69-- pg_stat_io test:
70-- verify_heapam always uses a BAS_BULKREAD BufferAccessStrategy, whereas a
71-- sequential scan does so only if the table is large enough when compared to
72-- shared buffers (see initscan()). CREATE DATABASE ... also unconditionally
73-- uses a BAS_BULKREAD strategy, but we have chosen to use a tablespace and
74-- verify_heapam to provide coverage instead of adding another expensive
75-- operation to the main regression test suite.
76--
77-- Create an alternative tablespace and move the heaptest table to it, causing
78-- it to be rewritten and all the blocks to reliably evicted from shared
79-- buffers -- guaranteeing actual reads when we next select from it in the
80-- same transaction.  The heaptest table is smaller than the default
81-- wal_skip_threshold, so a wal_level=minimal commit reads the table into
82-- shared_buffers.  A transaction delays that and excludes any autovacuum.
