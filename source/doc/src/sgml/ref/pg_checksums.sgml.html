36 <refsect1 id="r1-app-pgchecksums-1">
37  <title>Description</title>
38  <para>
39   <application>pg_checksums</application> checks, enables or disables data
40   checksums in a <productname>PostgreSQL</productname> cluster.  The server
41   must be shut down cleanly before running
42   <application>pg_checksums</application>. When verifying checksums, the exit
43   status is zero if there are no checksum errors, and nonzero if at least one
44   checksum failure is detected. When enabling or disabling checksums, the
45   exit status is nonzero if the operation failed.
46  </para>
47
48  <para>
49   When verifying checksums, every file in the cluster is scanned. When
50   enabling checksums, each relation file block with a changed checksum is
51   rewritten in-place.
52   Disabling checksums only updates the file <filename>pg_control</filename>.
53  </para>
54 </refsect1>

226 <refsect1>
227  <title>Notes</title>
228  <para>
229   Enabling checksums in a large cluster can potentially take a long time.
230   During this operation, the cluster or other programs that write to the
231   data directory must not be started or else data loss may occur.
232  </para>
233  <para>
234   When using a replication setup with tools which perform direct copies
235   of relation file blocks (for example <xref linkend="app-pgrewind"/>),
236   enabling or disabling checksums can lead to page corruptions in the
237   shape of incorrect checksums if the operation is not done consistently
238   across all nodes. When enabling or disabling checksums in a replication
239   setup, it is thus recommended to stop all the clusters before switching
240   them all consistently. Destroying all standbys, performing the operation
241   on the primary and finally recreating the standbys from scratch is also
242   safe.
243  </para>
244  <para>
245   If <application>pg_checksums</application> is aborted or killed while
246   enabling or disabling checksums, the cluster's data checksum configuration
247   remains unchanged, and <application>pg_checksums</application> can be
248   re-run to perform the same operation.
249  </para>
