31 <refsect1 id="app-pg-isready-description">
32  <title>Description</title>
33  <para>
34   <application>pg_isready</application> is a utility for checking the connection
35   status of a <productname>PostgreSQL</productname> database server. The exit
36   status specifies the result of the connection check.
37  </para>
38 </refsect1>

142 <refsect1>
143  <title>Exit Status</title>
144
145  <para>
146   <application>pg_isready</application> returns <literal>0</literal> to the shell if the server
147   is accepting connections normally, <literal>1</literal> if the server is rejecting
148   connections (for example during startup), <literal>2</literal> if there was no response to the
149   connection attempt, and <literal>3</literal> if no attempt was made (for example due to invalid
150   parameters).
151  </para>
152 </refsect1>

155  <title>Environment</title>
156
157  <para>
158   <command>pg_isready</command>, like most other <productname>PostgreSQL</productname>
159   utilities,
160   also uses the environment variables supported by <application>libpq</application>
161   (see <xref linkend="libpq-envars"/>).
162  </para>
163
164  <para>
165   The environment variable <envar>PG_COLOR</envar> specifies whether to use
166   color in diagnostic messages. Possible values are
167   <literal>always</literal>, <literal>auto</literal> and
168   <literal>never</literal>.
169  </para>

172 <refsect1 id="app-pg-isready-notes">
173  <title>Notes</title>
174
175  <para>
176   It is not necessary to supply correct user name, password, or database
177   name values to obtain the server status; however, if incorrect values
178   are provided, the server will log a failed connection attempt.
179  </para>
180 </refsect1>
