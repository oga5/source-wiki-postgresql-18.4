41 <refsect1>
42  <title>Description</title>
43
44  <para>
45   <command>ALTER OPERATOR</command> changes the definition of
46   an operator.
47  </para>
48
49  <para>
50   You must own the operator to use <command>ALTER OPERATOR</command>.
51   To alter the owner, you must be able to <literal>SET ROLE</literal> to the
52   new owning role, and that role must have <literal>CREATE</literal>
53   privilege on the operator's schema.
54   (These restrictions enforce that altering the owner
55   doesn't do anything you couldn't do by dropping and recreating the operator.
56   However, a superuser can alter ownership of any operator anyway.)
57  </para>
58 </refsect1>

171 <refsect1>
172  <title>Notes</title>
173
174  <para>
175   Refer to <xref linkend="xoper"/> and
176   <xref linkend="xoper-optimization"/> for further information.
177  </para>
178
179  <para>
180   Since commutators come in pairs that are commutators of each other,
181   <literal>ALTER OPERATOR SET COMMUTATOR</literal> will also set the
182   commutator of the <replaceable class="parameter">com_op</replaceable>
183   to be the target operator.  Likewise, <literal>ALTER OPERATOR SET
184   NEGATOR</literal> will also set the negator of
185   the <replaceable class="parameter">neg_op</replaceable> to be the
186   target operator.  Therefore, you must own the commutator or negator
187   operator as well as the target operator.
188  </para>
189 </refsect1>

191 <refsect1>
192  <title>Examples</title>
193
194  <para>
195   Change the owner of a custom operator <literal>a @@ b</literal> for type <type>text</type>:
196<programlisting>
197ALTER OPERATOR @@ (text, text) OWNER TO joe;
198</programlisting>
199  </para>
200
201  <para>
202   Change the restriction and join selectivity estimator functions of a
203   custom operator <literal>a &amp;&amp; b</literal> for
204   type <type>int[]</type>:
205<programlisting>
206ALTER OPERATOR &amp;&amp; (int[], int[]) SET (RESTRICT = _int_contsel, JOIN = _int_contjoinsel);
207</programlisting>
208  </para>
209
210  <para>
211   Mark the <literal>&amp;&amp;</literal> operator as being its own
212   commutator:
213<programlisting>
214ALTER OPERATOR &amp;&amp; (int[], int[]) SET (COMMUTATOR = &amp;&amp;);
215</programlisting>
216  </para>
217
218 </refsect1>
