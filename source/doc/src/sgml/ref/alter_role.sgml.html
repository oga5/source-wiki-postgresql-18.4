18  <refname>ALTER ROLE</refname>

22 <refsynopsisdiv>
23<synopsis>
24ALTER ROLE <replaceable class="parameter">role_specification</replaceable> [ WITH ] <replaceable class="parameter">option</replaceable> [ ... ]
25
26<phrase>where <replaceable class="parameter">option</replaceable> can be:</phrase>
27
28      SUPERUSER | NOSUPERUSER
29    | CREATEDB | NOCREATEDB
30    | CREATEROLE | NOCREATEROLE
31    | INHERIT | NOINHERIT
32    | LOGIN | NOLOGIN
33    | REPLICATION | NOREPLICATION
34    | BYPASSRLS | NOBYPASSRLS
35    | CONNECTION LIMIT <replaceable class="parameter">connlimit</replaceable>
36    | [ ENCRYPTED ] PASSWORD '<replaceable class="parameter">password</replaceable>' | PASSWORD NULL
37    | VALID UNTIL '<replaceable class="parameter">timestamp</replaceable>'
38
39ALTER ROLE <replaceable class="parameter">name</replaceable> RENAME TO <replaceable>new_name</replaceable>
40
41ALTER ROLE { <replaceable class="parameter">role_specification</replaceable> | ALL } [ IN DATABASE <replaceable class="parameter">database_name</replaceable> ] SET <replaceable>configuration_parameter</replaceable> { TO | = } { <replaceable>value</replaceable> | DEFAULT }
42ALTER ROLE { <replaceable class="parameter">role_specification</replaceable> | ALL } [ IN DATABASE <replaceable class="parameter">database_name</replaceable> ] SET <replaceable>configuration_parameter</replaceable> FROM CURRENT
43ALTER ROLE { <replaceable class="parameter">role_specification</replaceable> | ALL } [ IN DATABASE <replaceable class="parameter">database_name</replaceable> ] RESET <replaceable>configuration_parameter</replaceable>
44ALTER ROLE { <replaceable class="parameter">role_specification</replaceable> | ALL } [ IN DATABASE <replaceable class="parameter">database_name</replaceable> ] RESET ALL
45
46<phrase>where <replaceable class="parameter">role_specification</replaceable> can be:</phrase>
47
48    <replaceable class="parameter">role_name</replaceable>
49  | CURRENT_ROLE
50  | CURRENT_USER
51  | SESSION_USER
52</synopsis>

55 <refsect1 id="sql-alterrole-desc">
56  <title>Description</title>
57
58  <para>
59   <command>ALTER ROLE</command> changes the attributes of a
60   <productname>PostgreSQL</productname> role.
61  </para>
62
63  <para>
64   The first variant of this command listed in the synopsis can change
65   many of the role attributes that can be specified in
66   <link linkend="sql-createrole"><command>CREATE ROLE</command></link>.
67   (All the possible attributes are covered,
68   except that there are no options for adding or removing memberships; use
69   <link linkend="sql-grant"><command>GRANT</command></link> and
70   <link linkend="sql-revoke"><command>REVOKE</command></link> for that.)
71   Attributes not mentioned in the command retain their previous settings.
72   Database superusers can change any of these settings for any role, except
73   for changing the <literal>SUPERUSER</literal> property for the
74   <glossterm linkend="glossary-bootstrap-superuser">bootstrap superuser</glossterm>.
75   Non-superuser roles having <literal>CREATEROLE</literal> privilege can
76   change most of these properties, but only for non-superuser and
77   non-replication roles for which they have been granted
78   <literal>ADMIN OPTION</literal>. Non-superusers cannot change the
79   <literal>SUPERUSER</literal> property and can change the
80   <literal>CREATEDB</literal>, <literal>REPLICATION</literal>, and
81   <literal>BYPASSRLS</literal> properties only if they possess the
82   corresponding property themselves.
83   Ordinary roles can only change their own password.
84  </para>
85
86  <para>
87   The second variant changes the name of the role.
88   Database superusers can rename any role.
89   Roles having <literal>CREATEROLE</literal> privilege can rename non-superuser
90   roles for which they have been granted <literal>ADMIN OPTION</literal>.
91   The current session user cannot be renamed.
92   (Connect as a different user if you need to do that.)
93   Because <literal>MD5</literal>-encrypted passwords use the role name as
94   cryptographic salt, renaming a role clears its password if the
95   password is <literal>MD5</literal>-encrypted.
96  </para>
97
98  <para>
99   The remaining variants change a role's session default for a configuration
100   variable, either for all databases or, when the <literal>IN
101   DATABASE</literal> clause is specified, only for sessions in the named
102   database.  If <literal>ALL</literal> is specified instead of a role name,
103   this changes the setting for all roles.  Using <literal>ALL</literal>
104   with <literal>IN DATABASE</literal> is effectively the same as using the
105   command <literal>ALTER DATABASE ... SET ...</literal>.
106  </para>
107
108  <para>
109   Whenever the role subsequently
110   starts a new session, the specified value becomes the session
111   default, overriding whatever setting is present in
112   <filename>postgresql.conf</filename> or has been received from the <command>postgres</command>
113   command line. This only happens at login time; executing
114   <link linkend="sql-set-role"><command>SET ROLE</command></link> or
115   <link linkend="sql-set-session-authorization"><command>SET SESSION AUTHORIZATION</command></link> does not cause new
116   configuration values to be set.
117   Settings set for all databases are overridden by database-specific settings
118   attached to a role.  Settings for specific databases or specific roles override
119   settings for all roles.
120  </para>
121
122  <para>
123   Superusers can change anyone's session defaults. Roles having
124   <literal>CREATEROLE</literal> privilege can change defaults for non-superuser
125   roles for which they have been granted <literal>ADMIN OPTION</literal>.
126   Ordinary roles can only set defaults for themselves.
127   Certain configuration variables cannot be set this way, or can only be
128   set if a superuser issues the command.  Only superusers can change a setting
129   for all roles in all databases.
130  </para>
131 </refsect1>

247 <refsect1 id="sql-alterrole-notes">
248  <title>Notes</title>
249
250  <para>
251   Use <link linkend="sql-createrole"><command>CREATE ROLE</command></link>
252   to add new roles, and <link linkend="sql-droprole"><command>DROP ROLE</command></link> to remove a role.
253  </para>
254
255  <para>
256   <command>ALTER ROLE</command> cannot change a role's memberships.
257   Use <link linkend="sql-grant"><command>GRANT</command></link> and
258   <link linkend="sql-revoke"><command>REVOKE</command></link>
259   to do that.
260  </para>
261
262  <para>
263   Caution must be exercised when specifying an unencrypted password
264   with this command.  The password will be transmitted to the server
265   in cleartext, and it might also be logged in the client's command
266   history or the server log.  <xref linkend="app-psql"/>
267   contains a command
268   <command>\password</command> that can be used to change a
269   role's password without exposing the cleartext password.
270  </para>
271
272  <para>
273   It is also possible to tie a
274   session default to a specific database rather than to a role; see
275   <xref linkend="sql-alterdatabase"/>.
276   If there is a conflict, database-role-specific settings override role-specific
277   ones, which in turn override database-specific ones.
278  </para>

281 <refsect1 id="sql-alterrole-examples">
282  <title>Examples</title>
283
284  <para>
285   Change a role's password:
286
287<programlisting>
288ALTER ROLE davide WITH PASSWORD 'hu8jmn3';
289</programlisting>
290  </para>
291
292  <para>
293   Remove a role's password:
294
295<programlisting>
296ALTER ROLE davide WITH PASSWORD NULL;
297</programlisting>
298  </para>
299
300  <para>
301   Change a password expiration date, specifying that the password
302   should expire at midday on 4th May 2015 using
303   the time zone which is one hour ahead of <acronym>UTC</acronym>:
304<programlisting>
305ALTER ROLE chris VALID UNTIL 'May 4 12:00:00 2015 +1';
306</programlisting>
307  </para>
308
309  <para>
310   Make a password valid forever:
311<programlisting>
312ALTER ROLE fred VALID UNTIL 'infinity';
313</programlisting>
314  </para>
315
316  <para>
317   Give a role the ability to manage other roles and create new databases:
318
319<programlisting>
320ALTER ROLE miriam CREATEROLE CREATEDB;
321</programlisting>
322  </para>
323
324  <para>
325   Give a role a non-default setting of the
326   <xref linkend="guc-maintenance-work-mem"/> parameter:
327
328<programlisting>
329ALTER ROLE worker_bee SET maintenance_work_mem = 100000;
330</programlisting>
331  </para>
332
333  <para>
334   Give a role a non-default, database-specific setting of the
335   <xref linkend="guc-client-min-messages"/> parameter:
336
337<programlisting>
338ALTER ROLE fred IN DATABASE devel SET client_min_messages = DEBUG;
339</programlisting></para>
