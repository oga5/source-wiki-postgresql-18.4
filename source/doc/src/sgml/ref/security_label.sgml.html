57 <refsect1>
58  <title>Description</title>
59
60  <para>
61   <command>SECURITY LABEL</command> applies a security label to a database
62   object.  An arbitrary number of security labels, one per label provider, can
63   be associated with a given database object.  Label providers are loadable
64   modules which register themselves by using the function
65   <function>register_label_provider</function>.
66  </para>
67
68  <note>
69    <para>
70      <function>register_label_provider</function> is not an SQL function; it can
71      only be called from C code loaded into the backend.
72    </para>
73  </note>
74
75  <para>
76   The label provider determines whether a given label is valid and whether
77   it is permissible to assign that label to a given object.  The meaning of a
78   given label is likewise at the discretion of the label provider.
79   <productname>PostgreSQL</productname> places no restrictions on whether or how a
80   label provider must interpret security labels; it merely provides a
81   mechanism for storing them.  In practice, this facility is intended to allow
82   integration with label-based mandatory access control (MAC) systems such as
83   <productname>SELinux</productname>.  Such systems make all access control decisions
84   based on object labels, rather than traditional discretionary access control
85   (DAC) concepts such as users and groups.
86  </para>
87
88  <para>
89   You must own the database object to use <command>SECURITY LABEL</command>.
90  </para>

131      <literal>INOUT</literal>, or <literal>VARIADIC</literal>.
132      If omitted, the default is <literal>IN</literal>.
133      Note that <command>SECURITY LABEL</command> does not actually
134      pay any attention to <literal>OUT</literal> arguments, since only the input

204 <refsect1>
205  <title>Examples</title>
206
207  <para>
208   The following example shows how the security label of a table could
209   be set or changed:
210
211<programlisting>
212SECURITY LABEL FOR selinux ON TABLE mytable IS 'system_u:object_r:sepgsql_table_t:s0';
213</programlisting>
214
215   To remove the label:
216
217<programlisting>
218SECURITY LABEL FOR selinux ON TABLE mytable IS NULL;
219</programlisting>
220  </para>
221 </refsect1>
