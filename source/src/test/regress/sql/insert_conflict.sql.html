37-- specified
38explain (costs off) insert into insertconflicttest values(0, 'Crowberry') on conflict (fruit collate "C" text_pattern_ops, key) do nothing;
39-- Okay, but only accepts the single index where both opclass and collation are
40-- specified (plus expression variant)
41explain (costs off) insert into insertconflicttest values(0, 'Crowberry') on conflict (lower(fruit) collate "C", key, key) do nothing;
42-- Attribute appears twice, while not all attributes/expressions on attributes
43-- appearing within index definition match in terms of both opclass and
44-- collation.
45--
46-- Works because every attribute in inference specification needs to be
47-- satisfied once or more by cataloged index attribute, and as always when an
48-- attribute in the cataloged definition has a non-default opclass/collation,
49-- it still satisfied some inference attribute lacking any particular
50-- opclass/collation specification.
51--
52-- The implementation is liberal in accepting inference specifications on the
53-- assumption that multiple inferred unique indexes will prevent problematic
54-- cases.  It rolls with unique indexes where attributes redundantly appear
55-- multiple times, too (which is not tested here).
56explain (costs off) insert into insertconflicttest values(0, 'Crowberry') on conflict (fruit, key, fruit text_pattern_ops, key) do nothing;
57explain (costs off) insert into insertconflicttest values(0, 'Crowberry') on conflict (lower(fruit) collate "C" text_pattern_ops, key, key) do nothing;
