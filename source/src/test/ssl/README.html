3SSL regression tests
4====================
5
6This directory contains a test suite for SSL support. It tests both
7client-side functionality, i.e. verifying server certificates, and
8server-side functionality, i.e. certificate authorization.
9
10CAUTION: The test server run by this test is configured to listen for
11TCP connections on localhost. Any user on the same host is able to
12log in to the test server while the tests are running. Do not run this
13suite on a multi-user system where you don't trust all local users!
14
15Running the tests
16=================
17
18NOTE: You must have given the --enable-tap-tests argument to configure.
19Also, to use "make installcheck", you must have built and installed
20contrib/sslinfo in addition to the core code.
21
22Run
23    make check PG_TEST_EXTRA=ssl
24or
25    make installcheck PG_TEST_EXTRA=ssl
26You can use "make installcheck" if you previously did "make install".
27In that case, the code in the installation tree is tested.  With
28"make check", a temporary installation tree is built from the current
29sources and then tested.
30
31Either way, this test initializes, starts, and stops a test Postgres
32cluster that is accessible to other local users!
33
34See src/test/perl/README for more info about running these tests.
35
36Certificates
37============
38
39The test suite needs a set of public/private key pairs and certificates to
40run:
41
42root_ca
43	root CA, use to sign the server and client CA certificates.
44
45server_ca
46	CA used to sign server certificates.
47
48client_ca
49	CA used to sign client certificates.
50
51server-cn-only
52server-cn-and-alt-names
53server-single-alt-name
54server-multiple-alt-names
55server-no-names
56	server certificates, with small variations in the hostnames present
57        in the certificate. Signed by server_ca.
58
59server-password
60	same as server-cn-only, but password-protected.
61
62client
63	a client certificate, for user "ssltestuser". Signed by client_ca.
64
65client-revoked
66	like "client", but marked as revoked in the client CA's CRL.
67
68In addition, there are a few files that combine various certificates together
69in the same file:
70
71both-cas-1
72	Contains root_ca.crt, client_ca.crt and server_ca.crt, in that order.
73
74both-cas-2
75	Contains root_ca.crt, server_ca.crt and client_ca.crt, in that order.
76
77root+server_ca
78	Contains root_crt and server_ca.crt. For use as client's "sslrootcert"
79	option.
80
81root+client_ca
82	Contains root_crt and client_ca.crt. For use as server's "ssl_ca_file".
83
84client+client_ca
85	Contains client.crt and client_ca.crt in that order. For use as client's
86	certificate chain.
87
88There are also CRLs for each of the CAs: root.crl, server.crl and client.crl.
89
90For convenience, all of these keypairs and certificates are included in the
91ssl/ subdirectory. The Makefile also contains a rule, "make sslfiles", to
92recreate them if you need to make changes. "make sslfiles-clean" is required
93in order to recreate the full set of keypairs and certificates. To rebuild
94separate files, touch (or remove) the files in question and run "make sslfiles".
95This step requires at least OpenSSL 1.1.1.
96
97Note
98====
99
100These certificates are also used in other tests, e.g. the LDAP tests.
