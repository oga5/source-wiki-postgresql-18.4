src/common/unicode/README
1This directory contains tools to download new Unicode data files and
2generate static tables. These tables are used to normalize or
3determine various properties of Unicode data.
4
5The generated header files are copied to src/include/common/, and
6included in the source tree, so these tools are not normally required
7to build PostgreSQL.
8
9Update Unicode Version
10----------------------
11
12Edit src/Makefile.global.in and src/common/unicode/meson.build
13to update the UNICODE_VERSION.
14
15Then, generate the new header files with:
16
17    make update-unicode
18
19or if using meson:
20
21    ninja update-unicode
22
23from the top level of the source tree. Examine the result to make sure
24the changes look reasonable (that is, that the diff size and scope is
25comparable to the Unicode changes since the last update), and then
26commit it.
27
28Tests
29-----
30
31Normalization tests:
32
33The Unicode consortium publishes a comprehensive test suite for the
34normalization algorithm, in a file called NormalizationTest.txt. This
35directory also contains a perl script and some C code, to run our
36normalization code with all the test strings in NormalizationTest.txt.
37To download NormalizationTest.txt and run the tests:
38
39    make normalization-check
40
41This is also run as part of the update-unicode target.
42
43Category, Property and Case tests:
44
45The files case_test.c and category_test.c test Unicode categories,
46properties, and case mapping by exhaustively comparing results with
47ICU. For these tests to be effective, the version of the Unicode data
48files must be similar to the version of Unicode on which ICU is
49based. Mismatched Unicode versions will cause the tests to skip over
50codepoints that are assigned in one version and not the other, and may
51falsely report failures. This test is run as a part of the
52update-unicode target.
