src/backend/lib/README
1This directory contains a general purpose data structures, for use anywhere
2in the backend:
3
4bipartite_match.c - Hopcroft-Karp maximum cardinality algorithm for bipartite graphs
5
6bloomfilter.c - probabilistic, space-efficient set membership testing
7
8dshash.c - concurrent hash tables backed by dynamic shared memory areas
9
10hyperloglog.c - a streaming cardinality estimator
11
12ilist.c - single and double-linked lists
13
14integerset.c - a data structure for holding large set of integers
15
16knapsack.c - knapsack problem solver
17
18pairingheap.c - a pairing heap
19
20rbtree.c - a red-black tree
21
22
23Aside from the inherent characteristics of the data structures, there are a
24few practical differences between the binary heap and the pairing heap. The
25binary heap is fully allocated at creation, and cannot be expanded beyond the
26allocated size. The pairing heap on the other hand has no inherent maximum
27size, but the caller needs to allocate each element being stored in the heap,
28while the binary heap works with plain Datums or pointers.
29
30The linked-lists in ilist.c can be embedded directly into other structs, as
31opposed to the List interface in nodes/pg_list.h.
