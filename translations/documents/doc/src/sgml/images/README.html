画像
======

このディレクトリには、ドキュメントで使用する画像が格納されています。

画像の作成
----------

画像の作成にはさまざまなツールを使用できます。適切なツールの選択は、画像の種類によって異なります。私たちは、差分比較可能なソースファイルを含むワークフローを推奨しています。

以下のツールが使用可能です。

- Graphviz (https://graphviz.org/)
- Ditaa (http://ditaa.sourceforge.net/)

ドキュメントの最終的な出力形式（HTML、PDFなど）に画像を統合するための形式として、SVGを使用しています。したがって、使用するツールはSVGを出力できる必要があります。

このディレクトリには、一般的なスタイルを使用して一般的な入力形式からSVGをビルドするためのmakefileルールが含まれています。

fixup-svg.xslは、それらの外部ツールによって生成されたSVGファイルに対して、さまざまな問題を解決するための後処理を適用します。その中のコメントを参照し、必要に応じて調整や拡張を行ってください。

ソースファイルとSVG出力ファイルの両方がこのディレクトリにコミットされます。これにより、すべての開発者がすべてのツールをインストールしておく必要がなくなります。特定のツールによってSVG出力に不要な差分が生じる可能性があることは許容しますが、それに注意を払い、最小限に抑えるようにしてください。

DocBookでの画像の使用
---------------------

DocBookで画像を使用する例を以下に示します。

    <figure id="gin-internals-figure">
     <title>GIN Internals</title>
     <mediaobject>
      <imageobject>
       <imagedata fileref="images/gin.svg" format="SVG" width="100%"/>
      </imageobject>
     </mediaobject>
    </figure>

注意点:

- 実際の処理は <mediaobject> 要素で行われますが、通常は <figure> で囲み、本文中のどこかにその図への <xref> を配置する必要があります。リンクや説明なしに、ドキュメントに画像を配置しないでください。

- 異なる出力形式であっても、<imagedata> 要素は1つだけで済むように設定されています。

- format="SVG" 属性は必須です。これを省略しても動作するように見える場合がありますが、画像を明示的にSVGとして宣言した方が、スタイルシートはより適切に処理を行います。

- widthには何らかの値を設定する必要があります。これにより、PDF出力時に画像がページに合わせて拡大縮小されるようになります。（100%以外の幅が適切な場合もあります。）
