パーサー
======

このディレクトリは、SQLクエリのトークン化と解析以上のことを行います。また、オプティマイザおよびエグゼキュータに渡される様々な複雑なクエリのためのQuery構造体を作成します。

parser.c	ここから開始
scan.l		クエリをトークンに分割
scansup.c	入力文字列内のエスケープを処理
gram.y		トークンを解析し、「raw」なパースツリーを生成
analyze.c	最適化可能なクエリに対するパース解析の最上位レベル
parse_agg.c	SUM(col1)、AVG(col2)などの集約関数を処理
parse_clause.c	WHERE、ORDER BY、GROUP BYなどの句を処理
parse_coerce.c	式を異なるデータ型へ強制変換（キャスト）することを処理
parse_collate.c	完了した式に照合順序（collation）情報を割り当て
parse_cte.c	共通テーブル式（WITH句）を処理
parse_expr.c	col、col + 3、x = 3 or x = 4などの式を処理
parse_enr.c	一時的な名前付きリレーション（トリガー遷移テーブルなど）を処理
parse_func.c	関数、table.column、および列識別子を処理
parse_jsontable.c JSON_TABLEを処理
parse_merge.c	MERGEを処理
parse_node.c	様々な構造のためのノードを作成
parse_oper.c	式内の演算子を処理
parse_param.c	Paramsを処理（コアバックエンドで使用されるケース用）
parse_relation.c テーブルおよび列の処理のためのサポートルーチン
parse_target.c	クエリの結果リストを処理
parse_type.c	データ型処理のためのサポートルーチン
parse_utilcmd.c	ユーティリティコマンドのパース解析（実行時に行われる）

標準キーワードのテーブルとキーワード検索関数を含む src/common/keywords.c も参照してください。様々なフロントエンドコードでもそれを使用できるように分離されています。
