Snowball ベースのステミング
=======================

このモジュールは、Snowball プロジェクト (http://snowballstem.org、旧 http://snowball.tartarus.org) によって開発された単語ステミングコードを使用しています。このコードは、BSD スタイルのライセンスの下で公開されています。

Snowball プロジェクトは正式なリリースを頻繁に行わないため、git リポジトリから取得するのが最適です。

git clone https://github.com/snowballstem/snowball.git

派生ファイルのビルドは、以下のように簡単に行えます。

cd snowball
make

少なくとも Linux 上では、プラットフォーム固有の調整は不要です。

src/backend/snowball/libstemmer/ および src/include/snowball/libstemmer/ にある Postgres のファイルは、Snowball のファイルから直接取得したものであり、ファイルインクルードに関するわずかな調整のみが行われています。これらのファイルのほとんどは実際には派生ファイルであり、オリジナルのソースではないことに注意してください。オリジナルのソースは Snowball 言語で記述されており、Snowball プロジェクトの一部である Snowball-to-C コンパイラを使用してビルドされます。PostgreSQL の配布物に派生ファイルを含めているのは、ほとんどのインストール環境で Snowball コンパイラが利用できないためです。

現在、2025年2月19日時点の Snowball git コミット d19326ac6c1b9a417fc872f7c2f845265a5e9ece と同期しています。

新しい Snowball バージョンから PostgreSQL ソースを更新する手順:

0. まだ実行していない場合は、"make -C snowball" を実行します。

1. snowball/src_c/ にある *.c ファイルを src/backend/snowball/libstemmer にコピーし、"../runtime/header.h" を "header.h" に置換します。例:

for f in .../snowball/src_c/*.c
do
    sed 's|\.\./runtime/header\.h|header.h|' $f >libstemmer/`basename $f`
done

libstemmer/modules.txt で非標準 (nonstandard) とされているステマー ("kraaij_pohlmann" や "lovins" など) はコピーしないでください。

2. snowball/runtime/ にある *.c ファイルを src/backend/snowball/libstemmer にコピーし、<stdio.h> などのシステムヘッダーの直接インクルードを削除するように編集します。これらは "header.h" のみをインクルードする必要があります。(この削除は、largefile コンパイルオプションの影響を受ける一部のプラットフォームでの移植性の問題を回避するためです。)

3. snowball/src_c/ および snowball/runtime/ にある *.h ファイルを src/include/snowball/libstemmer にコピーします。執筆時点では、ヘッダーファイルに変更を加える必要はありません。ここでも、非標準ステマーの *.h ファイルは除外してください。

4. ステマーモジュールが追加または削除されていないか確認します。変更がある場合は、Makefile 内の OBJS リスト、meson.build 内の dict_snowball_sources リスト、dict_snowball.c 内の #include のリストと stemmer_modules[] テーブル、および textsearch.sgml 内のドキュメントにあるサンプル \dFd 出力を編集します。また、snowball_create.pl 内の @languages 配列と initdb.c 内の tsearch_config_languages[] テーブルの変更が必要になる場合もあります。

5. stopwords/ にある各種ストップワードファイルは、snowballstem.org ウェブサイトの各ページから個別にダウンロードする必要があります。これらのファイルは UTF-8 エンコーディングで保存する必要があることに注意してください。ファイルが追加または削除された場合は、Makefile 内の stop_files リストを更新してください (ただし、meson ツール側での調整は不要です)。
