#
# errcodes.txt
#      PostgreSQL エラーコード
#
# Copyright (c) 2003-2025, PostgreSQL Global Development Group
#
# このリストは、エラーコードを含むソースファイルを生成するための基礎となります。
# すべてのソースファイルが同じ内容を持つことを保証するため、共通の形式で保持されています。
# このファイルから生成されるファイルは以下の通りです。
#
#   src/include/utils/errcodes.h
#      ソースコードの他の部分で使用されるエラーコード定数を定義するマクロ
#
#   src/pl/plpgsql/src/plerrcodes.h
#      PL/pgSQL 条件名とそれに対応する SQLSTATE コードのリスト
#
#   src/pl/tcl/pltclerrcodes.h
#      PL/Tcl 用の同リスト
#
#   doc/src/sgml/errcodes-table.sgml
#      ドキュメントに含めるためのエラーコードの SGML テーブル
#
# このファイルの形式は、1行につき1つのエラーコードで、以下の空白区切りのフィールドで構成されます。
#
#      sqlstate    E/W/S    errcode_macro_name    spec_name
#
# sqlstate は SQLSTATE 規則に従った5文字の文字列です。
# 2番目のフィールドは、そのコードがエラー、警告、成功のいずれを意味するかを示します。
# errcode_macro_name は errcodes.h に記述される ERRCODE で始まる C マクロ名です。
# spec_name は小文字でアンダースコア区切りの名前で、PL/pgSQL の条件名として使用され、SGML リストにも含まれます。
# 最後のフィールドはオプションであり、存在しない場合は PL/pgSQL 条件および SGML エントリは生成されません。
#
# 空行およびハッシュ記号で始まる行はコメントです。
#
# また、以下の形式の特別な行もあります。
#
#      Section: セクションの説明
#
# つまり、"Section:" という文字列で始まる行です。これらは SQL 仕様で定義されているエラークラスを区切るために使用され、SGML 出力に必要です。
#
#
#      エラー用 SQLSTATE コード
#
# SQL99 のコードセットは、特に構文エラーや意味エラーの領域においてかなり乏しいものです。
# 私たちは IBM の DB2 からコードを借用し、有用なコードセットを開発するために独自のコードを考案しました。
#
# 新しいコードを追加する際は、最も適切なクラス（コード値の最初の2文字がクラスを識別します）に配置されるようにしてください。
# リストは、これを明確にするためにクラスごとに整理されています。
#
# 各クラスには汎用的な '000' サブクラスが必要です。ただし、汎用的な '000' サブクラスコードは、
# より具体的なサブクラスコードが定義されていない場合にのみエラーに使用されるべきです。
#
# SQL 仕様では、SQLSTATE コードのすべての要素は数字または大文字の ASCII 文字である必要があります。
#
# 0-4 または A-H で始まるクラスは標準によって定義されています。そのようなクラス内では、
# 標準で定義されたサブクラス値は 0-4 または A-H で始まる必要があります。
# 新しいエラーコードを定義する場合は、「実装定義クラス」（5-9 または I-Z で始まる）に含めるか、
# そのサブクラスが将来の標準バージョンで存在する可能性のあるエラーコードの範囲外（つまり、サブクラス値が 5-9 または I-Z で始まる）
# になるようにしてください。
#
# PostgreSQL によって定義される新しいエラーコードは、標準で定義されたクラス内では 'P' で始まるサブクラス値を持つという慣習があります。
# さらに、PostgreSQL クライアント（ecpg など）によって定義されるエラーコードは、'Y' で始まるクラス値を持っています。
