実装に関するメモと未解決の課題です...

まず、実装では相関Var（変数）を扱うために、新しいタイプのパラメータである PARAM_EXEC を使用します。query_planner() が呼び出されると、まず現在のクエリで参照されている上位クエリのすべての Var を、このタイプの Param に置き換えようとします。Var から Param へのマッピング、および Param から Var へのマッピングを保持するために、いくつかのグローバル変数が使用されます。

これに続いて、現在のクエリのすべての SubLink が処理されます。クエリの qual に見つかった各 SubLink に対して、対応するサブセレクトを計画するために union_planner()（旧 planner() 関数）が呼び出されます（union_planner() は「単純な」クエリに対して query_planner() を呼び出し、UNION をサポートします）。サブセレクトの計画が完了すると、オプティマイザは、それが相関クエリなのか、非相関クエリなのか、あるいは「間接的」相関クエリ（親の Var は参照しないが、祖父母の Var を参照している：親の観点からは非相関）なのかを把握します。

EXPRession 型または EXISTS 型の非相関サブクエリおよび間接的相関サブクエリについては、SubLink は SubLink->Oper リストからの「通常の」句に置き換えられます（このリストは単なる Oper ノードではなく、EXPR ノードのリストになるよう変更しました）。これらのノードの右辺は PARAM_EXEC パラメータに置き換えられます。これが新しいパラメータタイプの2つ目の用途です。実行時に、これらのパラメータはサブクエリの評価結果（つまり、サブクエリのターゲットリスト）から値を取得します。サブクエリ自体の実行計画は、親クエリの init plan になります。InitPlan はどのパラメータがサブクエリの結果から値を取得すべきかを把握しており、「オンデマンド」で実行されます（例：select * from table where x > 0 and y > (select max(a) from table_a) のようなクエリにおいて、x > 0 を満たすタプルが存在せず、かつ y がインデックススキャンで使用されない場合、サブクエリは全く実行されません）。

他のすべての型のサブクエリに対する SubLink は、新しいタイプの Expr ノードである SUBPLAN_EXPR に変換されます。Expr->args は「親」クエリからの相関変数です。Expr->oper は新しい SubPlan ノードです。

このノードは InitPlan にも使用されます。これには、サブクエリの範囲テーブル、親クエリの Var（つまり Expr->args）から値を取得する Param のインデックス、サブクエリの結果を代入する Param のインデックス（これは InitPlan 用）、SubLink、およびサブクエリの実行計画が保持されます。

Plan ノードは、親クエリや InitPlan からの Param への依存関係を把握し、変更された Param のリスト（上記のもの）を保持するように変更されました。これにより、リストが NULL でない場合は再スキャンが可能になります。また、InitPlans のリスト（実際には、現在のクエリに対するすべての InitPlan が最上位の Plan ノードに格納されています）やその他の SubPlans（plan->qual からのもの）のリストも追加され、それらを初期化し、変更された Param（それらの「関心」リストから）について通知できるようにしました。

すべての SubLink が処理された後、query_planner() は qual の正規化関数を呼び出し、「通常」の処理を行います。Param を使用することで、オプティマイザはほとんど変更されずに済みます。

さて、エグゼキュータについてです。ExecutorStart() や ExecutorEnd() を行わずに（リレーションやインデックスのオープン/クローズを行わず、多くの palloc() や pfree() を行わずに — これは SQL 関数が呼び出しのたびに行っていることです）サブプランを再評価するために、ExecReScan() は現在、ほとんどの Plan 型をサポートしています...

EXPLAIN の説明。

vac=> explain select * from tmp where x >= (select max(x2) from test2
where y2 = y and exists (select * from tempx where tx = x));

Seq Scan on tmp  (cost=40.03 size=101 width=8)
  SubPlan
  ^^^^^^^ サブクエリは Seq Scan の qual 内にあり、その計画は以下にあります
    ->  Aggregate  (cost=2.05 size=0 width=0)
          InitPlan
          ^^^^^^^^ EXISTS サブサブクエリはサブクエリの InitPlan です
            ->  Seq Scan on tempx  (cost=4.33 size=1 width=4)

                ->  Index Scan on test2  (cost=2.05 size=1 width=4)

いくつかのテスト結果。TMP は x, y (int4) を持つテーブルで、x は 0-9、y = 100 - x、1000 タプル（各タプルが10個の重複）。TEST2 は x2, y2 (int4) を持つテーブルで、x2 は 1-99、y2 = 100 - x2、10000 タプル（100個の重複）。

   試行

select * from tmp where x >= (select max(x2) from test2 where y2 = y);

   および

begin;
select y as ty, max(x2) as mx into table tsub from test2, tmp
where y2 = y group by ty;
vacuum tsub;
select x, y from tmp, tsub where x >= mx and y = ty;
drop table tsub;
end;

   test2(y2) にインデックスがない場合：

SubSelect         -> 320 秒
一時テーブル使用  -> 32 秒

   インデックスがある場合

SubSelect         -> 17 秒 (2MB のメモリ)
一時テーブル使用  -> 32 秒 (12MB のメモリ: -S 8192)
