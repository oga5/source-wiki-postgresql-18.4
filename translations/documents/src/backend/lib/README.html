src/backend/lib/README

このディレクトリには、バックエンドのどこからでも使用できる汎用データ構造が含まれています。

bipartite_match.c - 二部グラフのためのHopcroft-Karp最大濃度アルゴリズム

bloomfilter.c - 確率的で空間効率の良い集合メンバーシップテスト

dshash.c - 動的共有メモリ領域によってバックアップされた並行ハッシュテーブル

hyperloglog.c - ストリーミング濃度推定器

ilist.c - 単方向および双方向連結リスト

integerset.c - 整数の大きな集合を保持するためのデータ構造

knapsack.c - ナップサック問題ソルバー

pairingheap.c - ペアリングヒープ

rbtree.c - 赤黒木

データ構造固有の特性とは別に、バイナリヒープとペアリングヒープの間にはいくつかの実用的な違いがあります。バイナリヒープは作成時に完全に割り当てられ、割り当てられたサイズを超えて拡張することはできません。一方、ペアリングヒープには固有の最大サイズはありませんが、呼び出し元がヒープに格納される各要素を割り当てる必要があります。これに対し、バイナリヒープは単純なDatumまたはポインタで動作します。

ilist.cの連結リストは、nodes/pg_list.hのListインターフェースとは異なり、他の構造体に直接埋め込むことができます。
