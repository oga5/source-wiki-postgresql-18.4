tznames
=======

このディレクトリには、PostgreSQL用のタイムゾーンセットを含むファイルが格納されています。問題は、タイムゾーンの省略形が世界中で一意ではないことであり、`Default' セット内のタイムゾーン省略形が、使用したいものと衝突する可能性があります。これは、目的の省略形を定義するタイムゾーンセットを選択することで解決できます。すでにニーズを満たすファイルがここにあるかもしれません。もしなければ、独自に作成することも可能です。

これらのファイルの一つを使用するには、パラメータを設定する通常の方法のいずれかで、

   timezone_abbreviations = 'xyz'

を設定する必要があります。ここで xyz は、目的のタイムゾーン省略形を含むファイル名です。

もし、あなたの地理的な場所に適した省略形セットがここに提供されていない場合は、<pgsql-hackers@lists.postgresql.org> まで報告してください。あなたのタイムゾーン省略形セットは、将来のリリースに含まれる可能性があります。当面の間は、いつでも独自のセットを追加することができます。

通常、カスタム省略形セットは、`Default' セットをインクルードし、必要に応じて省略形を追加または上書きすることで作成されます。例については、`Australia' および `India' ファイルを参照してください。

Africa.txt などの名前のファイルは、タイムゾーン省略形ファイルとして直接使用することを意図したものではありません。これらには、必要に応じてカスタム省略形ファイルにコピーできる、タイムゾーン省略形の参照定義が含まれています。これらのファイルには、2010年頃の IANA タイムゾーンデータベースに示されていたタイムゾーン省略形のほとんどが含まれています。

しかし、これらの省略形の多くは単に IANA タイムゾーングループによって考案されたものであり、実社会では通用しないことが判明しています。IANA グループはその方針を変更し、現在では、実社会で広く知られた省略形がない場合は、数値による UTC オフセットを使用することを推奨しています。そのため、これらの省略形の多くは IANA データには現れなくなっており、これらのデータファイルでは「obsolete（廃止）」とマークされています。
