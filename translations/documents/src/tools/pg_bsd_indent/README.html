src/tools/pg_bsd_indent/README

これは、FreeBSDプロジェクトによって保守されている "indent" プログラムをわずかに修正したバージョンです。修正の主な目的は、非BSD系プラットフォームへの移植性を高めることですが、アップストリーム（本家）に採用してもらえなかったフォーマット用スイッチが1つ含まれています。

ビルドするには、周囲のPostgresソースツリーを構成（configure）してから、このディレクトリで "make" を実行してください。
オプションとして、"make test" を実行すると簡単なテストが行えます。

使用する前に、pg_bsd_indentをPATHの通った場所にインストールする必要があります。開発者であれば、周囲のPostgresビルドがインストールされる場所と同じ場所に置くことは避けるのが賢明でしょう。そのため、以下のようにインストールを実行してください。

	make install prefix=/usr/local

Mesonを使用してビルドする場合、標準のビルドターゲットでpg_bsd_indentのビルドとテストが行われますが、現時点ではどこかにインストールするための仕組みはありません。ビルドされた実行ファイル（build/src/tools/pg_bsd_indent/pg_bsd_indent）を、任意の場所に手動でコピーしてください。

もしindentのソースコードをハック（改造）する場合、既存のインデントスタイルに最も近い近似値は以下の通りです。

	./pg_bsd_indent -i4 -l79 -di12 -nfc1 -nlp -sac somefile.c

ただし、これが厳密に守られているわけではありません。
（「靴屋の子供は靴を履いていない」という諺がありましたね？）
Postgresスタイルに再インデントする予定はありません。そうするとFreeBSDのソースとの比較が困難になるためです。

これはC言語のインデントツールであり、元々はPDP-11 Unix用の配布テープを通じてイリノイ大学から提供されたものです。その後、CMUのJames Goslingによってハックされました。あまり美しいコードとは言えず、全面的に作り直す必要がありますが、おそらく現存するC言語のプリティプリンタとしては最も優れたものでしょう。

「カーネル標準形式（Kernel Normal Form）」を提供するための追加機能は、Sun Microsystemsのスタッフによって寄稿されました。

indentに関する回答をありがとうございます。私の当初の観察（あるいは誤った観察 :-）は間違っていました。UCBはイリノイ大学の著作権表示を保持していました。

問題は、私たちがindentを配布できるかどうか、そして配布できるならどのバージョンかということです。David Willcox（作者）は次のように述べています：

| indentがパブリックドメインであると私が主張する根拠について、何人かから尋ねられました。あの投稿をしたことを後悔することになるとは思っていました。
| 
| 少し歴史を話しましょう。1976年、私がイリノイ大学高度計算センターで働いていたプロジェクトで、Cコードのフォーマット方法を巡って大きな論争がありました。約1週間の争いの後、私は嫌気がさして、Cコードを再フォーマットするindentというプログラムを書きました。これには、好みの形式で出力できるようにするためのさまざまなオプションがありました。特に、当時支持されていたあらゆるフォーマットをサポートしていました。
| 
| それが私の最初の大きなCプログラムでした。ひどい出来でした。設計されたものではなく、なんとなく成長したようなものです。しかし、ほぼ期待通りに動作し、争いのほとんどを終息させました。
| 
| 形式的なものとして、私はイリノイ大学の著作権表示を含めました。しかし、私の理解では、その作業はARPAの契約下で行われたため、パブリックドメインであると考えていました。
| 
| 時は流れ、数年後、indentは初期のemacs配布物の一つに含まれていました。
| 
| さらに後になって、UC Berkeleyの誰かがイリノイ大学に電話をかけ、indentがパブリックドメインかどうかを尋ねました。彼らはemacs関連のものと一緒に、自分たちのUNIX配布物にそれを含めたかったのです。私はすでにイリノイ大学にはいませんでしたが、そこにいたRob Kolstadが私に尋ねてきました。私は彼らが使っても構わないと答えました。それ以来、それはBSD配布物に含まれるようになりました。
| 
| その過程で、他にも名前のない数人がこれに関わりました。バージョン7のCを理解できるように変換されました（オリジナルはバージョン6でした）。元のフィルタインターフェースから、現在の「ユーザーのファイルを上書きする」インターフェースに変更されました。$HOME/.indent.proファイルの解析機能が追加されました。さらにいくつかのフォーマットオプションも追加されました。
| 
| 私が現在持っているソースには2つの著作権表示があります。一つはイリノイ大学のオリジナル、もう一つはバークレーのものです。
| 
| 私は弁護士ではありませんし、著作権法を理解しているわけでもありません。私に関する限り、このプログラムの大部分、つまりイリノイ大学の著作権でカバーされているすべてはパブリックドメインであり、それだけの価値があるものです。バークレーの著作権はおそらく彼らの変更点のみをカバーすべきであり、それを配布することについて彼らがどう感じているかは分かりません。

いずれにせよ、イリノイ大学にはその著作権を明確化・変更する人はいないようですが、私は（作者の声明に基づき）その著作権のままのコードは配布可能であり、法的な問題を引き起こすことはないと確信しています。

したがって、問題はcomp.sources.unixを通じて「どの」バージョンを配布するかという点に絞られます。皆さんの許可が得られるのであれば（一部に著作権をお持ちですので）、最も最新と思われるindentの4.3バージョンを配布することを提案します。私は、作者からunixテープへの最初の投稿、その後のG-Emacsバージョン、4.nバージョン、sunバージョン、Unipressバージョンなど、知られているほぼすべてのバージョンのindentを持っています。皆さんがこれまでに行ってきたすべての作業を「過去に戻って」やり直す必要はないと私は考えています。

これについてどう思われるか、お返事をお待ちしています。もちろん、4.3バージョンを直接モデレーターに送っていただいても構いませんし、許可をいただければ私がソースを送ることもできます。あるいは4.3バージョンが使用不可であると教えていただければ、その場合は古いバージョンに戻さざるを得ないでしょう。いずれにせよ、comp.sources.unixにindentのバージョンを届けたいと思っています。

よろしくお願いします。	oz
