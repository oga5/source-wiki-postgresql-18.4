src/tools/pginclude/README

このディレクトリには、PostgreSQLソースコード内のヘッダーファイルとインクルードを管理するためのスクリプトと情報が含まれています。


include-what-you-use
====================

include-what-you-use (IWYU) (<https://include-what-you-use.org/>) は、Cソースファイル内の不足しているインクルードや不要なインクルードを見つけるためのツールです。

コンパイルデータベース（mesonによって生成される compile_commands.json）があれば、ソースツリー全体に対して次のように実行できます。

    iwyu_tool.py -p build .

（これは非常に多くの出力が出る可能性があります）、または個別のファイルに対しては次のように実行します。

    iwyu_tool.py -p build src/bin/psql/startup.c

その他、さまざまな実行オプションが利用可能です。

少なくともバージョン0.23を使用することを推奨します。それより前のバージョンでは、コンパイラの警告オプション -Wmissing-variable-declarations と互換性のないアドバイスが出力されます。

clangd（言語サーバー）は、IWYU形式のアドバイスを自動的に提供できます。詳細は <https://clangd.llvm.org/guides/include-cleaner> を参照してください。

ソースコードには、PostgreSQLのインクルードファイルの慣習（特定のソースファイルで使用されるシンボルに寄与していなくても、"postgres.h" のようなヘッダーは常にインクルードされるべきである、など）をIWYUに伝えたり、修正が困難な警告を抑制したりするために、いくつかの "IWYU pragma" コメントが含まれています。それらに関するドキュメントは <https://github.com/include-what-you-use/include-what-you-use/blob/master/docs/IWYUPragmas.md> を参照してください。

もちろん、このツールや他のツールによって提案されたインクルードの変更は、慎重に確認および検証する必要があります。一部のインクルードは特定のプラットフォームや特定のコンパイルオプションでのみ使用されるため、生成されたアドバイスを盲目的に従うことは推奨されません。


headerscheck
============

このスクリプトを実行すると、すべてのPostgresインクルードファイルが「スタンドアロン」でコンパイルできる、つまり postgres.h（または必要に応じて postgres_fe.h や c.h）以外の前提ヘッダーを必要としないというプロジェクトの慣習を満たしているかを確認できます。

少数のヘッダーファイルはこの要件から除外されており、headerscheckスクリプトではスキップされます。

このスクリプトを実行する簡単な方法は、ビルド完了後にトップレベルのビルドディレクトリで "make -s headerscheck" と入力することです。configureオプションに "--with-perl --with-python" を含めておく必要があります。そうしないと、関連するヘッダーが見つからないというエラーが発生する可能性が高いです。

現在のスクリプトの制限として、どのヘッダーがフロントエンド用でどれがバックエンド用かを正確に把握していないことが挙げられます。判断に迷う場合は、postgres_fe.h や c.h がより適切であっても、前提条件として postgres.h を使用します。また、マクロの内容はチェックされませんが、これは意図的なものです。


headerscheck --cplusplus
========================

--cplusplus モードの headerscheck を実行すると、すべてのPostgresインクルードファイルがC++コードとしてコンパイルできるというプロジェクトの慣習を満たしているかを確認できます。プロジェクトのコーディング言語はCですが、C++で拡張機能を記述する人もいるため、インクルードファイルがC++でクリーンであることは有益です。

少数のヘッダーファイルはこの要件から除外されており、--cplusplus モードのスクリプトではスキップされます。

このスクリプトを実行する簡単な方法は、ビルド完了後にトップレベルのビルドディレクトリで "make -s cpluspluscheck" と入力することです。configureオプションに "--with-perl --with-python" を含めておく必要があります。そうしないと、関連するヘッダーが見つからないというエラーが発生する可能性が高いです。

g++と互換性のないC++コンパイラを使用している場合は、適切な環境変数を設定してスクリプトの CXXFLAGS 設定を上書きする必要があるかもしれません。

現在のスクリプトの制限として、どのヘッダーがフロントエンド用でどれがバックエンド用かを正確に把握していないことが挙げられます。判断に迷う場合は、postgres_fe.h や c.h がより適切であっても、前提条件として postgres.h を使用します。また、マクロの内容はチェックされませんが、これは意図的なものです。
