LDAP機能のテスト
============================

このディレクトリには、LDAP機能のテストスイートが含まれています。これにはサーバーおよびクライアントツールを含むOpenLDAPの完全なインストールが必要であるため、デフォルトでは実行されないように分離されています。想定外の場所にOpenLDAPがある場合は、テストファイル内のパスをいくつか調整する必要があるかもしれません。

また、このテストスイートは、アクセス制御なしでlocalhost上のTCP/IP接続をリッスンするLDAPサーバーを作成します。そのため、信頼できないローカルユーザーが存在する可能性のあるシステムでこれを実行することは安全ではありません。

テストの実行
=================

注意: configureに --enable-tap-tests 引数を指定している必要があります。

実行方法:
    make check PG_TEST_EXTRA=ldap
または
    make installcheck PG_TEST_EXTRA=ldap
以前に "make install" を実行した場合は "make installcheck" を使用できます。その場合、インストールツリー内のコードがテストされます。"make check" を使用した場合は、現在のソースから一時的なインストールツリーが構築され、それがテストされます。

いずれの場合も、このテストはテスト用のPostgresクラスターとテスト用のLDAPサーバーを初期化、起動、停止します。

これらのテストの実行に関する詳細については、src/test/perl/README を参照してください。

要件
============

LDAPサーバーおよびクライアントツールが必要です。

Debian/Ubuntuパッケージ: slapd ldap-utils

RHEL/CentOS/Fedoraパッケージ: openldap-clients openldap-servers
(ビルドにはすでに openldap および openldap-devel が必要です。)

FreeBSD: openldap-server
(ビルドにはすでに openldap-client が必要です。portsソースツリーからビルドする場合は、net/openldap24-client および net/openldap24-server をビルドしてください。)

macOS: Appleが提供するバージョンのOpenLDAPを使用することは推奨しません。非常に古いうえに、Appleはslapdの起動規則を変更しているようです。テストファイル内のパスは、HomebrewまたはMacPortsを使用してOpenLDAPをインストールしたことを前提として設定されています。
