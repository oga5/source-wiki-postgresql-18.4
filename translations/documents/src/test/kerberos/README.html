Kerberos/GSSAPI 機能のテスト
=======================================

このディレクトリには、Kerberos/GSSAPI 機能のテストスイートが含まれています。これにはサーバーおよびクライアントツールを含む完全な MIT Kerberos のインストールが必要であるため、別途保持されており、デフォルトでは実行されません。

注意: このテストで実行されるテストサーバーは、localhost 上の TCP 接続をリッスンするように構成されています。テストの実行中は、同じホスト上のすべてのユーザーがテストサーバーにログイン可能です。すべてのローカルユーザーを信頼できないマルチユーザーシステムでは、このスイートを実行しないでください！また、このテストスイートは、実質的なアクセス制御なしで localhost 上の TCP/IP 接続をリッスンする KDC サーバーを作成します。

テストの実行
=================

注: configure に --enable-tap-tests 引数を指定している必要があります。また、"make installcheck" を使用するには、コアコードに加えて contrib/dblink および contrib/postgres_fdw をビルドしてインストールしておく必要があります。

実行するには
    make check PG_TEST_EXTRA=kerberos
または
    make installcheck PG_TEST_EXTRA=kerberos
以前に "make install" を実行した場合は "make installcheck" を使用できます。その場合、インストールツリー内のコードがテストされます。"make check" では、現在のソースから一時的なインストールツリーが構築され、それがテストされます。

いずれの場合も、このテストはテスト用の Postgres クラスターとテスト用の KDC サーバーを初期化、起動、停止します。

これらのテストの実行に関する詳細については、src/test/perl/README を参照してください。

要件
============

MIT Kerberos サーバーおよびクライアントツールが必要です。Heimdal はサポートされていません。

Debian/Ubuntu パッケージ: krb5-admin-server krb5-kdc krb5-user

RHEL/CentOS/Fedora パッケージ: krb5-server krb5-workstation

FreeBSD ポート: krb5 (ベースシステムには Heimdal が含まれています)
