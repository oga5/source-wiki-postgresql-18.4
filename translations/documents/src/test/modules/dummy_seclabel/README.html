src/test/modules/dummy_seclabel/README

dummy_seclabel モジュールは、SECURITY LABEL 文の回帰テストをサポートするためだけに存在します。本番環境での使用を意図したものではありません。

根拠
====

SECURITY LABEL 文を使用すると、ユーザーはデータベースオブジェクトにセキュリティラベルを割り当てることができます。ただし、セキュリティラベルはロード可能なモジュールによって明示的に許可されている場合にのみ割り当てることができるため、適切な回帰テストを可能にする目的でこのモジュールが提供されています。

本番環境での使用を意図したセキュリティラベルプロバイダーは、通常、SELinux のようなプラットフォーム固有の機能に依存します。このモジュールはプラットフォームに依存しないため、回帰テストに適しています。

使用方法
========

以下に使用例を示します。

# postgresql.conf
shared_preload_libraries = 'dummy_seclabel'

postgres=# CREATE TABLE t (a int, b text);
CREATE TABLE
postgres=# SECURITY LABEL ON TABLE t IS 'classified';
SECURITY LABEL

dummy_seclabel モジュールは、unclassified、classified、secret、top secret という 4 つのハードコードされたラベルのみを提供します。
これら以外の文字列をセキュリティラベルとして使用することはできません。

これらのラベルはアクセス制御を強制するために使用されるものではありません。SECURITY LABEL 文が期待通りに動作するかどうかを確認するためだけに使用されます。
