src/test/modules/test_regex/README

test_regex は、正規表現パッケージをテストするためのモジュールです。
主に Tcl の正規表現テストスイートを取り込めるようにすることを目的としています。
そのため、PostgreSQL の SQL レベルでは現在公開されていない正規表現機能を実行するための規定が設けられています。

現在、以下の関数が提供されています。

test_regex(pattern text, string text, flags text) returns setof text[]

パターンが無効な正規表現である場合はエラーを報告します。それ以外の場合、出力の最初の行には部分式の数が含まれ、その後に正規表現の re_info フィールド内のセットビットを報告する単語が続きます。
パターンが文字列と一致しない場合、出力はこれだけです。
パターンが一致する場合、次の行には最初の配列要素として一致全体が含まれます。括弧で囲まれた部分式がある場合、続く配列要素にはそれらの部分式への一致が含まれます。
"g" (glob) フラグが設定されている場合、追加の出力行で同様に後続の一致が報告されます。

"flags" 引数は、正規表現パッケージまたはテスト関数の動作を変更する、0 個以上の 1 文字フラグからなる文字列です。Tcl の reg.test ファイルで説明されているように、以下の通りです。

フラグ文字は複雑で少し折衷的です。一般的に、小文字はコンパイルオプション、大文字は期待される re_info ビット、非アルファベットは一致オプション、テストの実行方法の制御、またはテストオプションです。唯一の小さな驚きは、ARE がデフォルトであり、より単純な種類の RE を明示的に要求する必要があることです。フラグは以下の通りです。一部はあまり覚えやすくないことを認めます。

	-	何もしない（プレースホルダー）
	0	実際の文字列ではなくインデックスを報告する
		（これは Tcl の -indices スイッチの代わりです）
	!	部分一致を期待し、いずれにせよ開始位置を報告する
	%	マッチャー内の状態セットキャッシュを強制的に小さくする（キャッシュ置換をテストするため）
	^	文字列の先頭を行の先頭とみなさない
	$	文字列の末尾を行の末尾とみなさない
	*	Unicode 特有のテスト、大きな文字セットが必要
	+	偽の xy 等価クラスと ch 照合要素を提供する
		（注：等価クラスは実装されていますが、照合要素は実装されていないため、[.ch.] への参照は失敗します）
	,	REG_PROGRESS を設定する（REG_DEBUG ビルドでのみ有用）
	.	REG_DUMP を設定する（REG_DEBUG ビルドでのみ有用）
	:	REG_MTRACE を設定する（REG_DEBUG ビルドでのみ有用）
	;	REG_FTRACE を設定する（REG_DEBUG ビルドでのみ有用）

	&	ARE と BRE の両方としてテストする
		（Postgres では未実装、個別のテストを使用します）
	b	BRE
	e	ERE
	a	高度な機能ビットをオンにする（ERE でない場合はエラー）
	q	リテラル文字列、メタ文字は一切なし

	g	グローバルマッチ（すべての一致を検索）
	i	大文字小文字を区別しないマッチング
	o	（"opaque"）一致位置を返さない
	p	改行は半マジック、. および [^ からのみ除外される
	w	改行は半マジック、^ および $ に対してのみ重要
	n	改行は完全マジック、両方の効果を持つ
	x	拡張 RE 構文
	t	不完全一致の報告
	c	canmatch（Postgres 実装における "t0!" と同等）
	s	先頭でのみ一致（REG_BOSONLY）

	A	バックスラッシュ-_a_lphanumeric が見つかった
	B	ERE/ARE リテラル-_b_race ヒューリスティックが使用された
	E	[] 内でバックスラッシュ (_e_scape) が見つかった
	H	先読み (_h_ead) 制約が見つかった
	I	一致させるのが _i_mpossible（不可能）
	L	_l_ocale 特有の構成が見つかった
	M	移植性のない (_m_achine-specific) 構成が見つかった
	N	RE が空 (_n_ull) 文字列と一致する可能性がある
	P	非 _P_OSIX 構成が見つかった
	Q	{} _q_uantifier（量指定子）が見つかった
	R	後方参照 (_r_eference) が見つかった
	S	POSIX で指定されていない (_s_pecified) 構文が見つかった
	T	最短 (_t_iny) を優先する
	U	元の POSIX の失敗（ERE 内の対応する閉じ括弧がない）が見つかった (_u_gh)
