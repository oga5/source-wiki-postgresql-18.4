src/test/modules/libpq_pipeline/README

libpq 用のテストプログラムおよびライブラリ

Test::Differences がインストールされている場合、トレースファイルの差異は標準形式よりも読みやすい形式で表示されます。
=====================================

このモジュールは libpq の「パイプライン」モードをテストするために開発されましたが、特殊な C コードを必要とするあらゆる libpq テストに使用できます。

"make check" を実行すると、モジュール内のすべてのテストが一時的なサーバーインストールに対して実行されます。

特定のテストを手動で実行するには、以下のようにします。

    ./libpq_pipeline <テスト名> [ <接続文字列> ]

これは新しいサーバーを起動するのではなく、接続文字列で指定されたサーバー、または省略した場合はデフォルトのサーバーに接続します。利用可能なテスト名を確認するには、以下を実行してください。

    ./libpq_pipeline tests

このモジュールに新しいテストを追加するには、libpq_pipeline.c を編集する必要があります。
テストを実行する関数を追加し、新しいテストの名前がプログラムに渡されたときに main() がその関数を呼び出すように設定してください。その際、print_test_list() 関数にテスト名を追加するのを忘れないでください。忘れると TAP テストが実行されません。

テストにおいて Postgres プロトコルメッセージが送信される順序が決定論的である場合は、TAP テストによってメッセージシーケンスが検証されるように設定してください。まず、以下のようなコマンドを使用して、リファレンスとなるトレースファイルを生成します。

   ./libpq_pipeline -t traces/mynewtest.trace mynewtest

次に、t/001_libpq_pipeline.pl ファイル内の $cmptrace 定義のリストにテスト名を追加します。"make check" を数回実行して、トレース出力が実際に安定していることを確認してください。
