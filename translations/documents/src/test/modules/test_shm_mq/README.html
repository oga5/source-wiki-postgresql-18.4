src/test/modules/test_shm_mq/README

test_shm_mq は、動的共有メモリおよび共有メモリメッセージキュー機能を使用して、ユーザーバックエンドと1つ以上のバックグラウンドワーカーの連携を行う方法を示す例です。これ自体で何か有用なことを行うことを意図したものではなく、これらの機能の使用方法を示すデモンストレーションであり、またそれらの機能のユニットテストでもあります。

この拡張機能の関数は、プロセスのループを通じて同じメッセージを繰り返し送信します。メッセージのペイロード、送信に使用するメッセージキューのサイズ、およびループ内のプロセス数は設定可能です。最後に、メッセージが送信中に破損していないことを確認するために検証が行われる場合があります。

関数
=========


test_shm_mq(queue_size int8, message text,
            repeat_count int4 default 1, num_workers int4 default 1)
    RETURNS void

この関数は、メッセージを同期的に送受信します。ユーザーバックエンドは、指定されたサイズのメッセージキューを使用して、提供されたメッセージを最初のバックグラウンドワーカーに送信します。最初のバックグラウンドワーカーは、ワーカー数が1より大きい場合は2番目のバックグラウンドワーカーにメッセージを送信し、以下同様に続きます。最終的に、最後のバックグラウンドワーカーがユーザーバックエンドにメッセージを返送します。repeat_count が1より大きい場合、ユーザーバックエンドはメッセージを最初のワーカーに送り返します。メッセージがすべての連携プロセスによって repeat_count 回送受信されると、ユーザーバックエンドは最終的に受信したメッセージが最初に送信したものと一致することを確認し、一致しない場合はエラーをスローします。


test_shm_mq_pipelined(queue_size int8, message text,
                      repeat_count int4 default 1, num_workers int4 default 1,
                      verify bool default true)
    RETURNS void

この関数は、repeat_count で指定された回数だけ同じメッセージを、指定されたサイズのキューを使用して最初のバックグラウンドワーカーに送信します。これらのメッセージは、それぞれ指定されたサイズのキューを使用して、各バックグラウンドワーカーに順番に転送されます。最後に、最後のバックグラウンドワーカーがメッセージをユーザーバックエンドに返送します。ユーザーバックエンドは非ブロッキング送受信を使用するため、すべてのメッセージのコピーを送信し終える前に、メッセージのコピーの受信を開始できます。'verify' 引数は、受信したコピーが送信されたメッセージと一致するかどうかをチェックするかどうかを制御します。（これには無視できない時間がかかるため、ベンチマーク目的で無効にすると便利な場合があります。）
