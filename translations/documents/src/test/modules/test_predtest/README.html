src/test/modules/test_predtest/README

test_predtest は、src/backend/optimizer/util/predtest.c にあるオプティマイザの述語証明ロジックの正確性を検証するためのモジュールです。

このモジュールは、predtest.c で公開されている関数 predicate_implied_by() および predicate_refuted_by() を任意の論理式に対して直接適用し、その結果を直接検証できる関数を提供します。これはプランナの結果を確認することで間接的に行うことも可能ですが、期待通りの結果が得られたことを証明するエンドツーエンドのテストケースを構築するのは困難な場合があります。

一般的に、この関数の使用方法は以下の通りです。
	select * from test_predtest('query string')
ここで、クエリ文字列は2つの論理型カラムを返す SELECT 文である必要があります。例：

	select * from test_predtest($$
	select x, not x
	from (values (false), (true), (null)) as v(x)
	$$);

この関数は与えられたクエリを解析および計画し、SELECT リスト内の2つの論理式に対して predtest.c のコードを適用して、最初の式が2番目の式によって証明または反証できるかどうかを確認します。また、クエリを実行し、結果の行をチェックして、主張された含意または反証の関係が実際に成立しているかどうかを確認します。上記の例のように、可能な入力値の全セットに対して式を検証するようにクエリが設計されている場合、これは証明コードが正しい答えを出したかどうかを機械的にクロスチェックする手段となります。
