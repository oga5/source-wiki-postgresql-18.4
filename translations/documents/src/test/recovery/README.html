リカバリおよびレプリケーションの回帰テスト
=============================================

このディレクトリには、リカバリおよびレプリケーションのテストスイートが含まれています。

テストの実行
=================

注意: configure に --enable-tap-tests 引数を指定している必要があります。
また、"make installcheck" を使用するには、コアコードに加えて contrib/pg_prewarm、contrib/pg_stat_statements、および contrib/test_decoding をビルドしてインストールしておく必要があります。

実行方法:
    make check
または
    make installcheck
以前に "make install" を実行している場合は "make installcheck" を使用できます。
その場合、インストールツリー内のコードがテストされます。"make check" の場合は、現在のソースから一時的なインストールツリーが構築され、それがテストされます。

いずれの場合も、このテストでは複数のテスト用 Postgres クラスタの初期化、起動、停止が行われます。

WAL 一貫性チェックをテストしたい場合は、"make" コマンドに
PG_TEST_EXTRA=wal_consistency_checking
を追加してください。これはリソースを大量に消費するため、デフォルトでは実行されません。

これらのテストの実行に関する詳細については、src/test/perl/README を参照してください。
