SSL回帰テスト
=============

このディレクトリには、SSLサポートのテストスイートが含まれています。クライアント側の機能（サーバー証明書の検証など）と、サーバー側の機能（証明書の認証など）の両方をテストします。

注意: このテストで実行されるテストサーバーは、localhostでのTCP接続を待ち受けるように構成されています。テストの実行中は、同じホスト上のすべてのユーザーがテストサーバーにログイン可能です。ローカルの全ユーザーを信頼できないマルチユーザーシステムでは、このスイートを実行しないでください！

テストの実行
============

注意: configureに --enable-tap-tests 引数を指定している必要があります。また、"make installcheck" を使用するには、コアコードに加えて contrib/sslinfo をビルドおよびインストールしておく必要があります。

実行方法:
    make check PG_TEST_EXTRA=ssl
または
    make installcheck PG_TEST_EXTRA=ssl
以前に "make install" を実行した場合は "make installcheck" を使用できます。その場合、インストールツリー内のコードがテストされます。"make check" の場合は、現在のソースから一時的なインストールツリーが構築され、それがテストされます。

いずれの場合も、このテストは他のローカルユーザーがアクセス可能なテスト用Postgresクラスターを初期化、起動、停止します！

これらのテストの実行に関する詳細は、src/test/perl/README を参照してください。

証明書
======

テストスイートを実行するには、公開鍵/秘密鍵のペアと証明書のセットが必要です。

root_ca
	ルートCA。サーバーおよびクライアントのCA証明書に署名するために使用します。

server_ca
	サーバー証明書に署名するために使用するCA。

client_ca
	クライアント証明書に署名するために使用するCA。

server-cn-only
server-cn-and-alt-names
server-single-alt-name
server-multiple-alt-names
server-no-names
	サーバー証明書。証明書内のホスト名にわずかな違いがあります。server_caによって署名されています。

server-password
	server-cn-onlyと同じですが、パスワードで保護されています。

client
	ユーザー "ssltestuser" 用のクライアント証明書。client_caによって署名されています。

client-revoked
	"client" と同様ですが、クライアントCAのCRLで失効済みとしてマークされています。

さらに、複数の証明書を同じファイルにまとめたファイルがいくつかあります。

both-cas-1
	root_ca.crt、client_ca.crt、server_ca.crtをこの順序で含みます。

both-cas-2
	root_ca.crt、server_ca.crt、client_ca.crtをこの順序で含みます。

root+server_ca
	root_crtとserver_ca.crtを含みます。クライアントの "sslrootcert" オプション用です。

root+client_ca
	root_crtとclient_ca.crtを含みます。サーバーの "ssl_ca_file" 用です。

client+client_ca
	client.crtとclient_ca.crtをこの順序で含みます。クライアントの証明書チェーン用です。

各CA用のCRL（root.crl、server.crl、client.crl）もあります。

便宜上、これらの鍵ペアと証明書はすべて ssl/ サブディレクトリに含まれています。Makefileには、変更が必要な場合にそれらを再作成するための "make sslfiles" というルールも含まれています。完全な鍵ペアと証明書のセットを再作成するには、"make sslfiles-clean" が必要です。個別のファイルを再構築するには、対象のファイルをtouch（または削除）してから "make sslfiles" を実行してください。この手順には少なくともOpenSSL 1.1.1が必要です。

注意
====

これらの証明書は、LDAPテストなど、他のテストでも使用されます。
