src/common/unicode/README

このディレクトリには、新しいUnicodeデータファイルをダウンロードし、静的テーブルを生成するためのツールが含まれています。これらのテーブルは、Unicodeデータの正規化や各種プロパティの判定に使用されます。

生成されたヘッダーファイルは src/include/common/ にコピーされ、ソースツリーに含まれるため、通常、PostgreSQLのビルドにこれらのツールは必要ありません。

Unicodeバージョンの更新
----------------------

src/Makefile.global.in および src/common/unicode/meson.build を編集して、UNICODE_VERSION を更新してください。

その後、以下のコマンドで新しいヘッダーファイルを生成します。

    make update-unicode

または meson を使用している場合は、

    ninja update-unicode

をソースツリーのトップレベルから実行します。結果を確認し、変更が妥当であること（つまり、差分のサイズと範囲が前回の更新以降のUnicodeの変更と比較して妥当であること）を確認してからコミットしてください。

テスト
-----

正規化テスト:

Unicodeコンソーシアムは、正規化アルゴリズムのための包括的なテストスイートを NormalizationTest.txt というファイルで公開しています。このディレクトリには、NormalizationTest.txt 内のすべてのテスト文字列を使用して、我々の正規化コードを実行するためのPerlスクリプトとCコードが含まれています。NormalizationTest.txt をダウンロードしてテストを実行するには、以下を実行します。

    make normalization-check

これは update-unicode ターゲットの一部としても実行されます。

カテゴリ、プロパティ、およびケースのテスト:

case_test.c および category_test.c ファイルは、ICUの結果と網羅的に比較することで、Unicodeのカテゴリ、プロパティ、およびケースマッピングをテストします。これらのテストを有効にするには、Unicodeデータファイルのバージョンが、ICUが基づいているUnicodeのバージョンと類似している必要があります。Unicodeのバージョンが一致していないと、一方のバージョンには割り当てられているがもう一方には割り当てられていないコードポイントをテストがスキップしてしまい、誤って失敗が報告される可能性があります。このテストは update-unicode ターゲットの一部として実行されます。
