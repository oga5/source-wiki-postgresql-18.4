contrib/start-scripts/macos/README

macOSでシステム起動時にPostgreSQLサーバーを自動的に起動させるには、以下の手順を実行してください。

1. postgres-wrapper.shスクリプトを編集し、Postgresのインストール先が /usr/local/pgsql ではない場合は、スクリプト冒頭のファイルパス変数をインストール先に合わせて調整してください。

2. 修正したpostgres-wrapper.shスクリプトを適切なインストールディレクトリにコピーします。Postgresの実行ファイルを保持している場所である必要はありませんが、そこでも構いません。

3. org.postgresql.postgres.plistファイルを編集し、手順2で行った内容に合わせてpostgres-wrapper.shへのパスを調整してください。また、Postgresサーバーを「postgres」以外のユーザー名で実行する予定の場合は、UserNameパラメータの値をそれに合わせて調整してください。

4. 修正したorg.postgresql.postgres.plistファイルを /Library/LaunchDaemons/ にコピーします。この操作はroot権限で行う必要があります。
    sudo cp org.postgresql.postgres.plist /Library/LaunchDaemons
    root所有でない場合、ファイルは無視されるためです。

この時点で再起動すればサーバーが起動するはずです。再起動せずにテストしたい場合は、以下のコマンドを実行してください。
    sudo launchctl load /Library/LaunchDaemons/org.postgresql.postgres.plist
