default_table_access_method

概念

default_table_access_method は、PostgreSQLにおいてテーブルを作成する際、アクセス方法（Table Access Method）が明示的に指定されていない場合にデフォルトで適用されるストレージエンジンを決定するための設定パラメータ、およびそれに関連する定義です。

なぜ重要か

PostgreSQLはプラグイン可能なテーブルストレージ（Table Access Methods API）をサポートしています。これにより、標準の heap 以外のストレージエンジンを選択することが可能になります。default_table_access_method は、ユーザーが CREATE TABLE 時に USING 句を省略した場合に、どのストレージエンジンをデフォルトとして使用するかを制御するために重要な役割を果たします。

  • symbol:src/include/access/tableam.h#DEFAULT_TABLE_ACCESS_METHOD

    • デフォルトのテーブルアクセス方法の初期値（一般的には "heap"）を定義するマクロです。

エンドツーエンドの流れ

提供されたソースコードのコンテキストからは、詳細なエンドツーエンドの制御フロー（GUCの初期化から、CREATE TABLE 実行時にこの変数が参照されて実際のテーブルが作成されるまでの具体的なコールスタックなど）の完全な詳細はまだ解析されていません。

一般的には、テーブル作成処理においてアクセス方法が指定されていない場合、この default_table_access_method に格納されている文字列に対応するアクセス方法がルックアップされ、テーブルの作成に適用されます。

現時点では、提供されたシンボル情報からは、この変数やマクロに関する詳細な不変条件（例：設定可能な値の検証ルールや、無効なアクセス方法が指定された場合のフォールバック挙動など）は特定されていません。
