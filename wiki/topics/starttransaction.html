StartTransaction

概念

StartTransaction は、PostgreSQLにおいてトランザクションを開始するための処理を指します。この名称を持つ処理は、バックエンド（サーバー側）のトランザクション管理機構と、フロントエンド（pg_dump などのクライアントツール側）のデータベース接続制御の双方に存在しており、それぞれ異なる役割を担っています。

なぜ重要か

トランザクションの開始は、データの整合性を保ちながら一連の操作を実行するための基本処理です。

  • **フロントエンド（pg_dump）**: バックアップ対象のデータベースに対して一貫した状態を確保するために、BEGIN コマンドを送信してトランザクションを開始します。
  • **バックエンド**: 実際のトランザクション状態の遷移や管理を行うために重要ですが、その詳細な役割は提供された情報からは未解明です。

関与するモジュール

  • module:src/backend/access/transam
  • module:src/bin/pg_dump

主要エンティティ

  • symbol:src/backend/access/transam/xact.c#StartTransaction: バックエンド側のトランザクション開始処理。
  • symbol:src/bin/pg_dump/pg_backup_db.c#StartTransaction: フロントエンド（pg_dump）側のトランザクション開始処理。Archive *AHX を引数に取り、"BEGIN" コマンドを実行します。

エンドツーエンドの流れ

フロントエンドにおける処理の流れは以下の通りです：

1. StartTransaction(Archive *AHX) が呼び出される。 2. 引数 AHX が ArchiveHandle *AH にキャストされる。 3. ExecuteSqlCommand を呼び出し、SQL コマンド "BEGIN" を実行する。 4. 実行に失敗した場合、エラーメッセージ "could not start database transaction" を伴ってプロセスが終了する。

なお、フロントエンドにおける本関数の具体的な呼び出し元や、バックエンド側（src/backend/access/transam/xact.c）の StartTransaction を含めた全体的なエンドツーエンドの流れは現時点では未解明です。

重要な不変条件

  • フロントエンドの StartTransaction において、引数 AHX は有効な Archive 構造体ポインタでなければならず、内部で ArchiveHandle 型へキャストされるため、pg_backup_archiver.h で定義された構造体レイアウトと整合している必要があります。
  • バックエンド側における不変条件は現時点では未解明です。

よくある落とし穴

  • **名前の混同**: src/backend/access/transam/xact.c の StartTransaction（バックエンド用）と、src/bin/pg_dump/pg_backup_db.c の StartTransaction（フロントエンド用）は、名前は同じですが全く異なるコンテキストで動作する別個の関数です。これらを混同しないよう注意が必要です。
  • **エラー時の即時終了**: フロントエンドの StartTransaction は、SQLの実行に失敗すると ExecuteSqlCommand 経由でプロセスを即時終了（exit）するため、呼び出し側でエラーをキャッチして回復するような制御は行えません。
