UseScramPassthrough

概念

UseScramPassthrough は、外部サーバー（ForeignServer）またはユーザーマッピング（UserMapping）のオプション設定から、SCRAM認証のパススルー（use_scram_passthrough）が有効化されているかどうかを判定するための関数です。

なぜ重要か

外部データラッパ（postgres_fdw）やリモートデータベース接続ツール（dblink）において、接続確立時やSCRAMキー検証時に、SCRAMパススルー認証を適用すべきかどうかを制御するために重要です。これにより、外部サーバー単位またはユーザーマッピング単位で認証動作を柔軟に切り替えることができます。

関与するモジュール

  • module:contrib/dblink
  • module:contrib/postgres_fdw

主要エンティティ

  • symbol:contrib/postgres_fdw/connection.c#UseScramPassthrough: 外部サーバーとユーザーマッピングのオプションから use_scram_passthrough の設定値を評価する静的関数。
  • symbol:contrib/dblink/dblink.c#UseScramPassthrough: dblink 側で接続時のSCRAMキー検証の判定に利用されるシンボル。

エンドツーエンドの流れ

1. postgres_fdw による接続確立時、または dblink による接続時のSCRAMキー検証時に、対象の ForeignServer および UserMapping を引数として UseScramPassthrough が呼び出されます。 2. 関数はまず ForeignServer のオプション（server->options）から use_scram_passthrough を検索します。 3. サーバーオプションに見つからない場合は、次に UserMapping のオプション（user->options）から同オプションを検索します。 4. オプションが見つかった場合は defGetBoolean を用いて真偽値に変換して返します。どちらにも設定がない場合は false を返します。 5. 呼び出し元はこの返り値（true または false）に基づいて、SCRAMパススルー処理を適用するかどうかを決定します。

※なお、SCRAMパススルーが有効化された後に、具体的にどのように認証パケットが転送・検証されるかという詳細なエンドツーエンドの通信フローは、提供されたコンテキストからは不明です。

重要な不変条件

  • **優先順位の不変性**: オプションの検索において、サーバーレベルの設定（ForeignServer）が常にユーザーレベルの設定（UserMapping）よりも優先されます。
  • **デフォルト値**: 対象のオプションがどちらの定義にも存在しない場合、関数は常に false を返します。
  • **引数の有効性**: 入力となる server および user は有効なポインタである必要があります。

よくある落とし穴

  • ユーザーマッピング（UserMapping）側で use_scram_passthrough を true に設定していても、サーバー（ForeignServer）側で明示的に false に設定されている場合、サーバー側の設定が優先されるため、ユーザー側の設定は無視されます。
  • オプション値のパースには defGetBoolean が使用されるため、無効な形式の値を指定した場合の挙動は defGetBoolean の実装に依存します。
