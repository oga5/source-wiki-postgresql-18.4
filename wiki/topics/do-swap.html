do_swap

概念

do_swap は、PostgreSQLのソースコードにおいて異なる目的で使用されている2つのエンティティに関連する名称です。

1. **pg_upgrade における関数**: 旧クラスタと新クラスタの間でディレクトリ構造の入れ替えやカタログファイルの差し替えを統括する静的関数。 2. **sort_template.h におけるマクロ**: ソート処理のテンプレート内で要素の入れ替え（スワップ）を行うために定義されている DO_SWAP マクロ。

これらは名前が類似していますが、一方はアップグレード時のファイルシステム操作、もう一方は汎用的なソートアルゴリズムの実装という、全く異なる文脈で機能します。

なぜ重要か

  • **pg_upgrade の do_swap**: --swap モードでのアップグレード実行時に、新旧クラスタのデータディレクトリやテーブルスペース、カタログファイルを安全かつ正しい順序で入れ替えるための中心的な制御フローを提供します。
  • **sort_template.h の DO_SWAP**: PostgreSQL内部で高速かつインライン化されたソート処理を実現するための共通部品として機能していると推測されます（詳細な仕様は提供されたコンテキストからは不明です）。

エンドツーエンドの流れ

pg_upgrade における do_swap の流れ

1. **ソートの実行**: 入力されたファイル対応表（FileNameMap 配列）を、比較関数 FileNameMapCmp を用いて relfilenumber 順に qsort で並べ替えます。 2. **対象の判定と処理**:

  • **特定のテーブルスペース (old_tablespace) が指定されている場合**:
  • そのテーブルスペースに対して prepare_for_swap を実行します。
  • 成功した場合、swap_catalog_files を呼び出してカタログファイルを差し替えます。
  • **特定のテーブルスペースが指定されていない場合**:
  • まずメインのデータディレクトリである old_cluster.pgdata に対して prepare_for_swap および swap_catalog_files を実行します。
  • 続いて、os_info.old_tablespaces に登録されているすべてのテーブルスペースに対して順次同様の処理をループ実行します。

※ sort_template.h の DO_SWAP マクロが関与するソート処理のエンドツーエンドの流れについては、提供されたコンテキストからは不明です。

重要な不変条件

  • **ソート順の保証**: pg_upgrade の do_swap 内で、maps 配列は swap_catalog_files を呼び出す前に必ず relfilenumber 順にソートされていなければなりません。
  • **実行の依存関係**: prepare_for_swap が成功（真を返却）した場合にのみ、後続の swap_catalog_files が呼び出されます。

※ sort_template.h の DO_SWAP マクロに関する不変条件は、提供されたコンテキストからは不明です。

よくある落とし穴

  • **モードの混同**: pg_upgrade の do_swap は --swap モード専用の独立したコードパスです。通常のファイル転送処理とは異なるため、コードを変更する際は prepare_for_swap や swap_catalog_files との整合性を慎重に維持する必要があります。
  • **境界条件の未チェック**: maps 配列が空、または size 引数が不適切な状態で do_swap を呼び出すと、qsort や配列アクセスで予期せぬ動作（セグメンテーションフォールトなど）を引き起こす可能性があります。

※ sort_template.h の DO_SWAP マクロに関する落とし穴は、提供されたコンテキストからは不明です。
