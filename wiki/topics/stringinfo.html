StringInfo

概念

StringInfo は、PostgreSQL において可変長文字列バッファを管理・操作するためのインターフェースです。実体である StringInfoData 構造体へのポインタ型（typedef StringInfoData *StringInfo）として定義されており、動的なメモリ再割り当てを伴う文字列の追加や、バイナリデータの保持を抽象化します。

なぜ重要か

手動でのメモリ管理やバッファオーバーフローのリスクを排除し、安全かつ効率的に可変長文字列を扱うために重要です。バッファの自動拡張（repalloc）をカプセル化し、文字列長（len）、最大容量（maxlen）、走査用カーソル（cursor）を一元管理することで、一貫した文字列操作 API を提供します。

主要エンティティ

  • **symbol:src/include/lib/stringinfo.h#StringInfo**: StringInfoData 構造体へのポインタ型。
  • **symbol:src/common/stringinfo.c#initStringInfo**: StringInfoData 構造体をデフォルトサイズ（STRINGINFO_DEFAULT_SIZE）で初期化し、メモリを割り当てる関数。
  • **symbol:src/common/stringinfo.c#resetStringInfo**: メモリの再確保を行わずに、既存のバッファ内容を破棄して空の状態（長さ 0、カーソル 0）にリセットする関数。

エンドツーエンドの流れ

提供された情報に基づく基本的なライフサイクルは以下の通りですが、文字列の追加や自動拡張を伴う詳細なエンドツーエンドのフローは完全には判明していません。

1. **初期化**: initStringInfo を呼び出し、対象となる StringInfoData 構造体の各メンバ（data, len, maxlen, cursor）を初期化し、デフォルトサイズのバッファを割り当てます。 2. **操作（詳細未詳）**: appendStringInfo などの API（提供された情報には詳細なし）を介して、必要に応じてバッファを自動拡張しながら文字列操作を行います。 3. **リセット**: resetStringInfo を呼び出すことで、バッファのメモリ領域を解放・再確保することなく、先頭にヌル文字を書き込んで文字列長とカーソルを 0 にリセットし、再利用可能な状態にします。 4. **解放（詳細未詳）**: メモリコンテキストの生存期間に従って、または明示的な解放処理によってバッファメモリが解放されます。

重要な不変条件

  • **初期化の必須性**: StringInfo が指す StringInfoData は、操作を行う前に initStringInfo 等で適切に初期化されていなければなりません。
  • **書き込み可能バッファの制限**: 読み取り専用バッファ（maxlen == 0）に対する変更や拡張を伴う操作は禁止されています。
  • **リセット時のアサーション**: resetStringInfo を呼び出す際、対象バッファは str->maxlen != 0（書き込み可能）でなければならず、これを満たさない場合はアサーションエラーが発生します。

よくある落とし穴

  • **読み取り専用バッファへの操作**: maxlen == 0 の状態の StringInfo に対して resetStringInfo などの変更操作を行うと、アサーション失敗によりプログラムが停止します。
  • **メモリ管理のライフサイクル**: StringInfo のメモリ管理は palloc/pfree（フロントエンド環境では malloc/free）に依存しています。makeStringInfo() で動的に作成した場合は destroyStringInfo() による解放が必要ですが、スタック上に確保した StringInfoData を指している場合は data フィールドのみを解放する必要があるなど、確保方法に応じた適切なライフサイクル管理が必要です。
