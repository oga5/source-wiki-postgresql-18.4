VarReturningType

概念

VarReturningType は、PostgreSQLのクエリ解析および書き換え（Rewrite）フェーズにおいて、RETURNING 句に含まれる変数（Var 構造体）がどのタプルソース（OLD または NEW）を参照すべきかを指定するための列挙型（enum）です。

なぜ重要か

RETURNING 句を伴うデータ更新操作（INSERT、UPDATE、DELETE）において、更新前（OLD）と更新後（NEW）のどちらの行データを返すかを正確に制御・解決するために不可欠です。これにより、操作の種類に応じた適切なタプルソースの参照が可能になります。

  • symbol:src/backend/rewrite/rewriteManip.c#SetVarReturningType: クエリ書き換え時に VarReturningType を設定または操作するための関数（詳細な実装は提供された情報からは不明）。

エンドツーエンドの流れ

クエリの解析（Parser）および書き換え（Rewriter）の過程で、RETURNING 句内の変数に対して VarReturningType が設定されます。 ただし、解析から実行（Executor）に至る具体的なエンドツーエンドの制御フローや、SetVarReturningType がどのように呼び出されて伝播するかについての完全な詳細は、提供された情報からは未だ明らかではありません。

重要な不変条件

VarReturningType の各値は、操作の種類（INSERT/UPDATE/DELETE）に応じて以下の挙動を示す不変条件（前提条件）を持ちます。

  • **VAR_RETURNING_DEFAULT**: DELETE 操作時は OLD を返し、それ以外（INSERT/UPDATE）は NEW を返します。
  • **VAR_RETURNING_OLD**: DELETE または UPDATE 操作時は OLD を返し、それ以外は NULL を返します。
  • **VAR_RETURNING_NEW**: INSERT または UPDATE 操作時は NEW を返し、それ以外は NULL を返します。

よくある落とし穴

  • 列挙型として定義されているため、定義外の値を代入した場合の動作は未定義です。
  • 操作の種類（INSERT/UPDATE/DELETE）と VarReturningType の組み合わせが不適切な場合、実行時に意図しない NULL が返されたり、誤ったタプルが参照されたりする可能性があります。
