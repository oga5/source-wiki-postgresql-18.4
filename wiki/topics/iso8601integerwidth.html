ISO8601IntegerWidth

概念

ISO8601IntegerWidth は、ISO 8601 形式の文字列において、符号（-）を除いた数値部分の連続する桁数を算出するためのヘルパー関数です。PostgreSQL のサーバーサイド（バックエンド）とクライアントサイド（ECPG）の双方で、日付・時刻やインターバル文字列のパース処理を支えるために同名の関数が定義されています。

なぜ重要か

ISO 8601 形式のパース処理において、各フィールド（年、月、日など）の数値の桁数を正確に特定することは、後続のパース処理を正しく進めるために不可欠です。バックエンドのロジックを ECPG 側に移植することで、サーバーとクライアントの間で一貫したパース挙動を保証しています。

エンドツーエンドの流れ

1. **入力の受け取り**: パース処理の過程で、数値フィールドの開始位置を示すポインタ（fieldstart）が本関数に渡されます。 2. **符号のスキップ**: 先頭文字が負符号（'-'）である場合、それをスキップしてポインタを1つ進めます。 3. **桁数の算出**: 標準ライブラリ関数 strspn を使用し、'0' から '9' までの文字が連続する長さを計算します。 4. **結果の返却**: 算出された桁数を int 型として呼び出し元に返します。

※なお、バックエンド（datetime.c）における具体的な呼び出し元や、システム全体にわたる詳細なエンドツーエンドのデータフローは現時点では完全には判明していません。

重要な不変条件

  • **入力の妥当性**: 引数 fieldstart は有効な文字列を指している必要があります。
  • **符号の除外**: 算出される桁数には、先頭の負符号（-）は含まれません。
  • ※バックエンドと ECPG の実装間における挙動の完全な同一性（不変条件）の詳細は、現時点では未検証です。

よくある落とし穴

  • **型変換の不一致**: 内部で使用している strspn の戻り値は size_t ですが、本関数の戻り値は int です。極端に長い文字列を処理する場合、型オーバーフローに注意する必要があります。
  • **非数値文字列**: 入力文字列が数値で始まらない場合、または '-' のみの場合は 0 が返されるため、呼び出し側で適切にハンドリングする必要があります。
  • **コードの重複**: 本関数はバックエンドのロジックを ECPG に移植したものであるため、一方の挙動を変更する際、もう一方への追従を忘れると不整合が生じるリスクがあります。
