SetEpochTimestamp

概念

SetEpochTimestamp は、システムのエポック（Unixエポック）に対応する timestamp 値を取得するための関数です。PostgreSQLのバックエンド側（src/backend/utils/adt）およびクライアント側のECPGライブラリ（src/interfaces/ecpg/pgtypeslib）の双方に同名のシンボルが存在します。

なぜ重要か

データベースシステムや埋め込みSQL（ECPG）において、基準となるエポック時刻（Unixエポック）を表現する timestamp 値（内部的には int64 のマイクロ秒単位）を正確に取得し、日時計算や変換の基準点として利用するために重要です。

エンドツーエンドの流れ

src/interfaces/ecpg/pgtypeslib/timestamp.c における処理の流れは以下の通りです。

1. GetEpochTime() を呼び出し、現在のエポック時刻を struct tm 構造体に格納します。 2. GetEpochTime() が負の値を返した場合は、エラーとして 0 を返します。 3. 成功した場合、tm2timestamp() を使用して struct tm を timestamp 型に変換し、その値を返します。

なお、バックエンド側（src/backend/utils/adt/timestamp.c）における具体的な処理の流れや、両モジュールを跨ぐ全体的なエンドツーエンドの流れは現時点では不明です。

重要な不変条件

  • GetEpochTime() が正常に時刻を取得できる必要があります。
  • 内部的な変換関数 tm2timestamp() が成功することが前提となります。
  • バックエンド側における不変条件は現時点では不明です。

よくある落とし穴

  • src/interfaces/ecpg/pgtypeslib/timestamp.c の SetEpochTimestamp は static 関数であり、同ファイル内でのみ利用されることを意図しています。
  • GetEpochTime() が失敗した際に返される 0 はエラー状態を示すため、呼び出し側でこの値を有効なタイムスタンプとして扱わないよう注意が必要です。
