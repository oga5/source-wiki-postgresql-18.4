manifest_file

概念

manifest_file は、PostgreSQLのバックアップマニフェスト（backup_manifest）に含まれる個々のファイルに関するメタデータ（パス、サイズ、チェックサムなど）をメモリ上で保持・管理するための構造体です。

この構造体は、バックアップの統合（pg_combinebackup）や検証（pg_verifybackup）を行うツールにおいて、マニフェスト内のファイル情報をハッシュテーブル（simplehash）で効率的に検索・照合するために定義されています。

なぜ重要か

バックアップの検証や統合処理では、マニフェストに記載された大量のファイル情報を高速に検索し、実ファイルやストリームデータと照合する必要があります。manifest_file は、simplehash を用いたハッシュテーブルの要素として機能することで、ファイルパスをキーとした高速なルックアップを可能にし、バックアップデータの整合性検証や統合処理のパフォーマンスを支えています。

関与するモジュール

  • module:src/bin/pg_combinebackup
  • module:src/bin/pg_verifybackup

主要エンティティ

  • **symbol:src/bin/pg_combinebackup/load_manifest.h#manifest_file**: pg_combinebackup において、マニフェストから読み込んだファイルのメタデータ（パス、サイズ、チェックサム等）を保持する構造体。
  • **symbol:src/bin/pg_verifybackup/pg_verifybackup.h#manifest_file**: pg_verifybackup において、ファイルメタデータに加えて検証状態（matched、bad フラグ）を保持する構造体。

エンドツーエンドの流れ

1. **マニフェストの解析とロード**:

  • pg_combinebackup では、load_backup_manifest() を通じてマニフェストファイルが読み込まれ、manifest_file 構造体が初期化されてハッシュテーブル（manifest_data）に格納されます。
  • pg_verifybackup でも同様に、pg_verifybackup.c がマニフェスト解析時に本構造体をインスタンス化し、ハッシュテーブルに格納します。

    • 2. **検証・処理の実行**:

  • pg_verifybackup では、astreamer_verify.c がアーカイブストリームの処理中に本構造体を参照し、実ファイルとの照合結果に応じて matched（一致）や bad（不整合）フラグを更新します。
  • pg_combinebackup では、ハッシュテーブルに格納された manifest_file を用いて、ファイルの検索や統合処理を容易にします。

※なお、マニフェスト全体のロードから各ツールでの最終的な出力・判定に至る完全なエンドツーエンドの制御フローの詳細は、提供されたモジュール情報からはすべて明らかになっていません。

重要な不変条件

  • **ハッシュテーブルの要素としての制約**:
  • simplehash（lib/simplehash.h）の要素として利用されるため、構造体の先頭にはハッシュ状態を示す status フィールドが存在する必要があります。
  • **データの整合性**:
  • checksum_payload は、checksum_length に応じた正しいメモリ領域を指している必要があります。
  • pg_verifybackup において、pathname はマニフェスト内で一意である必要があります。
  • checksum_payload が NULL の場合、チェックサム検証はスキップされます。

よくある落とし穴

  • **名前の衝突**:
  • src/bin/pg_basebackup/pg_basebackup.c にも manifest_file という名前の変数が存在しますが、これは FILE * 型の変数であり、本構造体とは全く無関係です。
  • **ライフサイクル管理**:
  • pathname は const char * 型であり、マニフェスト読み込み時に確保されたメモリ領域を指すため、構造体と参照先メモリのライフサイクル管理に注意が必要です。
  • **ハッシュテーブル定義との同期**:
  • simplehash のマクロ展開と密接に関連しているため、manifest_file のフィールドを変更する場合は、ハッシュテーブルの定義側（load_manifest.c や pg_verifybackup.c）も併せて更新する必要があります。
