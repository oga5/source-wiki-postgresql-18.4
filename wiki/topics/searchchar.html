searchChar

概念

searchChar は、SP-GiST（Space-Partitioned GiST）インデックスのノードラベル配列から、指定された文字（int16 型）を二分探索（binary search）によって検索するためのヘルパー関数です。

なぜ重要か

SP-GiST インデックスにおけるテキスト処理（spgtextproc.c）や、名前型（name 型）のSP-GiST演算子クラスのテストモジュール（spgist_name_ops.c）において、ノード選択時（例: spg_text_choose）に適切な子ノードを迅速に特定、あるいは新しいノードの挿入位置を決定するための基本的な探索ロジックとして機能するため重要です。

エンドツーエンドの流れ

1. SP-GiSTのノード選択処理（spg_text_choose など）において、探索対象の文字 c（int16 型）、現在のノードが持つラベル配列 nodeLabels、およびその要素数 nNodes を指定して searchChar が呼び出されます。 2. searchChar は、nodeLabels に対して二分探索を実行します。

  • 中央値（middle / StopMiddle）を計算し、nodeLabels の要素と c を比較します。
  • 探索範囲を絞り込みながら一致する要素を探索します。

    • 3. 指定された文字 c が見つかった場合：

  • 戻り値として true を返し、ポインタ i にその一致したインデックスを格納します。

    • 4. 指定された文字 c が見つからなかった場合：

  • 戻り値として false を返し、ポインタ i に c を挿入すべき位置（StopHigh）を格納します。

    • 5. 呼び出し元は、この結果（見つかったか否か、およびインデックス i）を利用して、次のノードへの遷移や新規ノードの挿入などの処理を継続します。

※なお、SP-GiST全体のインデックス検索・挿入における完全なエンドツーエンドのフローの詳細は、提供されたコンテキストからは完全には明らかになっていません。

重要な不変条件

  • **ソート済みの配列**: 入力となる nodeLabels は、昇順にソートされている必要があります。
  • **有効な要素数**: nNodes は配列の有効な要素数である必要があります。
  • **戻り値とインデックスの関係**:
  • 戻り値が true の場合、*i は一致した要素のインデックスを指します。
  • 戻り値が false の場合、*i は c を挿入すべき適切な位置（StopHigh）を指します（nNodes が 0 の場合は 0 となります）。

よくある落とし穴

  • **文字コードの範囲**: int16 型の比較を行っているため、ラベルとして扱える文字コードの範囲に注意する必要があります。
  • **重複ラベルの挙動**: 配列内に重複するラベルが存在する場合の挙動は、標準的な二分探索の仕様に従うため、特定のインデックスが返される保証はありません。
  • **コードの重複と同期**: 本関数は src/backend/access/spgist/spgtextproc.c と src/test/modules/spgist_name_ops/spgist_name_ops.c の両方に定義されており、ロジックを共有しています。一方を変更する際は、もう一方のモジュールへの影響を考慮し同期する必要があります。
