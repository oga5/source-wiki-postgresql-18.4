EncodeSpecialTimestamp

概念

EncodeSpecialTimestamp は、PostgreSQL において特殊なタイムスタンプ値（無限大を表す infinity や -infinity）を、対応する文字列表現（EARLY や LATE）に変換するための関数です。

この関数は、PostgreSQL のコアバックエンド側（src/backend/utils/adt/timestamp.c）と、クライアント側の埋め込み SQL プリプロセッサ用ライブラリである ECPG（src/interfaces/ecpg/pgtypeslib/timestamp.c）の2箇所に、それぞれ同名の関数として定義されています。

なぜ重要か

タイムスタンプ型において、無限大（正の無限大・負の無限大）は通常の数値表現とは異なる特殊な内部表現（TIMESTAMP_IS_NOBEGIN や TIMESTAMP_IS_NOEND）を持ちます。これらをユーザーや他の処理系が理解できる一貫した文字列形式（EARLY / LATE）へ安全にデコードするために、バックエンドとクライアントライブラリ（ECPG）の双方でこの共通の変換処理が必要となります。

  • **symbol:src/interfaces/ecpg/pgtypeslib/timestamp.c#EncodeSpecialTimestamp**
  • ECPG ライブラリ側で定義される static 関数。特殊値以外が渡された場合は abort() を呼び出します。

エンドツーエンドの流れ

タイムスタンプ値を文字列に変換する処理（例えば、JSON へのシリアライズや ECPG でのホスト変数への代入など）の過程で、対象のタイムスタンプが特殊値（無限大）であるかどうかが判定されます。

1. 呼び出し元（バックエンドの json.c や timestamp.c、または ECPG の timestamp.c 内の関数）が、値が特殊値であることを検知します。 2. EncodeSpecialTimestamp が呼び出されます。 3. TIMESTAMP_IS_NOBEGIN（負の無限大）の場合は EARLY、TIMESTAMP_IS_NOEND（正の無限大）の場合は LATE という文字列定数を、呼び出し元から渡されたバッファ str に strcpy でコピーします。

※ なお、システム全体における完全なエンドツーエンドの呼び出しフローや、すべての呼び出し元における詳細な遷移は完全には明らかになっていません。

重要な不変条件

  • **入力値の制限**: 引数 dt は、必ず TIMESTAMP_IS_NOBEGIN または TIMESTAMP_IS_NOEND のいずれかを満たす特殊値でなければなりません。
  • **バッファサイズの確保**: 引数 str が指すバッファは、コピーされる文字列（EARLY または LATE）を格納するのに十分なサイズが呼び出し元で確保されている必要があります。

よくある落とし穴

  • **通常値の入力による異常終了**:
  • バックエンドの実装では、特殊値以外の通常のタイムスタンプ値が渡されると elog(ERROR) が発生し、トランザクションが中断されます。
  • ECPG の実装では、通常のタイムスタンプ値が渡されると abort() が呼び出され、プロセス全体が即座に異常終了します。
  • **バッファオーバーフローの危険性**:
  • EncodeSpecialTimestamp 内部ではバッファサイズの検証を行わずに strcpy を使用するため、呼び出し元で十分な領域を確保していない場合、バッファオーバーフローを引き起こす危険性があります。
