utf8_to_unicode

概念

utf8_to_unicode は、UTF-8 でエンコードされたマルチバイト文字のバイト列を、対応する Unicode コードポイント（pg_wchar 型）に変換するための関数です。入力されたバイトの先頭ビットパターン（1〜4バイト）を解析し、ビット演算によって1つのコードポイントを合成します。

なぜ重要か

PostgreSQL のマルチバイト文字処理において、UTF-8 文字列を Unicode コードポイントに変換する基盤的な処理を提供します。この関数は、src/backend、src/common、contrib などの広範なディレクトリから参照されており、マルチバイト文字列のフォーマットや検証、各種文字列操作の正確性を支える重要な役割を担っています。

関与するモジュール

  • module:src/fe_utils
  • module:src/include/mb

主要エンティティ

  • symbol:src/fe_utils/mbprint.c#utf8_to_unicode: UTF-8 バイト列から pg_wchar への変換ロジックを実装する static 関数。
  • symbol:src/include/mb/pg_wchar.h#utf8_to_unicode: pg_wchar 型の定義や関連するマルチバイト処理のインターフェースを提供するヘッダー。

エンドツーエンドの流れ

1. 呼び出し元（contrib/fuzzystrmatch/daitch_mokotoff.c や src/common/unicode_case.c など）が、UTF-8 文字列の先頭を指すポインタを utf8_to_unicode に渡します。 2. 関数内部で、先頭バイトのビットパターンを判定します：

  • 0xxxxxxx (1バイト)
  • 110xxxxx (2バイト)
  • 1110xxxx (3バイト)
  • 11110xxx (4バイト)

    • 3. 各バイト数に応じたビットシフトと結合演算を行い、pg_wchar 型の Unicode コードポイントを生成して返します。 4. 不正なバイトパターンの場合は 0xffffffff を返します。

なお、呼び出し元から本関数を経由した最終的な出力先や、システム全体における具体的なエンドツーエンドのデータフローの全容は、現時点では完全には判明していません。

重要な不変条件

  • **入力の有効性**: 入力ポインタ c は、少なくとも 1 バイト以上の有効なメモリ領域を指している必要があります。
  • **エラー時の戻り値**: 入力されたバイトシーケンスが UTF-8 として不正である場合、または 4 バイトを超えるシーケンスや不適切な継続バイトを持つ場合は、必ず 0xffffffff を返します。

よくある落とし穴

  • **静的定義と広範な参照**: 本関数は src/fe_utils/mbprint.c 内で static として定義されていますが、複数のディレクトリ（src/backend、src/common、contrib）から参照されています。ロジックを変更する際は、影響範囲が広範に及ぶため注意が必要です。
  • **UTF-8 仕様への準拠**: 変更を加える場合は、RFC 3629 などの UTF-8 仕様に厳密に準拠している必要があります。
