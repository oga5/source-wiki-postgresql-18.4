hstoreCheckKeyLen

概念

hstoreCheckKeyLen は、PostgreSQL の hstore データ型において、キーの長さが許容される制限内に収まっているかを検証するための関数（またはマクロ）です。この検証処理は、コアとなる hstore 拡張モジュールだけでなく、PL/Perl や PL/Python といった手続き言語とのデータ変換を行う統合モジュールでも個別に定義または利用されています。

なぜ重要か

hstore のキーには格納可能な最大長が存在します。PL/Perl や PL/Python などの外部言語環境から PostgreSQL の hstore 形式へデータを変換・インポートする際、キーの長さ制限を事前にチェックして不正なデータがシステムに侵入するのを防ぐために、この検証処理が極めて重要になります。

関与するモジュール

  • module:contrib/hstore
  • module:contrib/hstore_plperl
  • module:contrib/hstore_plpython

主要エンティティ

  • symbol:contrib/hstore/hstore_io.c#hstoreCheckKeyLen : hstore の入出力処理においてキーの長さを検証する基本実装。
  • symbol:contrib/hstore_plperl/hstore_plperl.c#hstoreCheckKeyLen : Perl のハッシュから hstore への変換時にキーの長さを検証する実装。
  • symbol:contrib/hstore_plpython/hstore_plpython.c#hstoreCheckKeyLen : Python の辞書から hstore への変換時にキーの長さを検証する実装。

エンドツーエンドの流れ

現時点では、各モジュール間における具体的な呼び出しシーケンスや、Perl/Python のデータ構造から hstore への変換処理における詳細なエンドツーエンドの制御フローは完全には明らかになっていません。一般的には、各言語のデータ構造（Perl の Hash や Python の Dict）を走査する過程で、各キーに対して hstoreCheckKeyLen が呼び出され、制限を超えている場合はエラーをスローする流れになると推測されます。

重要な不変条件

キーの長さが hstore で定義されている最大制限値（具体的なバイト数は提供されたコンテキストからは未検証）を超えないことが不変条件として要求されます。詳細な不変条件の検証ルールは現時点では未検証です。

よくある落とし穴

PL/Perl や PL/Python 側で非常に長い文字列をキーとして持つオブジェクトを生成し、それを hstore 型の列に格納しようとした場合、この検証によって変換エラーが発生します。具体的なエラーハンドリングの挙動や、マルチバイト文字の扱いに関する落とし穴については、現時点では未検証です。
