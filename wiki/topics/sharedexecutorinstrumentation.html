SharedExecutorInstrumentation

概念

SharedExecutorInstrumentation は、PostgreSQLの並列クエリ実行において、複数のワーカープロセスが収集した実行統計情報（Instrumentation）を共有メモリ上で集約・管理するための構造体です。

ヘッダーファイル src/include/executor/execParallel.h で不透明な構造体として宣言され、ソースファイル src/backend/executor/execParallel.c でその具体的なメンバが定義されています。

なぜ重要か

並列クエリの実行時、各ワーカープロセスが個別に収集した実行統計情報をリーダープロセスに集約し、EXPLAIN ANALYZE などでクエリ全体の実行結果として統合・表示できるようにするために不可欠です。共有メモリを介してリーダーとワーカー間で統計データを安全に受け渡す役割を担います。

関与するモジュール

  • module:src/backend/executor
  • module:src/include/executor

主要エンティティ

  • **symbol:src/include/executor/execParallel.h#SharedExecutorInstrumentation**

    • 構造体の型宣言。並列実行のライフサイクルを管理する ParallelExecutorInfo などの上位構造体から参照されます。

  • **symbol:src/backend/executor/execParallel.c#SharedExecutorInstrumentation**

    • 構造体の具体的な定義。以下のメンバを持ちます。 ``c struct SharedExecutorInstrumentation { int instrument_options; int instrument_offset; int num_workers; int num_plan_nodes; int plan_node_id[FLEXIBLE_ARRAY_MEMBER]; /* この後に num_plan_nodes * num_workers 個の Instrumentation オブジェクトが配置される */ }; ``

エンドツーエンドの流れ

1. **初期化**: リーダープロセスは、並列実行の初期化時（ExecInitParallelPlan など）に、並列実行プランのノード数（num_plan_nodes）およびワーカー数（num_workers）に基づき、動的共有メモリ（DSM）上に本構造体を割り当て、初期化します。 2. **書き込み**: 各ワーカープロセスは、並列実行中に自身の担当するプランノードの統計情報を、本構造体内の対応するオフセット位置に書き込みます。 3. **集約**: リーダープロセスは、ExecParallelFinish 等を通じてワーカーの終了を待機した後、本構造体から統計情報を読み取り、クエリ全体の実行結果として集約します。

*注意: 提供されたコンテキストからは、具体的な書き込み・読み取りを行う各実行ノードの個別関数や、詳細な同期処理の全容など、上記以外のエンドツーエンドのフローの詳細は判明していません。*

重要な不変条件

  • **メモリレイアウトの整合性**: plan_node_id は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、構造体の末尾には num_plan_nodes * num_workers 分の Instrumentation オブジェクトが連続して配置される必要があります。
  • **プロセス間の一貫性**: リーダープロセスとワーカープロセス間で、本構造体のメモリレイアウト（オフセット計算など）の整合性が完全に保たれている必要があります。

よくある落とし穴

  • **メモリレイアウト計算の誤り**: 本構造体は可変長配列メンバを含んでいるため、メモリ割り当てやオフセット計算の際には単純な sizeof ではなく、offsetof マクロ（例: offsetof(SharedExecutorInstrumentation, plan_node_id)）を正しく使用する必要があります。これを誤ると、メモリ破壊や不正なデータアクセス（オーバーラン）を引き起こします。
  • **共有メモリの確保失敗**: 共有メモリの確保に失敗した場合、並列実行の初期化自体が中断されます。
  • **不整合なパラメータ**: num_workers や num_plan_nodes の値に不整合がある場合、境界外アクセスが発生する危険性があります。

次に読む場所

  • **src/include/executor/execParallel.h**: 構造体の宣言と、これを利用する ParallelExecutorInfo などの関連データ構造を確認できます。
  • **src/backend/executor/execParallel.c**: 構造体の詳細な定義、offsetof を用いたメモリレイアウト計算、および ExecInitParallelPlan などの並列実行管理ロジックにおける具体的な操作方法を確認できます。
