variable

概念

variable は、ECPG（Embedded SQL for C）において、C言語のホスト変数とSQL変数との間のマッピング、およびデータ転送に必要なメタデータを保持するための構造体（struct variable）です。この構造体は、ホスト変数の型情報、メモリ上の位置（ポインタ）、配列サイズ、および対応するインジケータ変数の情報を管理します。

なぜ重要か

ECPGでは、プリプロセッサが埋め込みSQLを標準のCコードに変換し、ランタイムライブラリ（ecpglib）が実際のデータベース操作を実行します。struct variable は、プリプロセッサが解析したホスト変数の型やサイズなどの情報をランタイムに伝達し、SQL実行時に適切な型変換と安全なデータ転送を行うための架け橋として機能するため重要です。

関与するモジュール

  • module:src/interfaces/ecpg/ecpglib：ランタイムライブラリ。SQL実行時にホスト変数とのデータ転送を処理します。
  • module:src/interfaces/ecpg/preproc：プリプロセッサ。Cコード内の埋め込みSQLとホスト変数を解析します。

主要エンティティ

  • symbol:src/interfaces/ecpg/ecpglib/ecpglib_extern.h#variable：ランタイム側でホスト変数とインジケータ変数のメタデータ（型、ポインタ、サイズ、オフセットなど）を保持し、単方向連結リストを形成する構造体。
  • symbol:src/interfaces/ecpg/preproc/type.h#variable：プリプロセッサ側で定義・利用される変数関連のエンティティ（詳細な構造は提供されたコンテキストからは未詳）。

エンドツーエンドの流れ

1. **プリプロセス時**: プリプロセッサ（preproc）がCソースコード内のホスト変数定義とSQL文を解析します。 2. **コード生成**: プリプロセッサは、ホスト変数の情報を struct variable の形式でランタイムに渡すためのCコードを生成します。 3. **ランタイム実行時**: 生成されたコードが実行されると、ecpglib は struct variable の連結リスト（next ポインタによる管理）を受け取ります。 4. **データ転送**: ecpg_execute などの実行関数がこのリストを走査し、type や value ポインタ、インジケータ情報（ind_value など）を参照して、データベースとホスト変数間のデータ転送および型変換を行います。

*注意: プリプロセッサ側（preproc/type.h）における具体的な変数解析処理や、ランタイム側での詳細な関数呼び出しフローの全容は、提供された情報からは完全には明らかになっていません。*

重要な不変条件

  • **連結リスト構造**: 複数の変数は next ポインタを介して単方向連結リストとして管理されます。
  • **インジケータの対応関係**: 各ホスト変数に対して、対応するインジケータ変数（NULL値の表現などに使用）の情報を ind_ で始まるメンバ群（ind_type, ind_value など）として同一構造体内に保持できます。

よくある落とし穴

  • **NULLポインタ参照**: value や pointer が適切に初期化されず NULL のままデータ転送が試みられた場合、不正なメモリアクセス（セグメンテーションフォールトなど）が発生する可能性があります。
  • **バッファオーバーフローとデータ切り捨て**: arrsize や varcharsize に不適切な値が設定されている場合、メモリ境界を越えた書き込みや、意図しないデータの切り捨てが発生するリスクがあります。
  • **互換性の維持**: struct variable は ecpglib の内部インターフェースの一部であるため、この構造体を変更すると、プリプロセッサが生成するコードとの間でバイナリ互換性が失われる危険性があります。

  • src/interfaces/ecpg/preproc/type.h：プリプロセッサ側での変数情報の表現や型定義が記述されているヘッダーファイル。
