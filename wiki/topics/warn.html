warn

概念

warn という名前のシンボルは、PostgreSQLのソースコードにおいて、全く異なる2つの文脈で使用されています。

1. **ECPG（埋め込みSQL）テストにおける警告通知関数**: SQL実行時にカラムの切り詰め（truncation）が発生したことを検知し、標準エラー出力（stderr）に警告メッセージを出力する static void warn(void) 関数。 2. **タイムゾーン処理における警告状態の列挙型**: src/timezone/strftime.c 内で、時刻変換処理の進行状況や警告レベルを追跡・表現するための enum warn 列挙型。

なぜ重要か

  • **ECPGテストにおける重要性**: プリプロセッサが生成するCコードの期待値（expected output）として機能し、SQL実行時の警告（カラム切り詰めなど）が正しくハンドリングされ、適切な警告メッセージが出力されるかを検証するために重要です。
  • **タイムゾーン処理における重要性**: 時刻変換処理（strftime）の内部で、変換の進行状況や警告レベル（警告なし、部分的、特定対象、全体など）を追跡し、正確な時間変換を保証するために重要です。

関与するモジュール

  • module:src/interfaces/ecpg/test/expected
  • module:src/timezone

主要エンティティ

  • **symbol:src/interfaces/ecpg/test/expected/compat_oracle-char_array.c#warn**: カラム切り詰め時に "Warning: At least one column was truncated\n" を stderr に出力する static 関数。
  • **symbol:src/interfaces/ecpg/test/expected/preproc-whenever.c#warn**: 上記と同様に、カラム切り詰め時に警告を出力する static 関数。
  • **symbol:src/timezone/strftime.c#warn**: 時刻変換の警告状態を表現する列挙型（IN_NONE, IN_SOME, IN_THIS, IN_ALL）。

エンドツーエンドの流れ

提供された情報からは、これらのエンティティが関与する完全なエンドツーエンドの流れは部分的にしか判明していません。

  • **ECPGテスト**: SQL実行後に sqlca.sqlwarn[0] == 'W' などの警告状態が検知された際、あるいは sqlca 構造体の状態に基づき、警告ハンドラとして warn() 関数が呼び出され、fprintf を介して標準エラー出力に固定メッセージが出力されます。具体的な呼び出し元のコードは提供されていません。
  • **タイムゾーン処理**: strftime.c の内部処理において、時刻変換中に警告状態が変化する際に enum warn の各値（IN_NONE など）が使用されますが、具体的な遷移ロジックや呼び出し元・呼び出し先の詳細は不明です。

重要な不変条件

  • **ECPGテストの warn 関数**:
  • static 関数として定義されており、定義されたファイル内でのみ呼び出し可能です。
  • 呼び出し元は、SQL実行後の警告状態（例: sqlca の警告フラグ）を事前に検知している必要があります。
  • **タイムゾーンの enum warn**:
  • src/timezone/strftime.c の内部専用の定義であり、外部からは参照されません。

よくある落とし穴

  • **名前の衝突と混同**: warn という名前は非常に一般的であり、ECPGのテスト用関数とタイムゾーンの列挙型、さらには他のモジュール（vacuumlazy.c など）の汎用的な警告ログ出力処理と混同しないよう注意が必要です。
  • **自動生成コードの編集**: ECPGテスト内の warn 関数は、ECPGプリプロセッサが生成するCコードの期待値（expected）として定義されているため、手動で直接編集することは推奨されません。変更する場合は、プリプロセッサの生成ロジックおよびテスト期待値との整合性を保つ必要があります。
