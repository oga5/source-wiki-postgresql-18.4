pg_u_prop_uppercase

概念

pg_u_prop_uppercase は、指定された Unicode コードポイント（pg_wchar）が Unicode 規格における「大文字（Uppercase）」プロパティを保持しているかどうかを判定するための機能です。

なぜ重要か

PostgreSQL 内で Unicode 文字のプロパティを正確かつ高速に判定するために重要です。特に、頻出する ASCII 範囲（0x80 未満）の文字に対しては、ビットマスクを用いた定数時間での高速な判定を提供し、それ以上の文字に対しては範囲検索を行うことで、パフォーマンスと規格準拠を両立させています。

関与するモジュール

  • module:src/common
  • module:src/include/common

主要エンティティ

  • **symbol:src/common/unicode_category.c#pg_u_prop_uppercase** (関数)

    • 指定された pg_wchar コードポイントが大文字プロパティを持つかを判定する実体です。

  • **symbol:src/include/common/unicode_category_table.h#PG_U_PROP_UPPERCASE** (マクロ/定数)

    • ASCII 範囲の文字プロパティを判定する際に、unicode_opt_ascii テーブルに対して適用されるビットマスクです。

エンドツーエンドの流れ

1. **入力の受け取り**: 呼び出し元（例: src/common/unicode/category_test.c など）から pg_wchar code を引数として pg_u_prop_uppercase が呼び出されます。 2. **ASCII 最適化パス**: 入力 code が 0x80 未満の場合、静的テーブル unicode_opt_ascii を直接参照し、PG_U_PROP_UPPERCASE ビットマスクを用いて即座に結果（true または false）を返します。 3. **範囲検索パス**: 入力 code が 0x80 以上の場合、大文字プロパティを持つコードポイント範囲が定義された unicode_uppercase テーブルに対して range_search を実行し、該当する範囲が存在するかどうかで判定結果を返します。

※ なお、PostgreSQL 全体における文字分類処理や SQL 関数からこの判定がどのように呼び出されるかといった、より広範なエンドツーエンドの統合フローの詳細は現時点では判明していません。

重要な不変条件

  • **Unicode 規格の安定性**: Unicode 規格の安定性ポリシーに基づき、一度定義されたカテゴリおよびプロパティ値は固定されています。
  • **入力の前提**: 入力値は pg_wchar 型の有効なコードポイントであることを前提としています。

よくある落とし穴

  • **負の値や不正な入力**: 0x80 未満の範囲外（負の値など）が入力された場合の挙動については、提供されたソースコードの範囲からは明示されていません。
  • **静的テーブルの同期**: 本機能は unicode_category_table.h で定義された静的データに依存しています。Unicode バージョンの更新等に伴いテーブルを再生成・更新する際は、手動での書き換えではなく、専用の生成スクリプトを介して一貫性を維持する必要があります。
