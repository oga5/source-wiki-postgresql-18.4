PLy_elog

概念

PLy_elog は、PostgreSQL の PL/Python（Python 手続き言語）において、エラーや警告メッセージを処理・報告するための関数またはマクロです。PL/Python 実行環境における例外やエラー情報を、PostgreSQL の標準的なロギング機構（elog / ereport）へと橋渡しする役割を担っています。

なぜ重要か

PL/Python 本体だけでなく、jsonb_plpython のような関連する拡張モジュール（contrib）においても、データ変換処理などの実行中に発生したエラーを適切に捕捉し、PostgreSQL のエラーハンドリング機構に通知する必要があります。PLy_elog は、これらのモジュール間で一貫したエラー報告を行うための共通インターフェースを提供します。

エンドツーエンドの流れ

現時点では、Python の例外オブジェクトから PostgreSQL のエラーレベル（ERROR, WARNING など）への具体的なマッピングや、エラー発生時のスタックトレースの構築といった、詳細なエンドツーエンドの処理フローは完全には判明していません。

一般的な流れとしては、jsonb_plpython.c などの処理中にエラーが検出された際、PLy_elog が呼び出され、内部で Python のエラー状態が取得・整形された上で PostgreSQL のエラーログシステムに送出されます。

重要な不変条件

現時点では、PLy_elog 呼び出し時における Python のグローバルインタプリタロック（GIL）の保持状態や、PostgreSQL のメモリコンテキストに関する具体的な不変条件は判明していません。

よくある落とし穴

現時点では、PLy_elog を使用する際のエラー状態のクリア漏れや、特定の例外ハンドリングにおける具体的な落とし穴は判明していません。
