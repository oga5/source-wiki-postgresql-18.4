examine_attribute

概念

examine_attribute は、PostgreSQLの ANALYZE 処理（統計情報の収集・構築）において、特定の属性（テーブルの列、インデックス式、または拡張統計情報の式）に対する統計解析の準備と初期化を行う静的（static）関数です。

PostgreSQLのソースコード内には、役割とシグネチャが異なる2つの同名関数が存在します。 1. **テーブル列・インデックス式用**: src/backend/commands/analyze.c に定義され、テーブルの列やインデックス式を対象とします。 2. **拡張統計情報の式用**: src/backend/statistics/extended_stats.c に定義され、拡張統計情報における式（Node）を対象とします。

なぜ重要か

この関数は、統計情報収集プロセスにおいて、各属性のデータ型に応じた適切な解析ロジック（typanalyze 関数）をバインドし、統計計算に必要なメタデータ構造体（VacAttrStats）を正しく初期化する役割を担います。これにより、プランナが最適な実行計画を作成するために不可欠な統計情報の正確な生成が保証されます。

関与するモジュール

主要エンティティ

  • **symbol:src/backend/commands/analyze.c#examine_attribute**
  • テーブル列やインデックス式を解析するための関数。Relation、列番号（attnum）、インデックス式（index_expr）を受け取ります。
  • **symbol:src/backend/statistics/extended_stats.c#examine_attribute**
  • 拡張統計情報（Extended Statistics）における式を解析するための関数。式を表す Node のみを受け取ります。

エンドツーエンドの流れ

統計情報収集における examine_attribute の処理フローは、対象となるコンテキストに応じて以下の2つの経路に分かれます。

1. テーブル列およびインデックス列の解析（analyze.c）

1. do_analyze_rel から呼び出されます。 2. 対象列が分析可能か（削除済み列や仮想生成列でないか、統計ターゲット attstattarget が0より大きいかなど）を判定します。 3. pg_attribute システムカタログから情報を取得し、VacAttrStats 構造体を確保・初期化します。 4. 型固有の typanalyze 関数（定義されていない場合はデフォルトの std_typanalyze）を呼び出して、統計計算関数（compute_stats）などを設定します。

2. 拡張統計情報の式解析（extended_stats.c）

1. BuildRelationExtStatistics 内のループ処理から呼び出されます。 2. 与えられた式（Node *expr）から型情報（OID、typmod、照合順序）を抽出します。 3. TYPEOID システムキャッシュから型情報を取得し、VacAttrStats 構造体を確保・初期化します。 4. 型固有の typanalyze 関数（またはデフォルトの std_typanalyze）を呼び出し、検証を行います。

※ なお、これら2つの処理がどのように相互作用するか、あるいはシステム全体における完全なエンドツーエンドの実行フローの全容（例えば、呼び出しの正確な順序やトリガー条件など）は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **共通の不変条件**:
  • 解析対象の型情報がシステムカタログ（pg_type など）に存在しなければならず、キャッシュルックアップに失敗した場合は elog(ERROR) で処理を中断します。
  • typanalyze 関数の実行結果が不正である場合、または compute_stats が設定されていない場合は、解析対象外として NULL を返します。
  • **analyze.c 固有の不変条件**:
  • 削除済み列（attisdropped）および仮想生成列は分析対象外（NULL を返す）とします。
  • attstattarget が 0 の場合は分析をスキップします。
  • **extended_stats.c 固有の不変条件**:
  • VacAttrStats の attstattarget は -1 に初期化されます。
  • minrows が 0 以下の場合は NULL を返します。

よくある落とし穴

  • **同名関数の混同**: analyze.c と extended_stats.c に定義されている examine_attribute は、シグネチャ（引数）やメモリ管理方針が異なります。これらは別個の静的関数であるため、開発やデバッグの際に混同しないよう注意が必要です。
  • **メモリ管理の違い**: analyze.c 版は anl_context を使用してメモリ管理を行いますが、extended_stats.c 版では anl_context が NULL に設定されており、異なるメモリ管理方針がとられています。
