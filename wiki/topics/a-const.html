A_Const

概念

A_Const は、PostgreSQLのSQLパースツリーにおいてリテラル定数（数値、文字列、NULLなど）を表現するためのノード（構造体）です。

なぜ重要か

SQL文に含まれる具体的な定数値を保持し、構文解析（パーサー）から後続の意味解析フェーズ（parse_analyze）へ情報を引き渡す役割を担います。また、pg_node_attr マクロの指定により、ノードのコピー、比較、シリアライズ/デシリアライズ、およびクエリの同一性判定（Jumbling）において、カスタム処理が適用される重要なデータ構造です。

関与するモジュール

  • module:src/backend/nodes
  • module:src/include/nodes

主要エンティティ

  • symbol:src/include/nodes/parsenodes.h#A_Const: リテラル定数を表す構造体定義。
  • symbol:src/backend/nodes/readfuncs.c#_readA_Const: A_Const ノードを読み込むためのカスタム処理関数。

エンドツーエンドの流れ

1. **生成**: パーサー（src/backend/parser/gram.y）がSQL文を解析する際、または src/backend/nodes/makefuncs.c を通じて A_Const ノードが生成されます。 2. **操作と伝播**: 生成されたノードはパースツリーの一部として、copyfuncs.c や equalfuncs.c などのユーティリティによって複製や比較が行われ、後続の解析フェーズ（parse_analyze）へ渡されます。 3. **正規化**: クエリの同一性判定（queryjumblefuncs.c）において、定数値として識別・処理されます。

※なお、parse_analyze 内部における具体的な型決定や、実行プラン生成に至る詳細なエンドツーエンドの制御フローは、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **NULL値の扱い**: isnull フラグが true の場合、共用体 val（union ValUnion）の内容は無視されます。
  • **位置情報**: location フィールド（ParseLoc）はSQL文字列内のバイトオフセットを保持します。位置が不明な場合は -1 が設定されます。

よくある落とし穴

  • **位置情報の欠落**: location が -1 に設定されている場合、エラー発生時に正確なエラー位置をユーザーに提示できなくなります。
  • **フラグの不整合**: isnull フラグと実際の値の整合性が崩れると、意味解析フェーズで予期しない型チェックエラーを引き起こす可能性があります。
  • **自動生成コードとの同期**: A_Const は gen_node_support.pl による自動生成の対象です。フィールドを変更する際は、関連する funcs.c ファイル群の更新が必要になる場合があります。また、query_jumble 属性が付与されているため、クエリ正規化時の定数置換などの扱いに注意が必要です。

次に読む場所

  • src/include/nodes/parsenodes.h: A_Const 構造体および ValUnion の定義。
  • src/backend/nodes/readfuncs.c: _readA_Const を含む、ノードのデシリアライズ処理の実装。
