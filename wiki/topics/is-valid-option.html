is_valid_option

概念

is_valid_option は、PostgreSQL の外部データラッパ（FDW）モジュールにおいて、ユーザーが指定したオプション名（キーワード）が、特定のコンテキスト（カタログの OID）において有効なオプションであるかを判定するための静的（static）なヘルパー関数です。

なぜ重要か

この関数は、外部データラッパの定義や変更時（例: CREATE FOREIGN TABLE など）に、ユーザーが指定したオプションが正しいコンテキストで使われているかを検証するために重要です。無効なオプションや、適用できないコンテキスト（サーバー、外部テーブルなど）でのオプション指定を早期に検出し、エラーとして弾く役割を担っています。

関与するモジュール

本機能は、以下の2つの独立したモジュールに関与しています。

  • module:contrib/file_fdw
  • module:contrib/postgres_fdw

主要エンティティ

本トピックに関与する主要なエンティティは以下の通りです。

  • **symbol:contrib/file_fdw/file_fdw.c#is_valid_option**: file_fdw モジュール内の実装。valid_options 配列（FileFdwOption 構造体）を走査してオプションを検証します。
  • **symbol:contrib/postgres_fdw/option.c#is_valid_option**: postgres_fdw モジュール内の実装。postgres_fdw_options 配列（PgFdwOption 構造体）を走査してオプションを検証します。

エンドツーエンドの流れ

1. **バリデータの呼び出し**: ユーザーが FDW 関連のオブジェクトを定義または変更する際、各モジュールのバリデータ関数（file_fdw_validator または postgres_fdw_validator）が呼び出されます。 2. **オプションの検証**: バリデータ内部から、検証対象のオプション名とコンテキスト（カタログ OID）を引数として is_valid_option が呼び出されます。 3. **配列の線形探索**:

  • file_fdw では valid_options 配列を走査します。
  • postgres_fdw では postgres_fdw_options 配列を走査します。

    • 4. **判定と返却**: オプション名（strcmp による一致）とコンテキスト OID の両方が一致する定義が見つかれば true を返し、見つからなければ false を返します。

なお、これら以外の詳細なエンドツーエンドの制御フロー（例えば、検証失敗時のエラーハンドリングの具体的な伝播経路など）は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **配列の終端**:
  • file_fdw の valid_options 配列、および postgres_fdw の postgres_fdw_options 配列は、末尾の要素のオプション名（optname または keyword）が NULL で終端されていなければなりません。
  • **初期化状態**:
  • postgres_fdw においては、postgres_fdw_options 配列が初期化済みであることが Assert によって検証されます。

よくある落とし穴

  • **NULL ポインタの引き渡し**: 引数として渡されるオプション名（option または keyword）が NULL である場合、内部の strcmp 呼び出しによりセグメンテーション違反が発生する可能性があります。
  • **配列の終端忘れ**: 静的配列の末尾に NULL 終端がない場合、探索ループがメモリ境界を越えてアクセスする危険性があります。
  • **同名関数の混同**: file_fdw と postgres_fdw にそれぞれ同名の is_valid_option 関数が存在しますが、これらは完全に独立した static 関数であり、参照するオプション配列も異なります。一方を修正する際にもう一方と混同しないよう注意が必要です。
