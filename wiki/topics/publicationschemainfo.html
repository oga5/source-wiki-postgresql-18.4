PublicationSchemaInfo

概念

PublicationSchemaInfo は、PostgreSQL においてパブリケーション（論理レプリケーションの公開対象）とスキーマ（ネームスペース）の関連付け情報を表す概念、およびそれを扱うためのデータ構造・関数群です。システムカタログ pg_publication_namespace に格納されているパブリケーションとスキーマのメタデータを解決・保持するために使用されます。

なぜ重要か

PostgreSQL の論理レプリケーションでは、特定のスキーマに属するすべてのテーブルをパブリケーションの対象として指定できます。この関連付け情報をシステムカタログから正確に取得し、オブジェクトの依存関係解決や pg_dump によるバックアップ・リストア処理へ連携するために、この情報構造と解決ロジックが不可欠となります。

関与するモジュール

  • **module:src/backend/catalog**: システムカタログ（pg_publication_namespace）からの情報取得や、オブジェクトアドレス（ObjectAddress）を介した名前解決を担当します。
  • **module:src/bin/pg_dump**: データベースのダンプ出力時に、パブリケーションとスキーマの関連付け情報を保持・出力するために使用します。

主要エンティティ

  • **symbol:src/backend/catalog/objectaddress.c#getPublicationSchemaInfo** (関数)

    • 指定された ObjectAddress（pg_publication_namespace の OID）から、対応するパブリケーション名とスキーマ名をシステムキャッシュ経由で解決するバックエンド関数です。

  • **symbol:src/bin/pg_dump/pg_dump.h#PublicationSchemaInfo** (構造体/型)

    • pg_dump 内部でパブリケーションとスキーマの関連付け情報を管理するために定義されているデータ構造です（詳細な構造定義は提供されたコンテキストからは未解明）。

エンドツーエンドの流れ

1. **バックエンドにおける情報解決**:

  • getPublicationSchemaInfo が ObjectAddress を入力として呼び出されます。
  • PUBLICATIONNAMESPACE システムキャッシュを検索し、pg_publication_namespace のタプルを取得します。
  • タプルからパブリケーション OID とスキーマ OID を抽出し、それぞれ get_publication_name および get_namespace_name を用いて文字列名（pubname, nspname）に変換して呼び出し元に返します。

2. **クライアント（pg_dump）における処理**:

  • pg_dump 実行時に、スキーマ単位のパブリケーション情報をダンプするために PublicationSchemaInfo 構造体が使用されますが、具体的なダンプ生成やリストア時の一連のエンドツーエンドの流れは、提供されたコンテキストからは未解明です。

重要な不変条件

  • **OID の有効性**: getPublicationSchemaInfo に渡される object->objectId は、pg_publication_namespace カタログの有効な OID でなければなりません。
  • **メモリ管理の責任**: getPublicationSchemaInfo の呼び出し側は、出力引数として返される pubname および nspname のメモリ（pfree による解放）を管理する責任を負います。
  • **pg_dump 側の不変条件**: pg_dump.h における PublicationSchemaInfo のライフサイクルや整合性制約については、提供されたコンテキストからは未解明です。

よくある落とし穴

  • **存在しないオブジェクトへのアクセス**: getPublicationSchemaInfo の呼び出し時に missing_ok が false に設定されている場合、対象の OID がカタログに存在しないと ERROR が発生し、処理が中断されます。
  • **メモリリーク**: getPublicationSchemaInfo が正常に true を返した後に、呼び出し側で取得したパブリケーション名やスキーマ名のメモリ解放を忘れると、メモリリークの原因となります。

  • **src/bin/pg_dump/pg_dump.h**: PublicationSchemaInfo 構造体の定義と、それが pg_dump 内でどのように宣言されているかを確認できます。
