JumbleState

概念

JumbleState は、PostgreSQL においてクエリの構造的な同一性を識別するためのフィンガープリント（Query ID）を生成するプロセス（クエリジャンブル）において、その作業状態や中間データを保持するための構造体です。クエリツリーを走査しながら、ノード情報や定数の位置情報を収集・記録するコンテキストとして機能します。

なぜ重要か

クエリの正規化や統計情報の収集（例えば pg_stat_statements 拡張など）において、具体的な定数リテラルの値が異なっていても、構造的に同じクエリであれば同一の Query ID として識別する必要があります。JumbleState は、クエリツリーから定数リテラルの位置（clocations）を抽出してマスクし、構造情報のみからハッシュ値を計算するための中核的なデータ構造として重要です。

関与するモジュール

  • module:src/include/nodes
  • module:src/include/parser

主要エンティティ

  • **symbol:src/include/nodes/queryjumble.h#JumbleState**: 構造体の実体定義。jumble バッファ、定数位置情報（clocations）、NULLノードの遅延処理用カウンタ（pending_nulls）などのメンバを保持します。
  • **symbol:src/include/parser/analyze.h#JumbleState**: post_parse_analyze_hook_type などの解析フックで参照されるための typedef 宣言。実体は nodes/queryjumble.h に定義されています。

エンドツーエンドの流れ

1. クエリの解析完了後、post_parse_analyze_hook などを介して JumbleState が初期化され、処理に渡されます。 2. クエリツリー（Query）の再帰的な走査が行われ、ノードの型や構造情報が JumbleState の jumble バッファに書き込まれます。 3. 走査中に定数リテラルが検出されると、その位置と長さが clocations 配列に記録されます。 4. NULL ノードの連続は pending_nulls を用いて遅延処理され、バッファへの書き込みが最適化されます。 5. 最終的なバッファ内容に基づいてクエリのフィンガープリント（Query ID）が計算され、定数情報を除外した正規化クエリの生成などに利用されます。

*注意: 走査処理の具体的な関数群や、ハッシュ計算アルゴリズムの厳密な実装詳細など、エンドツーエンドの完全な制御フローは提供された情報からは未だすべて明らかになっていません。*

重要な不変条件

  • **定数マスクの整合性**: clocations 配列は、クエリ内の定数リテラルを正確にマスクするために、検出された定数の位置と長さを正しく保持しなければなりません。
  • **NULLノードの遅延フラッシュ**: pending_nulls はハッシュ化の効率化のための遅延処理機構であり、後続の追加処理や最終ハッシュ計算の前に適切にフラッシュされる必要があります。
  • **パラメータ再割り当ての制御**: has_squashed_lists が真（true）の場合、リスト圧縮による番号の欠落を回避するため、クエリが提供するパラメータ番号を無視して再割り当てを行う必要があります。
  • **アサーション検証**: USE_ASSERT_CHECKING が有効な場合、total_jumble_len を用いてバッファに追加されたバイト数の整合性が検証されます。

よくある落とし穴

  • **定義と宣言の分離**: JumbleState の実体定義は src/include/nodes/queryjumble.h にありますが、src/include/parser/analyze.h でも宣言されています。構造体に変更を加える場合は、両方のファイルの影響を確認する必要があります。
  • **NULLポインタ参照**: フック関数（post_parse_analyze_hook など）が呼び出された際、JumbleState が NULL であるか、あるいは適切に初期化されていない状態で参照された場合の挙動は呼び出し側の実装（pg_stat_statements など）に依存するため、ハンドリングに注意が必要です。
  • **統計情報コマンドとの混同**: src/backend/commands/analyze.c（テーブルの統計情報を収集する ANALYZE コマンド）は、クエリのフィンガープリント生成を行う本機能とは無関係であるため、混同しないようにしてください。
