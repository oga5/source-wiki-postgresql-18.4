cmpOffsetNumbers

概念

cmpOffsetNumbers は、PostgreSQLのページ内タプル位置を示す識別子である OffsetNumber 型の値を昇順にソートするための比較関数です。主に qsort のコールバック関数として使用されます。

なぜ重要か

OffsetNumber は16ビットの符号なし整数として定義されており、インデックスやテーブルのページ内でのタプル位置を特定するために頻繁に使用されます。複数の OffsetNumber をソートして処理する際（例えば、SP-GiSTの挿入処理やバキューム処理など）、標準的な比較ロジックを提供することで、一貫した順序でのデータ処理を可能にします。

関与するモジュール

  • module:src/backend/access/spgist

主要エンティティ

  • symbol:src/backend/access/spgist/spgdoinsert.c#cmpOffsetNumbers: SP-GiSTの挿入処理において、OffsetNumber を比較するために定義されている静的関数。
  • symbol:src/backend/access/heap/vacuumlazy.c#cmpOffsetNumbers: vacuumlazy.c 内に存在する同名の比較関数。

エンドツーエンドの流れ

spgdoinsert.c における具体的な流れは以下の通りです。

1. spgdoinsert.c 内の処理において、OffsetNumber の配列をソートする必要が生じます。 2. qsort の比較コールバックとして cmpOffsetNumbers が渡されます。 3. cmpOffsetNumbers は、与えられた2つのポインタを OffsetNumber * にキャストし、それぞれの値を pg_cmp_u16 を用いて比較します。 4. 比較結果（負、0、正の整数）を返し、qsort によって配列が昇順に整列されます。

なお、vacuumlazy.c における具体的なエンドツーエンドの流れや、これら2つのモジュール間での直接的な連携フローについては、現時点では詳細が提供されておらず不明です。

重要な不変条件

  • **型の前提**: OffsetNumber が16ビット符号なし整数（uint16）であることを前提としており、内部で pg_cmp_u16 を使用して比較を行います。
  • **入力の妥当性**: 引数として渡される const void * ポインタは、有効な OffsetNumber 型のメモリ領域を指していなければなりません。

よくある落とし穴

  • **NULLポインタの引き渡し**: 入力ポインタ a または b が NULL である場合、関数内でデリファレンスされるため、未定義の動作（セグメンテーションフォールトなど）を引き起こします。
  • **モジュール間の独立性**: spgdoinsert.c 内の cmpOffsetNumbers は static 関数として定義されています。vacuumlazy.c に存在する同名関数とは、名前と目的が同一ですが、直接的なコード共有や依存関係はありません。
