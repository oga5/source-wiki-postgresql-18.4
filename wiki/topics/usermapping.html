UserMapping

概念

UserMapping は、PostgreSQLの外部データラッパー（FDW）において、ローカルデータベースのユーザー（ロール）と、外部サーバー（Foreign Server）上の接続・認証用ユーザー情報を紐付けるためのメタデータ、およびそれを表すデータ構造です。

なぜ重要か

外部サーバーへの接続や認証パラメータ（パスワードなど）を決定するために不可欠な要素です。contrib/postgres_fdw や contrib/dblink などのモジュールが、リモートサーバーに対して適切な権限と認証情報を用いて安全に接続を確立するために使用されます。

関与するモジュール

主要エンティティ

  • symbol:src/include/foreign/foreign.h#UserMapping: ユーザーマッピング情報を保持するメモリ上の構造体。
  • symbol:src/backend/commands/foreigncmds.c#CreateUserMapping: CREATE USER MAPPING SQL文の実行時に、システムカタログ pg_user_mapping にマッピングを作成する関数。
  • symbol:src/backend/foreign/foreign.c#GetUserMapping: 指定されたユーザーと外部サーバーに対応するマッピング情報をシステムカタログから取得し、UserMapping 構造体として返却する関数。

エンドツーエンドの流れ

1. **定義と作成**:

  • 管理者またはユーザーが CREATE USER MAPPING SQL文を実行すると、src/backend/tcop/utility.c を経由して CreateUserMapping が呼び出されます。
  • CreateUserMapping は、対象サーバーの存在と権限を検証し、FDWのバリデーター関数を用いてオプションを検証・変換（transformGenericOptions）した上で、システムカタログ pg_user_mapping に新しいタプルを挿入します。

    • 2. **接続時の取得と利用**:

  • postgres_fdw や dblink が外部サーバーへ接続する際、GetUserMapping を呼び出します。
  • GetUserMapping は USERMAPPINGUSERSERVER システムキャッシュを検索し、該当するユーザーのマッピングを取得します。個別ユーザーのマッピングが存在しない場合は、PUBLIC（InvalidOid）のマッピングを再検索します。
  • 取得したカタログ情報から UserMapping 構造体を構築し、オプションを untransformRelOptions で DefElem リストに変換して呼び出し元に返却します。
  • 呼び出し元（FDW等）はこのオプションを解析し、リモート接続の確立に使用します。

重要な不変条件

  • **一意性の制約**: 同一のユーザーと同一の外部サーバーの組み合わせに対するユーザーマッピングは、システム内で一意でなければなりません（CreateUserMapping 内で重複チェックが行われます）。
  • **フォールバック動作**: 特定のユーザーに対するマッピングが存在しない場合、InvalidOid を用いて PUBLIC ユーザー向けのマッピングが検索されます。
  • **メモリ管理**: GetUserMapping が返却する UserMapping 構造体はシステムキャッシュのコピーであり、呼び出し側で適切にメモリ管理（解放など）を行う必要があります。また、構造体への変更はカタログへ自動的に反映されません。

よくある落とし穴

  • **重複エラー**: 同一ユーザー・サーバーの組み合わせが既に存在する場合、IF NOT EXISTS 指定がない限り CreateUserMapping は ERRCODE_DUPLICATE_OBJECT エラーを発生させます。
  • **未定義エラー**: 指定されたユーザーおよび PUBLIC のいずれにもマッピングが定義されていない場合、GetUserMapping は ERRCODE_UNDEFINED_OBJECT エラーを発生させます。
  • **カタログスキーマへの依存**: UserMapping は pg_user_mapping カタログに直接依存しているため、カタログスキーマが変更された場合は SysCacheGetAttr の属性インデックス（Anum_pg_user_mapping_umoptions など）の整合性を保つ必要があります。
  • **拡張機能との非互換**: ユーザーマッピングは現在、拡張機能（Extension）のメンバーとして扱われない仕様となっています。
