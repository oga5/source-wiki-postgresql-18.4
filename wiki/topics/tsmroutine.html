TsmRoutine

概念

TsmRoutine は、PostgreSQL の TABLESAMPLE 句で使用されるサンプリングメソッド（例: Bernoulli, System など）のインターフェースを定義する構造体です。プランナやエグゼキュータが、具体的なサンプリングアルゴリズムを抽象化された共通のインターフェースを介して利用できるように設計されています。

なぜ重要か

TsmRoutine は、TABLESAMPLE の拡張性を担保する中心的な API です。新しいサンプリングメソッドを追加する際、コアコード（プランナやエグゼキュータ）を直接修正することなく、この構造体を正しく実装・登録するだけで独自のサンプリングロジックを PostgreSQL に統合できるようになります。

主要エンティティ

  • **symbol:src/include/access/tsmapi.h#TsmRoutine**

    • サンプリングメソッドのインターフェースを定義する構造体。各実行フェーズに対応する関数ポインタ（SampleScanGetSampleSize、BeginSampleScan、NextSampleTuple など）や、パラメータ型リスト、クエリ/スキャンをまたぐ再現性の有無を示すフラグを保持します。

  • **symbol:src/backend/access/tablesample/tablesample.c#GetTsmRoutine**

    • 指定されたサンプリングメソッドのハンドラ OID (tsmhandler) を受け取り、対応する TsmRoutine 構造体を取得・検証する関数。

エンドツーエンドの流れ

1. **ハンドラの取得と検証**: GetTsmRoutine が呼び出されると、内部で OidFunctionCall1 を使用して引数 NULL でハンドラ関数を実行します。返された Datum を TsmRoutine * にキャストし、IsA マクロによる型チェックを行って検証します。 2. **プランニングフェーズ**: プランナは TsmRoutine の SampleScanGetSampleSize 関数ポインタを呼び出し、サンプリング後の推定ページ数とタプル数を算出してコスト見積もりに利用します。 3. **実行（エグゼキュータ）フェーズ**:

  • InitSampleScan で状態を初期化します（定義されている場合）。
  • BeginSampleScan でスキャンを開始します。
  • NextSampleBlock で対象ブロックを順次取得します（NULL の場合は通常のシーケンシャルスキャンにフォールバックします）。
  • NextSampleTuple で対象タプルを順次取得します。
  • EndSampleScan によりリソースを解放します（定義されている場合）。

*注意: 呼び出し元（nodeSamplescan.c や allpaths.c など）から GetTsmRoutine が呼び出される具体的なタイミングや、詳細なエンドツーエンドの制御フローの全容は、提供されたコンテキストからは完全には判明していません。*

重要な不変条件

  • **初期化の要件**: TsmRoutine 構造体の初期化には必ず makeNode(TsmRoutine) を使用しなければなりません。これにより、将来的なフィールド追加時のメモリ安全性が保証されます。
  • **型チェックの通過**: GetTsmRoutine が返す構造体は、IsA(routine, TsmRoutine) による型チェックを通過する必要があります。
  • **必須関数ポインタ**: BeginSampleScan、NextSampleTuple、SampleScanGetSampleSize は必須（非NULL）であり、これらが欠落している場合の動作は未定義です。
  • **オプション関数ポインタ**: InitSampleScan、NextSampleBlock、EndSampleScan は NULL を許容します。

よくある落とし穴

  • **必須関数の未実装**: BeginSampleScan、NextSampleTuple、SampleScanGetSampleSize のいずれかを NULL のままにすると、動作が未定義となりシステムがクラッシュするなどの問題を引き起こす可能性があります。
  • **不適切な初期化**: makeNode(TsmRoutine) を使用せずに構造体を初期化した場合、将来的に構造体に新しいフィールドが追加された際にメモリ安全性が損なわれるリスクがあります。
  • **ハンドラ関数の不正な戻り値**: ハンドラ関数が NULL を返したり、TsmRoutine 以外の型を返したりした場合、GetTsmRoutine は ERROR を発生させて処理を中断します。
