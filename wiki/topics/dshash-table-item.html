dshash_table_item

概念

dshash_table_item は、動的共有メモリ（DSA: Dynamic Shared Memory Area）上に構築されたハッシュテーブル（dshash）において、個々のエントリを管理するための内部ヘッダー構造体です。

この構造体はハッシュテーブルのバケット内連結リストのノードとして機能し、実際のユーザー定義データエントリの直前に配置されます。

なぜ重要か

動的共有メモリ上で動作するハッシュテーブルにおいて、各エントリのメタデータを管理し、バケット内の連結リストを構築するために不可欠な要素です。

1. **効率的な走査と比較**: 構造体内にハッシュ値（hash）をキャッシュすることで、衝突解決時のキー比較コストを削減します。 2. **共有メモリの抽象化**: 共有メモリ上のアドレスを直接的なポインタではなく dsa_pointer を使用して管理することで、プロセス間での安全なリンクを可能にします。 3. **密なメモリ配置**: ユーザー定義データとヘッダーをメモリ上で隣接させ、マクロによる高速な相互変換を実現しています。

主要エンティティ

  • **symbol:src/backend/lib/dshash.c#dshash_table_item**

    • 具体的な構造体定義です。以下のメンバを持ちます。

  • dsa_pointer next: 同一バケット内の次のアイテムへの共有メモリポインタ。
  • dshash_hash hash: 再計算を避けるために保持されるハッシュ値。
  • **symbol:src/include/lib/dshash.h#dshash_table_item**

    • 外部モジュール向けに公開されている不透明型（Opaque Type）としての宣言です。具体的なメンバ構成は隠蔽されています。

エンドツーエンドの流れ

1. **バケット操作の開始**: dshash.c 内のバケット操作関数（find_in_bucket や insert_into_bucket など）が呼び出されます。 2. **リストの走査**: dshash_table_item の next（dsa_pointer）を辿り、バケット内の連結リストを走査します。 3. **ハッシュ値の比較**: 走査中、まず dshash_table_item 内の hash フィールドを比較し、一致する場合のみ実際のキー比較を行います。 4. **データへのアクセス**:

  • アイテムからユーザーデータへアクセスする際は、ENTRY_FROM_ITEM マクロ等を使用して dshash_table_item の直後のアドレスを計算します。
  • 逆に、ユーザーデータからアイテムヘッダーを特定する際は、ITEM_FROM_ENTRY マクロ（dshash.c 内）を使用して逆算します。

※なお、ハッシュテーブル全体の初期化から破棄に至る完全なライフサイクルや、並行アクセス制御（ロック等）の具体的なエンドツーエンドのフローは、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **メモリ配置の整合性**: ユーザーデータは dshash_table_item の直後に配置されるため、メモリレイアウトは必ず MAXALIGN で調整されていなければなりません。
  • **ポインタの有効性**: next フィールドは dsa_pointer であり、常に共有メモリ上の有効な領域を指している必要があります。

よくある落とし穴

  • **構造体変更時のアライメント崩れ**: dshash_table_item のサイズやフィールドを変更すると、ENTRY_FROM_ITEM や ITEM_FROM_ENTRY マクロによる MAXALIGN を用いたオフセット計算の整合性が崩れ、メモリ破壊や不正アクセスを引き起こす危険性があります。
  • **共有メモリの枯渇**: 共有メモリ（DSA）が枯渇した場合、本構造体を含むエントリの新規確保に失敗します。
  • **無効なポインタ参照**: dsa_pointer が破損または無効な場合、リスト走査時に不正なメモリ参照が発生します。

次に読む場所

  • **src/backend/lib/dshash.c**: 構造体の具体的な定義、ENTRY_FROM_ITEM や ITEM_FROM_ENTRY マクロ、および find_in_bucket や insert_into_bucket などの具体的なバケット操作関数の実装が存在します。
  • **src/include/lib/dshash.h**: 外部公開用のヘッダーであり、不透明型としての宣言や関連する dshash_table などのインターフェイスが定義されています。
