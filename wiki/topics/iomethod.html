IoMethod

概念

IoMethod は、PostgreSQL の非同期 I/O (AIO) サブシステムにおいて、I/O 操作を実行するために使用する具体的な実装方式を選択・定義するための列挙型（enum）です。

なぜ重要か

PostgreSQL が動作する環境やコンパイル時の設定に応じて、最適な I/O 処理方式を柔軟に切り替えるために重要です。これにより、同期的な処理、ワーカースレッドを用いた非同期処理、あるいは Linux カーネルの高性能な io_uring インターフェースを用いた非同期処理を、統一されたインターフェースの下で管理できます。

関与するモジュール

  • module:src/backend/storage/aio
  • module:src/include/storage

主要エンティティ

  • symbol:src/include/storage/aio.h#IoMethod: I/O 実行方式（IOMETHOD_SYNC, IOMETHOD_WORKER, IOMETHOD_IO_URING）を定義する列挙型。
  • symbol:src/backend/storage/aio/aio.c#io_method: IoMethod に基づき、実際の I/O 処理を制御・管理する変数またはロジック。

エンドツーエンドの流れ

提供されたソース情報からは、IoMethod がシステム起動時やクエリ実行時にどのように決定され、個々の I/O 要求へ適用されるかという具体的なエンドツーエンドのフローは判明していません。

ただし、I/O 操作のコンテキストを保持する PgAioHandle 構造体がこの IoMethod を使用して実装方式を管理していることが分かっています。

重要な不変条件

  • **コンパイル時条件**: IOMETHOD_IO_URING は、コンパイル時に IOMETHOD_IO_URING_ENABLED が定義されている場合にのみ利用可能です。
  • **実行環境の制限**: io_uring は EXEC_BACKEND 環境と互換性がないため、この環境下では IOMETHOD_IO_URING を選択することはできません。

よくある落とし穴

  • EXEC_BACKEND が有効な環境において IOMETHOD_IO_URING を使用しようとすると、互換性の問題により失敗します。
  • 新しい I/O 方式を追加する場合、src/include/storage/aio.h の IoMethod 列挙型にメンバーを追加するだけでなく、src/backend/storage/aio/aio.c などの関連ソースコードに対応する処理ロジックを適切に実装する必要があります。

次に読む場所

  • src/include/storage/aio.h: IoMethod 列挙型および関連する PgAioHandle の定義。
  • src/backend/storage/aio/aio.c: IoMethod に基づく I/O 処理の実装。
