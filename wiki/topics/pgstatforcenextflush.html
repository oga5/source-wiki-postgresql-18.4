pgStatForceNextFlush

概念

pgStatForceNextFlush は、PostgreSQLの統計情報（statistics collector）サブシステムにおいて、保留中の統計データ（累積されたメトリクスなど）を次回のアクティビティ時に強制的にフラッシュ（書き出し）させるための制御機構です。

なぜ重要か

通常、統計情報のフラッシュはパフォーマンス向上のために遅延評価や一定の間隔をおいて行われます。しかし、テストや即時反映が求められる特定の状況において、このフラッシュを即座にトリガーする手段が必要となります。pgStatForceNextFlush はその制御フラグとして機能し、SQL関数 pg_stat_force_next_flush() を通じて外部からこの挙動を制御できるようにします。

関与するモジュール

  • module:src/backend/utils/activity: 統計情報の収集と管理のコアロジックを担当します。
  • module:src/backend/utils/adt: SQLから呼び出し可能な組み込み関数（システム管理関数など）を提供します。

主要エンティティ

  • symbol:src/backend/utils/activity/pgstat.c#pgStatForceNextFlush: 統計情報の次回フラッシュを強制するかどうかを保持する内部フラグ（変数）です。
  • symbol:src/backend/utils/adt/pgstatfuncs.c#pg_stat_force_next_flush: SQLから呼び出し可能な関数であり、内部で pgStatForceNextFlush フラグを有効化します。

エンドツーエンドの流れ

1. ユーザーまたはシステムプロセスが SQL から pg_stat_force_next_flush() を実行します。 2. src/backend/utils/adt/pgstatfuncs.c 内の同名関数が呼び出され、src/backend/utils/activity/pgstat.c で定義されている pgStatForceNextFlush フラグを true に設定します。 3. その後の統計情報更新タイミングにおいて、pgStatForceNextFlush が true であることが検知されると、通常の間隔制限を無視して統計データが即座にフラッシュされます。 4. フラッシュ完了後、このフラグは false にリセットされます。

*注意: 提供されたコンテキストからは、フラッシュが実行される具体的なトリガー関数やリセットの厳密なタイミングなど、詳細なエンドツーエンドの制御フローの全貌は未確定です。*

重要な不変条件

  • pgStatForceNextFlush が true に設定された場合、次回の統計情報書き出し処理において、時間ベースの抑制（スロットリング）をバイパスしてフラッシュが実行されなければなりません。
  • フラッシュ処理が正常に完了、あるいはスキップされた後、フラグは適切に初期状態（false）に戻る必要があります。

*注意: 内部的な排他制御や、複数セッションから同時に呼び出された場合の排他ロックの不変条件については、提供された情報からは未確定です。*

よくある落とし穴

  • **頻繁な呼び出しによるパフォーマンス低下**: pg_stat_force_next_flush() を頻繁に呼び出すと、統計情報の書き出し処理が多発し、I/Oや共有メモリへの書き込みオーバーヘッドが増大する可能性があります。
  • **非同期性の誤解**: フラグをセットした瞬間に同期的にフラッシュが完了するわけではなく、あくまで「次回のフラッシュ処理時」に強制されるため、極めて短い時間差での即時反映を期待すると、タイミングによってはわずかな遅延が生じる可能性があります。
