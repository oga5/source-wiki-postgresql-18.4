ECPGdebug

概念

ECPGdebug は、PostgreSQLの埋め込みSQL（ECPG）クライアントライブラリ（ecpglib）において、ランタイムデバッグログ出力を有効化・制御するためのAPI関数です。開発者やテストスイートが、接続処理やクエリ実行などの内部動作を追跡するために、ログの出力先ストリームとデバッグレベルを設定する目的で使用されます。

なぜ重要か

ECPGアプリケーションのデバッグや回帰テストにおいて、内部動作の可視化は不可欠です。ECPGdebug は以下の理由で重要な役割を果たします。

  • **ランタイム動作の可視化**: 接続、プリペアドステートメント、カーソル操作などの内部状態を追跡可能にします。
  • **回帰テストのサポート**: デバッグレベルに100を超える値を指定することで「内部回帰テストモード」が有効になり、テスト結果の検証を容易にします。
  • **スレッドセーフな設計**: マルチスレッド環境下でも、ミューテックスを用いて安全にデバッグ設定を切り替えることができます。

関与するモジュール

  • module:src/interfaces/ecpg/ecpglib: ECPGdebug の実体および排他制御、ログ出力処理（misc.c）を実装しています。
  • module:src/interfaces/ecpg/test/expected: ECPGdebug を呼び出して動作検証を行う、膨大な自動テスト群の期待される出力結果を定義しています。

主要エンティティ

  • **symbol:src/interfaces/ecpg/ecpglib/misc.c#ECPGdebug**: デバッグレベルと出力ストリームを設定する関数本体。
  • **simple_debug**: デバッグレベルを保持する内部グローバル変数。
  • **debugstream**: ログ出力先となる FILE ポインタ。
  • **ecpg_internal_regression_mode**: 内部回帰テストモードの有効化を示すフラグ。
  • **debug_init_mutex / debug_mutex**: 設定変更時およびログ出力時の競合を防ぐためのミューテックス。
  • **src/interfaces/ecpg/test/expected/*.c 内の ECPGdebug 参照**: テストコードからデバッグ機能を有効化するための呼び出し。

エンドツーエンドの流れ

1. **初期化と呼び出し**: アプリケーションまたはテストコード（例: connect-test1.c など）が、デバッグレベル n と出力先ストリーム dbgs を指定して ECPGdebug(n, dbgs) を呼び出します。 2. **排他制御の獲得**: debug_init_mutex と debug_mutex を順にロックし、他のスレッドによる設定変更やログ出力を一時的にブロックします。 3. **モード判定と設定**:

  • n > 100 の場合、ecpg_internal_regression_mode を true に設定し、デバッグレベルを n - 100 として simple_debug に格納します。
  • それ以外の場合、simple_debug に n を直接格納します。
  • debugstream に dbgs を割り当てます。

    • 4. **設定完了の通知**: debug_mutex を解放した後、ecpg_log を呼び出して新しいデバッグレベルが設定されたことをログに出力します。 5. **クリーンアップ**: debug_init_mutex を解放して処理を完了します。 6. **以降のログ出力**: ECPG内部の各処理で ecpg_log が呼び出された際、設定された simple_debug レベルと debugstream に基づいてログが書き出されます。

重要な不変条件

  • **スレッドセーフティ**: デバッグ設定の変更（ECPGdebug の実行）およびログ出力（ecpg_log の実行）は、debug_mutex および debug_init_mutex によって保護され、マルチスレッド環境下でも一貫性が保たれます。
  • **回帰テストモードの閾値**: デバッグレベル n が 100 を超える場合は、必ず ecpg_internal_regression_mode が true になり、実際のデバッグレベルは n - 100 に制限されます。

よくある落とし穴

  • **無効なファイルポインタ**: dbgs に無効な FILE ポインタ（すでにクローズされたものなど）を渡すと、内部の fprintf 呼び出し時に未定義動作やクラッシュを引き起こす可能性があります。
  • **パフォーマンスへの影響**: ECPGdebug を頻繁に呼び出したり、高頻度でログを出力させたりすると、ミューテックスの競合や ecpg_log 内部での malloc 処理により、アプリケーションのパフォーマンスに影響を与える可能性があります。
