interval_cmp_upper

概念

interval_cmp_upper は、PostgreSQLのGiSTインデックス処理などにおいて、区間（インターバル）の上限値（upper）に基づいて2つの要素を比較し、ソート順序を決定するための比較関数です。

同名の関数が src/backend/access/gist/gistproc.c と src/backend/utils/adt/rangetypes_gist.c の2箇所に存在しますが、これらはシグネチャや実装が異なる別個の関数として定義されています。

なぜ重要か

GiSTインデックスのページ分割（picksplit）アルゴリズムなどにおいて、データを特定の区間に分割してソートする処理が必要となります。interval_cmp_upper は、SplitInterval 構造体などの上限値（upper フィールド）を基準に qsort 等で高速にソートを行うための比較ロジックを提供し、インデックスの再配置や分割の最適化に寄与します。

関与するモジュール

  • module:src/backend/access/gist
  • module:src/backend/utils/adt

主要エンティティ

  • symbol:src/backend/access/gist/gistproc.c#interval_cmp_upper: SplitInterval 構造体の upper フィールド（float8 型）を float8_cmp_internal を用いて比較する静的関数。
  • symbol:src/backend/utils/adt/rangetypes_gist.c#interval_cmp_upper: 同名の別実装関数（詳細なシグネチャや実装は本コンテキストでは未定義）。

エンドツーエンドの流れ

1. GiSTインデックスの picksplit アルゴリズムなどのソート処理において、qsort の比較関数として interval_cmp_upper が呼び出されます。 2. src/backend/access/gist/gistproc.c の実装では、入力された const void * 引数（i1, i2）を const SplitInterval * にキャストします。 3. それぞれの構造体から upper フィールド（float8 型）の値を取得します。 4. float8_cmp_internal を呼び出して2つの値を比較し、その結果（upper1 < upper2 なら負、upper1 > upper2 なら正、等しければ 0）を返します。

なお、src/backend/utils/adt/rangetypes_gist.c 側の interval_cmp_upper を含めた全体的なエンドツーエンドの統合フローや、具体的な呼び出しシーケンスの全容は現時点では完全には判明していません。

重要な不変条件

  • src/backend/access/gist/gistproc.c の実装において、入力ポインタ i1 および i2 は、有効な SplitInterval 構造体を指している必要があります。
  • 比較対象となる SplitInterval 構造体には、float8 型の upper フィールドが存在しなければなりません。

よくある落とし穴

  • **同名関数の混同**: src/backend/access/gist/gistproc.c と src/backend/utils/adt/rangetypes_gist.c に同名の interval_cmp_upper 関数が存在しますが、これらはシグネチャが異なる別個の実装です。コードの変更や参照の際には、どちらのモジュールの関数であるかを混同しないよう注意が必要です。
  • **NULLポインタ**: 引数 i1 または i2 に NULL が渡された場合、関数内で安全なチェックは行われないため、セグメンテーション違反（未定義の動作）を引き起こします。
  • **NaNの扱い**: upper フィールドの値が NaN である場合の比較挙動は、内部で呼び出される float8_cmp_internal の実装に依存します。
