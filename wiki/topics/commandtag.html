CommandTag

概念

CommandTag は、PostgreSQLにおいて実行されたSQLコマンドの種類（例: INSERT, SELECT, CREATE TABLE など）を識別するための列挙型（enum）です。パースツリーからコマンドの種類を判定し、クライアントへのコマンド完了通知（CommandComplete）やイベントトリガー、DDLの逆パース（deparse）などの処理で利用されます。

なぜ重要か

  • **クライアント・サーバー間プロトコルの基盤**: クライアントに対して実行完了したコマンドの種類を通知する CommandComplete メッセージの生成に利用されます。
  • **イベントトリガーや機能制御**: イベントトリガーのフィルタリングや、コマンドごとの行数表示の可否判定、イベントトリガーの対応状況の確認などに使用されます。
  • **DDLの解析・テスト**: DDL操作の内容を保持する CollectedCommand からコマンドタグを生成し、DDLの逆パースやテストモジュール（test_ddl_deparse）での検証に役立ちます。

関与するモジュール

  • module:src/backend/tcop: コマンドタグの決定（utility.c）や、コマンド名取得などのユーティリティ処理を担当します。
  • module:src/include/tcop: CommandTag 列挙型の定義（cmdtag.h）や、コマンドリスト（cmdtaglist.h）を保持します。
  • module:src/test/modules/test_ddl_deparse: DDL逆パースのテストにおいて、コマンドタグを取得・検証する関数を提供します。

主要エンティティ

  • **symbol:src/include/tcop/cmdtag.h#CommandTag**: SQLコマンドの種類を識別する列挙型。tcop/cmdtaglist.h をインクルードして全SQLコマンドの列挙子を定義します。
  • **symbol:src/backend/tcop/utility.c#CreateCommandTag**: 与えられたSQLパースツリー（Node *parsetree）を解析し、対応する CommandTag を決定して返す関数です。
  • **symbol:src/test/modules/test_ddl_deparse/test_ddl_deparse.c#get_command_tag**: テストモジュール用の関数。CollectedCommand 構造体のパースツリーからコマンド名を取得し、テキスト形式（Datum）で返します。

エンドツーエンドの流れ

1. **パースツリーの解析**: SQL文が解析され、Node *parsetree（または RawStmt）が生成されます。 2. **タグの決定**: CreateCommandTag(parsetree) が呼び出され、ノードの型（T_InsertStmt, T_SelectStmt など）や内部フィールド（TransactionStmt の kind など）に基づいて適切な CommandTag が決定されます。 3. **コマンドの実行と完了通知**:

  • コマンド実行後、QueryCompletion 構造体などを通じて CommandTag が保持されます。
  • CommandTag から対応する文字列（コマンド名）が取得され、クライアントへ完了通知として送信されます。
  • イベントトリガー（src/backend/commands/event_trigger.c）などの処理でも、このタグ情報がフィルタリング等に利用されます。

    • 4. **テスト・デバッグ（DDL逆パース時など）**: test_ddl_deparse モジュールでは、CollectedCommand 内のパースツリーから CreateCommandName を介してコマンド名を取得し、テスト検証用のテキストとして出力します。

*注意: クライアントへの送信処理の具体的な関数呼び出しや、QueryCompletion への格納タイミングなど、詳細なエンドツーエンドの制御フローの全容は提供されたソースコードの範囲内では完全には網羅されていません。*

重要な不変条件

  • **定義の整合性**: 新しいSQLコマンドを追加する場合、src/include/tcop/cmdtaglist.h に定義を追加する必要があり、この列挙値は cmdtag.c 内の属性テーブル（tag_behavior）と整合していなければなりません。
  • **パースツリーの有効性**: CreateCommandTag に渡される parsetree は、nodeTag() で識別可能な有効なノードである必要があります。
  • **NULLの許容**: get_command_tag において、CollectedCommand の parsetree が NULL の場合は安全に NULL を返します。

よくある落とし穴

  • **未知のノード型**: CreateCommandTag において、未知のノードタイプが渡された場合、switch 文の default 節がないケースでは未定義の挙動となる可能性があります（一部のステートメント種別では CMDTAG_UNKNOWN を返します）。
  • **新規コマンド追加時の漏れ**: 新しいSQLコマンドを追加した際、cmdtaglist.h への追加や CreateCommandTag の switch 文へのケース追加を怠ると、正しい CommandTag が割り当てられず CMDTAG_UNKNOWN になったり、予期せぬ動作を引き起こしたりします。
  • **特殊なロジック**: INSERT コマンドのタグには、歴史的な理由により "0" を含む特殊なロジックが存在するため、文字列変換や処理の際には注意が必要です。
