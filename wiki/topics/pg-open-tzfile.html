pg_open_tzfile

概念

pg_open_tzfile は、PostgreSQLのタイムゾーンデータベースディレクトリ（pg_TZDIR()）から、指定されたタイムゾーン名に対応するファイルを読み取り専用で開くための低レベル関数です。

PostgreSQLのソースコード内には、同名の関数が異なる2つのファイル（src/bin/initdb/findtimezone.c と src/timezone/pgtz.c）に定義されており、それぞれ異なるアプローチでファイルオープンを試みます。

  • **findtimezone.c 版**: パスを単純に結合して直接 open() を試みるシンプルな実装です。
  • **pgtz.c 版**: 大文字小文字を区別しない再帰検索（scan_directory_ci()）をサポートし、必要に応じて正規化された名前（canonname）を取得できる、より堅牢な実装です。

なぜ重要か

PostgreSQLがタイムゾーン情報を正しく処理するためには、OSやビルド設定に応じたタイムゾーンデータベースファイルに確実にアクセスする必要があります。

  • **初期化時（initdb）**: システムの適切なデフォルトタイムゾーンを検出するために使用されます。
  • **サーバー実行時**: セッションのタイムゾーン設定などに応じて動的にタイムゾーンファイルをロードするために使用されます。
  • **プラットフォーム非依存の保証**: scan_directory_ci() を使用した大文字小文字を区別しない検索により、ファイルシステムが大文字小文字を区別するかどうかにかかわらず、一貫したタイムゾーン名の解決を保証します。また、Windows環境でのバイナリモードオープン（PG_BINARY）も保証されています。

エンドツーエンドの流れ

呼び出し元からファイル記述子が取得されるまでの流れは、実装によって異なります。

src/bin/initdb/findtimezone.c 版の流れ

1. 呼び出し元（src/timezone/localtime.c や src/timezone/pgtz.c など）から name と canonname が渡されます。 2. canonname が指定されていれば、name をそのまま canonname にコピーします。 3. pg_TZDIR() から取得したベースパスと name を結合してフルパスを作成します。 4. パス長が MAXPGPATH を超えないことを確認し、open() システムコールを O_RDONLY | PG_BINARY で呼び出してファイル記述子を返します。

src/timezone/pgtz.c 版の流れ

1. 呼び出し元（src/bin/initdb/findtimezone.c や src/timezone/localtime.c など）から name と canonname が渡されます。 2. canonname が NULL の場合、まず指定された名前で直接 open() を試みます。 3. 直接オープンに失敗した場合、または canonname が指定されている場合、パスをディレクトリレベルで分割し、scan_directory_ci() を用いて大文字小文字を区別せずに再帰的に検索します。 4. 検索に成功し、canonname が指定されていれば、見つかったパスから正規化された名前を canonname にコピーします。 5. 最終的に open() を呼び出してファイル記述子を返します。

※**未解決のフロー**: これら2つの実装がビルド時にどのように選択・リンクされるか、あるいは実行時にどのように相互作用するかといった、システム全体における詳細なエンドツーエンドの統合フローは、提供された情報からは完全には明らかではありません。

重要な不変条件

  • **パス長の制限**: 結合されたフルパス（または入力された name）は MAXPGPATH を超えてはなりません。超えた場合は即座に -1 を返します。
  • **バッファサイズ**: canonname が提供される場合、そのバッファサイズは TZ_STRLEN_MAX + 1 以上でなければなりません（src/timezone/pgtz.c 版）。
  • **ベースディレクトリの有効性**: pg_TZDIR() が有効なタイムゾーンディレクトリパスを返す必要があります。

よくある落とし穴

  • **同名関数の混同**: src/bin/initdb/findtimezone.c と src/timezone/pgtz.c に同名かつ同シグネチャの関数が存在するため、コードリーディングやデバッグ時にどちらの実装が呼び出されているかを混同しやすくなっています。
  • **エラー時の errno の不確実性**: src/bin/initdb/findtimezone.c 版では、パス長超過などの自作チェックで -1 を返す際、errno を明示的に設定しません。そのため、呼び出し元は失敗時に常に有効な errno が設定されていると仮定してはなりません。
  • **環境依存性**: pg_TZDIR() の返す値（環境変数やビルド設定）に依存するため、実行環境によってファイルの検索結果や挙動が変わる可能性があります。
