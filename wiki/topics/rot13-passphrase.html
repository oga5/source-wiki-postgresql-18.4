rot13_passphrase

概念

rot13_passphrase は、PostgreSQLのテスト用モジュールにおいて、パスワードやパスフレーズを難読化（ROT13変換）するために使用される関数の名前です。同名の関数が2つの異なるテストモジュールに存在しますが、それぞれシグネチャと目的が異なる独立した実装となっています。

なぜ重要か

これらの関数は、PostgreSQLのテスト環境において、認証プロセス（LDAP認証やSSLパスフレーズのコールバック）の動作を検証するために重要です。実運用環境でのセキュリティ保護を目的としたものではなく、テスト目的でパスワードやパスフレーズを簡易的に難読化・変換する仕組みを提供します。

関与するモジュール

  • module:src/test/modules/ldap_password_func
  • module:src/test/modules/ssl_passphrase_callback

主要エンティティ

  • **symbol:src/test/modules/ldap_password_func/ldap_password_func.c#rot13_passphrase**:

    • LDAP認証プロセス用の関数。シグネチャは static char *rot13_passphrase(char *pw)。入力されたパスワードを palloc で確保した新しい領域にコピーし、ROT13変換を施して返します。

  • **symbol:src/test/modules/ssl_passphrase_callback/ssl_passphrase_func.c#rot13_passphrase**:

    • OpenSSLのパスフレーズ要求用のコールバック関数。シグネチャは static int rot13_passphrase(char *buf, int size, int rwflag, void *userdata)。静的変数 ssl_passphrase の内容をバッファにコピーしてROT13変換を施し、その長さを返します。

エンドツーエンドの流れ

これら2つのモジュールはそれぞれ独立して動作し、相互に連携するエンドツーエンドのフローは存在しません。それぞれの動作フローは以下の通りです。

1. LDAPパスワードファンクションの流れ

  • モジュールの初期化時（_PG_init）に、本関数が ldap_password_hook に登録されます。
  • コアシステム（src/backend/libpq/auth.c）が認証を行う際、このフックを介して本関数が呼び出され、入力されたパスワードがROT13変換されます。

2. SSLパスフレーズコールバックの流れ

  • SSL_CTX_set_default_passwd_cb を介して、本関数がOpenSSLライブラリにコールバックとして登録されます。
  • OpenSSLがパスフレーズを要求した際、このコールバックが起動し、静的変数 ssl_passphrase をROT13変換した値がバッファに書き込まれて返されます。

重要な不変条件

  • **変換対象文字**: どちらの実装においても、ROT13変換の対象はアルファベット（A-Z, a-z）のみであり、数字や記号、マルチバイト文字などは変換されずそのまま維持されます。
  • **LDAP側**: 入力文字列 pw はヌル終端されている必要があります。また、palloc を使用するため、PostgreSQLのメモリコンテキスト内で呼び出される必要があります。
  • **SSL側**: 静的変数 ssl_passphrase が NULL でないことが Assert により保証されている必要があります。

よくある落とし穴

  • **同名関数の混同**: ldap_password_func.c と ssl_passphrase_func.c に定義されている rot13_passphrase は、名前は同じですがシグネチャ（引数や戻り値）も目的も全く異なります。コードの変更や参照の際には、どちらのモジュールを扱っているかを明確に区別する必要があります。
  • **セキュリティ上の誤解**: これらはテスト用の難読化処理に過ぎず、暗号学的なセキュリティ保護機能はありません。実運用環境で使用してはなりません。
  • **バッファオーバーフロー/切り詰め**: SSL側の実装では、提供されたバッファサイズ size を超えるパスフレーズは strlcpy によって切り詰められます。
