PqGSSSendConsumed

概念

PqGSSSendConsumed は、PostgreSQL の GSSAPI（Generic Security Services Application Program Interface）を用いたセキュアな通信において、送信バッファ内の処理済み（消費済み）データ量を管理または追跡するために、バックエンド（サーバー）側とフロントエンド（クライアント/libpq）側の両方で定義・利用されるシンボルです。

なぜ重要か

GSSAPI による暗号化通信では、送信データを暗号化してネットワークに送出する際、どこまで送信が完了（消費）したかを正確に追跡する必要があります。PqGSSSendConsumed は、バックエンドとフロントエンドのそれぞれにおいて、非ブロッキング I/O や部分的な書き込みが発生した際に、送信状態の整合性を維持するために重要な役割を果たしていると考えられます。
