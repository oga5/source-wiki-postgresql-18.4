pg_pread

概念

pg_pread は、Windows環境においてPOSIX標準の pread(2) 関数をエミュレートするための関数です。ファイル記述子（fd）の現在のファイルポインタを変更することなく、指定されたオフセット（offset）からデータを直接読み取る機能を提供します。

なぜ重要か

PostgreSQLのコアコード（SLRU、XLOGリーダー、バックアップ処理など）は、ファイルポインタの状態に依存せずに特定の位置からデータを読み取る pread の挙動に依存しています。Windows環境はネイティブでPOSIXの pread をサポートしていないため、pg_pread を通じてプラットフォーム間のI/O挙動の差異を吸収し、コードのポータビリティを確保するために極めて重要です。

関与するモジュール

  • module:src/include
  • module:src/port

主要エンティティ

  • symbol:src/include/port.h#pg_pread: 関数の宣言を提供します。
  • symbol:src/port/win32pread.c#pg_pread: Windows API（ReadFile）を用いた具体的なエミュレーション処理を実装しています。

エンドツーエンドの流れ

1. **呼び出し**: src/backend/access/transam/slru.c や src/backend/access/transam/xlogreader.c などの呼び出し元から、ファイル記述子、バッファ、サイズ、オフセットを指定して pg_pread が呼び出されます。 2. **ハンドル取得**: _get_osfhandle(fd) を使用して、Cランタイムのファイル記述子からWindowsネイティブのファイルハンドルを取得します。 3. **オフセット設定**: OVERLAPPED 構造体の Offset フィールドに指定された offset を設定します。 4. **読み取り実行**: ReadFile APIを呼び出し、OVERLAPPED 構造体を渡すことで、ファイルポインタを移動させずに指定位置からデータを読み取ります。 5. **結果の返却**:

  • 成功時は実際に読み取ったバイト数を返します。
  • ReadFile が失敗し、エラーコードが ERROR_HANDLE_EOF の場合は 0（EOF）を返します。
  • それ以外のエラーの場合は、_dosmaperr を用いてWindowsエラーコードをPOSIX準拠の errno に変換し、-1 を返します。

※なお、呼び出し元（slru.c や xlogreader.c など）における具体的なエラーハンドリングや、上位レイヤーから下位レイヤーに至る完全なエンドツーエンドの制御フローの詳細は、提供されたコンテキストからは未解明です。

重要な不変条件

  • **読み取りサイズ制限**: size は 1GB（1024 * 1024 * 1024 バイト）を超えることはできません。DWORD 型のオーバーフローを防ぐため、内部で Min(size, 1024 * 1024 * 1024) が適用され、自動的に切り詰められます。
  • **有効なハンドル**: 引数として渡される fd は、_get_osfhandle によって有効なWindowsファイルハンドルに変換可能でなければなりません。

よくある落とし穴

  • **ファイルポインタの非移動**: pg_pread はファイルポインタを更新しません。呼び出し側が「読み取りによってファイルポインタが進む」ことを期待している場合、意図しない挙動になります。
  • **無効なファイル記述子**: _get_osfhandle が無効なハンドルを返した場合、関数は errno を EBADF に設定して -1 を返します。
  • **1GBを超えるサイズ指定**: 1GBを超えるサイズを指定した場合、エラーにはならず自動的に1GBに制限されて読み取られます。呼び出し側が一度に1GB以上の読み取りを期待している場合、部分的な読み取りとして処理されるため、ループ等での再試行が必要になります。
