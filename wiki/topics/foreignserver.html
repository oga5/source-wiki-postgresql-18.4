ForeignServer

概念

ForeignServer は、PostgreSQLにおいて外部データソース（外部サーバー）の定義情報を表すデータ構造および概念です。SQLの CREATE SERVER コマンドによって定義され、システムカタログ pg_foreign_server に登録されます。外部データラッパー（FDW）が外部リソースへ接続・操作する際に必要となるメタデータ（接続オプション、FDWのOID、所有者など）を管理します。

なぜ重要か

外部データラッパー（FDW）を介して外部のデータソース（他のPostgreSQLインスタンス、ファイル、その他のデータベースなど）にアクセスする際、接続先サーバーの定義やオプション情報を一元管理するために不可欠です。これにより、SQL実行時や接続確立時に、適切な接続パラメータや認証情報をFDWが参照できるようになります。

関与するモジュール

提供された情報に基づくと、以下のモジュールが関与しています。

  • module:src/backend/commands: SQLコマンド（CREATE SERVER など）の実行処理を担当します。
  • module:src/backend/foreign: 外部サーバーなどの外部データ関連オブジェクトの取得やユーティリティ処理を担当します。
  • module:src/include/foreign: ForeignServer 構造体などのデータ定義を提供します。

主要エンティティ

  • symbol:src/include/foreign/foreign.h#ForeignServer: 外部サーバーのメタデータ（サーバーID、FDWのID、所有者、サーバー名、オプションなど）を保持するC言語の構造体です。
  • symbol:src/backend/commands/foreigncmds.c#CreateForeignServer: CREATE SERVER コマンドを処理し、新規の外部サーバー定義を pg_foreign_server カタログに登録する関数です。
  • symbol:src/backend/foreign/foreign.c#GetForeignServer: 指定された OID に対応する外部サーバーの定義情報（ForeignServer 構造体）をシステムカタログから取得する関数です。

エンドツーエンドの流れ

提供された情報からは、完全なエンドツーエンドの実行時フロー（クエリ実行時にどのように ForeignServer が参照され、接続が確立されるかなど）の全容は完全には明らかになっていません。現時点で判明している部分的な流れは以下の通りです。

1. **定義フェーズ (DDLの実行)**:

  • ユーザーが CREATE SERVER コマンドを実行すると、src/backend/tcop/utility.c などのユーティリティコマンド実行フローから CreateForeignServer が呼び出されます。
  • CreateForeignServer は、指定されたFDWの存在と権限を検証し、オプションをFDWのバリデーター関数で検証（transformGenericOptions）した上で、システムカタログ pg_foreign_server にタプルを挿入します。また、依存関係を pg_depend に記録します。

2. **参照フェーズ (DML実行や接続時)**:

  • FDW（例: postgres_fdw や file_fdw）やシステム内部の処理において、特定の外部サーバー情報が必要になると、サーバーの OID を用いて GetForeignServer が呼び出されます。
  • GetForeignServer は内部的に GetForeignServerExtended を呼び出し、システムカタログから取得した情報を ForeignServer 構造体にマッピングして呼び出し元に返します。

重要な不変条件

  • **一意性と存在**: CreateForeignServer で作成される外部サーバーは、同名のサーバーが既に存在する場合、IF NOT EXISTS が指定されていない限りエラー（ERRCODE_DUPLICATE_OBJECT）となります。また、GetForeignServer で取得する際は、指定された OID がシステムカタログ pg_foreign_server に存在しなければなりません（存在しない場合はエラーが発生します）。
  • **権限と依存関係**: 外部サーバーを作成するには、指定されたFDWに対する USAGE 権限が必要です。また、作成されたサーバーは、そのFDWや所有者、および作成時の拡張機能に対する依存関係（pg_depend）を持ちます。
  • **メモリ管理**: GetForeignServer などによってシステムキャッシュから生成された ForeignServer 構造体は、使用後に適切なメモリ管理（pfree など）が必要となります。

よくある落とし穴

  • **エラー制御の欠如**: GetForeignServer は GetForeignServerExtended のラッパーであり、対象のサーバーが存在しない場合は強制的にエラーを発生させます。サーバーの存在が不確実で、エラーにせず NULL を返したい場合は、直接 GetForeignServerExtended を MISSING_OK フラグ付きで呼び出す必要があります。
  • **オプションの解釈**: ForeignServer 構造体の options フィールドは単なる DefElem のリスト（汎用的なオプション表現）として保持されるため、FDWの実装側で適切に解析・検証を行う必要があります。
  • **オプション値のNULL可能性**: servertype や serverversion はオプション属性であるため、NULLになる可能性があります。FDW側でこれらを参照する際はNULLチェックが必要です。
