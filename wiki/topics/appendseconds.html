AppendSeconds

概念

AppendSeconds は、PostgreSQLのバックエンドおよびクライアントライブラリ（ECPG）において、秒の整数部および小数秒（fsec_t）を指定された精度でフォーマットし、文字列バッファに書き込むための低レベルな書式化関数です。

なぜ重要か

日付時刻（timestamp や interval など）の出力処理において、秒および小数秒のフォーマットは頻繁に行われます。AppendSeconds は、ゼロ埋めの制御や、指定された精度（precision）に応じた小数部の末尾ゼロの除去など、一貫したフォーマット処理を効率的に行うために重要な役割を果たします。

関与するモジュール

  • module:src/interfaces/ecpg/pgtypeslib

主要エンティティ

  • symbol:src/backend/utils/adt/datetime.c#AppendSeconds: 秒数と小数秒をフォーマットしてバッファに書き込むメインの実装関数。
  • src/interfaces/ecpg/pgtypeslib/interval.c 内の AppendSeconds: ECPGライブラリにおける呼び出し元、または同等の処理を行うエンティティ。

エンドツーエンドの流れ

1. **呼び出し**: timestamp や interval の出力処理（datetime.c 内の複数箇所、および interval.c）から、書き込み先バッファポインタ cp、秒の整数部 sec、小数秒 fsec、精度 precision、ゼロ埋めフラグ fillzeros を伴って呼び出されます。 2. **整数部の処理**: fillzeros が真の場合は pg_ultostr_zeropad を、偽の場合は pg_ultostr を使用して、秒の整数部をバッファに書き込みます。 3. **小数部の処理**: fsec が 0 でない場合、小数点 . を追加し、指定された precision に基づいて末尾のゼロを除去しながら逆順で数値を生成・書き込みます。 4. **精度不足時のフォールバック**: 指定された精度で小数部を表現しきれない場合（value が 0 にならない場合）、pg_ultostr を用いて最小限の有効幅で書き出します。 5. **返却**: 書き込みが完了したバッファの末尾ポインタを返します。

※ ECPG側（interval.c）における具体的な呼び出しや、詳細なエンドツーエンドの統合フローの全容は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • **精度の非負性**: precision >= 0 である必要があります（datetime.c の実装では Assert により保証されています）。
  • **データ型**: 小数秒 fsec は fsec_t（int32）として扱われます。

よくある落とし穴

  • **精度不足時の挙動**: 指定された precision が不足している場合、指定桁数を超えて数値全体が出力されるため、バッファサイズの見積もりに注意が必要です。
  • **小数部の末尾ゼロ除去**: 小数部の末尾ゼロ除去ロジックは逆順での数値生成に基づいているため、このロジックを変更する際は桁数計算の整合性を崩さないようにする必要があります。
  • **fsecが0の場合**: fsec が 0 の場合は小数部（小数点含む）自体が出力されません。
