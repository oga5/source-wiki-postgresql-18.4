IsCTIDVar

概念

IsCTIDVar は、PostgreSQLのクエリ最適化および実行フェーズにおいて、指定された変数（Var ノード）が対象リレーションの物理的なタプル識別子（ctid）を直接参照しているかどうかを判定するためのインライン関数です。

なぜ重要か

クエリプランナがTIDスキャン（TidScan）やTIDレンジスキャン（TidRangeScan）の経路（Path）を生成する際、およびエグゼキュータが実行時に引数をチェックする際に、対象の式が「現在のクエリレベルにおける正しいリレーションの ctid」を指しているかを厳密に検証する必要があります。この判定を誤ると、不正な物理アクセスや誤ったプラン生成につながるため、安全かつ正確な判定を行うために重要です。

関与するモジュール

  • module:src/backend/optimizer/path
  • module:src/backend/executor

  • **symbol:src/backend/executor/nodeTidscan.c#IsCTIDVar**:

    • エグゼキュータのTIDスキャンノード。実行時の引数チェックに本関数を使用します。

    • エグゼキュータのTIDレンジスキャンノード。同様に実行時の引数チェックに使用します。

エンドツーエンドの流れ

1. **プラン生成時**: オプティマイザ（tidpath.c）が制限条件のリストを走査する際、IsCTIDVar を使用して、条件式が対象リレーションの ctid を対象としているかを判定します。 2. **実行時**: エグゼキュータ（nodeTidscan.c / nodeTidrangescan.c）が、渡されたパラメータや引数が有効な ctid 参照であるかを検証するために IsCTIDVar を呼び出します。

*注意: プラン作成から実行完了に至る完全な制御フローや、詳細なエンドツーエンドの相互作用は、提供されたコンテキストからは完全には明らかになっていません。*

重要な不変条件

IsCTIDVar が true を返すためには、以下の条件がすべて満たされている必要があります。

  • **属性番号の一致**: var->varattno が SelfItemPointerAttributeNumber（ctid のシステム属性番号）であること。
  • **型の一致**: var->vartype が TIDOID であること。
  • **リレーションの一致**: var->varno が対象リレーションのID（rel->relid）と一致すること。
  • **Nulling Relsの不在**: var->varnullingrels が NULL であること（外側結合等によるNULL化の影響を受けないこと）。
  • **クエリレベルの一致**: var->varlevelsup が 0 であること（現在のクエリレベルでの参照であること）。

*注意: システム全体におけるその他の不変条件は、提供されたコンテキストからは完全には明らかになっていません。*

よくある落とし穴

  • **NULLポインタの危険性**:

    • 入力となる var または rel が NULL である場合、関数内でセグメンテーションフォールトが発生する可能性があります。呼び出し側で有効なポインタであることを保証する必要があります。

  • **外部クエリ参照の除外**:

    • サブクエリなどで外部のクエリレベルを参照している Var（varlevelsup > 0）は、たとえ ctid を参照していても IsCTIDVar では false と判定されます。

  • **結合（Join）による影響**:

    • varnullingrels が NULL でない場合（結合処理によってNULLになる可能性がある場合など）は、安全性のために false と判定されます。
