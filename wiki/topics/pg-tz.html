pg_tz

概念

pg_tz は、PostgreSQLにおいてタイムゾーン情報を保持するためのデータ構造です。

この構造体は、利用するコンテキストに応じて2つの異なる側面を持ちます。

  • **外部向け（不透明な構造体）**: src/include/pgtime.h において typedef struct pg_tz pg_tz; として宣言されており、外部モジュールに対しては内部構造が隠蔽されています。
  • **内部向け（具体的な構造体）**: src/timezone/pgtz.h において具体的なメンバ（正規化されたタイムゾーン名 TZname および遷移情報や閏秒情報を含む内部状態 struct state）を持つ構造体として定義されています。

なぜ重要か

pg_tz は、PostgreSQLにおける時刻計算やタイムゾーン変換（DetermineTimeZoneOffsetInternal() などによるオフセット計算）の基盤として機能します。また、セッションごとのタイムゾーン設定（session_timezone）やログ出力用のタイムゾーン設定（log_timezone）といった、システム全体のグローバルなタイムゾーン状態を管理・保持するためにも極めて重要です。

関与するモジュール

  • module:src/include (主に src/include/pgtime.h)
  • module:src/timezone (主に src/timezone/pgtz.h)

主要エンティティ

  • **symbol:src/include/pgtime.h#pg_tz**: 外部モジュール向けに提供される不透明な（opaque）構造体宣言。ポインタ経由でのみ操作されます。
  • **symbol:src/timezone/pgtz.h#pg_tz**: タイムゾーンの具体的なデータ（TZname や struct state）を保持する実体定義。

エンドツーエンドの流れ

1. **初期化**: タイムゾーン機能を使用する前に、pg_timezone_initialize() を呼び出して環境をセットアップします。 2. **ロード**: pg_tzset() などのAPIを介してタイムゾーン名（文字列）が指定されると、pg_open_tzfile により対応するファイルが特定され、tzload によってファイル内容が state 構造体に展開されて pg_tz インスタンスが構築されます。 3. **保持**: 構築された pg_tz へのポインタは、セッションやログのタイムゾーン設定（src/backend/commands/variable.c など）として保持されます。 4. **変換と参照**: 時刻変換処理（src/backend/utils/adt/datetime.c の DetermineTimeZoneOffsetInternal や FetchDynamicTimeZone など）において、保持された pg_tz ポインタが参照され、オフセット計算や時刻変換が実行されます。

重要な不変条件

  • **カプセル化の維持**: 外部モジュールから pg_tz の内部メンバに直接アクセスすることは禁止されています。操作は必ず pgtime.h で提供される公開APIを経由しなければなりません。
  • **名前の長さ制限**: TZname は TZ_STRLEN_MAX + 1 バイトの固定長配列であり、格納される正規化されたタイムゾーン名はこの制限内に収まる必要があります。
  • **初期化の前提**: タイムゾーンのロードや変換処理を行う前に、pg_timezone_initialize() による初期化が完了していなければなりません。

よくある落とし穴

  • **直接アクセスの試み**: pg_tz は外部からは不透明な構造体として扱われるため、src/timezone/pgtz.h を適切に考慮せずにメンバへ直接アクセスしようとするとコンパイルエラーになります。外部モジュールからは必ず pgtime.h のインターフェースを使用してください。
  • **バッファオーバーフローと切り詰め**: タイムゾーン名が TZ_STRLEN_MAX を超える場合、切り詰めやバッファオーバーフローが発生するリスクがあります。
  • **不正なタイムゾーンファイル**: state 構造体は tzfile 形式のバイナリデータに依存しているため、破損した、または不正なタイムゾーンファイルを読み込んだ場合、時刻変換処理において予期せぬ動作を引き起こす可能性があります。
  • **同名定義の混在**: src/include/pgtime.h と src/timezone/pgtz.h の両方に pg_tz に関連する定義が存在するため、インクルードパスやスコープの取り扱いに注意が必要です。

次に読む場所

  • src/include/pgtime.h: 外部公開されているタイムゾーン操作APIおよび不透明な構造体の宣言を確認する場合。
  • src/timezone/pgtz.h: pg_tz の具体的な構造体定義や、内部状態を管理する struct state の詳細を確認する場合。
