JsonTokenType

概念

JsonTokenType は、PostgreSQLのJSON解析プロセスにおいて、入力ストリームから切り出された各トークン（要素）の種類を識別するための列挙型（enum）です。JSON仕様（RFC 8259）で定義された構造要素（オブジェクトや配列の開始/終了、カンマ、コロン）やリテラル（文字列、数値、true、false、null）に対応する値を持ちます。

なぜ重要か

JSONの字句解析（レキシカルアナリシス）において、パーサーが現在処理している文字の並びがどのJSON要素に該当するかを識別し、後続のセマンティック解析（意味解析）やコールバック関数（JsonSemAction など）へ正確に伝えるための共通の識別子として機能するためです。これにより、イベント駆動型のJSON解析や、jsonb などの内部表現への変換が正しく制御されます。

エンドツーエンドの流れ

1. **字句解析**: パーサーが入力データを走査し、文字の並びに基づいて JsonTokenType のいずれかの値を特定します。 2. **状態管理**: 特定されたトークン種別は、解析状態を保持する構造体（JsonLexContext）に管理されます。 3. **コールバックへの伝播**: 解析コールバック関数（JsonSemAction）の引数として JsonTokenType が渡され、オブジェクトの開始/終了やスカラ値の処理といったイベント駆動型の解析が実行されます。 4. **セマンティック処理**: src/backend/utils/adt/jsonfuncs.c 内の各関数（get_scalar や elements_scalar など）や src/backend/utils/adt/jsonb.c の jsonb_in_scalar などの呼び出し元が、渡されたトークン種別に基づいて具体的な処理を行います。

※パーサー内部の具体的な文字走査アルゴリズムや、すべてのコールバック関数における詳細なエンドツーエンドの処理フローの全容は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • JSON_TOKEN_INVALID は、解析不能な不正トークンを示します。
  • JSON_TOKEN_END は、入力ストリームの終端に到達したことを示します。
  • 各列挙値は、JSON仕様（RFC 8259）で定義された構造要素およびリテラルと一対一で対応しています。

※その他の詳細な状態遷移や不変条件の全容は、提供された情報からは完全には明らかになっていません。

よくある落とし穴

  • **不正な構文のハンドリング**: JSON_TOKEN_INVALID が返された場合、パーサーは不正な JSON 構文を検出したことを意味するため、適切なエラーハンドリングが必要です。
  • **予期しないトークンの処理**: 予期しないトークン種別がコールバックに渡された場合、呼び出し元（セマンティック解析側）でエラーハンドリングを行う必要があります。
  • **拡張時の注意**: 新しい JSON 構文要素を追加する場合は、本列挙型に値を追加するだけでなく、パーサーの実装（jsonapi.c）および各コールバック関数での switch 文などを漏れなく更新する必要があります。
