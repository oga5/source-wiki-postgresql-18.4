WordBoundaryState

概念

WordBoundaryState は、Unicode文字列のタイトルケース変換（initcap 処理）において、文字列走査中の現在の位置、単語境界の判定状態、および走査対象のメタデータを保持するための構造体（struct）です。

この構造体は以下のメンバから構成されます：

  • str: 走査対象の文字列へのポインタ
  • len: 文字列の長さ
  • offset: 現在の走査位置（オフセット）
  • posix: POSIX準拠モードを示すフラグ
  • init: 初期化済みフラグ
  • prev_alnum: 直前の文字が英数字であったかを示すフラグ

なぜ重要か

文字列をタイトルケース（単語の先頭のみを大文字にし、それ以外を小文字にする）に変換する際、システムは「どこが単語の境界（開始位置）であるか」を正確に判定する必要があります。WordBoundaryState は、この判定を行うイテレータ関数（initcap_wbnext など）の間で走査状態を共有・追跡するためのコンテキストとして機能し、正確な文字変換処理を支えています。

関与するモジュール

本構造体は以下のモジュールにまたがって定義・利用されています：

  • module:src/backend/utils/adt
  • module:src/common/unicode

主要エンティティ

本トピックに関連する主要なエンティティは以下の通りです：

  • **symbol:src/backend/utils/adt/pg_locale_builtin.c#WordBoundaryState**: ビルトインロケール処理（strtitle_builtin）において、単語境界を追跡するために使用される構造体。
  • **symbol:src/common/unicode/case_test.c#WordBoundaryState**: Unicodeのケース変換テストにおいて、同様の単語境界判定状態を管理するために使用される構造体。
  • **initcap_wbnext**: 本構造体を引数として受け取り、pg_u_isalnum を用いて現在の文字と直前の文字の属性を比較しながら、単語境界を特定して構造体の状態（offset や prev_alnum）を更新する関数。

エンドツーエンドの流れ

1. **初期化**: タイトルケース変換処理の開始時に WordBoundaryState のインスタンスが作成されます。対象文字列 str、長さ len、初期オフセット offset（通常は 0）が設定され、初期化フラグ init が true に設定されます。 2. **境界の検出**: strtitle_builtin（またはテストコード）から initcap_wbnext が呼び出されます。 3. **状態の更新**: initcap_wbnext は、現在の文字が英数字であるかを判定し、直前の状態を示す prev_alnum と比較して単語の区切りを特定します。処理が進むにつれて offset と prev_alnum が逐次更新されます。 4. **変換の実行**: 検出された単語境界に基づいて、対象文字のケース変換が行われます。

*注意: posix フラグによる判定ロジックの具体的な分岐や、エラー発生時の詳細な制御フローの全貌は、提供された情報からは完全には明らかになっていません。*

重要な不変条件

  • **初期化の保証**: init フラグが true である場合のみ、構造体は有効な走査状態にあるとみなされます（走査開始時は false から始まり、適切な初期化関数によって true に設定されます）。
  • **メモリの妥当性**: str は、len で指定された範囲内で有効なメモリ領域を指し示している必要があります。
  • **状態の一貫性**: prev_alnum は、initcap_wbnext が単語境界を正しく判定できるように、直前の文字の英数字属性と常に一致するように更新されなければなりません。

よくある落とし穴

  • **バッファオーバーランの危険性**: len が実際の文字列長と一致しない場合、範囲外アクセスが発生する可能性があります。
  • **未定義動作**: str が NULL の状態で走査を開始した場合の動作は定義されていません。
  • **重複定義の同期**: 本構造体は pg_locale_builtin.c と case_test.c の両方に個別に定義されています。構造体のメンバを変更・拡張する場合は、バックエンドの実装とテストコードの実装の両方を同期して修正する必要があります。
