gcd

概念

gcd は、2つの符号なし32ビット整数（uint32）の最大公約数（Greatest Common Divisor）を計算するための静的関数です。ユークリッドの互除法を用いて実装されており、テーブルサンプリングメソッドのブロック選択ロジックなどで利用されます。

なぜ重要か

この関数は、tsm_system_rows および tsm_system_time モジュールにおいて、ランダムなブロック選択を行うための補助関数（random_relative_prime など）の基礎となる計算を提供します。互いに素な数（最大公約数が1である関係）を判定・生成するプロセスにおいて不可欠な役割を果たします。

エンドツーエンドの流れ

1. 各サンプリングモジュール（tsm_system_rows または tsm_system_time）の処理中に、ブロック選択ロジックの一部として random_relative_prime が呼び出されます。 2. random_relative_prime 内部から、2つの数値の最大公約数を求めるために gcd が呼び出されます。 3. gcd は、以下のユークリッドの互除法アルゴリズムを実行します。

  • a が 0 でない間、a の値を一時変数 c に退避し、a を b % a で更新し、b に c を代入するループを繰り返します。
  • a が 0 になるとループを抜け、最終的な b の値を返します。

※ なお、PostgreSQLのクエリ実行エンジンからこれらのサンプリングメソッドが呼び出される全体的なエンドツーエンドの制御フローや、上位システムとの詳細な相互作用については、提供されたコンテキストからは完全には明らかになっていません。

重要な不変条件

  • **入力の型制限**: 入力引数 a および b は uint32 の範囲内でなければなりません。
  • **ゼロ入力時の挙動**: gcd(x, 0) = x の関係が成り立ちます。片方の入力が 0 の場合はもう一方の入力値が返され、両方の入力が 0 の場合は 0 が返されます。
  • **ループの終了条件**: 剰余演算（b % a）を繰り返すことで、有限回のステップで必ず a が 0 に達し、ループが終了することが保証されています。

よくある落とし穴

  • **コードの重複と個別管理**: gcd 関数は tsm_system_rows.c と tsm_system_time.c のそれぞれに static 関数として独立して定義されています。そのため、一方の挙動を変更または最適化する場合は、もう一方のファイルも同様に修正する必要があります。
  • **他モジュールの同名関数との混同**: pgcrypto や PostgreSQLコア（src/backend/utils/adt/int.c など）にも同名または類似の関数が存在する場合がありますが、これらは本トピックが扱う tsm 関連の静的関数とは別個に管理されています。
