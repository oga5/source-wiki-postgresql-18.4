oauth_json_scalar

概念

oauth_json_scalar は、PostgreSQLのOAuth認証プロセスにおいて、JSON形式のレスポンスを解析する際にスカラー値（主に文字列）を抽出し、指定されたターゲットフィールドに格納するためのコールバック関数です。JSON解析器と連携し、必要な認証メタデータ（アクセストークンなど）を安全に取得する役割を持ちます。

なぜ重要か

OAuth認証フローでは、外部のIDプロバイダやトークンエンドポイントから返されるJSONレスポンスを正確にパースする必要があります。oauth_json_scalar は、単に値を抽出するだけでなく、JSONの構造検証（ネストレベルのチェック）、重複定義の防止、型チェック（文字列型であることの確認）を厳格に行うことで、不正なJSONレスポンスによるバッファオーバーフローや不正な状態遷移を防ぎ、安全なメモリ確保（strdup）を保証します。

エンドツーエンドの流れ

1. **解析の開始**: JSONパーサーがレスポンスデータを処理し、スカラー値（トークン）を検出すると、コールバックとして oauth_json_scalar が呼び出されます。 2. **コンテキストの取得**: 引数 state から json_ctx 構造体へのポインタを取得します。 3. **トップレベルの検証**: ctx->nested が0の場合、トップレベルがオブジェクト構造であるか検証します。オブジェクトでない場合はエラーを返します。 4. **ターゲットフィールドの処理**:

  • ctx->target_field が設定されている場合、現在のネストレベルが1であることを確認します。
  • すでに値が格納されている場合は、重複エラーを返します。
  • トークン型が JSON_TOKEN_STRING であることを検証します。
  • strdup を使用して token を複製し、*ctx->target_field に格納します。
  • 格納完了後、ctx->target_field および ctx->target_field_name を NULL にクリアします。

    • 5. **無視**: ターゲットフィールドが設定されていない場合は、値を処理せず JSON_SUCCESS を返します。

※ なお、libpq-oauth 側（oauth-curl.c）における具体的なHTTP通信や、全体のOAuth認証シーケンスにおける完全なエンドツーエンドの統合フローの詳細は、提供された情報からは未確定です。

重要な不変条件

  • **型キャストの妥当性**: state 引数は、常に有効な json_ctx 構造体インスタンスを指していなければなりません。
  • **オブジェクト構造**: 解析対象のJSONのトップレベルは、必ずオブジェクトでなければなりません。
  • **ネストレベルの制限**: ターゲットフィールドに値を格納する際、ネストレベルは正確に 1 でなければなりません（それ以外は内部エラー）。
  • **単一代入**: 同一のターゲットフィールドに対して、複数回値を書き込むことはできません（重複エラーの発生）。
  • **トークン型**: 抽出対象となるトークンは、必ず JSON_TOKEN_STRING（文字列）でなければなりません。

よくある落とし穴

  • **メモリリーク**: oauth_json_scalar は内部で strdup を使用して文字列を動的に確保します。呼び出し側（またはコンテキストの所有者）がこのメモリを適切に解放しない場合、メモリリークが発生します。
  • **ターゲットフィールドの自動クリア**: 値が一度格納されると、ctx->target_field は自動的に NULL にリセットされます。これは単一のフィールド抽出を前提とした設計であるため、同一コンテキストで連続して複数の値を抽出する場合は、状態管理に注意が必要です。
  • **ネストされたオブジェクトの制限**: ネストレベルが1より深い場所にあるスカラー値は、ターゲットフィールドが設定されていても抽出できず、エラー（内部エラーまたはパースエラー）となります。
