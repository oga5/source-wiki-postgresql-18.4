XLogReaderState

概念

XLogReaderState は、PostgreSQL の WAL（Write-Ahead Log）を読み取り、解析するためのセッション状態を管理するメイン構造体です。WAL の物理的な読み込みから、バイナリデータから論理的なレコード構造（DecodedXLogRecord）へのデコードまでの一連の状態を保持します。

なぜ重要か

WAL の解析は、データベースのクラッシュリカバリやレプリケーションだけでなく、WAL の検査・解析ツール（pg_walinspect や pg_waldump など）においても極めて重要です。XLogReaderState は、バックエンド（サーバープロセス内）とフロントエンド（スタンドアロンツール）の両方で共通して利用できる一貫した WAL 読み取りインターフェースを提供します。

関与するモジュール

  • module:src/include/access (WAL リーダーのコア定義)
  • module:contrib/pg_walinspect (WAL 解析ユーティリティ)

主要エンティティ

  • **symbol:src/include/access/xlogreader.h#XLogReaderState**

    • WAL 読み取りセッションの状態を保持する構造体。コールバック関数群（XLogReaderRoutine）やデコードされたレコードのキューなどを管理します。

  • **symbol:contrib/pg_walinspect/pg_walinspect.c#InitXLogReaderState**

    • 指定された LSN（Log Sequence Number）を基準に XLogReaderState を初期化し、最初に有効な WAL レコードを検索して読み取り準備を整えるヘルパー関数。

エンドツーエンドの流れ

1. **初期化とメモリ確保** InitXLogReaderState などの呼び出し元が、XLogReaderAllocate を用いて XLogReaderState を割り当てます。この際、WAL ページを読み込むためのコールバック関数（例: read_local_xlog_page_no_wait）を登録します。 2. **レコードの検索** XLogFindNextRecord を使用して、指定された LSN 以降で最初に有効な WAL レコードの位置を特定します。 3. **ページの読み込みとデコード** XLogReadRecord などの関数が実行されると、登録されたコールバックを介して WAL ページが内部バッファにロードされます。ロードされたバイナリデータは DecodedXLogRecord へデコードされ、内部キューに格納されます。 4. **データの抽出** 呼び出し元（pg_walinspect の各解析関数など）は、提供されているマクロ（XLogRecGet* 等）を介して、デコードされたレコードから XID やブロック参照などの必要な情報を取得します。

*注意: 先読み（read-ahead）や詳細なエラーハンドリングを伴う完全なライフサイクルおよび内部キューの挙動の全容は、提供されたコンテキストからは一部未検証です。*

重要な不変条件

  • **LSN の最小境界制限**

    • 読み取り開始基準となる LSN は XLOG_BLCKSZ 以上でなければなりません。ブートストラップ WAL ページは読み取り不可であるため、これ未満の LSN を指定することは禁止されています。

  • **カプセル化の維持**

    • XLogReaderState の内部フィールドを直接操作することは避け、提供されている API や XLogRecGet* などのマクロを介して操作する必要があります。

  • **環境に依存しない設計**

    • 本構造体はフロントエンドとバックエンドの双方で動作する設計であるため、コアの構造体や API 内部でバックエンド専用の ereport などを直接呼び出すことはできません。

よくある落とし穴

  • **無効な LSN の指定**

    • XLOG_BLCKSZ 未満の LSN を指定して初期化を試みると、ERRCODE_INVALID_PARAMETER_VALUE エラーが発生します。

  • **メモリ確保の失敗**

    • XLogReaderAllocate がメモリ不足により NULL を返した場合のハンドリングを怠ると、ヌルポインタ参照に繋がります。

  • **先読みバッファの未確保**

    • 先読み（read-ahead）機能を有効にする場合、事前に XLogReaderSetDecodeBuffer を使用して循環バッファを適切に確保しておかないと、期待通りに動作しません。
