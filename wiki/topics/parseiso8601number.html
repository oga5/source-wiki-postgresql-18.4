ParseISO8601Number

概念

ParseISO8601Number は、ISO 8601 形式の文字列から数値を解析し、整数部（int）と小数部（double）に分離して抽出するための関数です。PostgreSQL のバックエンド（datetime.c）と、クライアント側の埋め込み SQL プリプロセッサ用ライブラリである ecpg（interval.c）の双方に同名の関数が存在します。

なぜ重要か

ISO 8601 形式の時間間隔（Interval）などのパース処理において、文字列から数値部分を正確に切り出し、整数部と小数部に分けて処理することは、後続の日付・時間演算を正確に行うために不可欠です。また、バックエンドとクライアントライブラリ（ecpg）で同様のパースロジックを維持することで、一貫したデータ解析挙動を保証します。

エンドツーエンドの流れ

ecpg ライブラリ側（src/interfaces/ecpg/pgtypeslib/interval.c）における処理の流れは以下の通りです：

1. **呼び出し**: 同一ファイル内の DecodeISO8601Interval 等から、解析対象の文字列 str と解析終了位置を格納する endptr のポインタを伴って呼び出されます。 2. **検証と変換**: 入力文字列の妥当性を確認した後、標準ライブラリの strtod を使用して文字列を浮動小数点数に変換します。 3. **分離**: 変換された値に対して floor を使用した 0 への切り捨てを行い、整数部を ipart に、元の数値から整数部を引いた残りの値を小数部として fpart に格納します。 4. **返却**: 成功時は 0 を返し、呼び出し元は分離された値を用いて時間間隔の各フィールドを構築します。

※バックエンド側（src/backend/utils/adt/datetime.c）における詳細なエンドツーエンドの流れや呼び出し関係については、提供されたコンテキストからは不明です。

重要な不変条件

  • **入力文字の制限**: 解析対象の文字列 str の先頭文字は、数字、マイナス記号（-）、または小数点（.）である必要があります。これらに該当しない場合は DTERR_BAD_FORMAT が返されます。
  • **範囲制限**: 抽出された数値は INT_MIN から INT_MAX の範囲内に収まる必要があります。範囲外（オーバーフロー）の場合は DTERR_FIELD_OVERFLOW が返されます。

よくある落とし穴

  • **コンパイル最適化の影響**: 浮動小数点演算（strtod や floor）に依存しているため、コンパイル時に -ffast-math などの最適化オプションを使用すると、浮動小数点の挙動が変わり、解析精度や結果に影響を与える可能性があります。
  • **エラーハンドリング**: 数値として解釈できない文字列が渡された場合の DTERR_BAD_FORMAT や、オーバーフロー時の DTERR_FIELD_OVERFLOW などのエラー戻り値を、呼び出し側で適切にハンドリングする必要があります。
