Syn

概念

Syn は、PostgreSQLの全文検索（Text Search）機能において、同義語（シノニム）辞書のエントリを表現するために使用される構造体です。 この構造体は、検索対象となる入力単語（キー）と、それに対応する置換後または展開用の同義語（値）のペアを保持します。

ただし、PostgreSQLのソースツリー内には、用途や実装が異なる2つの独立した Syn 構造体が存在します。 1. **contrib/dict_xsyn/dict_xsyn.c の Syn**: 拡張シノニム辞書（dict_xsyn）用。キーに対して、未パースのシノニムリスト（文字列）を保持します。 2. **src/backend/tsearch/dict_synonym.c の Syn**: 標準の同義語辞書（dict_synonym）用。入力単語と置換後の単語の1対1のペア、およびその長さやフラグを保持します。

なぜ重要か

全文検索において、表記揺れや同義語を考慮した検索（シノニム展開・置換）を高速に行うために不可欠なデータ構造です。 辞書ファイルからロードされた大量の単語ペアをメモリ上に効率的に配置し、二分探索（bsearch）による $O(\log n)$ の高速なルックアップを実現するための基盤となります。

関与するモジュール

  • module:contrib/dict_xsyn
  • module:src/backend/tsearch

主要エンティティ

  • symbol:contrib/dict_xsyn/dict_xsyn.c#Syn: 拡張シノニム辞書用の構造体。key（検索キー）と value（未パースのシノニムリスト）を持ちます。
  • symbol:src/backend/tsearch/dict_synonym.c#Syn: 標準同義語辞書用の構造体。in（入力文字列）、out（出力文字列）、outlen（出力長）、flags（制御フラグ）を持ちます。

エンドツーエンドの流れ

辞書の初期化から検索時までの処理の流れは、両モジュールで類似しています。

1. **辞書の初期化とロード**:

  • contrib/dict_xsyn では、dxsyn_init 内で外部ファイルから辞書データが読み込まれ、read_dictionary を介して Syn 構造体の配列が構築されます。
  • src/backend/tsearch では、dsynonym_init 内で辞書ファイルから単語ペアが読み込まれ、Syn 構造体の配列が構築されます。

    • 2. **ソート**:

  • ロードされた Syn 配列は、検索キー（contrib/dict_xsyn では key、src/backend/tsearch では in）を基準として qsort により昇順にソートされます。

    • 3. **検索と置換/展開**:

  • contrib/dict_xsyn では、dxsyn_lexize において入力単語をキーとして bsearch を行い、一致した Syn の value（未パース文字列）をパースしてシノニムリストを展開します。
  • src/backend/tsearch では、dsynonym_lexize において入力単語をキーとして bsearch を行い、一致した Syn の out を置換後の単語として返します。

※なお、辞書ファイルのパース処理の詳細や、メモリ確保（palloc）の具体的な失敗時のエラーハンドリングなど、詳細なエンドツーエンドのフローの一部は本コンテキストからは完全には特定されていません。

重要な不変条件

  • **ソート済みの維持**: 検索（bsearch）を正しく動作させるため、Syn 配列は初期化時に必ず検索キー（key または in）の昇順でソートされていなければなりません。
  • **メモリ管理**: Syn 構造体内の文字列ポインタ（key/value、in/out）は、palloc 等によって確保されたメモリ領域を指す必要があります。

よくある落とし穴

  • **同名構造体の混同**: contrib/dict_xsyn/dict_xsyn.c と src/backend/tsearch/dict_synonym.c の両方に Syn という名前の構造体が定義されていますが、これらは全く別個の定義であり、フィールド構成も異なります。コードの修正やデバッグ時には、対象としているファイルがどちらであるかを厳密に確認する必要があります。
  • **未パース文字列の扱い（contrib/dict_xsyn）**: contrib/dict_xsyn の Syn における value フィールドは、パース前のシノニムリスト文字列です。利用時には dxsyn_lexize 内で適切にパース処理を行う必要があります。
