add_size_error

概念

add_size_error は、メモリ確保サイズの計算中に加算オーバーフローが発生したことを報告し、プロセスを安全に異常終了させるためのエラーハンドラ関数です。

なぜ重要か

メモリ割り当て時のサイズ計算における整数オーバーフローは、バッファオーバーフローなどの深刻なセキュリティ脆弱性やメモリ破損を引き起こす原因となります。add_size_error は、オーバーフローが検出された際に安全かつ確実にプロセスを終了させ、不正なメモリ確保を未然に防ぐ役割を果たします。

関与するモジュール

主要エンティティ

  • **symbol:src/common/fe_memutils.c#add_size_error**

    • フロントエンドのメモリユーティリティで定義されているエラーハンドラ関数。標準エラー出力へのメッセージ出力とプロセス終了を担います。

  • **symbol:src/backend/utils/mmgr/mcxt.c#add_size_error**

    • バックエンドのメモリコンテキスト管理モジュールから参照される、または定義されている同名のエラーハンドラ（詳細な実装や関係性は未検証）。

エンドツーエンドの流れ

1. **オーバーフローの検出**: src/common/fe_memutils.c 内のサイズ計算ロジック（または src/backend/utils/mmgr/mcxt.c）において、メモリ確保サイズ（Size 型、通常 size_t）の加算時にオーバーフローが検出されます。 2. **ハンドラの呼び出し**: オーバーフロー検出直後に add_size_error(s1, s2) が呼び出されます。 3. **エラー報告**: 標準エラー出力（stderr）に対して、invalid memory allocation request size s1 + s2 という形式のエラーメッセージを出力します。このメッセージは _() マクロによる国際化に対応しています。 4. **プロセスの終了**: exit(EXIT_FAILURE) を実行し、プロセスを終了します。

※ バックエンド側（mcxt.c）における具体的な呼び出しフローや、フロントエンドとバックエンド間での詳細な連携フローの全容は現時点では未検証です。

重要な不変条件

  • **非復帰性**: 本関数には pg_noreturn 属性が付与されており、呼び出し元へ制御が戻ることは絶対にありません。
  • **呼び出しの前提**: メモリ確保サイズの加算においてオーバーフローが検出された直後に呼び出されることを想定しています。

よくある落とし穴

  • **検知漏れ**: 呼び出し元におけるサイズ計算のオーバーフロー検知ロジック自体に不備がある場合、本関数は呼び出されず、不正なサイズでのメモリ確保が進行してしまう可能性があります。
  • **コンテキストの混在**: 本関数はフロントエンドおよびバックエンドの両方のコンテキストから参照される可能性があるため、変更を加える際は双方の動作環境への影響を考慮する必要があります。
