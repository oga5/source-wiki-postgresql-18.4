COMPAT_MODE

概念

COMPAT_MODE は、PostgreSQLのECPG（Embedded SQL for C）において、ターゲットとするデータベース製品の互換動作モードを識別・制御するための列挙型（enum）です。 プリプロセッサ（preproc）によるコード生成時や、ランタイムライブラリ（ecpglib）による実行時のデータ変換、SQL構文解析、エラー処理などの挙動を切り替えるために使用されます。

なぜ重要か

ECPGは、PostgreSQL標準の動作だけでなく、他社製データベース（InformixやOracleなど）向けに書かれた埋め込みSQLコードの移行や互換動作をサポートしています。COMPAT_MODE を用いて適切なモードを指定することで、それぞれのデータベース特有のデータ型の解釈や型変換ルール、SQL解釈を適用できるようになります。

関与するモジュール

  • module:src/interfaces/ecpg/ecpglib
  • module:src/interfaces/ecpg/preproc

主要エンティティ

  • symbol:src/interfaces/ecpg/ecpglib/ecpglib_extern.h#COMPAT_MODE: ランタイムライブラリ側で定義されている列挙型。接続管理やデータ変換関数などで参照されます。
  • symbol:src/interfaces/ecpg/preproc/preproc_extern.h#COMPAT_MODE: プリプロセッサ側で定義されている列挙型。SQL構文解析やCコード生成時のルール切り替えに使用されます。

どちらの定義も以下の形状を持ちます。

enum COMPAT_MODE
{
    ECPG_COMPAT_PGSQL = 0,
    ECPG_COMPAT_INFORMIX,
    ECPG_COMPAT_INFORMIX_SE,
    ECPG_COMPAT_ORACLE
};

エンドツーエンドの流れ

1. **プリプロセス時**: プリプロセッサが指定された互換モード（COMPAT_MODE）に基づき、SQLの解釈やCコード生成時の型変換ルールを決定します。 2. **実行時（接続・実行）**: ecpglib 内の接続管理（struct connection）やSQL実行の文脈（struct statement）において、このモードが保持・参照されます。 3. **データ変換時**: data.c などの関数において、COMPAT_MODE に応じた特定のデータベース製品特有のデータ変換やエラー処理が選択的に実行されます。

なお、プリプロセッサからランタイムへこのモード情報がどのように引き渡されるかなど、詳細なエンドツーエンドの制御フローの全容は、提供された情報からは完全には明らかになっていません。

重要な不変条件

  • COMPAT_MODE は 0 から始まる整数値として割り当てられます（ECPG_COMPAT_PGSQL = 0）。
  • struct connection 構造体は、接続状態の一部としてこの COMPAT_MODE を保持します。
  • プリプロセッサおよび ecpglib の各関数において、互換性モードを引数として受け取り、条件分岐の基準として利用されます。

よくある落とし穴

  • **未定義値の代入**: 列挙型で定義されていない整数値がキャストされて代入された場合、ライブラリ内の条件分岐で予期せぬ動作を引き起こす可能性があります。
  • **モードの不整合**: プリプロセッサによるコード生成時と、実行時のライブラリ処理（ecpglib）の間で互換性モードに不整合が生じた場合、予期しないSQL構文エラーやデータ変換エラーが発生する原因となります。
  • **マクロとの整合性**: ソースコード内では INFORMIX_MODE や ORACLE_MODE といったマクロが頻繁に使用されているため、これらと COMPAT_MODE 列挙型との整合性に注意する必要があります。
