ArrayIterator

概念

ArrayIterator は、PostgreSQLおよびその拡張モジュールにおいて、配列（ArrayType）の要素を順次走査・処理するための仕組みです。

この概念には、コアシステムが提供する汎用的なイテレータ構造体（ArrayIterator）と、特定の拡張モジュール（ltree）内で定義されている静的なヘルパー関数（array_iterator）の2つの異なる実装が存在します。

なぜ重要か

配列の走査処理を抽象化し、多次元配列の処理や要素ごとのコールバック適用を簡潔に行うために重要です。

  • **コアの ArrayIterator**: 配列のメタデータを考慮しつつ、配列要素を順次 Datum として取り出す抽象化レイヤーを提供し、呼び出し側が配列の内部表現を直接操作する複雑さを排除します。
  • **ltree の array_iterator**: ltree 配列に対して特定の条件判定（コールバック関数）を適用する処理を共通化し、コードの重複を避けます。

関与するモジュール

  • module:contrib/ltree
  • module:src/include/utils

主要エンティティ

  • **symbol:src/include/utils/array.h#ArrayIterator**: PostgreSQLコアが提供する、配列を反復処理するための抽象化されたイテレータ構造体（ArrayIteratorData * の不透明ポインタ）。
  • **symbol:contrib/ltree/_ltree_op.c#array_iterator**: ltree 型の配列を走査し、各要素に対して指定されたコールバック関数を適用して条件判定を行う static 関数。

エンドツーエンドの流れ

コアの ArrayIterator と ltree の array_iterator は独立して動作するため、統一された単一のエンドツーエンドの流れは存在しません。それぞれの処理の流れは以下の通りです。

コアの ArrayIterator の流れ

1. **初期化**: array_create_iterator を呼び出し、対象の ArrayType 配列、スライス次元数、メタデータ（ArrayMetaState）を渡して ArrayIterator を生成します。 2. **反復処理**: array_iterate を繰り返し呼び出し、次の要素が存在する限り Datum 値と NULL フラグを取得します。 3. **解放**: 走査完了後（array_iterate が false を返した後）、array_free_iterator を呼び出してイテレータのリソースを解放します。

ltree の array_iterator の流れ

1. **呼び出し**: ltree_isparent などの関数から、ArrayType 配列、コールバック関数（PGCALL2）、パラメータ等を指定して array_iterator を呼び出します。 2. **検証**: 配列の次元数と NULL の有無をチェックします。 3. **ループと評価**: 配列の各要素に対して、DirectFunctionCall2 を介してコールバック関数を実行します。 4. **早期返却**: コールバックが真を返した場合は即座に true を返し、必要に応じて合致した要素へのポインタを found に格納します。全要素の走査が終わっても条件を満たさない場合は false を返します。

重要な不変条件

  • **コアの ArrayIterator**:
  • array_iterate を呼び出す前に、必ず array_create_iterator で有効なイテレータを初期化しなければなりません。
  • 反復処理の終了後、必ず array_free_iterator を呼び出してメモリを解放しなければなりません。
  • **ltree の array_iterator**:
  • 入力配列は1次元配列でなければなりません（多次元配列は許容されません）。
  • 入力配列は NULL 値を含んではなりません。

よくある落とし穴

  • **同名シンボルの混同**: contrib/ltree/_ltree_op.c の array_iterator は static 関数であり、src/backend/utils/adt/array_userfuncs.c に存在する同名のシンボルや、コアの ArrayIterator 構造体とは異なります。
  • **リソースリーク**: コアの ArrayIterator を使用する際、走査完了後に array_free_iterator の呼び出しを怠るとメモリリークの原因となります。
  • **エラーの発生**: ltree の array_iterator は、多次元配列や NULL を含む配列が渡された場合に ereport によるエラー（ERRCODE_ARRAY_SUBSCRIPT_ERROR や ERRCODE_NULL_VALUE_NOT_ALLOWED）を発生させるため、呼び出し側で事前チェックまたは適切なデータ保証が必要です。
