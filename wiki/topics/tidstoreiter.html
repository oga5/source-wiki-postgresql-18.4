TidStoreIter

概念

TidStoreIter は、TidStore に格納されたすべての TID（ItemPointerData）を順次走査するためのイテレータ状態を保持する構造体です。

ヘッダーファイル src/include/access/tidstore.h では不透明な構造体（Opaque struct）として宣言されており、実装ファイル src/backend/access/common/tidstore.c において具体的なメンバが定義されています。

なぜ重要か

TidStore は、VACUUM 処理（vacuumlazy.c など）において不要なタプルの TID を効率的に管理するために使用されます。TidStoreIter は、対象の TidStore が共有メモリ上にあるかローカルメモリ上にあるかにかかわらず、内部のラディックツリー（lib/radixtree.h）の複雑な走査ロジックをカプセル化し、呼び出し側にシンプルなブロック単位の走査インターフェースを提供するために不可欠です。

主要エンティティ

  • **symbol:src/include/access/tidstore.h#TidStoreIter**

    • 外部モジュールに公開される不透明な構造体定義です。呼び出し側は内部の実装詳細に依存せずにイテレータを扱えます。

  • **symbol:src/backend/access/common/tidstore.c#TidStoreIter**

    • 具体的な構造体の定義です。対象の TidStore へのポインタ、共有/ローカル用のラディックツリーイテレータの共用体（tree_iter）、および走査結果を格納する TidStoreIterResult output を保持します。

エンドツーエンドの流れ

1. **イテレータの生成と初期化** 呼び出し側（vacuumlazy.c やテストモジュールなど）が TidStoreBeginIterate を呼び出します。これにより、呼び出し側のメモリコンテキスト上に TidStoreIter が palloc0 で確保され、初期化されます。 2. **順次走査** TidStoreIterateNext を繰り返し呼び出します。内部的には、TidStore の種類に応じて tree_iter.shared または tree_iter.local を介して次のブロックおよびオフセットを検索し、結果を output に格納して返します。 3. **リソースの解放** 走査が完了（または途中で中断）した際、呼び出し側は TidStoreEndIterate を呼び出してイテレータに割り当てられたメモリを解放します。

*注意: 走査中に TidStore 自体が変更された場合の挙動や、並行アクセス時における具体的なロックの獲得・解放の全ライフサイクルについては、提供された情報からは完全には明らかになっていません。*

重要な不変条件

  • **メモリ配置の不変条件**

    • 走査対象の TidStore が共有メモリ上にある場合でも、TidStoreIter 構造体自体のメモリは常に呼び出し側のローカルメモリコンテキストに確保されます。

  • **共用体アクセスの整合性**

    • 内部の tree_iter は共用体であるため、TidStoreIsShared() の判定結果に基づいて、対応するアクティブなメンバ（shared または local）に対してのみ正しくアクセスされなければなりません。

  • **リソース管理の義務**

    • メモリリークを防ぐため、TidStoreBeginIterate で開始した走査は、必ず TidStoreEndIterate の呼び出しで終了しなければなりません。

よくある落とし穴

  • **不透明型への直接アクセス**

    • TidStoreIter は tidstore.h で不透明型として定義されているため、外部モジュールから直接構造体のメンバ（tree_iter や output など）にアクセスしようとするとコンパイルエラーになります。必ず提供されている API 関数を経由して操作してください。

  • **共有メモリ走査時のロック欠如**

    • 共有メモリ版の TidStore を走査する際、適切なロックが保持されていない状態で走査を行うと、未定義の動作を引き起こす可能性があります。

  • **空のストアに対する考慮**

    • TidStore が空の場合、イテレータは即座に終了状態を返します。呼び出し側は初回の TidStoreIterateNext で走査が終了する可能性を考慮する必要があります。
