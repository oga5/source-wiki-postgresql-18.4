debug

概念

debug は、PostgreSQLの複数の独立したモジュール（バックエンドのテキスト検索ステム処理、初期化ツール、リカバリツール、テストスイート）において、デバッグ情報の出力やデバッグモードの有効化を制御するために定義されているシンボル（主にフラグ変数やマクロ）です。

なぜ重要か

各モジュールにおいて、開発やトラブルシューティング時に詳細な実行ログや内部状態を出力するために、この debug シンボルが参照されます。これにより、通常の実行フローを妨げることなく、必要に応じて詳細な診断情報を得ることができます。

各モジュールにおける debug シンボルの具体的な初期化、コマンドライン引数からのパース、およびそれらがどのように条件分岐やログ出力に影響を与えるかという具体的なエンドツーエンドの流れは、提供されたメタデータからは**未検証であり、詳細なフローは不明**です。

重要な不変条件

各 debug 変数が保持すべき値の範囲（例: booleanとしての0/1、あるいは詳細度を表す整数値）や、マルチスレッド/マルチプロセス環境におけるスレッドセーフ性などの**重要な不変条件は、現時点では不明**です。

よくある落とし穴

各モジュールで個別に debug という同名のシンボルが定義されているため、これらはグローバルに共有された単一の変数ではなく、それぞれのプログラムやライブラリ内で独立して動作します。そのため、あるツール（例: pg_rewind）でデバッグを有効にしても、他のコンポーネント（例: initdb やバックエンド）の debug フラグには直接影響を与えない点に注意が必要です。また、具体的な競合や副作用に関する落とし穴は**現時点では不明**です。
