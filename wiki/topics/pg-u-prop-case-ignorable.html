PG_U_PROP_CASE_IGNORABLE

概念

PG_U_PROP_CASE_IGNORABLE は、指定された Unicode コードポイントが大文字・小文字の変換（Casing）において無視されるべき文字（Case-ignorable）であるかを判定するためのプロパティ定義およびビットマスクです。

なぜ重要か

Unicode 規格に準拠した大文字・小文字の変換処理（Case mapping/folding）を正確に行うためには、特定の文脈で無視されるべき文字を正しく識別する必要があります。このマクロと関連する判定関数は、PostgreSQL 内部での Unicode 文字列操作の正確性を担保するために機能します。

関与するモジュール

  • symbol:src/common/unicode_category.c#pg_u_prop_case_ignorable

エンドツーエンドの流れ

1. **呼び出し**: src/common/unicode_case.c などの大文字・小文字処理モジュールから pg_u_prop_case_ignorable(code) が呼び出されます。 2. **ASCII 最適化**: 入力コードポイント code が 0x80 未満（ASCII 範囲）の場合、unicode_opt_ascii テーブルを直接参照し、ビットマスク PG_U_PROP_CASE_IGNORABLE を用いて高速に判定します。 3. **範囲検索**: code が 0x80 以上の非 ASCII 文字の場合、unicode_case_ignorable テーブルに対して range_search を実行し、該当する範囲に含まれるかを判定します。

※なお、呼び出し元である src/common/unicode_case.c における具体的な大文字・小文字変換処理の全体的なエンドツーエンドの制御フローの詳細は、現時点では提供された情報からは不明です。

重要な不変条件

  • **静的テーブルの不変性**: 判定に使用される unicode_opt_ascii および unicode_case_ignorable テーブルは固定されており、実行時に変更されません。
  • **Unicode 規格への準拠**: 本機能の判定基準は Unicode 規格の安定性ポリシーに基づいています。

よくある落とし穴

  • **テーブルの自動生成**: 本機能で参照される unicode_category_table.h などの定義はスクリプトによって自動生成されるため、手動で直接編集すると、将来の Unicode バージョンアップ時に変更が上書きされて失われる可能性があります。
  • **範囲外の入力**: code が有効な Unicode コードポイントの範囲外である場合の挙動は、内部の range_search の実装に依存します。
