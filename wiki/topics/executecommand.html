executeCommand

概念

executeCommand は、PostgreSQLのフロントエンドツール（CLIツール）において、結果セットを返さないSQLコマンドを実行するためのユーティリティ関数です。クエリの実行に失敗した場合は、エラーログを出力した上でプロセスを即座に異常終了（exit(1)）させる「失敗即終了」の設計パターンを持っています。

なぜ重要か

フロントエンドツール（pg_dumpall や pg_amcheck など）において、データベース接続の確立後に実行する初期化クエリや設定変更クエリなど、失敗が許されない定型的なSQLコマンドの実行コードを簡素化するために重要です。各呼び出し元で個別にエラーハンドリングや接続のクローズ処理（PQfinish）を記述する重複を排除し、一貫したエラーハンドリングとログ出力を提供します。

エンドツーエンドの流れ

1. **呼び出し**: フロントエンドツール（例: pg_dumpall）から、有効な PGconn 接続、実行したいSQLクエリ文字列、およびエコー表示の有無（echo）を指定して executeCommand を呼び出します。 2. **エコー表示**: echo 引数が true の場合、実行前にクエリ文字列を標準出力に表示します。 3. **クエリ実行**: libpq の PQexec を用いて、データベースに対してクエリを送信します。 4. **結果判定**:

  • **成功時**: クエリの実行結果ステータスが PGRES_COMMAND_OK である場合、PQclear で結果オブジェクトを解放し、呼び出し元に制御を戻します。
  • **失敗時**: 実行結果が NULL または PGRES_COMMAND_OK 以外の場合、pg_log_error および pg_log_error_detail を用いてエラーメッセージとクエリ内容をログ出力し、PQfinish で接続を閉じた後、exit(1) によりプロセスを終了します。

※なお、src/bin/pg_dump/pg_dumpall.c における executeCommand の具体的な内部フローや、src/fe_utils/query_utils.c の実装との詳細な相互作用（ラッパーなのか、あるいは同名の別実装なのかなど）については、提供された情報からは未検証です。

重要な不変条件

  • **接続の有効性**: 入力される conn は、有効な PGconn オブジェクトでなければなりません。
  • **期待される結果ステータス**: 実行されるクエリは、結果セットを返さない（PGRES_COMMAND_OK を期待する）ものである必要があります。
  • **失敗時のプロセス終了**: クエリ実行が失敗した場合、関数は呼び出し元に制御を戻さず、必ずプロセスを異常終了（exit(1)）させます。

よくある落とし穴

  • **エラーハンドリングの制限**: 本関数はエラー発生時にプロセスを即座に終了させるため、エラーを呼び出し元でキャッチしてリトライする、あるいはトランザクションをロールバックして処理を続行する、といった柔軟なエラーハンドリングを行うことはできません。そのような制御が必要な場合は、本関数を使用せず直接 libpq を使用する必要があります。
  • **結果セットを返すクエリへの不適合**: SELECT などの結果セットを返すクエリに対して使用すると、ステータスが PGRES_COMMAND_OK にならないため、クエリ自体が成功していてもエラーとみなされプロセスが終了してしまいます。
