state

概念

PostgreSQLのソースコードにおいて、state という名前の構造体（struct state）は、全く異なる2つのモジュールで独立して定義され、それぞれの文脈における「状態」を管理するために使用されています。

1. **正規表現NFA（非決定性有限オートマトン）の状態** (src/include/regex/regguts.h): 正規表現のコンパイルおよび最適化プロセスにおいて、状態遷移図（NFA）の各ノードを表現します。 2. **タイムゾーンデータベースの状態** (src/timezone/pgtz.h): IANAタイムゾーンデータベース（tzfile）のバイナリ形式をメモリ上に展開し、タイムゾーンの遷移情報、閏秒情報、および時刻型情報を保持します。

なぜ重要か

  • **正規表現NFAの状態**:

    • 正規表現パターンマッチングエンジンにおいて、効率的なNFAの構築、最適化、およびグラフ走査（到達可能性判定など）を行うための基盤となるデータ構造です。

  • **タイムゾーンデータベースの状態**:

    • セッションやログのタイムゾーン変換処理において、正確なオフセット、夏時間（DST）、および閏秒の計算を高速に行うために不可欠な情報を保持します。

関与するモジュール

  • module:src/include/regex
  • module:src/timezone

主要エンティティ

  • **symbol:src/include/regex/regguts.h#state**:

    • NFAの各状態を表現する構造体。状態番号、入力/出力アークのリスト、走査用の一時ポインタ、およびライブ状態/空き状態管理用の双方向リンクを保持します。

  • **symbol:src/timezone/pgtz.h#state**:

    • タイムゾーンの遷移時刻、型情報、閏秒情報などを保持する構造体。上位の pg_tz 構造体の内部状態として保持されます。

エンドツーエンドの流れ

これら2つの構造体は、それぞれ独立したフローで動作します。

1. 正規表現NFAの構築と走査

  • **構築**: src/backend/regex/regcomp.c において、正規表現のコンパイル中に struct state がインスタンス化され、struct arc を介して他の状態と接続されます。
  • **走査と最適化**: グラフ走査アルゴリズム（到達可能性判定や最適化など）において、tmp ポインタや next/prev リストを用いて状態が走査・管理されます。

2. タイムゾーンデータのロードと変換

  • **ロード**: src/timezone/localtime.c 内の tzload() が pg_open_tzfile() を介してタイムゾーンファイルを特定し、その内容を解析して struct state の各フィールド（ats、ttis、lsis など）を構築します。
  • **変換**: タイムゾーン変換処理を行う関数群（localtime.c など）が、この構築された state 構造体を参照して、適切なタイムゾーンオフセットや夏時間フラグを決定します。

※なお、メモリ確保失敗時などの詳細なエラーハンドリングの全容や、具体的なファイル破損時のフォールバックフローの詳細は、提供されたモジュール情報からは完全には特定されていません。

重要な不変条件

正規表現NFAの状態

  • no フィールドには、非負の整数または FREESTATE (-1) が格納されます。
  • ins および outs は、それぞれ struct arc の有効な連結リストを指す必要があります。
  • next および prev フィールドは、NFA内の全ライブ状態を管理するための双方向リストを構成します。

タイムゾーンデータベースの状態

  • 配列サイズ（ats、types、ttis、lsis）は、tzfile.h で定義された TZ_MAX_TIMES、TZ_MAX_TYPES、TZ_MAX_LEAPS などの定数によって決定されます。
  • defaulttype は、最近の tzdb リリースでは常に 0 ですが、2018e 以前のデータでは非ゼロとなる可能性があります。

よくある落とし穴

正規表現NFAの状態

  • **リスト操作の混同**: next フィールドは「ライブ状態の連結」と「空き状態の連結（再利用）」という二つの異なる役割を兼ねているため、状態のリスト操作を行う際は、現在の状態がどちらのリストに属しているかを厳密に区別する必要があります。
  • **一時ポインタの競合**: tmp ポインタは再帰的なアルゴリズムやグラフ走査で頻繁に書き換えられるため、複数のアルゴリズムを並行して実行する際には競合に注意する必要があります。

タイムゾーンデータベースの状態

  • **サイズ制限**: TZ_MAX_TIMES を超える遷移情報を持つファイルは、読み込み時にエラーとなる可能性があります。
  • **直接操作の禁止**: 外部モジュールからタイムゾーン機能を利用する場合は、本構造体を直接操作せず、pgtime.h のインターフェースを使用する必要があります。
  • **ファイル整合性**: 破損したタイムゾーンファイルが入力された場合、構造体の各フィールドに不正な値が設定されるリスクがあります。

次に読む場所

  • src/include/regex/regguts.h: 正規表現NFAの state および arc 構造体の定義。
  • src/timezone/pgtz.h: タイムゾーンの state 構造体および関連定数の定義。
  • src/backend/regex/regcomp.c: NFAの構築、最適化、および状態管理の実装。
  • src/timezone/localtime.c: タイムゾーンファイルのロード（tzload）と時刻変換処理の実装。
