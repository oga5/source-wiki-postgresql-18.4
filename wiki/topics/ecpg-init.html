ecpg_init

概念

ecpg_init は、PostgreSQLのECPG（Embedded SQL in C）において、異なる目的で定義されている2つの同名関数です。

1. **ECPGライブラリ（ecpglib）における ecpg_init**: 各種SQL操作（接続、実行、準備など）の開始前に、スレッド固有のSQLCA（SQL Communication Area）を初期化し、データベース接続の有効性を検証するガード関数。 2. **ECPGテストドライバ（pg_regress_ecpg）における ecpg_init**: 回帰テストの実行時に呼び出される初期化用の関数（現在は空の実装）。

なぜ重要か

  • **ecpglib側**: SQL実行や接続管理などの各操作の冒頭で呼び出され、スレッドごとのSQLCAの初期化と接続の存在確認を保証する役割を持ちます。これにより、未初期化状態でのアクセスや無効な接続によるクラッシュを防ぎます。
  • **テストドライバ側**: 将来的にテストドライバ固有の初期化処理が必要となった場合の拡張ポイントとして定義されています。

関与するモジュール

  • symbol:src/interfaces/ecpg/test/pg_regress_ecpg.c#ecpg_init: テストドライバ用のローカルな初期化関数。

エンドツーエンドの流れ

1. ECPGライブラリにおける処理フロー

  • connect.c、execute.c、prepare.c などの呼び出し元から ecpg_init が呼び出されます。
  • ECPGget_sqlca() を呼び出して、現在のスレッドに対応する sqlca_t 構造体を取得します。
  • ecpg_init_sqlca() を呼び出して、取得したSQLCAを初期化します。
  • 渡された connection 構造体ポインタが NULL でないか検証します。
  • 検証が成功すれば true を返し、失敗した場合は ecpg_raise を呼び出してエラーを設定し false を返します。

2. テストドライバにおける処理フロー

  • テストドライバの起動時に ecpg_init(argc, argv) が呼び出されますが、現時点では内部処理が空であるため、何も行わずに終了します。なお、テストドライバ内での具体的な呼び出し元は提供された情報からは特定されていません。

重要な不変条件

  • **ecpglib側**:
  • ecpg_init が false を返す場合、内部で ecpg_raise が呼び出され、適切なエラー状態（ECPG_OUT_OF_MEMORY または ECPG_NO_CONN）が設定されていなければなりません。
  • ECPGget_sqlca() が正常に動作する状態（呼び出し可能）である必要があります。
  • **テストドライバ側**:
  • 現在は空の関数であるため、実行時の不変条件は特にありません。

よくある落とし穴

  • **同名関数の混同**: ecpglib の ecpg_init（boolを返し、接続情報を引数に取る）と、テストドライバ pg_regress_ecpg.c 内の ecpg_init（static voidで、コマンドライン引数を取る）は、名前は同じですがシグネチャも役割も完全に異なります。静的解析ツールやコードリーディングの際にこれらを混同しないよう注意が必要です。
