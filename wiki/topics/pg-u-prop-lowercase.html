pg_u_prop_lowercase

概念

pg_u_prop_lowercase は、指定された Unicode コードポイント（pg_wchar）が Unicode 規格における「小文字（Lowercase）」プロパティを保持しているかどうかを判定するための機能です。

なぜ重要か

PostgreSQL 内部で Unicode 文字のカテゴリ判定を正確かつ高速に行うために重要です。特に ASCII 範囲（0x80 未満）の文字に対しては、ビットマスクを用いた定数時間での高速な判定を提供し、それ以上の範囲の文字に対しても二分探索を用いて効率的に判定する仕組みを提供します。

関与するモジュール

  • module:src/common
  • module:src/include/common

主要エンティティ

  • **symbol:src/common/unicode_category.c#pg_u_prop_lowercase**: 指定された Unicode コードポイントが小文字プロパティを持つかを判定する関数。
  • **symbol:src/include/common/unicode_category_table.h#PG_U_PROP_LOWERCASE**: ASCII 範囲の文字が小文字プロパティを持つかを判定する際に使用されるビットマスク定数。

エンドツーエンドの流れ

1. **入力の受け取り**: 呼び出し元から pg_wchar code として Unicode コードポイントが pg_u_prop_lowercase 関数に渡されます。 2. **ASCII 範囲の判定**:

  • 入力コードポイントが 0x80 未満の場合、静的テーブル unicode_opt_ascii を参照します。
  • PG_U_PROP_LOWERCASE ビットマスクを用いて、定数時間（O(1)）で小文字であるかを判定します。

    • 3. **ASCII 範囲外の判定**:

  • 入力コードポイントが 0x80 以上の場合、静的テーブル unicode_lowercase に対して range_search を実行します。
  • 二分探索により、該当するコードポイント範囲が存在するかを特定します。

    • 4. **結果の返却**: 判定結果を bool 値（小文字であれば true、それ以外は false）として呼び出し元に返します。

※なお、呼び出し元（category_test.c や unicode_category.c 内の他の関数など）から本関数が呼び出された後の、PostgreSQL 全体における上位レベルでの具体的なエンドツーエンドのユースケースやフローの全貌は、提供された情報からは完全には特定されていません。

重要な不変条件

  • **入力型**: 入力されるコードポイントは pg_wchar 型でなければなりません。
  • **判定基準の固定**: Unicode 規格の安定性ポリシーに基づき、判定に使用されるカテゴリ値は固定されています。
  • **ASCII 高速化**: 0x80 未満のコードポイントに対しては、必ず PG_U_PROP_LOWERCASE ビットマスクを用いた高速パスが適用されます。

※システム全体におけるその他の詳細な不変条件は、提供された情報からは完全には特定されていません。

よくある落とし穴

  • **パフォーマンス特性の差異**: 0x80 未満の ASCII 文字と 0x80 以上の文字では、判定アルゴリズムが O(1) のテーブル参照から二分探索（range_search）へと切り替わるため、処理コストが異なる点に注意が必要です。
  • **テーブルの直接編集**: 判定に使用される unicode_lowercase や unicode_opt_ascii などの内部テーブルは、unicode_category_table.h を生成するスクリプトによって管理されています。これらを手動で直接書き換えると、不整合が発生する原因になります。
