PG_U_PROP_HEX_DIGIT

概念

PG_U_PROP_HEX_DIGIT は、PostgreSQLにおいて指定された Unicode コードポイントが Unicode 規格の「16進数桁（Hex Digit）」プロパティを保持しているかを判定するために使用されるビットフラグ（またはプロパティ識別子）です。

なぜ重要か

Unicode コードポイントが16進数として有効かどうかを判定する処理は、SQLのパースや各種データ型のバリデーションなどで必要となります。PG_U_PROP_HEX_DIGIT を用いた ASCII 範囲の高速判定（定数時間）と、それ以上の範囲に対する二分探索を組み合わせることで、Unicode 規格に準拠した正確な判定を効率的に行うことができます。

関与するモジュール

  • module:src/common
  • module:src/include/common

主要エンティティ

  • **symbol:src/include/common/unicode_category_table.h#PG_U_PROP_HEX_DIGIT**: ASCII 範囲の文字プロパティを保持する unicode_opt_ascii テーブルなどで、16進数桁プロパティを示すビットフラグとして定義されています。
  • **symbol:src/common/unicode_category.c#pg_u_prop_hex_digit**: 指定された Unicode コードポイント（pg_wchar）が16進数桁プロパティを持つかを判定する関数です。

エンドツーエンドの流れ

1. 呼び出し元（src/common/unicode/category_test.c や src/common/unicode_category.c 内の他の処理など）が pg_u_prop_hex_digit(code) を呼び出します。 2. **ASCII 範囲（code < 0x80）の処理**:

  • 静的テーブル unicode_opt_ascii を参照し、対象コードポイントのエントリに対して PG_U_PROP_HEX_DIGIT ビットが立っているかを定数時間（$O(1)$）で判定します。

    • 3. **ASCII 範囲外（code >= 0x80）の処理**:

  • 静的テーブル unicode_hex_digit に対して range_search を実行し、二分探索によってコードポイントが該当する範囲に含まれるかを特定します。

    • 4. 判定結果（bool）を呼び出し元に返します。

※なお、これら以外の具体的な上位機能（SQLパーサや特定のデータ型入力関数など）からこの判定関数に至るまでの完全なエンドツーエンドの呼び出しフローは、提供されたコンテキストからは未詳です。

重要な不変条件

  • **ASCII 判定の整合性**: 0x80 未満のコードポイントに対しては、unicode_opt_ascii テーブル内の PG_U_PROP_HEX_DIGIT ビットの有無が、Unicode 規格における Hex Digit プロパティの有無と完全に一致している必要があります。
  • **探索範囲の不変性**: 0x80 以上のコードポイントに対しては、unicode_hex_digit テーブルに定義された範囲外の入力に対しては常に false を返す必要があります。

よくある落とし穴

  • **テーブルの自動生成**: unicode_hex_digit や unicode_opt_ascii などのテーブルは、Unicode 規格のデータからスクリプトによって自動生成されるヘッダーファイル（unicode_category_table.h）に依存しています。手動でこれらのテーブルや PG_U_PROP_HEX_DIGIT の定義を直接書き換えると、Unicode 規格との不整合や、自動生成スクリプトの再実行時に変更が消失する原因となります。
  • **文字コードの前提**: 入力される code は pg_wchar 型（Unicode コードポイント）である必要があります。マルチバイト文字列（UTF-8 など）をそのまま渡すと、不正な判定結果となります。
