dsm_impl_windows

概念

dsm_impl_windows は、Windows環境においてOSネイティブのAPI（CreateFileMapping や MapViewOfFile など）を使用し、動的共有メモリ（DSM）セグメントの作成、アタッチ、デタッチ、および破棄を行うためのプラットフォーム固有の実装です。

なぜ重要か

PostgreSQLはマルチプロセスアーキテクチャを採用しており、プロセス間でデータを共有するために共有メモリを必要とします。Windows環境においては、Unix系の共有メモリ（POSIXやSysVなど）とは異なるOS固有のAPIを用いて動的共有メモリを管理する必要があり、dsm_impl_windows がその具体的な低レベル操作を提供します。

関与するモジュール

  • module:src/backend/storage/ipc
  • module:src/include/storage

主要エンティティ

  • **symbol:src/backend/storage/ipc/dsm_impl.c#dsm_impl_windows (関数)**: Windows APIを直接呼び出して、指定されたDSM操作（作成、アタッチ、デタッチ、破棄）を実行する中心的な関数です。
  • **symbol:src/include/storage/dsm_impl.h#DSM_IMPL_WINDOWS (マクロ/定数)**: Windows用のDSM実装を識別・有効化するための定義です。

エンドツーエンドの流れ

1. **呼び出し**: src/backend/storage/ipc/dsm_impl.c 内の dsm_impl_op 関数から、操作（dsm_op）を指定して dsm_impl_windows が呼び出されます。 2. **名前解決**: 指定された dsm_handle からセグメント名を生成します。 3. **操作の分岐**:

  • **DSM_OP_DETACH または DSM_OP_DESTROY**: UnmapViewOfFile でメモリマッピングを解除し、CloseHandle でハンドルを閉じます。
  • **DSM_OP_CREATE**: CreateFileMapping を呼び出し、ページファイル上にセグメントを新規作成します。
  • **DSM_OP_ATTACH**: OpenFileMapping を呼び出し、既存のセグメントを開きます。

    • 4. **マッピングとサイズ検証**: CREATE または ATTACH の成功後、MapViewOfFile を使用してプロセスのアドレス空間にセグメントをマッピングし、VirtualQuery を用いて実際のサイズ（mapped_size）を取得・検証します。 5. **状態保存**: impl_private に hmap ハンドルを格納して呼び出し元に返します。

*注意: dsm_impl_op より上位の呼び出し元や、システム全体におけるDSMのライフサイクル全体にわたるエンドツーエンドの流れの詳細は、提供されたコンテキストからは完全には明らかになっていません。*

重要な不変条件

  • **名前付きセグメント**: セグメントは SEGMENT_NAME_PREFIX を用いた名前付き共有メモリセグメントとして管理されます。
  • **自動破棄**: Windowsの仕様により、セグメントへのすべての参照（ハンドル）が閉じられるとオブジェクトは自動的に破棄されます。このため、DSM_OP_DETACH と DSM_OP_DESTROY は実質的に同等の処理として扱われます。
  • **サイズ検証**: マッピングされた領域のサイズは、常に VirtualQuery を使用して検証され、mapped_size に反映されます。

よくある落とし穴

  • **カーネルオブジェクトの生存期間**: Windowsのカーネルオブジェクトは参照カウントで管理されるため、pin/unpin 関数との連携において、ハンドルが意図せず早く閉じられたり、逆に解放されずに残ったりしないよう注意が必要です。
  • **作成時の競合**: CreateFileMapping が ERROR_ALREADY_EXISTS や ERROR_ACCESS_DENIED を返した場合、作成処理は失敗します。
  • **エラーハンドリングとロールバック**: Windows APIの失敗時は、_dosmaperr を介してシステムエラーを errno に変換し、ereport で通知する必要があります。また、失敗時には確保済みのリソース（ハンドルやマッピング）を適切に解放するロールバック処理が不可欠です。
