TestSpec

概念

PostgreSQLのテストスイートにおいて、テストの実行仕様や検証パターンを定義するために用いられるデータ構造です。 この名前を持つ構造体は、用途の異なる2つのテストモジュール（分離レベルテストとIntegerSetのテスト）でそれぞれ独立して定義されています。

なぜ重要か

  • **分離レベルテスト（isolationtester）における重要性**:

    • .spec ファイルから解析されたテスト定義（セットアップSQL、セッション、実行順序であるパーミュテーションなど）を保持するルートデータ構造であり、並行処理やトランザクション分離レベルの検証を自動実行するために不可欠です。

  • **IntegerSetテストにおける重要性**:

    • IntegerSet（整数集合データ構造）の挙動を検証するために、生成する整数のビットパターン、繰り返し間隔、総数などのテスト仕様を定義し、様々な境界条件（Simple-8bエンコーディングの制約など）を網羅したテストケースを容易に追加・管理できるようにします。

関与するモジュール

  • module:src/test/isolation

主要エンティティ

  • symbol:src/test/isolation/isolationtester.h#TestSpec (構造体): 分離レベルテストの仕様全体を保持するルートデータ構造。
  • symbol:src/test/modules/test_integerset/test_integerset.c#test_spec (構造体): IntegerSet のテストにおいて、特定の整数パターンを生成・検証するための設定情報を保持する構造体。

エンドツーエンドの流れ

2つのエンティティは独立しているため、それぞれの流れを以下に示します。

1. 分離レベルテスト (isolationtester)

1. **初期化**: spec_yyparse() により .spec ファイルが読み込まれ、TestSpec 構造体が構築されます。 2. **検証とリンク**: isolationtester.c 内の check_testspec() が、Step や PermutationStep 内のポインタを解決し、実行可能な状態にリンクを構築します。 3. **実行**: 実行エンジンが TestSpec を走査し、定義された sessions に対して setupsqls を実行後、permutations に基づいて SQL を発行します。最後に teardownsql が実行されます。

2. IntegerSetテスト (test_integerset)

1. **定義**: src/test/modules/test_integerset/test_integerset.c 内の test_specs 配列にて、test_spec 構造体のインスタンスが静的に初期化されます。 2. **実行**: test_pattern 関数が引数として test_spec を受け取り、その設定（パターン文字列、間隔、値の数など）を参照して IntegerSet への値の追加や検証ロジックを制御・実行します。

重要な不変条件

  • **分離レベルテスト (TestSpec)**:
  • check_testspec() 関数が呼び出されるまで、構造体内のポインタ（特に PermutationStep 関連）は未解決の状態である可能性があります。
  • nsetupsqls、nsessions、npermutations は、それぞれ対応する配列（setupsqls、sessions、permutations）の要素数を正確に保持している必要があります。
  • **IntegerSetテスト (test_spec)**:
  • test_name および pattern_str は有効な文字列ポインタである必要があります。
  • spacing および num_values は uint64 型の範囲内である必要があります。

よくある落とし穴

  • **分離レベルテスト (TestSpec)**:
  • check_testspec() によるリンク解決に失敗した場合、実行エンジンはテストを正しく実行できません。
  • nsessions や npermutations が 0 の場合、テスト実行ロジックが空の処理を行う可能性があります。
  • 本構造体は specparse.y と密接に関連しているため、テスト定義の構文を変更する場合は、本構造体の定義とパーサーの両方を更新する必要があります。
  • **IntegerSetテスト (test_spec)**:
  • num_values に極端に大きな値を指定した場合、テスト実行時のメモリ消費量や処理時間に影響を与える可能性があります。
  • pattern_str の内容が IntegerSet の許容するビットパターンと整合しない場合、検証ロジックでエラーが発生します。
  • Simple-8b エンコーディングの制約（2^60）をテストする際は、spacing の値が境界条件を満たしているか確認する必要があります。
