pg_usleep

概念

pg_usleep は、プラットフォームに依存しない方法で呼び出し元プロセスを指定されたマイクロ秒数だけ一時停止（スリープ）させるための関数です。PostgreSQLの動作する環境（Unix系、Windows）や、シグナル処理の要件に応じて異なる実装が提供されています。

なぜ重要か

PostgreSQLはマルチプラットフォームで動作するため、OSごとのスリープAPI（Unixの nanosleep や Windowsの SleepEx / WaitForSingleObject）の違いを抽象化する必要があります。特にWindows環境のバックエンドにおいては、単にスリープするだけでなく、待機中にシグナルイベントを監視し、割り込み可能なスリープ（シグナルエミュレーション）を実現するために極めて重要です。

主要エンティティ

  • **symbol:src/backend/port/win32/signal.c#pg_usleep**

    • Windows環境のバックエンド向け実装。シグナルイベント（pgwin32_signal_event）を監視しながら待機し、シグナル受信時に EINTR を設定して復帰します。

  • **symbol:src/port/pgsleep.c#pg_usleep**

    • プラットフォーム非依存の汎用実装。Unix系では nanosleep を使用し、Windows環境（バックエンド以外）では SleepEx を使用して単純にスリープします。

エンドツーエンドの流れ

呼び出し元（contrib/auth_delay/auth_delay.c、src/backend/access/nbtree/nbtpage.c、src/backend/access/transam/xlog.c など）が pg_usleep(microsec) を呼び出した際、プラットフォームおよびビルド環境に応じて以下のいずれかの流れで処理されます。

1. Unix系環境 (src/port/pgsleep.c)

1. microsec を秒（tv_sec）とナノ秒（tv_nsec）に分割し、struct timespec を構築します。 2. nanosleep を呼び出してプロセスを停止させます。

2. Windows環境（汎用 / src/port/pgsleep.c）

1. microsec をミリ秒に換算します（500μs未満は1msに切り上げ、それ以外は四捨五入）。 2. SleepEx を呼び出してプロセスを停止させます。

3. Windows環境（バックエンド / src/backend/port/win32/signal.c）

1. microsec をミリ秒に換算します（500μs未満は1msに切り上げ）。 2. pgwin32_signal_event が未初期化（NULL）の場合、SleepEx を呼び出し、割り込み不可能なスリープを行います。 3. pgwin32_signal_event が初期化済みの場合、WaitForSingleObject を使用して指定時間待機します。 4. 待機中にシグナルイベントが通知された場合（WAIT_OBJECT_0）：

  • pgwin32_dispatch_queued_signals() を呼び出し、キューイングされたシグナルを処理します。
  • errno を EINTR に設定して復帰します。

重要な不変条件

  • **入力値の境界**: microsec が 0 以下の場合は、何も行わずに即座に復帰します（src/port/pgsleep.c）。
  • **最小待機時間（Windows）**: Windows環境では、待機時間が500マイクロ秒未満の場合、最小待機時間は1ミリ秒となります。
  • **シグナル割り込み時のエラー設定**: Windowsバックエンド版において、待機中にシグナルを検知して中断した場合は、必ず errno が EINTR に設定されます。

よくある落とし穴

  • **精度の限界**: OSのスケジューリング精度やカーネルティック周期に依存するため、指定した時間よりも長くスリープする可能性（最大20ms程度の誤差）があります。
  • **応答性の低下**: バックエンドプロセスで pg_usleep を使用して長時間スリープすると、シグナルやラッチへの応答性が低下します。そのため、即時応答や割り込みが必要なバックエンド処理では、pg_usleep ではなく WaitLatch() の使用が推奨されます。
  • **Windowsにおける実装の競合**: src/port/pgsleep.c の Windows 実装は、Windows バックエンド以外の環境向けです。Windows バックエンドでは src/backend/port/win32/signal.c の実装が優先して使用されます。
