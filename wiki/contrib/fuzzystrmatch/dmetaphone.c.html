contrib/fuzzystrmatch/dmetaphone.c

概要

本ファイルは、文字列の音韻的類似性を判定する「Double Metaphone」アルゴリズムの PostgreSQL 実装です。入力された文字列から、発音に基づく「プライマリ」および「オルタネート（代替）」の2種類のコードを生成します。

モジュール内での役割

contrib/fuzzystrmatch モジュールの一部として、SQL から呼び出し可能な dmetaphone および dmetaphone_alt 関数を提供します。主に検索クエリにおける発音の揺れを吸収するためのインデックス作成や、データクレンジングに利用されます。

責務

  • **SQL インターフェース**: PostgreSQL の text 型と C 実装の橋渡し。
  • **音韻コード生成**: Double Metaphone アルゴリズムによる文字列の符号化。
  • **メモリ管理**: PostgreSQL のメモリコンテキスト（palloc/repalloc）を利用したバッファ管理。

重要なシンボル

SQL インターフェース

  • dmetaphone(PG_FUNCTION_ARGS): プライマリコードを返す SQL 関数。
  • dmetaphone_alt(PG_FUNCTION_ARGS): オルタネートコードを返す SQL 関数。

データ構造と管理

  • metastring: 文字列バッファ、長さ、バッファサイズを保持する構造体。
  • NewMetaString, DestroyMetaString: metastring のライフサイクル管理。
  • IncreaseBuffer: バッファの動的拡張。

アルゴリズム実装

  • DoubleMetaphone(char *str, char **codes): アルゴリズムの核となる関数。プライマリとオルタネートを同時に計算し、結果を codes 配列に格納する。
  • MetaphAdd: コード生成バッファへの追記。
  • StringAt, GetAt, SetAt: 文字列の走査と照合を行うヘルパー。
  • IsVowel, SlavoGermanic, MakeUpper: 言語的特性に基づく判定および正規化。

制御フローとデータフロー

1. **入力**: PG_GETARG_TEXT_PP で text 型を受け取り、text_to_cstring で C 文字列に変換。 2. **初期化**: NewMetaString で入力文字列をラップし、大文字化。 3. **メインループ**: DoubleMetaphone 内で文字列を走査し、文字種（A-Z）に応じたルールを適用。 4. **コード生成**: MetaphAdd を用いてプライマリとオルタネートのバッファを構築。 5. **出力**: codes 配列から該当する結果を取り出し、cstring_to_text を経由して PG_RETURN_TEXT_P で返却。

読者向けメモ

  • **メモリ管理**: PostgreSQL のメモリコンテキストに依存しており、pfree を明示的に呼び出さない設計です。これはモジュール再読み込み時の安定性を考慮した実装上の判断です。
  • **パフォーマンス**: 各関数呼び出しでプライマリとオルタネートの両方を計算し、不要な方を破棄する設計のため、若干の非効率性があります。
  • **制約**: 入力文字列が NULL の場合、STRICT 宣言がないとセグメンテーションフォールトを引き起こす可能性があるため、SQL 宣言時には STRICT を指定することが推奨されます。
  • **実装の由来**: 本コードは Perl の Text::DoubleMetaphone モジュールを移植したものであり、アルゴリズムのロジックはオリジナルの C++ 実装に準拠しています。
