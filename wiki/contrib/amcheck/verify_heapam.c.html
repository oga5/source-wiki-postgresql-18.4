contrib/amcheck/verify_heapam.c

概要

verify_heapam.c は、PostgreSQL のヒープリレーション（テーブル）の物理的・論理的な整合性を検証するための実装ファイルです。ページ内のタプル構造、HOT（Heap Only Tuple）チェーンの整合性、トランザクション可視性、および TOAST データの健全性をチェックします。

モジュール内での役割

amcheck モジュールの一部として、ヒープテーブルに対する verify_heapam SQL 関数を提供します。他の検証ファイル（verify_nbtree.c 等）がインデックス構造に特化しているのに対し、本ファイルはテーブルのデータページそのものの物理的な破損を検出する役割を担います。

責務

  • **ページ走査**: ReadStream API を利用し、指定された範囲のヒープページを効率的に走査する。
  • **構造検証**: ページ内のラインポインタ、タプルヘッダー、属性オフセットの整合性を検証する。
  • **可視性チェック**: トランザクション ID（XID/MXID）の妥当性と、可視性ルールに基づく HOT チェーンの整合性を検証する。
  • **TOAST 検証**: メインテーブルの TOAST ポインタと、対応する TOAST テーブル内のチャンクの整合性を照合する。
  • **破損報告**: 検出された破損情報を tuplestore に蓄積し、SQL レベルで結果を返す。

重要なシンボル

コンテキスト管理

  • HeapCheckContext: 検証実行中の状態（リレーション情報、キャッシュされた XID 範囲、現在のページ/タプル位置、破損フラグ）を保持する構造体。
  • HeapCheckReadStreamData: ReadStream API 用のプライベートデータ。スキップオプション（all-visible 等）に基づきページを読み飛ばすロジックを制御する。

検証ロジック

  • verify_heapam: SQL から呼び出されるメイン関数。引数の検証、リレーションのオープン、検証ループの実行を行う。
  • check_tuple: タプル単位の検証フロー（ヘッダー、可視性、属性の順）。
  • check_tuple_header: タプルヘッダーの物理的な整合性（オフセット、長さ、ビットマップ）をチェック。
  • check_tuple_visibility: XID/MXID の妥当性と、タプルが現在有効か（あるいは削除済みか）を判定。
  • check_tuple_attribute: 属性のオフセット計算と、TOAST データの存在チェック。
  • check_toasted_attribute: TOAST テーブルをスキャンし、チャンクのシーケンスとサイズを検証。

ユーティリティ

  • report_corruption: 破損情報を tuplestore に記録する。
  • get_xid_status: XID のコミット状態を判定し、範囲外の XID を検出する。

制御フローとデータフロー

1. **初期化**: verify_heapam がリレーションを AccessShareLock でロックし、HeapCheckContext を初期化。 2. **走査**: read_stream_begin_relation を介してページを順次取得。heapcheck_read_stream_next_unskippable が可視性マップを参照して不要なページをスキップ。 3. **ページ内ループ**: ページ内の各ラインポインタを走査し、check_tuple を呼び出す。 4. **チェーン検証**: ページ内のタプル間リンク（HOT チェーン）を predecessor/successor 配列で追跡し、不整合をチェック。 5. **TOAST 検証**: メインテーブルのロックを解放した後、toasted_attributes リストに蓄積されたポインタに基づき、TOAST テーブルをスキャンして検証。 6. **終了**: tuplestore を通じて結果を返し、リソースを解放。

読者向けメモ

  • **メモリ所有権**: HeapCheckContext 内の toasted_attributes リストは、ページごとの検証後に list_free_deep で解放されます。
  • **安全性**: 本モジュールは破損したデータに対してもクラッシュしないよう設計されていますが、TOAST テーブル自体が深刻に破損している場合、インデックススキャン中にクラッシュする可能性があります。
  • **制約**: check_toast オプションを有効にすると、TOAST テーブルへのインデックススキャンが発生するため、パフォーマンスに影響を与えます。
  • **未実装/浅い部分**: 本ファイルはヒープの物理構造を検証しますが、論理的なデータ内容（型ごとの値の妥当性など）までは検証しません。また、check_tuple_visibility は一部の古いバイナリアップグレード用フラグ（HEAP_MOVED_OFF 等）を考慮した複雑なロジックを含んでいます。
