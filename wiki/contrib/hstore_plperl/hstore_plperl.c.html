contrib/hstore_plperl/hstore_plperl.c

概要

PostgreSQLの hstore 型と Perl のハッシュリファレンス間の相互変換機能を提供する拡張モジュールです。PL/Perl 環境下で hstore データを Perl の連想配列として扱い、またその逆を行うためのブリッジとして機能します。

モジュール内での役割

hstore モジュールと plperl モジュールの間に位置し、両者のデータ構造を変換するアダプター層です。PostgreSQL の外部関数インターフェース（FMGR）を介して、PL/Perl から呼び出し可能な変換関数を提供します。

責務

  • **データ変換**: hstore 構造体から Perl の HV (Hash Value) への変換。
  • **データ変換**: Perl のハッシュリファレンスから hstore 構造体への変換。
  • **動的リンク**: hstore モジュールの内部関数をロードし、クロスモジュール呼び出しを可能にする。

重要なシンボル

  • hstore_to_plperl: hstore を Perl のハッシュリファレンスに変換する SQL 関数。
  • plperl_to_hstore: Perl のハッシュリファレンスを hstore に変換する SQL 関数。
  • _PG_init: モジュールロード時に hstore モジュールの関数ポインタを解決する初期化関数。

追加の索引済みシンボル

  • Datum
  • hstoreCheckKeyLen_t
  • hstoreCheckValLen_t

制御フローとデータフロー

1. **初期化**: _PG_init が load_external_function を使用して hstore モジュールの関数ポインタ（hstorePairs 等）を静的変数に格納します。 2. **hstore → Perl**: hstore_to_plperl が hstore のエントリを走査し、Perl の newHV() で作成したハッシュにキーと値をコピーします。 3. **Perl → hstore**: plperl_to_hstore が Perl のハッシュを走査し、Pairs 配列を構築します。その後、hstoreUniquePairs で重複排除を行い、hstorePairs で最終的な hstore 構造体を生成します。

直接依存

  • postgres.h: PostgreSQL の基本型およびマクロ定義。
  • fmgr.h: 関数マネージャインターフェース。
  • hstore/hstore.h: hstore の内部構造体定義と操作用マクロ。
  • plperl.h: Perl インタープリタとの連携用ヘッダー。

関連コード

  • contrib/hstore: hstore 型の定義とコアロジック。
  • src/pl/plperl: PL/Perl 言語ハンドラのコア実装。

読者向けメモ

  • **動的ロード**: 本ファイルは hstore モジュールの関数を直接リンクせず、実行時に load_external_function で解決しています。これはモジュール間の疎結合を維持するための PostgreSQL 拡張の一般的なパターンです。
  • **メモリ管理**: plperl_to_hstore 内で palloc_array を使用しています。PostgreSQL のメモリコンテキスト管理下にあるため、Perl 側のメモリ管理と混同しないよう注意が必要です。
  • **Perl API**: dTHX マクロにより Perl インタープリタのコンテキストを取得しています。Perl API (newHV, hv_store, SvROK 等) の知識がコードの理解に不可欠です。
