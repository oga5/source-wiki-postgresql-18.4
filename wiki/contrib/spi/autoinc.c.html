contrib/spi/autoinc.c

概要

autoinc.c は、PostgreSQL の SPI (Server Programming Interface) を利用して実装されたトリガー関数です。テーブルへの INSERT または UPDATE 時に、指定されたカラムが未設定（または 0）である場合に、シーケンスから取得した値を自動的に割り当てる機能を提供します。

モジュール内での役割

contrib/spi モジュール群の一部として、データベースの自動化処理をトリガー経由で提供します。他のモジュールが参照整合性や日時更新を扱うのに対し、本ファイルは「シーケンス値による自動採番」という特定のユースケースに特化しています。

責務

  • **トリガー検証**: 呼び出し元が適切なトリガーイベント（BEFORE INSERT/UPDATE）であることを確認する。
  • **引数解析**: トリガー定義で指定された「カラム名」と「シーケンス名」のペアを解析する。
  • **値の自動生成**: 対象カラムが空または 0 の場合、nextval() を呼び出して新しい値を生成し、タプルを更新する。

重要なシンボル

  • autoinc: 本ファイルのメインとなるトリガー関数。PG_FUNCTION_INFO_V1 によりエクスポートされる。
  • TriggerData: SPI トリガー関数に渡される構造体。イベント情報やタプルデータへのアクセスに使用する。
  • heap_modify_tuple_by_cols: 既存のタプルをベースに、特定のカラム値を書き換えた新しいタプルを作成するユーティリティ。

制御フローとデータフロー

1. **初期チェック**: CALLED_AS_TRIGGER 等を用いて、関数がトリガーとして正しく呼び出されているか検証する。 2. **引数ループ**: トリガー引数（tgargs）を走査し、カラム名とシーケンス名のペアを順次処理する。 3. **値の判定**: SPI_getbinval で対象カラムの値を確認し、値が設定済みであればスキップする。 4. **シーケンス取得**: nextval を呼び出し、戻り値（int64）を int32 にキャストして取得する。0 が返った場合は再試行するロジックを含む。 5. **タプル更新**: 変更が必要なカラムがある場合、heap_modify_tuple_by_cols でタプルを再構築し、最終的な結果として返す。

直接依存

  • executor/spi.h: SPI インターフェースの利用。
  • commands/trigger.h: トリガー関連のデータ構造とマクロ。
  • commands/sequence.h: nextval 関数の呼び出し。
  • utils/rel.h / access/htup_details.h: リレーションおよびタプル操作。
  • catalog/pg_type.h
  • postgres.h
  • utils/builtins.h

関連コード

  • contrib/spi/refint.c: 参照整合性を扱う SPI トリガーの例。
  • src/backend/executor/spi.c: SPI の基盤実装。
  • src/backend/commands/sequence.c: シーケンス操作のバックエンド実装。

読者向けメモ

  • **型制約**: 本関数は INT4 型のカラムに対してのみ動作します。それ以外の型を指定するとエラーになります。
  • **再帰的呼び出し**: nextval を直接呼び出しているため、シーケンスのキャッシュやロックの挙動は標準のシーケンス操作に準拠します。
  • **エラー処理**: 引数の数が偶数でない場合や、存在しないカラムを指定した場合は elog や ereport で即座に処理を中断します。デバッグ時はトリガー定義の引数を確認してください。
