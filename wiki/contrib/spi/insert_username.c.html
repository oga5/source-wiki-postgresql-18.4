contrib/spi/insert_username.c

概要

insert_username.c は、PostgreSQL のトリガー関数として動作し、行の挿入または更新時に指定されたカラムへ現在のユーザー名を自動的に格納するモジュールです。

モジュール内での役割

contrib/spi モジュール群の一つとして、SPI (Server Programming Interface) を活用したトリガーの実装例を提供します。他のモジュールが複雑な整合性チェックや自動採番を行うのに対し、本ファイルは「現在のユーザー情報をメタデータとして記録する」というシンプルな用途に特化しています。

責務

  • **トリガー検証**: 呼び出し元が正しいトリガーコンテキスト（行レベル、BEFORE トリガー）であることを確認する。
  • **メタデータ注入**: 現在のセッションユーザー名を取得し、指定されたカラムに TEXT 型として値を設定する。
  • **タプル操作**: heap_modify_tuple_by_cols を使用して、既存のタプルを新しいユーザー名で更新した新しいタプルを生成する。

重要なシンボル

  • insert_username: 本ファイルのメイン関数。PG_FUNCTION_INFO_V1 マクロにより PostgreSQL から呼び出し可能な関数として登録されます。
  • TriggerData: トリガー実行時に渡されるコンテキスト情報。ここから対象のテーブルやタプル情報にアクセスします。

制御フローとデータフロー

1. **初期チェック**: CALLED_AS_TRIGGER 等を用いて、関数がトリガーとして正しく起動されたか検証します。 2. **イベント判定**: INSERT または UPDATE イベントであることを確認し、対象となるタプルを取得します。 3. **引数解析**: トリガー定義で指定されたカラム名（args[0]）を特定し、そのカラムが TEXT 型であることを検証します。 4. **値の生成**: GetUserNameFromId で現在のユーザー名を取得し、CStringGetTextDatum で PostgreSQL 内部形式に変換します。 5. **タプル構築**: heap_modify_tuple_by_cols を用いて、対象カラムを新しいユーザー名で書き換えたタプルを構築し、トリガーマネージャへ返却します。

  • executor/spi.h: SPI インターフェースの利用。
  • commands/trigger.h: トリガーデータ構造とマクロの利用。
  • utils/builtins.h: ユーザー名取得や型変換関数の利用。
  • access/htup_details.h: タプル操作関数の利用。
  • catalog/pg_type.h

関連コード

  • contrib/spi/autoinc.c: 同様にトリガーを利用した自動採番の実装例であり、コードの構造やエラーチェックの作法が類似しています。
  • src/backend/executor/spi.c: SPI の基盤実装。より深い理解が必要な場合に参照します。

読者向けメモ

  • 本関数は BEFORE トリガーとして定義されることを前提としています。AFTER トリガーで使用しても意図した動作（タプルの書き換え）は行われません。
  • 指定するカラムは必ず TEXT 型である必要があります。それ以外の型を指定した場合、実行時にエラーとなります。
  • 非常に軽量な実装であるため、トリガーの基本的な書き方を学ぶための入門用コードとして適しています。
