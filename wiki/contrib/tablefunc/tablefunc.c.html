contrib/tablefunc/tablefunc.c

概要

tablefunc.c は、PostgreSQL の拡張機能として、SQL クエリの結果を変換・再構成してセット（集合）を返す関数群を提供します。主に「クロス集計（Crosstab）」と「階層データ探索（Connectby）」という、標準 SQL では記述が困難なデータ操作を C 言語で実装しています。

モジュール内での役割

contrib/tablefunc モジュールの中心的な実装ファイルです。SPI (Server Programming Interface) を利用して内部的に SQL を実行し、その結果を Tuplestore に蓄積して呼び出し元へ返す、PostgreSQL の「セットを返す関数（SRF）」の典型的な実装パターンを示しています。

責務

  • **クロス集計**: 縦持ちのデータを横持ちに変換する crosstab および crosstab_hash の提供。
  • **階層探索**: 親子関係を持つテーブルから階層構造を再帰的に抽出する connectby の提供。
  • **統計関数**: ガウス分布に従う乱数を生成する normal_rand の提供。
  • **型整合性チェック**: SPI から取得したデータと、関数の戻り値型（TupleDesc）の互換性検証。

重要なシンボル

  • normal_rand: Box-Muller 法を用いた正規分布乱数生成関数。
  • crosstab: 順序付けられた入力から行列を作成する関数。
  • crosstab_hash: カテゴリをハッシュテーブルで管理し、欠損値を適切に扱うクロス集計関数。
  • connectby: 再帰的にツリー構造を探索する関数。
  • crosstab_HashEnt: カテゴリ名と属性インデックスをマッピングするハッシュテーブルエントリ構造体。

制御フローとデータフロー

1. **初期化**: SRF_FIRSTCALL_INIT または SPI_connect を用いて、関数呼び出しのコンテキストと SPI 接続を確立します。 2. **データ取得**: SPI_execute を呼び出し、ソースとなる SQL を実行します。 3. **変換処理**:

  • crosstab: SPI 結果をループし、BuildTupleFromCStrings を用いて行を再構築します。
  • connectby: build_tuplestore_recursively を呼び出し、深さ優先探索でツリーを辿りながら Tuplestore に格納します。

    • 4. **出力**: tuplestore_puttuple で構築したタプルを蓄積し、ReturnSetInfo を介して呼び出し元へ返却します。

読者向けメモ

  • 本ファイルは、PostgreSQL の拡張機能開発における「SPI を使った SRF 実装」の非常に良いサンプルです。
  • crosstab は入力データの順序に依存するため、使用時は ORDER BY が必須である点に注意してください。
  • connectby は再帰的なクエリ実行を行うため、深い階層や循環参照がある場合はパフォーマンスやスタックオーバーフローのリスクを考慮する必要があります。
  • xpfree や xpstrdup などの独自マクロは、メモリ管理を簡略化するためのユーティリティです。
