contrib/file_fdw/file_fdw.c

概要

file_fdw.c は、PostgreSQL の外部データラッパー（FDW）インターフェースを実装し、サーバー上のフラットファイル（CSV, Text 等）や外部プログラムの出力をテーブルとして読み込むための機能を提供します。

モジュール内での役割

contrib/file_fdw モジュールの核となる実装ファイルです。PostgreSQL の COPY コマンドのインフラストラクチャを再利用し、外部ソースからのデータストリームを PostgreSQL の実行エンジンが理解できるタプル形式に変換します。

責務

  • **FDW ハンドラの提供**: file_fdw_handler を通じて、プランナやエグゼキュータが呼び出すべきコールバック関数群を登録します。
  • **オプション検証**: file_fdw_validator により、外部テーブル定義時に指定されたオプション（filename, program, format 等）が妥当か、また実行権限が適切かをチェックします。
  • **プランニングとコスト見積もり**: 外部ファイルのサイズや行数を推定し、クエリプラン作成時のコスト計算を行います。
  • **データスキャン実行**: COPY コマンドの内部 API を利用して、外部ソースからデータを読み込み、TupleTableSlot へ格納します。

重要なシンボル

  • file_fdw_handler: FDW のエントリポイント。FdwRoutine 構造体を返します。
  • FileFdwPlanState: プランニング段階で保持される、ファイルパスや推定統計情報を含む構造体。
  • FileFdwExecutionState: 実行段階で保持される、CopyFromState を含む実行コンテキスト。
  • fileIterateForeignScan: 外部ソースから 1 行ずつデータを取得するメインループ。

制御フローとデータフロー

1. **プランニング**: fileGetForeignRelSize が呼び出され、ファイルサイズを元に統計情報を推定します。fileGetForeignPaths で単一のシーケンシャルスキャンパスを作成します。 2. **実行開始**: fileBeginForeignScan が BeginCopyFrom を呼び出し、ファイルまたはプログラムのストリームを開きます。 3. **データ取得**: fileIterateForeignScan が NextCopyFrom を繰り返し呼び出し、外部データを TupleTableSlot にマッピングします。 4. **終了**: fileEndForeignScan が EndCopyFrom を呼び出し、リソースを解放します。

読者向けメモ

  • **セキュリティ**: filename オプションは pg_read_server_files ロール、program オプションは pg_execute_server_program ロールが必要です。これらは file_fdw_validator 内でチェックされます。
  • **COPY 依存**: このファイルは COPY コマンドのロジックに強く依存しています。COPY の仕様変更が直接影響するため、読み解く際は src/backend/commands/copyfrom.c を併せて参照することを推奨します。
  • **エラーハンドリング**: on_error = 'ignore' オプションによるデータ型不一致時のスキップ処理が fileIterateForeignScan に実装されており、読み込み時の堅牢性を確保しています。
