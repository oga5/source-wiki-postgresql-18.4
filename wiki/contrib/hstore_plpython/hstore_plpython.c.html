contrib/hstore_plpython/hstore_plpython.c

概要

本ファイルは、PostgreSQLの hstore 型と Python の辞書型（dict）の間で相互変換を行うためのブリッジ機能を提供します。plpython 拡張と hstore 拡張の両方に依存し、実行時にそれらの関数を動的にロードすることで型変換を実現します。

モジュール内での役割

contrib/hstore_plpython モジュールの唯一のソースファイルであり、PostgreSQL の拡張機能としてロードされる共有ライブラリの本体です。SQL レベルで hstore_to_plpython および plpython_to_hstore 関数を公開し、データ変換のパイプラインを構築します。

責務

  • **動的リンク**: _PG_init を通じて、plpython および hstore モジュール内の関数ポインタを解決し、クロスモジュール呼び出しを可能にする。
  • **型変換 (hstore → Python)**: hstore データを走査し、Python の辞書オブジェクトを構築する。
  • **型変換 (Python → hstore)**: Python のマッピングオブジェクトを走査し、hstore 形式のペア配列を生成して hstore オブジェクトに変換する。

重要なシンボル

  • _PG_init: モジュールロード時に実行され、外部関数のポインタを解決する初期化関数。
  • hstore_to_plpython: hstore 型を Python の dict に変換する SQL 呼び出し可能関数。
  • plpython_to_hstore: Python の dict を hstore 型に変換する SQL 呼び出し可能関数。
  • PLyObject_AsString_p 等の関数ポインタ: 外部モジュール機能を呼び出すためのインターフェース。

制御フローとデータフロー

1. **初期化**: _PG_init が load_external_function を使用して、必要な外部関数（hstore 操作や Python 文字列変換など）のアドレスを取得します。 2. **変換 (hstore → Python)**:

  • hstore の内部構造を HS_COUNT 等で走査。
  • PLyUnicode_FromStringAndSize を用いてキーと値を Python 文字列に変換し、PyDict_SetItem で辞書に格納します。

    • 3. **変換 (Python → hstore)**:

  • PyMapping_Check で入力が辞書であることを確認。
  • PyMapping_Items で要素を取得し、Pairs 配列に変換。
  • hstoreUniquePairs で重複排除を行い、hstorePairs で最終的な HStore 構造体を生成します。

直接依存

  • postgres.h, fmgr.h: PostgreSQL のコア機能および関数管理。
  • hstore/hstore.h: hstore 型の定義と操作マクロ。
  • plpy_typeio.h, plpy_util.h: plpython 側の型変換およびユーティリティ定義。

関連コード

  • contrib/hstore: hstore 型の定義と操作関数群。
  • src/pl/plpython: plpython 言語ハンドラの実装。本ファイルが依存する関数群の定義元。

読者向けメモ

  • **動的ロードの仕組み**: 本ファイルは plpython や hstore の内部関数を直接リンクするのではなく、load_external_function を介して実行時に解決しています。これは、これらのモジュールが独立してロード可能であることを保証するためです。
  • **メモリ管理**: Python オブジェクトの参照カウント（Py_DECREF）と PostgreSQL のメモリコンテキスト（palloc）の混在に注意してください。特に plpython_to_hstore では PG_TRY/PG_FINALLY を使用して、エラー発生時にも Python オブジェクトのメモリリークを防ぐ設計になっています。
