contrib/oid2name/oid2name.c

概要

oid2name.c は、PostgreSQL のデータベースオブジェクト（テーブル、インデックス、シーケンス等）の OID やファイルノード番号と、データベース上の名前を相互にマッピングするためのコマンドラインツールです。ファイルシステム上の物理ファイルと論理的なデータベースオブジェクトを紐付ける際に利用されます。

モジュール内での役割

contrib/oid2name ディレクトリにおける唯一のソースファイルであり、PostgreSQL のフロントエンドライブラリ（libpq）を使用してデータベースに接続し、システムカタログを照会するスタンドアロンのユーティリティとして機能します。

責務

  • コマンドライン引数の解析と、検索対象（OID、テーブル名、ファイルノード）の管理。
  • libpq を用いたデータベース接続の確立と認証処理。
  • システムカタログ（pg_class, pg_database, pg_tablespace 等）に対する動的な SQL クエリの生成と実行。
  • 取得したクエリ結果の整形出力。

重要なシンボル

  • struct options: 実行時のオプションや接続パラメータを保持する構造体。
  • struct eary: 検索対象のリストを管理するための動的配列構造体。
  • sql_conn(): データベース接続を確立し、初期設定を行う関数。
  • sql_exec(): SQL を実行し、結果を整形して標準出力に表示する汎用関数。
  • sql_exec_searchtables(): 指定された条件（OID/名前/ファイルノード）に基づき、特定のオブジェクト情報を検索する関数。

制御フローとデータフロー

1. **初期化**: main 関数が options 構造体を初期化し、get_opts で引数を解析します。 2. **接続**: sql_conn が libpq を介してデータベースへ接続し、必要に応じてパスワード入力を促します。 3. **クエリ生成**: ユーザーの指定に応じて、sql_exec_dumpalltables や sql_exec_searchtables 等が適切な SQL 文を構築します。 4. **実行と表示**: sql_exec がクエリを発行し、結果セットの列幅を計算して表形式で出力します。 5. **終了**: PQfinish で接続を閉じ、プログラムを終了します。

直接依存

  • libpq-fe.h: データベース接続およびクエリ実行用。
  • postgres_fe.h: PostgreSQL フロントエンドアプリケーション用共通定義。
  • getopt_long.h / pg_getopt.h: コマンドライン引数解析用。
  • catalog/pg_class_d.h: システムカタログの定数定義。
  • common/connect.h
  • common/logging.h
  • common/string.h

関連コード

  • common/connect.h, common/logging.h: 接続管理やログ出力の共通ユーティリティ。
  • src/backend/catalog/: システムカタログの構造を理解する上で参照すべきディレクトリ。

読者向けメモ

  • 本ツールはシステムカタログを直接参照するため、実行には適切な権限を持つデータベースユーザーが必要です。
  • eary 構造体は、コマンドラインで指定された複数の検索対象を SQL の IN 句に変換するために使用されています。
  • sql_exec 内で列幅を動的に計算するロジックは、出力の可読性を高めるための工夫です。
  • 開発・デバッグ目的のツールであるため、複雑なエラーハンドリングよりも、クエリの柔軟な構築に重点が置かれています。
