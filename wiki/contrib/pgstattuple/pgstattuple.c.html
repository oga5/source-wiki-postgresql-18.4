contrib/pgstattuple/pgstattuple.c

概要

pgstattuple.c は、PostgreSQL のテーブルおよびインデックスの物理的な統計情報（タプル数、デッドタプル数、空き容量など）を算出するためのコアロジックを提供します。

モジュール内での役割

contrib/pgstattuple モジュールにおけるメインの実行エンジンです。ユーザーが SQL 関数を呼び出した際、対象リレーションの種別を判定し、適切なスキャン手法へディスパッチする役割を担います。

責務

  • **統計情報の集計**: テーブルおよびインデックスのページを走査し、ライブ/デッドタプルの数とサイズ、および空き容量を計算する。
  • **アクセス制御**: 実行ユーザーの権限チェック（バージョン依存）および、他セッションの一時テーブルへのアクセス制限。
  • **結果の構築**: 集計した pgstattuple_type 構造体を、SQL で利用可能な複合型（Datum）へ変換する。

重要なシンボル

  • pgstattuple_type: 統計結果を保持する構造体。
  • pgstat_relation: リレーションの種類（テーブル/インデックス）を判定し、適切なスキャン関数へ振り分けるエントリポイント。
  • pgstat_heap: ヒープテーブルをスキャンし、ライブ/デッドタプルと空き容量を算出する。
  • pgstat_index: インデックスを走査するための汎用的なループ処理。

制御フローとデータフロー

1. **エントリ**: pgstattuple() 等の SQL 関数が呼ばれ、pgstat_relation() を呼び出す。 2. **判定**: relkind に基づき、ヒープなら pgstat_heap()、インデックスなら pgstat_index() を実行。 3. **走査**:

  • ヒープの場合: table_beginscan_strat を使用して全ページを走査し、HeapTupleSatisfiesVisibility でタプルの生存状態を判定。
  • インデックスの場合: pagefn（pgstat_btree_page 等）を介して各インデックス構造特有のページ走査を行う。

    • 4. **集計**: 走査中に pgstattuple_type 構造体の各フィールドを更新。 5. **出力**: build_pgstattuple_type() が構造体の値を基に HeapTuple を生成し、Datum として返却。

  • pgstatapprox.c: ヒープの統計を近似計算で高速に行うための実装。

読者向けメモ

  • **権限**: pgstattuple 関数はバージョン 1.5 以降、デフォルトで PUBLIC から実行権限が剥奪されています。古いバージョンとの互換性のため、_v1_5 サフィックス付きの関数と古い関数が共存しています。
  • **ロック**: 走査中は AccessShareLock を保持し、整合性を確保します。
  • **パフォーマンス**: 全ページを物理的に読み込むため、大規模なテーブルに対して実行すると I/O 負荷が高くなります。CHECK_FOR_INTERRUPTS() が各ループに含まれており、クエリキャンセルには対応しています。
