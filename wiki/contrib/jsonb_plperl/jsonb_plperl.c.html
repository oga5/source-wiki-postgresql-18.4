contrib/jsonb_plperl/jsonb_plperl.c

概要

PostgreSQLの jsonb 型と Perl のデータ構造（スカラー、配列、ハッシュ）を相互変換するためのブリッジ機能を提供します。PL/Perl 関数内で JSON データを扱う際の型変換ロジックを実装しています。

モジュール内での役割

contrib/jsonb_plperl モジュールの中心的な変換エンジンです。PostgreSQL の内部表現である Jsonb 構造体と、Perl のインターフェースである SV (Scalar Value) 間のマッピングを担います。

責務

  • **Jsonb → Perl**: Jsonb の各ノードを再帰的に走査し、Perl の配列 (AV) やハッシュ (HV)、スカラーへ変換する。
  • **Perl → Jsonb**: Perl のデータ構造を走査し、JsonbParseState を用いて Jsonb 形式へ構築する。
  • **型整合性の維持**: 数値、文字列、真偽値、NULL の各型を、PostgreSQL と Perl の仕様に合わせて適切に変換する。

重要なシンボル

  • jsonb_to_plperl: SQL から呼び出される、jsonb を Perl オブジェクトへ変換するエントリポイント。
  • plperl_to_jsonb: SQL から呼び出される、Perl オブジェクトを jsonb へ変換するエントリポイント。
  • Jsonb_to_SV: Jsonb 構造体を Perl の SV に変換する再帰関数。
  • SV_to_JsonbValue: Perl の SV を JsonbValue に変換する再帰関数。

追加の索引済みシンボル

  • AV_to_JsonbValue
  • HV_to_JsonbValue
  • JsonbValue_to_SV

制御フローとデータフロー

1. **変換開始**: PG_FUNCTION_ARGS を介してデータを受け取り、dTHX マクロで Perl インタプリタのコンテキストを初期化します。 2. **走査**:

  • Jsonb 側は JsonbIterator を使用してトークン単位で順次処理します。
  • Perl 側は SvTYPE を判定し、AV (配列) なら AV_to_JsonbValue、HV (ハッシュ) なら HV_to_JsonbValue を呼び出す再帰構造をとります。

    • 3. **構築**:

  • Perl への変換時は newAV() や newHV() で動的に構造を生成します。
  • jsonb への変換時は pushJsonbValue を用いて JsonbParseState を更新し、最後に JsonbValueToJsonb で確定させます。

直接依存

  • postgres.h, fmgr.h: PostgreSQL のコア機能および関数呼び出しインターフェース。
  • plperl.h: Perl インタプリタとの連携用ヘッダー。
  • utils/jsonb.h: Jsonb 構造体および操作用 API。
  • math.h: 数値変換時の isinf, isnan チェックに使用。
  • utils/fmgrprotos.h

関連コード

  • src/backend/utils/adt/jsonb.c: Jsonb の基本的な操作や構造定義。
  • src/pl/plperl/: PL/Perl 言語ハンドラの本体。

読者向けメモ

  • **メモリ管理**: Perl のメモリ管理と PostgreSQL の palloc が混在するため、変換時のメモリリークには注意が必要です。
  • **再帰の深さ**: Perl のデータ構造が深くネストしている場合、スタックオーバーフローのリスクがあるため、変換は再帰的に行われます。
  • **制約**: jsonb は NaN や無限大を許容しないため、Perl からの変換時に明示的なエラーチェックを行っています。また、Perl の UV (Unsigned Integer) は文字列経由で numeric 型に変換される点に留意してください。
