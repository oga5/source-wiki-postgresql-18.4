contrib/pgcrypto/mbuf.h

概要

mbuf.h は、pgcrypto モジュール内で使用されるメモリバッファ管理およびストリーム処理のための抽象化レイヤーを定義します。暗号化処理におけるデータの読み書きを効率化し、フィルタリング（変換）処理をパイプライン化するための基盤を提供します。

モジュール内での役割

pgcrypto は、PGP データの暗号化・復号や圧縮など、多段階のデータ変換を必要とします。本ファイルは、メモリ上のデータ保持（MBuf）と、そのデータに対する「プッシュ型（書き込み）」および「プル型（読み込み）」のフィルタ処理のインターフェースを定義し、複雑な変換ロジックをモジュール化する役割を担います。

責務

  • **メモリバッファ管理**: 動的なメモリ確保と、データ読み書き位置の追跡。
  • **フィルタ抽象化**: PushFilter および PullFilter による、データ変換処理の連鎖（チェーン）の定義。
  • **ストリーム操作**: フィルタを通じたデータの読み込み・書き込み・フラッシュ処理の提供。

重要なシンボル

  • MBuf: データの読み書きを行うためのメモリバッファ構造体。
  • PushFilterOps / PullFilterOps: フィルタの動作（初期化、処理、解放）を定義する関数ポインタテーブル。
  • pushf_write / pullf_read: フィルタチェーンを介したデータの入出力関数。
  • GETBYTE: フィルタから1バイトを安全に読み取るためのマクロ。

追加の索引済みシンボル

  • PullFilter
  • PushFilter

制御フローとデータフロー

  • **MBuf**: mbuf_create で生成され、mbuf_append でデータを蓄積、mbuf_grab で読み出しを行います。
  • **フィルタチェーン**:
  • PushFilter は、データが流れてくるたびに push 関数を呼び出し、順次変換して最終的な出力先へ送ります。
  • PullFilter は、要求されたデータを pull 関数を通じて上流から引き出し、変換しながら呼び出し元へ返します。
  • フィルタは init 時に次のフィルタ（またはソース）へのポインタを保持し、再帰的な処理構造を形成します。

直接依存

本ヘッダーファイルは、特定の外部ヘッダーへの依存を持たず、純粋なインターフェース定義に徹しています。実装の詳細（mbuf.c）では postgres.h や px.h がインクルードされます。

関連コード

  • contrib/pgcrypto/mbuf.c: 本ヘッダーで宣言された構造体および関数の実装。
  • contrib/pgcrypto/px.h: pgcrypto の共通ユーティリティ関数（メモリ確保やエラー処理など）が定義されており、実装側で利用されます。

読者向けメモ

  • 本モジュールは、ストリーム処理を多用する PGP 関連の実装において中心的な役割を果たします。
  • PushFilter と PullFilter のどちらを使うかは、データの処理方向（書き込み主導か、読み込み主導か）に依存します。
  • GETBYTE マクロは、エラーハンドリングを簡略化するために頻用されるため、読み込み処理を追う際には注意してください。
