contrib/pgcrypto/pgp-encrypt.c

概要

OpenPGP 形式のデータ暗号化処理を実装するソースファイルです。pgcrypto モジュールにおいて、平文を暗号化された PGP パケットストリームへ変換するパイプラインを構築します。

モジュール内での役割

pgcrypto の暗号化サブシステムにおける「出力フィルタ」の集合体です。mbuf.h で定義されるバッファ操作と、pgp.h で定義される PGP 構造を組み合わせ、暗号化、圧縮、MAC（MDC）計算、パケットヘッダ付与を連鎖的に実行します。

責務

  • **暗号化パイプラインの構築**: 圧縮、MAC 計算、CFB 暗号化、パケットヘッダ付与を行うフィルタチェーンの生成。
  • **PGP パケットの生成**: render_newlen 等による可変長ヘッダの作成と、各種 PGP パケット（セッション鍵、リテラルデータ、圧縮データ）の書き込み。
  • **セッション鍵管理**: S2K（String-to-Key）アルゴリズムを用いた鍵導出と、公開鍵/共通鍵によるセッション鍵の保護。

重要なシンボル

  • pgp_encrypt: 本ファイルのメインエントリポイント。暗号化処理全体を統括する。
  • encrypt_filter: CFB モードでの暗号化を行う PushFilter 実装。
  • mdc_filter: SHA-1 を用いた MDC（Modification Detection Code）計算フィルタ。
  • pkt_stream_filter: ストリーム形式の PGP パケットヘッダを管理するフィルタ。
  • PGP_Context: 暗号化設定（アルゴリズム、鍵、モード）を保持する構造体。

制御フローとデータフロー

1. **初期化**: pgp_encrypt が呼び出されると、まず init_s2k_key と init_sess_key で暗号化鍵を準備します。 2. **フィルタチェーン構築**: pushf_create を用いて、データが流れるパイプラインを構築します。

  • 順序: [データ] -> [CRLF変換] -> [リテラルパケット] -> [圧縮] -> [MDC計算] -> [CFB暗号化] -> [出力]

    • 3. **データ処理**: mbuf_grab で取得した平文をチェーンの先頭に流し込み、各フィルタが順次処理（暗号化・圧縮等）を行いながら最終的な MBuf へ書き出します。 4. **終了処理**: pushf_flush により全フィルタのバッファをフラッシュし、pushf_free_all でリソースを解放します。

直接依存

  • postgres.h: PostgreSQL の基本型定義とメモリ管理（palloc 等）。
  • time.h: リテラルパケットのタイムスタンプ生成に使用。
  • mbuf.h: メモリバッファの抽象化。
  • pgp.h: PGP 形式の定義とコンテキスト構造体。
  • px.h: 共通暗号化プリミティブ（乱数生成、ハッシュ関数）。

関連コード

  • pgp-decrypt.c: 対となる復号処理の実装。
  • pgp-cfb.c: CFB モードの暗号化アルゴリズム詳細。
  • pgp-s2k.c: パスフレーズから鍵を生成する S2K アルゴリズム。
  • pgp.c: PGP パケットの共通定義とユーティリティ。

読者向けメモ

  • 本ファイルは「Push Filter」パターンを採用しています。データは pushf_write を通じてチェーンを通過し、各フィルタが自身の役割（暗号化や圧縮）を適用します。
  • MDC（Modification Detection Code）は、暗号化データの改ざん検知に不可欠です。ctx->disable_mdc が設定されていない限り、デフォルトで有効になるよう設計されています。
  • render_newlen は PGP 仕様（RFC 4880）に基づく可変長長さをエンコードする重要なヘルパー関数です。パケット形式を理解する際に最初に参照すべき箇所です。
