contrib/pgcrypto/crypt-md5.c

概要

FreeBSD 由来の MD5 ベースのパスワードハッシュアルゴリズム（いわゆる crypt(3) の MD5 版）を実装したファイルです。主にレガシーなパスワード形式との互換性維持のために使用されます。

モジュール内での役割

contrib/pgcrypto におけるパスワードハッシュ生成機能の一部です。px_crypt_md5 関数を提供し、px-crypt.c などの上位層から呼び出されることで、特定のハッシュアルゴリズムを抽象化されたインターフェース経由で提供します。

責務

  • MD5 を用いた UNIX 形式のパスワードハッシュ生成。
  • 入力されたパスワードとソルトから、$1$ で始まる標準的な MD5 ハッシュ文字列を構築する。
  • セキュリティ上の理由から、計算過程で使用した一時的なバッファを適切にゼロクリアする。

重要なシンボル

  • px_crypt_md5: このファイルのメイン関数。パスワードとソルトを受け取り、ハッシュ化された文字列を返します。
  • _crypt_to64: ハッシュ値を 64 進数（Base64 類似の独自エンコーディング）に変換する内部ヘルパー関数。
  • _crypt_a64: _crypt_to64 で使用される変換用文字テーブル。

制御フローとデータフロー

1. **初期化**: 入力されたソルトからマジック文字列（$1$）を解析し、有効なソルト部分を抽出します。 2. **コンテキスト準備**: px_find_digest を使用して MD5 用のダイジェストコンテキストを 2 つ確保します。 3. **ハッシュ計算**: パスワード、ソルト、マジック文字列を組み合わせ、複雑な反復処理（1000 回のループ）を経てハッシュ値を計算します。このループはブルートフォース攻撃に対する計算コストを意図的に高める役割を果たします。 4. **エンコーディング**: 計算されたバイナリハッシュ値を _crypt_to64 を用いて文字列形式に変換し、結果バッファに格納します。 5. **クリーンアップ**: px_memset を使用して機密情報を含む一時バッファを消去し、コンテキストを解放します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • px-crypt.h: px_crypt_md5 の宣言および関連する暗号化インターフェース。
  • px.h: ダイジェスト操作（px_find_digest, px_md_update 等）の抽象化レイヤー。

関連コード

  • contrib/pgcrypto/px-crypt.c: 本関数を呼び出すためのディスパッチャや、他のハッシュアルゴリズム（Blowfish, SHA 等）との統合点。
  • contrib/pgcrypto/crypt-gensalt.c: ソルト生成に関連するユーティリティ。

読者向けメモ

  • 本実装は FreeBSD のソースコードをベースとしており、現代的なハッシュアルゴリズム（bcrypt や argon2 など）と比較すると強度は低いです。新規の用途には推奨されません。
  • px_md_free を呼び出す前に必ず px_memset で一時バッファをクリアする設計になっており、メモリ上の機密情報漏洩を防ぐための配慮がなされています。
  • 1000 回のループ処理は、当時のハードウェア性能に基づいた計算負荷を意図したものであり、現代の環境では非常に高速に動作します。
