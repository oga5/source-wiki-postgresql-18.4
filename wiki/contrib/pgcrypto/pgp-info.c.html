contrib/pgcrypto/pgp-info.c

概要

PGP データパケットを解析し、暗号化に使用された鍵の識別子（Key ID）を抽出するためのユーティリティファイルです。

モジュール内での役割

contrib/pgcrypto サブシステムにおいて、暗号化されたデータや公開鍵パケットから、どの鍵で暗号化されているか、あるいはどの鍵が関連しているかを特定するメタデータ抽出層を担います。

責務

  • PGP パケットストリームの走査とパケットタグの識別。
  • 公開鍵（Public Key）および公開鍵暗号化セッション鍵（Public-Key Encrypted Session Key）パケットからの Key ID 抽出。
  • 抽出したバイナリ形式の Key ID を、可読な 16 進数文字列へ変換。
  • 複数の鍵が含まれる場合の整合性チェックとエラーハンドリング。

重要なシンボル

  • pgp_get_keyid: 公開インターフェース。MBuf から PGP データを読み込み、Key ID を文字列として dst に書き出す。
  • read_pubkey_keyid: 公開鍵パケットから Key ID を抽出する内部ヘルパー。
  • read_pubenc_keyid: 公開鍵暗号化セッション鍵パケットから Key ID を抽出する内部ヘルパー。
  • print_key: 8 バイトのバイナリ Key ID を 16 進数文字列に変換するフォーマッタ。

制御フローとデータフロー

1. **初期化**: pullf_create_mbuf_reader を用いて MBuf からストリームを生成。 2. **パケット走査**: pgp_parse_pkt_hdr でパケットタグを順次読み取り、switch 文で処理を分岐。 3. **抽出**: 鍵関連のパケット（PGP_PKT_PUBLIC_SUBKEY 等）に遭遇すると、専用の read_*_keyid 関数を呼び出し、keyid_buf に値を格納。 4. **終了判定**: 暗号化データパケット（SYMENCRYPTED_DATA）に到達するか、ストリーム終了までループ。 5. **結果生成**: 抽出された Key ID があれば print_key で文字列化し、なければ SYMKEY や ANYKEY などの定型文字列を返す。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • mbuf.h: メモリバッファ操作用。
  • pgp.h: PGP パケット構造および定数定義。
  • px.h: pgcrypto の共通エラーコードおよびユーティリティ。

関連コード

  • pgp.c: パケット解析の基盤となる関数群（pgp_parse_pkt_hdr 等）。
  • pgp-pubkey.c: 公開鍵の構造体定義と操作。
  • mbuf.c: ストリーム読み込みの低レイヤー処理。

読者向けメモ

  • 本ファイルは「鍵の特定」に特化しており、暗号化や復号そのものは行いません。
  • dst 引数には、Key ID 文字列（16 文字）と終端文字を格納できる十分なバッファ（17 バイト以上）が必要です。
  • any_key は、特定の鍵を指定しない（ワイルドカード的な）Key ID を表す定数として扱われます。
  • 複数の鍵が混在するパケットストリームに対しては、厳格な整合性チェック（PXE_PGP_MULTIPLE_KEYS）が行われる点に注意してください。
