contrib/pgcrypto/px-hmac.c

概要

px-hmac.c は、pgcrypto モジュール内で HMAC (Hash-based Message Authentication Code) アルゴリズムを実装するための汎用フレームワークを提供します。特定のハッシュ関数に依存せず、px_find_digest を介して提供される任意のメッセージダイジェストアルゴリズムを HMAC の基盤として利用可能にします。

モジュール内での役割

pgcrypto の暗号化抽象化レイヤー（px）の一部として機能します。上位層がハッシュアルゴリズムの種類を意識することなく、一貫したインターフェースで HMAC 計算を行えるようにするアダプターの役割を担います。

責務

  • **HMAC コンテキストの構築**: 指定されたハッシュアルゴリズムに基づき、PX_HMAC 構造体を初期化する。
  • **キーの処理**: HMAC 仕様に従い、入力キーをブロックサイズに合わせてパディングまたはハッシュ化し、ipad（Inner Pad）および opad（Outer Pad）を生成する。
  • **計算の抽象化**: init, update, finish, reset といった標準的な HMAC 操作を、内部のダイジェスト関数へ委譲する。
  • **メモリ管理**: HMAC 計算に使用する一時バッファやコンテキストの安全な確保と解放（ゼロクリアを含む）。

重要なシンボル

  • px_find_hmac: HMAC コンテキストを生成する公開関数。利用するハッシュアルゴリズム名を指定する。
  • PX_HMAC: HMAC 操作に必要な状態（ダイジェストコンテキスト、ipad、opad）を保持する構造体。
  • HMAC_IPAD / HMAC_OPAD: RFC 2104 で定義された XOR 用の定数。

制御フローとデータフロー

1. **初期化 (px_find_hmac)**: 指定されたハッシュアルゴリズムのダイジェストコンテキストを取得し、PX_HMAC 構造体を割り当てて関数ポインタをセットする。 2. **キー設定 (hmac_init)**: キーがブロックサイズより長い場合はハッシュ化し、それ以外はそのまま使用。キーと IPAD/OPAD を XOR して内部バッファに保持し、最初の ipad をダイジェストコンテキストに適用する。 3. **更新 (hmac_update)**: 内部のダイジェスト関数へデータを渡す。 4. **終了 (hmac_finish)**: 内部ダイジェストを一度完了させ、その結果に対して opad を用いた二段目のハッシュ計算を行い、最終的な HMAC 値を生成する。

直接依存

  • postgres.h: PostgreSQL の基本型定義およびメモリ管理関数（palloc, pfree 等）。
  • px.h: pgcrypto の抽象化レイヤー定義。PX_MD や PX_HMAC 構造体、ダイジェスト操作関数群。

関連コード

  • px.c: px_find_digest など、このファイルが依存する基盤機能の実装。
  • openssl.c: px レイヤーが利用する具体的なハッシュアルゴリズムの実装（OpenSSL 等）。

読者向けメモ

  • 本実装は RFC 2104 に準拠しています。
  • hmac_finish や hmac_free 内で px_memset を使用して機密情報（キーや中間ハッシュ）をゼロクリアしており、メモリ上の情報漏洩を防ぐ設計になっています。
  • px_find_hmac が返す PX_HMAC 構造体は、呼び出し側が責任を持って free メソッドを呼び出し、メモリを解放する必要があります。
