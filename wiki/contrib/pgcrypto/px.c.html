contrib/pgcrypto/px.c

概要

px.c は、pgcrypto モジュール全体で共通して利用されるユーティリティ関数および基盤機能を提供します。エラーハンドリング、デバッグ出力、および暗号化アルゴリズムとパディングを組み合わせた「コンボ」インターフェースの管理を担います。

モジュール内での役割

pgcrypto の中心的な抽象化レイヤーです。個別の暗号アルゴリズム（Blowfish, DES, AES等）の実装と、それらを呼び出す上位層（SQL関数など）の間に位置し、一貫したエラーコード体系と初期化インターフェースを提供します。

責務

  • **エラー管理**: PXE_* エラーコードと人間が読めるメッセージの対応付け、および PostgreSQL の ereport への変換。
  • **デバッグ支援**: 柔軟なデバッグハンドラの登録とログ出力機能。
  • **コンボ管理**: 暗号アルゴリズムとパディング方式を組み合わせた PX_Combo オブジェクトの生成とライフサイクル管理。
  • **文字列処理**: アルゴリズム名のエイリアス解決およびパース。

重要なシンボル

  • px_THROW_ERROR(int): pgcrypto 内部のエラーを PostgreSQL の ERROR に変換してスローする。
  • px_strerror(int): エラーコードに対応する説明文字列を返す。
  • px_find_combo(const char *, PX_Combo **): 指定された名前（例: aes/pad:pkcs）から暗号化コンボオブジェクトを生成する。
  • px_debug(const char *, ...): 登録されたデバッグハンドラ経由でログを出力する。

制御フローとデータフロー

1. **コンボ生成**: px_find_combo が文字列をパースし、parse_cipher_name で暗号名とパディング設定を分離します。 2. **初期化**: combo_init が呼び出されると、指定された鍵と IV を適切なサイズに調整し、内部の PX_Cipher を初期化します。 3. **実行**: combo_encrypt / combo_decrypt が呼び出されると、内部の PX_Cipher 実装へ処理を委譲します。 4. **エラー処理**: 処理中に発生したエラーは px_THROW_ERROR を通じて PostgreSQL のトランザクション制御に統合されます。

直接依存

  • postgres.h: PostgreSQL のコア機能（palloc, ereport 等）へのアクセス。
  • px.h: pgcrypto の共通型定義、エラーコード、およびインターフェース定義。

関連コード

  • px.h: 本ファイルのインターフェース定義。
  • pgcrypto.c: SQL 関数から本モジュールの関数を呼び出すエントリーポイント。
  • px-crypt.c: パスワードハッシュ化に関連するユーティリティ。

読者向けメモ

  • px_memset はコンパイラの最適化によって削除されないように意図的にラップされています。機密情報（鍵など）をメモリから消去する際に重要です。
  • PX_Combo は暗号化とパディングを抽象化するための構造体であり、新しい暗号アルゴリズムを追加する際は、このインターフェースに適合させる必要があります。
  • parse_cipher_name は破壊的な文字列操作を行うため、呼び出し元で pstrdup されたバッファを渡す必要があります。
