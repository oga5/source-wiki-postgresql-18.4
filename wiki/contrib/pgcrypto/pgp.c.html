contrib/pgcrypto/pgp.c

概要

pgp.c は、pgcrypto モジュールにおける OpenPGP 実装の基盤となるユーティリティ関数群を提供します。暗号アルゴリズム、ハッシュ関数、および圧縮アルゴリズムのメタデータ管理と、PGP 処理のコンテキスト管理を担います。

モジュール内での役割

pgcrypto サブシステムにおいて、PGP 関連の暗号操作（暗号化・復号）を行うための「設定ハブ」として機能します。具体的な暗号処理を行う pgp-encrypt.c や pgp-decrypt.c が、このファイルで定義されたコンテキスト構造体やアルゴリズム解決関数を利用します。

責務

  • **メタデータ管理**: 暗号アルゴリズムやダイジェストアルゴリズムの名称と内部コードの相互変換。
  • **コンテキスト管理**: PGP_Context の初期化、設定、および解放。
  • **設定の抽象化**: ユーザーが指定したアルゴリズム名から、内部で使用する PX_Cipher や PX_MD をロードするためのインターフェース提供。
  • **デフォルト値の保持**: 暗号化処理におけるデフォルトのアルゴリズムや S2K（String-to-Key）設定の管理。

重要なシンボル

  • PGP_Context: PGP 処理の状態を保持する主要な構造体。
  • pgp_init() / pgp_free(): コンテキストのライフサイクル管理。
  • pgp_load_cipher() / pgp_load_digest(): アルゴリズムコードから実際の暗号・ハッシュ実装をロードするファクトリ関数。
  • pgp_get_cipher_code() / pgp_get_digest_code(): 文字列名から内部コードへの変換。

制御フローとデータフロー

1. **初期化**: pgp_init() が呼ばれると、静的なデフォルト値で PGP_Context が初期化されます。 2. **設定**: pgp_set_* 系関数により、ユーザー指定のアルゴリズムやモードがコンテキストに適用されます。 3. **解決**: 暗号化・復号の実行時に、pgp_load_cipher() 等が呼ばれ、コンテキスト内のコードに基づき px レイヤーから適切な暗号実装が取得されます。 4. **終了**: pgp_free() により、メモリの安全な解放と機密情報のクリアが行われます。

直接依存

  • postgres.h: PostgreSQL の基本型およびメモリ管理機能。
  • pgp.h: PGP_Context 構造体定義および列挙型。
  • px.h: px_find_cipher や px_find_digest を含む、低レベル暗号抽象化レイヤー。

関連コード

  • pgp-encrypt.c / pgp-decrypt.c: 本ファイルで管理されるコンテキストを消費するメインロジック。
  • pgp-s2k.c: S2K（パスフレーズから鍵を生成する処理）の実装。
  • px.c: px_find_cipher 等の低レベル暗号ライブラリへのブリッジ。

読者向けメモ

  • 本ファイルはアルゴリズムの「カタログ」と「設定器」です。実際の暗号化アルゴリズムの数学的実装はここにはなく、px レイヤー（OpenSSL 等）に委譲されています。
  • cipher_list や digest_list を拡張することで、サポートするアルゴリズムを増やすことが可能です。
  • PGP_Context を操作する際は、必ず pgp_init で生成されたものを使用してください。
