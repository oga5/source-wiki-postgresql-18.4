contrib/pgcrypto/pgp-cfb.c

概要

OpenPGP 仕様で定義される CFB（Cipher Feedback）モードの暗号化および復号処理を実装するファイルです。標準的な CFB モードに加え、古い PGP バージョンで採用されていた特殊な再同期（resync）処理をサポートしています。

モジュール内での役割

contrib/pgcrypto における PGP 暗号化サブシステムの一部です。上位の暗号化/復号処理層から呼び出され、ブロック暗号の出力をストリームデータに変換する役割を担います。

責務

  • **CFB モードの抽象化**: 暗号化と復号の差異を隠蔽し、共通のインターフェースを提供します。
  • **PGP 特有の再同期処理**: 古い PGP 形式で必要となる、特定のブロック境界での複雑な再同期ロジックをカプセル化します。
  • **状態管理**: 暗号化コンテキスト（PGP_CFB）を保持し、ブロック単位の XOR 演算とフィードバック処理を管理します。

重要なシンボル

  • PGP_CFB: 暗号化コンテキストを保持する構造体。暗号器、現在のブロック位置、再同期フラグ、内部バッファを管理します。
  • pgp_cfb_create / pgp_cfb_free: コンテキストのライフサイクル管理を行います。
  • pgp_cfb_encrypt / pgp_cfb_decrypt: 公開インターフェース。内部で適切な mix 関数を選択し、cfb_process を呼び出します。

制御フローとデータフロー

1. **初期化**: pgp_cfb_create で暗号器を初期化し、IV（初期化ベクトル）をセットします。 2. **処理ループ**: cfb_process がメインループとして機能します。

  • ブロック暗号を使用して ctx->fr（フィードバックレジスタ）を暗号化し、ctx->fre を生成します。
  • mix_data 関数（normal または resync）を介して、入力データと ctx->fre の XOR を取り、出力を作成します。

    • 3. **再同期**: resync モードの場合、特定のブロック（ブロック番号2）で特殊なバイト操作を行い、レジスタを更新します。 4. **フィードバック**: 処理したデータがブロックサイズに達するたびに、ctx->encbuf を ctx->fr にコピーし、次のブロックの入力とします。

直接依存

  • postgres.h: PostgreSQL の基本型およびメモリ管理（palloc 等）を利用。
  • pgp.h: PGP 関連の定数やデータ構造定義。
  • px.h: PX_Cipher などの汎用暗号化インターフェース。

関連コード

  • pgp-encrypt.c / pgp-decrypt.c: 本ファイルを呼び出し、PGP パケットの暗号化・復号フローを制御する上位層。
  • px.c: PX_Cipher の実装基盤。

読者向けメモ

  • 本ファイル内の mix_encrypt_resync および mix_decrypt_resync は、PGP の古い仕様に起因する複雑なロジックを隔離しています。これらはコードの可読性を下げる要因ですが、仕様上の必要悪としてここに集約されています。
  • resync フラグが有効な場合、通常の CFB とは異なるレジスタ更新が行われるため、デバッグ時には ctx->resync の値に注意してください。
