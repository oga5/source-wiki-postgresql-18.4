contrib/pgcrypto/pgcrypto.c

概要

pgcrypto.c は、PostgreSQL の SQL インターフェースと pgcrypto モジュールの内部暗号化ライブラリ（px レイヤー）を接続するブリッジです。ユーザーが SQL から呼び出す暗号化・ハッシュ・乱数生成関数の実装を提供します。

モジュール内での役割

pgcrypto モジュールのフロントエンドとして機能します。内部の抽象化レイヤー（px.h で定義される PX_MD, PX_HMAC, PX_Combo など）を呼び出し、PostgreSQL のデータ型（bytea, text）と暗号化処理の間の変換を担います。

責務

  • **SQL 関数定義**: digest, hmac, crypt, encrypt, decrypt などの SQL インターフェースの提供。
  • **プロバイダーの解決**: find_provider を通じて、指定されたアルゴリズム名に対応する暗号化プロバイダーを動的に検索・初期化。
  • **GUC 管理**: pgcrypto.builtin_crypto_enabled 設定の管理。
  • **メモリ管理**: PostgreSQL のメモリコンテキスト内でのバッファ確保と解放。

重要なシンボル

  • _PG_init: モジュールロード時に実行され、GUC 変数を登録。
  • find_provider: アルゴリズム名から適切な暗号化/ハッシュ関数を検索する内部ヘルパー。
  • pg_digest, pg_hmac: ハッシュおよび HMAC 計算のメインエントリポイント。
  • pg_encrypt, pg_decrypt: 対称鍵暗号化のメインエントリポイント。
  • builtin_crypto_enabled: 組み込み暗号化機能の有効状態を保持するグローバル変数。

制御フローとデータフロー

1. **初期化**: _PG_init が GUC を登録し、設定を読み込みます。 2. **関数呼び出し**: SQL 関数（例: pg_encrypt）が呼ばれると、引数からアルゴリズム名を取得します。 3. **プロバイダー検索**: find_provider が px レイヤーの検索関数を呼び出し、暗号化コンテキスト（PX_Combo 等）を取得します。 4. **処理実行**: px_combo_init で鍵と IV を設定し、px_combo_encrypt/decrypt でデータを処理します。 5. **結果返却**: 処理結果を bytea 型に格納し、PostgreSQL のメモリ管理下で返却します。

直接依存

  • postgres.h, utils/builtins.h, utils/guc.h: PostgreSQL コア機能。
  • px.h, px-crypt.h: pgcrypto 内部の暗号化抽象化レイヤー。
  • pgcrypto.h: モジュール共通ヘッダー。
  • ctype.h
  • parser/scansup.h
  • varatt.h

関連コード

  • px.c: find_provider が呼び出す px_find_* 関数の実装元。
  • openssl.c: px レイヤーが OpenSSL を使用する場合のバックエンド実装。
  • pgp-pgsql.c: PGP 関連の SQL 関数実装。

読者向けメモ

  • 本ファイルはビジネスロジックそのものではなく、SQL と内部ライブラリの「接着剤」です。暗号化アルゴリズムの詳細を知りたい場合は px.c や openssl.c を参照してください。
  • find_provider は、ユーザーが指定したアルゴリズム名を小文字化して検索するため、大文字小文字を区別しない柔軟な指定が可能です。
  • PG_FREE_IF_COPY を適切に使用し、PostgreSQL のメモリ管理ルールに従っている点に注意してください。
