contrib/bloom/blvalidate.c

概要

blvalidate.c は、PostgreSQL の Bloom フィルタインデックス用演算子クラス（opclass）の定義が妥当であるかを検証するバリデータを提供します。

モジュール内での役割

contrib/bloom モジュールにおいて、ユーザーが定義した演算子クラスがインデックスの要件を満たしているかを確認するゲートキーパーの役割を果たします。CREATE OPERATOR CLASS 実行時に呼び出され、不適切な定義を未然に防ぎます。

責務

  • **サポート関数の検証**: 必須となるハッシュ関数等のシグネチャが正しいかを確認します。
  • **演算子の検証**: 演算子の戦略番号が Bloom インデックスの仕様範囲内か、また戻り値が boolean 型であるかを確認します。
  • **整合性チェック**: 演算子ファミリー内のグループ構成を解析し、必須のサポート関数が欠落していないかを検証します。

重要なシンボル

  • blvalidate(Oid opclassoid): このファイルの唯一の公開関数。システムカタログから演算子クラス情報を取得し、検証ロジックの起点となります。

追加の索引済みシンボル

  • blvalidate

制御フローとデータフロー

1. **カタログ取得**: SearchSysCache1 を使用して pg_opclass から対象の演算子クラス情報を取得します。 2. **ファミリー解析**: SearchSysCacheList1 で関連する演算子（AMOPSTRATEGY）とサポート関数（AMPROCNUM）のリストを取得します。 3. **個別検証**:

  • サポート関数に対し、check_amproc_signature 等を用いて型と引数の整合性を確認します。
  • 演算子に対し、戦略番号の範囲チェックおよび ORDER BY 演算子の禁止（Bloom は非対応）を適用します。

    • 4. **グループ検証**: identify_opfamily_groups を用いて、演算子と関数の組み合わせが完全であるかを確認し、必須関数（BLOOM_HASH_PROC 等）の欠落を報告します。

読者向けメモ

  • 本ファイルは CREATE OPERATOR CLASS の実行時にのみ動作します。
  • Bloom インデックスは ORDER BY 演算子をサポートしていないため、検証ロジック内で明示的に拒否している点に注意してください。
  • BLOOM_OPTIONS_PROC はオプション扱いであり、欠落していてもエラーにはなりません。
