contrib/unaccent/generate_unaccent_rules.py

概要

PostgreSQL の unaccent 拡張機能で使用される変換ルールファイル（unaccent.rules）を生成するための Python スクリプトです。Unicode 標準のデータセットを解析し、アクセント記号の除去や合字（リガチャ）の展開ルールを自動生成します。

モジュール内での役割

contrib/unaccent モジュールにおいて、実行時に使用される変換テーブルの「ソース」を生成するビルドツールです。このスクリプトの出力結果が、最終的に unaccent.c が読み込むルールセットの基盤となります。

責務

  • **UnicodeData.txt の解析**: 文字の分解情報（Decomposition）を読み取り、アクセント付き文字を基底文字へ変換するルールを抽出します。
  • **CLDR データの統合**: Unicode CLDR の Latin-ASCII.xml を解析し、より広範な文字変換ルールを補完します。
  • **ルールセットの正規化**: 重複の排除、特殊ケースのハンドリング、および PostgreSQL が読み込み可能な形式への整形を行います。

重要なシンボル

  • Codepoint: Unicode 文字の属性（ID、カテゴリ、分解情報）を保持するデータクラス。
  • is_letter_with_marks(): 文字がアクセント記号を含むかを判定し、再帰的に基底文字を特定します。
  • parse_cldr_latin_ascii_transliterator(): XML 形式の CLDR 変換ルールを解析し、Python のセット形式に変換します。
  • COMBINING_MARK_RANGES: 除去対象とする Unicode の結合記号範囲を定義した定数。

制御フローとデータフロー

1. **初期化**: 引数から入力ファイル（UnicodeData.txt, Latin-ASCII.xml）のパスを受け取ります。 2. **データロード**: UnicodeData.txt を解析し、全文字の分解情報を table 辞書に格納します。 3. **ルール抽出**:

  • all リストを走査し、アクセント付き文字や合字を特定して charactersSet に追加します。
  • オプションが有効な場合、CLDR データおよび special_cases() で定義された手動ルールをセットに統合します。

    • 4. **出力**: charactersSet をコードポイント順にソートし、print_record() を通じて標準出力にタブ区切り形式で書き出します。

直接依存

  • argparse: コマンドライン引数の解析。
  • xml.etree.ElementTree: CLDR XML ファイルのパース。
  • re: Unicode 分解情報の正規表現処理。
  • codecs: 出力エンコーディングの制御。

関連コード

  • unaccent.c: 本スクリプトが生成したルールファイルを読み込み、実際に文字列の変換を行う PostgreSQL 拡張の本体。

読者向けメモ

  • **データソースの鮮度**: このスクリプトは Unicode 標準の更新に依存します。最新の Unicode バージョンに対応させるには、適切な UnicodeData.txt を指定する必要があります。
  • **保守のヒント**: PLAIN_LETTER_RANGES は現在ラテン文字とギリシャ文字に限定されています。他の言語のサポートを追加する場合は、この範囲を拡張する必要があります。
  • **実行時の注意**: Latin-ASCII.xml の解析は XML 構造に強く依存しているため、CLDR のバージョンアップにより XML スキーマが変更された場合は、パーサーの修正が必要になる可能性があります。
