contrib/jsonb_plpython/jsonb_plpython.c

概要

本ファイルは、PostgreSQLの jsonb 型と Python のオブジェクト間での相互変換機能を提供する拡張モジュールです。plpython 拡張と連携し、データベース内の JSONB データを Python の辞書やリスト、数値型として操作可能にします。

モジュール内での役割

plpython 拡張のサブモジュールとして機能します。PostgreSQL の型システムと Python の型システムを橋渡しする変換ロジックをカプセル化しており、plpython 本体がロードされた後に動的に関数ポインタを取得して連携します。

責務

  • **JSONB から Python への変換**: JsonbContainer や JsonbValue を走査し、対応する Python オブジェクト（dict, list, Decimal, str, bool, None）を生成する。
  • **Python から JSONB への変換**: Python のシーケンスやマッピングを再帰的に解析し、JsonbParseState を用いて JsonbValue 構造体を構築する。
  • **型変換の整合性維持**: 数値の NaN/Infinity 判定や、Python の Decimal 型への変換など、JSON 仕様と PostgreSQL の制約を遵守した変換を行う。

重要なシンボル

  • plpython_to_jsonb: SQL から呼び出される、Python オブジェクトを JSONB に変換する関数。
  • jsonb_to_plpython: SQL から呼び出される、JSONB を Python オブジェクトに変換する関数。
  • PLyObject_FromJsonbContainer: JSONB のコンテナ（オブジェクト/配列）を再帰的に Python オブジェクトへ変換する中心的な関数。
  • PLyObject_ToJsonbValue: Python オブジェクトを JSONB の要素に変換する再帰的な変換器。

制御フローとデータフロー

1. **初期化**: _PG_init が実行され、plpython 本体から PLyObject_AsString などの関数ポインタをロードします。 2. **変換フロー (JSONB → Python)**:

  • jsonb_to_plpython が JsonbIterator を初期化。
  • PLyObject_FromJsonbContainer がトークンを順次読み取り、型に応じて PyDict_New や PyList_New を呼び出し、再帰的に構造を構築します。

    • 3. **変換フロー (Python → JSONB)**:

  • plpython_to_jsonb が JsonbParseState を初期化。
  • PLyObject_ToJsonbValue が Python オブジェクトの型を判定し、pushJsonbValue を介して JSONB 構造を構築します。

読者向けメモ

  • **動的リンク**: 本モジュールは plpython がロードされていることを前提としています。_PG_init 内で load_external_function を使用して関数ポインタを解決している点に注意してください。
  • **例外処理**: Python のメモリ管理（Py_DECREF）と PostgreSQL のエラーハンドリング（PG_TRY/PG_CATCH）が混在しています。変換中にエラーが発生した場合、メモリリークを防ぐために PG_CATCH ブロック内で適切に Py_XDECREF を呼び出す設計になっています。
  • **Decimal 変換**: JSONB の数値は Python の Decimal 型に変換されます。cdecimal が利用可能な場合はそれが優先されます。
