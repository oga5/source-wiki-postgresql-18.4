contrib/dict_int/dict_int.c

概要

dict_int.c は、PostgreSQL の全文検索機能（TSearch）において、整数値を正規化またはフィルタリングするための辞書テンプレートを実装したモジュールです。

モジュール内での役割

contrib/dict_int 拡張の一部として、テキスト検索における「整数」の扱いを定義します。入力されたトークンが整数であると見なされる場合、設定に基づいて桁数を制限したり、特定の条件で除外したりする役割を担います。

責務

  • **辞書設定の初期化**: maxlen（最大長）、rejectlong（長すぎる値の拒否）、absval（符号の無視）といったパラメータを解析し、DictInt 構造体に保持する。
  • **トークンの正規化**: 入力文字列を解析し、設定に従って切り詰め（trim）や除外を行う。
  • **TSearch インターフェースへの適合**: dintdict_init および dintdict_lexize を通じて、PostgreSQL の全文検索エンジンが期待する標準的な辞書 API を提供する。

重要なシンボル

  • DictInt: 辞書の設定値を保持する構造体。
  • dintdict_init: 辞書作成時に呼び出され、オプション引数を解析する関数。
  • dintdict_lexize: 実際のトークン処理を行う関数。入力トークンを解析し、結果として TSLexeme 配列を返す。

制御フローとデータフロー

1. **初期化**: dintdict_init が SQL の CREATE TEXT SEARCH DICTIONARY 実行時に呼ばれ、DictInt 構造体をメモリ上に構築します。 2. **字句解析**: dintdict_lexize が全文検索の実行中に呼び出されます。

  • absval が有効な場合、先頭の符号（+ / -）を無視して処理します。
  • maxlen を超える長さのトークンに対し、rejectlong が真ならトークンを破棄（NULL を返す）、偽なら maxlen で切り詰めます。
  • 最終的に処理された文字列を TSLexeme 配列に格納して返却します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • commands/defrem.h: DefElem 構造体など、辞書オプションの解析に必要な定義。
  • tsearch/ts_public.h: TSLexeme など、全文検索辞書の実装に必要なインターフェース。

関連コード

  • src/backend/tsearch/: PostgreSQL 本体の全文検索エンジン。辞書 API の仕様についてはこのディレクトリ内のヘッダーファイルが参考になります。
  • contrib/: 他の全文検索辞書の実装例（dict_xsyn など）が、辞書モジュールの実装パターンとして参考になります。

読者向けメモ

  • 本モジュールは非常にシンプルで、主に「文字列の切り詰め」と「条件付き除外」に特化しています。
  • dintdict_lexize 内で palloc0 を使用して TSLexeme 配列を確保していますが、これは TSearch エンジン側で解放されることを前提としています。
  • 整数値のバリデーション自体は本モジュールではなく、呼び出し側のトークナイザー（ts_token_type など）に依存している点に注意してください。
