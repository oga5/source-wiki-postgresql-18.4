contrib/postgres_fdw/connection.c

概要

connection.c は、postgres_fdw におけるリモート PostgreSQL サーバーへの接続ライフサイクルとトランザクション同期を管理するモジュールです。リモート接続をキャッシュし、ローカルのトランザクション状態（メインおよびサブトランザクション）をリモート側へ透過的に反映させる責務を持ちます。

モジュール内での役割

postgres_fdw サブシステムにおいて、リモート通信の「基盤」を提供します。postgres_fdw.c（プランナ・エグゼキュータ）がクエリ実行を要求する際、本モジュールが接続の確立・維持・破棄を抽象化します。

責務

  • **接続キャッシュ管理**: UserMapping OID をキーとしたハッシュテーブルによる接続の再利用。
  • **トランザクション同期**: ローカルのトランザクション深度（xact_depth）を追跡し、リモート側で START TRANSACTION や SAVEPOINT を発行して同期を維持。
  • **セッション構成**: リモート接続確立時に search_path や timezone などのセッションパラメータを強制設定。
  • **エラー・中断処理**: リモートクエリのキャンセルや、トランザクション異常終了時のクリーンアップ（ABORT/ROLLBACK）。
  • **セキュリティ**: ユーザーマッピングに基づく認証チェックと、非スーパーユーザーによる権限昇格の防止。

重要なシンボル

キャッシュ管理

  • ConnectionHash: 接続情報を保持するグローバルなハッシュテーブル。
  • ConnCacheEntry: 接続状態、トランザクション深度、準備済みステートメントの有無などを保持する構造体。
  • GetConnection(): 必要な接続を取得または確立するメインエントリポイント。

トランザクション・ライフサイクル

  • pgfdw_xact_callback() / pgfdw_subxact_callback(): ローカルのトランザクションイベントをフックし、リモート側のコミット/ロールバックをトリガーするコールバック。
  • begin_remote_xact(): リモート側で必要なトランザクションレベル（REPEATABLE READ 等）を開始する。

接続・通信

  • connect_pg_server(): libpq を使用してリモート接続を確立し、認証とセッション設定を行う。
  • pgfdw_exec_query() / pgfdw_get_result(): リモートクエリ実行と結果取得のラッパー。
  • pgfdw_cancel_query(): 実行中のリモートクエリをキャンセルする。

制御フローとデータフロー

1. **接続取得**: GetConnection() が呼ばれると、ConnectionHash を検索。存在しないか無効な場合、make_new_connection() を経由して connect_pg_server() を呼び出し、libpq で接続を確立。 2. **同期**: begin_remote_xact() がローカルの GetCurrentTransactionNestLevel() と xact_depth を比較し、不足分だけ SAVEPOINT を発行。 3. **終了処理**: トランザクション終了時、pgfdw_xact_callback() がハッシュテーブルをスキャンし、COMMIT または ABORT をリモートへ送信。 4. **無効化**: pgfdw_inval_callback() がシステムカタログの変更を検知し、関連する接続を invalidated フラグでマーク。

読者向けメモ

  • **メモリ管理**: ConnCacheEntry はバックエンドの生存期間中保持されます。接続失敗時のエラーハンドリングには PG_TRY/PG_CATCH が多用されており、リモート接続のクリーンアップが確実に行われるよう設計されています。
  • **トランザクションの制約**: リモート側は常に REPEATABLE READ 以上で動作します。これはスナップショットの一貫性を保つためですが、READ COMMITTED の完全なエミュレーションは行われません。
  • **未実装・浅い部分**: 接続の並列実行最適化や、複雑な分散トランザクションの完全な 2PC（2フェーズコミット）はサポートされていません。
  • **注意点**: disconnect_cached_connections は、同一セッション内の他ユーザーの接続も切断する可能性があるため、マルチユーザー環境での運用には注意が必要です。
