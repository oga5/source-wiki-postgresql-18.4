contrib/postgres_fdw/postgres_fdw.h

概要

postgres_fdw.h は、PostgreSQL の外部データラッパー（FDW）である postgres_fdw モジュールの中心的なヘッダーファイルです。リモートの PostgreSQL サーバーと通信し、クエリを最適化・実行するために必要なデータ構造と、各サブモジュール（deparse.c, connection.c 等）が公開する関数プロトタイプを定義しています。

モジュール内での役割

本モジュールは、PostgreSQL のプランナおよびエグゼキュータと、リモートサーバー間のインターフェースを担います。このヘッダーは、プランナが作成した RelOptInfo に付加される FDW 固有の情報を定義し、クエリのプッシュダウン判断やコスト見積もりに必要な情報を保持する役割を果たします。

責務

  • **プランナ情報の保持**: リモートテーブルのコスト、選択率、プッシュダウン可能な条件式などを管理する PgFdwRelationInfo の定義。
  • **インターフェースの集約**: postgres_fdw を構成する各ソースファイル（connection.c, deparse.c 等）が提供する外部関数の宣言。
  • **定数・列挙型の定義**: リモートサンプリング手法や接続状態など、モジュール全体で共有すべき設定の定義。

重要なシンボル

  • PgFdwRelationInfo: プランナがリモートテーブルに対して最適化を行う際に使用する構造体。プッシュダウンの可否やコスト見積もりの結果を保持します。
  • PgFdwConnState: リモート接続に関連する非同期リクエストの状態を管理する構造体。
  • PgFdwSamplingMethod: ANALYZE 実行時にリモートサーバーからデータをサンプリングする際の手法を定義する列挙型。

制御フローとデータフロー

本ファイル自体はデータ構造と関数宣言のみを提供します。 1. **プランニング時**: プランナが PgFdwRelationInfo を参照し、リモートで実行可能な条件（remote_conds）を特定します。 2. **デパース時**: deparse.c の関数群が PgFdwRelationInfo の情報を元に、ローカルのクエリツリーをリモート用の SQL 文字列に変換します。 3. **実行時**: connection.c の関数が PgFdwConnState を用いてリモート接続を管理し、生成された SQL を送信します。

直接依存

  • foreign/foreign.h: FDW の基本構造定義。
  • libpq-fe.h: リモート PostgreSQL サーバーとの通信用ライブラリ（libpq）。
  • nodes/pathnodes.h / nodes/execnodes.h: PostgreSQL のプランナおよびエグゼキュータのノード定義。
  • utils/relcache.h: リレーションキャッシュへのアクセス。
  • lib/stringinfo.h

関連コード

  • postgres_fdw.c: モジュールのエントリポイント。PgFdwRelationInfo の初期化や EXPLAIN 出力などを担当。
  • deparse.c: SQL 生成ロジック。PgFdwRelationInfo を入力としてリモート SQL を構築します。
  • connection.c: リモート接続のライフサイクル管理。

読者向けメモ

  • PgFdwRelationInfo は非常に巨大な構造体ですが、これはプランナがクエリ最適化の各段階で必要な情報を一元管理するためです。
  • pushdown_safe フラグは、そのリレーションがリモートで処理可能かどうかを判断する最も重要なフラグの一つです。
  • 新規に機能を追加する場合、まずは PgFdwRelationInfo に必要なフィールドが不足していないかを確認してください。
