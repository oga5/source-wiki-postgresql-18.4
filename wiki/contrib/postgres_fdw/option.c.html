contrib/postgres_fdw/option.c

概要

option.c は、postgres_fdw が使用する外部データラッパー（FDW）関連のオプションおよび GUC（Grand Unified Configuration）パラメータの定義と検証を担うモジュールです。

モジュール内での役割

postgres_fdw の設定管理の基盤です。ユーザーが CREATE SERVER や CREATE FOREIGN TABLE 等で指定したオプションが、システムカタログの定義に適合しているかを検証し、接続時に libpq へ渡すパラメータを抽出する役割を持ちます。

責務

  • **オプション検証**: postgres_fdw_validator を通じて、指定されたオプション名が正しいか、またその値が型や制約（数値範囲、権限など）を満たしているかをチェックする。
  • **設定の初期化**: libpq が提供するデフォルトオプションと、postgres_fdw 固有のオプションを統合したリストを管理する。
  • **接続パラメータ抽出**: 外部サーバー接続時に必要な libpq 関連オプションのみを抽出する。
  • **GUC 管理**: postgres_fdw.application_name などの拡張固有パラメータを定義・管理する。

重要なシンボル

  • postgres_fdw_validator: SQL レベルの検証関数。オプション指定時に呼び出される。
  • InitPgFdwOptions: 内部的なオプションリストを構築する初期化関数。
  • PgFdwOption: オプション名、適用対象（カタログOID）、libpq 依存フラグを保持する構造体。
  • ExtractConnectionOptions: 接続時に libpq へ渡すオプションを抽出するユーティリティ。
  • process_pgfdw_appname: application_name 内の % エスケープシーケンスを現在のプロセス情報で置換する。

制御フローとデータフロー

1. **初期化**: InitPgFdwOptions が初回呼び出し時に PQconndefaults() を取得し、postgres_fdw 固有のオプションとマージして静的配列 postgres_fdw_options を構築する。 2. **検証**: postgres_fdw_validator が untransformRelOptions でリスト化されたオプションを走査し、is_valid_option で存在確認を行う。 3. **型チェック**: 各オプション名に応じて、defGetBoolean や parse_int 等で値の妥当性を検証し、不正な場合は ereport(ERROR, ...) で処理を中断する。 4. **抽出**: 接続確立時、ExtractConnectionOptions がオプションリストから libpq 関連のものだけをフィルタリングして抽出する。

読者向けメモ

  • 本ファイルは、libpq のオプションと postgres_fdw 固有のオプションを区別なく扱えるように設計されています。
  • InitPgFdwOptions はバックエンドプロセスごとに一度だけ実行されるため、メモリ管理は比較的単純です。
  • 新しいオプションを追加する場合は、non_libpq_options 配列への追加と、postgres_fdw_validator 内での検証ロジックの追加が必要です。
  • process_pgfdw_appname は、リモート接続時のアプリケーション名に動的な情報（プロセスIDやデータベース名）を埋め込むための強力な機能を提供します。
