contrib/postgres_fdw/shippable.c

概要

リモートサーバーへ実行を委譲（プッシュダウン）可能なデータベースオブジェクト（関数、演算子、データ型）を判定するためのロジックを提供します。

モジュール内での役割

postgres_fdw において、クエリの最適化時に「ローカルで定義されたオブジェクトがリモート側でも同一の挙動を保証できるか」を判断する判断基準を管理します。

責務

  • **組み込みオブジェクトの判定**: FirstGenbkiObjectId を基準に、PostgreSQL 標準のオブジェクトを自動的に「出荷可能（shippable）」とみなす。
  • **拡張機能の検証**: ユーザーが指定した拡張機能に含まれるオブジェクトを、リモート実行可能として許可する。
  • **判定結果のキャッシュ**: 頻繁に行われるカタログ検索のコストを削減するため、サーバー単位での判定結果をハッシュテーブルで保持する。

重要なシンボル

  • is_shippable: 外部から呼び出されるメイン関数。オブジェクトがリモートで実行可能か判定する。
  • is_builtin: オブジェクトが組み込み（pg_catalog 等）であるかを判定する。
  • ShippableCacheHash: 判定結果を保持するハッシュテーブル。
  • ShippableCacheKey: キャッシュのキー。オブジェクトID、カタログID、サーバーIDで構成される。

追加の索引済みシンボル

  • InitializeShippableCache
  • InvalidateShippableCacheCallback
  • ShippableCacheEntry
  • lookup_shippable

制御フローとデータフロー

1. is_shippable が呼び出されると、まず is_builtin で組み込みオブジェクトかを確認し、該当すれば即座に true を返す。 2. 組み込みでない場合、ShippableCacheHash を検索する。 3. キャッシュミス時は lookup_shippable を呼び出し、getExtensionOfObject を用いて対象オブジェクトが許可された拡張機能に含まれるかを確認する。 4. 判定結果をキャッシュに格納し、結果を返す。 5. pg_foreign_server が更新された場合、InvalidateShippableCacheCallback がトリガーされ、キャッシュ全体がフラッシュされる。

  • utils/hsearch.h: キャッシュ用ハッシュテーブル操作。
  • utils/syscache.h: システムカタログのキャッシュおよび無効化コールバック登録。
  • access/transam.h
  • postgres_fdw.h
  • utils/inval.h

関連コード

  • postgres_fdw.h: PgFdwRelationInfo 構造体定義。
  • postgres_fdw.c: fpinfo（外部テーブル情報）の管理と本ファイルの呼び出し元。

読者向けメモ

  • **キャッシュの粒度**: キャッシュはバックエンドの生存期間中保持されます。ALTER SERVER 等でサーバー設定が変更された場合、コールバックによりキャッシュが全破棄される点に注意してください。
  • **設計上の制約**: 現在の判定ロジックは「拡張機能のメンバーであるか」に依存しています。将来的に個別の関数単位での許可リスト機能が追加される余地があります。
  • **組み込みの定義**: is_builtin は FirstGenbkiObjectId を境界としています。これは information_schema 等の動的なオブジェクトを除外するための保守的な境界です。
