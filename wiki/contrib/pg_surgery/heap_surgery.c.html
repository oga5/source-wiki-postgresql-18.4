contrib/pg_surgery/heap_surgery.c

概要

heap_surgery.c は、破損したヒープテーブルに対して直接的な修正を行うための低レベルな関数群を提供します。主に、整合性が失われたタプルを強制的に削除（Kill）したり、フリーズ（Freeze）状態に強制変更したりするために使用されます。

モジュール内での役割

contrib/pg_surgery モジュールの中心的な実装ファイルです。SQL インターフェース（heap_force_kill, heap_force_freeze）と、それらの処理を共通化するバックエンドロジックを提供します。

責務

  • **強制的な状態変更**: 指定された TID（タプル識別子）リストに基づき、ヒープページ内のタプル状態を直接書き換える。
  • **整合性チェック**: 操作対象がヒープテーブルであること、および実行者が所有者またはスーパーユーザーであることを検証する。
  • **WAL ログの生成**: ページ変更を永続化するために、必要な WAL ログ（log_newpage_buffer）を生成する。
  • **可視性マップの更新**: KILL 操作時に、必要に応じて可視性マップ（VM）のビットをクリアする。

重要なシンボル

  • HeapTupleForceOption: 実行する操作の種類（HEAP_FORCE_KILL または HEAP_FORCE_FREEZE）を定義する列挙型。
  • heap_force_kill: SQL から呼び出される、タプルを強制的に Dead にする関数。
  • heap_force_freeze: SQL から呼び出される、タプルを強制的に Frozen 状態にする関数。
  • heap_force_common: 上記2関数の実処理を行う共通関数。

制御フローとデータフロー

1. **入力検証**: TID 配列をソートし、ページ単位で処理できるように準備する。 2. **ページ走査**: find_tids_one_page を使用して、同じブロック内の TID をグループ化する。 3. **バッファロック**: 対象ブロックを ReadBuffer で読み込み、LockBufferForCleanup で排他制御を行う。 4. **状態変更**:

  • KILL: ItemIdSetDead を使用してタプルを無効化。
  • FREEZE: タプルのヘッダー情報を直接書き換え、FrozenTransactionId を設定。

    • 5. **コミット**: 変更があれば MarkBufferDirty を呼び出し、WAL に記録してロックを解放する。

読者向けメモ

  • **危険性**: 本モジュールの関数は、PostgreSQL の整合性チェックをバイパスしてデータを直接書き換えます。誤った使用はデータベースの深刻な破損を招くため、緊急時の最終手段としてのみ使用してください。
  • **リカバリ中の制限**: RecoveryInProgress() が真の場合、これらの関数は実行できません。
  • **ロック**: 操作対象のリレーションに対して RowExclusiveLock を取得します。
  • **WAL**: 変更は WAL に記録されるため、PITR（ポイントインタイムリカバリ）やレプリケーションには反映されます。
