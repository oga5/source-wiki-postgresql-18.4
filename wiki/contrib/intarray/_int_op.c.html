contrib/intarray/_int_op.c

概要

本ファイルは、PostgreSQLの intarray モジュールにおける整数配列操作のためのSQL呼び出し可能関数（Datum）の実装を提供します。配列の比較、集合演算、および配列の加工（ソート、抽出、要素の追加・削除）を行うためのインターフェースを定義しています。

モジュール内での役割

contrib/intarray サブシステムにおいて、ユーザーがSQLから直接利用する演算子や関数のバックエンド実装層です。_int.h で定義された内部ロジックと、PostgreSQLの関数呼び出し規約（PG_FUNCTION_ARGS）を橋渡しします。

責務

  • **比較・集合演算**: 配列同士の包含関係、等価性、重なり、和集合、積集合の計算。
  • **配列操作**: 配列のソート（昇順・降順）、重複排除、部分配列の抽出。
  • **要素管理**: 特定の整数要素の追加・削除、および配列同士の結合。

重要なシンボル

  • _int_contains / _int_contained: 配列の包含関係を判定する主要関数。
  • _int_union / _int_inter: 配列の和集合・積集合を生成する関数。
  • sort / sort_asc / sort_desc: 配列をソートする関数。
  • intarray_push_elem / intarray_del_elem: 配列に対する要素の追加・削除を行う関数。
  • intset_subtract: 集合演算としての差分を計算する関数。

追加の索引済みシンボル

  • icount
  • idx
  • intarray_push_array
  • intset
  • intset_union_elem

制御フローとデータフロー

1. **入力**: PG_GETARG_ARRAYTYPE_P_COPY 等を用いて、SQLから渡された ArrayType を取得します。 2. **検証**: CHECKARRVALID マクロで配列の整合性を確認します。 3. **処理**:

  • 多くの関数は SORT マクロや QSORT を使用して配列を正規化します。
  • inner_int_* 系の関数や _int_unique などの内部ヘルパーを呼び出し、計算を実行します。

    • 4. **出力**: PG_RETURN_* マクロを使用して、結果をPostgreSQLのデータ型として返却します。メモリ管理のため、不要になった配列は pfree で解放します。

直接依存

  • postgres.h: PostgreSQLのコア機能および型定義。
  • _int.h: intarray モジュール共通のデータ構造、マクロ（CHECKARRVALID, SORT, QSORT 等）、および内部関数プロトタイプ。

関連コード

  • _int_tool.c: 本ファイルから呼び出される inner_int_* 系の関数や、配列操作の低レベルな実装が含まれています。
  • _int_bool.c: 配列に対するブールクエリ演算の実装。
  • _int_gin.c / _int_gist.c: インデックスサポート関数。

読者向けメモ

  • **メモリ管理**: 本ファイル内の関数は、入力配列を PG_GETARG_ARRAYTYPE_P_COPY でコピーして操作することが多いです。これは、インプレースなソートや変更が元のデータに影響を与えないようにするためです。
  • **マクロの活用**: CHECKARRVALID や SORT などのマクロは _int.h で定義されており、これらが配列操作の安全性を担保しています。
  • **パフォーマンス**: 集合演算（union, inter, subtract）は、効率的な計算のために事前にソートと重複排除を行う前提で設計されています。
