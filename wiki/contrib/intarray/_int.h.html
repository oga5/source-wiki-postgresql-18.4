contrib/intarray/_int.h

概要

_int.h は、PostgreSQL の intarray モジュールにおける共通定義、マクロ、およびデータ構造を保持するヘッダーファイルです。整数配列の操作、GiST/GIN インデックス用のシグネチャ生成、およびブール検索クエリの処理に必要な基盤を提供します。

モジュール内での役割

intarray モジュールの中心的なヘッダーであり、他のすべてのソースファイル（_int_op.c, _int_gist.c, _int_gin.c 等）から参照されます。配列の正規化、ビットベクトル操作、およびクエリ構造体の定義を一元管理することで、モジュール全体の一貫性を維持しています。

責務

  • **配列操作の抽象化**: int4 配列へのアクセスや、ソート・重複排除（PREPAREARR）の標準化。
  • **インデックスサポート**: GiST/GIN インデックスで利用されるシグネチャ（BITVECP）の生成・操作ロジックの提供。
  • **クエリ定義**: QUERYTYPE 構造体を用いたブール検索の構文解析と実行のためのデータ構造定義。
  • **設定管理**: オペレータクラスのオプション（GISTIntArrayOptions 等）を介した動的なパラメータ取得。

重要なシンボル

  • **GISTTYPE**: GiST インデックスのキー構造体。
  • **QUERYTYPE**: ブール検索クエリを保持する可変長構造体。
  • **BITVECP**: シグネチャ（ビットベクトル）を扱うためのポインタ型。
  • **PREPAREARR(x)**: 配列をソートし、重複を除去する必須マクロ。
  • **GETBIT/SETBIT**: ビットベクトルに対するビット単位の操作用マクロ。

制御フローとデータフロー

本ファイルは実行コードを含みませんが、以下のフローを定義します： 1. **データ準備**: CHECKARRVALID で入力配列の妥当性を検証し、PREPAREARR でインデックス作成に適した形式へ変換します。 2. **シグネチャ生成**: gensign 関数と HASH マクロにより、整数値をビットベクトルへマッピングします。 3. **クエリ実行**: QUERYTYPE に格納されたポーランド記法形式のクエリを、signconsistent や execconsistent を通じて配列データと照合します。

直接依存

  • utils/array.h: PostgreSQL の標準配列型定義。
  • utils/memutils.h: メモリ管理ユーティリティ。

関連コード

  • _int_tool.c: 本ヘッダーで宣言された isort や _int_unique などのユーティリティ関数の実装。
  • _int_gist.c / _int_gin.c: 本ヘッダーのデータ構造を利用したインデックスアクセスメソッドの実装。
  • _int_bool.c: QUERYTYPE を用いたブール検索ロジックの実装。

読者向けメモ

  • **マクロの副作用**: ARRPTR や GETBYTE などのマクロは引数を複数回評価する可能性があるため、呼び出し時に副作用のある式を渡さないよう注意してください。
  • **NULL 値の扱い**: intarray は配列内の NULL 値をサポートしていません。CHECKARRVALID がエラーを投げるため、呼び出し側で事前にフィルタリングが必要です。
  • **可変長データ**: GISTTYPE や QUERYTYPE は FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリ確保時には CALCGTSIZE や COMPUTESIZE マクロを必ず使用してください。
