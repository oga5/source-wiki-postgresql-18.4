contrib/intarray/_int_tool.c

概要

_int_tool.c は、intarray モジュールにおける整数配列操作の低レベルなユーティリティ関数群を提供します。主に配列の集合演算（包含、重なり、和、積）や、メモリ管理を伴う配列の生成・リサイズ処理を担います。

モジュール内での役割

contrib/intarray の中核となる計算エンジンです。GiST や GIN インデックスのサポート関数や、演算子クラスの実装から呼び出される「配列処理の共通ライブラリ」として機能します。

責務

  • **集合演算の実行**: ソート済み配列を前提とした効率的な包含・重なり・和・積の計算。
  • **メモリ管理**: PostgreSQL の ArrayType 構造体の動的確保、リサイズ、コピー。
  • **ソートと一意化**: lib/sort_template.h を利用したソート処理および重複排除。
  • **補助操作**: 配列への要素追加、検索、シグネチャ生成（gensign）。

重要なシンボル

  • inner_int_contains / inner_int_overlap: 集合演算の基本関数。入力がソート済みであることを前提に線形時間で動作します。
  • inner_int_union / inner_int_inter: 配列の和集合・積集合を計算し、新しい ArrayType を返します。
  • new_intArrayType / resize_intArrayType: intarray 専用のメモリ確保・調整関数。
  • _int_unique: 配列内の重複要素をインプレースで削除します。

制御フローとデータフロー

1. **入力検証**: CHECKARRVALID マクロにより、入力された ArrayType が有効な整数配列であることを確認します。 2. **演算**: 多くの関数（inner_int_* 系）は、2つの配列ポインタを同時に走査するポインタ演算を行い、結果を構築します。 3. **メモリ確保**: new_intArrayType で必要なバイト数を計算し palloc0 を実行。必要に応じて repalloc でサイズを調整します。 4. **ソート**: isort_cmp を比較関数として lib/sort_template.h が生成するソート関数を呼び出し、順序を保証します。

直接依存

  • postgres.h: PostgreSQL の基本型とメモリ管理関数。
  • _int.h: モジュール共通の定義とマクロ。
  • lib/qunique.h: 重複排除アルゴリズム。
  • lib/sort_template.h: 型安全なソートテンプレート。
  • common/int.h: 整数演算の補助。
  • catalog/pg_type.h
  • limits.h

関連コード

  • _int_op.c: 本ファイルで定義された関数を利用して、SQL 演算子（@> や && など）を実装しています。
  • _int_gist.c / _int_gin.c: インデックスの構築・検索時に本ファイルの集合演算関数を呼び出します。

読者向けメモ

  • **前提条件**: inner_int_ 系関数は、入力配列がソート済みであることを前提としています。ソートされていない配列を渡すと誤った結果を返すため、呼び出し側で必ずソート済みであることを保証してください。
  • **メモリ管理**: 本ファイル内の関数が返す ArrayType は palloc されているため、呼び出し側で適切に管理（あるいは pfree）する必要があります。
  • **パフォーマンス**: ほとんどの集合演算は $O(N+M)$ の線形時間で動作するように設計されています。複雑なロジックを実装する際は、この計算量を維持することを推奨します。
