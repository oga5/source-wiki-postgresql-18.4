contrib/intarray/_int_gin.c

概要

PostgreSQL の intarray モジュールにおける GIN (Generalized Inverted Index) インデックスのサポートを実装するファイルです。整数配列に対する効率的な検索を実現するための演算子クラスのメソッドを提供します。

モジュール内での役割

contrib/intarray の中で、GIN インデックスのインデックス作成および検索フェーズにおける「クエリの抽出」と「整合性チェック」のロジックを担います。他のファイル（_int_bool.c など）で定義されたブール検索ロジックを GIN インデックスのフレームワークに統合する役割を果たします。

責務

  • **クエリの分解**: インデックス検索時に、クエリからインデックスが検索すべきキー（VAL）を抽出する。
  • **検索モードの決定**: クエリの内容に基づき、GIN が全スキャンを行うべきか、特定のキーのみを検索すべきかを判断する。
  • **整合性判定**: インデックスから返された結果が、実際のクエリ条件を満たしているかを判定し、再チェック（recheck）が必要かどうかを通知する。

重要なシンボル

  • ginint4_queryextract: GIN インデックスの extractQuery メソッド。クエリをインデックス検索用のキー配列に変換します。
  • ginint4_consistent: GIN インデックスの consistent メソッド。インデックスの検索結果がクエリ条件に適合するかを判定します。

制御フローとデータフロー

1. **ginint4_queryextract**:

  • BooleanSearchStrategy の場合、QUERYTYPE を解析し、必要な値（VAL）を抽出します。必須値がない場合は GIN_SEARCH_MODE_ALL を設定し、全スキャンを指示します。
  • 配列検索の場合、演算子（@> や && など）に応じて searchMode を設定し、配列要素をキーとして抽出します。

    • 2. **ginint4_consistent**:

  • 演算子戦略（StrategyNumber）に基づき、check[] 配列（インデックスキーの存在フラグ）を評価します。
  • RTOverlapStrategyNumber 等では recheck = false とし、損失のない検索であることを示します。
  • BooleanSearchStrategy の場合は gin_bool_consistent を呼び出し、複雑なブール条件を評価します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • _int.h: intarray モジュール共通のデータ構造（QUERYTYPE, ITEM 等）。
  • access/gin.h: GIN インデックスのインターフェース定義。
  • access/stratnum.h: 演算子戦略番号の定義。

関連コード

  • _int_bool.c: ブール検索の評価ロジック（gin_bool_consistent の実装元）。
  • _int_op.c: intarray がサポートする演算子の定義。

読者向けメモ

  • 本ファイルは GIN インデックスの標準的なインターフェースを実装しているため、GIN の仕組み（extractQuery と consistent）を理解していると読みやすいです。
  • recheck フラグの扱いに注意してください。一部の演算子（ContainedBy など）はインデックスの性質上、再チェックが必要なため true に設定されています。
  • 複雑なブール検索については、_int_bool.c のロジックに強く依存しているため、適宜参照してください。
