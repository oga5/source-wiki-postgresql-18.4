contrib/intarray/_int_selfuncs.c

概要

PostgreSQL の intarray 拡張における、整数配列演算子およびクエリ演算子（@@）の選択率（Selectivity）推定関数を実装するファイルです。クエリプランナが最適な実行計画を選択するために必要なコスト見積もりを提供します。

モジュール内での役割

contrib/intarray モジュールの一部として、インデックスや演算子の定義と連携し、プランナに対して統計情報に基づいたヒントを与えます。主に arraycontsel や arraycontjoinsel といった組み込みの配列選択率推定関数をラップし、intarray 特有の演算子に適合させる役割を担います。

責務

  • **演算子の選択率推定**: &&（重なり）、@>（含む）、<@（含まれる）演算子の選択率を計算する。
  • **クエリマッチングの推定**: @@ 演算子を用いた query_int 型に対する複雑な論理クエリの選択率を算出する。
  • **統計情報の活用**: pg_statistic から取得した Most Common Elements (MCE) リストを利用し、より精度の高い見積もりを行う。

重要なシンボル

  • _int_overlap_sel / _int_contains_sel / _int_contained_sel: 各演算子の制限選択率推定関数。
  • _int_matchsel: @@ 演算子用のメイン推定関数。
  • int_query_opr_selec: query_int のツリー構造を再帰的に走査し、論理演算（&, |, !）を考慮して選択率を計算する内部関数。

追加の索引済みシンボル

  • compare_val_int4

制御フローとデータフロー

1. **ラッパー関数**: _int_overlap_sel 等は、引数を標準の arraycontsel に転送する際、intarray 固有の演算子 OID を組み込みの配列演算子 OID に置き換えて呼び出します。 2. **_int_matchsel のフロー**:

  • get_restriction_variable で対象列の統計情報を取得。
  • pg_statistic から MCE リストを抽出し、int_query_opr_selec へ渡す。
  • int_query_opr_selec は query_int の構文木を再帰的に辿り、各要素の出現頻度を MCE から検索して確率を合成します。
  • 最終的な選択率に NULL 分率を適用し、結果を返します。

読者向けメモ

  • **再帰の深さ**: int_query_opr_selec は再帰関数であるため、複雑な query_int を扱う際は check_stack_depth() によるスタック保護が行われています。
  • **MCE の利用**: MCE に存在しない要素については minfreq / 2 を用いたヒューリスティックな推定が行われます。これは統計情報が不完全な場合の安全策です。
  • **拡張性**: 新しい演算子を追加する場合、既存のラッパー関数を参考に、適切な組み込み関数へ OID をマッピングするだけで対応可能です。
