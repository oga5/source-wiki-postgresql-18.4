contrib/cube/cubedata.h

概要

cubedata.h は、PostgreSQL の cube 拡張モジュールにおけるデータ構造 NDBOX の定義と、その操作に必要なマクロ、およびパーサーインターフェースを定義するヘッダーファイルです。

モジュール内での役割

cube モジュールの中心的なデータ型である NDBOX のレイアウトを規定します。このファイルは、データ構造のメモリ配置を定義することで、cube.c（実装）および cubescan.l / cubeparse.y（構文解析）の橋渡しをします。

責務

  • NDBOX 構造体のメモリレイアウトの定義。
  • NDBOX のヘッダー情報（次元数、ポイントフラグ）を操作するアクセスマクロの提供。
  • PostgreSQL の fmgr インターフェースを介したデータ変換マクロの提供。
  • 構文解析器（Flex/Bison）との間で共有される型定義と関数プロトタイプの提供。

重要なシンボル

  • NDBOX: 多次元ボックスを表現する可変長構造体。x 配列に座標を保持します。
  • header: 次元数とポイントフラグを格納するビットフィールド。
  • IS_POINT(cube): 構造体が「点」であるか「ボックス」であるかを判定します。
  • LL_COORD / UR_COORD: 指定した次元の左下・右上の座標を取得するアクセサ。
  • CUBE_SIZE(_dim): 指定次元数に基づいた NDBOX のメモリサイズを計算します。

追加の索引済みシンボル

  • yyscan_t

制御フローとデータフロー

本ファイルはロジックを含みませんが、データフローの基盤を提供します。 1. **データアクセス**: PG_GETARG_NDBOX_P で入力された Datum を NDBOX ポインタに変換します。 2. **座標取得**: LL_COORD や UR_COORD を通じて、header のフラグに基づき、メモリ上の適切な位置から double 型の座標値を読み取ります。 3. **解析フロー**: cube_scanner_init でスキャナを初期化し、cube_yyparse を通じて文字列から NDBOX 構造体を生成します。

直接依存

本ファイルは特定のソースファイルに依存せず、PostgreSQL のコアヘッダー（postgres.h 等）がインクルードされる環境下で利用されることを前提としています。

関連コード

  • cube.c: 本ヘッダーで定義された構造体を利用して、距離計算や演算子を実装しています。
  • cubescan.l / cubeparse.y: 文字列形式のキューブ表現を NDBOX に変換するためのスキャナおよびパーサーです。

読者向けメモ

  • **メモリレイアウト**: NDBOX は FLEXIBLE_ARRAY_MEMBER を使用しています。UR_COORD マクロは、ポイントフラグが立っている場合は LL_COORD と同じ値を返し、そうでない場合は x 配列の後半部分を参照する点に注意してください。
  • **バイナリ互換性**: header フィールドは古いバージョンとの互換性のため unsigned int として定義されています。直接操作せず、必ず提供されているマクロ（DIM, IS_POINT 等）を使用してください。
  • **拡張性**: CUBE_MAX_DIM は 100 に制限されています。これを超える次元数を扱う場合は、計算時のオーバーフローリスクを考慮する必要があります。
