contrib/xml2/xslt_proc.c

概要

PostgreSQL から XSLT 変換機能を利用可能にするための拡張モジュールです。libxslt を利用して、XML ドキュメントに対して XSLT スタイルシートを適用し、変換結果をテキストとして返します。

モジュール内での役割

contrib/xml2 モジュールの一部として、XML 処理機能の変換レイヤーを担います。xpath.c が XML のクエリ（XPath）を担当するのに対し、本ファイルはスタイルシートによるドキュメント変換を担当します。

責務

  • **XSLT 変換の実行**: xslt_process 関数を通じ、SQL から XML と XSLT を受け取り変換を実行する。
  • **セキュリティ管理**: libxslt のセキュリティ設定を適用し、ファイルシステムやネットワークへの不正なアクセスを禁止する。
  • **パラメータ解析**: SQL から渡された文字列形式のパラメータを、libxslt が解釈可能な形式に変換する。
  • **リソース管理**: libxml/libxslt が確保したメモリやコンテキストを、エラー発生時を含めて確実に解放する。

重要なシンボル

  • xslt_process: SQL から呼び出されるメインの関数。
  • parse_params: key=value,key=value 形式の文字列を libxslt 用の配列に変換する内部ヘルパー。
  • PgXmlErrorContext: xpath.c で定義されたエラーハンドリング用コンテキスト。

制御フローとデータフロー

1. **初期化**: pgxml_parser_init でエラーハンドリングをセットアップ。 2. **解析**: xmlReadMemory を用いて XML ドキュメントとスタイルシートをメモリ上に展開。 3. **変換準備**: xsltParseStylesheetDoc でスタイルシートをコンパイルし、xsltNewTransformContext で変換コンテキストを作成。 4. **セキュリティ適用**: xsltSetSecurityPrefs を呼び出し、外部リソースへのアクセスを制限。 5. **変換実行**: xsltApplyStylesheetUser で変換を行い、結果を文字列として抽出。 6. **クリーンアップ**: PG_TRY/PG_CATCH ブロックにより、異常終了時でも確実に xmlFreeDoc や xsltCleanupGlobals を実行してメモリリークを防ぐ。

直接依存

  • **PostgreSQL**: postgres.h, fmgr.h, utils/builtins.h, utils/xml.h
  • **libxml2**: libxml/xpath.h, libxml/tree.h, libxml/xmlmemory.h
  • **libxslt**: libxslt/xslt.h, libxslt/xsltInternals.h, libxslt/security.h, libxslt/transform.h, libxslt/xsltutils.h

関連コード

  • contrib/xml2/xpath.c: 本モジュールの基盤となる XML 解析およびエラーハンドリング機能が定義されています。

読者向けメモ

  • **ビルド要件**: USE_LIBXSLT マクロが定義されていない環境では、関数はエラーを返すのみとなります。
  • **セキュリティ**: 本実装では、XSLT 変換中のファイル読み書きやネットワークアクセスを厳格に禁止しています。これは SQL インジェクションやサーバーサイドの脆弱性を防ぐための重要な設計です。
  • **メモリ管理**: libxml/libxslt の API は PostgreSQL のメモリ管理（palloc 等）とは独立しているため、PG_TRY ブロック内での確実な解放が不可欠です。
