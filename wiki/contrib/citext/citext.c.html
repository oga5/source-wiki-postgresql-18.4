contrib/citext/citext.c

概要

citext.c は、PostgreSQL において大文字・小文字を区別しない文字列型（citext）を実現するためのコア実装です。標準の text 型の挙動を拡張し、比較、ソート、ハッシュ計算において大文字・小文字を無視した処理を提供します。

モジュール内での役割

contrib/citext モジュールの唯一のソースファイルとして、SQL レベルで定義された citext 型の演算子や関数を C 言語で実装しています。

責務

  • **正規化**: 入力文字列を str_tolower を用いて小文字に変換し、比較やハッシュ計算の基盤とする。
  • **比較演算**: citext 型に対する等価性、不等価性、大小比較の提供。
  • **インデックスサポート**: ハッシュインデックスや B-tree インデックスで利用可能な比較・ハッシュ関数の提供。
  • **集約関数**: min() や max() で利用される citext_smaller / citext_larger の実装。

重要なシンボル

  • citextcmp: 内部的な比較関数。str_tolower で正規化した後、varstr_cmp で照合順序（Collation）を考慮した比較を行う。
  • internal_citext_pattern_cmp: パターンマッチング用の比較関数。照合順序を無視したバイナリ比較を行う。
  • citext_hash / citext_hash_extended: ハッシュインデックス用のハッシュ計算関数。
  • citext_eq / citext_ne: 等価・不等価判定。strcmp を使用して効率化されている。

制御フローとデータフロー

1. **正規化**: すべての比較・ハッシュ関数は、まず str_tolower を呼び出し、入力された text 型データを小文字の C 文字列に変換します。 2. **比較**:

  • 通常の比較（citext_cmp 等）では、正規化後に varstr_cmp を呼び出し、PostgreSQL の照合順序設定に従って比較します。
  • パターン比較（citext_pattern_cmp 等）では、正規化後に memcmp を用いたバイナリ比較を行い、照合順序の影響を排除します。

    • 3. **メモリ管理**: str_tolower で生成された一時的な文字列は、処理終了後に必ず pfree で解放されます。また、PG_FREE_IF_COPY を使用して、TOAST された入力データのメモリリークを防ぎます。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • fmgr.h: 関数マネージャインターフェース。
  • utils/formatting.h: str_tolower 等の文字列変換ユーティリティ。
  • utils/varlena.h / varatt.h: 可変長データ型（text）の操作用。
  • common/hashfn.h: hash_any 等のハッシュ計算用。
  • catalog/pg_collation.h

関連コード

  • catalog/pg_collation.h: 照合順序の定義。
  • PostgreSQL コアの src/backend/utils/adt/varlena.c: varstr_cmp 等の比較ロジックの基盤。

読者向けメモ

  • **照合順序の扱い**: 現在の実装では、str_tolower に DEFAULT_COLLATION_OID を固定で使用しています。これは、大文字・小文字の変換ルールが照合順序に依存しないようにするためです。
  • **パフォーマンス**: citext_eq 等の等価判定では、strcoll のような重い照合順序依存の比較を避け、strcmp を使用して最適化されています。
  • **拡張性**: 今後の PostgreSQL コアのインフラが照合順序依存のハッシュや等価比較を完全にサポートすれば、この実装も簡素化される可能性があります。
