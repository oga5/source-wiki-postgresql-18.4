contrib/lo/lo.c

概要

contrib/lo/lo.c は、PostgreSQL のラージオブジェクト（Large Object: LO）の孤立化を防ぐためのトリガー関数を提供するモジュールです。テーブル内の特定の列に格納された OID が指すラージオブジェクトを、行の更新や削除に合わせて自動的に削除（unlink）します。

モジュール内での役割

contrib/lo モジュールの唯一のソースファイルであり、データベース管理者がトリガーとして登録するためのインターフェースを提供します。

責務

  • **孤立オブジェクトのクリーンアップ**: 行の削除時、または対象列の更新時に、古いラージオブジェクトを be_lo_unlink を呼び出して削除する。
  • **トリガーの検証**: 実行コンテキストが正しくトリガーとして呼び出されているか、および必要な引数（監視対象の列名）が存在するかを検証する。

重要なシンボル

  • lo_manage: 本ファイルのメイン関数。CREATE TRIGGER で指定されるトリガー関数本体。
  • TriggerData: トリガー実行時のコンテキスト情報（行データ、トリガー定義など）を保持する構造体。
  • be_lo_unlink: ラージオブジェクトを物理的に削除するための内部関数。

制御フローとデータフロー

1. **初期化と検証**: fcinfo から TriggerData を取得し、行レベルのトリガーとして正しく呼び出されているかを確認します。 2. **対象列の特定**: トリガー引数から監視対象の列名を取得し、SPI_fnumber で属性番号を特定します。 3. **更新処理**: TRIGGER_FIRED_BY_UPDATE の場合、対象列の値が変更されたかを bms_is_member で判定し、変更があれば旧値を be_lo_unlink で削除します。 4. **削除処理**: TRIGGER_FIRED_BY_DELETE の場合、削除される行の対象列にある OID を取得し、be_lo_unlink で削除します。 5. **戻り値**: 処理対象のタプル（rettuple）を返します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • commands/trigger.h: トリガー関連のデータ構造とマクロ。
  • executor/spi.h: Server Programming Interface (SPI) を介したデータアクセス。
  • utils/fmgrprotos.h: 関数マネージャのプロトタイプ定義。
  • utils/rel.h: リレーション（テーブル）情報の操作。

関連コード

  • src/backend/storage/large_object/inv_api.c: be_lo_unlink が定義されているラージオブジェクト API の実装。
  • src/backend/executor/spi.c: 本ファイルで使用している SPI_ 系関数の実装。

読者向けメモ

  • 本関数は「トリガー関数」として設計されています。CREATE TRIGGER 文で、監視対象の列名を引数として渡す必要があります。
  • SPI_getvalue を使用して文字列として OID を取得し、atooid で変換する手法をとっています。
  • 更新処理において、値が変更された場合のみ削除を行うため、無駄な unlink を抑制するロジックになっています。
