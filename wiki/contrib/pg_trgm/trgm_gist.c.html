contrib/pg_trgm/trgm_gist.c

概要

PostgreSQL の pg_trgm 拡張における GiST インデックスのサポートを実装するソースファイルです。文字列のトリグラム（3文字の連続）に基づいたインデックス構造の構築、検索、および最適化ロジックを提供します。

モジュール内での役割

pg_trgm モジュールにおいて、GiST インデックスの演算子クラス（gist_trgm_ops）のバックエンドとして機能します。GIN インデックスが転置インデックスとして機能するのに対し、本ファイルは GiST の木構造を用いた近傍検索やパターンマッチングを担います。

責務

  • **インデックスの圧縮と展開**: gtrgm_compress / decompress によるインデックスキーの最適化。
  • **検索の整合性判定**: gtrgm_consistent による、インデックスノードが検索条件を満たすかの判定。
  • **距離計算**: gtrgm_distance による類似度検索（k-NN）のサポート。
  • **木構造の維持**: gtrgm_union、gtrgm_same、gtrgm_penalty、gtrgm_picksplit を通じた GiST 木のバランス調整とノード管理。

重要なシンボル

  • TrgmGistOptions: インデックス作成時に指定可能なオプション（siglen）を保持する構造体。
  • gtrgm_consistent_cache: 検索時の CPU コストを抑えるため、抽出済みトリグラムや正規表現グラフをキャッシュする構造体。
  • gtrgm_consistent: 検索クエリとインデックスノードの署名（シグネチャ）を比較する中核関数。
  • gtrgm_picksplit: GiST 木の分割時に、ノード間の重なりを最小化する分割アルゴリズム。

制御フローとデータフロー

1. **インデックス構築**: gtrgm_compress が入力文字列からトリグラムを生成し、ビットベクトル（シグネチャ）にハッシュ化して格納します。 2. **検索**: gtrgm_consistent が呼び出されると、まず fcinfo->flinfo->fn_extra を通じてキャッシュを確認します。キャッシュがない場合、クエリからトリグラムを抽出し、戦略（LIKE, RegExp, Similarity）に応じてインデックスノードのビットベクトルと照合します。 3. **距離計算**: gtrgm_distance は類似度演算子（% など）に対し、インデックスキーとクエリの距離を計算し、近傍順に結果を返します。

直接依存

  • postgres.h: PostgreSQL の基本定義。

  • trgm.h: トリグラムのデータ構造と共通関数。
  • port/pg_bitutils.h: ビット操作（popcount 等）。
  • fmgr.h
  • varatt.h

関連コード

  • trgm_op.c: トリグラムの抽出や類似度計算のロジックが実装されており、本ファイルから頻繁に呼び出されます。
  • trgm_gin.c: 同じく pg_trgm の GIN インデックス実装。比較対象として参照されます。

読者向けメモ

  • **キャッシュの挙動**: gtrgm_consistent や gtrgm_distance は fn_extra を使用してクエリのトリグラムをキャッシュしますが、正規表現グラフの扱いに注意が必要です。現在の実装では、インデックスの再スキャン時にキャッシュがリークする可能性がある旨のコメントが残されています。
  • **シグネチャの性質**: GiST インデックスはビットベクトルによる近似検索を行うため、recheck フラグが true になるケースが多いです。これはインデックスが「候補」を絞り込み、最終的な判定はヒープ上の実データで行うことを意味します。
