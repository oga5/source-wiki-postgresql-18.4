contrib/pg_trgm/trgm_gin.c

概要

PostgreSQL の pg_trgm モジュールにおいて、GIN（Generalized Inverted Index）インデックスをサポートするための実装ファイルです。テキストデータからトリグラム（3文字の連続）を抽出し、GIN インデックスの構造に適合させるための関数群を提供します。

モジュール内での役割

pg_trgm の中で、GIN インデックスの演算クラス（opclass）のバックエンドとして機能します。B-tree や GiST とは異なり、GIN は転置インデックス構造を持つため、このファイルは「値の抽出」と「クエリの整合性判定」という GIN 特有のインターフェースを実装しています。

責務

  • **インデックスキーの抽出**: インデックス対象のテキストからトリグラムを生成し、GIN が扱える形式（Datum 配列）に変換する。
  • **クエリの解析**: 検索条件（LIKE, 正規表現, 類似度検索など）に基づき、インデックス検索に必要なトリグラムを抽出する。
  • **整合性判定**: インデックスからヒットしたトリグラムが、実際の検索条件を満たすかを判定する（consistent / triconsistent）。

重要なシンボル

  • gin_extract_value_trgm: インデックス構築時に呼び出され、テキストをトリグラムの配列に分解する。
  • gin_extract_query_trgm: 検索時に呼び出され、検索条件から必要なトリグラムを特定する。
  • gin_trgm_consistent: 検索結果の候補が条件を満たすか判定する（recheck が常に true となる）。
  • gin_trgm_triconsistent: 部分一致検索（GIN_MAYBE）をサポートするための 3 値論理判定関数。

追加の索引済みシンボル

  • gin_extract_trgm

制御フローとデータフロー

1. **抽出フェーズ**: generate_trgm を呼び出し、テキストをトリグラムのリストに変換。GIN の仕様に合わせて Datum 配列へ格納します。 2. **クエリフェーズ**: StrategyNumber に応じて処理を分岐します。

  • LIKE 検索: generate_wildcard_trgm でワイルドカードを考慮したトリグラムを抽出。
  • 正規表現検索: createTrgmNFA で NFA グラフを構築し、extra_data として保持。

    • 3. **判定フェーズ**: consistent 関数内で、インデックスから得られたトリグラムの集合と check 配列を比較。類似度検索の場合はトリグラムの出現数から上限値を計算し、閾値と比較します。

読者向けメモ

  • **recheck の必然性**: このファイル内の consistent 関数は、常に *recheck = true を設定します。これは、トリグラムのインデックスが「候補の絞り込み」には有効ですが、完全な一致判定には不十分（偽陽性を含む）であるためです。
  • **GIN 演算クラス**: 本ファイルで定義された関数は、SQL レベルで CREATE OPERATOR CLASS を通じて GIN インデックスと紐付けられます。
  • **正規表現の扱い**: 正規表現検索時は、トリグラムの組み合わせをグラフ構造（TrgmPackedGraph）として保持し、trigramsMatchGraph で効率的に判定を行っています。
