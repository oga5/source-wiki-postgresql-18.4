contrib/sepgsql/selinux.c

概要

本ファイルは、PostgreSQL の SE-PostgreSQL 拡張機能とカーネル空間の SELinux API（libselinux）との間の仲介役を担います。PostgreSQL 内部のオブジェクトクラスや権限コードと、SELinux ポリシー上の外部コードとの変換、およびアクセス制御判定の要求を行います。

モジュール内での役割

contrib/sepgsql モジュールにおける「低レイヤーのインターフェース」です。他のファイル（hooks.c や dml.c など）が PostgreSQL のフックから呼び出される際、最終的なセキュリティ判定やラベル計算を本ファイル経由で SELinux に委譲します。

責務

  • **マッピング管理**: PostgreSQL 内部のオブジェクトクラス（db_table 等）と SELinux 外部コードの対応表（selinux_catalog）の保持。
  • **アクセス判定**: security_compute_av を利用した権限チェックの実行と、結果の内部形式への変換。
  • **ラベル計算**: 新規オブジェクト作成時のデフォルトセキュリティコンテキストの取得。
  • **監査ログ**: 判定結果に基づいた PostgreSQL ログへの出力。

重要なシンボル

  • selinux_catalog: オブジェクトクラスと権限の変換テーブル。
  • sepgsql_compute_avd(): 指定されたコンテキストペアに対するアクセス権限の判定。
  • sepgsql_compute_create(): 新規オブジェクト作成時のデフォルトコンテキストの計算。
  • sepgsql_audit_log(): SELinux 判定結果のログ出力。
  • sepgsql_mode: 動作モード（Disabled, Default, Permissive, Internal）を保持する変数。

制御フローとデータフロー

1. **変換**: sepgsql_compute_avd は、PostgreSQL の内部クラスコードを string_to_security_class で SELinux 外部コードに変換します。 2. **判定**: security_compute_av_flags_raw を呼び出し、カーネルから av_decision を取得します。 3. **再変換**: 取得した外部ビットマスクを、selinux_catalog を参照して PostgreSQL 内部の権限ビットに再マッピングし、呼び出し元へ返します。 4. **ログ**: 判定結果が拒否された場合や監査設定に従い、sepgsql_audit_log が ereport(LOG, ...) を通じてログを生成します。

直接依存

  • postgres.h: PostgreSQL の基本定義。
  • lib/stringinfo.h: 監査ログ生成時の文字列バッファ操作。
  • sepgsql.h: モジュール共通の定数、型定義、および関数宣言。

関連コード

  • sepgsql.h: 本ファイルで使用される SEPG_CLASS_* や SEPG_DB_* 定数の定義元。
  • uavc.c: 本ファイルで計算された判定結果をキャッシュするユーザー空間 AVC（Access Vector Cache）の実装。
  • hooks.c: PostgreSQL の各フックから本ファイルの関数を呼び出す起点。

読者向けメモ

  • **実行時変換**: SELinux のクラスや権限コードはビルド時ではなく実行時のポリシーに依存するため、selinux_catalog を介した動的な変換が必須です。
  • **メモリ管理**: security_compute_create_name_raw 等の libselinux 関数は malloc でメモリを確保するため、PG_TRY/PG_FINALLY ブロックを使用して確実に freecon を呼び出し、メモリリークを防いでいます。
  • **未知の権限**: ポリシーに存在しない権限に遭遇した場合、security_deny_unknown() の設定に従い、デフォルトで拒否または許可する安全策が講じられています。
