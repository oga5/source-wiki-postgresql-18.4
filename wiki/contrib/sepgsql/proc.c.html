contrib/sepgsql/proc.c

概要

PostgreSQLの関数（プロシージャ）オブジェクトに対するSELinuxアクセス制御を実装するソースファイルです。関数の作成、削除、属性変更、実行、およびラベル変更時の権限チェックを担います。

モジュール内での役割

contrib/sepgsql サブシステムの一部として、pg_proc カタログに対する操作をフックし、SELinuxのポリシーに基づいたセキュリティ決定を強制します。

責務

  • **ライフサイクル管理**: 関数の作成時（post_create）にデフォルトのセキュリティラベルを計算・付与し、削除時（drop）に権限を検証する。
  • **属性変更の監視**: ALTER FUNCTION 等による名前空間の変更や属性変更（setattr）時の権限チェック。
  • **実行制御**: 関数呼び出し時の execute 権限の検証。
  • **ラベル管理**: SECURITY LABEL コマンドによる関数のラベル付け（relabel）の制御。

重要なシンボル

  • sepgsql_proc_post_create: 新規関数作成時に呼び出され、SELinuxコンテキストを計算してラベルを適用する。
  • sepgsql_proc_execute: 関数実行時に呼び出され、db_procedure:{execute} 権限を検証する。
  • sepgsql_proc_setattr: 関数定義変更時に、proleakproof 属性の変更などを検知し、必要な権限をチェックする。

追加の索引済みシンボル

  • sepgsql_proc_drop
  • sepgsql_proc_relabel

制御フローとデータフロー

1. **権限チェック**: 各関数は sepgsql_avc_check_perms または sepgsql_avc_check_perms_label を呼び出し、SELinuxのAVC（Access Vector Cache）を介してポリシー判定を行います。 2. **カタログ参照**: systable_beginscan や SearchSysCache1 を使用して、対象関数の現在の状態（名前空間、名前、引数型など）を取得します。 3. **ラベル計算**: sepgsql_compute_create を使用し、クライアントのコンテキストと親名前空間のコンテキストから、新規オブジェクトのデフォルトラベルを導出します。 4. **監査**: getObjectIdentity を用いて、ログ出力用のオブジェクト識別子を生成し、権限拒否時に詳細な監査情報を残します。

読者向けメモ

  • 本ファイルは pg_proc カタログに対するフックとして機能します。PostgreSQLのコアコードから呼び出されることを前提としています。
  • sepgsql_proc_post_create 内で SnapshotSelf を使用しているのは、トランザクション内で作成されたばかりのタプルを可視化するためです。
  • proleakproof 属性の変更は、セキュリティ上の重要度が高いため、SEPG_DB_PROCEDURE__INSTALL 権限を要求する設計になっています。
