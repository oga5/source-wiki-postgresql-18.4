contrib/sepgsql/sepgsql.h

概要

sepgsql.h は、PostgreSQL の SE-Linux 統合機能（SE-PostgreSQL）における中心的なヘッダーファイルです。SE-Linux のセキュリティコンテキストやアクセス制御の判定に必要な定数、オブジェクトクラス、アクセスベクタの定義、および各モジュールが公開する関数のプロトタイプ宣言を保持しています。

モジュール内での役割

contrib/sepgsql モジュール全体のインターフェース定義として機能します。このファイルで定義された定数や型は、hooks.c や selinux.c などの各実装ファイルで共通して参照され、PostgreSQL のデータベースオブジェクトに対するセキュリティポリシーの適用を標準化します。

責務

  • **定数定義**: SE-Linux の動作モード（Default, Permissive 等）およびデータベースオブジェクトクラス（Table, Schema, Procedure 等）の定義。
  • **アクセスベクタの集約**: 各オブジェクトクラスに対する操作（Read, Write, Create 等）をビットマスクとして定義し、ポリシー判定の基盤を提供。
  • **関数インターフェースの公開**: 各ソースファイルが提供する機能（ラベル管理、AVC キャッシュ操作、DML 権限チェック等）の宣言。

重要なシンボル

  • SEPGSQL_LABEL_TAG: セキュリティラベルを識別するためのタグ名。
  • SEPG_CLASS_DB_*: データベースオブジェクト（Table, Schema, Procedure 等）を SE-Linux のクラスとしてマッピングするための識別子。
  • SEPG_*__*: 各クラスに対する具体的な操作権限（例: SEPG_DB_TABLE__SELECT）。
  • sepgsql_avc_check_perms: AVC（Access Vector Cache）を使用して権限チェックを行う主要関数。

制御フローとデータフロー

本ファイル自体は宣言のみを行うため、直接的な制御フローは持ちません。しかし、ここでの定義は以下のフローを支えます： 1. **初期化**: sepgsql_avc_init 等によりキャッシュを準備。 2. **判定**: sepgsql_avc_check_perms が呼び出される際、本ファイルで定義された tclass（オブジェクトクラス）とビットマスク（アクセスベクタ）が引数として渡され、SE-Linux のポリシー決定エンジンへ送られます。 3. **監査**: 判定結果に基づき、sepgsql_audit_log が定義に従ってログを出力します。

直接依存

  • catalog/objectaddress.h: ObjectAddress 型の定義に使用。
  • fmgr.h: PostgreSQL の関数マネージャ関連。
  • <selinux/selinux.h> / <selinux/avc.h>: SE-Linux のカーネル/ライブラリレベルの API 定義。
  • selinux/avc.h
  • selinux/selinux.h

関連コード

  • selinux.c: SE-Linux との低レベルな通信およびポリシー計算の実装。
  • uavc.c: 権限チェックのキャッシュ（AVC）管理。
  • hooks.c: PostgreSQL のフックポイントを介したセキュリティチェックのトリガー。

読者向けメモ

  • **ビットマスクの注意**: SEPG_DIR__* や SEPG_DB_TABLE__* などの定義は、SE-Linux のポリシーファイルと整合している必要があります。これらを変更する場合は、対応するポリシー側の修正も不可欠です。
  • **読み進め方**: まず SEPG_CLASS_DB_* の定義を確認し、PostgreSQL のオブジェクトがどのように SE-Linux の世界にマッピングされているかを理解してから、uavc.c や hooks.c の実装を読むことを推奨します。
