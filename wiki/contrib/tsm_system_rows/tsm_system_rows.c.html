contrib/tsm_system_rows/tsm_system_rows.c

概要

SYSTEM_ROWS テーブルサンプルメソッドを実装するソースファイルです。指定された行数に達するまで、リレーション内のブロックをランダムな順序で走査し、各ブロック内の全タプルを読み取ることでサンプリングを行います。

モジュール内での役割

PostgreSQL の TABLESAMPLE 構文を拡張するプラグインの一部です。TsmRoutine 構造体を通じて、クエリ実行エンジンに対してサンプリングのロジックを公開します。

責務

  • **サンプリング戦略の定義**: リレーション全体を網羅的に走査するためのブロック選択アルゴリズム（線形合同法的なスキップ走査）の実装。
  • **プランナへの統計提供**: SYSTEM_ROWS を使用した際のページ数とタプル数の見積もり。
  • **実行時の状態管理**: SystemRowsSamplerData を用いた、スキャン中の現在位置や進捗状況の保持。

重要なシンボル

  • tsm_system_rows_handler: エントリポイント。TsmRoutine を構築し、各コールバック関数を登録します。
  • SystemRowsSamplerData: スキャン状態を保持する構造体。step（スキップ幅）や firstblock（開始位置）を保持し、再スキャン時の一貫性を保証します。
  • system_rows_nextsampleblock: 次に読み込むブロックを決定する関数。random_relative_prime を用いて決定したステップ幅でブロックを巡回します。

制御フローとデータフロー

1. **プランニング**: system_rows_samplescangetsamplesize が呼び出され、指定された行数から必要なページ数を見積もります。 2. **初期化**: system_rows_initsamplescan で状態構造体を確保し、system_rows_beginsamplescan でパラメータを検証します。 3. **ブロック選択**: system_rows_nextsampleblock が初回実行時に乱数で開始位置とステップ幅を決定し、以降はそれに基づいて次のブロックを返します。 4. **タプル取得**: system_rows_nextsampletuple が現在のブロック内のタプルを順次返し、ブロック内の全タプルを読み終えると次のブロックへ遷移します。

読者向けメモ

  • **再スキャンの一貫性**: このメソッドはクエリ内での再スキャンに対しては一貫性を保ちますが、クエリを跨ぐと nblocks の変化により結果が変わる可能性があります。
  • **アルゴリズムの特性**: random_relative_prime を用いたスキップ走査は、リレーション内の全ブロックを重複なく、かつランダムな順序で訪問するために設計されています。
  • **可視性チェック**: node->use_pagemode = true を強制しており、ページ単位の可視性チェックが必須となっています。
