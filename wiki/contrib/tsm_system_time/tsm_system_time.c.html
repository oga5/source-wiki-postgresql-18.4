contrib/tsm_system_time/tsm_system_time.c

概要

tsm_system_time.c は、PostgreSQL の TABLESAMPLE SYSTEM_TIME メソッドを実装するソースファイルです。このメソッドは、指定された時間制限内で可能な限り多くの行をランダムにサンプリングするために使用されます。

モジュール内での役割

本ファイルは tsm_system_time モジュールの唯一のソースであり、PostgreSQL のテーブルサンプリング API (tsmapi.h) に準拠したハンドラを提供します。クエリ実行時にテーブルスキャンを制御するプラグインとして機能します。

責務

  • **サンプリング戦略の提供**: 指定された時間内に収まるよう、リレーション全体を網羅するブロックサンプリングアルゴリズムを実装する。
  • **コスト見積もり**: プランナに対して、時間制限に基づいたスキャン対象ページ数とタプル数の概算を提示する。
  • **スキャン状態の管理**: SystemTimeSamplerData を用いて、スキャンの進捗、時間経過、およびブロック選択のステップサイズを保持する。

重要なシンボル

  • tsm_system_time_handler: TABLESAMPLE メソッドの登録用エントリポイント。TsmRoutine 構造体を初期化し、各コールバック関数を割り当てる。
  • SystemTimeSamplerData: サンプリングの実行状態を保持する構造体。
  • system_time_nextsampleblock: 次に読み込むブロックを決定する。線形合同法的なアプローチでブロックを巡回する。
  • system_time_nextsampletuple: 現在のブロック内のタプルを順次返す。

制御フローとデータフロー

1. **初期化**: tsm_system_time_handler が呼ばれ、メソッドのメタデータが設定される。 2. **プランニング**: system_time_samplescangetsamplesize が実行され、指定時間から読み取り可能なページ数が推定される。 3. **実行開始**: system_time_beginsamplescan でシード値と時間制限がセットされる。 4. **スキャンループ**:

  • system_time_nextsampleblock が呼び出される。初回のみランダムな開始位置とステップサイズを計算し、以降は時間経過を監視しながら次のブロックを選択する。
  • system_time_nextsampletuple がブロック内の全タプルを順次返す。

    • 5. **終了**: 時間制限に達するか、全ブロックを走査し終えると InvalidBlockNumber を返しスキャンを終了する。

直接依存

  • access/tsmapi.h: テーブルサンプリング API の定義。
  • utils/sampling.h: サンプリング用の乱数生成ユーティリティ。
  • optimizer/optimizer.h: プランナによるコスト見積もり用関数。
  • postgres.h, math.h, miscadmin.h, catalog/pg_type.h, utils/spccache.h: 基本的な型定義、数学関数、およびシステム管理用ヘッダ。

読者向けメモ

  • **非決定性**: このメソッドは時間依存であるため、同じクエリでも実行ごとに結果が異なる（repeatable_across_queries = false）。
  • **コスト見積もりの精度**: system_time_samplescangetsamplesize 内のコスト計算は、ページコストをミリ秒に換算するというヒューリスティックな手法をとっており、あくまで概算である点に注意してください。
  • **線形探索**: ブロック選択には、リレーションの全ブロックを網羅するために、nblocks と互いに素な step を用いた線形探索アルゴリズムが採用されています。
