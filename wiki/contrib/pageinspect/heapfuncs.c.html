contrib/pageinspect/heapfuncs.c

概要

heapfuncs.c は、PostgreSQL のヒープページ（テーブルデータが格納されるページ）の内部構造を低レベルで調査するための SQL 関数群を提供します。主にデバッグや破損調査を目的としており、スーパーユーザーのみが実行可能です。

モジュール内での役割

contrib/pageinspect モジュールの一部として、ヒープページ特有のデータ構造（行ポインタ、タプルヘッダ、ビットマップ、フラグ）を解析し、SQL レベルで可視化する役割を担います。

責務

  • **ページ解析**: heap_page_items を通じて、指定されたページ内の各行ポインタとタプルヘッダ情報を抽出する。
  • **データ復元**: tuple_data_split を通じて、生のタプルデータから属性値を分離・復元する。
  • **フラグ解釈**: heap_tuple_infomask_flags を通じて、タプルの t_infomask および t_infomask2 を人間が読める形式に変換する。
  • **安全性**: 破損したページを読み込む際にもクラッシュしないよう、境界チェックとバリデーションを行う。

重要なシンボル

  • heap_page_items: ページ内の全アイテムを走査するセットリターン関数（SRF）。
  • tuple_data_split: タプルのバイナリデータから属性値を配列として抽出する関数。
  • heap_tuple_infomask_flags: タプルの状態フラグをデコードする関数。
  • heap_page_items_state: heap_page_items の SRF 呼び出し間で状態を保持する構造体。

制御フローとデータフロー

1. **ページ走査**: heap_page_items は SRF_IS_FIRSTCALL でページヘッダを検証し、PageGetMaxOffsetNumber で走査範囲を決定します。 2. **アイテム抽出**: 各呼び出しで PageGetItemId を使用して行ポインタを取得し、PageGetItem でタプルヘッダを読み取ります。 3. **バリデーション**: lp_offset や t_hoff がページ境界内に収まっているかを確認し、不正な場合は NULL を返してクラッシュを回避します。 4. **データ変換**: tuple_data_split は、カタログ情報（TupleDesc）を参照し、t_bits（NULL ビットマップ）を考慮しながら、オフセット計算によって各属性の値を切り出します。

読者向けメモ

  • **セキュリティ**: 本ファイル内の関数はすべて superuser() チェックを行っています。これは、破損したページを読み込む際にメモリ破壊や情報漏洩のリスクがあるためです。
  • **破損への耐性**: ページが破損している場合でも、可能な限り情報を表示する設計になっています。そのため、返される値が「意味をなさない（nonsense）」可能性があることを前提に利用してください。
  • **古いデータ**: HeapTupleHeaderGetOidOld が示す通り、古いバージョンの PostgreSQL からアップグレードされたデータに含まれる OID の表示にも対応しています。
