contrib/pageinspect/ginfuncs.c

概要

GIN（Generalized Inverted Index）インデックスの物理ページ内容を調査するための SQL 関数群を提供します。低レベルなページ構造を解析し、メタデータやフラグ、リーフページ内のポストリストを可視化します。

モジュール内での役割

contrib/pageinspect モジュールの一部として、GIN インデックスの内部構造をデバッグ・解析するためのインターフェースを提供します。他の *funcs.c ファイルと同様、SQL レベルから PostgreSQL の内部データ構造へアクセスするブリッジとして機能します。

責務

  • **メタデータ解析**: GIN メタページからインデックスの統計情報やバージョンを抽出する。
  • **フラグ解析**: ページヘッダーの不透明領域（Opaque）から、GIN 固有のフラグを人間が読み取れる形式に変換する。
  • **リーフページ走査**: 圧縮されたポストリスト（Posting List）をデコードし、インデックスに含まれるアイテムポインタを列挙する。

重要なシンボル

  • gin_metapage_info: メタページの内容を構造体として返す関数。
  • gin_page_opaque_info: ページフラグと rightlink 等の情報を返す関数。
  • gin_leafpage_items: SRF（Set Returning Function）として、リーフページ内の全ポストリストを順次返す関数。
  • gin_leafpage_items_state: gin_leafpage_items の呼び出し間で状態を保持するためのコンテキスト構造体。

制御フローとデータフロー

1. **入力**: 全関数とも bytea 型の raw ページデータを受け取り、get_page_from_raw() でメモリ上の Page 構造体に変換します。 2. **検証**: PageGetSpecialSize() とフラグチェックにより、対象ページが期待される GIN ページタイプ（メタ、データ等）であるかを確認します。 3. **抽出**:

  • メタページや Opaque 情報は、GinPageGetMeta() や GinPageGetOpaque() マクロを使用して直接メモリから読み取ります。
  • gin_leafpage_items は SRF_IS_FIRSTCALL() で初期化を行い、GinDataLeafPageGetPostingList() を起点にセグメントを走査します。

    • 4. **出力**: heap_form_tuple() を用いて PostgreSQL の行形式に変換し、結果を返します。

関連コード

読者向けメモ

  • 本ファイル内の関数はスーパーユーザー権限を要求します。これは raw ページへのアクセスがセキュリティ上のリスクを伴うためです。
  • gin_leafpage_items は圧縮されたポストリストをデコードするため、他の関数よりも複雑な状態管理（FuncCallContext）を行っています。
  • GIN インデックスの構造（メタページ、データページ、ポストリストの圧縮形式）に関する知識があると、コードの意図がより明確に理解できます。
