contrib/pageinspect/gistfuncs.c

概要

GiST（Generalized Search Tree）インデックスの物理ページ内容を調査するための SQL 呼び出し可能関数群を提供します。低レベルなページ構造の解析やデバッグを目的としています。

モジュール内での役割

contrib/pageinspect モジュールの一部として、GiST インデックスに特化した解析機能を提供します。他の *funcs.c ファイルと同様に、生のページデータ（bytea）を引数に取り、人間が読み取り可能な形式や構造化されたデータとして出力します。

責務

  • **ページ検証**: 入力された bytea が有効な GiST ページであるか（gist_page_id や特殊領域サイズ）を検証する。
  • **メタデータ抽出**: gist_page_opaque_info を通じて、GiST ページヘッダーのフラグ、LSN、NSN、右リンクなどの情報を取得する。
  • **アイテム解析**: gist_page_items および gist_page_items_bytea を通じて、ページ内のインデックス項目（タプル）を走査し、その内容をデコードまたはダンプする。

重要なシンボル

  • verify_gist_page: 全ての関数が内部的に呼び出す検証用静的関数。
  • gist_page_opaque_info: ページヘッダーのメタデータを返す SQL 関数。
  • gist_page_items: インデックスの列定義に基づいてタプルをデコードして返す SQL 関数。
  • gist_page_items_bytea: タプルをデコードせず、バイナリデータとしてそのまま返す SQL 関数。

制御フローとデータフロー

1. **検証**: verify_gist_page が PageIsNew をチェックし、GiST 特有の GISTPageOpaqueData を確認します。 2. **走査**: PageGetMaxOffsetNumber を使用してページ内の全アイテムをループ処理します。 3. **変換**:

  • gist_page_items_bytea は IndexTuple をそのまま bytea にコピーして返します。
  • gist_page_items は index_deform_tuple を使用し、インデックスの定義に基づいて各属性値を文字列化して返します。

    • 4. **出力**: tuplestore を介して結果セットを構築し、SQL レイヤーへ返却します。

読者向けメモ

  • **権限**: 本ファイル内の関数は全て superuser() チェックを行っており、一般ユーザーは実行できません。
  • **安全性**: ページが破損している場合や、GiST 以外のページを渡した場合はエラーを発生させるか、検証ロジックで弾く設計になっています。
  • **デコード**: gist_page_items はインデックスの定義（pg_index 等）を参照するため、対象のインデックス OID を正しく指定する必要があります。
