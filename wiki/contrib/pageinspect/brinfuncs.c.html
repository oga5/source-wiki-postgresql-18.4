contrib/pageinspect/brinfuncs.c

概要

brinfuncs.c は、PostgreSQL の BRIN (Block Range Index) インデックスの内部構造を検査するための SQL 関数群を提供します。主にデバッグやインデックスの整合性確認を目的としており、生のページデータからメタデータやインデックス項目を抽出します。

モジュール内での役割

contrib/pageinspect モジュールの一部として、BRIN インデックスに特化した低レベルな解析機能を提供します。他の *funcs.c ファイルと同様に、pageinspect 拡張のインターフェースを通じてユーザーがインデックスの内部状態を直接参照できるようにします。

責務

  • BRIN ページの型（メタ、リビジョンマップ、通常）の識別。
  • BRIN インデックスページ内の各タプル（インデックス項目）のデコードと表示。
  • メタページ情報の抽出と表示。
  • リビジョンマップページに含まれる ItemPointer 配列の抽出。
  • 実行時の権限チェック（スーパーユーザーのみ許可）。

重要なシンボル

  • brin_page_type: ページがどの種類の BRIN ページであるかを判定する関数。
  • brin_page_items: ページ内の全インデックス項目をタプルとして返すセットリターニング関数。
  • brin_metapage_info: メタページの内容（マジックナンバー、バージョン等）を構造体として返す関数。
  • brin_revmap_data: リビジョンマップページ内の TID 配列を列挙する関数。
  • brin_column_state: 各カラムの出力関数情報を保持する内部構造体。

追加の索引済みシンボル

  • ItemPointerData
  • verify_brin_page

制御フローとデータフロー

1. **検証**: 各関数はまず verify_brin_page を呼び出し、入力された bytea が有効な BRIN ページであり、かつ期待される型（Meta, Revmap, Regular）と一致するかを確認します。 2. **デコード**: brin_page_items は brin_deform_tuple を使用して、物理的なページ上のタプルをメモリ上の BrinMemTuple 構造体に変換します。 3. **出力生成**: 変換されたデータは、各カラムの型に応じた出力関数（FmgrInfo）を介してテキスト形式に変換され、tuplestore を通じて SQL 結果セットとして返されます。 4. **コンテキスト管理**: brin_revmap_data のようなセットリターニング関数は、FuncCallContext を使用して複数回の呼び出し間で状態を保持します。

読者向けメモ

  • **権限**: 本ファイルの関数はすべてスーパーユーザー権限を要求します。これは生のページデータへのアクセスがセキュリティ上のリスクを伴うためです。
  • **バージョン依存**: brin_page_items は出力カラム数に厳格です。拡張のバージョンが古い場合、関数定義と実装の不一致によりエラーが発生するため、常に最新の拡張バージョンを使用してください。
  • **デバッグ用途**: 本ファイルはインデックスの破損調査や、BRIN の動作を深く理解するためのツールです。本番環境での運用には適していません。
