contrib/uuid-ossp/uuid-ossp.c

概要

PostgreSQL で UUID を生成するための拡張モジュールです。OSSP UUID、BSD UUID、または E2FS UUID ライブラリのいずれかをラップし、SQL 関数として UUID 生成機能を提供します。

モジュール内での役割

contrib/uuid-ossp ディレクトリの唯一のソースファイルであり、PostgreSQL の型システム（uuid 型）と外部の UUID ライブラリを橋渡しするアダプターとして機能します。

責務

  • **抽象化**: コンパイル時に選択された UUID ライブラリの差異を吸収し、統一された SQL インターフェースを提供します。
  • **UUID 生成**: v1（時間ベース）、v3/v5（名前空間ベース）、v4（乱数ベース）の生成をサポートします。
  • **リソース管理**: OSSP ライブラリ使用時に、セッション単位で UUID オブジェクトをキャッシュし、パフォーマンスと一貫性を維持します。

重要なシンボル

  • uuid_generate_internal: 各種 UUID 生成関数のバックエンドとなる共通関数。
  • get_cached_uuid_t: OSSP 使用時にセッション内で再利用される uuid_t オブジェクトを管理します。
  • dce_uuid_t: BSD/E2FS 環境で UUID の内部構造を扱うための構造体。
  • UUID_MAKE_V1 ～ V5: 生成する UUID のバージョンを指定する定数。

制御フローとデータフロー

1. **初期化**: PG_FUNCTION_INFO_V1 マクロにより、SQL から呼び出し可能な関数として登録されます。 2. **分岐**: コンパイル時の設定（HAVE_UUID_OSSP 等）に基づき、OSSP ライブラリを使うパスと、OS 組み込みの UUID 関数を使うパスに分かれます。 3. **生成**:

  • OSSP の場合: uuid_make を呼び出し、結果を文字列に変換して PostgreSQL の uuid_in 関数へ渡します。
  • 非 OSSP の場合: pg_cryptohash（MD5/SHA1）を使用して名前空間ベースの UUID を計算するか、OS の乱数生成器（arc4random 等）を利用して生成します。

    • 4. **変換**: 最終的に PostgreSQL の Datum 型に変換され、SQL レイヤーへ返されます。

読者向けメモ

  • **ライブラリの差異**: このファイルは configure スクリプトによる環境判定に強く依存しています。ビルドエラーが発生する場合は、システムに適切な UUID 開発ライブラリがインストールされているか確認してください。
  • **OSSP のキャッシュ**: OSSP 使用時の get_cached_uuid_t は、MAC アドレス取得のオーバーヘッド削減と衝突回避のために重要です。このキャッシュを無闇に変更すると、UUID の一意性に影響を与える可能性があります。
  • **v1mc**: uuid_generate_v1mc は、物理 MAC アドレスの代わりにランダムなマルチキャスト MAC アドレスを使用する特殊な v1 UUID 生成です。プライバシー保護やクラスタ環境での衝突回避に利用されます。
