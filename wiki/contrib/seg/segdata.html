contrib/seg/segdata.h

概要

segdata.h は、PostgreSQL の contrib/seg モジュールにおけるデータ構造の定義と、字句解析・構文解析器（Flex/Bison）とのインターフェースを定義するヘッダーファイルです。

モジュール内での役割

本モジュールは、数値範囲（Segment）を扱うためのデータ型を提供します。このヘッダーは、seg.c（実装）、segscan.l（字句解析）、segparse.y（構文解析）の各ファイル間で共有される共通の型定義と関数プロトタイプを保持する「契約」の役割を果たします。

責務

  • **データ構造の定義**: 数値範囲を表現する SEG 構造体の定義。
  • **解析器の統合**: Flex/Bison で生成された解析器と、PostgreSQL のバックエンドコードを接続するための型定義（yyscan_t）および関数宣言の提供。

重要なシンボル

  • struct SEG: 数値範囲の境界値（lower/upper）と、それぞれの有効桁数や拡張フラグを保持する主要なデータ構造。
  • yyscan_t: Flex が生成するスキャナ状態を保持するための不透明ポインタ。
  • seg_yylex / seg_yyparse: 解析器のメインエントリポイント。

追加の索引済みシンボル

  • SEG

制御フローとデータフロー

本ファイル自体は宣言のみですが、以下のフローを支えています。 1. seg_scanner_init でスキャナを初期化し、入力文字列を渡す。 2. seg_yyparse が呼び出され、内部で seg_yylex を介してトークンを消費する。 3. 解析結果が SEG 構造体に書き込まれ、呼び出し元へ返される。

直接依存

本ファイルは標準的な C の型定義を除き、他のヘッダーファイルへの直接的な #include を含んでいません。これは、依存関係の循環を避けるための設計上の配慮です。

関連コード

  • seg.c: SEG 構造体を利用した演算や、significant_digits などの補助関数の実装。
  • segscan.l: 字句解析ルール。seg_yylex を生成。
  • segparse.y: 構文解析ルール。seg_yyparse を生成。

読者向けメモ

  • **構造体のパディング**: SEG 構造体は float4 と char が混在しています。アライメントを意識したメモリレイアウトになっている点に注意してください。
  • **解析器の再入可能性**: yyscan_t を使用しているため、この解析器はスレッドセーフ（または再入可能）な設計になっています。PostgreSQL のバックエンドで解析を行う際は、このコンテキストを適切に管理する必要があります。
  • **escontext**: seg_yyerror や seg_yyparse に渡される struct Node *escontext は、エラー報告のコンテキストとして使用されます。PostgreSQL のエラーハンドリング機構と統合されています。
