contrib/ltree/_ltree_op.c

概要

本ファイルは、PostgreSQL の ltree 拡張において、ltree 型の配列（ltree[]）に対する演算子関数を定義するモジュールです。

モジュール内での役割

ltree モジュールにおける「配列演算層」を担います。単一の ltree に対する演算（ltree_op.c 等で定義）を、配列全体に対して適用するためのラッパーおよびイテレータを提供します。

責務

  • **配列イテレーション**: array_iterator 関数を通じ、配列内の各要素に対して指定された演算を適用し、いずれか一つでも条件を満たすか（ANY 検索）を判定する。
  • **演算子の提供**: ltree 配列と ltree、lquery、ltxtquery との間の包含判定やマッチングを行う SQL 呼び出し可能関数（_ltree_isparent 等）の実装。
  • **抽出機能**: 配列内で条件に合致した最初の要素を特定し、そのコピーを返す関数の提供。
  • **共通祖先探索**: 配列内の全 ltree 要素に対する最小共通祖先（LCA）の計算。

重要なシンボル

  • array_iterator: 配列を走査し、コールバック関数を用いて条件判定を行う汎用イテレータ。
  • NEXTVAL(x): ltree 配列内の次の要素へポインタを進めるためのアライメント調整マクロ。
  • _ltree_isparent / _ltq_regex / _ltxtq_exec: 配列に対する主要な検索演算子関数。
  • _lca: 配列内の全要素の最小共通祖先を求める関数。

追加の索引済みシンボル

  • PG_FUNCTION_ARGS

制御フローとデータフロー

1. **入力**: SQL から ArrayType として渡された ltree[] を受け取る。 2. **走査**: array_iterator が配列のデータポインタを順次取得し、NEXTVAL で要素を移動しながら、各要素に対して DirectFunctionCall2 で単一要素用の演算関数を呼び出す。 3. **判定**: いずれかの要素で真が返れば、即座に true を返し、配列全体の検索結果とする。 4. **抽出**: _extract_ 系関数は、array_iterator が見つけた要素を palloc でメモリ確保し、呼び出し元へコピーを返す。

直接依存

  • postgres.h: PostgreSQL の基本型およびマクロ。
  • ltree.h: ltree 型の構造体定義および演算子関数のプロトタイプ。
  • utils/array.h: PostgreSQL の配列操作 API（ArrayGetNItems, ARR_DATA_PTR 等）。
  • ctype.h

関連コード

  • ltree_op.c: 本ファイルが内部で呼び出す単一要素用の演算ロジック。
  • lquery_op.c: lquery（パターン）とのマッチングロジック。
  • ltxtquery_op.c: ltxtquery（ブールクエリ）の実行ロジック。

読者向けメモ

  • 本ファイル内の関数は、基本的に「配列のいずれか一つが条件を満たすか」という ANY 演算子としての挙動を前提としています。
  • NEXTVAL マクロは、ltree が可変長構造体であるため、INTALIGN を用いてメモリ上の境界を正しく計算する重要な役割を担っています。
  • _r_ が付く関数（例: _ltree_r_isparent）は、引数の順序を入れ替えて呼び出すためのラッパーです。
