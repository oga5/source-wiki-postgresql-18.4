contrib/ltree_plpython/ltree_plpython.c

概要

PostgreSQLの ltree 型を Python のリスト型へ変換するためのブリッジ機能を提供する拡張モジュールです。plpython と ltree の橋渡しを行い、データベース内の階層パスデータを Python スクリプト内で扱いやすい形式に変換します。

モジュール内での役割

ltree 拡張と plpython 拡張の間に位置するアダプターです。ltree の内部構造を直接操作し、Python の C API を介して Python オブジェクトを構築します。

責務

  • **動的リンク**: plpython モジュールが提供する関数ポインタをロードし、実行時に利用可能にする。
  • **型変換**: ltree の各レベル（ラベル）を抽出し、Python の文字列リスト（list）として再構築する。
  • **メモリ管理**: Python オブジェクト生成時のメモリ不足エラーのハンドリング。

重要なシンボル

  • ltree_to_plpython: SQL から呼び出されるメインの変換関数。
  • PLyUnicode_FromStringAndSize_p: plpython モジュールから動的に取得した文字列生成関数のポインタ。
  • _PG_init: モジュールロード時に実行され、外部関数ポインタを解決する初期化関数。

追加の索引済みシンボル

  • Py_ssize_t

制御フローとデータフロー

1. **初期化**: _PG_init が load_external_function を使用して plpython の関数アドレスを取得し、グローバル変数に保持します。 2. **変換開始**: ltree_to_plpython が呼び出されると、入力された ltree 構造体からレベル数を取得し、PyList_New で Python リストを確保します。 3. **反復処理**: ltree の各レベルを走査し、PLyUnicode_FromStringAndSize を用いて各ラベルを Python 文字列に変換してリストに格納します。 4. **終了**: 構築されたリストを Datum として PostgreSQL 側に返却します。

直接依存

  • postgres.h, fmgr.h: PostgreSQL のコア機能および関数管理インターフェース。
  • ltree/ltree.h: ltree 型の定義と操作マクロ。
  • plpy_util.h: plpython 関連のユーティリティ定義。

関連コード

  • contrib/ltree: ltree 型の定義元。
  • src/pl/plpython: plpython の実装本体。本ファイルが依存する PLyUnicode_FromStringAndSize の定義元。

読者向けメモ

  • **動的ロードの仕組み**: このモジュールは plpython に直接リンクせず、実行時に load_external_function で関数ポインタを取得しています。これは、plpython がロードされていない環境でもモジュール自体がロードエラーにならないようにするための設計です。
  • **メモリ管理**: PyList_New 等の Python C API を使用しているため、メモリ確保失敗時の ereport によるエラーハンドリングが重要です。
  • **拡張性**: ltree の構造（ltree_level）を直接走査しているため、ltree の内部表現が変更された場合は本コードも修正が必要です。
