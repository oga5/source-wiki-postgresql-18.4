TestSpec

定義

  • 種別: struct
  • 定義箇所: src/test/isolation/isolationtester.h:75

目的

PostgreSQLの分離レベルテスト（Isolation Tests）におけるテスト仕様全体を保持するルートデータ構造です。.spec ファイルから解析されたテスト定義を格納し、実行エンジンがテストを実行するためのスキーマを提供します。

シグネチャまたは形状

typedef struct
{
	char	  **setupsqls;
	int			nsetupsqls;
	char	   *teardownsql;
	Session   **sessions;
	int			nsessions;
	Permutation **permutations;
	int			npermutations;
} TestSpec;

入力と出力

  • **入力**: specparse.y によって解析されたテスト定義情報が、本構造体の各フィールドに格納されます。
  • **出力**: 実行エンジン（isolationtester.c）が本構造体を走査し、各セッションに対して順次 SQL を発行することでテストを実行します。

前提条件と不変条件

  • check_testspec() 関数が呼び出されるまで、構造体内のポインタ（特に PermutationStep 関連）は未解決の状態である可能性があります。
  • nsetupsqls、nsessions、npermutations は、それぞれ対応する配列の要素数を正確に保持している必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: spec_yyparse() により .spec ファイルが読み込まれ、TestSpec 構造体が構築されます。 2. **検証とリンク**: isolationtester.c 内の check_testspec() が、Step や PermutationStep 内のポインタを解決し、実行可能な状態にリンクを構築します。 3. **実行**: 実行エンジンが TestSpec を走査し、定義された sessions に対して setupsqls を実行後、permutations に基づいて SQL を発行します。最後に teardownsql が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/isolation/isolationtester.c 内の check_testspec、run_testspec、run_all_permutations、run_all_permutations_recurse、run_named_permutations、run_permutation、try_complete_steps、try_complete_step が本構造体を引数として参照しています。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Session: 独立したデータベース接続をシミュレートする単位。
  • Permutation: テストの実行順序を定義する構造体。
  • Step: 実行される個別の SQL コマンド。

失敗モードと境界条件

  • check_testspec() によるリンク解決に失敗した場合、実行エンジンはテストを正しく実行できません。
  • nsessions や npermutations が 0 の場合、テスト実行ロジックが空の処理を行う可能性があります。

保守担当者向けメモ

  • 本構造体は specparse.y と密接に関連しています。テスト定義の構文を変更する場合は、本構造体の定義とパーサーの両方を更新する必要があります。
  • PermutationStep 内のポインタ解決は check_testspec() に依存しているため、実行順序や依存関係のロジックを変更する際は注意してください。
