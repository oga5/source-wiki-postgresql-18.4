BarrierArriveAndDetachExceptLast

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/ipc/barrier.c:212

目的

バリアからプロセスを離脱させ、離脱したプロセスが最後の一人でない場合は false を、最後の一人である場合は true を返す。これにより、バリアの終了処理を特定のプロセス（最後の一人）に委譲する。

シグネチャまたは形状

bool BarrierArriveAndDetachExceptLast(Barrier *barrier);

入力と出力

  • **入力**: barrier (対象となる Barrier 構造体へのポインタ)
  • **出力**: bool
  • true: 呼び出し元がバリアの最後の参加者であった場合。
  • false: 呼び出し元以外にまだ参加者が残っている場合。

前提条件と不変条件

  • barrier は初期化済みであること。
  • 内部の mutex（スピンロック）により、参加者数の操作とフェーズ更新はアトミックに保護される。
  • 呼び出し時点で barrier->participants >= 1 であること。

アルゴリズムまたは振る舞い

1. barrier->mutex を取得する。 2. barrier->participants が 1 より大きい場合：

  • participants をデクリメントする。
  • mutex を解放し、false を返す。

    • 3. barrier->participants が 1 の場合：

  • barrier->phase をインクリメントする。
  • mutex を解放し、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHash.c
  • src/backend/executor/nodeHashjoin.c
  • **呼び出し先**:
  • SpinLockAcquire
  • SpinLockRelease

関連データ構造

  • Barrier: 同期状態（participants、phase、mutex）を保持する構造体。

失敗モードと境界条件

  • participants が 1 の状態で呼び出された場合、フェーズが更新され true が返される。
  • 呼び出し元が participants の整合性を維持していない場合（0以下の状態で呼び出すなど）、アサーションエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、バリアの終了処理をシリアルに行う必要があるケース（例：ハッシュ結合のバッチ処理終了時）で利用される。
  • 呼び出し元は、戻り値が true の場合にのみ、バリアに関連するリソースの解放や後処理を行う必要がある。
