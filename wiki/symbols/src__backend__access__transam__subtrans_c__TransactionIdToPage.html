TransactionIdToPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:60

目的

指定されたトランザクション ID (XID) が格納されている SLRU (Simple Least Recently Used) ページ番号を算出します。

シグネチャまたは形状

static inline int64 TransactionIdToPage(TransactionId xid)

入力と出力

  • **入力**: xid (TransactionId) - 対象のトランザクション ID。
  • **出力**: int64 - 対応する SLRU ページ番号。

前提条件と不変条件

  • SUBTRANS_XACTS_PER_PAGE 定数が定義されている必要があります。
  • xid は有効なトランザクション ID であると想定されます。

アルゴリズムまたは振る舞い

  • 入力された xid を SUBTRANS_XACTS_PER_PAGE で除算し、ページ番号を計算します。
  • int64 へのキャストを行い、ページ番号のラップアラウンドやオーバーフローを考慮した計算を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/clog.c 内の複数の箇所（clog.c:82, 186, 198, 236, 260, 273, 404, 418, 669, 737, 880, 895）から呼び出されています。
  • **呼び出し先**: 算術演算のみを行うため、他の関数呼び出しはありません。

関連データ構造

  • TransactionId: トランザクションを識別する 32 ビット整数。
  • SUBTRANS_XACTS_PER_PAGE: 1 ページあたりのトランザクション保持数。

失敗モードと境界条件

  • xid が 0 の場合、ページ番号 0 が返されます。
  • 非常に大きな xid を指定した場合、int64 の範囲内でページ番号が算出されます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁に呼び出されるためパフォーマンスに影響します。変更時は計算コストに注意してください。
  • clog.c からの呼び出しが多数確認されていますが、これは subtrans.c の実装が SLRU 構造を共有または類似のロジックで管理しているためです。
