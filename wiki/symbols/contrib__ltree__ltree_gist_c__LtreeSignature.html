LtreeSignature

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltree_gist.c:561

目的

GiST インデックスにおいて、ltree データの包含関係や一致判定を高速化するためのビットマップシグネチャを保持する構造体です。

シグネチャまたは形状

typedef struct LtreeSignature
{
	BITVECP		sign;
	int			siglen;
} LtreeSignature;

入力と出力

  • sign: シグネチャデータを格納するビットベクトルのポインタ。
  • siglen: シグネチャの長さ（バイト単位）。

前提条件と不変条件

  • siglen は ltree_gist_options によって設定される値と整合している必要があります。
  • sign が指すメモリ領域は、シグネチャの操作に十分なサイズが確保されている必要があります。

アルゴリズムまたは振る舞い

GiST インデックスの検索時、この構造体に含まれるビットマップ（シグネチャ）を「ふるい」として使用します。クエリ条件とインデックスノードのシグネチャを照合することで、不要なサブツリーの探索をスキップし、検索効率を向上させます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/ltree/ltree_gist.c および contrib/ltree/_ltree_gist.c 内の関数で、シグネチャの初期化や参照に使用されています。
  • 呼び出し先: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • BITVECP: シグネチャのビットマップを表現するための型。
  • ltree_gist: ltree データを GiST インデックス用に変換した構造体（本構造体はこの一部として利用される）。

失敗モードと境界条件

  • siglen が不適切な値である場合、ビットマップ操作時にメモリ破壊や誤った検索結果を招く可能性があります。

保守担当者向けメモ

  • 本構造体は ltree_gist.c と _ltree_gist.c の両方で参照されています。シグネチャの計算ロジックを変更する場合は、両ファイルの整合性に注意してください。
  • シグネチャの生成には crc32.h で定義されるハッシュ計算が利用されます。
