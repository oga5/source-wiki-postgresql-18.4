check_stack_depth

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/stack_depth.c:94

目的

再帰的な関数呼び出し等によるスタックオーバーフローを未然に防ぐため、現在のスタック使用量が設定上限を超えていないかを検証し、超過している場合にエラーを発生させる。

シグネチャまたは形状

void check_stack_depth(void)

入力と出力

  • 入力: なし（内部的にグローバル変数 max_stack_depth およびスタック基準点を参照）
  • 出力: なし（正常時は復帰、超過時は ERROR をスローして処理を中断）

前提条件と不変条件

  • プロセス起動時に set_stack_base() が呼び出され、スタックの基準位置が初期化されていること。
  • max_stack_depth が適切な値で設定されていること。

アルゴリズムまたは振る舞い

1. stack_is_too_deep() を呼び出し、現在のスタックポインタと基準点の差分が max_stack_depth を超えているか判定する。 2. 超過している場合、ereport(ERROR, ...) を実行する。 3. エラーメッセージには、現在の max_stack_depth の値を含め、ユーザーに対して設定値の引き上げやプラットフォーム制限の確認を促すヒントを表示する。

呼び出し元と呼び出し先

  • 呼び出し先: stack_is_too_deep()
  • 呼び出し元（直接的な証拠に基づく）:
  • contrib/intarray/_int_bool.c
  • contrib/intarray/_int_selfuncs.c
  • contrib/ltree/lquery_op.c
  • contrib/ltree/ltxtquery_io.c
  • contrib/ltree/ltxtquery_op.c
  • src/backend/access/gist/gist.c

関連データ構造

  • max_stack_depth: スタック上限を保持するグローバル変数。

失敗モードと境界条件

  • スタックが物理的な制限（OSの RLIMIT_STACK）に達する前に、PostgreSQL 側の論理的な制限として ERROR を発生させる。
  • 境界条件として、スタックの伸長方向（増加または減少）に関わらず、差分の絶対値を用いて判定を行う。

保守担当者向けメモ

  • 本関数は頻繁に呼び出される可能性があるため、判定ロジックは最小限のポインタ演算に留めること。
  • ereport のエラーコードとして ERRCODE_STATEMENT_TOO_COMPLEX を使用している。
