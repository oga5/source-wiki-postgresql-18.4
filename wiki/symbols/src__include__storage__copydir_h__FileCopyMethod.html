FileCopyMethod

定義

  • 種別: enum
  • 定義箇所: src/include/storage/copydir.h:16

目的

PostgreSQL におけるディレクトリやファイルのコピー操作において、使用するコピー方式を識別・指定するために使用されます。

シグネチャまたは形状

typedef enum FileCopyMethod
{
	FILE_COPY_METHOD_COPY,
	FILE_COPY_METHOD_CLONE,
} FileCopyMethod;

入力と出力

  • 入力: なし（列挙型の定義）
  • 出力: FILE_COPY_METHOD_COPY（標準的なファイルコピー）または FILE_COPY_METHOD_CLONE（ファイルシステムレベルのクローン/Reflink）のいずれかの値。

前提条件と不変条件

  • コピー処理の実行中、選択された方式は一貫している必要があります。
  • file_copy_method GUC 変数によって制御されます。

アルゴリズムまたは振る舞い

  • FILE_COPY_METHOD_COPY: データを読み取り、新しいファイルへ書き出す標準的なコピー処理を選択します。
  • FILE_COPY_METHOD_CLONE: 可能であればファイルシステムレベルのクローン機能（Reflink 等）を利用し、物理的なデータコピーを省略または最適化します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース excerpt には具体的な呼び出し元は記載されていません。
  • 呼び出し先: なし（列挙型定義のため）。

関連データ構造

  • file_copy_method: 本列挙型を利用して現在のコピー方式を保持する GUC 変数。

失敗モードと境界条件

  • 記載なし。

保守担当者向けメモ

  • 本列挙型は copydir() 関数による再帰的なディレクトリコピー処理の挙動を決定します。
  • 実装を変更する際は、PostgreSQL のデータ整合性確保のための fsync フローが、選択されたコピー方式に関わらず適切に維持されることを確認してください。
