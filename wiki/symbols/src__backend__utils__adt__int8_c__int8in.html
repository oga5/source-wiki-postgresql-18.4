int8in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int8.c:49

目的

SQLの bigint 型（64ビット整数）の入力表現（文字列）を、PostgreSQL内部の int64 型へ変換します。

シグネチャまたは形状

Datum int8in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される cstring 型の文字列。
  • **出力**: int64 型の Datum 値。

前提条件と不変条件

  • 入力文字列は有効な整数形式である必要があります。
  • 変換処理は pg_strtoint64_safe を介して行われ、エラー発生時には適切なエラーコンテキストが使用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) を使用して、入力引数から文字列ポインタを取得します。 2. pg_strtoint64_safe 関数を呼び出し、文字列を64ビット整数に変換します。この際、fcinfo->context をエラー報告用コンテキストとして渡します。 3. 変換結果を PG_RETURN_INT64 マクロを通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/define.c:191
  • src/backend/utils/adt/jsonpath_exec.c:1296
  • **呼び出し先**:
  • pg_strtoint64_safe (文字列から64ビット整数への変換関数)

関連データ構造

  • Datum: PostgreSQLの汎用データ表現型。
  • PG_FUNCTION_ARGS: 関数呼び出し時の引数コンテキスト。

失敗モードと境界条件

  • 入力文字列が整数として解釈できない場合、または64ビット整数の範囲（INT64_MIN ～ INT64_MAX）を超える場合は、pg_strtoint64_safe の仕様に基づきエラーが発生します。

保守担当者向けメモ

  • 本関数は int8 型の入力変換の入り口です。バイナリ形式の入力には int8recv が別途存在します。
  • 変換ロジックの詳細は pg_strtoint64_safe に依存しているため、オーバーフロー等の挙動変更が必要な場合はそちらを確認してください。
