PQsslAttribute

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-openssl.c:1670

目的

アクティブなSSL接続に関連する属性情報を取得する。

シグネチャまたは形状

const char *PQsslAttribute(PGconn *conn, const char *attribute_name);

入力と出力

  • **入力**:
  • conn: PGconn 構造体へのポインタ。
  • attribute_name: 取得したい属性名（"library", "key_bits", "cipher", "compression", "protocol", "alpn"）。
  • **出力**:
  • 成功時: 属性値を示す文字列（静的バッファまたはOpenSSL内部文字列へのポインタ）。
  • 失敗時または属性不明時: NULL。

前提条件と不変条件

  • conn が NULL の場合、attribute_name が "library" であれば "OpenSSL" を返す。
  • conn->ssl が NULL（非暗号化接続）の場合、すべての属性に対して NULL を返す。

アルゴリズムまたは振る舞い

1. conn が NULL かつ attribute_name が "library" なら "OpenSSL" を返す。 2. conn->ssl が NULL なら NULL を返す。 3. attribute_name に応じて以下のOpenSSL関数を呼び出し、結果を返す：

  • "library": "OpenSSL"
  • "key_bits": SSL_get_cipher_bits を使用し、数値を文字列化して返す。
  • "cipher": SSL_get_cipher を使用。
  • "compression": SSL_get_current_compression の結果に基づき "on" または "off" を返す。
  • "protocol": SSL_get_version を使用。
  • "alpn": SSL_get0_alpn_selected を使用し、取得したデータを文字列として返す。

    • 4. 一致する属性がない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c にて、接続情報の表示や確認のために複数回呼び出されている。
  • **呼び出し先**: OpenSSLライブラリの関数（SSL_get_cipher_bits, SSL_get_cipher, SSL_get_current_compression, SSL_get_version, SSL_get0_alpn_selected）。

関連データ構造

  • PGconn: libpqの接続状態を保持する構造体。conn->ssl を通じてOpenSSLの SSL オブジェクトを参照する。

失敗モードと境界条件

  • attribute_name が未定義の文字列である場合、NULL を返す。
  • "alpn" 属性において、データ長が255バイトを超える場合やデータが取得できない場合は空文字列 "" を返す。
  • "key_bits" および "alpn" は静的バッファを使用するため、呼び出しごとに内容が上書きされる可能性がある。

保守担当者向けメモ

  • 本関数は libpq のセキュア通信層における公開インターフェースの一部である。
  • 新しいSSL属性を追加する場合は、conn->ssl の有効性を確認した上で、適切なOpenSSL APIを呼び出すロジックを追加すること。
