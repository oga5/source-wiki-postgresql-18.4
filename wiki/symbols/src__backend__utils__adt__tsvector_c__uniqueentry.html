uniqueentry

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector.c:102

目的

入力された WordEntryIN 配列のトークンをソートし、同一語彙（lexeme）をマージして、tsvector の内部バイナリレイアウトを構築するための準備を行う。

シグネチャまたは形状

static int uniqueentry(WordEntryIN *a, int l, char *buf, int *outbuflen)

入力と出力

  • **入力**:
  • a: WordEntryIN 構造体の配列（トークンと位置情報のリスト）。
  • l: 配列 a の要素数。
  • buf: 語彙文字列が格納されたバッファ。
  • outbuflen: 構築後のデータに必要な合計サイズを格納する出力用ポインタ。
  • **出力**:
  • 戻り値: 重複排除後の有効な要素数。
  • outbuflen: 最終的なバイナリ表現に必要なバイト数。

前提条件と不変条件

  • l >= 1 であること。
  • buf は WordEntryIN 内の pos が指す文字列データを含んでいること。
  • 処理後、語彙はソートされ、同一語彙はマージ済みであること。

アルゴリズムまたは振る舞い

1. **ソート**: l > 1 の場合、compareentry を用いて a を語彙順にソートする。 2. **重複排除とマージ**:

  • 隣接する要素を比較し、語彙が一致する場合は位置情報（pos）を res にマージする。
  • 位置情報のマージには repalloc を使用し、uniquePos で位置情報の重複排除を行う。
  • 語彙が異なる場合は、現在の res のサイズを計算し、次の要素へ進む。

    • 3. **最終計算**: 最後の要素についてもサイズ計算を行い、outbuflen に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsvector.c 内の tsvectorin 関数（行267）。
  • **呼び出し先**:
  • qsort_arg: 配列のソート。
  • compareentry: 語彙の比較。
  • uniquePos: 位置情報の重複排除。
  • repalloc / pfree: メモリ管理。

関連データ構造

  • WordEntryIN: 入力処理用の一時的な構造体。
  • WordEntryPos: 位置情報と重みを保持する構造体。

失敗モードと境界条件

  • l が 1 未満の場合、Assert により停止する。
  • メモリ不足時には repalloc がエラーを発生させる可能性がある。
  • 位置情報が重複している場合、uniquePos が適切に処理を行う必要がある。

保守担当者向けメモ

  • 本関数は tsvector の構築において重要な「正規化」フェーズを担う。
  • repalloc を多用するため、大規模な tsvector を処理する際はメモリ断片化に注意が必要。
  • WordEntryPos のビット操作については tsearch/ts_type.h の定義と整合性を保つこと。
