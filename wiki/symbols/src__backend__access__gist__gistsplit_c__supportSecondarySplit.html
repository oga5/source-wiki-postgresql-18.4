supportSecondarySplit

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistsplit.c:257

目的

GiSTインデックスのマルチカラム分割において、後続の列（attno）に基づき、分割された左右のページ（sv）の整合性を調整し、ユニオンキーを再計算する。

シグネチャまたは形状

static void supportSecondarySplit(Relation r, GISTSTATE *giststate, int attno, GIST_SPLITVEC *sv, Datum oldL, Datum oldR)

入力と出力

  • **入力**:
  • r: 対象のリレーション。
  • giststate: GiSTインデックスの状態管理構造体。
  • attno: 現在処理中の列インデックス。
  • sv: 分割結果を保持する GIST_SPLITVEC 構造体。
  • oldL, oldR: 前段の分割で確定した左および右のユニオンキー。
  • **出力**: なし（sv 構造体の内容を直接更新）。

前提条件と不変条件

  • sv は初期分割が完了している状態であること。
  • oldL および oldR は、この列の分割において基準となる値であること。

アルゴリズムまたは振る舞い

1. **ペナルティ評価**: 左右の既存ユニオンキー（oldL, oldR）と、現在の分割結果（sv->spl_ldatum, sv->spl_rdatum）を用いて、gistpenalty を計算する。 2. **分割の最適化**:

  • 両方のユニオンキーが存在する場合、ペナルティの合計を比較し、より小さい方を選択する。
  • 片方のみ存在する場合は、そのユニオンキーに対するペナルティが最小となる側を選択する。

    • 3. **スワップ処理**: 選択された配置が現在の sv と異なる場合、sv 内の左右のデータ（タプル数、オフセット、ユニオンキー）を入れ替える。 4. **ユニオンキーの更新**: gistMakeUnionKey を呼び出し、現在の列 attno に基づいて左右のユニオンキーを再計算する。 5. **フラグのクリア**: sv->spl_ldatum_exists および sv->spl_rdatum_exists を false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistsplit.c 内の gistSplitByKey から呼び出される。
  • **呼び出し先**: gistpenalty, gistMakeUnionKey, gistentryinit（マクロ）。

関連データ構造

  • GIST_SPLITVEC: 分割結果を格納する構造体。
  • GISTSTATE: GiSTインデックスの操作状態を保持する構造体。
  • GISTENTRY: インデックスエントリの表現。

失敗モードと境界条件

  • sv->spl_ldatum_exists と sv->spl_rdatum_exists の両方が偽である場合、ユニオンキーの再計算はスキップされる。
  • ユーザー定義のペナルティ関数が不正確な場合、最適化が不完全になる可能性があるが、ユニオンキーの再計算によりインデックスの整合性は維持される。

保守担当者向けメモ

  • 本関数は、マルチカラムGiSTインデックスにおいて、最初の列で決まった分割を後続の列で洗練させるために不可欠である。
  • SWAPVAR マクロによる入れ替え処理は、sv 構造体のポインタや値を安全に交換するために使用されている。
