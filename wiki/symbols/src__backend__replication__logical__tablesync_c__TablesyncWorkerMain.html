TablesyncWorkerMain

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/tablesync.c:1746

目的

論理レプリケーションにおけるテーブル同期ワーカー（Tablesync Worker）のメインエントリポイントとして機能し、ワーカーの初期化から同期処理の実行、および終了処理までを統括します。

シグネチャまたは形状

void TablesyncWorkerMain(Datum main_arg)

入力と出力

  • **入力**: main_arg (Datum型) - ワーカーの識別に使用されるスロット番号（worker_slot）。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数はバックグラウンドワーカーとして起動されることを前提としています。
  • main_arg から取得した worker_slot が有効なワーカー状態を指している必要があります。

アルゴリズムまたは振る舞い

1. SetupApplyOrSyncWorker(worker_slot) を呼び出し、ワーカーの実行環境を初期化します。 2. run_tablesync_worker() を呼び出し、パブリッシャーからのデータコピーおよびキャッチアップ処理を実行します。 3. finish_sync_worker() を呼び出し、同期完了後の後処理（状態の更新やリソースの解放）を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/bgworker.c:134 (直接の起動元)
  • src/backend/replication/logical/launcher.c:496 (ワーカー起動時の関数名指定)
  • **呼び出し先**:
  • SetupApplyOrSyncWorker
  • run_tablesync_worker
  • finish_sync_worker

関連データ構造

  • Datum (引数として使用)
  • ワーカーの状態管理に関連する内部構造体（SetupApplyOrSyncWorker 等で参照）

失敗モードと境界条件

  • ワーカーの初期化失敗や、同期プロセス中のネットワークエラー、データコピーの失敗が発生した場合、run_tablesync_worker 内でエラーがスローされ、ワーカープロセスが終了します。

保守担当者向けメモ

  • 本関数は同期ワーカーのライフサイクルを定義する最上位の関数です。同期ロジックの変更は主に run_tablesync_worker 内で行われます。
  • 呼び出し元情報は bgworker および launcher からの参照に基づいています。
