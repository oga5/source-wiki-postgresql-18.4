InitLatch

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/latch.c:62

目的

プロセスローカルなラッチ構造体を初期化し、待機および通知処理が可能な状態に設定します。

シグネチャまたは形状

void InitLatch(Latch *latch)

入力と出力

  • **入力**: latch (初期化対象となる Latch 構造体へのポインタ)
  • **出力**: なし（副作用として latch の各フィールドを初期化）

前提条件と不変条件

  • latch は有効なメモリ領域を指している必要があります。
  • 本関数で初期化されたラッチは、プロセスローカルな用途（is_shared = false）として設定されます。

アルゴリズムまたは振る舞い

1. is_set を false に設定します。 2. maybe_sleeping を false に設定します。 3. owner_pid を現在のプロセス ID (MyProcPid) に設定します。 4. is_shared を false に設定します。 5. **WIN32環境の場合**: CreateEvent を呼び出し、Windows イベントオブジェクトを作成して latch->event に格納します。作成に失敗した場合は elog(ERROR) で終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/init/miscinit.c: InitLatch(MyLatch);
  • その他、プロセスローカルなラッチを生成する箇所（ソースコード上の言及あり）。
  • **呼び出し先**:
  • CreateEvent (WIN32環境のみ)
  • elog (エラー発生時)

関連データ構造

  • Latch: ラッチの状態を保持する構造体。is_set, maybe_sleeping, owner_pid, is_shared を含みます。

失敗モードと境界条件

  • **WIN32環境**: CreateEvent が失敗した場合、elog(ERROR) によりプロセスが異常終了します。

保守担当者向けメモ

  • 本関数はプロセスローカルなラッチ専用です。共有メモリ上のラッチを初期化する場合は InitSharedLatch を使用してください。
  • is_shared が false に固定されるため、このラッチは初期化したプロセス自身からのみセット可能です。
