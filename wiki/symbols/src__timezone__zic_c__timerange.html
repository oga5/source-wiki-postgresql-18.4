timerange

定義

  • 種別: struct
  • 定義箇所: src/timezone/zic.c:2045

目的

タイムゾーンデータにおける特定の期間（遷移時刻の範囲）および関連する閏秒の情報を保持するために使用されます。

シグネチャまたは形状

struct timerange
{
	int			defaulttype;
	ptrdiff_t	base,
				count;
	int			leapbase,
				leapcount;
};

入力と出力

  • 本構造体はデータ保持用であり、関数ではないため入出力はありません。
  • メンバ変数は、遷移時刻のインデックス範囲（base, count）および閏秒のインデックス範囲（leapbase, leapcount）を定義します。

前提条件と不変条件

  • base および count は、遷移時刻配列の有効な範囲を指す必要があります。
  • leapbase および leapcount は、閏秒配列の有効な範囲を指す必要があります。

アルゴリズムまたは振る舞い

  • limitrange() 関数において、指定された時間範囲（lo から hi）に基づいて遷移時刻と閏秒の範囲を絞り込む際に使用されます。
  • rangeall 変数として、全期間を対象とする範囲の保持にも利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: limitrange() 関数が引数として受け取ります。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。
  • 補足: timerange_option() 関数内で timerange という名前の引数が存在しますが、これは struct timerange 型ではなく char * 型であり、直接の関連性は不明です。

関連データ構造

  • zic_t: 遷移時刻の計算に使用される型。
  • struct zone や struct rule: zic.c 内でタイムゾーン定義を構成する他の主要構造体。

失敗モードと境界条件

  • base や count に不適切な値が設定された場合、TZif バイナリ生成時にメモリ破壊や不正なファイル出力が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は zic.c 内でのみ使用される内部的な管理用データ構造です。
  • タイムゾーンの遷移時刻計算ロジックを変更する際は、base と count の整合性が保たれているか確認してください。
