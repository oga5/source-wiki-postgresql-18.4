BufFileDumpBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/buffile.c:493

目的

メモリ上のバッファ内容を物理ファイルへ書き出し、バッファを空の状態にリセットします。

シグネチャまたは形状

static void BufFileDumpBuffer(BufFile *file)

入力と出力

  • **入力**: BufFile *file（書き出し対象のバッファ付きファイル構造体）
  • **出力**: なし（副作用としてディスクへの書き込みと構造体の状態更新を行う）

前提条件と不変条件

  • file は有効な BufFile 構造体を指している必要があります。
  • 物理ファイルサイズが MAX_PHYSICAL_FILESIZE (1GB) を超える場合、自動的に新しいセグメントへ拡張されます。

アルゴリズムまたは振る舞い

1. バッファ内の全データを物理ファイルへ書き出すまでループ処理を行います。 2. 現在のオフセットが物理ファイルの制限に達している場合、extendBufFile を呼び出して新しいセグメントを作成し、ファイル記述子を切り替えます。 3. FileWrite を使用してバッファデータをディスクへフラッシュします。I/O タイミング計測が有効な場合は pgBufferUsage を更新します。 4. 書き込み失敗時は ereport(ERROR, ...) を発生させます。 5. 書き込み完了後、論理的なファイル位置を維持しつつ、バッファの pos と nbytes を 0 にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/file/buffile.c 内の BufFileWrite (行688) および BufFileSeek (行723) から呼び出されます。
  • **呼び出し先**: extendBufFile (セグメント拡張時)、FileWrite (低レベルI/O)、ereport (エラー時)。

関連データ構造

  • BufFile: バッファ状態、ファイル記述子の配列、現在のオフセットを保持する主要構造体。
  • File: fd.c が管理する仮想ファイル記述子。

失敗モードと境界条件

  • **書き込み失敗**: FileWrite が 0 以下の値を返した場合、ファイルアクセスエラーとして ERROR を発生させます。
  • **セグメント境界**: バッファデータが物理ファイルの境界をまたぐ場合、ループ内で適切に分割して書き込まれます。
  • **シーク後の書き込み**: バッファがダーティな状態で逆方向へのシークが発生した場合、論理位置の整合性を保つためのオフセット調整が行われます。

保守担当者向けメモ

  • 本関数は BufFileLoadBuffer と異なり、バッファ全体を書き出す必要があるため、物理ファイル境界をまたぐループ構造になっています。
  • pgBufferUsage.temp_blks_written をインクリメントするため、一時ファイルの統計情報に直接影響を与えます。
