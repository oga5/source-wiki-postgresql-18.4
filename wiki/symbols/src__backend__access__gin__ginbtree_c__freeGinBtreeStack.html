freeGinBtreeStack

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbtree.c:197

目的

GIN B-treeのトラバーサル中に生成された GinBtreeStack 構造体のリストを解放し、保持されているバッファのピンを解除します。

シグネチャまたは形状

void freeGinBtreeStack(GinBtreeStack *stack)

入力と出力

  • **入力**: stack - 解放対象の GinBtreeStack 構造体へのポインタ（リストの先頭）。
  • **出力**: なし。

前提条件と不変条件

  • stack が指す各要素の buffer フィールドは、有効なバッファIDであるか、InvalidBuffer である必要があります。
  • リスト内の各要素は、pfree で解放可能なメモリ領域にある必要があります。

アルゴリズムまたは振る舞い

1. stack が NULL になるまで以下のループを繰り返します。 2. 現在のスタック要素の parent ポインタを一時変数 tmp に退避します。 3. stack->buffer が InvalidBuffer でない場合、ReleaseBuffer() を呼び出してバッファのピンを解除します。 4. pfree(stack) を呼び出してスタック要素のメモリを解放します。 5. stack を tmp に更新し、次の要素へ進みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbtree.c 内の各処理（分割や挿入の終了時など）。
  • src/backend/access/gin/ginget.c（スキャン終了時など）。
  • src/backend/access/gin/gininsert.c（挿入処理終了時）。
  • **呼び出し先**:
  • ReleaseBuffer（バッファ管理）
  • pfree（メモリ管理）

関連データ構造

  • GinBtreeStack: ツリー降下時のパスを保持する構造体。parent ポインタと buffer を含みます。

失敗モードと境界条件

  • stack が NULL の場合、何もせず正常に終了します。
  • buffer が InvalidBuffer の場合、バッファの解放処理はスキップされます。

保守担当者向けメモ

  • ginget.c の一部の箇所では、この関数を呼び出した後もバッファを保持し続けるために、事前に IncrBufferRefCount を呼び出して参照カウントを調整しているケースがあります。スタックを解放する際は、バッファのピン状態に注意してください。
