brin_form_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_tuple.c:98

目的

メモリ上の BrinMemTuple 構造体を、ディスク書き込みに適したオンディスク形式の BrinTuple へ変換（シリアライズ）します。

シグネチャまたは形状

BrinTuple *brin_form_tuple(BrinDesc *brdesc, BlockNumber blkno, BrinMemTuple *tuple, Size *size)

入力と出力

  • **入力**:
  • brdesc: BRINインデックスの記述子。
  • blkno: 対象となるブロック番号。
  • tuple: 変換元のメモリ内BRINタプル。
  • **出力**:
  • 戻り値: 生成された BrinTuple へのポインタ。
  • size: 生成されたタプルのサイズが格納される出力引数。

前提条件と不変条件

  • brdesc->bd_totalstored > 0 であること。
  • BrinMemTuple 内の各カラムについて、bv_allnulls が設定されている場合はデータ領域をスキップし、NULLビットマップを適切に設定すること。

アルゴリズムまたは振る舞い

1. メモリ上の BrinMemTuple を走査し、各カラムの値を抽出します。 2. bv_serialize が定義されている場合、値をシリアライズします。 3. TOAST_INDEX_HACK が有効な場合、varlena 型のデータに対して以下の処理を行います。

  • 外部ストレージにある場合は detoast_external_attr で取得します。
  • サイズが TOAST_INDEX_TARGET を超える場合は、可能な限りインライン圧縮を試みます。

    • 4. 抽出した値とNULL情報を基に、heap_fill_tuple を用いて物理的なタプルを構築します。 5. NULLビットマップ（allnulls と hasnulls の二重ビットマップ）を構築し、タプルヘッダーに付与します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c 内の複数の箇所（インデックス構築、挿入、更新処理など）。
  • **呼び出し先**: palloc、detoast_external_attr、toast_compress_datum、heap_fill_tuple 等のメモリ管理およびタプル操作ユーティリティ。

関連データ構造

  • BrinTuple: オンディスク上のBRINタプル構造体。
  • BrinMemTuple: メモリ上で操作するための構造体。
  • BrinDesc: BRINインデックスのメタデータ保持構造体。

失敗モードと境界条件

  • idxattno が brdesc->bd_totalstored を超過した場合、アサーションエラーが発生します。
  • TOAST処理において、圧縮やデトーストに失敗した場合の挙動は、各ユーティリティ関数の仕様に依存します。

保守担当者向けメモ

  • NULLビットマップは、一般的なPostgreSQLの慣習とは異なり、NULLを 1 として表現するため、ビット操作時には注意が必要です。
  • TOAST_INDEX_HACK は暫定的な実装であるため、将来的なインデックス再構築の最適化に伴い変更される可能性があります。
  • メモリ割り当てを最小限にするため、パフォーマンスが重要なパスでは既存のバッファ再利用を考慮してください。
