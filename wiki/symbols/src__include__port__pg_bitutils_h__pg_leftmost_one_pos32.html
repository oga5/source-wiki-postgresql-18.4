pg_leftmost_one_pos32

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_bitutils.h:40

目的

32ビット符号なし整数において、最上位ビット（MSB）が立っている位置（0から31のインデックス）を特定します。

シグネチャまたは形状

static inline int pg_leftmost_one_pos32(uint32 word)

入力と出力

  • **入力**: uint32 word（対象となる32ビット整数）
  • **出力**: int（最上位ビットの位置。0がLSB、31がMSBに対応）

前提条件と不変条件

  • 入力 word は 0 であってはなりません（Assert(word != 0) により検証されます）。0 を渡した場合の動作は未定義です。

アルゴリズムまたは振る舞い

環境に応じて以下のいずれかの手法で計算を行います。 1. **HAVE__BUILTIN_CLZ が定義されている場合**: コンパイラの組み込み関数 __builtin_clz を使用し、31 - __builtin_clz(word) を返します。 2. **_MSC_VER が定義されている場合**: MSVC の _BitScanReverse を使用して最上位ビットを検索します。 3. **上記以外**: 8ビットずつシフトしながら非ゼロのバイトを探索し、pg_leftmost_one_pos ルックアップテーブルを使用して位置を特定します。

  • pg_leftmost_one_pos: フォールバック時に使用されるルックアップテーブル（src/port/pg_bitutils.c で定義）。

失敗モードと境界条件

  • word == 0 を渡すとアサーションエラーが発生します。
  • 戻り値は 0 から 31 の範囲に収まります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスが重視される箇所で使用されます。
  • 新しいプラットフォームへの移植時は、HAVE__BUILTIN_CLZ や _MSC_VER の判定が適切に行われているか確認してください。
  • フォールバック実装はルックアップテーブルに依存しているため、テーブル定義との整合性に注意してください。
