get_opts

定義

  • 種別: function
  • 定義箇所: contrib/oid2name/oid2name.c:62

目的

コマンドライン引数を解析し、options 構造体に設定値を格納する。

シグネチャまたは形状

void get_opts(int argc, char **argv, struct options *my_opts)

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。
  • my_opts: 設定を格納する struct options へのポインタ。
  • **出力**: なし（my_opts を直接更新する）。

前提条件と不変条件

  • my_opts は呼び出し元で確保された有効なメモリ領域であること。
  • pg_logging_init が内部で呼び出され、ログ出力が初期化される。

アルゴリズムまたは振る舞い

1. my_opts の各メンバをデフォルト値（false または NULL）で初期化する。 2. --help または -? が指定された場合、help() を呼び出しプログラムを終了する。 3. --version または -V が指定された場合、バージョン情報を出力し終了する。 4. getopt_long を使用して引数をループ処理し、対応するフラグを my_opts に反映させる。

  • f, o, t オプションは add_one_elt を呼び出し、検索対象リストに追加する。
  • d, h, H, p, U オプションは pg_strdup で値をコピーする。
  • i, q, s, S, x オプションはフラグを true に設定する。

    • 5. 解析終了後、optind を確認し、余分な引数が存在する場合はエラーを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数 (contrib/oid2name/oid2name.c:599)
  • **呼び出し先**:
  • pg_logging_init, get_progname
  • help
  • getopt_long
  • add_one_elt
  • pg_strdup
  • pg_log_error, pg_log_error_hint

関連データ構造

  • struct options: 実行時オプションを保持する構造体。

失敗モードと境界条件

  • 不明なオプションが指定された場合、getopt_long がエラーを出力し、本関数は pg_log_error_hint を経由して終了コード 1 で終了する。
  • 引数が多すぎる場合、エラーメッセージを表示して終了コード 1 で終了する。

保守担当者向けメモ

  • long_options 配列に新しいオプションを追加する際は、getopt_long の引数文字列（"d:f:h:H:io:p:qsSt:U:x"）も併せて更新する必要がある。
  • H オプションは非推奨であるため、将来的な削除を検討すること。
