existsTimeLineHistory

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/timeline.c:221

目的

指定されたタイムラインID（probeTLI）に対応する履歴ファイルが存在するかを確認する。

シグネチャまたは形状

bool existsTimeLineHistory(TimeLineID probeTLI)

入力と出力

  • **入力**: probeTLI (確認対象の TimeLineID)
  • **出力**: 履歴ファイルが存在する場合は true、存在しない場合は false を返す。

前提条件と不変条件

  • TimeLineID が 1 の場合、履歴ファイルは存在しないものとして常に false を返す。
  • アーカイブリカバリが要求されている場合（ArchiveRecoveryRequested が真）、RestoreArchivedFile を介してアーカイブから履歴ファイルの取得を試みる。

アルゴリズムまたは振る舞い

1. probeTLI が 1 であれば false を返す。 2. ArchiveRecoveryRequested が有効な場合、TLHistoryFileName でファイル名を生成し、RestoreArchivedFile を呼び出して履歴ファイルをローカルに取得する。 3. それ以外の場合は TLHistoryFilePath を使用してローカルパスを特定する。 4. AllocateFile で該当ファイルを読み込みモード（"r"）でオープンする。 5. オープンに成功した場合は FreeFile でクローズし、true を返す。 6. オープンに失敗した場合、その原因がファイル不在（ENOENT）であれば false を返し、それ以外のエラーであれば FATAL で異常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/timeline.c (内部参照)
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/replication/walreceiver.c
  • **呼び出し先**:
  • TLHistoryFileName / TLHistoryFilePath
  • RestoreArchivedFile
  • AllocateFile / FreeFile
  • ereport

関連データ構造

  • TimeLineID: タイムラインを識別する整数型。

失敗モードと境界条件

  • probeTLI == 1: 履歴ファイルを持たないため、常に false を返す。
  • ファイルオープンエラー: ENOENT（ファイル不在）以外のエラーが発生した場合、ereport(FATAL, ...) によりプロセスが終了する。

保守担当者向けメモ

  • 本関数は履歴ファイルの存在確認のみを行い、内容は読み込まない。内容の解析が必要な場合は readTimeLineHistory を使用すること。
  • アーカイブリカバリ環境下では、ネットワークや外部ストレージへのアクセスが発生する可能性があるため、I/O 負荷に注意が必要。
