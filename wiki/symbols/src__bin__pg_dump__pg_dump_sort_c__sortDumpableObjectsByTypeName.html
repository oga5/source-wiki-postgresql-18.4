sortDumpableObjectsByTypeName

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump_sort.c:191

目的

ダンプ対象のオブジェクト配列を、型および名前の順序に基づいてソートし、安定した初期順序を決定する。

シグネチャまたは形状

void sortDumpableObjectsByTypeName(DumpableObject **objs, int numObjs)

入力と出力

  • **入力**:
  • objs: DumpableObject 構造体へのポインタ配列。
  • numObjs: 配列内の要素数。
  • **出力**: なし（objs 配列がインプレースで並び替えられる）。

前提条件と不変条件

  • objs は numObjs 個の要素を持つ有効な配列であること。
  • ソートの比較ロジックとして DOTypeNameCompare 関数が使用される。

アルゴリズムまたは振る舞い

1. numObjs が 1 より大きい場合のみ処理を実行する。 2. 標準ライブラリの qsort を使用し、DOTypeNameCompare を比較関数として objs を並び替える。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c:1164
  • **呼び出し先**: qsort (標準ライブラリ), DOTypeNameCompare (比較関数)

関連データ構造

  • DumpableObject: ダンプ対象のデータベースオブジェクトを表現する構造体。

失敗モードと境界条件

  • numObjs <= 1 の場合、ソート処理はスキップされ、何も変更されない。

保守担当者向けメモ

  • 本関数は pg_dump におけるソート処理の初期段階で使用される。
  • ソート順序の安定性はテスト（002_pg_upgrade.pl 等）の差分に影響するため、順序の変更が必要な場合は DOTypeNameCompare のタイブレーク処理を慎重に検討すること。
