DictSimple

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/dict_simple.c:23

目的

PostgreSQL の全文検索における「Simple」辞書の設定（ストップワードリストおよび受理フラグ）を保持する構造体です。

シグネチャまたは形状

typedef struct
{
	StopList	stoplist;
	bool		accept;
} DictSimple;

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。辞書初期化時および検索時に、この構造体のインスタンスがパラメータとして読み書きされます。

前提条件と不変条件

  • stoplist は readstoplist 関数によってメモリ上にロードされた有効なストップワードリストである必要があります。
  • accept は、ストップワードではない単語をそのまま返すか（true）、認識不能として扱うか（false）を決定するフラグとして機能します。

アルゴリズムまたは振る舞い

本構造体は、dsimple_init で初期化され、dsimple_lexize におけるトークンのフィルタリング処理の基準として参照されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • dsimple_init (src/backend/tsearch/dict_simple.c:34): インスタンスのメモリ確保に使用。
  • dsimple_lexize (src/backend/tsearch/dict_simple.c:78): 辞書処理時の設定読み出しに使用。
  • 呼び出し先:
  • 本構造体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • StopList: stoplist フィールドの型。ストップワードの集合を保持します。

失敗モードと境界条件

  • メモリ確保失敗: palloc0 が失敗した場合、PostgreSQL のエラーハンドリングによりプロセスが中断されます。
  • 設定不備: accept フラグが適切に設定されていない場合、意図しないフィルタリング結果（単語の誤った破棄または保持）を招く可能性があります。

保守担当者向けメモ

  • 本構造体は palloc0 で割り当てられるため、ライフサイクル管理には PostgreSQL のメモリコンテキストが関与します。
  • accept = false に設定することで、特定の単語を「認識不能」として扱うフィルタとして利用可能です。
