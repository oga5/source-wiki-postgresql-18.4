GetSubscription

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_subscription.c:71

目的

システムカタログ pg_subscription から指定された OID に対応するサブスクリプション情報を読み取り、メモリ上の Subscription 構造体に展開して返します。

シグネチャまたは形状

Subscription *GetSubscription(Oid subid, bool missing_ok)

入力と出力

  • **入力**:
  • subid (Oid): 取得対象のサブスクリプションの OID。
  • missing_ok (bool): true の場合、対象が見つからなくてもエラーにせず NULL を返します。
  • **出力**:
  • 成功時: 取得した情報を格納した Subscription 構造体へのポインタ。
  • 失敗時: missing_ok が false かつ対象が存在しない場合、elog(ERROR) により処理を中断します。

前提条件と不変条件

  • SUBSCRIPTIONOID システムキャッシュが利用可能であること。
  • 戻り値の Subscription 構造体は palloc で確保されるため、呼び出し元で適切に管理（または FreeSubscription による解放）する必要があります。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して pg_subscription カタログからタプルを検索します。 2. タプルが見つからない場合、missing_ok に応じて NULL を返すかエラーを発生させます。 3. GETSTRUCT でタプルから Form_pg_subscription を展開し、Subscription 構造体の各フィールドに値をコピーします。 4. conninfo、slotname、synccommit、publications、origin などの属性を SysCacheGetAttr を介して取得し、適切な型（文字列やリスト）に変換します。 5. superuser_arg を呼び出し、サブスクリプション所有者がスーパーユーザーであるかを判定して構造体に格納します。 6. ReleaseSysCache でタプルを解放し、構築した構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • SearchSysCache1, ReleaseSysCache
  • SysCacheGetAttr, SysCacheGetAttrNotNull
  • textarray_to_stringlist
  • superuser_arg
  • palloc, pstrdup

関連データ構造

  • Subscription: サブスクリプション情報を保持するメモリ上の構造体。
  • Form_pg_subscription: pg_subscription カタログのタプル構造。

失敗モードと境界条件

  • 指定された subid が存在しない場合、missing_ok が false であればエラーとなります。
  • subslotname が NULL の場合、構造体の slotname フィールドには NULL が設定されます。

保守担当者向けメモ

  • 本関数はシステムキャッシュを利用しているため、カタログの直接的な変更が必要な場合を除き、効率的なアクセス手段となります。
  • 構造体内の文字列フィールドは pstrdup 等で動的に確保されているため、不要になった際は FreeSubscription を使用してメモリリークを防いでください。
