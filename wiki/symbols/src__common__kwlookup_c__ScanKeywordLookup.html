ScanKeywordLookup

定義

  • 種別: function
  • 定義箇所: src/common/kwlookup.c:37

目的

入力文字列が特定のキーワードリストに含まれているかを、大文字・小文字を区別せずに高速に判定する。

シグネチャまたは形状

int ScanKeywordLookup(const char *str, const ScanKeywordList *keywords);

入力と出力

  • **入力**:
  • str: 検索対象の文字列（NULL終端）。
  • keywords: 検索対象のキーワードリスト構造体。
  • **出力**:
  • 成功時: キーワードリスト内でのインデックス（int）。
  • 失敗時: -1。

前提条件と不変条件

  • keywords->hash 関数は、大文字・小文字を区別しないハッシュ値を生成するものと仮定する。
  • パーフェクトハッシュ関数を使用しているため、ハッシュ値が有効範囲内であれば、そのインデックスのキーワードのみを検証すればよい。
  • ロケール依存の誤変換を防ぐため、tolower() は使用せず、ASCII範囲内での小文字化を行う。

アルゴリズムまたは振る舞い

1. **長さチェック**: str の長さが keywords->max_kw_len を超える場合、即座に -1 を返す。 2. **ハッシュ計算**: keywords->hash を呼び出し、候補となるインデックス h を取得する。 3. **範囲検証**: h が 0 以上 keywords->num_keywords 未満であることを確認し、範囲外なら -1 を返す。 4. **文字列照合**: GetScanKeyword(h, keywords) で取得したキーワードと str を1文字ずつ比較する。比較時に str の各文字を ASCII ベースで小文字化し、不一致があれば -1 を返す。 5. **完了**: 全文字が一致し、かつキーワード文字列も終端に達していれば h を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/ruleutils.c
  • src/bin/psql/tab-complete.in.c
  • src/fe_utils/string_utils.c
  • **呼び出し先**:
  • keywords->hash (関数ポインタ)
  • GetScanKeyword (インライン関数)

関連データ構造

  • ScanKeywordList: 検索対象のキーワード群、オフセット情報、ハッシュ関数を保持する構造体。

失敗モードと境界条件

  • 入力文字列がキーワードの最大長を超える場合、検索を行わず -1 を返す。
  • ハッシュ値が有効なインデックス範囲外である場合、検索は失敗とみなされる。
  • 文字列の長さが一致しても、内容が一致しない場合は -1 を返す。

保守担当者向けメモ

  • 本関数は SQL 標準に準拠した ASCII ベースの比較を行うため、ロケールに依存する tolower() を使用してはならない。
  • keywords 構造体のデータは src/tools/gen_keywordlist.pl によって生成されることを前提としている。
