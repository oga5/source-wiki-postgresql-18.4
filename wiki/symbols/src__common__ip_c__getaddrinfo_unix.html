getaddrinfo_unix

定義

  • 種別: function
  • 定義箇所: src/common/ip.c:152

目的

Unix ドメインソケット用の addrinfo 構造体を構築し、名前解決をエミュレートする。標準の getaddrinfo API が対応していない Unix ソケットパスを透過的に扱えるようにする。

シグネチャまたは形状

static int getaddrinfo_unix(const char *path, const struct addrinfo *hintsp, struct addrinfo **result);

入力と出力

  • **入力**:
  • path: Unix ソケットのパス文字列。
  • hintsp: フィルタリング条件を指定する addrinfo 構造体（NULL 可）。
  • **出力**:
  • result: 成功時に確保された addrinfo 構造体へのポインタを格納。
  • 戻り値: 成功時は 0、失敗時は EAI_FAIL または EAI_MEMORY を返す。

前提条件と不変条件

  • path の長さは sockaddr_un.sun_path のサイズ制限を超えてはならない。
  • hintsp が指定されている場合、ai_family は AF_UNIX である必要がある。
  • 成功時、result には calloc で確保された addrinfo および sockaddr_un が設定される。

アルゴリズムまたは振る舞い

1. path の長さを検証し、制限を超える場合は EAI_FAIL を返す。 2. hintsp が NULL の場合はデフォルト値（AF_UNIX, SOCK_STREAM）を使用し、指定がある場合はコピーする。 3. ai_family が AF_UNIX でない場合はエラーを返す。 4. addrinfo および sockaddr_un を動的に確保する。 5. addrinfo の各フィールド（ai_family, ai_socktype, ai_protocol）を設定する。 6. sockaddr_un にパスをコピーする。 7. パスが @ で始まる場合、Linux 抽象名前空間ソケットとして扱い、先頭をヌル文字に置換し、ai_addrlen を調整する。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_getaddrinfo_all（src/common/ip.c 内）。
  • **呼び出し先**: calloc, memcpy, strlen, strcpy などの標準ライブラリ関数。

関連データ構造

  • struct addrinfo: ネットワークアドレス情報を保持する標準構造体。
  • struct sockaddr_un: Unix ドメインソケットのアドレスを保持する構造体。

失敗モードと境界条件

  • path が sun_path の許容長を超える場合: EAI_FAIL を返す。
  • メモリ確保失敗時: EAI_MEMORY を返す。
  • hintsp->ai_family が AF_UNIX 以外の場合: EAI_FAIL を返す。

保守担当者向けメモ

  • 本関数で確保されたメモリは、呼び出し元で pg_freeaddrinfo_all を使用して解放する必要がある。
  • @ で始まるパスの処理は、Linux 抽象名前空間の慣習に従っている。他のプラットフォームでの挙動については注意が必要。
