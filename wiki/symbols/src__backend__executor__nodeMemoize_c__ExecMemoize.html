ExecMemoize

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMemoize.c:696

目的

パラメータ化されたプランノードの実行結果をキャッシュし、同一パラメータでの再スキャン時にサブプランの再実行を回避して計算コストを削減する。

シグネチャまたは形状

TupleTableSlot *ExecMemoize(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate (Memoizeノードの状態構造体)
  • **出力**: TupleTableSlot * (キャッシュから取得、またはサブプランから取得したタプル。スキャン終了時は NULL)

前提条件と不変条件

  • pstate は MemoizeState 型にキャスト可能であること。
  • ResetExprContext により、呼び出しごとに式評価用のメモリコンテキストがリセットされる。
  • キャッシュミス時にメモリ不足等でエントリ作成に失敗した場合、キャッシュをバイパスするモードへ遷移する。

アルゴリズムまたは振る舞い

ステートマシンとして動作し、以下の状態遷移を行う： 1. **MEMO_CACHE_LOOKUP**: 現在のパラメータでキャッシュを検索。

  • ヒット時: キャッシュされたタプルを順次返し、MEMO_CACHE_FETCH_NEXT_TUPLE へ遷移。
  • ミス時: サブプランを実行し、結果をキャッシュに格納して MEMO_FILLING_CACHE へ遷移。キャッシュ不可時は MEMO_CACHE_BYPASS_MODE へ遷移。

    • 2. **MEMO_CACHE_FETCH_NEXT_TUPLE**: キャッシュされたタプルリストを走査し、次のタプルを返す。 3. **MEMO_FILLING_CACHE**: サブプランから取得したタプルをキャッシュエントリに順次追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecProcNode (実行ノードのメインループから、mstate->ss.ps.ExecProcNode を介して呼び出される)。
  • **呼び出し先**: cache_lookup, cache_store_tuple, entry_purge_tuples, ExecProcNode (サブプランの実行用)。

関連データ構造

  • MemoizeState: ノードの実行状態を保持。
  • MemoizeEntry: キャッシュのハッシュテーブルに格納されるデータ構造。
  • TupleTableSlot: タプルの受け渡しに使用。

失敗モードと境界条件

  • **キャッシュミス**: 初回実行時やパラメータ変更時に発生。
  • **キャッシュオーバーフロー**: メモリ制限によりエントリ作成やタプル保存に失敗した場合、MEMO_CACHE_BYPASS_MODE へ遷移し、キャッシュを介さず直接サブプランを実行する。
  • **空の結果**: サブプランがタプルを返さない場合、MEMO_END_OF_SCAN となり NULL を返す。

保守担当者向けメモ

  • singlerow フラグが有効な場合、最初のタプル取得後にキャッシュエントリを完了と見なす最適化が行われる。
  • キャッシュエントリのパージは、不完全なスキャン結果が残っている場合に再スキャン時の整合性を保つために行われる。
