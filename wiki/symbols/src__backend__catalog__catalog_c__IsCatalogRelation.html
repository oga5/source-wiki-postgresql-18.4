IsCatalogRelation

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/catalog.c:103

目的

指定されたリレーションがシステムカタログであるかどうかを判定します。

シグネチャまたは形状

bool IsCatalogRelation(Relation relation)

入力と出力

  • **入力**: Relation relation（判定対象のリレーション記述子）
  • **出力**: bool（システムカタログであれば true、そうでなければ false）

前提条件と不変条件

  • 本関数はカタログアクセスを行いません。ロック取得中や再帰的な呼び出しにおいても安全に使用可能です。
  • 判定ロジックは、リレーションのOIDに基づいています。

アルゴリズムまたは振る舞い

1. RelationGetRelid(relation) を呼び出し、対象リレーションのOIDを取得します。 2. 取得したOIDを引数として IsCatalogRelationOid を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/catalog/index.c
  • src/backend/catalog/pg_publication.c
  • src/backend/catalog/toasting.c
  • src/backend/executor/execReplication.c
  • src/backend/parser/parse_clause.c
  • src/backend/storage/aio/read_stream.c
  • **呼び出し先**:
  • RelationGetRelid (マクロ)
  • IsCatalogRelationOid

関連データ構造

  • Relation: リレーション記述子。

失敗モードと境界条件

  • relation が NULL の場合、RelationGetRelid の実装に依存した動作となります（通常はセグメンテーション違反）。
  • OIDがシステム予約範囲外であっても、IsCatalogRelationOid の内部ロジックに従い正しく false を返します。

保守担当者向けメモ

  • 本関数はシステムカタログの定義を集中管理する役割を担っています。判定基準を変更する場合は IsCatalogRelationOid の実装を確認してください。
  • カタログアクセスを伴わない設計であるため、この特性を維持してください。
