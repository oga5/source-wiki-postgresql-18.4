GIST_SPLITVEC

定義

  • 種別: struct
  • 定義箇所: src/include/access/gist.h:143

目的

GiSTインデックスのページ分割（PickSplitメソッド）処理において、既存のタプルを新しい2つのページへどのように振り分けるか、および各ページに格納されるキーの結合（Union）情報を保持するために使用されます。

シグネチャまたは形状

typedef struct GIST_SPLITVEC
{
    OffsetNumber *spl_left;
    int          spl_nleft;
    Datum        spl_ldatum;
    bool         spl_ldatum_exists;

    OffsetNumber *spl_right;
    int          spl_nright;
    Datum        spl_rdatum;
    bool         spl_rdatum_exists;
} GIST_SPLITVEC;

入力と出力

  • **入力**: PickSplitメソッドの実装において、この構造体の各フィールドを初期化・設定します。
  • **出力**: 分割後の左右のページに配置されるタプルのオフセット配列（spl_left, spl_right）と、それらのキーの包含範囲を示すデータ（spl_ldatum, spl_rdatum）が後続のインデックス構築処理へ渡されます。

前提条件と不変条件

  • spl_left および spl_right は、分割対象となるタプルのオフセット番号を保持する配列を指す必要があります。
  • spl_nleft および spl_nright は、それぞれの配列の有効な要素数と一致しなければなりません。
  • spl_ldatum_exists および spl_rdatum_exists が true の場合、対応する spl_ldatum / spl_rdatum に有効な結合キーが格納されている必要があります。

アルゴリズムまたは振る舞い

1. PickSplitメソッドが、インデックスページ内のタプルを2つのグループに分割する戦略を決定します。 2. 分割結果に基づき、spl_left と spl_right にオフセット番号を格納し、それぞれの個数を spl_nleft / spl_nright に設定します。 3. 各グループのキーの包含範囲を計算し、spl_ldatum / spl_rdatum に格納します。 4. 呼び出し元は、これらの情報を使用して物理的なページ分割とタプルの再配置を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist 配下の各モジュール（btree_bit.c, btree_bool.c, btree_bytea.c, btree_cash.c, btree_date.c, btree_enum.c, btree_float4.c, btree_float8.c, btree_inet.c, btree_int2.c, btree_int4.c, btree_int8.c 等）において、PG_GETARG_POINTER(1) を介して取得される形で参照されています。
  • **呼び出し先**: 構造体定義のため、直接的な関数呼び出しはありません。

関連データ構造

  • OffsetNumber: インデックスページ内のタプル位置を示す型。
  • Datum: インデックスキーの値を保持する汎用型。

失敗モードと境界条件

  • 配列のメモリ割り当てが不十分な場合、または spl_nleft / spl_nright が実際の配列サイズと不整合を起こす場合にメモリ破壊の危険性があります。
  • spl_ldatum_exists が false の状態で spl_ldatum を参照した場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本構造体はGiSTインデックスのコアAPIの一部であり、変更はすべてのGiST演算クラスに影響を与えます。
  • contrib/btree_gist 以外の拡張モジュールでも同様のパターンで利用されている可能性があるため、変更時は広範な影響調査が必要です。
