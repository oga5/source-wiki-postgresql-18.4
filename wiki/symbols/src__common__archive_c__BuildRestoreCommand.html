BuildRestoreCommand

定義

  • 種別: function
  • 定義箇所: src/common/archive.c:38

目的

WALアーカイブからファイルを復元するための restore_command 文字列内のプレースホルダー（%f, %r, %p）を、実際の値に置換して実行可能なコマンド文字列を構築します。

シグネチャまたは形状

char *BuildRestoreCommand(const char *restoreCommand,
                          const char *xlogpath,
                          const char *xlogfname,
                          const char *lastRestartPointFname);

入力と出力

  • **入力**:
  • restoreCommand: 置換対象のテンプレート文字列。
  • xlogpath: ファイルパス（%p 用）。指定がある場合、OS固有の形式に変換されます。
  • xlogfname: ファイル名（%f 用）。
  • lastRestartPointFname: 最終再起動ポイントのファイル名（%r 用）。
  • **出力**:
  • 置換済みの新しい文字列（palloc で確保されたメモリ領域）。呼び出し側で解放する必要があります。

前提条件と不変条件

  • xlogpath が提供される場合、内部で pstrdup された後に make_native_path が適用されます。
  • 戻り値の文字列は、呼び出し側が責任を持って解放（pfree）する必要があります。

アルゴリズムまたは振る舞い

1. xlogpath が存在する場合、その複製を作成し make_native_path を呼び出してOS固有のパス形式に正規化します。 2. replace_percent_placeholders を呼び出し、テンプレート内の %f, %r, %p をそれぞれ xlogfname, lastRestartPointFname, 正規化済み nativePath に置換します。 3. 一時的に確保した nativePath を解放します。 4. 構築された置換後の文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogarchive.c:153
  • src/fe_utils/archive.c:49
  • **呼び出し先**:
  • make_native_path
  • replace_percent_placeholders
  • pstrdup / pfree

関連データ構造

  • なし（標準的な文字列ポインタを使用）。

失敗モードと境界条件

  • xlogpath が NULL の場合、%p の置換には NULL が渡されます。
  • メモリ不足時には palloc 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は置換ロジックのみを提供し、コマンドの実行は行いません。
  • FRONTEND マクロの定義により、バックエンドとフロントエンドの両方でコンパイル可能です。
