cfind

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexec.c:510

目的

バックリファレンス等を含む複雑な正規表現に対して、メインの NFA を用いたマッチングを実行する。

シグネチャまたは形状

static int cfind(struct vars *v, struct cnfa *cnfa, struct colormap *cm)

入力と出力

  • **入力**:
  • v: マッチング状態を保持する vars 構造体。
  • cnfa: 実行対象のコンパイル済み NFA。
  • cm: 文字クラスのカラーマッピング情報。
  • **出力**:
  • int: マッチング結果（成功またはエラーコード）。

前提条件と不変条件

  • v が有効な vars 構造体を指していること。
  • newdfa による DFA の生成が成功する必要がある。
  • REG_EXPECT フラグが有効な場合、v->details が非 NULL であること。

アルゴリズムまたは振る舞い

1. newdfa を使用して、検索用 DFA (s) と対象 NFA 用 DFA (d) を生成する。 2. DFA 生成に失敗した場合はエラーコードを返し、成功した場合は cfindloop を呼び出してマッチングを実行する。 3. cfindloop の完了後、生成した DFA を freedfa で解放する。 4. REG_EXPECT フラグが設定されている場合、マッチングの詳細情報（rm_extend）を更新する。 5. 最終的なマッチング結果を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regexec.c:309
  • **呼び出し先**: newdfa, cfindloop, freedfa, NOERR

関連データ構造

  • vars: マッチングのコンテキストを保持。
  • dfa: 遅延評価 DFA の状態管理。
  • cnfa: コンパイル済み NFA。
  • colormap: 文字クラスの定義。

失敗モードと境界条件

  • newdfa がメモリ不足等で NULL を返した場合、エラーとして終了する。
  • cfindloop 内でエラーが発生した場合、そのエラーコードが伝播される。

保守担当者向けメモ

  • 本関数はバックリファレンスを含む複雑なケースを扱うため、単純な find 関数よりも計算コストが高い。
  • REG_EXPECT 処理において v->details->rm_extend.rm_eo は現時点では未知の値として扱われている。
