ParsedWord

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_utils.h:80

目的

テキスト解析過程において、抽出された単語とその属性（位置情報やフラグ）を一時的に保持するための構造体です。

シグネチャまたは形状

typedef struct {
    uint16      flags;
    uint16      len;
    uint16      nvariant;
    uint16      alen;
    union {
        uint16      pos;
        uint16     *apos;
    }           pos;
    char       *word;
} ParsedWord;

入力と出力

本構造体はデータ保持用であり、関数ではありません。解析器によって生成され、ParsedText 構造体の一部として格納されるか、ソート処理等のために配列として扱われます。

前提条件と不変条件

  • apos 配列を使用する場合、apos[0] は配列内の要素数（apos[0] 自体を除く）を保持します。
  • alen は apos 配列の割り当て済みサイズを示します。
  • MAXNUMPOS を超える要素数は許可されません。

アルゴリズムまたは振る舞い

  • 単語の文字列データ (word) とその長さ (len) を保持します。
  • 位置情報 (pos) は単一の値、または複数の位置を保持する配列 (apos) として共用体で管理されます。
  • to_tsany.c において、compareWORD 関数を用いたソートや、uniqueWORD による重複排除の対象として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/to_tsany.c 内の各関数が、テキスト解析結果の格納やソート処理のために本構造体を利用しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ParsedText: ParsedWord の配列を保持する上位の構造体。
  • TSVector: 解析完了後に変換される最終的なデータ型。

失敗モードと境界条件

  • メモリ割り当て時、MaxAllocSize を超える要素数で配列を確保しようとすると、to_tsany.c 内でサイズが制限されるロジックが存在します。
  • apos 使用時の要素数が MAXNUMPOS を超えないよう管理する必要があります。

保守担当者向けメモ

  • pos 共用体の切り替えロジックは flags や nvariant と密接に関連しているため、変更時は to_tsany.c 側の比較・ソートロジックとの整合性に注意してください。
