DropOwnedObjects

定義

  • 種別: function
  • 定義箇所: src/backend/commands/user.c:1582

目的

指定されたロールが所有するデータベースオブジェクトを削除（または権限剥奪）する。

シグネチャまたは形状

void DropOwnedObjects(DropOwnedStmt *stmt)

入力と出力

  • **入力**: DropOwnedStmt *stmt（削除対象のロールリストおよび削除動作 behavior を含む構造体）
  • **出力**: なし（成功時はオブジェクトの削除を実行、失敗時はエラーをスロー）

前提条件と不変条件

  • 呼び出し元は、対象となるロールに対する適切な権限（has_privs_of_role）を保持している必要がある。
  • 権限チェックに失敗した場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーが発生する。

アルゴリズムまたは振る舞い

1. roleSpecsToIds を使用して、stmt->roles から対象ロールの OID リストを取得する。 2. リスト内の各ロールについて、現在のユーザー（GetUserId()）が対象ロールの権限を持っているか検証する。 3. 検証を通過した後、shdepDropOwned を呼び出し、指定された behavior（CASCADE/RESTRICT）に従って共有オブジェクトの削除処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:1818（DropOwnedStmt を受け取ったユーティリティコマンド実行時）
  • **呼び出し先**:
  • roleSpecsToIds
  • has_privs_of_role
  • GetUserId
  • GetUserNameFromId
  • shdepDropOwned

関連データ構造

  • DropOwnedStmt: 削除対象ロールと削除動作を保持する構文解析ノード。

失敗モードと境界条件

  • 指定されたロールが存在しない場合、roleSpecsToIds 内でエラーが発生する可能性がある。
  • 呼び出し元ユーザーが対象ロールの権限を持たない場合、ereport(ERROR, ...) により処理が中断される。

保守担当者向けメモ

  • 本関数は共有オブジェクトの依存関係管理（shdep）のラッパーとして機能している。
  • 削除ロジックの詳細は shdepDropOwned に依存しているため、オブジェクト削除の挙動を変更する場合はそちらを参照すること。
