ReorderBufferFreeTXN

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/reorderbuffer.c:457

目的

ReorderBufferTXN 構造体に関連付けられたリソースを解放し、メモリから破棄する。

シグネチャまたは形状

static void ReorderBufferFreeTXN(ReorderBuffer *rb, ReorderBufferTXN *txn)

入力と出力

  • rb: ReorderBuffer へのポインタ。トランザクションキャッシュの更新に使用される。
  • txn: 解放対象の ReorderBufferTXN 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • txn->size == 0 であること（すべての変更レコードが既に解放済みである必要がある）。
  • txn が指すメモリ領域は、本関数の実行後に無効となる。

アルゴリズムまたは振る舞い

1. rb->by_txn_last_xid が対象の txn->xid と一致する場合、キャッシュを無効化（InvalidTransactionId および NULL を設定）する。 2. txn->gid が存在する場合、メモリを解放し NULL を代入する。 3. txn->tuplecid_hash が存在する場合、ハッシュテーブルを破棄し NULL を代入する。 4. txn->invalidations および txn->invalidations_distributed が存在する場合、それぞれメモリを解放し NULL を代入する。 5. ReorderBufferToastReset を呼び出し、TOAST 関連のハッシュをリセットする。 6. txn->size が 0 であることを確認するアサーションを実行する。 7. txn 自体のメモリを pfree で解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/reorderbuffer.c 内の複数箇所（ReorderBufferFreeTXN の宣言および ReorderBufferFreeTXN(rb, txn) の呼び出し）。
  • 呼び出し先: pfree (メモリ解放), hash_destroy (ハッシュテーブル破棄), ReorderBufferToastReset (TOAST リセット)。

関連データ構造

  • ReorderBuffer: デコーディングセッションの状態管理構造体。
  • ReorderBufferTXN: 解放対象となるトランザクション状態保持構造体。

失敗モードと境界条件

  • txn->size が 0 でない場合、アサーションエラーが発生する。
  • rb または txn が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はトランザクションのクリーンアップの最終段階で呼び出されます。新しいフィールドを ReorderBufferTXN に追加した場合は、必ず本関数で適切に解放処理を行ってください。
  • txn->size のチェックは、変更レコードのリークを防ぐための重要な不変条件です。
