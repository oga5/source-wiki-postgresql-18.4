lazy_check_wraparound_failsafe

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:2949

目的

トランザクションIDの周回（wraparound）が差し迫っている場合に、VACUUM処理の安全装置（failsafe）を起動し、インデックス掃除などの非本質的なメンテナンスをスキップしてヒープスキャンを優先させる。

シグネチャまたは形状

static bool lazy_check_wraparound_failsafe(LVRelState *vacrel)

入力と出力

  • **入力**: LVRelState *vacrel（VACUUM実行中の状態を保持する構造体）
  • **出力**: bool（安全装置が有効化された、または既に有効である場合は true、それ以外は false）

前提条件と不変条件

  • VacuumFailsafeActive が true の場合、本関数は即座に true を返す。
  • vacuum_xid_failsafe_check が true を返す場合のみ、安全装置の有効化処理が実行される。

アルゴリズムまたは振る舞い

1. VacuumFailsafeActive を確認し、既に有効であれば早期リターンする。 2. vacuum_xid_failsafe_check を呼び出し、周回リスクを判定する。 3. リスクが検出された場合：

  • VacuumFailsafeActive を true に設定する。
  • vacrel->bstrategy を NULL にし、バッファアクセス戦略を無効化して共有バッファの全利用を許可する。
  • インデックスの真空化、インデックスのクリーンアップ、およびリレーションの切り詰め（Truncate）フラグをすべて false に設定する。
  • 進捗状況カウンタをリセットする。
  • 警告メッセージ（WARNING）をログ出力する。
  • VacuumCostActive を false に設定し、コスト制限を無効化する。

    • 4. 処理完了後、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/vacuumlazy.c 内の複数箇所（行 447, 832, 1267, 2596, 2625, 2643）。
  • **呼び出し先**: vacuum_xid_failsafe_check（外部関数）、pgstat_progress_update_multi_param（外部関数）、ereport（外部関数）。

関連データ構造

  • LVRelState: VACUUMの実行状態を管理する構造体。
  • VacuumFailsafeActive: 安全装置の有効状態を示すグローバル変数。
  • VacuumCostActive: VACUUMのコスト制限の有効状態を示すグローバル変数。

失敗モードと境界条件

  • 本関数は VACUUM ごとに一度だけ警告を発する設計となっている。
  • vacrel->bstrategy のメモリ解放は、本関数ではなく、そのメモリを割り当てた呼び出し元が責任を持つ前提となっている。

保守担当者向けメモ

  • 本関数は、maintenance_work_mem や autovacuum_work_mem の不足が原因で周回リスクが高まった際の緊急回避策である。
  • インデックス掃除を省略するため、本関数が作動した後はテーブルのインデックスにデッドエントリが残存する可能性がある。
