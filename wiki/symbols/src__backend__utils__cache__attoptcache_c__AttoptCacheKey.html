AttoptCacheKey

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/attoptcache.c:32

目的

属性オプション（attoptions）をキャッシュするためのハッシュテーブルにおいて、個別のキャッシュエントリを一意に識別するための検索キーとして機能します。

シグネチャまたは形状

typedef struct
{
	Oid			attrelid;
	int			attnum;
} AttoptCacheKey;

入力と出力

  • **入力**: リレーションのOID (attrelid) および属性番号 (attnum)。
  • **出力**: ハッシュテーブル検索時に使用される構造体。

前提条件と不変条件

  • AttoptCacheEntry 構造体の先頭メンバとして配置される必要があります（src/backend/utils/cache/attoptcache.c:40 の参照より）。
  • ハッシュテーブルのキーサイズとして sizeof(AttoptCacheKey) が使用されます。

アルゴリズムまたは振る舞い

  • get_attribute_options() がキャッシュを検索する際、この構造体に検索対象の attrelid と attnum を格納して hash_search に渡します。
  • ハッシュテーブルの構築時、ctl.keysize に本構造体のサイズが設定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: get_attribute_options 関数がキャッシュ検索およびエントリ作成時に本構造体を初期化して使用します。
  • **呼び出し先**: hash_search 関数（utils/hsearch.h）が本構造体をキーとしてハッシュテーブル操作を行います。

関連データ構造

  • AttoptCacheEntry: 本構造体をキーとして保持するキャッシュエントリ構造体。
  • AttoptCacheHash: 本構造体をキーとして使用するハッシュテーブル。

失敗モードと境界条件

  • 構造体内の attrelid または attnum が無効な値である場合、ハッシュテーブル内で一致するエントリが見つからず、キャッシュミスが発生します。

保守担当者向けメモ

  • 本構造体はハッシュテーブルのキーとして直接使用されるため、メンバの順序やパディングによるメモリレイアウトの変化に注意してください。
  • AttoptCacheEntry の先頭メンバであるという制約は、ハッシュテーブルの検索ロジックに依存しているため、構造体定義を変更する際は注意が必要です。
