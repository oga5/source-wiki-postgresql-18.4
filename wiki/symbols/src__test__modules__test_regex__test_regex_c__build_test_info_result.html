build_test_info_result

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_regex/test_regex.c:617

目的

コンパイルされた正規表現オブジェクト（regex_t）のメタ情報およびフラグ状態を、SQL で利用可能な text[] 配列形式に変換して出力する。

シグネチャまたは形状

ArrayType *build_test_info_result(regex_t *cpattern, test_re_flags *flags)

入力と出力

  • **入力**:
  • cpattern: コンパイル済みの正規表現構造体へのポインタ。
  • flags: 期待されるフラグ状態を保持する test_re_flags 構造体へのポインタ。
  • **出力**:
  • ArrayType *: 正規表現のサブ式数およびフラグの整合性チェック結果を含む text 型の配列。

前提条件と不変条件

  • cpattern は有効な regex_t 構造体を指している必要がある。
  • flags は test_re_flags 構造体として初期化済みであること。
  • infonames 配列内の各ビット定義は、regex_t.re_info のビットマスクと対応している必要がある。

アルゴリズムまたは振る舞い

1. 正規表現のサブ式数（re_nsub）を配列の最初の要素として格納する。 2. infonames テーブルを走査し、cpattern->re_info の各ビット状態と flags->info の期待値を比較する。

  • ビットが立っており、かつ期待通りであればフラグ名を配列に追加する。
  • ビットが立っているが期待されていない場合は "unexpected <name>!" を追加する。
  • ビットが立っていないが期待されている場合は "missing <name>!" を追加する。

    • 3. 構築した Datum 配列を construct_md_array を用いて text[] 型の ArrayType に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: test_regex 関数（src/test/modules/test_regex/test_regex.c:122）。
  • **呼び出し先**: cstring_to_text, construct_md_array（PostgreSQL 内部ユーティリティ）。

関連データ構造

  • regex_t: PostgreSQL の正規表現エンジンが使用する構造体。
  • test_re_flags: テスト用のフラグ設定を保持する構造体。
  • infoname: re_info ビットと文字列ラベルを対応付けるための内部構造体。

失敗モードと境界条件

  • re_nsub が 0 の場合でも、配列の最初の要素として正しく出力される。
  • infonames に定義されていないビットは無視される。
  • construct_md_array がメモリ不足等で失敗した場合、PostgreSQL のエラーハンドリング機構により例外が発生する。

保守担当者向けメモ

  • 本関数は text 型の配列をハードコードされた前提で構築しているため、将来的に戻り値の型や構造を変更する場合は construct_md_array の引数を確認すること。
  • infonames テーブルに新しい正規表現フラグを追加する場合は、regex/regex.h の定義と同期させる必要がある。
