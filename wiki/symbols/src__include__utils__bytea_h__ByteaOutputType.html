ByteaOutputType

定義

  • 種別: enum
  • 定義箇所: src/include/utils/bytea.h:19

目的

PostgreSQL における BYTEA 型（可変長バイナリデータ）の出力形式を規定する列挙型です。

シグネチャまたは形状

typedef enum
{
	BYTEA_OUTPUT_ESCAPE,
	BYTEA_OUTPUT_HEX,
} ByteaOutputType;

入力と出力

  • BYTEA_OUTPUT_ESCAPE: 従来のバックスラッシュエスケープ形式を指定します。
  • BYTEA_OUTPUT_HEX: 16 進数形式を指定します（現在のデフォルト）。

前提条件と不変条件

  • GUC（Grand Unified Configuration）システムと連携するため、この列挙値は int 型として扱われるグローバル変数 bytea_output と比較されることを前提としています。

アルゴリズムまたは振る舞い

バイナリデータを文字列に変換する関数（例: byteaout）が、現在の bytea_output 設定値を参照し、この列挙型に基づいて変換ロジックを切り替えます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/varlena.c 内のバイナリ入出力関数が、この列挙型を参照して変換処理を分岐させます。
  • 呼び出し先: GUC システムのバリデーションロジック（src/backend/utils/misc/guc_tables.c）が、設定値の妥当性確認に使用します。

関連データ構造

  • bytea_output: ByteaOutputType の値を保持するグローバル変数（int 型として宣言）。

失敗モードと境界条件

  • GUC システム経由で不正な値が設定された場合の挙動については、本ソース excerpt には記載がありません。

保守担当者向けメモ

  • bytea_output 変数は int として宣言されていますが、コード内では ByteaOutputType 列挙値との比較が前提となっています。
  • 出力形式の互換性に直接影響するため、変更時には varlena.c の変換ロジックへの影響を考慮してください。
