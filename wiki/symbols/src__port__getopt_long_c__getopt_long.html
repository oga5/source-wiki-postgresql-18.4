getopt_long

定義

  • 種別: function
  • 定義箇所: src/port/getopt_long.c:59

目的

POSIX 標準の getopt に加え、GNU スタイルのロングオプション（--option）の解析をサポートし、プラットフォーム間の差異を吸収する。

シグネチャまたは形状

int getopt_long(int argc, char *const argv[], const char *optstring, const struct option *longopts, int *longindex);

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数の数と配列。
  • optstring: 許可するショートオプションの文字列。
  • longopts: ロングオプションの定義配列。
  • longindex: 一致したロングオプションのインデックスを格納するポインタ（任意）。
  • **出力**:
  • 成功時: オプション文字（ショート）または longopts[i].val（ロング）。flag が設定されている場合は 0。
  • 終了時: -1。
  • エラー時: BADCH または BADARG。

前提条件と不変条件

  • argv は破壊的に操作され、非オプション引数は末尾へ移動される。
  • static 変数（place, nonopt_start 等）を使用しているため、スレッドセーフではない。
  • optind 等のグローバル変数は、解析状態を保持するために使用される。

アルゴリズムまたは振る舞い

1. **初期化**: place ポインタで現在の引数位置を追跡。 2. **非オプションの再配置**: 引数が - で始まらない場合、それを argv の末尾へ移動させ、すべてのオプションが前方に集まるように再配置する。 3. **ロングオプション解析**: -- で始まる引数を検知し、longopts と照合。引数の有無を確認し、必要に応じて optarg を更新。 4. **ショートオプション解析**: - で始まる単一文字を optstring と照合し、引数を処理。 5. **終了**: 全ての引数を処理し終えるか、非オプション引数のみになった時点で -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/oid2name/oid2name.c, contrib/vacuumlo/vacuumlo.c, src/bin/initdb/initdb.c, src/bin/pg_amcheck/pg_amcheck.c 等の PostgreSQL ユーティリティ。
  • **呼び出し先**: strcspn, strlen, strncmp, strchr, fprintf（標準ライブラリ関数）。

関連データ構造

  • struct option: ロングオプションの定義（getopt_long.h で定義）。
  • グローバル変数: optarg, optind, opterr, optopt。

失敗モードと境界条件

  • optstring の先頭が : の場合、エラーメッセージを出力せずに BADARG を返す。
  • 必須引数が不足している場合、opterr が真であればエラーメッセージを出力し、BADCH を返す。
  • 不明なオプションが指定された場合、BADCH を返す。

保守担当者向けメモ

  • 本関数は argv を書き換えるため、呼び出し元はその副作用を許容する必要がある。
  • static 変数による状態保持のため、同一プロセス内での複数回呼び出しや並列実行には注意が必要。
