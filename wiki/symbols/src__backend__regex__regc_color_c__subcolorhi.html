subcolorhi

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:366

目的

カラーマップ内の特定のエントリに対して、必要に応じて新しいサブカラーを割り当て、文字カウントを更新する。

シグネチャまたは形状

static color subcolorhi(struct colormap *cm, color *pco)

入力と出力

  • **入力**:
  • cm: カラーマップ構造体へのポインタ。
  • pco: 現在の色（color 型）を指すポインタ。
  • **出力**:
  • 成功時: 新しく割り当てられた、または既存のサブカラー（color 型）。
  • 失敗時: COLORLESS（CISERR() が真の場合）。

前提条件と不変条件

  • pco は有効なカラーマップエントリを指している必要がある。
  • newsub が呼び出され、カラーマップの整合性が維持される。
  • CISERR() によるエラーチェックが成功していること。

アルゴリズムまたは振る舞い

1. pco が指す現在の色 co を取得する。 2. newsub(cm, co) を呼び出し、新しいサブカラー sco を生成または取得する。 3. エラーが発生した場合は COLORLESS を返す。 4. co == sco の場合（既にサブカラー化されている）、現在の色をそのまま返す。 5. それ以外の場合、元の色 co の文字カウント（nuchrs）を減らし、新しいサブカラー sco の文字カウントを増やす。 6. *pco を新しいサブカラー sco で更新し、sco を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_color.c 内の複数箇所（行598, 896など）。
  • src/backend/regex/regcomp.c:117（プロトタイプ宣言）。
  • **呼び出し先**:
  • newsub
  • CISERR()（マクロ）

関連データ構造

  • colormap: カラーマップ全体を管理する構造体。
  • colordesc: カラーマップ内の各色に関する記述子（nuchrs を保持）。

失敗モードと境界条件

  • newsub が失敗した場合、COLORLESS を返す。
  • 既にサブカラーである場合は、重複処理を避けて即座にリターンする。

保守担当者向けメモ

  • 本関数はカラーマップの動的な再構成を行うため、nuchrs の増減が正しく行われているか注意すること。
  • newsub の実装変更が本関数の挙動に直接影響を与える。
