StartupProcessMain

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:215

目的

PostgreSQLの起動プロセス（Startupプロセス）のメインエントリポイントとして、データベースの初期化およびWALリプレイ（リカバリ）を実行し、完了後にプロセスを終了させる。

シグネチャまたは形状

void StartupProcessMain(const void *startup_data, size_t startup_data_len)

入力と出力

  • **入力**:
  • startup_data: 起動データ（現在は使用されていない）。
  • startup_data_len: データの長さ。0 であることが前提。
  • **出力**: なし（正常終了時は proc_exit(0) を呼び出す）。

前提条件と不変条件

  • startup_data_len は必ず 0 であること。
  • Postmasterからフォークされた直後の状態であり、シグナルはブロックされていること。

アルゴリズムまたは振る舞い

1. **初期化**: MyBackendType を B_STARTUP に設定し、AuxiliaryProcessMainCommon を実行。 2. **終了処理登録**: on_shmem_exit を使用して StartupProcExit を登録。 3. **シグナル設定**: pqsignal を用いて、SIGHUP、SIGTERM、SIGUSR2等のハンドラを設定し、SIGINTやSIGPIPEを無視するよう構成。 4. **タイマー設定**: スタンバイモード用のデッドロック、タイムアウト、ロックタイムアウトの各ハンドラを登録。 5. **シグナル解除**: sigprocmask により、フォーク時にブロックされていたシグナルを解除。 6. **リカバリ実行**: StartupXLOG() を呼び出し、WALリプレイによる整合性確保を行う。 7. **終了**: リカバリ完了後、proc_exit(0) を呼び出して正常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c（Postmasterによるプロセス起動時）。
  • **呼び出し先**:
  • AuxiliaryProcessMainCommon, on_shmem_exit, pqsignal, InitializeTimeouts, RegisterTimeout, sigprocmask, StartupXLOG, proc_exit。

関連データ構造

  • B_STARTUP（バックエンドタイプ）
  • StartupProcExit（終了時コールバック）
  • StartupProcSigHupHandler, StartupProcShutdownHandler, StartupProcTriggerHandler（シグナルハンドラ）

失敗モードと境界条件

  • startup_data_len != 0 の場合、Assert により異常終了する。
  • リカバリ処理（StartupXLOG）の失敗は、PostgreSQLの起動失敗として扱われる。

保守担当者向けメモ

  • 本関数はメインループを持たず、リカバリ完了後に終了する設計である。
  • シグナルハンドラの登録順序や解除タイミングは、Postmasterとの連携において重要である。
