DateTimeErrorExtra

定義

  • 種別: struct
  • 定義箇所: src/include/utils/datetime.h:290

目的

日付・時刻の解析処理において発生したエラーの詳細情報を保持し、エラーメッセージ生成時にタイムゾーン名や省略形などのコンテキスト情報を提供するために使用されます。

シグネチャまたは形状

typedef struct DateTimeErrorExtra
{
    const char *dtee_timezone;    /* 不正なタイムゾーン名 */
    const char *dtee_abbrev;      /* 関連するタイムゾーン省略形 */
} DateTimeErrorExtra;

入力と出力

本構造体は関数引数として渡され、エラー発生時にその内容が参照されます。

  • 入力: DTERR_BAD_TIMEZONE や DTERR_BAD_ZONE_ABBREV などのエラーコードに関連する文字列ポインタ。
  • 出力: なし（構造体メンバへの書き込みおよび読み取り）。

前提条件と不変条件

  • dtee_timezone は DTERR_BAD_TIMEZONE または DTERR_BAD_ZONE_ABBREV エラー時に設定されることが想定されています。
  • dtee_abbrev は DTERR_BAD_ZONE_ABBREV エラー時に設定されることが想定されています。

アルゴリズムまたは振る舞い

日付・時刻の解析関数（DecodeDateTime 等）がエラーを検出した際、呼び出し元で本構造体を初期化し、エラー詳細を格納します。その後、DateTimeParseError 関数等に渡されることで、エラー報告の文脈情報として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照/初期化）**:
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/utils/adt/date.c
  • src/backend/utils/adt/datetime.c
  • **呼び出し先**:
  • src/backend/utils/adt/datetime.c 内の DateTimeParseError 関数が、エラー報告の引数として本構造体を受け取ります。

関連データ構造

  • src/include/utils/datetime.h: 本構造体が定義されているヘッダーファイル。
  • src/backend/utils/adt/datetime.c: 本構造体を利用する解析ロジックの実装ファイル。

失敗モードと境界条件

  • 構造体内のポインタが NULL の場合、エラーメッセージ生成時に適切なハンドリングが行われない可能性があります。
  • エラーコードと構造体内の設定値が不整合（例：DTERR_BAD_TIMEZONE なのに dtee_abbrev のみを設定）である場合、エラーメッセージが不正確になる可能性があります。

保守担当者向けメモ

  • 本構造体はエラー報告の補助的な役割を担います。新しいエラー型を追加する際は、本構造体で保持すべき情報が不足していないか確認してください。
  • src/backend/utils/adt/datetime.c 内の DateTimeParseError がどのように各メンバを参照しているかを確認し、整合性を維持してください。
