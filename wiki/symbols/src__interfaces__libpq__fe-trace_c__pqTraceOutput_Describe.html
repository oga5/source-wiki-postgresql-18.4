pqTraceOutput_Describe

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:310

目的

PostgreSQL プロトコルにおける Describe メッセージの内容をデバッグ用ストリームへ出力する。

シグネチャまたは形状

static void pqTraceOutput_Describe(FILE *f, const char *message, int *cursor)

入力と出力

  • f: 出力先の FILE ストリーム。
  • message: 解析対象のメッセージデータを含むバッファ。
  • cursor: 現在の解析位置を示すポインタ。関数実行後、読み取ったバイト数分だけ進められる。
  • 出力: なし（副作用として f にテキストを出力）。

前提条件と不変条件

  • message および cursor は有効なメモリ領域を指していること。
  • cursor はメッセージ内の正しい読み取り開始位置を指していること。

アルゴリズムまたは振る舞い

1. ストリーム f に "Describe" という文字列とタブ文字を出力する。 2. pqTraceOutputByte1 を呼び出し、メッセージ内の 1 バイト（型指定子）を出力する。 3. pqTraceOutputString を呼び出し、メッセージ内の文字列（対象オブジェクト名）を出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-trace.c 内のメッセージディスパッチャ（pqTraceOutputMessage 等から呼び出される）。
  • 呼び出し先: pqTraceOutputByte1, pqTraceOutputString。

関連データ構造

  • FILE: 標準ライブラリの出力ストリーム。

失敗モードと境界条件

  • f が無効な場合、fprintf により書き込みエラーが発生する可能性がある。
  • message の内容がプロトコル仕様と一致しない場合、cursor が不正な位置へ進む可能性がある。

保守担当者向けメモ

  • 本関数はプロトコル v3 の Describe メッセージ専用である。
  • メッセージフォーマットの変更時には、pqTraceOutputByte1 および pqTraceOutputString の呼び出し順序がプロトコル定義と一致しているか確認すること。
