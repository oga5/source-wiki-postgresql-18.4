getpeereid

定義

  • 種別: function
  • 定義箇所: src/port/getpeereid.c:32

目的

UNIXドメインソケットの接続相手のユーザーID（UID）およびグループID（GID）を取得するためのポータブルなラッパー関数です。OS間で異なるソケット認証メカニズムを抽象化し、PostgreSQLのバックエンドおよびフロントエンドがプラットフォーム固有の実装を意識せずに接続元情報を取得できるようにします。

シグネチャまたは形状

int getpeereid(int sock, uid_t *uid, gid_t *gid);

入力と出力

  • **入力**:
  • sock: 認証情報を取得する対象のソケットディスクリプタ。
  • uid: 取得したUIDを格納するメモリ領域へのポインタ。
  • gid: 取得したGIDを格納するメモリ領域へのポインタ。
  • **出力**:
  • 成功時: 0 を返し、uid および gid 引数に値が書き込まれる。
  • 失敗時: -1 を返し、必要に応じて errno が設定される。

前提条件と不変条件

  • sock は有効なUNIXドメインソケットである必要があります。
  • Solaris環境で getpeerucred を使用する場合、内部で割り当てられた ucred_t 構造体は関数内で適切に解放されます。

アルゴリズムまたは振る舞い

コンパイル時の定義に基づき、以下のいずれかの手法で認証情報を取得します。 1. **Linux (SO_PEERCRED)**: getsockopt を使用して struct ucred から情報を抽出します。 2. **Debian/FreeBSD系 (LOCAL_PEERCRED)**: getsockopt を使用して struct xucred から情報を抽出します。 3. **Solaris (HAVE_GETPEERUCRED)**: getpeerucred を呼び出し、返された ucred_t からUID/GIDを取得後、ucred_free でメモリを解放します。 4. **その他**: 実装が存在しないプラットフォームでは errno を ENOSYS に設定し -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth.c
  • src/interfaces/libpq/fe-connect.c
  • **呼び出し先**:
  • プラットフォーム固有のシステムコール（getsockopt または getpeerucred）を呼び出します。

関連データ構造

  • uid_t, gid_t: ユーザーIDおよびグループIDを保持する標準型。
  • struct ucred (Linux用)
  • struct xucred (Debian/FreeBSD用)
  • ucred_t (Solaris用)

失敗モードと境界条件

  • ソケットオプションの取得失敗や、Solarisでの認証情報取得失敗時には -1 を返します。
  • Solarisにおいて取得したUIDまたはGIDが (uid_t)-1 または (gid_t)-1 の場合、エラーとして扱われます。
  • サポートされていないプラットフォームでは ENOSYS が返されます。

保守担当者向けメモ

  • 本関数はOSの認証機能のラッパーです。OS側でサポートがない場合、ローカル接続認証が制限される可能性があります。
  • Solarisの getpeerucred を使用する際は、ucred_t 構造体のメモリリークを防ぐため、必ず ucred_free() が呼び出されることを確認してください。
