ParseVariableBool

定義

  • 種別: function
  • 定義箇所: src/bin/psql/variables.c:108

目的

文字列形式の変数値（true/false, yes/no, on/off, 1/0）をブール値に変換し、検証を行う。

シグネチャまたは形状

bool ParseVariableBool(const char *value, const char *name, bool *result);

入力と出力

  • **入力**:
  • value: 解析対象の文字列。NULL の場合は空文字列として扱われる。
  • name: エラーメッセージに使用する変数名（NULL 指定可能）。
  • result: 変換結果を格納する bool 型へのポインタ。
  • **出力**:
  • 戻り値: 変換に成功した場合は true、失敗した場合は false を返す。
  • result: 変換成功時のみ、解析結果が書き込まれる。

前提条件と不変条件

  • value が NULL の場合、内部的に空文字列として処理される。
  • 変換に失敗した場合、result が指す値は変更されない。
  • name が指定されている場合、変換失敗時に pg_log_error を通じてエラーメッセージが出力される。

アルゴリズムまたは振る舞い

1. value が NULL ならば空文字列に置換する。 2. pg_strncasecmp または pg_strcasecmp を使用し、以下の文字列をブール値にマッピングする。

  • true, yes, on, 1 → true
  • false, no, off, 0 → false

    • 3. on/off の判定では、最小2文字以上の長さで比較を行う。 4. いずれにも該当しない場合、valid を false に設定し、name があればエラーログを出力する。 5. 最終的な valid の状態を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c および src/bin/psql/startup.c 内の複数の箇所で、設定値のパースに使用されている。
  • **呼び出し先**: pg_strncasecmp, pg_strcasecmp, pg_log_error（いずれも外部ユーティリティ）。

関連データ構造

  • なし。

失敗モードと境界条件

  • 未定義の文字列が渡された場合、false を返し、result は更新されない。
  • value が空文字列の場合、変換失敗として扱われる。

保守担当者向けメモ

  • 本関数は psql の設定変数管理における型変換ユーティリティの一部である。
  • 新たなブール値表現を追加する場合は、比較ロジックの優先順位と pg_strncasecmp の長さ指定に注意すること。
