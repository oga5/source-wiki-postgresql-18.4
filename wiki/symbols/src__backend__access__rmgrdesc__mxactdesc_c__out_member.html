out_member

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/mxactdesc.c:19

目的

MultiXactMember 構造体の内容（トランザクションIDおよびロック状態）を人間が読める文字列に変換し、指定されたバッファへ書き出す。

シグネチャまたは形状

static void out_member(StringInfo buf, MultiXactMember *member)

入力と出力

  • buf (入出力): 変換後の文字列が追記される StringInfo バッファ。
  • member (入力): 解析対象の MultiXactMember 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • buf は初期化済みであること。
  • member は有効なメモリ領域を指していること。
  • member->status は MultiXactStatus 列挙型のいずれか、または未知の値であること。

アルゴリズムまたは振る舞い

1. member->xid を文字列として buf に書き込む。 2. member->status の値に基づき、対応するロック状態の略称（例: (keysh), (sh), (forupd) 等）を buf に追記する。 3. 定義されていないステータス値の場合は (unk) を追記する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/rmgrdesc/mxactdesc.c 内の multixact_desc 関数（xlrec->members の各要素を処理する際に呼び出される）。
  • 呼び出し先: appendStringInfo および appendStringInfoString（PostgreSQLの文字列操作ユーティリティ）。

関連データ構造

  • MultiXactMember: マルチトランザクションのメンバ情報を保持する構造体。
  • StringInfo: ログ出力用の動的文字列バッファ。
  • MultiXactStatus: ロック状態を定義する列挙型。

失敗モードと境界条件

  • member->status が MultiXactStatus の定義範囲外である場合、default ケースにより (unk) と出力される。

保守担当者向けメモ

  • 本関数内の switch 文は MultiXactStatus 列挙型と密結合している。ロック状態の定義が変更された場合は、必ず本関数の switch 文を更新すること。
