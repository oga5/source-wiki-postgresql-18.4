statapprox_heap

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstatapprox.c:58

目的

ヒープテーブルの統計情報（タプル数、デッドタプル数、空き容量など）を、可視性マップを活用して高速に推定する。

シグネチャまたは形状

static void statapprox_heap(Relation rel, output_type *stat)

入力と出力

  • **入力**:
  • rel: 統計対象のヒープリレーション。
  • stat: 推定結果を格納する output_type 構造体へのポインタ。
  • **出力**:
  • なし（stat 構造体の各フィールドが更新される）。

前提条件と不変条件

  • rel はヒープテーブルであること。
  • 一時テーブル（RELATION_IS_OTHER_TEMP）は、セッションローカルなバッファ制約によりアクセス不可。
  • VM_ALL_VISIBLE ビットが立っているページは、実際にはタプルが更新されていてもスキャン対象外となるため、統計に誤差が生じる。

アルゴリズムまたは振る舞い

1. **初期化**: GetOldestNonRemovableTransactionId でトランザクションの境界を取得し、バルク読み込み用のバッファ戦略を準備する。 2. **ページ走査**: 全ページをループし、以下のいずれかの処理を行う。

  • **可視性マップ判定**: VM_ALL_VISIBLE が真の場合、FSMから空き容量を取得し、タプル長を近似してスキップする。
  • **詳細スキャン**: それ以外の場合、ページを読み込み、HeapTupleSatisfiesVacuum を用いて各タプルをライブ/デッドに分類して集計する。

    • 3. **外挿**: 未スキャンページ分を vac_estimate_reltuples で推定し、全体のタプル数を算出する。 4. **統計算出**: 全体長に対する各統計値の割合（パーセンテージ）を計算する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgstattuple/pgstatapprox.c 内の関数（pgstattuple_approx_internal 等から呼び出される）。
  • **呼び出し先**: GetOldestNonRemovableTransactionId, GetAccessStrategy, RelationGetNumberOfBlocks, VM_ALL_VISIBLE, GetRecordedFreeSpace, ReadBufferExtended, PageGetExactFreeSpace, HeapTupleSatisfiesVacuum, vac_estimate_reltuples 等。

関連データ構造

  • output_type: 推定結果を保持する構造体。
  • Relation: 対象テーブルのメタデータ。
  • Buffer: ページ読み込み用バッファ。

失敗モードと境界条件

  • nblocks == 0 の場合、パーセンテージ計算をスキップする。
  • HeapTupleSatisfiesVacuum が予期しない結果を返した場合、elog(ERROR) を発生させる。
  • stat->tuple_count が負にならないよう Max(stat->tuple_count, 0) で保護している。

保守担当者向けメモ

  • 本関数は近似値を提供するため、正確な値が必要な場合は pgstattuple を使用すること。
  • lazy_scan_heap と異なり、特殊なケースやエラーを無視して高速化を優先している。
