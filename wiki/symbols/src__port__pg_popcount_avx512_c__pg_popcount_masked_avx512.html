pg_popcount_masked_avx512

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_avx512.c:166

目的

Intel AVX-512 命令セットを使用し、メモリバッファ内の各バイトに対して指定されたマスクを適用した上で、ビットがセットされている総数を高速にカウントする。

シグネチャまたは形状

uint64 pg_popcount_masked_avx512(const char *buf, int bytes, bits8 mask)

入力と出力

  • **入力**:
  • buf: 対象データの開始ポインタ。
  • bytes: 処理するバイト数。
  • mask: 各バイトに対して論理積（AND）をとるための 8 ビットマスク。
  • **出力**:
  • マスク適用後のビットカウント合計値を uint64 型で返す。

前提条件と不変条件

  • 本関数は USE_AVX512_POPCNT_WITH_RUNTIME_CHECK が定義された環境でのみ使用可能。
  • 実行時に AVX-512 命令セット（VPOPCNTDQ, BW）が利用可能である必要がある。
  • buf が指すメモリ領域は bytes 分だけ有効であること。

アルゴリズムまたは振る舞い

1. **アライメント調整**: 入力バッファを 64 バイト（512 ビット）境界に合わせ、最初のチャンクをマスク付きでロードする。 2. **ループ処理**:

  • 512 ビット単位でデータをロードし、mask との論理積を計算する。
  • _mm512_popcnt_epi64 を用いて各 8 ビットのビット数をカウントし、累積レジスタ accum に加算する。

    • 3. **終了処理**: バッファの末尾が 512 ビット境界に満たない場合、残りのバイトに対してマスクを適用して処理する。 4. **集計**: _mm512_reduce_add_epi64 を使用して、ZMM レジスタ内の累積値を最終的な合計値に変換する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/pg_bitutils.c 内の pg_popcount_masked_optimized に関数ポインタとして代入されている。
  • **呼び出し先**: AVX-512 イントリンシック関数（_mm512_setzero_si512, _mm512_set1_epi8, _mm512_maskz_loadu_epi8, _mm512_and_si512, _mm512_popcnt_epi64, _mm512_add_epi64, _mm512_load_si512, _mm512_reduce_add_epi64）。

関連データ構造

  • __m512i: AVX-512 512 ビットレジスタ型。
  • __mmask64: 64 ビットのマスク型。

失敗モードと境界条件

  • bytes が 0 の場合、ループは実行されず 0 を返す。
  • アライメントされていないメモリ境界に対しては、_mm512_maskz_loadu_epi8 を使用して安全にロードを行う。
  • 最終チャンクの処理において、tail_idx を用いてバッファ長を超えないようにマスクを調整する。

保守担当者向けメモ

  • 本関数は pg_attribute_target により AVX-512 命令セットを有効化している。
  • 汎用的な pg_popcount のバックエンドとして機能するため、直接呼び出すのではなく pg_bitutils.h 経由のインターフェースを介して利用すること。
