TocEntry

定義

  • 種別: typedef
  • 定義箇所: src/bin/pg_dump/parallel.h:24

目的

pg_dump および pg_restore における並列処理において、ジョブ完了時にリーダープロセスで実行されるコールバック関数の型を定義します。

シグネチャまたは形状

typedef void (*ParallelCompletionPtr) (ArchiveHandle *AH,
                                       TocEntry *te,
                                       int status,
                                       void *callback_data);

入力と出力

  • **入力**:
  • ArchiveHandle *AH: アーカイブの制御ハンドル。
  • TocEntry *te: 処理対象のアーカイブエントリ。
  • int status: ジョブの実行結果ステータス。
  • void *callback_data: コールバック関数に渡される任意のユーザーデータ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本型定義は pg_backup_archiver.h に依存しています。
  • コールバック関数は、リーダープロセスがワーカーの完了を待機する際に呼び出されます。

アルゴリズムまたは振る舞い

並列処理の同期フェーズにおいて、リーダープロセスが WaitForWorkers を介してワーカーの完了を監視する際、ジョブ完了の通知とともに本関数が実行されます。これにより、リーダープロセス側でアーカイブの状態更新や後処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/parallel.c 内の並列処理制御ロジック（ワーカーの完了監視時）。
  • **呼び出し先**: 登録された具体的なコールバック関数（実装詳細は pg_backup_archiver.c 等のアーカイブ制御ロジックに依存）。

関連データ構造

  • ArchiveHandle: アーカイブ処理の基盤構造体。
  • TocEntry: ダンプ/リストア対象の個別のエントリ。
  • ParallelState: 並列処理全体の管理構造体。

失敗モードと境界条件

  • コールバック関数内でのエラーハンドリングは、呼び出し側の実装に依存します。
  • status 引数には、ワーカープロセスでの実行結果が反映されます。

保守担当者向けメモ

  • 本シンボルは parallel.h で定義されていますが、実際のコールバック処理は pg_backup_archiver.c などの上位モジュールで定義・登録されます。
  • Windows 環境ではスレッドベースの並列処理となるため、コールバック実行時のスレッドセーフティに留意してください。
