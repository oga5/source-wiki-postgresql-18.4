gbtree_ninfo

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_utils_num.h:33

目的

GiST 索引において数値型（整数、浮動小数点数、日付等）を扱うためのメタデータおよび関数ポインタを保持する記述子構造体です。各データ型固有の比較・計算ロジックを抽象化し、共通のユーティリティ関数から利用可能にします。

シグネチャまたは形状

typedef struct {
    enum gbtree_type t;
    int32 size;
    int32 indexsize;
    bool (*f_gt) (const void *, const void *, FmgrInfo *);
    bool (*f_ge) (const void *, const void *, FmgrInfo *);
    bool (*f_eq) (const void *, const void *, FmgrInfo *);
    bool (*f_le) (const void *, const void *, FmgrInfo *);
    bool (*f_lt) (const void *, const void *, FmgrInfo *);
    int  (*f_cmp) (const void *, const void *, FmgrInfo *);
    float8 (*f_dist) (const void *, const void *, FmgrInfo *);
} gbtree_ninfo;

入力と出力

本構造体自体はデータ保持用であり、各関数ポインタは以下の入出力を持ちます。

  • **入力**: 比較対象のデータポインタ（void *）および FmgrInfo *。
  • **出力**: 比較関数は bool または int、距離関数は float8 を返します。

前提条件と不変条件

  • size が 0 の場合、可変長型であることを示します。
  • indexsize はインデックスページに格納されるデータサイズであり、通常は範囲（下限と上限）を保持するため、型サイズの2倍以上である必要があります。

アルゴリズムまたは振る舞い

各データ型に対応する tinfo インスタンスとして静的に定義され、btree_gist の各メソッド（compress, union, consistent 等）がこの構造体を参照することで、型に応じた適切な比較や計算を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の各ファイルで tinfo として静的定義され、モジュール内の各 GiST メソッドから参照されます。
  • btree_bool.c, btree_cash.c, btree_date.c, btree_enum.c, btree_float4.c, btree_float8.c, btree_inet.c, btree_int2.c, btree_int4.c, btree_int8.c, btree_interval.c, btree_macaddr.c
  • **呼び出し先**: 構造体内の関数ポインタは、各データ型に対応する比較関数や距離計算関数を指します。

関連データ構造

  • gbtree_type: 保持するデータの型を識別する列挙型。
  • GBT_NUMKEY: 数値型キーの内部表現。

失敗モードと境界条件

  • indexsize の設定が不適切な場合、インデックスページへの格納時にデータ破損やオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 新しい数値型を btree_gist に追加する際は、本構造体を適切に初期化して各関数に渡す必要があります。
  • 構造体内の関数ポインタは、PostgreSQL の関数マネージャ（FmgrInfo）を介して呼び出されるため、実装時には適切なシグネチャを維持してください。
