ExecEvalSubroutineTemplate

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_types.c:96

目的

PostgreSQL の JIT コンパイルにおいて、式評価サブルーチンの関数シグネチャを LLVM IR 上で再現するためのテンプレートとして機能します。

シグネチャまたは形状

void ExecEvalSubroutineTemplate(ExprState *state, struct ExprEvalStep *op, ExprContext *econtext);

入力と出力

  • 入力:
  • state: 式評価の状態を保持する ExprState 構造体へのポインタ。
  • op: 実行対象のステップを定義する ExprEvalStep 構造体へのポインタ。
  • econtext: 式の評価コンテキストを保持する ExprContext 構造体へのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • 本関数は実行時に直接呼び出されるものではなく、LLVM が型情報を抽出するためのメタデータとして存在します。
  • AssertVariableIsOfType マクロにより、自身の型が ExecEvalSubroutine 型と一致することが検証されます。

アルゴリズムまたは振る舞い

1. 呼び出されると、自身の型が ExecEvalSubroutine 型であることをアサーションで確認します。 2. 本関数自体はロジックを実行せず、LLVM JIT エンジンがこの関数定義を読み取ることで、生成されるコードの関数シグネチャを決定します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/jit/llvm/llvmjit.c: LLVMGetNamedFunction を介して関数定義を取得。
  • src/backend/jit/llvm/llvmjit_expr.c: LLVMGetFunctionType を介して型情報を取得。
  • 呼び出し先:
  • AssertVariableIsOfType: 型の整合性チェックに使用。

関連データ構造

  • ExprState: 式評価の状態管理。
  • ExprEvalStep: 実行ステップの定義。
  • ExprContext: 評価コンテキスト。

失敗モードと境界条件

  • 本関数はメタデータ提供用であるため、実行時の失敗モードは定義されていません。ただし、LLVM が本関数を正しく認識できない場合、JIT コンパイルされたコードの生成に失敗します。

保守担当者向けメモ

  • 本関数は static にしないでください。static にすると Clang が不要なコードと判断し、ビットコードから削除してしまうため、LLVM が型情報を取得できなくなります。
  • 新しい式評価の型を追加する際は、このテンプレートを参考にシグネチャを定義してください。
