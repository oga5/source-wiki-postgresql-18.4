big5_to_euc_tw

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:67

目的

BIG5 エンコーディングから EUC_TW エンコーディングへの文字列変換を行う、PostgreSQL の動的ロード可能な変換関数です。

シグネチャまたは形状

Datum big5_to_euc_tw(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_CSTRING(2): 変換元文字列 (src)
  • PG_GETARG_CSTRING(3): 変換先バッファ (dest)
  • PG_GETARG_INT32(4): 入力文字列の長さ (len)
  • PG_GETARG_BOOL(5): エラーハンドリングフラグ (noError)
  • **出力**:
  • Datum 型として変換後のバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_BIG5, PG_EUC_TW) により、変換元が BIG5、変換先が EUC_TW であることが保証されます。

アルゴリズムまたは振る舞い

1. 引数から変換に必要なパラメータを取得します。 2. big52euc_tw 関数を呼び出し、実際のバイト変換ロジックを実行します。 3. 変換されたバイト数を戻り値として返します。 4. noError が true の場合、変換エラーが発生しても例外を投げず、処理を中断して変換済みのバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（SQL レベルからの呼び出し）。PG_FUNCTION_INFO_V1(big5_to_euc_tw) により登録されています。
  • **呼び出し先**: big52euc_tw（実際の変換ロジックを実行する内部関数）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約で使用される引数構造体。

失敗モードと境界条件

  • 入力文字列が不正なエンコーディングである場合、noError フラグの設定に応じて、例外を発生させるか、処理を中断して変換済みバイト数を返します。

保守担当者向けメモ

  • 本関数はインターフェース層であり、具体的な変換ロジックは big52euc_tw に依存しています。
  • 変換ロジックの詳細は src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c 内の big52euc_tw 実装を参照してください。
