HeapTupleSetHintBits

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_visibility.c:140

目的

タプルヘッダーの t_infomask にトランザクション状態（ヒントビット）を記録し、将来の可視性判定コストを削減するための公開インターフェースを提供します。

シグネチャまたは形状

void HeapTupleSetHintBits(HeapTupleHeader tuple, Buffer buffer, uint16 infomask, TransactionId xid)

入力と出力

  • tuple: 更新対象のタプルヘッダー。
  • buffer: タプルが格納されているバッファ。
  • infomask: 設定するビットマスク（例: HEAP_XMAX_COMMITTED）。
  • xid: ヒントビットに関連付けるトランザクションID。
  • 出力: なし（副作用としてバッファの内容を更新）。

前提条件と不変条件

  • 呼び出し元は、対象の buffer に対して少なくとも共有ロック（Shared Lock）を保持している必要があります。
  • 内部で SetHintBits を呼び出すため、同関数の不変条件（バッファのダーティ化など）に従います。

アルゴリズムまたは振る舞い

内部関数 SetHintBits を呼び出すラッパーです。タプルの t_infomask を更新し、トランザクションのコミットまたはアボート状態をタプルヘッダーにキャッシュすることで、次回の可視性判定時に重いCLOG参照を回避可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heapam.c（HeapTupleSetHintBits を直接呼び出し）。
  • 呼び出し先: SetHintBits（同ファイル内の内部関数）。

関連データ構造

  • HeapTupleHeader: タプルのメタデータ構造。
  • Buffer: PostgreSQLのバッファ管理構造。
  • TransactionId: トランザクション識別子。

失敗モードと境界条件

  • バッファがロックされていない状態で呼び出した場合、競合状態やデータ破損のリスクがあります。

保守担当者向けメモ

  • 本関数は SetHintBits のエクスポート版です。ロジックの変更が必要な場合は、主に SetHintBits 側を修正してください。
  • 可視性判定の最適化において重要な役割を果たすため、変更時はトランザクション状態の整合性に注意してください。
