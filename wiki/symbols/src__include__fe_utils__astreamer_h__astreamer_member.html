astreamer_member

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/astreamer.h:79

目的

アーカイブ（tar等）内の各ファイルやディレクトリに関するメタデータを保持し、ストリーム処理パイプラインを通じて伝達するために使用されます。

シグネチャまたは形状

typedef struct
{
	char		pathname[MAXPGPATH];
	pgoff_t		size;
	mode_t		mode;
	uid_t		uid;
	gid_t		gid;
	bool		is_regular;
	bool		is_directory;
	bool		is_symlink;
	char		linktarget[MAXPGPATH];
} astreamer_member;

入力と出力

  • **入力**: アーカイブのパーサー（astreamer_tar_parser 等）によって抽出されたメタデータが各フィールドに格納されます。
  • **出力**: パイプラインの後続ステージ（astreamer_extractor や検証用ストリーム等）が、この構造体の情報を参照してファイル作成や検証を行います。

前提条件と不変条件

  • pathname および linktarget は MAXPGPATH バイト以内の文字列である必要があります。
  • is_regular、is_directory、is_symlink の各フラグは、特殊なファイルタイプの場合、すべて false になる可能性があります。

アルゴリズムまたは振る舞い

アーカイブ内の各メンバーを処理する際、この構造体にメタデータが設定され、パイプラインの各ステージへ順次渡されます。各ステージは、この情報を基にファイルシステム操作やチェックサム計算などの処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元（参照・操作）**:
  • src/bin/pg_basebackup/astreamer_inject.c
  • src/bin/pg_verifybackup/astreamer_verify.c（member_verify_header, astreamer_verify_content, member_compute_checksum 等）
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数呼び出しを直接行うものではありません。

関連データ構造

  • astreamer: 本構造体を保持し、パイプライン処理を行う基底構造体。

失敗モードと境界条件

  • MAXPGPATH を超えるパス名やリンクターゲットがアーカイブに含まれる場合、バッファオーバーフローや切り捨てのリスクがあります。
  • フラグ（is_regular 等）が適切に設定されていない場合、後続の処理でファイルタイプを誤認する可能性があります。

保守担当者向けメモ

  • 本構造体はメモリコンテキストを持たないため、動的に確保された場合は明示的な解放が必要です。
  • pg_basebackup や pg_verifybackup で頻繁に使用されるため、フィールドの追加や変更を行う際は、これらのツール側の整合性も確認してください。
