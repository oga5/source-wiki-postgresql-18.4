SIInsertDataEntries

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/sinvaladt.c:369

目的

共有キャッシュ無効化メッセージを共有メモリ上の循環バッファに挿入し、全バックエンドに対してメッセージの存在を通知する。

シグネチャまたは形状

void SIInsertDataEntries(const SharedInvalidationMessage *data, int n)
  • data: 挿入するメッセージ配列へのポインタ。
  • n: 挿入するメッセージの数。

入力と出力

  • 入力: SharedInvalidationMessage 型のメッセージ群と、その個数 n。
  • 出力: なし（副作用として共有メモリの状態を更新）。

前提条件と不変条件

  • SInvalWriteLock を排他モードで取得して実行される。
  • 循環バッファの容量（MAXNUMMESSAGES）を超過しないよう、必要に応じて SICleanupQueue が呼び出され、空き領域が確保される。
  • maxMsgNum の更新は msgnumLock（スピンロック）によって保護される。

アルゴリズムまたは振る舞い

1. n が大きい場合に備え、WRITE_QUANTUM 単位で分割して処理を行う。 2. SInvalWriteLock を取得し、バッファの空き容量を確認する。容量不足または閾値超過時には SICleanupQueue(true, nthistime) を呼び出し、古いメッセージを削除する。 3. メッセージを循環バッファの適切なスロットにコピーする。 4. msgnumLock を使用して maxMsgNum を更新し、変更を確定させる。 5. 全バックエンドの ProcState 内の hasMessages フラグを true に設定し、メッセージの存在を通知する。 6. SInvalWriteLock を解放する（この際、メモリバリアが適用される）。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/sinval.c（直接の呼び出しを確認）。
  • 呼び出し先: SICleanupQueue（バッファ管理のため）、LWLockAcquire/LWLockRelease、SpinLockAcquire/SpinLockRelease。

関連データ構造

  • SISeg: 共有メモリ上のメイン構造体。
  • SharedInvalidationMessage: 無効化メッセージのデータ型。
  • ProcState: 各バックエンドの進捗状態を保持する構造体。

失敗モードと境界条件

  • バッファ溢れ: MAXNUMMESSAGES を超える場合、SICleanupQueue が強制的にクリーンアップを実行する。
  • ロック競合: 他のプロセスが SICleanupQueue を実行中である場合、ロック取得まで待機する。

保守担当者向けメモ

  • WRITE_QUANTUM 単位の分割は、ロック保持時間を制限し、他のプロセス（特に追いつこうとしているバックエンド）の応答性を維持するために重要である。
  • SInvalWriteLock の解放時にメモリバリアが保証されるため、hasMessages の更新は他のプロセスから確実に観測される。
