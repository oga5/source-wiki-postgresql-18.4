SendFunctionResult

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/fastpath.c:66

目的

Fastpath（PQfn）経由で実行された関数の結果を、クライアントとの通信プロトコル形式に変換して送信する。

シグネチャまたは形状

static void SendFunctionResult(Datum retval, bool isnull, Oid rettype, int16 format)

入力と出力

  • retval: 関数実行結果の Datum 値。
  • isnull: 結果が NULL かどうかを示すフラグ。
  • rettype: 戻り値のデータ型 OID。
  • format: 戻り値の形式（0: テキスト, 1: バイナリ）。
  • 出力: なし（pq_endmessage を介してクライアントへ送信）。

前提条件と不変条件

  • format が 0 または 1 であること。それ以外の場合はエラーが発生する。
  • isnull が真の場合、プロトコル上で -1 が送信される。

アルゴリズムまたは振る舞い

1. PqMsg_FunctionCallResponse メッセージを開始する。 2. isnull が真であれば、長さとして -1 を送信する。 3. isnull が偽の場合:

  • format == 0 (テキスト): getTypeOutputInfo で型情報を取得し、OidOutputFunctionCall で文字列変換後、pq_sendcountedtext で送信する。
  • format == 1 (バイナリ): getTypeBinaryOutputInfo で型情報を取得し、OidSendFunctionCall でバイト列変換後、サイズとデータを送信する。

    • 4. pq_endmessage を呼び出し、メッセージを確定・送信する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tcop/fastpath.c 内の関数（HandleFunctionRequest 等から呼び出される）。
  • 呼び出し先: pq_beginmessage, getTypeOutputInfo, OidOutputFunctionCall, pq_sendcountedtext, getTypeBinaryOutputInfo, OidSendFunctionCall, pq_sendint32, pq_sendbytes, pq_endmessage, ereport。

関連データ構造

  • StringInfoData: 通信メッセージ構築用バッファ。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • format が 0 または 1 以外の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告する。
  • メモリ割り当て（palloc）は MessageContext で行われるため、呼び出し側での明示的な解放は不要。

保守担当者向けメモ

  • 本関数は SQL を介さない Fastpath 専用の出力処理である。
  • 戻り値の型変換にはシステムカタログ（pg_type）の情報を利用するため、型情報の取得に失敗するケースを考慮する必要がある。
