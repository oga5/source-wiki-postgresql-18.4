QueryCompletion

定義

  • 種別: struct
  • 定義箇所: src/include/tcop/cmdtag.h:29

目的

SQLコマンドの実行完了状態を保持し、クライアントへ送信する CommandComplete メッセージの生成基盤として機能します。

シグネチャまたは形状

typedef struct QueryCompletion
{
	CommandTag	commandTag;
	uint64		nprocessed;
} QueryCompletion;

入力と出力

  • **入力**: commandTag（実行されたコマンドの種類）、nprocessed（処理された行数）。
  • **出力**: 本構造体は BuildQueryCompletionString 等の関数によって参照され、クライアントへの通知文字列（例: "INSERT 0 10"）に変換されます。

前提条件と不変条件

  • nprocessed はコマンド実行によって処理された行数を保持します。
  • commandTag は cmdtaglist.h で定義された有効なコマンド種別である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: InitializeQueryCompletion により構造体がリセットされます。 2. **設定**: コマンド実行後に SetQueryCompletion を用いてタグと行数が記録されます。 3. **文字列化**: BuildQueryCompletionString が本構造体を参照し、プロトコルに応じた完了メッセージ文字列を生成します。

呼び出し元と呼び出し先

  • **呼び出し元（参照・操作）**:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • contrib/sepgsql/hooks.c
  • src/backend/commands/createas.c
  • src/backend/commands/matview.c
  • src/backend/commands/portalcmds.c
  • src/backend/commands/prepare.c
  • src/backend/executor/spi.c
  • src/backend/replication/walsender.c
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • CommandTag: コマンドの種類を識別する列挙型。
  • COMPLETION_TAG_BUFSIZE: 文字列化の際に必要なバッファサイズ（64バイト）。

失敗モードと境界条件

  • BuildQueryCompletionString を使用する際は、バッファオーバーフローを防ぐため COMPLETION_TAG_BUFSIZE 以上の領域を確保する必要があります。
  • INSERT コマンドのタグ生成には、歴史的な理由により "0" を含める特殊なロジックが存在します。

保守担当者向けメモ

  • 新しいSQLコマンドを追加する場合は、cmdtaglist.h を編集してください。
  • 本構造体は tcop サブシステムにおいてクライアント通信の要となるため、変更時は cmdtag.c の文字列生成ロジックとの整合性に注意してください。
