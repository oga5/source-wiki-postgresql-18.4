BrinValues

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_tuple.h:29

目的

BRIN（Block Range Index）インデックス処理において、特定のページ範囲内における各カラムの要約値をメモリ上で保持・管理するために使用されます。

シグネチャまたは形状

typedef struct BrinValues
{
    AttrNumber    bv_attno;
    bool          bv_hasnulls;
    bool          bv_allnulls;
    Datum        *bv_values;
    Datum         bv_mem_value;
    MemoryContext bv_context;
    brin_serialize_callback_type bv_serialize;
} BrinValues;

入力と出力

  • 入力: インデックス対象カラムの属性番号、NULL 状態フラグ、および蓄積された要約値（Datum）。
  • 出力: 構造体内の各フィールドを通じて、インデックスの構築や検索に必要な要約情報を提供します。

前提条件と不変条件

  • bv_hasnulls は範囲内に NULL が存在する場合に真となります。
  • bv_allnulls は範囲内の値がすべて NULL である場合に真となります。
  • bv_context は、bv_values 等のメモリ管理に使用されるコンテキストを指す必要があります。

アルゴリズムまたは振る舞い

BRIN インデックスの構築および検索プロセスにおいて、BrinMemTuple の一部として各カラムの値を保持します。bv_serialize コールバックを介して、メモリ上の表現をディスク格納用の形式へ変換する処理をサポートします。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/pageinspect/brinfuncs.c
  • src/backend/access/brin/brin.c（check_null_keys 等）
  • src/backend/access/brin/brin_bloom.c
  • 呼び出し先:
  • 構造体内の bv_serialize フィールドに割り当てられた関数ポインタが呼び出されます（具体的な関数名はソース抜粋からは特定できません）。

関連データ構造

  • BrinMemTuple: BrinValues の配列を保持し、インデックス処理中のメモリ表現を構成します。
  • Datum: インデックスされる実際の値を保持します。
  • MemoryContext: メモリ割り当てのライフサイクルを管理します。

失敗モードと境界条件

  • bv_values が適切に初期化されていない場合、メモリ不正アクセスのリスクがあります。
  • bv_serialize が設定されていない状態でシリアライズを試みた場合、実行時エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はメモリ上の操作用であり、ディスク上の BrinTuple とは明確に区別して扱う必要があります。
  • bv_hasnulls と bv_allnulls のビットフラグ管理は、brin_tuple.c 内のロジックと整合させる必要があります。
