inet_net_pton_ipv6

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/inet_net_pton.c:427

目的

IPv6 アドレスを人間が読みやすいプレゼンテーション形式（文字列）から、システム内部で扱うネットワークバイトオーダーのバイナリ形式へ変換します。

シグネチャまたは形状

int inet_net_pton_ipv6(const char *src, u_char *dst);

入力と出力

  • **入力**:
  • src: 変換対象の IPv6 アドレス文字列（ヌル終端）。
  • dst: 変換後のバイナリデータを格納するバッファ（16バイト以上を想定）。
  • **出力**:
  • 成功時: 変換されたビット長（プレフィックス長）。
  • 失敗時: -1 を返し、errno を設定する。

前提条件と不変条件

  • dst は少なくとも16バイトのメモリ領域を指している必要があります。
  • 本関数は CIDR 表記を含まない（または省略された）IPv6 アドレスの変換を前提としています。

アルゴリズムまたは振る舞い

1. 内部関数 inet_cidr_pton_ipv6 を呼び出します。 2. その際、第3引数（サイズ）に 16 を固定値として渡します。 3. inet_cidr_pton_ipv6 の処理結果（パースされたバイナリとビット長）をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/inet_net_pton.c 内の pg_inet_net_pton から、アドレスファミリーが IPv6 であり、かつ CIDR ネットワーク用ではない（size が -1 ではない）場合に呼び出されます。
  • **呼び出し先**:
  • inet_cidr_pton_ipv6

関連データ構造

  • u_char: バイナリ形式のアドレスを保持するために使用される型。

失敗モードと境界条件

  • src が不正な IPv6 フォーマットである場合、inet_cidr_pton_ipv6 の仕様に従い -1 を返します。
  • 詳細は inet_cidr_pton_ipv6 のエラーハンドリングに依存します。

保守担当者向けメモ

  • 本関数は inet_cidr_pton_ipv6 への単なるラッパーです。IPv6 のパースロジックに変更を加える場合は、直接 inet_cidr_pton_ipv6 を参照してください。
