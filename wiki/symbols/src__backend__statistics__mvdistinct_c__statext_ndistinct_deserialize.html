statext_ndistinct_deserialize

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mvdistinct.c:249

目的

システムカタログ（pg_statistic_ext_data）に格納されたバイナリ形式（bytea）の多変量NDistinct統計情報を、メモリ上の MVNDistinct 構造体へ復元する。

シグネチャまたは形状

MVNDistinct *statext_ndistinct_deserialize(bytea *data)

入力と出力

  • **入力**: bytea *data - シリアライズされた統計情報のバイナリデータ。
  • **出力**: MVNDistinct * - 復元された統計情報構造体へのポインタ。入力が NULL の場合は NULL を返す。

前提条件と不変条件

  • 入力データは STATS_NDISTINCT_MAGIC マジックナンバーと STATS_NDISTINCT_TYPE_BASIC 型を持つ必要がある。
  • 復元された各アイテムの属性数（nattributes）は 2 以上 STATS_MAX_DIMENSIONS 以下であること。
  • 処理終了時、入力された bytea の全バイトが消費されていること（Assert による検証）。

アルゴリズムまたは振る舞い

1. data が NULL か確認し、ヘッダーサイズが最小要件を満たすか検証する。 2. バイナリデータからマジックナンバー、型、アイテム数（nitems）を読み取り、整合性をチェックする。 3. MVNDistinct 構造体および内部の MVNDistinctItem 配列をメモリ確保する。 4. 各アイテムについて、NDistinct値（double）、属性数（int）、および属性番号の配列（AttrNumber）を順次コピーする。 5. 最後に、消費したデータ長が元の bytea サイズと一致することを確認して構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/mvdistinct.c 内の統計情報読み込み処理（複数箇所）。
  • **呼び出し先**:
  • palloc0, palloc (メモリ確保)
  • elog (エラー報告)
  • memcpy (データコピー)

関連データ構造

  • MVNDistinct: 多変量NDistinct統計情報を保持するメイン構造体。
  • MVNDistinctItem: 個別の列組み合わせに対する統計情報（NDistinct値と属性番号リスト）を保持する構造体。

失敗モードと境界条件

  • data が NULL の場合は NULL を返す。
  • ヘッダーサイズが不足している場合、またはマジックナンバー・型が期待値と異なる場合は ERROR を発生させる。
  • アイテム数が 0 の場合、またはデータサイズが計算上の最小サイズを下回る場合は ERROR を発生させる。

保守担当者向けメモ

  • 本関数は statext_ndistinct_serialize と対になる処理である。データ形式を変更する場合は両方の関数を更新する必要がある。
  • AttrNumber 配列は個別に palloc されるため、メモリ解放時には MVNDistinct 構造体内の各アイテムの属性配列を個別に解放するロジックが必要となる点に注意すること。
