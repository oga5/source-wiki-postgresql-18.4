get_raw_page_internal

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/rawpage.c:144

目的

指定されたリレーションおよびフォークから、物理ストレージ上のデータベースページを読み出し、その内容を bytea 形式で取得する内部関数です。

シグネチャまたは形状

static bytea *get_raw_page_internal(text *relname, ForkNumber forknum, BlockNumber blkno)

入力と出力

  • **入力**:
  • relname: 対象リレーション名（text 型）。
  • forknum: 読み出し対象のフォーク番号（ForkNumber 型）。
  • blkno: 読み出し対象のブロック番号（BlockNumber 型）。
  • **出力**:
  • ページ内容を格納した bytea 型のポインタ。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要です。
  • 対象リレーションはストレージを持つ種類（RELKIND_HAS_STORAGE）である必要があります。
  • 他セッションのローカル一時テーブルへのアクセスは禁止されています。
  • 指定されたブロック番号は、対象リレーションの有効な範囲内である必要があります。

アルゴリズムまたは振る舞い

1. **権限チェック**: superuser() を呼び出し、権限がない場合はエラーを発生させます。 2. **リレーションのオープン**: relation_openrv を使用して対象リレーションを AccessShareLock で開きます。 3. **検証**: リレーションの種類とブロック番号の妥当性を確認し、不正な場合はエラーを発生させます。 4. **メモリ確保**: BLCKSZ + VARHDRSZ 分のメモリを palloc で確保し、bytea 構造体を初期化します。 5. **ページ読み込み**: ReadBufferExtended でバッファを読み込み、LockBuffer で共有ロックを取得します。 6. **コピー**: memcpy を使用してバッファ内のページデータを確保したメモリ領域へコピーします。 7. **クリーンアップ**: バッファのロック解除、バッファの解放、およびリレーションのクローズを行います。 8. **戻り値**: 取得したページデータを格納した bytea ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/rawpage.c 内の各ラッパー関数（get_raw_page 等）から呼び出されます。
  • **呼び出し先**:
  • superuser(), makeRangeVarFromNameList(), textToQualifiedNameList(), relation_openrv(), RelationGetRelationName(), RelationGetNumberOfBlocksInFork(), palloc(), ReadBufferExtended(), LockBuffer(), BufferGetPage(), ReleaseBuffer(), relation_close()。

関連データ構造

  • bytea: ページデータを格納する PostgreSQL の可変長データ型。
  • Relation: リレーションのメタデータ構造体。
  • Buffer: 共有バッファ内のページを指すハンドル。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー。
  • **不正なリレーション**: ストレージを持たないリレーションや他セッションの一時テーブルの場合、エラー。
  • **範囲外アクセス**: 指定したブロック番号がリレーションのサイズを超える場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー。

保守担当者向けメモ

  • 本関数は物理データに直接アクセスするため、セキュリティ上の制約が厳格です。
  • ページデータのコピーには BLCKSZ を使用しているため、PostgreSQL のページサイズ変更時には整合性に注意してください。
