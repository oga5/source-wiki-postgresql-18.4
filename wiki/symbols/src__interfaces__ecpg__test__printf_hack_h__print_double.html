print_double

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/printf_hack.h:5

目的

ECPGテストスイートにおいて、プラットフォーム間で異なる浮動小数点数（double）の指数表記を正規化し、テスト結果の比較（diff）を安定させる。

シグネチャまたは形状

static void print_double(double x)

入力と出力

  • 入力: double 型の値 x。
  • 出力: なし（標準出力へフォーマット済み文字列を書き出す）。

前提条件と不変条件

  • 本関数は static 関数として定義されており、インクルードした各ソースファイル内で個別にコンパイルされる。
  • Windows環境（WIN32 マクロ定義時）では、標準の printf("%g", x) の出力結果を文字列操作で加工する。

アルゴリズムまたは振る舞い

1. **非Windows環境**: printf("%g", x) を実行して終了する。 2. **Windows環境**:

  • sprintf を使用して x を文字列バッファ convert に変換する。
  • 文字列の末尾を走査し、指数部分が3桁（例: e+001）であるかを確認する。
  • 3桁表記の場合、指数部分の余分な 0 を削除して2桁（例: e+01）に詰め、文字列を終端する。
  • 加工後の文字列を printf で出力する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/test/expected/compat_informix-dec_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c
  • 呼び出し先:
  • 標準Cライブラリ関数: sprintf, strlen, printf

関連データ構造

  • 特になし（ローカルバッファ char convert[128] を使用）。

失敗モードと境界条件

  • convert バッファ（128バイト）を超える長さの文字列が生成された場合、sprintf によるバッファオーバーフローの可能性がある（ただし、double の標準的な出力形式では通常発生しない）。
  • 指数表記を含まない値や、3桁以外の指数表記を持つ値に対しては、文字列操作を行わずそのまま出力する。

保守担当者向けメモ

  • Windows環境でのみ文字列操作が発生するため、パフォーマンスが重視される箇所での使用には注意が必要。
  • 将来的に他のプラットフォームで同様のフォーマット差異が確認された場合は、#ifdef による条件分岐の拡張が必要となる。
