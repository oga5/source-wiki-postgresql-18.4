directTriConsistentFn

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginlogic.c:88

目的

GINインデックスにおいて、演算子クラス（opclass）が提供する三値論理整合性関数を直接呼び出すためのラッパー関数です。

シグネチャまたは形状

static GinTernaryValue directTriConsistentFn(GinScanKey key)

入力と出力

  • **入力**: GinScanKey key（スキャン状態とクエリ情報を含む構造体）
  • **出力**: GinTernaryValue（GIN_TRUE, GIN_FALSE, GIN_MAYBE のいずれか）

前提条件と不変条件

  • key->triConsistentFmgrInfo が有効な関数ポインタとして初期化されていること。
  • key 内の各フィールド（entryRes, strategy, query, nuserentries, extra_data, queryValues, queryCategories）が適切に設定されていること。

アルゴリズムまたは振る舞い

1. key->triConsistentFmgrInfo に格納された関数マネージャ情報を利用し、FunctionCall7Coll を介してopclass定義の整合性関数を呼び出します。 2. 呼び出し時には、key 内の各種パラメータ（照合順序、エントリ結果、戦略番号、クエリデータ等）を引数として渡します。 3. 戻り値を DatumGetGinTernaryValue で GinTernaryValue 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginlogic.c 内の ginInitConsistentFunction において、key->triConsistentFn に関数ポインタとして代入されます。
  • **呼び出し先**: FunctionCall7Coll（PostgreSQLの関数呼び出しインターフェース）。

関連データ構造

  • GinScanKey: GINスキャン中の状態を保持する主要な構造体。
  • GinTernaryValue: 三値論理（TRUE/FALSE/MAYBE）を表す列挙型。

失敗モードと境界条件

  • triConsistentFmgrInfo が未設定の場合、呼び出し時にエラーが発生します。
  • 内部で呼び出されるopclassの関数がエラーをスローした場合、そのエラーがそのまま伝播します。

保守担当者向けメモ

  • 本関数は、opclassが三値論理関数を直接実装している場合にのみ使用されます。
  • shimTriConsistentFn とは異なり、追加の計算や状態の書き換えは行わず、純粋な委譲を行います。
