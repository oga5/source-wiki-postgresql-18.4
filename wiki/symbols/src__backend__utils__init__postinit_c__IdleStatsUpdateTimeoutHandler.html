IdleStatsUpdateTimeoutHandler

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:1415

目的

アイドル状態の統計情報更新タイマーが期限切れとなった際に、保留フラグを立ててバックエンドプロセスに通知を行うためのシグナルハンドラです。

シグネチャまたは形状

static void IdleStatsUpdateTimeoutHandler(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数の更新とラッチのセットを行う）

前提条件と不変条件

  • 本関数はシグナルハンドラとして呼び出されることを前提としています。
  • MyLatch が初期化済みであり、有効な状態である必要があります。

アルゴリズムまたは振る舞い

1. IdleStatsUpdateTimeoutPending を true に設定し、統計情報の更新が必要であることを記録します。 2. InterruptPending を true に設定し、割り込み処理が保留中であることをメインループに伝えます。 3. SetLatch(MyLatch) を呼び出し、待機中のバックエンドプロセスを起床させます。

呼び出し元と呼び出し先

  • 呼び出し元: タイムアウト発生時にシグナルハンドラとして呼び出されます（具体的なシグナル登録箇所は postinit.c:769 付近の初期化処理に関連）。
  • 呼び出し先: SetLatch（storage/ipc.h 経由）。

関連データ構造

  • IdleStatsUpdateTimeoutPending (グローバルフラグ)
  • InterruptPending (グローバルフラグ)
  • MyLatch (プロセス固有のラッチ)

失敗モードと境界条件

  • 本関数自体は単純なフラグ操作とラッチ設定のみを行うため、関数内部での失敗モードはありません。
  • 割り込みが頻発する環境下では、InterruptPending の競合やシグナル処理の再入可能性に注意が必要です。

保守担当者向けメモ

  • 本関数は postinit.c 内で静的関数として定義されています。
  • 統計情報の更新タイミングを制御する重要なハンドラであるため、変更時は InterruptPending を参照するメインループ側の処理への影響を考慮してください。
