basebackup_options

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup.c:62

目的

BASE_BACKUP コマンドによって要求されたバックアップ処理の設定パラメータを保持し、バックアップの実行エンジン全体で共有するために使用されます。

シグネチャまたは形状

typedef struct
{
	const char *label;
	bool		progress;
	bool		fastcheckpoint;
	bool		nowait;
	bool		includewal;
	bool		incremental;
	uint32		maxrate;
	bool		sendtblspcmapfile;
	bool		send_to_client;
	bool		use_copytblspc;
	BaseBackupTargetHandle *target_handle;
	backup_manifest_option manifest;
	pg_compress_algorithm compression;
	pg_compress_specification compression_specification;
	pg_checksum_type manifest_checksum_type;
} basebackup_options;

入力と出力

  • **入力**: parse_basebackup_options 関数により、クライアントから送信されたコマンド引数（List *options）がパースされ、本構造体の各フィールドに格納されます。
  • **出力**: perform_base_backup 関数などのバックアップ実行ロジックに対し、バックアップの挙動を制御する設定値として参照されます。

前提条件と不変条件

  • parse_basebackup_options を通じて初期化されることが前提です。
  • 構造体内の各フィールドは、バックアップの開始から終了までのライフサイクルを通じて、バックアップの整合性を維持するための設定として機能します。

アルゴリズムまたは振る舞い

本構造体はデータ保持用であり、直接的なロジックは持ちません。バックアップの実行フローにおいて、以下の制御を決定づけます。

  • 圧縮アルゴリズムおよび仕様の指定。
  • WAL（Write Ahead Log）を含めるかどうかのフラグ管理。
  • 増分バックアップの有効化。
  • バックアップデータの転送先（target_handle）の指定。
  • バックアップマニフェストの生成設定。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c 内の parse_basebackup_options（設定の書き込み）。
  • src/backend/backup/basebackup.c 内の perform_base_backup（設定の読み取り）。
  • src/backend/backup/basebackup.c 内の SendBaseBackup（構造体のインスタンス化）。
  • **呼び出し先**: 構造体であるため、関数としての呼び出し先はありません。

関連データ構造

  • bbsink: バックアップデータの出力先や変換パイプラインを定義する構造体。
  • BaseBackupTargetHandle: バックアップの出力先を抽象化したハンドル。
  • backup_manifest_option: マニフェスト生成に関する設定。
  • pg_compress_algorithm / pg_compress_specification: 圧縮設定。

失敗モードと境界条件

  • parse_basebackup_options において不正なオプションが渡された場合、バックアップ処理が開始される前にエラーが発生します。
  • 構造体内のポインタ（target_handle 等）が適切に初期化されていない状態で perform_base_backup が参照した場合、セグメンテーション違反等の未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体に新しいバックアップオプションを追加する場合は、parse_basebackup_options 関数内のパース処理も併せて更新してください。
  • bbsink インターフェースとの連携が深いため、変更時は src/backend/backup/basebackup_sink.h の定義との整合性に注意してください。
