uuid_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/uuid.c:180

目的

バイナリ形式の入力データから uuid 型の内部表現（pg_uuid_t）を構築します。

シグネチャまたは形状

Datum uuid_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される StringInfo 型のバッファ。
  • **出力**: pg_uuid_t 構造体へのポインタを Datum として返却します。

前提条件と不変条件

  • 入力バッファには、UUID のサイズ（UUID_LEN）に相当するバイト数が格納されている必要があります。
  • palloc によりメモリが動的に確保されるため、呼び出し元で適切なメモリ管理が必要です。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、入力バッファ（StringInfo）を取得します。 2. palloc(UUID_LEN) を呼び出し、pg_uuid_t 構造体用に 16 バイトのメモリを確保します。 3. pq_getmsgbytes を用いてバッファから UUID_LEN 分のバイト列を読み取ります。 4. memcpy を使用して、読み取ったバイト列を uuid->data にコピーします。 5. 構築した pg_uuid_t へのポインタを PG_RETURN_POINTER で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: 不明（PostgreSQL の型システムにより、バイナリ入力時に自動的に呼び出されます）。
  • **呼び出し先**: palloc（メモリ確保）、pq_getmsgbytes（バイナリメッセージの読み取り）、memcpy（メモリコピー）。

関連データ構造

  • pg_uuid_t: UUID を保持する 16 バイトの構造体。
  • StringInfo: libpq/pqformat.h で定義される、バイナリ通信用のデータ構造。

失敗モードと境界条件

  • pq_getmsgbytes がバッファの不足を検知した場合、エラーが発生する可能性があります。
  • メモリ不足時には palloc がエラーをスローします。

保守担当者向けメモ

  • 本関数は uuid 型のバイナリ入力ハンドラです。RFC 9562 等の仕様変更に伴い UUID のバイナリ表現に変更が生じた場合は、本関数の UUID_LEN 処理を確認してください。
