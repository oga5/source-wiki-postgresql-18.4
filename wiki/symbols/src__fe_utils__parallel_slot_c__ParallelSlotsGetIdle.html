ParallelSlotsGetIdle

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:370

目的

並列処理用のデータベース接続スロットを取得します。指定されたデータベースへの接続済みスロットの再利用、新規接続、または既存スロットの再接続を行い、利用可能なスロットを返します。

シグネチャまたは形状

ParallelSlot *ParallelSlotsGetIdle(ParallelSlotArray *sa, const char *dbname)

入力と出力

  • **入力**:
  • sa: ParallelSlotArray 構造体へのポインタ。
  • dbname: 接続先のデータベース名（NULL可）。
  • **出力**:
  • 成功時: 利用可能になった ParallelSlot 構造体へのポインタ。
  • 失敗時: NULL（wait_on_slots で致命的なエラーが発生した場合）。

前提条件と不変条件

  • sa は非NULLであること。
  • sa->numslots > 0 であること。
  • 戻り値の ParallelSlot は inUse フラグが true に設定された状態で返される。

アルゴリズムまたは振る舞い

以下の優先順位でスロットを探索し、取得を試みます。 1. **既存の適合スロット**: find_matching_idle_slot を使用し、指定された dbname に既に接続済みのアイドルスロットがあればそれを返す。 2. **未接続スロット**: find_unconnected_slot を使用し、未使用のスロットがあれば connect_slot で接続して返す。 3. **再接続**: find_any_idle_slot を使用し、別のデータベースに接続されているアイドルスロットがあれば、切断後に指定の dbname へ再接続して返す。 4. **待機**: 上記のいずれも不可能な場合、wait_on_slots を呼び出してスロットが空くまでブロックする。スロットが解放されるかエラーが発生するまでループを継続する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c
  • **呼び出し先**:
  • find_matching_idle_slot
  • find_unconnected_slot
  • find_any_idle_slot
  • connect_slot
  • disconnectDatabase
  • wait_on_slots

関連データ構造

  • ParallelSlotArray: スロットの配列を保持する管理構造体。
  • ParallelSlot: 個別の接続状態（connection）と使用状況（inUse）を保持する構造体。

失敗モードと境界条件

  • 全てのスロットがビジー状態で、かつ wait_on_slots が致命的なエラーを検出した場合、NULL を返す。
  • Windows環境では select() の制限により、FD_SETSIZE を超えるファイルディスクリプタを扱うとエラー終了する可能性がある。

保守担当者向けメモ

  • 本関数はシングルスレッド環境での使用を前提としている。
  • wait_on_slots が返すエラーは、接続の致命的な障害を意味するため、呼び出し側で適切に処理する必要がある。
