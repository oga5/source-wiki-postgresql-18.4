BTStackData

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtree.h:743

目的

B-tree インデックスのツリー降下時に、ピボットタプルの位置を記録するためのスタック構造体です。

シグネチャまたは形状

typedef struct BTStackData
{
	BlockNumber bts_blkno;      /* ページ番号 */
	OffsetNumber bts_offset;    /* ページ内のオフセット */
	struct BTStackData *bts_parent; /* 親ノードへのポインタ */
} BTStackData;

入力と出力

  • 入力: ツリー降下中の各レベルにおけるブロック番号とオフセット。
  • 出力: 連結リスト形式で構成されたスタック（BTStack）。

前提条件と不変条件

  • ツリー降下時にルートからリーフへ向かって順次プッシュされます。
  • bts_parent は、より上位（ルートに近い）レベルのスタック要素を指します。

アルゴリズムまたは振る舞い

  • インデックスの検索や挿入時に、ツリーを降下する過程で通過したノードの位置情報を保持します。
  • ページ分割が発生した際、このスタックを遡ることで、親ノードに対して新しいピボットタプルを挿入する位置を特定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/nbtree/nbtsearch.c（検索時のスタック構築）、src/backend/access/nbtree/nbtinsert.c（挿入時のスタック利用）。
  • 呼び出し先: 直接的な関数呼び出しは定義されていませんが、palloc 等によるメモリ確保の対象となります。

関連データ構造

  • BTStack: BTStackData へのポインタ型。
  • BTScanInsertData: 挿入操作時に使用されるスキャンキー構造体。

失敗モードと境界条件

  • メモリ不足: palloc によるスタック要素の確保時に発生する可能性があります。
  • スタックの空: ルートノードに到達した際、bts_parent は NULL となります。

保守担当者向けメモ

  • BTStack は BTStackData * の別名として定義されています。
  • 構造体内の bts_blkno と bts_offset は、インデックスの再走査や分割時の親更新に不可欠な情報です。変更時は、ツリー降下ロジック全体への影響を考慮してください。
