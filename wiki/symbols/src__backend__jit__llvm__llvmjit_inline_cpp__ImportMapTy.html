ImportMapTy

定義

  • 種別: typedef
  • 定義箇所: src/backend/jit/llvm/llvmjit_inline.cpp:96

目的

JIT コンパイルにおけるモジュール境界を越えたインライン展開計画を保持するためのデータ構造です。どのモジュールからどの関数（またはシンボル）をインポートすべきかを管理します。

シグネチャまたは形状

typedef llvm::StringMap<llvm::StringSet<> > ImportMapTy;
  • llvm::StringMap のキーはモジュール識別子（文字列）を表します。
  • 値である llvm::StringSet<> は、そのモジュールからインポート対象となる関数名やシンボルの集合を保持します。

入力と出力

  • **入力**: llvm_build_inline_plan 関数によって構築され、インライン化対象の関数リストが格納されます。
  • **出力**: llvm_execute_inline_plan 関数に渡され、llvm::IRMover を用いた実際のコード統合処理の引数として利用されます。

前提条件と不変条件

  • llvm::StringMap および llvm::StringSet の特性上、同一モジュール内の重複するシンボルは自動的に排除されます。
  • インライン化計画が完了した後に llvm_execute_inline_plan へ渡される必要があります。

アルゴリズムまたは振る舞い

1. llvm_build_inline_plan が現在のモジュール内の外部参照をスキャンし、インライン化すべき関数を特定します。 2. 特定された関数を、その定義元モジュールごとに ImportMapTy へ分類・格納します。 3. llvm_execute_inline_plan がこのマップを走査し、各モジュールから必要な関数定義を現在のモジュールへ移動（インポート）します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • llvm_build_inline_plan (計画の構築)
  • llvm_execute_inline_plan (計画の実行)
  • **呼び出し先**:
  • 本シンボル自体はデータ構造の型定義であり、関数呼び出しは行いません。

関連データ構造

  • llvm::StringMap: モジュール単位の管理に使用。
  • llvm::StringSet: モジュール内の対象シンボル管理に使用。
  • InlineWorkList: インライン化の候補を探索する際に使用される補助構造体。

失敗モードと境界条件

  • メモリ不足: 大規模なモジュール群をインライン化する場合、StringMap および StringSet の構築によりメモリ消費量が増大する可能性があります。
  • 依存関係の欠如: インデックス（SummaryCache）に存在しないモジュールや関数が指定された場合、IRMover による統合プロセスで失敗する可能性があります。

保守担当者向けメモ

  • 本型は LLVM のデータ構造に直接依存しています。LLVM のバージョンアップ時に StringMap や StringSet の API 変更がないか注意してください。
  • llvm_inline_reset_caches を呼び出す際、このマップに関連するキャッシュの整合性が保たれていることを確認してください。
