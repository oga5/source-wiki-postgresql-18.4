decrypt_read

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:297

目的

PGP 復号パイプラインにおいて、入力ストリームからデータを読み出し、CFB（Cipher Feedback）モードで復号を行うためのフィルタ読み取り関数です。

シグネチャまたは形状

static int decrypt_read(void *priv, PullFilter *src, int len, uint8 **data_p, uint8 *buf, int buflen);

入力と出力

  • priv: 復号コンテキスト（PGP_CFB 構造体へのポインタ）。
  • src: 読み取り元の PullFilter。
  • len: 要求するデータ長。
  • data_p: 復号済みデータへのポインタを格納する出力先。
  • buf: 復号結果を書き込むためのバッファ。
  • buflen: buf のサイズ。
  • 戻り値: 読み取りに成功したバイト数、またはエラーを示す負の値。

前提条件と不変条件

  • priv は PGP_CFB 型としてキャスト可能であること。
  • buf は len バイト以上の書き込み領域を保持していること。

アルゴリズムまたは振る舞い

1. pullf_read を呼び出し、ソースフィルタ src から最大 len バイトの暗号化データを読み出す。 2. 読み取り成功時（res > 0）、pgp_cfb_decrypt を使用して tmp バッファ内のデータを復号し、結果を buf に格納する。 3. data_p を buf に設定し、復号済みデータへの参照を呼び出し元に提供する。 4. 読み取ったバイト数 res を返す。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-decrypt.c 内の decrypt_init において、フィルタ初期化時に参照されている。
  • 呼び出し先: pullf_read（データ読み取り）、pgp_cfb_decrypt（CFB 復号）。

関連データ構造

  • PGP_CFB: 復号状態を保持する構造体。
  • PullFilter: データの逐次読み取りを行うための抽象化構造体。

失敗モードと境界条件

  • pullf_read がエラーを返した場合、そのエラーコードがそのまま本関数の戻り値となる。
  • len が buflen を超える場合、バッファオーバーフローの可能性がある（呼び出し元での整合性保証に依存）。

保守担当者向けメモ

  • 本関数は PullFilter のコールバックとして設計されています。
  • 復号処理は pgp-cfb.c で定義された pgp_cfb_decrypt に依存しているため、復号アルゴリズムの変更時はそちらを確認してください。
