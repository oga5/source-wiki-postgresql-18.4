pgstat_fetch_stat_wal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_wal.c:66

目的

WAL（Write-Ahead Logging）統計情報の最新スナップショットを取得し、そのポインタを返します。

シグネチャまたは形状

PgStat_WalStats *pgstat_fetch_stat_wal(void)

入力と出力

  • **入力**: なし
  • **出力**: PgStat_WalStats 構造体へのポインタ。現在のプロセスローカルなWAL統計スナップショットを指します。

前提条件と不変条件

  • 本関数を呼び出す前に、統計サブシステムが初期化されている必要があります。
  • 返されるポインタは pgStatLocal.snapshot.wal を参照しており、呼び出しごとに pgstat_snapshot_fixed(PGSTAT_KIND_WAL) によって最新の状態に更新されます。

アルゴリズムまたは振る舞い

1. pgstat_snapshot_fixed(PGSTAT_KIND_WAL) を呼び出し、共有メモリ上のWAL統計をローカルスナップショットへ同期させます。 2. pgStatLocal.snapshot.wal のアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c:1705 (SQLインターフェース経由の統計参照)
  • **呼び出し先**:
  • pgstat_snapshot_fixed (統計サブシステムの内部関数)

関連データ構造

  • PgStat_WalStats: WAL統計情報を保持する構造体。
  • pgStatLocal: 統計サブシステムが保持するプロセスローカルな状態管理構造体。

失敗モードと境界条件

  • 統計サブシステムが未初期化の場合や、共有メモリへのアクセス権限がない状況下での挙動については、本ソースコードからは判断できません。

保守担当者向けメモ

  • 本関数は pgstat モジュールの公開APIの一部であり、SQL関数からWAL統計を参照する際の標準的な取得経路です。
  • pgstat_snapshot_fixed を介して同期を行うため、呼び出しのたびに最新の統計値が反映されます。
