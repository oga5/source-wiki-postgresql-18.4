TidStoreCreateShared

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tidstore.c:207

目的

共有メモリ上に TidStore を初期化し、複数のバックエンドプロセス間で TID（ItemPointerData）の集合を共有するためのストレージ領域を作成します。

シグネチャまたは形状

TidStore *TidStoreCreateShared(size_t max_bytes, int tranche_id)

入力と出力

  • **入力**:
  • max_bytes: ストレージが使用する最大メモリサイズ（バイト単位）。
  • tranche_id: 共有メモリのロック管理に使用する LWLock トランシェ ID。
  • **出力**:
  • 初期化された TidStore 構造体へのポインタ。

前提条件と不変条件

  • max_bytes に基づいて DSA（Dynamic Shared Area）のセグメントサイズが自動調整されます。
  • 内部で使用される DSA セグメントサイズは、max_bytes の 1/8 を超えない範囲で設定されます。
  • dsa_max_size は DSA_MIN_SEGMENT_SIZE 以上であることが保証されます。

アルゴリズムまたは振る舞い

1. TidStore 構造体用にローカルメモリを確保します。 2. 指定された max_bytes に応じて、DSA セグメントの初期サイズおよび最大サイズを計算します。 3. dsa_create_ext を呼び出し、共有メモリ領域（dsa_area）を作成します。 4. shared_ts_create を呼び出し、作成した dsa_area 上に共有ラディックツリーを構築します。 5. 確保した dsa_area とツリー構造を TidStore 構造体に格納して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuumparallel.c (VACUUM 並列処理)
  • src/test/modules/test_tidstore/test_tidstore.c (テストモジュール)
  • **呼び出し先**:
  • palloc0
  • dsa_create_ext
  • shared_ts_create (内部関数)

関連データ構造

  • TidStore: 本関数が初期化するメイン構造体。
  • dsa_area: 共有メモリ管理領域。

失敗モードと境界条件

  • max_bytes が極端に小さい場合、DSA セグメントサイズは DSA_MIN_SEGMENT_SIZE に切り詰められます。
  • メモリ不足時には palloc0 または dsa_create_ext がエラーを発生させます。

保守担当者向けメモ

  • 本関数で作成された TidStore は、他のバックエンドが TidStoreAttach を介して接続可能です。
  • 共有メモリを使用するため、呼び出し側で適切なロック管理（TidStoreLockExclusive 等）を行う必要があります。
