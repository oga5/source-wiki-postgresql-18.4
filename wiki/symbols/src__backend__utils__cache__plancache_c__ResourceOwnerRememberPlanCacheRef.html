ResourceOwnerRememberPlanCacheRef

定義

  • 種別: static inline function
  • 定義箇所: src/backend/utils/cache/plancache.c:125

目的

指定された CachedPlan への参照を、現在の ResourceOwner に登録し、リソースの追跡対象として管理します。

シグネチャまたは形状

static inline void ResourceOwnerRememberPlanCacheRef(ResourceOwner owner, CachedPlan *plan)

入力と出力

  • **入力**:
  • owner: 参照を保持する ResourceOwner オブジェクト。
  • plan: 追跡対象となる CachedPlan 構造体へのポインタ。
  • **出力**: なし（副作用として ResourceOwner に登録）。

前提条件と不変条件

  • owner が有効な ResourceOwner であること。
  • plan が有効な CachedPlan 構造体を指していること。
  • 内部的に planref_resowner_desc を使用してリソースの記述を行っています。

アルゴリズムまたは振る舞い

1. plan ポインタを Datum 型に変換します。 2. ResourceOwnerRemember 関数を呼び出し、owner に対して当該 plan の参照を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/plancache.c 内の複数箇所（行 1374, 1536, 1597 など）から呼び出されています。
  • **呼び出し先**:
  • ResourceOwnerRemember (utils/resowner.h)

関連データ構造

  • CachedPlan: 実行可能な計画ツリーを保持する構造体。
  • ResourceOwner: リソースの所有権とライフサイクルを管理する構造体。
  • planref_resowner_desc: リソース所有者による管理のための記述子。

失敗モードと境界条件

  • owner が NULL の場合、ResourceOwnerRemember の仕様に依存します。
  • メモリ不足等の理由で ResourceOwner への登録に失敗した場合、ResourceOwnerRemember がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は CachedPlan のメモリリークを防ぐためのリソース管理機構の一部です。
  • 対応する ResourceOwnerForgetPlanCacheRef が存在するため、参照の解放時には必ず対となる関数を呼び出す必要があります。
