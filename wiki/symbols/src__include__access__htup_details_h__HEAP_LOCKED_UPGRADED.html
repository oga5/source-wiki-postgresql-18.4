HEAP_LOCKED_UPGRADED

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/htup_details.h:254

目的

ヒープタプルの t_infomask を検査し、当該タプルが「ロックアップグレード」状態（MultiXactにおいて、ロックのみの保持から排他ロック等へ昇格した状態）にあるかどうかを判定します。

シグネチャまたは形状

static inline bool HEAP_LOCKED_UPGRADED(uint16 infomask)

入力と出力

  • **入力**: infomask (タプルの t_infomask フィールド)
  • **出力**: bool (条件を満たす場合は true、それ以外は false)

前提条件と不変条件

  • 入力される infomask は、有効なヒープタプルのヘッダーから取得されたものである必要があります。

アルゴリズムまたは振る舞い

以下のすべての条件を満たす場合に true を返します。 1. HEAP_XMAX_IS_MULTI フラグが立っている（MultiXactIDが使用されている）。 2. HEAP_XMAX_LOCK_ONLY フラグが立っている（ロックのみの操作である）。 3. HEAP_XMAX_EXCL_LOCK および HEAP_XMAX_KEYSHR_LOCK のいずれのフラグも立っていない。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgrowlocks/pgrowlocks.c
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_visibility.c
  • src/backend/access/transam/multixact.c
  • **呼び出し先**:
  • 提供されたソース情報には、本関数が内部で呼び出している関数やマクロの記述はありません。

関連データ構造

  • HeapTupleHeaderData (タプルヘッダー構造体)
  • t_infomask (タプルの状態を示すビットマスク)

失敗モードと境界条件

  • infomask が正しく初期化されていない、あるいは破損している場合、誤った判定結果を返す可能性があります。

保守担当者向けメモ

  • 本関数は、MultiXactIDに関連するロック状態の解釈において重要な役割を果たします。
  • 判定ロジックを変更する場合は、関連する HEAP_XMAX_* フラグの定義（htup_details.h 内）との整合性に注意してください。
