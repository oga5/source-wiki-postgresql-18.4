GetForeignServerByName

定義

  • 種別: function
  • 定義箇所: src/backend/foreign/foreign.c:182

目的

外部サーバーの名前（srvname）を指定して、その定義情報をシステムカタログから取得する。

シグネチャまたは形状

ForeignServer *GetForeignServerByName(const char *srvname, bool missing_ok)

入力と出力

  • **入力**:
  • srvname: 検索対象の外部サーバー名。
  • missing_ok: true の場合、サーバーが見つからないときにエラーを発生させず NULL を返す。false の場合はエラーを発生させる（get_foreign_server_oid の挙動に依存）。
  • **出力**:
  • 成功時: 外部サーバー定義を保持する ForeignServer 構造体へのポインタ。
  • 失敗時: missing_ok が true でサーバーが存在しない場合、NULL を返す。

前提条件と不変条件

  • 内部で get_foreign_server_oid を呼び出し、サーバーの OID を解決する。
  • 戻り値の ForeignServer 構造体は、GetForeignServer を介してメモリ上に構築される。

アルゴリズムまたは振る舞い

1. get_foreign_server_oid(srvname, missing_ok) を呼び出し、サーバーの OID を取得する。 2. OID が無効（!OidIsValid）であれば、NULL を返して終了する。 3. OID が有効であれば、GetForeignServer(serverid) を呼び出し、その結果を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/postgres_fdw/connection.c
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/foreigncmds.c
  • **呼び出し先**:
  • get_foreign_server_oid: サーバー名から OID を解決する。
  • GetForeignServer: OID から ForeignServer 構造体を取得する。

関連データ構造

  • ForeignServer: 外部サーバーの定義情報を保持する構造体。

失敗モードと境界条件

  • missing_ok が false の状態で存在しないサーバー名を指定した場合、get_foreign_server_oid 内部でエラーが発生する。
  • データベース接続やシステムカタログへのアクセス権限がない場合、カタログ検索が失敗する可能性がある。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な検索ロジックは get_foreign_server_oid および GetForeignServer に依存している。
  • 戻り値のメモリ管理については GetForeignServer の仕様に従うこと。
