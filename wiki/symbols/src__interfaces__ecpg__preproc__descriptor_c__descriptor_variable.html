descriptor_variable

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:330

目的

SQL記述子をECPGの変数として扱うための専用構造体（variable）を生成・返却します。記述子は通常の変数リストには含まれないため、本関数が静的な領域を用いて管理を行います。

シグネチャまたは形状

struct variable *descriptor_variable(const char *name, int input)

入力と出力

  • **入力**:
  • name: 記述子の名前（文字列）。
  • input: 記述子のインデックス（0または1）。descriptor_names 配列の添字として使用されます。
  • **出力**:
  • 記述子情報を保持する struct variable へのポインタ。

前提条件と不変条件

  • input 引数は 0 または 1 である必要があります（descriptor_names および varspace のサイズが2であるため）。
  • 記述子の名前は MAX_DESCRIPTOR_NAMELEN を超えてはなりません。

アルゴリズムまたは振る舞い

1. 静的領域として descriptor_names（名前格納用）と descriptor_type（型定義）、varspace（変数構造体）を保持します。 2. 指定された input インデックスに対応する descriptor_names の位置へ、name を strlcpy でコピーします。 3. 対応する varspace[input] のアドレスを返します。これにより、呼び出し元は記述子をECPGの変数として参照可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c 内の記述子操作関数（具体的な関数名はソース抜粋からは特定不可）。
  • **呼び出し先**:
  • strlcpy（標準ライブラリ関数）。

関連データ構造

  • struct variable: ECPGにおける変数を表現する構造体。
  • struct ECPGtype: 変数の型情報を保持する構造体。本関数では ECPGt_descriptor 型として初期化されます。

失敗モードと境界条件

  • input が 0 または 1 以外の場合、配列外アクセスが発生する可能性があります。
  • name が MAX_DESCRIPTOR_NAMELEN を超える場合、strlcpy により切り詰められます。

保守担当者向けメモ

  • 本関数は静的領域を再利用するため、同時に複数の記述子を扱う際は input インデックスの管理が重要です。
  • 記述子は通常の変数リストとは独立して管理される設計である点に注意してください。
