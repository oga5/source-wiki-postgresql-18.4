coerce_to_target_type

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_coerce.c:77

目的

式（Node）をターゲットのデータ型および型修飾子（typmod）へ変換するための最上位エントリーポイントです。SQLの CAST や暗黙的な型変換の要求に対し、適切な変換ノードを生成して式ツリーを修飾します。

シグネチャまたは形状

Node *coerce_to_target_type(ParseState *pstate, Node *expr, Oid exprtype,
                            Oid targettype, int32 targettypmod,
                            CoercionContext ccontext,
                            CoercionForm cformat,
                            int location)

入力と出力

  • **入力**:
  • pstate: パーサーの状態（NULL 可）。
  • expr: 変換対象の式ノード。
  • exprtype, targettype: 変換元および変換先の型OID。
  • targettypmod: ターゲット型の型修飾子（長さ制限等）。
  • ccontext, cformat: 変換の文脈と表示形式。
  • location: エラー報告用の構文位置。
  • **出力**: 変換後の Node*。変換不可能な場合は NULL を返します。

前提条件と不変条件

  • 変換前に can_coerce_type による可否判定が行われます。
  • 入力式が CollateExpr を持つ場合、一時的に剥がされ、変換後に再適用されます。
  • ターゲット型が照合順序（Collation）をサポートしない場合、CollateExpr は破棄されます。

アルゴリズムまたは振る舞い

1. **可否判定**: can_coerce_type を呼び出し、変換パスが存在するか確認します。存在しない場合は NULL を返します。 2. **照合順序の保護**: 式の最上位にある CollateExpr を一時的に除去します。 3. **型変換**: coerce_type を呼び出し、型変換用のノード（FuncExpr 等）を生成します。 4. **Typmod適用**: coerce_type_typmod を呼び出し、固定長型などの長さ制限を適用します。 5. **照合順序の復元**: ターゲット型が照合順序をサポートする場合、除去した CollateExpr を変換結果のノードに再ラップします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_subs.c, src/backend/catalog/heap.c, src/backend/commands/prepare.c, src/backend/commands/tablecmds.c, src/backend/executor/functions.c, src/backend/parser/analyze.c 等。
  • **呼び出し先**: can_coerce_type, coerce_type, coerce_type_typmod, type_is_collatable, makeNode。

関連データ構造

  • Node: PostgreSQLの構文解析ツリーの基本単位。
  • CollateExpr: 照合順序情報を保持するノード。
  • ParseState: パーサーのコンテキスト情報。

失敗モードと境界条件

  • can_coerce_type が変換不可と判断した場合、NULL を返します。
  • ターゲット型が CollateExpr をサポートしない場合、元の照合順序情報は失われます。
  • coerce_type や coerce_type_typmod の内部でエラーが発生した場合、例外がスローされる可能性があります。

保守担当者向けメモ

  • 本関数は元の式ツリーを破壊せず、新しいノードをラップして返す設計です。
  • CollateExpr の処理は coerce_type と重複するロジックを含んでいますが、変換の成否を正確に判定するために必要です。
  • 変換ノードの生成ロジックを変更する際は、coerce_type との整合性に注意してください。
