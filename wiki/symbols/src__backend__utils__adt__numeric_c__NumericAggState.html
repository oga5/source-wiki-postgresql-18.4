NumericAggState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/numeric.c:4913

目的

NUMERIC 型の集約計算（SUM、AVG、STDDEV 等）において、計算中の中間状態を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct NumericAggState
{
	bool		calcSumX2;		/* sumX2を計算するかどうか */
	MemoryContext agg_context;	/* 計算に使用するメモリコンテキスト */
	int64		N;				/* 処理された数値のカウント */
	NumericSumAccum sumX;		/* 数値の合計 */
	NumericSumAccum sumX2;		/* 数値の二乗の合計 */
	int			maxScale;		/* これまでに確認された最大スケール */
	int64		maxScaleCount;	/* 最大スケールを持つ値の個数 */
	int64		NaNcount;		/* NaN値の個数 */
	int64		pInfcount;		/* +Inf値の個数 */
	int64		nInfcount;		/* -Inf値の個数 */
} NumericAggState;

入力と出力

  • **入力**: do_numeric_accum 関数を介して Numeric 型の値を受け取り、状態を更新します。
  • **出力**: 集約の最終段階で、保持している sumX や N 等の値を基に最終的な集約結果が生成されます。

前提条件と不変条件

  • N には NaNcount、pInfcount、nInfcount は含まれません。これらを含めた総数が必要な場合は NA_TOTAL_COUNT() マクロを使用する必要があります。
  • agg_context は、集約計算中にメモリを割り当てるための有効なコンテキストである必要があります。

アルゴリズムまたは振る舞い

  • palloc0 を使用して初期化され、集約関数が新しい値を受け取るたびに do_numeric_accum によって内部の合計値（sumX, sumX2）や統計情報が更新されます。
  • calcSumX2 フラグが真の場合のみ、二乗和の計算が行われます。
  • maxScale および maxScaleCount を追跡することで、精度の管理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/numeric.c 内の集約関数群（do_numeric_accum、do_numeric_discard 等）から参照・更新されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、palloc0 によるメモリ確保が行われます。

関連データ構造

  • NumericSumAccum: sumX および sumX2 の保持に使用されます。
  • Numeric: 集約対象となる数値データ型。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • NaN や無限大（Inf）は N にカウントされず、個別のカウンタで管理されます。

保守担当者向けメモ

  • N と NaNcount 等のカウンタの使い分けに注意してください。
  • NumericSumAccum の内部実装が変更された場合、本構造体の計算ロジックも影響を受ける可能性があります。
