City

定義

  • **種別**: struct
  • **定義箇所**: src/include/optimizer/geqo_recombination.h:62

目的

遺伝的クエリ最適化（GEQO）における交叉（Recombination）処理中、各都市のツアー内での位置情報や使用状況を追跡・管理するために使用されます。

シグネチャまたは形状

typedef struct City
{
    int    tour2_position;
    int    tour1_position;
    int    used;
    int    select_list;
} City;

入力と出力

本構造体はデータ保持用であり、関数ではありません。geqo_recombination.c 内の alloc_city_table 関数によってメモリが確保され、各交叉アルゴリズム（pmx, cx, px, ox1, ox2 など）に作業用領域として渡されます。

前提条件と不変条件

  • alloc_city_table を使用して (num_gene + 1) 個分のメモリを確保する必要があります。
  • 確保されたメモリは、使用後に free_city_table を呼び出して解放する必要があります。

アルゴリズムまたは振る舞い

交叉アルゴリズム実行中、以下の目的で利用されます。

  • tour1_position, tour2_position: 親個体（ツアー）内での都市のインデックスを保持します。
  • used: 交叉処理において、その都市が既に子個体に割り当て済みであるかを示すフラグとして機能します。
  • select_list: 交叉アルゴリズムの種類に応じて、選択リストや補助的な状態管理に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/backend/optimizer/geqo/geqo_cx.c
  • src/backend/optimizer/geqo/geqo_main.c
  • src/backend/optimizer/geqo/geqo_ox1.c
  • src/backend/optimizer/geqo/geqo_ox2.c
  • src/backend/optimizer/geqo/geqo_px.c
  • src/backend/optimizer/geqo/geqo_recombination.c
  • **呼び出し先**: 構造体のため該当なし。

関連データ構造

  • Gene: ツアーを構成する遺伝子型。
  • Edge: エッジ組み換え交叉（ERX）で使用される関連構造体。

失敗モードと境界条件

  • メモリ確保失敗: palloc が失敗した場合、PostgreSQLのメモリ管理機構によりエラーがスローされます。
  • メモリリーク: free_city_table を呼び出さない場合、GEQOの反復処理中にメモリリークが発生します。

保守担当者向けメモ

  • src/bin/initdb/findtimezone.c に同名の City という文字列が含まれていますが、これらはタイムゾーン関連のコメントであり、本構造体とは無関係です。
  • 交叉関数は geqo_recombination.c 内で条件付きコンパイル（#if defined(...)）される場合があるため、実装変更時は注意してください。
