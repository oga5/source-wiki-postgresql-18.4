donothingStartup

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/dest.c:55

目的

クエリ実行結果の出力先として「何もしない（破棄）」を選択した場合の、初期化処理を実行します。

シグネチャまたは形状

static void donothingStartup(DestReceiver *self, int operation, TupleDesc typeinfo)

入力と出力

  • **入力**:
  • self: DestReceiver 構造体へのポインタ。
  • operation: 実行される操作の種類（int）。
  • typeinfo: タプルの記述情報（TupleDesc）。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数は DestReceiver のメソッドとして呼び出されます。
  • 状態を持たない静的な出力先（DestNone）に関連付けられているため、内部状態の変更やメモリ割り当ては行いません。

アルゴリズムまたは振る舞い

  • 関数本体は空であり、呼び出されても何も処理を行わず即座に復帰します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/dest.c 内の静的レシーバー定義（donothingDR）において、rStartup フィールドに登録されています。
  • **呼び出し先**: なし。

関連データ構造

  • DestReceiver: 出力先を抽象化する構造体。本関数はこの構造体の rStartup メソッドとして機能します。

失敗モードと境界条件

  • 処理内容が空であるため、本関数自体に起因する失敗モードはありません。

保守担当者向けメモ

  • 本関数は DestNone（結果破棄）用の静的レシーバーの一部です。このレシーバーは静的に定義されているため、rDestroy 等で解放してはなりません。
  • 他の出力先で初期化処理が必要な場合と同様のインターフェースを維持するために存在しています。
