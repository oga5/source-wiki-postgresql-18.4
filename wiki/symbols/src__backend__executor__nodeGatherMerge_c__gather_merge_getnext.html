gather_merge_getnext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:546

目的

並列クエリ実行において、複数のワーカーから提供されるソート済みタプルストリームをバイナリヒープを用いてマージし、順序を維持した単一のタプルを返却する。

シグネチャまたは形状

static TupleTableSlot *gather_merge_getnext(GatherMergeState *gm_state)

入力と出力

  • **入力**: GatherMergeState *gm_state（実行状態を保持する構造体）
  • **出力**: TupleTableSlot *（マージされた順序における次のタプル、または全ストリーム終了時は NULL）

前提条件と不変条件

  • gm_state->gm_heap はバイナリヒープとして初期化済みであること。
  • 初回呼び出し時は gm_state->gm_initialized が false であり、gather_merge_init が実行されること。

アルゴリズムまたは振る舞い

1. **初期化**: gm_initialized が偽の場合、gather_merge_init を呼び出し、各参加者から最初のタプルを取得してヒープを構築する。 2. **反復処理**:

  • ヒープの先頭（最小値）である参加者インデックス i を取得する。
  • gather_merge_readnext を呼び出し、参加者 i から次のタプルを読み込む。
  • 読み込み成功時は、そのタプルをヒープに再挿入（binaryheap_replace_first）して順序を再構築する。
  • 読み込み失敗（ストリーム終了）時は、参加者 i をヒープから削除する。

    • 3. **終了判定**: ヒープが空になった場合、gather_merge_clear_tuples を呼び出し、NULL を返す。 4. **返却**: ヒープが空でない場合、ヒープの先頭にある参加者のスロット gm_state->gm_slots[i] を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeGatherMerge.c 内の実行ループ（ExecGatherMerge 等からの参照が確認されている）。
  • **呼び出し先**:
  • gather_merge_init
  • gather_merge_readnext
  • gather_merge_clear_tuples
  • binaryheap 関連関数 (binaryheap_first, binaryheap_replace_first, binaryheap_remove_first, binaryheap_empty)

関連データ構造

  • GatherMergeState: 実行状態とヒープ、スロットを管理する構造体。
  • TupleTableSlot: タプルを保持するスロット。
  • binaryheap: 複数ストリームのマージに使用される優先度付きキュー。

失敗モードと境界条件

  • 全ての入力ストリームが空の場合、ヒープが空となり NULL を返す。
  • 参加者からの読み込みが終了した際、ヒープから正しく削除される必要がある。

保守担当者向けメモ

  • 本関数は MergeAppend の並列版としての役割を担う。
  • binaryheap の操作において、gather_merge_readnext の結果に基づきヒープを更新するロジックが順序維持の要となる。
  • gm_slots 配列とヒープ内のインデックスの整合性に注意すること。
