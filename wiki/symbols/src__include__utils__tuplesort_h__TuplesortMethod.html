TuplesortMethod

定義

  • 種別: enum
  • 定義箇所: src/include/utils/tuplesort.h:76

目的

PostgreSQLのタプルソート処理において、現在実行中または完了したソートアルゴリズムの種別を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	SORT_TYPE_STILL_IN_PROGRESS = 0,
	SORT_TYPE_TOP_N_HEAPSORT = 1 << 0,
	SORT_TYPE_QUICKSORT = 1 << 1,
	SORT_TYPE_EXTERNAL_SORT = 1 << 2,
	SORT_TYPE_EXTERNAL_MERGE = 1 << 3,
} TuplesortMethod;

入力と出力

  • 本シンボルは列挙型であり、関数への引数や構造体のメンバとして利用されます。
  • tuplesort_method_name(TuplesortMethod m) 関数により、列挙値に対応する文字列名を取得可能です。

前提条件と不変条件

  • 並列ソートのインフラストラクチャは、TuplesortMethod が 0（SORT_TYPE_STILL_IN_PROGRESS）であることを前提として動作します。

アルゴリズムまたは振る舞い

  • 各値はビットマスクとして定義されており、ソートの進行状態や手法（メモリ内ソート、外部ソート、マージ処理など）を表現します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/commands/explain.c: ソート手法の判定に使用。
  • src/include/nodes/execnodes.h: bits32 sortMethods メンバとして保持。
  • src/include/utils/tuplesort.h: TuplesortPublic 構造体の sortMethod メンバとして保持。
  • **呼び出し先**:
  • src/backend/utils/sort/tuplesort.c: tuplesort_method_name 関数が本型を引数として受け取ります。

関連データ構造

  • TuplesortPublic: ソート状態を保持する構造体で、本型をメンバとして含みます。
  • execnodes.h 内の sortMethods ビットマスク。

失敗モードと境界条件

  • 該当なし（列挙型のため、定義外の値が代入された場合の挙動は未定義）。

保守担当者向けメモ

  • 新しいソート手法を追加する場合は、ビットマスクの重複を避けて定義してください。
  • tuplesort_method_name 関数に新しい列挙値を追加した際は、対応する文字列変換ロジックを更新してください。
