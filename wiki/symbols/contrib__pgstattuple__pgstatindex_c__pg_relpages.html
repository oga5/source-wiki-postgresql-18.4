pg_relpages

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstatindex.c:391

目的

指定されたリレーション（テーブルまたはインデックス）の物理的なページ数を取得するSQL関数です。

シグネチャまたは形状

Datum pg_relpages(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介してリレーション名（text型）を受け取ります。
  • **出力**: リレーションの総ページ数を int64 型で返します。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要です。
  • 対象リレーションに対して AccessShareLock を取得します。

アルゴリズムまたは振る舞い

1. superuser() を呼び出し、権限がない場合はエラーを発生させます。 2. 入力されたリレーション名を RangeVar に変換します。 3. relation_openrv を使用して対象リレーションを AccessShareLock モードで開きます。 4. pg_relpages_impl 関数を呼び出し、その結果を PG_RETURN_INT64 で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されます（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: pg_relpages_impl を呼び出します。

関連データ構造

  • Relation: PostgreSQLの内部リレーション構造体。
  • RangeVar: リレーション名を解決するための構造体。

失敗モードと境界条件

  • スーパーユーザー権限がない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーとなります。
  • relation_openrv が失敗した場合（リレーションが存在しない等）、標準的なPostgreSQLのエラー処理に従います。

保守担当者向けメモ

  • 本関数は pgstattuple 拡張モジュールの一部です。
  • 1.5以降のバージョンでは superuser() チェックが関数内で行われていますが、古いバージョンとの互換性や権限管理の変更には注意してください。
  • pg_relpages_impl が実際のページカウントロジックを保持しているため、統計計算の変更はそちらを確認してください。
