gist_te

定義

  • 種別: function
  • 定義箇所: contrib/ltree/_ltree_gist.c:388

目的

GiSTインデックスにおいて、ltreeキーが指定されたltreeクエリを包含している可能性があるかを、シグネチャ（ビットベクトル）を用いて判定する。

シグネチャまたは形状

static bool gist_te(ltree_gist *key, ltree *query, int siglen)

入力と出力

  • **入力**:
  • key: 検査対象のltree_gist構造体。
  • query: 検索条件となるltree構造体。
  • siglen: シグネチャの長さ（バイト単位）。
  • **出力**:
  • bool: クエリの全レベルがキーのシグネチャに含まれている場合はtrue、それ以外はfalse。

前提条件と不変条件

  • keyがLTG_ISALLTRUEフラグを保持している場合、無条件でtrueを返す。
  • query内の各レベルは、ltree_crc32_szによって計算されたハッシュ値がkeyのビットベクトル内に存在する必要がある。

アルゴリズムまたは振る舞い

1. keyが全ビットセット状態（LTG_ISALLTRUE）であればtrueを返す。 2. query内の各レベルを順次走査する。 3. 各レベルの名称からltree_crc32_szを用いてハッシュ値（hv）を算出する。 4. AHASHVALマクロで算出したビット位置が、keyのシグネチャ（sign）内でセットされているかを確認する。 5. 一つでもビットがセットされていないレベルがあればfalseを返し、全レベルで確認が取れればtrueを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/_ltree_gist.c:525（直接的な呼び出しを確認）
  • **呼び出し先**:
  • ltree_crc32_sz（ハッシュ計算）
  • LTREE_FIRST, LTG_SIGN, LTG_ISALLTRUE, GETBIT, AHASHVAL, LEVEL_NEXT（マクロ群）

関連データ構造

  • ltree_gist: GiSTインデックス用のキー構造体。
  • ltree: ツリーパスを保持するデータ構造。
  • ltree_level: ltree内の個別のノードレベル。

失敗モードと境界条件

  • queryのレベル数が0の場合、ループは実行されずtrueを返す。
  • siglenが不適切な場合、AHASHVALによるビット位置計算で範囲外アクセスが発生する可能性がある（呼び出し側での保証に依存）。

保守担当者向けメモ

  • 本関数はltreeの包含演算子（@>）のGiSTインデックスサポートの一部として機能する。
  • ltree_crc32_szの計算結果とsiglenの整合性が、インデックスの正確性を左右する。
