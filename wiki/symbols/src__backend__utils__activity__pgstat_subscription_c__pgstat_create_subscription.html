pgstat_create_subscription

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_subscription.c:60

目的

論理レプリケーションのサブスクリプション作成時に、対応する統計エントリを初期化し、トランザクション管理下に登録する。

シグネチャまたは形状

void pgstat_create_subscription(Oid subid)

入力と出力

  • **入力**: subid (Oid) - 作成対象のサブスクリプションを一意に識別するID。
  • **出力**: なし。

前提条件と不変条件

  • pgstat_create_transactional を呼び出すことで、現在のトランザクションがロールバックされた場合に統計エントリが自動的に削除される不変条件を保証する。

アルゴリズムまたは振る舞い

1. pgstat_create_transactional を呼び出し、PGSTAT_KIND_SUBSCRIPTION 型の統計エントリをトランザクション管理下で作成する。 2. pgstat_get_entry_ref を呼び出し、サブスクリプション統計エントリの参照を取得（または作成）する。 3. pgstat_reset_entry を呼び出し、当該サブスクリプションの統計値を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/subscriptioncmds.c:807
  • **呼び出し先**:
  • pgstat_create_transactional
  • pgstat_get_entry_ref
  • pgstat_reset_entry

関連データ構造

  • PGSTAT_KIND_SUBSCRIPTION: 統計の種類を示す識別子。
  • Oid: サブスクリプション識別子。

失敗モードと境界条件

  • 統計サブシステムが初期化されていない場合や、メモリ不足等のリソース制約下では、基盤となる pgstat 関数群がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数はサブスクリプションの作成時にのみ呼び出される。削除時は pgstat_drop_subscription が対応する。
  • トランザクション連動型であるため、サブスクリプション作成のロールバック時に統計情報が残存しない設計となっている。
