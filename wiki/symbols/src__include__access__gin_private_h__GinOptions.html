GinOptions

定義

  • 種別: struct
  • 定義箇所: src/include/access/gin_private.h:26

目的

GIN (Generalized Inverted Index) インデックスの動作を制御する設定パラメータを保持します。reloptions を介してインデックスごとに設定可能な値を管理します。

シグネチャまたは形状

typedef struct GinOptions
{
    int32       vl_len_;        /* varlena header */
    bool        useFastUpdate;  /* use fast updates? */
    int         pendingListCleanupSize; /* maximum size of pending list */
} GinOptions;

入力と出力

  • **入力**: reloptions を通じて設定される useFastUpdate (bool) および pendingListCleanupSize (int)。
  • **出力**: インデックスのメタデータとして、スキャンや挿入処理の挙動を決定するフラグおよび閾値を提供します。

前提条件と不変条件

  • vl_len_ は可変長データ型のヘッダーであり、直接操作してはなりません。
  • pendingListCleanupSize が -1 の場合、デフォルト値が使用されることを示唆するロジックが gin_private.h 内に存在します。

アルゴリズムまたは振る舞い

  • useFastUpdate: true の場合、インデックス挿入時に保留リスト（pending list）を利用した高速更新が有効になります。
  • pendingListCleanupSize: 保留リストの最大サイズを制御します。この値が設定されている場合、インデックスのクリーンアップ処理のトリガーとして使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginutil.c: reloptions の定義およびサイズ計算に使用。
  • src/include/access/gin_private.h: マクロ内で relation->rd_options をキャストして各フィールドを参照。
  • **呼び出し先**: 情報なし。

関連データ構造

  • Relation: インデックスの rd_options フィールドを通じて本構造体にアクセスします。

失敗モードと境界条件

  • pendingListCleanupSize が -1 の場合、明示的な制限値ではなくデフォルトの動作が選択されます。
  • reloptions が正しくキャストされない場合、メモリ破壊や不正な値の読み取りが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は reloptions の一部として扱われるため、フィールドの追加や変更を行う際は ginutil.c のオプション定義と整合性を取る必要があります。
