heapam_fetch_row_version

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_handler.c:182

目的

指定されたタプル識別子（TID）に対応する行バージョンをヒープから取得し、提供されたスロットに格納する。

シグネチャまたは形状

static bool heapam_fetch_row_version(Relation relation, ItemPointer tid, Snapshot snapshot, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • relation: 対象のテーブルリレーション。
  • tid: 取得対象のタプルを指す ItemPointer。
  • snapshot: 可視性判定に使用するスナップショット。
  • slot: 取得したタプルを格納するための TupleTableSlot。
  • **出力**:
  • bool: タプルが正常に取得され、スロットに格納された場合は true、それ以外は false を返す。

前提条件と不変条件

  • slot は TTS_IS_BUFFERTUPLE 型（BufferHeapTupleTableSlot）でなければならない（Assert による検証）。

アルゴリズムまたは振る舞い

1. slot を BufferHeapTupleTableSlot 型としてキャストし、t_self に tid を設定する。 2. heap_fetch を呼び出し、指定されたスナップショットに基づいてヒープからタプルを検索する。 3. heap_fetch が成功した場合:

  • ExecStorePinnedBufferHeapTuple を使用して、取得したタプルとバッファをスロットに格納する（バッファのピンを保持）。
  • スロットの tts_tableOid にリレーションの OID を設定する。
  • true を返す。

    • 4. heap_fetch が失敗した場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_handler.c 内の heapam_methods 構造体（tuple_fetch_row_version フィールド）から参照されている。
  • **呼び出し先**:
  • heap_fetch（src/backend/access/heap/heapam.c で定義）
  • ExecStorePinnedBufferHeapTuple
  • RelationGetRelid

関連データ構造

  • Relation: テーブルリレーション情報。
  • ItemPointer: タプルの物理位置情報。
  • Snapshot: トランザクション可視性制御。
  • TupleTableSlot / BufferHeapTupleTableSlot: タプルデータを保持する実行エンジン用スロット。

失敗モードと境界条件

  • heap_fetch がタプルを見つけられない場合、またはスナップショットの条件を満たさない場合は false を返す。
  • slot が BufferHeapTupleTableSlot ではない場合、アサーション違反により異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は TableAmRoutine のコールバックとして実装されている。
  • バッファのピン管理は ExecStorePinnedBufferHeapTuple に依存しているため、スロットのライフサイクル管理と整合させる必要がある。
  • 複雑な可視性判定やヒープ検索ロジックは heap_fetch に委譲されているため、検索ロジックの変更時は heapam.c を参照すること。
