hex_dec_len

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:263

目的

16進数文字列をバイナリデータへデコードする際に必要なバッファサイズを計算します。

シグネチャまたは形状

static uint64 hex_dec_len(const char *src, size_t srclen)

入力と出力

  • **入力**:
  • src: デコード対象の16進数文字列（本関数内では未使用）。
  • srclen: 入力文字列の長さ。
  • **出力**:
  • デコード後のバイナリデータに必要なバイト数（uint64）。

前提条件と不変条件

  • 入力文字列は16進数形式であることを前提としています。
  • 戻り値は srclen を2で除算した値となります。

アルゴリズムまたは振る舞い

入力された文字列長 srclen を右に1ビットシフト（2で除算）することで、16進数表現（2文字で1バイト）からバイナリサイズを算出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/encode.c 内の hex_decode 関連処理において、メモリ確保のためのサイズ計算用として参照されています。
  • **呼び出し先**: なし。

関連データ構造

  • pg_encoding: 本関数は、この構造体の decode_len フィールドに登録される関数ポインタとして利用されます。

失敗モードと境界条件

  • srclen が奇数の場合、切り捨てが発生します。
  • 非常に大きな srclen が渡された場合のオーバーフローチェックは、本関数ではなく呼び出し側のメモリ確保ロジック（palloc 前のチェック）に依存します。

保守担当者向けメモ

  • 本関数は hex_enc_len や hex_encode と共に enclist 配列に登録され、binary_decode の実行フローの一部として呼び出されます。
  • 16進数からバイナリへの変換ロジックを変更する場合は、このサイズ計算式と実際のデコード処理の整合性を確認してください。
