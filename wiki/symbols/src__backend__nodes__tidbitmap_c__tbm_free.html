tbm_free

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/tidbitmap.c:310

目的

TIDBitmap 構造体およびその内部で確保されたメモリ領域を解放する。

シグネチャまたは形状

void tbm_free(TIDBitmap *tbm)

入力と出力

  • **入力**: tbm (TIDBitmap *) - 解放対象のビットマップ構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • tbm は tbm_create 等で適切に初期化された有効なポインタであること。
  • tbm が NULL の場合、本関数はセグメンテーション違反を引き起こす可能性がある（ソースコード上に NULL チェックは存在しない）。

アルゴリズムまたは振る舞い

1. tbm->pagetable が存在する場合、pagetable_destroy を呼び出してハッシュテーブルを破棄する。 2. tbm->spages が存在する場合、pfree を呼び出してメモリを解放する。 3. tbm->schunks が存在する場合、pfree を呼び出してメモリを解放する。 4. 最後に tbm 自身を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginget.c
  • src/backend/access/gin/ginscan.c
  • src/backend/executor/nodeBitmapAnd.c
  • src/backend/executor/nodeBitmapHeapscan.c
  • src/backend/executor/nodeBitmapOr.c
  • **呼び出し先**:
  • pagetable_destroy (ハッシュテーブル破棄用)
  • pfree (メモリ解放用)

関連データ構造

  • TIDBitmap: 本関数が管理・解放するメインの構造体。

失敗モードと境界条件

  • tbm が NULL の場合、未定義動作となる。
  • 内部ポインタが既に解放済みである場合、二重解放（double free）が発生する可能性がある。

保守担当者向けメモ

  • 本関数はメモリ管理を担うため、TIDBitmap 構造体のメンバ構成に変更があった場合は、解放順序や対象メンバの確認が必要である。
  • 共有メモリを使用するビットマップ（tbm_free_shared_area）とは解放対象が異なるため、混同しないよう注意すること。
