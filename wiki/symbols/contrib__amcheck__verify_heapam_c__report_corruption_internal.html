report_corruption_internal

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_heapam.c:914

目的

ヒープリレーションの検証中に検出された破損情報を、SQLレベルで報告するために tuplestore へ記録する内部ヘルパー関数です。

シグネチャまたは形状

static void report_corruption_internal(Tuplestorestate *tupstore, TupleDesc tupdesc,
                                       BlockNumber blkno, OffsetNumber offnum,
                                       AttrNumber attnum, char *msg);

入力と出力

  • **入力**:
  • tupstore: 破損情報を蓄積する Tuplestorestate。
  • tupdesc: 報告用タプルの構造定義。
  • blkno, offnum: 破損が発生したブロック番号とオフセット番号。
  • attnum: 破損した属性番号（該当しない場合は負の値）。
  • msg: 破損内容を示す文字列（呼び出し側で割り当てられたもの）。
  • **出力**: なし。副作用として tuplestore にタプルが追加されます。

前提条件と不変条件

  • msg 引数は、呼び出し側で動的に割り当てられたメモリ領域を指している必要があります。本関数は pfree(msg) を実行してメモリを解放するため、呼び出し側は関数呼び出し後にこのポインタを参照してはなりません。

アルゴリズムまたは振る舞い

1. 破損情報を格納するための values 配列と nulls 配列を初期化します。 2. blkno, offnum, attnum, msg を Datum に変換して配列にセットします。attnum が負の場合は nulls を真に設定します。 3. メモリリークを防ぐため、引数として受け取った msg を即座に pfree で解放します。 4. heap_form_tuple を使用して報告用タプルを作成します。 5. tuplestore_puttuple を呼び出し、作成したタプルを tupstore に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c 内の report_corruption および report_toast_corruption から呼び出されています。
  • **呼び出し先**:
  • pfree (メモリ解放)
  • heap_form_tuple (タプル構築)
  • tuplestore_puttuple (結果の格納)

関連データ構造

  • Tuplestorestate: 破損情報を一時的に保持する構造体。
  • TupleDesc: 報告用タプルのスキーマ定義。

失敗モードと境界条件

  • tuplestore が work_mem を超える容量を消費する可能性があります。本関数は msg を早期解放することで、メモリ消費の増大を抑制しています。

保守担当者向けメモ

  • 本関数は msg 引数の所有権を実質的に引き継ぎ、解放する設計になっています。将来的に msg の生成元が変更された場合でも、この解放ロジックが適切に機能するか確認してください。
