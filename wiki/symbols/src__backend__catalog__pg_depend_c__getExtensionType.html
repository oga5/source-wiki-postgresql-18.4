getExtensionType

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:831

目的

指定された拡張機能（Extension）に属する特定の名前を持つデータ型（Type）の OID を検索する。

シグネチャまたは形状

Oid getExtensionType(Oid extensionOid, const char *typname)

入力と出力

  • **入力**:
  • extensionOid: 検索対象の拡張機能の OID。
  • typname: 検索対象のデータ型名（文字列）。
  • **出力**:
  • 見つかった場合はその型の Oid、見つからない場合は InvalidOid を返す。

前提条件と不変条件

  • pg_depend システムカタログに対する AccessShareLock を取得して読み取りを行う。
  • 検索対象のオブジェクトは DEPENDENCY_EXTENSION 型の依存関係として pg_depend に記録されている必要がある。

アルゴリズムまたは振る舞い

1. pg_depend テーブルを DependReferenceIndexId インデックスを用いてスキャンし、指定された extensionOid に関連付けられたレコードを抽出する。 2. 抽出した各レコードについて、以下の条件を満たすか確認する。

  • classid が TypeRelationId であること。
  • deptype が DEPENDENCY_EXTENSION であること。

    • 3. 条件を満たす場合、TYPEOID システムキャッシュを使用して型情報を取得し、その名前が typname と一致するか比較する。 4. 一致した場合はその型の OID を返し、スキャンを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/extension.c:299
  • **呼び出し先**:
  • table_open, systable_beginscan, systable_getnext, systable_endscan, table_close (テーブルアクセスAPI)
  • SearchSysCache1, ReleaseSysCache (システムキャッシュAPI)

関連データ構造

  • pg_depend: 依存関係を保持するシステムカタログ。
  • pg_type: データ型の定義を保持するシステムカタログ。
  • Form_pg_depend: pg_depend のタプル構造体。
  • Form_pg_type: pg_type のタプル構造体。

失敗モードと境界条件

  • 指定された拡張機能が存在しない、またはその拡張機能に該当する型が含まれていない場合は InvalidOid を返す。
  • SearchSysCache1 で型情報が見つからない場合は、そのレコードをスキップして検索を継続する。

保守担当者向けメモ

  • 本関数は pg_depend のインデックススキャンに依存しているため、カタログのスキーマ変更時にはインデックス定義との整合性に注意すること。
  • SearchSysCache1 で取得したタプルは ReleaseSysCache で適切に解放されている。
