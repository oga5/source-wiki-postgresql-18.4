xpath_list

定義

  • 種別: function
  • 定義箇所: contrib/xml2/xpath.c:235

目的

XMLドキュメントに対してXPathクエリを実行し、その結果を特定の区切り文字で結合したテキストとして返すSQL呼び出し可能な関数です。

シグネチャまたは形状

Datum xpath_list(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): 対象のXMLドキュメント（text型）
  • PG_GETARG_TEXT_PP(1): 実行するXPath式（text型）
  • PG_GETARG_TEXT_PP(2): 結果の結合に使用する区切り文字（text型）
  • **出力**:
  • Datum: 評価結果のテキスト（text型）、または結果が空の場合はNULL。

前提条件と不変条件

  • libxml2の初期化が完了している必要があります。
  • pgxml_xpathによる評価後、cleanup_workspaceを呼び出してメモリを解放する必要があります。

アルゴリズムまたは振る舞い

1. 入力されたXPath式と区切り文字をxmlChar形式に変換します。 2. pgxml_xpathを呼び出し、XMLドキュメントに対してXPath評価を実行してxmlXPathObjectPtrを取得します。 3. pgxml_result_to_textを用いて、評価結果を区切り文字で結合したtext型に変換します。 4. cleanup_workspaceを呼び出し、ワークスペース内のリソースを解放します。 5. 変換したxpath文字列をpfreeで解放します。 6. 結果がNULLであればSQLのNULLを返し、そうでなければ変換後のテキストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSQLインターフェース（PG_FUNCTION_INFO_V1(xpath_list)により登録）。
  • **呼び出し先**: pgxml_texttoxmlchar, pgxml_xpath, pgxml_result_to_text, cleanup_workspace, pfree。

関連データ構造

  • xpath_workspace: XPath評価のコンテキストやドキュメントを保持する管理構造体。

失敗モードと境界条件

  • pgxml_xpathの評価中にエラーが発生した場合、PostgreSQLの例外処理メカニズムによりelog(ERROR)へ遷移します。
  • 評価結果が空の場合、xpresはNULLとなり、関数はSQLのNULLを返します。

保守担当者向けメモ

  • 本関数はlibxml2のメモリを直接扱うため、cleanup_workspaceの呼び出しを忘れるとメモリリークの原因となります。
  • pgxml_texttoxmlcharで割り当てられたメモリは、関数終了前にpfreeで解放する必要があります。
