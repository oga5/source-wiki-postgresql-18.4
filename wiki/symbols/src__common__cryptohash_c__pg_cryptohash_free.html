pg_cryptohash_free

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash.c:237

目的

ハッシュ計算に使用したコンテキスト構造体を安全に破棄し、メモリを解放する。

シグネチャまたは形状

void pg_cryptohash_free(pg_cryptohash_ctx *ctx)

入力と出力

  • **入力**: pg_cryptohash_ctx *ctx - 解放対象のコンテキストへのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • ctx が NULL の場合、本関数は何も行わず直ちに復帰する。
  • メモリ解放前に、機密情報を含む可能性のあるコンテキスト領域を explicit_bzero で確実に消去する。

アルゴリズムまたは振る舞い

1. ctx が NULL かどうかを判定し、該当する場合は終了する。 2. explicit_bzero を呼び出し、pg_cryptohash_ctx サイズ分のメモリ領域をゼロクリアする。 3. FREE マクロを使用して、コンテキストが占有していたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/libpq/auth-scram.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/checksum_helper.c
  • **呼び出し先**:
  • explicit_bzero（メモリ消去用）
  • FREE（メモリ解放用マクロ）

関連データ構造

  • pg_cryptohash_ctx: ハッシュアルゴリズムの状態を保持する不透明な構造体。

失敗モードと境界条件

  • ctx が NULL の場合は安全に無視される。
  • FREE マクロの挙動は、ビルド環境（フロントエンドかバックエンドか）に応じて free または pfree に解決される。

保守担当者向けメモ

  • 本関数はセキュリティ上の配慮から、解放前に explicit_bzero を使用してハッシュ状態を消去している。メモリ管理の変更を行う際は、この消去処理が維持されていることを確認すること。
