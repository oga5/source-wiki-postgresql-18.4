trgm_mb_char

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_regexp.c:243

目的

マルチバイト文字を固定長のバイト配列として保持し、トリグラム生成処理において文字単位の操作を可能にするためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	char		bytes[MAX_MULTIBYTE_CHAR_LEN];
} trgm_mb_char;

入力と出力

  • **入力**: convertPgWchar 関数等を通じて、pg_wchar 型の文字データが本構造体の bytes フィールドに格納されます。
  • **出力**: fillTrgm 関数等において、本構造体の配列がトリグラム（trgm 型）を生成するためのソースとして利用されます。

前提条件と不変条件

  • bytes 配列のサイズは MAX_MULTIBYTE_CHAR_LEN で固定されています。
  • マルチバイト文字をトリグラムの構成要素として扱うためのコンテナとして機能します。

アルゴリズムまたは振る舞い

正規表現からトリグラムを抽出する過程で、ワイド文字（pg_wchar）をバイト表現に変換して保持します。これにより、インデックス構築やマッチング判定の際に、マルチバイト文字をトリグラムの単位として一貫して扱えるようにします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • convertPgWchar (文字変換時)
  • fillTrgm (トリグラム生成時)
  • TrgmColorInfo 構造体内の wordChars フィールドとして使用
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • trgm: トリグラムの基本データ型。
  • TrgmColorInfo: 本構造体の配列（wordChars）をメンバとして保持します。

失敗モードと境界条件

  • MAX_MULTIBYTE_CHAR_LEN を超える文字データが渡された場合の挙動については、本構造体の定義からは明示されていません。

保守担当者向けメモ

  • 本構造体は trgm_regexp.c 内でのみ使用される補助的なデータ構造です。
  • マルチバイト文字のエンコーディング依存の処理を行う際は、tsearch/ts_locale.h 等のロケール関連関数との整合性に注意してください。
