astreamer_lz4_compressor_free

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_lz4.c:259

目的

LZ4圧縮ストリーマーに関連するリソースを解放し、メモリリークを防ぐ。

シグネチャまたは形状

static void astreamer_lz4_compressor_free(astreamer *streamer)

入力と出力

  • 入力: astreamer *streamer - 解放対象のストリーマーインスタンス。
  • 出力: なし。

前提条件と不変条件

  • 引数 streamer は astreamer_lz4_frame 構造体としてキャスト可能であること。
  • streamer->bbs_next が有効なストリーマーを指しているか、NULLであること。

アルゴリズムまたは振る舞い

1. streamer を astreamer_lz4_frame 型にキャストする。 2. astreamer_free を呼び出し、後続のストリーマー (bbs_next) を再帰的に解放する。 3. LZ4F_freeCompressionContext を呼び出し、LZ4圧縮コンテキストを解放する。 4. pfree を使用して、内部バッファ (bbs_buffer.data) を解放する。 5. pfree を使用して、ストリーマー構造体自体を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/fe_utils/astreamer_lz4.c 内のストリーマーインターフェース定義（.free メンバへの代入）。
  • 呼び出し先: astreamer_free（基底関数）、LZ4F_freeCompressionContext（外部ライブラリ）、pfree（メモリ管理）。

関連データ構造

  • astreamer: 基底ストリーマー構造体。
  • astreamer_lz4_frame: LZ4圧縮・伸長用の内部状態を保持する構造体。

失敗モードと境界条件

  • streamer が NULL の場合、ポインタ参照エラーが発生する可能性がある。
  • bbs_next が既に解放済みである場合、二重解放によるクラッシュの可能性がある。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの free コールバックとして登録される。
  • pfree によるメモリ解放順序は、構造体内部のポインタを先に解放し、最後に構造体本体を解放する順序を守ること。
