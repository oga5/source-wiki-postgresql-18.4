LtreeGistOptions

定義

  • **種別**: struct
  • **定義箇所**: contrib/ltree/ltree.h:310

目的

ltree 拡張モジュールの GiST インデックスにおけるシグネチャ長（siglen）を動的に管理するためのオプション構造体です。

シグネチャまたは形状

typedef struct
{
	int32		vl_len_;		/* varlena header */
	int			siglen;			/* signature length in bytes */
} LtreeGistOptions;

入力と出力

  • **入力**: インデックス作成時や操作時に PG_GET_OPCLASS_OPTIONS() を介して取得されるオプション設定。
  • **出力**: siglen フィールドを通じて、インデックスのシグネチャ長をバイト単位で提供します。

前提条件と不変条件

  • vl_len_ は可変長データ（varlena）のヘッダーであるため、直接操作してはなりません。
  • インデックス操作時は、必ず LTREE_GET_SIGLEN() マクロを介して siglen を取得する必要があります。

アルゴリズムまたは振る舞い

  • GiST インデックスの構築および検索ロジックにおいて、インデックスのシグネチャ長を動的に変更するための設定値として機能します。
  • init_local_reloptions を使用して、インデックスのオプションとして初期化されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/_ltree_gist.c: init_local_reloptions による初期化。
  • contrib/ltree/ltree_gist.c: オプションの解析および初期化。
  • contrib/ltree/ltree.h: LTREE_GET_SIGLEN マクロ内での参照。
  • **呼び出し先**: 呼び出し先に関する情報は提供されていません。

関連データ構造

  • ltree_gist: GiST インデックスのノード格納形式。本構造体で設定された siglen に基づいてシグネチャ長が決定されます。

失敗モードと境界条件

  • siglen の値が不正な場合、インデックスのシグネチャ計算や比較処理において予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • FLEXIBLE_ARRAY_MEMBER を含む構造体と同様に、メモリレイアウトには注意してください。
  • インデックスのディスクフォーマットとの互換性に影響するため、フィールドの追加や変更を行う際は慎重に検討してください。
