TwoPhaseStateData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/twophase.c:176

目的

共有メモリ上で準備済みトランザクション（Prepared Transactions）のリストと、それらを管理するためのメタデータを保持する構造体です。

シグネチャまたは形状

typedef struct TwoPhaseStateData
{
    GlobalTransaction freeGXacts;
    int               numPrepXacts;
    GlobalTransaction prepXacts[FLEXIBLE_ARRAY_MEMBER];
} TwoPhaseStateData;

入力と出力

  • **入力**: 構造体内の prepXacts 配列は max_prepared_xacts に基づく可変長サイズを持ちます。
  • **出力**: 準備済みトランザクションの管理状態（空きリストの先頭、有効なエントリ数、エントリ配列）を提供します。

前提条件と不変条件

  • 本構造体は共有メモリ上に配置されます。
  • prepXacts 配列のサイズは max_prepared_xacts に依存します。
  • 構造体へのアクセスは TwoPhaseStateLock によって保護される必要があります。

アルゴリズムまたは振る舞い

  • 準備済みトランザクションのライフサイクル管理において、freeGXacts を用いて利用可能なスロットを追跡します。
  • numPrepXacts は現在アクティブな準備済みトランザクションの数を保持し、状態の追跡に使用されます。
  • FLEXIBLE_ARRAY_MEMBER を使用して、起動時に設定された最大数に応じたメモリ領域を動的に確保します。

呼び出し元と呼び出し先

  • **呼び出し元**: TwoPhaseShmemInit において共有メモリ領域の初期化時に参照されます。
  • **呼び出し先**: 構造体内の各フィールドは、準備済みトランザクションの検索、割り当て、解放を行う関数群から直接操作されます。
  • **注記**: 具体的な関数呼び出しの連鎖については、提供されたソース excerpt には記載がありません。

関連データ構造

  • GlobalTransactionData: prepXacts 配列が保持する個々の準備済みトランザクションの状態構造体。
  • TwoPhaseState: 本構造体を指すグローバルなポインタ。

失敗モードと境界条件

  • max_prepared_xacts を超えるトランザクションの準備はできません。
  • 共有メモリの初期化時に TwoPhaseStateData のサイズ計算（offsetof を使用）が正しく行われない場合、メモリ破壊が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、サイズ計算には offsetof を使用することが必須です。
  • 構造体内のフィールドを更新する際は、必ず TwoPhaseStateLock を保持していることを確認してください。
