max_parallel_hazard_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/clauses.c:90

目的

クエリの式ツリーを走査する際、並列実行に対する危険度（Parallel Hazard）の状態を保持・伝播するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    char        max_hazard;      /* これまでに発見された最悪の並列危険度 */
    char        max_interesting; /* 興味対象となる最悪の並列危険度 */
    List       *safe_param_ids;  /* 安全とみなす PARAM_EXEC の ID リスト */
} max_parallel_hazard_context;

入力と出力

本構造体は関数引数として渡され、走査中に更新されます。

  • 入力: max_interesting（判定基準）および safe_param_ids（許容されるパラメータID）を初期化して使用します。
  • 出力: 走査完了後、max_hazard に式全体から判定された最悪の危険度が格納されます。

前提条件と不変条件

  • max_hazard および max_interesting は、PostgreSQLの並列安全性レベル（proparallel）に対応する値をとります。
  • 走査対象の式ツリーが NULL の場合、本構造体は更新されません。

アルゴリズムまたは振る舞い

式ツリーの走査（max_parallel_hazard_walker）において、以下の処理を行います。 1. 各ノードを再帰的に訪問し、そのノードが持つ並列危険度を評価します。 2. 評価結果と現在の max_hazard を比較し、より危険度が高い（制約が厳しい）値で max_hazard を更新します。 3. max_parallel_hazard_test を介して、個別の関数や演算子の危険度をコンテキストの状態に反映させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/clauses.c 内の max_parallel_hazard 関数などが、式ツリーの解析を開始する際に本構造体をインスタンス化します。
  • **呼び出し先**: max_parallel_hazard_walker および max_parallel_hazard_test が本構造体を引数として受け取り、状態を更新します。

関連データ構造

  • List: safe_param_ids を保持するために使用されます。
  • Node: max_parallel_hazard_walker が走査する対象のツリー構造です。

失敗モードと境界条件

  • safe_param_ids が NIL の場合、すべての PARAM_EXEC ノードは危険とみなされる可能性があります。
  • 走査中に未知のノード型に遭遇した場合、expression_tree_walker の仕様に従い、適切なエラーハンドリングまたはスキップが行われます。

保守担当者向けメモ

  • max_hazard の値は、並列クエリの実行可否を決定する重要な指標です。この構造体のフィールドを追加・変更する場合は、max_parallel_hazard_walker 内の比較ロジックに影響がないか確認してください。
