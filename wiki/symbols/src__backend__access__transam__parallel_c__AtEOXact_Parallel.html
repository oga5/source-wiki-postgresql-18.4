AtEOXact_Parallel

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/parallel.c:1281

目的

トランザクション終了時（コミットまたはアボート）に、未破棄の並列コンテキスト（ParallelContext）をクリーンアップし、リソースリークを防ぐ。

シグネチャまたは形状

void AtEOXact_Parallel(bool isCommit)

入力と出力

  • **入力**:
  • isCommit (bool): トランザクションがコミットされる場合は true、アボートされる場合は false。
  • **出力**: なし。

前提条件と不変条件

  • グローバル変数 pcxt_list が管理する並列コンテキストのリストを走査する。
  • トランザクション終了時にこの関数が呼び出されることが期待されている。

アルゴリズムまたは振る舞い

1. pcxt_list が空になるまで以下のループを繰り返す。 2. リストの先頭から ParallelContext を取得する。 3. isCommit が true の場合、並列コンテキストがリークしていることを示す警告（WARNING）をログに出力する。 4. DestroyParallelContext(pcxt) を呼び出し、該当する並列コンテキストを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c:2290 (コミット時: AtEOXact_Parallel(true))
  • src/backend/access/transam/xact.c:2911 (アボート時: AtEOXact_Parallel(false))
  • **呼び出し先**:
  • DestroyParallelContext (並列コンテキストの破棄)
  • elog (警告メッセージの出力)

関連データ構造

  • ParallelContext: 並列実行の単位を管理する構造体。
  • pcxt_list: 並列コンテキストを保持する dlist（ソースコード内の文脈より）。

失敗モードと境界条件

  • isCommit が true の場合に並列コンテキストが残存していると、elog により警告が発行される。
  • リストが空の場合は何も行わず正常終了する。

保守担当者向けメモ

  • 本関数はトランザクション終了処理のフックとして機能する。並列クエリ実行後にコンテキストが適切に破棄されていないケースを検知する役割を持つ。
  • DestroyParallelContext の内部実装が変更された場合、本関数での呼び出し順序や引数に影響がないか確認すること。
