CopyMultiInsertBufferCleanup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfrom.c:619

目的

COPY コマンド実行中に使用されたバルク挿入用バッファ（CopyMultiInsertBuffer）のリソースを解放し、関連する状態をクリーンアップする。

シグネチャまたは形状

static inline void CopyMultiInsertBufferCleanup(CopyMultiInsertInfo *miinfo, CopyMultiInsertBuffer *buffer)

入力と出力

  • **入力**:
  • miinfo: バルク挿入全体の管理情報。
  • buffer: 解放対象のバッファ構造体。
  • **出力**: なし（副作用としてメモリ解放および状態更新を行う）。

前提条件と不変条件

  • バッファ内のデータがすべてフラッシュ済みであること（buffer->nused == 0 であることを Assert で検証）。

アルゴリズムまたは振る舞い

1. resultRelInfo から自身へのバックリンク（ri_CopyMultiInsertBuffer）を NULL に設定する。 2. FDW（外部データラッパー）を使用していない場合、FreeBulkInsertState を呼び出して bistate を解放する。 3. バッファ内の各タプルスロット（slots）を ExecDropSingleTupleTableSlot で順次解放する。 4. FDW を使用していない場合、table_finish_bulk_insert を呼び出してテーブルアクセス層のバルク挿入処理を終了させる。 5. 最後に pfree(buffer) を実行し、バッファ構造体自体をメモリから解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c:707
  • src/backend/commands/copyfrom.c:721
  • **呼び出し先**:
  • FreeBulkInsertState
  • ExecDropSingleTupleTableSlot
  • table_finish_bulk_insert
  • pfree

関連データ構造

  • CopyMultiInsertInfo: バルク挿入の全体状態を保持。
  • CopyMultiInsertBuffer: 個別のテーブルに対する挿入バッファ。
  • ResultRelInfo: 挿入先テーブルの実行時情報。

失敗モードと境界条件

  • buffer->nused != 0 の場合、Assert によりアボートする。
  • FDW 使用時は bistate が NULL であることが期待される（Assert で検証）。

保守担当者向けメモ

  • 本関数は static inline で定義されており、copyfrom.c 内のバルク挿入終了処理において一貫して使用される。
  • メモリ管理は copycontext に依存しているため、本関数で解放されるリソースが他のコンテキストと混在していないか注意すること。
