spg_quad_inner_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgquadtreeproc.c:226

目的

SP-GiST四分木インデックスにおいて、内部ノードの検索時にクエリ条件を満たす可能性のある子ノード（象限）を特定し、探索範囲を絞り込む（プルーニング）。

シグネチャまたは形状

Datum spg_quad_inner_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgInnerConsistentIn（検索キー、ノード情報、プレフィックスデータ等）および spgInnerConsistentOut（探索対象ノードのリスト、距離情報等）を受け取る。
  • **出力**: void。結果は out 引数を通じて返される。

前提条件と不変条件

  • in->hasPrefix が真であること。
  • in->allTheSame が偽の場合、in->nNodes は必ず 4 であること。
  • 距離計算（KNN）が必要な場合、in->traversalValues を介して親ノードの境界ボックスが提供されること。

アルゴリズムまたは振る舞い

1. **初期化**: プレフィックスから重心（centroid）を取得。KNN検索時は親ノードの境界ボックスを準備。 2. **全ノード探索**: in->allTheSame が真の場合、全象限を探索対象としてマークし、距離計算が必要なら各象限の境界ボックスを生成して保存。 3. **条件判定**: 各スキャンキー（sk_strategy）に基づき、重心を基準として探索不要な象限をビットマスク which から除外する。

  • RTLeftStrategyNumber 等の空間演算子を用い、クエリと重心の位置関係から該当する象限を特定。
  • RTContainedByStrategyNumber の場合、クエリボックスの各頂点が属する象限を計算し、包含関係を判定。

    • 4. **探索対象の確定**: which ビットマスクに基づき、探索すべき象限（1〜4）を out->nodeNumbers に格納し、必要に応じて各象限の境界ボックスを out->traversalValues に計算・保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTのインデックススキャン処理（詳細はソースコード内に明示的な呼び出し元記述なし）。
  • **呼び出し先**: getQuadrant, getQuadrantArea, spg_key_orderbys_distances, box_copy, DirectFunctionCall2 等のヘルパー関数およびユーティリティ。

関連データ構造

  • spgInnerConsistentIn, spgInnerConsistentOut: SP-GiSTの内部整合性チェック用インターフェース構造体。
  • Point, BOX: 幾何学データ型。

失敗モードと境界条件

  • 未知の sk_strategy が渡された場合、elog(ERROR) を発生させる。
  • which が 0 になった場合、探索を即座に終了する。
  • in->level == 0 の場合、境界ボックスの初期値として無限大（inf）を使用する。

保守担当者向けメモ

  • 象限は 1（右上）から 4（左上）まで反時計回りに定義されている。
  • KNN検索時の traversalValues は、インデックスの深さに関わらず距離計算を可能にするための重要な設計である。
  • RTContainedByStrategyNumber 処理時の DatumGetPointP の使用は、ポインタのキャストを前提とした実装上の工夫である。
