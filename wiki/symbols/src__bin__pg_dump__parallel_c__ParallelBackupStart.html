ParallelBackupStart

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/parallel.c:898

目的

pg_dump または pg_restore における並列処理を開始し、指定された数のワーカープロセス（またはスレッド）を生成・初期化する。

シグネチャまたは形状

ParallelState *ParallelBackupStart(ArchiveHandle *AH)

入力と出力

  • **入力**: ArchiveHandle *AH（アーカイブのハンドル。numWorkers プロパティが 0 より大きい必要がある）。
  • **出力**: ParallelState *（並列実行状態を管理する構造体へのポインタ）。

前提条件と不変条件

  • AH->public.numWorkers > 0 であること。
  • numWorkers が 1 の場合は、ワーカー生成を行わず ParallelState を作成して即座に返却する。
  • ワーカー生成前に fflush(NULL) を呼び出し、stdio の状態を静止させる必要がある。

アルゴリズムまたは振る舞い

1. ParallelState 構造体を割り当て、ワーカー数分の TocEntry 配列と ParallelSlot 配列を初期化する。 2. Windows 環境では getThreadLocalPQExpBuffer を設定する。 3. shutdown_info.pstate に生成した pstate を登録し、終了ハンドラがワーカーをクリーンアップできるようにする。 4. リーダー接続のクエリキャンセル情報を一時的に無効化し、子プロセスへの継承を防ぐ。 5. 指定された数のワーカーをループで生成する:

  • 各ワーカー用に2つのパイプ（リーダー・ワーカー間通信用）を作成する。
  • Unix系では fork() を使用し、子プロセス内で RunWorker を実行する。
  • Windows系では _beginthreadex を使用し、init_spawned_worker_win32 を経由してワーカーを開始する。

    • 6. 全ワーカー生成後、リーダー側で SIGPIPE を無視する設定を行い、クエリキャンセル情報を再確立し、set_cancel_pstate を呼び出してシグナルハンドラがワーカーへシグナルを転送できるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_dump/pg_backup_directory.c
  • **呼び出し先**:
  • pg_malloc, pg_malloc0, pg_fatal
  • pgpipe
  • RunWorker (Unix系)
  • _beginthreadex (Windows系)
  • set_archive_cancel_info, set_cancel_pstate

関連データ構造

  • ParallelState: 並列実行の全体管理構造体。
  • ParallelSlot: 各ワーカーの通信パイプやプロセス/スレッドIDを保持。
  • ArchiveHandle: アーカイブの構成情報。
  • WorkerInfo: Windows環境でワーカーへ引数を渡すための一時構造体。

失敗モードと境界条件

  • numWorkers が 1 の場合、並列化処理をスキップして正常終了する。
  • パイプ作成（pgpipe）に失敗した場合、pg_fatal によりプロセスが異常終了する。
  • fork() に失敗した場合、pg_fatal によりプロセスが異常終了する。

保守担当者向けメモ

  • Windows環境では fork が存在しないため、スレッドとソケットを用いたパイプエミュレーションが行われている。
  • シグナルハンドラが子プロセスに継承されないよう、signal_info.pstate の設定タイミングに注意が必要。
  • closesocket を使用して、不要になったパイプのファイル記述子を適切に閉じることでリソースリークを防いでいる。
