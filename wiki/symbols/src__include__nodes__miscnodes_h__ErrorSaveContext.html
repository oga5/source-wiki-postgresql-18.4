ErrorSaveContext

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/miscnodes.h:44

目的

関数呼び出し間で「ソフトエラー（Soft Error）」情報を安全に伝播させるためのコンテキスト構造体です。例外を投げる代わりに、エラー状態を保持して呼び出し元へ通知するために使用されます。

シグネチャまたは形状

typedef struct ErrorSaveContext
{
    NodeTag     type;
    bool        error_occurred; /* エラー発生時に true に設定 */
    bool        details_wanted; /* 詳細情報の要求フラグ */
    ErrorData  *error_data;     /* エラー詳細情報 */
} ErrorSaveContext;

入力と出力

  • **入力**: 呼び出し元が makeNode(ErrorSaveContext) 等で初期化し、FunctionCallInfo.context 等に設定して渡します。
  • **出力**: 下位関数がエラーを検出した場合、error_occurred が true に更新され、必要に応じて error_data に詳細が格納されます。

前提条件と不変条件

  • NodeTag を保持するため、IsA(context, ErrorSaveContext) による実行時の型判別が可能です。
  • 使用する際は、呼び出し先がエラー情報を書き込むメモリコンテキストの生存期間に注意が必要です（通常は短命なコンテキストでの利用を想定）。

アルゴリズムまたは振る舞い

1. **初期化**: 呼び出し元がコンテキストを作成し、実行状態にセットします。 2. **エラー捕捉**: 下位関数がエラーを検出した際、例外を発生させる代わりに本構造体の error_occurred を true に設定します。 3. **判定**: 呼び出し元は SOFT_ERROR_OCCURRED(escontext) マクロを使用してエラーの有無を判定し、必要に応じて error_data を参照します。

呼び出し元と呼び出し先

  • **呼び出し元（例）**: src/backend/commands/copyfrom.c、src/backend/executor/execExpr.c 等で ErrorSaveContext が生成・利用されています。
  • **呼び出し先**: src/backend/executor/execExprInterp.c 等で Assert(IsA(fcinfo_in->context, ErrorSaveContext)) を通じて型確認が行われています。

関連データ構造

  • NodeTag: ノードの型識別子。
  • ErrorData: src/include/utils/elog.h で定義されるエラー詳細情報。

失敗モードと境界条件

  • error_data が NULL の場合、details_wanted が false であるか、詳細情報が記録されていない可能性があります。
  • 呼び出し元が ErrorSaveContext を適切に初期化していない場合、IsA チェックで失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は SQL ツリーの構築には使用されません。C 言語レベルの API 設計において、戻り値以外で状態を返すための補助的な仕組みです。
  • SOFT_ERROR_OCCURRED マクロと併用することで、エラー判定の簡潔さを維持してください。
