RangeBound

定義

  • 種別: struct
  • 定義箇所: src/include/utils/rangetypes.h:62

目的

範囲型（Range Types）の境界（下限または上限）をプログラム内で表現し、操作するための構造体です。

シグネチャまたは形状

typedef struct {
    Datum       val;        /* 境界値（存在する場合） */
    bool        infinite;   /* 境界が +/- 無限大かどうか */
    bool        inclusive;  /* 境界が包含的（inclusive）か除外的（exclusive）か */
    bool        lower;      /* 下限（lower）か上限（upper）か */
} RangeBound;

入力と出力

  • 本構造体は、range_deserialize 関数によって物理ストレージ形式（RangeType）から展開され、range_serialize 関数によって再びストレージ形式へ変換されます。

前提条件と不変条件

  • val フィールドは、infinite が true の場合は使用されません。
  • 範囲型の境界値は、要素型の typalign に依存して配置されます。

アルゴリズムまたは振る舞い

  • 範囲型のシリアライズ/デシリアライズ処理において、範囲の包含性や無限大の状態を保持するために使用されます。
  • 比較演算や集合演算（range_union_internal 等）を実行する際、この構造体に展開された状態でロジックが処理されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/multirangetypes.c 内の複数の関数（行 86, 87, 748, 807, 861, 862, 881, 882, 913, 1513, 1514, 1536）で、マルチレンジ型の操作に関連して使用されています。
  • 呼び出し先: 提供されたソース情報には、本構造体が直接呼び出す関数や、本構造体を呼び出す具体的な関数名は記載されていません。

関連データ構造

  • RangeType: 範囲型のメモリ上の表現。
  • RangeBound を含むマルチレンジ型関連の処理は src/include/utils/multirangetypes.h に定義されています。

失敗モードと境界条件

  • RANGE_LB_NULL および RANGE_UB_NULL フラグが定義されていますが、これらは現在使用されておらず、将来的な拡張のために予約されています。

保守担当者向けメモ

  • RangeBound は可変長構造体である RangeType の内部状態を扱うため、直接フィールドを操作する際は、rangetypes.c で定義されたシリアライズ/デシリアライズのロジックに従ってください。
