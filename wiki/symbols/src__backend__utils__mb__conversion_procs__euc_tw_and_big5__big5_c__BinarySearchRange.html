BinarySearchRange

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/big5.c:207

目的

変換テーブル（codes_t構造体配列）を二分探索し、BIG5とCNS 11643間の文字コード変換を計算によって導出します。

シグネチャまたは形状

static unsigned short BinarySearchRange(const codes_t *array, int high, unsigned short code)

入力と出力

  • **入力**:
  • array: 変換範囲の開始点と対応するピアコードを保持する codes_t 型の配列。
  • high: 配列の検索範囲上限（インデックス）。
  • code: 変換対象の文字コード。
  • **出力**: 変換後の文字コード（unsigned short）。変換不能な場合は 0 を返します。

前提条件と不変条件

  • array はコード値に基づいて昇順にソートされている必要があります。
  • BIG5の文字コード構造（0xA1-0xFEと0x40-0x7E/0xA1-0xFEの2領域）に基づいたオフセット計算が前提となります。

アルゴリズムまたは振る舞い

1. 指定された範囲内で code が含まれる区間を二分探索します。 2. 該当区間が見つかった場合、code の値に応じて以下の処理を行います。

  • **BIG5からCNSへの変換 (code >= 0xa140U)**:
  • BIG5の2つの領域（0x40-0x7Eと0xA1-0xFE）の不連続性を補正するバイアス（-0x22）を適用し、距離を算出します。
  • 算出した距離を基にCNSコードのオフセットを計算し、最終的なコードを導出します。
  • **CNSからBIG5への変換**:
  • ISO文字セットの範囲（0x21-0xFE）に基づき距離を算出します。
  • BIG5の領域構造に合わせてコードを再構成します。

    • 3. 該当する区間がない、または peer が 0 の場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mb/conversion_procs/euc_tw_and_big5/big5.c 内の以下の関数から呼び出されます。
  • BIG5toCNS（big5Level1ToCnsPlane1 および big5Level2ToCnsPlane2 を使用）
  • CNStoBIG5（cnsPlane1ToBig5Level1 および cnsPlane2ToBig5Level2 を使用）
  • **呼び出し先**:
  • 本関数は計算ロジックのみで構成されており、他の関数を呼び出しません。

関連データ構造

  • codes_t: 変換範囲の開始点と対応するピアコードを保持する構造体。

失敗モードと境界条件

  • 検索範囲内に code が存在しない場合、0 を返します。
  • array[mid].peer が 0 の場合、変換不能とみなして 0 を返します。
  • BIG5の特殊な領域境界（0x7Eと0xA1の間の不連続性）において、バイアス補正が正しく行われない場合は誤ったコードを生成する可能性があります。

保守担当者向けメモ

  • BIG5の文字コード構造（2つの領域）に依存した計算を行っているため、マッピングの修正や新しいコードの追加時には、distance 計算内のバイアス値（-0x22等）や領域判定ロジックを慎重に確認してください。
  • 本関数はフロントエンドとバックエンドの両方で利用されるため、PostgreSQLのコア機能への依存を避ける実装となっています。
