pg_bswap32

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/pg_bswap.h:63

目的

32ビット符号なし整数のバイト順序を反転（エンディアン変換）します。

シグネチャまたは形状

static inline uint32 pg_bswap32(uint32 x)

入力と出力

  • **入力**: uint32 x （変換対象の32ビット符号なし整数）
  • **出力**: uint32 （バイト順序が反転された値）

前提条件と不変条件

  • 入力値は符号なし整数として扱われます。符号付き整数を渡す場合は、事前にキャストを行う必要があります。
  • 本関数は、コンパイラ組み込み関数（__builtin_bswap32 等）が利用できない環境におけるフォールバック実装です。

アルゴリズムまたは振る舞い

ビットシフトと論理和演算を用いて、以下の順序でバイトを入れ替えます。 1. 第1バイト（最上位）を第4バイトへ移動 2. 第2バイトを第3バイトへ移動 3. 第3バイトを第2バイトへ移動 4. 第4バイト（最下位）を第1バイトへ移動 これらを論理和で結合し、32ビットの値を再構成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network.c:648
  • src/include/port/pg_crc32c.h:160 （FIN_CRC32C マクロ内）
  • **呼び出し先**: なし（純粋な算術演算のみを実行）
  • 注: pg_bswap32 は、pg_hton32、pg_ntoh32、および DatumBigEndianToNative マクロの定義にも使用されています。

関連データ構造

  • uint32 （標準的な32ビット符号なし整数型）

失敗モードと境界条件

  • 境界条件: 入力値が 0 または 0xFFFFFFFF の場合、反転後もそれぞれ同じ値となります。
  • 失敗モード: 特になし。

保守担当者向けメモ

  • 本関数はフォールバック実装です。環境によっては、コンパイル時に __builtin_bswap32 や _byteswap_ulong などのプラットフォーム固有の最適化されたマクロ定義が優先的に使用される場合があります。
  • 符号付き整数に適用する際は、予期せぬ動作を避けるため、呼び出し側で適切なキャストを行ってください。
