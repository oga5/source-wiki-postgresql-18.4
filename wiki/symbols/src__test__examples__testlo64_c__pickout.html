pickout

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlo64.c:78

目的

PostgreSQL のラージオブジェクトから、指定された 64 ビットオフセット位置を起点としてデータを読み取り、標準エラー出力へ表示する。

シグネチャまたは形状

static void pickout(PGconn *conn, Oid lobjId, int64_t start, int len)

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル (PGconn *)。
  • lobjId: 対象ラージオブジェクトの OID (Oid)。
  • start: 読み取り開始位置（64 ビットオフセット、int64_t）。
  • len: 読み取るバイト数 (int)。
  • **出力**: なし（標準エラー出力への書き出しのみ）。

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • lobjId は読み取り権限のあるラージオブジェクトを指していること。
  • len は読み取りバッファとして確保可能なサイズであること。

アルゴリズムまたは振る舞い

1. lo_open を使用してラージオブジェクトを読み取りモードで開く。 2. lo_lseek64 を使用して、指定された 64 ビットオフセット start へシークする。 3. lo_tell64 を呼び出し、現在の位置が start と一致することを確認する。 4. malloc で len + 1 バイトのバッファを確保する。 5. lo_read をループ内で呼び出し、指定された長さのデータを読み取る。 6. 読み取ったデータに終端文字を付与し、fprintf で標準エラー出力へ出力する。 7. 読み取り完了後、free でバッファを解放し、lo_close でラージオブジェクトを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/examples/testlo64.c:280 にて 4294967000U をオフセットとして呼び出されている。
  • **呼び出し先**: lo_open, lo_lseek64, lo_tell64, lo_read, lo_close (いずれも libpq API)。

関連データ構造

  • PGconn: PostgreSQL 接続オブジェクト。
  • Oid: ラージオブジェクト識別子。

失敗モードと境界条件

  • lo_open が失敗した場合、エラーメッセージを出力する。
  • lo_lseek64 が失敗した場合、PQerrorMessage を含むエラーメッセージを出力する。
  • lo_tell64 が期待値と一致しない場合、エラーメッセージを出力する。
  • lo_read が 0 以下の値を返した場合、ループを終了する。

保守担当者向けメモ

  • 本関数は 64 ビットオフセット API を使用しているため、4GB を超えるラージオブジェクトの検証に適している。
  • buf の確保に malloc を使用しているため、呼び出し側の len が極端に大きい場合はメモリ不足のリスクがある。
