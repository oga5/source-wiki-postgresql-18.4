BackendStartupData

定義

  • 種別: struct
  • 定義箇所: src/include/tcop/backend_startup.h:44

目的

Postmasterからバックエンドプロセスへ、接続確立に必要な初期化情報およびタイムスタンプを引き継ぐために使用されます。

シグネチャまたは形状

typedef struct BackendStartupData
{
    CAC_state    canAcceptConnections;
    TimestampTz  socket_created;
    TimestampTz  fork_started;
} BackendStartupData;

入力と出力

  • **入力**: Postmasterプロセスがバックエンド起動時に設定し、受け渡します。
  • **出力**: バックエンドプロセス側で接続状態の判定や、接続確立にかかる時間の計測に使用されます。

前提条件と不変条件

  • src/backend/tcop/backend_startup.c において、startup_data_len が sizeof(BackendStartupData) と一致することがアサーションにより検証されます。

アルゴリズムまたは振る舞い

1. **接続状態の管理**: canAcceptConnections フィールドにより、サーバーが現在接続を受け入れ可能な状態（CAC_state）であるかをバックエンドに伝達します。 2. **時間計測**:

  • socket_created: クライアントソケットが作成された時刻を保持します。
  • fork_started: Postmasterがプロセス生成（fork等）を開始した時刻を保持します。
  • これらは主にクライアント接続およびWAL sender接続のパフォーマンス分析やログ出力に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（設定側）**: src/backend/postmaster/postmaster.c および src/backend/postmaster/launch_backend.c がこの構造体を初期化・設定します。
  • **呼び出し先（参照側）**: src/backend/tcop/backend_startup.c がバックエンド起動時にこのデータを受け取り、初期化処理に利用します。

関連データ構造

  • CAC_state: 接続受け入れ状態を定義する列挙型。
  • TimestampTz: utils/timestamp.h で定義される時刻型。

失敗モードと境界条件

  • 構造体のサイズが期待値と異なる場合、バックエンドの初期化プロセスでアサーションエラーが発生します。
  • socket_created および fork_started は、クライアント接続およびWAL sender接続以外では使用されない可能性があります。

保守担当者向けメモ

  • 本構造体はバックエンド起動の初期フェーズに特化しています。認証後の詳細な処理については tcopprot.h を参照してください。
  • 構造体のフィールドを追加・変更する場合は、src/backend/tcop/backend_startup.c 側のサイズ検証ロジックと整合性を取る必要があります。
