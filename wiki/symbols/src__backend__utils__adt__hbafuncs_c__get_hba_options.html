get_hba_options

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/hbafuncs.c:51

目的

HbaLine 構造体に格納された認証設定情報を、SQLの text[] 型（テキスト配列）に変換して返します。

シグネチャまたは形状

static ArrayType *get_hba_options(HbaLine *hba)

入力と出力

  • **入力**: HbaLine *hba（認証設定行の構造体）
  • **出力**: ArrayType *（key=value 形式の文字列を格納した配列。オプションが存在しない場合は NULL）

前提条件と不変条件

  • noptions は MAX_HBA_OPTIONS を超えてはならない（Assert により検証）。
  • 認証方式（auth_method）に応じて、対応するオプションフィールドのみが配列に追加されます。

アルゴリズムまたは振る舞い

1. noptions を 0 に初期化し、options 配列を準備します。 2. hba->auth_method に基づき、以下の条件でオプションを CStringGetTextDatum を用いて配列に追加します。

  • GSS / SSPI: include_realm, krb_realm
  • 共通: usermap, clientcert
  • PAM: pamservice
  • LDAP: ldapserver, ldapport, ldapscheme, ldaptls, ldapprefix, ldapsuffix, ldapbasedn, ldapbinddn, ldapbindpasswd, ldapsearchattribute, ldapsearchfilter, ldapscope
  • RADIUS: radiusservers, radiussecrets, radiusidentifiers, radiusports
  • OAuth: issuer, scope, validator, delegate_ident_mapping

    • 3. オプションが1つ以上存在する場合、construct_array_builtin を使用して TEXTOID 型の配列を構築して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/hbafuncs.c:366（fill_hba_line 関数内から呼び出される）
  • **呼び出し先**: psprintf, CStringGetTextDatum, construct_array_builtin

関連データ構造

  • HbaLine: 認証設定行を保持する構造体（src/include/libpq/hba.h で定義）。
  • ArrayType: PostgreSQL の配列型内部表現。

失敗モードと境界条件

  • オプションが一つも設定されていない場合、NULL を返します。
  • MAX_HBA_OPTIONS を超えるオプションが追加された場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 新しい認証オプションを追加した場合は、MAX_HBA_OPTIONS の値が十分か確認してください。
  • 本関数は pg_hba_file_rules ビューの出力生成プロセスの一部として機能します。
