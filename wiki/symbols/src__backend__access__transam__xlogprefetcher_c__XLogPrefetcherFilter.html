XLogPrefetcherFilter

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xlogprefetcher.c:160

目的

リカバリ処理中、特定のデータブロックに対する先読み（Prefetching）を一時的に抑制するためのフィルタ条件を保持します。リプレイが追いつく前に存在しないファイルや切り詰められた範囲へアクセスし、エラーが発生することを防ぐために使用されます。

シグネチャまたは形状

typedef struct XLogPrefetcherFilter
{
	RelFileLocator rlocator;        /* 対象のリレーション */
	XLogRecPtr	filter_until_replayed; /* フィルタを解除するLSN */
	BlockNumber filter_from_block;  /* フィルタ対象の開始ブロック番号 */
	dlist_node	link;               /* リスト管理用リンク */
} XLogPrefetcherFilter;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • 入力: XLogPrefetcherAddFilter 等を通じて、リレーション情報、LSN、ブロック番号が設定されます。
  • 出力: XLogPrefetcherNextBlock 等の判定ロジックにおいて、先読みの可否を判断する材料として参照されます。

前提条件と不変条件

  • filter_until_replayed に到達するまでの間、rlocator で指定されたリレーションの filter_from_block 以降への先読みは抑制されるべきです。
  • dlist_node を介してリスト管理されるため、メモリ上の生存期間は XLogPrefetcher のライフサイクルに依存します。

アルゴリズムまたは振る舞い

  • XLogPrefetcher はこの構造体をリストとして保持します。
  • 先読み対象のブロックを特定する際、このフィルタリストを走査し、現在のリプレイ位置とブロック番号がフィルタ条件に合致する場合、当該ブロックの先読みをスキップします。
  • リプレイが進み、filter_until_replayed を超えたフィルタは順次解放されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogprefetcher.c 内のフィルタ管理関数（XLogPrefetcherAddFilter 等）および先読み判定ロジック。
  • 呼び出し先: 本構造体自体はデータ保持用のため、直接的な呼び出し先はありません。

関連データ構造

  • RelFileLocator: フィルタ対象のリレーションを特定するために使用。
  • XLogRecPtr: フィルタの有効期限（LSN）を管理。
  • dlist_node: フィルタリストの構築に使用。
  • XLogPrefetcher: 本構造体を管理するメイン構造体。

失敗モードと境界条件

  • フィルタリストの管理不備により、本来先読みすべきブロックが永続的にスキップされる可能性があります。
  • filter_until_replayed の設定値が誤っている場合、リカバリのパフォーマンス低下や、逆に保護対象外のアクセスによるエラーを招く恐れがあります。

保守担当者向けメモ

  • 本構造体は XLogPrefetcher の内部的な最適化機構の一部です。
  • フィルタの追加・削除ロジックを変更する際は、dlist の整合性が保たれていることを確認してください。
  • フィルタの適用範囲が広すぎると先読みの効果が減退するため、必要最小限の範囲で設定されるよう設計されています。
