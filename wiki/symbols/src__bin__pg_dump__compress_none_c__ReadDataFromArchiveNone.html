ReadDataFromArchiveNone

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_none.c:29

目的

アーカイブからデータを読み込み、圧縮処理を介さずにそのまま出力先へ転送する。

シグネチャまたは形状

static void ReadDataFromArchiveNone(ArchiveHandle *AH, CompressorState *cs)

入力と出力

  • **入力**:
  • AH: アーカイブのハンドル（ArchiveHandle）。
  • cs: 圧縮状態を管理する構造体（CompressorState）。readF 関数ポインタを介してデータソースにアクセスする。
  • **出力**: なし（副作用として ahwrite を通じたデータ転送を行う）。

前提条件と不変条件

  • cs->readF が有効な読み込み関数を指していること。
  • DEFAULT_IO_BUFFER_SIZE 分のメモリ確保が成功すること。

アルゴリズムまたは振る舞い

1. DEFAULT_IO_BUFFER_SIZE サイズのバッファを動的に確保する。 2. cs->readF を呼び出し、データがなくなるまでループする。 3. 読み込んだデータを ahwrite を使用してアーカイブハンドルへ書き出す。 4. ループ終了後、確保したバッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は確認できないが、src/bin/pg_dump/compress_none.c:69 にて cs->readData に本関数が代入されている。
  • **呼び出し先**: cs->readF（データ読み込み）、ahwrite（データ書き出し）、pg_malloc、free。

関連データ構造

  • ArchiveHandle: アーカイブ操作のコンテキスト。
  • CompressorState: 圧縮・非圧縮の抽象化状態を保持する構造体。

失敗モードと境界条件

  • pg_malloc によるメモリ確保失敗時は、内部で pg_fatal が呼び出されプロセスが終了する可能性がある。
  • cs->readF がエラーを返した場合、ループが終了し処理が完了する。

保守担当者向けメモ

  • 本関数は非圧縮モード専用の転送ロジックである。
  • private_data メンバには FILE* が格納されている前提で readF が動作する。
