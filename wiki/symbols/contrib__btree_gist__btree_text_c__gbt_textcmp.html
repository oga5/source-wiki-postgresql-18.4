gbt_textcmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_text.c:72

目的

GiSTインデックスにおいて、text型の2つの値を比較し、その順序関係を判定するための比較関数を提供します。

シグネチャまたは形状

int32 gbt_textcmp(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の text 型データへのポインタ。
  • collation: 照合順序（Collation）のOID。
  • flinfo: 関数マネージャ情報。
  • **出力**:
  • int32: a が b より小さい場合は負の値、等しい場合は 0、大きい場合は正の値を返します。

前提条件と不変条件

  • a および b は有効な text 型のデータである必要があります。
  • 内部で bttextcmp を呼び出すため、PostgreSQLの標準的なテキスト比較ルールに従います。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力ポインタ a および b を Datum 型に変換します。 2. DirectFunctionCall2Coll を介して、PostgreSQLの標準関数 bttextcmp を呼び出します。 3. bttextcmp の戻り値（int32）をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_text.c 内の tinfo 構造体（gbtree_vinfo）で参照されています。
  • **呼び出し先**: bttextcmp（PostgreSQLのコア比較関数）。

関連データ構造

  • gbtree_vinfo: text 型のインデックス操作に必要な比較関数を保持する構造体。

失敗モードと境界条件

  • a または b が NULL の場合、PointerGetDatum および bttextcmp の仕様に依存します。
  • 照合順序（collation）が不正な場合、bttextcmp の実行時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は text 型専用の比較ラッパーです。bpchar 型については別途 gbt_bpcharcmp が定義されています。
  • DirectFunctionCall2Coll を使用しているため、関数呼び出しのオーバーヘッドを最小限に抑えています。
