RelfilenumberMapInvalidateCallback

定義

  • 種別: static function
  • 定義箇所: src/backend/utils/cache/relfilenumbermap.c:51

目的

pg_class の変更に伴い、RelfilenumberMapHash キャッシュ内の整合性が失われたエントリを削除または無効化する。

シグネチャまたは形状

static void RelfilenumberMapInvalidateCallback(Datum arg, Oid relid)

入力と出力

  • **入力**:
  • arg: コールバック登録時に渡される Datum（未使用）。
  • relid: 無効化対象の Oid。InvalidOid の場合は全キャッシュのクリアを意味する。
  • **出力**: なし（副作用としてハッシュテーブルを更新）。

前提条件と不変条件

  • RelfilenumberMapHash が初期化済みであること（Assert により検証）。
  • キャッシュの整合性を維持するため、relid が InvalidOid の場合はすべてのエントリを削除する。

アルゴリズムまたは振る舞い

1. hash_seq_init を使用して RelfilenumberMapHash の全エントリを走査する。 2. 各エントリに対し、以下のいずれかの条件に合致する場合に hash_search (操作: HASH_REMOVE) を実行して削除する。

  • relid == InvalidOid（完全リセット）
  • entry->relid == InvalidOid（負のキャッシュエントリの削除）
  • entry->relid == relid（特定のリレーションのフラッシュ）

    • 3. 削除処理中にハッシュテーブルの不整合が検出された場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: CacheRegisterRelcacheCallback を介してリレーションキャッシュの無効化イベントから呼び出される。
  • **呼び出し先**: hash_seq_init, hash_seq_search, hash_search（いずれも utils/hsearch.h）。

関連データ構造

  • RelfilenumberMapHash: キャッシュの実体であるハッシュテーブル。
  • RelfilenumberMapEntry: キャッシュ内の各エントリ構造体。
  • HASH_SEQ_STATUS: ハッシュテーブル走査用の状態管理構造体。

失敗モードと境界条件

  • RelfilenumberMapHash が初期化されていない状態で呼び出された場合、Assert により停止する。
  • 削除対象のキーがハッシュテーブル内に見つからない場合、elog(ERROR) によりエラーとなる。

保守担当者向けメモ

  • 本関数は「負のキャッシュ（存在しないファイル番号の記録）」も削除対象とするため、カタログの変更時には必ず呼び出される必要がある。
  • 一時テーブルは本キャッシュの管理対象外であるため、本コールバックによる影響を受けない。
