base_yy_extra_type

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/gramparse.h:35

目的

PostgreSQL の SQL 解析フェーズにおいて、字句解析器（Flex）と構文解析器（Bison）の間で共有される実行状態を保持する構造体です。

シグネチャまたは形状

typedef struct base_yy_extra_type
{
    core_yy_extra_type core_yy_extra;
    bool        have_lookahead;
    int         lookahead_token;
    core_YYSTYPE lookahead_yylval;
    YYLTYPE     lookahead_yylloc;
    char       *lookahead_end;
    char        lookahead_hold_char;
    List       *parsetree;
} base_yy_extra_type;

入力と出力

  • **入力**: parser_init() を通じて解析器に渡され、字句解析および構文解析の過程で状態が更新されます。
  • **出力**: 構文解析の最終結果が parsetree フィールドに格納されます。

前提条件と不変条件

  • gram.h は scanner.h の後にインクルードされている必要があります（YYLTYPE の定義依存）。
  • pg_yyget_extra マクロは、この構造体のメモリレイアウトに依存した最適化を行っているため、構造体の変更には注意が必要です。

アルゴリズムまたは振る舞い

1. **初期化**: parser_init() により構造体が初期化されます。 2. **字句解析**: base_yylex() がトークンを生成し、必要に応じてルックアヘッド情報（have_lookahead 等）を更新します。 3. **構文解析**: base_yyparse() が字句解析器と連携し、構築された構文木を parsetree に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parser.c 内の yyextra 変数としてインスタンス化され、pg_yyget_extra マクロを通じて参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • core_yy_extra_type: 字句解析器のコア状態を保持。
  • List: 構文解析結果（parsetree）を保持するリスト構造。

失敗モードと境界条件

  • ルックアヘッド情報の不整合（have_lookahead が正しく管理されない場合）は、字句解析の誤動作を招く可能性があります。

保守担当者向けメモ

  • 本構造体はコア解析コンポーネント（parser.c, gram.y, scan.l）専用のプライベートなインターフェースです。外部モジュールからは参照しないでください。
