finalize_grouping_exprs_walker

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_agg.c:1615

目的

式ツリーを走査し、GroupingFunc ノード内の引数が現在のクエリレベルにおける有効なグループ化式であるかを検証し、対応する ressortgroupref を解決してノードに設定する。

シグネチャまたは形状

static bool finalize_grouping_exprs_walker(Node *node, substitute_grouped_columns_context *context)

入力と出力

  • **入力**:
  • node: 走査対象の式ノード。
  • context: グループ化句のリストや現在のクエリレベル（sublevels_up）を保持するコンテキスト。
  • **出力**:
  • 走査を継続する場合は false、中断する場合は true を返す。

前提条件と不変条件

  • context->sublevels_up は現在のクエリのネストレベルを正確に反映している必要がある。
  • GroupingFunc の引数は、現在のクエリレベルのグループ化式と一致しなければならない。

アルゴリズムまたは振る舞い

1. **ノードのスキップ**: Const や Param は常に有効とみなし、走査を終了する。 2. **Aggref の処理**: 現在のレベルの Aggref に遭遇した場合、その aggdirectargs に対して再帰的に走査を行う。より深いレベルの Aggref は無視する。 3. **GroupingFunc の解決**:

  • 現在のレベルに属する GroupingFunc を特定する。
  • 各引数が groupClauses 内のいずれかの式と一致するかを確認する（Var の場合は属性番号とレベルで照合、それ以外は equal() で比較）。
  • 一致した式の ressortgroupref を取得し、GroupingFunc->refs リストに格納する。

    • 4. **サブクエリの処理**: Query ノードに遭遇した場合は sublevels_up をインクリメントして再帰する。 5. **デフォルトの走査**: expression_tree_walker を使用してツリー全体を再帰的に走査する。

呼び出し元と呼び出し先

  • **呼び出し元**: finalize_grouping_exprs から呼び出される。また、自身を再帰的に呼び出す。
  • **呼び出し先**: expression_tree_walker, query_tree_walker, flatten_join_alias_vars, equal, ereport。

関連データ構造

  • substitute_grouped_columns_context: 走査状態を保持する構造体。
  • GroupingFunc: 走査対象となる集約関数ノード。
  • Aggref: 集約関数ノード。
  • TargetEntry: groupClauses 内の要素。

失敗モードと境界条件

  • 引数がグループ化式と一致しない場合、ERRCODE_GROUPING_ERROR を発生させてエラー終了する。
  • GroupingFunc が現在のレベルより深いレベルに属している場合は走査を停止する。

保守担当者向けメモ

  • sublevels_up の管理は Query ノードの出入りで適切に行われているため、再帰呼び出し時のカウンタ操作を維持すること。
  • GroupingFunc の引数解決において、hasJoinRTEs が真の場合は flatten_join_alias_vars を介して正規化を行う必要がある。
