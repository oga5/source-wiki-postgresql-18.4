format_operator_qualified

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regproc.c:798

目的

指定された演算子OIDを、スキーマ修飾された文字列形式（例: schema.operator）に変換して返します。

シグネチャまたは形状

char *format_operator_qualified(Oid operator_oid)

入力と出力

  • **入力**: operator_oid (Oid) - 変換対象となる演算子のOID。
  • **出力**: char * - 演算子を識別するスキーマ修飾された文字列。

前提条件と不変条件

  • 内部で format_operator_extended を呼び出しており、その動作に依存します。
  • 呼び出し元は、返された文字列のメモリ管理（解放）を行う必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った operator_oid を、FORMAT_OPERATOR_FORCE_QUALIFY フラグと共に format_operator_extended 関数へ渡します。 2. format_operator_extended の処理結果（スキーマ修飾が強制された演算子名）をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/regproc.h にて外部インターフェースとして宣言されています。
  • 具体的な呼び出し元のコード箇所は、提供されたソース情報からは特定できません。
  • **呼び出し先**:
  • format_operator_extended (同ファイル内)

関連データ構造

  • Oid: PostgreSQLのシステムオブジェクト識別子。

失敗モードと境界条件

  • 無効な operator_oid が渡された場合の挙動は、内部で呼び出される format_operator_extended の実装に依存します。

保守担当者向けメモ

  • 本関数は format_operator_extended のラッパーであり、特定のフラグ（FORMAT_OPERATOR_FORCE_QUALIFY）を固定的に適用する役割を持ちます。
  • 演算子のフォーマットロジックを変更する場合は、format_operator_extended を参照してください。
