llvm_assert_in_fatal_section

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_error.cpp:107

目的

現在の実行コンテキストが、LLVMの致命的エラーをPostgreSQLのFATALエラーとして処理する「致命的セクション」内にあることを検証する。

シグネチャまたは形状

void llvm_assert_in_fatal_section(void)

入力と出力

  • 入力: なし
  • 出力: なし（条件を満たさない場合、アサーション失敗によりプロセスが中断される）

前提条件と不変条件

  • 静的変数 fatal_new_handler_depth が 0 より大きいこと。
  • 本関数は、llvm_enter_fatal_on_oom() によってエラーハンドラが差し替えられた後に呼び出される必要がある。

アルゴリズムまたは振る舞い

1. fatal_new_handler_depth の値を確認する。 2. 値が 0 以下であれば、Assert マクロによりプログラムを停止させる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/jit/llvm/llvmjit.c:227
  • src/backend/jit/llvm/llvmjit.c:318
  • src/backend/jit/llvm/llvmjit.c:366
  • 呼び出し先:
  • Assert マクロ（PostgreSQLのデバッグ用アサーション）

関連データ構造

  • fatal_new_handler_depth: 現在のネストレベルを追跡する静的変数。

失敗モードと境界条件

  • fatal_new_handler_depth が 0 の状態で呼び出された場合、アサーション失敗によりプロセスが終了する。これは、JITコンパイルの保護範囲外で本関数が実行されたことを示す。

保守担当者向けメモ

  • 本関数はデバッグビルドでのみ有効なチェックを行う。リリースビルドでは最適化により削除される可能性があるため、実行時の厳密な制御フローには依存させないこと。
