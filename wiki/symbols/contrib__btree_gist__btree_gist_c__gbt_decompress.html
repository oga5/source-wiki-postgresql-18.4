gbt_decompress

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_gist.c:55

目的

GiST 索引の圧縮解除関数として機能するスタブです。本モジュールではデータ圧縮を行わないため、入力をそのまま透過的に返す役割を担います。

シグネチャまたは形状

Datum gbt_decompress(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS マクロを介して受け取る GiST 圧縮データ。
  • 出力: PG_GETARG_POINTER(0) で取得したポインタをそのまま Datum として返します。

前提条件と不変条件

  • 本関数は GiST 索引の仕様上要求されるインターフェースとして実装されています。
  • 圧縮解除の処理は行わず、入力されたポインタをそのまま返す不変条件を持ちます。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を呼び出し、第1引数として渡されたポインタを取得します。 2. PG_RETURN_POINTER を使用して、取得したポインタをそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • 呼び出し元: GiST 索引スキャン実行時に PostgreSQL の GiST フレームワークから呼び出されます（直接的な呼び出し元コードの詳細はソース抜粋からは不明）。
  • 呼び出し先: なし。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約で使用される引数セット。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。入力ポインタが有効である限り、常に成功します。

保守担当者向けメモ

  • 本関数はスタブ実装です。将来的に圧縮アルゴリズムを導入する場合を除き、ロジックを変更する必要はありません。
  • PG_FUNCTION_INFO_V1(gbt_decompress) により、PostgreSQL の関数マネージャに登録されています。
