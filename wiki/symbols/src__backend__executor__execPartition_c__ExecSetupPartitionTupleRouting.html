ExecSetupPartitionTupleRouting

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execPartition.c:217

目的

パーティション化されたテーブルに対するデータ操作（INSERT/UPDATE/MERGE）において、タプルを適切なパーティションへ振り分けるためのルーティング環境を初期化します。

シグネチャまたは形状

PartitionTupleRouting *ExecSetupPartitionTupleRouting(EState *estate, Relation rel)

入力と出力

  • **入力**:
  • estate: 実行状態を保持する EState 構造体。
  • rel: ルーティングの起点となるパーティション化されたテーブルの Relation。
  • **出力**:
  • 初期化された PartitionTupleRouting 構造体へのポインタ。

前提条件と不変条件

  • PartitionTupleRouting 構造体の各パーティション用 ResultRelInfo は、この関数内では生成されません。これらはルーティング時に必要となった時点でオンデマンドで生成されます。
  • palloc0 を使用してメモリを確保し、構造体のメンバはゼロ初期化されます。

アルゴリズムまたは振る舞い

1. PartitionTupleRouting 構造体をメモリ上に確保し、ルートテーブル情報と現在のメモリコンテキストを保持します。 2. ExecInitPartitionDispatchInfo を呼び出し、対象テーブルのパーティション階層を探索するための PartitionDispatch オブジェクトを初期化します。この際、親テーブルは存在しないため NULL を渡します。 3. 初期化済みの PartitionTupleRouting 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c
  • src/backend/executor/nodeModifyTable.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • ExecInitPartitionDispatchInfo (同ファイル内)

関連データ構造

  • PartitionTupleRouting: ルーティングに必要な情報を保持するメイン構造体。
  • PartitionDispatchData: パーティション階層の探索情報を保持。
  • ResultRelInfo: 各パーティションの実行時情報（オンデマンドで生成）。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • 指定された rel がパーティション化されていない場合や、カタログ情報に不整合がある場合の挙動については、本関数のソースコード内には明示的なエラーハンドリングの記述はありません。

保守担当者向けメモ

  • 本関数は、単一タプルの挿入など、ルーティングのオーバーヘッドを最小限に抑える必要があるケースを考慮して設計されています。
  • ResultRelInfo のオンデマンド生成という設計方針を維持するため、初期化ロジックを追加する際は注意してください。
