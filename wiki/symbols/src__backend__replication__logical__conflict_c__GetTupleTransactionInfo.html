GetTupleTransactionInfo

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/conflict.c:61

目的

指定されたタプルスロットからトランザクションID（xmin）を抽出し、設定が有効な場合に限り、そのトランザクションのコミットタイムスタンプおよびレプリケーションオリジン情報を取得します。

シグネチャまたは形状

bool GetTupleTransactionInfo(TupleTableSlot *localslot, TransactionId *xmin,
                             RepOriginId *localorigin, TimestampTz *localts)

入力と出力

  • **入力**:
  • localslot: 情報を取得する対象のタプルスロット。
  • xmin: トランザクションIDを格納するポインタ。
  • localorigin: レプリケーションオリジンIDを格納するポインタ。
  • localts: コミットタイムスタンプを格納するポインタ。
  • **出力**:
  • 戻り値: 成功した場合（または情報が取得できた場合）に true、それ以外は false。
  • 引数経由: xmin、localorigin、localts に各値が書き込まれる。

前提条件と不変条件

  • localslot は有効なタプルスロットである必要があります。
  • xmin の取得において、システム属性 MinTransactionIdAttributeNumber が存在しない（isnull）ことは許容されません（Assert により検証）。

アルゴリズムまたは振る舞い

1. slot_getsysattr を使用して、タプルから xmin（MinTransactionIdAttributeNumber）を取得します。 2. グローバル変数 track_commit_timestamp を確認します。

  • 無効な場合: localorigin に InvalidRepOriginId、localts に 0 を設定し、false を返します。

    • 3. 有効な場合: TransactionIdGetCommitTsData を呼び出し、その戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execReplication.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • slot_getsysattr
  • DatumGetTransactionId
  • TransactionIdGetCommitTsData

関連データ構造

  • TupleTableSlot: タプルデータへのアクセスに使用。
  • TransactionId: トランザクション識別子。
  • RepOriginId: レプリケーションオリジン識別子。
  • TimestampTz: コミット時刻。

失敗モードと境界条件

  • track_commit_timestamp が無効な場合、タイムスタンプとオリジン情報は取得できず、関数は false を返します。
  • TransactionIdGetCommitTsData が失敗した場合、その結果が関数の戻り値となります。

保守担当者向けメモ

  • 本関数は track_commit_timestamp 設定に強く依存しています。設定が無効な環境では、呼び出し側でタイムスタンプ情報が欠落することを想定したロジックが必要です。
