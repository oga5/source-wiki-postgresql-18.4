preparePresortedCols

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:163

目的

インクリメンタルソート実行時に、既にソート済みであると見なされるキー（接頭辞キー）の比較関数を事前にキャッシュし、実行時の比較処理を効率化する。

シグネチャまたは形状

static void preparePresortedCols(IncrementalSortState *node)

入力と出力

  • **入力**: IncrementalSortState *node （実行状態を保持するノード構造体）
  • **出力**: なし（副作用として node->presorted_keys に比較情報が格納される）

前提条件と不変条件

  • node->ss.ps.plan が IncrementalSort 型のプランノードとしてキャスト可能であること。
  • plannode->nPresortedCols が定義されており、対応するソート演算子が存在すること。

アルゴリズムまたは振る舞い

1. plannode->nPresortedCols 分の PresortedKeyData 配列をメモリ確保する。 2. 各ソートキーについて、対応する等価演算子（equality operator）を get_equality_op_for_ordering_op で取得する。 3. 取得した演算子から get_opcode で関数 OID を特定する。 4. fmgr_info_cxt を使用して、比較関数の情報を key->flinfo にキャッシュする。 5. InitFunctionCallInfoData を用いて、関数呼び出しに必要な fcinfo を初期化し、引数の isnull フラグを false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeIncrementalSort.c 内の初期化プロセス（ExecInitIncrementalSort 等の文脈から呼び出される）。
  • **呼び出し先**: get_equality_op_for_ordering_op, get_opcode, fmgr_info_cxt, InitFunctionCallInfoData 等の PostgreSQL 内部ユーティリティ関数。

関連データ構造

  • IncrementalSortState: 実行時の状態を保持する構造体。
  • PresortedKeyData: 各ソートキーの比較関数情報（flinfo, fcinfo）を保持する構造体。
  • IncrementalSort: プランノード構造体。

失敗モードと境界条件

  • 等価演算子が見つからない場合、または関数 OID が無効な場合は elog(ERROR) を発生させ処理を中断する。

保守担当者向けメモ

  • 本関数は CurrentMemoryContext を使用して fcinfo を割り当てているため、メモリコンテキストの生存期間に注意すること。
  • fcinfo の引数は isnull = false で固定されているため、NULL 値を含む可能性のあるデータに対しては別途考慮が必要である。
