PLy_subtransaction_enter

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_subxactobject.c:93

目的

Pythonの with 文（コンテキストマネージャ）によるサブトランザクションの開始処理を実行し、PostgreSQL内部のサブトランザクション状態を初期化・登録します。

シグネチャまたは形状

static PyObject *PLy_subtransaction_enter(PyObject *self, PyObject *unused);

入力と出力

  • **入力**:
  • self: PLySubtransactionObject 型のPythonオブジェクト。
  • unused: 使用されない引数。
  • **出力**: 成功時は self（参照カウントをインクリメント）、失敗時は NULL を返します。

前提条件と不変条件

  • self は PLySubtransactionObject 型である必要があります。
  • サブトランザクションが既に開始（started）または終了（exited）している場合、エラーとなります。
  • TopTransactionContext 上でメモリ確保が行われます。

アルゴリズムまたは振る舞い

1. started および exited フラグを確認し、不正な状態であれば PyExc_ValueError を送出します。 2. started フラグを true に設定します。 3. 現在のメモリコンテキストとリソースオーナーを保存するための PLySubtransactionData を TopTransactionContext に確保します。 4. BeginInternalSubTransaction(NULL) を呼び出し、PostgreSQL内部でサブトランザクションを開始します。 5. explicit_subtransactions グローバルリストの先頭に、保存したコンテキスト情報を追加します。 6. 呼び出し元のメモリコンテキストを復元します。 7. self の参照カウントを増やして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Pythonの plpy.subtransaction オブジェクトの __enter__ メソッドとして、Pythonインタープリタから呼び出されます。
  • **呼び出し先**:
  • BeginInternalSubTransaction (PostgreSQL内部関数)
  • MemoryContextAlloc / MemoryContextSwitchTo (PostgreSQLメモリ管理)
  • PLy_exception_set (エラーハンドリング)
  • lcons (リスト操作)

関連データ構造

  • PLySubtransactionObject: Python側から見えるサブトランザクションオブジェクト。
  • PLySubtransactionData: サブトランザクションのコンテキスト情報を保持する構造体。
  • explicit_subtransactions: アクティブなサブトランザクションを管理するグローバルリスト。

失敗モードと境界条件

  • **ValueError**: 既に開始済み、または終了済みのサブトランザクションに対して __enter__ が呼ばれた場合に発生します。
  • **メモリ不足**: MemoryContextAlloc が失敗した場合、PostgreSQLの標準的なエラー処理により中断されます。

保守担当者向けメモ

  • 本関数は TopTransactionContext を使用してデータを保持するため、サブトランザクションの生存期間が現在のトランザクション全体に依存している点に留意してください。
  • explicit_subtransactions リストの操作は、トランザクションのネスト順序を維持するために重要です。
