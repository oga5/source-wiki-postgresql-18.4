ProcessXLogDataMsg

定義

  • **種別**: 関数
  • **定義箇所**: src/bin/pg_basebackup/receivelog.c:1046

目的

ストリーミングレプリケーションプロトコルを通じて受信した XLogData メッセージ（WALデータ）を解析し、ローカルのWALファイルへ書き出す。

シグネチャまたは形状

static bool ProcessXLogDataMsg(PGconn *conn, StreamCtl *stream, char *copybuf, int len, XLogRecPtr *blockpos)

入力と出力

  • **入力**:
  • conn: サーバーとの接続ハンドル。
  • stream: ストリーミング設定を保持する StreamCtl 構造体。
  • copybuf: 受信したメッセージバッファ。
  • len: メッセージの長さ。
  • **出力**:
  • blockpos: 書き込み開始位置（LSN）を格納するポインタ。
  • **戻り値**: 処理が正常に完了した場合は true、エラー発生時は false。

前提条件と不変条件

  • still_sending が偽の場合、メッセージは無視され即座に true を返す。
  • メッセージヘッダー（17バイト）が最低限存在すること。
  • 受信したWALデータのオフセットが、現在開いている walfile の位置と一致すること（またはファイルが未オープンであること）。

アルゴリズムまたは振る舞い

1. **ヘッダー解析**: メッセージからWAL開始位置（dataStart）を抽出。 2. **整合性チェック**: 現在のWALオフセットと受信データのオフセットを比較し、不一致があればエラーを報告。 3. **書き込みループ**:

  • ファイルが未オープンであれば open_walfile を呼び出す。
  • WALセグメント境界を跨ぐ場合は、セグメント末尾までを書き込む。
  • stream->walmethod->ops->write を介してデータをディスクへ書き出す。
  • 書き込み成功後、位置情報を更新。

    • 4. **セグメント終了処理**: セグメント末尾に到達した場合、close_walfile を呼び出す。

  • stream_stop コールバックが真を返せば、サーバーへ COPY 終了を通知し、still_sending を偽に設定して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: HandleCopyStream（src/bin/pg_basebackup/receivelog.c:834）。
  • **呼び出し先**:
  • open_walfile / close_walfile
  • stream->walmethod->ops->write
  • stream->stream_stop
  • PQputCopyEnd / PQflush

関連データ構造

  • StreamCtl: ストリーミングの動作制御。
  • walfile: 現在書き込み中のファイルハンドルを保持する静的変数。
  • WalWriteMethod: 書き込み処理の抽象化レイヤー。

失敗モードと境界条件

  • **ヘッダー不足**: len がヘッダー長（17バイト）未満の場合。
  • **オフセット不整合**: 受信データが期待する位置と異なる場合。
  • **書き込み失敗**: walmethod->ops->write が要求バイト数と異なる値を返した場合。
  • **ファイル操作失敗**: open_walfile または close_walfile が失敗した場合。
  • **通信エラー**: PQputCopyEnd 等の通信関数が失敗した場合。

保守担当者向けメモ

  • walmethod->ops->write は単なるファイル書き込みではなく、圧縮やtar変換を含む可能性があるため、直接的なファイルシステム操作と混同しないこと。
  • still_sending はモジュール内の静的変数であり、本関数内での状態遷移に依存している。
