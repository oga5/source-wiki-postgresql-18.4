SH_NEXT

定義

  • 種別: function
  • 定義箇所: src/include/lib/simplehash.h:363

目的

ハッシュテーブル内のバケットインデックスを、オープンアドレス法に従って次の位置へ循環的に進める。

シグネチャまたは形状

static inline uint32 SH_NEXT(SH_TYPE * tb, uint32 curelem, uint32 startelem)

入力と出力

  • **入力**:
  • tb: ハッシュテーブル構造体へのポインタ。
  • curelem: 現在のバケットインデックス。
  • startelem: 探索を開始した元のバケットインデックス。
  • **出力**:
  • 次のバケットインデックス（uint32）。

前提条件と不変条件

  • tb->sizemask はテーブルサイズのべき乗から算出されたマスク値であること。
  • curelem は常に startelem とは異なる位置へ更新される必要がある（Assert により検証）。

アルゴリズムまたは振る舞い

1. 現在のインデックス curelem に 1 を加算し、tb->sizemask とのビット論理積をとることで、テーブルサイズ内での循環的なインクリメントを行う。 2. 更新後の curelem が startelem と一致した場合、アサーションエラーを発生させる。 3. 更新されたインデックスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/lib/simplehash.h 内のハッシュテーブル操作関数（挿入、検索、削除、反復処理など）から、バケットの線形探索を行う際に呼び出される。
  • **呼び出し先**: 直接的な関数呼び出し先はない（インライン関数）。

関連データ構造

  • SH_TYPE: 生成されるハッシュテーブルの構造体。sizemask メンバを介してテーブルサイズを参照する。

失敗モードと境界条件

  • **境界条件**: テーブルの末尾に達した場合、sizemask によるマスク処理によりインデックス 0 へラップアラウンドする。
  • **失敗モード**: 探索が一周して startelem に戻った場合、Assert によりプログラムが停止する（デバッグビルド時）。
