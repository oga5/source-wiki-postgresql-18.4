compareSyn

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_synonym.c:86

目的

Syn 構造体の配列を、入力単語（in フィールド）に基づいて辞書順に比較する。主に qsort によるソートおよび bsearch による検索の比較関数として使用される。

シグネチャまたは形状

static int compareSyn(const void *a, const void *b)

入力と出力

  • **入力**:
  • a, b: 比較対象となる Syn 構造体へのポインタ（const void * 型）。
  • **出力**:
  • strcmp の結果を返す。a の入力単語が b より小さい場合は負、等しい場合は 0、大きい場合は正の整数。

前提条件と不変条件

  • 引数 a および b は、内部的に Syn 構造体へのポインタとしてキャスト可能である必要がある。
  • Syn 構造体の in フィールドは、strcmp で比較可能なヌル終端文字列であること。

アルゴリズムまたは振る舞い

1. 引数 a と b を const Syn * にキャストする。 2. それぞれの in フィールド（入力単語）を strcmp に渡し、その結果をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • dsynonym_init: 辞書読み込み後の配列ソート時に qsort の引数として呼び出される。
  • dsynonym_lexize: 単語置換時のバイナリ検索時に bsearch の引数として呼び出される。
  • **呼び出し先**:
  • strcmp（標準Cライブラリ）

関連データ構造

  • Syn: 同義語ペア（入力語 in と出力語 out）を保持する構造体。

失敗モードと境界条件

  • a または b が NULL の場合、strcmp 内部でセグメンテーション違反が発生する可能性がある。
  • in フィールドがヌル終端されていない場合、strcmp はメモリ境界を超えて読み取る可能性がある。

保守担当者向けメモ

  • 本関数は qsort および bsearch の仕様に従うため、シグネチャの変更は不可。
  • 比較ロジックを大文字小文字非依存にする必要がある場合は、strcmp を strcasecmp 等へ変更することを検討すること（ただし、現在の実装は辞書構築時の正規化に依存している）。
