RenameEnumLabel

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_enum.c:606

目的

既存の列挙型（ENUM）のラベル名を変更します。

シグネチャまたは形状

void RenameEnumLabel(Oid enumTypeOid, const char *oldVal, const char *newVal)

入力と出力

  • **入力**:
  • enumTypeOid: 対象となる列挙型の OID。
  • oldVal: 変更前のラベル名。
  • newVal: 変更後の新しいラベル名。
  • **出力**: なし（成功時はシステムカタログ pg_enum を更新）。

前提条件と不変条件

  • newVal の長さは NAMEDATALEN - 1 バイト以下である必要があります。
  • oldVal は対象の列挙型に存在するラベルである必要があります。
  • newVal は対象の列挙型において既に存在してはなりません。

アルゴリズムまたは振る舞い

1. 新しいラベル名の長さを検証し、超過している場合はエラーを報告します。 2. LockDatabaseObject を使用して対象の列挙型を排他ロックします（コミットまで保持）。 3. pg_enum リレーションを RowExclusiveLock で開きます。 4. SearchSysCacheList1 を用いて、対象の列挙型に属する全メンバーを取得します。 5. 取得したリストを走査し、oldVal に一致するタプルを特定し、newVal が既に存在しないか確認します。 6. 一致するタプルが見つからない場合、または newVal が重複する場合はエラーを報告します。 7. 特定したタプルの書き込み可能なコピーを作成し、namestrcpy でラベルを更新します。 8. CatalogTupleUpdate を呼び出してシステムカタログを更新し、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/typecmds.c:1328 (DDL コマンド実行時)
  • **呼び出し先**:
  • LockDatabaseObject, table_open, SearchSysCacheList1, heap_copytuple, ReleaseCatCacheList, namestrcpy, CatalogTupleUpdate, heap_freetuple, table_close

関連データ構造

  • pg_enum: 列挙型の定義を保持するシステムカタログ。
  • CatCList: SearchSysCacheList1 が返すシステムカタログのリスト構造。
  • Form_pg_enum: pg_enum カタログのタプル構造体。

失敗モードと境界条件

  • **ERRCODE_INVALID_NAME**: newVal が長すぎる場合。
  • **ERRCODE_INVALID_PARAMETER_VALUE**: oldVal が存在しない場合。
  • **ERRCODE_DUPLICATE_OBJECT**: newVal が既に存在する場合。

保守担当者向けメモ

  • 本関数は pg_enum のインデックス整合性を維持するため、CatalogTupleUpdate を使用しています。
  • LockDatabaseObject は、同時実行される他のバックエンドによる変更を防ぐために使用されています。
  • newVal の重複チェックは、システムカタログのユニークインデックスでも捕捉されますが、ユーザーへの親切なエラーメッセージを提供するために明示的に行われています。
