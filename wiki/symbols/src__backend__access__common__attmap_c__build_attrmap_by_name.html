build_attrmap_by_name

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/attmap.c:174

目的

2つの TupleDesc 間で、列名に基づいて属性番号の対応表（AttrMap）を生成します。主にパーティションテーブルや継承関係にあるテーブル間で、物理的な列順序が異なる場合のマッピングに使用されます。

シグネチャまたは形状

AttrMap *build_attrmap_by_name(TupleDesc indesc, TupleDesc outdesc, bool missing_ok)

入力と出力

  • **入力**:
  • indesc: 入力側のタプル記述子。
  • outdesc: 出力側のタプル記述子。
  • missing_ok: true の場合、出力側の列が入力側に存在しなくてもエラーにしません。
  • **出力**:
  • 生成された AttrMap 構造体。attnums 配列には、出力側の各列に対応する入力側の列番号（1始まり）が格納されます。削除済み列や対応なしの場合は 0 となります。

前提条件と不変条件

  • outdesc の各列について、対応する名前の列が indesc に存在する必要があります（missing_ok が false の場合）。
  • 対応する列が見つかった場合、そのデータ型（atttypid）および型修飾子（atttypmod）が一致している必要があります。一致しない場合はエラーを報告します。

アルゴリズムまたは振る舞い

1. outdesc の列数分だけ AttrMap を確保します。 2. outdesc の各列を走査し、削除済み列（attisdropped）をスキップします。 3. 各列名に対し、indesc 内を検索します。パーティションテーブルでの利用を想定し、前回の検索位置から順次探索する最適化が行われています。 4. 一致する名前の列が見つかったら型チェックを行い、attrMap->attnums に入力側の列番号を記録します。 5. 全ての列の処理終了後、生成した AttrMap を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c
  • src/backend/catalog/partition.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/tablecmds.c

関連データ構造

  • AttrMap: 属性マッピングを保持する構造体。
  • TupleDesc: タプルの構造を定義する記述子。
  • Form_pg_attribute: 個々の属性の定義情報。

失敗モードと境界条件

  • **型不一致**: 名前が一致する列の型または型修飾子が異なる場合、ERRCODE_DATATYPE_MISMATCH でエラー終了します。
  • **列欠落**: missing_ok が false で、かつ outdesc の列が indesc に存在しない場合、ERRCODE_DATATYPE_MISMATCH でエラー終了します。

保守担当者向けメモ

  • 検索ループ内の nextindesc は、パーティションテーブルの列順序が一致しているケースを高速化するためのヒントです。修正時はこの最適化ロジックを維持してください。
  • attnums 配列のインデックスは出力側の列位置に対応しています。値が 0 の場合は、その列が削除済みであることを示します。
