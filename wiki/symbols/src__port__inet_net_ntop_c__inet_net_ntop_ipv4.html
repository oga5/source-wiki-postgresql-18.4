inet_net_ntop_ipv4

定義

  • 種別: function
  • 定義箇所: src/port/inet_net_ntop.c:113

目的

IPv4 アドレスのバイナリ形式を、CIDR 表記を含む人間が読み取り可能な文字列形式へ変換します。

シグネチャまたは形状

char *inet_net_ntop_ipv4(const u_char *src, int bits, char *dst, size_t size)

入力と出力

  • src: 変換対象の IPv4 アドレス（4オクテットのバイナリデータ）。
  • bits: CIDR マスク長（0〜32）。
  • dst: 変換結果を格納するバッファ。
  • size: dst バッファのサイズ。
  • 戻り値: 成功時は dst へのポインタ、失敗時は NULL を返します。

前提条件と不変条件

  • bits は 0 から 32 の範囲内である必要があります。
  • 変換後の文字列（CIDR 表記を含む）が size に収まる必要があります。
  • 4オクテットすべてが常にフォーマットされます。

アルゴリズムまたは振る舞い

1. bits が 0 未満または 32 を超える場合、errno に EINVAL を設定し NULL を返します。 2. 4オクテットの各値を . で区切って文字列化します。 3. bits が 32 以外の場合、/ に続けて bits の値を文字列の末尾に付与します。 4. バッファサイズが不足した場合は errno に EMSGSIZE を設定し NULL を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/inet_net_ntop.c 内の pg_inet_net_ntop から呼び出されることが示唆されています。
  • 呼び出し先: 標準ライブラリの SPRINTF マクロ（sprintf のラッパー）を使用します。

関連データ構造

  • u_char: 入力アドレスの型として使用。

失敗モードと境界条件

  • bits が範囲外: EINVAL を返し NULL を返却。
  • バッファ不足: EMSGSIZE を返し NULL を返却。
  • bits == 32: マスク長（/32）は出力されません。

保守担当者向けメモ

  • 本関数は ISC ライセンスに基づくコードを含みます。
  • SPRINTF マクロの戻り値の扱いに依存しているため、移植先環境での sprintf の挙動に注意してください。
  • ゼロ圧縮などの IPv6 特有のロジックは含まれていません。
