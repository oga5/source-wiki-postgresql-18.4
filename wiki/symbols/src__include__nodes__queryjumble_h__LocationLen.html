LocationLen

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/queryjumble.h:22

目的

クエリテキスト内の定数リテラルや特定のノード位置を追跡し、クエリの正規化やフィンガープリント（Query ID）生成時に定数をマスクするために使用されます。

シグネチャまたは形状

typedef struct LocationLen
{
    int         location;       /* クエリテキスト内の開始オフセット */
    int         length;         /* バイト単位の長さ、または無視する場合は -1 */
    bool        squashed;       /* 圧縮されたリストを表すか */
    bool        extern_param;   /* PARAM_EXTERN パラメータか */
} LocationLen;

入力と出力

本構造体はデータ保持用であり、関数ではありません。JumbleState 構造体の clocations 配列としてメモリ上に確保され、クエリ走査プロセスを通じて定数の位置情報が記録されます。

前提条件と不変条件

  • length が -1 の場合、そのエントリは無視されるものとして扱われます。
  • location はクエリテキスト内の有効なオフセットを指す必要があります。

アルゴリズムまたは振る舞い

クエリツリーの走査中に定数が見つかると、その位置と長さが本構造体に格納されます。pg_stat_statements 等の拡張機能は、この情報を利用してクエリ文字列から定数部分を抽出し、正規化されたクエリを生成します。また、comp_location 関数（pg_stat_statements.c 内）により、位置情報に基づいたソートが行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/nodes/queryjumblefuncs.c: JumbleState の初期化および定数記録時に使用。
  • contrib/pg_stat_statements/pg_stat_statements.c: クエリの正規化およびソート処理に使用。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • JumbleState: クエリのハッシュ化プロセス中に LocationLen の配列（clocations）を保持する作業用構造体。

失敗モードと境界条件

  • length に -1 を設定することで、特定の定数位置を処理対象から除外可能です。
  • メモリ割り当て失敗時の挙動については、palloc を使用しているため、PostgreSQL のメモリ管理ポリシー（エラー発生時のアボート）に従います。

保守担当者向けメモ

  • squashed フラグは、リスト構造が圧縮された際のパラメータ番号の再割り当てに関連しています。
  • extern_param フラグは、外部パラメータの識別に使用されます。
  • pg_stat_statements でのソート処理（comp_location）と整合性を保つため、構造体のメンバ順序や型定義を変更する際は注意してください。
