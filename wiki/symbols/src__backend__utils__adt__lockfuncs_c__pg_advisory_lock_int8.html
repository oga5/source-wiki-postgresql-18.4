pg_advisory_lock_int8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:625

目的

ユーザーが指定した64ビット整数値をキーとして、セッションレベルのアドバイザリーロック（排他モード）を取得します。

シグネチャまたは形状

Datum pg_advisory_lock_int8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る int64 型のキー値。
  • **出力**: なし（void）。ロックが取得されるまで待機します。

前提条件と不変条件

  • ロックキーは MyDatabaseId と組み合わされて管理されるため、データベースを跨いだロックは行えません。
  • 取得されるロックは排他（ExclusiveLock）モードです。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT64(0) により、引数からロックキーを取得します。 2. SET_LOCKTAG_INT64 マクロを使用して、キーを LOCKTAG 構造体に変換します。 3. LockAcquire を呼び出し、対象のタグに対して排他ロックを要求します。

  • 第3引数 true により、ロックが既に保持されている場合は待機します。
  • 第4引数 false により、デッドロック検出の対象となります。

    • 4. ロック取得完了後、PG_RETURN_VOID() を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/delay_execution/delay_execution.c:58 からの直接呼び出しが確認されています。
  • **呼び出し先**:
  • LockAcquire（src/backend/storage/lmgr/lock.c 内のロックマネージャ機能）。

関連データ構造

  • LOCKTAG: ロック対象を識別するための構造体。
  • int64: ロックキーとして使用されるデータ型。

失敗モードと境界条件

  • ロックマネージャの制限により、デッドロックが発生した場合はエラーがスローされます。
  • データベース接続が切断された場合、セッションスコープのアドバイザリーロックは自動的に解放されます。

保守担当者向けメモ

  • 本関数はセッションスコープのロックを管理します。トランザクション終了時に自動解放されるロックが必要な場合は、pg_advisory_xact_lock_int8 を使用してください。
