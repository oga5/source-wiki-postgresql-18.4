logicalrep_write_prepare_common

定義

  • src/backend/replication/logical/proto.c:154

目的

論理レプリケーションにおける「準備済みトランザクション（Two-Phase Commit）」に関連するメッセージ（PREPARE または STREAM_PREPARE）の共通データを、ネットワーク転送用バッファへシリアライズする。

シグネチャまたは形状

static void logicalrep_write_prepare_common(StringInfo out, LogicalRepMsgType type, ReorderBufferTXN *txn, XLogRecPtr prepare_lsn)

入力と出力

  • **入力**:
  • out: シリアライズ先となる StringInfo バッファ。
  • type: メッセージタイプ（LOGICAL_REP_MSG_PREPARE 等）。
  • txn: 準備済みトランザクション情報を含む ReorderBufferTXN 構造体。
  • prepare_lsn: 準備処理のログシーケンス番号（LSN）。
  • **出力**: なし（out バッファにバイナリデータが追記される）。

前提条件と不変条件

  • txn->gid が NULL であってはならない。
  • txn は rbtxn_is_prepared(txn) を満たす準備済みトランザクションである必要がある。
  • txn->xid は有効なトランザクションIDである必要がある。

アルゴリズムまたは振る舞い

1. 指定された type をバッファに書き込む。 2. 現在は固定値 0 である flags フィールドを書き込む。 3. 以下の順序でデータをバッファに書き込む:

  • prepare_lsn (int64)
  • txn->end_lsn (int64)
  • txn->xact_time.prepare_time (int64)
  • txn->xid (int32)
  • txn->gid (文字列)

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/proto.c 内の logicalrep_write_prepare
  • src/backend/replication/logical/proto.c 内のストリーミング準備処理（LOGICAL_REP_MSG_STREAM_PREPARE）
  • **呼び出し先**:
  • pq_sendbyte, pq_sendint64, pq_sendint32, pq_sendstring（libpq の通信用関数）

関連データ構造

  • StringInfo: 通信バッファ。
  • ReorderBufferTXN: トランザクションの状態を保持する構造体。
  • LogicalRepMsgType: メッセージ種別を定義する列挙型。

失敗モードと境界条件

  • txn が準備済み状態でない場合、または GID が欠落している場合は Assert により停止する。
  • バッファのメモリ不足時は pq_send* 関数群の内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はプロトコルメッセージの共通部分を扱うため、フィールドの追加や順序変更を行う際は、対応する読み込み関数（logicalrep_read_prepare_common 等）との完全な同期が必要である。
  • 現在 flags は 0 固定だが、将来的な拡張のために予約されている。
