QualCost

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/pathnodes.h:45

目的

クエリプランナにおいて、式（Qual）の評価にかかるコストを見積もるための構造体です。

シグネチャまたは形状

typedef struct QualCost
{
    Cost        startup;        /* one-time cost */
    Cost        per_tuple;      /* per-evaluation cost */
} QualCost;

入力と出力

  • 入力: startup（初期化コスト）、per_tuple（タプルごとの評価コスト）の2つの Cost 型フィールド。
  • 出力: なし（構造体としてコスト情報を保持）。

前提条件と不変条件

  • startup は式の評価を開始する前の一回限りのコストを表します。
  • per_tuple は各タプルに対して式を評価する際にかかるコストを表します。

アルゴリズムまたは振る舞い

プランナがクエリの実行計画を生成する際、特定の条件式やターゲットリストの評価コストを算出するために使用されます。この構造体は、コスト計算の集計結果を保持するコンテナとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • contrib/postgres_fdw/postgres_fdw.c（外部テーブルのコスト見積もり）
  • src/backend/optimizer/path/costsize.c（クエリプランナのコスト計算ロジック）
  • 呼び出し先: 情報なし。

関連データ構造

  • Cost: コスト値を保持する型（通常は double）。
  • RelOptInfo: リレーションの最適化情報。reltarget->cost などで QualCost が利用されます。

失敗モードと境界条件

  • 特になし。

保守担当者向けメモ

  • 本構造体は、プランナのコスト見積もりロジックにおいて頻繁に受け渡しされます。
  • postgres_fdw 等の外部データラッパーでも利用されているため、変更時は外部モジュールへの影響を考慮してください。
