ecpg_free_params

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:1105

目的

SQL 文の実行に関連して動的に確保されたパラメータ関連のメモリを解放し、statement 構造体のパラメータ管理状態をリセットします。

シグネチャまたは形状

void ecpg_free_params(struct statement *stmt, bool print)

入力と出力

  • **入力**:
  • stmt: 解放対象のパラメータを保持する statement 構造体へのポインタ。
  • print: true の場合、解放前に各パラメータの値をログ出力します。
  • **出力**: なし（副作用として stmt のメンバが更新されます）。

前提条件と不変条件

  • stmt は有効な statement 構造体を指している必要があります。
  • stmt->paramvalues、stmt->paramlengths、stmt->paramformats が指すメモリ領域は、ecpg_free で解放可能な形式で確保されている必要があります。

アルゴリズムまたは振る舞い

1. stmt->nparams 回ループし、各パラメータの値を保持する stmt->paramvalues[n] を解放します。

  • print が true の場合、解放前に print_param_value を呼び出し、パラメータ情報をログ出力します。

    • 2. パラメータの配列自体である stmt->paramvalues、stmt->paramlengths、stmt->paramformats をそれぞれ解放します。 3. stmt 構造体の各ポインタメンバを NULL に設定し、stmt->nparams を 0 にリセットして状態を初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/execute.c 内の複数の箇所（エラーハンドリングや実行後のクリーンアップ処理）。
  • **呼び出し先**: print_param_value（条件付き）、ecpg_free。

関連データ構造

  • struct statement: SQL 文の実行状態を管理する構造体。paramvalues、paramlengths、paramformats、nparams を保持します。

失敗モードと境界条件

  • stmt が NULL の場合、未定義の動作を引き起こす可能性があります。
  • nparams が 0 の場合、ループは実行されず、配列ポインタの解放とリセットのみが行われます。

保守担当者向けメモ

  • 本関数は ecpg_free を使用してメモリを解放します。メモリ管理の整合性を保つため、パラメータの確保には対応する ecpglib のメモリ確保関数を使用してください。
  • print 引数は主にデバッグ目的で使用されます。呼び出し元で false が指定されている場合、ログ出力はスキップされます。
