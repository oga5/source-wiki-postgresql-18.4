pg_popcount_masked_neon

定義

  • **種別**: function
  • **定義箇所**: src/port/pg_popcount_aarch64.c:403

目的

AArch64 Neon命令セットを利用し、指定されたマスクでフィルタリングされたバッファ内のビットカウント（popcount）を高速に計算する。

シグネチャまたは形状

uint64 pg_popcount_masked_neon(const char *buf, int bytes, bits8 mask)

入力と出力

  • **入力**:
  • buf: 対象データのバッファ先頭ポインタ。
  • bytes: バッファのバイト長。
  • mask: 各バイトに対して適用するビットマスク。
  • **出力**: マスクされたデータ内のセットビット（1）の総数（uint64）。

前提条件と不変条件

  • AArch64環境でNeon命令セットが利用可能であること。
  • buf は少なくとも bytes 分のメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. **初期化**: 4つの64ビットアキュムレータ（accum1〜accum4）を0で初期化する。 2. **メインループ**: 128ビット（uint8x16_t）単位でデータを読み込み、マスクを適用後、vcntq_u8（各バイトのビット数カウント）と加算命令を用いて並列処理を行う。命令レベルの並列性を高めるため、4レジスタ分を一度に処理する。 3. **端数処理**: メインループで処理しきれなかった残りのデータに対し、2レジスタ単位、8バイト単位、最後に1バイト単位の順で逐次処理を行う。 4. **集計**: 全てのアキュムレータを合計し、最終的なビット数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/pg_popcount_aarch64.c 内の pg_popcount_masked_optimized 関数ポインタへの代入先として参照される。
  • **呼び出し先**:
  • pg_popcount64（8バイト単位の処理時）
  • pg_number_of_ones（バイト単位の処理時）
  • Neon組み込み関数（vdupq_n_u8, vld1q_u8, vandq_u8, vcntq_u8, vpaddlq_u8, vpaddlq_u16, vpadalq_u32, vaddq_u64, vaddvq_u64）

関連データ構造

  • uint8x16_t, uint64x2_t: Neon SIMDレジスタ型。
  • pg_number_of_ones: バイト単位のフォールバック処理で使用されるルックアップテーブル。

失敗モードと境界条件

  • bytes が0以下の場合、ループは実行されず0を返す。
  • bytes がSIMDレジスタ幅の倍数でない場合、残りのデータは逐次処理によって正確にカウントされる。

保守担当者向けメモ

  • 本関数は pg_popcount_masked_optimized を介して動的に呼び出される。
  • ループ内のアキュムレータ数はパイプライン効率を考慮して設計されているため、変更時は命令の依存関係に注意すること。
