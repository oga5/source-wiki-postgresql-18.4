clamp_cardinality_to_long

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/costsize.c:264

目的

Cardinality 型（通常は double）の値を、システムで扱える long 型の範囲内に安全に変換（クランプ）します。

シグネチャまたは形状

long clamp_cardinality_to_long(Cardinality x)

入力と出力

  • **入力**: x (Cardinality) - 変換対象の数値。
  • **出力**: long - 変換後の値。

前提条件と不変条件

  • 入力値が NaN の場合は LONG_MAX を返します。
  • 入力値が 0 以下の場合は 0 を返します。
  • long が 64 ビット環境の場合、double から long への直接キャストによるオーバーフローを避けるため、比較演算を用いて安全性を担保しています。

アルゴリズムまたは振る舞い

1. isnan(x) を判定し、真であれば LONG_MAX を返します。 2. x <= 0 であれば 0 を返します。 3. x が (double) LONG_MAX 未満であるかを確認し、条件を満たす場合は (long) x を返し、それ以外の場合は LONG_MAX を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeSubplan.c:513
  • src/backend/optimizer/plan/createplan.c:2727, 2764, 6742
  • **呼び出し先**:
  • C 標準ライブラリの isnan 関数。

関連データ構造

  • Cardinality: 本関数が入力として受け取る型（double の別名）。

失敗モードと境界条件

  • **NaN**: isnan により LONG_MAX に変換されます。
  • **負数**: 0 にクランプされます。
  • **極大値**: long 型の最大値を超える値は LONG_MAX にクランプされます。

保守担当者向けメモ

  • 本関数は、プランナが算出した行数推定値などを、メモリ割り当てや配列サイズ指定などの long 型が必要なコンテキストへ安全に渡すために使用されます。
  • double から long への変換時に発生しうる精度やオーバーフローの問題を回避する設計となっています。
