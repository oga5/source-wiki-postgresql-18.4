validate_token

定義

  • 種別: function
  • 定義箇所: src/test/modules/oauth_validator/validator.c:121

目的

PostgreSQLのOAuthトークン検証フックとして、提供されたトークンとロールに基づき認証の可否を判定し、結果を ValidatorModuleResult に格納する。

シグネチャまたは形状

static bool validate_token(const ValidatorModuleState *state,
                           const char *token, const char *role,
                           ValidatorModuleResult *res)

入力と出力

  • **入力**:
  • state: モジュールの状態を保持する構造体。private_data を含む。
  • token: 検証対象のOAuthトークン文字列。
  • role: 認証対象のロール名。
  • res: 検証結果を書き込むための出力用構造体。
  • **出力**:
  • 戻り値: 常に true を返す。
  • res->authorized: GUC authorize_tokens の値に基づき設定される。
  • res->authn_id: GUC authn_id が設定されていればその値、未設定であれば role がコピーされる。

前提条件と不変条件

  • state->private_data が定数 PRIVATE_COOKIE と一致している必要がある。一致しない場合、elog(ERROR) により処理が中断される。

アルゴリズムまたは振る舞い

1. private_data の整合性を確認する。 2. トークン、ロール、および MyProcPort->hba から取得した oauth_issuer と oauth_scope をログに出力する。 3. authorize_tokens の値を用いて res->authorized を更新する。 4. authn_id の設定値（または role）を pstrdup を用いて res->authn_id に割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: validator_callbacks 構造体を通じて、PostgreSQLサーバーのOAuth検証フックから呼び出される。
  • **呼び出し先**:
  • elog: ログ出力およびエラー報告。
  • pstrdup: メモリ確保と文字列コピー。

関連データ構造

  • ValidatorModuleState: 検証モジュールの状態管理に使用。
  • ValidatorModuleResult: 検証結果の格納に使用。
  • MyProcPort: HBA設定（oauth_issuer, oauth_scope）の参照に使用。

失敗モードと境界条件

  • state->private_data が PRIVATE_COOKIE と一致しない場合、エラーが発生する。
  • authn_id が未設定の場合、role が認証IDとして使用される。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、実運用環境では推奨されないデバッグログ出力が含まれている。
  • res->authn_id は pstrdup でメモリが割り当てられるため、呼び出し側でのメモリ管理に注意が必要である。
