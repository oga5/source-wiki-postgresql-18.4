AnyArrayType

定義

  • 種別: union
  • 定義箇所: src/include/utils/array.h:177

目的

PostgreSQLにおいて、標準的なフラット配列（ArrayType）と、メモリ上で操作効率を高めた展開済み配列（ExpandedArrayHeader）を透過的に扱うための共用体です。

シグネチャまたは形状

typedef union AnyArrayType
{
    ArrayType    flt;
    ExpandedArrayHeader xpn;
} AnyArrayType;

入力と出力

  • 入力: Datum 型の配列データ。
  • 出力: AnyArrayType へのポインタ。PG_GETARG_ANY_ARRAY_P マクロや DatumGetAnyArrayP 関数を通じて取得されます。

前提条件と不変条件

  • 配列データは Datum として渡される際、適切に detoast されているか、展開済みオブジェクトとして管理されている必要があります。
  • 内部の flt（ArrayType）を使用する場合、データは MAXALIGN 境界に配置されている必要があります。

アルゴリズムまたは振る舞い

  • 本共用体は、配列の物理的なメモリレイアウト（フラットか展開済みか）を隠蔽します。
  • 呼び出し側は、この共用体を通じて、配列の型に関わらず統一的なインターフェースで配列データにアクセス可能です。
  • 展開済み配列の場合、メモリ上の構造体として直接操作され、必要に応じてフラット化されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • src/backend/utils/adt/array_expanded.c: 展開済み配列の生成および変換処理。
  • src/backend/utils/adt/arrayfuncs.c: 配列操作関数（PG_GETARG_ANY_ARRAY_P を使用する各種関数）。
  • **呼び出し先**:
  • 呼び出し先に関する情報は、提供されたソース excerpt には含まれていません。

関連データ構造

  • ArrayType: フラット配列のヘッダー構造体。
  • ExpandedArrayHeader: 展開済み配列のヘッダー構造体。

失敗モードと境界条件

  • dataoffset が 0 の場合、NULLビットマップは存在しません。
  • 配列要素が TOAST されている場合、アクセス前に detoast が必要です。

保守担当者向けメモ

  • 本シンボルは、配列操作の抽象化レイヤーの基盤です。新しい配列操作関数を追加する際は、AnyArrayType を介してフラット配列と展開済み配列の両方をサポートするように設計してください。
  • 構造体のメンバアクセス時には、対象がフラット配列か展開済み配列かを適切に判別する必要があります。
