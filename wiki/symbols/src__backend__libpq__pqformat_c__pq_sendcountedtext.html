pq_sendcountedtext

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:141

目的

文字列をサーバーからクライアントのエンコーディングへ変換し、長さ（4バイト整数）を付与した上で StringInfo バッファに書き込む。

シグネチャまたは形状

void pq_sendcountedtext(StringInfo buf, const char *str, int slen)

入力と出力

  • buf: 書き込み先の StringInfo バッファ。
  • str: 送信する文字列のポインタ。
  • slen: 文字列の長さ（バイト数）。
  • 出力: なし（buf の内容が更新される）。

前提条件と不変条件

  • buf は初期化済みであること。
  • str は少なくとも slen バイトの有効なメモリ領域を指していること。
  • エンコーディング変換が発生した場合、内部で割り当てられたメモリは関数終了前に解放される。

アルゴリズムまたは振る舞い

1. pg_server_to_client を呼び出し、サーバーエンコーディングからクライアントエンコーディングへの変換を試みる。 2. 変換が発生した場合（戻り値が str と異なる場合）：

  • 変換後の文字列長を再計算する。
  • pq_sendint32 で変換後の長さをバッファに書き込む。
  • appendBinaryStringInfoNT で変換後の文字列をバッファに追加する。
  • 変換時に確保されたメモリを pfree で解放する。

    • 3. 変換が発生しなかった場合：

  • 元の slen を pq_sendint32 で書き込む。
  • appendBinaryStringInfoNT で元の文字列をバッファに追加する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠に基づく）：
  • src/backend/access/common/printsimple.c
  • src/backend/access/common/printtup.c
  • src/backend/commands/explain_dr.c
  • src/backend/replication/logical/proto.c
  • src/backend/tcop/fastpath.c
  • 呼び出し先：
  • pg_server_to_client
  • pq_sendint32
  • appendBinaryStringInfoNT
  • pfree

関連データ構造

  • StringInfo: メッセージ構築に使用される動的バッファ構造体。

失敗モードと境界条件

  • pg_server_to_client が内部でエラーを発生させた場合、処理は中断される可能性がある。
  • 文字列が非常に長い場合、StringInfo のバッファ拡張に伴うメモリ不足が発生する可能性がある。

保守担当者向けメモ

  • 本関数はエンコーディング変換を伴うため、変換不要な場合は pq_sendbytes 等の利用を検討すること。
  • appendBinaryStringInfoNT を使用しているため、バッファにはヌル終端が保証されるが、プロトコル上の長さ指定にはヌル文字を含まない値が使用される。
