overexplain_intlist

定義

  • 種別: function
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:740

目的

整数型（int）、OID、または XID を含む PostgreSQL の List 構造体を文字列化し、EXPLAIN 出力の一部として ExplainState に追加する。

シグネチャまたは形状

static void overexplain_intlist(const char *qlabel, List *list, ExplainState *es);

入力と出力

  • qlabel: 出力プロパティのラベル名。
  • list: 処理対象の List（IntList, OidList, XidList のいずれか）。
  • es: EXPLAIN の状態を保持する ExplainState 構造体。
  • 出力: なし（ExplainPropertyText を介して es に書き込む）。

前提条件と不変条件

  • list が NIL の場合、プロパティ値として "none" が出力される。
  • list がサポートされていない型（IntList, OidList, XidList 以外）である場合、アサーションエラーが発生する。

アルゴリズムまたは振る舞い

1. StringInfoData を初期化し、一時的なバッファとして使用する。 2. list の型を判定し、対応するイテレータ（foreach_int, foreach_oid, foreach_xid）を用いて各要素をバッファにスペース区切りで追記する。 3. バッファにデータが格納されている場合、先頭のスペースを除去した文字列を ExplainPropertyText を使用して es に出力する。 4. 最後にバッファを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pg_overexplain/pg_overexplain.c 内の各ハンドラ関数（例: Relation OIDs, Executor Parameter Types, Result RTIs の出力時）。
  • 呼び出し先:
  • initStringInfo, appendStringInfo, appendStringInfoString, pfree（PostgreSQL ユーティリティ関数）。
  • ExplainPropertyText（EXPLAIN 出力用 API）。

関連データ構造

  • List: PostgreSQL の汎用リスト構造。
  • ExplainState: EXPLAIN コマンドの実行状態を管理する構造体。
  • StringInfoData: 文字列構築用の動的バッファ。

失敗モードと境界条件

  • list が空（NIL）の場合は "none" を出力して正常終了する。
  • サポート外のリスト型が渡された場合、Assert(false) によりデバッグビルドでは停止し、ログには "not an integer list" が記録される。

保守担当者向けメモ

  • 本関数は IntList, OidList, XidList のみを対象としている。新しいリスト型を追加する場合は、条件分岐の拡張が必要である。
  • ExplainPropertyText を使用しているため、出力フォーマット（TEXT, JSON 等）は ExplainState の設定に依存する。
