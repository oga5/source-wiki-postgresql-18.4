gistjoinvector

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistutil.c:113

目的

既存の IndexTuple 配列（ベクトル）の末尾に、別の IndexTuple 配列を結合して拡張する。

シグネチャまたは形状

IndexTuple *gistjoinvector(IndexTuple *itvec, int *len, IndexTuple *additvec, int addlen)

入力と出力

  • **入力**:
  • itvec: 拡張対象の IndexTuple 配列へのポインタ。
  • len: 拡張対象配列の現在の要素数（ポインタ経由で更新される）。
  • additvec: 追加する IndexTuple 配列。
  • addlen: 追加する要素数。
  • **出力**:
  • 拡張または再割り当てされた IndexTuple 配列へのポインタを返す。

前提条件と不変条件

  • itvec は repalloc 可能なメモリ領域を指している必要がある。
  • len は itvec の現在の要素数を正しく保持している必要がある。

アルゴリズムまたは振る舞い

1. repalloc を使用して、itvec が保持するメモリ領域を *len + addlen 分の IndexTuple を格納できるサイズに拡張する。 2. memmove を使用して、additvec の内容を拡張後の itvec の末尾（インデックス *len 以降）にコピーする。 3. *len を *len + addlen に更新する。 4. 拡張された配列のポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c:314
  • src/backend/access/gist/gistbuild.c:519
  • **呼び出し先**:
  • repalloc (PostgreSQL メモリ管理)
  • memmove (標準 C ライブラリ)

関連データ構造

  • IndexTuple: GiST インデックスの各エントリを表現するデータ構造。

失敗モードと境界条件

  • メモリ不足時には repalloc がエラーを発生させる。
  • itvec が NULL の場合、repalloc の仕様に従い新規割り当てとして動作する。

保守担当者向けメモ

  • 本関数はメモリの再割り当てを伴うため、呼び出し元は戻り値のポインタを必ず受け取り、古いポインタを破棄して新しいポインタで更新する必要がある。
