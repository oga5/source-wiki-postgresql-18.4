mark_async_capable_plan

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/createplan.c:1139

目的

指定された Plan ノードが非同期実行可能（async-capable）であるかを判定し、可能であれば当該ノードの async_capable フラグを true に設定する。

シグネチャまたは形状

static bool mark_async_capable_plan(Plan *plan, Path *path)

入力と出力

  • **入力**:
  • plan: 変換後の Plan ノード。
  • path: 対応する Path ノード。
  • **出力**:
  • bool: 非同期実行が可能であれば true、そうでなければ false を返す。

前提条件と不変条件

  • plan が Result ノード（ゲート用）を含む場合、非同期実行は不可とみなされる。
  • path のノードタイプに応じて、再帰的または外部インターフェース（FDW）経由で能力を判定する。

アルゴリズムまたは振る舞い

1. **ゲートチェック**: plan が Result ノードである場合、直ちに false を返す。 2. **ノードタイプ別の判定**:

  • T_SubqueryScanPath: trivial_subqueryscan であれば、サブプランに対して再帰的に判定を行う。
  • T_ForeignPath: FdwRoutine の IsForeignPathAsyncCapable コールバックを呼び出し、外部データラッパーの能力を確認する。
  • T_ProjectionPath: サブパスに対して再帰的に判定を行う。

    • 3. **フラグ設定**: 判定が成功した場合、plan->async_capable を true に設定し、true を返す。 4. **デフォルト**: 上記以外、または判定失敗時は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c:1368
  • src/backend/optimizer/plan/setrefs.c:1457（言及あり）
  • **呼び出し先**:
  • trivial_subqueryscan
  • mark_async_capable_plan（再帰呼び出し）
  • fdwroutine->IsForeignPathAsyncCapable

関連データ構造

  • Plan
  • Path
  • SubqueryScanPath
  • ForeignPath
  • ProjectionPath
  • FdwRoutine

失敗モードと境界条件

  • Result ノードが介在する場合、非同期実行は常に拒否される。
  • ForeignPath において IsForeignPathAsyncCapable が定義されていない場合、非同期実行は不可となる。
  • ProjectionPath においてサブパスが非同期非対応であれば、本関数も false を返す。

保守担当者向けメモ

  • 本関数は Path ツリーから Plan ツリーへの変換過程で、非同期実行の可否を伝播させる役割を持つ。
  • ProjectionPath の処理では、create_projection_plan() がサブプランをプルアップすることを前提にサブパスを評価している点に注意すること。
