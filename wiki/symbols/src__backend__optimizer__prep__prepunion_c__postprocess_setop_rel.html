postprocess_setop_rel

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/prep/prepunion.c:1330

目的

集合演算（UNION, INTERSECT, EXCEPT）の結果リレーションに対して、パス生成後の最終的な調整と最適パスの選択を行う。

シグネチャまたは形状

static void postprocess_setop_rel(PlannerInfo *root, RelOptInfo *rel)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報構造体。
  • rel: 集合演算の結果を保持する RelOptInfo 構造体。
  • **出力**: なし（rel 構造体内のパス情報が更新される）。

前提条件と不変条件

  • rel には既に集合演算に関連するパスが追加されている必要がある。
  • 本関数実行後、rel->cheapest_total_path が設定される。

アルゴリズムまたは振る舞い

1. **拡張機能のフック呼び出し**: create_upper_paths_hook が定義されている場合、UPPERREL_SETOP を引数としてフック関数を実行し、外部モジュールによるパス追加の機会を提供する。 2. **最適パスの選択**: set_cheapest(rel) を呼び出し、リレーション内のパスからコストが最小のものを特定し、rel の最適パスとして設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepunion.c 内の各所（plan_set_operations の処理フローにおいて、集合演算ノードのプランニング完了後に呼び出される）。
  • **呼び出し先**:
  • create_upper_paths_hook（存在する場合）
  • set_cheapest

関連データ構造

  • PlannerInfo: プランニング全体のコンテキスト。
  • RelOptInfo: 集合演算の結果リレーションを表現する構造体。
  • UPPERREL_SETOP: 集合演算の最終処理段階を示すリレーション種別。

失敗モードと境界条件

  • create_upper_paths_hook が NULL の場合はスキップされる。
  • rel にパスが一つも追加されていない状態で set_cheapest が呼ばれた場合の挙動は、set_cheapest の実装に依存する。

保守担当者向けメモ

  • 本関数は FDW（Foreign Data Wrapper）によるパス追加を直接はサポートしていないが、フック経由での拡張性は確保されている。
  • 集合演算のプランニング完了直後に必ず呼び出されるため、このリレーションに対する追加の最適化や制約適用が必要な場合は本関数を起点とする。
