gistFindCorrectParent

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:1026

目的

GiSTインデックスの挿入処理において、子ノードへのダウンリンクを保持する正しい親ページを再特定する。ページ分割等により親ノードが変更された場合に、正しい親を追跡・更新するために使用される。

シグネチャまたは形状

static void gistFindCorrectParent(Relation r, GISTInsertStack *child, bool is_build)

入力と出力

  • **入力**:
  • r: 対象のリレーション。
  • child: 現在の挿入スタック（親ノード情報を含む）。
  • is_build: インデックスビルド中かどうかを示すフラグ。
  • **出力**: なし（副作用として child スタック内の親情報を更新する）。

前提条件と不変条件

  • child->parent が有効なスタック構造を指していること。
  • ページ分割が発生した場合、右リンク（rightlink）を辿ることで正しい親ノードに到達可能であること。
  • インデックスビルド中以外は、ページ更新時にLSNが変化していることが前提となる。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: child->downlinkoffnum を使用して、現在の親ページにダウンリンクが依然として存在するか確認する。存在すれば終了。 2. **ページスキャン**: ページが変更されている場合、現在のページから右リンクを順次辿り、child->blkno を指すダウンリンクを探索する。 3. **再帰的探索**: ページチェーンの末尾に到達しても見つからない場合、gistFindPath を呼び出してツリー全体から親を再検索し、スタックを再構築した上で自身を再帰的に呼び出す。 4. **バッファ管理**: 探索中に不要となった親バッファは UnlockReleaseBuffer で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: gist.c 内の複数箇所（挿入処理の再帰的な親更新時）。
  • **呼び出し先**: gistFindPath（親が見つからない場合の再検索）、ReadBuffer, LockBuffer, UnlockReleaseBuffer 等のバッファ管理関数。

関連データ構造

  • GISTInsertStack: 挿入時のツリー降下経路を保持するスタック構造。
  • Relation: PostgreSQLのリレーション情報。

失敗モードと境界条件

  • **ルート分割**: ルートノードが分割された場合、通常の右リンク探索では親が見つからないため、gistFindPath による全域検索へフォールバックする。
  • **無効なオフセット**: child->downlinkoffnum が無効な場合、即座にページスキャンを開始する。

保守担当者向けメモ

  • 本関数は、並行処理によるページ分割と、インデックスビルド時の特殊なWAL挙動の両方を考慮する必要がある。
  • gistFindPath へのフォールバックは非常に稀なケース（ルート分割時など）を想定している。
  • 呼び出し元でバッファのロック状態が管理されているため、再帰呼び出し時のロック順序に注意すること。
