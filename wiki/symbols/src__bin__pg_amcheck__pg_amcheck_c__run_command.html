run_command

定義

  • 種別: function
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:931

目的

指定された並列スロットを使用して、データベースに対して検証用の SQL コマンドを非同期に発行する。

シグネチャまたは形状

static void run_command(ParallelSlot *slot, const char *sql)

入力と出力

  • **入力**:
  • slot: コマンド実行に使用する ParallelSlot 構造体へのポインタ。
  • sql: 実行する SQL クエリ文字列。
  • **出力**: なし（副作用としてデータベースへのクエリ送信を行う）。

前提条件と不変条件

  • slot は有効なデータベース接続を保持している必要がある。
  • opts.echo が真の場合、実行前に SQL が標準出力へエコーされる。

アルゴリズムまたは振る舞い

1. opts.echo が有効であれば、実行する sql を標準出力に表示する。 2. PQsendQuery を呼び出し、指定された slot の接続を通じて SQL を非同期に送信する。 3. PQsendQuery が 0（失敗）を返した場合、pg_log_error を使用してエラーメッセージと実行しようとした SQL をログ出力し、exit(1) でプログラムを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c:779
  • src/bin/pg_amcheck/pg_amcheck.c:795
  • **呼び出し先**:
  • PQsendQuery (libpq)
  • PQdb (libpq)
  • PQerrorMessage (libpq)
  • pg_log_error / pg_log_error_detail (common/logging.h)

関連データ構造

  • ParallelSlot: 並列実行管理のための接続情報を保持する構造体。
  • AmcheckOptions: グローバル変数 opts を通じて参照される実行オプション。

失敗モードと境界条件

  • PQsendQuery が失敗した場合、エラー詳細をログ出力した上でプロセスが強制終了する。

保守担当者向けメモ

  • 本関数は非同期実行を前提としているため、呼び出し側は後続の PQgetResult 等による結果取得およびスロットの解放を適切に行う必要がある。
