r_Step_4

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_english.c:793

目的

Porterステミングアルゴリズムの第4ステップを実行し、単語末尾の特定の接尾辞を削除する。

シグネチャまたは形状

static int r_Step_4(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体（ステミング対象の単語データを含む環境）。
  • 出力: 処理が成功した場合は 1、失敗した場合は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • z->c（現在のカーソル位置）が単語の末尾付近にあること。
  • r_R2 関数により、対象の接尾辞が単語の R2 領域内にあることが検証される。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置に設定する。 2. 単語末尾の文字が特定のビットマスク条件を満たすか確認し、満たさない場合は終了する。 3. find_among_b を使用し、テーブル a_7 に定義された接尾辞を後方から検索する。 4. 接尾辞が見つかった場合、z->bra を設定し、r_R2 を呼び出して対象が R2 領域内にあるか判定する。 5. among_var の値に応じて以下の処理を行う:

  • case 1: slice_del で接尾辞を削除する。
  • case 2: 直前の文字が 's' または 't' であることを確認した上で、slice_del で接尾辞を削除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_english.c:1052（および他言語実装の同名関数）。
  • 呼び出し先: find_among_b, r_R2, slice_del。

関連データ構造

  • struct SN_env: ステミング処理の環境状態を保持する構造体。
  • a_7: ステップ4で使用される接尾辞検索テーブル。

失敗モードと境界条件

  • find_among_b が接尾辞を見つけられない場合は 0 を返す。
  • r_R2 が条件を満たさない場合、その戻り値をそのまま返す。
  • slice_del がメモリ不足等で失敗した場合、負の値を返す。
  • 単語の境界（z->lb）を超えた操作は行われないようガードされている。

保守担当者向けメモ

  • 本関数は自動生成コードであるため、直接編集せず、Snowballのソース定義（.sblファイル）を修正すること。
  • case 2 における 's' または 't' のチェックは、特定の接尾辞（例: "-ies", "-ied" 等の派生）に対する例外的な削除ルールを実装している。
