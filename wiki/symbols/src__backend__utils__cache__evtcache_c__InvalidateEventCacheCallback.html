InvalidateEventCacheCallback

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/evtcache.c:254

目的

システムカタログ pg_event_trigger の更新に伴い、メモリ上のイベントトリガーキャッシュを無効化し、次回の検索時に再構築を強制する。

シグネチャまたは形状

static void InvalidateEventCacheCallback(Datum arg, int cacheid, uint32 hashvalue)

入力と出力

  • **入力**:
  • arg: コールバック登録時に渡されたデータ（未使用）。
  • cacheid: 無効化対象のキャッシュID。
  • hashvalue: 無効化対象のハッシュ値。
  • **出力**: なし（副作用としてキャッシュ状態を更新）。

前提条件と不変条件

  • 本関数はシステムカタログの無効化通知メカニズムから呼び出されることを前提とする。
  • キャッシュ状態 EventTriggerCacheState が ETCS_VALID の場合のみ、メモリの解放処理が行われる。

アルゴリズムまたは振る舞い

1. 現在のキャッシュ状態 EventTriggerCacheState を確認する。 2. 状態が ETCS_VALID である場合:

  • EventTriggerCacheContext をリセットしてメモリを解放する。
  • EventTriggerCache ポインタを NULL に初期化する。

    • 3. キャッシュ状態を ETCS_NEEDS_REBUILD に設定し、次回 EventCacheLookup 呼び出し時に再構築が行われるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**: システムの無効化通知サブシステム（src/backend/utils/cache/inval.c 等の関連機能から登録・起動される）。
  • **呼び出し先**: MemoryContextReset（メモリ解放のため）。

関連データ構造

  • EventTriggerCacheState: キャッシュの有効性を管理する状態変数。
  • EventTriggerCacheContext: キャッシュ保持用のメモリコンテキスト。
  • EventTriggerCache: キャッシュの実体であるハッシュテーブル。

失敗モードと境界条件

  • キャッシュが既に ETCS_NEEDS_REBUILD 状態である場合、メモリ解放処理はスキップされるが、状態は ETCS_NEEDS_REBUILD に維持されるため整合性は保たれる。

保守担当者向けメモ

  • 本関数は再構築中であっても呼び出される可能性がある。その場合、メモリ解放は行われないが、再構築完了後にキャッシュが即座に無効化されるよう ETCS_NEEDS_REBUILD への遷移が保証されている。
