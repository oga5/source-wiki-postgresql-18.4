plpgsql_append_source_text

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_scanner.c:434

目的

スキャナが保持する元のソース文字列から、指定された範囲のテキストを StringInfo バッファへ抽出・コピーする。

シグネチャまたは形状

void plpgsql_append_source_text(StringInfo buf, int startlocation, int endlocation, yyscan_t yyscanner)

入力と出力

  • buf: 出力先となる StringInfo 構造体へのポインタ。
  • startlocation: 抽出を開始するソース内のバイトオフセット。
  • endlocation: 抽出を終了するソース内のバイトオフセット。
  • yyscanner: スキャナの状態を保持する yyscan_t 型のコンテキスト。

前提条件と不変条件

  • startlocation <= endlocation であること（アサーションにより検証）。
  • yyscanner 内の yyextra が初期化されており、scanorig（元のソース文字列）が有効であること。

アルゴリズムまたは振る舞い

1. startlocation から endlocation までの長さを計算する。 2. yyextra->scanorig を基点とし、startlocation 分オフセットした位置から、計算した長さ分のバイナリデータを appendBinaryStringInfo を用いて buf に追加する。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/pl_scanner.c 内のコード（直接の呼び出し箇所あり）。
  • 呼び出し先: appendBinaryStringInfo（PostgreSQL コアの文字列操作関数）。
  • 補足: 外部からの呼び出しについては、src/pl/plpgsql/src/plpgsql.h:1318 に外部宣言が存在する。

関連データ構造

  • StringInfo: PostgreSQL の動的文字列バッファ。
  • yyscan_t: スキャナの内部状態を保持する不透明ポインタ。
  • plpgsql_yy_extra_type: yyscanner 経由でアクセスされる構造体で、scanorig を保持する。

失敗モードと境界条件

  • startlocation または endlocation が scanorig の有効範囲外である場合、メモリ破壊や不正な読み取りが発生する可能性がある。
  • buf が指す StringInfo のメモリ不足時には、appendBinaryStringInfo 内部でエラーが発生する。

保守担当者向けメモ

  • 本関数は、エラー報告やソースコードの再構築時に、字句解析対象の元のテキストを正確に取得するために使用される。
  • scanorig は plpgsql_scanner_init 等で設定されるため、スキャナのライフサイクルと整合している必要がある。
