SyncRequestType

定義

  • 種別: enum
  • 定義箇所: src/include/storage/sync.h:23

目的

PostgreSQLのストレージサブシステムにおいて、ファイルに対する同期（fsync）や削除などの管理操作を識別するために使用されます。

シグネチャまたは形状

typedef enum SyncRequestType
{
	SYNC_REQUEST,
	SYNC_UNLINK_REQUEST,
	SYNC_FORGET_REQUEST,
	SYNC_FILTER_REQUEST,
} SyncRequestType;

入力と出力

  • 本シンボル自体は列挙型であり、関数への引数として渡されることで、実行すべき操作の種類を決定します。

前提条件と不変条件

  • RegisterSyncRequest や RememberSyncRequest などの関数において、操作対象の FileTag と組み合わせて使用されます。

アルゴリズムまたは振る舞い

  • SYNC_REQUEST: 指定されたタグに対する fsync 操作をスケジュールします。
  • SYNC_UNLINK_REQUEST: 指定されたタグに対する unlink 操作をスケジュールします。
  • SYNC_FORGET_REQUEST: 指定されたタグに関連する保留中の操作をすべて破棄します。
  • SYNC_FILTER_REQUEST: マッチ関数に合致する保留中の操作をすべて破棄します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/checkpointer.c (ForwardSyncRequest)
  • src/backend/storage/sync/sync.c (RememberSyncRequest, RegisterSyncRequest)
  • **呼び出し先**:
  • 情報なし（本シンボルは列挙型であり、直接の呼び出し先を持ちません）。

関連データ構造

  • FileTag: 同期対象ファイルを一意に識別するために本型と併用されます。

失敗モードと境界条件

  • 無効な SyncRequestType が渡された場合の挙動については、各ハンドラの実装（sync.c 等）に依存します。

保守担当者向けメモ

  • 新しい同期操作の種類を追加する場合は、本列挙型を拡張し、対応するハンドラロジックを sync.c に実装する必要があります。
