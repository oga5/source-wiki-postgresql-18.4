case_index

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/common/unicode_case_table.h:13524

目的

入力されたUnicodeコードポイント（pg_wchar）を、大文字・小文字変換やケースフォールディングに使用される変換テーブルのインデックス（uint16）へ変換します。

シグネチャまたは形状

static inline uint16 case_index(pg_wchar cp)

入力と出力

  • **入力**: pg_wchar cp（対象のUnicodeコードポイント）
  • **出力**: uint16 型のインデックス値。該当するマッピングが存在しない場合は 0 を返します。

前提条件と不変条件

  • 本関数は src/include/common/unicode_case_table.h 内の静的データ構造と密接に結合しており、Perlスクリプトによって自動生成されたテーブル構造に依存しています。
  • 入力コードポイントの範囲に応じて、case_map 配列の特定のオフセット位置を参照します。

アルゴリズムまたは振る舞い

コードポイントの範囲に基づき、ルックアップテーブルのインデックスを算出する多段の条件分岐を実行します。 1. **高速パス**: cp < 0x0588 の場合、case_map[cp] を直接返します。 2. **範囲分岐**: 0x0588 以上のコードポイントに対しては、入れ子状の if-else 文を用いて、特定のUnicodeブロック範囲（例: 0x10A0〜0x1100、0x24B6〜0x2D2E など）を判定し、各範囲に応じたオフセット計算を行います。 3. **終了条件**: どの範囲にも該当しない場合、デフォルト値として 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode_case.c（case_index(u1) および case_index(ucs) として使用）
  • src/include/common/unicode_case_table.h 内の各テーブル定義コメント（ドキュメントとしての参照）
  • **呼び出し先**: 本関数は計算のみを行うインライン関数であり、他の関数を呼び出しません。

関連データ構造

  • case_map: インデックス算出後に参照される変換テーブル本体。
  • pg_wchar: PostgreSQL内部で使用されるワイド文字型。

失敗モードと境界条件

  • 定義された範囲外のコードポイントが入力された場合、関数は 0 を返します。
  • 範囲判定は厳密に定義されており、テーブル生成スクリプトのロジックと一致している必要があります。
