make_outputdirs

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.c:252

目的

pg_upgrade の実行に必要なログ出力用ディレクトリ構造（ルート、タイムスタンプベースのベースディレクトリ、ダンプ用、ログ用）を作成し、ログファイルの初期化と開始ラベルの書き込みを行う。

シグネチャまたは形状

static void make_outputdirs(char *pgdata)

入力と出力

  • **入力**: pgdata (新クラスタのデータディレクトリパス)
  • **出力**: なし（副作用としてファイルシステム上にディレクトリとログファイルを作成し、グローバル変数 log_opts を更新する）

前提条件と不変条件

  • pgdata は有効なパス文字列であること。
  • MAXPGPATH を超えるパス長はエラーとして処理される。
  • log_opts 構造体が適切に初期化可能であること。

アルゴリズムまたは振る舞い

1. pgdata 配下に BASE_OUTPUTDIR を含むルートディレクトリパスを生成。 2. 現在時刻に基づき、%Y%m%dT%H%M%S.%03d 形式のタイムスタンプ文字列を生成。 3. タイムスタンプを付与したベースディレクトリ、ダンプ用ディレクトリ、ログ用ディレクトリのパスを構築。 4. mkdir を使用して各ディレクトリを作成（ルートディレクトリのみ EEXIST を許容）。 5. INTERNAL_LOG_FILE を追記モードで開き、log_opts.internal に割り当て。 6. output_files 配列に含まれる各ファイルに対し、アップグレード開始時刻を記録したヘッダーを追記する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/pg_upgrade.c 内の main 関数（123行目）。
  • **呼び出し先**: pg_malloc0, snprintf, gettimeofday, strftime, localtime, mkdir, fopen_priv, fprintf, fclose 等の標準ライブラリおよびユーティリティ関数。

関連データ構造

  • log_opts: ログ出力先ディレクトリやファイルハンドルを保持するグローバル構造体。
  • output_files: ログ出力対象となるファイル名のリスト。

失敗モードと境界条件

  • パス長が MAXPGPATH を超える場合、pg_fatal により異常終了する。
  • ディレクトリ作成権限がない場合や、ログファイルのオープンに失敗した場合、pg_fatal により異常終了する。
  • mkdir 失敗時、ルートディレクトリ以外で EEXIST が発生した場合はエラーとして扱われる。

保守担当者向けメモ

  • 本関数は pg_upgrade の初期化フェーズで一度だけ呼び出されることを想定している。
  • WIN32 環境でのファイル排他制御については、fopen_priv の実装（file.c）に依存しているため、ログ出力時の挙動を調査する際はそちらを参照すること。
