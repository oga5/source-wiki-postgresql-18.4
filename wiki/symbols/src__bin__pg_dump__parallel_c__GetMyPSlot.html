GetMyPSlot

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/parallel.c:265

目的

現在実行中のプロセスまたはスレッドに対応する ParallelSlot 構造体を、ParallelState が保持するスロット配列から検索して取得する。

シグネチャまたは形状

static ParallelSlot *GetMyPSlot(ParallelState *pstate)

入力と出力

  • **入力**: ParallelState *pstate（並列実行全体の管理構造体）
  • **出力**: 現在の実行コンテキスト（プロセスIDまたはスレッドID）と一致する ParallelSlot へのポインタ。該当するものがない場合は NULL を返す。

前提条件と不変条件

  • pstate は有効な ParallelState 構造体を指している必要がある。
  • pstate->parallelSlot 配列が初期化されていること。

アルゴリズムまたは振る舞い

1. pstate->numWorkers で指定された回数だけ、parallelSlot 配列をループで走査する。 2. プラットフォームに応じて現在の識別子を取得する：

  • Windows (WIN32): GetCurrentThreadId() を使用。
  • その他: getpid() を使用。

    • 3. 配列内の各スロットが保持する ID と現在の識別子を比較する。 4. 一致したスロットのポインタを返し、ループ終了まで見つからない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/parallel.c 内の各処理（例: ShutdownInformation 関連の処理など）。
  • **呼び出し先**:
  • getpid() (Unix) または GetCurrentThreadId() (Windows) を呼び出す。

関連データ構造

  • ParallelState: 並列処理の全体状態を保持。
  • ParallelSlot: 個別のワーカーのプロセス/スレッド情報および通信パイプを保持。

失敗モードと境界条件

  • 呼び出し元がワーカープロセス/スレッド外である場合、または pstate が正しく初期化されていない場合、NULL が返される。

保守担当者向けメモ

  • 本関数は static であり、parallel.c 内でのみ使用される。
  • Windows 環境ではスレッドベース、Unix 環境ではプロセスベースの識別を行うため、プラットフォーム依存の #ifdef ブロックが維持されている。
