CreateOpFamily

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:242

目的

PostgreSQLのシステムカタログ pg_opfamily に新しい演算子ファミリー（Operator Family）のエントリを作成し、必要な依存関係を構築する。

シグネチャまたは形状

ObjectAddress CreateOpFamily(CreateOpFamilyStmt *stmt, const char *opfname, Oid namespaceoid, Oid amoid)

入力と出力

  • **入力**:
  • stmt: CREATE OPERATOR FAMILY 文の構文解析ツリー。
  • opfname: 作成する演算子ファミリーの名前。
  • namespaceoid: ファミリーが属する名前空間のOID。
  • amoid: 対象となるアクセスメソッド（AM）のOID。
  • **出力**:
  • 作成された演算子ファミリーの ObjectAddress を返す。

前提条件と不変条件

  • pg_opfamily テーブルに対する RowExclusiveLock を保持していること。
  • 指定された名前空間およびアクセスメソッドにおいて、同名の演算子ファミリーが既に存在していないこと。

アルゴリズムまたは振る舞い

1. SearchSysCacheExists3 を使用し、指定されたAMと名前空間で同名のファミリーが存在するか確認する。存在する場合はエラーを報告する。 2. GetNewOidWithIndex を用いて新しいOIDを取得する。 3. pg_opfamily カタログの各属性（OID、AM、名前、名前空間、所有者）を構築し、CatalogTupleInsert で挿入する。 4. 以下の依存関係を pg_depend に記録する：

  • アクセスメソッドへの DEPENDENCY_AUTO 依存。
  • 名前空間への DEPENDENCY_NORMAL 依存。
  • 所有者への依存。
  • 現在の拡張（Extension）への依存。

    • 5. イベントトリガーへの通知およびポスト作成フックの呼び出しを行う。 6. table_close でロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照に基づくと、主に src/backend/commands/opclasscmds.c 内の演算子クラス/ファミリー定義処理から呼び出される。
  • **呼び出し先**: table_open, SearchSysCacheExists3, GetNewOidWithIndex, heap_form_tuple, CatalogTupleInsert, recordDependencyOn, recordDependencyOnOwner, recordDependencyOnCurrentExtension, EventTriggerCollectSimpleCommand, InvokeObjectPostCreateHook, table_close 等のカタログ操作およびユーティリティ関数。

関連データ構造

  • CreateOpFamilyStmt: SQL文の解析結果を保持する構造体。
  • pg_opfamily: 演算子ファミリーを管理するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトを識別するための構造体。

失敗モードと境界条件

  • 同名の演算子ファミリーが既に存在する場合、ERRCODE_DUPLICATE_OBJECT でエラー終了する。
  • カタログ操作中に権限不足やシステムエラーが発生した場合、トランザクションがアボートされる可能性がある。

保守担当者向けメモ

  • 本関数はスーパーユーザー権限を前提とした操作を想定している。
  • 依存関係の追加順序や種類（DEPENDENCY_AUTO vs DEPENDENCY_NORMAL）は、オブジェクトの削除時の連鎖動作に影響するため、変更時には注意が必要である。
