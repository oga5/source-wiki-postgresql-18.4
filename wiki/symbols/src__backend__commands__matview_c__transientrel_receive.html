transientrel_receive

定義

  • 種別: function
  • 定義箇所: src/backend/commands/matview.c:508

目的

マテリアライズド・ビューの更新処理において、クエリ結果のタプルを一時リレーションへ挿入する DestReceiver のコールバック関数です。

シグネチャまたは形状

static bool transientrel_receive(TupleTableSlot *slot, DestReceiver *self);

入力と出力

  • **入力**:
  • slot: 挿入対象のタプルを含む TupleTableSlot。
  • self: DR_transientrel 構造体としてキャストされる DestReceiver。
  • **出力**:
  • 常に true を返します。

前提条件と不変条件

  • self は DR_transientrel 型の構造体である必要があります。
  • 対象となるリレーションは新規作成されたものであり、インデックスが存在しないことが前提です。

アルゴリズムまたは振る舞い

1. self を DR_transientrel 型にキャストし、内部の transientrel（リレーション）を取得します。 2. table_tuple_insert を呼び出し、提供された slot の内容を一時リレーションに挿入します。

  • この際、DR_transientrel 内の output_cid、ti_options、bistate を使用します。

    • 3. 入力 slot の型が対象リレーションの型と一致しない場合でも、table_tuple_insert の仕様により挿入が試行されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/matview.c 内で DestReceiver の receiveSlot フィールドに本関数が割り当てられています。
  • **呼び出し先**:
  • table_tuple_insert (PostgreSQL テーブルアクセス層)

関連データ構造

  • DR_transientrel: DestReceiver を継承した構造体で、一時リレーションの挿入に必要な状態（リレーションポインタ、コマンドID、挿入オプション等）を保持します。
  • TupleTableSlot: 挿入されるタプルデータを保持する構造体。

失敗モードと境界条件

  • table_tuple_insert が失敗した場合、例外が発生し処理が中断される可能性があります。
  • 入力 slot が適切に初期化されていない場合、挿入時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は REFRESH MATERIALIZED VIEW のデータ生成フェーズで使用されます。
  • インデックスが存在しない新規作成リレーションへの挿入を前提としているため、インデックス更新処理は含まれていません。
  • 効率化のため、入力 slot をそのまま table_tuple_insert に渡していますが、これによりタプルごとの型変換コストを回避しています。
