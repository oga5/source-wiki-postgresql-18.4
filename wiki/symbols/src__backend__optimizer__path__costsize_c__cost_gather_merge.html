cost_gather_merge

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/costsize.c:484

目的

GatherMergePath ノードの実行コスト（startup_cost, total_cost）および行数推定値を算出し、対象パス構造体に設定する。

シグネチャまたは形状

void cost_gather_merge(GatherMergePath *path, PlannerInfo *root,
                       RelOptInfo *rel, ParamPathInfo *param_info,
                       int input_disabled_nodes,
                       Cost input_startup_cost, Cost input_total_cost,
                       double *rows)

入力と出力

  • **入力**:
  • path: コストを計算する対象の GatherMergePath 構造体。
  • root, rel, param_info: プランナのコンテキストおよびリレーション情報。
  • input_disabled_nodes, input_startup_cost, input_total_cost: 入力パスのコストおよびノード状態。
  • rows: 行数推定値（指定がない場合は param_info または rel から取得）。
  • **出力**:
  • path 構造体の rows, disabled_nodes, startup_cost, total_cost フィールドを更新。

前提条件と不変条件

  • path->num_workers > 0 であること（アサーションにより保証）。
  • enable_gathermerge グローバル変数の値に基づき、disabled_nodes が適切に設定される。

アルゴリズムまたは振る舞い

1. **行数設定**: 引数 rows があればそれを優先し、なければ param_info または rel->rows を使用して path->path.rows を決定する。 2. **ヒープ管理コスト計算**: ワーカー数 + リーダー（N = num_workers + 1）に基づき、マージ操作に必要な比較コスト（2.0 * cpu_operator_cost * N * log2(N)）を算出。 3. **実行コスト計算**:

  • 各タプルの比較コストおよびヒープ管理コストを加算。
  • 並列セットアップコスト（parallel_setup_cost）を startup_cost に加算。
  • 並列通信コスト（parallel_tuple_cost）に 5% のオーバーヘッドを乗じて run_cost に加算。

    • 4. **結果の反映**: 入力コストと計算したコストを合計し、path に書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/pathnode.c:2135
  • **呼び出し先**:
  • LOG2 マクロ（math.h 関連）
  • cpu_operator_cost, parallel_setup_cost, parallel_tuple_cost, enable_gathermerge（グローバル変数）

関連データ構造

  • GatherMergePath: 本関数がコストを計算する対象のパス構造体。
  • PlannerInfo, RelOptInfo, ParamPathInfo: プランナの統計情報およびコンテキスト保持構造体。

失敗モードと境界条件

  • rows が NULL の場合、param_info または rel からの行数推定に依存する。
  • num_workers が 0 以下の場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は Gather と異なり、全ワーカーからタプルが揃うまで待機する特性を考慮し、通信コストに 5% の加算を行っている。
  • リーダープロセスもワーカーと同等の負荷をかける前提で計算されているが、実際にはリーダーの負荷が低い場合があるため、将来的な調整の余地がある。
