latin4_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/latin_and_mic/latin_and_mic.c:109

目的

ISO-8859-4 (LATIN4) エンコーディングの文字列を、PostgreSQL内部表現である MULE_INTERNAL (MIC) エンコーディングへ変換する。

シグネチャまたは形状

Datum latin4_to_mic(PG_FUNCTION_ARGS)

  • PostgreSQLの関数マネージャ（fmgr）インターフェースに準拠した形式。

入力と出力

  • 入力: PG_FUNCTION_ARGS を介して受け取る以下の引数
  • src: 変換元文字列（CSTRING）
  • dest: 変換先バッファ（CSTRING）
  • len: 変換対象の長さ（INT32）
  • noError: エラー発生時の挙動を制御するフラグ（BOOL）
  • 出力: 変換されたバイト数（PG_RETURN_INT32）

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_LATIN4, PG_MULE_INTERNAL) により、呼び出し時にエンコーディングIDがLATIN4からMICへの変換として妥当であることが検証される。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認する。 3. latin2mic ヘルパー関数を呼び出し、LC_ISO8859_4 を指定して変換を実行する。 4. 変換されたバイト数を戻り値として返す。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの変換フレームワーク（PG_FUNCTION_INFO_V1(latin4_to_mic) により登録）。
  • 呼び出し先: latin2mic（mb/pg_wchar.h で定義された変換ロジック）。

関連データ構造

  • PG_LATIN4: 変換元エンコーディングID。
  • PG_MULE_INTERNAL: 変換先エンコーディングID。
  • LC_ISO8859_4: latin2mic に渡されるロケール識別子。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS による検証失敗時はエラーが発生する。
  • 変換処理自体の失敗モードは latin2mic の実装に依存する。

保守担当者向けメモ

  • 本関数はボイラープレート的なラッパーであり、実際の変換ロジックは src/backend/utils/mb/wchar.c の latin2mic に委譲されている。
  • 新しいLATIN系エンコーディングを追加する際は、本関数のパターンを模倣すること。
