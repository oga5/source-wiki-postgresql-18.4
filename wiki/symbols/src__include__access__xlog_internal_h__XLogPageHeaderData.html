XLogPageHeaderData

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlog_internal.h:36

目的

PostgreSQLのWAL（先行書き込みログ）ファイルの各ページの先頭に配置されるメタデータ構造体です。WALの整合性チェック、タイムラインの追跡、およびページを跨ぐレコードの継続処理を管理します。

シグネチャまたは形状

typedef struct XLogPageHeaderData
{
    uint16      xlp_magic;      /* 整合性チェック用マジックナンバー */
    uint16      xlp_info;       /* フラグビット */
    TimeLineID  xlp_tli;        /* ページ内の最初のレコードのタイムラインID */
    XLogRecPtr  xlp_pageaddr;   /* このページのXLOGアドレス */
    uint32      xlp_rem_len;    /* 前ページから続くレコードの残りバイト数 */
} XLogPageHeaderData;

入力と出力

  • **入力**: WAL書き込み時に、ページ先頭のメタデータとして初期化されます。
  • **出力**: WAL読み込み時、XLogReaderState が本構造体を解析し、xlp_info フラグに基づいてレコードの継続性やロングヘッダーの有無を判定します。

前提条件と不変条件

  • xlp_rem_len は、現在のページにレコード全体が収まらない場合に、前ページから継続するデータの総長を保持します。
  • 継続データは必ずしもアライメントされていません。

アルゴリズムまたは振る舞い

1. **WAL書き込み**: XLogPageHeaderSize マクロ等を用いてページヘッダーを構築し、レコードをチェーン状に構成します。 2. **WAL読み込み**: XLogReaderState がページヘッダーを読み取り、xlp_info を参照して、ロングヘッダー（XLP_LONG_HEADER）の有無や、レコードの継続状態を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、具体的な関数レベルの呼び出し元は記載されていません。
  • **呼び出し先**: 提供されたソース情報には、具体的な関数レベルの呼び出し先は記載されていません。

関連データ構造

  • XLogPageHeader: XLogPageHeaderData へのポインタ型。
  • XLogLongPageHeaderData: 拡張ヘッダー情報（システムID等）を含む構造体。
  • XLogReaderState: WAL読み込み時に本構造体を参照する状態管理構造体。

失敗モードと境界条件

  • xlp_magic による整合性チェックに失敗した場合、WALファイルが破損しているとみなされます。
  • xlp_rem_len が正しく設定されていない場合、ページを跨ぐレコードの再構築が失敗します。

保守担当者向けメモ

  • 本構造体はフロントエンド（pg_receivewal 等）でも利用されるため、バックエンド固有のコードを含めないようにしてください。
  • ページヘッダーのサイズ変更やフィールド追加を行う際は、既存のツールとの互換性に注意が必要です。
