bbsink_gzip

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup_gzip.c:22

目的

PostgreSQLのベースバックアップ処理において、bbsink インターフェースを介してデータをgzip圧縮するための状態保持構造体です。

シグネチャまたは形状

typedef struct bbsink_gzip
{
    bbsink      base;           /* 共通のシンク情報 */
    int         compresslevel;  /* 圧縮レベル */
    z_stream    zstream;        /* zlib圧縮ストリーム状態 */
    size_t      bytes_written;  /* 出力バッファ内の書き込み済みバイト数 */
} bbsink_gzip;

入力と出力

  • **入力**: bbsink フレームワークを通じて渡されるバックアップデータストリーム。
  • **出力**: zlib により圧縮されたデータ。bytes_written を用いて管理されるバッファを経由し、後続の bbsink へ転送されます。

前提条件と不変条件

  • HAVE_LIBZ が定義されている環境でのみ有効です。
  • zstream は deflateInit2 等の zlib 関数によって適切に初期化されている必要があります。
  • base フィールドは bbsink としての共通インターフェースを保持し、パイプラインの一部として機能します。

アルゴリズムまたは振る舞い

  • bbsink フレームワークのレイヤーとして動作し、透過的にデータを圧縮します。
  • bbsink_gzip_archive_contents を通じて入力データを圧縮し、出力バッファが満杯になった時点で後続のシンクへデータを送出します。
  • bbsink_gzip_end_archive 呼び出し時に Z_FINISH を使用して圧縮ストリームを終了し、残存データをフラッシュします。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_gzip_new（構造体の割り当て）、および bbsink インターフェースを介したバックアップ処理フロー。
  • **呼び出し先**: zlib ライブラリ（deflate 等）、および後続の bbsink 実装（bbsink_archive_contents 等）。

関連データ構造

  • bbsink: 本構造体が継承する基底インターフェース。
  • z_stream: zlib が提供する圧縮状態管理構造体。

失敗モードと境界条件

  • zlib の初期化失敗や圧縮処理中のエラーが発生する可能性があります。
  • マニフェストファイルなどの圧縮対象外データは、本構造体による圧縮処理をバイパスし、そのまま後続へ転送されます。

保守担当者向けメモ

  • deflateInit2 の引数には 15 + 16 を指定し、gzip形式のヘッダーを生成するように設定されています。
  • メモリ管理には palloc/pfree を使用し、zlib のインターフェースに適合させています。
