combinebackup_per_wal_range_cb

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/load_manifest.c:292

目的

バックアップマニフェストのパース中に検出されたWAL範囲情報を、manifest_data 構造体が保持するリンクドリストに追加するコールバック関数です。

シグネチャまたは形状

static void combinebackup_per_wal_range_cb(JsonManifestParseContext *context, TimeLineID tli, XLogRecPtr start_lsn, XLogRecPtr end_lsn)

入力と出力

  • **入力**:
  • context: パース状態を保持する JsonManifestParseContext。private_data メンバに manifest_data 構造体へのポインタを保持している必要があります。
  • tli: タイムラインID。
  • start_lsn: WAL範囲の開始LSN。
  • end_lsn: WAL範囲の終了LSN。
  • **出力**: なし（副作用として context->private_data が指す構造体を更新）。

前提条件と不変条件

  • context->private_data は manifest_data 型の有効なポインタであること。
  • manifest_data 構造体の first_wal_range および last_wal_range は、リストの開始と末尾を正しく追跡していること。

アルゴリズムまたは振る舞い

1. palloc を使用して新しい manifest_wal_range 構造体をメモリ上に確保します。 2. 引数として受け取った tli、start_lsn、end_lsn を構造体に設定します。 3. リストの末尾に新しいノードを連結します。

  • リストが空（first_wal_range == NULL）の場合は、新しいノードを先頭および末尾として設定します。
  • リストが存在する場合は、現在の last_wal_range->next に新しいノードを接続し、last_wal_range を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/load_manifest.c 内のパース処理（JsonManifestParseContext の per_wal_range_cb フィールドに設定され、パースエンジンから間接的に呼び出されます）。
  • **呼び出し先**: palloc（PostgreSQLメモリ割り当て関数）。

関連データ構造

  • manifest_data: マニフェスト全体の情報を保持する構造体。
  • manifest_wal_range: WAL範囲を保持するリンクドリストのノード構造体。
  • JsonManifestParseContext: パース処理のコンテキスト。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、処理が中断されます。

保守担当者向けメモ

  • 本関数は simplehash.h を使用する他の関数とは異なり、リンクドリストの操作を行います。
  • manifest_data 構造体の first_wal_range と last_wal_range の整合性が維持されていることを確認してください。
