pgss_shmem_request

定義

  • 種別: function
  • 定義箇所: contrib/pg_stat_statements/pg_stat_statements.c:494

目的

PostgreSQL サーバー起動時に、pg_stat_statements 拡張機能が必要とする共有メモリ領域と LWLock トランシェを予約する。

シグネチャまたは形状

static void pgss_shmem_request(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリおよびロックの予約を行う）

前提条件と不変条件

  • shmem_request_hook に登録されることで呼び出される。
  • pgss_memsize() が計算する共有メモリサイズが、システム全体で確保可能な範囲内であること。

アルゴリズムまたは振る舞い

1. prev_shmem_request_hook が存在する場合、それを実行する（フックの連鎖）。 2. RequestAddinShmemSpace() を呼び出し、pgss_memsize() で算出されたサイズの共有メモリを要求する。 3. RequestNamedLWLockTranche() を呼び出し、pg_stat_statements 用の LWLock トランシェを 1 つ要求する。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の共有メモリ要求フック (shmem_request_hook) を介して呼び出される。
  • 呼び出し先:
  • prev_shmem_request_hook() (存在する場合)
  • RequestAddinShmemSpace()
  • pgss_memsize()
  • RequestNamedLWLockTranche()

関連データ構造

  • pgssSharedState (共有メモリ上のグローバル状態を管理する構造体)

失敗モードと境界条件

  • 共有メモリの要求量がシステム制限を超えた場合、PostgreSQL の共有メモリ初期化プロセスでエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は _PG_init() 内で shmem_request_hook = pgss_shmem_request; と設定されることでフックされる。
  • 共有メモリのレイアウトやロックの追加が必要になった場合は、pgss_memsize() の計算ロジックと本関数の要求内容を同期させる必要がある。
