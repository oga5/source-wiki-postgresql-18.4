r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_romanian.c:638

目的

ルーマニア語のステミング処理において、単語内の領域（RV, R1, R2）の境界を特定し、SN_env 構造体の整数配列 I に記録する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（ステミング環境。z->p に文字列、z->c にカーソル位置、z->l に文字列長を含む）
  • 出力: 常に 1 を返す。
  • 副作用: z->I[0]（R2の開始位置）、z->I[1]（R1の開始位置）、z->I[2]（RVの開始位置）を更新する。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。
  • 領域の境界は、母音と子音の並び順に基づいて決定される。

アルゴリズムまたは振る舞い

1. **初期化**: I[0], I[1], I[2] を文字列の末尾（z->l）に初期化する。 2. **RV領域の特定**:

  • 母音の直後の子音、または子音の直後の母音を探し、その後の位置を I[2] に記録する。
  • 条件に合致しない場合、単語の末尾が I[2] となる。

    • 3. **R1/R2領域の特定**:

  • 母音と子音の交互の並びをスキャンする。
  • 最初の母音と子音の並びの直後を I[1]（R1）に、さらにその後の母音と子音の並びの直後を I[0]（R2）に記録する。

    • 4. **カーソル復元**: 処理の各段階でカーソル位置 z->c を一時保存し、終了時に元の位置へ戻す。

呼び出し元と呼び出し先

  • 呼び出し元: romanian_UTF_8_stem（直接の呼び出し元はソースコード内で明示されていないが、Snowball生成コードの標準的な構成に基づく）。
  • 呼び出し先: in_grouping_U, out_grouping_U, skip_utf8（いずれも libstemmer 共通ユーティリティ）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。I 配列で領域境界を管理する。

失敗モードと境界条件

  • 文字列が空、または母音・子音のパターンが定義された領域条件を満たさない場合、領域境界は文字列の末尾に留まる。
  • skip_utf8 が失敗した場合、処理を中断し lab0 へジャンプする。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、対応する .sbl ファイルを修正すること。
  • g_v はルーマニア語の母音セットを指す。
  • z->I[0..2] の値は、後続の接尾辞削除処理（r_standard_suffix 等）の適用範囲を決定する重要な指標となる。
