FreeQueryDesc

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:105

目的

QueryDesc 構造体が保持するリソースを解放し、メモリをクリーンアップする。

シグネチャまたは形状

void FreeQueryDesc(QueryDesc *qdesc)

入力と出力

  • **入力**: qdesc (QueryDesc *) - 解放対象のクエリ記述子。
  • **出力**: なし。

前提条件と不変条件

  • qdesc->estate が NULL であること（実行中のクエリではないこと）が Assert により保証される必要がある。

アルゴリズムまたは振る舞い

1. qdesc->estate が NULL であることを検証する。 2. qdesc が保持する snapshot および crosscheck_snapshot に対して UnregisterSnapshot を呼び出し、参照カウントを減らす。 3. pfree(qdesc) を呼び出し、QueryDesc 構造体自体のメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyto.c
  • src/backend/commands/createas.c
  • src/backend/commands/explain.c
  • src/backend/commands/extension.c
  • src/backend/commands/matview.c
  • src/backend/commands/portalcmds.c
  • src/backend/executor/execParallel.c
  • src/backend/executor/functions.c
  • src/backend/executor/spi.c
  • **呼び出し先**:
  • UnregisterSnapshot
  • pfree

関連データ構造

  • QueryDesc: クエリ実行に必要な情報（計画、スナップショット、実行状態など）を保持する構造体。

失敗モードと境界条件

  • qdesc が NULL の場合、または qdesc->estate が NULL でない場合は Assert により異常終了する。

保守担当者向けメモ

  • 本関数はクエリ実行終了後のクリーンアップ処理の一部である。QueryDesc の生成元である CreateQueryDesc と対になる。
  • 実行中のクエリに対して呼び出すことはできないため、必ず ExecutorEnd 等で実行状態が終了した後に呼び出すこと。
