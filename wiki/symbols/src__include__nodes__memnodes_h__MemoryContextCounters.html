MemoryContextCounters

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/memnodes.h:29

目的

メモリコンテキストにおけるメモリ使用状況の統計情報を集約・保持するためのデータ構造です。

シグネチャまたは形状

typedef struct MemoryContextCounters
{
    Size        nblocks;        /* 確保されたmallocブロックの総数 */
    Size        freechunks;     /* 未使用チャンクの総数 */
    Size        totalspace;     /* mallocから要求されたバイト数の合計 */
    Size        freespace;      /* totalspaceのうち未使用の領域 */
} MemoryContextCounters;

入力と出力

本構造体は関数への引数として渡され、メモリ管理サブシステムによって統計値が書き込まれます。戻り値として使用されることはありません。

前提条件と不変条件

  • totalspace は freespace 以上の値である必要があります。
  • 各フィールドは Size 型であり、メモリ管理サブシステムによる集計結果が格納されます。

アルゴリズムまたは振る舞い

メモリコンテキストの統計情報を取得する際、各コンテキスト実装（aset.c, bump.c, generation.c 等）がこの構造体の各フィールドを計算し、呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元（推測）**: src/backend/commands/explain.c（実行計画のメモリ使用量表示）、src/backend/utils/adt/mcxtfuncs.c（メモリ統計関数）、src/backend/utils/mmgr/mcxt.c（コンテキスト統計の集約）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • MemoryContextData: メモリコンテキストの基底構造体。
  • MemoryContextMethods: メモリコンテキストの操作を定義する仮想関数テーブル。

失敗モードと境界条件

  • メモリ統計の計算中にコンテキストが破棄された場合、無効なポインタを参照するリスクがあります。
  • 非常に大規模なメモリ確保が行われている場合、Size 型のオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 新しいメモリコンテキストの実装を追加する際は、統計取得メソッド内で本構造体の各フィールドを適切に更新するようにしてください。
  • explain.c 等で表示用に使用されているため、フィールドの定義変更は表示ロジックへの影響を考慮してください。
