CycleCtr

定義

  • 種別: typedef (uint16)
  • 定義箇所: src/backend/storage/sync/sync.c:54

目的

チェックポイント処理において、ファイル同期要求の世代管理を行うためのカウンタ型です。チェックポイントの境界を跨いだ要求の処理順序を制御し、無限ループを防止するために使用されます。

シグネチャまたは形状

typedef uint16 CycleCtr;

入力と出力

  • 入力: 整数値（カウンタの増分）
  • 出力: uint16 型の数値

前提条件と不変条件

  • uint16 型として定義されており、オーバーフロー時はラップアラウンドが発生します。
  • sync_cycle_ctr との比較において、Assert((CycleCtr) (entry->cycle_ctr + 1) == sync_cycle_ctr) のような加算による順序比較が想定されています。

アルゴリズムまたは振る舞い

  • PendingFsyncEntry 構造体の一部として保持され、要求がいつ発生したかを記録します。
  • sync_cycle_ctr（現在の同期サイクル）および checkpoint_cycle_ctr（チェックポイント開始時のサイクル）と組み合わせて、要求が現在のチェックポイント処理対象であるかを判定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/sync/sync.c 内の各関数（SyncPreCheckpoint 等）が、この型を用いてカウンタ変数を操作・参照しています。
  • 呼び出し先: 本シンボルは型定義であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • PendingFsyncEntry: 同期要求を管理する構造体。cycle_ctr メンバとして本型を保持します。
  • PendingUnlinkEntry: 削除待ちファイルリストの構造体。

失敗モードと境界条件

  • uint16 の範囲を超えるカウンタのラップアラウンドが発生した場合、比較ロジックが正しく機能するよう設計上の注意が必要です。

保守担当者向けメモ

  • 本シンボルはチェックポイントの無限ループを防ぐための重要な制御変数です。ロジックを変更する際は、sync_cycle_ctr との比較演算がラップアラウンドを考慮していることを確認してください。
