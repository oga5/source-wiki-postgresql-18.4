tbm_create_pagetable

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/tidbitmap.c:280

目的

TIDBitmap 構造体において、単一ページ管理モードからハッシュテーブルを用いた複数ページ管理モードへ移行するための初期化を行う。

シグネチャまたは形状

static void tbm_create_pagetable(TIDBitmap *tbm)

入力と出力

  • **入力**: TIDBitmap *tbm（対象となるビットマップ構造体）
  • **出力**: なし（副作用として tbm の状態を更新）

前提条件と不変条件

  • tbm->status が TBM_HASH ではないこと。
  • tbm->pagetable が NULL であること。

アルゴリズムまたは振る舞い

1. pagetable_create を呼び出し、初期サイズ 128 でハッシュテーブルを生成して tbm->pagetable に割り当てる。 2. 現在の状態が TBM_ONE_PAGE である場合、tbm->entry1 に保持されている単一ページ情報をハッシュテーブルへ挿入する。

  • pagetable_insert でブロック番号に対応するエントリを確保。
  • memcpy を用いて tbm->entry1 の内容をハッシュテーブル上のエントリへコピーする。

    • 3. tbm->status を TBM_HASH に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/tidbitmap.c:1226
  • src/backend/nodes/tidbitmap.c:1298
  • **呼び出し先**:
  • pagetable_create
  • pagetable_insert

関連データ構造

  • TIDBitmap: ビットマップ全体を管理するメイン構造体。
  • PagetableEntry: ハッシュテーブルに格納されるページ情報。

失敗モードと境界条件

  • メモリ不足時には pagetable_create または pagetable_insert 内でエラーが発生する可能性がある。
  • TBM_ONE_PAGE 以外の状態から呼び出された場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は TIDBitmap のメモリ管理戦略を切り替える重要な境界点である。
  • memcpy によるコピー時、page->status を上書きしないよう一時変数 oldstatus で保護している点に注意すること。
