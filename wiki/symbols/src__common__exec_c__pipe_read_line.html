pipe_read_line

定義

  • 種別: function
  • 定義箇所: src/common/exec.c:351

目的

外部コマンドを実行し、その標準出力から最初の1行を読み取って呼び出し元に返します。

シグネチャまたは形状

char *pipe_read_line(char *cmd)

入力と出力

  • **入力**: cmd (実行するコマンド文字列)
  • **出力**: コマンドの出力の最初の1行（文字列）。失敗した場合は NULL を返します。

前提条件と不変条件

  • fflush(NULL) が呼び出され、すべてのオープン中のストリームがフラッシュされます。
  • popen(3) を使用するため、呼び出し元はコマンド実行環境の制限を受けます。
  • 戻り値の文字列は、呼び出し元で適切に解放される必要があります。

アルゴリズムまたは振る舞い

1. fflush(NULL) を実行し、バッファをフラッシュします。 2. popen(cmd, "r") でコマンドを実行します。失敗した場合は log_error を経由してエラーを報告し、NULL を返します。 3. pg_get_line(pipe_cmd, NULL) を呼び出し、パイプから1行読み取ります。 4. 読み取りに失敗した場合、ferror(pipe_cmd) を確認し、ファイル読み取りエラーかデータ不在かを判定して log_error で報告します。 5. pclose_check(pipe_cmd) を呼び出し、パイプを閉じるとともに終了ステータスを検証します。 6. 読み取った行（または NULL）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/pg_rewind.c:1110
  • src/bin/pg_upgrade/exec.c:443
  • src/common/exec.c:332
  • **呼び出し先**:
  • popen(3)
  • pg_get_line
  • pclose_check
  • log_error

関連データ構造

  • FILE (標準ライブラリのストリーム)

失敗モードと境界条件

  • popen が失敗した場合: ERRCODE_SYSTEM_ERROR を報告。
  • コマンドがデータを返さない場合: ERRCODE_NO_DATA を報告。
  • 読み取り中にエラーが発生した場合: errcode_for_file_access() に基づくエラーを報告。

保守担当者向けメモ

  • 本関数は pg_get_line を利用して行読み取りを行っています。
  • pclose_check を使用して終了ステータスのチェックを行っているため、コマンドが正常終了しなかった場合も考慮されます。
