injection_points_stats_fixed

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats_fixed.c:172

目的

インジェクションポイントの固定統計情報（attach, detach, run, cached, loaded）をSQL経由で取得し、レコードとして出力する。

シグネチャまたは形状

Datum injection_points_stats_fixed(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数インターフェース経由）
  • **出力**: 5つの INT8 型カラム（numattach, numdetach, numrun, numcached, numloaded）を持つ RECORD 型の Datum。統計が無効な場合は NULL を返す。

前提条件と不変条件

  • グローバル変数 inj_fixed_loaded および inj_stats_enabled が真であること。
  • pgstat フレームワークが PGSTAT_KIND_INJECTION_FIXED 種別の統計を保持していること。

アルゴリズムまたは振る舞い

1. 統計が有効か確認し、無効であれば NULL を返す。 2. pgstat_snapshot_fixed を呼び出し、現在の統計スナップショットを更新する。 3. pgstat_get_custom_snapshot_data を通じて統計データを取得する。 4. 5つのカラムを持つタプル記述子（TupleDesc）を生成し、各統計値を Int64GetDatum で変換して格納する。 5. heap_form_tuple を用いてレコードを構築し、PG_RETURN_DATUM で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • pgstat_snapshot_fixed
  • pgstat_get_custom_snapshot_data
  • CreateTemplateTupleDesc, TupleDescInitEntry, BlessTupleDesc
  • heap_form_tuple, HeapTupleGetDatum

関連データ構造

  • PgStat_StatInjFixedEntry: 統計値を保持する構造体。
  • TupleDesc: SQLレコードの構造定義。

失敗モードと境界条件

  • inj_fixed_loaded または inj_stats_enabled が偽の場合、統計データは返されず NULL となる。
  • 統計データが取得できない場合、メモリ割り当てやタプル構築の失敗によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は pgstat のカスタム統計機能を利用している。統計項目を追加・変更する場合は、タプル記述子の定義と values 配列のインデックスを同期させる必要がある。
