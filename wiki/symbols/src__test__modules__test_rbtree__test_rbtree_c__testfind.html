testfind

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_rbtree/test_rbtree.c:242

目的

PostgreSQL の赤黒木ライブラリ (lib/rbtree.h) における rbt_find 関数の検索機能の正当性を検証する。

シグネチャまたは形状

static void testfind(int size)

入力と出力

  • **入力**: size (int) - 挿入する要素数。
  • **出力**: なし。検証失敗時には elog(ERROR, ...) を通じてエラーを発生させる。

前提条件と不変条件

  • create_int_rbtree() によって初期化された有効な赤黒木が存在すること。
  • rbt_populate により、偶数のキーを持つノードが木に挿入されていること。

アルゴリズムまたは振る舞い

1. create_int_rbtree で空の木を作成し、rbt_populate で 0 から 2*(size-1) までの偶数キーを挿入する。 2. 挿入したすべての偶数キーについて rbt_find を実行し、ノードが正しく取得できるか、およびキー値が一致するかを確認する。 3. 挿入されていない奇数キー（-1 から 2*size の範囲）について rbt_find を実行し、ノードが取得されない（NULL が返る）ことを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_rbtree/test_rbtree.c 内の test_rb_tree 関数から呼び出される。
  • **呼び出し先**:
  • create_int_rbtree
  • rbt_populate
  • rbt_find
  • elog

関連データ構造

  • RBTree: 検証対象の赤黒木構造体。
  • IntRBTreeNode: 整数キーを保持するテスト用ノード構造体。

失敗モードと境界条件

  • 挿入したはずの要素が見つからない場合、elog(ERROR) を発行する。
  • 検索結果のキーが期待値と異なる場合、elog(ERROR) を発行する。
  • 挿入していない要素が検索で見つかった場合、elog(ERROR) を発行する。
  • 境界値として、挿入範囲外の最小値（-1）および最大値（2*size）の検索を試行する。

保守担当者向けメモ

  • 本関数は rbt_find の基本的な検索ロジックを検証する。木が再構成されるような複雑なケース（削除後など）の検証については、他のテスト関数が担当している可能性があるため、必要に応じて参照すること。
