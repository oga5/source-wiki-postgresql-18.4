cmpEntriesArg

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/ginutil.c:440

目的

GINインデックスのキー比較処理において、ソートや重複排除を行う際に必要な比較関数情報と実行時パラメータを保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	FmgrInfo   *cmpDatumFunc; /* 比較関数情報 */
	Oid			collation;    /* 照合順序 */
	bool		haveDups;     /* 重複の有無を追跡するフラグ */
} cmpEntriesArg;

入力と出力

本構造体は関数への引数として渡されるデータコンテナです。

  • **入力**: cmpDatumFunc（比較関数）、collation（照合順序）、haveDups（初期状態）。
  • **出力**: 比較処理の過程で haveDups が更新され、重複が検出された場合にその状態が保持されます。

前提条件と不変条件

  • cmpDatumFunc は有効な FmgrInfo を指している必要があります。
  • collation はインデックスの演算クラス（OpClass）に対応する適切な値である必要があります。

アルゴリズムまたは振る舞い

qsort_arg 等の比較関数に渡される補助データとして使用されます。比較関数内で本構造体へキャストして参照することで、動的な比較ロジック（照合順序の適用など）を実行しつつ、重複の発生を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginutil.c:549 にてインスタンス化されます。
  • **呼び出し先**: 比較関数内で (cmpEntriesArg *) arg としてキャストされ、内部の cmpDatumFunc が呼び出されます。

関連データ構造

  • FmgrInfo: PostgreSQL の関数マネージャ情報。
  • keyEntryData: GIN インデックスのキーエントリを管理する構造体。

失敗モードと境界条件

  • cmpDatumFunc が初期化されていない場合、比較実行時にセグメンテーションフォールトが発生する可能性があります。
  • collation が不適切な場合、予期しない順序でソートされる可能性があります。

保守担当者向けメモ

  • 本構造体は qsort_arg のような比較関数へのコンテキスト受け渡しを目的としています。
  • haveDups フラグは、インデックス構築時の重複排除の効率化に寄与します。変更を加える際は、比較関数側のロジックと整合性が取れているか確認してください。
