ExecAppendReInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:538

目的

並列クエリ実行において、Append ノードの共有メモリ状態（DSM）を初期状態にリセットする。

シグネチャまたは形状

void ExecAppendReInitializeDSM(AppendState *node, ParallelContext *pcxt)

入力と出力

  • node: AppendState 構造体へのポインタ。実行時の状態と並列実行用の共有状態（as_pstate）を保持する。
  • pcxt: ParallelContext 構造体へのポインタ。
  • 出力: なし（副作用として共有メモリ上の状態を更新）。

前提条件と不変条件

  • node->as_pstate が有効な ParallelAppendState を指していること。
  • node->as_nplans が、初期化対象となるサブプランの数として正しく設定されていること。

アルゴリズムまたは振る舞い

1. node->as_pstate から共有状態 pstate を取得する。 2. pstate->pa_next_plan を 0 にリセットし、最初のサブプランから再開するように設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execParallel.c:998 にて呼び出されていることが確認されている。
  • 呼び出し先: memset（標準ライブラリ関数）。

関連データ構造

  • AppendState: Append ノードの実行時状態を管理する構造体。
  • ParallelAppendState: 並列実行時にワーカー間で共有される状態（pa_next_plan や pa_finished を含む）。

失敗モードと境界条件

  • node または node->as_pstate が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は ExecReScanAppend 等の再スキャン処理の一環として、並列実行環境をクリーンな状態に戻すために使用される。
  • pa_finished のサイズは node->as_nplans に依存するため、プラン数が動的に変更されるようなケースでは整合性に注意が必要である。
