proclist_push_head_offset

定義

  • 種別: function
  • 定義箇所: src/include/storage/proclist.h:58

目的

共有メモリ上で管理される PGPROC の双方向連結リストにおいて、指定されたノードをリストの先頭に挿入する。

シグネチャまたは形状

static inline void proclist_push_head_offset(proclist_head *list, int procno, size_t node_offset)

入力と出力

  • list: 操作対象のリストヘッダ（proclist_head へのポインタ）。
  • procno: 挿入するプロセスの識別番号。
  • node_offset: PGPROC 構造体内の proclist_node メンバへのオフセット。
  • 戻り値: なし。

前提条件と不変条件

  • 挿入対象のノードは、現在どのリストにも属していないこと（node->next == 0 かつ node->prev == 0 であること）。
  • リストが空でない場合、list->tail は INVALID_PROC_NUMBER ではないこと。
  • 挿入するノードは、現在のリストの先頭や末尾と同一ではないこと。

アルゴリズムまたは振る舞い

1. proclist_node_get を使用して、procno と node_offset から対象ノードを取得する。 2. リストが空の場合:

  • ノードの next および prev を INVALID_PROC_NUMBER に設定する。
  • リストの head と tail を当該 procno に更新する。

    • 3. リストが空でない場合:

  • ノードの next を現在の list->head に設定する。
  • 現在の先頭ノードの prev を当該 procno に更新する。
  • ノードの prev を INVALID_PROC_NUMBER に設定する。
  • リストの head を当該 procno に更新する。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/storage/proclist.h 内の proclist_push_head マクロから利用される。
  • 呼び出し先: proclist_node_get を呼び出し、ノードのポインタを取得する。

関連データ構造

  • proclist_head: リストの先頭と末尾の procno を保持する。
  • proclist_node: リストの前後関係を保持する。
  • PGPROC: プロセス情報を保持する構造体。

失敗モードと境界条件

  • 挿入対象ノードが既にリストに属している場合、Assert により停止する可能性がある。
  • INVALID_PROC_NUMBER を含む無効なリスト状態が渡された場合、整合性が破壊される。

保守担当者向けメモ

  • 本関数はインライン関数であり、共有メモリ上のアドレス空間の違いを吸収するため、ポインタではなく procno を使用している。
  • node_offset には offsetof(PGPROC, member_name) を渡すことが想定されている。
