tidne

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:184

目的

2つの TID (Tuple Identifier) が等しくないかどうかを判定する。

シグネチャまたは形状

Datum tidne(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る2つの ItemPointer 型の引数。
  • **出力**: bool 型。2つの TID が等しくない場合に true、等しい場合に false を返す。

前提条件と不変条件

  • 入力される ItemPointer は有効なタプル位置を指している必要がある。
  • TID は物理的な位置を指すため、VACUUM FULL や CLUSTER 等でタプルが移動した場合は無効となる。

アルゴリズムまたは振る舞い

1. PG_GETARG_ITEMPOINTER マクロを使用して、引数から2つの ItemPointer を取得する。 2. ItemPointerCompare 関数を呼び出し、両者のブロック番号とオフセット番号を比較する。 3. 比較結果が 0 でない（等しくない）場合、true を返す。それ以外の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 不明（ソースコード内に明示的な呼び出し元情報は存在しません）。
  • **呼び出し先**: ItemPointerCompare

関連データ構造

  • ItemPointer: TID の内部表現。BlockIdData（ブロック番号）と OffsetNumber（オフセット番号）で構成される。

失敗モードと境界条件

  • ItemPointerCompare の仕様に依存する。
  • 引数が NULL の場合の挙動は、PostgreSQL の関数呼び出し規約（strict 関数か否か）に依存する。

保守担当者向けメモ

  • 本関数は tideq の否定論理として機能する。比較ロジックを変更する場合は ItemPointerCompare の実装を確認すること。
