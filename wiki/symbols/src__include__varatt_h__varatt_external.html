varatt_external

定義

  • 種別: struct
  • 定義箇所: src/include/varatt.h:32

目的

TOAST（The Oversized-Attribute Storage Technique）テーブルに格納された外部データへのポインタを保持するための構造体です。

シグネチャまたは形状

typedef struct varatt_external
{
    int32       va_rawsize;     /* 元のデータサイズ（ヘッダーを含む） */
    uint32      va_extinfo;     /* 外部保存サイズ（ヘッダーを除く）および圧縮手法 */
    Oid         va_valueid;     /* TOASTテーブル内での値の一意なID */
    Oid         va_toastrelid;  /* この値を保持するTOASTテーブルのRelID */
} varatt_external;

入力と出力

本構造体はデータアクセス層において、TOASTされたデータへの参照を保持するために使用されます。入力として、TOASTテーブルの識別子やデータIDを格納し、出力として、それらの情報を基に実際のデータ本体を特定・取得するためのメタデータを提供します。

前提条件と不変条件

  • va_rawsize にはヘッダーサイズが含まれます。
  • va_extinfo にはヘッダーを除いたサイズと圧縮手法の情報がエンコードされています。
  • 本構造体はアライメントが保証されない可能性があるため、アクセス時には memcpy 等によるコピーが推奨されます。

アルゴリズムまたは振る舞い

TOASTされたデータへのアクセスが必要な際、システムはこの構造体を参照して、どのテーブル（va_toastrelid）のどの行（va_valueid）からデータを読み出すかを決定します。また、va_extinfo を解析することで、データの圧縮状態やサイズを判定し、必要に応じて展開処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • contrib/amcheck/verify_heapam.c
  • src/backend/access/common/detoast.c
  • src/backend/access/common/toast_compression.c
  • src/backend/access/common/toast_internals.c
  • src/backend/replication/logical/reorderbuffer.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。データ取得処理は、本構造体を引数として受け取る detoast 関連の関数群によって実行されます。

関連データ構造

  • varlena: 可変長データの基底構造。
  • vartag_external: TOASTポインタの種類を識別する列挙型。
  • varattrib_4b / varattrib_1b: インラインデータのヘッダー構造。

失敗モードと境界条件

  • va_toastrelid が無効な場合、データ取得時にテーブル参照エラーが発生します。
  • 構造体への直接キャストによるアクセスは、アライメント違反によるクラッシュや誤ったデータ読み込みを引き起こす可能性があるため避けるべきです。

保守担当者向けメモ

  • エンディアン依存のビット配置があるため、va_extinfo の操作には必ず提供されているマクロを使用してください。
  • 1バイトヘッダーのデータと混同しないよう、VARATT_IS_EXTERNAL 等の述語マクロで形式を確認してから本構造体として扱う必要があります。
