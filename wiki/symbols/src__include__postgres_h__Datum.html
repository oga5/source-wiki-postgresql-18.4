Datum

定義

  • 種別: typedef
  • 定義箇所: src/include/postgres.h:69

目的

PostgreSQL の実行エンジン内において、任意のデータ型を保持・受け渡しするための汎用的なデータ表現型です。

シグネチャまたは形状

typedef uintptr_t Datum;

入力と出力

  • **入力**: C の基本型（bool, int, float, ポインタ等）を *GetDatum マクロを介して変換した値。
  • **出力**: DatumGet* マクロを介して元の型へ復元される値。

前提条件と不変条件

  • Datum は sizeof(void *) と同じサイズ（64bit 環境では 8 バイト、32bit 環境では 4 バイト）であると仮定されます。
  • 値渡し（pass-by-value）または参照渡し（pass-by-reference）のポインタを保持します。
  • 浮動小数点数などのプラットフォーム依存の型を扱う際は、直接キャストせず、必ず提供されている変換関数（Float8GetDatum 等）を使用する必要があります。

アルゴリズムまたは振る舞い

  • PostgreSQL の実行エンジン内で「共通通貨」として機能し、データ型を抽象化して効率的に受け渡します。
  • 変換関数はすべて static inline で定義されており、関数呼び出しのオーバーヘッドを最小限に抑えつつ型安全性を確保します。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • contrib/amcheck/verify_gin.c
  • contrib/amcheck/verify_heapam.c
  • contrib/amcheck/verify_nbtree.c
  • **呼び出し先**:
  • 提供されたソース情報には、Datum 自体が呼び出す関数や、Datum を直接呼び出す関数の具体的なリストは含まれていません。

関連データ構造

  • NullableDatum: 値と NULL フラグをペアで保持する構造体。
  • c.h: uintptr_t 等の基盤となる型定義を提供。

失敗モードと境界条件

  • 移植性の欠如: 直接キャストを行うと、プラットフォームごとのサイズや表現の違いにより移植性が損なわれます。
  • サイズ制約: void * のサイズを超えるデータを直接保持することはできません。

保守担当者向けメモ

  • バックエンドのソースコードでは、必ず postgres.h を先頭でインクルードしてください。
  • Datum は汎用型であるため、操作時には必ず適切な変換マクロ（DatumGet* / *GetDatum）を介して型安全性を維持してください。
