my_int128_compare

定義

  • 種別: static inline function
  • 定義箇所: src/tools/testint128.c:55

目的

ネイティブな int128 型の比較演算子を用いて、2 つの 128 ビット整数を比較し、その結果を標準的な比較値（-1, 0, 1）として返す。

シグネチャまたは形状

static inline int my_int128_compare(int128 x, int128 y)

入力と出力

  • **入力**: 比較対象となる int128 型の変数 x および y。
  • **出力**:
  • x < y の場合: -1
  • x > y の場合: 1
  • x == y の場合: 0

前提条件と不変条件

  • コンパイラが int128 型をネイティブにサポートしている環境で動作することを前提とする。

アルゴリズムまたは振る舞い

1. x と y を比較演算子 < で評価し、真であれば -1 を返す。 2. x と y を比較演算子 > で評価し、真であれば 1 を返す。 3. 上記のいずれでもない場合（等しい場合）、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/testint128.c 内の以下の箇所から呼び出されている。
  • 143行目: if (my_int128_compare(t1.i128, t2.i128) != ...)
  • 147行目: my_int128_compare(t1.i128, t2.i128), ...
  • 157行目: if (my_int128_compare(t1.i128, t2.i128) != ...)
  • 161行目: my_int128_compare(t1.i128, t2.i128), ...
  • **呼び出し先**: ネイティブの比較演算子（<, >）。

関連データ構造

  • int128: ネイティブな 128 ビット整数型。

失敗モードと境界条件

  • 特になし。入力された int128 値の全範囲に対して定義された挙動を行う。

保守担当者向けメモ

  • 本関数は testint128.c における検証の基準（リファレンス）として機能する。PostgreSQL 独自実装の比較ロジックが正しいかを判定するために使用されるため、本関数のロジックを変更する際は注意が必要である。
