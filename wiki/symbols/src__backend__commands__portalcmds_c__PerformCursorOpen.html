PerformCursorOpen

定義

  • 種別: function
  • 定義箇所: src/backend/commands/portalcmds.c:44

目的

SQLの DECLARE CURSOR コマンドを実行し、クエリのプラン作成からポータルオブジェクトの初期化までを行う。

シグネチャまたは形状

void PerformCursorOpen(ParseState *pstate, DeclareCursorStmt *cstmt, ParamListInfo params, bool isTopLevel)

入力と出力

  • **入力**:
  • pstate: 解析状態。
  • cstmt: DECLARE CURSOR 文の構文木。
  • params: クエリ実行時に使用するパラメータリスト。
  • isTopLevel: トップレベルの実行か否かを示すフラグ。
  • **出力**: なし（副作用としてポータルが生成され、実行準備が完了する）。

前提条件と不変条件

  • カーソル名は空文字列であってはならない。
  • WITH HOLD オプションがない場合、トランザクションブロック内で実行される必要がある。
  • WITH HOLD オプションがある場合、セキュリティ制限下（InSecurityRestrictedOperation）での実行は禁止される。
  • 実行されるクエリは SELECT 文でなければならない。

アルゴリズムまたは振る舞い

1. カーソル名のバリデーションとトランザクション状態の確認を行う。 2. 必要に応じてクエリの識別子（QueryId）を生成し、解析フックを実行する。 3. QueryRewrite を呼び出し、クエリを書き換える。 4. pg_plan_query を使用してクエリのプランを作成する。 5. CreatePortal でポータルを生成し、プランとクエリ文字列をポータルのメモリコンテキストにコピーする。 6. パラメータリストをコピーし、ポータルに設定する。 7. スクロールオプションを決定し、PortalStart を呼び出して実行準備を完了させる。 8. 実際のクエリ実行は行わず、PerformPortalFetch 呼び出しを待機する状態にする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c (直接の呼び出し)
  • src/backend/executor/spi.c (参照)
  • **呼び出し先**:
  • JumbleQuery, QueryRewrite, pg_plan_query, CreatePortal, PortalDefineQuery, PortalStart 等。

関連データ構造

  • DeclareCursorStmt: DECLARE CURSOR 文の構造体。
  • Portal: クエリ実行状態を保持するオブジェクト。
  • PlannedStmt: 生成されたクエリプラン。

失敗モードと境界条件

  • 空のカーソル名が指定された場合、ERRCODE_INVALID_CURSOR_NAME でエラー終了する。
  • トランザクションブロック外で WITH HOLD なしカーソルを作成しようとした場合、エラー終了する。
  • セキュリティ制限下で WITH HOLD カーソルを作成しようとした場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了する。
  • SELECT 以外のクエリが指定された場合、エラー終了する。

保守担当者向けメモ

  • 本関数はポータルの初期化のみを担当し、実際のデータ取得は PerformPortalFetch が担う。
  • PortalStart 呼び出し時に GetActiveSnapshot() を使用している点に注意。
  • copyObject(plan) を使用してプランをポータル専用のメモリコンテキストへコピーしており、メモリ管理の整合性が保たれている。
