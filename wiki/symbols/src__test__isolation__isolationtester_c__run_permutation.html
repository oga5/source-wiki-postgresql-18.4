run_permutation

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolationtester.c:519

目的

指定された順列（Permutation）に従い、PostgreSQLの分離レベルテストにおけるセットアップ、ステップ実行、および完了待機の一連のプロセスを制御する。

シグネチャまたは形状

static void run_permutation(TestSpec *testspec, int nsteps, PermutationStep **steps)

入力と出力

  • **入力**:
  • testspec: テスト仕様全体を保持する TestSpec 構造体。
  • nsteps: 実行するステップ数。
  • steps: 実行順序に従って並べられた PermutationStep の配列。
  • **出力**: なし（成功時はテストを実行し、失敗時は exit(1) によりプロセスを終了する）。

前提条件と不変条件

  • conns 配列（グローバル変数）が初期化済みであること。
  • 各セッションの接続が確立されており、application_name 等の設定が完了していること。
  • 実行中にセッションがロック待ち等でハングアップした場合、max_step_wait の2倍の時間でタイムアウトし、異常終了する。

アルゴリズムまたは振る舞い

1. **セットアップ**: 全体共通のSQLおよび各セッション固有のセットアップSQLを順次実行する。 2. **ステップ実行ループ**: 指定された各ステップを以下の手順で処理する。

  • **ブロック確認**: 対象セッションが以前のステップでアクティブな場合、try_complete_step を用いて完了を待機する。
  • **クエリ発行**: PQsendQuery を使用して非同期にSQLを発行する。
  • **状態更新**: iconn->active_step に現在のステップを記録し、ブロック条件（PSB_NUM_NOTICES 等）のターゲットを計算する。
  • **ポーリング**: try_complete_step および try_complete_steps を呼び出し、ステップの完了を監視する。
  • **待機管理**: 完了していないステップを waiting 配列に追加し、後続のステップ実行時に再評価する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/isolation/isolationtester.c 内の run_testspec および run_all_permutations_recurse（推論される呼び出し元）。
  • **呼び出し先**:
  • PQexec, PQsendQuery, PQresultStatus, PQclear, PQerrorMessage (libpq)
  • try_complete_step, try_complete_steps (isolationtester.c)
  • printResultSet, gettimeofday, exit, fprintf, printf

関連データ構造

  • TestSpec: テスト全体の構成情報。
  • PermutationStep: 実行対象のステップ定義。
  • IsoConnInfo: セッションごとの接続状態とアクティブなステップ情報。

失敗モードと境界条件

  • **セットアップ失敗**: PQexec が PGRES_COMMAND_OK 以外を返した場合、エラーメッセージを出力して終了する。
  • **タイムアウト**: ステップ実行が 2 * max_step_wait を超えた場合、アクティブなステップ名を表示して異常終了する。
  • **クエリ発行失敗**: PQsendQuery が失敗した場合、エラーを出力して終了する。

保守担当者向けメモ

  • 本関数は conns グローバル配列に強く依存している。
  • waiting 配列の管理には memmove を使用しており、ステップ数が多い場合の計算量に注意が必要。
  • ブロック条件の判定ロジックは try_complete_step に委譲されているため、同期ロジックの変更時はそちらも参照すること。
