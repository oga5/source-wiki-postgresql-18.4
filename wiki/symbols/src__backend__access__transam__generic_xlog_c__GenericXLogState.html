GenericXLogState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/generic_xlog.c:64

目的

汎用的なWAL（Write-Ahead Logging）レコードを構築する際、変更対象となる複数のページイメージと、それらに関連するメタデータを一時的に保持するための状態管理構造体です。

シグネチャまたは形状

struct GenericXLogState
{
    PGIOAlignedBlock images[MAX_GENERIC_XLOG_PAGES];
    GenericXLogPageData pages[MAX_GENERIC_XLOG_PAGES];
    bool        isLogged;
};

入力と出力

  • **入力**: GenericXLogStart() によりメモリが割り当てられ、GenericXLogRegisterBuffer() を通じてバッファの内容が images にコピーされます。
  • **出力**: GenericXLogFinish() に渡されることで、保持しているページイメージの差分が計算され、WALレコードとして書き出されます。

前提条件と不変条件

  • images フィールドは、アライメントを維持するため構造体の先頭に配置される必要があります。
  • MAX_GENERIC_XLOG_PAGES で定義された上限数までのバッファを登録可能です。
  • 構築中の状態は GenericXLogFinish() または GenericXLogAbort() が呼び出されるまで有効です。

アルゴリズムまたは振る舞い

1. **初期化**: GenericXLogStart() でメモリが確保され、状態が初期化されます。 2. **登録**: GenericXLogRegisterBuffer() により、対象バッファの現在のページイメージが images 配列に格納されます。 3. **編集**: 呼び出し元は、返されたページイメージを直接編集します。 4. **確定**: GenericXLogFinish() が呼び出されると、編集後のイメージと元のイメージを比較し、差分（デルタ）を抽出してWALレコードを生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/bloom/ 配下のインデックス操作関数（blinsert.c, blutils.c, blvacuum.c など）がこの構造体を保持・操作します。
  • **呼び出し先**: GenericXLogStart, GenericXLogRegisterBuffer, GenericXLogFinish などの汎用WAL API関数群が、この構造体を引数として受け取り操作します。

関連データ構造

  • GenericXLogPageData: 各ページに関するメタデータ（ページ番号やフラグなど）を保持します。
  • PGIOAlignedBlock: ページイメージを格納するためのアライメントされたブロック型です。

失敗モードと境界条件

  • MAX_GENERIC_XLOG_PAGES を超えるバッファを登録しようとした場合、エラーが発生します。
  • メモリ不足時には palloc_aligned による割り当てが失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は generic_xlog.c 内で完結する内部的な状態保持用ですが、contrib モジュール等からポインタとして参照されます。
  • 構造体のメンバ順序（特に images の先頭配置）は、メモリレイアウトの整合性に直結するため変更時に注意が必要です。
