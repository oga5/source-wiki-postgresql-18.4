SyncRepGetOldestSyncRecPtr

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:659

目的

同期スタンバイのリストを走査し、全スタンバイの中で最も遅れている（最小の）Write、Flush、およびApplyのWAL位置を特定する。

シグネチャまたは形状

static void SyncRepGetOldestSyncRecPtr(XLogRecPtr *writePtr,
                                       XLogRecPtr *flushPtr,
                                       XLogRecPtr *applyPtr,
                                       SyncRepStandbyData *sync_standbys,
                                       int num_standbys)

入力と出力

  • **入力**:
  • writePtr, flushPtr, applyPtr: 最小値を格納するためのポインタ（呼び出し元で InvalidXLogRecPtr に初期化されていることを前提とする）。
  • sync_standbys: 評価対象となる同期スタンバイのデータ配列。
  • num_standbys: 配列内のスタンバイ数。
  • **出力**:
  • 引数 writePtr, flushPtr, applyPtr を通じて、各指標の最小値が返される。

前提条件と不変条件

  • writePtr, flushPtr, applyPtr が指す値は、関数呼び出し前に InvalidXLogRecPtr で初期化されている必要がある。
  • sync_standbys 配列は有効な SyncRepStandbyData を num_standbys 個保持していること。

アルゴリズムまたは振る舞い

1. num_standbys 分だけループし、各スタンバイの write, flush, apply 位置を順次確認する。 2. 各ポインタが指す値が InvalidXLogRecPtr であるか、現在のスタンバイの進捗位置がポインタの値よりも小さい場合、その値をポインタに代入する。 3. これにより、全スタンバイの中で最も進捗が遅い（最小の）LSNが各ポインタに格納される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/syncrep.c 内の関数（SyncRepGetSyncRecPtr などからの参照が確認されている）。
  • **呼び出し先**: 直接的な関数呼び出しは含まない。

関連データ構造

  • XLogRecPtr: WAL位置を保持する型。
  • SyncRepStandbyData: 各スタンバイの進捗状態（write, flush, apply）を保持する構造体。

失敗モードと境界条件

  • num_standbys が 0 の場合、ポインタの値は更新されず、初期値（InvalidXLogRecPtr）のまま維持される。
  • InvalidXLogRecPtr が含まれる場合、比較ロジックによりそれが優先的に最小値として扱われる。

保守担当者向けメモ

  • 本関数は同期レプリケーションの進捗判定の基礎となる。計算ロジックを変更する場合は、SyncRepGetSyncRecPtr 等の呼び出し元への影響を考慮すること。
