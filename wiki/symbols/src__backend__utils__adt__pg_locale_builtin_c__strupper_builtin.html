strupper_builtin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_builtin.c:106

目的

PostgreSQLの組み込みロケールプロバイダにおいて、Unicode規格に基づいた文字列の大文字変換を行う。

シグネチャまたは形状

size_t strupper_builtin(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: 変換後の文字列を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換元の文字列。
  • srclen: src の長さ（バイト単位）。
  • locale: 組み込みロケール情報を含む pg_locale_t 構造体。
  • **出力**:
  • 変換後の文字列の長さ（バイト単位）を返す。

前提条件と不変条件

  • locale 引数は、組み込みロケールプロバイダによって初期化された有効な pg_locale_t 構造体を指している必要がある。
  • locale->info.builtin.casemap_full が、Unicode大文字変換のための有効なマッピングテーブルを参照していること。

アルゴリズムまたは振る舞い

1. locale 構造体内の info.builtin.casemap_full を参照する。 2. unicode_strupper 関数を呼び出し、指定されたマッピングテーブルを用いて src から dest への大文字変換を実行する。 3. 変換結果の長さを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c: ロケール抽象化レイヤーのメインディスパッチャから呼び出される。
  • **呼び出し先**:
  • unicode_strupper (定義箇所不明、common/unicode_case.h 経由の可能性が高い)

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。内部の info.builtin フィールドを通じて変換テーブルにアクセスする。

失敗モードと境界条件

  • destsize が変換後の文字列を格納するのに不十分な場合、unicode_strupper の仕様に従った切り詰めやエラーが発生する可能性がある（詳細は unicode_strupper の実装に依存）。

保守担当者向けメモ

  • 本関数は libc や ICU に依存せず、PostgreSQL 内部の Unicode 変換テーブルを使用する。
  • 変換ロジックの変更が必要な場合は、common/unicode_case.h および unicode_strupper の実装を確認すること。
