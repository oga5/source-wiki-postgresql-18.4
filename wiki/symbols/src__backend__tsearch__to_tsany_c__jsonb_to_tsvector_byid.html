jsonb_to_tsvector_byid

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/to_tsany.c:327

目的

指定された全文検索設定（Config OID）とフラグに基づき、jsonb 型のデータを全文検索用の tsvector 型へ変換する。

シグネチャまたは形状

Datum jsonb_to_tsvector_byid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 全文検索設定の OID。
  • PG_GETARG_JSONB_P(1): 変換対象の Jsonb データ。
  • PG_GETARG_JSONB_P(2): インデックス処理の挙動を制御するフラグを含む Jsonb データ。
  • **出力**:
  • PG_RETURN_TSVECTOR(result): 変換された TSVector 構造体。

前提条件と不変条件

  • 入力された Jsonb オブジェクトは有効である必要がある。
  • parse_jsonb_index_flags を介してフラグが正しく解析されること。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID および PG_GETARG_JSONB_P を使用して引数を取得する。 2. parse_jsonb_index_flags を呼び出し、第3引数から uint32 型のフラグを生成する。 3. jsonb_to_tsvector_worker を呼び出し、設定とフラグに基づいて jsonb を tsvector に変換する。 4. PG_FREE_IF_COPY を使用して、入力された Jsonb ポインタのメモリを適切に解放する。 5. 構築された TSVector を PG_RETURN_TSVECTOR で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出される（直接的な C 言語レベルの呼び出し元情報は提供されていない）。
  • **呼び出し先**:
  • parse_jsonb_index_flags
  • jsonb_to_tsvector_worker

関連データ構造

  • Jsonb: 入力データ構造。
  • TSVector: 出力データ構造。
  • Oid: 全文検索設定の識別子。

失敗モードと境界条件

  • jsonb_to_tsvector_worker 内で発生するエラー（設定の不備やメモリ不足など）が伝播する可能性がある。
  • 入力引数が不正な場合、PG_GETARG_* マクロの挙動に従いエラーが発生する。

保守担当者向けメモ

  • 本関数は jsonb_to_tsvector_worker のラッパーとして機能している。
  • メモリ管理において PG_FREE_IF_COPY を使用しているため、引数のコピーが発生した場合のみメモリ解放が行われる点に注意すること。
