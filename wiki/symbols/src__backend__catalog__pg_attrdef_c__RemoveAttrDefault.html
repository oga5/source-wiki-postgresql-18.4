RemoveAttrDefault

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_attrdef.c:151

目的

指定されたリレーション（テーブル）の特定の列に関連付けられたデフォルト値（pg_attrdef エントリ）を削除し、関連する依存関係を解消します。

シグネチャまたは形状

void RemoveAttrDefault(Oid relid, AttrNumber attnum, DropBehavior behavior, bool complain, bool internal)

入力と出力

  • **入力**:
  • relid: 対象テーブルの OID。
  • attnum: 対象列の属性番号。
  • behavior: 削除時の振る舞い（DROP_RESTRICT 等）。
  • complain: 削除対象が見つからない場合にエラーを投げるかどうかのフラグ。
  • internal: 内部的な削除（PERFORM_DELETION_INTERNAL）として処理するかどうかのフラグ。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • pg_attrdef カタログに対する RowExclusiveLock を取得して操作を行います。
  • 指定された relid と attnum の組み合わせに対して、pg_attrdef には最大で 1 つのエントリが存在することが前提です。

アルゴリズムまたは振る舞い

1. AttrDefaultRelationId を開きます。 2. Anum_pg_attrdef_adrelid および Anum_pg_attrdef_adnum をキーとして AttrDefaultIndexId を使用したスキャンを開始します。 3. 検索で見つかった各タプルに対し、ObjectAddress を構築し、performDeletion を呼び出して削除処理を実行します。 4. complain が true であり、かつ対象が見つからなかった場合は elog(ERROR) を発生させます。 5. スキャンを終了し、リレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c 内の ALTER TABLE 関連処理（ATDropColumn 等）から複数回呼び出されています。
  • **呼び出し先**: table_open, ScanKeyInit, systable_beginscan, systable_getnext, performDeletion, systable_endscan, table_close。

関連データ構造

  • Relation: pg_attrdef カタログへのアクセスに使用。
  • ScanKeyData: カタログ検索条件の定義に使用。
  • ObjectAddress: 削除対象のオブジェクト識別に使用。
  • Form_pg_attrdef: pg_attrdef カタログのタプル構造。

失敗モードと境界条件

  • 対象のデフォルト値が存在しない場合、complain が true であればエラーとなります。
  • performDeletion の内部ロジックにより、依存関係の制約（RESTRICT 等）に違反する場合は削除が拒否される可能性があります。

保守担当者向けメモ

  • 本関数は pg_attrdef のエントリを削除するのみであり、pg_attribute の atthasdef フラグの更新は呼び出し元または別の処理で行われる必要があります。
  • 生成列（Generated Column）の削除時には internal フラグを適切に設定し、依存関係の整合性を維持してください。
