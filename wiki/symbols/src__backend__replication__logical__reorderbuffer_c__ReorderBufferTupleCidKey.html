ReorderBufferTupleCidKey

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/reorderbuffer.c:136

目的

論理デコーディングにおいて、特定のタプルに対するコマンドID（CID）マッピングをハッシュテーブルで管理するための検索キーとして使用されます。

シグネチャまたは形状

typedef struct ReorderBufferTupleCidKey
{
    RelFileLocator rlocator; /* リレーションの識別子 */
    ItemPointerData tid;     /* タプルの物理位置（TID） */
} ReorderBufferTupleCidKey;

入力と出力

  • **入力**: 本構造体は、リレーションのロケータとタプルのTIDを保持するコンテナです。
  • **出力**: ハッシュテーブルの検索キーとして使用され、対応する ReorderBufferTupleCidEnt を特定するために利用されます。

前提条件と不変条件

  • rlocator と tid の組み合わせにより、データベース内の特定のタプルを一意に識別できる必要があります。
  • ハッシュテーブルのキーとして使用されるため、検索時には memset 等で構造体全体をゼロ初期化し、パディングによるゴミデータの影響を排除することが推奨されます。

アルゴリズムまたは振る舞い

  • ReorderBuffer 内で、カタログ変更を伴うトランザクションの CID マッピングを追跡する際に、このキーを用いてハッシュテーブル（hash_ctl.keysize に本構造体のサイズを指定）を操作します。
  • 変更レコードの処理時に、対象タプルの CID を解決するためのルックアップキーとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: reorderbuffer.c 内の複数の箇所（1843, 1857, 1867, 5350, 5356, 5532行目付近）で、ハッシュテーブルの定義や検索キーの初期化に使用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ReorderBufferTupleCidEnt: 本キーに対応するハッシュテーブルのエントリ構造体。
  • RelFileLocator: リレーションを識別するための構造体。
  • ItemPointerData: タプルの物理位置を示す TID。

失敗モードと境界条件

  • 構造体の初期化が不完全（memset 漏れなど）な場合、ハッシュ検索時に予期せぬ不一致が発生する可能性があります。
  • 存在しないタプルや無効な rlocator を指定した場合、ハッシュテーブル検索は失敗します。

保守担当者向けメモ

  • 本構造体はハッシュキーとして使用されるため、メンバの追加や変更を行う際は、ハッシュ関数の計算ロジックや比較ロジックに影響がないか確認してください。
  • 構造体のアライメントやパディングがハッシュの整合性に影響を与える可能性があるため、変更時は注意が必要です。
