bits_to_text

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/heapfuncs.c:61

目的

ビット配列（bits8）を、'0' と '1' からなる人間が読み取り可能な文字列形式に変換します。

シグネチャまたは形状

static char *bits_to_text(bits8 *bits, int len)

入力と出力

  • **入力**:
  • bits: 変換対象のビット列を保持する bits8 型のポインタ。
  • len: 変換するビット数。
  • **出力**:
  • 変換された文字列を格納した char *（呼び出し元で解放が必要）。

前提条件と不変条件

  • len は、bits が指すメモリ領域の有効なビット数である必要があります。
  • palloc を使用してメモリを確保するため、PostgreSQL のメモリコンテキスト内で呼び出される必要があります。

アルゴリズムまたは振る舞い

1. len + 1 バイトのメモリを palloc で確保します。 2. 0 から len - 1 までのインデックス i についてループを実行します。 3. 各ビット位置 i に対して、bits[(i / 8)] の該当ビット（1 << (i % 8)）を検査します。 4. ビットがセットされていれば '1'、そうでなければ '0' を文字列の該当位置に書き込みます。 5. 文字列の末尾にヌル終端文字 \0 を付与して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pageinspect/heapfuncs.c:250 において、tuphdr->t_bits を引数として呼び出されています。
  • **呼び出し先**: palloc（PostgreSQL メモリ管理関数）。

関連データ構造

  • bits8: PostgreSQL で定義された 8 ビット符号なし整数型。

失敗モードと境界条件

  • len が 0 の場合、ヌル終端文字のみを含む空文字列が返されます。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は static であり、heapfuncs.c 内でのみ利用されます。
  • ビット順序はリトルエンディアン的なビット配置（i % 8）を前提としています。
