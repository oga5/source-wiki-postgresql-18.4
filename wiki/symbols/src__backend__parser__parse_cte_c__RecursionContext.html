RecursionContext

定義

  • 種別: enum
  • 定義箇所: src/backend/parser/parse_cte.c:31

目的

再帰的共通テーブル式（CTE）の解析において、再帰参照が許可される場所と禁止される場所を区別し、SQL標準に従った構造であることを検証するための状態を保持します。

シグネチャまたは形状

typedef enum {
    RECURSION_OK,
    RECURSION_NONRECURSIVETERM,
    RECURSION_SUBLINK,
    RECURSION_OUTERJOIN,
    RECURSION_INTERSECT,
    RECURSION_EXCEPT,
} RecursionContext;

入力と出力

本シンボルは列挙型であり、関数への入力として渡されるか、構造体（CteState）のメンバとして保持されます。出力として、checkWellFormedRecursionWalker 等の再帰的走査関数において、現在の解析位置が再帰的に有効かどうかを判断する基準となります。

前提条件と不変条件

  • 再帰的CTEの検証中、CteState 構造体内の context フィールドとして保持されます。
  • 再帰的な構造（JOIN や SubLink など）を走査する際、現在のコンテキストを保存し、新しいコンテキストへ切り替えて検証を行う必要があります。

アルゴリズムまたは振る舞い

checkWellFormedRecursionWalker がツリーを走査する際、この列挙値を用いて現在の構文要素が再帰参照を許容するかを判定します。例えば、RECURSION_SUBLINK や RECURSION_OUTERJOIN などの値は、特定の構文構造内での再帰参照が禁止されていることを示します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_cte.c 内の checkWellFormedRecursionWalker 等が、再帰的な検証を行う際にこの型を参照・更新します。
  • **呼び出し先**: 本シンボル自体は型定義であり、関数呼び出しは行いません。

関連データ構造

  • CteState: 解析中の状態を保持する構造体。RecursionContext context フィールドを保持します。

失敗モードと境界条件

  • 適切なコンテキスト遷移を行わずに再帰参照を許可すると、SQL標準に違反する不正な再帰的CTEが生成される可能性があります。
  • 境界条件として、INTERSECT や EXCEPT などの集合演算子配下での再帰参照の可否判定に使用されます。

保守担当者向けメモ

  • 本列挙型は checkWellFormedRecursionWalker の状態遷移ロジックと密接に結びついています。新しい構文要素が再帰的CTEの検証に追加される場合は、この列挙型に適切な状態を追加し、ウォーカー関数でのハンドリングを更新してください。
