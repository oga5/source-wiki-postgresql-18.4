gistinsert

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:164

目的

GiSTインデックスへのタプル挿入要求を受け付け、適切なリーフノードへの配置を統括する公開APIです。

シグネチャまたは形状

bool gistinsert(Relation r, Datum *values, bool *isnull,
                ItemPointer ht_ctid, Relation heapRel,
                IndexUniqueCheck checkUnique,
                bool indexUnchanged,
                IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • r: 挿入対象のインデックスリレーション。
  • values, isnull: 挿入する属性値とそのNULLフラグ。
  • ht_ctid: ヒープタプルを指すTID。
  • indexInfo: インデックス情報およびキャッシュ用構造体。
  • **出力**:
  • 戻り値: 常に false を返します。

前提条件と不変条件

  • indexInfo->ii_AmCache が NULL の場合、GISTSTATE の初期化とメモリコンテキストの作成が行われます。
  • 挿入処理は giststate->tempCxt 上で実行され、処理終了後にリセットされます。

アルゴリズムまたは振る舞い

1. **初期化**: indexInfo->ii_AmCache を確認し、初回呼び出しであれば GISTSTATE を初期化してキャッシュします。 2. **タプル生成**: gistFormTuple を使用して、入力値からインデックス用の IndexTuple を作成し、TID を設定します。 3. **挿入実行**: gistdoinsert を呼び出し、ルートからリーフへの降下および実際のページへの挿入処理を実行します。 4. **クリーンアップ**: 一時メモリコンテキストをリセットし、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのインデックス管理システム（amroutine->aminsert 経由）。
  • **呼び出し先**:
  • initGISTstate (初期化時)
  • createTempGistContext (初期化時)
  • gistFormTuple
  • gistdoinsert

関連データ構造

  • GISTSTATE: GiSTインデックス操作の実行状態を保持する構造体。
  • IndexTuple: インデックスに格納されるタプル形式。
  • Relation: インデックスおよびヒープリレーションのメタデータ。

失敗モードと境界条件

  • メモリ不足時、MemoryContext 関連の処理でエラーが発生する可能性があります。
  • ページ分割が必要な場合、gistdoinsert 内部で再帰的なツリー更新が発生します。

保守担当者向けメモ

  • 本関数は GiST 挿入のフロントエンドであり、実際のツリー操作ロジックは gistdoinsert に委譲されています。
  • GISTSTATE のキャッシュ管理は、ステートメント内での初回呼び出し時にのみ行われるため、キャッシュのライフサイクルに注意してください。
