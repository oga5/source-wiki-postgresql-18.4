SerialIOData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:35

目的

array_agg 集約関数の並列クエリ実行時における、シリアライズ処理のための状態保持用構造体です。

シグネチャまたは形状

typedef struct SerialIOData
{
	FmgrInfo	typsend;
} SerialIOData;

入力と出力

  • **入力**: FmgrInfo 型の typsend を保持します。これは、配列要素をバイナリ形式に変換するための関数情報です。
  • **出力**: なし（本構造体は状態保持用です）。

前提条件と不変条件

  • fn_extra を介して関数呼び出し間でキャッシュされることを前提としています。
  • typsend は、対象となる配列要素の型に対応する送信関数（send function）の情報を含んでいる必要があります。

アルゴリズムまたは振る舞い

  • array_agg_serialize 関数内において、集約状態を転送可能な形式に変換する際、要素のシリアライズに必要な関数情報を保持するために使用されます。
  • 初回呼び出し時に fn_extra が未初期化であれば、この構造体を割り当てて初期化し、以降の呼び出しで再利用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_userfuncs.c 内の array_agg_serialize 関数が、この構造体を fcinfo->flinfo->fn_extra から取得または初期化して使用します。
  • **呼び出し先**: 保持している typsend を通じて、要素型の送信関数が呼び出されます。

関連データ構造

  • FmgrInfo: PostgreSQL の関数マネージャ情報。
  • DeserialIOData: 対となるデシリアライズ用構造体。

失敗モードと境界条件

  • メモリ不足: fn_extra の初期化時に palloc が失敗する可能性があります。
  • 型情報の不整合: typsend が正しく初期化されていない場合、シリアライズ処理中にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は array_agg の並列実行をサポートするためのキャッシュ機構の一部です。
  • fn_extra を利用したキャッシュ戦略を採用しているため、構造体の変更時には array_agg_serialize および array_agg_deserialize での整合性に注意してください。
