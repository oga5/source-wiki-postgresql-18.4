polymorphic_actuals

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/fmgr/funcapi.c:35

目的

多相型（Polymorphic types）を使用する関数において、実行時に確定した具体的な型（Oid）を保持するための構造体です。

シグネチャまたは形状

typedef struct polymorphic_actuals
{
	Oid			anyelement_type;	/* anyelement mapping, if known */
	Oid			anyarray_type;	/* anyarray mapping, if known */
	Oid			anyrange_type;	/* anyrange mapping, if known */
	Oid			anymultirange_type; /* anymultirange mapping, if known */
} polymorphic_actuals;

入力と出力

  • 入力: 各フィールドは Oid 型であり、PostgreSQLのシステムカタログ pg_type における型IDを保持します。
  • 出力: 本構造体は、多相型解決ロジック（resolve_*_from_others 系関数）において、推論された型情報を格納するコンテナとして機能します。

前提条件と不変条件

  • 各フィールドは、型が未確定または不明な場合、InvalidOid を保持することが想定されます。
  • 本構造体を利用する関数は、パーサーによって型の一貫性が保証されていることを前提としています。

アルゴリズムまたは振る舞い

  • 多相型（anyelement, anyarray, anyrange, anymultirange）のプレースホルダーを、呼び出し時の引数型から推論して具体化するプロセスで使用されます。
  • resolve_anyelement_from_others 等の関数が、本構造体を引数として受け取り、他の既知の型情報から不足している型を埋める処理を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/fmgr/funcapi.c 内の関数（例: resolve_anyelement_from_others 等の解決関数や、型解決を行う上位関数）でローカル変数としてインスタンス化されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Oid: PostgreSQLの型識別子。
  • pg_type: 本構造体が保持する型情報の参照先となるシステムカタログ。

失敗モードと境界条件

  • 呼び出し側のコンテキストで多相型の解決が不可能な場合、各フィールドは InvalidOid のままとなります。
  • 解決関数が本構造体を適切に初期化せずに参照した場合、未定義の型情報が使用されるリスクがあります。

保守担当者向けメモ

  • 本構造体は funcapi.c 内の多相型解決ロジックに特化しています。
  • 新たな多相型が PostgreSQL に追加された場合、本構造体にフィールドを追加する必要があります。
