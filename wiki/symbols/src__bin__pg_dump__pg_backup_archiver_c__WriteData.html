WriteData

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:1218

目的

アーカイブ処理において、ダンプ対象のデータブロックを現在のアーカイブ形式に応じた出力先へ書き出す。

シグネチャまたは形状

void WriteData(Archive *AHX, const void *data, size_t dLen)

入力と出力

  • AHX: アーカイブハンドル（Archive 型）。内部で ArchiveHandle にキャストされる。
  • data: 書き出すデータバッファへのポインタ。
  • dLen: 書き出すデータのバイト数。
  • 出力: なし（戻り値は void）。

前提条件と不変条件

  • AH->currToc が非 NULL であること。この条件を満たさない場合、DataDumper ルーチンのコンテキスト外で呼び出されたとみなされ、pg_fatal によりプログラムが終了する。

アルゴリズムまたは振る舞い

1. 引数 AHX を ArchiveHandle *AH にキャストする。 2. AH->currToc が設定されているか検証し、未設定であればエラーを報告する。 3. AH->WriteDataPtr 関数ポインタを介して、現在のアーカイブ形式（カスタム、ディレクトリ、tar等）固有の書き込み処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c 内の各処理（1652, 1685 行付近）。
  • src/bin/pg_dump/pg_dump.c（2399, 4113 行付近）。
  • **呼び出し先**:
  • AH->WriteDataPtr: アーカイブ形式ごとに定義された関数（pg_backup_custom.c, pg_backup_directory.c 等で実装）。

関連データ構造

  • ArchiveHandle: アーカイブの状態を保持するメイン構造体。WriteDataPtr 関数ポインタを保持する。
  • TocEntry: currToc として参照される、現在処理中のオブジェクトメタデータ。

失敗モードと境界条件

  • currToc が NULL の場合: pg_fatal により強制終了する。
  • WriteDataPtr 内部での書き込みエラー: 各アーカイブ形式の実装に依存するが、通常は pg_fatal 等で異常終了する。

保守担当者向けメモ

  • 本関数は抽象化レイヤーであり、実際の書き込みロジックは AH->WriteDataPtr に委譲される。特定のフォーマットで書き込み動作を追跡する場合は、_allocAH でどの関数が WriteDataPtr に割り当てられているかを確認すること。
