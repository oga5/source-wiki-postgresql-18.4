hashbeginscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:373

目的

ハッシュインデックスに対するスキャン操作を開始し、スキャン状態を初期化します。

シグネチャまたは形状

IndexScanDesc hashbeginscan(Relation rel, int nkeys, int norderbys)

入力と出力

  • **入力**:
  • rel: スキャン対象のインデックスリレーション。
  • nkeys: スキャンで使用する検索キーの数。
  • norderbys: ORDER BY 句の数。
  • **出力**:
  • 初期化された IndexScanDesc 構造体。

前提条件と不変条件

  • norderbys は 0 である必要があります（ハッシュインデックスは順序付けをサポートしないため、Assert で検証されます）。

アルゴリズムまたは振る舞い

1. RelationGetIndexScan を呼び出し、標準的なインデックススキャン記述子を生成します。 2. HashScanOpaqueData 構造体をメモリ確保し、スキャン固有の状態を初期化します。

  • バケットバッファ (hashso_bucket_buf, hashso_split_bucket_buf) を InvalidBuffer に設定。
  • バケットのステータスフラグ (hashso_buc_populated, hashso_buc_split) を false に設定。
  • 削除対象タプル管理用の killedItems を NULL、numKilled を 0 に初期化。

    • 3. 生成した不透明な状態構造体を scan->opaque に格納し、スキャン記述子を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hash.c 内の IndexAmRoutine 定義において、ambeginscan コールバックとして登録されています。
  • **呼び出し先**:
  • RelationGetIndexScan（PostgreSQL のインデックス AM API）
  • palloc（メモリ割り当て）

関連データ構造

  • IndexScanDesc: PostgreSQL の標準的なインデックススキャン記述子。
  • HashScanOpaqueData: ハッシュインデックス固有のスキャン状態を保持する構造体。

失敗モードと境界条件

  • norderbys != 0 の場合、アサーション違反によりプロセスが停止します。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数はハッシュインデックススキャンの初期化のみを担当します。実際の検索ロジックは hashgettuple および hashsearch.c 内の関数群に委譲されます。
  • HashScanOpaque の各フィールドは、スキャン中のバケットチェーンの追跡や、並行する VACUUM との整合性維持に不可欠です。変更を加える際は hashgettuple 側の参照整合性に注意してください。
