query_requires_rewrite_plan

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:519

目的

与えられた Query オブジェクトが、書き換え（rewrite）およびプランニング（planning）の処理を必要とするかどうかを判定する。

シグネチャまたは形状

bool query_requires_rewrite_plan(Query *query)

入力と出力

  • **入力**: Query *query - 解析済みのクエリツリー。
  • **出力**: bool - 書き換えとプランニングが必要な場合は true、不要な場合は false を返す。

前提条件と不変条件

  • 入力される query は解析済みであること。
  • CMD_UTILITY 以外のコマンド（最適化可能な文）は、常に書き換えとプランニングが必要とみなされる。

アルゴリズムまたは振る舞い

1. query->commandType が CMD_UTILITY でない場合、最適化可能な文と判断し true を返す。 2. CMD_UTILITY の場合、query->utilityStmt のノードタグを評価する。

  • T_DeclareCursorStmt, T_ExplainStmt, T_CreateTableAsStmt, T_CallStmt のいずれかであれば true を返す。
  • それ以外の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/plancache.c (複数箇所)
  • **呼び出し先**:
  • nodeTag() (マクロ)

関連データ構造

  • Query: PostgreSQL のクエリツリー構造体。
  • NodeTag: utilityStmt の種類を識別するために使用される。

失敗モードと境界条件

  • query が NULL の場合、query->commandType へのアクセスによりクラッシュする可能性がある（ソースコード上に明示的な NULL チェックはない）。

保守担当者向けメモ

  • 本関数内の utilityStmt の判定リストは、stmt_requires_parse_analysis() と一致させる必要がある。
  • 新しいユーティリティコマンドが追加され、それが書き換えやプランニングを必要とする場合は、本関数の switch 文にケースを追加すること。
