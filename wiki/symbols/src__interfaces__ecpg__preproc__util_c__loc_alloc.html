loc_alloc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:137

目的

ECPGプリプロセッサの構文解析フェーズにおいて、一時的なメモリ領域を効率的に確保します。個別の解放を必要とせず、解析サイクル単位での一括解放を前提としたメモリ管理を提供します。

シグネチャまたは形状

void *loc_alloc(size_t size)

入力と出力

  • **入力**: size_t size（確保したいバイト数）
  • **出力**: 確保されたメモリ領域へのポインタ（void *）。確保に失敗した場合は mm_alloc の挙動に従い終了します。

前提条件と不変条件

  • 確保されるメモリは MAXALIGN によってアライメントが調整されます。
  • 確保されたメモリは、呼び出し側で個別に free してはなりません。
  • 解析サイクルの終了時に reclaim_local_storage() が呼び出され、確保された全メモリが解放される必要があります。

アルゴリズムまたは振る舞い

1. 要求された size をアライメント調整します。 2. 現在のチャンク（loc_chunks）に十分な空き容量があるか確認します。 3. 空きがない場合、Max(size, LOC_CHUNK_MIN_SIZE) で新しいチャンクを mm_alloc で確保し、リストの先頭に追加します。 4. 現在のチャンクから要求サイズ分を切り出し、オフセットを更新してポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c
  • src/interfaces/ecpg/preproc/output.c
  • src/interfaces/ecpg/preproc/util.c 内の文字列操作関数群（loc_strdup, cat_str 等）
  • **呼び出し先**:
  • mm_alloc（新規チャンク確保時）

関連データ構造

  • loc_chunk: メモリ管理用のチャンク構造体。data（領域）、chunk_used（使用量）、chunk_avail（残量）、next（次へのポインタ）を保持します。
  • loc_chunks: 確保されたチャンクを管理するグローバルなリストのヘッド。

失敗モードと境界条件

  • LOC_CHUNK_MIN_SIZE（8192バイト）を超える要求があった場合、そのサイズに合わせてチャンクが確保されます。
  • メモリ不足時は mm_alloc 経由で致命的エラーとして処理されます。

保守担当者向けメモ

  • 本関数は loc_chunk リストを介してメモリを管理します。メモリリークを防ぐため、解析の節目で必ず reclaim_local_storage() が実行される設計であることを維持してください。
  • 非常に大きな単一オブジェクトを確保する場合、チャンクの断片化やメモリ効率に注意が必要です。
