fetch_array_arg_replace_nulls

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:80

目的

SQL関数に渡された配列引数を取得し、NULL値の場合は空の展開済み配列（ExpandedArrayHeader）を生成して返すヘルパー関数です。

シグネチャまたは形状

ExpandedArrayHeader *fetch_array_arg_replace_nulls(FunctionCallInfo fcinfo, int argno)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • argno: 取得対象の引数インデックス。
  • **出力**:
  • ExpandedArrayHeader *: 取得または生成された展開済み配列へのポインタ。

前提条件と不変条件

  • fcinfo->flinfo->fn_extra を使用して ArrayMetaState をキャッシュします。初回呼び出し時にメモリコンテキスト（fn_mcxt）上に構造体が割り当てられます。
  • 結果のメモリコンテキストは AggCheckCallContext に基づいて決定されます。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: fn_extra が未初期化の場合、ArrayMetaState を割り当てて初期化します。 2. **コンテキスト決定**: 集約関数として呼び出されているかを確認し、結果のメモリコンテキストを決定します。 3. **引数取得**:

  • 引数がNULLでない場合: PG_GETARG_EXPANDED_ARRAYX を使用して展開済み配列を取得します。
  • 引数がNULLの場合:
  • get_fn_expr_argtype で引数の型を特定します。
  • get_element_type で要素型を特定します。
  • construct_empty_expanded_array を呼び出し、空の配列を生成します。

    • 4. **返却**: 取得または生成した ExpandedArrayHeader を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/array_userfuncs.c 内の配列操作関数（例: array_append や array_prepend 等の内部実装箇所）。
  • **呼び出し先**:
  • MemoryContextAlloc
  • MemoryContextSwitchTo
  • PG_GETARG_EXPANDED_ARRAYX
  • get_fn_expr_argtype
  • get_element_type
  • construct_empty_expanded_array
  • ereport

関連データ構造

  • ExpandedArrayHeader: 展開済み配列のヘッダ構造。
  • ArrayMetaState: 配列のメタデータ（要素型など）を保持するキャッシュ構造体。
  • FunctionCallInfo: SQL関数呼び出しのコンテキスト情報。

失敗モードと境界条件

  • **型不整合**: get_fn_expr_argtype で型が特定できない場合、または get_element_type で配列型ではないと判断された場合、ERROR をスローします。

保守担当者向けメモ

  • 本関数は fn_extra を利用したキャッシュ戦略を採用しており、同一クエリ内での繰り返し呼び出しにおけるカタログ検索コストを削減しています。
  • 集約関数内での利用を想定したメモリ管理（AggCheckCallContext）が行われているため、呼び出し元のコンテキストに注意してください。
