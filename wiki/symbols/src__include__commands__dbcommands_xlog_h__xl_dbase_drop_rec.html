xl_dbase_drop_rec

定義

  • 種別: struct
  • 定義箇所: src/include/commands/dbcommands_xlog.h:48

目的

データベース削除操作を WAL（Write-Ahead Logging）に記録するための構造体です。クラッシュリカバリやレプリケーション時に、削除対象のデータベースに関連するテーブルスペース情報を特定し、物理的な削除処理を再実行するために使用されます。

シグネチャまたは形状

typedef struct xl_dbase_drop_rec
{
    Oid         db_id;
    int         ntablespaces;
    Oid         tablespace_ids[FLEXIBLE_ARRAY_MEMBER];
} xl_dbase_drop_rec;

入力と出力

  • **入力**: データベース削除時に dbcommands.c 等で本構造体に db_id（データベースOID）および ntablespaces（テーブルスペース数）、tablespace_ids（テーブルスペースOIDの配列）が格納されます。
  • **出力**: WAL レコードとして書き込まれ、リカバリ時に XLogRecGetData を通じて読み出されます。

前提条件と不変条件

  • tablespace_ids は FLEXIBLE_ARRAY_MEMBER を使用した可変長配列です。
  • 構造体のサイズ計算には、MinSizeOfDbaseDropRec（offsetof(xl_dbase_drop_rec, tablespace_ids)）を使用する必要があります。

アルゴリズムまたは振る舞い

1. **書き込み**: データベース削除コマンド実行時、関連するテーブルスペースの ID を本構造体に詰め、XLogRegisterData を介して WAL に記録します。 2. **読み込み**: リカバリプロセスにおいて、WAL レコードから本構造体へキャストしてデータを復元します。 3. **適用**: 復元された db_id と tablespace_ids を基に、ファイルシステム上の該当ディレクトリやファイルを削除する処理（dbase_redo）が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（書き込み側）**: src/backend/commands/dbcommands.c
  • **呼び出し先（読み込み側）**:
  • src/backend/access/rmgrdesc/dbasedesc.c（ログのデバッグ出力）
  • src/backend/commands/dbcommands.c（リカバリ時のリプレイ処理）
  • src/backend/postmaster/walsummarizer.c（WAL サマライズ処理）

関連データ構造

  • MinSizeOfDbaseDropRec: 本構造体の最小サイズを定義するマクロ。

失敗モードと境界条件

  • 可変長構造体であるため、メモリ確保やポインタ演算時に ntablespaces の値に基づいた適切なサイズ計算を行わない場合、バッファオーバーランや不正なメモリ参照が発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を含んでいるため、構造体自体のサイズ（sizeof）ではなく、MinSizeOfDbaseDropRec を基準とした動的なサイズ計算を徹底してください。
  • データベース削除の WAL リプレイロジック（dbase_redo）を変更する際は、本構造体に含まれるテーブルスペース ID の整合性が維持されていることを確認してください。
