gather_merge_init

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:442

目的

GatherMerge ノードの実行開始時に、各ワーカーおよびリーダープロセスから最初のタプルを取得し、バイナリヒープを構築してマージ処理の準備を行う。

シグネチャまたは形状

static void gather_merge_init(GatherMergeState *gm_state)

入力と出力

  • **入力**: GatherMergeState *gm_state（実行状態を保持する構造体）
  • **出力**: なし（副作用として gm_state 内のヒープおよびバッファを初期化する）

前提条件と不変条件

  • gather_merge_setup が呼び出され、必要なメモリ空間とスロットが確保されていること。
  • gm_state->nreaders が GatherMerge プランノードの num_workers 以下であること。

アルゴリズムまたは振る舞い

1. **状態のリセット**: リーダーおよび各ワーカーのタプルスロット、バッファ（nTuples, readCounter, done フラグ）を初期化する。 2. **ヒープの初期化**: binaryheap_reset を呼び出し、ヒープを空にする。 3. **タプルの読み込み**:

  • 各ソースに対して nowait モードで gather_merge_readnext を試行し、タプルが取得できた場合はヒープに追加する。
  • すべてのソースからタプルまたは終了通知が得られるまで、wait モードに切り替えて再試行（reread ループ）を行う。

    • 4. **ヒープの構築**: binaryheap_build を呼び出し、ヒープ構造を確定させる。 5. **完了フラグ**: gm_state->gm_initialized を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecGatherMerge（ソースコード内の文脈より）
  • **呼び出し先**: gather_merge_readnext, load_tuple_array, binaryheap_reset, binaryheap_add_unordered, binaryheap_build, ExecClearTuple, CHECK_FOR_INTERRUPTS

関連データ構造

  • GatherMergeState: 実行状態を管理するメイン構造体。
  • GMReaderTupleBuffer: 各ワーカーからのタプルを保持するバッファ。
  • binaryheap: ソート済みのタプルをマージするための優先度付きキュー。

失敗モードと境界条件

  • CHECK_FOR_INTERRUPTS() により、割り込み発生時に処理が中断される可能性がある。
  • ワーカーがタプルを生成しない場合や、すべてのワーカーが終了している場合でも、ヒープの構築と初期化は正常に完了する。

保守担当者向けメモ

  • 本関数は GatherMerge の初期化フェーズの核心であり、gm_initialized フラグによって一度だけ実行されることが想定されている。
  • reread ループは、並列ワーカーからのタプル到着待ちと、リーダーのローカルスキャンを同期させるための重要なロジックである。
