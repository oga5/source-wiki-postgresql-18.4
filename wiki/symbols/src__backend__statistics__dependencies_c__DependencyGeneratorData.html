DependencyGeneratorData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/statistics/dependencies.c:56

目的

関数従属性（Functional Dependencies）の計算プロセスにおいて、属性の組み合わせを効率的に生成・管理するための状態保持構造体です。

シグネチャまたは形状

typedef struct DependencyGeneratorData
{
	int			k;				/* 従属性のサイズ */
	int			n;				/* 可能な属性の数 */
	int			current;		/* 次に返す従属性のインデックス */
	AttrNumber	ndependencies;	/* 生成された従属性の総数 */
	AttrNumber *dependencies;	/* 事前に生成された従属性の配列 */
} DependencyGeneratorData;

入力と出力

  • **入力**: statext_dependencies_build 等の統計構築プロセスにおいて、メモリ確保時に sizeof(DependencyGeneratorData) が使用されます。
  • **出力**: 内部の dependencies 配列を通じて、生成された属性の組み合わせを順次提供します。

前提条件と不変条件

  • dependencies は、生成された従属性を保持する動的に割り当てられた配列である必要があります。
  • current は、ndependencies 未満の範囲でインクリメントされ、次の要素を指し示します。

アルゴリズムまたは振る舞い

  • ANALYZE 実行時に、列の組み合わせを網羅的に生成するためのイテレータとして機能します。
  • DependencyGenerator_init（推測される初期化処理）で設定された k（サイズ）と n（属性数）に基づき、DependencyGenerator_next を介して属性の組み合わせを反復的に提供します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/dependencies.c 内の statext_dependencies_build が、この構造体を palloc0 で割り当てて使用しています。
  • **呼び出し先**: 直接的な呼び出し先情報は提供されていませんが、DependencyGenerator 型（本構造体へのポインタ）として、生成ロジック全体で状態管理に使用されます。

関連データ構造

  • DependencyGenerator: 本構造体へのポインタ型 (typedef DependencyGeneratorData *DependencyGenerator)。

失敗モードと境界条件

  • メモリ不足時: palloc0 による割り当て失敗が発生する可能性があります。
  • 範囲外アクセス: current が ndependencies を超えないよう、生成ロジック側で制御する必要があります。

保守担当者向けメモ

  • 本構造体は dependencies.c 内の統計構築ロジックに特化しています。
  • 属性数に対して指数関数的に計算量が増加するため、k や n の値が極端に大きい場合のメモリ消費量に注意してください。
