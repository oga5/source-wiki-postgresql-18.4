boolor_statefunc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:311

目的

SQLの集約関数 bool_or の状態遷移を計算します。現在の集約状態と次の入力値の論理和（OR）を求めます。

シグネチャまたは形状

Datum boolor_statefunc(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの bool 値を受け取ります。
  • 第1引数: 現在の集約状態（bool）。
  • 第2引数: 新たに追加される値（bool）。
  • **出力**: Datum 型として、論理和の結果（bool）を返します。

前提条件と不変条件

  • PG_GETARG_BOOL マクロを使用して引数を取得するため、呼び出し元はPostgreSQLの関数呼び出し規約に従う必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_BOOL(0) で現在の状態を取得します。 2. PG_GETARG_BOOL(1) で次の入力値を取得します。 3. C言語の論理和演算子 || を適用し、結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLの集約関数 bool_or の実行時に、PostgreSQLの関数マネージャを介して呼び出されます。
  • **呼び出し先**: 内部的に PG_GETARG_BOOL および PG_RETURN_BOOL マクロを使用します。

関連データ構造

  • bool (C言語の論理型)
  • Datum (PostgreSQLの汎用データ型)

失敗モードと境界条件

  • 引数がNULLの場合の挙動は、PostgreSQLの関数呼び出し規約および集約関数のNULLハンドリング設定に依存します。

保守担当者向けメモ

  • 本関数は単純な論理和を計算するのみであり、状態の初期化や最終的な集約値の算出は別の関数（bool_accum 等）が担います。
