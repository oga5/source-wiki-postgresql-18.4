IspellDict

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/dicts/spell.h:184

目的

ISpell および Hunspell 形式の辞書データを保持し、辞書の構築から検索時の単語正規化までの一連のライフサイクルを管理するメイン構造体です。

シグネチャまたは形状

typedef struct {
    int maffixes;
    int naffixes;
    AFFIX *Affix;
    AffixNode *Suffix;
    AffixNode *Prefix;
    SPNode *Dictionary;
    const char **AffixData;
    int lenAffixData;
    int nAffixData;
    bool useFlagAliases;
    CMPDAffix *CompoundAffix;
    bool usecompound;
    FlagMode flagMode;
    CompoundAffixFlag *CompoundAffixFlags;
    int nCompoundAffixFlag;
    int mCompoundAffixFlag;
    MemoryContext buildCxt;
    SPELL **Spell;
    int nspell;
    int mspell;
    char *firstfree;
    size_t avail;
} IspellDict;

入力と出力

  • **入力**: 辞書構築時には NIStartBuild を介して初期化され、NIAddSpell 等で単語データが投入されます。
  • **出力**: 構築完了後、Dictionary（Trie構造）や接辞データが検索エンジンによって参照され、単語の正規化結果が出力されます。

前提条件と不変条件

  • 辞書構築に使用するメモリは buildCxt に依存します。
  • 構築プロセス終了時には NIFinishBuild を呼び出し、構築専用のフィールド（Spell 配列や一時メモリ）を解放する必要があります。
  • 構造体は割り当て時にゼロクリアされていることが前提となります。

アルゴリズムまたは振る舞い

1. **構築フェーズ**: NIStartBuild でメモリコンテキストを確保し、NIAddSpell 等でデータを読み込みます。 2. **最適化フェーズ**: リスト形式のデータを NISortDictionary 等で Trie 構造に変換し、検索効率を最大化します。 3. **実行フェーズ**: 構築済みの Trie を参照して単語の正規化を行います。 4. **クリーンアップ**: NIFinishBuild により、構築時にのみ必要な一時領域を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/dict_ispell.c および src/backend/tsearch/spell.c 内の各関数。
  • **呼び出し先**: 本構造体はデータ保持用であり、NIStartBuild, NIFinishBuild, NIAddSpell, compact_palloc0, lowerstr_ctx 等の関数によって操作されます。

関連データ構造

  • SPNode, AffixNode: 辞書検索用の Trie ノード。
  • SPELL: 辞書内の単語データ。
  • AFFIX: 接辞ルール。
  • CMPDAffix: 複合語ルール。
  • MemoryContext: 構築時のメモリ管理用。

失敗モードと境界条件

  • 構築中のメモリ不足は buildCxt の管理下で発生します。
  • FLEXIBLE_ARRAY_MEMBER を含む構造体が含まれるため、サイズ計算には SPNHDRSZ 等のマクロを使用する必要があります。
  • NIFinishBuild を呼び出さない場合、構築用の一時メモリがリークします。

保守担当者向けメモ

  • 本構造体は構築時と実行時で役割が明確に分かれています。コード修正時は「現在どのフェーズか」を意識してください。
  • Spell 配列や buildCxt などのフィールドは構築後に不要となるため、実行時のメモリフットプリントを考慮した設計になっています。
