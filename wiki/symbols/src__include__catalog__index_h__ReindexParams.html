ReindexParams

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/index.h:33

目的

REINDEX コマンドの実行時における動作オプションおよびターゲット設定を保持するための構造体です。

シグネチャまたは形状

typedef struct ReindexParams
{
    bits32      options;        /* REINDEXOPT_* ビットマスク */
    Oid         tablespaceOid;  /* インデックス移動先のテーブルスペースOID */
} ReindexParams;

入力と出力

  • 入力: REINDEX 文の解析結果に基づき、コマンドオプション（options）と移動先テーブルスペース（tablespaceOid）が設定されます。
  • 出力: インデックス再構築処理を行う各関数へ引数として渡され、処理の挙動を制御します。

前提条件と不変条件

  • tablespaceOid に InvalidOid が指定された場合、テーブルスペースの変更は行われません。
  • options は REINDEXOPT_* 定数群のビットマスクである必要があります。

アルゴリズムまたは振る舞い

  • REINDEX コマンド実行時、この構造体に格納されたフラグに基づいて、インデックスの再構築プロセス（並行処理の有無やテーブルスペースの変更など）が分岐します。
  • src/backend/catalog/index.c や src/backend/commands/indexcmds.c 内の関数において、再構築の実行パラメータとして参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/indexcmds.c（ReindexIndex, ReindexTable 等）および src/backend/commands/cluster.c でインスタンス化されます。
  • 呼び出し先: src/backend/catalog/index.c 内のインデックス再構築関連関数（index_concurrently_build 等）に引数として渡されます。

関連データ構造

  • ReindexStmt: SQL 文の解析結果を保持するノード。
  • IndexInfo: インデックスの定義情報を保持する構造体。

失敗モードと境界条件

  • tablespaceOid に存在しない OID を指定した場合、後続のカタログ操作や物理ファイル作成時にエラーが発生する可能性があります。
  • options に無効なビットフラグが含まれる場合の挙動は、各処理関数の実装に依存します。

保守担当者向けメモ

  • 本構造体は REINDEX の仕様変更に伴いフィールドが追加される可能性があるため、変更時は src/backend/commands/indexcmds.c での初期化箇所を確認してください。
  • tablespaceOid の取り扱いについては、InvalidOid が「変更なし」を意味する点に注意してください。
