varbit_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:451

目的

SQL文字列（バイナリ表現または16進数表現）を、PostgreSQLの内部データ構造である VarBit 型へ変換します。

シグネチャまたは形状

Datum varbit_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_CSTRING(0): 変換対象の文字列（'b'または'x'で始まる、あるいは接頭辞なしのビット列）。
  • PG_GETARG_INT32(2): atttypmod（ビット長制限）。
  • **出力**:
  • Datum: 変換された VarBit 構造体へのポインタ。

前提条件と不変条件

  • VarBit 構造体の最後のバイトの未使用ビットは、常にゼロでなければなりません。
  • 入力文字列が16進数形式の場合、その長さは VARBITMAXLEN / 4 を超えてはなりません。
  • atttypmod が指定されている場合、入力されたビット長はこれを超えてはなりません（超える場合はエラー）。

アルゴリズムまたは振る舞い

1. **接頭辞解析**: 文字列の先頭文字を確認し、バイナリ形式（'b'）か16進数形式（'x'）かを判定します。 2. **長さ計算**: 形式に応じて必要なビット長を算出します。 3. **メモリ確保**: palloc0 を使用して、必要なサイズの VarBit 構造体をゼロ初期化して確保します。 4. **変換処理**:

  • **バイナリ形式**: 各文字を走査し、'1'であれば対応するビットを立てます。
  • **16進数形式**: 各文字を4ビットの値に変換し、バイト単位で詰め込みます。

    • 5. **パディング**: palloc0 により、未使用ビットは自動的にゼロクリアされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gin/btree_gin.c 内で DirectFunctionCall3 を介して呼び出される事例があります。
  • **呼び出し先**:
  • palloc0（メモリ確保）
  • ereturn（エラー報告）
  • pg_mblen_cstr（マルチバイト文字長確認）

関連データ構造

  • VarBit: ビット列を保持する構造体。vl_len_（ヘッダ）、bit_len（ビット長）、bit_dat（データ）で構成されます。

失敗モードと境界条件

  • **ERRCODE_PROGRAM_LIMIT_EXCEEDED**: 16進数入力が VARBITMAXLEN を超える場合。
  • **ERRCODE_STRING_DATA_RIGHT_TRUNCATION**: 入力ビット長が atttypmod を超える場合。
  • **ERRCODE_INVALID_TEXT_REPRESENTATION**: 不正なバイナリ文字（'0', '1'以外）や16進数文字が含まれる場合。

保守担当者向けメモ

  • VarBit のデータ領域は最上位バイト（MSB）が先頭に配置されます。ビット操作を行う際はバイト境界を跨ぐシフト処理に注意してください。
  • varbit_recv() との整合性を維持するため、パディングのゼロクリア不変条件を常に遵守してください。
