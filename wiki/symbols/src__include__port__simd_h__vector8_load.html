vector8_load

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/port/simd.h:107

目的

メモリ上の連続したバイト列から Vector8 型の変数へデータをロードする。プラットフォーム固有の SIMD 命令または標準的なメモリコピーを透過的に選択する抽象化レイヤーである。

シグネチャまたは形状

static inline void vector8_load(Vector8 *v, const uint8 *s)

入力と出力

  • v (出力): ロード先の Vector8 型変数へのポインタ。
  • s (入力): ロード元の uint8 型配列の先頭ポインタ。
  • 戻り値: なし（v が指すメモリ領域が更新される）。

前提条件と不変条件

  • s が指すメモリ領域は、少なくとも sizeof(Vector8) バイト以上の有効なデータを含んでいる必要がある。
  • Vector8 のサイズはプラットフォーム依存である。

アルゴリズムまたは振る舞い

コンパイル時の機能フラグに基づき、以下のいずれかの命令を実行する。 1. USE_SSE2 が定義されている場合: _mm_loadu_si128 を使用し、非整列アクセスでロードする。 2. USE_NEON が定義されている場合: vld1q_u8 を使用してロードする。 3. 上記以外の場合: memcpy を使用して sizeof(Vector8) バイトをコピーする。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • src/backend/utils/adt/json.c
  • src/include/lib/radixtree.h
  • src/include/port/pg_lfind.h
  • src/include/utils/ascii.h
  • 呼び出し先:
  • プラットフォーム固有の組み込み関数（_mm_loadu_si128 または vld1q_u8）または標準ライブラリの memcpy。

関連データ構造

  • Vector8: SIMD 演算用の抽象型。プラットフォームに応じて __m128i や uint8x16_t 等にマッピングされる。

失敗モードと境界条件

  • s が指す領域が sizeof(Vector8) 未満である場合、バッファオーバーリードが発生する。
  • メモリ配置が Vector8 のアライメント要件を満たさない場合、アーキテクチャによっては性能低下や例外が発生する可能性がある（ただし、SSE2 の _mm_loadu_si128 は非整列アクセスを許容する）。

保守担当者向けメモ

  • 本関数はインライン展開を前提としている。
  • SIMD 非対応環境でのフォールバックとして memcpy を使用しているため、コンパイラの最適化に依存する。
  • Vector8 のサイズ変更や新しいアーキテクチャの追加時には、本関数の #if 分岐を更新する必要がある。
