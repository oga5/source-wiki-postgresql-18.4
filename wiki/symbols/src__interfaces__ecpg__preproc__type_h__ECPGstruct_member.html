ECPGstruct_member

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/preproc/type.h:10

目的

C言語の構造体や共用体のメンバー情報を保持し、ECPGプリプロセッサがホスト変数の型情報を解析・管理するための連結リスト構造を提供します。

シグネチャまたは形状

struct ECPGstruct_member
{
    char *name;
    struct ECPGtype *type;
    struct ECPGstruct_member *next;
};

入力と出力

  • **入力**: メンバー名（name）、型情報（type）、およびリストの次要素へのポインタ（next）。
  • **出力**: 構造体メンバーのリスト構造。

前提条件と不変条件

  • type は ECPGtype 構造体へのポインタであり、有効な型情報である必要があります。
  • リストの終端は next が NULL であることで示されます。
  • メモリは mm_alloc 等で動的に確保されることが前提であり、使用後は ECPGfree_struct_member による解放が必要です。

アルゴリズムまたは振る舞い

  • 構造体メンバーの連結リストを形成し、再帰的な型定義（構造体の中に構造体が含まれる場合など）を表現します。
  • ECPGmake_struct_member 関数により、新しいメンバーがリストの先頭に追加されます。
  • ECPGstruct_member_dup により、既存のメンバーリストの複製が可能です。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/preproc/preproc_extern.h: struct_member_list 配列の要素として使用。
  • src/interfaces/ecpg/preproc/type.c: ECPGmake_struct_member, ECPGmake_struct_type, ECPGfree_struct_member 等で操作。
  • **呼び出し先**:
  • struct ECPGtype（型情報の保持のため）。

関連データ構造

  • ECPGtype: メンバーの型定義を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: mm_alloc が失敗した場合の挙動は呼び出し側の実装に依存します。
  • リスト操作: next ポインタの不適切な管理はメモリリークやセグメンテーション違反を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は再帰的な型定義をサポートしているため、リストを走査する際は無限ループに陥らないよう注意してください。
  • type.c にある ECPGfree_struct_member を使用して、再帰的にメモリを解放する設計になっています。
