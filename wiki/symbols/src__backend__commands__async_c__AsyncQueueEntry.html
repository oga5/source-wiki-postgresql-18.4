AsyncQueueEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/async.c:177

目的

非同期通知（NOTIFY）の各メッセージを共有メモリ上のSLRUキュー内で表現するためのデータ構造です。

シグネチャまたは形状

typedef struct AsyncQueueEntry
{
    int         length;         /* エントリの合計確保長 */
    Oid         dboid;          /* 送信元のデータベースOID */
    TransactionId xid;          /* 送信元のXID */
    int32       srcPid;         /* 送信元のPID */
    char        data[NAMEDATALEN + NOTIFY_PAYLOAD_MAX_LENGTH];
} AsyncQueueEntry;

入力と出力

  • 入力: asyncQueueNotificationToEntry 関数により、Notification 構造体から本構造体へデータがコピーされます。
  • 出力: キューから読み出された後、リスナーバックエンドによって解析され、フロントエンドへの通知メッセージとして利用されます。

前提条件と不変条件

  • data フィールドには、チャネル名とペイロードが含まれます。
  • 構造体内の各フィールドは、int アラインメント要件を満たす必要があります。

アルゴリズムまたは振る舞い

  • トランザクションコミット時に、保留中の通知が AsyncQueueEntry 形式に変換され、共有メモリ上のSLRUキューへ書き込まれます。
  • 受信側バックエンドは、キュー内のエントリをスキャンし、自身のデータベースIDと一致し、かつ購読中のチャネルである場合にのみ処理を行います。

呼び出し元と呼び出し先

  • 呼び出し元: asyncQueueNotificationToEntry が本構造体を初期化します。また、キューのスキャン処理においてポインタ経由で参照されます。
  • 呼び出し先: 直接的な関数呼び出しは行いません。データコンテナとして機能します。

関連データ構造

  • Notification: 送信元のトランザクション内で一時的に保持される通知情報。
  • AsyncQueueControl: キュー全体の管理構造体。
  • SLRU: 通知キューの物理的な格納領域。

失敗モードと境界条件

  • data フィールドのサイズ制限（NAMEDATALEN + NOTIFY_PAYLOAD_MAX_LENGTH）を超える通知は格納できません。
  • キューのテール位置が適切に進まない場合、ディスク領域を消費し続ける可能性があります。

保守担当者向けメモ

  • AsyncQueueEntryEmptySize マクロが定義されており、データ部を除いた最小サイズ計算に使用されます。
  • 構造体のレイアウト変更は、共有メモリ上のデータ構造と直接整合するため、慎重に行う必要があります。
