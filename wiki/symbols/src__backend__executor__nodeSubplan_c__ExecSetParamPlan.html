ExecSetParamPlan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:1083

目的

InitPlan（クエリ実行中に一度だけ評価されるサブクエリ）を実行し、その結果を親プランのパラメータ（ParamExecData）に格納する。

シグネチャまたは形状

void ExecSetParamPlan(SubPlanState *node, ExprContext *econtext)

入力と出力

  • **入力**:
  • node: 実行対象の SubPlanState。
  • econtext: 実行コンテキスト。ecxt_param_exec_vals を通じてパラメータ値を更新する。
  • **出力**: なし（副作用として econtext 内のパラメータ値を更新）。

前提条件と不変条件

  • ANY_SUBLINK, ALL_SUBLINK, CTE_SUBLINK はサポート対象外であり、呼び出された場合はエラーとなる。
  • 相関サブクエリ（parParam または args を持つもの）は実行不可。
  • 実行時のスキャン方向は強制的に ForwardScanDirection に設定される。

アルゴリズムまたは振る舞い

1. **初期化**: 実行コンテキストをクエリ単位のメモリコンテキストに切り替える。 2. **プラン実行**: ExecProcNode を用いてサブプランを反復実行する。 3. **型別の処理**:

  • EXISTS_SUBLINK: 結果の有無に基づき true/false をパラメータに設定。
  • ARRAY_SUBLINK: 全行を収集し、配列としてパラメータに設定。
  • その他（EXPR_SUBLINK 等）: 最初の行を抽出し、各列の値を対応するパラメータにセットする。複数行が返された場合はエラー（ERRCODE_CARDINALITY_VIOLATION）を報告する。

    • 4. **終了処理**: 結果が得られなかった場合、EXISTS なら false を、それ以外ならパラメータを NULL に設定する。最後にメモリコンテキストとスキャン方向を復元する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c
  • src/backend/executor/nodeSubplan.c（ExecSetParamPlan 自身および関連する再スキャン処理）
  • **呼び出し先**:
  • ExecProcNode（プランの実行）
  • MemoryContextSwitchTo（メモリ管理）
  • heap_getattr / slot_getattr（タプルからの値抽出）
  • initArrayResultAny / accumArrayResultAny / makeArrayResultAny（配列生成）

関連データ構造

  • SubPlanState: サブプランの実行状態を保持。
  • ExprContext: パラメータ値（ecxt_param_exec_vals）を保持する実行コンテキスト。
  • ParamExecData: パラメータの値を格納する構造体。

失敗モードと境界条件

  • **エラー**:
  • サポート外の SubLinkType が渡された場合。
  • 相関サブクエリが渡された場合。
  • 単一値を期待するサブクエリが複数行を返した場合。
  • **境界条件**:
  • サブクエリが0行を返した場合、パラメータは NULL または false に設定される。

保守担当者向けメモ

  • 本関数は InitPlan の評価専用である。通常のサブクエリ実行には ExecSubPlan を使用すること。
  • node->curTuple や node->curArray を使用して、パラメータが参照する値をメモリコンテキストの切り替えを跨いで保持している点に注意。
