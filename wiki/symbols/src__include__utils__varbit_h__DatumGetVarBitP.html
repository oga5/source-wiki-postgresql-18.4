DatumGetVarBitP

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/varbit.h:44

目的

Datum 型として渡されたビット列データを、PostgreSQLの内部表現である VarBit 構造体ポインタへ変換（detoast）します。

シグネチャまたは形状

static inline VarBit *DatumGetVarBitP(Datum X)

入力と出力

  • **入力**: Datum X（ビット列データを含む Datum）
  • **出力**: VarBit *（detoast されたビット列構造体へのポインタ）

前提条件と不変条件

  • 入力 X は、有効なビット列データ（BIT または BIT VARYING）を指している必要があります。
  • PG_DETOAST_DATUM を使用しているため、入力が TOAST 圧縮されている場合は自動的に伸長されます。

アルゴリズムまたは振る舞い

1. 引数 X を PG_DETOAST_DATUM マクロに渡します。 2. 戻り値を VarBit * 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/varbit.h:62 にて定義されるマクロ PG_GETARG_VARBIT_P(n) から呼び出されます。
  • **呼び出し先**:
  • fmgr.h で定義される PG_DETOAST_DATUM を呼び出します。

関連データ構造

  • VarBit: ビット列を保持する構造体。vl_len_（varlenaヘッダー）、bit_len（有効ビット数）、bit_dat（データ本体）で構成されます。

失敗モードと境界条件

  • 入力が有効なビット列データでない場合、PG_DETOAST_DATUM の仕様に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は varlena 構造体と互換性のある VarBit へのアクセスを抽象化するものです。直接メモリを操作する際は、必ず VARBITLEN や VARBITBYTES などの専用マクロを併用してください。
