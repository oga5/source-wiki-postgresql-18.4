WalSummarizerMain

定義

  • **種別**: function
  • **定義箇所**: src/backend/postmaster/walsummarizer.c:213

目的

WAL Summarizer プロセスのメインループを実行し、WAL レコードをスキャンして変更ブロック情報を要約し、サマリファイルを生成・管理する。

シグネチャまたは形状

void WalSummarizerMain(const void *startup_data, size_t startup_data_len)

入力と出力

  • **入力**: startup_data（未使用、長さ0である必要がある）、startup_data_len。
  • **出力**: なし（プロセス終了時に proc_exit を呼び出す）。

前提条件と不変条件

  • startup_data_len は 0 であること。
  • プロセスは B_WAL_SUMMARIZER として初期化される。
  • sigsetjmp によるエラーハンドリング環境が構築され、エラー発生時はリソース解放後に10秒間待機して再開する。

アルゴリズムまたは振る舞い

1. **初期化**: シグナルハンドラの設定、共有メモリへのプロセス登録、メモリコンテキストの作成を行う。 2. **状態復旧**: GetOldestUnsummarizedLSN を呼び出し、要約開始位置を取得する。要約が無効な場合は終了する。 3. **メインループ**:

  • ProcessWalSummarizerInterrupts でシグナル処理を行う。
  • MaybeRemoveOldWalSummaries で古いサマリファイルを削除する。
  • GetLatestLSN で要約可能な最新位置を確認する。
  • タイムライン切り替えが必要な場合、readTimeLineHistory と tliSwitchPoint を使用して切り替えポイントを計算する。
  • 以降、WAL の読み込みと要約処理（ループ継続）へ移行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c（B_WAL_SUMMARIZER プロセス起動時）。
  • **呼び出し先**: AuxiliaryProcessMainCommon, GetOldestUnsummarizedLSN, ProcessWalSummarizerInterrupts, MaybeRemoveOldWalSummaries, GetLatestLSN, readTimeLineHistory, tliSwitchPoint 等。

関連データ構造

  • WalSummarizerCtl（共有メモリ上の制御構造体）
  • sigjmp_buf（エラー復旧用）
  • MemoryContext（メモリ管理用）

失敗モードと境界条件

  • **エラー発生時**: sigsetjmp により捕捉され、リソースを解放した後に 10 秒間待機（WaitLatch）してから処理を再開する。
  • **WAL 要約無効時**: GetOldestUnsummarizedLSN が無効な LSN を返した場合、プロセスは即座に終了する。
  • **シャットダウン**: SIGTERM 等のシグナルにより終了処理がトリガーされる。

保守担当者向けメモ

  • 本関数は sigsetjmp を使用した堅牢なエラーハンドリングを行っており、エラー発生時もプロセスを落とさず再試行する設計となっている。
  • タイムライン切り替えの計算は、current_tli と latest_tli が異なる場合にのみ実行される。
  • プロセス終了時は on_shmem_exit に登録された WalSummarizerShutdown が実行される。
