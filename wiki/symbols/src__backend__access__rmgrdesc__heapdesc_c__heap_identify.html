heap_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/heapdesc.c:388

目的

ヒープアクセスメソッドのWALレコードタイプ（infoフラグ）を、人間が判読可能な文字列に変換する。

シグネチャまたは形状

const char *heap_identify(uint8 info);

入力と出力

  • **入力**: uint8 info （WALレコードの xl_info フィールド）
  • **出力**: 対応する操作名を表す const char * 型の文字列。一致しない場合は NULL を返す。

前提条件と不変条件

  • 入力 info に含まれる XLR_INFO_MASK ビットは、比較前にマスク処理によって除去される。

アルゴリズムまたは振る舞い

1. 入力された info から XLR_INFO_MASK を除外する。 2. switch 文を用いて、ヒープ操作の種類（INSERT, DELETE, UPDATE, HOT_UPDATE, TRUNCATE, HEAP_CONFIRM, LOCK, INPLACE）および XLOG_HEAP_INIT_PAGE フラグの有無を判定する。 3. 該当する操作名（例: "INSERT", "INSERT+INIT"）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h において、リソースマネージャの定義（PG_RMGR）の一部として登録されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • access/heapam_xlog.h: ヒープWALレコードのフラグ定義（XLOG_HEAP_*）を保持。

失敗モードと境界条件

  • 定義されていない info ビットが渡された場合、id は初期値の NULL のままとなり、戻り値として NULL が返される。

保守担当者向けメモ

  • 新しいヒープWALレコードタイプを追加した際は、本関数の switch 文に必ずケースを追加すること。
  • 本関数は pg_waldump 等のログ解析ツールから参照されるため、文字列の変更は出力形式に影響を与える。
