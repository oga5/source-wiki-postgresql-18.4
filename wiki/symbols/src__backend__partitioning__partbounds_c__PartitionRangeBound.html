PartitionRangeBound

定義

  • 種別: struct
  • 定義箇所: src/backend/partitioning/partbounds.c:64

目的

Rangeパーティションにおける境界値を表現するための構造体です。パーティションの範囲を定義する値の集合と、それが下限か上限かを示すフラグを保持します。

シグネチャまたは形状

typedef struct PartitionRangeBound
{
	int			index;
	Datum	   *datums;
	PartitionRangeDatumKind *kind;
	bool		lower;
} PartitionRangeBound;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • index: パーティションの識別子。
  • datums: 境界値を格納する配列。
  • kind: 各 datum の種類（PartitionRangeDatumKind）。
  • lower: true の場合、この境界は範囲の下限（inclusive/exclusiveの判定基準）として機能します。

前提条件と不変条件

  • datums 配列は、対応するパーティションキーの定義に従ってソートされている必要があります。
  • lower フラグは、その境界が範囲の開始点か終了点かを一意に決定します。

アルゴリズムまたは振る舞い

本構造体は、create_range_bounds 等の関数において、パーティションの境界を正規化・比較する際の単位として利用されます。特に、Rangeパーティションの包含関係や重複チェックを行う際の比較演算の対象となります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partbounds.c 内の境界生成および比較関数（create_range_bounds 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PartitionBoundInfoData: 本構造体を保持し、パーティション全体の境界情報を管理する上位構造体。
  • PartitionRangeDatumKind: datums の各要素の性質（最小値、最大値、通常値など）を定義する列挙型。

失敗モードと境界条件

  • datums 配列が NULL である場合や、kind 配列との整合性が取れていない場合、境界の比較処理において予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/partitioning/partbounds.h にも関連定義が存在します。変更時はヘッダーファイルとの整合性に注意してください。
  • index フィールドは、パーティションの順序や検索効率に直結するため、境界のソート時に正しく割り当てられていることを確認してください。
