end_MultiFuncCall

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/funcapi.c:219

目的

集合を返す関数（SRF）の実行終了時に、登録済みの終了コールバックを解除し、リソースのクリーンアップを実行する。

シグネチャまたは形状

void end_MultiFuncCall(PG_FUNCTION_ARGS, FuncCallContext *funcctx)

入力と出力

  • **入力**:
  • PG_FUNCTION_ARGS: 関数呼び出し情報（fcinfo）。
  • FuncCallContext *funcctx: 終了対象の関数コンテキスト。
  • **出力**: なし（void）。

前提条件と不変条件

  • fcinfo->resultinfo が ReturnSetInfo 型としてキャスト可能であること。
  • rsi->econtext が有効な ExprContext を指していること。

アルゴリズムまたは振る舞い

1. fcinfo->resultinfo から ReturnSetInfo を取得する。 2. UnregisterExprContextCallback を呼び出し、ExprContext に登録されている shutdown_MultiFuncCall コールバックを解除する。 3. shutdown_MultiFuncCall を直接呼び出し、実際の終了処理（メモリ解放等）を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: 集合を返す関数（SRF）の実装において、終了処理が必要な箇所から呼び出される。
  • **呼び出し先**:
  • UnregisterExprContextCallback
  • shutdown_MultiFuncCall

関連データ構造

  • FuncCallContext: SRFの状態保持用構造体。
  • ReturnSetInfo: SRFの実行結果とコンテキスト情報を含む構造体。
  • ExprContext: 式の評価コンテキスト。

失敗モードと境界条件

  • fcinfo が NULL の場合、あるいは resultinfo が適切に初期化されていない場合はクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は shutdown_MultiFuncCall のラッパーとして機能する。終了処理のロジックを変更する場合は shutdown_MultiFuncCall を参照すること。
  • UnregisterExprContextCallback を介してコールバックを解除することで、二重解放や不正なメモリ参照を防いでいる。
