BTPS_State

定義

  • 種別: enum
  • 定義箇所: src/backend/access/nbtree/nbtree.c:54

目的

B-tree インデックスの並列スキャンにおける、各ワーカーのページ処理状態を管理するために使用されます。

シグネチャまたは形状

typedef enum
{
	BTPARALLEL_NOT_INITIALIZED,
	BTPARALLEL_NEED_PRIMSCAN,
	BTPARALLEL_ADVANCING,
	BTPARALLEL_IDLE,
	BTPARALLEL_DONE,
} BTPS_State;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • 並列スキャン実行時に、各ワーカーが現在どのフェーズにあるかを示す状態遷移の一部として利用されます。

アルゴリズムまたは振る舞い

  • BTPARALLEL_NOT_INITIALIZED: 初期化前状態。
  • BTPARALLEL_NEED_PRIMSCAN: プライマリスキャンが必要な状態。
  • BTPARALLEL_ADVANCING: ページを順次進めている状態。
  • BTPARALLEL_IDLE: 待機状態。
  • BTPARALLEL_DONE: スキャン完了状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/nbtree/nbtree.c 内の並列スキャン制御ロジック。
  • 呼び出し先: なし（列挙型のため）。

関連データ構造

  • BTParallelScanDescData: この列挙型をメンバ btps_pageStatus として保持し、並列スキャン時の共有状態を管理します。

失敗モードと境界条件

  • 状態遷移の整合性は、_bt_parallel_seize および _bt_parallel_release を介したロック制御に依存します。不適切なロック管理下での状態更新は競合を引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは並列スキャン時の状態遷移を定義するものです。状態遷移のロジックを変更する場合は、nbtree.c 内の並列スキャン制御関数（_bt_parallel_seize 等）の動作と整合しているか確認してください。
