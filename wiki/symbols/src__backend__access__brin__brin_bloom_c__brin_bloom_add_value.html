brin_bloom_add_value

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_bloom.c:537

目的

BRINインデックスのブルームフィルタに対して、新しい値をハッシュ化して追加し、インデックスの要約情報を更新する。

シグネチャまたは形状

Datum brin_bloom_add_value(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): BrinDesc *（インデックス記述子）
  • PG_GETARG_POINTER(1): BrinValues *（更新対象の列値構造体）
  • PG_GETARG_DATUM(2): Datum（追加する値）
  • PG_GETARG_DATUM(3): bool（値がNULLかどうかのフラグ）
  • **出力**:
  • PG_RETURN_BOOL(updated): フィルタが更新された場合に true を返す。

前提条件と不変条件

  • 入力値がNULLであることは許容されない（Assert(!isnull)）。
  • ブルームフィルタは BrinValues 構造体の bv_values[0] に格納される。
  • フィルタが未初期化（bv_allnulls が真）の場合、bloom_init を用いて新規作成される。

アルゴリズムまたは振る舞い

1. bv_allnulls が真であれば、brin_bloom_get_ndistinct と BloomGetFalsePositiveRate を用いて新しい BloomFilter を初期化する。 2. 既存のフィルタがある場合は、PG_DETOAST_DATUM を介して取得する。 3. bloom_get_procinfo で取得したハッシュ関数を使用し、入力値を uint32 のハッシュ値に変換する。 4. bloom_add_value を呼び出し、ハッシュ値をフィルタに追加する。 5. 更新されたフィルタを column->bv_values[0] に再格納し、更新フラグを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていません。
  • **呼び出し先**:
  • bloom_init
  • bloom_get_procinfo
  • bloom_add_value
  • FunctionCall1Coll

関連データ構造

  • BrinDesc: インデックスのメタデータ。
  • BrinValues: ページ範囲ごとの要約値を保持する構造体。
  • BloomFilter: ブルームフィルタのビットマップと設定を保持する構造体。
  • BloomOptions: 誤判定率などの設定オプション。

失敗モードと境界条件

  • フィルタサイズが BloomMaxFilterSize を超える場合、bloom_init 内でエラーが発生する可能性がある。
  • NULL値が渡された場合はアサーションエラーとなる。

保守担当者向けメモ

  • 本関数は「Less Hashing, Same Performance」論文に基づくハッシュ戦略を採用しています。
  • BloomFilter はTOASTされる可能性があるため、PG_DETOAST_DATUM による展開が必須です。
