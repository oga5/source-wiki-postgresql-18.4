pg_get_shmem_allocations

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:490

目的

共有メモリの割り当て状況を可視化するためのSQL関数（SRF: Set-Returning Function）です。ShmemIndex に登録された構造体、匿名領域、および未使用領域の情報を出力します。

シグネチャまたは形状

Datum pg_get_shmem_allocations(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (PostgreSQLの関数呼び出し規約に従う引数)
  • **出力**: tuplestore を介して、以下の4列からなる行セットを返します。

    • 1. name (text): 構造体名（匿名領域は <anonymous>） 2. off (int8): 共有メモリ先頭からのオフセット 3. size (int8): 割り当てサイズ 4. allocated_size (int8): 実際に確保されたサイズ

前提条件と不変条件

  • ShmemIndexLock を共有モードで取得し、共有メモリの整合性を保護します。
  • InitMaterializedSRF を呼び出し、結果セットを格納する tuplestore を初期化する必要があります。

アルゴリズムまたは振る舞い

1. ShmemIndexLock を共有ロックします。 2. hash_seq_init を使用して ShmemIndex ハッシュテーブルを走査し、各エントリの情報を tuplestore に格納します。 3. 走査中に named_allocated（名前付き構造体の合計サイズ）を計算します。 4. 匿名領域（ShmemSegHdr->freeoffset から named_allocated を引いたもの）を出力します。 5. 未使用領域（ShmemSegHdr->totalsize から freeoffset を引いたもの）を出力します。 6. LWLockRelease でロックを解放し、終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（直接のC言語レベルの呼び出し元はソースコード上では明示されていません）。
  • **呼び出し先**: InitMaterializedSRF, LWLockAcquire, hash_seq_init, hash_seq_search, tuplestore_putvalues, LWLockRelease 等のPostgreSQL内部API。

関連データ構造

  • ShmemIndex: 共有メモリ上の全データ構造を管理するハッシュテーブル。
  • ShmemSegHdr: 共有メモリセグメントのヘッダー（freeoffset や totalsize を保持）。
  • ShmemIndexEnt: ShmemIndex 内の各エントリ構造体。

失敗モードと境界条件

  • ShmemIndexLock が取得できない場合、処理はブロックされます。
  • 共有メモリが初期化されていない状態で呼び出された場合の挙動は、ShmemSegHdr へのアクセスにより未定義となります。

保守担当者向けメモ

  • 本関数は pg_get_shmem_allocations_numa と類似していますが、NUMAノード別の詳細情報は含みません。
  • tuplestore を使用しているため、メモリ消費量は結果セットの行数に比例します。
