report_error_fn

定義

  • 種別: typedef
  • 定義箇所: src/include/common/blkreftable.h:47

目的

ブロック参照テーブルの処理（読み込み等）において発生したエラーを、呼び出し元へ通知するためのコールバック関数型です。

シグネチャまたは形状

typedef void (*report_error_fn) (void *callback_arg, char *msg, ...) pg_attribute_printf(2, 3);

入力と出力

  • callback_arg: エラー処理コンテキストを保持する任意のデータへのポインタ。
  • msg: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • 戻り値: なし（void）。

前提条件と不変条件

  • pg_attribute_printf(2, 3) により、第2引数をフォーマット文字列、第3引数以降をその引数としてコンパイル時に型チェックが行われます。
  • ヘッダーファイルの記述によれば、この関数は戻らない（non-returning）ことが想定されています。

アルゴリズムまたは振る舞い

ブロック参照テーブルの読み込み処理中に不整合やエラーが検出された際、この関数が呼び出されます。具体的なエラー報告の振る舞い（ログ出力、例外送出、プロセス終了など）は、実装側で提供されるコールバック関数に依存します。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/blkreftable.c 内の読み込み処理ロジック（error_callback として参照）。
  • 呼び出し先: 実装依存（提供されたコールバック関数）。

関連データ構造

  • BlockRefTableReader: 本関数をエラーハンドラとして保持する構造体。

失敗モードと境界条件

  • 本関数自体がエラーを発生させることはありませんが、呼び出し元が提供する実装が適切にエラーを処理できない場合、システム全体の整合性に影響を与える可能性があります。

保守担当者向けメモ

  • 本関数は blkreftable モジュールのエラーハンドリングの抽象化層です。実装を追加する際は、pg_attribute_printf の制約に従い、フォーマット文字列と引数の整合性を維持してください。
