CHKVAL

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltxtquery_op.c:49

目的

ltree 型のデータと、それに対する検索クエリ（ltxtquery）のオペランドを照合する際に、評価対象のコンテキストを保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	ltree	   *node;
	char	   *operand;
} CHKVAL;

入力と出力

本構造体はデータ保持用であり、関数ではありません。ltree_execute 等の評価エンジンにおいて、照合対象の ltree ポインタと、クエリ内のオペランド文字列へのポインタを格納するコンテナとして機能します。

前提条件と不変条件

  • node は有効な ltree 構造体を指している必要があります。
  • operand は ltxtquery 内のオペランド文字列を指すポインタであり、照合ロジック（checkcondition_str）によってオフセット計算の基準として利用されます。

アルゴリズムまたは振る舞い

ltree_execute がクエリを再帰的に走査する際、この構造体を介して現在の照合対象データにアクセスします。照合関数（checkcondition_str）は、この構造体を void * として受け取り、内部で CHKVAL * にキャストして node の各レベルや operand の内容を参照します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/ltxtquery_op.c 内の ltxtq_exec 等が、評価開始時にこの構造体を初期化します。
  • **呼び出し先**: checkcondition_str がこの構造体を参照してマッチングを行います。
  • **注記**: contrib/intarray/_int_bool.c や src/backend/utils/adt/tsgistidx.c にも同名の構造体定義が存在しますが、これらは本ファイル（ltxtquery_op.c）の定義とは別個のコンテキストで使用されています。

関連データ構造

  • ltree: 照合対象となるツリー構造。
  • ltxtquery: 検索条件を保持するポーランド記法形式のクエリ構造。

失敗モードと境界条件

  • node が NULL の場合、参照時にセグメンテーションフォールトが発生する可能性があります。
  • operand が指すメモリ領域がクエリの有効範囲外である場合、不正なメモリ読み取りが発生します。

保守担当者向けメモ

  • 本構造体は他のモジュール（intarray 等）でも同名で定義されています。contrib/ltree 内で修正を行う際は、他のモジュールとの定義の競合や、キャスト時の型安全性に注意してください。
  • ltxtquery_op.c 内の checkcondition_str におけるキャスト処理は、この構造体のメモリレイアウトに強く依存しています。構造体のメンバ順序を変更する場合は、関連するすべてのキャスト箇所を更新する必要があります。
