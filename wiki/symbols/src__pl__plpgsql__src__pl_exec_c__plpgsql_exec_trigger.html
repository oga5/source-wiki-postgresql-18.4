plpgsql_exec_trigger

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_exec.c:934

目的

PL/pgSQL で記述されたトリガープロシージャを実行し、トリガーイベントに応じた実行状態の初期化、OLD/NEW レコードの準備、およびトリガー本体の実行と結果の返却を行う。

シグネチャまたは形状

HeapTuple plpgsql_exec_trigger(PLpgSQL_function *func, TriggerData *trigdata)

入力と出力

  • **入力**:
  • func: 実行対象の PL/pgSQL 関数構造体。
  • trigdata: トリガー実行時のコンテキスト情報（イベント、対象リレーション、タプル等）。
  • **出力**:
  • HeapTuple: トリガーによって変更または返却されるタプル。行レベルトリガー以外や NULL を返す場合は NULL。

前提条件と不変条件

  • func および trigdata は有効なポインタであること。
  • trigdata->tg_relation から取得される TupleDesc が、トリガーの実行環境と整合していること。
  • トリガー関数は RETURN 文で終了する必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: plpgsql_estate_setup で実行状態（PLpgSQL_execstate）を構築し、エラーコールバックをスタックに積む。 2. **データ準備**: copy_plpgsql_datums で変数を初期化し、OLD/NEW レコードを expanded_record としてセットアップする。 3. **トリガー種別処理**: trigdata->tg_event に基づき、INSERT/UPDATE/DELETE の各ケースで OLD/NEW タプルをレコードに割り当てる。UPDATE の BEFORE トリガーでは、生成列（stored generated columns）を NULL に設定する。 4. **実行**: SPI_register_trigger_data で遷移テーブルを可視化し、exec_toplevel_block を呼び出して関数本体を実行する。 5. **検証と終了**: 戻り値が PLPGSQL_RC_RETURN であることを確認し、必要に応じてタプルの構造変換（convert_tuples_by_position）を行い、結果の HeapTuple を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_handler.c 内の関数（トリガーハンドラ）。
  • **呼び出し先**: plpgsql_estate_setup, copy_plpgsql_datums, expanded_record_set_tuple, SPI_register_trigger_data, exec_toplevel_block, convert_tuples_by_position 等。

関連データ構造

  • PLpgSQL_execstate: 実行中の関数の状態を保持。
  • TriggerData: トリガーのイベント情報。
  • PLpgSQL_rec: OLD/NEW レコードを保持する構造体。
  • ExpandedRecordHeader: 拡張レコードのヘッダ。

失敗モードと境界条件

  • **戻り値なし**: 関数が RETURN 文に到達せずに終了した場合、ERRCODE_S_R_E_FUNCTION_EXECUTED_NO_RETURN_STATEMENT エラーを発生させる。
  • **型不一致**: トリガーがセットを返そうとした場合、ERRCODE_DATATYPE_MISMATCH エラーを発生させる。
  • **構造不一致**: 返されたタプルの構造がトリガー対象テーブルと一致しない場合、変換を試みるが、不可能な場合はエラーとなる。

保守担当者向けメモ

  • BEFORE トリガーにおける生成列の処理は、UPDATE 時の副作用を避けるための特殊なロジックである。
  • expanded_record を使用したタプル操作は、パフォーマンス最適化のために行われている。
  • 戻り値のタプル変換処理は、トリガー関数が返すタプルとテーブル定義の不一致を吸収するために重要である。
