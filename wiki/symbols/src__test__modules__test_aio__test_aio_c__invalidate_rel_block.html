invalidate_rel_block

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:456

目的

指定されたリレーションのブロックを強制的に無効化（破棄）するためのテスト用関数です。共有バッファまたはローカルバッファの状態を直接操作し、テスト環境において特定のブロックを強制的に解放させるために使用されます。

シグネチャまたは形状

Datum invalidate_rel_block(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象リレーションの OID。
  • PG_GETARG_UINT32(1): 無効化対象のブロック番号 (BlockNumber)。
  • **出力**: なし (PG_RETURN_VOID)。

前提条件と不変条件

  • 対象リレーションに対して AccessExclusiveLock を取得します。
  • PrefetchBuffer を使用してバッファを取得するため、対象ブロックがバッファキャッシュに存在する必要があります。
  • バッファがダーティな場合は、無効化の前にフラッシュ処理が行われます。

アルゴリズムまたは振る舞い

1. 対象リレーションを AccessExclusiveLock でオープンします。 2. PrefetchBuffer を呼び出し、対象ブロックのバッファ ID を取得します。 3. ReadRecentBuffer でバッファ内容の有効性を確認します。 4. バッファがダーティな場合、FlushLocalBuffer または FlushOneBuffer を用いてディスクへ書き出します。 5. バッファのロックを解除し、参照カウントを解放します。 6. InvalidateLocalBuffer または EvictUnpinnedBuffer を呼び出し、バッファをキャッシュから強制的に追い出します。 7. リレーションをクローズして終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由での呼び出しを想定（PG_FUNCTION_INFO_V1 による登録あり）。
  • **呼び出し先**: relation_open, PrefetchBuffer, ReadRecentBuffer, LockBuffer, FlushLocalBuffer, FlushOneBuffer, ReleaseBuffer, InvalidateLocalBuffer, EvictUnpinnedBuffer, relation_close 等の PostgreSQL 内部 API。

関連データ構造

  • Relation: リレーション記述子。
  • Buffer: バッファ識別子。
  • BufferDesc: バッファ記述子（GetLocalBufferDescriptor または GetBufferDescriptor で取得）。

失敗モードと境界条件

  • EvictUnpinnedBuffer が失敗した場合、elog(ERROR, "couldn't evict") を発生させます。
  • バッファが有効でない場合、無効化処理はスキップされます。

保守担当者向けメモ

  • 本関数は共有バッファの状態を直接操作する「ハック」を含んでおり、テスト専用です。本番コードでの利用は避けてください。
  • PrefetchBuffer を使用してバッファ ID を取得する手法は、他に適切な API が存在しないための暫定的な実装です。
