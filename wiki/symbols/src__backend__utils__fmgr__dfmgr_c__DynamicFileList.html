DynamicFileList

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/fmgr/dfmgr.c:46 および fmgr.h:782

目的

ロード済みの動的ライブラリ（共有オブジェクト）を追跡・管理するための連結リスト構造体です。

シグネチャまたは形状

struct DynamicFileList {
    DynamicFileList *next;
    dev_t device;
#ifndef WIN32
    ino_t inode;
#endif
    void *handle;
    const Pg_magic_struct *magic;
    char filename[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: internal_load_library 等でライブラリをロードする際、filename を含む可変長データとしてメモリ確保されます。
  • **出力**: get_next_loaded_module 等を通じて、ロード済みライブラリのリストを走査するために使用されます。

前提条件と不変条件

  • filename は FLEXIBLE_ARRAY_MEMBER を使用した可変長配列であり、構造体末尾に配置されます。
  • WIN32 環境以外では、ファイルの一意性を判定するために device と inode が使用されます。
  • 一度ロードされたライブラリは、現在の設計上アンロードされません。

アルゴリズムまたは振る舞い

  • internal_load_library は、新しいライブラリをロードする前に file_list を走査し、同一の device および inode を持つエントリが存在するか確認することで、重複ロードを防止します。
  • 新規ロード時は malloc を使用して offsetof(DynamicFileList, filename) + strlen(libname) + 1 のサイズでメモリを確保し、filename を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/fmgr/dfmgr.c 内の internal_load_library（リストへの追加）、get_next_loaded_module（リストの走査）、および src/backend/commands/extension.c（拡張機能管理）。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • Pg_magic_struct: ライブラリの ABI 互換性を検証するためのマジックブロック構造体。
  • file_list / file_tail: dfmgr.c 内で管理される、ロード済みライブラリリストの先頭および末尾ポインタ。

失敗モードと境界条件

  • メモリ不足時: malloc 失敗によりプロセスが終了する可能性があります。
  • ABI 不整合: magic フィールドが指す情報が PG_MODULE_MAGIC と一致しない場合、ロードが拒否されます。

保守担当者向けメモ

  • filename は可変長配列であるため、構造体のコピーやスタックへの割り当ては避けてください。
  • Windows 環境では inode が存在しないため、プラットフォーム固有の差異に注意してください。
  • リストの走査には get_next_loaded_module を使用することが推奨されます。
