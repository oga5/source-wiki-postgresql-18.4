get_stack_depth_rlimit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/stack_depth.c:175

目的

OSが設定しているスタックサイズの物理的な上限（リソース制限）を取得します。この値は、PostgreSQLのスタック監視機能において、設定可能な上限値の妥当性を検証するために使用されます。

シグネチャまたは形状

ssize_t get_stack_depth_rlimit(void);

入力と出力

  • **入力**: なし。
  • **出力**: OSのスタック制限値（バイト単位）。制限が無限の場合は SSIZE_MAX を返し、取得に失敗した場合は -1 を返します。

前提条件と不変条件

  • プロセス起動後に値が変更されることは想定されておらず、初回呼び出し時に取得した値を静的変数にキャッシュして再利用します。
  • Windows環境では、src/backend/Makefile で定義された WIN32_STACK_RLIMIT が返されます。

アルゴリズムまたは振る舞い

1. HAVE_GETRLIMIT が定義されている環境では、getrlimit(RLIMIT_STACK, ...) を呼び出します。 2. 取得した rlim_cur が RLIM_INFINITY の場合、または SSIZE_MAX を超える場合は SSIZE_MAX を返します。 3. それ以外の場合は、取得した値をそのまま返します。 4. 初回呼び出し時にのみOSへの問い合わせを行い、以降はキャッシュされた値を返します。

呼び出し元と呼び出し先

  • src/backend/utils/misc/guc.c:1613
  • src/backend/utils/misc/stack_depth.c:145
  • **呼び出し先**:

関連データ構造

  • struct rlimit: OSからスタック制限を取得するために使用される構造体。

失敗モードと境界条件

  • getrlimit が失敗した場合、-1 を返します。
  • スタック制限が SSIZE_MAX を超える場合、オーバーフローを防ぐために SSIZE_MAX に丸められます。

保守担当者向けメモ

  • 本関数はプロセス起動時に一度だけ評価されることを前提としています。
  • Windows環境におけるスタック制限値は、コンパイル時の設定に依存するため注意してください。
