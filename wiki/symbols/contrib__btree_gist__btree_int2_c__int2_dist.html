int2_dist

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int2.c:95

目的

2つの int2（2バイト整数）値間の絶対距離を計算します。

シグネチャまたは形状

Datum int2_dist(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS を介して2つの int16 値を受け取ります。
  • 出力: int16 型の絶対差分を Datum として返します。

前提条件と不変条件

  • 入力値の差分が int16 の範囲内に収まる必要があります。
  • PG_INT16_MIN は絶対値変換時にオーバーフローを引き起こす可能性があるため、計算結果がこの値になる場合はエラーとなります。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT16 を使用して2つの入力値 a と b を取得します。 2. pg_sub_s16_overflow を用いて a - b の減算を行い、オーバーフローをチェックします。 3. 減算結果 r が PG_INT16_MIN であるか、またはオーバーフローが発生した場合は ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE でエラーを報告します。 4. abs(r) により差分の絶対値を求めます。 5. PG_RETURN_INT16 を使用して結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先: pg_sub_s16_overflow（オーバーフローチェック用）、abs（標準Cライブラリ）、ereport（エラー報告用）。

関連データ構造

  • int16: PostgreSQL の 2バイト整数型。

失敗モードと境界条件

  • 減算結果が int16 の表現範囲（-32768〜32767）を超える場合、エラーとなります。
  • 減算結果が PG_INT16_MIN (-32768) となる場合、絶対値をとると int16 の範囲外（32768）となるため、エラーとなります。

保守担当者向けメモ

  • 本関数は btree_gist 拡張の一部として提供されていますが、インデックス操作のサポート関数とは独立したユーティリティ関数です。
  • int16 の範囲制限に厳格であるため、将来的に型が拡張される場合は注意が必要です。
