free_variable

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:82

目的

連結リスト構造を持つ variable 構造体のリストを走査し、各要素に対してメモリ解放処理を行う。

シグネチャまたは形状

static void free_variable(struct variable *var)

入力と出力

  • 入力: struct variable *var （解放対象のリストの先頭ポインタ）
  • 出力: なし（void）

前提条件と不変条件

  • var は NULL であるか、または next ポインタで連結された有効な variable 構造体のリストの先頭であること。
  • ecpg_free が内部で呼び出されるため、リスト内の各要素は ecpg_free で解放可能なメモリ領域である必要がある。

アルゴリズムまたは振る舞い

1. 引数 var が NULL になるまでループを実行する。 2. 現在のノードの next ポインタを一時変数 var_next に退避する。 3. ecpg_free(var) を呼び出し、現在のノードのメモリを解放する。 4. var に var_next を代入し、次のノードへ進む。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/ecpglib/execute.c 内の free_statement（推測される文脈）
  • src/interfaces/ecpg/ecpglib/execute.c 内の ECPGdo（stmt->inlist および stmt->outlist の解放時）
  • 呼び出し先:
  • ecpg_free（src/interfaces/ecpg/ecpglib/memory.c で定義されているメモリ解放関数）

関連データ構造

  • struct variable: ecpglib においてホスト変数の情報を保持する連結リスト構造体。

失敗モードと境界条件

  • var が NULL の場合、ループは実行されず何も行われない（安全に終了する）。

保守担当者向けメモ

  • 本関数はリストの走査と解放のみを担当する。個別の variable 構造体内部で保持されている動的確保されたメンバの解放については、ecpg_free の実装および variable 構造体の定義を確認すること。
