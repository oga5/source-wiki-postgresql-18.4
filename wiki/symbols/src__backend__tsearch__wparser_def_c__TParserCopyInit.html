TParserCopyInit

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/wparser_def.c:345

目的

既存の TParser 構造体の状態を基に、解析位置を現在のオフセットから開始する新しい TParser インスタンスを生成します。

シグネチャまたは形状

static TParser *TParserCopyInit(const TParser *orig)

入力と出力

  • **入力**: orig (既存の TParser 構造体へのポインタ)
  • **出力**: 新規にメモリ確保された TParser 構造体へのポインタ。解析対象文字列のポインタは orig の現在位置（posbyte および poschar）まで進められた状態で初期化されます。

前提条件と不変条件

  • orig は有効な TParser 構造体を指している必要があります。
  • palloc0 を使用してメモリを確保するため、PostgreSQL のメモリコンテキスト内での実行が前提となります。
  • 新しい TParser の状態（state）は TPS_Base にリセットされます。

アルゴリズムまたは振る舞い

1. palloc0 を用いて新しい TParser 構造体領域を確保します。 2. orig の charmaxlen、usewide をコピーします。 3. orig の現在の解析位置（posbyte および poschar）に基づき、文字列ポインタ（str, pgwstr, wstr）をオフセット分進めて設定します。 4. lenstr を残りのバイト数に合わせて調整します。 5. newTParserPosition(NULL) を呼び出し、解析状態を TPS_Base で初期化します。 6. WPARSER_TRACE が定義されている場合、デバッグ情報を標準エラー出力に出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/wparser_def.c 内の関数（631 行目および 659 行目付近で確認）。
  • **呼び出し先**:
  • palloc0
  • newTParserPosition

関連データ構造

  • TParser: 解析状態を保持するメイン構造体。
  • TParserPosition: 解析の現在地を管理する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • orig->state が NULL の場合、posbyte や poschar へのアクセスでセグメンテーション違反が発生する可能性があります（ソースコード上、orig が適切に初期化されていることが前提となっています）。

保守担当者向けメモ

  • 本関数で作成された TParser は、対応する TParserCopyClose を使用して解放する必要があります。
  • WPARSER_TRACE マクロによるデバッグ出力は、開発時の解析フロー追跡に有用です。
