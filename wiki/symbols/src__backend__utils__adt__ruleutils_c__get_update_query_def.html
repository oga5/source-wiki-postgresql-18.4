get_update_query_def

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ruleutils.c:7150

目的

UPDATE 文のパースツリー（Query 構造体）を解析し、対応する SQL 文字列を再構築（デパース）します。

シグネチャまたは形状

void get_update_query_def(Query *query, deparse_context *context)

入力と出力

  • **入力**:
  • query: デパース対象の UPDATE クエリツリー。
  • context: デパース処理の状態を保持する deparse_context 構造体（出力バッファ buf を含む）。
  • **出力**: なし（副作用として context->buf に SQL 文字列が追記されます）。

前提条件と不変条件

  • query->resultRelation が指す RangeTblEntry は RTE_RELATION である必要があります（Assert で検証済み）。
  • context は初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. **WITH 句**: get_with_clause を呼び出し、存在すれば WITH 句を出力します。 2. **UPDATE 句**: resultRelation から対象テーブルを特定し、UPDATE [ONLY] relname を出力します。 3. **エイリアス**: get_rte_alias を呼び出し、テーブルエイリアスが必要な場合は出力します。 4. **SET 句**: get_update_query_targetlist_def を呼び出し、更新対象の列と値を生成します。 5. **FROM 句**: get_from_clause を呼び出し、FROM 句が存在すれば出力します。 6. **WHERE 句**: query->jointree->quals が存在する場合、WHERE 句と条件式を get_rule_expr で再帰的に出力します。 7. **RETURNING 句**: query->returningList が存在する場合、get_returning_clause を呼び出し出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/ruleutils.c 内のクエリデパース処理（get_query_def 等）から呼び出されます。
  • **呼び出し先**:
  • get_with_clause, rt_fetch, generate_relation_name, get_rte_alias, get_update_query_targetlist_def, get_from_clause, appendContextKeyword, get_rule_expr, get_returning_clause。

関連データ構造

  • Query: デパース対象のクエリツリー。
  • deparse_context: 出力バッファや整形フラグを管理する状態オブジェクト。
  • RangeTblEntry: 更新対象テーブルのメタデータ。

失敗モードと境界条件

  • rte->rtekind が RTE_RELATION でない場合、アサーションエラーが発生します。
  • jointree->quals が NULL の場合、WHERE 句は出力されません。
  • returningList が空の場合、RETURNING 句は出力されません。

保守担当者向けメモ

  • 本関数は UPDATE 文の構文再構築の中核を担います。MERGE や INSERT 等の他の DML 文と同様に、get_query_def からのディスパッチ先として機能します。
  • PRETTY_INDENT フラグが有効な場合、インデント制御が行われます。
