pgwin32_setlocale

定義

  • 種別: function
  • 定義箇所: src/port/win32setlocale.c:171

目的

Windows 環境における標準ライブラリ setlocale() の不具合を吸収するためのラッパー関数です。ロケール名の正規化を行い、OS 側で認識されない形式や非 ASCII 文字を含む戻り値の問題を回避します。

シグネチャまたは形状

char *pgwin32_setlocale(int category, const char *locale);

入力と出力

  • **入力**:
  • category: ロケールのカテゴリ（LC_ALL 等）。
  • locale: 設定するロケール名（文字列）。NULL の場合は現在のロケールを取得します。
  • **出力**:
  • 成功時: 正規化されたロケール名を表す文字列へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • 本関数は setlocale を置き換えるため、ソースコード内で #undef setlocale が行われています。
  • map_locale が使用する静的バッファ aliasbuf を介して結果を返すため、スレッドセーフな環境での利用には注意が必要です。

アルゴリズムまたは振る舞い

1. **入力変換**: locale が NULL でない場合、map_locale 関数と locale_map_argument を用いて、Windows が許容する形式へロケール名を置換します。 2. **実行**: 変換後の引数を用いて、本来の setlocale() を呼び出します。 3. **出力変換**: setlocale() の戻り値が NULL でない場合、map_locale 関数と locale_map_result を用いて、非 ASCII 文字を排除した安全な文字列に変換します。 4. **戻り値**: 変換後の文字列を返します。setlocale の仕様に従い、戻り値の const は unconstify マクロによって解除されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて setlocale として定義されており、PostgreSQL の各所から標準関数として呼び出されます。
  • **呼び出し先**:
  • map_locale（内部ヘルパー）
  • setlocale（標準ライブラリ）

関連データ構造

  • locale_map_argument: 入力ロケール名の置換ルール。
  • locale_map_result: 出力ロケール名の置換ルール。

失敗モードと境界条件

  • setlocale() が NULL を返した場合、本関数もそのまま NULL を返します。
  • map_locale による文字列置換が失敗または不要な場合、元の文字列がそのまま使用されます。

保守担当者向けメモ

  • map_locale は静的バッファ aliasbuf を使用するため、呼び出しごとに結果が上書きされる可能性があります。
  • 本関数は Windows プラットフォーム固有の移植性レイヤーの一部であり、他のプラットフォームでは使用されません。
