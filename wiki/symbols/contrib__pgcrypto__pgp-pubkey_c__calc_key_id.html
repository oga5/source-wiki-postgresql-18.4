calc_key_id

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubkey.c:83

目的

OpenPGP 公開鍵のデータから一意な鍵 ID（Key ID）を計算し、PGP_PubKey 構造体に格納する。

シグネチャまたは形状

static int calc_key_id(PGP_PubKey *pk);

入力と出力

  • **入力**: PGP_PubKey *pk（鍵情報を含む構造体へのポインタ）
  • **出力**: 成功時は 0 を返し、pk->key_id に 8 バイトの計算結果を書き込む。失敗時は負の整数（エラーコード）を返す。

前提条件と不変条件

  • pk->algo に基づき、対応する公開鍵パラメータ（MPI）が pk->pub 内に正しく設定されていること。
  • SHA1 ハッシュ計算のためのメモリリソースが確保可能であること。

アルゴリズムまたは振る舞い

1. pgp_load_digest を使用して SHA1 ハッシュコンテキストを初期化する。 2. 鍵のアルゴリズム（RSA, DSA, ElGamal）に応じて、パケットヘッダおよび公開鍵パラメータの長さを算出する。 3. パケットヘッダ（0x99 等）、鍵のバージョン、作成時刻、アルゴリズムタイプをハッシュ化する。 4. pgp_mpi_hash を呼び出し、各アルゴリズム固有の MPI（多倍長整数）をハッシュコンテキストに追加する。 5. px_md_finish で SHA1 ハッシュ値を生成し、その下位 8 バイトを pk->key_id にコピーする。 6. 使用したハッシュコンテキストを解放し、一時的なハッシュバッファをゼロクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pubkey.c 内の internal_read_key 関数から、公開鍵パケットの解析処理中に呼び出される。
  • **呼び出し先**: pgp_load_digest, px_md_update, pgp_mpi_hash, px_md_finish, px_md_free, px_memset 等の暗号ユーティリティ関数。

関連データ構造

  • PGP_PubKey: 鍵情報を保持する主要な構造体。
  • PX_MD: ハッシュ計算用コンテキスト。

失敗モードと境界条件

  • pgp_load_digest が失敗した場合、そのエラーコードをそのまま返す。
  • 未対応の pk->algo が指定された場合、ハッシュ計算が不完全な状態で終了する可能性がある。

保守担当者向けメモ

  • 本関数は PGP バージョン 4 のパケット形式に依存している。
  • 鍵 ID は SHA1 ハッシュの下位 8 バイトを使用する仕様であるため、ハッシュアルゴリズムの変更は慎重に行うこと。
  • pk->pub 内の各 MPI 構造体へのアクセスは、pk->algo との整合性が取れていることを前提としている。
