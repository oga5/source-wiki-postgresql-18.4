SystemSamplerData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/tablesample/system.c:37

目的

TABLESAMPLE SYSTEM メソッドによるスキャン実行中の状態を保持するためのプライベート構造体です。

シグネチャまたは形状

typedef struct
{
	uint64		cutoff;			/* select blocks with hash less than this */
	uint32		seed;			/* random seed */
	BlockNumber nextblock;		/* next block to consider sampling */
	OffsetNumber lt;			/* last tuple returned from current block */
} SystemSamplerData;

入力と出力

  • 入力: system_beginsamplescan 等で初期化され、SampleScanState の tsm_state フィールドに格納されます。
  • 出力: スキャン処理の各ステップ（ブロック選択、タプル抽出）において、現在のスキャン位置や状態を更新・参照するために使用されます。

前提条件と不変条件

  • palloc0 を使用してメモリ確保されることが想定されています。
  • cutoff はサンプリング率に基づいて算出された閾値であり、ハッシュ値との比較に使用されます。

アルゴリズムまたは振る舞い

  • nextblock を用いてスキャン対象のブロックを追跡します。
  • lt (last tuple) を用いて、現在読み込んでいるブロック内でのタプル位置を管理します。
  • seed と cutoff を用いて、ブロック単位のサンプリングの再現性を保証します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/tablesample/system.c 内の system_beginsamplescan (メモリ確保)
  • system_nextsampleblock (状態参照・更新)
  • system_nextsampletuple (状態参照・更新)
  • 呼び出し先: 情報なし。

関連データ構造

  • SampleScanState: 本構造体を tsm_state として保持します。
  • TsmRoutine: TABLESAMPLE メソッドのインターフェースを定義します。

失敗モードと境界条件

  • メモリ確保失敗: palloc0 が失敗した場合の挙動は PostgreSQL の標準的なメモリ管理に従います。
  • スキャン終了: nextblock がテーブルの全ブロック数に達した際、あるいは lt がブロック内の最終タプルを超えた際の終了条件管理に依存します。

保守担当者向けメモ

  • SystemSamplerData は SampleScanState に格納されるため、実行終了時のクリーンアップ（メモリ解放）が適切に行われるよう注意してください。
  • ハッシュ計算には hash_any が使用されており、プラットフォーム間での一貫性が保たれています。
