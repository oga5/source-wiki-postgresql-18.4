inet_spg_config

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_spgist.c:50

目的

SP-GiSTインデックス構築時に、ネットワーク型（inet, cidr）のインデックス設定を初期化します。

シグネチャまたは形状

Datum inet_spg_config(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgConfigOut 構造体へのポインタを受け取ります。
  • **出力**: なし（PG_RETURN_VOID() を返します）。spgConfigOut 構造体の各フィールドを更新することで設定を反映します。

前提条件と不変条件

  • spgConfigOut 構造体が正しく割り当てられている必要があります。

アルゴリズムまたは振る舞い

1. spgConfigOut 構造体の各メンバに以下の値を設定します。

  • prefixType: CIDROID（プレフィックス型としてCIDRを使用）
  • labelType: VOIDOID（ラベルは使用しない）
  • canReturnData: true（インデックスから元のデータを取得可能）
  • longValuesOK: false（長い値の格納は許可しない）

    • 2. 処理終了後、PG_RETURN_VOID() を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックスの構築プロセスから呼び出されます（具体的な呼び出し元関数名はソースコードの抜粋からは特定できません）。
  • **呼び出し先**: なし。

関連データ構造

  • spgConfigOut: SP-GiSTインデックスの構成情報を保持する構造体。
  • CIDROID, VOIDOID: PostgreSQLの型ID。

失敗モードと境界条件

  • 供給された引数が無効なポインタである場合、メモリ保護違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数はSP-GiSTのインデックス定義の根幹を成すため、インデックスのデータ構造（プレフィックスの扱いなど）を変更する場合は、ここでの設定値と整合性を取る必要があります。
