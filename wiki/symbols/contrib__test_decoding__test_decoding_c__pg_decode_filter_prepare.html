pg_decode_filter_prepare

定義

  • 種別: function
  • 定義箇所: contrib/test_decoding/test_decoding.c:452

目的

論理デコーディングにおける「準備済みトランザクション（prepared transaction）」のフィルタリング判定を行うコールバック関数です。特定の条件に合致するトランザクションをデコード対象から除外するために使用されます。

シグネチャまたは形状

static bool pg_decode_filter_prepare(LogicalDecodingContext *ctx, TransactionId xid, const char *gid);

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • xid: 準備済みトランザクションのトランザクションID。
  • gid: 準備済みトランザクションに割り当てられたグローバル識別子（Global Transaction ID）。
  • **出力**:
  • bool: true を返すと当該トランザクションのデコードをスキップし、false を返すとデコードを継続します。

前提条件と不変条件

  • gid はヌル終端文字列である必要があります。
  • 内部で strstr を使用しているため、gid が NULL の場合は未定義の動作となります。

アルゴリズムまたは振る舞い

1. 引数 gid に文字列 "_nodecode" が含まれているかを確認します。 2. 含まれている場合、true を返し、当該トランザクションの処理をフィルタリング（除外）します。 3. 含まれていない場合、false を返し、デコード処理を許可します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの論理デコーディングコアエンジン（filter_prepare_cb コールバック経由）。
  • **呼び出し先**: strstr（標準Cライブラリ関数）。
  • **補足**: contrib/test_decoding/test_decoding.c:141 にて、プラグイン初期化時に cb->filter_prepare_cb へ本関数が登録されています。

関連データ構造

  • LogicalDecodingContext: 論理デコーディングのセッション状態を保持する構造体。

失敗モードと境界条件

  • gid が NULL の場合、strstr の呼び出しによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数はテスト用プラグインの一部であり、gid に特定のサフィックスが含まれるかという単純な文字列マッチングでフィルタリングを決定しています。実運用環境向けのプラグインを作成する際は、より堅牢な識別ロジックへの置き換えを検討してください。
