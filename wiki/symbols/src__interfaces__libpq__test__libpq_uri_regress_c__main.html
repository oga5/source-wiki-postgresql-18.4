目的

libpq の接続文字列（URI形式を含む）を解析し、デフォルト値と比較して差分を出力する回帰テスト用プログラムの起点。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**: コマンドライン引数 argv[1] として接続文字列を受け取る。
  • **出力**:
  • 成功時: デフォルト値と異なる接続オプションを keyword='value' 形式で標準出力し、最後に (local) または (inet) を出力して終了コード 0 を返す。
  • 失敗時: エラーメッセージを標準エラー出力に表示し、終了コード 1 を返す。

前提条件と不変条件

  • PQconninfoOption 構造体配列内のキーワード順序が、解析結果 (opts) とデフォルト値 (defs) で一致している必要がある。
  • Unix ドメインソケットの判定は host 値の先頭文字が '/' であるかどうかに依存する。

アルゴリズムまたは振る舞い

1. 引数が1つでない場合、終了コード 1 を返す。 2. PQconninfoParse で接続文字列を解析し、失敗した場合はエラーを出力して終了する。 3. PQconndefaults でデフォルト値を取得し、失敗した場合は終了する。 4. opts と defs を順次走査し、opt->val が存在し、かつデフォルト値と異なる場合にそのキーワードと値を標準出力する。 5. hostaddr が指定されているか、あるいは host が指定されており値が '/' で始まらない場合、接続タイプを inet と判定する。 6. 判定結果に基づき (local) または (inet) を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: 本プログラムはテスト用実行ファイルであり、外部からの呼び出し情報は提供されていない。
  • **呼び出し先**:
  • PQconninfoParse (libpq API)
  • PQconndefaults (libpq API)
  • fprintf, printf, strcmp (標準ライブラリ)

関連データ構造

  • PQconninfoOption: libpq の接続オプションを保持する構造体。

失敗モードと境界条件

  • 引数の数が不正な場合、即座に終了する。
  • PQconninfoParse が NULL を返した場合、エラーメッセージを表示して終了する。
  • PQconndefaults が NULL を返した場合、エラーメッセージを表示して終了する。
  • Windows 環境では Unix ドメインソケットの判定ロジックが意図通りに動作しない可能性がある。

保守担当者向けメモ

  • 本関数はライブラリの一部ではなく、回帰テスト用ツールである。
  • PQconninfoOption の順序依存性があるため、API 仕様変更時には注意が必要である。
