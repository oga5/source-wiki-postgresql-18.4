client_initial_response

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-oauth.c:105

目的

SASL OAUTHBEARER 認証メカニズムにおいて、サーバーへ送信する初期応答文字列を生成します。

シグネチャまたは形状

char *client_initial_response(PGconn *conn, bool discover);

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体へのポインタ。conn->oauth_token を参照します。
  • discover: パラメータ探索モード（Discovery）かどうかを示すフラグ。
  • **出力**:
  • 成功時: 構築された認証応答文字列（呼び出し元で解放が必要）。
  • 失敗時: NULL。エラー内容は conn に格納されます。

前提条件と不変条件

  • discover が false の場合、conn->oauth_token が設定されている必要があります。設定されていない場合はアサーションエラーが発生します。
  • 戻り値が NULL でない場合、呼び出し元は free() を使用してメモリを解放する責任があります。

アルゴリズムまたは振る舞い

1. discover が true の場合、認証スキームとトークンを空文字列として設定します。 2. discover が false の場合、認証スキームを "Bearer " に設定し、conn->oauth_token を使用します。 3. n,,auth=%s%s という形式（RFC 7628 準拠）でバッファを構築します。 4. 構築したバッファを strdup で複製して返します。 5. メモリ不足や内部エラー発生時は、libpq_append_conn_error を呼び出し NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-oauth.c 内の認証フロー処理（client_initial_response への参照が確認されています）。
  • **呼び出し先**: initPQExpBuffer, appendPQExpBuffer, termPQExpBuffer, libpq_append_conn_error 等の libpq 内部ユーティリティ。

関連データ構造

  • PGconn: 接続情報を保持する構造体。oauth_token フィールドを使用。
  • PQExpBufferData: 応答文字列構築用の一時バッファ。

失敗モードと境界条件

  • conn->oauth_token が未設定で discover が false の場合: アサーション失敗およびエラーメッセージの追加。
  • メモリ割り当て失敗時: NULL を返し、conn にエラーメッセージを設定。

保守担当者向けメモ

  • 本関数は RFC 7628 に基づく SASL 応答形式を生成します。形式を変更する場合は resp_format 定数を参照してください。
  • discover モードはサーバーからメタデータを取得するための特殊なケースであり、認証トークンを伴わない空の応答を生成します。
