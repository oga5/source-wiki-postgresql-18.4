ginFindParents

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbtree.c:217

目的

GINインデックスのツリー構造において、特定のページへのパスが失われた際に、ルートから再探索を行い、親ページへのスタックを再構築します。

シグネチャまたは形状

static void ginFindParents(GinBtree btree, GinBtreeStack *stack)

入力と出力

  • **入力**:
  • btree: GIN B-treeの操作コンテキスト。
  • stack: 現在のパス情報を持つスタック構造体（更新対象）。
  • **出力**: なし（副作用として stack の親ポインタを更新し、正しいパスを再構築する）。

前提条件と不変条件

  • ルートページに対するピン（pin）は保持されている必要があります（同時実行されるバキューム処理を排除するため）。
  • 処理開始時、スタックはルートまで巻き戻されます。

アルゴリズムまたは振る舞い

1. **スタックの巻き戻し**: ルートページ以外のスタック要素を解放し、ルートのみを残します。 2. **再探索**: ルートから開始し、btree->findChildPtr を用いてターゲットページへのパスを辿ります。 3. **不完全な分割の修復**: 探索中に GIN_INCOMPLETE_SPLIT フラグが設定されたページに遭遇した場合、ginFinishOldSplit を呼び出して分割処理を完了させます。 4. **右兄弟の追跡**: 現在のレベルでターゲットが見つからない場合、ginStepRight を使用して右兄弟へ移動します。 5. **スタックの更新**: ターゲットページを発見すると、新しいスタック要素を作成し、stack->parent に接続して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbtree.c 内の ginFinishSplit（再帰的呼び出し）およびその他の内部ルーチン。
  • **呼び出し先**:
  • ginFinishOldSplit: 不完全な分割の修復。
  • ginStepRight: 右兄弟への移動。
  • btree->getLeftMostChild / btree->findChildPtr: インデックス型固有の探索ロジック。
  • ReleaseBuffer, LockBuffer, ReadBuffer: バッファ管理。

関連データ構造

  • GinBtree: GIN B-treeの操作状態を保持する構造体。
  • GinBtreeStack: ツリーのパスを保持するスタック構造体。

失敗モードと境界条件

  • **Lost path**: リーフページに到達してもターゲットが見つからない場合、elog(ERROR) を発生させます。
  • **ルートのピン**: ルートページのピンは解放されません。
  • **不完全な分割**: 探索中に遭遇した不完全な分割は、再帰的に ginFindParents を呼び出す可能性があります。

保守担当者向けメモ

  • 本関数は、ツリー構造の整合性を維持するための重要なリカバリ手段です。
  • ginFinishOldSplit 呼び出し時の引数として渡されるスタックは、再帰的な修正を前提としています。
  • ptr->parent = root の設定は暫定的なものであり、後続の呼び出しで修正される可能性がある点に注意してください。
