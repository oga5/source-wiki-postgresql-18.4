gss_read

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-gssapi.c:434

目的

GSSAPI 通信において、基盤となるソケットからデータを読み込み、読み込み状態（成功、待機、失敗）を管理する。

シグネチャまたは形状

PostgresPollingStatusType gss_read(PGconn *conn, void *recv_buffer, size_t length, ssize_t *ret)

入力と出力

  • **入力**:
  • conn: PostgreSQL 接続オブジェクト。
  • recv_buffer: 読み込み先バッファ。
  • length: 読み込むバイト数。
  • ret: 読み込んだバイト数（またはエラーコード）を格納するポインタ。
  • **出力**:
  • 戻り値: PostgresPollingStatusType 型（PGRES_POLLING_OK, PGRES_POLLING_READING, PGRES_POLLING_FAILED のいずれか）。

前提条件と不変条件

  • pqsecure_raw_read が呼び出し可能であること。
  • ret ポインタは有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. pqsecure_raw_read を呼び出し、ソケットからデータを読み込む。 2. 読み込み結果が負の場合、EAGAIN, EWOULDBLOCK, EINTR であれば PGRES_POLLING_READING を返し、それ以外は PGRES_POLLING_FAILED を返す。 3. 読み込み結果が 0（EOF）の場合、pqReadReady を用いて接続状態を確認する。

  • 準備ができていない場合は PGRES_POLLING_READING を返す。
  • 再度 pqsecure_raw_read を試行し、結果が 0 であれば PGRES_POLLING_FAILED を返す。

    • 4. 正常に読み込めた場合は PGRES_POLLING_OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure-gssapi.c 内の複数箇所（パケットヘッダーやペイロードの読み込み時）。
  • **呼び出し先**: pqsecure_raw_read, pqReadReady。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。

失敗モードと境界条件

  • ソケットエラー発生時: PGRES_POLLING_FAILED を返す。
  • EOF 到達時: pqReadReady による再確認を行い、それでもデータがない場合は PGRES_POLLING_FAILED を返す。
  • 非ブロッキング I/O: EAGAIN 等の発生時は PGRES_POLLING_READING を返し、呼び出し元に再試行を促す。

保守担当者向けメモ

  • 本関数は暗号化されたストリームの「生の読み込み」を担当する低レイヤー関数である。
  • 復号処理（gss_unwrap）は本関数の呼び出し元で行われるため、本関数自体はパケットの断片化や再構築のロジックを含まない。
