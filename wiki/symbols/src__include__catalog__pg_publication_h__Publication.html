Publication

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/pg_publication.h:127

目的

論理レプリケーションにおけるパブリケーションの情報を、バックエンドのメモリ上で保持するためのデータ構造です。システムカタログ pg_publication の行情報を変換し、レプリケーションロジックで利用可能な形式として提供します。

シグネチャまたは形状

typedef struct Publication
{
    Oid            oid;
    char          *name;
    bool           alltables;
    bool           pubviaroot;
    PublishGencolsType pubgencols_type;
    PublicationActions pubactions;
} Publication;

入力と出力

  • **入力**: GetPublication や GetPublicationByName などの関数を通じて、システムカタログから読み取られたデータが本構造体に格納されます。
  • **出力**: レプリケーションの対象テーブルの判定、列フィルタリングの検証、および DML 操作の制御ロジックへ渡されます。

前提条件と不変条件

  • 本構造体は palloc によってメモリ確保されることが想定されています。
  • pubactions フィールドは、パブリケーションが許可する DML 操作（INSERT, UPDATE, DELETE, TRUNCATE）の有効/無効状態を保持します。

アルゴリズムまたは振る舞い

1. **初期化**: GetPublication 等の関数がカタログからタプルを取得し、本構造体の各フィールドに値を割り当てます。 2. **変換**: カタログ上の物理的な行データが、メモリ上の Publication 構造体へ変換されます。 3. **利用**: レプリケーションの実行時、この構造体を参照して、特定のテーブルがパブリケーションに含まれるか、どの操作が許可されているか、列リストの制約がどうなっているかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c: オブジェクトアドレスの管理に関連して参照。
  • src/backend/catalog/pg_publication.c: パブリケーションの取得、列リストの検証、およびレプリケーションロジックの実行時に参照。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありませんが、PublicationActions や PublishGencolsType といった関連型を内部に保持します。

関連データ構造

  • PublicationActions: DML 操作の有効/無効フラグを保持。
  • PublishGencolsType: 生成列の扱いを定義する列挙型。
  • FormData_pg_publication: システムカタログ pg_publication の物理行構造体。

失敗モードと境界条件

  • メモリ不足時には palloc が失敗する可能性があります。
  • 存在しないパブリケーション名や OID を指定して取得を試みた場合、呼び出し側の関数（GetPublication 等）がエラーを発生させるか、NULL を返す可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/catalog/pg_publication.c 内で動的に生成・破棄されるため、メモリ管理（palloc）のライフサイクルに注意してください。
  • パブリケーションの列フィルタリングやパーティションの扱いに関する変更を行う際は、本構造体のフィールド定義と、それを利用する pg_publication.c 内の検証ロジックを併せて確認してください。
