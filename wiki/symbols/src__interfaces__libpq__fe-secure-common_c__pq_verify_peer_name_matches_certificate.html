pq_verify_peer_name_matches_certificate

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-common.c:251

目的

SSL/TLS接続において、サーバー証明書内の識別名（DNS名またはIPアドレス）が接続先ホスト名と一致するかを検証する。

シグネチャまたは形状

bool pq_verify_peer_name_matches_certificate(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn （接続状態を保持する構造体）
  • **出力**: bool （検証成功時に true、失敗時に false を返す）

前提条件と不変条件

  • conn->sslmode が "verify-full" である場合にのみ検証を実行する。それ以外の場合は常に true を返す。
  • 接続先ホスト名（conn->connhost[...].host）が指定されている必要がある。空の場合はエラーとなる。

アルゴリズムまたは振る舞い

1. sslmode が "verify-full" でない場合は直ちに true を返す。 2. ホスト名が未指定または空文字の場合、libpq_append_conn_error を呼び出し false を返す。 3. pgtls_verify_peer_name_matches_certificate_guts を呼び出し、証明書内の名前情報を抽出・照合する。 4. 照合に失敗した場合（rc == 0）、names_examined（検証した名前の数）に基づき、conn->errorMessage に詳細なエラーメッセージを構築する。

  • 複数の名前がある場合は最初の名前のみを表示する。

    • 5. 最後に first_name を解放し、照合結果（rc == 1）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure-openssl.c:1491
  • **呼び出し先**:
  • pgtls_verify_peer_name_matches_certificate_guts （SSLライブラリ固有の検証ロジック）
  • libpq_append_conn_error （エラーメッセージの記録）
  • appendPQExpBuffer / appendPQExpBufferChar （バッファ操作）
  • free （メモリ解放）

関連データ構造

  • PGconn: 接続情報およびエラーメッセージバッファを保持。

失敗モードと境界条件

  • **ホスト名欠如**: ホスト名が指定されていない場合、検証を中断しエラーを返す。
  • **検証不一致**: 証明書内の名前とホスト名が一致しない場合、false を返し、エラーメッセージに証明書内の名前を含める。
  • **名前取得失敗**: 証明書からホスト名を取得できなかった場合、専用のエラーメッセージを生成する。

保守担当者向けメモ

  • 本関数は verify-full モードのメインエントリポイントである。
  • 実際の証明書抽出ロジックは pgtls_verify_peer_name_matches_certificate_guts に委譲されているため、SSLライブラリの追加・変更時はそちらを確認すること。
  • エラーメッセージ生成時に libpq_ngettext を使用しており、複数名存在する場合の文言対応が考慮されている。
