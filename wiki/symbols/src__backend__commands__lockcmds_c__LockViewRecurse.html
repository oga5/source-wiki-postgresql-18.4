LockViewRecurse

定義

  • 種別: function
  • 定義箇所: src/backend/commands/lockcmds.c:244

目的

ビューの定義（クエリツリー）を再帰的に走査し、そのビューが依存しているすべてのリレーションに対してロックを適用する。

シグネチャまたは形状

static void LockViewRecurse(Oid reloid, LOCKMODE lockmode, bool nowait, List *ancestor_views)

入力と出力

  • reloid: ロック対象のビューの OID。
  • lockmode: 適用するロックモード。
  • nowait: true の場合、ロック取得時に待機せずエラーを発生させる。
  • ancestor_views: 循環参照を防ぐために追跡している親ビューの OID リスト。
  • 出力: なし（副作用としてリレーションのロックを実行）。

前提条件と不変条件

  • 呼び出し元が、対象のビュー（reloid）に対して既に適切なロックを保持していること。
  • ビューの再帰処理中、ancestor_views により循環参照が管理されていること。

アルゴリズムまたは振る舞い

1. table_open を使用してビューを開く（NoLock モード）。 2. get_view_query でビューのクエリ定義を取得する。 3. ビューの security_invoker プロパティを確認し、権限チェックを行うユーザー（現在のユーザーまたはビュー所有者）を決定する。 4. LockViewRecurse_context を初期化し、現在のビューを ancestor_views に追加する。 5. LockViewRecurse_walker を呼び出し、クエリツリー内の全リレーションに対して再帰的にロックを適用する。 6. 処理終了後、ancestor_views から自身の OID を削除し、table_close でビューを閉じる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/lockcmds.c 内の LockTableCommand および LockViewRecurse_walker（再帰呼び出し）。
  • 呼び出し先: table_open, get_view_query, RelationHasSecurityInvoker, GetUserId, lappend_oid, LockViewRecurse_walker, list_delete_last, table_close。

関連データ構造

  • LockViewRecurse_context: 再帰走査中に必要なロックモード、権限チェック対象ユーザー、ビューの OID、および祖先ビューのリストを保持する構造体。

失敗モードと境界条件

  • ビューが循環参照を含んでいる場合、ancestor_views による追跡で保護される。
  • nowait が true の場合、依存リレーションのロック取得に失敗するとエラーが発生する。

保守担当者向けメモ

  • 本関数は query_tree_walker を利用した走査を行うため、ビュー定義の変更や新しいノードタイプの追加時には LockViewRecurse_walker の更新が必要となる可能性がある。
