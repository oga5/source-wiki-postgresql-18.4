SetDataDirectoryCreatePerm

定義

  • 種別: function
  • 定義箇所: src/common/file_perm.c:33

目的

PostgreSQLのデータディレクトリ（PGDATA）配下で新規作成されるファイルやディレクトリのアクセス権限（パーミッション）および umask 用のマスク値を、指定されたモードに基づいて一元的に設定する。

シグネチャまたは形状

void SetDataDirectoryCreatePerm(int dataDirMode)

入力と出力

  • **入力**: dataDirMode (int) - 基準となるディレクトリのモード（stat 構造体の st_mode 等）。
  • **出力**: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • グローバル変数 pg_dir_create_mode、pg_file_create_mode、pg_mode_mask が本関数によって更新される。
  • 入力された dataDirMode にグループ権限が含まれているかどうかが、設定値を決定する唯一の判定基準となる。

アルゴリズムまたは振る舞い

1. 入力された dataDirMode と PG_DIR_MODE_GROUP のビット論理積を評価する。 2. グループ権限が有効な場合:

  • pg_dir_create_mode を PG_DIR_MODE_GROUP に設定。
  • pg_file_create_mode を PG_FILE_MODE_GROUP に設定。
  • pg_mode_mask を PG_MODE_MASK_GROUP に設定。

    • 3. グループ権限が無効な場合:

  • pg_dir_create_mode を PG_DIR_MODE_OWNER に設定。
  • pg_file_create_mode を PG_FILE_MODE_OWNER に設定。
  • pg_mode_mask を PG_MODE_MASK_OWNER に設定。

  • **呼び出し先**:
  • ソースコード上での直接的な関数呼び出しは確認されません。グローバル変数への代入のみを行います。

関連データ構造

  • pg_dir_create_mode (グローバル変数)
  • pg_file_create_mode (グローバル変数)
  • pg_mode_mask (グローバル変数)

失敗モードと境界条件

  • Windows環境では、ファイルシステムの違いにより権限設定の自動適用がスキップされる場合がある（モジュール概要による）。
  • 入力値 dataDirMode が不正な場合でも、関数はビットマスクによる判定を継続する。

保守担当者向けメモ

  • 本関数はフロントエンドおよびバックエンドの両方から参照されるため、変更時は両環境への影響を考慮すること。
  • PG_DIR_MODE_GROUP 等の定数は src/include/common/file_perm.h で定義されている。
