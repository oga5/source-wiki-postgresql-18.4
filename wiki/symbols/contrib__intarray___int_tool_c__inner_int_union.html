inner_int_union

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:78

目的

2つの整数配列（ArrayType）の和集合を計算し、新しい配列として返します。

シグネチャまたは形状

ArrayType *inner_int_union(ArrayType *a, ArrayType *b)

入力と出力

  • **入力**:
  • a, b: ソート済みの ArrayType ポインタ。
  • **出力**:
  • 和集合の結果を格納した新しい ArrayType ポインタ。メモリは palloc されています。

前提条件と不変条件

  • 入力される a および b は、CHECKARRVALID マクロにより有効な整数配列であることが保証されている必要があります。
  • 入力配列はソート済みである必要があります。ソートされていない場合、正しい和集合は生成されません。

アルゴリズムまたは振る舞い

1. 入力配列が空の場合、他方の配列のコピーを返すか、両方空なら空の配列を返します。 2. 両方の配列が空でない場合、na + nb の領域を確保し、2つのポインタを同時に走査してマージ処理を行います。 3. マージ中、値が一致する場合は片方のみを結果に含め、異なる場合は小さい方の値を結果に追加します。 4. マージ完了後、resize_intArrayType を呼び出して実際の要素数に合わせてメモリを縮小します。 5. 結果の要素数が1より大きい場合、_int_unique を呼び出して重複を排除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c (GiST インデックス操作)
  • contrib/intarray/_int_op.c (SQL 演算子実装)
  • **呼び出し先**:
  • CHECKARRVALID (マクロ)
  • new_intArrayType
  • copy_intArrayType
  • resize_intArrayType
  • _int_unique

関連データ構造

  • ArrayType: PostgreSQL の標準的な配列構造体。

失敗モードと境界条件

  • 入力配列がソートされていない場合、結果は不正確になります。
  • メモリ不足時には palloc がエラーを発生させます。
  • 空配列の入力は適切に処理されます。

保守担当者向けメモ

  • 本関数は $O(N+M)$ の線形時間で動作します。この計算量を維持してください。
  • 返される配列は palloc されているため、呼び出し側でメモリ管理（pfree 等）を行う必要があります。
