CheckPointLogicalRewriteHeap

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/rewriteheap.c:1154

目的

論理レプリケーションで使用されるヒープ再構築時のマッピングファイル（map-*）を管理し、不要なファイルを削除し、必要なファイルをディスクへ同期（fsync）します。

シグネチャまたは形状

void CheckPointLogicalRewriteHeap(void)

入力と出力

  • **入力**: なし（内部的に ReplicationSlotsComputeLogicalRestartLSN() 等のグローバル状態を参照）
  • **出力**: なし（副作用として PG_LOGICAL_MAPPINGS_DIR 内のファイル操作を実行）

前提条件と不変条件

  • 呼び出し元は、チェックポイント処理の一部として本関数を実行している必要があります。
  • システム全体で同時に実行されるチェックポイントは一つのみであるため、ファイル削除処理における競合は発生しません。
  • 対象テーブルに対して AccessExclusiveLock が保持されている環境下での実行を想定しています。

アルゴリズムまたは振る舞い

1. **カットオフの決定**: GetRedoRecPtr() と論理レプリケーションスロットの最小再開LSNを比較し、削除基準となる cutoff LSN を算出します。 2. **ディレクトリ走査**: PG_LOGICAL_MAPPINGS_DIR 内の全ファイルを順次走査します。 3. **ファイル判定**: map- で始まるファイル名を解析し、ファイル名に含まれるLSN情報を抽出します。 4. **削除または同期**:

  • ファイルのLSNが cutoff より小さい場合、不要と判断して unlink() で削除します。
  • それ以外の場合、pg_fsync() を呼び出し、データの永続化を保証します。

    • 5. **ディレクトリ同期**: 最後に fsync_fname() を呼び出し、ディレクトリ自体の変更をディスクへ反映します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c (チェックポイント処理のメインフロー)
  • src/backend/access/heap/rewriteheap.c (コメント内での言及あり)
  • **呼び出し先**:
  • ReplicationSlotsComputeLogicalRestartLSN
  • AllocateDir, ReadDir, FreeDir
  • pg_fsync, fsync_fname
  • OpenTransientFile, CloseTransientFile

関連データ構造

  • PG_LOGICAL_MAPPINGS_DIR: マッピングファイルが格納されるディレクトリパス。
  • LOGICAL_REWRITE_FORMAT: ファイル名の解析に使用されるフォーマット文字列。

失敗モードと境界条件

  • **ファイル操作エラー**: unlink、open、fsync 等のシステムコールが失敗した場合、ereport(ERROR, ...) によりチェックポイント処理全体が中断されます。
  • **解析エラー**: ファイル名が期待するフォーマットと一致しない場合、elog(ERROR, ...) が発生します。
  • **空のLSN**: cutoff が InvalidXLogRecPtr の場合、すべてのマッピングファイルが保持対象となります。

保守担当者向けメモ

  • 本関数はチェックポイントの整合性を維持するために不可欠です。論理レプリケーションのクラッシュリカバリにおいて、マッピングファイルが正しく同期されていることが前提となります。
  • fsync の負荷を軽減する最適化の余地はありますが、現状では全対象ファイルの同期が優先されています。
