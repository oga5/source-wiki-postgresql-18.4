plperl_query_desc

定義

  • 種別: struct
  • 定義箇所: src/pl/plperl/plperl.c:188

目的

PL/Perl 内で SPI（Server Programming Interface）を使用して実行されるプリペアドステートメントのメタデータおよび実行状態を保持するために使用されます。

シグネチャまたは形状

typedef struct plperl_query_desc
{
    char        qname[24];
    MemoryContext plan_cxt;
    SPIPlanPtr  plan;
    int         nargs;
    Oid        *argtypes;
    FmgrInfo   *arginfuncs;
    Oid        *argtypioparams;
} plperl_query_desc;

入力と出力

  • **入力**: palloc0 を介してメモリが割り当てられ、SPI 実行に必要なクエリ情報や引数情報が格納されます。
  • **出力**: 構造体内の各フィールドは、SPI 実行時および引数の型変換時に参照されます。

前提条件と不変条件

  • plan_cxt は、この構造体自体を保持するメモリコンテキストを指す必要があります。
  • plan は SPI_prepare 等で生成された有効な SPIPlanPtr である必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、Perl 側から呼び出される spi_prepare 等の処理において、クエリのコンパイル結果と引数の型情報をキャッシュする役割を担います。
  • 実行時には、nargs に基づいて argtypes や arginfuncs を参照し、PostgreSQL のデータ型と Perl のデータ型間の変換を仲介します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plperl/plperl.c 内の SPI 関連関数（plperl_spi_prepare 等）からインスタンス化および参照されます。
  • **呼び出し先**: SPI API（SPI_execute_plan 等）および PostgreSQL の型変換関数（fmgr_info 等）が本構造体のフィールドを利用します。

関連データ構造

  • SPIPlanPtr: SPI 実行計画へのポインタ。
  • MemoryContext: 構造体の生存期間を管理するメモリコンテキスト。
  • FmgrInfo: 関数呼び出し情報。

失敗モードと境界条件

  • メモリ不足時に palloc0 が失敗する可能性があります。
  • nargs が実際の引数数と一致しない場合、型変換処理において不正なメモリ参照が発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は plperl_query_entry 等のハッシュテーブル管理用構造体と組み合わせて使用されることが多いため、キャッシュのライフサイクル管理には注意してください。
  • qname は固定長（24バイト）であるため、クエリ名の管理には制限があります。
