tlist_vinfo

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:41

目的

ターゲットリスト（TargetList）内の変数（Var）に関する情報を保持し、プランの後処理段階において、式内の変数参照をサブプランの出力位置へ効率的にマッピングするために使用されます。

シグネチャまたは形状

typedef struct {
    int         varno;            /* RT index of Var */
    AttrNumber  varattno;         /* attr number of Var */
    AttrNumber  resno;            /* TLE position of Var */
    Bitmapset  *varnullingrels;   /* Var's varnullingrels */
} tlist_vinfo;

入力と出力

本構造体はデータ保持用であり、関数ではありません。ターゲットリストの各要素を走査する際に、対応する変数の属性情報と出力位置（resno）を格納するために利用されます。

前提条件と不変条件

  • resno は、対応するターゲットリストエントリ（TLE）のインデックスを指します。
  • varnullingrels は、外部結合等で NULL になり得るリレーションの集合を保持します。

アルゴリズムまたは振る舞い

setrefs.c 内の処理において、ターゲットリストから indexed_tlist を構築する際、各変数の情報をこの構造体に詰め込みます。これにより、式ツリー内の変数を検索する際、リスト全体を線形探索する代わりに、この構造体を用いた高速な参照解決が可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/setrefs.c 内の関数（indexed_tlist を初期化する箇所など）。
  • **呼び出し先**: 構造体であるため、直接的な呼び出し先はありません。

関連データ構造

  • indexed_tlist: tlist_vinfo の配列を保持し、ターゲットリストの高速検索を可能にする構造体。

失敗モードと境界条件

  • ターゲットリストが空の場合、本構造体の配列も空となります。
  • varnullingrels が NULL の場合、その変数はどの外部結合の NULL 許容範囲にも含まれていないことを示します。

保守担当者向けメモ

  • tlist_vinfo は indexed_tlist の一部として動的にメモリ確保されることが多いため、list_length(tlist) に基づくサイズ計算の整合性に注意してください。
  • 変数の参照解決後にプランノードが書き換わるため、本構造体を使用してマッピングを行うタイミングは、プランの最終化プロセスにおいて極めて重要です。
