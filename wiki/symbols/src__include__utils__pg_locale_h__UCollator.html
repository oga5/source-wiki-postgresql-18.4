UCollator

定義

  • 種別: struct
  • 定義箇所: src/include/utils/pg_locale.h:134

目的

ICUライブラリにおける照合順序（Collation）処理をPostgreSQL内でカプセル化し、ロケール依存の文字列比較や変換を行うためのハンドルとして機能します。

シグネチャまたは形状

ICUライブラリの unicode/ucol.h で定義される不透明な構造体です。PostgreSQLの内部構造体（pg_locale_t 等）の一部として、以下の形式で保持されます。

struct {
    const char *locale;
    UCollator  *ucol;
} icu;

入力と出力

  • 入力: ロケール文字列（const char *loc_str）を基に生成されます。
  • 出力: UCollator * 型のポインタとして返され、以降の文字列比較操作の引数として利用されます。

前提条件と不変条件

  • ICUライブラリが有効なビルド環境であること。
  • pg_ucol_open 等のファクトリ関数を通じて適切に初期化されていること。
  • ライフサイクル管理が重要であり、メモリリークを防ぐために適切なタイミングで破棄される必要があります。

アルゴリズムまたは振る舞い

  • ICUの照合エンジンへのインターフェースを提供します。
  • pg_locale_t のメソッドテーブルを通じて、strncoll_icu などの関数から間接的に呼び出され、ロケールに応じた文字列比較ロジックを実行します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/pg_locale.c および src/backend/utils/adt/pg_locale_icu.c 内の関数群（pg_ucol_open, make_icu_collator, icu_set_collation_attributes など）。
  • 呼び出し先: ICUライブラリの関数（ucol_open 等、unicode/ucol.h 経由）。

関連データ構造

  • pg_locale_t: ロケール情報を保持する上位構造体。
  • collate_methods: 照合順序の挙動を定義する関数ポインタテーブル。

失敗モードと境界条件

  • ロケール文字列が不正な場合、pg_ucol_open はNULLを返す可能性があります。
  • 実行時にメモリ不足が発生した場合、ICUライブラリの動作に依存します。
  • 適切な破棄が行われない場合、メモリリークが発生します。

保守担当者向けメモ

  • 本シンボルはICUライブラリのC APIを直接ラップしています。ICUのバージョンアップやAPI変更の影響を直接受けるため、変更時は src/backend/utils/adt/pg_locale_icu.c の実装を確認してください。
  • U_SHOW_CPLUSPLUS_API を無効化してC APIのみを露出させているため、C++固有の機能は利用できません。
