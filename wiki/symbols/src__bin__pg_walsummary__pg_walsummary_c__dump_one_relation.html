dump_one_relation

定義

  • 種別: function
  • 定義箇所: src/bin/pg_walsummary/pg_walsummary.c:127

目的

特定のリレーションに関連するブロック情報を WAL サマリファイルから取得し、整形して標準出力へ表示する。

シグネチャまたは形状

static void dump_one_relation(ws_options *opt, RelFileLocator *rlocator,
                              ForkNumber forknum, BlockNumber limit_block,
                              BlockRefTableReader *reader)

入力と出力

  • **入力**:
  • opt: 実行オプション（quiet, individual フラグを含む）。
  • rlocator: 対象リレーションの識別情報。
  • forknum: 対象のフォーク番号。
  • limit_block: リミットブロック番号（存在する場合）。
  • reader: ブロック参照テーブルのリーダーインスタンス。
  • **出力**: なし（標準出力への書き込みのみ）。

前提条件と不変条件

  • block_buffer および block_buffer_size はグローバルまたはファイルスコープで管理される状態変数である。
  • reader は有効な状態であること。
  • メモリ割り当て失敗時は PostgreSQL の標準的なエラーハンドリングによりプロセスが終了する。

アルゴリズムまたは振る舞い

1. **リミットブロック出力**: limit_block が有効な場合、リレーション情報と共に表示する。 2. **バッファ確保**: block_buffer が未初期化であれば確保し、BlockRefTableReaderGetBlocks でブロック番号を読み込む。 3. **動的拡張**: 読み込んだブロック数がバッファ容量に達した場合、容量を倍増させながら全ブロックを読み込むまで繰り返す。 4. **静的チェック**: opt->quiet が真であれば、出力を行わず終了する。 5. **ソート**: compare_block_numbers を用いてブロック番号を昇順にソートする。 6. **範囲集約と出力**:

  • opt->individual が偽の場合、連続するブロック番号を start..end 形式に集約して出力する。
  • opt->individual が真の場合、各ブロックを個別に表示する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_walsummary/pg_walsummary.c 内のループ処理（BlockRefTableReaderNextRelation の反復内）。
  • **呼び出し先**: BlockRefTableReaderGetBlocks, qsort, printf, palloc_array, repalloc_array。

関連データ構造

  • ws_options: 実行時設定。
  • RelFileLocator: リレーションの識別子。
  • BlockRefTableReader: ブロック参照テーブルの読み込み用ハンドル。
  • block_buffer: ブロック番号を一時保持する動的配列。

失敗モードと境界条件

  • block_buffer の拡張時にメモリ不足が発生した場合、repalloc_array がエラーを発生させる。
  • ブロック参照テーブルの表現形式（ビットマップまたはオフセット配列）により、読み込み時の順序が異なるが、qsort により正規化される。

保守担当者向けメモ

  • 本関数は BlockRefTableReader のクライアント実装として機能する。
  • block_buffer のサイズ管理は本関数とグローバル変数に依存しているため、並列実行や再入が必要な場合は設計変更が必要となる。
