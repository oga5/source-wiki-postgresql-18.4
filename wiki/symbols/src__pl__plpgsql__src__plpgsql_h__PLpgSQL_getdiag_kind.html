PLpgSQL_getdiag_kind

定義

  • 種別: enum
  • 定義箇所: src/pl/plpgsql/src/plpgsql.h:148

目的

PL/pgSQL の GET DIAGNOSTICS 文において、取得対象とする診断情報の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum PLpgSQL_getdiag_kind
{
    PLPGSQL_GETDIAG_ROW_COUNT,
    PLPGSQL_GETDIAG_ROUTINE_OID,
    PLPGSQL_GETDIAG_CONTEXT,
    PLPGSQL_GETDIAG_ERROR_CONTEXT,
    PLPGSQL_GETDIAG_ERROR_DETAIL,
    PLPGSQL_GETDIAG_ERROR_HINT,
    PLPGSQL_GETDIAG_RETURNED_SQLSTATE,
    PLPGSQL_GETDIAG_COLUMN_NAME,
    PLPGSQL_GETDIAG_CONSTRAINT_NAME,
    PLPGSQL_GETDIAG_DATATYPE_NAME,
    PLPGSQL_GETDIAG_MESSAGE_TEXT,
    PLPGSQL_GETDIAG_TABLE_NAME,
    PLPGSQL_GETDIAG_SCHEMA_NAME,
} PLpgSQL_getdiag_kind;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • 各列挙子は、PL/pgSQL の実行エンジンが診断情報を抽出する際のキーとして一意に割り当てられています。

アルゴリズムまたは振る舞い

  • 本列挙型は、GET DIAGNOSTICS 文の構文解析および実行時に、どの診断項目を要求されているかを特定するための識別子として機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/pl/plpgsql/src/plpgsql.h:600: 診断値の要求を保持する構造体メンバとして使用。
  • src/pl/plpgsql/src/pl_funcs.c:300: plpgsql_getdiag_kindname 関数の引数として使用。
  • 呼び出し先: なし（列挙型定義のため）

関連データ構造

  • PLpgSQL_stmt: 本列挙型をメンバとして保持する可能性のある文の基底構造体。

失敗モードと境界条件

  • 定義外の値をキャストして使用した場合の動作は未定義です。

保守担当者向けメモ

  • 新しい診断項目を GET DIAGNOSTICS に追加する場合は、本列挙型に項目を追加し、対応する plpgsql_getdiag_kindname 関数（pl_funcs.c）にも文字列変換ロジックを追加してください。
