ECPGset_desc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/descriptor.c:612

目的

SQL 記述子（Descriptor）の指定されたインデックスに対して、アプリケーションから属性値やデータを設定します。

シグネチャまたは形状

bool ECPGset_desc(int lineno, const char *desc_name, int index, ...);

可変長引数（va_list）を使用して、設定対象の属性タイプ（enum ECPGdtype）とそれに対応する値を順次受け取ります。

入力と出力

  • **入力**:
  • lineno: エラー報告用行番号。
  • desc_name: 記述子名。
  • index: 設定対象の項目番号。
  • ...: ECPGd_data, ECPGd_indicator, ECPGd_length 等の属性タイプと、それに対応する値のペア。
  • **出力**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • ecpg_find_desc により、指定された desc_name が現在のスレッドにおいて有効である必要があります。
  • 可変長引数は ECPGd_EODT で終了する必要があります。
  • 内部で ecpg_alloc を使用してメモリを確保するため、呼び出し側はメモリ不足時の失敗を考慮する必要があります。

アルゴリズムまたは振る舞い

1. ecpg_find_desc で対象の記述子を検索します。 2. 指定された index に対応する descriptor_item を検索し、存在しない場合は新規作成して記述子リストに追加します。 3. 可変長引数をループで解析し、itemtype に応じて以下の処理を行います：

  • ECPGd_data: ecpg_store_input を経由してデータを変換し、set_desc_attr で属性を更新します。
  • その他の属性（indicator, length, precision, scale, type）: set_int_item を使用して値を設定します。

    • 4. 処理終了後、一時的に確保したメモリを解放し、va_end を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPG プリプロセッサが生成したコード（src/interfaces/ecpg/preproc/descriptor.c 等）から呼び出されます。
  • **呼び出し先**:
  • ecpg_find_desc
  • ecpg_alloc / ecpg_free
  • ecpg_store_input
  • set_desc_attr
  • set_int_item
  • ecpg_raise

関連データ構造

  • struct descriptor: 記述子全体を管理する構造体。
  • struct descriptor_item: 記述子内の各項目（列）の属性を保持する構造体。
  • struct variable: 変換対象の変数の型や値を保持する一時構造体。

失敗モードと境界条件

  • 記述子が見つからない場合、false を返します。
  • メモリ確保に失敗した場合、false を返します。
  • 未知の itemtype が渡された場合、ECPG_UNKNOWN_DESCRIPTOR_ITEM エラーを発生させ false を返します。
  • index が既存の記述子項目数を超えた場合、desc->count が更新されます。

保守担当者向けメモ

  • 可変長引数の解析順序は ECPG プリプロセッサの生成コードと厳密に一致している必要があります。
  • LC_NUMERIC の影響を受ける可能性があるため、数値変換時は注意が必要です。
  • スレッドローカルストレージを利用しているため、マルチスレッド環境での動作はスレッドごとの記述子管理に依存します。
