EventTriggerQueryState

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/event_trigger.c:61

目的

クエリ実行中におけるイベントトリガーの状態を保持・管理するための構造体です。DDLコマンドの収集、sql_drop イベントの追跡、およびテーブル再構築イベントの情報を保持します。

シグネチャまたは形状

typedef struct EventTriggerQueryState
{
    MemoryContext cxt;
    slist_head    SQLDropList;
    bool          in_sql_drop;
    Oid           table_rewrite_oid;
    int           table_rewrite_reason;
    bool          commandCollectionInhibited;
    CollectedCommand *currentCommand;
    List         *commandList;
    struct EventTriggerQueryState *previous;
} EventTriggerQueryState;

入力と出力

  • **入力**: クエリ開始時に MemoryContext を介して初期化されます。
  • **出力**: 実行されたDDLコマンドのリスト（commandList）や、削除対象オブジェクトのリスト（SQLDropList）を保持し、イベントトリガー関数から参照可能にします。

前提条件と不変条件

  • 実行は専用のメモリコンテキスト（cxt）で行われ、クエリ終了時に解放されます。
  • 再帰的なDDL実行を考慮し、previous ポインタによってスタック構造を形成します。
  • スタック深さのチェック（check_stack_depth）が再帰呼び出しの安全性を担保します。

アルゴリズムまたは振る舞い

1. **初期化**: クエリ開始時に EventTriggerBeginCompleteQuery によって生成され、currentEventTriggerState に割り当てられます。 2. **収集**: DDL実行中、EventTriggerCollectSimpleCommand 等を通じて commandList にコマンド情報が蓄積されます。 3. **状態保持**: in_sql_drop フラグや table_rewrite_oid を用いて、現在処理中のイベント種別を追跡します。 4. **終了**: クエリ終了時に EventTriggerEndCompleteQuery が呼ばれ、メモリコンテキストの解放とともに状態がクリーンアップされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/event_trigger.c 内のクエリライフサイクル管理関数（EventTriggerBeginCompleteQuery 等）。
  • **呼び出し先**: 具体的な関数呼び出し関係はソース抜粋からは特定できません。

関連データ構造

  • CollectedCommand: commandList に格納されるDDLコマンドの詳細情報。
  • slist_head: SQLDropList を管理するための連結リスト。
  • MemoryContext: 状態保持のためのメモリ管理単位。

失敗モードと境界条件

  • **再帰的呼び出し**: イベントトリガー関数自体がDDLを実行する場合、スタックオーバーフローのリスクがあるため、スタック深さの監視が必要です。
  • **スタンドアロンモード**: シングルユーザーモードではイベントトリガーは無効化されます。

保守担当者向けメモ

  • 本構造体は currentEventTriggerState 静的変数を通じてグローバルに参照されます。
  • 状態の追加や変更を行う際は、メモリコンテキストのライフサイクルと整合性が取れていることを確認してください。
  • 呼び出し元と呼び出し先の詳細な依存関係は、現在利用可能なソース情報からは不完全です。
