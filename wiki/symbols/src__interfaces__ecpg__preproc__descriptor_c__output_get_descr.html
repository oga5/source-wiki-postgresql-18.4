output_get_descr

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:174

目的

SQLの GET DESCRIPTOR 文を、ECPGランタイムライブラリが実行可能なCコード（ECPGget_desc 呼び出し）へ変換して出力する。

シグネチャまたは形状

void output_get_descr(const char *desc_name, const char *index)

入力と出力

  • **入力**:
  • desc_name: 記述子名を表す文字列。
  • index: 記述子のインデックスを表す文字列。
  • assignments (グローバル変数): 構文解析中に蓄積された記述子操作項目のリスト。
  • **出力**:
  • base_yyout (外部ストリーム): 生成されたCコードが書き込まれる。

前提条件と不変条件

  • assignments リストが適切に初期化・蓄積されていること。
  • base_yyout が書き込み可能な状態であること。
  • 処理終了時に drop_assignments() が呼び出され、一時的なリストが解放されること。

アルゴリズムまたは振る舞い

1. ECPGget_desc の関数呼び出しヘッダーを出力する。 2. グローバルな assignments リストを走査する。

  • 各項目に対して find_variable で変数情報を取得する。
  • ECPGd_nullable や ECPGd_key_member の場合は警告を発行する。
  • get_dtype で型情報を取得し、ECPGdump_a_type を用いて変数の型情報を出力ストリームに書き出す。

    • 3. drop_assignments() を呼び出し、リストをクリーンアップする。 4. ECPGd_EODT を出力して関数呼び出しを閉じる。 5. whenever_action(2 | 1) を呼び出し、エラーハンドリングを設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: 構文解析器（パーサー）から呼び出される（具体的な呼び出し箇所はソースコードの構文解析ロジックに依存）。
  • **呼び出し先**:
  • find_variable
  • get_dtype
  • ECPGdump_a_type
  • drop_assignments
  • whenever_action
  • mmerror

関連データ構造

  • struct assignment: 記述子操作項目を保持するリスト構造。
  • struct variable: 変数情報を保持する構造体。

失敗モードと境界条件

  • assignments リストが空の場合、ECPGget_desc の引数リストが不完全になる可能性がある。
  • find_variable が該当する変数を見つけられない場合、NULLポインタ参照が発生する可能性がある。

保守担当者向けメモ

  • 本関数は assignments リストを消費する設計であるため、呼び出し順序が重要である。
  • ECPGdump_a_type に渡される str_zero は一時的なメモリ確保であるため、ループ内で適切に free されている。
