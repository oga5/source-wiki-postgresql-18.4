目的

ECPG（Embedded SQL for C）のプリプロセッサテストにおいて、SQL文のインジケータ変数として利用される共用体型を定義します。

シグネチャまたは形状

typedef union { int integer; short smallint; } ind;

入力と出力

  • **入力**: int または short 型の値を保持可能。
  • **出力**: データベース操作において、対応するホスト変数のNULL状態やステータスを示すインジケータ値として機能します。

前提条件と不変条件

  • ECPGの EXEC SQL TYPE 指令によって生成された型定義です。
  • 共用体であるため、integer と smallint はメモリ領域を共有します。

アルゴリズムまたは振る舞い

  • SQL文の実行時に、ホスト変数とペアで渡されるインジケータ変数として使用されます。
  • データベースから取得した値がNULLであるか、あるいは切り捨てが発生したか等の情報を保持するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/preproc-variable.c 内の ECPGdo 呼び出しにおいて、インジケータ変数として参照されます。
  • **呼び出し先**: 情報なし。
  • **注意**: ツールが提示した src/backend/bootstrap/bootstrap.c や src/backend/commands/analyze.c 内の ind は、本シンボルとは異なる文脈（インデックス関連）の変数であり、本シンボルとは無関係です。

失敗モードと境界条件

  • 共用体であるため、integer メンバと smallint メンバを同時に使用することはできません。誤ったメンバへのアクセスはデータの不整合を招きます。
