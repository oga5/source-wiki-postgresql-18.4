FillPortalStore

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:994

目的

ポータルの実行結果を Tuplestore に蓄積し、後続の読み出しに備えるための準備と実行を行う。

シグネチャまたは形状

static void FillPortalStore(Portal portal, bool isTopLevel)

入力と出力

  • **入力**:
  • portal: 実行対象のポータル構造体。
  • isTopLevel: 呼び出しがトップレベルのトランザクションかを示すフラグ。
  • **出力**: なし（副作用として portal->holdStore に結果が格納される）。

前提条件と不変条件

  • portal->strategy が PORTAL_ONE_RETURNING、PORTAL_ONE_MOD_WITH、または PORTAL_UTIL_SELECT のいずれかであること。
  • PortalCreateHoldStore(portal) が呼び出され、保持用ストアが初期化可能であること。

アルゴリズムまたは振る舞い

1. PortalCreateHoldStore を呼び出し、ポータル内に結果保持用のストアを作成する。 2. DestTuplestore 型の DestReceiver を作成し、SetTuplestoreDestReceiverParams でストアとコンテキストを紐付ける。 3. portal->strategy に応じて実行関数を切り替える:

  • PORTAL_ONE_RETURNING / PORTAL_ONE_MOD_WITH: PortalRunMulti を呼び出し、主クエリの結果を Tuplestore に送る。
  • PORTAL_UTIL_SELECT: PortalRunUtility を呼び出し、ユーティリティコマンドの結果を Tuplestore に送る。

    • 4. 実行完了後、QueryCompletion 情報をポータルの qc フィールドにコピーする。 5. treceiver->rDestroy を呼び出し、レシーバーを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/pquery.c 内のポータル実行フロー（行760, 1425など）。
  • **呼び出し先**:
  • PortalCreateHoldStore
  • CreateDestReceiver
  • SetTuplestoreDestReceiverParams
  • PortalRunMulti
  • PortalRunUtility
  • CopyQueryCompletion

関連データ構造

  • Portal: 実行状態を保持する構造体。
  • DestReceiver: 結果の配送先を定義するインターフェース。
  • Tuplestore: 実行結果を一時的に保持するメモリ/ディスク上のストア。
  • QueryCompletion: コマンド実行結果のメタデータ。

失敗モードと境界条件

  • サポートされていない portal->strategy が渡された場合、elog(ERROR) により処理を中断する。
  • PortalCreateHoldStore や PortalRunMulti 等の内部でエラーが発生した場合、PostgreSQL のエラーハンドリング機構によりトランザクションがアボートされる可能性がある。

保守担当者向けメモ

  • 本関数は、クエリ結果を即座にクライアントへ返さず、後で再利用するために保持する必要があるケース（カーソルや RETURNING 句付きのクエリなど）で使用される。
  • isTopLevel フラグは、実行コンテキストの正当性を維持するために PortalRunMulti 等へ正しく伝播させる必要がある。
