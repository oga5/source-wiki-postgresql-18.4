MultiSortSupportData

定義

  • 種別: struct
  • 定義箇所: src/include/statistics/extended_stats_internal.h:44

目的

複数列にまたがる統計情報の構築において、各次元（列）のソートおよび比較処理を効率的に管理するためのデータ構造です。

シグネチャまたは形状

typedef struct MultiSortSupportData
{
    int         ndims;          /* 次元数 */
    SortSupportData ssup[FLEXIBLE_ARRAY_MEMBER]; /* 各次元のソートサポートデータ */
} MultiSortSupportData;

入力と出力

  • **入力**: 構造体生成時に次元数（ndims）が指定され、可変長配列 ssup に各次元のソートサポート情報が格納されます。
  • **出力**: 統計計算アルゴリズムにおいて、複数列の比較やソート順序の決定に使用されます。

前提条件と不変条件

  • ssup は FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体は palloc 等を用いて ndims に応じたサイズで動的に確保される必要があります。
  • ndims は保持する次元数と一致している必要があります。

アルゴリズムまたは振る舞い

  • 複数列のソート順序を抽象化し、utils/sortsupport.h で定義された SortSupportData を各次元に対して保持することで、列ごとの比較ロジックを統合的に扱います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/extended_stats.c
  • src/backend/statistics/mcv.c

    • （いずれも palloc0 または palloc を用いて、offsetof を介したサイズ計算によりインスタンスを生成しています）

  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出し先はソースから特定できません。

関連データ構造

  • MultiSortSupport: MultiSortSupportData へのポインタ型（typedef MultiSortSupportData *MultiSortSupport;）。
  • SortSupportData: 各次元のソート処理を定義する基盤構造体。

失敗モードと境界条件

  • ndims が 0 の場合や、メモリ確保時に ndims に基づく適切なサイズ計算が行われない場合、メモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を含んでいるため、インスタンス生成時には sizeof(MultiSortSupportData) ではなく offsetof(MultiSortSupportData, ssup) を用いてベースサイズを計算し、そこに ndims * sizeof(SortSupportData) を加算する慣習に従ってください。
