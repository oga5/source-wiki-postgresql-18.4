OSSLCipher

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/openssl.c:254

目的

OpenSSL の EVP API を使用した暗号化および復号処理における状態管理を行うためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct OSSLCipher
{
	EVP_CIPHER_CTX *evp_ctx;
	const EVP_CIPHER *evp_ciph;
	uint8		key[MAX_KEY];
	uint8		iv[MAX_IV];
	unsigned	klen;
	unsigned	init;
	const struct ossl_cipher *ciph;
	ResourceOwner owner;
} OSSLCipher;

入力と出力

  • **入力**: 暗号化/復号に必要な鍵 (key)、初期化ベクトル (iv)、および OpenSSL の暗号アルゴリズム定義 (evp_ciph) を保持します。
  • **出力**: evp_ctx を通じて OpenSSL の暗号化コンテキストを管理し、処理結果を保持します。

前提条件と不変条件

  • ResourceOwner を保持しており、トランザクション終了時やエラー発生時にリソースが適切に解放される必要があります。
  • init フラグにより、暗号化コンテキストが初期化済みであるかを追跡します。

アルゴリズムまたは振る舞い

  • pgcrypto の抽象インターフェースと OpenSSL の EVP API を橋渡しします。
  • px_find_cipher() 等によって生成され、EVP_CipherUpdate などの OpenSSL 関数を呼び出す際の基盤となります。
  • PostgreSQL のメモリ管理機構と統合されており、ResourceOwner を介してライフサイクルが管理されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/openssl.c 内の暗号操作関数（free_openssl_cipher 等）および px_find_cipher を呼び出す上位層。
  • **呼び出し先**: OpenSSL の EVP_CIPHER_CTX 関連関数。
  • 詳細はソースコード内の ResourceOwnerRememberOSSLCipher や ResourceOwnerForgetOSSLCipher 等の関数から参照されています。

関連データ構造

  • EVP_CIPHER_CTX: OpenSSL の暗号化コンテキスト。
  • ossl_cipher: OpenSSL 側の暗号アルゴリズム定義構造体。
  • ResourceOwner: PostgreSQL のリソース管理用構造体。

失敗モードと境界条件

  • evp_ctx の初期化失敗や、鍵長制限（MAX_KEY）を超過した場合の挙動は OpenSSL の API 依存となります。
  • トランザクションがアボートされた場合、ResOwnerReleaseOSSLCipher が呼び出され、メモリリークを防ぐための自動解放が行われます。

保守担当者向けメモ

  • 本構造体は ResourceOwner を保持しているため、新しいメソッドを追加する際は必ずリソースの登録・解除フローを維持してください。
  • OpenSSL のバージョンアップに伴い、EVP_CIPHER_CTX の取り扱いが変更される可能性があるため注意が必要です。
