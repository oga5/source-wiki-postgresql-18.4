contain_aggs_of_level_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteManip.c:28

目的

クエリツリーの走査中に、特定のネストレベルにおける集約関数（Aggref）の存在を追跡するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	int			sublevels_up;
} contain_aggs_of_level_context;

入力と出力

  • sublevels_up: 現在の走査位置から見て、対象とするクエリレベルが何階層上にあるかを示す整数値。

前提条件と不変条件

  • sublevels_up は、expression_tree_walker 等を用いた走査において、サブクエリに潜る際にインクリメントされ、戻る際にデクリメントされる必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、contain_aggs_of_level 関数などの走査ロジックにおいて、現在の走査対象ノードが目的のクエリレベルに属しているかを判定するための状態保持に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/rewrite/rewriteManip.c 内の走査関数（contain_aggs_of_level 等）から参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Aggref: 集約関数を表すノード。
  • Query: クエリツリーのルートノード。

失敗モードと境界条件

  • 構造体定義自体に失敗モードはありません。走査ロジックにおいて sublevels_up の管理を誤ると、誤ったクエリレベルの集約関数を検出する可能性があります。

保守担当者向けメモ

  • 本構造体は rewriteManip.c 内の Walker パターンによるツリー探索専用です。
  • sublevels_up の値が正しく維持されていることを確認してください。
