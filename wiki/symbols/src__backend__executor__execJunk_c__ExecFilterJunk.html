ExecFilterJunk

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execJunk.c:246

目的

入力タプルから「ジャンク属性」（クエリ実行には必要だが最終出力には不要なデータ）を除去し、クリーンなタプルを生成して返す。

シグネチャまたは形状

TupleTableSlot *ExecFilterJunk(JunkFilter *junkfilter, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • junkfilter: フィルタリングルール（マッピング情報）を保持する JunkFilter 構造体。
  • slot: ジャンク属性を含むソースタプル。
  • **出力**:
  • ジャンク属性が取り除かれた仮想タプルを格納した TupleTableSlot。

前提条件と不変条件

  • slot は slot_getallattrs を呼び出し可能な状態であること。
  • junkfilter->jf_cleanMap は 1-based の属性番号を保持しており、0 は削除対象の属性を指す。
  • junkfilter->jf_resultSlot が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. slot_getallattrs(slot) を呼び出し、入力タプルの全属性値を抽出する。 2. junkfilter からターゲットとなる記述子とマッピング情報（cleanMap）を取得する。 3. resultSlot を ExecClearTuple で初期化する。 4. cleanMap に基づき、入力タプルの属性を新しいスロットへ転送する。

  • cleanMap[i] が 0 の場合、結果スロットの該当位置を NULL に設定する。
  • それ以外の場合、old_values[j - 1] を結果スロットへコピーする。

    • 5. ExecStoreVirtualTuple(resultSlot) を呼び出し、構築された仮想タプルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c
  • src/backend/executor/execMain.c
  • src/backend/executor/functions.c
  • **呼び出し先**:
  • slot_getallattrs
  • ExecClearTuple
  • ExecStoreVirtualTuple

関連データ構造

  • JunkFilter: フィルタリングに必要なマッピング情報や結果用スロットを保持する構造体。
  • TupleTableSlot: タプルデータを保持する実行エンジンの基本構造。

失敗モードと境界条件

  • slot が NULL の場合や、junkfilter が指す cleanMap の範囲外にアクセスが発生する場合の挙動については、本ソース excerpt 内では定義されていません。

保守担当者向けメモ

  • 本関数は現在、仮想タプルを生成する実装となっています。
  • ソース内のコメントによれば、将来的には ProjectionInfo を活用したより効率的な手法への置き換えが検討されています。
