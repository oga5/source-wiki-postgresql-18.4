ss_scan_locations_t

定義

  • 種別: struct
  • 定義箇所: src/backend/access/common/syncscan.c:104

目的

共有メモリ上に配置される、同期スキャン（Synchronous Scan）用のスキャン位置管理リストを保持するメイン構造体です。

シグネチャまたは形状

typedef struct ss_scan_locations_t
{
    ss_lru_item_t *head;
    ss_lru_item_t *tail;
    ss_lru_item_t items[FLEXIBLE_ARRAY_MEMBER]; /* SYNC_SCAN_NELEM items */
} ss_scan_locations_t;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではありません。
  • items メンバは SYNC_SCAN_NELEM 個の要素を持つ可変長配列として機能します。

前提条件と不変条件

  • 共有メモリ上に割り当てられる必要があります。
  • head および tail は items 配列内の要素を指すポインタとして管理されます。
  • SyncScanLock による排他制御下でアクセスされることが前提です。

アルゴリズムまたは振る舞い

  • LRU（Least Recently Used）リストの管理基盤として機能します。
  • head は最も最近アクセスされたエントリを、tail は最も古いエントリを指します。
  • 新規スキャン位置の検索や更新時に、リストの順序が動的に変更されます。

呼び出し元と呼び出し先

  • 呼び出し元: SyncScanShmemInit（共有メモリ確保時）、および ss_search 等の同期スキャン関連関数。
  • 呼び出し先: 本構造体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ss_lru_item_t: リストの各要素を構成する構造体。
  • ss_scan_location_t: 各スキャン位置の情報を保持する構造体。

失敗モードと境界条件

  • SYNC_SCAN_NELEM を超えるテーブルが同時にスキャンされる場合、最も古いエントリが tail から押し出されます。
  • 共有メモリの初期化に失敗した場合、本構造体へのアクセスは未定義の動作を引き起こします。

保守担当者向けメモ

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリ割り当て時には offsetof(ss_scan_locations_t, items) + (N) * sizeof(ss_lru_item_t) のような計算式でサイズを算出する必要があります。
  • scan_locations 静的ポインタ変数を介して共有メモリ上のインスタンスにアクセスします。
