plpgsql_CastHashEntry

定義

  • 種別: struct
  • 定義箇所: src/pl/plpgsql/src/pl_exec.c:168

目的

PL/pgSQL の実行時における型キャスト式の評価状態を保持するためのハッシュテーブルエントリです。

シグネチャまたは形状

typedef struct {
    plpgsql_CastHashKey key;                  /* ハッシュキー（先頭必須） */
    plpgsql_CastExprHashEntry *cast_centry;  /* 対応する式エントリへのリンク */
    ExprState  *cast_exprstate;              /* 式の評価ツリー */
    bool        cast_in_use;                 /* 評価ツリー実行中は true */
    LocalTransactionId cast_lxid;            /* キャッシュ有効性判定用 LXID */
} plpgsql_CastHashEntry;

入力と出力

  • 本構造体は、hash_search を介してハッシュテーブルから取得または生成されます。
  • cast_exprstate は、cast_lxid が現在のトランザクションの LXID と一致する場合にのみ有効とみなされます。

前提条件と不変条件

  • key フィールドは構造体の先頭に配置されている必要があります。
  • cast_exprstate は、cast_lxid が現在の LocalTransactionId と一致しない場合、再構築または更新が必要です。

アルゴリズムまたは振る舞い

  • キャスト式の評価ツリー（cast_exprstate）をキャッシュし、繰り返し実行されるキャスト処理のオーバーヘッドを削減します。
  • cast_in_use フラグにより、現在評価中のツリーであることを追跡します。
  • トランザクション境界を跨ぐキャッシュの整合性を cast_lxid で管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: get_cast_hashentry 関数が本構造体の取得およびハッシュテーブル検索を行います。
  • **呼び出し先**: 直接的な関数呼び出し関係はソース抜粋からは不明です。

関連データ構造

  • plpgsql_CastHashKey: ハッシュテーブルの検索キー。
  • plpgsql_CastExprHashEntry: 対応する式エントリ。
  • ExprState: PostgreSQL の式評価ツリー。
  • PLpgSQL_execstate: 実行状態を保持する構造体（ハッシュテーブルを保持）。

失敗モードと境界条件

  • cast_lxid が現在のトランザクションと一致しない場合、キャッシュされた cast_exprstate は無効として扱われます。
  • ハッシュテーブルの初期化や検索に失敗した場合の挙動は、hash_search の仕様に依存します。

保守担当者向けメモ

  • 本構造体は src/pl/plpgsql/src/pl_exec.c 内のハッシュテーブル管理で使用されます。
  • cast_exprstate のメモリ管理は、PL/pgSQL の実行コンテキスト（eval_mcontext 等）と整合している必要があります。
