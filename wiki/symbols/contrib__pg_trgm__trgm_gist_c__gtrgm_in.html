gtrgm_in

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_gist.c:56

目的

gtrgm 型の入力変換関数として定義されていますが、現時点では明示的にサポートを拒否する役割を担っています。

シグネチャまたは形状

Datum gtrgm_in(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（PostgreSQL の関数呼び出し規約に従う引数群）
  • 出力: なし（関数内でエラーを発生させ、処理を中断します）

前提条件と不変条件

本関数が呼び出された場合、常にエラーが発生し、正常な戻り値は返されません。

アルゴリズムまたは振る舞い

1. ereport を呼び出し、ERRCODE_FEATURE_NOT_SUPPORTED エラーを発生させます。 2. エラーメッセージとして「cannot accept a value of type gtrgm」を出力します。 3. PG_RETURN_VOID() を呼び出しますが、エラーにより到達しません。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_trgm/trgm_gist.c 内の PG_FUNCTION_INFO_V1(gtrgm_in) マクロによる登録。
  • 呼び出し先: PostgreSQL のエラー報告システム (ereport)。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数インターフェース用マクロ。

失敗モードと境界条件

  • 本関数は常に失敗（エラー送出）するように設計されています。gtrgm 型の値を入力として受け入れることはできません。

保守担当者向けメモ

  • 本関数は gtrgm 型の入力変換を意図的に禁止しています。将来的にこの型を直接入力可能にする必要がある場合は、このエラー処理を削除し、適切なパースロジックを実装する必要があります。
