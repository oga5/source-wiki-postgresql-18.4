tsquery_rewrite_query

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_rewrite.c:279

目的

SQLクエリの結果セットとして提供される置換ルールに基づき、入力された tsquery を再帰的に書き換える。

シグネチャまたは形状

Datum tsquery_rewrite_query(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TSQUERY_COPY(0): 書き換え対象の tsquery。
  • PG_GETARG_TEXT_PP(1): 置換ルールを取得するためのSQLクエリ文字列。
  • **出力**:
  • Datum: 書き換え後の tsquery を返す。

前提条件と不変条件

  • 入力クエリのサイズが0の場合、置換を行わずそのまま返す。
  • 置換ルールを提供するSQLクエリは、tsquery 型の列を2つ（検索対象パターンと置換先パターン）返す必要がある。
  • 置換処理の過程で QTNode 形式に変換され、QTNTernary および QTNSort による正規化が行われる。

アルゴリズムまたは振る舞い

1. 入力 tsquery を QTNode 木構造に変換し、正規化する。 2. 指定されたSQLクエリをSPI経由で実行し、置換ルールセットを取得する。 3. ルールセットの各行に対し、findsubquery を呼び出して木構造内の部分木を置換する。 4. 置換が発生した場合は、再度 QTNTernary と QTNSort を適用して木を再構成する。 5. 全てのルール適用後、木構造をバイナリ形式に戻し、tsquery 型として出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（ts_rewrite 関数群のバックエンドとして機能）。
  • **呼び出し先**:
  • SPI関数群 (SPI_connect, SPI_prepare, SPI_cursor_open, SPI_cursor_fetch, SPI_finish 等)
  • QT2QTN, QTNTernary, QTNSort, QTNFree, QTNBinary, QTN2QT (木構造操作ユーティリティ)
  • findsubquery (置換ロジックの核)

関連データ構造

  • TSQuery: 全文検索クエリの内部表現。
  • QTNode: tsquery を操作するための木構造表現。
  • SPIPlanPtr, Portal: SQL実行のためのSPI関連構造体。

失敗モードと境界条件

  • **SPIエラー**: SQLクエリの準備やカーソルオープンに失敗した場合、elog(ERROR) で中断する。
  • **型不一致**: SQLクエリの戻り値が tsquery 型の2列でない場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告する。
  • **スタックオーバーフロー**: 再帰的な木構造操作を行うため、非常に深いクエリ構造に対してはスタック制限に抵触する可能性がある。

保守担当者向けメモ

  • 本関数は SPI を使用して動的にルールを適用するため、ルールセットが大規模な場合、パフォーマンスに影響を与える可能性がある。
  • QTN_NOCHANGE フラグの管理により、置換済みのノードに対する冗長な再帰を抑制している。
  • findsubquery は再帰関数であるため、変更を加える際はスタック消費量に注意すること。
