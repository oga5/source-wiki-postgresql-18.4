PLy_procedure_get

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_procedure.c:65

目的

PL/Python関数またはトリガーの実行に必要な PLyProcedure 構造体をキャッシュから取得します。キャッシュに存在しない場合、または既存のキャッシュが古く無効な場合は、新規に作成してキャッシュを更新します。

シグネチャまたは形状

PLyProcedure *PLy_procedure_get(Oid fn_oid, Oid fn_rel, bool is_trigger)

入力と出力

  • **入力**:
  • fn_oid: 対象関数のOID。
  • fn_rel: トリガーの場合は関連するリレーションのOID、それ以外は InvalidOid。
  • is_trigger: 関数がトリガーであるかどうかを示すフラグ。
  • **出力**:
  • 実行可能な PLyProcedure 構造体へのポインタを返します。

前提条件と不変条件

  • fn_oid に対応するシステムカタログ（pg_proc）のタプルが存在する必要があります。
  • is_trigger が真かつ fn_rel が InvalidOid の場合、キャッシュは使用されません。
  • キャッシュ操作は PG_TRY ブロック内で保護され、エラー発生時にはキャッシュエントリが適切にクリーンアップされます。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して pg_proc タプルを取得します。 2. キャッシュが使用可能な場合、PLy_procedure_cache から fn_oid と fn_rel をキーとして検索します。 3. キャッシュが見つからない場合、PLy_procedure_create を呼び出して新規作成します。 4. キャッシュが見つかった場合、PLy_procedure_valid で有効性を確認します。無効であれば PLy_procedure_delete で解放し、再作成します。 5. 処理の成功・失敗に関わらず、最後に ReleaseSysCache でタプルを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_main.c (複数の箇所で関数実行やトリガー実行時に呼び出し)
  • **呼び出し先**:
  • SearchSysCache1, ReleaseSysCache
  • hash_search
  • PLy_procedure_create
  • PLy_procedure_valid
  • PLy_procedure_delete
  • elog (エラー時)

関連データ構造

  • PLyProcedure: 関数実行状態を保持する主要構造体。
  • PLyProcedureKey: キャッシュ検索用のキー構造体。
  • PLyProcedureEntry: キャッシュ内のエントリ構造体。
  • PLy_procedure_cache: 関数定義を保持するグローバルなハッシュテーブル。

失敗モードと境界条件

  • pg_proc の検索に失敗した場合、elog(ERROR) により処理が中断されます。
  • PG_CATCH ブロックにより、処理中にエラーが発生した場合はキャッシュから該当エントリが削除され、メモリリークや不正なエントリの残留を防ぎます。

保守担当者向けメモ

  • 本関数はキャッシュの整合性維持に責任を持ちます。PLy_procedure_valid の判定ロジックが変更された場合、キャッシュの再生成タイミングに影響するため注意してください。
  • トリガー関数は fn_rel を含むため、通常の関数とは異なるキャッシュキーのライフサイクルを持つ点に留意してください。
