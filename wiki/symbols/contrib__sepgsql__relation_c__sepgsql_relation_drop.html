sepgsql_relation_drop

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/relation.c:415

目的

リレーション（テーブル、ビュー、シーケンス等）の削除操作において、SELinuxのセキュリティポリシーに基づいた権限チェックを実行します。

シグネチャまたは形状

void sepgsql_relation_drop(Oid relOid)

入力と出力

  • **入力**: relOid (削除対象リレーションのOID)
  • **出力**: なし（権限違反時はエラーをスローし、トランザクションを中断する）

前提条件と不変条件

  • 対象リレーションがシステムカタログに存在すること。
  • sepgsql モジュールが有効であり、SELinuxポリシーがロードされていること。
  • インデックスの場合、TOASTテーブルに関連するものは無視される。

アルゴリズムまたは振る舞い

1. **種別判定**: get_rel_relkind でリレーションの種類を特定し、対応するSELinuxクラス（db_table, db_sequence, db_view）を決定する。 2. **スキーマ権限確認**: 親スキーマに対する db_schema:{remove_name} 権限を検証する。 3. **インデックスの特殊処理**: RELKIND_INDEX の場合、sepgsql_index_modify を呼び出して終了する。 4. **オブジェクト権限確認**: 対象オブジェクトに対する drop 権限を検証する。 5. **カラム権限確認**: テーブルまたはパーティションテーブルの場合、そのテーブルに属する全カラムに対して db_column:{drop} 権限を個別に検証する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:177
  • **呼び出し先**:
  • get_rel_relkind, get_rel_namespace, getObjectIdentity
  • sepgsql_avc_check_perms (権限チェック実行)
  • sepgsql_index_modify (インデックス削除時)
  • SearchSysCacheList1, ReleaseCatCacheList (カラム走査用)

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • ObjectAddress: 権限チェック対象のオブジェクトを特定するために使用。
  • Form_pg_attribute: カラムの属性情報。

失敗モードと境界条件

  • **権限不足**: sepgsql_avc_check_perms が権限違反を検知した場合、例外が発生し処理が中断される。
  • **TOASTインデックス**: PG_TOAST_NAMESPACE に属するインデックスは、チェック対象外として即時リターンする。
  • **不明なリレーション種別**: 定義外の relkind は無視される。

保守担当者向けメモ

  • 本関数はDDLフックとして動作するため、カタログ操作には SnapshotSelf が使用される前提で設計されています。
  • カラムの権限チェックは、テーブル削除時にそのテーブルに属する全カラムの削除権限を再帰的に確認する仕様となっています。
