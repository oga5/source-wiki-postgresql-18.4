strfold_builtin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_builtin.c:114

目的

PostgreSQLの組み込みロケールプロバイダにおいて、Unicode規格に基づいた文字列の「折り畳み（fold）」処理を実行する。

シグネチャまたは形状

size_t strfold_builtin(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: 変換結果を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換対象のソース文字列。
  • srclen: ソース文字列の長さ（バイト単位）。
  • locale: 組み込みロケール情報を含む pg_locale_t 構造体。
  • **出力**:
  • 変換後の文字列長（バイト単位）を size_t で返す。

前提条件と不変条件

  • locale 引数は有効な pg_locale_t 構造体を指している必要がある。
  • locale->info.builtin.casemap_full が、Unicode変換テーブルを参照するために適切に初期化されていること。

アルゴリズムまたは振る舞い

1. locale 構造体内の info.builtin.casemap_full フラグを取得する。 2. unicode_strfold 関数を呼び出し、指定された変換テーブルを用いて src から dest へ文字列の折り畳み変換を行う。 3. 変換されたバイト数を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c:1332 （ロケールプロバイダのメインディスパッチャ経由）
  • **呼び出し先**:
  • unicode_strfold （common/unicode_case.h で定義された関数）

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。内部の info.builtin メンバが組み込みロケールの設定を保持する。

失敗モードと境界条件

  • destsize が変換後の文字列を格納するのに不十分な場合、unicode_strfold の仕様に従った切り詰めやエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は src/backend/utils/adt/pg_locale.c からの委譲先として機能する。
  • 変換ロジックの詳細は common/unicode_case.h に依存しているため、Unicode規格の更新に伴う変更は主にそちらで行うこと。
