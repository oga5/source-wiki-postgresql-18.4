to_regoperator

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regproc.c:693

目的

演算子名（および引数型）を表す文字列を、システムカタログ上の演算子 OID に変換します。エラー発生時に例外をスローせず、安全に結果を返すためのラッパー関数です。

シグネチャまたは形状

Datum to_regoperator(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介して取得される、演算子名を含むテキスト文字列（例: "operator_name(type, type)"）。
  • **出力**: 成功した場合は演算子の OID を含む Datum を返し、解決に失敗した場合は NULL を返します。

前提条件と不変条件

  • 入力文字列は regoperatorin 関数が解釈可能な形式である必要があります。
  • ErrorSaveContext を使用してエラーを捕捉するため、呼び出し元でトランザクションの中断を発生させずに解決の成否を判定可能です。

アルゴリズムまたは振る舞い

1. 入力された text 型の引数を C 文字列に変換します。 2. ErrorSaveContext を初期化し、エラー発生時の安全なハンドリングを準備します。 3. DirectInputFunctionCallSafe を呼び出し、regoperatorin を用いて文字列から OID への変換を試みます。 4. 変換が成功すればその Datum を返し、失敗した場合は PG_RETURN_NULL() により NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから直接呼び出されることを想定しています。
  • **呼び出し先**:
  • text_to_cstring: 入力テキストの変換。
  • regoperatorin: 実際の名前解決と OID 変換を行うコア関数。
  • DirectInputFunctionCallSafe: エラーを抑制しつつ関数を呼び出すためのユーティリティ。

関連データ構造

  • ErrorSaveContext: エラー情報を保持し、例外を回避するための構造体。
  • Datum: PostgreSQL の内部データ表現。

失敗モードと境界条件

  • 入力文字列が不正な形式である場合や、該当する演算子が存在しない場合は、regoperatorin がエラーを報告し、to_regoperator は NULL を返します。
  • 存在しない型や曖昧な演算子指定の場合も同様に NULL が返されます。

保守担当者向けメモ

  • 本関数は regoperatorin の安全な呼び出しインターフェースです。ロジックの変更が必要な場合は、まず regoperatorin 側の名前解決ロジックを確認してください。
  • ErrorSaveContext の使用により、呼び出し側でエラーをハンドリングする設計となっているため、この関数自体でエラーをスローしないよう注意してください。
