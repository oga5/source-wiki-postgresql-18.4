pgstat_tracks_backend_bktype

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_backend.c:364

目的

指定されたバックエンド種別（BackendType）が、統計収集の対象であるかどうかを判定します。

シグネチャまたは形状

bool pgstat_tracks_backend_bktype(BackendType bktype)

入力と出力

  • **入力**: bktype (判定対象の BackendType 列挙値)
  • **出力**: 統計収集対象であれば true、それ以外であれば false を返します。

前提条件と不変条件

  • BackendType の全列挙値が switch 文で網羅されている必要があります。
  • 新しいバックエンド種別が追加された場合、本関数の switch 文を更新して適切な判定結果を割り当てる必要があります。

アルゴリズムまたは振る舞い

bktype に基づき以下の通り判定します。

  • **対象外 (false)**: B_INVALID, B_AUTOVAC_LAUNCHER, B_DEAD_END_BACKEND, B_ARCHIVER, B_LOGGER, B_BG_WRITER, B_CHECKPOINTER, B_IO_WORKER, B_STARTUP
  • **対象 (true)**: B_AUTOVAC_WORKER, B_BACKEND, B_BG_WORKER, B_STANDALONE_BACKEND, B_SLOTSYNC_WORKER, B_WAL_RECEIVER, B_WAL_SENDER, B_WAL_SUMMARIZER, B_WAL_WRITER

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/backend_status.c:497
  • src/backend/utils/activity/pgstat_backend.c:61, 77, 136, 275
  • src/backend/utils/adt/pgstatfuncs.c:1979
  • **呼び出し先**: 本関数は列挙値の判定のみを行うため、他の関数を呼び出しません。

関連データ構造

  • BackendType (列挙型)

失敗モードと境界条件

  • 定義されていない BackendType 値が渡された場合、デフォルトで false を返します。

保守担当者向けメモ

  • 新しいバックエンド種別を追加する際は、必ず本関数の switch 文を確認し、統計収集の要否を判断して追加してください。これにより、未対応の種別に対する警告や誤判定を防ぎます。
