index_form_tuple_context

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/indextuple.c:64

目的

指定された属性値から新しい IndexTuple を構築し、呼び出し元が指定したメモリコンテキスト上に割り当てます。

シグネチャまたは形状

IndexTuple index_form_tuple_context(TupleDesc tupleDescriptor,
                                    const Datum *values,
                                    const bool *isnull,
                                    MemoryContext context)

入力と出力

  • **入力**:
  • tupleDescriptor: タプルの構造定義。
  • values: 属性値の配列。
  • isnull: 各属性がNULLかどうかを示すブール配列。
  • context: タプルを割り当てるメモリコンテキスト。
  • **出力**:
  • 構築された IndexTuple へのポインタ。

前提条件と不変条件

  • 属性数（natts）が INDEX_MAX_KEYS 以下であること。
  • インデックス行は外部ストレージ（TOAST）を保持できないため、外部データはインライン化または圧縮される。
  • 構築後のタプルサイズが INDEX_SIZE_MASK を超えないこと。

アルゴリズムまたは振る舞い

1. **TOAST処理**: TOAST_INDEX_HACK が有効な場合、外部データ（EXTERNAL）をデトーストし、必要に応じて圧縮を試みる。 2. **NULL判定**: NULL値の有無を確認し、infomask に INDEX_NULL_MASK を設定する。 3. **サイズ計算**: heap_compute_data_size を使用してデータ領域のサイズを算出し、ヘッダーサイズを加算してメモリを確保する。 4. **タプル構築**: heap_fill_tuple を呼び出し、データをメモリ領域に書き込む。 5. **メタデータ設定**: 可変長属性の有無を infomask に反映し、最終的なサイズを t_info に格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/indextuple.c (ラッパー関数 index_form_tuple)
  • src/backend/utils/sort/tuplesortvariants.c
  • **呼び出し先**:
  • detoast_external_attr (TOAST処理)
  • toast_compress_datum (圧縮処理)
  • heap_compute_data_size (サイズ計算)
  • MemoryContextAllocZero (メモリ確保)
  • heap_fill_tuple (データ書き込み)

関連データ構造

  • IndexTuple: インデックス行のメモリ表現。
  • TupleDesc: タプルの記述子。
  • Datum: 属性値の汎用型。

失敗モードと境界条件

  • numberOfAttributes > INDEX_MAX_KEYS: ERRCODE_TOO_MANY_COLUMNS エラーを発生させる。
  • size > INDEX_SIZE_MASK: ERRCODE_PROGRAM_LIMIT_EXCEEDED エラーを発生させる。

保守担当者向けメモ

  • 本関数は index_form_tuple のメモリコンテキスト指定版です。
  • TOAST_INDEX_HACK が定義されている環境では、一時的なメモリ割り当てが発生するため、処理終了後に pfree で適切に解放されます。
  • heap_fill_tuple が生成する tupmask から、インデックスに必要な INDEX_VAR_MASK を抽出するロジックが含まれています。
