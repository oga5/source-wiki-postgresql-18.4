LOCALLOCKOWNER

定義

  • 種別: struct
  • 定義箇所: src/include/storage/lock.h:415

目的

特定のロックを保持しているリソース所有者（ResourceOwner）と、その所有者によるロックの保持回数を管理するために使用されます。

シグネチャまたは形状

typedef struct LOCALLOCKOWNER
{
    struct ResourceOwnerData *owner;
    int64       nLocks;
} LOCALLOCKOWNER;

入力と出力

  • **入力**: owner（ResourceOwnerData へのポインタ）、nLocks（保持回数）。
  • **出力**: なし（構造体定義）。

前提条件と不変条件

  • owner が NULL の場合、そのロックはセッション単位で保持されていることを示します。
  • owner が NULL でない場合、そのロックは現在のトランザクション単位で保持されていることを示します。
  • 循環参照を避けるため、ResourceOwnerData は前方宣言として扱われます。

アルゴリズムまたは振る舞い

各バックエンドプロセスがローカルで保持する LOCALLOCK 構造体内に配列として格納され、ロックの取得・解放時にこの配列を走査して所有者ごとの保持回数を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/lock.c 内のロック管理関数群（LockAcquire, LockRelease 等）が、LOCALLOCK 構造体内の lockOwners 配列を操作する際に参照します。
  • **呼び出し先**: 情報なし。

関連データ構造

  • LOCALLOCK: 本構造体を配列として保持する親構造体。
  • ResourceOwnerData: ロックの所有権を管理するリソース所有者構造体。

失敗モードと境界条件

  • LOCALLOCK 内の lockOwners 配列のサイズが不足した場合、src/backend/storage/lmgr/lock.c にて再割り当てが行われます。

保守担当者向けメモ

  • LOCALLOCK 構造体の lockOwners フィールドを介してアクセスされます。
  • ロックの所有権管理において、セッションレベルとトランザクションレベルを区別する重要な役割を担っています。
