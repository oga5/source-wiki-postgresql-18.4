switchToPresortedPrefixMode

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:285

目的

Full Sort モードから Prefix Sort モードへ移行し、共通の接頭辞を持つタプルグループに対して残りのキーのみを用いた効率的なソートを実行する。

シグネチャまたは形状

static void switchToPresortedPrefixMode(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（IncrementalSortState 構造体を含む実行状態）。
  • **出力**: なし（副作用として node->prefixsort_state の初期化またはリセット、および node->execution_status の更新を行う）。

前提条件と不変条件

  • pstate は IncrementalSortState にキャスト可能であること。
  • node->fullsort_state にソート対象のタプルが保持されていること。
  • node->transfer_tuple は、現在のグループ判定に使用される有効なスロットであること。

アルゴリズムまたは振る舞い

1. **状態初期化**: prefixsort_state が未作成であれば、tuplesort_begin_heap を用いて接頭辞を除いた残りのカラムでソートするように初期化する。既に存在する場合は tuplesort_reset を行う。 2. **バウンド設定**: node->bounded が真の場合、残りのソート可能数に合わせて tuplesort_set_bound を適用する。 3. **タプル転送**: fullsort_state からタプルを順次取り出し、isCurrentGroup を用いて現在の接頭辞グループに属するか判定する。

  • 同一グループであれば prefixsort_state に投入する。
  • グループ境界に達した場合は、そのタプルを保持したままループを終了する。

    • 4. **実行状態の更新**:

  • 全タプルが同一グループであれば、INCSORT_LOADPREFIXSORT へ移行し、さらなる入力を待機する。
  • グループが終了した場合は、tuplesort_performsort を実行し、INCSORT_READPREFIXSORT へ移行してソート済みタプルの返却準備を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecIncrementalSort（src/backend/executor/nodeIncrementalSort.c 内の複数箇所）。
  • **呼び出し先**: tuplesort_begin_heap, tuplesort_reset, tuplesort_set_bound, tuplesort_puttupleslot, tuplesort_gettupleslot, tuplesort_performsort, isCurrentGroup, ExecCopySlot, ExecClearTuple。

関連データ構造

  • IncrementalSortState: 本ノードの実行状態を保持する主要構造体。
  • Tuplesortstate: prefixsort_state として使用されるソート用状態。
  • PlanState: 実行計画のノード状態。

失敗モードと境界条件

  • node->n_fullsort_remaining が 0 になった場合、現在のグループの終了を検知し、ソートを実行する。
  • group_pivot が空の場合、最初のタプルをピボットとして設定する。
  • bounded モードでは、bound_Done を更新し、ソート範囲を動的に調整する。

保守担当者向けメモ

  • 本関数は ExecIncrementalSort の状態遷移ロジックの中核を担う。
  • transfer_tuple は fullsort_state 内のメモリを参照しているため、fullsort_state のリセットタイミングには注意が必要である。
  • isCurrentGroup の比較ロジックが変更された場合、本関数でのグループ判定結果に直接影響する。
