pg_atomic_flag

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/arch-x86.h:55

目的

アトミックなフラグ操作（テスト・セットおよびクリア）を行うためのメモリ領域を定義します。

シグネチャまたは形状

typedef struct pg_atomic_flag
{
    volatile char value;
} pg_atomic_flag;

入力と出力

  • 入力: volatile char 型のメンバ value を保持します。
  • 出力: なし（データ構造の定義）。

前提条件と不変条件

  • volatile 修飾子により、コンパイラによる最適化（レジスタへのキャッシュなど）を抑制し、メモリ上の値を直接参照することを保証します。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行い、具体的なアトミック操作は pg_atomic_test_set_flag_impl や pg_atomic_clear_flag_impl などのインライン関数によって提供されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/include/port/atomics.h: pg_atomic_init_flag, pg_atomic_test_set_flag, pg_atomic_unlocked_test_flag, pg_atomic_clear_flag
  • src/include/port/atomics/arch-x86.h: pg_atomic_test_set_flag_impl, pg_atomic_clear_flag_impl
  • src/include/port/atomics/generic-gcc.h: pg_atomic_test_set_flag_impl, pg_atomic_unlocked_test_flag_impl, pg_atomic_clear_flag_impl
  • src/backend/postmaster/autovacuum.c: wi_dobalance 変数の型として使用
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • pg_atomic_uint32
  • pg_atomic_uint64

失敗モードと境界条件

  • アトミック操作の整合性は、本構造体を指すポインタが適切にアライメントされていることに依存します。

保守担当者向けメモ

  • 本構造体は arch-x86.h および generic-gcc.h の両方で定義されています。環境に応じて適切な定義が選択される仕組みです。
  • value メンバは char 型ですが、アトミック操作時にはアーキテクチャ固有の命令（lock プレフィックス付き命令など）が適用されることを前提としています。
