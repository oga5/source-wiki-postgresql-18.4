set_foreign_rel_properties

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/relnode.c:588

目的

外部テーブル（Foreign Table）を含む結合リレーションにおいて、結合後のリレーションノード（joinrel）に対して、外部データラッパー（FDW）に関連するプロパティ（サーバーID、ユーザーID、FDWルーチン等）を適切に伝播・設定する。

シグネチャまたは形状

static void set_foreign_rel_properties(RelOptInfo *joinrel, RelOptInfo *outer_rel, RelOptInfo *inner_rel)

入力と出力

  • **入力**:
  • joinrel: 結合結果として作成される RelOptInfo 構造体（出力先）。
  • outer_rel, inner_rel: 結合対象となる左右のリレーションノード。
  • **出力**: なし（joinrel の各フィールドを直接更新する）。

前提条件と不変条件

  • outer_rel および inner_rel は初期化済みであること。
  • 外部テーブルの結合において、両者が同一のサーバー（serverid）を参照している場合にのみプロパティが設定される。

アルゴリズムまたは振る舞い

1. outer_rel と inner_rel の serverid が一致するか確認する。 2. 一致する場合、以下のいずれかの条件で joinrel のプロパティを更新する：

  • 両者の userid が一致する場合：そのまま値をコピーし、useridiscurrent を論理和で更新する。
  • inner_rel の userid が無効で、outer_rel の userid が現在のユーザーと一致する場合：outer_rel の情報を継承し、useridiscurrent を true に設定する。
  • outer_rel の userid が無効で、inner_rel の userid が現在のユーザーと一致する場合：inner_rel の情報を継承し、useridiscurrent を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/relnode.c 内の build_join_rel（773行目および957行目付近）。
  • **呼び出し先**: GetUserId()（現在のユーザーIDを取得する関数）。

関連データ構造

  • RelOptInfo: プランナがリレーション情報を保持するために使用する主要な構造体。

失敗モードと境界条件

  • serverid が一致しない場合、または userid の条件を満たさない場合、joinrel の外部テーブル関連プロパティは更新されず、初期値（通常は無効値）のままとなる。

保守担当者向けメモ

  • 本関数は、FDW を使用した結合において、リモートサーバーへのプッシュダウンが可能かどうかを判断する重要な基盤となる。
  • useridiscurrent の扱いは、リモートクエリ実行時の権限チェックに直結するため、変更時は fdwroutine の仕様と照らし合わせること。
