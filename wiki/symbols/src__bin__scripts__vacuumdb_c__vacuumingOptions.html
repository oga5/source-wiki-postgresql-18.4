vacuumingOptions

定義

  • 種別: struct
  • 定義箇所: src/bin/scripts/vacuumdb.c:31

目的

vacuumdb ユーティリティにおいて、ユーザーがコマンドライン引数を通じて指定した VACUUM および ANALYZE の実行オプションを保持・管理するために使用されます。

シグネチャまたは形状

typedef struct vacuumingOptions
{
    bool        analyze_only;
    bool        verbose;
    bool        and_analyze;
    bool        full;
    bool        freeze;
    bool        disable_page_skipping;
    bool        skip_locked;
    int         min_xid_age;
    int         min_mxid_age;
    int         parallel_workers;
    bool        no_index_cleanup;
    bool        force_index_cleanup;
    bool        do_truncate;
    bool        process_main;
    bool        process_toast;
    bool        skip_database_stats;
    char       *buffer_usage_limit;
    bool        missing_stats_only;
} vacuumingOptions;

入力と出力

  • **入力**: コマンドライン引数の解析結果が各フィールドに格納されます。parallel_workers はユーザー指定がない場合 -1 となります。
  • **出力**: 構造体の各フィールド値は、prepare_vacuum_command() 等の関数で SQL 文を構築する際のパラメータとして参照されます。

前提条件と不変条件

  • parallel_workers が 0 以上の場合、ユーザーが並列度を指定したことを示します。
  • サーバーバージョンとの互換性は、この構造体自体ではなく、これを利用する関数（vacuum_one_database 等）側で PQserverVersion を用いて検証されます。

アルゴリズムまたは振る舞い

本構造体はデータの保持のみを行い、ロジックは含みません。実行時に main 関数で初期化され、retrieve_objects や prepare_vacuum_command などの関数へポインタとして渡され、SQL 生成の制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（初期化）、および vacuum_one_database、retrieve_objects、prepare_vacuum_command 等の各処理関数。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • VacObjFilter: 処理対象（DB、テーブル、スキーマ）を絞り込むための構造体。
  • ParallelSlot: 並列実行時に使用される接続管理構造体。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、buffer_usage_limit は文字列ポインタであるため、メモリ管理や不正な値のバリデーションが利用側で必要となります。

保守担当者向けメモ

  • 新しい VACUUM オプションが PostgreSQL サーバーに追加された場合、この構造体にフィールドを追加し、prepare_vacuum_command での SQL 生成ロジックを更新してください。
  • parallel_workers の初期値 -1 の扱いに注意してください。
