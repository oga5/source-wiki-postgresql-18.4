manifest_data

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.h:75

目的

バックアップマニフェスト全体の解析結果を保持し、検証プロセスにおいてファイル情報や WAL 範囲を効率的に検索・照合するためのルート構造体。

シグネチャまたは形状

typedef struct manifest_data
{
	int			version;
	uint64		system_identifier;
	manifest_files_hash *files;
	manifest_wal_range *first_wal_range;
	manifest_wal_range *last_wal_range;
} manifest_data;

入力と出力

  • **入力**: マニフェストファイルの内容を解析した結果が各フィールドに格納される。
  • **出力**: 検証プロセスにおいて、ファイルパスによる高速検索（files）や WAL 範囲の確認（first_wal_range, last_wal_range）の基盤として利用される。

前提条件と不変条件

  • files フィールドは lib/simplehash.h を用いて生成されたハッシュテーブルであり、ファイルパスをキーとして manifest_file を検索するために使用される。
  • system_identifier はバックアップ元のデータベースクラスタを一意に識別する。

アルゴリズムまたは振る舞い

1. pg_verifybackup.c 等のメインロジックによってマニフェストが解析され、本構造体が構築される。 2. 検証実行中、ファイルパスをキーとして files ハッシュテーブルを検索し、対応する manifest_file を取得して整合性を確認する。 3. WAL 範囲のリスト（first_wal_range から last_wal_range）を辿ることで、バックアップに含まれる WAL の連続性を検証する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/load_manifest.c において、マニフェストの読み込みおよびメモリ確保の対象として参照されている。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはない。

関連データ構造

  • manifest_files_hash: ファイルパスをキーとするハッシュテーブル。
  • manifest_wal_range: WAL 範囲を管理するリスト構造。
  • manifest_file: 個別のファイル情報を保持する構造体。

失敗モードと境界条件

  • マニフェストの解析に失敗した場合、本構造体は適切に初期化されない可能性がある。
  • files ハッシュテーブルの検索時にキーが存在しない場合、検証対象外のファイルとして扱われる。

保守担当者向けメモ

  • manifest_files_hash は simplehash.h のテンプレート生成マクロによって定義されているため、ハッシュテーブルの仕様変更時は pg_verifybackup.h 内の SH_ マクロ定義を確認すること。
  • pg_combinebackup でも利用されているため、構造体の変更は両ツールへの影響を考慮する必要がある。
