gbt_byteagt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bytea.c:23

目的

bytea型に対する「より大きい（greater than）」比較演算を、GiSTインデックスの内部ロジックから呼び出し可能な形式で提供します。

シグネチャまたは形状

bool gbt_byteagt(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のbyteaデータへのポインタ。
  • collation: 照合順序（本関数では使用されません）。
  • flinfo: 関数マネージャ情報（本関数では使用されません）。
  • **出力**:
  • a > b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効なbyteaデータ構造を指している必要があります。
  • PostgreSQLの標準関数 byteagt が正常に動作する環境である必要があります。

アルゴリズムまたは振る舞い

1. a と b を Datum 型に変換します。 2. DirectFunctionCall2 を介して、PostgreSQLの標準比較関数 byteagt を呼び出します。 3. byteagt の戻り値を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: byteagt（PostgreSQL標準のbytea比較関数）。
  • **呼び出し元**: ソースコード内の静的解析によれば、contrib/btree_gist/btree_bytea.c:76 にて参照されています。

関連データ構造

  • bytea: PostgreSQLの可変長バイナリデータ型。
  • Datum: PostgreSQLの汎用データ表現型。

失敗モードと境界条件

  • byteagt 関数が内部的にエラーを発生させた場合、本関数もそのエラーを伝播します。
  • 入力値が NULL の場合の挙動は、呼び出し元のGiSTインデックスのロジックに依存します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な比較ロジックは byteagt に依存しています。
  • collation や flinfo 引数は現在の実装では使用されていませんが、インターフェースの互換性維持のために保持されています。
