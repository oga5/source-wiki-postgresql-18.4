AtEOXact_Namespace

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/namespace.c:4514

目的

トランザクション終了時（EOXact）に、セッションの一時名前空間（Temporary Namespace）の状態を管理・クリーンアップする。

シグネチャまたは形状

void AtEOXact_Namespace(bool isCommit, bool parallel)

入力と出力

  • isCommit (bool): トランザクションがコミットされる場合は true、アボートされる場合は false。
  • parallel (bool): パラレルワーカーのコンテキストであるかを示す。
  • 出力: なし（副作用としてグローバル変数およびプロセス状態を更新）。

前提条件と不変条件

  • myTempNamespaceSubID が InvalidSubTransactionId でない場合にのみ処理が実行される。
  • parallel が true の場合、本関数は何も行わない。

アルゴリズムまたは振る舞い

1. **コミット時 (isCommit == true)**:

  • RemoveTempRelationsCallback を before_shmem_exit に登録し、バックエンド終了時に一時テーブルを削除する準備を行う。

    • 2. **アボート時 (isCommit == false)**:

  • 一時名前空間の OID (myTempNamespace, myTempToastNamespace) を InvalidOid にリセットする。
  • 検索パスのキャッシュフラグ (baseSearchPathValid, searchPathCacheValid) を false に設定し、再構築を強制する。
  • MyProc->tempNamespaceId を InvalidOid にリセットする。

    • 3. **共通**:

  • 処理完了後、myTempNamespaceSubID を InvalidSubTransactionId にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c 内のトランザクション終了処理から呼び出される。
  • **呼び出し先**: before_shmem_exit（コミット時）。

関連データ構造

  • MyProc: プロセスごとの共有メモリ状態。
  • myTempNamespace, myTempToastNamespace: 現在のセッションの一時名前空間を保持するグローバル変数。
  • baseSearchPathValid, searchPathCacheValid: 検索パスのキャッシュ有効性フラグ。

失敗モードと境界条件

  • パラレルワーカー実行中は、一時名前空間の操作を行わない（parallel 引数によるガード）。
  • アボート時の MyProc->tempNamespaceId リセットはアトミックであると仮定されている。

保守担当者向けメモ

  • 本関数はトランザクションのコミット/アボートのたびに呼ばれるため、一時名前空間のライフサイクル管理において重要な役割を果たす。
  • アボート時に baseSearchPathValid を無効化することで、次のトランザクションで検索パスが確実に再計算されるようになっている。
