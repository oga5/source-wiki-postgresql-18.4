ScanCKeywordLookup

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/c_keywords.c:35

目的

入力された文字列がC言語の予約語であるかを判定し、該当する場合は対応するトークン値を返却します。

シグネチャまたは形状

int ScanCKeywordLookup(const char *text)

入力と出力

  • **入力**: text (解析対象の文字列)
  • **出力**: 予約語と一致した場合は対応するトークン値（int）、一致しない場合は -1 を返します。

前提条件と不変条件

  • 完全ハッシュ関数（ScanCKeywords_hash_func）を使用しているため、ハッシュ値はキーワードリストのインデックスと直接対応します。
  • C言語のキーワード判定のため、大文字と小文字を区別した厳密な比較が行われます。

アルゴリズムまたは振る舞い

1. **長さチェック**: 入力文字列の長さが ScanCKeywords.max_kw_len を超える場合、即座に -1 を返します。 2. **ハッシュ計算**: ScanCKeywords_hash_func を用いてハッシュ値 h を算出します。 3. **範囲検証**: h が有効なキーワード範囲内（0 以上 ScanCKeywords.num_keywords 未満）であることを確認し、範囲外であれば -1 を返します。 4. **照合**: GetScanKeyword で取得したキーワードと入力文字列を strcmp で比較します。 5. **結果返却**: 一致すれば ScanCKeywordTokens[h] を返し、不一致であれば -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/parser.c 等の字句解析ロジックから呼び出されることが想定されています（ソースコード内の直接的な呼び出し箇所は、提供された抜粋内では特定されていません）。
  • **呼び出し先**:
  • ScanCKeywords_hash_func
  • GetScanKeyword
  • strcmp
  • strlen

関連データ構造

  • ScanCKeywords: キーワードリストのメタデータ（最大長、総数など）を保持する構造体。
  • ScanCKeywordTokens: 各キーワードに対応するトークン値の配列。

失敗モードと境界条件

  • 入力文字列が長すぎる場合、ハッシュ計算を行わずに -1 を返します。
  • ハッシュ値が範囲外の場合、不一致とみなして -1 を返します。
  • 文字列が一致しない場合、-1 を返します。

保守担当者向けメモ

  • 本関数は完全ハッシュを利用しているため、ハッシュ衝突が発生しない前提で設計されています。
  • c_kwlist.h の変更に伴い、ハッシュ関数やトークン配列が再生成される仕組みになっています。
