ReadDimensionInt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:518

目的

文字列から配列の次元または境界値を表す整数を解析し、内部表現へ変換します。

シグネチャまたは形状

static bool ReadDimensionInt(char **srcptr, int *result, const char *origStr, Node *escontext);

入力と出力

  • srcptr (入出力): 解析対象の文字列ポインタ。解析終了後の位置に更新されます。
  • result (出力): 解析された整数値が格納されます。
  • origStr (入力): エラーメッセージ生成用の元の文字列。
  • escontext (入力): エラー報告用コンテキスト。
  • 戻り値: 成功時に true、範囲外エラー等の失敗時に false を返します。

前提条件と不変条件

  • *srcptr が指す文字列が、数値（符号付き）で開始されている必要があります。
  • 解析される値は、PG_INT32_MIN から PG_INT32_MAX の範囲内に収まる必要があります。

アルゴリズムまたは振る舞い

1. 先頭文字が数字、+、または - でない場合、解析をスキップし result に 0 を設定して true を返します。 2. strtol を使用して文字列を long 型に変換し、srcptr を進めます。 3. 変換結果が int32 の範囲を超えている場合、escontext を通じてエラーを報告し false を返します。 4. 正常な場合、値を int にキャストして result に格納し true を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/arrayfuncs.c 内の配列解析ロジック（ReadArrayDimensions 等から呼び出されることが示唆されています）。
  • 呼び出し先: strtol (標準ライブラリ)、ereturn (エラー報告マクロ)。

関連データ構造

  • Node: エラーコンテキストとして使用されます。

失敗モードと境界条件

  • **範囲外エラー**: 解析された整数が int32 の範囲（PG_INT32_MIN ～ PG_INT32_MAX）を超えた場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーとなります。
  • **非数値**: 先頭が数値でない場合はエラーではなく 0 を返して終了します。

保守担当者向けメモ

  • 本関数は配列の次元定義をパースする際の低レベルユーティリティです。
  • strtol の結果を int にキャストしているため、プラットフォーム依存の long のサイズに関わらず int32 への適合性が保証されています。
