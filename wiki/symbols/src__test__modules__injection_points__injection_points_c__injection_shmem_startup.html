injection_shmem_startup

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_points.c:150

目的

PostgreSQLの共有メモリ初期化プロセスにおいて、インジェクションポイント機能で使用する共有メモリ領域（InjectionPointSharedState）の確保またはアタッチを行う。

シグネチャまたは形状

static void injection_shmem_startup(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 inj_state を初期化）

前提条件と不変条件

  • AddinShmemInitLock を使用して排他制御を行う。
  • prev_shmem_startup_hook が設定されている場合、本関数の実行前に呼び出される。
  • inj_state は共有メモリ上の InjectionPointSharedState 構造体を指すように設定される。

アルゴリズムまたは振る舞い

1. prev_shmem_startup_hook が存在すれば実行する。 2. AddinShmemInitLock を取得し、共有メモリ領域 injection_points を確保または既存領域へアタッチする。 3. 初回初期化（found が false）の場合、injection_point_init_state を呼び出して共有メモリの状態を初期化する。 4. AddinShmemInitLock を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: shmem_startup_hook に本関数が登録されることで、PostgreSQLの共有メモリ起動シーケンスから呼び出される。
  • 呼び出し先:
  • prev_shmem_startup_hook()
  • LWLockAcquire() / LWLockRelease()
  • ShmemInitStruct()
  • injection_point_init_state()

関連データ構造

  • InjectionPointSharedState: 共有メモリ上で管理されるインジェクションポイントの状態構造体。
  • AddinShmemInitLock: 共有メモリ初期化時の競合を防ぐためのロック。

失敗モードと境界条件

  • ShmemInitStruct が失敗した場合、PostgreSQLの共有メモリ割り当てルールに従い、プロセスが異常終了する可能性がある。
  • 共有メモリが既に初期化されている場合、found フラグにより injection_point_init_state の再実行はスキップされる。

保守担当者向けメモ

  • 本関数は shmem_startup_hook に登録されるフック関数である。
  • injection_point_init_state による初期化は、DSM（動的共有メモリ）経由の動的初期化と共有されているため、変更時は両方のパスに影響がないか注意すること。
