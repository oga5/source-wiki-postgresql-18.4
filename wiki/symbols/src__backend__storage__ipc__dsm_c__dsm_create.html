dsm_create

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm.c:515

目的

動的共有メモリ（DSM）セグメントを新規に作成し、呼び出し元プロセスにマッピングされた dsm_segment 構造体を返します。

シグネチャまたは形状

dsm_segment *dsm_create(Size size, int flags)

入力と出力

  • **入力**:
  • size: 作成するセグメントのサイズ（バイト単位）。
  • flags: 動作オプション（例: DSM_CREATE_NULL_IF_MAXSEGMENTS）。
  • **出力**:
  • 成功時: 初期化済みの dsm_segment 構造体へのポインタ。
  • 失敗時: flags に DSM_CREATE_NULL_IF_MAXSEGMENTS が指定されている場合は NULL、それ以外はエラーを発生させる。

前提条件と不変条件

  • ポストマスター環境下、またはシングルユーザーモードで実行されること（Assert による検証）。
  • dsm_init_done が偽の場合、内部的に dsm_backend_startup() が呼び出される。
  • 制御セグメント内のスロットの参照カウント（refcnt）は、作成直後に 2 に設定される。

アルゴリズムまたは振る舞い

1. dsm_create_descriptor() でセグメント記述子を確保する。 2. 設定されている場合、メイン共有メモリ領域（dsm_main_space_fpm）からの割り当てを試みる。 3. メイン領域が利用できない場合、dsm_impl_op を用いて OS レベルの共有メモリセグメントを作成する。 4. DynamicSharedMemoryControlLock を取得し、制御セグメント内の空きスロットを検索する。 5. 空きスロットが見つかった場合、セグメント情報を登録し、参照カウントを 2 に初期化してロックを解放する。 6. スロットが満杯の場合、確保したリソースを解放し、エラー処理（または NULL 返却）を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c
  • src/backend/access/common/session.c
  • src/backend/access/transam/parallel.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/backend/storage/ipc/dsm_registry.c
  • src/backend/utils/mmgr/dsa.c
  • **呼び出し先**:
  • dsm_create_descriptor()
  • dsm_impl_op()
  • LWLockAcquire() / LWLockRelease()
  • FreePageManagerGet() / FreePageManagerPut()
  • ResourceOwnerForgetDSM()

関連データ構造

  • dsm_segment: セグメントの管理構造体。
  • dsm_control_header: 制御セグメントのヘッダー。
  • dsm_control_item: 制御セグメント内の各セグメント管理項目。
  • FreePageManager: メイン共有メモリ領域の管理用構造体。

失敗モードと境界条件

  • **セグメント上限超過**: nitems >= dsm_control->maxitems の場合、DSM_CREATE_NULL_IF_MAXSEGMENTS フラグの有無に応じて NULL を返すか、エラーを発生させる。
  • **メモリ不足**: OS レベルのセグメント作成失敗時はエラーとなる。
  • **異常終了**: ポストマスター環境下での呼び出しはアサートにより制限される。

保守担当者向けメモ

  • 参照カウントが 2 から開始されるのは、1 がセグメント破棄のトリガーとして予約されているためである。
  • dsm_main_space_fpm を使用する場合、セグメントのハンドル生成には make_main_region_dsm_handle が使用される。
  • エラー発生時のクリーンアップ（pfree や dlist_delete）は、リソースリークを防ぐために厳密に行う必要がある。
