SystemAttributeDefinition

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:235

目的

システム属性（ctid, xmin 等）のメタデータである FormData_pg_attribute 構造体へのポインタを、属性番号（attno）に基づいて取得する。

シグネチャまたは形状

const FormData_pg_attribute *SystemAttributeDefinition(AttrNumber attno)

入力と出力

  • **入力**: attno (負の属性番号)
  • **出力**: 対応するシステム属性の定義情報を含む FormData_pg_attribute 構造体への定数ポインタ。

前提条件と不変条件

  • attno は負の値である必要がある。
  • attno はシステム属性の有効範囲内（SysAtt 配列のサイズ内）である必要がある。

アルゴリズムまたは振る舞い

1. 入力された attno が 0 以上、または定義されたシステム属性の範囲外である場合、elog(ERROR) を発生させる。 2. 範囲内であれば、内部の静的配列 SysAtt から -attno - 1 のインデックスを使用して該当する属性定義を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/spi.c
  • src/backend/optimizer/util/plancat.c
  • src/backend/parser/parse_relation.c
  • src/backend/parser/parse_utilcmd.c
  • **呼び出し先**:
  • elog (エラー発生時)

関連データ構造

  • FormData_pg_attribute: システムカタログ pg_attribute の行形式を定義する構造体。
  • SysAtt: ファイル内で定義されているシステム属性の静的配列（ソースコード上のハードコーディング箇所）。

失敗モードと境界条件

  • attno >= 0 または定義範囲外の負の値を指定した場合、エラーとして処理を中断する。

保守担当者向けメモ

  • 本関数はシステム属性の定義を静的に保持する SysAtt 配列に依存している。
  • ソースコード内に「UGLY HARD CODED BADNESS」とコメントされている通り、システム属性の定義はハードコーディングされており、変更には細心の注意が必要である。
