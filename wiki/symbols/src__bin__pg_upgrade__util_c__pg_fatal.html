pg_fatal

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/util.c:269

目的

致命的なエラーを報告し、プログラムを異常終了させるための共通インターフェースを提供します。

シグネチャまたは形状

void pg_fatal(const char *fmt, ...)

入力と出力

  • **入力**: fmt（printf形式のフォーマット文字列）、および可変長引数。
  • **出力**: なし（関数は復帰しません）。

前提条件と不変条件

  • 呼び出し元は、プログラムの継続が不可能な致命的エラーが発生した状態である必要があります。
  • 本関数は呼び出し元へ制御を戻しません。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数を初期化します。 2. pg_log_v を PG_FATAL 型で呼び出し、ログ出力およびユーザーへのメッセージ表示を行います。 3. va_end で引数リストを終了します。 4. printf で終了メッセージを出力します。 5. exit(1) を呼び出し、プロセスを終了します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_log_v
  • **呼び出し元**: ツール内の各所から呼び出されています（例: contrib/oid2name/oid2name.c, src/bin/initdb/initdb.c 等）。

関連データ構造

  • eLogType（pg_log_v の引数として使用）

失敗モードと境界条件

  • 本関数自体が失敗することはありませんが、exit(1) によりプロセスが強制終了されるため、呼び出し元で確保されたリソースのクリーンアップが適切に行われない可能性があります。

保守担当者向けメモ

  • pg_log_v が内部で改行を付加するため、fmt に改行を含める必要はありません。
  • ソースコード上の /* NOTREACHED */ コメントが示す通り、本関数以降のコードは実行されません。
