dsa_create_ext

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:420

目的

動的共有メモリ（DSM）セグメントを新規作成し、その領域内にプロセス間で共有可能なメモリ管理領域（dsa_area）を初期化する。

シグネチャまたは形状

dsa_area *dsa_create_ext(int tranche_id, size_t init_segment_size, size_t max_segment_size)

入力と出力

  • **入力**:
  • tranche_id: ロック取得に使用するトランシェID。
  • init_segment_size: 最初に作成する DSM セグメントのサイズ。
  • max_segment_size: この DSA 領域が許容する最大セグメントサイズ。
  • **出力**:
  • 初期化された dsa_area 構造体へのポインタ。

前提条件と不変条件

  • 作成された DSM セグメントは dsm_pin_segment によりピン留めされ、DSA が寿命を明示的に制御する状態となる。
  • 制御セグメントがデタッチされる際、dsa_on_dsm_detach_release_in_place が実行されるよう登録される。

アルゴリズムまたは振る舞い

1. dsm_create を呼び出し、指定されたサイズで DSM セグメントを確保する。 2. dsm_pin_segment を実行し、セグメントをピン留めする。 3. create_internal を呼び出し、DSM セグメントのアドレス上に制御オブジェクトを配置して dsa_area を構築する。 4. on_dsm_detach を使用し、制御セグメントのデタッチ時にクリーンアップ処理が実行されるようフックを登録する。 5. 初期化済みの dsa_area を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tidstore.c:230
  • **呼び出し先**:
  • dsm_create
  • dsm_pin_segment

  • dsm_segment_address
  • dsm_segment_handle

関連データ構造

  • dsa_area: DSA 領域の状態を保持する構造体。
  • dsm_segment: 基盤となる動的共有メモリセグメント。

失敗モードと境界条件

  • DSM セグメントの作成失敗（メモリ不足等）時は dsm_create がエラーを発生させる。
  • 内部的な領域構築（create_internal）が失敗した場合の挙動は、当該関数の実装に依存する。

保守担当者向けメモ

  • 本関数は dsa_create_in_place_ext と異なり、新規セグメントの作成から行う。
  • ピン留めされたセグメントは、DSA 領域の生存期間を保証するために不可欠である。
