DefineOpClass

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:332

目的

CREATE OPERATOR CLASS SQLコマンドのメインエントリポイントとして、新しい演算子クラスをシステムカタログに定義し、必要な依存関係を構築します。

シグネチャまたは形状

ObjectAddress DefineOpClass(CreateOpClassStmt *stmt)

入力と出力

  • **入力**: CreateOpClassStmt *stmt（解析済みの CREATE OPERATOR CLASS 文の構造体）
  • **出力**: ObjectAddress（作成された演算子クラスのオブジェクト識別子）

前提条件と不変条件

  • 呼び出し元はスーパーユーザーである必要があります。
  • 指定されたアクセスメソッド（AM）が存在する必要があります。
  • 対象の名前空間に対する CREATE 権限が必要です。

アルゴリズムまたは振る舞い

1. **名前空間の特定**: QualifiedNameGetCreationNamespace を使用して、演算子クラスを作成する名前空間を特定し、権限を確認します。 2. **AMの検証**: SearchSysCache1 でアクセスメソッドを検索し、GetIndexAmRoutineByAmId でAMのルーチン情報を取得します。 3. **演算子ファミリーの解決**: stmt->opfamilyname が指定されていればそのOIDを取得し、未指定の場合は既存のファミリーを検索するか、CreateOpFamily を呼び出して新規作成します。 4. **アイテムの走査**: stmt->items リストをループし、演算子（OPCLASS_ITEM_OPERATOR）やサポート関数の定義を検証・抽出します。

  • 演算子番号がAMの許容範囲内かチェックします。
  • 演算子や関数のOIDを解決します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQLコマンド実行時）
  • **呼び出し先**:
  • QualifiedNameGetCreationNamespace, object_aclcheck, SearchSysCache1, GetIndexAmRoutineByAmId
  • get_opfamily_oid, CreateOpFamily
  • LookupOperWithArgs, LookupOperName

関連データ構造

  • CreateOpClassStmt: SQL文の解析結果を保持する構造体。
  • IndexAmRoutine: アクセスメソッドの仕様（戦略数、サポート関数数など）を保持する構造体。
  • OpFamilyMember: 演算子やサポート関数の情報を保持する内部構造体。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザーでない場合、または名前空間への作成権限がない場合にエラーを報告します。
  • **オブジェクト不在**: 指定したアクセスメソッドが存在しない場合にエラーを報告します。
  • **範囲外エラー**: 演算子番号がAMの定義する amstrategies の範囲外である場合にエラーを報告します。

保守担当者向けメモ

  • 現在、演算子クラスの作成にはスーパーユーザー権限が必須です。これは、提供された演算子や関数がAMの期待する仕様と整合しているかを検証する安全な手段がないためです。
  • NOT_USED としてコメントアウトされているコードブロックは、将来的に検証ロジックが改善された場合に再検討可能です。
