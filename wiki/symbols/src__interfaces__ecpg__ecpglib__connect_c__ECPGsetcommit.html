ECPGsetcommit

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/connect.c:157

目的

ECPG接続におけるオートコミットモード（on / off）の切り替えを行い、必要に応じてトランザクションの開始（BEGIN）または終了（COMMIT）をデータベースに対して発行する。

シグネチャまたは形状

bool ECPGsetcommit(int lineno, const char *mode, const char *connection_name)

入力と出力

  • **入力**:
  • lineno: 呼び出し元のソースコード行番号（ログ出力用）。
  • mode: 設定するモード文字列（"on" または "off"）。
  • connection_name: 対象とする接続名（NULLの場合はカレント接続）。
  • **出力**:
  • bool: 処理が成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • ecpg_get_connection により、指定された接続名に対応する connection 構造体が取得可能であること。
  • ecpg_init による接続の初期化が成功すること。

アルゴリズムまたは振る舞い

1. ecpg_get_connection で対象の接続を取得し、ecpg_init で初期化を確認する。 2. mode が "off" で現在の autocommit が有効な場合:

  • トランザクションが PQTRANS_IDLE 状態であれば、BEGIN を実行する。
  • autocommit フラグを false に設定する。

    • 3. mode が "on" で現在の autocommit が無効な場合:

  • トランザクションが PQTRANS_IDLE 状態以外であれば、COMMIT を実行する。
  • autocommit フラグを true に設定する。

    • 4. 各SQL実行結果は ecpg_check_PQresult で検証され、エラー時は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGで生成されたCコード（例: sql-array.c, sql-createtableas.c 等のテストコード）。
  • **呼び出し先**:
  • ecpg_get_connection
  • ecpg_init
  • ecpg_log
  • PQtransactionStatus
  • PQexec
  • ecpg_check_PQresult
  • PQclear

関連データ構造

  • struct connection: 接続状態および autocommit フラグを保持する構造体。
  • PGresult: libpq のクエリ結果構造体。

失敗モードと境界条件

  • ecpg_init が失敗した場合、関数は即座に false を返す。
  • PQexec による BEGIN または COMMIT が失敗した場合、ecpg_check_PQresult を経由して false を返す。
  • mode に "on"/"off" 以外の値が渡された場合、内部の strncmp 条件に合致せず、状態変更は行われない。

保守担当者向けメモ

  • 本関数は libpq のトランザクション状態（PQTRANS_IDLE）に依存して動作する。
  • autocommit の切り替えは、接続構造体の autocommit メンバとデータベース側のトランザクション状態を同期させる必要があるため、状態遷移の整合性に注意すること。
