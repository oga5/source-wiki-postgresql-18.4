r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_catalan.c:1183

目的

単語内の語幹抽出処理において、接尾辞削除の可否を判定するための基準となる領域（R1, R2）をマークします。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体）
  • 出力: int（常に 1 を返します）
  • 副作用: z->I[1] にR1の開始位置、z->I[0] にR2の開始位置を格納します。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要があります。
  • z->l は入力文字列の長さを保持している必要があります。

アルゴリズムまたは振る舞い

1. 初期状態として、R1およびR2の開始位置を文字列の末尾（z->l）に設定します。 2. 母音（g_v）の出現を基準に領域を特定します。

  • out_grouping_U（非母音）の後に in_grouping_U（母音）が続く箇所を検索します。
  • 最初にこのパターンが見つかった位置をR1の開始位置（z->I[1]）とします。
  • さらにその後に続く同様のパターンを検索し、見つかった位置をR2の開始位置（z->I[0]）とします。

    • 3. 検索に失敗した場合は、初期値（文字列末尾）を維持します。

呼び出し元と呼び出し先

  • 呼び出し元: catalan_UTF_8_stem（ステミング処理のメインエントリーポイント）から呼び出されることが想定されています。
  • 呼び出し先: out_grouping_U および in_grouping_U（いずれも libstemmer の共通ユーティリティ）。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。
  • g_v: 母音を定義するグループ構造体。

失敗モードと境界条件

  • 文字列が短い、または母音のパターンが存在しない場合、R1およびR2は文字列の末尾に設定されます。
  • 処理中に検索が失敗した場合は lab0 ラベルへジャンプし、カーソル位置を復元して終了します。
