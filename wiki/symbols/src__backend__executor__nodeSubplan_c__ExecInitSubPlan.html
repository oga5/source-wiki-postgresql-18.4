ExecInitSubPlan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:809

目的

サブクエリ（SubPlan）を実行するための状態管理構造体 SubPlanState を初期化し、実行に必要なメモリコンテキストや投影（Projection）情報を構築する。

シグネチャまたは形状

SubPlanState *ExecInitSubPlan(SubPlan *subplan, PlanState *parent)

入力と出力

  • **入力**:
  • subplan: 初期化対象の SubPlan ノード。
  • parent: このサブプランを保持する親の PlanState。
  • **出力**:
  • 初期化済みの SubPlanState 構造体へのポインタ。

前提条件と不変条件

  • parent->state（EState）内に対応する subplan->plan_id の実行状態が既に存在している必要がある。
  • subplan->plan_id に対応する planstate が見つからない場合、エラーが発生する。
  • ハッシュ化が必要なサブプランの場合、testexpr は OpExpr または AND 句である必要がある。

アルゴリズムまたは振る舞い

1. SubPlanState を割り当て、parent および estate とリンクする。 2. ExecInitExpr を呼び出し、サブプランのテスト式（testexpr）を初期化する。 3. InitPlan（一度だけ評価されるサブプラン）の場合、出力パラメータ（setParam）を estate->es_param_exec_vals に登録し、実行時に評価されるよう設定する。 4. subplan->useHashTable が真の場合、ハッシュテーブル用のメモリコンテキスト（hashtablecxt, hashtempcxt）および式評価用のコンテキスト（innerecontext）を作成する。 5. testexpr を解析し、ハッシュ結合に必要な左辺・右辺のターゲットリスト、等価関数、ハッシュ関数を抽出・初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExpr.c
  • src/backend/executor/execMain.c
  • src/backend/executor/execProcnode.c
  • **呼び出し先**:
  • ExecInitExpr
  • AllocSetContextCreate
  • CreateExprContext
  • fmgr_info
  • get_compatible_hash_operators
  • list_nth（estate->es_subplanstates へのアクセス）

関連データ構造

  • SubPlanState: 本関数が構築するメインの実行状態構造体。
  • SubPlan: 実行対象のサブクエリ定義。
  • PlanState: 親プランの実行状態。
  • EState: クエリ実行全体のグローバル状態。

失敗モードと境界条件

  • planstate が NULL の場合: 「subplan was not initialized」というエラーで停止する。
  • testexpr がハッシュ化に適さない型の場合: 「unrecognized testexpr type」エラーで停止する。
  • 互換性のあるハッシュ演算子が見つからない場合: 「could not find compatible hash operator」エラーで停止する。

保守担当者向けメモ

  • InitPlan はクエリ実行中に一度だけ評価されるため、parent->chgParam の更新は行わない。
  • ハッシュテーブルの生成は本関数では行わず、初回実行時に遅延評価される。
  • CTE_SUBLINK の場合はパラメータ評価の対象外となるため注意が必要。
