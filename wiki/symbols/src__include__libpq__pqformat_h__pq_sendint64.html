pq_sendint64

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/libpq/pqformat.h:151

目的

8バイトの整数値（uint64）をバイナリ形式で StringInfo バッファに追加します。

シグネチャまたは形状

static inline void pq_sendint64(StringInfo buf, uint64 i)

入力と出力

  • **入力**:
  • buf: データを書き込む対象の StringInfo 構造体へのポインタ。
  • i: 書き込む uint64 型の整数値。
  • **出力**: なし（副作用として buf の内容が更新されます）。

前提条件と不変条件

  • buf は初期化済みの有効な StringInfo 構造体である必要があります。
  • 書き込みに必要なバッファ容量が不足している場合、内部で enlargeStringInfo が呼び出され、バッファが自動的に拡張されます。

アルゴリズムまたは振る舞い

1. enlargeStringInfo を呼び出し、8バイトの書き込みに必要な領域を確保します。 2. pq_writeint64 を呼び出し、指定された整数値をネットワークバイトオーダー（ビッグエンディアン）に変換してバッファに書き込みます。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • enlargeStringInfo
  • pq_writeint64
  • **呼び出し元**:
  • src/backend/backup/basebackup_copy.c
  • src/backend/libpq/pqformat.c
  • src/backend/replication/logical/proto.c

関連データ構造

  • StringInfo: 通信データのバッファを管理する構造体。

失敗モードと境界条件

  • メモリ不足が発生した場合、enlargeStringInfo 内でエラーが発生し、処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、呼び出し時にバッファの動的拡張を自動的に処理します。バッファ容量が既に確保されていることが保証されている場合は、オーバーヘッドを避けるために pq_writeint64 を直接使用することを検討してください。
