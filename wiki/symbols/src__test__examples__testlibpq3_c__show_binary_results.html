show_binary_results

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlibpq3.c:59

目的

バイナリ形式で取得された PostgreSQL のクエリ結果（PGresult）を解析し、各フィールドの値を標準出力に整形して表示するヘルパー関数です。

シグネチャまたは形状

static void show_binary_results(PGresult *res)

入力と出力

  • **入力**: PGresult *res - バイナリ形式のデータを含む結果セットへのポインタ。
  • **出力**: なし（標準出力への書き出しのみ）。

前提条件と不変条件

  • res は有効な PGresult オブジェクトを指している必要があります。
  • 結果セットには "i" (INT4), "t" (TEXT), "b" (BYTEA) という名前のフィールドが含まれていることを前提としています。
  • 各フィールドの値が NULL でないことを前提としています（NULL チェックは行われません）。

アルゴリズムまたは振る舞い

1. PQfnumber を使用して、結果セット内の "i"、"t"、"b" フィールドのインデックスを特定します。 2. PQntuples で取得した行数分ループし、各行に対して以下を実行します。

  • PQgetvalue で各フィールドのポインタを取得します。
  • "i" フィールド（INT4）に対し、ntohl を用いてネットワークバイトオーダーからホストバイトオーダーへ変換します。
  • "t" フィールド（TEXT）を C 文字列として扱います。
  • "b" フィールド（BYTEA）に対し、PQgetlength で取得した長さを基に、各バイトを 8 進数形式（\000）で出力します。

    • 3. 各フィールドのバイト長と値を標準出力に表示します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/examples/testlibpq3.c 内の main 関数（185行目および223行目）。
  • **呼び出し先**: libpq API（PQfnumber, PQntuples, PQgetvalue, PQgetlength）および標準ライブラリの ntohl, printf。

関連データ構造

  • PGresult: libpq がクエリ結果を保持するために使用する構造体。

失敗モードと境界条件

  • **NULL 値**: フィールド値が NULL の場合、PQgetvalue は空文字列を返す可能性がありますが、本関数は NULL チェックを行わないため、予期せぬ動作を引き起こす可能性があります。
  • **フィールドの欠如**: PQfnumber がフィールドを見つけられない場合、インデックスとして -1 が返され、その後の PQgetvalue 等でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は testlibpq3.c 専用のデモンストレーション用コードです。実運用環境で使用する場合は、NULL 値の判定（PQgetisnull）を追加することを推奨します。
  • バイナリデータの出力には PQgetlength を使用しており、strlen を使用しないことで、bytea 型に含まれるヌルバイトを安全に処理しています。
