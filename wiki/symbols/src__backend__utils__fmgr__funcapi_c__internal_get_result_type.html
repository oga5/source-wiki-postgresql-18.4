internal_get_result_type

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/funcapi.c:429

目的

関数呼び出し情報に基づき、関数の戻り値の型（TypeFuncClass）を分類し、必要に応じて型IDおよびタプル記述子（TupleDesc）を解決・構築する内部ユーティリティ関数です。

シグネチャまたは形状

static TypeFuncClass internal_get_result_type(Oid funcid,
                                              Node *call_expr,
                                              ReturnSetInfo *rsinfo,
                                              Oid *resultTypeId,
                                              TupleDesc *resultTupleDesc)

入力と出力

  • **入力**:
  • funcid: 対象関数のOID。
  • call_expr: 関数呼び出しの式ツリー（多相型解決に使用）。
  • rsinfo: ReturnSetInfo 構造体（TYPEFUNC_RECORD の場合に期待される記述子を取得）。
  • resultTypeId: 戻り値の型OIDを格納するポインタ（任意）。
  • resultTupleDesc: 戻り値のタプル記述子を格納するポインタ（任意）。
  • **出力**:
  • 戻り値: TypeFuncClass（TYPEFUNC_COMPOSITE, TYPEFUNC_SCALAR, TYPEFUNC_RECORD 等）。

前提条件と不変条件

  • funcid に対応する pg_proc エントリがシステムカタログに存在すること。
  • 多相型が含まれる場合、call_expr から実際の型が推論可能であること。

アルゴリズムまたは振る舞い

1. **カタログ検索**: PROCOID キャッシュから関数定義を取得。 2. **OUTパラメータの確認**: build_function_result_tupdesc_t を呼び出し、OUTパラメータがある場合は複合型として処理。多相型が含まれる場合は resolve_polymorphic_tupdesc で解決を試みる。 3. **スカラー多相型の解決**: 戻り値が多相型の場合、exprType(call_expr) を用いて実際の型を特定。 4. **型分類**: get_type_func_class を使用して戻り値のクラスを判定。

  • TYPEFUNC_COMPOSITE / TYPEFUNC_COMPOSITE_DOMAIN: 関連するタプル記述子をコピーして取得。
  • TYPEFUNC_RECORD: rsinfo->expectedDesc が存在すればそれを採用し、TYPEFUNC_COMPOSITE として扱う。

    • 5. **クリーンアップ**: ReleaseSysCache を呼び出し、検索したキャッシュエントリを解放して終了。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • get_func_result_type
  • get_expr_result_type
  • get_call_result_type

    • （いずれも src/backend/utils/fmgr/funcapi.c 内）

  • **呼び出し先**:
  • SearchSysCache1, ReleaseSysCache
  • build_function_result_tupdesc_t
  • resolve_polymorphic_tupdesc
  • exprType
  • get_type_func_class
  • lookup_rowtype_tupdesc_copy

関連データ構造

  • TypeFuncClass: 戻り値の型分類を示す列挙型。
  • ReturnSetInfo: SRFの実行コンテキスト情報。
  • TupleDesc: 複合型の構造定義。
  • Form_pg_proc: pg_proc カタログの行構造。

失敗モードと境界条件

  • funcid が無効な場合、elog(ERROR) で停止。
  • 多相型の解決に失敗し、かつ型が特定できない場合、ereport(ERROR) で停止。
  • TYPEFUNC_RECORD で rsinfo が提供されない場合、タプル記述子は解決されず NULL となる。

保守担当者向けメモ

  • 本関数は get_call_result_type 等の公開APIのバックエンドとして機能します。
  • TYPEFUNC_RECORD の解決において rsinfo->expectedDesc に依存しているため、呼び出し側が適切なコンテキストを提供しているか確認が必要です。
