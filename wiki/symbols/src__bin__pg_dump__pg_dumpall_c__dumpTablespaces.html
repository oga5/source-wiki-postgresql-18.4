dumpTablespaces

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dumpall.c:1381

目的

PostgreSQL クラスタからユーザー定義のテーブルスペース情報を抽出し、CREATE TABLESPACE 等の SQL 文として出力する。

シグネチャまたは形状

static void dumpTablespaces(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（データベース接続ハンドル）
  • **出力**: なし（直接グローバル変数 OPF に SQL を書き出す）

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • 組み込みのテーブルスペース（pg_ で始まる名前）は除外される。
  • binary_upgrade モード時は、OID を保持するための binary_upgrade_set_next_pg_tablespace_oid が出力される。

アルゴリズムまたは振る舞い

1. pg_catalog.pg_tablespace からテーブルスペースの属性（OID、名前、所有者、場所、ACL、オプション、コメント）を取得する。 2. 取得したテーブルスペースが存在する場合、ヘッダーを出力する。 3. 各テーブルスペースに対して以下の処理を行う：

  • CREATE TABLESPACE 文を構築する。
  • LOCATION を判定し、絶対パスであればそのまま、相対パスであれば空文字列として出力する。
  • オプション（spcoptions）がある場合は ALTER TABLESPACE ... SET を出力する。
  • skip_acls が無効な場合、buildACLCommands を用いて権限設定を出力する。
  • no_comments が無効な場合、COMMENT ON TABLESPACE を出力する。
  • no_security_labels が無効な場合、buildShSecLabels を用いてセキュリティラベルを出力する。

    • 4. 最後に PQclear で結果セットを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dumpall.c 内の main 関数（行665）。
  • **呼び出し先**: executeQuery, PQntuples, createPQExpBuffer, atooid, PQgetvalue, appendPQExpBuffer, appendPQExpBufferStr, fmtId, is_absolute_path, appendStringLiteralConn, buildACLCommands, pg_log_error, PQfinish, exit_nicely, buildShSecLabels, PQclear, destroyPQExpBuffer。

関連データ構造

  • PGconn: データベース接続状態。
  • PGresult: クエリ結果セット。
  • PQExpBuffer: SQL 文構築用のバッファ。

失敗モードと境界条件

  • buildACLCommands が失敗した場合、エラーログを出力して exit_nicely(1) によりプロセスを終了する。
  • テーブルスペースが存在しない場合、何も出力せず正常終了する。

保守担当者向けメモ

  • 本関数は pg_dumpall の一部であり、クラスタ全体のダンプ順序において重要な役割を果たす。
  • fmtId を使用して識別子をクォートしているため、テーブルスペース名に特殊文字が含まれていても安全に出力される。
  • is_absolute_path によるパス判定ロジックは、インプレーステーブルスペースのダンプ要件に依存している。
