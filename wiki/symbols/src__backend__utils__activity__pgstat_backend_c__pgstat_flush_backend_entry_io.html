pgstat_flush_backend_entry_io

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/activity/pgstat_backend.c:165

目的

プロセスローカルなバッファ（PendingBackendStats）に蓄積された I/O 統計情報を、共有メモリ上の統計エントリへ反映（フラッシュ）し、ローカルバッファをリセットする。

シグネチャまたは形状

void pgstat_flush_backend_entry_io(PgStat_EntryRef *entry_ref)

入力と出力

  • **入力**: entry_ref (共有メモリ上の統計エントリへの参照)
  • **出力**: なし（副作用として共有メモリの状態を更新し、ローカルバッファをクリアする）

前提条件と不変条件

  • backend_has_iostats フラグが true の場合にのみ、共有メモリへの書き込み処理が実行される。
  • 共有メモリへの書き込みは、entry_ref が適切にロックされた状態で行われる必要がある（呼び出し元の責務）。

アルゴリズムまたは振る舞い

1. backend_has_iostats を確認し、false であれば何もせず終了する。 2. entry_ref から共有メモリ上の PgStatShared_Backend 構造体を取得する。 3. IOOBJECT_NUM_TYPES、IOCONTEXT_NUM_TYPES、IOOP_NUM_TYPES の各次元でループし、ローカルの PendingBackendStats.pending_io に蓄積された値を共有メモリ側の統計値に加算する。 4. 時間統計については INSTR_TIME_GET_MICROSEC を使用してマイクロ秒単位に変換して加算する。 5. MemSet を使用してローカルの pending_io バッファをゼロクリアする。 6. backend_has_iostats を false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat_flush_backend（src/backend/utils/activity/pgstat_backend.c:297）から呼び出される。
  • **呼び出し先**: 共有メモリ操作および統計計算に関連する内部マクロ・関数（INSTR_TIME_GET_MICROSEC, MemSet 等）。

関連データ構造

  • PgStat_EntryRef: 共有メモリ上の統計エントリへの参照を保持。
  • PgStatShared_Backend: 共有メモリ上に配置されたバックエンド統計の本体。
  • PendingBackendStats: プロセスローカルな統計保持用構造体。

失敗モードと境界条件

  • 統計情報が蓄積されていない場合（backend_has_iostats が false）、処理は早期リターンし、共有メモリへの変更は行われない。
  • 共有メモリへのアクセスは、呼び出し元が適切なロックを保持していることを前提としている。

保守担当者向けメモ

  • 本関数はメモリ割り当てを行わない設計となっているため、クリティカルセクション内からの呼び出しが可能である。
  • 統計の加算処理は、各次元の定数（IOOBJECT_NUM_TYPES 等）に依存しているため、統計項目が増減した場合はこれらの定数定義と整合性を保つ必要がある。
