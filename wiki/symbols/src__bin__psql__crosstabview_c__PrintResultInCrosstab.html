PrintResultInCrosstab

定義

  • 種別: function
  • 定義箇所: src/bin/psql/crosstabview.c:103

目的

クエリ結果セットをクロス集計（ピボットテーブル）形式に変換し、psql 上で表示するメインエントリポイントです。

シグネチャまたは形状

bool PrintResultInCrosstab(const PGresult *res)

入力と出力

  • **入力**: res (実行済みの PGresult ポインタ)
  • **出力**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • res は PGRES_TUPLES_OK ステータスである必要があります。
  • クエリ結果は少なくとも3列以上を含んでいる必要があります。
  • 垂直ヘッダー（行）と水平ヘッダー（列）に使用する列は、互いに異なる列である必要があります。
  • 内部的に使用される CROSSTABVIEW_MAX_COLUMNS (1600) を超える列が生成される場合はエラーとなります。

アルゴリズムまたは振る舞い

1. **初期化**: 行・列用の avl_tree を初期化します。 2. **引数解析**: pset.ctv_args を参照し、行・列・データ・ソート用の各カラムインデックスを特定します。 3. **集計**: 全タプルを走査し、行と列のヘッダー値を avl_tree に挿入して一意な値のセットを作成します。 4. **配列化**: avlCollectFields を使用して、木構造からソート済みの pivot_field 配列を生成します。 5. **ソート**: ソート列が指定されている場合、rankSort を呼び出して列の順序を調整します。 6. **描画**: printCrosstab を呼び出し、構築されたヘッダーとデータを用いてテーブルを出力します。 7. **終了処理**: 使用したメモリ（AVL木および配列）を解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/common.c:1057
  • **呼び出し先**: avlInit, avlMergeValue, avlCollectFields, rankSort, printCrosstab, avlFree, pg_malloc, pg_free 等。

関連データ構造

  • pivot_field: ヘッダー値やランク情報を保持する構造体。
  • avl_tree: ヘッダー値の重複排除とソートに使用される平衡二分探索木。

失敗モードと境界条件

  • 結果セットが3列未満の場合、または垂直・水平ヘッダーが同一列を指定している場合はエラーとなります。
  • データ列が指定されておらず、かつ結果セットが3列以外の場合、一意なデータ列を特定できずエラーとなります。
  • CROSSTABVIEW_MAX_COLUMNS を超える列が生成されるとエラーとなります。
  • indexOfColumn 等の内部関数で列の特定に失敗した場合、処理を中断します。

保守担当者向けメモ

  • 本関数は \crosstabview コマンドの実行フローを統括します。
  • メモリ解放は error_return ラベルで一元管理されており、異常終了時も適切にクリーンアップされます。
