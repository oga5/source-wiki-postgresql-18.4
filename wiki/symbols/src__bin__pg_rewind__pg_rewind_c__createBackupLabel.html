createBackupLabel

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/pg_rewind.c:962

目的

ターゲットデータディレクトリに backup_label ファイルを生成し、リカバリ開始位置とチェックポイント位置を記録することで、同期後のクラスタが新しいタイムラインで起動できるようにする。

シグネチャまたは形状

static void createBackupLabel(XLogRecPtr startpoint, TimeLineID starttli, XLogRecPtr checkpointloc)

入力と出力

  • **入力**:
  • startpoint: リカバリを開始する WAL 位置。
  • starttli: リカバリ開始時のタイムライン ID。
  • checkpointloc: チェックポイントの WAL 位置。
  • **出力**: なし（副作用としてターゲットディレクトリに backup_label ファイルを作成）。

前提条件と不変条件

  • WalSegSz が適切に初期化されていること。
  • open_target_file および write_target_range がターゲットディレクトリへの書き込み権限を持っていること。

アルゴリズムまたは振る舞い

1. XLByteToSeg を使用して startpoint から WAL セグメント番号を算出する。 2. XLogFileName を使用して、対応する WAL ファイル名を生成する。 3. 現在時刻を取得し、フォーマットされた文字列を作成する。 4. snprintf を用いて、backup_label ファイルの内容（WAL 位置、チェックポイント位置、バックアップ手法、開始時刻など）をバッファに書き込む。 5. open_target_file("backup_label", true) を呼び出し、ターゲットディレクトリにファイルを作成する。 6. write_target_range を使用してバッファの内容をファイルに書き込み、close_target_file で閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c:680
  • **呼び出し先**:
  • XLByteToSeg, XLogFileName (マクロ/関数)
  • time, localtime, strftime (標準ライブラリ)
  • snprintf (標準ライブラリ)
  • open_target_file, write_target_range, close_target_file (ファイル操作ユーティリティ)
  • pg_fatal (エラーハンドリング)

関連データ構造

  • XLogRecPtr: WAL 位置を保持するデータ型。
  • TimeLineID: タイムライン識別子。

失敗モードと境界条件

  • snprintf の結果がバッファサイズ（1000バイト）を超える場合、pg_fatal によりプログラムが異常終了する。
  • ターゲットディレクトリへのファイル作成や書き込みが権限不足やディスクフルで失敗した場合、open_target_file 等の内部でエラーが発生する。
