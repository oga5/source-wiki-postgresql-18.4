ParallelState

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/parallel.h:55

目的

pg_dump および pg_restore における並列処理の実行状態を管理する構造体です。リーダープロセスとワーカープロセス間の通信、および各ワーカーのタスク割り当て状況を保持します。

シグネチャまたは形状

typedef struct ParallelState
{
    int           numWorkers;      /* 許可されたワーカー数 */
    TocEntry    **te;              /* 各ワーカーが処理中のアイテム、またはNULL */
    ParallelSlot *parallelSlot;    /* 各ワーカーに関するプライベート情報 */
} ParallelState;

入力と出力

  • 入力: numWorkers は並列実行されるワーカーの最大数を定義します。
  • 出力: te 配列を通じて、各ワーカーが現在担当している TocEntry を追跡します。

前提条件と不変条件

  • te および parallelSlot 配列は、numWorkers のサイズを持ちます。
  • ParallelSlot は不透明ポインタとして扱われ、内部構造は parallel.c でのみ定義されます。

アルゴリズムまたは振る舞い

  • リーダープロセスは本構造体を使用して、ワーカーのライフサイクル（開始、ジョブ割り当て、終了）を制御します。
  • DispatchJobForTocEntry を通じて TocEntry がワーカーに割り当てられ、te 配列が更新されます。
  • WaitForWorkers などの関数により、ワーカーの完了監視や同期が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/parallel.c 内の複数の関数（GetMyPSlot, ShutdownWorkersHard, WaitForTerminatingWorkers, set_cancel_pstate, GetIdleWorker, HasEveryWorkerTerminated, ListenToWorkers, getMessageFromWorker, sendMessageToWorker など）が ParallelState を引数として受け取ります。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • TocEntry: 各ワーカーが処理する対象アイテム。
  • ParallelSlot: ワーカーごとの詳細な実行状態を保持する構造体。
  • ArchiveHandle: 並列処理の基盤となるアーカイブ制御構造体。

失敗モードと境界条件

  • Windows 環境では、MAXIMUM_WAIT_OBJECTS の制限により、numWorkers は最大 64 に制限されます。
  • ワーカープロセスはリーダーがカタログを読み込んだ後にフォークされるため、カタログ情報へのアクセス権を継承します。

保守担当者向けメモ

  • 本構造体のメモリ管理は parallel.c 内の関数で行われます。
  • ParallelSlot の内部構造を変更する場合は、parallel.c の実装との整合性に注意してください。
