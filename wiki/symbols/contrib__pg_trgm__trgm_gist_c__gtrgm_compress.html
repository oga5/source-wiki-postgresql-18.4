gtrgm_compress

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_gist.c:114

目的

GiST インデックス構築時、入力データ（文字列または既存のシグネチャ）をトリグラムベースのインデックスキーに圧縮・変換する。

シグネチャまたは形状

Datum gtrgm_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体を受け取る。
  • **出力**: 圧縮または処理済みの GISTENTRY を Datum として返す。

前提条件と不変条件

  • GET_SIGLEN() により、現在のインデックス設定に基づいたシグネチャ長が取得可能であること。
  • entry->leafkey が真の場合、entry->key はテキストデータとして扱われる。

アルゴリズムまたは振る舞い

1. **リーフノードの場合**: generate_trgm を呼び出し、入力文字列からトリグラムを生成して新しい GISTENTRY を作成・返却する。 2. **内部ノードの場合**: 既存のシグネチャが「すべて真（全ビットが1）」でないかを確認する。

  • 全ビットが1でない場合、gtrgm_alloc を使用してシグネチャを再割り当てし、新しい GISTENTRY を作成して返却する。
  • すべてのビットが1である場合は、元の entry をそのまま返す。

    • 3. いずれの条件にも該当しない場合は、元の entry をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセスから呼び出される（直接的な呼び出し元関数の詳細はソースコード内に明示なし）。
  • **呼び出し先**: generate_trgm（トリグラム生成）、gtrgm_alloc（メモリ確保）、gistentryinit（エントリ初期化）。

関連データ構造

  • GISTENTRY: GiST インデックスのエントリ構造体。
  • TRGM: トリグラムデータを保持する構造体。
  • BITVECP: シグネチャ（ビットベクトル）を操作するためのポインタ型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる可能性がある。
  • シグネチャが既に「すべて真」の状態である場合、圧縮処理をスキップして元のエントリを返す。

保守担当者向けメモ

  • 本関数は GiST インデックスの圧縮フェーズを担う。trgm_op.c で定義される generate_trgm に強く依存している。
  • siglen の値によってシグネチャのビット操作範囲が動的に決定される点に注意すること。
