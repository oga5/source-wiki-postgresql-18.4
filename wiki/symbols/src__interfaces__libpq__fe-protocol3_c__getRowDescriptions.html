getRowDescriptions

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:517

目的

PostgreSQL プロトコル 3.0 における RowDescription ('T') メッセージを解析し、結果セットの属性定義（列名、型 ID、サイズ等）を PGresult 構造体に格納する。

シグネチャまたは形状

static int getRowDescriptions(PGconn *conn, int msgLength)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体。
  • msgLength: 受信したメッセージの長さ。
  • **出力**:
  • 戻り値: 成功時およびエラー処理完了時に 0 を返す。
  • 副作用: conn->result に解析結果を格納、またはエラー発生時に conn->errorMessage を更新し pqSaveErrorResult を実行する。

前提条件と不変条件

  • pqParseInput3 によってメッセージタイプ 'T' とメッセージ長が既に読み取り済みであること。
  • conn->inBuffer にメッセージの残りのデータ（フィールド数および各フィールドの属性情報）が格納されていること。

アルゴリズムまたは振る舞い

1. **結果オブジェクトの準備**: 既存の PGresult があれば再利用し、なければ PQmakeEmptyPGresult で新規作成する。 2. **フィールド数の取得**: メッセージから 2 バイトのフィールド数を読み取る。 3. **メモリ確保**: フィールド数に基づき PGresAttDesc 配列を pqResultAlloc で確保する。 4. **属性情報の反復解析**: 各フィールドについて、名前、テーブル ID、列 ID、型 ID、型サイズ、型修飾子、フォーマットを順次読み取る。

  • 2 バイト整数（columnid, typlen, format）は int16 にキャストして符号を補正する。
  • フォーマットが 1（バイナリ）でない列が存在する場合、result->binary を 0 に設定する。

    • 5. **状態更新**: 成功時は conn->asyncStatus を PGASYNC_READY に更新する。 6. **エラー処理**: 解析中にデータ不足やメモリ不足が発生した場合、advance_and_error ラベルへ遷移し、エラーメッセージを構築して inCursor を進める。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c 内のメッセージディスパッチ処理（pqParseInput3 からの呼び出しが示唆されている）。
  • **呼び出し先**: PQmakeEmptyPGresult, pqGetInt, pqGets, pqResultAlloc, pqResultStrdup, PQclear, pqClearAsyncResult, appendPQExpBuffer, pqSaveErrorResult 等。

関連データ構造

  • PGconn: 接続状態および入力バッファを管理。
  • PGresult: 解析結果を保持する構造体。
  • PGresAttDesc: 各列のメタデータを保持する構造体。

失敗モードと境界条件

  • **データ不足**: メッセージ長が期待値に満たない場合、エラーメッセージを生成し、入力カーソルをスキップして処理を終了する。
  • **メモリ不足**: pqResultAlloc や pqResultStrdup が失敗した場合、エラー結果を生成して接続状態をエラーに遷移させる。
  • **不正な型情報**: 2 バイト整数の読み取り時に符号付き変換を行うことで、プロトコル上の表現と内部表現の不整合を防ぐ。

保守担当者向けメモ

  • 本関数はプロトコル 3.0 専用である。
  • pqGetInt は 2 バイト整数を符号なしとして読み取るため、int16 へのキャストによる符号補正が必須である点に注意すること。
  • エラー発生時に inCursor を msgLength 分進めることで、後続のメッセージ解析への影響を遮断している。
