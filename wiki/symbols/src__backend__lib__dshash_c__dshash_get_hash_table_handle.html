dshash_get_hash_table_handle

定義

  • 種別: function
  • 定義箇所: src/backend/lib/dshash.c:366

目的

指定された dshash_table に対応する共有メモリ上のハンドル（dshash_table_handle）を取得します。

シグネチャまたは形状

dshash_table_handle dshash_get_hash_table_handle(dshash_table *hash_table)

入力と出力

  • **入力**: hash_table (dshash_table *) - 操作対象のハッシュテーブルへのポインタ。
  • **出力**: dshash_table_handle - 共有メモリ上のハッシュテーブルを識別するためのハンドル。

前提条件と不変条件

  • hash_table は有効なポインタである必要があります。
  • hash_table->control->magic が DSHASH_MAGIC と一致している必要があります（Assert による検証）。

アルゴリズムまたは振る舞い

1. hash_table が指す制御ブロックの magic 値を検証します。 2. 制御ブロック内に保持されている handle を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/launcher.c
  • src/backend/storage/ipc/dsm_registry.c
  • src/backend/utils/activity/pgstat_shmem.c
  • src/backend/utils/cache/typcache.c
  • **呼び出し先**:
  • 本関数は内部で他の関数を呼び出しません。

関連データ構造

  • dshash_table: バックエンドローカルなハッシュテーブルの状態を保持する構造体。
  • dshash_table_control: 共有メモリ上に配置されるハッシュテーブルの制御ブロック。

失敗モードと境界条件

  • hash_table が NULL の場合、または magic 値が不正な場合はアサーション失敗によりプロセスが異常終了します。

保守担当者向けメモ

  • 本関数は、共有メモリ上のハッシュテーブルを他プロセスと共有するために必要な識別子を取得する軽量なアクセサです。
  • dshash_table_handle は、dshash_attach 等でハッシュテーブルを再接続する際に使用されます。
