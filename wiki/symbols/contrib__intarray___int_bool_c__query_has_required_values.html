query_has_required_values

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_bool.c:396

目的

QUERYTYPE 構造体で保持されるクエリツリー内に、評価に必須となる値（VAL 型のノード）が含まれているかを判定する。

シグネチャまたは形状

bool query_has_required_values(QUERYTYPE *query)

入力と出力

  • **入力**: QUERYTYPE *query - 評価対象のクエリ構造体。
  • **出力**: bool - クエリ内に必須値が存在する場合は true、それ以外（空のクエリや必須値を含まない場合）は false を返す。

前提条件と不変条件

  • query ポインタは有効である必要がある。
  • query->size が 0 以下の場合、必須値は存在しないとみなされる。

アルゴリズムまたは振る舞い

1. query->size が 0 以下であれば false を即座に返す。 2. GETQUERY(query) マクロを使用してクエリの先頭アドレスを取得し、query->size - 1 を加算してポーランド記法で表現されたクエリツリーの末尾（ルートノード）を特定する。 3. contains_required_value 関数を呼び出し、末尾ノードから再帰的に必須値の有無を探索する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gin.c:37 (gin_bool_consistent 等の文脈で使用)
  • **呼び出し先**:
  • contains_required_value (同ファイル内)

関連データ構造

  • QUERYTYPE: クエリを保持する構造体。
  • ITEM: クエリツリーを構成するノード（値または演算子）。

失敗モードと境界条件

  • query->size が 0 の場合、探索を行わずに false を返す。
  • query が指すメモリ領域が不正な場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は contains_required_value へのラッパーとして機能する。
  • ポーランド記法におけるクエリツリーの末尾から探索を開始する仕様であるため、ツリー構造の変更時には GETQUERY のオフセット計算が妥当か確認すること。
