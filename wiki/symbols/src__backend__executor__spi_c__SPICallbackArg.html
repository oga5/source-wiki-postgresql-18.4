SPICallbackArg

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/spi.c:53

目的

SPI（Server Programming Interface）経由で実行されるクエリの解析および実行処理において、エラー報告やコンテキスト情報としてクエリ文字列とその解析モードを保持するために使用されます。

シグネチャまたは形状

typedef struct SPICallbackArg
{
	const char *query;
	RawParseMode mode;
} SPICallbackArg;

入力と出力

  • **入力**: 構造体初期化時に、実行対象のクエリ文字列 (query) と解析モード (mode) が設定されます。
  • **出力**: エラー発生時のコールバック関数等において、この構造体を通じて現在のクエリコンテキストが参照されます。

前提条件と不変条件

  • query は、当該 SPI 操作の生存期間中、有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

SPI 内部の各実行関数（SPI_execute 等）において、エラー発生時に詳細な情報を出力するための引数としてインスタンス化されます。src/backend/executor/spi.c 内の複数の箇所でローカル変数として宣言され、エラーハンドリングのコンテキストを伝達します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/spi.c 内の SPI_execute 関連関数群（行番号 1587, 2080, 2226, 2334, 2410 付近）。
  • **呼び出し先**: エラーハンドリング用のコールバック関数（carg としてキャストされ参照されるケースを確認）。

関連データ構造

  • RawParseMode: クエリの解析モードを定義する列挙型。

失敗モードと境界条件

  • query が NULL または解放済みのメモリを指している場合、エラー報告時に不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は SPI 内部のスタック管理や実行フローにおいて、エラー発生時のデバッグ情報を補完する役割を担っています。変更を加える際は、SPI のエラーハンドリング機構との整合性に注意してください。
