JsonManifestParseState

定義

  • 種別: struct
  • 定義箇所: src/common/parse_manifest.c:67

目的

バックアップマニフェスト（JSON形式）を解析する際、現在の解析フェーズや抽出中のデータ、およびマニフェスト全体のメタデータを保持する内部状態管理構造体です。

シグネチャまたは形状

typedef struct {
    JsonManifestParseContext *context;
    JsonManifestSemanticState state;
    JsonManifestFileField file_field;
    char *pathname;
    char *encoded_pathname;
    char *size;
    char *algorithm;
    pg_checksum_type checksum_algorithm;
    char *checksum;
    JsonManifestWALRangeField wal_range_field;
    char *timeline;
    char *start_lsn;
    char *end_lsn;
    bool saw_version_field;
    char *manifest_version;
    char *manifest_system_identifier;
    char *manifest_checksum;
} JsonManifestParseState;

入力と出力

  • **入力**: 解析対象のJSONデータから抽出された各フィールドの文字列値が、解析プロセスを通じて本構造体の各メンバに一時的に格納されます。
  • **出力**: 解析完了時、蓄積されたデータは JsonManifestParseContext で定義されたコールバック関数へ渡され、適切な型（uint64 や XLogRecPtr 等）へ変換されます。

前提条件と不変条件

  • 本構造体は src/common/parse_manifest.c 内の解析関数によって初期化および管理されます。
  • ファイルパスのエンコード（Encoded-Path）は16進数文字列として保持され、後続の処理でバイナリへ復号されます。

アルゴリズムまたは振る舞い

1. **状態保持**: JsonManifestSemanticState を用いて、現在処理中のJSONオブジェクト（ファイルリスト、WAL範囲など）を追跡します。 2. **データ蓄積**: JSONの各フィールドが走査されるたび、対応するメンバ変数に値が格納されます。 3. **確定処理**: オブジェクトの終了時に json_manifest_finalize_file や json_manifest_finalize_wal_range 等が呼び出され、蓄積された文字列データが型変換されてコールバックへ通知されます。

呼び出し元と呼び出し先

  • **呼び出し元**: json_parse_manifest およびインクリメンタル解析用関数（json_parse_manifest_incremental_init 等）によってインスタンス化されます。
  • **呼び出し先**: json_manifest_finalize_version、json_manifest_finalize_system_identifier、json_manifest_finalize_file、json_manifest_finalize_wal_range、verify_manifest_checksum などの内部関数が本構造体を引数として受け取ります。

関連データ構造

  • JsonManifestParseContext: 解析結果を受け取るコールバック群。
  • JsonManifestSemanticState: 解析の進捗状態を管理する列挙型または構造体。
  • JsonManifestFileField / JsonManifestWALRangeField: 各オブジェクト内のフィールド位置を特定するための補助構造体。

失敗モードと境界条件

  • 解析中にエラーが発生した場合、json_manifest_parse_failure が呼び出され、pg_unreachable() により処理が停止します。
  • インクリメンタル解析を行う場合、マニフェストの最後のチャンクにチェックサムが含まれていないと検証に失敗します。

保守担当者向けメモ

  • 本構造体はフロントエンド（postgres_fe.h）向けに設計されています。
  • pathname 等の文字列メンバは、解析プロセス中に動的に割り当てられるため、メモリ管理に注意してください。
