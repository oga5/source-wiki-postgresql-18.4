RelationInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:157

目的

検証対象となる個々のリレーション（ヒープテーブルまたはB-tree索引）のメタデータを保持し、並列検証プロセスにおける各リレーションの識別とSQL構築を支援する。

シグネチャまたは形状

typedef struct RelationInfo
{
	const DatabaseInfo *datinfo;
	Oid			reloid;
	bool		is_heap;
	char	   *nspname;
	char	   *relname;
	int			relpages;
	int			blocks_to_check;
	char	   *sql;
} RelationInfo;

入力と出力

  • 本構造体は pg_malloc0 を介してメモリ確保され、各フィールドが初期化される。
  • sql フィールドは、検証クエリの実行中に動的に設定され、実行後に pg_free される。

前提条件と不変条件

  • is_heap が true の場合はヒープテーブル、false の場合はB-tree索引として扱われる。
  • datinfo は他の RelationInfo インスタンスと共有される可能性がある。

アルゴリズムまたは振る舞い

  • pg_amcheck の並列実行において、各スロットのコンテキストとして渡され、検証対象の特定および検証用SQLの生成に使用される。

呼び出し元と呼び出し先

  • 呼び出し元: pg_amcheck.c 内のリスト走査処理や並列スロットのコンテキスト初期化箇所（例: 2205行目付近）。
  • 呼び出し先: prepare_heap_command および prepare_btree_command に引数として渡され、SQL構築に利用される。また、verify_heap_slot_handler や verify_btree_slot_handler 内でコンテキストとして参照される。

関連データ構造

  • DatabaseInfo: 所属するデータベースの情報を保持し、datinfo フィールドから参照される。

失敗モードと境界条件

  • sql フィールドのメモリ管理（pg_free）が適切に行われない場合、メモリリークが発生する可能性がある。

保守担当者向けメモ

  • 本構造体は pg_amcheck の並列処理の核となるデータ保持用である。フィールドの追加や変更を行う際は、prepare_heap_command および prepare_btree_command でのSQL構築ロジックへの影響を確認すること。
