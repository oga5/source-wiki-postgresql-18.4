PgFdwConnState

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/postgres_fdw.h:137

目的

リモートの PostgreSQL サーバーに対する非同期リクエストの状態を管理するための構造体です。

シグネチャまたは形状

typedef struct PgFdwConnState
{
    AsyncRequest *pendingAreq;    /* 実行待ちの非同期リクエスト */
} PgFdwConnState;

入力と出力

  • **入力**: AsyncRequest 型のポインタを保持し、非同期処理の追跡に使用されます。
  • **出力**: 接続状態を保持するコンテナとして機能し、GetConnection や pgfdw_exec_query などの関数を通じて接続管理やクエリ実行のコンテキストとして利用されます。

前提条件と不変条件

  • pendingAreq は、現在処理中の非同期リクエストを指すか、リクエストが存在しない場合は NULL である必要があります。

アルゴリズムまたは振る舞い

本構造体は、postgres_fdw におけるリモート接続のライフサイクル管理において、非同期操作の追跡ポイントとして機能します。クエリ実行時や接続取得時に、この構造体を通じて現在の非同期リクエストの状態が参照・更新されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照・利用）**:
  • contrib/postgres_fdw/connection.c: GetConnection, pgfdw_exec_query
  • contrib/postgres_fdw/postgres_fdw.c: 各種エグゼキュータ関連処理
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • AsyncRequest: pendingAreq フィールドが参照する非同期リクエストの定義。
  • PGconn: libpq によるリモート接続オブジェクト（pgfdw_exec_query 等で併用される）。

失敗モードと境界条件

  • pendingAreq が不正なメモリ領域を指している場合、非同期リクエストの処理時にクラッシュする可能性があります。
  • 接続が切断された状態での非同期リクエストの保持については、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は postgres_fdw の接続管理における重要な状態保持コンテナです。非同期処理のロジックを変更する際は、pendingAreq のライフサイクル管理が適切に行われているかを確認してください。
  • ツールによる解析結果から、connection.c および postgres_fdw.c の複数の箇所で本構造体がインスタンス化または参照されていることが確認されています。
