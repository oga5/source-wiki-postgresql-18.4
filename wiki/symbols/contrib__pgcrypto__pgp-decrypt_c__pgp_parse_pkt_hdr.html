pgp_parse_pkt_hdr

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:128

目的

OpenPGP パケットのヘッダーを解析し、パケットタグとデータ長を特定する。

シグネチャまたは形状

int pgp_parse_pkt_hdr(PullFilter *src, uint8 *tag, int *len_p, int allow_ctx)

入力と出力

  • src: 読み取り対象の PullFilter。
  • tag: 解析されたパケットタグを格納するポインタ。
  • len_p: 解析されたパケット長を格納するポインタ。
  • allow_ctx: コンテキスト依存の長さ（Indeterminate length）を許可するかどうかのフラグ。
  • 戻り値: 成功時は 0、EOF 到達時は 0、エラー時は負の px エラーコードを返す。

前提条件と不変条件

  • src は有効な PullFilter であること。
  • パケットヘッダーの先頭バイトの最上位ビット（bit 7）が 1 であること（PGP パケットの仕様）。

アルゴリズムまたは振る舞い

1. pullf_read を使用してヘッダーの最初の 1 バイトを読み取る。 2. 読み取ったバイトの bit 7 が 0 の場合、パケットヘッダーではないと判断し PXE_PGP_CORRUPT_DATA を返す。 3. bit 6 が 1 の場合（New Packet Format）、タグを抽出し parse_new_len を呼び出して長さを取得する。 4. bit 6 が 0 の場合（Old Packet Format）、タグと長さタイプ（lentype）を抽出し、以下のように処理する：

  • lentype が 3 の場合、allow_ctx が真であれば PKT_CONTEXT を返し、偽であれば PXE_PGP_CORRUPT_DATA を返す。
  • それ以外の場合、parse_old_len を呼び出して長さを取得する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-decrypt.c (複数箇所)
  • contrib/pgcrypto/pgp-info.c
  • contrib/pgcrypto/pgp-pubkey.c
  • **呼び出し先**:
  • pullf_read
  • parse_new_len
  • parse_old_len
  • px_debug

関連データ構造

  • PullFilter: データを逐次処理するためのフィルタ構造体。

失敗モードと境界条件

  • pullf_read がエラーを返した場合、そのエラーコードをそのまま返す。
  • パケットヘッダーの形式が不正（bit 7 が 0）な場合、PXE_PGP_CORRUPT_DATA を返す。
  • Old Packet Format で長さタイプが 3 であり、かつ allow_ctx が偽の場合、PXE_PGP_CORRUPT_DATA を返す。

保守担当者向けメモ

  • 本関数は RFC 4880 に基づくパケット解析を行う。
  • px_debug によるログ出力は、パケットヘッダーの不整合を検出した際に行われる。
