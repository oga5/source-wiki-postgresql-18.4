set_new_cluster_char_signedness

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.c:404

目的

新クラスタの char 型の符号付き/符号なし設定（char signedness）を、ユーザー指定値または旧クラスタの設定に合わせて調整する。

シグネチャまたは形状

static void set_new_cluster_char_signedness(void)

入力と出力

  • **入力**: グローバル変数 user_opts.char_signedness および old_cluster.controldata.default_char_signedness を参照。
  • **出力**: なし（副作用として新クラスタの制御データが更新される）。

前提条件と不変条件

  • user_opts および old_cluster が初期化済みであること。
  • new_cluster のパス情報が設定済みであること。

アルゴリズムまたは振る舞い

1. ターゲットとなる符号付き設定を決定する。

  • user_opts.char_signedness が -1 でない場合、その値を優先する。
  • それ以外の場合は、old_cluster.controldata.default_char_signedness を継承する。

    • 2. 新クラスタの現在の設定（new_cluster.controldata.default_char_signedness）と比較する。 3. 設定が異なる場合、pg_resetwal --char-signedness コマンドを実行して新クラスタの設定を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/pg_upgrade.c 内の main 関数（161行目）。
  • **呼び出し先**: 外部コマンド pg_resetwal を exec_prog 経由で呼び出す。

関連データ構造

  • user_opts: ユーザー指定のオプションを保持する構造体。
  • old_cluster, new_cluster: クラスタの制御データ（controldata）を保持する構造体。

失敗モードと境界条件

  • pg_resetwal の実行に失敗した場合、exec_prog の内部処理によりアップグレードプロセスが中断される可能性がある。
  • user_opts.char_signedness が不正な値の場合の挙動は、本関数のロジック外で制御される。

保守担当者向けメモ

  • 本関数は pg_upgrade の初期化フェーズで呼び出される。
  • pg_resetwal を直接呼び出すため、新クラスタのバイナリパス（new_cluster.bindir）が正しく設定されていることが前提となる。
