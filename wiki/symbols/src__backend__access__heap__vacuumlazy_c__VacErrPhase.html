VacErrPhase

定義

  • 種別: enum
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:224

目的

Lazy Vacuum（同時実行VACUUM）処理中にエラーが発生した際、どのフェーズで問題が起きたかを特定するためのコンテキスト追跡用列挙型です。

シグネチャまたは形状

typedef enum
{
	VACUUM_ERRCB_PHASE_UNKNOWN,
	VACUUM_ERRCB_PHASE_SCAN_HEAP,
	VACUUM_ERRCB_PHASE_VACUUM_INDEX,
	VACUUM_ERRCB_PHASE_VACUUM_HEAP,
	VACUUM_ERRCB_PHASE_INDEX_CLEANUP,
	VACUUM_ERRCB_PHASE_TRUNCATE,
} VacErrPhase;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • VACUUM処理の各主要ステップにおいて、現在のフェーズを追跡するために使用されます。
  • VACUUM_ERRCB_PHASE_UNKNOWN が初期状態として想定されます。

アルゴリズムまたは振る舞い

VACUUM処理の進行に合わせて、エラーコールバック関数が現在のフェーズを報告するために参照します。各列挙値は以下の処理フェーズに対応します。

  • SCAN_HEAP: ヒープスキャンおよびデッドタプルの特定。
  • VACUUM_INDEX: インデックスのクリーンアップ（TID削除）。
  • VACUUM_HEAP: ヒープ上のデッドタプル削除（Pruning）。
  • INDEX_CLEANUP: インデックスの最終クリーンアップ。
  • TRUNCATE: テーブル末尾のページ切り詰め。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/vacuumlazy.c 内の各処理関数が、エラー発生時のコンテキストとしてこの型を使用します。
  • 呼び出し先: 本シンボル自体は型定義であり、直接の呼び出し先はありません。

関連データ構造

  • LVRelState: VACUUM実行中の状態を保持する構造体であり、この中で現在のフェーズを保持するメンバとして使用される可能性があります。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 新たなVACUUMのサブフェーズが追加された場合は、この列挙型に値を追加し、エラー報告ロジックを更新してください。
