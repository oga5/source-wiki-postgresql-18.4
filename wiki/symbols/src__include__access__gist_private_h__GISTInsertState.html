GISTInsertState

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/gist_private.h:251

目的

GiSTインデックスへの挿入処理中に、対象リレーションや挿入パスの状態を保持・管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    Relation         r;          /* 挿入対象のインデックスリレーション */
    Relation         heapRel;    /* 対応するヒープリレーション */
    Size             freespace;  /* ページ内に残すべき空き容量 */
    bool             is_build;   /* インデックス構築中か否かのフラグ */
    GISTInsertStack *stack;      /* ツリー降下時のパスを記録するスタック */
} GISTInsertState;

入力と出力

  • **入力**: 挿入操作開始時に memset で初期化され、各フィールドが設定されます。
  • **出力**: 挿入処理の過程で stack フィールドが更新され、ページ分割や再試行の制御に使用されます。

前提条件と不変条件

  • r および heapRel は有効なリレーション記述子である必要があります。
  • stack は挿入処理の進行に伴い、ツリーのルートからリーフへのパスを保持します。

アルゴリズムまたは振る舞い

1. 挿入処理（gistdoinsert 等）の開始時に初期化されます。 2. ツリーを降下する際、現在のパスを stack に記録します。 3. ページ分割が必要な場合、gistfixsplit や gistfinishsplit がこの状態を参照し、分割後のツリー構造を調整します。 4. is_build フラグにより、通常の挿入とインデックス構築時の挙動を切り替えます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c 内の挿入関連関数（gistdoinsert 等）。
  • **呼び出し先**: gistfixsplit, gistinserttuple, gistinserttuples, gistfinishsplit 等の内部関数に引数として渡されます。

関連データ構造

  • GISTInsertStack: 挿入パスを保持するスタック構造。
  • Relation: PostgreSQLの標準的なリレーション構造体。
  • GISTSTATE: インデックスのサポート関数情報を保持する構造体（gistfixsplit 等で併用されます）。

失敗モードと境界条件

  • 挿入対象がクローズされている場合や、メモリ不足によるスタック割り当て失敗時に異常終了する可能性があります。
  • ページ分割が連続して発生する場合、stack の整合性が維持される必要があります。

保守担当者向けメモ

  • 本構造体は gist.c 内の静的関数間で共有されるため、フィールドの追加・変更時は gist.c 内のすべての参照箇所を確認してください。
  • is_build フラグは、インデックス構築時の最適化パスを制御するため、通常の挿入ロジックと混同しないよう注意が必要です。
