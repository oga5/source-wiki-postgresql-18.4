be_gssapi_get_delegation

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-gssapi.c:781

目的

現在の接続（Port）において、GSSAPIによる資格情報の委任（delegation）が有効かどうかを判定する。

シグネチャまたは形状

bool be_gssapi_get_delegation(Port *port);

入力と出力

  • **入力**: Port *port （接続情報を保持する構造体へのポインタ）
  • **出力**: bool （委任が有効な場合は true、それ以外は false）

前提条件と不変条件

  • port が NULL であるか、port->gss が初期化されていない場合、関数は安全に false を返す。
  • 戻り値は port->gss->delegated_creds の値に直接依存する。

アルゴリズムまたは振る舞い

1. 引数 port および port->gss が有効かを確認する。無効な場合は false を返す。 2. port->gss->delegated_creds の値を評価し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/postgres_fdw/connection.c
  • src/backend/utils/activity/backend_status.c
  • src/backend/utils/init/postinit.c
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない。

関連データ構造

  • Port: src/include/libpq/libpq-be.h で定義される接続状態を管理する構造体。
  • gss_delegated_creds: Port 構造体内のGSSAPI関連情報に含まれるフラグ。

失敗モードと境界条件

  • port が NULL の場合、セグメンテーション違反を起こさず false を返す。
  • GSSAPIコンテキストが確立されていない（port->gss が NULL）場合、false を返す。

保守担当者向けメモ

  • 本関数は、GSSAPI認証後の委任状態を外部モジュールへ公開するためのアクセサである。
  • contrib モジュールやバックエンドの初期化処理において、委任された資格情報の有無を確認するために頻繁に使用される。
