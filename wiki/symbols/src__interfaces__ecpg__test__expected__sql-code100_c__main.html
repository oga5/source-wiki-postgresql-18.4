目的

ECPG（Embedded SQL for C）において、SQL実行時に「データが見つからない」ことを示す SQLCODE 100 が正しく検知・処理されるかを検証する回帰テスト用関数。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム実行時に環境変数等を通じてデータベースへ接続）
  • 出力: 正常終了時に 0 を返却。エラー発生時には標準エラー出力（stderr）および標準出力（printf）へ診断メッセージを出力。

前提条件と不変条件

  • データベース ecpg1_regression が利用可能であること。
  • ECPGdebug(1, stderr) により、実行時の詳細なデバッグログが stderr に出力される。
  • 各SQL操作後、sqlca.sqlcode を参照して結果を判定する。

アルゴリズムまたは振る舞い

1. **接続と初期化**: ECPGconnect でデータベースに接続し、test テーブルを作成して初期データを投入する。 2. **エラー条件の誘発**: 存在しない条件（WHERE index = -1）を指定して UPDATE、DELETE、INSERT ... SELECT を実行する。 3. **検証**: 各操作後に sqlca.sqlcode が 100（No data found）であることを確認する。100 以外の場合はエラーとしてメッセージを出力する。 4. **終了処理**: test テーブルを削除し、ECPGdisconnect で接続を終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGtrans, ECPGdisconnect（いずれも ecpglib ライブラリ関数）

関連データ構造

  • sqlca: SQL通信領域（sqlca_t 構造体）。実行結果のステータスコードやエラーメッセージを保持する。

失敗モードと境界条件

  • sqlca.sqlcode が 100 以外の値である場合、予期せぬエラーとして標準出力にエラー内容が表示される。
  • データベース接続失敗やテーブル作成失敗など、各ステップでエラーが発生した場合は sqlca.sqlcode を通じて通知される。

保守担当者向けメモ

  • 本関数はプリプロセッサが生成したコードであり、直接編集しないこと。
  • sqlca.sqlcode == 100 は成功に近い警告コードとして扱われており、このテストはアプリケーションがこの状態を正しくハンドリングできるかを保証するものである。
