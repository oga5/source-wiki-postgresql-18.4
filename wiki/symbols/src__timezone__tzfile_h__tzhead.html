tzhead

定義

  • 種別: struct
  • 定義箇所: src/timezone/tzfile.h:39

目的

IANA Time Zone Database (TZif) 形式のバイナリファイルの先頭 44 バイトを定義し、後続する可変長データ（遷移時間、オフセット、略称など）のメタデータを提供します。

シグネチャまたは形状

struct tzhead
{
    char    tzh_magic[4];       /* "TZif" マジックナンバー */
    char    tzh_version[1];     /* '\0', '2', または '3' */
    char    tzh_reserved[15];   /* 予約領域（ゼロであること） */
    char    tzh_ttisutcnt[4];   /* 遷移時間フラグの数 */
    char    tzh_ttisstdcnt[4];  /* 遷移時間フラグの数 */
    char    tzh_leapcnt[4];     /* 閏秒の数 */
    char    tzh_timecnt[4];     /* 遷移時間の数 */
    char    tzh_typecnt[4];     /* ローカル時刻型の数 */
    char    tzh_charcnt[4];     /* 略称文字列のバイト数 */
};

入力と出力

  • **入力**: TZif 形式のバイナリファイル。
  • **出力**: 構造体内の各メンバは char 配列として保持され、読み込み時に detzcode 等を用いて整数値へ変換されます。

前提条件と不変条件

  • すべてのメンバは char 配列として定義されており、アーキテクチャ間のアライメント問題を回避しています。
  • tzh_reserved は必ずゼロである必要があります。
  • v2/v3 形式の場合、ファイル内に本構造体が二重に存在し、後半で 64 ビットタイムスタンプを扱う拡張データブロックが定義されます。

アルゴリズムまたは振る舞い

1. ファイル読み込み時、先頭 44 バイトを本構造体にキャストします。 2. tzh_version を確認し、バージョンに応じたデータレイアウトの解釈を行います。 3. 各カウント値（tzh_timecnt 等）を読み取り、後続する可変長データのオフセットを計算します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/timezone/localtime.c がファイル読み込みおよび解析の基盤として使用しています。
  • **呼び出し先**: 本構造体自体はデータ定義であり、関数を呼び出しません。

関連データ構造

  • struct state: localtime.c 内でタイムゾーン状態を保持するために使用されます。

失敗モードと境界条件

  • エンディアンの不一致: 読み込み時に適切なエンディアン変換を行わない場合、各カウント値が正しく解釈されません。
  • バージョン不整合: tzh_version が期待する形式と異なる場合、後続のデータオフセット計算が誤る可能性があります。

保守担当者向けメモ

  • 本構造体はバイナリのポータビリティを確保するため、メンバはすべて char 配列です。
  • システムインクルードディレクトリへのコピーは厳禁です。
  • 拡張データブロック（v2以降）の解析時には、ヘッダーが二重に存在することを考慮してください。
