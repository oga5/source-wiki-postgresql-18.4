pg_atomic_exchange_u64_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic-msvc.h:100

目的

MSVC環境において、64ビット符号なし整数に対してアトミックな値の交換（Exchange）操作を提供します。

シグネチャまたは形状

static inline uint64 pg_atomic_exchange_u64_impl(volatile pg_atomic_uint64 *ptr, uint64 newval)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint64 構造体へのポインタ。
  • newval: ptr に書き込む新しい64ビット値。
  • **出力**:
  • 交換前の ptr が保持していた元の64ビット値を返します。

前提条件と不変条件

  • 本関数は _WIN64 環境（64ビットビルド）での使用を前提としています。
  • ptr はアトミック操作のために適切にアライメントされている必要があります。

アルゴリズムまたは振る舞い

1. MSVCの組み込み関数 _InterlockedExchange64 を呼び出します。 2. ptr->value の値を newval でアトミックに置換します。 3. 置換前の値を戻り値として返します。 4. Interlocked APIの特性として、操作にはメモリバリアが含まれます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics.h:508 からの呼び出しが確認されています。
  • **呼び出し先**: MSVC組み込み関数 _InterlockedExchange64 を呼び出します。

関連データ構造

  • pg_atomic_uint64: アトミック操作対象の値を保持する構造体。

失敗モードと境界条件

  • 本関数は64ビットビルド環境専用です。32ビット環境で呼び出した場合の挙動については、本ソース excerpt 内に定義がありません。

保守担当者向けメモ

  • 本関数は generic-msvc.h 内で定義されており、atomics.h を経由して利用されることを想定しています。
  • _InterlockedExchange64 はメモリバリアを内包するため、呼び出し時のメモリ順序付けの副作用に注意してください。
