PLyCursorObject

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_cursorobject.h:11

目的

PostgreSQLのカーソルをPythonオブジェクトとしてラップし、PL/Python内でのクエリ結果の反復処理を管理する。

シグネチャまたは形状

typedef struct PLyCursorObject
{
    PyObject_HEAD
    char       *portalname;
    PLyDatumToOb result;
    bool        closed;
    MemoryContext mcxt;
} PLyCursorObject;

入力と出力

  • **入力**: Python側からの plpy.cursor() 呼び出し時にインスタンス化される。
  • **出力**: Pythonのイテレータプロトコルを通じて、PostgreSQLのポータルからフェッチされたタプルをPythonオブジェクトとして返す。

前提条件と不変条件

  • portalname は有効なPostgreSQLポータル名を参照している必要がある。
  • mcxt はカーソルがクローズされるまで維持されるメモリコンテキストを保持する。
  • PythonのガベージコレクションとPostgreSQLのトランザクション状態の整合性が保たれている必要がある。

アルゴリズムまたは振る舞い

1. PyObject_New によりメモリが割り当てられ、Pythonオブジェクトとして初期化される。 2. SPI（Server Programming Interface）を介してポータルがオープンされ、その名前が portalname に格納される。 3. Pythonの tp_iternext プロトコルを通じて、ポータルから順次タプルをフェッチし、result を用いてPython型へ変換する。 4. カーソルがクローズされると closed フラグが更新される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_cursorobject.c 内の関数（PLy_cursor 等）によりインスタンス化される。
  • **呼び出し先**: src/pl/plpython/plpy_spi.c が提供するSPI関数群を介してポータル操作を行う。

関連データ構造

  • PLyDatumToOb: PostgreSQLのデータ型をPythonオブジェクトへ変換するための型変換定義。
  • MemoryContext: カーソルに関連付けられたメモリ管理領域。

失敗モードと境界条件

  • PyObject_New が失敗した場合、メモリ不足等の理由で NULL が返される。
  • ポータルのオープン失敗やフェッチ時のエラーは、SPI層の例外として伝播する。

保守担当者向けメモ

  • 本構造体は MemoryContext を保持しているため、メモリリークを防ぐために適切なタイミングでのクローズ処理が必須である。
  • portalname はSPIのライフサイクルと密接に関連しているため、トランザクション終了後のポータル無効化に注意すること。
