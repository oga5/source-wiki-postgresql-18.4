check_agglevels_and_constraints

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_agg.c:306

目的

集約関数（Aggref）またはグループ化操作（GroupingFunc）がクエリ内の妥当な位置に配置されているかを検証し、その集約レベルを決定する。

シグネチャまたは形状

static void check_agglevels_and_constraints(ParseState *pstate, Node *expr)

入力と出力

  • pstate: 現在の解析状態（ParseState）。
  • expr: 検証対象のノード（Aggref または GroupingFunc）。
  • 出力: なし（副作用として pstate のフラグ更新やエラー発生を行う）。

前提条件と不変条件

  • expr は Aggref または GroupingFunc のいずれかである必要がある。
  • pstate->p_expr_kind を通じて、現在の式が配置されている構文上のコンテキストが特定可能であること。

アルゴリズムまたは振る舞い

1. **レベル決定**: check_agg_arguments を呼び出し、引数から集約のネストレベル（min_varlevel）を計算し、対象ノードの agglevelsup フィールドに格納する。 2. **フラグ更新**: 計算されたレベルに基づき、該当するクエリレベルの ParseState を遡り、p_hasAggs を true に設定する。 3. **配置検証**: pstate->p_expr_kind を参照し、現在のコンテキストが集約関数やグループ化操作を許可しているか判定する。

  • JOIN 条件、FROM 句、ウィンドウフレーム定義など、許可されていない場所で使用されている場合はエラーを報告する。
  • WHERE 句や HAVING 句など、コンテキストに応じて適切なエラーメッセージまたはエラー種別（errkind）を決定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_agg.c 内の集約変換処理（transformAggregateCall 等のフロー内）。
  • **呼び出し先**:
  • check_agg_arguments（集約レベルの計算）。
  • pstate->parentParseState（クエリレベルの遡り）。

関連データ構造

  • ParseState: クエリ解析の状態を保持。
  • Aggref: 集約関数ノード。
  • GroupingFunc: グループ化操作ノード。

失敗モードと境界条件

  • **不正な配置**: 集約関数が許可されていない場所（例: JOIN 条件）で使用された場合、エラーが発生する。
  • **ネストの不整合**: check_agg_arguments が検出する集約のネストレベルが不正な場合、解析エラーとなる。
  • **EXPR_KIND_NONE**: p_expr_kind が EXPR_KIND_NONE の場合はアサーション失敗となる。

保守担当者向けメモ

  • 本関数は sublevels_up の管理に依存している。pstate を遡るロジックは、サブクエリ内の集約を正しく処理するために不可欠である。
  • 新しい構文コンテキスト（EXPR_KIND_*）を追加した際は、本関数の switch 文に適切な許可/禁止ルールを追加する必要がある。
