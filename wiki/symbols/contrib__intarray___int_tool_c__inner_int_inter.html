inner_int_inter

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:135

目的

2つのソート済み整数配列（ArrayType）の積集合を計算し、新しい ArrayType として返します。

シグネチャまたは形状

ArrayType *inner_int_inter(ArrayType *a, ArrayType *b)

入力と出力

  • **入力**:
  • a, b: ソート済みの ArrayType ポインタ。
  • **出力**:
  • 積集合を含む新しい ArrayType ポインタ。積集合が空の場合は要素数0の配列を返します。

前提条件と不変条件

  • 入力配列 a および b はソート済みである必要があります。
  • 戻り値の ArrayType は palloc されているため、呼び出し側で適切にメモリ管理（pfree 等）を行う必要があります。

アルゴリズムまたは振る舞い

1. 入力配列のいずれかが空の場合、要素数0の配列を返します。 2. 両方の配列を先頭から走査し、ポインタ演算を用いて共通する要素を抽出します。 3. 比較ロジック:

  • da[i] < db[j] の場合: i を進める。
  • da[i] == db[j] の場合: 重複を避けて結果配列 dr に格納し、i と j を進める。
  • da[i] > db[j] の場合: j を進める。

    • 4. 抽出された要素数に応じて、結果配列を resize_intArrayType で調整して返します。共通要素がない場合はメモリを解放し、空の配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c:495
  • contrib/intarray/_int_op.c:158
  • **呼び出し先**:
  • new_intArrayType
  • resize_intArrayType
  • pfree

関連データ構造

  • ArrayType: PostgreSQL の標準的な配列構造体。

失敗モードと境界条件

  • 入力配列が空の場合、要素数0の配列を正常に返します。
  • 共通要素が存在しない場合、メモリを解放した上で要素数0の配列を返します。

保守担当者向けメモ

  • 本関数は線形時間 $O(N+M)$ で動作します。
  • 入力配列がソートされていない場合、結果は保証されません。呼び出し側でソート状態を担保してください。
