infer_collation_opclass_match

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/plancat.c:992

目的

ON CONFLICT 句などで指定された推論要素（InferenceElem）が、対象インデックスの演算子クラス（opclass）および照合順序（collation）と一致するかを検証する。

シグネチャまたは形状

static bool infer_collation_opclass_match(InferenceElem *elem, Relation idxRel, List *idxExprs)

入力と出力

  • **入力**:
  • elem: 検証対象の推論要素。
  • idxRel: 比較対象のインデックスリレーション。
  • idxExprs: インデックス式（idxRel が式インデックスの場合）。
  • **出力**:
  • bool: 指定された条件に合致するインデックス属性が一つでも見つかれば true、それ以外は false。

前提条件と不変条件

  • elem の infercollid および inferopclass が共に InvalidOid の場合、無条件で true を返す。
  • インデックス定義および推論句には RelabelType は含まれないものとして処理する。

アルゴリズムまたは振る舞い

1. elem に inferopclass が指定されている場合、対応する演算子族（opfamily）と入力型を get_opclass_family および get_opclass_input_type で取得する。 2. インデックスの各属性を順次走査する。 3. inferopclass が指定されている場合、インデックス属性の演算子族および入力型と一致するかを確認し、不一致ならスキップする。 4. infercollid が指定されている場合、インデックス属性の照合順序と一致するかを確認し、不一致ならスキップする。 5. 条件に合致した場合：

  • elem->expr が Var 型であれば、その属性番号がインデックスの属性番号と一致するかを確認する。
  • それ以外（式インデックス）であれば、idxExprs から対応する式を取得し、equal() 関数で一致を判定する。

    • 6. いずれかの属性で一致が確認できれば true を返し、全属性を走査しても見つからなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/plancat.c 内の infer_arbiter_indexes から呼び出される。
  • **呼び出し先**: get_opclass_family, get_opclass_input_type, IsA, list_nth, equal。

関連データ構造

  • InferenceElem: 推論対象の式、照合順序、演算子クラスを保持する構造体。
  • Relation: インデックスのメタデータ（rd_att, rd_opfamily, rd_indcollation 等）を保持。

失敗モードと境界条件

  • elem に制約がない場合は常に成功（true）する。
  • インデックスが式インデックスである場合、idxExprs リストのインデックスと属性の対応関係を nplain（単純属性数）を用いて計算する。

保守担当者向けメモ

  • 本関数は match_index_to_operand() と異なり、RelabelType の考慮を意図的に省略している。
  • インデックスの属性番号 attno が 0 の場合は式インデックスを指す。
