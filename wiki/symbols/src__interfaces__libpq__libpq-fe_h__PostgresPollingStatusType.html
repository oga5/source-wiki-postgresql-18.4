PostgresPollingStatusType

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-fe.h:113

目的

非同期接続や認証処理におけるポーリング状態を表現し、クライアントが次に取るべきアクション（読み込み待機、書き込み待機、完了など）を指示するために使用されます。

シグネチャまたは形状

typedef enum {
    PGRES_POLLING_FAILED = 0,
    PGRES_POLLING_READING,
    PGRES_POLLING_WRITING,
    PGRES_POLLING_OK,
    PGRES_POLLING_ACTIVE
} PostgresPollingStatusType;

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: 非同期処理の現在のステータスを示す定数

前提条件と不変条件

  • PGRES_POLLING_READING および PGRES_POLLING_WRITING は、ポーリングを再試行する前に select() 等のシステムコールを使用してソケットの状態を確認できることを示します。
  • PGRES_POLLING_ACTIVE は未使用であり、後方互換性のために保持されています。

アルゴリズムまたは振る舞い

  • PGRES_POLLING_FAILED: 処理が失敗したことを示します。
  • PGRES_POLLING_READING: ソケットからの読み込みが可能になるまで待機する必要があります。
  • PGRES_POLLING_WRITING: ソケットへの書き込みが可能になるまで待機する必要があります。
  • PGRES_POLLING_OK: 処理が正常に完了したことを示します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/bin/pg_upgrade/task.c
  • src/include/libpq/libpq-be-fe-helpers.h
  • src/interfaces/libpq-oauth/oauth-curl.c
  • src/interfaces/libpq-oauth/oauth-curl.h
  • src/interfaces/libpq/fe-auth-oauth.c
  • 呼び出し先: 情報なし。

関連データ構造

  • PGconn: データベース接続を管理する不透明構造体。本列挙型は主に PGconn を用いた非同期処理のステータスとして利用されます。

失敗モードと境界条件

  • PGRES_POLLING_FAILED が返された場合、接続または認証プロセスは失敗しており、それ以上のポーリングは継続できません。

保守担当者向けメモ

  • PGRES_POLLING_ACTIVE は現在使用されていませんが、既存コードとの互換性維持のため削除しないでください。
  • 新しいステータスを追加する場合は、既存の列挙値の順序や数値（0 から開始）に注意してください。
