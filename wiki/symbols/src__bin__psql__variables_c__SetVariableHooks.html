SetVariableHooks

定義

  • 種別: function
  • 定義箇所: src/bin/psql/variables.c:383

目的

変数に対して代入フック（VariableAssignHook）および置換フック（VariableSubstituteHook）を登録し、即座に現在の値に対してこれらのフックを適用して内部状態を同期させる。

シグネチャまたは形状

void SetVariableHooks(VariableSpace space, const char *name,
                      VariableSubstituteHook shook,
                      VariableAssignHook ahook)

入力と出力

  • space: 変数を格納する VariableSpace 構造体。
  • name: 登録対象の変数名。
  • shook: 値の正規化を行う置換フック関数。
  • ahook: 値の妥当性を検証する代入フック関数。
  • 出力: なし（void）。

前提条件と不変条件

  • space および name が NULL の場合、何も行わずに終了する。
  • name は valid_variable_name の検証を通過する必要がある。
  • 変数リストは名前順（strcmp 順）にソートされている必要がある。

アルゴリズムまたは振る舞い

1. 指定された name を変数リストから検索する。 2. 発見した場合、その変数のフックを更新し、置換フックがあれば値を正規化し、代入フックがあれば値を検証する。 3. 発見できなかった場合、新しいエントリを作成してリストに挿入し、同様にフックを適用する。 4. フックの適用順序は、まず shook で値を変換し、その結果に対して ahook を実行する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/startup.c（AUTOCOMMIT, ON_ERROR_STOP, QUIET, SINGLELINE, SINGLESTEP, FETCH_COUNT, HISTFILE, HISTSIZE, IGNOREEOF, ECHO, ECHO_HIDDEN, ON_ERROR_ROLLBACK の初期化時）。
  • 呼び出し先: valid_variable_name, pg_malloc, pg_strdup, 登録された shook および ahook 関数。

関連データ構造

  • VariableSpace: 変数リストのヘッダー。
  • struct _variable: 変数名、値、およびフック関数を保持する内部構造体。

失敗モードと境界条件

  • space や name が無効な場合、処理はスキップされる。
  • valid_variable_name が偽を返す名前の場合、処理はスキップされる。
  • メモリ不足時は pg_malloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は、変数が存在しない場合に新規作成する機能も兼ねている。
  • フック関数が NULL の場合は適用されない。
  • 代入フックが値を拒否した場合の挙動は、フック関数側の実装に依存する。
