set_ps_display_with_len

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/ps_status.c:452

目的

バックエンドプロセスの現在の活動状態を示す文字列を更新し、OSのプロセス表示（psコマンド等）に反映させる。

シグネチャまたは形状

void set_ps_display_with_len(const char *activity, size_t len)

入力と出力

  • activity: プロセス名に付与する活動状態を表す文字列。
  • len: activity の長さ（バイト数）。
  • 戻り値: なし。

前提条件と不変条件

  • strlen(activity) == len であること（Assertによる検証）。
  • PS_USE_NONE が定義されていない環境でのみ動作する。
  • update_ps_display_precheck() が真を返す場合にのみ更新処理が実行される。

アルゴリズムまたは振る舞い

1. update_ps_display_precheck() を呼び出し、更新の可否を確認する。 2. 既存のサフィックス情報をクリアする（ps_buffer_nosuffix_len = 0）。 3. ps_buffer の固定部分（ps_buffer_fixed_size）の直後に activity をコピーする。

  • バッファ容量が不足する場合は、末尾を切り詰めて終端文字 \0 を付与する。
  • 十分な容量がある場合は、len 分をコピーし、ps_buffer_cur_len を更新する。

    • 4. flush_ps_display() を呼び出し、OS固有のメカニズムを通じてカーネルへ新しいプロセス名を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/postgres.c（クエリ実行の各段階で呼び出される）。
  • **呼び出し先**:
  • update_ps_display_precheck()
  • flush_ps_display()

関連データ構造

  • ps_buffer: プロセス名全体を保持するグローバルバッファ。
  • ps_buffer_fixed_size: プロセス名の固定部分の長さ。
  • ps_buffer_cur_len: 現在のバッファ内の有効な文字列長。
  • ps_buffer_nosuffix_len: サフィックスを除いたバッファ長。

失敗モードと境界条件

  • activity の長さがバッファ容量を超える場合、バッファサイズに合わせて切り詰めが行われる。
  • update_ps_display_precheck() が偽を返す場合、関数は何もせず即座に復帰する。

保守担当者向けメモ

  • 本関数は PS_USE_CLOBBER_ARGV 等のOS固有の実装に依存する flush_ps_display() を呼び出すため、プラットフォームごとの挙動に注意が必要。
  • Assert による検証を行っているため、呼び出し側は必ず正しい len を渡す必要がある。
