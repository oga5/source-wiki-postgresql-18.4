pg_partition_root

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/partitionfuncs.c:163

目的

指定されたリレーション（パーティションまたはテーブル）が属するパーティションツリーのルート（最上位の親）リレーションの OID を取得します。

シグネチャまたは形状

Datum pg_partition_root(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してリレーションの OID を受け取ります。
  • **出力**: 成功時はルートリレーションの OID を Datum 型で返します。入力リレーションがパーティションツリーの一部として無効な場合は NULL を返します。

前提条件と不変条件

  • 入力された OID は check_rel_can_be_partition によって検証され、パーティションツリーのメンバーとして有効である必要があります。
  • 戻り値の rootrelid は、有効なパーティションツリーメンバーである限り、常に有効な OID であることが保証されます（Assert(OidIsValid(rootrelid))）。

アルゴリズムまたは振る舞い

1. check_rel_can_be_partition(relid) を呼び出し、対象がパーティションツリーに含まれるか確認します。無効な場合は NULL を返します。 2. get_partition_ancestors(relid) を呼び出し、祖先リレーションのリストを取得します。 3. 祖先リストが空（NIL）の場合、入力リレーション自身がルートであると判断し、その OID を返します。 4. 祖先リストが存在する場合、リストの末尾（llast_oid）がルートであるため、その値を取得します。 5. 取得したリストを list_free で解放し、ルート OID を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出されることを想定しています（具体的な SQL クエリ等の呼び出し元情報はソースコード内に記載なし）。
  • **呼び出し先**:
  • check_rel_can_be_partition (内部ヘルパー)
  • get_partition_ancestors (カタログ操作)
  • llast_oid (リスト操作)
  • list_free (メモリ管理)

関連データ構造

  • Oid: リレーション識別子。
  • List: 祖先リレーションを保持するためのリスト構造。

失敗モードと境界条件

  • 入力リレーションがパーティションツリーの一部ではない場合、check_rel_can_be_partition が false を返し、本関数は NULL を返します。
  • 入力リレーション自身がルートである場合、祖先リストが NIL となり、即座に自身の OID を返します。

保守担当者向けメモ

  • 本関数は pg_partition_ancestors と同様に get_partition_ancestors に依存しています。カタログ定義の変更時には、祖先取得ロジックの整合性に注意してください。
  • list_free によるメモリ解放は必須です。祖先リストの取得後に早期リターンを行う場合は、必ずリストの解放漏れがないことを確認してください。
