pa_get_xact_state

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/applyparallelworker.c:1334

目的

並列適用ワーカーの共有メモリ状態から、現在のトランザクション状態（ParallelTransState）をスレッドセーフに取得します。

シグネチャまたは形状

static ParallelTransState pa_get_xact_state(ParallelApplyWorkerShared *wshared);

入力と出力

  • **入力**: ParallelApplyWorkerShared *wshared（ワーカーの共有メモリ状態構造体へのポインタ）
  • **出力**: ParallelTransState（現在のトランザクション状態を示す列挙値）

前提条件と不変条件

  • 引数 wshared は有効なメモリ領域を指している必要があります。
  • 内部で wshared->mutex を使用するため、呼び出し元は当該ミューテックスの保持状態を考慮する必要はありません（関数内で取得・解放を行うため）。

アルゴリズムまたは振る舞い

1. wshared->mutex を用いてスピンロックを取得します。 2. 共有メモリ上の wshared->xact_state をローカル変数にコピーします。 3. スピンロックを解放します。 4. コピーしたトランザクション状態を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/applyparallelworker.c:560 (アサーションチェック)
  • src/backend/replication/logical/applyparallelworker.c:1269 (状態比較)
  • src/backend/replication/logical/applyparallelworker.c:1313 (状態確認)
  • **呼び出し先**:
  • SpinLockAcquire
  • SpinLockRelease

関連データ構造

  • ParallelApplyWorkerShared: ワーカー間で共有される状態を保持する構造体。
  • ParallelTransState: トランザクションの進行状況を示す列挙型。

失敗モードと境界条件

  • wshared が NULL の場合、セグメンテーション違反が発生します。
  • スピンロックの取得に失敗する状況（システムリソース枯渇等）では、呼び出し元で待機またはエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は共有メモリへのアクセスを同期化するラッパーです。状態の読み取り頻度が高いため、スピンロックの競合がパフォーマンスに影響を与える可能性があります。
  • 戻り値の ParallelTransState は、呼び出し直後に別のプロセスによって変更される可能性があるため、値の保持期間には注意してください。
