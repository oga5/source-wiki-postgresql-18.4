PostmasterIsAlive

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/pmsignal.h:99

目的

子プロセスがポストマスター（親プロセス）の生存状態を効率的に確認するために使用されます。

シグネチャまたは形状

static inline bool PostmasterIsAlive(void)

入力と出力

  • **入力**: なし
  • **出力**: bool 型。ポストマスターが生存している場合は true、終了している（またはその可能性がある）場合は false を返します。

前提条件と不変条件

  • 本関数は、ポストマスターの子プロセスから呼び出されることを前提としています。
  • postmaster_possibly_dead 変数の状態に依存します。

アルゴリズムまたは振る舞い

1. postmaster_possibly_dead フラグをチェックします。このフラグが false（生存中）であると推測される場合、即座に true を返します。 2. フラグが true（死亡の可能性がある）場合、PostmasterIsAliveInternal() を呼び出し、より詳細な生存確認（システムコール等を含む可能性のある処理）を実行してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuum.c
  • src/backend/postmaster/pgarch.c
  • src/backend/postmaster/startup.c
  • src/backend/storage/ipc/waiteventset.c
  • **呼び出し先**:
  • PostmasterIsAliveInternal()

関連データ構造

  • postmaster_possibly_dead: 生存状態を保持するフラグ（sig_atomic_t）。

失敗モードと境界条件

  • OSが親プロセス終了通知（PDEATHSIG等）をサポートしている場合、シグナルハンドラによって postmaster_possibly_dead が更新されます。
  • PostmasterIsAliveInternal() は、OS固有の機能を利用する「低速パス」として機能します。

保守担当者向けメモ

  • 本関数はインライン化されており、頻繁な呼び出しによるオーバーヘッドを最小限に抑える設計です。
  • src/include/storage/pmsignal.h:107 にて、PostmasterIsAlive() を PostmasterIsAliveInternal() に置換するマクロ定義が存在します。
