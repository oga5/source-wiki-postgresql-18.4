IncrementVarSublevelsUp_context

定義

  • **種別**: struct
  • **定義箇所**: src/backend/rewrite/rewriteManip.c:805

目的

クエリツリー内の Var ノードのネストレベル（varlevelsup）を調整する際、走査中の状態を保持するために使用されるコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    int delta_sublevels_up;
    int min_sublevels_up;
} IncrementVarSublevelsUp_context;

入力と出力

  • delta_sublevels_up: Var ノードの varlevelsup に加算する値。
  • min_sublevels_up: この値以上の varlevelsup を持つ Var ノードのみを対象とするための閾値。

前提条件と不変条件

  • 走査対象の Var ノードの varlevelsup が min_sublevels_up 以上である場合のみ、その値が delta_sublevels_up 分だけ増加されます。

アルゴリズムまたは振る舞い

  • expression_tree_walker 等の走査関数と組み合わせて使用されます。
  • ツリーを走査し、条件に合致する Var ノードを見つけると、その varlevelsup を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteManip.c 内の関数（895行目および918行目付近でインスタンス化を確認）。
  • **呼び出し先**: 構造体定義のため、直接的な呼び出し先はありません。

関連データ構造

  • Var ノード（nodes/parsetree.h）: 本構造体によって varlevelsup が操作される対象。

失敗モードと境界条件

  • 供給されたソース情報には、特定の失敗モードや境界条件に関する記述はありません。

保守担当者向けメモ

  • 本構造体は rewriteManip.c 内のツリー操作ロジックにおいて、ネストされたクエリレベルをまたぐ変数参照を正しく調整するために不可欠です。
  • 走査ロジックを修正する際は、min_sublevels_up の判定条件が意図したスコープを正しくカバーしているか確認してください。
