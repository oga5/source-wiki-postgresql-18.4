ExecReScanSetOp

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSetOp.c:692

目的

SetOp 実行ノードの状態をリセットし、再スキャン（再実行）を可能にする。

シグネチャまたは形状

void ExecReScanSetOp(SetOpState *node)

入力と出力

  • **入力**: SetOpState *node（再スキャン対象の実行ノード状態）
  • **出力**: なし（副作用としてノード内部状態を初期化）

前提条件と不変条件

  • node は SetOpState 構造体として有効であること。
  • node->ps.plan から取得される戦略（SETOP_HASHED または SETOP_SORTED）に基づき動作する。

アルゴリズムまたは振る舞い

1. **共通リセット**: ps_ResultTupleSlot をクリアし、setop_done を false、numOutput を 0 にリセットする。 2. **戦略別処理**:

  • **SETOP_HASHED の場合**:
  • ハッシュテーブルが未構築（table_filled が false）であれば即座に終了する。
  • 子プランのパラメータ変更（chgParam）がなく、かつハッシュテーブルが構築済みであれば、イテレータをリセットして終了する。
  • それ以外の場合、tableContext をリセットし、ハッシュテーブルを空の状態へ初期化して table_filled を false にする。
  • **SETOP_SORTED の場合**:
  • need_init を true に設定し、次回の実行時に各入力の再読み込みを強制する。

    • 3. **子ノードの再スキャン**: 子プラン（outerPlan および innerPlan）の chgParam が NULL の場合、ExecReScan を呼び出して子ノードを再スキャンする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:290
  • **呼び出し先**:
  • ExecClearTuple
  • ResetTupleHashIterator
  • MemoryContextReset
  • ResetTupleHashTable
  • ExecReScan

関連データ構造

  • SetOpState: 実行ノードの状態を保持する構造体。
  • SetOp: プランノードの定義。
  • PlanState: 子プランの実行状態。

失敗モードと境界条件

  • 子プランの chgParam が設定されている場合、ExecReScan は明示的に呼び出されず、後続の ExecProcNode 呼び出し時に自動的に再スキャンされる。
  • ハッシュテーブルが未構築の状態で再スキャンが要求された場合、メモリ解放や再構築は行われない。

保守担当者向けメモ

  • UNION 演算には関与しないため、本関数は INTERSECT または EXCEPT の実行ノードに対してのみ適用される。
  • ハッシュテーブルのメモリ管理は tableContext に依存しており、再スキャン時の効率化のために MemoryContextReset を利用している点に注意すること。
