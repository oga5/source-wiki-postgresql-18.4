array_unnest_fctx

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:6269

目的

SQL関数 unnest を実行する際、配列の要素を走査するための状態を保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    array_iter iter;      /* 配列走査用のイテレータ */
    int        nextelem;  /* 次に取得する要素のインデックス */
    int        numelems;  /* 配列内の総要素数 */
    int16      elmlen;    /* 要素の長さ（バイト数） */
    bool       elmbyval;  /* 要素が値渡し（by-value）かどうか */
    char       elmalign;  /* 要素のアライメント要件 */
} array_unnest_fctx;

入力と出力

  • **入力**: palloc を介してメモリが割り当てられ、配列のメタデータおよびイテレータ状態が格納されます。
  • **出力**: 構造体内の各フィールドは、後続の unnest 処理において各要素の抽出および型情報の特定に使用されます。

前提条件と不変条件

  • palloc によってメモリが割り当てられている必要があります。
  • iter フィールドは、有効な配列データに対して初期化されている必要があります。

アルゴリズムまたは振る舞い

1. unnest 実行時に本構造体が palloc で確保されます。 2. 配列のメタデータ（要素長、値渡しの可否、アライメント）が構造体にキャッシュされます。 3. iter を使用して配列内を順次走査し、nextelem を更新しながら各要素を抽出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arrayfuncs.c 内の unnest 実装関数（該当箇所: 6306行目付近）。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • array_iter: 配列要素を走査するためのイテレータ型。
  • ArrayType: 配列の内部表現を定義する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 配列が空の場合、numelems が 0 となり、走査は即座に終了します。

保守担当者向けメモ

  • 本構造体は unnest の実行状態を保持するため、関数呼び出し間で状態を維持する必要がある場合に利用されます。
  • elmlen や elmbyval などのフィールドは、配列要素の型情報をキャッシュすることで、走査ごとの型カタログ検索コストを削減しています。
