AtEOXact_SMgr

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/smgr.c:1016

目的

トランザクション終了時に、ストレージマネージャが管理する不要な SMgrRelation オブジェクトをクリーンアップする。

シグネチャまたは形状

void AtEOXact_SMgr(void)

入力と出力

  • 入力: なし
  • 出力: なし

前提条件と不変条件

  • トランザクションの終了処理の一環として呼び出される。
  • ピン留め（pincount > 0）されていない SMgrRelation オブジェクトのみが破棄の対象となる。

アルゴリズムまたは振る舞い

内部で smgrdestroyall() を呼び出し、現在キャッシュされているすべての SMgrRelation オブジェクトのうち、ピン留めされていないものを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xact.c (トランザクション終了処理)
  • src/backend/postmaster/ 配下の各プロセス（autovacuum, bgwriter, checkpointer, walwriter 等のメインループ内）
  • 呼び出し先:
  • smgrdestroyall()

関連データ構造

  • SMgrRelation: ストレージマネージャが管理するリレーションハンドル。
  • SMgrRelationHash: キャッシュされている SMgrRelation を保持するハッシュテーブル。

失敗モードと境界条件

  • 本関数自体はエラーを返さないが、内部の smgrdestroyall() がファイル記述子のクローズ処理を伴うため、システムリソースの枯渇や I/O エラーの影響を受ける可能性がある。

保守担当者向けメモ

  • 本関数はトランザクション境界でのリソース管理を担うため、メモリリークやファイル記述子のリークを防ぐ重要な役割を持つ。
  • 割り込み処理との競合を避けるため、呼び出し元で適切な割り込み制御が行われていることを確認すること。
