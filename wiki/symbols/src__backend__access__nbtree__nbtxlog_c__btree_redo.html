btree_redo

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtxlog.c:1017

目的

B-tree インデックスの WAL（Write-Ahead Log）リプレイにおけるメインエントリポイントとして、WAL レコードの種類を解析し、適切な物理的再構築処理へディスパッチします。

シグネチャまたは形状

void btree_redo(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象の WAL レコード）
  • **出力**: なし（副作用としてインデックスページの状態を更新）

前提条件と不変条件

  • opCtx（メモリコンテキスト）が有効であること。
  • リプレイ処理は単一のインデックスページに対する操作として完結し、並行更新が発生しない環境（リカバリ時等）で実行されること。

アルゴリズムまたは振る舞い

1. XLogRecGetInfo を用いてレコードタイプを抽出します。 2. opCtx にメモリコンテキストを切り替え、リプレイに必要な一時領域を確保します。 3. switch 文により、挿入（INSERT）、分割（SPLIT）、削除（DELETE）、メタページ更新（NEWROOT、META_CLEANUP）などの各 WAL レコードタイプに応じた専用関数を呼び出します。 4. 未知のレコードタイプが渡された場合は PANIC レベルでエラーを発生させます。 5. 処理終了後、メモリコンテキストを元に戻し、opCtx をリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されたリソースマネージャ（RM_BTREE_ID）経由で、WAL リプレイエンジンから呼び出されます。
  • **呼び出し先**: btree_xlog_insert, btree_xlog_split, btree_xlog_dedup, btree_xlog_vacuum, btree_xlog_delete, btree_xlog_mark_page_halfdead, btree_xlog_unlink_page, btree_xlog_newroot, btree_xlog_reuse_page, _bt_restore_meta。

関連データ構造

  • XLogReaderState: WAL レコードを保持する構造体。
  • opCtx: リプレイ処理中に使用されるメモリコンテキスト。

失敗モードと境界条件

  • 未知の WAL レコードタイプが渡された場合、elog(PANIC, ...) によりリカバリが停止します。
  • 各サブ関数（btree_xlog_* 等）内でバッファ読み込みやページ操作に失敗した場合、それらの関数がエラーを発生させます。

保守担当者向けメモ

  • 本関数はリプレイ処理のディスパッチャであるため、新しい WAL レコードタイプを追加する際は、必ず switch 文にケースを追加してください。
  • リカバリ時はロックが簡略化されているため、ページ単位の整合性維持に注意を払う必要があります。
