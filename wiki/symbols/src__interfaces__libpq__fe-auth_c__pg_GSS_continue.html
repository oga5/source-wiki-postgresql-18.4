pg_GSS_continue

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth.c:60

目的

GSSAPI 認証プロセスの継続処理を実行します。サーバーから送られてくる認証トークンを処理し、セキュリティコンテキストの初期化または更新を行います。

シグネチャまたは形状

static int pg_GSS_continue(PGconn *conn, int payloadlen)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体へのポインタ。
  • payloadlen: サーバーから受信した認証トークンのバイト長。
  • **出力**:
  • STATUS_ERROR (-1): メモリ確保失敗、通信エラー、または GSSAPI 認証エラーが発生した場合。

前提条件と不変条件

  • conn->gctx が GSS_C_NO_CONTEXT でない場合、payloadlen 分のデータが入力バッファに存在している必要があります。
  • 認証完了時には conn->client_finished_auth が true に設定され、conn->gssapi_used が true になります。

アルゴリズムまたは振る舞い

1. **トークン読み込み**: 既存のコンテキストがある場合、pqGetnchar を使用して入力バッファからトークンを読み込みます。 2. **認証コンテキスト更新**: gss_init_sec_context を呼び出し、GSSAPI セキュリティコンテキストを更新します。この際、conn->gssdelegation 設定に応じて委任フラグを制御します。 3. **応答送信**: GSSAPI が生成した出力トークンがある場合、pqPacketSend を介して PqMsg_GSSResponse メッセージをサーバーへ送信します。 4. **状態判定**:

  • GSS_S_COMPLETE: 認証成功。コンテキストを解放し、完了フラグを立てます。
  • GSS_S_CONTINUE_NEEDED: 認証継続。
  • その他: pg_GSS_error を呼び出し、エラーとして終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth.c 内の認証フロー制御ロジックから呼び出されます。
  • **呼び出し先**:
  • pqGetnchar, pqPacketSend, pqTraceOutputMessage (libpq 内部関数)
  • gss_init_sec_context, gss_release_buffer, gss_release_name, gss_delete_sec_context (GSSAPI ライブラリ関数)
  • pg_GSS_have_cred_cache, pg_GSS_error (同ファイル内のヘルパー)

関連データ構造

  • PGconn: 接続情報および認証状態（gctx, gcred, gtarg_nam 等）を保持。
  • gss_buffer_desc: GSSAPI の入出力トークンを保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: malloc 失敗時に libpq_append_conn_error を経由して STATUS_ERROR を返します。
  • **通信エラー**: pqGetnchar または pqPacketSend が失敗した場合に STATUS_ERROR を返します。
  • **GSSAPI エラー**: maj_stat が完了または継続以外の場合、コンテキストを破棄しエラーを報告します。

保守担当者向けメモ

  • 本関数は非同期認証フローの一部として機能します。
  • gss_init_sec_context の呼び出し前後で ginbuf および goutbuf のメモリ管理（free および gss_release_buffer）が確実に行われるよう注意してください。
