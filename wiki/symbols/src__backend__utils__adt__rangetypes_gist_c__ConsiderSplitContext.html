ConsiderSplitContext

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/rangetypes_gist.c:71

目的

GiSTインデックスの picksplit メソッドにおいて、範囲型データの分割候補を評価および保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    TypeCacheEntry *typcache;
    bool        has_subtype_diff;
    int         entries_count;
    bool        first;
    RangeBound *left_upper;
    RangeBound *right_lower;
    float4      ratio;
    float4      overlap;
    int         common_left;
    int         common_right;
} ConsiderSplitContext;

入力と出力

  • 本構造体自体はデータ保持用であり、関数呼び出しの引数として渡されることで、分割アルゴリズムの状態を共有します。
  • first フラグにより、現在までに有効な分割候補が選択されたかどうかを追跡します。

前提条件と不変条件

  • typcache は対象となる範囲型の型キャッシュエントリを指している必要があります。
  • entries_count は分割対象となるインデックスエントリの総数を保持します。

アルゴリズムまたは振る舞い

  • range_gist_picksplit 内で初期化され、range_gist_consider_split 関数によって更新されます。
  • 分割の評価指標として、ratio（分割比率）、overlap（左右の述語間の重なり）、および common_left/common_right（各側に割り当てられる共通エントリ数）を管理し、最適な分割を決定するためのヒューリスティックに使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rangetypes_gist.c 内の range_gist_picksplit および range_gist_consider_split。
  • src/backend/access/gist/gistproc.c において、g_box_consider_split 等のコンテキストとして利用される可能性があります（直接的な呼び出し関係の詳細はソースコードの文脈に依存します）。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • RangeBound: 範囲の境界情報を保持するために使用されます。
  • TypeCacheEntry: 範囲型の型情報を保持します。

失敗モードと境界条件

  • first が true の状態では、まだ有効な分割候補が計算されていないことを示します。
  • entries_count が 0 または不適切な値の場合、分割ロジックが正しく動作しない可能性があります。

保守担当者向けメモ

  • 本構造体は range_gist_picksplit の分割ロジックと密結合しています。分割アルゴリズム（特に double_sorting_split）を変更する際は、本構造体のフィールドが適切に更新されているか確認してください。
  • gistproc.c からの参照については、あくまでヒューリスティックな情報であり、実際の呼び出しフローについては該当ファイルの静的解析結果を併せて参照してください。
