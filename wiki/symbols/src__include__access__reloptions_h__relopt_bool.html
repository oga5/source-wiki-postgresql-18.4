relopt_bool

定義

  • 種別: struct
  • 定義箇所: src/include/access/reloptions.h:91

目的

リレーションやテーブルスペースのオプション設定において、真偽値（bool）をとるパラメータを定義・保持するための構造体です。

シグネチャまたは形状

typedef struct relopt_bool
{
    relopt_gen    gen;
    bool          default_val;
} relopt_bool;

入力と出力

  • **入力**: オプション定義時に default_val として真偽値を指定します。
  • **出力**: 解析済みオプションの値を参照する際、default_val フィールドからデフォルト値を取得可能です。

前提条件と不変条件

  • gen フィールド（relopt_gen 型）が構造体の先頭に配置されており、汎用的なオプション管理関数との互換性が保たれています。
  • reloptions.c 内の関数によって、RELOPT_TYPE_BOOL 型として識別される必要があります。

アルゴリズムまたは振る舞い

  • reloptions.c において、init_bool_reloption 等の関数を通じて初期化されます。
  • ユーザーが指定したオプション値がない場合、この構造体に格納された default_val がデフォルト値として使用されます。
  • 実行時には、options[i].gen を relopt_bool* にキャストすることで、特定のオプションのデフォルト値にアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元（参照・初期化）**:
  • src/backend/access/common/reloptions.c: boolRelOpts[] 配列の定義、init_bool_reloption による初期化、および allocate_reloption を介したメモリ確保。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • relopt_gen: 全てのオプション型が継承する共通ヘッダー。
  • relopt_value: 解析後の値を保持する構造体。

失敗モードと境界条件

  • relopt_gen の型識別子が RELOPT_TYPE_BOOL と一致しない状態でキャストを行うと、メモリ破壊や不正な値の読み取りが発生する可能性があります。

保守担当者向けメモ

  • 新規の真偽値オプションを追加する場合は、reloptions.c 内の boolRelOpts 配列に定義を追加するか、init_bool_reloption を使用してください。
  • 構造体のメモリレイアウトは relopt_gen に依存しているため、gen フィールドの変更は全オプション型に影響します。
