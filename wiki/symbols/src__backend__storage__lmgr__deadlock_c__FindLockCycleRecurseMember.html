FindLockCycleRecurseMember

定義

  • src/backend/storage/lmgr/deadlock.c:535

目的

ロック待ちグラフを再帰的に探索し、特定のプロセス（checkProc）が関与するデッドロックサイクルを検出する。また、解決可能な「ソフトエッジ」を特定し、必要に応じてオートバキュームプロセスのキャンセル候補を記録する。

シグネチャまたは形状

static bool FindLockCycleRecurseMember(
    PGPROC *checkProc,
    PGPROC *checkProcLeader,
    int depth,
    EDGE *softEdges,
    int *nSoftEdges
)

入力と出力

  • **入力**:
  • checkProc: 探索対象のプロセス。
  • checkProcLeader: checkProc が属するロックグループのリーダー。
  • depth: 現在の再帰深さ。
  • softEdges: 検出されたソフトエッジを格納する配列（出力用）。
  • nSoftEdges: softEdges の現在の要素数（入出力用）。
  • **出力**:
  • 戻り値: デッドロックサイクルが検出された場合は true、それ以外は false。

前提条件と不変条件

  • LOCKTAG_RELATION_EXTEND 型のロックはデッドロックサイクルに関与しないため、即座に false を返す。
  • ロックグループのメンバーは、自身や同一グループ内の他メンバーをブロックしない。
  • deadlockDetails 配列は、サイクル検出時に再帰深さに応じて更新される。

アルゴリズムまたは振る舞い

1. **ハードエッジの探索**: checkProc が保持するロックの procLocks リストを走査し、競合するロックを保持しているプロセスを特定する。競合が見つかれば FindLockCycleRecurse を再帰的に呼び出す。 2. **オートバキュームの判定**: checkProc が自身（MyProc）であり、かつ競合相手が PROC_IS_AUTOVACUUM フラグを持つ場合、blocking_autovacuum_proc に記録する。 3. **ソフトエッジの探索**: ロックの待ち行列（waitProcs）を走査する。提案された待ち順序（waitOrders）が存在する場合はそれを使用し、存在しない場合は実際の待ち行列を使用する。 4. **サイクル確定**: 競合するプロセスが見つかり、再帰呼び出しでサイクルが検出された場合、deadlockDetails に情報を格納し、ソフトエッジであれば softEdges 配列に追加して true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: FindLockCycleRecurse（deadlock.c 内の再帰探索プロセス）。
  • **呼び出し先**: FindLockCycleRecurse（再帰）、GetLocksMethodTable。

関連データ構造

  • PGPROC: プロセス情報。
  • LOCK: ロック状態。
  • EDGE: 待ちグラフのエッジ情報。
  • DEADLOCK_INFO: デッドロック発生時の詳細情報。
  • PROCLOCK: プロセスとロックの関連付け。

失敗モードと境界条件

  • nSoftEdges が MaxBackends を超えることはアサーションで禁止されている。
  • 待ち行列の走査において、ロックグループのメンバーに到達した時点で探索を打ち切る最適化が行われている。

保守担当者向けメモ

  • 本関数は DeadLockCheck の一部として実行される。
  • blocking_autovacuum_proc への代入はロックなしで行われるため、PROC_IS_AUTOVACUUM フラグの不変性（一度セットされるとリセットされない）に依存している。
  • 待ち行列の再構成（waitOrders）が考慮されているため、単純なグラフ探索よりも複雑な条件分岐が含まれている。
