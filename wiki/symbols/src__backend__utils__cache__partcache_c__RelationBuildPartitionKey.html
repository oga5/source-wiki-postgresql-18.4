RelationBuildPartitionKey

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/partcache.c:77

目的

指定されたリレーション（パーティションテーブル）のパーティションキー情報をシステムカタログから読み取り、PartitionKeyData 構造体を構築してリレーションキャッシュ（relcache）用に準備する。

シグネチャまたは形状

static void RelationBuildPartitionKey(Relation relation)

入力と出力

  • **入力**: Relation relation（対象となるリレーション構造体）
  • **出力**: なし（副作用として、リレーションに関連付けられたメモリコンテキスト内に PartitionKeyData を構築する）

前提条件と不変条件

  • 対象リレーションが pg_partitioned_table カタログに存在すること。
  • パーティション戦略（LIST, RANGE, HASH）が有効であること。
  • パーティションキーの構築は一度行われると変更されない前提である。

アルゴリズムまたは振る舞い

1. PARTRELID システムキャッシュから対象リレーションの情報を取得する。 2. CurTransactionContext をベースに専用のメモリコンテキスト（partition key）を作成する。 3. pg_partitioned_table カタログから戦略、キー列、演算子クラス、照合順序、式情報を読み取る。 4. パーティション式が存在する場合、定数簡約（eval_const_expressions）と演算子関数IDの修正（fix_opfuncids）を行う。 5. 各キー列について、演算子ファミリーからサポート関数（procnum）を検索し、FmgrInfo を初期化する。 6. 各列の型情報（型ID、型修飾子、照合順序、サイズ等）を収集し、構造体に格納する。 7. 最後にシステムキャッシュのタプルを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: RelationGetPartitionKey（src/backend/utils/cache/partcache.c 内）から呼び出されることが想定されている。
  • **呼び出し先**: SearchSysCache1, SysCacheGetAttr, eval_const_expressions, fix_opfuncids, get_opfamily_proc, fmgr_info_cxt 等。

関連データ構造

  • PartitionKeyData: パーティションキーの定義を保持する主要構造体。
  • Relation: リレーションキャッシュの基本構造体。
  • pg_partitioned_table: パーティションテーブルの定義を保持するシステムカタログ。

失敗モードと境界条件

  • PARTRELID キャッシュ検索失敗時: elog(ERROR) を発生させる。
  • 無効なパーティション戦略: elog(ERROR) を発生させる。
  • 演算子クラスのサポート関数が見つからない場合: ereport(ERROR) を発生させる。
  • パーティションキー式と列数の不整合: elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数で確保されるメモリは CacheMemoryContext の子コンテキストに配置される。relcache エントリの破棄時に適切に解放される設計となっている。
  • パーティションキーの構築はオンデマンドで行われるため、パフォーマンスクリティカルなパスでは注意が必要である。
