pg_decode_commit_txn

定義

  • 種別: function
  • 定義箇所: contrib/test_decoding/test_decoding.c:321

目的

論理デコーディングにおけるトランザクションコミットイベントを処理し、テキスト形式で出力プラグインのバッファへ書き出す。

シグネチャまたは形状

static void pg_decode_commit_txn(LogicalDecodingContext *ctx, ReorderBufferTXN *txn, XLogRecPtr commit_lsn)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • txn: コミットされるトランザクション情報。
  • commit_lsn: コミット時のWALログシーケンス番号。
  • **出力**: なし（戻り値は void）。副作用として ctx->out に COMMIT メッセージが書き込まれる。

前提条件と不変条件

  • ctx->output_plugin_private に TestDecodingData が格納されていること。
  • txn->output_plugin_private に TestDecodingTxnData が格納されていること。
  • 本関数実行後、txn->output_plugin_private は NULL にリセットされる。

アルゴリズムまたは振る舞い

1. TestDecodingTxnData を取得し、トランザクション内で変更が発生したか（xact_wrote_changes）を確認する。 2. txndata をメモリから解放し、txn->output_plugin_private を NULL に設定する。 3. data->skip_empty_xacts が有効かつ変更がない場合、何も出力せずに終了する。 4. OutputPluginPrepareWrite を呼び出し、出力バッファを準備する。 5. data->include_xids の設定に応じて COMMIT または COMMIT <xid> をバッファに追記する。 6. data->include_timestamp が有効な場合、txn->xact_time.commit_time を文字列変換して追記する。 7. OutputPluginWrite を呼び出し、バッファの内容を確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの論理デコーディングエンジン（_PG_output_plugin_init で commit_cb として登録）。
  • **呼び出し先**:
  • pfree
  • OutputPluginPrepareWrite
  • appendStringInfo / appendStringInfoString
  • timestamptz_to_str
  • OutputPluginWrite

関連データ構造

  • LogicalDecodingContext: デコーディング状態を保持するコア構造体。
  • ReorderBufferTXN: トランザクションの詳細情報を保持する構造体。
  • TestDecodingData: プラグイン全体の設定を保持。
  • TestDecodingTxnData: トランザクションごとの状態を保持。

失敗モードと境界条件

  • skip_empty_xacts オプションが有効な場合、変更を含まないトランザクションは出力されない。
  • メモリ不足時には pfree 等の内部関数によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は commit_cb として登録されるコールバックである。
  • txn->output_plugin_private の解放責任を持つため、他のコールバックから参照されないタイミングで呼び出される必要がある。
