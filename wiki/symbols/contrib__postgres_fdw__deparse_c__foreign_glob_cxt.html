foreign_glob_cxt

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/deparse.c:66

目的

外部テーブルのプランニングおよびデパース処理において、クエリ全体のグローバルな状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct foreign_glob_cxt
{
    PlannerInfo *root;      /* グローバルなプランナ状態 */
    RelOptInfo *foreignrel; /* プランニング対象の外部リレーション */
    Relids      relids;     /* 基底リレーションの識別子セット */
} foreign_glob_cxt;

入力と出力

本構造体は関数引数として渡されることで、プランニング中の情報を共有します。

  • 入力: PlannerInfo（プランナ状態）、RelOptInfo（外部リレーション情報）、Relids（スキャン対象の基底リレーション）。
  • 出力: なし（構造体内のフィールドが参照・更新される）。

前提条件と不変条件

  • root および foreignrel は、プランニングの実行中に有効なポインタである必要があります。
  • relids は、対象となるスキャン操作に関連するリレーションを正確に反映している必要があります。

アルゴリズムまたは振る舞い

本構造体は、postgres_fdw におけるクエリのデパース（SQL文字列への変換）プロセスにおいて、プランニングの文脈を維持するために使用されます。主に、式ツリーの走査や安全性判定を行う関数間で、クエリ全体のメタデータを共有する役割を担います。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/postgres_fdw/deparse.c 内の関数（例: 246 行目付近でインスタンス化され、120 行目や 311 行目の関数へ引数として渡される）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • PlannerInfo: PostgreSQL プランナのグローバル状態。
  • RelOptInfo: リレーションの最適化情報。
  • Relids: リレーション識別子の集合。
  • deparse_expr_cxt: SQL 構築中の状態を保持する別のコンテキスト構造体。

失敗モードと境界条件

  • 構造体内のポインタが NULL の状態で参照された場合、セグメンテーション違反が発生する可能性があります。
  • プランニングの文脈外で本構造体を使用した場合、整合性の取れないデータにアクセスするリスクがあります。

保守担当者向けメモ

  • 本構造体は deparse.c 内の複数の関数で共有されるため、フィールドの追加や変更を行う際は、依存するすべての関数（is_foreign_expr 等の呼び出し元）への影響を確認してください。
