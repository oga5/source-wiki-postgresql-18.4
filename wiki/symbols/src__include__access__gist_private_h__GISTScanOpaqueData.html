GISTScanOpaqueData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/gist_private.h:154

目的

GiST インデックスのスキャン実行時における非公開の状態管理を行うための構造体です。

シグネチャまたは形状

typedef struct GISTScanOpaqueData
{
    GISTSTATE  *giststate;
    Oid        *orderByTypes;
    pairingheap *queue;
    MemoryContext queueCxt;
    bool        qual_ok;
    bool        firstCall;
    IndexOrderByDistance *distances;
    OffsetNumber *killedItems;
    int         numKilled;
    BlockNumber curBlkno;
    GistNSN     curPageLSN;
    GISTSearchHeapItem pageData[BLCKSZ / sizeof(IndexTupleData)];
    OffsetNumber nPageData;
    OffsetNumber curPageData;
    MemoryContext pageDataCxt;
} GISTScanOpaqueData;

入力と出力

  • **入力**: スキャン開始時に GISTSTATE や orderByTypes が設定されます。
  • **出力**: pageData 配列を介して、インデックススキャンで取得されたタプル情報が呼び出し元に提供されます。

前提条件と不変条件

  • queue は pairingheap を使用して未訪問アイテムを管理します。
  • killedItems が使用されない場合、値は NULL となります。
  • pageData は非順序検索時にヒープタプルを格納するために使用されます。

アルゴリズムまたは振る舞い

  • pairingheap を利用し、距離順または深さ優先順でインデックスページおよびヒープタプルを探索します。
  • gistgettuple 呼び出しの初回までは firstCall が true を保持します。
  • インデックスのみのスキャン（index-only scan）時には、pageDataCxt を使用して取得したタプルのメモリ管理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistscan.c にて palloc0 を用いてインスタンス化されます。
  • **呼び出し先**: 構造体内部の各フィールドは、GiST インデックスの検索処理（gistgettuple 等）において参照・更新されます。

関連データ構造

  • GISTSTATE: インデックスのサポート関数や記述子を保持。
  • GISTSearchHeapItem: 非順序検索時に格納されるタプル情報。
  • pairingheap: 検索順序制御のためのヒープ構造。

失敗モードと境界条件

  • qual_ok が false の場合、検索条件を満たすタプルが存在しないことを示します。
  • pageData のサイズは BLCKSZ に依存しており、格納可能なタプル数には物理的な制限があります。

保守担当者向けメモ

  • 本構造体は GiST インデックスの内部状態を保持するため、スキャン処理の変更時には gistscan.c との整合性に注意してください。
  • pageData のメモリ管理は pageDataCxt に依存しているため、リークを防ぐための適切なコンテキスト管理が必要です。
