get_var1

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-outofscope.c:172

目的

ECPGの回帰テストにおいて、カーソル操作に使用する構造体メモリをヒープ上に確保し、ECPGランタイムへ変数を登録するために使用されます。

シグネチャまたは形状

static void get_var1(MYTYPE **myvar0, MYNULLTYPE **mynullvar0)

入力と出力

  • **入力**:
  • myvar0: MYTYPE 構造体ポインタを格納するためのポインタ。
  • mynullvar0: MYNULLTYPE 構造体ポインタを格納するためのポインタ。
  • **出力**:
  • 引数経由で確保されたメモリ領域のポインタを返却します。

前提条件と不変条件

  • 呼び出し側は、MYTYPE および MYNULLTYPE のメモリ領域を解放する責任を負います。
  • ECPGset_var が成功し、SQL操作の準備が整っている必要があります。

アルゴリズムまたは振る舞い

1. malloc を使用して MYTYPE および MYNULLTYPE 用のメモリをヒープ上に確保します。 2. ECPGset_var を呼び出し、確保した変数をECPGランタイムに登録します。 3. DECLARE 文を実行し、カーソル mycur を定義します。 4. sqlca.sqlcode を確認し、エラーが発生した場合は exit(1) で終了します。 5. 確保したポインタを引数 myvar0 および mynullvar0 に代入して呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/preproc-outofscope.c:325 (main関数内)
  • **呼び出し先**: malloc, ECPGset_var (ECPGライブラリ関数)

関連データ構造

  • MYTYPE: データベース行データを保持する構造体。
  • MYNULLTYPE: NULL状態を追跡するインジケータ構造体。
  • sqlca: SQL通信領域。

失敗モードと境界条件

  • malloc が失敗した場合、プログラムは異常終了する可能性があります。
  • ECPGset_var 後の sqlca.sqlcode が負の値（エラー）である場合、exit(1) によりプロセスが終了します。

保守担当者向けメモ

  • 本関数はECPGプリプロセッサが生成したテストコードの一部です。手動で修正する際は、プリプロセッサの生成ルールと整合性を保つ必要があります。
  • ECPGset_var の引数 0 および 1 は、ECPGが内部的に管理する変数インデックスに対応しています。
