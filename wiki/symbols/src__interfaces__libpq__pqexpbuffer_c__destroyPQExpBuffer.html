destroyPQExpBuffer

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.c:113

目的

PQExpBuffer 構造体によって管理されている動的バッファのメモリを解放し、構造体自体を破棄する。

シグネチャまたは形状

void destroyPQExpBuffer(PQExpBuffer str)

入力と出力

  • **入力**: PQExpBuffer str（解放対象のバッファ構造体へのポインタ）
  • **出力**: なし

前提条件と不変条件

  • str が NULL の場合、本関数は何もしない。
  • str が有効なポインタである場合、内部で保持されている文字列バッファのメモリが termPQExpBuffer によって解放され、その後 str 自体が free される。

アルゴリズムまたは振る舞い

1. 引数 str が NULL でないことを確認する。 2. termPQExpBuffer(str) を呼び出し、構造体内部のデータ領域を解放する。 3. free(str) を呼び出し、構造体本体のメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し先**: termPQExpBuffer
  • **呼び出し元**:
  • src/bin/initdb/initdb.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_createsubscriber.c

    • (上記はソースコード内の直接的な呼び出し箇所に基づく)

関連データ構造

  • PQExpBuffer: 動的に拡張可能な文字列バッファを保持する構造体。

失敗モードと境界条件

  • str が NULL の場合、安全に無視される。
  • termPQExpBuffer 内部でのメモリ解放処理に依存する。

保守担当者向けメモ

  • 本関数は malloc で確保されたメモリを解放するため、必ず対応する初期化関数（createPQExpBuffer 等）で生成されたオブジェクトに対して呼び出すこと。
  • termPQExpBuffer との役割分担に注意すること（termPQExpBuffer は構造体内部の解放のみを行い、destroyPQExpBuffer は構造体自体の解放も行う）。
