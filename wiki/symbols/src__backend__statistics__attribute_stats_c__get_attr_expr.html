get_attr_expr

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/attribute_stats.c:555

目的

インデックスリレーションにおいて、指定された属性番号（attnum）に対応するインデックス式（Node）を取得します。

シグネチャまたは形状

static Node *get_attr_expr(Relation rel, int attnum)

入力と出力

  • **入力**:
  • rel: 対象の Relation 構造体。
  • attnum: 取得対象の属性番号。
  • **出力**:
  • 指定された属性がインデックス式である場合、その Node を返します。
  • インデックスではない場合、式が存在しない場合、または指定された属性が単純な列参照である場合は NULL を返します。

前提条件と不変条件

  • rel は有効なリレーション構造体である必要があります。
  • attnum は 1 から始まる属性番号です。
  • インデックス式が存在する場合、rel->rd_indexprs に格納されている必要があります。

アルゴリズムまたは振る舞い

1. **リレーション種別の確認**: rel がインデックス（RELKIND_INDEX または RELKIND_PARTITIONED_INDEX）でない場合は NULL を返します。 2. **式の存在確認**: インデックス式リストが空であれば NULL を返します。 3. **属性の判定**: rel->rd_index->indkey を参照し、指定された attnum が単純な列参照（値が 0 以外）であれば NULL を返します。 4. **式の探索**: rel->rd_indexprs リストを走査し、インデックスキー内の 0（式を表すプレースホルダー）の出現回数をカウントして、対応するインデックス式を特定します。 5. **返却**: 特定した Node を返します。リストの要素数が不足している場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/attribute_stats.c 内の統計情報更新処理（行 624 など）。
  • **呼び出し先**: RelationGetIndexExpressions、list_head、lnext、lfirst などのリスト操作マクロ。

関連データ構造

  • Relation: リレーションのメタデータ構造体。
  • Node: PostgreSQL の構文木ノード。
  • Form_pg_index: rel->rd_index を通じてアクセスされるインデックスの定義情報。

失敗モードと境界条件

  • **エラー**: インデックスキーの定義と rd_indexprs リストの整合性が取れていない場合（リストの要素数が足りない場合）、ERROR が発生します。
  • **境界条件**: attnum がインデックスの範囲外である場合、またはインデックス式が存在しない場合は NULL を返します。

保守担当者向けメモ

  • 本関数は、インデックス式に対する統計情報を扱う際に、どの式がどの属性に対応するかを特定するために使用されます。
  • indkey の値が 0 である位置をカウントして indexprs リストを辿るロジックであるため、インデックス定義の変更時には整合性に注意してください。
