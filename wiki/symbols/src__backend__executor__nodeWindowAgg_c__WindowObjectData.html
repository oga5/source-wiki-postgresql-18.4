WindowObjectData

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeWindowAgg.c:63

目的

ウィンドウ関数APIを通じて、実行中のウィンドウ関数に対してコンテキスト情報およびデータアクセス手段を提供します。

シグネチャまたは形状

typedef struct WindowObjectData
{
    NodeTag     type;
    WindowAggState *winstate;    /* 親のWindowAggState */
    List       *argstates;       /* 関数の引数用ExprStateツリー */
    void       *localmem;        /* WinGetPartitionLocalMemory用チャンク */
    int         markptr;         /* この関数用のtuplestoreマークポインタ */
    int         readptr;         /* この関数用のtuplestore読み取りポインタ */
    int64       markpos;         /* markptrが位置する行番号 */
    int64       seekpos;         /* readptrが位置する行番号 */
} WindowObjectData;

入力と出力

  • **入力**: WindowAggState からの参照および tuplestore を介したパーティション内のタプルデータ。
  • **出力**: ウィンドウ関数の計算結果（WindowObject を介して間接的に提供）。

前提条件と不変条件

  • WindowObject 型（WindowObjectData*）として使用される際、IsA(winobj, WindowObjectData) が真である必要があります。
  • markptr および readptr は、tuplestore 内の有効な読み取り位置を指している必要があります。

アルゴリズムまたは振る舞い

  • ウィンドウ関数実行時に、パーティション内の特定の行へのアクセスや、フレーム境界の管理を行います。
  • localmem を通じて、パーティションごとの一時的なメモリ領域を保持します。
  • tuplestore のポインタ（markptr, readptr）を操作することで、ウィンドウフレーム内の行を走査します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeWindowAgg.c 内の ExecWindowAgg 等の関数で makeNode(WindowObjectData) により生成されます。
  • **呼び出し先**: WindowObject を受け取るウィンドウ関数API（WinGetFuncArgInPartition 等）が本構造体を参照します。
  • ※詳細なコールグラフは提供されたソース範囲外のため不明です。

関連データ構造

  • WindowAggState: 親となる実行状態構造体。
  • WindowObject: WindowObjectData へのポインタ型。
  • tuplestore: パーティションデータを保持するストレージ。

失敗モードと境界条件

  • winstate が NULL の場合や、tuplestore のポインタが無効な位置を指している場合は未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/windowapi.h で公開されており、ウィンドウ関数APIの基盤です。変更時はAPIの互換性に注意してください。
  • markpos および seekpos は行番号を追跡しており、フレーム計算のロジックと密接に関連しています。
