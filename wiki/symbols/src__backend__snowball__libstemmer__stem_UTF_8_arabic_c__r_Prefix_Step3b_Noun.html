r_Prefix_Step3b_Noun

目的

アラビア語のステミング処理において、名詞の特定の接頭辞パターンを検索し、一致した場合は削除または置換を行う。

シグネチャまたは形状

static int r_Prefix_Step3b_Noun(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体）
  • 出力: 処理が成功した場合は 1、一致しない場合やエラー時は 0 または負の値を返す。

前提条件と不変条件

  • z->c（現在のカーソル位置）が有効な範囲内であること。
  • z->p（文字列バッファ）が UTF-8 エンコーディングであること。
  • 処理対象の文字列長が 3 バイトを超えている必要がある。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c を開始点 z->bra に設定する。 2. カーソル位置の次（z->c + 1）の文字コードが 168 または 131 であるかを確認する。 3. find_among を使用し、テーブル a_7（サイズ 4）に基づいて接頭辞を検索する。 4. 一致した場合、z->ket を現在のカーソル位置に設定し、以下のいずれかの操作を行う：

  • case 1: slice_del で接頭辞を削除する。
  • case 2: slice_from_s で s_59 に置換する。
  • case 3: slice_from_s で s_60 に置換する。

    • 5. 各操作において、文字列長が 3 バイト以下の場合は処理を中断する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_arabic.c:1629
  • 呼び出し先: find_among, len_utf8, slice_del, slice_from_s（いずれも Snowball ランタイム関数）

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_7: 接頭辞検索用の among 構造体テーブル。

失敗モードと境界条件

  • 文字列長が 3 バイト以下の場合は処理をスキップし 0 を返す。
  • find_among が一致を見つけられない場合は 0 を返す。
  • slice_del または slice_from_s が負の値を返した場合（メモリ不足等）、その値をそのまま戻り値として伝播させる。

保守担当者向けメモ

  • UTF-8 のマルチバイト文字を扱うため、カーソル操作には skip_utf8 等のヘルパー関数が前提となっている。
