目的

pg_recvlogical クライアントツールのエントリーポイントであり、コマンドライン引数の解析、設定の初期化、および論理レプリケーションストリーミングの実行制御を行う。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**: コマンドライン引数 (argc, argv)。接続先情報、レプリケーションスロット操作、ストリーミングの開始・終了位置、およびファイル出力設定など。
  • **出力**: 正常終了時は 0、エラー発生時や不正な引数の場合は 1 を返す。

前提条件と不変条件

  • pg_logging_init および set_pglocale_pgservice により、ログ出力とロケールが初期化されている必要がある。
  • getopt_long を使用して引数を解析し、グローバルな設定変数（outfile, fsync_interval, replication_slot 等）を更新する。

アルゴリズムまたは振る舞い

1. **初期化**: ログとロケールの設定を行う。 2. **ヘルプ・バージョン表示**: --help や --version が指定された場合は即座に終了する。 3. **引数解析**: getopt_long を用いて、接続オプション、レプリケーションオプション、およびアクション（スロット作成・削除・開始）を解析する。 4. **オプション検証**: sscanf を用いて startpos および endpos を 16 進数文字列から uint64 へ変換する。 5. **引数チェック**: 解析終了後、余分なコマンドライン引数が存在しないかを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（プログラムの起動時）。
  • **呼び出し先**:
  • usage(): ヘルプ表示時に呼び出される。
  • pg_fatal / pg_log_error: エラー発生時に呼び出される。
  • getopt_long: 引数解析に使用。
  • pg_strdup / pg_realloc: メモリ確保に使用。
  • ※本関数以降のストリーミング処理（StreamLogicalLog 等）への遷移は、提供されたソース抜粋範囲外のため不明。

関連データ構造

  • struct option long_options[]: コマンドライン引数の定義。
  • options (char配列): レプリケーションプラグインに渡すオプションを保持する動的配列。

失敗モードと境界条件

  • **引数エラー**: 不正なオプションや解析不能な LSN 形式が指定された場合、エラーメッセージを出力して終了する。
  • **メモリ不足**: pg_strdup や pg_realloc 失敗時の挙動は基盤ライブラリに依存する。
  • **引数過多**: optind < argc の条件により、未知の引数が残っている場合はエラーとして処理される。

保守担当者向けメモ

  • 本関数は引数解析の大部分を担っている。ストリーミングのメインループへ移行する前の状態管理（do_create_slot, do_start_slot 等のフラグ設定）が適切に行われているか確認すること。
  • getopt_long の定義と switch 文のケースが一致しているか、新しいオプション追加時に注意が必要。
