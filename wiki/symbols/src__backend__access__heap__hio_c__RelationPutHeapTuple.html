RelationPutHeapTuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/hio.c:34

目的

指定されたバッファ内のページへタプルを物理的に挿入し、タプルの t_self（および非投機的挿入の場合は t_ctid）を更新する。

シグネチャまたは形状

void RelationPutHeapTuple(Relation relation, Buffer buffer, HeapTuple tuple, bool token)

入力と出力

  • relation: 対象のリレーション。
  • buffer: タプルを挿入する対象のバッファ。
  • tuple: 挿入対象のタプル構造体。
  • token: 投機的挿入（speculative insertion）かどうかを示すフラグ。
  • 出力: なし（副作用として buffer 内のページおよび tuple 構造体が更新される）。

前提条件と不変条件

  • token が真の場合、タプルは投機的挿入の要件を満たしている必要がある（HeapTupleHeaderIsSpeculative が真）。
  • タプルのヒントビットの組み合わせが不正でないこと（HEAP_XMAX_COMMITTED と HEAP_XMAX_IS_MULTI が同時に設定されていないこと）。

アルゴリズムまたは振る舞い

1. BufferGetPage を用いてバッファからページヘッダーを取得する。 2. PageAddItem を呼び出し、ページ内にタプルを格納する。格納に失敗した場合は PANIC を発生させる。 3. 挿入された位置（ブロック番号とオフセット番号）を ItemPointerSet を用いて tuple->t_self に書き込む。 4. token が偽（通常の挿入）の場合、ページ内のタプルヘッダーにある t_ctid を tuple->t_self で更新する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heapam.c 内の複数の箇所（heap_insert 等のタプル挿入処理）。
  • 呼び出し先: BufferGetPage, PageAddItem, ItemPointerSet, PageGetItemId, PageGetItem, BufferGetBlockNumber。

関連データ構造

  • Relation: リレーション記述子。
  • Buffer: バッファマネージャのバッファ識別子。
  • HeapTuple: タプルデータ構造。

失敗モードと境界条件

  • ページ内にタプルを格納する十分な空き領域がない場合、elog(PANIC, ...) によりシステムが停止する。
  • 不正なヒントビットの組み合わせが検出された場合、アサーション違反により停止する。

保守担当者向けメモ

  • 本関数内での ereport(ERROR) は禁止されている。バッファロックを保持した状態でのエラーはクリーンアップが困難なため、致命的な失敗には PANIC を使用すること。
  • 投機的挿入（token が真）の場合、t_ctid の更新はスキップされる。これは t_ctid フィールドがトークン保持のために使用されるためである。
