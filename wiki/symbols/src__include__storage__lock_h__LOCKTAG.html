LOCKTAG

定義

  • 種別: struct
  • 定義箇所: src/include/storage/lock.h:165

目的

PostgreSQLのロックマネージャにおいて、ロック対象となるリソース（リレーション、ページ、タプル等）を一意に識別するために使用されるタグ構造体です。

シグネチャまたは形状

typedef struct LOCKTAG
{
	uint32		locktag_field1;
	uint32		locktag_field2;
	uint32		locktag_field3;
	uint16		locktag_field4;
	uint8		locktag_type;
	uint8		locktag_lockmethodid;
} LOCKTAG;

入力と出力

  • **入力**: ロック対象の識別情報（フィールド1〜4）およびロック種別（locktag_type）、ロックメソッドID（locktag_lockmethodid）。
  • **出力**: 共有メモリ上のハッシュテーブルにおけるキーとして機能し、ロックの競合判定や管理の起点となります。

前提条件と不変条件

  • 構造体全体のサイズは16バイトに収まるよう設計されており、この制約を維持する必要があります。
  • locktag_type は LockTagType 列挙型に基づきます。

アルゴリズムまたは振る舞い

1. ロック要求時、LOCKTAG からハッシュ値を算出し、共有メモリ上の適切なロックパーティションを特定します。 2. ロックテーブル内でこのタグをキーとして検索し、対応する LOCK オブジェクトを特定または生成します。 3. 競合チェックおよび待機キューへの追加処理において、このタグがリソースの同一性を保証します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/heap/heapam.c
  • src/backend/catalog/index.c
  • src/backend/catalog/pg_subscription.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/tablecmds.c
  • src/backend/storage/ipc/standby.c（ResolveRecoveryConflictWithLock 等）
  • **呼び出し先**: 構造体定義のため、特定の関数呼び出し先はありません。

関連データ構造

  • LOCK: 共有メモリ上でロック状態を保持する構造体。
  • LockTagType: ロック対象の種類を定義する列挙型。
  • LOCKMETHODID: ロックメソッドを識別するID。

失敗モードと境界条件

  • 16バイトというサイズ制限があるため、フィールドの追加や拡張には厳格な制約が伴います。
  • 共有メモリ上のハッシュテーブルで管理されるため、メモリ不足やハッシュ衝突の管理が重要となります。

保守担当者向けメモ

  • 本構造体はフロントエンドコードからはインクルードできません。
  • ロックテーブルはパーティション化されており、LockHashPartition マクロを使用してアクセス先を決定します。
  • フィールドの変更は、ロックの識別ロジック全体に影響を与えるため慎重に行ってください。
