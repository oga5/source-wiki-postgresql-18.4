get_errno_symbol

定義

  • 種別: function
  • 定義箇所: src/port/strerror.c:112

目的

数値の errno 値を、対応する標準的なエラーシンボル名（文字列）に変換します。OS が提供するエラーメッセージが空または無効な場合に、デバッグ情報を補完するために使用されます。

シグネチャまたは形状

static char *get_errno_symbol(int errnum);

入力と出力

  • **入力**: errnum (int型) - 変換対象のエラー番号。
  • **出力**: char * - 対応するシンボル名（例: "ENOENT"）。該当するシンボルが見つからない場合は NULL を返します。

前提条件と不変条件

  • 網羅的なマッピングではなく、主要なエラーコードのみを対象としています。
  • プラットフォーム固有のヘッダー定義（EAGAIN, ELOOP 等）に基づき、条件付きコンパイルによって利用可能なシンボルが決定されます。

アルゴリズムまたは振る舞い

1. errnum を引数として switch 文を実行します。 2. 定義済みの各エラー定数（E2BIG, EACCES, ENOENT 等）に一致する場合、そのシンボル名を文字列リテラルとして返します。 3. 一致するケースがない場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/strerror.c 内の pg_strerror_r から呼び出されます。
  • **呼び出し先**: 本関数はライブラリ関数を呼び出さず、定数値を返すのみです。

関連データ構造

  • 特になし（標準的な errno 定数を使用）。

失敗モードと境界条件

  • **失敗モード**: 引数 errnum が本関数内の switch 文で定義されていない値である場合、NULL を返します。
  • **境界条件**: EAGAIN や EWOULDBLOCK など、プラットフォームによって定義が重複または欠落する可能性がある定数については、プリプロセッサディレクティブにより安全に処理されます。

保守担当者向けメモ

  • 本関数は pg_strerror_r のフォールバックロジックの一部です。新しいエラーコードのサポートを追加する場合は、switch 文に case を追加してください。
  • 戻り値が NULL の場合、呼び出し元は数値ベースのエラーメッセージ（例: "operating system error %d"）を生成する設計になっています。
