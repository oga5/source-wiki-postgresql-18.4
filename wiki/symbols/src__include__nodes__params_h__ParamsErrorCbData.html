ParamsErrorCbData

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/params.h:154

目的

エラー発生時にパラメータ情報をコンテキストとして保持し、エラーメッセージやデバッグ情報に含めるためのデータ構造です。

シグネチャまたは形状

typedef struct ParamsErrorCbData
{
	const char *portalName;
	ParamListInfo params;
} ParamsErrorCbData;

入力と出力

  • **入力**: portalName（ポータル名を示す文字列へのポインタ）、params（パラメータリスト情報へのポインタ）。
  • **出力**: なし（本構造体自体はデータ保持用）。

前提条件と不変条件

  • portalName は、エラー発生時に参照可能な有効な文字列を指している必要があります。
  • params は、エラー発生時に有効な ParamListInfo 構造体を指している必要があります。

アルゴリズムまたは振る舞い

本構造体は、エラーハンドリングのコールバック関数に渡される引数として使用されます。エラー発生時に portalName と params を参照することで、どのポータルでどのようなパラメータが使用されていたかを特定する補助を行います。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/tcop/postgres.c:1644
  • src/backend/tcop/postgres.c:2123
  • **呼び出し先**:
  • src/backend/nodes/params.c:409（arg としてキャストされ、内部データとして利用される）

関連データ構造

  • ParamListInfo: パラメータの集合を保持する構造体。本構造体の params フィールドとして利用されます。

失敗モードと境界条件

  • portalName が NULL の場合、エラーレポート時にポータル名が正しく表示されない可能性があります。
  • params が NULL の場合、エラーハンドリング中にパラメータ情報を参照しようとするとセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は主にエラー報告の文脈で使用されます。エラーコールバックの登録・解除ロジックを変更する際は、src/backend/tcop/postgres.c でのインスタンス化とライフサイクル管理に注意してください。
