MVNDistinctItem

定義

  • 種別: struct
  • 定義箇所: src/include/statistics/statistics.h:26

目的

列の組み合わせに対する多変量NDistinct（個体数）統計情報を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct MVNDistinctItem
{
    double      ndistinct;      /* ndistinct value for this combination */
    int         nattributes;    /* number of attributes */
    AttrNumber *attributes;     /* attribute numbers */
} MVNDistinctItem;

入力と出力

  • 入力: 統計情報の構築時（ANALYZE等）に、対象となる列の組み合わせと算出された個体数を受け取ります。
  • 出力: プランナがクエリの選択率を推定する際、特定の列の組み合わせに対する個体数情報を提供します。

前提条件と不変条件

  • nattributes は、この統計情報が対象とする列の数を表します。
  • attributes は、対象列を識別する AttrNumber の配列を指します。
  • ndistinct は、指定された列の組み合わせにおける個体数の推定値です。

アルゴリズムまたは振る舞い

本構造体は、特定の列の組み合わせに対する統計的な個体数を保持します。プランナは、クエリ内の句に含まれる列と本構造体の attributes を照合し、ndistinct を用いて選択率を推定します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/statistics/mvdistinct.c: 統計情報の生成および処理において、配列の確保や要素へのアクセスに使用。
  • src/backend/utils/adt/selfuncs.c: 選択率推定処理において、統計情報の検索や評価に使用。
  • 呼び出し先: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • MVNDistinct: MVNDistinctItem を複数保持する上位の構造体であり、items フィールドとして MVNDistinctItem の可変長配列（FLEXIBLE_ARRAY_MEMBER）を含みます。

失敗モードと境界条件

  • FLEXIBLE_ARRAY_MEMBER を使用した構造体であるため、メモリ確保やポインタ操作を行う際は、構造体のサイズ計算を正しく行う必要があります。
  • attributes 配列が指すメモリ領域の管理は、上位の MVNDistinct 構造体のライフサイクルに依存します。

保守担当者向けメモ

  • 本構造体は拡張統計情報の一部として使用されます。新しい統計型を追加する際は、整合性を保つために statistics.h 内の他の構造体定義と併せて確認してください。
  • attributes はポインタであるため、シリアライズおよびデシリアライズの際には、メモリレイアウトの整合性に注意してください。
