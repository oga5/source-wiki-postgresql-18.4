shutdown_validator_library

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-oauth.c:805

目的

ロード済みの外部OAuth検証モジュールに対して終了処理（シャットダウン）を実行し、リソースの解放を促す。

シグネチャまたは形状

static void shutdown_validator_library(void *arg);

入力と出力

  • **入力**: void *arg（未使用）
  • **出力**: なし（void）

前提条件と不変条件

  • ValidatorCallbacks が初期化されており、有効なポインタを保持していること。
  • validator_module_state が、ロードされた検証モジュールの現在の状態を指していること。

アルゴリズムまたは振る舞い

1. ValidatorCallbacks 構造体の shutdown_cb メンバが NULL でないかを確認する。 2. shutdown_cb が存在する場合、validator_module_state を引数としてその関数を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c:796 にて、mcb->func に関数ポインタとして登録されている。
  • **呼び出し先**: ValidatorCallbacks->shutdown_cb（外部検証モジュールが提供するコールバック関数）。

関連データ構造

  • ValidatorCallbacks: 外部検証モジュールの関数ポインタを保持する構造体。
  • validator_module_state: 検証モジュールが保持する内部状態。

失敗モードと境界条件

  • ValidatorCallbacks が NULL の場合、あるいは shutdown_cb が NULL の場合、関数は何も実行せずに終了する（安全なガード条件）。

保守担当者向けメモ

  • 本関数は、認証メカニズムの終了処理の一環として呼び出されることを想定している。
  • arg 引数は現在の実装では使用されていないが、将来的な拡張やインターフェースの互換性のために保持されている。
