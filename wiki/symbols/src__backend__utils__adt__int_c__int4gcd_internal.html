int4gcd_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int.c:1261

目的

2つの int32 値の最大公約数（GCD）を計算します。

シグネチャまたは形状

static int32 int4gcd_internal(int32 arg1, int32 arg2)

入力と出力

  • **入力**: arg1 (int32), arg2 (int32)
  • **出力**: 両引数の最大公約数（int32）。結果は常に正の値として返されます。

前提条件と不変条件

  • INT_MIN（PG_INT32_MIN）を含む入力値を安全に処理します。
  • 内部計算は負の空間で行われ、最終的に正の値に変換されます。

アルゴリズムまたは振る舞い

1. **正規化**: 入力値を負の空間に変換し、絶対値の大きい方を arg2 に配置します。 2. **特殊ケース処理**: arg1 が PG_INT32_MIN の場合、arg2 が 0 または PG_INT32_MIN であればエラー（ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE）を報告します。arg2 が -1 の場合は 1 を返します。 3. **ユークリッドの互除法**: arg2 が 0 になるまで剰余演算を繰り返します。 4. **結果の正数化**: 最終的な結果が負であれば正に反転させて返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/int.c 内の他の関数から呼び出されています（詳細はソースコードの文脈に依存）。
  • **呼び出し先**: ereport（エラー報告用マクロ）を呼び出します。

関連データ構造

  • int32: PostgreSQLの4バイト整数型。

失敗モードと境界条件

  • **範囲外エラー**: arg1 が PG_INT32_MIN で、かつ arg2 が 0 または PG_INT32_MIN の場合に ereport でトランザクションを中断します。
  • **浮動小数点例外の回避**: INT_MIN % -1 のような環境依存の例外を回避するため、明示的なチェックを行っています。

保守担当者向けメモ

  • 本関数は int4 型の演算用ユーティリティです。
  • INT_MIN の扱いは複雑であるため、ロジックを変更する際は負の空間での計算結果が正しく正数化されることを確認してください。
