GinScanItem

定義

  • 種別: struct
  • 定義箇所: contrib/amcheck/verify_gin.c:36

目的

GINインデックスの深さ優先探索（DFS）における走査状態を保持するためのスタック要素として使用されます。

シグネチャまたは形状

typedef struct GinScanItem
{
    int            depth;
    IndexTuple     parenttup;
    BlockNumber    parentblk;
    BlockNumber    blkno;
    struct GinScanItem *next;
} GinScanItem;

入力と出力

  • 本構造体は関数引数として渡されるのではなく、palloc を用いてメモリ上に動的に確保され、スタック構造として連結されます。
  • 出力として、走査中の特定のインデックスページ位置（blkno）およびその親のコンテキスト情報を提供します。

前提条件と不変条件

  • next ポインタにより、単方向連結リスト（スタック）を形成します。
  • depth は現在の走査階層を追跡するために使用されます。

アルゴリズムまたは振る舞い

  • GINインデックスの検証プロセスにおいて、次に探索すべきページを管理するスタックのノードとして機能します。
  • gin_check_parent_keys_consistency 内で、インデックスの階層構造を辿る際の現在の探索位置を記録するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_gin.c 内の gin_check_parent_keys_consistency 関数が、探索スタックの構築および管理のために本構造体をインスタンス化します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • GinPostingTreeScanItem: ポスティングツリーの検証時に使用される類似の走査用構造体。
  • IndexTuple: GINインデックス内のタプル情報。

失敗モードと境界条件

  • メモリ不足時には palloc によりエラーが発生する可能性があります。
  • スタックの深さがインデックスの物理的な階層を超えて不適切に管理された場合、探索ロジックの不整合を招く可能性があります。

保守担当者向けメモ

  • 本構造体は contrib/amcheck/verify_gin.c 内でのみ使用されるプライベートなデータ構造です。
  • 探索スタックの操作（プッシュ/ポップ）は、gin_check_parent_keys_consistency 内のロジックに依存しているため、変更時は当該関数のスタック管理フローを確認してください。
