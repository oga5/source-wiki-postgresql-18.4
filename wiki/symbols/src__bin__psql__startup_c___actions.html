_actions

定義

  • 種別: enum
  • 定義箇所: src/bin/psql/startup.c:46

目的

psql 起動時に実行される単発のアクション（コマンドやスクリプト実行）の種類を識別するために使用されます。

シグネチャまたは形状

enum _actions
{
    ACT_SINGLE_QUERY,
    ACT_SINGLE_SLASH,
    ACT_FILE,
};

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力は持ちません。
  • psql の実行フローにおいて、コマンドライン引数から解析されたアクションを特定する識別子として利用されます。

前提条件と不変条件

  • psql の起動プロセスにおいて、ユーザーが指定した実行モードを決定する際に使用されます。
  • 実行されるアクションは、SimpleActionList 構造体を通じてリスト化されることが想定されています。

アルゴリズムまたは振る舞い

  • ACT_SINGLE_QUERY: -c オプションで指定された SQL クエリを実行します。
  • ACT_SINGLE_SLASH: -c オプション（またはそれに準ずるもの）で指定されたメタコマンドを実行します。
  • ACT_FILE: -f オプションで指定されたファイルの内容を読み込み、実行します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/startup.c 内の引数解析およびアクション処理ロジック（simple_action_list_append 等）から参照されています。
  • 呼び出し先: 本シンボル自体は列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • SimpleActionList: アクションを保持するリスト構造。
  • SimpleActionListCell: リストの各要素。
  • adhoc_opts: コマンドライン引数の解析結果を格納する構造体。

失敗モードと境界条件

  • 列挙型であるため、定義外の値が代入された場合の挙動は未定義です。
  • アクションのリストが空の場合、psql は対話モード（MainLoop）へ移行します。

保守担当者向けメモ

  • 新しい起動時アクションを追加する場合は、この列挙型に値を追加し、startup.c 内の処理ロジックを更新してください。
