listAllDbs

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:945

目的

データベース一覧を表示するメタコマンド（\l）の実行ロジックを提供します。

シグネチャまたは形状

bool listAllDbs(const char *pattern, bool verbose)

入力と出力

  • **入力**:
  • pattern: データベース名のフィルタリングに使用するパターン（NULL可）。
  • verbose: 詳細表示（サイズ、テーブルスペース、説明）を有効にするフラグ。
  • **出力**:
  • bool: 処理が成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • pset.sversion（サーバーバージョン）に基づき、クエリの生成内容が動的に変化します。
  • pattern が指定された場合、validateSQLNamePattern による検証を通過する必要があります。

アルゴリズムまたは振る舞い

1. PQExpBuffer を初期化し、pg_catalog.pg_database を対象とした SQL クエリを構築します。 2. サーバーバージョンに応じて、ロケールプロバイダー、ロケール、ICUルールなどの列を条件付きで追加します。 3. verbose が true の場合、pg_database_size や pg_tablespace 情報を結合します。 4. pattern が存在する場合、validateSQLNamePattern を呼び出し、WHERE 句を構築します。 5. PSQLexec を実行して結果を取得し、printQuery を用いて整形出力します。 6. 使用したバッファおよび結果セットを解放し、終了します。

呼び出し元と呼び出し先

  • src/bin/psql/command.c:2351
  • src/bin/psql/startup.c:339
  • **呼び出し先**:
  • initPQExpBuffer, printfPQExpBuffer, appendPQExpBuffer, appendPQExpBufferStr, termPQExpBuffer

  • PSQLexec

  • PQclear

関連データ構造

  • PGresult: クエリ結果の保持に使用。
  • PQExpBufferData: SQL クエリ構築用バッファ。
  • printQueryOpt: 出力オプションの保持に使用。

失敗モードと境界条件

  • validateSQLNamePattern が失敗した場合、バッファを解放して false を返します。
  • PSQLexec が NULL を返した場合、処理を中断して false を返します。
  • サーバーバージョンが古い場合、一部の列（Locale等）は NULL として出力されます。

保守担当者向けメモ

  • 新しいデータベース属性を追加する場合は、pset.sversion による条件分岐を確認してください。
  • verbose モード時のクエリ結合は pg_tablespace に依存しているため、カタログ構造の変更時には注意が必要です。
