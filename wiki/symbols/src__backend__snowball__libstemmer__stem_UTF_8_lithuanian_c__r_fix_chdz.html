r_fix_chdz

目的

リトアニア語のステミング処理において、語幹抽出後に発生する特定の綴り（chdz 等）の不整合を補正する。

シグネチャまたは形状

static int r_fix_chdz(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境構造体）
  • **出力**: 成功時に 1、処理対象外または失敗時に 0、内部エラー発生時に負の値を返す。

前提条件と不変条件

  • z->c（現在のカーソル位置）が有効な範囲内にあること。
  • SN_env 構造体が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c を z->ket に設定する。 2. カーソル位置の直前の文字が特定の条件（UTF-8コードポイント 141 または 190）を満たすか確認する。満たさない場合は終了する。 3. find_among_b を使用して、定義済みのテーブル a_3 に対して後方マッチングを行う。 4. マッチした場合、z->bra を現在のカーソル位置に設定し、マッチしたケースに応じて slice_from_s を呼び出し、文字列を置換する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_lithuanian.c 内のステミング処理フロー（800行目および812行目付近）。
  • **呼び出し先**: find_among_b（Snowballランタイム関数）、slice_from_s（Snowballランタイム関数）。

関連データ構造

  • struct SN_env: ステミング処理の環境状態を保持する構造体。
  • a_3: find_among_b で使用される接尾辞マッチング用のテーブル。

失敗モードと境界条件

  • カーソル位置が文字列の先頭に近い場合、またはマッチするパターンが存在しない場合は 0 を返す。
  • slice_from_s が失敗した場合（メモリ不足や不正な操作）、負の戻り値をそのまま伝播させる。
