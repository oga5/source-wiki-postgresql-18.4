DecodeSpecConfirm

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/decode.c:1215

目的

WALレコードから「Speculative Insertion（投機的挿入）」の確定イベントを抽出し、論理デコーディングの再順序付けバッファ（ReorderBuffer）へキューイングする。

シグネチャまたは形状

static void DecodeSpecConfirm(LogicalDecodingContext *ctx, XLogRecordBuffer *buf)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • buf: 解析対象のWALレコードおよびLSN情報を含むバッファ。
  • **出力**: なし（副作用として ctx->reorder に変更が追加される）。

前提条件と不変条件

  • buf->record が有効なWALレコードを指していること。
  • 処理対象のデータベースOIDが ctx->slot->data.database と一致する場合のみ処理が進行する。

アルゴリズムまたは振る舞い

1. **フィルタリング**:

  • レコードのブロックタグからデータベースOIDを取得し、現在のスロットのデータベースと一致しない場合は早期リターンする。
  • FilterByOrigin を呼び出し、レプリケーションオリジンによるフィルタリング条件に合致する場合は処理を中断する。

    • 2. **変更の作成**:

  • ReorderBufferAllocChange を使用して REORDER_BUFFER_CHANGE_INTERNAL_SPEC_CONFIRM 型の変更オブジェクトを確保する。
  • レコードからオリジンIDを取得し、ターゲットとなるリレーションのロケータ（RelFileLocator）をコピーする。

    • 3. **フラグ設定**: clear_toast_afterwards を true に設定し、TOASTデータのクリアを指示する。 4. **キューイング**: ReorderBufferQueueChange を呼び出し、トランザクションIDとLSNを紐付けてバッファへ登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c 内のデコード処理フロー（行533付近）。
  • **呼び出し先**:
  • XLogRecGetBlockTag, XLogRecGetOrigin, XLogRecGetXid (WAL解析用マクロ)
  • FilterByOrigin
  • ReorderBufferAllocChange
  • ReorderBufferQueueChange

関連データ構造

  • LogicalDecodingContext: 論理デコーディングの全体状態。
  • XLogRecordBuffer: 解析中のWALレコード情報。
  • ReorderBufferChange: 再順序付けバッファに格納される変更内容。
  • RelFileLocator: 対象リレーションの識別子。

失敗モードと境界条件

  • データベースOIDが一致しない場合、またはオリジンフィルタに該当する場合は、何も行わず正常終了する。

保守担当者向けメモ

  • 本関数は内部的な「Spec Confirm」イベントを扱う。論理デコーディングにおいて、投機的挿入が確定した際のTOASTデータ処理を正しく管理するために重要である。
  • clear_toast_afterwards の設定は、このイベントが確定した際に不要なTOASTデータをクリーンアップするための不変条件として機能する。
