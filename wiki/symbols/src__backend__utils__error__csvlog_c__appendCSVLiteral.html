appendCSVLiteral

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/csvlog.c:36

目的

文字列を CSV 形式のフィールドとして整形し、StringInfo バッファに追加します。特に、CSV ログ出力においてダブルクォーテーションのエスケープ処理を確実に行うために使用されます。

シグネチャまたは形状

static inline void appendCSVLiteral(StringInfo buf, const char *data)

入力と出力

  • **入力**:
  • buf: 文字列を格納する StringInfo 構造体へのポインタ。
  • data: CSV リテラルとして追加する対象の文字列（const char *）。
  • **出力**: なし（buf が指すバッファの内容が更新されます）。

前提条件と不変条件

  • data が NULL の場合、関数は何もせず即座に復帰します。
  • 出力される文字列は、常にダブルクォーテーション " で囲まれます。

アルゴリズムまたは振る舞い

1. data が NULL であれば終了します。 2. バッファの先頭に " を追加します。 3. data 内の各文字を走査し、文字が " であれば、CSV のエスケープルールに従い " を追加してダブルクォーテーションを重ねます。 4. その後、元の文字をバッファに追加します。 5. 最後にバッファの末尾に " を追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/error/csvlog.c 内の各箇所から、エラーメッセージ、詳細、ヒント、ユーザー名、データベース名などのログフィールドをバッファへ書き込むために呼び出されます。
  • **呼び出し先**: lib/stringinfo.h で定義されている appendStringInfoCharMacro を使用してバッファ操作を行います。

関連データ構造

  • StringInfo: ログメッセージを構築するための可変長文字列バッファ。
  • ErrorData: エラー情報が格納された構造体（csvlog.c 内でこの関数の引数として参照される各フィールドのソース）。

失敗モードと境界条件

  • data が空文字列の場合、"" として出力されます。
  • data 内にダブルクォーテーションが含まれる場合、正しく "" に変換されます。
  • メモリ不足等のバッファ操作に伴う失敗モードについては、StringInfo の実装に依存します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、csvlog.c 内でのみ使用されます。
  • CSV のエスケープルール（ダブルクォーテーションの二重化）を変更する場合は、本関数のロジックを修正する必要があります。
