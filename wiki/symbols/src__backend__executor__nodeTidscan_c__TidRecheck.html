TidRecheck

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/nodeTidscan.c:402

目的

EvalPlanQual（同時実行制御）において、指定されたタプルが TidScan ノードで対象としている TID（Tuple Identifier）リストに含まれているかを再検証する。

シグネチャまたは形状

bool TidRecheck(TidScanState *node, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • node: TidScanState 構造体へのポインタ。スキャン状態および TID リストを保持。
  • slot: 検証対象のタプルを格納する TupleTableSlot。
  • **出力**:
  • bool: タプルが対象 TID リストに含まれていれば true、そうでなければ false を返す。

前提条件と不変条件

  • node->tss_isCurrentOf が真の場合、常に true を返す。
  • node->tss_TidList が未初期化（NULL）の場合、内部で TidListEval(node) が呼び出され、TID リストが生成される。

アルゴリズムまたは振る舞い

1. tss_isCurrentOf フラグを確認し、真であれば即座に true を返す。 2. tss_TidList が NULL の場合、TidListEval を呼び出して TID リストを構築する。 3. slot->tts_tid（対象タプルの TID）をキーとして、bsearch を用いて tss_TidList 内をバイナリサーチする。 4. 比較には itemptr_comparator を使用する。 5. 検索結果が NULL でなければ true を、そうでなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内のコメントおよび参照情報から、EvalPlanQual 実行時の再検証メソッドとして利用されることが示唆されている（直接的な呼び出し元関数の明示的なリストは提供されていない）。
  • **呼び出し先**:
  • TidListEval: TID リストの構築が必要な場合に呼び出される。
  • bsearch: 標準ライブラリ関数。
  • itemptr_comparator: TID 比較用関数。

関連データ構造

  • TidScanState: スキャン状態を保持する構造体。
  • TupleTableSlot: タプルデータを保持する構造体。
  • ItemPointerData: TID を表すデータ構造。

失敗モードと境界条件

  • tss_TidList が空の場合、bsearch は NULL を返し、結果として false が返される。
  • slot 内の TID が無効な場合、検索結果は NULL となる。

保守担当者向けメモ

  • 本関数は ExecScanRecheckMtd インターフェースの一部として登録されている。
  • TidListEval の遅延評価により、初回呼び出し時にコストが発生する点に注意すること。
