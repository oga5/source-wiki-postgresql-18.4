pgstat_io_snapshot_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_io.c:308

目的

共有メモリ上のI/O統計情報をローカルのスナップショット領域へコピーし、統計ビュー（pg_stat_io）生成のための整合性のあるデータセットを作成します。

シグネチャまたは形状

void pgstat_io_snapshot_cb(void)

入力と出力

  • **入力**: なし（pgStatLocal.shmem->io に格納された共有メモリ上の統計情報を参照）。
  • **出力**: なし（pgStatLocal.snapshot.io 構造体および stat_reset_timestamp を更新）。

前提条件と不変条件

  • pgStatLocal.shmem が初期化されていること。
  • 各バックエンドタイプに対応する LWLock を使用して、共有メモリへのアクセスを排他制御すること。

アルゴリズムまたは振る舞い

1. BACKEND_NUM_TYPES 分のループを実行し、各バックエンドタイプごとの統計を処理します。 2. 各タイプに対応する LWLock を LW_SHARED モードで取得します。 3. ループの初回（i == 0）のみ、共有メモリ上の統計リセット時刻 (stat_reset_timestamp) をスナップショットへコピーします。 4. 共有メモリ上の PgStat_BktypeIO 構造体の内容を、構造体代入を用いてスナップショット領域へコピーします。 5. ロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:446（pgstat_snapshot_cb 構造体の一部として登録）。
  • **呼び出し先**: LWLockAcquire, LWLockRelease（共有メモリ保護のため）。

関連データ構造

  • PgStat_BktypeIO: バックエンドタイプごとのI/O統計を保持する構造体。
  • pgStatLocal: 統計サブシステムのローカル状態を保持するグローバル変数。

失敗モードと境界条件

  • 共有メモリが未初期化の場合、ポインタ参照によりクラッシュする可能性があります。
  • LWLock の取得に失敗した場合、デッドロックや待機状態が発生する可能性があります。

保守担当者向けメモ

  • 構造体代入 (*bktype_snap = *bktype_shstats) を使用しているため、PgStat_BktypeIO 構造体の定義変更時には注意が必要です。
  • 統計リセット時刻のコピーはインデックス 0 のロックに依存しているため、ロックの順序や範囲を変更する際は注意してください。
