SampleNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSamplescan.c:41

目的

TABLESAMPLE 句を用いたスキャン実行において、サンプリング手法に基づいた次のタプルを取得し、結果スロットへ格納して返します。

シグネチャまたは形状

static TupleTableSlot *SampleNext(SampleScanState *node)

入力と出力

  • **入力**: SampleScanState *node（サンプリングスキャンの実行状態を保持する構造体）
  • **出力**: TupleTableSlot *（取得されたタプルを格納したスロット、またはスキャン終了時の NULL）

前提条件と不変条件

  • node は有効な SampleScanState を指している必要があります。
  • 初回呼び出し時に node->begun が偽であれば、内部で tablesample_init が呼び出され、スキャンが初期化されます。

アルゴリズムまたは振る舞い

1. node->begun を確認し、偽であれば tablesample_init(node) を呼び出してサンプリングの初期化（パラメータ評価や BeginSampleScan の実行）を行います。 2. tablesample_getnext(node) を呼び出し、サンプリング手法（TsmRoutine）に従って次のタプルを取得します。 3. 取得したタプルを格納した TupleTableSlot を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecScan 経由で実行エンジンから呼び出されます（src/backend/executor/nodeSamplescan.c:84 にて ExecScanAccessMtd として登録）。
  • **呼び出し先**:
  • tablesample_init（初回スキャン時）
  • tablesample_getnext

関連データ構造

  • SampleScanState: スキャンノードの実行状態を保持する構造体。
  • TupleTableSlot: タプルを一時的に保持するための構造体。
  • TsmRoutine: サンプリング手法のインターフェース定義（src/include/access/tsmapi.h）。

失敗モードと境界条件

  • スキャン対象のテーブルから取得可能なタプルがなくなった場合、tablesample_getnext の振る舞いに従い NULL を返します。
  • 初回初期化時にサンプリング手法のパラメータ評価等でエラーが発生した場合、例外がスローされます。

保守担当者向けメモ

  • 本関数は ExecScan のコールバックとして機能します。スキャンロジックの変更は tablesample_getnext 側で行うのが適切です。
  • node->begun フラグによる初期化制御は、ReScan 発生時等の再実行フローと密接に関係しています。
