sync_pgdata

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:98

目的

PostgreSQLのデータディレクトリ全体を永続化（同期）し、クラッシュ後の整合性を保証します。

シグネチャまたは形状

void sync_pgdata(const char *pg_data,
                 int serverVersion,
                 DataDirSyncMethod sync_method,
                 bool sync_data_files)

入力と出力

  • pg_data: データディレクトリのパス。
  • serverVersion: サーバーバージョン（pg_xlog と pg_wal の命名規則判定に使用）。
  • sync_method: 同期手法（DATA_DIR_SYNC_METHOD_SYNCFS または DATA_DIR_SYNC_METHOD_FSYNC）。
  • sync_data_files: true の場合、テーブルスペース等のデータファイルも同期対象に含める。
  • 出力: なし（成功時）。エラー発生時はログ出力またはプロセス終了。

前提条件と不変条件

  • pg_wal がシンボリックリンクである場合、別途走査対象として処理される。
  • sync_method が SYNCFS の場合、環境が HAVE_SYNCFS をサポートしている必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: pg_wal および pg_tblspc のパスを構築し、pg_wal がシンボリックリンクかどうかを lstat で判定する。 2. **同期処理の分岐**:

  • DATA_DIR_SYNC_METHOD_SYNCFS: do_syncfs を使用。データディレクトリ、テーブルスペース、およびシンボリックリンクである pg_wal に対してファイルシステム単位の同期を実行する。
  • DATA_DIR_SYNC_METHOD_FSYNC: walkdir を使用。
  • オプションで pre_sync_fname を呼び出し、カーネルへ同期のヒントを通知する。
  • fsync_fname を呼び出し、各ファイルに対して fsync を実行する。sync_data_files が false の場合は base ディレクトリを除外する。

    • 3. **順序の保証**: pg_wal やテーブルスペースがシンボリックリンクの場合、メインの走査とは別に個別に同期処理を行う。

呼び出し元と呼び出し先

  • 呼び出し元: initdb, pg_basebackup, pg_checksums, pg_combinebackup, pg_rewind 等のフロントエンドツール。
  • 呼び出し先: do_syncfs, walkdir, pre_sync_fname, fsync_fname, lstat, opendir, readdir, closedir, psprintf, pfree。

関連データ構造

  • DataDirSyncMethod: 同期手法を定義する列挙型。
  • struct dirent: ディレクトリ走査に使用。
  • struct stat: ファイル属性の取得に使用。

失敗モードと境界条件

  • syncfs がサポートされていない環境で SYNCFS 手法を選択した場合、エラーを出力して終了する。
  • ディレクトリのオープンや読み取りに失敗した場合、エラーログを出力する。
  • pg_tblspc 内のディレクトリが複数回 fsync される可能性があるが、これは許容される設計となっている。

保守担当者向けメモ

  • 本関数は FRONTEND マクロが定義されている環境でのみ有効です。
  • pg_wal の命名変更（pg_xlog からの移行）を考慮したパス構築を行っています。
  • fsync の順序はクラッシュ耐性を維持するために重要であり、変更時はディレクトリのメタデータ永続化順序に注意してください。
