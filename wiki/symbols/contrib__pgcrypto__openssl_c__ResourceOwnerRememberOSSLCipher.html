ResourceOwnerRememberOSSLCipher

定義

  • 種別: static inline function
  • 定義箇所: contrib/pgcrypto/openssl.c:280

目的

OpenSSL の暗号化コンテキスト (OSSLCipher) を PostgreSQL の ResourceOwner に登録し、トランザクション終了時やエラー発生時に確実にリソースが解放されるよう管理する。

シグネチャまたは形状

static inline void ResourceOwnerRememberOSSLCipher(ResourceOwner owner, OSSLCipher *od)

入力と出力

  • owner: リソースを所有する ResourceOwner オブジェクト。
  • od: 登録対象の OSSLCipher 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • od は有効な OSSLCipher 構造体を指している必要がある。
  • osslcipher_resowner_desc がリソース記述子として定義されている必要がある。

アルゴリズムまたは振る舞い

内部で ResourceOwnerRemember を呼び出し、od を PointerGetDatum で変換して owner に紐付ける。これにより、PostgreSQL のリソース管理機構が当該コンテキストのライフサイクルを追跡可能となる。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/openssl.c:810 (暗号化コンテキストの初期化処理内)
  • 呼び出し先: ResourceOwnerRemember (PostgreSQL コアのメモリ管理関数)

関連データ構造

  • OSSLCipher: 暗号化/復号用のコンテキスト構造体。
  • ResourceOwner: PostgreSQL のリソース管理用オブジェクト。
  • osslcipher_resowner_desc: リソースの解放処理等を定義した記述子。

失敗モードと境界条件

  • owner が NULL の場合、ResourceOwnerRemember の挙動に依存する（通常はエラーとなる）。
  • メモリ不足等の理由で ResourceOwner への登録に失敗した場合、PostgreSQL のエラーハンドリング機構により例外が発生する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、openssl.c 内でのみ使用される。
  • 新たな暗号化コンテキストを生成する際は、必ず本関数を呼び出してリソースリークを防止すること。
  • 登録解除には対となる ResourceOwnerForgetOSSLCipher を使用すること。
