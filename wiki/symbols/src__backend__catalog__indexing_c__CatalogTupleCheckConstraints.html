CatalogTupleCheckConstraints

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/indexing.c:194

目的

システムカタログのタプルに対して、定義された制約（現在は NOT NULL 制約のみ）が満たされているかを検証する。

シグネチャまたは形状

static void CatalogTupleCheckConstraints(Relation heapRel, HeapTuple tup)

入力と出力

  • **入力**:
  • heapRel: 対象となるシステムカタログのリレーション記述子。
  • tup: 検証対象の HeapTuple。
  • **出力**: なし（検証失敗時は Assert により異常終了）。

前提条件と不変条件

  • システムカタログに対してのみ使用されることを想定している。
  • 現在、システムカタログでサポートされている制約は attnotnull（非NULL制約）のみである。

アルゴリズムまたは振る舞い

1. HeapTupleHasNulls(tup) を使用し、タプル内に NULL 値が含まれているかを確認する。 2. NULL 値が含まれる場合、リレーションのタプル記述子（TupleDesc）を取得する。 3. 全属性をループで走査し、各属性の attnotnull フラグと、タプルのビットマップ上の NULL 状態を比較する。 4. attnotnull が真かつ該当属性が NULL である場合、Assert により検証エラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/indexing.c 内の CatalogTupleInsert、CatalogTupleInsertWithInfo、CatalogTupleUpdate、CatalogTupleUpdateWithInfo から呼び出される。
  • **呼び出し先**:
  • HeapTupleHasNulls (マクロ)
  • RelationGetDescr (マクロ)
  • TupleDescAttr (マクロ)
  • att_isnull (マクロ)

関連データ構造

  • Relation: システムカタログのリレーション情報。
  • HeapTuple: 操作対象のタプルデータ。
  • TupleDesc: タプルの属性定義情報。

失敗モードと境界条件

  • **失敗モード**: attnotnull が設定されている属性に NULL 値が格納されている場合、Assert が失敗する。
  • **境界条件**: タプルに NULL 値が含まれていない場合、ループ処理はスキップされ、検証は行われない。

保守担当者向けメモ

  • 本関数はデバッグビルド時の整合性チェックを主目的としている。
  • 将来的に他のカタログ制約が追加された場合、本関数内で検証ロジックを拡張する必要がある。
