GinQualCounts

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/selfuncs.c:7977

目的

GIN（Generalized Inverted Index）インデックスにおけるクエリ条件の評価結果を保持し、選択率推定やコスト計算のための統計情報を集計するために使用されます。

シグネチャまたは形状

typedef struct
{
	bool		attHasFullScan[INDEX_MAX_KEYS];
	bool		attHasNormalScan[INDEX_MAX_KEYS];
	double		partialEntries;
	double		exactEntries;
	double		searchEntries;
	double		arrayScans;
} GinQualCounts;

入力と出力

本構造体は関数間でポインタとして受け渡され、集計結果を格納するコンテナとして機能します。

  • 入力: 統計計算を行う関数（src/backend/utils/adt/selfuncs.c 内の関連関数）が、この構造体のインスタンスへのポインタを受け取ります。
  • 出力: 各フィールドに計算されたスキャン回数やエントリ数が書き込まれます。

前提条件と不変条件

  • attHasFullScan および attHasNormalScan は、インデックスのキー数（INDEX_MAX_KEYS）に基づいた配列として管理されます。
  • partialEntries, exactEntries, searchEntries, arrayScans は、推定された回数やエントリ数を表す double 型の値です。

アルゴリズムまたは振る舞い

GIN インデックスのクエリ評価プロセスにおいて、各キーや条件がインデックススキャンに与える影響を定量化します。具体的には、インデックスのフルスキャンや通常スキャンの有無、および検索に伴うエントリの処理数を累積的に記録します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/selfuncs.c 内の複数の関数（行 7995, 8111, 8162, 8173, 8216, 8287）で、ローカル変数として宣言または引数として参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • INDEX_MAX_KEYS: インデックスの最大キー数を定義する定数。

失敗モードと境界条件

  • INDEX_MAX_KEYS を超えるインデックスキーに対するアクセスは、配列の境界外参照を引き起こす可能性があるため、呼び出し側のロジックで適切に制御されている必要があります。

保守担当者向けメモ

  • 本構造体は GIN インデックスのコスト推定ロジックに深く依存しています。統計情報の計算ロジックを変更する際は、selfuncs.c 内の関連する集計関数との整合性を確認してください。
  • 呼び出し元情報の詳細は、selfuncs.c 内の各参照箇所における具体的な関数呼び出しフローを参照してください。
