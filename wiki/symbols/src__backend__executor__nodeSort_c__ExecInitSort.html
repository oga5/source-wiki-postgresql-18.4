ExecInitSort

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSort.c:220

目的

Sort プランノードを実行するための SortState 構造体を初期化し、下位ノードの準備を行う。

シグネチャまたは形状

SortState *ExecInitSort(Sort *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Sort プランノード。
  • estate: クエリ実行状態を保持する EState。
  • eflags: 実行フラグ（EXEC_FLAG_REWIND 等）。
  • **出力**:
  • 初期化済みの SortState 構造体へのポインタ。

前提条件と不変条件

  • Sort ノードは ExecQual や ExecProject を呼び出さないため、ExprContext の初期化は行わない。
  • 呼び出し元から渡された eflags から REWIND, BACKWARD, MARK/RESTORE フラグを除去した上で、下位ノードの初期化を行う。

アルゴリズムまたは振る舞い

1. SortState 構造体を割り当て、ExecProcNode フィールドに ExecSort を設定する。 2. eflags に基づき、randomAccess フラグを設定する。 3. tuplesortstate を NULL に初期化し、ソート完了フラグ sort_Done を false に設定する。 4. ExecInitNode を呼び出し、下位ノードを初期化する。 5. ExecCreateScanSlotFromOuterPlan を使用してスキャンスロットを作成する。 6. ExecInitResultTupleSlotTL を使用して戻り値用スロットを初期化する。 7. 下位ノードのタプル記述子（outerTupDesc）を確認し、カラム数が 1 の場合は datumSort を true に、それ以外は false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:321（ExecInitNode 経由での呼び出しが確認されている）。
  • **呼び出し先**:
  • makeNode
  • ExecInitNode
  • ExecCreateScanSlotFromOuterPlan
  • ExecInitResultTupleSlotTL
  • ExecGetResultType
  • outerPlan (マクロ)
  • outerPlanState (マクロ)

関連データ構造

  • SortState: 本関数が初期化する主要な構造体。
  • Sort: プランノード。
  • EState: 実行状態。
  • TupleDesc: タプルの構造情報。

失敗モードと境界条件

  • 下位ノードの初期化失敗時にエラーが発生する可能性がある。
  • outerTupDesc が NULL の場合、ExecGetResultType の挙動に依存する（本ソース内では明示的なチェックなし）。

保守担当者向けメモ

  • 本ノードはブロッキング操作であり、全入力を読み込むまで最初のタプルを返さない。
  • datumSort は単一カラムのソート時にのみ有効であり、パフォーマンス最適化の鍵となる。
  • ExecSort 内での CHECK_FOR_INTERRUPTS() の重要性を考慮し、大規模ソート時の割り込み処理が適切に行われるよう設計されている。
