GISTTYPE

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int.h:93

目的

PostgreSQL の GiST インデックスにおいて、インデックスキーを保持するための可変長構造体です。

シグネチャまたは形状

typedef struct {
    int32       vl_len_;        /* varlena header */
    int32       flag;
    char        data[FLEXIBLE_ARRAY_MEMBER];
} GISTTYPE;

入力と出力

  • **入力**: FLEXIBLE_ARRAY_MEMBER を含むため、メモリ確保時には CALCGTSIZE や COMPUTESIZE マクロを用いて必要なサイズを算出する必要があります。
  • **出力**: インデックスのキーとして格納されるデータ構造体。

前提条件と不変条件

  • vl_len_ は varlena ヘッダーであるため、直接操作してはなりません。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズは固定ではありません。
  • 配列内の NULL 値はサポートされていません。

アルゴリズムまたは振る舞い

GiST インデックスにおけるキーの格納および管理を行います。flag フィールドによりインデックスの状態を制御し、data フィールドに実際のインデックスデータを保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_gist.c において、GISTTYPE 型へのキャストや、ghstore_alloc を介したメモリ確保、hemdist や unionkey などの関数内での引数として参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • BITVECP: シグネチャ（ビットベクトル）を扱うためのポインタ型。
  • QUERYTYPE: ブール検索クエリを保持する構造体。

失敗モードと境界条件

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、誤ったサイズ計算でメモリを確保するとバッファオーバーフローやメモリ破損を引き起こす可能性があります。
  • CHECKARRVALID マクロによる検証を通過しないデータは、本構造体で扱う前にフィルタリングする必要があります。

保守担当者向けメモ

  • vl_len_ フィールドには直接触れないでください。
  • メモリ確保時には必ず CALCGTSIZE や COMPUTESIZE マクロを使用してください。
  • hstore モジュールからも参照されているため、変更時は影響範囲に注意してください。
