目的

ECPG（Embedded SQL for C）プリプロセッサによって生成されたコードの回帰テストとして、カーソル操作（宣言、オープン、フェッチ）の正当性を検証する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は受け取らない）。
  • 出力: 成功時に 0 を返し、エラー発生時に exit(1) を呼び出す。標準出力にフェッチした結果を出力する。

前提条件と不変条件

  • 実行環境において ecpg1_regression および ecpg2_regression データベースへの接続が可能であること。
  • 各SQL操作の直後に sqlca.sqlcode を確認し、負の値であれば異常終了する不変条件を持つ。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、2つのデータベースへ接続する。 2. **環境構築**: test1 および test2 データベースにテーブル t1 を作成し、初期データを投入する。 3. **トランザクション**: 各接続で commit を実行する。 4. **カーソル操作**:

  • 動的カーソル名（curname1）を使用してカーソルを宣言およびオープンする。
  • FETCH 文を実行し、結果をC言語の変数 id および t に格納する。
  • 取得したデータを printf で出力する。

    • 5. **終了処理**: 接続を維持したまま終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（エントリポイント）。
  • 呼び出し先:
  • ECPGconnect: データベース接続。
  • ECPGdo: SQL文の実行（set, create, insert, declare, open, fetch）。
  • ECPGtrans: トランザクション制御。
  • ECPGset_var: カーソル名のバインド。
  • printf: 結果の出力。
  • exit: エラー時の終了。

関連データ構造

  • sqlca: SQL通信領域（エラーコードの確認に使用）。
  • varchar_1: カーソル名を保持するための構造体。
  • id, t: フェッチ結果を格納するC言語変数。

失敗モードと境界条件

  • データベース接続失敗時、またはSQL実行時に sqlca.sqlcode < 0 となった場合、即座に exit(1) で終了する。
  • カーソル操作において、動的なカーソル名が正しく解決されない場合、エラーが発生する。

保守担当者向けメモ

  • 本関数は自動生成されたコードであるため、直接編集しないこと。修正が必要な場合は対応する .pgc ファイルを編集し、プリプロセッサを再実行すること。
  • ECPGdebug(1, stderr) が有効であるため、実行時の詳細なログは標準エラー出力から確認可能。
