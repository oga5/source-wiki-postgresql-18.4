build_test_match_result

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_regex/test_regex.c:691

目的

正規表現のマッチング結果を SQL で利用可能な text[] 型の配列として構築する。

シグネチャまたは形状

static ArrayType *build_test_match_result(test_regex_ctx *matchctx)

入力と出力

  • **入力**: test_regex_ctx *matchctx（マッチング状態、パターン情報、および結果格納用バッファを含むコンテキスト）
  • **出力**: ArrayType *（構築されたテキスト配列）

前提条件と不変条件

  • matchctx 内の match_locs には、現在のマッチング位置情報が格納されている必要がある。
  • matchctx->elems および matchctx->nulls は、結果配列を構築するための十分な領域が確保されている必要がある。
  • indices フラグが有効な場合、Tcl 互換のインデックス形式（so eo-1）で出力される。

アルゴリズムまたは振る舞い

1. matchctx->next_match を基に、現在のマッチング対象となる match_locs のオフセットを算出する。 2. 各パターンについてループし、以下のいずれかの形式で elems に格納する:

  • indices が真の場合: マッチング開始位置と終了位置（so と eo-1）を文字列化。
  • マッチング失敗（so < 0）の場合: nulls を真に設定。
  • buf が存在する場合: pg_wchar2mb_with_len を使用してマルチバイト文字列へ変換。
  • それ以外の場合: text_substr を呼び出し、元の文字列から部分文字列を抽出。

    • 3. REG_EXPECT フラグが有効な場合、追加で rm_extend の詳細情報を配列の末尾に追加する。 4. construct_md_array を使用して、蓄積された elems と nulls から text[] 型の配列を生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_regex/test_regex.c 内の test_regex 関数（SRF の反復処理フェーズ）。
  • **呼び出し先**: cstring_to_text, pg_wchar2mb_with_len, cstring_to_text_with_len, DirectFunctionCall3（text_substr を呼び出し）, construct_md_array。

関連データ構造

  • test_regex_ctx: マッチングのコンテキストおよび結果保持用構造体。

失敗モードと境界条件

  • len が matchctx->conv_bufsiz を超える場合、Assert により異常終了する可能性がある。
  • indices モードにおいて、so < 0 の場合は eo をそのまま出力する（Tcl 互換性のための特殊処理）。

保守担当者向けメモ

  • 本関数は test_regex 関数から反復的に呼び出されるため、メモリ管理は matchctx に依存している。
  • text 型の配列構築に関するハードコードされた前提があるため、型定義の変更時には注意が必要。
