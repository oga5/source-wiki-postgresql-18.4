PgStat_ArchiverStats

定義

  • 種別: struct
  • 定義箇所: src/include/pgstat.h:217

目的

WAL アーカイバプロセスの実行状況に関する統計情報を保持するための構造体です。成功・失敗したアーカイブ処理の回数、対象となった WAL ファイル名、およびタイムスタンプを記録します。

シグネチャまたは形状

typedef struct PgStat_ArchiverStats
{
    PgStat_Counter archived_count;
    char           last_archived_wal[MAX_XFN_CHARS + 1];
    TimestampTz    last_archived_timestamp;
    PgStat_Counter failed_count;
    char           last_failed_wal[MAX_XFN_CHARS + 1];
    TimestampTz    last_failed_timestamp;
    TimestampTz    stat_reset_timestamp;
} PgStat_ArchiverStats;

入力と出力

  • **入力**: アーカイバプロセスによるアーカイブ成功時および失敗時のイベント情報。
  • **出力**: pgstat_fetch_stat_archiver() を通じて、統計ビュー（pg_stat_archiver 等）へ現在の統計状態を提供します。

前提条件と不変条件

  • archived_count および failed_count は PgStat_Counter（int64）型であり、累積的に増加します。
  • last_archived_wal および last_failed_wal は MAX_XFN_CHARS + 1 のサイズを持ち、WAL ファイル名を格納します。
  • stat_reset_timestamp は統計が最後にリセットされた時刻を保持します。

アルゴリズムまたは振る舞い

1. アーカイブ成功時、archived_count がインクリメントされ、last_archived_wal にファイル名が、last_archived_timestamp に現在時刻が記録されます。 2. アーカイブ失敗時、failed_count がインクリメントされ、last_failed_wal にファイル名が、last_failed_timestamp に現在時刻が記録されます。 3. 統計情報の取得要求があった場合、共有メモリ上のデータが読み取られ、ビューを通じてユーザーに公開されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat_archiver.c: 統計の更新およびスナップショット取得に使用。
  • src/backend/utils/adt/pgstatfuncs.c: SQL インターフェース経由での統計取得に使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PgStat_Counter: カウンタに使用される 64 ビット整数型。
  • TimestampTz: 時刻管理に使用される型。

失敗モードと境界条件

  • 統計リセット操作が行われた場合、各カウンタは 0 に、タイムスタンプはリセット時刻に更新されます。
  • WAL ファイル名が MAX_XFN_CHARS を超えることはありませんが、文字列バッファの境界には注意が必要です。

保守担当者向けメモ

  • 本構造体のメンバを変更した場合は、統計ファイルの互換性を維持するため PGSTAT_FILE_FORMAT_ID の更新を検討してください。
  • 共有メモリ上の統計データとローカルスナップショットの間でデータが同期される仕組み（pgstat_archiver.c 内のロジック）を理解した上で修正を行ってください。
