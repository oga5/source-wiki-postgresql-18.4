Pg_magic_struct

定義

  • 種別: struct
  • 定義箇所: src/include/fmgr.h:477

目的

動的にロードされる拡張モジュールと PostgreSQL バックエンド間での ABI（Application Binary Interface）およびバージョンの整合性を検証するためのメタデータ構造体です。

シグネチャまたは形状

typedef struct
{
	int			len;			/* sizeof(this struct) */
	Pg_abi_values abi_fields;	/* ABI 関連フィールド */
	const char *name;			/* モジュール名（任意） */
	const char *version;		/* モジュールバージョン（任意） */
} Pg_magic_struct;

入力と出力

  • **入力**: PG_MODULE_MAGIC マクロを通じて、コンパイル時に構造体の各フィールドが初期化されます。
  • **出力**: PGModuleMagicFunction 型の関数が、この構造体へのポインタを返します。

前提条件と不変条件

  • len フィールドには、必ず sizeof(Pg_magic_struct) が格納されている必要があります。
  • 拡張モジュールは、バックエンドが期待する ABI 形式と一致している必要があります。

アルゴリズムまたは振る舞い

1. 拡張モジュールがロードされる際、バックエンドは PG_MAGIC_FUNCTION_NAME で定義された関数を呼び出し、Pg_magic_struct を取得します。 2. バックエンドは取得した構造体の len が自身の期待するサイズと一致するかを確認します。 3. abi_fields を比較し、バックエンドとモジュール間で ABI の互換性があるかを検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/fmgr/dfmgr.c 内の動的ライブラリロード処理において、magic_func を介して呼び出されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出しません。

関連データ構造

  • Pg_abi_values: ABI 互換性を判定するためのフィールド群。
  • PGModuleMagicFunction: Pg_magic_struct を返す関数の型定義。

失敗モードと境界条件

  • len が sizeof(Pg_magic_struct) と一致しない場合、ABI 互換性エラーとしてロードが拒否されます。
  • abi_fields がバックエンドの期待値と一致しない場合、ロード時に不整合が検出されます。

保守担当者向けメモ

  • name および version フィールドは、PG_MODULE_MAGIC_EXT マクロを使用しない限りゼロ（NULL）で初期化されます。
  • 構造体のサイズやフィールド定義を変更する場合は、ABI 互換性に影響するため慎重に行う必要があります。
