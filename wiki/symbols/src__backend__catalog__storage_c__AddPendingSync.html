AddPendingSync

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/storage.c:85

目的

トランザクション終了時に同期（fsync）が必要なリレーションを追跡するための管理リストに、指定されたリレーションを追加します。

シグネチャまたは形状

static void AddPendingSync(const RelFileLocator *rlocator)

入力と出力

  • **入力**: rlocator (同期対象となるリレーションの物理的な位置情報)
  • **出力**: なし（副作用として pendingSyncHash へのエントリ追加）

前提条件と不変条件

  • pendingSyncHash が未初期化の場合は、TopTransactionContext をメモリコンテキストとしてハッシュテーブルが作成されます。
  • 指定された rlocator が既にハッシュテーブル内に存在しないこと（Assert(!found) により保証）。

アルゴリズムまたは振る舞い

1. pendingSyncHash が存在しない場合、hash_create を呼び出して初期化します。 2. hash_search を使用し、rlocator をキーとしてハッシュテーブルにエントリを挿入します。 3. 挿入されたエントリの is_truncated フラグを false に初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/storage.c 内の複数箇所（RelationCreateStorage 等から呼び出されていることが示唆されています）。
  • **呼び出し先**:
  • hash_create (ハッシュテーブル初期化)
  • hash_search (エントリの挿入)

関連データ構造

  • RelFileLocator: リレーションの物理的な識別子。
  • PendingRelSync: 同期待ちリレーションの状態を保持する構造体。
  • pendingSyncHash: 同期待ちリレーションを管理するハッシュテーブル。

失敗モードと境界条件

  • pendingSyncHash への挿入時に既にエントリが存在する場合、Assert によりプログラムが停止します。
  • メモリ不足時には hash_create または hash_search がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は static であり、src/backend/catalog/storage.c 内でのみ使用されます。
  • is_truncated フラグの初期値は false です。このフラグが後にどのように更新・参照されるかについては、関連する smgrDoPendingSyncs 等のロジックを確認してください。
