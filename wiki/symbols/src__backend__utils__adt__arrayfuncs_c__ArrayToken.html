ArrayToken

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:57

目的

配列のテキスト表現を解析する際、入力文字列から抽出される構成要素のトークン種別を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
    ATOK_LEVEL_START,  /* 配列の開始（'{'） */
    ATOK_LEVEL_END,    /* 配列の終了（'}'） */
    ATOK_DELIM,        /* 要素間の区切り文字 */
    ATOK_ELEM,         /* 通常の要素 */
    ATOK_ELEM_NULL,    /* NULL要素 */
    ATOK_ERROR,        /* 解析エラー */
} ArrayToken;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • ReadArrayToken 関数が入力文字列をスキャンし、この列挙値のいずれかを返却することで、配列解析の制御フローを決定します。

アルゴリズムまたは振る舞い

  • 配列の構文解析において、現在のスキャン位置がどのトークンに該当するかを分類します。
  • ATOK_LEVEL_START および ATOK_LEVEL_END はネスト構造の深さを管理するために使用されます。
  • ATOK_ELEM は実際のデータ値の読み取りを、ATOK_ELEM_NULL はNULL値の処理をトリガーします。

呼び出し元と呼び出し先

  • 呼び出し元: ReadArrayToken 関数（src/backend/utils/adt/arrayfuncs.c:104）がこの型を戻り値として返します。
  • 呼び出し先: 本シンボルは列挙型であるため、直接の呼び出し先はありません。

関連データ構造

  • ReadArrayToken: この列挙型を生成する解析関数。

失敗モードと境界条件

  • ATOK_ERROR: 解析中に予期しない文字や構文違反が検出された場合に返されます。

保守担当者向けメモ

  • 新しい配列の構文要素を追加する必要がある場合は、この列挙型を拡張し、ReadArrayToken 内の解析ロジックを更新してください。
