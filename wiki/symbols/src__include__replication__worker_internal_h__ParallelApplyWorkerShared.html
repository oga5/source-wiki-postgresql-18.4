ParallelApplyWorkerShared

定義

  • 種別: struct
  • 定義箇所: src/include/replication/worker_internal.h:138

目的

論理レプリケーションにおけるリーダーワーカーと並列適用ワーカー間で、トランザクションのコミット順序制御、ストリーミング状態の同期、およびファイルセットの共有を行うための共有メモリ構造体です。

シグネチャまたは形状

typedef struct ParallelApplyWorkerShared
{
    slock_t        mutex;
    TransactionId  xid;
    ParallelTransState xact_state;
    uint16         logicalrep_worker_generation;
    int            logicalrep_worker_slot_no;
    pg_atomic_uint32 pending_stream_count;
    XLogRecPtr     last_commit_end;
    PartialFileSetState fileset_state;
    FileSet        fileset;
} ParallelApplyWorkerShared;

入力と出力

  • **入力**: リーダーワーカーによるトランザクション状態の更新、コミット終了位置（last_commit_end）の書き込み、およびファイルセットの共有。
  • **出力**: 並列適用ワーカーによる xact_state の PARALLEL_TRANS_FINISHED への遷移、および pending_stream_count の監視。

前提条件と不変条件

  • xact_state は、トランザクション終了コマンド（STREAM_COMMIT等）の処理において、リーダーワーカーが並列ワーカーの完了を待機するための同期点として機能します。
  • fileset は、共有メモリ解放後も存続する必要があるため、SharedFileSet ではなく FileSet が使用されます。

アルゴリズムまたは振る舞い

1. **コミット順序制御**: 並列適用ワーカーはトランザクション終了コマンドを処理後、xact_state を PARALLEL_TRANS_FINISHED に設定します。リーダーワーカーはこの状態になるまで待機します。 2. **ストリーミング管理**: pending_stream_count を用いて、キュー内に未処理のストリーミングブロックが存在するかを並列ワーカーが確認します。 3. **シリアライズ移行**: PARTIAL_SERIALIZE モード移行時、リーダーワーカーは変更をファイルに書き出し、fileset を介して並列ワーカーと共有します。並列ワーカーはその後、スプールされたメッセージを適用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/applyparallelworker.c 内の各関数（pa_get_xact_state, pa_set_xact_state, pa_set_fileset_state 等）が本構造体を操作します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • ParallelTransState: トランザクションの実行状態を管理する列挙型。
  • PartialFileSetState: ファイルセットのシリアライズ状態を管理する列挙型。
  • FileSet: ストリーミングトランザクションの永続化に使用されるファイル管理構造体。
  • LogicalRepWorker: 関連するワーカーのスロット情報。

失敗モードと境界条件

  • fileset_state の遷移が不適切な場合、リーダーと並列ワーカー間の同期が崩れ、コミット順序の整合性が損なわれる可能性があります。
  • pending_stream_count の不整合は、並列ワーカーの待機状態に影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体はリーダーと並列ワーカー間の複雑な同期を担うため、フィールドの追加や変更を行う際は、コミット順序の整合性に細心の注意を払ってください。
  • fileset の生存期間は共有メモリの解放後も続く設計であるため、メモリ管理のライフサイクルを誤らないようにしてください。
