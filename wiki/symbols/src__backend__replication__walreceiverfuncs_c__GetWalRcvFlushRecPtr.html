GetWalRcvFlushRecPtr

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiverfuncs.c:331

目的

共有メモリ上の WalRcv 構造体から、WALレシーバーが現在までに受信し、ディスクへフラッシュした最新のWAL位置（LSN）、および関連するタイムラインIDとチャンク開始位置を取得します。

シグネチャまたは形状

XLogRecPtr GetWalRcvFlushRecPtr(XLogRecPtr *latestChunkStart, TimeLineID *receiveTLI)

入力と出力

  • **入力**:
  • latestChunkStart: 最後に受信したチャンクの開始位置を格納するポインタ（不要な場合は NULL を指定可能）。
  • receiveTLI: 受信したWALのタイムラインIDを格納するポインタ（不要な場合は NULL を指定可能）。
  • **出力**:
  • 戻り値: XLogRecPtr 型のフラッシュ済み最新WAL位置。

前提条件と不変条件

  • 共有メモリ上の WalRcv 構造体が初期化されている必要があります。
  • 内部で WalRcv->mutex スピンロックを使用するため、呼び出し元はスピンロックの保持状態を考慮する必要はありません。

アルゴリズムまたは振る舞い

1. WalRcv 共有メモリ構造体へのポインタを取得します。 2. SpinLockAcquire を用いて walrcv->mutex をロックします。 3. 共有メモリから flushedUpto を読み取り、戻り値として保持します。 4. latestChunkStart が NULL でない場合、walrcv->latestChunkStart をコピーします。 5. receiveTLI が NULL でない場合、walrcv->receivedTLI をコピーします。 6. SpinLockRelease を用いてロックを解放します。 7. 取得した flushedUpto を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogfuncs.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/replication/walsender.c
  • **呼び出し先**:
  • SpinLockAcquire
  • SpinLockRelease

関連データ構造

  • WalRcvData (共有メモリ上のグローバル構造体 WalRcv)
  • XLogRecPtr
  • TimeLineID

失敗モードと境界条件

  • 共有メモリが初期化されていない状態で呼び出された場合、未定義の動作を引き起こす可能性があります。
  • latestChunkStart や receiveTLI に無効なポインタを渡した場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は共有メモリへの安全なアクセスを保証するため、必ずスピンロックを介して値を読み取ります。
  • 統計情報の取得やリカバリ進捗の監視において頻繁に呼び出されるため、ロックの競合がパフォーマンスに影響を与える可能性があります。
