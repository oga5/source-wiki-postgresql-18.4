pgwin32_fopen

定義

  • 種別: function
  • 定義箇所: src/port/open.c:204

目的

Windows 環境において、標準の fopen() を代替し、POSIX 互換のファイルオープン操作を提供します。

シグネチャまたは形状

FILE *pgwin32_fopen(const char *fileName, const char *mode)

入力と出力

  • **入力**:
  • fileName: 開く対象のファイルパス（文字列）。
  • mode: ファイルオープンモード（文字列、r, w, a, +, b, t 等）。
  • **出力**: 成功時は FILE* を返し、失敗時は NULL を返します。

前提条件と不変条件

  • 本関数は WIN32 マクロが定義されている環境でのみ有効です。
  • src/include/port.h により、fopen は本関数へマクロ置換されます。

アルゴリズムまたは振る舞い

1. **モード解析**: 引数 mode を解析し、対応する POSIX オープンフラグ（O_RDONLY, O_WRONLY, O_RDWR, O_CREAT, O_TRUNC, O_APPEND, O_BINARY, O_TEXT）を構築します。 2. **ファイルオープン**: 構築したフラグを用いて pgwin32_open を呼び出し、ファイル記述子（fd）を取得します。 3. **エラーハンドリング**: pgwin32_open が -1 を返した場合は NULL を返します。 4. **ストリーム生成**: 成功した場合、_fdopen を呼び出してファイル記述子から FILE* ストリームを生成し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port.h にて fopen として定義されており、PostgreSQL のコアおよびフロントエンドツールから広く参照されます。
  • **呼び出し先**:
  • pgwin32_open（ファイル記述子の取得）
  • _fdopen（C ランタイムのストリーム生成）

関連データ構造

  • FILE（標準 C ライブラリのストリーム構造体）

失敗モードと境界条件

  • pgwin32_open が失敗した場合（ファイルが存在しない、権限不足、共有違反など）、本関数も NULL を返します。
  • 共有違反が発生した場合、pgwin32_open 内部の再試行ロジックにより最大 30 秒間待機します。

保守担当者向けメモ

  • 本関数は _fdopen を使用して FILE* を生成するため、Windows の C ランタイム（CRT）の制約を受けます。
  • O_CLOEXEC のサポートは pgwin32_open 側で処理されます。
  • STATUS_DELETE_PENDING などの Windows 特有のステータスコードは、pgwin32_open 経由で適切に処理される設計となっています。
