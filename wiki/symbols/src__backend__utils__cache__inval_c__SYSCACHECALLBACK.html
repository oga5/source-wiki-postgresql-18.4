SYSCACHECALLBACK

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/inval.c:276

目的

システムキャッシュの無効化イベントが発生した際に実行されるコールバック関数とその引数、および同一キャッシュに対する複数のコールバックを連結するためのリンク情報を保持する構造体です。

シグネチャまたは形状

struct SYSCACHECALLBACK {
    int16       id;         /* キャッシュ番号 */
    int16       link;       /* 同一キャッシュに対する次コールバックのインデックス+1 */
    SyscacheCallbackFunction function;
    Datum       arg;
};

入力と出力

  • 入力: syscache_callback_list 配列の要素として静的に確保されます。
  • 出力: キャッシュ無効化イベント発生時に、登録された function が arg を引数として呼び出されます。

前提条件と不変条件

  • syscache_callback_list 配列のサイズは MAX_SYSCACHE_CALLBACKS によって制限されます。
  • link フィールドは、同一キャッシュIDを持つコールバックのリストを辿るためのインデックスとして機能します。

アルゴリズムまたは振る舞い

  • キャッシュ無効化イベントが発生すると、対応するキャッシュIDに関連付けられたコールバック関数が順次実行されます。
  • link フィールドにより、単一のキャッシュIDに対して複数のコールバックを登録・管理することが可能です。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/inval.c 内の関数（795行目および1908行目付近）で、syscache_callback_list を走査する際に参照されています。
  • 呼び出し先: function フィールドに格納された SyscacheCallbackFunction 型の関数が実行されます。

関連データ構造

  • syscache_callback_list: SYSCACHECALLBACK 型のグローバルな静的配列。
  • SyscacheCallbackFunction: コールバック関数の型定義。

失敗モードと境界条件

  • MAX_SYSCACHE_CALLBACKS を超えるコールバックの登録は、配列の境界外アクセスを引き起こす可能性があるため、登録数には上限があります。

保守担当者向けメモ

  • 本構造体は inval.c 内で管理されるコールバック機構の基盤です。コールバックの追加や削除を行う際は、link によるリスト構造の整合性が保たれていることを確認してください。
