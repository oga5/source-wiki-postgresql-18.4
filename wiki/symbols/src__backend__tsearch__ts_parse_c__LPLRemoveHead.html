LPLRemoveHead

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_parse.c:85

目的

ListParsedLex 構造体から先頭の要素（head）を取り出し、リストから切り離して返します。

シグネチャまたは形状

static ParsedLex *LPLRemoveHead(ListParsedLex *list)

入力と出力

  • **入力**: ListParsedLex *list（操作対象のリスト構造体へのポインタ）
  • **出力**: ParsedLex *（リストから取り外された先頭要素へのポインタ。リストが空の場合は NULL）

前提条件と不変条件

  • list が NULL でないこと。
  • リストの head が NULL になった場合、tail も自動的に NULL に更新される不変条件が維持されます。

アルゴリズムまたは振る舞い

1. 現在の list->head を戻り値用の変数 res に保持します。 2. list->head が存在する場合、list->head を次の要素（next）へ進めます。 3. 更新後の list->head が NULL（リストが空になった）であれば、list->tail を NULL に設定します。 4. 保持していた元の先頭要素 res を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/ts_parse.c 内の LexizeExec 関数から呼び出されています。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • ListParsedLex: トークンリストを管理する構造体。
  • ParsedLex: リスト内の各要素を構成するデータ構造。

失敗モードと境界条件

  • **空リスト**: list->head が NULL の場合、関数は単に NULL を返し、リスト構造体に変更を加えません。

保守担当者向けメモ

  • 本関数は LexizeExec におけるトークン処理のパイプライン（towork リストから waste リストへの移動など）で頻繁に使用されます。
  • リストのポインタ操作のみを行う軽量な関数ですが、呼び出し側で list が正しく初期化されていることを確認してください。
