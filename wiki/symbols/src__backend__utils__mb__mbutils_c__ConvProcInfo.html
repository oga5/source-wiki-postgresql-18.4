ConvProcInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mb/mbutils.c:55

目的

サーバーエンコーディングとクライアントエンコーディング間での相互変換を行う関数（FmgrInfo）をキャッシュし、変換処理のオーバーヘッドを削減するために使用されます。

シグネチャまたは形状

typedef struct ConvProcInfo
{
    int         s_encoding;     /* サーバーエンコーディングID */
    int         c_encoding;     /* クライアントエンコーディングID */
    FmgrInfo    to_server_info; /* クライアントからサーバーへの変換関数情報 */
    FmgrInfo    to_client_info; /* サーバーからクライアントへの変換関数情報 */
} ConvProcInfo;

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドはエンコーディングの識別子（int）および関数マネージャのキャッシュ情報（FmgrInfo）を保持します。

前提条件と不変条件

  • TopMemoryContext 上に割り当てられ、セッション終了まで保持されることが想定されています。
  • 変換関数情報は fmgr を介してロードされるため、カタログ検索が可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. エンコーディング変換要求が発生した際、既存の ConvProcList 内を走査し、一致する s_encoding および c_encoding を持つ ConvProcInfo を検索します。 2. キャッシュが存在しない場合、新たに ConvProcInfo をメモリ確保し、FindDefaultConversionProc を用いて変換関数をロードしてキャッシュに追加します。 3. 以降の変換処理では、キャッシュされた FmgrInfo を使用して OidFunctionCall6 等で変換を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mb/mbutils.c 内のエンコーディング変換管理ロジック（pg_client_to_server 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出しません。保持する to_server_info および to_client_info が fmgr 経由で実際の変換関数を呼び出すために使用されます。

関連データ構造

  • FmgrInfo: 関数マネージャが管理する関数呼び出し情報。
  • ConvProcList: ConvProcInfo を要素として保持する List。

失敗モードと境界条件

  • メモリ不足時: MemoryContextAlloc が失敗する可能性があります。
  • 変換関数未定義時: FindDefaultConversionProc が適切な関数を見つけられない場合、変換処理がエラーとなります。

保守担当者向けメモ

  • 本構造体は TopMemoryContext に配置されるため、キャッシュの追加時にはメモリリークが発生しないよう注意してください。
  • トランザクション外でのカタログ検索を伴う初期化には制限があるため、呼び出しタイミングに注意が必要です。
