AttrMissing

定義

  • 種別: struct
  • 定義箇所: src/include/access/tupdesc_details.h:22

目的

テーブル定義の変更（ALTER TABLE ADD COLUMN 等）により、物理タプル上に存在しない属性に対して、読み取り時に供給すべきデフォルト値を保持するために使用されます。

シグネチャまたは形状

typedef struct AttrMissing
{
	bool		am_present;		/* true if non-NULL missing value exists */
	Datum		am_value;		/* value when attribute is missing */
} AttrMissing;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。

  • 入力: 属性が欠落している際のデフォルト値（am_value）およびその有効性（am_present）。
  • 出力: タプルスキャンやデータ変換処理において、欠落属性の代替値として利用されます。

前提条件と不変条件

  • am_present が false の場合、属性は欠落しており、かつデフォルト値も存在しない（NULLとして扱われる）ことを示します。
  • am_present が true の場合、am_value に有効なデフォルト値が格納されています。

アルゴリズムまたは振る舞い

タプルスキャンや変換処理において、特定の属性が物理タプル上に存在しないと判断された際、この構造体を参照してデフォルト値を供給します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/access/common/heaptuple.c
  • src/backend/access/common/tupdesc.c
  • src/backend/executor/execTuples.c
  • src/backend/utils/cache/relcache.c
  • src/include/access/tupdesc.h
  • 呼び出し先:
  • 本構造体はデータ構造定義であり、関数を呼び出すことはありません。

関連データ構造

  • TupleDesc: タプル記述子。tupdesc.h で定義され、AttrMissing の配列を保持する可能性があります。
  • Datum: PostgreSQL における汎用的なデータ値表現。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、am_value に格納される Datum がメモリ管理上の制約（ポインタ型の場合の生存期間など）に違反している場合、データアクセス時に問題が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は tupdesc.h と密接に関連しますが、循環参照を避けるために分離されています。インクルードする際は依存関係の連鎖に注意してください。
  • ALTER TABLE 後の古いタプル読み取りにおいて不可欠な構造体です。変更を加える際は、タプル変換処理への影響を考慮してください。
