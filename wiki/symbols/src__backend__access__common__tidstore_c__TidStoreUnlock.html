TidStoreUnlock

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tidstore.c:300

目的

TidStore に対して取得済みの共有メモリロックを解放します。

シグネチャまたは形状

void TidStoreUnlock(TidStore *ts)

入力と出力

  • **入力**: ts (TidStore *) - ロックを解放する対象の TidStore 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • ts は有効な TidStore 構造体を指している必要があります。
  • 共有メモリモードで作成された TidStore に対してのみ、ロック解放処理が実行されます。

アルゴリズムまたは振る舞い

1. TidStoreIsShared(ts) を呼び出し、対象の TidStore が共有メモリを使用しているか判定します。 2. 共有メモリを使用している場合、内部の shared_ts_unlock 関数を呼び出し、共有メモリ上のロックを解放します。 3. ローカルメモリモードの場合は何もしません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_tidstore/test_tidstore.c (テストモジュール内の複数箇所)
  • **呼び出し先**:
  • TidStoreIsShared (インライン判定)
  • shared_ts_unlock (内部関数)

関連データ構造

  • TidStore: TID 集合を管理するメイン構造体。

失敗モードと境界条件

  • ts が NULL の場合の動作は定義されていません。
  • 共有メモリロックを保持していない状態で本関数を呼び出した場合の動作は、基盤となるロック機構（LWLock 等）の仕様に依存します。

保守担当者向けメモ

  • 本関数は TidStore の共有メモリ版におけるロック管理のラッパーです。ロックの取得側である TidStoreLockExclusive や TidStoreLockShare と対になる操作です。
