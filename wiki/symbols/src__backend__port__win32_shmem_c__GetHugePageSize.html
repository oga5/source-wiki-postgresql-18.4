GetHugePageSize

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_shmem.c:629

目的

Windows 環境において、Huge Page（ラージページ）のサイズおよびメモリマッピング用フラグを取得するためのインターフェースを提供します。

シグネチャまたは形状

void GetHugePageSize(Size *hugepagesize, int *mmap_flags)

入力と出力

  • **入力**:
  • hugepagesize: Huge Page サイズを格納する Size 型へのポインタ。
  • mmap_flags: メモリマッピング用フラグを格納する int 型へのポインタ。
  • **出力**:
  • なし（引数ポインタ経由で値を書き戻す）。

前提条件と不変条件

  • 引数が NULL でない場合、各値は 0 に初期化されます。
  • 本実装（win32_shmem.c）では、Huge Page サイズおよびフラグとして常に 0 を返します。

アルゴリズムまたは振る舞い

1. hugepagesize が NULL でなければ、その指す先に 0 を代入します。 2. mmap_flags が NULL でなければ、その指す先に 0 を代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c
  • src/backend/storage/ipc/shmem.c
  • src/backend/port/sysv_shmem.c（注: sysv_shmem.c 内の呼び出しは、プラットフォーム条件に応じた条件付きコンパイルの一部である可能性があります）
  • **呼び出し先**:
  • なし。

関連データ構造

  • Size（postgres.h で定義される型）

失敗モードと境界条件

  • 引数に NULL が渡された場合、その引数に対する書き込み処理はスキップされ、安全に終了します。

保守担当者向けメモ

  • 本関数は Windows 環境用のスタブ実装です。Huge Page のサポートを Windows 上で有効化する場合、GetLargePageMinimum 等の Windows API を用いた実装への置き換えが必要です。
  • src/include/storage/pg_shmem.h で宣言されており、プラットフォーム間で共通のインターフェースとして機能します。
