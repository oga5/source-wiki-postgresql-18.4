init_tablespaces

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/tablespace.c:18

目的

pg_upgrade におけるテーブルスペースのパス情報を収集し、新旧クラスタ間でのバージョン固有のディレクトリサフィックスを初期化・検証する。

シグネチャまたは形状

void init_tablespaces(void)

入力と出力

  • 入力: なし（グローバル変数 old_cluster, new_cluster, os_info を参照）
  • 出力: なし（副作用としてグローバル状態を更新し、検証失敗時にプロセスを終了する）

前提条件と不変条件

  • get_tablespace_paths() が実行され、os_info 構造体が適切に初期化されている必要がある。
  • old_cluster および new_cluster 構造体が、クラスタ情報で初期化済みであること。

アルゴリズムまたは振る舞い

1. get_tablespace_paths() を呼び出し、旧クラスタからテーブルスペースのパス情報を取得する。 2. set_tablespace_directory_suffix() を用いて、旧クラスタおよび新クラスタそれぞれのバージョン固有のディレクトリサフィックスを生成・設定する。 3. テーブルスペースが存在する場合、新旧クラスタのサフィックスを比較する。 4. サフィックスが同一（＝システムカタログのバージョンが同一）である場合、テーブルスペースを使用しているアップグレードは安全ではないと判断し、pg_fatal でエラー終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/check.c:608
  • 呼び出し先:
  • get_tablespace_paths()
  • set_tablespace_directory_suffix()
  • pg_fatal()

関連データ構造

  • ClusterInfo (新旧クラスタの情報を保持)
  • os_info (テーブルスペースのパスや個数などのOSレベルの情報を保持)

失敗モードと境界条件

  • 旧クラスタと新クラスタのシステムカタログバージョンが同一であり、かつテーブルスペースが使用されている場合、pg_fatal によりアップグレードが中断される。
  • get_tablespace_paths 内でのパス取得や stat 検証に失敗した場合、その下位関数でエラーが発生する。

保守担当者向けメモ

  • 本関数はグローバルな os_info 構造体を更新するため、呼び出し順序が重要である。
  • テーブルスペースの作成・削除を行う関数ではなく、あくまでアップグレード前のパス管理と検証を行うためのものである。
