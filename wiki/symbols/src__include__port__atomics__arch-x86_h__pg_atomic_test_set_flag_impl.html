pg_atomic_test_set_flag_impl

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/atomics/arch-x86.h:134

目的

x86アーキテクチャにおいて、アトミックにフラグをセットし、以前の状態を取得する（Test-and-Set）。

シグネチャまたは形状

bool pg_atomic_test_set_flag_impl(volatile pg_atomic_flag *ptr)

入力と出力

  • **入力**: ptr - 操作対象となる pg_atomic_flag 構造体へのポインタ。
  • **出力**: フラグが以前 0 であった場合（セットに成功した場合）に true を返し、以前 1 であった場合（既にセットされていた場合）に false を返す。

前提条件と不変条件

  • ptr は有効な pg_atomic_flag 構造体を指している必要がある。
  • volatile 修飾により、コンパイラによる最適化でメモリ読み書きが省略されないことが保証される。

アルゴリズムまたは振る舞い

1. ローカル変数 _res を 1 で初期化する。 2. インラインアセンブラを用いて lock xchgb 命令を実行する。

  • ptr->value の値を _res にロードし、同時に ptr->value に 1 を書き込む。
  • lock プレフィックスにより、マルチプロセッサ環境でのアトミック性が保証される。

    • 3. memory クライバにより、コンパイラに対してメモリバリアを指示する。 4. _res が 0 であったかどうかを評価し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics.h:185 にて呼び出されている。
  • **呼び出し先**: CPUの xchgb 命令を直接発行する。

関連データ構造

  • pg_atomic_flag: アトミック操作対象となるフラグ構造体。

失敗モードと境界条件

  • 本関数はハードウェア命令に依存するため、メモリが正しくアライメントされていない場合や、無効なアドレスが渡された場合には未定義動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本実装はx86のTSO（Total Store Order）モデルに基づいている。
  • xchgb は常にロックされるため、競合が発生した際のオーバーヘッドに注意すること。
