freeHyperLogLog

定義

  • 種別: function
  • 定義箇所: src/backend/lib/hyperloglog.c:150

目的

hyperLogLogState 構造体が内部で保持するレジスタ配列（hashesArr）のメモリを解放します。

シグネチャまたは形状

void freeHyperLogLog(hyperLogLogState *cState)

入力と出力

  • **入力**: cState - 解放対象の hyperLogLogState 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • cState は有効なポインタである必要があります。
  • cState->hashesArr は NULL であってはなりません（Assert により検証されます）。

アルゴリズムまたは振る舞い

1. cState->hashesArr が NULL でないことを確認します。 2. pfree を呼び出し、hashesArr に割り当てられていたメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAgg.c:3218 での呼び出しが確認されています。
  • **呼び出し先**: pfree（PostgreSQL のメモリ解放関数）を呼び出します。

関連データ構造

  • hyperLogLogState: HLL の状態を保持する構造体。本関数はこの構造体内の動的メモリを管理します。

失敗モードと境界条件

  • cState->hashesArr が NULL の場合、アサーション違反によりプロセスが異常終了する可能性があります（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は hashesArr のメモリのみを解放します。hyperLogLogState 構造体自体のメモリ解放は呼び出し元で行う必要があります。
