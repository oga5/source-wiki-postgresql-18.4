WordEntryIN

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsvector.c:26

目的

tsvector の入力処理および構築プロセスにおいて、語彙（lexeme）とその位置情報（position）を一時的に保持するための作業用構造体です。

シグネチャまたは形状

typedef struct {
    WordEntry    entry;    /* 必須: compareentry で使用するため先頭に配置 */
    WordEntryPos *pos;     /* 位置情報の配列 */
    int          poslen;   /* pos 配列の要素数 */
} WordEntryIN;

入力と出力

本構造体は関数間のデータ受け渡しに使用されます。

  • **入力**: tsvectorin 等の処理過程で、解析されたトークン情報と位置情報が格納されます。
  • **出力**: uniqueentry 関数等により、ソートや重複排除の対象として利用されます。

前提条件と不変条件

  • entry フィールドは構造体の先頭に配置する必要があります。これは compareentry 関数が WordEntryIN へのポインタを WordEntry としてキャストして扱うためです。

アルゴリズムまたは振る舞い

  • tsvector の構築過程において、複数のトークンをソートし、同一語彙をマージする際の単位として機能します。
  • qsort_arg を用いたソート処理において、compareentry 関数と共に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: tsvectorin および tsvectorrecv 内で、入力データの解析・構築時に動的に割り当てられます。
  • **呼び出し先**: compareentry（ソート用比較関数）が本構造体の entry フィールドを参照します。

関連データ構造

  • WordEntry: tsvector の内部バイナリ表現における語彙エントリ。
  • WordEntryPos: 位置情報と重みを保持するデータ型。

失敗モードと境界条件

  • メモリ割り当て: palloc および repalloc を使用して動的に配列を確保するため、メモリ不足時にはエラーが発生します。
  • ソート順序: compareentry の仕様に従い、語彙が正しくソートされていることが uniqueentry による重複排除の前提となります。

保守担当者向けメモ

  • WordEntry が構造体の先頭にあるという制約は、メモリレイアウトに依存する重要な不変条件です。構造体のメンバ順序を変更する際は注意してください。
  • 大規模な tsvector を処理する場合、WordEntryIN の配列確保がメモリ消費のボトルネックとなる可能性があります。
