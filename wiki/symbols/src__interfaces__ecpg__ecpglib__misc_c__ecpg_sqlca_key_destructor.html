ecpg_sqlca_key_destructor

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/misc.c:95

目的

スレッド終了時に、スレッドローカルな sqlca_t 構造体のメモリを解放するためのデストラクタ関数です。

シグネチャまたは形状

static void ecpg_sqlca_key_destructor(void *arg)

入力と出力

  • **入力**: void *arg - 解放対象となる sqlca_t 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 arg は、ECPGget_sqlca 内で動的に確保された有効なメモリ領域を指している必要があります。
  • 本関数は pthread_key_create のデストラクタとして登録されることを前提としています。

アルゴリズムまたは振る舞い

1. 受け取ったポインタ arg に対して free() を実行し、スレッド固有の sqlca 領域を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: スレッド終了時に pthread ライブラリによって自動的に呼び出されます。
  • **呼び出し先**: free()（標準ライブラリ関数）。

関連データ構造

  • sqlca_t: ECPGが使用するSQL通信領域構造体。

失敗モードと境界条件

  • arg が NULL の場合、free() の仕様に従い何も行われません。

保守担当者向けメモ

  • 本関数は pthread_key_create に登録されるコールバックであるため、直接呼び出すことは想定されていません。
  • ECPGget_sqlca で割り当てられたメモリのライフサイクル管理を担っているため、メモリ割り当てロジックを変更する際は本関数の整合性を確認してください。
