EphemeralNamedRelationData

定義

  • 種別: struct
  • 定義箇所: src/include/utils/queryenvironment.h:50

目的

カタログに存在しない一時的な名前付きリレーション（ENR）のメタデータと、実行時のデータアクセス用ポインタを保持するコンテナ構造体です。

シグネチャまたは形状

typedef struct EphemeralNamedRelationData
{
    EphemeralNamedRelationMetadataData md;
    void       *reldata;
} EphemeralNamedRelationData;

入力と出力

  • **入力**: EphemeralNamedRelationMetadataData 型のメタデータおよび、実行時にアクセス可能なデータへの汎用ポインタ（void *reldata）。
  • **出力**: なし（データ構造体）。

前提条件と不変条件

  • md フィールドには、リレーション名や型情報などのメタデータが格納されている必要があります。
  • reldata は、クエリ実行コンテキストにおいて有効なデータ構造を指している必要があります。

アルゴリズムまたは振る舞い

本構造体は、クエリ実行中に一時的なリレーションを管理するためのデータ保持単位として機能します。md を通じてリレーションのスキーマ情報（TupleDesc 等）を特定し、reldata を通じて実際のデータへアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/spi.c 内の関数が、本構造体のメモリを確保（palloc）して使用していることが確認されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • EphemeralNamedRelationMetadataData: リレーションのメタデータを保持する構造体。
  • EphemeralNamedRelation: 本構造体へのポインタ型（typedef EphemeralNamedRelationData *EphemeralNamedRelation）。
  • QueryEnvironment: 本構造体を管理するコンテナ。

失敗モードと境界条件

  • メモリ確保失敗: palloc を使用しているため、メモリ不足時にはエラーが発生します。
  • ライフサイクル不一致: reldata が指すデータが、QueryEnvironment の生存期間よりも先に解放された場合、ダングリングポインタとなります。

保守担当者向けメモ

  • 本構造体は queryenvironment.h で定義されており、queryenvironment.c で実装される管理ロジックと密接に関連しています。
  • reldata は void * であるため、キャスト時の型安全性は呼び出し側の実装に依存します。変更を加える際は、関連するキャスト箇所を確認してください。
