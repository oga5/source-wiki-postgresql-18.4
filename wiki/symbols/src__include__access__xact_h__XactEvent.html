XactEvent

定義

  • 種別: enum
  • 定義箇所: src/include/access/xact.h:126

目的

トランザクションのライフサイクルにおける特定のフェーズ（コミット、アボート、準備など）を識別し、登録されたコールバック関数をトリガーするためのイベント種別を定義します。

シグネチャまたは形状

typedef enum
{
	XACT_EVENT_COMMIT,
	XACT_EVENT_PARALLEL_COMMIT,
	XACT_EVENT_ABORT,
	XACT_EVENT_PARALLEL_ABORT,
	XACT_EVENT_PREPARE,
	XACT_EVENT_PRE_COMMIT,
	XACT_EVENT_PARALLEL_PRE_COMMIT,
	XACT_EVENT_PRE_PREPARE,
} XactEvent;

入力と出力

  • **入力**: トランザクション制御関数（CallXactCallbacks 等）が、現在のトランザクション状態に応じてこの列挙値を引数として渡します。
  • **出力**: 該当イベントに登録されたコールバック関数が呼び出されます。

前提条件と不変条件

  • トランザクションの各フェーズにおいて、対応する XactEvent が CallXactCallbacks を通じて発行される必要があります。
  • コールバック関数は void (*XactCallback) (XactEvent event, void *arg) の形式に従う必要があります。

アルゴリズムまたは振る舞い

トランザクションの状態遷移に伴い、CallXactCallbacks(XactEvent event) が呼び出されます。この関数は、登録済みの全コールバックを順次実行し、各モジュール（postgres_fdw, sepgsql, plpgsql 等）がトランザクションの終了や準備に合わせてリソースの解放や状態の更新を行えるようにします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c: CallXactCallbacks 関数内。
  • **呼び出し先**:
  • contrib/postgres_fdw/connection.c: pgfdw_xact_callback
  • contrib/sepgsql/label.c: sepgsql_xact_callback
  • src/pl/plpgsql/src/pl_exec.c: plpgsql_xact_cb

関連データ構造

  • XactCallback: XactEvent を引数に取る関数ポインタ型。

失敗モードと境界条件

  • 供給されたソース情報には、特定の失敗モードや境界条件に関する記述はありません。

保守担当者向けメモ

  • 本列挙型に新しいイベントを追加した場合は、CallXactCallbacks を呼び出すトランザクション制御ロジック側で、そのイベントが適切に処理されることを確認してください。
  • PARALLEL 系イベントは、パラレルクエリ実行時のワーカープロセスにおけるトランザクション状態と同期するために使用されます。
