EOH_get_flat_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandeddatum.c:74

目的

展開済みオブジェクト（Expanded Object）をフラット化（シリアライズ）した際に必要となるバイトサイズを取得します。

シグネチャまたは形状

Size EOH_get_flat_size(ExpandedObjectHeader *eohptr)

入力と出力

  • **入力**: eohptr - 展開済みオブジェクトの共通ヘッダーへのポインタ。
  • **出力**: Size 型のバイトサイズ。

前提条件と不変条件

  • eohptr は有効な ExpandedObjectHeader を指している必要があります。
  • eohptr->eoh_methods が初期化されており、かつ get_flat_size メソッドが実装されている必要があります。

アルゴリズムまたは振る舞い

1. 引数として渡された eohptr が保持するメソッドテーブル（eoh_methods）にアクセスします。 2. メソッドテーブル内の get_flat_size 関数ポインタを呼び出し、その結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/detoast.c
  • src/backend/access/common/heaptuple.c
  • src/backend/executor/execTuples.c
  • src/backend/utils/adt/datum.c
  • **呼び出し先**:
  • eohptr->eoh_methods->get_flat_size（各データ型固有の実装）

関連データ構造

  • ExpandedObjectHeader: 展開済みオブジェクトの共通ヘッダー。
  • ExpandedObjectMethods: 型固有の操作メソッドを定義する構造体。

失敗モードと境界条件

  • eohptr が NULL の場合、セグメンテーション違反が発生します。
  • メソッドテーブルが正しく設定されていない場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のサイズ計算ロジックは各データ型（array_expanded.c や expandedrecord.c など）のメソッド実装に依存します。
  • 呼び出し元で Datum から ExpandedObjectHeader を取得する際は、DatumGetEOHP マクロを介して安全に変換してください。
