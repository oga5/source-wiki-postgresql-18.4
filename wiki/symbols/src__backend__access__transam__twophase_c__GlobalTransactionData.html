GlobalTransactionData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/transam/twophase.c:147

目的

準備済みトランザクション（GXACT）の状態を保持し、2フェーズコミット（2PC）におけるトランザクションのライフサイクル管理およびクラッシュリカバリを可能にするためのデータ構造です。

シグネチャまたは形状

typedef struct GlobalTransactionData
{
    GlobalTransaction next;             /* 自由リストへのリンク */
    int               pgprocno;         /* 関連付けられたダミーPGPROCのID */
    TimestampTz       prepared_at;      /* 準備時刻 */
    XLogRecPtr        prepare_start_lsn;/* 準備レコード開始のXLOGオフセット */
    XLogRecPtr        prepare_end_lsn;  /* 準備レコード終了のXLOGオフセット */
    TransactionId     xid;              /* GXACT ID */
    Oid               owner;            /* トランザクション実行ユーザーのID */
    ProcNumber        locking_backend;  /* 現在処理中のバックエンド */
    bool              valid;            /* PGPROCエントリがproc arrayにあるか */
    bool              ondisk;           /* 準備状態ファイルがディスクにあるか */
    bool              inredo;           /* xlog_redo経由で追加されたか */
    char              gid[GIDSIZE];     /* 準備済みトランザクションに割り当てられたGID */
} GlobalTransactionData;

入力と出力

  • **入力**: PREPARE TRANSACTION 実行時に生成され、GID、XID、所有者情報などが設定されます。
  • **出力**: COMMIT PREPARED または ROLLBACK PREPARED 実行時に、WALおよびディスク上の状態が読み取られ、トランザクションの終了処理に使用されます。

前提条件と不変条件

  • 準備済みトランザクションの操作は TwoPhaseStateLock によって保護される必要があります。
  • prepare_start_lsn はWALから状態を読み戻すために使用され、prepare_end_lsn はコミット前に待機すべきLSNとして使用されます。
  • valid フラグが真の場合、対応するダミー PGPROC が proc array に存在することが保証されます。

アルゴリズムまたは振る舞い

1. **準備**: MarkAsPreparing によりGIDが予約され、GlobalTransactionData が初期化されます。 2. **永続化**: チェックポイント時に ondisk フラグが管理され、pg_twophase ディレクトリへの書き出しが行われます。 3. **完了**: FinishPreparedTransaction が呼び出されると、本構造体の情報に基づき、WALまたはディスクから状態を復元してリソースを解放します。 4. **リカバリ**: 起動時に restoreTwoPhaseData 等が本構造体を再構築し、システムの一貫性を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/twophase.c 内の各関数（MarkAsPreparing, FinishPreparedTransaction 等）が本構造体を操作します。
  • **呼び出し先**: 直接的な関数呼び出し関係はソース抜粋からは特定できません。

関連データ構造

  • GlobalTransaction: GlobalTransactionData へのポインタ型。
  • PGPROC: 準備済みトランザクションがシステム上で実行中として扱われるために使用するダミー構造体。
  • TwoPhaseStateData: 共有メモリ上で GlobalTransactionData を管理するグローバル構造体。

失敗モードと境界条件

  • max_prepared_xacts GUC変数により、同時に存在可能な GlobalTransactionData の最大数が制限されます。
  • 共有メモリ上のロック競合が発生した場合、デッドロックを避けるための適切なロック順序の遵守が必要です。

保守担当者向けメモ

  • 本構造体は src/include/access/twophase.h で GlobalTransaction 型としてポインタが公開されています。
  • 準備済みトランザクションはWALとディスクの二重管理下にあり、リカバリ時はこれらを統合して一貫性を維持する設計となっています。
