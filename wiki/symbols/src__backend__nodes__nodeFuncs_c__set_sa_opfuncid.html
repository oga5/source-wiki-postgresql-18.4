set_sa_opfuncid

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/nodeFuncs.c:1878

目的

ScalarArrayOpExpr ノード内の演算子関数 ID (opfuncid) が未設定（InvalidOid）である場合に、演算子 OID (opno) から対応する関数 ID を解決して設定します。

シグネチャまたは形状

void set_sa_opfuncid(ScalarArrayOpExpr *opexpr)

入力と出力

  • **入力**: ScalarArrayOpExpr *opexpr - 処理対象となるスカラー配列演算子式ノード。
  • **出力**: なし（引数 opexpr の opfuncid フィールドを直接更新します）。

前提条件と不変条件

  • opexpr は有効な ScalarArrayOpExpr 構造体へのポインタである必要があります。
  • get_opcode が呼び出されるため、システムカタログから演算子情報が取得可能である必要があります。

アルゴリズムまたは振る舞い

1. opexpr->opfuncid が InvalidOid であるかを確認します。 2. InvalidOid の場合、get_opcode(opexpr->opno) を呼び出し、演算子 opno に対応する関数 ID を取得します。 3. 取得した関数 ID を opexpr->opfuncid に代入します。 4. すでに opfuncid が設定されている場合は、何も行いません。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/nodeFuncs.c (内部の走査処理)
  • src/backend/optimizer/path/costsize.c
  • src/backend/optimizer/plan/setrefs.c
  • src/backend/optimizer/util/clauses.c
  • **呼び出し先**:
  • get_opcode (システムカタログから演算子関数 ID を取得する関数)

関連データ構造

  • ScalarArrayOpExpr: src/include/nodes/execnodes.h 等で定義される、IN や ANY/ALL 演算子を含む式ノード。

失敗モードと境界条件

  • opexpr が NULL の場合、セグメンテーション違反が発生する可能性があります。
  • opno が無効な演算子である場合、get_opcode の挙動に依存します。

保守担当者向けメモ

  • 本関数は fix_opfuncids 等のツリー走査処理の一部として利用されます。ノードの属性を最適化するフェーズで呼び出されることが多いため、変更時はクエリ最適化への影響を考慮してください。
