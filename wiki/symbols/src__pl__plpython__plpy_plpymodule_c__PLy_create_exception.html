PLy_create_exception

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:207

目的

Python の例外クラスを動的に生成し、指定された Python モジュールに登録する。

シグネチャまたは形状

static PyObject *PLy_create_exception(char *name, PyObject *base, PyObject *dict, const char *modname, PyObject *mod)

入力と出力

  • **入力**:
  • name: 例外クラス名（例: "plpy.Error"）。
  • base: 基底クラス（NULL 可）。
  • dict: クラス属性を定義する辞書（NULL 可）。
  • modname: モジュール内の属性名。
  • mod: 例外を登録する対象の Python モジュールオブジェクト。
  • **出力**:
  • 生成された例外クラスの PyObject ポインタ。

前提条件と不変条件

  • PyErr_NewException が失敗した場合、PLy_elog(ERROR, ...) を呼び出し、PostgreSQL 側でエラーを報告する。
  • 参照カウントの管理: PyModule_AddObject は参照カウントを増加させないため、関数内で Py_INCREF を2回実行する（1つはモジュール登録用、もう1つは呼び出し元での保持用）。

アルゴリズムまたは振る舞い

1. PyErr_NewException を呼び出し、新しい例外クラスを作成する。 2. 作成した例外オブジェクトに対して Py_INCREF を実行し、PyModule_AddObject を介してモジュールに登録する。 3. 呼び出し元が永続的な変数に保持できるよう、再度 Py_INCREF を実行して参照カウントを増やした状態でオブジェクトを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_plpymodule.c 内の初期化処理（PLy_exc_error, PLy_exc_fatal, PLy_exc_spi_error の生成時）。
  • PLy_generate_spi_exceptions 関数内（exception_map に基づく動的生成時）。
  • **呼び出し先**:
  • PyErr_NewException (Python C API)
  • PyModule_AddObject (Python C API)
  • PLy_elog (PostgreSQL/PL/Python ログ出力)

関連データ構造

  • PyObject (Python C API の汎用オブジェクト型)

失敗モードと境界条件

  • PyErr_NewException が NULL を返した場合、PostgreSQL の ERROR レベルで処理が中断される。

保守担当者向けメモ

  • 本関数は、Python の参照カウント管理を明示的に行う必要がある。PyModule_AddObject の仕様（参照カウントを消費しない）に依存しているため、将来的な Python API の変更時には注意が必要である。
  • 呼び出し元で参照を保持することを前提とした設計となっている。
