ParallelApplyWorkerMain

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/applyparallelworker.c:865

目的

論理レプリケーションにおける並列適用ワーカーのメインエントリーポイントとして、共有メモリの初期化、通信路の確立、および適用ループの実行を行う。

シグネチャまたは形状

void ParallelApplyWorkerMain(Datum main_arg)
  • main_arg: ワーカーのスロット番号（int）を保持する Datum 型。

入力と出力

  • **入力**: main_arg（ワーカー識別用）、および MyBgworkerEntry->bgw_extra に格納された動的共有メモリ（DSM）ハンドル。
  • **出力**: なし（終了時はプロセス終了）。

前提条件と不変条件

  • BackgroundWorker として起動されていること。
  • DSMセグメントおよび共有メモリテーブル（shm_toc）が正しく配置されていること。
  • 呼び出し元（リーダーワーカー）によって、必要な共有メモリ構造が初期化済みであること。

アルゴリズムまたは振る舞い

1. **シグナルハンドラ設定**: SIGHUP, SIGTERM, SIGUSR2 を設定し、バックグラウンドワーカーのシグナルブロックを解除。 2. **メモリ接続**: dsm_attach によりDSMへ接続し、shm_toc を介して共有メモリ上の制御情報およびメッセージキュー（mq）を取得。 3. **ワーカー登録**: logicalrep_worker_attach で自身のスロットを確保し、pa_shutdown を終了時コールバックとして登録。 4. **エラーキュー設定**: pq_redirect_to_shm_mq を使用し、エラー出力を共有メモリキューへリダイレクト。 5. **レプリケーション起点設定**: replorigin_session_setup を呼び出し、リーダーと同一のレプリケーション起点でセッションを確立。 6. **メインループ**: LogicalParallelApplyLoop を呼び出し、リーダーからのメッセージ待機・適用処理へ移行。 7. **終了**: ループを抜けることは想定されておらず、到達した場合は Assert(false) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: postmaster（BackgroundWorker の起動プロセス）。
  • **呼び出し先**: dsm_attach, shm_toc_attach, logicalrep_worker_attach, before_shmem_exit, pq_redirect_to_shm_mq, InitializeLogRepWorker, StartTransactionCommand, replorigin_session_setup, CacheRegisterSyscacheCallback, LogicalParallelApplyLoop 等。

関連データ構造

  • ParallelApplyWorkerShared: 共有メモリ上のワーカー状態管理構造体。
  • shm_mq: リーダー・ワーカー間の通信用メッセージキュー。
  • LogicalRepWorker: 論理レプリケーションワーカーの内部状態を保持する構造体。

失敗モードと境界条件

  • **DSM接続失敗**: dsm_attach が失敗した場合、ERROR で終了。
  • **TOC不正**: shm_toc_attach でマジックナンバーが不一致の場合、ERROR で終了。
  • **通信切断**: リーダーからの SIGTERM または SIGUSR2 受信により、LogicalParallelApplyLoop を経由して終了処理が実行される。

保守担当者向けメモ

  • 本関数は LogicalParallelApplyLoop に制御を渡した後は戻らない設計である。
  • SIGUSR2 はリーダーワーカーからの正常終了要求として扱われるため、デバッグ時にはシグナル送信元と意図を区別すること。
  • pq_redirect_to_shm_mq によるエラー出力のリダイレクトは、並列ワーカーで発生したエラーをリーダーへ伝播させるために不可欠である。
