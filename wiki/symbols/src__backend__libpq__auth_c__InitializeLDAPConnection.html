InitializeLDAPConnection

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth.c:2216

目的

LDAP認証のための接続ハンドルを初期化し、プロトコルバージョン設定およびTLSセッションの確立を行う。

シグネチャまたは形状

static int InitializeLDAPConnection(Port *port, LDAP **ldap);

入力と出力

  • **入力**:
  • port: 接続情報を保持する Port 構造体（hba 設定を含む）。
  • ldap: 初期化されたLDAPハンドルを格納するためのポインタ。
  • **出力**:
  • 成功時は STATUS_OK を返す。
  • 失敗時は STATUS_ERROR を返す。

前提条件と不変条件

  • port->hba が有効であること。
  • LDAPライブラリ（OpenLDAPまたはWinLDAP）がリンクされていること。

アルゴリズムまたは振る舞い

1. port->hba->ldapscheme に基づき、接続スキーム（ldap または ldaps）を決定する。 2. プラットフォームに応じてLDAPハンドルを初期化する:

  • **Win32**: ldap_sslinit または ldap_init を使用。
  • **OpenLDAP**: ldap_initialize を使用し、必要に応じてDNS SRVレコードからサーバーリストを解決する。
  • **その他**: ldap_init を使用。

    • 3. LDAP_OPT_PROTOCOL_VERSION を LDAP_VERSION3 に設定する。 4. port->hba->ldaptls が真の場合、ldap_start_tls_s を呼び出しTLSセッションを開始する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c 内の CheckLDAPAuth から呼び出される（ソースコード上の直接的な参照箇所: src/backend/libpq/auth.c:2487）。
  • **呼び出し先**: ldap_init, ldap_sslinit, ldap_initialize, ldap_set_option, ldap_start_tls_s, ldap_unbind 等のLDAPライブラリ関数。

関連データ構造

  • Port: 接続要求の詳細と HBA 設定を保持する構造体。
  • LDAP: LDAPライブラリが提供する接続ハンドル。

失敗モードと境界条件

  • LDAPサーバーへの接続初期化失敗（ldap_init 等がNULLを返す）。
  • プロトコルバージョンの設定失敗。
  • TLSセッションの開始失敗。
  • DNS SRVレコードの解決失敗（OpenLDAP環境でサーバー指定がない場合）。
  • 各エラー発生時には ereport(LOG, ...) を通じてログを出力し、STATUS_ERROR を返す。

保守担当者向けメモ

  • 本関数はプラットフォーム依存のLDAP実装（Win32 vs OpenLDAP）を抽象化している。
  • 新しいLDAPライブラリ機能を追加する場合は、#ifdef ブロックの条件分岐に注意すること。
  • 認証失敗時の詳細なエラー理由はセキュリティ上の理由からクライアントには通知されず、サーバーログにのみ記録される。
