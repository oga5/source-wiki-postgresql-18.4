pg_abs_s16

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:136

目的

符号付き16ビット整数（int16）の絶対値を、オーバーフローを回避しつつ符号なし16ビット整数（uint16）として安全に取得する。

シグネチャまたは形状

static inline uint16 pg_abs_s16(int16 a)

入力と出力

  • **入力**: int16 a（絶対値を求める対象の整数）
  • **出力**: uint16（a の絶対値）

前提条件と不変条件

  • int16 型の最小値（INT16_MIN）を絶対値に変換する際、通常の符号反転ではオーバーフローが発生するが、本関数は型を拡張することでこれを回避する。

アルゴリズムまたは振る舞い

1. 引数 a を int32 型にキャストして拡張する。 2. 拡張された値に対して abs() 関数を適用する。 3. 得られた int32 型の結果を uint16 型にキャストして戻り値とする。 この手順により、INT16_MIN を含むすべての int16 値に対して安全な変換を保証する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**: 標準ライブラリの abs() 関数を呼び出します。

関連データ構造

  • int16, int32, uint16: PostgreSQL で定義される標準的な整数型。

失敗モードと境界条件

  • a = -32768 (INT16_MIN) の場合: int32 への拡張により abs() が正しく 32768 を返し、最終的に uint16 の範囲内に収まるため、オーバーフローや未定義動作は発生しません。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、ヘッダーファイル内で完結しています。
  • 移植性を担保するため、コンパイラ依存の組み込み関数ではなく標準の abs() を型拡張と組み合わせて使用しています。
