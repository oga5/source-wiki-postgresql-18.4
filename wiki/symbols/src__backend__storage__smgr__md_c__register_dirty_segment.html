register_dirty_segment

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/md.c:1507

目的

リレーションの特定のセグメントがダーティ（未同期）であることを記録し、チェックポインタプロセスに対して fsync を要求する。

シグネチャまたは形状

static void register_dirty_segment(SMgrRelation reln, ForkNumber forknum, MdfdVec *seg)

入力と出力

  • **入力**:
  • reln: 対象リレーションのストレージマネージャ構造体。
  • forknum: 対象のフォーク番号。
  • seg: 対象セグメントのファイル記述子およびメタデータを含む MdfdVec 構造体。
  • **出力**: なし（副作用として同期要求の登録または直接の fsync 実行を行う）。

前提条件と不変条件

  • 一時リレーション（SmgrIsTemp(reln) が真）に対しては呼び出してはならない（アサーションにより強制）。

アルゴリズムまたは振る舞い

1. INIT_MD_FILETAG を使用して、リレーションのロケータ、フォーク番号、セグメント番号から FileTag を生成する。 2. RegisterSyncRequest を呼び出し、チェックポインタへの同期要求を試みる。 3. 要求キューが満杯で登録に失敗した場合:

  • DEBUG1 レベルでログを出力する。
  • FileSync を呼び出し、当該セグメントに対して直接 fsync を実行する。
  • 実行した fsync の所要時間を pgstat_count_io_op_time を用いて統計情報に記録する（IOCONTEXT_NORMAL として計上）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/smgr/md.c 内の複数の箇所（mdextend, mdwritev 等のセグメント更新処理）。
  • **呼び出し先**:
  • RegisterSyncRequest (同期要求の登録)
  • FileSync (キュー溢れ時の直接同期)
  • pgstat_prepare_io_time / pgstat_count_io_op_time (統計計測)
  • data_sync_elevel (エラーレベル判定)

関連データ構造

  • SMgrRelation: ストレージマネージャのリレーション情報。
  • MdfdVec: セグメントファイル記述子とセグメント番号を保持する構造体。
  • FileTag: 同期対象ファイルを一意に識別するためのタグ。

失敗モードと境界条件

  • **キュー溢れ**: RegisterSyncRequest が失敗した場合、バックエンドプロセス自身が FileSync を実行して同期を完了させる。
  • **同期失敗**: FileSync がエラーを返した場合、data_sync_elevel に基づいてエラー（通常は ERROR）を発生させる。

保守担当者向けメモ

  • 本関数はチェックポインタへの非同期通知を優先する設計である。
  • fsync を直接実行する場合、現在の IOContext を特定できないため、統計上は一律 IOCONTEXT_NORMAL として記録される。
