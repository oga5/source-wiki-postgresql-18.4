socket_set

定義

  • 種別: struct
  • 定義箇所: src/bin/pgbench/pgbench.c:97

目的

pgbench におけるネットワーク I/O の多重化（ppoll または select）を抽象化するためのデータ構造です。

シグネチャまたは形状

typedef struct socket_set
{
    int         maxfds;     /* 割り当てられた pollfds[] 配列の長さ */
    int         curfds;     /* 現在使用中の要素数 */
    struct pollfd pollfds[FLEXIBLE_ARRAY_MEMBER];
} socket_set;

入力と出力

  • 入力: alloc_socket_set 関数により、指定したカウント数に基づいて動的にメモリが確保されます。
  • 出力: pollfds 配列を通じて、監視対象のファイルディスクリプタの状態が保持されます。

前提条件と不変条件

  • pollfds は FLEXIBLE_ARRAY_MEMBER を使用した可変長配列として定義されており、構造体インスタンスの末尾に配置されます。
  • curfds は常に maxfds 以下である必要があります。

アルゴリズムまたは振る舞い

  • pgbench のメインループにおいて、複数のデータベース接続を効率的に監視するために使用されます。
  • add_socket_to_set で監視対象を追加し、wait_on_socket_set を介してプラットフォーム依存の I/O 待機処理が行われます。
  • socket_has_input により、特定のファイルディスクリプタにデータが到着しているかを確認します。

呼び出し元と呼び出し先

  • 呼び出し元: pgbench.c 内のメイン処理フロー（main 関数周辺）でインスタンスが生成されます。
  • 呼び出し先: 以下の関数群がこの構造体を操作します。
  • alloc_socket_set
  • free_socket_set
  • clear_socket_set
  • add_socket_to_set
  • wait_on_socket_set
  • socket_has_input

関連データ構造

  • struct pollfd: poll.h で定義される標準的な構造体で、socket_set の内部配列として利用されます。

失敗モードと境界条件

  • alloc_socket_set によるメモリ確保失敗時の挙動については、本ソース抜粋からは判断できません。
  • maxfds を超えるソケットの追加は、配列の範囲外アクセスを引き起こす可能性があるため、呼び出し側での管理が必須です。

保守担当者向けメモ

  • 本構造体は pgbench のネットワーク抽象化レイヤーの核です。プラットフォーム間で poll と select の挙動を統一するために使用されています。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には注意が必要です。
