get_sock_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/option.c:498

目的

指定されたクラスタの Unix ドメインソケットディレクトリを特定し、ClusterInfo 構造体に設定する。

シグネチャまたは形状

void get_sock_dir(ClusterInfo *cluster);

入力と出力

  • **入力**: ClusterInfo *cluster（対象のクラスタ情報へのポインタ）
  • **出力**: なし（副作用として cluster->sockdir を更新）

前提条件と不変条件

  • WIN32 環境以外では、postmaster.pid ファイルが存在し、読み取り可能である必要がある。
  • user_opts グローバル変数が初期化されていること。

アルゴリズムまたは振る舞い

1. **Windows 環境**: cluster->sockdir を NULL に設定する。 2. **非 Windows 環境**:

  • user_opts.live_check が偽、または対象が new_cluster である場合、user_opts.socketdir をそのまま cluster->sockdir に代入する。
  • それ以外（ライブチェック時）の場合:
  • cluster->pgdata 内の postmaster.pid を開く。
  • ファイルの特定の行（LOCK_FILE_LINE_PORT および LOCK_FILE_LINE_SOCKET_DIR）を読み取る。
  • ポート番号を old_cluster.port に反映し、ソケットディレクトリパスを cluster->sockdir に格納する。
  • 読み取ったパスから改行コードを除去する。
  • ポート番号がデフォルト値と異なり、かつ読み取った値と一致しない場合は警告を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/pg_upgrade.c のメイン処理にて old_cluster および new_cluster に対して呼び出される。
  • **呼び出し先**: pg_fatal（エラー時）、pg_log（警告時）、fopen, fgets, sscanf, pg_strdup, pg_strip_crlf 等の標準およびユーティリティ関数。

関連データ構造

  • ClusterInfo: ソケットディレクトリやポート番号を保持する構造体。
  • user_opts: ユーザー指定のオプションを保持するグローバル構造体。

失敗モードと境界条件

  • postmaster.pid が存在しない、または読み取り権限がない場合、pg_fatal によりプログラムが異常終了する。
  • ファイルの読み取り中に予期せぬ EOF に達した場合、pg_fatal により異常終了する。

保守担当者向けメモ

  • 本関数は postmaster.pid のレイアウトに強く依存している。LOCK_FILE_LINE_PORT や LOCK_FILE_LINE_SOCKET_DIR の定義が変更された場合は注意が必要。
  • Windows 環境ではソケットディレクトリの概念がないため、明示的に NULL を設定している。
