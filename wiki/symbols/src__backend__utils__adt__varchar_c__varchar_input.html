varchar_input

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varchar.c:456

目的

文字列入力を varchar(n) 型の内部表現（VarChar）に変換し、指定された型修飾子（atttypmod）に基づく長さ制限の検証と切り詰めを行う。

シグネチャまたは形状

VarChar *varchar_input(const char *s, size_t len, int32 atttypmod, Node *escontext)

入力と出力

  • **入力**:
  • s: 入力文字列のポインタ。
  • len: 入力文字列のバイト長。
  • atttypmod: 型修飾子（最大長 + VARHDRSZ）。
  • escontext: エラー処理用コンテキスト。
  • **出力**:
  • 成功時: 変換された VarChar 構造体へのポインタ。
  • 失敗時: escontext を介したエラー報告（NULL を返す）。

前提条件と不変条件

  • VarChar 型と text 型はバイナリ互換である。
  • atttypmod が VARHDRSZ 以上の場合、有効な長さ制限として扱われる。

アルゴリズムまたは振る舞い

1. atttypmod から最大許容バイト長（maxlen）を算出する。 2. 入力長 len が maxlen を超える場合:

  • マルチバイト文字を考慮し、pg_mbcharcliplen を用いて切り詰め可能な境界を特定する。
  • 切り詰め対象の末尾部分にスペース以外の文字が含まれていないか検証する。
  • スペース以外の文字が含まれる場合は ERRCODE_STRING_DATA_RIGHT_TRUNCATION エラーを発生させる。
  • 検証を通過すれば、末尾のスペースを切り詰めた長さに更新する。

    • 3. cstring_to_text_with_len を呼び出し、文字列を VarChar 構造体に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/varchar.c 内の varcharin および varcharrecv 関数（推測される）。
  • 同ファイル内の他の箇所で、文字列から varchar への変換が必要な際に呼び出される。
  • **呼び出し先**:
  • pg_mbcharcliplen: マルチバイト文字を考慮した長さ計算。
  • ereturn: エラー処理。
  • cstring_to_text_with_len: 内部表現への変換。

関連データ構造

  • VarChar: PostgreSQL の可変長文字列型。
  • Node: エラーコンテキスト用構造体。

失敗モードと境界条件

  • **切り詰めエラー**: 指定された長さを超える文字列で、かつ末尾にスペース以外の文字が含まれる場合にエラーとなる。
  • **メモリ不足**: cstring_to_text_with_len 内部でのメモリ確保失敗時にエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は varchar 型の入力処理の中核であり、text 型とのバイナリ互換性に依存している。
  • マルチバイト文字の切り詰め処理は pg_mbcharcliplen に依存しているため、文字エンコーディングの変更時には注意が必要である。
