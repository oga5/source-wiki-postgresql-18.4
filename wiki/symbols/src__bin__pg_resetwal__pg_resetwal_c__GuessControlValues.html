GuessControlValues

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:659

目的

pg_control ファイルが読み込めない、または破損している場合に、データベースの制御情報をデフォルト値で初期化する。

シグネチャまたは形状

static void GuessControlValues(void)

入力と出力

  • 入力: なし（グローバル変数 ControlFile を直接操作する）
  • 出力: なし（副作用として ControlFile 構造体の各フィールドを初期化し、グローバル変数 guessed を true に設定する）

前提条件と不変条件

  • ControlFile 構造体は、本関数によってゼロクリアされた後、PostgreSQL のバージョンに応じた定数やデフォルト値で埋められる。
  • システム識別子（system_identifier）は、現在時刻とプロセスIDを用いて生成される。

アルゴリズムまたは振る舞い

1. guessed フラグを true に設定する。 2. ControlFile 構造体を memset でゼロクリアする。 3. PG_CONTROL_VERSION および CATALOG_VERSION_NO を設定する。 4. 現在時刻とプロセスIDを組み合わせ、一意な system_identifier を生成する。 5. チェックポイント関連のフィールド（redo, TimeLineID, nextXid, nextOid 等）を初期値で埋める。 6. データベースの状態を DB_SHUTDOWNED に設定する。 7. WAL レベル、接続制限、ブロックサイズ、アライメント等のシステムパラメータをデフォルト値で初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（src/bin/pg_resetwal/pg_resetwal.c:408）
  • 呼び出し先: gettimeofday, getpid, time, FullTransactionIdFromEpochAndXid 等の標準ライブラリおよび内部関数。

関連データ構造

  • ControlFileData (グローバル変数 ControlFile として使用)

失敗モードと境界条件

  • 本関数自体はエラーを返さないが、生成された推測値はあくまでデフォルトであり、実際のデータベースの状態と一致しない可能性がある。
  • 実行には -f (force) オプションが必要となるケースが多い。

保守担当者向けメモ

  • 現在は固定的なデフォルト値を使用しているが、将来的には古い WAL セグメントを解析して、より正確な TimeLineID や nextXID を導出するロジックの追加が検討されている。
