InitializeQueryCompletion

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/cmdtag.c:39

目的

QueryCompletion 構造体を初期化し、コマンドタグを未知の状態に、処理行数をゼロにリセットする。

シグネチャまたは形状

void InitializeQueryCompletion(QueryCompletion *qc)

入力と出力

  • **入力**: QueryCompletion *qc （初期化対象の構造体へのポインタ）
  • **出力**: なし（引数 qc のフィールドを直接更新する）

前提条件と不変条件

  • 引数 qc は有効なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. qc->commandTag に CMDTAG_UNKNOWN を代入する。 2. qc->nprocessed に 0 を代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/spi.c
  • src/backend/tcop/pquery.c
  • **呼び出し先**:
  • 特になし（構造体のメンバへの直接代入のみを行う）

関連データ構造

  • QueryCompletion: コマンド実行結果のメタデータ（コマンドタグと処理行数）を保持する構造体。

失敗モードと境界条件

  • qc が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は QueryCompletion を再利用する際の標準的なリセット手段として使用される。新しいフィールドが QueryCompletion 構造体に追加された場合は、本関数で適切に初期化されているか確認すること。
