get_user_name_or_exit

定義

  • 種別: function
  • 定義箇所: src/common/username.c:73

目的

現在のユーザー名を取得し、取得に失敗した場合は指定されたプログラム名と共にエラーメッセージを標準エラー出力へ書き出し、プロセスを即座に終了（exit(1)）させる。

シグネチャまたは形状

const char *get_user_name_or_exit(const char *progname);

入力と出力

  • **入力**: progname (エラーメッセージの接頭辞として使用されるプログラム名)
  • **出力**: 成功時は現在のユーザー名を表す const char * を返す。失敗時は戻り値を返さず、プロセスが終了する。

前提条件と不変条件

  • 内部で get_user_name を呼び出す。
  • 取得失敗時に生成されるエラーメッセージは palloc されているため、呼び出し元での管理に注意が必要（ただし本関数内で終了するため、実質的なメモリリークの懸念は限定的）。

アルゴリズムまたは振る舞い

1. get_user_name(&errstr) を呼び出し、ユーザー名を取得する。 2. 戻り値が NULL の場合:

  • fprintf(stderr, "%s: %s\n", progname, errstr) を実行し、エラーを標準エラー出力に書き出す。
  • exit(1) を呼び出し、プロセスを終了させる。

    • 3. 戻り値が有効な文字列である場合、そのポインタをそのまま呼び出し元へ返す。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • ユーザー名の解決に失敗した場合、必ず exit(1) で終了する。
  • get_user_name が返すエラーメッセージ（errstr）の内容に依存する。

保守担当者向けメモ

  • 本関数はスレッドセーフではない。
  • Windows環境では get_user_name が静的バッファを使用するため、複数回呼び出した場合、以前の戻り値が上書きされる可能性がある点に注意すること。
