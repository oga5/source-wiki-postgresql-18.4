sepgsql_schema_rename

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/schema.c:228

目的

スキーマの名前変更操作に関連する権限チェックを実行します。

シグネチャまたは形状

void sepgsql_schema_rename(Oid namespaceId)

入力と出力

  • **入力**: namespaceId (Oid) - 名前変更対象となるスキーマの識別子。
  • **出力**: なし。権限チェックに失敗した場合はエラーを発生させます。

前提条件と不変条件

  • 指定された namespaceId が有効なスキーマを参照している必要があります。
  • 内部で呼び出される check_schema_perms により、権限違反時にはトランザクションが中断されます。

アルゴリズムまたは振る舞い

1. 指定された namespaceId に対して、SEPG_DB_SCHEMA__ADD_NAME および SEPG_DB_SCHEMA__REMOVE_NAME 権限の有無を検証します。 2. check_schema_perms の第3引数に true を渡しているため、権限が不足している場合は即座にエラーがスローされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/proc.c:282
  • contrib/sepgsql/relation.c:684
  • **呼び出し先**:
  • check_schema_perms (内部ユーティリティ関数)

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • namespaceId が存在しない場合や、現在のセキュリティコンテキストにおいて必要な権限（ADD_NAME および REMOVE_NAME）が許可されていない場合、check_schema_perms を通じてエラーが発生します。

保守担当者向けメモ

  • 本関数は、スキーマの名前変更が実行される直前のフックとして機能します。
  • 権限チェックのロジックを変更する場合は、check_schema_perms の仕様を確認してください。
