pg_char_to_encoding

定義

  • 種別: function
  • 定義箇所: src/common/encnames.c:551

目的

文字列として与えられたエンコーディング名を、PostgreSQL 内部で使用されるエンコーディング ID (pg_enc) に変換します。

シグネチャまたは形状

int pg_char_to_encoding(const char *name);

入力と出力

  • **入力**: name (エンコーディング名を表す文字列)
  • **出力**: 成功した場合は対応するエンコーディング ID（整数）、失敗した場合は -1 を返します。

前提条件と不変条件

  • name が NULL または空文字列の場合、-1 を返します。
  • name の長さが NAMEDATALEN 以上の場合、テーブルには存在しないとみなされ -1 を返します。
  • 内部テーブル pg_encname_tbl はアルファベット順にソートされている必要があります。

アルゴリズムまたは振る舞い

1. 入力文字列を clean_encoding_name に渡し、英数字のみの小文字に正規化します。 2. 正規化されたキーを用いて、静的テーブル pg_encname_tbl に対して二分探索を行います。 3. 文字列比較には strcmp を使用し、一致した場合は対応する position->encoding を返します。 4. 探索範囲内で一致が見つからない場合は -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/conversioncmds.c
  • src/backend/commands/copy.c
  • src/backend/utils/adt/ascii.c
  • src/backend/utils/adt/xml.c
  • src/backend/utils/mb/mbutils.c
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_upgrade/check.c
  • src/bin/psql/describe.c
  • **呼び出し先**:
  • clean_encoding_name (正規化用)
  • strcmp (標準ライブラリ)

関連データ構造

  • pg_encname_tbl: エンコーディング名と ID のマッピングを保持する静的テーブル。
  • pg_enc: エンコーディング ID を表す型。

失敗モードと境界条件

  • 入力が NULL または空文字の場合、即座に -1 を返します。
  • NAMEDATALEN を超える長さの入力は、探索を行わずに -1 を返します。
  • 該当するエンコーディング名がテーブルに存在しない場合、-1 を返します。

保守担当者向けメモ

  • 本関数は mb/pg_wchar.h にてマクロとしても定義されています。
  • 検索対象の pg_encname_tbl を更新する際は、二分探索が正しく機能するようアルファベット順のソート順序を維持してください。
