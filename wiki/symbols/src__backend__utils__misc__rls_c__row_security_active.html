row_security_active

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/rls.c:141

目的

SQLインターフェースを通じて、指定されたテーブル（OID）に対して行レベルセキュリティ（RLS）が現在有効であるかどうかを判定します。

シグネチャまたは形状

Datum row_security_active(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してテーブルのOIDを受け取ります。
  • **出力**: RLSが有効な場合は true、それ以外の場合は false を Datum 型として返します。

前提条件と不変条件

  • 内部で check_enable_rls を呼び出し、現在のユーザーコンテキストおよびテーブル設定に基づいた判定を行います。
  • check_enable_rls の第3引数に true を渡しているため、エラー抑制が有効な状態で判定が行われます。

アルゴリズムまたは振る舞い

1. 引数からテーブルのOIDを取得します。 2. check_enable_rls(tableoid, InvalidOid, true) を呼び出し、RLSの適用状態（CheckEnableRlsResult）を取得します。 3. 取得したステータスが RLS_ENABLED と一致するかを比較します。 4. 比較結果（真偽値）を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: check_enable_rls（同ファイル内）。
  • **呼び出し元**: SQLエンジンからの直接呼び出しが想定されますが、ソースコード内の明示的な呼び出し元情報は提供されていません。

関連データ構造

  • CheckEnableRlsResult: check_enable_rls が返す列挙型。
  • RLS_ENABLED: CheckEnableRlsResult の値の一つ。

失敗モードと境界条件

  • check_enable_rls が内部的にシステムカタログ（pg_class）へのアクセスを行うため、無効なOIDが指定された場合やカタログアクセスに失敗した場合は、その関数に準じたエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はSQLからRLSの状態を確認するためのラッパーです。ロジックの変更は check_enable_rls に集約されています。
  • RLS_NONE_ENV などの他のステータスは、本関数では false として扱われます。
