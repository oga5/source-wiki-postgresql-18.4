JsonbSubWorkspace

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonbsubs.c:27

目的

jsonb 型に対する添字（subscripting）操作の実行時において、添字の型情報や変換後の値を保持するためのワークスペース構造体です。

シグネチャまたは形状

typedef struct JsonbSubWorkspace
{
	bool		expectArray;	/* jsonb root is expected to be an array */
	Oid		   *indexOid;		/* OID of coerced subscript expression */
	Datum	   *index;			/* Subscript values in Datum format */
} JsonbSubWorkspace;

入力と出力

本構造体は直接的な関数ではありませんが、実行時に SubscriptingRefState の workspace フィールドとして割り当てられ、以下の情報を保持します。

  • **入力**: 添字式が評価された結果（Datum）およびその型情報（Oid）。
  • **出力**: jsonb 構造に対する要素取得（fetch）や更新（assign）処理に必要な正規化されたパス情報。

前提条件と不変条件

  • indexOid および index は、添字の数に応じた配列として確保されます。
  • indexOid に格納される型は、int4（配列インデックス用）または text（オブジェクトキー用）のいずれかである必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: jsonb_exec_setup 内で palloc0 を用いてメモリが確保されます。この際、構造体自体のサイズに加えて、添字の数に応じた配列領域が確保されます。 2. **保持**: 実行フェーズにおいて、jsonb_subscript_check_subscripts がユーザー指定の添字を評価し、その結果を本構造体の index 配列に格納します。 3. **利用**: jsonb_subscript_fetch および jsonb_subscript_assign が、このワークスペースを参照して jsonb 内部の対象要素を特定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonbsubs.c: jsonb_exec_setup（確保）、jsonb_subscript_check_subscripts、jsonb_subscript_fetch、jsonb_subscript_assign（参照）。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • SubscriptingRefState: 実行時の添字操作状態を管理する親構造体。本構造体はこの workspace フィールドに格納されます。

失敗モードと境界条件

  • メモリ不足: palloc0 によるメモリ確保時に失敗する可能性があります。
  • 型不整合: indexOid が int4 または text 以外の場合、後続の jsonb 操作関数で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は jsonb の添字操作専用です。スライス操作には対応していません。
  • index 配列のサイズは、実行時の添字の数に依存するため、動的なメモリレイアウトを前提としています。変更時は jsonb_exec_setup での確保ロジックと整合性をとる必要があります。
