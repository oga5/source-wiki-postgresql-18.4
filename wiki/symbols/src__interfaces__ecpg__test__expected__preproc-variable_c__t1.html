定義

目的

ECPGプリプロセッサのテストスイートにおいて、SQL文とC言語構造体間のマッピングを検証するためのデータ構造です。

シグネチャまたは形状

struct t1 {
    struct varchar_2 {
        int len;
        char arr[BUFFERSIZ];
    } name;
};

入力と出力

  • 入力: データベースから取得した文字列データが name.arr に格納され、その長さが name.len に設定されます。
  • 出力: ECPGの EXEC SQL 文を通じて、データベースへの挿入や更新操作のホスト変数として利用されます。

前提条件と不変条件

  • BUFFERSIZ マクロで定義されたサイズが name.arr の容量として確保されている必要があります。
  • ECPGプリプロセッサによって生成されたコードであるため、構造体のメモリレイアウトはECPGの型変換規則に準拠している必要があります。

アルゴリズムまたは振る舞い

ECPGの EXEC SQL TYPE 指令やホスト変数バインドのテストにおいて、データベースの行データを受け取るコンテナとして機能します。ECPGdo 関数呼び出し時に、この構造体のメンバがSQL文のパラメータとして渡されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/preproc-variable.c 内のテストロジック。
  • 呼び出し先: ECPGライブラリ関数（ECPGdo 等）。
  • 注意: ツールが提示した contrib/ 配下の t1 への参照は、本シンボルとは無関係な同名の別識別子である可能性が高いです。

関連データ構造

  • varchar_2: t1 構造体内部で定義されているネストされた構造体。
  • t2: 同様に定義された対となる構造体。

失敗モードと境界条件

  • name.len が BUFFERSIZ を超えるデータを扱おうとした場合、バッファオーバーフローやデータ切り捨てが発生する可能性があります。
  • データベース接続が確立されていない状態で本構造体を使用すると、ECPGライブラリ経由でエラーが返されます。
