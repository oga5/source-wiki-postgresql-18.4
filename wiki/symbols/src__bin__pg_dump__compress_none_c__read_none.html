read_none

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_none.c:86

目的

圧縮を行わない（None）モードにおいて、CompressFileHandle が保持するファイルストリームからデータを読み込むための抽象化関数です。

シグネチャまたは形状

static size_t read_none(void *ptr, size_t size, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • ptr: 読み込んだデータを格納するバッファへのポインタ。
  • size: 読み込むバイト数。
  • CFH: CompressFileHandle 構造体へのポインタ。private_data メンバに FILE* が格納されていることを前提とします。
  • **出力**:
  • 実際に読み込まれたバイト数を size_t 型で返します。

前提条件と不変条件

  • CFH->private_data は有効な FILE* である必要があります。
  • 読み込み処理中にファイルエラーが発生した場合、pg_fatal を呼び出しプロセスを終了させます。

アルゴリズムまたは振る舞い

1. CFH->private_data から FILE* を取得します。 2. fread を使用して、指定されたサイズ分を ptr に読み込みます。 3. ferror を用いてファイル操作の成否を確認し、エラーが発生している場合は pg_fatal で致命的なエラーとして報告します。 4. 読み込んだバイト数を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_none.c 内の InitCompressFileHandleNone 関数において、CFH->read_func にこの関数が割り当てられます。
  • **呼び出し先**: 標準ライブラリの fread および ferror を呼び出します。また、エラー時には pg_fatal を呼び出します。

関連データ構造

  • CompressFileHandle: 圧縮・非圧縮の抽象化レイヤーで使用される構造体。本関数はこの構造体の private_data を介してファイルポインタにアクセスします。

失敗モードと境界条件

  • ファイル読み込み中にエラーが発生した場合、pg_fatal によりプログラムが終了します。
  • fread の仕様に従い、EOF（ファイル終端）に達した場合は、読み込めたバイト数が要求サイズより少なくなる可能性があります。

保守担当者向けメモ

  • 本関数は compress_io モジュールが期待するインターフェースに適合させるためのラッパーです。
  • private_data メンバには常に FILE* が格納される設計となっているため、型キャストの安全性は InitCompressFileHandleNone での初期化に依存しています。
