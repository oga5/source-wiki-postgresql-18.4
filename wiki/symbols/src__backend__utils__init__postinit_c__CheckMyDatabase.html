CheckMyDatabase

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:322

目的

データベース接続時の検証（権限、接続制限、ロケール設定）および、システムカタログからのデータベース情報の読み込みと適用を行う。

シグネチャまたは形状

static void CheckMyDatabase(const char *name, bool am_superuser, bool override_allow_connections)

入力と出力

  • **入力**:
  • name: 接続対象のデータベース名。
  • am_superuser: 呼び出し元がスーパーユーザーであるかを示すフラグ。
  • override_allow_connections: 接続許可チェックをスキップするかどうかを示すフラグ。
  • **出力**: なし（成功時は正常終了、失敗時は FATAL エラーでプロセスを終了）。

前提条件と不変条件

  • MyDatabaseId が正しく設定されていること。
  • システムカタログキャッシュ（DATABASEOID）が利用可能であること。
  • 呼び出し前に ACL フレームワークが初期化されていること。

アルゴリズムまたは振る舞い

1. **カタログ検索**: pg_database システムカタログから対象データベースのタプルを取得。 2. **整合性チェック**: データベース名が OID と一致するか確認。 3. **接続許可チェック**: IsUnderPostmaster が真の場合、datallowconn および override_allow_connections に基づき接続可否を判定。 4. **権限チェック**: スーパーユーザーでない場合、ACL_CONNECT 権限を検証。 5. **接続数制限**: 通常のバックエンドプロセスの場合、datconnlimit を超えていないか確認。 6. **エンコーディング設定**: SetDatabaseEncoding を呼び出し、server_encoding および client_encoding を GUC に反映。 7. **ロケール設定**: LC_COLLATE および LC_CTYPE を適用し、OS との互換性を検証。 8. **照合順序（Collation）検証**: データベースの照合順序バージョンをチェックし、不一致がある場合は警告を出力。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/init/postinit.c 内の InitPostgres 関数（1188行目付近）。
  • **呼び出し先**: SearchSysCache1, ereport, SetDatabaseEncoding, SetConfigOption, pg_perm_setlocale, init_database_collation, get_collation_actual_version 等の内部ユーティリティ。

関連データ構造

  • Form_pg_database: pg_database カタログの行構造体。
  • HeapTuple: システムカタログから取得したタプル。

失敗モードと境界条件

  • **FATAL エラー**: データベースが存在しない、接続が許可されていない、権限不足、接続数上限超過、またはロケールが OS でサポートされていない場合に発生。
  • **警告**: 照合順序のバージョン不一致時に発生。
  • **競合**: 接続数制限のチェックには、PGPROC 作成後のレースコンディションが存在するため、制限値は近似値として扱われる。

保守担当者向けメモ

  • 本関数は InitPostgres の初期化フローの一部であり、カタログキャッシュ構築後に実行される必要がある。
  • override_allow_connections はバックグラウンドプロセスが接続制限を回避するために使用される。
  • ロケール設定の失敗はデータベースの動作に致命的な影響を与えるため、FATAL で停止する設計となっている。
