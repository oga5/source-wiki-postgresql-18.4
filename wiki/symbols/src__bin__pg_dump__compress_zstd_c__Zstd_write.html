Zstd_write

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_zstd.c:354

目的

CompressFileHandle を介して提供されたデータを Zstandard アルゴリズムで圧縮し、関連付けられたファイルストリームへ書き出す。

シグネチャまたは形状

static void Zstd_write(const void *ptr, size_t size, CompressFileHandle *CFH)

入力と出力

  • ptr: 書き込み対象のデータバッファへのポインタ。
  • size: 書き込むデータのバイト数。
  • CFH: 圧縮状態とファイルポインタを保持する CompressFileHandle 構造体。
  • 出力: なし（副作用として CFH 内のファイルストリームへ圧縮データを書き出す）。

前提条件と不変条件

  • CFH->private_data は ZstdCompressorState 型としてキャスト可能であること。
  • ZSTD_compressStream2 がエラーを返さない限り、入力データはすべて消費されるまでループ処理される。

アルゴリズムまたは振る舞い

1. CFH->private_data から ZstdCompressorState を取得する。 2. zstdcs->cstream が未初期化の場合、ZSTD_CStreamOutSize に基づいて出力バッファを確保し、_ZstdCStreamParams を用いて圧縮ストリームを初期化する。 3. 入力データがすべて処理されるまで、以下のループを実行する：

  • ZSTD_compressStream2 を呼び出し、データを圧縮して output->dst に格納する。
  • 圧縮エラーが発生した場合は pg_fatal で終了する。
  • fwrite を使用して、圧縮されたチャンクを zstdcs->fp へ書き出す。書き込みバイト数が期待値と異なる場合は pg_fatal で終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/compress_zstd.c 内で CFH->write_func に代入されており、pg_dump のアーカイブ書き込み処理から間接的に呼び出される。
  • 呼び出し先: ZSTD_compressStream2 (Zstd ライブラリ), fwrite (標準ライブラリ), pg_malloc0, pg_fatal。

関連データ構造

  • CompressFileHandle: 圧縮処理の抽象化ハンドル。
  • ZstdCompressorState: Zstd 固有の圧縮コンテキスト（ストリーム、バッファ、ファイルポインタ）。

失敗モードと境界条件

  • 圧縮ストリームの初期化失敗: _ZstdCStreamParams が NULL を返した場合、pg_fatal により異常終了する。
  • 圧縮処理エラー: ZSTD_isError が真となる場合、エラー名と共に pg_fatal で異常終了する。
  • ファイル書き込み失敗: fwrite の戻り値が期待値と一致しない場合、errno に基づくエラーメッセージと共に pg_fatal で異常終了する。

保守担当者向けメモ

  • 本関数は USE_ZSTD が定義されている環境でのみ有効である。
  • input->pos が input->size に達するまでループを継続する設計であり、部分的な書き込みが発生しないよう制御されている。
