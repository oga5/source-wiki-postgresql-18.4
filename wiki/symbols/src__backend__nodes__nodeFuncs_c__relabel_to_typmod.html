relabel_to_typmod

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/nodeFuncs.c:688

目的

既存の式ノードに対して、新しい型修飾子（typmod）を適用した RelabelType ノードを生成または適用する。

シグネチャまたは形状

Node *relabel_to_typmod(Node *expr, int32 typmod)

入力と出力

  • **入力**:
  • expr: 変換対象の式ノード（Node *）。
  • typmod: 適用する新しい型修飾子（int32）。
  • **出力**:
  • 変換後の式ノード（Node *）。内部で applyRelabelType を呼び出し、その結果を返す。

前提条件と不変条件

  • 入力 expr は有効な式ノードであること。
  • 内部的に exprType(expr) および exprCollation(expr) を使用して、現在の式の型と照合順序を抽出する。

アルゴリズムまたは振る舞い

1. exprType(expr) を呼び出し、式の現在の型 OID を取得する。 2. exprCollation(expr) を呼び出し、式の現在の照合順序を取得する。 3. applyRelabelType を以下の引数で呼び出す:

  • expr: 対象ノード。
  • exprType(expr) の結果: 現在の型。
  • typmod: 指定された新しい型修飾子。
  • exprCollation(expr) の結果: 現在の照合順序。
  • COERCE_EXPLICIT_CAST: 変換の強制レベル。
  • -1: 位置情報（指定なし）。
  • false: 実行時の型チェックフラグ。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/datetime.c:4980
  • src/backend/utils/adt/numeric.c:1231
  • src/backend/utils/adt/timestamp.c:1316
  • src/backend/utils/adt/varbit.c:726
  • src/backend/utils/adt/varchar.c:589
  • **呼び出し先**:
  • applyRelabelType (同ファイル内)
  • exprType (同ファイル内)
  • exprCollation (同ファイル内)

関連データ構造

  • Node: PostgreSQL の構文解析木や計画木の基本構造体。
  • RelabelType: applyRelabelType によって生成されるノード型。

失敗モードと境界条件

  • expr が NULL の場合の挙動については、exprType および exprCollation の実装に依存する。
  • typmod が対象の型に対して無効な値である場合、applyRelabelType の内部ロジックに従いエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、データ型変換の際に型修飾子のみを変更するラッパーとして機能する。
  • 変換ロジックの変更が必要な場合は、applyRelabelType の実装を確認すること。
