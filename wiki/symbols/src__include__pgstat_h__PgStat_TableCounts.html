PgStat_TableCounts

定義

  • 種別: struct
  • 定義箇所: src/include/pgstat.h:137

目的

バックエンドプロセスがテーブル単位で収集する統計情報を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct PgStat_TableCounts
{
	PgStat_Counter numscans;
	PgStat_Counter tuples_returned;
	PgStat_Counter tuples_fetched;
	PgStat_Counter tuples_inserted;
	PgStat_Counter tuples_updated;
	PgStat_Counter tuples_deleted;
	PgStat_Counter tuples_hot_updated;
	PgStat_Counter tuples_newpage_updated;
	bool		truncdropped;
	PgStat_Counter delta_live_tuples;
	PgStat_Counter delta_dead_tuples;
	PgStat_Counter changed_tuples;
	PgStat_Counter blocks_fetched;
	PgStat_Counter blocks_hit;
} PgStat_TableCounts;

入力と出力

  • 入力: 各種テーブル操作（スキャン、タプル操作、ブロックアクセス）が発生するたびに、対応するカウンタがインクリメントされます。
  • 出力: 統計情報のフラッシュ時に、この構造体の内容が共有メモリ上の統計エントリへ反映されます。

前提条件と不変条件

  • PgStat_Counter 型（int64）を使用して統計を蓄積します。
  • truncdropped はテーブルの切り詰めや削除が発生したかを示すフラグとして機能します。

アルゴリズムまたは振る舞い

バックエンドローカルでテーブルごとのイベント発生回数を蓄積し、pgstat_report_stat() が呼び出された際に共有メモリへ情報を転送することで、統計システム全体への反映を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/activity/pgstat_relation.c において、統計情報のサイズ計算や管理に使用されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • PgStat_TableStatus: バックエンドローカルで保持されるテーブル統計のステータス構造体。
  • PgStat_Counter: 統計カウンタに使用される int64 型。

失敗モードと境界条件

  • 統計情報のフラッシュが失敗した場合、ローカルのカウンタと共有メモリ上の値に乖離が生じる可能性があります。
  • 構造体の定義変更時には、PGSTAT_FILE_FORMAT_ID の更新が必要です。

保守担当者向けメモ

  • 本構造体は PgStat_TableStatus の一部として使用されます。
  • 統計情報の正確性を維持するため、新しい統計項目を追加する際は、関連する pgstat_relation.c の更新を忘れないようにしてください。
