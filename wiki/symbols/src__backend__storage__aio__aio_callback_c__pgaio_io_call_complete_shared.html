pgaio_io_call_complete_shared

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_callback.c:224

目的

非同期I/O（AIO）操作の完了時、共有リソース（バッファ等）に関連する後処理を順次実行し、結果を集約する。

シグネチャまたは形状

void pgaio_io_call_complete_shared(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh（処理対象のAIOハンドル）
  • **出力**: なし（副作用として ioh->distilled_result を更新）

前提条件と不変条件

  • START_CRIT_SECTION() によりクリティカルセクション内で実行される。
  • ioh->target が有効な範囲（PGAIO_TID_INVALID 〜 PGAIO_TID_COUNT）にあること。
  • ioh->op が有効な範囲（PGAIO_OP_INVALID 〜 PGAIO_OP_COUNT）にあること。
  • 各コールバックの実行結果 result.status は PGAIO_RS_UNKNOWN になってはならない。

アルゴリズムまたは振る舞い

1. PgAioResult を初期化し、低レベルI/Oの結果をセットする。 2. ハンドルに登録されたコールバックを、登録順序の逆順（スタック的）にループ処理する。 3. 各コールバックの complete_shared 関数を呼び出し、戻り値の result を次のコールバックへ引き継ぐ。 4. 最終的な集約結果を ioh->distilled_result に格納する。 5. END_CRIT_SECTION() を呼び出し、クリティカルセクションを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio.c:540
  • **呼び出し先**:
  • 登録された各コールバックの complete_shared 関数（aio_handle_cbs 配列経由）
  • pgaio_debug_io（デバッグログ出力）

関連データ構造

  • PgAioHandle: I/O操作の状態とコールバック情報を保持。
  • PgAioResult: I/O完了後の状態とエラー情報を保持。
  • PgAioHandleCallbacksEntry: コールバック関数の定義テーブル。
  • aio_handle_cbs: 利用可能なコールバックの静的配列。

失敗モードと境界条件

  • コールバックが登録されていない場合、ループはスキップされる。
  • complete_shared が定義されていないコールバックは無視される。
  • 低レベルI/Oの結果は常に PGAIO_RS_OK として初期化される。

保守担当者向けメモ

  • 本関数は pgaio_io_call_complete_local と類似の構造を持つため、将来的な重複排除が推奨される。
  • コールバックの実行順序は、後から登録されたものが先に結果を加工する仕様である。
