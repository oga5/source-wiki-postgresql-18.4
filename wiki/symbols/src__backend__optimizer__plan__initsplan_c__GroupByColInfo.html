GroupByColInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/initsplan.c:88

目的

GROUP BY 句に含まれるカラムの情報を保持し、一意性制約に基づいた冗長な GROUP BY カラムの削除（最適化）を支援するために使用されます。

シグネチャまたは形状

typedef struct GroupByColInfo
{
    AttrNumber  attno;          /* var->varattno */
    List       *eq_opfamilies;  /* mergejoin opfamilies of sgc->eqop */
    Oid         coll;           /* var->varcollid */
} GroupByColInfo;

入力と出力

  • 本構造体は、remove_useless_groupby_columns 関数内で palloc を用いて動的に生成されます。
  • attno には対象カラムの属性番号、eq_opfamilies には等価演算子の演算子族、coll には照合順序（Collation）の OID が格納されます。

前提条件と不変条件

  • remove_useless_groupby_columns において、GROUP BY カラムがテーブルの一意性インデックスと照合可能であるかを判断するためのメタデータとして機能します。

アルゴリズムまたは振る舞い

  • remove_useless_groupby_columns 関数内で、クエリの GROUP BY 句を走査する際に、各カラムの属性情報を抽出して本構造体に格納します。
  • 格納された情報は、後続の処理で一意性制約との比較を行う際の比較キーとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/initsplan.c 内の remove_useless_groupby_columns 関数が、本構造体のインスタンスを生成および参照しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • AttrNumber: PostgreSQL の属性番号型。
  • List: PostgreSQL の汎用リスト構造。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • eq_opfamilies が空リストである場合、等価性の判定において特定の最適化が適用されない可能性があります。

保守担当者向けメモ

  • 本構造体は initsplan.c 内でのみ使用される一時的なデータ構造です。
  • GROUP BY の最適化ロジックを変更する際は、attno や coll の整合性がプランナの他の部分（特に pathnode.h 関連）と矛盾しないよう注意してください。
