proclist_contains_offset

定義

  • 種別: function
  • 定義箇所: src/include/storage/proclist.h:145

目的

指定されたプロセス（procno）が、特定の proclist_head が管理する双方向連結リストに含まれているかを判定します。

シグネチャまたは形状

static inline bool proclist_contains_offset(const proclist_head *list, int procno, size_t node_offset)

入力と出力

  • **入力**:
  • list: 検索対象のリスト構造体へのポインタ。
  • procno: 検索対象のプロセス番号。
  • node_offset: PGPROC 構造体内の proclist_node メンバへのオフセット。
  • **出力**:
  • bool: ノードがリストに含まれていると判断された場合に true を返します。

前提条件と不変条件

  • proclist_node_get を介して対象ノードが解決可能であること。
  • リストの整合性を確認するためのアサーション（Assert）が有効なビルドでは、ノードがリストの先頭または末尾である場合に、リストの head または tail と procno が一致することが検証されます。

アルゴリズムまたは振る舞い

1. proclist_node_get を使用して、指定された procno と node_offset から共有メモリ上の proclist_node を取得します。 2. ノードの prev および next が共に 0（INVALID_PROC_NUMBER）である場合、そのノードはいかなるリストにも属していないと判断し false を返します。 3. それ以外の場合、リストに含まれていると見なし true を返します。 4. アサーション有効時には、ノードがリストの端（先頭または末尾）にある場合に限り、リスト構造との整合性を O(1) で検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/storage/proclist.h 内で利用されています。
  • **呼び出し先**: proclist_node_get を呼び出します。

関連データ構造

  • proclist_head: リストの先頭と末尾を保持する構造体。
  • proclist_node: リストの前後関係を保持するノード構造体。
  • PGPROC: プロセス情報を保持する構造体。

失敗モードと境界条件

  • ノードがどのリストにも属していない場合、false を返します。

保守担当者向けメモ

  • 本関数はスピンロックを保持した状態で呼び出されることが多いため、リスト全体を走査するような実装は避ける必要があります。
  • アサーションによる検証は、デバッグ目的でリストの端の整合性を確認するために提供されています。
