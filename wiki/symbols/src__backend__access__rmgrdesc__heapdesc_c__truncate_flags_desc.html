truncate_flags_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/heapdesc.c:54

目的

ヒープのTRUNCATE操作に関連するWALレコードのフラグビットを、人間が読み取れる文字列形式に変換してバッファへ出力します。

シグネチャまたは形状

static void truncate_flags_desc(StringInfo buf, uint8 flags)

入力と出力

  • **入力**:
  • buf: 出力先の StringInfo バッファ。
  • flags: 解析対象の uint8 型フラグ値。
  • **出力**: なし（buf の内容が更新されます）。

前提条件と不変条件

  • buf は初期化済みである必要があります。
  • flags に含まれるビットが XLH_TRUNCATE_CASCADE または XLH_TRUNCATE_RESTART_SEQS のいずれか、あるいは両方に対応していることを前提とします。

アルゴリズムまたは振る舞い

1. flags: [ という文字列をバッファに追加します。 2. XLH_TRUNCATE_CASCADE ビットが立っていれば "CASCADE, " を追加します。 3. XLH_TRUNCATE_RESTART_SEQS ビットが立っていれば "RESTART_SEQS, " を追加します。 4. 末尾に , が残っている場合（フラグが1つ以上追加された場合）、その2文字を削除して整形します。 5. 最後に ] を追加して閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/rmgrdesc/heapdesc.c 内の heap2_desc 関数から呼び出されます（xlrec->flags を引数として渡す箇所）。
  • **呼び出し先**: appendStringInfoString および appendStringInfoChar（lib/stringinfo.h）。

関連データ構造

  • StringInfo: PostgreSQLの動的文字列バッファ。
  • xl_heap_truncate: ヒープTRUNCATE操作のWALレコード構造体（access/heapam_xlog.h で定義）。

失敗モードと境界条件

  • フラグが0の場合、flags: [] と出力されます。
  • バッファの末尾がスペースであるかを確認する際、Assert により , の存在を検証しています。

保守担当者向けメモ

  • 新しいTRUNCATE関連のフラグが追加された場合は、本関数内の条件分岐に追加してください。
  • StringInfo の操作において、末尾のカンマ削除処理はバッファの整合性に依存するため、変更時は buf->len の操作に注意してください。
