unicode_category

定義

  • 種別: function
  • 定義箇所: src/common/unicode_category.c:84

目的

指定された Unicode コードポイントの一般カテゴリ（General Category）を判定し、対応する列挙値を返します。

シグネチャまたは形状

pg_unicode_category unicode_category(pg_wchar code)

入力と出力

  • **入力**: pg_wchar code（判定対象の Unicode コードポイント）
  • **出力**: pg_unicode_category 型の列挙値（カテゴリ情報）

前提条件と不変条件

  • code は 0x10ffff 以下である必要があります（Assert により検証）。
  • Unicode 規格の安定性ポリシーに基づき、カテゴリ値は固定されています。

アルゴリズムまたは振る舞い

1. **ASCII 最適化**: code が 0x80 未満の場合、unicode_opt_ascii テーブルを参照して定数時間で結果を返します。 2. **二分探索**: 0x80 以上のコードポイントに対しては、unicode_categories テーブルに対して二分探索を実行します。 3. **探索失敗時**: テーブル内に該当する範囲が見つからない場合、PG_U_UNASSIGNED を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/varlena.c
  • src/common/unicode/case_test.c
  • src/common/unicode/category_test.c
  • **呼び出し先**:
  • unicode_opt_ascii（テーブル）
  • unicode_categories（テーブル）
  • lengthof() マクロ

関連データ構造

  • pg_wchar: ワイド文字型。
  • pg_unicode_category: カテゴリを表す列挙型。
  • unicode_categories: カテゴリ判定用の静的データテーブル（common/unicode_category_table.h で定義）。

失敗モードと境界条件

  • code > 0x10ffff: アサーション違反が発生します。
  • 該当するカテゴリが存在しない場合: PG_U_UNASSIGNED が返されます。

保守担当者向けメモ

  • 本関数はフロントエンドとバックエンドの両方から利用されるため、変更時は両環境への影響を考慮してください。
