pg_encrypt_iv

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:348

目的

指定されたアルゴリズム、鍵、および初期化ベクトル（IV）を使用して、データを対称鍵暗号化する SQL インターフェース関数。

シグネチャまたは形状

Datum pg_encrypt_iv(PG_FUNCTION_ARGS)

  • 引数: SQL 経由で bytea (データ), bytea (鍵), bytea (IV), text (アルゴリズム名) を受け取る。
  • 戻り値: 暗号化された bytea 型データ。

入力と出力

  • 入力:
  • PG_GETARG_BYTEA_PP(0): 暗号化対象データ
  • PG_GETARG_BYTEA_PP(1): 暗号化鍵
  • PG_GETARG_BYTEA_PP(2): 初期化ベクトル (IV)
  • PG_GETARG_TEXT_PP(3): 暗号化アルゴリズム名
  • 出力: 暗号化されたバイナリデータ (bytea) を返す。

前提条件と不変条件

  • find_provider により、指定されたアルゴリズムに対応する PX_Combo プロバイダーが取得可能であること。
  • PostgreSQL のメモリコンテキスト内で動作し、palloc によるメモリ確保が成功すること。

アルゴリズムまたは振る舞い

1. find_provider を呼び出し、指定されたアルゴリズムの暗号化コンテキスト (PX_Combo) を取得する。 2. px_combo_encrypt_len で必要な出力バッファサイズを計算し、palloc でメモリを確保する。 3. px_combo_init を呼び出し、鍵と IV を設定する。 4. px_combo_encrypt を呼び出し、データを暗号化する。 5. px_combo_free でコンテキストを解放する。 6. エラー発生時は px_strerror を用いて例外をスローする。 7. PG_FREE_IF_COPY を使用して、入力引数のメモリを適切に解放する。

呼び出し元と呼び出し先

  • 呼び出し元: SQL インターフェース（PG_FUNCTION_INFO_V1(pg_encrypt_iv) により登録）。
  • 呼び出し先: find_provider, px_combo_init, px_combo_encrypt, px_combo_free, px_strerror（いずれも px レイヤーの関数）。

関連データ構造

  • PX_Combo: 暗号化処理の抽象化コンテキスト。
  • bytea, text: PostgreSQL のデータ型。

失敗モードと境界条件

  • アルゴリズム名が不正な場合、find_provider がエラーを発生させる。
  • px_combo_init または px_combo_encrypt が失敗した場合、px_strerror を含む ereport(ERROR, ...) が発生し、トランザクションが中断される。

保守担当者向けメモ

  • 本関数は SQL インターフェースと px レイヤーの仲介役です。暗号化アルゴリズムの具体的な実装については px.c や openssl.c を参照してください。
  • メモリリークを防ぐため、PG_FREE_IF_COPY の呼び出し順序や対象に変更を加える際は注意してください。
