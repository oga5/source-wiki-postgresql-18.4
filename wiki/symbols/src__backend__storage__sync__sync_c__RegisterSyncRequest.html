RegisterSyncRequest

定義

  • 種別: function
  • 定義箇所: src/backend/storage/sync/sync.c:579

目的

ファイル同期（fsync）や削除（unlink）などの要求を管理システムに登録する。要求元がチェックポインタプロセスであるか、それ以外のバックエンドであるかに応じて、要求をローカルキューへ追加するか、チェックポインタへ転送する。

シグネチャまたは形状

bool RegisterSyncRequest(const FileTag *ftag, SyncRequestType type, bool retryOnError)

入力と出力

  • **入力**:
  • ftag: 対象ファイルを識別する FileTag 構造体へのポインタ。
  • type: 要求の種類（SyncRequestType）。
  • retryOnError: キューイング失敗時にリトライを行うかどうかのフラグ。
  • **出力**:
  • bool: 要求の登録（または転送）が成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • pendingOps が NULL の場合、スタンドアロンバックエンドまたはスタートアッププロセスとして動作しており、同期状態はローカルで管理される。
  • retryOnError が true の場合、チェックポインタへの転送が失敗すると、成功するまで WaitLatch を用いて待機とリトライを繰り返す。

アルゴリズムまたは振る舞い

1. pendingOps が存在する場合、RememberSyncRequest を呼び出し、ローカルキューに要求を追加して終了する。 2. pendingOps が存在しない場合（チェックポインタ以外からの呼び出し）、ForwardSyncRequest を呼び出してチェックポインタへ要求を転送する。 3. 転送が失敗し、かつ retryOnError が true の場合、WaitLatch で 10ms 待機した後に再度転送を試みる。 4. 転送成功、またはリトライ不要と判断された時点で処理を終了し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/slru.c
  • src/backend/catalog/storage.c
  • src/backend/storage/smgr/md.c
  • **呼び出し先**:
  • RememberSyncRequest (ローカル管理時)
  • ForwardSyncRequest (転送時)
  • WaitLatch (リトライ待機時)

関連データ構造

  • FileTag: 同期対象ファイルを識別する構造体。
  • SyncRequestType: 要求の種類を定義する列挙型。
  • pendingOps: ローカルでの同期要求を管理するハッシュテーブル。

失敗モードと境界条件

  • ForwardSyncRequest が失敗した場合、retryOnError が false であれば即座に false を返して終了する。
  • retryOnError が true の場合、チェックポインタがキューを空にするまでループが継続する可能性がある。

保守担当者向けメモ

  • ForwardSyncRequest ループ内での CHECK_FOR_INTERRUPTS の必要性については検討の余地がある。特に SYNC_UNLINK_REQUEST の場合、エラーでループを抜けるとファイルがディスク上に残存するリスクがあるため、チェックポインタが確実にキューを処理することを前提としている。
