icount

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_op.c:187

目的

PostgreSQLの整数配列（intarray）に含まれる要素数をカウントし、その結果を int32 型として返します。

シグネチャまたは形状

Datum icount(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される ArrayType 型の配列。
  • **出力**: 配列内の要素数を示す int32 値。

前提条件と不変条件

  • 入力された配列は ArrayType として有効である必要があります。
  • PG_GETARG_ARRAYTYPE_P を使用して配列を取得します。

アルゴリズムまたは振る舞い

1. PG_GETARG_ARRAYTYPE_P(0) を呼び出し、引数から配列ポインタを取得します。 2. ARRNELEMS(a) マクロを使用して、配列内の要素数を取得します。 3. PG_FREE_IF_COPY(a, 0) を呼び出し、必要に応じてメモリを解放します。 4. PG_RETURN_INT32(count) を使用して、計算された要素数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから直接呼び出されます（PG_FUNCTION_INFO_V1(icount) により登録）。
  • **呼び出し先**: PostgreSQLの内部マクロ（ARRNELEMS, PG_GETARG_ARRAYTYPE_P, PG_FREE_IF_COPY, PG_RETURN_INT32）を使用しています。

関連データ構造

  • ArrayType: PostgreSQLの標準的な配列データ構造。

失敗モードと境界条件

  • 入力が有効な配列でない場合、ARRNELEMS マクロの挙動に依存します。
  • 空の配列が渡された場合、要素数として 0 を返します。

保守担当者向けメモ

  • 本関数は単純な要素数カウントを行うラッパーです。配列の正規化（ソートや重複排除）は行いません。
  • メモリ管理のため、PG_FREE_IF_COPY を適切に使用してコピーされた配列の解放を行っています。
