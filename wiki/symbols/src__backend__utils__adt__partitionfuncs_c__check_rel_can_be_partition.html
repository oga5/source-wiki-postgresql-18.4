check_rel_can_be_partition

定義

  • static bool check_rel_can_be_partition(Oid relid)
  • 定義箇所: src/backend/utils/adt/partitionfuncs.c:33

目的

指定されたリレーション（OID）がパーティションツリーの一部として有効であるかを検証します。

シグネチャまたは形状

  • 引数: Oid relid（検証対象のリレーションID）
  • 戻り値: bool（有効な場合は true、存在しないかパーティションツリーに含まれない場合は false）

入力と出力

  • 入力: relid を使用してシステムカタログ（pg_class）を検索します。
  • 出力: 判定結果をブール値で返します。エラーは発生させません。

前提条件と不変条件

  • SearchSysCacheExists1 を使用して、リレーションがシステムカタログに存在することを確認します。
  • get_rel_relkind および get_rel_relispartition を使用してリレーションの属性を取得します。

アルゴリズムまたは振る舞い

1. RELOID システムキャッシュを検索し、リレーションが存在しない場合は false を返します。 2. リレーションの relkind（種類）と relispartition（パーティションフラグ）を取得します。 3. 以下のいずれかの条件を満たす場合に true を返します。

  • relispartition が真である。
  • RELKIND_HAS_PARTITIONS(relkind) マクロが真である（パーティションを保持可能なリレーション型である）。

    • 4. 上記のいずれでもない場合は false を返します。

関連データ構造

  • Oid: リレーションの識別子。
  • pg_class: システムカタログ（キャッシュ経由でアクセス）。

失敗モードと境界条件

  • 指定された relid が存在しない場合、false を返します。
  • パーティションツリーに関与しないリレーション型（通常のテーブルやインデックスなど、パーティション属性を持たないもの）に対しては false を返します。

保守担当者向けメモ

  • 本関数はエラーを投げない設計であるため、呼び出し側で戻り値を確認し、必要に応じて適切なエラーハンドリングを行う必要があります。
