DynamicZoneAbbrev

定義

  • 種別: struct
  • 定義箇所: src/include/utils/datetime.h:224

目的

タイムゾーンの省略形（abbreviation）に関連付けられた動的なタイムゾーン情報を保持するための構造体です。

シグネチャまたは形状

typedef struct DynamicZoneAbbrev
{
    pg_tz      *tz;                             /* NULL if not yet looked up */
    char        zone[FLEXIBLE_ARRAY_MEMBER];    /* NUL-terminated zone name */
} DynamicZoneAbbrev;

入力と出力

  • **入力**: タイムゾーン名（zone メンバ）および対応する pg_tz 構造体へのポインタ。
  • **出力**: タイムゾーンの解決状態を保持するメモリブロック。

前提条件と不変条件

  • zone メンバは FLEXIBLE_ARRAY_MEMBER として定義されており、構造体の末尾に可変長の文字列を保持します。
  • tz メンバは、タイムゾーンがまだ検索されていない場合は NULL となります。

アルゴリズムまたは振る舞い

  • タイムゾーンの省略形に対応する動的な情報を管理します。
  • src/backend/utils/adt/datetime.c において、abbrevs[] 配列の後に配置される形で動的に割り当てられ、タイムゾーンの検索結果をキャッシュする役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c 内のタイムゾーン処理ロジック（5041行目付近での割り当てなど）。
  • **呼び出し先**: タイムゾーンの解決時に pg_tz 構造体を参照します。

関連データ構造

  • pg_tz: タイムゾーン情報を保持する構造体。
  • datetkn: 日付・時刻トークンを保持する構造体。

失敗モードと境界条件

  • タイムゾーン名が解決できない場合、tz メンバは NULL のまま維持されます。
  • 可変長メンバを使用しているため、メモリ割り当て時には offsetof(DynamicZoneAbbrev, zone) を用いたサイズ計算が必要です。

保守担当者向けメモ

  • DynamicZoneAbbrev は abbrevs[] 配列の直後に配置される設計であるため、メモリレイアウトの変更には注意が必要です。
  • タイムゾーンの検索は遅延評価される可能性があるため、tz メンバが NULL であるケースを常に考慮してください。
