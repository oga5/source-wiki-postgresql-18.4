do_lo_export

定義

  • 種別: function
  • 定義箇所: src/bin/psql/large_obj.c:141

目的

データベース内のラージオブジェクトをファイルシステム上のファイルへエクスポートする。psql の \lo_export メタコマンドの実装として機能する。

シグネチャまたは形状

bool do_lo_export(const char *loid_arg, const char *filename_arg)

入力と出力

  • **入力**:
  • loid_arg: エクスポート対象のラージオブジェクト OID を表す文字列。
  • filename_arg: 出力先ファイルパスを表す文字列。
  • **出力**:
  • 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • ラージオブジェクト操作はトランザクション内で行う必要がある。
  • pset.db が有効なデータベース接続を保持していること。

アルゴリズムまたは振る舞い

1. start_lo_xact を呼び出し、トランザクション状態を確認・開始する。 2. SetCancelConn を用いて割り込み処理を有効化する。 3. libpq の lo_export を呼び出し、OID に対応するラージオブジェクトをファイルへ書き出す。 4. ResetCancelConn で割り込み処理を無効化する。 5. lo_export の戻り値が 1 でない場合、PQerrorMessage をログ出力し、fail_lo_xact を介してトランザクションをロールバック（または終了）して false を返す。 6. 成功時は finish_lo_xact でトランザクションを終了（コミット）する。 7. print_lo_result を呼び出し、操作結果を報告して true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:2393
  • **呼び出し先**:
  • start_lo_xact, finish_lo_xact, fail_lo_xact
  • SetCancelConn, ResetCancelConn
  • lo_export (libpq)
  • atooid
  • pg_log_info, PQerrorMessage
  • print_lo_result

関連データ構造

  • pset (グローバル設定構造体): データベース接続情報 (pset.db) を保持。

失敗モードと境界条件

  • start_lo_xact が失敗した場合、即座に false を返す。
  • lo_export が 1 以外の値を返した場合、エラーとして処理される。
  • finish_lo_xact が失敗した場合、false を返す。

保守担当者向けメモ

  • lo_export の戻り値の仕様が不明確であるため、status != 1 をエラー条件として判定している。
  • トランザクション管理は start_lo_xact / finish_lo_xact / fail_lo_xact に委譲されており、本関数はそれらの戻り値に従う必要がある。
