pg_prevpower2_32

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/port/pg_bitutils.h:234

目的

与えられた 32 ビット符号なし整数 num 以下で最大の 2 のべき乗を算出する。

シグネチャまたは形状

static inline uint32 pg_prevpower2_32(uint32 num)

入力と出力

  • **入力**: num (uint32) - 対象となる 32 ビット整数。
  • **出力**: num 以下の最大の 2 のべき乗（uint32）。

前提条件と不変条件

  • pg_leftmost_one_pos32 が 0 以外の入力に対して正しく最上位ビット位置を返すことを前提としている。
  • 入力 num が 0 の場合、pg_leftmost_one_pos32 の挙動に依存する（一般に 0 の入力は未定義動作やアサーションエラーの対象となる）。

アルゴリズムまたは振る舞い

1. pg_leftmost_one_pos32(num) を呼び出し、num の最上位ビット（MSB）の位置を取得する。 2. 数値 1 を取得した位置まで左シフトし、その結果を返す。これにより、MSB 以外のビットをすべて切り捨てた値が算出される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c:1295 にて max_buckets の計算に使用されている。
  • **呼び出し先**: pg_leftmost_one_pos32 を呼び出す。

関連データ構造

  • pg_prevpower2_size_t マクロ（src/include/port/pg_bitutils.h:438）により、size_t 型の処理に対して本関数がエイリアスとして定義されている。

失敗モードと境界条件

  • 入力 num が 0 の場合、pg_leftmost_one_pos32 の仕様に基づき、予期しない結果やアサーションエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結する。
  • 64 ビット版については pg_prevpower2_64 が別途存在する。
  • 移植性を高めるため、ハードウェア命令が利用できない環境では pg_bitutils.h 内のフォールバック実装が pg_leftmost_one_pos32 を通じて使用される。
