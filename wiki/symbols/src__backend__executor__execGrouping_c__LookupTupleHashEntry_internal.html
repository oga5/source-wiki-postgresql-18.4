LookupTupleHashEntry_internal

定義

  • 種別: static inline function
  • 定義箇所: src/backend/executor/execGrouping.c:468

目的

指定されたハッシュテーブル内から、入力タプルに対応するエントリを検索または作成します。

シグネチャまたは形状

static inline TupleHashEntry LookupTupleHashEntry_internal(
    TupleHashTable hashtable, 
    TupleTableSlot *slot, 
    bool *isnew, 
    uint32 hash
)

入力と出力

  • **入力**:
  • hashtable: 操作対象の TupleHashTable 構造体。
  • slot: 検索または挿入のキーとなるタプルを含む TupleTableSlot。
  • isnew: 新規作成された場合に true を格納するフラグへのポインタ（NULL の場合は検索のみ）。
  • hash: 事前に計算されたタプルのハッシュ値。
  • **出力**:
  • 検索または作成された TupleHashEntry へのポインタ。

前提条件と不変条件

  • isnew が指定されている場合、新規エントリ作成時には hashtable->tablecxt にメモリコンテキストが切り替えられます。
  • isnew が指定されていない場合、または既存エントリが見つかった場合は、メモリコンテキストの切り替えは行われません。

アルゴリズムまたは振る舞い

1. isnew が NULL でない場合:

  • tuplehash_insert_hash を呼び出し、ハッシュテーブルへの挿入を試みます。
  • 既存のエントリが見つかった場合、*isnew を false に設定します。
  • 新規作成された場合、*isnew を true に設定し、hashtable->tablecxt に切り替えた上で ExecCopySlotMinimalTupleExtra を使用してタプルをコピーし、entry->firstTuple に格納します。この際、additionalsize 分の領域も確保されます。

    • 2. isnew が NULL の場合:

  • tuplehash_lookup_hash を呼び出し、既存のエントリを検索します。

    • 3. 最終的に見つかった、または作成された TupleHashEntry を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execGrouping.c 内の LookupTupleHashEntry 等から呼び出されます。
  • **呼び出し先**:
  • tuplehash_insert_hash (lib/simplehash.h)
  • tuplehash_lookup_hash (lib/simplehash.h)
  • MemoryContextSwitchTo
  • ExecCopySlotMinimalTupleExtra

関連データ構造

  • TupleHashTable: ハッシュテーブルの管理構造体。
  • TupleHashEntryData: ハッシュテーブル内の各エントリを保持する構造体。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • メモリ不足時には MemoryContextSwitchTo や ExecCopySlotMinimalTupleExtra の内部でエラーが発生する可能性があります。
  • isnew が NULL の場合、エントリが存在しないときは NULL が返される可能性があります（tuplehash_lookup_hash の挙動に依存）。

保守担当者向けメモ

  • 本関数は static inline であり、パフォーマンスを重視して execGrouping.c 内の複数の検索関数から共有されています。
  • additionalsize を利用した追加データ領域の確保は、タプルと付加情報を同一メモリブロックに配置することで効率化を図っています。変更を加える際はメモリレイアウトに注意してください。
