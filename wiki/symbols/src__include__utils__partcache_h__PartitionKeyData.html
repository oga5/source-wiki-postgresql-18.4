PartitionKeyData

定義

  • 種別: struct
  • 定義箇所: src/include/utils/partcache.h:25

目的

パーティションテーブルの定義情報をリレーションキャッシュ（relcache）内で保持し、クエリ最適化や実行時にパーティションキー情報へ定数時間でアクセスするためのメタデータ構造体です。

シグネチャまたは形状

typedef struct PartitionKeyData
{
    PartitionStrategy strategy;
    int16       partnatts;
    AttrNumber *partattrs;
    List       *partexprs;
    Oid        *partopfamily;
    Oid        *partopcintype;
    FmgrInfo   *partsupfunc;
    Oid        *partcollation;
    Oid        *parttypid;
    int32      *parttypmod;
    int16      *parttyplen;
    bool       *parttypbyval;
    char       *parttypalign;
    Oid        *parttypcoll;
} PartitionKeyData;

入力と出力

本構造体は直接的な関数ではありませんが、RelationGetPartitionKey() を通じてリレーションから取得されます。出力として、パーティション戦略、列数、属性番号、式リスト、および各列の型情報や演算子族情報を提供します。

前提条件と不変条件

  • **不変性**: パーティションキーは一度構築されると変更されないため、リレーションが開かれている間はポインタを安全に保持可能です。
  • **メモリ管理**: CacheMemoryContext の子コンテキストで管理され、リレーションが閉じられるまで生存します。

アルゴリズムまたは振る舞い

リレーションキャッシュの rd_partkey フィールドとして保持されます。キャッシュ未構築時は RelationBuildPartitionKey() がシステムカタログ（pg_partitioned_table）を検索してメモリを割り当て、構造体を構築します。構築後は、インライン関数（get_partition_... 系）を通じて各フィールドが参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/partcache.c:108 にて sizeof(PartitionKeyData) が使用されています。
  • **呼び出し先**: 構造体内部で List や FmgrInfo などの型を使用していますが、具体的な関数呼び出し関係は本ソース抜粋からは特定できません。

関連データ構造

  • PartitionKey (typedef: struct PartitionKeyData *)
  • PartitionStrategy
  • AttrNumber
  • List
  • FmgrInfo

失敗モードと境界条件

  • キャッシュ未構築時にアクセスした場合、RelationGetPartitionKey() を介さない直接参照は未定義の動作を引き起こす可能性があります。
  • partattrs が 0 の場合、対応する partexprs リストから式を取得する必要があります。

保守担当者向けメモ

  • 本構造体への直接アクセスは避け、get_partition_strategy() や get_partition_col_attnum() などの提供されているインライン関数を使用してください。これにより、将来的な構造変更に対する抽象化が維持されます。
