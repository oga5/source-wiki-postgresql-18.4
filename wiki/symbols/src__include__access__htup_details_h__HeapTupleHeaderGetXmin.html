HeapTupleHeaderGetXmin

定義

  • 種別: function
  • 定義箇所: src/include/access/htup_details.h:328

目的

ヒープタプルのヘッダーから、そのタプルを挿入したトランザクションID（Xmin）を取得する。

シグネチャまたは形状

static inline TransactionId HeapTupleHeaderGetXmin(const HeapTupleHeaderData *tup)

入力と出力

  • **入力**: tup (const HeapTupleHeaderData *) - 対象となるヒープタプルのヘッダー構造体へのポインタ。
  • **出力**: TransactionId - タプルの挿入トランザクションID。

前提条件と不変条件

  • 入力ポインタ tup は有効な HeapTupleHeaderData を指している必要がある。
  • HeapTupleHeaderXminFrozen の判定結果に基づき、凍結済みタプルの場合は FrozenTransactionId を返す。

アルゴリズムまたは振る舞い

1. HeapTupleHeaderXminFrozen(tup) を呼び出し、タプルが凍結されているかを確認する。 2. 凍結されている場合は FrozenTransactionId を返す。 3. 凍結されていない場合は HeapTupleHeaderGetRawXmin(tup) を呼び出し、ヘッダーに記録されている生のトランザクションIDを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • contrib/amcheck/verify_nbtree.c
  • contrib/pg_visibility/pg_visibility.c
  • src/backend/access/heap/heapam.c
  • **呼び出し先**:
  • HeapTupleHeaderXminFrozen
  • HeapTupleHeaderGetRawXmin

関連データ構造

  • HeapTupleHeaderData: タプルのヘッダー構造。
  • TransactionId: トランザクションIDを表す型。
  • FrozenTransactionId: 凍結済みタプルを示す定数。

失敗モードと境界条件

  • tup が NULL の場合の挙動は未定義（セグメンテーション違反の可能性がある）。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、頻繁に呼び出されるためパフォーマンスへの影響を考慮すること。
  • 凍結済みタプルの判定ロジック（HeapTupleHeaderXminFrozen）に変更があった場合、本関数の戻り値にも直接影響する。
