get_ten

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc2004_sjis2004/euc2004_sjis2004.c:224

目的

Shift_JIS 形式の文字コードにおける第 2 バイトから、JIS X 0213 規格に基づく「点（ten）」を算出し、同時に「区（ku）」の偶奇フラグを決定する。

シグネチャまたは形状

static int get_ten(int b, int *ku);

入力と出力

  • **入力**:
  • b: Shift_JIS の第 2 バイト値（int）。
  • ku: 区の偶奇を格納するためのポインタ（int *）。
  • **出力**:
  • 戻り値: 計算された「点」（成功時）、またはエラー時は -1。
  • *ku: 偶奇フラグ（1 は奇数区、0 は偶数区）。

前提条件と不変条件

  • 入力 b は Shift_JIS の第 2 バイトとして有効な範囲（0x40〜0xFC）にあることが期待される。
  • ku は有効なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. b の値に応じて以下の範囲で計算を行う：

  • 0x40 ～ 0x7E: ten = b - 0x3f, ku = 1
  • 0x80 ～ 0x9E: ten = b - 0x40, ku = 1
  • 0x9F ～ 0xFC: ten = b - 0x9e, ku = 0

    • 2. 上記範囲外の場合、ten に -1 を返し、ku に 0 を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mb/conversion_procs/euc2004_sjis2004/euc2004_sjis2004.c 内の変換ロジック（317, 330, 344, 371 行目付近）。
  • **呼び出し先**:
  • なし（純粋な計算関数）。

関連データ構造

  • なし。

失敗モードと境界条件

  • b が 0x40 未満、または 0x7F（Shift_JIS で使用されない領域）、あるいは 0xFD 以上の場合、エラーとして ten = -1 を返す。

保守担当者向けメモ

  • 本関数は JIS X 0213 のマッピングロジックに深く依存している。変換ロジックの修正時は、Shift_JIS のバイトオフセットと JIS コードの対応関係を規格書と照合すること。
