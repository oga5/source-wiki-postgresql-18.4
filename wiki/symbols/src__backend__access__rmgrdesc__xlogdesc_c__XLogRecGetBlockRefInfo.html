XLogRecGetBlockRefInfo

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/rmgrdesc/xlogdesc.c:230

目的

WALレコードに含まれるブロック参照情報を走査し、人間が読める形式で StringInfo バッファに記述する。また、Full Page Image (FPI) のデータ長を計算する。

シグネチャまたは形状

void XLogRecGetBlockRefInfo(XLogReaderState *record, bool pretty, bool detailed_format, StringInfo buf, uint32 *fpi_len)

入力と出力

  • **入力**:
  • record: 解析対象の XLogReaderState。
  • pretty: 改行やインデントによる整形を行うか。
  • detailed_format: 詳細なブロック情報（オフセット、ホール、圧縮方式等）を出力するか。
  • buf: 出力先の StringInfo バッファ。
  • fpi_len: FPIデータの合計長を格納するポインタ（NULL可）。
  • **出力**: なし（buf への追記および fpi_len への加算）。

前提条件と不変条件

  • record は NULL であってはならない（Assert による検証）。
  • fpi_len が提供されている場合、レコード内の各ブロックのFPI長が加算される。

アルゴリズムまたは振る舞い

1. XLogRecMaxBlockId を用いてレコード内の全ブロックを走査する。 2. XLogRecGetBlockTagExtended でブロックのタグ情報を取得し、取得できない場合はスキップする。 3. detailed_format が真の場合:

  • ブロックID、リレーショナルロケータ、フォーク番号、ブロック番号を詳細に出力する。
  • FPIが存在する場合、その圧縮方式（pglz, lz4, zstd）やホール情報を出力する。

    • 4. detailed_format が偽の場合:

  • 簡潔な形式でブロック情報を出力する。
  • FPIが存在する場合は "FPW" または "FPW for WAL verification" を付記する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_walinspect/pg_walinspect.c
  • src/bin/pg_waldump/pg_waldump.c
  • **呼び出し先**:
  • XLogRecMaxBlockId, XLogRecGetBlockTagExtended, XLogRecHasBlockImage, XLogRecGetBlock, XLogRecBlockImageApply（いずれも XLogReaderState 関連のアクセサ）。
  • appendStringInfo 系関数。

関連データ構造

  • XLogReaderState: WALレコード読み取り状態を保持する構造体。
  • StringInfo: 文字列構築用バッファ。
  • RelFileLocator: リレーションの識別子。

失敗モードと境界条件

  • XLogRecGetBlockTagExtended が失敗したブロックは無視される。
  • fpi_len が NULL の場合、FPI長の計算はスキップされる。
  • 圧縮方式が既知の定数（BKPIMAGE_COMPRESS_...）に一致しない場合、"unknown" と記録される。

保守担当者向けメモ

  • 新しい圧縮方式を追加した際は、BKPIMAGE_COMPRESSED 判定および method の if-else ブロックを更新する必要がある。
  • detailed_format の有無により出力形式が大きく異なるため、変更時は pg_waldump 等の出力への影響を確認すること。
