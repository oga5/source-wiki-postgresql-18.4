hash_xlog_init_bitmap_page

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash_xlog.c:62

目的

ハッシュインデックスのビットマップページを初期化し、対応するメタページを更新するWALリプレイ処理を実行します。

シグネチャまたは形状

static void hash_xlog_init_bitmap_page(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファおよびディスク上のページ状態を更新）。

前提条件と不変条件

  • record から xl_hash_init_bitmap_page 構造体が正しく抽出可能であること。
  • リプレイ中は他のプロセスによる同時アクセスがないため、メタページの更新時にビットマップページのロックを保持する必要はありません。

アルゴリズムまたは振る舞い

1. **ビットマップページの初期化**: XLogInitBufferForRedo を使用してビットマップページを初期化し、_hash_initbitmapbuffer で内容を適用します。 2. **永続化**: ページにLSNを書き込み、ダーティとしてマークします。INIT_FORKNUM の場合は FlushOneBuffer を呼び出してディスク同期を強制します。 3. **メタページの更新**: XLogReadBufferForRedo を呼び出し、BLK_NEEDS_REDO が返された場合にメタページを更新します。

  • メタページの hashm_mapp 配列に新しいビットマップページを追加し、hashm_nmaps をインクリメントします。

    • 4. **終了処理**: 取得したバッファをアンロックおよび解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hash_xlog.c 内の hash_redo（推測されるディスパッチャ）。
  • **呼び出し先**:
  • XLogRecGetData, XLogInitBufferForRedo, XLogReadBufferForRedo, XLogRecGetBlockTag
  • _hash_initbitmapbuffer
  • PageSetLSN, MarkBufferDirty, FlushOneBuffer, UnlockReleaseBuffer
  • BufferGetPage, HashPageGetMeta, BufferIsValid

関連データ構造

  • xl_hash_init_bitmap_page: WALレコードから取得されるビットマップ初期化情報。
  • HashMetaPage: ハッシュインデックスのメタデータ構造。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、メタページの更新はスキップされます。
  • INIT_FORKNUM の場合のみ、明示的な FlushOneBuffer が実行されます。

保守担当者向けメモ

  • 本関数は冪等である必要があります。
  • INIT_FORKNUM に対する FlushOneBuffer は、インデックス作成時にメタページのフルページイメージが記録されないことへの対策です。
