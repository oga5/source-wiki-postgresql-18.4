ginCompareEntries

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginutil.c:392

目的

GINインデックスにおいて、2つのエントリ（キー）を比較し、その順序関係を決定する。

シグネチャまたは形状

int ginCompareEntries(GinState *ginstate, OffsetNumber attnum,
                      Datum a, GinNullCategory categorya,
                      Datum b, GinNullCategory categoryb)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの実行時状態構造体。
  • attnum: 比較対象の属性番号。
  • a, b: 比較する値（Datum）。
  • categorya, categoryb: 値のNULLカテゴリ（GinNullCategory）。
  • **出力**:
  • a < b の場合は負の整数、a > b の場合は正の整数、等しい場合は 0 を返す。

前提条件と不変条件

  • ginstate は initGinState によって初期化済みであること。
  • attnum は有効な属性番号であること。

アルゴリズムまたは振る舞い

1. **カテゴリ比較**: categorya と categoryb が異なる場合、カテゴリ値に基づいて順序を決定する。 2. **NULLカテゴリの判定**: 両方のカテゴリが GIN_CAT_NORM_KEY（通常キー）でない場合、カテゴリが同一であれば等しいとみなして 0 を返す。 3. **値の比較**: 両者が GIN_CAT_NORM_KEY である場合、ginstate にキャッシュされた compareFn を呼び出し、supportCollation を適用して比較結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginget.c
  • src/backend/access/gin/ginscan.c
  • src/backend/access/gin/ginutil.c (ginCompareAttEntries 内)
  • **呼び出し先**:
  • ginstate->compareFn（OpClassの比較関数）

関連データ構造

  • GinState: インデックスの演算クラス情報やサポート関数を保持する構造体。
  • GinNullCategory: NULL値やキーのカテゴリを定義する列挙型。

失敗モードと境界条件

  • attnum が範囲外の場合、ginstate->compareFn 配列へのアクセスでメモリ違反が発生する可能性がある。
  • compareFn が未定義の演算クラスに対して呼び出された場合、実行時エラーとなる。

保守担当者向けメモ

  • 本関数は ginCompareAttEntries からも利用されており、GINインデックスのソート順序を決定する中核的な関数である。
  • compareFn の呼び出しには FunctionCall2Coll を使用しており、照合順序（Collation）を考慮した比較が行われる点に注意すること。
