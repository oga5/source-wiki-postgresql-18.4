AlterSubscription

定義

  • 種別: function
  • 定義箇所: src/backend/commands/subscriptioncmds.c:1099

目的

論理レプリケーションにおける既存のサブスクリプション設定（オプション等）を変更する。

シグネチャまたは形状

ObjectAddress AlterSubscription(ParseState *pstate, AlterSubscriptionStmt *stmt, bool isTopLevel)

入力と出力

  • **入力**:
  • pstate: パーサ状態。
  • stmt: 変更内容を保持する AlterSubscriptionStmt 構造体。
  • isTopLevel: 最上位レベルのコマンドかどうかを示すフラグ。
  • **出力**: 変更されたサブスクリプションの ObjectAddress を返す。

前提条件と不変条件

  • 対象のサブスクリプションが存在すること。
  • 実行ユーザーがサブスクリプションの所有者であること。
  • password_required=false のサブスクリプションを操作する場合、スーパーユーザーである必要がある。
  • AccessExclusiveLock を取得し、対象サブスクリプションを排他制御する。

アルゴリズムまたは振る舞い

1. SUBSCRIPTIONNAME システムキャッシュから対象サブスクリプションを検索・コピーする。 2. 所有権および権限チェックを行う。 3. stmt->kind に基づき分岐し、parse_subscription_options で指定されたオプションを解析する。 4. ALTER_SUBSCRIPTION_OPTIONS の場合、以下の検証と更新を行う：

  • slot_name: 有効なサブスクリプションに対して NONE を設定することは不可。
  • two_phase: slot_name との同時変更は不可。また、変更前に CheckAlterSubOption による検証を行う。
  • その他、synchronous_commit, binary, streaming, disableonerr, password_required, run_as_owner 等の属性を pg_subscription カタログのタプルに反映する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c (SQLコマンド実行時)
  • **呼び出し先**:
  • parse_subscription_options (オプション解析)
  • CheckAlterSubOption (オプション変更の整合性チェック)
  • GetSubscription (サブスクリプション情報の取得)
  • LockSharedObject (排他ロック)
  • SearchSysCacheCopy2 (カタログ検索)

関連データ構造

  • AlterSubscriptionStmt: SQLコマンドの構文解析結果。
  • SubOpts: 解析されたオプション値を保持する内部構造体。
  • pg_subscription: サブスクリプション情報を格納するシステムカタログ。

失敗モードと境界条件

  • サブスクリプションが存在しない場合、ERRCODE_UNDEFINED_OBJECT でエラー。
  • 所有者以外が実行した場合、ACLCHECK_NOT_OWNER でエラー。
  • 権限不足（password_required 関連）の場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー。
  • 無効なオプション組み合わせ（slot_name と two_phase の同時変更など）の場合、ERRCODE_SYNTAX_ERROR 等でエラー。

保守担当者向けメモ

  • 本関数は switch 文でサブコマンドを処理する構造のため、新しい変更項目を追加する際は該当する case 内で replaces 配列を適切に設定すること。
  • two_phase 変更時のワーカー終了処理など、レプリケーションの整合性に直結する処理が含まれているため、変更時は worker.c 等の関連箇所との整合性に注意が必要。
