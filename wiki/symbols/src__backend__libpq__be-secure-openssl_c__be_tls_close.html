be_tls_close

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-openssl.c:734

目的

PostgreSQL バックエンドの TLS セッションを終了し、関連する OpenSSL リソースおよびピア情報を解放します。

シグネチャまたは形状

void be_tls_close(Port *port)

入力と出力

  • **入力**: Port *port - セッション情報を保持する PostgreSQL のポート構造体。
  • **出力**: なし（副作用として port 構造体内の SSL 関連フィールドをクリアします）。

前提条件と不変条件

  • port が有効なポインタであること。
  • port->ssl が存在する場合、OpenSSL のセッション状態が適切に管理されていること。

アルゴリズムまたは振る舞い

1. **SSL セッションの終了**: port->ssl が存在する場合、SSL_shutdown を呼び出して TLS シャットダウンを実行し、SSL_free でオブジェクトを解放します。その後、port->ssl を NULL に、port->ssl_in_use を false に設定します。 2. **ピア証明書の解放**: port->peer が存在する場合、X509_free で解放し、port->peer を NULL に設定します。 3. **識別情報の解放**: port->peer_cn および port->peer_dn が存在する場合、pfree を用いてメモリを解放し、それぞれ NULL に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure.c:171 から呼び出されます。
  • **呼び出し先**: OpenSSL ライブラリの SSL_shutdown、SSL_free、X509_free、および PostgreSQL のメモリ管理関数 pfree を呼び出します。

関連データ構造

  • Port: src/include/libpq/libpq-be.h で定義される、接続状態を保持する構造体。

失敗モードと境界条件

  • port が NULL の場合、あるいは port 内の各ポインタが既に NULL である場合、関数は何も行わず安全に終了します。

保守担当者向けメモ

  • 本関数は be-secure.c の抽象化レイヤーから呼び出されることを想定しています。
  • pfree を使用しているため、PostgreSQL のメモリコンテキストが有効な状態で呼び出される必要があります。
