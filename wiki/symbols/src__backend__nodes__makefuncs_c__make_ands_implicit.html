make_ands_implicit

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/makefuncs.c:809

目的

論理積（AND）条件式を、リスト形式の「暗黙的なAND」表現に変換します。クエリの WHERE 句などで、複数の条件がANDで結合されている状態をリストとして正規化するために使用されます。

シグネチャまたは形状

List *make_ands_implicit(Expr *clause)

入力と出力

  • **入力**: Expr *clause（解析対象の式ノード）
  • **出力**: List *（条件式のリスト。ANDで結合された各要素のリスト、または空リスト NIL）

前提条件と不変条件

  • 入力 clause が NULL の場合、論理的に「真（TRUE）」とみなされ、空リスト NIL が返されます。
  • 入力 clause が定数の「真（TRUE）」である場合も、空リスト NIL が返されます。

アルゴリズムまたは振る舞い

1. **NULLチェック**: clause が NULL ならば NIL を返す。 2. **ANDノードの判定**: is_andclause(clause) が真であれば、その BoolExpr 内の args（ANDで結合された式のリスト）をそのまま返す。 3. **定数判定**: clause が定数ノードであり、かつその値が真であれば NIL を返す。 4. **単一条件**: 上記のいずれにも該当しない場合、clause を単一要素として含むリストを list_make1(clause) で生成して返す。

関連データ構造

  • Expr: 式ノードの基底型。
  • BoolExpr: 論理演算（AND/OR/NOT）を保持するノード。
  • Const: 定数ノード。
  • List: PostgreSQLのリスト構造。

失敗モードと境界条件

  • clause が NULL の場合、エラーにはならず NIL を返す安全な設計となっています。
  • Const ノードが NULL 値（constisnull）を持つ場合、あるいは偽（FALSE）である場合は、単一要素のリストとして扱われます。

保守担当者向けメモ

  • 本関数はパーサーが WHERE 句のないクエリに対して qual フィールドを NULL に設定する仕様を考慮しています。このため、NULL 入力を「偽」ではなく「真（空リスト）」として扱う点に注意してください。
