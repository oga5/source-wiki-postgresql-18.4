ReadNextTransactionId

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/transam.h:314

目的

システム全体で次に割り当てられる予定のトランザクションID（XID）を、32ビットの TransactionId 型として取得します。

シグネチャまたは形状

static inline TransactionId ReadNextTransactionId(void)

入力と出力

  • **入力**: なし。
  • **出力**: 次回割り当て予定の32ビットトランザクションIDを返します。

前提条件と不変条件

  • 共有メモリ上のトランザクション状態（TransamVariables）が初期化されている必要があります。
  • 内部的に ReadNextFullTransactionId() を呼び出し、その結果を XidFromFullTransactionId() で32ビットに変換します。

アルゴリズムまたは振る舞い

1. ReadNextFullTransactionId() を呼び出し、64ビットの FullTransactionId を取得します。 2. 取得した64ビット値を XidFromFullTransactionId() に渡し、下位32ビットの TransactionId を抽出して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: ReadNextFullTransactionId(), XidFromFullTransactionId()
  • **呼び出し元（直接的な証拠）**:
  • src/backend/access/gin/ginvacuum.c
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/access/transam/commit_ts.c
  • src/backend/access/transam/xact.c
  • src/backend/commands/vacuum.c
  • src/backend/postmaster/autovacuum.c

関連データ構造

  • TransactionId: 32ビットのトランザクション識別子。
  • FullTransactionId: エポックを含む64ビットのトランザクション識別子。
  • TransamVariablesData: 共有メモリ上で次回のXID生成状態を保持する構造体。

失敗モードと境界条件

  • 本関数自体は単純な値の読み取りですが、トランザクションIDのラップアラウンド（循環）が発生する環境下では、返される値が過去のIDと比較して「古い」と判定される可能性があるため、呼び出し側での適切な比較処理（TransactionIdPrecedes 等）が必要です。

保守担当者向けメモ

  • 64ビットの FullTransactionId が導入されたため、32ビットの TransactionId を返す本関数は、既存の古いインターフェースとの互換性を維持するために提供されています。新しいコードでは可能な限り64ビット版の利用を検討してください。
