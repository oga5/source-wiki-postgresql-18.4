IndexSupportsBackwardScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAmi.c:602

目的

指定されたインデックスが、アクセスメソッド（AM）レベルで後方スキャン（逆順スキャン）をサポートしているかを判定する。

シグネチャまたは形状

static bool IndexSupportsBackwardScan(Oid indexid)

入力と出力

  • **入力**: indexid (Oid) - 判定対象のインデックスのオブジェクト識別子。
  • **出力**: bool - インデックスが後方スキャンをサポートしていれば true、そうでなければ false。

前提条件と不変条件

  • indexid はシステムカタログ pg_class に存在する有効なインデックスであること。
  • 内部で SearchSysCache1 を使用するため、システムキャッシュが利用可能な状態であること。

アルゴリズムまたは振る舞い

1. RELOID システムキャッシュを使用して、指定された indexid に対応する pg_class タプルを取得する。 2. タプルからインデックスのアクセスメソッド ID (relam) を抽出する。 3. GetIndexAmRoutineByAmId を呼び出し、該当するアクセスメソッドの API 構造体 (IndexAmRoutine) を取得する。 4. API 構造体の amcanbackward フラグを参照し、その値を結果とする。 5. 取得した amroutine および ht_idxrel（システムキャッシュタプル）を解放し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execAmi.c 内の IndexScan および IndexOnlyScan ノードに関連する処理から呼び出される。
  • **呼び出し先**:
  • SearchSysCache1 (システムキャッシュ検索)
  • GetIndexAmRoutineByAmId (AM API 構造体の取得)
  • pfree (メモリ解放)
  • ReleaseSysCache (キャッシュタプルの解放)

関連データ構造

  • Oid: オブジェクト識別子。
  • HeapTuple: システムカタログのタプル表現。
  • Form_pg_class: pg_class カタログの構造体。
  • IndexAmRoutine: インデックスアクセスメソッドの機能フラグを保持する構造体。

失敗モードと境界条件

  • 指定された indexid がシステムキャッシュで見つからない場合、elog(ERROR) により処理が中断される。
  • GetIndexAmRoutineByAmId が失敗した場合の挙動は、当該関数の実装に依存する。

保守担当者向けメモ

  • 本関数は pg_class およびインデックス AM の API 定義に依存している。インデックス AM の機能拡張や変更が行われた場合、本関数のロジック自体は変更不要だが、amcanbackward フラグの解釈が変更される可能性がある。
