spg_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/spgdesc.c:131

目的

SP-GiSTインデックスのWALレコードの info フィールドを、人間が判読可能な操作名（文字列）に変換します。

シグネチャまたは形状

const char *spg_identify(uint8 info);

入力と出力

  • **入力**: uint8 info - WALレコードの info フィールド。
  • **出力**: const char * - 対応する操作名の文字列リテラル。一致しない場合は NULL を返します。

前提条件と不変条件

  • 入力 info に対して XLR_INFO_MASK を用いたビットマスク処理が行われ、補助フラグが除外された状態で操作タイプが判定されます。

アルゴリズムまたは振る舞い

1. info & ~XLR_INFO_MASK を計算し、純粋な操作タイプを抽出します。 2. switch 文により、以下のいずれかの定数と照合します。

  • XLOG_SPGIST_ADD_LEAF, MOVE_LEAFS, ADD_NODE, SPLIT_TUPLE, PICKSPLIT, VACUUM_LEAF, VACUUM_ROOT, VACUUM_REDIRECT

    • 3. 一致した操作に対応する文字列（例: "ADD_LEAF"）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h 内の PG_RMGR マクロ定義を通じて、PostgreSQLのWALリソースマネージャインターフェースから参照されます。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • access/spgxlog.h: SP-GiSTのWALレコードに関連する定数定義を提供します。

失敗モードと境界条件

  • info が既知のSP-GiST操作タイプと一致しない場合、NULL を返します。

保守担当者向けメモ

  • 新しいWALレコードタイプをSP-GiSTに追加した際は、本関数の switch 文に必ず新しい case を追加してください。
