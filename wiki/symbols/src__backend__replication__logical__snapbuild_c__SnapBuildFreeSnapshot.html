SnapBuildFreeSnapshot

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/snapbuild.c:251

目的

論理デコーディングで使用される SNAPSHOT_HISTORIC_MVCC 型のスナップショットをメモリから解放する。

シグネチャまたは形状

static void SnapBuildFreeSnapshot(Snapshot snap)

入力と出力

  • **入力**: Snapshot snap - 解放対象のスナップショット構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • snap->snapshot_type が SNAPSHOT_HISTORIC_MVCC であること。
  • snap->curcid が FirstCommandId であること。
  • snap->suboverflowed が false であること。
  • snap->takenDuringRecovery が false であること。
  • snap->regd_count が 0 であること。

アルゴリズムまたは振る舞い

1. 上記の前提条件を Assert で検証する。 2. snap->copied が真の場合、コピーされたスナップショットは解放できないため ERROR を発生させる。 3. snap->active_count が 0 以外の場合、アクティブなスナップショットは解放できないため ERROR を発生させる。 4. pfree(snap) を呼び出し、メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/snapbuild.c 内の関数（行348など）。
  • **呼び出し先**: pfree（メモリ管理関数）。

関連データ構造

  • Snapshot: 論理デコーディングにおけるMVCCスナップショットを保持する構造体。

失敗モードと境界条件

  • **ERROR**: コピー済みのスナップショット（copied が真）を渡した場合。
  • **ERROR**: アクティブなスナップショット（active_count が 0 以外）を渡した場合。
  • **Assert失敗**: SNAPSHOT_HISTORIC_MVCC 以外の型や、期待される状態（curcid 等）を満たさないスナップショットを渡した場合。

保守担当者向けメモ

  • 本関数は SNAPSHOT_HISTORIC_MVCC 専用の解放処理です。汎用的なスナップショット解放関数ではないため、他のスナップショット型を扱う際には注意してください。
