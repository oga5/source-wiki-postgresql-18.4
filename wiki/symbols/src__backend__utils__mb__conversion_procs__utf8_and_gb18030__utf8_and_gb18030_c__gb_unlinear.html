gb_unlinear

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c:46

目的

GB18030 エンコーディングの 4 バイト文字表現において、線形空間上の数値（インデックス）を、実際の 4 バイトのバイト列形式に変換します。

シグネチャまたは形状

static inline uint32 gb_unlinear(uint32 lin)

入力と出力

  • **入力**: lin (uint32) - GB18030 の 4 バイト文字範囲を線形化した数値。
  • **出力**: uint32 - GB18030 の 4 バイト文字を構成するバイト列をビッグエンディアン形式でパックした値（0xR0R1R2R3）。

前提条件と不変条件

  • 入力 lin は GB18030 の 4 バイト文字範囲に対応する有効な線形値である必要があります。
  • GB18030-2000 仕様に基づいた計算式を使用しています。

アルゴリズムまたは振る舞い

1. 入力 lin を除算および剰余演算を用いて 4 つのバイト成分（r0, r1, r2, r3）に分解します。

  • r0 = 0x81 + lin / 12600
  • r1 = 0x30 + (lin / 1260) % 10
  • r2 = 0x81 + (lin / 10) % 126
  • r3 = 0x30 + lin % 10

    • 2. 分解した各バイトをビットシフトし、論理和をとることで 32 ビットの整数値として結合して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c 内の変換ロジック（例: 164 行目付近）から呼び出されます。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • 特になし（プリミティブな数値計算のみ）。

失敗モードと境界条件

  • 入力 lin が想定される範囲外である場合、計算結果は仕様外のバイト列となります。本関数自体でのバリデーションは行われません。

保守担当者向けメモ

  • 本関数は gb_linear 関数の逆変換です。
  • GB18030 の 4 バイト文字の構造（81-FE 30-39 81-FE 30-39）を維持するための計算ロジックであるため、仕様変更がない限り修正は不要です。
