目的

PostgreSQL の ECPG における bytea 型のデータ送受信およびカーソル操作の整合性を検証する回帰テストのエントリポイント。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（コマンドライン引数は使用しない）
  • 出力: 終了コード（成功時は 0、エラー時は sqlprint() による標準エラー出力への診断メッセージ）

前提条件と不変条件

  • ECPG ランタイムライブラリが利用可能であること。
  • データベース ecpg1_regression への接続が可能であること。
  • bytea 型を格納する構造体（bytea_1 ～ bytea_4）の len メンバが、データ操作前に適切に設定されていること。

アルゴリズムまたは振る舞い

1. **初期化**: init() マクロにより、送受信用の構造体配列とインジケータ変数をリセット。 2. **データ準備**: send_buf に 0xff から降順のバイナリパターンを生成。 3. **データベース接続**: ECPGconnect を実行。 4. **テーブル作成**: test テーブル（data1, data2 カラム）を作成。 5. **SQL 実行**:

  • INSERT 文で bytea データを投入。
  • SELECT 文でデータを取得し、recv_buf 等へ格納。
  • カーソル操作および PREPARE された動的 SQL を用いたデータ操作を実行。

    • 6. **検証**: dump_binary() を呼び出し、取得したバイナリデータの整合性を標準出力へ書き出す。 7. **終了**: 接続を切断し、終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラム実行時）
  • 呼び出し先:
  • ECPGconnect, ECPGdo, ECPGprepare, ECPGallocate_desc, ECPGset_var, ECPGdeallocate（ECPG ランタイム関数）
  • dump_binary（デバッグ用出力関数）
  • sqlprint（エラー出力）
  • memset（標準ライブラリ）

関連データ構造

  • struct bytea_1 ～ bytea_4: bytea 型を表現する構造体（len と arr を保持）。
  • ind: SQL インジケータ変数配列。

失敗モードと境界条件

  • sqlca.sqlcode < 0 の場合、sqlprint() が呼び出され、エラー詳細が標準エラー出力に表示される。
  • recv_short_buf を使用したテストケースにより、バッファサイズが不足した場合の挙動を確認する。

保守担当者向けメモ

  • 本関数はプリプロセッサ生成コードであるため、直接編集しないこと。
  • ECPGdebug(1, stderr) が有効であり、実行時に詳細な通信ログが標準エラー出力に出力される。
  • DATA_SIZE や LACK_SIZE の定義変更は、テストの境界条件に直接影響する。
