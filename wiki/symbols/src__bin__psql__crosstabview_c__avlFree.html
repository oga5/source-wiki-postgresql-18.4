avlFree

定義

  • 種別: function
  • 定義箇所: src/bin/psql/crosstabview.c:447

目的

AVL木構造全体およびそのノードが保持するメモリを再帰的に解放する。

シグネチャまたは形状

static void avlFree(avl_tree *tree, avl_node *node)

入力と出力

  • tree: 解放対象の木構造へのポインタ。tree->end（番兵）の解放に使用される。
  • node: 再帰的に解放を開始する起点となるノード。
  • 戻り値: なし（void）。

前提条件と不変条件

  • node は tree に属する有効なノードであること。
  • tree->end は、木構造の末端を示す番兵として適切に初期化されていること。

アルゴリズムまたは振る舞い

1. node の左右の子ノードを順に確認する。 2. 子ノードが tree->end でない場合、その子ノードに対して avlFree を再帰的に呼び出し、その後 pg_free でメモリを解放する。 3. node が tree->root と一致する場合、ルートノード自体を解放する。 4. 最後に tree->end を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/crosstabview.c 内の piv_columns および piv_rows の破棄処理（273行目、274行目）。
  • 呼び出し先: avlFree（再帰呼び出し）、pg_free（メモリ解放）。

関連データ構造

  • avl_tree: 木のルートおよび番兵（end）を保持する構造体。
  • avl_node: 木の各ノード。children[2] 配列で子ノードを保持する。

失敗モードと境界条件

  • node が tree->end を指している場合、そのノード自体は解放されない（再帰の停止条件）。
  • ルートノードが tree->end と等しい場合、ルートの解放はスキップされる。

保守担当者向けメモ

  • 本関数は avl_tree の破棄専用であり、個別のノード削除には対応していない。
  • pg_free を使用しているため、PostgreSQL のメモリ管理コンテキスト下で実行される必要がある。
