add_nulling_relids_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteManip.c:44

目的

クエリツリーの走査中に、特定の関係（リレーション）のNULL許容属性（varnullingrels や phnullingrels）を更新するためのコンテキスト情報を保持する構造体です。

シグネチャまたは形状

typedef struct
{
	const Bitmapset *target_relids;
	const Bitmapset *added_relids;
	int			sublevels_up;
} add_nulling_relids_context;

入力と出力

本構造体は関数への引数として渡されるコンテキストオブジェクトです。

  • target_relids: 処理対象とするリレーションIDの集合。
  • added_relids: 追加または適用するNULL許容リレーションIDの集合。
  • sublevels_up: 現在のネストレベル（サブクエリの深さ）。

前提条件と不変条件

  • target_relids および added_relids は Bitmapset へのポインタであり、読み取り専用として扱われます。
  • sublevels_up は、ツリー走査の深さに応じて適切に管理される必要があります。

アルゴリズムまたは振る舞い

本構造体は、expression_tree_walker 等を用いたツリー走査において、現在の走査状態を保持するために使用されます。走査中に Query ノードに遭遇した際、sublevels_up をインクリメントしてサブクエリ内を探索する際の追跡に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/rewrite/rewriteManip.c 内の関数（1255行目および1268行目付近）でインスタンス化および参照されています。
  • 呼び出し先: 具体的な関数呼び出しの詳細は提供されたソース抜粋には含まれていません。

関連データ構造

  • Bitmapset: リレーションIDの集合を保持するために使用されます。

失敗モードと境界条件

  • target_relids または added_relids が NULL の場合、あるいは sublevels_up が不適切な値である場合の挙動については、本構造体の定義のみからは判断できません。

保守担当者向けメモ

  • 本構造体は rewriteManip.c 内のツリー走査ロジックと密接に関連しています。
  • sublevels_up の管理ルール（サブクエリ突入時のインクリメント、脱出時のデクリメント）を遵守してください。
