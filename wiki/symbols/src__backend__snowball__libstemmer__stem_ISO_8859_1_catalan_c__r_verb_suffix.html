r_verb_suffix

目的

カタルーニャ語のステミング処理において、単語末尾の動詞接尾辞を識別し、特定の領域（R1またはR2）内にある場合に削除する。

シグネチャまたは形状

static int r_verb_suffix(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: int（成功時に1、マッチしない場合に0、エラー発生時に負の値を返す）

前提条件と不変条件

  • z->c は処理対象の単語内の現在位置を指す。
  • find_among_b を使用するため、単語は逆方向に走査される。
  • 削除操作は r_R1 または r_R2 による領域チェックを通過した場合のみ実行される。

アルゴリズムまたは振る舞い

1. 現在位置 z->c を z->ket に設定する。 2. a_3 テーブルを使用して、283個の候補から後方一致検索を行う。 3. マッチした among_var に応じて分岐する:

  • case 1: r_R1 を呼び出し、領域内であれば slice_del で接尾辞を削除する。
  • case 2: r_R2 を呼び出し、領域内であれば slice_del で接尾辞を削除する。

    • 4. マッチしなかった場合は 0 を返し、削除処理が成功した場合は 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_ISO_8859_1_catalan.c および src/backend/snowball/libstemmer/stem_UTF_8_catalan.c 内のメインステミング処理フローから呼び出される。
  • 呼び出し先: find_among_b, r_R1, r_R2, slice_del（いずれもSnowballランタイム関数）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_3: 動詞接尾辞の探索用 among 構造体テーブル。

失敗モードと境界条件

  • find_among_b がマッチしない場合、関数は即座に 0 を返す。
  • r_R1 または r_R2 が失敗（0以下を返す）した場合、その戻り値をそのまま伝播させて終了する。
  • slice_del が失敗（負の値を返す）した場合、その戻り値をそのまま伝播させて終了する。
