r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_basque.c:906

目的

バスク語のステミング処理において、単語内の語幹抽出対象領域（RV, R1, R2）を特定し、その境界位置を SN_env 構造体の整数配列 I に記録する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z（ステミング環境。現在の単語データおよび領域境界を保持する配列 I を含む）
  • **出力**: 常に 1 を返す。副作用として z->I[0]（R2）、z->I[1]（R1）、z->I[2]（RV）が更新される。

前提条件と不変条件

  • z は有効な SN_env 構造体へのポインタであること。
  • g_v（母音グループ）が定義されていること。
  • 領域境界は単語の末尾（z->l）で初期化される。

アルゴリズムまたは振る舞い

1. **初期化**: I[0]〜I[2] を単語長 z->l に設定する。 2. **RV領域の決定**: 母音・子音の並び順に基づいて、単語内の RV 領域の開始位置を探索し I[2] に格納する。 3. **R1/R2領域の決定**:

  • 母音と子音の切り替わりを追跡し、最初の母音の後の子音の直後を R1 の開始位置として I[1] に格納する。
  • 同様に、R1 の開始位置からさらに母音と子音の切り替わりを追跡し、R2 の開始位置を I[0] に格納する。

    • 4. 探索中にマッチングが失敗した場合は、各段階でデフォルト値（単語長）を維持する。

呼び出し元と呼び出し先

  • **呼び出し元**: basque_UTF_8_stem（ステミング処理のメインエントリーポイント）。
  • **呼び出し先**: in_grouping_U, out_grouping_U, skip_utf8（Snowball ランタイムのユーティリティ関数）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。I 配列（領域境界）、c（現在のカーソル位置）、l（単語長）を使用。

失敗モードと境界条件

  • 単語が短すぎる場合や、母音・子音のパターンが適合しない場合は、領域境界が単語の末尾に留まる。
  • skip_utf8 が不正なシーケンスを検出した場合、探索を中断し、その時点での境界値を保持する。

保守担当者向けメモ

  • 本関数は Snowball コンパイラにより自動生成されている。ロジックの変更が必要な場合は、直接編集せずソースとなる .sbl ファイルを修正すること。
  • I[0]=R2, I[1]=R1, I[2]=RV のマッピングは、他の言語のステマーと共通の規約に従っている。
