BufTableHashCode

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/buf_table.c:77

目的

共有バッファハッシュテーブル（SharedBufHash）における、特定の BufferTag に対応するハッシュ値を計算します。この値は、バッファマッピングのロックパーティションを特定するために使用されます。

シグネチャまたは形状

uint32 BufTableHashCode(BufferTag *tagPtr)

入力と出力

  • **入力**: tagPtr（ハッシュ計算対象となる BufferTag 構造体へのポインタ）
  • **出力**: uint32 型のハッシュ値

前提条件と不変条件

  • SharedBufHash が初期化済みである必要があります。
  • 本関数自体はロックを保持しません。呼び出し側は、本関数で得られたハッシュ値を用いて適切な BufMappingLock を取得する責任があります。

アルゴリズムまたは振る舞い

内部的に get_hash_value 関数を呼び出し、共有ハッシュテーブル SharedBufHash の設定に基づいて tagPtr のハッシュ値を算出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c（バッファの検索、挿入、削除の各操作前）
  • **呼び出し先**:
  • get_hash_value（共有ハッシュテーブルの内部関数）

関連データ構造

  • BufferTag: バッファを識別するためのキー構造体。
  • SharedBufHash: バッファマッピングを管理する共有ハッシュテーブル。

失敗モードと境界条件

  • tagPtr が NULL の場合の挙動は、get_hash_value の実装に依存します。
  • 共有メモリが初期化されていない状態で呼び出された場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • ハッシュ計算を独立させているのは、ロック取得前にハッシュ値が必要であり、かつ計算の重複を避けるためです。
  • 呼び出し側で BufTableHashCode() を呼び出した後、必ず BufMappingPartitionLock() を適用する一連のパターンが src/include/storage/buf_internals.h で規定されています。
