make_rfile

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/reconstruct.c:510

目的

再構築プロセスで使用するソースファイル（rfile）のメタデータ構造体を初期化し、対象ファイルを読み取り専用で開く。

シグネチャまたは形状

static rfile *make_rfile(char *filename, bool missing_ok)

入力と出力

  • **入力**:
  • filename: 開く対象のファイルパス。
  • missing_ok: true の場合、ファイルが存在しなくてもエラーとせず NULL を返す。
  • **出力**:
  • 成功時: 初期化された rfile 構造体へのポインタ。
  • 失敗時: missing_ok が true かつファイルが存在しない場合は NULL。それ以外は pg_fatal によりプロセス終了。

前提条件と不変条件

  • rfile 構造体は pg_malloc0 によりゼロ初期化される。
  • ファイルは O_RDONLY | PG_BINARY モードで開かれる。

アルゴリズムまたは振る舞い

1. rfile 構造体用のメモリを確保し、ファイル名をコピーする。 2. 指定された filename を開く。 3. ファイルオープンに失敗した場合:

  • missing_ok が true かつエラーが ENOENT であれば、確保したメモリを解放して NULL を返す。
  • それ以外の場合は pg_fatal を呼び出し、エラーメッセージを出力して終了する。

    • 4. 成功した場合は、ファイル記述子を格納した rfile 構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_combinebackup/reconstruct.c:184
  • src/bin/pg_combinebackup/reconstruct.c:461
  • **呼び出し先**:
  • pg_malloc0, pstrdup, open, pg_free, pg_fatal（いずれもPostgreSQLのユーティリティ関数）。

関連データ構造

  • rfile: 再構築に使用するソースファイルのメタデータ（ファイル名、ファイル記述子など）を保持する構造体。

失敗モードと境界条件

  • ファイルが存在しない場合、missing_ok が false であれば pg_fatal により異常終了する。
  • メモリ確保失敗時は pg_malloc0 の挙動に従い終了する。
  • ファイルオープン失敗（権限不足等）時は pg_fatal により終了する。

保守担当者向けメモ

  • 本関数は rfile のライフサイクル管理の起点となる。返された rfile は後続の処理で適切にクローズおよび解放される必要がある。
