CreateProceduralLanguage

定義

  • 種別: function
  • 定義箇所: src/backend/commands/proclang.c:36

目的

CREATE [OR REPLACE] LANGUAGE 文のバックエンド処理を実行し、PostgreSQLの手続き型言語（PL）をシステムカタログ pg_language に登録または更新します。

シグネチャまたは形状

ObjectAddress CreateProceduralLanguage(CreatePLangStmt *stmt)

入力と出力

  • **入力**: CreatePLangStmt *stmt（SQL文の解析結果を保持する構造体）
  • **出力**: ObjectAddress（作成または更新された言語オブジェクトの識別子）

前提条件と不変条件

  • 実行ユーザーはスーパーユーザーである必要があります。
  • 指定されたハンドラ関数は language_handler 型を戻り値として持つ必要があります。
  • OR REPLACE が指定されていない場合、同名の言語が既に存在するとエラーになります。

アルゴリズムまたは振る舞い

1. **権限チェック**: superuser() を呼び出し、権限がない場合はエラーを発生させます。 2. **関数検証**: LookupFuncName を使用して、ハンドラ関数、インラインハンドラ、バリデータ関数の存在とシグネチャを検証します。 3. **カタログ操作**: pg_language を RowExclusiveLock で開き、既存の言語であれば heap_modify_tuple で更新、新規であれば heap_form_tuple で挿入します。 4. **依存関係構築**: recordDependencyOn... 系関数を用いて、言語と所有者、拡張機能、および各ハンドラ関数との依存関係を pg_depend に記録します。 5. **フック**: InvokeObjectPostCreateHook を呼び出し、外部モジュールへ作成を通知します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（CREATE LANGUAGE 文の実行時）
  • **呼び出し先**: LookupFuncName, get_func_rettype, table_open, SearchSysCache1, heap_modify_tuple, CatalogTupleUpdate, GetNewOidWithIndex, heap_form_tuple, CatalogTupleInsert, recordDependencyOnOwner, recordDependencyOnCurrentExtension, record_object_address_dependencies, InvokeObjectPostCreateHook 等。

関連データ構造

  • CreatePLangStmt: SQL文の構文解析結果。
  • pg_language: 言語定義を保持するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトの識別情報。

失敗モードと境界条件

  • **権限不足**: スーパーユーザー以外が実行した場合、ERRCODE_INSUFFICIENT_PRIVILEGE で失敗します。
  • **型不一致**: ハンドラ関数の戻り値が language_handler でない場合、ERRCODE_WRONG_OBJECT_TYPE で失敗します。
  • **重複**: 言語が既に存在し、かつ OR REPLACE が指定されていない場合、ERRCODE_DUPLICATE_OBJECT で失敗します。

保守担当者向けメモ

  • REPLACE 実行時、OID、所有者、権限は維持されます。
  • 依存関係の更新時、所有者や権限に関連する共有依存関係は変更されません。
  • NOT_USED マクロで囲まれた所有者チェックコードは、現状では superuser() チェックにより冗長となっています。
