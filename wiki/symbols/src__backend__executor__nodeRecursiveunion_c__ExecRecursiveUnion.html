ExecRecursiveUnion

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeRecursiveunion.c:80

目的

SQLの WITH RECURSIVE 句における再帰的クエリ評価を実行します。非再帰項（ベースケース）と再帰項（再帰ステップ）を順次実行し、結果を統合します。

シグネチャまたは形状

TupleTableSlot *ExecRecursiveUnion(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（RecursiveUnionState としてキャストされる実行ノードの状態）。
  • **出力**: TupleTableSlot *（再帰クエリの結果として得られたタプル、または終了時の NULL）。

前提条件と不変条件

  • pstate は RecursiveUnionState 型であること。
  • node->working_table および node->intermediate_table が初期化済みであること。
  • CHECK_FOR_INTERRUPTS() により、実行中に割り込みチェックが行われる。

アルゴリズムまたは振る舞い

1. **非再帰項の実行**: recursing フラグが偽の場合、outerPlan を実行。重複排除が必要な場合（plan->numCols > 0）、ハッシュテーブルで確認し、新規タプルのみを working_table に格納して返す。完了後、recursing を真にする。 2. **再帰項の実行**: innerPlan を実行。

  • innerPlan が空の場合、intermediate_table が空なら終了。空でなければ、working_table をクリアし、intermediate_table とスワップして再帰ステップを継続する。
  • 重複排除が必要な場合、ハッシュテーブルで確認。
  • 新規タプルであれば intermediate_table に格納し、呼び出し元へ返す。

    • 3. **メモリ管理**: 各ハッシュテーブルルックアップ後に node->tempContext をリセットし、メモリ消費を抑制する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（ExecProcNode 経由）。
  • **呼び出し先**: ExecProcNode（outerPlan および innerPlan の実行）、LookupTupleHashEntry（重複排除）、tuplestore_puttupleslot（タプル保存）、tuplestore_clear（タプルストアのクリア）、MemoryContextReset。

関連データ構造

  • RecursiveUnionState: 再帰実行の状態を保持。
  • RecursiveUnion: 実行計画ノード。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • plan->numCols > 0 の場合、ハッシュテーブルによる重複排除が発生し、メモリ使用量が増大する可能性がある。
  • intermediate_empty フラグが再帰終了の判定基準となる。
  • innerPlan が空になった際、working_table と intermediate_table のスワップにより、メモリの再割り当てを回避する最適化が行われる。

保守担当者向けメモ

  • UNION（重複排除あり）と UNION ALL（重複排除なし）で動作が分岐するため、修正時は両方のパスを確認すること。
  • 無限ループを避けるため、intermediate_empty フラグの遷移ロジックを慎重に扱うこと。
  • tempContext のリセット漏れはメモリリークに直結するため注意が必要。
