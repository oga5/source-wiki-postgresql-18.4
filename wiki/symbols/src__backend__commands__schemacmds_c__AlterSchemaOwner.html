AlterSchemaOwner

定義

  • 種別: function
  • 定義箇所: src/backend/commands/schemacmds.c:329

目的

指定されたスキーマの所有者を変更し、システムカタログの更新および依存関係の再計算を行う。

シグネチャまたは形状

ObjectAddress AlterSchemaOwner(const char *name, Oid newOwnerId)

入力と出力

  • **入力**:
  • name: 変更対象のスキーマ名。
  • newOwnerId: 新しい所有者のロールOID。
  • **出力**:
  • ObjectAddress: 変更されたスキーマのオブジェクトアドレス（クラスIDとオブジェクトOID）。

前提条件と不変条件

  • pg_namespace システムカタログに対する RowExclusiveLock が保持される。
  • 指定されたスキーマ名がシステムカタログに存在すること。

アルゴリズムまたは振る舞い

1. pg_namespace リレーションを RowExclusiveLock でオープンする。 2. NAMESPACENAME システムキャッシュを使用して対象スキーマを検索する。存在しない場合はエラーを発生させる。 3. 取得したタプルからスキーマのOIDを抽出する。 4. AlterSchemaOwner_internal を呼び出し、所有者変更の共通ロジック（ACL再計算や依存関係更新）を実行する。 5. ObjectAddressSet を使用して戻り値のアドレスを構築する。 6. システムキャッシュを解放し、リレーションをクローズして終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c:847 (SQLの ALTER SCHEMA ... OWNER TO 文の処理)
  • **呼び出し先**:
  • table_open / table_close
  • SearchSysCache1 / ReleaseSysCache
  • AlterSchemaOwner_internal
  • ObjectAddressSet

関連データ構造

  • ObjectAddress: 操作対象のオブジェクトを識別する構造体。
  • Form_pg_namespace: pg_namespace カタログのタプル構造。
  • NamespaceRelationId: pg_namespace システムカタログのOID。

失敗モードと境界条件

  • 指定されたスキーマ名が存在しない場合、ERRCODE_UNDEFINED_SCHEMA でエラー終了する。
  • 権限チェック（AlterSchemaOwner_internal 内で実施）に失敗した場合、エラー終了する。

保守担当者向けメモ

  • 本関数は高レベルなインターフェースであり、実際のカタログ更新ロジックは AlterSchemaOwner_internal にカプセル化されている。
  • 所有者変更に伴うイベントトリガーやキャッシュ無効化の通知は、内部関数側で処理される。
