bbsink_copystream_begin_manifest

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_copy.c:259

目的

バックアップマニフェストの送信開始をクライアントに通知するため、COPYプロトコル上でマニフェスト開始を示す識別子を送信する。

シグネチャまたは形状

static void bbsink_copystream_begin_manifest(bbsink *sink)

入力と出力

  • 入力: bbsink *sink （対象のシンク構造体）
  • 出力: なし（副作用としてネットワークストリームへメッセージを送信）

前提条件と不変条件

  • bbsink_copystream が初期化され、COPYモードに移行済みであること。

アルゴリズムまたは振る舞い

1. pq_beginmessage を使用して PqMsg_CopyData メッセージの構築を開始する。 2. pq_sendbyte を使用して、マニフェストデータであることを示すタイプバイト 'm' をバッファに書き込む。 3. pq_endmessage を呼び出し、構築したメッセージをクライアントへ送信する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/backup/basebackup_copy.c 内の bbsink_copystream_ops 構造体を通じて、バックアップ処理のメインループから呼び出される。
  • 呼び出し先: pq_beginmessage, pq_sendbyte, pq_endmessage （libpqプロトコル関数）。

関連データ構造

  • bbsink: basebackup_sink.h で定義されるシンクインターフェース。
  • StringInfoData: メッセージ構築用の一時バッファ。

失敗モードと境界条件

  • ネットワークエラーが発生した場合、pq_ 系関数内部でエラーが送出される可能性がある。

保守担当者向けメモ

  • 本関数は bbsink_copystream_ops の begin_manifest フィールドに登録されている。
  • 送信されるタイプバイト 'm' は、クライアント側で受信したデータがマニフェストであることを識別するために使用される。
