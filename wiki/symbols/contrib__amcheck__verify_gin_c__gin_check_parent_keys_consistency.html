gin_check_parent_keys_consistency

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_gin.c:389

目的

GINインデックスの物理的な整合性を検証するため、ルートページから深さ優先探索（DFS）を行い、親ページと子ページのキー関係およびツリー構造の不変条件をチェックする。

シグネチャまたは形状

static void gin_check_parent_keys_consistency(Relation rel, Relation heaprel, void *callback_state, bool readonly)

入力と出力

  • **入力**:
  • rel: 検証対象のGINインデックスリレーション。
  • heaprel: 関連するヒープリレーション。
  • callback_state: 呼び出し元から渡される状態情報。
  • readonly: 読み取り専用モードフラグ。
  • **出力**: なし（検証失敗時は ereport(ERROR, ...) により例外をスロー）。

前提条件と不変条件

  • 検証は AccessShareLock 下で実行されるため、並行処理によるページ分割（Split）が発生し得る。
  • 全ての枝においてツリーの高さが一致していること。
  • 各ページ内のタプルは適切に順序付けられ、親ページのキーと整合していること。

アルゴリズムまたは振る舞い

1. メモリコンテキストを作成し、GinState を初期化する。 2. ルートページをスタックに積み、DFSを開始する。 3. 各ページに対して以下を実行する:

  • check_index_page によるヘッダーの健全性確認。
  • ページ分割が発生している場合、右兄弟ページをスタックに追加する。
  • リーフページ到達時にツリーの深さを記録し、全枝で一致するか確認する。
  • ページ内のタプル順序を検証し、親ページのダウンリンクとの整合性を確認する。

    • 4. 矛盾を検知した場合は ERRCODE_INDEX_CORRUPTED を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_gin.c 内の gin_index_check からの呼び出しが確認されている。
  • **呼び出し先**: check_index_page, gintuple_get_key, gintuple_get_attrnum, ginCompareAttEntries, PageGetItemIdCareful 等。

関連データ構造

  • GinScanItem: 走査中のスタック状態を保持する構造体。
  • GinState: GINインデックスの操作状態を管理する構造体。

失敗モードと境界条件

  • **タプル順序の不整合**: ページ内のタプルが昇順でない場合、エラーとなる。
  • **ツリーの高さの不一致**: 枝によってリーフまでの深さが異なる場合、エラーとなる。
  • **タプルサイズの不整合**: ItemId と IndexTuple のサイズが一致しない場合、エラーとなる。
  • **並行処理**: ページ分割による一時的な不整合は許容されるが、親キーとの比較で矛盾が解消できない場合はエラーとなる。

保守担当者向けメモ

  • 本関数は AccessShareLock で動作するため、検証中にインデックスが更新される可能性がある。
  • 検証失敗時は物理的な破損を示唆するため、REINDEX が推奨される。
  • DEBUG3 ログにより、ページ処理の詳細な追跡が可能である。
