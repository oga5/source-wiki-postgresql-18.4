tarGets

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_tar.c:417

目的

TARアーカイブ内の特定のメンバーから、改行文字または指定された長さの上限に達するまで文字列を読み取ります。

シグネチャまたは形状

char *tarGets(char *buf, size_t len, TAR_MEMBER *th)

入力と出力

  • **入力**:
  • buf: 読み取った文字列を格納するバッファ。
  • len: 読み取りの最大長。
  • th: 対象となる TAR_MEMBER 構造体へのポインタ。
  • **出力**:
  • 成功時は buf へのポインタを返します。
  • EOFに達し、かつ読み取ったデータが空の場合は NULL を返します。

前提条件と不変条件

  • th->fileLen および th->pos を使用して、論理的なEOFを超えた読み取りが行われないよう制限されます。
  • 読み取り成功時には、th->pos が読み取ったバイト数分だけ更新されます。

アルゴリズムまたは振る舞い

1. 現在の読み取り位置とファイル長に基づき、読み取り可能な最大長を len に再設定します。 2. _tarReadRaw を呼び出し、1バイトずつ buf に格納します。 3. 改行文字 (\n) が検出されるか、len に達するまでループを継続します。 4. 読み取り終了後、バッファの末尾にヌル終端文字を付与します。 5. 読み取った文字列の長さを計算し、th->pos を進めます。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、特定の呼び出し元は特定されていません。
  • **呼び出し先**: _tarReadRaw を呼び出します。

関連データ構造

  • TAR_MEMBER: TARアーカイブ内のメンバー情報を保持する構造体。

失敗モードと境界条件

  • _tarReadRaw が0以下の値を返した場合（読み取りエラーまたは物理的なEOF）、ループを中断し、それまでに読み取った内容を返します。
  • 読み取り開始時点でEOFに達している場合、NULL を返します。

保守担当者向けメモ

  • 本関数は標準の fgets に類似した動作をしますが、TAR_MEMBER コンテキスト内の位置情報を管理する点が異なります。
  • 圧縮形式には対応していないため、本関数は非圧縮のTARストリームに対してのみ動作します。
