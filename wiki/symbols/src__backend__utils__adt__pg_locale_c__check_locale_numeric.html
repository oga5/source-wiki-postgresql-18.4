check_locale_numeric

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:377

目的

GUC（Grand Unified Configuration）パラメータとして設定されたロケール名が、数値フォーマット（LC_NUMERIC）としてシステム上で有効かどうかを検証します。

シグネチャまたは形状

bool check_locale_numeric(char **newval, void **extra, GucSource source);

入力と出力

  • **入力**:
  • newval: 設定しようとしているロケール名の文字列へのポインタ。
  • extra: GUC フック用の補助データ（本関数では使用されません）。
  • source: 設定のソース（GUC 設定元）。
  • **出力**:
  • 戻り値: 指定されたロケールが有効であれば true、無効であれば false を返します。

前提条件と不変条件

  • LC_NUMERIC カテゴリに対するロケール検証を行います。
  • 内部で呼び出される check_locale が、OS のロケール環境に依存して妥当性を判断します。

アルゴリズムまたは振る舞い

1. 引数として受け取った newval を、LC_NUMERIC カテゴリを指定して check_locale 関数へ渡します。 2. check_locale の判定結果（成功または失敗）をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc_tables.c: GUC パラメータの検証フックとして登録されています。
  • **呼び出し先**:
  • check_locale(int category, const char *locale, char **canonname): ロケール名の妥当性チェックを実行する共通関数。

関連データ構造

  • GucSource: 設定値の由来を示す列挙型。

失敗モードと境界条件

  • 指定されたロケール名が現在の OS 環境でサポートされていない場合、check_locale が false を返し、本関数も false を返します。
  • newval が NULL の場合の挙動は check_locale の実装に依存します。

保守担当者向けメモ

  • 本関数は LC_NUMERIC 専用のラッパーです。同様のロジックが LC_TIME や LC_MONETARY 用にも存在するため、修正時は他の check_locale_* 関数との整合性に注意してください。
