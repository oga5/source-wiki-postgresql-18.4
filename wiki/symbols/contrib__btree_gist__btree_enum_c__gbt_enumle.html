gbt_enumle

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:51

目的

PostgreSQL の ENUM 型に対する GiST インデックスの「小なりイコール（<=）」比較操作を提供します。

シグネチャまたは形状

static bool gbt_enumle(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の ENUM 値を指すポインタ（内部的に Oid として扱われる）。
  • flinfo: 関数マネージャー情報。
  • **出力**:
  • bool: a <= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な Oid を指している必要があります。
  • ENUM 型は内部的に OID として順序付けられていることを前提としています。

アルゴリズムまたは振る舞い

1. a と b が指す値を Oid にキャストし、ObjectIdGetDatum を用いて Datum 型に変換します。 2. CallerFInfoFunctionCall2 を介して、PostgreSQL 標準の enum_le 関数を呼び出します。 3. enum_le の結果（Datum）を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_enum.c 内の tinfo 構造体（gbtree_ninfo）の初期化時に参照されています。
  • **呼び出し先**: enum_le（PostgreSQL 標準の ENUM 比較関数）。

関連データ構造

  • tinfo: gbtree_ninfo 構造体。本関数を比較演算子として保持し、GiST ユーティリティ関数へ提供します。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、メモリ参照エラーが発生する可能性があります。
  • flinfo が無効な場合、CallerFInfoFunctionCall2 の実行時に例外が発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された汎用 GiST ユーティリティからコールバックされることを想定しています。
  • ENUM 型の比較ロジックに変更が必要な場合は、標準の enum_le の仕様を確認してください。
