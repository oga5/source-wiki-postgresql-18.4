assign_timezone

定義

  • 種別: function
  • 定義箇所: src/backend/commands/variable.c:380

目的

GUC（Grand Unified Configuration）サブシステムにおいて、セッションのタイムゾーン設定を更新し、関連するキャッシュを無効化する。

シグネチャまたは形状

void assign_timezone(const char *newval, void *extra)

入力と出力

  • newval: 設定されるタイムゾーン名の文字列（本関数内では直接使用されない）。
  • extra: check_timezone 等のバリデーションフェーズで準備された、解決済みの pg_tz 構造体へのポインタ。
  • 出力: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • extra 引数は、有効な pg_tz 構造体へのポインタを指している必要がある。
  • 本関数は、GUCフレームワークの assign_hook として呼び出されることを前提としている。

アルゴリズムまたは振る舞い

1. extra が指す pg_tz 構造体のポインタを、グローバル変数 session_timezone に代入する。 2. ClearTimeZoneAbbrevCache() を呼び出し、datetime.c が保持しているタイムゾーン略称のキャッシュをクリアする。これにより、次回の日付時刻処理で新しいタイムゾーン設定が反映される。

呼び出し元と呼び出し先

  • 呼び出し元: GUCサブシステム（src/backend/utils/misc/guc_tables.c 等でフックとして登録）。
  • 呼び出し先: ClearTimeZoneAbbrevCache()（utils/datetime.h 経由）。

関連データ構造

  • pg_tz: タイムゾーン情報を保持する構造体。
  • session_timezone: セッションの現在のタイムゾーンを保持するグローバル変数。

失敗モードと境界条件

  • extra が NULL の場合、デリファレンス時にクラッシュする可能性がある（GUCフレームワークのバリデーションが正常に機能していることが前提）。

保守担当者向けメモ

  • 本関数は assign_hook であるため、バリデーションは呼び出し前の check_timezone で完了している必要がある。
  • タイムゾーン変更に伴うキャッシュの整合性を保つため、ClearTimeZoneAbbrevCache の呼び出しは必須である。
