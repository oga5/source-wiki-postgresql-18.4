mda_get_offset_values

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayutils.c:182

目的

多次元配列のインデックス走査において、現在の次元位置から次の次元へ移動する際に必要なオフセット値の配列（dist）を計算します。

シグネチャまたは形状

void mda_get_offset_values(int n, int *dist, const int *prod, const int *span)

入力と出力

  • **入力**:
  • n: 配列の次元数。
  • prod: 各次元の要素数の積（prod[i] は次元 i のステップサイズ）。
  • span: 各次元の範囲（要素数）。
  • **出力**:
  • dist: 計算されたオフセット値が格納される配列（呼び出し元で確保済みであること）。

前提条件と不変条件

  • dist, prod, span は少なくとも n 個の要素を持つ配列である必要があります。
  • n は正の整数であることを前提としています。

アルゴリズムまたは振る舞い

1. dist[n - 1] を 0 に初期化します。 2. 最下位次元から順に（n-2 から 0 まで）、以下の計算を行います。

  • dist[j] を prod[j] - 1 で初期化します。
  • j より深い各次元 i について、(span[i] - 1) * prod[i] を dist[j] から減算します。

    • 3. この処理により、多次元配列のインデックスを線形メモリ空間上で走査する際のステップ値が決定されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arrayfuncs.c: 複数箇所（行 5070, 5133, 5219）で呼び出されています。
  • **呼び出し先**:
  • 本関数は計算ロジックのみで構成されており、他の関数を呼び出しません。

関連データ構造

  • ArrayType: PostgreSQLの配列内部表現（utils/array.h で定義）。本関数は、この構造体に含まれる次元情報や範囲情報を処理するために使用されます。

失敗モードと境界条件

  • n が 0 以下の場合、配列アクセス違反が発生する可能性があります。
  • dist, prod, span が n 未満のサイズである場合、メモリ破壊が発生します。

保守担当者向けメモ

  • 本関数は「Multi-Dimensional Array」操作用ユーティリティの一部です。
  • mda_ 系関数は、呼び出し元が既に次元や境界の妥当性を確認していることを前提としています。
  • 内部のループ計算は、多次元配列の線形配置に基づいた数学的なオフセット計算であるため、変更時は配列の次元構造との整合性に注意してください。
