ordered_set_transition_multi

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/orderedsetaggs.c:382

目的

複数カラムを入力として受け取る順序集合集約関数において、入力行をソート用バッファ（tuplesort）へ蓄積するための遷移関数。

シグネチャまたは形状

Datum ordered_set_transition_multi(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（集約状態および入力カラム群）。
  • **出力**: Datum（更新された OSAPerGroupState へのポインタ）。

前提条件と不変条件

  • 最初の呼び出し時、PG_ARGISNULL(0) が真であれば ordered_set_startup を呼び出し、状態を初期化する必要がある。
  • 状態が初期化済みの場合、PG_GETARG_POINTER(0) から OSAPerGroupState を取得する。
  • 蓄積される行は、osastate->qstate->tupslot を介して tuplesort に渡される。

アルゴリズムまたは振る舞い

1. **状態の取得**: 初回呼び出しであれば ordered_set_startup を実行し、そうでなければ既存の OSAPerGroupState を取得する。 2. **タプルの構築**:

  • ExecClearTuple でスロットをクリアする。
  • 入力引数をスロットの各属性に格納する。
  • AGGKIND_HYPOTHETICAL（仮説的集約）の場合、ソート対象の行を識別するためのフラグ値（0）をスロットの末尾に追加する。

    • 3. **ソートへの投入**: tuplesort_puttupleslot を呼び出し、構築したタプルを tuplesort オブジェクトに書き込む。 4. **カウンタ更新**: osastate->number_of_rows をインクリメントする。 5. **返却**: 更新された osastate を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の集約実行エンジン（nodeAgg.c 等から間接的に呼び出される）。
  • **呼び出し先**:
  • ordered_set_startup: 初期化時。
  • ExecClearTuple, ExecStoreVirtualTuple: タプル操作。
  • tuplesort_puttupleslot: ソートバッファへの書き込み。

関連データ構造

  • OSAPerGroupState: グループごとの実行状態を保持。
  • TupleTableSlot: 入力行を一時的に保持する構造。
  • Aggref: 集約関数のメタデータ（aggkind の判定に使用）。

失敗モードと境界条件

  • tuplesort のメモリ制限（work_mem）を超過した場合、一時ファイルが生成される可能性がある。
  • Assert により、構築したタプルの属性数が slot->tts_tupleDescriptor->natts と一致することが検証される。

保守担当者向けメモ

  • 本関数は複数カラム入力を扱うため、ordered_set_transition（単一カラム用）とは異なるタプル構築ロジックを持つ。
  • 仮説的集約におけるフラグ値の付与は、ソート順序を制御するための重要な実装である。
