目的

ECPG（Embedded SQL for C）プリプロセッサの回帰テスト用エントリポイントであり、INSERT および UPDATE 文（特に RETURNING 句）の動作検証を行う。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム引数を使用しない）
  • 出力: 成功時に 0 を返し、標準出力にテスト結果の検証用文字列を出力する。

前提条件と不変条件

  • データベース ecpg1_regression への接続が可能であること。
  • 実行環境において ECPGdo および ECPGconnect 等のECPGランタイムライブラリが利用可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、ECPGconnect でデータベースに接続する。 2. **DDL実行**: create table insupd_test (a int, b int) を実行し、テスト用テーブルを作成する。 3. **DML実行**:

  • INSERT 文を複数回実行し、データを投入する。一部の INSERT では RETURNING a を使用し、ホスト変数 i4 に結果を格納する。
  • UPDATE 文を実行し、RETURNING a を使用して更新後の値を配列 i3 に格納する。
  • 追加の UPDATE 文でテーブルの状態を操作する。

    • 4. **検証**: SELECT 文を実行し、結果を配列 i1, i2 に取得する。printf を用いて期待値と一致するかを確認する。 5. **終了**: ECPGdisconnect で接続を終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGdisconnect, sqlprint, printf

関連データ構造

  • sqlca: SQL実行結果や警告を保持するグローバル構造体。
  • ホスト変数: i1, i2, i3（配列）, i4（整数）。

失敗モードと境界条件

  • sqlca.sqlcode < 0 の場合、エラーが発生したとみなされ sqlprint() が呼び出される。
  • sqlca.sqlwarn[0] == 'W' の場合、警告として sqlprint() が呼び出される。
