InitCompressorGzip

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:238

目的

CompressorState 構造体に対し、GZIP 圧縮・解凍処理に必要な関数ポインタおよび設定を割り当て、初期化を行う。

シグネチャまたは形状

void InitCompressorGzip(CompressorState *cs, const pg_compress_specification compression_spec);

入力と出力

  • **入力**:
  • cs: 初期化対象となる CompressorState 構造体へのポインタ。
  • compression_spec: 圧縮設定を指定する pg_compress_specification 構造体。
  • **出力**: なし（cs が指す構造体のメンバを更新する）。

前提条件と不変条件

  • cs は有効なメモリ領域を指している必要がある。
  • cs->writeF が設定されている場合、DeflateCompressorInit が呼び出され、内部的な圧縮状態が構築される。

アルゴリズムまたは振る舞い

1. cs の関数ポインタメンバに GZIP 実装を割り当てる:

  • readData = ReadDataFromArchiveGzip
  • writeData = WriteDataToArchiveGzip
  • end = EndCompressorGzip

    • 2. compression_spec を cs->compression_spec に格納する。 3. cs->writeF が非 NULL の場合、DeflateCompressorInit(cs) を呼び出し、圧縮用バッファや zlib ストリームの初期化を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_io.c:135
  • **呼び出し先**:
  • DeflateCompressorInit (条件付き)

関連データ構造

  • CompressorState: 圧縮サブシステムの基盤となる状態管理構造体。
  • pg_compress_specification: 圧縮レベルやアルゴリズム等の設定を保持する構造体。

失敗モードと境界条件

  • HAVE_LIBZ が定義されていない環境では、初期化関数はエラーを発生させるスタブとして動作する（ソース概要より）。
  • データが空の場合、Write が呼び出される前に End が呼び出される可能性がある。

保守担当者向けメモ

  • 本関数は compress_io.c が定義する汎用インターフェースに対する GZIP 固有の実装である。
  • gzgetc マクロはライブラリとの不整合を避けるため #undef されている点に注意すること。
