FullTransactionIdNewer

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/transam.h:359

目的

2つの FullTransactionId を比較し、より新しい（後の）トランザクションIDを返します。

シグネチャまたは形状

static inline FullTransactionId FullTransactionIdNewer(FullTransactionId a, FullTransactionId b)

入力と出力

  • **入力**: 比較対象となる2つの FullTransactionId 型の値 a および b。
  • **出力**: a と b のうち、FullTransactionIdFollows の判定に基づき、より新しいと判断された FullTransactionId。

前提条件と不変条件

  • FullTransactionIdIsValid マクロにより、入力値が有効なトランザクションIDであることが確認されます。
  • 比較ロジックは FullTransactionIdFollows に依存しており、トランザクションIDの循環やエポックを考慮した順序関係が維持されます。

アルゴリズムまたは振る舞い

1. a が無効な場合、b を返します。 2. b が無効な場合、a を返します。 3. FullTransactionIdFollows(a, b) が真であれば a を返し、そうでなければ b を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/procarray.c 内の複数の箇所（GlobalVisSharedRels, GlobalVisCatalogRels, GlobalVisDataRels 等の可視性判定に関連する処理）から呼び出されています。
  • **呼び出し先**: FullTransactionIdIsValid および FullTransactionIdFollows を内部的に使用します。

関連データ構造

  • FullTransactionId: 64ビットのトランザクション識別子。

失敗モードと境界条件

  • 入力値のいずれか一方が無効（InvalidTransactionId 等）である場合、有効な方の値を優先して返します。両方が無効な場合の挙動は、FullTransactionIdIsValid の定義に依存します。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、頻繁に呼び出される可視性判定処理のパフォーマンスに影響します。変更時は FullTransactionIdFollows の比較ロジックとの整合性に注意してください。
