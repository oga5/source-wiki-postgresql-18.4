LookupWSErrorMessage

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/win32.c:217

目的

Winsockエラーコードに対応する説明文を静的テーブルから検索し、指定されたバッファにコピーします。

シグネチャまたは形状

static int LookupWSErrorMessage(DWORD err, char *dest);

入力と出力

  • **入力**:
  • err: 検索対象のWinsockエラーコード (DWORD)。
  • dest: エラーメッセージを格納するためのバッファ (char *)。
  • **出力**:
  • 成功時: 1 を返し、dest に対応する説明文を書き込む。
  • 失敗時: 0 を返し、dest は変更されない。

前提条件と不変条件

  • dest は、エラーメッセージを格納するのに十分なサイズを持つバッファである必要があります。
  • WSErrors テーブルは description メンバが NULL であるエントリで終端されている必要があります。

アルゴリズムまたは振る舞い

1. WSErrors テーブルを先頭から順に走査します。 2. 現在のエントリの error フィールドが引数 err と一致する場合、その description を dest にコピーし、1 を返して終了します。 3. テーブルの終端（description が NULL のエントリ）に達した場合は、何もコピーせず 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/win32.c 内の winsock_strerror 関数から呼び出されます。
  • **呼び出し先**: 外部関数は呼び出しません。

関連データ構造

  • WSErrorEntry: エラーコードと説明文のペアを保持する構造体。
  • WSErrors: WSErrorEntry の静的配列。

失敗モードと境界条件

  • 指定されたエラーコードが WSErrors テーブルに存在しない場合、関数は 0 を返し、呼び出し元は別の手段（FormatMessage 等）でエラーメッセージを取得する必要があります。

保守担当者向けメモ

  • 本関数は線形探索を行うため、WSErrors テーブルのサイズが増大した場合はパフォーマンスに影響する可能性がありますが、現在の用途では許容範囲内とされています。
