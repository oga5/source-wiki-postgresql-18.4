InternalIpcSemaphoreCreate

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_sema.c:96

目的

System V セマフォセットをカーネルに要求し、新規作成する。

シグネチャまたは形状

static IpcSemaphoreId InternalIpcSemaphoreCreate(IpcSemaphoreKey semKey, int numSems, bool retry_ok)

入力と出力

  • **入力**:
  • semKey: セマフォセットを識別するためのキー。
  • numSems: 作成するセマフォの数。
  • retry_ok: 作成失敗時に呼び出し元へ制御を戻す（-1 を返す）か、FATAL エラーとして終了させるかを指定するフラグ。
  • **出力**:
  • 成功時: 作成されたセマフォセットの ID (semId)。
  • 失敗時: retry_ok が真かつ特定の errno の場合 -1。それ以外は ereport(FATAL, ...) によりプロセス終了。

前提条件と不変条件

  • semget システムコールを IPC_CREAT | IPC_EXCL フラグ付きで呼び出し、既存のセマフォセットとの衝突を回避する。
  • IPCProtection 定数で定義された権限設定が適用される。

アルゴリズムまたは振る舞い

1. semget を呼び出し、指定されたキーと数でセマフォセットの作成を試みる。 2. 成功した場合は semId を返す。 3. 失敗した場合、errno を保存し、retry_ok が真かつエラーが EEXIST, EACCES, EINVAL, EIDRM のいずれかであれば -1 を返す。 4. 上記以外、または retry_ok が偽の場合は ereport(FATAL, ...) を呼び出す。特に ENOSPC の場合は、カーネルパラメータ（SEMMNI, SEMMNS）の不足を示唆するヒントをログに出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/sysv_sema.c 内の IpcSemaphoreCreate 関数から呼び出される。
  • **呼び出し先**:
  • semget (システムコール)
  • ereport (PostgreSQL エラー報告メカニズム)

関連データ構造

  • IpcSemaphoreId: セマフォセット識別子（int）。
  • IpcSemaphoreKey: セマフォセットを特定するキー（key_t）。

失敗モードと境界条件

  • **EEXIST**: 指定キーのセマフォセットが既に存在する。
  • **EACCES**: 権限不足。
  • **EINVAL**: numSems がカーネルの SEMMSL 制限を超えている、またはキーが無効。
  • **EIDRM**: セマフォセットが削除対象となっている。
  • **ENOSPC**: システム全体のセマフォセット数（SEMMNI）またはセマフォ数（SEMMNS）の制限超過。

保守担当者向けメモ

  • 本関数は semget のラッパーであり、PostgreSQL のセマフォ管理における低レベルなリソース確保を担当する。
  • EINVAL はプラットフォームの制限（SEMMSL）に依存するため、このエラーが発生した場合はカーネル設定の確認が必要である。
  • retry_ok パラメータは、呼び出し元がセマフォの再利用やクリーンアップを試みるための柔軟性を提供するために存在する。
