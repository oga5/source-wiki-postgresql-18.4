check_lateral_ref_ok

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:490

目的

SQLクエリにおける LATERAL 参照の妥当性を検証し、制約に違反している場合にエラーを報告する。

シグネチャまたは形状

static void check_lateral_ref_ok(ParseState *pstate, ParseNamespaceItem *nsitem, int location)

入力と出力

  • pstate: 現在の解析状態を保持する ParseState 構造体へのポインタ。
  • nsitem: 参照対象の ParseNamespaceItem。
  • location: エラー発生時のパーサー位置情報。
  • 戻り値: なし（正常時は何もせず終了、異常時は ereport(ERROR, ...) により処理を中断）。

前提条件と不変条件

  • nsitem が p_lateral_only フラグを保持している場合、その参照が許可されているか（p_lateral_ok）を検証する。
  • SQL:2008規格に基づき、LATERAL参照が許可されていない文脈でのアクセスはエラーとする。

アルゴリズムまたは振る舞い

1. nsitem->p_lateral_only が真かつ nsitem->p_lateral_ok が偽であるかを確認する。 2. 条件に合致する場合、ereport を呼び出し ERRCODE_INVALID_COLUMN_REFERENCE エラーを発生させる。 3. エラーメッセージ生成時、pstate->p_target_nsitem との比較を行い、対象テーブルがクエリの特定の箇所から参照不可であることを示すヒント、または結合型（INNER/LEFT以外）に関する詳細情報を付与する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_relation.c 内の複数箇所（行229, 276, 933付近）で、名前空間アイテムの解決プロセス中に呼び出される。
  • 呼び出し先: ereport（PostgreSQLのエラー報告マクロ）、parser_errposition。

関連データ構造

  • ParseState: 解析状態を管理。
  • ParseNamespaceItem: 名前空間内のアイテム（RTEと可視性フラグを含む）。
  • RangeTblEntry: テーブルやサブクエリ等のリレーション情報。

失敗モードと境界条件

  • p_lateral_only が真であっても p_lateral_ok が真であれば、エラーは発生しない。
  • ターゲットとなる nsitem が p_target_nsitem と一致する場合、より具体的なヒントメッセージが生成される。

保守担当者向けメモ

  • 本関数はLATERAL制約の厳格な適用を担う。エラーメッセージの文言はSQL標準に準拠しているため、変更時は規格との整合性に注意すること。
  • p_lateral_ok フラグの管理ロジックが変更された場合、本関数による検証結果に直接影響を与える。
