prepare_common

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:158

目的

SQLステートメントのプリペア処理を共通化し、準備されたステートメントを接続オブジェクトのリストに登録する。

シグネチャまたは形状

static bool prepare_common(int lineno, struct connection *con, const char *name, const char *variable);

入力と出力

  • **入力**:
  • lineno: 呼び出し元の行番号。
  • con: 接続先を表す connection 構造体。
  • name: プリペアドステートメントの識別名。
  • variable: 実行するSQLクエリ文字列。
  • **出力**:
  • 成功時は true、失敗時は false を返す。

前提条件と不変条件

  • con は有効な接続オブジェクトであること。
  • ecpg_alloc によるメモリ確保が成功していること。
  • PQprepare がバックエンド側で正常に完了すること。

アルゴリズムまたは振る舞い

1. prepared_statement および statement 構造体をメモリ確保する。 2. statement 構造体にクエリ文字列をコピーし、replace_variables を呼び出してホスト変数を ? に置換する。 3. PQprepare を実行し、バックエンドにクエリを送信する。 4. ecpg_check_PQresult で結果を検証し、失敗時は確保したメモリを解放して終了する。 5. 成功時、prepared_statement を接続オブジェクトの prep_stmts リストの先頭に挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/prepare.c 内の関数（ECPGprepare 等から間接的に呼び出される）。
  • **呼び出し先**:
  • ecpg_alloc, ecpg_free, ecpg_strdup, ecpg_log, ecpg_check_PQresult
  • replace_variables
  • PQprepare, PQclear

関連データ構造

  • struct connection: 接続状態とプリペアドステートメントのリストを保持。
  • struct prepared_statement: プリペアドステートメントのメタデータとリスト構造を保持。
  • struct statement: 実行対象のSQLコマンドとコンテキストを保持。

失敗モードと境界条件

  • メモリ確保失敗時（ecpg_alloc がNULLを返す）は false を返す。
  • PQprepare がエラーを返した場合、確保したメモリを解放し false を返す。
  • 接続が確立されていない場合や無効なクエリの場合は ecpg_check_PQresult により失敗する。

保守担当者向けメモ

  • 本関数は prepared_statement リストの先頭に新しいエントリを追加するスタック構造をとる。
  • replace_variables によるクエリの書き換えが副作用として発生するため、呼び出し時のクエリ文字列の扱いに注意すること。
  • メモリ管理は ecpg_alloc/ecpg_free に依存しており、エラーパスでのリーク防止が重要である。
