build_inet_union_key

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_gist.c:471

目的

GiSTインデックスのノード統合処理において、複数のネットワークアドレス情報を保持するための新しい GistInetKey 構造体を生成・初期化します。

シグネチャまたは形状

GistInetKey *build_inet_union_key(int family, int minbits, int commonbits, unsigned char *addr)

入力と出力

  • **入力**:
  • family: アドレスファミリー（IPv4/IPv6）。
  • minbits: 統合対象の最小ネットマスク長。
  • commonbits: 共通プレフィックス長。
  • addr: 共通プレフィックス部分のアドレスバイト列。
  • **出力**:
  • 初期化済みの GistInetKey 構造体へのポインタ。

前提条件と不変条件

  • palloc0 によりメモリがゼロクリアされた状態で構造体が確保されます。
  • commonbits が0より大きい場合、指定された addr から必要なバイト数がコピーされます。
  • 最終的な GistInetKey は SET_GK_VARSIZE マクロにより、可変長データとしてのヘッダーが正しく設定されます。

アルゴリズムまたは振る舞い

1. GistInetKey 用のメモリを確保し、ゼロクリアします。 2. 引数で渡された family、minbits、commonbits を構造体の各フィールドに設定します。 3. commonbits に基づき、アドレスバイト列をコピーします。 4. コピーしたバイト列の末尾において、commonbits で指定された範囲外の不要なビットをマスク処理（ゼロクリア）します。 5. SET_GK_VARSIZE を呼び出し、構造体のサイズ情報をヘッダーに書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_gist.c:472
  • src/backend/utils/adt/network_gist.c:531
  • src/backend/utils/adt/network_gist.c:777
  • src/backend/utils/adt/network_gist.c:787
  • **呼び出し先**:
  • palloc0 (PostgreSQLメモリ管理)
  • memcpy (標準Cライブラリ)
  • SET_GK_VARSIZE (マクロ)

関連データ構造

  • GistInetKey: ネットワーク型GiSTインデックスのキー構造体。

失敗モードと境界条件

  • commonbits が8の倍数でない場合、最後のバイトの不要なビットをクリアする処理が実行されます。
  • メモリ確保失敗時は palloc0 の仕様に従い、PostgreSQLの標準的なエラーハンドリング（ereport(ERROR, ...)）がトリガーされます。

保守担当者向けメモ

  • 本関数は GistInetKey の内部構造に直接依存しています。構造体のレイアウト変更時には、ビットマスク処理および SET_GK_VARSIZE の整合性を再確認してください。
