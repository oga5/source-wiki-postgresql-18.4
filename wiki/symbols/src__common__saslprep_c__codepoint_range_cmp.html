codepoint_range_cmp

定義

  • 種別: function
  • 定義箇所: src/common/saslprep.c:968

目的

Unicodeコードポイントが、定義された範囲（下限・上限）に含まれるかどうかを判定するための比較関数です。

シグネチャまたは形状

static int codepoint_range_cmp(const void *a, const void *b)

入力と出力

  • **入力**:
  • a: 比較対象のコードポイント（pg_wchar * としてキャストして使用）。
  • b: 範囲を示す配列（pg_wchar[2] を想定し、range[0] が下限、range[1] が上限）。
  • **出力**:
  • -1: *a が範囲の下限より小さい場合。
  • 1: *a が範囲の上限より大きい場合。
  • 0: *a が範囲内（下限以上、上限以下）にある場合。

前提条件と不変条件

  • b は少なくとも2つの要素を持つ pg_wchar 型の配列を指している必要があります。
  • a は単一の pg_wchar 値を指している必要があります。
  • b が指す範囲は range[0] <= range[1] であることが前提です。

アルゴリズムまたは振る舞い

1. a が指す値をキーとして取得します。 2. b が指す配列の第1要素（下限）と比較し、キーが小さければ -1 を返します。 3. b が指す配列の第2要素（上限）と比較し、キーが大きければ 1 を返します。 4. 上記のいずれでもない場合、キーは範囲内にあると判断し 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/saslprep.c 内の is_code_in_table 関数から、bsearch の比較関数として呼び出されます。
  • **呼び出し先**: 外部関数への依存はありません。

関連データ構造

  • pg_wchar: PostgreSQLでUnicodeコードポイントを扱うための型。
  • prohibited_output_ranges 等の範囲テーブル（bsearch の検索対象）。

失敗モードと境界条件

  • a または b が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こします。
  • 範囲テーブルの定義が不正（range[0] > range[1]）な場合、常に 1 または -1 が返され、正しい判定が行われません。

保守担当者向けメモ

  • 本関数は bsearch と組み合わせて使用されることを前提としています。
  • pg_wchar のサイズやエンコーディングの整合性は、呼び出し側で保証する必要があります。
