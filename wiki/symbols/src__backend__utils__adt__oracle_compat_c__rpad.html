rpad

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/oracle_compat.c:260

目的

入力文字列（string1）の末尾に、指定された文字列（string2）を繰り返し連結し、指定された文字数（len）までパディングまたは切り詰めを行う。

シグネチャまたは形状

Datum rpad(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): パディング対象の文字列（text 型）。
  • PG_GETARG_INT32(1): 目標とする文字数（int32）。負の値は 0 として扱われる。
  • PG_GETARG_TEXT_PP(2): パディングに使用する文字列（text 型）。
  • **出力**:
  • Datum: パディングまたは切り詰め後の新しい text 型文字列。

前提条件と不変条件

  • len が負の場合、0 に補正される。
  • string1 の長さが len を超える場合、string1 は先頭から len 文字で切り詰められる。
  • string2 が空（長さ 0 以下）の場合、パディングは行われず、string1 の切り詰めのみが行われる。
  • マルチバイト文字環境において、文字境界を正しく維持して処理される。

アルゴリズムまたは振る舞い

1. string1 および string2 の長さを取得し、string1 を len 文字まで切り詰める。 2. 出力に必要な最大バイト数を計算し、オーバーフローチェックを行った上でメモリを確保する。 3. string1 の内容を結果バッファにコピーする。 4. len に達するまで、string2 の内容を繰り返しコピーする。string2 の末尾に達した場合は先頭に戻る（ラップアラウンド）。 5. 最終的なバイトサイズを確定し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出される（ソース内のコメントより、rpad(text, int4, text) として定義されている）。
  • **呼び出し先**:
  • pg_mbstrlen_with_len, pg_mblen_unbounded, pg_mblen_range: マルチバイト文字の長さ計算。
  • pg_mul_s32_overflow, pg_add_s32_overflow: バイトサイズ計算時のオーバーフローチェック。
  • palloc: メモリ確保。
  • memcpy: 文字列コピー。

関連データ構造

  • text: PostgreSQL の可変長文字列型。VARDATA_ANY や VARSIZE_ANY_EXHDR マクロを用いて操作される。

失敗モードと境界条件

  • **メモリ不足**: palloc が失敗した場合、PostgreSQL のエラーハンドリングによりアボートする。
  • **サイズ制限超過**: 計算された出力サイズが AllocSizeIsValid を満たさない場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーを報告する。
  • **負の長さ**: len が負の場合は 0 として処理される。

保守担当者向けメモ

  • 本関数はマルチバイト文字を考慮した走査を行っているため、新しい文字列操作関数を追加する際は pg_mblen_range 等の利用パターンを参考にすること。
  • string2 が空の場合の挙動（パディングなし）は仕様である。
