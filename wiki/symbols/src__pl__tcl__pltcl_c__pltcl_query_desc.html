pltcl_query_desc

定義

  • 種別: struct
  • 定義箇所: src/pl/tcl/pltcl.c:172

目的

SPI（Server Programming Interface）を介して実行される準備済みクエリのメタデータおよび実行計画を保持するための構造体です。

シグネチャまたは形状

typedef struct pltcl_query_desc
{
    char        qname[20];
    SPIPlanPtr  plan;
    int         nargs;
    Oid        *argtypes;
    FmgrInfo   *arginfuncs;
    Oid        *argtypioparams;
} pltcl_query_desc;

入力と出力

  • **入力**: 構造体内の各フィールドは、クエリの識別名、SPI 実行計画、および引数の型情報や変換関数情報を保持します。
  • **出力**: なし（本構造体はデータ保持用です）。

前提条件と不変条件

  • qname は最大 20 バイトの固定長文字列です。
  • plan は SPI_prepare 等で生成された有効な SPIPlanPtr である必要があります。
  • argtypes、arginfuncs、argtypioparams は nargs の数に応じた配列として割り当てられる必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、Tcl インタプリタ内で SQL クエリを再利用可能にするためのキャッシュ（query_hash）の要素として使用されます。
  • クエリ実行時に Tcl_GetHashValue を通じて取得され、SPI 実行計画の参照や引数の型変換に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/tcl/pltcl.c 内のクエリ準備および実行ロジック（palloc0 によるメモリ確保および Tcl_GetHashValue による取得）。
  • **呼び出し先**: SPI 関連関数（SPIPlanPtr を介した実行）および PostgreSQL の型変換関数（FmgrInfo を介した呼び出し）。

関連データ構造

  • SPIPlanPtr: SPI 実行計画へのポインタ。
  • Tcl_HashTable: 本構造体を格納するハッシュテーブル（query_hash）。
  • FmgrInfo: PostgreSQL の関数マネージャ情報。

失敗モードと境界条件

  • qname の長さを超える識別名は切り詰められる可能性があります。
  • メモリ不足時には palloc0 がエラーを発生させます。
  • キャッシュ内の plan が無効化された場合、後続の SPI 実行でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は query_hash ハッシュテーブルのバリューとして管理されています。
  • 引数情報の配列（argtypes 等）は、構造体のライフサイクルに合わせて適切にメモリ管理（palloc）を行う必要があります。
