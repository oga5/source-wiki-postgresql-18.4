statext_is_kind_built

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/extended_stats.c:385

目的

指定された拡張統計種別が、対象の統計データタプル（pg_statistic_ext_data）において構築済みであるか（値が存在するか）を判定します。

シグネチャまたは形状

bool statext_is_kind_built(HeapTuple htup, char type)

入力と出力

  • **入力**:
  • htup: pg_statistic_ext_data カタログの HeapTuple。
  • type: 判定対象の統計種別（STATS_EXT_NDISTINCT, STATS_EXT_DEPENDENCIES, STATS_EXT_MCV, STATS_EXT_EXPRESSIONS のいずれか）。
  • **出力**:
  • 指定された種別に対応する属性が NULL でなければ true、そうでなければ false を返します。

前提条件と不変条件

  • type 引数は、定義された STATS_EXT_* 定数のいずれかである必要があります。それ以外の場合はエラーが発生します。
  • htup は pg_statistic_ext_data カタログの有効なタプルであると想定されます。

アルゴリズムまたは振る舞い

1. type に基づき、pg_statistic_ext_data カタログ内の対応する属性番号（attnum）を決定します。 2. heap_attisnull を使用して、指定された attnum が htup 内で NULL かどうかを確認します。 3. NULL でない場合（値が格納されている場合）に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/plancat.c: クエリプランナが統計情報の利用可否を判断する際に使用。
  • **呼び出し先**:
  • heap_attisnull（PostgreSQL内部関数）
  • elog（エラー発生時）

関連データ構造

  • HeapTuple: 統計データが格納されているカタログタプル。
  • pg_statistic_ext_data: 拡張統計の実データが格納されるシステムカタログ。

失敗モードと境界条件

  • 未定義の type が渡された場合、elog(ERROR, ...) により処理が中断されます。

保守担当者向けメモ

  • 新しい統計種別を拡張統計に追加する場合は、本関数の switch 文に新しい case を追加し、対応する pg_statistic_ext_data の属性番号を指定する必要があります。
