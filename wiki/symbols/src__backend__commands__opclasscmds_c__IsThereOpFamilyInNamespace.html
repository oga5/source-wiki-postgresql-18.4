IsThereOpFamilyInNamespace

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:1852

目的

指定された名前空間において、特定のアクセスメソッドに対する演算子ファミリー（Operator Family）が既に存在するかを確認し、存在する場合はエラーを発生させることで重複定義を防止します。

シグネチャまたは形状

void IsThereOpFamilyInNamespace(const char *opfname, Oid opfmethod, Oid opfnamespace)

入力と出力

  • **入力**:
  • opfname: 確認対象の演算子ファミリー名。
  • opfmethod: 対象となるアクセスメソッドのOID。
  • opfnamespace: 検索対象の名前空間（スキーマ）のOID。
  • **出力**: なし。
  • **副作用**: 重複が見つかった場合、ereport(ERROR, ...) によりトランザクションを中断します。

前提条件と不変条件

  • opfmethod および opfnamespace は有効なOIDである必要があります。
  • 本関数は、新しい演算子ファミリーを作成する直前の検証ステップとして呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. SearchSysCacheExists3 を使用し、システムカタログ pg_opfamily を OPFAMILYAMNAMENSP キャッシュ経由で検索します。 2. 検索キーとして opfmethod（アクセスメソッド）、opfname（ファミリー名）、opfnamespace（名前空間）の組み合わせを使用します。 3. 該当するレコードが存在する場合、ERRCODE_DUPLICATE_OBJECT エラーコードとともに、対象の演算子ファミリーが既に存在することを示すエラーメッセージを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c (2箇所)
  • **呼び出し先**:
  • SearchSysCacheExists3 (システムキャッシュ検索)
  • get_am_name (エラーメッセージ生成用)
  • get_namespace_name (エラーメッセージ生成用)
  • ereport (エラー通知)

関連データ構造

  • pg_opfamily: 演算子ファミリー情報を保持するシステムカタログ。
  • OPFAMILYAMNAMENSP: pg_opfamily に対するシステムキャッシュのルックアップキー。

失敗モードと境界条件

  • 指定された名前空間に同名の演算子ファミリーが存在する場合、常にエラーとなります。
  • システムキャッシュが利用できない、あるいはカタログが破損している場合、検索結果が正しく得られない可能性があります。

保守担当者向けメモ

  • 本関数は、CREATE OPERATOR FAMILY や ALTER OPERATOR FAMILY コマンドの実行時に、名前空間の整合性を保つためのガードレールとして機能します。
  • エラーメッセージにはアクセスメソッド名とスキーマ名が含まれるため、デバッグ時にどのAMに対する重複かを特定しやすくなっています。
