BlockRefTableReader

定義

  • 種別: struct
  • 定義箇所: src/common/blkreftable.c:200

目的

ディスク上のバイナリ形式で保存されたブロック参照テーブルを読み込み、増分処理を行うための状態保持構造体です。

シグネチャまたは形状

struct BlockRefTableReader
{
    BlockRefTableBuffer buffer;
    char       *error_filename;
    report_error_fn error_callback;
    void       *error_callback_arg;
    uint32      total_chunks;
    uint32      consumed_chunks;
    uint16     *chunk_size;
    uint16      chunk_data[MAX_ENTRIES_PER_CHUNK];
    uint32      chunk_position;
};

入力と出力

  • **入力**: BlockRefTableBuffer を介した外部ストレージからのバイナリデータ。
  • **出力**: BlockRefTableReaderNextRelation や BlockRefTableReaderGetBlocks を通じた、リレーション単位の変更ブロック情報の復元。

前提条件と不変条件

  • 読み込み対象のデータは、tablespace -> database -> relfilenumber -> forknumber の順でソートされている必要があります。
  • chunk_data は MAX_ENTRIES_PER_CHUNK の固定長配列として確保されています。

アルゴリズムまたは振る舞い

1. **初期化**: CreateBlockRefTableReader により生成され、I/O コールバックを保持します。 2. **読み込み**: BlockRefTableRead を内部的に呼び出し、CRC チェックを行いながらデータを順次消費します。 3. **状態管理**: total_chunks と consumed_chunks により読み込み進捗を管理し、chunk_position で現在のチャンク内のオフセットを追跡します。 4. **エラーハンドリング**: 読み込み失敗時には error_callback を通じて通知を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c
  • src/backend/backup/walsummaryfuncs.c
  • src/bin/pg_walsummary/pg_walsummary.c
  • **呼び出し先**:
  • BlockRefTableRead (内部関数)
  • BlockRefTableReaderNextRelation
  • BlockRefTableReaderGetBlocks

関連データ構造

  • BlockRefTableBuffer: I/O バッファリング用構造体。
  • BlockRefTableEntry: 復元された変更ブロック情報を格納する構造体。

失敗モードと境界条件

  • **CRC エラー**: 読み込みデータが破損している場合、整合性チェックにより失敗します。
  • **メモリ不足**: palloc0 による構造体確保時に失敗する可能性があります。
  • **チャンク境界**: MAX_ENTRIES_PER_CHUNK を超えるデータアクセスは、構造体の定義上境界条件となります。

保守担当者向けメモ

  • 本構造体は src/include/common/blkreftable.h にも定義が存在します。
  • chunk_data のサイズは MAX_ENTRIES_PER_CHUNK に依存するため、この定数を変更する場合は注意が必要です。
  • I/O はコールバック経由で行われるため、ファイル以外のストリームへの適用が可能です。
