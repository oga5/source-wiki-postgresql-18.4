sepgsql_avc_exit

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:476

目的

PostgreSQL プロセス終了時に、SELinux のステータス監視用リソースを解放し、クリーンアップを行う。

シグネチャまたは形状

static void sepgsql_avc_exit(int code, Datum arg)

入力と出力

  • **入力**:
  • code: プロセスの終了コード（on_proc_exit の仕様に基づく）。
  • arg: 終了処理に渡される補助データ（本関数では使用されない）。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数は on_proc_exit を介して登録されるクリーンアップハンドラとして機能する。
  • selinux_status_close() を呼び出すことで、SELinux の状態監視に関連するファイル記述子やメモリリソースを解放する。

アルゴリズムまたは振る舞い

1. プロセス終了イベントが発生した際、on_proc_exit により呼び出される。 2. selinux_status_close() を実行し、SELinux のステータス通知メカニズムを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/uavc.c 内の on_proc_exit(sepgsql_avc_exit, 0); により登録される。
  • **呼び出し先**: selinux_status_close()（SELinux ライブラリ関数）。

関連データ構造

  • なし。

失敗モードと境界条件

  • selinux_status_close() の内部挙動に依存する。本関数自体はエラーコードを返さない。

保守担当者向けメモ

  • 本関数はプロセス終了時のクリーンアップ専用であるため、通常の権限チェックフローからは呼び出されない。
  • storage/ipc.h で定義されるプロセス終了処理の一部として管理されている。
