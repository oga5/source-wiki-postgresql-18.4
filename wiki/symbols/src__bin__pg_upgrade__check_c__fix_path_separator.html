fix_path_separator

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/check.c:549

目的

ファイルパス内のディレクトリ区切り文字を、実行環境のOSに合わせて正規化します。

シグネチャまたは形状

char *fix_path_separator(char *path)

入力と出力

  • **入力**: path (正規化対象のパス文字列)
  • **出力**:
  • Windows環境の場合: 入力パスの / を \ に置換した新しい文字列（pg_strdup によりメモリ確保）。
  • その他環境の場合: 入力された path ポインタをそのまま返却。

前提条件と不変条件

  • Windows環境では、戻り値は呼び出し元で解放する必要があります（pg_strdup を使用しているため）。
  • 非Windows環境では、入力ポインタがそのまま返されるため、解放は不要です。

アルゴリズムまたは振る舞い

1. WIN32 マクロが定義されている場合:

  • 入力文字列の複製を作成します。
  • 文字列を走査し、/ が出現するたびに \ に置換します。
  • 変換後の文字列を返します。

    • 2. WIN32 マクロが定義されていない場合:

  • 入力された path を変更せずそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c:1048
  • src/bin/pg_upgrade/check.c:1052
  • src/bin/pg_upgrade/check.c:1055
  • **呼び出し先**:
  • pg_strdup (Windows環境時のみ)

関連データ構造

特になし。

失敗モードと境界条件

  • pg_strdup がメモリ不足により失敗した場合、内部で pg_fatal 等が呼び出される可能性があります（pg_strdup の実装依存）。
  • 空文字列やNULLポインタが渡された場合の挙動は、標準的な文字列操作関数に準じます。

保守担当者向けメモ

  • Windows環境でのみメモリ割り当てが発生する非対称な設計となっているため、呼び出し元でのメモリ管理（free の要否）には注意が必要です。
