transformContainerType

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_node.c:188

目的

添字操作（subscripting）を行う対象の型を、実際に操作可能な基底型へと正規化します。

シグネチャまたは形状

void transformContainerType(Oid *containerType, int32 *containerTypmod)

入力と出力

  • **入力**:
  • containerType: 処理対象の型のOIDへのポインタ。
  • containerTypmod: 処理対象の型のtypmodへのポインタ。
  • **出力**:
  • 引数として渡された containerType および containerTypmod が、正規化された値で上書きされます。

前提条件と不変条件

  • 入力された型がドメイン型である場合、その基底型へ解決されます。
  • int2vector および oidvector 型は、それぞれ int2[] および oid[] として扱われます。

アルゴリズムまたは振る舞い

1. getBaseTypeAndTypmod を呼び出し、ドメイン型をその基底型へ変換し、対応する typmod を抽出します。 2. 型が INT2VECTOROID の場合は INT2ARRAYOID に変換します。 3. 型が OIDVECTOROID の場合は OIDARRAYOID に変換します。

  • これは、スライス操作の結果が元のベクトル型の制約を満たさない可能性があるため、より汎用的な配列型として扱う必要があるためです。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_node.c (内部)
  • src/backend/parser/parse_target.c
  • **呼び出し先**:
  • getBaseTypeAndTypmod (utils/lsyscache.h)

関連データ構造

  • Oid: PostgreSQLの型識別子。
  • INT2VECTOROID, OIDVECTOROID, INT2ARRAYOID, OIDARRAYOID: catalog/pg_type.h で定義される型OID。

失敗モードと境界条件

  • 特記事項なし。

保守担当者向けメモ

  • ドメイン型に対する添字操作は、常にその基底コンテナ型に対して行われます。ドメイン作成者が添字操作を隠蔽する手段は提供されていません。
