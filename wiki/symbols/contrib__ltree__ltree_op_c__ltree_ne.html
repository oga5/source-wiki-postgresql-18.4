ltree_ne

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:129

目的

2つの ltree 型の値が等しくないかどうかを判定する SQL 比較演算子（!=）の実装。

シグネチャまたは形状

Datum ltree_ne(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS を介して2つの ltree ポインタを受け取る。
  • 出力: bool 型（Datum 形式）。両者が等しくない場合に true、等しい場合に false を返す。

前提条件と不変条件

  • RUNCMP マクロが定義されている環境で実行されること。
  • RUNCMP は内部的に ltree_compare を呼び出し、比較結果を res 変数に格納する。

アルゴリズムまたは振る舞い

1. RUNCMP マクロを実行し、引数として渡された2つの ltree を比較する。 2. 比較結果 res が 0 でない（等しくない）場合、true を返す。 3. 比較結果 res が 0 である（等しい）場合、false を返す。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の SQL エンジン（演算子 != の呼び出し時）。
  • 呼び出し先: RUNCMP マクロ（内部で ltree_compare を呼び出す）。

関連データ構造

  • ltree: 比較対象となる階層ラベルパス構造体。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約に基づく引数コンテナ。

失敗モードと境界条件

  • 入力が不正な ltree 構造である場合、ltree_compare の挙動に依存する。
  • メモリ不足等のシステムエラー時は PostgreSQL の標準的なエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数は ltree_compare の結果を反転させて判定を行う単純なラッパーである。
  • 比較ロジックの変更が必要な場合は ltree_compare を修正すること。
  • PG_FUNCTION_INFO_V1(ltree_ne) により、PostgreSQL の関数マネージャに登録されている。
