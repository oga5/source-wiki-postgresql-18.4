PageSetHasFreeLinePointers

定義

  • 種別: function
  • 定義箇所: src/include/storage/bufpage.h:402

目的

ページヘッダーのフラグ pd_flags に PD_HAS_FREE_LINES ビットをセットし、ページ内に再利用可能なラインポインタが存在することを示す。

シグネチャまたは形状

static inline void PageSetHasFreeLinePointers(Page page)

入力と出力

  • **入力**: Page page （操作対象のページへのポインタ）
  • **出力**: なし（副作用としてページヘッダーを直接更新）

前提条件と不変条件

  • 引数 page は有効な PageHeaderData 構造体としてキャスト可能であること。
  • PD_HAS_FREE_LINES はWALログに記録されない「ヒントビット」として扱われる。

アルゴリズムまたは振る舞い

1. 引数として渡された page を PageHeader 型にキャストする。 2. キャストしたヘッダーの pd_flags メンバに対して、ビット論理和（|=）を用いて PD_HAS_FREE_LINES フラグをセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/page/bufpage.c 内の複数箇所（行809, 892）から呼び出されていることが確認されている。
  • **呼び出し先**: なし（インライン関数であり、直接的な関数呼び出しは行わない）。

関連データ構造

  • Page: char * として定義されるページへのポインタ型。
  • PageHeaderData: ページ先頭のメタデータ構造体。pd_flags を保持する。

失敗モードと境界条件

  • page が NULL または不正なメモリ領域を指している場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数はパフォーマンス最適化のためのヒントビットを操作する。このフラグは常に正確であるとは限らないため、整合性チェックの厳密な根拠として使用すべきではない。
