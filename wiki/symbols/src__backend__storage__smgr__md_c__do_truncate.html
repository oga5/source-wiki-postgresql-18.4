do_truncate

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/md.c:342

目的

指定されたパスのファイルをサイズ 0 に切り詰める（truncate）ための内部ヘルパー関数です。エラー発生時には警告ログを出力し、呼び出し側での重複したエラーハンドリングを回避します。

シグネチャまたは形状

static int do_truncate(const char *path)

入力と出力

  • **入力**: path (切り詰め対象ファイルのパス)
  • **出力**: pg_truncate の戻り値（成功時は 0、失敗時は -1）

前提条件と不変条件

  • pg_truncate が利用可能であること。
  • 呼び出し側は、本関数が失敗時に errno を保持しつつ警告を発行することに依存します。

アルゴリズムまたは振る舞い

1. pg_truncate(path, 0) を呼び出し、対象ファイルをサイズ 0 に切り詰めます。 2. 戻り値が負であり、かつエラー原因が ENOENT（ファイルが存在しない）でない場合、ereport を使用して WARNING レベルのログを出力します。 3. ログ出力後に元の errno を復元し、pg_truncate の戻り値を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_truncate
  • **呼び出し元**: src/backend/storage/smgr/md.c 内の複数箇所（リレーションの切り詰め処理に関連する箇所）。
  • **注意**: src/bin/scripts/vacuumdb.c に存在する do_truncate という名称の識別子は、本関数とは無関係な変数であり、呼び出し関係はありません。

関連データ構造

  • 特になし（標準的なファイルパス文字列と整数型を使用）。

失敗モードと境界条件

  • **ファイルが存在しない場合**: pg_truncate が失敗し errno が ENOENT となる場合、本関数は警告を出力せずにそのまま戻り値を返します。
  • **権限不足やその他の I/O エラー**: pg_truncate が失敗し ENOENT 以外の場合、WARNING ログが発行されます。

保守担当者向けメモ

  • 本関数は md.c 内のローカルなヘルパーです。ファイル削除ではなく「切り詰め」を行うため、他のバックエンドがファイル記述子を保持している可能性がある状況でも安全に使用されることを意図しています。
