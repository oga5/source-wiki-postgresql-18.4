ossl_cipher_lookup

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/openssl.c:751

目的

OpenSSL の暗号アルゴリズム名と、それに対応する ossl_cipher 構造体（実装定義）を紐付けるためのマッピング用構造体です。

シグネチャまたは形状

struct ossl_cipher_lookup
{
	const char *name;
	const struct ossl_cipher *ciph;
};

入力と出力

  • name: 暗号アルゴリズムの名称（文字列）。
  • ciph: 対応する ossl_cipher 構造体へのポインタ。

前提条件と不変条件

  • name は NULL でない文字列である必要があります。
  • ciph は有効な ossl_cipher 構造体を指している必要があります。

アルゴリズムまたは振る舞い

この構造体は、暗号アルゴリズムを検索するための静的な配列（ossl_cipher_types）を構成するために使用されます。特定のアルゴリズム名で検索を行う際、この構造体の配列を走査し、一致する name を持つエントリから対応する ciph を取得します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/openssl.c 内の検索ロジックにおいて、ossl_cipher_types 配列の要素として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • ossl_cipher: 暗号化処理の具体的な実装を保持する構造体。
  • ossl_cipher_types: ossl_cipher_lookup 型の静的配列。

失敗モードと境界条件

  • 検索時に一致する name が存在しない場合、検索処理側で適切なエラーハンドリングが必要です。

保守担当者向けメモ

  • 新しい暗号アルゴリズムを pgcrypto に追加する際は、この構造体を用いて ossl_cipher_types 配列にエントリを追加してください。
