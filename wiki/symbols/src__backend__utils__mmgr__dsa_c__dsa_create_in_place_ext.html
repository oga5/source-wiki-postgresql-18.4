dsa_create_in_place_ext

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:470

目的

既存のメモリ領域（place）内に動的共有メモリ領域（DSA）を初期化し、管理構造体を作成する。

シグネチャまたは形状

dsa_area *dsa_create_in_place_ext(void *place, size_t size, int tranche_id, dsm_segment *segment, size_t init_segment_size, size_t max_segment_size)

入力と出力

  • **入力**:
  • place: DSA 領域を構築するメモリの開始アドレス。
  • size: place が指すメモリのサイズ。
  • tranche_id: ロック管理に使用するトランシェ ID。
  • segment: 関連付けられた DSM セグメント（任意）。
  • init_segment_size: 初期セグメントサイズ。
  • max_segment_size: 最大セグメントサイズ。
  • **出力**:
  • 初期化された dsa_area 構造体へのポインタ。

前提条件と不変条件

  • place は、指定された size 分のメモリ領域として有効であること。
  • segment が提供される場合、そのセグメントがデタッチされた際に dsa_on_dsm_detach_release_in_place が呼び出されるよう登録される。

アルゴリズムまたは振る舞い

1. create_internal を呼び出し、指定された place 上に DSA 制御構造体を構築する。 2. segment 引数が NULL でない場合、on_dsm_detach を使用して、当該セグメントがデタッチされる際に dsa_on_dsm_detach_release_in_place が実行されるようコールバックを登録する。 3. 初期化された dsa_area 構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/utils/dsa.h で宣言されており、外部から呼び出し可能。具体的な呼び出し元コードの詳細は提供されていない。
  • **呼び出し先**: create_internal（内部関数）、on_dsm_detach。

関連データ構造

  • dsa_area: DSA 領域を管理する状態構造体。
  • dsm_segment: 基盤となる動的共有メモリセグメント。

失敗モードと境界条件

  • place が不十分なサイズである場合や、内部的なメモリ割り当てに失敗した場合の挙動は create_internal の実装に依存する。

保守担当者向けメモ

  • 本関数は、既存のメモリ領域を再利用して DSA を初期化する際に使用される。
  • segment が提供された場合のクリーンアップ処理（on_dsm_detach）は、共有メモリのライフサイクル管理において重要である。
