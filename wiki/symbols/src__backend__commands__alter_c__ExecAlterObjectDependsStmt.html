ExecAlterObjectDependsStmt

定義

  • 種別: function
  • 定義箇所: src/backend/commands/alter.c:470

目的

データベースオブジェクトと拡張（Extension）間の依存関係（DEPENDENCY_AUTO_EXTENSION）を動的に追加または削除する。

シグネチャまたは形状

ObjectAddress ExecAlterObjectDependsStmt(AlterObjectDependsStmt *stmt, ObjectAddress *refAddress)

入力と出力

  • **入力**:
  • stmt: 実行対象の AlterObjectDependsStmt 構造体（対象オブジェクト情報、拡張名、操作種別を含む）。
  • refAddress: 拡張のオブジェクトアドレスを格納するためのポインタ（NULL可）。
  • **出力**:
  • 処理対象となったオブジェクトの ObjectAddress を返す。

前提条件と不変条件

  • 対象オブジェクトに対する AccessExclusiveLock を取得する。
  • 呼び出し元ユーザーが対象オブジェクトの所有者である必要がある（check_object_ownership による検証）。
  • 拡張（Extension）の所有権チェックは行わない。

アルゴリズムまたは振る舞い

1. get_object_address_rv を呼び出し、対象オブジェクトの OID とロックを取得する。 2. check_object_ownership で権限を検証し、リレーションがロックされている場合は table_close で解放する。 3. get_object_address で対象拡張の OID を特定し、refAddress が指定されていれば値を格納する。 4. stmt->remove が真の場合:

  • deleteDependencyRecordsForSpecific を呼び出し、該当する依存関係レコードを削除する。

    • 5. stmt->remove が偽の場合:

  • getAutoExtensionsOfObject で既存の依存関係を確認し、重複がないことを確認した上で recordDependencyOn を呼び出し、依存関係を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c（パーサーからのディスパッチ経由）
  • **呼び出し先**:
  • get_object_address_rv, get_object_address
  • check_object_ownership
  • table_close
  • deleteDependencyRecordsForSpecific
  • getAutoExtensionsOfObject
  • recordDependencyOn

関連データ構造

  • AlterObjectDependsStmt: 構文解析結果を保持するノード。
  • ObjectAddress: オブジェクトのカタログIDとOIDを保持する構造体。

失敗モードと境界条件

  • 対象オブジェクトが存在しない場合、get_object_address_rv がエラーを発生させる。
  • 呼び出しユーザーが所有者でない場合、check_object_ownership がエラーを発生させる。
  • 拡張名が解決できない場合、get_object_address がエラーを発生させる。

保守担当者向けメモ

  • 本関数は ALTER ... DEPENDS ON EXTENSION ... 構文を処理する。
  • 依存関係の重複を防ぐため、追加処理前に getAutoExtensionsOfObject を介したチェックを行っている。
