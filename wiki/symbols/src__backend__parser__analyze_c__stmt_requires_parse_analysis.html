stmt_requires_parse_analysis

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:446

目的

与えられた RawStmt（生の解析ツリー）が、セマンティック解析（意味解析）を必要とする文であるか、あるいは CMD_UTILITY としてそのままラップすべき文であるかを判定する。

シグネチャまたは形状

bool stmt_requires_parse_analysis(RawStmt *parseTree)

入力と出力

  • **入力**: RawStmt *parseTree - 構文解析器から出力された生の解析ツリー。
  • **出力**: bool - 解析が必要な場合は true、それ以外（ユーティリティコマンド等）は false。

前提条件と不変条件

  • parseTree は有効な RawStmt 構造体へのポインタであること。
  • 判定ロジックは、PostgreSQL のクエリ変換プロセスにおける「最適化可能な文」と「ユーティリティ文」の分類に基づいている。

アルゴリズムまたは振る舞い

1. parseTree->stmt の nodeTag を評価する。 2. 以下のいずれかのタグであれば true を返す：

  • DML系: T_InsertStmt, T_DeleteStmt, T_UpdateStmt, T_MergeStmt, T_SelectStmt, T_ReturnStmt, T_PLAssignStmt
  • 特殊ケース: T_DeclareCursorStmt, T_ExplainStmt, T_CreateTableAsStmt, T_CallStmt

    • 3. 上記以外（default）の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/plancache.c: プランキャッシュの生成プロセスで使用。
  • src/backend/parser/analyze.c: 内部的な解析処理の分岐判断に使用。
  • src/backend/optimizer/plan/setrefs.c: 参照解決時の判定に使用。
  • **呼び出し先**:
  • nodeTag() マクロ（ノード種別の取得）。

関連データ構造

  • RawStmt: 構文解析直後の SQL 文を保持する構造体。
  • Query: 解析完了後に生成されるクエリツリー（本関数は、この変換が必要かを判定する）。

失敗モードと境界条件

  • parseTree が NULL の場合、nodeTag へのアクセスによりクラッシュする可能性がある（ソースコード上でのガードは存在しない）。
  • 新しい SQL 文法を追加した際、本関数にケースを追加し忘れると、その文が誤って CMD_UTILITY として扱われる可能性がある。

保守担当者向けメモ

  • 本関数で定義されているリストは、src/backend/parser/analyze.c 内の他の判定ロジックと同期させる必要がある。
  • 修正を行う際は、src/include/parser/analyze.h の宣言と整合性を保つこと。
