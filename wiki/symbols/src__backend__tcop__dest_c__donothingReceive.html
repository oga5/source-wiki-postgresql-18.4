donothingReceive

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/dest.c:49

目的

クエリ実行結果として渡されたタプルを、何も処理せずに破棄（無視）するためのレシーバー関数です。

シグネチャまたは形状

static bool donothingReceive(TupleTableSlot *slot, DestReceiver *self);

入力と出力

  • **入力**:
  • slot: 処理対象のタプルを含む TupleTableSlot。
  • self: 呼び出し元の DestReceiver 構造体へのポインタ。
  • **出力**:
  • 常に true を返します。

前提条件と不変条件

  • 本関数は DestReceiver インターフェースの一部として呼び出されます。
  • 状態を持たないため、副作用やメモリの解放は行いません。

アルゴリズムまたは振る舞い

1. 受け取った slot および self に対して何のアクションも行いません。 2. 処理成功を示す true を即座に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/dest.c 内の静的レシーバー定義（donothingDR）において、receiveSlot メソッドとして参照されています。
  • **呼び出し先**: なし。

関連データ構造

  • DestReceiver: 出力先を抽象化する構造体。本関数はこの構造体のメソッドとして登録されます。
  • TupleTableSlot: 実行エンジンがタプルを保持するために使用する構造体。

失敗モードと境界条件

  • 失敗モードはありません。常に成功を返します。

保守担当者向けメモ

  • 本関数は静的なレシーバーの一部です。rDestroy メソッド等で解放してはならないオブジェクトで使用されることを前提としています。
  • 新しい出力先を追加する際、タプルを単に破棄したい場合は本関数を再利用してください。
