try_table_open

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/table.c:59

目的

指定された OID に対応するリレーションをオープンし、それがテーブルであることを検証した上で Relation 構造体を返します。リレーションが存在しない場合はエラーを発生させずに NULL を返します。

シグネチャまたは形状

Relation try_table_open(Oid relationId, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relationId (Oid): オープン対象のリレーション識別子。
  • lockmode (LOCKMODE): リレーション取得時に適用するロックモード。
  • **出力**:
  • Relation: オープンに成功し、かつテーブルであることが確認されたリレーションのポインタ。存在しない場合は NULL。

前提条件と不変条件

  • try_relation_open が正常に動作する環境であること。
  • 返却される Relation は、validate_relation_kind による検証を通過しており、インデックスや複合型ではないことが保証されます。

アルゴリズムまたは振る舞い

1. try_relation_open(relationId, lockmode) を呼び出し、リレーションの取得を試みます。 2. リレーションが存在しない場合（NULL が返された場合）、即座に NULL を返して終了します。 3. リレーションが存在する場合、validate_relation_kind(r) を呼び出し、対象がテーブルであることを検証します。 4. 検証を通過した Relation ポインタを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • try_relation_open
  • validate_relation_kind
  • **呼び出し元**:
  • src/backend/catalog/index.c
  • src/backend/commands/indexcmds.c
  • src/backend/optimizer/util/inherit.c
  • src/backend/replication/logical/relation.c

関連データ構造

  • Relation: オープンされたリレーション情報を保持する構造体。

失敗モードと境界条件

  • 指定された relationId が存在しない場合、NULL を返します。
  • validate_relation_kind の検証に失敗した場合（対象がテーブルではない場合）、内部で ereport(ERROR) が発行され、処理が中断されます。

保守担当者向けメモ

  • 本関数は try_relation_open のラッパーであり、テーブルアクセスのためのガードレールとして機能します。
  • 実際のデータ操作ロジックは含まず、リレーションの種別チェックに特化しています。
