WaitEventInjectionPointNew

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/wait_event.c:168

目的

インジェクションポイント専用の新しい待機イベントを動的に登録し、その一意な識別子（ID）を取得します。

シグネチャまたは形状

uint32 WaitEventInjectionPointNew(const char *wait_event_name)

入力と出力

  • **入力**: wait_event_name (待機イベント名を表す文字列)
  • **出力**: 登録された待機イベントを一意に識別する uint32 型のID

前提条件と不変条件

  • 共有メモリ上にカスタム待機イベント用のハッシュテーブルが初期化されている必要があります。
  • 登録可能なカスタム待機イベント数には上限（WAIT_EVENT_CUSTOM_HASH_MAX_SIZE = 128）があります。

アルゴリズムまたは振る舞い

1. PG_WAIT_INJECTIONPOINT（待機クラス）と引数で受け取った wait_event_name を指定して WaitEventCustomNew を呼び出します。 2. WaitEventCustomNew 内部で共有メモリ上の排他制御を行い、一意なIDの割り当てとハッシュテーブルへの登録を実行します。 3. 割り当てられたIDを呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/injection_points/injection_points.c:302
  • **呼び出し先**: WaitEventCustomNew

関連データ構造

  • PG_WAIT_INJECTIONPOINT: 待機イベントのクラス定数。
  • WaitEventCustomHashByInfo / WaitEventCustomHashByName: 共有メモリ上のハッシュテーブル（WaitEventCustomNew 経由で使用）。

失敗モードと境界条件

  • 登録上限（128個）を超えた場合、WaitEventCustomNew の内部実装に従いエラーが発生する可能性があります。
  • 共有メモリが初期化されていない状態で呼び出された場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は WaitEventCustomNew のラッパーであり、インジェクションポイントという特定の用途に限定したインターフェースを提供します。
  • 待機イベント名の文字列は共有メモリ上のハッシュテーブルに格納されるため、メモリ消費量と登録数に注意してください。
