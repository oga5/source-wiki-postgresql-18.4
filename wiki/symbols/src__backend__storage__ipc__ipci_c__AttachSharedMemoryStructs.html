AttachSharedMemoryStructs

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipci.c:172

目的

PostgreSQL のバックエンドプロセスにおいて、共有メモリ内のデータ構造への接続および再初期化を行う。主に EXEC_BACKEND 環境（Windows 等）で、子プロセスが既存の共有メモリ領域を正しく参照できるようにするために使用される。

シグネチャまたは形状

void AttachSharedMemoryStructs(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上のポインタ設定およびグローバル変数の初期化を行う）

前提条件と不変条件

  • MyProc が初期化済みであること（Assert(MyProc != NULL)）。
  • プロセスが Postmaster の管理下にあること（Assert(IsUnderPostmaster)）。

アルゴリズムまたは振る舞い

1. InitializeFastPathLocks() を呼び出し、高速パスロックのグループ数を再計算する（EXEC_BACKEND モードでの継承不備を補完）。 2. CreateOrAttachShmemStructs() を呼び出し、各サブシステムの共有メモリデータ構造を初期化またはアタッチする。 3. shmem_startup_hook が設定されている場合、これを実行し、ロード済みモジュールによる共有メモリ割り当てのセットアップを許可する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/lmgr/proc.c
  • 呼び出し先:
  • InitializeFastPathLocks
  • CreateOrAttachShmemStructs
  • shmem_startup_hook (関数ポインタ)

関連データ構造

  • MyProc: 現在のプロセスのプロセス制御ブロック。
  • shmem_startup_hook: 共有メモリ初期化完了後に実行される拡張モジュール用フック。

失敗モードと境界条件

  • MyProc が NULL の場合、アサーション違反により異常終了する。
  • IsUnderPostmaster が偽の場合、アサーション違反により異常終了する。

保守担当者向けメモ

  • 本関数は EXEC_BACKEND 環境とそれ以外で共有メモリの初期化パスを統合する役割を持つ。
  • 新しいサブシステムが共有メモリを必要とする場合、CreateOrAttachShmemStructs 経由で初期化ロジックが確実に呼び出されるようにする必要がある。
