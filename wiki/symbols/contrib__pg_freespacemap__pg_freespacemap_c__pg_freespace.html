pg_freespace

定義

  • 種別: function
  • 定義箇所: contrib/pg_freespacemap/pg_freespacemap.c:27

目的

PostgreSQLのフリースペースマップ（FSM）から、指定されたリレーションの特定のページにおける空き容量をSQLインターフェース経由で取得する。

シグネチャまたは形状

Datum pg_freespace(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象リレーションのOID。
  • PG_GETARG_INT64(1): 対象のブロック番号。
  • **出力**:
  • PG_RETURN_INT16(freespace): 指定ブロックの空き容量（バイト単位の近似値）。

前提条件と不変条件

  • 対象リレーションは物理ストレージを持つ必要がある（RELKIND_HAS_STORAGE）。
  • ブロック番号は 0 以上 MaxBlockNumber 以下である必要がある。
  • AccessShareLock を取得してリレーションをオープンするため、読み取り操作に対して安全である。

アルゴリズムまたは振る舞い

1. relation_open を呼び出し、AccessShareLock モードで対象リレーションを開く。 2. リレーションが物理ストレージを持たない場合、ERRCODE_WRONG_OBJECT_TYPE でエラーを報告する。 3. ブロック番号が範囲外の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告する。 4. GetRecordedFreeSpace を呼び出し、FSMから空き容量を取得する。 5. relation_close でロックを解放し、取得した空き容量を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • relation_open / relation_close
  • GetRecordedFreeSpace
  • ereport

関連データ構造

  • Relation (リレーション記述子)

失敗モードと境界条件

  • リレーションが物理ストレージを持たない場合（ビューや外部テーブル等）はエラーとなる。
  • ブロック番号が負の値、または MaxBlockNumber を超える場合はエラーとなる。

保守担当者向けメモ

  • 本関数は読み取り専用のインターフェースであり、FSMの更新ロジックは含まない。
  • AccessShareLock を使用しているため、テーブルの同時実行性には影響を与えない。
