PQsetdbLogin

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-connect.c:2230

目的

PostgreSQL データベースへの接続を確立し、指定された接続パラメータ（ホスト、ポート、オプション、TTY、データベース名、ユーザー名、パスワード）を適用した PGconn オブジェクトを返します。

シグネチャまたは形状

PGconn *PQsetdbLogin(const char *pghost, const char *pgport, const char *pgoptions,
                     const char *pgtty, const char *dbName, const char *login,
                     const char *pwd);

入力と出力

  • **入力**:
  • pghost, pgport, pgoptions, pgtty, dbName, login, pwd: 各接続パラメータ（文字列）。NULL または空文字列を指定可能。
  • **出力**:
  • 接続状態を保持する PGconn 構造体へのポインタ。メモリ確保に失敗した場合は NULL を返す。

前提条件と不変条件

  • pqMakeEmptyPGconn() により、conn->errorMessage は空の状態で初期化されることが期待されます。
  • 接続初期化の各ステップで発生するエラーメッセージは、errorMessage バッファに追記されます。

アルゴリズムまたは振る舞い

1. pqMakeEmptyPGconn() で PGconn 構造体を確保します。 2. dbName が接続文字列（URI 等）と見なされる場合、connectOptions1() で解析します。それ以外の場合はデフォルト値を設定後、dbName を構造体に格納します。 3. 指定された各パラメータ（pghost, pgport, pgoptions, login, pwd）が提供されている場合、既存の値を解放し、strdup() で新しい値を構造体にコピーします。 4. pqConnectOptions2() を呼び出し、派生オプションを計算します。 5. pqConnectDBStart() を呼び出し、接続プロセスを開始します。 6. 接続開始に成功した場合、pqConnectDBComplete() を呼び出して接続を完了させます。 7. 最終的に PGconn オブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: libpq-fe.h で定義されており、アプリケーションコードから直接呼び出されることが想定されています。
  • **呼び出し先**:
  • pqMakeEmptyPGconn
  • recognized_connection_string
  • connectOptions1
  • pqConnectOptions2
  • pqConnectDBStart
  • pqConnectDBComplete
  • libpq_append_conn_error

関連データ構造

  • PGconn: 接続状態を保持する主要な構造体。

失敗モードと境界条件

  • **メモリ不足**: パラメータのコピー（strdup）に失敗した場合、oom_error ラベルへジャンプし、conn->status を CONNECTION_BAD に設定した上でエラーメッセージを付与し、conn を返します。
  • **接続失敗**: connectOptions1 や pqConnectOptions2 が失敗した場合、その時点での conn を返します。

保守担当者向けメモ

  • 本関数は古い形式の接続 API です。内部的には PQconnectdb 等と同様の PGconn 構築ロジックを共有しています。
  • dbName が接続文字列として認識されるかどうかの判定には recognized_connection_string が使用されます。
  • 接続パラメータの更新時に free() と strdup() を使用しているため、メモリリークや二重解放に注意してください。
