tfunc_fold

定義

  • 種別: function
  • 定義箇所: src/common/unicode/case_test.c:323

目的

Unicode 文字列のケースフォールディング（Case Folding）テストを実行するためのラッパー関数です。PostgreSQL の Unicode 変換ロジックを検証するテストスイートの一部として機能します。

シグネチャまたは形状

static size_t tfunc_fold(char *dst, size_t dstsize, const char *src, ssize_t srclen);

入力と出力

  • **入力**:
  • dst: 変換結果を格納するバッファへのポインタ。
  • dstsize: dst バッファのサイズ。
  • src: 変換対象のソース文字列。
  • srclen: ソース文字列の長さ（バイト単位）。
  • **出力**:
  • 変換後の文字列の長さを size_t 型で返します。

前提条件と不変条件

  • 内部で unicode_strfold を呼び出しており、その第5引数に true を渡すことで、Unicode 標準に準拠したケースフォールディングを強制しています。

アルゴリズムまたは振る舞い

1. 受け取った引数をそのまま unicode_strfold 関数へ転送します。 2. unicode_strfold の実行結果（変換後のバイト長）をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode/case_test.c 内の test_convert 関数から、特定のテストケース（例: "ıiIİ" や "σςΣ ΣΣΣ"）の検証用関数ポインタとして呼び出されます。
  • **呼び出し先**:
  • unicode_strfold（PostgreSQL 内部の Unicode 変換関数）。

関連データ構造

  • 本関数は TestFunc 型のシグネチャに適合するように設計されており、test_convert 関数による網羅的なバッファ境界テストで使用されます。

失敗モードと境界条件

  • unicode_strfold の挙動に依存します。dstsize が変換後の文字列を格納するのに不十分な場合、切り詰めが発生する可能性があります。

保守担当者向けメモ

  • 本関数はテスト専用のラッパーです。ロジックの変更は unicode_strfold 側で行う必要があります。
  • テストケースを追加する際は、test_convert を介して本関数を呼び出す形式に従ってください。
