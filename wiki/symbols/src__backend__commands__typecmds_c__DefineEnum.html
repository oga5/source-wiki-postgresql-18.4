DefineEnum

定義

  • 種別: function
  • 定義箇所: src/backend/commands/typecmds.c:1180

目的

SQLの CREATE TYPE ... AS ENUM コマンドを処理し、新しい列挙型（Enum）とその対応する配列型をシステムカタログに登録する。

シグネチャまたは形状

ObjectAddress DefineEnum(CreateEnumStmt *stmt)

入力と出力

  • **入力**: CreateEnumStmt *stmt（パーサーが生成した列挙型定義の構文木）
  • **出力**: ObjectAddress（作成された列挙型のOIDとカタログIDを含む構造体）

前提条件と不変条件

  • 呼び出し元は、対象のスキーマに対する CREATE 権限を保持している必要がある。
  • 列挙型の名前が既存の型と重複する場合、それが自動生成された配列型であれば名前を変更して回避を試みる。それ以外の場合はエラーとなる。

アルゴリズムまたは振る舞い

1. **名前空間の特定**: QualifiedNameGetCreationNamespace を使用して、型を作成するスキーマを決定する。 2. **権限チェック**: object_aclcheck により、対象スキーマへの作成権限を確認する。 3. **名前衝突の解決**: GetSysCacheOid2 で既存の型を確認し、必要に応じて moveArrayTypeName で既存の配列型を退避させる。 4. **型作成**:

  • AssignTypeArrayOid で配列型のOIDを確保する。
  • TypeCreate を呼び出し、pg_type に列挙型を登録する。
  • EnumValuesCreate を呼び出し、pg_enum に列挙値のリストを登録する。
  • 再度 TypeCreate を呼び出し、対応する配列型を登録する。

    • 5. **メモリ解放**: makeArrayTypeName で生成した一時的な名前文字列を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c の ProcessUtility 経由で呼び出される。
  • **呼び出し先**: QualifiedNameGetCreationNamespace, object_aclcheck, GetSysCacheOid2, moveArrayTypeName, AssignTypeArrayOid, TypeCreate, EnumValuesCreate, makeArrayTypeName 等。

関連データ構造

  • CreateEnumStmt: 列挙型定義の構文木。
  • ObjectAddress: 作成されたオブジェクトの識別子。
  • pg_type, pg_enum: 操作対象のシステムカタログ。

失敗モードと境界条件

  • **権限不足**: スキーマへの作成権限がない場合、aclcheck_error が発生する。
  • **名前重複**: 既存の型名と衝突し、かつ退避できない場合は ERRCODE_DUPLICATE_OBJECT エラーが発生する。

保守担当者向けメモ

  • 列挙型は内部的に TYPTYPE_ENUM として扱われ、pg_enum カタログに値が格納される。
  • 列挙型を作成すると、自動的にその配列型も作成されるため、TypeCreate が2回呼び出される点に注意すること。
