LZ4Stream_write

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:572

目的

指定されたバッファの内容を LZ4 形式で圧縮し、CompressFileHandle が保持するファイルストリームへ書き込む。

シグネチャまたは形状

static void LZ4Stream_write(const void *ptr, size_t size, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • ptr: 書き込むデータへのポインタ。
  • size: 書き込むデータのバイト数。
  • CFH: 圧縮状態と出力先ファイルポインタを保持する CompressFileHandle 構造体。
  • **出力**: なし（成功時はストリームへ書き込み、失敗時は pg_fatal で終了）。

前提条件と不変条件

  • CFH->private_data は LZ4State 構造体へのポインタとしてキャスト可能であること。
  • LZ4Stream_init による遅延初期化が内部的に行われるため、呼び出し前に明示的な初期化は不要。

アルゴリズムまたは振る舞い

1. LZ4Stream_init を呼び出し、未初期化であればコンテキストを生成する。 2. 入力データを DEFAULT_IO_BUFFER_SIZE 単位のチャンクに分割して処理する。 3. LZ4F_compressUpdate を使用してチャンクを圧縮し、結果を state->buffer に格納する。 4. 圧縮されたデータを fwrite で出力先ファイルストリームへ書き出す。 5. 書き込み失敗時は pg_fatal を呼び出し、プロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_lz4.c 内で CFH->write_func に割り当てられている。直接の呼び出し元はソースコード上では明示されていない。
  • **呼び出し先**: LZ4Stream_init, LZ4F_compressUpdate, LZ4F_getErrorName, LZ4F_isError, fwrite, pg_fatal。

関連データ構造

  • CompressFileHandle: 圧縮処理の抽象化インターフェース。
  • LZ4State: LZ4 圧縮コンテキスト、バッファ、およびファイルポインタを保持する構造体。

失敗モードと境界条件

  • **初期化失敗**: LZ4Stream_init が失敗した場合、pg_fatal によりエラーメッセージを出力して終了する。
  • **圧縮エラー**: LZ4F_compressUpdate がエラーを返した場合、pg_fatal によりエラーメッセージを出力して終了する。
  • **書き込みエラー**: fwrite の戻り値が期待値と異なる場合、errno に基づくエラーメッセージを出力して終了する。

保守担当者向けメモ

  • 本関数は CompressFileHandle の関数ポインタ経由で呼び出されることを前提としている。
  • DEFAULT_IO_BUFFER_SIZE を超えるデータはループ処理されるため、メモリ消費量はバッファサイズに依存する。
