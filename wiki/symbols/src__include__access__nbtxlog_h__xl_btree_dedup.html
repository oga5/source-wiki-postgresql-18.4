xl_btree_dedup

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtxlog.h:170

目的

B-treeインデックスにおける重複キー圧縮（Deduplication）操作をWAL（Write-Ahead Log）に記録するための構造体です。

シグネチャまたは形状

typedef struct xl_btree_dedup
{
    uint16      nintervals;
    /* DEDUPLICATION INTERVALS FOLLOW */
} xl_btree_dedup;

入力と出力

  • **入力**: nintervals（重複排除された間隔の数）。構造体の直後に可変長データとして「重複排除間隔（Deduplication Intervals）」が続く形式をとります。
  • **出力**: WALレコードの一部としてディスクへ書き出されます。

前提条件と不変条件

  • nintervals は、構造体の直後に続くデータセットの個数と一致する必要があります。
  • 本構造体は、XLogRecGetData を介してWALレコードから取得されることを前提としています。

アルゴリズムまたは振る舞い

1. **記録**: B-treeの重複排除処理時に、変更内容を記述するために本構造体が初期化されます。 2. **再生**: クラッシュリカバリ時、btree_redo がWALレコードを読み込み、本構造体を使用してページ内の重複排除状態を再構築します。既存のタプルと本構造体内のメタデータを組み合わせて最終的なページ状態を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtdedup.c: 重複排除処理の実行時に本構造体を使用。
  • **呼び出し先**:
  • src/backend/access/nbtree/nbtxlog.c: btree_redo 内でWALレコードの解析に使用。
  • src/backend/access/rmgrdesc/nbtdesc.c: ログ内容のデバッグ出力（人間が読める形式への変換）に使用。

関連データ構造

  • SizeOfBtreeDedup: 本構造体のサイズを計算するためのマクロ（offsetof を使用）。
  • xl_btree_update: 重複排除に関連する他の更新操作を定義する構造体。

失敗モードと境界条件

  • WALレコードのデータ長が nintervals と整合しない場合、REDO処理時にデータ破損や不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は可変長データを含むため、メモリレイアウトを扱う際は SizeOfBtreeDedup マクロを適切に使用してください。
  • REDO処理は冪等性を維持する必要があるため、ページが既に更新済みである場合はLSNチェックを介してスキップするロジックが nbtxlog.c 側で求められます。
