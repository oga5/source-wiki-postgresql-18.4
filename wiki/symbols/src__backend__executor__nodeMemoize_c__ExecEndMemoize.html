ExecEndMemoize

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMemoize.c:1079

目的

Memoize 実行ノードの終了処理を行い、キャッシュ用メモリコンテキストの解放および統計情報の集約を行う。

シグネチャまたは形状

void ExecEndMemoize(MemoizeState *node)

入力と出力

  • **入力**: MemoizeState *node （終了対象のメモ化ノード状態構造体）
  • **出力**: なし（副作用としてリソース解放および統計情報の書き出しを行う）

前提条件と不変条件

  • node は初期化済みであること。
  • USE_ASSERT_CHECKING が有効なビルドでは、キャッシュ内のメモリ使用量（node->mem_used）と、実際にハッシュテーブルを走査して算出したメモリ使用量の整合性が検証される。

アルゴリズムまたは振る舞い

1. **メモリ整合性チェック** (USE_ASSERT_CHECKING 有効時): ハッシュテーブルを走査し、各エントリおよびタプルのメモリ消費量を再計算して node->mem_used と一致するか検証する。 2. **統計情報の共有**: 並列ワーカーとして実行されている場合、node->stats を共有メモリ上の node->shared_info にコピーし、EXPLAIN ANALYZE で参照可能にする。 3. **リソース解放**: node->tableContext を削除し、キャッシュに関連するメモリを解放する。 4. **サブプランの終了**: ExecEndNode を呼び出し、配下のサブプランを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c:718
  • **呼び出し先**:
  • ExecEndNode (サブプラン終了用)
  • MemoryContextDelete
  • memoize_start_iterate / memoize_iterate (アサート有効時のみ)

関連データ構造

  • MemoizeState: ノードの実行状態を保持する構造体。
  • MemoizeInstrumentation: 統計情報保持用構造体。

失敗モードと境界条件

  • 並列ワーカー実行時、node->shared_info が NULL の場合は統計情報のコピーをスキップする。
  • node->hashtable が NULL の場合、アサートチェックはスキップされる。

保守担当者向けメモ

  • EXPLAIN ANALYZE の統計情報は、並列ワーカー終了時にこの関数を通じて共有メモリへ書き出されるため、統計情報の項目を追加・変更する際は MemoizeInstrumentation 構造体と本関数のコピー処理を同期させる必要がある。
