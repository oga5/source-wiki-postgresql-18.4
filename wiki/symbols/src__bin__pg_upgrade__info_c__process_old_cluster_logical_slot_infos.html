process_old_cluster_logical_slot_infos

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:689

目的

旧クラスタから取得した論理レプリケーションスロットのクエリ結果（PGresult）を解析し、データベース情報構造体（DbInfo）に格納する。

シグネチャまたは形状

static void process_old_cluster_logical_slot_infos(DbInfo *dbinfo, PGresult *res, void *arg)

入力と出力

  • **入力**:
  • dbinfo: スロット情報を格納する対象の DbInfo 構造体へのポインタ。
  • res: 論理スロット情報を保持する PGresult オブジェクト。
  • arg: 使用されていない引数。
  • **出力**: なし（dbinfo の slot_arr フィールドが更新される）。

前提条件と不変条件

  • process_old_cluster_logical_slot_infos が UpgradeTaskProcessCB 型であることをアサーションで検証している。
  • res には slot_name, plugin, two_phase, failover, caught_up, invalid の各カラムが含まれている必要がある。

アルゴリズムまたは振る舞い

1. PQntuples(res) を用いてスロット数を取得する。 2. スロットが存在する場合、LogicalSlotInfo 配列をメモリ確保する。 3. 各カラムのインデックスを PQfnumber で特定する。 4. 各スロットについて以下の処理をループで実行する：

  • slotname と plugin を pg_strdup で複製。
  • two_phase, failover, caught_up, invalid の各フラグを、値が "t" であるかどうかの比較結果（真偽値）として格納。

    • 5. 最終的に dbinfo->slot_arr に確保した配列とスロット数を割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し箇所は確認されていないが、UpgradeTaskProcessCB 型として登録される設計である。
  • **呼び出し先**: PQntuples, PQfnumber, PQgetvalue, pg_malloc, pg_strdup, strcmp 等のライブラリ関数。

関連データ構造

  • DbInfo: データベース情報を保持する構造体。
  • LogicalSlotInfo: 論理レプリケーションスロットの属性（名前、プラグイン、状態等）を保持する構造体。

失敗モードと境界条件

  • num_slots が 0 の場合、slot_arr.slots は NULL に、slot_arr.nslots は 0 に設定される。
  • メモリ確保失敗時は pg_malloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は PostgreSQL 17 以降の論理スロット移行機能に関連する。
  • two_phase 等のフラグは文字列 "t" との比較に依存しているため、クエリ結果のフォーマット変更には注意が必要である。
