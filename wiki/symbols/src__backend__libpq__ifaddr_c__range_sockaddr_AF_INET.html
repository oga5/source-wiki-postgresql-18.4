range_sockaddr_AF_INET

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/ifaddr.c:65

目的

IPv4アドレスが指定されたネットワーク範囲（サブネット）に含まれるかを判定します。

シグネチャまたは形状

static int range_sockaddr_AF_INET(const struct sockaddr_in *addr,
                                  const struct sockaddr_in *netaddr,
                                  const struct sockaddr_in *netmask);

入力と出力

  • **入力**:
  • addr: 判定対象のIPv4アドレス。
  • netaddr: ネットワークアドレス。
  • netmask: ネットワークマスク。
  • **出力**:
  • 1: アドレスが範囲内にある場合。
  • 0: アドレスが範囲外にある場合。

前提条件と不変条件

  • 入力される各構造体は sockaddr_in 型であり、IPv4アドレスとして適切に初期化されている必要があります。
  • ビット演算による比較を行うため、ネットワークバイトオーダーで格納されていることが前提となります。

アルゴリズムまたは振る舞い

1. addr と netaddr の各ビットに対して排他的論理和（XOR）をとります。 2. その結果に対して netmask でビットごとの論理積（AND）をとります。 3. 結果が 0 であれば、指定されたネットワーク範囲内であると判定し 1 を返します。それ以外は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_range_sockaddr（src/backend/libpq/ifaddr.c 内）。
  • **呼び出し先**: 特になし（標準的なビット演算のみを実行）。

関連データ構造

  • struct sockaddr_in: IPv4アドレスを保持する標準的なソケットアドレス構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動は定義されていません。
  • ネットワークマスクが不正な値（例: 連続しないビットパターン）である場合、論理的なサブネット判定として意図しない結果を返す可能性があります。

保守担当者向けメモ

  • 本関数は pg_range_sockaddr からの内部的な委譲先として実装されています。
  • IPv6版の range_sockaddr_AF_INET6 と同様のロジックですが、IPv4専用の sin_addr.s_addr を直接操作するため、移植や修正の際はIPv4の構造に依存している点に注意してください。
