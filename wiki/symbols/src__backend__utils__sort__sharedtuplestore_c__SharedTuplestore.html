SharedTuplestore

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/sharedtuplestore.c:59

目的

PostgreSQL の並列クエリ実行において、複数のバックエンド間でタプルを共有・交換するための共有メモリ上の制御構造体です。

シグネチャまたは形状

struct SharedTuplestore
{
    int         nparticipants;    /* 書き込み可能な参加者数 */
    int         flags;           /* SHARED_TUPLESTORE_XXX フラグ */
    size_t      meta_data_size;  /* タプルごとのヘッダーサイズ */
    char        name[NAMEDATALEN]; /* ストアの名称 */

    /* 参加者ごとの共有状態が続く */
    SharedTuplestoreParticipant participants[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • 入力: sts_initialize を通じて共有メモリ上に初期化されます。
  • 出力: 構造体自体は共有メモリ上の状態を保持し、各バックエンドが SharedTuplestoreAccessor を介して操作します。

前提条件と不変条件

  • 共有メモリ上に配置される必要があります。
  • participants 配列は可変長であり、初期化時に指定された参加者数に応じてメモリが確保される必要があります。

アルゴリズムまたは振る舞い

  • 参加プロセスごとのロックと読み取り位置を保持し、並列スキャン時の同期制御を提供します。
  • 書き込みは各プロセスが専用の一時ファイルに対して行うため、書き込み時のロックは不要です。
  • 並列スキャン時は LWLock を使用して、読み取り位置の更新とチャンクの確保を排他的に行います。

呼び出し元と呼び出し先

  • 呼び出し元: sts_estimate（サイズ計算）、sts_initialize（初期化）、sts_attach（接続）などで参照されます。
  • 呼び出し先: 構造体内部で SharedTuplestoreParticipant を保持しています。

関連データ構造

  • SharedTuplestoreParticipant: 各参加者の状態を管理する構造体。
  • SharedTuplestoreAccessor: 各バックエンドが操作に使用するローカル構造体。
  • SharedTuplestoreChunk: データを管理するチャンク単位。

失敗モードと境界条件

  • name が NAMEDATALEN を超える場合、エラーが発生します。
  • 参加者数やメタデータサイズが不適切に設定された場合、共有メモリの確保やアクセスで不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリレイアウトの計算には offsetof を使用してください。
  • 並列スキャンは「全参加者の全タプル」を読み出しますが、特定の順序は保証されません。
