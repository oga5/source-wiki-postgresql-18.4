MakeTidOpExpr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidrangescan.c:56

目的

TID範囲スキャンにおいて、ctid を含む比較演算子（OpExpr）を解析し、スキャン範囲の境界条件（下限または上限）を決定するための内部表現 TidOpExpr を生成する。

シグネチャまたは形状

TidOpExpr *MakeTidOpExpr(OpExpr *expr, TidRangeScanState *tidstate)

入力と出力

  • **入力**:
  • expr: 解析対象の OpExpr 構造体。
  • tidstate: 現在の TidRangeScanState へのポインタ。
  • **出力**:
  • TidOpExpr: 生成された比較条件の内部表現。

前提条件と不変条件

  • expr の左右いずれかの引数が ctid 変数である必要がある。
  • expr->opno は TIDLessOperator, TIDLessEqOperator, TIDGreaterOperator, TIDGreaterEqOperator のいずれかである必要がある。

アルゴリズムまたは振る舞い

1. get_leftop および get_rightop を使用して演算子の左右の引数を取得する。 2. IsCTIDVar を用いて、どちらの引数が ctid かを判定する。ctid でない方の引数を ExecInitExpr で評価可能な状態（ExprState）に変換する。 3. invert フラグを管理し、ctid が右辺にある場合は比較方向を反転させる。 4. expr->opno に基づき、TidOpExpr の exprtype を TIDEXPR_LOWER_BOUND（下限）または TIDEXPR_UPPER_BOUND（上限）に設定する。 5. inclusive フラグを演算子に応じて設定する（<= や >= の場合は true）。 6. 構築した TidOpExpr を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeTidrangescan.c 内の TidExprListCreate から呼び出される（直接の証拠）。
  • **呼び出し先**: ExecInitExpr（式評価の初期化）、palloc（メモリ確保）、elog（エラー報告）。

関連データ構造

  • TidOpExpr: 本関数が生成する構造体。
  • OpExpr: PostgreSQL の演算子式構造体。
  • TidRangeScanState: TID範囲スキャンの実行状態を保持する構造体。

失敗モードと境界条件

  • ctid を含む引数が見つからない場合、elog(ERROR) によりエラー終了する。
  • 未定義の演算子（TID 関連以外の演算子）が渡された場合、elog(ERROR) によりエラー終了する。

保守担当者向けメモ

  • 現在 inclusive は初期化時に false で固定された後、演算子に応じて上書きされる実装となっている。
  • invert フラグの管理は、ctid の位置関係による比較方向の反転を正しく処理するために重要である。
