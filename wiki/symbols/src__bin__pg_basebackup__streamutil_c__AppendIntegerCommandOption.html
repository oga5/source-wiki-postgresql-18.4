AppendIntegerCommandOption

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/streamutil.c:789

目的

レプリケーションコマンドの文字列構築において、整数型のオプションをバッファに追加します。サーバーのバージョンに応じた構文（旧形式または新形式）を吸収する役割を持ちます。

シグネチャまたは形状

void AppendIntegerCommandOption(PQExpBuffer buf, bool use_new_option_syntax, char *option_name, int32 option_value)

入力と出力

  • buf (入力/出力): コマンド文字列を保持する PQExpBuffer。
  • use_new_option_syntax (入力): 新しい括弧付き構文を使用するかどうかを示すフラグ。
  • option_name (入力): 追加するオプションの名称。
  • option_value (入力): オプションに設定する整数値。
  • 戻り値: なし。

前提条件と不変条件

  • buf は初期化済みである必要があります。
  • option_name は有効な文字列ポインタである必要があります。

アルゴリズムまたは振る舞い

1. AppendPlainCommandOption を呼び出し、option_name を buf に追加します（構文の切り替えはこの関数内で行われます）。 2. appendPQExpBuffer を使用し、buf の末尾にスペースと option_value を整数として追記します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_basebackup/pg_basebackup.c (複数箇所で WAIT, MAX_RATE 等のオプション追加に使用)
  • 呼び出し先:
  • AppendPlainCommandOption (同ファイル内)
  • appendPQExpBuffer (libpq)

関連データ構造

  • PQExpBuffer: コマンド文字列を動的に構築するための構造体。

失敗モードと境界条件

  • buf がメモリ不足状態にある場合、appendPQExpBuffer の内部仕様に従い、以降の操作は無効化されます。

保守担当者向けメモ

  • 本関数はレプリケーションコマンドの構文差異を隠蔽するためのヘルパーです。新しいレプリケーションコマンドを追加する際は、本関数または関連する Append...CommandOption 関数群を利用して一貫性を保ってください。
