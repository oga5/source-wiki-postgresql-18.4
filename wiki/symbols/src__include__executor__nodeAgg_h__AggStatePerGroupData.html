AggStatePerGroupData

定義

  • 種別: struct
  • 定義箇所: src/include/executor/nodeAgg.h:250

目的

集約演算において、各グループ内の各集約関数が保持すべき作業用状態（中間遷移値）を管理する。

シグネチャまたは形状

typedef struct AggStatePerGroupData {
    Datum       transValue;       /* 現在の遷移値 */
    bool        transValueIsNull; /* 遷移値がNULLか否か */
    bool        noTransValue;     /* 遷移値が未設定か否か */
} AggStatePerGroupData;

入力と出力

  • 入力: 集約関数の遷移関数（transfn）による計算結果。
  • 出力: 最終関数（finalfn）へ渡される中間状態、または次の行の計算に使用される遷移値。

前提条件と不変条件

  • noTransValue は初期状態において transValueIsNull と同じ値を持つ。
  • noTransValue が真の場合、transValue は未設定とみなされる。
  • 最初の非NULL入力が処理されると、noTransValue と transValueIsNull は両方とも偽にクリアされる。
  • transfn がNULLを返した場合、そのNULL状態は保持され、以降の入力値で自動的に上書きされることはない。

アルゴリズムまたは振る舞い

集約処理の実行中、グループごとに本構造体が割り当てられる。入力行が処理されるたびに transfn が呼び出され、その結果が transValue に格納される。この際、NULL値の取り扱いと初期値の判定ロジック（noTransValue）に基づき、状態が更新される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/nodeAgg.c 内の集約初期化および実行ロジック（ExecInitAgg 等）において、メモリ確保および状態管理のために参照される。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはない。

関連データ構造

  • AggStatePerTransData: 集約の遷移関数実行に必要なメタデータを保持。
  • AggStatePerAggData: 最終関数実行に必要なメタデータを保持。
  • AggStatePerGroup: AggStatePerGroupData へのポインタ型（src/include/nodes/execnodes.h で定義）。

失敗モードと境界条件

  • メモリ不足: palloc0 等による構造体確保時に発生する可能性がある。
  • NULL値の伝播: transfn がNULLを返した場合、その状態が維持されるため、後続の入力値による上書きが発生しない点に注意が必要。

保守担当者向けメモ

  • 本構造体は AggState 内で配列として管理されることが一般的である。
  • transValue は Datum 型であるため、ポインタを保持する場合はメモリコンテキストの管理に注意すること。
  • noTransValue と transValueIsNull のフラグ管理は、集約関数の初期値処理と密接に関連しているため、変更時は nodeAgg.c のロジックと整合性を取ること。
