xl_replorigin_set

定義

  • 種別: struct
  • 定義箇所: src/include/replication/origin.h:18

目的

論理レプリケーションにおける「レプリケーション起点（Replication Origin）」の進捗状態をWAL（Write-Ahead Log）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_replorigin_set
{
	XLogRecPtr	remote_lsn;
	RepOriginId node_id;
	bool		force;
} xl_replorigin_set;

入力と出力

  • **入力**: レプリケーション起点の進捗更新時に、リモート側のLSN (remote_lsn)、対象の起点ID (node_id)、および強制フラグ (force) を保持します。
  • **出力**: WALレコードとして書き込まれ、リカバリ時に replorigin_redo を通じてレプリケーション状態の復元に使用されます。

前提条件と不変条件

  • node_id は RepOriginId 型（uint16）であり、有効なレプリケーション起点を指す必要があります。
  • remote_lsn は、リモートノードから受信した最新のLSNを保持します。

アルゴリズムまたは振る舞い

1. レプリケーション起点の進捗更新時（replorigin_session_advance 等）に、本構造体に値を設定します。 2. WALレコードとして書き込まれることで、クラッシュリカバリ時にこの構造体の内容が読み取られます。 3. リカバリ処理において、記録された node_id に対応する起点の進捗が remote_lsn に更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/origin.c: 進捗更新処理において本構造体を初期化し、WALへの書き込みを行います。
  • **呼び出し先**:
  • src/backend/access/rmgrdesc/replorigindesc.c: WALレコードのデコードおよび説明出力に使用されます。

関連データ構造

  • RepOriginId: 起点を識別する内部ID。
  • XLogRecPtr: WAL上の位置を示すポインタ型。

失敗モードと境界条件

  • force フラグが設定されている場合、通常の進捗更新ルールを上書きして強制的に状態を更新する挙動をとります。
  • 無効な node_id が指定された場合、リカバリ時や状態更新時に整合性エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はWALのバイナリレイアウトに直接影響するため、フィールドの追加や変更を行う際は、replorigindesc.c のデコードロジックおよびリカバリ処理との整合性を確認してください。
