目的

PostgreSQL のコマンドラインユーティリティ createuser のメインエントリポイントとして、ユーザーからの引数解析、データベース接続の確立、および CREATE ROLE SQL コマンドの構築と実行を制御します。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。
  • オプション引数: ロール属性（superuser, createdb 等）、接続先情報（host, port, username）、および対話モードフラグ。
  • **出力**:
  • 正常終了時は 0、エラー発生時は 1 を返します。

前提条件と不変条件

  • pg_logging_init およびロケール設定が初期化されている必要があります。
  • getopt_long によって解析されるオプションは、long_options 構造体で定義された仕様に従う必要があります。
  • 属性の指定状態は trivalue 型（TRI_DEFAULT, TRI_YES, TRI_NO）で管理され、デフォルト値と明示的な指定が区別されます。

アルゴリズムまたは振る舞い

1. **初期化**: ロギングとロケールを初期化し、ヘルプ・バージョンオプションを処理します。 2. **引数解析**: getopt_long を使用してコマンドライン引数をループ処理し、ロール属性や接続パラメータを内部変数に格納します。 3. **状態管理**:

  • SimpleStringList を使用して、ロール名やメンバーシップ情報を保持します。
  • trivalue 型を用いて、各ロール属性の指定状態を追跡します。

    • 4. **引数検証**: 解析終了後、残りの引数（ロール名）を特定します。 5. **後続処理**: 接続確立、SQL 文の構築、およびサーバーへの実行（ソース抜粋範囲外）。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（プログラムのエントリポイント）。
  • **呼び出し先**:
  • pg_logging_init, get_progname, set_pglocale_pgservice
  • handle_help_version_opts
  • getopt_long
  • simple_string_list_append
  • option_parse_int
  • pg_strdup
  • exit
  • pg_log_error_hint

関連データ構造

  • struct option long_options[]: コマンドラインオプションの定義。
  • SimpleStringList: ロール名やメンバーシップ情報を保持するリスト。
  • enum trivalue: 属性の指定状態（デフォルト/YES/NO）を管理する列挙型。
  • ConnParams: 接続パラメータを保持する構造体。
  • PQExpBufferData: SQL 文構築用のバッファ。

失敗モードと境界条件

  • 不正なオプションや引数が指定された場合、pg_log_error_hint を出力して exit(1) で終了します。
  • option_parse_int による数値変換に失敗した場合、exit(1) で終了します。

保守担当者向けメモ

  • 本関数は createuser の制御フローの起点です。SQL の構築ロジックやサーバーとの通信処理は、本関数の後半（ソース抜粋範囲外）で実行されます。
  • trivalue 型の使用は、ユーザーの明示的な指定とデフォルト値を区別するために不可欠です。変更時はこの型の一貫性に注意してください。
