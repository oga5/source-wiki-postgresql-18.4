GISTPageOpaqueData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/gist.h:80

目的

GiST インデックスページの末尾（Special Space）に配置されるメタデータ構造体です。ページ分割の追跡や、インデックスページとしての識別、状態管理に使用されます。

シグネチャまたは形状

typedef struct GISTPageOpaqueData
{
    PageGistNSN nsn;        /* ページ分割時に変更されるノードシーケンス番号 */
    BlockNumber rightlink;  /* 次のページへのリンク（存在する場合） */
    uint16      flags;      /* ページの状態フラグ */
    uint16      gist_page_id; /* GiST インデックス識別用 ID */
} GISTPageOpaqueData;

入力と出力

本構造体は関数ではなくデータ構造であるため、入力と出力の概念は適用されません。メモリ上の特定のページレイアウトとして存在します。

前提条件と不変条件

  • GiST インデックスの各ページにおいて、末尾の Special Space に配置される必要があります。
  • PageInit 関数等でページを初期化する際、sizeof(GISTPageOpaqueData) が Special Space のサイズとして指定される必要があります。
  • nsn は並行処理中のページ分割を検知するために使用され、ページ分割時にのみ更新されます。

アルゴリズムまたは振る舞い

  • **ページ分割検知**: nsn を親ノードの LSN と比較することで、スキャン中に子ノードが分割されたかどうかを判定します。
  • **ページリンク**: rightlink により、インデックス内のページ間を横断的に辿ることが可能です。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/pageinspect/gistfuncs.c: ページ構造の検証に使用。
  • contrib/pgstattuple/pgstattuple.c: 統計情報取得時のページ判定に使用。
  • src/backend/access/gist/gistbuild.c: インデックス構築時の空き容量計算に使用。
  • src/backend/access/gist/gistutil.c: ページ初期化および構造検証に使用。
  • **呼び出し先**:
  • 本構造体はデータ定義であり、特定の関数を呼び出すことはありません。

関連データ構造

  • GISTPageOpaque: GISTPageOpaqueData へのポインタ型。
  • PageGistNSN: nsn フィールドで使用される、ページ分割追跡用の LSN。

失敗モードと境界条件

  • PageGetSpecialSize が MAXALIGN(sizeof(GISTPageOpaqueData)) と一致しない場合、ページ構造が破損している、あるいは GiST インデックスページではないと判断されます。
  • ページ内の空き容量計算（BLCKSZ - SizeOfPageHeaderData - sizeof(GISTPageOpaqueData)）において、アライメントを考慮しない計算を行うとメモリレイアウトの不整合を招く恐れがあります。

保守担当者向けメモ

  • 本構造体のサイズを変更する場合、MAXALIGN を介した計算を行っている箇所（gist_private.h や gist.h 内のマクロ）が正しく追従しているか確認してください。
  • nsn の更新ロジックは GiST の並行制御の根幹であるため、変更時は WAL ログの整合性に十分注意してください。
