ECPGfree_struct_member

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/type.c:617

目的

ECPGプリプロセッサにおいて、構造体のメンバーリスト（ECPGstruct_member）が保持する動的メモリを再帰的に解放する。

シグネチャまたは形状

void ECPGfree_struct_member(struct ECPGstruct_member *rm)

入力と出力

  • **入力**: rm - 解放対象となる ECPGstruct_member 構造体のリストの先頭ポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • rm が指すリストは、next ポインタによって連結されている必要がある。
  • リストの終端は NULL であること。
  • 各メンバー内の name フィールドおよび type フィールドは、適切に割り当てられたメモリを指している必要がある。

アルゴリズムまたは振る舞い

1. rm が NULL でない限り、リストを順次走査する。 2. 現在のノード p を保持し、リストのポインタを rm->next へ進める。 3. p->name を解放する。 4. ECPGfree_type(p->type) を呼び出し、メンバーの型情報を再帰的に解放する。 5. 最後にノード p 自体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/ecpg.c:432
  • src/interfaces/ecpg/preproc/type.c:658
  • src/interfaces/ecpg/preproc/variable.c:282
  • **呼び出し先**:
  • ECPGfree_type (型情報の解放)
  • free (標準ライブラリ)

関連データ構造

  • ECPGstruct_member: 構造体のメンバー情報を保持するリスト構造。
  • ECPGtype: メンバーの型情報を保持する構造体。

失敗モードと境界条件

  • rm が NULL の場合、何も行わずに正常終了する。
  • リストの各要素が指すポインタが不正な場合、free 呼び出し時に未定義動作が発生する可能性がある。

保守担当者向けメモ

  • 本関数は ECPGstruct_member のリストを走査する唯一のクリーンアップ関数である。構造体定義の変更時には、ECPGfree_type との整合性を維持すること。
  • メモリ管理は mm_alloc 等のラッパー経由で行われることが推奨されているため、本関数内で直接 free を呼ぶ際は、割り当て元との整合性に注意が必要である。
