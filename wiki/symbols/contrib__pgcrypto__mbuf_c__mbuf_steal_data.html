mbuf_steal_data

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/mbuf.c:161

目的

MBuf 構造体が保持するデータバッファの所有権を呼び出し元へ移譲し、バッファへのアクセスを無効化します。

シグネチャまたは形状

int mbuf_steal_data(MBuf *mbuf, uint8 **data_p)

入力と出力

  • **入力**:
  • mbuf: 操作対象の MBuf 構造体へのポインタ。
  • data_p: バッファの先頭アドレスを格納するための uint8 ポインタへのポインタ。
  • **出力**:
  • 戻り値: 移譲されたバッファのサイズ（バイト単位）。

前提条件と不変条件

  • mbuf は有効な MBuf 構造体を指している必要があります。
  • 本関数実行後、mbuf 構造体内のデータポインタおよび関連する境界ポインタはすべて NULL にリセットされます。

アルゴリズムまたは振る舞い

1. mbuf_size(mbuf) を呼び出し、現在のバッファサイズを取得します。 2. mbuf->no_write を true に設定し、書き込みを禁止します。 3. mbuf->own_data を false に設定し、バッファの所有権を放棄します。 4. mbuf->data の値を *data_p に代入し、呼び出し元へバッファの参照を渡します。 5. mbuf 構造体内の data、data_end、read_pos、buf_end をすべて NULL に初期化します。 6. 取得したサイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c (関数内にて res_len の取得に使用)
  • **呼び出し先**:
  • mbuf_size (同ファイル内)

関連データ構造

  • MBuf: 本関数によって内部データが抽出・リセットされる対象の構造体。

失敗モードと境界条件

  • mbuf が NULL の場合、ポインタ参照によるクラッシュが発生する可能性があります。
  • 本関数はバッファの所有権を移譲するため、呼び出し後に mbuf_free を呼び出す際は、内部データが既に NULL になっていることを考慮する必要があります。

保守担当者向けメモ

  • 本関数は、メモリの再割り当てやコピーを避け、バッファの所有権を効率的に外部へ引き渡すために使用されます。
  • 呼び出し元は、受け取った *data_p のメモリ解放責任を負う必要があります。
