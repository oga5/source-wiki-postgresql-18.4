sepgsql_relation_truncate

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/relation.c:523

目的

リレーション（テーブルまたはパーティションテーブル）に対する TRUNCATE 操作の実行権限を、SELinux ポリシーに基づいて検証する。

シグネチャまたは形状

void sepgsql_relation_truncate(Oid relOid)

入力と出力

  • **入力**:
  • relOid: 対象リレーションの OID。
  • **出力**: なし（権限チェックに失敗した場合は例外が発生する）。

前提条件と不変条件

  • 対象リレーションが RELKIND_RELATION または RELKIND_PARTITIONED_TABLE である必要がある。それ以外のリレーション種別（ビューやシーケンス等）に対しては何も行わない。

アルゴリズムまたは振る舞い

1. get_rel_relkind(relOid) を呼び出し、対象リレーションの種別を取得する。 2. 種別がテーブルまたはパーティションテーブルであれば、SELinux クラス SEPG_CLASS_DB_TABLE を指定する。それ以外の場合は処理を終了する。 3. getObjectIdentity を使用して、対象リレーションの識別名（audit_name）を取得する。 4. sepgsql_avc_check_perms を呼び出し、db_table クラスに対する SEPG_DB_TABLE__TRUNCATE 権限を検証する。 5. 最後に pfree(audit_name) を呼び出し、メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:198
  • **呼び出し先**:
  • get_rel_relkind
  • getObjectIdentity
  • sepgsql_avc_check_perms
  • pfree

関連データ構造

  • ObjectAddress: 権限チェック対象のオブジェクトを特定するために使用。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • 対象リレーションがテーブルまたはパーティションテーブル以外の場合、権限チェックはスキップされる。
  • SELinux ポリシーにより truncate 権限が拒否された場合、sepgsql_avc_check_perms 内でエラーが発生し、トランザクションが中断される。

保守担当者向けメモ

  • 本関数は TRUNCATE コマンド実行時のセキュリティフックとして機能する。
  • インデックスや他のリレーション種別に対する挙動を変更する場合は、switch 文のケース追加を検討すること。
