CreateTupleQueueReader

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tqueue.c:138

目的

並列クエリ実行において、共有メモリキューからタプルを読み出すための TupleQueueReader 構造体を初期化し、割り当てを行う。

シグネチャまたは形状

TupleQueueReader *CreateTupleQueueReader(shm_mq_handle *handle)

入力と出力

  • **入力**: shm_mq_handle *handle（共有メモリキューへのハンドル）
  • **出力**: 初期化された TupleQueueReader 構造体へのポインタ。

前提条件と不変条件

  • palloc0 を使用してメモリを確保するため、PostgreSQL のメモリコンテキストが有効である必要がある。
  • handle は有効な共有メモリキューハンドルを指していること。

アルゴリズムまたは振る舞い

1. palloc0 を用いて TupleQueueReader サイズのメモリ領域をゼロ初期化して確保する。 2. 確保した構造体の queue メンバに、引数で受け取った handle を格納する。 3. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:906（pei->reader[i] = CreateTupleQueueReader(pei->tqueue[i]);）
  • **呼び出し先**: palloc0（PostgreSQL メモリ管理関数）

関連データ構造

  • TupleQueueReader: 受信側の状態を保持する構造体。
  • shm_mq_handle: 共有メモリキューの制御ハンドル。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • handle が NULL の場合、構造体は作成されるが、その後の読み出し操作で異常が発生する可能性がある。

保守担当者向けメモ

  • 本関数は TupleQueueReader のメモリ確保とハンドル設定のみを行う。実際のタプル読み出しは TupleQueueReaderNext が担当する。
  • 読み取ったタプルは共有メモリ上のポインタを直接指すため、メモリ管理のライフサイクルに注意すること。
