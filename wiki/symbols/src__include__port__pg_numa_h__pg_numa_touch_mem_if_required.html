pg_numa_touch_mem_if_required

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_numa.h:27

目的

指定されたメモリ領域を強制的にアクセス（タッチ）することで、ページフォールトを誘発し、物理メモリへのマッピングを確定させるために使用されます。これにより、後続の NUMA ノード配置照会処理が正確な結果を得られるようにします。

シグネチャまたは形状

static inline void pg_numa_touch_mem_if_required(void *ptr)

入力と出力

  • **入力**: ptr (void *) - 物理メモリへのマッピングを確定させたいメモリ領域の先頭アドレス。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 ptr は有効なメモリ領域を指している必要があります。
  • 本関数は volatile 修飾子を介してメモリ読み取りを行うため、コンパイラによる最適化でアクセスが省略されることはありません。

アルゴリズムまたは振る舞い

1. volatile uint64 型の変数 touch を宣言します。 2. ptr が指すメモリ位置から uint64 サイズの値を読み取り、touch に代入します。 3. この読み取り操作により、OS は対象ページに対してページフォールトを発生させ、物理メモリを割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_buffercache/pg_buffercache_pages.c:379
  • src/backend/storage/ipc/shmem.c:645
  • **呼び出し先**:
  • 特になし（標準的なメモリ読み取り操作のみを実行）。

関連データ構造

  • uint64: PostgreSQL の基本型定義（c.h 経由）であり、メモリ読み取りの単位として使用されます。

失敗モードと境界条件

  • ptr が NULL または不正なメモリアドレスである場合、セグメンテーション違反が発生します。
  • メモリが既に物理的に割り当てられている場合、単なる読み取り操作として完了します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • pg_attribute_unused() が使用されているのは、読み取った値自体は利用されないため、コンパイラによる警告を抑制する目的です。
  • NUMA 関連の照会処理（pg_numa_query_pages 等）の直前に呼び出すことが必須です。省略すると、カーネルがページを物理的に割り当てていない場合に正しいノード情報が取得できない可能性があります。
