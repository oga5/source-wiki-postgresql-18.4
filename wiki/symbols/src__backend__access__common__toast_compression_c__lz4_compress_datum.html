lz4_compress_datum

定義

  • src/backend/access/common/toast_compression.c:138

目的

入力された varlena データを LZ4 アルゴリズムを用いて圧縮し、圧縮後のデータを格納した新しい varlena 構造体を返します。

シグネチャまたは形状

struct varlena *lz4_compress_datum(const struct varlena *value)

入力と出力

  • **入力**: 圧縮対象となる varlena 構造体へのポインタ (value)。
  • **出力**: 圧縮されたデータを含む varlena 構造体へのポインタ。圧縮効率が悪い（圧縮後の方がサイズが大きい）場合、または圧縮が不要と判断された場合は NULL を返します。

前提条件と不変条件

  • ビルド時に USE_LZ4 が定義されている必要があります。未定義の場合、本関数は NO_LZ4_SUPPORT() を呼び出しエラーとなります。
  • 圧縮後のデータサイズが元のデータサイズを超える場合、メモリを解放し NULL を返すことで、圧縮を行わない選択をします。

アルゴリズムまたは振る舞い

1. VARSIZE_ANY_EXHDR を用いて入力データのサイズを取得します。 2. LZ4_compressBound を使用して、圧縮後の最大サイズを算出します。 3. palloc を用いて、最大サイズと VARHDRSZ_COMPRESSED（ヘッダー分）の合計メモリを確保します。 4. LZ4_compress_default を呼び出し、データを圧縮します。 5. 圧縮結果の長さ (len) が 0 以下であればエラー (elog) を発生させます。 6. 圧縮後のサイズが元のサイズより大きい場合、確保したメモリを pfree し NULL を返します。 7. 圧縮成功時、SET_VARSIZE_COMPRESSED を用いてヘッダーを更新し、圧縮済みデータを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/toast_internals.c:71
  • **呼び出し先**:
  • LZ4_compressBound (LZ4ライブラリ)
  • LZ4_compress_default (LZ4ライブラリ)
  • palloc / pfree (PostgreSQLメモリ管理)
  • elog (エラー報告)

関連データ構造

  • varlena: PostgreSQLの可変長データ型。
  • VARHDRSZ_COMPRESSED: 圧縮された varlena のヘッダーサイズ。

失敗モードと境界条件

  • **LZ4未サポート**: USE_LZ4 が無効な環境で呼び出された場合、NO_LZ4_SUPPORT() によりエラーとなります。
  • **圧縮失敗**: LZ4_compress_default が 0 以下の値を返した場合、elog(ERROR, ...) によりトランザクションが中断されます。
  • **圧縮効率**: 圧縮後のデータが元のデータより大きい場合、圧縮は破棄され NULL が返されます。

保守担当者向けメモ

  • 本関数は varlena ヘッダーの圧縮フラグを考慮したメモリレイアウトを前提としています。
  • LZ4ライブラリのバージョン依存性については、lz4_decompress_datum_slice 等の関連関数における制約も併せて確認してください。
