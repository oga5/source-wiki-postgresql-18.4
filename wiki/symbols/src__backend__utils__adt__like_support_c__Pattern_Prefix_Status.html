Pattern_Prefix_Status

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/like_support.c:67

目的

パターンマッチング（LIKE や正規表現など）において、抽出された固定接頭辞（prefix）がパターン全体に対してどのような性質を持つかを示す状態を定義します。

シグネチャまたは形状

typedef enum
{
    Pattern_Prefix_None,    /* 固定接頭辞なし */
    Pattern_Prefix_Partial, /* 部分的な接頭辞（パターンの一部のみ固定） */
    Pattern_Prefix_Exact    /* パターン全体が固定文字列と一致 */
} Pattern_Prefix_Status;

入力と出力

  • 本シンボルは列挙型であり、関数への入力や出力として Pattern_Prefix_Status 型の変数が使用されます。

前提条件と不変条件

  • like_support.c 内のパターン解析関数（pattern_fixed_prefix 等）が、パターン文字列を解析した結果としてこのいずれかの値を返します。

アルゴリズムまたは振る舞い

  • プランナがインデックススキャンを利用可能かどうかを判断する際、このステータスを参照します。
  • Pattern_Prefix_Exact の場合、インデックススキャンは完全一致検索に最適化されます。
  • Pattern_Prefix_Partial の場合、インデックススキャンは接頭辞を用いた範囲検索（>= や <）に変換されます。
  • Pattern_Prefix_None の場合、インデックスによる最適化は行われません。

呼び出し元と呼び出し先

  • **呼び出し元**: like_regex_support や、like_fixed_prefix、regex_fixed_prefix などの内部解析関数がこの型を生成・利用します。
  • **呼び出し先**: 本シンボル自体は型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • Pattern_Type: パターンマッチングの種類を定義する列挙型。
  • Const: パターン文字列を保持するノード構造体。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 新しいパターンマッチング手法を追加する際は、この列挙型が網羅的であるか確認してください。
  • Pattern_Prefix_Partial と Pattern_Prefix_Exact の境界は、ロケールや照合順序（Collation）に依存する可能性があるため、解析ロジックの変更時には注意が必要です。
