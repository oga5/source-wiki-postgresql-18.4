pg_euc_mblen

定義

  • 種別: function
  • 定義箇所: src/common/wchar.c:148

目的

EUC（Extended Unix Code）エンコーディングにおける、指定されたマルチバイト文字のバイト長を判定します。

シグネチャまたは形状

static inline int pg_euc_mblen(const unsigned char *s)

入力と出力

  • **入力**: s - 判定対象のマルチバイト文字列の先頭ポインタ。
  • **出力**: 文字のバイト長（1, 2, または 3 を返す）。

前提条件と不変条件

  • 入力ポインタ s は有効なメモリ領域を指している必要があります。
  • SS2（Single Shift 2）および SS3（Single Shift 3）定数が定義されている環境を前提とします。

アルゴリズムまたは振る舞い

1. 先頭バイト *s が SS2 の場合、長さ 2 を返します。 2. 先頭バイト *s が SS3 の場合、長さ 3 を返します。 3. IS_HIGHBIT_SET(*s) が真（最上位ビットが立っている）の場合、長さ 2 を返します。 4. 上記いずれにも該当しない場合、ASCII文字とみなし長さ 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/wchar.c 内の複数の箇所から呼び出されています（行149, 192, 223, 446）。
  • **呼び出し先**: 本関数はインライン関数であり、他の関数を呼び出しません。

関連データ構造

  • mb/pg_wchar.h: エンコーディング関連の定数およびマクロ定義を提供します。

失敗モードと境界条件

  • 入力ポインタが指す先が不正なエンコーディングである場合、本関数は単にバイト長を返すのみであり、妥当性の検証は行いません。
  • IS_HIGHBIT_SET マクロの定義に依存するため、環境ごとのビット操作の挙動に左右されます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスが重視されるパスで使用されます。
  • EUCエンコーディングの仕様変更や、新しいEUC派生エンコーディングの追加時には、このバイト長判定ロジックが適切か再確認してください。
