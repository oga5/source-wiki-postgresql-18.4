prefix_init

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:243

目的

PGP 暗号化データの復号開始時に、暗号ブロックの先頭に含まれるランダムなプレフィックス（チェック用データ）を読み取り、整合性を検証する。

シグネチャまたは形状

static int prefix_init(void **priv_p, void *arg, PullFilter *src);

入力と出力

  • **入力**:
  • priv_p: 未使用。
  • arg: PGP_Context 構造体へのポインタ（暗号アルゴリズム情報を含む）。
  • src: データを読み出すための PullFilter。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は負の px エラーコード（PXE_BUG や PXE_PGP_CORRUPT_DATA 等）を返す。

前提条件と不変条件

  • ctx->cipher_algo から取得されるブロックサイズが PGP_MAX_BLOCK 以下であること。
  • 読み取り対象のストリームには、少なくとも「ブロックサイズ + 2」バイトのデータが存在すること。

アルゴリズムまたは振る舞い

1. pgp_get_cipher_block_size を呼び出し、現在の暗号アルゴリズムのブロックサイズ（len）を取得する。 2. pullf_read_max を使用して、ストリームから len + 2 バイトを読み込む。 3. 読み取ったバッファの末尾 2 バイト（buf[len] と buf[len+1]）が、それぞれ len - 2 バイト目と len - 1 バイト目の値と一致するかを確認する。 4. 一致しない場合、ctx->corrupt_prefix フラグを立ててエラーを報告する。 5. 使用した一時バッファを px_memset でゼロクリアし、機密情報の漏洩を防ぐ。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、復号パイプラインの初期化プロセスで参照されていることが示唆される。
  • **呼び出し先**: pgp_get_cipher_block_size（ブロックサイズ取得）、pullf_read_max（データ読み取り）、px_debug（エラーログ出力）、px_memset（メモリクリア）。

関連データ構造

  • PGP_Context: 復号状態とアルゴリズム情報を保持。
  • PullFilter: ストリーム読み取り用のフィルタ構造体。

失敗モードと境界条件

  • len > PGP_MAX_BLOCK: PXE_BUG を返す。
  • 読み取りデータ長が不足している場合: PXE_PGP_CORRUPT_DATA を返す。
  • プレフィックスのチェックサムが一致しない場合: ctx->corrupt_prefix を 1 に設定し、PXE_PGP_CORRUPT_DATA を返す。

保守担当者向けメモ

  • 本関数は「クイックチェック」として機能する。エラー発生時は pgp_decrypt 側で適切に処理されることを想定している。
  • プレフィックスの不一致はデータ破損または鍵の不一致を示すため、セキュリティ上の理由から詳細なエラー内容は隠蔽される。
