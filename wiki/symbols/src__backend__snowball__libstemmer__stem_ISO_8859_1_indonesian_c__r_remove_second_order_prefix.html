r_remove_second_order_prefix

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_indonesian.c:273

目的

インドネシア語のステミング処理において、単語の2次接頭辞（second-order prefix）を特定し、除去または置換を行う。

シグネチャまたは形状

static int r_remove_second_order_prefix(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体。現在の単語データやワークスペース I を保持）
  • 出力: int（成功時は 1、処理対象外または失敗時は 0、エラー時は負の値を返す）

前提条件と不変条件

  • z->c が現在の処理位置を指していること。
  • z->p に単語データが格納されていること。
  • 処理の過程で z->I[1]（単語内の母音数）が更新される。

アルゴリズムまたは振る舞い

1. 現在位置の次（z->c + 1）が文字コード 101（'e'）であるかを確認し、一致しない場合は終了する。 2. find_among(z, a_4, 6) を呼び出し、a_4 配列に基づいたパターンマッチングを行う。 3. マッチした among_var の値に応じて以下の処理を実行する:

  • **case 1**: 接頭辞を削除し、z->I[0] を 2 に設定、母音数 z->I[1] を 1 減らす。
  • **case 2**: 接頭辞を s_5 に置換し、母音数 z->I[1] を 1 減らす。
  • **case 3**: 接頭辞を削除し、z->I[0] を 4 に設定、母音数 z->I[1] を 1 減らす。
  • **case 4**: 接頭辞を s_6 に置換し、z->I[0] を 4 に設定、母音数 z->I[1] を 1 減らす。

    • 4. slice_del または slice_from_s が負の値を返した場合は、その値をそのまま呼び出し元へ伝播させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_ISO_8859_1_indonesian.c 内のステミング処理フロー（行369, 382）。また、stem_UTF_8_indonesian.c からも参照されている。
  • 呼び出し先: find_among, slice_del, slice_from_s（いずれもSnowballランタイムユーティリティ）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_4: 2次接頭辞のパターンを定義した among 構造体の配列。

失敗モードと境界条件

  • z->c + 1 が文字列長 z->l を超える場合、または文字コードが一致しない場合は 0 を返す。
  • find_among がマッチしない場合は 0 を返す。
  • slice_del や slice_from_s が内部エラー（負の値）を返した場合、そのエラーコードを返す。

保守担当者向けメモ

  • 本関数は自動生成コードの一部である。アルゴリズムの変更が必要な場合は、直接編集せず元のSnowballスクリプト（.sbl）を修正すること。
  • z->I[0] は接頭辞の種類を追跡するフラグとして使用されている。
