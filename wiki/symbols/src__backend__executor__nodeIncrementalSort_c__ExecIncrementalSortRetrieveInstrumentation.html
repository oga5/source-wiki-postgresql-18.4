ExecIncrementalSortRetrieveInstrumentation

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:1232

目的

パラレルクエリ実行時において、各ワーカーから収集されたインクリメンタルソートの統計情報（SharedIncrementalSortInfo）をローカルメモリにコピーし、ノードの状態を更新する。

シグネチャまたは形状

void ExecIncrementalSortRetrieveInstrumentation(IncrementalSortState *node)

入力と出力

  • **入力**: IncrementalSortState *node（インクリメンタルソートノードの状態構造体）
  • **出力**: なし（副作用として node->shared_info が更新される）

前提条件と不変条件

  • node->shared_info が NULL の場合、本関数は何も行わず即座に復帰する。
  • node->shared_info には、ワーカー数に応じた IncrementalSortInfo 配列が含まれている必要がある。

アルゴリズムまたは振る舞い

1. node->shared_info が NULL かどうかを確認し、該当する場合は処理を終了する。 2. SharedIncrementalSortInfo 構造体のサイズを、offsetof を用いて計算する（ヘッダー部分とワーカー数分の IncrementalSortInfo 配列の合計）。 3. 計算したサイズ分を palloc でメモリ確保する。 4. 共有メモリ上の統計情報を確保したローカルメモリ領域へ memcpy でコピーする。 5. node->shared_info ポインタを、コピーしたローカルメモリ領域のアドレスで更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1090
  • **呼び出し先**:
  • palloc (メモリ確保)
  • memcpy (メモリコピー)

関連データ構造

  • IncrementalSortState: インクリメンタルソートノードの実行状態を保持する構造体。
  • SharedIncrementalSortInfo: パラレルクエリ環境下で共有されるソート統計情報。
  • IncrementalSortInfo: 個別のワーカーにおけるソート統計情報。

失敗モードと境界条件

  • node->shared_info が NULL の場合、安全に早期復帰する。
  • メモリ不足時には palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数はパラレルクエリの統計収集プロセスにおいて重要である。
  • SharedIncrementalSortInfo の構造変更（特に可変長配列部分）を行う際は、offsetof を使用したサイズ計算ロジックが正しく維持されているか確認すること。
