SearchSysCacheLocked1

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/syscache.c:286

目的

システムカタログのタプルを検索し、そのタプルに対して InplaceUpdateTupleLock を取得してロックした状態で返す。主にインプレース更新を行う直前の整合性確保に使用される。

シグネチャまたは形状

HeapTuple SearchSysCacheLocked1(int cacheId, Datum key1)

入力と出力

  • **入力**:
  • cacheId: 検索対象のシステムキャッシュID。
  • key1: 検索に使用するキー値（Datum）。
  • **出力**:
  • 検索された HeapTuple。見つからない場合は NULL。

前提条件と不変条件

  • 呼び出し側は、返されたタプルに対して適切なタイミングでロックを解放する必要がある（直接的な解放関数はソース内に明示されていないが、LockRelease が内部で使用されている）。
  • インプレース更新の競合を防ぐため、タプルの取得とロック取得のシーケンスが厳密に管理されている。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を呼び出し、対象タプルを検索する。 2. 取得したタプルの t_self（TID）を記録し、LockAcquire を用いて LOCKTAG_TUPLE を取得する。 3. ロック取得後、再度 SearchSysCache1 を呼び出し、TID が一致するかを確認する。 4. TID が一致しない場合は、古いロックを解放してループを継続し、最新のタプルに対して再度ロック取得を試みる。 5. AcceptInvalidationMessages を呼び出し、保留中のキャッシュ無効化メッセージを処理する。 6. TID が一致した時点で、ロックされたタプルを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/catalog/aclchk.c
  • src/backend/commands/tablecmds.c
  • src/backend/utils/cache/syscache.c (内部的な再帰的利用の可能性あり)
  • **呼び出し先**:
  • SearchSysCache1
  • LockAcquire / LockRelease
  • AcceptInvalidationMessages
  • ReleaseSysCache

関連データ構造

  • CatCache: システムキャッシュの管理構造体。
  • HeapTuple: 検索対象のタプル。
  • LOCKTAG: ロック対象を識別するためのタグ。

失敗モードと境界条件

  • 対象のタプルが存在しない場合、NULL を返す。
  • 同時実行環境において、タプルが更新された場合はループ内で再試行が行われる。
  • ItemPointer が無効な場合や、検索結果が一致しない場合はロックの解放と再取得が繰り返される。

保守担当者向けメモ

  • 本関数は複雑なロックシーケンスを伴うため、インプレース更新を行う特定のケース以外での使用は避けること。
  • SearchSysCacheCopy との統合が検討されることがあるが、本関数はロック取得後の整合性を保証する点に特化している。
