AtSubStart_ResourceOwner

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xact.c:1282

目的

サブトランザクションの開始時に、そのサブトランザクション専用のリソースオーナー（ResourceOwner）を作成し、現在のリソース管理コンテキストとして設定する。

シグネチャまたは形状

static void AtSubStart_ResourceOwner(void)

入力と出力

  • **入力**: なし（グローバルな CurrentTransactionState を参照）。
  • **出力**: なし（副作用として CurrentTransactionState 内の curTransactionOwner を更新し、グローバル変数 CurTransactionResourceOwner および CurrentResourceOwner を更新する）。

前提条件と不変条件

  • CurrentTransactionState が指すトランザクション状態において、親トランザクションが存在すること（s->parent != NULL）がアサーションにより保証されている。

アルゴリズムまたは振る舞い

1. 現在のトランザクション状態（CurrentTransactionState）を取得する。 2. 親トランザクションのリソースオーナー（s->parent->curTransactionOwner）を親として、新しいリソースオーナーを ResourceOwnerCreate で作成する。 3. 作成したリソースオーナーを、現在のトランザクション状態の curTransactionOwner に割り当てる。 4. グローバル変数 CurTransactionResourceOwner および CurrentResourceOwner を、新しく作成したリソースオーナーで更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c 内のサブトランザクション開始処理（src/backend/access/transam/xact.c:5083 等）。
  • **呼び出し先**: ResourceOwnerCreate（リソースオーナーの生成）。

関連データ構造

  • TransactionStateData: トランザクションの状態を保持する構造体。
  • ResourceOwner: リソース（メモリ、ロック、ファイル等）の所有権を管理する構造体。

失敗モードと境界条件

  • ResourceOwnerCreate がメモリ不足等で失敗した場合、例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数はサブトランザクションの階層構造をリソース管理に反映させるための重要なステップである。親トランザクションのリソースオーナーを継承する形となるため、サブトランザクション終了時のリソース解放順序と整合している必要がある。
