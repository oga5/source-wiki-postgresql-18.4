transformTableLikeClause

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_utilcmd.c:1125

目的

CREATE TABLE ... (LIKE ...) 構文において、指定された親リレーションから属性、制約、および関連するメタデータを抽出し、新しいテーブル定義（CreateStmtContext）へ統合する。

シグネチャまたは形状

static void transformTableLikeClause(CreateStmtContext *cxt, TableLikeClause *table_like_clause)

入力と出力

  • **入力**:
  • cxt: 変換中のテーブル定義状態を保持する CreateStmtContext 構造体。
  • table_like_clause: 解析された LIKE 句のノード情報。
  • **出力**: なし（副作用として cxt 内の columns リストや alist に定義を追加する）。

前提条件と不変条件

  • table_like_clause->relation が指すリレーションが、テーブル、ビュー、マテリアライズドビュー、複合型、外部テーブル、またはパーティションテーブルのいずれかであること。
  • 呼び出し元で適切なロック（AccessShareLock）が取得されていること。
  • 外部テーブル（cxt->isforeign が真）の場合、ID列、ストレージ設定、圧縮設定のコピーはスキップされる。

アルゴリズムまたは振る舞い

1. **リレーションのオープン**: relation_openrv を使用して対象リレーションを開き、型チェックを行う。 2. **権限チェック**: 対象が複合型の場合は ACL_USAGE を、それ以外は ACL_SELECT 権限を確認する。 3. **属性のコピー**:

  • 親リレーションの全属性を走査し、削除済み列を除外して ColumnDef を生成する。
  • オプションに応じて GENERATED 属性、ID列（シーケンス生成を含む）、ストレージ設定、圧縮設定をコピーする。
  • CREATE_TABLE_LIKE_COMMENTS 指定時は、列コメントを抽出し CommentStmt を cxt->alist に追加する。

    • 4. **制約のコピー**:

  • NOT NULL 制約を抽出し cxt->nnconstraints に追加する。
  • コメントオプションが有効な場合、制約のコメントも cxt->alist に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_utilcmd.c 内の transformCreateStmt から呼び出される。
  • **呼び出し先**: relation_openrv, getIdentitySequence, sequence_options, generateSerialExtraStmts, GetComment, RelationGetNotNullConstraints 等。

関連データ構造

  • CreateStmtContext: 変換中のテーブル定義状態を保持。
  • TableLikeClause: LIKE 句の構文解析結果。
  • TupleDesc: リレーションの属性定義情報。
  • ColumnDef: 新規作成される列の定義ノード。

失敗モードと境界条件

  • 指定されたリレーションがサポート対象外の型（relkind）である場合、ERRCODE_WRONG_OBJECT_TYPE でエラー終了する。
  • 権限チェックで ACLCHECK_OK が返らない場合、aclcheck_error によりエラー終了する。

保守担当者向けメモ

  • 本関数は CREATE TABLE の解析フェーズで実行される。
  • cxt->alist に追加された CommentStmt は、テーブル作成後の後処理として実行される必要がある。
  • LIKE 句の処理は SQL99 準拠のため、属性の順序を維持して cxt->columns に追加する設計となっている。
