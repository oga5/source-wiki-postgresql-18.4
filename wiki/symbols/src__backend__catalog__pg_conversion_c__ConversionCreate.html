ConversionCreate

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_conversion.c:37

目的

新しい文字コード変換規則をシステムカタログ pg_conversion に登録し、関連する依存関係（プロシージャ、名前空間、所有者、拡張）を構築します。

シグネチャまたは形状

ObjectAddress ConversionCreate(const char *conname, Oid connamespace,
                               Oid conowner,
                               int32 conforencoding, int32 contoencoding,
                               Oid conproc, bool def)

入力と出力

  • **入力**:
  • conname: 変換規則名。
  • connamespace: 変換規則が属する名前空間のOID。
  • conowner: 変換規則の所有者のOID。
  • conforencoding, contoencoding: 変換元のエンコーディングIDと変換先のエンコーディングID。
  • conproc: 変換を実行するプロシージャのOID。
  • def: デフォルト変換として登録するかどうかのフラグ。
  • **出力**:
  • 作成された変換規則の ObjectAddress を返します。

前提条件と不変条件

  • conname が NULL であってはなりません。
  • 同一名前空間内に同名の変換規則が存在してはなりません。
  • def が真の場合、指定された名前空間において、同一のエンコーディングペアに対するデフォルト変換が既に存在してはなりません。

アルゴリズムまたは振る舞い

1. SearchSysCacheExists2 を使用して名前の重複をチェックし、必要に応じて FindDefaultConversion でデフォルト変換の重複を検証します。 2. table_open で pg_conversion を RowExclusiveLock モードで開きます。 3. GetNewOidWithIndex で新しいOIDを取得し、heap_form_tuple を用いてカタログタプルを構築します。 4. CatalogTupleInsert でカタログへ挿入します。 5. recordDependencyOn を呼び出し、プロシージャ、名前空間、所有者、および現在の拡張に対する依存関係を記録します。 6. InvokeObjectPostCreateHook を呼び出し、作成後のフック処理を実行します。 7. 最後にテーブルをクローズし、作成したオブジェクトの ObjectAddress を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/conversioncmds.c:132
  • **呼び出し先**:
  • SearchSysCacheExists2, FindDefaultConversion, table_open, GetNewOidWithIndex, heap_form_tuple, CatalogTupleInsert, recordDependencyOn, recordDependencyOnOwner, recordDependencyOnCurrentExtension, InvokeObjectPostCreateHook, table_close

関連データ構造

  • pg_conversion: 操作対象のシステムカタログ。
  • ObjectAddress: 戻り値として使用されるオブジェクト識別構造体。
  • HeapTuple: カタログ挿入時に使用されるタプル形式。

失敗モードと境界条件

  • conname が NULL の場合、elog(ERROR) で停止します。
  • 同名の変換規則が存在する場合、ERRCODE_DUPLICATE_OBJECT でエラーを報告します。
  • デフォルト変換が重複する場合、ERRCODE_DUPLICATE_OBJECT でエラーを報告します。

保守担当者向けメモ

  • 本関数はカタログ操作に特化しており、実際の文字コード変換ロジックは含みません。
  • 依存関係の追加順序は、カタログの整合性を維持するために重要です。
  • CONDEFAULT キャッシュを利用した検索を行っているため、カタログ定義の変更時にはキャッシュキーの整合性に注意してください。
