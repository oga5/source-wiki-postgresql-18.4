ResourceOwnerRememberWaitEventSet

定義

  • 種別: static inline function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:222

目的

指定された WaitEventSet を ResourceOwner に登録し、リソースのライフサイクル管理（所有権の追跡）を行う。

シグネチャまたは形状

static inline void ResourceOwnerRememberWaitEventSet(ResourceOwner owner, WaitEventSet *set)

入力と出力

  • 入力:
  • owner: リソースを所有する ResourceOwner 構造体へのポインタ。
  • set: 登録対象の WaitEventSet 構造体へのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • owner が有効な ResourceOwner であること。
  • set が初期化済みの WaitEventSet であること。
  • wait_event_set_resowner_desc がリソース記述子として定義されていること。

アルゴリズムまたは振る舞い

1. PointerGetDatum(set) を使用して WaitEventSet ポインタを Datum 型に変換する。 2. ResourceOwnerRemember を呼び出し、owner に対して set の追跡を開始する。この際、リソースの解放時に使用される記述子 wait_event_set_resowner_desc を渡す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/ipc/waiteventset.c:421（CreateWaitEventSet 内で呼び出されていることが確認できる）。
  • 呼び出し先:
  • ResourceOwnerRemember（PostgreSQL のリソース管理サブシステム）。

関連データ構造

  • ResourceOwner: リソースの所有権を管理する構造体。
  • WaitEventSet: 待機対象のイベント群を保持する構造体。
  • wait_event_set_resowner_desc: ResourceOwner がリソースを解放する際に使用するメタデータ。

失敗モードと境界条件

  • owner が NULL の場合、ResourceOwnerRemember の内部実装に依存する（通常はクラッシュまたはアサーションエラー）。
  • メモリ不足により ResourceOwner への登録が失敗した場合、ResourceOwnerRemember がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、主に CreateWaitEventSet 内でのリソース登録を簡略化するために使用される。
  • リソースの解放時には対となる ResourceOwnerForgetWaitEventSet が呼び出される必要がある。
