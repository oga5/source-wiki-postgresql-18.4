pro

定義

  • 種別: struct
  • 定義箇所: src/tools/pg_bsd_indent/args.c:86

目的

pg_bsd_indent における設定オプションの定義テーブルです。コマンドライン引数やプロファイルファイル（.indent.pro）で指定されたオプションを解析し、対応するグローバル変数へ値を反映させるためのマッピングを提供します。

シグネチャまたは形状

struct pro {
    const char *p_name;    /* オプション名（例: -bl, -cli） */
    int         p_type;    /* 型（PRO_BOOL, PRO_INT, PRO_SPECIAL） */
    int         p_default; /* デフォルト値（int型の場合） */
    int         p_special; /* 型に応じた特殊処理フラグ */
    int        *p_obj;     /* 関連付けられたグローバル変数へのポインタ */
};

入力と出力

  • **入力**: set_option 関数が引数文字列をこのテーブルと照合します。
  • **出力**: PRO_BOOL または PRO_INT 型の場合、p_obj が指すグローバル変数が更新されます。PRO_SPECIAL 型の場合、指定された特殊処理が実行されます。

前提条件と不変条件

  • テーブルの末尾は {0, 0, 0, 0, 0} で終了する必要があります。
  • eqin 関数による前方一致検索を行うため、長い名前のオプション（例: -lp）は、そのサブストリングであるオプション（例: -l）よりも先に定義する必要があります。

アルゴリズムまたは振る舞い

  • **テーブル駆動**: オプションの追加は、この配列に要素を追加するだけで完結します。
  • **解析フロー**:

    • 1. set_defaults() が p_default を使用してグローバル変数を初期化します。 2. set_option() が入力文字列を p_name と照合します。 3. 一致した場合、p_type に応じて値を代入するか、p_special に基づく特殊処理（型定義の追加やバージョン表示など）を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: set_defaults() および set_option() がこのテーブルを参照します。
  • **呼び出し先**: このシンボル自体はデータ構造であり、関数を呼び出しません。
  • **注意**: ツールが検出した他のファイル（contrib/amcheck 等）内の「pro」という文字列は、本シンボルとは無関係な単語（pro forma等）であり、呼び出し関係はありません。

関連データ構造

  • indent_globs.h: p_obj が指し示す対象となるグローバル変数が定義されています。

失敗モードと境界条件

  • 不明なオプションや引数不足が検出された場合、errx を呼び出してプログラムが即座に終了します。

保守担当者向けメモ

  • オプションを追加する際は、既存の定義との前方一致順序に注意してください。
  • PRO_SPECIAL 型の追加が必要な場合は、set_option 関数内のロジックを併せて更新する必要があります。
