ExecPrepareTuplestoreResult

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execSRF.c:863

目的

集合を返す関数（SRF）の実行結果を保持する Tuplestore を SetExprState に関連付け、結果読み取り用のスロットを準備および検証する。

シグネチャまたは形状

static void ExecPrepareTuplestoreResult(SetExprState *sexpr, ExprContext *econtext, Tuplestorestate *resultStore, TupleDesc resultDesc)

入力と出力

  • **入力**:
  • sexpr: SRF の実行状態を保持する構造体。
  • econtext: 式評価コンテキスト。
  • resultStore: 関数が生成した結果を格納する Tuplestore。
  • resultDesc: 結果の行構造を記述する TupleDesc。
  • **出力**: なし（sexpr の内部状態を更新する）。

前提条件と不変条件

  • sexpr->func.fn_mcxt が有効なメモリコンテキストであること。
  • resultDesc が動的に割り当てられた場合（tdrefcount == -1）、本関数内で解放される。
  • sexpr->shutdown_reg が false の場合、ShutdownSetExpr が econtext にコールバックとして登録される。

アルゴリズムまたは振る舞い

1. sexpr->funcResultStore に resultStore を設定する。 2. sexpr->funcResultSlot が未初期化の場合、sexpr->func.fn_mcxt コンテキスト下で MakeSingleTupleTableSlot を用いてスロットを作成する。

  • 行型は sexpr->funcResultDesc または resultDesc から決定する。いずれも不明な場合はエラーを報告する。

    • 3. resultDesc が提供されている場合、sexpr->funcResultDesc との整合性を tupledesc_match で検証する。 4. resultDesc が動的割り当て（tdrefcount == -1）であれば FreeTupleDesc で解放する。 5. sexpr->shutdown_reg が未設定であれば、RegisterExprContextCallback を呼び出し、クリーンアップ関数 ShutdownSetExpr を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execSRF.c 内の関数（ExecMakeTableFunctionResult 等から参照されている）。
  • **呼び出し先**: MemoryContextSwitchTo, CreateTupleDescCopy, MakeSingleTupleTableSlot, tupledesc_match, FreeTupleDesc, RegisterExprContextCallback, ShutdownSetExpr。

関連データ構造

  • SetExprState: SRF 実行状態を管理する構造体。
  • Tuplestorestate: 結果セットを保持するストレージ。
  • TupleDesc: 行のメタデータ。
  • ExprContext: 式評価の実行環境。

失敗モードと境界条件

  • sexpr->funcResultDesc および resultDesc の双方が NULL の場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。
  • tupledesc_match が不一致を検出した場合、エラーが発生する。

保守担当者向けメモ

  • 本関数は SRF の実行終了時にリソースを確実に解放するため、ShutdownSetExpr の登録を必須としている。新しい状態を追加する際は、この登録漏れに注意すること。
  • resultDesc のメモリ管理（tdrefcount による判定）は、クエリ実行中のメモリリークを防ぐために重要である。
