r_Step_1a

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_english.c:487

目的

Porterステミングアルゴリズムの「Step 1a」を実装し、単語末尾の複数形や所有格の接尾辞を削除または置換する。

シグネチャまたは形状

static int r_Step_1a(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（Snowballの実行環境状態）
  • 出力: 成功時に 1、失敗または処理対象外の場合に 0、メモリ操作エラー時に負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 内部で find_among_b を使用して後方からパターンマッチングを行うため、環境内のカーソル位置（z->c）が適切に管理されている必要がある。

アルゴリズムまたは振る舞い

1. **所有格・複数形の処理**:

  • 単語末尾が 's または s で終わる場合、a_1 テーブルを用いて接尾辞を検索し、一致すれば slice_del で削除する。

    • 2. **接尾辞の置換・削除**:

  • 単語末尾が d または s で終わる場合、a_2 テーブルを用いて検索を行う。
  • among_var の値に応じて以下の処理を行う:
  • 1: 接尾辞を s_2 に置換。
  • 2: 接尾辞を s_3 または s_4 に置換（UTF-8文字の長さを考慮したスキップ処理を含む）。
  • 3: 直前の文字が母音（g_v グループ）であるかを確認し、条件を満たせば接尾辞を削除。

呼び出し元と呼び出し先

  • 呼び出し元: stem_UTF_8_english.c 内のメイン処理フロー（english_UTF_8_stem からの呼び出しが想定される）。
  • 呼び出し先: find_among_b, slice_del, slice_from_s, skip_b_utf8, out_grouping_b_U。

関連データ構造

  • struct SN_env: ステミング処理の環境状態を保持。
  • a_1, a_2: 接尾辞検索用のテーブル。
  • g_v: 母音グループの定義。

失敗モードと境界条件

  • slice_del や slice_from_s が負の値を返した場合、エラーとしてそのまま伝播させる。
  • z->c が境界（z->lb）を超えた場合、処理を中断して 0 を返す。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによる自動生成コードである。ロジックの変更は直接行わず、対応する .sbl 定義ファイルを修正して再生成すること。
  • UTF-8のマルチバイト文字操作が含まれるため、skip_b_utf8 の挙動を変更する際は注意が必要である。
