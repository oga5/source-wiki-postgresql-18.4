create_ctas_internal

定義

  • 種別: function
  • 定義箇所: src/backend/commands/createas.c:81

目的

CREATE TABLE AS (CTAS) および CREATE MATERIALIZED VIEW コマンドにおいて、ターゲットとなるリレーション（テーブルまたはマテリアライズドビュー）の物理的な作成と、関連するメタデータ（TOASTテーブル、マテリアライズドビューのクエリ定義）の保存を行う。

シグネチャまたは形状

static ObjectAddress create_ctas_internal(List *attrList, IntoClause *into)

入力と出力

  • **入力**:
  • attrList: 作成するテーブルの列定義リスト。
  • into: IntoClause 構造体（ターゲットリレーション名、オプション、アクセスメソッド、マテリアライズドビューの場合はクエリ情報を含む）。
  • **出力**:
  • ObjectAddress: 作成されたリレーションのオブジェクトIDとクラスID。

前提条件と不変条件

  • into->viewQuery が存在する場合、マテリアライズドビューとして扱われる。
  • DefineRelation を呼び出すため、対象となるリレーションが既に存在する場合はエラーとなる。
  • TOASTテーブル作成後、CommandCounterIncrement() が実行され、後続の処理から可視となることが保証される。

アルゴリズムまたは振る舞い

1. IntoClause の情報に基づき、内部的に CreateStmt 構造体を構築する。 2. DefineRelation を呼び出し、テーブルを物理的に作成する。 3. NewRelationCreateToastTable を呼び出し、TOASTテーブルを生成する。 4. マテリアライズドビューの場合、StoreViewQuery を呼び出し、クエリ定義をシステムカタログに保存する。 5. 作成したリレーションの ObjectAddress を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/createas.c 内の CTAS 実行フロー（ExecCreateTableAs 等）。
  • **呼び出し先**: DefineRelation, NewRelationCreateToastTable, StoreViewQuery, transformRelOptions, heap_reloptions, copyObject, CommandCounterIncrement。

関連データ構造

  • IntoClause: CTAS コマンドの構文解析結果を保持。
  • CreateStmt: DefineRelation に渡すためのテーブル定義ノード。
  • ObjectAddress: 作成されたリレーションの識別子。

失敗モードと境界条件

  • ターゲットとなるリレーションが既に存在する場合、DefineRelation によりエラーが発生する。
  • reloptions の解析に失敗した場合、エラーが発生する。

保守担当者向けメモ

  • 本関数は CREATE TABLE AS と CREATE MATERIALIZED VIEW の両方のパスを統合しているため、修正時は両方の挙動に影響がないか確認すること。
  • StoreViewQuery は引数のクエリツリーを書き換える可能性があるため、copyObject を使用してコピーを渡している。
