ProcessClientReadInterrupt

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/postgres.c:501

目的

クライアントからの読み込み操作中に発生した割り込み要求を適切に処理し、プロセスの状態を整合させる。

シグネチャまたは形状

void ProcessClientReadInterrupt(bool blocked)

入力と出力

  • **入力**:
  • blocked (bool): 読み込み操作がブロック中であるかを示すフラグ。
  • **出力**: なし（副作用として割り込み処理を実行）。

前提条件と不変条件

  • 実行前後で errno の値が保存・復元される。
  • DoingCommandRead が真の場合、一般的な割り込みチェックおよび保留中の通知・キャッシュ更新処理が行われる。

アルゴリズムまたは振る舞い

1. 現在の errno を保存する。 2. DoingCommandRead が真の場合:

  • CHECK_FOR_INTERRUPTS() を呼び出し、一般的な割り込みを処理する。
  • catchupInterruptPending が真であれば ProcessCatchupInterrupt() を実行する。
  • notifyInterruptPending が真であれば ProcessNotifyInterrupt(true) を実行する。

    • 3. DoingCommandRead が偽かつ ProcDiePending が真の場合:

  • blocked が真であれば CHECK_FOR_INTERRUPTS() を呼び出す。
  • blocked が偽であれば SetLatch(MyLatch) を呼び出し、プロセスラッチを設定して終了処理を促す。

    • 4. 保存した errno を復元する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/async.c
  • src/backend/libpq/be-secure.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • CHECK_FOR_INTERRUPTS()
  • ProcessCatchupInterrupt()
  • ProcessNotifyInterrupt()
  • SetLatch()

関連データ構造

  • DoingCommandRead (グローバル変数)
  • ProcDiePending (グローバル変数)
  • catchupInterruptPending (グローバル変数)
  • notifyInterruptPending (グローバル変数)
  • MyLatch (プロセスラッチ)

失敗モードと境界条件

  • blocked 引数の値によって、プロセス終了要求（ProcDiePending）時の挙動が分岐する。読み込みがブロックされていない場合はラッチを設定して終了を待機し、ブロックされている場合は即座に割り込みチェックを行う。

保守担当者向けメモ

  • 本関数はクライアント通信の待機ループにおける安全な割り込み処理の要所である。
  • DoingCommandRead の状態管理と、ProcDiePending 時のラッチ操作の整合性に注意すること。
