TSTernaryValue

定義

  • 種別: enum
  • 定義箇所: src/include/tsearch/ts_utils.h:130

目的

全文検索におけるクエリ実行時、位置情報（positional data）の欠如により一致判定が確定できないケースを扱うための3値論理値を提供します。

シグネチャまたは形状

typedef enum
{
    TS_NO,      /* 確実に不一致 */
    TS_YES,     /* 確実に一致 */
    TS_MAYBE,   /* 位置情報不足のため一致を検証不可 */
} TSTernaryValue;

入力と出力

  • 本シンボル自体は列挙型であり、関数ではありません。
  • 主に TS_execute 関連の再帰的な一致判定処理において、戻り値として使用されます。

前提条件と不変条件

  • TS_execute において、句検索（<N> 演算子）などの位置情報を必要とする処理を行う際に使用されます。
  • GIN インデックスの GinTernaryValue と同等の値として扱われることが期待されています（src/backend/utils/adt/tsginidx.c のコメントより）。

アルゴリズムまたは振る舞い

  • TS_NO: クエリ条件を満たさないことが確定している状態。
  • TS_YES: クエリ条件を満たすことが確定している状態。
  • TS_MAYBE: 検索対象のデータに位置情報が含まれていないため、クエリの条件（特に句検索）を完全に検証できない状態。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/utils/adt/tsvector_op.c: TS_execute_recurse 等の実行関数。
  • src/backend/utils/adt/tsginidx.c: GIN インデックスの検索処理。
  • src/backend/utils/adt/tsgistidx.c: GiST インデックスの検索処理。
  • src/backend/utils/adt/tsrank.c: ランク計算処理。
  • src/backend/tsearch/wparser_def.c: パーサー定義関連。
  • **呼び出し先**:
  • 列挙型であるため、直接の呼び出し先はありません。

関連データ構造

  • TS_execute: 本列挙型を戻り値として利用する主要な実行関数。
  • ExecPhraseData: 句検索時に位置情報を保持・伝播するための構造体。

失敗モードと境界条件

  • 位置情報を持たない tsvector に対して句検索を実行した場合、判定結果が TS_MAYBE となることが境界条件となります。

保守担当者向けメモ

  • 本シンボルは、インデックスの検索結果とクエリ実行エンジンの判定結果を統合する際の重要なインターフェースです。
  • tsginidx.c 等で言及されている通り、他の3値論理型（GinTernaryValue）との値の互換性を維持する必要があります。
