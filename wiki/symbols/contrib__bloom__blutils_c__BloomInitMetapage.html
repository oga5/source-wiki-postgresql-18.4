BloomInitMetapage

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blutils.c:450

目的

Bloom インデックスのメタページ（ブロック番号 0）を新規作成し、初期化を行う。

シグネチャまたは形状

void BloomInitMetapage(Relation index, ForkNumber forknum)

入力と出力

  • **入力**:
  • index: 対象となるインデックスのリレーション。
  • forknum: 操作対象のフォーク番号（MAIN_FORKNUM や INIT_FORKNUM）。
  • **出力**: なし。副作用として、指定されたフォークのブロック 0 にメタページが書き込まれる。

前提条件と不変条件

  • インデックスの初期化フェーズで呼び出されることを想定している。
  • 同時実行される挿入処理が存在しない環境で実行される必要がある（排他ロックの取得による保護）。
  • 作成されるページは必ず BLOOM_METAPAGE_BLKNO（0）である必要がある。

アルゴリズムまたは振る舞い

1. ReadBufferExtended を使用し、指定されたフォークの新規ページ（P_NEW）を確保する。 2. 確保したバッファに対して排他ロックを取得し、ブロック番号が BLOOM_METAPAGE_BLKNO であることを検証する。 3. GenericXLogStart を呼び出し、WAL ログ記録を開始する。 4. GenericXLogRegisterBuffer を用いてバッファを登録し、BloomFillMetapage を呼び出してメタページの内容を初期化する。 5. GenericXLogFinish で WAL ログを確定させる。 6. バッファのロックを解除し、解放する。

失敗モードと境界条件

  • ReadBufferExtended が失敗した場合、例外が発生する。
  • Assert により、確保したブロック番号が 0 でない場合に異常終了する。

保守担当者向けメモ

  • 本関数はインデックス作成時のメタデータ設定に特化している。
  • BloomFillMetapage がメタページの具体的なレイアウトを決定するため、メタデータ構造を変更する場合は BloomFillMetapage を参照すること。
