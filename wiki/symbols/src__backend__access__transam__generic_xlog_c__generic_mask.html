generic_mask

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/generic_xlog.c:538

目的

汎用WAL（Write-Ahead Logging）におけるページ比較時、チェックサムやLSN、およびページ内の未使用領域（ホール）を無視して比較可能にするためのマスク処理を行う。

シグネチャまたは形状

void generic_mask(char *page, BlockNumber blkno)

入力と出力

  • page: マスク対象となるページデータのポインタ。
  • blkno: 対象ページのブロック番号（本関数内では未使用）。
  • 戻り値: なし。

前提条件と不変条件

  • page は有効なページデータへのポインタであること。
  • ページ内の pd_lower から pd_upper までの領域は、再利用可能な空き領域として扱われる。

アルゴリズムまたは振る舞い

1. mask_page_lsn_and_checksum(page) を呼び出し、ページヘッダー内のLSNおよびチェックサム値をマスクする。 2. mask_unused_space(page) を呼び出し、ページ内の未使用領域（pd_lower から pd_upper の間）をゼロクリアする。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/access/rmgrlist.h において、リソースマネージャ（RM_GENERIC_ID）のマスク関数として登録されている。
  • 呼び出し先:
  • mask_page_lsn_and_checksum
  • mask_unused_space

関連データ構造

  • BlockNumber: PostgreSQLのブロック識別子。

失敗モードと境界条件

  • page が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は、リカバリ時や差分計算時にページ内容を比較する際、物理的な不一致（LSNの差異や未使用領域のゴミデータ）を排除するために不可欠である。
  • ページレイアウトの変更や、マスク対象の追加が必要な場合は、access/bufmask.h で定義されている関連関数と併せて修正すること。
