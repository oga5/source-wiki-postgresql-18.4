ExecInitCteScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCtescan.c:174

目的

WITH 句で定義された共通テーブル式（CTE）をスキャンするための CteScanState ノードを初期化し、CTE 結果を保持する Tuplestore へのアクセス権を確立します。

シグネチャまたは形状

CteScanState *ExecInitCteScan(CteScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の CteScan プランノード。
  • estate: 実行時状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの CteScanState 構造体へのポインタ。

前提条件と不変条件

  • CteScan ノードは子プランを持たないこと（outerPlan および innerPlan が NULL であること）。
  • EXEC_FLAG_MARK フラグはサポートされない。
  • eflags には強制的に EXEC_FLAG_REWIND が付与される。
  • estate->es_param_exec_vals を介して、CTE のリーダーノードの状態が管理されていること。

アルゴリズムまたは振る舞い

1. CteScanState を割り当て、基本フィールドを初期化する。 2. estate->es_subplanstates から対応する CTE サブプランの PlanState を取得する。 3. estate->es_param_exec_vals を参照し、自身がリーダー（最初の初期化ノード）か否かを判定する。

  • **リーダーの場合**: tuplestore_begin_heap で新しい Tuplestore を作成し、自身の leader フィールドに自身をセットする。
  • **リーダーでない場合**: 既存のリーダーの Tuplestore に対して新しい読み取りポインタ（readptr）を割り当て、初期位置へリセットする。

    • 4. 式コンテキストの作成、スキャンタプルスロットの初期化、および射影情報の構築を行う。 5. node->scan.plan.qual に基づき、フィルタ条件（qual）を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c（ExecInitNode 経由での呼び出しが示唆される）。
  • **呼び出し先**: makeNode, list_nth, tuplestore_begin_heap, tuplestore_alloc_read_pointer, ExecAssignExprContext, ExecInitScanTupleSlot, ExecInitQual 等。

関連データ構造

  • CteScanState: CTE スキャンノードの実行時状態。
  • CteScan: プランノード。
  • Tuplestore: CTE 結果を格納するストレージ。
  • ParamExecData: リーダーノードの共有状態を保持するパラメータデータ。

失敗モードと境界条件

  • eflags に EXEC_FLAG_MARK が含まれている場合、Assert により停止する。
  • prmdata->execPlan が NULL でない場合、または prmdata->isnull が真の場合、Assert により停止する。
  • メモリ不足時には tuplestore がディスクへスピルする可能性がある。

保守担当者向けメモ

  • リーダーノードの選定は Param を介した動的な仕組みであるため、このノードの初期化順序に依存している。
