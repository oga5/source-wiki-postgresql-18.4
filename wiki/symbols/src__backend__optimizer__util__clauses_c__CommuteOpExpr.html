CommuteOpExpr

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/clauses.c:2147

目的

二項演算子を含む式（OpExpr）の引数を入れ替え、対応する交換演算子（commutator）に置き換えることで、式の構造を変換します。

シグネチャまたは形状

void CommuteOpExpr(OpExpr *clause)

入力と出力

  • **入力**: clause - 変換対象となる OpExpr 構造体へのポインタ。
  • **出力**: なし（引数として渡された clause を直接書き換えるインプレース操作）。

前提条件と不変条件

  • clause は二項演算子（引数が2つ）である必要があります。
  • 対象の演算子に対して、システムカタログ上で有効な交換演算子（commutator）が定義されている必要があります。

アルゴリズムまたは振る舞い

1. **検証**: 入力された clause が演算子式であり、かつ引数が2つであることを確認します。満たさない場合はエラーを発生させます。 2. **検索**: get_commutator を使用し、現在の演算子に対応する交換演算子の OID を取得します。取得できない場合はエラーを発生させます。 3. **変換**:

  • clause->opno を取得した交換演算子の OID に更新します。
  • clause->opfuncid を InvalidOid にリセットします。
  • clause->args 内の2つの引数を入れ替えます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c:5372
  • **呼び出し先**:
  • get_commutator (システムカタログアクセス)
  • elog (エラー報告)
  • linitial, lsecond (リスト操作)

関連データ構造

  • OpExpr: PostgreSQL のクエリツリーにおける演算子式を表すノード。

失敗モードと境界条件

  • **エラー**: 以下の条件で elog(ERROR, ...) が発生し、処理が中断されます。
  • 入力が二項演算子ではない、または引数が2つではない場合。
  • 対応する交換演算子がシステムカタログに見つからない場合。

保守担当者向けメモ

  • 本関数は clause を直接書き換えるため、呼び出し側で元の構造を保持する必要がある場合は、事前にコピーを作成してください。
  • opresulttype, opretset, opcollid, inputcollid は交換操作によって変更する必要がないため、そのまま維持されます。
