ExplainQuery

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:175

目的

EXPLAIN コマンドのメインエントリポイントとして、クエリの実行計画を生成し、指定されたフォーマットで整形して出力する。

シグネチャまたは形状

void ExplainQuery(ParseState *pstate, ExplainStmt *stmt, ParamListInfo params, DestReceiver *dest)

入力と出力

  • **入力**:
  • pstate: パーサーの状態。
  • stmt: 解析済みの EXPLAIN 文。
  • params: クエリ実行時に使用するパラメータリスト。
  • dest: 結果を出力するための DestReceiver。
  • **出力**: なし（dest を通じてクライアントへ結果を送信する）。

前提条件と不変条件

  • ExplainState のインデントレベルは、処理終了時に 0 である必要がある（Assert(es->indent == 0)）。
  • QueryRewrite を呼び出すため、クエリは解析済みである必要がある。

アルゴリズムまたは振る舞い

1. NewExplainState で状態を初期化し、ParseExplainOptionList でオプションを適用する。 2. JumbleQuery が有効な場合、クエリのハッシュ値を計算する。 3. post_parse_analyze_hook が定義されていれば実行する。 4. QueryRewrite を呼び出し、クエリを書き換える。 5. ExplainBeginOutput で出力の開始処理を行う。 6. 書き換えられた各クエリに対して ExplainOneQuery を呼び出し、実行計画を生成・出力する。 7. ExplainEndOutput で出力の終了処理を行う。 8. ExplainResultDesc を用いて結果のメタデータを取得し、dest を介して整形済みの文字列をクライアントへ送信する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:866 から呼び出される。
  • **呼び出し先**: ParseExplainOptionList, JumbleQuery, QueryRewrite, ExplainBeginOutput, ExplainOneQuery, ExplainSeparatePlans, ExplainEndOutput, ExplainResultDesc 等。

関連データ構造

  • ExplainState: EXPLAIN の実行状態を保持する構造体。
  • ExplainStmt: EXPLAIN コマンドの構文解析結果。
  • DestReceiver: クエリ結果の出力先を制御するインターフェース。
  • TupOutputState: タプル出力の管理用構造体。

失敗モードと境界条件

  • 書き換え後のクエリリストが空（rewritten == NIL）の場合、テキスト形式であれば「Query rewrites to nothing」と出力する。
  • es->format に応じて、テキスト形式の場合は複数行、それ以外（JSON/XML等）の場合は単一行として出力する。

保守担当者向けメモ

  • 本関数は EXPLAIN のフロー制御を担う。個別のプランノードの出力ロジックを変更する場合は、ExplainOneQuery 経由で呼び出される ExplainPrintPlan や ExplainNode を参照すること。
  • ANALYZE オプションが指定された場合、ExplainOneQuery 内で実際にクエリが実行されるため、副作用を伴う点に注意が必要である。
