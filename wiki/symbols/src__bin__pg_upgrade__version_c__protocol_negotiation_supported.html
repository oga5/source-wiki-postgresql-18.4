protocol_negotiation_supported

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/version.c:35

目的

旧サーバー（ソースクラスター）が NegotiateProtocolVersion をサポートしているかどうかを判定します。

シグネチャまたは形状

bool protocol_negotiation_supported(const ClusterInfo *cluster);

入力と出力

  • **入力**: ClusterInfo 構造体へのポインタ（旧クラスターの情報）。
  • **出力**: bool 型。サポートされている場合は true、それ以外は false を返します。

前提条件と不変条件

  • ClusterInfo 構造体にはメジャーバージョン情報が含まれている必要があります。
  • 2018年2月のパッチ適用済みサーバー（9.3.21, 9.4.16, 9.5.11, 9.6.7, 10.2）も NegotiateProtocolVersion をサポートしていますが、本関数はメジャーバージョンのみを参照するため、これらパッチ適用済みサーバーに対しては false を返す保守的な設計となっています。

アルゴリズムまたは振る舞い

  • GET_MAJOR_VERSION(cluster->major_version) を使用して、メジャーバージョンが 1100（PostgreSQL 11）以上であるかを判定します。
  • 11未満のバージョンは、プロトコルネゴシエーションをサポートしていないものとみなします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/dump.c
  • src/bin/pg_upgrade/server.c
  • src/bin/pg_upgrade/task.c
  • **呼び出し先**:
  • 本関数はマクロ GET_MAJOR_VERSION を使用しますが、関数呼び出しは行いません。

関連データ構造

  • ClusterInfo: pg_upgrade.h で定義される、クラスターのメタデータ（メジャーバージョン等）を保持する構造体。

失敗モードと境界条件

  • メジャーバージョンが 11 未満のサーバーに対しては、パッチ適用状況に関わらず一律で false を返します。これは、古いサーバーでは新しいプロトコルを利用できないため、機能上の損失がないという前提に基づいています。

保守担当者向けメモ

  • 本関数はメジャーバージョンのみで判定を行っています。将来的にパッチ適用状況を詳細に判定する必要が生じた場合は、判定ロジックの拡張を検討してください。
