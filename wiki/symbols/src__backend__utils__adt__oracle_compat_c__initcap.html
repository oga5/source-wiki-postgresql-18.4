initcap

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:113

目的

入力文字列の各単語の先頭文字を大文字に、残りの文字を小文字に変換した新しい文字列を生成します。Oracle データベースとの互換性を目的とした関数です。

シグネチャまたは形状

Datum initcap(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る text 型の文字列。
  • **出力**: 変換後の文字列を格納した text 型の Datum。

前提条件と不変条件

  • 入力文字列は PostgreSQL の text 型として有効である必要があります。
  • 内部的に PG_GET_COLLATION() を使用しており、現在の照合順序（Collation）設定に依存した変換が行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP(0) を使用して入力文字列を取得します。 2. str_initcap を呼び出し、文字列データ、サイズ、および現在の照合順序を渡して変換を実行します。 3. str_initcap が返した C 文字列を cstring_to_text で PostgreSQL の text 型に変換します。 4. str_initcap が確保したメモリを pfree で解放します。 5. PG_RETURN_TEXT_P を使用して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます。src/bin/pg_upgrade/check.c 等のシステムチェック処理で参照されています。
  • **呼び出し先**:
  • str_initcap (コアロジック)
  • cstring_to_text (型変換)
  • pfree (メモリ管理)

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • 入力文字列が NULL の場合、PG_GETARG_TEXT_PP の仕様に従い処理されます。
  • メモリ不足時には palloc (内部で使用) によりエラーが発生する可能性があります。
  • マルチバイト文字を含む場合、str_initcap の実装に基づき、文字境界を正しく維持して変換されます。

保守担当者向けメモ

  • 本関数は src/backend/utils/formatting.c に定義されている str_initcap のラッパーです。ロジックの変更が必要な場合は、まず formatting.c 側の実装を確認してください。
  • 照合順序に依存するため、ロケール設定によって変換結果が異なる場合があります。
