ParallelSlotsTerminate

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:478

目的

ParallelSlotArray が管理するすべてのデータベース接続をクローズし、リソースを解放します。

シグネチャまたは形状

void ParallelSlotsTerminate(ParallelSlotArray *sa)

入力と出力

  • **入力**: sa - 終了対象の ParallelSlotArray 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • sa は ParallelSlotsSetup 等によって適切に初期化されている必要があります。
  • 内部の各スロットにおいて、接続が存在しない（NULL）場合はスキップされます。

アルゴリズムまたは振る舞い

1. sa->numslots で指定された回数分、スロット配列を反復処理します。 2. 各スロットの connection フィールドを確認します。 3. 接続が確立されている（conn != NULL）場合、disconnectDatabase(conn) を呼び出して接続を終了します。 4. 接続が NULL のスロットは無視して処理を継続します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c
  • **呼び出し先**:
  • disconnectDatabase（src/fe_utils/connect_utils.c で定義）

関連データ構造

  • ParallelSlotArray: 並列接続を管理する配列構造体。
  • ParallelSlot: 個別の接続状態を保持する構造体。

失敗モードと境界条件

  • sa が NULL の場合、あるいは sa->slots が不正な場合、セグメンテーション違反が発生する可能性があります。
  • 接続が既に閉じられている場合や NULL の場合は安全にスキップされます。

保守担当者向けメモ

  • 本関数は接続のクローズのみを行い、ParallelSlotArray 自体のメモリ解放は行いません。配列のメモリ解放が必要な場合は、別途適切な処理を行ってください。
