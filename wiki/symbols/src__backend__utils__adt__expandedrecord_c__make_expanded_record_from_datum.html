make_expanded_record_from_datum

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandedrecord.c:579

目的

既存の複合型 Datum から、メモリ上で効率的に操作可能な「展開されたレコード（Expanded Record）」オブジェクトを生成する。

シグネチャまたは形状

Datum make_expanded_record_from_datum(Datum recorddatum, MemoryContext parentcontext)

入力と出力

  • **入力**:
  • recorddatum: 展開対象となる複合型の Datum。
  • parentcontext: 新規オブジェクトのメモリコンテキストを作成するための親コンテキスト。
  • **出力**:
  • 展開されたレコードへの読み書き可能な Datum（EOHPGetRWDatum により取得）。

前提条件と不変条件

  • 入力 recorddatum は複合型である必要がある。
  • HeapTupleHeaderHasExternal(tuphdr) が真である（外部ストレージに依存している）状態は許可されない。
  • 生成されたオブジェクトは ER_MAGIC で識別され、ER_FLAG_FVALUE_VALID フラグが設定される。

アルゴリズムまたは振る舞い

1. **メモリ確保**: parentcontext の下に専用のメモリコンテキストを作成し、ExpandedRecordHeader を初期化する。 2. **データコピー**: 入力 recorddatum を detoast し、heap_copytuple を用いてプライベートなメモリコンテキストへコピーする。 3. **メタデータ設定**: コピーしたタプルから型ID (er_decltypeid, er_typeid) および型修飾子 (er_typmod) を抽出してヘッダーに格納する。 4. **フラット化状態の保持**: コピーしたタプルへのポインタ (fvalue) とその範囲 (fstartptr, fendptr) を記録する。 5. **遅延評価**: タプルの分解（deconstruction）や TupleDesc の取得は、必要になるまで行わない。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/expandedrecord.c:939 (直接の呼び出しを確認)
  • **呼び出し先**:
  • AllocSetContextCreate, MemoryContextAllocZero, EOH_init_header, DatumGetHeapTupleHeader, HeapTupleHeaderGetDatumLength, ItemPointerSetInvalid, MemoryContextSwitchTo, heap_copytuple, HeapTupleHeaderGetTypeId, HeapTupleHeaderGetTypMod, HeapTupleHeaderHasExternal, EOHPGetRWDatum

関連データ構造

  • ExpandedRecordHeader: 展開されたレコードのメタデータと状態を保持する構造体。
  • HeapTupleData: 物理的なタプル表現。
  • ER_methods: 展開オブジェクトのプロトコルメソッドテーブル。

失敗モードと境界条件

  • detoast 処理が必要な場合、呼び出し元のコンテキストでメモリリークが発生する可能性がある（ソースコード上の注釈による）。
  • 外部ストレージ（TOAST）を持つタプルが渡された場合、Assert により停止する可能性がある。

保守担当者向けメモ

  • 本関数はタプルの分解を遅延評価するため、生成直後の ExpandedRecordHeader は分解済みフィールド情報を持たない。
  • ER_FLAG_HAVE_EXTERNAL は設定されない前提で動作する。
