FreeWaitEventSetAfterFork

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:523

目的

プロセスが fork() された直後に、子プロセス側で不要となった WaitEventSet 構造体および関連する OS リソースを解放する。

シグネチャまたは形状

void FreeWaitEventSetAfterFork(WaitEventSet *set)

入力と出力

  • **入力**: set - 解放対象の WaitEventSet 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • set は有効な WaitEventSet 構造体を指している必要がある。
  • 本関数は fork() 直後の子プロセスから呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. コンパイル時のプラットフォーム設定（WAIT_USE_EPOLL 等）に基づき、OS 固有のリソースをクリーンアップする。

  • WAIT_USE_EPOLL が定義されている場合: set->epoll_fd を close() し、ReleaseExternalFD() を呼び出す。
  • WAIT_USE_KQUEUE が定義されている場合: ReleaseExternalFD() を呼び出す（kqueue は通常子プロセスに継承されないため）。

    • 2. pfree(set) を呼び出し、WaitEventSet 構造体自体が使用していたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c:1861
  • **呼び出し先**: close() (OS), ReleaseExternalFD(), pfree()

関連データ構造

  • WaitEventSet: 待機対象のイベント群を保持するメイン構造体。

失敗モードと境界条件

  • set が NULL の場合の挙動は定義されていない。
  • ReleaseExternalFD() の内部状態に依存する。

保守担当者向けメモ

  • 本関数は fork() 後に親プロセスのリソースを子プロセスが引き継がないようにするための安全策である。
  • プラットフォーム固有の待機メカニズム（epoll/kqueue）が追加・変更された場合は、本関数の条件分岐も更新する必要がある。
