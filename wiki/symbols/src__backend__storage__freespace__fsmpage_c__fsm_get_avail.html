fsm_get_avail

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/fsmpage.c:121

目的

FSM（空き領域マップ）ページ内の指定されたスロットにおける現在の空き容量を取得します。

シグネチャまたは形状

uint8 fsm_get_avail(Page page, int slot)

入力と出力

  • **入力**:
  • page: 操作対象の Page 構造体。
  • slot: 取得対象のリーフノードインデックス。
  • **出力**:
  • 指定されたスロットの空き容量を示す uint8 型の値。

前提条件と不変条件

  • slot は LeafNodesPerPage 未満である必要があります（Assert により検証）。
  • ページは FSMPage 構造として初期化されている必要があります。

アルゴリズムまたは振る舞い

1. PageGetContents(page) を使用して、ページの内容を FSMPage 構造体としてキャストします。 2. fp_nodes 配列内のリーフノード領域（NonLeafNodesPerPage オフセット以降）から、指定された slot に対応する値を読み取ります。 3. 読み取った値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/freespace/freespace.c:267
  • src/backend/storage/freespace/freespace.c:903
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（マクロ PageGetContents を使用）。

関連データ構造

  • FSMPage: FSM ページの内部構造を定義するデータ構造。
  • NonLeafNodesPerPage: リーフノードの開始位置を決定するための定数。

失敗モードと境界条件

  • slot が LeafNodesPerPage 以上の場合、アサーション違反によりプロセスが停止する可能性があります（デバッグビルド時）。
  • ページが有効な FSM ページでない場合、メモリ上の不正な位置を参照する可能性があります。

保守担当者向けメモ

  • 本関数は単一バイトの読み取りを行うため、バッファロックなしで呼び出し可能です。
  • fp_nodes 配列のインデックス計算において、NonLeafNodesPerPage を加算することでリーフノードへアクセスする設計となっています。
