ModifyTableContext

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeModifyTable.c:86

目的

ModifyTable ノードによるデータ操作（INSERT, UPDATE, DELETE, MERGE）実行中に、状態管理、トランザクション情報、およびタプル処理に必要なコンテキストを保持する。

シグネチャまたは形状

typedef struct ModifyTableContext
{
    ModifyTableState *mtstate;
    EPQState   *epqstate;
    EState     *estate;
    TupleTableSlot *planSlot;
    TM_FailureData tmfd;
    TupleTableSlot *cpDeletedSlot;
    TupleTableSlot *cpUpdateReturningSlot;
} ModifyTableContext;

入力と出力

  • **入力**: ExecModifyTable 等の関数において、実行状態を共有するための引数として渡される。
  • **出力**: 構造体内の各フィールド（planSlot や cpDeletedSlot 等）を通じて、操作中のタプルデータや同時実行制御の結果が更新・参照される。

前提条件と不変条件

  • mtstate, epqstate, estate は、実行ノードの初期化時に適切に設定されている必要がある。
  • planSlot は、サブプランから供給されるタプルを保持し、格納されない「ジャンク」カラムへのアクセスに使用される。

アルゴリズムまたは振る舞い

  • **状態保持**: 実行中のノード状態（mtstate）や同時実行制御状態（epqstate）を保持し、各操作関数へ伝播させる。
  • **パーティション対応**: クロスパーティション更新（cpDeletedSlot, cpUpdateReturningSlot）が発生した際、ルートテーブルの記述子に変換されたタプルを保持し、RETURNING 句の処理をサポートする。
  • **競合管理**: tmfd を通じて、更新または削除対象のタプルに対する同時実行時の競合情報を保持する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecInsert, ExecDelete, ExecMerge, ExecProcessReturning, ExecOnConflictUpdate 等の nodeModifyTable.c 内の関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しの引数として渡されることで、各操作ロジックが状態を参照する。

関連データ構造

  • ModifyTableState: ノード全体の実行状態。
  • EPQState: EvalPlanQual（同時実行制御）の状態。
  • TupleTableSlot: タプルデータを保持するスロット。
  • TM_FailureData: 同時更新失敗時の詳細情報。

失敗モードと境界条件

  • 同時更新が発生した場合、tmfd に情報が格納され、呼び出し側のロジックで再試行やエラー処理が判断される。
  • パーティションルーティングを伴う操作では、cpDeletedSlot 等が適切に初期化されていない場合、RETURNING 句の処理で不整合が生じる可能性がある。

保守担当者向けメモ

  • 本構造体は nodeModifyTable.c 内の多くの関数で共有されるため、フィールドの追加や変更は、関連する ExecInsert や ExecDelete 等の全操作ロジックに影響を与える。
  • クロスパーティション更新時の RETURNING 処理は複雑であるため、cpDeletedSlot 等のライフサイクル管理には注意が必要。
