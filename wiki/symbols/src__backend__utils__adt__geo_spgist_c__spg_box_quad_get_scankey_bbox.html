spg_box_quad_get_scankey_bbox

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_spgist.c:530

目的

SP-GiSTインデックスの検索処理において、ScanKey から幾何学的な境界ボックス（BOX）を抽出します。

シグネチャまたは形状

BOX *spg_box_quad_get_scankey_bbox(ScanKey sk, bool *recheck)

入力と出力

  • **入力**:
  • sk: 検索条件を保持する ScanKey 構造体。
  • recheck: 検索結果の再チェックが必要かを示すフラグへのポインタ（POLYGON 型の場合に使用）。
  • **出力**:
  • 抽出された BOX 構造体へのポインタ。

前提条件と不変条件

  • sk->sk_subtype は BOXOID または POLYGONOID である必要があります。それ以外の場合はエラーが発生します。
  • POLYGON 型の場合、境界ボックス（boundbox）が有効であると仮定されます。

アルゴリズムまたは振る舞い

1. sk->sk_subtype に基づき、以下の処理を行います。

  • BOXOID: DatumGetBoxP を使用して sk->sk_argument から直接 BOX を取得します。
  • POLYGONOID: DatumGetPolygonP を使用して POLYGON を取得し、その boundbox メンバを返します。この際、is_bounding_box_test_exact が偽を返す場合は、recheck フラグを真に設定します。

    • 2. 未知の型が指定された場合は elog(ERROR) を発行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/geo_spgist.c:615
  • src/backend/utils/adt/geo_spgist.c:764
  • **呼び出し先**:
  • DatumGetBoxP (マクロ)
  • DatumGetPolygonP (マクロ)
  • is_bounding_box_test_exact
  • elog

関連データ構造

  • BOX: 矩形を表現する構造体。
  • ScanKey: インデックス検索条件を保持する構造体。
  • POLYGON: 多角形を表現する構造体。

失敗モードと境界条件

  • sk->sk_subtype がサポート対象外（BOXOID または POLYGONOID 以外）の場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は POLYGON の境界ボックスによる近似検索をサポートしており、必要に応じて recheck フラグを制御します。変更を加える際は、is_bounding_box_test_exact の判定ロジックとの整合性に注意してください。
