EA_get_flat_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_expanded.c:232

目的

展開済み配列（ExpandedArrayHeader）をフラットな ArrayType 形式にシリアライズするために必要なバイトサイズを算出します。

シグネチャまたは形状

static Size EA_get_flat_size(ExpandedObjectHeader *eohptr)

入力と出力

  • **入力**: ExpandedObjectHeader *eohptr（展開済み配列オブジェクトへのポインタ）
  • **出力**: Size（フラット化に必要なバイト数）

前提条件と不変条件

  • eohptr は ExpandedArrayHeader としてキャスト可能であること。
  • eah->ea_magic == EA_MAGIC が真であること（整合性チェック）。
  • 配列の要素数（nelems）と次元情報（dims）が整合していること。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: 既にフラット化された値（fvalue）が存在する場合はそのサイズを返し、計算済みの flat_size があればそれを返します。 2. **サイズ計算**: 展開された要素（dvalues）とNULLフラグ（dnulls）を走査し、各要素のデータ長とアライメントを考慮して合計サイズを算出します。 3. **オーバーヘッド加算**: 配列の次元数とNULLビットマップの有無に基づき、ARR_OVERHEAD_WITHNULLS または ARR_OVERHEAD_NONULLS を加算します。 4. **キャッシュ保存**: 計算結果を eah->flat_size に格納し、次回以降の呼び出しに備えます。 5. **エラー処理**: 計算中に MaxAllocSize を超えるサイズが要求された場合、エラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**: ExpandedObjectMethods 構造体を通じて、展開済みオブジェクトのシリアライズ処理から呼び出されます。具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**: ArrayGetNItems、att_addlength_datum、att_align_nominal、AllocSizeIsValid などのマクロおよび関数。

関連データ構造

  • ExpandedArrayHeader: 展開済み配列のメイン構造体。
  • ExpandedObjectHeader: 展開済みオブジェクトの基底ヘッダー。
  • ArrayType: フラットな配列の内部表現。

失敗モードと境界条件

  • **メモリ制限**: 計算されたサイズが MaxAllocSize を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラー終了します。
  • **整合性エラー**: EA_MAGIC が一致しない場合、アサーション違反となります。

保守担当者向けメモ

  • 本関数は ExpandedObjectMethods の一部として定義されています。
  • dnulls が存在する場合、実際の配列内にNULLが含まれていなくても、NULLビットマップ分のオーバーヘッドがサイズ計算に含まれる仕様です。
