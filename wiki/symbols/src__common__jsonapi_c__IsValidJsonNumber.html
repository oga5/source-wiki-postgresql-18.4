IsValidJsonNumber

定義

  • 種別: function
  • 定義箇所: src/common/jsonapi.c:338

目的

与えられた文字列がJSONの数値として妥当であるかを検証します。

シグネチャまたは形状

bool IsValidJsonNumber(const char *str, size_t len)

入力と出力

  • **入力**:
  • str: 検証対象の文字列へのポインタ。
  • len: 文字列の長さ。
  • **出力**:
  • bool: 文字列全体が有効なJSON数値として解析可能な場合は true、それ以外は false を返します。

前提条件と不変条件

  • len が 0 以下の場合、常に false を返します。
  • 内部で JsonLexContext を一時的に生成して利用します。
  • json_lex_number 関数が期待する通り、先頭のマイナス記号（-）は処理の過程で考慮されます。

アルゴリズムまたは振る舞い

1. len <= 0 の場合は直ちに false を返します。 2. JsonLexContext のダミーインスタンスを初期化します。 3. 文字列の先頭が - の場合、その文字をスキップして残りの部分を数値として解析対象にします。 4. json_lex_number を呼び出し、数値の解析を行います。 5. 解析結果として numeric_error が発生しておらず、かつ解析された数値の長さ（total_len）が入力文字列の長さと完全に一致する場合にのみ true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_io.c（hstore_io.c:1379 および 1527 での利用を確認）
  • **呼び出し先**:
  • json_lex_number（内部で呼び出し）

関連データ構造

  • JsonLexContext: 解析状態を保持するために一時的に使用される構造体。

失敗モードと境界条件

  • 空文字列または長さ 0 の入力は false となります。
  • 数値の形式がJSON仕様に適合しない場合（例: 不正な指数表記、先頭の不要なゼロなど）、json_lex_number がエラーを検出し、結果として false が返されます。
  • 文字列の一部のみが数値として有効であっても、文字列全体が数値でなければ false となります。

保守担当者向けメモ

  • json_lex_number に渡すために const char * の定数性をキャストで外す処理が含まれています。
  • 本関数は src/include/common/jsonapi.h で公開されており、外部モジュールからも利用可能です。
