ArraySortCachedInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:54

目的

配列のソート処理において、型情報や比較演算子の検索コストを削減するためのキャッシュ構造体。

シグネチャまたは形状

typedef struct ArraySortCachedInfo
{
	ArrayMetaState array_meta;	/* array_create_iterator用メタデータ */
	Oid			elem_lt_opr;	/* 要素型の"<"演算子 */
	Oid			elem_gt_opr;	/* 要素型の">"演算子 */
	Oid			array_type;		/* 配列型のpg_type OID */
} ArraySortCachedInfo;

入力と出力

本構造体は関数呼び出しの fn_extra に格納され、後続の呼び出しで再利用されます。直接的な入力/出力関数ではありません。

前提条件と不変条件

  • fn_extra を介してキャッシュされるため、同一クエリ内での繰り返し実行時に有効です。
  • ArrayMetaState は array_create_iterator で使用可能な状態である必要があります。

アルゴリズムまたは振る舞い

配列ソート関数において、カタログから演算子や型情報を毎回検索する代わりに、初回実行時に本構造体を初期化して fn_extra に保存します。以降の呼び出しでは、このキャッシュを参照することで処理を最適化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_userfuncs.c 内のソート関連関数（cache_info 変数として参照）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ArrayMetaState: 配列反復処理のためのメタデータ。
  • Oid: PostgreSQLのシステムカタログ識別子。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc 失敗時）に例外が発生する可能性があります。
  • キャッシュ初期化時に指定された型がソートをサポートしていない場合、演算子検索が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は fn_extra を利用した最適化戦略の一部です。変更を加える際は、array_userfuncs.c 内の fn_extra 初期化ロジック（1919行目付近）との整合性に注意してください。
