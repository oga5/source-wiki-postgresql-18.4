registered_buffer

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xloginsert.c:68

目的

WALレコード構築時に、変更対象となる各ブロックのメタデータ、状態、および関連データを一時的に保持するために使用されます。

シグネチャまたは形状

typedef struct {
    bool        in_use;
    uint8       flags;
    RelFileLocator rlocator;
    ForkNumber  forkno;
    BlockNumber block;
    const PageData *page;
    uint32      rdata_len;
    XLogRecData *rdata_head;
    XLogRecData *rdata_tail;
    XLogRecData bkp_rdatas[2];
    char        compressed_page[COMPRESS_BUFSIZE];
} registered_buffer;

入力と出力

本構造体は関数への引数ではなく、xloginsert.c 内の静的配列 registered_buffers の要素として管理されます。XLogRegisterBuffer 等の関数を通じて各フィールドが設定され、XLogRecordAssemble で読み取られてWALレコードの構築に使用されます。

前提条件と不変条件

  • in_use が true の場合のみ、当該スロットは有効な登録情報を含みます。
  • XLogRegisterBuffer を呼び出す前に、対象のバッファが排他ロックされている必要があります。
  • compressed_page は、wal_compression が有効な場合にフルページイメージ（FPI）の圧縮結果を格納するために使用されます。

アルゴリズムまたは振る舞い

1. **登録**: XLogRegisterBuffer 等により、対象ブロックの識別子（rlocator, forkno, block）とデータがスロットに記録されます。 2. **連結**: rdata_head および rdata_tail を用いて、当該ブロックに関連するデータチャンクのチェーンを管理します。 3. **組み立て**: XLogRecordAssemble 実行時に、各スロットの情報を走査し、必要に応じて compressed_page への圧縮処理を行い、最終的なWALレコードのヘッダーとデータ部を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: XLogRegisterBuffer, XLogRegisterBufData, XLogRecordAssemble 等の xloginsert.c 内の関数が本構造体を操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • XLogRecData: レコードデータのチェーンを構成する構造体。
  • RelFileLocator: 変更対象の物理ファイルを識別する構造体。
  • registered_buffers: 本構造体を要素とする、レコード構築用の静的配列。

失敗モードと境界条件

  • COMPRESS_BUFSIZE を超える圧縮データは保持できません。
  • XLogEnsureRecordSpace を事前に呼び出さない場合、登録可能なバッファ数（max_registered_buffers）の上限に達するとエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は xloginsert.c 内の作業領域の一部です。XLogResetInsertion が呼び出されると、in_use フラグがリセットされ、スロットが解放されます。
  • bkp_rdatas は XLogRecordAssemble 内での一時的な参照保持に特化したフィールドであり、外部からの直接操作は避けてください。
