WALDumpOpenSegment

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/pg_waldump.c:337

目的

XLogReader が WAL セグメントファイルを読み込む際に、指定されたセグメント番号に対応するファイルを開く。

シグネチャまたは形状

static void WALDumpOpenSegment(XLogReaderState *state, XLogSegNo nextSegNo, TimeLineID *tli_p);

入力と出力

  • **入力**:
  • state: XLogReaderState 構造体へのポインタ。ファイルディスクリプタの格納先として使用される。
  • nextSegNo: 開くべき WAL セグメント番号。
  • tli_p: タイムライン ID (TimeLineID) へのポインタ。
  • **出力**: なし（成功時は state->seg.ws_file にファイルディスクリプタが設定される）。

前提条件と不変条件

  • state->segcxt.ws_dir に有効なディレクトリパスが設定されていること。
  • state->segcxt.ws_segsize に正しいセグメントサイズが設定されていること。

アルゴリズムまたは振る舞い

1. XLogFileName を使用して、指定されたタイムラインとセグメント番号からファイル名 fname を生成する。 2. open_file_in_directory を呼び出し、対象ディレクトリからファイルを開く。 3. **フォローモードの考慮**: ファイルが存在しない場合（errno == ENOENT）、最大 10 回（合計 5 秒間）まで 500ms 間隔で再試行を行う。 4. ファイルが正常に開けた場合、そのディスクリプタを state->seg.ws_file に格納して終了する。 5. 再試行してもファイルが見つからない場合、またはその他のエラーが発生した場合は pg_fatal を呼び出し、プログラムを異常終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_waldump/pg_waldump.c 内の XLogReader 初期化処理にて、XLogReaderState の .segment_open コールバックとして登録されている。
  • **呼び出し先**:
  • XLogFileName
  • open_file_in_directory
  • pg_usleep
  • pg_fatal

関連データ構造

  • XLogReaderState: WAL 読み込みの状態を保持する構造体。
  • XLogSegNo: WAL セグメント番号を表現する型。
  • TimeLineID: タイムラインを識別する型。

失敗モードと境界条件

  • **ファイル不在**: 指定されたセグメントファイルがディレクトリ内に存在しない場合、5 秒間の待機後に pg_fatal で終了する。
  • **権限エラー・その他**: open_file_in_directory が ENOENT 以外のエラーを返した場合、即座に pg_fatal で終了する。

保守担当者向けメモ

  • 本関数は pg_waldump のフォローモード（継続的な WAL 読み込み）において、新しいセグメントが生成されるまでのラグを吸収する重要な役割を担っている。
  • pg_usleep によるポーリング間隔や再試行回数は、環境やサーバーの負荷に応じて調整が必要になる可能性がある。
