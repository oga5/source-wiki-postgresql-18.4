SecLabelItem

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.c:88

目的

データベースオブジェクトに適用されたセキュリティラベル情報を保持し、ダンプ処理中に管理するための構造体です。

シグネチャまたは形状

typedef struct
{
	const char *provider;		/* ラベルプロバイダ名 */
	const char *label;			/* オブジェクトのセキュリティラベル */
	Oid			classoid;		/* オブジェクトのカタログOID */
	Oid			objoid;			/* オブジェクトのOID */
	int			objsubid;		/* サブオブジェクトID（テーブル列番号など） */
} SecLabelItem;

入力と出力

本構造体は、システムカタログから取得したセキュリティラベルの属性を格納するコンテナとして機能します。関数 findSecLabels を通じて、特定のオブジェクトに関連付けられたラベル情報のリストを取得するために使用されます。

前提条件と不変条件

  • classoid および objoid は、PostgreSQL システムカタログ内の有効なオブジェクトを指している必要があります。
  • objsubid は、対象オブジェクトがテーブル列などのサブオブジェクトである場合にのみ意味を持ちます。

アルゴリズムまたは振る舞い

  • pg_dump 実行時、システムカタログからセキュリティラベル情報を読み取る際にこの構造体の配列が割り当てられます。
  • findSecLabels 関数は、指定されたオブジェクト（classoid, objoid）に対応する SecLabelItem のリストを検索・抽出する役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_dump.c 内のダンプ処理ロジックにおいて、オブジェクトのメタデータ収集時に呼び出されます。
  • **呼び出し先**: pg_malloc を使用してメモリを確保する処理が含まれます。
  • 詳細な呼び出し関係については、ソースコードの解析範囲内では特定の関数呼び出しの連鎖までは特定されていません。

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。
  • pg_dump.c 内の静的変数 seclabels が、この構造体の配列を保持するために使用されることがあります。

失敗モードと境界条件

  • メモリ不足時: pg_malloc によるメモリ確保失敗が発生する可能性があります。
  • 検索失敗: findSecLabels が該当するラベルを見つけられない場合、戻り値やポインタの状態に注意が必要です。

保守担当者向けメモ

  • 本構造体は pg_dump におけるセキュリティラベルの永続化をサポートします。変更を加える際は、pg_dump.h での定義との整合性に注意してください。
  • objsubid の値が 0 の場合は、オブジェクト全体を指すものとして扱われます。
