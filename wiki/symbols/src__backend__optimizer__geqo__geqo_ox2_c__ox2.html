ox2

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_ox2.c:48

目的

遺伝的クエリ最適化（GEQO）において、2つの親個体（ツアー）から「OX2（Order Crossover 2）」アルゴリズムを用いて新しい子孫個体を生成する。

シグネチャまたは形状

void ox2(PlannerInfo *root, Gene *tour1, Gene *tour2, Gene *offspring, int num_gene, City *city_table)

入力と出力

  • **入力**:
  • root: クエリ最適化のコンテキスト情報。
  • tour1, tour2: 交叉の元となる親個体（遺伝子の配列）。
  • num_gene: 遺伝子の総数。
  • city_table: 交叉処理用の一時作業領域。
  • **出力**:
  • offspring: 生成された子孫個体が格納される配列。

前提条件と不変条件

  • city_table は、少なくとも num_gene 以上のサイズを持つ必要がある。
  • 本関数は単一スレッドでの実行を前提としている（city_table を共有作業領域として使用するため）。
  • 遺伝子の順序は、巡回セールスマン問題（TSP）の解候補として妥当である必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: city_table の used フラグをリセットし、select_list を初期化する。 2. **選択**: tour1 からランダムに num_positions 個（全遺伝子の1/3〜2/3）の遺伝子を選択し、city_table に記録する。 3. **圧縮**: 選択された遺伝子を select_list 内で連続した位置に詰め直す。 4. **生成**: tour2 を走査し、その位置の遺伝子が tour1 から選択済みであれば select_list の値で置き換え、未選択であれば tour2 の値をそのまま採用して offspring を構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_main.c:216
  • **呼び出し先**:
  • geqo_randint（乱数生成）
  • 直接的な関数呼び出しの詳細はソース抜粋からは不明だが、GEQOの交叉演算インターフェースの一部として機能する。

関連データ構造

  • Gene: 遺伝子を表す型。
  • City: 交叉プロセス中に遺伝子の選択状態や順序を追跡するための構造体。

失敗モードと境界条件

  • num_gene が極端に小さい場合、geqo_randint の範囲指定により予期せぬ挙動が発生する可能性がある。
  • city_table が適切に初期化されていない場合、メモリ破壊や不正な遺伝子生成を招く。

保守担当者向けメモ

  • 本実装は #if defined(OX2) で囲まれており、コンパイル時に有効化されている必要がある。
  • city_table は一時的な作業領域として使用されるため、再帰呼び出しや並列実行は避けること。
  • 遺伝子の「位置」よりも「順序」の保持を重視するアルゴリズムである。
