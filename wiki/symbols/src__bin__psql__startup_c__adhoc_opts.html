adhoc_opts

定義

  • 種別: struct
  • 定義箇所: src/bin/psql/startup.c:66

目的

psql 起動時のコマンドライン引数から得られる一時的な設定値および実行アクションを保持するための構造体です。

シグネチャまたは形状

struct adhoc_opts
{
    char       *dbname;
    char       *host;
    char       *port;
    char       *username;
    char       *logfilename;
    bool        no_readline;
    bool        no_psqlrc;
    bool        single_txn;
    bool        list_dbs;
    SimpleActionList actions;
};

入力と出力

  • **入力**: parse_psql_options 関数において、コマンドライン引数（argc, argv）を解析した結果が本構造体の各フィールドに格納されます。
  • **出力**: 構造体に格納された値は、接続パラメータの決定や、psql の動作モード（スクリプト実行やデータベース一覧表示など）の制御に使用されます。

前提条件と不変条件

  • actions フィールドは SimpleActionList 型であり、実行すべき一連のアクションを保持します。
  • 構造体の各ポインタフィールド（dbname 等）は、解析結果に応じて動的に割り当てられるか、NULL となります。

アルゴリズムまたは振る舞い

1. main 関数内で本構造体のインスタンスが宣言されます。 2. parse_psql_options にポインタが渡され、コマンドライン引数に基づいて各フィールドが設定されます。 3. 解析後、これらの値はデータベース接続の確立や、psql の初期化プロセスにおいて参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/startup.c 内の main 関数および parse_psql_options 関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SimpleActionList: 実行すべきアクションのリストを管理するために使用されます。

失敗モードと境界条件

  • 引数解析時にメモリ不足が発生した場合、getopt_long の挙動やその後の処理に依存します。
  • 必須の接続情報が指定されていない場合、各フィールドはデフォルト値または NULL を保持したままとなります。

保守担当者向けメモ

  • 本構造体は psql の起動時のみ使用される一時的な設定保持用です。実行中の永続的な設定管理には pset（PsqlSettings）が使用されるため、混同しないよう注意してください。
  • 新しいコマンドラインオプションを追加する際は、本構造体にフィールドを追加し、parse_psql_options 内で解析ロジックを更新する必要があります。
