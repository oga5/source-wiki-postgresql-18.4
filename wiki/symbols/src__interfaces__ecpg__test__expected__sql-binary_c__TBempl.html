TBempl

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-binary.c:32

目的

ECPG（Embedded SQL for C）のテストにおいて、データベースの empl テーブルの行構造を模倣するためのデータ構造です。

シグネチャまたは形状

struct TBempl {
    long idnum;
    char name[21];
    short accs;
    char byte[20];
};

入力と出力

  • 本構造体は、ECPG プリプロセッサによって生成される SQL 文のホスト変数として使用されます。
  • データベースからフェッチされたデータが各メンバに格納され、C 言語側で参照可能となります。

前提条件と不変条件

  • ecpg プリプロセッサの declare section 内で定義されることを前提としています。
  • メンバ byte はバイナリデータ（bytea 型）の保持に使用されます。

アルゴリズムまたは振る舞い

  • データベースのテーブル行と C 言語のメモリレイアウトをマッピングします。
  • ECPG ライブラリが SQL 文を実行する際、この構造体の各メンバのアドレスがバインド変数として渡されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/sql-binary.c:61 にてインスタンス化されています。
  • 呼び出し元: src/interfaces/ecpg/test/expected/preproc-type.c:94 にてインスタンス化されています。
  • 呼び出し先: ECPG ライブラリの内部関数（ECPGt_long, ECPGt_char, ECPGt_short 等）が、この構造体の各メンバを操作します。

関連データ構造

  • sqlca: SQL 通信領域。本構造体を用いた SQL 操作の結果確認に使用されます。

失敗モードと境界条件

  • name メンバ（21バイト）および byte メンバ（20バイト）のサイズを超えるデータを格納しようとした場合、バッファオーバーフローのリスクがあります。
  • accs メンバはプラットフォーム間のエンディアン依存性があるため、テスト実行時には出力がスキップされる場合があります。

保守担当者向けメモ

  • 本構造体は binary.pgc から生成されたコードの一部です。直接編集せず、元の .pgc ファイルを修正してください。
  • byte メンバにバイナリデータを扱う際は、メモリの動的割り当てと解放の整合性に注意してください。
