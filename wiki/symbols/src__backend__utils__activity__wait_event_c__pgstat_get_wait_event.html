pgstat_get_wait_event

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/wait_event.c:430

目的

バックエンドプロセスの待機状態を示す整数値（wait_event_info）を、人間が判読可能な待機イベント名の文字列に変換する。

シグネチャまたは形状

const char *pgstat_get_wait_event(uint32 wait_event_info)

入力と出力

  • **入力**: wait_event_info (uint32型)。待機クラスとイベントIDをエンコードした値。
  • **出力**: 待機イベント名を表す const char * 型の文字列。

前提条件と不変条件

  • wait_event_info が 0 の場合、待機中ではないと判断され NULL を返す。
  • wait_event_info は上位ビットでクラスを、下位ビットでイベントIDを保持している必要がある。

アルゴリズムまたは振る舞い

1. wait_event_info が 0 ならば NULL を返す。 2. WAIT_EVENT_CLASS_MASK および WAIT_EVENT_ID_MASK を用いて、クラスIDとイベントIDを抽出する。 3. クラスIDに基づき switch 文で分岐し、対応するサブシステム（LWLock, Lock, IO, IPC, Timeout, Client, Activity, Extension, InjectionPoint）の識別関数を呼び出す。 4. どのクラスにも該当しない場合は "unknown wait event" を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c:453
  • src/backend/utils/adt/pgstatfuncs.c:839
  • **呼び出し先**:
  • GetLWLockIdentifier
  • GetLockNameFromTagType
  • GetWaitEventCustomIdentifier
  • pgstat_get_wait_bufferpin
  • pgstat_get_wait_activity
  • pgstat_get_wait_client
  • pgstat_get_wait_ipc
  • pgstat_get_wait_timeout
  • pgstat_get_wait_io

関連データ構造

  • wait_event_info (uint32): 待機状態を表現するビットフィールド。
  • WaitEventBufferPin, WaitEventActivity, WaitEventClient, WaitEventIPC, WaitEventTimeout, WaitEventIO: 各クラスに対応する列挙型または型定義。

失敗モードと境界条件

  • wait_event_info が定義済みのクラスIDのいずれにも一致しない場合、デフォルトケースとして "unknown wait event" が返される。

保守担当者向けメモ

  • 新しい待機クラスを追加する場合は、本関数の switch 文に新しい case を追加し、対応する識別関数を呼び出す必要がある。
  • 頻繁に呼び出される可能性があるため、各識別関数は効率的に名前を解決するように実装すること。
