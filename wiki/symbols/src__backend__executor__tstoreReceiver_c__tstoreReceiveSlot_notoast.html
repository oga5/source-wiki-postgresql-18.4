tstoreReceiveSlot_notoast

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/tstoreReceiver.c:121

目的

実行エンジンから受け取ったタプルを、変換処理（TOAST展開やタプルマップ）を行わずにそのまま Tuplestore へ格納する。

シグネチャまたは形状

static bool tstoreReceiveSlot_notoast(TupleTableSlot *slot, DestReceiver *self)

入力と出力

  • **入力**:
  • slot: 格納対象の TupleTableSlot。
  • self: TStoreState 構造体としてキャストされる DestReceiver インターフェース。
  • **出力**:
  • 常に true を返す。

前提条件と不変条件

  • self は TStoreState 型として安全にキャスト可能であること。
  • myState->tstore が初期化済みであること。

アルゴリズムまたは振る舞い

1. self を TStoreState 型にキャストし、内部の tstore（Tuplestore）への参照を取得する。 2. tuplestore_puttupleslot を呼び出し、slot の内容を tstore に書き込む。 3. 処理完了後、true を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（DestReceiver インターフェース経由）。tstoreStartupReceiver 内で、変換が不要な場合の受信関数として self->pub.receiveSlot に割り当てられる。
  • **呼び出し先**: tuplestore_puttupleslot（src/include/utils/tuplestore.h）。

関連データ構造

  • TStoreState: DestReceiver を継承したプライベート構造体。
  • TupleTableSlot: 実行エンジンがタプルを保持するために使用する構造体。
  • Tuplestore: タプルの格納先となるメモリまたは一時ファイル上のストレージ。

失敗モードと境界条件

  • Tuplestore への書き込み時にメモリ不足やディスクフルが発生した場合、tuplestore_puttupleslot 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、TOAST データの展開やタプル構造の変換が不要な場合にのみ使用される。
  • tstoreStartupReceiver での条件分岐により、実行時に動的に割り当てられる。
