StorePreparedStatement

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:391

目的

セッション内で再利用可能なプリペアドステートメントを、バックエンドローカルのハッシュテーブルに登録・永続化する。

シグネチャまたは形状

void StorePreparedStatement(const char *stmt_name, CachedPlanSource *plansource, bool from_sql)

入力と出力

  • **入力**:
  • stmt_name: プリペアドステートメントの識別名。
  • plansource: キャッシュ対象の計画ソース (CachedPlanSource)。
  • from_sql: SQLコマンド経由で作成されたかを示すフラグ。
  • **出力**: なし（成功時はハッシュテーブルへ登録、失敗時はエラーをスロー）。

前提条件と不変条件

  • prepared_queries ハッシュテーブルが初期化されていること（未初期化の場合は関数内で InitQueryHashTable() を呼び出す）。
  • 指定された stmt_name がハッシュテーブル内に存在しないこと。

アルゴリズムまたは振る舞い

1. prepared_queries が未初期化であれば InitQueryHashTable() を実行する。 2. hash_search を使用して、ハッシュテーブルに stmt_name でエントリを確保する。 3. 同名のステートメントが既に存在する場合、ERRCODE_DUPLICATE_PSTATEMENT エラーを発生させる。 4. 確保したエントリに plansource、from_sql、および現在のタイムスタンプを格納する。 5. SaveCachedPlan(plansource) を呼び出し、計画ソースを永続メモリへ移動させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/prepare.c (PREPARE文の処理フロー)
  • src/backend/tcop/postgres.c (拡張クエリプロトコルのParseメッセージ処理)
  • **呼び出し先**:
  • InitQueryHashTable()
  • hash_search()
  • GetCurrentStatementStartTimestamp()
  • SaveCachedPlan()
  • ereport() (エラー発生時)

関連データ構造

  • PreparedStatement: ハッシュテーブルに格納される構造体。
  • CachedPlanSource: 計画のキャッシュ基盤となる構造体。
  • prepared_queries: セッションローカルのハッシュテーブル。

失敗モードと境界条件

  • **重複エラー**: 同名のプリペアドステートメントが既に存在する場合、ereport(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数はセッションローカルな管理を行うため、他セッションとの共有は考慮不要。
  • SaveCachedPlan を呼び出すことで、計画ソースのメモリ寿命をセッション終了まで延長する点に注意すること。
