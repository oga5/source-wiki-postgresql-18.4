pg_popcount_masked_choose

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_aarch64.c:96

目的

AArch64環境において、実行時にCPUの機能（SVEの有無等）を判定し、最適なマスク付きビットカウント実装へ動的に切り替えるための初期化関数です。

シグネチャまたは形状

static uint64 pg_popcount_masked_choose(const char *buf, int bytes, bits8 mask)

入力と出力

  • buf: カウント対象のデータバッファへのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • mask: 各バイトに対して適用するビットマスク。
  • 戻り値: マスクされたデータ内のセットビット（1）の総数。

前提条件と不変条件

  • 本関数は、pg_popcount_masked_optimized 関数ポインタが初期化されていない、あるいは再初期化が必要なタイミングで呼び出されます。
  • choose_popcount_functions() が実行されることで、以降の呼び出し先が最適化された実装に差し替えられます。

アルゴリズムまたは振る舞い

1. choose_popcount_functions() を呼び出し、CPUの機能判定と関数ポインタの更新を行います。 2. 更新された pg_popcount_masked_optimized を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/pg_bitutils.c 内のビット操作ユーティリティ層。
  • 呼び出し先: choose_popcount_functions（CPU機能判定）、pg_popcount_masked_optimized（実処理）。

関連データ構造

  • pg_popcount_masked_optimized: 最適化された実装を指す関数ポインタ。

失敗モードと境界条件

  • bytes が 0 の場合、各実装の仕様に従い 0 を返します。
  • CPU機能判定が失敗した場合の挙動は choose_popcount_functions の実装に依存します。

保守担当者向けメモ

  • 本関数はディスパッチャとして機能します。直接的な計算ロジックは含まれておらず、実処理は pg_popcount_masked_optimized に委譲されます。
  • POPCNT_AARCH64 が定義されている環境でのみ有効です。
