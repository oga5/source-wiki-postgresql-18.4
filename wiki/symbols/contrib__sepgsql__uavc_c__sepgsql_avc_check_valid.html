sepgsql_avc_check_valid

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:151

目的

SELinux ポリシーの更新状況を監視し、ユーザー空間アクセスベクタキャッシュ（AVC）の妥当性を検証する。

シグネチャまたは形状

static bool sepgsql_avc_check_valid(void)

入力と出力

  • 入力: なし
  • 出力: bool 型。キャッシュが有効な場合は true、ポリシー更新によりキャッシュがフラッシュされた場合は false を返す。

前提条件と不変条件

  • 本関数は、権限チェック処理を行うループ内から呼び出されることを想定している。
  • selinux_status_updated() が正の値を返す場合、ポリシーが変更されたとみなされる。

アルゴリズムまたは振る舞い

1. selinux_status_updated() を呼び出し、SELinux の状態更新を確認する。 2. 更新が検出された場合（戻り値 > 0）、sepgsql_avc_reset() を呼び出してキャッシュを全消去し、false を返す。 3. 更新がない場合、何も行わず true を返す。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/sepgsql/uavc.c 内の各権限チェック処理（do-while ループの条件式として使用）。
  • 呼び出し先: selinux_status_updated()（SELinux 状態確認）、sepgsql_avc_reset()（キャッシュリセット）。

関連データ構造

  • avc_cache（キャッシュエントリの保持に使用される構造体）

失敗モードと境界条件

  • 本関数自体はエラーを返さないが、ポリシー更新時にキャッシュをリセットするため、呼び出し元はキャッシュの再構築が発生することを前提とした制御フロー（再試行ループ）を維持する必要がある。

保守担当者向けメモ

  • 権限チェックはシステムコールを跨ぐ可能性があるため、本関数をループ内で使用するイディオムは必須である。
  • キャッシュのメモリ管理は avc_mem_cxt に依存しており、sepgsql_avc_reset による一括解放がキャッシュの整合性を保証する。
