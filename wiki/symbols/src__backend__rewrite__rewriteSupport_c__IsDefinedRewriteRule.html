IsDefinedRewriteRule

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteSupport.c:31

目的

指定されたテーブル（リレーション）に、特定の名前を持つ書き換えルールが存在するかを判定します。

シグネチャまたは形状

bool IsDefinedRewriteRule(Oid owningRel, const char *ruleName)

入力と出力

  • **入力**:
  • owningRel: ルールを所有するテーブルのOID。
  • ruleName: 検索対象のルール名（文字列）。
  • **出力**:
  • bool: 指定したテーブルに該当するルールが存在する場合は true、存在しない場合は false を返します。

前提条件と不変条件

  • システムカタログ pg_rewrite が検索可能な状態である必要があります。
  • SearchSysCacheExists2 を使用しているため、システムキャッシュが有効な環境で動作します。

アルゴリズムまたは振る舞い

1. RULERELNAME システムキャッシュインデックスを使用して、owningRel（OID）と ruleName（文字列）の組み合わせを検索します。 2. キャッシュ内に該当するエントリが存在するかを確認し、その結果を真偽値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/rewrite/rewriteDefine.c:832（ルール定義の重複チェック等で使用）
  • **呼び出し先**:
  • SearchSysCacheExists2（システムキャッシュ検索関数）

関連データ構造

  • pg_rewrite: ルール定義を保持するシステムカタログ。
  • RULERELNAME: pg_rewrite のシステムキャッシュ検索用インデックス。

失敗モードと境界条件

  • owningRel が無効なOIDである場合、キャッシュ検索は失敗し false を返します。
  • ruleName がNULLまたは不正なポインタである場合の挙動は、SearchSysCacheExists2 の実装に依存します。

保守担当者向けメモ

  • 本関数はカタログへの直接アクセスを避け、システムキャッシュを利用することで高速な判定を実現しています。
  • ルール定義の作成・削除を行う rewriteDefine.c 等で、名前の衝突チェックや存在確認のために利用されます。
