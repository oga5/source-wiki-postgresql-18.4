PageGetFreeSpaceForMultipleTuples

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/bufpage.c:932

目的

ページ内に複数のタプルを新たに追加する際、アイテムID配列の領域を確保した上で、タプルデータ用に利用可能な残りの空き容量を計算します。

シグネチャまたは形状

Size PageGetFreeSpaceForMultipleTuples(const PageData *page, int ntups)

入力と出力

  • **入力**:
  • page: 対象となるページへのポインタ。
  • ntups: 追加予定のタプル数。
  • **出力**:
  • Size: タプルデータ格納用に利用可能なバイト数。空き容量が不足している場合は 0 を返します。

前提条件と不変条件

  • page は PageHeaderData としてキャスト可能である必要があります。
  • 内部計算では符号付き整数を使用しており、pd_lower が pd_upper を超えている（ページが破損または初期状態）場合でも、論理的に妥当な挙動を維持します。

アルゴリズムまたは振る舞い

1. ページヘッダーから pd_upper（タプルデータの開始位置）と pd_lower（アイテムID配列の終了位置）を取得します。 2. ページ全体の物理的な空き領域（pd_upper - pd_lower）を算出します。 3. 追加するタプル数 ntups 分のアイテムID配列領域（ntups * sizeof(ItemIdData)）を確保できるか判定します。 4. 確保できない場合は 0 を返します。 5. 確保可能な場合、物理的な空き領域からアイテムID配列分を差し引いた残りのサイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hashovfl.c
  • src/backend/access/hash/hashpage.c
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • PageHeaderData: ページ内の pd_lower および pd_upper を保持する構造体。
  • ItemIdData: ページ内のアイテムID配列の各要素。

失敗モードと境界条件

  • ntups 分のアイテムID配列を確保できない場合、空き容量として 0 を返します。
  • ページが満杯に近い場合、計算結果が負にならないよう符号付き演算で処理されます。

保守担当者向けメモ

  • 本関数は、タプル挿入前に「挿入可能か」を事前判定するために使用されます。
  • 戻り値はタプルデータ本体を格納するためのサイズであり、アイテムID配列自体のサイズは含まれません。
