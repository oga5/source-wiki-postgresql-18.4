ReadStream

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/aio/read_stream.c:92

目的

リレーションデータの読み込みにおいて、適応的な先読み（Look-ahead）とバッチ処理を行うための状態管理構造体です。単一ブロックごとの I/O 待機を回避し、I/O 効率を最大化します。

シグネチャまたは形状

struct ReadStream {
    int16 max_ios, io_combine_limit, ios_in_progress, queue_size, max_pinned_buffers;
    int16 forwarded_buffers, pinned_buffers, distance, initialized_buffers;
    int read_buffers_flags;
    bool sync_mode, batch_mode, advice_enabled, temporary;
    BlockNumber buffered_blocknum;
    ReadStreamBlockNumberCB callback;
    void *callback_private_data;
    BlockNumber seq_blocknum, seq_until_processed;
    BlockNumber pending_read_blocknum;
    int16 pending_read_nblocks;
    size_t per_buffer_data_size;
    void *per_buffer_data;
    InProgressIO *ios;
    int16 oldest_io_index, next_io_index;
    bool fast_path;
    int16 oldest_buffer_index, next_buffer_index;
    Buffer buffers[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: ReadStreamBlockNumberCB 型のコールバック関数を保持し、これを通じて読み込むべきブロック番号を取得します。
  • **出力**: 循環キュー（buffers）を介して、ピン留めされたバッファを消費者に提供します。

前提条件と不変条件

  • READ_STREAM_USE_BATCHING フラグが有効な場合、コールバック関数は pgaio_enter_batchmode() の制約を遵守する必要があります。
  • 連続するブロックは pending_read に蓄積され、一定量または制限に達した時点で StartReadBuffers が発行されます。

アルゴリズムまたは振る舞い

1. **先読み制御**: キャッシュヒット時は先読み距離を 1 に縮小し、I/O 発生時は指数関数的に増加させます。 2. **バッチ I/O**: コールバックから取得したブロック番号をマージし、StartReadBuffers で一括読み込みを行います。 3. **キュー管理**: 循環キューを用いて、進行中の I/O とピン留めされたバッファを追跡します。 4. **最適化**: キャッシュヒット率が高い場合、キュー管理をスキップする「Fast Path」が動作します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c、contrib/pg_prewarm/autoprewarm.c、src/backend/access/heap/heapam.c 等の各スキャン処理。
  • **呼び出し先**: StartReadBuffers、WaitReadBuffers、および登録された ReadStreamBlockNumberCB コールバック関数。

関連データ構造

  • InProgressIO: 進行中の I/O 操作を管理する構造体。
  • Buffer: バッファマネージャが管理するバッファ識別子。

失敗モードと境界条件

  • I/O が分割される際、フロー制御のために buffered_blocknum を使用してブロック番号の「アンゲット（unget）」をサポートします。
  • バッチモード使用時のデッドロック回避制約に違反すると、システムが停止する可能性があります。

保守担当者向けメモ

  • buffers フィールドは FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体の割り当て時にはサイズ計算に注意してください。
  • デバッグ時は、Fast Path が有効な場合とそうでない場合でキュー管理の挙動が異なる点に留意してください。
