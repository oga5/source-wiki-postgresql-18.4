itemptr_encode

定義

  • 種別: static inline function
  • 定義箇所: src/include/catalog/index.h:189

目的

ItemPointer（TID）をソート可能な int64 値に変換し、インデックス検証処理におけるバルク処理のパフォーマンスを最適化する。

シグネチャまたは形状

static inline int64 itemptr_encode(ItemPointer itemptr)

入力と出力

  • **入力**: ItemPointer itemptr（変換対象のタプル識別子）
  • **出力**: int64（エンコードされた値）

前提条件と不変条件

  • 2の補数表現を前提とする。
  • エンコード後の値は、ブロック番号とオフセット番号のビット配置により、負の値にはならない。

アルゴリズムまたは振る舞い

1. ItemPointerGetBlockNumber を用いてブロック番号を取得し、上位 32 ビットへシフトする。 2. ItemPointerGetOffsetNumber を用いてオフセット番号を取得し、下位 16 ビットに配置する。 3. これらをビット論理和で結合し、int64 型として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/index.c:3486 にて呼び出しを確認。
  • **呼び出し先**: ItemPointerGetBlockNumber および ItemPointerGetOffsetNumber（マクロ）。

関連データ構造

  • ItemPointer: タプル識別子。
  • BlockNumber: ブロック番号。
  • OffsetNumber: オフセット番号。

失敗モードと境界条件

  • 入力された ItemPointer が無効な場合、エンコード結果も無意味な値となる。
  • ブロック番号が 32 ビット、オフセット番号が 16 ビットの範囲を超えないことを前提としている。

保守担当者向けメモ

  • itemptr_decode と対になる関数である。
  • ソート順序を維持したまま変換を行うため、ビット配置の変更はソートロジックに影響を与える点に注意すること。
