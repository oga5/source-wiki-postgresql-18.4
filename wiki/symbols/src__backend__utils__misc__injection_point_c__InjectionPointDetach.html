InjectionPointDetach

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/injection_point.c:361

目的

共有メモリ上に登録された特定のインジェクションポイントを無効化（削除）します。

シグネチャまたは形状

bool InjectionPointDetach(const char *name);

入力と出力

  • **入力**: name (削除対象のインジェクションポイント名)
  • **出力**: bool (指定された名前のポイントが見つかり、正常に削除された場合は true、それ以外は false)

前提条件と不変条件

  • --enable-injection-points を指定してビルドされている必要があります。無効なビルドでは elog(ERROR) が発生します。
  • 内部的に InjectionPointLock（排他ロック）を取得するため、呼び出し元はロックの競合を考慮する必要があります。

アルゴリズムまたは振る舞い

1. InjectionPointLock を排他モードで取得します。 2. 共有メモリ上の ActiveInjectionPoints 配列を末尾から走査し、指定された name に一致するエントリを探します。 3. エントリの generation カウンタをインクリメントし、偶数にすることで「空きスロット」としてマークします。 4. 削除したエントリが max_inuse（使用中の最大インデックス）に該当する場合、末尾から空きスロットをスキップして max_inuse を再計算し、更新します。 5. ロックを解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/injection_points/injection_points.c 内のテストコードから呼び出されています。
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (共有メモリロック管理)
  • pg_atomic_read_u32 / pg_atomic_write_u32 / pg_atomic_read_u64 / pg_atomic_write_u64 (アトミック操作)
  • strcmp (文字列比較)

関連データ構造

  • InjectionPointsCtl: 共有メモリ上の管理構造体。
  • InjectionPointEntry: 個々のインジェクションポイント定義。

失敗モードと境界条件

  • --enable-injection-points が無効なビルドでは、関数呼び出し自体がエラーとなります。
  • 指定した名前のインジェクションポイントが存在しない場合、false を返します。
  • 共有メモリのロック取得に失敗した場合や、他のプロセスとの競合による影響は、PostgreSQL の標準的なロック管理機構に依存します。

保守担当者向けメモ

  • 本関数はテストおよびデバッグ専用です。本番環境での使用は想定されていません。
  • generation カウンタの偶数/奇数判定によるロックフリーな読み取り設計を維持するため、削除時のカウンタ更新は必須です。
