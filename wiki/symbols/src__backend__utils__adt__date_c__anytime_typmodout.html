anytime_typmodout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/date.c:93

目的

TIME および TIMETZ 型の型修飾子（typmod）を、SQLの型定義文字列（例: (6) without time zone）に変換して返します。

シグネチャまたは形状

char *anytime_typmodout(bool istz, int32 typmod)

入力と出力

  • **入力**:
  • istz (bool): タイムゾーン付き（TIMETZ）か否か（TIME）を指定します。
  • typmod (int32): 型の精度（秒の小数部桁数）。負の値は精度指定なしを意味します。
  • **出力**:
  • char *: 変換された文字列。pstrdup または psprintf を使用してメモリが割り当てられます。

前提条件と不変条件

  • typmod が 0 以上の場合、精度指定を含む文字列が生成されます。
  • typmod が負の場合、精度指定なしの文字列が生成されます。
  • istz が true の場合は " with time zone"、false の場合は " without time zone" が文字列の末尾に付加されます。

アルゴリズムまたは振る舞い

1. istz に基づき、タイムゾーンの有無を示す文字列 tz を決定します。 2. typmod が 0 以上であれば、psprintf を用いて (精度)tz の形式で文字列を構築します。 3. typmod が負であれば、pstrdup を用いて tz のみをコピーして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/date.c:1635 (TIME 型の出力処理)
  • src/backend/utils/adt/date.c:2460 (TIMETZ 型の出力処理)
  • **呼び出し先**:
  • psprintf (メモリ割り当てと書式化)
  • pstrdup (文字列の複製)

関連データ構造

  • TimeADT: TIME 型の内部表現。
  • TimeTzADT: TIMETZ 型の内部表現。

失敗モードと境界条件

  • メモリ不足時には psprintf や pstrdup がエラーを発生させる可能性があります。
  • typmod の値自体に対する妥当性検証は本関数内では行われません（呼び出し元で anytime_typmod_check 等を用いて事前に行うことが想定されます）。

保守担当者向けメモ

  • 本関数は TIME および TIMETZ 型の typmodout 関数として共通利用されています。
  • 戻り値は palloc されたメモリ領域であるため、呼び出し側で適切に管理される必要があります。
