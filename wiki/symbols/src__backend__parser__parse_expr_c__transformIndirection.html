transformIndirection

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_expr.c:436

目的

生の式（A_Indirection）に含まれる配列の添字操作およびフィールド選択を、意味解析済みの式ツリーへ変換する。

シグネチャまたは形状

static Node *transformIndirection(ParseState *pstate, A_Indirection *ind)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体。
  • ind: 変換対象の A_Indirection ノード（配列添字やフィールド参照のリスト）。
  • **出力**: 変換後の Node。

前提条件と不変条件

  • ind->arg は transformExprRecurse により再帰的に変換可能であること。
  • A_Indices（添字）と String（フィールド名）が混在するリストを処理する。

アルゴリズムまたは振る舞い

1. ind->arg を transformExprRecurse で変換し、ベースとなる式を取得する。 2. ind->indirection リストを走査し、以下の処理を行う:

  • A_Indices の場合: subscripts リストに蓄積する。
  • A_Star の場合: エラーを報告する（サポート外）。
  • String（フィールド名）の場合:
  • 蓄積された subscripts があれば transformContainerSubscripts を呼び出し、配列操作を確定させる。
  • ParseFuncOrColumn を呼び出し、フィールド選択を解決する。解決できない場合は unknown_attribute を呼び出す。

    • 3. ループ終了後、残存する subscripts があれば再度 transformContainerSubscripts を呼び出して処理を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_expr.c 内の式変換処理（transformExprRecurse 等からの呼び出しが示唆される）。
  • **呼び出し先**:
  • transformExprRecurse
  • transformContainerSubscripts
  • ParseFuncOrColumn
  • unknown_attribute

関連データ構造

  • A_Indirection: 構文解析器が出力する生のインダイレクションノード。
  • ParseState: 解析中の状態管理構造体。
  • A_Indices: 配列の添字情報。

失敗モードと境界条件

  • A_Star が指定された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーとなる。
  • フィールド名が解決できない場合、unknown_attribute によりエラーが報告される。

保守担当者向けメモ

  • 本関数は配列の添字とフィールド選択の順序を正しく処理する必要がある。subscripts リストを適宜フラッシュするロジックが重要である。
  • A_Indirection のリスト構造が複雑なため、変更時は transformContainerSubscripts との整合性に注意すること。
