reset_tsvector_parser

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_parser.c:80

目的

既存の TSVectorParseState オブジェクトの解析対象文字列（バッファ）を新しい入力文字列で上書きし、パーサーの状態をリセットします。

シグネチャまたは形状

void reset_tsvector_parser(TSVectorParseState state, char *input)

入力と出力

  • **入力**:
  • state: 解析状態を保持する TSVectorParseState 構造体へのポインタ。
  • input: 新たに解析対象とする文字列へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • state は init_tsvector_parser 等によって初期化済みである必要があります。
  • input は解析終了まで有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った input ポインタを、state 構造体内の prsbuf メンバに代入します。 2. これにより、以降のトークン解析処理（gettoken_tsvector 等）は、新しい input 文字列の先頭から開始されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery.c: reset_tsvector_parser(state->valstate, state->buf); として呼び出されています。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • TSVectorParseState: 解析の進捗やバッファ情報を保持する構造体。

失敗モードと境界条件

  • state が NULL の場合、セグメンテーション違反が発生します。
  • input が NULL の場合、prsbuf に NULL が設定され、後続のトークン解析関数でヌルポインタ参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数は単にバッファポインタを差し替えるのみであり、解析状態（有限オートマトンの状態コードなど）の初期化は行いません。状態を完全に初期化する必要がある場合は、init_tsvector_parser の利用を検討してください。
