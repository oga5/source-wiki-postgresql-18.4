bbsink_gzip_archive_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_gzip.c:166

目的

入力バッファ内のデータを zlib を用いて gzip 圧縮し、圧縮後のデータを後続の bbsink へ転送する。

シグネチャまたは形状

static void bbsink_gzip_archive_contents(bbsink *sink, size_t len)

入力と出力

  • sink: 処理対象の bbsink_gzip 構造体（bbsink として渡される）。
  • len: 圧縮対象となる入力バッファ（sink->base.bbs_buffer）内のデータ長。
  • 出力: 圧縮されたデータは sink->base.bbs_next が指す後続シンクへ bbsink_archive_contents を介して渡される。

前提条件と不変条件

  • sink は bbsink_gzip 型へキャスト可能であること。
  • sink->base.bbs_buffer に圧縮対象データが格納されていること。
  • sink->base.bbs_next が有効な後続シンクとして設定されていること。

アルゴリズムまたは振る舞い

1. z_stream の入力ポインタ（next_in）を現在のバッファに設定する。 2. 入力データが残っている間、以下のループを実行する。

  • 出力バッファの空き領域を z_stream の出力ポインタ（next_out）に割り当てる。
  • deflate(zs, Z_NO_FLUSH) を呼び出し、データを圧縮する。
  • 圧縮後のデータ量を mysink->bytes_written に更新する。
  • 出力バッファが満杯になった場合、bbsink_archive_contents を呼び出して後続シンクへデータを転送し、bytes_written をリセットする。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink インターフェースの .archive_contents 関数ポインタ経由で呼び出される（直接の呼び出し元コードはソース抜粋からは特定不可）。
  • 呼び出し先: deflate (zlib)、bbsink_archive_contents (後続シンクへの転送)。

関連データ構造

  • bbsink_gzip: 圧縮状態（z_stream）やバッファ管理情報を保持する構造体。
  • bbsink: バックアップ処理のパイプラインを構成する基本構造体。

失敗モードと境界条件

  • deflate が Z_STREAM_ERROR を返した場合、elog(ERROR) により処理を中断する。これはメモリ破壊やプログラミングエラーを示唆する。

保守担当者向けメモ

  • 本関数は Z_NO_FLUSH を使用しているため、データがバッファサイズに達するまで後続へ送出されない。
  • 最終的なフラッシュ処理については bbsink_gzip_end_archive を参照すること。
