TidStoreLockShare

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tidstore.c:293

目的

TidStore 構造体に対して共有ロック（Shared Lock）を取得し、並行アクセスを制御します。

シグネチャまたは形状

void TidStoreLockShare(TidStore *ts)

入力と出力

  • **入力**: TidStore *ts - ロック対象の TidStore 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • ts は有効な TidStore 構造体を指している必要があります。
  • 共有メモリ上で動作する TidStore である場合にのみロック操作が実行されます。

アルゴリズムまたは振る舞い

1. TidStoreIsShared(ts) を呼び出し、対象の TidStore が共有メモリを使用しているか判定します。 2. 共有メモリを使用している場合、内部の shared_ts_lock_share 関数を呼び出し、共有ロックを取得します。 3. ローカルメモリ上の TidStore の場合は、何も行わずに終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_tidstore/test_tidstore.c (テストモジュール)
  • **呼び出し先**:
  • shared_ts_lock_share (内部関数)

関連データ構造

  • TidStore: ロック対象となるデータ構造。
  • LWLock: shared_ts_lock_share 経由で間接的に利用される PostgreSQL の軽量ロック。

失敗モードと境界条件

  • ts が NULL の場合の挙動は定義されていません。
  • 共有メモリを使用していない TidStore に対して呼び出した場合、ロックは取得されず、副作用も発生しません。

保守担当者向けメモ

  • 本関数は共有メモリ版の TidStore を使用する際の並行制御インターフェースです。
  • ロックの解放には TidStoreUnlock を使用してください。
