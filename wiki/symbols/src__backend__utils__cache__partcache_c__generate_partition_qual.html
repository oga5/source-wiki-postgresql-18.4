generate_partition_qual

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/partcache.c:336

目的

指定されたパーティションリレーションが満たすべき制約式（qual）のリストを構築し、リレーションキャッシュ（relcache）に格納・返却する。

シグネチャまたは形状

static List *generate_partition_qual(Relation rel)

入力と出力

  • **入力**: rel (Relation) - 制約を生成対象とするパーティションリレーション。
  • **出力**: List * - パーティションの境界条件を表現する制約式のリスト。

前提条件と不変条件

  • check_stack_depth() により、深いパーティション階層によるスタックオーバーフローを防止する。
  • rel->rd_partcheckvalid が真の場合、キャッシュ済みの rd_partcheck のコピーを即座に返す。
  • パーティションキーの属性番号（attnos）は、親テーブルの定義から現在のリレーションの定義へ変換される。
  • 処理完了時、rel->rd_partcheckcxt はキャッシュされた制約式を保持するメモリコンテキストとして初期化される。

アルゴリズムまたは振る舞い

1. キャッシュ済みであればそのコピーを返却する。 2. 親テーブルを AccessShareLock でオープンする。 3. pg_class システムカタログから relpartbound を取得し、get_qual_from_partbound を用いて境界条件を制約式に変換する。 4. 親テーブルがさらにパーティションである場合、再帰的に generate_partition_qual を呼び出し、自身の制約と連結する。 5. map_partition_varattnos を使用し、制約式内の変数属性番号を現在のリレーションの定義に適合させる。 6. 結果を CacheMemoryContext の子コンテキストにコピーし、rd_partcheck に格納して rd_partcheckvalid を真にする。 7. 親テーブルのロックを解放し、生成したリストを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/partcache.c 内の内部関数から呼び出される（再帰呼び出しを含む）。
  • **呼び出し先**:
  • check_stack_depth()
  • get_partition_parent()
  • relation_open() / relation_close()
  • SearchSysCache1() / ReleaseSysCache()
  • get_qual_from_partbound()
  • map_partition_varattnos()
  • copyObject()
  • AllocSetContextCreate()

関連データ構造

  • Relation: リレーションキャッシュ構造体。
  • PartitionBoundSpec: パーティション境界定義。
  • List: 制約式のリスト。
  • MemoryContext: キャッシュ保持用のメモリコンテキスト。

失敗モードと境界条件

  • pg_class のルックアップに失敗した場合、elog(ERROR) を発生させる。
  • パーティション階層が非常に深い場合、check_stack_depth() によりスタックオーバーフローが検出される。
  • rd_partcheckcxt が既に存在する場合、アサーションエラーとなる。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、パーティションツリーの深さに注意が必要。
  • メモリリークを防ぐため、CacheMemoryContext の子コンテキスト管理が厳密に行われている。
  • map_partition_varattnos は varno = 1 を前提としているため、将来的な構造変更時にはこの仮定を確認すること。
