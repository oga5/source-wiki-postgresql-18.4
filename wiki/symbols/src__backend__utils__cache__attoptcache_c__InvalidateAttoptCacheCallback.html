InvalidateAttoptCacheCallback

定義

  • 種別: static void 関数
  • 定義箇所: src/backend/utils/cache/attoptcache.c:52

目的

pg_attribute システムカタログの更新イベントに応答し、AttoptCacheHash 内の属性オプションキャッシュエントリを破棄してメモリを解放する。

シグネチャまたは形状

static void InvalidateAttoptCacheCallback(Datum arg, int cacheid, uint32 hashvalue)

入力と出力

  • arg: 未使用（コールバック登録時の引数）。
  • cacheid: 無効化対象のキャッシュID。
  • hashvalue: 無効化対象のハッシュ値。0 の場合はキャッシュ全体を対象とする。
  • 出力: なし（副作用としてキャッシュエントリの削除とメモリ解放を行う）。

前提条件と不変条件

  • AttoptCacheHash が初期化済みであること。
  • キャッシュエントリの opts フィールドは pfree 可能なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. hashvalue が 0 の場合、hash_seq_init を用いてハッシュテーブル全体を走査対象とする。 2. hashvalue が 0 以外の場合、hash_seq_init_with_hash_value を用いて指定されたハッシュ値に一致するエントリのみを走査対象とする。 3. 走査ループ内で各 AttoptCacheEntry を取得する。 4. エントリ内の opts が存在する場合、pfree でメモリを解放する。 5. hash_search を HASH_REMOVE モードで呼び出し、ハッシュテーブルからエントリを削除する。 6. 削除に失敗した場合（NULL が返された場合）、elog(ERROR) を発生させハッシュテーブルの破損を報告する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/attoptcache.c 内の初期化処理（CacheRegisterSyscacheCallback への登録用）。src/backend/utils/cache/catcache.c からの言及あり。
  • 呼び出し先: hash_seq_init, hash_seq_init_with_hash_value, hash_seq_search, pfree, hash_search, elog。

関連データ構造

  • AttoptCacheHash: 属性オプションを保持するハッシュテーブル。
  • AttoptCacheEntry: キャッシュ内の各エントリ構造体。

失敗モードと境界条件

  • hashvalue が 0 の場合、キャッシュ全体の無効化が行われる。
  • ハッシュテーブルの検索・削除処理中に内部不整合が検出された場合、elog(ERROR) によりトランザクションがアボートされる。

保守担当者向けメモ

  • 本関数は CacheRegisterSyscacheCallback を介してシステムキャッシュの無効化通知を受け取るために使用される。
  • キャッシュエントリの削除時に pfree を行うため、opts フィールドのメモリ管理には注意が必要である。
