blackhole_get_sink

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_target.c:193

目的

バックアップターゲットとして「blackhole（ブラックホール）」が選択された際に、データ転送パイプラインを構築するためのシンク生成関数として機能します。

シグネチャまたは形状

static bbsink *blackhole_get_sink(bbsink *next_sink, void *detail_arg)

入力と出力

  • **入力**:
  • next_sink: 現在のパイプラインにおける後続の bbsink オブジェクト。
  • detail_arg: ターゲット詳細情報（本関数では使用されません）。
  • **出力**:
  • next_sink: 受け取った next_sink をそのまま返します。

前提条件と不変条件

  • 本関数は blackhole ターゲットの定義の一部として呼び出されます。
  • next_sink が有効な bbsink ポインタであることを前提とします。

アルゴリズムまたは振る舞い

  • 渡された next_sink を変更することなく、そのまま呼び出し元へ返します。これにより、ブラックホールターゲットは実質的にデータ転送パイプラインにおいて「何もしない（データを捨てる）」ノードとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: BaseBackupGetSink() からの内部的な呼び出しが想定されます。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • bbsink: バックアップデータの転送パイプラインを構成する抽象インターフェース。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。入力された next_sink をそのまま返すという単純な動作を行います。

保守担当者向けメモ

  • 本関数は builtin_backup_targets 配列内で blackhole ターゲットに関連付けられています。
  • 拡張モジュールが独自のターゲットを追加する際の get_sink 実装の最小構成例として参照可能です。
