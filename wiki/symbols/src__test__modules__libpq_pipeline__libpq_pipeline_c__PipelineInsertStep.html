PipelineInsertStep

定義

  • 種別: enum
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:994

目的

test_pipelined_insert 関数におけるパイプライン処理の進行状況を追跡するための状態定義です。

シグネチャまたは形状

enum PipelineInsertStep
{
	BI_BEGIN_TX,
	BI_DROP_TABLE,
	BI_CREATE_TABLE,
	BI_PREPARE,
	BI_INSERT_ROWS,
	BI_COMMIT_TX,
	BI_SYNC,
	BI_DONE,
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • libpq のパイプラインモードにおける一連のデータベース操作手順を順序立てて表現します。
  • BI_BEGIN_TX から始まり BI_DONE で終了する線形的な遷移を想定しています。

アルゴリズムまたは振る舞い

  • 各列挙子は、パイプラインを通じて送信されるクエリや同期ポイントの各段階を表します。
  • BI_BEGIN_TX（トランザクション開始）から BI_COMMIT_TX（コミット）、BI_SYNC（パイプライン同期）を経て BI_DONE に至るまで、テストの進行状態を管理します。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/libpq_pipeline/libpq_pipeline.c:1013 にて send_step 変数の初期値として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • test_pipelined_insert: 本列挙型を使用してパイプラインのステップを制御する関数。

失敗モードと境界条件

  • 状態遷移の順序が正しく制御されない場合、パイプラインの整合性が失われ、PQpipelineSync による同期や結果取得でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本列挙型はテストコード専用のステートマシンです。新しいパイプライン操作ステップを追加する場合は、この列挙型を拡張し、対応する処理を test_pipelined_insert 内のロジックに追加してください。
