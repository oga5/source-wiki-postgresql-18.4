fix_join_expr_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:65

目的

結合ノード（Join Node）の式ツリーを走査・修正する際に、必要なコンテキスト情報とマッピング情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    PlannerInfo *root;
    indexed_tlist *outer_itlist;
    indexed_tlist *inner_itlist;
    Index       acceptable_rel;
    int         rtoffset;
    NullingRelsMatch nrm_match;
    double      num_exec;
} fix_join_expr_context;

入力と出力

本構造体は関数への引数として渡されるコンテキストオブジェクトです。

  • **入力**: 走査対象のプラン情報 (root)、結合の左右のターゲットリスト (outer_itlist, inner_itlist)、および変換ルール (acceptable_rel, rtoffset, nrm_match) を保持します。
  • **出力**: 構造体自体は変更可能なコンテキストとして機能し、走査過程で参照されます。

前提条件と不変条件

  • outer_itlist および inner_itlist は、サブプランの出力タプルを高速検索するための indexed_tlist 構造体である必要があります。
  • 結合ノードの式変換を行う前に、このコンテキストが適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

fix_join_expr_mutator 関数において、式ツリーを再帰的に走査する際の引数として使用されます。このコンテキストに基づき、式内の Var ノードを、実行時のスロット参照（OUTER_VAR や INNER_VAR 等）へ変換するためのマッピングが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/setrefs.c 内の結合ノード処理箇所（例: 3113 行目付近）。
  • **呼び出し先**: fix_join_expr_mutator 関数（3126 行目）がこの構造体を引数として受け取ります。

関連データ構造

  • PlannerInfo: プラン全体のグローバル情報。
  • indexed_tlist: ターゲットリストの高速検索用構造体。
  • NullingRelsMatch: 外部結合における NULL 許容関係の照合ルール。

失敗モードと境界条件

  • 構造体内のポインタが NULL の場合、fix_join_expr_mutator 内での参照時にクラッシュする可能性があります。
  • rtoffset が正しく設定されていない場合、レンジテーブルのインデックス解決が不整合を起こす可能性があります。

保守担当者向けメモ

  • 本構造体は setrefs.c におけるプランの後処理フェーズでのみ使用されます。
  • 結合ノードの式変換ロジックを変更する場合は、このコンテキストに含まれる nrm_match や acceptable_rel の定義が最新のプランナの要件を満たしているか確認してください。
