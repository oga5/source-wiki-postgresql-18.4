shm_mq_get_receiver

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_mq.c:241

目的

共有メモリメッセージキュー（shm_mq）に現在登録されている受信側プロセス（PGPROC）のポインタを取得します。

シグネチャまたは形状

PGPROC *shm_mq_get_receiver(shm_mq *mq);

入力と出力

  • **入力**: mq (共有メモリメッセージキュー構造体へのポインタ)
  • **出力**: 受信側として登録されている PGPROC 構造体へのポインタ。登録されていない場合は NULL を返します。

前提条件と不変条件

  • 引数 mq は有効な共有メモリ領域を指している必要があります。
  • 内部的に mq->mq_mutex スピンロックを使用して同期を行います。

アルゴリズムまたは振る舞い

1. mq->mq_mutex スピンロックを取得します。 2. キュー構造体内の mq_receiver フィールドの値を読み取ります。 3. mq->mq_mutex スピンロックを解放します。 4. 読み取った PGPROC ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/shm_mq.c:825 (受信側プロセスの判定に使用)
  • src/backend/storage/ipc/shm_mq.c:965 (受信側が設定されているかの確認に使用)
  • **呼び出し先**:
  • SpinLockAcquire
  • SpinLockRelease

関連データ構造

  • shm_mq: メッセージキュー本体。
  • PGPROC: PostgreSQL のプロセス制御構造体。

失敗モードと境界条件

  • mq が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • 受信側がまだ設定されていない場合、NULL を返します。これはエラーではなく正常な状態として扱われます。

保守担当者向けメモ

  • 本関数はスピンロックを使用してアトミックに値を読み取ります。呼び出し元で受信側プロセスが終了・変更される可能性がある場合、この値は取得時点でのスナップショットであることに注意してください。
