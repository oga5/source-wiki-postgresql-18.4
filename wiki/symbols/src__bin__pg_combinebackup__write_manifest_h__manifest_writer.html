manifest_writer

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_combinebackup/write_manifest.h:20

目的

バックアップの整合性を保証するための backup_manifest ファイルを生成・書き出す際のコンテキストを保持する不透明（opaque）な構造体です。

シグネチャまたは形状

typedef struct manifest_writer manifest_writer;

※ヘッダーファイル上では不透明な型として定義されており、内部構造は src/bin/pg_combinebackup/write_manifest.c で定義されます。

入力と出力

  • 入力: create_manifest_writer() により初期化され、add_file_to_manifest() を通じてファイルメタデータやチェックサムを受け取ります。
  • 出力: finalize_manifest() を通じて、最終的な JSON 形式の backup_manifest ファイルをディスクへ書き出します。

前提条件と不変条件

  • pg_combinebackup ツール内での使用を前提としています。
  • マニフェストの書き込み処理中は、バッファリングおよびチェックサム計算の状態が本構造体によって一貫して管理されます。

アルゴリズムまたは振る舞い

  • JSON ヘッダーの初期化、ファイルエントリの逐次追加、および最終的な WAL 範囲の記録とファイルのクローズを管理します。
  • バッファが一定サイズを超えた場合、適宜ディスクへフラッシュする振る舞いを含みます。

呼び出し元と呼び出し先

  • 呼び出し元: pg_combinebackup.c および backup_label.c から参照・利用されています。
  • 呼び出し先: write_manifest.c 内の flush_manifest、add_file_to_manifest、finalize_manifest 等の関数によって操作されます。

関連データ構造

  • common/checksum_helper.h: チェックサム計算アルゴリズムの定義に使用されます。

失敗モードと境界条件

  • マニフェストファイルへの書き込み失敗や、ディスク容量不足などの I/O エラーが発生する可能性があります。
  • 構造体の詳細は非公開であるため、直接的なフィールド操作は行わず、提供されたインターフェース関数を介して操作する必要があります。

保守担当者向けメモ

  • 本構造体はフロントエンド専用の実装です。バックエンドの AddFileToBackupManifest とは機能が類似していますが、実装は独立しています。
  • マニフェストのパスは常に backup_manifest という固定名で生成されます。
