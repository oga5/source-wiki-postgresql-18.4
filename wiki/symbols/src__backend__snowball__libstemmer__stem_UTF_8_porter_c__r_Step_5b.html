r_Step_5b

目的

Porter Stemming アルゴリズムの最終段階（Step 5b）として、単語末尾の二重 'l' を削除する処理を行う。

シグネチャまたは形状

static int r_Step_5b(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: int（成功時に 1、失敗または条件不一致時に 0、エラー時に負の値を返す）

前提条件と不変条件

  • z->c（現在のカーソル位置）が有効な範囲内であること。
  • r_R2(z) が呼び出され、現在の位置が R2 領域内にあることが検証されること。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置 z->c に設定する。 2. 文字列末尾から 'l' を探し、存在しなければ終了する。 3. カーソルを一つ戻し、z->bra を設定する。 4. r_R2(z) を呼び出し、現在の位置が R2 領域内にあるか確認する。条件を満たさない場合は終了する。 5. 再度、文字列末尾から 'l' を探し、存在しなければ終了する。 6. slice_del(z) を呼び出し、該当する 'l' を削除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_porter.c および src/backend/snowball/libstemmer/stem_ISO_8859_1_porter.c 内のメインステミング処理フローから参照されている。
  • 呼び出し先: r_R2（領域判定）、slice_del（文字列操作）。

関連データ構造

  • SN_env: ステミング処理中のカーソル位置、領域境界、文字列バッファを保持する構造体。

失敗モードと境界条件

  • 文字列が短すぎる場合や、末尾に 'l' が存在しない場合は 0 を返す。
  • r_R2 が失敗した場合、その戻り値をそのまま返す。
  • slice_del が失敗した場合、その負の戻り値を返す。
