JsonTableExecContext

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/jsonpath_exec.c:221

目的

JSON_TABLE 関数の実行時におけるプラン状態とコンテキスト情報を保持するための構造体です。

シグネチャまたは形状

typedef struct JsonTableExecContext
{
    int            magic;
    JsonTablePlanState *rootplanstate;
    JsonTablePlanState **colplanstates;
} JsonTableExecContext;

入力と出力

  • **入力**: palloc0 を用いてメモリ確保され、JsonTableInitPlan 等の関数を通じて初期化されます。
  • **出力**: JSON_TABLE の実行エンジンが、行セットの生成や列値の抽出を行う際の参照先として利用します。

前提条件と不変条件

  • magic フィールドは、構造体の健全性チェック（sanity checking）のための識別子として使用されます。
  • rootplanstate は、ルートの jsonpath から評価される行ソースのプラン状態を保持します。
  • colplanstates は、ネストされた列を含むすべての列に対する JsonTablePlanState の配列を保持します。

アルゴリズムまたは振る舞い

  • JSON_TABLE 実行時に、プランツリーの走査状態を管理します。
  • JsonTableInitPlan によりプランの初期化が行われ、実行に必要なコンテキストが構築されます。
  • 実行中、JsonTableExecContext は不透明なポインタ（opaque）として保持され、必要に応じて型キャストしてアクセスされます。

呼び出し元と呼び出し先

  • **呼び出し元**: JsonTableInitPlan、および JSON_TABLE 実行に関連する内部関数（src/backend/utils/adt/jsonpath_exec.c 内の 4085 行目以降のロジックなど）。
  • **呼び出し先**: JsonTablePlanState 構造体を参照し、プランの実行状態を管理します。

関連データ構造

  • JsonTablePlanState: JSON_TABLE のプランツリーの状態を管理する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • magic フィールドによる健全性チェックが失敗した場合、実行時エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は JSON_TABLE 実行専用であり、他の jsonpath 実行ロジックとは独立した状態管理を行います。
  • 構造体の変更時は、JsonTableInitPlan および opaque ポインタを介したアクセス箇所との整合性に注意してください。
