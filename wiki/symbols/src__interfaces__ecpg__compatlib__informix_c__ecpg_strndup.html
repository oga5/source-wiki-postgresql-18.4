ecpg_strndup

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/compatlib/informix.c:178

目的

指定された文字列 str から最大 len バイトまでの内容を複製し、新しいメモリ領域を確保して返却する。

シグネチャまたは形状

static char *ecpg_strndup(const char *str, size_t len)

入力と出力

  • **入力**:
  • str: 複製対象のヌル終端文字列。
  • len: 複製する最大バイト数。
  • **出力**:
  • 成功時: 複製された文字列を格納したメモリ領域へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • str は有効なヌル終端文字列であること。
  • 戻り値が NULL でない場合、呼び出し元は返却されたポインタに対して free() を実行する責任を負う。

アルゴリズムまたは振る舞い

1. 入力文字列 str の長さを strlen で取得する。 2. 実際にコピーする長さ use_len を、len と real_len の小さい方の値として決定する。 3. malloc を用いて use_len + 1 バイトのメモリを確保する。 4. メモリ確保に成功した場合、memcpy で str の内容をコピーし、末尾にヌル文字 \0 を付与する。 5. メモリ確保に失敗した場合、errno に ENOMEM を設定し、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/compatlib/informix.c 内の変換関数（例: deccvasc 等）から呼び出される。
  • **呼び出し先**: strlen, malloc, memcpy（標準 C ライブラリ関数）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **メモリ不足**: malloc が失敗した場合、errno に ENOMEM が設定され、関数は NULL を返す。
  • **長さの指定**: len が str の実際の長さより大きい場合、str 全体がコピーされる。len が小さい場合、指定された長さで切り詰められる。

保守担当者向けメモ

  • 本関数は static であり、informix.c 内でのみ使用されるユーティリティである。
  • Informix 互換の変換関数において、入力文字列を安全に複製するために利用されている。
