record_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rowtypes.c:822

目的

2つのレコード（HeapTupleHeader）を比較し、その大小関係または等価性を判定する内部関数です。record_lt、record_gt、record_eq などのSQL演算子の基盤として機能します。

シグネチャまたは形状

static int record_cmp(FunctionCallInfo fcinfo)

入力と出力

  • **入力**: FunctionCallInfo（比較対象の2つのレコードを引数として保持）。
  • **出力**: int 型の比較結果（0 は等価、-1 は第1引数が小さい、1 は第1引数が大きい）。

前提条件と不変条件

  • check_stack_depth() により、再帰的なレコード比較におけるスタックオーバーフローが防止されます。
  • 比較対象の列数が異なる場合や、列の型が一致しない場合はエラーとなります。
  • 削除された列（attisdropped）は比較対象から除外されます。
  • NULL 値は「NULL は非 NULL より大きい」というルールで扱われます（2つの NULL は等価とみなされます）。

アルゴリズムまたは振る舞い

1. **型情報の取得**: lookup_rowtype_tupdesc を使用して、各レコードの型定義（TupleDesc）を取得します。 2. **キャッシュの利用**: fcinfo->flinfo->fn_extra を使用し、比較に必要な型情報や比較関数ポインタをキャッシュします。 3. **タプルの分解**: heap_deform_tuple を用いて、レコードをフィールド配列に変換します。 4. **逐次比較**: 論理的な列順序に従い、各フィールドを順次比較します。

  • 型が異なる場合は ERRCODE_DATATYPE_MISMATCH でエラーを報告します。
  • 各フィールドの型に対応する比較関数を lookup_type_cache で取得し、再帰的に呼び出します。
  • 差異が見つかった時点で直ちに結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rowtypes.c 内の比較演算子実装（record_lt, record_gt, record_eq, record_ne, record_cmp のラッパー関数など）。
  • **呼び出し先**:
  • lookup_rowtype_tupdesc
  • heap_deform_tuple
  • lookup_type_cache
  • 各フィールド型に対応する比較関数（typentry->cmp_proc_finfo）

関連データ構造

  • RecordCompareData: 比較実行時の型情報やキャッシュを保持する構造体。
  • ColumnCompareData: 各列の比較関数情報を保持する構造体。
  • TupleDesc: レコードの構造定義。
  • HeapTupleHeader: レコードの内部表現。

失敗モードと境界条件

  • **型不一致**: 比較する列の型が異なる場合、ERRCODE_DATATYPE_MISMATCH を発生させます。
  • **比較関数未定義**: フィールド型に対して比較関数が見つからない場合、ERRCODE_UNDEFINED_FUNCTION を発生させます。
  • **スタック溢れ**: 深いネストを持つレコード比較時にスタック制限を超えるとエラーとなります。

保守担当者向けメモ

  • 本関数は fn_extra を介してキャッシュを保持するため、型定義が実行中に変更される可能性（ALTER TYPE 等）には注意が必要です。
  • 物理的な列位置ではなく論理的な列順序で比較が行われるため、attisdropped フラグの処理が重要です。
