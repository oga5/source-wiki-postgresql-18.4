OverridingKind

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/primnodes.h:26

目的

SQLの INSERT 文等における OVERRIDING 句の指定状態を表現するために使用されます。列の値をユーザーが指定するか、システムが生成するかを制御するフラグとして機能します。

シグネチャまたは形状

typedef enum OverridingKind
{
	OVERRIDING_NOT_SET = 0,
	OVERRIDING_USER_VALUE,
	OVERRIDING_SYSTEM_VALUE,
} OverridingKind;

入力と出力

  • 入力: SQL解析時に OVERRIDING SYSTEM VALUE または OVERRIDING USER VALUE 句が指定された場合、対応する列挙値が設定されます。
  • 出力: 後の書き換え（Rewrite）処理や実行計画生成において、列の挿入値を決定するための判断基準として参照されます。

前提条件と不変条件

  • OVERRIDING_NOT_SET がデフォルト値（0）です。
  • 相互排他的な状態として扱われます。

アルゴリズムまたは振る舞い

  • OVERRIDING_USER_VALUE: ユーザーが明示的に指定した値を優先します。
  • OVERRIDING_SYSTEM_VALUE: システムが生成した値を優先します。
  • OVERRIDING_NOT_SET: 句が指定されていない状態を示します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/rewrite/rewriteHandler.c: 書き換え処理において参照。
  • src/include/nodes/parsenodes.h: 解析ツリー内のノード（InsertStmt 等）のフィールドとして保持。
  • src/include/nodes/primnodes.h: 関連するノード定義内で保持。
  • 呼び出し先: 実行時の具体的な値決定ロジックについては、本ソース excerpt からは直接確認できません。

関連データ構造

  • parsenodes.h 内の各種ステートメントノード（InsertStmt 等）のフィールドとして組み込まれます。

失敗モードと境界条件

  • 句が指定されていない場合に OVERRIDING_NOT_SET が維持されることが前提です。不正な値が設定された場合の挙動については、本ソース excerpt には記載がありません。

保守担当者向けメモ

  • query_jumble_ignore 属性が付与されている箇所があるため、クエリのハッシュ化（Jumbling）時にはこの値が無視される設計となっています。変更を加える際は、クエリの正規化やキャッシュへの影響を考慮してください。
