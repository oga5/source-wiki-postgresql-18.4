WindowStatePerAggData

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeWindowAgg.c:107

目的

ウィンドウ関数として実行される集約関数の実行時状態を保持し、遷移関数や逆遷移関数を用いた集約計算を管理する。

シグネチャまたは形状

typedef struct WindowStatePerAggData {
    Oid transfn_oid, invtransfn_oid, finalfn_oid;
    FmgrInfo transfn, invtransfn, finalfn;
    int numFinalArgs;
    Datum initValue;
    bool initValueIsNull;
    Datum resultValue;
    bool resultValueIsNull;
    int16 inputtypeLen, resulttypeLen, transtypeLen;
    bool inputtypeByVal, resulttypeByVal, transtypeByVal;
    int wfuncno;
    MemoryContext aggcontext;
    Datum transValue;
    bool transValueIsNull;
    int64 transValueCount;
    bool restart;
} WindowStatePerAggData;

入力と出力

  • **入力**: pg_aggregate カタログから取得した初期値 (initValue) および遷移関数の OID。
  • **出力**: resultValue として計算結果を保持し、transValue として現在の集約状態を保持する。

前提条件と不変条件

  • transfn は必須だが、invtransfn および finalfn は InvalidOid の可能性がある。
  • FmgrInfo 構造体には、対応する OID が有効な場合にのみ関数呼び出しに必要な情報が格納される。
  • aggcontext は、遷移値や関連する補助データを保持するためのメモリコンテキストとして機能する。

アルゴリズムまたは振る舞い

  • eval_windowaggregates() 内で、フレームの移動に伴う状態更新を管理する。
  • transfn を用いて行データを集約し、invtransfn が定義されている場合はフレームの移動時に効率的な増分更新（逆遷移）を行う。
  • restart フラグにより、現在のサイクルで集約の再計算が必要かどうかを制御する。
  • 型の長さ (len) と値渡しフラグ (byVal) を保持し、データのコピーや削除を適切に処理する。

呼び出し元と呼び出し先

  • **呼び出し元**: initialize_peragg 関数により初期化され、WindowAggState 内で管理される。
  • **呼び出し先**: FmgrInfo を介して SQL の集約関数（遷移関数、逆遷移関数、最終計算関数）を呼び出す。
  • 補足: 具体的な関数呼び出しの連鎖については、ソースコードの断片からは詳細なコールグラフを特定できない。

関連データ構造

  • WindowStatePerFuncData: 関連付けられたウィンドウ関数の状態（wfuncno でインデックス参照）。
  • WindowAggState: ウィンドウ集約ノード全体の実行状態を保持する構造体。

失敗モードと境界条件

  • invtransfn が未定義の場合、フレーム移動時に再計算が発生し、パフォーマンスが低下する可能性がある。
  • work_mem を超えるデータ量はディスクへスピルするため、メモリ管理 (aggcontext) に依存する。

保守担当者向けメモ

  • 本構造体は nodeWindowAgg.c における集約処理の中核である。
  • 逆遷移関数 (invtransfn) の実装有無がパフォーマンスに直結するため、最適化の際はここを確認すること。
  • inputtype, resulttype, transtype に関する型情報は、メモリ管理やデータコピーの正確性に不可欠である。
