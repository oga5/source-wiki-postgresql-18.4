ColumnIOData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonfuncs.c:162

目的

JSON/JSONBデータとPostgreSQLの複合型（RowType）との相互変換において、各カラムの入出力メタデータをキャッシュするために使用されます。

シグネチャまたは形状

typedef struct ColumnIOData ColumnIOData;

※構造体の詳細なメンバ定義は、ソースコード上の宣言および contrib/hstore/hstore_io.c での利用状況から、入出力関数や型情報を含むメタデータ保持用として構成されています。

入力と出力

  • **入力**: 複合型の各フィールドに対する型情報、入出力関数ポインタ。
  • **出力**: 変換処理に必要なキャッシュ済みメタデータ。

前提条件と不変条件

  • jsonfuncs.c 内の変換処理（populate_record 等）において、型変換のオーバーヘッドを削減するためにキャッシュとして利用されます。
  • RecordIOData を直接 ColumnIOData.io 共用体内で使用することはできません。

アルゴリズムまたは振る舞い

  • 複合型の各フィールドに対して、一度型情報を解決した後に本構造体にキャッシュすることで、再帰的な変換処理時のルックアップコストを低減します。
  • contrib/hstore/hstore_io.c においても同様の構造体名が利用されており、ncolumns * sizeof(ColumnIOData) の形式でメモリ確保が行われるケースが確認されています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonfuncs.c 内の populate_record 関連関数。
  • **呼び出し先**: contrib/hstore/hstore_io.c において、my_extra->columns[i] を通じてカラム情報の参照が行われています。

関連データ構造

  • RecordIOData: 複合型全体の入出力状態を管理する構造体。
  • ScalarIOData: スカラー型の入出力メタデータ。

失敗モードと境界条件

  • メモリ確保時（ncolumns * sizeof(ColumnIOData)）のサイズ計算において、ncolumns が適切でない場合、メモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は jsonfuncs.c と hstore_io.c の両方で定義・利用されています。変更を加える際は、両モジュール間でのバイナリ互換性や構造の一貫性に注意してください。
  • ColumnIOData のメンバ定義は jsonfuncs.c の 162 行目以降のコンテキストで詳細を確認可能です。
