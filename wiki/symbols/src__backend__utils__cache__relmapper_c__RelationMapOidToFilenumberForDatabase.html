RelationMapOidToFilenumberForDatabase

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/cache/relmapper.c:264

目的

指定されたデータベースパスにあるマップファイル（pg_filenode.map）を読み込み、特定のカタログOIDに対応する物理ファイル番号（RelFileNumber）を解決する。

シグネチャまたは形状

RelFileNumber RelationMapOidToFilenumberForDatabase(char *dbpath, Oid relationId)

入力と出力

  • **入力**:
  • dbpath (char *): 対象データベースのディレクトリパス。
  • relationId (Oid): 検索対象のリレーションOID。
  • **出力**:
  • 成功時: 対応する RelFileNumber。
  • 失敗時: InvalidRelFileNumber（OIDが見つからない場合）。

前提条件と不変条件

  • read_relmap_file を呼び出し、指定されたパスからマップファイルを読み込む。
  • 読み込み時にエラーが発生した場合（ファイルが存在しない、破損している等）、read_relmap_file の仕様に従い ERROR が発生する可能性がある。

アルゴリズムまたは振る舞い

1. read_relmap_file を呼び出し、指定された dbpath のマップファイルを RelMapFile 構造体に読み込む。 2. 読み込んだマップ内の全エントリをループで走査する。 3. relationId と一致する mapoid を持つエントリが見つかった場合、対応する mapfilenumber を返す。 4. ループ終了まで一致するものがない場合、InvalidRelFileNumber を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c (データベース操作に関連する処理で参照)
  • **呼び出し先**:
  • read_relmap_file (内部関数)

関連データ構造

  • RelMapFile: マップファイルの内容を保持する構造体。
  • RelFileNumber: 物理ファイル番号を表す型。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • マップファイルが読み込めない場合、read_relmap_file によりエラーがスローされる。
  • 指定されたOIDがマップファイル内に存在しない場合は InvalidRelFileNumber を返す。

保守担当者向けメモ

  • 本関数は、現在アクティブなメモリ上のマップではなく、ディスク上のマップファイルを直接読み込むため、呼び出しコストが高い点に注意すること。
  • データベースのコピーや移動など、外部データベースのメタデータを参照する必要がある場面で使用される。
