sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/thread-descriptor.c:43

目的

ECPG（Embedded SQL for C）における SQL 通信領域（SQLCA: SQL Communication Area）を定義する構造体。SQL 文の実行結果、エラーコード、警告、および処理された行数などの診断情報を保持する。

シグネチャまたは形状

struct sqlca_t {
    char    sqlcaid[8];
    long    sqlabc;
    long    sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    } sqlerrm;
    char    sqlerrp[8];
    long    sqlerrd[6];
    char    sqlwarn[8];
    char    sqlstate[5];
};

入力と出力

  • **入力**: SQL 文の実行に伴い、ECPG ライブラリが本構造体の各フィールドを更新する。
  • **出力**: アプリケーションは sqlcode や sqlstate を参照してエラーハンドリングを行い、sqlerrd[2] から処理された行数を取得する。

前提条件と不変条件

  • sqlerrd[2] は INSERT, UPDATE, DELETE 文の実行後に処理された行数を格納する。
  • sqlwarn[0] は、他の警告フラグのいずれかが設定された場合に 'W' に設定される。

アルゴリズムまたは振る舞い

ECPG ライブラリ関数（ECPGallocate_desc 等）の実行後、ライブラリ内部でこの構造体が更新される。呼び出し側は、各 SQL 操作の直後に sqlcode が負の値でないかを確認することで、処理の成否を判定する。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはない。

関連データ構造

  • sqlda_t: SQL 記述子領域。
  • sqlvar_t: SQL 記述子内の変数定義。

失敗モードと境界条件

  • sqlcode が負の場合、SQL エラーが発生していることを示す。
  • sqlerrmc のバッファサイズ（SQLERRMC_LEN）を超えるエラーメッセージは切り詰められる可能性がある。

保守担当者向けメモ

  • 本構造体は ECPG の標準的なインターフェースであるため、定義の変更はライブラリ全体およびプリプロセッサの出力に影響を与える。
  • thread-descriptor.c 等のテストコードでは、マルチスレッド環境下での sqlca の整合性が検証されている。
