RemoveSQLFunctionCache

定義

  • 種別: function
  • 定義箇所: src/backend/executor/functions.c:2013

目的

SQL関数実行に関連付けられたキャッシュエントリの参照カウントを減らし、リソースの解放を管理する。

シグネチャまたは形状

static void RemoveSQLFunctionCache(void *arg)

入力と出力

  • **入力**: arg (SQL関数実行状態を保持する SQLFunctionCache 構造体へのポインタ)
  • **出力**: なし（副作用としてキャッシュエントリの参照カウントを更新）

前提条件と不変条件

  • arg は SQLFunctionCache 構造体としてキャスト可能であること。
  • fcache->func が存在する場合、その use_count は 0 より大きいこと（Assert により検証）。

アルゴリズムまたは振る舞い

1. 引数 arg を SQLFunctionCache 型にキャストする。 2. fcache->func が NULL でない場合、以下の処理を行う。

  • fcache->func->cfunc.use_count をデクリメントする。
  • fcache->func を NULL に設定し、二重解放や誤った参照を防止する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照情報に基づくと、src/backend/executor/functions.c 内で fcache->mcb.func に代入されており、メモリコンテキストのクリーンアップコールバックとして登録されていることが示唆される。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • SQLFunctionCache: 関数実行ごとの状態を保持する構造体。
  • SQLFunctionHashEntry: 関数定義に基づく長寿命なキャッシュエントリ。

失敗モードと境界条件

  • fcache->func が NULL の場合、参照カウントの操作はスキップされる。
  • use_count が 0 の状態で呼び出された場合、Assert が有効なビルドではプログラムが停止する。

保守担当者向けメモ

  • 本関数はメモリコンテキストの破棄時に呼び出されるコールバックとして設計されている。
  • SQLFunctionHashEntry のライフサイクル管理において、参照カウントの整合性を維持する重要な役割を担う。
