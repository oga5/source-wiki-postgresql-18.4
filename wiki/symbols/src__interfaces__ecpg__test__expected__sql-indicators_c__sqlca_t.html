sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-indicators.c:32

目的

ECPG（Embedded SQL for C）におけるSQL通信領域（SQL Communication Area）を定義する構造体。データベース操作の結果として返されるエラーコード、警告、処理行数などの診断情報を保持する。

シグネチャまたは形状

struct sqlca_t
{
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: ECPGランタイムライブラリがSQL文の実行結果を本構造体の各フィールドに書き込む。
  • **出力**: アプリケーション側で sqlcode や sqlstate を参照し、エラーハンドリングや処理結果の判定を行う。

前提条件と不変条件

  • sqlerrd[2] には、INSERT、UPDATE、DELETE 文実行後に処理された行数が格納される。
  • sqlwarn[1] が 'W' の場合、ホスト変数への格納時に文字列の切り捨てが発生したことを示す。
  • sqlstate[5] には標準SQL準拠のエラーコードが格納される。

アルゴリズムまたは振る舞い

ECPGプリプロセッサが生成するコードにおいて、SQL文の実行ごとにこの構造体が更新される。アプリケーションはこれを通じて、データベースとの通信状態や実行結果の成否を監視する。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGget_sqlca() を介して、src/interfaces/ecpg/ecpglib/ 配下の各関数（connect.c, data.c, descriptor.c 等）および src/interfaces/ecpg/compatlib/informix.c から参照・更新される。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはない。

関連データ構造

  • sqlda_t: 動的SQLで使用される記述子領域。
  • sqlvar_t: sqlda_t 内で使用される列定義構造体。

失敗モードと境界条件

  • sqlerrmc は SQLERRMC_LEN で定義されたサイズ制限があるため、長いエラーメッセージは切り捨てられる可能性がある。
  • sqlerrd 配列の各要素は特定の用途（OID、処理行数など）に固定されており、用途外のアクセスは無効な値を返す。

保守担当者向けメモ

  • 本構造体はECPGのランタイムライブラリと密接に結合している。定義を変更する場合は、ecpglib 側の整合性を必ず確認すること。
  • テストコード（sql-indicators.c 等）は、この構造体のフィールド値を検証することでECPGの動作を保証している。
