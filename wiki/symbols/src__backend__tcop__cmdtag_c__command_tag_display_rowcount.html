command_tag_display_rowcount

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/cmdtag.c:59

目的

指定されたコマンドタグが、完了メッセージにおいて処理行数の表示を必要とするかどうかを判定する。

シグネチャまたは形状

bool command_tag_display_rowcount(CommandTag commandTag)

入力と出力

  • **入力**: CommandTag 型の列挙値。
  • **出力**: 行数表示が必要な場合は true、不要な場合は false を返す。

前提条件と不変条件

  • commandTag は有効な CommandTag 列挙値である必要がある。
  • tag_behavior 配列が初期化済みであること。

アルゴリズムまたは振る舞い

1. グローバルな tag_behavior 配列に対し、引数 commandTag をインデックスとしてアクセスする。 2. 該当する CommandTagBehavior 構造体の display_rowcount メンバの値を参照し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/cmdtag.c:146 (BuildQueryCompletionString 内)
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない（tag_behavior 配列へのアクセスのみ）。

関連データ構造

  • CommandTag: コマンドの種類を識別する列挙型。
  • CommandTagBehavior: 各コマンドタグの属性を保持する構造体。
  • tag_behavior: 全コマンドタグの属性を格納する静的配列。

失敗モードと境界条件

  • commandTag が配列の範囲外である場合、未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は tag_behavior 配列の構造に強く依存している。新しいコマンドタグを追加する際は、src/include/tcop/cmdtaglist.h を更新することで tag_behavior が自動的に構成される。
