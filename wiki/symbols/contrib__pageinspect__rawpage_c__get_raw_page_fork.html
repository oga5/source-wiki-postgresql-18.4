get_raw_page_fork

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/rawpage.c:125

目的

指定されたリレーション、フォーク名、およびブロック番号に基づき、物理ストレージからデータベースページを読み出し、その内容を bytea 形式で取得します。

シグネチャまたは形状

Datum get_raw_page_fork(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relname (text): 対象リレーション名。
  • forkname (text): 対象フォーク名（例: 'main', 'fsm', 'vm'）。
  • blkno (uint32): 読み出すブロック番号。
  • **出力**:
  • bytea: 読み出されたページイメージ。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要です（rawpage.c の責務に基づく）。
  • forkname は有効なフォーク名として解決可能である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して、SQL 引数からリレーション名、フォーク名、ブロック番号を取得します。 2. forkname_to_number を呼び出し、文字列形式のフォーク名を内部的な ForkNumber 型に変換します。 3. get_raw_page_internal を呼び出し、指定されたリレーションのフォークおよびブロック番号からページデータを取得します。 4. 取得したページデータを PG_RETURN_BYTEA_P を通じて SQL レイヤーへ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（PG_FUNCTION_INFO_V1 による登録あり）。
  • **呼び出し先**:
  • forkname_to_number
  • get_raw_page_internal

関連データ構造

  • ForkNumber: フォークを識別するための列挙型。
  • bytea: PostgreSQL の可変長バイナリデータ型。

失敗モードと境界条件

  • 無効なリレーション名や存在しないブロック番号を指定した場合、get_raw_page_internal 経由でエラーが発生します。
  • 不正なフォーク名を指定した場合、forkname_to_number がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は pageinspect 拡張機能の一部であり、物理データへの直接アクセスを行うため、セキュリティ上の制約（スーパーユーザー権限）が適用されます。
  • get_raw_page_internal の実装変更が本関数の挙動に直接影響します。
