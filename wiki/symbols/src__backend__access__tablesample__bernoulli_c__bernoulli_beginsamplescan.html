bernoulli_beginsamplescan

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/bernoulli.c:135

目的

TABLESAMPLE BERNOULLI メソッドにおけるスキャン開始時の初期化処理を行い、サンプリングのカットオフ値とシード値を設定します。

シグネチャまたは形状

static void bernoulli_beginsamplescan(SampleScanState *node, Datum *params, int nparams, uint32 seed)

入力と出力

  • node: サンプリングスキャンの状態を保持する SampleScanState 構造体。
  • params: ユーザーが指定したサンプリング率（パーセンテージ）。
  • nparams: パラメータ数。
  • seed: サンプリングの再現性を確保するための乱数シード。
  • 出力: なし（node->tsm_state 内の BernoulliSamplerData を更新）。

前提条件と不変条件

  • params[0] に指定されたサンプリング率は 0 から 100 の範囲内である必要があります。範囲外または NaN の場合はエラーが発生します。
  • node->tsm_state は BernoulliSamplerData 型として初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. **パラメータ検証**: 指定されたパーセンテージが 0〜100 の範囲内かを確認します。 2. **カットオフ算出**: (PG_UINT32_MAX + 1) * percent / 100 を計算し、タプル選択の閾値（cutoff）を決定します。 3. **状態保存**: sampler->cutoff に算出値を、sampler->seed に引数のシードを格納します。また、sampler->lt を InvalidOffsetNumber に初期化します。 4. **スキャン最適化設定**:

  • 全ページ走査のため node->use_bulkread を true に設定します。
  • サンプリング率が 25% 以上の場合、効率化のため node->use_pagemode を有効にします。

呼び出し元と呼び出し先

  • 呼び出し元: tsm_bernoulli_handler 内で TsmRoutine のコールバックとして登録され、実行エンジンから呼び出されます。
  • 呼び出し先: DatumGetFloat4（マクロ）、rint（数学関数）、ereport（エラー報告）。

関連データ構造

  • BernoulliSamplerData: スキャン中の状態（カットオフ値、シード値、オフセット）を保持します。
  • SampleScanState: スキャン全体の実行状態を管理します。

失敗モードと境界条件

  • サンプリング率が 0 未満、100 超、または NaN の場合、ERRCODE_INVALID_TABLESAMPLE_ARGUMENT でエラー終了します。
  • サンプリング率が 0 または 100 の極端な値であっても、カットオフ計算により正しく動作するよう設計されています。

保守担当者向けメモ

  • node->use_pagemode の 25% という閾値は限定的な実験に基づくものです。パフォーマンスチューニングの際は、この値を調整対象として検討してください。
