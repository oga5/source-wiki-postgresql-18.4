patternsel_common

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_support.c:482

目的

パターンマッチング演算子（LIKE、正規表現等）に対するクエリの選択率（Selectivity）を推定する汎用ロジックを提供します。プランナが統計情報を基に、パターンマッチングがどの程度の行にヒットするかを予測するために使用されます。

シグネチャまたは形状

double patternsel_common(PlannerInfo *root, Oid oprid, Oid opfuncid, List *args, int varRelid, Oid collation, Pattern_Type ptype, bool negate);

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報。
  • oprid, opfuncid: 演算子および関数の識別子。
  • args: 演算子の引数リスト。
  • varRelid: 変数のリレーションID。
  • collation: 照合順序。
  • ptype: パターンマッチングの種類（Pattern_Type）。
  • negate: 否定演算子（NOT LIKE 等）かどうかを示すフラグ。
  • **出力**:
  • double: 推定された選択率（0.0 から 1.0 の範囲）。

前提条件と不変条件

  • 入力引数が「変数 op 定数」の形式である必要があります。それ以外の場合はデフォルトの推定値を返します。
  • 定数が NULL の場合は、演算子が厳密（strict）であると仮定し、0.0 を返します。
  • 右辺の定数は TEXTOID または BYTEAOID である必要があります。
  • 左辺の変数は TEXTOID, NAMEOID, BPCHAROID, BYTEAOID のいずれかである必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: 否定フラグに応じてデフォルトの選択率を設定します。 2. **検証**: 入力が「変数 op 定数」の形式でない場合、または型がサポート対象外の場合はデフォルト値を返します。 3. **演算子マッピング**: 左辺の型に応じて、対応する等価（=）、小なり（<）、大なりイコール（>=）演算子を選択します。 4. **接頭辞抽出**: pattern_fixed_prefix を呼び出し、パターンから固定接頭辞を抽出します。 5. **選択率計算**:

  • パターンが完全一致（Pattern_Prefix_Exact）の場合、var_eq_const を使用して等価演算子としての選択率を計算します。
  • それ以外の場合、ヒストグラムと最頻値（MCV）の統計情報を組み合わせて、パターン全体のヒット率を推定します。

呼び出し元と呼び出し先

  • **呼び出し元**: like_regex_support 等のサポート関数から呼び出されます。
  • **呼び出し先**: get_restriction_variable, ReleaseVariableStats, pattern_fixed_prefix, var_eq_const 等のユーティリティ関数を呼び出します。

関連データ構造

  • PlannerInfo: プランナの情報を保持する構造体。
  • Pattern_Type: パターンマッチングの種類を定義する列挙型。
  • VariableStatData: 変数の統計情報を保持する構造体。
  • Const: 定数ノード。

失敗モードと境界条件

  • サポートされていない型が渡された場合、デフォルトの選択率を返します。
  • 定数が NULL の場合、常に 0.0 を返します。
  • 統計情報（statsTuple）が有効でない場合、ヒストグラムを用いた詳細な推定は行われず、デフォルト値や簡易的な計算に依存します。

保守担当者向けメモ

  • 本関数は、SQL 呼び出し可能な関数と内部的な選択率推定ロジックの間の「インピーダンス不整合」を解消する役割を担っています。
  • prefix の型変換（TEXT から BPCHAR への変換など）が必要な場合、consttype を直接書き換える処理が含まれています。
  • 非 C ロケールにおける接頭辞生成の複雑なロジックは like_fixed_prefix 等に依存しているため、デバッグ時はそちらの挙動も確認してください。
