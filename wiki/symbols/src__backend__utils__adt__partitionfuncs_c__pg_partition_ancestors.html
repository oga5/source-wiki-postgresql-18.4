pg_partition_ancestors

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/partitionfuncs.c:200

目的

指定されたリレーション（パーティションまたはテーブル）からルート（最上位の親）までの全祖先リレーションの OID を列挙する。

シグネチャまたは形状

Datum pg_partition_ancestors(PG_FUNCTION_ARGS)

  • SQL から呼び出し可能な SRF（Set Returning Function）として実装。

入力と出力

  • **入力**: PG_GETARG_OID(0) により、対象リレーションの OID を取得。
  • **出力**: 祖先リレーションの OID を含む行セット。

前提条件と不変条件

  • check_rel_can_be_partition が真を返すリレーションであること。
  • 初回呼び出し時に multi_call_memory_ctx を使用して祖先リストを保持し、後続の呼び出しでそのリストを順次消費する。

アルゴリズムまたは振る舞い

1. **初回呼び出し**:

  • check_rel_can_be_partition で有効性を確認。無効な場合は終了。
  • get_partition_ancestors を呼び出し、祖先リストを取得。
  • 取得したリストの先頭に、入力された relid 自身を lcons_oid で追加。
  • リストを funcctx->user_fctx に格納。

    • 2. **各呼び出し**:

  • funcctx->user_fctx からリストを取得。
  • call_cntr がリスト長に達するまで、順次 OID を SRF_RETURN_NEXT で返却。
  • 全ての要素を返却後、SRF_RETURN_DONE で終了。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/bin/psql/describe.c（psql のメタコマンド等で使用）
  • **呼び出し先**:
  • check_rel_can_be_partition
  • get_partition_ancestors
  • lcons_oid
  • list_nth_oid

関連データ構造

  • FuncCallContext: SRF の状態保持に使用。
  • List: 祖先 OID の保持に使用。

失敗モードと境界条件

  • check_rel_can_be_partition が false を返した場合、結果セットは空となる。
  • 入力された OID がパーティションツリーに属さない場合、空の結果を返す。

保守担当者向けメモ

  • 本関数は SRF_IS_FIRSTCALL を用いた標準的な SRF 実装パターンに従っている。
  • 祖先リストの構築には catalog/partition.h で定義された関数群を利用しているため、カタログ構造の変更時には注意が必要。
