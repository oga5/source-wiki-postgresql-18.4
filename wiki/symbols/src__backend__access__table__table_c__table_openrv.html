table_openrv

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/table.c:82

目的

RangeVar で指定されたリレーションをオープンし、それがテーブルであることを検証した上で Relation 構造体を返します。

シグネチャまたは形状

Relation table_openrv(const RangeVar *relation, LOCKMODE lockmode);

入力と出力

  • **入力**:
  • relation: オープン対象のリレーションを指定する RangeVar 構造体へのポインタ。
  • lockmode: リレーション取得時に適用するロックモード（例: AccessShareLock 等）。
  • **出力**:
  • 指定されたリレーションの Relation 構造体ポインタ。

前提条件と不変条件

  • 指定された relation がシステム内に存在すること。
  • 呼び出し後、呼び出し元は適切なタイミングで table_close を呼び出し、リソースを解放する責任を負う。

アルゴリズムまたは振る舞い

1. relation_openrv(relation, lockmode) を呼び出し、リレーションをオープンして Relation 構造体を取得する。 2. validate_relation_kind(r) を呼び出し、取得したリレーションがテーブルとして有効か（インデックスや複合型ではないか）を検証する。 3. 検証を通過した Relation ポインタを呼び出し元へ返す。検証に失敗した場合は validate_relation_kind 内でエラーが発生する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • src/backend/bootstrap/bootstrap.c
  • src/backend/catalog/toasting.c
  • src/backend/commands/copy.c
  • src/backend/commands/publicationcmds.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • relation_openrv (低レベルなリレーションオープン)
  • validate_relation_kind (リレーションの型検証)

関連データ構造

  • Relation: オープンされたリレーションを表現する構造体。
  • RangeVar: リレーションを特定するための名前空間や名前を含む構造体。

失敗モードと境界条件

  • 指定された名前のリレーションが存在しない場合、relation_openrv がエラーを発生させる。
  • リレーションがテーブルではない（インデックスやシーケンス等）場合、validate_relation_kind が ereport(ERROR) を発行し、処理を中断する。

保守担当者向けメモ

  • 本関数は relation_openrv のラッパーであり、テーブルアクセスのためのガードレール（型検証）を追加する役割を持つ。
  • 実際のデータアクセスロジックは含まれていないため、AM（Access Method）固有の処理が必要な場合は tableam.c 側のインターフェースを参照すること。
