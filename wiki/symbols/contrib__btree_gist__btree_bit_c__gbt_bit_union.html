gbt_bit_union

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:164

目的

GiST インデックス構築時において、複数のビット列（BIT/VARBIT）エントリを統合し、それらを包含する最小の範囲（Union）を計算する。

シグネチャまたは形状

Datum gbt_bit_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *entryvec（統合対象のインデックスエントリ群）
  • PG_GETARG_POINTER(1): int32 *size（結果のサイズを格納する出力用ポインタ）
  • **出力**:
  • Datum: 統合されたビット列範囲を表すポインタ。

前提条件と不変条件

  • tinfo（gbtree_vinfo 構造体）が、BIT/VARBIT 型の処理に必要な比較・変換関数を保持している必要がある。
  • PG_GET_COLLATION() を介して適切な照合順序が渡されること。

アルゴリズムまたは振る舞い

1. entryvec と size を引数として受け取る。 2. btree_utils_var.c で定義された汎用関数 gbt_var_union を呼び出す。 3. その際、型固有の定義を含む tinfo と、現在の関数呼び出し情報 fcinfo->flinfo を渡す。 4. gbt_var_union が計算した結果をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの構築プロセス（GiST フレームワークから間接的に呼び出される）。
  • **呼び出し先**: gbt_var_union（btree_utils_var.c 内の汎用関数）。

関連データ構造

  • GistEntryVector: GiST インデックスのノードエントリを保持する構造体。
  • tinfo: gbtree_vinfo 型。BIT/VARBIT 型の GiST 処理を定義するテーブル。

失敗モードと境界条件

  • entryvec が空、または不正なメモリ領域を指している場合は、gbt_var_union の内部でエラーが発生する可能性がある。
  • メモリ不足時には PostgreSQL のメモリ管理機構によりエラーがスローされる。

保守担当者向けメモ

  • 本関数は gbt_var_union への単なるラッパーである。型固有のロジックを変更する場合は、tinfo の定義内容を確認すること。
  • PG_FUNCTION_INFO_V1(gbt_bit_union) により、PostgreSQL の関数マネージャに登録されている。
