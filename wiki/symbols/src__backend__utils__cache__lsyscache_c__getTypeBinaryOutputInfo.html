getTypeBinaryOutputInfo

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:3150

目的

指定されたデータ型のバイナリ出力関数（typsend）のOIDと、その型が可変長（varlena）であるかどうかの情報を取得する。

シグネチャまたは形状

void getTypeBinaryOutputInfo(Oid type, Oid *typSend, bool *typIsVarlena)

入力と出力

  • **入力**:
  • type: 情報を取得する対象のデータ型のOID。
  • typSend: 出力関数OIDを格納するためのポインタ。
  • typIsVarlena: 型が可変長かどうかを示すフラグを格納するためのポインタ。
  • **出力**:
  • なし（引数のポインタ経由で値を返却）。

前提条件と不変条件

  • type に対応する pg_type システムカタログのエントリが存在すること。
  • 該当する型が定義済み（typisdefined が真）であること。
  • 該当する型にバイナリ出力関数が定義されていること。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して TYPEOID キャッシュから対象型の HeapTuple を検索する。 2. キャッシュミスが発生した場合はエラーを報告する。 3. GETSTRUCT マクロで Form_pg_type 構造体を取得する。 4. 型がシェル型（未定義）である場合、またはバイナリ出力関数（typsend）が設定されていない場合はエラーを報告する。 5. pt->typsend を typSend に代入する。 6. typIsVarlena を (!pt->typbyval) && (pt->typlen == -1) の論理値で設定する。 7. ReleaseSysCache を呼び出し、参照カウントを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printtup.c
  • src/backend/commands/copyto.c
  • src/backend/commands/explain_dr.c
  • src/backend/tcop/fastpath.c
  • src/backend/utils/adt/array_userfuncs.c
  • src/backend/utils/adt/rowtypes.c
  • **呼び出し先**:
  • SearchSysCache1
  • ReleaseSysCache
  • elog / ereport

関連データ構造

  • Form_pg_type: pg_type システムカタログの行構造体。

失敗モードと境界条件

  • **キャッシュミス**: SearchSysCache1 が有効なタプルを返さない場合、elog(ERROR) で停止する。
  • **未定義型**: typisdefined が偽の場合、ERRCODE_UNDEFINED_OBJECT でエラーを報告する。
  • **関数未定義**: typsend が無効なOIDの場合、ERRCODE_UNDEFINED_FUNCTION でエラーを報告する。

保守担当者向けメモ

  • 本関数は pg_type キャッシュを直接操作するラッパーである。
  • typIsVarlena の判定ロジック（!typbyval && typlen == -1）は、PostgreSQLにおける可変長型の標準的な定義に基づいている。
