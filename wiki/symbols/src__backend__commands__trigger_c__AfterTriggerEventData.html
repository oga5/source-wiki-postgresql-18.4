AfterTriggerEventData

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/trigger.c:3707

目的

AFTER トリガーの実行イベントをキューイングする際に、個々のイベントに関連するタプル情報やパーティション情報を保持するために使用されます。

シグネチャまたは形状

typedef struct AfterTriggerEventData
{
    TriggerFlags ate_flags;     /* ステータスビットおよび共有データへのオフセット */
    ItemPointerData ate_ctid1;  /* 挿入、削除、または更新前のタプル */
    ItemPointerData ate_ctid2;  /* 更新後の新しいタプル */
    Oid             ate_src_part; /* ソースパーティションの OID */
    Oid             ate_dst_part; /* 宛先パーティションの OID */
} AfterTriggerEventData;

入力と出力

  • **入力**: トリガー実行時に ExecBRInsertTriggers 等の実行エンジンから、タプルの物理位置（ItemPointerData）やパーティション OID が設定されます。
  • **出力**: トランザクション終了時や SET CONSTRAINTS 実行時に、キューから読み出され、トリガー関数の実行コンテキストとして利用されます。

前提条件と不変条件

  • ate_ctid1 および ate_ctid2 は、対象となるテーブル内のタプルを指す有効な ItemPointer である必要があります。
  • パーティションテーブルの更新時には、ate_src_part と ate_dst_part に正しいパーティション OID が格納されている必要があります。

アルゴリズムまたは振る舞い

  • AfterTriggerSaveEvent を通じてキューに保存されます。
  • パーティションテーブルの更新時には、ソースと宛先のパーティションを跨ぐ処理をサポートするため、それぞれの OID を保持します。
  • 実行時には、これらのデータをもとに AfterTriggerFireDeferred がトリガー関数を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/trigger.c 内のトリガーイベント生成処理（AfterTriggerSaveEvent 等）。
  • **呼び出し先**: この構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。この構造体を参照する関数は src/backend/commands/trigger.c 内に存在します。

関連データ構造

  • TriggerFlags: イベントの状態管理に使用。
  • ItemPointerData: タプルの物理位置を特定するために使用。
  • AfterTriggerEvent: AfterTriggerEventData へのポインタ型。

失敗モードと境界条件

  • メモリ不足: 大量にトリガーイベントが発生した場合、キューのメモリ管理に依存します。
  • 無効な OID: パーティションテーブルの更新時に、存在しないパーティション OID が指定された場合、タプル取得時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/commands/trigger.c 内で定義されており、ate_src_part や ate_dst_part はパーティションテーブルの更新時にのみ利用されます。
  • ate_ctid1 や ate_ctid2 の解釈は、トリガーの種類（INSERT/UPDATE/DELETE）に応じて変化するため、処理を追加する際は TriggerFlags の状態ビットを確認してください。
