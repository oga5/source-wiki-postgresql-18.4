pg_checksum_final

定義

  • 種別: function
  • 定義箇所: src/common/checksum_helper.c:175

目的

チェックサム計算の最終化を行い、計算結果をバッファに書き込みます。SHA系アルゴリズムの場合は、計算完了後に内部コンテキストのリソース解放も行います。

シグネチャまたは形状

int pg_checksum_final(pg_checksum_context *context, uint8 *output);

入力と出力

  • context: チェックサム計算の状態を保持する pg_checksum_context へのポインタ。
  • output: 計算結果を格納するバッファへのポインタ。
  • 戻り値: 成功時は書き込まれたバイト数（int）、失敗時は -1 を返します。

前提条件と不変条件

  • context は pg_checksum_init によって初期化済みであること。
  • SHA系アルゴリズムを使用する場合、本関数を呼び出すことで内部コンテキストが解放されるため、呼び出し後の再利用はできません。
  • 出力バッファサイズが PG_CHECKSUM_MAX_LENGTH を超えないことがコンパイル時に保証されています。

アルゴリズムまたは振る舞い

1. context->type に基づき、以下の処理を分岐します。

  • CHECKSUM_TYPE_NONE: 何も行いません。
  • CHECKSUM_TYPE_CRC32C: FIN_CRC32C マクロで計算を完了し、結果を output にコピーします。
  • CHECKSUM_TYPE_SHA224/256/384/512: pg_cryptohash_final を呼び出して計算を完了し、成功時に pg_cryptohash_free でコンテキストを解放します。

    • 2. 最後に、書き込まれたバイト数が PG_CHECKSUM_MAX_LENGTH 以下であることを Assert で検証します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/backup/backup_manifest.c
  • src/bin/pg_combinebackup/backup_label.c
  • src/bin/pg_combinebackup/pg_combinebackup.c
  • src/bin/pg_combinebackup/reconstruct.c
  • src/bin/pg_combinebackup/write_manifest.c
  • src/bin/pg_verifybackup/astreamer_verify.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • 呼び出し先:
  • pg_cryptohash_final (SHA系の場合)
  • pg_cryptohash_free (SHA系の場合)

関連データ構造

  • pg_checksum_context: アルゴリズムの種類と内部状態を保持する構造体。

失敗モードと境界条件

  • pg_cryptohash_final が負の値を返した場合、本関数は -1 を返して終了します。

保守担当者向けメモ

  • SHA系アルゴリズムを使用する場合、メモリリークを防ぐため必ず本関数を呼び出してリソースを解放してください。
  • 出力バッファの最大長は PG_CHECKSUM_MAX_LENGTH で定義されており、StaticAssertDecl によりコンパイル時に安全性が担保されています。
