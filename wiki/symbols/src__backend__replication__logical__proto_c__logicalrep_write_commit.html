logicalrep_write_commit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/proto.c:77

目的

論理レプリケーションにおけるトランザクションのコミット情報を、ネットワーク転送可能なバイナリ形式にシリアライズする。

シグネチャまたは形状

void logicalrep_write_commit(StringInfo out, ReorderBufferTXN *txn, XLogRecPtr commit_lsn)

入力と出力

  • **入力**:
  • out: シリアライズ結果を格納する StringInfo バッファ。
  • txn: コミット対象のトランザクション情報を含む ReorderBufferTXN 構造体。
  • commit_lsn: コミット時のログシーケンス番号（XLogRecPtr）。
  • **出力**: なし（out バッファにバイナリデータが追記される）。

前提条件と不変条件

  • out は初期化済みであること。
  • txn は有効なポインタであり、end_lsn および xact_time.commit_time が適切に設定されていること。

アルゴリズムまたは振る舞い

1. LOGICAL_REP_MSG_COMMIT メッセージタイプ識別子をバッファに書き込む。 2. 現在未使用のフラグフィールド（0）を書き込む。 3. 以下の順序でデータをバッファに書き込む:

  • commit_lsn (int64)
  • txn->end_lsn (int64)
  • txn->xact_time.commit_time (int64)

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/pgoutput/pgoutput.c:654 から呼び出されていることが確認されている。
  • **呼び出し先**: libpq/pqformat.h で定義された pq_sendbyte および pq_sendint64 を使用してバッファ操作を行う。

関連データ構造

  • StringInfo: 通信用の可変長バッファ。
  • ReorderBufferTXN: トランザクションの状態を保持する構造体。
  • XLogRecPtr: WAL上の位置を示す型。

失敗モードと境界条件

  • StringInfo バッファのメモリ不足が発生した場合、pq_send* 関数群を通じてエラーが送出される可能性がある。

保守担当者向けメモ

  • 本関数はプロトコルの一部であり、変更時は logicalproto.h の定義および対応するデシリアライズ関数（logicalrep_read_commit）と整合性を保つこと。
  • 現在フラグフィールドは未使用だが、将来的な拡張のために予約されている。
