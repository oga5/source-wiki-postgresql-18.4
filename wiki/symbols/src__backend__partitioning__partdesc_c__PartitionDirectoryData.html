PartitionDirectoryData

定義

  • 種別: struct
  • 定義箇所: src/backend/partitioning/partdesc.c:35

目的

クエリ実行中に複数のリレーションに対するパーティション記述子（PartitionDesc）を一括管理し、一貫性を保証するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct PartitionDirectoryData
{
	MemoryContext pdir_mcxt;
	HTAB	   *pdir_hash;
	bool		omit_detached;
} PartitionDirectoryData;

入力と出力

  • **入力**: 構造体生成時にメモリコンテキストおよびデタッチ済みパーティションを除外するかどうかのフラグが設定されます。
  • **出力**: 内部のハッシュテーブル（pdir_hash）を通じて、リレーションOIDに対応するパーティション記述子エントリを提供します。

前提条件と不変条件

  • pdir_mcxt は、ディレクトリに関連付けられたメモリコンテキストであり、ディレクトリの生存期間を管理します。
  • pdir_hash は、リレーションOIDをキーとして PartitionDirectoryEntry を格納するハッシュテーブルです。

アルゴリズムまたは振る舞い

  • クエリ実行中に必要となるパーティション情報をキャッシュし、同一リレーションに対する重複したカタログ読み取りを抑制します。
  • omit_detached フラグに基づき、デタッチ処理中のパーティションを動的に除外する制御を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partdesc.c 内の CreatePartitionDirectory 関数にて、palloc を用いて本構造体のメモリが確保されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PartitionDirectory: 本構造体へのポインタ型（typedef struct PartitionDirectoryData *PartitionDirectory）。
  • PartitionDirectoryEntry: ハッシュテーブル内に格納される個別のエントリ。
  • HTAB: utils/hsearch.h で定義されるハッシュテーブル構造。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • ハッシュテーブルの初期化失敗や、エントリ検索時のカタログ不整合については、上位の RelationBuildPartitionDesc 等の再試行ロジックに依存します。

保守担当者向けメモ

  • 本構造体は CacheMemoryContext とは独立したライフサイクルを持つことが可能です。クエリ実行終了時に適切に破棄されるよう注意してください。
  • omit_detached の扱いは、並行DDL実行時の整合性に直結するため、変更時は partdesc.c のキャッシュ無効化ロジックと照らし合わせてください。
