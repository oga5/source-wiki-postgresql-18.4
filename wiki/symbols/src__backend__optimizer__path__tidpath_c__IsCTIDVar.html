IsCTIDVar

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/tidpath.c:55

目的

指定された Var ノードが、対象リレーションの物理的なタプル識別子（ctid）を参照しているかどうかを判定します。

シグネチャまたは形状

static inline bool IsCTIDVar(Var *var, RelOptInfo *rel)

入力と出力

  • **入力**:
  • var: 判定対象の Var ノードへのポインタ。
  • rel: 比較対象となるリレーションの RelOptInfo 構造体へのポインタ。
  • **出力**:
  • bool: 対象が ctid 列であれば true、そうでなければ false を返します。

前提条件と不変条件

  • var は有効な Var ノードである必要があります。
  • rel は var が属するリレーションの情報を保持している必要があります。

アルゴリズムまたは振る舞い

以下のすべての条件を満たす場合に true を返します。 1. var->varattno が SelfItemPointerAttributeNumber（ctid の属性番号）と一致する。 2. var->vartype が TIDOID である。 3. var->varno が rel->relid と一致する。 4. var->varnullingrels が NULL である。 5. var->varlevelsup が 0 である（現在のクエリレベルで定義されている）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/tidpath.c 内の各関数（TidQualFromRestrictInfoList 等）から、式が ctid を参照しているかの確認に使用。
  • src/backend/executor/nodeTidscan.c および src/backend/executor/nodeTidrangescan.c から、実行時の引数チェックに使用。
  • **呼び出し先**:
  • 本関数は static inline であり、他の関数を呼び出しません。

関連データ構造

  • Var: PostgreSQLの式ツリーにおける変数ノード。
  • RelOptInfo: プランナが保持するリレーションの最適化情報。
  • SelfItemPointerAttributeNumber: ctid を指すシステム属性番号。

失敗モードと境界条件

  • var が NULL の場合、または rel が NULL の場合はセグメンテーションフォールトが発生する可能性があります。
  • 外部クエリレベルを参照している Var（varlevelsup > 0）は、この関数では false と判定されます。

保守担当者向けメモ

  • vartype のチェックは安全策として実装されています。
  • varnullingrels や varlevelsup のチェックは、現在のクエリコンテキストにおける正確な ctid 参照を保証するために重要です。
