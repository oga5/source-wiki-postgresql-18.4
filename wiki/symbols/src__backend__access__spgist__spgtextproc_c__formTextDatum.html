formTextDatum

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgtextproc.c:112

目的

指定されたバイト列から、PostgreSQL の text 型として有効な Datum を生成する。

シグネチャまたは形状

static Datum formTextDatum(const char *data, int datalen)

入力と出力

  • **入力**:
  • data: 格納する文字列データへのポインタ。
  • datalen: 文字列の長さ（バイト単位）。
  • **出力**:
  • Datum: 生成された text 型のデータへのポインタを保持する Datum 型の値。

前提条件と不変条件

  • palloc を使用してメモリを確保するため、呼び出し元は適切なメモリコンテキストにいる必要がある。
  • datalen が 0 の場合、空の text 型データが生成される。

アルゴリズムまたは振る舞い

1. datalen + VARHDRSZ 分のメモリを palloc で確保する。 2. データ長に応じて、以下のいずれかの形式で text 型のヘッダーを構築する。

  • datalen + VARHDRSZ_SHORT が VARATT_SHORT_MAX 以下の場合: ショートヘッダー形式（SET_VARSIZE_SHORT）を使用し、データをコピーする。
  • それ以外の場合: 通常のヘッダー形式（SET_VARSIZE）を使用し、データをコピーする。

    • 3. 確保したメモリ領域を PointerGetDatum で Datum 型に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgtextproc.c 内の複数箇所（インデックス構築および検索処理）。
  • src/test/modules/spgist_name_ops/spgist_name_ops.c 内の複数箇所。
  • **呼び出し先**:
  • palloc (メモリ確保)
  • SET_VARSIZE_SHORT / SET_VARSIZE (ヘッダー設定マクロ)
  • memcpy (データコピー)
  • PointerGetDatum (型変換)

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ表現型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • data が NULL で datalen が 0 の場合、ヘッダーのみが初期化された空の text データが生成される。

保守担当者向けメモ

  • 本関数は text 型の内部表現（ヘッダーの圧縮形式）を自動的に判定するため、呼び出し側でヘッダーサイズを意識する必要はない。
  • spgtextproc.c 内の他の関数から頻繁に呼び出されるため、パフォーマンスに影響を与える可能性がある。
