decodePageSplitRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistxlog.c:222

目的

WALレコードのデータ領域から、GiSTインデックスのページ分割時に格納された複数の IndexTuple をデコードして復元します。

シグネチャまたは形状

static IndexTuple *decodePageSplitRecord(char *begin, int len, int *n)

入力と出力

  • **入力**:
  • begin: デコード対象のデータ領域の先頭ポインタ。
  • len: データ領域の総バイト数。
  • n: 復元されたタプル数を格納するための出力用ポインタ。
  • **出力**:
  • 復元された IndexTuple 配列へのポインタを返します。メモリは palloc で確保されます。

前提条件と不変条件

  • begin から始まる領域の先頭には、タプル数を示す int 型の値が格納されている必要があります。
  • 処理終了時、ptr が指す位置と begin の差分が len と一致すること（Assert(ptr - begin == len)）が保証されます。

アルゴリズムまたは振る舞い

1. begin の先頭からタプル数 n を読み取ります。 2. n に基づいて IndexTuple 配列をメモリ確保します。 3. ループ内で各タプルのサイズを IndexTupleSize で取得し、ポインタを順次進めながら配列に格納します。 4. 各ステップで、現在のポインタがデータ領域の範囲内にあるかを Assert で検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistxlog.c 内の gistRedoPageSplitRecord から呼び出されます。
  • **呼び出し先**: IndexTupleSize マクロを使用します。

関連データ構造

  • IndexTuple: GiSTインデックスの各エントリを表す構造体。

失敗モードと境界条件

  • Assert による検証を行っているため、デバッグビルドにおいてデータ領域のサイズとタプル構造が不整合な場合、プログラムが停止します。
  • palloc を使用しているため、メモリ不足時にはエラーが発生します。

保守担当者向けメモ

  • 本関数はWALリプレイ時にのみ使用されます。
  • IndexTuple のポインタは begin が指すメモリ領域を直接参照しているため、呼び出し元で begin が指すメモリが解放される前に本関数の戻り値を利用する必要があります。
