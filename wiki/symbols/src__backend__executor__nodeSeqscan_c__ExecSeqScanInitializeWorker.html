ExecSeqScanInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:398

目的

並列クエリ実行時、各ワーカープロセスにおいてシーケンシャルスキャンの状態を初期化し、共有メモリ上のスキャン記述子に接続する。

シグネチャまたは形状

void ExecSeqScanInitializeWorker(SeqScanState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の SeqScanState 構造体。
  • pwcxt: 並列ワーカーのコンテキスト情報（共有メモリのTOCを含む）。
  • **出力**: なし（node->ss.ss_currentScanDesc にスキャン記述子が設定される）。

前提条件と不変条件

  • pwcxt が有効であり、node に対応するプランノードIDの並列スキャン記述子が共有メモリ（TOC）内に存在すること。

アルゴリズムまたは振る舞い

1. pwcxt->toc から、現在のプランノードID (node->ss.ps.plan->plan_node_id) をキーとして ParallelTableScanDesc を検索する。 2. 取得した ParallelTableScanDesc を使用して、table_beginscan_parallel を呼び出し、テーブルの並列スキャンを開始する。 3. 生成されたスキャン記述子を node->ss.ss_currentScanDesc に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1343（並列ワーカーの初期化プロセスから呼び出される）。
  • **呼び出し先**:
  • shm_toc_lookup（共有メモリからのデータ検索）
  • table_beginscan_parallel（テーブルアクセスメソッドによるスキャン開始）

関連データ構造

  • SeqScanState: シーケンシャルスキャンの実行状態を保持する構造体。
  • ParallelWorkerContext: 並列ワーカーの実行コンテキスト。
  • ParallelTableScanDesc: 並列スキャンにおける共有状態。

失敗モードと境界条件

  • shm_toc_lookup が指定されたIDを見つけられない場合、false 引数によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は並列ワーカー側でのみ実行される。リーダープロセス側での初期化は ExecSeqScanInitializeDSM が担当する。
