AlterOpFamily

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:816

目的

ALTER OPERATOR FAMILY SQLコマンドの実行エンジンとして、演算子ファミリーに対する演算子やサポート関数の追加（ADD）または削除（DROP）を処理する。

シグネチャまたは形状

Oid AlterOpFamily(AlterOpFamilyStmt *stmt)

入力と出力

  • **入力**: AlterOpFamilyStmt *stmt（解析済みのSQLステートメント構造体）
  • **出力**: Oid（変更対象となった演算子ファミリーのOID）

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要である。
  • 指定されたアクセスメソッド（AM）および演算子ファミリーが存在しなければならない。
  • 演算子番号およびサポート関数番号は、対象AMの定義する範囲内である必要がある。

アルゴリズムまたは振る舞い

1. AMNAME システムキャッシュを検索し、アクセスメソッドのOIDとルーチン情報を取得する。 2. get_opfamily_oid を呼び出し、対象となる演算子ファミリーのOIDを特定する。 3. スーパーユーザー権限を確認し、不足している場合はエラーを投げる。 4. stmt->isDrop フラグに基づき、以下のいずれかを呼び出す：

  • AlterOpFamilyDrop: メンバーの削除処理。
  • AlterOpFamilyAdd: メンバーの追加処理。

    • 5. 処理完了後、演算子ファミリーのOIDを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQLコマンド実行のディスパッチャ）
  • **呼び出し先**:
  • SearchSysCache1, ReleaseSysCache
  • GetIndexAmRoutineByAmId
  • get_opfamily_oid
  • AlterOpFamilyAdd / AlterOpFamilyDrop

関連データ構造

  • AlterOpFamilyStmt: SQL解析結果を保持する構造体。
  • IndexAmRoutine: アクセスメソッドの機能ルーチン定義。
  • Form_pg_am: pg_am システムカタログのタプル構造。

失敗モードと境界条件

  • **ERRCODE_UNDEFINED_OBJECT**: 指定されたアクセスメソッドが存在しない場合。
  • **ERRCODE_INSUFFICIENT_PRIVILEGE**: スーパーユーザー権限がない場合。
  • 演算子番号がアクセスメソッドの許容範囲外である場合、後続の Add/Drop 関数内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は ADD と DROP の分岐点であり、実際のカタログ更新ロジックは AlterOpFamilyAdd および AlterOpFamilyDrop に委譲されている。
  • スーパーユーザー権限のチェックを解除する場合、関連するセキュリティ上の懸念事項（NOT_USED と記されたコードセクション）を再検討する必要がある。
