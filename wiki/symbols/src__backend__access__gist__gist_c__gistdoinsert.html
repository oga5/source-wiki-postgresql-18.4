gistdoinsert

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:638

目的

GiSTインデックスにおいて、ルートからリーフまでツリーを降下し、指定されたタプルを適切な位置に挿入するメインループを実行します。

シグネチャまたは形状

void gistdoinsert(Relation r, IndexTuple itup, Size freespace, GISTSTATE *giststate, Relation heapRel, bool is_build)

入力と出力

  • **入力**:
  • r: 対象のインデックスリレーション。
  • itup: 挿入するインデックスタプル。
  • freespace: ページ分割の判断に使用する空き容量。
  • giststate: GiSTの実行状態管理構造体。
  • heapRel: ヒープリレーション。
  • is_build: インデックス構築中か否かを示すフラグ。
  • **出力**: なし（副作用としてインデックスページへの挿入およびツリー構造の更新を行う）。

前提条件と不変条件

  • 挿入処理は常にルートノード（GIST_ROOT_BLKNO）から開始されます。
  • ツリー降下中、並行処理によるページ分割や削除が発生した場合は、親ノードへ戻って再試行する仕組みが維持されます。
  • インデックス構築中（is_build が真）は、WALログの記録が省略されます。

アルゴリズムまたは振る舞い

1. **初期化**: 挿入状態（GISTInsertState）を初期化し、ルートノードをスタックの起点とします。 2. **ツリー降下ループ**:

  • retry_from_parent フラグが立っている場合、親ノードへ戻り再試行します。
  • ページを読み込み、共有ロックを取得して整合性を確認します。
  • 並行処理による分割や削除が検出された場合、親ノードへ戻り再試行します。
  • **内部ノードの場合**: gistchoose で最小ペナルティのパスを選択し、必要に応じて gistgetadjusted でキーを更新し、gistinserttuple で再帰的に挿入を試みます。
  • **リーフノードの場合**: 適切な位置にタプルを挿入します（ソースコードの続きで定義）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c:190 (gistinsert)
  • src/backend/access/gist/gistbuild.c:865
  • **呼び出し先**:
  • gistcheckpage, gistfixsplit, gistchoose, gistgetadjusted, gistinserttuple
  • ReadBuffer, LockBuffer, BufferGetPage, PageGetLSN, BufferGetLSNAtomic 等のストレージ管理関数。

関連データ構造

  • GISTSTATE: GiSTの実行状態。
  • GISTInsertStack: ツリー降下中のパスを保持するスタック構造。
  • IndexTuple: 挿入対象のインデックスデータ。

失敗モードと境界条件

  • **無効なタプル**: 内部ノードに古い形式の無効なタプルが存在する場合、エラー（ereport(ERROR, ...)）を発生させます。
  • **並行競合**: ページ分割や削除が進行中の場合、ロックを解放して親ノードから再試行します。
  • **ページ溢れ**: gistinserttuple 内でページ分割が発生した場合、親ノードへのダウンリンク挿入が必要となります。

保守担当者向けメモ

  • 本関数は共有ロックから排他ロックへの昇格を伴うため、デッドロックを避けるための慎重なロック管理が行われています。
  • is_build が真の場合、LSNの代わりに GistBuildLSN が使用される特殊なパスを通る点に注意してください。
  • ページ分割の再帰的な挙動を理解するには、gistfinishsplit および gistinserttuple の実装と併せて確認してください。
