hstoreValidOldFormat

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_compat.c:170

目的

与えられた HStore データが旧形式（hstore-old）として妥当であるかを検証し、その適合度を判定します。

シグネチャまたは形状

static int hstoreValidOldFormat(HStore *hs)

入力と出力

  • **入力**: HStore *hs（検証対象のデータ構造）
  • **出力**: int 型のステータスコード
  • 0: 旧形式として無効
  • 1: 旧形式として妥当だが、サイズに不一致がある（変換が必要な可能性）
  • 2: 旧形式として完全に妥当

前提条件と不変条件

  • sizeof(HOldEntry) が 2 * sizeof(HEntry) と等しいこと（コンパイル時に StaticAssertStmt で検証）。
  • hs->size_ に HS_FLAG_NEWVERSION フラグが立っていないこと。

アルゴリズムまたは振る舞い

1. HS_FLAG_NEWVERSION フラグを確認し、立っていれば即座に 0 を返します。 2. 要素数 count が 0 の場合は 2 を返します。 3. count が上限（0xFFFFFFF）を超えている、または計算上のデータサイズが実際の VARSIZE(hs) を超える場合は 0 を返します。 4. 最初の要素のオフセットが 0 であることを確認します。 5. 各要素のキー長（keylen）が非減少であることを確認します。 6. 各要素のオフセット（pos）が、前の要素の終了位置と一致しているか（連続性）を確認します。 7. 最終的な計算サイズと実際の VARSIZE(hs) を比較し、一致すれば 2、異なれば 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_compat.c 内の関数（hstoreUpgrade 等）から呼び出されます。
  • **呼び出し先**: ARRPTR マクロ、CALCDATASIZE マクロ、VARSIZE マクロを使用します。

関連データ構造

  • HStore: 検証対象のメイン構造体。
  • HOldEntry: 旧形式で使用されるエントリ構造体。

失敗モードと境界条件

  • count が負の値や過大な値の場合、またはメモリレイアウトが期待される連続性を満たさない場合に 0 を返します。
  • VARSIZE と計算上のサイズが一致しない場合、形式は妥当と見なされますが、変換が必要な状態（1）として扱われます。

保守担当者向けメモ

  • 本関数は旧形式のメモリレイアウトに強く依存しています。プラットフォーム間で HOldEntry のサイズが異なる場合、動作が保証されません。
  • 戻り値の 1 は、データが旧形式として解釈可能であるものの、現在のメモリサイズと乖離があることを示しており、後続の変換処理のトリガーとなります。
