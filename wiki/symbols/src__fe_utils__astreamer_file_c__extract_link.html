extract_link

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_file.c:358

目的

ファイルシステム上にシンボリックリンクを作成します。

シグネチャまたは形状

static void extract_link(const char *filename, const char *linktarget)

入力と出力

  • filename: 作成するシンボリックリンクのパス（入力）。
  • linktarget: シンボリックリンクが指し示すターゲットパス（入力）。
  • 戻り値: なし（成功時）。失敗時は pg_fatal を呼び出しプロセスを終了します。

前提条件と不変条件

  • POSIX環境における symlink() システムコールが利用可能であること。
  • filename および linktarget は有効な文字列ポインタであること。

アルゴリズムまたは振る舞い

1. symlink(linktarget, filename) を実行し、linktarget を指す filename という名前のシンボリックリンクを作成します。 2. symlink の戻り値が 0 以外（エラー）の場合、pg_fatal を呼び出し、エラーメッセージとシステムエラー情報（%m）を出力してプログラムを終了します。

呼び出し元と呼び出し先

  • 呼び出し元: src/fe_utils/astreamer_file.c 内の astreamer_extractor_content（255行目付近）。
  • 呼び出し先: unistd.h で定義される symlink() システムコール。

関連データ構造

  • なし。

失敗モードと境界条件

  • symlink システムコールが失敗した場合（権限不足、パスの存在、ファイルシステム制限など）、pg_fatal により即座に実行が停止されます。

保守担当者向けメモ

  • 本関数は astreamer_extractor によるアーカイブ展開処理の一部として使用されます。
  • エラー発生時の挙動は pg_fatal に依存しており、呼び出し側で個別にエラーハンドリングを行う必要はありません。
