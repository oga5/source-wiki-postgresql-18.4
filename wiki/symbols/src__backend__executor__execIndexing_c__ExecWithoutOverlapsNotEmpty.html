ExecWithoutOverlapsNotEmpty

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execIndexing.c:1154

目的

WITHOUT OVERLAPS 制約が定義されたカラムに対し、空（empty）の範囲型またはマルチ範囲型値が挿入・更新されることを防ぎ、エラーを発生させる。

シグネチャまたは形状

static void ExecWithoutOverlapsNotEmpty(Relation rel, NameData attname, Datum attval, char typtype, Oid atttypid)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • attname: チェック対象のカラム名。
  • attval: チェック対象のデータ（Datum）。
  • typtype: 型のカテゴリ（TYPTYPE_RANGE または TYPTYPE_MULTIRANGE）。
  • atttypid: 型のOID。
  • **出力**: なし。空値が検出された場合はエラーにより処理を中断する。

前提条件と不変条件

  • typtype は TYPTYPE_RANGE または TYPTYPE_MULTIRANGE のいずれかである必要がある。それ以外の場合は elog(ERROR) が発生する。

アルゴリズムまたは振る舞い

1. typtype に基づき、attval を適切な型（RangeType または MultirangeType）に変換する。 2. RangeIsEmpty または MultirangeIsEmpty を使用して、値が空であるかを判定する。 3. 値が空である場合、ERRCODE_CHECK_VIOLATION を伴うエラーを発生させ、リレーション名とカラム名を含むメッセージを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execIndexing.c 内の他のルーチンから呼び出される（詳細はソースコード内の行番号 147 および 766 を参照）。
  • **呼び出し先**: RangeIsEmpty、MultirangeIsEmpty、DatumGetRangeTypeP、DatumGetMultirangeTypeP などの型操作関数。

関連データ構造

  • Relation: リレーション情報。
  • RangeType, MultirangeType: PostgreSQLの範囲型およびマルチ範囲型。

失敗モードと境界条件

  • typtype が範囲型・マルチ範囲型以外の場合: elog(ERROR) により強制終了。
  • 値が空（empty）の場合: ereport(ERROR, ...) により CHECK_VIOLATION を発生させる。

保守担当者向けメモ

  • 本関数は WITHOUT OVERLAPS 制約の整合性チェックの一部として機能する。
  • 新たな範囲型関連の型が導入された場合、switch 文の拡張が必要となる可能性がある。
