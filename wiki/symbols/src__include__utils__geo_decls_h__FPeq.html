FPeq

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/geo_decls.h:46

目的

浮動小数点数同士の「曖昧な比較（Fuzzy comparison）」を行い、計算誤差を許容した等価判定を提供します。

シグネチャまたは形状

bool FPeq(double A, double B)

入力と出力

  • **入力**: 比較対象となる2つの倍精度浮動小数点数 A および B。
  • **出力**: A と B が等価であるとみなされる場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • EPSILON 定数に基づいた許容誤差範囲内で判定が行われます。
  • 数値解析の厳密な手法ではなく、計算誤差を考慮した簡易的な比較です。
  • **推移律の欠如**: 浮動小数点演算の性質上、推移律（A=B かつ B=C ならば A=C）が満たされない可能性があります。

アルゴリズムまたは振る舞い

以下の論理和（OR）条件で判定します。 1. A == B が真であること。 2. fabs(A - B) <= EPSILON が真であること（差の絶対値が許容誤差以下であること）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c: GISTインデックスの座標比較に使用。
  • src/backend/utils/adt/geo_ops.c: 幾何型（BOX, LINE, Point）の演算結果の比較に使用。
  • **呼び出し先**:
  • math.h の fabs 関数を呼び出します。

関連データ構造

  • 本関数は double 型の値を直接比較します。幾何データ型（Point, BOX, LINE 等）の各フィールドの比較に利用されます。

失敗モードと境界条件

  • **NaNの扱い**: NaN が含まれる場合、本関数は常に false を返します。
  • **精度**: EPSILON の値に依存するため、極端に大きな値や小さな値の比較では精度上の制約を受けます。

保守担当者向けメモ

  • 本ルーチンは数値解析の専門家によって設計されたものではありません。厳密な計算が求められる箇所での使用には注意が必要です。
  • PATH や POLYGON などの可変長データ型を扱う際は、事前に PG_DETOAST_DATUM によるTOAST解除が完了していることを確認してください。
