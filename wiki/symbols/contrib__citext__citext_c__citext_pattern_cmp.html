citext_pattern_cmp

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:126

目的

citext 型の2つの値を、照合順序（Collation）を無視したパターンマッチング用の比較基準に基づいて比較します。

シグネチャまたは形状

Datum citext_pattern_cmp(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る2つの text 型データ（left, right）。
  • **出力**: int32 型の比較結果（left < right なら負、left == right なら 0、left > right なら正）。

前提条件と不変条件

  • 入力された text 型データは、PG_GETARG_TEXT_PP を使用して適切に展開される必要があります。
  • 処理終了後、PG_FREE_IF_COPY を用いて入力データのメモリを解放し、メモリリークを防ぐ必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して、引数 left および right を取得します。 2. internal_citext_pattern_cmp を呼び出し、照合順序を考慮しないバイナリ比較を実行します。 3. PG_FREE_IF_COPY を使用して、必要に応じて一時的なメモリを解放します。 4. 比較結果を PG_RETURN_INT32 で返します。

呼び出し元と呼び出し先

  • **呼び出し先**: internal_citext_pattern_cmp
  • **呼び出し元**: ソースコード内の PG_FUNCTION_INFO_V1(citext_pattern_cmp) マクロ定義および内部的な参照が存在します。それ以外の外部からの呼び出し関係については、提供された情報からは特定できません。

関連データ構造

  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • 入力データが TOAST されている場合、PG_GETARG_TEXT_PP および PG_FREE_IF_COPY によって適切に処理されます。

保守担当者向けメモ

  • 本関数はパターンマッチング用であるため、通常の citext_cmp と異なり、照合順序の影響を受けないバイナリ比較を行う internal_citext_pattern_cmp を利用しています。修正時はこの比較ロジックの分離を維持してください。
