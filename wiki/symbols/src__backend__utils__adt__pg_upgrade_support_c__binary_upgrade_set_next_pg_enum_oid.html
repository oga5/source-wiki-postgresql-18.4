binary_upgrade_set_next_pg_enum_oid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_upgrade_support.c:162

目的

pg_upgrade 実行時に、次に作成される pg_enum エントリに対して特定の OID を強制的に割り当てるためのグローバル変数を設定します。

シグネチャまたは形状

Datum binary_upgrade_set_next_pg_enum_oid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して、次に使用する OID を受け取ります。
  • **出力**: なし（PG_RETURN_VOID() を返します）。

前提条件と不変条件

  • サーバーが binary_upgrade モードで起動されている必要があります。CHECK_IS_BINARY_UPGRADE マクロにより、この条件が満たされない場合はエラーが発生します。

アルゴリズムまたは振る舞い

1. 引数から OID を取得します。 2. CHECK_IS_BINARY_UPGRADE を実行し、アップグレードモードであることを検証します。 3. グローバル変数 binary_upgrade_next_pg_enum_oid に取得した OID を代入します。 4. 処理を終了し、void を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c において、アップグレード時のデータダンプ出力の一部として SQL 経由で呼び出されます。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、CHECK_IS_BINARY_UPGRADE マクロが内部で実行されます。

関連データ構造

  • binary_upgrade_next_pg_enum_oid: catalog/binary_upgrade.h で宣言されているグローバル変数。

失敗モードと境界条件

  • binary_upgrade モード以外で呼び出された場合、CHECK_IS_BINARY_UPGRADE によりエラーが発生します。

保守担当者向けメモ

  • 本関数はシステムカタログの整合性を直接操作する強力なツールです。誤用はカタログの破損を招くため、必ず pg_upgrade の制御下でのみ使用されることを前提としています。
