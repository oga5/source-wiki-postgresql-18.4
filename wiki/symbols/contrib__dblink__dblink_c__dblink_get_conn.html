dblink_get_conn

定義

  • 種別: function
  • 定義箇所: contrib/dblink/dblink.c:202

目的

指定された接続名または接続文字列を使用して、リモートデータベースへの接続（PGconn）を取得または確立します。

シグネチャまたは形状

static void dblink_get_conn(char *conname_or_str, PGconn *volatile *conn_p, char **conname_p, volatile bool *freeconn_p)

入力と出力

  • **入力**:
  • conname_or_str: 接続名、または接続文字列。
  • **出力**:
  • conn_p: 取得した PGconn オブジェクトへのポインタ。
  • conname_p: 接続名（名前付き接続の場合）または NULL。
  • freeconn_p: 呼び出し元で接続の解放（libpqsrv_disconnect）が必要かを示すフラグ。

前提条件と不変条件

  • dblink_init() が呼び出され、内部状態が初期化されている必要があります。
  • 接続文字列が指定された場合、dblink_security_check によるセキュリティ検証を通過する必要があります。

アルゴリズムまたは振る舞い

1. getConnectionByName を呼び出し、既存の接続を検索します。 2. 既存の接続が見つかった場合、その接続を再利用し、freeconn を false に設定します。 3. 見つからない場合、接続文字列として解釈し、libpqsrv_connect で新規接続を確立します。

  • 接続失敗時は ereport(ERROR, ...) で処理を中断します。
  • 接続成功後、dblink_security_check で権限を確認し、必要に応じてクライアントエンコーディングをデータベースエンコーディングと同期させます。
  • 新規接続の場合、freeconn を true に設定します。

    • 4. 取得した接続情報およびフラグを引数経由で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c 内の各関数（行 738, 753, 1431, 1446 など）。
  • **呼び出し先**:
  • getConnectionByName
  • get_connect_string
  • dblink_connstr_check
  • WaitEventExtensionNew
  • libpqsrv_connect
  • PQstatus, PQerrorMessage, pchomp, libpqsrv_disconnect
  • dblink_security_check
  • PQclientEncoding, GetDatabaseEncoding, PQsetClientEncoding, GetDatabaseEncodingName

関連データ構造

  • remoteConn: 接続状態を保持する構造体。
  • PGconn: libpq の接続ハンドル。

失敗モードと境界条件

  • 接続名が存在せず、かつ接続文字列としても不正な場合、エラーが発生します。
  • リモート接続が CONNECTION_BAD となった場合、エラーメッセージを伴って ERROR レベルで終了します。

保守担当者向けメモ

  • 本関数は volatile 修飾子を使用してポインタを渡しており、エラー発生時のメモリコンテキスト切り替えや PG_TRY ブロック内での安全性を考慮した設計となっています。
