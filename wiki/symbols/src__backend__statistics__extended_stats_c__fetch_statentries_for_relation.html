fetch_statentries_for_relation

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/extended_stats.c:418

目的

指定されたリレーション（テーブル）に関連付けられた拡張統計情報の定義を pg_statistic_ext カタログから取得し、StatExtEntry 構造体のリストとして構築する。

シグネチャまたは形状

static List *fetch_statentries_for_relation(Relation pg_statext, Oid relid)

入力と出力

  • **入力**:
  • pg_statext: pg_statistic_ext カタログを開いた状態の Relation オブジェクト。
  • relid: 統計情報を取得する対象テーブルの Oid。
  • **出力**:
  • List *: 取得した統計オブジェクトの定義を保持する StatExtEntry 構造体のリスト。

前提条件と不変条件

  • pg_statext は pg_statistic_ext カタログを指している必要がある。
  • stxkind カラムは 1 次元の char 配列であり、有効な統計種別（NDistinct, Dependencies, MCV, Expressions）のみを含む必要がある。

アルゴリズムまたは振る舞い

1. StatisticExtRelidIndexId インデックスを使用して、指定された relid に一致する pg_statistic_ext のタプルをスキャンする。 2. 各タプルから以下の情報を抽出する:

  • statOid, schema, name, columns (ビットマップセット)。
  • stattarget (設定値、未設定時は -1)。
  • stxkind (統計種別のリスト)。

    • 3. stxexprs カラムが存在する場合、文字列をノードツリーに変換し、eval_const_expressions による定数評価と fix_opfuncids による関数 ID の修正を行う。 4. 構築した StatExtEntry をリストに追加し、スキャン終了後にリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/extended_stats.c 内の BuildRelationExtStatistics および統計情報を必要とする他の関数から呼び出される。
  • **呼び出し先**:
  • systable_beginscan, systable_getnext, systable_endscan (カタログスキャン用)。
  • SysCacheGetAttr, stringToNode, eval_const_expressions, fix_opfuncids (データ抽出および式処理用)。

関連データ構造

  • StatExtEntry: 拡張統計の定義を保持する構造体。
  • pg_statistic_ext: 拡張統計情報を格納するシステムカタログ。

失敗モードと境界条件

  • stxkind が 1 次元の char 配列でない場合、または型が CHAROID でない場合は elog(ERROR) を発生させる。
  • 該当する統計オブジェクトが存在しない場合、空のリスト (NIL) を返す。

保守担当者向けメモ

  • 式の評価において eval_const_expressions を使用しているのは、プランナがクエリ内の句と照合する際に正規化された形式が必要となるためである。この処理は必須であり、削除してはならない。
