get_attribute_options

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/attoptcache.c:130

目的

指定されたリレーション（attrelid）および属性番号（attnum）に対応する属性オプション（attoptions）をキャッシュから取得します。キャッシュミス時はシステムカタログ pg_attribute から情報を読み込み、キャッシュを更新します。

シグネチャまたは形状

AttributeOpts *get_attribute_options(Oid attrelid, int attnum);

入力と出力

  • **入力**:
  • attrelid (Oid): 対象リレーションのOID。
  • attnum (int): 対象属性の番号。
  • **出力**:
  • 成功時: 属性オプションを格納した AttributeOpts 構造体へのポインタ（呼び出し元のメモリコンテキストにコピー済み）。
  • 該当なし/オプション未設定時: NULL。

前提条件と不変条件

  • AttoptCacheHash が初期化されている必要があります（未初期化時は関数内で InitializeAttoptCache() を呼び出します）。
  • 返却される AttributeOpts は呼び出し元で pfree 可能です。
  • キャッシュエントリの構築は、pg_attribute の読み込みが完了した後に実行されます。

アルゴリズムまたは振る舞い

1. AttoptCacheHash を検索し、エントリが存在すればその内容をコピーして返却します。 2. キャッシュミス時:

  • SearchSysCache2 を使用して ATTNUM システムキャッシュから pg_attribute のタプルを取得します。
  • タプルが存在しない、または attoptions が空の場合は opts を NULL とします。
  • タプルが存在する場合、attribute_reloptions() で解析し、CacheMemoryContext 上に確保した領域へコピーします。
  • hash_search (HASH_ENTER) を用いてキャッシュエントリを登録します。

    • 3. 最後に、キャッシュされた opts を呼び出し元のメモリコンテキストにコピーして返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c
  • src/backend/statistics/extended_stats.c
  • **呼び出し先**:
  • InitializeAttoptCache
  • hash_search
  • SearchSysCache2
  • SysCacheGetAttr
  • attribute_reloptions
  • ReleaseSysCache
  • MemoryContextAlloc
  • palloc

関連データ構造

  • AttoptCacheKey: キャッシュ検索用のキー（attrelid, attnum）。
  • AttoptCacheEntry: キャッシュ内のデータ構造。
  • AttributeOpts: 属性オプションの実体。
  • AttoptCacheHash: 属性オプションを保持するハッシュテーブル。

失敗モードと境界条件

  • 存在しない属性が指定された場合、HeapTupleIsValid が偽となり、NULL を返します。
  • pg_attribute にオプションが設定されていない場合、isNull が真となり、NULL を返します。
  • メモリ不足時は palloc 等がエラーを発生させます。

保守担当者向けメモ

  • 本関数は CacheMemoryContext を使用してキャッシュを保持します。キャッシュエントリの操作時はメモリコンテキストの生存期間に注意してください。
  • キャッシュエントリの opts は pfree 可能な領域であるため、呼び出し元で受け取ったデータは適切に管理してください。
  • pg_attribute の更新イベントが発生すると、InvalidateAttoptCacheCallback によってキャッシュエントリが破棄される仕組みになっています。
