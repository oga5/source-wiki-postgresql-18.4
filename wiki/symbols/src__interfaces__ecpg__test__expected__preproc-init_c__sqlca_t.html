sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-init.c:30

目的

SQL 実行結果、エラー情報、警告、および診断情報を保持するための構造体。ECPG（Embedded SQL for C）アプリケーションにおいて、データベース操作の成否を判定するために使用される。

シグネチャまたは形状

struct sqlca_t {
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: ECPG ランタイムライブラリ（ECPGdo() 等）が SQL 実行後にこの構造体のメンバを更新する。
  • **出力**: アプリケーションコードが sqlcode や sqlstate を参照し、エラーハンドリングや制御フローの分岐に使用する。

前提条件と不変条件

  • sqlca インスタンスは ECPGget_sqlca() を通じて取得されることが一般的である。
  • sqlerrd[2] は、INSERT、UPDATE、DELETE 文の実行後に処理された行数を保持する。
  • sqlwarn[0] は、他の警告フラグが設定された場合に 'W' に設定される。

アルゴリズムまたは振る舞い

ECPG プリプロセッサによって生成されたコードは、SQL 文の実行直後に sqlca.sqlcode をチェックする。sqlcode < 0 の場合、WHENEVER 文で定義されたエラーハンドリングアクション（関数呼び出し等）が実行される。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはない。

関連データ構造

  • sqlda_t: 動的 SQL のための記述子領域。
  • sqlvar_t: sqlda_t 内で使用される変数定義構造体。

失敗モードと境界条件

  • sqlerrmc は SQLERRMC_LEN で定義されたサイズ制限があるため、長いエラーメッセージは切り詰められる可能性がある。
  • sqlcode が 0 の場合は成功、正の値は警告や「データなし（100）」、負の値はエラーを示す。

保守担当者向けメモ

  • 本構造体は ECPG のランタイムライブラリとプリプロセッサ間で共有される重要なインターフェースである。定義を変更する場合は、ecpglib および関連する互換ライブラリへの影響を考慮すること。
  • sqlerrd 配列の各要素の意味（OID、処理行数など）はソース内のコメントに準拠している。
