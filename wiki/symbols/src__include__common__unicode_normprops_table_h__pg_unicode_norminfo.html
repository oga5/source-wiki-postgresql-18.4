pg_unicode_norminfo

定義

  • 種別: struct
  • 定義箇所: src/include/common/unicode_normprops_table.h:19

目的

Unicode 正規化（NFC/NFKC）における「クイックチェック（Quick Check）」処理のための管理構造体です。コードポイントの正規化状態を高速に判定するためのルックアップテーブル、ハッシュ関数、および要素数を保持します。

シグネチャまたは形状

typedef struct
{
	const pg_unicode_normprops *normprops;
	qc_hash_func	hash;
	int		num_normprops;
} pg_unicode_norminfo;

入力と出力

  • **入力**: 構造体内の hash 関数ポインタに対し、対象のコードポイントを渡すことでインデックスを算出します。
  • **出力**: 算出されたインデックスを用いて normprops 配列から pg_unicode_normprops 構造体を取得し、正規化状態（YES/NO/MAYBE）を判定します。

前提条件と不変条件

  • 本構造体は自動生成スクリプト（generate-unicode_normprops_table.pl）によって生成される静的データです。
  • hash 関数は、指定されたデータセットに対して衝突が発生しない「完全ハッシュ関数」であることが前提です。

アルゴリズムまたは振る舞い

1. 呼び出し元が pg_unicode_norminfo を参照します。 2. 対象コードポイントを hash 関数に渡し、テーブル内のインデックスを算出します。 3. normprops 配列の算出インデックス位置からプロパティ情報を取得します。 4. 取得したプロパティに基づき、正規化の要否を判断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/unicode_norm.c 内の qc_hash_lookup 関数が本構造体を参照しています。
  • **呼び出し先**: UnicodeNormInfo_NFC_QC および UnicodeNormInfo_NFKC_QC として定義された静的インスタンスが、それぞれ対応する NFC_QC_hash_func や NFKC_QC_hash_func を保持しています。

関連データ構造

  • pg_unicode_normprops: クイックチェック状態を保持する個々の要素。
  • qc_hash_func: インデックス算出用の関数型。

失敗モードと境界条件

  • 本構造体は静的データとして定義されているため、実行時のメモリ割り当て失敗等のリスクはありません。
  • 完全ハッシュ関数の性質上、テーブル範囲外へのアクセスが発生しないよう設計されています。
