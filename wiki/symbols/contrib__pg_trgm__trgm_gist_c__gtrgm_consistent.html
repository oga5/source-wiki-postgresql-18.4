gtrgm_consistent

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_gist.c:196

目的

GiST インデックスの検索において、インデックスノードが指定された検索条件（演算子）を満たすかどうかを判定します。

シグネチャまたは形状

Datum gtrgm_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • entry: 検査対象の GiST インデックスエントリ。
  • query: 検索条件となるテキスト。
  • strategy: 演算子戦略番号（Similarity, Like, RegExp 等）。
  • recheck: 検索結果の再評価が必要かを示すフラグ（出力用）。
  • **出力**:
  • Datum: 判定結果（bool）。条件を満たす場合は true。

前提条件と不変条件

  • fcinfo->flinfo->fn_extra を介して、クエリのトリグラム抽出結果をキャッシュします。
  • キャッシュは strategy、query のサイズ、および内容が一致する場合にのみ再利用されます。
  • インデックスノードが ISALLTRUE である場合、常に true を返します。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: fn_extra を参照し、現在のクエリと戦略に適合するキャッシュがあれば利用します。なければ generate_trgm や createTrgmNFA を用いてトリグラムを抽出し、キャッシュを更新します。 2. **戦略別判定**:

  • **Similarity系**: cnt_sml（リーフノード）または cnt_sml_sign_common（非リーフノード）を用いて類似度を計算し、閾値と比較します。SimilarityStrategyNumber 以外は recheck を true に設定します。
  • **Like/Equal系**: trgm_contained_by を用いてトリグラムの包含関係を判定します。常に recheck を true に設定します。
  • **RegExp系**: createTrgmNFA で生成されたグラフを用いて判定を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックススキャンエンジン（直接的な呼び出し元関数の詳細はソースコード内に明示されていません）。
  • **呼び出し先**: generate_trgm, generate_wildcard_trgm, createTrgmNFA, cnt_sml, cnt_sml_sign_common, trgm_contained_by 等。

関連データ構造

  • GISTENTRY: インデックスエントリの構造体。
  • gtrgm_consistent_cache: 検索クエリのトリグラムや正規表現グラフを保持するキャッシュ構造体。
  • TRGM: トリグラムデータ。

失敗モードと境界条件

  • **未対応の戦略**: 未定義の戦略番号が渡された場合、elog(ERROR) を発生させます。
  • **ケースセンシティブ**: IGNORECASE が定義されていない環境で大文字小文字を区別しない演算子（~~*, ~*）が使用された場合、エラーとなります。
  • **メモリ**: キャッシュ生成時のメモリ確保に失敗した場合は例外が発生します。
  • **正規表現**: 正規表現グラフの扱いに起因するメモリリークの可能性がある旨が指摘されています。

保守担当者向けメモ

  • 本関数は GiST インデックスの「候補」を絞り込む役割を担うため、多くのケースで recheck が true となり、ヒープでの再検証が行われます。
  • fn_extra を使用したキャッシュのライフサイクル管理（特に正規表現グラフの解放）には注意が必要です。
