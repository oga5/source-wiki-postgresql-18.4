comp_trgm

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_op.c:284

目的

トリグラム（3文字の連続）の比較を行うための標準的な比較関数を提供します。主に qsort や qunique などの標準ライブラリ関数から呼び出されるコールバックとして機能します。

シグネチャまたは形状

static int comp_trgm(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つのトリグラムへのポインタ a および b。
  • **出力**: a が b より小さい場合は負の整数、等しい場合は 0、大きい場合は正の整数を返します。

前提条件と不変条件

  • CMPTRGM マクロが定義されている必要があります。
  • システムの文字符号（signed/unsigned）の差異を吸収するため、直接比較を行わず必ず CMPTRGM マクロを経由する必要があります。

アルゴリズムまたは振る舞い

内部で CMPTRGM(a, b) を呼び出すだけのラッパー関数です。このマクロは、実行時のシステム設定（CMPTRGM_CHOOSE により決定）に基づき、符号付きまたは符号なしの比較ロジックを動的に選択します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_op.c 内の qsort および qunique 呼び出し箇所（行 572, 573, 1103, 1104）。
  • **呼び出し先**:
  • CMPTRGM マクロ（実行時に最適化された比較関数へ展開）。

関連データ構造

  • trgm: トリグラムを保持する型。

失敗モードと境界条件

  • 入力ポインタ a または b が NULL の場合の挙動は、CMPTRGM マクロの実装に依存します。

保守担当者向けメモ

  • 本関数はトリグラムのソートおよび重複排除の基盤です。比較ロジックを変更する場合は、個別の比較関数ではなく CMPTRGM マクロの定義または CMPTRGM_CHOOSE による初期化ロジックを確認してください。
