WALInsertLockPadded

定義

  • 種別: typedef (共用体)
  • 定義箇所: src/backend/access/transam/xlog.c:382

目的

WALInsertLock を CPU キャッシュラインサイズにパディングし、共有メモリ上での配置において「偽の共有（False Sharing）」を防止するために使用されます。

シグネチャまたは形状

typedef union WALInsertLockPadded
{
	WALInsertLock l;
	char		pad[PG_CACHE_LINE_SIZE];
} WALInsertLockPadded;

入力と出力

  • **入力**: なし（データ構造の定義）。
  • **出力**: なし。

前提条件と不変条件

  • PG_CACHE_LINE_SIZE は、ターゲットプラットフォームのキャッシュラインサイズに一致している必要があります。
  • WALInsertLock 構造体は、pad 配列のサイズ（キャッシュラインサイズ）以下である必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、WALInsertLock を char 配列の先頭に配置し、残りの領域をパディングすることで、メモリ上のサイズを強制的にキャッシュライン境界に揃えます。
  • これにより、複数の WALInsertLock が同一のキャッシュラインに混在することを防ぎ、マルチコア環境下でのロック取得時のキャッシュコヒーレンシトラフィックを低減します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c 内の共有メモリ割り当て処理（XLOGShmemInit 相当の初期化フロー）において、WALInsertLocks 配列の確保に使用されます。
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • WALInsertLock: 実際の WAL 挿入ロックを保持する構造体。
  • XLogCtlData: WALInsertLocks 配列を管理する上位の共有メモリ構造体。

失敗モードと境界条件

  • **メモリ割り当て**: 共有メモリの確保時に NUM_XLOGINSERT_LOCKS に基づいて計算されるサイズが不足すると、システム初期化に失敗します。
  • **アライメント**: allocptr の計算において、sizeof(WALInsertLockPadded) を用いた剰余演算によるアライメント調整が行われており、この計算が誤るとメモリレイアウトが崩れる可能性があります。

保守担当者向けメモ

  • 本構造体のサイズを変更する場合は、PG_CACHE_LINE_SIZE の定義と整合しているか確認してください。
  • WALInsertLock のサイズが PG_CACHE_LINE_SIZE を超えると、パディングの効果が失われるだけでなく、コンパイルエラーやメモリレイアウトの不整合を招く恐れがあります。
