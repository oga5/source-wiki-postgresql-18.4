ParseVariableDouble

定義

  • 種別: function
  • 定義箇所: src/bin/psql/variables.c:194

目的

文字列形式の変数値から倍精度浮動小数点数（double）への安全な変換と、指定された範囲内であることの検証を行う。

シグネチャまたは形状

bool ParseVariableDouble(const char *value, const char *name, double *result, double min, double max)

入力と出力

  • **入力**:
  • value: 変換対象の文字列。
  • name: エラーメッセージに使用する変数名（NULL 指定可）。
  • min, max: 値の許容範囲（境界値を含む）。
  • **出力**:
  • 戻り値: 変換および検証が成功した場合は true、失敗した場合は false。
  • result: 変換成功時に格納される double 値。

前提条件と不変条件

  • value が NULL または空文字列の場合、変換は失敗する。
  • 変換後の値が min 未満、または max を超える場合は失敗する。
  • 変換失敗時には、name が指定されていれば pg_log_error を通じてエラーメッセージが出力される。

アルゴリズムまたは振る舞い

1. value が空でないことを確認する。 2. strtod を使用して文字列を double に変換する。 3. 変換結果が数値として妥当（errno が正常、かつ文字列全体が消費されている）かを確認する。 4. 変換された値が [min, max] の範囲内にあるか検証する。 5. 範囲外や変換エラー（ERANGE 等）が発生した場合は、適切なエラーメッセージをログ出力し false を返す。 6. すべての条件を満たした場合、result に値を代入し true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/startup.c:958 (WATCH_INTERVAL の設定)
  • src/bin/psql/command.c:3389 (変数設定に関連する処理)
  • **呼び出し先**:
  • strtod (標準ライブラリ)
  • pg_log_error (エラー報告用)

関連データ構造

  • 特になし（プリミティブ型および標準ライブラリの型を使用）。

失敗モードと境界条件

  • **入力不正**: value が NULL または空文字列の場合。
  • **構文エラー**: strtod が数値を正しく解析できない場合。
  • **範囲外**: 変換値が min 未満または max を超える場合。
  • **オーバーフロー/アンダーフロー**: errno が ERANGE となり、値が HUGE_VAL 等に達している場合。

保守担当者向けメモ

  • strtod のプラットフォーム依存の挙動（特に空文字列やゼロに近い値の扱い）を吸収する実装となっている。
  • name 引数が NULL の場合、エラーメッセージの出力は抑制される。
