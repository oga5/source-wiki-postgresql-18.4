CreateReplicationSlot

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/streamutil.c:583

目的

PostgreSQLサーバーに対して CREATE_REPLICATION_SLOT コマンドを発行し、物理または論理レプリケーションスロットを作成します。

シグネチャまたは形状

bool CreateReplicationSlot(PGconn *conn, const char *slot_name, const char *plugin,
                           bool is_temporary, bool is_physical, bool reserve_wal,
                           bool slot_exists_ok, bool two_phase, bool failover)

入力と出力

  • **入力**:
  • conn: サーバーへのアクティブな PGconn 接続。
  • slot_name: 作成するスロット名。
  • plugin: 論理スロットの場合の出力プラグイン名（物理スロット時は NULL）。
  • is_temporary: 一時スロットとして作成するかどうか。
  • is_physical: 物理スロットを作成するかどうか。
  • reserve_wal: 物理スロットでWALを予約するかどうか。
  • slot_exists_ok: 既にスロットが存在する場合にエラーとせず成功とみなすか。
  • two_phase: 論理スロットで2フェーズコミットを有効にするか。
  • failover: 論理スロットでフェイルオーバーを有効にするか。
  • **出力**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • is_physical が true の場合、plugin は NULL でなければならない。
  • is_physical が false の場合、plugin は NULL であってはならない。
  • two_phase および failover は、物理スロット（is_physical）と同時に指定できない。
  • slot_name は NULL であってはならない。

アルゴリズムまたは振る舞い

1. サーバーバージョン（PQserverVersion）に基づき、PostgreSQL 15以降で導入された括弧付きの新しいオプション構文を使用するかを決定します。 2. PQExpBuffer を用いて CREATE_REPLICATION_SLOT コマンド文字列を動的に構築します。 3. サーバーバージョンに応じて、RESERVE_WAL、FAILOVER、TWO_PHASE、SNAPSHOT などのオプションをコマンドに追加します。 4. PQexec を通じてコマンドを実行します。 5. 実行結果が PGRES_TUPLES_OK でない場合、slot_exists_ok が true かつエラーコードが ERRCODE_DUPLICATE_OBJECT であれば成功として終了します。 6. 正常終了時は、サーバーから返される結果セット（1行4列）を確認し、バッファと結果オブジェクトを解放して true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_receivewal.c
  • src/bin/pg_recvlogical.c
  • **呼び出し先**:
  • libpq 関数群 (PQexec, PQserverVersion, PQresultStatus, PQresultErrorField, PQntuples, PQnfields, PQclear, PQerrorMessage)
  • pqexpbuffer 関数群 (createPQExpBuffer, appendPQExpBuffer, appendPQExpBufferStr, appendPQExpBufferChar, destroyPQExpBuffer)
  • streamutil.c 内のヘルパー関数 (AppendPlainCommandOption, AppendStringCommandOption)

関連データ構造

  • PGconn: libpqの接続ハンドル。
  • PGresult: クエリ実行結果。
  • PQExpBuffer: コマンド文字列構築用バッファ。

失敗モードと境界条件

  • サーバー接続が切断されている場合、PQexec が失敗します。
  • サーバーバージョンが古い場合、新しいオプション（FAILOVER 等）を指定しても無視されるか、サーバー側でエラーとなります。
  • 既に存在するスロットに対して slot_exists_ok が false の場合、エラーとしてログ出力され false が返ります。
  • サーバーからの応答が期待される形式（1行4列）でない場合、エラーとしてログ出力され false が返ります。

保守担当者向けメモ

  • サーバーバージョンによるコマンド構文の差異（特にPostgreSQL 15以降の括弧付き構文）を維持する必要があります。
  • 新しいレプリケーションオプションを追加する場合は、AppendPlainCommandOption 等のヘルパー関数を適切に使用してください。
