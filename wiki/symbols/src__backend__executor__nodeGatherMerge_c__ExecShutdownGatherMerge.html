ExecShutdownGatherMerge

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:303

目的

GatherMerge 実行ノードに関連する並列実行リソースを解放し、ワーカープロセスを停止する。

シグネチャまたは形状

void ExecShutdownGatherMerge(GatherMergeState *node)

入力と出力

  • **入力**: GatherMergeState *node - 停止対象の GatherMerge ノードの状態構造体。
  • **出力**: なし。

前提条件と不変条件

  • node が指す GatherMergeState は初期化済みであること。
  • node->pei が有効な場合、ExecParallelCleanup を通じて並列コンテキストが適切に破棄される。

アルゴリズムまたは振る舞い

1. ExecShutdownGatherMergeWorkers(node) を呼び出し、並列ワーカーの停止処理を実行する。 2. node->pei（並列実行情報）が NULL でない場合、ExecParallelCleanup(node->pei) を呼び出して並列コンテキストを破棄し、node->pei を NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c:812
  • src/backend/executor/nodeGatherMerge.c:294
  • **呼び出し先**:
  • ExecShutdownGatherMergeWorkers
  • ExecParallelCleanup

関連データ構造

  • GatherMergeState: GatherMerge ノードの実行状態を保持する構造体。
  • ParallelExecutorInfo (pei): 並列実行に関連するコンテキスト情報。

失敗モードと境界条件

  • node が NULL の場合の挙動は未定義（ソースコード上でのチェックは明示されていない）。
  • node->pei が既に NULL の場合、ExecParallelCleanup はスキップされるため安全である。

保守担当者向けメモ

  • 本関数は GatherMerge ノードの終了処理の一部であり、リソースリークを防ぐために必ず呼び出される必要がある。
  • 並列ワーカーの停止とコンテキストの破棄という二段階の処理を行っているため、並列インフラストラクチャに変更があった場合は本関数の順序を確認すること。
