SpinDelayStatus

定義

  • 種別: struct
  • 定義箇所: src/include/storage/s_lock.h:729

目的

スピンロック取得の競合発生時における、試行回数、遅延時間、および発生場所（ファイル名、行番号、関数名）を追跡・管理するための構造体です。

シグネチャまたは形状

typedef struct
{
	int			spins;
	int			delays;
	int			cur_delay;
	const char *file;
	int			line;
	const char *func;
} SpinDelayStatus;

入力と出力

本構造体は、スピンロック待機処理の各関数に対してポインタとして渡されます。

  • **入力**: init_spin_delay により初期化され、perform_spin_delay および finish_spin_delay に状態として渡されます。
  • **出力**: 待機ループ内での試行回数や遅延時間の累積値が更新されます。

前提条件と不変条件

  • init_spin_delay を呼び出して初期化してから使用する必要があります。
  • 構造体内の file, line, func は、デバッグやログ出力のためにスピンロックが発生したコンテキストを保持します。

アルゴリズムまたは振る舞い

1. **初期化**: init_spin_delay で待機状態をセットアップします。 2. **待機実行**: perform_spin_delay が呼び出されるたびに、spins や delays がインクリメントされ、cur_delay に基づいて CPU 負荷を軽減する待機（PAUSE 命令等）が行われます。 3. **終了**: finish_spin_delay により、待機処理の完了に伴う後処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/lmgr/lwlock.c
  • src/backend/storage/lmgr/s_lock.c
  • **呼び出し先**:
  • init_spin_delay (初期化)
  • perform_spin_delay (待機実行)
  • finish_spin_delay (終了処理)

関連データ構造

  • slock_t: スピンロックの本体を保持する型。

失敗モードと境界条件

  • 規定回数（NUM_DELAYS）を超えてもロックが取得できない場合、システムはデッドロックやスタックと見なし PANIC を発生させます。

保守担当者向けメモ

  • 本構造体はスピンロックの待機戦略を可視化・制御するためのものです。新しいアーキテクチャへの移植や待機ロジックの変更時には、perform_spin_delay 内の cur_delay の更新アルゴリズムと併せて確認してください。
