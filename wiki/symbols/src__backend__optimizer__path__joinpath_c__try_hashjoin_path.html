try_hashjoin_path

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinpath.c:1221

目的

2つの結合対象リレーション（outer_path と inner_path）に対してハッシュ結合（Hash Join）パスを生成し、コスト評価を経て有効であれば結合リレーションのパスリストに追加する。

シグネチャまたは形状

static void try_hashjoin_path(PlannerInfo *root,
                              RelOptInfo *joinrel,
                              Path *outer_path,
                              Path *inner_path,
                              List *hashclauses,
                              JoinType jointype,
                              JoinPathExtraData *extra)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • joinrel: 結合対象のリレーション情報。
  • outer_path, inner_path: 結合する左右のパス。
  • hashclauses: ハッシュ結合に使用する条件式のリスト。
  • jointype: 結合タイプ（INNER, LEFT, SEMI 等）。
  • extra: 結合メタデータ（SpecialJoinInfo や restrictlist を含む）。
  • **出力**: なし（成功時に joinrel->pathlist を更新）。

前提条件と不変条件

  • ハッシュ結合パスは出力パスキー（pathkeys）を持たない。
  • 外部結合（Outer Join）が関与する場合、入力パスがその外部結合をパラメータとして含んでいると無効とみなされる。
  • パラメータ化されたパスの場合、そのパラメータが extra->param_source_rels と重なりを持つ必要がある。

アルゴリズムまたは振る舞い

1. **妥当性チェック**: 外部結合の制約およびパラメータ化の妥当性を検証し、不適格な場合は早期リターンする。 2. **コスト計算**: initial_cost_hashjoin を呼び出し、ハッシュ結合の起動コストと総コストを算出する。 3. **パスの選別**: add_path_precheck を使用し、既存のパスと比較してコスト的に優位であるかを確認する。 4. **パスの生成と追加**: 条件を満たす場合、create_hashjoin_path でパスノードを作成し、add_path で joinrel に登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinpath.c 内の各所（add_paths_to_joinrel 等から呼び出される）。
  • **呼び出し先**:
  • calc_non_nestloop_required_outer
  • initial_cost_hashjoin
  • add_path_precheck
  • create_hashjoin_path
  • add_path
  • bms_is_member, bms_overlap, bms_free（ビットマップ操作）

関連データ構造

  • RelOptInfo: 結合対象のリレーション。
  • Path: 結合の入力および生成されるパス。
  • JoinPathExtraData: 結合処理に必要な追加情報。
  • JoinCostWorkspace: コスト計算結果を保持する作業領域。

失敗モードと境界条件

  • パラメータ化が不適切な場合、またはコストが既存のパスより劣る場合はパスを追加せずに終了する。
  • メモリリークを防ぐため、パスを拒否する際には required_outer ビットマップを解放する。

保守担当者向けメモ

  • 本関数は parallel_hash を false として固定で呼び出している。並列ハッシュ結合の生成には try_partial_hashjoin_path が別途存在する。
  • ハッシュ結合はソート順序を保持しないため、パスキーの考慮は不要である。
