server_get_sink

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_target.c:202

目的

server ターゲット用のバックアップシンク（bbsink）パイプラインを生成します。

シグネチャまたは形状

static bbsink *server_get_sink(bbsink *next_sink, void *detail_arg);

入力と出力

  • next_sink: パイプラインの次段となる bbsink オブジェクト。
  • detail_arg: ターゲットの詳細情報（server_check_detail で検証されたデータ）。
  • 戻り値: サーバーサイドバックアップ用に初期化された新しい bbsink オブジェクト。

前提条件と不変条件

  • bbsink_server_new が正常にメモリを確保し、オブジェクトを構築できること。

アルゴリズムまたは振る舞い

1. bbsink_server_new を呼び出し、next_sink をラップしたサーバーサイド専用のシンクオブジェクトを生成します。 2. 生成されたシンクオブジェクトを呼び出し元へ返します。

呼び出し元と呼び出し先

  • 呼び出し元: BaseBackupGetSink（src/backend/backup/basebackup_target.c 内のターゲット登録リスト経由で参照）。
  • 呼び出し先: bbsink_server_new（src/backend/backup/basebackup_server.c で定義）。

関連データ構造

  • bbsink: バックアップデータの転送パイプラインを構成する抽象インターフェース。
  • bbsink_server: server ターゲット固有のシンク実装。

失敗モードと境界条件

  • メモリ不足時には bbsink_server_new 内部でエラーが発生し、処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は BaseBackupTargetType 構造体の一部として登録されており、server ターゲットが選択された際に動的に呼び出されます。
  • サーバーサイドバックアップの具体的なデータ転送ロジックについては src/backend/backup/basebackup_server.c を参照してください。
