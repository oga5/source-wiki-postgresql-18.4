ExecAppendInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:517

目的

Append ノードの並列実行に必要な共有メモリ領域（DSM）を割り当て、初期化を行う。

シグネチャまたは形状

void ExecAppendInitializeDSM(AppendState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の AppendState 構造体。
  • pcxt: 並列実行コンテキスト (ParallelContext)。
  • **出力**: なし（副作用として node および共有メモリを更新）。

前提条件と不変条件

  • node->pstate_len に必要な共有メモリサイズが設定されていること。
  • pcxt->toc が有効なテーブル・オブ・コンテンツ（TOC）を保持していること。

アルゴリズムまたは振る舞い

1. shm_toc_allocate を使用し、node->pstate_len 分の共有メモリを pcxt->toc から確保する。 2. 確保したメモリ領域を memset でゼロクリアする。 3. LWLockInitialize を呼び出し、ParallelAppendState 内の pa_lock を LWTRANCHE_PARALLEL_APPEND で初期化する。 4. shm_toc_insert を使用し、プランノード ID をキーとして pstate を TOC に登録する。 5. node->as_pstate に確保した pstate を格納する。 6. node->choose_next_subplan を choose_next_subplan_for_leader に設定し、リーダープロセス用のサブプラン選択ロジックを割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:497 (確認済み)
  • **呼び出し先**:
  • shm_toc_allocate
  • memset
  • LWLockInitialize
  • shm_toc_insert
  • choose_next_subplan_for_leader (関数ポインタへの代入)

関連データ構造

  • AppendState: 実行時の状態を保持。
  • ParallelAppendState: 並列実行時に共有される状態（ロックや進捗管理）。
  • ParallelContext: 並列実行のコンテキスト。

失敗モードと境界条件

  • 共有メモリの割り当て失敗時、shm_toc_allocate がエラーを発生させる可能性がある。
  • node->pstate_len が不適切な場合、メモリ破壊や不正アクセスが発生する可能性がある。

保守担当者向けメモ

  • 本関数はリーダープロセスによって呼び出されることを前提としている。
  • choose_next_subplan の切り替えは、並列実行時のリーダーとワーカーの役割分担を決定する重要な処理である。
