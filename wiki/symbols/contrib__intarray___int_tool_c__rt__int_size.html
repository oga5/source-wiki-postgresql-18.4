rt__int_size

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:183

目的

PostgreSQL の ArrayType 配列に含まれる要素数を取得し、浮動小数点数として出力します。

シグネチャまたは形状

void rt__int_size(ArrayType *a, float *size)

入力と出力

  • **入力**:
  • a: 要素数をカウントする対象の ArrayType 構造体へのポインタ。
  • size: 結果を格納するための float 型変数へのポインタ。
  • **出力**:
  • なし（size が指すメモリ領域に結果が書き込まれます）。

前提条件と不変条件

  • a は有効な ArrayType である必要があります（呼び出し側で CHECKARRVALID マクロ等による検証が推奨されます）。
  • size は書き込み可能な有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. ARRNELEMS(a) マクロを使用して、配列 a 内の要素数を取得します。 2. 取得した要素数を float 型にキャストします。 3. キャストした値を size が指すアドレスに代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c: GiST インデックスの計算処理（union や inter のサイズ計算など）において頻繁に呼び出されています。
  • **呼び出し先**:
  • なし（本関数は直接的な計算のみを行い、他の関数を呼び出しません）。

関連データ構造

  • ArrayType: PostgreSQL の標準的な配列データ構造。

失敗モードと境界条件

  • a が NULL の場合、ARRNELEMS マクロの内部実装に依存してセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は非常に単純なラッパーであり、主に GiST インデックスのペナルティ計算やサイズ評価のインターフェースを統一するために使用されています。
  • ARRNELEMS は配列のメタデータから要素数を取得するマクロであり、配列の内容を走査しないため $O(1)$ で動作します。
