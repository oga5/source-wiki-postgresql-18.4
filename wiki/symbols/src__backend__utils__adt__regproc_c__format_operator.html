format_operator

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regproc.c:792

目的

指定された演算子OIDを、人間が読み取り可能な文字列形式（例: schema.operator_name）に変換します。

シグネチャまたは形状

char *format_operator(Oid operator_oid)

入力と出力

  • **入力**: operator_oid (Oid) - 変換対象の演算子OID。
  • **出力**: char * - 演算子名を表す文字列。メモリは palloc によって割り当てられます。

前提条件と不変条件

  • 入力された operator_oid がシステムカタログ pg_operator に存在する必要があります。
  • 内部的に format_operator_extended を呼び出すため、その前提条件（システムキャッシュの有効性など）に依存します。

アルゴリズムまたは振る舞い

1. 引数 operator_oid を受け取ります。 2. format_operator_extended(operator_oid, 0) を呼び出します。 3. 拡張フラグに 0 を指定することで、デフォルトのフォーマット設定で演算子名を解決し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: format_operator_extended
  • **呼び出し元**:
  • contrib/bloom/blvalidate.c
  • src/backend/access/brin/brin_validate.c
  • src/backend/access/gin/ginvalidate.c
  • src/backend/access/gist/gistvalidate.c

    • (主にインデックスの検証処理において、エラーメッセージ等で演算子名を表示するために使用されています)

関連データ構造

  • Oid: PostgreSQLのシステムオブジェクト識別子。
  • pg_operator: 演算子情報を保持するシステムカタログ。

失敗モードと境界条件

  • 指定された operator_oid が存在しない場合、format_operator_extended の挙動に従い、エラーが発生するか、あるいは無効な名前が返される可能性があります。

保守担当者向けメモ

  • 本関数は format_operator_extended の単なるラッパーです。フォーマットの挙動を変更したい場合は format_operator_extended を修正してください。
  • 戻り値の文字列は呼び出し元で適切に解放（pfree）される必要があります。
