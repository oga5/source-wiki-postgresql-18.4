XmlTableSetDocument

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/xml.c:4730

目的

XMLTABLE 演算子で使用されるドキュメントコンテキストを初期化し、入力されたXMLデータを解析して内部構造（xmlDocPtr および xmlXPathContextPtr）を構築します。

シグネチャまたは形状

static void XmlTableSetDocument(TableFuncScanState *state, Datum value)

入力と出力

  • **入力**:
  • state: TableFuncScanState 構造体へのポインタ。
  • value: 解析対象のXMLデータを含む Datum。
  • **出力**: なし（副作用として state 内の XmlTableBuilderData にドキュメントとXPathコンテキストを格納）。

前提条件と不変条件

  • USE_LIBXML が定義されている環境であること。
  • state から取得される XmlTableBuilderData が有効であること。
  • value は有効な xml 型のデータであること。

アルゴリズムまたは振る舞い

1. GetXmlTableBuilderPrivateData を呼び出し、現在の実行コンテキストを取得します。 2. xml_out_internal を使用して value を文字列に変換します。 3. PG_TRY ブロック内で以下を実行します。

  • xmlCtxtReadMemory を呼び出し、文字列からXMLドキュメントを生成します。
  • 解析失敗時やエラー発生時は xml_ereport でエラーを報告します。
  • xmlXPathNewContext を呼び出し、ドキュメントに対するXPath評価用コンテキストを生成します。
  • XPathコンテキストのノードをドキュメントルートに設定します。

    • 4. PG_CATCH ブロックにて、エラー発生時に確保したドキュメントとXPathコンテキストを解放し、例外を再スローします。 5. 成功時、生成された doc と xpathcxt を xtCxt に保存します。

呼び出し元と呼び出し先

  • **呼び出し元**: XmlTableRoutine 構造体の .SetDocument フィールドを通じて呼び出されます。
  • **呼び出し先**: GetXmlTableBuilderPrivateData, xml_out_internal, pg_xmlCharStrndup, xmlCtxtReadMemory, xml_ereport, xmlXPathNewContext, xmlXPathFreeContext, xmlFreeDoc。

関連データ構造

  • TableFuncScanState: 実行状態を保持する構造体。
  • XmlTableBuilderData: XMLTABLE の構築に必要な内部データを保持する構造体。
  • xmlDocPtr, xmlXPathContextPtr: libxml2 のドキュメントおよびXPathコンテキスト構造体。

失敗モードと境界条件

  • USE_LIBXML が未定義の場合、NO_XML_SUPPORT() マクロによりエラーとなります。
  • XMLの解析に失敗した場合、ERRCODE_INVALID_XML_DOCUMENT でエラーを報告します。
  • メモリ不足等でXPathコンテキストの確保に失敗した場合、ERRCODE_OUT_OF_MEMORY でエラーを報告します。

保守担当者向けメモ

  • 本関数は libxml2 のグローバルな状態に依存するため、PG_TRY/PG_CATCH を用いた確実なリソース解放が必須です。
  • xml_out_internal を経由することで、エンコーディングプロパティを除去した文字列として処理されます。
