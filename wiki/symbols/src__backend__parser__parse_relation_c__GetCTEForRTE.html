GetCTEForRTE

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:564

目的

指定された RangeTblEntry (RTE) に対応する CommonTableExpr (CTE) 定義を、現在のパーサー状態のスコープから検索して取得する。

シグネチャまたは形状

CommonTableExpr *GetCTEForRTE(ParseState *pstate, RangeTblEntry *rte, int rtelevelsup)

入力と出力

  • **入力**:
  • pstate: 現在のクエリ解析状態。
  • rte: 検索対象の RTE_CTE 型の RangeTblEntry。
  • rtelevelsup: RTEが参照するスコープレベルのオフセット。
  • **出力**:
  • 一致する CommonTableExpr 構造体へのポインタ。

前提条件と不変条件

  • rte->rtekind は RTE_CTE である必要がある（Assert で検証）。
  • rte に含まれる ctelevelsup と引数 rtelevelsup の合計値に基づき、親 ParseState を遡る必要がある。

アルゴリズムまたは振る舞い

1. rte->ctelevelsup と rtelevelsup を加算し、対象のCTEが定義されているスコープレベルまで pstate を遡る。 2. 到達した pstate の p_ctenamespace リストを走査する。 3. リスト内の各 CommonTableExpr の ctename と、対象 rte の ctename を比較する。 4. 一致する名前が見つかった場合、その CommonTableExpr を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_target.c (複数箇所)
  • **呼び出し先**:
  • elog (エラー発生時)

関連データ構造

  • RangeTblEntry: クエリ内のテーブルやCTEを表現する構造体。
  • CommonTableExpr: CTEの定義を保持する構造体。
  • ParseState: クエリ解析中の状態を保持する構造体。

失敗モードと境界条件

  • 指定されたレベルまで pstate を遡れない場合、elog(ERROR) により処理を中断する。
  • p_ctenamespace 内に一致する名前のCTEが見つからない場合、elog(ERROR) により処理を中断する。

保守担当者向けメモ

  • 本関数は RTE_CTE 型のRTEに対してのみ有効です。
  • rtelevelsup の計算ロジックは、ネストされたクエリやサブクエリにおけるCTEの可視性管理と密接に関連しています。変更時は ParseState の階層構造に注意してください。
