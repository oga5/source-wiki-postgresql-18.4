bytesToHex

定義

  • 種別: function
  • 定義箇所: src/common/md5_common.c:27

目的

16バイトのバイナリデータを、対応する32文字の16進数文字列（ヌル終端）に変換する内部ヘルパー関数です。

シグネチャまたは形状

static void bytesToHex(uint8 b[16], char *s)

入力と出力

  • **入力**:
  • b[16]: 変換対象の16バイトのバイナリデータ。
  • s: 変換結果を格納するためのバッファ（少なくとも33バイト以上の領域が必要）。
  • **出力**: なし（s が指すメモリ領域に結果が書き込まれます）。

前提条件と不変条件

  • b は16バイトのデータを含んでいる必要があります。
  • s は変換後の32文字とヌル終端文字を格納するために、最低33バイトのメモリ領域を確保している必要があります。

アルゴリズムまたは振る舞い

1. 16バイトの各バイトを順次処理します。 2. 各バイトを上位4ビットと下位4ビットに分割します。 3. それぞれの値を 0123456789abcdef のインデックスとして参照し、対応する16進数文字を取得します。 4. 取得した2文字を順次 s に書き込みます。 5. 全ての処理終了後、s の末尾にヌル終端文字 (\0) を付与します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/md5_common.c 内の pg_md5_hash 等から呼び出されます。
  • **呼び出し先**: 外部関数は呼び出しません。

関連データ構造

  • uint8: バイナリデータの型として使用。

失敗モードと境界条件

  • **境界条件**: s に十分なバッファサイズが割り当てられていない場合、バッファオーバーランが発生します。本関数自体はバッファサイズの検証を行いません。

保守担当者向けメモ

  • 本関数は static であり、md5_common.c 内でのみ使用される内部ユーティリティです。
  • セキュリティ上の理由から、MD5の利用は認証目的以外では推奨されません。
