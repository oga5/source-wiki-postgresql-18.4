ConnectDatabase

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/connectdb.c:39

目的

データベース接続の確立、認証プロンプトの処理、およびサーバーバージョンとの互換性検証を一元的に行う。

シグネチャまたは形状

PGconn *ConnectDatabase(const char *dbname, const char *connection_string,
                        const char *pghost, const char *pgport, const char *pguser,
                        trivalue prompt_password, bool fail_on_error, const char *progname,
                        const char **connstr, int *server_version, char *password,
                        char *override_dbname)

入力と出力

  • **入力**: 接続先情報（dbname, connection_string, pghost, pgport, pguser）、認証設定（prompt_password, password）、動作制御（fail_on_error, progname）、出力用ポインタ（connstr, server_version, override_dbname）。
  • **出力**: 成功時は確立された PGconn オブジェクトへのポインタ。失敗時かつ fail_on_error が false の場合は NULL を返す。

前提条件と不変条件

  • 接続文字列（connection_string）が提供される場合、その中の dbname は無視される。
  • 認証が必要な場合、simple_prompt を用いて対話的にパスワードを取得する。
  • 接続成功後、サーバーバージョンが取得できない場合は致命的エラーとして終了する。

アルゴリズムまたは振る舞い

1. 必要に応じてパスワードを対話的に取得する。 2. PQconninfoParse で接続文字列を解析し、他の引数とマージして PQconnectdbParams を呼び出す。 3. 認証エラーが発生し、かつパスワードが未指定であれば、パスワードを再取得して接続を再試行する。 4. 接続成功後、必要に応じて constructConnStr を用いて接続文字列を生成する。 5. サーバーのバージョンを確認し、呼び出し元が要求していれば server_version 引数に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_db.c、src/bin/pg_dump/pg_dumpall.c など。
  • **呼び出し先**: libpq 関数群（PQconnectdbParams, PQstatus, PQconnectionNeedsPassword, PQfinish, PQparameterStatus, PQserverVersion）、simple_prompt、constructConnStr。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PQconninfoOption: 接続パラメータの解析結果を保持する構造体。

失敗モードと境界条件

  • 接続失敗時、fail_on_error が true であれば pg_fatal によりプログラムを終了させる。
  • サーバーバージョンが解析不能な場合、致命的エラーとして終了する。
  • 接続文字列の解析に失敗した場合、エラーメッセージを出力して終了する。

保守担当者向けメモ

  • 本関数は libpq をラップしており、接続パラメータの優先順位管理が複雑であるため、変更時は libpq の仕様と pg_dump の要件の両方を考慮すること。
  • constructConnStr はセキュリティ上の理由からパスワードや dbname を除外して生成されるため、サブプロセスへの引き継ぎ時に機密情報が漏洩しないよう設計されている。
