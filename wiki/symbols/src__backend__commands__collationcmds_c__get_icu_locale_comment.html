get_icu_locale_comment

定義

  • 種別: function
  • 定義箇所: src/backend/commands/collationcmds.c:645

目的

ICUロケール名に対応する表示名を英語で取得し、コメントとして利用可能な文字列を生成する。

シグネチャまたは形状

char *get_icu_locale_comment(const char *localename)

入力と出力

  • **入力**: localename (ICUロケール名を示す文字列)
  • **出力**: 成功時は英語の表示名を含む動的に割り当てられた文字列（palloc）、失敗時は NULL。

前提条件と不変条件

  • localename は有効なICUロケール名であること。
  • 取得した表示名にASCII範囲外の文字が含まれる場合、本関数は NULL を返す。

アルゴリズムまたは振る舞い

1. uloc_getDisplayName を呼び出し、指定されたロケールの英語表示名を取得する。 2. ICUの処理でエラーが発生した場合は NULL を返す。 3. 取得した表示名（UChar配列）を走査し、すべての文字がASCII範囲内（127以下）であることを確認する。範囲外の文字が含まれる場合は NULL を返す。 4. 条件を満たす場合、palloc でメモリを確保し、UChar から char へ変換してコピーし、ヌル終端文字列として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/collationcmds.c 内のロケール処理箇所（例: 1019行目付近）。
  • **呼び出し先**: ICUライブラリの uloc_getDisplayName。

関連データ構造

  • UChar: ICUの文字型。
  • UErrorCode: ICUのステータス管理用列挙型。

失敗モードと境界条件

  • uloc_getDisplayName が失敗した場合、NULL を返す。
  • 表示名に非ASCII文字が含まれる場合、NULL を返す。
  • メモリ不足時は palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数はASCII文字のみを許可する制約があるため、多言語対応の表示名が必要な場合は制約の緩和を検討すること。
  • displayname バッファサイズ（128）は固定であるため、非常に長いロケール名に対しては注意が必要。
