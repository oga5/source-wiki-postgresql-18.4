PsqlVarEnumError

定義

  • 種別: function
  • 定義箇所: src/bin/psql/variables.c:485

目的

列挙型として定義された psql 変数に対して、許可されていない値が設定された際に標準エラー出力へ警告メッセージを表示する。

シグネチャまたは形状

void PsqlVarEnumError(const char *name, const char *value, const char *suggestions);

入力と出力

  • name: エラーが発生した変数名（文字列）。
  • value: ユーザーが指定した無効な値（文字列）。
  • suggestions: 許可されている値のリストを示す文字列（例: "on, off, auto"）。
  • 出力: なし（void）。副作用として pg_log_error を介してログを出力する。

前提条件と不変条件

  • 引数 name, value, suggestions は NULL であってはならない。
  • pg_log_error が利用可能であること。

アルゴリズムまたは振る舞い

1. pg_log_error を呼び出し、指定された変数名、無効な値、および利用可能な値のリストをフォーマットして出力する。 2. 出力形式は "unrecognized value "value" for "name"\nAvailable values are: suggestions." となる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/psql/command.c: \set コマンド等での変数設定時。
  • src/bin/psql/startup.c: 初期設定値の検証時。
  • 呼び出し先:
  • pg_log_error (PostgreSQL 共通ログ出力関数)。

関連データ構造

  • なし。

失敗モードと境界条件

  • suggestions に空文字列を渡した場合、エラーメッセージには "Available values are: ." と表示される。
  • 引数に NULL を渡した場合、pg_log_error の実装に依存してセグメンテーションフォールトが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、変数の代入フック（Assign Hook）内でバリデーションエラーを報告するために使用される。
  • 新しい列挙型変数を追加する際は、本関数を使用して一貫したエラーメッセージを提供すること。
