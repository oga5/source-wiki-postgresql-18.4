spgvacuumscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgvacuum.c:799

目的

SP-GiST インデックスの物理的な全ページを走査し、不要なタプルの削除、リダイレクト情報の整理、および空き領域の FSM（Free Space Map）への返却を行う。

シグネチャまたは形状

static void spgvacuumscan(spgBulkDeleteState *bds)

入力と出力

  • **入力**: spgBulkDeleteState *bds（VACUUM 実行中の状態、統計情報、保留リスト等を保持する構造体へのポインタ）
  • **出力**: なし（副作用として bds->stats の更新、インデックスメタページの更新、および FSM の更新を行う）

前提条件と不変条件

  • bds が初期化済みであること。
  • インデックスの物理的な全ページ（メタページを除く）を走査する。
  • 走査中に発生した REDIRECT 先の TID は bds->pendingList に蓄積され、ページ処理の合間に spgprocesspending によって処理される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: initSpGistState を呼び出し、スキャン用の統計情報をリセットする。 2. **走査準備**: read_stream_begin_relation を使用し、物理順でインデックスページを読み込むストリームを開始する。 3. **メインループ**:

  • インデックスの現在のブロック数を取得し、全ページを走査するまで繰り返す。
  • read_stream_next_buffer でページを取得し、spgvacuumpage を呼び出して各ページの整理を行う。
  • ページ処理後、bds->pendingList に項目があれば spgprocesspending を呼び出して保留中の TID を処理する。

    • 4. **終了処理**:

  • SpGistUpdateMetaPage を呼び出し、メタページを更新する。
  • 削除されたページがある場合、IndexFreeSpaceMapVacuum を呼び出して FSM を更新する。
  • 最終的な統計情報を bds->stats に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: spgbulkdelete（src/backend/access/spgist/spgvacuum.c 内の複数箇所）
  • **呼び出し先**:
  • initSpGistState, GetActiveSnapshot, read_stream_begin_relation, LockRelationForExtension, RelationGetNumberOfBlocks, vacuum_delay_point, read_stream_next_buffer, spgvacuumpage, spgprocesspending, read_stream_reset, read_stream_end, SpGistUpdateMetaPage, IndexFreeSpaceMapVacuum

関連データ構造

  • spgBulkDeleteState: VACUUM 実行中の状態を保持する主要構造体。
  • ReadStream: ページ走査に使用するストリーム構造体。

失敗モードと境界条件

  • **同時実行**: インデックスの切り詰め（Truncation）は、同時実行中の挿入操作との競合リスクがあるため無効化されている。
  • **ロック**: 新規または一時的なリレーション以外では、ブロック拡張時に ExclusiveLock を取得する。

保守担当者向けメモ

  • 本関数は物理順の走査を行うため、スキャン開始後に新規追加されたリーフページも確実に処理される設計となっている。
  • spgprocesspending を呼び出すタイミングは、ページ処理の整合性を保つために重要である。
