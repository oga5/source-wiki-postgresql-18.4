brin_copy_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_tuple.c:445

目的

既存の BrinTuple をメモリ上の新しい領域、または既存のバッファへコピーします。

シグネチャまたは形状

BrinTuple *brin_copy_tuple(BrinTuple *tuple, Size len, BrinTuple *dest, Size *destsz)

入力と出力

  • **入力**:
  • tuple: コピー元の BrinTuple ポインタ。
  • len: コピーするバイトサイズ。
  • dest: コピー先のバッファ（NULL 指定可能）。
  • destsz: dest バッファの現在のサイズ（バイト単位）。NULL 指定可能。
  • **出力**:
  • コピー先の BrinTuple ポインタを返します。

前提条件と不変条件

  • destsz が NULL または *destsz == 0 の場合、関数内で palloc(len) を用いてメモリが確保されます。
  • len が *destsz を超える場合、repalloc によりバッファが拡張され、*destsz が len に更新されます。

アルゴリズムまたは振る舞い

1. コピー先バッファが未指定またはサイズ不足の場合、palloc または repalloc を用いて必要なメモリを確保・拡張します。 2. memcpy を使用して、tuple から dest へ len バイトのデータをコピーします。 3. 確保または拡張された dest ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c: brin_copy_tuple を複数箇所で呼び出し。
  • src/backend/access/brin/brin_pageops.c: brin_copy_tuple を呼び出し。
  • **呼び出し先**:
  • palloc / repalloc (メモリ管理)
  • memcpy (メモリコピー)

関連データ構造

  • BrinTuple: オンディスク上のBRINタプル構造体。

失敗モードと境界条件

  • メモリ不足時には palloc / repalloc がエラーを発生させます。
  • destsz が NULL の場合、常に新しいメモリ領域が割り当てられます。

保守担当者向けメモ

  • 本関数は、ループ内でのメモリ割り当てを避けるために既存のバッファを再利用する設計になっています。パフォーマンスが重要なパスでは、dest バッファを再利用する呼び出しパターンを優先してください。
