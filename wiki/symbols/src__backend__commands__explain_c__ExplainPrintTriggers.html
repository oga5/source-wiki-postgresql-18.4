ExplainPrintTriggers

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:831

目的

クエリ実行計画に関連するトリガー情報を取得し、EXPLAIN 出力の一部として整形・出力する。

シグネチャまたは形状

void ExplainPrintTriggers(ExplainState *es, QueryDesc *queryDesc)

入力と出力

  • **入力**:
  • es: 出力状態を管理する ExplainState 構造体へのポインタ。
  • queryDesc: 実行中のクエリ情報を含む QueryDesc 構造体へのポインタ。
  • **出力**: なし（es が保持するバッファにトリガー情報が書き込まれる）。

前提条件と不変条件

  • queryDesc->estate が有効であること。
  • ExplainState が初期化済みであること。

アルゴリズムまたは振る舞い

1. queryDesc->estate から、結果リレーション（es_opened_result_relations）、タプルルーティングリレーション（es_tuple_routing_result_relations）、およびトリガー対象リレーション（es_trig_target_relations）のリストを取得する。 2. ExplainOpenGroup を呼び出し、出力形式に応じた "Triggers" グループを開始する。 3. 複数のリレーションが存在する場合、トリガー出力時にリレーション名を表示するフラグ show_relname を有効にする。 4. 取得した各リスト（resultrels, routerels, targrels）を順に走査し、各リレーションに対して report_triggers を呼び出してトリガー情報を出力する。 5. ExplainCloseGroup を呼び出し、"Triggers" グループを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c:633
  • contrib/auto_explain/auto_explain.c:412
  • **呼び出し先**:
  • ExplainOpenGroup / ExplainCloseGroup
  • report_triggers（内部ヘルパー関数）

関連データ構造

  • ExplainState: 出力形式やバッファを管理。
  • QueryDesc: 実行計画と実行状態（EState）を保持。
  • ResultRelInfo: トリガー情報を含むリレーション情報。

失敗モードと境界条件

  • リストが空の場合、トリガー情報は出力されない。
  • show_relname の判定により、単一リレーションの場合はリレーション名が省略される場合がある。

保守担当者向けメモ

  • 本関数は EXPLAIN 出力のトリガーセクションを生成する。トリガーの表示形式を変更する場合は、本関数ではなく report_triggers の修正を検討すること。
