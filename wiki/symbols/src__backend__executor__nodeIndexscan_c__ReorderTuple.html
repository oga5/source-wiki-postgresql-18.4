ReorderTuple

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeIndexscan.c:50

目的

インデックススキャンにおいて、ORDER BY 句の条件を満たすためにタプルを一時的に保持し、再ソートを行うための優先度付きキュー（pairingheap）用ノード構造体。

シグネチャまたは形状

typedef struct {
    pairingheap_node ph_node;   /* pairingheap用ノード */
    HeapTuple    htup;          /* 保持するタプル */
    Datum       *orderbyvals;   /* ORDER BY句の評価値 */
    bool        *orderbynulls;  /* ORDER BY句のNULLフラグ */
} ReorderTuple;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではない。
  • palloc を用いてメモリ確保され、pairingheap に格納される。

前提条件と不変条件

  • pairingheap のノードとして機能するため、ph_node が正しく初期化されている必要がある。
  • orderbyvals および orderbynulls は、対応するインデックススキャンノードの ORDER BY 式評価結果と整合している必要がある。

アルゴリズムまたは振る舞い

  • IndexNextWithReorder において、インデックスから取得したタプルを本構造体に格納し、pairingheap にプッシュする。
  • pairingheap の特性を利用して、ORDER BY 順序に従ったタプルの取り出しを可能にする。
  • 順序評価が必要なタプルをキューに蓄積し、必要に応じて pairingheap_remove_first で最小（または最大）要素を順次取得する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeIndexscan.c 内の IndexNextWithReorder および関連する再ソート処理関数。
  • **呼び出し先**: pairingheap ライブラリの各関数（pairingheap_first, pairingheap_remove_first など）。

関連データ構造

  • pairingheap_node: pairingheap で管理するための基底構造。
  • HeapTuple: PostgreSQLのヒープタプル表現。
  • IndexScanState: 本構造体を管理するキュー（iss_ReorderQueue）を保持する親ノード。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生する可能性がある。
  • キューが空の状態で pairingheap_remove_first を呼び出した場合の挙動は、pairingheap の仕様に依存する。

保守担当者向けメモ

  • 本構造体は nodeIndexscan.c 内でのみ使用されるプライベートな構造体である。
  • orderbyvals と orderbynulls のメモリ管理は、IndexScanState のライフサイクルと同期させる必要がある。
