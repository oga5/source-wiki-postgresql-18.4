gtsquery_same

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_gist.c:106

目的

2つの TSQuerySign（tsquery のシグネチャ）が同一であるかを判定し、GiST インデックスの same メソッドとして機能する。

シグネチャまたは形状

Datum gtsquery_same(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TSQUERYSIGN(0): 比較対象のシグネチャ a
  • PG_GETARG_TSQUERYSIGN(1): 比較対象のシグネチャ b
  • PG_GETARG_POINTER(2): 結果を格納する bool 型のポインタ
  • **出力**:
  • result ポインタ経由で、a == b の真偽値が返される。

前提条件と不変条件

  • 入力される TSQuerySign は、tsquery を圧縮したビットマップ表現であること。
  • result ポインタは有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. 第1引数と第2引数から TSQuerySign 型の値を抽出する。 2. 第3引数から結果格納用のポインタを取得する。 3. 2つのシグネチャ a と b を直接比較（a == b）する。 4. 比較結果を result が指すメモリ領域に書き込み、そのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスのメソッドとして、PostgreSQL のインデックス管理サブシステムから呼び出される（具体的な呼び出し元コードの詳細は提供されていない）。
  • **呼び出し先**: なし。

関連データ構造

  • TSQuerySign: tsquery を表現する固定長のビットマップ型。

失敗モードと境界条件

  • PG_GETARG_POINTER(2) が NULL ポインタである場合、メモリ書き込み時にエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は GiST インデックスの same インターフェースを実装しており、インデックスの重複判定に使用される。
  • 比較ロジックは単純な値の等価比較であるため、TSQuerySign の定義変更がない限り修正は不要である。
