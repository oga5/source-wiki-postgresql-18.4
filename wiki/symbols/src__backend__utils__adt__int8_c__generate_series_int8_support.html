generate_series_int8_support

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/int8.c:1458

目的

generate_series(start, finish, step) 関数のプランナサポート関数として、クエリ実行前に生成される行数を推定します。

シグネチャまたは形状

Datum generate_series_int8_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して SupportRequestRows 構造体へのポインタを受け取ります。
  • **出力**: 推定行数が設定された SupportRequestRows 構造体へのポインタ、または推定不能な場合は NULL を返します。

前提条件と不変条件

  • 入力ノードが SupportRequestRows 型である必要があります。
  • 引数が定数（Const）として評価可能な場合にのみ、行数の計算が行われます。
  • step が 0 の場合、行数の計算は行われません。

アルゴリズムまたは振る舞い

1. rawreq が SupportRequestRows であるかを確認します。 2. estimate_expression_value を使用して、generate_series の引数（start, finish, step）を定数として評価を試みます。 3. 引数のいずれかが NULL 定数である場合、戻り行数を 0 と推定します。 4. すべての引数が定数である場合、以下の式を用いて行数を計算し、req->rows に格納します。

  • rows = floor((finish - start + step) / step)
  • 計算には double 型を使用し、オーバーフローを回避します。

    • 5. 計算結果を PG_RETURN_POINTER で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のプランナ（クエリ最適化プロセス）から呼び出されます。
  • **呼び出し先**: estimate_expression_value を呼び出し、式ツリーの定数評価を行います。

関連データ構造

  • SupportRequestRows: プランナが行数推定を要求するために使用する構造体。
  • FuncExpr: generate_series 関数呼び出しの構文木。
  • Const: 引数の定数表現。

失敗モードと境界条件

  • 引数が定数として評価できない場合、行数推定は行われず NULL が返されます。
  • step が 0 の場合、除算を避けるため行数推定はスキップされます。
  • 引数が NULL 定数の場合は 0 行とみなされます。

保守担当者向けメモ

  • 本関数は generate_series のプランナ最適化を目的としており、実行時のロジック（generate_series_step_int8）とは独立しています。
  • 行数計算式は step の正負に関わらず機能するように設計されています。
