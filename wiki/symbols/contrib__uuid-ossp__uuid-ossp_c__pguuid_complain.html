pguuid_complain

定義

  • 種別: function
  • 定義箇所: contrib/uuid-ossp/uuid-ossp.c:124

目的

OSSP UUID ライブラリから返されたエラーコードを解析し、PostgreSQL のエラー報告メカニズムを通じて例外を発生させる。

シグネチャまたは形状

static void pguuid_complain(uuid_rc_t rc)

入力と出力

  • **入力**: rc (uuid_rc_t 型) - OSSP UUID ライブラリが返すリターンコード。
  • **出力**: なし。関数内で ereport(ERROR, ...) を呼び出すため、呼び出し元には戻らずにトランザクションが中断される。

前提条件と不変条件

  • OSSP UUID ライブラリがエラー状態（成功以外のリターンコード）を返した際に呼び出されることを前提とする。
  • uuid_error(rc) が有効な文字列を返すか、あるいは NULL を返すかのいずれかである必要がある。

アルゴリズムまたは振る舞い

1. uuid_error(rc) を呼び出し、リターンコードに対応するエラーメッセージ文字列を取得する。 2. メッセージが取得できた場合（err != NULL）、ERRCODE_EXTERNAL_ROUTINE_EXCEPTION を用いてそのメッセージを報告する。 3. メッセージが取得できなかった場合、リターンコードの数値を直接含めた汎用的なエラーメッセージを報告する。 4. いずれの場合も ereport の ERROR レベルにより、呼び出し元の処理を停止させる。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/uuid-ossp/uuid-ossp.c 内の各 UUID 生成関数（uuid_generate_internal 等）から、ライブラリ呼び出しの戻り値チェックとして呼び出される。
  • **呼び出し先**: uuid_error（OSSP UUID ライブラリ関数）、ereport（PostgreSQL コア関数）。

関連データ構造

  • uuid_rc_t: OSSP UUID ライブラリで定義されるリターンコード型。

失敗モードと境界条件

  • uuid_error(rc) が NULL を返す場合、数値コードのみを含むエラーメッセージが生成される。
  • 本関数自体がエラーを発生させるため、呼び出し元でこの関数を呼んだ後の後続処理は実行されない。

保守担当者向けメモ

  • 本関数は OSSP UUID ライブラリを使用する環境でのみ有効である。コンパイル時に OSSP UUID が選択されていない場合、この関数は使用されない。
