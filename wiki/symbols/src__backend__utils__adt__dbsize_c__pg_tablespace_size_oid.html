pg_tablespace_size_oid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/dbsize.c:280

目的

指定された OID に対応するテーブルスペースの物理的なディスクサイズ（バイト単位）を計算します。

シグネチャまたは形状

Datum pg_tablespace_size_oid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してテーブルスペースの OID を取得。
  • **出力**: int64 型のバイト数。対象が存在しない場合はエラー、計算不能な場合は NULL を返却。

前提条件と不変条件

  • 指定された OID がシステムカタログ pg_tablespace に存在すること。
  • 実行ユーザーが対象オブジェクトへのアクセス権限を有していること（権限チェックは calculate_tablespace_size 内部で実施）。

アルゴリズムまたは振る舞い

1. SearchSysCacheExists1 を使用し、指定された OID が存在するか確認する。存在しない場合は ERRCODE_UNDEFINED_OBJECT でエラーを発生させる。 2. calculate_tablespace_size(tblspcOid) を呼び出し、物理サイズを算出する。 3. 算出結果が負の値であれば PG_RETURN_NULL() を返す。 4. 正常な値であれば PG_RETURN_INT64(size) を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（詳細な呼び出し元情報はソースコード内に明示的な記述なし）。
  • **呼び出し先**: SearchSysCacheExists1, calculate_tablespace_size。

関連データ構造

  • Oid: テーブルスペース識別子。
  • int64: サイズ保持用データ型。

失敗モードと境界条件

  • **OID 不一致**: 指定した OID が存在しない場合、エラーをスローする。
  • **計算失敗**: calculate_tablespace_size が負の値を返した場合、SQL レベルで NULL を返す。

保守担当者向けメモ

  • 本関数はテーブルスペースの存在確認を事前に行うことで、エラーメッセージの制御を行っています。
  • 物理的なファイル走査を伴うため、実行時間はテーブルスペース内のファイル数に依存します。
