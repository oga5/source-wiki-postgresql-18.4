statext_dependencies_deserialize

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/dependencies.c:498

目的

システムカタログ（pg_statistic_ext_data）に格納されたバイナリ形式の関数従属性データ（bytea）を、メモリ上の MVDependencies 構造体へ復元する。

シグネチャまたは形状

MVDependencies *statext_dependencies_deserialize(bytea *data)

入力と出力

  • **入力**: bytea *data - シリアライズされた関数従属性のバイナリデータ。
  • **出力**: MVDependencies * - 復元された関数従属性の構造体。data が NULL の場合は NULL を返す。

前提条件と不変条件

  • 入力データは STATS_DEPS_MAGIC および STATS_DEPS_TYPE_BASIC に適合している必要がある。
  • 属性数 k は 2 以上 STATS_MAX_DIMENSIONS 以下であること。
  • 復元後の MVDependencies 構造体は、入力された bytea 全体を過不足なく消費している必要がある。

アルゴリズムまたは振る舞い

1. data のヘッダーを検証し、MVDependencies 構造体を初期化する。 2. ヘッダーからマジックナンバー、型、従属性の数（ndeps）を読み取り、整合性をチェックする。 3. ndeps に基づいて MVDependencies 構造体のメモリを再確保（repalloc）する。 4. 各従属性について以下の処理を繰り返す：

  • 従属性の度合い（degree）と属性数（k）を読み取る。
  • MVDependency 構造体を割り当て、属性番号の配列をコピーする。
  • 復元した MVDependency を dependencies->deps 配列に格納する。

    • 5. 最後に、読み取ったデータ長が入力 bytea のサイズと一致することを確認し、構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/dependencies.c 内の統計情報読み込み処理（複数箇所）。
  • **呼び出し先**:
  • palloc0, repalloc (メモリ管理)
  • memcpy (データコピー)
  • elog (エラー報告)

関連データ構造

  • MVDependencies: 関数従属性のリストを保持するメイン構造体。
  • MVDependency: 個別の関数従属性（度合いと属性番号のリスト）を保持する構造体。

失敗モードと境界条件

  • data が NULL の場合は NULL を返す。
  • ヘッダーサイズが不足している場合、またはマジックナンバーや型が期待値と異なる場合は elog(ERROR) で中断する。
  • ndeps が 0 の場合、またはデータ消費量が入力サイズと一致しない場合はエラーとなる。

保守担当者向けメモ

  • 本関数は statext_dependencies_serialize と対になる処理である。データ形式を変更する場合は、両方の関数を同期させる必要がある。
  • Assert による検証が各ステップで行われており、バイナリデータの破損を厳格にチェックする設計となっている。
