AlterPublicationOwner_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/publicationcmds.c:2092

目的

指定されたOIDを持つパブリケーションの所有者を変更する。

シグネチャまたは形状

void AlterPublicationOwner_oid(Oid pubid, Oid newOwnerId)

入力と出力

  • pubid (Oid): 変更対象のパブリケーションのOID。
  • newOwnerId (Oid): 新しい所有者のロールOID。
  • 出力: なし。成功時はシステムカタログを更新し、失敗時はエラーを報告する。

前提条件と不変条件

  • pg_publication システムカタログに対する RowExclusiveLock を保持する必要がある。
  • 指定された pubid に対応するパブリケーションが存在しなければならない。

アルゴリズムまたは振る舞い

1. table_open を呼び出し、pg_publication テーブルを RowExclusiveLock モードで開く。 2. SearchSysCacheCopy1 を使用して、PUBLICATIONOID キャッシュから対象パブリケーションのタプルをコピーとして取得する。 3. タプルが存在しない場合、ERRCODE_UNDEFINED_OBJECT エラーを発生させる。 4. AlterPublicationOwner_internal を呼び出し、実際の所有者変更処理（カタログ更新および権限チェック等）を実行する。 5. heap_freetuple で取得したタプルを解放する。 6. table_close でテーブルを閉じる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_shdepend.c:1695 (ロール削除等の依存関係処理に伴う所有者変更)
  • 呼び出し先: AlterPublicationOwner_internal

関連データ構造

  • pg_publication: パブリケーションの定義を保持するシステムカタログ。
  • HeapTuple: システムカタログの行データ。

失敗モードと境界条件

  • 指定した pubid が存在しない場合、ereport(ERROR, ...) によりトランザクションが中断される。

保守担当者向けメモ

  • 本関数は所有者変更のインターフェースであり、実際のカタログ更新ロジックは AlterPublicationOwner_internal に委譲されている。
  • キャッシュの無効化や権限チェックの詳細は AlterPublicationOwner_internal を参照すること。
