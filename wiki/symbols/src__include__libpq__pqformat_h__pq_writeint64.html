pq_writeint64

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/libpq/pqformat.h:87

目的

64ビット整数値をネットワークバイトオーダー（ビッグエンディアン）に変換し、StringInfo バッファの末尾に書き込みます。

シグネチャまたは形状

static inline void pq_writeint64(StringInfoData *pg_restrict buf, uint64 i)

入力と出力

  • **入力**:
  • buf: 書き込み対象の StringInfoData 構造体へのポインタ。
  • i: 書き込む uint64 型の整数値。
  • **出力**: なし（副作用として buf の内容と len が更新されます）。

前提条件と不変条件

  • buf が指すバッファには、sizeof(uint64) 分の空き容量が既に確保されている必要があります（buf->len + sizeof(uint64) <= buf->maxlen）。
  • この条件は Assert によって検証されます。

アルゴリズムまたは振る舞い

1. pg_hton64 を使用して、入力値 i をホストバイトオーダーからネットワークバイトオーダーへ変換します。 2. buf->data の現在の末尾位置（buf->len）に、変換後の値を memcpy でコピーします。 3. buf->len を sizeof(uint64) 分だけ増加させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/libpq/pqformat.h 内のコードから参照されています。
  • **呼び出し先**:
  • pg_hton64（port/pg_bswap.h 由来）
  • memcpy（標準ライブラリ）

関連データ構造

  • StringInfoData: 通信バッファを管理する構造体（lib/stringinfo.h で定義）。

失敗モードと境界条件

  • バッファの容量不足（buf->len + 8 > buf->maxlen）が発生した場合、アサーション違反によりプログラムが停止します。動的なバッファ拡張が必要な場合は、本関数ではなく pq_sendint64 を使用してください。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁な呼び出しによるオーバーヘッドを最小化する設計となっています。
  • pg_restrict キーワードが使用されており、コンパイラによる最適化が期待されています。
