BF_set_key

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/crypt-blowfish.c:550

目的

入力パスワードから Blowfish 暗号の P-box を初期化し、鍵拡張を行うための関数です。

シグネチャまたは形状

static void BF_set_key(const char *key, BF_key expanded, BF_key initial, int sign_extension_bug)

入力と出力

  • key: 入力パスワード文字列（ヌル終端）。
  • expanded: 拡張された鍵を格納する出力配列。
  • initial: 初期化された P-box 状態を格納する出力配列。
  • sign_extension_bug: 符号拡張の互換性バグを再現するかを指定するフラグ。

前提条件と不変条件

  • key はヌル終端されている必要があります。
  • expanded および initial は BF_N + 2 個の BF_word を保持できるサイズである必要があります。

アルゴリズムまたは振る舞い

1. BF_N + 2 回のループを実行し、各ステップで 4 バイトずつパスワードから読み取ります。 2. パスワード文字列が尽きた場合は、先頭に戻って循環的に読み取ります。 3. sign_extension_bug が真の場合、signed char として符号拡張を行い、偽の場合は unsigned char として扱います。 4. 読み取った 4 バイトを BF_word に結合し、expanded 配列に格納します。 5. 同時に、BF_init_state.P の対応する要素と tmp の XOR をとり、initial 配列に格納します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/crypt-blowfish.c 内の _crypt_blowfish_rn から呼び出されます。
  • 呼び出し先: 直接的な関数呼び出しはありませんが、BF_init_state 構造体を参照します。

関連データ構造

  • BF_key: 鍵データを保持する配列型。
  • BF_word: 暗号処理に使用されるワード型。
  • BF_init_state: Blowfish の初期状態を保持するグローバル構造体。

失敗モードと境界条件

  • パスワードが短い場合、循環的に読み取られる仕様です。
  • sign_extension_bug の設定により、文字の符号解釈が変化するため、bcrypt 互換性の維持に依存します。

保守担当者向けメモ

  • 本関数はオリジナルの bcrypt 実装との互換性を維持するため、意図的な符号拡張の挙動を含んでいます。変更時は注意してください。
  • contrib/pgcrypto/openssl.c 内のコメントによれば、本関数は長いキーを黙って切り詰める仕様であるため、呼び出し側でのキー長管理が必要です。
