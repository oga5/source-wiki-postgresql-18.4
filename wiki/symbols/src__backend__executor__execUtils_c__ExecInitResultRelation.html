ExecInitResultRelation

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execUtils.c:879

目的

クエリ実行状態（EState）において、更新対象となるリレーション（Result Relation）の初期化と登録を行う。

シグネチャまたは形状

void ExecInitResultRelation(EState *estate, ResultRelInfo *resultRelInfo, Index rti)

入力と出力

  • **入力**:
  • estate: クエリ実行状態を保持する EState 構造体へのポインタ。
  • resultRelInfo: 初期化対象の ResultRelInfo 構造体へのポインタ。
  • rti: レンジテーブルインデックス（Range Table Index）。
  • **出力**: なし（副作用として estate 内の構造を更新）。

前提条件と不変条件

  • estate は初期化済みであること。
  • rti は有効なレンジテーブルインデックスであること。

アルゴリズムまたは振る舞い

1. ExecGetRangeTableRelation を呼び出し、指定された rti に対応するリレーションをオープンする。 2. InitResultRelInfo を呼び出し、resultRelInfo をリレーション情報で初期化する。 3. estate->es_result_relations が未割り当ての場合、es_range_table_size に基づいて配列を確保する。 4. estate->es_result_relations の rti - 1 番目の要素に resultRelInfo を格納する。 5. estate->es_opened_result_relations リストの末尾に resultRelInfo を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c
  • src/backend/executor/nodeModifyTable.c
  • **呼び出し先**:
  • ExecGetRangeTableRelation
  • InitResultRelInfo
  • palloc0
  • lappend

関連データ構造

  • EState: クエリ実行状態を管理する構造体。
  • ResultRelInfo: 更新対象リレーションの情報を保持する構造体。
  • Relation: リレーションのメタデータを保持する構造体。

失敗モードと境界条件

  • ExecGetRangeTableRelation が失敗した場合、本関数も処理を継続できない。
  • estate->es_result_relations のメモリ確保失敗時は palloc0 によりエラーが発生する。

保守担当者向けメモ

  • 本関数は es_result_relations 配列と es_opened_result_relations リストの両方を管理しており、一貫性を保つ必要がある。
  • es_opened_result_relations への追加は、全配列を走査せずにオープン済みのリレーションを効率的に処理するために行われる。
