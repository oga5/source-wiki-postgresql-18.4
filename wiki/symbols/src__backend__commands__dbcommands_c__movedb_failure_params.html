movedb_failure_params

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/dbcommands.c:95

目的

データベース移動操作（ALTER DATABASE ... SET TABLESPACE 等）が失敗した際に、エラークリーンアップ処理へ引き渡すためのパラメータを保持する構造体です。

シグネチャまたは形状

typedef struct
{
	Oid			dest_dboid;		/* 移動対象のデータベースOID */
	Oid			dest_tsoid;		/* 移動先のテーブルスペースOID */
} movedb_failure_params;

入力と出力

  • **入力**: 構造体の各フィールド（dest_dboid, dest_tsoid）は、移動操作を開始する際に設定されます。
  • **出力**: エラー発生時にクリーンアップ関数へ渡され、物理的な残骸の特定に使用されます。

前提条件と不変条件

  • dest_dboid および dest_tsoid は、有効なデータベースおよびテーブルスペースのOIDである必要があります。
  • 構造体は、エラー発生時にクリーンアップ処理が実行されるスコープ内でインスタンス化される必要があります。

アルゴリズムまたは振る舞い

データベース移動処理中にエラーが発生した場合、この構造体に格納された情報を用いて、部分的に作成されたディレクトリや一時的な状態を破棄するクリーンアップ処理が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dbcommands.c 内のデータベース移動ロジック（ALTER DATABASE 関連処理）。
  • **呼び出し先**: エラー発生時のクリーンアップコールバック関数（DatumGetPointer を介してキャストされ利用される）。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、この構造体が保持するOIDが不正な場合、クリーンアップ処理が意図した対象を削除できない可能性があります。

保守担当者向けメモ

  • 本構造体は、エラー発生時のリカバリを確実にするための補助データです。移動処理のロジックを変更する際は、クリーンアップが必要な状態を正しく反映するように本構造体の内容を更新してください。
