pg_advisory_lock_shared_int8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:658

目的

ユーザーが指定した64ビット整数値をキーとして、共有モード（ShareLock）のアドバイザリーロックを取得します。

シグネチャまたは形状

Datum pg_advisory_lock_shared_int8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して int64 型のキーを受け取ります。
  • **出力**: void（PG_RETURN_VOID() により返却）。

前提条件と不変条件

  • ロックのキーは、現在のセッションの MyDatabaseId と組み合わされて管理されます。
  • データベースを跨いだロックは行えません。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT64(0) を使用して、引数からロックキーを取得します。 2. SET_LOCKTAG_INT64 マクロを用いて、取得したキーを LOCKTAG 構造体に変換します。 3. LockAcquire 関数を呼び出し、以下の引数でロックを要求します。

  • locktag: 生成したタグ。
  • lockmode: ShareLock（共有ロック）。
  • sessionLevel: true（セッションスコープ）。
  • dontWait: false（ロックが取得できるまで待機する）。

    • 4. ロック取得完了後、制御を戻します。

呼び出し元と呼び出し先

  • **呼び出し先**: LockAcquire（ロックマネージャのコア機能）。
  • **呼び出し元**: 提供されたソース情報には、本関数を直接呼び出している具体的な関数名は記載されていません。

関連データ構造

  • LOCKTAG: ロック対象を一意に識別するための構造体。
  • ShareLock: ロックマネージャで定義される共有ロックモード。

失敗モードと境界条件

  • 指定されたロックが既に他のプロセスによって排他的に保持されている場合、ロックが解放されるまで待機します。
  • ロックマネージャの制限により、システムリソースが枯渇している場合はエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はアドバイザリーロックのSQLインターフェースの一部です。
  • 述語ロック（Predicate Locks）とは管理体系が異なるため、通常のロックマネージャのロジックに従います。
  • 呼び出し元情報の詳細は、現在のソース範囲外のため不明です。
