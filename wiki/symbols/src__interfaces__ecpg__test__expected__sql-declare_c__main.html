目的

ECPG（Embedded SQL for C）プリプロセッサの回帰テスト用プログラムのエントリポイントであり、SQLのDECLAREステートメントやカーソル操作、複数接続環境における動作を検証する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（引数なし）
  • 出力: 成功時に 0 を返す。

前提条件と不変条件

  • ロケールが "C" に設定されていること。
  • ECPGdebug(1, stderr) により、実行時の詳細なデバッグログが標準エラー出力に出力される。
  • 各SQL操作の実行後、sqlca.sqlcode を確認し、負の値（エラー）の場合は sqlprint() を呼び出してエラー情報を出力する。

アルゴリズムまたは振る舞い

1. **初期化**: ロケールの設定とデバッグログの有効化を行う。 2. **接続**: ecpg1_regression および ecpg2_regression データベースへ接続し、それぞれ con1, con2 という名前で識別する。 3. **テーブル作成**: con1 および con2 の両方で source テーブルを作成する。 4. **データ挿入**: 各接続先の source テーブルにテストデータを挿入する。 5. **テスト実行**: commitTable() を呼び出した後、execute_test() を実行してカーソル操作やプリペアドステートメントの検証を行う。 6. **終了処理**: source テーブルを削除し、ECPGdisconnect を用いてすべての接続を終了する。

呼び出し元と呼び出し先

  • 呼び出し元: プログラムのエントリポイントのため、OSにより呼び出される。
  • 呼び出し先:
  • ECPGconnect, ECPGdo, ECPGdisconnect (ECPGライブラリ関数)
  • commitTable()
  • execute_test()
  • sqlprint()

関連データ構造

  • sqlca: ECPGのエラー情報を保持するグローバルな通信領域構造体。

失敗モードと境界条件

  • 各SQL操作において sqlca.sqlcode < 0 となった場合、sqlprint() が実行され、エラー内容が標準エラー出力に書き出される。

保守担当者向けメモ

  • 本関数は .pgc ファイルから生成されたCコードである。#line 指示子により元のソースコードとの対応が維持されている。
  • データベース接続の管理（con1, con2）がテストの核心であるため、接続設定の変更時には注意が必要。
