koi8u_to_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_cyrillic/utf8_and_cyrillic.c:110

目的

KOI8-U エンコーディングから UTF-8 エンコーディングへの文字列変換を行う PostgreSQL のロード可能な変換プロシージャです。

シグネチャまたは形状

Datum koi8u_to_utf8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取ります。
  • src (CSTRING): 変換元文字列。
  • dest (CSTRING): 変換先バッファ。
  • len (INT32): 変換対象の長さ。
  • noError (BOOL): 変換エラー発生時の挙動フラグ。
  • **出力**: Datum 型として、変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_KOI8U, PG_UTF8) により、エンコーディング ID が KOI8-U から UTF-8 への変換として妥当である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. CHECK_ENCODING_CONVERSION_ARGS で引数の妥当性を検証します。 3. LocalToUtf 関数を呼び出し、koi8u_to_unicode_tree マップを使用して変換を実行します。 4. 変換されたバイト数を PG_RETURN_INT32 で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のエンコーディング変換サブシステム（直接的な呼び出し元関数の詳細はソース内に記載なし）。
  • **呼び出し先**: LocalToUtf（PostgreSQL 内部の汎用変換関数）。

関連データ構造

  • koi8u_to_unicode_tree: 変換ロジックに使用される変換マップ。

失敗モードと境界条件

  • noError 引数が false の場合、変換エラー発生時に PostgreSQL のエラーハンドリング機構により処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(koi8u_to_utf8) によりエントリポイントとして登録されています。
  • 変換マップ koi8u_to_utf8.map はビルド時にインクルードされます。
