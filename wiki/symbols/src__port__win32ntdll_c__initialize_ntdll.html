initialize_ntdll

定義

  • 種別: function
  • 定義箇所: src/port/win32ntdll.c:38

目的

Windows環境において、ntdll.dll を動的にロードし、PostgreSQLが必要とするNT層のシステム関数ポインタを解決・初期化します。

シグネチャまたは形状

int initialize_ntdll(void);

入力と出力

  • **入力**: なし
  • **出力**: 成功時は 0 を返し、失敗時は -1 を返します。失敗時には内部で _dosmaperr を呼び出し、Windowsエラーコードを標準の errno に変換します。

前提条件と不変条件

  • initialized フラグにより、二重ロードを防止します。一度成功すると、以降の呼び出しは即座に 0 を返します。
  • スレッド安全性に関する排他制御は行われていません。PostgreSQLの起動初期段階での呼び出しを前提としています。

アルゴリズムまたは振る舞い

1. initialized フラグを確認し、既に初期化済みであれば終了します。 2. LoadLibraryEx を使用して ntdll.dll をロードします。 3. routines 配列に定義された各関数名に対し、GetProcAddress を用いてアドレスを解決します。 4. 解決したアドレスを対応する関数ポインタ変数に格納します。 5. 全ての解決が成功した場合、initialized を true に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/port/open.c:71
  • src/port/win32fdatasync.c:36
  • **呼び出し先**:
  • LoadLibraryEx (Windows API)
  • GetProcAddress (Windows API)
  • FreeLibrary (Windows API)
  • _dosmaperr (PostgreSQL内部関数)

関連データ構造

  • routines: ntdll.dll からロードする関数名と、その格納先ポインタを保持する内部配列。
  • initialized: 初期化状態を保持する静的フラグ。

失敗モードと境界条件

  • ntdll.dll のロード失敗: LoadLibraryEx が失敗した場合、-1 を返します。
  • 関数アドレスの解決失敗: GetProcAddress が NULL を返した場合、既にロードした ntdll.dll を FreeLibrary で解放し、-1 を返します。

保守担当者向けメモ

  • 新しいNT関数を追加する場合は、src/include/port/win32ntdll.h に型定義と extern 宣言を追加し、src/port/win32ntdll.c の routines 配列にエントリを追加してください。
  • 本関数は静的リンクが推奨されない ntdll.dll への安全なアクセス手段を提供します。
