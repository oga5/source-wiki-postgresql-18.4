InternalGrant

定義

  • 種別: struct
  • 定義箇所: src/include/tcop/deparse_utility.h:69
  • 補足: 本シンボルは utils/aclchk_internal.h:31 にも定義が存在します。

目的

権限付与（GRANT/REVOKE）コマンドの実行状態を保持し、DDLの追跡や監査ログ、レプリケーション等の目的でコマンド構造を管理するために使用されます。

シグネチャまたは形状

構造体の詳細は utils/aclchk_internal.h で定義されています。本ファイル（deparse_utility.h）内では、CollectedCommand 構造体の共用体メンバとして以下のように参照されます。

struct {
    InternalGrant *istmt;
} grant;

入力と出力

  • 入力: GrantStmt（通常のパースツリー）から変換された情報を受け取ります。
  • 出力: 権限チェックやカタログ更新処理を行う各関数へ、権限付与のパラメータを伝播します。

前提条件と不変条件

  • CollectedCommand 内で使用される場合、type フィールドが適切なコマンド種別であることを前提とします。
  • 権限チェックや実行処理の過程で、有効な InternalGrant ポインタが保持されている必要があります。

アルゴリズムまたは振る舞い

1. GrantStmt を InternalGrant 形式へ変換します。 2. 変換された InternalGrant を介して、対象オブジェクトの権限チェックやカタログ操作（ExecGrant_Relation 等）を実行します。 3. 階層的なコマンド管理が必要な場合、CollectedCommand の一部として保持されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/aclchk.c 内の ExecGrantStmt_oids 等の関数が本構造体を利用します。
  • 呼び出し先: ExecGrant_Relation、ExecGrant_Language_check、ExecGrant_Largeobject、ExecGrant_Parameter 等の権限実行関数に引数として渡されます。

関連データ構造

  • CollectedCommand: 本構造体を保持する親構造体。
  • GrantStmt: 変換元のパースツリーノード。
  • AclMode: 権限モードの定義。

失敗モードと境界条件

  • InternalGrant ポインタが NULL の状態で実行関数に渡された場合、セグメンテーション違反が発生する可能性があります。
  • 変換元の GrantStmt が不正な場合、InternalGrant への変換処理でエラーが発生します。
