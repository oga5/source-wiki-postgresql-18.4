IndexAttrBitmapKind

定義

  • 種別: enum
  • 定義箇所: src/include/utils/relcache.h:67

目的

リレーションのインデックス属性に関する特定の情報をビットマップ形式で取得する際、その対象種別を指定するために使用されます。

シグネチャまたは形状

typedef enum IndexAttrBitmapKind
{
    INDEX_ATTR_BITMAP_KEY,
    INDEX_ATTR_BITMAP_PRIMARY_KEY,
    INDEX_ATTR_BITMAP_IDENTITY_KEY,
    INDEX_ATTR_BITMAP_HOT_BLOCKING,
    INDEX_ATTR_BITMAP_SUMMARIZED,
} IndexAttrBitmapKind;

入力と出力

  • 本シンボル自体は列挙型であり、関数への引数として渡されることで、取得対象の属性種別を決定します。

前提条件と不変条件

  • nodes/bitmapset.h で定義されるビットマップ操作機能と組み合わせて使用されます。

アルゴリズムまたは振る舞い

  • RelationGetIndexAttrBitmap 関数において、第2引数として渡されることで、以下のいずれかの情報を選択的に取得します。
  • INDEX_ATTR_BITMAP_KEY: インデックスキー属性。
  • INDEX_ATTR_BITMAP_PRIMARY_KEY: 主キー属性。
  • INDEX_ATTR_BITMAP_IDENTITY_KEY: ID キー属性。
  • INDEX_ATTR_BITMAP_HOT_BLOCKING: HOT（Heap Only Tuple）更新を阻害する属性。
  • INDEX_ATTR_BITMAP_SUMMARIZED: 要約された属性。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/relcache.c 内の RelationGetIndexAttrBitmap 関数がこの型を引数として受け取ります。
  • 呼び出し先: 本シンボル自体は列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • Relation: リレーション記述子。
  • Bitmapset: インデックス属性を保持するために使用されるデータ構造（nodes/bitmapset.h 依存）。

失敗モードと境界条件

  • 未定義の列挙値を渡した場合の挙動は、呼び出し側の実装（RelationGetIndexAttrBitmap）に依存します。

保守担当者向けメモ

  • 新たなインデックス属性の分類が必要になった場合は、この列挙型に値を追加し、RelationGetIndexAttrBitmap の実装を更新してください。
