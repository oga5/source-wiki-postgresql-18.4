rt_num_entries

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_radixtree/test_radixtree.c:110

目的

指定された適応型基数木（Radix Tree）に現在格納されているキーの総数を取得します。

シグネチャまたは形状

static uint64 rt_num_entries(rt_radix_tree *tree)

入力と出力

  • **入力**: tree (基数木構造体 rt_radix_tree へのポインタ)
  • **出力**: uint64 型のキー総数

前提条件と不変条件

  • tree は有効な rt_radix_tree 構造体を指している必要があります。
  • tree->ctl が初期化されていることが前提となります。

アルゴリズムまたは振る舞い

1. 引数として受け取った tree の制御構造体（ctl）にアクセスします。 2. ctl 構造体内に保持されている num_keys フィールドの値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_radixtree/test_radixtree.c 内のテスト関数（test_empty 等）から、木が空であることの検証（EXPECT_TRUE(rt_num_entries(radixtree) == 0)）に使用されています。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行いません。構造体のメンバ参照のみを行います。

関連データ構造

  • rt_radix_tree: 基数木のメイン構造体。
  • rt_radix_tree_ctl (推測): tree->ctl が参照する制御構造体。

失敗モードと境界条件

  • tree が NULL の場合、デリファレンスによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は lib/radixtree.h で定義された基数木実装の内部状態をテスト目的で読み取るためのラッパーです。
  • 共有メモリ（DSA）対応のテスト時においても、ctl 経由で正しく値が取得できることを前提としています。
