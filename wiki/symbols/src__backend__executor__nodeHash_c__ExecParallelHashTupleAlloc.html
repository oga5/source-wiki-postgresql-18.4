ExecParallelHashTupleAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:2975

目的

並列ハッシュ結合において、共有メモリ（DSA）上にタプルを格納するためのメモリ領域を確保する。

シグネチャまたは形状

HashJoinTuple ExecParallelHashTupleAlloc(HashJoinTable hashtable, size_t size, dsa_pointer *shared)

入力と出力

  • **入力**:
  • hashtable: ハッシュ結合の状態を保持する HashJoinTable 構造体。
  • size: 確保するタプルのサイズ（バイト単位）。
  • shared: 確保された領域の DSA ポインタを格納する出力用引数。
  • **出力**:
  • 確保されたメモリ領域を指す HashJoinTuple 型のポインタ。確保に失敗した場合（成長処理が必要な場合など）は NULL を返す。

前提条件と不変条件

  • size は MAXALIGN によってアライメント調整される。
  • hashtable->area が有効な DSA（Dynamic Shared Area）を指していること。
  • 並列実行時、バッチやバケットの拡張が必要な場合は、呼び出し元で再試行を行う必要がある。

アルゴリズムまたは振る舞い

1. **高速パス**: 現在のバックエンドが保持する current_chunk に十分な空きがある場合、ロックを取得せずにメモリを割り当て、shared ポインタを更新して即座に返す。 2. **低速パス**:

  • pstate->lock（排他ロック）を取得する。
  • 他の参加者がバッチやバケットの拡張を要求している場合、ロックを解放し、拡張処理（ExecParallelHashIncreaseNumBatches 等）を呼び出した後 NULL を返す。
  • メモリ制限や負荷係数（load factor）の閾値を超過した場合、拡張フラグを立ててロックを解放し NULL を返す。
  • 条件を満たす場合、dsa_allocate を使用して新しいチャンクを確保し、hashtable のチャンクリストに追加する。
  • 確保したチャンクを current_chunk として更新し、ロックを解放して確保した領域を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c 内の複数箇所（ExecHashTableInsert 等の文脈で利用）。
  • **呼び出し先**:
  • ExecParallelHashIncreaseNumBatches
  • ExecParallelHashIncreaseNumBuckets
  • dsa_allocate / dsa_get_address
  • LWLockAcquire / LWLockRelease

関連データ構造

  • HashJoinTable: ハッシュ結合の全状態を保持。
  • ParallelHashJoinState: 並列実行時の共有状態。
  • HashMemoryChunk: 共有メモリ上のチャンク管理構造。
  • dsa_pointer: DSA 内のメモリ位置を示すポインタ。

失敗モードと境界条件

  • **拡張要求**: バッチやバケットの拡張が必要な場合、この関数は NULL を返し、呼び出し元に再試行を促す。
  • **メモリ不足**: space_allowed を超過した場合、バッチ拡張フラグを立てて NULL を返す。
  • **巨大タプル**: HASH_CHUNK_THRESHOLD を超えるタプルは、通常のチャンク共有を行わず、個別のチャンクとして確保される。

保守担当者向けメモ

  • 本関数は並列実行時の共有メモリ管理の核心部であり、ロックの取得範囲がパフォーマンスに直結する。
  • current_chunk の更新ロジックは、並列ワーカー間でのメモリ断片化を抑制するように設計されている。
  • 呼び出し元は、本関数が NULL を返した場合の再試行ロジックを必ず実装すること。
