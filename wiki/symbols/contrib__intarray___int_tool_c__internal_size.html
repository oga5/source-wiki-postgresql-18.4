internal_size

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:294

目的

整数配列で表現された範囲の集合の合計サイズ（要素数）を計算します。

シグネチャまたは形状

int internal_size(int *a, int len)

入力と出力

  • **入力**:
  • a: 範囲のペア（開始値と終了値）が交互に格納された整数配列へのポインタ。
  • len: 配列 a の要素数。
  • **出力**:
  • 計算された合計サイズを int 型で返します。オーバーフローが発生した場合は -1 を返します。

前提条件と不変条件

  • 入力配列 a は、範囲のペア（a[0] が開始、a[1] が終了、a[2] が次の開始…）として構成されている必要があります。
  • int64 を用いて計算を行うため、中間結果が int の範囲を超えても一時的に保持可能です。

アルゴリズムまたは振る舞い

1. 配列を 2 要素ずつ走査します。 2. 各ペアについて、直前の範囲と重複していない場合（!i || a[i] != a[i - 1]）に、その範囲のサイズ（a[i+1] - a[i] + 1）を size に加算します。 3. 最終的な size が INT_MAX を超えるか INT_MIN 未満である場合はオーバーフローとみなし -1 を返します。 4. 正常な場合は size を int にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c:272
  • contrib/intarray/_int_gist.c:334
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • int 型の配列（範囲ペアを保持）。

失敗モードと境界条件

  • **オーバーフロー**: 計算結果が int 型の範囲（INT_MIN ～ INT_MAX）を超えた場合、-1 を返します。
  • **重複範囲**: 連続する範囲が重複している場合、重複分はカウントされません。

保守担当者向けメモ

  • 本関数は _int_gist.c における GiST インデックスのサイズ計算に使用されています。
  • 範囲のペアが正しくソート・構成されていることを前提としています。入力データの整合性は呼び出し側で担保してください。
