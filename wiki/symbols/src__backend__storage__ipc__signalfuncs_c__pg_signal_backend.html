pg_signal_backend

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/signalfuncs.c:50

目的

指定されたPIDを持つPostgreSQLバックエンドプロセスに対して、権限チェックを行った上でシグナルを送信する。

シグネチャまたは形状

static int pg_signal_backend(int pid, int sig);

入力と出力

  • **入力**:
  • pid: 対象となるバックエンドプロセスのプロセスID。
  • sig: 送信するシグナル番号（例: SIGINT, SIGTERM）。
  • **出力**:
  • 成功時は SIGNAL_BACKEND_SUCCESS を返す。
  • 失敗時は SIGNAL_BACKEND_ERROR、SIGNAL_BACKEND_NOAUTOVAC、SIGNAL_BACKEND_NOSUPERUSER、または SIGNAL_BACKEND_NOPERMISSION のいずれかを返す。

前提条件と不変条件

  • BackendPidGetProc(pid) により、対象PIDが有効なバックエンドプロセスとして認識される必要がある。
  • 呼び出し元は、対象プロセスに対する適切な権限（スーパーユーザー権限、または対象ロールの権限）を有している必要がある。
  • オートバキュームワーカに対しては、ROLE_PG_SIGNAL_AUTOVACUUM_WORKER 権限が必要。

アルゴリズムまたは振る舞い

1. BackendPidGetProc を呼び出し、PIDから PGPROC 構造体を取得する。取得できない場合は警告を発しエラーを返す。 2. ターゲットのロール権限を検証する。

  • スーパーユーザーまたはロール未設定のプロセスの場合、スーパーユーザー権限を要求する（ただし、オートバキュームワーカは例外）。
  • それ以外の場合、呼び出し元がターゲットロールの権限を持つか、ROLE_PG_SIGNAL_BACKEND 権限を持つかを確認する。

    • 3. 権限検証を通過した場合、kill() システムコールを実行する。

  • HAVE_SETSID が定義されている環境では、プロセスグループ全体（-pid）に対してシグナルを送信する。
  • それ以外の場合は、単一プロセス（pid）に対して送信する。

    • 4. kill() が失敗した場合は警告を発し、エラーコードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pg_cancel_backend (SQL関数)
  • pg_terminate_backend (SQL関数)
  • **呼び出し先**:
  • BackendPidGetProc
  • GetNumberFromPGProc
  • pgstat_get_backend_type_by_proc_number
  • superuser / superuser_arg
  • has_privs_of_role
  • kill (システムコール)

関連データ構造

  • PGPROC: バックエンドプロセスの状態を保持する構造体。
  • BackendType: プロセスの種類（B_AUTOVAC_WORKER 等）。

失敗モードと境界条件

  • **PID無効**: 対象PIDがバックエンドとして存在しない場合、警告を出して終了する。
  • **権限不足**: 必要な権限がない場合、適切なエラーコードを返してシグナル送信を拒否する。
  • **競合状態**: BackendPidGetProc 呼び出し後から kill 実行までの間にプロセスが終了した場合、kill が失敗する可能性があるが、これは警告として処理される。

保守担当者向けメモ

  • 本関数はSQLインターフェースのバックエンドロジックとして機能する。
  • PIDの再利用による誤ったプロセスへのシグナル送信リスクは、現在のPID割り当てロジック上極めて低いと想定されている。
  • 権限エラーは呼び出し元で ereport(ERROR, ...) を発行してトランザクションを中断させる設計となっている。
