ResourceOwnerRememberHMAC

定義

  • 種別: function
  • 定義箇所: src/common/hmac_openssl.c:85

目的

PostgreSQLのバックエンド環境において、HMAC計算用コンテキスト（pg_hmac_ctx）を現在のリソース所有者（ResourceOwner）に登録し、トランザクション終了時やエラー発生時のメモリリークを防止する。

シグネチャまたは形状

static inline void ResourceOwnerRememberHMAC(ResourceOwner owner, pg_hmac_ctx *ctx)

入力と出力

  • owner: リソースを管理する ResourceOwner オブジェクト。
  • ctx: 登録対象の pg_hmac_ctx 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • バックエンド環境でのみ使用されることを想定している。
  • ctx は有効な pg_hmac_ctx 構造体を指している必要がある。
  • hmac_resowner_desc がリソース管理用の記述子として定義されている必要がある。

アルゴリズムまたは振る舞い

1. PointerGetDatum(ctx) を使用して、コンテキストポインタを Datum 型に変換する。 2. ResourceOwnerRemember 関数を呼び出し、指定された owner に対して ctx をリソースとして登録する。この際、hmac_resowner_desc をリソースの識別および解放処理の定義として渡す。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/hmac_openssl.c 内のコンテキスト作成処理（pg_hmac_create 等）から呼び出される。
  • 呼び出し先: ResourceOwnerRemember（PostgreSQLのコアリソース管理機能）。

関連データ構造

  • pg_hmac_ctx: HMAC計算の状態を保持する構造体。
  • ResourceOwner: リソースのライフサイクルを管理するオブジェクト。
  • hmac_resowner_desc: リソース所有者管理のためのメタデータ（記述子）。

失敗モードと境界条件

  • owner が NULL の場合、ResourceOwnerRemember の内部実装に依存する（通常はアサーション失敗やセグメンテーション違反の可能性がある）。
  • メモリ不足等のリソース登録失敗時は、ResourceOwnerRemember がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、主に src/common/hmac_openssl.c 内での利用に限定されている。
  • バックエンド環境以外（フロントエンド等）では、この関数は使用されないか、あるいはマクロ等で無効化される設計となっている。
