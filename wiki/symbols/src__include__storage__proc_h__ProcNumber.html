ProcNumber

定義

  • 種別: struct（PGPROC 構造体内のメンバとして定義）
  • 定義箇所: src/include/storage/proc.h:208

目的

PostgreSQLのバックエンドプロセスを一意に識別するための識別子です。通常のバックエンド、準備されたトランザクション、および未使用のプロセスエントリを区別するために使用されます。

シグネチャまたは形状

typedef された整数型（storage/procnumber.h で定義）であり、PGPROC 構造体内の vxid 構造体の一部として保持されます。

入力と出力

  • 入力: GetNumberFromPGProc(proc) を介して取得される値。
  • 出力: プロセス識別子（ProcNumber 型）。未使用時は INVALID_PROC_NUMBER を返します。

前提条件と不変条件

  • 通常のバックエンドでは、GetNumberFromPGProc(proc) の戻り値と一致する必要があります。
  • 準備されたトランザクション（prepared xacts）の場合、そのトランザクションを処理した元のバックエンドの ID を保持します。
  • 未使用の PGPROC エントリには INVALID_PROC_NUMBER が設定されます。

アルゴリズムまたは振る舞い

プロセス管理において、バックエンドのライフサイクルやトランザクションの所有権を追跡するために参照されます。特に ProcGlobal を通じたプロセス状態の集約や、ロックマネージャでのプロセス特定に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/access/transam/multixact.c: PreparedXactOldestMemberMXactIdSlot 等
  • src/backend/access/transam/parallel.c: parallel_leader_proc_number
  • src/backend/access/transam/twophase.c: locking_backend
  • src/backend/access/transam/xlog.c: walwriterProc
  • src/backend/catalog/catalog.c, namespace.c, storage.c
  • 呼び出し先:
  • 提供されたソースからは、具体的な関数呼び出し先は特定できません。

関連データ構造

  • PGPROC: プロセス制御構造体。
  • vxid: ProcNumber と LocalTransactionId を保持する構造体。
  • PROC_HDR: プロセス管理のグローバルヘッダー。

失敗モードと境界条件

  • INVALID_PROC_NUMBER: プロセスが未使用または無効であることを示す境界値です。

保守担当者向けメモ

  • PGPROC 内の vxid.procNumber は、並行制御において重要な識別子です。ProcArrayLock 等の適切なロックなしにアクセスすると不整合を招く可能性があるため、変更時は ProcArray 周りの設計ドキュメントを併読してください。
