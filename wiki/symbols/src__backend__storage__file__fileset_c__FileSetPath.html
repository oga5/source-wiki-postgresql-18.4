FileSetPath

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fileset.c:171

目的

指定された FileSet とテーブルスペース OID に基づき、一時ファイルセットを格納するためのディレクトリパスを生成する。

シグネチャまたは形状

static void FileSetPath(char *path, FileSet *fileset, Oid tablespace)

入力と出力

  • **入力**:
  • path: パス文字列を格納するためのバッファ（MAXPGPATH 以上のサイズが必要）。
  • fileset: 対象となる FileSet 構造体へのポインタ。
  • tablespace: 配置先のテーブルスペース OID。
  • **出力**:
  • path 引数に、生成されたディレクトリパスが書き込まれる。

前提条件と不変条件

  • path バッファは MAXPGPATH の長さを保持できること。
  • fileset は FileSetInit によって初期化済みであり、creator_pid および number が有効な値であること。

アルゴリズムまたは振る舞い

1. TempTablespacePath を呼び出し、指定された tablespace に対応する一時ディレクトリのベースパスを取得する。 2. snprintf を使用し、ベースパス、PG_TEMP_FILE_PREFIX、fileset->creator_pid、fileset->number を組み合わせて、.../pgsql_tmp/pgsql_tmp<pid>.<number>.fileset という形式のパスを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/fileset.c 内の複数箇所（ディレクトリパスの構築が必要な処理）。
  • **呼び出し先**:
  • TempTablespacePath（commands/tablespace.h 由来）

関連データ構造

  • FileSet: 一時ファイルセットを管理する構造体。
  • Oid: テーブルスペースを識別する型。

失敗モードと境界条件

  • path バッファが MAXPGPATH より小さい場合、パスが切り詰められる可能性がある。
  • tablespace が無効な場合、TempTablespacePath の挙動に依存する。

保守担当者向けメモ

  • 本関数は static であり、fileset.c 内でのみ使用される内部ヘルパー関数である。
  • パス形式を変更する場合は、FileSet を利用する他の関数（FileSetCreate 等）との整合性に注意すること。
