ListenActionKind

定義

  • 種別: enum
  • 定義箇所: src/backend/commands/async.c:332

目的

非同期通知機能（NOTIFY, LISTEN, UNLISTEN）における、リスナー管理操作の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	LISTEN_LISTEN,
	LISTEN_UNLISTEN,
	LISTEN_UNLISTEN_ALL,
} ListenActionKind;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: LISTEN_LISTEN（購読開始）、LISTEN_UNLISTEN（特定チャネルの購読解除）、LISTEN_UNLISTEN_ALL（全チャネルの購読解除）のいずれかの識別子。

前提条件と不変条件

  • async.c 内のリスナー管理ロジックにおいて、アクションの種類を判別する唯一の手段として使用されます。

アルゴリズムまたは振る舞い

  • queue_listen 関数において、実行すべきリスナー操作を決定するための引数として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/async.c 内の queue_listen 関数がこの型を引数として受け取ります。
  • 呼び出し先: なし（列挙型のため、直接の関数呼び出しは発生しません）。

関連データ構造

  • ListenAction: この列挙型をメンバとして保持する構造体。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 新たなリスナー操作が追加される場合は、この列挙型に値を追加し、queue_listen 内の処理を更新する必要があります。
