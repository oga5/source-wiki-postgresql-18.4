SpGistInitBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgutils.c:721

目的

SP-GiSTインデックスのバッファ（ページ）を初期化し、指定されたフラグを設定します。

シグネチャまたは形状

void SpGistInitBuffer(Buffer b, uint16 f)

入力と出力

  • **入力**:
  • b: 初期化対象のバッファ。
  • f: ページに設定するフラグ（SPGIST_LEAF や SPGIST_NULLS など）。
  • **出力**: なし（副作用としてバッファの内容が初期化されます）。

前提条件と不変条件

  • BufferGetPageSize(b) == BLCKSZ であること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. 指定されたバッファ b のページサイズがシステム標準の BLCKSZ と一致することを確認します。 2. BufferGetPage(b) を介してページを取得し、SpGistInitPage を呼び出してページヘッダーの初期化およびフラグ f の適用を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgdoinsert.c
  • src/backend/access/spgist/spginsert.c
  • src/backend/access/spgist/spgutils.c
  • src/backend/access/spgist/spgxlog.c
  • **呼び出し先**:
  • SpGistInitPage

関連データ構造

  • Buffer: PostgreSQLのバッファ管理構造体。
  • Page: ページレイアウトを管理するデータ構造。

失敗モードと境界条件

  • バッファサイズが BLCKSZ と異なる場合、アサーション違反により異常終了します。

保守担当者向けメモ

  • 本関数は、新規に割り当てられたページや再利用されるページをSP-GiSTの物理構造として初期化するための標準的な手段です。
  • ページフラグ f の定義については access/spgist_private.h を参照してください。
