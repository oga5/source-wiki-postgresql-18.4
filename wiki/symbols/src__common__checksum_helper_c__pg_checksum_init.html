pg_checksum_init

定義

  • 種別: function
  • 定義箇所: src/common/checksum_helper.c:82

目的

チェックサム計算のためのコンテキスト構造体（pg_checksum_context）を初期化し、指定されたアルゴリズムに応じた内部状態を準備します。

シグネチャまたは形状

int pg_checksum_init(pg_checksum_context *context, pg_checksum_type type);

入力と出力

  • **入力**:
  • context: 初期化対象となる pg_checksum_context 構造体へのポインタ。
  • type: 使用するチェックサムアルゴリズム（pg_checksum_type 列挙型）。
  • **出力**:
  • 成功時は 0 を返します。
  • SHA系アルゴリズムでメモリ確保や初期化に失敗した場合は -1 を返します。

前提条件と不変条件

  • SHA系アルゴリズム（SHA224/256/384/512）を選択した場合、内部で動的にメモリが確保されます。
  • 呼び出し元は、計算終了後に必ず pg_checksum_final を呼び出し、リソースを解放する責任があります。

アルゴリズムまたは振る舞い

1. context->type に指定された type を設定します。 2. type に基づき以下の処理を行います：

  • CHECKSUM_TYPE_NONE: 何も行いません。
  • CHECKSUM_TYPE_CRC32C: INIT_CRC32C マクロを使用して CRC32C コンテキストを初期化します。
  • CHECKSUM_TYPE_SHA224/256/384/512:
  • pg_cryptohash_create でハッシュコンテキストを生成します。
  • 生成に失敗した場合は -1 を返します。
  • pg_cryptohash_init でハッシュ計算を初期化します。初期化に失敗した場合は pg_cryptohash_free でメモリを解放し、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/bin/pg_combinebackup/backup_label.c
  • src/bin/pg_combinebackup/pg_combinebackup.c
  • src/bin/pg_combinebackup/reconstruct.c
  • src/bin/pg_combinebackup/write_manifest.c
  • src/bin/pg_verifybackup/astreamer_verify.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • **呼び出し先**:
  • pg_cryptohash_create
  • pg_cryptohash_init
  • pg_cryptohash_free

関連データ構造

  • pg_checksum_context: チェックサム計算の状態を保持する構造体。
  • pg_checksum_type: アルゴリズムを指定する列挙型。

失敗モードと境界条件

  • SHA系アルゴリズムにおいて、システムメモリ不足等で pg_cryptohash_create が NULL を返した場合、関数は -1 を返して終了します。
  • pg_cryptohash_init が失敗した場合、確保済みのメモリを解放した上で -1 を返します。

保守担当者向けメモ

  • 本関数は pg_checksum_update および pg_checksum_final とセットで利用されるライフサイクル管理の起点です。
  • SHA系アルゴリズムを使用する場合、メモリリークを防ぐため、必ず対応する pg_checksum_final が実行されるフローであることを確認してください。
