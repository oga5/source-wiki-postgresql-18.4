array_iter

定義

  • 種別: struct
  • 定義箇所: src/include/utils/arrayaccess.h:33

目的

PostgreSQLの配列データ（フラットな配列および展開された配列）に対して、要素単位で順次アクセスするための状態管理用構造体です。

シグネチャまたは形状

typedef struct array_iter
{
    Datum      *datumptr;    /* 展開済み配列用: Datum配列へのポインタ */
    bool       *isnullptr;   /* 展開済み配列用: NULLフラグ配列へのポインタ */
    char       *dataptr;     /* フラット配列用: データ領域の現在位置 */
    bits8      *bitmapptr;   /* フラット配列用: NULLビットマップの現在バイト */
    int         bitmask;     /* フラット配列用: 現在のビットマスク */
} array_iter;

入力と出力

本構造体はデータ保持用であり、直接的な入出力関数ではありません。array_iter_setup() によって初期化され、array_iter_next() の実行時に内部状態が更新されます。

前提条件と不変条件

  • datumptr が NULL か否かで、対象がフラットな配列か展開された配列かが判別されます。
  • フラットな配列を走査する場合、呼び出し側は要素型に応じた elmlen, elmbyval, elmalign を正確に提供する必要があります。

アルゴリズムまたは振る舞い

  • **展開済み配列**: datumptr および isnullptr を使用し、インデックスによる直接アクセスを行います。
  • **フラットな配列**: dataptr を要素サイズとアライメントに基づいて進め、bitmapptr と bitmask を用いてNULL状態を判定・更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arrayfuncs.c 内の複数の関数（array_iter 型の変数を宣言して使用）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、操作は array_iter_setup および array_iter_next が行います。

関連データ構造

  • ArrayType: フラットな配列の内部表現。
  • ExpandedArrayHeader: 展開された配列の内部表現。

失敗モードと境界条件

  • 順次アクセス専用です。ランダムアクセスが必要な場合は、本構造体ではなくインデックス計算による直接アクセスが必要です。
  • 展開済み配列の内部にフラットな配列が含まれる場合、array_iter_setup が適切に分岐処理を行う必要があります。

保守担当者向けメモ

  • 本構造体はインライン関数での利用を想定した軽量なインターフェースの一部です。フィールドの追加や変更を行う際は、array_iter_next の走査ロジックへの影響を考慮してください。
