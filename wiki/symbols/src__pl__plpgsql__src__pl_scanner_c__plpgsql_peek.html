plpgsql_peek

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_scanner.c:451

目的

現在の字句解析位置から次のトークンを消費せずに先読み（peek）し、そのトークン種別を返します。

シグネチャまたは形状

int plpgsql_peek(yyscan_t yyscanner)

入力と出力

  • **入力**: yyscan_t yyscanner（スキャナの状態を保持するハンドル）
  • **出力**: int（先読みしたトークンの識別子）

前提条件と不変条件

  • yyscanner は plpgsql_scanner_init によって初期化済みである必要があります。
  • 内部的に push_back_token を使用するため、プッシュバックスタックの容量（MAX_PUSHBACKS = 4）を超えない範囲で呼び出される必要があります。

アルゴリズムまたは振る舞い

1. internal_yylex を呼び出し、現在の入力ストリームから次のトークン tok1 とその補助データ aux1 を取得します。 2. push_back_token を呼び出し、取得した tok1 と aux1 をスキャナのプッシュバックスタックに戻します。 3. 取得した tok1 を呼び出し元に返します。これにより、次回の字句解析呼び出しで同じトークンが再度取得されます。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • internal_yylex: トークンの取得に使用。
  • push_back_token: 取得したトークンの復元に使用。
  • **呼び出し元**:
  • 提供されたソース情報には、具体的な呼び出し元の関数名は記載されていません。plpgsql.h:1321 に外部宣言が存在します。

関連データ構造

  • yyscan_t: スキャナの内部状態を保持する不透明なポインタ型。
  • TokenAuxData: トークンに関連する補助情報（位置情報や値など）を保持する構造体。

失敗モードと境界条件

  • プッシュバックスタックが満杯の状態で呼び出された場合、push_back_token の実装依存の挙動（エラーまたはスタックオーバーフロー）が発生する可能性があります。

保守担当者向けメモ

  • 本関数はトークンを消費せずに状態を維持する「先読み」専用です。2つ先のトークンを確認したい場合は plpgsql_peek2 を使用してください。
