pg_get_logical_snapshot_info

定義

  • 種別: function
  • 定義箇所: contrib/pg_logicalinspect/pg_logicalinspect.c:133

目的

論理デコーディングで使用されるスナップショットファイル（.snap）の内容を読み込み、その内部状態（SnapBuildState、トランザクションID範囲、コミット済みトランザクション配列など）をSQLの行形式で出力する。

シグネチャまたは形状

Datum pg_get_logical_snapshot_info(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: SQL関数引数としてスナップショットファイル名（text型）を受け取る。
  • **出力**: 14列からなる複合型（Datum）を返す。

前提条件と不変条件

  • 呼び出し元はPostgreSQLのSQL関数インターフェースを介して実行する必要がある。
  • 戻り値の型は複合型（TYPEFUNC_COMPOSITE）である必要がある。
  • 内部的に SnapBuildRestoreSnapshot を使用するため、PostgreSQLの内部構造体レイアウトと整合している必要がある。
  • 処理終了時に values 配列の要素数が PG_GET_LOGICAL_SNAPSHOT_INFO_COLS（14）と一致すること。

アルゴリズムまたは振る舞い

1. get_call_result_type で戻り値の型を検証する。 2. parse_snapshot_filename を呼び出し、ファイル名からLSNを抽出する。 3. SnapBuildRestoreSnapshot を呼び出し、ディスク上のバイナリデータを SnapBuildOnDisk 構造体に復元する。 4. 復元された ondisk.builder の各フィールドを values 配列にマッピングする。

  • トランザクションID配列（committed および catchange）については、construct_array_builtin を使用してSQL配列型に変換する。

    • 5. heap_form_tuple を使用してタプルを構築し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロによる登録）。
  • **呼び出し先**:
  • get_call_result_type
  • parse_snapshot_filename
  • SnapBuildRestoreSnapshot
  • get_snapbuild_state_desc
  • construct_array_builtin
  • heap_form_tuple

関連データ構造

  • SnapBuildOnDisk: ディスク上のスナップショット状態を保持する構造体。
  • SnapBuildState: 論理デコーディングのビルド状態。
  • TupleDesc: SQL関数の戻り値のタプル定義。

失敗モードと境界条件

  • 戻り値の型が複合型でない場合、elog(ERROR) を発生させる。
  • ファイル名が解析できない場合、parse_snapshot_filename がエラーを発生させる可能性がある。
  • トランザクションID配列が空の場合、対応する列は NULL として扱われる。

保守担当者向けメモ

  • 本関数は SnapBuildRestoreSnapshot という内部APIに依存しているため、PostgreSQLのバージョンアップ時に構造体のレイアウト変更の影響を受ける可能性がある。
  • トランザクションID配列の構築には palloc を使用しているため、メモリ管理には注意すること。
