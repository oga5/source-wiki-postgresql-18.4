pos_trgm

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_op.c:52

目的

文字列内のトリグラム（3文字の連続）とその出現位置情報を保持し、単語単位の類似度計算（Word Similarity）においてスライディングウィンドウ的な探索を行うために使用されます。

シグネチャまたは形状

typedef struct
{
	trgm		trg;
	int			index;
} pos_trgm;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • trg: trgm 型のトリグラムデータ。
  • index: 文字列内でのトリグラムの開始位置を示す整数値。

前提条件と不変条件

  • index は対象文字列内での有効なオフセットを指す必要があります。
  • word_similarity 計算の文脈では、この構造体の配列がソートされた状態で処理されることが前提となります。

アルゴリズムまたは振る舞い

単語単位の類似度計算において、文字列をトリグラムに分解する際、各トリグラムにその出現位置を付与して管理します。これにより、単純な集合比較ではなく、位置情報を考慮した動的な類似度探索が可能となります。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_op.c 内の calc_word_similarity 等の関数が、メモリ確保（palloc）やソート（qsort）の対象として本構造体を利用しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • trgm: trgm.h で定義されるトリグラムの基本型。
  • growable_trgm_array: 動的に拡張可能なトリグラム配列管理用の構造体。

失敗モードと境界条件

  • palloc によるメモリ確保時に、len が大きすぎる場合は PostgreSQL のメモリ制限により失敗する可能性があります。
  • qsort 実行時に比較関数（comp_ptrgm）が正しく実装されていない場合、ソート順序の不整合が発生します。

保守担当者向けメモ

  • 本構造体は word_similarity ロジックの心臓部です。位置情報が重要なため、配列を操作する際は index の整合性を維持してください。
  • 比較関数 comp_ptrgm を変更する場合は、pos_trgm の全フィールド（trg と index）が正しく評価されるか確認してください。
