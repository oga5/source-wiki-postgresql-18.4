ShutdownWorkersHard

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/parallel.c:396

目的

並列実行中の全ワーカープロセスに対し、強制的な終了処理を実行する。

シグネチャまたは形状

static void ShutdownWorkersHard(ParallelState *pstate)

入力と出力

  • **入力**: ParallelState *pstate（並列実行状態を保持する構造体へのポインタ）
  • **出力**: なし（副作用としてワーカーの終了を待機する）

前提条件と不変条件

  • pstate が有効な ParallelState 構造体を指していること。
  • ワーカーの初期化が一部失敗している場合でも、parallelSlot 内のパイプ記述子が適切に処理されるよう設計されている。

アルゴリズムまたは振る舞い

1. **通信路の切断**: 全ワーカーの書き込み用パイプ（pipeWrite）を closesocket で閉じ、ワーカーに終了を通知する。 2. **終了要求の送信**:

  • **非Windows環境**: 各ワーカープロセスに対して SIGTERM シグナルを送信する。
  • **Windows環境**: signal_info_lock を取得し、各ワーカーのデータベース接続に対して PQcancel を発行してクエリをキャンセルする。

    • 3. **待機**: WaitForTerminatingWorkers(pstate) を呼び出し、全ワーカーのプロセス終了を待機する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/parallel.c:207（宣言）
  • src/bin/pg_dump/parallel.c:356（呼び出し）
  • **呼び出し先**:
  • closesocket
  • kill (非Windows)
  • PQcancel (Windows)
  • EnterCriticalSection / LeaveCriticalSection (Windows)
  • WaitForTerminatingWorkers

関連データ構造

  • ParallelState: 並列処理全体の管理構造体。
  • ParallelSlot: 個々のワーカーのプロセスIDやパイプ、接続情報を保持する構造体。

失敗モードと境界条件

  • ワーカーの初期化が不完全な場合、pipeWrite が 0 である可能性があるが、本関数はエラーを無視して処理を継続する。
  • Windows環境では、AH->connCancel が存在する場合のみキャンセル処理が実行される。

保守担当者向けメモ

  • 本関数は強制終了用であるため、正常終了時のクリーンアップには別の関数を使用すること。
  • Windows環境での実装は signal_info_lock を介した排他制御に依存しているため、ロックの順序に注意が必要。
