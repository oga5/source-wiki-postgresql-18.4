AdjustFractDays

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datetime.c:578

目的

浮動小数点数で表現された日数（frac）を、指定されたスケール（scale）に基づいて整数日数と端数に分割し、pg_itm_in 構造体の日付フィールドを更新する。

シグネチャまたは形状

static bool AdjustFractDays(double frac, int scale, struct pg_itm_in *itm_in)

入力と出力

  • **入力**:
  • frac: 調整対象の浮動小数点数（絶対値が1未満であることを前提とする）。
  • scale: 日数への換算倍率（例: 7, DAYS_PER_MONTH）。
  • itm_in: 更新対象の pg_itm_in 構造体へのポインタ。
  • **出力**:
  • 成功時は true、itm_in->tm_mday の加算時にオーバーフローが発生した場合は false を返す。

前提条件と不変条件

  • 入力 frac の絶対値は 1 未満であると仮定される。
  • itm_in は有効な pg_itm_in 構造体を指している必要がある。

アルゴリズムまたは振る舞い

1. frac が 0 の場合、何もせず true を返す。 2. frac に scale を乗算し、整数部分を extra_days として抽出する。 3. itm_in->tm_mday に extra_days を加算する。この際、pg_add_s32_overflow を使用してオーバーフローチェックを行う。 4. 残りの端数（frac - extra_days）を AdjustFractMicroseconds に渡し、USECS_PER_DAY をスケールとして残りの時間調整を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/datetime.c 内の複数箇所（主に日付計算処理）。
  • src/interfaces/ecpg/pgtypeslib/interval.c（ECPGライブラリ）。
  • **呼び出し先**:
  • AdjustFractMicroseconds（端数処理のため）。
  • pg_add_s32_overflow（オーバーフローチェックのため）。

関連データ構造

  • struct pg_itm_in: 日付・時刻の内部表現を保持する構造体。

失敗モードと境界条件

  • itm_in->tm_mday への加算時に 32 ビット整数のオーバーフローが発生した場合、false を返す。

保守担当者向けメモ

  • 本関数は frac の絶対値が 1 未満であることを前提に設計されているため、呼び出し側でこの制約が守られていることを確認すること。
  • itm_in->tm_mday が既に非ゼロである場合、加算によってオーバーフローする可能性がある点に注意が必要。
