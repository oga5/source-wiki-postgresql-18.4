copy_data_dest_cb

定義

  • 種別: typedef
  • 定義箇所: src/include/commands/copy.h:97

目的

COPY TO 操作において、出力データを受け取り処理するためのコールバック関数型を定義します。

シグネチャまたは形状

typedef void (*copy_data_dest_cb) (void *data, int len);

入力と出力

  • data: 書き込み対象となるデータバッファへのポインタ。
  • len: 書き込むデータのバイト長。
  • 戻り値: なし（void）。

前提条件と不変条件

  • 本コールバックは、COPY TO 実行中にデータが生成されるたびに呼び出されます。
  • len は data が指す有効なバイト数と一致している必要があります。

アルゴリズムまたは振る舞い

  • 呼び出し元から提供されたデータバッファを、特定の宛先（ファイル、パイプ、または内部バッファなど）へ転送する役割を担います。
  • 具体的な書き込み先の実装は、この関数ポインタを保持するコンテキストに依存します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/copyto.c 内のデータ出力処理フローから参照されています。
  • 呼び出し先: 本シンボル自体は関数型であり、具体的な実装関数が実行時に割り当てられます。

関連データ構造

  • CopyToState: COPY TO の実行状態を管理する構造体内で、データ転送用コールバックとして保持されます。

失敗モードと境界条件

  • data が NULL である場合や、len が負の値である場合の挙動は、実装側の定義に依存します。
  • 外部リソース（ファイルやパイプ）への書き込みに失敗した場合、呼び出し元でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本型定義は COPY コマンドの出力処理を抽象化するものです。新しい出力先を追加する場合は、このシグネチャに適合する関数を実装し、CopyToState に設定してください。
