HandleSlotSyncMessageInterrupt

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/slotsync.c:1235

目的

スロット同期プロセスに対して、外部からの割り込み要求（シャットダウン要求）を通知し、安全な停止を準備するために使用されます。

シグネチャまたは形状

void HandleSlotSyncMessageInterrupt(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバルなフラグ変数を更新する副作用を持つ）

前提条件と不変条件

  • 本関数はシグナルハンドラから呼び出されることを想定しています。
  • InterruptPending および SlotSyncShutdownPending フラグが適切に設定されることで、メインループが停止要求を検知できる状態になります。

アルゴリズムまたは振る舞い

1. InterruptPending を true に設定し、割り込みが発生したことをシステムに通知します。 2. SlotSyncShutdownPending を true に設定し、スロット同期ワーカーに対してシャットダウンを要求します。 3. プロセスシグナルハンドラ（procsignal_sigusr1_handler）によってラッチが設定されることを前提としています。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/ipc/procsignal.c:699（シグナル処理の文脈）
  • 呼び出し先:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • InterruptPending（グローバル変数）
  • SlotSyncShutdownPending（グローバル変数）

失敗モードと境界条件

  • 本関数自体は単純なフラグ設定であるため、失敗モードはありません。
  • 割り込みが複数回発生した場合でも、フラグは冪等に true に設定されます。

保守担当者向けメモ

  • 本関数は slotsync.h で宣言されており、外部モジュール（特に procsignal.c）からシグナルハンドラ経由で呼び出されます。
  • シャットダウン処理の詳細は、このフラグを監視している ReplSlotSyncWorkerMain のループ内ロジックに依存します。
