dshash_parameters

定義

  • 種別: struct
  • 定義箇所: src/include/lib/dshash.h:54

目的

動的共有メモリ（DSM）上に構築される並行ハッシュテーブル（dshash）の挙動を定義するための設定構造体です。

シグネチャまたは形状

typedef struct dshash_parameters
{
    size_t      key_size;       /* キーのサイズ（エントリの先頭バイト数） */
    size_t      entry_size;     /* エントリの合計サイズ */
    dshash_compare_function compare_function;   /* 比較関数 */
    dshash_hash_function hash_function;         /* ハッシュ関数 */
    dshash_copy_function copy_function;         /* コピー関数 */
    int         tranche_id;     /* ロックに使用するトランシェID */
} dshash_parameters;

入力と出力

  • **入力**: ハッシュテーブルの作成またはアタッチ時に、呼び出し元からこの構造体のインスタンスが渡されます。
  • **出力**: なし（本構造体は設定値の保持のみを行います）。

前提条件と不変条件

  • dshash_parameters 内の関数ポインタ（compare_function, hash_function, copy_function）はプロセス間で共有できません。そのため、各プロセスが dshash_attach を呼び出す際に、必ず再指定する必要があります。

アルゴリズムまたは振る舞い

  • ハッシュテーブルの初期化およびアタッチ時の構成パラメータとして利用されます。
  • key_size と entry_size は、ハッシュテーブル内の各エントリのメモリレイアウトを決定します。
  • tranche_id は、並行アクセス制御のための LWLock 割り当てに使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • dshash_create（src/backend/lib/dshash.c）
  • dshash_attach（src/backend/lib/dshash.c）
  • 各種バックエンド実装（logical/launcher.c, dsm_registry.c, pgstat_shmem.c, typcache.c 等）にて静的に定義されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • dshash_table: 本パラメータを用いて生成されるハッシュテーブル本体。
  • dshash_compare_function, dshash_hash_function, dshash_copy_function: 各種操作関数。

失敗モードと境界条件

  • 不適切な key_size や entry_size を指定した場合の動作は未定義です。
  • 関数ポインタの再設定を怠った場合、アタッチ先のプロセスで不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるものではなく、各プロセスがローカルに保持する設定値です。
  • 構造体メンバの追加や変更を行う際は、dshash_create および dshash_attach を呼び出す全箇所での整合性に注意してください。
