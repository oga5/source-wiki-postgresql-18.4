xl_hash_delete

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash_xlog.h:185

目的

ハッシュインデックスにおけるタプル削除操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_hash_delete
{
    bool        clear_dead_marking;
    bool        is_primary_bucket_page;
} xl_hash_delete;

入力と出力

  • **入力**: 削除操作時に clear_dead_marking（LH_PAGE_HAS_DEAD_TUPLES フラグの解除有無）および is_primary_bucket_page（プライマリバケットページか否か）のフラグを受け取ります。
  • **出力**: WALレコードのペイロードとして書き込まれ、リカバリ時に再適用されます。

前提条件と不変条件

  • SizeOfHashDelete マクロにより、構造体のサイズが定義されています。
  • リカバリ時には、XLogRecGetData を通じてこの構造体にキャストして読み取られます。

アルゴリズムまたは振る舞い

1. **書き込み**: ハッシュインデックスの削除処理において、操作の属性を本構造体に格納し、WALレコードの一部として永続化します。 2. **リカバリ**: hash_redo 関数がWALレコードを読み込む際、この構造体を参照して対象ページの状態（デッドタプルフラグの有無やバケット種別）を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hash.c: 削除処理実行時に本構造体のインスタンスが生成されます。
  • **呼び出し先**:
  • src/backend/access/hash/hash_xlog.c: リカバリ処理（hash_redo）において、WALレコードからデータを抽出するために参照されます。
  • src/backend/access/rmgrdesc/hashdesc.c: ログ出力（hash_desc）において、レコード内容のデバッグ表示に使用されます。

関連データ構造

  • SizeOfHashDelete: 本構造体のサイズを計算するためのマクロ。

失敗モードと境界条件

  • 構造体のフィールドサイズや配置が hash_xlog.c でのキャストと一致しない場合、リカバリ時にデータ破損や誤ったページ操作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はハッシュインデックスの物理的な削除操作を再現するために不可欠です。構造体に変更を加える場合は、hash_redo および hash_desc の実装と整合性を確認してください。
