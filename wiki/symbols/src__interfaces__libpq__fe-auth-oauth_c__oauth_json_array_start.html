oauth_json_array_start

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-oauth.c:257

目的

JSON 解析プロセスにおいて、配列の開始（[）を検出し、解析状態（json_ctx）のネストレベルを更新および検証する。

シグネチャまたは形状

JsonParseErrorType oauth_json_array_start(void *state);

入力と出力

  • **入力**: void *state（実際には struct json_ctx * 型としてキャストして使用）。
  • **出力**: 解析が成功した場合は JSON_SUCCESS、エラーが発生した場合は JSON_SEM_ACTION_FAILED を返す。

前提条件と不変条件

  • state は有効な json_ctx 構造体を指している必要がある。
  • 配列の開始は、ルートレベル（ctx->nested == 0）では許可されない。
  • フィールドのターゲットが設定されている場合、そのフィールドは文字列型である必要があるため、配列の開始はエラーとなる。

アルゴリズムまたは振る舞い

1. ctx->nested が 0 の場合、トップレベル要素がオブジェクトではないとしてエラーを記録する。 2. ctx->target_field が設定されている場合、フィールドが文字列ではないとしてエラーを記録する。 3. ctx->nested をインクリメントする。 4. インクリメント後の ctx->nested が MAX_SASL_NESTING_LEVEL を超えた場合、ネストが深すぎるとしてエラーを記録する。 5. エラーが記録されている場合は JSON_SEM_ACTION_FAILED を返し、それ以外の場合は JSON_SUCCESS を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-auth-oauth.c:552（sem.array_start への代入）
  • src/interfaces/libpq-oauth/oauth-curl.c:865（sem.array_start への代入）
  • **呼び出し先**:
  • oauth_json_set_error（エラー発生時）
  • oauth_json_has_error（戻り値判定時）

関連データ構造

  • json_ctx: 解析状態を保持する構造体。nested（現在のネスト深さ）および target_field（現在解析中のフィールド名）を含む。

失敗モードと境界条件

  • **トップレベルの配列**: ctx->nested が 0 の状態で呼び出されると失敗する。
  • **フィールド内での配列**: ctx->target_field が存在する場合、配列の開始は許可されず失敗する。
  • **ネスト制限超過**: MAX_SASL_NESTING_LEVEL を超える深いネストは失敗する。

保守担当者向けメモ

  • 本関数は json_ctx の状態遷移を管理するコールバック関数として設計されている。
  • MAX_SASL_NESTING_LEVEL の定義値については common/jsonapi.h を参照すること。
