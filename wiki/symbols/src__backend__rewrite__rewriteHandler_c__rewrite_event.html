rewrite_event

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteHandler.c:50

目的

クエリ書き換え処理中に、現在処理対象となっているリレーションと、適用中のルールイベントの種類を追跡するために使用されます。

シグネチャまたは形状

typedef struct rewrite_event
{
	Oid			relation;		/* ルールを持つリレーションのOID */
	CmdType		event;			/* 実行中のルールイベントの種類 */
} rewrite_event;

入力と出力

本構造体はデータ保持用であり、関数ではありません。メモリ割り当て時には palloc を介して生成されます。

前提条件と不変条件

  • relation は有効なリレーションの OID である必要があります。
  • event は CmdType 列挙型（SELECT, INSERT, UPDATE, DELETE 等）のいずれかである必要があります。

アルゴリズムまたは振る舞い

クエリ書き換えの再帰的な処理において、無限ループを防止するための追跡用データとして利用されます。処理中のリレーションをリストに格納し、スタックのように管理することで、同一リレーションに対するルールの再帰的適用を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteHandler.c 内のクエリ書き換えロジック（RewriteQuery 等の周辺処理）から、リストへの追加や参照が行われます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な呼び出し先はありません。

関連データ構造

  • CmdType: ルールのイベント種別を定義する列挙型。
  • List: rewrite_event を保持するために使用される PostgreSQL のリスト構造。

失敗モードと境界条件

  • メモリ不足時には palloc によりエラーが発生する可能性があります。
  • 循環参照を検知するためのリスト管理が不適切な場合、無限ループに陥るリスクがあります。

保守担当者向けメモ

  • 本構造体は、主にビューの展開やルール適用時の再帰制御に使用されます。
  • activeRIRs 等のリスト管理と併せて参照されることが多いため、変更時は書き換えエンジンの再帰停止条件に影響がないか確認してください。
