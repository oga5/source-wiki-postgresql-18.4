exec_init_tuple_store

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_exec.c:3668

目的

PL/pgSQL 関数が集合（Set）を返す際に使用する tuplestore を初期化し、実行状態（PLpgSQL_execstate）に設定する。

シグネチャまたは形状

static void exec_init_tuple_store(PLpgSQL_execstate *estate)

入力と出力

  • **入力**: PLpgSQL_execstate *estate（実行状態を保持する構造体）。
  • **出力**: なし（副作用として estate 内の tuple_store および tuple_store_desc を更新）。

前提条件と不変条件

  • estate->rsi（ReturnSetInfo）が有効であり、かつ SFRM_Materialize モードをサポートしていること。
  • rsi->expectedDesc が NULL でないこと。

アルゴリズムまたは振る舞い

1. **検証**: 呼び出し元が集合を返せるコンテキストか、および Materialize モードが許可されているかをチェックし、不適格な場合はエラーを報告する。 2. **コンテキスト切り替え**: メモリコンテキストを estate->tuple_store_cxt に、リソースオーナーを estate->tuple_store_owner に一時的に切り替える。これにより、サブトランザクション等の影響を受けずにリソースを確保する。 3. **初期化**: tuplestore_begin_heap を呼び出し、tuplestore を作成する。 4. **復元**: 元のメモリコンテキストとリソースオーナーを復元する。 5. **メタデータ設定**: estate->tuple_store_desc に rsi->expectedDesc を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_exec.c 内の関数（行 3340, 3560 付近）。
  • **呼び出し先**: tuplestore_begin_heap（PostgreSQL 内部 API）。

関連データ構造

  • PLpgSQL_execstate: 実行状態を管理するメイン構造体。
  • ReturnSetInfo: 集合を返す関数の情報を保持する構造体。
  • tuplestore: 集合結果を一時的に保持するためのストレージ。

失敗モードと境界条件

  • rsi が NULL であるか、ReturnSetInfo 型でない場合にエラー。
  • SFRM_Materialize モードが許可されていない、または expectedDesc が NULL の場合にエラー。

保守担当者向けメモ

  • 本関数は tuplestore の作成場所を estate が保持する特定のメモリコンテキストとリソースオーナーに固定することで、例外処理ブロック（サブトランザクション）を跨いでもリソースが正しく維持されるように設計されている。
