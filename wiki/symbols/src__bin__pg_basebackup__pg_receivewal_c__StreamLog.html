StreamLog

定義

  • **種別**: function
  • **定義箇所**: src/bin/pg_basebackup/pg_receivewal.c:499

目的

PostgreSQL サーバーから WAL（Write-Ahead Log）データをストリーミング受信し、ローカルのディレクトリへ保存するメインループを実行する。

シグネチャまたは形状

static void StreamLog(void)

入力と出力

  • **入力**: グローバル変数（conn, replication_slot, basedir, compression_algorithm, compresslevel, verbose, synchronous, do_sync, WalSegSz, standby_message_timeout）を介して設定を取得。
  • **出力**: なし（戻り値は void）。処理結果はローカルファイルシステムへの書き込みおよび標準出力へのログ出力として反映される。

前提条件と不変条件

  • conn が確立されているか、GetConnection() により確立可能であること。
  • サーバーのバージョンがストリーミングに対応していること。
  • stream.startpos および stream.timeline が InvalidXLogRecPtr ではない状態でストリーミングが開始されること。

アルゴリズムまたは振る舞い

1. **接続確立**: GetConnection() でサーバーとのレプリケーション接続を確保し、CheckServerVersionForStreaming() でバージョン互換性を確認する。 2. **システム識別**: RunIdentifySystem() でサーバーのシステム識別子、現在の LSN、タイムライン ID を取得する。 3. **開始位置決定**:

  • FindStreamingStart() でローカルディレクトリを走査し、再開位置を特定する。
  • 失敗した場合は、レプリケーションスロット情報（PostgreSQL 15以降）またはサーバーの現在のフラッシュ位置をフォールバックとして使用する。

    • 4. **ストリーミング準備**: 開始位置を WAL セグメントの境界に調整し、CreateWalDirectoryMethod() で書き込みメソッドを初期化する。 5. **データ受信**: ReceiveXlogStream() を呼び出し、サーバーから WAL を受信して保存する。 6. **終了処理**: 書き込みメソッドの終了処理を行い、接続を閉じてリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_receivewal.c 内の main 関数から呼び出される。
  • **呼び出し先**:
  • GetConnection, CheckServerVersionForStreaming, RunIdentifySystem, FindStreamingStart, GetSlotInformation, XLogSegmentOffset, pg_log_info, CreateWalDirectoryMethod, ReceiveXlogStream, PQfinish, PQserverVersion

関連データ構造

  • StreamCtl: ストリーミングの制御パラメータを保持する構造体。
  • WalMethod: ファイルシステムへの書き込み抽象化レイヤー。

失敗モードと境界条件

  • **接続失敗**: GetConnection() が失敗した場合、関数は即座に終了する。
  • **バージョン不一致**: CheckServerVersionForStreaming() が失敗した場合、exit(1) でプロセスを終了する。
  • **識別失敗**: RunIdentifySystem() が失敗した場合、exit(1) でプロセスを終了する。
  • **書き込み失敗**: stream.walmethod->ops->finish() が失敗した場合、エラーログを出力して終了する。

保守担当者向けメモ

  • ストリーミング開始位置の決定ロジック（FindStreamingStart 後のフォールバック）は、PostgreSQL のバージョンアップに伴い変更される可能性があるため注意すること。
  • stream.walmethod の操作は walmethods.c に依存しているため、圧縮アルゴリズムの追加や変更時には同ファイルの確認が必要。
