ResourceOwnerEnlarge

定義

  • 種別: function
  • 定義箇所: src/backend/utils/resowner/resowner.c:451

目的

リソースオーナーが保持するリソース追跡用ストレージの容量を確保します。ResourceOwnerRemember() を呼び出してリソースを登録する前に、十分な空き容量があることを保証するために使用されます。

シグネチャまたは形状

void ResourceOwnerEnlarge(ResourceOwner owner)

入力と出力

  • **入力**: owner (リソースオーナー構造体へのポインタ)
  • **出力**: なし（副作用として owner 内部のストレージ状態が更新されます）

前提条件と不変条件

  • リソースの解放処理（releasing フラグ）が開始されていないこと。開始後に呼び出した場合はエラーとなります。
  • 呼び出し後、owner->nhash <= owner->grow_at が成立すること。

アルゴリズムまたは振る舞い

1. **初期チェック**: owner->releasing が真であればエラーを投げる。 2. **容量確認**: 現在の配列使用数（narr）が固定長配列の制限（RESOWNER_ARRAY_SIZE）未満であれば、何もせず終了する。 3. **ハッシュテーブル拡張**:

  • 現在のハッシュテーブルが満杯（narr + nhash >= grow_at）の場合、容量を2倍に拡張する。
  • 新しいハッシュテーブルを TopMemoryContext に割り当て、既存のハッシュエントリを新しいテーブルへ再配置（ResourceOwnerAddToHash）する。
  • 古いハッシュテーブルを解放する。

    • 4. **配列からハッシュへの移行**: 固定長配列（arr）に保持されていた全てのリソースをハッシュテーブルへ移動し、配列のカウンタ（narr）を 0 にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/openssl.c
  • src/backend/access/common/tupdesc.c
  • src/backend/jit/llvm/llvmjit.c
  • src/backend/storage/buffer/bufmgr.c
  • **呼び出し先**:
  • ResourceOwnerAddToHash (ハッシュテーブルへのエントリ追加)
  • MemoryContextAllocZero (メモリ確保)
  • pfree (旧ハッシュテーブルの解放)
  • elog (エラー報告)

関連データ構造

  • ResourceOwnerData: リソースオーナーの内部状態を保持する構造体。
  • ResourceElem: リソースを格納するハッシュテーブルの要素。

失敗モードと境界条件

  • **解放中の呼び出し**: owner->releasing が設定されている場合、elog(ERROR) により処理が中断されます。
  • **メモリ不足**: MemoryContextAllocZero が失敗した場合、PostgreSQL の標準的なメモリ管理ルールに従い、呼び出し元へ制御が戻らない可能性があります。

保守担当者向けメモ

  • 本関数は ResourceOwnerRemember の前提条件となるため、リソース登録を行うコードパスでは必ず先行して呼び出す必要があります。
  • ハッシュテーブルの容量は常に2のべき乗を維持する設計となっています。
