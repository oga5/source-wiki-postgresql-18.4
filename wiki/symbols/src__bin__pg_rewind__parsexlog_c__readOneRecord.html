readOneRecord

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/parsexlog.c:123

目的

指定された WAL 位置から単一の WAL レコードを読み込み、そのレコードの終了位置（次のレコードの開始位置）を返します。

シグネチャまたは形状

XLogRecPtr readOneRecord(const char *datadir, XLogRecPtr ptr, int tliIndex, const char *restoreCommand)

入力と出力

  • **入力**:
  • datadir: データディレクトリのパス。
  • ptr: 読み込みを開始する WAL の LSN（Log Sequence Number）。
  • tliIndex: タイムラインインデックス。
  • restoreCommand: WAL セグメントを復元するためのコマンド文字列。
  • **出力**:
  • 読み込んだレコードの直後の LSN（XLogRecPtr 型）。

前提条件と不変条件

  • WalSegSz がグローバルに定義されている必要があります。
  • SimpleXLogPageRead がコールバックとして利用可能である必要があります。
  • 読み込み対象の WAL レコードが指定した ptr に存在する必要があります。

アルゴリズムまたは振る舞い

1. XLogReaderAllocate を使用して XLogReaderState を初期化し、SimpleXLogPageRead をページ読み込みコールバックとして設定します。 2. XLogBeginRead で読み込み位置を ptr にセットします。 3. XLogReadRecord を呼び出し、レコードを取得します。 4. 取得したレコードの終了位置（xlogreader->EndRecPtr）を保持します。 5. XLogReaderFree でリーダーを解放し、開いているファイルディスクリプタ xlogreadfd があればクローズします。 6. 保持した終了位置を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/pg_rewind.c:417
  • **呼び出し先**:
  • XLogReaderAllocate
  • XLogBeginRead
  • XLogReadRecord
  • SimpleXLogPageRead (コールバックとして)
  • XLogReaderFree

関連データ構造

  • XLogReaderState: WAL 解析のメイン状態構造体。
  • XLogPageReadPrivate: SimpleXLogPageRead に渡されるプライベートデータ構造。
  • XLogRecPtr: WAL 上の論理位置を示す型。

失敗モードと境界条件

  • XLogReaderAllocate がメモリ不足で失敗した場合、pg_fatal を呼び出しプロセスを終了します。
  • XLogReadRecord がレコードの読み込みに失敗した場合（record == NULL）、エラーメッセージの有無に応じて pg_fatal を呼び出しプロセスを終了します。
  • xlogreadfd が開かれている場合、関数終了時に必ずクローズされます。

保守担当者向けメモ

  • 本関数は単発のレコード読み込み用であり、ループ処理は含みません。
  • xlogreadfd はグローバル変数として管理されているため、並列実行には適していません。
