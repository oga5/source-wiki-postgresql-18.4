AlterSubscription_refresh

定義

  • 種別: function
  • 定義箇所: src/backend/commands/subscriptioncmds.c:819

目的

論理レプリケーションにおける REFRESH PUBLICATION コマンドを実行し、ローカルのサブスクリプションとリモートのパブリケーション間でのテーブル同期状態を更新します。

シグネチャまたは形状

static void AlterSubscription_refresh(Subscription *sub, bool copy_data, List *validate_publications)

入力と出力

  • **入力**:
  • sub: 対象となるサブスクリプション構造体。
  • copy_data: 新規テーブル追加時にデータをコピーするかどうかを指定するフラグ。
  • validate_publications: 検証対象のパブリケーションリスト（NULL可）。
  • **出力**: なし（成功時はカタログ更新とワーカー制御を実行、失敗時はエラーをスロー）。

前提条件と不変条件

  • libpqwalreceiver ライブラリがロード可能であること。
  • パブリッシャーへの接続が確立できること。
  • pg_subscription_rel に対して AccessExclusiveLock を取得し、同期処理中の競合を防止する。

アルゴリズムまたは振る舞い

1. **接続**: walrcv_connect を使用してパブリッシャーに接続。 2. **検証**: validate_publications が指定されていればパブリケーションの存在を確認。 3. **リスト取得**: リモートのパブリケーションからテーブルリストを取得し、ローカルの pg_subscription_rel と比較。 4. **差分処理**:

  • リモートに存在しローカルにないテーブル: AddSubscriptionRelState で追加。
  • ローカルに存在しリモートにないテーブル: RemoveSubscriptionRel で削除し、関連するレプリケーションワーカーを停止。

    • 5. **後処理**: READY 状態でないテーブルについては、必要に応じてレプリケーションオリジンを削除。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/subscriptioncmds.c 内の AlterSubscription 関数から複数回参照。また src/backend/replication/logical/tablesync.c にも言及あり。
  • **呼び出し先**: walrcv_connect, fetch_table_list, GetSubscriptionRelations, AddSubscriptionRelState, RemoveSubscriptionRel, logicalrep_worker_stop, replorigin_drop_by_name 等。

関連データ構造

  • Subscription: サブスクリプション情報を保持。
  • SubscriptionRelState: ローカルのテーブル同期状態。
  • SubRemoveRels: 削除対象テーブルの情報を一時保持する内部構造体。

失敗モードと境界条件

  • パブリッシャーへの接続失敗時は ERRCODE_CONNECTION_FAILURE でエラー終了。
  • サポートされていないリレーション種別（CheckSubscriptionRelkind）が含まれる場合はエラー。
  • トランザクション内で実行される場合、カタログロックにより他のワーカーの再起動と競合しないよう制御される。

保守担当者向けメモ

  • PG_TRY/PG_FINALLY ブロックにより、接続エラー発生時でもリソースが適切に解放されるよう設計されています。
  • pg_subscription_rel のロックは、同期処理中の整合性を保つために不可欠です。変更を加える際は、ワーカーのライフサイクルへの影響を考慮してください。
