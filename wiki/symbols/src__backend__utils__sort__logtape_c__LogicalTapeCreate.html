LogicalTapeCreate

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/logtape.c:679

目的

既存の LogicalTapeSet 内に新しい論理テープを作成し、その管理構造体へのポインタを返します。

シグネチャまたは形状

LogicalTape *LogicalTapeCreate(LogicalTapeSet *lts)

入力と出力

  • **入力**: lts - テープを追加する対象の LogicalTapeSet 構造体へのポインタ。
  • **出力**: 新規作成された LogicalTape 構造体へのポインタ。

前提条件と不変条件

  • lts は有効な LogicalTapeSet を指している必要があります。
  • リーダープロセス（lts->worker == -1）において lts->fileset が存在する場合、新しいテープの作成は禁止されており、エラーが発生します。

アルゴリズムまたは振る舞い

1. 現在のプロセスがリーダープロセスであり、かつファイルセットが関連付けられているかを確認します。条件に合致する場合、書き込み不可の制約により ERROR を送出します。 2. 内部関数 ltsCreateTape(lts) を呼び出し、テープセット内に新しい論理テープを割り当てて初期化します。 3. 初期化された LogicalTape 構造体を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c:3007
  • src/backend/utils/sort/tuplesort.c:1961
  • **呼び出し先**:
  • ltsCreateTape (内部関数)
  • elog (エラー発生時)

関連データ構造

  • LogicalTapeSet: テープセット全体を管理する構造体。
  • LogicalTape: 個別の論理テープの状態を保持する構造体。

失敗モードと境界条件

  • **エラー**: リーダープロセスで lts->fileset が設定されている状態で本関数を呼び出すと、"cannot create new tapes in leader process" というメッセージで elog(ERROR) が発生します。

保守担当者向けメモ

  • 本関数は、リーダープロセスでのテープ作成を制限しています。これは BufFileOpenShared() で開かれたファイルが読み取り専用であることに起因する制約ですが、将来的に書き込みを許可する設計変更がなされる可能性がコメントに示唆されています。
