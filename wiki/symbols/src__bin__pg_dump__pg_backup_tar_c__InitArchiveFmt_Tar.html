InitArchiveFmt_Tar

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_tar.c:120

目的

pg_dump のアーカイブ形式として TAR 形式を使用するためのハンドラ関数ポインタを初期化し、TAR 固有のコンテキストをセットアップする。

シグネチャまたは形状

void InitArchiveFmt_Tar(ArchiveHandle *AH)

入力と出力

  • **入力**: ArchiveHandle *AH（アーカイブ処理の基盤となるハンドル構造体）
  • **出力**: なし（副作用として AH の関数ポインタ群および formatData フィールドを更新する）

前提条件と不変条件

  • AH は有効な ArchiveHandle 構造体へのポインタであること。
  • TAR 形式では圧縮（compression_spec.algorithm != PG_COMPRESSION_NONE）はサポートされない。

アルゴリズムまたは振る舞い

1. **関数ポインタの割り当て**: AH 内の各種操作ポインタ（StartDataPtr, WriteDataPtr, ClosePtr 等）を TAR 形式用の実装関数（_StartData, _WriteData, _CloseArchive 等）で初期化する。 2. **コンテキストの初期化**: lclContext 構造体をメモリ確保し、AH->formatData に格納する。 3. **モード別処理**:

  • **書き込みモード (archModeWrite)**: 指定されたファイルまたは標準出力 (stdout) をバイナリ書き込みモードで開き、ファイル位置のシーク可否を確認する。
  • **読み込みモード**: 指定されたファイルまたは標準入力 (stdin) をバイナリ読み込みモードで開き、toc.dat を読み込んでアーカイブのヘッダーと目次（TOC）をロードする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_archiver.c:2482
  • **呼び出し先**: pg_malloc0_object, fopen, checkSeek, pg_fatal, tarOpen, ReadHead, ReadToc, tarClose

関連データ構造

  • ArchiveHandle: アーカイブ処理の抽象化レイヤー。
  • lclContext: TAR 形式特有のコンテキスト（ファイルハンドル tarFH、位置情報 tarFHpos、シーク可否フラグ hasSeek 等を保持）。

失敗モードと境界条件

  • ファイルのオープンに失敗した場合、pg_fatal を呼び出しプロセスを終了する。
  • 圧縮アルゴリズムが指定されている場合、pg_fatal を呼び出しエラー終了する。
  • 標準入出力が利用できない場合、エラーを報告する。

保守担当者向けメモ

  • 本関数は pg_backup_archiver.c から呼び出されるエントリポイントである。
  • setvbuf によるバッファリング制御は現在コメントアウトされているため、必要に応じて挙動を確認すること。
  • TAR 形式は圧縮をサポートしていないため、将来的な拡張時には gzdopen 等のバッファリングがファイル位置制御に与える影響を考慮する必要がある。
