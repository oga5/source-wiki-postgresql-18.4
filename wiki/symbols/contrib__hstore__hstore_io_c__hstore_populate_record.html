hstore_populate_record

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_io.c:993

目的

既存のレコード（行型）のフィールド値を、hstore 型のキーと値のペアで更新または生成する。

シグネチャまたは形状

Datum hstore_populate_record(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: record 型（更新対象の行データ、または NULL）。
  • 第2引数: hstore 型（更新に使用するキーと値のペア）。
  • **出力**: 更新された record 型の Datum を返す。

前提条件と不変条件

  • 第1引数は必ず行型（rowtype）である必要がある。
  • hstore が空の場合、第1引数が非 NULL であればそのまま元のレコードを返す。
  • RecordIOData を用いて、呼び出し間で I/O 情報をキャッシュする。

アルゴリズムまたは振る舞い

1. 引数の型情報を取得し、レコードの TupleDesc を特定する。 2. fcinfo->flinfo->fn_extra を使用して、列ごとの I/O 情報をキャッシュする。 3. レコードが提供されている場合は heap_deform_tuple で分解し、提供されていない場合は全フィールドを NULL として初期化する。 4. レコードの各属性に対し、hstore 内に一致するキーがあるか hstoreFindKey で検索する。 5. キーが見つかった場合、またはドメイン制約チェックのために必要な場合、InputFunctionCall を用いて値を変換し、レコードのフィールドを更新する。 6. 最終的に構築された値を基に新しいタプルを生成する（ソースコード末尾で InputFunctionCall を呼び出し中）。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • lookup_rowtype_tupdesc_domain
  • heap_deform_tuple
  • hstoreFindKey
  • getTypeInputInfo
  • InputFunctionCall
  • MemoryContextAlloc

関連データ構造

  • HStore: 入力データ構造。
  • RecordIOData: 列ごとの I/O 情報を保持するキャッシュ構造体。
  • TupleDesc: レコードの構造定義。
  • HeapTupleHeader: 入力レコードのヘッダー。

失敗モードと境界条件

  • 第1引数がレコード型でない場合、ERRCODE_DATATYPE_MISMATCH でエラーを報告する。
  • 両方の引数が NULL の場合、NULL を返す。
  • 削除された列（attisdropped）は処理対象外としてスキップされる。

保守担当者向けメモ

  • fn_extra を使用したキャッシュの更新ロジック（record_type や record_typmod の変更検知）は、レコードの構造が動的に変わるケースを考慮している。
  • ドメイン型の制約チェックを強制するため、キーが存在しない場合や値が NULL の場合でも InputFunctionCall が呼び出される設計になっている点に注意すること。
