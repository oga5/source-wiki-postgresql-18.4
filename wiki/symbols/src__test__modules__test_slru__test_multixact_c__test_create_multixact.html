test_create_multixact

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_multixact.c:27

目的

現在のトランザクションIDを使用して新しいMultiXact IDを生成し、テスト目的でその値をSQL層へ返却する。

シグネチャまたは形状

Datum test_create_multixact(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQLの関数呼び出し規約に基づく引数）
  • **出力**: Datum 型として MultiXactId を返却する。

前提条件と不変条件

  • 本関数はPostgreSQLのトランザクション内で呼び出される必要がある。
  • GetCurrentTransactionId() が有効なトランザクションIDを返す状態であること。

アルゴリズムまたは振る舞い

1. MultiXactIdSetOldestMember() を呼び出し、古いメンバーの追跡情報を更新する。 2. MultiXactIdCreate を呼び出し、現在のトランザクションIDを MultiXactStatusUpdate および MultiXactStatusForShare のステータスで登録する。 3. 生成された MultiXactId を PG_RETURN_TRANSACTIONID マクロを介して呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由での呼び出し（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**:
  • MultiXactIdSetOldestMember()
  • MultiXactIdCreate()
  • GetCurrentTransactionId()
  • PG_RETURN_TRANSACTIONID()

関連データ構造

  • MultiXactId: MultiXactを識別するための型。
  • MultiXactStatus: MultiXactのメンバー状態を定義する列挙型。

失敗モードと境界条件

  • トランザクションが開始されていない場合、GetCurrentTransactionId() の呼び出しによりエラーが発生する可能性がある。
  • MultiXactの生成上限に達している場合、MultiXactIdCreate 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、実運用環境での使用は想定されていない。
  • MultiXactIdCreate の引数として固定のステータス（UpdateおよびForShare）を使用している点に注意すること。
