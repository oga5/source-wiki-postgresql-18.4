config_generic

定義

  • 種別: struct
  • 定義箇所: src/include/utils/guc_tables.h:171

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおける、すべての設定変数の基底となる共通構造体です。設定変数のメタデータ、現在の値のソース、および実行時の状態管理を統合的に保持します。

シグネチャまたは形状

struct config_generic {
    const char *name;
    GucContext  context;
    enum config_group group;
    const char *short_desc;
    const char *long_desc;
    int         flags;
    enum config_type vartype;
    int         status;
    GucSource   source;
    GucSource   reset_source;
    GucContext  scontext;
    GucContext  reset_scontext;
    Oid         srole;
    Oid         reset_srole;
    GucStack   *stack;
    void       *extra;
    dlist_node  nondef_link;
    slist_node  stack_link;
    slist_node  report_link;
    char       *last_reported;
    char       *sourcefile;
    int         sourceline;
};

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。ただし、GUCシステム内の各関数（InitializeOneGUCOption 等）がこの構造体を引数として受け取り、設定値の適用や検証を行います。

前提条件と不変条件

  • name フィールドは構造体の先頭に配置される必要があります。
  • vartype は起動時にのみ設定されます。
  • extra ポインタは、check_hook で生成された値を assign_hook へ渡すためのブリッジとして機能します。

アルゴリズムまたは振る舞い

  • **設定管理**: source や scontext を通じて、現在の値がどこから（設定ファイル、SQLコマンド等）設定されたかを追跡します。
  • **スタック管理**: SET LOCAL 等による一時的な変更が発生した場合、stack フィールドを用いて以前の値を保持し、トランザクション終了時に復元します。
  • **リスト管理**: nondef_link（デフォルト値以外）、stack_link（スタックあり）、report_link（クライアントへの報告が必要）といったリストノードを保持し、GUCシステムが効率的に変数を走査できるようにします。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/transam/xlog.c
  • src/backend/commands/explain.c
  • src/backend/utils/misc/guc.c（InitializeOneGUCOption, RemoveGUCFromLists, set_guc_source, push_old_value, ReportGUCOption, reapply_stacked_values 等）
  • **呼び出し先**:
  • 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • GucStack: 一時的な設定変更を管理するスタック構造。
  • dlist_node, slist_node: lib/ilist.h で定義されたリスト管理用ノード。
  • config_bool, config_int 等: 本構造体を継承（包含）する各型固有の構造体。

失敗モードと境界条件

  • sourcefile が NULL の場合、その設定は設定ファイル由来ではないことを示します。
  • last_reported が NULL の場合、クライアントへ未送信であることを示します。

保守担当者向けメモ

  • 本構造体はGUCシステムの基盤です。フィールドの追加や変更は、guc.c 内のすべての操作関数に影響を与える可能性があるため注意してください。
  • extra ポインタのライフサイクル管理は、各変数の assign_hook 実装に依存します。
