PG_Locale_Strategy

定義

  • 種別: enum
  • 定義箇所: src/backend/regex/regc_pg_locale.c:63

目的

PostgreSQLの正規表現エンジンにおいて、文字プロパティ判定（isalpha 等）を行う際に使用するロケール処理戦略を識別するために定義されています。

シグネチャまたは形状

typedef enum {
    PG_REGEX_STRATEGY_C,
    PG_REGEX_STRATEGY_BUILTIN,
    PG_REGEX_STRATEGY_LIBC_WIDE,
    PG_REGEX_STRATEGY_LIBC_1BYTE,
    PG_REGEX_STRATEGY_ICU,
} PG_Locale_Strategy;

入力と出力

  • 本シンボルは列挙型であり、入力や出力を持つ関数ではありません。
  • 状態変数 pg_regex_strategy の値として保持され、文字判定ロジックの分岐条件として使用されます。

前提条件と不変条件

  • 正規表現コンパイル時に pg_set_regex_collation によって適切な戦略が選択・設定される必要があります。

アルゴリズムまたは振る舞い

正規表現エンジンは、本列挙型の値に基づいて以下のいずれかの判定手法を選択します。

  • PG_REGEX_STRATEGY_C: エンコーディングに依存しないCロケール処理。
  • PG_REGEX_STRATEGY_BUILTIN: PostgreSQL組み込みのUnicodeセマンティクス。
  • PG_REGEX_STRATEGY_LIBC_WIDE: locale_t を用いたワイド文字対応のlibc関数。
  • PG_REGEX_STRATEGY_LIBC_1BYTE: locale_t を用いた1バイト文字対応のlibc関数。
  • PG_REGEX_STRATEGY_ICU: ICUライブラリの uchar.h 関数。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/regex/regc_pg_locale.c 内の静的変数 pg_regex_strategy の型定義および、同ファイル内の関数（pg_wc_... 等）で戦略判定に使用されます。
  • 呼び出し先: なし（列挙型定義のため）。

関連データ構造

  • pg_regex_strategy: 本列挙型を保持する静的変数。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 新しいロケール判定手法を追加する場合は、本列挙型にメンバを追加し、pg_set_regex_collation および各 pg_wc_ 関数で対応する分岐を実装してください。
