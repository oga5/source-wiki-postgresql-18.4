CreateOneShotCachedPlan

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/plancache.c:297

目的

一度きりの実行を想定した軽量な CachedPlanSource を初期化し、生成します。このキャッシュは無効化（Invalidation）の追跡を行わない点に特徴があります。

シグネチャまたは形状

CachedPlanSource *CreateOneShotCachedPlan(RawStmt *raw_parse_tree, const char *query_string, CommandTag commandTag)

入力と出力

  • **入力**:
  • raw_parse_tree: 解析対象の構文木。
  • query_string: クエリ文字列（必須）。
  • commandTag: コマンドの種類を示すタグ。
  • **出力**:
  • 初期化された CachedPlanSource 構造体へのポインタ。

前提条件と不変条件

  • query_string は NULL であってはならない。
  • 呼び出し元のメモリコンテキストで palloc0 を使用して構造体が割り当てられる。

アルゴリズムまたは振る舞い

1. CachedPlanSource 構造体をメモリ上に確保し、ゼロクリアする。 2. magic フィールドに CACHEDPLANSOURCE_MAGIC を設定する。 3. 入力引数（raw_parse_tree, query_string, commandTag）を構造体に格納する。 4. is_oneshot フラグを true に設定し、無効化追跡や永続化に関連するフラグ（is_valid, is_saved 等）を false に初期化する。 5. その他のフィールド（param_types, query_list 等）を空またはデフォルト値に設定する。 6. 初期化済みの構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/spi.c:2362
  • **呼び出し先**:
  • palloc0 (メモリ割り当て)

関連データ構造

  • CachedPlanSource: クエリのソースおよび計画状態を保持する主要構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • query_string が NULL の場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数で作成されたキャッシュは無効化の追跡を行わないため、DDL 等によるシステムカタログの変更に対して自動的な再検証は行われません。
  • is_oneshot が true であるため、通常のキャッシュ管理フローとは異なるライフサイクルを持つことに注意してください。
