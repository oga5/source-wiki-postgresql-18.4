get_matching_partitions

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partprune.c:846

目的

PartitionPruneContext と一連の PartitionPruneStep を用いて、クエリの条件式に基づきスキャン対象となるパーティションのインデックス集合（Bitmapset）を特定します。

シグネチャまたは形状

Bitmapset *get_matching_partitions(PartitionPruneContext *context, List *pruning_steps)

入力と出力

  • **入力**:
  • context: パーティション境界情報や戦略を含む PartitionPruneContext。
  • pruning_steps: 評価対象となる PartitionPruneStep のリスト。
  • **出力**:
  • スキャン対象となるパーティションインデックスを保持する Bitmapset。

前提条件と不変条件

  • pruning_steps が空の場合、すべてのパーティションが対象となります（context->nparts > 0 を前提）。
  • final_result（最後のステップの結果）は NULL であってはなりません。
  • final_result->scan_null が真の場合、PARTITION_STRATEGY_LIST である必要があります。
  • scan_default が真の場合、PARTITION_STRATEGY_LIST または PARTITION_STRATEGY_RANGE である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: ステップが空であれば全パーティションを返します。 2. **ステップ実行**: pruning_steps を順次走査し、ステップの型に応じて以下の関数を呼び出します。

  • T_PartitionPruneStepOp: perform_pruning_base_step を実行。
  • T_PartitionPruneStepCombine: perform_pruning_combine_step を実行。
  • 各ステップの結果は results 配列に格納され、後続のステップで参照されます。

    • 3. **インデックス収集**: 最終ステップの結果に含まれる境界オフセットから、context->boundinfo->indexes を参照して実際のパーティションインデックスを特定します。 4. **特殊パーティションの追加**:

  • インデックスが負（範囲外やハッシュの未定義値）の場合はデフォルトパーティションをマークします。
  • scan_null が真であれば null_index を、scan_default が真であれば default_index を結果の Bitmapset に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execPartition.c (実行時プルーニング)
  • src/backend/partitioning/partprune.c (内部的な再帰呼び出し等)
  • **呼び出し先**:
  • perform_pruning_base_step
  • perform_pruning_combine_step
  • bms_add_range, bms_add_member, bms_next_member
  • partition_bound_has_default, partition_bound_accepts_nulls

関連データ構造

  • PartitionPruneContext: プルーニングに必要な境界情報や戦略を保持。
  • PartitionPruneStep: 演算または結合の単位。
  • PruneStepResult: 各ステップの実行結果（境界オフセット、NULL/デフォルトのフラグ）。
  • Bitmapset: 最終的なパーティションインデックスの集合。

失敗モードと境界条件

  • nodeTag が未知のステップ型である場合、elog(ERROR) を発生させます。
  • パーティションインデックスが負の場合、デフォルトパーティションの存在を確認し、必要に応じてスキャン対象に追加します。

保守担当者向けメモ

  • 本関数はプランニング時と実行時の両方で使用されます。
  • results 配列は num_steps 分のメモリを確保し、ステップの依存関係を解決するために使用されます。
  • Bitmapset の操作が多いため、メモリ管理とビット演算の整合性に注意してください。
