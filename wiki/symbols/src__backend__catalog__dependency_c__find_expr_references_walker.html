find_expr_references_walker

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/dependency.c:1697

目的

式ツリーを走査し、式内で参照されているデータベースオブジェクト（テーブル、列、型、関数、演算子など）を特定して、依存関係リストに登録する。

シグネチャまたは形状

static bool find_expr_references_walker(Node *node, find_expr_references_context *context)

入力と出力

  • **入力**:
  • node: 走査対象の式ツリーノード。
  • context: 依存関係の蓄積先（ObjectAddresses）およびリレーションテーブル（rtables）を含むコンテキスト構造体。
  • **出力**:
  • 戻り値: 走査を継続する場合は false、停止する場合は true。

前提条件と不変条件

  • context が有効なポインタであること。
  • Var ノードを処理する場合、context->rtables が varlevelsup および varno に対して十分な長さを持っていること。
  • node が NULL の場合は即座に false を返す。

アルゴリズムまたは振る舞い

1. **Var ノード**: 参照先の RangeTblEntry を取得し、列番号（varattno）が有効であれば、リレーションまたは関数に対する依存関係を add_object_address で記録する。 2. **Const ノード**: 定数のデータ型および照合順序（デフォルト以外）に対する依存関係を記録する。また、regclass や regproc 等の OID 参照型の場合、システムカタログを検索して該当オブジェクトへの依存関係を追加する。regrole 型はサポート外としてエラーを投げる。 3. **Param ノード**: パラメータのデータ型および照合順序に対する依存関係を記録する。 4. **FuncExpr ノード**: 関数 ID に対する依存関係を記録し、引数の走査を継続する。 5. **再帰**: expression_tree_walker 等を通じて、式ツリーの各ノードに対して再帰的に適用される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c 内の式評価に関連する箇所（find_expr_references 等から間接的に呼び出される）。
  • **呼び出し先**: add_object_address, process_function_rte_ref, SearchSysCacheExists1, expression_tree_walker。

関連データ構造

  • find_expr_references_context: 走査中の状態と依存関係リストを保持。
  • ObjectAddresses: 依存関係のリスト。
  • Var, Const, Param, FuncExpr: PostgreSQLのノード構造体。

失敗モードと境界条件

  • varlevelsup や varno が範囲外の場合、elog(ERROR) で異常終了する。
  • regrole 型の定数が検出された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーを報告する。
  • 循環参照やスタックオーバーフローについては、呼び出し元の findDependentObjects 等で管理されるスタック深さ制限に依存する。

保守担当者向けメモ

  • Var ノードの処理において、JOIN のエイリアス変数などは明示的に無視される設計となっている。
  • 新しいノードタイプや定数型を追加する場合は、switch 文および IsA チェックを拡張する必要がある。
  • reg* 型の定数処理は、システムカタログの OID 整合性に強く依存している。
