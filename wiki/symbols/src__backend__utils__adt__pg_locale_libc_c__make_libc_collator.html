make_libc_collator

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:496

目的

OSのロケール設定に基づき、照合順序（LC_COLLATE）および文字分類（LC_CTYPE）を処理するための locale_t ハンドルを生成する。

シグネチャまたは形状

static locale_t make_libc_collator(const char *collate, const char *ctype)

入力と出力

  • **入力**:
  • collate: 照合順序に使用するロケール名（文字列）。
  • ctype: 文字分類に使用するロケール名（文字列）。
  • **出力**:
  • 成功時: 生成された locale_t 型のロケールハンドル。
  • 失敗時: report_newlocale_failure を呼び出し、エラーを報告する。

前提条件と不変条件

  • collate と ctype が同一の場合、POSIX環境では newlocale を使用し、Windows環境では _create_locale を使用する。
  • collate と ctype が異なる場合、POSIX環境では個別に newlocale を呼び出して合成するが、Windows環境ではサポートされていないためエラーを発生させる。
  • "C" または "POSIX" ロケールが指定された場合、それらはデフォルトのロケールとして扱われ、明示的な newlocale 呼び出しはスキップされる。

アルゴリズムまたは振る舞い

1. collate と ctype の文字列比較を行う。 2. **同一の場合**:

  • "C" または "POSIX" 以外であれば、newlocale（POSIX）または _create_locale（Windows）でハンドルを生成する。

    • 3. **異なる場合**:

  • POSIX環境: LC_COLLATE_MASK を指定して collate 用のロケールを作成し、次に LC_CTYPE_MASK を指定して ctype 用のロケールを重ね合わせる。
  • Windows環境: ereport(ERROR, ...) を発行し、機能非対応として処理を中断する。

    • 4. 生成に失敗した場合は report_newlocale_failure を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale_libc.c 内の create_pg_locale_libc から呼び出されることが確認されている。
  • **呼び出し先**:
  • newlocale / freelocale (POSIX)
  • _create_locale (Windows)
  • report_newlocale_failure (エラーハンドリング)
  • strcmp (標準Cライブラリ)

関連データ構造

  • locale_t: OSが提供するロケールオブジェクトの型。

失敗モードと境界条件

  • **ロケール生成失敗**: newlocale 等が NULL を返した場合、report_newlocale_failure を経由してエラー終了する。
  • **Windowsの制約**: collate と ctype が異なる値を要求された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーとなる。

保守担当者向けメモ

  • 本関数はプラットフォーム依存のロケールAPIを抽象化している。Windows環境で collate と ctype の分離をサポートするには、pg_locale_t 構造体自体を拡張し、2つのロケールハンドルを保持する設計変更が必要となる可能性がある。
