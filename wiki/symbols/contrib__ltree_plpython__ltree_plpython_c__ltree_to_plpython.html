ltree_to_plpython

定義

  • 種別: function
  • 定義箇所: contrib/ltree_plpython/ltree_plpython.c:37

目的

PostgreSQL の ltree 型データを、Python のリスト型オブジェクトへ変換する。

シグネチャまたは形状

Datum ltree_to_plpython(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して渡される ltree 型のデータ。
  • **出力**: 変換された Python リストオブジェクトを指す Datum 型。

前提条件と不変条件

  • 入力された ltree 構造体は有効であること。
  • plpython モジュールが初期化され、必要な関数ポインタ（PLyUnicode_FromStringAndSize）が解決済みであること。

アルゴリズムまたは振る舞い

1. PG_GETARG_LTREE_P(0) を使用して入力 ltree を取得する。 2. ltree のレベル数 (numlevel) に基づき、PyList_New で Python リストを確保する。メモリ確保に失敗した場合は ereport でエラーを発生させる。 3. LTREE_FIRST で最初のレベルを取得し、ループ内で PLyUnicode_FromStringAndSize を用いて各ラベルを Python 文字列に変換する。 4. PyList_SetItem を使用して、変換した文字列をリストの各インデックスに格納する。 5. LEVEL_NEXT を使用して次のレベルへ順次移動する。 6. PG_FREE_IF_COPY で入力メモリを解放し、構築したリストを PointerGetDatum で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数呼び出しインターフェース（PG_FUNCTION_INFO_V1 により登録）。
  • **呼び出し先**:
  • PyList_New, PyList_SetItem: Python C API。
  • PLyUnicode_FromStringAndSize: plpython モジュールから動的に取得した関数。
  • PG_GETARG_LTREE_P, PG_FREE_IF_COPY, PointerGetDatum: PostgreSQL のマクロ。

関連データ構造

  • ltree: PostgreSQL の階層パスデータ構造。
  • ltree_level: ltree 内の個々のラベル要素。
  • PyObject: Python のオブジェクト構造体。

失敗モードと境界条件

  • **メモリ不足**: PyList_New が失敗した場合、ERRCODE_OUT_OF_MEMORY でエラーを報告する。
  • **データ構造の不整合**: ltree の内部構造が変更された場合、LEVEL_NEXT 等のマクロによる走査が正しく機能しない可能性がある。

保守担当者向けメモ

  • 本関数は plpython に直接リンクせず、実行時に _PG_init で解決された関数ポインタを利用する設計である。
  • ltree の内部表現に依存しているため、ltree 型の定義に変更があった場合は本関数の走査ロジックを再確認すること。
