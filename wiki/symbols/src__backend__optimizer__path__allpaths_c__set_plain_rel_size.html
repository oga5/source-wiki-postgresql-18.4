set_plain_rel_size

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/allpaths.c:571

目的

プレーンなリレーション（テーブル等）に対して、サイズ見積もり（行数や幅など）を計算し、リレーションのメタデータに反映させる。

シグネチャまたは形状

static void set_plain_rel_size(PlannerInfo *root, RelOptInfo *rel, RangeTblEntry *rte)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: サイズ見積もり対象となるリレーション情報。
  • rte: リレーションに対応するレンジテーブルエントリ。
  • **出力**: なし（rel 構造体の内部フィールドを更新する）。

前提条件と不変条件

  • rel は RTE_RELATION 型のプレーンなリレーションであること。
  • サイズ見積もりの前に、部分インデックスの述語条件が評価されている必要がある。

アルゴリズムまたは振る舞い

1. check_index_predicates(root, rel) を呼び出し、部分インデックスの述語がリレーションのサイズ見積もりに影響を与える可能性があるため、これを評価する。 2. set_baserel_size_estimates(root, rel) を呼び出し、リレーションの出力行数や幅などの見積もり値を計算し、rel 構造体に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/allpaths.c 内の set_rel_size 関数から呼び出される。
  • **呼び出し先**:
  • check_index_predicates
  • set_baserel_size_estimates

関連データ構造

  • PlannerInfo: プランナのコンテキスト情報。
  • RelOptInfo: オプティマイザが使用するリレーションの統計情報やパスを保持する構造体。
  • RangeTblEntry: クエリ内のテーブル参照情報。

失敗モードと境界条件

  • 特筆すべき失敗モードは定義されていない。入力ポインタが NULL の場合はセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は set_rel_size からの分岐先の一つであり、プレーンなテーブルに対する見積もりを担当する。
  • 部分インデックスの評価がサイズ見積もりに先行する必要があるため、処理順序の変更には注意が必要。
