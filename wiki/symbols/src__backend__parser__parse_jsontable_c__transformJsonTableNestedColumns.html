transformJsonTableNestedColumns

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_jsontable.c:451

目的

JSON_TABLE 構文における NESTED COLUMNS 句を再帰的に処理し、実行計画ツリー（JsonTablePlan）を構築します。複数のネストされたカラム定義が存在する場合、それらを結合して単一のプランを生成します。

シグネチャまたは形状

static JsonTablePlan *transformJsonTableNestedColumns(JsonTableParseContext *cxt, List *passingArgs, List *columns)

入力と出力

  • **入力**:
  • cxt: 解析コンテキスト（JsonTableParseContext）。
  • passingArgs: JSONパス評価時に渡される引数リスト。
  • columns: NESTED COLUMNS 句を含むカラム定義のリスト。
  • **出力**:
  • 構築された JsonTablePlan 構造体へのポインタ。

前提条件と不変条件

  • columns リスト内の各要素は JsonTableColumn ノードである必要があります。
  • JTC_NESTED 型のカラムのみが処理対象となります。

アルゴリズムまたは振る舞い

1. columns リストを走査し、JTC_NESTED 型のノードを抽出します。 2. 各ネストされたカラムに対し、名前が未設定であれば generateJsonTablePathName を呼び出して一意な名前を割り当てます。 3. transformJsonTableColumns を再帰的に呼び出し、ネストされたカラムのサブプランを生成します。 4. 複数のネストされたプランが存在する場合、makeJsonTableSiblingJoin を使用してそれらを順次結合（UNION相当の処理）します。 5. 最終的な結合済みプランを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_jsontable.c:363
  • **呼び出し先**:
  • transformJsonTableColumns
  • generateJsonTablePathName
  • makeJsonTableSiblingJoin

関連データ構造

  • JsonTablePlan: 実行計画のノード。
  • JsonTableColumn: カラム定義を保持するノード。
  • JsonTableParseContext: 解析状態を保持する共有コンテキスト。

失敗モードと境界条件

  • columns が空の場合、plan は NULL のまま返されます。
  • JTC_NESTED 以外のカラムがリストに含まれている場合、それらはスキップされます。

保守担当者向けメモ

  • 本関数は NESTED 句の再帰的な処理を担います。makeJsonTableSiblingJoin による結合は、行セットの UNION を意味するため、プランナがこの構造を正しく解釈できることを前提としています。
