RangeBox

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/geo_spgist.c:109

目的

2次元の矩形（BOX）をSP-GiSTインデックスで扱う際、その範囲をX軸とY軸の2つの Range に分解して保持するための構造体です。

シグネチャまたは形状

typedef struct
{
    Range       left;
    Range       right;
} RangeBox;

入力と出力

  • **入力**: BOX 型データから getRangeBox 関数を介して生成されます。
  • **出力**: 空間検索における象限判定（getQuadrant）や、重なり判定（overlap2D, contain2D 等）の引数として利用されます。

前提条件と不変条件

  • left フィールドは範囲の下限側（low）、right フィールドは上限側（high）の情報を保持します。
  • 空間インデックスの構築および探索において、矩形の境界条件を表現する不変の単位として扱われます。

アルゴリズムまたは振る舞い

  • 本構造体は、2次元の矩形を4次元空間（x_low, x_high, y_low, y_high）の点としてマッピングする過程で、各次元の範囲を管理するために使用されます。
  • nextRectBox 関数において、現在の探索範囲と組み合わせて次の探索対象となる象限を計算する際の基準となります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/geo_spgist.c 内の各関数（getRangeBox, nextRectBox, overlap2D, contain2D, contained2D 等）から参照・生成されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Range: RangeBox を構成する要素。
  • RectBox: 4次元空間における探索範囲を保持する構造体。
  • BOX: 本構造体の元となる幾何データ型。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保に失敗した場合、PostgreSQLの標準的なエラーハンドリングによりアボートします。
  • 境界値: Range の値が不正な場合（下限が上限を超える等）、後続の空間演算関数で予期せぬ挙動を引き起こす可能性があります。

保守担当者向けメモ

  • RangeBox は traversalMemoryContext 上で動的に確保されることが多いため、メモリ管理には注意が必要です。
  • 4次元空間へのマッピングロジックを変更する場合、RangeBox の left/right の解釈と getQuadrant のロジックが整合していることを確認してください。
