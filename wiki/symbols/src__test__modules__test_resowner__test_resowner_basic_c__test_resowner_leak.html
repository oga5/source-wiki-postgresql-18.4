test_resowner_leak

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_resowner/test_resowner_basic.c:139

目的

ResourceOwner 機構において、リソースを明示的に解放（ResourceOwnerForget）せずに所有者を削除した場合の挙動を検証するテスト関数です。

シグネチャまたは形状

Datum test_resowner_leak(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（PostgreSQLの関数呼び出し規約に従う引数）
  • 出力: void（PG_RETURN_VOID() を介して返却）

前提条件と不変条件

  • CurrentResourceOwner が有効な親所有者として存在していること。
  • string_desc がリソースの解放処理（ReleaseString）を定義していること。

アルゴリズムまたは振る舞い

1. ResourceOwnerCreate を呼び出し、CurrentResourceOwner を親とする新しい ResourceOwner を作成します。 2. ResourceOwnerEnlarge で所有者の容量を確保します。 3. ResourceOwnerRemember を使用して、文字列リソースを所有者に登録します。 4. 意図的に ResourceOwnerForget を呼び出さず、リソースをリークさせた状態にします。 5. RESOURCE_RELEASE_BEFORE_LOCKS、RESOURCE_RELEASE_LOCKS、RESOURCE_RELEASE_AFTER_LOCKS の各フェーズで ResourceOwnerRelease を実行します。 6. ResourceOwnerDelete を呼び出し、所有者を破棄します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先: ResourceOwnerCreate, ResourceOwnerEnlarge, ResourceOwnerRemember, ResourceOwnerRelease, ResourceOwnerDelete。

関連データ構造

  • ResourceOwner: リソースのライフサイクルを管理する構造体。
  • string_desc: リソースの解放コールバックを定義する ResourceOwnerDesc 構造体。

失敗モードと境界条件

  • 本関数は意図的なリソースリークを発生させるテストケースであるため、正常系として動作します。
  • ResourceOwnerRelease 呼び出し後の操作制限など、ResourceOwner の内部状態遷移に関する制約に違反した場合、PostgreSQLの内部エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はテストモジュールの一部であり、実運用コードではありません。
  • リソース解放の順序や、リーク時の挙動を検証する目的で使用されます。
  • 呼び出し元および呼び出し先の詳細なスタックトレースは、PostgreSQLのバックエンド実装（src/backend/utils/resowner/resowner.c）に依存します。
