handle_sig_alarm

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:363

目的

SIGALRM シグナルを受信した際に、期限切れとなったタイムアウトタスクを特定して実行し、必要に応じて次回のタイマーを再スケジュールするシグナルハンドラです。

シグネチャまたは形状

static void handle_sig_alarm(SIGNAL_ARGS)

入力と出力

  • **入力**: SIGNAL_ARGS（シグナルハンドラの標準引数）。
  • **出力**: なし（副作用としてプロセスの状態更新およびタイマーの再設定を行う）。

前提条件と不変条件

  • HOLD_INTERRUPTS() により、ハンドラ実行中の割り込みを禁止する。
  • alarm_enabled が真の場合のみ、タイムアウト処理を実行する。
  • active_timeouts 配列は期限順にソートされている必要がある。

アルゴリズムまたは振る舞い

1. HOLD_INTERRUPTS() を呼び出し、割り込みを禁止する。 2. SetLatch(MyLatch) を呼び出し、待機中のプロセスを起床させる。 3. signal_pending フラグを false にリセットする。 4. alarm_enabled が有効な場合、disable_alarm() で一時的にタイマーを停止する。 5. active_timeouts を走査し、現在時刻 (now) を過ぎているタスクを順次実行する。

  • remove_timeout_index(0) でタスクをリストから削除する。
  • timeout_handler() を呼び出す。
  • 繰り返し実行が必要なタスク（interval_in_ms > 0）の場合、次回の期限を計算して enable_timeout() で再登録する。

    • 6. 全ての処理終了後、schedule_alarm(now) を呼び出し、次のタイマーを予約する。 7. RESUME_INTERRUPTS() を呼び出し、割り込みを許可して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: SIGALRM シグナル発生時にカーネルから呼び出される。src/backend/utils/misc/timeout.c 内で pqsignal(SIGALRM, handle_sig_alarm) として登録される。
  • **呼び出し先**: SetLatch, disable_alarm, remove_timeout_index, timeout_handler (各タスクのコールバック), enable_timeout, schedule_alarm, GetCurrentTimestamp, TimestampTzPlusMilliseconds。

関連データ構造

  • active_timeouts: 実行待ちのタイムアウトタスクを保持する配列。
  • timeout_params: 個々のタイムアウト設定を保持する構造体。
  • alarm_enabled: タイムアウト処理の有効/無効を制御するフラグ。
  • signal_pending: シグナルが保留中かを示すフラグ。

失敗モードと境界条件

  • num_active_timeouts が 0 の場合、タスク実行ループはスキップされる。
  • タイムアウト処理中に now を更新し、ハンドラ実行時間の増大に対応する。
  • new_fin_time の計算において、大幅な遅延が発生した場合は現在時刻を基準に再計算を行う。

保守担当者向けメモ

  • ハンドラ内での elog(FATAL) や longjmp は状態の不整合を招くため避けること。
  • CheckDeadLock のような重い処理がハンドラ内で呼び出される可能性があるため、実行時間の考慮が必要。
  • signal_pending は alarm_enabled の状態に関わらずリセットされる。
