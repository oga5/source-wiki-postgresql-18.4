ForkNumber

定義

  • 種別: enum
  • 定義箇所: src/include/common/relpath.h:56

目的

PostgreSQLの物理ストレージ層において、リレーション（テーブルやインデックス）を構成する個別のデータファイル（フォーク）を識別するために使用されます。

シグネチャまたは形状

typedef enum ForkNumber
{
    InvalidForkNumber = -1,
    MAIN_FORKNUM = 0,
    FSM_FORKNUM,
    VISIBILITYMAP_FORKNUM,
    INIT_FORKNUM,
} ForkNumber;

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: リレーションの各フォークを指す整数値

前提条件と不変条件

  • MAIN_FORKNUM は 0 から始まります。
  • 新しいフォークを追加する場合、MAX_FORKNUM および FORKNAMECHARS の更新、ならびに src/common/relpath.c 内の forkNames 配列への追記が必須です。

アルゴリズムまたは振る舞い

リレーションの物理パスを生成する際、どのフォークのファイルにアクセスするかを決定するための識別子として機能します。GetRelationPath 等のパス生成関数において、ファイル名の一部を決定する根拠となります。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • contrib/bloom/bloom.h / blutils.c (BloomInitMetapage)
  • contrib/pageinspect/rawpage.c (get_raw_page_internal)
  • contrib/pg_buffercache/pg_buffercache_pages.c
  • contrib/pg_prewarm/autoprewarm.c / pg_prewarm.c
  • contrib/pg_visibility/pg_visibility.c
  • 呼び出し先: 情報なし

関連データ構造

  • RelFileLocator: リレーションを特定するための構造体（src/include/storage/relfilelocator.h で定義）

失敗モードと境界条件

  • InvalidForkNumber (-1) は無効なフォーク状態を表します。
  • 定義された範囲外の値を指定した場合の動作は、利用側の実装に依存します。

保守担当者向けメモ

  • 本シンボルは物理ストレージの構造に直結しています。フォークの追加・変更時は必ず src/common/relpath.c の対応する配列と同期させてください。
