AllocSetReset

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/aset.c:536

目的

AllocSet メモリコンテキスト内の全メモリチャンクを解放し、コンテキストを初期状態へリセットします。

シグネチャまたは形状

void AllocSetReset(MemoryContext context)

入力と出力

  • **入力**: MemoryContext context（リセット対象の AllocSet コンテキスト）
  • **出力**: なし（副作用としてコンテキストの状態が更新される）

前提条件と不変条件

  • context は AllocSetIsValid を満たす有効な AllocSet であること。
  • リセット完了後、context->mem_allocated はキーパーブロック（コンテキストヘッダーと共有される初期ブロック）のサイズと一致すること。

アルゴリズムまたは振る舞い

1. MEMORY_CONTEXT_CHECKING が有効な場合、AllocSetCheck を呼び出しメモリの整合性を検証する。 2. コンテキスト内の全フリーリストを MemSetAligned でクリアする。 3. キーパーブロック以外の全ブロックを走査し、free() を実行してメモリを OS に返却する。 4. キーパーブロックについては、メモリを解放せず、内部の freeptr を初期位置に巻き戻し、再利用可能な状態にする。 5. set->nextBlockSize を set->initBlockSize にリセットし、次回の割り当てサイズシーケンスを初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c において、MemoryContextMethods の reset メソッドとして登録されている。
  • **呼び出し先**:
  • AllocSetCheck (デバッグ時)
  • free() (標準ライブラリ)
  • wipe_mem() (デバッグ時)
  • VALGRIND_MAKE_MEM_NOACCESS (Valgrind 使用時)

関連データ構造

  • AllocSet: メモリコンテキストの主要構造体。
  • AllocBlock: 管理対象のメモリブロック。
  • MemoryContext: 抽象インターフェース。

失敗モードと境界条件

  • コンテキストが破損している場合、AllocSetCheck によりエラーが発生する可能性がある。
  • キーパーブロックのみが存在する状態では、ブロックの解放処理はスキップされ、単なるポインタのリセットのみが行われる。

保守担当者向けメモ

  • 本関数は AllocSetDelete と異なり、コンテキスト自体は破棄せず再利用可能な状態に保つ。
  • KeeperBlock はコンテキスト構造体とメモリを共有しているため、解放対象から除外するロジックが必須である。
