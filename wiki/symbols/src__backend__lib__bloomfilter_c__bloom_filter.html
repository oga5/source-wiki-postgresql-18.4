bloom_filter

定義

  • 種別: struct
  • 定義箇所: src/backend/lib/bloomfilter.c:44

目的

集合への要素の包含判定を確率的に行うためのデータ構造です。メモリ効率を重視し、偽陽性を許容しつつ偽陰性を発生させない設計となっています。

シグネチャまたは形状

struct bloom_filter
{
    int         k_hash_funcs;
    uint64      seed;
    uint64      m;
    unsigned char bitset[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: 構造体内の seed（ハッシュ関数用）、m（ビットセットサイズ）、および k_hash_funcs（使用するハッシュ関数の数）。
  • **出力**: bitset メンバを介して、要素の存在判定結果（包含の有無）を保持します。

前提条件と不変条件

  • m（ビットセットサイズ）は 2 のべき乗である必要があります。
  • m は最大 2^32 ビット（512MB）に制限されています。
  • bitset は FLEXIBLE_ARRAY_MEMBER を使用した可変長配列として定義されています。

アルゴリズムまたは振る舞い

  • **ハッシュ生成**: 拡張ダブルハッシュ法を用い、k 個のハッシュ値を算出します。
  • **ビット操作**: ビットセットサイズが 2 のべき乗であることを利用し、剰余演算をビットマスクで高速化します。
  • **判定**: 要素追加時はビットセットの該当位置を OR 操作でセットし、判定時は AND 操作で全ビットがセットされているかを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • src/backend/utils/adt/acl.c
  • src/backend/lib/bloomfilter.c 内の各関数（bloom_create, bloom_add_element, bloom_lacks_element 等）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数呼び出しの対象ではありません。操作関数（bloom_create 等）が本構造体を引数として受け取ります。

関連データ構造

  • src/include/lib/bloomfilter.h: 公開インターフェース定義。

失敗モードと境界条件

  • **メモリ制限**: ビットセットサイズが 512MB を超えるような設定は、32 ビットハッシュ値の範囲制限によりサポートされません。
  • **偽陽性**: 確率的なデータ構造であるため、要素が含まれていない場合でも「含まれている可能性がある」と判定される場合があります。

保守担当者向けメモ

  • bitset の埋まり具合を確認する際は bloom_prop_bits_set を使用してください。理想的な埋まり具合は 0.5 前後です。
  • 構造体のメモリ確保には offsetof(bloom_filter, bitset) を使用した動的なサイズ計算が必要です。
