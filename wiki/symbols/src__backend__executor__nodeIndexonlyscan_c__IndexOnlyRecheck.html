IndexOnlyRecheck

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:325

目的

インデックスのみスキャン（Index-Only Scan）実行中に、EvalPlanQual（EPQ）によるタプルの再チェック要求が発生した際、エラーを発生させて処理を中断する。

シグネチャまたは形状

static bool IndexOnlyRecheck(IndexOnlyScanState *node, TupleTableSlot *slot)

入力と出力

  • 入力:
  • node: IndexOnlyScanState 構造体へのポインタ。
  • slot: 再チェック対象の TupleTableSlot。
  • 出力:
  • 常にエラーを投げるため、呼び出し元へ制御は戻らない。

前提条件と不変条件

  • インデックスのみスキャンは、EvalPlanQual（行レベルロックの再チェック）をサポートしていない。

アルゴリズムまたは振る舞い

1. elog(ERROR, ...) を呼び出し、インデックスのみスキャンではEPQ再チェックがサポートされていない旨を通知する。 2. コンパイラの警告を抑制するためのダミーの return false を含む。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/nodeIndexonlyscan.c:349 にて ExecScanRecheckMtd として登録されている。
  • 呼び出し先: elog（PostgreSQLのエラー報告関数）。

関連データ構造

  • IndexOnlyScanState: インデックスのみスキャンの実行状態を保持する構造体。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • 本関数が呼び出された時点で、常に ERROR レベルの例外が発生し、トランザクションまたはサブトランザクションがアボートされる。

保守担当者向けメモ

  • インデックスのみスキャンがEPQをサポートしていないという設計上の制約を反映した実装である。将来的にEPQをサポートする必要が生じた場合、この関数を適切に実装し直す必要がある。
