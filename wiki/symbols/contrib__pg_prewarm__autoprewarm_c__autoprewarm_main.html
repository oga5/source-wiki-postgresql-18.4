autoprewarm_main

定義

  • 種別: function
  • 定義箇所: contrib/pg_prewarm/autoprewarm.c:166

目的

PostgreSQLのバックグラウンドワーカーとして、共有バッファ状態の永続化（ダンプ）と、サーバー起動時のバッファ再構築（プリウォーム）を管理するメインループを実行する。

シグネチャまたは形状

void autoprewarm_main(Datum main_arg)

入力と出力

  • **入力**: Datum main_arg（バックグラウンドワーカーの引数）。
  • **出力**: なし（関数は終了またはプロセス終了までループする）。

前提条件と不変条件

  • apw_state（共有メモリ上の制御構造体）が初期化されていること。
  • 実行時に SIGTERM、SIGHUP、SIGUSR1 に対するシグナルハンドラが設定されていること。
  • 既に別の autoprewarm ワーカーが稼働している場合、本関数は即座に終了する。

アルゴリズムまたは振る舞い

1. **初期化**: シグナルハンドラを設定し、共有メモリ領域へアタッチする。before_shmem_exit に apw_detach_shmem を登録する。 2. **排他制御**: 共有メモリ内の bgworker_pid を確認し、自身が唯一のリーダーワーカーであることを保証する。 3. **初回ロード**: 共有メモリが新規作成された場合、apw_load_buffers() を呼び出し、既存のダンプファイルからバッファ情報を読み込む。 4. **メインループ**: ShutdownRequestPending が真になるまで以下を繰り返す。

  • SIGHUP 受信時に設定ファイルを再読み込みする。
  • autoprewarm_interval に基づき、次回のダンプ時刻まで WaitLatch で待機する。
  • 待機時間が経過した場合、apw_dump_now(true, false) を呼び出し、現在のバッファ状態をファイルへ書き出す。

    • 5. **終了処理**: ループ終了時、シャットダウン要求が保留中でなければ apw_dump_now(true, true) を呼び出し、最終的な状態を保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: バックグラウンドワーカーの起動プロセス（bgworker フレームワーク経由）。
  • **呼び出し先**:
  • apw_init_shmem, apw_detach_shmem
  • apw_load_buffers
  • apw_dump_now
  • ProcessConfigFile
  • WaitLatch, ResetLatch

関連データ構造

  • AutoPrewarmSharedState（共有メモリ上の制御構造体）
  • MyProcPid（現在のプロセスID）
  • MyLatch（プロセス待機用ラッチ）

失敗モードと境界条件

  • **多重起動**: 既にワーカーが稼働している場合、ログを出力して終了する。
  • **シャットダウン中の割り込み**: apw_load_buffers() 中にシャットダウン要求が発生した場合、不完全な状態でのダンプを防ぐため、ループ後の最終ダンプをスキップする。
  • **間隔設定**: autoprewarm_interval <= 0 の場合、定期的なダンプを行わず、シャットダウン時のみダンプを行う待機モードへ移行する。

保守担当者向けメモ

  • 本関数はリーダーワーカーとして動作する。個別のデータベースに対するプリウォーム処理は、本関数から起動される別のワーカー（autoprewarm_database_main）が担当する。
  • apw_dump_now の呼び出しは、共有メモリのロック状態やシャットダウンのタイミングに依存するため、変更時は排他制御の整合性に注意すること。
