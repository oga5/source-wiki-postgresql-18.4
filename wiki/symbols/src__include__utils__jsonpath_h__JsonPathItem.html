JsonPathItem

定義

  • **種別**: struct
  • **定義箇所**: src/include/utils/jsonpath.h:135

目的

SQL/JSON Path 式のバイナリ形式を、実行時に走査・評価するための抽象化構造体です。

シグネチャまたは形状

typedef struct JsonPathItem {
    JsonPathItemType type;
    int32            nextPos;
    char            *base;
    union {
        struct { int32 left; int32 right; } args;
        int32        arg;
        struct { int32 nelems; struct { int32 from; int32 to; } *elems; } array;
        struct { uint32 first; uint32 last; } anybounds;
        struct { char *data; int32 datalen; } value;
        struct { int32 expr; char *pattern; int32 patternlen; uint32 flags; } like_regex;
    } content;
} JsonPathItem;

入力と出力

  • **入力**: JsonPath 型のバイナリデータから構築されます。base ポインタは、現在のノードが属するバイナリデータ内の相対位置の基準点となります。
  • **出力**: content 共用体を通じて、演算子のオペランド、配列インデックス、リテラル値、または正規表現パターンなどのノード固有情報を提供します。

前提条件と不変条件

  • nextPos は、現在のノードから次のノードへの相対的なオフセットを保持します。
  • base ポインタは、オンディスク形式の JsonPath 構造を指し、すべての内部位置情報はこれに対する相対値として扱われます。
  • JsonPathItemType の順序はオンディスク形式と直結しているため、後方互換性の維持が必要です。

アルゴリズムまたは振る舞い

  • パスの走査は、演算子ではなく「左オペランド」を起点とする設計です。
  • jspInit および jspGetNext 関数（本構造体を使用）により、リンクを辿ってパスを降下（descend）します。
  • content 共用体は、type に応じて適切なデータ構造を選択し、jspGetArg 等のアクセサを介してノードの属性を取得します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonb_gin.c 内の GIN インデックス関連関数（jsonb_ops__add_path_item, jsonb_path_ops__add_path_item など）から参照されています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しは含みません。操作には jspInit や jspGetNext などの関連関数が使用されます。

関連データ構造

  • JsonPath: オンディスク形式のバイナリ JSON Path。
  • JsonPathItemType: ノードの種類を識別する列挙型。
  • JsonPathParseItem: 解析時の中間表現。

失敗モードと境界条件

  • base ポインタが無効なメモリ領域を指している場合、走査時に未定義の動作を引き起こします。
  • nextPos が不正なオフセット値を持つ場合、リンクの走査が異常終了する可能性があります。

保守担当者向けメモ

  • JsonPathItemType の変更はオンディスク形式に影響するため、追加は必ず末尾に行うこと。
  • デバッグ時には jspOperationName を使用して、列挙型から演算子名を取得することが推奨されます。
