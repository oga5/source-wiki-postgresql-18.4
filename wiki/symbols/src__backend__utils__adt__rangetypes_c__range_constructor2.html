range_constructor2

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes.c:378

目的

2つの引数（下限値と上限値）から範囲型（RangeType）を構築します。

シグネチャまたは形状

Datum range_constructor2(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの Datum を受け取ります。
  • **出力**: 構築された RangeType を Datum として返します（PG_RETURN_RANGE_P マクロを使用）。

前提条件と不変条件

  • 呼び出し元の関数式から対象の範囲型 OID を取得できる必要があります。
  • range_get_typcache により、対象の範囲型に関するメタデータ（TypeCacheEntry）が取得可能である必要があります。
  • 構築される範囲は、下限が包含（inclusive）、上限が非包含（exclusive）の [a, b) 形式として初期化されます。

アルゴリズムまたは振る舞い

1. get_fn_expr_rettype を用いて、関数の戻り値型から範囲型の OID を特定します。 2. range_get_typcache を呼び出し、型キャッシュを取得します。 3. 引数から RangeBound 構造体（lower および upper）を生成します。

  • 引数が NULL の場合、infinite フラグを立てて無限大として扱います。
  • lower は包含（inclusive = true）、upper は非包含（inclusive = false）に設定されます。

    • 4. make_range を呼び出し、境界値に基づいた範囲型を構築します。 5. 構築された範囲型を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/typecmds.c:1781 にて、この関数名が文字列として参照されています。
  • **呼び出し先**:
  • get_fn_expr_rettype
  • range_get_typcache
  • make_range

関連データ構造

  • RangeType: 範囲型の内部表現。
  • RangeBound: 範囲の境界を定義する構造体。
  • TypeCacheEntry: 範囲型のメタデータキャッシュ。

失敗モードと境界条件

  • 引数が NULL の場合、infinite として処理されます。
  • make_range 内部で正規化（canonical）関数が呼び出される場合、その関数の制約に従います。

保守担当者向けメモ

  • 本関数は [a, b) 形式の範囲構築専用です。3引数版の range_constructor3 と比較して、境界の包含フラグが固定されている点に注意してください。
  • 範囲型のメモリ管理は varlena 形式に従うため、構築後のメモリ解放は PostgreSQL のメモリコンテキスト管理に依存します。
