WordEntry

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_type.h:41

目的

tsvector 内の個々の語彙（lexeme）を管理するためのメタデータ構造体です。語彙の長さ、位置情報フラグ、およびデータ本体へのオフセットを保持します。

シグネチャまたは形状

typedef struct
{
	uint32
				haspos:1,
				len:11,			/* MAX 2Kb */
				pos:20;			/* MAX 1Mb */
} WordEntry;

入力と出力

  • 入力: tsvector 内のバイナリデータの一部として配置されます。
  • 出力: len フィールドにより語彙の長さを、pos フィールドによりデータ本体の開始位置を、haspos フラグにより位置情報の有無をそれぞれ提供します。

前提条件と不変条件

  • len は最大 2KB、pos は最大 1MB までの値を保持可能です。
  • tsvector 内では、ヘッダーの直後に WordEntry の配列が配置され、その後に実際の文字列データと位置情報ベクトルが続くメモリレイアウトを前提としています。

アルゴリズムまたは振る舞い

  • tsvector 内の語彙を検索・走査する際、ARRPTR() マクロで取得したポインタを介してこの構造体にアクセスします。
  • pos フィールドは、tsvector のデータ領域内における語彙の開始位置を指し示します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/tsearch/to_tsany.c
  • src/backend/tsearch/ts_typanalyze.c
  • src/backend/utils/adt/tsginidx.c
  • src/backend/utils/adt/tsgistidx.c
  • src/backend/utils/adt/tsrank.c
  • 呼び出し先: 呼び出し先に関する情報は提供されていません。

関連データ構造

  • TSVectorData: WordEntry を含む配列を保持する親構造体。
  • WordEntryPos: 語彙に関連付けられた位置情報を格納する構造体。

失敗モードと境界条件

  • len や pos のビット幅を超えるデータ長を持つ語彙を扱う場合、オーバーフローが発生する可能性があります。
  • tsvector のメモリレイアウト（アライメントや varlena ヘッダー）が正しく維持されていない場合、ポインタ演算によるアクセスで不正なメモリ参照が発生するリスクがあります。

保守担当者向けメモ

  • FLEXIBLE_ARRAY_MEMBER を使用した TSVectorData の一部として配置されるため、構造体のサイズ計算には CALCDATASIZE や COMPUTESIZE マクロを使用してください。
  • 構造体のビットフィールド定義は、プラットフォームごとのエンディアンやアライメントの影響を受ける可能性があるため、バイナリ互換性を維持する際は注意が必要です。
