gets_none

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_none.c:119

目的

非圧縮モードにおけるアーカイブデータからの文字列読み込み処理を提供します。

シグネチャまたは形状

char *gets_none(char *ptr, int size, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • ptr: 読み込んだデータを格納するバッファへのポインタ。
  • size: 読み込む最大バイト数。
  • CFH: 非圧縮ファイルハンドルを保持する CompressFileHandle 構造体。
  • **出力**:
  • 成功時は ptr を返し、失敗または EOF 到達時は NULL を返します。

前提条件と不変条件

  • CFH->private_data には有効な FILE* が格納されている必要があります。

アルゴリズムまたは振る舞い

1. CFH->private_data を FILE* にキャストします。 2. 標準ライブラリ関数 fgets を呼び出し、指定されたファイルストリームからデータを読み込みます。 3. fgets の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_none.c 内で CFH->gets_func に関数ポインタとして代入されています。
  • **呼び出し先**: 標準ライブラリの fgets を呼び出します。

関連データ構造

  • CompressFileHandle: 非圧縮 I/O の状態を保持する構造体。private_data メンバを介して FILE* を管理します。

失敗モードと境界条件

  • fgets が失敗した場合、または EOF に達した場合は NULL を返します。
  • CFH が NULL である場合や、private_data が無効なファイルポインタである場合の挙動は未定義です。

保守担当者向けメモ

  • 本関数は fgets の単純なラッパーです。エラー発生時の詳細な診断が必要な場合は、呼び出し元で ferror 等を確認する必要があります。
