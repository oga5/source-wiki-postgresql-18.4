set_cheapest

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/pathnode.c:271

目的

RelOptInfo が保持する候補パスリスト（pathlist）を走査し、起動コストおよび総コストの観点から最適なパスを特定して、リレーションの最適パス属性を確定させる。

シグネチャまたは形状

void set_cheapest(RelOptInfo *parent_rel)

入力と出力

  • **入力**: parent_rel (最適化対象のリレーション情報 RelOptInfo へのポインタ)
  • **出力**: なし（副作用として parent_rel の各フィールドを更新）

前提条件と不変条件

  • parent_rel は有効な RelOptInfo 構造体であること。
  • parent_rel->pathlist が空であってはならない（空の場合はエラーを投げる）。
  • 実行終了後、parent_rel->cheapest_total_path は必ず非 NULL となる。

アルゴリズムまたは振る舞い

1. parent_rel->pathlist を走査し、パスを「パラメータ化されたパス」と「非パラメータ化パス」に分類する。 2. **非パラメータ化パス**:

  • 起動コスト（STARTUP_COST）および総コスト（TOTAL_COST）が最小のものを追跡する。
  • コストが等しい場合は、pathkeys を比較し、より優れたソート順を持つパスを優先する。

    • 3. **パラメータ化パス**:

  • パラメータ化の度合い（PATH_REQ_OUTER）を比較し、最小のパラメータ化を持つパスの中で総コストが最小のものを best_param_path として追跡する。

    • 4. **結果の格納**:

  • cheapest_startup_path および cheapest_total_path を設定する。
  • 非パラメータ化パスが存在しない場合、best_param_path を cheapest_total_path として代入する。
  • cheapest_unique_path を NULL に初期化する。
  • cheapest_parameterized_paths リストを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_eval.c
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/joinrels.c
  • src/backend/optimizer/plan/planmain.c
  • src/backend/optimizer/plan/planner.c
  • **呼び出し先**:
  • compare_path_costs
  • compare_pathkeys
  • bms_subset_compare
  • lfirst, lappend, lcons (リスト操作)

関連データ構造

  • RelOptInfo: パスリストおよび確定した最適パスを保持する構造体。
  • Path: 候補となる実行パス。
  • PathKeys: パスのソート順序情報。

失敗モードと境界条件

  • parent_rel->pathlist が NIL の場合、elog(ERROR) によりクエリプラン生成失敗として処理を中断する。

保守担当者向けメモ

  • 本関数はプランナの最終段階に近い位置で呼び出される。
  • cheapest_unique_path は本関数では計算されず、必要に応じて別途計算される設計となっている。
  • パラメータ化パスの比較において、BMS_DIFFERENT（パラメータ化の包含関係が不明）の場合は古いパスを維持する保守的な挙動をとる。
