int8gcd_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int8.c:605

目的

2つの64ビット整数（int64）の最大公約数（GCD）を計算します。

シグネチャまたは形状

static int64 int8gcd_internal(int64 arg1, int64 arg2)

入力と出力

  • **入力**: arg1 (int64), arg2 (int64)
  • **出力**: 両引数の最大公約数を表す正の int64 値。

前提条件と不変条件

  • INT64_MIN を含むあらゆる64ビット整数を処理可能です。
  • 内部計算は負の空間で行われ、最終結果は必ず正の値として返されます。

アルゴリズムまたは振る舞い

1. **正規化**: 入力値を負の空間で扱い、絶対値が大きい方を arg1 に配置します。 2. **特殊ケース処理**: arg1 が PG_INT64_MIN の場合、arg2 が 0 または PG_INT64_MIN であればエラーを報告します。また、arg2 が -1 の場合は 1 を即座に返します。 3. **ユークリッドの互除法**: arg2 が 0 になるまで剰余演算を繰り返します。 4. **結果の正値化**: 最終的な arg1 が負であれば符号を反転させ、正の値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/int8.c 内の関数（int8gcd 等のSQLインターフェース関数から参照されていることが示唆されています）。
  • **呼び出し先**: ereport（エラー発生時）。

関連データ構造

  • int64 (PostgreSQLの64ビット整数型)

失敗モードと境界条件

  • **範囲外エラー**: arg1 が PG_INT64_MIN であり、かつ arg2 が 0 または PG_INT64_MIN の場合、ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE でエラーとなります。
  • **浮動小数点例外の回避**: 特定の環境で発生しうる INT64_MIN % -1 の例外を回避するガードロジックが含まれています。

保守担当者向けメモ

  • 本関数は int8 型の算術演算の一部として実装されています。
  • INT64_MIN の扱いは複雑であるため、ロジックを変更する際は負の空間での計算ロジックを維持してください。
