addrange

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_cvec.c:92

目的

正規表現のブラケット式（[...]）解析において、文字セット（cvec）に新しい文字範囲を追加する。

シグネチャまたは形状

static void addrange(struct cvec *cv, chr from, chr to)

入力と出力

  • **入力**:
  • cv: 操作対象の cvec 構造体へのポインタ。
  • from: 追加する範囲の開始文字。
  • to: 追加する範囲の終了文字。
  • **出力**: なし（副作用として cv の内容を更新）。

前提条件と不変条件

  • cv->nranges が cv->rangespace 未満であること（アサーションにより検証）。
  • cvec 構造体は regguts.h で定義されたレイアウトに従う。

アルゴリズムまたは振る舞い

1. cv->ranges 配列の現在のインデックス（nranges * 2）に from を格納する。 2. その直後のインデックス（nranges * 2 + 1）に to を格納する。 3. cv->nranges をインクリメントし、範囲の総数を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコードの文脈上、regcomp.c 内のブラケット式解析ロジックから呼び出されることが想定されるが、具体的な呼び出し箇所は本抜粋からは特定できない。
  • **呼び出し先**: なし。
  • **注意**: ツールが検出した src/backend/access/brin/brin.c 内の addrange は変数名であり、本関数とは無関係である。

関連データ構造

  • struct cvec: 文字セットを保持する構造体。ranges 配列、範囲数 nranges、および最大容量 rangespace を含む。

失敗モードと境界条件

  • cv->nranges が cv->rangespace に達している場合、アサーション失敗によりプログラムが停止する可能性がある。

保守担当者向けメモ

  • 本関数は static であり、regcomp.c にインクルードされる regc_cvec.c 内でのみ使用される。
  • cvec 内部のメモリ管理は getcvec 等の関数が担うため、本関数を呼び出す前に十分な容量が確保されていることを確認する必要がある。
