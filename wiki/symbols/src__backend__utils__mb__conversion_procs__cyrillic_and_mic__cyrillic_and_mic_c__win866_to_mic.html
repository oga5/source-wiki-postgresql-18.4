win866_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:405

目的

WIN866 エンコーディングから MULE_INTERNAL エンコーディングへの文字列変換を行う。

シグネチャまたは形状

Datum win866_to_mic(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得
  • src (CSTRING): 変換元文字列
  • dest (CSTRING): 変換先バッファ
  • len (INT32): 文字列長
  • noError (BOOL): エラーハンドリングフラグ
  • **出力**: Datum 型として変換されたバイト数を返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_WIN866, PG_MULE_INTERNAL) により、ソースエンコーディングが WIN866、ターゲットが MULE_INTERNAL であることが検証される。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認する。 3. latin2mic_with_table を呼び出し、win8662koi マッピングテーブルを使用して変換を実行する。 4. 変換されたバイト数を PG_RETURN_INT32 で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: latin2mic_with_table（変換処理の実行）。

関連データ構造

  • win8662koi: WIN866 から KOI8-R への変換テーブル。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断される。
  • noError フラグの値に応じて、変換中の不正な文字に対する挙動が latin2mic_with_table 内で制御される。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_ARGS を引数に取るため、直接呼び出さず PostgreSQL の関数呼び出し規約に従うこと。
  • 変換ロジックは latin2mic_with_table に依存しており、マッピングテーブル win8662koi の整合性が変換結果に直結する。
