unlimit_core_size

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress.c:173

目的

プロセスが生成するコアダンプファイルのサイズ制限を、システムが許可する最大値（ハードリミット）まで引き上げることで、テスト失敗時のデバッグ情報を確保する。

シグネチャまたは形状

static void unlimit_core_size(void)

入力と出力

  • 入力: なし（システムリソース設定を直接参照）
  • 出力: なし（副作用として setrlimit を実行）

前提条件と不変条件

  • sys/resource.h が提供する getrlimit および setrlimit が利用可能であること。
  • RLIMIT_CORE リソースタイプがシステムでサポートされていること。

アルゴリズムまたは振る舞い

1. getrlimit(RLIMIT_CORE, &lim) を呼び出し、現在のコアダンプ制限を取得する。 2. ハードリミット (lim.rlim_max) が 0 の場合、コアダンプが禁止されていると判断し、diag 関数で警告を出力して終了する。 3. ハードリミットが無限大 (RLIM_INFINITY) であるか、現在のソフトリミット (lim.rlim_cur) がハードリミットより小さい場合、ソフトリミットをハードリミットの値に更新する。 4. setrlimit(RLIMIT_CORE, &lim) を呼び出し、新しい制限を適用する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/regress/pg_regress.c 内のメイン処理フロー（main 相当の初期化プロセス）から呼び出される。
  • 呼び出し先: getrlimit, setrlimit (システムコール)、diag (ログ出力関数)。

関連データ構造

  • struct rlimit: リソース制限値を保持する構造体。

失敗モードと境界条件

  • ハードリミットが 0 に設定されている環境では、コアダンプの制限解除は行われず、警告メッセージが出力される。
  • setrlimit が権限不足等の理由で失敗した場合の戻り値チェックは行われていない。

保守担当者向けメモ

  • 本関数は src/bin/pg_ctl/pg_ctl.c にも同名の関数が存在するが、本ページは pg_regress.c に定義されたものに限定している。
  • プラットフォームによって RLIMIT_CORE の挙動が異なる可能性があるため、移植性を考慮する場合は sys/resource.h の仕様を確認すること。
