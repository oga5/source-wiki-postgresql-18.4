ConnCacheEntry

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/connection.c:54

目的

リモートの PostgreSQL サーバーへの接続状態、トランザクション深度、および関連するメタデータを保持し、postgres_fdw における接続キャッシュ（ConnectionHash）の各エントリとして機能する。

シグネチャまたは形状

typedef struct ConnCacheEntry
{
    ConnCacheKey key;
    PGconn      *conn;
    int          xact_depth;
    bool         have_prep_stmt;
    bool         have_error;
    bool         changing_xact_state;
    bool         parallel_commit;
    bool         parallel_abort;
    bool         invalidated;
    bool         keep_connections;
    Oid          serverid;
    uint32       server_hashvalue;
    uint32       mapping_hashvalue;
    PgFdwConnState state;
} ConnCacheEntry;

入力と出力

  • 本構造体は ConnectionHash ハッシュテーブルの要素として管理される。
  • conn が NULL の場合、xact_depth 以降のフィールドは無効とみなされる。

前提条件と不変条件

  • key フィールドは構造体の先頭に配置されなければならない。
  • xact_depth は 0（トランザクション未開始）、1（メイントランザクション）、2以上（サブトランザクションのネスト）の値をとり、リモート側の状態と同期される。

アルゴリズムまたは振る舞い

  • GetConnection 等の関数により、接続の確立、トランザクションの開始、およびエラー発生時のクリーンアップ状態の管理に使用される。
  • invalidated が真の場合、再接続が必要な状態であることを示す。
  • have_error は、リモート側でサブトランザクションがアボートした際に設定され、以降のトランザクション状態の整合性を制御する。

呼び出し元と呼び出し先

  • **呼び出し元**: connection.c 内の GetConnection、make_new_connection、disconnect_pg_server、begin_remote_xact、pgfdw_reset_xact_state、pgfdw_abort_cleanup 等が本構造体を引数として操作する。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • ConnCacheKey: ハッシュテーブルの検索キー。
  • ConnectionHash: 本構造体を格納するハッシュテーブル。
  • PGconn: libpq が提供する物理的な接続ハンドル。
  • PgFdwConnState: 接続ごとの追加状態を保持する列挙型または構造体。

失敗モードと境界条件

  • conn が NULL の場合、接続が確立されていないか、切断済みであることを示す。
  • リモート側でエラーが発生した場合、have_error が設定され、pgfdw_reject_incomplete_xact_state_change 等によって以降の利用が制限される可能性がある。

保守担当者向けメモ

  • ユーザーマッピング単位でキャッシュされるため、認証情報が異なる場合は別エントリとして扱われる。
  • トランザクションのネストレベル（xact_depth）の管理は、ローカルのトランザクション状態とリモートの SAVEPOINT を同期させるために不可欠である。
