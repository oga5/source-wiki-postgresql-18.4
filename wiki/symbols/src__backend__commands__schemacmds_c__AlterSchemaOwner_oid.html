AlterSchemaOwner_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/schemacmds.c:306

目的

指定されたスキーマ（OID）の所有者を新しい所有者（OID）に変更します。

シグネチャまたは形状

void AlterSchemaOwner_oid(Oid schemaoid, Oid newOwnerId)

入力と出力

  • **入力**:
  • schemaoid: 対象スキーマのOID。
  • newOwnerId: 新しい所有者のロールOID。
  • **出力**: なし（成功時はカタログ更新を実行、失敗時はエラーをスロー）。

前提条件と不変条件

  • pg_namespace システムカタログに対する RowExclusiveLock が取得可能であること。
  • 指定された schemaoid がシステムカタログ上に存在すること（存在しない場合は elog(ERROR) を発生させる）。

アルゴリズムまたは振る舞い

1. table_open を呼び出し、pg_namespace システムカタログを RowExclusiveLock モードで開く。 2. SearchSysCache1 を使用して、対象スキーマのタプルをキャッシュから取得する。 3. キャッシュヒットを確認し、無効な場合はエラーをスローする。 4. AlterSchemaOwner_internal を呼び出し、所有者変更の共通ロジック（ACL更新や依存関係処理）を実行する。 5. ReleaseSysCache でキャッシュタプルを解放する。 6. table_close で pg_namespace のロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_shdepend.c:1657
  • **呼び出し先**:
  • table_open / table_close
  • SearchSysCache1 / ReleaseSysCache
  • AlterSchemaOwner_internal

関連データ構造

  • pg_namespace: スキーマ情報を保持するシステムカタログ。
  • HeapTuple: システムカタログの行データ。

失敗モードと境界条件

  • 指定した schemaoid が存在しない場合、elog(ERROR) によりトランザクションが中断される。
  • ロック取得やカタログ更新時にデータベースの整合性制約に抵触した場合、例外が発生する。

保守担当者向けメモ

  • 本関数は所有者変更のラッパーであり、実際の更新ロジックは AlterSchemaOwner_internal に集約されています。変更を加える際は、依存関係の整合性に注意してください。
