ExecReScanValuesScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeValuesscan.c:327

目的

ValuesScanState ノードのスキャン位置を先頭にリセットし、実行状態を初期化する。

シグネチャまたは形状

void ExecReScanValuesScan(ValuesScanState *node)

入力と出力

  • **入力**: ValuesScanState *node（スキャン対象の VALUES ノードの状態構造体）
  • **出力**: なし（副作用として node の内部状態が更新される）

前提条件と不変条件

  • node は ExecInitValuesScan によって適切に初期化されている必要がある。
  • node->ss.ps.ps_ResultTupleSlot が存在する場合、その内容はクリアされる。

アルゴリズムまたは振る舞い

1. node->ss.ps.ps_ResultTupleSlot が割り当てられている場合、ExecClearTuple を呼び出してスロットを空にする。 2. ExecScanReScan(&node->ss) を呼び出し、共通のスキャン再スキャン処理を実行する。 3. node->curr_idx を -1 に設定し、次回の ValuesNext 呼び出しで最初の行から評価が開始されるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c 内の ExecReScan 経由で呼び出されることが確認されている。
  • **呼び出し先**:
  • ExecClearTuple
  • ExecScanReScan

関連データ構造

  • ValuesScanState: VALUES スキャンノードの実行時状態を保持する構造体。

失敗モードと境界条件

  • node が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • ps_ResultTupleSlot が NULL の場合、そのクリア処理はスキップされる。

保守担当者向けメモ

  • 本関数は VALUES 句の再スキャンを制御する。curr_idx を -1 にリセットすることで、ValuesNext が次回呼び出された際にインデックス 0 からの評価を強制する仕組みとなっている。
