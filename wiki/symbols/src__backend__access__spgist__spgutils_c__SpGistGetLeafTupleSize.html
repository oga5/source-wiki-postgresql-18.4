SpGistGetLeafTupleSize

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgutils.c:817

目的

SP-GiSTインデックスのリーフタプルを構築する際に必要なメモリサイズを計算します。

シグネチャまたは形状

Size SpGistGetLeafTupleSize(TupleDesc tupleDescriptor, const Datum *datums, const bool *isnulls)

入力と出力

  • **入力**:
  • tupleDescriptor: タプルの属性定義情報。
  • datums: 格納する値の配列。
  • isnulls: 各値がNULLかどうかを示すフラグ配列。
  • **出力**:
  • Size: アライメント調整済みのタプルに必要な合計バイト数。

前提条件と不変条件

  • natts == 1 の場合、互換性維持のためNULLビットマスクは使用されません。
  • natts > 1 の場合、いずれかの属性がNULLであればNULLビットマスクが必要と判断されます。
  • 計算されたサイズは、将来的にデッドタプル（SGDTSIZE）へ置換可能であることを保証するため、最小サイズが SGDTSIZE 以上になるよう調整されます。

アルゴリズムまたは振る舞い

1. **NULLマスク判定**: 属性数が1より大きい場合、isnulls 配列を走査し、NULLが存在すれば needs_null_mask を真とします。 2. **データサイズ計算**: heap_compute_data_size を用いて、属性値の合計サイズを算出します。 3. **合計サイズ算出**: ヘッダーサイズ（SGLTHDRSZ）とデータサイズを加算し、MAXALIGN でアライメント調整を行います。 4. **最小サイズ保証**: 計算結果が SGDTSIZE 未満の場合は、SGDTSIZE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgdoinsert.c (複数箇所)
  • **呼び出し先**:
  • heap_compute_data_size (PostgreSQLコアのヒープタプル計算関数)

関連データ構造

  • TupleDesc: タプルのスキーマ情報。
  • SGLTHDRSZ: リーフタプルのヘッダーサイズを計算するマクロ。
  • SGDTSIZE: デッドタプルの最小サイズを定義する定数。

失敗モードと境界条件

  • 属性数が1の場合、NULLの有無に関わらずビットマスクは省略されます。
  • 計算結果が極端に小さい場合でも、デッドタプルへの置換を考慮して SGDTSIZE が下限値として適用されます。

保守担当者向けメモ

  • 本関数はSP-GiSTの物理レイアウトに依存します。タプルヘッダーの構造や SGDTSIZE の定義が変更された場合は、本関数の計算ロジックも整合性を確認する必要があります。
