pairingheap_SpGistSearchItem_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgscan.c:40

目的

SP-GiSTのKNN検索（Order-by）において、優先度付きキュー（ペアリングヒープ）内の探索アイテムの順序を決定する比較関数です。

シグネチャまたは形状

static int pairingheap_SpGistSearchItem_cmp(const pairingheap_node *a, const pairingheap_node *b, void *arg)

入力と出力

  • **入力**:
  • a, b: 比較対象の pairingheap_node（実際には SpGistSearchItem 型）。
  • arg: スキャン状態を保持する SpGistScanOpaque 構造体。
  • **出力**:
  • a が b より優先される場合は 1、逆の場合は -1、等価の場合は 0 を返します。

前提条件と不変条件

  • arg は有効な SpGistScanOpaque 構造体へのポインタである必要があります。
  • sa->distances および sb->distances は、so->numberOfNonNullOrderBys に基づいて比較されます。

アルゴリズムまたは振る舞い

1. **NULL値の判定**: isNull フラグを持つアイテムを優先的に処理します。 2. **距離比較**: numberOfNonNullOrderBys に基づき、各距離を順に比較します。

  • NaN が含まれる場合、NaN を含むアイテムを優先します。
  • 距離が異なる場合、距離が小さい方を優先します。

    • 3. **ノード種別の優先順位**: 距離が等しい場合、深さ優先探索を維持するため、リーフノード（isLeaf）を内部ノードよりも優先します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgscan.c:163 にて、pairingheap_allocate の引数として渡されます。
  • **呼び出し先**: 比較ロジック内で isnan() マクロを使用します。

関連データ構造

  • SpGistSearchItem: 比較対象となる探索アイテム。
  • SpGistScanOpaque: スキャン状態および距離情報を保持する構造体。
  • pairingheap: 優先度付きキューを実装するデータ構造。

失敗モードと境界条件

  • numberOfNonNullOrderBys が 0 の場合、距離比較はスキップされ、ノード種別（リーフか否か）のみで順序が決定されます。
  • NaN が距離に含まれる場合、特定の順序付けルール（NaN を優先）に従って処理されます。

保守担当者向けメモ

  • 本関数は pairingheap の比較関数として登録されるため、シグネチャを変更する際は pairingheap_allocate の呼び出し箇所との整合性に注意してください。
  • KNN検索の順序付けロジックを変更する場合は、距離比較のループおよび isLeaf の優先順位付けの論理を確認してください。
