sqlname

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/include/sqlda-native.h:18

目的

ECPGの動的SQL実行において、カラム名やテーブル名などの識別子を保持するための構造体です。

シグネチャまたは形状

struct sqlname
{
    short       length;
    char        data[NAMEDATALEN];
};

入力と出力

  • **入力**: length（識別子の有効長）、data（識別子文字列本体）。
  • **出力**: sqlvar_struct の一部として、ECPGランタイムがデータベースから取得したメタデータ（カラム名等）を格納するために使用されます。

前提条件と不変条件

  • data 配列のサイズは NAMEDATALEN（64バイト）で固定されています。
  • length は data に格納された文字列の有効な長さを表します。

アルゴリズムまたは振る舞い

  • src/interfaces/ecpg/ecpglib/sqlda.c において、データベースから取得した識別子文字列の長さを length に設定し、data 領域へ strcpy を用いてコピーする処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/ecpglib/sqlda.c: メタデータの書き込みおよび読み取りに使用。
  • src/interfaces/ecpg/include/sqlda-native.h: struct sqlvar_struct のメンバとして定義。
  • src/interfaces/ecpg/test/expected/compat_informix-describe.c: テストコード内での参照。
  • **呼び出し先**: 特になし（データ構造定義のため）。

関連データ構造

  • struct sqlvar_struct: 本構造体をメンバとして保持し、カラムの属性を定義します。
  • NAMEDATALEN: 識別子の最大長を規定する定数。

失敗モードと境界条件

  • NAMEDATALEN を超える識別子を格納しようとした場合、バッファオーバーフローのリスクがあります。
  • データベース側のスキーマ定義と NAMEDATALEN が不一致の場合、メタデータの切り詰めが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はECPGネイティブな形式です。sqlda-compat.h で定義される互換用構造体（char *sqlname）とは型が異なるため、混同しないよう注意してください。
