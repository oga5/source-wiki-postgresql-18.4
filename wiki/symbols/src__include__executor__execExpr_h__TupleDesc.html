TupleDesc

定義

  • 種別: struct
  • 定義箇所: src/include/executor/execExpr.h:322

目的

タプル（行）の構造を記述するメタデータを提供します。実行エンジンにおいて、特定のタプルがどのような属性構成を持つかを識別するために使用されます。

シグネチャまたは形状

src/include/executor/execExpr.h 内の fetch 構造体の一部として、known_desc というフィールド名で保持されています。

入力と出力

  • 入力: タプル操作に関連するコンテキスト情報。
  • 出力: タプルの属性定義（型情報や列数など）。

前提条件と不変条件

  • fetch 構造体内で使用される場合、known_desc はタプルの構造が既知である場合にのみ有効な値となります。

アルゴリズムまたは振る舞い

実行エンジンがタプルから値を取得する際、そのタプルの構造を特定するために参照されます。特に fetch 構造体においては、last_var（取得対象の属性番号）や fixed（スロットの型が固定か否か）といった情報と組み合わせて、タプルアクセスの最適化や検証に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/amcheck/verify_heapam.c および contrib/dblink/dblink.c 等の外部モジュールで、タプル記述子の取得や検証のために参照されています。
  • 呼び出し先: 提供されたソース excerpt には、本シンボルが直接呼び出す関数や、本シンボルを呼び出す具体的な関数チェーンに関する情報は含まれていません。

関連データ構造

  • TupleTableSlotOps: タプルの種類を定義する構造体。
  • fetch 構造体: TupleDesc を保持する親構造体。

失敗モードと境界条件

  • known_desc が NULL の場合、タプルの構造が実行時に特定できない状態を示唆します。
  • fixed フラグが偽の場合、kind フィールドの信頼性が保証されないため、TupleDesc の利用には注意が必要です。
