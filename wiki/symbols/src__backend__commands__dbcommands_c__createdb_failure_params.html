createdb_failure_params

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/dbcommands.c:88

目的

データベース作成処理（CREATE DATABASE）が失敗した際に、クリーンアップ処理（createdb_failure_callback）へ渡すためのコンテキスト情報を保持する。

シグネチャまたは形状

typedef struct
{
	Oid			src_dboid;		/* テンプレートデータベースのOID */
	Oid			dest_dboid;		/* 作成しようとしていたデータベースのOID */
	CreateDBStrategy strategy;	/* データベース作成戦略 */
} createdb_failure_params;

入力と出力

本構造体は、createdb() 内でローカル変数としてインスタンス化され、エラー発生時のコールバック関数に引数として渡される。

前提条件と不変条件

  • src_dboid および dest_dboid は有効なデータベースOIDであること。
  • strategy は CreateDBStrategy 列挙型で定義されたいずれかの値であること。

アルゴリズムまたは振る舞い

データベース作成中にエラーが発生した場合、この構造体に格納された情報を用いて、部分的に作成されたディレクトリや共有バッファの破棄といったクリーンアップが実行される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/dbcommands.c 内の createdb() 関数。
  • 呼び出し先: エラー発生時に PG_ENSURE_ERROR_CLEANUP を通じて createdb_failure_callback が本構造体を参照する。

関連データ構造

  • CreateDBStrategy: データベース作成手法（WALログベースかファイルコピーベースか）を定義する列挙型。

失敗モードと境界条件

  • 構造体内のOIDが不正な場合、クリーンアップ処理が意図したディレクトリやリソースを正しく特定できない可能性がある。

保守担当者向けメモ

  • 本構造体は非トランザクション的なファイル操作を伴うデータベース作成処理の整合性を保つための重要なデータ構造である。変更を加える際は、createdb_failure_callback 側のロジックと整合性が取れているか確認すること。
