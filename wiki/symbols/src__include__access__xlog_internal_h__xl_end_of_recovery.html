xl_end_of_recovery

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlog_internal.h:300

目的

リカバリ処理の終了時に記録されるWALレコードのデータ構造を定義します。リカバリ完了時点のタイムライン情報やWALレベルの状態を保持するために使用されます。

シグネチャまたは形状

typedef struct xl_end_of_recovery
{
    TimestampTz end_time;
    TimeLineID  ThisTimeLineID;
    TimeLineID  PrevTimeLineID;
    int         wal_level;
} xl_end_of_recovery;

入力と出力

  • **入力**: リカバリ終了時のシステム状態（タイムラインID、タイムスタンプ、WALレベル）。
  • **出力**: WALレコードの一部としてディスクへ書き込まれるバイナリデータ。

前提条件と不変条件

  • ThisTimeLineID はリカバリ完了後の新しいタイムラインIDを指します。
  • PrevTimeLineID はフォーク元となった直前のタイムラインIDを指します。

アルゴリズムまたは振る舞い

リカバリ終了時にこの構造体に値を格納し、XLogRegisterData を介してWALレコードとして書き込まれます。リカバリ処理やWALサマライザがこのレコードを読み込む際は、XLogRecGetData を通じてメモリ上に復元されます。

呼び出し元と呼び出し先

  • **呼び出し元（書き込み側）**: src/backend/access/transam/xlog.c
  • **呼び出し先（読み込み側）**:
  • src/backend/access/rmgrdesc/xlogdesc.c (記述生成)
  • src/backend/access/transam/xlog.c (リカバリ処理)
  • src/backend/access/transam/xlogrecovery.c (リカバリ処理)
  • src/backend/postmaster/walsummarizer.c (WALサマライザ)

関連データ構造

  • TimeLineID: タイムライン識別子。
  • TimestampTz: タイムスタンプ型。

失敗モードと境界条件

  • 構造体のサイズが固定であるため、memcpy 等で読み込む際にレコードのデータ長が sizeof(xl_end_of_recovery) と一致しない場合、データ破損の可能性があります。

保守担当者向けメモ

  • 本構造体はWALの物理フォーマットの一部であるため、フィールドの追加や変更を行う際は、既存のWALとの後方互換性に注意してください。
  • xlogdesc.c 内の記述関数も併せて更新する必要があります。
