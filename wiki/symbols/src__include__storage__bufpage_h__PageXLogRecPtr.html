PageXLogRecPtr

定義

  • 種別: struct
  • 定義箇所: src/include/storage/bufpage.h:98

目的

WAL（Write-Ahead Log）上の位置を指し示すためのポインタ構造体です。主にページヘッダーにおいて、そのページが最後に変更されたログ位置（LSN）を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	uint32		xlogid;			/* 上位ビット */
	uint32		xrecoff;		/* 下位ビット */
} PageXLogRecPtr;

入力と出力

  • 入力: 64ビットのログ位置情報を上位32ビット（xlogid）と下位32ビット（xrecoff）に分割して保持します。
  • 出力: PageXLogRecPtrGet 関数（src/include/storage/bufpage.h:105）を介して、構造体から値を取得可能です。

前提条件と不変条件

  • xlogid と xrecoff の組み合わせにより、WALストリーム内の一意な位置を表現します。
  • ページヘッダー内の pd_lsn フィールドとして利用される際、そのページに対する最後の変更を特定する不変の参照点となります。

アルゴリズムまたは振る舞い

  • ページが更新されるたびに、対応するWALレコードのLSNがこの構造体に格納されます。
  • ページがディスクから読み込まれた際、この値はページがどのログ位置まで反映済みであるかを判断する基準となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/storage/bufpage.h 内の PageHeaderData 構造体で pd_lsn として利用されています。
  • 呼び出し先: src/include/access/gist.h において PageGistNSN 型の定義に使用されています。

関連データ構造

  • PageHeaderData: ページヘッダーのメタデータ構造体。PageXLogRecPtr 型の pd_lsn をメンバに持ちます。
  • PageXLogRecPtrGet: 本構造体の値を取得するためのインライン関数。

失敗モードと境界条件

  • ログ位置が64ビットの範囲を超えた場合、xlogid のオーバーフローが発生する可能性があります。
  • ページが初期化されていない場合や破損している場合、このポインタが指す値は無効なログ位置を示す可能性があります。

保守担当者向けメモ

  • 本構造体は access/xlogdefs.h で定義されるLSN関連の概念と密接に関連しています。変更を加える際は、WALのログシーケンス番号の管理ルールと整合性を保つ必要があります。
