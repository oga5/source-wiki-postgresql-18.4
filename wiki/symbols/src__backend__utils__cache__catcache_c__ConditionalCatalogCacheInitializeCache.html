ConditionalCatalogCacheInitializeCache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/catcache.c:1073

目的

指定されたカタログキャッシュ（CatCache）が未初期化状態（cc_tupdesc が NULL）である場合に、遅延初期化を実行する。

シグネチャまたは形状

static void ConditionalCatalogCacheInitializeCache(CatCache *cache)

入力と出力

  • **入力**: CatCache *cache - 初期化対象のカタログキャッシュ構造体へのポインタ。
  • **出力**: なし。必要に応じて cache->cc_tupdesc が設定される。

前提条件と不変条件

  • USE_ASSERT_CHECKING が有効な場合、TYPEOID または ATTNUM 以外のキャッシュについては、トランザクション状態であるか、あるいはリレーションの取得が可能である必要がある。
  • TYPEOID および ATTNUM キャッシュについては、トランザクション外での実行が許容される。

アルゴリズムまたは振る舞い

1. デバッグビルド時、キャッシュの ID に応じてリレーション取得の妥当性を検証する。 2. cache->cc_tupdesc が NULL であるかを確認する。 3. NULL であれば CatalogCacheInitializeCache(cache) を呼び出し、キャッシュの初期化を完了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/catcache.c 内の複数箇所（SearchCatCache 等のキャッシュアクセス関数から呼び出される）。
  • **呼び出し先**:
  • CatalogCacheInitializeCache
  • AssertCouldGetRelation (デバッグビルド時のみ)

関連データ構造

  • CatCache: キャッシュ管理構造体。cc_tupdesc フィールドを介して初期化状態を保持する。

失敗モードと境界条件

  • cc_tupdesc が既に初期化済みの場合、本関数は何も行わず即座に復帰する。
  • メモリ不足（OOM）が発生した場合、CatalogCacheInitializeCache 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_attribute_always_inline が指定されており、ホットパスでのオーバーヘッドを最小化するように設計されている。
  • TYPEOID や ATTNUM キャッシュはトランザクション外で初期化される可能性があるため、アサーションの条件分岐において特別な扱いを受けている。
