GinPageIsRecyclable

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginvacuum.c:800

目的

GINインデックスのページが、他のトランザクションから参照される可能性がなく、再利用可能であるかを判定します。

シグネチャまたは形状

bool GinPageIsRecyclable(Page page)

入力と出力

  • **入力**: Page page（判定対象のGINインデックスページ）
  • **出力**: bool（true の場合、ページは再利用可能。false の場合、再利用不可）

前提条件と不変条件

  • 入力された page は有効なGINインデックスページである必要があります。
  • ページが削除済み（GinPageIsDeleted）である場合、そのページに関連付けられた削除トランザクションID（delete_xid）が判定の基準となります。

アルゴリズムまたは振る舞い

1. **新規ページ判定**: PageIsNew(page) が真であれば、初期化されていないページとみなし true を返します。 2. **削除フラグ確認**: GinPageIsDeleted(page) が偽であれば、まだ使用中のページとみなし false を返します。 3. **削除XIDの確認**: ページから GinPageGetDeleteXid(page) を取得します。

  • delete_xid が無効な場合、即座に true を返します。

    • 4. **可視性チェック**: GlobalVisCheckRemovableXid(NULL, delete_xid) を呼び出します。この関数は、delete_xid を参照している可能性のあるアクティブなスキャンが他に存在しないかを判定し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginutil.c:325
  • src/backend/access/gin/ginvacuum.c:758
  • **呼び出し先**:
  • PageIsNew
  • GinPageIsDeleted
  • GinPageGetDeleteXid
  • TransactionIdIsValid
  • GlobalVisCheckRemovableXid

関連データ構造

  • Page: PostgreSQLのバッファページ構造。
  • TransactionId: トランザクション識別子。

失敗モードと境界条件

  • ページが破損している場合や、正しく初期化されていないバッファが渡された場合の挙動は未定義です。
  • GlobalVisCheckRemovableXid の判定結果に依存するため、システム全体のトランザクション状態に左右されます。

保守担当者向けメモ

  • 本関数は、ginDeletePage 等で削除されたページが、他のスキャンから安全に参照されなくなったことを確認するために使用されます。
  • GlobalVisCheckRemovableXid は、削除されたページがまだ他のスキャンから参照されていないかを慎重に判定するため、このロジックを変更する際は並行スキャンとの整合性に注意してください。
