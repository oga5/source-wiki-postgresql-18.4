report_extension_updates

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/version.c:196

目的

アップグレード後のデータベースにおいて、インストール済みの拡張機能のうち、バージョン更新が必要なものを検出し、ユーザーに対して更新用SQLスクリプトの生成と実行を促す。

シグネチャまたは形状

void report_extension_updates(ClusterInfo *cluster)

入力と出力

  • **入力**: ClusterInfo *cluster（対象となるクラスター情報）
  • **出力**: なし（副作用として update_extensions.sql ファイルの生成および標準出力へのメッセージ表示を行う）

前提条件と不変条件

  • UpgradeTask フレームワークが初期化可能であること。
  • データベースに対して pg_available_extensions ビューをクエリする権限が必要。
  • update_extensions.sql ファイルへの書き込み権限が必要。

アルゴリズムまたは振る舞い

1. upgrade_task_create() でタスクを生成する。 2. pg_available_extensions を参照し、installed_version が default_version と一致しない拡張機能を抽出するクエリを定義する。 3. upgrade_task_add_step() を介して、抽出結果を process_extension_updates 関数で処理するステップを登録する。 4. upgrade_task_run() を実行し、更新が必要な拡張機能があれば update_extensions.sql に SQL を書き出す。 5. 更新が必要な拡張機能が存在した場合、その旨をユーザーに通知し、生成されたファイルパスを表示する。存在しない場合は check_ok() を呼び出す。 6. 最後に upgrade_task_free() でタスクリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c:806
  • **呼び出し先**:
  • upgrade_task_create
  • upgrade_task_add_step
  • process_extension_updates (コールバックとして登録)
  • upgrade_task_run
  • upgrade_task_free
  • prep_status, report_status, pg_log, check_ok (ユーティリティ関数)

関連データ構造

  • ClusterInfo: アップグレード対象のクラスター情報。
  • UpgradeTask: 拡張機能更新のタスク管理用構造体。
  • UpgradeTaskReport: 更新対象のファイルパスやファイルポインタを保持する構造体。

失敗モードと境界条件

  • update_extensions.sql の作成に失敗した場合、ファイル操作に関連するエラーが発生する可能性がある。
  • 拡張機能の更新クエリ実行時にデータベース接続エラーが発生した場合、タスク実行が失敗する可能性がある。

保守担当者向けメモ

  • 本関数は UpgradeTask フレームワークに依存している。拡張機能の更新ロジックに変更を加える場合は、process_extension_updates の実装も併せて確認すること。
  • 生成される SQL ファイル名は update_extensions.sql に固定されている。
