UpgradeTaskSlot

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/task.c:102

目的

pg_upgrade におけるデータベースタスクの並列実行において、個別の接続状態と実行進捗を管理するためのステートマシン用スロット構造体です。

シグネチャまたは形状

typedef struct UpgradeTaskSlot
{
	UpgradeTaskSlotState state; /* スロットの状態 */
	int			db_idx;			/* 割り当てられたデータベースのインデックス */
	int			step_idx;		/* 現在実行中のタスクステップのインデックス */
	PGconn	   *conn;			/* 管理対象の libpq 接続 */
	bool		ready;			/* 処理準備完了フラグ */
	bool		select_mode;	/* select() の監視モード (true: 読み込み, false: 書き込み) */
	int			sock;			/* 接続ソケットのファイル記述子 */
} UpgradeTaskSlot;

入力と出力

  • **入力**: upgrade_task_run 内で pg_malloc0 により初期化され、process_slot 等の関数へ引数として渡されます。
  • **出力**: 状態遷移および libpq 接続の更新を通じて、非同期クエリ実行の結果を保持します。

前提条件と不変条件

  • libpq の非同期 API (PQconnectStart, PQsendQuery 等) を使用する前提で設計されています。
  • db_idx および step_idx は、タスク実行中に有効な範囲内にある必要があります。

アルゴリズムまたは振る舞い

  • process_slot 関数により、state に基づいて接続の確立からクエリの送信、結果の処理までを非ブロッキングで遷移させます。
  • wait_on_slots 関数は、本構造体の sock と select_mode を使用して select() を呼び出し、I/O 待機を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: upgrade_task_run (初期化), wait_on_slots (I/O 監視), process_slot (状態遷移)。
  • **呼び出し先**: start_conn, process_query_result 等のタスク処理関数。

関連データ構造

  • UpgradeTaskSlotState: スロットの現在の状態を定義する列挙型。
  • UpgradeTask: 実行すべきタスクステップの集合を保持する構造体。
  • PGconn: libpq の接続ハンドル。

失敗モードと境界条件

  • 接続失敗時やクエリ実行エラー時は、state の遷移を通じて適切にエラーハンドリングを行う必要があります。
  • sock が無効な場合、select() 呼び出し時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は task.c 内の並列実行フレームワークの中核です。状態遷移ロジックを変更する際は、process_slot 内のステートマシンとの整合性に注意してください。
  • select_mode は select() の引数（readfds か writefds か）を決定するために使用されます。
