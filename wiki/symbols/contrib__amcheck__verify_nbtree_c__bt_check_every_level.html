bt_check_every_level

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_nbtree.c:377

目的

B-Tree インデックスの構造的整合性を検証するため、ルートからリーフまで全レベルを走査し、各ページおよびレベル間の不変条件をチェックする。

シグネチャまたは形状

static void bt_check_every_level(Relation rel, Relation heaprel, bool heapkeyspace,
                                 bool readonly, bool heapallindexed, bool rootdescend,
                                 bool checkunique)

入力と出力

  • **入力**:
  • rel: 検証対象のインデックスリレーション。
  • heaprel: 対応するヒープリレーション（heapallindexed が有効な場合に使用）。
  • heapkeyspace, readonly, heapallindexed, rootdescend, checkunique: 検証の厳密さやモードを制御するフラグ。
  • **出力**: なし（検証失敗時はエラーをスロー）。

前提条件と不変条件

  • TransactionIdIsValid(RecentXmin) が真であること。
  • rootdescend が指定されている場合、readonly モードであること。
  • B-Tree バージョン 4 未満のインデックスでは rootdescend は使用不可。

アルゴリズムまたは振る舞い

1. **初期化**: BtreeCheckState を構築し、heapallindexed が有効な場合は Bloom フィルタを作成してスナップショットを登録する。 2. **メタページ取得**: インデックスのメタページから真のルート（btm_root）を特定する。 3. **レベル走査**: ルートから開始し、bt_check_level_from_leftmost を用いてレベルごとに左端から右リンクを辿りながら検証を行う。 4. **ヒープ整合性検証**: 全レベルの走査完了後、heapallindexed が有効であれば table_beginscan_strat を使用してヒープをスキャンし、インデックスとの不整合を検出する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_nbtree.c 内の bt_index_check および bt_index_parent_check から呼び出される。
  • **呼び出し先**: bt_check_level_from_leftmost（レベル走査）、bloom_create（フィルタ作成）、table_beginscan_strat（ヒープスキャン）等。

関連データ構造

  • BtreeCheckState: 検証中の状態、Bloom フィルタ、スナップショット等を保持。
  • BTMetaPageData: インデックスのメタデータ。
  • BtreeLevel: 現在走査中のレベル情報。

失敗モードと境界条件

  • **整合性エラー**: インデックス構造が破損している場合、ERRCODE_INDEX_CORRUPTED 等でエラーをスローする。
  • **スナップショット不整合**: IsolationUsesXactSnapshot() 環境下でインデックスの indcheckxmin とスナップショットが矛盾する場合、ERRCODE_T_R_SERIALIZATION_FAILURE をスローする。
  • **機能制限**: B-Tree バージョン 4 未満で rootdescend を指定すると ERRCODE_FEATURE_NOT_SUPPORTED をスローする。

保守担当者向けメモ

  • heapallindexed モードは大規模テーブルで高負荷となるため注意が必要。
  • readonly モードでは、より厳密な親子の整合性チェックが実行される。
  • btm_fastroot と btm_root の不一致は、インデックスの破損ではなく「スキニー」な B-Tree の特性として許容される。
