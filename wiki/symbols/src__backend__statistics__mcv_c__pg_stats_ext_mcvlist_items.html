pg_stats_ext_mcvlist_items

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mcv.c:1337

目的

多変量MCV（Most Common Values）リストのバイナリデータを展開し、SQLから可視化可能なレコードセットとして出力する。

シグネチャまたは形状

Datum pg_stats_ext_mcvlist_items(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型のMCVリストデータ（PG_GETARG_BYTEA_P(0)）。
  • **出力**: 以下のカラムを持つレコードのセット（Set Returning Function）。
  • index (int4): アイテムのインデックス。
  • values (text[]): 各次元の値の文字列表現。
  • nulls (bool[]): 各次元のNULLフラグ。
  • frequency (float8): 観測された頻度。
  • base_frequency (float8): 独立性を仮定した場合のベース頻度。

前提条件と不変条件

  • SRF_IS_FIRSTCALL() で MCVList 構造体をデシリアライズし、funcctx->user_fctx に保持することで呼び出し間での状態を維持する。
  • 戻り値の型が TYPEFUNC_COMPOSITE であることを要求する。
  • mcvlist->items の各次元に対して、型に応じた出力関数（getTypeOutputInfo）を動的に呼び出して文字列変換を行う。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時に statext_mcv_deserialize を実行し、MCVリストをメモリ上に展開する。 2. **反復処理**: funcctx->call_cntr を用いて、リスト内の全アイテム（nitems）を順次処理する。 3. **変換**: 各アイテムの次元ごとに、getTypeOutputInfo で取得した関数を用いて値を文字列化し、ArrayBuildState を介して text[] 型の配列を構築する。 4. **出力**: 構築した値を heap_form_tuple でタプル化し、SRF_RETURN_NEXT で返却する。全アイテム終了後は SRF_RETURN_DONE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（直接的なC言語レベルの呼び出し元はソース内に明示なし）。
  • **呼び出し先**: statext_mcv_deserialize, get_call_result_type, BlessTupleDesc, TupleDescGetAttInMetadata, getTypeOutputInfo, fmgr_info, FunctionCall1, cstring_to_text, accumArrayResult, makeArrayResult, heap_form_tuple。

関連データ構造

  • MCVList: 多変量統計情報を保持する構造体。
  • MCVItem: 個別の値の組み合わせと頻度を保持する構造体。
  • FuncCallContext: SRF（Set Returning Function）の実行状態を保持するコンテキスト。

失敗モードと境界条件

  • get_call_result_type が TYPEFUNC_COMPOSITE を返さない場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーを報告する。
  • 入力データが不正な形式である場合、statext_mcv_deserialize 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は SRF（Set Returning Function）の定型的な実装パターンに従っている。
  • values 配列の構築において、各次元の型に応じた出力関数を動的に解決しているため、新しいデータ型を追加する際は型情報の整合性に注意が必要。
