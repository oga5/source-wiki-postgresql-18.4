split_to_stringlist

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress.c:232

目的

入力文字列を特定の区切り文字で分割し、その結果を文字列リスト（_stringlist）に追加する。

シグネチャまたは形状

static void split_to_stringlist(const char *s, const char *delim, _stringlist **listhead)

入力と出力

  • **入力**:
  • s: 分割対象の文字列（const char *）。
  • delim: 区切り文字を指定する文字列（const char *）。
  • listhead: 文字列リストの先頭ポインタへのポインタ（_stringlist **）。
  • **出力**: なし（副作用として listhead が指すリストが更新される）。

前提条件と不変条件

  • s は pg_strdup によって複製されるため、呼び出し元で s を解放する必要はない。
  • listhead は add_stringlist_item が受け入れ可能な有効なポインタであること。

アルゴリズムまたは振る舞い

1. pg_strdup を使用して入力文字列 s のコピーを作成する。 2. strsep を使用して、指定された delim を境界として文字列を順次分割する。 3. 分割された各トークンを add_stringlist_item に渡し、リストに追加する。 4. 最後に、手順1で確保したコピー用メモリを free して解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/regress/pg_regress.c:2157（dblist の構築）
  • src/test/regress/pg_regress.c:2204（extraroles の構築）
  • **呼び出し先**:
  • pg_strdup
  • strsep
  • add_stringlist_item
  • free

関連データ構造

  • _stringlist: 文字列を保持するためのリスト構造体。

失敗モードと境界条件

  • s が NULL の場合、pg_strdup の挙動に依存する。
  • delim が空文字列の場合、strsep の仕様に従い文字列全体が単一のトークンとして扱われる可能性がある。

保守担当者向けメモ

  • strsep は破壊的な操作を行うため、入力文字列のコピーを作成してから処理する設計になっている。
  • 本関数は pg_regress におけるコマンドライン引数の解析（データベース名やロール名のリスト化）で利用されている。
