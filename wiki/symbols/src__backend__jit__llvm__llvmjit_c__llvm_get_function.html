llvm_get_function

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit.c:361

目的

JIT コンパイル済みの関数ポインタを、指定された LLVMJitContext 内のハンドルから検索・取得します。

シグネチャまたは形状

void *llvm_get_function(LLVMJitContext *context, const char *funcname);

入力と出力

  • **入力**:
  • context: JIT コンパイル状態を保持する LLVMJitContext 構造体へのポインタ。
  • funcname: 検索対象の関数名（非マングル名）。
  • **出力**:
  • 成功時: 解決された関数アドレス（void *）。
  • 失敗時: elog(ERROR) によりエラーを報告し、制御を戻さない。

前提条件と不変条件

  • llvm_assert_in_fatal_section() により、致命的なエラーセクション内での呼び出しであることを検証します。
  • context->compiled が偽の場合、内部で llvm_compile_module(context) が自動的に呼び出され、モジュールのコンパイルとエミットが行われます。

アルゴリズムまたは振る舞い

1. コンテキストが未コンパイル状態であれば、llvm_compile_module を実行してコードを生成します。 2. context->handles に格納された各 LLVMJitHandle に対して、LLVMOrcLLJITLookup を用いて指定された名前のシンボルを検索します。 3. 検索処理にかかった時間は、context->base.instr.emission_counter に累積されます（LLJIT は初回参照時にコードをエミットするため）。 4. シンボルが見つかった場合、そのアドレスを返します。 5. 全てのハンドルを探索してもシンボルが見つからない場合は、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit_expr.c:2996 (直接的な呼び出し)
  • **呼び出し先**:
  • llvm_compile_module (コンテキストが未コンパイルの場合)
  • LLVMOrcLLJITLookup (LLVM C API)
  • elog (エラー報告)

関連データ構造

  • LLVMJitContext: JIT コンパイルのライフサイクルとハンドルを管理する構造体。
  • LLVMJitHandle: ORC JIT の実行エンジンインスタンスを保持する構造体。

失敗モードと境界条件

  • 指定された funcname がどのハンドルにも存在しない場合、elog(ERROR) が発生します。
  • LLVMOrcLLJITLookup がエラーを返した場合、llvm_error_message を用いてエラー内容と共に elog(ERROR) が発生します。

保守担当者向けメモ

  • 本関数は ORC JIT のシンボルテーブルを直接参照するため、渡す関数名はマングルされていない名前である必要があります。
  • 実行時間の計測（emission_counter）は、LLJIT の遅延エミット特性を考慮して、検索処理とエミット処理を合算しています。
