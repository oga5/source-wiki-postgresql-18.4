ExecInitExprRec

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExpr.c:918

目的

解析・計画された式ツリー（Expr）を再帰的に走査し、実行時に評価可能な命令ステップ（ExprEvalStep）のリストを構築して ExprState に追加します。

シグネチャまたは形状

static void ExecInitExprRec(Expr *node, ExprState *state, Datum *resv, bool *resnull)

入力と出力

  • node: 処理対象の式ツリーノード。
  • state: 式評価の状態を保持する ExprState 構造体。
  • resv: 評価結果を格納する Datum へのポインタ。
  • resnull: 評価結果が NULL かどうかを示すフラグへのポインタ。

前提条件と不変条件

  • resv および resnull は NULL であってはならない。
  • 複雑な式によるスタックオーバーフローを防ぐため、内部で check_stack_depth() を呼び出す。
  • ExprEvalStep の出力先（resvalue, resnull）は、呼び出し元から指定されたアドレスに固定される。

アルゴリズムまたは振る舞い

1. ExprEvalStep の初期化を行い、呼び出し元から指定された結果格納先をセットする。 2. nodeTag(node) に基づき、ノードの種類に応じた処理を行う：

  • T_Var: 変数参照。システム列、ユーザー列、または行全体（ExecInitWholeRowVar）を判別し、適切な EEOP_* 命令コードを割り当てる。
  • T_Const: 定数。EEOP_CONST 命令を生成する。
  • T_Param: パラメータ。PARAM_EXEC または PARAM_EXTERN を判別し、必要に応じて paramCompile フックを利用する。
  • T_Aggref: 集約関数。EEOP_AGGREF 命令を生成し、親ノードが AggState であることを検証した上で aggstate->aggs リストに追加する。

    • 3. 生成されたステップを ExprEvalPushStep を介して ExprState の命令配列に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecInitExpr およびその関連関数（ExecInitExprList 等）から再帰的に呼び出される。
  • **呼び出し先**: ExecInitWholeRowVar, ExprEvalPushStep, check_stack_depth, elog。また、Param ノードの処理において params->paramCompile が定義されていればそれを呼び出す。

関連データ構造

  • ExprState: 式評価の実行状態と命令リストを保持。
  • ExprEvalStep: 実行時の命令単位。
  • Expr: 計画段階の式ツリーノード。
  • AggState: T_Aggref 処理時に参照される親ノードの状態。

失敗モードと境界条件

  • check_stack_depth() により、過度に深い式ツリーに対してエラーを発生させる。
  • 未知の paramkind が渡された場合、elog(ERROR) を送出する。
  • Aggref が AggState 以外のプランノードで見つかった場合、プランナーの不整合としてエラーを発生させる。

保守担当者向けメモ

  • 本関数は再帰的に動作するため、新しいノードタイプを追加する際はスタック消費量に注意すること。
  • ExprEvalStep の順序は execExprInterp.c のインタプリタと密接に関連しているため、命令コードの追加や変更時は整合性を確認すること。
  • T_Var の処理において、varreturningtype に応じた EEO_FLAG_HAS_OLD / HAS_NEW フラグの更新が適切に行われているか確認が必要。
