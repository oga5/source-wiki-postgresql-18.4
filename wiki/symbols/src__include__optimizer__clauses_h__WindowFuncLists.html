WindowFuncLists

定義

  • 種別: struct
  • 定義箇所: src/include/optimizer/clauses.h:19

目的

クエリ内のウィンドウ関数を収集・整理し、オプティマイザがウィンドウ処理を最適化するための情報を保持する構造体です。

シグネチャまたは形状

typedef struct {
    int         numWindowFuncs; /* 見つかったWindowFuncの総数 */
    Index       maxWinRef;      /* windowFuncs配列のインデックス範囲 (0 .. maxWinRef) */
    List      **windowFuncs;    /* 各winrefに対応するWindowFuncのリスト */
} WindowFuncLists;

入力と出力

  • 本構造体は、式ツリーを走査する過程でウィンドウ関数を抽出するために使用されます。
  • find_window_functions 関数によって初期化・構築され、ウィンドウ関数の参照（winref）ごとに分類されたリストが格納されます。

前提条件と不変条件

  • windowFuncs 配列は maxWinRef に基づいてインデックス付けされます。
  • numWindowFuncs は、走査中に発見された WindowFunc ノードの総数と一致する必要があります。

アルゴリズムまたは振る舞い

  • オプティマイザが式ツリーを走査する際、find_window_functions_walker が各ノードを探索し、ウィンドウ関数を発見するたびに本構造体へ追加します。
  • 収集されたリストは、後のプラン生成フェーズにおいて、どのウィンドウ関数がどのウィンドウ定義（winref）に属するかを特定するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c 内の optimize_window_clauses や select_active_windows など、プラン生成プロセスでウィンドウ関数の最適化を行う関数群から参照されます。
  • **呼び出し先**: src/backend/optimizer/util/clauses.c の find_window_functions_walker が本構造体を更新します。

関連データ構造

  • WindowFunc: 式ツリー内のウィンドウ関数ノード。
  • List: PostgreSQLの汎用リスト構造。windowFuncs はこのリストの配列として保持されます。
  • PlannerInfo: クエリ全体の計画情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保に失敗する可能性があります。
  • 構造体の初期化漏れ: find_window_functions を経由せずに使用した場合、未初期化のポインタによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は clauses.c の内部的な走査ロジックと密接に結合しています。構造体のフィールドを変更する場合は、find_window_functions_walker の実装も併せて更新してください。
  • maxWinRef の値が適切に管理されているか、インデックス範囲外アクセスが発生しないよう注意が必要です。
