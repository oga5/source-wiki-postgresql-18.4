pg_gen_salt

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:167

目的

指定されたアルゴリズム名に基づき、パスワードハッシュ生成用のソルト文字列を生成する SQL インターフェース関数。

シグネチャまたは形状

Datum pg_gen_salt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る text 型の引数（アルゴリズム名）。
  • **出力**: text 型の生成されたソルト文字列。

前提条件と不変条件

  • 入力引数は text 型であること。
  • buf は PX_MAX_SALT_LEN + 1 バイトの固定長バッファであり、これを超える長さのソルトは生成できない。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP(0) で入力アルゴリズム名を取得する。 2. text_to_cstring_buffer を使用して、入力値をローカルの buf にコピーする。 3. px_gen_salt(buf, buf, 0) を呼び出し、ソルトを生成する。第3引数の 0 はデフォルトの反復回数または設定を意味する。 4. px_gen_salt が負の値を返した場合、px_strerror を用いてエラーメッセージを構築し、ereport(ERROR, ...) で処理を中断する。 5. 成功時、cstring_to_text_with_len を用いて結果を text 型に変換し、PG_RETURN_TEXT_P で返却する。 6. PG_FREE_IF_COPY により、入力引数のメモリを適切に解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出される（直接的な C 言語レベルの呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**: px_gen_salt（内部暗号化ライブラリ）、px_strerror。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • PX_MAX_SALT_LEN: ソルトの最大長を規定する定数。

失敗モードと境界条件

  • px_gen_salt が負の値を返した場合、ERRCODE_INVALID_PARAMETER_VALUE エラーが発生する。
  • 入力されたアルゴリズム名がサポートされていない場合や、内部的な生成処理に失敗した場合にエラーとなる。

保守担当者向けメモ

  • 本関数は SQL 関数 gen_salt(text) の実装である。
  • px_gen_salt の第3引数に 0 を渡しているため、反復回数等の調整が必要な場合は pg_gen_salt_rounds を参照すること。
