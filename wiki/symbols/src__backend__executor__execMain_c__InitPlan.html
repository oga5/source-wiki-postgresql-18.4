InitPlan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execMain.c:835

目的

クエリ実行に必要なプランツリーの初期化、権限チェック、および実行環境（EState）の構築を行う。

シグネチャまたは形状

static void InitPlan(QueryDesc *queryDesc, int eflags)

入力と出力

  • **入力**:
  • queryDesc: 実行対象のクエリ情報（プランツリー、リレーション情報、実行状態 EState を含む）。
  • eflags: 実行フラグ（スキャン方法やリワインドの可否などを制御）。
  • **出力**: なし（queryDesc->estate 内の各フィールドを更新し、実行可能な状態にする）。

前提条件と不変条件

  • estate->es_subplanstates が空（NIL）であること。
  • ExecInitNode を呼び出す前に、サブプランの初期化が完了している必要がある。
  • ExecRowMark の初期化において、rti が es_range_table_size の範囲内であること。

アルゴリズムまたは振る舞い

1. **権限チェック**: ExecCheckPermissions を実行し、クエリ内のリレーションに対するアクセス権を検証する。 2. **実行状態の構築**: ExecInitRangeTable でレンジテーブルを初期化し、パーティションの初期プルーニング（ExecDoInitialPruning）を行う。 3. **行マークの処理**: plannedstmt->rowMarks が存在する場合、ExecRowMark 配列を構築し、各リレーションのロック状態やアクセス権を検証する。 4. **サブプランの初期化**: plannedstmt->subplans を走査し、各サブプランに対して ExecInitNode を呼び出して es_subplanstates に格納する。 5. **メインプランの初期化**: ExecInitNode をメインのプランツリーに対して呼び出し、planstate を生成する。 6. **ジャンクフィルタの準備**: SELECT クエリの場合、ターゲットリスト内のジャンク属性を処理するためのフィルタを準備する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecutorStart（src/backend/executor/execMain.c）から呼び出される。
  • **呼び出し先**: ExecCheckPermissions, ExecInitRangeTable, ExecDoInitialPruning, ExecGetRangeTableRelation, CheckValidRowMarkRel, ExecInitNode, ExecGetResultType 等。

関連データ構造

  • QueryDesc: クエリ実行の全体情報を保持。
  • EState: クエリ実行中の状態を保持する主要な構造体。
  • PlannedStmt: プランツリーやレンジテーブルを含む計画情報。
  • PlanState: プランノードごとの実行状態。
  • ExecRowMark: 行ロックや更新のためのマーク情報。

失敗モードと境界条件

  • markType が不明な場合、elog(ERROR) を発生させる。
  • ExecInitNode の再帰的な呼び出しにおいて、リソース確保やメモリ割り当てに失敗した場合はエラーとなる。
  • パーティションプルーニングにより除外されたリレーションに対する行マークは無視される。

保守担当者向けメモ

  • 本関数はクエリ実行の準備フェーズにおける中心的な役割を果たす。
  • ExecInitNode の呼び出し順序（サブプランが先、メインプランが後）は、依存関係上重要であるため変更しないこと。
  • ExecRowMark の構築ロジックは、プランノードの実行時動作に直結するため、変更時は nodeLockRows.c 等の関連ノードへの影響を考慮すること。
