FreeWorkerInfo

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/autovacuum.c:1605

目的

自動バキュームワーカープロセスの終了時に、共有メモリ上のワーカー管理情報を解放し、ランチャープロセスに対して再調整を通知する。

シグネチャまたは形状

static void FreeWorkerInfo(int code, Datum arg)

入力と出力

  • **入力**:
  • code: on_shmem_exit コールバック用の終了コード。
  • arg: on_shmem_exit コールバック用の引数。
  • **出力**: なし（副作用として共有メモリの状態を更新）。

前提条件と不変条件

  • MyWorkerInfo が有効なワーカー情報を指していること。
  • AutovacuumLock を排他モードで取得して共有メモリを操作する。

アルゴリズムまたは振る舞い

1. MyWorkerInfo が NULL でない場合、AutovacuumLock を排他取得する。 2. 現在のランチャーPIDをローカル変数に保存する。 3. MyWorkerInfo を AutoVacuumShmem の管理リストから削除し、各フィールド（wi_dboid, wi_tableoid 等）を初期化する。 4. 解放した MyWorkerInfo を av_freeWorkers リストの先頭にプッシュする。 5. MyWorkerInfo を NULL に設定し、自身の所有権を放棄する。 6. AutoVacuumShmem->av_signal[AutoVacRebalance] を true に設定し、残存ワーカーのコスト制限再計算をトリガーする。 7. AutovacuumLock を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: on_shmem_exit を介してプロセス終了時に呼び出される。
  • **呼び出し先**: LWLockAcquire, LWLockRelease, dlist_delete, dclist_push_head, pg_atomic_clear_flag 等の共有メモリ管理およびリスト操作関数。

関連データ構造

  • WorkerInfoData: ワーカーの状態を保持する構造体。
  • AutoVacuumShmemStruct: 自動バキューム全体の共有メモリ状態。

失敗モードと境界条件

  • MyWorkerInfo が NULL の場合は何も行わない。
  • ランチャーのPIDが取得と使用の間に変更されるリスクがあるが、プロセス終了後のPID再利用は稀であると想定して処理を続行する。

保守担当者向けメモ

  • 本関数は on_shmem_exit に登録されるため、ワーカープロセスが異常終了した場合でも確実に実行される。
  • AutovacuumLock の取得は必須であり、デッドロックを避けるため、本関数内では他のロック取得を行わないこと。
