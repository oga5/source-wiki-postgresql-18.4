ExecForeignScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeForeignscan.c:117

目的

外部テーブル（Foreign Table）に対するスキャン操作を実行し、取得したタプルを返却する。

シグネチャまたは形状

TupleTableSlot *ExecForeignScan(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（実行中の ForeignScanState ノードの状態）
  • **出力**: TupleTableSlot *（取得されたタプル、またはスキャン終了時の NULL）

前提条件と不変条件

  • pstate は ForeignScanState 型にキャスト可能であること。
  • EvalPlanQual（同時実行制御）がアクティブな場合、CMD_SELECT 以外の操作（直接更新など）はスキップされる。

アルゴリズムまたは振る舞い

1. pstate から ForeignScanState および ForeignScan 計画ノードを取得する。 2. EvalPlanQual がアクティブかつ操作が CMD_SELECT でない場合は、直ちに NULL を返す。 3. ExecScan を呼び出し、以下のメソッドを介してスキャンを実行する。

  • ForeignNext: 外部データソースから次のタプルを取得する。
  • ForeignRecheck: EvalPlanQual 実行時にタプルの再検証を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジンのメインループ（ExecProcNode 経由）。
  • **呼び出し先**:
  • ExecScan（共通ユーティリティ）
  • ForeignNext（FDWイテレータのラッパー）
  • ForeignRecheck（EPQ用再検証関数）

関連データ構造

  • ForeignScanState: 実行時のノード状態。
  • ForeignScan: 計画ノード。
  • EState: 実行状態（es_epq_active の確認に使用）。

失敗モードと境界条件

  • EvalPlanQual 実行中の非 SELECT 操作は、処理を行わず NULL を返す。
  • 外部データソースがタプルを使い果たした場合は、ExecScan を通じて NULL が返される。

保守担当者向けメモ

  • 本関数は ExecScan を呼び出すための薄いラッパーである。
  • 実際のデータ取得ロジックは FdwRoutine を通じて外部モジュールに委譲されている。
  • EvalPlanQual 処理中は直接更新（DirectModify）がスキップされる仕様に留意すること。
