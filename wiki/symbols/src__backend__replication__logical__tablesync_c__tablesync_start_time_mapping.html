tablesync_start_time_mapping

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/tablesync.c:420

目的

論理レプリケーションにおけるテーブル同期ワーカーの開始時刻を追跡・管理するためのデータ構造です。

シグネチャまたは形状

struct tablesync_start_time_mapping
{
    Oid         relid;            /* テーブルのOID */
    TimestampTz last_start_time;  /* 最後に同期を開始した時刻 */
};

入力と出力

本構造体はメモリ上のマッピングエントリとして利用されます。入力はテーブルの識別子（relid）とタイムスタンプであり、出力はそれらを保持する構造体インスタンスです。

前提条件と不変条件

  • relid は対象となるテーブルを一意に識別する OID である必要があります。
  • last_start_time は TimestampTz 型で保持され、同期プロセスの再試行制御等に利用されます。

アルゴリズムまたは振る舞い

本構造体は、同期ワーカーの起動履歴を管理するためのハッシュテーブル等のエントリとして使用されます。特定のテーブルに対する同期処理の開始時刻を記録することで、同期の頻度や再試行の判断に寄与します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/tablesync.c 内のハッシュテーブル初期化処理（ctl.entrysize の設定）およびエントリ操作処理（hentry へのポインタ代入）から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TablesyncWorkerMain: 同期ワーカーのメイン処理。
  • SyncingTablesState: 同期状態を管理する列挙型。

失敗モードと境界条件

  • ハッシュテーブルのメモリ確保失敗時や、無効な relid が指定された場合の挙動については、本構造体単体では定義されていません。

保守担当者向けメモ

  • 本構造体は tablesync.c 内の共有メモリまたはローカルメモリ上の管理用ハッシュテーブルで使用されています。
  • 構造体のサイズ変更やフィールド追加を行う際は、関連するハッシュテーブルの初期化ロジック（entrysize 指定箇所）との整合性に注意してください。
