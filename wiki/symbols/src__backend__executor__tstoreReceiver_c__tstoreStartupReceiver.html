tstoreStartupReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tstoreReceiver.c:55

目的

Tuplestore へのタプル格納処理を開始するための初期化を行い、タプルの属性や変換要件に基づいて最適な受信関数（receiveSlot）を選択・設定する。

シグネチャまたは形状

static void tstoreStartupReceiver(DestReceiver *self, int operation, TupleDesc typeinfo)

入力と出力

  • **入力**:
  • self: TStoreState 構造体へのポインタ（DestReceiver としてキャスト）。
  • operation: 実行操作の種類（未使用）。
  • typeinfo: 格納されるタプルの構造定義（TupleDesc）。
  • **出力**: なし（副作用として self の状態を更新）。

前提条件と不変条件

  • myState->cxt に有効なメモリコンテキストが割り当てられていること。
  • detoast（TOAST展開）と target_tupdesc（タプル変換）の同時使用はサポートされておらず、detoast が必要な場合は tupmap が NULL である必要がある（Assert で保護）。

アルゴリズムまたは振る舞い

1. **TOAST 展開の判定**: myState->detoast が有効な場合、typeinfo 内の属性を走査し、attlen == -1（可変長属性）が存在するかを確認する。 2. **タプル変換の準備**: myState->target_tupdesc が設定されている場合、convert_tuples_by_position を呼び出し、変換マップ（tupmap）を作成する。 3. **受信関数の選択**:

  • **TOAST 展開が必要な場合**: receiveSlot を tstoreReceiveSlot_detoast に設定し、変換用ワークスペース（outvalues, tofree）を確保する。
  • **変換が必要な場合**: receiveSlot を tstoreReceiveSlot_tupmap に設定し、変換用スロット（mapslot）を作成する。
  • **上記以外**: receiveSlot を tstoreReceiveSlot_notoast に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（DestReceiver インターフェース経由）。
  • **呼び出し先**:
  • convert_tuples_by_position
  • MemoryContextAlloc
  • MakeSingleTupleTableSlot
  • tstoreReceiveSlot_detoast / tstoreReceiveSlot_tupmap / tstoreReceiveSlot_notoast（関数ポインタの割り当て先）

関連データ構造

  • TStoreState: 状態管理用構造体。
  • DestReceiver: 実行エンジンの出力先インターフェース。
  • TupleDesc: タプルの属性定義。
  • TupleTableSlot: タプル保持用スロット。

失敗モードと境界条件

  • detoast と tupmap が同時に要求された場合、Assert によりデバッグビルドで停止する。
  • メモリ不足時には MemoryContextAlloc がエラーを発生させる。

保守担当者向けメモ

  • 新しい変換ロジックを追加する場合は、本関数の条件分岐を拡張し、対応する tstoreReceiveSlot_* 関数を実装すること。
  • detoast 処理は myState->cxt 上で行われるため、メモリリークを防ぐために tstoreShutdownReceiver での適切な解放処理と整合性を保つ必要がある。
