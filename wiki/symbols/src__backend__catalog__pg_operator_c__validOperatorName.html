validOperatorName

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_operator.c:67

目的

演算子名がPostgreSQLの構文規則およびSQL標準との互換性において妥当であるかを検証する。

シグネチャまたは形状

static bool validOperatorName(const char *name)

入力と出力

  • **入力**: name (検証対象の演算子名文字列)
  • **出力**: bool (妥当であれば true、不正であれば false)

前提条件と不変条件

  • name はヌル終端文字列であること。
  • 演算子名として使用可能な文字セットは ~!@#^&|?+-*/%<>=` に限定される。

アルゴリズムまたは振る舞い

1. **長さチェック**: 文字列が空、または NAMEDATALEN 以上であれば false を返す。 2. **文字セット検証**: strspn を使用し、許可された文字以外が含まれていれば false を返す。 3. **コメント禁止**: /* または -- を含む場合は false を返す。 4. **SQL標準互換性チェック**: 2文字以上の演算子で、末尾が + または - の場合、その直前に ~!@#^&|?% のいずれかが含まれていなければ false を返す（=- のような誤解釈を防ぐため）。 5. **予約語チェック**: != はパーサーが <> に変換するため、演算子名としては false` を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_operator.c 内の OperatorCreate (2箇所)
  • **呼び出し先**:
  • 標準Cライブラリ関数: strlen, strspn, strstr, strchr, strcmp

関連データ構造

  • なし

失敗モードと境界条件

  • name が NAMEDATALEN 以上の場合に失敗する。
  • != を指定した場合に失敗する。
  • 演算子名として許可されていない文字が含まれる場合に失敗する。
  • 演算子名の中にコメント開始シーケンスが含まれる場合に失敗する。

保守担当者向けメモ

  • 本関数は src/backend/parser/scan.l 内の op_chars 定義と同期させる必要がある。変更時は字句解析ルールとの整合性に注意すること。
