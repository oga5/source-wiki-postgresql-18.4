RelationBuildPartitionDesc

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partdesc.c:133

目的

カタログ（pg_inherits, pg_class）からパーティション情報を収集し、リレーションのパーティション記述子（PartitionDesc）を構築する。

シグネチャまたは形状

static PartitionDesc RelationBuildPartitionDesc(Relation rel, bool omit_detached)

入力と出力

  • **入力**:
  • rel: 対象となるパーティション化テーブルのリレーション。
  • omit_detached: デタッチ済みパーティションを除外するかどうかを指定するフラグ。
  • **出力**:
  • 構築された PartitionDesc 構造体。

前提条件と不変条件

  • PartitionBoundSpec がカタログから取得できない場合、並行 DDL（DETACH CONCURRENTLY）による不整合とみなし、一度だけ再試行を行う。
  • 構築される記述子は、CacheMemoryContext の子コンテキストで管理される必要がある。

アルゴリズムまたは振る舞い

1. find_inheritance_children_extended を呼び出し、対象テーブルの子パーティション OID リストを取得する。 2. 各パーティションについて、システムキャッシュ（RELOID）から relpartbound 属性を取得する。 3. キャッシュから取得できない場合、pg_class を直接スキャンして取得を試みる。 4. 取得に失敗した場合は AcceptInvalidationMessages を実行し、一度だけ処理を retry ラベルへ戻す。 5. 取得した PartitionBoundSpec を検証し、partition_bounds_create を用いて境界情報（PartitionBoundInfo）を生成する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/partitioning/partdesc.c 内の RelationGetPartitionDesc から呼び出される。
  • **呼び出し先**:
  • find_inheritance_children_extended
  • SearchSysCache1 / SysCacheGetAttr
  • systable_beginscan / systable_getnext
  • partition_bounds_create
  • AcceptInvalidationMessages

関連データ構造

  • PartitionDesc: パーティションのメタデータ構造体。
  • PartitionBoundSpec: カタログに格納されているパーティション境界の定義。
  • PartitionBoundInfo: 正規化された境界情報。

失敗モードと境界条件

  • **カタログ不整合**: DETACH CONCURRENTLY 等の並行操作により relpartbound が消失した場合、再試行を行う。再試行後も取得できない場合はエラー（elog(ERROR)）となる。
  • **カタログ破損**: is_default パーティションの OID が pg_partitioned_table の定義と一致しない場合、エラーとなる。

保守担当者向けメモ

  • 本関数は並行 DDL との競合を解決するための retry ロジックを含んでいる。このロジックは無限ループを避けるため、retried フラグにより最大1回に制限されている。
  • メモリ管理は呼び出し側のコンテキストに依存する部分があるため、メモリリークの発生には注意が必要である。
