GeneratePruningStepsContext

定義

  • 種別: struct
  • 定義箇所: src/backend/partitioning/partprune.c:111

目的

パーティションプルーニング（不要なパーティションの除外）のためのステップ生成プロセスにおいて、現在の解析状態、生成されたステップ、およびクエリ条件の特性を保持するために使用されるコンテキスト構造体です。

シグネチャまたは形状

typedef struct GeneratePruningStepsContext
{
    RelOptInfo *rel;            /* 対象のパーティション化リレーション */
    PartClauseTarget target;    /* ステップ生成のユースケース */
    List       *steps;          /* PartitionPruneStep のリスト */
    bool        has_mutable_op; /* 安定（stable）演算子の有無 */
    bool        has_mutable_arg;/* 実行時パラメータ以外の可変引数の有無 */
    bool        has_exec_param; /* PARAM_EXEC パラメータの有無 */
    bool        contradictory;  /* 条件の矛盾が証明されたか */
    int         next_step_id;   /* 次のステップ ID */
} GeneratePruningStepsContext;

入力と出力

本構造体は関数 gen_partprune_steps 等の引数として渡され、解析結果を蓄積します。

  • **入力**: RelOptInfo（対象リレーション）および PartClauseTarget（プルーニングの目的）。
  • **出力**: steps リストに格納された PartitionPruneStep オブジェクト群、およびクエリ条件の性質を示すフラグ群。

前提条件と不変条件

  • next_step_id はステップ生成時に一意の ID を割り当てるために使用されます。
  • steps リストは、gen_partprune_steps_internal 等の内部関数によって構築されます。

アルゴリズムまたは振る舞い

クエリの WHERE 句を走査し、パーティションキーとマッチする条件を抽出して PartitionPruneStep に変換します。この過程で、演算子の安定性やパラメータの有無をフラグとして記録し、最終的に実行時プルーニングが必要かどうかを判定するためのメタデータを構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: gen_partprune_steps およびその内部再帰関数（gen_partprune_steps_internal, gen_prune_step_op, gen_prune_step_combine, gen_prune_steps_from_opexps, match_clause_to_partition_key, get_steps_using_prefix, get_steps_using_prefix_recurse）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • RelOptInfo: プランニング対象のリレーション情報。
  • PartClauseTarget: プルーニングの対象（プランニング時か実行時か）。
  • PartitionPruneStep: 生成されるプルーニングの論理ステップ。

失敗モードと境界条件

  • contradictory フラグが true に設定された場合、条件式が矛盾しており、対象パーティションがゼロであることを示します。
  • has_exec_param が true の場合、プランニング段階では完全なプルーニングが不可能であり、実行時プルーニングが必要であることを示唆します。

保守担当者向けメモ

  • 本構造体は partprune.c 内の複数のステップ生成関数間で共有されるため、フラグの更新タイミングに注意してください。
  • next_step_id はステップ間の依存関係を構築する際に重要となるため、ステップ追加時には必ずインクリメントしてください。
