SortCoordinateData

定義

  • 種別: struct
  • 定義箇所: src/include/utils/tuplesort.h:46

目的

並列ソート処理において、リーダープロセスとワーカープロセス間で共有される状態情報を保持するために使用されます。

シグネチャまたは形状

typedef struct SortCoordinateData
{
    bool        isWorker;
    int         nParticipants;
    Sharedsort *sharedsort;
} SortCoordinateData;

入力と出力

  • **入力**: palloc0 等によりメモリ確保され、各フィールドが初期化されます。
  • **出力**: SortCoordinate 型（SortCoordinateData へのポインタ）として、ソート処理の調整用データ構造として利用されます。

前提条件と不変条件

  • isWorker が false の場合、そのプロセスはリーダーとして扱われます。
  • nParticipants はリーダープロセスによって設定され、起動された参加者数を示します（ワーカーの場合は -1 に設定されます）。
  • sharedsort は共有メモリ上の Sharedsort 構造体を指す不透明なポインタです。

アルゴリズムまたは振る舞い

並列ソート実行時、リーダーとワーカーは本構造体を介して共有メモリ上のソート状態（Sharedsort）にアクセスし、ソートの進行状況やマージ処理を同期します。リーダーは自身もワーカーとして参加する場合、そのための状態も本構造体を通じて管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の各モジュールで palloc0 を用いてインスタンス化されています。
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • **呼び出し先**: 共有メモリ管理（storage/dsm.h）およびソート処理本体（tuplesort.c）と連携します。

関連データ構造

  • Sharedsort: 共有メモリ上に配置されるソートの内部状態。
  • SortCoordinate: SortCoordinateData へのポインタ型。

失敗モードと境界条件

  • メモリ不足時に palloc0 が失敗する可能性があります。
  • 並列ソートのリーダーとワーカー間で nParticipants の整合性が取れていない場合、ソート処理が正しく完了しない可能性があります。

保守担当者向けメモ

  • 本構造体は並列ソートの制御に不可欠です。変更を加える際は、tuplesort.c における共有メモリのレイアウトおよびリーダー/ワーカー間の同期ロジックへの影響を考慮してください。
