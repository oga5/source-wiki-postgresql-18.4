hashvalidate

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashvalidate.c:39

目的

PostgreSQL のハッシュインデックスにおける演算子クラス（opclass）および演算子ファミリー（opfamily）の整合性を検証します。カタログ登録時に呼び出され、ハッシュインデックスの要件を満たさない定義を排除します。

シグネチャまたは形状

bool hashvalidate(Oid opclassoid)

入力と出力

  • **入力**: opclassoid (検証対象の演算子クラスの OID)
  • **出力**: 検証に合格した場合は true、不整合が検出された場合は false を返します。不整合時には INFO レベルのメッセージがログ出力されます。

前提条件と不変条件

  • pg_amproc および pg_amop システムカタログにアクセス可能であること。
  • 演算子ファミリー内のサポート関数は、左辺型と右辺型が一致している必要があります。
  • ハッシュインデックスは ORDER BY 演算子をサポートしません。

アルゴリズムまたは振る舞い

1. **メタデータ取得**: 指定された opclassoid から演算子ファミリー情報を取得します。 2. **サポート関数の検証**:

  • HASHSTANDARD_PROC (int4 戻り値) および HASHEXTENDED_PROC (int8 戻り値) のシグネチャをチェックし、ハッシュ可能な型リストを作成します。
  • HASHOPTIONS_PROC のシグネチャを検証します。

    • 3. **演算子の検証**:

  • 戦略番号が有効範囲内（1 ～ HTMaxStrategyNumber）であるか確認します。
  • ORDER BY 演算子が含まれていないか確認します。
  • 演算子の左右の型が、ハッシュ可能な型リストに含まれているか確認します。

    • 4. **グループ整合性チェック**: identify_opfamily_groups を使用し、演算子と関数のセットが不足していないか（特に等価演算子の欠落）を検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hash.c 内の amroutine->amvalidate フィールドを通じて呼び出されます。
  • **呼び出し先**: SearchSysCache1, SearchSysCacheList1, check_amproc_signature, check_amop_signature, identify_opfamily_groups 等のユーティリティ関数。

関連データ構造

  • Form_pg_opclass, Form_pg_amproc, Form_pg_amop: システムカタログの構造体。
  • OpFamilyOpFuncGroup: 演算子ファミリーのグループ化情報。

失敗モードと境界条件

  • **キャッシュ検索失敗**: opclassoid が無効な場合、elog(ERROR) で停止します。
  • **型不一致**: サポート関数の左右の型が異なる場合、検証失敗となります。
  • **シグネチャ不正**: サポート関数や演算子のシグネチャが期待される型と一致しない場合、検証失敗となります。
  • **演算子欠落**: 必要な等価演算子が存在しない場合、検証失敗となります。

保守担当者向けメモ

  • 本関数は amvalidate API を実装しており、ファミリー全体の整合性を検証します。
  • hashadjustmembers と連携して依存関係を管理しているため、変更時はカタログ定義との整合性に注意してください。
