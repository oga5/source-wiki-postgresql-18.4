worker_spi_launch

定義

  • 種別: function
  • 定義箇所: src/test/modules/worker_spi/worker_spi.c:391

目的

SQLから呼び出し可能なバックグラウンドワーカーを動的に起動し、そのプロセスIDを返却する。

シグネチャまたは形状

Datum worker_spi_launch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_INT32(0): ワーカー識別用整数。
  • PG_GETARG_OID(1): 接続先データベースのOID。
  • PG_GETARG_OID(2): 実行ロールのOID。
  • PG_GETARG_ARRAYTYPE_P(3): ワーカー起動フラグの配列（TEXT型）。
  • **出力**:
  • 成功時: 起動したワーカーのプロセスID (int32)。
  • 失敗時: NULL またはエラーによるトランザクション中断。

前提条件と不変条件

  • 引数のフラグ配列は1次元かつNULLを含まないこと。
  • フラグ配列の各要素は "ALLOWCONN" または "ROLELOGINCHECK" であること。
  • RegisterDynamicBackgroundWorker が成功すること。

アルゴリズムまたは振る舞い

1. BackgroundWorker 構造体を初期化し、ライブラリ名 worker_spi、関数名 worker_spi_main を設定。 2. 引数からフラグ配列を解析し、内部ビットフラグを構築。 3. データベースおよびロールのOIDが未指定の場合、GUC変数から取得。 4. 接続情報とフラグを bgw_extra 領域に格納。 5. RegisterDynamicBackgroundWorker でワーカーを登録。 6. WaitForBackgroundWorkerStartup を呼び出し、ワーカーの起動完了を待機。 7. 起動状態を確認し、成功した場合はプロセスIDを返却。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 により登録）。
  • **呼び出し先**:
  • RegisterDynamicBackgroundWorker
  • WaitForBackgroundWorkerStartup
  • get_database_oid, get_role_oid
  • deconstruct_array_builtin

関連データ構造

  • BackgroundWorker: ワーカーの構成情報を保持。
  • BackgroundWorkerHandle: ワーカーの制御ハンドル。

失敗モードと境界条件

  • **エラー**:
  • フラグ配列が1次元でない、またはNULLを含む場合。
  • 未定義のフラグ文字列が指定された場合。
  • WaitForBackgroundWorkerStartup が BGWH_STOPPED または BGWH_POSTMASTER_DIED を返した場合。
  • **境界条件**:
  • dboid や roleoid が無効な場合、GUC変数によるフォールバックが行われる。

保守担当者向けメモ

  • 本関数は worker_spi_main を動的に起動するエントリーポイントです。
  • bgw_extra へのデータ格納順序は worker_spi_main 側の読み出し順序と一致させる必要があります。
  • 起動待機処理 (WaitForBackgroundWorkerStartup) を含んでいるため、ワーカーの起動が完了するまでSQL実行がブロックされます。
