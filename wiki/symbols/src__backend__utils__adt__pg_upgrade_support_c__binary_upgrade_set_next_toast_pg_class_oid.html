binary_upgrade_set_next_toast_pg_class_oid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_upgrade_support.c:140

目的

pg_upgrade 実行中において、次に作成される TOAST テーブルの pg_class OID を強制的に指定するために使用されます。

シグネチャまたは形状

Datum binary_upgrade_set_next_toast_pg_class_oid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して、設定対象となる OID を受け取ります。
  • **出力**: PG_RETURN_VOID() により、値を返しません。

前提条件と不変条件

  • サーバーが binary_upgrade モードで起動されている必要があります（CHECK_IS_BINARY_UPGRADE マクロにより検証されます）。この条件を満たさない場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. 引数から OID を取得します。 2. CHECK_IS_BINARY_UPGRADE を実行し、アップグレードモードであることを確認します。 3. グローバル変数 binary_upgrade_next_toast_pg_class_oid に取得した OID を代入します。 4. 処理を終了し、void を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c:5751 において、SQL クエリ経由で呼び出されます。
  • **呼び出し先**: 本関数内から直接呼び出される関数はありません。

関連データ構造

  • binary_upgrade_next_toast_pg_class_oid: この関数が値を設定するグローバル変数です（src/include/catalog/binary_upgrade.h で宣言）。

失敗モードと境界条件

  • binary_upgrade モード以外で呼び出された場合、CHECK_IS_BINARY_UPGRADE マクロによって処理が中断されます。

保守担当者向けメモ

  • 本関数はシステムカタログの整合性を直接操作する強力なツールです。誤用はデータベースの破損を招くため、pg_upgrade 以外のコンテキストでの利用は想定されていません。
