free_device_authz

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq-oauth/oauth-curl.c:150

目的

device_authz 構造体が保持する動的に割り当てられた各文字列フィールドのメモリを解放する。

シグネチャまたは形状

static void free_device_authz(struct device_authz *authz);

入力と出力

  • 入力: struct device_authz *authz （解放対象の構造体へのポインタ）
  • 出力: なし（void）

前提条件と不変条件

  • 引数 authz は NULL であってはならない（free 呼び出し時に構造体メンバへのアクセスが発生するため）。
  • 構造体内の各文字列メンバは、malloc 等で動的に確保されているか、あるいは NULL である必要がある。

アルゴリズムまたは振る舞い

以下の順序で構造体メンバのメモリを解放する： 1. device_code 2. user_code 3. verification_uri 4. verification_uri_complete 5. expires_in_str 6. interval_str

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq-oauth/oauth-curl.c 内の async_ctx 破棄処理（actx->authz を引数として渡す箇所）から呼び出される。
  • 呼び出し先: 標準ライブラリの free()。

関連データ構造

  • device_authz: OAuth 2.0 Device Authorization レスポンスを保持する構造体。

失敗モードと境界条件

  • authz が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • 構造体メンバが初期化されていない（ゴミ値が入っている）場合、free() が不正なポインタを処理しようとしてクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は構造体自体のメモリは解放しない。構造体本体の解放は呼び出し側で行う必要がある。
  • 新たなフィールドが device_authz 構造体に追加された場合は、本関数にも対応する free() を追加すること。
