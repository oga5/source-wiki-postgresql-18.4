brin_desummarize_range

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin.c:1490

目的

指定されたブロック番号を含むBRINインデックスのページ範囲の要約を解除（desummarize）し、Revmapから該当範囲の要約情報を削除する。

シグネチャまたは形状

Datum brin_desummarize_range(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象となるBRINインデックスのOID。
  • PG_GETARG_INT64(1): 要約解除対象のブロック番号（heapBlk64）。
  • **出力**: なし（PG_RETURN_VOID()）。

前提条件と不変条件

  • リカバリ中ではないこと。
  • 指定されたOIDが有効なBRINインデックスであること。
  • 呼び出し元がインデックスの所有者であること。
  • インデックスが有効（indisvalid）であること。
  • デッドロック回避のため、テーブル（heapRel）を先にロックし、その後にインデックス（indexRel）をロックすること。

アルゴリズムまたは振る舞い

1. 引数の検証およびリカバリ状態の確認を行う。 2. 対象テーブルとインデックスを ShareUpdateExclusiveLock モードでオープンする。 3. インデックスがBRIN形式であること、および呼び出し元が所有者であることを確認する。 4. インデックスとテーブルの対応関係を再検証する。 5. インデックスが有効な場合、brinRevmapDesummarizeRange を呼び出し、完了するまでループ処理を行う。 6. 最後に両リレーションをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース内では明示的な呼び出し元は特定されていません。
  • **呼び出し先**:
  • brinRevmapDesummarizeRange (内部処理の実行)
  • IndexGetRelation, table_open, index_open, relation_close (リレーション管理)
  • object_ownercheck, aclcheck_error (権限チェック)

関連データ構造

  • Relation (インデックスおよびヒープリレーションの管理)
  • BRIN_AM_OID (BRINインデックスの識別子)

失敗モードと境界条件

  • **リカバリ中**: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE を送出。
  • **ブロック番号不正**: heapBlk64 が負または MaxBlockNumber を超える場合、ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE を送出。
  • **オブジェクト種別不一致**: 対象がBRINインデックスでない場合、ERRCODE_WRONG_OBJECT_TYPE を送出。
  • **権限不足**: 所有者でない場合、ACLCHECK_NOT_OWNER エラーを送出。
  • **インデックス無効**: indisvalid が偽の場合、DEBUG1 レベルでメッセージを出力し処理を終了。

保守担当者向けメモ

  • brin_summarize_range とは異なり、オートバキュームによる呼び出しは想定されていないため、ユーザーIDの切り替えは行われません。
  • brinRevmapDesummarizeRange が要約解除の核心部分を担います。
