pgp_s2k_fill

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-s2k.c:222

目的

新規暗号化処理において、OpenPGPのString-to-Key (S2K) パラメータ（ソルトや反復回数）を初期化します。

シグネチャまたは形状

int pgp_s2k_fill(PGP_S2K *s2k, int mode, int digest_algo, int count);

入力と出力

  • **入力**:
  • s2k: 初期化対象の PGP_S2K 構造体へのポインタ。
  • mode: 使用するS2Kモード（PGP_S2K_SIMPLE, PGP_S2K_SALTED, PGP_S2K_ISALTED）。
  • digest_algo: 使用するハッシュアルゴリズムのID。
  • count: 反復回数計算のためのベース値。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時はエラーコード（PXE_NO_RANDOM または PXE_PGP_BAD_S2K_MODE）を返します。

前提条件と不変条件

  • s2k ポインタは有効なメモリ領域を指している必要があります。
  • pg_strong_random がシステム上で利用可能である必要があります。

アルゴリズムまたは振る舞い

1. s2k->mode および s2k->digest_algo を引数の値で設定します。 2. mode に応じて以下の処理を行います：

  • PGP_S2K_SIMPLE: 特に追加処理なし。
  • PGP_S2K_SALTED: pg_strong_random を使用して s2k->salt を生成します。
  • PGP_S2K_ISALTED: s2k->salt を生成し、さらに乱数を用いて decide_s2k_iter を呼び出し、s2k->iter を決定します。

    • 3. 未定義のモードが指定された場合は PXE_PGP_BAD_S2K_MODE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-encrypt.c:569
  • **呼び出し先**:
  • pg_strong_random (乱数生成)
  • decide_s2k_iter (反復回数の決定)

関連データ構造

  • PGP_S2K: S2Kパラメータを保持する構造体（pgp.h で定義）。

失敗モードと境界条件

  • pg_strong_random が失敗した場合、PXE_NO_RANDOM を返します。
  • サポートされていない mode が渡された場合、PXE_PGP_BAD_S2K_MODE を返します。

保守担当者向けメモ

  • 本関数は暗号化の初期化フェーズでのみ使用されます。
  • decide_s2k_iter のロジック変更は、本関数の PGP_S2K_ISALTED モードの挙動に直接影響します。
