CatalogCacheCreateEntry

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/catcache.c:2143

目的

システムカタログキャッシュ（CatCache）に新しいエントリ（CatCTup）を作成し、ハッシュテーブルへ挿入する。

シグネチャまたは形状

static CatCTup *CatalogCacheCreateEntry(CatCache *cache, HeapTuple ntp, Datum *arguments, uint32 hashValue, Index hashIndex)

入力と出力

  • **入力**:
  • cache: 対象の CatCache 構造体。
  • ntp: キャッシュするタプル（負のキャッシュの場合は NULL）。
  • arguments: 負のキャッシュ作成時に使用するキーの配列。
  • hashValue: 計算済みのハッシュ値。
  • hashIndex: ハッシュテーブル内のバケットインデックス。
  • **出力**:
  • 作成された CatCTup 構造体へのポインタ。失敗時は NULL。

前提条件と不変条件

  • CacheMemoryContext が有効であること。
  • ntp が指定されている場合、そのタプルは CacheMemoryContext 内にコピーされる。
  • ntp に外部（TOAST）フィールドが含まれる場合、インライン化して展開される。
  • 参照カウント（refcount）は初期値 0 で作成される。

アルゴリズムまたは振る舞い

1. **TOAST展開**: ntp が存在し、外部フィールドを持つ場合、toast_flatten_tuple を用いてインライン化する。この際、catcache_in_progress_stack を使用して、処理中の無効化イベントを追跡する。 2. **メモリ確保**: CacheMemoryContext に切り替え、CatCTup 構造体とタプルデータを一括で確保する。 3. **データコピー**: タプル内容を確保した領域にコピーする。 4. **キー抽出**: キャッシュキーをタプルから抽出（または負のキャッシュ用にコピー）する。 5. **リスト登録**: dlist_push_head を使用して、指定されたバケットの先頭にエントリを挿入する。 6. **統計更新**: キャッシュ内のエントリ数（cc_ntup）をインクリメントする。 7. **リハッシュ**: キャッシュの負荷率（cc_ntup > cc_nbuckets * 2）が閾値を超えた場合、RehashCatCache を呼び出してバケットを拡張する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/catcache.c 内のキャッシュ検索およびミス処理関数群（SearchCatCacheMiss 等）。
  • **呼び出し先**: toast_flatten_tuple, heap_freetuple, MemoryContextSwitchTo, palloc, CatCacheCopyKeys, dlist_push_head, RehashCatCache。

関連データ構造

  • CatCache: キャッシュ管理構造体。
  • CatCTup: キャッシュされたタプルエントリ。
  • CatCInProgress: 処理中のキャッシュエントリを追跡するためのスタック構造。

失敗モードと境界条件

  • **デバッグビルド**: USE_ASSERT_CHECKING が有効な場合、テスト目的で約0.1%の確率で NULL を返す。
  • **無効化**: TOAST展開中に無効化イベントが発生し、in_progress_ent.dead が真となった場合、確保したメモリを解放して NULL を返す。

保守担当者向けメモ

  • 本関数は CacheMemoryContext を操作するため、呼び出し元でのメモリコンテキスト管理に注意すること。
  • CatCTup の refcount は初期値 0 であるため、呼び出し側で適切に管理（ピン留め）する必要がある。
