float8_pl

定義

  • 種別: function
  • 定義箇所: src/include/utils/float.h:157

目的

2つの float8 型の値を加算し、結果がオーバーフローした際にPostgreSQLのエラー報告メカニズムを介して処理を中断する。

シグネチャまたは形状

static inline float8 float8_pl(const float8 val1, const float8 val2)

入力と出力

  • **入力**: val1 (float8), val2 (float8)
  • **出力**: 加算結果 (float8)

前提条件と不変条件

  • 入力値が有限数である場合、加算結果が無限大（inf）になった場合はオーバーフローとみなす。
  • 入力値のいずれかが既に無限大である場合は、加算結果が無限大であってもエラーを発生させない。

アルゴリズムまたは振る舞い

1. val1 と val2 の加算を実行する。 2. 結果が isinf() であり、かつ入力値 val1 および val2 のいずれも無限大ではない場合、float_overflow_error() を呼び出して処理を中断する。 3. 上記条件に該当しない場合、計算結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c
  • src/backend/utils/adt/float.c
  • src/backend/utils/adt/geo_ops.c
  • **呼び出し先**:
  • isinf() (標準ライブラリ <math.h>)
  • float_overflow_error() (src/backend/utils/adt/float.c で実装)

関連データ構造

  • float8: PostgreSQLにおける倍精度浮動小数点型。

失敗モードと境界条件

  • **オーバーフロー**: 有限数同士の加算結果が無限大になる場合、float_overflow_error() によりエラーが発生する。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスを最適化している。
  • エラー報告は pg_noinline 属性を持つ関数（float_overflow_error）に分離されており、コードの肥大化を抑制している。
