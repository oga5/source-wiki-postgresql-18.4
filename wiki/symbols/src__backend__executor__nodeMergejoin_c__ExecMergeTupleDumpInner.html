ExecMergeTupleDumpInner

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMergejoin.c:552

目的

マージ結合処理において、現在保持している内部（inner）タプルの内容をデバッグ出力する。

シグネチャまたは形状

void ExecMergeTupleDumpInner(MergeJoinState *mergestate)

入力と出力

  • **入力**: MergeJoinState *mergestate - マージ結合ノードの実行状態を保持する構造体。
  • **出力**: なし（標準出力へのデバッグ情報の書き出しのみ）。

前提条件と不変条件

  • mergestate が有効なポインタであること。
  • mergestate->mj_InnerTupleSlot が初期化されていること。

アルゴリズムまたは振る舞い

1. mergestate から内部タプルのスロット (mj_InnerTupleSlot) を取得する。 2. "==== inner tuple ====" というヘッダーを出力する。 3. スロットが空（TupIsNull）であれば (nil) と出力する。 4. スロットにタプルが存在する場合、MJ_debugtup マクロを使用してタプルの詳細情報を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeMergejoin.c 内のデバッグトレース箇所。
  • **呼び出し先**: MJ_debugtup（マクロ）。

関連データ構造

  • MergeJoinState: マージ結合の実行状態を管理する構造体。
  • TupleTableSlot: タプルを保持するスロット構造体。

失敗モードと境界条件

  • mergestate が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • MJ_debugtup マクロが定義されていない環境や、デバッグ出力が無効なビルド設定では期待した出力が得られない可能性がある。

保守担当者向けメモ

  • 本関数はデバッグ専用のユーティリティである。本番環境のパフォーマンスには影響しないが、出力フォーマットを変更する場合は ExecMergeTupleDumpOuter や ExecMergeTupleDumpMarked との整合性に注意すること。
