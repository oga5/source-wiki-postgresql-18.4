WalSndState

定義

  • 種別: enum
  • 定義箇所: src/include/replication/walsender_private.h:24

目的

WAL Senderプロセスのライフサイクルにおける現在の実行状態を定義し、共有メモリ上でプロセスの進捗状況を追跡するために使用されます。

シグネチャまたは形状

typedef enum WalSndState
{
    WALSNDSTATE_STARTUP = 0,
    WALSNDSTATE_BACKUP,
    WALSNDSTATE_CATCHUP,
    WALSNDSTATE_STREAMING,
    WALSNDSTATE_STOPPING,
} WalSndState;

入力と出力

  • 入力: WalSndSetState() 関数を通じて状態遷移の引数として渡されます。
  • 出力: WalSndGetStateString() 関数により、現在の状態を表す文字列として出力されます。

前提条件と不変条件

  • 状態の更新は、WalSndSetState() を介して行われる必要があります。
  • 共有メモリ上の WalSnd 構造体に含まれる state フィールドとして保持されます。

アルゴリズムまたは振る舞い

  • WALSNDSTATE_STARTUP: プロセス開始直後の初期状態。
  • WALSNDSTATE_BACKUP: ベースバックアップ実行中の状態。
  • WALSNDSTATE_CATCHUP: スタンバイがプライマリに追いつくためのデータ転送中の状態。
  • WALSNDSTATE_STREAMING: 通常のWALストリーミング実行中の状態。
  • WALSNDSTATE_STOPPING: プロセス終了処理中の状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/walsender.c 内のプロセス管理ロジックが状態遷移をトリガーします。
  • 呼び出し先: src/backend/replication/syncrep.c 等の同期レプリケーション管理モジュールが、現在の状態を参照して待機キューの制御を行います。

関連データ構造

  • WalSnd: 各WAL Senderプロセスの状態を保持する共有メモリ構造体。
  • WalSndCtlData: 全WAL Senderを統括するグローバル構造体。

失敗モードと境界条件

  • 状態遷移の整合性は WalSndSetState() の実装に依存します。
  • 共有メモリ上のデータであるため、プロセス間での競合を防ぐためのスピンロック制御が前提となります。

保守担当者向けメモ

  • 本列挙型は walsender.c の内部状態を外部（syncrep.c 等）に公開する役割を持ちます。
  • 新しい状態を追加する場合は、WalSndGetStateString() への対応も必要です。
