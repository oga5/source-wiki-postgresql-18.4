InitSync

定義

  • 種別: function
  • 定義箇所: src/backend/storage/sync/sync.c:123

目的

ファイル同期（fsync）要求を管理するためのハッシュテーブルおよび削除待ちリストを初期化します。

シグネチャまたは形状

void InitSync(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 pendingOpsCxt、pendingOps、pendingUnlinks を初期化）

前提条件と不変条件

  • プロセスがスタンドアロン（!IsUnderPostmaster）であるか、チェックポインタプロセス（AmCheckpointerProcess()）である場合にのみ実行されます。
  • pendingOpsCxt はクリティカルセクション内でのメモリ確保が許可される設定で作成されます。

アルゴリズムまたは振る舞い

1. 条件を満たす場合、TopMemoryContext の子として pendingOpsCxt を作成します。 2. MemoryContextAllowInCriticalSection を呼び出し、メモリ不足時のパニックを許容しつつクリティカルセクション内での操作を可能にします。 3. FileTag をキー、PendingFsyncEntry をエントリとするハッシュテーブル pendingOps を作成します。 4. 削除待ちファイルリスト pendingUnlinks を NIL に初期化します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/init/postinit.c:642
  • 呼び出し先:
  • AllocSetContextCreate
  • MemoryContextAllowInCriticalSection
  • hash_create

関連データ構造

  • pendingOps: fsync 要求を管理するハッシュテーブル。
  • pendingUnlinks: 削除待ちのファイルリスト。
  • pendingOpsCxt: 上記データ構造を保持するメモリコンテキスト。

失敗モードと境界条件

  • メモリ不足が発生した場合、AllocSetContextCreate または hash_create の内部でエラーが発生します。
  • 特にチェックポインタが fsync 要求を吸収する際、メモリ不足が発生すると PANIC に至る可能性があります。

保守担当者向けメモ

  • 本関数はチェックポイント処理の基盤となるデータ構造を準備します。
  • ハッシュテーブルのサイズは 100 エントリで固定されています。
