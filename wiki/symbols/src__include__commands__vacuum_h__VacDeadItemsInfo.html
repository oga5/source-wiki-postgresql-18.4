VacDeadItemsInfo

定義

  • **種別**: struct
  • **定義箇所**: src/include/commands/vacuum.h:292

目的

削除対象（Dead）となったタプルのTID（Tuple ID）を管理する TidStore に関連する補助情報を保持します。

シグネチャまたは形状

typedef struct VacDeadItemsInfo
{
    size_t      max_bytes;      /* TidStoreが使用可能な最大バイト数 */
    int64       num_items;      /* 現在の登録エントリ数 */
} VacDeadItemsInfo;

入力と出力

本構造体はデータ保持用であり、関数ではありません。TidStore の容量制限（max_bytes）と現在の格納数（num_items）を管理します。

前提条件と不変条件

  • num_items は現在の TidStore 内の有効なエントリ数を反映している必要があります。
  • max_bytes は TidStore が消費するメモリ上限を規定します。

アルゴリズムまたは振る舞い

  • VACUUM 処理において、削除対象タプルの情報を収集・保持するために使用されます。
  • 並列VACUUM実行時には、ParallelVacuumState を介してワーカープロセス間で共有される状態の一部として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照・操作）**:
  • src/backend/access/heap/vacuumlazy.c: dead_items_info としてインスタンス化および管理。
  • src/backend/commands/vacuumparallel.c: 並列処理における共有状態の取得（parallel_vacuum_get_dead_items）および初期化。
  • src/backend/commands/vacuum.c: 引数として受け取り処理。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • TidStore: 本構造体が補助情報を保持する対象のデータ構造。
  • ParallelVacuumState: 並列VACUUM実行時に本構造体を保持する上位構造体。

失敗モードと境界条件

  • num_items が TidStore の許容範囲を超過した場合の挙動については、TidStore 側の実装に依存します。
  • メモリ割り当て失敗時（palloc 失敗時）に本構造体の生成が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は VACUUM のメモリ管理と並列処理の同期において重要な役割を果たします。
  • TidStore の仕様変更に伴い、max_bytes の解釈や num_items のカウントロジックに変更が必要となる場合があります。
