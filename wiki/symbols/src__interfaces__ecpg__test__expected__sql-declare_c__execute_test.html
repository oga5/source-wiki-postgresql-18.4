execute_test

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-declare.c:216

目的

ECPG（Embedded SQL for C）の DECLARE ステートメント、カーソル操作、およびプリペアドステートメントの動作を検証する回帰テストを実行する。

シグネチャまたは形状

void execute_test(void)

入力と出力

  • 入力: なし（グローバルなデータベース接続および静的変数を使用）
  • 出力: なし（副作用として標準エラー出力へのログ出力およびテスト結果の検証を行う）

前提条件と不変条件

  • 実行前に main 関数等でデータベース接続（con1 等）が確立されている必要がある。
  • 各 SQL 操作後に sqlca.sqlcode を確認し、エラー発生時には sqlprint() を呼び出す不変条件を持つ。

アルゴリズムまたは振る舞い

以下のテストケースを順次実行する： 1. **ケース 1**: AT 句なしでの PREPARE および DECLARE CURSOR の実行と、カーソルを用いたデータ取得。 2. **ケース 2**: con1 接続を指定した PREPARE および DECLARE CURSOR の実行と、カーソルを用いたデータ取得。 3. **ケース 3**: AT 句なしでの PREPARE および EXECUTE ステートメントによる直接実行。 各ステップで reset() を呼び出し、カーソルのクローズとステートメントの解放（ECPGdeallocate）を行う。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/sql-declare.c 内の main 関数（推定）。
  • 呼び出し先: ECPGprepare, ECPGdo, ECPGdeallocate, reset, sqlprint, printResult。

関連データ構造

  • sqlca: エラー状態を保持する構造体。
  • f1, f2, f3: 取得したデータを格納する配列。

失敗モードと境界条件

  • sqlca.sqlcode < 0: SQL 実行エラーが発生した場合、sqlprint() によりエラー詳細が出力される。
  • ECPG_NOT_FOUND: FETCH 操作時にデータがなくなった場合、ループを終了する。

保守担当者向けメモ

  • 本関数はプリプロセッサが生成した C コードであり、#line 指示子により declare.pgc と対応している。
  • 接続先（con1 等）の指定有無による挙動の差異を検証しているため、接続管理の変更時には注意が必要。
