pg_tzset

定義

  • 種別: function
  • 定義箇所: src/timezone/pgtz.c:233

目的

指定されたタイムゾーン名に対応する pg_tz 構造体をロードし、キャッシュ経由で返却する。

シグネチャまたは形状

pg_tz *pg_tzset(const char *tzname)

入力と出力

  • **入力**: tzname (タイムゾーン名を示す文字列)
  • **出力**: 成功時は pg_tz 構造体へのポインタ、失敗時は NULL を返す。

前提条件と不変条件

  • tzname の長さは TZ_STRLEN_MAX 以下であること。
  • 内部の timezone_cache ハッシュテーブルが初期化されていること（未初期化の場合は自動的に init_timezone_hashtable が呼び出される）。
  • タイムゾーン名は内部で大文字に変換され、大文字小文字を区別しない検索が行われる。

アルゴリズムまたは振る舞い

1. **検証**: tzname の長さを確認し、超過していれば NULL を返す。 2. **キャッシュ検索**: tzname を大文字に変換し、timezone_cache を検索する。ヒットした場合は即座に pg_tz を返す。 3. **ロード**:

  • GMT の場合: tzparse を使用して初期化する。
  • それ以外の場合: tzload を試行し、失敗した場合は tzparse による POSIX 形式の解析を試みる。

    • 4. **キャッシュ登録**: ロードしたタイムゾーン情報を timezone_cache に格納し、正規化された名前と状態を保存して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/variable.c
  • src/backend/utils/adt/datetime.c
  • src/backend/utils/adt/timestamp.c
  • contrib/postgres_fdw/connection.c
  • src/bin/initdb/findtimezone.c
  • **呼び出し先**:
  • init_timezone_hashtable
  • hash_search
  • tzparse (localtime.c)
  • tzload (localtime.c)

関連データ構造

  • pg_tz: タイムゾーン情報を保持する構造体。
  • pg_tz_cache: キャッシュ用のハッシュテーブルエントリ。
  • struct state: タイムゾーンの内部状態（localtime.c 由来）。

失敗モードと境界条件

  • tzname が TZ_STRLEN_MAX を超える場合、NULL を返す。
  • タイムゾーン名が不正で tzload および tzparse の両方に失敗した場合、NULL を返す。
  • GMT の初期化に失敗した場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • タイムゾーン名はハッシュテーブル検索のために常に大文字に正規化される。
  • tzparse は POSIX 形式のタイムゾーン指定にも対応している。
  • キャッシュは hash_search を使用しており、メモリ管理は PostgreSQL のメモリコンテキストに依存する。
