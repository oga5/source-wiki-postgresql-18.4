timerCA

定義

  • 種別: struct
  • 定義箇所: src/backend/port/win32/timer.c:23

目的

Windows環境において、メインスレッドとタイマー管理スレッド間でタイマー設定および同期用オブジェクトを共有するための構造体です。

シグネチャまたは形状

typedef struct timerCA
{
	struct itimerval value;
	HANDLE		event;
	CRITICAL_SECTION crit_sec;
} timerCA;

入力と出力

本構造体はデータ保持用であり、関数ではないため入出力はありません。各メンバは以下の役割を持ちます。

  • value: タイマーの期限設定（itimerval 構造体）。
  • event: タイマー管理スレッドを起床させるためのWin32イベントハンドル。
  • crit_sec: スレッド間のデータ競合を防ぐためのクリティカルセクション。

前提条件と不変条件

  • timerCommArea（本構造体の静的インスタンス）へのアクセスは、必ず crit_sec によって保護される必要があります。
  • event は CreateEvent によって適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • setitimer が呼び出されると、本構造体の value が更新され、event に対して SetEvent が発行されます。
  • pg_timer_thread は、この構造体の event を WaitForSingleObjectEx で監視し、タイムアウトや設定変更を検知します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/port/win32/timer.c 内の setitimer および pg_timer_thread が本構造体を参照・更新します。
  • 呼び出し先: 構造体自体は関数ではないため、呼び出し先はありません。

関連データ構造

  • itimerval: タイマーのインターバルと期限を保持するために使用されます。
  • timerCommArea: src/backend/port/win32/timer.c 内で定義される timerCA 型の静的インスタンス。

失敗モードと境界条件

  • crit_sec の初期化失敗や、event の生成失敗が発生した場合、タイマー機能は正常に動作しません。
  • 本構造体は ITIMER_REAL のみに対応しており、インターバルタイマー（it_interval）の機能はサポートされていません。

保守担当者向けメモ

  • 本構造体はWindows固有の移植性レイヤーの一部です。スレッド同期のロジックを変更する際は、CRITICAL_SECTION のデッドロックに十分注意してください。
  • value メンバの更新時には、必ず crit_sec を取得して排他制御を行ってください。
