BlockRefTableKey

定義

  • 種別: struct
  • 定義箇所: src/common/blkreftable.c:47

目的

ブロック参照テーブルにおいて、特定の変更済みブロックを追跡するためのハッシュテーブルのキーとして使用される構造体です。リレーションフォークを一意に識別します。

シグネチャまたは形状

typedef struct BlockRefTableKey
{
	RelFileLocator rlocator;
	ForkNumber	forknum;
} BlockRefTableKey;

入力と出力

  • 入力: RelFileLocator（リレーションの識別子）および ForkNumber（フォーク番号）。
  • 出力: なし（データ構造体）。

前提条件と不変条件

  • BlockRefTableKey を初期化する際は、パディング領域を含めてゼロクリアすることが推奨されます（{0} による初期化）。これは memcmp を用いた比較やハッシュ計算の整合性を保つためです。

アルゴリズムまたは振る舞い

  • 本構造体は simplehash.h を用いたハッシュテーブルのキーとして利用されます。
  • ハッシュ計算には hash_bytes が使用され、比較には memcmp が使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/blkreftable.c 内の複数の関数（BlockRefTableMarkBlockModified 等）で、テーブル操作のためのキーとしてインスタンス化されます。
  • 呼び出し先: 直接的な関数呼び出しはありません。

関連データ構造

  • RelFileLocator: リレーションを識別する構造体。
  • ForkNumber: フォークを識別する列挙型。
  • BlockRefTable: 本構造体をキーとして保持するハッシュテーブル。

失敗モードと境界条件

  • 構造体内のパディングに不定値が含まれる場合、ハッシュ値の不一致や比較エラーが発生する可能性があります。必ずゼロ初期化を行ってください。

保守担当者向けメモ

  • 本構造体は simplehash の SH_KEY_TYPE として定義されています。構造体を変更する場合は、ハッシュ関数および比較ロジック（SH_EQUAL）との整合性に注意してください。
