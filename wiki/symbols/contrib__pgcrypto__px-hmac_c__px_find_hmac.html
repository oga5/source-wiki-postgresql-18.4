px_find_hmac

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px-hmac.c:141

目的

指定されたハッシュアルゴリズム名に基づき、HMAC 計算を行うための PX_HMAC コンテキストを初期化・構築します。

シグネチャまたは形状

int px_find_hmac(const char *name, PX_HMAC **res);

入力と出力

  • **入力**:
  • name: 使用するハッシュアルゴリズム名（文字列）。
  • res: 初期化された PX_HMAC 構造体へのポインタを格納する出力先。
  • **出力**:
  • 成功時は 0 を返し、res に構築済みコンテキストを格納します。
  • 失敗時はエラーコード（PXE_HASH_UNUSABLE_FOR_HMAC 等）を返します。

前提条件と不変条件

  • 指定された name に対応するダイジェストアルゴリズムが px_find_digest で取得可能であること。
  • 取得したダイジェストアルゴリズムのブロックサイズが 2 以上であること。
  • 呼び出し側は、返された PX_HMAC 構造体を最終的に解放する責任を負います。

アルゴリズムまたは振る舞い

1. px_find_digest を呼び出し、指定されたハッシュアルゴリズムの PX_MD コンテキストを取得します。 2. ダイジェストのブロックサイズを確認し、2 未満であればエラーを返してダイジェストコンテキストを解放します。 3. PX_HMAC 構造体および内部の ipad（Inner Pad）、opad（Outer Pad）用バッファをメモリ確保します。 4. PX_HMAC 構造体の各メソッドポインタ（init, update, finish, reset, free 等）を、px-hmac.c 内の対応する静的関数に割り当てます。 5. 構築した構造体を res に格納して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgcrypto.c:139 (find_provider 経由)
  • **呼び出し先**:
  • px_find_digest (ダイジェストコンテキスト取得)
  • px_md_block_size (ブロックサイズ確認)
  • palloc (メモリ確保)
  • px_md_free (エラー時のメモリ解放)

関連データ構造

  • PX_HMAC: HMAC 操作の状態を保持する構造体。
  • PX_MD: 基盤となるメッセージダイジェストコンテキスト。

失敗モードと境界条件

  • px_find_digest が失敗した場合、そのエラーコードをそのまま返します。
  • ブロックサイズが 2 未満の場合、PXE_HASH_UNUSABLE_FOR_HMAC を返します。
  • メモリ不足時は palloc によりエラーが発生します。

保守担当者向けメモ

  • 本関数は RFC 2104 に準拠した HMAC 実装の入り口です。
  • 割り当てられた PX_HMAC 構造体は、内部で palloc を使用しているため、必ず対応する解放処理が必要です。
  • ipad および opad のサイズは、基盤となるダイジェストアルゴリズムのブロックサイズに依存します。
