ProcessCopyOptions

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/copy.c:535

目的

COPY コマンドのオプションリスト（DefElem のリスト）を解析し、CopyFormatOptions 構造体に設定値を反映させる。

シグネチャまたは形状

void ProcessCopyOptions(ParseState *pstate, CopyFormatOptions *opts_out, bool is_from, List *options);

入力と出力

  • **入力**:
  • pstate: パーサー状態（エラー報告に使用）。
  • opts_out: 設定を格納する CopyFormatOptions 構造体へのポインタ。NULL の場合は内部でメモリ確保される。
  • is_from: COPY FROM（読み込み）か COPY TO（書き出し）かを指定するフラグ。
  • options: DefElem のリスト。
  • **出力**: なし（opts_out を直接更新）。

前提条件と不変条件

  • opts_out が NULL の場合、関数内で palloc0 を使用してメモリが確保される。
  • 各オプションは一度のみ指定可能であり、重複した場合は errorConflictingDefElem が呼び出される。
  • is_from フラグは、header オプション等の処理において読み込みと書き出しの挙動を切り替えるために使用される。

アルゴリズムまたは振る舞い

1. opts_out の初期化（file_encoding を -1 に設定）。 2. options リストを走査し、各 DefElem の defname に基づいて以下の処理を行う：

  • format: text, csv, binary のいずれかを判定し、opts_out に反映。
  • freeze, delimiter, null, default, header, quote, escape: 各種値を構造体に格納。
  • force_quote, force_not_null, force_null, convert_selectively: 列名リストまたは A_Star を解析し、構造体に格納。
  • encoding: 文字コード名をエンコーディング ID に変換。
  • on_error, log_verbosity: 関連するヘルパー関数を呼び出し、値を設定。

    • 3. 不正な値や不明なオプションが指定された場合、ereport(ERROR, ...) により処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c:1574
  • src/backend/commands/copyto.c:703
  • contrib/file_fdw/file_fdw.c:341
  • **呼び出し先**:
  • defGetCopyHeaderChoice, defGetCopyOnErrorChoice, defGetCopyLogVerbosityChoice 等のヘルパー関数。
  • defGetString, defGetBoolean, defGetCopyRejectLimitOption（ソースコード末尾の断片より推測）。

関連データ構造

  • CopyFormatOptions: COPY コマンドの全設定を保持する構造体。
  • DefElem: SQL 文のオプション定義を保持するノード。
  • ParseState: パーサーのコンテキスト情報。

失敗モードと境界条件

  • **重複オプション**: 同じオプションが複数回指定された場合、エラーとなる。
  • **不正な値**: サポートされていないフォーマット名や、無効なエンコーディング名が指定された場合、エラーとなる。
  • **引数の不整合**: force_quote 等でリスト以外の引数が渡された場合、エラーとなる。

保守担当者向けメモ

  • 本関数は COPY コマンドのセットアップにおける厳格な整合性チェックを担う。オプションを追加・変更する際は、既存の errorConflictingDefElem による重複チェックと整合するように実装すること。
  • is_from フラグの扱いは、読み込みと書き出しで挙動が異なるオプション（header 等）があるため、変更時には注意が必要。
  • convert_selectively は SQL から直接アクセスできない内部オプションである点に留意すること。
