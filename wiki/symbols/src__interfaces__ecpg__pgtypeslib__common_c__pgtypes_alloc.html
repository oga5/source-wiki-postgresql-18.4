pgtypes_alloc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/common.c:9

目的

calloc をラップし、メモリ確保失敗時に errno を ENOMEM に設定する安全なメモリ確保インターフェースを提供します。

シグネチャまたは形状

char *pgtypes_alloc(long size)

入力と出力

  • **入力**: size (確保するメモリのバイト数)
  • **出力**: 確保されたゼロ初期化済みメモリ領域へのポインタ。確保に失敗した場合は NULL を返します。

前提条件と不変条件

  • 確保されたメモリはゼロ初期化されます。
  • メモリ確保失敗時には、呼び出し元が errno を参照することでエラーを検知できることが期待されます。

アルゴリズムまたは振る舞い

1. calloc(1L, size) を呼び出し、指定されたサイズのメモリを確保します。 2. 確保されたポインタが NULL である場合、errno に ENOMEM を代入します。 3. 確保したポインタ（または NULL）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/pgtypeslib/common.c
  • src/interfaces/ecpg/pgtypeslib/datetime.c
  • src/interfaces/ecpg/pgtypeslib/dt_common.c
  • **呼び出し先**:
  • calloc (標準ライブラリ関数)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • メモリ不足により calloc が失敗した場合、errno が ENOMEM に設定され、NULL が返されます。

保守担当者向けメモ

  • 本関数は pgtypeslib 内のメモリ確保の標準的な手段です。
  • 確保したメモリを解放する際は、Windows環境との互換性を考慮し、必ず PGTYPESchar_free を使用してください。
