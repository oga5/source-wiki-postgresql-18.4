TimestampDifferenceMilliseconds

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/timestamp.c:1760

目的

2つのタイムスタンプ（TimestampTz）間の差分を計算し、ミリ秒単位の long 型として返します。

シグネチャまたは形状

long TimestampDifferenceMilliseconds(TimestampTz start_time, TimestampTz stop_time)

入力と出力

  • **入力**:
  • start_time: 開始時刻（TimestampTz）
  • stop_time: 終了時刻（TimestampTz）
  • **出力**:
  • 差分（ミリ秒単位の long 型）。範囲は 0 から INT_MAX まで。

前提条件と不変条件

  • TimestampTz はマイクロ秒単位の int64 値として扱われます。
  • 戻り値は常に非負の値となります。

アルゴリズムまたは振る舞い

1. **初期チェック**: start_time >= stop_time の場合、経過時間はゼロまたは負であるとみなし、即座に 0 を返します。 2. **オーバーフロー検出**: stop_time - start_time の計算においてオーバーフローが発生する場合、INT_MAX を返します。 3. **範囲チェック**: 計算された差分が INT_MAX ミリ秒を超える場合、INT_MAX を返します。 4. **変換**: 上記以外の場合、マイクロ秒単位の差分を (diff + 999) / 1000 によりミリ秒単位に切り上げて返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c
  • contrib/postgres_fdw/connection.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/backup/basebackup_copy.c
  • src/backend/commands/analyze.c
  • **呼び出し先**:
  • pg_sub_s64_overflow（オーバーフロー検出用）

関連データ構造

  • TimestampTz: int64 型（マイクロ秒単位のタイムスタンプ）。

失敗モードと境界条件

  • start_time が stop_time より大きい場合、負の差分ではなく 0 が返されます。
  • 計算結果が long 型の表現範囲や INT_MAX を超える場合、上限値である INT_MAX にクランプされます。

保守担当者向けメモ

  • 本関数はミリ秒単位の精度を保証するため、計算時に切り上げ処理を行っています。
  • タイムスタンプの無限大（infinity）を扱う際、オーバーフローチェックが正しく機能するように設計されています。変更を加える際は pg_sub_s64_overflow の使用を維持してください。
