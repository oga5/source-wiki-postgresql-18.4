gbt_int8_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int8.c:127

目的

GiSTインデックス内の int8 型キーをフェッチし、インデックスエントリから元の値を復元します。

シグネチャまたは形状

Datum gbt_int8_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、インデックスから取得された値（int8）を返します。

前提条件と不変条件

  • tinfo（gbtree_ninfo 型の静的変数）が適切に初期化されている必要があります。
  • 入力される GISTENTRY は、int8 型のGiSTインデックスエントリとして有効である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. 共通関数 gbt_num_fetch を呼び出し、entry と tinfo を渡して値を抽出します。 3. 抽出された結果を PG_RETURN_POINTER を介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックス管理機構（直接的な呼び出し元コードはソース抜粋内には存在しません）。
  • **呼び出し先**: gbt_num_fetch（btree_utils_num.c で定義）。

関連データ構造

  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。
  • tinfo: int8 型のメタデータ（比較関数や距離関数を含む）を保持する gbtree_ninfo 構造体。

失敗モードと境界条件

  • entry が NULL の場合、または不正な形式である場合の挙動は gbt_num_fetch の実装に依存します。

保守担当者向けメモ

  • 本関数は int8 型専用のラッパーであり、ロジックの大部分は btree_utils_num.c の gbt_num_fetch に委譲されています。
  • PG_FUNCTION_INFO_V1(gbt_int8_fetch) マクロにより、PostgreSQLの関数マネージャに登録されています。
