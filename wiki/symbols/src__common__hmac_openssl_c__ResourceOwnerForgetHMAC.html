ResourceOwnerForgetHMAC

定義

  • 種別: function
  • 定義箇所: src/common/hmac_openssl.c:90

目的

バックエンド環境において、ResourceOwner が管理するリソースリストから特定の HMAC コンテキスト（pg_hmac_ctx）の追跡を解除する。

シグネチャまたは形状

static inline void ResourceOwnerForgetHMAC(ResourceOwner owner, pg_hmac_ctx *ctx)

入力と出力

  • owner: リソースを所有する ResourceOwner オブジェクト。
  • ctx: 追跡を解除する対象の pg_hmac_ctx 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • ResourceOwnerRememberHMAC によって、対象の ctx が既に当該 owner に登録されている必要がある。
  • 本関数は src/common/hmac_openssl.c 内で定義されており、主にバックエンド環境でのメモリリーク防止を目的としている。

アルゴリズムまたは振る舞い

1. PointerGetDatum(ctx) を用いて pg_hmac_ctx ポインタを Datum 型に変換する。 2. ResourceOwnerForget 関数を呼び出し、hmac_resowner_desc を参照してリソースリストから該当エントリを削除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/hmac_openssl.c 内の pg_hmac_free 等（コンテキスト解放時）。
  • 呼び出し先: ResourceOwnerForget（utils/resowner.h で定義されるコア関数）。

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する構造体。
  • ResourceOwner: リソースのライフサイクルを管理する PostgreSQL のバックエンド構造体。
  • hmac_resowner_desc: リソース所有者管理のための記述子。

失敗モードと境界条件

  • owner が無効な場合や、ctx が当該 owner に登録されていない場合に ResourceOwnerForget がどのように振る舞うかは、utils/resowner.h の実装に依存する。

保守担当者向けメモ

  • 本関数は static inline で定義されている。
  • バックエンド環境でのみ有効なリソース管理機構の一部であるため、フロントエンド（postgres_fe.h を使用する環境）では呼び出されない設計となっている。
