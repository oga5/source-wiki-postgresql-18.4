GetTableAmRoutine

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableamapi.c:27

目的

指定されたテーブルアクセス手法（TAM）のハンドラ関数を呼び出し、返された TableAmRoutine 構造体の取得と、必須コールバックの実装有無の検証を行う。

シグネチャまたは形状

const TableAmRoutine *GetTableAmRoutine(Oid amhandler)

入力と出力

  • **入力**: amhandler (Oid) - TAMハンドラ関数のOID。
  • **出力**: const TableAmRoutine * - 検証済みのTAMルーチン構造体へのポインタ。

前提条件と不変条件

  • amhandler は OidFunctionCall0 を介して実行可能であること。
  • 返される構造体は TableAmRoutine 型であり、かつ IsA マクロによる型チェックを通過すること。
  • TableAmRoutine 構造体内のすべての必須関数ポインタ（スキャン、挿入、更新、削除、バキューム、統計情報取得等）が NULL でないこと。

アルゴリズムまたは振る舞い

1. OidFunctionCall0(amhandler) を呼び出し、TAMハンドラから Datum を取得する。 2. 取得した Datum を TableAmRoutine 構造体へキャストする。 3. 構造体が NULL であるか、型が正しくない場合は ERROR を発生させる。 4. 構造体内の各関数ポインタに対して Assert を実行し、必須の実装が欠けていないかを確認する。 5. 検証を通過した構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/relcache.c: リレーションキャッシュ構築時に relation->rd_amhandler を引数として呼び出される。
  • **呼び出し先**:
  • OidFunctionCall0: ハンドラ関数の実行。
  • DatumGetPointer: Datum から構造体ポインタへの変換。
  • IsA: 型チェック。
  • elog: エラー報告。

関連データ構造

  • TableAmRoutine: TAMの機能を提供する関数ポインタ群を保持する構造体（access/tableam.h で定義）。

失敗モードと境界条件

  • ハンドラ関数が NULL を返した場合、または TableAmRoutine 型ではない場合、elog(ERROR) により処理が中断される。
  • 必須コールバックのいずれかが NULL の場合、Assert によりデバッグビルド環境でアサーション失敗が発生する。

保守担当者向けメモ

  • 本関数内の Assert は、新しいTAM実装を追加する際の必須インターフェースのチェックリストとして機能する。
  • 将来的に新しいコールバックが追加された場合、本関数に Assert を追加することで、既存のTAM実装の更新漏れを検知できる。
