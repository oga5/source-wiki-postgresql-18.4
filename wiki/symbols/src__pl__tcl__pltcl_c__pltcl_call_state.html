pltcl_call_state

定義

  • 種別: struct
  • 定義箇所: src/pl/tcl/pltcl.c:216

目的

PostgreSQL の関数、トリガー、またはイベントトリガーが PL/Tcl を介して実行される際の、現在の呼び出しコンテキストを保持するために使用されます。

シグネチャまたは形状

typedef struct pltcl_call_state
{
    FunctionCallInfo fcinfo;
    TriggerData *trigdata;
    pltcl_proc_desc *prodesc;
    TupleDesc   ret_tupdesc;
    AttInMetadata *attinmeta;
    ReturnSetInfo *rsi;
    Tuplestorestate *tuple_store;
    MemoryContext tuple_store_cxt;
    ResourceOwner tuple_store_owner;
} pltcl_call_state;

入力と出力

本構造体は関数引数や実行状態を格納するコンテナであり、直接的な入力・出力関数はありません。各フィールドは実行中の関数呼び出しに関連する PostgreSQL の内部データ構造（FunctionCallInfo, TriggerData 等）を参照します。

前提条件と不変条件

  • fcinfo は通常の関数呼び出し時に設定されます（トリガーの場合は NULL）。
  • trigdata は通常のトリガー実行時に設定されます（イベントトリガーや関数呼び出し時は NULL）。
  • ret_tupdesc および attinmeta は、関数がタプルを返す場合（fn_retistuple または fn_retisset）に初期化されます。

アルゴリズムまたは振る舞い

PL/Tcl の実行ハンドラ（pltcl_func_handler 等）において、呼び出しごとに本構造体のインスタンスが生成・管理されます。特に集合を返す関数（SRF）の実行時には、tuple_store を介して結果セットの蓄積と管理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: pltcl_func_handler, pltcl_trigger_handler, pltcl_event_trigger_handler 等のハンドラ関数内でインスタンス化されます。
  • **呼び出し先**: pltcl_init_tuple_store 等の内部関数へ引数として渡され、状態の初期化や更新に使用されます。

関連データ構造

  • pltcl_proc_desc: 実行対象の関数定義情報。
  • FunctionCallInfo: PostgreSQL の関数呼び出し情報。
  • TriggerData: トリガー実行時のコンテキスト。
  • Tuplestorestate: SRF の結果セットを保持するストレージ。

失敗モードと境界条件

  • tuple_store 関連のメモリ管理において、tuple_store_cxt や tuple_store_owner が適切に設定されていない場合、メモリリークやリソース解放の不整合が発生する可能性があります。
  • prodesc が NULL のまま実行が進むと、関数定義の特定に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は実行中の関数呼び出しのライフサイクルに紐付いています。PG_TRY/CATCH ブロック内でのクリーンアップ時には、tuple_store 関連のリソースが正しく解放されることを確認してください。
  • pltcl_current_call_state という静的変数で現在の呼び出し状態が追跡されています。再帰的な呼び出しやネストされた実行時の挙動については、このグローバルな状態管理に注意が必要です。
