pair_encode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_ops.c:254

目的

2つの浮動小数点数（float8）をカンマ区切りの文字列形式に変換し、指定された StringInfo バッファへ追加します。

シグネチャまたは形状

static void pair_encode(float8 x, float8 y, StringInfo str)

入力と出力

  • **入力**:
  • x, y: 変換対象の float8 値。
  • str: 出力先となる StringInfo 構造体へのポインタ。
  • **出力**: なし（戻り値は void）。str が指すバッファに "x,y" 形式の文字列が追記されます。

前提条件と不変条件

  • str は appendStringInfo が呼び出し可能な有効な StringInfo オブジェクトである必要があります。
  • 内部で float8out_internal を使用して文字列化を行うため、メモリ割り当てが発生します。

アルゴリズムまたは振る舞い

1. float8out_internal を呼び出し、x と y をそれぞれ文字列に変換します。 2. appendStringInfo を使用し、"x,y" の形式で str に書き込みます。 3. 変換に使用した一時的な文字列バッファを pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/geo_ops.c:364
  • src/backend/utils/adt/geo_ops.c:4690
  • **呼び出し先**:
  • float8out_internal
  • appendStringInfo
  • pfree

関連データ構造

  • float8: PostgreSQLの8バイト浮動小数点型。
  • StringInfo: PostgreSQLの動的文字列バッファ構造体。

失敗モードと境界条件

  • メモリ不足が発生した場合、pfree や appendStringInfo 内部のメモリ管理関数によりエラーが報告される可能性があります。
  • float8 の値が無限大や NaN の場合、float8out_internal の挙動に依存します。

保守担当者向けメモ

  • 本関数は static であり、主に幾何データ型の出力処理における座標ペアのシリアライズに使用されます。
  • float8out_internal が割り当てたメモリを確実に解放するため、pfree が適切に呼び出されていることを維持してください。
