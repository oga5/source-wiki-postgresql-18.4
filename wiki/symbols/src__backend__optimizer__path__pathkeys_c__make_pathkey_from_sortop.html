make_pathkey_from_sortop

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/pathkeys.c:255

目的

ソート演算子（ordering_op）に基づいて、正規化された PathKey を生成または取得します。

シグネチャまたは形状

PathKey *make_pathkey_from_sortop(PlannerInfo *root,
                                  Expr *expr,
                                  Oid ordering_op,
                                  bool reverse_sort,
                                  bool nulls_first,
                                  Index sortref,
                                  bool create_it)

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報。
  • expr: ソート対象の式。
  • ordering_op: ソートに使用する演算子の OID。
  • reverse_sort: 降順ソートかどうか。
  • nulls_first: NULL を先頭に配置するかどうか。
  • sortref: ソート参照インデックス。
  • create_it: 既存のパスキーが見つからない場合に新規作成するかどうか。
  • **出力**:
  • 成功時: 正規化された PathKey 構造体へのポインタ。
  • 失敗時: 演算子が有効なソート演算子でない場合、elog(ERROR) により中断。

前提条件と不変条件

  • 指定された ordering_op は、システムカタログ pg_amop に存在する有効なソート演算子である必要があります。
  • 返される PathKey は、make_canonical_pathkey の規約に従い正規化されていることが保証されます。

アルゴリズムまたは振る舞い

1. get_ordering_op_properties を呼び出し、指定された ordering_op に関連付けられた演算子族（opfamily）、入力型（opcintype）、および比較型（cmptype）を取得します。 2. exprCollation を使用して、式から照合順序（collation）を抽出します。 3. 取得した情報を引数として make_pathkey_from_sortinfo を呼び出し、正規化された PathKey を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/pathkeys.c 内のコードから参照されています。
  • **呼び出し先**:
  • get_ordering_op_properties
  • exprCollation
  • make_pathkey_from_sortinfo

関連データ構造

  • PathKey: 本関数が生成する主要な構造体。
  • PlannerInfo: プランナのグローバル状態。
  • Expr: ソート対象の式。

失敗モードと境界条件

  • get_ordering_op_properties が失敗した場合（無効な演算子 OID が渡された場合）、エラーが発生します。

保守担当者向けメモ

  • 本関数は make_pathkey_from_sortinfo のラッパーとして機能し、演算子 OID から必要な属性を解決する役割を担います。
  • SortGroupClause が照合順序情報を持たないため、式自体から照合順序を抽出する処理が必須となっています。
