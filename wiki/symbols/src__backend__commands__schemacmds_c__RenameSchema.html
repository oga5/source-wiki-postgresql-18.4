RenameSchema

定義

  • 種別: function
  • 定義箇所: src/backend/commands/schemacmds.c:248

目的

データベース内の既存スキーマの名前を変更する。

シグネチャまたは形状

ObjectAddress RenameSchema(const char *oldname, const char *newname)

入力と出力

  • **入力**:
  • oldname: 変更対象の既存スキーマ名。
  • newname: 新しいスキーマ名。
  • **出力**:
  • ObjectAddress: 変更されたスキーマのオブジェクトアドレス（NamespaceRelationId とスキーマの Oid を保持）。

前提条件と不変条件

  • pg_namespace システムカタログに対する RowExclusiveLock が保持される。
  • 呼び出し元は、対象スキーマの所有者である必要がある。
  • 呼び出し元は、現在のデータベースに対する CREATE 権限を持つ必要がある。
  • newname が既に存在する場合、エラーとなる。
  • allowSystemTableMods が無効な場合、pg_ で始まる名前への変更は禁止される。

アルゴリズムまたは振る舞い

1. NamespaceRelationId をロックし、NAMESPACENAME システムキャッシュから oldname に該当するタプルを取得する。存在しない場合はエラーを報告する。 2. newname が既存でないことを確認する。 3. object_ownercheck により所有者権限を、object_aclcheck によりデータベースの CREATE 権限を検証する。 4. システム名保護ルール（IsReservedName）に基づき、newname の妥当性をチェックする。 5. namestrcpy を使用してタプル内の名前を更新し、CatalogTupleUpdate でシステムカタログを書き換える。 6. InvokeObjectPostAlterHook を呼び出し、イベントトリガーやキャッシュ無効化の通知を行う。 7. ObjectAddressSet で戻り値を設定し、リソースを解放して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/alter.c:388（ALTER SCHEMA ... RENAME TO ... 文の処理）。
  • **呼び出し先**:
  • table_open, table_close
  • SearchSysCacheCopy1, CatalogTupleUpdate
  • object_ownercheck, object_aclcheck
  • IsReservedName
  • InvokeObjectPostAlterHook
  • ObjectAddressSet
  • heap_freetuple

関連データ構造

  • ObjectAddress: 操作対象の識別に使用。
  • Form_pg_namespace: pg_namespace カタログの構造体。
  • NamespaceRelationId: pg_namespace システムカタログの OID。

失敗モードと境界条件

  • ERRCODE_UNDEFINED_SCHEMA: oldname が存在しない場合。
  • ERRCODE_DUPLICATE_SCHEMA: newname が既に存在する場合。
  • ACLCHECK_NOT_OWNER: 所有者権限がない場合。
  • ERRCODE_RESERVED_NAME: pg_ で始まる名前への変更を試みた場合（システム保護）。

保守担当者向けメモ

  • 本関数は pg_namespace カタログを直接更新するため、呼び出し時には適切なロック管理が必須である。
  • InvokeObjectPostAlterHook の呼び出しにより、スキーマ名変更に伴う依存関係やキャッシュの整合性が維持される。
