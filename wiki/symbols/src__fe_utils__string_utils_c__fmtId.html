fmtId

定義

  • 種別: function
  • 定義箇所: src/fe_utils/string_utils.c:247

目的

SQL識別子を安全にクォートし、SQL文中で使用可能な形式に変換します。

シグネチャまたは形状

const char *fmtId(const char *rawid);

入力と出力

  • **入力**: rawid (変換対象の識別子文字列)
  • **出力**: クォート処理済みの文字列を指す const char * 型のポインタ。

前提条件と不変条件

  • 内部で getFmtEncoding() を使用してエンコーディングを取得するため、事前に適切なエンコーディング設定が必要です。
  • 戻り値は内部の静的バッファを指しており、次に fmtId 系関数が呼び出されると内容が上書きされます。そのため、戻り値は即座に使用するか、別の領域へコピーする必要があります。

アルゴリズムまたは振る舞い

1. 現在のエンコーディング設定を getFmtEncoding() で取得します。 2. 取得したエンコーディングを引数として fmtIdEnc() を呼び出し、実際のクォート処理を委譲します。 3. fmtIdEnc が生成した結果をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: getFmtEncoding(), fmtIdEnc()
  • **呼び出し元**: src/bin/pg_dump/dumputils.c 内の各関数（オブジェクト名や所有者名のクォート処理に使用）。

関連データ構造

  • PQExpBuffer (内部的に使用されるバッファ管理構造体)

失敗モードと境界条件

  • 内部バッファを使用するため、マルチスレッド環境ではスレッドセーフではありません。
  • 戻り値のポインタを保持し続けることはできません。

保守担当者向けメモ

  • 本関数は fmtIdEnc のラッパーです。エンコーディングを明示的に指定したい場合は fmtIdEnc() を直接使用することを推奨します。
  • 戻り値の生存期間に関する制約を呼び出し側に徹底させる必要があります。
