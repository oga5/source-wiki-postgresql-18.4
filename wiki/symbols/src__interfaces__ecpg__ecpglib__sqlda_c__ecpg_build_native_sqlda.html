ecpg_build_native_sqlda

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/sqlda.c:411

目的

PGresult オブジェクトから ECPG ネイティブモードの SQLDA 構造体を構築し、メタデータを初期化します。

シグネチャまたは形状

struct sqlda_struct *ecpg_build_native_sqlda(int line, PGresult *res, int row, enum COMPAT_MODE compat);

入力と出力

  • **入力**:
  • line: 呼び出し元のソースコード行番号（ログ出力用）。
  • res: 変換対象の PGresult ポインタ。
  • row: 処理対象の行インデックス。
  • compat: 互換モード指定。
  • **出力**:
  • 成功時: 初期化済みの struct sqlda_struct * を返します。
  • 失敗時: NULL を返します。

前提条件と不変条件

  • sqlda_native_total_size によって算出されたメモリサイズが、ecpg_alloc による確保に成功している必要があります。
  • 戻り値の SQLDA は、メタデータとデータ領域が連続した単一のメモリブロックとして確保されます。

アルゴリズムまたは振る舞い

1. sqlda_native_total_size を呼び出し、必要なメモリサイズを算出します。 2. ecpg_alloc でメモリを確保し、memset でゼロクリアします。 3. sqldaid に "SQLDA " を設定します。 4. PQnfields(res) を用いてフィールド数（sqld, sqln）を設定し、sqldabc を計算します。 5. 各フィールドについてループし、以下を実行します：

  • sqlda_dynamic_type で PostgreSQL の型を ECPG の型に変換して sqltype に格納。
  • PQfname で取得した列名を sqlname にコピー。

    • 6. 構築した sqlda ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/descriptor.c:969
  • src/interfaces/ecpg/ecpglib/execute.c:1807
  • **呼び出し先**:
  • sqlda_native_total_size
  • ecpg_alloc
  • PQnfields
  • ecpg_log
  • sqlda_dynamic_type
  • PQftype
  • PQfname

関連データ構造

  • struct sqlda_struct: ECPG ネイティブモードの SQLDA 構造体。
  • PGresult: libpq のクエリ結果構造体。
  • struct sqlvar_struct: SQLDA 内の各列情報を保持する構造体。

失敗モードと境界条件

  • ecpg_alloc がメモリ不足で失敗した場合、NULL を返します。
  • res が無効な場合、PQnfields 等の libpq 関数呼び出し時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はネイティブモード専用です。Informix 互換モードについては ecpg_build_compat_sqlda を参照してください。
  • メモリは単一ブロックで確保されるため、解放時は free(sqlda) のみで完結します。
  • sqlda_dynamic_type による型変換ロジックの変更は、本関数にも影響を与えるため注意してください。
