inet_cidr_pton_ipv6

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/inet_net_pton.c:437

目的

IPv6 アドレス（CIDR 表記を含む可能性がある文字列）を、ネットワークバイトオーダーのバイナリ形式に変換し、プレフィックス長を抽出します。

シグネチャまたは形状

int inet_cidr_pton_ipv6(const char *src, u_char *dst, size_t size)

入力と出力

  • **入力**:
  • src: 変換対象の IPv6 アドレス文字列（CIDR 表記を含む）。
  • dst: 変換結果を格納するバッファ。
  • size: dst バッファのサイズ。
  • **出力**:
  • 成功時: 抽出されたプレフィックス長（ビット数）。
  • 失敗時: -1 を返し、errno を設定。

前提条件と不変条件

  • size は NS_IN6ADDRSZ（通常 16 バイト）以上である必要があります。
  • src が指す文字列は、標準的な IPv6 表記（:: を含む）または IPv4 互換アドレス形式である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: 内部バッファをゼロクリアし、パース状態を初期化します。 2. **文字列走査**: 文字列を 1 文字ずつ走査し、16 進数、コロン（:）、ドット（.）、スラッシュ（/）を識別します。

  • 16 進数: 数値を蓄積し、16 ビット単位でバッファに書き込みます。
  • コロン: ゼロ圧縮（::）の検出や、16 ビット区切りの処理を行います。
  • ドット: IPv4 互換アドレスとして getv4 を呼び出し、残りの 4 バイトを処理します。
  • スラッシュ: getbits を呼び出し、プレフィックス長を確定させます。

    • 3. **ゼロ圧縮の展開**: :: が存在する場合、メモリをシフトしてゼロを補完します。 4. **終了処理**: プレフィックス長が指定されていない場合はデフォルトの 128 を適用し、結果を dst にコピーして返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/inet_net_pton.c 内の関数から参照されています。
  • **呼び出し先**:
  • getv4: IPv4 互換アドレスのパースに使用。
  • getbits: CIDR プレフィックス長のパースに使用。
  • memset, memcpy, strchr: 標準ライブラリ関数。

関連データ構造

  • NS_IN6ADDRSZ: IPv6 アドレスのバイトサイズ（16）。
  • NS_INT16SZ: 16 ビット整数のバイトサイズ（2）。
  • NS_INADDRSZ: IPv4 アドレスのバイトサイズ（4）。

失敗モードと境界条件

  • size < NS_IN6ADDRSZ: errno に EMSGSIZE を設定し -1 を返します。
  • 不正なフォーマット（不正な文字、連続するコロンの誤用、バッファオーバーフローの可能性など）: errno に ENOENT を設定し -1 を返します。

保守担当者向けメモ

  • 本関数は ISC ライセンス由来のコードを含みます。
  • memmove の代わりに手動でメモリシフトを行っている箇所があるため、バッファ操作の変更には注意してください。
  • getv4 や getbits の戻り値が正であることを前提としています。
