SubscriptionRelState

定義

  • **種別**: struct
  • **定義箇所**: src/include/catalog/pg_subscription_rel.h:78

目的

論理レプリケーションにおける、サブスクリプションと個々のテーブル（リレーション）の紐付け状態をメモリ上で管理するための構造体です。

シグネチャまたは形状

typedef struct SubscriptionRelState
{
	Oid			relid;    /* リレーションのOID */
	XLogRecPtr	lsn;      /* 同期進捗を示すログシーケンス番号 */
	char		state;    /* 現在の同期フェーズ */
} SubscriptionRelState;

入力と出力

  • 本構造体は、カタログ pg_subscription_rel から読み取られた情報、またはプロセス間通信で受け渡される状態情報を保持するために使用されます。
  • state フィールドには、SUBREL_STATE_* 定数（i, d, f, s, r など）が格納されます。

前提条件と不変条件

  • lsn は access/xlogdefs.h で定義された XLogRecPtr 型であり、同期の進捗を追跡します。
  • state は、永続化される状態（i, d, f, s, r）と、プロセス間通信専用の非永続状態（w, c）のいずれかを取ります。

アルゴリズムまたは振る舞い

  • テーブル同期プロセス（tablesync.c）において、同期フェーズの遷移や進捗の更新時に本構造体のインスタンスが生成・更新されます。
  • memcpy を用いてリスト内の要素からコピーされるなど、メモリ上での状態保持に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照・生成）**:
  • src/backend/catalog/pg_subscription.c
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/tablesync.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。状態の更新には UpdateSubscriptionRelState 等のAPIが利用されます。

関連データ構造

  • FormData_pg_subscription_rel: カタログ pg_subscription_rel の行構造体。
  • List: nodes/pg_list.h で定義され、SubscriptionRelState のポインタを保持するために使用されます。

失敗モードと境界条件

  • lsn フィールドは CATALOG_VARLEN マクロの影響を受ける可能性があるため、直接アクセスせず提供されたAPI経由での操作が推奨されます。
  • state に未定義の文字が設定された場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • SubscriptionRelState はメモリ内での管理用であり、カタログの永続化形式である FormData_pg_subscription_rel と混同しないよう注意してください。
  • 構造体のフィールド追加や変更を行う際は、pg_subscription_rel.c 内の関連関数（Add, Remove, Update）への影響を確認してください。
