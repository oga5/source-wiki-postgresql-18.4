SetOpStatePerGroupData

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeSetOp.c:62

目的

SQL の集合演算（INTERSECT および EXCEPT）において、グループごとの左側入力と右側入力のタプル出現回数を保持し、出力すべき行数を決定するための作業状態を管理する。

シグネチャまたは形状

typedef struct SetOpStatePerGroupData
{
    int64       numLeft;        /* number of left-input dups in group */
    int64       numRight;       /* number of right-input dups in group */
} SetOpStatePerGroupData;

入力と出力

  • **入力**: 実行ノードが処理する各グループ内のタプル出現数（int64）。
  • **出力**: なし（本構造体は状態保持用であり、計算結果は set_output_count 等の関数で利用される）。

前提条件と不変条件

  • numLeft および numRight は、特定のグループ内におけるタプルの出現回数を表す。
  • 本構造体は SetOp ノードの実行コンテキスト内で、グループ化処理の進行に合わせて更新される。

アルゴリズムまたは振る舞い

  • SetOp ノードが INTERSECT や EXCEPT を実行する際、各グループの左側入力と右側入力の出現回数をカウントし、本構造体に格納する。
  • このカウント値に基づき、SQL 仕様に従って最終的に出力すべき行数が決定される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSetOp.c 内の実行ロジック（ExecSetOp 等）から、グループごとの状態管理のために参照される。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはない。

関連データ構造

  • SetOpStatePerGroup: SetOpStatePerGroupData へのポインタ型（typedef SetOpStatePerGroupData *SetOpStatePerGroup;）。
  • SetOpState: src/include/nodes/execnodes.h で定義されるノード全体の実行状態構造体。

失敗モードと境界条件

  • 構造体自体に失敗モードは存在しない。
  • カウント値が int64 の範囲を超えるような極端なデータセットの場合、オーバーフローの可能性があるが、本構造体はその値を保持するのみである。

保守担当者向けメモ

  • 本構造体は nodeSetOp.c におけるグループごとの集計ロジックの中核である。
  • UNION 演算には使用されないため、修正時は INTERSECT / EXCEPT の仕様変更にのみ留意すること。
