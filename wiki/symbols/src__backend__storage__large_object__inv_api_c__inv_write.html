inv_write

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:542

目的

ラージオブジェクト（LO）の指定されたオフセット位置へデータを書き込み、必要に応じて pg_largeobject カタログテーブルのページ更新または新規挿入を行う。

シグネチャまたは形状

int inv_write(LargeObjectDesc *obj_desc, const char *buf, int nbytes)

入力と出力

  • **入力**:
  • obj_desc: 操作対象のラージオブジェクト記述子。
  • buf: 書き込むデータを含むバッファ。
  • nbytes: 書き込むバイト数。
  • **出力**:
  • 戻り値: 実際に書き込まれたバイト数。

前提条件と不変条件

  • obj_desc が有効であること。
  • obj_desc->flags に IFS_WRLOCK が設定されていること（書き込み権限の検証）。
  • 書き込み後のオフセットが MAX_LARGE_OBJECT_SIZE を超えないこと。
  • nbytes が 0 以下の場合は何もせず 0 を返す。

アルゴリズムまたは振る舞い

1. open_lo_relation() を呼び出し、pg_largeobject テーブルとインデックスをオープンする。 2. systable_beginscan_ordered を使用し、対象の loid と pageno をキーにスキャンを開始する。 3. nbytes 分のデータが書き込まれるまで、以下のループを繰り返す：

  • 既存のページが存在する場合、そのデータを読み込み、オフセット位置に新しいデータをマージして CatalogTupleUpdateWithInfo で更新する。
  • ページが存在しない（またはページを跨ぐ）場合、新しいページを作成し CatalogTupleInsertWithInfo で挿入する。
  • ページ間に「穴」がある場合はゼロ埋めを行う。

    • 4. 処理の過程で obj_desc->offset を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-fsstubs.c (複数の箇所から呼び出しあり)
  • **呼び出し先**:
  • open_lo_relation()
  • CatalogOpenIndexes()
  • systable_beginscan_ordered()
  • systable_getnext_ordered()
  • getdatafield()
  • CatalogTupleUpdateWithInfo() / CatalogTupleInsertWithInfo()

関連データ構造

  • LargeObjectDesc: LOのメタデータと状態を保持。
  • pg_largeobject: LOのデータを格納するシステムカタログ。
  • ScanKeyData: スキャン条件の定義に使用。

失敗モードと境界条件

  • **権限エラー**: IFS_WRLOCK がない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了。
  • **サイズ超過**: 書き込み後のサイズが MAX_LARGE_OBJECT_SIZE を超える場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了。
  • **データ破損**: pg_largeobject 内に NULL フィールドが存在する場合、elog でエラー終了。

保守担当者向けメモ

  • 本関数は LOBLKSIZE（通常2KB）単位で処理を行う。
  • pg_largeobject.data はTOASTされている可能性があるため、getdatafield を介したアクセスが必須である。
  • 書き込み完了後は CatalogCloseIndexes 等のクリーンアップが適切に行われる必要がある（ソース excerpt には CatalogCloseIndexes の呼び出しが明示されていないため、後続の処理または呼び出し元での管理を確認すること）。
