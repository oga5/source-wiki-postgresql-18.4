standby_desc_invalidations

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/standbydesc.c:100

目的

共有キャッシュ無効化メッセージ（SharedInvalidationMessage）の内容を人間が読める形式に変換し、指定された StringInfo バッファへ出力します。

シグネチャまたは形状

void standby_desc_invalidations(StringInfo buf,
                                int nmsgs, SharedInvalidationMessage *msgs,
                                Oid dbId, Oid tsId,
                                bool relcacheInitFileInval)

入力と出力

  • **入力**:
  • buf: 出力先の StringInfo バッファ。
  • nmsgs: 無効化メッセージの数。
  • msgs: 無効化メッセージの配列。
  • dbId, tsId: データベースおよびテーブルスペースの識別子。
  • relcacheInitFileInval: リレーションキャッシュ初期化ファイルの無効化フラグ。
  • **出力**: なし（buf を介して文字列を追記）。

前提条件と不変条件

  • nmsgs が 0 以下の場合は何も行わず即座に復帰します。
  • msgs 配列は少なくとも nmsgs 個の要素を持つ必要があります。

アルゴリズムまたは振る舞い

1. relcacheInitFileInval が真の場合、dbId と tsId を含む文字列をバッファに追記します。 2. "; inval msgs:" という文字列をバッファに追記します。 3. msgs 配列をループし、各メッセージの id に基づいて以下のいずれかの情報をバッファに追記します。

  • catcache: msg->id
  • catalog: msg->cat.catId
  • relcache: msg->rc.relId
  • smgr: 固定文字列
  • relmap: msg->rm.dbId
  • snapshot: msg->sn.relId
  • relsync: msg->rs.relid
  • その他: unrecognized id と数値を出力。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/rmgrdesc/heapdesc.c:257
  • src/backend/access/rmgrdesc/standbydesc.c:72
  • src/backend/access/rmgrdesc/xactdesc.c:350, 413, 481
  • **呼び出し先**:
  • appendStringInfo (マクロ/関数)
  • appendStringInfoString (マクロ/関数)

関連データ構造

  • StringInfo: PostgreSQLの動的文字列バッファ。
  • SharedInvalidationMessage: 共有キャッシュ無効化を表現する構造体。

失敗モードと境界条件

  • nmsgs が 0 の場合、処理はスキップされます。
  • 未定義の msg->id が渡された場合、"unrecognized id" として数値が出力されます。

保守担当者向けメモ

  • 本関数はスタンバイ関連のWALレコードだけでなく、ヒープやトランザクションのWALレコード解析からも呼び出される汎用ユーティリティです。
  • 新しい無効化メッセージタイプが追加された場合は、ループ内の条件分岐にケースを追加してください。
