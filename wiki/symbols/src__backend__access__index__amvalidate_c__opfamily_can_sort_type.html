opfamily_can_sort_type

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/amvalidate.c:270

目的

指定された演算子ファミリーが、特定のデータ型に対してソート順序を提供できる（B-tree インデックスとして利用可能である）かを判定します。

シグネチャまたは形状

bool opfamily_can_sort_type(Oid opfamilyoid, Oid datatypeoid)

入力と出力

  • **入力**:
  • opfamilyoid (Oid): 検査対象の演算子ファミリーのOID。
  • datatypeoid (Oid): 検査対象のデータ型のOID。
  • **出力**:
  • bool: 指定されたデータ型が当該ファミリー内でソート可能であれば true、そうでなければ false を返します。

前提条件と不変条件

  • opclass_for_family_datatype が B-tree アクセスメソッド (BTREE_AM_OID) を対象として検索を行うため、本関数は実質的に「そのファミリーが B-tree 演算子クラスとして当該型をサポートしているか」を検証します。

アルゴリズムまたは振る舞い

1. opclass_for_family_datatype を呼び出し、B-tree アクセスメソッドにおいて、指定された opfamilyoid と datatypeoid に対応する演算子クラス（OpClass）を検索します。 2. 検索結果が有効な OID であれば true を、無効（InvalidOid）であれば false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistvalidate.c
  • src/backend/access/spgist/spgutils.c
  • src/backend/access/spgist/spgvalidate.c
  • **呼び出し先**:
  • opclass_for_family_datatype (同一ファイル内)

関連データ構造

  • Oid: PostgreSQLのシステムカタログ識別子。
  • BTREE_AM_OID: B-tree アクセスメソッドを識別する定数。

失敗モードと境界条件

  • opfamilyoid または datatypeoid が無効な場合、opclass_for_family_datatype の仕様に従い、結果として false が返されます。

保守担当者向けメモ

  • 本関数は src/include/access/amvalidate.h で公開されており、主にインデックスの検証ロジックにおいて、ソート順序の依存関係を確認するために利用されます。
