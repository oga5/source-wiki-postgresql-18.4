xl_multixact_create

定義

  • 種別: struct
  • 定義箇所: src/include/access/multixact.h:73

目的

マルチトランザクション（MultiXact）の作成時に、その状態をWAL（Write-Ahead Log）へ記録するための構造体です。リカバリ時にマルチトランザクションのメンバー情報を再構築するために使用されます。

シグネチャまたは形状

typedef struct xl_multixact_create
{
    MultiXactId mid;            /* new MultiXact's ID */
    MultiXactOffset moff;       /* its starting offset in members file */
    int32       nmembers;       /* number of member XIDs */
    MultiXactMember members[FLEXIBLE_ARRAY_MEMBER];
} xl_multixact_create;

入力と出力

  • **入力**: MultiXactId（新規ID）、MultiXactOffset（メンバーファイル内の開始オフセット）、メンバー数、および MultiXactMember の配列。
  • **出力**: WALレコードとして永続化されるデータ構造。

前提条件と不変条件

  • members フィールドは可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、構造体の末尾に配置されます。
  • SizeOfMultiXactCreate マクロ（offsetof(xl_multixact_create, members)）を使用して、ヘッダー部分のサイズを計算可能です。

アルゴリズムまたは振る舞い

1. マルチトランザクション作成時、MultiXactIdCreate 内で本構造体が初期化されます。 2. メンバー情報が members 配列に格納されます。 3. WALレコードとして書き込まれ、リカバリ時にはこのレコードを読み込むことで、SLRU（pg_multixact/members）上の状態を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/multixact.c 内のマルチトランザクション作成処理。
  • **呼び出し先**: src/backend/access/rmgrdesc/mxactdesc.c（WALレコードのデコードおよび説明出力に使用）。
  • **補足**: 呼び出し元・呼び出し先の詳細なコールグラフは、提供されたソース範囲内では限定的です。

関連データ構造

  • MultiXactId: マルチトランザクションを識別する32ビット整数。
  • MultiXactMember: トランザクションIDとロック状態を保持する構造体。
  • MultiXactOffset: メンバーファイル内の位置を示すオフセット。

失敗モードと境界条件

  • メンバー数が非常に多い場合、WALレコードの最大サイズ制限に抵触する可能性があります。
  • 構造体の可変長部分のメモリ確保が正しく行われない場合、バッファオーバーランのリスクがあります。

保守担当者向けメモ

  • 本構造体はWALのフォーマットを直接規定しているため、フィールドの追加や変更を行う際は、rmgrdesc 側のデコード処理も併せて更新する必要があります。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には必ず提供されているマクロを使用してください。
