do_field

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-print.c:340

目的

PGresult オブジェクトの特定のフィールド値を整形し、指定された出力ストリーム（fout）への書き出し、または後続の出力処理のためのバッファリングを行う。

シグネチャまたは形状

static bool do_field(const PQprintOpt *po, const PGresult *res,
                     const int i, const int j, const int fs_len,
                     char **fields,
                     const int nFields, char const **fieldNames,
                     unsigned char *fieldNotNum, int *fieldMax,
                     const int fieldMaxLen, FILE *fout)

入力と出力

  • **入力**:
  • po: 出力オプション（PQprintOpt）。
  • res: 対象のクエリ結果セット。
  • i, j: 現在処理中の行および列のインデックス。
  • fields: 出力用バッファ配列。
  • fieldNames: フィールド名の配列。
  • fieldNotNum: 数値判定フラグの配列。
  • fieldMax: 各列の最大幅を保持する配列。
  • fout: 出力先のファイルストリーム。
  • **出力**: 成功時に true、メモリ不足時に false を返す。

前提条件と不変条件

  • po->align が有効な場合、fieldNotNum および fieldMax 配列は適切に初期化されている必要がある。
  • PQgetlength および PQgetvalue が有効な結果を返すことを前提とする。

アルゴリズムまたは振る舞い

1. **値の取得**: PQgetlength と PQgetvalue で対象フィールドの値を取得する。 2. **空値の判定**: 値が空の場合、po->align または po->expanded が有効であればスキップし、それ以外の場合は区切り文字を出力して終了する。 3. **数値判定**: po->align が有効かつ fieldNotNum[j] が未設定の場合、フィールド値が数値のみで構成されているかを確認し、非数値が含まれる場合は fieldNotNum[j] をセットする。 4. **出力処理**:

  • po->expanded が無効かつ（po->align または po->html3）が有効な場合、値を fields バッファにコピーし、fieldMax[j] を更新する。
  • po->expanded が有効な場合、HTML形式またはテキスト形式でヘッダーと値を整形して fout に出力する。
  • 上記以外（標準的な表形式）の場合、値を直接 fout に出力し、列の区切り文字または改行を付与する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-print.c 内のループ処理から呼び出される。
  • **呼び出し先**: PQgetlength, PQgetvalue, PQmblenBounded (libpq API)、および標準ライブラリの malloc, fprintf, fputs, fputc。

関連データ構造

  • PQprintOpt: 出力形式を制御する構造体。
  • PGresult: クエリ結果を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: fields へのメモリ確保に失敗した場合、エラーメッセージを stderr に出力し false を返す。
  • **数値判定の限界**: 数値判定ロジックは簡易的であり、特定の文字配置（例：末尾以外での 'E' の出現など）に対しては不完全な判定となる可能性がある。

保守担当者向けメモ

  • 本関数はレガシーな出力機能の一部であり、現代のアプリケーションでの使用は推奨されない。
  • 数値判定ロジックは厳密な型チェックを意図したものではないため、変更時は既存の出力フォーマットへの影響に注意すること。
