sigexit_handler

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_recvlogical.c:676

目的

シグナル（SIGINT または SIGTERM）を受信した際に、プログラムを安全に終了させるためのフラグを設定する。

シグネチャまたは形状

static void sigexit_handler(SIGNAL_ARGS)

※ SIGNAL_ARGS は libpq/pqsignal.h で定義されるシグナルハンドラの引数。

入力と出力

  • 入力: シグナル発生時の引数（SIGNAL_ARGS）。
  • 出力: なし（void）。グローバル変数の状態を更新する。

前提条件と不変条件

  • 本関数はシグナルハンドラとして呼び出されるため、非同期シグナル安全な操作のみを行う必要がある。
  • stop_reason および time_to_abort がグローバルスコープで定義されていること。

アルゴリズムまたは振る舞い

1. グローバル変数 stop_reason に STREAM_STOP_SIGNAL を代入する。 2. グローバル変数 time_to_abort を true に設定する。これにより、メインループ（StreamLogicalLog 等）が次回のループ判定時に終了処理へ移行する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（シグナル発生時）。
  • 呼び出し先: なし。
  • 参照元（設定箇所）:
  • src/bin/pg_basebackup/pg_recvlogical.c: main 関数内で pqsignal(SIGINT, sigexit_handler) および pqsignal(SIGTERM, sigexit_handler) として登録。
  • src/bin/pg_basebackup/pg_receivewal.c: 同様にシグナルハンドラとして登録。

関連データ構造

  • stop_reason: 終了理由を保持する列挙型変数。
  • time_to_abort: 終了を指示するブール型フラグ。

失敗モードと境界条件

  • 本関数自体は失敗しない。
  • シグナルが頻繁に発生した場合、グローバル変数の書き込みが繰り返されるが、副作用はない。

保守担当者向けメモ

  • 本関数は pg_recvlogical.c と pg_receivewal.c の両方で利用されている。変更を加える際は、両方のツールにおける終了挙動に影響がないか確認すること。
  • Windows 環境ではシグナル処理が異なる場合があるため、プラットフォーム固有の制約に注意すること。
