find_header

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-armor.c:265

目的

PGP ASCII アーマー形式のデータから、開始または終了のヘッダー行（-----BEGIN... または -----END...）を検索し、その位置と長さを特定します。

シグネチャまたは形状

int find_header(const uint8 *data, const uint8 *datend, const uint8 **start_p, int is_end);

入力と出力

  • data: 検索対象のバッファ開始ポインタ。
  • datend: 検索対象のバッファ終了ポインタ。
  • start_p: ヘッダーの開始位置を格納する出力用ポインタ。
  • is_end: 0 の場合は開始ヘッダー、それ以外の場合は終了ヘッダーを検索します。
  • 戻り値: 成功時はヘッダー行の長さ（バイト数）、失敗時は PXE_PGP_CORRUPT_ARMOR を返します。

前提条件と不変条件

  • data から datend までの範囲が有効なメモリ領域であること。
  • ヘッダーは行の先頭（バッファの開始位置または直前が \n）から始まる必要があります。

アルゴリズムまたは振る舞い

1. find_str を使用して、指定されたセパレータ（-----BEGIN または -----END）を検索します。 2. セパレータが行の先頭に位置していることを確認します。 3. セパレータ以降のヘッダーテキスト部分に不正な文字（制御文字など）が含まれていないか検証します。 4. ヘッダーの末尾に続く 5 文字がセパレータと一致することを確認します。 5. ヘッダー行の直後に改行コード（\n または \r\n）が存在することを確認します。 6. 成功した場合、start_p にヘッダー開始位置を格納し、ヘッダー全体の長さを返します。

呼び出し元と呼び出し先

  • 呼び出し元: pgp_armor_decode（contrib/pgcrypto/pgp-armor.c 内の複数箇所）
  • 呼び出し先: find_str（同ファイル内）

関連データ構造

  • uint8: バイトデータ型。
  • PXE_PGP_CORRUPT_ARMOR: エラーコード（pgp.h で定義）。

失敗モードと境界条件

  • セパレータが見つからない場合、PXE_PGP_CORRUPT_ARMOR を返します。
  • ヘッダー行内に不正な文字が含まれる場合、PXE_PGP_CORRUPT_ARMOR を返します。
  • ヘッダーの末尾が正しく閉じられていない場合、PXE_PGP_CORRUPT_ARMOR を返します。
  • 改行コードの形式が不正な場合、PXE_PGP_CORRUPT_ARMOR を返します。

保守担当者向けメモ

  • 本関数は PGP アーマーの構造解析において重要な役割を果たします。
  • find_str への依存があるため、検索ロジックを変更する際は注意してください。
  • エラーコード PXE_PGP_CORRUPT_ARMOR は呼び出し元で適切に処理されることが期待されています。
