pg_tablespace_databases

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/misc.c:223

目的

指定されたテーブルスペースOIDを使用しているデータベースのOIDを列挙する集合を返す関数（SRF）です。

シグネチャまたは形状

Datum pg_tablespace_databases(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) によりテーブルスペースのOIDを取得。
  • **出力**: tuplestore を介してデータベースOIDの集合を返却。

前提条件と不変条件

  • InitMaterializedSRF を使用してマテリアライズドSRFとして初期化される。
  • GLOBALTABLESPACE_OID が指定された場合は警告を発し、空の結果を返す。
  • 存在しないテーブルスペースやアクセス権限のないディレクトリに対しては、ENOENT 以外のエラーを除き、警告を発して空の結果を返す。

アルゴリズムまたは振る舞い

1. 指定されたOIDが DEFAULTTABLESPACE_OID かどうかを判定し、対象ディレクトリ（base または pg_tblspc/<oid>/<version>）を決定する。 2. AllocateDir を使用してディレクトリを開く。 3. ReadDir を用いてディレクトリ内の各エントリを走査する。 4. 各エントリがデータベースディレクトリであるかを確認し、directory_is_empty で中身が空でないことを検証する。 5. 有効なデータベースディレクトリであれば、そのOIDを tuplestore_putvalues を用いて結果セットに追加する。 6. 最後に FreeDir でディレクトリを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（直接的なC言語レベルの呼び出し元はソース内に明示なし）。
  • **呼び出し先**: InitMaterializedSRF, psprintf, AllocateDir, ReadDir, atooid, directory_is_empty, pfree, tuplestore_putvalues, FreeDir, ereport。

関連データ構造

  • ReturnSetInfo: SRFの結果セットを管理するための構造体。
  • DIR: ディレクトリ操作用ハンドル。
  • dirent: ディレクトリ内のエントリ情報。

失敗モードと境界条件

  • GLOBALTABLESPACE_OID 指定時はデータベースを含まないため空を返す。
  • ディレクトリが開けない場合、ENOENT であれば警告を出して終了し、それ以外のエラーは ereport(ERROR, ...) で処理を中断する。
  • directory_is_empty が真を返すディレクトリ（空のデータベースディレクトリ）は結果から除外される。

保守担当者向けメモ

  • 本関数はファイルシステムを直接走査するため、テーブルスペースの物理レイアウトに依存しています。
  • directory_is_empty の判定ロジックが、データベースの存在確認の唯一の基準となっています。
