pgstat_reset_database_timestamp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_database.c:406

目的

指定されたデータベースの統計情報リセット時刻（stat_reset_timestamp）を更新する。

シグネチャまたは形状

void pgstat_reset_database_timestamp(Oid dboid, TimestampTz ts)

入力と出力

  • dboid (Oid): 対象とするデータベースのOID。
  • ts (TimestampTz): 設定するリセット時刻。
  • 戻り値: なし。

前提条件と不変条件

  • 呼び出し元は、対象のデータベース統計エントリが存在することを前提とする。
  • 内部で共有メモリ上の統計エントリをロックするため、呼び出し後に必ずロックが解放される。

アルゴリズムまたは振る舞い

1. pgstat_get_entry_ref_locked を呼び出し、指定された dboid に対応する共有メモリ上の統計エントリへの参照を取得し、ロックする。 2. 取得した PgStatShared_Database 構造体の stats.stat_reset_timestamp フィールドに、引数 ts を代入する。 3. pgstat_unlock_entry を呼び出し、エントリのロックを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/activity/pgstat.c:865
  • 呼び出し先:
  • pgstat_get_entry_ref_locked
  • pgstat_unlock_entry

関連データ構造

  • PgStatShared_Database: 共有メモリ上に保持されるデータベース統計構造体。
  • PgStat_EntryRef: 統計エントリへのアクセスとロック状態を管理する構造体。

失敗モードと境界条件

  • 指定した dboid が存在しない場合、pgstat_get_entry_ref_locked の内部挙動に依存する（通常はエラーが発生する可能性がある）。

保守担当者向けメモ

  • 本関数はデータベース単位の統計リセット処理の一部として使用される。
  • 統計情報の共有メモリ操作を行うため、ロックの取得・解放の順序が正しく維持されていることを確認すること。
