目的

PostgreSQLの制御ファイル（global/pg_control）を読み込み、その内容を解析して人間が読める形式で標準出力に表示するコマンドラインツールのメインエントリポイントです。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • コマンドライン引数: データディレクトリ（-D オプションまたは位置引数）。
  • 環境変数: PGDATA（引数未指定時）。
  • **出力**:
  • 標準出力: 制御ファイルの内容（バージョン、システム識別子、チェックポイント情報など）。
  • 終了コード: 成功時は 0、引数エラーや読み込み失敗時は 1。

前提条件と不変条件

  • 実行環境は pg_logging_init およびロケール設定が初期化されていること。
  • get_controlfile により取得される ControlFileData 構造体が有効であること。
  • WalSegSz が正の整数であること（0以下の場合はWALファイル名の計算がスキップされる）。

アルゴリズムまたは振る舞い

1. **引数解析**: getopt_long を使用し、データディレクトリパスを特定する。 2. **ファイル読み込み**: get_controlfile を呼び出し、制御ファイルをメモリにロードし、CRCチェックサムを検証する。 3. **検証**: IsValidWalSegSize を用いてWALセグメントサイズが妥当か確認し、異常があれば警告を出力する。 4. **データ変換**:

  • time_t 型のタイムスタンプを localtime と strftime で文字列化する。
  • XLByteToSeg を用いてチェックポイントのREDO位置からWALファイル名を算出する。
  • 認証用ナンスを16進数文字列に変換する。

    • 5. **表示**: 取得した各フィールドを printf で整形して出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: OSのシェル（プログラムのエントリポイント）。
  • **呼び出し先**:
  • get_controlfile (制御ファイルの読み込みとCRC検証)
  • usage (ヘルプ表示)
  • dbState (データベース状態の文字列変換)
  • XLByteToSeg, XLogFileName (WAL関連計算)
  • pg_logging_init, get_progname (ロギング・ユーティリティ)
  • **注記**: ツールが提示した contrib/amcheck 関連の呼び出し元情報は、本シンボルとは無関係なコードベース内の同名参照である可能性が高く、本関数の呼び出し元としては不適切です。

関連データ構造

  • ControlFileData: 制御ファイルの内容を保持する構造体。
  • struct option: コマンドライン引数解析用。
  • XLogSegNo: WALセグメント番号。

失敗モードと境界条件

  • **引数不足**: データディレクトリが指定されていない場合、エラーメッセージを表示して終了する。
  • **CRC不一致**: 制御ファイルの破損またはバージョン不整合を検出し、警告を表示する。
  • **無効なWALセグメントサイズ**: 範囲外のサイズが検出された場合、警告を表示する。
  • **タイムスタンプ変換失敗**: localtime がNULLを返す場合、??? と表示する。

保守担当者向けメモ

  • ControlFileData 構造体の定義（src/include/catalog/pg_control.h）が変更された場合、本関数内の出力ロジックも必ず更新してください。
  • 本関数はフロントエンド環境で動作するため、バックエンドのライブラリ（特にタイムゾーン関連）を直接利用できない制約があります。
