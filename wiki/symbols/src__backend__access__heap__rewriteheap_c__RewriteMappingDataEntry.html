RewriteMappingDataEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/access/heap/rewriteheap.c:204

目的

テーブル再構築時における、古いタプル位置（TID）と新しいタプル位置の対応関係を管理するためのデータ構造です。論理レプリケーション用のマッピング情報を保持するために使用されます。

シグネチャまたは形状

typedef struct RewriteMappingDataEntry
{
    LogicalRewriteMappingData map; /* 古い位置と新しい位置の対応 */
    dlist_node    node;            /* リスト管理用ノード */
} RewriteMappingDataEntry;

入力と出力

  • 入力: LogicalRewriteMappingData 型のデータ。
  • 出力: なし（構造体としてメモリ上に保持される）。

前提条件と不変条件

  • dlist_node を介して二重連結リストの一部として管理されることが想定されています。
  • 構造体内の map フィールドには、ヒープ再構築プロセスで確定した有効な TID マッピングが含まれている必要があります。

アルゴリズムまたは振る舞い

  • dlist の要素としてリストに格納され、再構築処理の過程で収集されたマッピング情報を保持します。
  • dclist_container マクロを使用して、リストノードから構造体本体へのポインタを取得する操作が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/rewriteheap.c 内の関数群（palloc によるメモリ確保およびリストへの追加操作）。
  • 呼び出し先: 直接的な関数呼び出しはありませんが、dlist 操作関数群によって操作されます。

関連データ構造

  • LogicalRewriteMappingData: マッピングの本体データ。
  • dlist_node: リスト構造を形成するためのノード。

失敗モードと境界条件

  • メモリ不足: palloc を使用して確保されるため、システムメモリが枯渇した場合はエラーとなります。

保守担当者向けメモ

  • 本構造体は rewriteheap.c 内の論理デコーディング支援機能に特化しています。
  • dlist を使用しているため、リスト操作時のポインタ整合性に注意してください。
  • sizeof(RewriteMappingDataEntry) がメモリ確保時に使用されているため、構造体メンバを変更する際は注意が必要です。
