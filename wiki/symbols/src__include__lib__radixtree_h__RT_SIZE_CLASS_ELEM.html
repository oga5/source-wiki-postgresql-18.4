RT_SIZE_CLASS_ELEM

定義

  • 種別: struct
  • 定義箇所: src/include/lib/radixtree.h:642

目的

適応型基数木（ART）における各ノードサイズクラス（4, 16, 48, 256スロット）のメタデータを保持し、ノードの動的な拡張やメモリ割り当てを管理するための構造体です。

シグネチャまたは形状

typedef struct RT_SIZE_CLASS_ELEM
{
	const char *name;
	int			fanout;
	size_t		allocsize;
} RT_SIZE_CLASS_ELEM;

入力と出力

本構造体はデータ操作の直接的な入力・出力には関与しません。主に RT_SIZE_CLASS_INFO 配列の要素として、ノード管理ロジックに対する設定値を提供します。

前提条件と不変条件

  • name: ノード種別を識別する文字列（デバッグ用）。
  • fanout: 当該ノードが保持可能な最大子ノード数。
  • allocsize: 当該ノード構造体を確保するために必要なメモリサイズ。

アルゴリズムまたは振る舞い

適応型基数木において、ノードが満杯になった際に RT_SIZE_CLASS_INFO を参照し、現在のノード種別から次のサイズクラスへ移行するための情報を取得するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/lib/radixtree.h 内のノード拡張ロジック（RT_GROW_NODE_* 関連処理）において、RT_SIZE_CLASS_INFO を通じて参照されます。
  • **呼び出し先**: 情報なし。

関連データ構造

  • RT_SIZE_CLASS_INFO: 本構造体のインスタンスを保持する静的配列。
  • RT_NODE_* (4, 16, 48, 256): 各サイズクラスに対応するノード構造体。

失敗モードと境界条件

本構造体自体はデータ保持用であるため、直接的な失敗モードはありません。ただし、RT_SIZE_CLASS_INFO 配列のインデックス範囲外アクセスが発生した場合、ノード管理ロジックが未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はマクロテンプレートの一部として定義されており、RT_SIZE_CLASS_ELEM マクロを通じて展開されます。
  • 変更を加える際は、RT_SIZE_CLASS_INFO 配列の定義順序と、ノード拡張ロジックにおけるインデックスの整合性に注意してください。
