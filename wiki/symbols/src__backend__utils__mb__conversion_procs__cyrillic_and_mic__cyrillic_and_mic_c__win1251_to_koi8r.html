win1251_to_koi8r

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:453

目的

WIN1251 エンコーディングから KOI8-R エンコーディングへ文字列を変換する。

シグネチャまたは形状

Datum win1251_to_koi8r(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得
  • 第3引数: ソース文字列 (unsigned char *)
  • 第4引数: 宛先バッファ (unsigned char *)
  • 第5引数: 文字列長 (int)
  • 第6引数: エラーハンドリングフラグ (bool)
  • **出力**: 変換されたバイト数 (int32) を PG_RETURN_INT32 で返却

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_WIN1251, PG_KOI8R) により、ソースエンコーディングが WIN1251、宛先エンコーディングが KOI8-R であることが検証される。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認する。 3. local2local 関数を呼び出し、win12512koi マッピングテーブルを用いて変換を実行する。 4. 変換されたバイト数を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: local2local（変換処理の実体）。

関連データ構造

  • win12512koi: WIN1251 から KOI8-R への変換用マッピングテーブル。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断される可能性がある。
  • noError フラグが false の場合、変換不可能な文字が含まれるとエラーが発生する可能性がある（local2local の仕様に依存）。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_ARGS を引数に取るため、SQL インターフェース経由での利用を前提としている。直接呼び出す場合は PostgreSQL の関数呼び出し規約に従うこと。
  • 変換ロジックの変更は win12512koi テーブルの整合性に注意すること。
