StrategyFreeBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/freelist.c:362

目的

指定されたバッファ記述子（BufferDesc）を共有バッファのフリーリストに追加し、再利用可能な状態にする。

シグネチャまたは形状

void StrategyFreeBuffer(BufferDesc *buf)

入力と出力

  • **入力**: buf - フリーリストへ戻す対象のバッファ記述子へのポインタ。
  • **出力**: なし（副作用として共有メモリ上のフリーリスト状態を更新）。

前提条件と不変条件

  • StrategyControl->buffer_strategy_lock スピンロックを使用して、フリーリストの整合性を保護する。
  • バッファが既にフリーリストに含まれている場合は、リスト構造を破壊しないよう二重登録を回避する。

アルゴリズムまたは振る舞い

1. buffer_strategy_lock を獲得する。 2. 対象バッファの freeNext フィールドを確認し、FREENEXT_NOT_IN_LIST である（リストに含まれていない）ことを検証する。 3. リストに含まれていない場合：

  • 現在の firstFreeBuffer を対象バッファの freeNext に設定する。
  • リストが空だった場合（freeNext < 0）、lastFreeBuffer を対象バッファの ID に更新する。
  • firstFreeBuffer を対象バッファの ID に更新する。

    • 4. buffer_strategy_lock を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/bufmgr.c 内のバッファ管理処理（bufmgr.c:2101, 2264, 2686, 2766 など）。
  • **呼び出し先**: 内部的な関数呼び出しはなし。スピンロック操作（SpinLockAcquire, SpinLockRelease）を利用。

関連データ構造

  • BufferDesc: バッファのメタデータを保持する構造体。freeNext フィールドでフリーリストの連結を管理する。
  • BufferStrategyControl: 共有メモリ上のグローバルな戦略管理構造体。firstFreeBuffer および lastFreeBuffer を保持する。

失敗モードと境界条件

  • **二重登録**: 既にフリーリストにあるバッファを渡した場合、if (buf->freeNext == FREENEXT_NOT_IN_LIST) のチェックにより無視されるため、リストの破壊は発生しない。

保守担当者向けメモ

  • 本関数はフリーリストの連結操作のみを行う。バッファのピン解除やフラッシュ処理は呼び出し元（bufmgr.c）の責任である。
  • buffer_strategy_lock はフリーリストの操作に特化しているため、ロック保持時間は最小限に抑えられている。
