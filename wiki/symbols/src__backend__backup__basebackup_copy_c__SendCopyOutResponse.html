SendCopyOutResponse

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_copy.c:316

目的

クライアントに対して CopyOutResponse メッセージを送信し、バックアップデータの転送（COPYモード）を開始するためのプロトコル制御を行います。

シグネチャまたは形状

static void SendCopyOutResponse(void)

入力と出力

  • 入力: なし
  • 出力: なし（内部で pq_endmessage を通じてネットワークバッファへ書き出し）

前提条件と不変条件

  • PostgreSQLのフロントエンド/バックエンドプロトコルにおけるCOPYモードへの遷移時に呼び出される必要があります。

アルゴリズムまたは振る舞い

1. pq_beginmessage を使用して PqMsg_CopyOutResponse メッセージの構築を開始します。 2. pq_sendbyte で全体フォーマットとして 0（テキスト形式）を送信します。 3. pq_sendint16 で列数（natts）として 0 を送信します。 4. pq_endmessage でメッセージを確定し、クライアントへ送信します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/backup/basebackup_copy.c 内の bbsink_copystream_begin_backup から呼び出されます。
  • 呼び出し先: libpq/pqformat.h で定義された pq_beginmessage, pq_sendbyte, pq_sendint16, pq_endmessage を使用します。

関連データ構造

  • StringInfoData: メッセージ構築用の一時バッファとして使用されます。

失敗モードと境界条件

  • ネットワークエラーやクライアントとの接続断が発生した場合、libpq 側の関数群によってエラーが報告されます。

保守担当者向けメモ

  • 本関数は、バックアップのストリーム転送を開始する際の定型的なプロトコルハンドシェイクです。列数 0 は、このCOPYストリームが特定のテーブル行データではなく、バイナリデータ（アーカイブやマニフェスト）の転送用であることを示しています。
