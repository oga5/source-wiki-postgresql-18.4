g_int_penalty

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_gist.c:363

目的

GiST インデックスにおいて、新しいエントリを既存のインデックスツリーのノードに追加する際の「ペナルティ（コスト）」を計算します。この値は、挿入によってインデックスの範囲がどれだけ拡大するかを示します。

シグネチャまたは形状

Datum g_int_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): 既存のインデックスエントリ (GISTENTRY *)
  • PG_GETARG_POINTER(1): 追加対象の新しいエントリ (GISTENTRY *)
  • PG_GETARG_POINTER(2): 計算結果を格納するポインタ (float *)
  • **出力**: PG_RETURN_POINTER(result) を通じて、計算されたペナルティ値（float）を返します。

前提条件と不変条件

  • 入力される GISTENTRY のキーは、ArrayType として解釈可能である必要があります。
  • inner_int_union および rt__int_size が正常に動作するデータ形式である必要があります。

アルゴリズムまたは振る舞い

1. inner_int_union を使用し、既存のキーと新しいキーの和集合（union）を計算します。 2. rt__int_size を使用して、和集合のサイズ（tmp1）と、元のキーのサイズ（tmp2）をそれぞれ算出します。 3. tmp1 - tmp2 を計算し、その差分をペナルティとして result に格納します。 4. 計算に使用した一時的な和集合オブジェクトを pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの挿入処理（PostgreSQL コアの GiST インデックス管理機構から呼び出されます）。
  • **呼び出し先**:
  • inner_int_union: 和集合の計算。
  • rt__int_size: サイズの算出。
  • pfree: メモリ解放。
  • 注: ソースコード上の直接的な呼び出し元情報は、PG_FUNCTION_INFO_V1(g_int_penalty) マクロによる登録のみが確認されています。

関連データ構造

  • GISTENTRY: GiST インデックスのノードエントリを保持する構造体。
  • ArrayType: intarray モジュールで扱われる PostgreSQL の配列型。

失敗モードと境界条件

  • メモリ不足時に inner_int_union が失敗する可能性があります。
  • 入力された Datum が不正なポインタである場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は gist__int_ops 演算子クラスの一部です。
  • 範囲の拡大度合いを単純な差分で評価しているため、インデックスのバランス調整において重要な役割を果たします。
  • _int_tool.c で定義されている rt__int_size の実装変更が、本関数の計算結果に直接影響を与えます。
