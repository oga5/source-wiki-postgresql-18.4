gbt_byteacmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bytea.c:63

目的

bytea型（可変長バイナリデータ）の比較演算をGiSTインデックスの内部ロジックへ提供する。

シグネチャまたは形状

int32 gbt_byteacmp(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • 入力:
  • a, b: 比較対象のbyteaデータへのポインタ。
  • collation: 照合順序（本関数では未使用）。
  • flinfo: 関数マネージャ情報（本関数では未使用）。
  • 出力:
  • int32: aがbより小さい場合は負、等しい場合は0、大きい場合は正の値を返す。

前提条件と不変条件

  • 入力ポインタaおよびbは有効なbyteaデータ構造を指している必要がある。
  • 内部的にPostgreSQL標準のbyteacmp関数を呼び出すため、その仕様に準拠する。

アルゴリズムまたは振る舞い

1. PointerGetDatumを使用して、入力されたvoidポインタをDatum型に変換する。 2. DirectFunctionCall2を介して、PostgreSQLの標準比較関数であるbyteacmpを呼び出す。 3. byteacmpの戻り値をDatumGetInt32でint32型に変換して返却する。

呼び出し元と呼び出し先

  • 呼び出し先: byteacmp（PostgreSQL標準の比較関数）。
  • 呼び出し元: ソースコード内の静的解析では、contrib/btree_gist/btree_bytea.c内のtinfo構造体初期化時（81行目）に参照されていることが確認できる。その他の具体的な呼び出し元はソースから特定不可。

関連データ構造

  • tinfo: gbtree_vinfo構造体。本関数を比較ロジックとして保持し、btree_utils_var.cの汎用関数へ提供する。

失敗モードと境界条件

  • byteacmpの仕様に依存する。入力がNULLの場合や不正なbytea構造である場合の挙動は、標準のbyteacmpの動作に従う。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な比較ロジックはsrc/backend/utils/adt/varlena.cのbyteacmpに委譲されている。
  • collationおよびflinfo引数は、GiSTの汎用インターフェースに合わせるために定義されているが、現在の実装では使用されていない。
