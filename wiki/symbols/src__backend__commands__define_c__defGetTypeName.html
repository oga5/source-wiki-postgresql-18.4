defGetTypeName

定義

  • 種別: function
  • 定義箇所: src/backend/commands/define.c:270

目的

DefElem ノードからデータ型名（TypeName）を抽出・変換する。

シグネチャまたは形状

TypeName *defGetTypeName(DefElem *def)

入力と出力

  • **入力**: DefElem *def （解析対象の定義要素）
  • **出力**: TypeName * （抽出された型名構造体）
  • **例外**: def->arg が NULL の場合、またはノードが T_TypeName または T_String 以外の場合、ERRCODE_SYNTAX_ERROR でエラー終了する。

前提条件と不変条件

  • def は有効なポインタであること。
  • 戻り値は TypeName 構造体へのポインタであり、呼び出し側で適切に管理されること。

アルゴリズムまたは振る舞い

1. def->arg が NULL であればエラーを報告する。 2. nodeTag(def->arg) を判定する:

  • T_TypeName: そのまま TypeName * として返す。
  • T_String: 互換性のため、文字列を makeTypeNameFromNameList を用いて TypeName に変換して返す。
  • その他: 期待する型ではないとしてエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/aggregatecmds.c
  • src/backend/commands/operatorcmds.c
  • src/backend/commands/sequence.c
  • src/backend/commands/typecmds.c
  • **呼び出し先**:
  • makeTypeNameFromNameList (内部的な変換用)
  • ereport (エラー報告用)

関連データ構造

  • DefElem: SQLコマンドのオプションや定義を保持するノード。
  • TypeName: データ型名を表現する構造体。

失敗モードと境界条件

  • def->arg が NULL の場合: ERRCODE_SYNTAX_ERROR をスロー。
  • def->arg が T_TypeName でも T_String でもない場合: ERRCODE_SYNTAX_ERROR をスロー。

保守担当者向けメモ

  • 本関数は後方互換性のため、文字列リテラルを型名として扱う処理を含んでいる。
  • 新しいノード型をサポートする必要がある場合は、switch 文にケースを追加すること。
