InvalidationMsgsGroup

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/cache/inval.c:184

目的

キャッシュ無効化メッセージの論理的なグループを管理するための構造体です。現在のコマンドまたはトランザクション全体で発生した無効化イベントを追跡し、適切なタイミングで一括処理するために使用されます。

シグネチャまたは形状

typedef struct InvalidationMsgsGroup
{
	int			firstmsg[2];	/* 関連配列における開始インデックス */
	int			nextmsg[2];		/* 終了インデックス + 1 */
} InvalidationMsgsGroup;

入力と出力

  • **入力**: 本構造体は、AddInvalidationMessage 等の関数を通じて、特定の無効化メッセージをグループに追加する際のターゲットとして渡されます。
  • **出力**: 蓄積されたメッセージの範囲（インデックス）を保持し、後続の処理（コミット時の共有メモリへの書き出し等）で参照されます。

前提条件と不変条件

  • firstmsg および nextmsg は、無効化メッセージを格納する配列（InvalMessageArray 等）のインデックスを指します。
  • 配列のインデックスは [2] のサイズを持ち、サブグループごとの管理を可能にしています。

アルゴリズムまたは振る舞い

  • **メッセージ蓄積**: AddInvalidationMessage などの関数により、メッセージがグループに追加されるたびに nextmsg が更新されます。
  • **階層管理**: CurrentCmdInvalidMsgs（現在のコマンド用）や PriorCmdInvalidMsgs（前コマンド用）としてインスタンス化され、コマンド境界やトランザクション終了時にメッセージの伝播を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • AddInvalidationMessage
  • AppendInvalidationMessageSubGroup
  • AddCatcacheInvalidationMessage
  • AddCatalogInvalidationMessage
  • AddRelcacheInvalidationMessage
  • AddRelsyncInvalidationMessage
  • AddSnapshotInvalidationMessage
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • InvalMessageArray: メッセージの実体が格納される配列。
  • TransInvalidationInfo: トランザクションごとの無効化状態を保持する構造体で、本構造体をメンバとして含みます。

失敗モードと境界条件

  • インデックスが配列の最大サイズを超過する場合の挙動については、本構造体単体では規定されておらず、操作関数側の実装に依存します。

保守担当者向けメモ

  • 本構造体は抽象化を維持するためにマクロ経由での操作が推奨されています。
  • firstmsg と nextmsg のインデックスが指す配列の整合性は、inval.c 内のメッセージ追加関数群によって保証されています。
