join_is_removable

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:170

目的

与えられた左結合（JOIN_LEFT）が、結合結果に影響を与えずに削除可能かどうかを判定します。右側のリレーションが結合条件によって一意であることが証明できる場合に、結合を削除するための前段階として使用されます。

シグネチャまたは形状

static bool join_is_removable(PlannerInfo *root, SpecialJoinInfo *sjinfo)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • sjinfo: 判定対象の結合に関する情報（SpecialJoinInfo）。
  • **出力**:
  • bool: 削除可能な場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • 結合タイプが JOIN_LEFT である必要があります。
  • 右側のリレーションが単一のベースリレーションである必要があります。
  • 結合対象の右側リレーションが、クエリの resultRelation（MERGE 文などで発生）と一致してはなりません。

アルゴリズムまたは振る舞い

1. **初期チェック**: 結合タイプ、右側リレーションの単一性、および resultRelation との重複を検証します。 2. **一意性サポート確認**: rel_supports_distinctness を呼び出し、右側リレーションがそもそも一意性をサポートしているかを確認します。 3. **属性使用状況の確認**: 結合より上位で右側リレーションの属性が使用されていないか、attr_needed を走査して検証します。 4. **PlaceHolderVar の検証**: 結合より上位で必要とされる PlaceHolderVar が、右側リレーションに依存していないか、または削除後も評価場所が確保できるかを検証します。 5. **結合条件の収集**: 結合条件のうち、マージ結合可能（mergejoinable）かつプッシュダウンされていないものを収集します。 6. **一意性の証明**: 収集した条件に基づき、rel_is_distinct_for を用いて右側リレーションの一意性を証明します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/analyzejoins.c 内の remove_useless_joins から呼び出されます。
  • **呼び出し先**:
  • bms_get_singleton_member, bms_union, bms_copy, bms_add_member, bms_is_subset, bms_is_member, bms_overlap (ビットマップ操作)
  • find_base_rel
  • rel_supports_distinctness
  • pull_varnos
  • clause_sides_match_join
  • rel_is_distinct_for

関連データ構造

  • PlannerInfo: プランニング全体のコンテキスト。
  • SpecialJoinInfo: 結合の制約情報。
  • RelOptInfo: リレーションの最適化情報。
  • PlaceHolderInfo: プレースホルダー変数に関する情報。
  • RestrictInfo: 結合条件などの制約情報。

失敗モードと境界条件

  • 結合タイプが JOIN_LEFT 以外の場合、即座に false を返します。
  • 右側リレーションが複数ある場合、判定不能として false を返します。
  • 結合より上位で右側リレーションの属性や PlaceHolderVar が必要とされる場合、削除不可と判断します。
  • マージ結合可能な条件が見つからない、または一意性が証明できない場合は false を返します。

保守担当者向けメモ

  • 本関数は JOIN_LEFT の削除に特化しています。将来的に他の結合タイプへの拡張が検討される可能性があります。
  • PlaceHolderVar の評価位置（ph_eval_at）に関するロジックは複雑であり、結合削除後に評価場所が消失しないよう厳密にチェックしています。
  • rel_is_distinct_for による一意性証明は、現在マージ結合可能な条件に依存しています。他の証明手法（一意インデックスの利用など）を追加する余地があります。
