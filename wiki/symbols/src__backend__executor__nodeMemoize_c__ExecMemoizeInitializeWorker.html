ExecMemoizeInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMemoize.c:1235

目的

並列クエリ実行時、各ワーカープロセスにおいて共有メモリ上の統計情報（shared_info）を検索し、MemoizeState ノードに紐付ける。

シグネチャまたは形状

void ExecMemoizeInitializeWorker(MemoizeState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の MemoizeState 構造体へのポインタ。
  • pwcxt: 並列ワーカーのコンテキスト情報（ParallelWorkerContext）。
  • **出力**: なし（node->shared_info が更新される）。

前提条件と不変条件

  • pwcxt->toc が有効であり、プランノード ID に対応するエントリが共有メモリ内に存在すること。

アルゴリズムまたは振る舞い

1. pwcxt->toc（Table of Contents）から、現在のプランノード ID (node->ss.ps.plan->plan_node_id) をキーとして shm_toc_lookup を実行する。 2. 取得した共有メモリ上の統計情報ポインタを node->shared_info に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1402 にて呼び出されることが確認されている。
  • **呼び出し先**: shm_toc_lookup（共有メモリ内のデータ検索関数）。

関連データ構造

  • MemoizeState: ノードの実行状態を保持する構造体。
  • ParallelWorkerContext: 並列実行時のワーカーコンテキスト。

失敗モードと境界条件

  • shm_toc_lookup の第3引数が true で指定されているため、該当するキーが見つからない場合はエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は並列クエリ実行時のワーカー初期化フェーズでのみ使用される。
  • 共有メモリのレイアウト変更やプランノード ID の管理に変更がある場合は、本関数の検索ロジックに影響が及ぶ可能性がある。
