Edge

定義

  • 種別: struct
  • 定義箇所: src/include/optimizer/geqo_recombination.h:35

目的

遺伝的クエリ最適化（GEQO）におけるエッジ組み換え交叉（ERX: Edge Recombination Crossover）アルゴリズムにおいて、都市間の接続関係（エッジ）を保持するために使用されます。

シグネチャまたは形状

typedef struct Edge
{
    Gene        edge_list[4];    /* list of edges */
    int         total_edges;
    int         unused_edges;
} Edge;

入力と出力

本構造体はデータ保持用であり、関数ではありません。src/backend/optimizer/geqo/geqo_erx.c 内の関数において、Edge 型の配列（テーブル）としてメモリ確保され、交叉処理の過程で各都市のエッジ情報が読み書きされます。

前提条件と不変条件

  • edge_list は最大4つの接続先（Gene）を保持します。
  • total_edges はその都市に接続されているエッジの総数を表します。
  • unused_edges は、交叉処理中にまだ処理されていないエッジの残数を追跡するために使用されます。

アルゴリズムまたは振る舞い

ERXアルゴリズムにおいて、親個体のツアーから抽出された接続関係をこの構造体に格納します。交叉プロセスでは、このテーブルを参照して次に選択すべき都市を決定し、選択済みの都市を remove_gene 関数等でマークすることで、ツアーの構築を進行させます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**: src/backend/optimizer/geqo/geqo_erx.c 内の gimme_edge, remove_gene, gimme_gene, edge_failure 等の関数がこの構造体を操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Gene: ツアー内の都市を識別するための型。
  • PlannerInfo: GEQOのコンテキスト情報。
  • Edge 配列: geqo_erx.c 内で palloc を用いて動的に確保される作業用テーブル。

失敗モードと境界条件

  • メモリ確保: palloc を使用しているため、メモリ不足時にはシステム標準の例外が発生します。
  • 呼び出し側での解放: palloc で確保された Edge テーブルは、使用後に必ず対応する解放関数（free_edge_table 等）を呼び出してメモリリークを防ぐ必要があります。

保守担当者向けメモ

  • 本構造体は geqo_erx.c のロジックに強く依存しています。変更を加える際は、ERXアルゴリズムにおけるエッジリストの最大数（4）が現在の実装要件を満たしているか確認してください。
  • contrib/fuzzystrmatch 等で見られる「Edge」という用語は、本構造体とは無関係な文字列処理上の用語であり、混同しないよう注意してください。
