rebuild_placeholder_attr_needed

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:328

目的

クエリ内のすべての PlaceHolderInfo を走査し、各プレースホルダ式が参照する変数を特定した上で、それらの変数が評価されるべきリレーション（ph_eval_at）に対して必要な属性として再登録する。

シグネチャまたは形状

void rebuild_placeholder_attr_needed(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root - 現在のクエリ最適化状態を保持する構造体。root->placeholder_list を参照する。
  • **出力**: なし（副作用として root 内の属性必要情報を更新する）。

前提条件と不変条件

  • root->placeholder_list が有効な PlaceHolderInfo のリストを保持していること。
  • add_vars_to_attr_needed が呼び出し可能であること。

アルゴリズムまたは振る舞い

1. root->placeholder_list に含まれる各 PlaceHolderInfo を順次処理する。 2. pull_var_clause を使用し、プレースホルダ式（ph_var->phexpr）から以下の要素を含む変数のリストを抽出する。

  • 集約関数、ウィンドウ関数、および他のプレースホルダを含む。

    • 3. 抽出した変数リストと、そのプレースホルダの評価レベル（ph_eval_at）を add_vars_to_attr_needed に渡し、必要な属性としてマークする。 4. 抽出に使用した変数リストのメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/analyzejoins.c:659
  • src/backend/optimizer/plan/analyzejoins.c:2074
  • **呼び出し先**:
  • pull_var_clause
  • add_vars_to_attr_needed
  • list_free

関連データ構造

  • PlannerInfo: クエリ全体の最適化情報を保持。
  • PlaceHolderInfo: プレースホルダの評価レベルや式を保持。

失敗モードと境界条件

  • placeholder_list が空の場合、何も行わず正常終了する。
  • pull_var_clause が抽出する変数が存在しない場合、add_vars_to_attr_needed は何も行わない。

保守担当者向けメモ

  • 本関数は、結合の再構成や最適化の過程で、式の依存関係が変更された際に属性の必要性を再計算するために使用される。
  • pull_var_clause のフラグ設定（PVC_RECURSE_...）は、プレースホルダ内の複雑な式を正しく追跡するために重要である。
