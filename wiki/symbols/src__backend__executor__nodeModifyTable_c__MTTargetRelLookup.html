MTTargetRelLookup

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeModifyTable.c:75

目的

ModifyTable ノードにおいて、ターゲットとなるリレーション（テーブル）を効率的に検索・特定するためのハッシュテーブル用エントリ構造体です。

シグネチャまたは形状

typedef struct MTTargetRelLookup
{
	Oid			relationOid;	/* hash key, must be first */
	int			relationIndex;	/* rel's index in resultRelInfo[] array */
} MTTargetRelLookup;

入力と出力

  • **入力**: relationOid（検索キーとなるリレーションのOID）
  • **出力**: relationIndex（resultRelInfo[] 配列内でのインデックス位置）

前提条件と不変条件

  • relationOid はハッシュテーブルのキーとして機能するため、構造体の先頭に配置される必要があります。
  • relationIndex は、実行ノードが保持する resultRelInfo 配列の有効なインデックスを指している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、ハッシュテーブルの各エントリとして使用されます。
  • 実行時に特定のリレーションに対する ResultRelInfo を高速に取得するために、OID をキーとして relationIndex を解決するルックアップ処理に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeModifyTable.c 内のハッシュテーブル操作箇所（hash_ctl.entrysize の設定および mtlookup ポインタを介したアクセス）。
  • **呼び出し先**: 特定の関数呼び出し関係はソース抜粋からは不明です。

関連データ構造

  • resultRelInfo[]: ターゲットリレーションの情報を保持する配列。
  • ModifyTableContext: 本構造体を利用するコンテキストの一部として使用される可能性があります。

失敗モードと境界条件

  • ハッシュテーブルの初期化やエントリサイズの設定（sizeof(MTTargetRelLookup)）が正しく行われていない場合、メモリ破壊や検索失敗が発生します。
  • 存在しない relationOid で検索を行った場合の挙動は、ハッシュテーブルの実装に依存します。

保守担当者向けメモ

  • 本構造体は nodeModifyTable.c 内のハッシュテーブル管理に密接に関連しています。変更を加える際は、ハッシュキーとしての relationOid の配置順序を維持してください。
