GinBuffer

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/gininsert.c:1174

目的

GINインデックス構築時において、特定のキーに関連付けられたTID（タプル識別子）リストを一時的に蓄積・管理するためのバッファ構造体です。

シグネチャまたは形状

typedef struct GinBuffer
{
	OffsetNumber attnum;
	GinNullCategory category;
	Datum		key;
	Size		keylen;
	int16		typlen;
	bool		typbyval;
	int			maxitems;
	int			nitems;
	int			nfrozen;
	SortSupport ssup;
	ItemPointerData *items;
} GinBuffer;

入力と出力

  • **入力**: palloc0 等でメモリを確保し、GinBufferInit 等で初期化して使用します。
  • **出力**: 蓄積されたTIDリストは、GinBufferStoreTuple などの操作を通じてインデックスページへ書き込まれます。

前提条件と不変条件

  • items 配列には、maxitems で指定された上限までTIDを格納可能です。
  • ssup（SortSupport）は、キーの比較やソートのために適切に設定されている必要があります。

アルゴリズムまたは振る舞い

  • GINインデックス構築プロセスにおいて、ヒープから抽出されたキーとTIDを一時的に保持します。
  • nitems が maxitems に達した際や、特定の条件（GinBufferShouldTrim 等）を満たした際に、蓄積されたデータをインデックスへフラッシュする処理をサポートします。
  • GinBufferReset により、バッファの内容をクリアして再利用可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gininsert.c 内のインデックス構築処理フローから参照されます。
  • **呼び出し先**: AssertCheckGinBuffer による検証、GinBufferIsEmpty、GinBufferKeyEquals、GinBufferStoreTuple、GinBufferReset、GinBufferTrim などの専用関数が本構造体を操作します。

関連データ構造

  • ItemPointerData: 蓄積されるTIDの型。
  • SortSupport: キーのソート順序を定義する構造体。
  • GinTuple: インデックスへ書き込む際のタプル表現。

失敗モードと境界条件

  • maxitems を超えてTIDを追加しようとした場合、メモリ不足やバッファオーバーフローを防ぐための適切なチェック（GinBufferShouldTrim 等）が必要です。
  • key が存在しない場合（keylen == 0）、適切に処理される必要があります。

保守担当者向けメモ

  • 本構造体はインデックス構築のパフォーマンスに直結するため、maintenance_work_mem に基づく maxitems の設定が適切か確認してください。
  • nfrozen フィールドは、既に確定（凍結）されたTIDの管理に関連しており、転置リストの圧縮ロジックと密接に関係しています。
