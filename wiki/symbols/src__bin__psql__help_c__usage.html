usage

定義

  • 種別: function
  • 定義箇所: src/bin/psql/help.c:47

目的

psql のコマンドラインオプションに関するヘルプ情報を標準出力またはページャ経由で表示する。

シグネチャまたは形状

void usage(unsigned short int pager);

入力と出力

  • **入力**: pager (unsigned short int) - 0以外の場合、pset.popt.topt に基づくページングを有効にする。
  • **出力**: なし（標準出力またはページャへヘルプテキストを出力）。

前提条件と不変条件

  • pset 構造体が初期化されていること。
  • PQExpBuffer を使用してメモリ上でヘルプテキストを構築するため、十分なメモリが利用可能であること。

アルゴリズムまたは振る舞い

1. initPQExpBuffer でバッファを初期化する。 2. HELP0 および HELPN マクロを使用して、コマンドライン引数、入出力オプション、出力形式、接続オプション等のヘルプテキストをバッファに追加する。 3. バッファ内の改行文字をカウントし、行数を算出する。 4. PageOutput を呼び出し、pager 引数が真であればページャを起動し、出力先ファイルポインタを取得する。 5. バッファの内容を fputs で出力先へ書き出す。 6. ClosePager でページャを終了し、termPQExpBuffer でバッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述なし。
  • **呼び出し先**: initPQExpBuffer, PageOutput, fputs, ClosePager, termPQExpBuffer。

関連データ構造

  • PQExpBufferData: ヘルプテキスト構築用バッファ。
  • pset: settings.h で定義される psql の設定状態保持構造体。

失敗モードと境界条件

  • メモリ不足時: PQExpBuffer の内部処理で失敗する可能性がある。
  • ページャ起動失敗時: PageOutput が標準出力を返す挙動に依存する。

保守担当者向けメモ

  • ヘルプテキストの内容を変更した場合は、必ず ref/psql-ref.sgml（DocBook）と同期させること。
  • HELPN マクロを使用する際は、PACKAGE_BUGREPORT や PACKAGE_NAME 等の定数が正しく定義されている環境であることを前提とする。
