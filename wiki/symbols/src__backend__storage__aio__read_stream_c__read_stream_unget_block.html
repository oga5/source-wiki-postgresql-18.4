read_stream_unget_block

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/read_stream.c:210

目的

読み込み処理において、一度取得したブロック番号をストリームのバッファへ一時的に差し戻す（アンゲットする）ことで、再利用を可能にする。

シグネチャまたは形状

static inline void read_stream_unget_block(ReadStream *stream, BlockNumber blocknum)

入力と出力

  • **入力**:
  • stream: 操作対象の ReadStream 構造体へのポインタ。
  • blocknum: 差し戻す対象の BlockNumber。
  • **出力**: なし（void）。

前提条件と不変条件

  • stream->buffered_blocknum が InvalidBlockNumber であること（一度に差し戻せるのは1ブロックのみ）。
  • blocknum が InvalidBlockNumber ではないこと。

アルゴリズムまたは振る舞い

1. stream->buffered_blocknum が空であることをアサートで確認する。 2. blocknum が有効であることをアサートで確認する。 3. 指定された blocknum を stream->buffered_blocknum に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/read_stream.c:486（直接的な呼び出し）
  • **呼び出し先**:
  • なし（インライン関数であり、外部関数呼び出しは含まない）。

関連データ構造

  • ReadStream: ストリームの状態を保持する構造体。buffered_blocknum フィールドを介して本関数と連携する。

失敗モードと境界条件

  • 既に buffered_blocknum に値が設定されている状態で呼び出した場合、アサート違反により異常終了する。
  • InvalidBlockNumber を差し戻そうとした場合、アサート違反により異常終了する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを考慮してヘッダーではなくソースファイル内で定義されている。
  • 呼び出し元で buffered_blocknum が空であることを保証するロジックが重要である。
