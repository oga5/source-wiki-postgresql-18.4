statext_dependencies_serialize

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/dependencies.c:443

目的

MVDependencies 構造体で保持されている関数従属性の統計情報を、システムカタログ（pg_statistic_ext_data）へ格納可能なバイナリ形式（bytea）に変換する。

シグネチャまたは形状

bytea *statext_dependencies_serialize(MVDependencies *dependencies)

入力と出力

  • **入力**: MVDependencies *dependencies （シリアライズ対象の関数従属性データ構造）
  • **出力**: bytea * （シリアライズされたバイナリデータ。palloc0 によりメモリ確保される）

前提条件と不変条件

  • dependencies ポインタは有効であること。
  • シリアライズ処理中、内部の Assert により、計算されたバッファサイズと実際に書き込まれたデータサイズが一致することが保証される。

アルゴリズムまたは振る舞い

1. VARHDRSZ およびヘッダーサイズに基づき、必要なメモリ量を算出する。 2. 各従属性（MVDependency）の属性数に応じて必要なサイズを累積加算する。 3. palloc0 でメモリを確保し、SET_VARSIZE でサイズを設定する。 4. magic、type、ndeps のヘッダー情報をコピーする。 5. 各従属性について、degree（double）、nattributes（AttrNumber）、および属性番号の配列を順次コピーする。 6. 書き込み完了後、バッファの末尾が計算したサイズと一致することを確認し、ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/extended_stats.c:804
  • **呼び出し先**: 直接的な関数呼び出しは行わず、メモリ操作（memcpy）およびメモリ確保（palloc0）を行う。

関連データ構造

  • MVDependencies: 関数従属性の集合を保持する構造体。
  • MVDependency: 個別の関数従属性（度合いと属性リスト）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • 内部の Assert により、バッファオーバーフローや書き込みサイズ不一致が検出される。

保守担当者向けメモ

  • 本関数は pg_statistic_ext_data への書き込み用バイナリを生成する。データ構造の変更（MVDependency のメンバ追加など）を行う場合は、本関数のシリアライズ順序と、対応する statext_dependencies_deserialize のデシリアライズ順序を同期させる必要がある。
