PrintfArgValue

定義

  • 種別: typedef union
  • 定義箇所: src/port/snprintf.c:152

目的

printf ファミリーのフォーマット処理において、可変長引数（va_list）から抽出された値を、型を保持したまま一時的に格納するための共用体です。

シグネチャまたは形状

typedef union
{
	int			i;
	long		l;
	long long	ll;
	double		d;
	char	   *cptr;
} PrintfArgValue;

入力と出力

  • **入力**: va_list から読み取られた各型の値。
  • **出力**: 変換処理（fmtstr, fmtint, fmtfloat 等）へ渡される値の保持。

前提条件と不変条件

  • %n$ 形式の引数指定が使用される際、引数は事前に find_arguments() によって解析され、この型の配列に格納されます。
  • 格納される値の型は、対応する PrintfArgType によって管理されます。

アルゴリズムまたは振る舞い

dopr() 関数内において、フォーマット文字列中に %n$ 指定子が見つかった場合、find_arguments() が va_list を走査し、引数をこの共用体配列にコピーします。これにより、引数の順序を任意に入れ替えて参照することが可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: find_arguments() がこの型の配列を生成・設定します。また、dopr() がこの配列を参照してフォーマット変換を行います。
  • **呼び出し先**: 変換処理を行う各ヘルパー関数（fmtstr, fmtint, fmtfloat 等）が、この共用体から値を取り出して使用します。

関連データ構造

  • PrintfArgType: この共用体に格納された値の型を識別するために使用されます。
  • PrintfTarget: 出力先を管理する構造体。

失敗モードと境界条件

  • PG_NL_ARGMAX (31) を超える引数インデックスを指定した場合の挙動は、find_arguments() の実装に依存します。
  • 共用体であるため、書き込んだ型と異なる型で読み出した場合の動作は未定義です（型管理が正しく行われていることが前提となります）。

保守担当者向けメモ

  • 本シンボルは src/port/snprintf.c 内の POSIX 準拠引数制御に不可欠です。
  • 新しい型をサポートする必要がある場合は、この共用体にメンバを追加し、find_arguments() および型判定ロジックを更新してください。
