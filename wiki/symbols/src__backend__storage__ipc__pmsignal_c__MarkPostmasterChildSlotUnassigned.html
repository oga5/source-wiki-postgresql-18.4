MarkPostmasterChildSlotUnassigned

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:248

目的

Postmasterが管理する共有メモリ上の子プロセススロットを解放し、状態を PM_CHILD_UNUSED に更新する。

シグネチャまたは形状

bool MarkPostmasterChildSlotUnassigned(int slot);

入力と出力

  • **入力**: slot (int) - 解放対象のスロット番号（1始まり）。
  • **出力**: bool - スロットの状態が更新前は PM_CHILD_ASSIGNED であった場合に true を返し、それ以外（既に未使用など）の場合は false を返す。

前提条件と不変条件

  • slot は 1 以上 num_child_flags 以下である必要がある（Assert により検証）。
  • PMSignalState が初期化済みであること。

アルゴリズムまたは振る舞い

1. 指定された slot を0始まりのインデックスに変換する。 2. PMSignalState->PMChildFlags 配列の該当インデックスを確認し、現在の値が PM_CHILD_ASSIGNED であるかを判定して結果を保持する。 3. 該当スロットの値を PM_CHILD_UNUSED に設定する。 4. 手順2で保持した判定結果を戻り値として返す。

  • 子プロセスのクラッシュ時などに二重に呼び出される可能性があるため、状態遷移の整合性については厳密なアサーションを行わない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/pmchild.c:266 からの呼び出しが確認されている。
  • **呼び出し先**: 本関数内での関数呼び出しは発生しない。

関連データ構造

  • PMSignalData: 共有メモリ上のメイン構造体。PMChildFlags 配列を保持する。
  • PM_CHILD_ASSIGNED, PM_CHILD_UNUSED: スロットの状態を示す列挙値。

失敗モードと境界条件

  • slot が範囲外の場合、Assert が有効なビルドではプロセスが停止する。
  • 同一スロットに対して複数回呼び出された場合、2回目以降は false が返されるが、エラーとは見なされない。

保守担当者向けメモ

  • 本関数は postmaster.c における子プロセスのライフサイクル管理の一環として使用される。
  • 共有メモリ上のフラグ操作はアトミックに行われることが前提となっている。
