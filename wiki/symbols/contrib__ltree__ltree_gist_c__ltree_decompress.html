ltree_decompress

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_gist.c:112

目的

GiST インデックスに格納された圧縮状態の ltree_gist データを展開し、インデックス操作に必要な形式へ変換します。

シグネチャまたは形状

Datum ltree_decompress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体を受け取ります。
  • **出力**: 展開された GISTENTRY へのポインタを Datum として返します。

前提条件と不変条件

  • 入力された entry->key は ltree_gist 型のデータである必要があります。
  • PG_DETOAST_DATUM により、必要に応じて TOAST されたデータが展開されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) で GISTENTRY を取得します。 2. PG_DETOAST_DATUM を使用して、インデックスキーを ltree_gist 構造体へ展開します。 3. 展開後のデータが元の entry->key と異なる（TOAST 展開が発生した）場合:

  • 新しい GISTENTRY をメモリ上に確保（palloc）します。
  • gistentryinit を呼び出し、展開されたキーと元のコンテキスト情報（リレーション、ページ、オフセット）を用いて新しいエントリを初期化します。
  • 新しいエントリを返します。

    • 4. 展開が発生しなかった場合は、元の entry をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構から呼び出されます（直接的な呼び出し元コードの詳細は提供されていません）。
  • **呼び出し先**: PG_DETOAST_DATUM, palloc, gistentryinit などの PostgreSQL 内部マクロおよび関数。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • ltree_gist: ltree 型の GiST インデックス用データ構造。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 入力データが不正な形式である場合、PG_DETOAST_DATUM 等で例外が発生する可能性があります。

保守担当者向けメモ

  • 本関数は GiST インデックスの読み出し時に自動的に呼び出されます。
  • ltree_gist 構造体の変更や、TOAST 処理の最適化を行う際は、gistentryinit に渡す引数の整合性に注意してください。
