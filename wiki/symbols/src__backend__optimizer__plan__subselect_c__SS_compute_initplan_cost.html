SS_compute_initplan_cost

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/plan/subselect.c:2311

目的

指定された InitPlan リストの実行コストを合計し、それらが並列実行に対して安全かどうかを判定します。

シグネチャまたは形状

void SS_compute_initplan_cost(List *init_plans, Cost *initplan_cost_p, bool *unsafe_initplans_p)

入力と出力

  • **入力**:
  • init_plans: 評価対象の SubPlan ノードのリスト。
  • **出力**:
  • *initplan_cost_p: 計算されたコストの合計値。
  • *unsafe_initplans_p: リスト内に parallel_safe ではない SubPlan が一つでも含まれる場合に true が設定される。

前提条件と不変条件

  • 各 InitPlan は、トップレベルのプラン開始時に一度だけ実行されるものと仮定して計算されます。これは保守的な過大評価であり、実際には実行されない、あるいは遅延実行される可能性があります。

アルゴリズムまたは振る舞い

1. initplan_cost を 0、unsafe_initplans を false に初期化します。 2. init_plans リストを走査し、各 SubPlan について以下を実行します。

  • startup_cost と per_call_cost を加算して initplan_cost を更新します。
  • parallel_safe フラグを確認し、一つでも false があれば unsafe_initplans を true に設定します。

    • 3. 計算結果をポインタ引数 initplan_cost_p および unsafe_initplans_p に格納して終了します。

関連データ構造

  • SubPlan: InitPlan として扱われるノード構造体。startup_cost, per_call_cost, parallel_safe フィールドを参照します。

失敗モードと境界条件

  • init_plans が空リストの場合、コストは 0、安全フラグは false となります。

保守担当者向けメモ

  • 本関数はコスト見積もりの保守的なアプローチをとっています。将来的に実行タイミングの最適化が進んだ場合、この見積もりロジックの精度向上が検討される可能性があります。
