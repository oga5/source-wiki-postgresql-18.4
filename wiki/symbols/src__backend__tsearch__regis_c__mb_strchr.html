mb_strchr

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/regis.c:176

目的

マルチバイト文字列 str 内に、特定のマルチバイト文字（または文字列）c が含まれているかを判定する。

シグネチャまたは形状

static bool mb_strchr(char *str, char *c)

入力と出力

  • **入力**:
  • str: 検索対象のヌル終端文字列。
  • c: 検索する文字（マルチバイト対応）。
  • **出力**:
  • bool: c が str 内に見つかった場合は true、それ以外は false を返す。

前提条件と不変条件

  • str および c は有効なヌル終端文字列であること。
  • pg_mblen_cstr を使用するため、PostgreSQL のマルチバイト環境下で動作することを前提としている。

アルゴリズムまたは振る舞い

1. c のバイト長を pg_mblen_cstr で取得する。 2. str を先頭から順に走査する。 3. 現在の走査位置の文字長（pg_mblen_cstr）が c のバイト長と一致する場合のみ、バイト単位で内容を比較する。 4. 一致する文字が見つかるか、str の終端に達するまで繰り返す。 5. 一致した場合は true を返し、走査を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/regis.c 内の RS_execute 関数から、RegisNode のデータと照合するために呼び出される。
  • **呼び出し先**:
  • pg_mblen_cstr（src/backend/tsearch/ts_locale.h で定義）を呼び出し、マルチバイト文字の長さを取得する。

関連データ構造

  • RegisNode: 本関数は、この構造体の data フィールドに含まれる文字集合とのマッチングに使用される。

失敗モードと境界条件

  • str が空文字列の場合、false を返す。
  • c が空文字列の場合の挙動は未定義（pg_mblen_cstr の仕様に依存）。
  • マルチバイト文字の長さが一致しない場合、その位置の文字は不一致とみなされる。

保守担当者向けメモ

  • 本関数は標準の strchr と異なり、マルチバイト文字を正しく扱うためのラッパーとして機能する。
  • パフォーマンスは str の長さと文字数に依存する。頻繁に呼び出されるため、最適化が必要な場合は pg_mblen_cstr の呼び出しコストを考慮すること。
