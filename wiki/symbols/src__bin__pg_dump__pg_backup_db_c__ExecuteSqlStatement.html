ExecuteSqlStatement

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:216

目的

ArchiveHandle を介してデータベースに対して SQL 文を実行し、結果を検証する。

シグネチャまたは形状

void ExecuteSqlStatement(Archive *AHX, const char *query)

入力と出力

  • **入力**:
  • AHX: Archive 型のポインタ（内部で ArchiveHandle にキャストされる）。
  • query: 実行する SQL 文を含むヌル終端文字列。
  • **出力**: なし（成功時は正常終了、失敗時はプロセスを終了する）。

前提条件と不変条件

  • AHX は有効な ArchiveHandle 構造体としてキャスト可能であること。
  • AH->connection が有効な libpq 接続を保持していること。

アルゴリズムまたは振る舞い

1. AHX を ArchiveHandle 型にキャストする。 2. PQexec を使用して、現在の接続に対して query を実行する。 3. PQresultStatus を確認し、結果が PGRES_COMMAND_OK でない場合は die_on_query_failure を呼び出してプロセスを終了する。 4. PQclear を呼び出し、実行結果のメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の各所（設定コマンドの実行やデータダンプ時のクエリ発行など）。
  • **呼び出し先**: libpq の PQexec および PQclear、エラーハンドリング用の die_on_query_failure。

関連データ構造

  • ArchiveHandle: データベース接続情報（connection）を保持する構造体。
  • PGresult: libpq のクエリ実行結果を格納する構造体。

失敗モードと境界条件

  • クエリ実行が PGRES_COMMAND_OK 以外を返した場合、die_on_query_failure によりプログラムが異常終了する。

保守担当者向けメモ

  • 本関数はエラー発生時に即座にプロセスを終了させる設計であるため、呼び出し側でエラーを回復させることはできない。
  • ArchiveHandle の connection が NULL の場合や接続が切断されている場合の挙動については、libpq の仕様に依存する。
