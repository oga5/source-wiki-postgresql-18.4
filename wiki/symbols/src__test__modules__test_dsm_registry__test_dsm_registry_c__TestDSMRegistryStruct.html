TestDSMRegistryStruct

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_dsm_registry/test_dsm_registry.c:21

目的

PostgreSQLの動的共有メモリ（DSM）レジストリ機能のテストにおいて、共有メモリ上に配置されるデータ構造を提供し、値の保持と排他制御の検証を行う。

シグネチャまたは形状

typedef struct TestDSMRegistryStruct
{
	int			val;
	LWLock		lck;
} TestDSMRegistryStruct;

入力と出力

  • **入力**: 共有メモリセグメント確保時に本構造体のサイズ（sizeof(TestDSMRegistryStruct)）が使用される。
  • **出力**: 共有メモリ上のデータ領域として、整数値（val）およびロック制御用オブジェクト（lck）を保持する。

前提条件と不変条件

  • 共有メモリセグメント内に配置されることを前提とする。
  • lck メンバは、val へのアクセスを同期するための軽量ロック（LWLock）として機能する。

アルゴリズムまたは振る舞い

  • val メンバは共有メモリを介してプロセス間で共有される整数値を保持する。
  • lck メンバは、set_val_in_shmem（排他モード）および get_val_in_shmem（共有モード）によるアクセス制御の基盤となる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_dsm_registry/test_dsm_registry.c 内の tdr_attach_shmem 関数が、共有メモリ上のポインタを本構造体にキャストして利用する。
  • tdr_init_shmem 関数が、共有メモリ確保時のサイズ指定に使用する。
  • **呼び出し先**:
  • LWLock 型を保持しており、storage/lwlock.h で定義されたロック操作関数群の対象となる。

関連データ構造

  • tdr_state: 本構造体を指す静的ポインタとして、モジュール内で共有メモリ領域へのアクセスに使用される。

失敗モードと境界条件

  • 共有メモリの初期化に失敗した場合、本構造体へのアクセスは不正なメモリ参照を引き起こす可能性がある。
  • ロックの取得に失敗した場合、データの一貫性が保証されない。

保守担当者向けメモ

  • 本構造体はテスト用モジュールの一部であり、実運用環境での共有メモリ管理の最小構成例として参照すること。
  • lck メンバの初期化およびトランシェ登録は tdr_attach_shmem 内で行われるため、構造体定義自体には初期化ロジックは含まれない。
