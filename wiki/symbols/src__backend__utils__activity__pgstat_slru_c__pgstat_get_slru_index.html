pgstat_get_slru_index

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_slru.c:131

目的

SLRU（Simple Least Recently Used）キャッシュの名称（文字列）に対応する統計管理用のインデックス番号を返します。

シグネチャまたは形状

int pgstat_get_slru_index(const char *name)

入力と出力

  • **入力**: name (SLRUキャッシュを識別する文字列)
  • **出力**: int 型のインデックス値。

前提条件と不変条件

  • slru_names 配列が定義されており、SLRU_NUM_ELEMENTS がその要素数として正しく機能している必要があります。
  • 戻り値は常に 0 から SLRU_NUM_ELEMENTS - 1 の範囲内に収まります。

アルゴリズムまたは振る舞い

1. slru_names 配列を先頭から順に走査します。 2. 引数 name と一致する文字列が見つかった場合、そのインデックスを返します。 3. 一致するものが見つからなかった場合、配列の最後の要素（「その他」用インデックス）である SLRU_NUM_ELEMENTS - 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/slru.c:281
  • src/backend/utils/activity/pgstat_slru.c:51
  • **呼び出し先**:
  • strcmp (標準Cライブラリ)

関連データ構造

  • slru_names: SLRU名称を保持する静的配列（定義箇所は pgstat_slru.c 内）。
  • SLRU_NUM_ELEMENTS: SLRUの総数を定義する定数。

失敗モードと境界条件

  • 引数 name が slru_names に存在しない場合、エラーにはならず、デフォルトの「その他」インデックス（配列の末尾）が返されます。

保守担当者向けメモ

  • 新しいSLRUキャッシュタイプを追加した際は、slru_names 配列を更新する必要があります。
  • 本関数は文字列比較を行うため、頻繁に呼び出されるパスではパフォーマンスへの影響を考慮してください。
