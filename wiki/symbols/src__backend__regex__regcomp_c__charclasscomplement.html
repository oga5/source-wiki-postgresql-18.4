charclasscomplement

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regcomp.c:1518

目的

正規表現における補集合文字クラス（例: [^a-z]）に対応する NFA 遷移を生成する。

シグネチャまたは形状

static void charclasscomplement(struct vars *v, enum char_classes cls, struct state *lp, struct state *rp)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体。
  • cls: 対象とする文字クラスの列挙型。
  • lp, rp: 遷移の始点および終点となる NFA 状態。
  • **出力**: なし（副作用として NFA に遷移を追加）。

前提条件と不変条件

  • v が有効な vars 構造体であること。
  • NOERR() マクロにより、コンパイルエラー発生時は即座に処理が中断される。

アルゴリズムまたは振る舞い

1. newstate() で一時的なダミー状態 cstate を作成する。 2. cclasscvec() を呼び出し、指定された文字クラスに対応する cvec（文字ベクトル）を取得する。 3. subcolorcvec() を使用して、cstate 間で文字クラスに対応する遷移を構築する。 4. okcolors() を呼び出し、カラーマップの整合性を維持する。 5. colorcomplement() を呼び出し、cstate に構築された遷移の補集合を lp から rp への遷移として生成する。 6. dropstate() で一時的な cstate を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regcomp.c 内の複数箇所（1000行目、1706行目など）から呼び出される。
  • **呼び出し先**: newstate, cclasscvec, subcolorcvec, okcolors, colorcomplement, dropstate（いずれも regcomp.c または関連モジュール内の関数）。

関連データ構造

  • struct vars: コンパイルのコンテキスト。
  • struct state: NFA の状態。
  • struct cvec: 文字クラスの集合を保持する構造体。

失敗モードと境界条件

  • NOERR() マクロにより、内部関数（newstate 等）でエラーが発生した場合、呼び出し元へエラー状態が伝播する。
  • cclasscvec が取得できない場合や、メモリ不足等の状況で処理が中断される。

保守担当者向けメモ

  • 本関数は okcolors() を内部で呼び出すため、呼び出し側で重複して呼び出す必要はない。
  • 補集合の生成には一時的な状態遷移が必要となるため、cstate の作成と破棄が必須である。
