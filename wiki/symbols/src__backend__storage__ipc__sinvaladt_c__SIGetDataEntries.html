SIGetDataEntries

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/sinvaladt.c:472

目的

自プロセスの進捗に基づき、共有メモリ上の循環バッファから未読のキャッシュ無効化メッセージを抽出する。

シグネチャまたは形状

int SIGetDataEntries(SharedInvalidationMessage *data, int datasize)

入力と出力

  • **入力**:
  • data: メッセージを格納するバッファへのポインタ。
  • datasize: data 配列の最大要素数。
  • **出力**:
  • 読み取ったメッセージ数（整数）。
  • 状態がリセットされた場合は -1 を返す。

前提条件と不変条件

  • 呼び出し元は MyProcNumber が有効な状態であること。
  • SInvalReadLock を共有モードで取得し、読み取り終了後に解放する。
  • 読み取り進捗（nextMsgNum）は、共有メモリ上の maxMsgNum を超えない範囲で更新される。

アルゴリズムまたは振る舞い

1. stateP->hasMessages を確認し、メッセージがない場合は即座に 0 を返す。 2. SInvalReadLock を取得し、hasMessages を false にリセットする。 3. スピンロックを用いて共有メモリ上の最新メッセージ番号 maxMsgNum を取得する。 4. resetState が真の場合、nextMsgNum を max に更新し、フラグをクリアして -1 を返す。 5. nextMsgNum から max に達するまで、または datasize に達するまで、循環バッファからメッセージを data にコピーし、nextMsgNum を進める。 6. 全てのメッセージを読み終えた場合（nextMsgNum >= max）、signaled フラグを false にする。読み残しがある場合は hasMessages を true に戻す。 7. ロックを解放し、読み取ったメッセージ数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/sinval.c 内の関数群（sinval.c:58, sinval.c:98, sinval.c:122）。
  • **呼び出し先**:
  • LWLockAcquire/LWLockRelease（SInvalReadLock）
  • SpinLockAcquire/SpinLockRelease（msgnumLock）

関連データ構造

  • SISeg: 共有メモリ上のメイン構造体。
  • ProcState: 各バックエンドの進捗状態を保持する構造体。
  • SharedInvalidationMessage: 抽出されるメッセージの型。

失敗モードと境界条件

  • **バッファ溢れ**: datasize を超えるメッセージがある場合、読み取れる分だけを返し、残りは次回の呼び出しに持ち越す。
  • **リセット状態**: プロセスが遅延しすぎてバッファから溢れた場合、resetState が真となり、キャッシュの全破棄を促す -1 が返される。
  • **並列実行**: 複数のバックエンドが同時に SIGetDataEntries を呼び出しても、SInvalReadLock により安全に実行される。

保守担当者向けメモ

  • 本関数は共有ロックを取得するが、自身の ProcState を更新するため、他のバックエンドの ProcState と競合しない範囲で並列実行が許可されている。
  • hasMessages フラグの操作は、メッセージの取りこぼしを防ぐための重要な最適化である。
