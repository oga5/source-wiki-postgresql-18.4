PerformPortalClose

定義

  • 種別: function
  • 定義箇所: src/backend/commands/portalcmds.c:223

目的

指定された名前のポータルを破棄し、関連するリソースを解放する。

シグネチャまたは形状

void PerformPortalClose(const char *name)

入力と出力

  • **入力**: name (ポータル名。NULL の場合は全ポータルを対象とする)
  • **出力**: なし（成功時）。エラー発生時は例外をスローする。

前提条件と不変条件

  • name が空文字列 ("") の場合、ERRCODE_INVALID_CURSOR_NAME エラーが発生する。
  • name が指定されている場合、その名前のポータルが存在しなければならない。存在しない場合は ERRCODE_UNDEFINED_CURSOR エラーが発生する。

アルゴリズムまたは振る舞い

1. name が NULL の場合、PortalHashTableDeleteAll() を呼び出し、すべてのポータルを削除して終了する。 2. name が空文字列でないことを確認する。 3. GetPortalByName(name) を使用してポータルオブジェクトを取得する。 4. 取得したポータルが有効でない場合、エラーを報告する。 5. PortalDrop(portal, false) を呼び出し、ポータルを破棄する。この際、必要に応じて PortalCleanup が副作用として実行される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:701 (SQLコマンド処理経由)
  • **呼び出し先**:
  • PortalHashTableDeleteAll
  • GetPortalByName
  • PortalIsValid
  • PortalDrop (内部で PortalCleanup を呼び出す)

関連データ構造

  • Portal: ポータルの状態を保持する構造体。

失敗モードと境界条件

  • **空のポータル名**: プロトコルレベルの無名ポータルと競合するため、明示的に拒否される。
  • **存在しないポータル**: 指定された名前のポータルが見つからない場合、エラーとなる。

保守担当者向けメモ

  • 本関数は CLOSE SQLコマンドのバックエンド実装である。
  • PortalDrop を呼び出すことで、ポータルに関連付けられたメモリやエグゼキュータの状態が適切にクリーンアップされる。
