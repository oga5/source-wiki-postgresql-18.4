PlanCacheMode

定義

  • 種別: enum
  • 定義箇所: src/include/utils/plancache.h:31

目的

クエリ実行計画のキャッシュ生成戦略（汎用計画かカスタム計画か）を制御するためのモードを指定する。

シグネチャまたは形状

typedef enum
{
	PLAN_CACHE_MODE_AUTO,
	PLAN_CACHE_MODE_FORCE_GENERIC_PLAN,
	PLAN_CACHE_MODE_FORCE_CUSTOM_PLAN,
} PlanCacheMode;

入力と出力

  • 入力: なし（列挙型定数）
  • 出力: なし

前提条件と不変条件

  • PLAN_CACHE_MODE_AUTO: システムが自動的に汎用計画とカスタム計画のどちらが最適かを判断する。
  • PLAN_CACHE_MODE_FORCE_GENERIC_PLAN: 常に汎用計画（パラメータ値に依存しない計画）の使用を強制する。
  • PLAN_CACHE_MODE_FORCE_CUSTOM_PLAN: 常にカスタム計画（特定のパラメータ値に最適化された計画）の使用を強制する。

アルゴリズムまたは振る舞い

CachedPlanSource における計画選択ロジックにおいて、この列挙値が参照され、計画の再利用方針が決定される。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報には具体的な呼び出し元は記載されていない。
  • 呼び出し先: なし（列挙型定義のため）。

関連データ構造

  • CachedPlanSource: このモード設定を保持し、計画のライフサイクル管理に使用する構造体。

失敗モードと境界条件

  • 未定義の列挙値を指定した場合の動作は未定義。

保守担当者向けメモ

  • 本シンボルは src/backend/utils/cache/plancache.c における計画選択ロジックの根幹に関わる。変更時はキャッシュの無効化および再計画の挙動に影響を与えるため注意すること。
