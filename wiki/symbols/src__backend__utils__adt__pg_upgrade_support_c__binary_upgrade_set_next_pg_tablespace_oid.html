binary_upgrade_set_next_pg_tablespace_oid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_upgrade_support.c:41

目的

pg_upgrade 実行時に、次に作成されるテーブルスペースに対して特定の OID を強制的に割り当てるためのグローバル変数を設定します。

シグネチャまたは形状

Datum binary_upgrade_set_next_pg_tablespace_oid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して、次に割り当てるべきテーブルスペースの OID を受け取ります。
  • **出力**: なし（PG_RETURN_VOID() を返します）。

前提条件と不変条件

  • サーバーが binary_upgrade モードで起動されている必要があります。CHECK_IS_BINARY_UPGRADE マクロにより、この条件が満たされない場合はエラーが発生します。

アルゴリズムまたは振る舞い

1. 引数から OID を取得します。 2. CHECK_IS_BINARY_UPGRADE マクロを実行し、アップグレードモードであることを検証します。 3. グローバル変数 binary_upgrade_next_pg_tablespace_oid に取得した OID を代入します。 4. 処理を終了し、void を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dumpall.c 内の SQL 生成ロジックから呼び出されます。
  • **呼び出し先**: CHECK_IS_BINARY_UPGRADE マクロ（定義元は catalog/binary_upgrade.h 経由の可能性が高いが、本ソース内ではマクロとして展開）。

関連データ構造

  • binary_upgrade_next_pg_tablespace_oid: 次回のテーブルスペース作成時に使用される OID を保持するグローバル変数。

失敗モードと境界条件

  • binary_upgrade モード以外で呼び出された場合、CHECK_IS_BINARY_UPGRADE によりエラーが発生し、処理が中断されます。

保守担当者向けメモ

  • 本関数は pg_upgrade によるカタログの物理的な復元を支援するための管理用インターフェースです。誤用はシステムカタログの整合性を破壊するため、必ず binary_upgrade モード下でのみ使用されることを前提としています。
