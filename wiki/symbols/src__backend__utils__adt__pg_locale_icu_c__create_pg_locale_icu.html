create_pg_locale_icu

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_icu.c:141

目的

ICUライブラリを利用したロケールおよび照合順序（Collation）設定を保持する pg_locale_t 構造体を初期化し、生成する。

シグネチャまたは形状

pg_locale_t create_pg_locale_icu(Oid collid, MemoryContext context)

入力と出力

  • **入力**:
  • collid: 照合順序のOID。DEFAULT_COLLATION_OID の場合はデータベースのデフォルト設定を使用する。
  • context: 構造体割り当てに使用するメモリコンテキスト。
  • **出力**:
  • 初期化済みの pg_locale_t 構造体へのポインタ。

前提条件と不変条件

  • ビルド時に USE_ICU が定義されている必要がある。
  • collid に対応するシステムカタログ（pg_database または pg_collation）のレコードが存在すること。
  • ICUがサポートされていないビルド環境で呼び出された場合、エラーを発生させる。

アルゴリズムまたは振る舞い

1. **カタログ検索**: collid に応じて pg_database または pg_collation システムカタログからロケール文字列およびICUルールを取得する。 2. **コレータ生成**: make_icu_collator を呼び出し、取得したロケールとルールに基づいて UCollator を生成する。 3. **構造体初期化**: 指定された context 上に pg_locale_struct を割り当て、ICU関連情報を格納する。 4. **メソッド選択**: データベースのエンコーディングが PG_UTF8 であるかを確認し、最適化された collate_methods_icu_utf8 または汎用的な collate_methods_icu をメソッドテーブルとして割り当てる。 5. **結果返却**: 設定済みの構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c 内のロケール管理ロジックから呼び出される。
  • **呼び出し先**:
  • SearchSysCache1, SysCacheGetAttr, SysCacheGetAttrNotNull: カタログ情報取得。
  • make_icu_collator: ICUコレータの生成。
  • MemoryContextAllocZero, MemoryContextStrdup: メモリ確保。
  • GetDatabaseEncoding: エンコーディング判定。

関連データ構造

  • pg_locale_t (構造体 pg_locale_struct): ロケールと照合順序の情報を保持する。
  • UCollator: ICUの照合順序オブジェクト。
  • Form_pg_collation: pg_collation カタログの行構造。

失敗モードと境界条件

  • **キャッシュルックアップ失敗**: 指定されたOIDがカタログに存在しない場合、elog(ERROR) でトランザクションを中断する。
  • **ICU未サポート**: ビルド時にICUが有効でない場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーを報告する。

保守担当者向けメモ

  • 本関数は pg_locale_t のICUプロバイダー固有の初期化を担当する。
  • データベースエンコーディングがUTF-8の場合、比較処理の高速化のために専用のメソッドテーブルが選択される点に注意すること。
