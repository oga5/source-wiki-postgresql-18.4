InitializeSession

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/session.c:53

目的

現在のバックエンドプロセスにおけるセッション状態を初期化し、グローバル変数 CurrentSession を割り当てる。

シグネチャまたは形状

void InitializeSession(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 CurrentSession が初期化される）

前提条件と不変条件

  • TopMemoryContext が初期化済みであること。
  • 本関数はセッション開始時に一度だけ呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. TopMemoryContext 上に Session 構造体サイズのメモリをゼロクリアして確保する。 2. 確保したメモリ領域のポインタをグローバル変数 CurrentSession に代入する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c:1218
  • 呼び出し先: MemoryContextAllocZero（utils/memutils.h）

関連データ構造

  • Session: セッション情報を保持する構造体。
  • CurrentSession: セッション状態を保持するグローバル変数。
  • TopMemoryContext: メモリ割り当ての基点となるメモリコンテキスト。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocZero がエラーを発生させる。

保守担当者向けメモ

  • 本関数は並列クエリ実行時のセッション共有基盤の起点となる。
  • CurrentSession のライフサイクルはバックエンドプロセスの生存期間に依存する。
