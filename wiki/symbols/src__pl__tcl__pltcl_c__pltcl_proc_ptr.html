pltcl_proc_ptr

定義

  • 種別: struct
  • 定義箇所: src/pl/tcl/pltcl.c:206

目的

PL/Tcl において、コンパイル済み関数キャッシュ（ハッシュテーブル）の各エントリを保持するためのラッパー構造体です。

シグネチャまたは形状

typedef struct pltcl_proc_ptr
{
	pltcl_proc_key proc_key;	/* Hash key (must be first!) */
	pltcl_proc_desc *proc_ptr;
} pltcl_proc_ptr;

入力と出力

  • **入力**: 構造体内の proc_key はハッシュテーブルの検索キーとして機能し、proc_ptr は対応する関数記述子へのポインタを保持します。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • proc_key フィールドは構造体の先頭に配置される必要があります。これはハッシュテーブルのキーとして直接参照されるためです。

アルゴリズムまたは振る舞い

  • 本構造体は、PL/Tcl の関数キャッシュハッシュテーブルの要素として使用されます。
  • ハッシュテーブルの初期化時（src/pl/tcl/pltcl.c:460）に、hash_ctl.entrysize として本構造体のサイズが指定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/tcl/pltcl.c 内の関数キャッシュ管理ロジック。
  • **呼び出し先**: なし（データ構造体のため）。
  • 補足: src/pl/tcl/pltcl.c:1423 にて、本構造体へのポインタ proc_ptr がローカル変数として宣言されています。

関連データ構造

  • pltcl_proc_key: ハッシュテーブルの検索キー。
  • pltcl_proc_desc: 実際の関数定義やコンパイル済み状態を保持する構造体。

失敗モードと境界条件

  • 構造体定義自体に失敗モードはありませんが、ハッシュテーブルのサイズ計算時に sizeof(pltcl_proc_ptr) を使用するため、定義の変更はハッシュテーブルの整合性に直接影響します。

保守担当者向けメモ

  • proc_key が構造体の先頭にあるという制約は、ハッシュテーブルの実装に依存する重要な不変条件です。構造体のメンバ順序を変更する際は注意してください。
