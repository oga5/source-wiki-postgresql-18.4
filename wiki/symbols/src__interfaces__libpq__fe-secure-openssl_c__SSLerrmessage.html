SSLerrmessage

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-openssl.c:1553

目的

OpenSSL のエラーコード（unsigned long）を、人間が読み取り可能な文字列メッセージに変換する。

シグネチャまたは形状

static char *SSLerrmessage(unsigned long ecode)

入力と出力

  • **入力**: ecode (OpenSSL のエラーコード)
  • **出力**: エラー内容を記述した char * 型の文字列。メモリは malloc で確保される。

前提条件と不変条件

  • 呼び出し元は、返された文字列に対して適切なタイミングでメモリ解放を行う責任がある。
  • malloc が失敗した場合は、静的なエラー文字列 ssl_nomem が返される。

アルゴリズムまたは振る舞い

1. SSL_ERR_LEN 分のメモリを確保する。 2. ecode が 0 の場合、「no SSL error reported」を返す。 3. ERR_reason_error_string(ecode) を使用して標準的なエラー文字列を取得し、存在すればそれを返す。 4. 特定の TLS アラート（SSL_AD_NO_APPLICATION_PROTOCOL）に該当する場合、専用のメッセージを返す。 5. システムエラー（ERR_SYSTEM_ERROR）の場合は strerror_r を使用してシステムのエラーメッセージを取得する。 6. 上記のいずれにも該当しない場合、「SSL error code [ecode]」という形式の数値メッセージを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure-openssl.c 内の各箇所（証明書、鍵、CA ファイルの読み込みエラー時など）。
  • **呼び出し先**: ERR_reason_error_string, ERR_GET_LIB, ERR_GET_REASON, ERR_SYSTEM_ERROR, strerror_r, malloc, snprintf, strlcpy。

関連データ構造

  • SSL_ERR_LEN: エラーメッセージバッファのサイズを規定する定数。
  • ssl_nomem: メモリ確保失敗時に返される静的文字列。

失敗モードと境界条件

  • malloc が失敗した場合、ssl_nomem を返す。
  • ERR_reason_error_string が NULL を返す場合、フォールバック処理（TLS アラート判定やシステムエラー判定）へ移行する。
  • OpenSSL のバージョンや環境（OpenSSL 3.0.0 以降など）により、エラー解決のロジックが条件分岐する。

保守担当者向けメモ

  • 本関数は libpq のエラーハンドリングにおいて、OpenSSL のエラーキューから取得したコードを可読化する重要な役割を担う。
  • ERR_SYSTEM_ERROR マクロの有無は OpenSSL のバージョンに依存するため、移植性を考慮した実装となっている。
