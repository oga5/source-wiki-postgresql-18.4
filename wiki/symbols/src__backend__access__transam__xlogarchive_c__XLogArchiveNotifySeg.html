XLogArchiveNotifySeg

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:491

目的

指定されたWALセグメント番号とタイムラインIDからWALファイル名を生成し、アーカイブプロセスに対して当該ファイルのアーカイブが必要であることを通知する。

シグネチャまたは形状

void XLogArchiveNotifySeg(XLogSegNo segno, TimeLineID tli)

入力と出力

  • **入力**:
  • segno (XLogSegNo): 通知対象のWALセグメント番号。
  • tli (TimeLineID): 通知対象のタイムラインID。
  • **出力**: なし（void）。

前提条件と不変条件

  • tli は 0 であってはならない（Assert により検証済み）。
  • wal_segment_size が適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

1. tli が 0 でないことを確認する。 2. XLogFileName を呼び出し、引数として渡された tli と segno から、対応するWALファイル名（xlog）を生成する。 3. 生成したファイル名を引数として XLogArchiveNotify を呼び出し、アーカイブ状態を .ready に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c:2487
  • **呼び出し先**:
  • XLogFileName (ファイル名生成)
  • XLogArchiveNotify (アーカイブ通知)

関連データ構造

  • XLogSegNo: WALセグメント番号を表す型。
  • TimeLineID: タイムラインIDを表す型。
  • MAXFNAMELEN: ファイル名格納用バッファの最大長。

失敗モードと境界条件

  • tli が 0 の場合、アサーション違反によりプロセスが停止する可能性がある（デバッグビルド時）。
  • XLogArchiveNotify 内部でファイルシステム操作（.ready ファイルの作成）が失敗した場合、アーカイブ通知が正しく行われない可能性がある。

保守担当者向けメモ

  • 本関数は XLogArchiveNotify のラッパーであり、セグメント番号からファイル名を導出する責務を持つ。
  • ファイル名生成ロジックを変更する場合は XLogFileName の仕様を確認すること。
