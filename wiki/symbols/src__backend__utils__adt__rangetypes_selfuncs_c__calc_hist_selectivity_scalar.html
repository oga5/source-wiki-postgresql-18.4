calc_hist_selectivity_scalar

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/rangetypes_selfuncs.c:595

目的

範囲型の統計情報（ヒストグラム）に基づき、特定の境界値が分布全体の中でどの程度の位置（選択率）にあるかを推定します。

シグネチャまたは形状

static double calc_hist_selectivity_scalar(
    TypeCacheEntry *typcache, 
    const RangeBound *constbound,
    const RangeBound *hist, 
    int hist_nvalues, 
    bool equal
)

入力と出力

  • **入力**:
  • typcache: 型固有のキャッシュ情報。
  • constbound: 比較対象の範囲境界値。
  • hist: 境界ヒストグラムの配列。
  • hist_nvalues: ヒストグラムの要素数。
  • equal: 境界値の包含関係を示すフラグ。
  • **出力**:
  • double: 0.0 から 1.0 の範囲で表される推定選択率。

前提条件と不変条件

  • hist は昇順にソートされている必要があります。
  • hist_nvalues は 2 以上であることを前提としています（除算 hist_nvalues - 1 を含むため）。

アルゴリズムまたは振る舞い

1. **ビン探索**: rbound_bsearch を呼び出し、constbound がヒストグラムのどのビンに該当するかを特定します。 2. **基本選択率の算出**: 特定されたインデックスに基づき、先行する全ビンの割合を計算します。 3. **線形補間**: 該当ビン内での相対的な位置を get_position で取得し、ビン間の累積分布を線形補間して加算します。 4. **結果返却**: 最終的な選択率を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/multirangetypes_selfuncs.c 内の複数の箇所から、マルチレンジ型の選択率計算のために呼び出されています。
  • **呼び出し先**:
  • rbound_bsearch: ヒストグラム内の位置特定。
  • get_position: ビン内での相対位置の計算。

関連データ構造

  • RangeBound: 範囲の境界値を保持する構造体。
  • TypeCacheEntry: 型固有の比較関数やサブ差分関数を保持するキャッシュ構造体。

失敗モードと境界条件

  • hist_nvalues が 1 以下の場合、除算によるゼロ除算が発生する可能性があります。
  • get_position が型固有の subdiff 関数に依存しているため、対象の型で subdiff が未定義または不適切な場合、推定精度が低下します。

保守担当者向けメモ

  • 本関数は「範囲の長さ」と「下限値」が独立しているという仮定に基づいています。
  • get_position の戻り値は、型キャッシュ内の subdiff 関数に依存するため、カスタム範囲型を追加する際は subdiff の実装が重要です。
