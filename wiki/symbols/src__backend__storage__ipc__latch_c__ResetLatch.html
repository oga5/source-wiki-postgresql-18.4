ResetLatch

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/latch.c:373

目的

ラッチの状態をクリアし、待機ループの開始準備を行うために使用されます。

シグネチャまたは形状

void ResetLatch(Latch *latch)

入力と出力

  • **入力**: latch (操作対象の Latch 構造体へのポインタ)
  • **出力**: なし（副作用として latch->is_set が false に設定される）

前提条件と不変条件

  • 呼び出し元プロセスが当該ラッチの所有者であること (latch->owner_pid == MyProcPid)。
  • ラッチがスリープ状態ではないこと (latch->maybe_sleeping == false)。
  • 上記条件は Assert により検証されます。

アルゴリズムまたは振る舞い

1. 所有権とスリープ状態の整合性を検証します。 2. latch->is_set を false に設定します。 3. pg_memory_barrier() を実行し、is_set の更新がメインメモリにフラッシュされることを保証します。これにより、並行して実行される SetLatch がフラグ更新を見逃す競合を防ぎます。

  • **呼び出し先**:

関連データ構造

  • Latch: src/include/storage/latch.h で定義される、ラッチの状態（is_set 等）を保持する構造体。

失敗モードと境界条件

  • 所有者以外のプロセスから呼び出された場合、Assert が有効なビルドではプロセスが停止します。
  • WaitLatch の直前に呼び出さない場合、シグナルが ResetLatch の直前に到着した際にイベントを見逃す競合が発生する可能性があります。

保守担当者向けメモ

  • WaitLatch を使用するループ構造では、必ず WaitLatch の呼び出し前に本関数を配置してください。
  • SetLatch との競合を避けるため、メモリバリアの順序は厳密に維持する必要があります。
