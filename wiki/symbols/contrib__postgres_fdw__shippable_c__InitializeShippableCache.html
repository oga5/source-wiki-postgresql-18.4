InitializeShippableCache

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/shippable.c:90

目的

リモートサーバーへのオブジェクト実行委譲（プッシュダウン）判定結果を保持するためのハッシュテーブルを初期化し、キャッシュ無効化のためのコールバックを登録します。

シグネチャまたは形状

static void InitializeShippableCache(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 ShippableCacheHash を初期化し、システムキャッシュコールバックを登録）

前提条件と不変条件

  • 本関数は、キャッシュが未初期化の状態で呼び出されることを前提としています。
  • ShippableCacheHash がグローバル変数として定義されている必要があります。

アルゴリズムまたは振る舞い

1. ShippableCacheKey をキー、ShippableCacheEntry をエントリサイズとするハッシュテーブル ShippableCacheHash を作成します（初期サイズ 256）。 2. CacheRegisterSyscacheCallback を使用し、FOREIGNSERVEROID システムカタログの変更を監視します。 3. pg_foreign_server が更新された際に InvalidateShippableCacheCallback が実行されるよう登録します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/postgres_fdw/shippable.c 内の関数（キャッシュへのアクセス時に遅延初期化として呼び出される）。
  • 呼び出し先: hash_create（ハッシュテーブル作成）、CacheRegisterSyscacheCallback（コールバック登録）。

関連データ構造

  • ShippableCacheHash: 判定結果を保持するハッシュテーブル。
  • ShippableCacheKey: キャッシュの検索キー。
  • ShippableCacheEntry: キャッシュのエントリ構造体。
  • InvalidateShippableCacheCallback: キャッシュ無効化時に呼び出される関数。

失敗モードと境界条件

  • メモリ不足時には hash_create がエラーを発生させます。
  • 本関数は複数回呼び出される可能性があるため、キャッシュの二重初期化を防ぐガードロジックが呼び出し元で適切に管理されている必要があります。

保守担当者向けメモ

  • 本関数は postgres_fdw の最適化フェーズにおけるキャッシュ基盤を構築します。
  • pg_foreign_server の変更時にキャッシュを破棄する仕組みは、リモートサーバーの設定変更が即座にプッシュダウン判定に反映されることを保証します。
