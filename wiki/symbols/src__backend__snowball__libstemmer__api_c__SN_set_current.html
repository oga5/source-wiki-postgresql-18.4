SN_set_current

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/api.c:51

目的

ステミング処理対象の文字列を SN_env 環境にロードし、処理開始位置を初期化します。

シグネチャまたは形状

int SN_set_current(struct SN_env * z, int size, const symbol * s)

入力と出力

  • **入力**:
  • z: ステミング環境を保持する SN_env 構造体へのポインタ。
  • size: 入力文字列 s のサイズ。
  • s: ステミング対象となる文字列（symbol 型の配列）。
  • **出力**:
  • int: replace_s の実行結果（成功時は 0、失敗時は非 0 のエラーコード）。

前提条件と不変条件

  • z は SN_create_env によって適切に初期化されている必要があります。
  • 内部バッファの更新後、カーソル位置 z->c は必ず 0 にリセットされます。

アルゴリズムまたは振る舞い

1. replace_s(z, 0, z->l, size, s, NULL) を呼び出し、環境 z 内の既存文字列を s で置き換えます。 2. 環境内のカーソル位置 z->c を 0 に設定します。 3. replace_s の戻り値（エラーコード）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/dict_snowball.c:324 で呼び出されています。
  • **呼び出し先**: replace_s（utilities.c で定義）を呼び出します。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。
  • symbol: header.h で定義される文字型（システム依存）。

失敗モードと境界条件

  • replace_s がメモリ確保等で失敗した場合、そのエラーコードがそのまま返されます。
  • size が負の値である場合や、s が NULL の場合の挙動は replace_s の実装に依存します。

保守担当者向けメモ

  • 本関数は SN_env の状態を直接変更するため、並行処理を行う場合は呼び出し元で排他制御が必要です。
  • replace_s の戻り値を呼び出し元で適切にチェックし、エラー発生時のメモリリークや不正な状態遷移を防いでください。
