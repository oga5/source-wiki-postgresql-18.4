AtInplace_Inval

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/inval.c:1262

目的

トランザクションを伴わない「インプレース（In-place）」更新処理において、蓄積されたキャッシュ無効化メッセージを処理し、共有メモリへ伝播させる。

シグネチャまたは形状

void AtInplace_Inval(void)

入力と出力

  • 入力: なし（グローバル変数 inplaceInvalInfo を参照）
  • 出力: なし（副作用として共有メモリへのメッセージ送信およびリレーションキャッシュ初期化ファイルの無効化を行う）

前提条件と不変条件

  • CritSectionCount > 0 であること（クリティカルセクション内での実行が必須）。
  • inplaceInvalInfo が初期化されている場合にのみ処理が実行される。

アルゴリズムまたは振る舞い

1. inplaceInvalInfo が NULL の場合は即座に終了する。 2. inplaceInvalInfo->CurrentCmdInvalidMsgs に蓄積された無効化メッセージを ProcessInvalidationMessagesMulti を介して共有メモリへ送信する。 3. inplaceInvalInfo->RelcacheInitFileInval が真であれば、RelationCacheInitFilePostInvalidate() を呼び出し、リレーションキャッシュの初期化ファイルを無効化する。 4. 最後に inplaceInvalInfo を NULL にリセットする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heapam.c:6630
  • 呼び出し先: ProcessInvalidationMessagesMulti, RelationCacheInitFilePostInvalidate

関連データ構造

  • TransInvalidationInfo (グローバル変数 inplaceInvalInfo の型)

失敗モードと境界条件

  • クリティカルセクション外で呼び出された場合、Assert により停止する。
  • inplaceInvalInfo が NULL の場合、何も行わず正常終了する。

保守担当者向けメモ

  • 本関数は relmapper 等のトランザクション管理外の更新で使用される。通常のトランザクション処理（AtEOXact_Inval）とは異なるパスである点に注意すること。
