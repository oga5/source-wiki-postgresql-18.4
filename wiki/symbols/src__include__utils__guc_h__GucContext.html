GucContext

定義

  • 種別: enum
  • 定義箇所: src/include/utils/guc.h:71

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおいて、設定パラメータの変更が許可されるタイミングや適用範囲を定義するコンテキストを識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	PGC_INTERNAL,
	PGC_POSTMASTER,
	PGC_SIGHUP,
	PGC_SU_BACKEND,
	PGC_BACKEND,
	PGC_SUSET,
	PGC_USERSET,
} GucContext;

入力と出力

  • 入力: 設定変更操作（SetConfigOption 等）の引数として渡され、現在の実行環境のコンテキストと比較されます。
  • 出力: 設定変更が許可されるかどうかの判定基準として機能します。

前提条件と不変条件

  • 列挙型の順序は、設定の優先順位や適用可否を決定するロジックに直結しているため、変更には注意が必要です。
  • 設定変更時には、現在のコンテキストが対象パラメータの許可するコンテキストと一致、または上位である必要があります。

アルゴリズムまたは振る舞い

  • 設定変更のライフサイクル管理において、いつ設定が変更されたかを追跡します。
  • PGC_POSTMASTER はサーバー起動時にのみ設定可能なパラメータを示し、PGC_USERSET は通常のユーザーがセッション内で変更可能なパラメータを示します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の例）:
  • src/backend/tcop/postgres.c: set_debug_options, set_plan_disabling_options, process_postgres_switches
  • src/backend/utils/fmgr/fmgr.c: superuser() の判定結果に基づくコンテキスト選択
  • src/backend/utils/misc/guc.c: ProcessConfigFileInternal 等の内部処理
  • 呼び出し先: 本シンボルは列挙型であるため、関数としての呼び出し先はありません。

関連データ構造

  • GucSource: 設定値の優先順位を決定するソース（PGC_S_FILE, PGC_S_SESSION 等）。
  • ConfigVariable: 設定ファイル解析時に生成される名前・値ペアのリスト構造体。

失敗モードと境界条件

  • 許可されていないコンテキストで設定を変更しようとした場合、GUCシステムはエラーを発生させます。
  • 実行時のコンテキストとパラメータ定義のコンテキストの不一致は、設定の適用失敗を引き起こします。

保守担当者向けメモ

  • フック関数（CheckHook, AssignHook）の実装において、現在のコンテキストを考慮した妥当性チェックを行うことが重要です。
  • スレッドセーフティを確保するため、変更時のロックやコンテキストの整合性確認を怠らないでください。
