apw_load_buffers

定義

  • 種別: function
  • 定義箇所: contrib/pg_prewarm/autoprewarm.c:291

目的

サーバー再起動時に、ディスク上のダンプファイル（AUTOPREWARM_FILE）から以前の共有バッファ状態を読み込み、データベースごとにバックグラウンドワーカーを起動してプリウォーム処理を再開する。

シグネチャまたは形状

static void apw_load_buffers(void)

入力と出力

  • **入力**: AUTOPREWARM_FILE（テキスト形式のブロック情報ダンプファイル）。
  • **出力**: なし（副作用として共有メモリ状態の更新およびデータベースごとのワーカー起動を行う）。

前提条件と不変条件

  • apw_state->lock を介した排他制御により、ダンプファイルへの同時アクセスを防止する。
  • プリウォーム処理は have_free_buffer() が真である間のみ継続される。
  • データベース接続が不可能なグローバルオブジェクトのみのブロック情報はスキップされる。

アルゴリズムまたは振る舞い

1. **排他制御**: apw_state->pid_using_dumpfile を確認し、他プロセスがダンプファイルを使用中の場合は処理を中断する。 2. **読み込み**: AUTOPREWARM_FILE を開き、全レコードを動的共有メモリ（DSM）上に展開する。 3. **ソート**: qsort を使用し、apw_compare_blockinfo に基づいてブロック情報をソートする。 4. **ワーカー起動**: データベースごとにグループ化し、apw_start_database_worker() を呼び出してプリウォームを実行する。 5. **クリーンアップ**: DSM セグメントを解放し、apw_state のロックを解除して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: autoprewarm_main（contrib/pg_prewarm/autoprewarm.c:224）。
  • **呼び出し先**: apw_start_database_worker、dsm_create、dsm_detach、AllocateFile、FreeFile、LWLockAcquire、LWLockRelease、have_free_buffer。

関連データ構造

  • BlockInfoRecord: 読み込まれる各ブロックのメタデータ構造体。
  • AutoPrewarmSharedState: プリウォームの進捗やワーカー状態を管理する共有メモリ構造体。

失敗モードと境界条件

  • **ファイル不在**: AUTOPREWARM_FILE が存在しない場合はエラーとせず静かに終了する。
  • **ファイル破損**: fscanf による読み込みが期待した形式と異なる場合、ERROR を発生させる。
  • **シャットダウン要求**: ShutdownRequestPending が真になった場合、ワーカーの起動を停止し、処理を中断する。
  • **バッファ枯渇**: have_free_buffer() が偽の場合、それ以上のワーカー起動を停止する。

保守担当者向けメモ

  • 本関数は apw_state->lock を保持したまま apw_start_database_worker を呼び出すことはないが、処理の各ステップで ShutdownRequestPending をチェックし、安全に中断できるように設計されている。
  • ダンプファイル形式はテキストベースであり、<<%d>> で始まるレコード数と、それに続く CSV 形式のブロック情報で構成される。
