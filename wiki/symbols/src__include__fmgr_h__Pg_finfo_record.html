Pg_finfo_record

定義

  • 種別: struct
  • 定義箇所: src/include/fmgr.h:394

目的

C 言語で実装された PostgreSQL 関数が、どの呼び出し規約バージョン（ABI バージョン）を使用しているかを宣言するためのメタデータ構造体です。

シグネチャまたは形状

typedef struct
{
    int api_version; /* 呼び出し規約のバージョン番号 */
} Pg_finfo_record;

入力と出力

  • **入力**: api_version フィールドに整数値を保持します。
  • **出力**: なし（本構造体は関数呼び出しのメタデータとして参照されます）。

前提条件と不変条件

  • api_version は、PostgreSQL の関数マネージャがサポートする呼び出し規約のバージョンと一致している必要があります。
  • バージョン 1 の関数では、通常 1 が設定されます。

アルゴリズムまたは振る舞い

  • 関数マネージャ（fmgr）は、動的ライブラリからこの構造体を取得し、api_version を検証することで、関数呼び出しが安全に行えるかを判断します。
  • PG_FUNCTION_INFO_V1 マクロを使用することで、この構造体の定義と初期化が自動的に行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/fmgr/fmgr.c: fmgr_info 等の関数が、ロードされた関数の情報を取得するために参照します。
  • src/include/fmgr.h: fetch_finfo_record 関数が外部ライブラリからこのレコードを取得します。
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • PGFInfoFunction: Pg_finfo_record を返す関数ポインタ型。
  • FmgrInfo: 関数呼び出しに必要なメタデータを保持する構造体。

失敗モードと境界条件

  • api_version がバックエンドの期待する値と一致しない場合、関数マネージャは呼び出しを拒否し、エラーを発生させます。
  • 動的ライブラリ内に対応するシンボルが存在しない場合、fetch_finfo_record は失敗します。

保守担当者向けメモ

  • 将来的にバージョン番号が 1 を超える場合、フィールドが追加される可能性があります。
  • PG_FUNCTION_INFO_V1 マクロ経由で利用することが推奨されており、手動での構造体定義は避けてください。
