TupleTableSlot

定義

  • 種別: typedef
  • 定義箇所: src/include/foreign/fdwapi.h:48

目的

PostgreSQL の実行エンジンにおいて、タプル（行データ）を一時的に保持および管理するための汎用的なコンテナ構造体です。FDW（外部データラッパー）においては、外部ソースから取得した行データをエンジンへ渡す際の受け渡し口として機能します。

シグネチャまたは形状

構造体の内部定義は本ヘッダーファイルには含まれていませんが、FDW API においては ForeignScanState や各種データ操作関数（IterateForeignScan 等）の引数や戻り値として頻繁に参照されます。

入力と出力

  • **入力**: FDW のスキャン関数や更新関数において、外部ソースから読み取ったデータがこのスロットに格納されます。
  • **出力**: エンジン側のノード（ForeignScanState 等）は、このスロットを介してタプルデータにアクセスします。

前提条件と不変条件

  • FDW の実行フェーズにおいて、IterateForeignScan はタプルを取得するたびに TupleTableSlot にその内容を格納する必要があります。
  • スロットは、実行ノードのライフサイクルに合わせて適切に初期化および解放される必要があります。

アルゴリズムまたは振る舞い

FDW の実行フローにおける振る舞いは以下の通りです： 1. BeginForeignScan でスロットが準備されます。 2. IterateForeignScan が呼び出されるたびに、外部ソースから取得したタプルがスロットに格納されます。 3. RecheckForeignScan などの関数では、既にスロットに格納されているタプルに対して再評価（フィルタ条件のチェック等）が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/file_fdw や contrib/postgres_fdw などの FDW 実装において、スキャン関数や更新関数（postgresExecForeignInsert 等）から参照されます。
  • **呼び出し先**: 本シンボル自体はデータ構造であるため、特定の関数を呼び出すことはありません。

関連データ構造

  • ForeignScanState: スキャン実行時の状態を保持し、スロットを管理します。
  • FdwRoutine: FDW が実装すべきインターフェースを定義する構造体であり、この中で TupleTableSlot を扱う関数ポインタが定義されています。

失敗モードと境界条件

  • スロットが空（NULL）の状態でタプルへのアクセスを試みた場合、実行時エラーが発生する可能性があります。
  • 外部ソースから取得したデータがスロットの定義と一致しない場合、データ変換エラーや不整合が生じる可能性があります。

保守担当者向けメモ

  • TupleTableSlot の詳細な内部構造については、src/include/nodes/execnodes.h などの実行ノード関連のヘッダーファイルを参照してください。
  • FDW 開発時には、メモリリークを防ぐため、スロットのライフサイクル管理が EState（実行状態）と正しく同期されていることを確認してください。
