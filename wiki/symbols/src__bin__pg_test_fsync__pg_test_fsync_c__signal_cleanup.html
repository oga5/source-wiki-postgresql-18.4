signal_cleanup

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:596

目的

プログラムの割り込み（SIGINT, SIGTERM, SIGHUP）発生時に、テスト用の一時ファイルを削除し、プロセスを安全に終了させるためのシグナルハンドラ。

シグネチャまたは形状

static void signal_cleanup(SIGNAL_ARGS);

入力と出力

  • 入力: SIGNAL_ARGS（シグナルハンドラとして呼び出される際の引数）。
  • 出力: なし（_exit(1) によりプロセスを終了する）。

前提条件と不変条件

  • グローバル変数 needs_unlink が真であれば、filename で指定されたファイルが存在するとみなして削除を試みる。
  • 標準出力への改行出力は、エラーが発生しても無視される。

アルゴリズムまたは振る舞い

1. needs_unlink が真の場合、unlink(filename) を実行して一時ファイルを削除する。 2. 標準出力 (STDOUT_FILENO) に改行文字 \n を書き込み、出力行を完結させる。 3. _exit(1) を呼び出し、プロセスを即座に終了する。

呼び出し元と呼び出し先

  • 呼び出し元: pqsignal を介して SIGINT, SIGTERM, SIGHUP シグナルから呼び出される。
  • 呼び出し先: unlink, write, _exit（いずれも標準システムコール）。

関連データ構造

  • needs_unlink: ファイル削除が必要かを示すフラグ（グローバル変数）。
  • filename: 削除対象のファイルパスを保持する文字列（グローバル変数）。

失敗モードと境界条件

  • unlink の失敗: 内部的にエラーは無視されるため、ファイル削除に失敗しても処理は継続される。
  • write の失敗: 標準出力への改行書き込み失敗は無視される。

保守担当者向けメモ

  • 本関数はシグナルハンドラとして動作するため、非同期シグナル安全（async-signal-safe）な関数のみを使用すること。
  • _exit() を使用しているため、標準の exit() と異なり atexit で登録された関数や標準ライブラリのクリーンアップ処理は実行されない点に注意。
