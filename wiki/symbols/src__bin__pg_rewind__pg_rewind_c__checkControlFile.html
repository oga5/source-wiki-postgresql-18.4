checkControlFile

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/pg_rewind.c:1005

目的

PostgreSQLの制御ファイル（ControlFileData）の整合性を、CRCチェックサムを用いて検証する。

シグネチャまたは形状

static void checkControlFile(ControlFileData *ControlFile)

入力と出力

  • **入力**: ControlFileData *ControlFile - 検証対象となる制御ファイル構造体へのポインタ。
  • **出力**: なし（戻り値なし）。検証に失敗した場合は pg_fatal を呼び出し、プログラムを終了させる。

前提条件と不変条件

  • 入力された ControlFile ポインタは有効であること。
  • ControlFile 構造体内の crc フィールドには、計算済みのチェックサム値が格納されていること。

アルゴリズムまたは振る舞い

1. INIT_CRC32C を使用してCRC計算用変数を初期化する。 2. COMP_CRC32C を使用し、ControlFile 構造体の先頭から crc フィールドの直前までのデータ範囲に対してCRCを計算する。 3. FIN_CRC32C で計算を完了させる。 4. EQ_CRC32C を使用して、計算結果と構造体内の crc フィールド値を比較する。 5. 値が一致しない場合、pg_fatal を呼び出してエラーメッセージを出力し、プロセスを異常終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c:1048 にて呼び出されていることが確認できる。
  • **呼び出し先**: INIT_CRC32C, COMP_CRC32C, FIN_CRC32C, EQ_CRC32C（CRC計算用マクロ）、および pg_fatal（エラー終了用関数）。
  • 注: ツールが提示した src/backend/postmaster/postmaster.c 内の同名関数は、本シンボルとは別の定義（バックエンド側の実装）であり、本関数の呼び出し元ではない。

関連データ構造

  • ControlFileData: catalog/pg_control.h で定義される、PostgreSQLの制御ファイル情報を保持する構造体。

失敗モードと境界条件

  • **失敗モード**: 制御ファイルのデータが破損している、あるいは読み込み時に不整合が生じている場合、CRC不一致により pg_fatal が発生する。
  • **境界条件**: 構造体の crc フィールド自体は計算対象外（offsetof を使用して除外）であるため、構造体のレイアウト変更には依存しない。

保守担当者向けメモ

  • 本関数は pg_rewind ユーティリティ専用の検証ロジックである。バックエンド側の同名関数と混同しないよう注意すること。
  • CRC計算アルゴリズムの変更が必要な場合は、src/include/port/pg_crc32c.h 等の基盤定義と整合性を取ること。
