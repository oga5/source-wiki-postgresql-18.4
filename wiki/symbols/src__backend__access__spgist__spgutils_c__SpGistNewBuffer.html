SpGistNewBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgutils.c:393

目的

SP-GiSTインデックスにおいて、新しいページ（バッファ）を確保します。既存の空き領域（FSM）を再利用するか、必要に応じてリレーションを拡張して新規ページを割り当てます。

シグネチャまたは形状

Buffer SpGistNewBuffer(Relation index)

入力と出力

  • **入力**: index (対象となるSP-GiSTインデックスのリレーション)
  • **出力**: 確保されたページの Buffer（バッファ記述子）

前提条件と不変条件

  • SpGistBlockIsFixed で判定される固定ページ（メタページ等）は、FSMから取得された場合でも再利用対象から除外されます。
  • 取得したバッファは、呼び出し元で適切に初期化または使用されることが想定されています。

アルゴリズムまたは振る舞い

1. **FSM検索**: GetFreeIndexPage を使用して、FSMから空きページブロック番号を取得します。 2. **再利用判定**:

  • 取得したブロックが固定ページであればスキップします。
  • ReadBuffer で読み込み、ConditionalLockBuffer でロックを試みます。
  • ページが「新規（PageIsNew）」「削除済み（SpGistPageIsDeleted）」「空（PageIsEmpty）」のいずれかであれば、そのバッファを返します。
  • 再利用不可の場合はロックを解除し、次のFSM候補を探します。

    • 3. **拡張**: FSMから適切なページが見つからない場合、ExtendBufferedRel を呼び出してリレーションを物理的に拡張し、新しいバッファを確保して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spginsert.c (インデックス初期化時)
  • src/backend/access/spgist/spgutils.c (SpGistGetBuffer 内)
  • **呼び出し先**:
  • GetFreeIndexPage, SpGistBlockIsFixed, ReadBuffer, ConditionalLockBuffer, BufferGetPage, PageIsNew, SpGistPageIsDeleted, PageIsEmpty, LockBuffer, ReleaseBuffer, ExtendBufferedRel

関連データ構造

  • Relation: インデックスのメタデータおよび物理構造へのアクセスに使用。
  • Buffer: バッファマネージャが管理するページ記述子。

失敗モードと境界条件

  • FSMに空きがない場合、自動的にリレーションの拡張が行われます。
  • 他のプロセスが並行してページを再利用しようとした場合、ConditionalLockBuffer が失敗し、別の候補を探すか拡張処理へ移行します。

保守担当者向けメモ

  • 本関数はFSMを優先的に利用しますが、競合が発生した場合は即座にロックを諦めて次の候補へ進む設計になっています。
  • ExtendBufferedRel による拡張は排他的な操作を伴うため、頻繁な呼び出しはパフォーマンスに影響を与える可能性があります。
