PageGetItemIdCareful

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_gin.c:760

目的

GINインデックスページ内の指定されたオフセットにある ItemId を取得し、その内容が物理的に妥当かつGINインデックスの制約に適合しているかを検証する。

シグネチャまたは形状

static ItemId PageGetItemIdCareful(Relation rel, BlockNumber block, Page page, OffsetNumber offset)

入力と出力

  • **入力**:
  • rel: 対象のインデックスリレーション。
  • block: ページ番号（エラー報告用）。
  • page: 検査対象のページポインタ。
  • offset: 取得するアイテムのオフセット番号。
  • **出力**:
  • 妥当性が確認された ItemId 構造体へのポインタ。
  • **戻り値**: 成功時は ItemId を返し、検証失敗時は ereport(ERROR, ...) により処理を中断する。

前提条件と不変条件

  • GINインデックスのページレイアウトが標準的な形式に従っていること。
  • ItemId が指し示すデータ領域が、ページ内の有効なタプル領域（BLCKSZ から GinPageOpaqueData を除いた範囲）を超えていないこと。
  • GINインデックスでは使用されない LP_REDIRECT、LP_UNUSED、LP_DEAD フラグが設定されていないこと。
  • ItemId の長さが0ではないこと。

アルゴリズムまたは振る舞い

1. PageGetItemId を呼び出し、指定されたオフセットの ItemId を取得する。 2. ItemIdGetOffset と ItemIdGetLength を加算し、ページ末尾の GinPageOpaqueData 領域を侵害していないか確認する。侵害している場合は ERRCODE_INDEX_CORRUPTED を送出する。 3. ItemId のフラグ（Redirect, Unused, Dead）および長さ（0）をチェックし、GINで許可されない状態であれば ERRCODE_INDEX_CORRUPTED を送出する。 4. すべてのチェックを通過した場合にのみ ItemId を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c 内の複数箇所（行459, 506, 744）。
  • **呼び出し先**:
  • PageGetItemId (マクロ)
  • ItemIdGetOffset, ItemIdGetLength, ItemIdGetFlags, ItemIdIsRedirected, ItemIdIsUsed, ItemIdIsDead (マクロ)
  • ereport

関連データ構造

  • ItemId: ページ内のアイテム位置を管理する構造体。
  • GinPageOpaqueData: GINインデックスページ末尾の不透明データ領域。

失敗モードと境界条件

  • **破損検知**: ページ内のタプル領域外を指すポインタや、GINで禁止されているフラグが設定されている場合、ERRCODE_INDEX_CORRUPTED を伴うエラーを発生させる。
  • **境界条件**: offset がページ内の有効な範囲外である場合、PageGetItemId の挙動に依存する。

保守担当者向けメモ

  • 本関数は contrib/amcheck/verify_nbtree.c にも同名の関数が存在するが、GIN用は GinPageOpaqueData を考慮して境界チェックを行っている点が異なる。
  • 検証失敗時はインデックスの物理的な破損を示唆するため、呼び出し側で適切にエラーを伝播させること。
