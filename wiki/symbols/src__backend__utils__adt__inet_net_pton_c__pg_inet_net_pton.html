pg_inet_net_pton

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/inet_net_pton.c:60

目的

人間が読みやすい形式（プレゼンテーション形式）のネットワークアドレス文字列を、システム内部で扱うネットワークバイトオーダーのバイナリ形式へ変換します。

シグネチャまたは形状

int pg_inet_net_pton(int af, const char *src, void *dst, size_t size);

入力と出力

  • **入力**:
  • af: アドレスファミリー（PGSQL_AF_INET または PGSQL_AF_INET6）。
  • src: 変換対象の文字列（IPアドレスまたはCIDR表記）。
  • dst: 変換後のバイナリデータを格納するバッファ。
  • size: 処理モードを指定する値。-1 の場合はホストアドレスとして扱い、それ以外は CIDR ネットワークのビット長として扱います。
  • **出力**:
  • 成功時: ネットワークのビット長（CIDRの場合）または 0 以上の値。
  • 失敗時: -1 を返し、errno に EAFNOSUPPORT 等のエラーコードを設定します。

前提条件と不変条件

  • af に指定されたアドレスファミリーがサポート対象外の場合、関数は即座に失敗します。
  • size が -1 の場合、CIDR 表記のパースは行われず、純粋なホストアドレス変換として動作します。

アルゴリズムまたは振る舞い

1. af に基づき、IPv4 (PGSQL_AF_INET) か IPv6 (PGSQL_AF_INET6) かを判定します。 2. size 引数の値を確認します。

  • size == -1: inet_net_pton_ipv4 または inet_net_pton_ipv6 を呼び出し、ホストアドレスとして変換します。
  • size != -1: inet_cidr_pton_ipv4 または inet_cidr_pton_ipv6 を呼び出し、CIDR ネットワークとして変換します。

    • 3. サポートされていない af が渡された場合は errno に EAFNOSUPPORT を設定して -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/network.c 内の関数から呼び出されることが確認されています。
  • **呼び出し先**:
  • inet_net_pton_ipv4 / inet_net_pton_ipv6
  • inet_cidr_pton_ipv4 / inet_cidr_pton_ipv6

関連データ構造

  • 本関数は inet 型や cidr 型の内部表現を構築するための基盤として利用されます。

失敗モードと境界条件

  • 不正なフォーマットの文字列が入力された場合、各変換関数（inet_net_pton_ipv4 等）の仕様に従いエラーを返します。
  • 未定義のアドレスファミリーが指定された場合、errno が EAFNOSUPPORT に設定されます。

保守担当者向けメモ

  • 本関数は ISC ライセンス由来のコードを含んでおり、PostgreSQL 固有のメモリ管理ではなく標準的な errno を使用します。
  • size 引数に -1 を渡すインターフェースは本モジュール特有の設計であるため、呼び出し元の意図を十分に確認してください。
