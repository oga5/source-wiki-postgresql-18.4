ExecCheckPermissions

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execMain.c:581

目的

クエリ実行時に必要なリレーションに対するアクセス権限を検証します。

シグネチャまたは形状

bool ExecCheckPermissions(List *rangeTable, List *rteperminfos, bool ereport_on_violation)

入力と出力

  • **入力**:
  • rangeTable: クエリのレンジテーブルリスト。
  • rteperminfos: 検証対象の権限情報リスト（RTEPermissionInfo）。
  • ereport_on_violation: 権限違反時にエラーを報告するかどうかを指定するフラグ。
  • **出力**:
  • 権限チェックが成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • rteperminfos 内の各 RTEPermissionInfo は有効な relid を保持している必要があります。
  • USE_ASSERT_CHECKING が有効な場合、rangeTable と rteperminfos の整合性が検証されます。具体的には、perminfoindex を持つ RangeTblEntry がリレーションまたはビューであること、およびインデックスの重複がないことが保証されます。

アルゴリズムまたは振る舞い

1. **整合性チェック**（デバッグビルド時のみ）: rangeTable を走査し、rteperminfos との対応関係およびインデックスの整合性を確認します。 2. **権限検証**: rteperminfos リストを反復し、各要素に対して ExecCheckOneRelPerms を呼び出します。 3. **違反時の処理**: ExecCheckOneRelPerms が false を返した場合、ereport_on_violation が true であれば aclcheck_error を呼び出してエラーを発生させ、そうでなければ false を返して終了します。 4. **フックの実行**: ExecutorCheckPerms_hook が設定されている場合、そのフックを呼び出し、その結果を最終的な戻り値とします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copy.c
  • src/backend/executor/execMain.c
  • src/backend/parser/parse_relation.c
  • src/backend/utils/adt/ri_triggers.c
  • **呼び出し先**:
  • ExecCheckOneRelPerms
  • aclcheck_error
  • ExecutorCheckPerms_hook（存在する場合）

関連データ構造

  • RangeTblEntry: クエリ内のリレーション定義。
  • RTEPermissionInfo: 特定のリレーションに対する権限要件を保持する構造体。

失敗モードと境界条件

  • 権限不足: ExecCheckOneRelPerms が失敗し、ereport_on_violation が true の場合はエラーが発生します。
  • フックによる上書き: ExecutorCheckPerms_hook が設定されている場合、標準の権限チェックロジックとは異なる挙動や結果となる可能性があります。

保守担当者向けメモ

  • 本関数は ExecutorCheckPerms_hook を介して外部モジュールから介入可能です。デバッグ時にはフックの有無を確認してください。
  • USE_ASSERT_CHECKING 下での整合性チェックは、開発時のデータ構造の不整合を早期発見するために重要です。
