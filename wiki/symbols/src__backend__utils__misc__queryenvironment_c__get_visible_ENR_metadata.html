get_visible_ENR_metadata

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/queryenvironment.c:44

目的

指定されたクエリ環境（QueryEnvironment）から、名前（refname）に対応する一時的な名前付きリレーション（ENR）のメタデータを取得します。

シグネチャまたは形状

EphemeralNamedRelationMetadata get_visible_ENR_metadata(QueryEnvironment *queryEnv, const char *refname)

入力と出力

  • **入力**:
  • queryEnv: 検索対象のクエリ環境へのポインタ。
  • refname: 検索するリレーションの名前（文字列）。
  • **出力**:
  • 該当する ENR が見つかった場合はそのメタデータ（EphemeralNamedRelationMetadata）へのポインタを返します。
  • 見つからない場合、または queryEnv が NULL の場合は NULL を返します。

前提条件と不変条件

  • refname は NULL であってはなりません（Assert により検証されます）。
  • queryEnv が NULL の場合、関数は安全に NULL を返します。

アルゴリズムまたは振る舞い

1. refname が NULL でないことを確認します。 2. queryEnv が NULL であれば直ちに NULL を返します。 3. get_ENR(queryEnv, refname) を呼び出し、環境内から該当する EphemeralNamedRelation を検索します。 4. 検索結果が存在する場合、その構造体内のメタデータフィールド（md）へのポインタを返します。 5. 存在しない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_enr.c (2箇所で参照)
  • **呼び出し先**:
  • get_ENR (同ファイル内)

関連データ構造

  • QueryEnvironment: ENR を保持するコンテナ。
  • EphemeralNamedRelation: ENR の実体。
  • EphemeralNamedRelationMetadata: 返されるメタデータ構造体。

失敗モードと境界条件

  • queryEnv が NULL の場合、エラーにはならず NULL を返します。
  • 指定された refname が環境内に存在しない場合、NULL を返します。

保守担当者向けメモ

  • 本関数は get_ENR のラッパーとして機能し、メタデータへのアクセスを抽象化しています。
  • QueryEnvironment の内部実装（現在はリスト）が変更されても、本関数のインターフェースは維持される設計となっています。
