index_unchanged_by_update

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execIndexing.c:1011

目的

UPDATE操作において、特定のインデックスが論理的に変更されていないかを判定し、インデックス更新処理をスキップするためのヒント（indexUnchanged）を生成する。

シグネチャまたは形状

static bool index_unchanged_by_update(ResultRelInfo *resultRelInfo, EState *estate, IndexInfo *indexInfo, Relation indexRelation)

入力と出力

  • **入力**:
  • resultRelInfo: 対象リレーション情報。
  • estate: 実行状態。
  • indexInfo: インデックスのメタデータ（キャッシュを含む）。
  • indexRelation: インデックスのリレーション。
  • **出力**:
  • bool: インデックスが変更されていない場合は true、変更されている可能性がある場合は false。

前提条件と不変条件

  • indexInfo->ii_CheckedUnchanged が true の場合、キャッシュされた indexInfo->ii_IndexUnchanged の値を即座に返す。
  • インデックスキー属性のみを対象とし、INCLUDEインデックスの非キー列の変更は本判定に影響を与えない。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: 既に判定済みであればキャッシュ値を返す。 2. **キー列の照合**: インデックスの各キー属性が、UPDATE対象の列（updatedCols および extraUpdatedCols）に含まれているかを確認する。

  • キー列が更新対象に含まれていれば、ii_IndexUnchanged を false に設定し終了する。

    • 3. **式インデックスの処理**: インデックスが式を含む場合、index_expression_changed_walker を使用して、更新対象列が式内で使用されているかを再帰的に走査する。 4. **結果の保存**: 判定結果を indexInfo->ii_IndexUnchanged に保存し、呼び出し元へ返す。

  • 部分インデックスの述語（predicate）は本判定では考慮しない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execIndexing.c 内の更新処理ルーチン（ExecInsertIndexTuples 等から参照される）。
  • **呼び出し先**:
  • ExecGetUpdatedCols, ExecGetExtraUpdatedCols
  • RelationGetIndexExpressions
  • index_expression_changed_walker
  • bms_is_member, bms_union, bms_free

関連データ構造

  • ResultRelInfo: 更新対象リレーションの実行時情報。
  • IndexInfo: インデックスの属性情報と判定キャッシュを保持。
  • Bitmapset: 更新された列の集合を表現。

失敗モードと境界条件

  • **式インデックス**: キー列が 0 以下（式）の場合、単純な列比較をスキップし、後続の式走査へ移行する。
  • **メモリ管理**: extraUpdatedCols が存在する場合のみ allUpdatedCols のメモリ確保と解放を行う。

保守担当者向けメモ

  • 本関数はパフォーマンス最適化のためのヒント生成を目的としており、ここで true を返してもインデックスの整合性が損なわれることはない（index_insert 側で最終的な更新判断が行われるため）。
  • 部分インデックスの述語が変更された場合でも、この関数は true を返す可能性がある点に注意すること。
