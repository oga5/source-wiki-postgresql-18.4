WaitForTerminatingWorkers

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/parallel.c:447

目的

すべての並列ワーカープロセス（またはスレッド）が終了するまで待機し、各ワーカーの終了状態を同期・更新する。

シグネチャまたは形状

static void WaitForTerminatingWorkers(ParallelState *pstate)

入力と出力

  • **入力**: ParallelState *pstate（並列実行状態を保持する構造体へのポインタ）
  • **出力**: なし（副作用として pstate 内のワーカー状態を更新）

前提条件と不変条件

  • pstate は有効なメモリ領域を指していること。
  • HasEveryWorkerTerminated(pstate) が真になるまでループが継続する。
  • 終了したワーカーのステータスは WRKR_TERMINATED に更新され、対応する te[] エントリは NULL に設定される。

アルゴリズムまたは振る舞い

1. HasEveryWorkerTerminated が真になるまでループを実行する。 2. **プラットフォーム依存の待機処理**:

  • **非Windows**: wait() を使用して子プロセスの終了を待機し、終了したPIDに対応する ParallelSlot を特定してPIDをクリアする。
  • **Windows**: WaitForMultipleObjects() を使用してスレッドの終了を待機し、終了したスレッドハンドルに対応する ParallelSlot を特定する。その後、CloseHandle でハンドルを解放し、スレッドIDをリセットする。

    • 3. 共通処理として、特定したスロットの workerStatus を WRKR_TERMINATED に更新し、pstate->te[j] を NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/parallel.c 内の複数箇所（行441、1080など）。
  • **呼び出し先**:
  • HasEveryWorkerTerminated
  • wait (非Windows)
  • WaitForMultipleObjects (Windows)
  • CloseHandle (Windows)
  • pg_malloc / free (Windows)

関連データ構造

  • ParallelState: 並列処理全体の管理構造体。
  • ParallelSlot: 各ワーカーのプロセスIDやスレッドハンドル、ステータスを保持する構造体。

失敗モードと境界条件

  • Windows環境において WaitForMultipleObjects が WAIT_FAILED を返した場合、Assert により停止する。
  • 待機対象のワーカーが pstate 内で見つからない場合、Assert(j < pstate->numWorkers) により停止する。

保守担当者向けメモ

  • Windows環境では pg_malloc でメモリを確保しているため、ループ内での free 漏れに注意すること。
  • プラットフォームごとのプロセス/スレッド管理の差異（pid vs hThread）がコードの分岐要因となっている。
