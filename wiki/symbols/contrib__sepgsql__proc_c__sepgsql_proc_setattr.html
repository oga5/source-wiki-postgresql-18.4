sepgsql_proc_setattr

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/proc.c:234

目的

関数（プロシージャ）の属性変更時におけるSELinuxアクセス制御の検証および、関連するスキーマ情報の更新を行う。

シグネチャまたは形状

void sepgsql_proc_setattr(Oid functionId)

入力と出力

  • **入力**: functionId (対象関数のOID)
  • **出力**: なし（権限拒否時はエラーをスロー）

前提条件と不変条件

  • pg_proc カタログに対する AccessShareLock を保持していること。
  • 対象の関数がシステムカタログ上に存在すること。

アルゴリズムまたは振る舞い

1. **カタログ取得**: systable_beginscan を用いて変更後の関数定義を、SearchSysCache1 を用いて変更前の関数定義を取得する。 2. **スキーマ更新**: 名前空間（pronamespace）や関数名が変更された場合、sepgsql_schema_remove_name、sepgsql_schema_add_name、sepgsql_schema_rename を呼び出し、スキーマレベルの管理情報を更新する。 3. **権限チェック**:

  • 基本権限として SEPG_DB_PROCEDURE__SETATTR を要求する。
  • proleakproof 属性が false から true に変更される場合、セキュリティ上の重要度が高いため、追加で SEPG_DB_PROCEDURE__INSTALL 権限を要求する。

    • 4. **監査**: getObjectIdentity を使用して対象関数の識別子を生成し、sepgsql_avc_check_perms を介してSELinuxポリシー判定を行う。 5. **クリーンアップ**: 取得したキャッシュおよびスキャンを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:245
  • **呼び出し先**:
  • table_open, systable_beginscan, systable_getnext, systable_endscan, table_close
  • SearchSysCache1, ReleaseSysCache
  • sepgsql_schema_remove_name, sepgsql_schema_add_name, sepgsql_schema_rename
  • getObjectIdentity, sepgsql_avc_check_perms

関連データ構造

  • pg_proc (カタログ)
  • Form_pg_proc (関数定義構造体)
  • ObjectAddress (SELinux権限チェック用オブジェクト情報)

失敗モードと境界条件

  • 指定された functionId がカタログに見つからない場合、elog(ERROR) を発生させる。
  • SELinuxポリシーにより権限が拒否された場合、sepgsql_avc_check_perms 内でエラーが発生する。

保守担当者向けメモ

  • proleakproof 属性の変更はセキュリティに直結するため、厳格な権限チェック（INSTALL権限）が課されている点に注意すること。
  • スキーマ情報の更新は、関数定義の変更がスキーマのセキュリティコンテキストに影響を与える場合に必要となる。
