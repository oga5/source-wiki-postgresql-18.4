QTNFree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_util.c:63

目的

QTNode ツリー構造を再帰的に解放し、関連付けられたメモリを動的に解放する。

シグネチャまたは形状

void QTNFree(QTNode *in)

入力と出力

  • **入力**: in (QTNode *) - 解放対象のツリーのルートノード。
  • **出力**: なし（void）。

前提条件と不変条件

  • in が NULL の場合、何もせず正常終了する。
  • 再帰的な処理を行うため、スタックオーバーフローを避けるための check_stack_depth() が呼び出される。
  • QTNode の各フィールド（word, child, valnode）は、対応するフラグ（QTN_WORDFREE, QTN_NEEDFREE）が設定されている場合にのみ pfree される。

アルゴリズムまたは振る舞い

1. in が NULL であれば即座にリターンする。 2. スタック深さをチェックする。 3. in->valnode->type が QI_VAL であり、かつ QTN_WORDFREE フラグが立っている場合、in->word を解放する。 4. in->valnode->type が QI_OPR の場合、子ノードに対して再帰的に QTNFree を呼び出す。 5. 子ノードの配列 in->child が存在すれば解放する。 6. QTN_NEEDFREE フラグが立っている場合、in->valnode を解放する。 7. 最後にノード自身 in を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_op.c
  • src/backend/utils/adt/tsquery_rewrite.c
  • **呼び出し先**:
  • check_stack_depth()
  • pfree()
  • QTNFree() (再帰呼び出し)

関連データ構造

  • QTNode: 全文検索クエリを表現するツリー構造体。

失敗モードと境界条件

  • **スタックオーバーフロー**: 再帰的な構造であるため、極端に深いツリーに対してはスタック不足が発生する可能性がある。
  • **メモリ管理**: フラグ（QTN_NEEDFREE, QTN_WORDFREE）が正しく設定されていない場合、メモリリークまたは二重解放が発生する可能性がある。

保守担当者向けメモ

  • 本関数は再帰的であるため、ツリーの深さ制限に注意すること。
  • QTNode のメモリ管理はフラグに依存しているため、ノードを構築する際のフラグ設定と本関数での解放ロジックに整合性を持たせること。
