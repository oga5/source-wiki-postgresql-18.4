ginxlogVacuumDataLeafPage

定義

  • 種別: struct
  • 定義箇所: src/include/access/ginxlog.h:143

目的

GINインデックスのデータリーフページに対するバキューム操作をWAL（Write-Ahead Log）に記録するための構造体です。

シグネチャまたは形状

typedef struct ginxlogVacuumDataLeafPage
{
    ginxlogRecompressDataLeaf data;
} ginxlogVacuumDataLeafPage;

入力と出力

  • **入力**: WALレコードのブロックデータとして格納されます。
  • **出力**: リカバリ処理時に、この構造体に含まれる ginxlogRecompressDataLeaf を用いて対象ページの再圧縮が行われます。

前提条件と不変条件

  • ginxlogRecompressDataLeaf 型のフィールド data を保持しており、この構造体自体がWALレコードのペイロードとして XLogRecGetBlockData を介して取得されることを前提としています。

アルゴリズムまたは振る舞い

1. WAL再生時、XLogRecGetBlockData を使用してレコードから本構造体を取得します。 2. 内部の data フィールド（ginxlogRecompressDataLeaf）に格納された情報に基づき、GINデータリーフページの再圧縮処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginxlog.c: WALレコードの読み取り時に使用。
  • src/backend/access/rmgrdesc/gindesc.c: WALレコードのデコード（説明出力）時に使用。
  • **呼び出し先**:
  • 構造体内部で ginxlogRecompressDataLeaf を参照しています。

関連データ構造

  • ginxlogRecompressDataLeaf: 本構造体がラップしている主要なデータ構造。

失敗モードと境界条件

  • WALレコードのデータ長が本構造体のサイズと一致しない場合、XLogRecGetBlockData による取得時に不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は ginxlogRecompressDataLeaf のラッパーです。WALのバイナリ互換性を維持するため、フィールドの追加や変更を行う際は、パディングやアライメントに注意してください。
