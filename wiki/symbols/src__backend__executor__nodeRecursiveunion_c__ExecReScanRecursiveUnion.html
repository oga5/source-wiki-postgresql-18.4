ExecReScanRecursiveUnion

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeRecursiveunion.c:310

目的

再帰的クエリ実行ノード（RecursiveUnion）の状態をリセットし、再スキャンを可能にする。

シグネチャまたは形状

void ExecReScanRecursiveUnion(RecursiveUnionState *node)

入力と出力

  • **入力**: RecursiveUnionState *node（再帰実行ノードの状態構造体）
  • **出力**: なし（副作用としてノードの状態を初期化する）

前提条件と不変条件

  • node は有効な RecursiveUnionState を指している必要がある。
  • 再帰項（innerPlan）の chgParam が更新され、再スキャンがトリガーされる状態にあること。

アルゴリズムまたは振る舞い

1. **再帰項の準備**: innerPlan の chgParam に plan->wtParam を追加し、再帰項が再スキャンを認識できるようにする。 2. **非再帰項の再スキャン**: outerPlan の chgParam が NULL の場合、ExecReScan(outerPlan) を呼び出して非再帰項をリセットする。 3. **メモリ解放**: node->tableContext が存在する場合、MemoryContextReset を実行してハッシュテーブルのストレージを解放する。 4. **ハッシュテーブルのクリア**: plan->numCols > 0 の場合、ResetTupleHashTable を呼び出して重複排除用ハッシュテーブルを空にする。 5. **状態のリセット**:

  • recursing フラグを false に設定。
  • intermediate_empty フラグを true に設定。
  • working_table および intermediate_table の tuplestore をクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:154（ExecReScan 経由での呼び出しが確認されている）。
  • **呼び出し先**:
  • bms_add_member
  • ExecReScan
  • MemoryContextReset
  • ResetTupleHashTable
  • tuplestore_clear

関連データ構造

  • RecursiveUnionState: 再帰実行の現在の状態（ハッシュテーブル、メモリコンテキスト、タプルストア）を保持。
  • RecursiveUnion: 実行計画ノード。
  • PlanState: outerPlan および innerPlan の状態。

失敗モードと境界条件

  • node->tableContext が NULL の場合、メモリリセットはスキップされる。
  • plan->numCols が 0 の場合（UNION ALL の場合など）、ハッシュテーブルのクリアはスキップされる。

保守担当者向けメモ

  • 本関数は再帰クエリの再実行時に呼び出されるため、working_table と intermediate_table の完全なクリアが重要である。
  • innerPlan への chgParam 追加は、再帰ステップが確実に再評価されることを保証するための必須処理である。
