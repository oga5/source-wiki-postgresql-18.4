pgstat_count_slru_page_zeroed

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_slru.c:58

目的

SLRU（Simple Least Recently Used）キャッシュにおいて、ページがゼロ初期化された回数の統計カウンタをインクリメントします。

シグネチャまたは形状

void pgstat_count_slru_page_zeroed(int slru_idx)

入力と出力

  • **入力**: slru_idx (int) - 対象となるSLRUキャッシュを識別するためのインデックス。
  • **出力**: なし。副作用として、ローカルの統計カウンタ pending_SLRUStats が更新されます。

前提条件と不変条件

  • slru_idx は有効なSLRU識別インデックスである必要があります。
  • 本関数は、各バックエンドのローカルメモリ領域を更新するため、スレッドセーフな共有メモリ操作を伴いません。

アルゴリズムまたは振る舞い

1. get_slru_entry(slru_idx) を呼び出し、指定されたインデックスに対応する統計エントリを取得します。 2. 取得したエントリの blocks_zeroed メンバを 1 加算します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/slru.c:412 から呼び出されています。
  • **呼び出し先**: get_slru_entry（インライン関数と推測される）を呼び出します。

関連データ構造

  • pending_SLRUStats: 各バックエンドが保持するローカルな統計カウンタ配列。本関数はこの配列内の特定の要素を更新します。

失敗モードと境界条件

  • slru_idx が不正な値（範囲外）である場合、get_slru_entry の実装に依存してメモリ破壊やクラッシュが発生する可能性があります。

保守担当者向けメモ

  • 本関数は slru.c からの統計収集フックとして機能します。
  • 統計の共有メモリへの反映は、別途 pgstat_slru_flush_cb によって行われるため、本関数単体では永続化されません。
