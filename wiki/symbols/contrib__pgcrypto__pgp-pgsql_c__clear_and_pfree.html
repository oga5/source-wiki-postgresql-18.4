clear_and_pfree

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pgsql.c:96

目的

機密情報（鍵や平文など）を含むメモリ領域を安全に消去し、PostgreSQLのメモリコンテキストから解放する。

シグネチャまたは形状

static void clear_and_pfree(text *p)

入力と出力

  • **入力**: text *p - 消去および解放対象となるPostgreSQLのtext型データへのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 p は palloc 等で確保された有効なメモリ領域を指している必要がある。
  • VARSIZE_ANY(p) を使用してメモリサイズを特定するため、p は有効なPostgreSQLの可変長データ構造（varlena）である必要がある。

アルゴリズムまたは振る舞い

1. px_memset(p, 0, VARSIZE_ANY(p)) を呼び出し、対象メモリ領域全体をゼロで上書きする。 2. pfree(p) を呼び出し、メモリ領域を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c:434
  • contrib/pgcrypto/pgp-pgsql.c:447
  • contrib/pgcrypto/pgp-pgsql.c:541
  • **呼び出し先**:
  • px_memset (機密情報のゼロクリア用)
  • pfree (PostgreSQLのメモリ解放用)

関連データ構造

  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • p が NULL の場合、VARSIZE_ANY や pfree の挙動に依存する（通常はセグメンテーション違反を引き起こす可能性がある）。

保守担当者向けメモ

  • 本関数は、メモリ上に機密情報が残存するリスクを低減するために使用される。機密データを扱う変数を解放する際は、通常の pfree ではなく必ず本関数を使用すること。
