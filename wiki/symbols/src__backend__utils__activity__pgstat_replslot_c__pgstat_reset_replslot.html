pgstat_reset_replslot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_replslot.c:41

目的

指定された名前の論理レプリケーションスロットに関連付けられた統計カウンタをリセットします。

シグネチャまたは形状

void pgstat_reset_replslot(const char *name)

入力と出力

  • **入力**: name (const char *) - リセット対象のレプリケーションスロット名。
  • **出力**: なし。
  • **副作用**: 指定されたスロットが論理スロットである場合、共有メモリ上の統計情報が初期化されます。

前提条件と不変条件

  • name は NULL であってはなりません。
  • 物理レプリケーションスロットに対しては統計情報を収集していないため、本関数を呼び出しても何も行われません。

アルゴリズムまたは振る舞い

1. ReplicationSlotControlLock を共有モードで取得します。 2. SearchNamedReplicationSlot を使用して、指定された名前のスロットを検索します。 3. スロットが存在しない場合は ERROR を発生させます。 4. スロットが存在し、かつ論理スロット（SlotIsLogical が真）である場合、pgstat_reset を呼び出して当該スロットの統計情報をリセットします。 5. ReplicationSlotControlLock を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c:2015 (SQL関数経由の呼び出し)
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease
  • SearchNamedReplicationSlot
  • SlotIsLogical
  • ReplicationSlotIndex
  • pgstat_reset
  • ereport

関連データ構造

  • ReplicationSlot: レプリケーションスロットの制御構造体。
  • ReplicationSlotControlLock: スロット制御のための共有ロック。

失敗モードと境界条件

  • 指定された名前のスロットが存在しない場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了します。
  • 物理スロットが指定された場合、統計のリセットは行われず、正常終了します。

保守担当者向けメモ

  • 本関数は論理レプリケーションスロットの統計のみを対象としています。
  • ReplicationSlotControlLock の取得は、スロットの存在確認およびインデックス取得の整合性を保証するために必須です。
