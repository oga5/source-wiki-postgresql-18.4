InitBufTable

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/buffer/buf_table.c:50

目的

共有メモリ上に、ディスク上のページ（BufferTag）とメモリ上のバッファスロット（buffer ID）を対応付けるためのハッシュテーブル（SharedBufHash）を初期化します。

シグネチャまたは形状

void InitBufTable(int size)

入力と出力

  • **入力**: size (int) - 作成するハッシュテーブルのサイズ。
  • **出力**: なし（副作用としてグローバル変数 SharedBufHash を初期化）。

前提条件と不変条件

  • 本関数は共有メモリの初期化フェーズで呼び出されることを想定しています。
  • 呼び出し時点では、ハッシュテーブル操作のためのロック（BufMappingLock）は不要であると見なされます。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を設定し、キーサイズを BufferTag、エントリサイズを BufferLookupEnt に指定します。 2. パーティション数として NUM_BUFFER_PARTITIONS を設定します。 3. ShmemInitHash を呼び出し、SharedBufHash を以下のフラグで初期化します。

  • HASH_ELEM: 要素の格納。
  • HASH_BLOBS: キーがバイナリデータ（BufferTag）であることの指定。
  • HASH_PARTITION: ハッシュテーブルをパーティション分割し、並行アクセスを可能にする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/freelist.c 内の初期化処理から呼び出されます。
  • **呼び出し先**: ShmemInitHash（共有メモリハッシュテーブルの作成）。

関連データ構造

  • BufferTag: ハッシュテーブルのキー。
  • BufferLookupEnt: ハッシュテーブルのエントリ構造体。
  • SharedBufHash: 初期化対象となるグローバルなハッシュテーブルポインタ。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitHash 内部でエラーが発生します。

保守担当者向けメモ

  • 本関数はバッファ管理システムの起動時に一度だけ実行される必要があります。
  • size 引数には、バッファ数とパーティション数の合計値などが渡される設計となっています。
