closedir

定義

  • 種別: function
  • 定義箇所: src/port/dirent.c:126

目的

Windows環境において、opendir で開かれたディレクトリ走査用ハンドルをクローズし、関連するメモリリソースを解放します。

シグネチャまたは形状

int closedir(DIR *d);

入力と出力

  • **入力**: DIR *d（ディレクトリ走査状態を保持する構造体へのポインタ）
  • **出力**: 成功した場合は 0、失敗した場合は 0 以外（FindClose の戻り値に基づく）を返します。

前提条件と不変条件

  • 引数 d は opendir によって初期化された有効なポインタである必要があります。
  • d->handle は INVALID_HANDLE_VALUE であるか、有効なWindows検索ハンドルである必要があります。

アルゴリズムまたは振る舞い

1. d->handle が INVALID_HANDLE_VALUE でない場合、Windows APIの FindClose(d->handle) を呼び出します。 2. FindClose の戻り値（成功時非ゼロ）を反転させ、ret に格納します。 3. d->dirname を解放します。 4. d 構造体自体を解放します。 5. ret を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/fd.c
  • src/bin/pg_archivecleanup/pg_archivecleanup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • src/bin/pg_checksums/pg_checksums.c
  • src/bin/pg_combinebackup/pg_combinebackup.c
  • src/bin/pg_resetwal/pg_resetwal.c
  • **呼び出し先**:
  • FindClose (Windows API)
  • free (標準ライブラリ)

関連データ構造

  • DIR: ディレクトリ走査の状態を管理する構造体。src/include/port/win32_msvc/dirent.h で定義されています。

失敗モードと境界条件

  • FindClose が失敗した場合、そのエラーコードが戻り値に反映されます。
  • d が NULL の場合、free 呼び出し時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はWindows専用の実装です。
  • d_ino や d_reclen 等のPOSIXフィールドはWindows APIでは提供されないため、本関数でのクリーンアップ対象外です。
  • fd.c 内では closedir の代わりに FreeDir を使用することが推奨されている箇所があるため、呼び出し元を選択する際は注意してください。
