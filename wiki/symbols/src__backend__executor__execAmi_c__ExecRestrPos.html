ExecRestrPos

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAmi.c:375

目的

プランノードの実行状態において、以前にマークされたスキャン位置へ戻す操作を実行する。

シグネチャまたは形状

void ExecRestrPos(PlanState *node)

入力と出力

  • **入力**: PlanState *node - 操作対象となるプランノードの状態構造体。
  • **出力**: なし（副作用としてノードのスキャン位置を復元する）。

前提条件と不変条件

  • 対象ノードは、マークされた位置を保持する機能をサポートしている必要がある。
  • 呼び出し前に ExecMarkPos が実行され、有効な位置がマークされていることが期待される。

アルゴリズムまたは振る舞い

1. nodeTag(node) を使用して、ノードの型を判定する。 2. ノード型に応じて、以下の専用関数を呼び出す：

  • T_IndexScanState: ExecIndexRestrPos
  • T_IndexOnlyScanState: ExecIndexOnlyRestrPos
  • T_CustomScanState: ExecCustomRestrPos
  • T_MaterialState: ExecMaterialRestrPos
  • T_SortState: ExecSortRestrPos
  • T_ResultState: ExecResultRestrPos

    • 3. 対応していないノード型が渡された場合、elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeMergejoin.c
  • src/backend/executor/nodeResult.c
  • **呼び出し先**:
  • ExecIndexRestrPos, ExecIndexOnlyRestrPos, ExecCustomRestrPos, ExecMaterialRestrPos, ExecSortRestrPos, ExecResultRestrPos（各ノード型の専用実装）

関連データ構造

  • PlanState: 実行中のプランノードの状態を保持する基底構造体。

失敗モードと境界条件

  • 未対応のノード型が渡された場合、エラーが発生する。
  • マークされていない状態で呼び出された場合の挙動は、各ノードの専用実装に依存する。

保守担当者向けメモ

  • 新しいプランノードがマーク/リストア機能をサポートする場合、本関数の switch 文に新しい case を追加する必要がある。
  • MergeJoin 等の再スキャンが必要なノードでのみ利用されるため、すべてのノードがこの機能を実装しているわけではない点に注意すること。
