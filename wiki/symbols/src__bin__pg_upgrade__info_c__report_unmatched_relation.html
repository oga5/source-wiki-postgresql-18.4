report_unmatched_relation

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:213

目的

新旧 PostgreSQL クラスタ間で対応するリレーション（テーブル、インデックス、TOAST）が見つからなかった場合に、詳細な診断メッセージをログ出力する。

シグネチャまたは形状

static void report_unmatched_relation(const RelInfo *rel, const DbInfo *db, bool is_new_db)

入力と出力

  • rel: 照合に失敗したリレーション情報 (RelInfo)。
  • db: 対象のリレーションが属するデータベース情報 (DbInfo)。
  • is_new_db: true の場合は新しいクラスタでの不一致、false の場合は古いクラスタでの不一致を示す。
  • 出力: なし（pg_log を通じて標準エラー出力等へ警告メッセージを送信）。

前提条件と不変条件

  • rel および db は有効なポインタであること。
  • db->rel_arr には、当該データベース内の全リレーション情報が含まれていること。

アルゴリズムまたは振る舞い

1. 対象リレーションの名称を reldesc バッファに格納する。 2. 対象がインデックス (rel->indtable) である場合、db 内の全リレーションを走査し、親テーブルを特定して reldesc に追記する。 3. 対象が TOAST テーブル (rel->toastheap) である場合、同様に親テーブルを特定して reldesc に追記する。 4. 特定できなかった場合は OID を直接メッセージに含める。 5. is_new_db の値に基づき、新旧どちらのクラスタで不一致が発生したかを明示する警告メッセージを生成する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/info.c 内の gen_db_file_maps 等、リレーションの照合処理を行う関数群から呼び出される。
  • 呼び出し先: snprintf, strlen, pg_log (PostgreSQL 内部ユーティリティ)。

関連データ構造

  • RelInfo: リレーションの OID、名前、親テーブル/TOAST への参照を保持。
  • DbInfo: データベース名およびそのデータベース内のリレーション配列 (RelInfoArr) を保持。

失敗モードと境界条件

  • reldesc バッファ（1000バイト）を超える長い名前を持つリレーションの場合、メッセージが切り詰められる可能性がある。
  • データベース内のリレーション数が非常に多い場合、インデックスや TOAST の親テーブル検索（線形探索）がパフォーマンスに影響を与える可能性がある。

保守担当者向けメモ

  • 本関数は gen_db_file_maps における照合失敗時のデバッグ支援を主目的としている。
  • メッセージの多言語化（_() マクロ）に対応しているため、修正時は翻訳ファイルへの影響を考慮すること。
