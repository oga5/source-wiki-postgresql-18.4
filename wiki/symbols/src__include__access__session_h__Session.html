Session

定義

  • 種別: struct
  • 定義箇所: src/include/access/session.h:25

目的

クライアントセッション内で複数のバックエンドプロセスが共有すべき状態をカプセル化し、セッションスコープの共有メモリ管理を行うためのデータ構造です。

シグネチャまたは形状

typedef struct Session
{
    dsm_segment *segment;
    dsa_area   *area;
    struct SharedRecordTypmodRegistry *shared_typmod_registry;
    dshash_table *shared_record_table;
    dshash_table *shared_typmod_table;
} Session;

入力と出力

  • **入力**: InitializeSession() により TopMemoryContext 上に確保されます。
  • **出力**: セッション固有の動的共有メモリ（DSM）セグメント、動的共有領域（DSA）、および型修飾子レジストリへのポインタを保持します。

前提条件と不変条件

  • Session 構造体内の各フィールドは、セッションの生存期間中ずっと有効である必要があります。
  • shared_typmod_registry 関連のフィールドは typcache.c によって管理されます。

アルゴリズムまたは振る舞い

1. **初期化**: InitializeSession() が呼び出されると、Session オブジェクトが生成されます。 2. **共有メモリ管理**: GetSessionDsmHandle() を通じて DSM セグメントが作成され、shm_toc を用いて DSA 領域や型レジストリ領域が初期化されます。 3. **プロセス間共有**: 並列クエリ実行時、リーダープロセスが生成したハンドルをワーカープロセスが受け取り、AttachSession() を実行することで同一のセッション状態にアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/session.c 内の CurrentSession 初期化処理で参照されています。
  • **呼び出し先**: lib/dshash.h（dshash_table 型の定義として利用）、dsm_segment、dsa_area。
  • 注: 上記以外の具体的な関数呼び出し関係は、提供されたソース情報からは特定できません。

関連データ構造

  • dsm_segment: セッションスコープの DSM セグメント。
  • dsa_area: セッションスコープの DSA 領域。
  • SharedRecordTypmodRegistry: typcache.c で管理される共有型修飾子レジストリ。
  • dshash_table: 共有ハッシュテーブル。

失敗モードと境界条件

  • メモリ確保失敗時（MemoryContextAllocZero 等）の挙動については、本構造体の定義からは直接確認できません。
  • セッション生存期間外でのアクセスは未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • typcache.h を直接インクルードせず、前方宣言を使用することで依存関係を最小化しています。
  • 将来的なバックエンドとクライアント接続の分離を見据えた設計となっています。
