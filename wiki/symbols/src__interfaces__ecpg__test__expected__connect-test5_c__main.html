目的

ECPG（Embedded C for PostgreSQL）のデータベース接続および切断機能の正当性を検証するための回帰テスト用エントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は受け取らない）
  • 出力: 正常終了時に 0 を返却する。

前提条件と不変条件

  • ECPGdebug(1, stderr) が呼び出され、デバッグログが標準エラー出力に有効化されていること。
  • 実行環境において、テスト対象のデータベース（ecpg2_regression）およびユーザー（regress_ecpg_user1, regress_ecpg_user2）がアクセス可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: デバッグ出力を有効化する。 2. **接続・切断の網羅的テスト**:

  • 固定文字列および変数を用いた接続を順次実行する。
  • 接続文字列に unix:postgresql:// 形式や接続パラメータ（connect_timeout, client_encoding, gssencmode 等）を含むURLを指定した接続をテストする。
  • ユーザー認証情報（ユーザー名、パスワード）の受け渡しを検証する。
  • ECPGdisconnect を用いて、接続名指定による切断を繰り返す。

    • 3. **境界条件の検証**:

  • 同一接続名での二重接続試行。
  • 存在しない接続名に対する切断試行。

    • 4. **終了**: 処理完了後、0 を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先:
  • ECPGconnect: データベース接続の確立。
  • ECPGdisconnect: データベース接続の切断。
  • ECPGdo: SQL文（ALTER USER）の実行。
  • ECPGtrans: トランザクション（COMMIT）の確定。
  • strcpy: 接続名やDB名の文字列コピー。

関連データ構造

  • char db[200]: データベース名を保持するバッファ。
  • char id[200]: 接続名を保持するバッファ。
  • char *user: ユーザー名を保持するポインタ。

失敗モードと境界条件

  • 存在しない接続名に対する ECPGdisconnect の呼び出し（異常系挙動の確認）。
  • 同一接続名での連続した ECPGconnect 呼び出し（二重接続の挙動確認）。
  • 接続文字列の解析エラー（URLパラメータの不正など）。
