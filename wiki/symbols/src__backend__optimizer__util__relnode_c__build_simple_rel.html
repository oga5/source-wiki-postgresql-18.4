build_simple_rel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/relnode.c:191

目的

クエリプランナにおけるベースリレーション（RelOptInfo）の生成および初期化を行います。テーブル、サブクエリ、関数、CTE などの各リレーションに対して、プランナが最適化を行うためのデータ構造を構築します。

シグネチャまたは形状

RelOptInfo *build_simple_rel(PlannerInfo *root, int relid, RelOptInfo *parent)

入力と出力

  • **入力**:
  • root: 現在のプランナのグローバル情報。
  • relid: 対象リレーションの Range Table Index。
  • parent: 継承関係における親リレーション（存在しない場合は NULL）。
  • **出力**:
  • 初期化済みの RelOptInfo 構造体へのポインタ。

前提条件と不変条件

  • relid は 0 より大きく、root->simple_rel_array_size 未満であること。
  • 指定された relid に対応するリレーションが既に存在しないこと（存在する場合はエラー）。
  • root->simple_rte_array[relid] が有効な RangeTblEntry を指していること。

アルゴリズムまたは振る舞い

1. **ノード生成**: makeNode(RelOptInfo) を呼び出し、reloptkind を parent の有無に応じて RELOPT_BASEREL または RELOPT_OTHER_MEMBER_REL に設定します。 2. **基本属性の初期化**: relids（ビットマップ）、コスト関連フラグ、ターゲットリスト、パスリストなどを初期化します。 3. **権限情報の解決**: RTE_RELATION の場合、userid を設定します。継承関係（parent あり）の場合は親の情報を引き継ぎます。 4. **継承情報の伝播**: parent が指定されている場合、parent から nulling_relids や lateral_relids などの情報をコピーします。 5. **RTE 種別ごとの詳細設定**:

  • RTE_RELATION: get_relation_info を呼び出し、カタログから統計情報を取得します。
  • RTE_SUBQUERY 等: 属性範囲（min_attr, max_attr）と関連配列を確保します。
  • RTE_RESULT: 属性を持たないものとして初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/initsplan.c
  • src/backend/optimizer/plan/planmain.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/prep/prepunion.c
  • src/backend/optimizer/util/inherit.c

関連データ構造

  • RelOptInfo: 本関数が構築する主要な構造体。
  • PlannerInfo: プランナの全体状態を保持。
  • RangeTblEntry: リレーションのソース情報。

失敗モードと境界条件

  • relid が既に存在する場合、elog(ERROR) により処理を中断します。
  • rtekind が未定義の場合、elog(ERROR) により処理を中断します。

保守担当者向けメモ

  • 本関数は RelOptInfo の初期化において非常に多くのフィールドを扱います。フィールドの追加や変更を行う際は、継承関係（parent）における伝播ロジックが正しく維持されているか確認してください。
  • RTE_RELATION 以外の種別における属性管理（attr_needed 等）は palloc0 でメモリ確保されるため、メモリリークに注意してください。
