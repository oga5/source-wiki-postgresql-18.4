ExecGrant_Largeobject

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/aclchk.c:2267

目的

ラージオブジェクト（Large Object）に対する GRANT または REVOKE 文の実行結果を反映し、システムカタログ pg_largeobject_metadata の ACL 情報を更新する。

シグネチャまたは形状

static void ExecGrant_Largeobject(InternalGrant *istmt)

入力と出力

  • **入力**: InternalGrant *istmt（実行対象の権限操作情報を含む構造体）
  • **出力**: なし（副作用としてシステムカタログを更新）

前提条件と不変条件

  • istmt->objects には対象となるラージオブジェクトの OID リストが含まれている必要がある。
  • pg_largeobject_metadata テーブルに対する RowExclusiveLock が保持されていること。
  • pg_largeobject_metadata にはシステムキャッシュが存在しないため、systable_beginscan を用いた直接的なスキャンが必要である。

アルゴリズムまたは振る舞い

1. istmt->all_privs が指定されている場合、権限を ACL_ALL_RIGHTS_LARGEOBJECT に設定する。 2. pg_largeobject_metadata を開き、対象の各ラージオブジェクトについて以下を繰り返す。

  • OID をキーにカタログをスキャンし、対象タプルを取得する。
  • 既存の ACL を取得し、存在しない場合は acldefault でデフォルト値を生成する。
  • select_best_grantor を呼び出し、権限付与者と付与可能な権限を決定する。
  • restrict_and_check_grant で権限の妥当性を検証し、警告やエラーを処理する。
  • merge_acl_with_grant で新しい ACL を生成する。
  • CatalogTupleUpdate を用いてカタログを更新する。
  • recordExtensionInitPriv で拡張機能の初期権限を更新する。
  • updateAclDependencies で共有依存関係を同期する。
  • CommandCounterIncrement を呼び出し、重複処理時のエラーを防止する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c 内の ExecuteGrantStmt（istmt->objtype == ACL_OBJECT_LARGEOBJECT の場合）。
  • **呼び出し先**: table_open, systable_beginscan, systable_getnext, acldefault, select_best_grantor, restrict_and_check_grant, merge_acl_with_grant, aclmembers, heap_modify_tuple, CatalogTupleUpdate, recordExtensionInitPriv, updateAclDependencies, CommandCounterIncrement, table_close。

関連データ構造

  • InternalGrant: 権限操作の定義を保持する構造体。
  • Form_pg_largeobject_metadata: pg_largeobject_metadata カタログのタプル形式。
  • Acl: アクセス制御リストのデータ構造。

失敗モードと境界条件

  • 対象のラージオブジェクトの OID がカタログ内に存在しない場合、elog(ERROR) により処理が中断される。
  • 権限操作が標準規格に違反する場合、restrict_and_check_grant 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_largeobject_metadata を直接操作するため、システムキャッシュの欠如を考慮したスキャン処理が必須である。
  • 拡張機能の初期権限追跡（recordExtensionInitPriv）が含まれているため、関連するカタログ更新時には依存関係の整合性に注意すること。
