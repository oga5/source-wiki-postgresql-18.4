eqin

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/args.c:232

目的

文字列 s1 が文字列 s2 の先頭部分と完全に一致するかを判定し、一致した場合は s2 の残りの部分（s1 の長さ分進んだ位置）を返します。

シグネチャまたは形状

const char *eqin(const char *s1, const char *s2)

入力と出力

  • **入力**:
  • s1: 比較対象の接頭辞となる文字列。
  • s2: 比較対象の文字列全体。
  • **出力**:
  • 一致した場合: s2 内の s1 の直後の位置を指すポインタ。
  • 一致しない場合: NULL。

前提条件と不変条件

  • s1 および s2 はヌル終端文字列である必要があります。
  • s1 が空文字列の場合、ループは実行されず s2 の先頭アドレスが返されます。

アルゴリズムまたは振る舞い

1. s1 の各文字を順次走査します。 2. s1 の文字と s2 の対応する位置の文字を比較します。 3. 一致しない文字が見つかった時点で NULL を返します。 4. s1 の終端まで全て一致した場合、s2 の残りの部分を指すポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/args.c 内の set_option 関数から呼び出され、コマンドライン引数のオプション名照合に使用されます。
  • **呼び出し先**: 外部関数は呼び出しません。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • s2 が s1 より短い場合、s1 の走査中に s2 の終端に達するため、不一致として NULL が返されます。

保守担当者向けメモ

  • 本関数は pro[] テーブルのオプション名検索における前方一致判定に使用されます。
  • pro[] テーブルにおいて、長い名前のオプション（例: -lp）が短い名前のオプション（例: -l）よりも先に定義されている必要があります。これは本関数が最初に見つかった一致を返すためです。
