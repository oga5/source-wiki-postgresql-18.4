parse

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/parse.c:48

目的

字句解析されたトークンを現在の構文コンテキストに適用し、インデントレベルの算出およびスタックベースの構文還元を行うことで、parser_state を更新する。

シグネチャまたは形状

void parse(int tk)
  • tk: 字句解析器から渡されるトークンコード。

入力と出力

  • **入力**: トークンコード tk。グローバル変数 ps（parser_state）が保持する現在の解析状態。
  • **出力**: なし（副作用としてグローバル変数 ps のスタック p_stack、インデントレベル i_l_follow、ind_level 等を更新）。

前提条件と不変条件

  • ps.tos（スタックポインタ）が ps.p_stack の範囲内であること。
  • reduce() 関数が呼び出されることで、スタックが常に安定した状態に保たれること。

アルゴリズムまたは振る舞い

1. **前処理**: if 文で else を伴わない場合、スタック上の ifhead を stmt に還元する。 2. **状態遷移**: switch(tk) により、トークン種別（decl, ifstmt, lbrace, whilestmt 等）に応じたスタック操作とインデントレベルの調整を行う。 3. **スタック更新**: 新しいトークンをスタックにプッシュし、必要に応じてインデントレベルを増減させる。 4. **還元**: reduce() を呼び出し、スタック上のパターンを評価して文を畳み込む。 5. **デバッグ出力**: #ifdef debug が有効な場合、処理前後のトークンおよびスタック状態を標準出力に書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上では明示的な呼び出し元は記述されていない（indent.c がメインループとして呼び出すことが示唆されている）。
  • **呼び出し先**: reduce()（内部関数）、diag2()（エラー報告用）。
  • *注: ツールが提示した他の parse という名称の関数（contrib/ 配下等）は、本関数とは無関係な別シンボルである。*

関連データ構造

  • parser_state (ps): 解析状態を保持するグローバル構造体。
  • p_stack: ps 内の構文スタック。
  • il: ps 内の各スタックレベルに対応するインデントレベル配列。

失敗モードと境界条件

  • **スタックオーバーフロー**: ps.tos がスタックの最大サイズを超えた場合、errx(1, "Parser stack overflow") を呼び出しプログラムを終了する。
  • **構文エラー**: 未知のトークンや不正なネスト（else の不一致や rbrace のエラー）が検出された場合、diag2() を通じてエラーを報告する。

保守担当者向けメモ

  • reduce() は無限ループで記述されているため、新しい構文ルールを追加する際はスタックの収束性に注意すること。
  • ps.p_stack はインデント計算の根拠となるため、変更時は indent_globs.h の定義と整合性を確認すること。
