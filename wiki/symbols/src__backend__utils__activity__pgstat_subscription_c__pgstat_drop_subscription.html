pgstat_drop_subscription

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_subscription.c:78

目的

論理レプリケーションのサブスクリプション削除に伴い、対応する統計エントリを削除する。

シグネチャまたは形状

void pgstat_drop_subscription(Oid subid)

入力と出力

  • **入力**: subid (Oid) - 削除対象のサブスクリプション識別子。
  • **出力**: なし。

前提条件と不変条件

  • 本関数はトランザクションと連動して動作する。
  • 統計エントリの削除は pgstat_drop_transactional を介して行われるため、呼び出し元のトランザクションがロールバックされた場合、統計エントリの削除も自動的に取り消される。

アルゴリズムまたは振る舞い

1. pgstat_drop_transactional を呼び出し、PGSTAT_KIND_SUBSCRIPTION 型の統計エントリを削除対象として登録する。 2. データベース識別子として InvalidOid を指定し、サブスクリプション固有の統計情報を対象とする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/subscriptioncmds.c:1814
  • **呼び出し先**: pgstat_drop_transactional (統計サブシステム内部関数)

関連データ構造

  • PGSTAT_KIND_SUBSCRIPTION: 統計情報の種類を識別する定数。

失敗モードと境界条件

  • 指定された subid が存在しない場合や、統計システムが初期化されていない場合の挙動については、本ソース抜粋からは判断できない。

保守担当者向けメモ

  • 本関数は pgstat_create_subscription と対になるライフサイクル管理関数である。
  • 統計エントリの削除は即時ではなくトランザクション管理下で行われるため、削除の確定タイミングはトランザクションのコミットに依存する。
