CopyDest

定義

  • 種別: enum
  • 定義箇所: src/backend/commands/copyto.c:43

目的

COPY TO 操作におけるデータの出力先（転送先）の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum CopyDest
{
    COPY_FILE,      /* ファイル（またはパイプ経由のプログラム）への出力 */
    COPY_FRONTEND,  /* クライアント（フロントエンド）への出力 */
    COPY_CALLBACK,  /* コールバック関数への出力 */
} CopyDest;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • COPY TO 操作の実行状態を保持する構造体において、出力先を決定するフラグとして利用されます。

アルゴリズムまたは振る舞い

  • COPY_FILE: ファイルシステム上のファイル、または外部プログラムへのパイプを通じてデータを出力します。
  • COPY_FRONTEND: libpq プロトコルを介して、接続中のクライアントへデータを送信します。
  • COPY_CALLBACK: 登録されたコールバック関数を呼び出してデータを受け渡します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/copyto.c 内の CopyToStateData 構造体（copy_dest フィールド）にて参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • CopyToStateData: COPY TO 操作の全状態を保持する構造体であり、この列挙型をメンバとして保持します。

失敗モードと境界条件

  • 列挙型であるため、定義されていない値が代入された場合の動作は未定義です。

保守担当者向けメモ

  • 新しい出力先タイプを追加する場合は、この列挙型を拡張し、CopyToStateData を利用する各関数で分岐処理を更新する必要があります。
