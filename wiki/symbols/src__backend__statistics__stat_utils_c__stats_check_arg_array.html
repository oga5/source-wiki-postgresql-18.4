stats_check_arg_array

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/stat_utils.c:58

目的

統計情報の更新関数に渡される配列型の引数が、システムで許容される形式（1次元かつNULLを含まない）であることを検証します。

シグネチャまたは形状

bool stats_check_arg_array(FunctionCallInfo fcinfo, struct StatsArgInfo *arginfo, int argnum)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • arginfo: 引数名などのメタデータを含む構造体配列。
  • argnum: 検証対象の引数インデックス。
  • **出力**:
  • true: 引数がNULLであるか、検証を通過した場合。
  • false: 検証に失敗し、警告（WARNING）が発行された場合。

前提条件と不変条件

  • argnum で指定された引数は配列型として扱われます。
  • 引数がNULLの場合、検証はスキップされ true を返します。

アルゴリズムまたは振る舞い

1. PG_ARGISNULL を用いて引数がNULLか判定し、NULLであれば即座に true を返します。 2. DatumGetArrayTypeP で引数を ArrayType に変換します。 3. ARR_NDIM を確認し、次元数が1でない場合は WARNING を発行して false を返します。 4. array_contains_nulls を実行し、配列内にNULL要素が含まれている場合は WARNING を発行して false を返します。 5. 上記の条件をすべて満たす場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/attribute_stats.c: MOST_COMMON_FREQS_ARG, MOST_COMMON_ELEM_FREQS_ARG, ELEM_COUNT_HISTOGRAM_ARG の検証に使用。
  • **呼び出し先**:
  • PG_ARGISNULL, PG_GETARG_DATUM, DatumGetArrayTypeP, ARR_NDIM, array_contains_nulls, ereport（PostgreSQL内部マクロおよび関数）。

関連データ構造

  • FunctionCallInfo: 関数呼び出しのコンテキスト。
  • StatsArgInfo: 引数のメタデータ（エラーメッセージで使用される引数名など）。
  • ArrayType: PostgreSQLの配列データ構造。

失敗モードと境界条件

  • **多次元配列**: ARR_NDIM(arr) != 1 の場合、ERRCODE_INVALID_PARAMETER_VALUE を伴う警告を発行。
  • **NULL要素**: 配列内にNULLが含まれる場合、ERRCODE_INVALID_PARAMETER_VALUE を伴う警告を発行。

保守担当者向けメモ

  • 本関数はバリデーション専用であり、統計情報の計算ロジックには関与しません。
  • 警告レベル（WARNING）で処理しているため、呼び出し側は戻り値を確認し、必要に応じて処理を中断する必要があります。
