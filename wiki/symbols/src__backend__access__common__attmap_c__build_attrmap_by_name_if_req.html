build_attrmap_by_name_if_req

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/attmap.c:260

目的

2つの TupleDesc 間で列名に基づいた属性マッピングを生成し、変換が不要な場合はリソースを解放して NULL を返すラッパー関数です。

シグネチャまたは形状

AttrMap *build_attrmap_by_name_if_req(TupleDesc indesc, TupleDesc outdesc, bool missing_ok)

入力と出力

  • **入力**:
  • indesc: 入力側のタプル記述子。
  • outdesc: 出力側のタプル記述子。
  • missing_ok: 列が見つからない場合にエラーとせず許容するかどうか。
  • **出力**:
  • 変換が必要な場合は AttrMap*（呼び出し元で解放が必要）。
  • 変換が不要な場合は NULL。

前提条件と不変条件

  • indesc と outdesc は有効な TupleDesc である必要があります。
  • 内部で呼び出される build_attrmap_by_name がマッピングを生成し、check_attrmap_match がそのマッピングが恒等変換（1対1対応）であるかを判定します。

アルゴリズムまたは振る舞い

1. build_attrmap_by_name を呼び出し、列名に基づく AttrMap を作成します。 2. check_attrmap_match を呼び出し、生成されたマッピングが変換不要な状態（1対1の対応）であるかを確認します。 3. 変換不要と判定された場合、free_attrmap でメモリを解放し NULL を返します。 4. 変換が必要な場合、作成した AttrMap をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tupconvert.c
  • src/backend/commands/tablecmds.c
  • src/backend/executor/execMain.c
  • src/backend/executor/execPartition.c
  • src/backend/executor/execUtils.c
  • src/backend/replication/pgoutput/pgoutput.c
  • **呼び出し先**:
  • build_attrmap_by_name
  • check_attrmap_match
  • free_attrmap

関連データ構造

  • AttrMap: 属性番号の対応関係を保持する構造体。
  • TupleDesc: タプルの型定義情報。

失敗モードと境界条件

  • build_attrmap_by_name が失敗した場合（列の不一致等）、その関数内でエラーが発生し処理が中断されます。
  • missing_ok が false の場合、列名が一致しないとエラーが発生します。

保守担当者向けメモ

  • 本関数は、変換処理の実行前に「実際に変換が必要か」を判定する最適化レイヤーとして機能します。
  • check_attrmap_match の判定ロジックが、不要なタプルコピーを回避するための重要なポイントとなります。
