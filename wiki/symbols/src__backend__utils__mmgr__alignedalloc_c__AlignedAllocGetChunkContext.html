AlignedAllocGetChunkContext

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/alignedalloc.c:135

目的

アライメント調整されたメモリチャンクのポインタから、そのチャンクが属する MemoryContext を取得する。

シグネチャまたは形状

MemoryContext AlignedAllocGetChunkContext(void *pointer)

入力と出力

  • **入力**: pointer (アライメント調整されたメモリ領域を指すポインタ)
  • **出力**: MemoryContext (当該チャンクを管理するメモリコンテキスト)

前提条件と不変条件

  • pointer は palloc_aligned() 等で確保された有効なアライメント調整済みメモリを指している必要がある。
  • MemoryChunkIsExternal(redirchunk) が偽であること（内部管理チャンクであること）をアサートする。

アルゴリズムまたは振る舞い

1. PointerGetMemoryChunk() を用いて、入力ポインタから MemoryChunk ヘッダーを取得する。 2. VALGRIND_MAKE_MEM_DEFINED を呼び出し、ヘッダー領域へのアクセスを許可する。 3. MemoryChunkGetBlock() を介して、アライメント調整前の「真の」メモリブロックを特定する。 4. GetMemoryChunkContext() を呼び出し、ブロックから所属する MemoryContext を抽出する。 5. VALGRIND_MAKE_MEM_NOACCESS を呼び出し、ヘッダー領域へのアクセスを再び禁止する。 6. 取得した MemoryContext を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c:96 (MCTX_ALIGNED_REDIRECT_ID のメソッドテーブル経由)
  • **呼び出し先**:
  • PointerGetMemoryChunk
  • MemoryChunkIsExternal
  • MemoryChunkGetBlock
  • GetMemoryChunkContext
  • VALGRIND_MAKE_MEM_DEFINED / VALGRIND_MAKE_MEM_NOACCESS

関連データ構造

  • MemoryChunk: メモリ管理用のヘッダー構造体。
  • MemoryContext: メモリ管理の基盤となるコンテキスト構造体。

失敗モードと境界条件

  • 不正なポインタが渡された場合、PointerGetMemoryChunk や MemoryChunkGetBlock の内部でメモリ破壊やセグメンテーション違反が発生する可能性がある。
  • MEMORY_CONTEXT_CHECKING が有効な場合、メモリ破壊が検出される可能性がある。

保守担当者向けメモ

  • 本関数は palloc_aligned で確保されたメモリに対するデコレータとして機能する。
  • MemoryChunk ヘッダーはアライメント調整されたポインタの直前に配置される前提で実装されている。
  • Valgrind マクロによるアクセス制御は、ヘッダー情報への不正アクセスを防ぐための重要な保護層である。
