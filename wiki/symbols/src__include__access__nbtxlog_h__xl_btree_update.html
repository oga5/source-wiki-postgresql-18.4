xl_btree_update

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/nbtxlog.h:264

目的

B-treeインデックスにおける重複キー圧縮（Deduplication）に伴う、削除されたタプルID（TID）の情報をWAL（Write-Ahead Log）に記録するための構造体です。

シグネチャまたは形状

typedef struct xl_btree_update
{
    uint16      ndeletedtids;
    /* POSTING LIST uint16 OFFSETS TO A DELETED TID FOLLOW */
} xl_btree_update;

入力と出力

  • **入力**: 削除対象のTID数（ndeletedtids）。
  • **出力**: 構造体直後に続く、削除されたTIDへのオフセット配列（uint16のリスト）。

前提条件と不変条件

  • 構造体の直後に、ndeletedtids で指定された数分の uint16 型オフセット値がメモリ上に連続して配置されている必要があります。

アルゴリズムまたは振る舞い

  • B-treeの重複キー圧縮処理において、どのTIDが削除されたかを追跡するために使用されます。
  • REDO処理時には、この構造体と後続のオフセット配列を読み取り、インデックスページ上の該当するPosting Listから対象TIDを削除することでページ状態を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtpage.c および src/backend/access/nbtree/nbtxlog.c 内の重複キー削除処理に関連する関数群から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • xl_btree_dedup: 重複キー圧縮に関連する他のWALレコード構造体。

失敗モードと境界条件

  • ndeletedtids が0の場合、後続のオフセットデータが存在しない可能性があります。
  • WALレコードの解析時に、構造体直後のメモリ領域が期待されるサイズ（ndeletedtids * sizeof(uint16)）を確保していない場合、メモリ破壊や不正な読み取りが発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は可変長データ（uint16配列）を伴うため、ポインタ演算によるアクセス時には、nbtxlog.c 内の updates = (xl_btree_update *) ((char *) updatedoffsets + ...) のような実装パターンを遵守してください。
  • nbtdesc.c でログのデバッグ出力を行う際も、同様にオフセット配列の走査が必要です。
