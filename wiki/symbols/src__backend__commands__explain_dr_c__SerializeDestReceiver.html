SerializeDestReceiver

定義

  • **種別**: struct
  • **定義箇所**: src/backend/commands/explain_dr.c:32

目的

EXPLAIN (SERIALIZE) 実行時に、クエリ結果のシリアライズ処理に伴うオーバーヘッド（CPU時間、バッファ使用量、バイト数）を計測・収集するための専用レシーバー構造体です。実際のネットワーク送信は行わず、シリアライズコストの定量化を目的とします。

シグネチャまたは形状

typedef struct SerializeDestReceiver
{
    DestReceiver pub;
    ExplainState *es;
    int8         format;
    TupleDesc    attrinfo;
    int          nattrs;
    FmgrInfo     *finfos;
    MemoryContext tmpcontext;
    StringInfoData buf;
    SerializeMetrics metrics;
} SerializeDestReceiver;

入力と出力

  • **入力**: DestReceiver インターフェースを介して受け取るタプルデータ。
  • **出力**: 収集された SerializeMetrics（GetSerializationMetrics 関数経由で ExplainState に渡される）。

前提条件と不変条件

  • DestReceiver を継承しており、pub フィールドが先頭に配置されている必要があります。
  • buf（StringInfoData）は、実際のネットワーク送信を行わず、シリアライズ後のデータ長を計測するために使用されます。
  • tmpcontext は、行ごとのメモリ管理に使用され、大規模な結果セット処理時のメモリリークを防ぎます。

アルゴリズムまたは振る舞い

1. **初期化**: CreateExplainSerializeDestReceiver により生成され、serializeAnalyzeStartup でメモリコンテキストとバッファが準備されます。 2. **データ処理**: serializeAnalyzeReceive が呼び出されるたび、以下の処理をシミュレートします。

  • serialize_prepare_info を用いた出力関数のキャッシュ。
  • slot_getallattrs によるタプルのデコード。
  • pq_send... 系関数による StringInfo バッファへのデータ構築（送信はスキップ）。
  • 経過時間とバッファ使用量の計測および metrics への加算。

    • 3. **終了**: serializeAnalyzeShutdown によりリソースが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain_dr.c 内の各関数（serialize_prepare_info, serializeAnalyzeReceive, serializeAnalyzeShutdown, CreateExplainSerializeDestReceiver, GetSerializationMetrics）から参照されています。
  • **呼び出し先**: 構造体内部で ExplainState や TupleDesc、FmgrInfo、MemoryContext、StringInfoData、SerializeMetrics を保持・操作します。

関連データ構造

  • DestReceiver: 基底となるレシーバー構造体。
  • ExplainState: 計測結果を保持する上位のステート。
  • SerializeMetrics: 収集された計測メトリクスを格納する構造体。
  • StringInfoData: シリアライズされたデータを一時的に保持するバッファ。

失敗モードと境界条件

  • GetSerializationMetrics は、本レシーバーが使用されていない場合（例: CREATE TABLE AS 等）にはゼロを返すよう設計されています。
  • ネットワーク送信を行わないため、pq_endmessage_reuse は呼び出されません。

保守担当者向けメモ

  • 本構造体は「実際にデータを送らない」ことが重要です。バッファの長さ（len）のみをカウントすることで、ネットワーク負荷を除外した純粋なシリアライズコストを測定しています。
  • src/include/commands/explain_dr.h に公開インターフェースが定義されています。
