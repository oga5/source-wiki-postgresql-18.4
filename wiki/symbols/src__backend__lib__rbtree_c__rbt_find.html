rbt_find

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:144

目的

赤黒木（RBTree）内から、指定されたキーと一致するノードを検索します。

シグネチャまたは形状

RBTNode *rbt_find(RBTree *rbt, const RBTNode *data)

入力と出力

  • **入力**:
  • rbt: 検索対象の RBTree 構造体へのポインタ。
  • data: 検索キーとして使用する RBTNode 構造体（呼び出し側で定義されたデータ構造の先頭メンバ）。
  • **出力**:
  • キーが一致した場合は該当する RBTNode へのポインタ。
  • 一致するノードが見つからない場合は NULL。

前提条件と不変条件

  • rbt は rbt_create によって初期化済みであること。
  • rbt->comparator コールバック関数が、data と木内のノードを比較可能であること。
  • 木の構造は赤黒木の不変条件（平衡性）を維持していること。

アルゴリズムまたは振る舞い

1. 木のルートノードから探索を開始します。 2. 現在のノードが RBTNIL（番兵）に到達するまで、以下のループを繰り返します。

  • rbt->comparator を呼び出し、検索キーと現在のノードを比較します。
  • 比較結果が 0（一致）の場合、そのノードを返します。
  • 比較結果が負の場合、左の子ノードへ移動します。
  • 比較結果が正の場合、右の子ノードへ移動します。

    • 3. ループ終了まで一致が見つからない場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_rbtree/test_rbtree.c 内のテスト関数群から、挿入後の検索確認やノードの存在確認のために呼び出されています。
  • **呼び出し先**:
  • rbt->comparator（RBTree 作成時に登録された比較コールバック関数）。

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTNode: 木の各ノードのヘッダー。
  • RBTNIL: 葉ノードを示す静的な番兵。

失敗モードと境界条件

  • 木が空（rbt->root == RBTNIL）の場合、直ちに NULL を返します。
  • 検索キーが木内に存在しない場合、NULL を返します。

保守担当者向けメモ

  • 本関数は侵入型設計に基づいています。data 引数は RBTNode 型ですが、実際には呼び出し側が定義した構造体の先頭メンバとして渡されることを前提としています。
  • RBTNIL を使用して終端判定を行っているため、NULLポインタの直接参照は発生しません。
