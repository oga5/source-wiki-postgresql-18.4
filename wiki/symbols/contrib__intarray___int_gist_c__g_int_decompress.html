g_int_decompress

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_gist.c:288

目的

GiST インデックス検索時に、圧縮された整数配列（範囲表現）を元の配列形式に復元します。

シグネチャまたは形状

Datum g_int_decompress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体を受け取ります。
  • **出力**: 復元された配列を含む新しい GISTENTRY 構造体を Datum として返します。

前提条件と不変条件

  • 入力された ArrayType が有効であること（CHECKARRVALID）。
  • 圧縮された配列の要素数が MAXNUMELTS を超える場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. 入力された GISTENTRY から配列データを取得します。 2. 配列が空、または圧縮されていない（要素数が 2 * num_ranges 未満）場合は、そのまま返します。 3. 圧縮されている場合、各範囲ペア（開始値、終了値）を走査し、範囲内のすべての整数を新しい配列に展開します。 4. 展開後の配列を GISTENTRY に格納し、メモリを適切に管理（必要に応じて pfree）して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの検索処理（PostgreSQL コアの GiST インターフェース経由）。
  • **呼び出し先**: internal_size, new_intArrayType, gistentryinit, palloc, pfree 等のメモリ・配列操作関数。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • ArrayType: PostgreSQL の配列型。
  • MAXNUMELTS: 圧縮キーのスパース性を制御する上限値。

失敗モードと境界条件

  • **エラー**: 展開後の配列サイズが MAXNUMELTS を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED を発生させ、gist__intbig_ops への移行を推奨します。
  • **境界条件**: 空の配列や、圧縮されていない配列が入力された場合は、変換を行わずにそのまま返します。

保守担当者向けメモ

  • 本関数は gist__int_ops の一部です。データがスパースな場合はインデックスが肥大化するため、_intbig_gist.c で定義される gist__intbig_ops の利用が推奨される旨がエラーメッセージに含まれています。
