DefineRange

定義

  • 種別: function
  • 定義箇所: src/backend/commands/typecmds.c:1379

目的

SQLの CREATE TYPE ... AS RANGE コマンドを処理し、新しい範囲型（Range Type）をシステムカタログに登録する。

シグネチャまたは形状

ObjectAddress DefineRange(ParseState *pstate, CreateRangeStmt *stmt)

入力と出力

  • **入力**:
  • pstate: パーサーの状態。
  • stmt: CREATE TYPE 文の構文木（CreateRangeStmt）。
  • **出力**:
  • 作成された範囲型の ObjectAddress を返す。

前提条件と不変条件

  • 対象のスキーマに対する CREATE 権限が必要。
  • subtype パラメータの指定が必須。
  • 範囲型のサブタイプに擬似型（TYPTYPE_PSEUDO）を指定することは不可。
  • canonical 関数を指定する場合、事前にシェル型（Shell Type）が作成されている必要がある。

アルゴリズムまたは振る舞い

1. **名前解決**: 指定された型名から名前空間を特定し、権限を確認する。 2. **重複チェック**: 同名の型が既に存在するか確認し、存在する場合はエラーを投げる（ただし、自動生成された配列型の場合はリネームを試みる）。 3. **パラメータ解析**: stmt->params を走査し、subtype, subtype_opclass, collation, canonical, subtype_diff, multirange_type_name を抽出する。 4. **検証**: サブタイプの有効性、照合順序（Collation）の整合性、サポート関数の存在確認を行う。 5. **属性取得**: サブタイプの長さ、値渡しフラグ、アライメント情報を取得する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQLコマンド実行時）。
  • **呼び出し先**: QualifiedNameGetCreationNamespace, object_aclcheck, GetSysCacheOid2, moveArrayTypeName, typenameTypeId, findRangeSubOpclass, get_collation_oid, findRangeCanonicalFunction, findRangeSubtypeDiffFunction, get_typlenbyvalalign 等。

関連データ構造

  • CreateRangeStmt: 構文解析結果を保持する構造体。
  • ObjectAddress: 作成されたオブジェクトの識別子。
  • pg_type: 型定義を格納するシステムカタログ。

失敗モードと境界条件

  • 指定された型名が既に存在する場合、エラー（ERRCODE_DUPLICATE_OBJECT）。
  • 必須の subtype が欠落している場合、エラー（ERRCODE_SYNTAX_ERROR）。
  • サブタイプが照合順序をサポートしていないのに collation が指定された場合、エラー（ERRCODE_WRONG_OBJECT_TYPE）。
  • シェル型なしで canonical 関数を指定した場合、エラー（ERRCODE_INVALID_OBJECT_DEFINITION）。

保守担当者向けメモ

  • 本関数はカタログ操作のフロントエンドであり、実際の登録処理は後続のカタログ操作関数へ引き継がれる。
  • multirange_type_name の処理が含まれており、マルチレンジ型との関連性が深い。
  • 権限チェックや名前解決のロジックは他の Define... 系関数と共通のパターンに従っている。
