fmtIdEnc

定義

  • 種別: function
  • 定義箇所: src/fe_utils/string_utils.c:100

目的

SQL識別子を安全にクォートし、SQLクエリ内で使用可能な文字列を生成する。識別子がキーワードである場合や、識別子として不適切な文字を含む場合にダブルクォートを付与する。

シグネチャまたは形状

const char *fmtIdEnc(const char *rawid, int encoding)

入力と出力

  • **入力**:
  • rawid: クォート対象の識別子文字列。
  • encoding: 文字列のエンコーディング（pg_wchar.h で定義される値）。
  • **出力**:
  • クォート処理済みの識別子を含む静的バッファへのポインタ。

前提条件と不変条件

  • 戻り値は getLocalPQExpBuffer() が管理する共有バッファを指す。
  • **重要**: 戻り値は、次に fmtIdEnc や関連するバッファ再利用関数が呼び出されると内容が上書きされるため、呼び出し直後に使用またはコピーする必要がある。
  • スレッドセーフではない。

アルゴリズムまたは振る舞い

1. **クォート判定**: quote_all_identifiers が真、または識別子の先頭文字が英字・アンダースコア以外、あるいは文字列内に英数字・アンダースコア以外の文字が含まれる場合にクォートが必要と判断する。 2. **キーワード判定**: クォート不要と判断された場合でも、ScanKeywordLookup を用いて予約語（UNRESERVED_KEYWORD 以外）であるかを確認し、該当すればクォートする。 3. **文字列構築**:

  • クォート不要な場合はそのままバッファへコピー。
  • クォートが必要な場合は " で囲み、内部の " を "" にエスケープする。

    • 4. **マルチバイト処理**: encoding に基づき文字長を判定。不正なマルチバイト文字が含まれる場合は、サーバー側でエラーを誘発する無効なシーケンスに置換する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/scripts/dropdb.c
  • src/bin/scripts/dropuser.c
  • src/bin/scripts/reindexdb.c
  • src/fe_utils/string_utils.c（fmtId, fmtQualifiedIdEnc 等）
  • **呼び出し先**:
  • getLocalPQExpBuffer()
  • ScanKeywordLookup()
  • appendPQExpBufferStr() / appendPQExpBufferChar()
  • pg_encoding_mblen() / pg_encoding_verifymbchar() / pg_encoding_set_invalid()

関連データ構造

  • PQExpBuffer: 文字列構築用バッファ。
  • ScanKeywords: SQLキーワード定義リスト。

失敗モードと境界条件

  • rawid が空文字列の場合の挙動は呼び出し元に依存する。
  • 不正なマルチバイト文字が含まれる場合、サーバー側でエラーとなるようエスケープ処理を強制する。
  • バッファ拡張（enlargePQExpBuffer）が失敗した場合の挙動は、内部の PQExpBuffer 実装に依存する。

保守担当者向けメモ

  • fmtId() は内部で getFmtEncoding() を使用して本関数を呼び出すラッパーである。
  • 識別子のクォート規則は scan.l の識別子生成規則と一致させる必要がある。
  • 戻り値のバッファ再利用に関する制約を、利用側で厳守すること。
