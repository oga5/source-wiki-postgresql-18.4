pg_u_prop_cased

定義

  • 種別: function
  • 定義箇所: src/common/unicode_category.c:143

目的

指定された Unicode コードポイントが「Cased（大文字・小文字の区別を持つ）」プロパティを持つかどうかを判定します。

シグネチャまたは形状

bool pg_u_prop_cased(pg_wchar code)

入力と出力

  • **入力**: pg_wchar code（判定対象の Unicode コードポイント）
  • **出力**: bool（対象が Cased であれば true、そうでなければ false）

前提条件と不変条件

  • 入力コードポイントは UTF-8 エンコーディングを前提とした pg_wchar 型であること。
  • Unicode 規格の安定性ポリシーに基づき、カテゴリ値は固定されているものとして扱われます。

アルゴリズムまたは振る舞い

1. **ASCII 最適化**: code < 0x80 の場合、unicode_opt_ascii テーブルを参照し、PG_U_PROP_CASED ビットが立っているかを定数時間で判定して返します。 2. **一般カテゴリ判定**: 0x80 以上のコードポイントに対しては、unicode_category(code) を呼び出し、その結果を PG_U_CATEGORY_MASK でビットマスク化します。 3. **プロパティ判定**: 以下のいずれかの条件を満たす場合に true を返します。

  • カテゴリマスクが PG_U_LT_MASK（Letter, Titlecase）と一致する。
  • pg_u_prop_lowercase(code) が真である。
  • pg_u_prop_uppercase(code) が真である。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode/category_test.c:71
  • src/common/unicode_case.c:327, 351
  • **呼び出し先**:
  • unicode_category(code)
  • pg_u_prop_lowercase(code)
  • pg_u_prop_uppercase(code)

関連データ構造

  • unicode_opt_ascii: ASCII 文字のプロパティを保持する静的テーブル。
  • PG_U_LT_MASK: Titlecase 文字を識別するためのビットマスク。

失敗モードと境界条件

  • 0x80 未満のコードポイントについては、テーブル参照による高速な判定が行われます。
  • 0x80 以上のコードポイントについては、内部で二分探索（range_search）を伴う unicode_category() が実行されるため、処理コストが異なります。
