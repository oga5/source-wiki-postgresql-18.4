inv_seek

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:387

目的

オープンされているラージオブジェクト（LO）の読み書き位置（オフセット）を、指定された基準点からの相対位置へ変更します。

シグネチャまたは形状

int64 inv_seek(LargeObjectDesc *obj_desc, int64 offset, int whence)

入力と出力

  • **入力**:
  • obj_desc: 操作対象の LargeObjectDesc 構造体へのポインタ。
  • offset: 移動量（バイト単位）。
  • whence: 基準点。SEEK_SET（先頭）、SEEK_CUR（現在位置）、SEEK_END（末尾）のいずれか。
  • **出力**:
  • 変更後の新しいオフセット値を int64 で返します。

前提条件と不変条件

  • obj_desc は有効なポインタである必要があります（Assert による検証あり）。
  • 読み取りまたは書き込みの権限チェックは本関数内では行われません。
  • 算出されたオフセットが負の値、または MAX_LARGE_OBJECT_SIZE を超える場合はエラーとなります。

アルゴリズムまたは振る舞い

1. whence に基づき、新しいオフセット newoffset を算出します。

  • SEEK_SET: offset をそのまま使用。
  • SEEK_CUR: 現在の obj_desc->offset に offset を加算。
  • SEEK_END: inv_getsize(obj_desc) で取得したサイズに offset を加算。

    • 2. 算出された newoffset が有効範囲内（0以上かつ MAX_LARGE_OBJECT_SIZE 以下）であることを確認します。 3. obj_desc->offset を更新し、その値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-fsstubs.c（複数の箇所で呼び出しを確認）
  • **呼び出し先**:
  • inv_getsize（whence == SEEK_END の場合）
  • ereport（不正な whence または範囲外のオフセット指定時）

関連データ構造

  • LargeObjectDesc: ラージオブジェクトの状態を保持する構造体。offset フィールドが本関数によって直接更新されます。

失敗モードと境界条件

  • whence に SEEK_SET, SEEK_CUR, SEEK_END 以外が指定された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了します。
  • 算出されたオフセットが負または MAX_LARGE_OBJECT_SIZE を超える場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了します。

保守担当者向けメモ

  • 本関数はオフセットの更新のみを行い、実際のデータアクセスは行いません。
  • int64 の加算時にオーバーフローが発生する可能性がありますが、最終的な結果が範囲外であればエラーとして処理されるため、個別のオーバーフローチェックは省略されています。
