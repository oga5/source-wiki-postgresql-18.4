element

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_locale.c:375

目的

正規表現内の文字名（例: [[:NUL:]]）を、対応する chr 型の文字コードへ変換します。

シグネチャまたは形状

static chr element(struct vars *v, const chr *startp, const chr *endp)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体へのポインタ。
  • startp: 文字名の開始位置を指す chr 型ポインタ。
  • endp: 文字名の終了位置（直後）を指す chr 型ポインタ。
  • **出力**:
  • 変換後の chr 型文字コード。失敗時は 0 を返します。

前提条件と不変条件

  • startp < endp であること（アサーションにより検証）。
  • 文字名が1文字の場合は、その文字自体を返します。
  • 複数文字の名前の場合、cnames テーブルに定義されている必要があります。

アルゴリズムまたは振る舞い

1. 名前が1文字の場合、その文字をそのまま返します。 2. 複数文字の場合、cnames テーブルを走査し、pg_char_and_wchar_strncmp を用いて名前の一致を検索します。 3. 一致が見つかった場合、対応する cn->code を CHR() マクロで変換して返します。 4. 一致が見つからない場合、ERR(REG_ECOLLATE) を呼び出してエラーを記録し、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコードの文脈からは直接的な呼び出し元は特定できません。
  • **呼び出し先**:
  • pg_char_and_wchar_strncmp: 文字名の比較に使用。
  • ERR(): エラー発生時に呼び出されます。
  • NOTE(): ロケール依存処理の通知に使用されます。
  • **注意**: ツールが提示した contrib/ 以下の各ファイルにおける element への言及は、本関数とは無関係な一般的な用語としての使用であり、本関数の呼び出し元ではありません。

関連データ構造

  • struct vars: コンパイル状態を管理する構造体。
  • struct cname: 文字名とコードの対応を保持するテーブル構造体。
  • chr: 正規表現エンジン内で使用される文字型。

失敗モードと境界条件

  • **失敗**: cnames テーブルに存在しない文字名が渡された場合、REG_ECOLLATE エラーを発生させます。
  • **境界条件**: startp と endp が等しい場合、アサーションにより停止します。

保守担当者向けメモ

  • cnames テーブルの定義と同期して動作します。
  • pg_char_and_wchar_strncmp を使用しているため、ロケール設定に依存した比較が行われます。
  • 本関数は regcomp.c からインクルードされるモジュールの一部であるため、変更時は正規表現コンパイル全体への影響を考慮してください。
