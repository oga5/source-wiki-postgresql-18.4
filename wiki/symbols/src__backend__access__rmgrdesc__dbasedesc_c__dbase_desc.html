dbase_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/dbasedesc.c:21

目的

データベースの作成および削除操作に関連する WAL（Write-Ahead Log）レコードの内容を解析し、人間が読み取り可能な形式で文字列バッファへ出力します。

シグネチャまたは形状

void dbase_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 出力文字列を格納するための StringInfo バッファ。
  • record: 解析対象の XLogReaderState 構造体へのポインタ。
  • **出力**: なし（buf を通じて文字列が書き込まれます）。

前提条件と不変条件

  • record は有効な XLogReaderState を指している必要があります。
  • XLogRecGetInfo(record) で取得した値から XLR_INFO_MASK を除外した値が、データベース操作（XLOG_DBASE_CREATE_FILE_COPY、XLOG_DBASE_CREATE_WAL_LOG、XLOG_DBASE_DROP）のいずれかに合致する必要があります。

アルゴリズムまたは振る舞い

1. XLogRecGetData を用いて WAL レコードのペイロードを取得します。 2. XLogRecGetInfo と XLR_INFO_MASK を用いて操作タイプを特定します。 3. 操作タイプに応じて以下の処理を行います：

  • XLOG_DBASE_CREATE_FILE_COPY: xl_dbase_create_file_copy_rec 構造体としてキャストし、コピー元とコピー先のテーブルスペース ID およびデータベース ID を出力します。
  • XLOG_DBASE_CREATE_WAL_LOG: xl_dbase_create_wal_log_rec 構造体としてキャストし、作成されたディレクトリのテーブルスペース ID およびデータベース ID を出力します。
  • XLOG_DBASE_DROP: xl_dbase_drop_rec 構造体としてキャストし、削除対象の各テーブルスペース ID とデータベース ID をループで出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されているリソースマネージャ（RM_DBASE_ID）の記述子ルーチンとして登録されており、WAL 解析ツール（pg_waldump 等）から間接的に呼び出されます。
  • **呼び出し先**: XLogRecGetData、XLogRecGetInfo、appendStringInfo、appendStringInfoString。

関連データ構造

  • StringInfo: 文字列バッファ管理構造体。
  • XLogReaderState: WAL レコード読み取り状態。
  • xl_dbase_create_file_copy_rec, xl_dbase_create_wal_log_rec, xl_dbase_drop_rec: データベース操作ごとの WAL レコード構造体（commands/dbcommands_xlog.h で定義）。

失敗モードと境界条件

  • info が既知のデータベース操作タイプと一致しない場合、何も出力されません。
  • xlrec へのキャストは、レコードタイプが正しいという前提に依存しています。

保守担当者向けメモ

  • 新しいデータベース操作タイプが commands/dbcommands_xlog.h に追加された場合は、本関数の if/else if 分岐を更新する必要があります。
