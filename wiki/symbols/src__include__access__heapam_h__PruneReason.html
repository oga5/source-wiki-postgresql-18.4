PruneReason

定義

  • 種別: enum
  • 定義箇所: src/include/access/heapam.h:266

目的

ヒープページ内の不要領域を剪定（Prune）する際の理由を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
    PRUNE_ON_ACCESS,        /* on-access pruning */
    PRUNE_VACUUM_SCAN,      /* VACUUM 1st heap pass */
    PRUNE_VACUUM_CLEANUP,   /* VACUUM 2nd heap pass */
} PruneReason;

入力と出力

本シンボルは列挙型であり、関数への引数として渡されることで、剪定処理のコンテキストを制御します。

前提条件と不変条件

  • 剪定処理を行う関数（heap_page_prune_and_freeze 等）において、現在の処理フェーズを正確に反映する必要があります。

アルゴリズムまたは振る舞い

  • PRUNE_ON_ACCESS: 通常のアクセス時に発生するオンデマンドな剪定。
  • PRUNE_VACUUM_SCAN: VACUUM の第1パス（ヒープスキャン）における剪定。
  • PRUNE_VACUUM_CLEANUP: VACUUM の第2パス（クリーンアップ）における剪定。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/pruneheap.c および src/include/access/heapam.h 内の関数がこの列挙値を受け取ります。
  • 呼び出し先: 本シンボル自体は列挙型であり、直接の呼び出し先はありません。

関連データ構造

  • HeapPageFreeze: ページ内のタプル凍結状態を管理する構造体。
  • PruneFreezeResult: 剪定および凍結処理の結果を保持する構造体。

失敗モードと境界条件

  • 未定義の列挙値を渡した場合の動作は未定義です。

保守担当者向けメモ

  • VACUUM のフェーズ追加や剪定ロジックの変更に伴い、本列挙型に値を追加する必要があるか検討してください。
