CatalogId

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_backup.h:277

目的

PostgreSQLのシステムカタログ上のオブジェクトを一意に識別するための構造体です。pg_dump において、データベース内の特定のオブジェクト（テーブルや列など）を追跡・管理するために使用されます。

シグネチャまたは形状

typedef struct
{
	Oid			tableoid;
	Oid			oid;
} CatalogId;

入力と出力

  • **入力**: tableoid（オブジェクトが属するシステムカタログのOID）および oid（オブジェクト自体のOID）。
  • **出力**: なし（データ構造体）。

前提条件と不変条件

  • 構造体内に未使用のバイト（パディング）を含めてはならないという制約があります。
  • tableoid と oid の組み合わせにより、データベース内のオブジェクトを一意に特定します。

アルゴリズムまたは振る舞い

  • 本構造体は、pg_dump におけるオブジェクトのハッシュテーブル管理のキーとして利用されます。
  • src/bin/pg_dump/common.c において、hash_bytes を用いて CatalogId 全体をハッシュ化し、オブジェクトの検索や重複排除のキーとして使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/bin/pg_dump/common.c: ハッシュテーブルのキー定義 (SH_KEY_TYPE) および検索関数 findObjectByCatalogId 等で使用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • DumpId: pg_dump が実行時に割り当てる論理的な識別子。CatalogId がシステム上の永続的なIDであるのに対し、DumpId は依存関係解決のために使用されます。
  • DumpableObject: CatalogId を用いてハッシュテーブルに登録されるオブジェクト構造体。

失敗モードと境界条件

  • 構造体内のパディングによるメモリレイアウトの不整合は、ハッシュ計算の結果に影響を与える可能性があるため、定義上の不変条件として厳格に管理されています。

保守担当者向けメモ

  • CatalogId は pg_dump の内部的なハッシュテーブルのキーとして機能するため、構造体のメンバ変更はハッシュ計算ロジック（SH_HASH_KEY マクロ等）に直接影響します。変更時は src/bin/pg_dump/common.c の整合性を確認してください。
