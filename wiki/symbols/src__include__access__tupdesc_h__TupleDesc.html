TupleDesc

定義

  • 種別: typedef struct TupleDescData *TupleDesc;
  • 定義箇所: src/include/access/tupdesc.h:145

目的

リレーションやクエリ結果の行構造（列数、各列の型、制約など）を保持するメタデータ構造体へのポインタ型です。実行エンジンやアクセスメソッドがメモリ上のタプルを解釈するための基盤となります。

シグネチャまたは形状

TupleDescData 構造体へのポインタです。

入力と出力

  • **入力**: CreateTupleDesc 等により生成され、属性数（natts）に基づいたメモリ領域が確保されます。
  • **出力**: 実行エンジンは TupleDescCompactAttr() 等を介して属性情報にアクセスし、タプルのバイト列を解析（変形）します。

前提条件と不変条件

  • tdrefcount（参照カウント）によるライフサイクル管理が行われます。
  • tdrefcount が -1 の場合、キャッシュ管理対象外（スタック上や一時的なメモリコンテキスト）であることを示し、Pin/Release 操作は無効となります。
  • FormData_pg_attribute（カタログ由来の定義）を直接変更した後は、必ず populate_compact_attribute() を呼び出し、compact_attrs との整合性を維持する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: natts に基づきメモリ確保。 2. **同期**: カタログ定義変更時に compact_attrs を更新。 3. **アクセス**: 実行エンジンがタプル変形（Deformation）を行う際に、CompactAttribute を通じて高速に属性情報へアクセス。 4. **解放**: 参照カウントが 0 になると FreeTupleDesc() によりメモリ解放。

関連データ構造

  • TupleDescData: 実体となる構造体。
  • CompactAttribute: タプル変形を高速化するための軽量属性情報。
  • FormData_pg_attribute: カタログ由来の属性定義。

失敗モードと境界条件

  • populate_compact_attribute() の呼び出し漏れによる CompactAttribute との不整合。
  • 参照カウント管理の不備によるメモリリークまたは早期解放。

保守担当者向けメモ

  • CompactAttribute は 16 バイトに制限されています。拡張時はタプル変形性能への影響を考慮してください。
  • TupleDesc は TupleDescData へのポインタであるため、メモリ管理時には参照カウントの増減（PinTupleDesc / ReleaseTupleDesc）を適切に行ってください。
