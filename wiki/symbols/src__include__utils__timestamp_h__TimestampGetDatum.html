TimestampGetDatum

定義

  • 種別: function
  • 定義箇所: src/include/utils/timestamp.h:45

目的

内部的な Timestamp 型の値を、PostgreSQLの実行エンジンが扱う汎用的な Datum 型へ変換する。

シグネチャまたは形状

static inline Datum TimestampGetDatum(Timestamp X)

入力と出力

  • **入力**: Timestamp X（内部表現である int64 値）
  • **出力**: Datum（PostgreSQLの関数マネージャや実行エンジンで受け渡し可能な型）

前提条件と不変条件

  • Timestamp 型は int64 として実装されているため、本関数は Int64GetDatum を介して変換を行う。
  • インライン関数として定義されており、呼び出し時のオーバーヘッドを最小限に抑える設計となっている。

アルゴリズムまたは振る舞い

引数として受け取った Timestamp 値を、Int64GetDatum マクロを用いて Datum 型にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gin/btree_gin.c
  • contrib/btree_gist/btree_ts.c および btree_utils_num.c
  • src/backend/executor/execExprInterp.c
  • src/backend/utils/adt/date.c
  • src/backend/utils/adt/formatting.c
  • src/backend/utils/adt/jsonpath_exec.c
  • src/backend/utils/adt/timestamp.c
  • **呼び出し先**:
  • Int64GetDatum（fmgr.h 等で定義されるマクロ）

関連データ構造

  • Timestamp: 本関数が扱う内部日時表現（int64）。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • 本関数自体は単純なキャストを行うインライン関数であり、特有の失敗モードはない。
  • 入力値 X が有効な Timestamp 範囲内であることを前提とする。

保守担当者向けメモ

  • Timestamp 型の定義変更（int64 からの変更など）が発生した場合は、本関数の実装も追従する必要がある。
  • パフォーマンス向上のため、インライン定義を維持すること。
