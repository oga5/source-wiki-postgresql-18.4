SplitInterval

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gist/gistproc.c:305

目的

GiSTインデックスのノード分割（PickSplit）処理において、各エントリの境界値（下限および上限）を保持し、ソートおよび分割候補の評価を行うためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    float8      lower,
                upper;
} SplitInterval;

入力と出力

  • 本構造体は、gist_box_picksplit 等の関数内で、インデックスエントリの座標範囲を表現するために使用されます。
  • lower および upper フィールドは、float8 型の境界値を格納します。

前提条件と不変条件

  • lower は通常、対象となる幾何データ（ボックス等）の最小座標値を、upper は最大座標値を表します。
  • qsort を用いたソート処理において、lower または upper の値に基づいて順序付けが行われます。

アルゴリズムまたは振る舞い

  • GiSTのDouble Sortingアルゴリズムにおいて、全エントリの SplitInterval 配列が生成されます。
  • interval_cmp_lower および interval_cmp_upper 関数により、それぞれ lower 値と upper 値を基準とした比較が行われ、ソートされます。
  • ソートされた配列は、ノード分割時の最適な分割軸とグループ分けを決定するために走査されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistproc.c 内の gist_box_picksplit が、メモリ確保およびソートの起点となります。
  • **呼び出し先**: qsort 関数が本構造体を要素とする配列をソートするために呼び出されます。

関連データ構造

  • ConsiderSplitContext: 分割候補の品質評価に使用されるコンテキスト構造体。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • palloc によるメモリ確保失敗時にエラーが発生する可能性があります。
  • nentries が0の場合の挙動については、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は gistproc.c 内の幾何データ型専用の分割ロジックに密結合しています。
  • lower と upper の比較ロジックを変更する場合は、interval_cmp_lower および interval_cmp_upper の実装と整合性を取る必要があります。
