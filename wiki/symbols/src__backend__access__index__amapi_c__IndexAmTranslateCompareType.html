IndexAmTranslateCompareType

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/amapi.c:147

目的

指定されたインデックス・アクセスメソッド（AM）および比較型（CompareType）に対応する戦略番号（StrategyNumber）を取得します。

シグネチャまたは形状

StrategyNumber IndexAmTranslateCompareType(CompareType cmptype, Oid amoid, Oid opfamily, bool missing_ok)

入力と出力

  • **入力**:
  • cmptype: 変換対象の比較型。
  • amoid: 対象アクセスメソッドのOID。
  • opfamily: 演算子ファミリーのOID。
  • missing_ok: 変換が見つからない場合にエラーを発生させるか（false）否か（true）を指定するフラグ。
  • **出力**:
  • 成功時: 対応する StrategyNumber。
  • 失敗時: missing_ok が true の場合は InvalidStrategy を返します。

前提条件と不変条件

  • B-tree AM（BTREE_AM_OID）の場合、COMPARE_INVALID より大きく COMPARE_GT 以下の範囲であれば、カタログ検索をスキップして直接変換値を返します。
  • それ以外のAMについては、GetIndexAmRoutineByAmId を介して取得した IndexAmRoutine 構造体内の amtranslatecmptype 関数ポインタが有効である必要があります。

アルゴリズムまたは振る舞い

1. **最適化パス**: amoid が B-tree であり、かつ cmptype が標準的な範囲内であれば、即座に値を返します。 2. **ルーチン取得**: GetIndexAmRoutineByAmId を呼び出し、対象AMのルーチン構造体を取得します。 3. **変換実行**: ルーチン内の amtranslatecmptype 関数を呼び出し、戦略番号を導出します。関数が定義されていない場合は InvalidStrategy とします。 4. **エラーハンドリング**: missing_ok が false で、かつ変換結果が InvalidStrategy である場合、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/tablecmds.c
  • src/backend/executor/execReplication.c
  • src/backend/replication/logical/relation.c
  • src/backend/utils/adt/selfuncs.c
  • **呼び出し先**:
  • GetIndexAmRoutineByAmId
  • amroutine->amtranslatecmptype（各AM固有の実装）

関連データ構造

  • IndexAmRoutine: AMの機能実装を保持する構造体。
  • CompareType: 比較操作の種類を定義する列挙型。
  • StrategyNumber: インデックスの戦略番号。

失敗モードと境界条件

  • amtranslatecmptype が定義されていないAMに対して missing_ok=false で呼び出した場合、エラーが発生します。
  • amoid が無効な場合、GetIndexAmRoutineByAmId の仕様に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • B-treeの最適化パスは src/backend/access/index/amapi.c 内にハードコードされています。新しい比較型を追加する際は、このショートカット条件の更新が必要になる場合があります。
