ResourceOwnerRemember

定義

  • 種別: function
  • 定義箇所: src/backend/utils/resowner/resowner.c:523

目的

指定されたリソースオーナー（ResourceOwner）に対し、管理対象のリソースを登録します。

シグネチャまたは形状

void ResourceOwnerRemember(ResourceOwner owner, Datum value, const ResourceOwnerDesc *kind)

入力と出力

  • **入力**:
  • owner: リソースを登録する対象の ResourceOwner 構造体。
  • value: 登録するリソースの識別子（Datum 型）。
  • kind: リソースの解放フェーズや優先順位を定義する ResourceOwnerDesc 構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • kind で指定される release_phase および release_priority が 0 以外であること。
  • 対象の owner が解放処理中（releasing）ではなく、かつソート済み（sorted）ではないこと。
  • 本関数を呼び出す前に、必ず ResourceOwnerEnlarge() を呼び出し、リソースを格納するための十分な容量が確保されていること。

アルゴリズムまたは振る舞い

1. kind の妥当性をアサーションで検証します。 2. owner の状態（releasing および sorted）を確認し、解放処理が開始されていないことを保証します。 3. owner->narr が固定長配列の最大サイズ（RESOWNER_ARRAY_SIZE）を超えていないか確認します。超えている場合はエラーを発生させます。 4. owner->arr 配列の現在のインデックス位置に value と kind を格納します。 5. owner->narr をインクリメントし、登録数を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の各サブシステムでリソース管理のために利用されています。
  • contrib/pgcrypto/openssl.c
  • src/backend/access/common/tupdesc.c
  • src/backend/jit/llvm/llvmjit.c
  • src/backend/storage/file/fd.c
  • src/backend/storage/ipc/dsm.c
  • **呼び出し先**: 本関数内から直接呼び出される関数は確認されていません（アサーションおよびエラーハンドリングのみ）。

関連データ構造

  • ResourceOwnerData: リソースオーナーの内部状態を保持する構造体。
  • ResourceOwnerDesc: リソースの解放方法を定義する記述子。
  • ResourceElem: リソースの項目を保持する構造体。

失敗モードと境界条件

  • **エラー**: ResourceOwnerEnlarge() を呼び忘れた場合、または配列の容量が不足している場合に elog(ERROR) が発生します。
  • **境界条件**: owner->narr が RESOWNER_ARRAY_SIZE に達している状態で呼び出すと失敗します。

保守担当者向けメモ

  • 本関数は固定長配列への追加のみを行います。ハッシュテーブルへの移行が必要な大規模なリソース管理については、ResourceOwnerEnlarge() の設計と併せて確認してください。
  • 解放処理が開始された後の登録は禁止されているため、呼び出しタイミングの整合性に注意してください。
