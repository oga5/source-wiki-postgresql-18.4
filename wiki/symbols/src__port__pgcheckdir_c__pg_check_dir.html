pg_check_dir

定義

  • 種別: function
  • 定義箇所: src/port/pgcheckdir.c:32

目的

指定されたディレクトリの存在を確認し、その中身の状態（空、隠しファイルのみ、マウントポイント、通常ファイル存在）を分類して返す。主に initdb 等でデータディレクトリの初期化可否を判定するために使用される。

シグネチャまたは形状

int pg_check_dir(const char *dir)

入力と出力

  • **入力**: dir (検査対象のディレクトリパス)
  • **出力**: 以下のいずれかの整数値を返す。
  • 0: ディレクトリが存在しない
  • 1: 空である
  • 2: ドットファイルのみ存在する（非Windows環境）
  • 3: lost+found ディレクトリのみ存在する（非Windows環境）
  • 4: 通常のファイルやディレクトリが存在する
  • -1: アクセスエラー

前提条件と不変条件

  • opendir が失敗した場合、errno が ENOENT であれば 0 を返し、それ以外は -1 を返す。
  • 走査中、. および .. エントリは常に無視される。
  • 戻り値の判定には優先順位があり、通常ファイルが一つでもあれば 4 が優先される。

アルゴリズムまたは振る舞い

1. opendir() でディレクトリを開く。 2. readdir() を用いてディレクトリ内の全エントリをループで走査する。 3. 非Windows環境では、ファイル名が . で始まる場合は dot_found フラグを、lost+found の場合は mount_found フラグを立てる。 4. 上記以外のファイルが見つかった場合、即座に result = 4 を設定しループを終了する。 5. closedir() でディレクトリを閉じ、走査中の errno を保持したまま結果を返す。 6. 最終的な戻り値は、result が 1 の場合に限り、mount_found (3) または dot_found (2) のフラグ状態を反映する。

関連データ構造

  • DIR: ディレクトリストリームを保持する構造体。
  • struct dirent: ディレクトリ内のエントリ情報。

失敗モードと境界条件

  • ディレクトリが開けない場合や closedir に失敗した場合は -1 を返す。
  • readdir 実行中にエラーが発生した場合、result は -1 に設定される。
  • Windows環境では lost+found やドットファイルの特別な判定ロジックは無効化される。

保守担当者向けメモ

  • 本関数はプラットフォーム依存の挙動（特に非Windows環境での隠しファイルや lost+found の扱い）を含む。
  • readdir 実行中の errno を適切に保存・復元する設計となっており、呼び出し元がエラー原因を特定できるように配慮されている。
