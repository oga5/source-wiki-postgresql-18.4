DefineAttr

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/bootstrap/bootstrap.c:521

目的

ブートストラップモードにおいて、システムカタログの属性（列）定義をメモリ上の attrtypes 配列に登録する。

シグネチャまたは形状

void DefineAttr(char *name, char *type, int attnum, int nullness)

入力と出力

  • **入力**:
  • name: 属性名。
  • type: 型名。
  • attnum: 属性番号（0から開始）。
  • nullness: NULL許容フラグ（BOOTCOL_NULL_FORCE_NOT_NULL, BOOTCOL_NULL_FORCE_NULL, BOOTCOL_NULL_AUTO）。
  • **出力**: なし（副作用として attrtypes 配列を更新）。

前提条件と不変条件

  • boot_reldesc が NULL であること（開いているリレーションがない状態であること）。
  • attrtypes 配列が適切に確保されていること。

アルゴリズムまたは振る舞い

1. **初期化**: attrtypes[attnum] が未確保であれば AllocateAttribute() で確保し、メモリをゼロクリアする。 2. **型解決**: gettype(type) を呼び出し、型情報を取得する。

  • Typ リストが構築済みであれば、Ap（型情報キャッシュ）から属性の長さ、アライメント、ストレージ等を設定する。
  • 未構築であれば、ハードコードされた TypInfo 配列から情報を取得する。

    • 3. **照合順序の強制**: 照合順序（Collation）が有効な場合、C_COLLATION_OID に強制設定する。 4. **NULL制約の決定**:

  • 明示的な指定がある場合はそれに従う。
  • BOOTCOL_NULL_AUTO の場合、固定長かつ先行する全列が固定長かつNOT NULLであれば、当該列もNOT NULLとみなす。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は提供されていません。
  • **呼び出し先**: AllocateAttribute(), gettype(), namestrcpy(), elog(), MemSet(), closerel()。

関連データ構造

  • attrtypes: 属性定義を保持するグローバル配列。
  • TypInfo: 型情報のハードコードされた定義。
  • Ap: 型情報キャッシュへのポインタ。

失敗モードと境界条件

  • boot_reldesc が NULL でない場合、警告を出力し closerel(NULL) を実行してリレーションを閉じる。
  • attnum が配列の範囲外である場合の挙動は、本ソース抜粋からは不明。

保守担当者向けメモ

  • 本関数はブートストラップの初期段階でカタログ構造を定義するために使用される。
  • 照合順序を C に強制するロジックは、template0 のクローン時にデータベース間で一貫した動作を保証するために不可欠である。
