ProcWaitStatus

定義

  • 種別: enum
  • 定義箇所: src/include/storage/proc.h:137

目的

バックエンドプロセスがロック待ち等の待機状態にある際の、処理結果や現在のステータスを表現するために使用されます。

シグネチャまたは形状

typedef enum
{
	PROC_WAIT_STATUS_OK,        /* 待機が正常に完了 */
	PROC_WAIT_STATUS_WAITING,   /* 現在待機中 */
	PROC_WAIT_STATUS_ERROR,     /* 待機中にエラーが発生 */
} ProcWaitStatus;

入力と出力

  • **入力**: ProcWakeup() 等の関数において、プロセスの状態遷移を決定する引数として渡されます。
  • **出力**: WaitOnLock() 等の待機処理を行う関数が、待機結果を呼び出し元へ返す戻り値として使用されます。

前提条件と不変条件

  • PGPROC 構造体の waitStatus フィールドに格納される値として利用されます。
  • 待機状態の参照時には、必要に応じて volatile 修飾子を用いたアクセスが行われます（例: src/backend/storage/lmgr/proc.c:1470）。

アルゴリズムまたは振る舞い

  • プロセスがロック取得のために待機キューに入る際、状態は PROC_WAIT_STATUS_WAITING に設定されます。
  • 待機対象のロックが解放された際、ProcWakeup() を通じて PROC_WAIT_STATUS_OK または PROC_WAIT_STATUS_ERROR が設定され、待機プロセスが起床します。

呼び出し元と呼び出し先

  • **呼び出し元（参照・設定）**:
  • src/backend/storage/lmgr/lock.c: WaitOnLock() 内で待機結果として利用。
  • src/backend/storage/lmgr/proc.c: ProcWakeup() 内で状態更新に使用。
  • **呼び出し先**:
  • 本シンボル自体は列挙型であり、関数呼び出しは行いません。

関連データ構造

  • PGPROC: 各バックエンドのプロセス情報を保持する構造体。waitStatus フィールドでこの型を使用。
  • LOCALLOCK: ロックマネージャが管理するローカルロック情報。

失敗モードと境界条件

  • PROC_WAIT_STATUS_ERROR は、デッドロック検出やプロセス終了要求など、待機が正常に継続できない状況を示します。

保守担当者向けメモ

  • PGPROC->waitStatus を読み取る際は、並行制御の観点から適切なロック保持や volatile キャストが必要になる場合があります。実装変更時は src/backend/storage/lmgr/proc.c の該当箇所を参照してください。
