jsonb_subscript_fetch_old

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonbsubs.c:322

目的

jsonb 型の添字操作において、対象の jsonb 値から指定されたパス（添字）に対応する要素を取得し、実行状態へ格納する。

シグネチャまたは形状

static void jsonb_subscript_fetch_old(ExprState *state, ExprEvalStep *op, ExprContext *econtext)

入力と出力

  • **入力**:
  • op->resvalue: 取得対象となる jsonb 値（Datum）。
  • op->resnull: 対象値が NULL かどうかを示すフラグ。
  • op->d.sbsref.state: 添字操作の状態を保持する SubscriptingRefState。ここから upperindex（添字の配列）と numupper（添字の数）が参照される。
  • **出力**:
  • sbsrefstate->prevvalue: 取得した要素の Datum 値。
  • sbsrefstate->prevnull: 取得した要素が NULL かどうかを示すフラグ。

前提条件と不変条件

  • op->resvalue は jsonb 型のポインタとして解釈可能であること。
  • sbsrefstate は初期化済みであること。

アルゴリズムまたは振る舞い

1. op->resnull を確認し、対象の jsonb が NULL の場合は、結果の prevvalue を (Datum) 0、prevnull を true に設定して終了する。 2. 対象が NULL でない場合、DatumGetJsonbP を用いて jsonb 構造体へ変換する。 3. jsonb_get_element を呼び出し、sbsrefstate に格納された添字情報（upperindex, numupper）に基づいて要素を抽出する。 4. 抽出結果を sbsrefstate->prevvalue および sbsrefstate->prevnull に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonbsubs.c 内で、SubscriptRoutines 構造体の sbs_fetch_old フィールドに代入されている。
  • **呼び出し先**:
  • jsonb_get_element (内部的な要素抽出処理)

関連データ構造

  • ExprState, ExprEvalStep, ExprContext: PostgreSQL の式評価フレームワークで使用される構造体。
  • SubscriptingRefState: 添字操作の実行時状態を保持する構造体。
  • Jsonb: jsonb 型の内部表現。

失敗モードと境界条件

  • 対象の jsonb が NULL の場合、エラーにはならず NULL を返す。
  • jsonb_get_element が内部的に失敗（不正なパス指定など）した場合の挙動は、同関数の実装に依存する。

保守担当者向けメモ

  • 本関数は SubscriptRoutines を介して呼び出されるレガシーなフェッチ処理である。
  • jsonb の添字操作において、スライス操作はサポートされていない。
