inet_gist_picksplit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_gist.c:662

目的

GiSTインデックスのページ分割時に、ネットワークアドレス（INET/CIDR）の集合を2つのグループに分割し、それぞれの親ノードとなる代表キー（spl_ldatum, spl_rdatum）を計算する。

シグネチャまたは形状

Datum inet_gist_picksplit(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *（分割対象のインデックスエントリ群）
  • PG_GETARG_POINTER(1): GIST_SPLITVEC *（分割結果を格納する構造体）
  • **出力**:
  • GIST_SPLITVEC * を含む Datum。spl_left, spl_right に分割されたオフセット番号が格納され、spl_ldatum, spl_rdatum に新しい親キーが設定される。

前提条件と不変条件

  • 入力されるエントリは GistInetKey 構造体として扱われる。
  • 分割後の各サイドにおいて、minbits（最小ネットマスク長）と commonbits（共通プレフィックス長）が可能な限り高く設定される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: 全入力エントリの共通パラメータを calc_inet_union_params で算出。 2. **分割戦略**:

  • **アドレスファミリーが混在する場合**: ファミリーの種類に基づいてグループ分けを行う。
  • **同一ファミリーの場合**: 共通プレフィックスの直後のビット位置を順次走査し、ビット値（0または1）に基づいて分割を試みる。
  • **分割不能な場合**: 最終手段として、エントリを単純に50-50で分割する。

    • 3. **再計算**: 分割された各グループに対し calc_inet_union_params_indexed を呼び出し、最適な親キーを再計算する。 4. **結果格納**: build_inet_union_key を用いて生成したキーを splitvec にセットして終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの管理フレームワーク（直接的な呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**:
  • calc_inet_union_params
  • calc_inet_union_params_indexed
  • build_inet_union_key
  • DatumGetInetKeyP (マクロ)
  • gk_ip_family, gk_ip_addr (マクロ)

関連データ構造

  • GistInetKey: ネットワークアドレスのインデックス表現。
  • GistEntryVector: 分割対象のインデックスエントリ配列。
  • GIST_SPLITVEC: 分割結果を保持する標準的なGiST構造体。

失敗モードと境界条件

  • **ビット走査の限界**: 共通ビットが最大ビット数に達した場合、ビット単位の分割を諦め、50-50の強制分割へ移行する。
  • **メモリ不足**: palloc によるメモリ確保が失敗した場合はエラーとなる。

保守担当者向けメモ

  • 本関数は network_gist.c 内の他のヘルパー関数と密接に連携している。特に calc_inet_union_params_indexed の結果がインデックスの検索効率に直結するため、分割ロジックを変更する際は検索時の枝刈り性能への影響を考慮すること。
