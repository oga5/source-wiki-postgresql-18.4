is_encoding_supported_by_icu

定義

  • 種別: function
  • 定義箇所: src/common/encnames.c:463

目的

指定された内部エンコーディング ID が、ICU（International Components for Unicode）ライブラリによってサポートされているかどうかを判定します。

シグネチャまたは形状

bool is_encoding_supported_by_icu(int encoding)

入力と出力

  • **入力**: encoding (int) - 判定対象の PostgreSQL 内部エンコーディング ID。
  • **出力**: bool - サポートされている場合は true、それ以外（無効な ID または ICU 未対応）の場合は false。

前提条件と不変条件

  • 入力される encoding は、PostgreSQL のバックエンドエンコーディングとして有効な範囲内である必要があります。
  • pg_enc2icu_tbl テーブルが、エンコーディング ID をインデックスとして ICU 名へのマッピング（または NULL）を保持していることを前提とします。

アルゴリズムまたは振る舞い

1. PG_VALID_BE_ENCODING(encoding) マクロを使用して、入力された ID が有効なバックエンドエンコーディング範囲内かを確認します。範囲外であれば直ちに false を返します。 2. 有効な範囲内であれば、pg_enc2icu_tbl テーブルの該当インデックスを参照します。 3. テーブルの値が NULL でなければ true を、NULL であれば false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/namespace.c:2351
  • src/backend/commands/collationcmds.c:337
  • src/backend/commands/dbcommands.c:1145
  • src/bin/initdb/initdb.c:2302
  • **呼び出し先**:
  • PG_VALID_BE_ENCODING マクロ（src/include/mb/pg_wchar.h で定義）

関連データ構造

  • pg_enc2icu_tbl: 内部エンコーディング ID から ICU エンコーディング名へのマッピングを保持する配列。

失敗モードと境界条件

  • PG_VALID_BE_ENCODING の範囲外の値を渡した場合、関数は安全に false を返します。
  • ICU でサポートされていないエンコーディング ID に対しては、テーブルの値が NULL であるため false を返します。

保守担当者向けメモ

  • 本関数はバックエンドエンコーディングのみを対象としています。
  • pg_enc2icu_tbl のサイズや内容は src/include/mb/pg_wchar.h の定義と密接に関連しているため、エンコーディングを追加する際はテーブルの整合性に注意してください。
