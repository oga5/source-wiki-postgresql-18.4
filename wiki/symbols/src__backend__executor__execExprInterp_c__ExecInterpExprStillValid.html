ExecInterpExprStillValid

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExprInterp.c:2294

目的

式評価の初回実行時に、スキーマ変更等により式が依然として有効であるかを検証し、以降の実行を最適化するためのラッパー関数です。

シグネチャまたは形状

Datum ExecInterpExprStillValid(ExprState *state, ExprContext *econtext, bool *isNull);

入力と出力

  • **入力**:
  • state: 評価対象の式状態 (ExprState)。
  • econtext: 実行コンテキスト (ExprContext)。
  • isNull: 結果が NULL かどうかを格納するフラグへのポインタ。
  • **出力**:
  • Datum: 式の評価結果。

前提条件と不変条件

  • 初回実行時に CheckExprStillValid を呼び出し、属性と Var ノードの互換性を検証する必要があります。
  • 成功後は state->evalfunc を state->evalfunc_private に書き換えることで、次回以降の検証処理をスキップする不変条件を持ちます。

アルゴリズムまたは振る舞い

1. CheckExprStillValid(state, econtext) を呼び出し、スキーマ変更等による不整合がないか確認します。 2. state->evalfunc を state->evalfunc_private に更新し、以降の呼び出しで本関数をバイパスするようにします。 3. 更新された state->evalfunc を実行し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c (初期化処理にて state->evalfunc に代入)
  • src/backend/jit/llvm/llvmjit_expr.c および llvmjit_types.c (JIT コンパイル時の参照)
  • **呼び出し先**:
  • CheckExprStillValid (検証用)
  • state->evalfunc (更新後の実行関数)

関連データ構造

  • ExprState: 式の実行状態を保持する構造体。
  • ExprContext: 実行時のコンテキスト情報。

失敗モードと境界条件

  • CheckExprStillValid 内でスキーマ不整合が検出された場合、エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は式評価のホットパスにおける初回実行時のガードとして機能します。
  • evalfunc の書き換えは、実行効率を維持するための重要な最適化です。変更を加える際は、このポインタの付け替えが正しく行われることを確認してください。
