getTSCurrentConfig

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/ts_cache.c:555

目的

現在のセッションにおけるデフォルトの全文検索設定（Text Search Configuration）の OID を取得します。キャッシュを利用して検索コストを削減します。

シグネチャまたは形状

Oid getTSCurrentConfig(bool emitError)

入力と出力

  • **入力**:
  • emitError (bool): 設定が見つからない、または設定されていない場合にエラーを投げるかどうかを指定します。
  • **出力**:
  • Oid: 見つかった設定の OID。設定が未定義で emitError が false の場合は InvalidOid を返します。

前提条件と不変条件

  • GUC変数 TSCurrentConfig が設定されている必要があります。
  • 初回呼び出し時に TSConfigCacheHash が未初期化であれば、init_ts_config_cache() が内部で呼び出されます。
  • キャッシュされた TSCurrentConfigCache が有効な場合、システムカタログへの再検索は行われません。

アルゴリズムまたは振る舞い

1. キャッシュ変数 TSCurrentConfigCache が有効な OID を保持していれば、即座にそれを返します。 2. GUC変数 TSCurrentConfig が空または NULL の場合、emitError に応じてエラーを発生させるか InvalidOid を返します。 3. TSConfigCacheHash が NULL の場合、init_ts_config_cache() を呼び出して初期化します。 4. stringToQualifiedNameList を用いて設定名を解析し、get_ts_config_oid を呼び出して OID を取得します。 5. 取得した OID を TSCurrentConfigCache に格納し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/to_tsany.c (複数の箇所で emitError=true として呼び出し)
  • src/backend/tsearch/wparser.c (複数の箇所で emitError=true として呼び出し)
  • **呼び出し先**:
  • init_ts_config_cache
  • stringToQualifiedNameList
  • get_ts_config_oid
  • elog (エラー発生時)

関連データ構造

  • TSCurrentConfig (GUC変数)
  • TSCurrentConfigCache (キャッシュ用グローバル変数)
  • TSConfigCacheHash (設定キャッシュ用ハッシュテーブル)
  • ErrorSaveContext (エラー抑制用コンテキスト)

失敗モードと境界条件

  • **設定未定義**: TSCurrentConfig が未設定の場合、emitError が true なら ERROR を発生させ、false なら InvalidOid を返します。
  • **名前解決失敗**: stringToQualifiedNameList が失敗した場合、InvalidOid が返されます。
  • **初回実行**: 初回呼び出し時はキャッシュ構築のため、システムカタログへのアクセスが発生します。

保守担当者向けメモ

  • 本関数は TSCurrentConfigCache を更新するため、カタログ変更時には InvalidateTSCacheCallBack を通じた無効化が適切に行われる必要があります。
  • emitError が false の場合、ErrorSaveContext を使用して名前解析時のエラーを抑制する設計になっています。
