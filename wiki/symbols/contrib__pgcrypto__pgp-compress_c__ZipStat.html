ZipStat

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/pgp-compress.c:48

目的

PGPデータストリームの圧縮処理において、zlib ライブラリのストリーム状態と、圧縮データの一時バッファを保持するために使用される構造体です。

シグネチャまたは形状

struct ZipStat
{
    uint8       type;
    int         buf_len;
    int         hdr_done;
    z_stream    stream;
    uint8       buf[ZIP_OUT_BUF];
};

入力と出力

本構造体は関数引数として直接渡されるのではなく、フィルタのプライベートデータ（priv）として保持され、圧縮処理の各段階で参照・更新されます。

前提条件と不変条件

  • z_stream メンバは、zlib の関数（deflate 等）に渡す前に適切に初期化されている必要があります。
  • buf メンバは ZIP_OUT_BUF サイズの固定長配列であり、圧縮データの書き出し用バッファとして機能します。

アルゴリズムまたは振る舞い

  • 圧縮フィルタの実行中、stream を通じて zlib の圧縮アルゴリズムを管理します。
  • hdr_done フラグにより、PGP圧縮ヘッダーの処理状態を追跡します。
  • buf_len は buf 内の有効なデータ長を管理し、下位フィルタへの書き出し時に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-compress.c 内の圧縮関連関数（compress_init, compress_process, compress_flush, compress_free）が、priv ポインタをキャストして本構造体を参照しています。
  • 呼び出し先: zlib ライブラリの関数群（deflate 等）が stream メンバを介して呼び出されます。

関連データ構造

  • z_stream: zlib が提供する圧縮状態管理構造体。
  • PushFilter: 本構造体を内部状態として保持するフィルタインターフェース。

失敗モードと境界条件

  • zlib のメモリ確保失敗時、z_alloc を経由して PostgreSQL のメモリ管理機構がエラーを発生させる可能性があります。
  • buf のサイズ（ZIP_OUT_BUF）を超えるデータが一度に処理される場合、バッファのフラッシュ処理が適切に行われる必要があります。

保守担当者向けメモ

  • 本構造体は圧縮専用です。伸長処理については DecomprData 構造体が対応します。
  • zlib がインストールされていない環境では、本構造体を含む圧縮機能は無効化されます。
