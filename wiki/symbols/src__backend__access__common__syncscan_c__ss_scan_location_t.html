ss_scan_location_t

定義

  • 種別: struct
  • 定義箇所: src/backend/access/common/syncscan.c:91

目的

同期スキャン（Synchronous Scan）において、特定のテーブルに対する現在のスキャン進行位置を保持するために使用されます。

シグネチャまたは形状

typedef struct ss_scan_location_t
{
	RelFileLocator relfilelocator;	/* identity of a relation */
	BlockNumber location;		/* last-reported location in the relation */
} ss_scan_location_t;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • relfilelocator: 対象リレーションを一意に識別する情報。
  • location: 最後に報告されたブロック番号。

前提条件と不変条件

  • relfilelocator は、共有メモリ上でスキャン位置を追跡する対象のリレーションを特定するために使用されます。
  • location は、同期スキャン機能によって定期的に更新される進行状況を反映します。

アルゴリズムまたは振る舞い

本構造体は、ss_scan_locations_t が管理するリスト内の各要素として機能します。同期スキャン実行時、この構造体を通じて各バックエンドが現在のスキャン位置を共有し、I/O 負荷を低減するための調整が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/common/syncscan.c 内の ss_search 等で、共有メモリ上のエントリとして参照されます。
  • 呼び出し先: 情報なし。

関連データ構造

  • ss_scan_locations_t: 本構造体を複数保持し、共有メモリ上でスキャン位置リストを管理するメイン構造体。
  • RelFileLocator: リレーションの識別に使用。
  • BlockNumber: ブロック位置の表現に使用。

失敗モードと境界条件

  • 共有メモリ上のエントリ数が SYNC_SCAN_NELEM を超えた場合、LRU アルゴリズムにより古いエントリが押し出されます。
  • ss_report_location による更新は LWLockConditionalAcquire を介して行われるため、ロック競合時には更新がスキップされる可能性があります。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、アライメントやサイズ変更には注意が必要です。
  • スキャン位置のラップアラウンド処理は本構造体ではなく、呼び出し側のスキャンロジックに依存します。
