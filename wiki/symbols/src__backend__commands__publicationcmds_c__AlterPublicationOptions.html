AlterPublicationOptions

定義

  • 種別: function
  • 定義箇所: src/backend/commands/publicationcmds.c:980

目的

ALTER PUBLICATION 文の実行時、パブリケーションのオプション（publish アクション、publish_via_partition_root、publish_generated_columns）を解析し、システムカタログ pg_publication を更新する。

シグネチャまたは形状

static void AlterPublicationOptions(ParseState *pstate, AlterPublicationStmt *stmt, Relation rel, HeapTuple tup)

入力と出力

  • **入力**:
  • pstate: パーサー状態。
  • stmt: AlterPublicationStmt 構文解析ノード。
  • rel: pg_publication システムカタログのリレーション。
  • tup: 更新対象の pg_publication カタログタプル。
  • **出力**: なし（副作用としてカタログ更新とキャッシュ無効化を実行）。

前提条件と不変条件

  • publish_via_partition_root を false に変更する場合、そのパブリケーションに含まれるパーティションテーブルに対して、行フィルタ（WHERE句）やカラムリストが定義されていないこと。定義されている場合はエラーとなる。

アルゴリズムまたは振る舞い

1. parse_publication_options を呼び出し、SQL文からオプション値を抽出する。 2. publish_via_partition_root を false に変更する試行がある場合、対象パブリケーションの全パーティションテーブルを走査し、行フィルタやカラムリストの有無を pg_publication_rel から確認する。違反があればエラーを投げる。 3. 変更対象のオプション値を heap_modify_tuple でタプルに反映する。 4. CatalogTupleUpdate でシステムカタログを更新し、CommandCounterIncrement を発行する。 5. キャッシュ無効化処理へ移行する。puballtables が真であれば全リレーションキャッシュを無効化し、それ以外の場合は関連するパーティションツリーを特定して無効化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/publicationcmds.c 内の AlterPublication 関数から呼び出される。
  • **呼び出し先**: parse_publication_options, LockDatabaseObject, GetPublicationRelations, SearchSysCache2, heap_modify_tuple, CatalogTupleUpdate, CommandCounterIncrement, CacheInvalidateRelcacheAll, GetPubPartitionOptionRelations 等。

関連データ構造

  • AlterPublicationStmt: SQL文の解析結果を保持するノード。
  • Form_pg_publication: pg_publication カタログの構造体。
  • PublicationActions: パブリケーションのアクション設定を保持する構造体。

失敗モードと境界条件

  • **エラー**: publish_via_partition_root を false に設定する際、既存のパーティションテーブルに行フィルタやカラムリストが存在する場合に ERRCODE_INVALID_PARAMETER_VALUE を発生させる。
  • **境界条件**: テーブルが並行して削除された場合、キャッシュルックアップが失敗する可能性があるが、その場合は静かにスキップする。

保守担当者向けメモ

  • 本関数はキャッシュ無効化の起点となるため、カタログ更新後の CommandCounterIncrement の呼び出し順序は厳守すること。
  • パーティションテーブルの階層構造を扱うため、GetPubPartitionOptionRelations を用いた再帰的なキャッシュ無効化が正しく行われているか注意が必要。
