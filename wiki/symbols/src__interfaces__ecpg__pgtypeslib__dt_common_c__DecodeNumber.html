DecodeNumber

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/dt_common.c:1198

目的

文字列として与えられた数値フィールドを解析し、日付や時刻の構成要素（年、月、日、通日など）として struct tm に反映させる。

シグネチャまたは形状

static int DecodeNumber(int flen, char *str, int fmask, int *tmask, struct tm *tm, fsec_t *fsec, bool *is2digits, bool EuroDates);

入力と出力

  • **入力**:
  • flen: 文字列の長さ。
  • str: 解析対象の文字列。
  • fmask: 現在までに解析済みのフィールドを示すビットマスク。
  • EuroDates: 欧州形式（日/月/年）を優先するかどうかのフラグ。
  • **出力**:
  • tmask: 今回の解析で特定されたフィールドを示すビットマスク。
  • tm: 更新される struct tm 構造体。
  • fsec: 小数秒（存在する場合）。
  • is2digits: 2桁の年であるかを示すフラグ。
  • 戻り値: 成功時は 0、失敗時は -1。

前提条件と不変条件

  • str は数値として解釈可能であること。
  • tm は呼び出し元で適切に初期化されていること。
  • 3桁の数値は「通日（DOY）」として扱われる場合がある。
  • 4桁以上の数値は原則として「年」として扱われる。

アルゴリズムまたは振る舞い

1. strtoint を用いて文字列を整数 val に変換する。 2. 小数点が含まれる場合、2桁以下の数値であれば小数秒として処理し、それ以外は DecodeNumberField へ委譲する。 3. flen == 3 かつ fmask に年が含まれる場合、val を通日（DOY）とみなし、j2date を用いて年・月・日に変換する。 4. flen >= 4 の場合、val を年として設定する。既に年が設定されている場合は、既存の値を日として再解釈を試みる。 5. 上記以外の場合、fmask の状態と EuroDates フラグに基づき、val を月または日として割り当てる。 6. 最後に、2桁または4桁以上の数値を年として割り当てるフォールバック処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/pgtypeslib/dt_common.c 内の解析関数（1398行目、2117行目など）。
  • **呼び出し先**: strtoint, strtod, DecodeNumberField, j2date, date2j。
  • 注: src/backend/utils/adt/datetime.c に同名の関数が存在するが、本ページは ecpg ライブラリ内の実装を対象としている。

関連データ構造

  • struct tm: 日付・時刻の分解値を保持する標準構造体。
  • fsec_t: 小数秒を保持する型。
  • DTK_M(...): フィールドマスク用の定数群。

失敗モードと境界条件

  • 文字列が数値として解析できない場合、または str が空の場合は -1 を返す。
  • 小数点以降の解析で不正な形式が検出された場合、-1 を返す。
  • どのフィールドにも適合しない数値が渡された場合、-1 を返す。

保守担当者向けメモ

  • 本関数は EuroDates の設定により、日付の解釈順序（日/月か月/日か）が動的に変化するため、修正時はこの条件分岐に注意すること。
  • 3桁の通日（DOY）処理は、年が既に確定していることを前提としている。
