StartupSUBTRANS

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:308

目的

PostgreSQL 起動時に、サブトランザクションログ（pg_subtrans）の現在アクティブなページをゼロクリアします。本モジュールはクラッシュリカバリを目的としないため、再起動時には既存のディスク上のデータを破棄し、初期化を行う必要があります。

シグネチャまたは形状

void StartupSUBTRANS(TransactionId oldestActiveXID)

入力と出力

  • **入力**: oldestActiveXID (現在アクティブな最古のトランザクションID)
  • **出力**: なし（副作用として共有メモリ上の SLRU ページおよびディスク上の対応ページがゼロクリアされる）

前提条件と不変条件

  • pg_subtrans はクラッシュリカバリをサポートしないため、起動時に内容をリセットすることが前提です。
  • TransamVariables->nextXid が有効な値として参照可能である必要があります。
  • ページ番号の計算において、TransactionId のラップアラウンドが考慮されます。

アルゴリズムまたは振る舞い

1. oldestActiveXID から開始ページ (startPage) を算出します。 2. TransamVariables->nextXid から終了ページ (endPage) を算出します。 3. startPage から endPage までループし、以下の処理を実行します。

  • SimpleLruGetBankLock を使用して、対象ページに対応する SLRU バンクロックを取得します。
  • ZeroSUBTRANSPage を呼び出し、対象ページをゼロクリアします。
  • ページ番号が MaxTransactionId を超える場合は 0 にラップアラウンドします。

    • 4. 最後に取得したロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c: リカバリ処理の過程で呼び出されます。
  • **呼び出し先**:
  • TransactionIdToPage: ページ番号の算出。
  • SimpleLruGetBankLock: SLRU バンクロックの取得。
  • LWLockAcquire / LWLockRelease: ロックの排他制御。
  • ZeroSUBTRANSPage: ページの初期化。

関連データ構造

  • SubTransCtl: SLRU 管理のための共有メモリ制御構造体。
  • TransamVariables: トランザクション管理のグローバル変数群。

失敗モードと境界条件

  • **ラップアラウンド**: startPage が MaxTransactionId を超える場合、正しく 0 にリセットされる設計になっています。
  • **ロック競合**: 複数のページを処理する際、バンクロックが切り替わるタイミングで適切にロックの解放と再取得が行われます。

保守担当者向けメモ

  • 本関数は pg_subtrans の初期化のみを担います。新しいページが必要になった際の拡張は ExtendSUBTRANS が担当します。
  • 起動時のゼロクリアは、ディスク上の古いデータが誤って参照されることを防ぐための重要な安全策です。
