IndexStateFlagsAction

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/index.h:24

目的

インデックスの状態遷移（有効化、無効化、削除準備など）を制御するためのアクションを定義する列挙型です。

シグネチャまたは形状

typedef enum
{
	INDEX_CREATE_SET_READY,
	INDEX_CREATE_SET_VALID,
	INDEX_DROP_CLEAR_VALID,
	INDEX_DROP_SET_DEAD,
} IndexStateFlagsAction;

入力と出力

  • 本シンボルは index_set_state_flags(Oid indexId, IndexStateFlagsAction action) 関数の引数として入力されます。
  • 出力値はありません（列挙型の各メンバは、インデックスの状態変更操作を識別する定数として機能します）。

前提条件と不変条件

  • index_set_state_flags を通じて、カタログ内のインデックス状態が適切に更新されることを前提としています。

アルゴリズムまたは振る舞い

  • INDEX_CREATE_SET_READY: インデックスを「準備完了（ready）」状態に設定します。
  • INDEX_CREATE_SET_VALID: インデックスを「有効（valid）」状態に設定します。
  • INDEX_DROP_CLEAR_VALID: インデックスの「有効」フラグを解除します。
  • INDEX_DROP_SET_DEAD: インデックスを「削除対象（dead）」状態に設定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/index.c 内の index_set_state_flags 関数がこの列挙型を利用します。
  • 呼び出し先: 情報なし。

関連データ構造

  • Oid: インデックスを特定するための識別子。

失敗モードと境界条件

  • 無効な IndexStateFlagsAction 値が渡された場合の挙動については、ソースコードの文脈からは定義されていません。

保守担当者向けメモ

  • 本列挙型は index_set_state_flags の動作を決定づけるため、インデックスのライフサイクル管理（特に並行作成や削除処理）に変更を加える際は、各フラグがカタログのどの状態に対応するかを src/backend/catalog/index.c の実装と照らし合わせて確認してください。
