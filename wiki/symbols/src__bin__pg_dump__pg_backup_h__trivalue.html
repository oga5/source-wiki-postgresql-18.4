trivalue

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/pg_backup.h:32

目的

ユーザーの選択や設定状態を「デフォルト」「いいえ」「はい」の3値で表現するための列挙型です。

シグネチャまたは形状

typedef enum trivalue
{
    TRI_DEFAULT,
    TRI_NO,
    TRI_YES,
} trivalue;

入力と出力

  • 入力: なし（列挙定数）
  • 出力: TRI_DEFAULT (0), TRI_NO (1), TRI_YES (2) のいずれかの整数値。

前提条件と不変条件

  • TRI_DEFAULT が初期値として使用されることが一般的です。

アルゴリズムまたは振る舞い

  • ユーザー入力のプロンプト制御や、設定の有効/無効を切り替える際の論理状態として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • contrib/vacuumlo/vacuumlo.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_dump/connectdb.c / connectdb.h
  • src/bin/pg_dump/pg_backup.h (メンバ変数 promptPassword)
  • src/bin/pg_dump/pg_backup_db.c
  • src/bin/pg_dump/pg_dumpall.c
  • src/bin/psql/command.c
  • src/bin/psql/settings.h
  • 呼び出し先: なし（本シンボルはデータ型定義です）。

関連データ構造

  • Archive 構造体（promptPassword メンバとして保持）
  • RestoreOptions / DumpOptions（文脈により関連）

失敗モードと境界条件

  • 列挙型のため、定義外の整数値が代入された場合の挙動は未定義です。

保守担当者向けメモ

  • 本シンボルは pg_dump だけでなく psql や vacuumlo など、複数の PostgreSQL クライアントツールで共通して利用されています。変更を加える際は、これら全ツールへの影響を考慮してください。
