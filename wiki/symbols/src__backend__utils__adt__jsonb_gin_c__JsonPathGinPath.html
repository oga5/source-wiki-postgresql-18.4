JsonPathGinPath

定義

  • 種別: union
  • 定義箇所: src/backend/utils/adt/jsonb_gin.c:125

目的

JSONBのGINインデックスにおいて、検索対象となるパス情報を保持するための共用体です。使用する演算子クラス（jsonb_ops または jsonb_path_ops）に応じて、パスの表現形式を切り替えます。

シグネチャまたは形状

typedef union JsonPathGinPath
{
    JsonPathGinPathItem *items; /* list of path items (jsonb_ops) */
    uint32      hash;           /* hash of the path (jsonb_path_ops) */
} JsonPathGinPath;

入力と出力

  • **入力**: jsonb_ops 使用時は JsonPathGinPathItem のリストへのポインタ、jsonb_path_ops 使用時はパスのハッシュ値（uint32）を受け取ります。
  • **出力**: 演算子クラスに応じた適切な形式でパス情報を保持し、インデックス抽出やノード抽出処理へ提供します。

前提条件と不変条件

  • jsonb_ops 演算子クラスを使用する場合、items フィールドが有効です。
  • jsonb_path_ops 演算子クラスを使用する場合、hash フィールドが有効です。
  • どちらのフィールドを使用するかは、呼び出し元のコンテキスト（演算子クラスの選択）に依存します。

アルゴリズムまたは振る舞い

  • jsonb_ops モードでは、パスを構成するアイテムのリストを保持し、詳細な構造比較を可能にします。
  • jsonb_path_ops モードでは、パス全体をハッシュ化した値を保持し、高速な一致判定を可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**: jsonb_ops__add_path_item, jsonb_path_ops__add_path_item, jsonb_ops__extract_nodes, jsonb_path_ops__extract_nodes, extract_jsp_path_expr_nodes, extract_jsp_path_expr などから参照されています。
  • **呼び出し先**: 本シンボル自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • JsonPathGinPathItem: jsonb_ops モードで使用されるパスアイテムのリスト構造。
  • JsonPathGinContext: インデックス抽出時のコンテキスト情報。
  • JsonPathItem: JSONPathの解析済みアイテム。

失敗モードと境界条件

  • 適切な演算子クラスと一致しないフィールドにアクセスした場合、不正なメモリ参照や誤ったハッシュ値の解釈が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は共用体であるため、アクセス時には現在使用している演算子クラスがどちらであるかを常に意識する必要があります。
  • jsonb_path_ops はハッシュ値のみを保持するため、構造の復元は不可能である点に注意してください。
