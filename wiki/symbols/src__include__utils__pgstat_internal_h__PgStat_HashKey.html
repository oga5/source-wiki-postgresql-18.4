PgStat_HashKey

定義

  • **種別**: struct
  • **定義箇所**: src/include/utils/pgstat_internal.h:52

目的

PostgreSQLの累積統計システムにおいて、共有メモリ上の統計エントリを一意に識別するためのハッシュキーとして機能します。

シグネチャまたは形状

typedef struct PgStat_HashKey
{
    PgStat_Kind kind;    /* 統計エントリの種類 */
    Oid         dboid;   /* データベースID（共有オブジェクトの場合はInvalidOid） */
    uint64      objid;   /* オブジェクトID（テーブル、関数等）または識別子 */
} PgStat_HashKey;

入力と出力

  • **入力**: 統計情報の種類、データベースOID、オブジェクトIDの組み合わせ。
  • **出力**: 共有ハッシュテーブル内での統計エントリ検索および識別に使用されるキー構造体。

前提条件と不変条件

  • 共有メモリ上の統計システムにおいて、統計エントリを特定するための固定的な識別子として使用されます。
  • 共有オブジェクトに関連する統計の場合、dboid には InvalidOid が設定されます。

アルゴリズムまたは振る舞い

  • 統計システム内のハッシュテーブル操作において、キーとして利用されます。
  • src/backend/utils/activity/pgstat.c 内の pgstat_hash_hash_key および pgstat_cmp_hash_key 関数により、ハッシュ値の計算やキーの比較が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c 内の統計情報検索、初期化、およびエントリ管理ロジックから頻繁に参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PgStatShared_HashEntry: 共有統計ハッシュテーブルの各エントリ。本構造体をキーとして保持します。
  • PgStat_Kind: 統計の種類を定義する列挙型。

失敗モードと境界条件

  • objid の解釈は kind に依存します。
  • 共有オブジェクトとデータベース固有オブジェクトの混同を避けるため、dboid の設定には注意が必要です。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置される可能性があるため、構造体のパディングやアライメントには注意を払ってください。
  • 統計種別を追加する際は、PgStat_Kind と併せて本構造体の objid が適切に識別子として機能するか確認してください。
