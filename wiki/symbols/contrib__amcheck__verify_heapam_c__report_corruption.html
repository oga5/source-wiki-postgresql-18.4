report_corruption

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_heapam.c:950

目的

ヒープリレーションの検証中に検出された破損情報を tuplestore に記録し、検証コンテキストの破損フラグを更新する。

シグネチャまたは形状

static void report_corruption(HeapCheckContext *ctx, char *msg)

入力と出力

  • **入力**:
  • ctx: 検証中の状態を保持する HeapCheckContext 構造体へのポインタ。
  • msg: 検出された破損内容を示すエラーメッセージ文字列。
  • **出力**: なし（副作用として ctx の状態を更新）。

前提条件と不変条件

  • ctx は有効な HeapCheckContext を指している必要がある。
  • ctx->tupstore および ctx->tupdesc が初期化されていること。

アルゴリズムまたは振る舞い

1. report_corruption_internal を呼び出し、ctx 内の tupstore、tupdesc、現在のブロック番号 (blkno)、オフセット番号 (offnum)、属性番号 (attnum)、およびメッセージ msg を渡してエラー情報を記録する。 2. ctx->is_corrupt を true に設定し、検証対象のリレーションに破損が存在することをマークする。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c 内の各検証ロジック（行番号 202, 528, 536, 552, 559, 566, 588, 595, 603, 680, 688, 721）。
  • **呼び出し先**: report_corruption_internal。

関連データ構造

  • HeapCheckContext: 検証中のリレーション情報や現在の走査位置、エラー蓄積用 tuplestore を保持する構造体。

失敗モードと境界条件

  • ctx が NULL の場合、あるいは ctx 内の tupstore が適切に初期化されていない場合はクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は破損検出時の標準的な報告手段である。破損が検出された場合、検証処理を即座に停止するか継続するかは呼び出し側のロジックに依存する。
