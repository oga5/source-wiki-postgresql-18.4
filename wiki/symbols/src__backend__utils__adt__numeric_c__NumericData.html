NumericData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/numeric.c:156

目的

PostgreSQL における NUMERIC 型（任意精度数値型）のオンディスク表現を定義する構造体です。

シグネチャまたは形状

struct NumericData
{
    int32       vl_len_;        /* varlena header */
    union NumericChoice choice; /* choice of format */
};

入力と出力

  • 本構造体は Numeric 型（typedef struct NumericData *Numeric）として、PostgreSQL のデータ型システム内で Datum として扱われます。
  • vl_len_ は PostgreSQL の可変長データ（varlena）ヘッダーであり、直接操作してはなりません。

前提条件と不変条件

  • vl_len_ フィールドは PostgreSQL の varlena 規約に従う必要があります。
  • choice フィールドは、数値の格納形式（Short 形式または Long 形式）を保持する共用体です。

アルゴリズムまたは振る舞い

  • NUMERIC 型の物理的な格納形式を提供します。
  • 小さな数値に対してはディスク容量を節約する NumericShort 形式を、それ以外には NumericLong 形式を選択的に使用する構造をサポートします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/numeric.c 内の算術演算および型変換関数群。
  • 呼び出し先: NumericChoice 共用体（src/include/utils/numeric.h で定義）。
  • 注: 具体的な関数レベルの呼び出し関係については、提供されたソース情報からは詳細な追跡が不可能なため、これ以上の情報は不明です。

関連データ構造

  • NumericChoice: NumericData 内の形式を選択する共用体。
  • NumericVar: 算術演算時に使用されるメモリ上の作業用構造体。
  • NumericShort / NumericLong: NumericData が内部的に取り得る具体的なデータ形式。

失敗モードと境界条件

  • vl_len_ を直接操作した場合、メモリ破損やデータ型の不整合を引き起こす可能性があります。
  • Numeric 型のメモリ管理は palloc に依存しており、適切に扱われない場合はメモリリークの原因となります。

保守担当者向けメモ

  • 本構造体は src/include/utils/numeric.h で Numeric 型として型定義（typedef）されています。
  • 内部構造を変更する際は、NumericShort および NumericLong との互換性、および varlena ヘッダーの整合性に注意してください。
