AtEOXact_SPI

定義

  • 種別: function
  • 定義箇所: src/backend/executor/spi.c:427

目的

トランザクション終了時（コミットまたはアボート）に、SPI スタックをクリーンアップし、SPI 接続状態を整合させる。

シグネチャまたは形状

void AtEOXact_SPI(bool isCommit)

入力と出力

  • isCommit (bool): トランザクションがコミットされる場合は true、アボートされる場合は false を指定する。
  • 出力: なし。

前提条件と不変条件

  • SPI スタック (_SPI_stack) が有効であること。
  • internal_xact フラグが設定された接続エントリに到達するまで、スタックを遡ってポップする。

アルゴリズムまたは振る舞い

1. _SPI_stack を末尾から順に走査する。 2. internal_xact が true である接続エントリに到達するまで、各エントリの outer_processed、outer_tuptable、outer_result をグローバル変数に復元し、スタックをポップする。 3. _SPI_connected を更新し、_SPI_current を適切なスタック位置へ再設定する。 4. 正常なコミット時にスタックが空でない（found が true）場合、SPI_finish の呼び出し漏れを示唆する警告をログに出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c: トランザクション終了処理の過程で呼び出される。
  • **呼び出し先**:
  • ereport: 警告出力時に使用。

関連データ構造

  • _SPI_stack: SPI 接続状態を保持する内部スタック。
  • _SPI_connection: スタック内の各要素を定義する構造体。

失敗モードと境界条件

  • トランザクション終了時に SPI スタックが空でない場合、コミット時は警告 (WARNING) が発生する。
  • _SPI_connected が負の値になった場合、_SPI_current は NULL に設定される。

保守担当者向けメモ

  • SPI_finish() と異なり、メモリコンテキストの明示的な解放は行わない（親コンテキストの終了時に自動的に解放されるため）。
  • トランザクション終了時のクリーンアップに特化しており、メモリコンテキストの切り替えによるリスクを回避している。
