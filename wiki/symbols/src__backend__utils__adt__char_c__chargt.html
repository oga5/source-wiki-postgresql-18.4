chargt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/char.c:162

目的

PostgreSQLの内部型 "char"（1バイト文字型）の2つの値に対し、「より大きい」という比較演算を実行します。

シグネチャまたは形状

Datum chargt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの "char" 型の値。
  • **出力**: 比較結果を示す bool 値（Datum 型として返却）。第1引数が第2引数より大きい場合に true を返します。

前提条件と不変条件

  • 入力値は1バイトの "char" 型として扱われます。
  • 比較は符号なし8ビット整数 (uint8) として実行されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_CHAR マクロを使用して、引数から2つの1バイト文字を取得します。 2. 取得した各文字を uint8 にキャストします。 3. キャスト後の値を比較し、第1引数が第2引数より大きいかどうかを判定します。 4. 判定結果を PG_RETURN_BOOL を通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（演算子呼び出し経由）。ソースコード内での直接的な呼び出し元は確認されていません。
  • **呼び出し先**: PG_GETARG_CHAR および PG_RETURN_BOOL マクロを使用します。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • "char": PostgreSQLの1バイト固定長文字型。

失敗モードと境界条件

  • 比較は符号なしとして行われるため、文字コードの数値的な大小関係に基づきます。
  • 境界条件として、0 から 255 までの全範囲の値が比較可能です。

保守担当者向けメモ

  • 本関数は uint8 として比較を行いますが、int4 との相互変換を行う関数群では int8（符号付き）として扱われるという歴史的な不整合が存在します。この挙動の差異に注意してください。
  • SQL標準の CHAR(n) 型（bpchar）とは異なる型を対象としているため、混同を避けてください。
