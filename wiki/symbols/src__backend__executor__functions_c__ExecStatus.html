ExecStatus

定義

  • 種別: enum
  • 定義箇所: src/backend/executor/functions.c:63

目的

SQL関数の実行プロセスにおける現在の状態を追跡・管理するために使用されます。

シグネチャまたは形状

typedef enum
{
    F_EXEC_START,
    F_EXEC_RUN,
    F_EXEC_DONE,
} ExecStatus;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • SQL関数の実行状態を保持する変数（例: execution_state 構造体内のメンバ）として使用されることが想定されています。

アルゴリズムまたは振る舞い

SQL関数の実行フローにおいて、以下の状態遷移を表現します。

  • F_EXEC_START: 関数の実行開始準備状態。
  • F_EXEC_RUN: 関数内のクエリを実行中。
  • F_EXEC_DONE: すべてのクエリの実行が完了した状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/functions.c 内の関数実行ロジック（execution_state を操作する箇所）。
  • 呼び出し先: 情報なし。

関連データ構造

  • execution_state: SQL関数内の各クエリの実行状態を管理する構造体。この中で ExecStatus 型のメンバが定義されています。

失敗モードと境界条件

  • 列挙型であるため、定義された値以外の値が代入された場合の挙動は未定義です。

保守担当者向けメモ

  • 本シンボルは src/backend/executor/functions.c 内のSQL関数実行エンジン専用です。
  • 状態遷移のロジックを変更する場合は、functions.c 内の実行ループ（fmgr_sql 等）における状態管理フローを確認してください。
