statext_dependencies_build

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/dependencies.c:347

目的

統計対象の列セットから関数従属性（Functional Dependencies）を計算し、MVDependencies 構造体として構築する。

シグネチャまたは形状

MVDependencies *statext_dependencies_build(StatsBuildData *data)

入力と出力

  • **入力**: StatsBuildData *data （統計構築に必要な属性情報やデータを含む構造体）
  • **出力**: MVDependencies * （計算された関数従属性のリスト。従属性が一つも検出されない場合は NULL を返す可能性がある）

前提条件と不変条件

  • data->nattnums >= 2 であること（アサーションにより強制）。
  • 内部計算用の一時メモリコンテキスト cxt を作成し、計算ごとにリセットすることでメモリリークを防ぐ。

アルゴリズムまたは振る舞い

1. 2列の組み合わせから data->nattnums 列までのすべての組み合わせを順次生成する。 2. DependencyGenerator を使用して、各サイズ k ごとの属性の組み合わせを列挙する。 3. dependency_degree 関数を呼び出し、各組み合わせの従属性の度合いを計算する。 4. 度合いが 0.0 より大きい場合のみ、MVDependency 構造体を割り当てて結果リストに追加する。 5. 構築した MVDependencies 構造体を返し、一時メモリコンテキストを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/extended_stats.c:207
  • **呼び出し先**:
  • DependencyGenerator_init, DependencyGenerator_next, DependencyGenerator_free
  • dependency_degree
  • palloc0, repalloc, MemoryContextSwitchTo, MemoryContextReset, MemoryContextDelete

関連データ構造

  • MVDependencies: 構築された従属性の集合を保持する。
  • MVDependency: 個別の関数従属性（属性リストと度合い）を保持する。
  • StatsBuildData: 統計構築の入力データ。
  • DependencyGenerator: 属性の組み合わせを生成するためのイテレータ。

失敗モードと境界条件

  • data->nattnums < 2 の場合、アサーション違反により停止する。
  • 従属性の度合いが 0.0 の場合、その組み合わせは結果に含めない。
  • メモリ不足時には palloc0 等がエラーを発生させる。

保守担当者向けメモ

  • 属性数に対して指数関数的に計算量が増加するため、大規模なテーブルでの ANALYZE 実行時にはパフォーマンスへの影響を考慮すること。
  • dependencies リストの拡張には repalloc を使用しており、ndeps の増加に伴いメモリ再配置が発生する。
