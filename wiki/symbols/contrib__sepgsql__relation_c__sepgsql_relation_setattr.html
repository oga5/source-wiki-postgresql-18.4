sepgsql_relation_setattr

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/relation.c:614

目的

リレーション（テーブル、ビュー、シーケンス等）の属性変更時におけるSELinuxセキュリティ権限の検証および、関連するスキーマ情報の更新を行う。

シグネチャまたは形状

void sepgsql_relation_setattr(Oid relOid)

入力と出力

  • **入力**: relOid (対象リレーションのOID)
  • **出力**: なし（権限チェック失敗時はエラーをスロー）

前提条件と不変条件

  • SnapshotSelf を使用してカタログ（pg_class）をスキャンするため、現在のトランザクション内で変更された最新のレコードを参照する。
  • インデックスに対しては sepgsql_index_modify を呼び出し、処理を委譲する。

アルゴリズムまたは振る舞い

1. **リレーション種別の判定**: get_rel_relkind により対象の種別を特定し、対応するSELinuxクラス（db_table, db_sequence, db_view）を決定する。 2. **カタログ情報の取得**: SnapshotSelf を用いて最新の pg_class レコードを取得し、同時に RELOID システムキャッシュから旧レコードを取得する。 3. **スキーマ変更の追跡**: 名前空間（relnamespace）やリレーション名が変更されている場合、sepgsql_schema_remove_name / sepgsql_schema_add_name / sepgsql_schema_rename を呼び出し、スキーマ関連の管理情報を更新する。 4. **権限検証**: sepgsql_avc_check_perms を呼び出し、対象オブジェクトに対して SEPG_DB_TABLE__SETATTR 権限があるかを検証する。 5. **リソース解放**: 取得したシステムキャッシュおよびカタログスキャンを終了し、メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/hooks.c:237
  • contrib/sepgsql/relation.c:748
  • **呼び出し先**:
  • sepgsql_index_modify (インデックスの場合)
  • sepgsql_schema_remove_name / sepgsql_schema_add_name / sepgsql_schema_rename
  • sepgsql_avc_check_perms
  • table_open / systable_beginscan / systable_getnext / systable_endscan / table_close
  • SearchSysCache1 / ReleaseSysCache

関連データ構造

  • Form_pg_class: pg_class カタログの構造体。
  • ObjectAddress: 権限チェック対象のオブジェクト識別子。

失敗モードと境界条件

  • 対象のOIDが pg_class に存在しない場合、elog(ERROR) により処理を中断する。
  • SELinuxポリシーにより SETATTR 権限が拒否された場合、sepgsql_avc_check_perms 内でエラーが発生する。

保守担当者向けメモ

  • インデックス操作は親テーブルの属性変更として扱われるため、本関数ではなく sepgsql_index_modify が処理する。
  • 将来的な拡張として、テーブルスペースの変更時にシステムカタログの db_tuple:{use} 権限をチェックする実装が検討されている。
