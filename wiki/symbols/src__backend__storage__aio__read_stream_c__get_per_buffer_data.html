get_per_buffer_data

定義

  • 種別: static inline function
  • 定義箇所: src/backend/storage/aio/read_stream.c:150

目的

ReadStream 構造体が管理するバッファごとの補助データ領域において、指定されたインデックスに対応するメモリ位置を計算して返します。

シグネチャまたは形状

void *get_per_buffer_data(ReadStream *stream, int16 buffer_index)

入力と出力

  • **入力**:
  • stream: 対象の ReadStream 構造体へのポインタ。
  • buffer_index: 取得対象のバッファインデックス（int16）。
  • **出力**:
  • 指定されたインデックスに対応する per_buffer_data 領域の先頭アドレスを void * として返します。

前提条件と不変条件

  • stream->per_buffer_data が有効なメモリ領域を指している必要があります。
  • buffer_index は、ストリームが確保したバッファ範囲内（0 以上、かつ割り当てられたバッファ数未満）である必要があります。

アルゴリズムまたは振る舞い

1. stream->per_buffer_data（データ領域のベースアドレス）を取得します。 2. stream->per_buffer_data_size（バッファごとのデータサイズ）に buffer_index を乗算します。 3. ベースアドレスに計算したオフセットを加算し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/read_stream.c 内の複数箇所（行 462, 906, 965 付近）で、バッファに関連付けられたメタデータへのアクセスに使用されています。
  • **呼び出し先**:
  • なし（算術演算のみを行うインライン関数）。

関連データ構造

  • ReadStream: ストリームの状態を保持する構造体。per_buffer_data（データ領域のポインタ）と per_buffer_data_size（各要素のサイズ）を保持します。

失敗モードと境界条件

  • buffer_index が負の値、またはストリームの容量を超えている場合、メモリ破壊や不正なメモリアクセスを引き起こす可能性があります。本関数自体には境界チェックは含まれていません。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、static inline として定義されています。変更を加える際は、パフォーマンスへの影響を考慮してください。
  • per_buffer_data_size は ReadStream 生成時に決定される固定値であるため、実行時のサイズ変更は想定されていません。
