BF_encode

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/crypt-gensalt.c:123

目的

Blowfish アルゴリズムで使用されるバイナリデータを、BF_itoa64 テーブルに基づいた Base64 風の ASCII 文字列へエンコードします。

シグネチャまたは形状

static void BF_encode(char *dst, const BF_word *src, int size)

入力と出力

  • dst: エンコード結果を格納する出力バッファ（呼び出し元で十分な領域を確保する必要がある）。
  • src: エンコード対象のバイナリデータ（BF_word 型の配列）。
  • size: エンコードするバイト数。
  • 出力: なし（dst に直接書き込まれる）。

前提条件と不変条件

  • src は size バイト以上の有効なメモリ領域を指していること。
  • BF_itoa64 テーブルが利用可能であること。
  • size は正の整数であること。

アルゴリズムまたは振る舞い

1. 入力バイナリを 3 バイトずつ読み込み、BF_itoa64 テーブルを用いて 4 文字の ASCII 文字列に変換します。 2. 変換ロジックは以下の通りです：

  • 1 バイト目から上位 6 ビットを取り出し、インデックスとして変換。
  • 残りの 2 ビットと 2 バイト目の上位 4 ビットを結合して変換。
  • 2 バイト目の下位 4 ビットと 3 バイト目の上位 2 ビットを結合して変換。
  • 3 バイト目の下位 6 ビットを変換。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pgcrypto/crypt-blowfish.c:404
  • contrib/pgcrypto/crypt-blowfish.c:747
  • contrib/pgcrypto/crypt-gensalt.c:183
  • 呼び出し先:
  • 本関数は外部関数を呼び出しません。

関連データ構造

  • BF_word: エンコード対象のデータ型。
  • BF_itoa64: 変換に使用される 64 文字の定数テーブル（本ファイル内で定義）。

失敗モードと境界条件

  • size が 0 の場合、ループは実行されず何も書き込まれません。
  • dst バッファが size に応じた十分な長さを保持していない場合、バッファオーバーランが発生する可能性があります。

保守担当者向けメモ

  • 本関数は crypt-blowfish.c が利用できない環境でも Blowfish のソルト生成を完結させるために crypt-gensalt.c に配置されています。
  • _rn 接尾辞を持つ関数群から呼び出される再入可能な実装です。
