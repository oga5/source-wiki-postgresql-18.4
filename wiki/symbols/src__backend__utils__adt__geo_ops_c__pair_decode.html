pair_decode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_ops.c:211

目的

文字列形式の幾何データ（座標ペア）を解析し、2つの float8 値に変換します。

シグネチャまたは形状

static bool pair_decode(char *str, float8 *x, float8 *y, char **endptr_p,
                        const char *type_name, const char *orig_string,
                        Node *escontext);

入力と出力

  • **入力**:
  • str: 解析対象の文字列。
  • type_name: エラー報告用の型名。
  • orig_string: エラー報告用の元の文字列全体。
  • escontext: エラー処理用ノード。
  • **出力**:
  • x, y: 解析された座標値が格納されるポインタ。
  • endptr_p: 解析終了位置を指すポインタ（NULL指定時は文字列末尾まで解析）。
  • 戻り値: 成功時に true、失敗時に false。

前提条件と不変条件

  • str は LDELIM（開始デリミタ）で囲まれているか、あるいはデリミタなしの形式である必要があります。
  • 座標間は DELIM で区切られている必要があります。
  • 浮動小数点数の解析には single_decode を利用します。

アルゴリズムまたは振る舞い

1. 先頭の空白をスキップします。 2. LDELIM が存在する場合、それを消費し、終了時に RDELIM を期待するフラグを立てます。 3. single_decode を呼び出し、最初の値（x）を抽出します。 4. DELIM を確認し、続いて single_decode で2番目の値（y）を抽出します。 5. RDELIM が期待されている場合、それを消費し、後続の空白をスキップします。 6. endptr_p が提供されている場合は解析終了位置を返し、提供されていない場合は文字列末尾（\0）であることを検証します。 7. 解析失敗時は ereturn マクロを介してエラーを報告し、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/geo_ops.c 内の各幾何型（点、円など）の入力関数から呼び出されます。
  • **呼び出し先**:
  • single_decode（座標値の解析）
  • ereturn（エラー報告）

関連データ構造

  • float8: 座標値を保持する型。
  • Node: エラーコンテキスト。

失敗モードと境界条件

  • 構文が不正な場合（デリミタの不一致、数値形式の誤り、末尾に余分な文字がある場合など）は false を返します。
  • escontext を通じて ERRCODE_INVALID_TEXT_REPRESENTATION エラーが発行されます。

保守担当者向けメモ

  • 本関数は geo_ops.c 内の複数の幾何型入出力処理で再利用されています。
  • LDELIM や RDELIM などのデリミタ定義は geo_decls.h に依存しています。
