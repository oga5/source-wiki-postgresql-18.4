WriteTempFileBlock

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuildbuffers.c:757

目的

GiST インデックスのバッファリングビルドにおいて、メモリ上のページデータを一時ファイル（BufFile）の指定されたブロック位置へ書き出す。

シグネチャまたは形状

static void WriteTempFileBlock(BufFile *file, long blknum, const void *ptr)

入力と出力

  • **入力**:
  • file: 書き込み先の一時ファイルハンドル（BufFile *）。
  • blknum: 書き込み対象のブロック番号（long）。
  • ptr: 書き込むデータが格納されたメモリ領域へのポインタ（const void *）。
  • **出力**: なし（成功時）。失敗時はエラーを投げる。

前提条件と不変条件

  • ptr は少なくとも BLCKSZ バイトの有効なデータを含んでいる必要がある。
  • file は有効な BufFile インスタンスであること。

アルゴリズムまたは振る舞い

1. BufFileSeekBlock を呼び出し、一時ファイル内の指定されたブロック番号 blknum へシークする。 2. シークに失敗した場合（戻り値が 0 以外）、elog(ERROR) を発生させて処理を中断する。 3. BufFileWrite を使用して、ptr から BLCKSZ バイトのデータをファイルへ書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuildbuffers.c 内の gistUnloadNodeBuffer 等から呼び出される（行 257, 371）。
  • **呼び出し先**:
  • BufFileSeekBlock
  • BufFileWrite
  • elog

関連データ構造

  • BufFile: storage/buffile.h で定義される一時ファイル操作用構造体。

失敗モードと境界条件

  • シーク操作が失敗した場合、elog(ERROR) によりトランザクションまたはプロセスが停止する。
  • BLCKSZ 単位での書き込みが行われるため、データサイズが BLCKSZ に満たない場合や、ファイルシステム上の容量制限に達した場合は BufFileWrite 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は gistbuildbuffers.c 内の静的ヘルパー関数であり、エラーハンドリングを elog(ERROR) に依存しているため、呼び出し側での個別のエラーチェックは不要である。
  • BLCKSZ は PostgreSQL の標準的なページサイズを指す。
