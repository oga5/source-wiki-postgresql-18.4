checkcondition_bit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsgistidx.c:307

目的

GiST インデックスの内部ノードにおいて、tsquery のオペランドがインデックスキー（シグネチャ）に含まれているかを判定する。

シグネチャまたは形状

static TSTernaryValue checkcondition_bit(void *checkval, QueryOperand *val, ExecPhraseData *data)

入力と出力

  • **入力**:
  • checkval: SignTSVector 構造体へのポインタ（インデックスキー）。
  • val: 照合対象の QueryOperand。
  • data: ExecPhraseData（本関数内では未使用）。
  • **出力**:
  • TS_MAYBE: 条件に合致する可能性がある場合。
  • TS_NO: 条件に合致しないことが確定している場合。

前提条件と不変条件

  • checkval は SignTSVector 型としてキャスト可能であること。
  • インデックスキーがビットベクトル形式（SIGNKEY）であることを前提としている。

アルゴリズムまたは振る舞い

1. val->prefix が真の場合、シグネチャツリーではプレフィックス検索をサポートできないため、常に TS_MAYBE を返す。 2. val->valcrc（ハッシュ値）と GETSIGLEN(key) を用いて、シグネチャ内の対応するビット位置を算出する。 3. GETBIT マクロにより、該当ビットが立っているかを確認する。

  • ビットが立っていれば TS_MAYBE を返す。
  • ビットが立っていなければ TS_NO を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: gtsvector_consistent 関数から TS_execute のコールバックとして呼び出される。
  • **呼び出し先**: GETBIT, GETSIGN, GETSIGLEN, HASHVAL などのマクロ。
  • *注記*: ツールにより contrib/intarray や contrib/ltree 内の同名関数への参照が検出されていますが、これらは別モジュールのシンボルであり、本関数との直接的な呼び出し関係はソースコード上からは確認できません。

関連データ構造

  • SignTSVector: GiST インデックスに格納されるキー構造体。
  • QueryOperand: 検索対象のクエリ条件。

失敗モードと境界条件

  • val->prefix が設定されている場合、インデックスによる絞り込みができず、常に TS_MAYBE を返すことで再チェックを強制する。

保守担当者向けメモ

  • 本関数は「不正確（inexact）」なインデックス検索の一部であり、TS_MAYBE を返した場合、後続のヒープ再チェックが必要となります。
  • siglen の値によってビットベクトルのサイズが動的に変化するため、GETSIGLEN マクロを介したアクセスが必須です。
