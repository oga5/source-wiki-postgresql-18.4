amvalidate

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/amapi.c:173

目的

指定された演算子クラス（opclass）の整合性を、対応するインデックス・アクセスメソッド（AM）の検証ロジックを介して確認する。

シグネチャまたは形状

Datum amvalidate(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) により、検証対象の演算子クラスの OID を取得する。
  • **出力**: PG_RETURN_BOOL(result) により、検証結果（bool）を返す。

前提条件と不変条件

  • pg_opclass システムカタログに指定された OID が存在すること。
  • 対象の AM が amvalidate 関数ポインタを IndexAmRoutine 構造体内に定義していること。

アルゴリズムまたは振る舞い

1. CLAOID システムキャッシュを使用して、指定された opclassoid に対応する pg_opclass タプルを取得する。 2. タプルから AM の OID (opcmethod) を抽出する。 3. GetIndexAmRoutineByAmId を呼び出し、該当 AM の IndexAmRoutine 構造体を取得する。 4. 取得した構造体の amvalidate 関数ポインタが NULL でないことを確認する（NULL の場合はエラーを投げる）。 5. AM 固有の検証関数を呼び出し、結果を保持する。 6. pfree で amroutine を解放し、検証結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルから呼び出される（直接的な C 言語レベルの呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**:
  • SearchSysCache1 / ReleaseSysCache
  • GetIndexAmRoutineByAmId
  • amroutine->amvalidate (各 AM 実装の検証関数)

関連データ構造

  • IndexAmRoutine: AM の機能ルーチンを保持する構造体。
  • Form_pg_opclass: pg_opclass カタログのタプル形式。

失敗モードと境界条件

  • **キャッシュ検索失敗**: 指定された OID が pg_opclass に存在しない場合、elog(ERROR) を発生させる。
  • **検証関数未定義**: AM が amvalidate を実装していない場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は SQL レベルのインターフェースであり、各 AM の検証ロジックへのディスパッチャーとして機能する。
  • GetIndexAmRoutineByAmId が返すメモリは palloc されているため、必ず pfree を行う必要がある。
