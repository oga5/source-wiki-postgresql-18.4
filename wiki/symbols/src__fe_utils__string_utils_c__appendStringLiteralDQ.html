appendStringLiteralDQ

定義

  • 種別: function
  • 定義箇所: src/fe_utils/string_utils.c:483

目的

文字列をPostgreSQLのドル引用符（Dollar Quoting）形式でエスケープし、指定されたバッファに追加する。

シグネチャまたは形状

void appendStringLiteralDQ(PQExpBuffer buf, const char *str, const char *dqprefix)

入力と出力

  • buf: 文字列を追加する対象の PQExpBuffer。
  • str: 引用符で囲む対象の文字列。
  • dqprefix: ドル引用符のタグに使用する接頭辞（NULL可）。
  • 出力: なし（buf の内容が更新される）。

前提条件と不変条件

  • buf は有効な PQExpBuffer であること。
  • str は NULL 終端文字列であること。
  • 生成されるドル引用符のタグ（$prefix_XXXXXXX$）は、元の文字列 str 内に含まれないことが保証される。

アルゴリズムまたは振る舞い

1. dqprefix を基に初期の区切りタグを生成する。 2. str 内に現在の区切りタグが存在するかを確認する。 3. 存在する場合、_XXXXXXX というパターンのサフィックスを順次追加し、str に含まれない一意なタグが見つかるまで繰り返す。 4. 確定したタグで str を囲み、buf に書き込む。 5. 内部で使用した一時バッファを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_dump.c（pg_dump における関数ソースの出力処理などで使用）。
  • 呼び出し先: createPQExpBuffer, appendPQExpBufferChar, appendPQExpBufferStr, destroyPQExpBuffer（いずれも libpq のバッファ操作関数）。

関連データ構造

  • PQExpBuffer: 文字列を動的に保持・拡張するための構造体。

失敗モードと境界条件

  • メモリ不足: createPQExpBuffer が失敗した場合、プログラムは終了する可能性がある。
  • str が非常に長い場合、タグの探索に時間がかかる可能性がある。

保守担当者向けメモ

  • 本関数はドル引用符を使用するため、エスケープ処理を簡略化できるが、タグの衝突回避ロジックがループ構造になっている点に注意すること。
  • suffixes 配列の変更は、タグ生成の衝突回避戦略に直接影響する。
