ParseComplexProjection

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_func.c:1911

目的

関数呼び出し構文（func(arg)）として記述された式が、実際には複合型に対する列アクセス（プロジェクション）である場合に、それを解決して FieldSelect ノードまたは Var ノードを生成する。

シグネチャまたは形状

Node *ParseComplexProjection(ParseState *pstate, const char *funcname, Node *first_arg, int location)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態。
  • funcname: 解決対象の属性名（列名）。
  • first_arg: 複合型を持つ式ノード。
  • location: 構文解析上の位置情報。
  • **出力**:
  • 解決に成功した場合: FieldSelect ノードまたは Var ノード。
  • 解決に失敗した場合: NULL。

前提条件と不変条件

  • first_arg は複合型（RECORD 型または行型）である必要がある。
  • funcname は、対象の複合型が持つ属性名と一致しなければならない。

アルゴリズムまたは振る舞い

1. **行全体変数の最適化**: first_arg が行全体を表す Var（varattno == InvalidAttrNumber）である場合、scanNSItemForColumn を呼び出し、名前空間から直接列を解決する。 2. **タプル記述子の取得**: 上記以外の場合、get_expr_result_tupdesc（first_arg が RECORD 型の場合は expandRecordVariable を併用）を用いて、式の戻り値の構造（TupleDesc）を取得する。 3. **属性照合**: 取得した TupleDesc 内の各属性を走査し、funcname と一致し、かつ削除されていない（!attisdropped）属性を探す。 4. **ノード生成**: 一致する属性が見つかった場合、FieldSelect ノードを構築し、対象の型情報（atttypid, atttypmod, attcollation）をコピーして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_func.c 内の ParseFuncOrColumn から複数箇所で呼び出されている。
  • **呼び出し先**:
  • GetNSItemByRangeTablePosn, scanNSItemForColumn
  • expandRecordVariable, get_expr_result_tupdesc
  • TupleDescAttr, makeNode

関連データ構造

  • ParseState: 解析状態を保持する構造体。
  • TupleDesc: 複合型の構造定義。
  • FieldSelect: 複合型から特定のフィールドを抽出するノード。
  • Var: 変数ノード。

失敗モードと境界条件

  • tupdesc が取得できない場合（NULL）、解決不能として NULL を返す。
  • funcname に一致する属性が存在しない場合、NULL を返す。
  • 削除済みの属性（attisdropped）は無視される。

保守担当者向けメモ

  • 本関数は ParseFuncOrColumn の補助的な役割を担う。
  • RECORD 型の解決には expandRecordVariable が必要であり、型解決の複雑さがこの関数に集約されている。
  • FieldSelect 生成時に属性の照合順序（attcollation）を正しく引き継ぐことが、後続の parse_collate.c での処理において重要である。
