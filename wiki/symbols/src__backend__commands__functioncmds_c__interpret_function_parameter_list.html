interpret_function_parameter_list

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/functioncmds.c:182

目的

関数、プロシージャ、または集約関数の定義時に指定された引数リストを解析し、型情報、モード（IN/OUT等）、デフォルト値、および戻り値の型を抽出・検証する。

シグネチャまたは形状

void interpret_function_parameter_list(ParseState *pstate,
                                      List *parameters,
                                      Oid languageOid,
                                      ObjectType objtype,
                                      oidvector **parameterTypes,
                                      List **parameterTypes_list,
                                      ArrayType **allParameterTypes,
                                      ArrayType **parameterModes,
                                      ArrayType **parameterNames,
                                      List **inParameterNames_list,
                                      List **parameterDefaults,
                                      Oid *variadicArgType,
                                      Oid *requiredResultType)

入力と出力

  • **入力**: pstate（パーサー状態）、parameters（FunctionParameterのリスト）、languageOid（言語）、objtype（オブジェクト種別）。
  • **出力**: parameterTypes（oidvector）、parameterTypes_list（型リスト）、allParameterTypes、parameterModes、parameterNames（各配列）、inParameterNames_list、parameterDefaults、variadicArgType、requiredResultType。

前提条件と不変条件

  • parameters リスト内の各要素は FunctionParameter 構造体であること。
  • 指定された型はシステムカタログに存在し、呼び出し元に ACL_USAGE 権限があること。
  • VARIADIC パラメータは、入力パラメータの最後である必要がある。

アルゴリズムまたは振る舞い

1. 引数リストを走査し、各引数の型を LookupTypeName で解決する。 2. シェル型（未定義型）の利用をチェックし、言語やオブジェクト種別に応じてエラーまたは通知を発生させる。 3. ACL_USAGE 権限を検証する。 4. setof 引数の使用を禁止する（関数、プロシージャ、集約関数すべてでエラー）。 5. fpmode に基づき、入力パラメータ（inTypes）と出力パラメータを分類する。 6. VARIADIC パラメータの場合、その型が配列型であることを検証する。 7. プロシージャの場合、出力パラメータが存在すれば戻り値型を RECORDOID に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/aggregatecmds.c:307
  • src/backend/commands/functioncmds.c:1174
  • **呼び出し先**:
  • LookupTypeName, typeTypeId, ReleaseSysCache, object_aclcheck, aclcheck_error_type, get_element_type 等のユーティリティ関数。

関連データ構造

  • FunctionParameter: 引数の定義を保持する構造体。
  • ParseState: パーサーの状態管理。
  • oidvector: 引数型の配列表現。
  • pg_type: 型情報を格納するシステムカタログ。

失敗モードと境界条件

  • **型未定義**: LookupTypeName が失敗した場合、エラーを報告。
  • **権限不足**: ACL_USAGE がない場合、aclcheck_error_type でエラー。
  • **不正な引数順序**: VARIADIC の後に他の入力パラメータがある場合、エラー。
  • **不正な型**: 集約関数やプロシージャで setof 引数を使用した場合、エラー。

保守担当者向けメモ

  • 本関数は pg_proc の引数関連フィールドを構築するための中心的なロジックです。
  • プロシージャと関数の間で OUT パラメータの扱いが微妙に異なるため（特に CALL 文との兼ね合い）、修正時は objtype による条件分岐に注意してください。
