pg_log_generic_v

定義

  • 種別: function
  • 定義箇所: src/common/logging.c:218

目的

PostgreSQLフロントエンドプログラムにおいて、可変長引数を受け取り、ログレベルや設定に基づいた整形済みメッセージを stderr へ出力する。

シグネチャまたは形状

void pg_log_generic_v(enum pg_log_level level, enum pg_log_part part,
                      const char *pg_restrict fmt, va_list ap)

入力と出力

  • **入力**:
  • level: ログの重要度（pg_log_level 列挙型）。
  • part: メッセージの役割（pg_log_part 列挙型）。
  • fmt: フォーマット文字列。
  • ap: フォーマット引数リスト。
  • **出力**: なし（void）。副作用として stderr への書き出しを行う。

前提条件と不変条件

  • progname が初期化されていること。
  • level および fmt が非NULLであること。
  • fmt の末尾に改行文字を含めないこと（関数内で自動付与されるため）。

アルゴリズムまたは振る舞い

1. **フィルタリング**: level が現在のグローバル閾値 __pg_log_level 未満であれば即座に終了する。 2. **同期**: stdout をフラッシュし、出力順序を保証する。 3. **フック実行**: 設定されていれば log_pre_callback および log_locus_callback を実行し、位置情報を取得する。 4. **ヘッダー出力**: log_flags に応じてプログラム名、ファイル名、行番号を stderr に出力する。 5. **接頭辞付与**: part と level に応じて「error:」「warning:」「detail:」「hint:」等の接頭辞を付与する。 6. **メッセージ構築**: vsnprintf でメッセージを構築し、末尾の改行を調整した上で stderr に出力する。メモリ不足時は vfprintf で直接出力するフォールバックを備える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/logging.c 内のラッパー関数。
  • src/bin/pg_combinebackup/load_manifest.c
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • src/bin/pg_walsummary/pg_walsummary.c
  • **呼び出し先**:
  • log_pre_callback (設定時)
  • log_locus_callback (設定時)
  • vsnprintf, vfprintf, fprintf, fflush, free, pg_malloc_extended

関連データ構造

  • enum pg_log_level: ログレベル定義。
  • enum pg_log_part: ログパーツ定義。
  • __pg_log_level: 現在のログ出力閾値。
  • log_flags: 出力制御フラグ。

失敗モードと境界条件

  • pg_malloc_extended が失敗した場合、メモリ確保をスキップして vfprintf による直接出力へ切り替える。
  • required_len が2未満の場合、改行のストリップ処理は行われない。

保守担当者向けメモ

  • 本関数はフロントエンド専用であり、バックエンドコードでの使用は禁止されている。
  • 直接呼び出すのではなく、pg_log_error() 等のヘッダーで定義されたマクロ経由での利用を推奨する。
