get_tablespace

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/spccache.c:106

目的

指定されたテーブルスペースOIDに対応するキャッシュエントリ（TableSpaceCacheEntry）を取得します。キャッシュに存在しない場合は、システムカタログ pg_tablespace から情報を読み込み、キャッシュを構築して返します。

シグネチャまたは形状

static TableSpaceCacheEntry *get_tablespace(Oid spcid)

入力と出力

  • **入力**: spcid (Oid) - 対象のテーブルスペースOID。InvalidOid の場合は MyDatabaseTableSpace が使用されます。
  • **出力**: TableSpaceCacheEntry * - キャッシュされたテーブルスペース情報へのポインタ。

前提条件と不変条件

  • TableSpaceCacheHash が初期化されている必要があります（未初期化時は内部で InitializeTableSpaceCache() が呼び出されます）。
  • pg_tablespace の読み込み処理は、キャッシュエントリの作成前に行う必要があります（読み込み中のキャッシュフラッシュを避けるため）。

アルゴリズムまたは振る舞い

1. **OID解決**: spcid が InvalidOid ならばデフォルトのテーブルスペースに置換します。 2. **キャッシュ検索**: TableSpaceCacheHash を検索し、エントリが見つかれば即座に返します。 3. **カタログ参照**: キャッシュミス時、SearchSysCache1 を用いて pg_tablespace から情報を取得します。 4. **オプション解析**: カタログの spcoptions 属性を tablespace_reloptions で解析し、CacheMemoryContext に割り当てます。 5. **キャッシュ登録**: 解析結果を TableSpaceCacheEntry に格納し、ハッシュテーブルに登録して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • get_tablespace_page_costs (src/backend/utils/cache/spccache.c:186)
  • get_tablespace_io_concurrency (src/backend/utils/cache/spccache.c:217)
  • get_tablespace_maintenance_io_concurrency (src/backend/utils/cache/spccache.c:231)
  • **呼び出し先**:
  • InitializeTableSpaceCache
  • hash_search
  • SearchSysCache1
  • SysCacheGetAttr
  • tablespace_reloptions
  • MemoryContextAlloc
  • ReleaseSysCache

関連データ構造

  • TableSpaceCacheEntry: キャッシュの各エントリを保持する構造体。
  • TableSpaceCacheHash: キャッシュを管理するハッシュテーブル。
  • TableSpaceOpts: テーブルスペースのオプション設定を保持する構造体。

失敗モードと境界条件

  • **存在しないOID**: SearchSysCache1 でエントリが見つからない場合、オプションは NULL として扱われ、キャッシュエントリが作成されます。
  • **オプション未設定**: spcoptions が NULL の場合、オプションは NULL として扱われます。

保守担当者向けメモ

  • 本関数は static であり、モジュール内部でのみ使用されます。
  • キャッシュの無効化は InvalidateTableSpaceCacheCallback によって行われ、ハッシュテーブル全体がクリアされる戦略をとっています。
