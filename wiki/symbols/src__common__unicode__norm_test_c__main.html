目的

Unicode 正規化機能（unicode_normalize）の正確性を検証するためのテストスイートを実行する。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • 入力: argc および argv（本関数内では使用されていない）。
  • 出力: テスト成功時は 0 を返し、失敗時は 1 を返して異常終了する。

前提条件と不変条件

  • UnicodeNormalizationTests 配列が norm_test_table.h に定義されており、終端が input[0] == 0 であること。
  • unicode_normalize 関数が 0 から 3 までの正規化形式（NFC, NFD, NFKC, NFKD）を正しく処理できること。

アルゴリズムまたは振る舞い

1. UnicodeNormalizationTests 配列を順次イテレートする。 2. 各テストケースに対し、4 つの正規化形式（0〜3）をループで適用する。 3. unicode_normalize を呼び出し、結果を取得する。 4. pg_wcscmp を使用して、期待値（test->output[form]）と計算結果を比較する。 5. 不一致がある場合、print_wchar_str を用いて入力、期待値、結果を標準出力にダンプし、exit(1) で終了する。 6. 全テストケースが完了した時点で「All tests successful!」と出力し、exit(0) で正常終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（テストプログラムの実行時）。
  • 呼び出し先: unicode_normalize, pg_wcscmp, print_wchar_str, printf, exit。
  • 補足: ツールによる contrib/amcheck 等の参照結果は、本シンボルとは無関係な「main relation」に関する言及であり、本関数の呼び出し関係を示すものではない。

関連データ構造

  • pg_unicode_test: テストケースを保持する構造体。
  • UnicodeNormalizationTests: テストデータの配列（norm_test_table.h で定義）。
  • pg_wchar: 正規化対象および結果の文字列型。

失敗モードと境界条件

  • 正規化結果が期待値と一致しない場合、エラーメッセージを出力して exit(1) で終了する。
  • テストデータ配列の終端条件（input[0] == 0）に達するまでループを継続する。

保守担当者向けメモ

  • 本関数はスタンドアロンのテストツールであり、サーバー本体のコードとは独立している。
  • print_wchar_str が静的バッファを使用しているため、本テストはシングルスレッドで実行する必要がある。
