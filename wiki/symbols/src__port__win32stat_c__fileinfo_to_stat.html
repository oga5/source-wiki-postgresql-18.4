fileinfo_to_stat

定義

  • 種別: function
  • 定義箇所: src/port/win32stat.c:67

目的

Windows API の BY_HANDLE_FILE_INFORMATION 構造体から取得したファイルメタデータを、PostgreSQL が内部で使用する POSIX 互換の struct stat 形式へ変換する。

シグネチャまたは形状

static int fileinfo_to_stat(HANDLE hFile, struct stat *buf);

入力と出力

  • **入力**:
  • hFile: 対象ファイルの Windows ハンドル。
  • buf: 変換結果を格納する struct stat 構造体へのポインタ。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は -1 を返し、GetLastError() を通じてエラーコードを errno にマッピングする。

前提条件と不変条件

  • hFile は有効なファイルハンドルであること。
  • buf は書き込み可能なメモリ領域を指していること。
  • Windows XP および Windows Server 2003 以降の環境で動作することを前提としている。

アルゴリズムまたは振る舞い

1. memset で buf をゼロクリアする。 2. GetFileInformationByHandle を呼び出し、ファイル情報を取得する。失敗した場合は _dosmaperr でエラーを処理して終了する。 3. ftLastWriteTime、ftLastAccessTime、ftCreationTime をそれぞれ filetime_to_time 関数を用いて time_t 形式に変換し、st_mtime、st_atime、st_ctime に代入する。

  • アクセス時刻や作成時刻がゼロの場合は、更新時刻（st_mtime）で補完する。

    • 4. fileattr_to_unixmode を呼び出し、Windows のファイル属性を POSIX の st_mode に変換する。 5. nNumberOfLinks を st_nlink に、ファイルサイズ（上位・下位 32bit）を st_size に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/port/win32stat.c 内の _pglstat64 および _pgfstat64 から呼び出される。
  • **呼び出し先**:
  • GetFileInformationByHandle (Windows API)
  • _dosmaperr
  • filetime_to_time
  • fileattr_to_unixmode

関連データ構造

  • BY_HANDLE_FILE_INFORMATION: Windows API のファイル情報構造体。
  • struct stat: POSIX 準拠のファイル状態構造体。

失敗モードと境界条件

  • GetFileInformationByHandle が失敗した場合、関数は -1 を返し、呼び出し元にエラーを伝播させる。
  • ファイルサイズが 4GB を超える場合、nFileSizeHigh と nFileSizeLow をビットシフトおよび論理和で結合し、64bit 整数として正しく処理する。

保守担当者向けメモ

  • 本関数は Windows 固有のファイルシステム情報を POSIX 形式にマッピングする重要な変換層である。
  • fileattr_to_unixmode は所有者の権限のみを考慮する仕様であるため、グループや他人の権限ビットは設定されない点に留意すること。
