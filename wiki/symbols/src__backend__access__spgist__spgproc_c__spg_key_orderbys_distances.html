spg_key_orderbys_distances

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgproc.c:62

目的

SP-GiSTインデックスの順序付けスキャン（KNN探索）において、インデックスキーと複数のスキャンキー間の距離を計算し、その結果を配列として提供します。

シグネチャまたは形状

double *spg_key_orderbys_distances(Datum key, bool isLeaf, ScanKey orderbys, int norderbys);

入力と出力

  • **入力**:
  • key: 距離計算対象となるインデックスキー（Datum）。
  • isLeaf: key がリーフ（点）か内部ノード（ボックス）かを指定するフラグ。
  • orderbys: スキャンキーの配列。
  • norderbys: スキャンキーの数。
  • **出力**:
  • 計算された距離を格納した double 型の配列（palloc によりメモリ確保）。

前提条件と不変条件

  • orderbys が指す各スキャンキーの引数は、Point 型として解釈可能である必要があります。
  • isLeaf が true の場合、key は Point 型として解釈されます。
  • isLeaf が false の場合、key は軸平行な BOX 型として解釈されます。

アルゴリズムまたは振る舞い

1. norderbys 分の double 配列をメモリ確保します。 2. 各スキャンキーに対してループ処理を行います。 3. isLeaf の値に基づき、以下のいずれかの距離計算関数を呼び出します。

  • isLeaf == true: point_point_distance を使用。
  • isLeaf == false: point_box_distance を使用。

    • 4. 計算結果を配列に格納し、最後に配列のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgkdtreeproc.c
  • src/backend/access/spgist/spgquadtreeproc.c
  • src/backend/utils/adt/geo_spgist.c
  • **呼び出し先**:
  • point_point_distance
  • point_box_distance
  • palloc (PostgreSQLメモリ管理)

関連データ構造

  • ScanKey: スキャン条件を保持する構造体。
  • Point, BOX: 幾何計算に使用されるデータ型。

失敗モードと境界条件

  • メモリ不足時に palloc がエラーを発生させる可能性があります。
  • key や orderbys->sk_argument が期待される型（Point または BOX）と一致しない場合、データ破損やクラッシュの原因となります。

保守担当者向けメモ

  • 本関数は spgquadtreeproc.c や spgkdtreeproc.c における最近傍探索の基盤です。
  • 新しい幾何データ型を追加する際は、isLeaf の判定ロジックおよび距離計算関数の拡張が必要になる場合があります。
