ResourceOwnerForgetOSSLDigest

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/openssl.c:83

目的

PostgreSQL の ResourceOwner 管理下にある OSSLDigest オブジェクトの追跡を解除します。これにより、トランザクション終了時やエラー発生時に、当該リソースが自動的に解放対象として処理されることを防ぎます。

シグネチャまたは形状

static inline void ResourceOwnerForgetOSSLDigest(ResourceOwner owner, OSSLDigest *digest)

入力と出力

  • owner: リソースを所有する ResourceOwner オブジェクト。
  • digest: 追跡を解除する OSSLDigest 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • digest は、事前に ResourceOwnerRememberOSSLDigest を通じて owner に登録されている必要があります。
  • 本関数は ResourceOwnerForget をラップしており、ossldigest_resowner_desc をリソース記述子として使用します。

アルゴリズムまたは振る舞い

1. digest ポインタを PointerGetDatum により Datum 型に変換します。 2. ResourceOwnerForget を呼び出し、指定された owner から digest の登録を削除します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/openssl.c 内のコード（digest のライフサイクル終了時）。
  • 呼び出し先: ResourceOwnerForget（PostgreSQL コアのリソース管理関数）。

関連データ構造

  • OSSLDigest: ハッシュ計算用のコンテキストを保持する構造体。
  • ResourceOwner: PostgreSQL のリソース管理機構。

失敗モードと境界条件

  • owner が無効な場合や、digest が当該 owner に登録されていない場合の挙動は、ResourceOwnerForget の実装に依存します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、pgcrypto モジュール内でのリソースリーク防止を目的としています。
  • ResourceOwner への登録・解除の対（ResourceOwnerRememberOSSLDigest / ResourceOwnerForgetOSSLDigest）が正しく維持されていることを確認してください。
