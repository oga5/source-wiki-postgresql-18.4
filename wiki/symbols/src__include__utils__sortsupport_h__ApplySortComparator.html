ApplySortComparator

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/sortsupport.h:199

目的

SortSupport フレームワークを利用して、NULL 値の扱いとソート順序（昇順/降順）を考慮したデータ比較を行うための標準化されたインターフェースを提供します。

シグネチャまたは形状

static inline int ApplySortComparator(Datum datum1, bool isNull1,
                                      Datum datum2, bool isNull2,
                                      SortSupport ssup)

入力と出力

  • **入力**:
  • datum1, datum2: 比較対象のデータ（Datum 型）。
  • isNull1, isNull2: 各データが NULL かどうかを示すフラグ。
  • ssup: ソート設定を保持する SortSupport 構造体へのポインタ。
  • **出力**:
  • int: 比較結果。datum1 < datum2 なら負の値、datum1 > datum2 なら正の値、等価なら 0 を返します。

前提条件と不変条件

  • ssup は有効な SortSupportData を指している必要があります。
  • ssup->comparator は、NULL 以外のデータに対する比較関数として適切に初期化されている必要があります。
  • NULL の配置（ssup_nulls_first）およびソート順序（ssup_reverse）は、SortSupport 構造体内で一貫して管理されている必要があります。

アルゴリズムまたは振る舞い

1. **NULL 判定**: どちらか一方または両方が NULL の場合、ssup->ssup_nulls_first に基づいて NULL を先頭または末尾に配置するよう比較結果を決定します。両方 NULL の場合は 0 を返します。 2. **データ比較**: 両方とも NULL でない場合、ssup->comparator を呼び出して比較を実行します。 3. **順序の反転**: ssup->ssup_reverse が真の場合、比較結果の符号を反転（INVERT_COMPARE_RESULT）させ、降順ソートを実現します。

関連データ構造

  • SortSupportData: 比較関数やソートオプションを保持する構造体。

失敗モードと境界条件

  • ssup が NULL の場合、デリファレンスによりクラッシュします。
  • ssup->comparator が未設定（NULL）の場合、実行時にクラッシュします。

保守担当者向けメモ

  • 本関数はインライン化されており、ソートのループ内での呼び出しコストを最小化する設計です。変更を加える際は、パフォーマンスへの影響を考慮してください。
  • 省略キー（Abbreviated Key）を使用する場合、本関数は完全な比較が必要な局面で利用されます。
