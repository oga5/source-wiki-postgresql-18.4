split_pathtarget_item

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/tlist.c:41

目的

PathTarget 内のサブ式を管理し、ソートやグループ化の参照情報（sortgroupref）を保持するためのデータ構造です。主に SRF（セットを返す関数）を含む式をプランナが分解・再構築する際に使用されます。

シグネチャまたは形状

typedef struct {
    Node   *expr;           /* PathTarget のサブ式 */
    Index   sortgroupref;   /* sortgroupref、設定がない場合は 0 */
} split_pathtarget_item;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではありません。
  • add_sp_item_to_pathtarget 関数等において、PathTarget への追加対象として入力されます。

前提条件と不変条件

  • expr は有効な Node ツリーである必要があります。
  • sortgroupref は、ソートやグループ化のメタデータとして利用されない場合は 0 を保持します。

アルゴリズムまたは振る舞い

  • PathTarget の分解プロセスにおいて、式ツリーの走査結果を一時的に保持します。
  • add_sp_item_to_pathtarget を通じて、重複する式がないかを確認しつつ PathTarget に統合されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/tlist.c 内の SRF 分解ロジック（split_pathtarget_at_srfs 関連処理）からインスタンス化されます。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PathTarget: 本構造体が管理する式を保持するターゲットリスト表現。
  • Node: expr フィールドが参照する基底ノード型。

失敗モードと境界条件

  • メモリ不足時: palloc による割り当て失敗が発生する可能性があります。
  • 重複登録: add_sp_item_to_pathtarget は、同一の式を持つ項目が既に存在する場合、追加を抑制するロジックを含んでいます。

保守担当者向けメモ

  • 本構造体は tlist.c 内の SRF 分解処理に特化した一時的なコンテナです。
  • sortgroupref の値が 0 の場合、その式はソートやグループ化のキーとして明示的に参照されていないことを示します。
