transformTargetList

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_target.c:120

目的

構文解析後のターゲットリスト（ResTargetのリスト）を、プランナーが処理可能なTargetEntryのリストへ変換する。ワイルドカード（*）の展開や、代入処理に伴うメタデータの付与を行う。

シグネチャまたは形状

List *transformTargetList(ParseState *pstate, List *targetlist, ParseExprKind exprKind)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持するParseState構造体。
  • targetlist: 変換対象のResTargetノードのリスト。
  • exprKind: 式のコンテキストを示すParseExprKind（例: EXPR_KIND_UPDATE_SOURCEなど）。
  • **出力**:
  • 変換済みのTargetEntryノードを含むList。

前提条件と不変条件

  • pstate->p_multiassign_exprs は関数開始時に空（NIL）でなければならない。
  • UPDATE 処理時以外は、pstate->p_multiassign_exprs は関数終了時に空に戻される。

アルゴリズムまたは振る舞い

1. **ワイルドカード展開の判定**: exprKind が EXPR_KIND_UPDATE_SOURCE でない場合、something.* の展開を有効化する。 2. **リストの走査**: targetlist の各要素を順次処理する。

  • ColumnRef または A_Indirection で末尾が A_Star の場合、ExpandColumnRefStar または ExpandIndirectionStar を呼び出し、展開結果をリストに追加する。
  • それ以外の場合は、transformTargetEntry を呼び出し、単一の式として変換してリストに追加する。

    • 3. **マルチアサインの処理**: UPDATE 文で生成された p_multiassign_exprs が存在する場合、それを結果リストの末尾に連結し、pstate の該当フィールドをクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c（SELECT句、RETURNING句、INSERT/UPDATEのターゲットリスト処理など）
  • **呼び出し先**:
  • ExpandColumnRefStar
  • ExpandIndirectionStar
  • transformTargetEntry

関連データ構造

  • ParseState: 解析状態を管理。
  • ResTarget: 構文解析直後のターゲット項目。
  • TargetEntry: 意味解析後のターゲット項目。
  • ColumnRef, A_Indirection, A_Star: 構文解析ツリーのノード。

失敗モードと境界条件

  • UPDATE 文の実行時、p_multiassign_exprs が存在する場合は必ず結果リストに統合される。
  • * 展開は UPDATE のソース式（SET 句の右辺など）では抑制される。

保守担当者向けメモ

  • 本関数は SELECT や INSERT、UPDATE のターゲットリスト構築の共通エントリポイントである。
  • * 展開のロジックは、ColumnRef と A_Indirection の両方に対応しているため、新しい構文要素を追加する際は注意が必要。
  • p_multiassign_exprs の扱いは UPDATE 文特有の挙動であるため、変更時は transformUpdateStmt との整合性に留意すること。
