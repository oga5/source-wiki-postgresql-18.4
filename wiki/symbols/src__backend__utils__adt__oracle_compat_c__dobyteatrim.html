dobyteatrim

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:554

目的

bytea 型のデータに対して、指定されたバイトセットに含まれる文字を文字列の先頭（ltrim）および末尾（rtrim）から除去する共通処理を提供します。

シグネチャまたは形状

bytea *dobyteatrim(bytea *string, bytea *set, bool doltrim, bool dortrim)

入力と出力

  • **入力**:
  • string: 対象となる bytea 型データ。
  • set: 除去対象となるバイト値を含む bytea 型データ。
  • doltrim: 先頭からの除去を行うかを示すフラグ。
  • dortrim: 末尾からの除去を行うかを示すフラグ。
  • **出力**: 除去処理後の新しい bytea 型データ（palloc によりメモリ確保）。

前提条件と不変条件

  • string または set の長さが 0 以下の場合、処理を行わず入力された string をそのまま返します。
  • 処理後のデータ長に応じたメモリ領域が palloc で確保されます。

アルゴリズムまたは振る舞い

1. string と set のデータ長を取得します。 2. doltrim が真の場合、string の先頭から順に set 内のいずれかのバイトと一致するかを確認し、一致する間ポインタを進め、長さを減らします。 3. dortrim が真の場合、string の末尾から順に set 内のいずれかのバイトと一致するかを確認し、一致する間ポインタを戻し、長さを減らします。 4. 最終的な長さに基づいて新しい bytea 構造体を確保し、結果をコピーして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/oracle_compat.c 内の btrim（両端）、ltrim（先頭）、rtrim（末尾）相当の関数から呼び出されます。
  • **呼び出し先**:
  • palloc: メモリ確保に使用。
  • memcpy: 結果のコピーに使用。

関連データ構造

  • bytea: PostgreSQL の可変長バイナリデータ型。VARSIZE_ANY_EXHDR や VARDATA_ANY マクロを使用して内部データにアクセスします。

失敗モードと境界条件

  • 入力データが空（長さ 0）の場合、即座に終了します。
  • set に含まれるバイトが string の全バイトと一致する場合、結果の長さは 0 となります。

保守担当者向けメモ

  • 本関数は bytea 型専用であり、マルチバイト文字の境界を考慮する dotrim とは異なり、単純なバイト単位の走査を行います。
  • palloc を使用しているため、呼び出し元でメモリ管理が適切に行われていることを確認してください。
