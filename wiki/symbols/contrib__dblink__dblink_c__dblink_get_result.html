dblink_get_result

定義

  • 種別: function
  • 定義箇所: contrib/dblink/dblink.c:708

目的

リモートデータベースから非同期クエリの結果セットを取得し、PostgreSQL のレコードセットとして返却します。

シグネチャまたは形状

Datum dblink_get_result(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（関数呼び出し時の引数コンテキスト）。
  • **出力**: Datum 型（リモートクエリの結果セットを含む Tuplestore の SRF）。

前提条件と不変条件

  • dblink_send_query 等によって、既にリモート接続上で非同期クエリが開始されている必要があります。
  • 内部的に dblink_record_internal を呼び出すため、その関数が要求するメモリコンテキストおよび接続状態の要件に準拠します。

アルゴリズムまたは振る舞い

1. dblink_record_internal を呼び出します。 2. その際、第2引数に true を渡し、非同期結果取得モードであることを明示します。 3. リモートからの結果を Tuplestore にマテリアライズし、呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: dblink_record_internal（ソースコード上の直接的な呼び出し）。

関連データ構造

  • PG_FUNCTION_ARGS: 関数呼び出しの引数とコンテキスト情報。
  • Tuplestore: 結果セットを保持するために使用される内部構造。

失敗モードと境界条件

  • リモート接続が確立されていない場合や、クエリが実行されていない場合はエラーが発生します。
  • リモート側でエラーが発生した場合、dblink_res_error を経由してローカルの ereport として通知されます。

保守担当者向けメモ

  • 本関数は dblink_record_internal の単なるラッパーです。ロジックの変更が必要な場合は dblink_record_internal を確認してください。
  • 非同期処理の特性上、接続状態の管理については remoteConn 構造体のライフサイクルに依存します。
