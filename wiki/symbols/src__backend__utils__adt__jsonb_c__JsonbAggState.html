JsonbAggState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonb.c:36

目的

SQL 集約関数（jsonb_agg や jsonb_object_agg 等）の実行中、中間状態を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct JsonbAggState
{
	JsonbInState *res;
	JsonTypeCategory key_category;
	Oid			key_output_func;
	JsonTypeCategory val_category;
	Oid			val_output_func;
} JsonbAggState;

入力と出力

  • **入力**: 集約関数の遷移関数（transfn）において、PG_GETARG_POINTER(0) を介して現在の状態として取得されます。
  • **出力**: 状態更新後、次の遷移関数呼び出しのためにポインタとして返却されます。

前提条件と不変条件

  • 集約関数の実行コンテキスト（メモリコンテキスト）において、palloc を用いて割り当てられる必要があります。
  • res フィールドは、構築中の JSONB 構造を管理する JsonbInState を指します。

アルゴリズムまたは振る舞い

集約関数の状態遷移において、以下の役割を果たします。 1. **状態保持**: 複数の行にわたる集約処理の間、JsonbInState を通じて構築中の JSONB データを保持します。 2. **型情報の管理**: キーおよび値の型カテゴリ（JsonTypeCategory）と、それらを文字列化するための出力関数（Oid）を保持し、各行の入力値を適切に変換して JSONB 構造へ追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonb.c 内の集約関数関連の実装（jsonb_agg_transfn 等の遷移関数および最終化関数）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、内部の res フィールドを通じて JsonbInState 関連の操作関数が利用されます。

関連データ構造

  • JsonbInState: 構築中の JSONB 状態を管理する構造体。
  • JsonTypeCategory: 入力値の型カテゴリを識別するための列挙型。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • コンテキスト不整合: 集約関数の実行コンテキスト外で本構造体を参照しようとした場合、メモリ保護違反や不正なポインタ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は jsonb_agg 系の集約関数に特化しています。新しい集約関数を追加する際は、key_category や val_category の設定が適切に行われているか確認してください。
  • JsonbInState のライフサイクルと同期しているため、メモリコンテキストの切り替え時には注意が必要です。
