OutputPluginUpdateProgress

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logical.c:719

目的

論理デコーディングの進捗状況を更新し、出力プラグインに対して現在の書き込み位置とトランザクションIDを通知する。

シグネチャまたは形状

void OutputPluginUpdateProgress(struct LogicalDecodingContext *ctx, bool skipped_xact)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングの状態を保持する LogicalDecodingContext 構造体へのポインタ。
  • skipped_xact: 現在のトランザクションがスキップされたかどうかを示すフラグ。
  • **出力**: なし（void）。

前提条件と不変条件

  • ctx が有効な LogicalDecodingContext を指していること。
  • ctx->update_progress が設定されている場合のみ、プラグイン側のコールバックが実行される。

アルゴリズムまたは振る舞い

1. ctx->update_progress が存在するか確認する。存在しない場合は何もせず終了する。 2. 存在する場合、ctx 内の現在の書き込み位置 (write_location)、トランザクションID (write_xid)、および引数 skipped_xact を渡して ctx->update_progress コールバックを呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/logical.c
  • src/backend/replication/pgoutput/pgoutput.c
  • **呼び出し先**:
  • ctx->update_progress（LogicalDecodingContext に登録された関数ポインタ）

関連データ構造

  • LogicalDecodingContext: 論理デコーディングのコンテキストを保持する構造体。update_progress コールバック関数ポインタを保持する。

失敗モードと境界条件

  • ctx->update_progress が NULL の場合、関数は安全に早期リターンする。

保守担当者向けメモ

  • 本関数は src/include/replication/output_plugin.h で外部公開されている。
  • pgoutput プラグイン等から頻繁に呼び出されるため、パフォーマンスへの影響を考慮すること。
