shm_mq_attach

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_mq.c:289

目的

共有メモリメッセージキュー (shm_mq) にアクセスするためのプライベートハンドル (shm_mq_handle) を作成し、初期化する。

シグネチャまたは形状

shm_mq_handle *shm_mq_attach(shm_mq *mq, dsm_segment *seg, BackgroundWorkerHandle *handle);

入力と出力

  • **入力**:
  • mq: 接続対象の共有メモリキューへのポインタ。
  • seg: キューが配置されている動的共有メモリ（DSM）セグメント（NULL可）。
  • handle: バックグラウンドワーカーのハンドル（NULL可）。
  • **出力**:
  • 成功時: 初期化された shm_mq_handle 構造体へのポインタ。

前提条件と不変条件

  • 呼び出し元プロセスが、キューの受信側 (mq_receiver) または送信側 (mq_sender) として登録されている必要がある (Assert(mq->mq_receiver == MyProc || mq->mq_sender == MyProc))。

アルゴリズムまたは振る舞い

1. shm_mq_handle 構造体用にメモリを割り当てる。 2. 引数で受け取ったキュー情報、セグメント、ワーカーハンドルをハンドル構造体に格納する。 3. バッファ状態、保留中の送受信バイト数、フラグ類を初期化する。 4. seg が指定されている場合、on_dsm_detach を使用して、セグメントデタッチ時に shm_mq_detach_callback が呼び出されるよう登録する。 5. 初期化済みのハンドルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c
  • src/backend/executor/execParallel.c
  • src/backend/replication/logical/applyparallelworker.c
  • **呼び出し先**:
  • palloc (メモリ割り当て)
  • on_dsm_detach (セグメントデタッチ時のコールバック登録)

関連データ構造

  • shm_mq: 共有メモリ上のキュー本体。
  • shm_mq_handle: プロセスごとのキューアクセス用プライベートハンドル。
  • dsm_segment: 動的共有メモリセグメント。
  • BackgroundWorkerHandle: バックグラウンドワーカーの管理用ハンドル。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • 前提条件（MyProc の一致）が満たされない場合、アサーション違反によりプロセスが停止する。

保守担当者向けメモ

  • 本関数は shm_mq_handle の各フィールドをゼロまたはデフォルト値で初期化する。特に mqh_context に CurrentMemoryContext を保持するため、ハンドルの生存期間とメモリコンテキストの整合性に注意すること。
