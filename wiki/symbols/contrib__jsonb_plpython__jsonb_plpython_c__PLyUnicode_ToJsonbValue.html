PLyUnicode_ToJsonbValue

定義

  • 種別: function
  • 定義箇所: contrib/jsonb_plpython/jsonb_plpython.c:81

目的

Python の文字列オブジェクトを PostgreSQL の JsonbValue 構造体に変換し、JSONB 文字列型として格納する。

シグネチャまたは形状

static void PLyUnicode_ToJsonbValue(PyObject *obj, JsonbValue *jbvElem)

入力と出力

  • **入力**:
  • obj: 変換対象の Python オブジェクト（文字列型を想定）。
  • jbvElem: 変換結果を格納する JsonbValue 構造体へのポインタ。
  • **出力**: なし（jbvElem を直接書き換える）。

前提条件と不変条件

  • obj は PLyObject_AsString が処理可能な Python オブジェクトであること。
  • jbvElem は有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. jbvElem->type を jbvString に設定する。 2. PLyObject_AsString(obj) を呼び出し、Python オブジェクトを C 文字列に変換して jbvElem->val.string.val に代入する。 3. strlen を用いて文字列長を計算し、jbvElem->val.string.len に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/jsonb_plpython/jsonb_plpython.c:297
  • contrib/jsonb_plpython/jsonb_plpython.c:427
  • **呼び出し先**:
  • PLyObject_AsString (外部関数、plpy_typeio.c 等で定義)
  • strlen (標準ライブラリ)

関連データ構造

  • PyObject: Python のオブジェクト表現。
  • JsonbValue: PostgreSQL の JSONB 要素を表現する構造体。

失敗モードと境界条件

  • PLyObject_AsString が内部でエラーを発生させた場合、PostgreSQL のエラーハンドリング機構により処理が中断される可能性がある。
  • obj が文字列として解釈できない場合、PLyObject_AsString の実装依存の挙動となる。

保守担当者向けメモ

  • 本関数は文字列変換専用である。数値や他の型については PLyNumber_ToJsonbValue 等の別関数が担当する。
  • PLyObject_AsString の戻り値のメモリ管理（解放の要否）については、plpython 側の規約に従うこと。
