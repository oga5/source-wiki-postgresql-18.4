hstore_exec_setup

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_subs.c:263

目的

hstore 型に対する添字演算の実行に必要な関数ポインタを、PostgreSQL の実行エンジンへ登録するために使用されます。

シグネチャまたは形状

static void hstore_exec_setup(const SubscriptingRef *sbsref, SubscriptingRefState *sbsrefstate, SubscriptExecSteps *methods)

入力と出力

  • **入力**:
  • sbsref: 添字演算ノード。
  • sbsrefstate: 添字演算の実行状態を保持する構造体。
  • methods: 実行エンジンが呼び出す関数ポインタを格納する構造体。
  • **出力**: なし（methods 構造体の各フィールドを更新することで結果を返します）。

前提条件と不変条件

  • sbsrefstate->numlower が 0 であること（単一の添字のみをサポート）。
  • sbsrefstate->numupper が 1 であること（キー指定が一つであることを保証）。

アルゴリズムまたは振る舞い

1. Assert を使用して、添字の数が期待通り（下限なし、上限1つ）であることを検証します。 2. methods 構造体に以下の関数ポインタを割り当てます。

  • sbs_check_subscripts: NULL（チェック不要）。
  • sbs_fetch: hstore_subscript_fetch を設定。
  • sbs_assign: hstore_subscript_assign を設定。
  • sbs_fetch_old: NULL。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_subs.c 内の hstore_subscript_handler 構造体初期化時に参照されます。
  • **呼び出し先**: 実行エンジンが methods を通じて hstore_subscript_fetch および hstore_subscript_assign を呼び出します。

関連データ構造

  • SubscriptingRef: 添字演算の構文ノード。
  • SubscriptingRefState: 実行時の添字状態。
  • SubscriptExecSteps: 実行ステップを定義する関数ポインタ群。

失敗モードと境界条件

  • numlower や numupper が前提条件を満たさない場合、Assert によりデバッグビルド環境で停止します。

保守担当者向けメモ

  • 本関数は hstore の添字操作のセットアップを担うエントリポイントです。将来的に添字の検証ロジックが必要になった場合は、sbs_check_subscripts に適切な関数を割り当ててください。
