pq_sendint8

定義

  • 種別: static inline function
  • 定義箇所: src/include/libpq/pqformat.h:127

目的

通信バッファ（StringInfo）に対して、8ビット整数（uint8）を安全に追加するために使用されます。

シグネチャまたは形状

static inline void pq_sendint8(StringInfo buf, uint8 i)

入力と出力

  • **入力**:
  • buf: データを書き込む対象の StringInfo 構造体へのポインタ。
  • i: 書き込む対象の uint8 型の値。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • buf は有効な StringInfo 構造体を指している必要があります。
  • 内部で enlargeStringInfo を呼び出すため、メモリ確保が失敗した場合は呼び出し元で例外が発生する可能性があります。

アルゴリズムまたは振る舞い

1. enlargeStringInfo(buf, sizeof(uint8)) を呼び出し、バッファに1バイト分の空き容量を確保します。 2. pq_writeint8(buf, i) を呼び出し、バッファの現在位置に値を書き込みます。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • enlargeStringInfo
  • pq_writeint8
  • **呼び出し元（直接的な証拠に基づく）**:
  • contrib/ltree/ltree_io.c
  • contrib/ltree/ltxtquery_io.c
  • src/backend/replication/logical/proto.c
  • src/backend/utils/adt/jsonb.c
  • src/backend/utils/adt/jsonpath.c
  • src/backend/utils/adt/tsquery.c

関連データ構造

  • StringInfo: 通信データのバッファを管理する構造体。

失敗モードと境界条件

  • buf が NULL の場合、enlargeStringInfo 内部でクラッシュする可能性があります。
  • メモリ不足時には enlargeStringInfo がエラーを発生させます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁な呼び出しによるオーバーヘッドを抑える設計となっています。
  • バッファの容量が既に確保されていることが保証されている場合は、pq_writeint8 を直接使用することで最適化が可能です。
