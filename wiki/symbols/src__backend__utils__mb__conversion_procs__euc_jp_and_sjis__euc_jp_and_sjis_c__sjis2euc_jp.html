sjis2euc_jp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c:640

目的

Shift-JIS（SJIS）形式の文字列をEUC_JP形式に変換します。

シグネチャまたは形状

static int sjis2euc_jp(const unsigned char *sjis, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • sjis: 変換元のSJIS文字列へのポインタ。
  • p: 変換後のEUC_JP文字列を格納するバッファへのポインタ。
  • len: 変換対象のバイト長。
  • noError: 不正な文字を検出した際にエラーを報告せず中断するかどうかを指定するフラグ。
  • **出力**:
  • 戻り値: 変換されたバイト数。
  • p: 変換後の文字列（ヌル終端済み）。

前提条件と不変条件

  • sjis は有効なSJISバイト列であること。
  • p は変換結果を格納するのに十分な領域を確保していること。
  • 変換処理は len が0になるまで継続される。

アルゴリズムまたは振る舞い

1. 入力文字列を1文字ずつ走査する。 2. ASCII文字（高ビットが立っていない文字）はそのままコピーする。 3. JIS X0201（1バイトカナ）は SS2 を付与して変換する。 4. JIS X0208/X0212および拡張文字（IBM漢字、UDC）については、以下の処理を行う：

  • ibmkanji テーブルを参照し、NEC選定IBM漢字やIBM漢字の変換を行う。
  • 範囲に応じてJIS X0208またはJIS X0212（SS3 を付与）へマッピングする。
  • マッピング不明なNEC選定IBM漢字は PGEUCALTCODE に置換する。

    • 5. 変換終了後、バッファ p にヌル文字を書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の sjis_to_euc_jp 関数（src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c:90）から呼び出される。
  • **呼び出し先**:
  • pg_encoding_verifymbchar: 文字列の妥当性検証。
  • report_invalid_encoding: 不正なエンコーディング検出時のエラー報告。

関連データ構造

  • ibmkanji: SJISとEUC_JP間の特殊な文字変換テーブル。
  • PGEUCALTCODE: マッピングが定義されていない場合の代替コード。

失敗モードと境界条件

  • pg_encoding_verifymbchar が負の値を返した場合、noError が偽であれば report_invalid_encoding を呼び出しエラーを発生させる。
  • 入力文字列が0バイトの場合、または不正な文字が含まれる場合、noError が真であれば処理を中断する。

保守担当者向けメモ

  • 本関数は sjis.map で定義されたマッピングテーブルに強く依存している。拡張文字の追加や変更が必要な場合は、テーブル構造の確認が必要である。
