applyMemoryGroupPermissions

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/SectionMemoryManager.cpp:327

目的

JITコンパイルされたメモリ領域（MemoryGroup）に対して、指定された権限を適用し、管理状態を更新する。

シグネチャまたは形状

std::error_code applyMemoryGroupPermissions(MemoryGroup &MemGroup, unsigned Permissions);

入力と出力

  • **入力**:
  • MemGroup: 権限変更対象となるメモリグループ（PendingMemおよびFreeMemを保持）。
  • Permissions: 適用するメモリ保護フラグ（mprotect相当）。
  • **出力**:
  • std::error_code: 成功時はデフォルト値（エラーなし）、失敗時はOSレベルのエラーコードを返す。

前提条件と不変条件

  • MemGroup.PendingMem に含まれるすべてのメモリブロックは、この関数呼び出し後に権限が変更され、リストからクリアされる。
  • FreeMem 内のブロックは、ページ境界に合わせて調整（trimBlockToPageSize）される必要がある。

アルゴリズムまたは振る舞い

1. MemGroup.PendingMem 内の各 MemoryBlock に対して、MMapper->protectMappedMemory を呼び出し、指定された Permissions を適用する。 2. 権限適用が完了した PendingMem リストをクリアする。 3. MemGroup.FreeMem 内の各ブロックに対し、trimBlockToPageSize を適用してページ境界に合わせる。 4. FreeMem 内の各ブロックの PendingPrefixIndex を無効値（-1）にリセットする。 5. FreeMem 内でサイズが 0 になったブロックを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/SectionMemoryManager.cpp:278 (CodeMemの処理)
  • src/backend/jit/llvm/SectionMemoryManager.cpp:288 (RODataMemの処理)
  • **呼び出し先**:
  • MMapper->protectMappedMemory
  • trimBlockToPageSize
  • erase_if (標準ライブラリまたはユーティリティ)

関連データ構造

  • MemoryGroup: メモリブロックの管理単位。
  • MemoryBlock: 確保されたメモリ領域の抽象。
  • FreeMemBlock: 空きメモリ領域の管理構造。
  • MMapper: メモリ保護操作を実行する MemoryMapper インスタンス。

失敗モードと境界条件

  • protectMappedMemory がエラーを返した場合、その時点で処理を中断し、エラーコードを呼び出し元へ伝播させる。
  • FreeMem 内のブロックがページサイズ未満にトリミングされた結果、サイズが 0 になった場合は自動的に破棄される。

保守担当者向けメモ

  • 本関数は finalizeMemory の一部として実行され、JIT生成コードの実行権限を確定させる重要なステップである。
  • PendingMem のクリアと FreeMem の再調整は不可分な操作として設計されているため、変更時は整合性に注意すること。
