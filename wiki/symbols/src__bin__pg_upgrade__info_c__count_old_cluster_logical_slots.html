count_old_cluster_logical_slots

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:743

目的

旧クラスタ内に存在する論理レプリケーションスロットの総数を算出します。

シグネチャまたは形状

int count_old_cluster_logical_slots(void);

入力と出力

  • 入力: なし（グローバル変数 old_cluster を参照）
  • 出力: int 型の論理スロット総数

前提条件と不変条件

  • old_cluster が初期化され、各データベースの論理スロット情報（slot_arr）が適切に読み込まれている必要があります。

アルゴリズムまたは振る舞い

1. slot_count を 0 で初期化します。 2. old_cluster.dbarr に格納されている全データベースを走査します。 3. 各データベースの slot_arr.nslots（論理スロット数）を slot_count に加算します。 4. 最終的な合計値を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_upgrade/check.c:2067
  • src/bin/pg_upgrade/function.c:109
  • src/bin/pg_upgrade/pg_upgrade.c:209
  • 呼び出し先: なし（本関数は計算のみを行い、他の関数を呼び出しません）

関連データ構造

  • old_cluster: pg_upgrade における旧クラスタのメタデータを保持するグローバル変数。
  • DbInfo: old_cluster.dbarr の各要素。
  • LogicalSlotInfoArr: DbInfo 内で論理スロット情報を保持する構造体。

失敗モードと境界条件

  • old_cluster.dbarr.dbs が空の場合、0 を返します。
  • データベースごとのスロット数が未設定（0）の場合、そのデータベースは加算対象外となります。

保守担当者向けメモ

  • 本関数は論理スロットの存在確認や移行前のチェック処理において頻繁に使用されます。
  • PostgreSQL 17 未満のクラスタでは論理スロットの概念が存在しないため、呼び出し側でバージョンチェックが行われているか確認してください。
