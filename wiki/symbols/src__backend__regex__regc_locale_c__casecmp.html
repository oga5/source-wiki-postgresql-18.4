casecmp

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_locale.c:757

目的

正規表現のバックリファレンス照合時などに使用される、大文字小文字を区別しない文字列比較を行う。

シグネチャまたは形状

int casecmp(const chr *x, const chr *y, size_t len)

入力と出力

  • **入力**:
  • x: 比較対象の文字列（chr 型ポインタ）
  • y: 比較対象の文字列（chr 型ポインタ）
  • len: 比較する文字数
  • **出力**:
  • 0: 両方の文字列が一致（または大文字小文字変換後に一致）した場合。
  • nonzero: 一致しなかった場合。

前提条件と不変条件

  • x および y は、少なくとも len 分の要素を持つ有効なメモリ領域を指している必要がある。
  • pg_wc_tolower 関数が、現在のロケール設定に基づいて適切に小文字変換を行うことが前提となる。

アルゴリズムまたは振る舞い

1. 指定された len 分だけ、x と y の各文字を順次比較する。 2. 各位置において、*x と *y が等しいか、あるいは pg_wc_tolower(*x) と pg_wc_tolower(*y) が等しければ、その文字は一致とみなす。 3. いずれの条件も満たさない（不一致）場合、直ちに 1 を返して終了する。 4. 全ての文字で一致が確認された場合、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regcomp.c: 正規表現コンパイル時に、REG_ICASE フラグが指定されている場合に g->compare 関数ポインタとして設定される。
  • **呼び出し先**:
  • pg_wc_tolower: ロケール依存の小文字変換関数。

関連データ構造

  • chr: 正規表現エンジン内で使用される文字型。

失敗モードと境界条件

  • len が 0 の場合、ループは実行されず 0 を返す（空文字列同士の比較として一致とみなされる）。
  • pg_wc_tolower の挙動はデータベースのエンコーディングおよびロケール設定に依存する。

保守担当者向けメモ

  • 本関数は regcomp.c からも参照されるため、シグネチャの変更には注意が必要である。
  • regcomp.c:534 にある通り、大文字小文字を区別しないマッチングが必要な場合にのみ動的に選択される。
