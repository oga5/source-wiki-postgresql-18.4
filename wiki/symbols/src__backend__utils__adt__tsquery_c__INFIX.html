INFIX

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsquery.c:967

目的

ポーランド記法（前置記法）で表現された tsquery 構造を、人間が読み取り可能な中置記法（infix）の文字列へ変換する際の作業状態を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	QueryItem  *curpol;
	char	   *buf;
	char	   *cur;
	char	   *op;
	int			buflen;
} INFIX;

入力と出力

  • **入力**: tsquery の内部表現である QueryItem 配列（curpol）および出力用バッファ。
  • **出力**: buf に格納された中置記法の文字列。

前提条件と不変条件

  • curpol は有効な tsquery のポーランド記法配列を指している必要があります。
  • buf は変換後の文字列を格納するのに十分なサイズ（buflen）を確保している必要があります。

アルゴリズムまたは振る舞い

  • tsquery.c 内の関数 infix がこの構造体を使用して再帰的にツリーを走査します。
  • curpol を順次読み進め、演算子とオペランドを適切な括弧付きの中置形式に並べ替えて buf に書き込みます。
  • 演算子の優先順位に基づき、必要に応じて括弧を挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery.c 内の infix 関数（再帰的に自身を呼び出す）。
  • contrib/intarray/_int_bool.c および contrib/ltree/ltxtquery_io.c においても同名の構造体が定義・利用されています。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • QueryItem: tsquery の内部表現を構成する要素。
  • TSQuery: tsquery 型の全体構造。

失敗モードと境界条件

  • buflen を超える文字列生成が発生した場合、バッファオーバーフローのリスクがあります（呼び出し側の実装に依存）。
  • 不正なポーランド記法構造が curpol に渡された場合、変換処理が異常終了または不正な文字列を生成する可能性があります。

保守担当者向けメモ

  • 本構造体は tsquery.c だけでなく、contrib 配下のモジュールでも同様の定義が重複して存在します。修正を行う際は、これらの関連ファイルへの影響範囲を確認してください。
  • 再帰処理を行うため、スタック深さの制限に注意が必要です。
