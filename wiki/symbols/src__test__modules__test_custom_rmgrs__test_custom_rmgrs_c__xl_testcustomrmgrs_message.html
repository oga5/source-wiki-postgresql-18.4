xl_testcustomrmgrs_message

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_custom_rmgrs/test_custom_rmgrs.c:32

目的

カスタムWALリソースマネージャがWAL（Write-Ahead Log）に書き込むメッセージデータのレイアウトを定義する構造体です。

シグネチャまたは形状

typedef struct xl_testcustomrmgrs_message
{
    Size        message_size;    /* メッセージのサイズ */
    char        message[FLEXIBLE_ARRAY_MEMBER]; /* ペイロード */
} xl_testcustomrmgrs_message;

入力と出力

  • **入力**: WAL書き込み時に message_size と可変長の message を保持します。
  • **出力**: WALレコードの解析時（descコールバック等）に、この構造体へキャストすることで格納されたデータにアクセス可能です。

前提条件と不変条件

  • message フィールドは FLEXIBLE_ARRAY_MEMBER として定義されており、構造体の末尾に可変長のデータが続くことを前提としています。
  • SizeOfTestCustomRmgrsMessage マクロ（offsetofを使用）により、ヘッダー部分のサイズが計算可能です。

アルゴリズムまたは振る舞い

  • WALレコード挿入時、この構造体形式でデータが構築されます。
  • WAL読み込み時、rec ポインタを xl_testcustomrmgrs_message * にキャストすることで、格納されたメッセージ内容を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • test_custom_rmgrs_insert_wal_record 関数内でレコード構築時に使用。
  • testcustomrmgrs_desc 関数内でWALレコードのデコード時に使用。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（データ構造体のため）。

関連データ構造

  • RmgrData: この構造体を用いたレコードを処理するカスタムRmgrの定義に使用されます。

失敗モードと境界条件

  • message_size と実際の message の長さが一致しない場合、データの読み取り時にメモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はテストモジュール用です。実運用環境で同様の構造体を設計する場合は、アライメントやパディングに注意し、pg_attribute_packed 等の属性付与を検討してください。
