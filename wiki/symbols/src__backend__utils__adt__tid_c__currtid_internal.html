currtid_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:295

目的

指定されたリレーションにおいて、特定の TID (Tuple Identifier) が指すタプルの最新バージョンを検索し、その TID を取得する。

シグネチャまたは形状

ItemPointer currtid_internal(Relation rel, ItemPointer tid)

入力と出力

  • **入力**:
  • rel: 対象リレーションの記述子。
  • tid: 検索対象の初期 TID。
  • **出力**:
  • ItemPointer: 最新のタプル位置を指す新しい ItemPointer 構造体。

前提条件と不変条件

  • 呼び出し元は、対象リレーションに対する ACL_SELECT 権限を保持している必要がある。
  • リレーションがビュー (RELKIND_VIEW) である場合、currtid_for_view へ処理が委譲される。
  • ストレージを持たないリレーション（RELKIND_HAS_STORAGE が偽）に対して呼び出した場合、エラーが発生する。

アルゴリズムまたは振る舞い

1. palloc を使用して結果格納用の ItemPointerData をメモリ確保する。 2. pg_class_aclcheck を呼び出し、現在のユーザーに SELECT 権限があるか検証する。権限がない場合は aclcheck_error を投げる。 3. リレーションがビューであれば currtid_for_view を呼び出し、その結果を返す。 4. ストレージを持たないリレーションであれば ERRCODE_FEATURE_NOT_SUPPORTED エラーを投げる。 5. ItemPointerCopy で入力 tid を結果領域にコピーする。 6. GetLatestSnapshot を用いてスナップショットを登録し、table_beginscan_tid でスキャンを開始する。 7. table_tuple_get_latest_tid を呼び出し、最新の TID を特定する。 8. スキャンを終了し、スナップショットを登録解除して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tid.c:399 (currtid_byrelname 内)
  • src/backend/utils/adt/tid.c:430 (currtid_for_view 内)
  • **呼び出し先**:
  • pg_class_aclcheck, aclcheck_error, currtid_for_view
  • table_beginscan_tid, table_tuple_get_latest_tid, table_endscan
  • RegisterSnapshot, UnregisterSnapshot

関連データ構造

  • ItemPointer / ItemPointerData: タプルの物理位置（ブロック番号とオフセット番号）を保持する構造体。
  • Relation: リレーションのメタデータおよびアクセスメソッドへのアクセスポイント。
  • Snapshot: タプルの可視性を判定するためのスナップショット。

失敗モードと境界条件

  • **権限エラー**: SELECT 権限がない場合、aclcheck_error により処理が中断される。
  • **サポート外の型**: ストレージを持たないリレーション（ビュー以外）に対してはエラーとなる。
  • **メモリ不足**: palloc が失敗した場合は PostgreSQL の標準的なメモリ不足エラーが発生する。

保守担当者向けメモ

  • 本関数は table_tuple_get_latest_tid を利用して物理的なタプル追跡を行うため、VACUUM FULL 等でタプルが移動した後の最新位置を特定する際に重要となる。
  • ビューに対する処理は currtid_for_view に依存しており、複雑なビュー定義では追跡が失敗する可能性がある点に注意すること。
