spgRedoMoveLeafs

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgxlog.c:170

目的

SP-GiST インデックスにおいて、リーフタプルを別のページへ移動させる WAL レコードのリプレイを行う。

シグネチャまたは形状

static void spgRedoMoveLeafs(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象の WAL レコード）。
  • **出力**: なし（副作用としてバッファおよびインデックスページを更新）。

前提条件と不変条件

  • リプレイ時は、ソースページ、宛先ページ、親ページの順序で更新を行うことで整合性を維持する。
  • WAL レコード内のタプルはアライメントされていないため、アクセス時には memcpy を使用してヘッダーをコピーする必要がある。

アルゴリズムまたは振る舞い

1. fillFakeState を呼び出し、リプレイに必要な SpGistState を構築する。 2. **宛先ページ（Destination）の更新**:

  • newPage フラグが立っている場合は SpGistInitBuffer で初期化し、それ以外は XLogReadBufferForRedo で取得する。
  • addOrReplaceTuple を使用して、レコードに含まれるリーフタプルをページに挿入する。

    • 3. **ソースページ（Source）の更新**:

  • spgPageIndexMultiDelete を呼び出し、移動対象のタプルを削除する。
  • インデックス構築中であれば SPGIST_PLACEHOLDER を、通常時は SPGIST_REDIRECT を使用してリダイレクトポインタを設定する。

    • 4. **親ページ（Parent）の更新**:

  • spgUpdateNodeLink を使用し、親ノードのダウンリンクを新しい宛先ページとオフセット番号へ更新する。

    • 5. 各ステップで PageSetLSN と MarkBufferDirty を実行し、変更を確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgxlog.c 内の spg_redo（レコードタイプに基づくディスパッチ）。
  • **呼び出し先**: fillFakeState, XLogRecGetBlockTag, XLogInitBufferForRedo, SpGistInitBuffer, XLogReadBufferForRedo, BufferGetPage, addOrReplaceTuple, PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer, spgPageIndexMultiDelete, PageGetItem, PageGetItemId, spgUpdateNodeLink。

関連データ構造

  • spgxlogMoveLeafs: WAL レコードのデータ構造。
  • SpGistState: インデックスの状態管理構造体。
  • SpGistLeafTupleData: リーフタプルのヘッダー情報。
  • SpGistInnerTuple: 親ノードのタプル構造。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、そのページの更新はスキップされる。
  • 宛先ページが新規作成される場合と既存ページの場合で初期化処理が分岐する。

保守担当者向けメモ

  • リプレイ処理は、スタンバイ側での読み取り時にリンク切れが発生しないよう、宛先ページへの挿入を最初に行う必要がある。
  • 削除処理において、インデックス構築中か否かでプレースホルダーの扱いが異なる点に注意すること。
