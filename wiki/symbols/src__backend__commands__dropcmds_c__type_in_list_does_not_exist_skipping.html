type_in_list_does_not_exist_skipping

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dropcmds.c:205

目的

IF EXISTS 句を伴う削除操作において、指定された型リストの中に存在しない型が含まれているかを判定し、ユーザーに通知するための警告メッセージと対象名を構築します。

シグネチャまたは形状

static bool type_in_list_does_not_exist_skipping(List *typenames, const char **msg, char **name)

入力と出力

  • **入力**:
  • typenames: 検査対象の TypeName リスト。
  • msg: 警告メッセージを格納するためのポインタのポインタ。
  • name: 存在しない型名を格納するためのポインタのポインタ。
  • **出力**:
  • 戻り値: 存在しない型が見つかった場合は true、すべて存在する場合は false。

前提条件と不変条件

  • typenames リスト内の各要素は TypeName ノードである必要があります。
  • msg および name は、呼び出し元で確保された領域を指すポインタである必要があります。

アルゴリズムまたは振る舞い

1. typenames リストを走査し、各 TypeName に対して LookupTypeNameOid を呼び出します。 2. LookupTypeNameOid が無効な OID を返した場合、型が存在しないと判断します。 3. まず schema_does_not_exist_skipping を呼び出し、スキーマの不在が原因であるかを確認します。スキーマ不在であればその結果を返します。 4. スキーマが存在する場合、型自体が存在しないものとして、msg に "type \"%s\" does not exist, skipping" を設定し、name に該当する型名を文字列化して格納し、true を返します。 5. リスト内のすべての型が存在する場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dropcmds.c 内の複数の箇所（主に DROP 文の処理フロー中）から呼び出されています。
  • **呼び出し先**:
  • LookupTypeNameOid
  • schema_does_not_exist_skipping
  • TypeNameToString

関連データ構造

  • List: 削除対象の型名のリスト。
  • TypeName: SQL 文で指定された型を表現するノード。

失敗モードと境界条件

  • typenames が空リストの場合、ループは実行されず false を返します。
  • TypeName が NULL の場合、その要素はスキップされます。

保守担当者向けメモ

  • 本関数は IF EXISTS 処理におけるエラーメッセージの生成を標準化する役割を担っています。型名以外のオブジェクトに対する同様のチェックが必要な場合は、既存の does_not_exist_skipping 関連関数との整合性に注意してください。
