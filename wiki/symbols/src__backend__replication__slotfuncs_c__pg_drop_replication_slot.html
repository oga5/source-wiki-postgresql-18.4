pg_drop_replication_slot

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:217

目的

SQLインターフェースを通じて、指定された名前のレプリケーションスロットを削除します。

シグネチャまたは形状

Datum pg_drop_replication_slot(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_NAME(0) を介してスロット名（Name型）を受け取ります。
  • **出力**: void（PG_RETURN_VOID() を使用）。

前提条件と不変条件

  • 実行には適切な権限が必要です（CheckSlotPermissions() により検証）。
  • スロット操作に必要なシステム要件を満たしている必要があります（CheckSlotRequirements() により検証）。

アルゴリズムまたは振る舞い

1. CheckSlotPermissions() を呼び出し、呼び出し元にスロット削除権限があるか確認します。 2. CheckSlotRequirements() を呼び出し、現在のシステム状態がスロット削除を許可しているか確認します。 3. ReplicationSlotDrop(NameStr(*name), true) を呼び出し、共有メモリ上のスロット状態を解放し、ディスク上の永続化データを削除します。 4. 処理完了後、PG_RETURN_VOID() を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQLコマンド経由（例: SELECT pg_drop_replication_slot(...)）
  • src/bin/pg_basebackup/pg_createsubscriber.c:1455 からの呼び出しが確認されています。
  • **呼び出し先**:
  • CheckSlotPermissions()
  • CheckSlotRequirements()
  • ReplicationSlotDrop()

関連データ構造

  • Name: スロット名を保持する型。
  • ReplicationSlot: ReplicationSlotDrop 内部で管理される共有メモリ上の構造体。

失敗モードと境界条件

  • 権限不足やシステム要件を満たさない場合、上記チェック関数内でエラーが発生し、処理が中断されます。
  • 指定した名前のスロットが存在しない場合、ReplicationSlotDrop 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はSQLレイヤーと内部の ReplicationSlotDrop を繋ぐブリッジです。スロットのライフサイクル管理の核心部分は src/backend/replication/slot.c に実装されています。
