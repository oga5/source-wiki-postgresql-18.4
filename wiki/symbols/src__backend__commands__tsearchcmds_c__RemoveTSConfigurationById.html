RemoveTSConfigurationById

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tsearchcmds.c:1107

目的

指定された OID に対応する全文検索設定（Text Search Configuration）をシステムカタログから削除します。

シグネチャまたは形状

void RemoveTSConfigurationById(Oid cfgId)

入力と出力

  • **入力**: cfgId (Oid) - 削除対象の全文検索設定の OID。
  • **出力**: なし。

前提条件と不変条件

  • 呼び出し元は、対象の OID が存在することを保証するか、存在しない場合にエラーが発生することを許容する必要があります。
  • RowExclusiveLock を使用してカタログを操作するため、適切なトランザクションコンテキストが必要です。

アルゴリズムまたは振る舞い

1. pg_ts_config カタログを RowExclusiveLock で開き、指定された cfgId をシステムキャッシュ（TSCONFIGOID）から検索します。 2. キャッシュ内にタプルが存在しない場合は elog(ERROR) を発生させます。 3. CatalogTupleDelete を使用して pg_ts_config から該当エントリを削除し、キャッシュを解放します。 4. pg_ts_config_map カタログを RowExclusiveLock で開きます。 5. pg_ts_config_map 内の mapcfg 列が cfgId と一致するすべてのエントリをスキャンし、それらを順次削除します。 6. 各カタログのロックを解除して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c:1424（依存関係削除処理から呼び出されることが確認されています）。
  • **呼び出し先**: table_open, SearchSysCache1, CatalogTupleDelete, ReleaseSysCache, table_close, ScanKeyInit, systable_beginscan, systable_getnext, systable_endscan。

関連データ構造

  • pg_ts_config: 全文検索設定のメインカタログ。
  • pg_ts_config_map: 設定と辞書のマッピングを保持するカタログ。

失敗モードと境界条件

  • 指定された cfgId が pg_ts_config に存在しない場合、elog(ERROR) によりトランザクションがアボートされます。

保守担当者向けメモ

  • 本関数はカタログ上のエントリを物理的に削除するのみです。依存関係のチェックやイベントトリガーの通知は、呼び出し元（dependency.c 等）で適切に処理されていることを前提としています。
