pg_tz_cache

定義

  • 種別: struct
  • 定義箇所: src/timezone/pgtz.c:191

目的

タイムゾーンの検索および読み込みコストを削減するため、ハッシュテーブル内でキャッシュされたタイムゾーン情報を保持するデータ構造です。

シグネチャまたは形状

typedef struct
{
	char		tznameupper[TZ_STRLEN_MAX + 1];
	pg_tz		tz;
} pg_tz_cache;

入力と出力

本構造体はハッシュテーブルのエントリとして機能します。

  • **入力**: タイムゾーン名（大文字変換済み）をキーとしてハッシュテーブルに格納されます。
  • **出力**: 読み込み済みの pg_tz 構造体を含むキャッシュエントリとして返されます。

前提条件と不変条件

  • tznameupper は、対応するタイムゾーン名をすべて大文字に変換した文字列を保持します。
  • timezone_cache ハッシュテーブルの各エントリサイズとして本構造体が指定されています。

アルゴリズムまたは振る舞い

  • pg_tzset がタイムゾーンを要求する際、まずハッシュテーブルを検索します。
  • キャッシュヒット時、hash_search を通じて本構造体へのポインタが取得され、内部の pg_tz が利用されます。
  • キャッシュミス時は、新たに読み込まれた情報が本構造体に格納され、ハッシュテーブルに登録されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/timezone/pgtz.c 内の pg_tzset 関数が、ハッシュテーブル操作（hash_search）を通じて本構造体を参照・更新します。
  • **呼び出し先**: 直接的な関数呼び出しはありません。データ構造として pg_tz を内包しています。

関連データ構造

  • pg_tz: タイムゾーンの定義情報を保持する構造体（pgtz.h で定義）。
  • timezone_cache: 本構造体をエントリとして管理するハッシュテーブル。

失敗モードと境界条件

  • TZ_STRLEN_MAX を超える長さのタイムゾーン名は、tznameupper に収まらない可能性があります。
  • ハッシュテーブルの初期化（init_timezone_hashtable）が完了していない状態でアクセスした場合、未定義の動作となります。

保守担当者向けメモ

  • 本構造体は src/timezone/pgtz.c 内のキャッシュ管理専用です。
  • pg_tz 構造体の定義変更に伴い、本構造体のメモリレイアウトも影響を受ける可能性があるため注意してください。
