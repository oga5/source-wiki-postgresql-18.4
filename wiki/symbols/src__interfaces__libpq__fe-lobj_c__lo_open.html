lo_open

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:56

目的

データベース内のラージオブジェクトをオープンし、操作用のファイルディスクリプタを取得します。

シグネチャまたは形状

int lo_open(PGconn *conn, Oid lobjId, int mode);

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • lobjId: オープン対象のラージオブジェクトのOID。
  • mode: アクセスモード（読み取り専用、書き込み専用、またはその組み合わせ）。
  • **出力**:
  • 成功時: ラージオブジェクトを操作するためのファイルディスクリプタ（int）。
  • 失敗時: -1。

前提条件と不変条件

  • lo_initialize(conn) が内部で呼び出され、必要なサーバー側関数のOIDが PGconn 構造体にキャッシュされている必要があります。
  • ラージオブジェクト操作は、現在のトランザクション内で実行されます。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、接続の初期化状態を確認します。 2. lobjId と mode を PQArgBlock 配列に格納します。 3. PQfn (Fastpath) を使用して、サーバー側の lo_open 関数を呼び出します。 4. サーバーからの応答が PGRES_COMMAND_OK であれば、取得したファイルディスクリプタを返します。 5. 処理終了後、PQclear を呼び出して結果セットを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_dump/pg_backup_null.c
  • src/bin/pg_dump/pg_dump.c
  • src/interfaces/libpq/fe-lobj.c (内部的な再帰的利用の可能性あり)
  • **呼び出し先**:
  • lo_initialize: 接続の初期化。
  • PQfn: サーバー側関数の実行。
  • PQresultStatus, PQclear: 結果の確認と解放。

関連データ構造

  • PGconn: 接続状態およびラージオブジェクト関数のキャッシュを保持。
  • PQArgBlock: サーバー側関数へ渡す引数構造体。
  • PGresult: サーバーからの実行結果を保持。

失敗モードと境界条件

  • lo_initialize が失敗した場合、-1 を返します。
  • サーバー側でエラーが発生した場合（PQresultStatus が PGRES_COMMAND_OK 以外）、PQclear を実行した上で -1 を返します。

保守担当者向けメモ

  • lo_open は libpq のラージオブジェクトAPIの主要なエントリポイントです。
  • サーバー側の lo_open 関数のOIDは lo_initialize によって解決されます。
  • 非常に大きなサイズを扱う場合、内部的な型変換（int32 へのキャスト等）に注意してください。
