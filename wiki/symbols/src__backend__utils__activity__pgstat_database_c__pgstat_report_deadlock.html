pgstat_report_deadlock

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_database.c:124

目的

現在のデータベースで発生したデッドロックの回数を統計情報として記録する。

シグネチャまたは形状

void pgstat_report_deadlock(void)

入力と出力

  • 入力: なし（グローバル変数 MyDatabaseId および pgstat_track_counts を参照）
  • 出力: なし（副作用としてデータベース統計エントリの deadlocks カウンタをインクリメント）

前提条件と不変条件

  • pgstat_track_counts が真である場合にのみ統計の更新が行われる。
  • MyDatabaseId が有効なデータベースIDを指している必要がある。

アルゴリズムまたは振る舞い

1. pgstat_track_counts フラグを確認し、偽であれば即座に終了する。 2. pgstat_prep_database_pending(MyDatabaseId) を呼び出し、現在のデータベースに対応する PgStat_StatDBEntry 構造体を取得（または作成）する。 3. 取得した構造体の deadlocks メンバを 1 インクリメントする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/deadlock.c:1131
  • 呼び出し先: pgstat_prep_database_pending

関連データ構造

  • PgStat_StatDBEntry: データベース単位の統計情報を保持する構造体。

失敗モードと境界条件

  • pgstat_track_counts が無効な場合、統計は記録されない。
  • メモリ不足等の理由で pgstat_prep_database_pending が失敗した場合の挙動については、本ソースコード内では明示されていない。

保守担当者向けメモ

  • 本関数はデッドロック検出ロジックから直接呼び出される。統計情報の集計タイミングについては pgstat_database.c の「制御フローとデータフロー」を参照すること。
