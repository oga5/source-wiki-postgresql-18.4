InjectionPointCondition

定義

  • 種別: struct
  • 定義箇所: src/test/modules/injection_points/injection_points.c:57

目的

インジェクションポイントの実行可否を判定するための条件を保持する構造体です。特定のプロセスID（PID）に限定した実行制御などを実現するために使用されます。

シグネチャまたは形状

typedef struct InjectionPointCondition
{
    InjectionPointConditionType type;
    int            pid;
} InjectionPointCondition;

入力と出力

本構造体はデータ保持用であり、直接的な関数入出力は持ちません。ただし、injection_point_allowed 関数に引数として渡され、実行許可の判定に使用されます。

前提条件と不変条件

  • type フィールドは InjectionPointConditionType 列挙型に従う必要があります。
  • pid フィールドは、インジェクションポイントの実行を許可する対象のプロセスIDを格納します。

アルゴリズムまたは振る舞い

インジェクションポイントがトリガーされた際、この構造体を参照して現在のプロセスが条件を満たしているか（例：指定されたPIDと一致するか）が検証されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/injection_points/injection_points.c 内の各関数（injection_error, injection_notice, injection_wait 等）で private_data からキャストして取得されます。
  • インジェクションポイントの登録処理（injection_points_attach 関連）において初期化されます。
  • **呼び出し先**:
  • injection_point_allowed(InjectionPointCondition *condition) 関数に渡され、判定ロジックの入力として利用されます。

関連データ構造

  • InjectionPointConditionType: 条件の種類を定義する列挙型。

失敗モードと境界条件

  • private_data が NULL である場合や、正しく InjectionPointCondition 型にキャストできない場合に不正なメモリ参照が発生する可能性があります。
  • pid が無効なプロセスIDである場合、意図したプロセスでインジェクションポイントが実行されない可能性があります。

保守担当者向けメモ

  • 本構造体はテストモジュール専用です。本番環境のコードパスには影響しません。
  • 構造体の拡張が必要な場合は、InjectionPointConditionType との整合性に注意してください。
