spg_range_quad_inner_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_spgist.c:299

目的

SP-GiSTインデックスの内部ノードにおいて、検索条件（スキャンキー）に基づき、探索対象となる子ノード（象限）を絞り込む。

シグネチャまたは形状

Datum spg_range_quad_inner_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgInnerConsistentIn（検索条件、ノード情報）および spgInnerConsistentOut（探索対象ノードの出力先）を受け取る。
  • **出力**: spgInnerConsistentOut 構造体の nNodes および nodeNumbers を更新し、探索すべき子ノードのインデックスを格納する。

前提条件と不変条件

  • in->allTheSame が真の場合、すべてのノードを探索対象とする。
  • in->hasPrefix が偽（重心なし）の場合、ノード数は2（空範囲用と非空範囲用）である。
  • in->hasPrefix が真（重心あり）の場合、ノード数は4または5（各象限および空範囲用）である。

アルゴリズムまたは振る舞い

1. **全探索判定**: allTheSame が真であれば、全ノードを探索対象として即時終了する。 2. **重心なしノードの処理**:

  • 検索条件（sk_strategy）に基づき、空範囲か否かを判定し、ビットマスク which を用いて不要なノードを枝刈りする。

    • 3. **重心ありノードの処理**:

  • prefixDatum から重心となる範囲を取得し、各象限（1〜5）に対応するビットマスクを初期化する。
  • 各スキャンキーの戦略に従い、範囲の境界条件（minLower, maxLower等）を評価し、探索不要な象限のビットを順次クリアする。
  • RANGESTRAT_CONTAINS_ELEM のような特殊な戦略は、単一要素を範囲に変換して処理する。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックスの走査プロセスから呼び出される（ソースコード上の直接的な呼び出し元情報は提供されていない）。
  • **呼び出し先**: RangeIsEmpty, DatumGetRangeTypeP, range_get_typcache, range_deserialize 等の範囲型操作関数を呼び出す。

関連データ構造

  • spgInnerConsistentIn, spgInnerConsistentOut: SP-GiSTの内部整合性チェック用インターフェース構造体。
  • RangeBound, RangeType: 範囲型の内部表現。

失敗モードと境界条件

  • **未認識の戦略**: default ケースにおいて、未知の StrategyNumber が渡された場合は elog(ERROR) を発生させる。
  • **空範囲の扱い**: RANGESTRAT_CONTAINS 等の戦略において、空範囲が入力された場合の特殊な枝刈りロジックが適用される。

保守担当者向けメモ

  • 本関数は、adjacent_inner_consistent と同様に、検索精度向上のために needPrevious フラグを用いた前段の重心情報の利用を考慮した設計となっている。
  • 枝刈りロジックはビットマスク操作に依存しているため、象限の定義（1〜5）とビット位置の対応関係を変更する際は注意が必要である。
