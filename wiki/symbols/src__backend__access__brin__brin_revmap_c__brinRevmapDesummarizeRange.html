brinRevmapDesummarizeRange

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_revmap.c:322

目的

指定されたヒープブロック範囲に対応するBRINインデックスの要約情報を削除（desummarize）し、リバースレンジマップ（revmap）およびインデックスページの状態を整合させる。

シグネチャまたは形状

bool brinRevmapDesummarizeRange(Relation idxrel, BlockNumber heapBlk)

入力と出力

  • **入力**:
  • idxrel: 対象のBRINインデックスリレーション。
  • heapBlk: 要約解除対象のヒープブロック番号。
  • **出力**:
  • bool: 処理が正常に完了した場合は true。対象ページが既に通常のページではない等の理由で再試行が必要な場合は false。

前提条件と不変条件

  • 呼び出し元は ShareUpdateExclusiveLock を保持している必要がある。
  • revmapページおよびインデックスページは、整合性が保たれている必要がある（不整合時はエラーをスロー）。
  • 処理対象のインデックスページは BRIN_IS_REGULAR_PAGE であること。

アルゴリズムまたは振る舞い

1. brinRevmapInitialize でrevmapの状態を初期化する。 2. revmap_get_blkno で対象ヒープブロックに対応するrevmapページを特定する。存在しない場合は終了する。 3. brinLockRevmapPageForUpdate でrevmapページを排他ロックし、対応するインデックスタプルへのポインタ（TID）を取得する。 4. TIDが無効な場合は終了する。 5. インデックスページを読み込み、BRIN_IS_REGULAR_PAGE であるか確認する。そうでない場合は false を返す。 6. インデックスページ内のオフセットが有効か検証し、無効な場合はエラーを報告する。 7. クリティカルセクションを開始し、以下を実行する：

  • brinSetHeapBlockItemptr でrevmap上のTIDを無効化する。
  • PageIndexTupleDeleteNoCompact でインデックスページからタプルを削除する。
  • WALが必要な場合、XLOG_BRIN_DESUMMARIZE レコードを書き込む。

    • 8. バッファのロックを解除し、brinRevmapTerminate でリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c:1561
  • **呼び出し先**:
  • brinRevmapInitialize, brinRevmapTerminate
  • revmap_get_blkno, brinLockRevmapPageForUpdate
  • brinSetHeapBlockItemptr
  • ReadBuffer, LockBuffer, UnlockReleaseBuffer
  • PageIndexTupleDeleteNoCompact
  • XLogInsert (WAL記録時)

関連データ構造

  • BrinRevmap: revmapのキャッシュ状態を保持。
  • RevmapContents: revmapページ内のTID配列構造。
  • xl_brin_desummarize: WAL記録用の構造体。

失敗モードと境界条件

  • **インデックス破損**: regOffset がページ内の最大オフセットを超えている、または ItemId が使用されていない場合、ERRCODE_INDEX_CORRUPTED でエラーとなる。
  • **再試行**: 対象ページが通常のBRINページ構造を維持していない場合、false を返して呼び出し元に制御を戻す。

保守担当者向けメモ

  • 本関数は、クラッシュや中断によって残存したプレースホルダタプルを削除する役割も担う。
  • WAL記録時には XLogRegisterBuffer を使用してrevmapとインデックスページの両方を保護している。
