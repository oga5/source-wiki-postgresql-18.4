GenerateRecoveryConfig

定義

  • **種別**: 関数
  • **定義箇所**: src/fe_utils/recovery_gen.c:27

目的

PostgreSQLのスタンバイサーバー構築に必要なリカバリ設定（primary_conninfo および primary_slot_name）を含む設定ファイルの内容を生成し、PQExpBuffer として返却します。

シグネチャまたは形状

PQExpBuffer GenerateRecoveryConfig(PGconn *pgconn, const char *replication_slot, char *dbname);

入力と出力

  • **入力**:
  • pgconn: 接続済みの PGconn オブジェクト。
  • replication_slot: レプリケーションスロット名（NULL可）。
  • dbname: データベース名（NULL可）。
  • **出力**:
  • 生成された設定内容を保持する PQExpBuffer。メモリ不足時は pg_fatal によりプロセスが終了します。

前提条件と不変条件

  • pgconn は NULL であってはなりません。
  • PostgreSQL 12未満のサーバーに対しては、standby_mode = 'on' が設定内容に含まれます。
  • PQconninfo で取得した接続オプションのうち、replication、dbname、fallback_application_name は除外されます。

アルゴリズムまたは振る舞い

1. createPQExpBuffer でバッファを初期化します。 2. サーバーバージョンが MINIMUM_VERSION_FOR_RECOVERY_GUC 未満の場合、standby_mode = 'on' を追記します。 3. PQconninfo で取得した接続オプションを走査し、appendConnStrVal を用いて primary_conninfo 文字列を構築します。 4. dbname が指定されている場合、接続文字列の末尾に dbname を追加します。 5. 構築した文字列を escape_quotes でエスケープし、primary_conninfo = '...' の形式でバッファに書き込みます。 6. replication_slot が指定されている場合、primary_slot_name = '...' を追記します。 7. 最後に PQconninfoFree で接続オプションを解放し、バッファを返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_rewind/pg_rewind.c
  • **呼び出し先**:
  • PQconninfo, PQserverVersion, PQconninfoFree (libpq)
  • createPQExpBuffer, initPQExpBuffer, appendPQExpBuffer, appendPQExpBufferStr, appendPQExpBufferChar, termPQExpBuffer, PQExpBufferDataBroken, PQExpBufferBroken (libpq)
  • appendConnStrVal, escape_quotes (内部ユーティリティ)
  • pg_fatal (logging.h)

関連データ構造

  • PQExpBuffer: 設定文字列を保持する可変長バッファ。
  • PGconn: サーバー接続情報。
  • PQconninfoOption: 接続オプションの配列。

失敗モードと境界条件

  • **メモリ不足**: PQExpBuffer の確保や操作に失敗した場合、pg_fatal を呼び出しプロセスを終了します。
  • **バージョン依存**: PostgreSQL 12を境界として、standby_mode の出力有無が切り替わります。
  • **エスケープ処理**: 接続オプション個別のエスケープと、設定ファイル全体としてのクォート処理の二段階が行われます。

保守担当者向けメモ

  • primary_conninfo の生成において、libpq の接続オプションと設定ファイル用のエスケープ処理が混在しているため、修正時はエスケープの二重適用に注意してください。
  • replication_slot は ReplicationSlotValidateName に準拠した文字のみが想定されています。
