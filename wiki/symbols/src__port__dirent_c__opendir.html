opendir

定義

  • 種別: function
  • 定義箇所: src/port/dirent.c:32

目的

Windows環境において、POSIX準拠のディレクトリ走査を開始するためのディレクトリハンドルを初期化します。

シグネチャまたは形状

DIR *opendir(const char *dirname)

入力と出力

  • **入力**: dirname (走査対象のディレクトリパス)
  • **出力**: 成功時はディレクトリ走査状態を保持する DIR 構造体へのポインタ、失敗時は NULL を返します。

前提条件と不変条件

  • 指定された dirname がディレクトリとして存在する必要があります。
  • DIR 構造体および内部のパス文字列用メモリの確保に成功する必要があります。
  • Windows APIの GetFileAttributes がディレクトリ属性を正しく判定できる環境であること。

アルゴリズムまたは振る舞い

1. GetFileAttributes を呼び出し、パスの存在確認とディレクトリ属性の検証を行います。 2. DIR 構造体およびパス格納用のメモリを動的に確保します。 3. パスの末尾にディレクトリ区切り文字（/ または \）がない場合は \ を補完し、さらに検索用ワイルドカード * を付与します。 4. d->handle を INVALID_HANDLE_VALUE に初期化し、d_ino、d_reclen を 0、d_type を DT_UNKNOWN に設定して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/file/fd.c、src/bin/pg_archivecleanup/pg_archivecleanup.c、src/bin/pg_basebackup/pg_receivewal.c、src/bin/pg_checksums/pg_checksums.c、src/bin/pg_combinebackup/pg_combinebackup.c、src/bin/pg_dump/dumputils.c、src/bin/pg_resetwal/pg_resetwal.c など。
  • **呼び出し先**: Windows API (GetFileAttributes)、標準ライブラリ (malloc, strlen, strcpy, strcat, free)。

関連データ構造

  • DIR: ディレクトリ走査の状態を管理する構造体（src/include/port/win32_msvc/dirent.h で定義）。

失敗モードと境界条件

  • パスが存在しない場合: errno を ENOENT に設定し NULL を返す。
  • パスがディレクトリでない場合: errno を ENOTDIR に設定し NULL を返す。
  • メモリ確保に失敗した場合: errno を ENOMEM に設定し NULL を返す。

保守担当者向けメモ

  • 本実装はWindows専用です。
  • d_ino や d_reclen はWindows APIで取得できないため、常に0に固定されています。
  • 呼び出し元で closedir を呼び出し、メモリとハンドルを解放する必要があります。
