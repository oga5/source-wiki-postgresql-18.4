GinBtreeDataLeafInsertData

定義

  • 種別: struct
  • 定義箇所: src/include/access/gin_private.h:194

目的

GINインデックスのリーフページに対するデータ挿入操作において、挿入対象となるアイテムポインタのリストとその状態を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	ItemPointerData *items;
	uint32		nitem;
	uint32		curitem;
} GinBtreeDataLeafInsertData;

入力と出力

  • items: 挿入対象の ItemPointerData 配列へのポインタ。
  • nitem: 挿入するアイテムの総数。
  • curitem: 現在処理中のアイテムインデックス（進捗管理用）。

前提条件と不変条件

  • nitem は挿入されるアイテムの有効な数を示す必要があります。
  • items は少なくとも nitem 分のメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

GINインデックスのデータページへの挿入処理において、挿入すべきアイテム群を一時的に保持し、ページ内の適切な位置へ書き込むためのコンテキストとして機能します。curitem を用いて、バッチ処理や反復的な挿入操作の進捗を追跡します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/gin/gindatapage.c 内のデータページ操作関数（例: gindatapage.c:1913 でインスタンス化され、gindatapage.c:453 で参照される）。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ItemPointerData: GINインデックスが管理するTID（Tuple Identifier）の基本単位。
  • GinBtreeData: GINのB-tree操作を定義する構造体。

失敗モードと境界条件

  • items が NULL の状態でアクセスした場合、セグメンテーション違反が発生します。
  • curitem が nitem を超えてインクリメントされた場合、範囲外アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は gindatapage.c におけるデータページ挿入ロジックと密接に結合しています。変更を加える際は、同ファイル内の挿入アルゴリズムへの影響を確認してください。
