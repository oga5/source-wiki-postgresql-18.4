lexi

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/lexi.c:215

目的

入力ソースコードからトークンを一つずつ切り出し、その種類を識別して呼び出し元に返す字句解析器のメイン関数です。

シグネチャまたは形状

int lexi(struct parser_state *state)

入力と出力

  • **入力**: struct parser_state *state（現在の構文解析状態を保持する構造体へのポインタ）。
  • **出力**: トークンの種類を示す整数コード（indent_codes.h で定義）。

前提条件と不変条件

  • buf_ptr が有効な入力バッファを指していること。
  • s_token および e_token がトークン格納用のバッファとして利用可能であること。
  • specials テーブルはアルファベット順にソートされている必要がある（bsearch 使用のため）。

アルゴリズムまたは振る舞い

1. **空白スキップ**: buf_ptr を進めて空白およびタブを読み飛ばす。 2. **トークン抽出**:

  • 英数字または数値リテラルの場合: 数値リテラル（基数判定含む）または識別子としてバッファにコピーする。
  • 識別子の場合: specials テーブルを二分探索してキーワードを判定。該当しない場合は typenames 配列を検索し、型名かどうかを判定する。

    • 3. **状態更新**:

  • state->last_u_d（単項/二項演算子フラグ）を更新し、後続の整形処理を補助する。
  • state->keyword にキーワードコードを格納する。

    • 4. **戻り値**: 識別したトークンの種類に応じたコードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/indent.c のメインループ内から複数回呼び出される。
  • **呼び出し先**: fill_buffer(), diag2(), bsearch(), strcmp_type(), strspn(), strchr(), isdigit() 等。

関連データ構造

  • struct parser_state: 解析状態を保持。
  • specials[]: 予約語とコードの対応表。
  • typenames: ユーザー定義型名を保持する動的配列。

失敗モードと境界条件

  • **バッファ境界**: buf_ptr が buf_end に達した際、fill_buffer() を呼び出して入力を補充する。
  • **リテラルエラー**: 数値リテラルの解析中に不正な形式が検出された場合、diag2() でエラー報告を行う。
  • **型名判定**: typedef の自動認識が不完全な場合、add_typename による明示的な登録が必要。

保守担当者向けメモ

  • is_func_definition を用いた先読みロジックと連携しているため、修正時はバッファの整合性に注意すること。
  • specials 配列を編集する際は、必ずアルファベット順を維持すること。
