pg_walfile_name_offset

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogfuncs.c:373

目的

指定されたWALログシーケンス番号（LSN）に対応するWALファイル名と、そのファイル内でのオフセット位置を算出するSQLインターフェース関数。

シグネチャまたは形状

Datum pg_walfile_name_offset(PG_FUNCTION_ARGS)
  • 引数: PG_GETARG_LSN(0) によりLSNを受け取る。
  • 戻り値: (file_name: text, file_offset: int4) を含む HeapTuple。

入力と出力

  • 入力: XLogRecPtr 型のLSN値。
  • 出力: 2列のレコード（file_name: WALセグメントファイル名、file_offset: セグメント内のバイトオフセット）。

前提条件と不変条件

  • サーバーがリカバリ中（RecoveryInProgress()）である場合、本関数は実行できずエラーを発生させる。
  • 結果のタプル記述子は、pg_proc エントリと一致する必要がある。

アルゴリズムまたは振る舞い

1. RecoveryInProgress() をチェックし、リカバリ中であればエラーを投げる。 2. 結果格納用のタプル記述子（file_name と file_offset）を作成し、BlessTupleDesc で初期化する。 3. XLByteToSeg を用いてLSNからセグメント番号を算出する。 4. XLogFileName を用いて、現在のタイムラインIDとセグメント番号からWALファイル名を生成する。 5. XLogSegmentOffset を用いて、セグメント内のオフセットを算出する。 6. heap_form_tuple を使用して結果タプルを構築し、PG_RETURN_DATUM で返却する。

呼び出し元と呼び出し先

  • 呼び出し元: SQLインターフェース経由で呼び出される（直接的なC言語レベルの呼び出し元はソース内に明示されていない）。
  • 呼び出し先: RecoveryInProgress, CreateTemplateTupleDesc, TupleDescInitEntry, BlessTupleDesc, XLByteToSeg, XLogFileName, GetWALInsertionTimeLine, XLogSegmentOffset, heap_form_tuple 等の内部API。

関連データ構造

  • XLogRecPtr: LSNを保持する型。
  • HeapTuple: SQLへ返却する結果行の構造体。
  • TupleDesc: タプルのスキーマ定義。

失敗モードと境界条件

  • リカバリ実行中に呼び出された場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーが発生する。
  • wal_segment_size が不正な値である場合、計算結果に影響を及ぼす可能性がある（ただし、本関数内ではグローバル変数として参照）。

保守担当者向けメモ

  • 本関数はSQLインターフェースであるため、戻り値のタプル構造を変更した場合は、対応する pg_proc のカタログ定義も更新する必要がある。
  • RecoveryInProgress() チェックは、本ファイル内の他のWAL関連関数と共通の設計パターンである。
