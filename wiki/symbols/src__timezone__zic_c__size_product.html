size_product

定義

  • 種別: function
  • 定義箇所: src/timezone/zic.c:417

目的

メモリ割り当て時に、要素数と要素サイズの積を計算し、算術オーバーフローが発生しないことを保証する。

シグネチャまたは形状

static size_t size_product(size_t nitems, size_t itemsize)

入力と出力

  • **入力**:
  • nitems: 確保する要素数。
  • itemsize: 各要素のサイズ（バイト単位）。
  • **出力**:
  • 計算結果である nitems * itemsize を size_t 型で返す。

前提条件と不変条件

  • SIZE_MAX を超える積が発生する場合、関数は呼び出し元へ戻らずに終了する。

アルゴリズムまたは振る舞い

1. SIZE_MAX / itemsize < nitems を評価し、積が size_t の範囲を超えるか判定する。 2. オーバーフローが検出された場合、memory_exhausted(_("size overflow")) を呼び出し、プログラムを終了させる。 3. オーバーフローが発生しない場合、nitems * itemsize を計算して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/timezone/zic.c:464 (erealloc の引数として使用)
  • src/timezone/zic.c:2100 (emalloc の引数として使用)
  • src/timezone/zic.c:3725 (emalloc の引数として使用)
  • **呼び出し先**:
  • memory_exhausted: オーバーフロー検出時に呼び出される。

関連データ構造

  • なし。

失敗モードと境界条件

  • itemsize が 0 の場合、除算エラーを避けるための明示的なチェックはソース内に存在しない（SIZE_MAX / 0 は未定義動作となる可能性がある）。
  • nitems または itemsize が SIZE_MAX に近い値の場合、オーバーフロー判定が機能する。

保守担当者向けメモ

  • 本関数は zic ユーティリティ内での安全なメモリ確保を目的としている。
  • memory_exhausted はプログラムを終了させるため、本関数を呼び出す際はメモリ確保の失敗が致命的であることを前提とする必要がある。
