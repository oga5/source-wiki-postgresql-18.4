AlterForeignServerOwner_internal

定義

  • 種別: function
  • 定義箇所: src/backend/commands/foreigncmds.c:348

目的

外部サーバー（Foreign Server）の所有者を変更し、システムカタログ pg_foreign_server の更新および依存関係の再設定を行う。

シグネチャまたは形状

static void AlterForeignServerOwner_internal(Relation rel, HeapTuple tup, Oid newOwnerId)

入力と出力

  • rel: pg_foreign_server システムカタログのリレーション（オープン済み）。
  • tup: 変更対象の外部サーバーのタプル。
  • newOwnerId: 新しい所有者の OID。
  • 出力: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • rel は RowExclusiveLock でロックされている必要がある。
  • スーパーユーザーでない場合、呼び出し元は現在のサーバー所有者である必要がある。
  • newOwnerId は有効なロールである必要がある。
  • 新しい所有者は、当該サーバーが使用する外部データラッパー（FDW）に対して USAGE 権限を持っている必要がある。

アルゴリズムまたは振る舞い

1. **権限チェック**: スーパーユーザーでない場合、現在の所有者であるかを確認し、newOwnerId への変更権限および FDW への USAGE 権限を検証する。 2. **タプル更新準備**: pg_foreign_server の srvowner フィールドを newOwnerId に更新する。 3. **ACL 更新**: サーバーに ACL が設定されている場合、aclnewowner を用いて所有者変更に伴う ACL の再構築を行う。 4. **カタログ更新**: heap_modify_tuple と CatalogTupleUpdate を使用してシステムカタログを永続化する。 5. **依存関係更新**: changeDependencyOnOwner を呼び出し、所有者変更を依存関係システムに反映させる。 6. **フック実行**: InvokeObjectPostAlterHook を呼び出し、変更を外部サブシステムに通知する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/foreigncmds.c 内の AlterForeignServerOwner 関連関数（AlterForeignServerOwner および AlterForeignServerOwner_oid）。
  • 呼び出し先: superuser, object_ownercheck, check_can_set_role, object_aclcheck, GetForeignDataWrapper, aclcheck_error, heap_getattr, aclnewowner, heap_modify_tuple, CatalogTupleUpdate, changeDependencyOnOwner, InvokeObjectPostAlterHook。

関連データ構造

  • Form_pg_foreign_server: pg_foreign_server カタログの構造体。
  • Relation: システムカタログのリレーション情報。
  • HeapTuple: 変更対象のタプルデータ。
  • Acl: アクセス制御リスト。

失敗モードと境界条件

  • 所有者でない場合や FDW への USAGE 権限がない場合は aclcheck_error によりエラーが発生する。
  • newOwnerId が無効な場合、check_can_set_role によりエラーが発生する。
  • ACL が NULL の場合、ACL の更新はスキップされる。

保守担当者向けメモ

  • 本関数は内部的な変更処理のみを担当する。外部からの呼び出しは AlterForeignServerOwner 等の公開インターフェースを経由すること。
  • pg_foreign_server のスキーマ変更（列追加など）が発生した場合は、repl_val 等の配列サイズや Anum 定数を確認すること。
