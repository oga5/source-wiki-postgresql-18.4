sepgsql_avc_compute

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:199

目的

SELinux のアクセス制御決定（AVD）をカーネルから取得し、ユーザー空間アクセスベクタキャッシュ（AVC）に新規エントリとして登録する。

シグネチャまたは形状

static avc_cache *sepgsql_avc_compute(const char *scontext, const char *tcontext, uint16 tclass)

入力と出力

  • **入力**:
  • scontext: 主体（Subject）のセキュリティコンテキスト。
  • tcontext: 対象（Target）のセキュリティコンテキスト。
  • tclass: オブジェクトクラス。
  • **出力**:
  • 構築された avc_cache 構造体へのポインタ。

前提条件と不変条件

  • avc_mem_cxt メモリコンテキストが初期化されていること。
  • tcontext が無効な場合、sepgsql_avc_unlabeled() を介して代替コンテキストが使用される。
  • tclass が SEPG_CLASS_DB_PROCEDURE の場合、実行時に切り替えるべきセキュリティラベル（ncontext）が計算・キャッシュされる。

アルゴリズムまたは振る舞い

1. **ハッシュ計算**: sepgsql_avc_hash を用いてスロットインデックスを決定する。 2. **検証**: security_check_context_raw で tcontext を検証し、無効な場合は sepgsql_avc_unlabeled を使用する。 3. **AVD 取得**: sepgsql_compute_avd を呼び出し、権限判定結果を取得する。 4. **プロシージャラベル計算**: tclass が DB_PROCEDURE の場合、sepgsql_compute_create で実行時のセキュリティラベルを計算する。 5. **キャッシュ生成**: avc_mem_cxt 上に avc_cache オブジェクトを割り当て、各フィールドをコピーする。 6. **メモリ管理**: キャッシュ数が avc_threshold を超えた場合、sepgsql_avc_reclaim を呼び出してエントリを解放する。 7. **登録**: 新規エントリを avc_slots の先頭に挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**: sepgsql_avc_lookup（contrib/sepgsql/uavc.c:321）。
  • **呼び出し先**: sepgsql_avc_hash, security_check_context_raw, sepgsql_avc_unlabeled, sepgsql_compute_avd, sepgsql_compute_create, palloc0, pstrdup, pfree, MemoryContextSwitchTo, sepgsql_avc_reclaim, lcons。

関連データ構造

  • avc_cache: キャッシュエントリを保持する構造体。
  • av_decision: SELinux の権限判定結果を格納する構造体。
  • avc_slots: キャッシュエントリを格納するハッシュテーブルのスロット配列。

失敗モードと境界条件

  • tcontext が無効な場合、ucontext を使用して処理を継続する。
  • ncontext が scontext と同一の場合、キャッシュには保存されない。
  • キャッシュエントリ数が閾値を超えると、sepgsql_avc_reclaim による LRU ベースの解放が発生する。

保守担当者向けメモ

  • 本関数は avc_mem_cxt コンテキスト内でメモリを割り当てるため、呼び出し元でのメモリコンテキスト管理に注意すること。
  • DB_PROCEDURE クラスのキャッシュには ncontext が含まれるため、信頼されたプロシージャの判定ロジックと密接に関連している。
