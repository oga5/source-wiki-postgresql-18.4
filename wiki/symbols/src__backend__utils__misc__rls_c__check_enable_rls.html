check_enable_rls

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/rls.c:51

目的

指定されたテーブルに対して行レベルセキュリティ（RLS）を適用すべきか、バイパスすべきか、あるいは環境依存の判定が必要かを決定する。

シグネチャまたは形状

int check_enable_rls(Oid relid, Oid checkAsUser, bool noError)

入力と出力

  • **入力**:
  • relid: 対象テーブルのOID。
  • checkAsUser: 権限チェックに使用するユーザーID（InvalidOidの場合は現在のユーザーを使用）。
  • noError: true の場合、RLSが無効な状況でエラーを発生させない。
  • **出力**:
  • RLS_NONE: RLSを適用しない。
  • RLS_NONE_ENV: 現時点では適用しないが、環境（ユーザー等）の変化により適用対象となる可能性がある。
  • RLS_ENABLED: RLSを適用する。

前提条件と不変条件

  • 組み込みリレーション（FirstNormalObjectId 未満）に対しては常に RLS_NONE を返す。
  • pg_class システムカタログの relrowsecurity フラグが false の場合、RLSは適用されない。

アルゴリズムまたは振る舞い

1. **メタデータ取得**: RELOID システムキャッシュからテーブルの relrowsecurity および relforcerowsecurity を取得する。 2. **バイパス判定**:

  • BYPASSRLS 権限を持つユーザー（スーパーユーザー含む）は RLS_NONE_ENV を返す。
  • テーブル所有者は、FORCE ROW SECURITY が設定されておらず、かつ InNoForceRLSOperation() が真でない限り RLS_NONE_ENV を返す。

    • 3. **エラーチェック**: RLSが有効で、かつGUC変数 row_security が無効（かつ noError が false）の場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーをスローする。 4. **最終判定**: 上記のいずれにも該当しない場合、RLS_ENABLED を返す。

  • **呼び出し元**:
  • src/backend/access/index/genam.c
  • src/backend/commands/copy.c

  • **呼び出し先**:

関連データ構造

  • Form_pg_class: テーブルのRLS設定フラグ（relrowsecurity, relforcerowsecurity）を保持。
  • RLS_NONE, RLS_NONE_ENV, RLS_ENABLED: 戻り値として使用される列挙型定数。

失敗モードと境界条件

  • テーブルOIDがシステムカタログに存在しない場合、RLS_NONE を返す。
  • row_security GUCが無効な状態でRLS対象テーブルにアクセスし、noError が false の場合、エラーが発生する。

保守担当者向けメモ

  • RLS_NONE_ENV はプランキャッシュの無効化トリガーとして機能するため、プランナがキャッシュ戦略を決定する際に重要となる。
  • InNoForceRLSOperation() は参照整合性チェック等の特殊な文脈でRLSを抑制するための仕組みであり、所有者権限チェックと組み合わせて使用される。
