pgfnames

定義

  • 種別: function
  • 定義箇所: src/common/pgfnames.c:36

目的

指定されたディレクトリ内のファイル名一覧を取得し、動的に確保された文字列配列として返します。

シグネチャまたは形状

char **pgfnames(const char *path)

入力と出力

  • **入力**: path (走査対象のディレクトリパス)
  • **出力**: ファイル名のリスト（char ** 型）。リストの末尾は NULL で終端されます。ディレクトリのオープンに失敗した場合は NULL を返します。

前提条件と不変条件

  • 戻り値の配列および各文字列は palloc / pstrdup で確保されるため、呼び出し元は必ず pgfnames_cleanup を使用してメモリを解放する必要があります。
  • ディレクトリ内の . および .. は結果に含まれません。

アルゴリズムまたは振る舞い

1. opendir で指定パスを開きます。失敗時は警告をログ出力し NULL を返します。 2. 初期サイズ 200 のポインタ配列を確保します。 3. readdir をループし、各エントリ名（. と .. を除く）を pstrdup でコピーして配列に格納します。 4. 配列が不足した場合は repalloc でサイズを倍増させます。 5. ループ終了後、配列の末尾に NULL を代入し、closedir を呼び出します。 6. 読み取り中にエラーが発生した場合は警告をログ出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/findtimezone.c:664 での利用が確認されています。
  • **呼び出し先**: opendir, readdir, closedir (システムコール)、palloc, repalloc, pstrdup, pfree (メモリ管理)、pg_log_warning (ログ出力)。

関連データ構造

  • DIR: ディレクトリ操作用構造体。
  • struct dirent: ディレクトリ内の各エントリ情報を保持する構造体。

失敗モードと境界条件

  • ディレクトリが開けない場合: 警告をログ出力し NULL を返します。
  • ディレクトリ読み取り中にエラーが発生した場合: 警告をログ出力し、その時点までに取得できたリストを返します。
  • ディレクトリが空の場合: NULL で終端された空の配列（ポインタのみ）を返します。

保守担当者向けメモ

  • FRONTEND マクロの定義状況により、ログ出力関数（elog または pg_log_warning）の挙動が切り替わります。
  • メモリリークを防ぐため、本関数で確保されたメモリは必ず pgfnames_cleanup で解放してください。
