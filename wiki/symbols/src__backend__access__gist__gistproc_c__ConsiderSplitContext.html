ConsiderSplitContext

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gist/gistproc.c:283

目的

GiSTインデックスのノード分割処理において、複数の分割候補の中から最適な分割軸と境界を評価・保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    int         entriesCount;   /* 分割対象の総エントリ数 */
    BOX         boundingBox;    /* 全エントリを包含する最小境界矩形 */
    bool        first;          /* 分割が未選択であればtrue */
    float8      leftUpper;      /* 左側グループの上限 */
    float8      rightLower;     /* 右側グループの下限 */
    float4      ratio;          /* 分割比率 */
    float4      overlap;        /* 分割後のオーバーラップ量 */
    int         dim;            /* 分割軸の次元 */
    float8      range;          /* 選択された軸におけるMBRの幅 */
} ConsiderSplitContext;

入力と出力

本構造体は関数間で受け渡される状態保持用オブジェクトです。

  • **入力**: g_box_consider_split 等の評価関数に対し、現在の分割候補の品質を蓄積するために渡されます。
  • **出力**: 評価の結果、最適な分割パラメータ（dim, leftUpper, rightLower 等）が更新されます。

前提条件と不変条件

  • entriesCount は分割対象となる全エントリ数と一致している必要があります。
  • first フラグは、初期化直後には true であり、最初の有効な分割候補が評価された時点で false に更新されます。

アルゴリズムまたは振る舞い

GiSTのDouble Sortingアルゴリズムにおいて、各軸（X軸、Y軸）の分割候補を走査する際、この構造体を使用して以下の評価を行います。 1. 各分割候補のオーバーラップ量と面積増加を計算。 2. 既存の候補よりも品質が高い（オーバーラップが小さい、または面積増加が少ない）場合、構造体内の各フィールドを新しい値で更新。 3. 最終的に、最もペナルティの低い分割軸と境界が本構造体に保持されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c 内の分割ロジック。
  • src/backend/utils/adt/rangetypes_gist.c 内の範囲型分割ロジック。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。評価関数である g_box_consider_split や range_gist_consider_split が本構造体を操作します。

関連データ構造

  • BOX: 幾何データ型の境界矩形。
  • SplitInterval: 分割軸上の間隔を管理する構造体。

失敗モードと境界条件

  • エントリ数が0または1の場合、分割は行われません。
  • すべての分割候補が同等の品質である場合、最初に評価された候補が保持される可能性があります。

保守担当者向けメモ

  • 本構造体は gistproc.c と rangetypes_gist.c の両方で利用されています。構造体のフィールドを変更する場合は、両方の実装における評価ロジックへの影響を確認してください。
  • first フラグの管理は、分割候補の初期化と更新の境界条件を制御する重要な役割を担っています。
