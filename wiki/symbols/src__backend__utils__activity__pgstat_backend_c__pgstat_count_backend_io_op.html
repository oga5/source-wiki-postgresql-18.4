pgstat_count_backend_io_op

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_backend.c:73

目的

バックエンドプロセスにおける I/O 操作の回数およびバイト数を、プロセスローカルな統計バッファ（PendingBackendStats）に加算する。

シグネチャまたは形状

void pgstat_count_backend_io_op(IOObject io_object, IOContext io_context, IOOp io_op, uint32 cnt, uint64 bytes)

入力と出力

  • **入力**:
  • io_object: 対象となる I/O オブジェクト（例: データファイル、WAL）。
  • io_context: I/O が発生したコンテキスト。
  • io_op: 実行された I/O 操作の種類。
  • cnt: 加算する操作回数。
  • bytes: 加算するデータ量（バイト単位）。
  • **出力**: なし（副作用として静的バッファを更新）。

前提条件と不変条件

  • pgstat_tracks_backend_bktype(MyBackendType) が真である場合にのみ統計が更新される。
  • pgstat_tracks_io_op のアサーションにより、指定された操作が現在のバックエンド種別で追跡対象であることが保証される。

アルゴリズムまたは振る舞い

1. 現在のバックエンド種別が統計収集対象か判定し、対象外であれば即座にリターンする。 2. pgstat_tracks_io_op を呼び出し、引数の組み合わせが妥当か検証する。 3. PendingBackendStats.pending_io 内の該当する counts および bytes 配列に、引数の値を加算する。 4. backend_has_iostats フラグを true に設定し、統計更新があったことを記録する。 5. pgstat_report_fixed フラグを true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_io.c（直接の呼び出しを確認）。
  • **呼び出し先**: pgstat_tracks_backend_bktype および pgstat_tracks_io_op（いずれもマクロまたはインライン関数と推測されるが、本ソースからは詳細な定義元は特定不可）。

関連データ構造

  • PendingBackendStats: プロセスローカルな統計保持用構造体。
  • IOObject, IOContext, IOOp: I/O 統計を分類するための列挙型。

失敗モードと境界条件

  • 統計対象外のバックエンド種別から呼び出された場合は何もしない。
  • メモリ割り当てを行わない設計のため、クリティカルセクション内からの呼び出しにも対応可能。

保守担当者向けメモ

  • 本関数はメモリ確保を避けるため、静的領域のバッファを直接更新する。
  • 統計の共有メモリへの反映は、後続の pgstat_flush_backend によって行われる。
