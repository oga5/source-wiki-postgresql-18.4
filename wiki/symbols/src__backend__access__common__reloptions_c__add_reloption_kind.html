add_reloption_kind

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/reloptions.c:693

目的

拡張モジュールや新しいアクセスメソッドが、独自のストレージパラメータ種別（relopt_kind）を動的に割り当てるためのインターフェースを提供します。

シグネチャまたは形状

relopt_kind add_reloption_kind(void);

入力と出力

  • **入力**: なし
  • **出力**: 新しく割り当てられた relopt_kind 値（ビットフラグ）を返します。

前提条件と不変条件

  • 内部変数 last_assigned_kind が保持する値が RELOPT_KIND_MAX を超えていないこと。
  • 戻り値はビット演算によるフラグとして機能するため、移植性を考慮して最上位ビットは使用されません。

アルゴリズムまたは振る舞い

1. 現在の last_assigned_kind が RELOPT_KIND_MAX に達しているか確認し、超えている場合はエラー（ERRCODE_PROGRAM_LIMIT_EXCEEDED）を発生させます。 2. last_assigned_kind を左に 1 ビットシフトします。 3. 更新された値を relopt_kind 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blutils.c
  • src/test/modules/dummy_index_am/dummy_index_am.c
  • **呼び出し先**:
  • ereport（エラー発生時）

関連データ構造

  • relopt_kind: ストレージパラメータの種別を識別するための列挙型またはビットマスク。

失敗モードと境界条件

  • **失敗モード**: ユーザー定義のパラメータ種別数が上限（RELOPT_KIND_MAX）に達した場合、プログラム制限超過エラーが発生します。

保守担当者向けメモ

  • 本関数はビットシフトによって新しい種別を生成するため、呼び出し順序によって割り当てられる値が決定されます。
  • 拡張モジュールがこの関数を呼び出すことで、コアシステムと競合しない独自のオプション種別を安全に確保できます。
