ascii_safe_strlcpy

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ascii.c:173

目的

文字列を安全にコピーし、非ASCII文字を ? に置換することで、エラーを発生させずに文字列をASCII範囲内に収める。主にポストマスタープロセス内での使用を想定している。

シグネチャまたは形状

void ascii_safe_strlcpy(char *dest, const char *src, size_t destsiz)

入力と出力

  • dest: コピー先のバッファ（出力）。
  • src: コピー元の文字列（入力）。
  • destsiz: dest バッファのサイズ。
  • 戻り値: なし。

前提条件と不変条件

  • destsiz が 0 の場合、関数は何もせず直ちに終了する。
  • コピー後の文字列は必ずヌル終端される。
  • ereport(ERROR) を呼び出さない設計である。

アルゴリズムまたは振る舞い

1. destsiz が 0 の場合は終了する。 2. destsiz - 1 バイト分を上限として、src から文字を読み込む。 3. 読み込んだ文字 ch がヌル文字 (\0) ならばループを終了する。 4. 以下の条件で dest に書き込む文字を決定する:

  • 32から127の範囲のASCII文字、または改行・復帰・タブ (\n, \r, \t) はそのままコピーする。
  • それ以外の文字は ? に置換する。

    • 5. ループ終了後、dest の末尾にヌル文字を書き込む。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/bgworker.c
  • src/backend/utils/activity/backend_status.c
  • 呼び出し先:
  • 外部関数への依存なし（標準的なCのポインタ操作のみ）。

関連データ構造

  • なし。

失敗モードと境界条件

  • destsiz が 1 の場合: ループ条件 while (--destsiz > 0) が直ちに偽となり、ヌル終端のみが行われる。
  • src が destsiz より長い場合: バッファサイズに収まるように切り詰められ、適切にヌル終端される。

保守担当者向けメモ

  • 本関数は ereport(ERROR) を投げないため、エラーハンドリングが制限される環境（ポストマスター等）での利用に適している。
  • 変換テーブルを使用せず、条件分岐によって文字を判定している。
