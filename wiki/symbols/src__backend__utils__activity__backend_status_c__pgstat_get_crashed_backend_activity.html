pgstat_get_crashed_backend_activity

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:1073

目的

バックエンドプロセスがクラッシュした際、ポストマスターが共有メモリ上の情報を直接読み取り、当該プロセスが実行していたクエリ文字列を抽出するために使用されます。

シグネチャまたは形状

const char *pgstat_get_crashed_backend_activity(int pid, char *buffer, int buflen)

入力と出力

  • **入力**:
  • pid: 情報を取得したいバックエンドプロセスのプロセスID。
  • buffer: 抽出したクエリ文字列を格納するための呼び出し元が提供するバッファ。
  • buflen: buffer のサイズ。
  • **出力**:
  • 成功時は buffer へのポインタを返す。
  • 失敗時（PIDが見つからない、共有メモリ未初期化、クエリ文字列が空など）は NULL を返す。

前提条件と不変条件

  • 共有メモリ（BackendStatusArray および BackendActivityBuffer）が初期化済みであること。
  • 読み取り対象のクエリ文字列は、BackendActivityBuffer の範囲内に存在しなければならない。

アルゴリズムまたは振る舞い

1. 共有メモリ上の BackendStatusArray を走査し、指定された pid に一致するエントリを検索する。 2. 一致するエントリが見つかった場合、その st_activity_raw ポインタが BackendActivityBuffer の有効範囲内にあるか検証する。 3. クエリ文字列が空（\0）であれば NULL を返す。 4. ascii_safe_strlcpy を使用し、ASCIIセーフな文字のみを buffer にコピーする。この際、buflen と pgstat_track_activity_query_size の小さい方を上限とする。 5. コピーした文字列を格納した buffer を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c:2809 で呼び出されていることが確認されている。
  • **呼び出し先**: ascii_safe_strlcpy を呼び出す。

関連データ構造

  • PgBackendStatus: 各バックエンドの状態を保持する構造体。
  • BackendStatusArray: 全バックエンドのステータスを格納する共有メモリ上の配列。
  • BackendActivityBuffer: クエリ文字列が格納される共有メモリ領域。

失敗モードと境界条件

  • 共有メモリが未初期化の場合、NULL を返す。
  • 指定された pid が BackendStatusArray 内に見つからない場合、NULL を返す。
  • st_activity_raw が BackendActivityBuffer の範囲外を指している場合、安全のため NULL を返す。
  • クエリ文字列が空の場合、NULL を返す。

保守担当者向けメモ

  • 本関数は通常の統計情報読み取りプロトコル（st_changecount による楽観的並行制御）を無視して共有メモリを直接読み取るため、クラッシュ時の緊急的な情報取得に限定して使用すること。
  • マルチバイト文字の扱いは、ASCIIセーフな文字のみをコピーすることで回避しているため、非ASCII文字は失われる仕様である。
