array_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/rmgrdesc_utils.c:23

目的

WAL（Write-Ahead Log）レコードに含まれる配列データを、一貫したフォーマット（[elem1, elem2, ...]）で StringInfo バッファへ出力するための汎用ユーティリティ関数です。

シグネチャまたは形状

void array_desc(StringInfo buf, void *array, size_t elem_size, int count,
                void (*elem_desc) (StringInfo buf, void *elem, void *data),
                void *data)

入力と出力

  • buf: 出力先の StringInfo バッファ。
  • array: 処理対象の配列の先頭アドレス。
  • elem_size: 配列の各要素のサイズ（バイト単位）。
  • count: 配列の要素数。
  • elem_desc: 各要素を文字列化してバッファに書き込むためのコールバック関数。
  • data: コールバック関数に渡す任意のコンテキストデータ。
  • 出力: なし（buf の内容を更新）。

前提条件と不変条件

  • count が 0 の場合、" []" がバッファに追加されます。
  • count が 1 以上の場合、" [" で始まり、各要素が elem_desc で処理され、要素間は ", " で区切られ、最後に "]" が追加されます。

アルゴリズムまたは振る舞い

1. count == 0 の場合、" []" を出力して終了します。 2. count > 0 の場合、" [" を出力します。 3. 0 から count - 1 までループし、以下の処理を行います。

  • (char *) array + elem_size * i により、現在の要素のアドレスを計算します。
  • elem_desc を呼び出し、要素をバッファに書き込みます。
  • 最後の要素でない場合、", " をバッファに追加します。

    • 4. ループ終了後、"]" をバッファに追加します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/rmgrdesc/heapdesc.c (複数箇所)
  • src/backend/access/rmgrdesc/nbtdesc.c
  • 呼び出し先:
  • elem_desc 引数として渡された関数ポインタ（offset_elem_desc 等）。
  • appendStringInfoString / appendStringInfoChar (PostgreSQL 内部 API)。

関連データ構造

  • StringInfo: WAL 記述用バッファ。
  • OffsetNumber, Oid: 本関数で頻繁に処理されるデータ型。

失敗モードと境界条件

  • count が負の値の場合の挙動は定義されていません。
  • elem_size が 0 の場合、同じメモリ位置に対して elem_desc が count 回呼び出されます。

保守担当者向けメモ

  • 新しい WAL レコードタイプで配列を出力する際は、手動ループを避け本関数を使用することで出力形式の整合性を維持してください。
  • elem_desc に渡す data 引数は、特定のテーブル定義など追加のコンテキストが必要な場合に活用可能です。単純な型変換であれば NULL を指定してください。
