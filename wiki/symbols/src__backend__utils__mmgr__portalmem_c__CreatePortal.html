CreatePortal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:174

目的

新規ポータルオブジェクトを作成し、メモリコンテキストの割り当て、リソース所有者の生成、およびポータルハッシュテーブルへの登録を行う。

シグネチャまたは形状

Portal CreatePortal(const char *name, bool allowDup, bool dupSilent)

入力と出力

  • **入力**:
  • name: ポータル名（文字列）。
  • allowDup: 同名のポータルが存在する場合に上書きを許可するかどうか。
  • dupSilent: 同名のポータルが存在する場合に警告を出力しないかどうか。
  • **出力**:
  • 初期化済みの Portal 構造体へのポインタ。

前提条件と不変条件

  • name は有効なポインタであること。
  • TopPortalContext が初期化されていること。
  • CurTransactionResourceOwner が現在のトランザクションに対して有効であること。

アルゴリズムまたは振る舞い

1. GetPortalByName を呼び出し、同名のポータルが存在するか確認する。 2. 同名ポータルが存在する場合、allowDup が偽ならエラーを発生させ、dupSilent が偽なら警告を出した上で PortalDrop を実行して既存ポータルを破棄する。 3. TopPortalContext からメモリを確保し、Portal 構造体を初期化する。 4. AllocSetContextCreate を用いてポータル専用のメモリコンテキストを作成する。 5. ResourceOwnerCreate を呼び出し、現在のトランザクションに関連付けられたリソース所有者を作成する。 6. ポータルのステータスを PORTAL_NEW に設定し、トランザクションID、ネストレベル、タイムスタンプ等のメタデータを初期化する。 7. PortalHashTableInsert を呼び出し、ポータルをハッシュテーブルに登録する。 8. MemoryContextSetIdentifier を使用して、デバッグ用にメモリコンテキストを識別する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/portalcmds.c
  • src/backend/executor/spi.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • GetPortalByName, PortalDrop, MemoryContextAllocZero, AllocSetContextCreate, ResourceOwnerCreate, GetCurrentSubTransactionId, GetCurrentTransactionNestLevel, GetCurrentStatementStartTimestamp, PortalHashTableInsert, MemoryContextSetIdentifier

関連データ構造

  • Portal: ポータルの状態を保持する主要構造体。
  • PortalHashTable: 全ポータルを管理するハッシュテーブル。
  • TopPortalContext: ポータル構造体割り当て用のメモリコンテキスト。

失敗モードと境界条件

  • 同名のポータルが存在し、かつ allowDup が false の場合、ERRCODE_DUPLICATE_CURSOR エラーが発生する。
  • メモリ不足時には MemoryContextAllocZero 等がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数はポータルのライフサイクル管理の起点となる。
  • PortalDrop を呼び出す際は、ポータルがピン留めされていないことを確認する必要がある。
  • WITH HOLD カーソルの永続化処理は、本関数ではなくトランザクション終了時の PreCommit_Portals で行われる。
