pg_safe_snapshot_blocking_pids

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:572

目的

指定されたプロセスID（PID）が GetSafeSnapshot で待機している場合に、その待機原因となっているプロセスIDのリストを配列として取得します。

シグネチャまたは形状

Datum pg_safe_snapshot_blocking_pids(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_INT32(0): 調査対象のプロセスID（blocked_pid）。
  • **出力**:
  • Datum: ブロックしているプロセスIDを格納した int4 型の配列。

前提条件と不変条件

  • MaxBackends 分のメモリ領域を確保するため、システムが許容する最大バックエンド数に依存します。
  • 戻り値は construct_array_builtin を介して構築されるため、PostgreSQLの配列型として返却されます。

アルゴリズムまたは振る舞い

1. MaxBackends 分のメモリを palloc で確保し、ブロッキングPID格納用のバッファを準備します。 2. GetSafeSnapshotBlockingPids を呼び出し、指定されたPIDをブロックしているプロセスIDをバッファに収集します。 3. 収集されたPIDの数が1以上の場合は、それらを Datum 型の配列に変換します。 4. construct_array_builtin を使用して、INT4OID 型のPostgreSQL配列を生成し、呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • palloc: メモリ確保。
  • GetSafeSnapshotBlockingPids: ブロッキングPIDの取得。
  • Int32GetDatum: int から Datum への変換。
  • construct_array_builtin: 配列の構築。
  • **呼び出し元**:
  • ソースコード内の直接的な呼び出し元情報は提供されていません。

関連データ構造

  • Datum: PostgreSQLの汎用データ表現。
  • INT4OID: int4 型のオブジェクト識別子。

失敗モードと境界条件

  • ブロッキングPIDが存在しない場合、num_blockers は 0 となり、空の配列が返却されます。
  • palloc によるメモリ確保が失敗した場合は、PostgreSQLの標準的なエラーハンドリング（メモリ不足エラー）が発生します。

保守担当者向けメモ

  • 本関数は GetSafeSnapshotBlockingPids の結果をSQL層へ公開するラッパーです。
  • MaxBackends を使用した固定長バッファ確保を行っているため、将来的にバックエンド数の上限が動的に変化するような変更がある場合は注意が必要です。
