FreeAccessStrategy

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/freelist.c:722

目的

BufferAccessStrategy オブジェクトを解放し、関連付けられたメモリをシステムに返却する。

シグネチャまたは形状

void FreeAccessStrategy(BufferAccessStrategy strategy)

入力と出力

  • **入力**: strategy (BufferAccessStrategy) - 解放対象のバッファアクセス戦略オブジェクト。
  • **出力**: なし。

前提条件と不変条件

  • strategy が NULL の場合、関数は何もしない（安全に無視される）。
  • strategy が有効なポインタである場合、pfree を介してメモリが解放される。

アルゴリズムまたは振る舞い

1. 引数 strategy が NULL でないことを確認する。 2. pfree(strategy) を呼び出し、当該オブジェクトのメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blscan.c
  • src/backend/access/heap/heapam.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/storage/buffer/bufmgr.c
  • **呼び出し先**:
  • pfree (メモリ解放関数)

関連データ構造

  • BufferAccessStrategy: バッファリング戦略を保持する構造体（BufferAccessStrategyData へのポインタ）。

失敗モードと境界条件

  • NULL ポインタが渡された場合、クラッシュせずに正常終了する。
  • 解放済みのポインタを渡した場合の動作は、pfree の仕様に依存する。

保守担当者向けメモ

  • 本関数は GetAccessStrategy 等で割り当てられたメモリを解放するための対となる関数である。
  • デフォルトの戦略（NULL）に対して呼び出しても安全な設計となっている。
