replorigin_get_progress

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/origin.c:1036

目的

指定されたレプリケーション起点（RepOriginId）の現在の進捗状況（リモートLSN）を取得します。

シグネチャまたは形状

XLogRecPtr replorigin_get_progress(RepOriginId node, bool flush)

入力と出力

  • **入力**:
  • node: 取得対象のレプリケーション起点のID。
  • flush: true の場合、対応するローカルLSNまでWALをフラッシュします。
  • **出力**:
  • 該当する起点の remote_lsn を返します。見つからない場合は InvalidXLogRecPtr を返します。

前提条件と不変条件

  • ReplicationOriginLock を共有モードで取得し、共有メモリ上の起点リストを保護します。
  • 個別の ReplicationState にアクセスする際は、その状態の lock を共有モードで取得します。

アルゴリズムまたは振る舞い

1. ReplicationOriginLock を取得し、共有メモリ上の全レプリケーション状態を走査します。 2. 指定された node ID と一致する ReplicationState を検索します。 3. 一致する状態が見つかった場合、その lock を取得して remote_lsn と local_lsn を読み取り、ロックを解放します。 4. ReplicationOriginLock を解放します。 5. flush が true かつ local_lsn が有効な場合、XLogFlush(local_lsn) を呼び出してローカルのWALをディスクへ書き出します。 6. 取得した remote_lsn を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c:1536
  • src/backend/replication/logical/origin.c:1529
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (共有メモリおよび各状態のロック)
  • XLogFlush (条件付き)

関連データ構造

  • RepOriginId: 起点を識別するID。
  • ReplicationState: 共有メモリ上で進捗を保持する構造体。
  • XLogRecPtr: LSN（ログシーケンス番号）の型。

失敗モードと境界条件

  • 指定された node が存在しない場合、remote_lsn は InvalidXLogRecPtr のまま返されます。
  • flush が true であっても、local_lsn が InvalidXLogRecPtr の場合は XLogFlush は実行されません。

保守担当者向けメモ

  • 8バイトのLSN読み取りの整合性を保つため、必ず state->lock を経由してアクセスしてください。
  • max_active_replication_origins に基づくループ処理を行っているため、起点数が多い環境での頻繁な呼び出しには注意が必要です。
