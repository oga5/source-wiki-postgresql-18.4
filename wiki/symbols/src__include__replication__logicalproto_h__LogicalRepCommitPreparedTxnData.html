LogicalRepCommitPreparedTxnData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/logicalproto.h:156

目的

論理レプリケーションにおいて、準備済みトランザクション（Prepared Transaction）のコミット情報を保持し、ワイヤプロトコル経由で転送するためのデータ構造です。

シグネチャまたは形状

typedef struct LogicalRepCommitPreparedTxnData
{
    XLogRecPtr    commit_lsn;    /* コミット時のLSN */
    XLogRecPtr    end_lsn;       /* トランザクション終了時のLSN */
    TimestampTz   commit_time;   /* コミット時刻 */
    TransactionId xid;           /* トランザクションID */
    char          gid[GIDSIZE];  /* グローバル識別子 */
} LogicalRepCommitPreparedTxnData;

入力と出力

本構造体は、論理レプリケーションのストリームから読み取られたデータ（入力）を保持し、サブスクライバー側でのコミット処理に利用されます。

前提条件と不変条件

  • gid フィールドは GIDSIZE で定義された固定長配列であり、準備済みトランザクションを一意に識別するために使用されます。
  • commit_lsn および end_lsn は、PostgreSQLのWAL（Write-Ahead Log）上の有効な位置を指している必要があります。

アルゴリズムまたは振る舞い

論理レプリケーションのワーカープロセスが、パブリッシャーから送信されたコミット準備済みメッセージを受信した際、この構造体にデシリアライズされたデータが格納されます。その後、この情報を用いてローカル側で該当する準備済みトランザクションのコミットが実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/worker.c において、受信したメッセージを処理する際に本構造体がインスタンス化されます。
  • **呼び出し先**: src/backend/replication/logical/proto.c 内の logicalrep_read_commit_prepared 関数が、ストリームからデータを読み取り本構造体に書き込みます。

関連データ構造

  • XLogRecPtr: WAL上の位置を示す型。
  • TransactionId: トランザクション識別子。
  • TimestampTz: 時刻情報。

失敗モードと境界条件

  • gid が GIDSIZE を超える場合や、ストリームから読み取られたデータが不正な形式である場合、デシリアライズ処理が失敗する可能性があります。
  • ネットワーク切断やパケット破損により、LSNやトランザクションIDが不整合な値となる可能性があります。

保守担当者向けメモ

  • 本構造体は論理レプリケーションのプロトコルの一部です。フィールドの追加や変更を行う際は、logicalrep_read_commit_prepared および対応する書き込み側の関数（logicalrep_write_commit_prepared）との互換性を維持してください。
  • GIDSIZE の定義が変更された場合、本構造体のサイズも影響を受けるため注意が必要です。
