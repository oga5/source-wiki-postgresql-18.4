filemap_t

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/filemap.h:89

目的

pg_rewind におけるデータディレクトリ同期計画の最終的な集約構造体です。ソースクラスタとターゲットクラスタの比較結果に基づき、実行すべきファイル操作のリストを保持します。

シグネチャまたは形状

typedef struct filemap_t
{
    uint64      total_size;
    uint64      fetch_size;
    int         nentries;
    file_entry_t *entries[FLEXIBLE_ARRAY_MEMBER];
} filemap_t;

入力と出力

  • **入力**: decide_file_actions() により生成され、calculate_totals() に渡されることで統計情報が更新されます。
  • **出力**: entries 配列に格納された file_entry_t のリストおよび、total_size と fetch_size の集計値。

前提条件と不変条件

  • entries は FLEXIBLE_ARRAY_MEMBER を使用した可変長配列であり、メモリ確保時には offsetof(filemap_t, entries) を用いる必要があります。
  • total_size および fetch_size は、calculate_totals() が呼び出されるまで正しい値が保証されません。

アルゴリズムまたは振る舞い

1. decide_file_actions() がハッシュテーブルを走査し、各ファイルの同期アクションを決定して filemap_t を構築します。 2. calculate_totals() が entries を走査し、ソースクラスタの総サイズと転送が必要なバイト数を計算して構造体に格納します。 3. perform_rewind() がこの構造体を参照し、決定された順序に従って実際のファイル操作を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/filemap.c（decide_file_actions 等）、src/bin/pg_rewind/pg_rewind.c（perform_rewind 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • file_entry_t: 同期対象の各ファイルの状態とアクションを保持する構造体。
  • file_action_t: 各ファイルに対して実行される操作（COPY, TRUNCATE, REMOVE 等）を定義する列挙型。

失敗モードと境界条件

  • メモリ確保時、entries 配列のサイズが不足すると未定義の動作を引き起こす可能性があります。
  • calculate_totals() 実行前に統計情報を参照した場合、初期化されていない値（0）が返されます。

保守担当者向けメモ

  • filemap.c 内で pg_malloc を使用して動的にメモリ確保が行われています。構造体を拡張する際は、entries が末尾にあることを考慮してください。
  • entries のソート順序は同期の安全性に直結するため、変更時は decide_file_actions() のロジックを確認してください。
