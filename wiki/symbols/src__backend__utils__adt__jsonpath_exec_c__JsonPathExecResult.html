JsonPathExecResult

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/jsonpath_exec.c:135

目的

SQL/JSON パス言語のクエリ実行結果の状態を表現するために使用されます。

シグネチャまたは形状

typedef enum JsonPathExecResult
{
	jperOk = 0,
	jperNotFound = 1,
	jperError = 2
} JsonPathExecResult;

入力と出力

  • 本シンボルは列挙型であり、関数からの戻り値として使用されます。
  • 実行エンジン内の各評価関数が、処理の成否や結果の有無を呼び出し元へ伝達するために出力します。

前提条件と不変条件

  • jperOk: クエリまたは式が正常に評価され、結果が得られたことを示します。
  • jperNotFound: クエリにマッチする要素が存在しなかったことを示します。
  • jperError: 実行中にエラーが発生したことを示します。

アルゴリズムまたは振る舞い

  • executeJsonPath をはじめとするパス実行関数群が、処理結果をこの列挙値で返却します。
  • 呼び出し元は、この戻り値を判定することで、後続の処理（値の抽出やエラーハンドリング）を制御します。

呼び出し元と呼び出し先

  • 呼び出し元: executeJsonPath および executeItem 系の再帰的実行関数群が、この型を戻り値として利用しています。
  • 呼び出し先: 列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • JsonPathExecContext: 実行時の状態を保持する構造体であり、この列挙型と組み合わせて実行フローを管理します。

失敗モードと境界条件

  • jperError が返される場合、SQL/JSON 標準に準拠したエラーハンドリングが内部で発生しています。
  • jperNotFound は、パス式が空の結果セットを生成した際の正常な終了状態として扱われます。

保守担当者向けメモ

  • 本列挙型は src/backend/utils/adt/jsonpath_exec.c 内の再帰的な実行ロジック全体で共通のステータスコードとして機能しています。
  • 新たな実行状態を追加する場合は、既存の executeItem 系の関数がこの戻り値を適切に伝播・評価しているか確認してください。
