printTableInit

定義

  • 種別: function
  • 定義箇所: src/fe_utils/print.c:3171

目的

printTableContent 構造体を初期化し、表形式データの出力に必要なメモリ領域を確保する。

シグネチャまたは形状

void printTableInit(printTableContent *const content, const printTableOpt *opt,
                    const char *title, const int ncolumns, const int nrows)

入力と出力

  • **入力**:
  • content: 初期化対象の printTableContent 構造体へのポインタ。
  • opt: 出力オプションを保持する printTableOpt 構造体。
  • title: 表のタイトル文字列。
  • ncolumns: 列数。
  • nrows: 行数。
  • **出力**: なし（content 構造体の各フィールドを更新）。

前提条件と不変条件

  • content は有効なメモリ領域を指している必要がある。
  • ncolumns および nrows は、メモリ確保時に SIZE_MAX を超えるオーバーフローを引き起こさない範囲であること。

アルゴリズムまたは振る舞い

1. content 構造体に opt、title、ncolumns、nrows を設定する。 2. pg_malloc0 を使用して、ヘッダー用メモリ（ncolumns + 1 分）を確保する。 3. ncolumns * nrows の計算結果が SIZE_MAX を超えないか検証し、超える場合はエラーメッセージを出力して exit(EXIT_FAILURE) を実行する。 4. pg_malloc0 を使用して、セル用メモリ（total_cells + 1 分）を確保する。 5. cellmustfree、footers を NULL に初期化する。 6. pg_malloc0 を使用して、アライメント用メモリ（ncolumns + 1 分）を確保する。 7. 構造体内のポインタ（header, cell, footer, align）を確保した各領域の先頭に設定し、cellsadded を 0 にリセットする。

呼び出し元と呼び出し先

  • src/bin/psql/command.c:839

関連データ構造

  • printTableContent: 表のデータ、ヘッダー、フッター、オプションを保持するメイン構造体。
  • printTableOpt: 出力形式やオプションを定義する構造体。

失敗モードと境界条件

  • **メモリ不足**: pg_malloc0 が失敗した場合、内部的にエラーが発生する可能性がある。
  • **整数オーバーフロー**: ncolumns * nrows が SIZE_MAX を超える場合、プログラムはエラーメッセージを出力して異常終了する。

保守担当者向けメモ

  • 本関数で確保したメモリは、後続の printTableCleanup で解放されることが想定されている。
  • cellmustfree が NULL に初期化されるため、動的に確保したセル文字列を管理する場合は、呼び出し側で適切に設定する必要がある。
