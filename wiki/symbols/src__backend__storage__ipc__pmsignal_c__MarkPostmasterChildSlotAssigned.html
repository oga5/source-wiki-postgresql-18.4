MarkPostmasterChildSlotAssigned

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:229

目的

Postmasterが管理する子プロセス用スロットを「割り当て済み（ASSIGNED）」状態にマークし、プロセス管理のライフサイクルを開始します。

シグネチャまたは形状

void MarkPostmasterChildSlotAssigned(int slot)

入力と出力

  • **入力**: slot (int) - 割り当てる対象のスロット番号（1始まり）。
  • **出力**: なし。

前提条件と不変条件

  • slot は 1 以上 num_child_flags 以下である必要があります（Assertによる検証）。
  • 対象のスロットは、操作前に PM_CHILD_UNUSED 状態である必要があります。

アルゴリズムまたは振る舞い

1. 引数 slot が有効範囲内にあるか検証します。 2. 内部インデックス調整のため slot をデクリメントします。 3. 共有メモリ上の PMSignalState->PMChildFlags[slot] が PM_CHILD_UNUSED でない場合、致命的エラー（FATAL）を発生させます。 4. 対象スロットのフラグを PM_CHILD_ASSIGNED に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/pmchild.c:194 からの呼び出しが確認されています。
  • **呼び出し先**: 本関数内での直接的な関数呼び出しはありませんが、共有メモリ構造体 PMSignalState を参照します。

関連データ構造

  • PMSignalData: 共有メモリ上のメイン構造体。PMChildFlags 配列を保持します。
  • PM_CHILD_UNUSED, PM_CHILD_ASSIGNED: スロットの状態を示す列挙値。

失敗モードと境界条件

  • **FATALエラー**: 指定されたスロットが既に PM_CHILD_UNUSED 以外の状態（既に割り当て済みなど）である場合、プロセスは FATAL で終了します。
  • **アサーション失敗**: slot が範囲外の場合、デバッグビルドではアサーション失敗が発生します。

保守担当者向けメモ

  • 本関数は PMSignalState を直接操作するため、呼び出し側で適切な排他制御が行われていることを前提としています。
  • num_child_flags はPostmasterのローカル変数と共有メモリ上の値が同期している必要があります。
