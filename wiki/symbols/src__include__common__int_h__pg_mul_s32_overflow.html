pg_mul_s32_overflow

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:186

目的

2つの int32 型整数の乗算を行い、結果が int32 の範囲を超える（オーバーフローする）かどうかを判定する。

シグネチャまたは形状

static inline bool pg_mul_s32_overflow(int32 a, int32 b, int32 *result)

入力と出力

  • **入力**:
  • a: 乗数 (int32)
  • b: 被乗数 (int32)
  • result: 計算結果を格納するポインタ (int32 *)
  • **出力**:
  • 戻り値: オーバーフローが発生した場合は true、それ以外は false を返す。
  • *result: 演算成功時は計算結果、オーバーフロー時はダミー値 0x5EED が書き込まれる。

前提条件と不変条件

  • コンパイラが __builtin_mul_overflow をサポートしている場合、それを利用する。
  • サポートしていない環境では、int64 へのキャストによる中間計算を行い、結果が PG_INT32_MAX から PG_INT32_MIN の範囲内にあるかを検証する。

アルゴリズムまたは振る舞い

1. HAVE__BUILTIN_OP_OVERFLOW が定義されている場合、コンパイラ組み込み関数を使用して演算と判定を行う。 2. 未定義の場合、a と b を int64 にキャストして乗算し、その結果が int32 の許容範囲を超えているか比較する。 3. 範囲外であれば *result に 0x5EED を代入し true を返す。 4. 範囲内であれば *result に計算結果をキャストして代入し false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/datetime.c
  • src/backend/utils/adt/formatting.c
  • src/backend/utils/adt/int.c
  • **呼び出し先**:
  • コンパイラ組み込み関数 __builtin_mul_overflow（利用可能な場合）。

関連データ構造

  • int32 (PostgreSQL 標準の 32bit 符号付き整数型)
  • int64 (PostgreSQL 標準の 64bit 符号付き整数型)

失敗モードと境界条件

  • 乗算結果が int32 の最小値 (PG_INT32_MIN) または最大値 (PG_INT32_MAX) を超える場合にオーバーフローとして検出される。
  • オーバーフロー発生時の *result の値は、警告回避のためのダミー値 (0x5EED) であり、計算結果としては無効である。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、ヘッダーファイル内で完結している。
  • 移植性を担保するため、コンパイラ固有の組み込み関数と、型拡張による手動チェックの二段構えで実装されている。
