PutMemoryContextsStatsTupleStore

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/mcxtfuncs.c:69

目的

特定の MemoryContext の統計情報を取得し、SQLの結果セットとして返却するための Tuplestore に格納する。

シグネチャまたは形状

static void PutMemoryContextsStatsTupleStore(
    Tuplestorestate *tupstore,
    TupleDesc tupdesc,
    MemoryContext context,
    HTAB *context_id_lookup
)

入力と出力

  • **入力**:
  • tupstore: 結果を蓄積する Tuplestorestate。
  • tupdesc: 結果セットのタプル定義。
  • context: 統計対象の MemoryContext。
  • context_id_lookup: コンテキストIDを検索するためのハッシュテーブル。
  • **出力**: なし（tupstore への書き込みが副作用として発生する）。

前提条件と不変条件

  • context は有効なメモリコンテキストであること（Assert(MemoryContextIsValid(context))）。
  • context_id_lookup には、対象コンテキストおよびその全祖先コンテキストのIDが登録されていること。

アルゴリズムまたは振る舞い

1. **パスの構築**: 対象コンテキストから親を辿り、context_id_lookup を用いて各階層のIDを取得し、リスト（path）を作成する。 2. **統計取得**: context->methods->stats を呼び出し、メモリ使用量（合計、空き、ブロック数など）を MemoryContextCounters に格納する。 3. **識別子の整形**: name および ident を取得する。dynahash 型の場合は name を ident で上書きする。ident が長すぎる場合は pg_mbcliplen で切り詰める。 4. **タプル作成**: コンテキスト名、識別子、型名、階層レベル、パス配列、および統計値をタプルとして構築する。 5. **格納**: tuplestore_putvalues を使用して tupstore に書き込み、一時リスト path を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_get_backend_memory_contexts（src/backend/utils/adt/mcxtfuncs.c 内）。
  • **呼び出し先**:
  • hash_search: ハッシュテーブルからのID取得。
  • context->methods->stats: メモリ統計の取得。
  • pg_mbcliplen: 識別子の切り詰め。
  • tuplestore_putvalues: 結果の格納。

関連データ構造

  • Tuplestorestate: 結果セットの保持用。
  • MemoryContextCounters: メモリ統計情報の格納用。
  • MemoryContextId: ハッシュテーブル内のエントリ構造体。

失敗モードと境界条件

  • context_id_lookup 内に対象コンテキストが見つからない場合、elog(ERROR, "hash table corrupted") を発生させる。
  • ident が MEMORY_CONTEXT_IDENT_DISPLAY_SIZE を超える場合、安全に切り詰められる。

保守担当者向けメモ

  • 本関数は pg_get_backend_memory_contexts から呼び出され、メモリコンテキストの階層構造をSQLから可視化する役割を担う。
  • dynahash コンテキストの表示ロジックは、ログ出力との一貫性を保つために特別扱いされている点に注意すること。
