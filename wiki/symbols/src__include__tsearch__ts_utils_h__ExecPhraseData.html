ExecPhraseData

定義

  • **種別**: struct
  • **定義箇所**: src/include/tsearch/ts_utils.h:159

目的

全文検索における句検索（phrase search）実行時、一致した語彙の位置情報およびマッチング状態を保持・伝播するために使用されます。

シグネチャまたは形状

typedef struct ExecPhraseData
{
    int           npos;      /* 報告された位置の数 */
    bool          allocated; /* pos が palloc されたメモリを指しているか */
    bool          negate;    /* 位置がクエリの NOT 一致箇所であるか */
    WordEntryPos *pos;       /* 順序付けられた重複のない語彙位置 */
    int           width;     /* 語彙単位でのマッチ幅（-1） */
} ExecPhraseData;

入力と出力

  • **入力**: 検索実行エンジンがクエリ評価時にこの構造体を生成または更新します。
  • **出力**: TS_phrase_output 等の関数へ渡され、句検索の条件判定や結果の構築に使用されます。

前提条件と不変条件

  • pos フィールドは、重複のない順序付けられた語彙位置を保持する必要があります。
  • allocated が true の場合、pos は palloc によって確保されたメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

  • 句検索の評価プロセスにおいて、各ノードの一致判定結果（位置情報）を収集します。
  • negate フラグにより、通常の検索結果と NOT 条件による除外箇所を区別します。
  • width はマッチした句の範囲を定義し、位置情報の計算に寄与します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • checkcondition_HL (src/backend/tsearch/wparser_def.c)
  • checkcondition_gin (src/backend/utils/adt/tsginidx.c)
  • checkcondition_arr / checkcondition_bit (src/backend/utils/adt/tsgistidx.c)
  • checkcondition_str (src/backend/utils/adt/tsvector_op.c)
  • tsrank.c 内のランク計算処理
  • **呼び出し先**:
  • TS_phrase_output (src/backend/utils/adt/tsvector_op.c) が本構造体を入力として受け取ります。

関連データ構造

  • WordEntryPos: pos 配列の各要素として使用される位置情報型。

失敗モードと境界条件

  • allocated フラグの管理を誤ると、メモリリークまたは不正なメモリ解放が発生する可能性があります。
  • pos が指す配列のサイズ（npos）と実際のメモリ確保量に不整合がある場合、バッファオーバーランのリスクがあります。

保守担当者向けメモ

  • 本構造体は TS_execute を介した3値論理評価フローにおいて重要な役割を果たします。
  • 句検索の演算子（<N>）の挙動を変更する際は、width および pos の計算ロジックが tsvector_op.c 等の関連関数と整合していることを確認してください。
