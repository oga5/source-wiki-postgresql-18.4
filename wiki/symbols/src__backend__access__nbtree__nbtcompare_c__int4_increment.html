int4_increment

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:239

目的

B-tree インデックスのスキップスキャンにおいて、int4 型のキー値を 1 増加させるための補助関数です。

シグネチャまたは形状

static Datum int4_increment(Relation rel, Datum existing, bool *overflow)

入力と出力

  • **入力**:
  • rel: 対象となるリレーション（未使用）。
  • existing: 増加対象の int4 値（Datum 型）。
  • overflow: オーバーフロー発生時に true を書き込むためのポインタ。
  • **出力**:
  • 成功時: existing + 1 を Datum 型で返す。
  • 失敗時（オーバーフロー時）: (Datum) 0 を返す。

前提条件と不変条件

  • existing は int32 として解釈可能であること。
  • overflow ポインタは有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. existing を int32 に変換します。 2. 値が PG_INT32_MAX と等しい場合、overflow を true に設定し、0 を返して終了します。 3. それ以外の場合、overflow を false に設定し、値に 1 を加算した結果を Datum 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内で、SkipSupport 構造体の increment フィールドに本関数が登録されています。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • SkipSupport: インデックススキップスキャン用の増減ロジックを保持する構造体。

失敗モードと境界条件

  • **境界条件**: existing が PG_INT32_MAX の場合、加算は行われず overflow フラグが立てられます。

保守担当者向けメモ

  • 本関数は SkipSupport インターフェースの一部として使用されます。新しい整数型を追加する際は、同様のパターンに従ってください。
  • rel 引数は現在の実装では使用されていませんが、インターフェース定義上保持されています。
