toast_fetch_datum_slice

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/detoast.c:395

目的

TOASTテーブルに格納された外部データ（varlena）から、指定されたオフセットと長さの範囲（スライス）をメモリ上に取得・復元します。

シグネチャまたは形状

struct varlena *toast_fetch_datum_slice(struct varlena *attr, int32 sliceoffset, int32 slicelength)

入力と出力

  • **入力**:
  • attr: 外部ストレージを指すvarlena構造体ポインタ。
  • sliceoffset: 取得を開始するオフセット。
  • slicelength: 取得するデータの長さ。
  • **出力**:
  • 復元されたデータを含むvarlena構造体へのポインタ。メモリはpallocにより割り当てられます。

前提条件と不変条件

  • 入力attrは、VARATT_IS_EXTERNAL_ONDISKが真である（ディスク上のTOASTデータである）必要があります。
  • 圧縮されたデータの場合、sliceoffsetは0でなければなりません（先頭からの取得のみ許可）。
  • slicelengthが負の値の場合、またはsliceoffset + slicelengthがデータサイズを超える場合は、可能な範囲まで調整されます。

アルゴリズムまたは振る舞い

1. VARATT_EXTERNAL_GET_POINTERを用いて、アライメントを考慮した外部ポインタ情報を取得します。 2. 圧縮データかつslicelength > 0の場合、圧縮情報（va_tcinfo）のサイズ分をslicelengthに加算します。 3. pallocで結果格納用のメモリを確保し、圧縮状態に応じてヘッダーを設定します。 4. slicelengthが0の場合は即座に空の構造体を返します。 5. table_openでTOASTテーブルを開き、table_relation_fetch_toast_sliceを呼び出してチャンクデータを読み込みます。 6. table_closeでテーブルを閉じ、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: detoast_attr_slice（src/backend/access/common/detoast.c内）から呼び出されます。
  • **呼び出し先**: table_open, table_relation_fetch_toast_slice, table_close（いずれもテーブルアクセス層の関数）。

関連データ構造

  • varlena: 可変長データの基本構造体。
  • varatt_external: 外部TOASTデータのポインタ情報。

失敗モードと境界条件

  • VARATT_IS_EXTERNAL_ONDISKでないデータが渡された場合、elog(ERROR)により処理が中断されます。
  • sliceoffsetがデータサイズ以上の場合は、長さ0として扱われます。

保守担当者向けメモ

  • 本関数はメモリを動的に割り当てるため、呼び出し元でのpfreeによる解放が必須です。
  • 圧縮データのスライス取得は、圧縮アルゴリズムの制約により「先頭からのプレフィックス」のみがサポートされています。
