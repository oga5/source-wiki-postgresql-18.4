opclass_for_family_datatype

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/index/amvalidate.c:235

目的

指定されたアクセスメソッド（AM）および演算子ファミリーに属し、かつ特定のデータ型をサポートする演算子クラス（OpClass）の OID を検索します。

シグネチャまたは形状

Oid opclass_for_family_datatype(Oid amoid, Oid opfamilyoid, Oid datatypeoid)

入力と出力

  • **入力**:
  • amoid: 対象アクセスメソッドの OID。
  • opfamilyoid: 検索対象の演算子ファミリーの OID。
  • datatypeoid: 演算子クラスがサポートすべきデータ型の OID。
  • **出力**:
  • 条件に一致する演算子クラスの OID。該当するものがない場合は InvalidOid を返します。

前提条件と不変条件

  • SearchSysCacheList1 を使用して CLAAMNAMENSP キャッシュを走査します。
  • 演算子ファミリーが指定されたアクセスメソッドに属しているかの明示的なチェックは行いません。

アルゴリズムまたは振る舞い

1. CLAAMNAMENSP システムキャッシュから、指定された amoid に関連付けられたすべての演算子クラスのリストを取得します。 2. 取得したリストを反復処理し、各演算子クラスの定義（pg_opclass）を確認します。 3. opcfamily が opfamilyoid と一致し、かつ opcintype が datatypeoid と一致するエントリを検索します。 4. 一致するエントリが見つかった時点でその OID を保持し、ループを終了します。 5. 最後にキャッシュリストを解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hashvalidate.c
  • src/backend/access/index/amvalidate.c
  • src/backend/access/nbtree/nbtvalidate.c
  • **呼び出し先**:
  • SearchSysCacheList1
  • ReleaseCatCacheList
  • GETSTRUCT (マクロ)

関連データ構造

  • CatCList: システムキャッシュから取得したリスト構造。
  • Form_pg_opclass: pg_opclass システムカタログの行構造体。

失敗モードと境界条件

  • 指定された条件に合致する演算子クラスが存在しない場合、InvalidOid を返します。
  • キャッシュリストの取得に失敗した場合の挙動は、SearchSysCacheList1 の仕様に依存します。

保守担当者向けメモ

  • 本関数は全演算子クラスを走査するため、効率は最適ではありませんが、他に利用可能なインデックスが存在しないため現状の設計となっています。
