ItemPointerIndicatesMovedPartitions

定義

  • 種別: function
  • 定義箇所: src/include/storage/itemptr.h:196

目的

指定されたアイテムポインタ（ItemPointerData）が、通常のタプル位置ではなく「パーティション間移動（Moved Partitions）」という特殊な状態を示しているかどうかを判定する。

シグネチャまたは形状

static inline bool ItemPointerIndicatesMovedPartitions(const ItemPointerData *pointer)

入力と出力

  • **入力**: pointer - 検査対象の ItemPointerData 構造体へのポインタ。
  • **出力**: bool - 移動済みパーティションを示す特殊値と一致する場合は true、それ以外は false。

前提条件と不変条件

  • pointer は有効な ItemPointerData 構造体を指している必要がある。
  • 本関数は、ItemPointerData が MovedPartitionsOffsetNumber および MovedPartitionsBlockNumber と一致するかを判定する。

アルゴリズムまたは振る舞い

1. ItemPointerGetOffsetNumber(pointer) を呼び出し、オフセット番号が MovedPartitionsOffsetNumber と等しいかを確認する。 2. ItemPointerGetBlockNumberNoCheck(pointer) を呼び出し、ブロック番号が MovedPartitionsBlockNumber と等しいかを確認する。 3. 両方の条件が真である場合にのみ true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c:6058
  • src/backend/access/heap/heapam_handler.c:412
  • src/backend/executor/execReplication.c:144
  • src/backend/executor/nodeModifyTable.c:3401
  • src/include/access/htup_details.h:487
  • **呼び出し先**:
  • ItemPointerGetOffsetNumber
  • ItemPointerGetBlockNumberNoCheck

関連データ構造

  • ItemPointerData: タプル位置を保持する 6 バイトの構造体。
  • MovedPartitionsOffsetNumber: 移動済みパーティションを示す特殊なオフセット値。
  • MovedPartitionsBlockNumber: 移動済みパーティションを示す特殊なブロック番号。

失敗モードと境界条件

  • pointer が NULL の場合、未定義の動作を引き起こす可能性がある。
  • 本関数は ItemPointerData の内容が特殊なマジックナンバーと一致するかを判定するのみであり、ポインタ自体の妥当性（ItemPointerIsValid）は検証しない。

保守担当者向けメモ

  • 本関数はインライン化されており、パフォーマンスが重視されるパスで使用される。
  • MovedPartitionsOffsetNumber や MovedPartitionsBlockNumber の定義が変更された場合、本関数の判定ロジックも影響を受けるため注意が必要である。
