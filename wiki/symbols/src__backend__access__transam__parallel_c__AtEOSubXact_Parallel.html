AtEOSubXact_Parallel

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/parallel.c:1260

目的

サブトランザクションの終了（コミットまたはアボート）時に、そのサブトランザクションに関連付けられた並列コンテキスト（ParallelContext）をクリーンアップする。

シグネチャまたは形状

void AtEOSubXact_Parallel(bool isCommit, SubTransactionId mySubId)

入力と出力

  • isCommit (bool): サブトランザクションがコミットされる場合は true、アボートされる場合は false。
  • mySubId (SubTransactionId): 終了対象のサブトランザクション ID。
  • 戻り値: なし。

前提条件と不変条件

  • pcxt_list に格納されている並列コンテキストは、サブトランザクション ID の昇順で管理されている必要がある。
  • isCommit が true の場合、未終了の並列コンテキストが存在すると警告（WARNING）がログ出力される。

アルゴリズムまたは振る舞い

1. pcxt_list の先頭から順に、現在のサブトランザクション ID (mySubId) と一致する ParallelContext を走査する。 2. pcxt->subid が mySubId と一致しなくなるまでループを継続する。 3. ループ内で、isCommit が true の場合は「leaked parallel context」という警告を出す。 4. DestroyParallelContext(pcxt) を呼び出し、該当する並列コンテキストを破棄する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xact.c:5123 (サブトランザクションコミット時)
  • src/backend/access/transam/xact.c:5308 (サブトランザクションアボート時)
  • 呼び出し先:
  • DestroyParallelContext (並列コンテキストの破棄)
  • dlist_is_empty, dlist_head_element (リスト操作)
  • elog (警告出力)

関連データ構造

  • ParallelContext: 並列実行のコンテキストを保持する構造体。
  • pcxt_list: 並列コンテキストを管理するグローバルなリスト（parallel.c 内で定義）。

失敗モードと境界条件

  • pcxt_list が空の場合、何も行わずに終了する。
  • 終了対象のサブトランザクション ID に関連付けられたコンテキストが存在しない場合、ループは即座に終了する。

保守担当者向けメモ

  • 本関数はサブトランザクションの終了処理の一部として呼び出される。並列クエリ実行中にサブトランザクションが終了した場合、リソースリークを防ぐために必ず本関数が実行される必要がある。
  • isCommit が true の際の警告は、並列クエリがサブトランザクションの終了を待たずに完了していない（リークしている）ことを示唆する。
