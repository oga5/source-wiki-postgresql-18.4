spgRedoPickSplit

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgxlog.c:528

目的

SP-GiST インデックスの分割操作（PickSplit）に関連する WAL レコードをリプレイし、ソースページ、デスティネーションページ、および内部ノードの物理的な状態を復元します。

シグネチャまたは形状

static void spgRedoPickSplit(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象の WAL レコード）。
  • **出力**: なし（副作用としてバッファの内容を更新）。

前提条件と不変条件

  • WAL レコード内のタプルはアライメントされていないため、アクセス時に memcpy を用いてヘッダーをコピーする必要があります。
  • ホットスタンバイ環境では、リーフタプルや新しい内部タプルの挿入が完了するまで、ソースページ等のロックを保持し、リダイレクトタプルによるリンク切れを防ぐ必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: fillFakeState を用いてリプレイに必要な SpGistState を構築します。 2. **ソースページ処理**: isRootSplit でない場合、XLogReadBufferForRedo または XLogInitBufferForRedo を使用してソースページを取得し、spgPageIndexMultiDelete で不要なタプルを削除（リダイレクトまたはプレースホルダー化）します。 3. **デスティネーションページ処理**: XLogRecHasBlockRef で存在を確認し、必要に応じて初期化または読み込みを行います。 4. **タプル復元**: leafPageSelect に基づき、各リーフタプルをソースまたはデスティネーションページへ addOrReplaceTuple で書き込みます。 5. **LSN 更新**: 変更されたソースおよびデスティネーションページの LSN を更新し、MarkBufferDirty を呼び出します。 6. **内部ノード処理**: 新しい内部タプルを対象ページに書き込みます。innerIsParent が真の場合は、spgUpdateNodeLink を用いて親ノードのリンクを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgxlog.c 内の spg_redo 関数（レコードタイプに応じたディスパッチ）。
  • **呼び出し先**: fillFakeState, XLogRecGetData, XLogRecGetBlockTag, XLogInitBufferForRedo, XLogReadBufferForRedo, spgPageIndexMultiDelete, addOrReplaceTuple, PageSetLSN, MarkBufferDirty, spgUpdateNodeLink 等。

関連データ構造

  • spgxlogPickSplit: WAL レコードのデータ構造。
  • SpGistState: インデックス操作の状態管理構造体。
  • SpGistInnerTupleData, SpGistLeafTupleData: タプルヘッダー情報。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、ページ更新はスキップされます。
  • isRootSplit の場合、ソースページは操作対象外となります。
  • initSrc や initDest が指定されている場合、ページは再初期化されます。

保守担当者向けメモ

  • リプレイ関数内では、親ノードの更新よりも子ページ（リーフや新しいノード）の更新を優先して処理し、スタンバイ側での読み取り整合性を確保しています。
  • ページ更新の順序とロックの保持期間は、スタンバイ時の競合回避において重要です。
