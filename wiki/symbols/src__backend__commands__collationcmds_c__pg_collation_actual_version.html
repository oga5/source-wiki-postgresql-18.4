pg_collation_actual_version

定義

  • 種別: function
  • 定義箇所: src/backend/commands/collationcmds.c:506

目的

指定された照合順序（Collation）の現在のライブラリ環境におけるバージョン文字列を取得します。

シグネチャまたは形状

Datum pg_collation_actual_version(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) により照合順序の OID を取得。
  • **出力**: バージョン情報を含む text 型の Datum を返却。バージョンが特定できない場合は NULL を返却。

前提条件と不変条件

  • 入力された OID が DEFAULT_COLLATION_OID の場合、現在のデータベースの照合順序設定を使用します。
  • それ以外の場合、pg_collation システムカタログから該当する照合順序情報を検索します。
  • pg_collation から取得する場合、プロバイダが COLLPROVIDER_DEFAULT であることは許容されません（Assert により検証）。

アルゴリズムまたは振る舞い

1. 入力 OID に基づき、pg_database または pg_collation システムキャッシュから対象の照合順序情報を取得します。 2. 取得したタプルからプロバイダ（collprovider）とロケール名（collcollate または colllocale）を抽出します。 3. 抽出したプロバイダとロケールを引数として get_collation_actual_version を呼び出し、現在のライブラリ環境におけるバージョンを特定します。 4. 取得したバージョン文字列を text 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/initdb.c: unicode 照合順序のバージョン更新処理で呼び出されます。
  • **呼び出し先**:
  • get_collation_actual_version: バージョン特定のための内部関数。
  • SearchSysCache1, ReleaseSysCache: システムカタログ検索用。

関連データ構造

  • Form_pg_database, Form_pg_collation: カタログタプルの構造体。
  • HeapTuple: システムキャッシュから取得されるタプル。

失敗モードと境界条件

  • 指定された OID が存在しない場合、ERRCODE_UNDEFINED_OBJECT でエラーを報告します。
  • データベースの OID が無効な場合も同様にエラーとなります。

保守担当者向けメモ

  • 本関数はプラットフォーム依存のロケール情報を扱うため、OS や ICU ライブラリの更新時にバージョンが変化する可能性があります。
  • COLLPROVIDER_DEFAULT はユーザー定義の照合順序には使用できないため、カタログ検索時には明示的に除外されています。
