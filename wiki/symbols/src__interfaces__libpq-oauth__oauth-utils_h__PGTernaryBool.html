PGTernaryBool

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq-oauth/oauth-utils.h:71

目的

真偽値に「不明（Unknown）」の状態を加えた、3値論理による状態管理を提供します。主にOAuthの認証状態や、接続設定のデフォルト値の保持に使用されます。

シグネチャまたは形状

typedef enum
{
    PG_BOOL_UNKNOWN = 0,
    PG_BOOL_YES,
    PG_BOOL_NO
} PGTernaryBool;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: PG_BOOL_UNKNOWN (0), PG_BOOL_YES, PG_BOOL_NO のいずれかの値。

前提条件と不変条件

  • PG_BOOL_UNKNOWN が初期値として使用されることが想定されています。

アルゴリズムまたは振る舞い

  • 状態を「未確定」「真」「偽」の3段階で保持します。
  • libpq-oauth モジュール内では、認証の初期化成功フラグなどの状態管理に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/interfaces/libpq-oauth/oauth-curl.c: init_successful 変数の型として使用。
  • src/interfaces/libpq/libpq-int.h: default_transaction_read_only および in_hot_standby メンバーの型として使用。
  • 呼び出し先: なし（データ型定義のため）。

関連データ構造

  • PGconn（libpq-int.h 内）: 接続状態の管理メンバーとして本型を利用。

失敗モードと境界条件

  • 整数値としてキャストされた場合、0が PG_BOOL_UNKNOWN となるため、論理演算を行う際は明示的な比較が必要です。

保守担当者向けメモ

  • 本型は libpq の内部構造体（libpq-int.h）と libpq-oauth モジュールの双方で共有されています。変更を加える際は、libpq 側の依存関係に影響がないか確認してください。
