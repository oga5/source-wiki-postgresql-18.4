get_sub_conninfo

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:344

目的

サブスクライバー（ターゲット）サーバーへの接続に使用する libpq 接続文字列を生成します。

シグネチャまたは形状

static char *get_sub_conninfo(const struct CreateSubscriberOptions *opt)

入力と出力

  • **入力**: CreateSubscriberOptions 構造体へのポインタ。これに含まれるポート番号、ソケットディレクトリ、ユーザー名を参照します。
  • **出力**: 生成された接続文字列を保持する char * 型のメモリ領域。呼び出し元で解放する必要があります。

前提条件と不変条件

  • opt 引数は有効な CreateSubscriberOptions 構造体を指している必要があります。
  • progname グローバル変数が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. createPQExpBuffer() でバッファを初期化します。 2. opt->sub_port を用いて port を追加します。 3. Windows 以外の環境では、opt->socket_dir を用いて host を追加します。 4. opt->sub_username が NULL でない場合、user を追加します。 5. progname を fallback_application_name として追加します。 6. バッファの内容を pg_strdup() で複製し、バッファを破棄した後にその複製を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_createsubscriber.c:2322 にて sub_base_conninfo の初期化に使用されています。
  • **呼び出し先**: createPQExpBuffer, appendConnStrItem, pg_strdup, destroyPQExpBuffer を呼び出します。

関連データ構造

  • CreateSubscriberOptions: 接続設定を保持する構造体。
  • PQExpBuffer: 接続文字列構築用の一時バッファ。

失敗モードと境界条件

  • メモリ不足が発生した場合、pg_strdup や createPQExpBuffer 内部でエラーが発生する可能性があります。
  • opt->sub_username が NULL の場合、接続文字列に user パラメータは含まれません。

保守担当者向けメモ

  • 本関数は pg_createsubscriber ユーティリティ内でのみ使用されます。
  • Windows 環境では socket_dir が無視されるため、ホスト指定が必要な場合は注意が必要です。
