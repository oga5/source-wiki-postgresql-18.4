BitmapAnd

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/plannodes.h:442

目的

複数のビットマップスキャン計画を論理積（AND）で結合し、共通するタプルを抽出するための実行計画ノードです。

シグネチャまたは形状

typedef struct BitmapAnd
{
	Plan		plan;
	List	   *bitmapplans;
} BitmapAnd;

入力と出力

  • **入力**: bitmapplans フィールドに格納された、ビットマップを生成する複数の子プランノードのリスト。
  • **出力**: 子プランノードのビットマップの論理積結果。

前提条件と不変条件

  • plan メンバを構造体の先頭に保持し、Plan 構造体としてキャスト可能です。
  • 通常の Plan ノードが使用する lefttree および righttree は使用されません。

アルゴリズムまたは振る舞い

  • 複数の子プランを実行し、それぞれのビットマップ結果に対して論理積演算を行います。
  • ExecProcNode 呼び出し規約には対応しておらず、専用の実行関数（ExecInitBitmapAnd 等）を通じて処理されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c: ExecInitBitmapAnd を呼び出し初期化を行います。
  • src/backend/commands/explain.c: EXPLAIN 出力生成時にノード型判定や名称取得のために参照されます。
  • **呼び出し先**:
  • src/backend/executor/nodeBitmapAnd.c: ExecInitBitmapAnd（初期化）、ExecEndBitmapAnd（終了）、ExecReScanBitmapAnd（再スキャン）などの実行時ルーチンが定義されています。

関連データ構造

  • Plan: 基底構造体。
  • List: 子プランノードを保持するリスト構造。

失敗モードと境界条件

  • ExecProcNode を介して呼び出された場合、エラー（elog(ERROR, ...)）が発生します。

保守担当者向けメモ

  • 本ノードは BitmapOr と対になる構造です。
  • 実行時には ExecInitBitmapAnd を通じて全ての子プランのサブスキャンを開始する設計となっています。
