ExtensibleNodeEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/nodes/extensible.c:29

目的

拡張機能が定義したカスタムノードのメソッド構造体を、ノード名（識別子）と紐付けて管理するためのレジストリ用データ構造です。

シグネチャまたは形状

typedef struct
{
	char		extnodename[EXTNODENAME_MAX_LEN];
	const void *extnodemethods;
} ExtensibleNodeEntry;

入力と出力

  • extnodename: ノードを一意に識別する名称（最大64バイト）。
  • extnodemethods: ノードの操作（コピー、比較、シリアライズ等）を定義したメソッド構造体へのポインタ。

前提条件と不変条件

  • extnodename は EXTNODENAME_MAX_LEN（64バイト）を超えてはなりません。
  • 本構造体は extensible_node_methods ハッシュテーブルの要素として使用されます。

アルゴリズムまたは振る舞い

  • 拡張機能が RegisterExtensibleNodeMethods を呼び出す際、この構造体が生成され、ハッシュテーブルに格納されます。
  • コアシステムがノードを処理する際、GetExtensibleNodeMethods を通じてこの構造体を検索し、格納された extnodemethods を取得して処理を委譲します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/nodes/extensible.c 内のハッシュテーブル操作関数（hash_search を使用する箇所）から参照されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ExtensibleNodeMethods: extnodemethods が指し示す実際のメソッド群。
  • CustomScanMethods: 類似の目的で使用されるカスタムスキャン用のメソッド構造体。

失敗モードと境界条件

  • ハッシュテーブルへの登録時に同一の extnodename が既に存在する場合、重複登録エラーが発生する可能性があります。
  • GetExtensibleNodeMethods で missing_ok フラグが偽の場合、該当する名前の ExtensibleNodeEntry が見つからなければエラーとなります。

保守担当者向けメモ

  • 本構造体は src/backend/nodes/extensible.c 内の内部的なハッシュテーブル管理用です。
  • 拡張機能の開発者が直接この構造体を操作することはなく、公開APIである RegisterExtensibleNodeMethods 等を介して間接的に利用されます。
