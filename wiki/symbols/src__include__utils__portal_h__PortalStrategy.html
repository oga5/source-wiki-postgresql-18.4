PortalStrategy

定義

  • 種別: enum
  • 定義箇所: src/include/utils/portal.h:89

目的

ポータル（Portal）が扱うクエリの種類を識別し、実行エンジン（Executor）が適切な実行戦略を選択するための分類を提供します。

シグネチャまたは形状

typedef enum PortalStrategy
{
	PORTAL_ONE_SELECT,
	PORTAL_ONE_RETURNING,
	PORTAL_ONE_MOD_WITH,
	PORTAL_UTIL_SELECT,
	PORTAL_SELECT,
	PORTAL_MULTI_QUERY,
} PortalStrategy;

入力と出力

  • 入力: ChoosePortalStrategy(List *stmts) 関数により、解析済みのステートメントリストから決定されます。
  • 出力: PortalData 構造体の strategy メンバとして保持されます。

前提条件と不変条件

  • ポータルは「1つのソースSQLクエリ」に対して1つ作成されます。
  • PORTAL_ONE_SELECT 以外の戦略では、部分的な実行やサスペンドはサポートされません。

アルゴリズムまたは振る舞い

クエリの実行計画に基づき、以下のいずれかに分類されます。

  • PORTAL_ONE_SELECT: 単一のSELECTクエリ。
  • PORTAL_ONE_RETURNING: RETURNING句を伴う単一のデータ変更クエリ。
  • PORTAL_ONE_MOD_WITH: WITH句を伴うデータ変更クエリ。
  • PORTAL_UTIL_SELECT: ユーティリティコマンド（SELECTを返すもの）。
  • PORTAL_MULTI_QUERY: 複数のクエリを含むもの。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tcop/pquery.c にて使用されています。
  • 呼び出し先: src/include/tcop/pquery.h で宣言されている ChoosePortalStrategy 関数がこの値を決定します。

関連データ構造

  • PortalData: ポータルの状態を保持する構造体であり、strategy メンバを通じて本列挙型を参照します。

失敗モードと境界条件

  • 実行戦略の誤判定は、クエリ実行時の予期せぬ動作やリソース管理の不整合を招く可能性があります。

保守担当者向けメモ

  • 本列挙型は、ポータルのライフサイクル管理および実行エンジンとの橋渡しにおいて中心的な役割を果たします。新しいクエリタイプや実行戦略を追加する際は、この列挙型と ChoosePortalStrategy のロジックを同期させる必要があります。
