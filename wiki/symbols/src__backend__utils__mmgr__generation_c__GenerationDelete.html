GenerationDelete

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/generation.c:327

目的

GenerationContext を破棄し、関連付けられたすべてのメモリブロックおよびコンテキストヘッダー自体を解放する。

シグネチャまたは形状

void GenerationDelete(MemoryContext context)

入力と出力

  • **入力**: context (破棄対象の MemoryContext)
  • **出力**: なし

前提条件と不変条件

  • context は GenerationContext 型として初期化されている必要がある。
  • 本関数呼び出し後、context が指すメモリ領域へのアクセスは無効となる。

アルゴリズムまたは振る舞い

1. GenerationReset(context) を呼び出し、KeeperBlock 以外のすべての GenerationBlock を解放する。 2. free(context) を実行し、コンテキストヘッダーと KeeperBlock を含むメモリ領域を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c (メモリコンテキスト削除のディスパッチテーブル経由)
  • **呼び出し先**:
  • GenerationReset
  • free (標準ライブラリ)

関連データ構造

  • MemoryContext: 汎用的なメモリコンテキストの基底型。
  • GenerationContext: 本関数が管理する世代別メモリコンテキストの構造体。

失敗モードと境界条件

  • context が NULL の場合、標準の free() の挙動に従う（通常は未定義動作または無視）。
  • GenerationReset が内部で保持するブロックリストを正しく走査できない場合、メモリリークが発生する可能性がある。

保守担当者向けメモ

  • 本関数は MemoryContextMethods の delete_context フィールドとして登録されている。
  • KeeperBlock はコンテキストヘッダーと同一のメモリブロック内に存在するため、free(context) の一括解放によって同時に破棄される設計となっている。
