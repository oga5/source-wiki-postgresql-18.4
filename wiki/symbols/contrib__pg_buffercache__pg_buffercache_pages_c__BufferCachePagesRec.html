BufferCachePagesRec

定義

  • **種別**: struct
  • **定義箇所**: contrib/pg_buffercache/pg_buffercache_pages.c:39

目的

PostgreSQL の共有バッファキャッシュ内の各バッファのメタデータを保持し、SQL インターフェースを通じてバッファの状態を可視化するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    uint32      bufferid;
    RelFileNumber relfilenumber;
    Oid         reltablespace;
    Oid         reldatabase;
    ForkNumber  forknum;
    BlockNumber blocknum;
    bool        isvalid;
    bool        isdirty;
    uint16      usagecount;
    int32       pinning_backends;
} BufferCachePagesRec;

入力と出力

本構造体は関数への入力や出力として直接渡されるものではなく、pg_buffercache_pages 関数が共有バッファの状態をスキャンした結果を格納するためのコンテナとして使用されます。

前提条件と不変条件

  • pinning_backends は int32 型で十分なサイズを確保しています。これは MAX_BACKENDS の制限により、単一のバッファが過剰な数のバックエンドによってピン留めされることがなく、かつ各バックエンドが PrivateRefCount 機構によりバッファを一度しかピン留めできないためです。

アルゴリズムまたは振る舞い

本構造体は、pg_buffercache_pages 関数において NBuffers 分の配列としてメモリ上に確保されます。各フィールドには、対応するバッファヘッダーから抽出されたメタデータがコピーされます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_buffercache_pages 関数（contrib/pg_buffercache/pg_buffercache_pages.c）が、バッファキャッシュの情報を収集する際にこの構造体の配列を割り当て、各フィールドに値を設定します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • BufferDesc: storage/bufmgr.h で定義されるバッファ記述子。本構造体の元となる情報を保持しています。
  • BufferCachePagesContext: pg_buffercache_pages の実行コンテキストとして、本構造体の配列を保持するために使用されます。

失敗モードと境界条件

  • NBuffers が非常に大きい環境では、本構造体の配列を確保するために大量のメモリを消費する可能性があります。

保守担当者向けメモ

  • 本構造体のフィールドを変更する場合は、pg_buffercache_pages 関数内のデータ抽出ロジック（バッファヘッダーからのコピー処理）も併せて更新する必要があります。
  • 統計情報の取得はアトミックな読み取りを優先しているため、本構造体に格納される値は「調査時点でのスナップショット」であり、システム全体の一貫性を保証するものではありません。
