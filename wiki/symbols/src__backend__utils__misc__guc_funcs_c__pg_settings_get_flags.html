pg_settings_get_flags

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc_funcs.c:541

目的

指定されたGUC（Grand Unified Configuration）パラメータ名に対応するフラグ情報を取得し、テキスト配列として返却する。

シグネチャまたは形状

Datum pg_settings_get_flags(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_DATUM(0) を介して取得されるGUCパラメータ名（text型）。
  • **出力**: パラメータに設定されているフラグのリスト（text[]型）。該当するパラメータが存在しない場合は NULL を返す。

前提条件と不変条件

  • MAX_GUC_FLAGS（6）は、チェック対象のフラグ数と一致している必要がある。
  • find_option がパラメータを特定できない場合、関数は即座に NULL を返す。

アルゴリズムまたは振る舞い

1. find_option を呼び出し、引数で指定された名前のGUCパラメータレコードを取得する。 2. レコードが存在しない場合、PG_RETURN_NULL() で終了する。 3. record->flags をビットマスクで評価し、以下のフラグがセットされているか順次確認する。

  • GUC_EXPLAIN, GUC_NO_RESET, GUC_NO_RESET_ALL, GUC_NO_SHOW_ALL, GUC_NOT_IN_SAMPLE, GUC_RUNTIME_COMPUTED

    • 4. 該当するフラグ名を CStringGetTextDatum で変換し、flags 配列に格納する。 5. construct_array_builtin を使用して、収集したフラグの配列を構築し、PG_RETURN_ARRAYTYPE_P で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は明示されていない（SQLインターフェース経由での呼び出しが想定される）。
  • **呼び出し先**: find_option（GUCパラメータ検索用内部関数）。

関連データ構造

  • struct config_generic: GUCパラメータの共通定義構造体。
  • ArrayType: PostgreSQLの配列型表現。

失敗モードと境界条件

  • 指定されたパラメータ名が存在しない場合、find_option が ERROR を発生させる可能性がある（find_option の引数設定による）。
  • cnt が MAX_GUC_FLAGS を超えることは、現在の実装ロジック上発生しない（Assert で検証済み）。

保守担当者向けメモ

  • 新しいGUCフラグが追加された場合、本関数のフラグ判定ロジックおよび MAX_GUC_FLAGS の値を更新する必要がある。
