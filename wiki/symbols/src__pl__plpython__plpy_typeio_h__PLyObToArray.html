PLyObToArray

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_typeio.h:99

目的

PythonオブジェクトからPostgreSQLの配列型へ変換する際に必要なメタデータと変換情報を保持します。

シグネチャまたは形状

typedef struct PLyObToArray
{
	PLyObToDatum *elm;			/* conversion info for array's element type */
	Oid			elmbasetype;	/* element base type */
} PLyObToArray;

入力と出力

  • **入力**: PythonのリストやシーケンスオブジェクトをPostgreSQLの配列データへ変換するための設定情報。
  • **出力**: 変換処理の実行時に使用される、要素型ごとの変換関数および型情報。

前提条件と不変条件

  • elm フィールドは、配列要素の変換を担う PLyObToDatum 構造体を指す必要があります。
  • elmbasetype は、配列要素の基底型を示す有効な Oid である必要があります。

アルゴリズムまたは振る舞い

PythonオブジェクトからPostgreSQL配列への変換プロセスにおいて、この構造体は配列内の各要素を個別に変換するためのキャッシュされた変換関数（elm）を提供します。これにより、再帰的な型変換がサポートされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_typeio.h:142 にて、より上位の変換状態を保持する構造体の一部として定義されています。
  • **呼び出し先**: 具体的な変換関数（PLyObToDatum が保持する関数）が呼び出されます。

関連データ構造

  • PLyObToDatum: 配列要素の変換情報を保持する構造体。
  • PLyObToScalar, PLyObToTuple, PLyObToDomain: 他の型変換用構造体。

失敗モードと境界条件

  • 変換対象のPythonオブジェクトが配列（シーケンス）ではない場合、または要素の型変換に失敗した場合の挙動は、この構造体を利用する変換関数側の実装に依存します。

保守担当者向けメモ

  • 本構造体は plpy_typeio.c での変換処理において、配列の再帰的な処理を制御するために使用されます。
  • 構造体内の elm は、配列の要素型に応じて動的に初期化されるため、メモリ管理には注意が必要です。
