RegisterExtensibleNodeEntry

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/extensible.c:38

目的

拡張ノードまたはカスタムスキャンメソッドを管理するためのハッシュテーブルに、新しいノード型のエントリを登録します。

シグネチャまたは形状

static void RegisterExtensibleNodeEntry(HTAB **p_htable, const char *htable_label, const char *extnodename, const void *extnodemethods)

入力と出力

  • p_htable: 登録先ハッシュテーブルへのポインタ。NULLの場合は初期化されます。
  • htable_label: ハッシュテーブル作成時に使用する識別ラベル。
  • extnodename: 登録するノード型の名称。
  • extnodemethods: ノードに関連付けるメソッド構造体へのポインタ。
  • 出力: なし（成功時）。失敗時はエラーを報告します。

前提条件と不変条件

  • extnodename は EXTNODENAME_MAX_LEN（64バイト）未満である必要があります。
  • 指定された extnodename が既にハッシュテーブルに存在する場合、登録は失敗します。

アルゴリズムまたは振る舞い

1. p_htable が NULL の場合、hash_create を用いてハッシュテーブルを初期化します。 2. extnodename の長さが EXTNODENAME_MAX_LEN 以上であればエラーを報告します。 3. hash_search を使用して、指定された名前でハッシュテーブルを検索します。 4. 既にエントリが存在する場合（found が真）、ERRCODE_DUPLICATE_OBJECT でエラーを報告します。 5. 新規エントリを作成し、extnodemethods を格納します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • RegisterExtensibleNodeMethods (同ファイル内)
  • RegisterCustomScanMethods (同ファイル内)
  • 呼び出し先:
  • hash_create (PostgreSQL ハッシュテーブル API)
  • hash_search (PostgreSQL ハッシュテーブル API)
  • elog / ereport (エラー報告)

関連データ構造

  • ExtensibleNodeEntry: ハッシュテーブルに格納されるエントリ構造体。
  • HTAB: utils/hsearch.h で定義されるハッシュテーブル構造体。

失敗モードと境界条件

  • extnodename が長すぎる場合: ERROR を発生させます。
  • 重複する名前で登録しようとした場合: ERRCODE_DUPLICATE_OBJECT を伴う ERROR を発生させます。

保守担当者向けメモ

  • 本関数は内部的な登録ロジックを共通化したものです。直接呼び出すのではなく、公開APIである RegisterExtensibleNodeMethods 等を経由して利用してください。
  • ハッシュテーブルの初期化は遅延評価されるため、初回呼び出し時にのみ実行されます。
