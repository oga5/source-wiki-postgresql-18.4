validate_exec

定義

  • 種別: function
  • 定義箇所: src/common/exec.c:87

目的

指定されたパスがファイルとして存在し、かつ実行可能および読み取り可能であることを検証する。

シグネチャまたは形状

int validate_exec(const char *path);

入力と出力

  • **入力**: path (検証対象のファイルパス文字列)
  • **出力**:
  • 0: 検証成功（読み取り・実行可能）
  • -1: ファイルが存在しない、または通常ファイルではない
  • -2: 実行可能だが読み取り不可（動的ロードに失敗するケース）

前提条件と不変条件

  • 対象パスがシンボリックリンクである場合、stat(2) の挙動に依存する（リンク先を評価する）。
  • Windows環境では、パス末尾に .exe がない場合に自動補完を試みる。

アルゴリズムまたは振る舞い

1. **Windows補完**: WIN32 定義時、.exe 拡張子がない場合は付与して検証する。 2. **存在確認**: stat(2) を呼び出し、ファイルが存在し、かつ通常ファイル (S_ISREG) であることを確認する。ディレクトリ等の場合は errno を設定して失敗する。 3. **権限検証**:

  • **POSIX**: access(2) を使用し、R_OK（読み取り）および X_OK（実行）を確認する。
  • **Windows**: stat 結果のモードビット (S_IRUSR, S_IXUSR) を直接参照する。

    • 4. **戻り値判定**: 実行権限と読み取り権限の組み合わせに基づき、成功または特定の失敗コードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/exec.c
  • src/common/exec.c (内部のパス解決ロジック内)
  • **呼び出し先**:
  • stat(2)
  • access(2) (POSIXのみ)

関連データ構造

  • struct stat: ファイル属性の取得に使用。

失敗モードと境界条件

  • **ファイル不在**: stat が失敗し -1 を返す。
  • **非通常ファイル**: ディレクトリの場合は EISDIR、それ以外は EPERM を errno に設定し -1 を返す。
  • **権限不足**: 読み取り不可の場合は -2、実行不可の場合は -1 を返す。

保守担当者向けメモ

  • access(2) は実効ユーザーIDに基づくため、setuidプログラム等では注意が必要。
  • Windows環境での .exe 補完ロジックは、MAXPGPATH を超えるパス長に対して安全なバッファ操作を行っている。
