binary_upgrade_set_next_multirange_pg_type_oid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_upgrade_support.c:74

目的

pg_upgrade 実行時において、次に作成されるマルチレンジ型の OID をシステムカタログへ強制的に割り当てるための予約値を設定します。

シグネチャまたは形状

Datum binary_upgrade_set_next_multirange_pg_type_oid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して、次に使用すべき OID を受け取ります。
  • **出力**: なし（void）。

前提条件と不変条件

  • サーバーが binary_upgrade モードで起動されている必要があります。この条件は CHECK_IS_BINARY_UPGRADE マクロによって検証され、モード外であればエラーが発生します。

アルゴリズムまたは振る舞い

1. 引数から OID を取得します。 2. CHECK_IS_BINARY_UPGRADE を実行し、アップグレードモードであることを確認します。 3. グローバル変数 binary_upgrade_next_mrng_pg_type_oid に取得した OID を代入します。 4. PG_RETURN_VOID() を返し、処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の SQL クエリ実行箇所から呼び出されます。
  • **呼び出し先**: 本関数自体は他の関数を呼び出しません。

関連データ構造

  • binary_upgrade_next_mrng_pg_type_oid: catalog/binary_upgrade.h で宣言されているグローバル変数。本関数によって値が保持されます。

失敗モードと境界条件

  • binary_upgrade モード以外で呼び出された場合、CHECK_IS_BINARY_UPGRADE によりエラーが発生します。

保守担当者向けメモ

  • 本関数は pg_upgrade によるカタログ移行の整合性維持に不可欠です。グローバル変数の更新は、後続のカタログ挿入処理において当該 OID が使用されることを前提としています。
