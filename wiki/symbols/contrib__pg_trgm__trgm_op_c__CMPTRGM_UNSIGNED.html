CMPTRGM_UNSIGNED

定義

  • 種別: static inline function
  • 定義箇所: contrib/pg_trgm/trgm_op.c:204

目的

トリグラム（3文字の連続）を符号なし文字として比較するための比較関数です。システム環境における文字の符号解釈に依存せず、一貫した比較結果を提供します。

シグネチャまたは形状

static inline int CMPTRGM_UNSIGNED(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つのメモリ領域へのポインタ a および b。
  • **出力**: 比較結果を示す整数。a < b なら負、a > b なら正、a == b なら 0 を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、少なくとも3バイト以上の有効なメモリ領域を指している必要があります。
  • 内部で CMPCHAR マクロを利用しており、各バイトを unsigned char としてキャストして比較します。

アルゴリズムまたは振る舞い

1. 1バイト目から順に CMPCHAR を用いて比較を行います。 2. 1バイト目が等しい場合、2バイト目を比較します。 3. 2バイト目も等しい場合、3バイト目を比較します。 4. 最初に差異が見つかったバイトの比較結果を即座に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_op.c 内の CMPTRGM_CHOOSE 関数において、システム設定に応じて関数ポインタ CMPTRGM に代入されます。
  • **呼び出し先**: CMPCHAR マクロ（trgm.h で定義）。

関連データ構造

  • TRGM: トリグラムを保持する型。本関数はこの型の比較に使用されます。

失敗モードと境界条件

  • 入力ポインタが NULL または3バイト未満の領域を指している場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本関数は CMPTRGM_SIGNED と対をなします。比較ロジックを変更する場合は、両方の関数に一貫性を持たせてください。
  • 直接呼び出すのではなく、CMPTRGM 関数ポインタを経由して利用することが推奨されています。
