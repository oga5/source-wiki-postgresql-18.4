make_distinct_op

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_expr.c:3083

目的

SQLの IS DISTINCT FROM 構文に対応する演算子ノードを生成し、意味解析を行う。

シグネチャまたは形状

Expr *make_distinct_op(ParseState *pstate, List *opname, Node *ltree, Node *rtree, int location)

入力と出力

  • **入力**:
  • pstate: パーサーの状態管理構造体。
  • opname: 演算子名（リスト形式）。
  • ltree, rtree: 比較対象となる左辺および右辺の式ノード。
  • location: 構文解析上の位置情報。
  • **出力**:
  • T_DistinctExpr 型としてタグ付けされた Expr 構造体へのポインタ。

前提条件と不変条件

  • make_op を介して生成された OpExpr 構造体と DistinctExpr 構造体がメモリレイアウト上で互換性を持つことを前提としている。
  • 戻り値の演算子は boolean 型を返す必要がある。

アルゴリズムまたは振る舞い

1. make_op を呼び出し、指定された演算子名と左右の式から OpExpr ノードを生成する。 2. 生成された演算子の戻り値型が BOOLOID（boolean）でない場合、エラーを報告する。 3. 演算子が集合（set）を返す場合、エラーを報告する。 4. NodeSetTag を使用して、ノードの型タグを T_DistinctExpr に書き換える。 5. 変換済みのノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_expr.c 内の複数箇所（1062行目、3062行目など）。
  • **呼び出し先**: make_op（src/backend/parser/parse_oper.c 関連）、ereport、parser_errposition。

関連データ構造

  • ParseState: パーサーのコンテキスト。
  • OpExpr: 演算子式ノード。
  • DistinctExpr: IS DISTINCT FROM を表現するノード。

失敗モードと境界条件

  • **型不一致**: 演算子が boolean を返さない場合、ERRCODE_DATATYPE_MISMATCH を発生させる。
  • **集合返却**: 演算子が集合を返す場合、ERRCODE_DATATYPE_MISMATCH を発生させる。

保守担当者向けメモ

  • 本関数は OpExpr を DistinctExpr にキャストして再利用するハック的な実装を行っているため、構造体の定義変更時には注意が必要である。
