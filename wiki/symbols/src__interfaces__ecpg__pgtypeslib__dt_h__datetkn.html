datetkn

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/dt.h:203

目的

日付・時刻の解析処理において、文字列トークンとそれに対応する型および値を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
    char        token[TOKMAXLEN + 1]; /* NUL終端されたトークン文字列 */
    char        type;                 /* フィールド型コード */
    int32       value;                /* 型に応じた値 */
} datetkn;

入力と出力

本構造体はデータ保持を目的としており、関数としての入出力は持ちません。各フィールドは解析ロジックによって読み書きされます。

前提条件と不変条件

  • token フィールドは常に NUL 終端されている必要があります。
  • value フィールドの意味は type フィールドの値に依存します。

アルゴリズムまたは振る舞い

日付・時刻の解析時に、入力文字列をトークンとして照合するために使用されます。メモリ効率を考慮し、サイズが小さく保たれています。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、ECPG ライブラリ内での直接的な呼び出し元は特定できません。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。
  • **備考**: PostgreSQL バックエンドの src/backend/utils/adt/datetime.c において、datebsearch 関数や datetktbl 配列などで同様の構造体が参照されていることが確認されています。

関連データ構造

  • TOKMAXLEN: token 配列のサイズを規定する定数。

失敗モードと境界条件

  • token の長さが TOKMAXLEN を超える場合、切り詰めやバッファオーバーランのリスクがあるため、定義された範囲内での使用が必須です。

保守担当者向けメモ

  • 本構造体はバックエンドの datetime.h と同期しています。変更を加える際は、バックエンド側との整合性に十分注意してください。
  • メモリ効率が重視されているため、フィールドの追加やサイズ変更は慎重に行ってください。
