PQresultSetInstanceData

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/libpq-events.c:141

目的

PGresult オブジェクトに関連付けられた、特定のイベントプロシージャに対応するインスタンスデータを更新します。

シグネチャまたは形状

int PQresultSetInstanceData(PGresult *result, PGEventProc proc, void *data);

入力と出力

  • **入力**:
  • result: 対象となる PGresult 構造体へのポインタ。
  • proc: データを更新する対象のイベントプロシージャ（関数ポインタ）。
  • data: 設定するインスタンスデータへのポインタ。
  • **出力**:
  • int: 成功した場合は true (1)、失敗した場合は false (0) を返します。

前提条件と不変条件

  • result および proc が NULL でないこと。
  • 指定された proc が、対象の PGresult に対して事前に登録されている必要があります。

アルゴリズムまたは振る舞い

1. result または proc が NULL の場合、直ちに false を返します。 2. result->events 配列を走査し、登録されているイベントプロシージャの中から proc と一致するエントリを検索します。 3. 一致するエントリが見つかった場合、そのエントリの data フィールドを引数 data で更新し、true を返します。 4. 配列内に一致する proc が存在しない場合、何も変更せず false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、具体的な呼び出し元は特定できません。
  • **呼び出し先**: 内部的な関数呼び出しは行いません。

関連データ構造

  • PGresult: イベント管理用の配列 events を保持する構造体。
  • PGEventProc: イベント処理関数の型定義。

失敗モードと境界条件

  • result または proc が NULL の場合、処理は行われず false が返されます。
  • 登録されていない proc を指定した場合、検索が失敗し false が返されます。

保守担当者向けメモ

  • 本関数は PGresult に紐づくデータの更新のみを行います。データの登録（PGEVT_REGISTER イベントの発火など）は PQregisterEventProc 等の別のメカニズムで行われる必要があります。
  • イベント識別には関数ポインタのアドレスが使用されるため、同一プロシージャを複数回登録することはできません。
