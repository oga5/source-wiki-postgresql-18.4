plpgsql_ns_push

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_funcs.c:53

目的

PL/pgSQL の名前空間（スコープ）管理において、新しいレベルをスタックにプッシュし、ラベルを登録します。

シグネチャまたは形状

void plpgsql_ns_push(const char *label, PLpgSQL_label_type label_type)

入力と出力

  • **入力**:
  • label: スコープに関連付けるラベル名（NULL の場合は空文字列として扱われる）。
  • label_type: ラベルの型を示す PLpgSQL_label_type 列挙値。
  • **出力**: なし（void）。

前提条件と不変条件

  • plpgsql_ns_additem を呼び出して内部的な名前空間スタックを更新します。
  • label が NULL の場合、内部で空文字列 "" に正規化されます。

アルゴリズムまたは振る舞い

1. 引数 label が NULL であるかを確認し、NULL であれば空文字列に置換します。 2. plpgsql_ns_additem を呼び出し、PLPGSQL_NSTYPE_LABEL 型のアイテムとして、指定された label と label_type を現在の名前空間スタックに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpgsql/src/pl_comp.c:271
  • src/pl/plpgsql/src/pl_comp.c:804
  • **呼び出し先**:
  • plpgsql_ns_additem

関連データ構造

  • PLpgSQL_label_type: ラベルの種別を定義する列挙型。
  • PLpgSQL_nsitem: 名前空間スタックを構成する要素（plpgsql_ns_additem 経由で操作）。

失敗モードと境界条件

  • label に NULL を渡した場合の安全な処理が実装されています。
  • 名前空間スタックのメモリ不足や、plpgsql_ns_additem 内部でのエラー発生時は、PostgreSQL のエラーハンドリング機構に依存します。

保守担当者向けメモ

  • 本関数は名前空間の階層構造を構築するための基本操作です。
  • plpgsql_ns_pop と対になる操作であるため、スコープの開始と終了が正しく対応しているか確認してください。
