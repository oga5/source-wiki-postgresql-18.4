pgaio_io_get_handle_data

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_callback.c:155

目的

PgAioHandle に関連付けられた補助データ（handle data）へのポインタと、そのデータ長を取得します。

シグネチャまたは形状

uint64 *pgaio_io_get_handle_data(PgAioHandle *ioh, uint8 *len);

入力と出力

  • **入力**:
  • ioh: データを取得する対象の PgAioHandle 構造体へのポインタ。
  • len: 取得したデータの長さを格納するための uint8 型変数へのポインタ。
  • **出力**:
  • 戻り値: 共有メモリ上の補助データ領域を指す uint64 型のポインタ。
  • len 引数経由: 補助データの長さ（バイト単位）。

前提条件と不変条件

  • ioh->handle_data_len が 0 より大きいこと（アサーションにより検証）。
  • pgaio_ctl が初期化されており、有効な共有メモリ領域を指していること。

アルゴリズムまたは振る舞い

1. ioh->handle_data_len が 0 より大きいことを確認します。 2. ioh->handle_data_len の値を len 引数に書き込みます。 3. pgaio_ctl->handle_data 配列の、ioh->iovec_off で指定されたオフセット位置にあるアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c (行 6791 および 7201)
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • PgAioHandle: I/O操作を管理するハンドル構造体。
  • pgaio_ctl: AIOサブシステム全体の制御情報を保持する共有メモリ上の構造体。

失敗モードと境界条件

  • handle_data_len が 0 の場合、アサーション失敗によりプロセスが異常終了します。
  • iovec_off が pgaio_ctl->handle_data の有効範囲外である場合、メモリ破壊や不正なメモリアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数は pgaio_io_set_handle_data_64 等で設定されたデータを読み出すためのアクセサです。
  • 共有メモリ上のデータに直接アクセスするため、呼び出し側は適切なロックやライフサイクル管理が保証されているコンテキストで実行する必要があります。
