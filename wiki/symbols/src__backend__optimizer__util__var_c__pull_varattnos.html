pull_varattnos

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/var.c:295

目的

指定された式ツリー（node）を走査し、特定のリレーション（varno）を参照しているすべての属性番号（varattno）を収集してビットマップセット（varattnos）に格納します。

シグネチャまたは形状

void pull_varattnos(Node *node, Index varno, Bitmapset **varattnos)

入力と出力

  • **入力**:
  • node: 走査対象の式ツリーまたはノード。
  • varno: 属性を抽出対象とするリレーションのインデックス。
  • varattnos: 収集した属性番号を保持する Bitmapset へのポインタのポインタ。
  • **出力**:
  • *varattnos: 抽出された属性番号が追加された Bitmapset。

前提条件と不変条件

  • varattnos が指すポインタは、初期状態では NULL または既存の Bitmapset を指している必要があります。
  • 走査中に見つかった属性番号は、既存のビットマップセットに論理和（OR）で追加されます。

アルゴリズムまたは振る舞い

1. pull_varattnos_context 構造体を初期化し、対象の varno と現在の varattnos セットを格納します。 2. pull_varattnos_walker を呼び出し、式ツリーを再帰的に走査します。 3. 走査過程で対象の varno を持つ Var ノードが見つかるたびに、その varattno をコンテキスト内のビットマップセットに追加します。 4. 走査終了後、更新されたビットマップセットを *varattnos に書き戻します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/deparse.c
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/catalog/partition.c
  • src/backend/commands/copy.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/statscmds.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • pull_varattnos_walker (直接の走査実行)

関連データ構造

  • Bitmapset: 属性番号の集合を効率的に保持するために使用。
  • pull_varattnos_context: 走査中の状態（対象 varno と収集済み varattnos）を保持する内部構造体。

失敗モードと境界条件

  • node が NULL の場合、何も収集されず正常に終了します。
  • varno に一致する Var ノードがツリー内に存在しない場合、varattnos は変更されません。

保守担当者向けメモ

  • 本関数は walker パターンを利用しており、サブクエリ内の変数（sublevels_up）の扱いについては pull_varattnos_walker の実装に依存します。
  • 属性番号の収集は Bitmapset を介して行われるため、メモリ管理は Bitmapset の仕様に従います。
