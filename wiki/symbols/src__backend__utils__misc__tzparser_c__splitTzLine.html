splitTzLine

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/tzparser.c:97

目的

タイムゾーン設定ファイルの各行を解析し、略称、オフセット、夏時間フラグ等の情報を tzEntry 構造体に抽出する。

シグネチャまたは形状

static bool splitTzLine(const char *filename, int lineno, char *line, tzEntry *tzentry)

入力と出力

  • **入力**:
  • filename: 解析対象のファイル名。
  • lineno: 現在の行番号。
  • line: 解析対象の文字列（破壊的に変更される）。
  • tzentry: 解析結果を格納する tzEntry 構造体へのポインタ。
  • **出力**:
  • 成功時は true を返し、tzentry に値が設定される。
  • 失敗時は false を返し、GUC_check_errmsg を介してエラーメッセージが設定される。

前提条件と不変条件

  • line 引数は strtok_r によるトークン化のために破壊的に変更される。
  • タイムゾーンファイル名は英字のみに制限されている。
  • 呼び出し元は elog(ERROR) を避けるため、本関数が返すエラーメッセージを適切に処理する必要がある。

アルゴリズムまたは振る舞い

1. tzentry にファイル名と行番号を記録する。 2. strtok_r を使用して行をトークン化し、略称（abbrev）を取得する。 3. 次のトークン（オフセットまたはゾーン名）を取得する。 4. トークンが数値、+、- で始まる場合、オフセット値として解析する。

  • オフセット後に D があれば夏時間（DST）フラグを立てる。

    • 5. トークンが数値以外の場合、ゾーン名として扱い、オフセットを 0 に設定する（ゾーンの妥当性検証は行わない）。 6. 残りのトークンが # で始まらない場合、構文エラーとして処理する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/tzparser.c 内の ParseTzFile 関数から呼び出される。
  • **呼び出し先**: strtok_r, pstrdup, isdigit, strtol, pg_strcasecmp, GUC_check_errmsg 等の標準ライブラリおよび PostgreSQL 内部関数。

関連データ構造

  • tzEntry: 解析されたタイムゾーン情報を保持する構造体。

失敗モードと境界条件

  • **略称欠落**: 行の先頭に略称がない場合。
  • **オフセット欠落**: 略称の後にオフセットまたはゾーン名がない場合。
  • **数値解析失敗**: オフセットとして指定された文字列が数値として不正な場合。
  • **構文エラー**: 行の末尾に # 以外の予期しない文字が存在する場合。

保守担当者向けメモ

  • addToArray は strcmp を使用してソート順を維持しており、datetime.c 側の検索ロジックと密結合しているため、変更時には注意が必要である。
  • ゾーン名の検証を行わないのは、セッションで利用されない可能性のあるゾーンのロードを避けるためである。
