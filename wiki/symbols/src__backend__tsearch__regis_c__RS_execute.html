RS_execute

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/regis.c:207

目的

コンパイル済みの Regis 構造体を使用して、対象文字列が指定された正規表現パターンに適合するかを判定する。

シグネチャまたは形状

bool RS_execute(Regis *r, char *str)

入力と出力

  • **入力**:
  • r: コンパイル済みの Regis 構造体へのポインタ。
  • str: マッチング対象の文字列（C文字列）。
  • **出力**:
  • bool: パターンに適合した場合は true、適合しない場合は false を返す。

前提条件と不変条件

  • r は RS_compile によって適切に初期化されている必要がある。
  • str は有効な文字列であること。
  • マルチバイト文字を含む場合、pg_mblen_cstr を用いて文字境界が正しく処理される。

アルゴリズムまたは振る舞い

1. 入力文字列 str の文字数をカウントする。 2. 文字数が r->nchar（パターンが必要とする最小文字数）未満であれば false を返す。 3. r->issuffix が真の場合、文字列の末尾から r->nchar 分だけ開始位置を調整する。 4. RegisNode の連結リストを順次走査する。

  • RSF_ONEOF: 現在の文字が ptr->data に含まれるか mb_strchr で確認し、含まれなければ false を返す。
  • RSF_NONEOF: 現在の文字が ptr->data に含まれるか mb_strchr で確認し、含まれていれば false を返す。

    • 5. 全てのノードで適合を確認できれば true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/spell.c:2181
  • **呼び出し先**:
  • pg_mblen_cstr (マクロ/関数)
  • mb_strchr (src/backend/tsearch/regis.c)
  • elog (エラー発生時)

関連データ構造

  • Regis: パターン情報を保持する構造体。
  • RegisNode: パターンの各要素を保持する連結リストノード。

失敗モードと境界条件

  • 文字列長がパターン長より短い場合、即座に false を返す。
  • 未知の RegisNode 型が検出された場合、elog(ERROR) により処理を中断する。

保守担当者向けメモ

  • 本関数は汎用的な正規表現エンジンではなく、ISpell 辞書等の特定の用途に最適化された限定的なマッチング処理を行う。
  • mb_strchr を介してマルチバイト文字を処理しているため、文字コードセットの変更時には注意が必要である。
