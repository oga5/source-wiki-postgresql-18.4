JsonTablePlanRowSource

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonpath_exec.c:167

目的

JSON_TABLE 関数実行時において、現在処理中の行ソース（値）を保持するための構造体です。

シグネチャまたは形状

typedef struct JsonTablePlanRowSource
{
	Datum		value;
	bool		isnull;
} JsonTablePlanRowSource;

入力と出力

  • **入力**: Datum 型のデータ値および bool 型の NULL フラグ。
  • **出力**: なし（本構造体は状態保持に使用されます）。

前提条件と不変条件

  • value は Datum 型として保持され、isnull が true の場合は value は無効とみなされます。

アルゴリズムまたは振る舞い

  • JSON_TABLE のプラン状態管理において、行生成のソースとなる値を一時的に保持します。
  • JsonTablePlanState 構造体の一部として組み込まれ、行の走査や値の抽出プロセスで参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonpath_exec.c 内の JSON_TABLE 実行ロジック。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。
  • **参照箇所**:
  • src/backend/utils/adt/jsonpath_exec.c:200 (JsonTablePlanState 内のメンバとして定義)
  • src/backend/utils/adt/jsonpath_exec.c:4461 (ポインタ経由でのアクセス)

関連データ構造

  • JsonTablePlanState: 本構造体をメンバとして保持し、JSON_TABLE の実行状態を管理します。

失敗モードと境界条件

  • isnull フラグの管理が不適切な場合、value に不正な Datum が含まれる可能性があります。

保守担当者向けメモ

  • 本構造体は JSON_TABLE 実行時の行ソース管理に特化しています。JSON_TABLE 関連のロジックを変更する際は、この構造体の Datum が指すメモリコンテキストの生存期間に注意してください。
