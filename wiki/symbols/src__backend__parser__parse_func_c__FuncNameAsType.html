FuncNameAsType

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_func.c:1880

目的

指定された名前（List *funcname）がシステムカタログ上の有効な型名であるかを判定し、該当する場合はその型OIDを返します。

シグネチャまたは形状

Oid FuncNameAsType(List *funcname)

入力と出力

  • **入力**: funcname (型名を表す名前リスト)
  • **出力**: 成功時は該当する型の Oid、失敗または型として不適格な場合は InvalidOid

前提条件と不変条件

  • temp_ok=false で LookupTypeNameExtended を呼び出すことで、SECURITY DEFINER 関数の安全性を保護する契約を遵守しています。
  • 戻り値が有効なOIDであるためには、対象が定義済み（typisdefined）であり、かつ複合型（typeTypeRelid が無効）ではない必要があります。

アルゴリズムまたは振る舞い

1. LookupTypeNameExtended を使用して、名前リストから型情報を検索します。 2. 検索結果が NULL の場合は InvalidOid を返します。 3. 取得した型情報が「定義済み」かつ「複合型ではない」ことを確認します。 4. 条件を満たす場合、typeTypeId を介して型OIDを取得します。 5. ReleaseSysCache を呼び出してシステムキャッシュを解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_func.c 内の解析処理（1486行目付近）から呼び出されます。
  • **呼び出し先**: LookupTypeNameExtended, makeTypeNameFromNameList, GETSTRUCT, typeTypeRelid, typeTypeId, ReleaseSysCache 等のシステムカタログ操作関数。

関連データ構造

  • List: 名前リストを保持する構造体。
  • Type: システムキャッシュ内の型情報を指すハンドル。
  • Form_pg_type: pg_type カタログの行構造体。

失敗モードと境界条件

  • 名前が型として存在しない場合、InvalidOid を返します。
  • 検索された型が未定義（typisdefined が偽）であるか、複合型である場合は InvalidOid を返します。

保守担当者向けメモ

  • 本関数は、関数呼び出し構文が実際には型キャストや型参照である可能性を排除するために使用されます。
  • temp_ok=false の指定は、セキュリティ上の制約を維持するために重要であるため、変更時は注意してください。
