IndexAttributeBitMapData

定義

  • 種別: struct
  • 定義箇所: src/include/access/itup.h:55

目的

インデックスタプル（IndexTuple）内の各属性が NULL 値であるかどうかを管理するためのビットマップ構造体です。

シグネチャまたは形状

typedef struct IndexAttributeBitMapData
{
	bits8		bits[(INDEX_MAX_KEYS + 8 - 1) / 8];
}			IndexAttributeBitMapData;

入力と出力

  • **入力**: INDEX_MAX_KEYS（インデックスの最大キー数）に基づき、必要なビット配列サイズが決定されます。
  • **出力**: インデックスタプルのヘッダー直後に配置される固定長のビットマップ領域を提供します。

前提条件と不変条件

  • IndexTupleData の直後に配置されることが想定されています。
  • ビットマップのサイズは属性数に関わらず INDEX_MAX_KEYS に基づく固定長であり、一定のメモリ領域を消費します。

アルゴリズムまたは振る舞い

  • インデックスタプルの t_info フラグで NULL ビットマップの存在が示されている場合、この構造体を使用して各属性の NULL 状態を判定します。
  • メモリレイアウト上、IndexTupleData の直後に配置され、その後に実際の属性値が MAXALIGN 境界で並びます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/access/itup.h 内のサイズ計算ロジック（IndexTupleSize 等）で参照されています。
  • src/include/access/spgist_private.h でサイズ計算時に参照されています。
  • **呼び出し先**:
  • 提供されたソース情報からは、特定の関数呼び出し関係は特定できません。

関連データ構造

  • IndexTupleData: インデックスタプルの基底構造体。
  • IndexAttributeBitMap: IndexAttributeBitMapData へのポインタ型。

失敗モードと境界条件

  • INDEX_MAX_KEYS を超える属性を扱うことはできません。
  • メモリ配置において MAXALIGN を考慮しないポインタ演算を行うと、データ破損やアライメント違反を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は固定長であるため、属性数が少ない場合でも一定のメモリを消費します。
  • IndexTuple の物理レイアウトを直接操作する際は、itup.h で定義されたマクロや関数を介してアクセスし、手動のポインタ演算を最小限に留めてください。
