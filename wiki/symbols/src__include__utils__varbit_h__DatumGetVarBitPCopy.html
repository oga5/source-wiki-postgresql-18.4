DatumGetVarBitPCopy

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/varbit.h:50

目的

Datum 形式のビット列データを detoast（展開）し、そのコピーをメモリ上に作成して VarBit 構造体へのポインタとして返します。

シグネチャまたは形状

static inline VarBit *DatumGetVarBitPCopy(Datum X)

入力と出力

  • **入力**: Datum X（ビット列データを含む Datum）
  • **出力**: VarBit *（展開・コピーされたビット列データへのポインタ）

前提条件と不変条件

  • 入力 Datum X は、varlena 構造体と互換性のあるビット列データである必要があります。
  • 戻り値のメモリ領域は、呼び出し元が適切に管理（必要に応じて解放）する必要があります。

アルゴリズムまたは振る舞い

1. PG_DETOAST_DATUM_COPY(X) マクロを呼び出します。 2. このマクロにより、入力が TOAST されている場合は展開され、メモリ上に新しいコピーが作成されます。 3. 作成されたメモリ領域の先頭アドレスを VarBit * 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/varbit.h:63 にて定義されるマクロ PG_GETARG_VARBIT_P_COPY(n) から直接参照されています。
  • **呼び出し先**:
  • PG_DETOAST_DATUM_COPY（fmgr.h で定義される PostgreSQL の関数マネージャインターフェース）。

関連データ構造

  • VarBit: ビット列を保持する構造体。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • メモリ不足時には PG_DETOAST_DATUM_COPY の内部実装に従い、エラーが発生する可能性があります。
  • 入力が有効なビット列データでない場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は varlena 構造体のコピーを生成するため、メモリリークを防ぐために戻り値のライフサイクルを呼び出し側で管理してください。
  • VarBit 構造体の内部フィールド（bit_len 等）にアクセスする際は、必ず varbit.h で提供されているマクロ（VARBITLEN 等）を使用してください。
