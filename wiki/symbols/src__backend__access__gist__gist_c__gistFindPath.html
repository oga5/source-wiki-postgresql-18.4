gistFindPath

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:913

目的

GiSTインデックス内の特定のブロック（child）を指す親ノードのパスを、ルートから幅優先探索で再構築する。

シグネチャまたは形状

GISTInsertStack *gistFindPath(Relation r, BlockNumber child, OffsetNumber *downlinkoffnum)

入力と出力

  • **入力**:
  • r: 対象のリレーション。
  • child: 探索対象のブロック番号。
  • downlinkoffnum: 見つかった親ノード内でのダウンリンクのオフセット番号を格納するポインタ。
  • **出力**:
  • 成功時: child を指す親ノードまでのパスを保持する GISTInsertStack 構造体の先頭ポインタ。
  • 失敗時: elog(ERROR) により処理を中断。

前提条件と不変条件

  • 探索はルートノード（GIST_ROOT_BLKNO）から開始される。
  • 内部ノードは削除されないものと仮定される。
  • 探索中にページ分割が発生した場合、GistPageGetNSN と rightlink を用いて右側のページを探索キューに追加し、整合性を維持する。

アルゴリズムまたは振る舞い

1. ルートノードをキューに入れ、幅優先探索を開始する。 2. 各ページを GIST_SHARE ロックで読み込み、リーフページであればそのパスを終了する。 3. 内部ページの場合、各タプルを走査し、t_tid が child と一致すればその親ノードとオフセットを返す。 4. 一致しない場合、そのタプルの指す子ノードをキューに追加する。 5. 探索中にページ分割が検出された場合（GistFollowRight が真、またはNSNの不整合）、右側のページをキューの優先位置に追加して探索を継続する。 6. 全てのページを走査しても child が見つからない場合はエラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c 内のコードから呼び出されている（詳細はソース内の gistFindPath 参照）。
  • **呼び出し先**: ReadBuffer, LockBuffer, gistcheckpage, BufferGetPage, GistPageIsLeaf, UnlockReleaseBuffer, PageGetMaxOffsetNumber, PageGetItemId, PageGetItem, ItemPointerGetBlockNumber 等のバッファ管理およびページ操作関数。

関連データ構造

  • GISTInsertStack: ツリーのパスを保持するスタック構造。
  • Relation: インデックスのリレーション情報。
  • Buffer: ページバッファ。
  • IndexTuple: インデックス内のタプル。

失敗モードと境界条件

  • **エラー**: 指定された child ブロックがインデックス内に存在しない場合、elog(ERROR) で停止する。
  • **並行処理**: 探索中にページ分割が発生した場合、右側のページを探索対象に含めることで整合性を確保している。

保守担当者向けメモ

  • 本関数はツリーの再構築を行うため、コストが高い処理である。
  • GistFollowRight が真の場合にエラーを投げる挙動は、分割中のツリー保護が期待通り機能していることを前提としている。
