ltree_gist

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltree.h:267

目的

PostgreSQL の ltree 拡張モジュールにおいて、GiST インデックスのリーフおよび非リーフノードで利用されるデータ格納形式を定義します。

シグネチャまたは形状

typedef struct
{
	int32		vl_len_;
	uint32		flag;
	char		data[FLEXIBLE_ARRAY_MEMBER];
} ltree_gist;

入力と出力

  • **入力**: 可変長データ（varlena）として PostgreSQL のメモリ上に配置されます。
  • **出力**: GiST インデックスの検索や構築プロセスにおいて、シグネチャ（ビットベクトル）や比較キーとして利用されます。

前提条件と不変条件

  • vl_len_ は PostgreSQL の可変長データヘッダーであり、直接操作してはなりません。
  • data フィールドは FLEXIBLE_ARRAY_MEMBER として定義されているため、構造体のサイズ計算には MAXALIGN を含む専用のマクロを使用する必要があります。
  • インデックス操作時は、LTREE_GET_SIGLEN() マクロを介してシグネチャ長を取得する必要があります。

アルゴリズムまたは振る舞い

  • GiST インデックスの構築および検索ロジックにおいて、インデックスキーのシリアライズおよびデシリアライズの単位として機能します。
  • ltree_gist.c および _ltree_gist.c 内の関数において、インデックスノード間の距離計算（hemdist）や、ノードの結合（unionkey）の対象として処理されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • contrib/ltree/_ltree_gist.c: インデックスのキー取得（GETENTRY）、距離計算（hemdist）、ノード結合（unionkey）、および PG_GETARG_POINTER を介した引数としての利用。
  • **呼び出し先**:
  • ltree_gist_alloc: インデックスノードのメモリ確保時に呼び出されます。

関連データ構造

  • LtreeGistOptions: インデックスのシグネチャ長を制御するために使用されます。
  • BITVECP: シグネチャのビットベクトル操作に関連します。
  • GISTENTRY: PostgreSQL の GiST インデックスインターフェースにおいて、キーを保持する構造体です。

失敗モードと境界条件

  • sizeof を直接使用して構造体サイズを計算すると、アライメント違反が発生します。
  • インデックスのシグネチャ長が動的に変更される可能性があるため、固定長と見なした処理は誤動作の原因となります。

保守担当者向けメモ

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリレイアウトの変更には細心の注意を払ってください。
  • 過去のディスクフォーマットとの互換性制約があるため、構造体のフィールド順序やサイズ変更は慎重に行う必要があります。
