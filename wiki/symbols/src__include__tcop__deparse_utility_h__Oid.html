Oid

定義

  • 種別: struct
  • 定義箇所: src/include/tcop/deparse_utility.h:61

目的

PostgreSQLのシステムカタログ内のオブジェクトを一意に識別するための識別子（Object Identifier）を保持します。本ファイル内では、主に ALTER TABLE などのユーティリティコマンドの実行対象や関連するカタログ情報を管理するために使用されます。

シグネチャまたは形状

struct {
    Oid objectId;
    Oid classId;
    List *subcmds;
} alterTable;

入力と出力

  • **入力**: ユーティリティコマンド実行時に、対象となるオブジェクトの objectId およびその所属するカタログの classId が設定されます。
  • **出力**: 保持された objectId および classId は、後続のイベントトリガーや監査ログ生成プロセスにおいて、対象オブジェクトを特定するために参照されます。

前提条件と不変条件

  • objectId および classId は、PostgreSQLのシステムカタログにおいて有効な値である必要があります。
  • alterTable 構造体の一部として使用される場合、type フィールドによる型判定を経てアクセスされることが前提となります。

アルゴリズムまたは振る舞い

本シンボル自体はデータ構造の定義であり、ロジックは持ちません。CollectedCommand 構造体の一部として、コマンド実行時のコンテキストを保持し、後続の deparse（再構築）処理のために情報を永続化する役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、特定の関数による直接的な呼び出し関係は特定できません。
  • **呼び出し先**: 提供されたソース情報からは、特定の関数を呼び出す関係は特定できません。
  • **注記**: contrib/amcheck 等の外部モジュールで Oid 型自体は広く利用されていますが、本ヘッダーファイル内の alterTable 構造体内の Oid フィールドに対する具体的な操作関数は本抜粋からは不明です。

関連データ構造

  • CollectedCommand: 本構造体を含む親のデータ構造。
  • List: subcmds フィールドで使用されるPostgreSQLのリスト構造。

失敗モードと境界条件

  • 無効な Oid（InvalidOid）が設定された場合、後続のオブジェクト参照処理でエラーが発生する可能性があります。
  • classId が不適切なカタログを指している場合、オブジェクトの特定に失敗します。

保守担当者向けメモ

  • 本シンボルは CollectedCommand 内の共用体の一部として定義されています。アクセスする際は、必ず親構造体の type フィールドを確認し、alterTable メンバが有効であることを保証してください。
  • Oid はPostgreSQL全体で多用される型ですが、本ファイルではDDLの追跡という文脈で利用されている点に注意してください。
