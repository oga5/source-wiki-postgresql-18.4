ExecEvalExpr

定義

  • 種別: function
  • 定義箇所: src/include/executor/executor.h:388

目的

クエリ実行エンジンにおいて、コンパイル済みの式状態（ExprState）を評価し、その結果を Datum として取得する。

シグネチャまたは形状

static inline Datum ExecEvalExpr(ExprState *state, ExprContext *econtext, bool *isNull)

入力と出力

  • **入力**:
  • state: 評価対象の式状態構造体（ExprState）。
  • econtext: 評価に使用するコンテキスト（ExprContext）。
  • isNull: 結果が NULL かどうかを格納するためのブール値ポインタ。
  • **出力**:
  • 評価結果を Datum 型で返す。

前提条件と不変条件

  • state（ExprState）は NULL であってはならない。
  • 呼び出し前に ExprContext が適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

ExprState 構造体内に保持されている関数ポインタ evalfunc を呼び出し、式を評価する。インライン関数として定義されており、実行時のオーバーヘッドを最小化する設計となっている。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/commands/copyfromparse.c
  • src/backend/commands/tablecmds.c
  • src/backend/commands/typecmds.c
  • src/backend/executor/execGrouping.c
  • **呼び出し先**:
  • state->evalfunc: ExprState 内の関数ポインタを介して実行される。具体的な実装は evalfunc が指す関数に依存する。

関連データ構造

  • ExprState: 評価対象の式と実行用関数を保持する。
  • ExprContext: 式評価に必要な実行時状態（タプルデータ等）を保持する。
  • Datum: 評価結果の汎用データ型。

失敗モードと境界条件

  • state が NULL の場合、未定義の動作（クラッシュ）を引き起こす可能性がある。
  • evalfunc が適切に設定されていない場合、実行時エラーが発生する。

保守担当者向けメモ

  • 本関数は static inline であり、頻繁に呼び出されるため、変更時はパフォーマンスへの影響を考慮すること。
  • ExprState の構築については ExecInitExpr を参照すること。
