QueryItem

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsrank.c:508

目的

全文検索における tsquery の構成要素（演算子または検索語）を表現するための構造体です。

シグネチャまたは形状

struct QueryItem {
    int32 type;      /* 項目タイプ (QI_VAL, QI_OPR 等) */
    /* その他、タイプに応じたフィールドが含まれる */
};

*注: ソース excerpt には構造体全体のフィールド定義は含まれていませんが、tsquery.c 等の参照から type フィールドの存在が確認できます。*

入力と出力

  • 本構造体は、tsquery の内部表現としてメモリ上に配置されます。
  • tsquery の解析および評価プロセスにおいて、各ノードの型判定や演算処理の入力として利用されます。

前提条件と不変条件

  • tsquery の実行エンジンは、本構造体の type フィールドを検証し、未知の型に対してはエラーを発生させる必要があります（tsquery.c:917）。

アルゴリズムまたは振る舞い

  • tsquery の木構造を走査する際、各ノードとして QueryItem が参照されます。
  • 演算子ノードの場合、子ノードへの再帰的なアクセスや、適合度計算（calc_rank 等）における論理演算の制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/tsearch/ts_parse.c
  • src/backend/tsearch/ts_selfuncs.c
  • src/backend/utils/adt/tsginidx.c
  • src/backend/utils/adt/tsquery.c
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、特定の関数を呼び出すものではありません。操作は tsquery.c 等の関数群によって行われます。

関連データ構造

  • TSQuery: QueryItem の配列を保持する上位のデータ構造。
  • QueryRepresentation: コンパイルされた文書表現内で QueryItem のポインタ配列を保持します。

失敗モードと境界条件

  • 不正な type 値を持つ QueryItem が渡された場合、elog(ERROR) により処理が中断されます。
  • 複雑なクエリ構造において、再帰的な走査を行う際にスタックオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は tsrank.c だけでなく、tsearch モジュール全体で共有される基盤的なデータ構造です。変更を加える際は、tsquery のバイナリ互換性および関連するインデックス（GIN/GiST）への影響を考慮してください。
