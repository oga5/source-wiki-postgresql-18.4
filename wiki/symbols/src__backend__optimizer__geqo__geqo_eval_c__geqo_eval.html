geqo_eval

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_eval.c:56

目的

遺伝的アルゴリズム（GEQO）において、指定された結合順序（ツアー）に基づき、クエリプランの総コスト（適応度）を評価する。

シグネチャまたは形状

Cost geqo_eval(PlannerInfo *root, Gene *tour, int num_gene)

入力と出力

  • **入力**:
  • root: プランナのグローバル状態。
  • tour: 結合順序を定義する遺伝子配列。
  • num_gene: 遺伝子配列の長さ。
  • **出力**:
  • 算出されたコスト（Cost）。有効なプランが構築できない場合は DBL_MAX を返す。

前提条件と不変条件

  • root->join_rel_level が NULL であること（Assertで検証）。
  • 評価中に root->join_rel_list および root->join_rel_hash が一時的に変更されるが、関数終了時には呼び出し前の状態に復元されること。

アルゴリズムまたは振る舞い

1. **メモリ管理**: 一時的なプラン構築用メモリコンテキストを作成し、現在のコンテキストを切り替える。 2. **状態の退避**: root 内の結合リレーションリストの長さとハッシュテーブルを保存し、ハッシュテーブルを一時的に NULL に設定する。 3. **プラン構築**: gimme_tree を呼び出し、ツアーに基づいた結合順序を生成する。 4. **コスト算出**: 構築された joinrel の cheapest_total_path から総コストを取得する。 5. **状態復元**: root->join_rel_list を元の長さに切り詰め、ハッシュテーブルを復元する。 6. **クリーンアップ**: 一時メモリコンテキストを削除し、元のメモリコンテキストへ戻す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_main.c: geqo_eval を呼び出し個体の適応度を評価。
  • src/backend/optimizer/geqo/geqo_pool.c: プール内の個体評価時に呼び出し。
  • **呼び出し先**:
  • gimme_tree: ツアーに基づいたプランツリー構築。
  • list_truncate: リストの復元。
  • MemoryContextSwitchTo, AllocSetContextCreate, MemoryContextDelete: メモリ管理。

関連データ構造

  • PlannerInfo: プランナのグローバル状態。
  • Gene: 遺伝子配列の要素。
  • RelOptInfo: 結合リレーション情報。
  • Path: コスト算出に使用されるパス情報。

失敗モードと境界条件

  • gimme_tree が有効なプランを返さない場合、fitness は DBL_MAX となる。
  • 評価中にエラーが発生した場合、一時メモリコンテキストはプランナの親コンテキストに紐付いているため、自動的に解放される。

保守担当者向けメモ

  • root->join_rel_list の切り詰めは、gimme_tree が追加したエントリがリストの末尾に追加されることを前提としている。
  • 現在、部分的な結果取得やパラメータ化されたパスの最適化はサポートされていない。
