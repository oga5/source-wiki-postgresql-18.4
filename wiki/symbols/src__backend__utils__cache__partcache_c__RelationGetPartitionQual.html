RelationGetPartitionQual

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/partcache.c:276

目的

指定されたリレーションがパーティションである場合に、そのパーティションが満たすべき制約式（qual）のリストを取得する。

シグネチャまたは形状

List *RelationGetPartitionQual(Relation rel)

入力と出力

  • **入力**: Relation rel （対象のリレーション）
  • **出力**: List * （パーティション制約を表す式のリスト。パーティションでない場合は NIL）

前提条件と不変条件

  • 入力された rel は有効なリレーション構造体であること。
  • パーティション制約の構築は、リレーションキャッシュ（relcache）の管理下で行われる。

アルゴリズムまたは振る舞い

1. rel->rd_rel->relispartition を確認し、対象がパーティションでない場合は直ちに NIL を返す。 2. パーティションである場合、generate_partition_qual(rel) を呼び出して制約式を生成・取得し、その結果を返す。 3. generate_partition_qual は再帰的に親テーブルの制約を辿り、現在のリレーションの属性番号に変換する処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/executor/execMain.c
  • src/backend/optimizer/util/plancat.c
  • **呼び出し先**:
  • generate_partition_qual （同ファイル内）

関連データ構造

  • Relation: リレーション情報を保持する構造体。
  • List: PostgreSQL のリスト構造（制約式の保持に使用）。

失敗モードと境界条件

  • **パーティションではない場合**: relispartition が偽であれば、処理を行わず NIL を返す。
  • **スタックオーバーフロー**: generate_partition_qual が再帰的に動作するため、非常に深いパーティション階層ではスタックオーバーフローのリスクがある（内部で check_stack_depth() による保護が行われている）。

保守担当者向けメモ

  • 本関数が返す制約式は、rd_partcheck にキャッシュされる設計となっている。
  • メモリ管理は CacheMemoryContext の子コンテキストで行われ、relcache エントリの破棄時に自動的に解放される。
