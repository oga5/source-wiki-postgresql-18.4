bpchar_larger

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varchar.c:946

目的

2つの bpchar (char(n)) 値を比較し、大きい方の値を返します。

シグネチャまたは形状

Datum bpchar_larger(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの BpChar 型のデータを受け取ります。
  • **出力**: Datum 型として、入力された2つの値のうち、比較結果が大きい方の値を返します。

前提条件と不変条件

  • 入力値は PG_GETARG_BPCHAR_PP マクロにより、必要に応じて TOAST 展開された状態で取得されます。
  • 比較には現在のセッションの照合順序（Collation）が使用されます。

アルゴリズムまたは振る舞い

1. bcTruelen を使用して、各入力値の末尾空白を除いた「真の長さ」を計算します。 2. varstr_cmp を呼び出し、計算した長さと現在の照合順序に基づいて2つの値を比較します。 3. 比較結果が 0 以上（第1引数が第2引数以上）であれば第1引数を、そうでなければ第2引数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの GREATEST 関数等から呼び出されることが想定されますが、ソースコード内での直接的な呼び出し元情報は提供されていません。
  • **呼び出し先**:
  • bcTruelen: 真の長さの計算。
  • varstr_cmp: ロケールを考慮した文字列比較。
  • PG_GET_COLLATION: 現在の照合順序の取得。
  • PG_RETURN_BPCHAR_P: 結果の返却。

関連データ構造

  • BpChar: char(n) 型の内部表現。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • 入力が NULL の場合、PG_GETARG_BPCHAR_PP の挙動に従い、呼び出し元で適切に処理される必要があります。
  • 照合順序の設定によっては、比較結果が予期しない順序になる可能性があります。

保守担当者向けメモ

  • 本関数は bpchar_smaller と対になる実装です。
  • 末尾空白を無視する仕様は SQL 標準に準拠しており、bcTruelen による事前計算が比較の正確性を担保しています。
