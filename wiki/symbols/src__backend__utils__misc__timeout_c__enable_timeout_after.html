enable_timeout_after

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:559

目的

指定されたミリ秒後に発生する単発のタイムアウトイベントを登録し、タイマーをスケジュールする。

シグネチャまたは形状

void enable_timeout_after(TimeoutId id, int delay_ms)

入力と出力

  • id: タイムアウトの種類を識別する TimeoutId。
  • delay_ms: タイムアウトまでの待機時間（ミリ秒単位）。
  • 戻り値: なし。

前提条件と不変条件

  • 呼び出し時に disable_alarm() が実行され、シグナル割り込みが一時的に無効化される。
  • 内部で GetCurrentTimestamp() を使用して現在時刻を取得し、期限を算出する。

アルゴリズムまたは振る舞い

1. disable_alarm() を呼び出し、シグナルハンドラによる競合を防止する。 2. GetCurrentTimestamp() で現在時刻を取得する。 3. TimestampTzPlusMilliseconds() を用いて、現在時刻に delay_ms を加算した期限時刻を算出する。 4. enable_timeout() を呼び出し、指定された id と期限を管理対象として登録する。 5. schedule_alarm() を呼び出し、登録された全タイムアウトの中で最も早い期限に合わせて setitimer を再設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c (トランザクションタイムアウト)
  • src/backend/storage/lmgr/proc.c (デッドロックタイムアウト)
  • src/backend/tcop/backend_startup.c (スタートアップパケットタイムアウト)
  • src/backend/tcop/postgres.c (クライアント接続チェック、アイドルセッション、ステートメントタイムアウト等)
  • **呼び出し先**:
  • disable_alarm()
  • GetCurrentTimestamp()
  • TimestampTzPlusMilliseconds()
  • enable_timeout()
  • schedule_alarm()

関連データ構造

  • TimeoutId: タイムアウトを識別する列挙型。
  • active_timeouts: 有効なタイムアウトを保持する内部配列。

失敗モードと境界条件

  • delay_ms が負の値や極端に大きい値である場合の挙動は、基盤となる TimestampTz 計算関数に依存する。
  • シグナルハンドラ実行中の割り込み禁止状態（HOLD_INTERRUPTS）との整合性に注意が必要。

保守担当者向けメモ

  • 本関数は単発のタイムアウト用である。繰り返し発生するタイムアウトには enable_timeout_every を使用すること。
  • schedule_alarm はシステムコールを伴うため、頻繁な呼び出しはパフォーマンスに影響を与える可能性がある。
