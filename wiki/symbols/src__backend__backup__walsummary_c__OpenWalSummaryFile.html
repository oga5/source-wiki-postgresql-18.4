OpenWalSummaryFile

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummary.c:204

目的

指定された WalSummaryFile 構造体の情報に基づき、対応するWAL要約ファイルを読み取り専用モードで開きます。

シグネチャまたは形状

File OpenWalSummaryFile(WalSummaryFile *ws, bool missing_ok)

入力と出力

  • **入力**:
  • ws: 開く対象の要約ファイルを特定するための WalSummaryFile 構造体へのポインタ。tli、start_lsn、end_lsn を使用します。
  • missing_ok: true の場合、ファイルが存在しなくてもエラーを発生させず、無効なファイル記述子を返します。
  • **出力**:
  • 成功時: 開かれたファイルの File 記述子。
  • 失敗時: missing_ok が false の場合、ereport(ERROR) により処理を中断します。

前提条件と不変条件

  • ws が指す構造体には、有効な tli、start_lsn、end_lsn が格納されている必要があります。
  • ファイルパスは pg_wal/summaries/ ディレクトリ配下の命名規則（TLI + StartLSN + EndLSN）に従って生成されます。

アルゴリズムまたは振る舞い

1. ws の各フィールドを基に、snprintf を用いてファイルパス文字列を構築します。 2. PathNameOpenFile を呼び出し、O_RDONLY モードでファイルを開きます。 3. ファイルオープンに失敗した場合、missing_ok が false かつエラー原因が「ファイルが存在しないこと（ENOENT）」以外であれば、ereport でエラーを報告します。 4. 正常に開けた場合、または missing_ok が true でファイルが存在しない場合は、その結果（ファイル記述子または負の値）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c:582
  • src/backend/backup/walsummaryfuncs.c:102
  • **呼び出し先**:
  • PathNameOpenFile (PostgreSQLのファイルI/O抽象化レイヤー)
  • ereport (エラー報告用マクロ)

関連データ構造

  • WalSummaryFile: 要約ファイルのメタデータ（タイムライン、LSN範囲）を保持する構造体。
  • File: PostgreSQL内部のファイル記述子型。

失敗モードと境界条件

  • **ファイル不在**: missing_ok が false の場合、ファイルが存在しないと ERROR レベルでログ出力され、トランザクションがアボートされます。
  • **権限エラー等**: ファイルが存在しても読み取り権限がない場合などは、missing_ok の値に関わらずエラーとして報告されます。

保守担当者向けメモ

  • ファイルパスの生成には XLOGDIR マクロが使用されており、pg_wal ディレクトリ配下の構成に依存しています。
  • 本関数は読み取り専用でのオープンを前提としています。書き込みが必要な場合は別の手段を検討してください。
