win32_check_directory_write_permissions

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/exec.c:287

目的

Windows環境において、現在の作業ディレクトリに対する書き込み権限があるかを検証します。

シグネチャまたは形状

static int win32_check_directory_write_permissions(void)

入力と出力

  • **入力**: なし
  • **出力**:
  • 0: 書き込み権限が確認できた場合（成功）。
  • -1: 書き込み権限がない、またはファイルの作成に失敗した場合。

前提条件と不変条件

  • GLOBALS_DUMP_FILE で定義されるファイル名が、カレントディレクトリで作成可能である必要があります。
  • この関数はWindows環境での実行を想定しています。

アルゴリズムまたは振る舞い

1. GLOBALS_DUMP_FILE を O_RDWR | O_CREAT モードで開き、読み書き可能なファイルを作成します。 2. ファイルの作成に成功した場合、直ちに close(fd) でファイルを閉じます。 3. 作成したファイルを unlink() で削除します。 4. unlink() の戻り値を関数の結果として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/exec.c 内のロジックから呼び出されます。
  • **呼び出し先**:
  • open() (標準ライブラリ)
  • close() (標準ライブラリ)
  • unlink() (標準ライブラリ)

関連データ構造

  • GLOBALS_DUMP_FILE: ファイル作成に使用される定数（pg_upgrade.h で定義）。

失敗モードと境界条件

  • open() が失敗した場合（権限不足やディスクフルなど）、-1 を返します。
  • check モードであっても、実際に一時ファイルを作成・削除する副作用が発生します。

保守担当者向けメモ

  • 本関数はWindows特有の権限チェックを目的としています。
  • unlink() の戻り値をそのまま返しているため、ファイル作成には成功したが削除に失敗した場合も考慮した設計となっています。
