mcelem_array_contain_overlap_selec

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_selfuncs.c:520

目的

配列の包含演算子（@>）および重複演算子（&&）に対する選択率（Selectivity）を、最頻要素（MCELEM）統計情報を用いて推定します。

シグネチャまたは形状

static Selectivity mcelem_array_contain_overlap_selec(
    Datum *mcelem, 
    int nmcelem,
    float4 *numbers, 
    int nnumbers,
    Datum *array_data, 
    int nitems,
    Oid operator, 
    TypeCacheEntry *typentry
)

入力と出力

  • **入力**:
  • mcelem: 最頻要素の配列。
  • nmcelem: 最頻要素の数。
  • numbers: 各最頻要素の出現頻度（末尾3要素は最小頻度、最大頻度、NULL頻度）。
  • array_data: 比較対象の定数配列。
  • nitems: 定数配列の要素数。
  • operator: 演算子OID（OID_ARRAY_CONTAINS_OP か否か）。
  • typentry: 型情報のキャッシュ。
  • **出力**:
  • Selectivity（0.0〜1.0の範囲）: 推定された選択率。

前提条件と不変条件

  • nnumbers が nmcelem + 3 でない場合、統計情報は無視され、デフォルト値が使用されます。
  • array_data 内の重複要素は無視されます。
  • 各要素の出現は独立であると仮定して計算されます。
  • 中間結果は CLAMP_PROBABILITY マクロにより、0.0〜1.0の範囲に収められます。

アルゴリズムまたは振る舞い

1. **初期化**: 演算子が @> なら初期値を 1.0、&& なら 0.0 に設定します。 2. **探索戦略**: nitems と nmcelem の規模に基づき、バイナリサーチまたは線形スキャンを選択します。 3. **反復処理**: array_data の各要素に対し、mcelem 内の対応する頻度を検索します。

  • 統計情報が見つかった場合、その頻度を使用します。
  • 見つからない場合、minfreq / 2 と DEFAULT_CONTAIN_SEL の小さい方を採用します。

    • 4. **集計**:

  • @> の場合: 各要素の選択率を積算します。
  • && の場合: selec = selec + elem_selec - selec * elem_selec の式で確率を合成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_selfuncs.c 内の calc_arraycontsel 等から呼び出されます。
  • **呼び出し先**: element_compare（要素比較）、find_next_mcelem（バイナリサーチ用）、floor_log2（探索戦略決定用）を呼び出します。

関連データ構造

  • TypeCacheEntry: 型固有の比較関数を取得するために使用。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • 統計情報（numbers）が不完全な場合、DEFAULT_CONTAIN_SEL を用いたフォールバックが行われます。
  • array_data が空の場合、初期値（@> なら 1.0、&& なら 0.0）がそのまま返されます。

保守担当者向けメモ

  • 本関数は「各要素の出現は独立である」という仮定に基づいています。複雑な相関があるデータ分布では推定誤差が生じる可能性があります。
  • 統計情報が不足している場合、ANALYZE による統計情報の更新が推奨されます。
