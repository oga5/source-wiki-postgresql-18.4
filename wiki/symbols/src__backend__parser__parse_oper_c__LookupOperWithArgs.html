LookupOperWithArgs

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_oper.c:132

目的

ObjectWithArgs 構造体で指定された演算子名と引数型リストに基づき、システムカタログから対応する演算子の OID を検索・特定します。

シグネチャまたは形状

Oid LookupOperWithArgs(ObjectWithArgs *oper, bool noError)

入力と出力

  • **入力**:
  • oper: 演算子名と引数型リストを含む ObjectWithArgs 構造体へのポインタ。
  • noError: 演算子が見つからない場合にエラーを投げるか（false）、InvalidOid を返すか（true）を制御するフラグ。
  • **出力**:
  • 成功時は該当する演算子の Oid。見つからない場合は noError に応じてエラー発生または InvalidOid を返却。

前提条件と不変条件

  • oper->objargs は必ず2つの要素（左辺と右辺の型）を持つ必要があります（Assert で検証済み）。

アルゴリズムまたは振る舞い

1. oper->objargs から左辺および右辺の型定義（TypeName）を取得します。 2. LookupTypeNameOid を使用して、各型名に対応する OID を解決します。型が指定されていない（NULL）場合は InvalidOid を割り当てます。 3. 解決した型 OID と演算子名を引数として LookupOperName を呼び出し、最終的な演算子 OID を取得して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/opclasscmds.c
  • src/backend/commands/operatorcmds.c
  • **呼び出し先**:
  • LookupTypeNameOid
  • LookupOperName

関連データ構造

  • ObjectWithArgs: 演算子名と引数型のリストを保持する構造体。
  • TypeName: SQL 内の型指定を表現する構造体。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • oper->objargs の要素数が2でない場合、アサーション違反により異常終了します。
  • 指定された型名が存在しない場合、LookupTypeNameOid の挙動に従い、noError が false であればエラーが発生します。
  • 演算子名と型シグネチャが一致しない場合、LookupOperName の挙動に従い、noError が false であればエラーが発生します。

保守担当者向けメモ

  • 本関数は主に DDL コマンドやオブジェクトアドレス解決の文脈で使用されます。
  • LookupOperName への委譲を行っているため、演算子解決のコアロジックは LookupOperName に依存しています。
