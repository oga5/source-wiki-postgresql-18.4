ValidateSlotSyncParams

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/slotsync.c:1084

目的

論理レプリケーションスロットの同期機能を開始するために必要なシステム設定（GUC）が適切に構成されているかを検証します。

シグネチャまたは形状

bool ValidateSlotSyncParams(int elevel)

入力と出力

  • **入力**: elevel（ereport で使用するエラーレベル）
  • **出力**: 設定がすべて有効であれば true、一つでも不備があれば false を返します。

前提条件と不変条件

以下のすべての条件が満たされている必要があります。 1. wal_level が logical 以上であること。 2. primary_slot_name が設定されていること。 3. hot_standby_feedback が有効であること。 4. primary_conninfo が設定されていること。

アルゴリズムまたは振る舞い

1. wal_level を確認し、論理レプリケーションに必要なレベルに達していない場合はエラーを報告して false を返します。 2. PrimarySlotName が空または未設定の場合、エラーを報告して false を返します。 3. hot_standby_feedback が無効な場合、エラーを報告して false を返します。 4. PrimaryConnInfo が空または未設定の場合、エラーを報告して false を返します。 5. すべてのチェックを通過した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c
  • src/backend/replication/slotfuncs.c
  • **呼び出し先**:
  • ereport（エラー報告用マクロ）

関連データ構造

  • PrimarySlotName (GUC)
  • PrimaryConnInfo (GUC)
  • wal_level (GUC)
  • hot_standby_feedback (GUC)

失敗モードと境界条件

  • 各チェック項目で不備がある場合、指定された elevel に応じてエラーが報告されます。
  • elevel が ERROR の場合、関数は戻り値を返す前に処理を中断します。

保守担当者向けメモ

  • 本関数は、スロット同期ワーカーの起動前や、SQL 関数 pg_sync_replication_slots() の実行前に呼び出されるガードレールとして機能します。
  • 新たな同期要件が追加された場合は、この検証ロジックにも条件を追加してください。
