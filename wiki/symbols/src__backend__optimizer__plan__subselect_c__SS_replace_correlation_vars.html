SS_replace_correlation_vars

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/subselect.c:1970

目的

式ツリー内の相関変数（親クエリの変数）を探索し、それらを適切な Param ノードに置換することで、サブクエリが実行時に親クエリの値を参照できるようにします。

シグネチャまたは形状

Node *SS_replace_correlation_vars(PlannerInfo *root, Node *expr)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo 構造体。
  • expr: 相関変数の置換対象となる式ツリー。
  • **出力**:
  • 置換処理が適用された新しい式ツリー（Node *）。

前提条件と不変条件

  • 本関数は、サブクエリの計画化プロセスにおいて、相関変数が解決可能な状態にある必要があります。
  • 内部的に replace_correlation_vars_mutator を呼び出すため、ツリーウォークのための追加のセットアップは不要です。

アルゴリズムまたは振る舞い

1. 入力された式ツリー expr に対して replace_correlation_vars_mutator を再帰的に適用します。 2. ツリー内の各ノードを走査し、親クエリのスコープに属する変数を特定します。 3. 特定された変数を、実行時に値を受け渡すための Param ノードに変換・置換します。 4. 変換後の式ツリーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c:1385, 1390
  • **呼び出し先**:
  • replace_correlation_vars_mutator（内部的な再帰処理）

関連データ構造

  • PlannerInfo: クエリの計画生成に必要な情報（パラメータ情報など）を保持。
  • Node: PostgreSQLの式ツリーを構成する基本構造。
  • Param: 相関変数の置換先となるノード。

失敗モードと境界条件

  • 供給されたソース情報には、特定の失敗モードや境界条件に関する明示的な記述はありません。

保守担当者向けメモ

  • 本関数は SS_process_sublinks の実行前、あるいは特定の計画化フェーズの適切なタイミングで呼び出す必要があります。
  • 関連するコメントによれば、本処理はサブクエリの展開後に実行されることが前提となっています。
  • setrefs.c で行われるパラメータ参照の解決と密接に関連しているため、変更時はパラメータの整合性に注意してください。
