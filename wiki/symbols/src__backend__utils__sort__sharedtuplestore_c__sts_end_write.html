sts_end_write

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sharedtuplestore.c:212

目的

共有タプルストアへの書き込み処理を終了し、関連するリソースを解放およびクリーンアップします。

シグネチャまたは形状

void sts_end_write(SharedTuplestoreAccessor *accessor)

入力と出力

  • **入力**: accessor (SharedTuplestoreAccessor *) - 操作対象の共有タプルストアへのアクセス用構造体。
  • **出力**: なし（副作用として状態を更新）。

前提条件と不変条件

  • accessor が有効であること。
  • 書き込みが進行中（accessor->write_file != NULL）である場合にのみ、リソース解放処理が実行されます。

アルゴリズムまたは振る舞い

1. accessor->write_file が NULL でないことを確認します。 2. sts_flush_chunk(accessor) を呼び出し、バッファに残っているデータをフラッシュします。 3. BufFileClose(accessor->write_file) を呼び出し、一時ファイルを閉じます。 4. pfree(accessor->write_chunk) により、書き込み用チャンクメモリを解放します。 5. accessor 内のポインタを NULL にリセットします。 6. 共有状態の participants 配列において、該当する参加者の writing フラグを false に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHash.c (複数の箇所でハッシュテーブルのバッチ処理終了時に呼び出し)
  • src/backend/executor/nodeHashjoin.c
  • **呼び出し先**:
  • sts_flush_chunk
  • BufFileClose
  • pfree

関連データ構造

  • SharedTuplestoreAccessor: 書き込み状態を保持するローカル構造体。
  • SharedTuplestore: 共有メモリ上の制御構造体。
  • BufFile: 一時ファイル操作用インターフェース。

失敗モードと境界条件

  • accessor->write_file が既に NULL の場合、本関数は何も行わず安全に終了します。

保守担当者向けメモ

  • 本関数は書き込み終了時のクリーンアップを担うため、並列スキャンを開始する前や、処理の完了時に必ず呼び出される必要があります。
  • writing フラグの更新は、共有メモリ上の状態を正しく反映させるために重要です。
