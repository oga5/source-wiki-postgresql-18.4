library_name_compare

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/function.c:28

目的

pg_upgrade において、共有ライブラリのリストをソートするための比較関数です。ライブラリのロード順序を安定させ、依存関係の解決を確実にするために使用されます。

シグネチャまたは形状

static int library_name_compare(const void *p1, const void *p2)

入力と出力

  • **入力**: LibraryInfo 構造体へのポインタ p1 および p2。
  • **出力**: 比較結果を示す整数値。
  • p1 が p2 より小さい場合は負の値。
  • p1 が p2 より大きい場合は正の値。
  • 等しい場合は 0。

前提条件と不変条件

  • 入力されるポインタ p1 および p2 は、LibraryInfo 構造体としてキャスト可能である必要があります。
  • LibraryInfo 構造体には name (文字列) および dbnum (整数) フィールドが存在することが前提です。

アルゴリズムまたは振る舞い

以下の優先順位で比較を行います。

1. **名前の長さ**: strlen を使用してライブラリ名の長さを比較し、短いものを優先します。 2. **辞書順**: 長さが同じ場合、strcmp を使用してライブラリ名を辞書順で比較します。 3. **データベース番号**: 名前も同一である場合、dbnum を使用して数値比較を行い、順序を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/function.c 内の check_loadable_libraries 関数から qsort の引数として呼び出されます。
  • **呼び出し先**: strlen、strcmp、pg_cmp_size、pg_cmp_s32 を呼び出します。

関連データ構造

  • LibraryInfo: ライブラリ名とデータベース番号を保持する構造体（pg_upgrade.h で定義）。

失敗モードと境界条件

  • p1 または p2 が NULL の場合、あるいは LibraryInfo 構造体として正しくキャストできないメモリ領域を指している場合は未定義の動作となります。

保守担当者向けメモ

  • 本関数によるソート順序は、ライブラリ間の依存関係（例: hstore と hstore_plpython）を考慮して設計されています。順序の変更はアップグレード時のライブラリロード失敗を招く可能性があるため、慎重に行ってください。
