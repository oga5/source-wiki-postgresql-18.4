r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_serbian.c:5064

目的

セルビア語のステミング処理において、語幹抽出の境界（R1領域）を特定し、ステミング環境構造体 SN_env の整数配列 I に格納する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（ステミング環境へのポインタ）
  • 出力: int（常に 1 を返す）
  • 副作用: z->I[0] および z->I[1] の値を更新する。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • z->p に処理対象の文字列が格納されていること。

アルゴリズムまたは振る舞い

1. **R1の初期化**: z->I[1] を 1 に設定し、特定の文字グループ（g_sa）の出現に基づいて調整する。 2. **領域の計算**:

  • 母音グループ（g_v）の出現位置を探索し、z->I[0] に記録する。
  • 文字列内の特定の文字（'r'）の出現位置や、文字グループ（g_rg）との照合を行い、語幹の境界を絞り込む。

    • 3. **状態の復元**: 探索中に使用した一時的なインデックス（c1, c2, c3, c4）を用いて、処理終了時にカーソル位置を適切に復元する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内には明示的な呼び出し元は記載されていない。
  • 呼び出し先: out_grouping_U, in_grouping_U, skip_utf8（いずれもSnowball共通ユーティリティ）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。I 配列（整数）、c（カーソル）、l（文字列長）、p（文字列データ）を含む。

失敗モードと境界条件

  • 文字列が空、または探索対象のパターンが見つからない場合、各 lab ブロックへジャンプし、デフォルト値または直前の状態を維持して処理を継続する。
  • インデックスが文字列の境界（z->l）を超えないよう、skip_utf8 等で保護されている。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接Cファイルを編集せず、対応する .sbl ソースファイルを修正すること。
  • z->I 配列のインデックス（0, 1）は、Snowball言語の領域定義に対応している。デバッグ時は z->I の値が期待する境界位置と一致しているか確認すること。
