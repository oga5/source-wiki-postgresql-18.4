DirectoryMethodFile

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/walmethods.c:82

目的

ディレクトリ形式で WAL（Write-Ahead Log）を保存する際の、個別のファイルハンドルおよび圧縮状態を管理するための構造体です。

シグネチャまたは形状

typedef struct DirectoryMethodFile
{
    Walfile     base;
    int         fd;
    char       *fullpath;
    char       *temp_suffix;
#ifdef HAVE_LIBZ
    gzFile      gzfp;
#endif
#ifdef USE_LZ4
    LZ4F_compressionContext_t ctx;
    size_t      lz4bufsize;
    void       *lz4buf;
#endif
} DirectoryMethodFile;

入力と出力

  • **入力**: Walfile 基底構造体を通じて初期化され、ファイルパスや圧縮設定を受け取ります。
  • **出力**: ファイル記述子（fd）および圧縮ライブラリ（zlib/LZ4）のコンテキストを保持し、後続の書き込み操作に提供します。

前提条件と不変条件

  • base メンバが構造体の先頭に配置されており、Walfile 型へのキャストが可能です。
  • 圧縮が有効な場合、gzfp または lz4buf 等の関連リソースが適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • ディレクトリ保存メソッドにおけるファイル操作の抽象化レイヤーとして機能します。
  • 圧縮が有効な場合、書き込みデータは内部バッファ（lz4buf）または gzFile を経由して処理されます。
  • temp_suffix を使用して、書き込み中のファイルに一時的な接尾辞を付与し、完了後にリネームする処理をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/walmethods.c 内のディレクトリメソッド関連関数（dir_open_for_write 等）からインスタンス化および参照されます。
  • **呼び出し先**: OS のファイル操作関数（fsync 等）や、圧縮ライブラリ（gzflush 等）を呼び出します。

関連データ構造

  • Walfile: 本構造体の基底となる抽象構造体。
  • WalWriteMethod: ディレクトリメソッド全体を管理する上位構造体。

失敗モードと境界条件

  • 圧縮ライブラリが未定義（HAVE_LIBZ や USE_LZ4 が無効）の場合、対応するメンバは構造体に含まれません。
  • ファイルオープン失敗やディスクフル等のエラー発生時は、WalWriteMethod 側のエラーハンドリング機構（lasterrstring 等）を通じて報告されます。

保守担当者向けメモ

  • 本構造体は walmethods.c 内のディレクトリメソッド専用です。tar メソッドについては TarMethodFile を参照してください。
  • 圧縮ライブラリの追加や変更を行う際は、条件付きコンパイル（#ifdef）の範囲に注意してください。
