sepgsql_database_post_create

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/database.c:32

目的

新規作成されたデータベースに対して、SELinuxのセキュリティラベルを計算・適用し、必要な権限チェックを行う。

シグネチャまたは形状

void sepgsql_database_post_create(Oid databaseId, const char *dtemplate)

入力と出力

  • **入力**:
  • databaseId: 新規作成されたデータベースのOID。
  • dtemplate: テンプレートデータベース名（NULLの場合は "template1" が使用される）。
  • **出力**: なし（副作用としてカタログへのセキュリティラベル永続化を行う）。

前提条件と不変条件

  • データベース作成処理の直後に呼び出されることを想定している。
  • pg_database カタログに該当するOIDのタプルが存在する必要がある。

アルゴリズムまたは振る舞い

1. **テンプレートDBのラベル取得**: dtemplate を基にソースデータベースのセキュリティラベルを取得する。 2. **ソースDBの権限確認**: ソースデータベースに対する db_database:{getattr} 権限を sepgsql_avc_check_perms_label で検証する。 3. **新規DBのラベル計算**: クライアントのセキュリティコンテキストとテンプレートDBのラベルを基に、sepgsql_compute_create を用いて新規データベースのデフォルトラベルを生成する。 4. **作成権限の確認**: 新規データベース名に対して db_database:{create} 権限を検証する。 5. **ラベルの永続化**: SetSecurityLabel を呼び出し、計算されたラベルをシステムカタログに保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:111（フックポイント経由）。
  • **呼び出し先**:
  • get_database_oid, sepgsql_get_label, sepgsql_avc_check_perms_label
  • table_open, systable_beginscan, systable_getnext, systable_endscan, table_close
  • sepgsql_compute_create, sepgsql_get_client_label
  • SetSecurityLabel

関連データ構造

  • ObjectAddress: データベースオブジェクトの識別に使用。
  • Form_pg_database: pg_database カタログのタプル構造。
  • StringInfoData: 監査ログ用の文字列バッファ。

失敗モードと境界条件

  • get_database_oid がテンプレートDBを見つけられない場合、エラーが発生する可能性がある。
  • systable_getnext で新規作成したはずのデータベースタプルが見つからない場合、elog(ERROR) により処理が中断される。
  • SELinuxポリシーにより権限が拒否された場合、sepgsql_avc_check_perms_label 内でエラーが発生する。

保守担当者向けメモ

  • テンプレートDB名が指定されない場合のデフォルト値は "template1" に固定されている。
  • sepgsql_compute_create は、将来的にオブジェクト名を用いた特殊なラベル処理に対応する可能性がある。
  • 処理の最後に pfree を行い、動的に確保したラベル文字列のメモリを解放している。
