truncate_target_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:211

目的

ターゲットデータディレクトリ内の指定されたファイルを、指定されたサイズに切り詰める（truncate）。

シグネチャまたは形状

void truncate_target_file(const char *path, off_t newsize)

入力と出力

  • **入力**:
  • path: ターゲットディレクトリからの相対パス。
  • newsize: 切り詰め後のファイルサイズ（off_t）。
  • **出力**: なし（成功時は副作用としてファイルシステムを更新、失敗時は pg_fatal によりプロセスを終了）。

前提条件と不変条件

  • path は path_is_safe_for_extraction によって安全性が検証されなければならない。
  • datadir_target がグローバル変数として定義されている必要がある。
  • dry_run が有効な場合、ファイルシステムへの変更は行われない。

アルゴリズムまたは振る舞い

1. path_is_safe_for_extraction(path) を呼び出し、パスが安全でない場合はエラーで終了する。 2. dry_run フラグを確認し、真であれば何もせず終了する。 3. datadir_target と path を結合し、対象ファイルのフルパスを生成する。 4. open() を用いて書き込み専用モード（O_WRONLY）でファイルを開く。失敗時はエラーで終了する。 5. ftruncate() を呼び出し、ファイルを newsize に切り詰める。失敗時はエラーで終了する。 6. close() を呼び出し、ファイル記述子を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c:605
  • **呼び出し先**:
  • path_is_safe_for_extraction
  • open
  • ftruncate
  • close
  • pg_fatal

関連データ構造

  • off_t: ファイルサイズを表す型。
  • MAXPGPATH: パス文字列の最大長を定義する定数。

失敗モードと境界条件

  • **パスの安全性**: path_is_safe_for_extraction が偽を返す場合、pg_fatal により異常終了する。
  • **ファイルオープン失敗**: 指定されたパスが存在しない、または権限がない場合、pg_fatal により異常終了する。
  • **切り詰め失敗**: ftruncate が失敗した場合（ディスク容量不足やI/Oエラー等）、pg_fatal により異常終了する。

保守担当者向けメモ

  • 本関数は pg_rewind のファイル同期プロセスの一部であり、filemap で決定された変更を適用するために使用されます。
  • dry_run モードのサポートは必須であり、新しいロジックを追加する際は必ずチェックを維持してください。
