PgBenchExprType

定義

  • 種別: enum
  • 定義箇所: src/bin/pgbench/pgbench.h:57

目的

pgbench の式評価エンジンにおいて、抽象構文木（AST）を構成する各ノードの種類を識別するために使用されます。

シグネチャまたは形状

typedef enum PgBenchExprType
{
	ENODE_CONSTANT,
	ENODE_VARIABLE,
	ENODE_FUNCTION,
} PgBenchExprType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: PgBenchExpr 構造体内の etype フィールドとして保持され、評価時のノード種別判定に使用されます。

前提条件と不変条件

  • PgBenchExpr 構造体において、この列挙値はノードの性質を決定する不変の識別子として機能します。

アルゴリズムまたは振る舞い

  • ENODE_CONSTANT: 定数ノードであることを示します。
  • ENODE_VARIABLE: 変数参照ノードであることを示します。
  • ENODE_FUNCTION: 関数呼び出しノードであることを示します。
  • 評価エンジンは、この列挙値に基づいて switch 文等でノードごとの処理を分岐させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pgbench/pgbench.h 内の PgBenchExpr 構造体定義（112行目）で型として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • PgBenchExpr: 本列挙型を保持する AST ノード構造体。

失敗モードと境界条件

  • 定義外の値を etype に設定した場合、評価エンジン側で未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しいノードタイプ（例: 演算子や制御構文）を追加する場合は、本列挙型に要素を追加し、pgbench.c 側の評価ロジックを更新する必要があります。
