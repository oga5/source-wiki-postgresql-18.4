sort_pool

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_pool.c:134

目的

遺伝的アルゴリズムにおける個体群（Pool）内の染色体（Chromosome）を、評価値（コスト）に基づいて昇順（優秀な順）に並べ替えます。

シグネチャまたは形状

void sort_pool(PlannerInfo *root, Pool *pool)

入力と出力

  • **入力**:
  • root: クエリ最適化のコンテキスト情報（PlannerInfo）。
  • pool: ソート対象の個体群を保持する Pool 構造体へのポインタ。
  • **出力**: なし（pool->data が直接ソートされます）。

前提条件と不変条件

  • pool および pool->data が有効なメモリ領域を指していること。
  • pool->size が pool->data に格納されている実際の個体数と一致していること。

アルゴリズムまたは振る舞い

標準ライブラリの qsort を使用して、pool->data 配列内の Chromosome 要素を並べ替えます。比較には compare 関数が使用されます。評価値が小さいほど「優秀」と見なされるため、コストの低い順に整列されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_main.c:122
  • **呼び出し先**: qsort（標準ライブラリ）、compare（同ファイル内の比較関数）

関連データ構造

  • Pool: 個体群を管理する構造体。
  • Chromosome: 個体情報を保持する構造体。

失敗モードと境界条件

  • pool->size が 0 または 1 の場合、qsort は何もしません。
  • pool->data が NULL の場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は geqo_main.c から一度だけ呼び出されることを想定しています。
  • 評価値が小さいほど優秀と判断されるため、compare 関数の実装を変更する際はソート順序に注意してください。
