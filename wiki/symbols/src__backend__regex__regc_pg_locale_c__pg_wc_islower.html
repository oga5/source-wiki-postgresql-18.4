pg_wc_islower

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_pg_locale.c:415

目的

与えられた文字（pg_wchar）が小文字であるかどうかを、現在のロケール戦略に基づいて判定します。

シグネチャまたは形状

int pg_wc_islower(pg_wchar c);

入力と出力

  • **入力**: pg_wchar c（判定対象の文字コード）
  • **出力**: int（小文字であれば非ゼロ、そうでなければ 0）

前提条件と不変条件

  • pg_regex_strategy が適切に初期化されている必要があります。
  • pg_wchar は符号なし型として扱われます。
  • PG_REGEX_STRATEGY_LIBC_WIDE または PG_REGEX_STRATEGY_LIBC_1BYTE を使用する場合、pg_regex_locale が有効なロケール情報を保持している必要があります。

アルゴリズムまたは振る舞い

現在の pg_regex_strategy に応じて以下のいずれかの判定ロジックを実行します。 1. **PG_REGEX_STRATEGY_C**: pg_char_properties 配列を参照し、ASCII範囲内での小文字判定を行います。 2. **PG_REGEX_STRATEGY_BUILTIN**: pg_u_islower(c) を呼び出し、組み込みのUnicodeテーブルで判定します。 3. **PG_REGEX_STRATEGY_LIBC_WIDE**: wchar_t のサイズに応じて iswlower_l を使用します。 4. **PG_REGEX_STRATEGY_LIBC_1BYTE**: islower_l を使用し、1バイト文字として判定します。 5. **PG_REGEX_STRATEGY_ICU**: u_islower(c) を呼び出します（USE_ICU 定義時）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_locale.c:651 (pg_ctype_get_cache への引数として使用)
  • src/backend/regex/regc_locale.c:700
  • **呼び出し先**:
  • pg_u_islower (BUILTIN戦略時)
  • iswlower_l / islower_l (LIBC戦略時)
  • u_islower (ICU戦略時)

関連データ構造

  • pg_regex_strategy: 判定戦略を決定する列挙型。
  • pg_regex_locale: ロケール情報を保持する構造体。
  • pg_char_properties: Cロケール用の文字プロパティテーブル。

失敗モードと境界条件

  • pg_regex_strategy が未知の値である場合、関数は 0 を返します。
  • PG_REGEX_STRATEGY_LIBC_WIDE において wchar_t が小さい環境で c が 0xFFFF を超える場合、iswlower_l の呼び出しをスキップし、後続の LIBC_1BYTE 判定へフォールスルーする可能性があります。

保守担当者向けメモ

  • 本関数は regcomp.c からインクルードされることを前提としています。
  • ICUサポート (USE_ICU) はコンパイル時の条件に依存します。
  • キャッシュ機構 (pg_ctype_get_cache) と組み合わせて使用されるため、判定ロジックの変更はキャッシュの整合性に影響を与える可能性があります。
