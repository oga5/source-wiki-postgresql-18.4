ExecInitResult

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeResult.c:179

目的

Result ノードの実行時状態である ResultState を作成し、式コンテキストの初期化、子ノードの構築、および投影（Projection）に必要な設定を行う。

シグネチャまたは形状

ResultState *ExecInitResult(Result *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Result プランノード。
  • estate: クエリ実行全体の実行状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの ResultState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_MARK または EXEC_FLAG_BACKWARD が含まれる場合、outerPlan(node) が存在しなければならない（Result ノード自体は単独での Mark/Restore をサポートしないため）。
  • innerPlan(node) は常に NULL である必要がある。

アルゴリズムまたは振る舞い

1. ResultState 構造体を割り当て、ps.ExecProcNode に ExecResult を設定する。 2. rs_done を false に初期化し、resconstantqual の有無に基づいて rs_checkqual を設定する。 3. ExecAssignExprContext を呼び出し、式コンテキストを作成する。 4. ExecInitNode を呼び出し、子ノード（outerPlan）を初期化する。 5. ExecInitResultTupleSlotTL を呼び出し、仮想タプルスロットを初期化する。 6. ExecAssignProjectionInfo を呼び出し、投影情報を設定する。 7. ExecInitQual を使用して、ノードの qual および resconstantqual を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:167（ExecInitNode 経由での呼び出しが確認されている）。
  • **呼び出し先**: makeNode, ExecAssignExprContext, ExecInitNode, ExecInitResultTupleSlotTL, ExecAssignProjectionInfo, ExecInitQual。

関連データ構造

  • ResultState: Result ノードの実行時状態を保持する構造体。
  • Result: プランツリー上のノード定義。
  • EState: クエリ実行状態。
  • PlanState: ResultState の基底構造体。

失敗モードと境界条件

  • Assert により、サポートされていないフラグの組み合わせや、innerPlan の存在が検出された場合に異常終了する。
  • メモリ不足時は makeNode 等の内部関数によりエラーが発生する。

保守担当者向けメモ

  • Result ノードは子ノードを持たない場合、一度だけ実行される特殊な挙動をとる。この初期化ロジックは、その「一度だけ」の実行に必要な定数評価の準備を含んでいる。
  • resconstantqual は EXPLAIN で表示される「One-Time Filter」に対応する。
