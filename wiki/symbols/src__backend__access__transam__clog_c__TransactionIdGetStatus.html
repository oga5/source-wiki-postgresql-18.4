TransactionIdGetStatus

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:734

目的

指定されたトランザクションID（XID）の現在のコミット状態（XidStatus）と、その状態に関連付けられたログシーケンス番号（LSN）を取得する。

シグネチャまたは形状

XidStatus TransactionIdGetStatus(TransactionId xid, XLogRecPtr *lsn)

入力と出力

  • **入力**:
  • xid: 状態を確認する対象のトランザクションID。
  • lsn: 取得したLSNを格納するためのポインタ（出力用）。
  • **出力**:
  • 戻り値: トランザクションの状態を示す XidStatus 型の値。
  • lsn: 引数経由で、当該トランザクション状態に関連する最新のLSNが書き込まれる。

前提条件と不変条件

  • XactCtl（CLOGの共有メモリ制御構造体）が初期化済みであること。
  • 内部で SimpleLruReadPage_ReadOnly を呼び出し、対象ページに対するロックを取得する。
  • 呼び出し側は、本関数がロックを保持したまま読み取りを行うため、短時間で処理を終える必要がある。

アルゴリズムまたは振る舞い

1. TransactionIdToPage 等を用いて、指定された xid に対応するページ番号、バイト位置、ビットシフト量を算出する。 2. SimpleLruReadPage_ReadOnly を呼び出し、対象ページを読み込み、ロックを獲得する。 3. 算出されたオフセットに基づき、CLOGページ上のビット列から XidStatus を抽出する。 4. GetLSNIndex を用いて、当該スロットに対応するLSNインデックスを特定し、XactCtl から対応する group_lsn を取得して出力する。 5. LWLockRelease により、SimpleLruGetBankLock で取得したロックを解放する。 6. 抽出した状態を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/transam.c（複数箇所）
  • **呼び出し先**:
  • SimpleLruReadPage_ReadOnly
  • GetLSNIndex
  • LWLockRelease
  • SimpleLruGetBankLock

関連データ構造

  • XidStatus: トランザクションの状態（コミット、アボート等）を保持する列挙型。
  • XactCtl: CLOGの共有メモリ状態を管理する SlruCtlData 構造体。
  • XLogRecPtr: WALのログシーケンス番号。

失敗モードと境界条件

  • 指定された xid が無効な場合や、CLOGページが破損している場合の挙動については、SimpleLruReadPage_ReadOnly のエラー処理に依存する。

保守担当者向けメモ

  • 本関数はロックを保持して読み取りを行うため、パフォーマンスへの影響を考慮すること。
  • 非同期コミット時には、WALがフラッシュされるまでLSNが追跡される仕組みが組み込まれている。
