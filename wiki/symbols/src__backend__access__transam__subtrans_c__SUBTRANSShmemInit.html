SUBTRANSShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:219

目的

サブトランザクション管理用の共有メモリ領域（SLRU）を初期化し、必要に応じてバッファ数を自動調整する。

シグネチャまたは形状

void SUBTRANSShmemInit(void)

入力と出力

  • **入力**: なし（グローバル変数 subtransaction_buffers および SubTransCtl を使用）。
  • **出力**: なし。SubTransCtl 構造体の初期化および subtransaction_buffers GUC パラメータの設定を行う。

前提条件と不変条件

  • SubTransCtl が指す共有メモリ領域が割り当て済みであること。
  • 本関数は共有メモリの初期化フェーズで呼び出されること。
  • subtransaction_buffers が 0 の場合、SUBTRANSShmemBuffers() を用いて自動計算される。

アルゴリズムまたは振る舞い

1. **バッファ数の自動調整**: subtransaction_buffers が 0 の場合、SUBTRANSShmemBuffers() で算出した値を SetConfigOption を用いて設定する。設定が失敗した場合は PGC_S_OVERRIDE を使用して強制適用する。 2. **SLRU 初期化**:

  • SubTransCtl->PagePrecedes に SubTransPagePrecedes を割り当てる。
  • SimpleLruInit を呼び出し、pg_subtrans ディレクトリを対象とした SLRU 制御構造を初期化する。

    • 3. **検証**: SlruPagePrecedesUnitTests を実行し、SLRU のページ順序ロジックの整合性を確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c 内の CreateSharedMemoryAndSemaphores から呼び出される。
  • **呼び出し先**: SUBTRANSShmemBuffers, SetConfigOption, SimpleLruInit, SubTransPagePrecedes, SlruPagePrecedesUnitTests。

関連データ構造

  • SubTransCtl: サブトランザクション SLRU の制御構造体。
  • LWTRANCHE_SUBTRANS_BUFFER, LWTRANCHE_SUBTRANS_SLRU: SLRU 操作に使用される軽量ロック（LWLock）のトランシェ。

失敗モードと境界条件

  • subtransaction_buffers の設定が競合する場合、PGC_S_OVERRIDE を用いて上書きを試みる。
  • SimpleLruInit 内部でメモリ確保やディレクトリ操作に失敗した場合、PostgreSQL の標準的なエラーハンドリングによりプロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数は PostgreSQL 起動時の共有メモリ初期化シーケンスの一部である。
  • pg_subtrans はクラッシュリカバリを目的としないため、再起動時にはデータがゼロクリアされる前提で設計されている。
