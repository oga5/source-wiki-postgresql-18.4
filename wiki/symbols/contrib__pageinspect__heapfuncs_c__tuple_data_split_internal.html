tuple_data_split_internal

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/heapfuncs.c:312

目的

ヒープタプルのバイナリデータから、カタログ情報（TupleDesc）に基づき各属性値を抽出し、bytea 型の配列として返却する。主に破損したページや低レベルなデータ構造の調査に使用される。

シグネチャまたは形状

Datum tuple_data_split_internal(Oid relid, char *tupdata, uint16 tupdata_len, 
                                uint16 t_infomask, uint16 t_infomask2, 
                                bits8 *t_bits, bool do_detoast)

入力と出力

  • **入力**:
  • relid: 対象リレーションの OID。
  • tupdata: タプルデータの先頭ポインタ。
  • tupdata_len: タプルデータの長さ。
  • t_infomask, t_infomask2: タプルの状態フラグ。
  • t_bits: NULL ビットマップ。
  • do_detoast: TOAST データの展開を行うかどうかのフラグ。
  • **出力**:
  • 属性値を格納した Datum（bytea 型の配列）。

前提条件と不変条件

  • 対象リレーションは AccessShareLock でオープンされる。
  • リレーションがヒープアクセスメソッド（HEAP_TABLE_AM_OID）を使用していること。
  • 属性数がタプルヘッダの宣言数と整合していること。

アルゴリズムまたは振る舞い

1. relid から TupleDesc を取得し、属性数（nattrs）を特定する。 2. initArrayResult で結果格納用の配列を初期化する。 3. 各属性について以下の処理を繰り返す：

  • t_infomask と t_bits を参照し、属性が NULL かどうかを判定する。
  • 非 NULL の場合、attlen に基づきオフセットを計算し、データ長を特定する。
  • do_detoast が真かつ可変長データ（attlen == -1）の場合は pg_detoast_datum_copy を実行する。
  • 抽出したデータを accumArrayResult で配列に追加する。

    • 4. 全属性の処理終了後、tupdata_len と最終的なオフセットが一致することを確認し、配列を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pageinspect/heapfuncs.c 内の tuple_data_split 関数から呼び出される。
  • **呼び出し先**: relation_open, RelationGetDescr, initArrayResult, TupleDescCompactAttr, att_pointer_alignby, VARSIZE_ANY, pg_detoast_datum_copy, palloc, accumArrayResult, relation_close 等。

関連データ構造

  • TupleDesc: 属性定義情報。
  • ArrayBuildState: 結果構築用の中間状態。
  • CompactAttribute: 属性の物理レイアウト情報。

失敗モードと境界条件

  • **ERRCODE_FEATURE_NOT_SUPPORTED**: ヒープ AM 以外が指定された場合。
  • **ERRCODE_DATA_CORRUPTED**:
  • タプルヘッダの属性数が記述子より多い場合。
  • 可変長データの先頭バイトが不正な場合。
  • データの読み取り中に予期せぬ終端に達した場合。
  • 全データを走査した結果が tupdata_len と一致しない場合。

保守担当者向けメモ

  • 本関数は superuser() 権限を必要とする関数の内部処理として実装されている。
  • 破損したページを扱うため、メモリ破壊を防ぐための境界チェックが随所で行われている。
  • ALTER TABLE ADD COLUMN 等で発生する「ヘッダ上の属性数と記述子の不一致」を許容するロジックが含まれている。
