CatalogOpenIndexes

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/indexing.c:42

目的

システムカタログの更新時に必要なインデックス情報をロードし、インデックス操作用の状態構造体（CatalogIndexState）を初期化する。

シグネチャまたは形状

CatalogIndexState CatalogOpenIndexes(Relation heapRel)

入力と出力

  • **入力**: heapRel (対象となるシステムカタログの Relation 記述子)
  • **出力**: CatalogIndexState (実体は ResultRelInfo 構造体へのポインタ)

前提条件と不変条件

  • heapRel は有効なシステムカタログのリレーション記述子であること。
  • 戻り値の CatalogIndexState は、後続のインデックス更新処理（CatalogIndexInsert 等）で使用される。
  • 本関数はトリガーを起動しない設定（ri_TrigDesc = NULL）で初期化を行う。

アルゴリズムまたは振る舞い

1. ResultRelInfo 構造体をメモリ上に確保する。 2. ri_RangeTableIndex をダミー値 0 で初期化する。 3. ri_RelationDesc に引数の heapRel を設定する。 4. ri_TrigDesc を NULL に設定する。 5. ExecOpenIndices(resultRelInfo, false) を呼び出し、対象リレーションに関連付けられたすべてのインデックスをオープンする。 6. 初期化済みの resultRelInfo を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c (複数箇所)
  • src/backend/catalog/index.c
  • src/backend/catalog/indexing.c (内部の最適化関数等)
  • **呼び出し先**:
  • makeNode (PostgreSQLのノード作成マクロ)
  • ExecOpenIndices (インデックスオープン処理の基盤関数)

関連データ構造

  • CatalogIndexState: ResultRelInfo の別名。インデックス更新に必要なリソースを保持する。
  • Relation: ヒープリレーションの記述子。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性がある。
  • ExecOpenIndices が失敗した場合、PostgreSQLの標準的なエラーハンドリングにより処理が中断される。

保守担当者向けメモ

  • 本関数で確保したリソースは、必ず対応する CatalogCloseIndexes で解放する必要がある。
  • 複数行の操作を行う場合は、オーバーヘッドを削減するために *WithInfo 系関数（CatalogTupleInsertWithInfo 等）の使用を検討すること。
  • システムカタログ特有の制約により、式インデックスや部分インデックスはサポートされていない点に注意。
