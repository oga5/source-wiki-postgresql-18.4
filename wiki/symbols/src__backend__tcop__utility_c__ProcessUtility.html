ProcessUtility

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/utility.c:498

目的

PostgreSQLにおけるユーティリティコマンド（CREATE, ALTER, DROP, VACUUM 等の非DML文）の実行を制御するメインエントリーポイントです。

シグネチャまたは形状

void ProcessUtility(PlannedStmt *pstmt,
                    const char *queryString,
                    bool readOnlyTree,
                    ProcessUtilityContext context,
                    ParamListInfo params,
                    QueryEnvironment *queryEnv,
                    DestReceiver *dest,
                    QueryCompletion *qc)

入力と出力

  • **入力**:
  • pstmt: 実行対象の計画済みステートメント。
  • queryString: 元のSQL文字列。
  • readOnlyTree: パーサーが生成したツリーを保護するためのフラグ。
  • context: 実行コンテキスト。
  • params, queryEnv: パラメータおよびクエリ環境。
  • dest: 結果の出力先。
  • qc: コマンド完了情報を格納する構造体。
  • **出力**: なし（void）。qc を通じて実行結果が返されます。

前提条件と不変条件

  • pstmt は PlannedStmt 型であり、commandType が CMD_UTILITY である必要があります。
  • queryString は NULL であってはなりません。
  • qc が指定されている場合、その commandTag は CMDTAG_UNKNOWN である必要があります。

アルゴリズムまたは振る舞い

1. **フックの確認**: ProcessUtility_hook が設定されている場合、それを呼び出します。 2. **標準実行**: フックがない場合、standard_ProcessUtility を呼び出してコマンドを実行します。 3. **委譲**: 実際のコマンド実行ロジックは、フックまたは標準実装のいずれかに完全に委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/ 配下の各コマンド実装や、src/backend/access/transam/xact.c 等のトランザクション制御層から呼び出されます。
  • **呼び出し先**: ProcessUtility_hook（設定時）または standard_ProcessUtility。

関連データ構造

  • PlannedStmt: 実行対象のコマンド構造体。
  • ProcessUtilityContext: 実行コンテキストを定義する列挙型。
  • QueryCompletion: コマンド完了後の状態を保持する構造体。

失敗モードと境界条件

  • Assert 文により、不正な pstmt 型や NULL 文字列が渡された場合に異常終了します。
  • 実行環境（読み取り専用トランザクション、パラレルモード等）の制約違反がある場合、standard_ProcessUtility 内でエラーが発生します。

保守担当者向けメモ

  • 本関数は単なるディスパッチャーであり、実際のロジックは standard_ProcessUtility にあります。
  • プラグインによる拡張をサポートするため、必ず ProcessUtility_hook を経由する設計となっています。
