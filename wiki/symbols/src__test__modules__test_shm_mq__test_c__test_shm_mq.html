test_shm_mq

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/test.c:42

目的

PostgreSQLの共有メモリメッセージキュー（shm_mq）の同期通信機能を検証するためのテスト関数です。SQLインターフェースを通じて呼び出され、バックグラウンドワーカーと共有メモリを介したメッセージの送受信ループを実行します。

シグネチャまたは形状

Datum test_shm_mq(PG_FUNCTION_ARGS)
  • 引数: SQL経由で以下の4つを受け取る（PG_GETARG_*を使用）

    • 1. queue_size (int64): キューのサイズ 2. message (text*): 送信するメッセージ内容 3. loop_count (int32): ループ回数 4. nworkers (int32): 使用するワーカー数

入力と出力

  • 入力: 共有メモリ設定、メッセージデータ、ループ回数、ワーカー数。
  • 出力: PG_RETURN_VOID()。成功時は終了し、失敗時は ereport によりエラーを発生させる。

前提条件と不変条件

  • loop_count は 0 以上であること。
  • nworkers は 0 より大きいこと（自身でブロックするため、ワーカーが必須）。
  • 共有メモリセグメントおよびバックグラウンドワーカーのセットアップが test_shm_mq_setup によって正常に完了すること。

アルゴリズムまたは振る舞い

1. **初期化**: test_shm_mq_setup を呼び出し、共有メモリとワーカーを準備。 2. **初回送信**: shm_mq_send を使用して初期メッセージを送信。 3. **ループ処理**:

  • shm_mq_receive でメッセージを受信。
  • loop_count が 0 に達するまで、受信したメッセージを shm_mq_send で再送する。

    • 4. **検証**: ループ終了後、verify_message を呼び出し、送受信データの整合性を確認。 5. **終了処理**: dsm_detach で共有メモリセグメントを解放。

呼び出し元と呼び出し先

  • 呼び出し元: SQLインターフェース（PG_FUNCTION_INFO_V1 により登録）。
  • 呼び出し先:
  • test_shm_mq_setup (環境構築)
  • shm_mq_send / shm_mq_receive (キュー操作)
  • verify_message (データ検証)
  • dsm_detach (メモリ解放)

関連データ構造

  • dsm_segment: 動的共有メモリセグメント。
  • shm_mq_handle: メッセージキューのハンドル（送信用 outqh、受信用 inqh）。
  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • loop_count < 0: ERRCODE_INVALID_PARAMETER_VALUE でエラー。
  • nworkers <= 0: ERRCODE_INVALID_PARAMETER_VALUE でエラー。
  • shm_mq_send または shm_mq_receive が SHM_MQ_SUCCESS 以外を返した場合: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー。

保守担当者向けメモ

  • 本関数はブロッキングインターフェースを使用しているため、デッドロックを避けるために適切なワーカー数とキューサイズの設定が必要です。
  • verify_message による比較処理は、パイプラインテストとは異なり、同期テストの最終段階で必ず実行されます。
