pgwin32_is_admin

定義

  • 種別: function
  • 定義箇所: src/port/win32security.c:48

目的

現在のプロセスが Windows の管理者グループ（Administrators または Power Users）の権限で実行されているかを判定します。

シグネチャまたは形状

int pgwin32_is_admin(void);

入力と出力

  • **入力**: なし
  • **出力**:
  • 1: 現在のプロセスが管理者または Power Users グループのメンバーである場合。
  • 0: いずれのグループにも属さない場合。

前提条件と不変条件

  • Windows API (AllocateAndInitializeSid, CheckTokenMembership) が利用可能であること。
  • 判定に必要な SID の生成およびトークンチェックが成功すること。

アルゴリズムまたは振る舞い

1. AllocateAndInitializeSid を使用し、SECURITY_NT_AUTHORITY を基に「Administrators」および「Power Users」グループの SID を生成します。 2. CheckTokenMembership を呼び出し、現在のアクセストークンが上記いずれかの SID を保持しているかを確認します。 3. FreeSid を呼び出し、生成した SID リソースを解放します。 4. 判定結果（いずれかのグループに属しているか）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/main/main.c:470 での参照が確認されています。
  • **呼び出し先**: Windows API (AllocateAndInitializeSid, CheckTokenMembership, FreeSid) および内部関数 log_error。

関連データ構造

  • SID_IDENTIFIER_AUTHORITY: SID 生成時に使用される Windows 構造体。
  • PSID: セキュリティ識別子を保持するポインタ型。

失敗モードと境界条件

  • SID の生成またはトークンチェックが失敗した場合、log_error を通じてエラーコードを出力し、exit(1) によりプロセスを終了します。これは本関数が初期化フェーズで呼び出されることを想定しているためです。

保守担当者向けメモ

  • 本関数は ereport() が利用できない極めて早い段階で呼び出されるため、エラーハンドリングには標準エラー出力へ直接書き出す log_error を使用しています。
  • 判定ロジックは Windows のアクセストークンに基づいています。
