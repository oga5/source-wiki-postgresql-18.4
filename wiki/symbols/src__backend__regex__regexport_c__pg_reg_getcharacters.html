pg_reg_getcharacters

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:265

目的

コンパイル済みの正規表現が保持するカラーマップから、指定されたカラー ID に対応する文字コードのリストを抽出する。

シグネチャまたは形状

void pg_reg_getcharacters(const regex_t *regex, int co, pg_wchar *chars, int chars_len)

入力と出力

  • regex: コンパイル済みの正規表現オブジェクト（regex_t）。
  • co: 抽出対象のカラー ID。
  • chars: 抽出した文字コードを格納する呼び出し元提供のバッファ。
  • chars_len: chars バッファのサイズ。

前提条件と不変条件

  • regex は有効な正規表現オブジェクトであること（REMAGIC チェック済み）。
  • co が有効な範囲（1 から cm->max まで）であること。
  • PSEUDO フラグが立っているカラーは対象外とする。

アルゴリズムまたは振る舞い

1. regex からカラーマップ（colormap）を取得する。 2. co が範囲外、または chars_len が 0 以下の場合は即座に終了する。 3. 指定されたカラーが PSEUDO フラグを持つ場合は終了する。 4. CHR_MIN から MAX_SIMPLE_CHR までの範囲でカラーマップ（locolormap）を線形走査する。 5. マップ内の値が指定された co と一致する場合、その文字コードを chars バッファに書き込み、バッファの残量を減らす。 6. バッファが満杯になるか、走査範囲が終了するまで繰り返す。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_trgm/trgm_regexp.c:800 からの呼び出しが確認されている。
  • 呼び出し先: 本関数は内部ヘルパーを呼び出さず、regex 構造体内の colormap を直接参照する。

関連データ構造

  • regex_t: 正規表現のコンパイル結果を保持する構造体。
  • colormap: 正規表現内の文字クラス情報を管理する構造体。
  • pg_wchar: 文字コードを表す型。

失敗モードと境界条件

  • co が 0 以下、またはカラーマップの最大値を超えている場合は何も行わない。
  • chars_len が 0 以下の場合、何も行わない。
  • PSEUDO フラグを持つカラーに対しては何も行わない。
  • バッファサイズが不足している場合、バッファに収まる分だけを書き込んで終了する。

保守担当者向けメモ

  • 本関数はカラーマップを線形走査するため、呼び出し頻度が高い場合はパフォーマンスに影響を与える可能性がある。
  • locolormap のみを走査対象としており、高位文字マップ（high map）は考慮していない。
