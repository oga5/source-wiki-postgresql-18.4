get_exec_path

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:370

目的

PostgreSQLの関連バイナリ（pg_ctl や pg_resetwal など）の絶対パスを、実行中の pg_createsubscriber と同一ディレクトリ内から検索して取得する。

シグネチャまたは形状

static char *get_exec_path(const char *argv0, const char *progname)

入力と出力

  • **入力**:
  • argv0: 呼び出し元プログラムの実行パス（argv[0]）。
  • progname: 検索対象のプログラム名（例: "pg_ctl"）。
  • **出力**:
  • 成功時: 検索されたプログラムの絶対パスを格納した char *（呼び出し元で解放が必要）。
  • 失敗時: pg_fatal を呼び出し、プロセスを終了させる。

前提条件と不変条件

  • progname で指定されたプログラムが、pg_createsubscriber と同じディレクトリに存在することが期待される。
  • MAXPGPATH バッファサイズに収まるパス長であること。

アルゴリズムまたは振る舞い

1. find_other_exec を使用し、argv0 を基準に progname を検索する。 2. 検索結果が正常（ret >= 0）であれば、そのパスを返す。 3. 検索に失敗した場合（ret < 0）:

  • find_my_exec で自身のパスを特定し、エラーメッセージを構築する。
  • ret == -1 の場合は「ファイル未発見」、それ以外の場合は「バージョン不一致」として pg_fatal を発行し終了する。

    • 4. 成功時はデバッグログにパスを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_createsubscriber.c:2399 (pg_ctl_path の取得)
  • src/bin/pg_basebackup/pg_createsubscriber.c:2400 (pg_resetwal_path の取得)
  • **呼び出し先**:
  • psprintf
  • pg_malloc
  • find_other_exec
  • find_my_exec
  • pg_fatal
  • pg_log_debug

関連データ構造

  • MAXPGPATH: パス文字列の最大長を規定する定数。

失敗モードと境界条件

  • 指定されたプログラムが同一ディレクトリに見つからない場合、pg_fatal によりプログラムが異常終了する。
  • バージョン不一致（find_other_exec が正の値を返さない場合）も同様に pg_fatal を引き起こす。

保守担当者向けメモ

  • 本関数は pg_createsubscriber が依存する外部ツールを安全に特定するためのユーティリティです。
  • 実行環境の PATH に依存せず、自身のバイナリ配置場所を基準にする設計となっています。
