verifier_context

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.h:87

目的

PostgreSQL のバックアップ検証プロセス全体で共有される、実行時の状態および設定情報を保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct verifier_context
{
	manifest_data *manifest;
	char	   *backup_directory;
	SimpleStringList ignore_list;
	char		format;
	bool		skip_checksums;
	bool		exit_on_error;
	bool		saw_any_error;
} verifier_context;

入力と出力

本構造体は関数間でポインタとして受け渡され、検証の進行状況（エラー発生有無）や設定（スキップフラグ等）を共有します。直接的な戻り値はありません。

前提条件と不変条件

  • manifest フィールドは、解析済みの manifest_data を指している必要があります。
  • format はバックアップ形式（'p': plain, 't': tar）を示す文字を保持します。

アルゴリズムまたは振る舞い

検証プロセスにおいて、以下の役割を果たします。

  • ignore_list を用いた検証対象外パスの管理。
  • skip_checksums によるチェックサム検証の有効/無効制御。
  • saw_any_error による検証中のエラー発生状態の追跡。
  • exit_on_error によるエラー発生時の即時終了フラグの管理。

呼び出し元と呼び出し先

本構造体は pg_verifybackup のメインロジックおよびアーカイブストリーム処理で参照されます。

  • **主な呼び出し元**:
  • src/bin/pg_verifybackup/pg_verifybackup.c: verify_tar_backup, verify_plain_backup_directory, verify_backup_checksums 等の各検証関数。
  • src/bin/pg_verifybackup/astreamer_verify.c: astreamer_verify_content_new 等のストリーム処理関数。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • manifest_data: 検証対象のバックアップマニフェスト情報。
  • SimpleStringList: 除外リスト（ignore_list）の管理に使用。

失敗モードと境界条件

  • saw_any_error が true に設定された場合、検証プロセスはエラー状態として扱われます。
  • exit_on_error が true の場合、エラー発生時に検証処理を中断するロジックの判定基準となります。

保守担当者向けメモ

  • verifier_context 内のフィールドを追加・変更する場合は、pg_verifybackup.c および astreamer_verify.c 内の各検証関数への影響を確認してください。
  • ignore_list の操作には fe_utils/simple_list.h の関数群を使用してください。
