fail_token

定義

  • 種別: function
  • 定義箇所: src/test/modules/oauth_validator/fail_validator.c:40

目的

OAuthトークン検証プロセスにおいて、意図的に FATAL エラーを発生させ、システムが検証失敗をどのように処理するかをテストするためのコールバック関数です。

シグネチャまたは形状

static bool fail_token(const ValidatorModuleState *state,
                       const char *token, 
                       const char *role,
                       ValidatorModuleResult *res);

入力と出力

  • 入力:
  • state: 検証モジュールの状態（ValidatorModuleState）
  • token: 検証対象のトークン文字列
  • role: 関連付けられたロール名
  • res: 検証結果を格納する構造体（ValidatorModuleResult）
  • 出力: なし（関数は戻り値に到達せず、プロセスを終了させる）

前提条件と不変条件

  • 本関数は elog(FATAL, ...) を実行するため、呼び出された時点でバックエンドプロセスが強制終了されます。
  • pg_unreachable() が呼び出されるため、本関数が正常に呼び出し元へ制御を戻すことはありません。

アルゴリズムまたは振る舞い

1. elog(FATAL, "fail_validator: sentinel error") を実行し、PostgreSQLのログにエラーを出力してバックエンドプロセスを停止させます。 2. pg_unreachable() を呼び出し、コンパイラに対して本関数が戻り値を持たないことを伝達します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLのOAuth検証フック実行エンジン（直接的な呼び出し元コードは本ファイル内の validator_callbacks 構造体による参照）。
  • 呼び出し先: elog（PostgreSQLのログ出力およびエラー処理関数）。

関連データ構造

  • ValidatorModuleState: 検証モジュールの状態を保持する構造体。
  • ValidatorModuleResult: 検証結果を格納する構造体。
  • validator_callbacks: 本関数を validate_cb フィールドに登録している構造体。

失敗モードと境界条件

  • 本関数自体が「検証失敗時の異常系動作をテストする」という失敗モードを意図的に引き起こすための実装です。
  • FATAL エラーをスローするため、呼び出し側のトランザクションやプロセスは即座に中断されます。

保守担当者向けメモ

  • 本関数はテスト専用です。本番環境のコードには決して組み込まないでください。
  • elog(FATAL) はプロセスを終了させるため、デバッグ目的以外での使用は避けてください。
