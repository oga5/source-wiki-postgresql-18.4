XactCallbackItem

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xact.c:308

目的

トランザクションのコミットやアボート時に実行されるコールバック関数を保持するためのリスト要素です。

シグネチャまたは形状

typedef struct XactCallbackItem
{
    struct XactCallbackItem *next;
    XactCallback callback;
    void       *arg;
} XactCallbackItem;

入力と出力

  • next: リスト内の次の要素へのポインタ。
  • callback: トランザクション終了時に呼び出される関数ポインタ。
  • arg: コールバック関数に渡される任意の引数。

前提条件と不変条件

  • Xact_callbacks を起点とする単方向連結リストの一部として管理されます。
  • リストの要素は TopMemoryContext 上に割り当てられます。

アルゴリズムまたは振る舞い

  • トランザクション終了処理において、登録されたコールバックリストを順次走査し、各 callback を arg を引数として実行します。
  • リストへの追加および削除操作を通じて、動的にコールバックを登録・解除可能です。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c 内のコールバック登録関数およびトランザクション終了処理関数。
  • 呼び出し先: callback フィールドに格納された関数（具体的な関数名はソース抜粋からは特定できません）。

関連データ構造

  • XactCallback (型定義)
  • Xact_callbacks (静的リストのヘッドポインタ)

失敗モードと境界条件

  • メモリ不足時: MemoryContextAlloc が失敗した場合、トランザクションのコールバック登録に失敗します。
  • リストが空の場合: Xact_callbacks が NULL の場合、コールバックは実行されません。

保守担当者向けメモ

  • 本構造体はトランザクションのライフサイクルに密接に関連しています。コールバック内で実行される処理がトランザクションの整合性を損なわないよう注意してください。
  • 登録されたコールバックの実行順序は、リストへの追加順序に依存します。
