be_lo_close

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-fsstubs.c:125

目的

SQLインターフェースから呼び出され、指定されたラージオブジェクト（LO）ディスクリプタをクローズし、関連するリソースを解放します。

シグネチャまたは形状

Datum be_lo_close(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される int32 型のラージオブジェクトディスクリプタ（fd）。
  • **出力**: 成功時は 0 を返します。

前提条件と不変条件

  • fd は、現在開かれている有効なラージオブジェクトディスクリプタである必要があります。
  • 内部の cookies 配列が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32(0) を使用して引数から fd を取得します。 2. fd が負の値、cookies_size 以上、または cookies[fd] が NULL である場合、ERRCODE_UNDEFINED_OBJECT エラーを発生させます。 3. FSDB マクロが定義されている場合、デバッグログを出力します。 4. closeLOfd(fd) を呼び出し、ディスクリプタに関連付けられたリソースをクローズします。 5. PG_RETURN_INT32(0) を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（直接の呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: closeLOfd（内部関数）。

関連データ構造

  • cookies: 開いているラージオブジェクトディスクリプタを保持する動的配列。
  • LargeObjectDesc: cookies 配列に格納されるラージオブジェクトの記述子構造体。

失敗モードと境界条件

  • **無効なディスクリプタ**: fd が範囲外であるか、既にクローズされている（cookies[fd] == NULL）場合、エラーが発生します。

保守担当者向けメモ

  • 本関数はトランザクション境界でのリソース管理の一部を担います。
  • FSDB マクロを有効にすることで、DEBUG4 レベルのログ出力による詳細な追跡が可能です。
