get_destination_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_receivewal.c:234

目的

指定されたディレクトリパスを開き、WAL ファイル操作のためのディレクトリストリームを取得する。

シグネチャまたは形状

static DIR *get_destination_dir(char *dest_folder)

入力と出力

  • **入力**: dest_folder (ディレクトリパスを示す文字列へのポインタ)
  • **出力**: 成功時は DIR * 型のディレクトリストリームを返す。失敗時は pg_fatal を呼び出し、プロセスを終了させる。

前提条件と不変条件

  • dest_folder は NULL であってはならない（Assert により検証）。

アルゴリズムまたは振る舞い

1. dest_folder が NULL でないことを確認する。 2. opendir() を呼び出し、指定されたディレクトリを開く。 3. ディレクトリのオープンに失敗した場合、pg_fatal を使用してエラーメッセージを出力し、プログラムを終了させる。 4. 成功した場合、取得した DIR * を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_receivewal.c:276
  • src/bin/pg_basebackup/pg_receivewal.c:823
  • **呼び出し先**:
  • opendir() (標準ライブラリ)
  • pg_fatal() (PostgreSQL 共通ログユーティリティ)

関連データ構造

  • DIR: ディレクトリストリームを保持する構造体。

失敗モードと境界条件

  • dest_folder が存在しない、またはアクセス権限がない場合、opendir() が失敗し、pg_fatal によりエラー終了する。

保守担当者向けメモ

  • 本関数は pg_receivewal におけるディレクトリ操作の標準的なラッパーです。ディレクトリのクローズには close_destination_dir を使用してください。
