nameout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:70

目的

PostgreSQLの内部データ型 name を、クライアントやログ出力に適した cstring（ヌル終端文字列）形式に変換します。

シグネチャまたは形状

Datum nameout(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して name 型のデータ（Name 構造体へのポインタ）を受け取ります。
  • **出力**: Datum 型として、変換された cstring へのポインタを返します。

前提条件と不変条件

  • 入力された Name 型データは、NAMEDATALEN バイト以内の固定長文字列として格納されている必要があります。
  • pstrdup を使用してメモリを動的に確保するため、呼び出し元はメモリ管理（コンテキストのクリーンアップ）に責任を持つ必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_NAME(0) を使用して、入力された name 型の Datum を Name 構造体ポインタとして取得します。 2. NameStr(*s) マクロにより、内部の文字列データを取り出します。 3. pstrdup() を呼び出し、文字列を現在のメモリコンテキストにコピーして cstring を生成します。 4. PG_RETURN_CSTRING() を介して、生成された文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/trigger.c（トリガー名等の取得時）
  • src/backend/utils/adt/like_support.c（LIKE 演算子のサポート関数）
  • **呼び出し先**:
  • pstrdup（メモリ確保とコピー）
  • NameStr（マクロによるデータアクセス）

関連データ構造

  • Name: name 型の内部表現。NAMEDATALEN バイトの固定長配列。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • メモリ不足時には pstrdup がエラーを発生させ、処理が中断されます。
  • name 型の内部データが正しくヌル終端されていない場合、NameStr マクロの挙動に依存します（通常、name 型は適切に管理されていることが前提です）。

保守担当者向けメモ

  • 本関数は name 型の標準的な出力変換関数です。システムカタログの識別子を人間が読める形式に変換する際に多用されます。
  • NAMEDATALEN の制限は name 型の定義に依存するため、将来的なサイズ変更の際は Name 型の定義箇所を確認してください。
