CreateConversionCommand

定義

  • 種別: function
  • 定義箇所: src/backend/commands/conversioncmds.c:31

目的

SQLの CREATE CONVERSION コマンドを処理し、ユーザー定義の文字コード変換ルールをシステムカタログ pg_conversion に登録する。

シグネチャまたは形状

ObjectAddress CreateConversionCommand(CreateConversionStmt *stmt)

入力と出力

  • **入力**: CreateConversionStmt *stmt（解析済みのSQL構文木）
  • **出力**: ObjectAddress（作成された変換オブジェクトの識別子）

前提条件と不変条件

  • 変換先のスキーマに対する CREATE 権限が必要。
  • ソースおよびターゲットのエンコーディングがシステムでサポートされていること。
  • SQL_ASCII を含む変換は禁止（内部的に高速パスが使用されるため）。
  • 指定された変換関数は (int4, int4, cstring, internal, int4, bool) の引数構成を持ち、int4 を返す必要がある。
  • 変換関数に対して EXECUTE 権限が必要。

アルゴリズムまたは振る舞い

1. **名前解決**: QualifiedNameGetCreationNamespace で変換オブジェクトを作成するスキーマを特定。 2. **権限確認**: object_aclcheck でスキーマへの作成権限を検証。 3. **エンコーディング検証**: pg_char_to_encoding でエンコーディングの存在を確認し、SQL_ASCII が含まれていないかチェック。 4. **関数検証**: LookupFuncName で変換関数を特定し、戻り値型が int4 であることを確認。 5. **権限確認**: 関数に対する EXECUTE 権限を検証。 6. **動作テスト**: 空文字列を引数として変換関数を OidFunctionCall6 で呼び出し、正常に動作するか（戻り値が0か）を事前確認。 7. **永続化**: ConversionCreate を呼び出し、カタログへ登録。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQL解析後のディスパッチ）
  • **呼び出し先**: QualifiedNameGetCreationNamespace, object_aclcheck, pg_char_to_encoding, LookupFuncName, get_func_rettype, OidFunctionCall6, ConversionCreate

関連データ構造

  • CreateConversionStmt: SQL構文解析結果を保持する構造体。
  • ObjectAddress: 作成されたオブジェクトのカタログ上の位置情報。

失敗モードと境界条件

  • スキーマが存在しない、または権限がない場合にエラー。
  • 未サポートのエンコーディング名が指定された場合にエラー。
  • SQL_ASCII を含む変換定義はエラー。
  • 変換関数が見つからない、または戻り値型が不適切な場合にエラー。
  • 変換関数が空文字列の入力に対して不正な結果を返した場合にエラー。

保守担当者向けメモ

  • 変換関数は (int4, int4, cstring, internal, int4, bool) という固定のシグネチャを要求する。
  • SQL_ASCII の制限は、PostgreSQL内部の高速パスとの競合を避けるための仕様である。
