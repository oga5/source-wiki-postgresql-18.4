foreign_expr_walker

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/deparse.c:309

目的

式ツリーを再帰的に走査し、その式がリモートの PostgreSQL サーバーで安全に実行可能（shippable）であるかを判定します。

シグネチャまたは形状

static bool foreign_expr_walker(Node *node,
                                foreign_glob_cxt *glob_cxt,
                                foreign_loc_cxt *outer_cxt,
                                foreign_loc_cxt *case_arg_cxt)

入力と出力

  • **入力**:
  • node: 走査対象の式ノード。
  • glob_cxt: リモートサーバー情報やリレーション情報を含むグローバルコンテキスト。
  • outer_cxt: 親ノードから引き継がれる照合順序（Collation）状態。
  • case_arg_cxt: CASE 式の引数に関するコンテキスト。
  • **出力**:
  • bool: 式がリモート実行可能であれば true、そうでなければ false を返します。

前提条件と不変条件

  • node が NULL の場合は常に true を返します。
  • リモートテーブルに属さない Var や、照合順序がデフォルトではない Const は、文脈に応じて安全性が判定されます。
  • システム列（ctid を除く）はリモート送信不可とみなされます。

アルゴリズムまたは振る舞い

1. **ノード種別による分岐**: nodeTag(node) に基づき処理を振り分けます。 2. **T_Var の処理**:

  • リモートテーブルの列であれば、照合順序の安全性を確認します。
  • 他のテーブルの列であれば、デフォルト照合順序以外は原則として「安全ではない」とマークします。

    • 3. **T_Const の処理**:

  • regproc, regclass 等のシステムカタログ参照型は、is_shippable を用いてリモートサーバーでの利用可否を検証します。
  • TSConfig 等の特定の型は、FirstNormalObjectId 未満の OID であれば出荷可能とみなす緩和ルールを適用します。

    • 4. **再帰的走査**: 子ノードが存在する場合、inner_cxt を用いて再帰的に走査を行い、すべての部分式が安全であることを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**: is_foreign_expr および deparse.c 内の各種ノード処理関数（SubscriptingRef, FuncExpr, OpExpr 等の走査時）。
  • **呼び出し先**: is_shippable（定数の出荷可能性判定）、bms_is_member（リレーション判定）、および再帰的な自身（foreign_expr_walker）。

関連データ構造

  • foreign_glob_cxt: 走査全体で共有されるコンテキスト。
  • foreign_loc_cxt: ツリーの階層ごとに保持される照合順序状態。
  • PgFdwRelationInfo: fpinfo として参照され、リモートサーバーのメタデータを提供します。
  • FDWCollateState: 照合順序の安全性を表す列挙型。

失敗モードと境界条件

  • **MULTIEXPR パラメータ**: T_Param において MULTIEXPR 型が検出された場合、リモート実行不可として false を返します。
  • **システム列**: ctid 以外のシステム列はリモート実行不可と判定されます。
  • **照合順序の不一致**: 照合順序が不明またはデフォルト以外である場合、文脈に応じて安全性が否定されます。

保守担当者向けメモ

  • 新しいノード型が PostgreSQL コアに追加された場合、本関数に適切なハンドラを追加する必要があります。
  • is_shippable の判定ロジックは shippable.c に依存しているため、出荷可能性の基準を変更する場合はそちらも確認してください。
