ExecSetVariableStmt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc_funcs.c:42

目的

SQLの SET コマンド（および関連する変数設定操作）を実行し、GUC（Grand Unified Configuration）パラメータの更新やトランザクション制御を行うためのエントリポイントです。

シグネチャまたは形状

void ExecSetVariableStmt(VariableSetStmt *stmt, bool isTopLevel)

入力と出力

  • **入力**:
  • stmt: 構文解析された VariableSetStmt 構造体へのポインタ。
  • isTopLevel: 現在の文がトップレベルで実行されているかを示すフラグ。
  • **出力**: なし（副作用としてGUC設定の変更やエラー発生を行う）。

前提条件と不変条件

  • 並列モード（IsInParallelMode()）実行中は、パラメータの変更は禁止されており、エラーが発生します。
  • SET LOCAL を含む一部の操作は、トランザクションブロック内での実行が求められ、そうでない場合は WarnNoTransactionBlock により警告されます。

アルゴリズムまたは振る舞い

1. **アクション決定**: stmt->is_local に基づき、GUC_ACTION_LOCAL または GUC_ACTION_SET を設定します。 2. **並列モードチェック**: 並列実行中であればエラーを報告します。 3. **ステートメント種別の分岐**:

  • VAR_SET_VALUE / VAR_SET_CURRENT: set_config_option を呼び出し、値を更新します。
  • VAR_SET_MULTI: TRANSACTION、SESSION CHARACTERISTICS、TRANSACTION SNAPSHOT の各特殊ケースを処理します。SetPGVariable を介して個別の設定を行うか、ImportSnapshot を実行します。
  • VAR_SET_DEFAULT / VAR_RESET: set_config_option を使用してデフォルト値へのリセットを行います。
  • VAR_RESET_ALL: ResetAllOptions を呼び出し、全設定をリセットします。

    • 4. **フック実行**: 最後に InvokeObjectPostAlterHookArgStr を呼び出し、パラメータ変更後のフック処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:875
  • **呼び出し先**: set_config_option, ExtractSetVariableArgs, WarnNoTransactionBlock, SetPGVariable, ImportSnapshot, ResetAllOptions, InvokeObjectPostAlterHookArgStr

関連データ構造

  • VariableSetStmt: SQLの SET 文の解析結果を保持する構造体。
  • GucAction: 設定適用のアクション種別（LOCAL/SET）。

失敗モードと境界条件

  • **並列モード**: ERRCODE_INVALID_TRANSACTION_STATE を発生させます。
  • **不正な要素**: VAR_SET_MULTI において予期しない名前が渡された場合、elog(ERROR) で停止します。

保守担当者向けメモ

  • 本関数はSQLインターフェースの窓口であり、実際の値の検証や保持ロジックは guc.c の set_config_option に依存しています。
  • VAR_SET_MULTI の分岐は、特定のSQL構文（SET TRANSACTION 等）に強く依存しているため、構文の拡張時には整合性に注意してください。
