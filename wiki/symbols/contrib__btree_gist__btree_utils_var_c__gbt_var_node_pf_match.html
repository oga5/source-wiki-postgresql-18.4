gbt_var_node_pf_match

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_var.c:200

目的

GiST インデックスの検索処理において、ノードのキー範囲がクエリ条件と共通プレフィックス（Prefix）を共有しているかを判定し、インデックスの枝刈りを最適化します。

シグネチャまたは形状

static bool gbt_var_node_pf_match(const GBT_VARKEY_R *node, const bytea *query, const gbtree_vinfo *tinfo)

入力と出力

  • **入力**:
  • node: 比較対象となる GiST ノードの範囲（下限および上限）。
  • query: 検索条件となる bytea 型のデータ。
  • tinfo: 可変長型ごとの比較・変換ロジックを保持する gbtree_vinfo 構造体。
  • **出力**:
  • bool: 共通プレフィックスによる一致が認められる場合は true、それ以外は false。

前提条件と不変条件

  • tinfo->trnc が真であること（プレフィックス圧縮が有効であること）が判定の前提となります。

アルゴリズムまたは振る舞い

1. tinfo->trnc が偽であれば、直ちに false を返します。 2. tinfo->trnc が真の場合、ノードの lower（下限）または upper（上限）のいずれかが gbt_bytea_pf_match を用いて query とプレフィックス一致するかを評価します。 3. いずれか一方でも一致すれば true を返し、一致しなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_utils_var.c 内の gbt_var_consistent 関数から複数回呼び出されています（行 581, 588, 597, 604, 611）。
  • **呼び出し先**: gbt_bytea_pf_match を呼び出します。

関連データ構造

  • GBT_VARKEY_R: 可変長キーの範囲（下限・上限）を保持する構造体。
  • gbtree_vinfo: 可変長型ごとの動作を定義する設定構造体。
  • bytea: PostgreSQL の可変長バイナリデータ型。

失敗モードと境界条件

  • tinfo が NULL の場合や、node 内のポインタが不正な場合は未定義の動作となります。
  • プレフィックス圧縮が適用されていないデータ型に対しては、常に false を返します。

保守担当者向けメモ

  • 本関数は gbt_var_consistent による検索の枝刈りロジックの一部です。
  • gbt_bytea_pf_match の実装に依存しているため、プレフィックスの照合ロジックを変更する場合は両方の整合性を確認してください。
