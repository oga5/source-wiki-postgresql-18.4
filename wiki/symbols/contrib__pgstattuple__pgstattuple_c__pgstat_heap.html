pgstat_heap

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:315

目的

ヒープテーブルを物理的に走査し、ライブタプルおよびデッドタプルの数、サイズ、および空き容量を算出する。

シグネチャまたは形状

static Datum pgstat_heap(Relation rel, FunctionCallInfo fcinfo)

入力と出力

  • **入力**:
  • rel: 統計対象のリレーション（Relation）。
  • fcinfo: SQL 関数呼び出し情報（FunctionCallInfo）。
  • **出力**:
  • Datum: 集計結果を格納した複合型（pgstattuple_type 構造体を変換したもの）。

前提条件と不変条件

  • 対象リレーションはヒープアクセスメソッドを使用している必要がある（シーケンスを除く）。
  • 走査中は AccessShareLock を保持する。
  • 同期スキャン（Syncscan）は無効化され、ブロック 0 から順次走査される。

アルゴリズムまたは振る舞い

1. table_beginscan_strat を用いてスキャンを開始する。 2. heap_getnext で全タプルを走査し、HeapTupleSatisfiesVisibility で生存状態を判定して stat 構造体に集計する。 3. タプル走査と並行して、未走査のページに対して PageGetExactFreeSpace を実行し、空き容量を算出する。 4. 全ページの走査完了後、relation_close でロックを解放する。 5. build_pgstattuple_type を呼び出し、集計結果を Datum に変換して返す。 6. 各ループ内で CHECK_FOR_INTERRUPTS() を実行し、クエリキャンセルに対応する。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat_relation（contrib/pgstattuple/pgstattuple.c:260）。
  • **呼び出し先**: table_beginscan_strat, heap_getnext, LockBuffer, HeapTupleSatisfiesVisibility, ReadBufferExtended, PageGetExactFreeSpace, table_endscan, relation_close, build_pgstattuple_type。

関連データ構造

  • pgstattuple_type: 統計情報を保持する構造体。
  • HeapScanDesc: ヒープスキャン状態を保持する構造体。
  • SnapshotData: タプルの可視性判定に使用するスナップショット。

失敗モードと境界条件

  • ヒープ以外のアクセスメソッドが指定された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーを報告する。
  • 大規模なテーブルに対して実行すると、全ページを物理的に読み込むため I/O 負荷が高くなる。

保守担当者向けメモ

  • 本関数は AccessShareLock を保持して実行されるため、長時間実行される可能性がある。
  • pgstattuple バージョン 1.5 以降、実行権限が制限されている点に注意すること。
