timestamp

定義

  • 種別: typedef
  • 定義箇所: src/interfaces/ecpg/include/pgtypes_timestamp.h:10

目的

ECPG（Embedded C for PostgreSQL）アプリケーションにおいて、データベースのタイムスタンプ型をC言語の変数として扱うためのデータ型定義です。

シグネチャまたは形状

typedef int64 timestamp;

入力と出力

  • **入力**: データベースから取得したタイムスタンプ値、またはC言語プログラム内で生成された int64 値。
  • **出力**: ECPGライブラリの変換関数（PGTYPEStimestamp_to_asc 等）を通じて文字列や他の形式へ変換される際の基底データ。

前提条件と不変条件

  • 内部的には int64 型として表現されます。
  • ECPGライブラリのタイムスタンプ関連関数（PGTYPEStimestamp_from_asc, PGTYPEStimestamp_add_interval 等）で利用されることを前提としています。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、直接的なアルゴリズムは持ちません。ECPGライブラリが提供する関数群を通じて、文字列との相互変換や interval 型との算術演算が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGアプリケーションコード。
  • **呼び出し先**: 本シンボル自体は型定義であるため、直接的な呼び出し先はありません。関連する関数として src/interfaces/ecpg/pgtypeslib/timestamp.c に実装されたタイムスタンプ操作関数群が利用されます。
  • **注記**: ツールによる参照調査では utils/timestamp.h への言及がありますが、これらはPostgreSQLサーバー内部のコードであり、本ヘッダーファイル（src/interfaces/ecpg/include/pgtypes_timestamp.h）の直接的な呼び出し元であるという証拠は提供されていません。

関連データ構造

  • TimestampTz: タイムゾーン付きタイムスタンプを保持する int64 型のエイリアス。
  • interval: タイムスタンプの演算に使用される型（pgtypes_interval.h で定義）。

失敗モードと境界条件

  • int64 型の範囲を超える値の代入や演算を行った場合、オーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本型はC++からの利用を考慮し、extern "C" ブロック内で定義されることが推奨されます。
  • 内部表現が int64 であるため、プラットフォーム間でのサイズ整合性に注意してください。
