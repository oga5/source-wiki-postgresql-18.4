RegisterCustomRmgr

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/rmgr.c:106

目的

拡張機能が独自のWAL（Write-Ahead Log）処理を行うためのリソースマネージャ（Resource Manager）をシステムに登録します。

シグネチャまたは形状

void RegisterCustomRmgr(RmgrId rmid, const RmgrData *rmgr)

入力と出力

  • **入力**:
  • rmid: 登録するリソースマネージャのID（RM_MIN_CUSTOM_ID から RM_MAX_CUSTOM_ID の範囲内である必要がある）。
  • rmgr: 登録するリソースマネージャの定義を含む RmgrData 構造体へのポインタ。
  • **出力**: なし（成功時は RmgrTable に登録され、ログが出力される）。

前提条件と不変条件

  • rmgr->rm_name は空文字列であってはならない。
  • rmid はカスタムリソースマネージャとして有効な範囲内であること。
  • 本関数は shared_preload_libraries による初期化プロセス中にのみ呼び出し可能。
  • 指定された rmid および rm_name は、既存の登録済みリソースマネージャと重複してはならない。

アルゴリズムまたは振る舞い

1. rm_name の妥当性を検証し、無効な場合はエラーを報告する。 2. rmid がカスタム範囲内か確認し、範囲外であればエラーを報告する。 3. process_shared_preload_libraries_in_progress フラグを確認し、初期化フェーズ外であればエラーを報告する。 4. 指定された rmid が既に RmgrTable で使用されていないか確認する。 5. 全既存リソースマネージャを走査し、同じ名前のマネージャが存在しないか確認する。 6. 検証を通過した場合、RmgrTable[rmid] に rmgr の内容をコピーして登録を完了し、ログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_custom_rmgrs/test_custom_rmgrs.c（テストモジュールによる直接呼び出しの証拠あり）。
  • **呼び出し先**:
  • ereport: エラー報告およびログ出力に使用。
  • strlen, pg_strcasecmp: 文字列操作および比較に使用。

関連データ構造

  • RmgrData: リソースマネージャの定義（名前、Redo関数、記述関数など）を保持する構造体。
  • RmgrTable: 全リソースマネージャを保持するグローバル配列。

失敗モードと境界条件

  • **エラー**:
  • 名前が空またはNULLの場合。
  • IDがカスタム範囲外の場合。
  • shared_preload_libraries 以外で呼び出された場合。
  • 指定したIDが既に登録済みの場合。
  • 指定した名前が既に登録済みの場合。

保守担当者向けメモ

  • 本関数はリカバリ開始前に全マネージャが確実に登録されていることを保証するため、shared_preload_libraries 経由での登録を強制しています。
  • RmgrId は uint8 型であり、最大256個までしか定義できないため、IDの割り当てには注意が必要です。
