PgStat_BgWriterStats

定義

  • 種別: struct
  • 定義箇所: src/include/pgstat.h:238

目的

PostgreSQL のバックグラウンドライター（Background Writer）プロセスに関する統計情報を保持・管理するためのデータ構造です。

シグネチャまたは形状

typedef struct PgStat_BgWriterStats
{
	PgStat_Counter buf_written_clean;
	PgStat_Counter maxwritten_clean;
	PgStat_Counter buf_alloc;
	TimestampTz stat_reset_timestamp;
} PgStat_BgWriterStats;

入力と出力

  • **入力**: 各フィールドは PgStat_Counter（int64）型および TimestampTz 型であり、バックグラウンドライターの動作に伴い更新されます。
  • **出力**: pgstat_fetch_stat_bgwriter() 関数を通じて、統計ビュー等で利用可能な形式で取得されます。

前提条件と不変条件

  • buf_written_clean、maxwritten_clean、buf_alloc は累積カウンタであり、負の値にはなりません。
  • stat_reset_timestamp は統計が最後にリセットされた時刻を保持します。

アルゴリズムまたは振る舞い

バックグラウンドライターがバッファをクリーンアップした回数、クリーンアップの制限に達した回数、およびバッファを割り当てた回数を追跡します。これらの値は共有メモリ上の統計情報として蓄積され、必要に応じてリセットや取得が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_bgwriter.c 内で、統計情報の初期化、更新、および共有メモリへの反映に使用されています。
  • **呼び出し先**: 統計取得関数 pgstat_fetch_stat_bgwriter() が本構造体を返します。
  • **その他**: src/include/utils/pgstat_internal.h にて、共有メモリ上の統計管理構造体の一部として参照されています。

関連データ構造

  • PgStat_Counter: カウンタ値に使用される型。
  • TimestampTz: 時刻管理に使用される型。

失敗モードと境界条件

  • 統計情報の取得時に共有メモリの状態が不整合である場合、不正確な値が返される可能性があります。
  • カウンタが int64 の最大値を超えた場合のオーバーフロー挙動については、標準的な整数演算のルールに従います。

保守担当者向けメモ

  • 本構造体のメンバを変更した場合は、統計ファイルのフォーマット互換性に影響を与える可能性があるため、PGSTAT_FILE_FORMAT_ID の更新を検討してください。
  • 統計の集計ロジックについては src/backend/utils/activity/pgstat_bgwriter.c を参照してください。
