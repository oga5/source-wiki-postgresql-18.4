PLy_exec_function

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_exec.c:52

目的

SQL 関数および集合を返す関数（SRF）の実行を制御する、PL/Python のメインエントリーポイントです。Python コードの実行環境構築、引数の管理、および PostgreSQL と Python 間のデータ変換を統括します。

シグネチャまたは形状

Datum PLy_exec_function(FunctionCallInfo fcinfo, PLyProcedure *proc)

入力と出力

  • **入力**:
  • fcinfo: PostgreSQL の関数呼び出し情報。
  • proc: 実行対象の PL/Python プロシージャ定義。
  • **出力**:
  • Datum: Python コードの実行結果を PostgreSQL のデータ型に変換した値。

前提条件と不変条件

  • 再帰呼び出し時には、PLy_global_args_push により外部レベルの引数がスタックに退避されている必要があります。
  • PG_TRY ブロック内で実行され、エラー発生時には PG_CATCH を通じて SPI 接続の切断やメモリ解放が保証されます。
  • SRF の場合、funcctx->multi_call_memory_ctx に PLySRFState が割り当てられ、クリーンアップコールバックが登録されます。

アルゴリズムまたは振る舞い

1. **引数退避**: PLy_global_args_push を呼び出し、再帰呼び出しに備えてグローバル辞書の状態を保存します。 2. **SRF 初期化**: 集合を返す関数の場合、初回呼び出し時に PLySRFState を作成し、イテレータを管理します。 3. **実行**:

  • 初回または非 SRF の場合、PLy_function_build_args で引数を構築し、PLy_procedure_call で Python コードを実行します。
  • SRF の継続呼び出し時は、PLy_function_restore_args で引数を復元します。

    • 4. **イテレーション**: SRF の場合、PyObject_GetIter でイテレータを作成し、PyIter_Next で値を順次取得します。 5. **終了処理**: SPI_finish を呼び出し、戻り値を PostgreSQL の Datum に変換します。void 型の場合は None であることを検証します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_main.c (直接の呼び出しを確認)
  • **呼び出し先**:
  • PLy_global_args_push / PLy_global_args_pop
  • PLy_function_build_args
  • PLy_procedure_call
  • PLy_function_restore_args / PLy_function_save_args
  • SPI_finish
  • PLy_elog

関連データ構造

  • PLyProcedure: プロシージャの定義とキャッシュ状態を保持。
  • PLySRFState: SRF のイテレータ状態および保存された引数を保持。
  • FunctionCallInfo: PostgreSQL 側の関数呼び出し情報。

失敗モードと境界条件

  • **型不一致**: 戻り値が期待される型と異なる場合（例: void 関数が None 以外を返す）、ereport(ERROR) を発生させます。
  • **イテレーション失敗**: SRF でイテレータが取得できない、または取得中にエラーが発生した場合、ereport(ERROR) を発生させます。
  • **SPI 失敗**: SPI_finish が正常終了しない場合、elog(ERROR) を発生させます。

保守担当者向けメモ

  • PG_TRY ブロック内での Py_DECREF 漏れはメモリリークの原因となるため、修正時には参照カウントの管理に注意してください。
  • SRF のイテレータは MemoryContextRegisterResetCallback を使用して解放されるため、メモリコンテキストのライフサイクルと整合性を保つ必要があります。
