pgstat_bktype_io_stats_valid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_io.c:36

目的

指定されたバックエンドタイプのI/O統計構造体（PgStat_BktypeIO）の内容が、現在の統計ルールに照らして整合性が取れているかを検証します。

シグネチャまたは形状

bool pgstat_bktype_io_stats_valid(PgStat_BktypeIO *backend_io, BackendType bktype)

入力と出力

  • **入力**:
  • backend_io: 検証対象のI/O統計データ構造へのポインタ。
  • bktype: 統計に関連付けられたバックエンドのタイプ。
  • **出力**:
  • bool: 統計データが妥当であれば true、不整合があれば false を返します。

前提条件と不変条件

  • pgstat_tracks_io_op 関数が定義する統計対象ルールに基づき、データがフィルタリングされている必要があります。
  • I/O時間が記録されている場合、対応する操作回数は必ず1以上でなければなりません。

アルゴリズムまたは振る舞い

1. 全てのI/Oオブジェクト、コンテキスト、操作の組み合わせ（IOOBJECT_NUM_TYPES × IOCONTEXT_NUM_TYPES × IOOP_NUM_TYPES）を走査します。 2. pgstat_tracks_io_op を呼び出し、その組み合わせが統計対象であるかを確認します。 3. **統計対象の場合**: 実行時間が0以外であるにもかかわらず、操作回数が0以下である場合は不整合とみなし false を返します。 4. **統計対象外の場合**: 操作回数が0以外である場合は不整合とみなし false を返します。 5. 全ての走査を通過した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat_io.c (内部アサーション)
  • src/backend/utils/adt/pgstatfuncs.c (統計関数内のアサーション)
  • **呼び出し先**:
  • pgstat_tracks_io_op (統計対象の判定)

関連データ構造

  • PgStat_BktypeIO: バックエンドごとのI/O統計を保持する構造体。
  • BackendType: バックエンドの種類を識別する列挙型。

失敗モードと境界条件

  • 統計対象外の項目に値が設定されている場合、検証は失敗します。
  • 統計対象項目において、時間計測値と回数計測値の間に論理的な矛盾（時間はあるが回数がない）がある場合、検証は失敗します。

保守担当者向けメモ

  • 本関数は主にデバッグや整合性チェックのためのアサーションとして使用されます。
  • 新しいI/O操作やコンテキストを追加した際は、pgstat_tracks_io_op のロジックと併せて本関数の検証範囲が適切か確認してください。
