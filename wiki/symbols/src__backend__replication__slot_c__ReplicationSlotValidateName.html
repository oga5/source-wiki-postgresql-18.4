ReplicationSlotValidateName

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slot.c:263

目的

レプリケーションスロット名がシステム上の命名規則（英数字およびアンダースコアのみ）に適合しているかを検証し、不適合な場合にエラー報告を行う。

シグネチャまたは形状

bool ReplicationSlotValidateName(const char *name, int elevel)

入力と出力

  • **入力**:
  • name: 検証対象のスロット名（文字列）。
  • elevel: 検証失敗時に使用するエラーレベル（ERROR, DEBUG2 等）。
  • **出力**:
  • bool: 名前が有効であれば true、無効であれば false を返す。

前提条件と不変条件

  • 内部検証ロジック ReplicationSlotValidateNameInternal に依存している。
  • ReplicationSlotValidateNameInternal が生成するエラーメッセージは翻訳済みであるため、本関数では errmsg_internal を使用して二重翻訳を回避する。

アルゴリズムまたは振る舞い

1. ReplicationSlotValidateNameInternal を呼び出し、名前の妥当性を確認する。 2. 検証に失敗した場合:

  • 指定された elevel で ereport を発行し、エラーメッセージとヒント（存在する場合）を報告する。
  • 内部で割り当てられたエラーメッセージ用メモリを pfree で解放する。
  • false を返す。

    • 3. 検証に成功した場合、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/replication/slot.c
  • **呼び出し先**:
  • ReplicationSlotValidateNameInternal（ソースコード上の直接の依存先）

関連データ構造

  • 特になし（標準的な文字列およびエラー報告用構造体を使用）。

失敗モードと境界条件

  • 名前が命名規則（[a-z0-9_]）に違反している場合、elevel に応じてエラーまたは警告が発生する。
  • メモリ不足等の理由で ReplicationSlotValidateNameInternal がエラー情報を生成できない場合の挙動は、当該内部関数の実装に依存する。

保守担当者向けメモ

  • 本関数は検証ロジックのラッパーであり、実際のルール変更は ReplicationSlotValidateNameInternal で行うこと。
  • エラーメッセージの二重翻訳を避けるため、errmsg_internal を使用する設計を維持すること。
