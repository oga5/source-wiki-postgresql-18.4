execute_attr_map_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tupconvert.c:153

目的

既存の HeapTuple を、指定された TupleConversionMap に基づいて新しい構造（TupleDesc）を持つタプルへ変換する。

シグネチャまたは形状

HeapTuple execute_attr_map_tuple(HeapTuple tuple, TupleConversionMap *map)

入力と出力

  • **入力**:
  • tuple: 変換元の HeapTuple。
  • map: 変換ルールとワークスペースを保持する TupleConversionMap 構造体。
  • **出力**:
  • 変換後の新しい HeapTuple を返す。

前提条件と不変条件

  • map->attrMap->maplen は map->outdesc->natts と一致しなければならない（Assert による検証）。
  • map 内の invalues/inisnull 配列は、heap_deform_tuple の仕様に合わせてインデックス 1 から属性値が格納されるようオフセットされている必要がある。

アルゴリズムまたは振る舞い

1. heap_deform_tuple を使用し、入力タプルの属性値を map 内のワークスペース（invalues および inisnull）へ展開する。 2. attrMap に定義された対応関係に従い、入力側の属性値を新しいタプル用のワークスペース（outvalues および outisnull）へ転送する。

  • attrMap->attnums[i] が 0 の場合、対応する属性は存在しない（削除済み）とみなされ、NULL がセットされる。

    • 3. heap_form_tuple を呼び出し、map->outdesc に基づいて新しいタプルを構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c
  • src/backend/executor/execExprInterp.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • heap_deform_tuple (PostgreSQL 内部関数)
  • heap_form_tuple (PostgreSQL 内部関数)

関連データ構造

  • HeapTuple: 変換対象および変換結果のタプル構造。
  • TupleConversionMap: 変換に必要な属性マッピング情報（AttrMap）と、変換用ワークスペース（Datum 配列等）を保持する。

失敗モードと境界条件

  • map が指す TupleDesc と入力タプルの構造に不整合がある場合、heap_deform_tuple 等の内部関数でエラーが発生する可能性がある。
  • 変換後のタプル構築時にメモリ不足が発生した場合、heap_form_tuple がエラーを投げる。

保守担当者向けメモ

  • 本関数は HeapTuple 専用である。TupleTableSlot を扱う場合は execute_attr_map_slot を使用すること。
  • map 内の TupleDesc は参照のみであり、本関数内で解放してはならない。
  • attrMap のインデックス 0 は NULL を表すため、配列操作時にはオフセットの扱いに注意が必要である。
