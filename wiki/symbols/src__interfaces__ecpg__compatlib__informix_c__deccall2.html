deccall2

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/compatlib/informix.c:47

目的

Informix の decimal 型に対する演算を、PostgreSQL の numeric 型を用いた演算へ委譲するための内部ラッパー関数です。

シグネチャまたは形状

static int deccall2(decimal *arg1, decimal *arg2, int (*ptr) (numeric *, numeric *))

入力と出力

  • **入力**:
  • arg1, arg2: 演算対象となる decimal 型へのポインタ。
  • ptr: 2つの numeric 型ポインタを引数に取り、int を返す関数ポインタ。
  • **出力**:
  • 成功時: ptr が返す演算結果（int）。
  • 失敗時: ECPG_INFORMIX_OUT_OF_MEMORY（メモリ確保失敗時）。

前提条件と不変条件

  • arg1 および arg2 は有効な decimal 構造体を指している必要があります。
  • ptr に渡される関数は、numeric 型の演算を実行する PGTYPESnumeric ライブラリの関数である必要があります。

アルゴリズムまたは振る舞い

1. PGTYPESnumeric_new() を使用して、2つの numeric 型一時領域（a1, a2）を確保します。 2. PGTYPESnumeric_from_decimal() を呼び出し、入力された decimal 型の値を numeric 型へ変換します。 3. 変換に成功した場合、関数ポインタ ptr を介して a1 と a2 を引数に演算を実行します。 4. 演算結果を保持したまま、確保した numeric 型領域を PGTYPESnumeric_free() で解放します。 5. 演算結果を呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/compatlib/informix.c 内の deccmp 関数から呼び出されていることが確認されています。
  • **呼び出し先**:
  • PGTYPESnumeric_new
  • PGTYPESnumeric_from_decimal
  • PGTYPESnumeric_free
  • ptr で指定された関数（例: PGTYPESnumeric_cmp）

関連データ構造

  • decimal: Informix 互換の数値型。
  • numeric: PostgreSQL 内部の数値型。

失敗モードと境界条件

  • PGTYPESnumeric_new によるメモリ確保が失敗した場合、ECPG_INFORMIX_OUT_OF_MEMORY を返します。
  • PGTYPESnumeric_from_decimal による型変換が失敗した場合、確保済みのメモリを解放した上で ECPG_INFORMIX_OUT_OF_MEMORY を返します。

保守担当者向けメモ

  • 本関数は decimal 型の演算を numeric 型に変換して処理する共通パターンを実装しています。同様の処理を行う deccall3 と対になる構造です。
  • メモリリークを防ぐため、変換失敗時のエラーパスにおいて必ず PGTYPESnumeric_free が呼び出されるよう設計されています。
