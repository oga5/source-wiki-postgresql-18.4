range_lower

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes.c:447

目的

範囲型（RangeType）から下限値を取得します。

シグネチャまたは形状

Datum range_lower(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される RangeType 型のデータ。
  • **出力**: 範囲の下限値（Datum）。範囲が空であるか、下限値が無限大（infinite）の場合は NULL を返します。

前提条件と不変条件

  • 入力された RangeType は、操作前にデトースト（detoast）されている必要があります。
  • 内部的に range_get_typcache を使用するため、型キャッシュが有効である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_RANGE_P(0) を使用して、入力された範囲型データを取得します。 2. range_get_typcache を呼び出し、対象範囲型のメタデータを取得します。 3. range_deserialize を使用して、範囲型データを RangeBound 構造体（下限・上限・空フラグ）に展開します。 4. 範囲が空であるか、下限が無限大（lower.infinite）であるかを確認します。 5. 条件に該当する場合は PG_RETURN_NULL() を返します。 6. それ以外の場合は、lower.val に格納されている下限値を PG_RETURN_DATUM() で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**:
  • range_get_typcache
  • range_deserialize
  • PG_GETARG_RANGE_P（マクロ）
  • PG_RETURN_NULL（マクロ）
  • PG_RETURN_DATUM（マクロ）

関連データ構造

  • RangeType: 範囲型を表現する可変長データ構造。
  • RangeBound: 範囲の境界（下限・上限）を保持する構造体。
  • TypeCacheEntry: 範囲型の型情報キャッシュ。

失敗モードと境界条件

  • **空の範囲**: empty が真の場合、下限値は存在しないため NULL を返します。
  • **無限大の下限**: lower.infinite が真の場合、有限の下限値を持たないため NULL を返します。

保守担当者向けメモ

  • 本関数は範囲型のメタデータに依存するため、型キャッシュの整合性が重要です。
  • 境界値の比較や判定には、直接メンバにアクセスするのではなく、提供されている RangeBound 構造体のフラグ（infinite 等）を適切に使用してください。
