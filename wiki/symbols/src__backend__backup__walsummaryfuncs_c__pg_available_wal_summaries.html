pg_available_wal_summaries

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummaryfuncs.c:31

目的

pg_wal/summaries ディレクトリ内に存在する利用可能なWALサマリーファイルのリストを、SQLクエリの結果セットとして取得します。

シグネチャまたは形状

Datum pg_available_wal_summaries(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (SQL関数インターフェース経由)
  • **出力**: Datum (SRFとして、TLI、開始LSN、終了LSNを含むタプルのセットを返す)

前提条件と不変条件

  • InitMaterializedSRF を呼び出し、結果を格納するための tuplestore が初期化されている必要があります。
  • 処理中、CHECK_FOR_INTERRUPTS() が呼び出され、クエリのキャンセル要求を監視します。

アルゴリズムまたは振る舞い

1. InitMaterializedSRF を使用して、結果セットを保持する tuplestore を準備します。 2. GetWalSummaries(0, InvalidXLogRecPtr, InvalidXLogRecPtr) を呼び出し、利用可能なサマリーファイルのリストを取得します。 3. 取得したリストを反復処理し、各サマリーファイルについて以下の操作を行います。

  • CHECK_FOR_INTERRUPTS() を実行し、割り込みを確認します。
  • サマリーのTLI、開始LSN、終了LSNを抽出します。
  • heap_form_tuple を用いてタプルを生成し、tuplestore_puttuple で結果セットに追加します。

    • 4. 最後に (Datum) 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SQL関数として呼び出される）。
  • **呼び出し先**:
  • InitMaterializedSRF
  • GetWalSummaries
  • CHECK_FOR_INTERRUPTS
  • heap_form_tuple
  • tuplestore_puttuple

関連データ構造

  • WalSummaryFile: サマリーファイルのメタデータ（TLI、開始LSN、終了LSN）を保持する構造体。
  • ReturnSetInfo: SRFの戻り値情報を管理する構造体。

失敗モードと境界条件

  • サマリーファイルが存在しない場合、空の結果セットが返されます。
  • 大量のサマリーファイルが存在する場合、tuplestore への格納に伴うメモリ消費が発生します。
  • 実行中にユーザーがクエリをキャンセルした場合、CHECK_FOR_INTERRUPTS() により処理が中断されます。

保守担当者向けメモ

  • 本関数はSRF（Set Returning Function）として実装されており、メモリ管理は tuplestore に依存しています。
  • NUM_WS_ATTS 定数に基づいた列定義が想定されています。
  • GetWalSummaries の引数（0, InvalidXLogRecPtr, InvalidXLogRecPtr）は、全サマリーを取得するためのデフォルト値として使用されています。
