ExecGetJunkAttribute

定義

  • 種別: function
  • 定義箇所: src/include/executor/executor.h:220

目的

TupleTableSlot から「ジャンク属性（Junk Attribute）」と呼ばれる、クエリ結果には出力されないが実行時に必要な補助的な属性値を取得するために使用されます。

シグネチャまたは形状

static inline Datum ExecGetJunkAttribute(TupleTableSlot *slot, AttrNumber attno, bool *isNull)

入力と出力

  • **入力**:
  • slot: 対象の TupleTableSlot。
  • attno: 取得する属性の番号（AttrNumber）。
  • isNull: 値が NULL かどうかを格納するためのブール値ポインタ。
  • **出力**:
  • 戻り値: 取得した属性値を表す Datum。

前提条件と不変条件

  • attno > 0 であること（アサーションにより強制）。

アルゴリズムまたは振る舞い

1. attno が正の値であることを確認する。 2. slot_getattr を呼び出し、指定されたスロットから属性値を取得して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/executor/execMain.c
  • src/backend/executor/nodeLockRows.c
  • src/backend/executor/nodeModifyTable.c
  • **呼び出し先**:
  • slot_getattr（スロットから属性値を抽出する低レベル関数）

関連データ構造

  • TupleTableSlot: タプルを保持する実行時のデータ構造。
  • Datum: PostgreSQLの汎用データ型。
  • AttrNumber: 属性番号を識別する型。

失敗モードと境界条件

  • attno が 0 以下の場合、アサーション違反により異常終了する可能性があります。
  • slot が無効な場合や、attno がスロット内の有効な属性範囲外である場合の挙動は slot_getattr の仕様に依存します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを重視して頻繁に呼び出されることを想定しています。
  • 実行エンジン内のジャンクフィルタリング処理において、特定の属性値を抽出する際の標準的なインターフェースとして機能しています。
