gbt_cashge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_cash.c:34

目的

money 型の値に対して「以上（greater than or equal to）」の比較を行い、GiST インデックスの検索条件適合判定を支援します。

シグネチャまたは形状

static bool gbt_cashge(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の Cash 型値へのポインタ。
  • b: 比較対象の Cash 型値へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • bool: *a >= *b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は Cash 型としてキャスト可能な有効なメモリ領域を指している必要があります。
  • money 型は内部的に 64bit 整数として扱われます。

アルゴリズムまたは振る舞い

1. 引数 a と b を Cash 型のポインタにキャストします。 2. それぞれの指す値を比較演算子 >= で評価します。 3. 評価結果をそのまま戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_cash.c:85 において、tinfo 構造体（gbtree_ninfo）のメンバとして登録されています。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行いません。Cash 型の組み込み比較演算子を利用します。

関連データ構造

  • Cash: money 型を表現するデータ構造。
  • gbtree_ninfo: btree_cash.c 内で定義される tinfo 構造体。本関数はこの構造体を通じて GiST の汎用関数から間接的に利用されます。

失敗モードと境界条件

  • ポインタ a または b が NULL の場合、セグメンテーション違反が発生する可能性があります。
  • money 型の比較ロジックに依存するため、値の範囲外やオーバーフローに関する挙動は money 型の定義に従います。

保守担当者向けメモ

  • 本関数は btree_utils_num.h で提供される数値型 GiST インデックスの枠組みの一部です。
  • ロジックの変更が必要な場合は、btree_utils_num.c で定義されている汎用的な数値処理ロジックとの整合性を確認してください。
