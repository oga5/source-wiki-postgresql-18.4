RelationCacheInitFileRemove

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relcache.c:6899

目的

データベース内の全テーブルスペースおよびデータベースディレクトリを走査し、リレーションキャッシュの初期化ファイル（pg_internal.init）を削除する。

シグネチャまたは形状

void RelationCacheInitFileRemove(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてファイルシステム上の初期化ファイルを削除）

前提条件と不変条件

  • 呼び出し元は、ファイルシステム操作に必要な権限を有している必要がある。
  • PG_TBLSPC_DIR（pg_tblspc）ディレクトリが存在し、アクセス可能であること。

アルゴリズムまたは振る舞い

1. global/pg_internal.init を削除する。 2. デフォルトのテーブルスペースディレクトリ（base）に対して RelationCacheInitFileRemoveInDir を実行する。 3. pg_tblspc ディレクトリを走査し、各テーブルスペース内のデータベースディレクトリに対して RelationCacheInitFileRemoveInDir を再帰的に実行する。 4. 走査に使用したディレクトリハンドルを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/heap/heapam.c
  • src/backend/access/transam/xlog.c
  • 呼び出し先:
  • unlink_initfile
  • RelationCacheInitFileRemoveInDir
  • AllocateDir
  • ReadDirExtended
  • FreeDir

関連データ構造

  • DIR（ディレクトリハンドル）
  • struct dirent（ディレクトリ内のエントリ情報）

失敗モードと境界条件

  • ディレクトリのオープン失敗や読み取り失敗が発生した場合、ReadDirExtended 等の内部関数がエラーを報告する。
  • 削除対象のファイルが存在しない場合、unlink_initfile の実装に依存するが、通常はエラーとして扱われない。

保守担当者向けメモ

  • 本関数は、キャッシュの整合性が失われた際や、システムカタログの更新に伴いキャッシュを強制的に再構築させるために呼び出される。
  • RelationCacheInitFileRemoveInDir が各ディレクトリ内のファイル削除を担当するため、ディレクトリ構造の変更時にはこの連携に注意すること。
