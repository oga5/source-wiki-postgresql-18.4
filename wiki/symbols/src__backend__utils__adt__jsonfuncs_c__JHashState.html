JHashState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonfuncs.c:136

目的

JSONデータ処理関数において、解析中の状態およびハッシュテーブルによる集計情報を保持するための構造体です。

シグネチャまたは形状

typedef struct JHashState
{
	JsonLexContext *lex;           /* JSON解析コンテキスト */
	const char *function_name;     /* 呼び出し元の関数名 */
	HTAB	   *hash;              /* 集計用ハッシュテーブル */
	char	   *saved_scalar;      /* 一時保存されたスカラー値 */
	const char *save_json_start;   /* JSON開始位置のポインタ */
	JsonTokenType saved_token_type; /* 保存されたトークン型 */
} JHashState;

入力と出力

  • 本構造体は palloc0 を介してメモリ確保され、JSON処理関数の実行状態として利用されます。
  • 呼び出し元関数内で JHashState * 型としてキャストされ、状態の受け渡しに使用されます。

前提条件と不変条件

  • lex フィールドは有効な JsonLexContext を指している必要があります。
  • hash フィールドは、使用前に適切なハッシュテーブルとして初期化されている必要があります。

アルゴリズムまたは振る舞い

  • JSONの解析プロセスにおいて、解析器のコールバック関数間で共有される状態を保持します。
  • トークン解析中に特定されたスカラー値やトークン型を一時的に退避（saved_scalar, saved_token_type）し、後続の処理で再利用する仕組みを提供します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonfuncs.c 内のJSON処理関数群（3816, 3827, 3855, 3881, 3932, 3945 行付近で参照）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • JsonLexContext: JSON解析の基盤となるコンテキスト。
  • HTAB: PostgreSQLのハッシュテーブル構造体。
  • JsonTokenType: JSONトークンの種別定義。

失敗モードと境界条件

  • メモリ不足時に palloc0 が失敗する可能性があります。
  • ハッシュテーブルの初期化や操作に失敗した場合、後続の解析処理が異常終了する可能性があります。

保守担当者向けメモ

  • 本構造体は jsonfuncs.c 内の特定の関数群でのみ使用されるプライベートな状態保持用構造体です。
  • 構造体のフィールドを追加・変更する場合は、jsonfuncs.c 内のすべての参照箇所（特にキャストを行っている箇所）で整合性が取れているか確認してください。
