pg_get_timezone_offset

定義

  • 種別: function
  • 定義箇所: src/timezone/localtime.c:1964

目的

指定されたタイムゾーン（pg_tz）が持つすべての時刻情報（ttinfo）において、UTCオフセットが単一の値であるかを確認し、そのオフセット値を取得する。

シグネチャまたは形状

bool pg_get_timezone_offset(const pg_tz *tz, long int *gmtoff)

入力と出力

  • **入力**:
  • tz: タイムゾーン情報を保持する pg_tz 構造体へのポインタ。
  • gmtoff: UTCオフセット（秒単位）を格納するための long int 型へのポインタ。
  • **出力**:
  • 戻り値: すべての ttinfo が同一のオフセットを持つ場合は true、異なるオフセットが含まれる場合は false を返す。
  • gmtoff: 戻り値が true の場合、その共通のオフセット値が書き込まれる。

前提条件と不変条件

  • tz は有効な pg_tz 構造体を指している必要がある。
  • タイムゾーンの内部状態（state）には、少なくとも1つ以上の ttinfo が含まれていることが前提となる。

アルゴリズムまたは振る舞い

1. tz->state を参照し、保持されているすべての ttinfo（sp->ttis）を走査する。 2. インデックス 0 の ttinfo が持つ UTCオフセット（tt_utoff）を基準値とする。 3. インデックス 1 から sp->typecnt まで順次比較を行い、すべての tt_utoff が基準値と一致するかを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/datetime.c:2362
  • src/backend/utils/adt/timestamp.c:6443
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わず、pg_tz 構造体内の state メンバを参照する。

関連データ構造

  • pg_tz: タイムゾーン情報を保持する構造体。
  • state: pg_tz 内に含まれる、タイムゾーンの遷移ルールや時刻情報を管理する構造体。
  • ttinfo: state 内で定義される、特定の期間における時刻属性（オフセット等）を保持する構造体。

失敗モードと境界条件

  • タイムゾーンが歴史的に複数の異なるオフセット（夏時間の有無や過去の変更など）を持つ場合、この関数は false を返す。
  • sp->typecnt が 1 の場合はループが実行されず、常に true が返される。

保守担当者向けメモ

  • 本関数は、タイムゾーンが「固定されたオフセットを持つか」を判定するために使用される。
  • 戻り値が false の場合、gmtoff の値は更新されないため、呼び出し側で戻り値を必ず確認する必要がある。
