string_to_uuid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/uuid.c:130

目的

文字列形式の UUID（標準的な 32 桁の 16 進数、または {} で囲まれた形式）を解析し、内部表現である pg_uuid_t 構造体に変換します。

シグネチャまたは形状

static void string_to_uuid(const char *source, pg_uuid_t *uuid, Node *escontext);

入力と出力

  • **入力**:
  • source: 解析対象の文字列（char *）。
  • uuid: 変換結果を格納する pg_uuid_t 構造体へのポインタ。
  • escontext: エラー処理用のコンテキスト（Node *）。
  • **出力**: なし（成功時は uuid を更新、失敗時は escontext を介してエラーを報告）。

前提条件と不変条件

  • source はヌル終端文字列である必要があります。
  • UUID は 16 バイトのデータとして pg_uuid_t に格納されます。
  • 入力文字列は、ハイフンを含む標準的な UUID 形式、または {} で囲まれた形式である必要があります。

アルゴリズムまたは振る舞い

1. 文字列先頭が { の場合、フラグを立ててスキップします。 2. 16 バイト分（UUID_LEN）のループを実行し、2 文字ずつ 16 進数として読み取ります。 3. 読み取った 2 文字が 16 進数でない場合はエラーとみなします。 4. ハイフン（-）は特定のバイト位置でスキップされます。 5. { で開始した場合は、末尾に } が存在することを確認します。 6. 文字列の末尾に余分な文字がないことを確認します。 7. 構文エラー発生時は ereturn マクロによりエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/uuid.c 内の関数（uuid_in 等）。
  • contrib/uuid-ossp/uuid-ossp.c（外部モジュールからの参照）。
  • **呼び出し先**:
  • isxdigit: 文字が 16 進数か判定。
  • strtoul: 16 進文字列を数値に変換。
  • ereturn: エラー処理用マクロ。
  • memcpy: 文字列バッファのコピー。

関連データ構造

  • pg_uuid_t: UUID を保持する 16 バイトの構造体。
  • Node: エラー処理コンテキスト。

失敗モードと境界条件

  • **構文エラー**: 文字列が UUID 形式に適合しない場合（ERRCODE_INVALID_TEXT_REPRESENTATION を発生）。
  • **境界条件**: 文字列が短すぎる場合、または {} の閉じ忘れ、末尾の不正な文字が含まれる場合にエラーとなります。

保守担当者向けメモ

  • 本関数は uuid_in 等の SQL 型変換関数の内部実装として利用されています。
  • RFC 9562 準拠の UUID 生成ロジックとは異なり、こちらは純粋な文字列解析に特化しています。
  • escontext を使用しているため、呼び出し側でエラーハンドリングの挙動を制御可能です。
