heap_prune_satisfies_vacuum

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/pruneheap.c:916

目的

ヒープページ内のタプルが、VACUUM処理またはオンアクセス・プランニングの観点で削除可能（DEAD）かどうかを判定する。

シグネチャまたは形状

HTSV_Result heap_prune_satisfies_vacuum(PruneState *prstate, HeapTuple tup, Buffer buffer)

入力と出力

  • **入力**:
  • prstate: 剪定処理の状態を保持する構造体。
  • tup: 判定対象のヒープタプル。
  • buffer: タプルが格納されているバッファ。
  • **出力**:
  • HTSV_Result: タプルの可視性状態（HEAPTUPLE_DEAD, HEAPTUPLE_LIVE, HEAPTUPLE_RECENTLY_DEAD 等）。

前提条件と不変条件

  • HeapTupleSatisfiesVacuumHorizon を用いて、タプルの xmax に基づく初期判定を行う。
  • 凍結（Freezing）対象のタプルに対しては、OldestXmin を超えて削除判定を行わない不変条件がある。

アルゴリズムまたは振る舞い

1. HeapTupleSatisfiesVacuumHorizon を呼び出し、基本的な可視性を判定する。 2. 結果が HEAPTUPLE_RECENTLY_DEAD 以外であれば、その結果をそのまま返す。 3. HEAPTUPLE_RECENTLY_DEAD の場合、以下の条件で HEAPTUPLE_DEAD への昇格を試みる：

  • prstate->cutoffs が存在し、タプルの dead_after（xmax）が OldestXmin より古い場合。
  • GlobalVisTestIsRemovableXid を用いて、現在のグローバル可視性状態に基づきタプルが削除可能と判断された場合。

    • 4. 上記いずれにも該当しない場合、元の HEAPTUPLE_RECENTLY_DEAD を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/pruneheap.c 内の剪定処理ループ（heap_page_prune_and_freeze 等から呼び出される）。
  • **呼び出し先**: HeapTupleSatisfiesVacuumHorizon, GlobalVisTestIsRemovableXid。

関連データ構造

  • PruneState: 剪定処理のコンテキスト。
  • HTSV_Result: 可視性判定結果の列挙型。
  • HeapTuple: 判定対象のタプル構造体。

失敗モードと境界条件

  • prstate->cutoffs が NULL の場合、OldestXmin による判定はスキップされる（オンアクセス・プランニング時など）。
  • GlobalVisState の更新状況により、VACUUM開始時よりも後にタプルが削除可能と判定される場合がある。

保守担当者向けメモ

  • 本関数は、VACUUMの整合性を保ちつつ、可能な限り多くのタプルを削除対象として特定するために使用される。
  • HEAPTUPLE_RECENTLY_DEAD を HEAPTUPLE_DEAD に昇格させるロジックは、インデックスの更新を伴わないHOTチェーン剪定の効率に直結する。
