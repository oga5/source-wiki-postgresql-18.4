BufferGetBlock

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/storage/bufmgr.h:383

目的

バッファID（Buffer）から、対応するメモリ上のブロック（ページ）の先頭アドレスを取得します。

シグネチャまたは形状

Block BufferGetBlock(Buffer buffer)

入力と出力

  • **入力**: buffer (Buffer 型) - 取得対象のバッファID。
  • **出力**: Block 型 - バッファ内のページデータが格納されているメモリ上のポインタ。

前提条件と不変条件

  • BufferIsValid(buffer) が真である必要があります（関数内で Assert されています）。
  • 共有バッファの場合、BufferBlocks が初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. 引数の buffer が有効なIDであるか検証します。 2. BufferIsLocal(buffer) を判定します。

  • **ローカルバッファの場合**: LocalBufferBlockPointers 配列から、負のバッファIDをインデックスに変換してポインタを取得します。
  • **共有バッファの場合**: BufferBlocks（共有メモリ上のバッファプール先頭）を基準とし、バッファIDからオフセットを計算してポインタを算出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_buffercache/pg_buffercache_pages.c
  • src/backend/access/heap/heapam.c
  • src/backend/access/transam/xloginsert.c
  • src/backend/storage/buffer/bufmgr.c
  • **呼び出し先**:
  • BufferIsValid (マクロ/インライン関数)
  • BufferIsLocal (マクロ/インライン関数)

関連データ構造

  • Buffer: バッファを識別する整数型。
  • Block: ページデータへのポインタ型。
  • BufferBlocks: 共有バッファプール全体の先頭を指すグローバル変数。
  • LocalBufferBlockPointers: ローカルバッファのブロックポインタを保持する配列。

失敗モードと境界条件

  • 無効なバッファIDを渡した場合、Assert が有効なビルドではプログラムが停止します。
  • 共有バッファとローカルバッファの境界を正しく判定できない場合、誤ったメモリ領域を参照するリスクがあります。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、インライン化されています。変更時は bufmgr.h の依存関係に注意してください。
  • BufferBlocks および LocalBufferBlockPointers のメモリレイアウトに強く依存しているため、バッファ管理の初期化ロジックを変更する際は本関数の計算式が妥当か確認が必要です。
