ProgressCommandType

定義

  • 種別: enum
  • 定義箇所: src/include/utils/backend_progress.h:22

目的

PostgreSQLの長時間実行コマンド（VACUUM, ANALYZE等）の進捗状況を追跡・識別するための列挙型です。

シグネチャまたは形状

typedef enum ProgressCommandType
{
	PROGRESS_COMMAND_INVALID,
	PROGRESS_COMMAND_VACUUM,
	PROGRESS_COMMAND_ANALYZE,
	PROGRESS_COMMAND_CLUSTER,
	PROGRESS_COMMAND_CREATE_INDEX,
	PROGRESS_COMMAND_BASEBACKUP,
	PROGRESS_COMMAND_COPY,
} ProgressCommandType;

入力と出力

  • 入力: なし（列挙定数）
  • 出力: なし

前提条件と不変条件

  • PROGRESS_COMMAND_INVALID は、進捗報告がアクティブでない状態を示します。
  • コマンドの開始時に pgstat_progress_start_command を通じて設定され、終了時に PROGRESS_COMMAND_INVALID にリセットされる必要があります。

アルゴリズムまたは振る舞い

  • バックエンドプロセスが実行中のコマンド種別を共有メモリ上の PgBackendStatus 構造体に保持するために使用されます。
  • 監視ツール（pg_stat_progress_* ビューなど）は、この値に基づいて進捗パラメータの解釈を行います。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/activity/backend_progress.c: pgstat_progress_start_command の引数として使用。
  • src/backend/utils/adt/pgstatfuncs.c: 進捗統計情報の取得時に参照。
  • 呼び出し先: なし（列挙型のため）

関連データ構造

  • PgBackendStatus: st_progress_command フィールドでこの型を保持します。
  • PGSTAT_NUM_PROGRESS_PARAM: 進捗情報のパラメータ配列サイズを定義します。

失敗モードと境界条件

  • 未定義のコマンド種別が設定された場合、進捗情報の読み取り側で予期しない動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しい長時間実行コマンドの進捗追跡を追加する場合は、本列挙型に新しい定数を追加し、src/include/commands/progress.h でパラメータのインデックス定義を割り当ててください。
