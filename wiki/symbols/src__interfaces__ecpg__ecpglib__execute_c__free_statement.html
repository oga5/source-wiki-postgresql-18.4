free_statement

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:95

目的

SQL 文の実行に関連する statement 構造体およびその内部で保持されている動的メモリを解放する。

シグネチャまたは形状

static void free_statement(struct statement *stmt)

入力と出力

  • 入力: struct statement *stmt（解放対象のステートメント構造体へのポインタ）
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • stmt が NULL の場合、関数は何もせず即座に復帰する。
  • stmt が指す構造体内の各メンバ（inlist, outlist, command, name, oldlocale）は、ecpg_free で解放可能な状態である必要がある。

アルゴリズムまたは振る舞い

1. stmt が NULL かどうかを確認し、該当する場合は終了する。 2. stmt->inlist および stmt->outlist を free_variable を用いて解放する。 3. stmt->command および stmt->name を ecpg_free で解放する。 4. HAVE_USELOCALE が定義されていない場合、stmt->oldlocale を ecpg_free で解放する。 5. 最後に stmt 構造体自体を ecpg_free で解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/ecpglib/execute.c:2232（ECPGdo の終了処理など）
  • 呼び出し先: free_variable, ecpg_free

関連データ構造

  • struct statement: SQL 実行のライフサイクルを管理する構造体。

失敗モードと境界条件

  • stmt が無効なメモリ領域を指している場合、ecpg_free 呼び出し時に未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • HAVE_USELOCALE の有無によって oldlocale の解放条件が分岐するため、ロケール管理の変更時には注意が必要である。
