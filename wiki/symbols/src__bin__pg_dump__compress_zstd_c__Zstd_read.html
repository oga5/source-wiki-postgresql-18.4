Zstd_read

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_zstd.c:429

目的

Zstandard 圧縮されたアーカイブからデータを読み込み、復元するためのインターフェース関数です。

シグネチャまたは形状

size_t Zstd_read(void *ptr, size_t size, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • ptr: 読み込んだデータを格納するバッファへのポインタ。
  • size: 読み込むバイト数。
  • CFH: 圧縮処理の状態を保持する CompressFileHandle 構造体。
  • **出力**:
  • 実際に読み込まれたバイト数を size_t 型で返します。

前提条件と不変条件

  • CFH は有効な Zstd 圧縮コンテキストで初期化されている必要があります。
  • 内部的に Zstd_read_internal を呼び出し、解凍処理を実行します。

アルゴリズムまたは振る舞い

1. Zstd_read_internal を呼び出し、第4引数に true を渡して解凍処理を実行します。 2. 内部関数が処理した結果（読み込みバイト数）をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: CompressFileHandle の read_func フィールドに本関数が代入されており、アーカイブ読み込み処理を行う上位モジュールから間接的に呼び出されます（src/bin/pg_dump/compress_zstd.c:565）。
  • **呼び出し先**: Zstd_read_internal を呼び出します。

関連データ構造

  • CompressFileHandle: 圧縮・解凍処理の抽象化に使用される構造体。
  • ZstdCompressorState: Zstd_read_internal 経由で参照される、Zstd 固有のコンテキスト情報。

失敗モードと境界条件

  • 読み込みエラーや EOF 到達時の挙動は、内部で呼び出される Zstd_read_internal の実装に依存します。

保守担当者向けメモ

  • 本関数は Zstd_read_internal への単なるラッパーです。解凍ロジックの変更が必要な場合は Zstd_read_internal を参照してください。
  • USE_ZSTD マクロが定義されていない環境では、本モジュールはスタブとして動作します。
