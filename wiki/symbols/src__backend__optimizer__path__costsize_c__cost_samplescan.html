cost_samplescan

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/costsize.c:369

目的

TABLESAMPLE 句を使用したスキャン操作（SampleScan）の実行コスト（startup_cost, total_cost）および行数推定値を算出し、指定された Path 構造体に設定する。

シグネチャまたは形状

void cost_samplescan(Path *path, PlannerInfo *root, RelOptInfo *baserel, ParamPathInfo *param_info)

入力と出力

  • **入力**:
  • path: コストを書き込む対象の Path 構造体。
  • root: プランナのグローバル情報。
  • baserel: 対象となるベースリレーションの情報。
  • param_info: パラメータ化されたパスの場合の補足情報。
  • **出力**:
  • path の rows, startup_cost, total_cost フィールドが更新される。

前提条件と不変条件

  • baserel->relid が 0 より大きいこと。
  • 対象リレーションが TABLESAMPLE 句を持っていること。
  • baserel->pages および baserel->tuples には、サンプリング手法がアクセスするページ数とタプル数が事前に設定されていること。

アルゴリズムまたは振る舞い

1. **行数設定**: param_info があればその行数を、なければ baserel->rows を path->rows に設定する。 2. **I/Oコスト計算**: テーブルスペースのコスト設定を取得し、サンプリング手法が NextSampleBlock をサポートしていればランダムアクセス、そうでなければシーケンシャルアクセスのコストを適用して run_cost に加算する。 3. **CPUコスト計算**:

  • 制限句（Qual）の評価コストを算出し、startup_cost と run_cost に加算。
  • タプル処理コスト（cpu_tuple_cost）を baserel->tuples に乗じて run_cost に加算。
  • ターゲットリストの評価コストを startup_cost および run_cost に加算。

    • 4. **コスト確定**: path->startup_cost と path->total_cost を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/pathnode.c
  • **呼び出し先**:
  • planner_rt_fetch, GetTsmRoutine, get_tablespace_page_costs, get_restriction_qual_cost 等。

関連データ構造

  • Path, PlannerInfo, RelOptInfo, ParamPathInfo, TableSampleClause, TsmRoutine

失敗モードと境界条件

  • TABLESAMPLE 句が欠如している場合、Assert により停止する。
  • baserel->pages や baserel->tuples が適切に設定されていない場合、計算結果が不正確になる。

保守担当者向けメモ

  • 本関数は TABLESAMPLE パラメータ自体の評価コストを無視している。これは、通常スキャンごとに一度しか評価されず、定数であることが多いためである。
  • サンプリング手法内部での計算コストも考慮対象外としている。
