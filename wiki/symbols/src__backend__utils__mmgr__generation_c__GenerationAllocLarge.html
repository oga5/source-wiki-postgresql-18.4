GenerationAllocLarge

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/generation.c:341

目的

GenerationContext において、通常のブロック管理では収まらない大きなサイズのメモリ要求に対し、専用の独立したブロックを割り当てる。

シグネチャまたは形状

static void *GenerationAllocLarge(MemoryContext context, Size size, int flags)

入力と出力

  • **入力**:
  • context: 対象の GenerationContext。
  • size: 要求するメモリサイズ（バイト単位）。
  • flags: メモリ割り当ての動作を制御するフラグ。
  • **出力**:
  • 成功時: 割り当てられたメモリ領域へのポインタ。
  • 失敗時: MemoryContextAllocationFailure を呼び出し、その戻り値を返す。

前提条件と不変条件

  • size は MemoryContextCheckSize により、指定された flags に基づく制限内である必要がある。
  • 割り当てられたブロックは GenerationContext の blocks リストの先頭に挿入される。
  • チャンクヘッダーは MCTX_GENERATION_ID で外部管理としてマークされる。

アルゴリズムまたは振る舞い

1. **サイズ計算**: 要求サイズにヘッダーサイズとアライメント調整を加算し、必要なブロックサイズを算出する。 2. **メモリ確保**: malloc を使用してブロック全体を確保する。失敗した場合はエラーハンドラへ遷移する。 3. **メタデータ初期化**: ブロック構造体（GenerationBlock）を初期化し、チャンク数 nchunks を 1 に設定する。 4. **チャンク設定**: ブロック内の領域をチャンクとして構成し、外部管理フラグを付与する。 5. **デバッグ/検証**: MEMORY_CONTEXT_CHECKING が有効な場合はセンチネル値を設定し、RANDOMIZE_ALLOCATED_MEMORY が有効な場合はメモリを乱数で埋める。 6. **リスト登録**: ブロックをコンテキストの blocks リストに追加する。 7. **Valgrind 連携**: パディング領域およびチャンクヘッダーを NOACCESS に設定し、不正アクセスを検知可能にする。 8. **返却**: チャンクのデータ領域へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: GenerationAlloc からの呼び出しが確認されている。
  • **呼び出し先**: MemoryContextCheckSize, malloc, MemoryContextAllocationFailure, MemoryChunkSetHdrMaskExternal, dlist_push_head, VALGRIND_MAKE_MEM_NOACCESS 等。

関連データ構造

  • GenerationContext: メモリコンテキストの管理構造体。
  • GenerationBlock: メモリブロックの管理単位。
  • MemoryChunk: チャンクヘッダー。

失敗モードと境界条件

  • malloc が NULL を返した場合、MemoryContextAllocationFailure が呼び出される。
  • size が制限を超えている場合、MemoryContextCheckSize によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は「大きな」割り当て専用であり、通常の GenerationAlloc のフローとは異なり、ブロックを再利用せず常に新規 malloc を行う。
  • VALGRIND_MAKE_MEM_NOACCESS の呼び出しは、メモリの安全性を確保するために重要であるため、変更時は注意すること。
