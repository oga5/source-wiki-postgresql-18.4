ResetCatalogCache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/catcache.c:746

目的

指定されたカタログキャッシュ（CatCache）内の全エントリを無効化または削除し、キャッシュの状態をリセットします。

シグネチャまたは形状

static void ResetCatalogCache(CatCache *cache, bool debug_discard)

入力と出力

  • **入力**:
  • cache: リセット対象の CatCache 構造体へのポインタ。
  • debug_discard: デバッグ用フラグ。false の場合、構築中のキャッシュエントリも無効化対象となります。
  • **出力**: なし（副作用としてキャッシュの内容を更新）。

前提条件と不変条件

  • cache ポインタは有効である必要があります。
  • 参照カウント（refcount）が 0 より大きいエントリは、即座に削除されず dead フラグが立てられます。

アルゴリズムまたは振る舞い

1. **リストの処理**: キャッシュ内の全リスト（CatCList）を走査し、参照中のものは dead フラグを立て、それ以外は CatCacheRemoveCList で削除します。 2. **タプルの処理**: キャッシュ内の全タプル（CatCTup）を走査し、自身または所属するリストが参照中の場合は dead フラグを立て、それ以外は CatCacheRemoveCTup で削除します。 3. **構築中エントリの処理**: debug_discard が false の場合、catcache_in_progress_stack を走査し、対象キャッシュに関連する構築中のエントリを dead に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/catcache.c 内の ResetCatalogCaches および ResetCatalogCachesExt から呼び出されます。
  • **呼び出し先**:
  • CatCacheRemoveCList
  • CatCacheRemoveCTup

関連データ構造

  • CatCache: キャッシュ管理構造体。
  • CatCTup: キャッシュされたタプルエントリ。
  • CatCList: 検索結果リスト。
  • CatCInProgress: 構築中のキャッシュエントリを追跡するスタック構造。

失敗モードと境界条件

  • 参照カウントが 0 でないエントリはメモリから即座に解放されず、参照が解放された後に削除される「遅延削除」の挙動をとります。
  • CATCACHE_STATS が定義されている場合、タプル削除ごとに cc_invals カウンタが増加します。

保守担当者向けメモ

  • 本関数はキャッシュの整合性を強制的にリセットする強力な操作です。
  • dead フラグが立てられたエントリは、参照カウントが 0 になった時点で最終的に解放されるため、メモリ管理の整合性に注意してください。
