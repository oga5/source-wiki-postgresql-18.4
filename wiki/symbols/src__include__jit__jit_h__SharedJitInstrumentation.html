SharedJitInstrumentation

定義

  • 種別: struct
  • 定義箇所: src/include/jit/jit.h:51

目的

並列クエリ実行時において、複数のワーカープロセス間で JIT コンパイルの統計情報（JitInstrumentation）を共有するためのデータ構造です。

シグネチャまたは形状

typedef struct SharedJitInstrumentation
{
    int         num_workers;
    JitInstrumentation jit_instr[FLEXIBLE_ARRAY_MEMBER];
} SharedJitInstrumentation;

入力と出力

  • **入力**: 構造体内の num_workers は、共有されるワーカーの数を指定します。
  • **出力**: jit_instr 配列を通じて、各ワーカーの JIT 実行統計情報が格納されます。

前提条件と不変条件

  • jit_instr はフレキシブル配列メンバーとして定義されており、num_workers に応じたメモリ領域を動的に確保して使用する必要があります。
  • 並列実行環境下での共有メモリ領域として割り当てられることを前提としています。

アルゴリズムまたは振る舞い

  • 並列クエリの実行計画において、各ワーカーが生成した JIT 統計情報を集約するために使用されます。
  • execParallel.c 等で、共有メモリのオフセット計算（offsetof(SharedJitInstrumentation, jit_instr)）を行い、ワーカーごとの統計領域へアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/commands/explain.c: 実行計画の統計表示時に参照。
  • src/backend/executor/execParallel.c: 並列実行時のメモリ確保および統計情報の書き込み・読み取りに使用。
  • src/include/executor/execParallel.h: ParallelExecutorState 等の構造体内でポインタとして保持。
  • src/include/nodes/execnodes.h: PlanState 内の worker_jit_instrument として保持。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • JitInstrumentation: 各ワーカーの個別の統計情報を保持する構造体。
  • ParallelExecutorState: 並列実行の全体状態を管理する構造体（execParallel.h）。

失敗モードと境界条件

  • num_workers が 0 の場合、またはメモリ確保が不十分な場合に、jit_instr 配列へのアクセスが境界外参照を引き起こす可能性があります。
  • 共有メモリのサイズ計算時に offsetof を誤ると、データ破損が発生します。

保守担当者向けメモ

  • 本構造体はフレキシブル配列を使用しているため、sizeof(SharedJitInstrumentation) は配列部分を含まないサイズを返します。メモリ確保時には必ず num_workers を考慮したサイズ計算を行ってください。
  • src/backend/executor/execParallel.c でのメモリレイアウト計算ロジックと密結合しているため、構造体の変更時は当該ファイルのオフセット計算ロジックも併せて確認してください。
