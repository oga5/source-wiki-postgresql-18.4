PLyUnicode_Bytes

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_util.c:18

目的

PythonのUnicodeオブジェクトを、PostgreSQLのサーバーエンコーディングに適合したバイト列（bytesオブジェクト）へ変換する。

シグネチャまたは形状

PyObject *PLyUnicode_Bytes(PyObject *unicode)

入力と出力

  • **入力**: unicode (Python Unicodeオブジェクト)
  • **出力**: サーバーエンコーディングに変換された PyObject* (bytesオブジェクト)

前提条件と不変条件

  • 入力された unicode オブジェクトは有効なPython Unicode型であること。
  • 変換処理中、Pythonの参照カウントが適切に管理されること。
  • データベースエンコーディングがUTF-8でない場合、pg_any_to_server による変換が行われる。

アルゴリズムまたは振る舞い

1. PyUnicode_AsUTF8String を呼び出し、UnicodeオブジェクトをUTF-8バイト列に変換する。 2. データベースのエンコーディングがUTF-8と異なる場合、pg_any_to_server を用いてサーバーエンコーディングへ変換する。この際、PostgreSQLの例外処理（PG_TRY/PG_CATCH）を介して安全性を確保する。 3. 変換後のデータから PyBytes_FromStringAndSize を用いて新しいPython bytesオブジェクトを生成する。 4. pg_any_to_server がメモリを動的に割り当てた場合は pfree で解放する。 5. 内部で使用した一時的なUTF-8バイト列の参照カウントを Py_DECREF で減らし、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_typeio.c (複数箇所)
  • src/pl/plpython/plpy_util.c:83
  • **呼び出し先**:
  • PyUnicode_AsUTF8String, PyBytes_AsString, PyBytes_FromStringAndSize, Py_DECREF (Python C API)
  • PLy_elog (エラー報告)
  • GetDatabaseEncoding, pg_any_to_server, pfree (PostgreSQL内部関数)

関連データ構造

  • PyObject (Python C APIの基本型)

失敗モードと境界条件

  • PyUnicode_AsUTF8String が失敗した場合、PLy_elog(ERROR, ...) を呼び出し処理を中断する。
  • PyBytes_AsString が失敗した場合、一時オブジェクトの参照カウントを解放した上で PLy_elog(ERROR, ...) を呼び出す。
  • サーバーエンコーディング変換中にエラーが発生した場合、PG_CATCH ブロックで参照カウントを解放し、例外を再スローする。

保守担当者向けメモ

  • 本関数はUTF-8を中間エンコーディングとして利用する設計である。
  • pg_any_to_server がメモリを割り当てた場合、必ず pfree を呼び出す必要がある。
  • Pythonオブジェクトの参照カウント管理は、PostgreSQLの例外発生時にもリークしないよう慎重に実装されている。
