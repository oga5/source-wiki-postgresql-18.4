ExecMakeFunctionResultSet

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execSRF.c:496

目的

プロジェクション（SELECT リスト等）内で呼び出される集合を返す関数（SRF）を実行し、結果を逐次取得する。

シグネチャまたは形状

Datum ExecMakeFunctionResultSet(SetExprState *fcache,
                                ExprContext *econtext,
                                MemoryContext argContext,
                                bool *isNull,
                                ExprDoneCond *isDone)

入力と出力

  • **入力**:
  • fcache: SRF の実行状態を保持する SetExprState。
  • econtext: 式評価用のコンテキスト。
  • argContext: 引数評価用のメモリコンテキスト。
  • **出力**:
  • 戻り値: 関数実行結果の Datum。
  • isNull: 結果が NULL かどうかを示すフラグ。
  • isDone: 関数の完了状態（ExprSingleResult, ExprMultipleResult, ExprEndResult）。

前提条件と不変条件

  • check_stack_depth() によりスタックオーバーフローを防止する。
  • ValuePerCall モードの場合、引数の評価結果は argContext 上に保持され、複数行の返却に耐える生存期間を持つ必要がある。
  • fcache->funcResultStore が存在する場合、関数は既にマテリアライズ済みであり、そこから行を読み出す。

アルゴリズムまたは振る舞い

1. **Tuplestore 読み出し**: fcache->funcResultStore があれば、そこから tuplestore_gettupleslot で行を取得し、結果を返す。空になれば tuplestore_end を呼び出し終了する。 2. **引数評価**: setArgsValid が偽の場合、ExecEvalFuncArgs を呼び出し引数を評価する。 3. **関数呼び出し**: ReturnSetInfo を準備し、FunctionCallInvoke を実行する。

  • fn_strict かつ引数に NULL が含まれる場合は関数を呼び出さず、空の結果を返す。

    • 4. **プロトコル処理**:

  • SFRM_ValuePerCall: 複数行の結果が返る場合、setArgsValid を真にして次回の呼び出しに備える。
  • SFRM_Materialize: 関数が Tuplestore を返した場合、ExecPrepareTuplestoreResult を呼び出し、restart ラベルへ遷移して読み出しを開始する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeProjectSet.c
  • src/backend/executor/execSRF.c（再帰的または内部的な呼び出し）
  • **呼び出し先**:
  • ExecEvalFuncArgs
  • FunctionCallInvoke
  • ExecPrepareTuplestoreResult
  • tuplestore_gettupleslot / tuplestore_end
  • RegisterExprContextCallback (ShutdownSetExpr)

関連データ構造

  • SetExprState: SRF の実行状態を保持。
  • ReturnSetInfo: 関数と実行エンジン間の通信用構造体。
  • ExprContext: 式評価の実行環境。

失敗モードと境界条件

  • **スタックオーバーフロー**: check_stack_depth() によりエラーが発生する場合がある。
  • **プロトコル違反**: SFRM_Materialize モードで isDone が ExprSingleResult でない場合、ERRCODE_E_R_I_E_SRF_PROTOCOL_VIOLATED でエラーとなる。
  • **メモリ管理**: argContext と econtext の切り替えが不適切な場合、メモリリークや不正なメモリ参照が発生する可能性がある。

保守担当者向けメモ

  • tupledesc_match は、関数が返す行型とクエリが期待する行型が一致するかを検証する。動的な型（RECORD 型など）を扱う際の重要なチェックポイントである。
