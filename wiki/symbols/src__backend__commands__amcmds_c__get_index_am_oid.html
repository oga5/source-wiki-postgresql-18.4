get_index_am_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/amcmds.c:162

目的

指定されたアクセスメソッド名に対応するインデックスアクセスメソッドの OID を取得します。

シグネチャまたは形状

Oid get_index_am_oid(const char *amname, bool missing_ok)

入力と出力

  • **入力**:
  • amname: 検索対象のアクセスメソッド名（文字列）。
  • missing_ok: true の場合、対象が見つからない場合にエラーを発生させず InvalidOid を返します。false の場合はエラーを発生させます。
  • **出力**:
  • 成功時: 対応するアクセスメソッドの Oid。
  • 失敗時: missing_ok が true の場合に InvalidOid。

前提条件と不変条件

  • 検索対象のアクセスメソッドは pg_am システムカタログに登録されている必要があります。
  • AMTYPE_INDEX 型のアクセスメソッドのみを対象とします。

アルゴリズムまたは振る舞い

1. 内部関数 get_am_type_oid を呼び出します。 2. その際、第2引数に AMTYPE_INDEX を指定し、インデックス専用のアクセスメソッドであることを明示して検索を実行します。 3. 検索結果（OID またはエラー）をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c:1653
  • src/backend/commands/opclasscmds.c:790
  • src/backend/parser/parse_utilcmd.c:2505
  • **呼び出し先**:
  • get_am_type_oid (同ファイル内)

関連データ構造

  • pg_am: アクセスメソッド情報を保持するシステムカタログ。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • amname に該当するアクセスメソッドが存在しない場合、missing_ok が false であればエラーが発生します。
  • 指定された名前のアクセスメソッドが存在しても、それが AMTYPE_INDEX ではない場合、get_am_type_oid の仕様に従い検索に失敗します。

保守担当者向けメモ

  • 本関数は get_am_type_oid のラッパーです。インデックス以外のアクセスメソッド（テーブルなど）を検索する場合は、get_table_am_oid 等の適切な関数を使用してください。
