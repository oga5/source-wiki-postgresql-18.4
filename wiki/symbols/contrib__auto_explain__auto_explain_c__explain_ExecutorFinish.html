explain_ExecutorFinish

定義

  • 種別: function
  • 定義箇所: contrib/auto_explain/auto_explain.c:349

目的

PostgreSQL のクエリ実行プロセスにおける ExecutorFinish フェーズをフックし、ネストレベルの追跡を行いながら、本来の実行処理（標準または以前に登録されたフック）を安全に呼び出す。

シグネチャまたは形状

static void explain_ExecutorFinish(QueryDesc *queryDesc)

入力と出力

  • **入力**: QueryDesc *queryDesc - 実行中のクエリに関する情報を含む記述子。
  • **出力**: なし（void）。

前提条件と不変条件

  • nesting_level グローバル変数が、クエリのネスト深さを正確に反映していること。
  • PG_TRY ブロックにより、内部で例外が発生した場合でも nesting_level のデクリメントが保証されること。

アルゴリズムまたは振る舞い

1. nesting_level をインクリメントする。 2. PG_TRY ブロックを開始する。 3. prev_ExecutorFinish が設定されていればそれを呼び出し、設定されていなければ standard_ExecutorFinish を呼び出す。 4. PG_FINALLY ブロックにて、例外の有無にかかわらず nesting_level をデクリメントする。 5. PG_END_TRY で処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の実行エンジン（ExecutorFinish_hook を介して呼び出される）。
  • **呼び出し先**:
  • prev_ExecutorFinish（以前に登録されたフック関数）
  • standard_ExecutorFinish（PostgreSQL 標準の実行終了処理）

関連データ構造

  • QueryDesc: クエリ実行状態を保持する構造体。
  • nesting_level: 現在のクエリ実行のネスト深さを追跡する静的変数。

失敗モードと境界条件

  • PG_TRY ブロック内の呼び出し先でエラーが発生した場合、PG_FINALLY により nesting_level の整合性は維持される。
  • 呼び出し先関数が異常終了した場合、PostgreSQL のエラー処理機構に制御が移る。

保守担当者向けメモ

  • 本関数はフックの連鎖を維持するために prev_ExecutorFinish を経由する設計となっている。
  • nesting_level の操作は、再帰的なクエリ実行時のログ出力制御に不可欠であるため、変更時は注意が必要。
