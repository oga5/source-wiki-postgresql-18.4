CleanQuerytext

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:85

目的

SQLクエリ文字列から先頭および末尾の空白を除去し、指定された位置情報（location）と長さ（len）を正規化して、クエリの有効な部分文字列を返します。

シグネチャまたは形状

const char *CleanQuerytext(const char *query, int *location, int *len)

入力と出力

  • **入力**:
  • query: 対象となるSQLクエリ文字列。
  • location: クエリ内の開始位置（ポインタ）。-1 の場合は不明とみなされます。
  • len: クエリの長さ（ポインタ）。0 または -1 の場合は文字列の末尾までとみなされます。
  • **出力**:
  • 戻り値: 正規化されたクエリ文字列の開始位置を指すポインタ。
  • location: 更新された開始位置。
  • len: 更新されたクエリの長さ。

前提条件と不変条件

  • location が 0 以上の場合、*location は query の長さ以下である必要があります。
  • len が 0 より大きい場合、*len は query の長さ以下である必要があります。
  • 空白の判定には scanner_isspace() を使用し、レクサーの挙動と一致させます。

アルゴリズムまたは振る舞い

1. **オフセット適用**: location が 0 以上であれば、その位置までポインタを進めます。len が 0 以下であれば、残りの文字列全体を対象とします。 2. **未知の位置の処理**: location が -1 の場合、開始位置を 0 とし、長さは文字列全体とみなします。 3. **先頭空白除去**: scanner_isspace() を用いて、先頭の空白をスキップし、location を進め len を減らします。 4. **末尾空白除去**: scanner_isspace() を用いて、末尾の空白をスキップし、len を減らします。 5. **結果更新**: 最終的な location と len を呼び出し元に書き戻し、調整後の文字列ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/backend/commands/extension.c
  • **呼び出し先**:
  • strlen (標準ライブラリ)
  • scanner_isspace (パーサー関連)

関連データ構造

  • 特になし（標準的なC文字列と整数ポインタを使用）。

失敗モードと境界条件

  • location が -1 の場合、安全のために文字列全体を対象として再計算します。
  • 空白のみの文字列や空文字列が渡された場合、len が 0 になり、適切に処理されます。
  • Assert により、不正な位置や長さが指定された場合にデバッグビルドで停止します。

保守担当者向けメモ

  • 本関数は、パーサーが生成したステートメント位置情報が不完全な場合や、後続の処理でクエリのトリミングが必要な場合に利用されます。
  • scanner_isspace を使用しているため、レクサーの空白定義に変更があった場合は注意が必要です。
