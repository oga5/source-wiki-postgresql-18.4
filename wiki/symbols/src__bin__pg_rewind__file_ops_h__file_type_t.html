file_type_t

定義

  • 種別: typedef
  • 定義箇所: src/bin/pg_rewind/file_ops.h:26

目的

pg_rewind において処理対象となるファイルやディレクトリの種別を識別するために使用される列挙型（またはその別名）です。

シグネチャまたは形状

src/bin/pg_rewind/filemap.h:37 にて定義されています。

入力と出力

本シンボル自体は型定義であり、関数ではありません。主に process_source_file や process_target_file などの関数において、引数としてファイル種別を渡すために使用されます。

前提条件と不変条件

filemap.h で定義された列挙値のいずれかをとることが想定されます。

アルゴリズムまたは振る舞い

ファイルシステム上のオブジェクト（通常ファイル、ディレクトリ、シンボリックリンク等）を区別し、pg_rewind の同期処理において適切な操作を選択するための分類基準として機能します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/bin/pg_rewind/filemap.c: process_source_file, process_target_file
  • src/bin/pg_rewind/filemap.h: 構造体メンバ target_type, source_type
  • src/bin/pg_rewind/libpq_source.c: ローカル変数 type
  • **呼び出し先**:
  • 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • file_entry_t (ファイル情報を保持する構造体)
  • process_file_callback_t (コールバック関数の引数として使用)

失敗モードと境界条件

型定義であるため、本シンボル自体に失敗モードはありません。ただし、未定義の値を代入した場合の挙動は、これを利用する各関数の実装に依存します。

保守担当者向けメモ

  • src/bin/pg_rewind/filemap.h にある列挙定義と同期している必要があります。変更時は両ファイルを確認してください。
