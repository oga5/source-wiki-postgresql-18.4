ExplainPrintJIT

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:900

目的

クエリ実行計画の EXPLAIN 出力において、JIT（Just-In-Time）コンパイルに関する統計情報および設定を表示する。

シグネチャまたは形状

static void ExplainPrintJIT(ExplainState *es, int jit_flags, JitInstrumentation *ji)

入力と出力

  • **入力**:
  • es: 出力状態を管理する ExplainState 構造体。
  • jit_flags: JIT の有効化オプション（PGJIT_INLINE, PGJIT_OPT3 等）。
  • ji: JIT 実行統計を保持する JitInstrumentation 構造体。
  • **出力**: なし（es->str に文字列を追加、または es を介して構造化データを出力）。

前提条件と不変条件

  • ji が NULL であるか、ji->created_functions == 0 の場合は何も出力せずに終了する。
  • es->format に応じて、テキスト形式または構造化形式（JSON/XML等）で出力が分岐する。

アルゴリズムまたは振る舞い

1. ji 内の各カウンタ（生成、インライン化、最適化、エミッション）を合計し、総時間を算出する。 2. ExplainOpenGroup を使用して "JIT" グループを開始する。 3. es->format が EXPLAIN_FORMAT_TEXT の場合:

  • インデントを調整し、関数数とオプション設定（Inlining, Optimization 等）をテキスト出力する。
  • es->analyze かつ es->timing が真の場合、各フェーズの実行時間をミリ秒単位で出力する。

    • 4. それ以外（JSON/XML等）の場合:

  • 関数数、オプション設定、およびタイミング情報を階層構造として出力する。

    • 5. ExplainCloseGroup で "JIT" グループを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c 内の複数箇所（ExplainOnePlan 等から呼び出される）。
  • **呼び出し先**: ExplainOpenGroup, ExplainCloseGroup, ExplainPropertyInteger, ExplainPropertyBool, ExplainPropertyFloat, ExplainIndentText, appendStringInfo 等の ExplainState 操作関数。

関連データ構造

  • ExplainState: 出力先やフォーマットを保持する構造体。
  • JitInstrumentation: JIT コンパイルの統計情報（カウンタ）を保持する構造体。

失敗モードと境界条件

  • ji が NULL の場合、安全に早期リターンする。
  • es->analyze や es->timing が偽の場合、詳細なタイミング情報は出力されない。

保守担当者向けメモ

  • テキスト出力形式の変更は ExplainPrintJIT 内の if (es->format == EXPLAIN_FORMAT_TEXT) ブロックを修正する。
  • 構造化データ形式の変更は else ブロックを修正する。
  • ji->deform_counter は generation_counter に含まれているため、総時間計算時には加算しないこと。
