ClosePager

定義

  • 種別: function
  • 定義箇所: src/fe_utils/print.c:3140

目的

開いているページャ（more や less 等）のパイプを閉じ、関連するシグナルハンドラの状態を復元する。

シグネチャまたは形状

void ClosePager(FILE *pagerpipe)

入力と出力

  • **入力**: pagerpipe (開いているページャのパイプへのファイルポインタ)
  • **出力**: なし（副作用としてパイプをクローズし、シグナル状態を復元する）

前提条件と不変条件

  • pagerpipe が stdout と等しい場合、本関数は何もしない。
  • pagerpipe が NULL の場合、本関数は何もしない。

アルゴリズムまたは振る舞い

1. pagerpipe が有効かつ stdout ではないことを確認する。 2. グローバル変数 cancel_pressed が真の場合、pagerpipe に対して「Interrupted」という警告メッセージを出力する。 3. pclose(pagerpipe) を呼び出し、パイプを閉じる。 4. restore_sigpipe_trap() を呼び出し、SIGPIPE のトラップ状態を復元する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c
  • src/bin/psql/common.c
  • src/bin/psql/help.c
  • src/bin/psql/input.c
  • src/fe_utils/print.c
  • **呼び出し先**:
  • pclose (標準ライブラリ)
  • restore_sigpipe_trap (内部関数)
  • fprintf (標準ライブラリ)

関連データ構造

  • FILE (標準ライブラリ)
  • cancel_pressed (グローバル変数)

失敗モードと境界条件

  • pagerpipe が stdout の場合、クローズ処理はスキップされる。
  • ユーザーが処理を中断（cancel_pressed が真）した場合、ページャに対して警告メッセージが書き込まれるが、ページャが既に終了している場合はメッセージが破棄される可能性がある。

保守担当者向けメモ

  • 本関数は PageOutput で開始されたページャセッションを終了させるために使用する。
  • SIGPIPE のハンドリングを適切に終了させるため、必ず pclose とセットで呼び出すこと。
