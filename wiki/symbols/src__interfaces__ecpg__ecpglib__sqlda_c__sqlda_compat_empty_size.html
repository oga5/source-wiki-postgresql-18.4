sqlda_compat_empty_size

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/sqlda.c:44

目的

Informix 互換モードの SQLDA 構造体において、データ領域を含まないメタデータ部分（構造体本体、フィールド定義、およびフィールド名文字列）に必要なメモリサイズを計算します。

シグネチャまたは形状

static long sqlda_compat_empty_size(const PGresult *res)

入力と出力

  • **入力**: const PGresult *res - 取得したクエリ結果セット。
  • **出力**: long - メタデータ格納に必要な合計バイト数（アライメント調整済み）。

前提条件と不変条件

  • res は有効な PGresult ポインタである必要があります。
  • 戻り値は、sqlda_compat 構造体、sqlvar_compat 配列、および各列名の文字列を格納するための連続したメモリブロックのサイズを保証します。

アルゴリズムまたは振る舞い

1. PQnfields(res) を用いて列数（sqld）を取得します。 2. sqlda_compat 構造体本体と、列数分の sqlvar_compat 構造体配列のサイズを合計します。 3. 各列の列名文字列の長さ（strlen + 終端文字分）を合計に加算します。 4. ecpg_sqlda_align_add_size を呼び出し、最初のフィールド値の配置に向けたアライメント調整（int 型基準）を適用します。 5. 計算された最終的なオフセット値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/sqlda.c 内の ecpg_build_compat_sqlda 関数（161行目および266行目）。
  • **呼び出し先**:
  • PQnfields (libpq)
  • PQfname (libpq)
  • strlen (標準ライブラリ)
  • ecpg_sqlda_align_add_size (内部ユーティリティ)

関連データ構造

  • struct sqlda_compat: Informix 互換モードの SQLDA 構造体。
  • struct sqlvar_compat: 各列の情報を保持する構造体。
  • PGresult: libpq のクエリ結果オブジェクト。

失敗モードと境界条件

  • res が NULL の場合、PQnfields 等の libpq 関数呼び出しにより未定義の動作が発生する可能性があります。
  • 列数が 0 の場合、構造体本体のみのサイズが計算されます。

保守担当者向けメモ

  • 本関数はメモリレイアウトの計算のみを担当します。実際のメモリ確保は呼び出し側で行われます。
  • Informix 互換モードの構造体定義に変更があった場合、sizeof の計算式を更新する必要があります。
  • ecpg_sqlda_align_add_size によるアライメント調整は、プラットフォーム固有の構造体パディング規則に依存するため、移植時には注意が必要です。
