UtilityReturnsTuples

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/utility.c:2027

目的

指定されたユーティリティ文（parsetree）が、実行結果としてタプルを返すかどうかを判定する。

シグネチャまたは形状

bool UtilityReturnsTuples(Node *parsetree)

入力と出力

  • **入力**: Node *parsetree（解析済みのユーティリティ文ノード）
  • **出力**: bool（タプルを返す場合は true、それ以外は false）

前提条件と不変条件

  • 入力される parsetree は、PostgreSQL のユーティリティ文ノードである必要がある。
  • T_FetchStmt や T_ExecuteStmt の場合、内部でポータルやプリペアドステートメントの検索を行うため、実行環境が適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

ノードの型（nodeTag）に基づき、以下の通り判定を行う： 1. **T_CallStmt**: 関数呼び出しの結果型が RECORDOID であれば true。 2. **T_FetchStmt**: ismove が真なら false。それ以外はポータルを検索し、tupDesc が存在すれば true。 3. **T_ExecuteStmt**: プリペアドステートメントを検索し、結果記述子（resultDesc）が存在すれば true。 4. **T_ExplainStmt / T_VariableShowStmt**: 常に true。 5. **その他**: false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/pquery.c: ユーティリティ文の実行計画生成および実行フローにおいて、結果の有無を確認するために呼び出される。
  • src/backend/tcop/utility.c: 内部的なユーティリティ処理フローから参照される。
  • **呼び出し先**:
  • GetPortalByName (ポータル検索)
  • PortalIsValid (ポータル検証)
  • FetchPreparedStatement (プリペアドステートメント検索)

関連データ構造

  • Node: 構文解析ツリーのノード。
  • CallStmt, FetchStmt, ExecuteStmt, ExplainStmt, VariableShowStmt: 各ユーティリティ文の構造体。
  • Portal: FetchStmt の実行状態を保持する構造体。
  • PreparedStatement: ExecuteStmt の対象となる準備済み文。

失敗モードと境界条件

  • T_FetchStmt または T_ExecuteStmt において、対象のポータルやプリペアドステートメントが見つからない場合、エラーを発生させずに false を返す。
  • T_FetchStmt で ismove が指定されている場合は、タプルを返さないものとして false を返す。

保守担当者向けメモ

  • 本関数は src/backend/tcop/pquery.c 内のロジックと同期を保つ必要がある。新しいユーティリティ文でタプルを返すものを追加した場合は、必ず本関数の switch 文にケースを追加すること。
