koi8r_to_win866

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:469

目的

KOI8-R エンコーディングから WIN866 エンコーディングへ文字列を変換する PostgreSQL 変換プロシージャ。

シグネチャまたは形状

Datum koi8r_to_win866(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得
  • src (CSTRING): 変換元の文字列バッファ
  • dest (CSTRING): 変換後の文字列格納先バッファ
  • len (INT32): 変換対象のバイト長
  • noError (BOOL): エラーハンドリングのフラグ
  • **出力**: Datum 型として変換されたバイト数を返却

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_KOI8R, PG_WIN866) により、ソースエンコーディングが KOI8-R、宛先エンコーディングが WIN866 であることが検証される。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認する。 3. local2local 関数を呼び出し、koi2win866 変換テーブルを用いて変換を実行する。 4. 変換されたバイト数を PG_RETURN_INT32 で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1(koi8r_to_win866) により登録）。
  • **呼び出し先**: local2local（変換処理の実行）。

関連データ構造

  • koi2win866: KOI8-R から WIN866 への変換用マッピングテーブル（unsigned char 配列）。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS に適合しないエンコーディング ID が渡された場合、エラーが発生する。
  • noError フラグの設定に応じて、変換中の不正な文字に対する挙動が local2local の仕様に従い決定される。

保守担当者向けメモ

  • 本関数は local2local を利用した直接変換を行う。変換テーブル koi2win866 の内容を変更する場合は、対応する文字コードの整合性に注意すること。
