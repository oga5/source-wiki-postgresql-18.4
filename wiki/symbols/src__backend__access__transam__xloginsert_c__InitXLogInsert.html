InitXLogInsert

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xloginsert.c:1347

目的

WAL（Write-Ahead Logging）レコード構築に必要なメモリ領域および作業用データ構造を初期化します。

シグネチャまたは形状

void InitXLogInsert(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 xloginsert_cxt, registered_buffers, rdatas, hdr_scratch が初期化されます）

前提条件と不変条件

  • 本関数は各バックエンドプロセスにつき一度だけ呼び出されることを想定しています。
  • USE_ASSERT_CHECKING が有効な場合、WAL レコードのデコードに必要な最大領域が妥当であるか検証されます。

アルゴリズムまたは振る舞い

1. **メモリコンテキスト作成**: xloginsert_cxt が未初期化の場合、TopMemoryContext 配下に「WAL record construction」という名称のメモリコンテキストを作成します。 2. **バッファ領域の確保**: registered_buffers が未初期化の場合、XLR_NORMAL_MAX_BLOCK_ID + 1 個分の registered_buffer 構造体配列を確保します。 3. **データチェーン領域の確保**: rdatas が未初期化の場合、XLR_NORMAL_RDATAS 個分の XLogRecData 構造体配列を確保します。 4. **ヘッダー領域の確保**: hdr_scratch が未初期化の場合、HEADER_SCRATCH_SIZE 分のメモリを確保します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c（バックエンド初期化時）
  • 呼び出し先: AllocSetContextCreate, MemoryContextAllocZero, MemoryContextAlloc（PostgreSQL メモリ管理 API）

関連データ構造

  • xloginsert_cxt: WAL 構築用メモリコンテキスト
  • registered_buffers: 登録されたブロック情報を保持する配列
  • rdatas: レコードデータを保持するチェーン構造
  • hdr_scratch: レコードヘッダー構築用の一時領域

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc 等がエラーを発生させ、プロセスが中断されます。
  • XLR_NORMAL_MAX_BLOCK_ID や XLR_NORMAL_RDATAS を超える登録を行う場合は、別途 XLogEnsureRecordSpace による拡張が必要です。

保守担当者向けメモ

  • 本関数で確保されるメモリはバックエンドの生存期間中維持されます。
  • registered_buffers 等のグローバル変数は、本関数によって一度だけ初期化される設計です。
