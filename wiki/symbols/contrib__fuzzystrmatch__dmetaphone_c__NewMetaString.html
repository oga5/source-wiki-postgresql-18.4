NewMetaString

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/dmetaphone.c:234

目的

Double Metaphone アルゴリズムで使用される文字列操作用構造体 metastring を初期化し、メモリを割り当てる。

シグネチャまたは形状

static metastring *NewMetaString(const char *init_str)

入力と出力

  • **入力**: init_str (初期化に使用する文字列、NULL の場合は空文字列として扱われる)
  • **出力**: 初期化済みの metastring 構造体へのポインタ

前提条件と不変条件

  • META_MALLOC マクロによりメモリが確保される。
  • s->str は init_str の長さに対して 7 バイトの余裕を持って確保される。
  • s->free_string_on_destroy は 1 に設定され、破棄時にメモリ解放が必要であることを示す。
  • assert により、メモリ確保の成功が保証される。

アルゴリズムまたは振る舞い

1. metastring 構造体自体のメモリを確保する。 2. init_str が NULL の場合は空文字列を代入する。 3. 文字列の長さを取得し、成長を見越して length + 7 バイトのバッファを確保する。 4. init_str を確保したバッファにコピーする。 5. 破棄時のメモリ解放フラグを立てて構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/fuzzystrmatch/dmetaphone.c:408 (入力文字列の初期化)
  • contrib/fuzzystrmatch/dmetaphone.c:412 (プライマリコード用バッファの初期化)
  • contrib/fuzzystrmatch/dmetaphone.c:413 (セカンダリコード用バッファの初期化)
  • **呼び出し先**:
  • strlen, memcpy (標準ライブラリ)
  • META_MALLOC (マクロ)

関連データ構造

  • metastring: 文字列のバッファ、現在の長さ、バッファサイズ、解放フラグを保持する構造体。

失敗モードと境界条件

  • init_str が NULL の場合、空文字列として安全に処理される。
  • メモリ確保に失敗した場合、assert によりプログラムが停止する。

保守担当者向けメモ

  • 本関数は PostgreSQL のメモリコンテキスト内で動作することを前提としている。
  • s->bufsize の計算における + 7 は、アルゴリズム実行中の文字列伸長に対するヒューリスティックな余裕である。
