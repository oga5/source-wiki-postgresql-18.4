SerializedSnapshotData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/time/snapmgr.c:250

目的

スナップショット情報を外部プロセスへの受け渡しや永続化のために、固定長のヘッダーとしてシリアライズする際のデータ構造です。

シグネチャまたは形状

typedef struct SerializedSnapshotData
{
	TransactionId xmin;
	TransactionId xmax;
	uint32		xcnt;
	int32		subxcnt;
	bool		suboverflowed;
	bool		takenDuringRecovery;
	CommandId	curcid;
} SerializedSnapshotData;

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。シリアライズ処理において、この構造体のサイズ（sizeof(SerializedSnapshotData)）がメモリレイアウトのオフセット計算の基準として使用されます。

前提条件と不変条件

  • 本構造体はスナップショットのメタデータのみを保持します。
  • 実際のトランザクションID配列（xip）やサブトランザクションID配列は、本構造体の直後のメモリ領域に配置されることが前提となっています。

アルゴリズムまたは振る舞い

スナップショットをシリアライズする際、まずこの構造体に各フィールドをコピーし、続いて後続のメモリ領域に可変長のトランザクションID配列を書き込むという手順で利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/time/snapmgr.c 内のシリアライズ関連処理（1707行目以降のコード群）から参照されています。
  • **呼び出し先**: 本構造体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • SnapshotData: 本構造体の元となる、実行時に使用されるスナップショット構造体。

失敗モードと境界条件

  • 本構造体は固定長ですが、後続する可変長データ（xip等）のサイズ計算を誤ると、メモリ破壊や不正なデータ読み込みが発生する可能性があります。

保守担当者向けメモ

  • 本構造体のフィールドを変更する場合は、シリアライズおよびデシリアライズを行うコード（snapmgr.c 内の関連関数）のオフセット計算ロジックをすべて更新する必要があります。
  • sizeof(SerializedSnapshotData) を基準としたポインタ演算が行われているため、パディングによるサイズ変化に注意してください。
