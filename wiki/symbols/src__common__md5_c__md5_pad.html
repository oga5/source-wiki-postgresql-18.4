md5_pad

定義

  • 種別: function
  • 定義箇所: src/common/md5.c:309

目的

MD5 アルゴリズムにおける最終パディング処理を実行し、メッセージ長をハッシュ計算の末尾に付与する。

シグネチャまたは形状

static void md5_pad(pg_md5_ctx *ctx)

入力と出力

  • **入力**: pg_md5_ctx *ctx（ハッシュ計算の状態を保持するコンテキスト構造体）。
  • **出力**: なし（副作用として ctx 内のバッファおよび内部状態を更新）。

前提条件と不変条件

  • ctx は初期化済みであること。
  • md5_calc を呼び出すことで、バッファ内のデータが MD5 アルゴリズムに従って変換される。

アルゴリズムまたは振る舞い

1. 現在のバッファ残量（gap）を計算し、パディングデータ（md5_paddat）をバッファにコピーする。 2. バッファが一杯になった場合は md5_calc を呼び出して中間状態を更新し、残りのパディングをバッファに書き込む。 3. メッセージ長（ctx->md5_n8）をバッファの末尾（56バイト目以降）に配置する。この際、WORDS_BIGENDIAN マクロに基づき、エンディアンに応じたバイト順序の調整を行う。 4. 最後に md5_calc を呼び出し、最終的なハッシュ状態を確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/md5.c 内の pg_md5_final から呼び出される。
  • **呼び出し先**: md5_calc を内部で呼び出す。

関連データ構造

  • pg_md5_ctx: ハッシュ計算の状態を保持する構造体。
  • md5_paddat: パディングに使用される定数データ。

失敗モードと境界条件

  • バッファの空き容量が 8 バイト以下の場合は、一度 md5_calc を実行してバッファをフラッシュしてから残りのパディングを適用する。

保守担当者向けメモ

  • 本関数は RFC 1321 に準拠した実装である。
  • エンディアン依存の処理が含まれているため、プラットフォーム固有のバイトオーダー変更時には注意が必要である。
