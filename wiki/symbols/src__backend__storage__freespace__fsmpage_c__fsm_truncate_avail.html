fsm_truncate_avail

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/fsmpage.c:312

目的

FSMページ内の指定されたスロット以降のリーフノードをクリアし、必要に応じてページ全体のツリー構造を再構築する。

シグネチャまたは形状

bool fsm_truncate_avail(Page page, int nslots)

入力と出力

  • **入力**:
  • page: 操作対象のFSMページ。
  • nslots: 残すべきリーフノードの数（このインデックス以降がクリア対象となる）。
  • **出力**:
  • bool: リーフノードのクリアによってページ内容に変更があった場合に true を返す。

前提条件と不変条件

  • nslots は 0 以上かつ LeafNodesPerPage 未満であること（Assert により検証）。
  • 呼び出し元は、対象のページに対して適切なバッファロックを保持している必要がある。

アルゴリズムまたは振る舞い

1. fsmpage->fp_nodes 配列の NonLeafNodesPerPage + nslots 以降の全リーフノードを走査する。 2. 走査中に値が 0 以外のノードを発見した場合、changed フラグを true に設定する。 3. 対象範囲の全ノードを 0 にクリアする。 4. changed が true の場合、fsm_rebuild_page(page) を呼び出し、変更を反映するためにツリー構造全体を再計算する。 5. 最終的な変更有無（changed）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/freespace/freespace.c:319
  • **呼び出し先**:
  • fsm_rebuild_page (条件付き)

関連データ構造

  • FSMPage: ページ内のノード配列（fp_nodes）を保持する構造体。
  • NonLeafNodesPerPage, NodesPerPage: storage/fsm_internals.h で定義される定数。

失敗モードと境界条件

  • nslots が範囲外の場合、アサーション違反により異常終了する可能性がある。
  • ページ内のノードが既にすべて 0 である場合、changed は false となり、再構築処理はスキップされる。

保守担当者向けメモ

  • 本関数は、FSMページの切り詰め操作において整合性を保つために使用される。
  • fsm_rebuild_page の呼び出しはコストが高いため、実際に値が変更された場合のみ実行されるよう最適化されている。
