ParseLoc

定義

  • 種別: typedef
  • 定義箇所: src/include/nodes/nodes.h:246

目的

SQLクエリ文字列内における特定のトークンや文の開始位置（オフセット）を保持するために使用されます。

シグネチャまたは形状

typedef int ParseLoc;

入力と出力

  • 入力: SQLクエリ文字列内のバイトオフセットを表す整数値。
  • 出力: なし（型定義）。

前提条件と不変条件

  • 値が -1 の場合、位置情報が不明であることを示します。
  • 負の値を許容する整数型として定義されています。

アルゴリズムまたは振る舞い

パーサーがクエリを解析する際、各ノードに関連するソースコード上の位置を記録するために使用されます。エラーメッセージの生成や、クエリの特定部分を指し示す際に参照されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/commands/extension.c
  • src/include/nodes/parsenodes.h（多数のノード構造体でフィールドとして使用）
  • 呼び出し先: なし（プリミティブ型のため）。

関連データ構造

  • parsenodes.h 内の各種ノード構造体（例: location フィールドとして保持される）。

失敗モードと境界条件

  • 位置情報が特定できない場合は -1 が代入されます。
  • 文字列の長さを超える値が設定された場合の挙動は、利用側のロジックに依存します。

保守担当者向けメモ

  • 本型は単なる int の別名であるため、型安全性を高めるための静的解析やアサーションは、利用側の構造体定義や関数実装で行う必要があります。
