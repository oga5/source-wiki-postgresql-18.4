PQping

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-connect.c:835

目的

指定された接続文字列を使用してデータベースサーバーへの接続を試行し、サーバーが応答可能かどうかを確認（ping）します。

シグネチャまたは形状

PGPing PQping(const char *conninfo);

入力と出力

  • **入力**: conninfo (PostgreSQL 接続文字列)
  • **出力**: PGPing 型の列挙値（サーバーの状態を示す）

前提条件と不変条件

  • conninfo は有効な接続文字列である必要があります。
  • 内部で PQconnectStart を呼び出すため、呼び出し元は libpq の接続ライフサイクル管理に準拠する必要があります。

アルゴリズムまたは振る舞い

1. PQconnectStart(conninfo) を呼び出し、接続プロセスを開始します。 2. internal_ping(conn) を呼び出し、接続状態を確認します。 3. PQfinish(conn) を呼び出し、接続に使用したリソースを解放します。 4. 最終的な PGPing の結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/server.c:282 での参照が確認されています。
  • **呼び出し先**:
  • PQconnectStart (接続開始)
  • internal_ping (状態確認)
  • PQfinish (リソース解放)

関連データ構造

  • PGconn: 接続状態を保持する構造体。
  • PGPing: 接続結果を表す列挙型。

失敗モードと境界条件

  • conninfo が無効な場合、PQconnectStart が失敗した状態の PGconn が返され、その後の internal_ping で適切なエラー状態が処理されます。
  • サーバーがダウンしている場合やネットワーク到達不能な場合、internal_ping はそれに応じた PGPing 値を返します。
