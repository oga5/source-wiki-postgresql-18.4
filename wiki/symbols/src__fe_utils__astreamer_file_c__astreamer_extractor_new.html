astreamer_extractor_new

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_file.c:185

目的

アーカイブストリーミングフレームワークにおいて、受信したアーカイブメンバーをファイルシステム上に展開するための astreamer_extractor コンテキストを初期化し、生成します。

シグネチャまたは形状

astreamer *astreamer_extractor_new(const char *basepath,
                                   const char *(*link_map) (const char *),
                                   void (*report_output_file) (const char *))

入力と出力

  • **入力**:
  • basepath: 展開先のルートディレクトリパス。
  • link_map: シンボリックリンクのターゲットを変換するためのコールバック関数（任意）。
  • report_output_file: 展開されたファイル名を報告するためのコールバック関数（任意）。
  • **出力**:
  • 初期化された astreamer 構造体へのポインタを返します。

前提条件と不変条件

  • palloc0 によりメモリがゼロクリアされた状態で astreamer_extractor が割り当てられます。
  • base.bbs_ops には astreamer_extractor_ops が静的に割り当てられます。
  • basepath は内部で pstrdup され、コンテキストの一部として保持されます。

アルゴリズムまたは振る舞い

1. astreamer_extractor 構造体分のメモリを確保します。 2. astreamer_extractor_ops を基底構造体の操作テーブルとして設定します。 3. 引数として渡された basepath を複製して保持します。 4. link_map および report_output_file コールバックをコンテキストに格納します。 5. 初期化済みの astreamer 基底構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_basebackup.c:1164 から呼び出されていることが確認されています。
  • **呼び出し先**: palloc0、pstrdup（PostgreSQLメモリ管理関数）を呼び出します。

関連データ構造

  • astreamer_extractor: 本関数で生成されるコンテキスト構造体。
  • astreamer: 基底構造体。
  • astreamer_extractor_ops: 展開処理の各フェーズ（content, finalize, free）を定義する操作テーブル。

失敗モードと境界条件

  • メモリ不足時には palloc0 または pstrdup がエラーを発生させ、プロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は astreamer_extractor のインスタンス生成のみを担当します。実際の展開ロジックは astreamer_extractor_content 等の操作関数に委譲されています。
  • link_map や report_output_file が NULL の場合、それぞれの機能は無効化されます。
