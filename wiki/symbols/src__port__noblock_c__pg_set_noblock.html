pg_set_noblock

定義

  • 種別: function
  • 定義箇所: src/port/noblock.c:24

目的

指定されたソケット（ファイル記述子）を非ブロッキングモードに設定するためのポータブルなラッパー関数です。プラットフォーム固有のシステムコール差異を隠蔽し、統一されたインターフェースを提供します。

シグネチャまたは形状

bool pg_set_noblock(pgsocket sock)

入力と出力

  • **入力**: pgsocket sock（設定対象のソケット記述子）
  • **出力**: bool（成功時に true、失敗時に false を返す）

前提条件と不変条件

  • pgsocket 型は c.h に定義されている必要があります。
  • POSIX 環境では <fcntl.h> が利用可能である必要があります。

アルゴリズムまたは振る舞い

プラットフォームに応じて以下の処理を行います。 1. **POSIX 環境 (!defined(WIN32))**:

  • fcntl(sock, F_GETFL) で現在のフラグを取得します。失敗時は false を返します。
  • fcntl(sock, F_SETFL, (flags | O_NONBLOCK)) を実行し、非ブロッキングフラグを付与します。
  • fcntl の戻り値が -1 の場合は false を返し、それ以外は true を返します。

    • 2. **Windows 環境 (WIN32)**:

  • ioctlsocket(sock, FIONBIO, &ioctlsocket_ret) を呼び出します（ioctlsocket_ret は 1）。
  • ioctlsocket が 0 を返せば成功（true）、それ以外は失敗（false）と判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/pqcomm.c:295
  • src/backend/postmaster/postmaster.c:3619
  • src/interfaces/libpq/fe-connect.c:3368
  • **呼び出し先**:
  • POSIX: fcntl
  • Windows: ioctlsocket

関連データ構造

  • pgsocket: src/include/c.h で定義されるソケット型。

失敗モードと境界条件

  • fcntl または ioctlsocket がシステムエラーを返した場合、関数は false を返します。
  • プラットフォーム間でエラー判定基準が異なります（POSIX は -1、Windows は非ゼロでエラー）。

  • 修正を行う際は、POSIX と Windows の両方のエラーハンドリング仕様に留意してください。
