makeTargetEntry

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/makefuncs.c:288

目的

クエリの射影リスト（TargetList）を構成する TargetEntry ノードを生成し、初期化する。

シグネチャまたは形状

TargetEntry *makeTargetEntry(Expr *expr, AttrNumber resno, char *resname, bool resjunk)

入力と出力

  • **入力**:
  • expr: 射影対象となる式。
  • resno: 射影リスト内での属性番号。
  • resname: 属性名（または NULL）。
  • resjunk: 内部処理用（Junk）属性かどうかを示すフラグ。
  • **出力**:
  • 初期化済みの TargetEntry 構造体へのポインタ。

前提条件と不変条件

  • makeNode(TargetEntry) を介して CurrentMemoryContext にメモリが割り当てられる。
  • ressortgroupref は 0、resorigtbl は InvalidOid、resorigcol は 0 に固定初期化される。これらを変更する場合は、呼び出し側が明示的に設定する必要がある。

アルゴリズムまたは振る舞い

1. makeNode(TargetEntry) によりメモリを確保し、ノード型を初期化する。 2. 引数 expr, resno, resname, resjunk を構造体の対応するフィールドに代入する。 3. 頻繁に変更されないフィールド（ressortgroupref, resorigtbl, resorigcol）をデフォルト値で初期化する。 4. 構築した TargetEntry ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/executor/functions.c
  • src/backend/executor/nodeSubplan.c
  • src/backend/optimizer/plan/createplan.c
  • **呼び出し先**:
  • makeNode（マクロ）

関連データ構造

  • TargetEntry: 本関数が生成するノード構造体。
  • Expr: TargetEntry が保持する式ノード。

失敗モードと境界条件

  • メモリ不足時には makeNode 経由でエラーが発生する可能性がある。
  • resname に NULL を渡すことは許容される。

保守担当者向けメモ

  • 本関数は、頻繁に使用されるフィールドのみを引数に取ることで、呼び出し側のコードを簡素化している。特殊なフィールド設定が必要な場合は、本関数を呼び出した後に手動でフィールドを上書きすること。
