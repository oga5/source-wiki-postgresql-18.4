BloomInitPage

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blutils.c:403

目的

Bloom インデックスのページを初期化し、PostgreSQL のページ構造および Bloom インデックス固有のメタデータを設定します。

シグネチャまたは形状

void BloomInitPage(Page page, uint16 flags)

入力と出力

  • **入力**:
  • page: 初期化対象となるメモリ上のページバッファ。
  • flags: ページの状態を示すフラグ（例: BLOOM_META）。
  • **出力**: なし（page が直接更新されます）。

前提条件と不変条件

  • page は BLCKSZ バイトのメモリ領域として確保されている必要があります。
  • ページ内の BloomPageOpaqueData 領域が、インデックスのメタデータ保持のために使用されます。

アルゴリズムまたは振る舞い

1. PageInit を呼び出し、ページ全体を標準的な PostgreSQL ページ形式で初期化します。 2. BloomPageGetOpaque を使用して、ページ末尾の BloomPageOpaque 領域を取得します。 3. 指定された flags を opaque->flags に格納します。 4. opaque->bloom_page_id に定数 BLOOM_PAGE_ID を設定し、Bloom インデックスのページであることを識別可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blinsert.c: インデックス構築およびタプル挿入時のページ初期化に使用。
  • contrib/bloom/blutils.c: メタページの初期化（BloomInitMetapage 等）に使用。
  • **呼び出し先**:
  • PageInit (PostgreSQL 内部関数)
  • BloomPageGetOpaque (マクロ)

関連データ構造

  • BloomPageOpaqueData: ページ末尾に配置される Bloom インデックス固有のヘッダ構造体。
  • Page: PostgreSQL の標準的なページ表現。

失敗モードと境界条件

  • page が NULL または BLCKSZ 未満の領域である場合、メモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はページレイアウトの初期化のみを行います。ページ内のデータ領域のクリアや、WAL ログへの記録は呼び出し側の責任で行う必要があります。
  • BLOOM_PAGE_ID の値は、インデックスのバージョン管理や整合性チェックに使用されるため、変更時には注意してください。
