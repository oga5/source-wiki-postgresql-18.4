pg_b64_decode

定義

  • 種別: function
  • 定義箇所: src/common/base64.c:115

目的

Base64形式でエンコードされた文字列をバイナリデータに復元します。

シグネチャまたは形状

int pg_b64_decode(const char *src, int len, uint8 *dst, int dstlen);

入力と出力

  • **入力**:
  • src: デコード対象のBase64文字列。
  • len: src のバイト長。
  • dst: デコード結果を格納するバッファ。
  • dstlen: dst バッファの最大サイズ。
  • **出力**:
  • 成功時: デコードされたバイナリのバイト数（正の整数）。
  • 失敗時: -1。

前提条件と不変条件

  • 入力文字列に空白文字（スペース、タブ、改行、復帰）が含まれていないこと。
  • dst バッファは、デコード結果を格納するのに十分なサイズであること。
  • 処理中、出力バッファの書き込み位置が dstlen を超えないよう厳格にチェックされます。

アルゴリズムまたは振る舞い

1. 入力文字列を1文字ずつ走査し、b64lookup テーブルを用いて6ビットの数値に変換します。 2. 4文字（24ビット）単位で蓄積し、3バイトのバイナリデータに復元します。 3. パディング文字 = が出現した場合、シーケンスの終了とみなし、パディング数に応じて出力バイト数を調整します。 4. 変換中に不正な文字、空白、またはバッファオーバーフローを検知した場合は、エラー処理へ移行します。 5. 正常終了時、書き込んだバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c (SCRAM認証関連)
  • src/interfaces/libpq/fe-auth-scram.c
  • src/interfaces/libpq/fe-connect.c
  • **呼び出し先**:
  • memset (エラー発生時のバッファクリア)
  • b64lookup (文字からインデックスへの変換テーブル)

関連データ構造

  • b64lookup: デコード時に使用される文字変換用ルックアップテーブル。

失敗モードと境界条件

  • **エラー発生時**: memset により dst バッファ全体をゼロクリアし、-1 を返します。
  • **境界条件**:
  • 空白文字が含まれる場合、エラーとなります。
  • 入力文字列の長さが4の倍数でない、またはパディングが不正な場合、エラーとなります。
  • 出力バッファサイズが不足する場合、エラーとなります。

保守担当者向けメモ

  • 本関数は厳格なBase64デコーダーであり、空白文字を許容しません。
  • 呼び出し側は、事前に pg_b64_dec_len を使用して必要なバッファサイズを確保することが推奨されます。
  • エラー発生時の dst ゼロクリアは、セキュリティ上の機密情報漏洩を防ぐための重要な処理です。
