xl_xact_subxacts

定義

  • 種別: struct
  • 定義箇所: src/include/access/xact.h:261

目的

トランザクションのコミットまたはアボート時に、そのトランザクションに関連するサブトランザクションのIDリストをWAL（Write-Ahead Logging）へ記録するための構造体です。

シグネチャまたは形状

typedef struct xl_xact_subxacts
{
    int         nsubxacts;
    TransactionId subxacts[FLEXIBLE_ARRAY_MEMBER];
} xl_xact_subxacts;

入力と出力

  • **入力**: トランザクション終了処理において、現在アクティブなサブトランザクションの数（nsubxacts）と、それらのトランザクションID配列（subxacts）が格納されます。
  • **出力**: WALレコードの一部としてシリアライズされ、リカバリ時にサブトランザクションの状態を再現するために使用されます。

前提条件と不変条件

  • subxacts は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、構造体自体のサイズは offsetof(xl_xact_subxacts, subxacts) で計算される MinSizeOfXactSubxacts を基準とします。
  • WALレコード内に本構造体が存在するかどうかは、XINFO_HAS_SUBXACT フラグによって制御されます。

アルゴリズムまたは振る舞い

1. トランザクション終了時、サブトランザクションが存在する場合に本構造体がWALレコードに書き込まれます。 2. リカバリ時、xact_redo 等の処理において、WALデータから本構造体へキャストしてサブトランザクションIDリストを読み取ります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c 内のトランザクション終了処理（コミット/アボート）において、WALログ記録時にインスタンス化されます。
  • **呼び出し先**: src/backend/access/rmgrdesc/xactdesc.c において、WALレコードの解析およびデバッグ出力のために参照されます。

関連データ構造

  • TransactionId: サブトランザクションを識別する型。
  • xl_xact_commit / xl_xact_abort: 本構造体が含まれる可能性のある親WALレコード構造体。

失敗モードと境界条件

  • サブトランザクション数が0の場合、本構造体はWALレコードに含まれない（または nsubxacts が0となる）設計です。
  • 可変長配列のため、直接的なメモリ操作を行う際は必ず MinSizeOfXactSubxacts を用いたオフセット計算が必要です。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体ポインタを直接扱う際はメモリレイアウトに注意してください。
  • WALレコードの解析には、必ず ParseCommitRecord 等の提供されている解析関数を経由することを推奨します。
