IsPagerNeeded

定義

  • 種別: function
  • 定義箇所: src/fe_utils/print.c:3402

目的

出力内容の行数に基づき、ページャ（more/less等）が必要かどうかを判定し、出力先ストリームを適切に設定する。

シグネチャまたは形状

static void IsPagerNeeded(const printTableContent *cont, int extra_lines, bool expanded, FILE **fout, bool *is_pager)

入力と出力

  • cont: 出力対象のテーブル内容（printTableContent）。
  • extra_lines: ページャ起動判定に加算する追加行数。
  • expanded: 拡張表示モード（expanded）かどうかを示すフラグ。
  • fout: 出力先ファイルポインタ。stdout の場合にのみページャの起動を検討し、必要に応じて更新される。
  • is_pager: ページャが起動されたかどうかを返す出力用フラグ。

前提条件と不変条件

  • *fout が stdout である場合のみ、ページャの起動判定が行われる。
  • *fout が stdout 以外の場合、*is_pager は常に false となる。

アルゴリズムまたは振る舞い

1. *fout が stdout かどうかを確認する。 2. expanded モードに応じて、基本行数を計算する。

  • expanded が true の場合: (列数 + 1) * 行数
  • expanded が false の場合: 行数 + 1

    • 3. tuples_only が false の場合、フッターの数（cont->footers のリスト長）を lines に加算する。 4. PageOutput(lines + extra_lines, cont->opt) を呼び出し、必要に応じてページャを起動して *fout を更新する。 5. *fout が stdout と異なる場合、*is_pager を true に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/fe_utils/print.c 内の複数箇所（printTable 等の出力処理関数）。
  • 呼び出し先: PageOutput（src/fe_utils/print.c 内）。

関連データ構造

  • printTableContent: テーブルのデータ、行数、列数、フッター、オプションを保持する構造体。

失敗モードと境界条件

保守担当者向けメモ

  • 本関数は stdout への出力を前提としているため、ファイル出力や他のストリームへのリダイレクト時にはページャが起動されない設計となっている。
  • ページャの起動判定ロジックを変更する場合は、PageOutput の仕様と整合性を保つこと。
