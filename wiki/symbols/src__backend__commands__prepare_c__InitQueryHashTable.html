InitQueryHashTable

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:371

目的

セッション内でプリペアドステートメントを管理するためのハッシュテーブル prepared_queries を初期化します。

シグネチャまたは形状

static void InitQueryHashTable(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 prepared_queries を初期化する副作用を持つ）

前提条件と不変条件

  • prepared_queries が未初期化であること。
  • ハッシュテーブルのキーサイズは NAMEDATALEN、エントリサイズは PreparedStatement 構造体のサイズに固定されます。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を設定し、キーサイズを NAMEDATALEN、エントリサイズを sizeof(PreparedStatement) に指定します。 2. hash_create を呼び出し、"Prepared Queries" という名称で初期サイズ 32 のハッシュテーブルを作成します。 3. 作成したハッシュテーブルのポインタをグローバル変数 prepared_queries に格納します。フラグとして HASH_ELEM および HASH_STRINGS を使用します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/prepare.c 内の他の関数（具体的な呼び出し箇所はソースコード上の参照として存在しますが、本関数がどのタイミングで実行されるかの詳細な制御フローは提供されていません）。
  • 呼び出し先: hash_create（PostgreSQL のハッシュテーブル管理ユーティリティ）。

関連データ構造

  • prepared_queries: プリペアドステートメントを保持するハッシュテーブル。
  • PreparedStatement: ハッシュテーブルのエントリとして格納される構造体。

失敗モードと境界条件

  • hash_create がメモリ不足等で失敗した場合、PostgreSQL のエラーハンドリング機構により ereport(ERROR) が発生します。

保守担当者向けメモ

  • 本関数はセッションローカルなハッシュテーブルを構築します。このテーブルはバックエンドの生存期間中、プリペアドステートメントの永続的なストレージとして機能します。
