like_escape_bytea

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:448

目的

bytea 型のパターン文字列に対して、指定されたエスケープ文字を適用した新しいパターン文字列を生成します。

シグネチャまたは形状

Datum like_escape_bytea(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BYTEA_PP(0): パターン文字列 (bytea *)
  • PG_GETARG_BYTEA_PP(1): エスケープ文字 (bytea *)
  • **出力**:
  • PG_RETURN_BYTEA_P: エスケープ処理後の bytea * 型の結果

前提条件と不変条件

  • 入力引数は bytea 型のポインタとして取得されます。
  • 内部的に text 型へのキャストを行って処理を委譲するため、入力データは text 型として解釈可能な形式である必要があります。

アルゴリズムまたは振る舞い

1. 引数からパターン文字列とエスケープ文字列を取得します。 2. SB_do_like_escape 関数を呼び出し、bytea 型の入力を text 型にキャストしてエスケープ処理を実行します。 3. 処理結果を PG_RETURN_BYTEA_P を通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_ARGS を介した呼び出し）。
  • **呼び出し先**: SB_do_like_escape

関連データ構造

  • bytea: PostgreSQLの可変長バイナリデータ型。
  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • 入力が無効な bytea 構造である場合、基盤となる PG_GETARG_BYTEA_PP マクロによりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は SB_do_like_escape への単なるラッパーです。ロジックの変更が必要な場合は SB_do_like_escape の実装を確認してください。
  • bytea と text の相互キャストを行っているため、型変換時のメモリレイアウトに注意してください。
