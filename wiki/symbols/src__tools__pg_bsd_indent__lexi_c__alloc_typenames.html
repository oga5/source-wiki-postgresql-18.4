alloc_typenames

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/lexi.c:676

目的

ユーザー定義の型名を保持するための動的配列 typenames を初期化し、メモリを割り当てる。

シグネチャまたは形状

void alloc_typenames(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 typenames および typename_count を更新）

前提条件と不変条件

  • typenames は本関数によって初期化される必要がある。
  • typename_count は初期値として 16 が設定される。

アルゴリズムまたは振る舞い

1. malloc を使用して、const char * 型のポインタを 16 個格納できるメモリ領域を確保する。 2. 確保した領域の先頭アドレスをグローバル変数 typenames に代入する。 3. typename_count に 16 を代入する。 4. メモリ確保に失敗した場合は err(1, NULL) を呼び出し、プログラムを終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/tools/pg_bsd_indent/indent.c:112
  • 呼び出し先: malloc (標準ライブラリ), err (標準ライブラリ)

関連データ構造

  • typenames: 型名を保持する動的配列（const char **）。
  • typename_count: typenames 配列の現在の容量（整数）。

失敗モードと境界条件

  • メモリ不足時: malloc が NULL を返した場合、err 関数によりエラーメッセージを出力してプロセスが終了する。

保守担当者向けメモ

  • 本関数で確保される初期容量は 16 です。add_typename 関数において、この容量を超過した場合の動的な再割り当て処理が適切に行われているか確認してください。
