utf8_to_unicode

定義

  • 種別: function
  • 定義箇所: src/fe_utils/mbprint.c:52

目的

UTF-8 でエンコードされたマルチバイト文字の先頭バイト列を、対応する Unicode コードポイント（pg_wchar）に変換します。

シグネチャまたは形状

static pg_wchar utf8_to_unicode(const unsigned char *c)

入力と出力

  • **入力**: c - UTF-8 文字列の先頭を指す unsigned char 型のポインタ。
  • **出力**: 変換された Unicode コードポイントを pg_wchar 型で返します。

前提条件と不変条件

  • 入力ポインタ c は、少なくとも 1 バイト以上の有効なメモリ領域を指している必要があります。
  • UTF-8 のバイトシーケンスが不正な場合、戻り値として 0xffffffff が返されます。

アルゴリズムまたは振る舞い

入力されたバイトの先頭ビットパターンに基づき、以下の通り変換を行います。 1. 0xxxxxxx: 1 バイト文字（ASCII）としてそのまま返します。 2. 110xxxxx: 2 バイト文字として、下位 5 ビットと 6 ビットを結合します。 3. 1110xxxx: 3 バイト文字として、下位 4 ビット、6 ビット、6 ビットを結合します。 4. 11110xxx: 4 バイト文字として、下位 3 ビット、6 ビット、6 ビット、6 ビットを結合します。 5. 上記以外: 不正なシーケンスとみなし 0xffffffff を返します。

  • **呼び出し先**:
  • 本関数は算術演算のみを実行し、他の関数を呼び出しません。

関連データ構造

  • pg_wchar: Unicode コードポイントを保持するための型。

失敗モードと境界条件

  • 入力が UTF-8 として不正なバイトパターンである場合、0xffffffff を返します。
  • 4 バイトを超えるシーケンスや、不適切な継続バイトを持つシーケンスは、すべて失敗として扱われます。

保守担当者向けメモ

  • 本関数は src/fe_utils/mbprint.c 内で static として定義されていますが、複数のディレクトリ（src/backend, src/common, contrib）から参照されています。
  • ロジックを変更する際は、UTF-8 の仕様（RFC 3629）に準拠しているか確認してください。
  • pg_wcssize や pg_wcsformat と同様に、マルチバイト処理の基盤となるため、変更時の影響範囲は広範囲に及びます。
