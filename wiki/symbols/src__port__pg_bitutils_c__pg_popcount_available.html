pg_popcount_available

定義

  • 種別: function
  • 定義箇所: src/port/pg_bitutils.c:138

目的

実行時のCPU機能判定を行い、ハードウェアによる POPCNT 命令のサポート状況を確認する。

シグネチャまたは形状

static bool pg_popcount_available(void)

入力と出力

  • 入力: なし
  • 出力: bool 型。POPCNT 命令が利用可能な場合は true、それ以外は false を返す。

前提条件と不変条件

  • コンパイル時に HAVE__GET_CPUID または HAVE__CPUID のいずれかが定義されている必要がある。
  • 定義されていない場合、コンパイルエラーとなる。

アルゴリズムまたは振る舞い

1. 4つの要素を持つ符号なし整数配列 exx を初期化する。 2. 環境に応じて __get_cpuid または __cpuid を呼び出し、CPU情報を取得する。 3. 取得した情報のうち、ECXレジスタ（exx[2]）の第23ビットを確認する。 4. 第23ビットがセットされていれば POPCNT 命令がサポートされていると判定し、その結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/pg_bitutils.c 内の choose_popcount_functions（pg_popcount_available() を呼び出し、その結果に基づいて関数ポインタを切り替える）。
  • 呼び出し先: __get_cpuid または __cpuid（プラットフォーム固有のCPUID取得関数）。

関連データ構造

  • なし

失敗モードと境界条件

  • cpuid 命令が利用できない環境では、コンパイル時に #error により停止する。

保守担当者向けメモ

  • 本関数はx86_64環境における動的ディスパッチの基盤となる。
  • CPUIDのビット判定ロジックはハードウェア仕様に依存するため、変更時はIntel/AMDの仕様書を確認すること。
