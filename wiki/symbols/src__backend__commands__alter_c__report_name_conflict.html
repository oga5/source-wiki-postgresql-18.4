report_name_conflict

定義

  • 種別: function
  • 定義箇所: src/backend/commands/alter.c:75

目的

オブジェクトの名称重複エラーを標準化された形式で報告する。

シグネチャまたは形状

static void report_name_conflict(Oid classId, const char *name)

入力と出力

  • **入力**:
  • classId (Oid): 対象オブジェクトのカタログID。
  • name (const char *): 重複しているオブジェクト名。
  • **出力**: なし。エラー発生時に ereport を介して処理を中断する。

前提条件と不変条件

  • classId は、関数内で定義された特定のシステムカタログID（EventTriggerRelationId 等）のいずれかである必要がある。
  • 未対応の classId が渡された場合、elog(ERROR) により強制終了する。

アルゴリズムまたは振る舞い

1. classId に基づき、対応するエラーメッセージのフォーマット文字列（msgfmt）を選択する。 2. ereport を呼び出し、ERRCODE_DUPLICATE_OBJECT エラーコードと共に、指定された名前を埋め込んだエラーメッセージを生成してスローする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/alter.c 内のオブジェクト変更処理（ALTER ステートメント実行時など）。
  • **呼び出し先**: gettext_noop（メッセージ翻訳用）、elog、ereport。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • EventTriggerRelationId, ForeignDataWrapperRelationId 等のカタログID定数。

失敗モードと境界条件

  • classId がサポート対象外の場合、elog(ERROR) によりバックエンドプロセスが停止する。
  • 呼び出し元で既にオブジェクトの存在確認が済んでいることを前提としている。

保守担当者向けメモ

  • 新しいオブジェクト型をサポートする場合は、switch 文に新しい case を追加し、対応するエラーメッセージを定義すること。
