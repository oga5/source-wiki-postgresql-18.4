tidrecv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:138

目的

クライアントから送信されたバイナリ形式のデータ（TID）を読み取り、PostgreSQL 内部で使用される ItemPointer 構造体に変換します。

シグネチャまたは形状

Datum tidrecv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して取得される StringInfo 型のバッファ。このバッファには、BlockNumber（ブロック番号）と OffsetNumber（オフセット番号）がバイナリ形式で格納されている必要があります。
  • **出力**: 変換された ItemPointer を指す Datum を返却します。

前提条件と不変条件

  • 入力バッファは libpq のバイナリプロトコルに従い、BlockNumber と OffsetNumber の順序でデータが配置されている必要があります。
  • palloc を使用してメモリを確保するため、メモリコンテキストが適切に設定されている必要があります。

アルゴリズムまたは振る舞い

1. pq_getmsgint を使用して、バッファから BlockNumber を読み取ります。 2. 同様に pq_getmsgint を使用して、バッファから OffsetNumber を読み取ります。 3. palloc を用いて ItemPointerData 構造体分のメモリを動的に確保します。 4. ItemPointerSet マクロを使用して、読み取ったブロック番号とオフセット番号を確保したメモリ領域に格納します。 5. PG_RETURN_ITEMPOINTER マクロを介して、構築された ItemPointer を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し元は確認されていませんが、PostgreSQL の型システム（fmgr）を通じて、バイナリ形式の TID を受け取る際に動的に呼び出されます。
  • **呼び出し先**:
  • pq_getmsgint: バイナリバッファからの数値読み取り。
  • palloc: メモリ確保。
  • ItemPointerSet: ItemPointer 構造体への値設定。

関連データ構造

  • ItemPointerData: TID の内部表現。
  • StringInfo: バイナリデータ転送用のバッファ構造体。

失敗モードと境界条件

  • pq_getmsgint がバッファの終端に達した場合や、不正なデータ形式が渡された場合、基盤となる libpq のエラーハンドリングにより例外が発生する可能性があります。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は TID 型のバイナリ受信を担当します。TID の内部構造（BlockIdData と OffsetNumber）に変更があった場合は、pq_getmsgint の引数や ItemPointerSet の呼び出しに影響が出る可能性があるため注意してください。
