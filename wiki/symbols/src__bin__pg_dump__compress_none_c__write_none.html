write_none

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_none.c:99

目的

圧縮を行わないデータストリームにおいて、指定されたバッファの内容をファイルへ直接書き出すためのインターフェースを提供します。

シグネチャまたは形状

static void write_none(const void *ptr, size_t size, CompressFileHandle *CFH)

入力と出力

  • ptr: 書き込むデータが格納されたバッファへのポインタ。
  • size: 書き込むデータのバイト数。
  • CFH: 圧縮ファイルハンドル。private_data メンバに FILE* が格納されていることが前提となります。
  • 出力: なし（成功時）。失敗時は pg_fatal によりプロセスが終了します。

前提条件と不変条件

  • CFH->private_data は有効な FILE* である必要があります。
  • fwrite が期待するバイト数を書き込めない場合、エラーとして扱われます。

アルゴリズムまたは振る舞い

1. errno を 0 にリセットします。 2. fwrite を使用して、ptr から size バイトを CFH->private_data が指すファイルストリームへ書き出します。 3. 書き込み結果（ret）が要求された size と一致しない場合、errno が設定されていればそれを保持し、設定されていなければ ENOSPC を割り当てた上で pg_fatal を呼び出し、エラーメッセージを出力して終了します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/compress_none.c 内で CFH->write_func に割り当てられます。具体的な呼び出し箇所はソースコードの文脈からは特定できません。
  • 呼び出し先: 標準ライブラリの fwrite およびエラー報告用の pg_fatal。

関連データ構造

  • CompressFileHandle: ファイル操作の抽象化に使用される構造体。

失敗モードと境界条件

  • 書き込みバイト数が要求と一致しない場合、pg_fatal により致命的エラーとして処理されます。
  • errno が設定されていない書き込み失敗時には、ディスクフルを想定して ENOSPC が使用されます。

保守担当者向けメモ

  • 本関数は CompressFileHandle の write_func インターフェースに適合するラッパーです。
  • private_data の型キャスト（FILE*）に依存しているため、構造体の定義変更時には注意が必要です。
