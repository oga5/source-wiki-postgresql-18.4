MakeTupleTableSlot

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execTuples.c:1300

目的

実行エンジンにおけるタプル保持用コンテナである TupleTableSlot を新規に割り当て、初期化する。

シグネチャまたは形状

TupleTableSlot *MakeTupleTableSlot(TupleDesc tupleDesc, const TupleTableSlotOps *tts_ops)

入力と出力

  • **入力**:
  • tupleDesc: スロットが扱うタプルの記述子（NULL 可）。
  • tts_ops: スロットの振る舞いを定義する関数テーブル（TupleTableSlotOps）。
  • **出力**:
  • 初期化済みの TupleTableSlot 構造体へのポインタ。

前提条件と不変条件

  • tts_ops は有効な関数テーブルを指している必要がある。
  • tupleDesc が指定された場合、スロットの属性値配列（tts_values）と NULL 配列（tts_isnull）は、スロット構造体の直後にメモリ配置される。
  • tupleDesc が指定された場合、PinTupleDesc により参照カウントが増加する。

アルゴリズムまたは振る舞い

1. tts_ops->base_slot_size を基に、必要なメモリサイズを計算する。tupleDesc がある場合は、属性数に応じた Datum 配列と bool 配列の領域を MAXALIGN を考慮して追加確保する。 2. palloc0 でメモリを確保し、tts_ops を設定する。 3. スロットのフラグを TTS_FLAG_EMPTY に設定し、tupleDesc があれば TTS_FLAG_FIXED を付与する。 4. tts_mcxt に現在のメモリコンテキストを記録する。 5. tupleDesc がある場合、確保したメモリ領域内の適切なオフセット位置を tts_values および tts_isnull に割り当てる。 6. tts_ops->init(slot) を呼び出し、スロット形式固有の初期化を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execMain.c (複数箇所)
  • src/backend/executor/execTuples.c
  • src/backend/replication/pgoutput/pgoutput.c
  • **呼び出し先**:
  • palloc0
  • PinTupleDesc
  • tts_ops->init (関数ポインタ経由)

関連データ構造

  • TupleTableSlot: タプルのコンテナ。
  • TupleTableSlotOps: スロットの操作を抽象化する関数テーブル。
  • TupleDesc: タプルの属性定義。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • tupleDesc が NULL の場合、属性配列の確保は行われず、TTS_FLAG_FIXED フラグも設定されない。

保守担当者向けメモ

  • 本関数はスロットのメモリレイアウトを決定する重要な箇所である。tts_ops の実装を変更する際は、base_slot_size との整合性に注意すること。
  • tupleDesc を渡す場合、スロットのライフサイクル終了時に PinTupleDesc と対になる解放処理が適切に行われることを確認すること。
