tarHeaderOffset

定義

  • 種別: enum
  • 定義箇所: src/include/pgtar.h:37

目的

tar アーカイブの 512 バイトヘッダーブロック内における、各フィールドの開始オフセット（バイト単位）を定義する。

シグネチャまたは形状

enum tarHeaderOffset {
    TAR_OFFSET_NAME = 0,
    TAR_OFFSET_MODE = 100,
    TAR_OFFSET_UID = 108,
    TAR_OFFSET_GID = 116,
    TAR_OFFSET_SIZE = 124,
    TAR_OFFSET_MTIME = 136,
    TAR_OFFSET_CHECKSUM = 148,
    TAR_OFFSET_TYPEFLAG = 156,
    TAR_OFFSET_LINKNAME = 157,
    TAR_OFFSET_MAGIC = 257,
    TAR_OFFSET_VERSION = 263,
    TAR_OFFSET_UNAME = 265,
    TAR_OFFSET_GNAME = 297,
    TAR_OFFSET_DEVMAJOR = 329,
    TAR_OFFSET_DEVMINOR = 337,
    TAR_OFFSET_PREFIX = 345
};

入力と出力

  • 入力: なし（定数定義）
  • 出力: 各フィールドの開始位置を示す整数値

前提条件と不変条件

  • tar ヘッダーブロックは 512 バイトで構成される。
  • 定義されたオフセットは、この 512 バイトブロック内での相対位置である。
  • ブロックの末尾 12 バイトは未割り当てである。

アルゴリズムまたは振る舞い

tar アーカイブの読み書き処理において、ヘッダー内の特定のメタデータ（ファイル名、サイズ、パーミッション等）へアクセスするためのインデックスとして機能する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定不可。
  • 呼び出し先: なし（定数定義のため）。

関連データ構造

  • TAR_BLOCK_SIZE: tar アーカイブの基本単位（512 バイト）。
  • tarFileType: ファイルの種類を識別するフラグ（TAR_OFFSET_TYPEFLAG で参照）。

失敗モードと境界条件

  • 定義されたオフセット値は固定であり、実行時の失敗モードは存在しない。
  • 512 バイトの境界を超えたアクセスは、tar 形式の仕様違反となる。

保守担当者向けメモ

  • 本列挙型は POSIX 準拠の tar ヘッダー構造に基づいている。
  • 数値フィールド（TAR_OFFSET_MODE 等）を扱う際は、直接値を代入せず print_tar_number 関数を介してエンコードすること。
