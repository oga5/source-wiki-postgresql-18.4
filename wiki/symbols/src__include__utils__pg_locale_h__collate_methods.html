collate_methods

定義

  • 種別: struct
  • 定義箇所: src/include/utils/pg_locale.h:73

目的

照合順序（Collation）処理のプロバイダー（libc, ICU, builtin）を抽象化し、文字列の比較や変換を行うための関数ポインタテーブルを定義する。

シグネチャまたは形状

struct collate_methods {
    int (*strncoll)(const char *arg1, ssize_t len1, const char *arg2, ssize_t len2, pg_locale_t locale);
    size_t (*strnxfrm)(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale);
    size_t (*strnxfrm_prefix)(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale);
    bool strxfrm_is_safe;
};

入力と出力

  • strncoll: 2つの文字列（長さ指定あり）とロケールオブジェクトを受け取り、比較結果（整数）を返す。
  • strnxfrm / strnxfrm_prefix: 文字列を照合可能な形式に変換し、変換後の長さを返す。
  • strxfrm_is_safe: strnxfrm の結果が信頼できるかを示す真偽値。

前提条件と不変条件

  • strncoll および strnxfrm は必須の実装である。
  • strnxfrm_prefix はオプションである。
  • strxfrm_is_safe が false の場合、プランナの推定目的以外での使用は制限される。

アルゴリズムまたは振る舞い

データベース実行時に、指定された pg_locale_t が保持するこの構造体のメソッドを介して、適切なロケール処理（ICUまたはlibcの関数）へディスパッチを行う。

呼び出し元と呼び出し先

  • 呼び出し元: pg_locale_t を利用する文字列操作関数（例: pg_strcoll）。
  • 呼び出し先（実装例）:
  • collate_methods_icu / collate_methods_icu_utf8 (src/backend/utils/adt/pg_locale_icu.c)
  • collate_methods_libc / collate_methods_libc_win32_utf8 (src/backend/utils/adt/pg_locale_libc.c)

関連データ構造

  • pg_locale_t: この構造体へのポインタを保持し、照合順序のプロバイダーをカプセル化する。

失敗モードと境界条件

  • strxfrm_is_safe が false の場合、正確な照合結果を必要とする処理では使用されない。

保守担当者向けメモ

  • 本構造体はポリモーフィズムを実現するためのインターフェースである。新しい照合プロバイダーを追加する場合は、この構造体を実装した静的インスタンスを定義し、pg_newlocale_from_collation で割り当てられるようにする必要がある。
