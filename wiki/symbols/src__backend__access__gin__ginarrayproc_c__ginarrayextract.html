ginarrayextract

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginarrayproc.c:32

目的

GINインデックス構築時、配列型の入力値を個々の要素に分解し、インデックスキーとして登録可能な形式に変換する。

シグネチャまたは形状

Datum ginarrayextract(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取る。

    • 1. ArrayType*: インデックス対象の配列（コピーが作成される）。 2. int32*: 抽出された要素数を格納するポインタ。 3. bool**: 各要素の NULL フラグ配列を格納するポインタ。

  • **出力**: Datum*（抽出された要素の配列）。

前提条件と不変条件

  • 入力配列の型情報（長さ、値渡し可否、アライメント）が get_typlenbyvalalign により正しく取得できること。
  • deconstruct_array を使用して配列を分解するため、戻り値の Datum 配列は入力配列のメモリ領域を指している。そのため、呼び出し側で入力配列を解放してはならない。

アルゴリズムまたは振る舞い

1. PG_GETARG_ARRAYTYPE_P_COPY(0) により、入力配列のコピーを作成し、処理中にデータが消失しないようにする。 2. get_typlenbyvalalign を呼び出し、配列要素の型属性を取得する。 3. deconstruct_array を実行し、配列を Datum の配列（elems）と NULL フラグの配列（nulls）に分解する。 4. 抽出された要素数 nelems を nkeys に代入する。 5. nulls を nullFlags に代入する。 6. 分解された elems を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginarrayproc.c 内の ginarrayextract_2args から、引数チェックを経て呼び出される。
  • **呼び出し先**:
  • get_typlenbyvalalign
  • deconstruct_array

関連データ構造

  • ArrayType: PostgreSQLの配列型構造体。
  • Datum: 汎用データ型。

失敗モードと境界条件

  • 入力配列が空の場合、nelems は 0 となり、空の配列が返される。
  • ginarrayextract_2args から呼び出される際、引数の数が不適切な場合は elog(ERROR, ...) により処理が中断される。

保守担当者向けメモ

  • 本関数は ginarrayextract_2args との互換性を維持している。新規実装では 3 引数版を使用すること。
  • 戻り値の elems は入力配列のメモリを指しているため、メモリ管理には注意が必要である。
