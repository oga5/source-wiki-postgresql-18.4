remove_useless_groupby_columns

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/initsplan.c:423

目的

一意性インデックスを利用して、GROUP BY 句に含まれる冗長なカラムを特定し、削除することでクエリ実行計画を最適化します。

シグネチャまたは形状

void remove_useless_groupby_columns(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（現在のクエリのプランナ情報）
  • **出力**: なし（副作用として root->processed_groupClause を直接操作する可能性がある）

前提条件と不変条件

  • GROUP BY 項目が2つ以上存在すること。
  • クエリに groupingSets が含まれていないこと。
  • 対象となる GROUP BY 項目が現在のクエリレベルの単純な Var であること。
  • 継承テーブル（パーティションテーブルを除く）は対象外。

アルゴリズムまたは振る舞い

1. **初期スキャン**: processed_groupClause を走査し、各リレーションごとの GROUP BY カラムを groupbyattnos および groupbycols に収集します。 2. **最適化判定**: 各リレーションに対し、一意性インデックス（unique かつ immediate、かつ述語なし）を検索します。 3. **機能的依存性の検証**: インデックスの各カラムが NOT NULL であるか、または NULLS NOT DISTINCT が設定されていることを確認します。また、インデックスの演算子族と照合順序が GROUP BY 項目と互換性があるか検証します。 4. **冗長性の特定**: インデックスの全カラムが GROUP BY 項目に含まれている場合、それ以外の GROUP BY カラムは冗長とみなされます（ソースコードの抜粋はここまで）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planmain.c:173
  • **呼び出し先**: get_sortgroupclause_tle, bms_add_member, get_mergejoin_opfamilies, collations_agree_on_equality 等のユーティリティ関数。

関連データ構造

  • PlannerInfo: プランナの全体状態を保持。
  • GroupByColInfo: GROUP BY カラムの属性（attno, eq_opfamilies, coll）を保持する一時構造体。
  • IndexOptInfo: リレーションのインデックス情報。

失敗モードと境界条件

  • GROUP BY 項目が1つ以下の場合は即座に終了します。
  • 式インデックスや、述語付きインデックス（indpred != NIL）はサポート対象外のためスキップされます。
  • 継承テーブルの親テーブルは、子テーブルで重複行が発生する可能性があるためスキップされます。

保守担当者向けメモ

  • 本関数は PlannerInfo を直接書き換えるため、呼び出しタイミングに注意してください。
  • XXX コメントにある通り、現在は単純な Var のみを対象としていますが、将来的には安定した式（stable expressions）の削除も検討の余地があります。
  • インデックスの nullsnotdistinct 属性の扱いは、機能的依存性を判断する上で重要です。
