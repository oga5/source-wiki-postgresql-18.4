plpgsql_compile_callback

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:166

目的

PL/pgSQL 関数のソースコードを解析し、実行可能な PLpgSQL_function 構造体（中間表現）を構築する内部コンパイル処理を実行します。

シグネチャまたは形状

static void plpgsql_compile_callback(
    FunctionCallInfo fcinfo,
    HeapTuple procTup,
    const CachedFunctionHashKey *hashkey,
    CachedFunction *cfunc,
    bool forValidator
)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • procTup: pg_proc システムカタログのタプル。
  • hashkey: キャッシュ検索用キー。
  • cfunc: コンパイル対象の CachedFunction（PLpgSQL_function にキャストして使用）。
  • forValidator: 検証モード（CREATE FUNCTION 時など）か否かを示すフラグ。
  • **出力**: なし（cfunc が指す構造体内部を更新）。

前提条件と不変条件

  • コンパイル中のメモリは plpgsql_compile_tmp_cxt で管理され、成功時にのみ CacheMemoryContext へ昇格されます。
  • 再入不可であり、コンパイル中に別の PL/pgSQL 関数が呼び出されることは想定されていません。
  • forValidator が真の場合、構文チェックが厳格化されます。

アルゴリズムまたは振る舞い

1. **初期化**: スキャナを初期化し、エラー追跡用のコンテキストを設定します。 2. **メモリ管理**: 関数専用のメモリコンテキストを作成し、解析中のメモリ割り当てを管理します。 3. **メタデータ設定**: PLpgSQL_function 構造体に OID、シグネチャ、トリガー種別、名前解決オプション等を設定します。 4. **名前空間構築**: plpgsql_ns_init および plpgsql_ns_push を呼び出し、パラメータ用の名前空間を準備します。 5. **パラメータ処理**: get_func_arg_info で引数情報を取得し、多相型（polymorphic types）の解決を行った後、各引数に対応する変数を生成してデータムリストに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、plpgsql_compile からの呼び出しが想定されています。
  • **呼び出し先**: plpgsql_scanner_init, plpgsql_ns_init, plpgsql_ns_push, plpgsql_start_datums, plpgsql_build_datatype, plpgsql_build_variable, get_func_arg_info, cfunc_resolve_polymorphic_argtypes 等。

関連データ構造

  • PLpgSQL_function: コンパイル結果を保持するメイン構造体。
  • FunctionCallInfo: 実行時の関数呼び出し情報。
  • HeapTuple: pg_proc カタログ情報。
  • PLpgSQL_variable: 関数内の変数定義。

失敗モードと境界条件

  • 引数の型がサポートされていない（疑似型など）場合、ereport(ERROR, ...) により中断されます。
  • コンパイル失敗時は、一時メモリコンテキストが破棄されることでメモリリークを防ぎます。

保守担当者向けメモ

  • コンパイル成功時にのみ func_cxt を CacheMemoryContext へ昇格させる仕組みを維持してください。
  • resolve_column_ref 等の名前解決ロジックと密接に関連しているため、変更時は名前空間のスタック操作に注意が必要です。
