TransactionGroupUpdateXidStatus

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:440

目的

複数のバックエンドプロセスによるトランザクション状態（CLOG）更新をグループ化し、SLRU ページロックの競合を削減することで、コミット処理の並列性能を向上させる。

シグネチャまたは形状

static bool TransactionGroupUpdateXidStatus(TransactionId xid, XidStatus status, XLogRecPtr lsn, int64 pageno)

入力と出力

  • **入力**:
  • xid: 更新対象のトランザクションID。
  • status: 設定するトランザクション状態（コミット/アボート等）。
  • lsn: 関連する WAL の LSN。
  • pageno: 更新対象の CLOG ページ番号。
  • **出力**:
  • bool: グループ更新が成功（または参加）した場合は true、ページ番号の不一致等によりグループ化できず、呼び出し元で個別に更新すべき場合は false。

前提条件と不変条件

  • TransactionIdIsValid(xid) が真であること。
  • 呼び出し元は、サブトランザクション数が THRESHOLD_SUBTRANS_CLOG_OPT 以下であることを保証する必要がある。
  • グループリーダーは、自身のページと異なるページを更新しようとする後続プロセスをグループに含めない。

アルゴリズムまたは振る舞い

1. **グループ参加**: 呼び出し元プロセスは自身の情報を PGPROC に格納し、ProcGlobal->clogGroupFirst を通じてキューイングを試みる。 2. **リーダー判定**:

  • 既にリストが存在し、ページ番号が一致する場合、リーダーの更新完了を待機する（セマフォによるスリープ）。
  • リストが空、または自身が最初のプロセスである場合、自身がリーダーとなる。

    • 3. **リーダーの処理**:

  • 対象ページの SLRU バンクロックを LW_EXCLUSIVE で取得する。
  • キュー内の全プロセスを順次処理し、TransactionIdSetPageStatusInternal を呼び出して状態を書き込む。
  • 処理中にページ番号が変わる場合は、必要に応じてロックを解放・再取得する。
  • 全ての更新完了後、参加プロセスをセマフォで起床させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/clog.c 内の TransactionIdSetTreeStatus から呼び出される。
  • **呼び出し先**:
  • SimpleLruGetBankLock
  • LWLockAcquire / LWLockRelease
  • TransactionIdSetPageStatusInternal
  • PGSemaphoreLock / PGSemaphoreUnlock

関連データ構造

  • PGPROC: 各バックエンドのグループ更新用状態（clogGroupMember, clogGroupMemberXid 等）を保持。
  • ProcGlobal: グループの先頭プロセスを指す clogGroupFirst を保持。
  • XactCtl: SLRU 制御構造体。

失敗モードと境界条件

  • **ページ不一致**: 異なる CLOG ページを更新しようとするプロセスが混在した場合、グループ化を中断し false を返す。
  • **ロック競合**: 異なる SLRU バンクへのアクセスが発生した場合、リーダーはロックを切り替える。
  • **ABA 問題**: clogGroupFirst の操作において、アトミックな交換処理により回避している。

保守担当者向けメモ

  • 本関数は TransactionIdSetTreeStatus の最適化パスである。
  • ページ更新中にロックを切り替える際、既に更新済みのプロセスの起床を待たずにロックを解放しているが、これは実装の簡略化のためであり、さらなる最適化の余地がある。
