WakeupWalSummarizer

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/walsummarizer.c:640

目的

WAL Summarizer プロセスに対して、待機状態から処理を再開するよう通知（シグナル送信）を行う。

シグネチャまたは形状

void WakeupWalSummarizer(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として WAL Summarizer プロセスのラッチがセットされる）

前提条件と不変条件

  • WalSummarizerCtl が初期化されていること。初期化されていない場合、関数は何もせず終了する。
  • WALSummarizerLock を介して共有メモリ上のプロセス番号を取得する。

アルゴリズムまたは振る舞い

1. WalSummarizerCtl が NULL でないことを確認する。 2. WALSummarizerLock を共有モードで取得し、共有メモリから summarizer_pgprocno を読み取る。 3. ロックを解放する。 4. 取得した pgprocno が INVALID_PROC_NUMBER でない場合、該当プロセスのラッチ（procLatch）を SetLatch でセットし、プロセスを起床させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c:7337
  • 呼び出し先: LWLockAcquire, LWLockRelease, SetLatch

関連データ構造

  • WalSummarizerCtl: WAL Summarizer の状態を保持する共有メモリ上の制御構造体。
  • ProcGlobal: 全プロセスの情報を保持するグローバル構造体。

失敗モードと境界条件

  • WalSummarizerCtl が NULL の場合、安全に早期リターンする。
  • summarizer_pgprocno が INVALID_PROC_NUMBER の場合、ラッチ操作はスキップされる。

保守担当者向けメモ

  • 本関数は WAL の更新が発生した際に、要約処理を即座にトリガーするために呼び出される。
  • WALSummarizerLock の取得は共有モードで行われるため、他の読み取り処理と競合しない設計となっている。
