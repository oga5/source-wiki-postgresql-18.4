forkname_chars

定義

  • 種別: function
  • 定義箇所: src/common/relpath.c:80

目的

与えられた文字列の先頭が、既知のフォーク名（main, fsm, vm, init等）で始まるかを判定し、対応するフォーク番号を特定します。

シグネチャまたは形状

int forkname_chars(const char *str, ForkNumber *fork);

入力と出力

  • **入力**:
  • str: 検査対象の文字列。
  • fork: フォーク番号を格納するためのポインタ（NULL許容）。
  • **出力**:
  • 戻り値: 一致したフォーク名の長さ（バイト数）。一致しない場合は 0。
  • *fork: 一致した場合は対応する ForkNumber が代入され、一致しない場合は InvalidForkNumber が代入されます（fork が NULL でない場合）。

前提条件と不変条件

  • forkNames 配列が ForkNumber の定義と整合している必要があります。
  • forkNum は 1 から MAX_FORKNUM の範囲で走査されます。

アルゴリズムまたは振る舞い

1. 1 から MAX_FORKNUM までのフォーク番号を順に走査します。 2. 各フォーク番号に対応する文字列（forkNames[forkNum]）の長さを取得します。 3. strncmp を使用して、入力文字列 str の先頭がそのフォーク名と一致するかを比較します。 4. 一致した場合、fork ポインタが有効であればその番号を書き込み、一致した長さを返して終了します。 5. 全てのフォーク名を走査しても一致しない場合、fork ポインタが有効であれば InvalidForkNumber を書き込み、0 を返します。

関連データ構造

  • ForkNumber: フォーク番号を定義する列挙型。
  • forkNames: フォーク番号と文字列の対応を保持する配列。

失敗モードと境界条件

  • str が空文字列またはフォーク名を含まない場合、戻り値は 0 となり、*fork には InvalidForkNumber が設定されます。
  • fork 引数に NULL を渡した場合、フォーク番号の書き込みはスキップされます。

保守担当者向けメモ

  • 新しいフォークを追加する場合は、ForkNumber 列挙型および forkNames 配列の更新が必要です。
  • 本関数はファイル名のプレフィックス解析に依存しているため、フォーク名の命名規則を変更する際は影響範囲に注意してください。
