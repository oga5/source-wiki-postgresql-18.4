SplitTextOutputData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/varlena.c:111

目的

文字列分割処理（split_text 等）において、分割された結果を蓄積・管理するための状態保持構造体です。

シグネチャまたは形状

typedef struct
{
	ArrayBuildState *astate;
	Tuplestorestate *tupstore;
	TupleDesc	tupdesc;
} SplitTextOutputData;

入力と出力

本構造体自体はデータ保持用であり、直接的な入出力関数はありません。split_text_accum_result 関数を介して、分割された各要素を astate または tupstore に蓄積するために使用されます。

前提条件と不変条件

  • astate または tupstore のいずれかが、分割結果の出力先として適切に初期化されている必要があります。
  • tupdesc は、tupstore を使用する場合のタプル構造定義として利用されます。

アルゴリズムまたは振る舞い

  • 分割処理の実行中、各要素を split_text_accum_result に渡すことで、動的に結果を構築します。
  • astate が設定されている場合は配列形式での構築を、tupstore が設定されている場合はタプルセットとしての構築をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/varlena.c 内の文字列分割関連関数（split_text 等）でインスタンスが生成・使用されます。
  • **呼び出し先**:
  • split_text 関数が本構造体を引数として受け取り、処理を制御します。
  • split_text_accum_result 関数が本構造体を引数として受け取り、結果の蓄積を行います。

関連データ構造

  • ArrayBuildState: 配列構築用状態。
  • Tuplestorestate: タプルセット構築用状態。
  • TupleDesc: タプルの記述子。

失敗モードと境界条件

  • メモリ不足時、astate や tupstore の構築・拡張に失敗する可能性があります。
  • tupstore 使用時に tupdesc が適切に設定されていない場合、実行時エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は split_text 関連の内部処理専用です。変更を加える際は、split_text および split_text_accum_result への影響を確認してください。
  • astate と tupstore の使い分けは、呼び出し側のコンテキスト（関数が何を返すか）に依存します。
