compare_files

定義

  • 種別: function
  • 定義箇所: contrib/basic_archive/basic_archive.c:224

目的

2つのファイルの内容が同一であるかをバイト単位で比較し、重複排除の判定を行う。

シグネチャまたは形状

static bool compare_files(const char *file1, const char *file2)

入力と出力

  • **入力**: 比較対象となる2つのファイルパス（file1, file2）。
  • **出力**: 両者の内容が完全に一致する場合は true、それ以外の場合は false を返す。

前提条件と不変条件

  • 指定されたファイルは読み取り可能であること。
  • ファイルのオープンおよびクローズには OpenTransientFile および CloseTransientFile を使用する。

アルゴリズムまたは振る舞い

1. OpenTransientFile を用いて両ファイルを読み取り専用で開く。 2. 4096バイトのバッファを2つ用意し、ループ内で各ファイルからデータを読み込む。 3. 読み込んだバイト数（buf1_len, buf2_len）が一致し、かつ memcmp による内容比較で差分がないことを確認する。 4. 読み込み中にエラーが発生した場合は ereport(ERROR, ...) で処理を中断する。 5. 比較終了後、CloseTransientFile で両ファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/basic_archive/basic_archive.c 内の basic_archive_file 関数（166行目付近）から呼び出される。
  • **呼び出し先**: OpenTransientFile, read, memcmp, CloseTransientFile, ereport 等のPostgreSQL内部関数および標準ライブラリ関数。

関連データ構造

  • 特になし（固定サイズのスタックバッファ buf1, buf2 を使用）。

失敗モードと境界条件

  • **ファイルオープン失敗**: 指定されたパスが存在しない、または権限がない場合、ereport によりエラー終了する。
  • **読み込みエラー**: 読み込み中にシステムエラーが発生した場合、ereport によりエラー終了する。
  • **ファイルサイズ不一致**: 読み込みバイト数が異なる場合、直ちに false を返し比較を終了する。

保守担当者向けメモ

  • 本関数は basic_archive_file において、アーカイブ先ディレクトリに同名ファイルが存在する場合の重複チェックに使用される。
  • 比較処理は CMP_BUF_SIZE (4096バイト) 単位で行われるため、非常に大きなファイルに対してもメモリ消費を抑えた比較が可能である。
