wait_result_to_str

定義

  • 種別: function
  • 定義箇所: src/common/wait_error.c:32

目的

wait(2) や system(3) 等から得られるプロセス終了ステータスを、人間が判読可能な説明文字列に変換します。

シグネチャまたは形状

char *wait_result_to_str(int exitstatus);

入力と出力

  • **入力**: exitstatus (int型) - wait 系システムコールが返す終了ステータス。
  • **出力**: char * - ステータスの内容を示す文字列。pstrdup を使用して動的にメモリが確保されているため、呼び出し側で解放する必要があります。

前提条件と不変条件

  • exitstatus が -1 の場合、errno に基づくエラーメッセージが生成されます。
  • WIFEXITED または WIFSIGNALED マクロがステータスの解析に使用されます。

アルゴリズムまたは振る舞い

1. exitstatus == -1: errno に基づくエラーメッセージを生成します。 2. WIFEXITED(exitstatus) が真の場合:

  • 終了コードが 126 なら「command not executable」、127 なら「command not found」を返します。
  • それ以外は「child process exited with exit code %d」という形式の文字列を生成します。

    • 3. WIFSIGNALED(exitstatus) が真の場合:

  • Windows環境では例外コードを、それ以外ではシグナル番号と pg_strsignal による説明文を生成します。

    • 4. いずれにも該当しない場合:

  • 「child process exited with unrecognized status %d」を返します。

    • 5. 最後に、生成した文字列を pstrdup で複製して返します。

関連データ構造

  • なし（標準的な wait ステータスの整数値を使用）。

失敗モードと境界条件

  • exitstatus が -1 の場合、errno が正しく設定されていないと不適切なエラーメッセージが生成される可能性があります。
  • メモリ不足時には pstrdup が失敗する可能性があります。

保守担当者向けメモ

  • 本関数が返す文字列は palloc されたメモリ領域であるため、メモリリークを防ぐために呼び出し側で必ず解放してください。
  • Windows環境とPOSIX環境でシグナル情報の取得方法が異なるため、修正時は条件付きコンパイルに注意してください。
