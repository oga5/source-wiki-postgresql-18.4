InProgressIO

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/aio/read_stream.c:83

目的

進行中の非同期 I/O 操作と、それに関連付けられたバッファの状態を追跡するために使用される構造体です。

シグネチャまたは形状

typedef struct InProgressIO
{
	int16		buffer_index;
	ReadBuffersOperation op;
} InProgressIO;

入力と出力

  • buffer_index: 関連するバッファのインデックス。
  • op: 実行中または保留中の ReadBuffersOperation 構造体。

前提条件と不変条件

  • ReadStream 構造体内で配列として確保され、管理されます。
  • ReadStream の初期化時に、最大 I/O 数に基づいてメモリが割り当てられます。

アルゴリズムまたは振る舞い

  • ReadStream がバッチ処理を行う際、複数のブロック読み込み要求を InProgressIO に格納し、StartReadBuffers を通じて I/O を開始します。
  • 読み込み完了後、対応するバッファの状態を更新するために参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: read_stream.c 内の read_stream_start_pending_read 等、ストリームの I/O 管理ロジックから参照されます。
  • 呼び出し先: 直接的な関数呼び出しは行いません。データ保持用の構造体です。

関連データ構造

  • ReadStream: 本構造体を配列として保持する親構造体。
  • ReadBuffersOperation: I/O 操作の詳細を定義する構造体。

失敗モードと境界条件

  • ReadStream の初期化時に指定された max_ios を超える I/O は管理できません。
  • メモリ割り当て時（sizeof(InProgressIO) * Max(1, max_ios)）に失敗した場合、ストリームの生成が失敗します。

保守担当者向けメモ

  • 本構造体は read_stream.c 内でのみ使用される内部的な管理用データです。
  • ReadStream のメモリレイアウトの一部として動的に確保されるため、構造体のサイズ変更は read_stream_begin_impl 等のメモリ確保ロジックに影響を与えます。
