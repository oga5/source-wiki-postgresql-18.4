ParseState

定義

  • 種別: struct
  • 定義箇所: src/include/parser/parse_node.h:23

目的

SQL 文の解析（Parse Analysis）プロセス全体を通じて、クエリのコンテキスト情報や状態を保持・管理するための中心的なデータ構造です。

シグネチャまたは形状

typedef struct ParseState ParseState;

※構造体の内部フィールド定義は本ソース抜粋には含まれていません。

入力と出力

  • 入力: make_parsestate() 関数により生成され、解析対象のクエリや親の ParseState を受け取ります。
  • 出力: 解析完了後、free_parsestate() によってリソースが解放されます。

前提条件と不変条件

  • 解析中、ソーステキスト（p_sourcetext）を保持し続ける必要があります。
  • サブクエリの解析時には、親の ParseState を参照する parentParseState がリンクされます。

アルゴリズムまたは振る舞い

  • **状態追跡**: 範囲テーブル（p_rtable）や名前空間（p_namespace）を管理し、構文木からクエリ木への変換を支援します。
  • **エラー報告**: parser_errposition() を介して、構文木のバイトオフセットをソーステキスト上の位置に変換し、エラーメッセージの文脈を特定します。
  • **拡張性**: パーサーフックを定義し、外部モジュールによる解析ロジックの介入を許可します。

呼び出し元と呼び出し先

  • **呼び出し元（例）**:
  • contrib/pg_stat_statements/pg_stat_statements.c: pgss_post_parse_analyze
  • src/backend/catalog/heap.c: cookConstraint
  • src/backend/catalog/aclchk.c: ExecAlterDefaultPrivilegesStmt
  • **呼び出し先**:
  • 本構造体は transformExpr 等の解析関数に渡され、内部で参照・更新されます。

関連データ構造

  • ParseExprKind: 式の変換コンテキストを識別する列挙型。
  • ParseNamespaceItem: 名前解決のための名前空間要素。
  • nodes/parsenodes.h: 構文木およびクエリ木のノード定義。
  • utils/queryenvironment.h / utils/relcache.h: 実行環境およびリレーションキャッシュへのアクセス。

失敗モードと境界条件

  • 解析中にソース文字列が失われると、エラー位置の特定が不可能になります。
  • ParseExprKind を用いて、特定の句（WHERE 句など）で許可されない構文（集約関数の使用など）をチェックする制約があります。

保守担当者向けメモ

  • ParseState は非常に多くのフィールドを持つため、名前解決やクエリ構成要素の追跡に使用される各フィールドの役割を理解してください。
  • src/backend/parser/parse_node.c に実装されているライフサイクル管理関数（make_parsestate, free_parsestate）が、リソース管理の起点となります。
