XmlTableInitOpaque

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xml.c:4682

目的

XMLTABLE 演算子の実行に必要な内部状態（XmlTableBuilderData）を初期化し、libxml2 のパーサコンテキストを準備する。

シグネチャまたは形状

static void XmlTableInitOpaque(TableFuncScanState *state, int natts)

入力と出力

  • state: 実行状態を保持する TableFuncScanState 構造体へのポインタ。初期化された opaque メンバが格納される。
  • natts: 抽出対象となる列数。
  • 戻り値: なし。

前提条件と不変条件

  • USE_LIBXML が定義されている環境で動作する。
  • state->opaque は本関数によって XmlTableBuilderData 型のメモリ領域が割り当てられ、初期化される。
  • libxml2 のエラーハンドラが pg_xml_init によりセッション単位で設定される。

アルゴリズムまたは振る舞い

1. XmlTableBuilderData 構造体をメモリ確保し、natts に基づいて XPath コンパイル用配列を初期化する。 2. pg_xml_init を呼び出し、PostgreSQL 用のエラーハンドリングコンテキストを構築する。 3. PG_TRY ブロック内で libxml2 の xmlInitParser および xmlNewParserCtxt を実行し、パーサコンテキストを生成する。 4. 生成失敗時やエラー発生時は xml_ereport を通じてエラーを報告する。 5. PG_CATCH ブロックにて、エラー発生時にパーサコンテキストの解放とエラーコンテキストの破棄を行い、例外を再送出する。 6. 成功時、構築したコンテキストを state->opaque に格納する。 7. USE_LIBXML が未定義の場合は NO_XML_SUPPORT() を呼び出しエラーとする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/xml.c 内の XmlTableRoutine 構造体（.InitOpaque フィールド）を通じて呼び出される。
  • 呼び出し先: palloc0, pg_xml_init, xmlInitParser, xmlNewParserCtxt, xml_ereport, xmlFreeParserCtxt, pg_xml_done, PG_RE_THROW。

関連データ構造

  • TableFuncScanState: 実行ノードの状態を保持する構造体。
  • XmlTableBuilderData: XMLTABLE 実行用のプライベートデータ構造体。
  • PgXmlErrorContext: libxml2 のエラーを PostgreSQL の ereport に変換するためのコンテキスト。

失敗モードと境界条件

  • libxml2 のパーサコンテキスト生成に失敗した場合、ERRCODE_OUT_OF_MEMORY でエラーを報告する。
  • libxml2 内部でエラーが発生した場合、xmlerrcxt->err_occurred を介して捕捉され、xml_ereport により処理される。
  • USE_LIBXML が無効なビルド環境では、実行時にエラーとなる。

保守担当者向けメモ

  • 本関数は libxml2 のグローバルな状態管理と PostgreSQL のメモリ管理を橋渡しする重要な初期化ポイントである。
  • PG_TRY/PG_CATCH ブロック内でのリソース解放は、メモリリークを防ぐために必須である。
