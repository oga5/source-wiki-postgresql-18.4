hbaPort

定義

  • 種別: typedef struct
  • 定義箇所: src/include/libpq/hba.h:173

目的

src/include/libpq/libpq-be.h で定義されている Port 構造体の前方宣言です。循環参照を回避しつつ、HBA（ホストベース認証）関連の関数が接続情報を参照できるようにするために使用されます。

シグネチャまたは形状

typedef struct Port hbaPort;

入力と出力

  • 本シンボル自体は型定義であるため、直接的な入力・出力はありません。
  • hbaPort 型のポインタは、認証方式の決定や検証を行う関数群の引数として渡されます。

前提条件と不変条件

  • hbaPort は Port 構造体の別名であるため、実体となる Port 構造体がメモリ上で有効である必要があります。
  • 認証処理の文脈において、この構造体には接続元の IP アドレス、ユーザー名、データベース名などの接続情報が格納されていることが期待されます。

アルゴリズムまたは振る舞い

  • 本シンボルは型定義であり、それ自体にアルゴリズムは含まれません。
  • 認証ロジック（hba_getauthmethod 等）において、この型を通じてクライアント接続情報が参照され、pg_hba.conf のルールとの照合に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/libpq/auth.c: ident_inet(), auth_peer()
  • src/backend/libpq/hba.c: check_hostname(), check_hba(), hba_getauthmethod()
  • src/include/libpq/hba.h: hba_getauthmethod() の宣言
  • **呼び出し先**:
  • 本シンボルは型定義であるため、関数としての呼び出し先はありません。

関連データ構造

  • Port: hbaPort が指し示す実体となる構造体（src/include/libpq/libpq-be.h で定義）。
  • HbaLine: hbaPort の情報と照合される認証ルールを保持する構造体。

失敗モードと境界条件

  • 型定義であるため、本シンボル自体に失敗モードはありません。
  • 参照先の Port 構造体が初期化されていない、あるいは不正なメモリ領域を指している場合、これを利用する関数群（hba_getauthmethod 等）でセグメンテーション違反や未定義動作が発生する可能性があります。

保守担当者向けメモ

  • hbaPort は libpq-be.h の Port 構造体への前方宣言です。hba.h と libpq-be.h の間での循環参照を避けるための設計上の工夫です。
  • 構造体の定義を変更する場合は、src/include/libpq/libpq-be.h の Port 構造体を確認してください。
