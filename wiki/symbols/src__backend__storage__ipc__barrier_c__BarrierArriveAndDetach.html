BarrierArriveAndDetach

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/barrier.c:202

目的

バリアへの到達を通知し、同時に当該プロセスをバリアから離脱させる。

シグネチャまたは形状

bool BarrierArriveAndDetach(Barrier *barrier);

入力と出力

  • **入力**: barrier (操作対象の Barrier 構造体へのポインタ)
  • **出力**: bool 型。呼び出しプロセスがバリアの最後の参加者であった場合に true を返し、それ以外の場合は false を返す。

前提条件と不変条件

  • barrier は BarrierInit によって初期化済みであること。
  • 呼び出しプロセスは、当該バリアに現在参加している必要がある。

アルゴリズムまたは振る舞い

1. 内部関数 BarrierDetachImpl(barrier, true) を呼び出す。 2. BarrierDetachImpl 内で、バリアの参加者数から自身のカウントを減算する。 3. 自身が最後の参加者である場合、バリアのフェーズを進行させ、待機中のプロセスを起床させる。 4. 自身が最後の参加者であるかどうかの判定結果（bool）を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHash.c (複数箇所)
  • **呼び出し先**:
  • BarrierDetachImpl (直接の内部実装)

関連データ構造

  • Barrier: 同期状態（参加者数、フェーズ、スピンロック、条件変数）を保持する構造体。

失敗モードと境界条件

  • 参加していないバリアに対して呼び出した場合の動作は未定義（ソースコード上での明示的なエラーハンドリングは記述されていない）。
  • 最後の参加者が離脱する際、バリアのフェーズ更新と待機プロセスの起床がアトミックに行われる。

保守担当者向けメモ

  • 本関数は BarrierDetachImpl のラッパーであり、引数 arrive に true を渡すことで「到達」と「離脱」を同時に処理する。
  • 呼び出し側で戻り値（true）を確認することで、そのプロセスがバリアの最後の離脱者としてシリアルな後処理を担当すべきかを判断できる。
