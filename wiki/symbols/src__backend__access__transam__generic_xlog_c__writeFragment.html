writeFragment

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/generic_xlog.c:89

目的

汎用WALレコードの構築過程において、ページ内の変更箇所（フラグメント）を GenericXLogPageData 構造体のデルタバッファへ書き込み、状態を更新します。

シグネチャまたは形状

static void writeFragment(GenericXLogPageData *pageData, OffsetNumber offset, OffsetNumber length, const char *data)

入力と出力

  • **入力**:
  • pageData: 変更対象のページデータ状態を保持する構造体。
  • offset: ページ内での変更開始位置。
  • length: 変更データのバイト長。
  • data: 書き込む変更データのソースポインタ。
  • **出力**: なし（pageData 構造体内の delta バッファおよび deltaLen を直接更新）。

前提条件と不変条件

  • pageData->delta バッファに、offset、length、および data を格納するための十分な空き容量が存在すること。この条件は Assert により検証されます。

アルゴリズムまたは振る舞い

1. pageData->delta 内の現在の書き込み位置（deltaLen）を特定します。 2. offset（オフセット値）をバッファにコピーし、ポインタを進めます。 3. length（データ長）をバッファにコピーし、ポインタを進めます。 4. data（実際の変更内容）をバッファにコピーし、ポインタを進めます。 5. 最終的な書き込み位置を pageData->deltaLen に反映させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/generic_xlog.c 内の computeDelta 関数（行196および217）から呼び出されます。
  • **呼び出し先**: 標準ライブラリの memcpy を使用してメモリコピーを行います。

関連データ構造

  • GenericXLogPageData: 汎用WALレコード生成時にページごとの差分情報を保持する構造体。

失敗モードと境界条件

  • **バッファオーバーフロー**: pageData->delta の容量を超えるデータを書き込もうとした場合、Assert によりプログラムが停止します（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は computeDelta によって計算された差分をシリアライズする役割を担います。
  • デルタバッファのサイズ制限は GenericXLogPageData の定義に依存するため、変更時は構造体のサイズ定義を確認してください。
