FreePageBtreeHeader

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/freepage.c:77

目的

メモリ断片化を抑制するために使用される B-tree ノードの共通ヘッダを定義します。この構造体は、B-tree ノードがリーフ（葉）であるか内部ノードであるかを識別し、ツリー構造を維持するために使用されます。

シグネチャまたは形状

typedef struct FreePageBtreeHeader
{
	int			magic;			/* FREE_PAGE_LEAF_MAGIC or FREE_PAGE_INTERNAL_MAGIC */
	Size		nused;			/* 使用中のアイテム数 */
	RelptrFreePageBtree parent; /* 親ノードへの相対ポインタ */
} FreePageBtreeHeader;

入力と出力

本構造体は関数ではなくデータ型であるため、直接的な入力・出力はありません。メモリ管理サブシステム内において、B-tree ノードとしてメモリ上に配置されます。

前提条件と不変条件

  • magic フィールドには、FREE_PAGE_LEAF_MAGIC または FREE_PAGE_INTERNAL_MAGIC のいずれかが格納されている必要があります。
  • parent は RelptrFreePageBtree 型であり、共有メモリ環境下で有効な相対ポインタとして機能します。

アルゴリズムまたは振る舞い

  • B-tree ノードの先頭に配置され、ノードのメタデータを保持します。
  • nused は、ノード内に格納されているキーの数を追跡します。
  • parent を通じて、ツリーの再帰的な調整や探索時の上位ノードへの遡りを可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/mmgr/freepage.c 内の B-tree 操作関数群（ノードの初期化や検索時）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • 注: ツールによる解析では、src/backend/utils/mmgr/freepage.c 内の複数の箇所で本構造体のサイズ計算や変数宣言に使用されていることが確認されています。

関連データ構造

  • FreePageBtree: B-tree ノード全体を表現する構造体。
  • FreePageBtreeInternalKey: 内部ノード用のキー構造。
  • FreePageBtreeLeafKey: リーフノード用のキー構造。
  • RelptrFreePageBtree: ノード間のリンクに使用される相対ポインタ型。

失敗モードと境界条件

  • メモリ管理の再帰的制約により、管理対象領域が枯渇した場合、新しい B-tree ノード（本ヘッダを含む）の確保に失敗する可能性があります。
  • magic 値が不正な場合、B-tree の整合性が損なわれ、メモリ管理操作が異常終了する可能性があります。

保守担当者向けメモ

  • 本構造体は FPM_PAGE_SIZE からヘッダサイズを差し引いた領域をキーの格納に使用するため、構造体のサイズ変更はノードあたりの収容可能キー数に直接影響します。
  • 共有メモリ環境での利用を前提としているため、ポインタの保持には必ず Relptr を使用してください。
