InjectionPointRun

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/injection_point.c:544

目的

指定された名前のインジェクションポイントに関連付けられたコールバック関数を実行します。テストやデバッグ目的で、実行コード中に動的なフックを挿入するために使用されます。

シグネチャまたは形状

void InjectionPointRun(const char *name, void *arg)

入力と出力

  • name (const char *): 実行対象のインジェクションポイント名。
  • arg (void *): コールバック関数に渡される任意の引数。
  • 戻り値: なし。

前提条件と不変条件

  • ビルド時に --enable-injection-points が有効である必要があります。無効なビルド環境で呼び出された場合、エラーが発生します。
  • 内部で InjectionPointCacheRefresh を呼び出し、キャッシュの整合性を確保した上で実行されます。

アルゴリズムまたは振る舞い

1. USE_INJECTION_POINTS が定義されている場合:

  • InjectionPointCacheRefresh(name) を呼び出し、対象のキャッシュエントリを取得します。
  • キャッシュエントリが存在する場合、その callback フィールドに格納された関数を、name、private_data、および呼び出し元から渡された arg を引数として実行します。

    • 2. USE_INJECTION_POINTS が定義されていない場合:

  • elog(ERROR) を発行し、機能がサポートされていない旨を通知します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/utils/injection_point.h で定義された INJECTION_POINT(name, arg) マクロ経由で、PostgreSQL の各コードパスから呼び出されます。
  • 呼び出し先: InjectionPointCacheRefresh を呼び出します。また、キャッシュエントリ内の callback 関数ポインタを介して外部ライブラリの関数を実行します。

関連データ構造

  • InjectionPointCacheEntry: キャッシュされたインジェクションポイントの定義（コールバック関数やプライベートデータを含む）。

失敗モードと境界条件

  • --enable-injection-points なしでビルドされた環境で呼び出すと、実行時エラーとなります。
  • 指定された name に対応するインジェクションポイントが登録されていない場合、InjectionPointCacheRefresh が NULL を返し、コールバックは実行されません。

保守担当者向けメモ

  • 本関数は本番環境での使用を想定していません。
  • 動的ライブラリのロードを伴うため、セキュリティおよび安定性の観点から慎重に扱う必要があります。
  • 共有メモリ上の generation カウンタを用いたロックフリーな読み取り設計が InjectionPointCacheRefresh 側で考慮されています。
