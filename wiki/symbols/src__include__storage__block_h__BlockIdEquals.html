BlockIdEquals

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/block.h:91

目的

2つの BlockIdData 構造体が同一のブロックを指しているかどうかを判定します。

シグネチャまたは形状

static inline bool BlockIdEquals(const BlockIdData *blockId1, const BlockIdData *blockId2)

入力と出力

  • **入力**: 比較対象となる2つの BlockIdData 構造体へのポインタ (blockId1, blockId2)。
  • **出力**: 両者の bi_hi および bi_lo メンバが完全に一致する場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • 入力されるポインタは有効な BlockIdData 構造体を指している必要があります。
  • BlockIdData は BlockNumber を2つの16ビット値に分割して保持する構造体であり、本関数はこの内部表現の完全一致を検証します。

アルゴリズムまたは振る舞い

1. blockId1 の上位16ビット (bi_hi) と blockId2 の上位16ビットを比較します。 2. blockId1 の下位16ビット (bi_lo) と blockId2 の下位16ビットを比較します。 3. 両方の比較結果が真である場合にのみ true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース抜粋内には具体的な呼び出し元の記述はありません。
  • **呼び出し先**: 外部関数や他のシンボルへの依存はありません。

関連データ構造

  • BlockIdData: BlockNumber を保持するためのオンディスク用構造体。
  • BlockNumber: 32ビットの論理ブロック番号。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の動作は未定義です。
  • 構造体内のパディングや初期化されていないメモリ領域については考慮されません（メンバ単位の比較を行うため）。

保守担当者向けメモ

  • 本関数はインライン化されており、パフォーマンスを重視する箇所での使用を想定しています。
  • BlockId の構造変更（bi_hi, bi_lo の型や名称変更）が発生した場合は、本関数の実装も追従する必要があります。
