detoast_external_attr

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/detoast.c:44

目的

TOASTされたデータ（外部格納、間接参照、展開済みオブジェクト）を、メモリ上の標準的な varlena 形式へ復元（フラット化）します。

シグネチャまたは形状

struct varlena *detoast_external_attr(struct varlena *attr);

入力と出力

  • **入力**: struct varlena *attr - 復元対象の可変長データ。
  • **出力**: struct varlena * - 復元されたデータへのポインタ。入力が既にプレーンな値であればそのまま返されます。

前提条件と不変条件

  • 戻り値のメモリは palloc によって確保されるため、呼び出し元が pfree する責任を負います（入力がプレーンな値である場合を除く）。
  • 間接参照（VARATT_IS_EXTERNAL_INDIRECT）の場合、ネストされた間接参照は許可されません。

アルゴリズムまたは振る舞い

入力データの形式に応じて以下の処理を行います。 1. **外部格納（On-disk）**: toast_fetch_datum を呼び出し、TOASTテーブルからデータを取得します。 2. **間接参照**: VARATT_EXTERNAL_GET_POINTER で実体を取得し、必要に応じて再帰的に detoast_external_attr を呼び出します。その後、呼び出し元のメモリコンテキストへコピーを作成します。 3. **展開済みオブジェクト**: EOH_get_flat_size でサイズを算出し、EOH_flatten_into を用いてメモリ上にフラットな形式で展開します。 4. **プレーン値**: 何もせず入力ポインタをそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c, src/backend/access/brin/brin_tuple.c, src/backend/access/common/indextuple.c, src/backend/access/heap/heaptoast.c 等から参照されています。
  • **呼び出し先**: toast_fetch_datum, EOH_get_flat_size, EOH_flatten_into, および再帰的に detoast_external_attr を呼び出します。

関連データ構造

  • struct varlena: PostgreSQLの可変長データ構造。
  • struct varatt_indirect: 間接参照用の構造体。
  • ExpandedObjectHeader: 展開済みオブジェクトのヘッダー。

失敗モードと境界条件

  • 入力が間接参照の場合、ネストされた間接参照が検出されると Assert が失敗します。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は呼び出し元が pfree することを前提にメモリを割り当てるため、メモリリークに注意してください。
  • 圧縮データの展開については、本関数ではなく toast_decompress_datum 等の役割となります。
