mbuf_grab

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/mbuf.c:148

目的

MBuf 構造体から指定された長さのデータを読み取り、バッファ内の読み込み位置を更新する。

シグネチャまたは形状

int mbuf_grab(MBuf *mbuf, int len, uint8 **data_p);

入力と出力

  • **入力**:
  • mbuf: 対象の MBuf 構造体へのポインタ。
  • len: 読み取りを要求するバイト数。
  • data_p: 読み取ったデータ領域の開始アドレスを格納するためのポインタ。
  • **出力**:
  • 戻り値: 実際に読み取られたバイト数（int）。

前提条件と不変条件

  • mbuf は有効な MBuf 構造体を指している必要がある。
  • 呼び出し後、mbuf->no_write が true に設定され、バッファへの書き込みが禁止される。

アルゴリズムまたは振る舞い

1. 要求された len がバッファ内の利用可能なデータ量（mbuf_avail(mbuf)）を超える場合、len を利用可能な最大値に切り詰める。 2. mbuf->no_write を true に設定する。 3. data_p に現在の読み込み位置（mbuf->read_pos）を代入する。 4. mbuf->read_pos を読み取ったバイト数分だけ進める。 5. 実際に読み取ったバイト数 len を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/mbuf.c (内部参照)
  • contrib/pgcrypto/pgp-encrypt.c:697
  • **呼び出し先**:
  • mbuf_avail (マクロまたは関数)

関連データ構造

  • MBuf: データを保持する可変長バッファ構造体。

失敗モードと境界条件

  • 要求された len が 0 の場合、0 を返し、読み込み位置は変更されない。
  • バッファが空の場合、0 を返し、data_p には現在の read_pos が代入される。

保守担当者向けメモ

  • 本関数はバッファの読み込み位置を不可逆的に進めるため、ストリーム処理のパイプラインにおいてデータの消費を行う際に使用される。
  • no_write フラグが設定されるため、この関数を呼び出した後は、同一の MBuf に対して mbuf_append 等の書き込み操作を行わないよう注意が必要である。
