GetACPEncoding

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:2483

目的

Windows環境におけるANSIコードページ（ACP）に対応するPostgreSQL内部エンコーディングIDを取得します。

シグネチャまたは形状

static int GetACPEncoding(void);

入力と出力

  • 入力: なし
  • 出力: int 型のPostgreSQLエンコーディングIDを返します。

前提条件と不変条件

  • 初回呼び出し時に pg_codepage_to_encoding(GetACP()) を実行し、その結果を静的変数にキャッシュします。
  • 2回目以降の呼び出しでは、キャッシュされた値を返します。

アルゴリズムまたは振る舞い

1. 静的変数 encoding が初期値 -2 であるか確認します。 2. -2 の場合、GetACP() を呼び出して現在のシステムACPを取得し、それを pg_codepage_to_encoding() でPostgreSQLのエンコーディングIDに変換して encoding に代入します。 3. キャッシュされた encoding の値を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/error/elog.c:2555 (GetMessageEncoding() 内での比較に使用)
  • 呼び出し先:
  • GetACP() (システム依存のACP取得)
  • pg_codepage_to_encoding() (コードページからPostgreSQLエンコーディングIDへの変換)

関連データ構造

  • なし（内部的な静的変数 encoding を使用）

失敗モードと境界条件

  • GetACP() および pg_codepage_to_encoding() の戻り値に依存します。
  • 呼び出し元以外の具体的な失敗時の挙動については、提供されたソースからは不明です。

保守担当者向けメモ

  • 本関数はWindows環境でのエンコーディング判定を最適化するためのキャッシュ機構です。
  • GetMessageEncoding() との比較を通じて、メッセージエンコーディングがシステムACPと一致するかを判定する際に利用されます。
