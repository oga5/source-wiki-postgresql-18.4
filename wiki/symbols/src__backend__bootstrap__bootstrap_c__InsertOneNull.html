InsertOneNull

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/bootstrap/bootstrap.c:694

目的

ブートストラップモードにおいて、現在構築中のリレーション（boot_reldesc）の指定された列に NULL 値を挿入するための準備を行う。

シグネチャまたは形状

void InsertOneNull(int i)

入力と出力

  • **入力**:
  • i (int): NULL を設定する列のインデックス（0始まり）。
  • **出力**: なし（副作用としてグローバルな状態を更新）。

前提条件と不変条件

  • i は 0 以上 MAXATTR 未満であること（Assert により検証）。
  • boot_reldesc が現在開かれているリレーションを指していること。

アルゴリズムまたは振る舞い

1. 指定された列インデックス i が NOT NULL 制約を持つかを確認する。 2. 制約違反がある場合は ERROR を発生させ、処理を中断する。 3. 制約違反がない場合、グローバル配列 values[i] に NULL を設定し、Nulls[i] を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、BKIコマンドの解析プロセス（boot_yyparse 等）から呼び出されることが示唆される。
  • **呼び出し先**:
  • elog（エラー報告およびデバッグログ出力）
  • TupleDescAttr（属性情報の取得）
  • RelationGetRelationName（リレーション名の取得）
  • PointerGetDatum（NULL の変換）

関連データ構造

  • boot_reldesc: 現在操作中のリレーション記述子。
  • values[]: 挿入対象タプルの値を保持するグローバル配列。
  • Nulls[]: 挿入対象タプルの各列が NULL かどうかを示すグローバル配列。

失敗モードと境界条件

  • **制約違反**: attnotnull が設定されている列に対して本関数を呼び出した場合、ERROR が発生する。
  • **範囲外アクセス**: i が MAXATTR を超える場合、Assert によりアボートする。

保守担当者向けメモ

  • 本関数は values および Nulls というグローバル変数に依存しているため、呼び出し順序と現在のリレーション状態に注意が必要である。
  • DEBUG4 レベルのログが出力されるため、ブートストラップ時のデータ挿入過程を追跡する際に利用可能である。
