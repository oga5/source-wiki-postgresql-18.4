COMPAT_MODE

定義

  • 種別: enum
  • 定義箇所: src/interfaces/ecpg/preproc/preproc_extern.h:140

目的

ECPG（Embedded SQL for C）プリプロセッサおよびライブラリにおいて、ターゲットとするデータベース製品の互換性モードを識別・制御するために使用されます。

シグネチャまたは形状

enum COMPAT_MODE
{
    ECPG_COMPAT_PGSQL = 0,
    ECPG_COMPAT_INFORMIX,
    ECPG_COMPAT_INFORMIX_SE,
    ECPG_COMPAT_ORACLE
};

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: プリプロセッサおよびライブラリ内の各処理において、特定のデータベース製品に合わせた挙動（SQL構文解析やデータ変換など）を選択するためのフラグとして機能します。

前提条件と不変条件

  • 0から始まる整数値として割り当てられます。
  • プリプロセッサおよび ecpglib の各関数において、互換性モードを引数として受け取り、条件分岐の基準として利用されます。

アルゴリズムまたは振る舞い

  • ECPG_COMPAT_PGSQL: PostgreSQL 標準の動作。
  • ECPG_COMPAT_INFORMIX / ECPG_COMPAT_INFORMIX_SE: Informix 互換の動作。
  • ECPG_COMPAT_ORACLE: Oracle 互換の動作。
  • 各関数内でこの列挙値に基づき、SQLの解釈やCコード生成時の型変換ルールが切り替えられます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/ecpglib_extern.h
  • 呼び出し先: 該当なし（本シンボルは列挙型定義であり、関数ではないため）。

関連データ構造

  • enum ARRAY_TYPE: data.c 内の関数で COMPAT_MODE と併用されることが多い型。
  • struct connection: ecpglib_extern.h 内で COMPAT_MODE を保持する構造体。

失敗モードと境界条件

  • 未定義の列挙値を渡した場合の挙動は、各関数の実装に依存します。
  • 互換性モードの不整合は、プリプロセッサによるコード生成時または実行時のライブラリ処理において、予期しないSQL構文エラーやデータ変換エラーを引き起こす可能性があります。

保守担当者向けメモ

  • 新しいデータベース製品のサポートを追加する場合、本列挙型に新しいメンバを追加し、各処理系（parser.c, output.c, ecpglib 等）の条件分岐を更新する必要があります。
  • INFORMIX_MODE や ORACLE_MODE といったマクロがソースコード内で頻繁に使用されているため、これらと本列挙型の整合性に注意してください。
