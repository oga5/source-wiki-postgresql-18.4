pgstat_report_subscription_conflict

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_subscription.c:45

目的

論理レプリケーションのサブスクリプションにおいて発生した競合（Conflict）の統計情報をインクリメントする。

シグネチャまたは形状

void pgstat_report_subscription_conflict(Oid subid, ConflictType type)

入力と出力

  • subid (Oid): 統計を更新する対象のサブスクリプションID。
  • type (ConflictType): 発生した競合の種類。
  • 戻り値: なし。

前提条件と不変条件

  • 呼び出し元は、有効なサブスクリプションIDを指定する必要がある。
  • pgstat_prep_pending_entry を介して、バックエンドローカルな統計エントリが準備可能であること。

アルゴリズムまたは振る舞い

1. pgstat_prep_pending_entry を呼び出し、指定された subid に対応するサブスクリプション統計エントリの参照を取得する。 2. 取得したエントリ内の conflict_count 配列に対し、引数 type で指定されたインデックスの値をインクリメントする。 3. この更新はバックエンドローカルな保留状態として保持され、後続のフラッシュ処理によって共有メモリに反映される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/conflict.c:122
  • 呼び出し先: pgstat_prep_pending_entry（統計サブシステム内部関数）

関連データ構造

  • PgStat_BackendSubEntry: サブスクリプション統計を保持するバックエンドローカルな構造体。
  • PgStat_EntryRef: 統計エントリへの参照を管理する構造体。
  • ConflictType: 競合の種類を定義する列挙型。

失敗モードと境界条件

  • subid が無効な場合、pgstat_prep_pending_entry の挙動に依存する。
  • type が ConflictType の範囲外である場合、配列外アクセスが発生する可能性がある（呼び出し元での保証が必要）。

保守担当者向けメモ

  • 本関数は統計の「定義」と「操作」を分離する設計に従っている。
  • 統計値の加算は pgstat_subscription_flush_cb が実行されるまで共有メモリには反映されない。
  • ConflictType に新しい種類を追加した場合は、PgStat_BackendSubEntry 構造体の conflict_count 配列サイズが十分であることを確認すること。
