slock_t

定義

  • 種別: typedef
  • 定義箇所: src/include/storage/s_lock.h:129

目的

PostgreSQL におけるスピンロックの状態を保持するための型です。ハードウェア固有の原子操作（Test-and-Set）を抽象化し、プラットフォーム間で一貫したロック状態の表現を提供します。

シグネチャまたは形状

typedef unsigned char slock_t;

入力と出力

  • 本型はデータ構造のメンバとして定義され、S_LOCK マクロや TAS（Test-and-Set）関数への引数として渡されます。
  • TAS 操作の結果として、ロックの取得成功または失敗を示す状態が内部的に更新されます。

前提条件と不変条件

  • スピンロックとして機能するためには、メモリ上の配置において適切なアライメントが維持されている必要があります。
  • S_UNLOCK を実行する際は、メモリバリアを伴うことが必須であり、単なる代入操作以上の整合性保証が求められます。

アルゴリズムまたは振る舞い

  • slock_t 型の変数は、TAS() 命令を通じてアトミックに操作されます。
  • ロック取得失敗時には s_lock() 関数へ制御が移り、TAS_SPIN ループと perform_spin_delay() を用いた待機戦略が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/access/transam/parallel.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/port/atomics.c
  • **呼び出し先**:
  • 直接的な関数呼び出し関係はありません（型定義のため）。ただし、この型を引数として TAS() や S_LOCK() マクロが操作を行います。

関連データ構造

  • SpinDelayStatus: スピン待機の試行回数や遅延時間を追跡するために併用されます。

失敗モードと境界条件

  • s_lock() 内で規定回数（NUM_DELAYS）を超えてもロックが取得できない場合、デッドロックやスタックと見なされ PANIC が発生します。

保守担当者向けメモ

  • 本型はプラットフォームにより定義が異なる場合があります（現在は unsigned char）。
  • 直接操作せず、必ず spin.h が提供する高レベル API を経由して利用してください。
  • 新しいアーキテクチャへの移植時には、TAS および S_UNLOCK の実装と併せて本型の定義が適切か確認が必要です。
