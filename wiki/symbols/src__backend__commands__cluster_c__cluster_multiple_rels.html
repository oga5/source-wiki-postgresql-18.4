cluster_multiple_rels

定義

  • 種別: function
  • 定義箇所: src/backend/commands/cluster.c:262

目的

複数のリレーション（テーブル）に対して、それぞれ個別のトランザクション内でクラスタリング処理を実行する。

シグネチャまたは形状

static void cluster_multiple_rels(List *rtcs, ClusterParams *params)

入力と出力

  • rtcs: クラスタリング対象のリレーション情報 (RelToCluster 構造体のリスト)。
  • params: クラスタリングの動作パラメータ (ClusterParams)。
  • 戻り値: なし。

前提条件と不変条件

  • 本関数呼び出し前にアクティブなトランザクションが存在する場合、それをコミットして終了させる必要がある（本関数内で CommitTransactionCommand を呼び出すため）。
  • 各リレーションの処理は独立したトランザクションで行われるため、リレーション間の整合性は個別のコミットに依存する。

アルゴリズムまたは振る舞い

1. PopActiveSnapshot および CommitTransactionCommand を実行し、既存のトランザクションを終了する。 2. rtcs リストを反復処理する。 3. 各リレーションに対して以下の手順を繰り返す：

  • StartTransactionCommand で新しいトランザクションを開始する。
  • PushActiveSnapshot でトランザクションスナップショットを設定する。
  • table_open を使用し、AccessExclusiveLock を取得して対象テーブルを開く。
  • cluster_rel を呼び出し、対象テーブルのクラスタリングを実行する。
  • PopActiveSnapshot でスナップショットを解除し、CommitTransactionCommand でトランザクションをコミットする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/cluster.c 内の cluster 関数（rtcs リストを生成後に呼び出し）。
  • 呼び出し先: table_open, cluster_rel, StartTransactionCommand, CommitTransactionCommand, PushActiveSnapshot, PopActiveSnapshot。

関連データ構造

  • RelToCluster: クラスタリング対象のテーブルOIDとインデックスOIDを保持する構造体。
  • ClusterParams: クラスタリングのオプション設定を保持する構造体。

失敗モードと境界条件

  • table_open でロック取得に失敗した場合や、cluster_rel 内でエラーが発生した場合は、現在のトランザクションがアボートされる。
  • 複数テーブル処理中にエラーが発生した場合、それ以前にコミットされたテーブルの処理はロールバックされない。

保守担当者向けメモ

  • 本関数は「1テーブル1トランザクション」という設計思想に基づいている。長時間ロックによるデッドロックを回避するための重要な設計であるため、処理順序やトランザクション境界の変更には注意が必要。
  • cluster_rel はリレーションをクローズするが、ロックは保持したままになる点に留意すること。
