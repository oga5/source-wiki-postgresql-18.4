spg_xlog_startup

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgxlog.c:975

目的

SP-GiST インデックスの WAL リプレイ処理に必要な一時的なメモリコンテキストを初期化します。

シグネチャまたは形状

void spg_xlog_startup(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 opCtx にメモリコンテキストを割り当てる）

前提条件と不変条件

  • 本関数は WAL リプレイの開始時に呼び出される必要があります。
  • opCtx はリプレイ処理全体で共有される一時的なメモリ領域として使用されます。

アルゴリズムまたは振る舞い

1. AllocSetContextCreate を呼び出し、CurrentMemoryContext を親として「SP-GiST temporary context」という名称のメモリコンテキストを作成します。 2. 作成したコンテキストをグローバル変数 opCtx に格納します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/access/rmgrlist.h において、SP-GiST のリソースマネージャ定義（RM_SPGIST_ID）の一部として登録されており、WAL リプレイ開始時にシステムから呼び出されます。
  • 呼び出し先: AllocSetContextCreate（メモリ管理用 API）。

関連データ構造

  • opCtx: リプレイ中に使用される MemoryContext 型のグローバル変数。

失敗モードと境界条件

  • メモリ不足時には AllocSetContextCreate がエラーを発生させ、リプレイ処理が中断されます。

保守担当者向けメモ

  • 本関数で作成された opCtx は、対応する spg_xlog_cleanup 関数内で解放される必要があります。リプレイ処理中にメモリリークが発生しないよう、一時的な割り当ては必ずこのコンテキストを使用してください。
