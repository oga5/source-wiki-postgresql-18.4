JsonUniqueBuilderState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/json.c:69

目的

JSONオブジェクト構築時におけるキーの重複チェックおよび、NULL値を持つキーのスキップ処理を管理するための状態保持構造体です。

シグネチャまたは形状

typedef struct JsonUniqueBuilderState
{
	JsonUniqueCheckState check;      /* 重複チェック用状態 */
	StringInfoData skipped_keys;     /* NULL値を持つスキップされたキーのバッファ */
	MemoryContext mcxt;              /* スキップされたキーを保存するためのメモリコンテキスト */
} JsonUniqueBuilderState;

入力と出力

本構造体自体はデータ保持用ですが、以下の関数を通じて操作されます。

  • 入力: json_unique_builder_init にポインタを渡し初期化します。
  • 出力: json_unique_builder_get_throwawaybuf を通じて、スキップされたキーを格納するためのバッファを取得します。

前提条件と不変条件

  • mcxt は、スキップされたキーのライフサイクルを管理するために適切に設定されている必要があります。
  • check フィールドは、JSON解析におけるキーの重複を追跡するために使用されます。

アルゴリズムまたは振る舞い

  • JSON構築プロセスにおいて、特定のキーがNULL値を持つ場合にそれを skipped_keys バッファへ退避させます。
  • 重複チェック機能と連携し、構築中のJSONオブジェクトの整合性を維持します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/json.c 内のJSON構築ロジック（例: json_unique_builder_init 等の関数内）。
  • 呼び出し先: JsonUniqueCheckState（重複チェック用）および StringInfoData（バッファ管理用）の各操作関数。

関連データ構造

  • JsonUniqueCheckState: 重複チェックの核となる構造体。
  • StringInfoData: 可変長文字列バッファ。
  • MemoryContext: メモリ割り当ての管理単位。

失敗モードと境界条件

  • メモリ不足: mcxt 上でのメモリ割り当て失敗時にエラーが発生する可能性があります。
  • バッファオーバーフロー: StringInfoData の自動拡張機能に依存しますが、極端に大きなキーセットを扱う場合はメモリ制約を受けます。

保守担当者向けメモ

  • 本構造体は json_object_agg 等の集約処理で利用されます。
  • skipped_keys のメモリ管理は mcxt に依存しているため、集約終了時のメモリ解放タイミングに注意してください。
