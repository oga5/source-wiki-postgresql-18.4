index_beginscan_internal

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/indexam.c:313

目的

インデックススキャンを開始するための共通処理を実行し、各インデックスアクセス手法（AM）固有の初期化ロジックを呼び出して IndexScanDesc を生成する。

シグネチャまたは形状

static IndexScanDesc index_beginscan_internal(
    Relation indexRelation,
    int nkeys,
    int norderbys,
    Snapshot snapshot,
    ParallelIndexScanDesc pscan,
    bool temp_snap
)

入力と出力

  • **入力**:
  • indexRelation: 対象のインデックスリレーション。
  • nkeys: スキャンキーの数。
  • norderbys: ソート順序指定の数。
  • snapshot: スキャンに使用するスナップショット。
  • pscan: 並列スキャン用の記述子（存在しない場合は NULL）。
  • temp_snap: スナップショットが一時的かどうかを示すフラグ。
  • **出力**:
  • 初期化済みの IndexScanDesc 構造体へのポインタ。

前提条件と不変条件

  • RELATION_CHECKS マクロにより、リレーションが有効であることが保証される。
  • CHECK_REL_PROCEDURE(ambeginscan) により、対象のインデックス AM が ambeginscan 関数を実装している必要がある。
  • スキャン期間中、リレーションの参照カウントがインクリメントされ保持される。

アルゴリズムまたは振る舞い

1. インデックスリレーションの整合性チェックと AM の ambeginscan 実装確認を行う。 2. AM が述語ロックをサポートしていない場合、PredicateLockRelation を呼び出しロックを取得する。 3. RelationIncrementReferenceCount を呼び出し、スキャン終了までリレーションが解放されないよう参照カウントを増やす。 4. インデックス AM 固有の ambeginscan を呼び出し、スキャン記述子を取得する。 5. 取得したスキャン記述子に並列スキャン情報 (pscan) と一時スナップショットフラグ (temp_snap) を設定し、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • index_beginscan (src/backend/access/index/indexam.c:266)
  • index_beginscan_bitmap (src/backend/access/index/indexam.c:298)
  • index_beginscan の他のバリエーション（src/backend/access/index/indexam.c:596）
  • **呼び出し先**:
  • indexRelation->rd_indam->ambeginscan (AM 固有の初期化関数)
  • PredicateLockRelation
  • RelationIncrementReferenceCount

関連データ構造

  • IndexScanDesc: スキャン状態を保持する構造体。
  • Relation: インデックスリレーションのメタデータ。
  • ParallelIndexScanDesc: 並列スキャン用の状態管理構造体。

失敗モードと境界条件

  • CHECK_REL_PROCEDURE により、AM が ambeginscan を定義していない場合はエラーが発生する。
  • リレーションが不正な状態である場合、RELATION_CHECKS によりエラーが発生する。

保守担当者向けメモ

  • 本関数は index_beginscan 系のバリエーションにおける共通ロジックをカプセル化している。
  • スキャン終了時には、本関数でインクリメントされた参照カウントを index_endscan 等で適切にデクリメントする必要がある。
