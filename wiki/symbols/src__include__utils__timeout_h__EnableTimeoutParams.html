EnableTimeoutParams

定義

  • 種別: struct
  • 定義箇所: src/include/utils/timeout.h:58

目的

複数のタイムアウトイベントを一度に設定するためのパラメータを保持する構造体です。

シグネチャまたは形状

typedef struct
{
    TimeoutId    id;            /* タイムアウト識別子 */
    TimeoutType  type;          /* TMPARAM_AFTER または TMPARAM_AT */
    int          delay_ms;      /* TMPARAM_AFTER/EVERY で使用 */
    TimestampTz  fin_time;      /* TMPARAM_AT で使用 */
} EnableTimeoutParams;

入力と出力

  • 本構造体は enable_timeouts(const EnableTimeoutParams *timeouts, int count) 関数への入力として使用されます。
  • 出力（戻り値）はありません。

前提条件と不変条件

  • delay_ms は type が TMPARAM_AFTER または TMPARAM_EVERY の場合にのみ使用されます。
  • fin_time は type が TMPARAM_AT の場合にのみ使用されます。

アルゴリズムまたは振る舞い

  • enable_timeouts 関数にこの構造体の配列を渡すことで、複数のタイムアウトをまとめてスケジュールします。
  • 内部的には all_timeouts 配列が更新され、指定された時刻または経過時間に基づいたタイマー割り込みが管理されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/storage/ipc/standby.c
  • src/backend/storage/lmgr/proc.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数呼び出しは行いません。この構造体を受け取る enable_timeouts 関数が src/backend/utils/misc/timeout.c で定義されています。

関連データ構造

  • TimeoutId: タイムアウトの種類を識別する列挙型。
  • TimeoutType: タイムアウトの指定方法（TMPARAM_AFTER, TMPARAM_AT 等）を定義する型。
  • TimestampTz: 時刻を保持するための型（datatype/timestamp.h で定義）。

失敗モードと境界条件

  • 構造体内の id に無効な値が指定された場合の挙動については、本ソース抜粋からは不明です。
  • count 引数と配列のサイズが一致しない場合、メモリ破壊や未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • src/backend/utils/misc/timeout.c の enable_timeouts 実装と密結合しています。変更時はシグナルハンドラへの影響を考慮してください。
  • SIGALRM を使用した非同期処理の一部であるため、設定値の整合性は重要です。
