ParseExprKind

定義

  • 種別: enum
  • 定義箇所: src/include/parser/parse_node.h:38

目的

SQL 文の解析（Parse Analysis）プロセスにおいて、現在解析中の式がどの SQL 句（WHERE, SELECT, JOIN 等）に属しているかを識別するために使用されます。

シグネチャまたは形状

列挙型であり、EXPR_KIND_NONE（0）から EXPR_KIND_CYCLE_MARK まで、SQL の各構文要素に対応する値を保持します。

入力と出力

  • **入力**: 解析関数（transformExpr 等）の引数として渡され、現在の解析コンテキストを伝達します。
  • **出力**: 解析ロジック内での条件分岐やエラーメッセージの生成に使用されます。

前提条件と不変条件

  • 解析状態（ParseState）の一部として、解析の進捗に応じて適切な値が設定される必要があります。
  • 新しい列挙値を追加する際は、関連する解析関数（parse_agg.c 等）で網羅的なチェックが行われることが期待されます。

アルゴリズムまたは振る舞い

  • **コンテキストの特定**: 解析関数が特定の句（例：WHERE 句）を処理する際、その式が許可されているか、あるいは特定の関数（集約関数など）が使用可能かを判定する基準となります。
  • **エラー報告**: 不適切な場所で式が使用された場合、この列挙値に基づいて「WHERE 句では集約関数は使用できません」といった具体的なエラーメッセージを生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_clause.c 内の各解析関数（transformWhereClause, transformTargetList 等）が、解析対象の句に応じて本列挙値を引数として渡します。
  • **呼び出し先**: src/backend/parser/parse_agg.c 等の解析ロジックが、本列挙値を参照して式の妥当性を検証します。

関連データ構造

  • ParseState: 解析中のクエリ状態を保持する構造体であり、この中で現在の解析コンテキストとして利用されます。

失敗モードと境界条件

  • EXPR_KIND_NONE: 式の解析コンテキスト外であることを示します。
  • EXPR_KIND_OTHER: 拡張機能用に予約されています。
  • 不適切な ParseExprKind を指定して解析関数を呼び出した場合、意図しないバリデーションエラーや、逆に本来禁止されるべき構文が許可されるリスクがあります。

保守担当者向けメモ

  • 本列挙型に新しい値を追加した場合は、parse_agg.c などの解析ロジックで switch 文の網羅性が維持されているか確認してください。コンパイラが警告を発する場合があります。
