AssertCheckItemPointers

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gininsert.c:1199

目的

GinBuffer 内に保持されているTID（タプル識別子）リストの整合性と順序を検証するデバッグ用アサーション関数です。

シグネチャまたは形状

static void AssertCheckItemPointers(GinBuffer *buffer)

入力と出力

  • **入力**: GinBuffer *buffer - 検証対象のバッファ構造体へのポインタ。
  • **出力**: なし（検証失敗時はアサーションエラーによりプロセスが停止）。

前提条件と不変条件

  • USE_ASSERT_CHECKING が定義されている環境でのみ有効です。
  • buffer->items が NULL であってはなりません。
  • buffer->nitems は 0 より大きい必要があります。
  • buffer->items 内の各TIDは有効（ItemPointerIsValid）である必要があります。

アルゴリズムまたは振る舞い

1. buffer->items が存在し、かつ nitems > 0 であることを確認します。 2. バッファ内の全TIDを走査します。 3. 各TIDが有効であることを確認します。 4. 2番目以降のTIDについて、直前のTIDよりも値が大きい（ItemPointerCompare の結果が負）ことを確認し、昇順にソートされていることを保証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gininsert.c 内の複数箇所（行: 1241, 1540, 1715, 1746, 1775, 1901, 1935, 1962）。
  • **呼び出し先**: ItemPointerIsValid および ItemPointerCompare（いずれもPostgreSQLのコアユーティリティ関数）。

関連データ構造

  • GinBuffer: GINインデックス構築中にキーごとのTIDリストを一時的に保持する構造体。

失敗モードと境界条件

  • buffer->items が NULL の場合、または nitems が 0 の場合にアサーション失敗となります。
  • TIDが有効でない場合、またはTIDリストが昇順に並んでいない場合にアサーション失敗となります。

保守担当者向けメモ

  • 本関数はデバッグビルド時のみ実行されます。本番環境のパフォーマンスには影響しません。
  • TIDの順序検証は、GINインデックスの転置リスト構築における重要な不変条件であるため、ロジック変更時は本関数によるチェックが維持されていることを確認してください。
