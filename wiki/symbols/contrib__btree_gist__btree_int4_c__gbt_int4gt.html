gbt_int4gt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int4.c:27

目的

2つの int4（4バイト整数）値を比較し、左辺が右辺より大きいかどうかを判定する。

シグネチャまたは形状

static bool gbt_int4gt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の int32 値へのポインタ。
  • b: 比較対象の int32 値へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • bool: *a > *b であれば true、そうでなければ false を返す。

前提条件と不変条件

  • a および b は、有効な int32 値を指している必要がある。
  • 本関数は btree_gist モジュール内での比較ロジックとして使用される。

アルゴリズムまたは振る舞い

1. ポインタ a が指す値を int32 としてキャストし、デリファレンスする。 2. ポインタ b が指す値を int32 としてキャストし、デリファレンスする。 3. 両者の値を比較し、a が b より大きい場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int4.c 内の tinfo 構造体定義（82行目）において、比較関数テーブルの一部として参照されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • int32: 比較対象となる PostgreSQL の 4 バイト整数型。

失敗モードと境界条件

  • ポインタが NULL の場合、デリファレンスによりセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された汎用的な GiST 比較ロジックから、tinfo 構造体を介して呼び出されることを想定している。
  • int4 型の比較ロジックを修正・拡張する場合は、同様のパターンで実装されている他の比較関数（gbt_int4lt, gbt_int4eq 等）との整合性に注意すること。
