AlterTypeOwner_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/typecmds.c:3952

目的

指定されたOIDを持つデータ型の所有者を変更します。

シグネチャまたは形状

void AlterTypeOwner_oid(Oid typeOid, Oid newOwnerId, bool hasDependEntry)

入力と出力

  • typeOid: 変更対象の型のOID。
  • newOwnerId: 新しい所有者のロールOID。
  • hasDependEntry: 依存関係エントリを更新するかどうかを示すフラグ。

前提条件と不変条件

  • pg_type システムカタログに対する RowExclusiveLock を保持する必要があります。
  • 指定された typeOid がシステムカタログ pg_type に存在すること。

アルゴリズムまたは振る舞い

1. pg_type リレーションをオープンし、TYPEOID システムキャッシュから対象の型タプルを取得します。 2. 型が複合型（TYPTYPE_COMPOSITE）である場合、ATExecChangeOwner を呼び出して関連する pg_class エントリを修正し、そこから AlterTypeOwnerInternal を介して pg_type エントリを更新します。 3. 複合型でない場合は、直接 AlterTypeOwnerInternal を呼び出して所有者を変更します。 4. hasDependEntry が真の場合、changeDependencyOnOwner を呼び出して所有者依存関係を更新します。 5. InvokeObjectPostAlterHook を呼び出し、変更後のフック処理を実行します。 6. システムキャッシュを解放し、リレーションをクローズします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_shdepend.c:1653
  • src/backend/commands/typecmds.c:3930
  • **呼び出し先**:
  • table_open, table_close
  • SearchSysCache1, ReleaseSysCache
  • ATExecChangeOwner (複合型の場合)
  • AlterTypeOwnerInternal
  • changeDependencyOnOwner (条件付き)
  • InvokeObjectPostAlterHook

関連データ構造

  • Form_pg_type: pg_type カタログの行構造体。
  • Relation: pg_type リレーションへのハンドル。
  • HeapTuple: システムキャッシュから取得した型タプル。

失敗モードと境界条件

  • SearchSysCache1 が有効なタプルを返さない場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は、型の所有権変更の最終的なカタログ更新を担当します。複合型とそれ以外でパスが分岐するため、修正時は両方のパスへの影響を考慮してください。
