init_procedure_caches

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_procedure.c:29

目的

PL/Python関数およびトリガー定義を管理するためのグローバルハッシュテーブル PLy_procedure_cache を初期化します。

シグネチャまたは形状

void init_procedure_caches(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 PLy_procedure_cache にハッシュテーブルを割り当てます）

前提条件と不変条件

  • 本関数は、PL/Pythonの初期化プロセスにおいて一度だけ呼び出されることを想定しています。
  • PLyProcedureKey および PLyProcedureEntry 構造体が定義されている必要があります。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を初期化し、キーサイズを PLyProcedureKey、エントリサイズを PLyProcedureEntry に設定します。 2. hash_create を呼び出し、"PL/Python procedures" という名前で初期サイズ 32 のハッシュテーブルを作成します。 3. HASH_ELEM および HASH_BLOBS フラグを指定し、ハッシュテーブルを生成します。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_main.c:127
  • 呼び出し先: hash_create (PostgreSQLのハッシュテーブル管理関数)

関連データ構造

  • PLyProcedureKey: キャッシュの検索キー。
  • PLyProcedureEntry: キャッシュに格納されるエントリ。
  • PLy_procedure_cache: 本関数によって初期化されるグローバルハッシュテーブル。

失敗モードと境界条件

  • hash_create がメモリ不足等で失敗した場合、PostgreSQLの標準的なエラーハンドリング（elog 等）によりプロセスが中断される可能性があります。

保守担当者向けメモ

  • 本関数で作成されるハッシュテーブルは、PL/Pythonの実行ライフサイクル全体で共有されます。
  • キャッシュのサイズ（初期値 32）を調整する必要がある場合は、本関数の hash_create 引数を変更してください。
