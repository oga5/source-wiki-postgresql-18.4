LexemeInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/dict_thesaurus.c:30

目的

シソーラス辞書における単語（レキシーム）の置換ルールおよびそのバリエーションを管理するためのデータ構造です。

シグネチャまたは形状

typedef struct LexemeInfo {
    uint32 idsubst;             /* DictThesaurus->subst 内のエントリ番号 */
    uint16 posinsubst;          /* エントリ内の位置情報 */
    uint16 tnvariant;           /* 1つのバリエーションに含まれるレキシームの総数 */
    struct LexemeInfo *nextentry;
    struct LexemeInfo *nextvariant;
} LexemeInfo;

入力と出力

  • 本構造体は palloc を介してメモリ確保されます。
  • addCompiledLexeme 等の関数を通じて、シソーラスのコンパイル時に生成・構築されます。

前提条件と不変条件

  • nextentry は同一の置換ルールに関連する次のエントリを指します。
  • nextvariant は異なるバリエーションの先頭エントリを指します。
  • tnvariant は、マッチング対象となるフレーズの長さを定義します。

アルゴリズムまたは振る舞い

  • シソーラス辞書の検索時、入力されたトークン列が定義されたルールに合致するかを判定するために使用されます。
  • findVariant 関数において、現在のトークン位置と照らし合わせながら、置換対象となるフレーズのバリエーションを探索する際にトラバースされます。

呼び出し元と呼び出し先

  • **呼び出し元**: addCompiledLexeme, matchIdSubst, findVariant 等の関数が本構造体を操作します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • DictThesaurus: シソーラス辞書のメイン構造体。本構造体は DictThesaurus が保持する置換ルールの一部として機能します。
  • TheLexeme: コンパイル前のレキシーム情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • tnvariant が 0 の場合や、リストの終端（NULL）に達した際の処理は、各操作関数（findVariant 等）の実装に依存します。

保守担当者向けメモ

  • 本構造体はリンクリスト形式で構成されているため、探索時のポインタ操作には注意が必要です。
  • idsubst は DictThesaurus 内の置換結果配列へのインデックスとして機能するため、辞書更新時の整合性に注意してください。
