sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-declare.c:38

目的

ECPG（Embedded SQL for C）アプリケーションにおいて、SQL 操作の結果やエラー情報を保持・伝達するための通信領域（SQL Communication Area）を定義する。

シグネチャまたは形状

struct sqlca_t {
    char    sqlcaid[8];
    long    sqlabc;
    long    sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    } sqlerrm;
    char    sqlerrp[8];
    long    sqlerrd[6];
    char    sqlwarn[8];
    char    sqlstate[5];
};

入力と出力

  • **入力**: SQL ステートメントの実行結果に基づき、ECPG ライブラリが各フィールドを更新する。
  • **出力**: アプリケーションは sqlcode や sqlstate を参照し、エラーハンドリングや後続処理の判断を行う。

前提条件と不変条件

  • sqlerrd[2] は、INSERT、UPDATE、DELETE ステートメント実行後に処理された行数を保持する。
  • sqlwarn[0] は、他の警告フラグ（sqlwarn[1] 等）のいずれかが設定された場合に 'W' に設定される。

アルゴリズムまたは振る舞い

各 SQL 操作後に、ECPG ライブラリは実行結果をこの構造体に反映させる。アプリケーションはこれを確認することで、SQL の成功・失敗を判定する。

呼び出し元と呼び出し先

  • **呼び出し元（参照・更新）**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • **呼び出し先**:
  • ECPGget_sqlca() を介してインスタンスを取得するパターンが一般的である。

関連データ構造

  • sqlda_t（SQL Descriptor Area）: 動的 SQL におけるメタデータ管理に使用される。

失敗モードと境界条件

  • sqlerrmc は SQLERRMC_LEN で定義された長さに制限される。
  • sqlstate は 5 文字の標準 SQL 準拠のエラーコードを保持する。

保守担当者向けメモ

  • 本構造体は ECPG のエラーハンドリングの基盤である。変更時は ecpglib および compatlib への影響を考慮すること。
  • sqlerrd や sqlwarn の各要素の定義は、PostgreSQL の ECPG 仕様に厳密に従う必要がある。
