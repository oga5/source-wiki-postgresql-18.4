infobits_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/heapdesc.c:25

目的

ヒープWALレコードに含まれるフラグビット（infobits）を人間が読み取り可能な文字列形式に変換し、指定されたバッファへ出力する。

シグネチャまたは形状

void infobits_desc(StringInfo buf, uint8 infobits, const char *keyname)

入力と出力

  • **入力**:
  • buf: 出力先の StringInfo バッファ。
  • infobits: 解析対象のビットフラグ（uint8）。
  • keyname: 出力文字列の先頭に付与するラベル名。
  • **出力**: なし（buf の内容が更新される）。

前提条件と不変条件

  • buf は有効な StringInfo 構造体であること。
  • buf の末尾がスペースでないことがアサーションにより保証される。

アルゴリズムまたは振る舞い

1. keyname をバッファに書き込み、[ を開始する。 2. infobits の各ビット（XLHL_XMAX_IS_MULTI, XLHL_XMAX_LOCK_ONLY, XLHL_XMAX_EXCL_LOCK, XLHL_XMAX_KEYSHR_LOCK, XLHL_KEYS_UPDATED）を順次チェックし、該当するものがあれば文字列を , 区切りで追加する。 3. 末尾に , が付加された場合、それを削除して整形する。 4. 最後に ] を閉じて終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/rmgrdesc/heapdesc.c 内の各箇所（xlrec->infobits_set または xlrec->old_infobits_set を引数として渡す）。
  • **呼び出し先**: appendStringInfo, appendStringInfoString, appendStringInfoChar（いずれも lib/stringinfo.h 由来）。

関連データ構造

  • StringInfo: 文字列構築用のバッファ構造体。
  • xl_heap_insert 等のWALレコード構造体（infobits_set フィールドを保持）。

失敗モードと境界条件

  • infobits が 0 の場合、keyname: [] と出力される。
  • バッファの末尾処理において、アサーションにより不正な状態（末尾がスペースである等）が検出される。

保守担当者向けメモ

  • 新しいフラグビットが追加された場合は、本関数のビットチェックロジックにも追加が必要。
  • 本関数は pg_waldump 等のログ解析ツールでのみ使用されるため、実行時のパフォーマンス影響はない。
