to_tsvector

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/to_tsany.c:269

目的

入力されたテキストデータを、現在の全文検索設定（default_text_search_config）に基づいて tsvector 型へ変換します。

シグネチャまたは形状

Datum to_tsvector(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る text 型のデータ。
  • **出力**: Datum 型として返される tsvector 構造体。

前提条件と不変条件

  • getTSCurrentConfig(true) を呼び出し、現在のセッションにおける全文検索設定の OID を取得できる必要があります。
  • 入力データは text 型として正しくポインタ参照可能である必要があります。

アルゴリズムまたは振る舞い

1. getTSCurrentConfig(true) を呼び出し、現在の全文検索設定（cfgId）を取得します。 2. 取得した cfgId と入力テキストを引数として、to_tsvector_byid 関数を呼び出します。 3. to_tsvector_byid の実行結果（tsvector）を PG_RETURN_DATUM を介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsvector_op.c 内の各所から呼び出されています。
  • **呼び出し先**:
  • getTSCurrentConfig
  • to_tsvector_byid

関連データ構造

  • text: 入力文字列を保持する PostgreSQL の標準型。
  • tsvector: 全文検索用のバイナリ形式データ構造。

失敗モードと境界条件

  • 現在の全文検索設定が未設定の場合、getTSCurrentConfig がエラーを発生させる可能性があります。
  • 入力テキストの解析過程で辞書やパーサーがエラーを返した場合、そのエラーが伝播します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の変換ロジックは to_tsvector_byid に委譲されています。
  • 変換ロジックの変更が必要な場合は to_tsvector_byid を参照してください。
