GetPGVariableResultDesc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc_funcs.c:393

目的

指定されたGUCパラメータ名に対応する、結果セットのタプル記述子（TupleDesc）を生成します。

シグネチャまたは形状

TupleDesc GetPGVariableResultDesc(const char *name)

入力と出力

  • **入力**: name (GUCパラメータ名、または "all")
  • **出力**: TupleDesc (SQLクエリの結果セット構造を定義する記述子)

前提条件と不変条件

  • name が "all" の場合、3列（name, setting, description）の TEXT 型タプル記述子を返します。
  • name が特定のパラメータ名の場合、そのパラメータ名を列名とする1列の TEXT 型タプル記述子を返します。
  • GetConfigOptionByName を呼び出すため、指定されたパラメータ名がシステムに存在する必要があります。

アルゴリズムまたは振る舞い

1. name が "all" と一致するか判定します。 2. "all" の場合:

  • CreateTemplateTupleDesc(3) で3列の記述子を作成します。
  • 各列（name, setting, description）を TEXTOID 型で初期化します。

    • 3. それ以外の場合:

  • GetConfigOptionByName を呼び出し、正規化されたパラメータ名を取得します。
  • CreateTemplateTupleDesc(1) で1列の記述子を作成します。
  • 取得したパラメータ名を列名とし、TEXTOID 型で初期化します。

    • 4. 生成した tupdesc を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:2122
  • **呼び出し先**:
  • guc_name_compare
  • CreateTemplateTupleDesc
  • TupleDescInitEntry
  • GetConfigOptionByName

関連データ構造

  • TupleDesc: 結果セットのメタデータを保持する構造体。
  • TEXTOID: PostgreSQLのテキスト型識別子。

失敗モードと境界条件

  • GetConfigOptionByName が存在しないパラメータ名を受け取った場合、内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は SHOW コマンド等の結果セットのスキーマを決定するために使用されます。
  • 列の追加や変更を行う際は、TupleDescInitEntry の呼び出し順序と列数に注意してください。
