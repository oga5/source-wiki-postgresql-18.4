gbt_int4_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int4.c:125

目的

GiST インデックス内の int4 型データに対して、インデックスエントリから元の値をフェッチ（取得）する機能を提供します。

シグネチャまたは形状

Datum gbt_int4_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、インデックスエントリから抽出された値を返します。

前提条件と不変条件

  • GISTENTRY ポインタが有効である必要があります。
  • 内部で tinfo（gbtree_ninfo 構造体）を参照しており、これが int4 型用に正しく初期化されていることが前提です。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. 共通ユーティリティ関数である gbt_num_fetch を呼び出します。その際、int4 型固有の設定情報である tinfo を渡します。 3. gbt_num_fetch の戻り値を PG_RETURN_POINTER を介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構（直接的な呼び出し元コードはソース内に明示されていませんが、PG_FUNCTION_INFO_V1(gbt_int4_fetch) により関数として登録されています）。
  • **呼び出し先**: gbt_num_fetch（btree_utils_num.c で定義された共通ユーティリティ関数）。

関連データ構造

  • GISTENTRY: GiST インデックスエントリを保持する構造体。
  • tinfo: int4 型の比較関数やサイズ情報を保持する gbtree_ninfo 構造体。

失敗モードと境界条件

  • GISTENTRY が NULL の場合や、不正なポインタが渡された場合の挙動は、呼び出し元の GiST フレームワークの仕様に依存します。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の汎用ロジックをラップしているだけです。ロジックの変更が必要な場合は、まず tinfo の定義内容を確認してください。
  • int4 型特有の処理を追加する場合、本関数自体ではなく tinfo が参照する比較関数や btree_utils_num 側のロジックを検討してください。
