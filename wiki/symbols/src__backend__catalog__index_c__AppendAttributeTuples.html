AppendAttributeTuples

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/index.c:511

目的

新しいインデックスの属性情報をシステムカタログ pg_attribute に登録します。

シグネチャまたは形状

static void AppendAttributeTuples(Relation indexRelation, const Datum *attopts, const NullableDatum *stattargets)

入力と出力

  • **入力**:
  • indexRelation: 登録対象のインデックスリレーション。
  • attopts: 属性オプションの配列（Datum 型）。NULL の場合は無視されます。
  • stattargets: 統計ターゲットの配列（NullableDatum 型）。NULL の場合は無視されます。
  • **出力**: なし（副作用として pg_attribute を更新）。

前提条件と不変条件

  • indexRelation は有効なリレーション構造体であること。
  • pg_attribute システムカタログに対する RowExclusiveLock が取得可能であること。

アルゴリズムまたは振る舞い

1. attopts および stattargets が提供されている場合、FormExtraData_pg_attribute 配列を確保し、各属性のオプションと統計ターゲットを格納します。 2. table_open を使用して pg_attribute を RowExclusiveLock モードで開きます。 3. CatalogOpenIndexes を呼び出し、pg_attribute に関連するインデックスの更新状態を準備します。 4. RelationGetDescr でインデックスのタプル記述子を取得します。 5. InsertPgAttributeTuples を呼び出し、カタログへ属性タプルを挿入します。 6. CatalogCloseIndexes でインデックスの更新状態を終了し、table_close で pg_attribute を閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/index.c 内のインデックス作成処理（index_create 等）から呼び出されます。
  • **呼び出し先**: table_open, CatalogOpenIndexes, RelationGetDescr, InsertPgAttributeTuples, CatalogCloseIndexes, table_close。

関連データ構造

  • Relation: インデックスおよび pg_attribute のリレーション情報。
  • FormExtraData_pg_attribute: 属性の追加情報を保持する内部構造体。
  • CatalogIndexState: カタログ更新時のインデックス状態管理用。

失敗モードと境界条件

  • attopts や stattargets が NULL の場合、対応するフィールドはデフォルト値（NULL）として扱われます。
  • カタログのロック取得や挿入処理において、トランザクションの整合性制約に違反した場合はエラーが発生します。

保守担当者向けメモ

  • 本関数は pg_attribute への書き込みを行うため、呼び出し側で適切なトランザクション状態とロックが維持されていることを確認してください。
  • InsertPgAttributeTuples への依存が強いため、カタログスキーマの変更時には当該関数と併せて確認が必要です。
