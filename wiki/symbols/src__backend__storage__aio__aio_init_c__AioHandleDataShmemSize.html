AioHandleDataShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_init.c:78

目的

非同期I/O（AIO）サブシステムにおいて、各I/Oハンドルが保持するデータ領域に必要な共有メモリサイズを算出します。

シグネチャまたは形状

Size AioHandleDataShmemSize(void)

入力と出力

  • **入力**: グローバル変数 io_max_combine_limit、io_max_concurrency、および関数 AioProcs() が返すプロセス数。
  • **出力**: 計算された Size 型のメモリサイズ。

前提条件と不変条件

  • io_max_combine_limit、AioProcs()、io_max_concurrency の積がメモリサイズとして算出されます。
  • 各バッファが参照する iovec に関連付けられたデータ領域として uint64 型のサイズが使用されます。

アルゴリズムまたは振る舞い

1. io_max_combine_limit と AioProcs() の積を計算します。 2. その結果に io_max_concurrency を乗算します。 3. 最終的に sizeof(uint64) を乗算し、必要なバイト数を返します。 4. 計算には mul_size を使用し、オーバーフローを考慮した安全なサイズ算出を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio_init.c 内の共有メモリサイズ合計算出処理（AioShmemSize 内の計算過程）。
  • src/backend/storage/aio/aio_init.c 内の ShmemInitStruct 呼び出し時（"AioHandleData" 領域の確保）。
  • **呼び出し先**:
  • mul_size（PostgreSQLのメモリサイズ計算用ユーティリティ）。
  • AioProcs()（プロセス数を取得する関数）。

関連データ構造

  • PgAioHandle（本関数が算出するメモリ領域に関連する構造体）。

失敗モードと境界条件

  • mul_size による計算でオーバーフローが発生した場合、PostgreSQLの標準的なエラー処理（ereport(ERROR, ...)）が呼び出される可能性があります。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定する重要な計算の一部です。io_max_combine_limit や io_max_concurrency の定義を変更する際は、本関数の算出結果が共有メモリの割り当て上限を超えないか確認してください。
