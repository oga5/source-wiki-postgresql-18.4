SyncRepReleaseWaiters

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:473

目的

WAL送信プロセス（WALSender）がスタンバイサーバーからの進捗報告を受け取り、同期コミットを待機しているバックエンドプロセスを解放する。

シグネチャまたは形状

void SyncRepReleaseWaiters(void)

入力と出力

  • **入力**: 引数なし。内部的に MyWalSnd（現在のWALSender状態）および WalSndCtl（共有メモリ上の制御データ）を参照する。
  • **出力**: なし。副作用として、待機中のバックエンドプロセスのラッチを解除し、共有メモリ上の lsn 情報を更新する。

前提条件と不変条件

  • 呼び出し元はWALSenderプロセスである必要がある。
  • MyWalSnd->sync_standby_priority が0より大きく、かつ状態が WALSNDSTATE_STREAMING または WALSNDSTATE_STOPPING であること。
  • MyWalSnd->flush が有効なLSNであること。

アルゴリズムまたは振る舞い

1. **初期チェック**: WALSenderが同期スタンバイとして機能していない、またはストリーミング中でない場合は即座に終了する。 2. **排他制御**: SyncRepLock を排他モードで取得する。 3. **同期状態判定**: SyncRepGetSyncRecPtr を呼び出し、現在の同期スタンバイ群の進捗（Write/Flush/Apply）を計算する。 4. **通知**: 自身が同期スタンバイとして認識された場合、必要に応じてログを出力する。 5. **待機解除**: 取得した各LSN（Write/Flush/Apply）が共有メモリ上の記録より進んでいる場合、SyncRepWakeQueue を呼び出して該当する待機中のバックエンドをウェイクアップする。 6. **終了**: SyncRepLock を解放し、デバッグログを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c
  • **呼び出し先**:
  • SyncRepGetSyncRecPtr
  • SyncRepWakeQueue
  • LWLockAcquire / LWLockRelease
  • ereport / elog

関連データ構造

  • WalSndCtlData: 共有メモリ上のWAL送信制御構造体。
  • SyncRepLock: 待機キューを保護するLWLock。
  • SyncRepConfig: 同期レプリケーションの設定情報。

失敗モードと境界条件

  • **同期対象外**: sync_standby_priority が0の場合、待機解除を行わずに終了する。
  • **無効なLSN**: MyWalSnd->flush が無効な場合、処理をスキップする。
  • **同期スタンバイ不足**: SyncRepGetSyncRecPtr が同期条件を満たさないと判断した場合、待機解除は行われない。

保守担当者向けメモ

  • 本関数は SyncRepLock を保持して SyncRepWakeQueue を呼び出すため、デッドロックを避けるよう注意が必要である。
  • announce_next_takeover フラグは、スタンバイの同期状態遷移を追跡するために使用される。
