makeSimpleA_Expr

定義

  • src/backend/nodes/makefuncs.c:47

目的

A_Expr（構文解析ツリーにおける演算子式ノード）を生成し、指定された引数で初期化する。

シグネチャまたは形状

A_Expr *makeSimpleA_Expr(A_Expr_Kind kind, char *name, Node *lexpr, Node *rexpr, int location)

入力と出力

  • **入力**:
  • kind: 演算子の種類（A_Expr_Kind）。
  • name: 演算子名を表す文字列。
  • lexpr: 左辺のノード。
  • rexpr: 右辺のノード。
  • location: クエリ文字列内での位置情報。
  • **出力**:
  • 初期化済みの A_Expr 構造体へのポインタ。

前提条件と不変条件

  • makeNode(A_Expr) により、CurrentMemoryContext 上にメモリが確保される。
  • name は list_make1(makeString(name)) を通じてリスト形式に変換され、A_Expr.name に格納される。

アルゴリズムまたは振る舞い

1. makeNode(A_Expr) を呼び出し、ノードのメモリを確保する。 2. 引数 kind を a->kind に設定する。 3. 引数 name を makeString でラップし、さらに list_make1 でリスト化して a->name に設定する。 4. 引数 lexpr、rexpr、location をそれぞれ対応するフィールドに代入する。 5. 構築された A_Expr ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_clause.c
  • src/backend/parser/parse_expr.c
  • **呼び出し先**:
  • makeNode (マクロ)
  • list_make1
  • makeString

関連データ構造

  • A_Expr: 構文解析ツリーで使用される演算子式ノード。
  • A_Expr_Kind: 演算子の種類を定義する列挙型。

失敗モードと境界条件

  • メモリ不足時には makeNode 経由でエラーが発生する可能性がある。
  • lexpr や rexpr に NULL を渡すことは可能だが、後続の処理でノードの型チェックを行う際に注意が必要。

保守担当者向けメモ

  • 本関数は単純な演算子式を生成するためのヘルパーであり、複雑な演算子名（修飾名など）を扱う場合は別の構築ロジックが必要となる場合がある。
  • name フィールドがリスト構造である点に注意すること。
