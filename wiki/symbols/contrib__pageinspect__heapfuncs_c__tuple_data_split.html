tuple_data_split

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/heapfuncs.c:435

目的

生のヒープタプルデータから属性値を分離・復元し、SQLで利用可能な形式（配列）として抽出する。

シグネチャまたは形状

Datum tuple_data_split(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relid (Oid): 対象リレーションのOID。
  • raw_data (bytea): タプルのバイナリデータ。
  • t_infomask, t_infomask2 (uint16): タプルの状態フラグ。
  • t_bits_str (text): NULLビットマップの文字列表現。
  • do_detoast (bool, オプション): TOASTデータの自動展開フラグ。
  • **出力**:
  • 復元された属性値の配列（Datum）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • t_infomask に HEAP_HASNULL が含まれる場合、t_bits_str は NULL であってはならず、かつ期待されるビット長と一致しなければならない。
  • t_infomask に HEAP_HASNULL が含まれない場合、t_bits_str は NULL でなければならない。

アルゴリズムまたは振る舞い

1. 引数を取得し、スーパーユーザー権限を検証する。 2. raw_data が NULL の場合は即座に NULL を返す。 3. t_infomask を確認し、NULL ビットマップが存在する場合は t_bits_str を bits8 配列に変換する。 4. tuple_data_split_internal を呼び出し、実際のデータ分割処理を実行する。 5. 変換に使用した t_bits メモリを解放する。 6. 結果を Datum として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 マクロによる登録）。
  • **呼び出し先**: tuple_data_split_internal を呼び出す。

関連データ構造

  • HeapTupleHeader（access/htup_details.h）: タプルヘッダの構造。
  • bits8（postgres.h）: NULL ビットマップの表現。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了する。
  • **データ破損**:
  • t_bits_str が NULL であるべき時に存在する場合、またはその逆の場合。
  • t_bits_str の長さが期待値と一致しない場合。
  • 上記の場合、ERRCODE_DATA_CORRUPTED でエラー終了する。

保守担当者向けメモ

  • 本関数は tuple_data_split_internal のラッパーとして機能している。
  • 破損したページを扱う可能性があるため、メモリ管理とバリデーションには注意が必要である。
