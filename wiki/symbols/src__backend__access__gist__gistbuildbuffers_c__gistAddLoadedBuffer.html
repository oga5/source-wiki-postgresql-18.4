gistAddLoadedBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuildbuffers.c:197

目的

GiST インデックスのバッファリングビルドにおいて、メモリ上に保持されているノードバッファ（GISTNodeBuffer）のリストに、指定されたバッファを追加・登録します。

シグネチャまたは形状

static void gistAddLoadedBuffer(GISTBuildBuffers *gfbb, GISTNodeBuffer *nodeBuffer)

入力と出力

  • **入力**:
  • gfbb: バッファリングビルド全体の管理構造体。
  • nodeBuffer: 追加対象のノードバッファ。
  • **出力**: なし（gfbb 内の loadedBuffers 配列が更新されます）。

前提条件と不変条件

  • nodeBuffer が一時バッファ（isTemp が真）である場合、この関数は何も行わず即座に復帰します。
  • gfbb->loadedBuffers 配列の容量が不足している場合、repalloc を用いて容量を2倍に拡張します。

アルゴリズムまたは振る舞い

1. nodeBuffer->isTemp を確認し、真であれば処理を終了します。 2. gfbb->loadedBuffersCount が gfbb->loadedBuffersLen に達している場合、gfbb->loadedBuffersLen を2倍に拡張し、repalloc でメモリを再確保します。 3. gfbb->loadedBuffers 配列の末尾に nodeBuffer を格納します。 4. gfbb->loadedBuffersCount をインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuildbuffers.c 内の関数（gistGetNodeBuffer 等）から呼び出されています。
  • **呼び出し先**:
  • repalloc（メモリ再確保用）

関連データ構造

  • GISTBuildBuffers: バッファリングビルドの状態を保持する構造体。loadedBuffers 配列およびそのサイズ情報を管理します。
  • GISTNodeBuffer: 個別のノードバッファを定義する構造体。

失敗モードと境界条件

  • メモリ不足時には repalloc がエラーを発生させ、システム全体で例外処理が行われます。
  • isTemp フラグが設定されたバッファは、このリストには追加されません。

保守担当者向けメモ

  • 本関数は、メモリ上にロードされているバッファを追跡するための管理用リストを操作します。
  • loadedBuffers 配列の拡張は指数的に行われるため、頻繁な再確保は発生しない設計となっています。
