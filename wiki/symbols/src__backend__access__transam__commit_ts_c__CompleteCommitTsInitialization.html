CompleteCommitTsInitialization

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/commit_ts.c:641

目的

track_commit_timestamp GUCパラメータの状態に基づき、コミットタイムスタンプ機能の有効化または無効化を行い、システムの状態を整合させる。

シグネチャまたは形状

void CompleteCommitTsInitialization(void)

入力と出力

  • 入力: なし（グローバル変数 track_commit_timestamp を参照）
  • 出力: なし（副作用として機能の有効化/無効化を実行）

前提条件と不変条件

  • 本関数は、リカバリの開始時や XLOG_PARAMETER_CHANGE レコードの適用時など、システム設定が変更されたタイミングで呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. グローバル変数 track_commit_timestamp を評価する。 2. track_commit_timestamp が偽の場合、DeactivateCommitTs() を呼び出し、機能を無効化して不要なデータを削除する。 3. track_commit_timestamp が真の場合、ActivateCommitTs() を呼び出し、機能を有効化する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xlog.c:6187
  • 呼び出し先:
  • DeactivateCommitTs()
  • ActivateCommitTs()

関連データ構造

  • track_commit_timestamp (GUC変数)

失敗モードと境界条件

  • 特記事項なし。

保守担当者向けメモ

  • 本関数はプライマリおよびスタンバイの両方で実行される。スタンバイにおいては、制御ファイルの内容やWALのパラメータ変更レコードの再生に応じて、機能の有効状態を同期させる役割を担う。
