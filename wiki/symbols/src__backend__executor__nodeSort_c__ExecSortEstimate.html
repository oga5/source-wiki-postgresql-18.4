ExecSortEstimate

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSort.c:415

目的

並列クエリ実行時、ソートノードの統計情報（TuplesortInstrumentation）を共有メモリに格納するために必要な領域サイズを見積もる。

シグネチャまたは形状

void ExecSortEstimate(SortState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: ソートノードの状態を保持する SortState 構造体。
  • pcxt: 並列クエリのコンテキスト情報を含む ParallelContext 構造体。
  • **出力**: なし（副作用として pcxt->estimator に見積もり値が追加される）。

前提条件と不変条件

  • node->ss.ps.instrument が有効であり、かつ pcxt->nworkers > 0 である場合にのみ見積もり処理が実行される。

アルゴリズムまたは振る舞い

1. インストゥルメンテーション（計測）が不要、またはワーカーが存在しない場合は即座に終了する。 2. ワーカー数に基づき、必要な共有メモリサイズを計算する。

  • TuplesortInstrumentation の配列サイズを算出。
  • SharedSortInfo 構造体のオフセットを加算。

    • 3. shm_toc_estimate_chunk を呼び出し、計算したサイズを共有メモリの TOC（Table of Contents）見積もりに登録する。 4. shm_toc_estimate_keys を呼び出し、1つのキーを登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c:294 からの呼び出しが確認されている。
  • **呼び出し先**:
  • shm_toc_estimate_chunk
  • shm_toc_estimate_keys
  • mul_size / add_size（サイズ計算用ユーティリティ）

関連データ構造

  • SortState: ソートノードの実行状態。
  • ParallelContext: 並列実行のコンテキスト。
  • TuplesortInstrumentation: ソート処理の統計情報。
  • SharedSortInfo: 並列ワーカー間で共有されるソート情報。

失敗モードと境界条件

  • pcxt->nworkers が 0 の場合、見積もりは行われない。
  • node->ss.ps.instrument が NULL の場合、見積もりは行われない。

保守担当者向けメモ

  • 本関数は並列クエリの初期化フェーズで呼び出される。共有メモリのレイアウト変更や SharedSortInfo 構造の変更時には、サイズ計算ロジックを更新する必要がある。
