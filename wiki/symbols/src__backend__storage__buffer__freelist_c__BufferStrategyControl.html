BufferStrategyControl

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/buffer/freelist.c:30

目的

PostgreSQLの共有バッファプールにおけるバッファ置換戦略（クロック掃引法）およびフリーリストの状態を管理するグローバルな制御構造体です。

シグネチャまたは形状

typedef struct {
    slock_t             buffer_strategy_lock;
    pg_atomic_uint32    nextVictimBuffer;
    int                 firstFreeBuffer;
    int                 lastFreeBuffer;
    uint32              completePasses;
    pg_atomic_uint32    numBufferAllocs;
    int                 bgwprocno;
} BufferStrategyControl;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。各メンバはバッファマネージャの各関数（StrategyGetBuffer 等）によって読み書きされます。

前提条件と不変条件

  • buffer_strategy_lock: 構造体内の各メンバ（特にフリーリスト関連）へのアクセスを保護するスピンロックです。
  • nextVictimBuffer: クロック掃引のハンド位置を保持します。値は単調増加し、実際のバッファインデックスを得るには NBuffers での剰余演算が必要です。
  • lastFreeBuffer: firstFreeBuffer が -1（リストが空）の場合、本メンバの値は未定義となります。

アルゴリズムまたは振る舞い

  • **クロック掃引管理**: nextVictimBuffer を用いてバッファプールを巡回し、再利用候補を特定します。
  • **フリーリスト管理**: firstFreeBuffer と lastFreeBuffer を用いて、未使用バッファのリストを高速に管理します。
  • **統計収集**: completePasses および numBufferAllocs により、バッファ割り当て状況を追跡します。
  • **通知**: bgwprocno を介して、バックグラウンドライター（bgwriter）への通知制御を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/freelist.c 内の StrategyInitialize（初期化時）およびバッファ割り当て・解放に関連する各関数。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • BufferDesc: バッファ記述子（storage/buf_internals.h で定義）。
  • BufferAccessStrategyData: プロセス固有のリングバッファ戦略を管理する構造体。

失敗モードと境界条件

  • フリーリストが空（firstFreeBuffer == -1）の場合、lastFreeBuffer は参照してはなりません。
  • 統計カウンタは、単一のbgwriterサイクル内でオーバーフローしない十分なビット幅を持つ必要があります。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されます。
  • buffer_strategy_lock はフリーリスト操作の保護を主目的としており、クロック掃引のメインループではロック競合を避ける設計が求められます。
  • nextVictimBuffer の更新にはアトミック操作が使用されています。
