date_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/date.c:113

目的

文字列形式の日付入力を受け取り、PostgreSQLの内部表現である DateADT（エポックからの経過日数）に変換する。

シグネチャまたは形状

Datum date_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して文字列を受け取る。
  • **出力**: 変換された DateADT 値を Datum 型として返す。解析失敗時は PG_RETURN_NULL() を実行する。

前提条件と不変条件

  • 入力文字列は ParseDateTime および DecodeDateTime で解析可能な形式であること。
  • 変換後の日付は、IS_VALID_JULIAN および IS_VALID_DATE の範囲内であること。範囲外の場合はエラーを発生させる。

アルゴリズムまたは振る舞い

1. ParseDateTime で文字列をフィールドに分解し、DecodeDateTime で日付情報（pg_tm）へ変換する。 2. dtype に基づき以下の処理を行う：

  • DTK_DATE: 通常の日付として処理。
  • DTK_EPOCH: エポック時刻を取得。
  • DTK_LATE / DTK_EARLY: 無限大（Infinity）として定数を返す。

    • 3. date2j 関数を用いてユリウス日を計算し、POSTGRES_EPOCH_JDATE を減算して DateADT を算出する。 4. 算出された値が有効範囲内であることを検証し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていないが、SQLレベルでの DATE 型入力関数として利用される。
  • **呼び出し先**: ParseDateTime, DecodeDateTime, GetEpochTime, date2j, DateTimeParseError, DateTimeErrorExtra 等。

関連データ構造

  • DateADT: int32 型（日付の内部表現）。
  • pg_tm: 日付フィールドを保持する構造体。
  • DateTimeErrorExtra: エラー発生時の詳細情報を保持する構造体。

失敗モードと境界条件

  • **解析エラー**: 文字列が日付として不正な場合、DateTimeParseError を経由してエラーを報告する。
  • **範囲外エラー**: IS_VALID_JULIAN または IS_VALID_DATE のチェックに失敗した場合、ERRCODE_DATETIME_VALUE_OUT_OF_RANGE エラーを発生させる。

保守担当者向けメモ

  • 本関数は DATE 型の入力インターフェースであり、date_out と対になる。
  • DATEVAL_NOBEGIN や DATEVAL_NOEND を使用して無限大を表現しているため、これら定数の変更時には注意が必要。
  • date2j による計算結果が int32 の範囲に収まることが前提となっている。
