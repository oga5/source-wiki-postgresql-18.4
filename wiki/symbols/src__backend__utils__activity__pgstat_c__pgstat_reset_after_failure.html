pgstat_reset_after_failure

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat.c:2043

目的

統計システムにおいて、障害発生後に統計情報を初期化・リセットし、整合性を回復させるために使用されます。

シグネチャまたは形状

static void pgstat_reset_after_failure(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の統計状態を更新）

前提条件と不変条件

  • 呼び出し時に GetCurrentTimestamp() を使用して現在の時刻を取得します。
  • PgStat_Kind の範囲（PGSTAT_KIND_MIN から PGSTAT_KIND_MAX）で反復処理を行います。

アルゴリズムまたは振る舞い

1. 現在のタイムスタンプを取得します。 2. 全ての統計種別を走査し、fixed_amount が設定されている（固定数の）統計種別に対して、対応する reset_all_cb コールバックを実行してリセットします。 3. pgstat_drop_all_entries() を呼び出し、可変数の統計エントリをすべて破棄します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/activity/pgstat.c 内の複数箇所（詳細はソースコードの参照箇所に依存）。
  • 呼び出し先:
  • pgstat_get_kind_info
  • kind_info->reset_all_cb
  • pgstat_drop_all_entries

関連データ構造

  • PgStat_Kind: 統計種別の列挙型。
  • PgStat_KindInfo: 統計種別の振る舞いを定義する構造体。

失敗モードと境界条件

  • pgstat_get_kind_info(kind) が NULL を返す場合、その種別はスキップされます。
  • fixed_amount が偽である種別は、ループ内でのリセット対象から除外されます。

保守担当者向けメモ

  • 本関数は統計システムのリカバリ処理の一部です。統計種別の追加や削除を行う際は、PgStat_KindInfo の定義が本関数のループ処理と整合しているか確認してください。
