RunWorker

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/parallel.c:830

目的

並列処理におけるワーカープロセスのメインループを実行し、リーダープロセスから送信されるジョブコマンドを処理する。

シグネチャまたは形状

static void RunWorker(ArchiveHandle *AH, ParallelSlot *slot)

入力と出力

  • **入力**:
  • AH: アーカイブのハンドル情報。
  • slot: ワーカーの通信パイプ情報を含む ParallelSlot 構造体。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • slot には有効なパイプファイルディスクリプタが格納されている必要がある。
  • AH は CloneArchive によってワーカー専用のインスタンスに複製される。

アルゴリズムまたは振る舞い

1. slot から読み込み用および書き込み用のパイプディスクリプタを取得する。 2. CloneArchive を呼び出し、ワーカー専用のアーカイブ状態とデータベース接続を作成する。 3. set_cancel_slot_archive を呼び出し、シグナルハンドラが現在のアーカイブを参照できるように設定する。 4. AH->SetupWorkerPtr を介してワーカーのセットアップ関数を実行する。 5. WaitForCommands を呼び出し、パイプ経由でコマンドを受信・実行するループに入る。 6. 終了時、set_cancel_slot_archive をリセットし、データベース接続の切断とアーカイブのクリーンアップ（DeCloneArchive）を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/parallel.c 内の並列処理開始ロジック（直接の呼び出し元関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: CloneArchive, set_cancel_slot_archive, AH->SetupWorkerPtr, WaitForCommands, DisconnectDatabase, DeCloneArchive。

関連データ構造

  • ArchiveHandle: アーカイブの操作状態を保持。
  • ParallelSlot: ワーカーの通信パイプや状態を管理。

失敗モードと境界条件

  • データベース接続のクローン失敗や、パイプ通信の異常終了時にプロセスが終了する可能性がある。
  • シグナル受信時の挙動は set_cancel_slot_archive で管理される。

保守担当者向けメモ

  • 本関数は Unix/Windows 両対応の並列処理の基盤である。Windows 環境では fork の代わりにスレッドが使用されるため、CloneArchive による状態分離が重要となる。
  • AH->SetupWorkerPtr は実行時に動的に決定されるため、デバッグ時は ArchiveHandle の初期化箇所を確認すること。
