PLyProcedureKey

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_procedure.h:53

目的

PL/Pythonにおけるプロシージャ（関数およびトリガー）のキャッシュ検索用キーとして機能します。関数OIDとリレーションOIDを組み合わせることで、特定の実行コンテキストを一意に識別します。

シグネチャまたは形状

typedef struct PLyProcedureKey
{
	Oid			fn_oid;			/* function OID */
	Oid			fn_rel;			/* triggered-on relation or InvalidOid */
} PLyProcedureKey;

入力と出力

  • **入力**: 構造体初期化時に fn_oid（関数識別子）および fn_rel（トリガーの場合は対象リレーションOID、それ以外は InvalidOid）を受け取ります。
  • **出力**: キャッシュハッシュテーブルの検索キーとして利用されます。

前提条件と不変条件

  • fn_rel は、トリガー関数でない場合は InvalidOid である必要があります。
  • キャッシュハッシュテーブルのキーとして使用されるため、構造体全体がハッシュ計算の対象となります。

アルゴリズムまたは振る舞い

PLy_procedure_get が呼び出された際、この構造体に値をセットしてハッシュテーブルを検索します。キャッシュミスが発生した場合は、このキー情報をもとにシステムカタログからプロシージャ定義を読み込み、新しいキャッシュエントリを作成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_procedure.c 内のキャッシュ管理ロジック（PLy_procedure_get 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • PLyProcedureEntry: キャッシュハッシュテーブルのエントリ構造体であり、本キーを保持します。
  • PLyProcedure: キャッシュされるプロシージャの本体情報。

失敗モードと境界条件

  • fn_rel に誤ったOIDが指定された場合、キャッシュの不整合や誤ったプロシージャの取得を招く可能性があります。
  • ハッシュテーブルのサイズ設定（hash_ctl.keysize）と本構造体のサイズが一致している必要があります。

保守担当者向けメモ

  • トリガー関数はリレーションに依存するため、キャッシュキーに fn_rel を含めることで、同一関数が異なるテーブルでトリガーとして使用される際の整合性を確保しています。
  • 構造体のメンバ変更時は、plpy_procedure.c 側のハッシュテーブル初期化処理（keysize の設定）を必ず更新してください。
