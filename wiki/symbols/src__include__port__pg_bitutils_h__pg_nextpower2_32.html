pg_nextpower2_32

定義

  • **種別**: static inline 関数
  • **定義箇所**: src/include/port/pg_bitutils.h:188

目的

与えられた 32 ビット符号なし整数 num 以上で、最小の 2 のべき乗（power of 2）を算出する。

シグネチャまたは形状

uint32 pg_nextpower2_32(uint32 num)

入力と出力

  • **入力**: num (32ビット符号なし整数)
  • **出力**: num 以上の最小の 2 のべき乗値。

前提条件と不変条件

  • num > 0 であること。
  • num <= PG_UINT32_MAX / 2 + 1 であること（オーバーフロー防止のため）。
  • 上記条件は Assert により検証される。

アルゴリズムまたは振る舞い

1. num が既に 2 のべき乗であるかを確認する（num & (num - 1) == 0）。該当する場合は num をそのまま返す。 2. そうでない場合、pg_leftmost_one_pos32 を使用して最上位ビット（MSB）の位置を特定する。 3. MSB の位置に基づき、ビットシフト演算（1 << (pos + 1)）を行い、次の 2 のべき乗を算出する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginfast.c
  • src/backend/access/hash/hashpage.c
  • src/backend/access/hash/hashsort.c
  • src/backend/access/heap/heapam.c
  • src/backend/access/table/tableam.c
  • src/backend/commands/explain_state.c
  • **呼び出し先**:
  • pg_leftmost_one_pos32（MSB 位置取得用）

関連データ構造

  • 特になし（プリミティブ型 uint32 のみを扱う）。

失敗モードと境界条件

  • num = 0 を渡した場合、Assert が有効なビルドではアサーションエラーが発生する。
  • num が PG_UINT32_MAX / 2 + 1 を超える場合、Assert により停止する。

保守担当者向けメモ

  • 本関数はインライン展開されるため、呼び出し側のパフォーマンスに直接影響する。
  • 64 ビット版が必要な場合は pg_nextpower2_64 を参照すること。
  • 新しいプラットフォームへの移植時は、pg_leftmost_one_pos32 が効率的に実装されているか確認すること。
