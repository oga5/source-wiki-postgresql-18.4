spgist_name_compress

定義

  • 種別: function
  • 定義箇所: src/test/modules/spgist_name_ops/spgist_name_ops.c:495

目的

SP-GiSTインデックスに格納する name 型の値を、内部ストレージ形式である text 型に変換（圧縮）する。

シグネチャまたは形状

Datum spgist_name_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される name 型のデータム。
  • **出力**: formTextDatum によって生成された、text 型のデータムを返す。

前提条件と不変条件

  • 入力値は name 型として正当であること。
  • 内部的に NameStr マクロを使用して文字列ポインタを取得するため、有効な Name 構造体へのポインタが必要。

アルゴリズムまたは振る舞い

1. PG_GETARG_NAME(0) を使用して、入力された name 型のデータムを取得する。 2. NameStr マクロにより、name 構造体から C 文字列（char *）を取り出す。 3. strlen で文字列長を計算する。 4. formTextDatum を呼び出し、取得した文字列と長さを基に text 型のデータムを構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTのインデックス挿入処理（PostgreSQLのインデックスアクセスメソッド層から呼び出される）。
  • **呼び出し先**: formTextDatum（text 型データム構築用ヘルパー）。
  • 補足: 呼び出し元の具体的な関数名はソースコードの断片からは特定できません。

関連データ構造

  • Name: PostgreSQLの name 型を表現する構造体。
  • text: 変換先の可変長データ型。

失敗モードと境界条件

  • 入力が NULL の場合、PG_GETARG_NAME の仕様に従い動作する（通常はエラーまたは NULL 処理）。
  • 文字列長が text 型の許容範囲を超える場合、formTextDatum の実装に依存する。

保守担当者向けメモ

  • 本関数は照合順序（Collation）を考慮しないバイナリ変換を行う。
  • spgtextproc.c のロジックを模倣しており、name 型を text 型として扱うためのブリッジとして機能する。
