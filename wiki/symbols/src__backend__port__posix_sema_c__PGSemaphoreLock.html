PGSemaphoreLock

定義

  • 種別: function
  • 定義箇所: src/backend/port/posix_sema.c:318

目的

指定されたセマフォに対してロック操作（待機）を行い、リソースの排他制御を実現する。

シグネチャまたは形状

void PGSemaphoreLock(PGSemaphore sema)

入力と出力

  • **入力**: sema (PGSemaphore) - ロック対象のセマフォへのポインタ。
  • **出力**: なし。成功時は呼び出し元に復帰する。

前提条件と不変条件

  • sema は PGSemaphoreCreate 等によって適切に初期化されている必要がある。
  • PG_SEM_REF(sema) マクロにより、内部の sem_t 構造体へアクセス可能であること。

アルゴリズムまたは振る舞い

1. sem_wait を呼び出し、セマフォのカウントが正になるまで待機する。 2. sem_wait がシグナル割り込み（EINTR）によって中断された場合、ループして再試行する。 3. sem_wait がその他のエラーを返した場合、elog(FATAL, ...) を呼び出し、プロセスを異常終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/clog.c
  • src/backend/storage/ipc/procarray.c
  • src/backend/storage/lmgr/lwlock.c
  • **呼び出し先**:
  • sem_wait (POSIX API)

関連データ構造

  • PGSemaphore: POSIX セマフォ実装における sem_t * の抽象型。
  • PG_SEM_REF(x): PGSemaphore から内部の sem_t を抽出するマクロ。

失敗モードと境界条件

  • **シグナル割り込み**: EINTR が発生した場合は自動的に再試行するため、呼び出し元には影響しない。
  • **致命的エラー**: sem_wait が EINTR 以外のエラー（無効なセマフォなど）を返した場合、FATAL レベルのログを出力してプロセスを停止させる。

保守担当者向けメモ

  • 本関数は POSIX セマフォ実装における標準的なロック処理である。
  • sysv_sema.c の実装と同様の設計思想（シグナル割り込みへの耐性）が適用されている。
