shm_mq_receive

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_mq.c:571

目的

共有メモリメッセージキューからメッセージを受信します。

シグネチャまたは形状

shm_mq_result shm_mq_receive(shm_mq_handle *mqh, Size *nbytesp, void **datap, bool nowait)

入力と出力

  • mqh: キューへのアクセス用プライベートハンドル。
  • nbytesp: 受信したメッセージのサイズを格納するポインタ（出力）。
  • datap: 受信したメッセージデータへのポインタを格納するポインタ（出力）。
  • nowait: true の場合、データがない場合にブロッキングせず SHM_MQ_WOULD_BLOCK を返します。
  • 戻り値: SHM_MQ_SUCCESS（成功）、SHM_MQ_WOULD_BLOCK（待機が必要）、SHM_MQ_DETACHED（相手側が切断済み）のいずれか。

前提条件と不変条件

  • 呼び出し元プロセスが、当該キューの受信側（mq->mq_receiver == MyProc）として登録されている必要があります。
  • メッセージサイズが MaxAllocSize を超える場合はエラーとなります。

アルゴリズムまたは振る舞い

1. **接続確認**: 送信側が未接続の場合、nowait 設定に応じて待機または状態確認を行います。 2. **消費済み領域の解放**: 蓄積された消費済みデータ量がリングバッファサイズの1/4を超えた場合、共有メモリ上の読み取りポインタを更新します。 3. **メッセージ長取得**: メッセージの長さを示すワードを読み取ります。必要に応じて複数回に分けて読み込み、再構築します。 4. **データ読み取り**: メッセージ本体を読み取ります。

  • 共有メモリ上で連続している場合は、その領域を直接指すポインタを返します。
  • リングバッファを跨いでいる場合は、内部バッファにコピーして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: parallel.c、tqueue.c、pqmq.c、applyparallelworker.c、および test_shm_mq モジュールから呼び出されます。
  • **呼び出し先**: shm_mq_receive_bytes、shm_mq_counterparty_gone、shm_mq_wait_internal、shm_mq_inc_bytes_read 等を内部的に呼び出します。

関連データ構造

  • shm_mq: 共有メモリ上のキュー本体。
  • shm_mq_handle: プロセス固有のキュー操作用ハンドル。

失敗モードと境界条件

  • **メッセージサイズ超過**: nbytes > MaxAllocSize の場合、ereport(ERROR) で停止します。
  • **相手側の切断**: 送信側が終了している場合、SHM_MQ_DETACHED を返します。
  • **バッファ不足**: メッセージがリングバッファを跨ぐ場合、mqh->mqh_buffer を動的に拡張します。

保守担当者向けメモ

  • SetLatch() の呼び出し頻度を抑えるため、読み取りポインタの更新は一定量溜まるまで遅延されます。
  • メッセージ長ワードがアライメント境界を跨ぐ可能性があるため、sizeof(Size) > MAXIMUM_ALIGNOF の環境では再構築処理が実行されます。
