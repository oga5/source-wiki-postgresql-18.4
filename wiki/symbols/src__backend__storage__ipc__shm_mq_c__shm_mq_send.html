shm_mq_send

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_mq.c:328

目的

共有メモリメッセージキューに対して、単一のデータブロックを送信するためのインターフェースを提供します。

シグネチャまたは形状

shm_mq_result shm_mq_send(shm_mq_handle *mqh, Size nbytes, const void *data, bool nowait, bool force_flush);

入力と出力

  • **入力**:
  • mqh: 送信先キューのハンドル。
  • nbytes: 送信するデータのバイト数。
  • data: 送信するデータへのポインタ。
  • nowait: true の場合、キューが満杯でも待機せず SHM_MQ_WOULD_BLOCK を返します。
  • force_flush: true の場合、送信後に共有メモリ上のポインタ更新を強制的に反映させます。
  • **出力**:
  • shm_mq_result: 送信結果（成功、キューのデタッチ、またはブロック状態）。

前提条件と不変条件

  • mqh は有効な shm_mq_handle を指している必要があります。
  • 内部的に shm_mq_iovec 構造体へデータをマッピングし、shm_mq_sendv を呼び出すラッパーとして動作します。

アルゴリズムまたは振る舞い

1. 指定された data ポインタと nbytes を shm_mq_iovec 構造体に格納します。 2. shm_mq_sendv を呼び出し、単一のベクトルとしてデータをキューへ書き込みます。 3. shm_mq_sendv の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/tqueue.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/test/modules/test_shm_mq/test.c
  • src/test/modules/test_shm_mq/worker.c
  • **呼び出し先**:
  • shm_mq_sendv

関連データ構造

  • shm_mq_handle: キューへのアクセス用プライベートハンドル。
  • shm_mq_iovec: 送信データと長さを保持する構造体。
  • shm_mq_result: 送信処理のステータスコード。

失敗モードと境界条件

  • キューが満杯の場合、nowait が false であれば待機し、true であれば SHM_MQ_WOULD_BLOCK を返します。
  • 相手側プロセスがデタッチ済みの場合、SHM_MQ_DETACHED が返されます。

保守担当者向けメモ

  • 本関数は shm_mq_sendv の単一要素版です。ロジックの変更は shm_mq_sendv に集約してください。
  • force_flush フラグは、頻繁なポインタ更新によるオーバーヘッドを避けるための最適化に関連しています。
