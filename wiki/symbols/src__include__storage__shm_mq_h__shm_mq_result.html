shm_mq_result

定義

  • 種別: enum
  • 定義箇所: src/include/storage/shm_mq.h:36

目的

共有メモリメッセージキュー（shm_mq）に対する送受信操作の結果状態を表現し、呼び出し元に対して操作の成否や待機が必要か、あるいは接続が切断されたかを通知するために使用されます。

シグネチャまたは形状

typedef enum
{
    SHM_MQ_SUCCESS,
    SHM_MQ_WOULD_BLOCK,
    SHM_MQ_DETACHED,
} shm_mq_result;

入力と出力

  • 本シンボルは関数の戻り値として使用されます。
  • SHM_MQ_SUCCESS: メッセージの送受信が完了したことを示します。
  • SHM_MQ_WOULD_BLOCK: 操作が完了せず、後続の再試行が必要であることを示します。
  • SHM_MQ_DETACHED: 通信相手のプロセスがキューからデタッチしたことを示します。

前提条件と不変条件

  • shm_mq は単一リーダー・単一ライターの制約を持つため、本列挙型を返す関数は、特定のプロセスがリーダーまたはライターとして正しく紐付けられている環境で呼び出される必要があります。

アルゴリズムまたは振る舞い

  • shm_mq_send および shm_mq_receive 等の操作関数が、内部的なリングバッファの状態や同期ラッチの状況に基づき、この列挙値のいずれかを返します。
  • SHM_MQ_DETACHED が返された場合、通信相手が終了しているため、呼び出し元は速やかに処理を終了する等の対応が求められます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な言及）**:
  • src/backend/access/transam/parallel.c
  • src/backend/executor/tqueue.c
  • src/backend/libpq/pqmq.c
  • src/backend/replication/logical/applyparallelworker.c
  • **呼び出し先**:
  • 本シンボルは列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • shm_mq: 共有メモリ上のキュー本体。
  • shm_mq_handle: 各バックエンドが保持するプライベートな操作用ハンドル。

失敗モードと境界条件

  • SHM_MQ_WOULD_BLOCK: キューが満杯（送信時）または空（受信時）の場合に発生します。
  • SHM_MQ_DETACHED: 相手プロセスが shm_mq_detach を実行した、あるいは異常終了した際に発生します。

保守担当者向けメモ

  • SHM_MQ_DETACHED を受け取った際の挙動は、各実装箇所で一貫した終了処理が行われているか注意してください。
  • 本列挙型は shm_mq_send_bytes および shm_mq_receive_bytes の戻り値として内部的に利用されています。
