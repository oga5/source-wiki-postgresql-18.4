MorphOpaque

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/to_tsany.c:25

目的

全文検索クエリの解析時において、設定情報（Config OID）および単語間の演算子情報を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct MorphOpaque
{
    Oid         cfg_id;
    int         qoperator;
} MorphOpaque;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。cfg_id には全文検索設定の OID が、qoperator には単語間を接続する演算子（通常は OP_PHRASE）が格納されます。

前提条件と不変条件

  • qoperator は、複数の単語に解析されたクエリにおいて、それらが隣接する位置にある場合に適用されます。
  • OP_PHRASE が指定された場合、複雑な形態素の単語位置が tsvector と正確に一致する必要があります。

アルゴリズムまたは振る舞い

クエリ解析プロセスにおいて、設定された辞書やパーサーを介して単語を正規化する際、この構造体を通じて解析コンテキストを維持します。特に、単語が複数の形態素に分割された場合の結合ロジックを制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/to_tsany.c 内の各関数（to_tsquery 関連の処理など）において、ローカル変数としてインスタンス化または Datum からのポインタキャストを通じて参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TSVectorBuildState: tsvector 構築時の状態管理に使用される関連構造体。

失敗モードと境界条件

  • Datum からのキャスト時に不正なポインタを参照した場合、メモリ保護違反が発生する可能性があります。
  • cfg_id が無効な OID である場合、後続の辞書検索や解析処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は tsvector のバイナリレイアウト操作と密接に関連しています。変更を加える際は、to_tsany.c 内のメモリ配置やアライメント（SHORTALIGN 等）への影響を考慮してください。
  • qoperator の値がどのようにクエリツリーの構築に影響するかについては、parse_tsquery および pushval_morph の実装を参照してください。
