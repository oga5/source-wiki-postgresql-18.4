appendStringInfoString

定義

  • 種別: function
  • 定義箇所: src/common/stringinfo.c:229

目的

既存の StringInfo バッファの末尾に、ヌル終端された C 文字列を安全に追加する。

シグネチャまたは形状

void appendStringInfoString(StringInfo str, const char *s)

入力と出力

  • str: 操作対象の StringInfoData 構造体へのポインタ。
  • s: 追加するヌル終端文字列。
  • 出力: なし（str が指すバッファの内容が更新される）。

前提条件と不変条件

  • str は初期化済みであること。
  • str が読み取り専用モード（maxlen == 0）でないこと。
  • s は有効なヌル終端文字列であること。

アルゴリズムまたは振る舞い

1. strlen(s) を用いて追加する文字列の長さを算出する。 2. appendBinaryStringInfo を呼び出し、算出した長さ分のデータを str のバッファ末尾にコピーする。 3. appendBinaryStringInfo の内部ロジックにより、必要に応じてバッファの自動拡張（倍増戦略）とヌル終端の維持が行われる。

呼び出し元と呼び出し先

  • 呼び出し先: appendBinaryStringInfo
  • 呼び出し元（直接的な証拠）:
  • contrib/cube/cube.c
  • contrib/dblink/dblink.c

関連データ構造

  • StringInfoData: バッファの長さ、容量、データポインタを保持する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc/repalloc が失敗した場合、PostgreSQL のエラーハンドリング機構によりアボートされる。
  • 読み取り専用バッファ: maxlen == 0 のバッファに対して呼び出した場合、内部の Assert により停止する可能性がある。

保守担当者向けメモ

  • 本関数は appendBinaryStringInfo のラッパーであり、文字列の長さを自動的に計算する利便性を提供する。
  • 頻繁に呼び出す場合は、文字列の長さを事前に把握しているなら appendBinaryStringInfo を直接使用することで strlen の呼び出しコストを削減できる。
