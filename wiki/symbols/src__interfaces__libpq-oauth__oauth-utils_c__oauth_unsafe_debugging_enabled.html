oauth_unsafe_debugging_enabled

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq-oauth/oauth-utils.c:148

目的

環境変数 PGOAUTHDEBUG の値に基づき、OAuth 関連のデバッグ出力を有効にするべきかどうかを判定します。

シグネチャまたは形状

bool oauth_unsafe_debugging_enabled(void);

入力と出力

  • **入力**: なし（環境変数 PGOAUTHDEBUG を直接参照）
  • **出力**: bool 型。環境変数が "UNSAFE" と一致する場合に true を返し、それ以外（未設定または他の値）の場合は false を返します。

前提条件と不変条件

  • 環境変数の取得には標準ライブラリの getenv を使用します。
  • 戻り値は、呼び出し時点の環境変数の状態に依存します。

アルゴリズムまたは振る舞い

1. getenv("PGOAUTHDEBUG") を呼び出し、環境変数の値を取得します。 2. 取得した値が NULL でなく、かつ文字列 "UNSAFE" と完全に一致するかを strcmp で比較します。 3. 一致した場合は true を、一致しない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq-oauth/oauth-curl.c:2797
  • src/interfaces/libpq/fe-auth-oauth.c:386, 880, 894
  • **呼び出し先**:
  • getenv (標準ライブラリ)
  • strcmp (標準ライブラリ)

関連データ構造

  • なし

失敗モードと境界条件

  • getenv が NULL を返した場合（環境変数が未設定の場合）、env && ... の評価により安全に false を返します。
  • 環境変数の値が "UNSAFE" 以外の文字列である場合、デバッグ機能は無効となります。

保守担当者向けメモ

  • 本関数は「unsafe」という名称の通り、デバッグ目的で機密情報がログに出力される可能性があるため、本番環境での使用には注意が必要です。
  • libpq 側（fe-auth-oauth.c）と libpq-oauth モジュール側の双方から参照されています。変更時は両方のコンテキストへの影響を確認してください。
