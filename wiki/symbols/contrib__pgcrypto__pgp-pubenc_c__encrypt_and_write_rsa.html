encrypt_and_write_rsa

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubenc.c:163

目的

RSA公開鍵を用いてセッション鍵を暗号化し、その結果をPGPパケットストリームへ書き出す。

シグネチャまたは形状

static int encrypt_and_write_rsa(PGP_Context *ctx, PGP_PubKey *pk, PushFilter *pkt);

入力と出力

  • ctx: PGPコンテキスト。
  • pk: 暗号化に使用する公開鍵構造体。
  • pkt: 暗号化データを書き出す出力フィルタ。
  • 戻り値: 成功時は 0、失敗時は負のエラーコードを返す。

前提条件と不変条件

  • pk はRSA公開鍵として初期化されている必要がある。
  • 処理の過程で生成される一時的な平文データおよびパディング済みデータは、関数終了時に必ずメモリから解放・ゼロクリアされる。

アルゴリズムまたは振る舞い

1. create_secmsg を呼び出し、セッション鍵とチェックサムを結合してパディングを施したメッセージ（MPI）を作成する。 2. pgp_rsa_encrypt を呼び出し、公開鍵 pk を用いてメッセージを暗号化する。 3. pgp_mpi_write を呼び出し、暗号化されたMPIを pkt ストリームへ書き出す。 4. 成功・失敗に関わらず、生成したMPI（m および c）を pgp_mpi_free で解放する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-pubenc.c 内の pgp_write_pubenc_sesskey（switch 文のRSAケースから呼び出される）。
  • 呼び出し先: create_secmsg, pgp_rsa_encrypt, pgp_mpi_write, pgp_mpi_free。

関連データ構造

  • PGP_Context: PGP処理の全体状態を保持。
  • PGP_PubKey: 公開鍵情報（RSA鍵のモジュラス等を含む）。
  • PGP_MPI: 多倍長整数（暗号化対象および結果）。
  • PushFilter: 出力ストリームの抽象化。

失敗モードと境界条件

  • create_secmsg または pgp_rsa_encrypt が負の値を返した場合、エラーとして処理を中断し、メモリを解放してその値を返す。
  • RSA鍵のサイズが不足している場合や、メモリ確保に失敗した場合にエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はRSA専用の暗号化フローを実装している。新しい公開鍵アルゴリズムを追加する際は、同様の構造を持つ関数を別途定義し、pgp_write_pubenc_sesskey からのディスパッチを追加すること。
  • pgp_mpi_free によるメモリ管理は、goto err を経由することで確実に実行される設計となっている。
