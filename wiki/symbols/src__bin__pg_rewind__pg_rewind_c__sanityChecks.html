sanityChecks

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/pg_rewind.c:734

目的

pg_rewind 実行前に、ソースおよびターゲットのクラスタが同期可能な状態にあるか、整合性と互換性を検証する。

シグネチャまたは形状

static void sanityChecks(void)

入力と出力

  • **入力**: グローバル変数 ControlFile_target、ControlFile_source、および datadir_source を参照。
  • **出力**: なし。検証に失敗した場合は pg_fatal を呼び出し、プログラムを異常終了させる。

前提条件と不変条件

  • ControlFile_target および ControlFile_source が読み込み済みであること。
  • ターゲットクラスタが実行中ではないこと（停止状態であること）。

アルゴリズムまたは振る舞い

1. **システム識別子の照合**: system_identifier が一致することを確認。 2. **バージョン互換性の確認**: 両クラスタの pg_control_version および catalog_version_no が現在の pg_rewind と一致することを確認。 3. **データ保護設定の検証**: ターゲットクラスタでデータチェックサムが有効か、または wal_log_hints が有効であることを確認。 4. **停止状態の確認**:

  • ターゲットクラスタが DB_SHUTDOWNED または DB_SHUTDOWNED_IN_RECOVERY であること。
  • datadir_source が指定されている場合、ソースクラスタも同様に停止状態であることを確認。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数 (src/bin/pg_rewind/pg_rewind.c:350) から呼び出される。
  • **呼び出し先**: pg_fatal（エラー発生時）。

関連データ構造

  • ControlFile_target, ControlFile_source: catalog/pg_control.h で定義される制御ファイル構造体。

失敗モードと境界条件

  • **システム不一致**: 異なるクラスタ間での実行を拒否。
  • **バージョン不整合**: 互換性のない PostgreSQL バージョン間での実行を拒否。
  • **設定不備**: チェックサムや wal_log_hints が無効な場合、データ破損のリスクがあるため拒否。
  • **稼働中**: ターゲット（およびソース）がクリーンに停止していない場合、安全のため実行を拒否。

保守担当者向けメモ

  • ソースが稼働中であっても技術的には同期可能な場合があるが、現在は安全性を優先して停止を強制している。
