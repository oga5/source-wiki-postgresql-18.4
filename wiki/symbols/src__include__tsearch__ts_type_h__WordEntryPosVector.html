WordEntryPosVector

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_type.h:65

目的

tsvector 内の各語彙に関連付けられた位置情報（WordEntryPos）の配列を保持するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct
{
	uint16		npos;
	WordEntryPos pos[FLEXIBLE_ARRAY_MEMBER];
} WordEntryPosVector;

入力と出力

  • **入力**: npos（位置情報の個数）、および pos 配列（可変長）。
  • **出力**: なし（データ構造体）。

前提条件と不変条件

  • pos メンバは FLEXIBLE_ARRAY_MEMBER として定義されており、構造体の末尾に可変長のデータが続くことを前提としています。
  • npos は pos 配列に格納されている WordEntryPos 要素の数を表します。

アルゴリズムまたは振る舞い

  • tsvector 内の特定の語彙に対する位置情報の集合を表現します。
  • メモリレイアウト上、npos の直後に WordEntryPos 型の配列が連続して配置されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/utils/adt/tsrank.c: ランキング計算処理において、位置情報の配列として参照。
  • src/backend/utils/adt/tsvector_op.c: tsvector の操作や変換処理において、位置情報ベクトルのポインタとして参照。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • WordEntryPos: pos 配列の各要素の型。
  • TSVectorData: tsvector の全体構造を定義する構造体。
  • WordEntryPosVector1: WordEntryPosVector の特殊ケース（要素数が1つ）として定義されています。

失敗モードと境界条件

  • 可変長配列を使用しているため、構造体のサイズ計算には sizeof(WordEntryPosVector) だけでなく、npos に基づいた動的な計算が必要です。
  • メモリ確保時に npos の値が不正な場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、この構造体を含むメモリ領域を確保する際は、offsetof(WordEntryPosVector, pos) + npos * sizeof(WordEntryPos) のような計算式を用いて必要なサイズを算出してください。
  • tsrank.c や tsvector_op.c でのポインタ操作が主な利用例であるため、変更を加える際はこれらの実装におけるメモリレイアウトの前提を破壊しないよう注意してください。
