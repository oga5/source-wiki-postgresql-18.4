list_delete_last

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:956

目的

リストの末尾要素を削除し、更新されたリストを返します。

シグネチャまたは形状

List *list_delete_last(List *list)

入力と出力

  • **入力**: list (操作対象の List 構造体へのポインタ)
  • **出力**: 末尾要素が削除された後の List 構造体へのポインタ。リストが空になった場合は NIL を返します。

前提条件と不変条件

  • 操作の開始時に check_list_invariants(list) が呼び出され、リストの整合性が検証されます。

アルゴリズムまたは振る舞い

1. リストが NIL の場合はそのまま NIL を返します。 2. リストの長さが 1 以下の場合、list_free(list) を呼び出してメモリを解放し、NIL を返します。 3. リストの長さが 2 以上の場合、list_truncate(list, list_length(list) - 1) を呼び出し、末尾を除いたリストを返します。

関連データ構造

  • List: PostgreSQLの配列ベースの動的リスト構造体。

失敗モードと境界条件

  • 入力が NIL の場合、エラーにはならず NIL を返します。
  • 要素数が 1 のリストに対して実行した場合、リスト自体が解放され NIL が返されます。

保守担当者向けメモ

  • 本関数はリストの末尾を削除する際、リストが空になるケースを明示的に処理し、メモリ解放を行っています。
  • list_truncate はリストが空になる場合にメモリを解放しない仕様であるため、本関数で list_free を呼び出すことでメモリリークを防いでいます。
