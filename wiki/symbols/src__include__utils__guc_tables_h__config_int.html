config_int

定義

  • 種別: struct
  • 定義箇所: src/include/utils/guc_tables.h:230

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおいて、整数型の設定変数を定義および管理するための構造体です。

シグネチャまたは形状

struct config_int
{
    struct config_generic gen;
    int            *variable;
    int             boot_val;
    int             min;
    int             max;
    GucIntCheckHook check_hook;
    GucIntAssignHook assign_hook;
    GucShowHook     show_hook;
    int             reset_val;
    void           *reset_extra;
};

入力と出力

  • **入力**: config_generic を継承した共通メタデータ、および整数値の範囲（min, max）とデフォルト値（boot_val）。
  • **出力**: 設定値の検証・割り当て・表示を行うためのフック関数（check_hook, assign_hook, show_hook）を保持します。

前提条件と不変条件

  • gen フィールドは構造体の先頭に配置され、config_generic とのキャスト互換性を持ちます。
  • variable は、設定値が格納されるメモリ上の実体（整数型変数）を指す必要があります。
  • min および max は、設定値の有効範囲を規定します。

アルゴリズムまたは振る舞い

  • GUCシステムは、本構造体を通じて整数型パラメータの値を管理します。
  • check_hook は値の変更前に検証を行い、assign_hook は値の適用時に実行されます。
  • reset_val および reset_extra は、トランザクション終了時や設定のリセット時に元の状態へ復元するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc.c 内の各関数（call_int_check_hook 等）が、本構造体を参照して値の検証や更新を行います。
  • **呼び出し先**: check_hook, assign_hook, show_hook に登録された関数が実行時に呼び出されます。

関連データ構造

  • config_generic: 全てのGUC変数の基底構造体。
  • GucIntCheckHook, GucIntAssignHook, GucShowHook: 整数型変数専用のフック関数型。

失敗モードと境界条件

  • min および max の範囲外の値が設定された場合、check_hook による検証でエラーが発生する可能性があります。
  • variable ポインタが正しく初期化されていない場合、メモリ破壊や不正な値の参照を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は config_generic を継承しているため、gconf 型のポインタを struct config_int * にキャストして操作するコードが guc.c に多数存在します。
  • extra ポインタ（reset_extra）は、複雑な設定値の検証ロジックと適用ロジックの間で状態を共有するためのブリッジとして機能します。
