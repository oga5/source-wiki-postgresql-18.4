WalRcvExecStatus

定義

  • 種別: enum
  • 定義箇所: src/include/replication/walreceiver.h:202

目的

WALレシーバープロセスがプライマリサーバーに対して実行したクエリやコマンドの結果状態を分類するために使用されます。

シグネチャまたは形状

typedef enum {
    WALRCV_ERROR,
    WALRCV_OK_COMMAND,
    WALRCV_OK_TUPLES,
    WALRCV_OK_COPY_IN,
    WALRCV_OK_COPY_OUT,
    WALRCV_OK_COPY_BOTH,
} WalRcvExecStatus;

入力と出力

  • 入力: クエリ実行の結果（エラー、コマンド完了、タプル返却、COPYモード開始など）。
  • 出力: 上記列挙型のいずれかの値。

前提条件と不変条件

  • WALRCV_ERROR はクエリ実行時にエラーが発生したことを示します。
  • WALRCV_OK_COPY_IN, WALRCV_OK_COPY_OUT, WALRCV_OK_COPY_BOTH は、レプリケーションプロトコルにおける COPY モードの開始状態を表現します。

アルゴリズムまたは振る舞い

  • クエリ実行の結果に応じて、後続の処理（タプルの取得やストリーミングデータの送受信など）を制御するためのステータスコードとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは、具体的な関数呼び出しの文脈は特定できません。
  • 呼び出し先: 本シンボルは列挙型であり、直接的な呼び出し先はありません。
  • 参照箇所: src/include/replication/walreceiver.h:220 にて WalRcvExecResult 構造体のメンバとして使用されています。

関連データ構造

  • WalRcvExecResult: 本列挙型をメンバ status として保持する構造体。

失敗モードと境界条件

  • WALRCV_ERROR が返された場合、クエリ実行が失敗したことを意味するため、呼び出し側でのエラーハンドリングが必要です。

保守担当者向けメモ

  • 本列挙型は、libpq 等の通信バックエンドが実行結果を報告する際のインターフェースの一部です。新しい通信プロトコルや実行モードを追加する際は、この列挙型に状態を追加する必要があるか検討してください。
