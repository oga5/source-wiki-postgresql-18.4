目的

ECPG（Embedded SQL for C）ライブラリのマルチスレッド環境におけるスレッドセーフ性を検証するための回帰テストプログラムのエントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（グローバル変数 nthreads および iterations に依存）。
  • 出力: 成功時に 0 を返し、標準出力に "Success." を出力。メモリ確保失敗時に 1 を返す。

前提条件と不変条件

  • 実行環境が POSIX スレッド（pthread）または Windows スレッドをサポートしていること。
  • データベース ecpg1_regression への接続が可能であること。
  • テスト終了時のテーブル test_thread の行数が nthreads * iterations と一致すること。

アルゴリズムまたは振る舞い

1. **環境初期化**: ECPGconnect を使用してデータベースに接続し、既存の test_thread テーブルを削除・再作成して切断します。 2. **スレッド生成**: nthreads 分のスレッドを生成し、各スレッドで test_thread 関数を実行します。 3. **並行実行**: 各スレッドは独立したデータベース接続を確立し、test_thread テーブルに対してデータを挿入します。 4. **同期**: 全スレッドの終了を待機（pthread_join または WaitForMultipleObjects）します。 5. **検証**: 再度データベースに接続し、SELECT COUNT(*) を実行して挿入された行数を取得します。 6. **結果判定**: 取得した行数が期待値と一致するかを確認し、結果を標準出力に表示して終了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS のプロセスローダー（エントリポイント）。
  • 呼び出し先:
  • ECPGconnect, ECPGdo, ECPGtrans, ECPGdisconnect（ECPGランタイム関数）
  • test_thread（スレッド実行関数）
  • pthread_create, pthread_join（POSIX環境）または CreateThread, WaitForMultipleObjects（Windows環境）
  • calloc, free, fprintf, printf（標準ライブラリ）

関連データ構造

  • threads: スレッドハンドルを保持する配列（pthread_t* または HANDLE*）。
  • l_rows: 挿入された行数を格納する整数変数。

失敗モードと境界条件

  • メモリ不足: calloc が失敗した場合、エラーメッセージを出力して 1 を返します。
  • データベース操作失敗: DROP TABLE や CONNECT 等の ECPG 処理が失敗した場合、テストは異常終了する可能性があります。
  • 行数不一致: 期待値と実際の行数が異なる場合、エラーメッセージを出力します。

保守担当者向けメモ

  • 本関数は .pgc ファイルから生成されたコードです。直接編集せず、元の .pgc ファイルを修正してください。
  • デバッグ出力（ECPGdebug）は、スレッドの実行順序が不定であるため、回帰テストの安定性の観点から無効化されています。
