pgtls_get_peer_certificate_hash

定義

  • src/interfaces/libpq/fe-secure-openssl.c:339

目的

TLS接続のピア証明書のハッシュ値を計算し、呼び出し元に返却する。

シグネチャまたは形状

char *pgtls_get_peer_certificate_hash(PGconn *conn, size_t *len);

入力と出力

  • **入力**:
  • conn: PGconn 構造体へのポインタ。
  • len: ハッシュ値のサイズを格納する出力用ポインタ。
  • **出力**:
  • 成功時: 計算されたハッシュ値を含む動的に確保されたメモリ領域（呼び出し元で解放が必要）。
  • 失敗時: NULL を返し、conn->errorMessage にエラー詳細を記録する。

前提条件と不変条件

  • conn->peer に有効なピア証明書（X509）が保持されていること。
  • 失敗時は *len が 0 に設定される。

アルゴリズムまたは振る舞い

1. conn->peer が存在しない場合は NULL を返す。 2. 証明書の署名アルゴリズム（NID）を取得する（OpenSSLのバージョンに応じて X509_get_signature_info または OBJ_find_sigid_algs を使用）。 3. 署名アルゴリズムが MD5 または SHA-1 の場合は、RFC 5929 に従い SHA-256 をハッシュアルゴリズムとして選択する。それ以外の場合は、署名アルゴリズムに対応するダイジェストアルゴリズムを使用する。 4. X509_digest を用いて証明書のハッシュを生成する。 5. 生成されたハッシュを malloc で確保したメモリにコピーし、そのサイズを *len に格納して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-scram.c (確認済み)
  • **呼び出し先**: OpenSSL API (X509_get_signature_info, OBJ_find_sigid_algs, EVP_sha256, EVP_get_digestbynid, X509_digest, OBJ_nid2sn) および libpq 内部関数 (libpq_append_conn_error)。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。
  • X509: OpenSSL の証明書構造体。

失敗モードと境界条件

  • conn->peer が未設定の場合: NULL を返す。
  • 署名アルゴリズムの特定に失敗した場合: エラーを記録し NULL を返す。
  • ダイジェストアルゴリズムの取得に失敗した場合: エラーを記録し NULL を返す。
  • ハッシュ生成に失敗した場合: エラーを記録し NULL を返す。
  • メモリ確保 (malloc) に失敗した場合: エラーを記録し NULL を返す。

保守担当者向けメモ

  • 本関数は RFC 5929 に準拠したハッシュ計算を行う。
  • malloc で確保されたメモリは呼び出し元で解放する必要がある。
  • OpenSSL 1.1.1 未満の環境とそれ以降の環境で、署名アルゴリズムの取得方法が条件分岐されている点に注意すること。
