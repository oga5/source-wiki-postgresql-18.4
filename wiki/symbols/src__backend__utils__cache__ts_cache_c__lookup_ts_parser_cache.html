lookup_ts_parser_cache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/ts_cache.c:112

目的

全文検索パーサーの定義をシステムカタログから取得し、メモリ上のハッシュテーブルにキャッシュすることで、以降のアクセスを高速化します。

シグネチャまたは形状

TSParserCacheEntry *lookup_ts_parser_cache(Oid prsId)

入力と出力

  • **入力**: prsId (Oid) - 検索対象のパーサーのOID。
  • **出力**: TSParserCacheEntry* - キャッシュされたパーサー定義情報へのポインタ。

前提条件と不変条件

  • 初回呼び出し時に TSParserCacheHash ハッシュテーブルおよび CacheMemoryContext が初期化されます。
  • キャッシュエントリは isvalid フラグを持ち、無効化コールバックにより整合性が維持されます。
  • 辞書やパーサーのメソッド（prsstart, prstoken, prsend）のOIDは有効である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: TSParserCacheHash が未作成の場合、作成しシステムカタログの無効化コールバックを登録します。 2. **高速ルックアップ**: 直近で使用した lastUsedParser と一致し、かつ有効であれば即座に返します。 3. **ハッシュ検索**: TSParserCacheHash を検索し、存在しないか無効であればシステムカタログ (pg_ts_parser) を SearchSysCache1 で検索します。 4. **検証と構築**: カタログから取得したメソッドOIDが有効か確認し、fmgr_info_cxt を用いて関数情報をキャッシュエントリに格納します。 5. **更新**: 最後に使用したエントリとして lastUsedParser を更新し、エントリを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tsearchcmds.c, src/backend/tsearch/ts_parse.c, src/backend/tsearch/wparser.c 等の全文検索関連モジュール。
  • **呼び出し先**: hash_create, CacheRegisterSyscacheCallback, CreateCacheMemoryContext, hash_search, SearchSysCache1, fmgr_info_cxt 等。

関連データ構造

  • TSParserCacheEntry: パーサーのメソッド情報やOIDを保持する構造体。
  • TSParserCacheHash: パーサーキャッシュを管理するハッシュテーブル。
  • lastUsedParser: 単一エントリの高速キャッシュ用ポインタ。

失敗モードと境界条件

  • 指定された prsId がシステムカタログに存在しない場合、elog(ERROR) で停止します。
  • パーサーに必要なメソッド（prsstart, prstoken, prsend）のOIDが不正な場合、elog(ERROR) で停止します。

保守担当者向けメモ

  • キャッシュエントリはバックエンドの生存期間中保持されます。
  • InvalidateTSCacheCallBack により、カタログ更新時に isvalid が false に設定され、次回のルックアップ時に再構築が強制されます。
