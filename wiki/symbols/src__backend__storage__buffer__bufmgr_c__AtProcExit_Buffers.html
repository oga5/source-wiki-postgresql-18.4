AtProcExit_Buffers

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/bufmgr.c:4041

目的

バックエンドプロセスの終了時に、共有バッファに関連するリソースのクリーンアップと整合性チェックを行う。

シグネチャまたは形状

static void AtProcExit_Buffers(int code, Datum arg)

入力と出力

  • code: プロセス終了時の終了コード。
  • arg: on_shmem_exit 登録時に渡される引数（未使用）。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • LWLock（軽量ロック）へのアクセスが可能である必要がある。
  • プロセス終了処理（on_shmem_exit）のコールバックとして呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. UnlockBuffers() を呼び出し、現在のプロセスが保持している可能性のあるバッファロックをすべて解放する。 2. CheckForBufferLeaks() を呼び出し、ピン留めされたままのバッファがないか確認する。 3. AtProcExit_LocalBuffers() を呼び出し、ローカルバッファの終了処理を実行する。

呼び出し元と呼び出し先

  • 呼び出し元: on_shmem_exit(AtProcExit_Buffers, 0) を通じて、共有メモリ終了処理の登録時に呼び出される。
  • 呼び出し先:
  • UnlockBuffers()
  • CheckForBufferLeaks()
  • AtProcExit_LocalBuffers()

関連データ構造

  • 特になし（バッファマネージャの内部状態およびローカルバッファ管理構造体を参照）。

失敗モードと境界条件

  • CheckForBufferLeaks() がバッファリークを検出した場合、PostgreSQL のエラーハンドリングに従い、警告やエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は on_shmem_exit に登録されるため、プロセス終了時のクリーンアップ順序において重要な役割を果たす。
  • localbuf.c 側の終了処理（AtProcExit_LocalBuffers）と連携しているため、ローカルバッファの仕様変更時には本関数の呼び出し順序や内容に注意が必要である。
