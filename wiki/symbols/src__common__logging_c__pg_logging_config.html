pg_logging_config

定義

  • 種別: function
  • 定義箇所: src/common/logging.c:165

目的

ログ出力フレームワークの動作フラグ（log_flags）を更新し、プログラム名や位置情報の出力制御を動的に変更します。

シグネチャまたは形状

void pg_logging_config(int new_flags)

入力と出力

  • **入力**: new_flags (int型) - 設定するログフラグのビットマスク。
  • **出力**: なし（void型）。

前提条件と不変条件

  • 本関数はフロントエンドプログラムでの使用を前提としています。
  • 設定された log_flags は、後続の pg_log_generic 等によるログ出力時に参照されます。

アルゴリズムまたは振る舞い

1. 引数として受け取った new_flags を、モジュール内の静的変数 log_flags に代入します。 2. これにより、以降のログ出力において、プログラム名やファイル位置情報の付与ルールが変更されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c
  • src/bin/psql/startup.c
  • **呼び出し先**:
  • 本関数は内部で他の関数を呼び出しません。

関連データ構造

  • log_flags: src/common/logging.c 内で定義されている静的変数。ログ出力の挙動を制御するビットフラグを保持します。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。引数として渡された整数値がそのままフラグとして適用されます。

保守担当者向けメモ

  • 本関数は pg_log_generic 等が参照するグローバルな設定状態を書き換えます。マルチスレッド環境での安全性については、PostgreSQLのフロントエンドライブラリの設計方針に従ってください。
