pgoutput_startup

定義

  • 種別: function
  • 定義箇所: src/backend/replication/pgoutput/pgoutput.c:448

目的

論理レプリケーション出力プラグインの初期化を行い、クライアントからのオプション解析、プロトコルバージョンの検証、およびメモリコンテキストとキャッシュのセットアップを行う。

シグネチャまたは形状

static void pgoutput_startup(LogicalDecodingContext *ctx, OutputPluginOptions *opt, bool is_init)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • opt: 出力プラグインのオプション設定。
  • is_init: スロット初期化中かどうかを示すフラグ。
  • **出力**: なし（ctx->output_plugin_private に PGOutputData を格納し、opt->output_type を設定）。

前提条件と不変条件

  • is_init が false の場合、クライアントから渡されたオプション（ctx->output_plugin_options）が有効であること。
  • PGOutputData 用のメモリコンテキストが正常に作成されること。

アルゴリズムまたは振る舞い

1. **メモリ管理**: PGOutputData 構造体を割り当て、専用のメモリコンテキスト（context, cachectx, pubctx）を作成する。 2. **クリーンアップ登録**: pgoutput_memory_context_reset をリセットコールバックとして登録する。 3. **プロトコル設定**: opt->output_type を OUTPUT_PLUGIN_BINARY_OUTPUT に設定する。 4. **初期化モード判定**:

  • is_init が true の場合、ストリーミングと二相コミットを無効化する。
  • is_init が false の場合:
  • parse_output_parameters でパラメータを解析。
  • プロトコルバージョンがサポート範囲内か検証し、範囲外であればエラーを投げる。
  • ストリーミングおよび二相コミットの要求を検証し、サポート外であればエラーを投げる。
  • パブリケーションのシステムカタログ変更を監視するためのコールバック（publication_invalidation_cb, rel_sync_cache_relation_cb）を登録する。
  • リレーションスキーマキャッシュを初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: _PG_output_plugin_init でコールバックとして登録され、論理デコーディング開始時に呼び出される。
  • **呼び出し先**: parse_output_parameters, AllocSetContextCreate, MemoryContextRegisterResetCallback, CacheRegisterSyscacheCallback, CacheRegisterRelSyncCallback, init_rel_sync_cache。

関連データ構造

  • PGOutputData: プラグインのセッション状態を保持。
  • LogicalDecodingContext: 論理デコーディングの基盤コンテキスト。
  • OutputPluginOptions: プラグインの動作オプション。

失敗モードと境界条件

  • **プロトコル不一致**: クライアントが要求したバージョンがサーバーのサポート範囲外の場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー。
  • **機能サポート不足**: ストリーミングや二相コミットが要求されたが、プロトコルバージョンが不足している、またはプラグインがサポートしていない場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー。

保守担当者向けメモ

  • publication_callback_registered は静的変数であり、プロセス内で一度だけコールバック登録を行うためのガードとして機能している。
  • is_init モードと通常モードで設定されるフラグが異なるため、変更を加える際は両方のパスを確認すること。
