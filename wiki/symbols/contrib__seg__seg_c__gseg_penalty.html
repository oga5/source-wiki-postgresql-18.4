gseg_penalty

定義

  • 種別: function
  • 定義箇所: contrib/seg/seg.c:273

目的

GiST インデックスにおいて、新しい区間（SEG）を既存のインデックスノードに追加する際の「ペナルティ」を計算します。この値は、新しい区間を挿入したことによってインデックスの範囲がどれだけ拡大するかを表します。

シグネチャまたは形状

Datum gseg_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *origentry（既存のインデックスエントリ）
  • PG_GETARG_POINTER(1): GISTENTRY *newentry（追加対象の新しいエントリ）
  • PG_GETARG_POINTER(2): float *result（計算結果を格納するポインタ）
  • **出力**:
  • PG_RETURN_POINTER(result): 計算されたペナルティ値（float）を格納したポインタを返します。

前提条件と不変条件

  • origentry および newentry の key メンバには、有効な SEG データが含まれている必要があります。
  • result は呼び出し元によって確保されたメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. seg_union 関数を呼び出し、既存の区間と新しい区間の和集合（包含する最小の区間）を計算します。 2. rt_seg_size を使用して、和集合のサイズ（tmp1）と、元の区間のサイズ（tmp2）をそれぞれ算出します。 3. *result = tmp1 - tmp2 により、挿入によるサイズの増加分をペナルティとして算出します。 4. GIST_DEBUG マクロが定義されている場合、計算結果を stderr に出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックスマネージャから呼び出されます（直接的な呼び出し元コードはソース内に明示されていません）。
  • **呼び出し先**:
  • seg_union: 和集合の計算に使用。
  • rt_seg_size: 区間のサイズ計算に使用。
  • DatumGetSegP: Datum から SEG 構造体への変換に使用。

関連データ構造

  • SEG: 区間データを保持する構造体。
  • GISTENTRY: GiST インデックスのノードエントリを保持する構造体。

失敗モードと境界条件

  • seg_union や rt_seg_size 内部で不正な SEG データが渡された場合、予期せぬ動作やクラッシュが発生する可能性があります。
  • 浮動小数点の計算精度に依存するため、極端に大きな値や小さな値の差分計算において精度上の制約を受ける可能性があります。

保守担当者向けメモ

  • 本関数は GiST インデックスの penalty メソッドとして登録されています。
  • GIST_DEBUG を有効にするとデバッグ出力が行われるため、インデックスの挙動調査に活用可能です。
