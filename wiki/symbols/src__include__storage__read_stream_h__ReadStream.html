ReadStream

定義

  • 種別: struct
  • 定義箇所: src/include/storage/read_stream.h:67

目的

リレーションデータの読み込みにおいて、将来必要となるブロックを予測して先読み（Look-ahead）を行うための不透明な状態保持構造体です。I/O待ちによるストールを最小限に抑え、バッチ処理や非同期I/O（AIO）を活用して効率的なデータアクセスを実現します。

シグネチャまたは形状

typedef struct ReadStream ReadStream;

※不透明な構造体として定義されており、内部メンバは公開されていません。

入力と出力

  • 入力: read_stream_begin_relation を介して初期化され、コールバック関数やフラグ（READ_STREAM_MAINTENANCE 等）を受け取ります。
  • 出力: read_stream_next_buffer を通じて、要求されたブロックのバッファが呼び出し元に返却されます。

前提条件と不変条件

  • 呼び出し元は、read_stream_begin_relation で指定したコールバック関数を通じて、次に読み込むべきブロック番号を正確に提供する必要があります。
  • READ_STREAM_USE_BATCHING フラグを使用する場合、コールバック関数内でのブロッキング操作は厳しく制限されます。

アルゴリズムまたは振る舞い

1. **初期化**: ストリームを開始し、先読み戦略を確立します。 2. **先読み制御**: キャッシュヒット率やI/O履歴に基づき、先読み距離を動的に調整します。 3. **I/O発行**: 連続するブロックの読み込み要求をまとめ、StartReadBuffers() 等を用いて効率的にI/Oを発行します。 4. **バッファ取得**: 必要なI/Oが完了するまで待機し、バッファを透過的に提供します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • contrib/pg_prewarm/autoprewarm.c
  • contrib/pg_visibility/pg_visibility.c
  • src/backend/access/gist/gistvacuum.c
  • src/backend/access/heap/heapam.c
  • **呼び出し先**:
  • storage/bufmgr.h（バッファ管理）
  • storage/smgr.h（ストレージ管理）
  • src/backend/storage/aio/read_stream.c（実装本体）

関連データ構造

  • ReadStreamBlockNumberCB: 次のブロック番号を決定するコールバック関数型。
  • BlockRangeReadStreamPrivate: 関連する内部構造体。

失敗モードと境界条件

  • 供給されたコールバック関数が不正なブロック番号を返した場合の挙動は、呼び出し元の実装に依存します。
  • 非同期I/Oのバッチ処理モードでは、コールバック内の制約違反がシステム全体のストールを招く可能性があります。

保守担当者向けメモ

  • 本構造体は不透明型であるため、内部メンバへの直接アクセスは避け、提供されているAPI（read_stream_next_buffer 等）を介して操作してください。
  • 先読みアルゴリズムは自動適応型であるため、呼び出し側での複雑なチューニングは不要です。
