gbt_cashle

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_cash.c:44

目的

PostgreSQLの money 型の値に対し、GiST インデックス内での「以下（less than or equal to）」比較判定を提供します。

シグネチャまたは形状

bool gbt_cashle(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**: 比較対象となる2つの Cash 型へのポインタ a および b、および関数マネージャ情報 flinfo。
  • **出力**: a <= b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な Cash 型の値を指している必要があります。
  • flinfo は使用されていませんが、関数シグネチャの互換性のために保持されています。

アルゴリズムまたは振る舞い

1. 引数 a および b を const Cash * 型にキャストします。 2. C言語の標準的な比較演算子 <= を用いて、2つの money 値の大小関係を評価します。 3. 評価結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_cash.c 内の tinfo 構造体（gbtree_ninfo）の比較関数として登録されており、GiST インデックスの検索処理（gbt_cash_consistent 経由）から間接的に呼び出されます。
  • **呼び出し先**: 直接的な関数呼び出しはありません（組み込みの <= 演算子を使用）。

関連データ構造

  • Cash: PostgreSQLの money 型を表現する内部データ構造。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。
  • money 型の内部表現（64bit整数）に基づく比較を行うため、値の範囲外による境界条件は money 型自体の仕様に依存します。

保守担当者向けメモ

  • 本関数は btree_utils_num.h で定義された数値型 GiST インデックスの汎用ロジックから利用されることを想定しています。
  • flinfo 引数は現在の実装では未使用ですが、将来的な拡張やインターフェースの整合性のためにシグネチャに含まれています。
