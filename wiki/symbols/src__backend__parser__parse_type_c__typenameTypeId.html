typenameTypeId

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_type.c:290

目的

TypeName 構造体で指定された型名から、システムカタログ上の型識別子（Oid）を直接取得します。

シグネチャまたは形状

Oid typenameTypeId(ParseState *pstate, const TypeName *typeName)

入力と出力

  • **入力**:
  • pstate: 構文解析状態（エラー位置報告に使用）。
  • typeName: 解決対象の型名情報。
  • **出力**:
  • 解決された型の Oid。

前提条件と不変条件

  • typeName は有効な TypeName 構造体である必要があります。

アルゴリズムまたは振る舞い

1. typenameType(pstate, typeName, NULL) を呼び出し、対象の型に対応する Type（HeapTuple）を取得します。 2. 取得したタプルから pg_type システムカタログの oid フィールドを抽出します。 3. ReleaseSysCache を呼び出し、取得したタプルの参照カウントを解放します。 4. 抽出した Oid を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • typenameType: 型の検索と検証を行う。
  • ReleaseSysCache: キャッシュエントリの解放を行う。
  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/aggregatecmds.c
  • src/backend/commands/functioncmds.c
  • src/backend/commands/opclasscmds.c

    • (各所にて、DDLコマンド実行時の型解決に使用されています)

関連データ構造

  • TypeName: 構文解析中の型指定を保持する構造体。
  • Oid: PostgreSQLのオブジェクト識別子。
  • Form_pg_type: pg_type システムカタログの行構造。

失敗モードと境界条件

  • 指定された型名が見つからない場合、または型が未定義（シェル型）である場合、typenameType 内でエラーが発生し、処理が中断されます。

保守担当者向けメモ

  • 本関数は typenameType のラッパーであり、型修飾子（typmod）を必要としない場合に利用します。
  • 取得した Type タプルは関数内で確実に解放されるため、呼び出し側で解放処理を行う必要はありません。
