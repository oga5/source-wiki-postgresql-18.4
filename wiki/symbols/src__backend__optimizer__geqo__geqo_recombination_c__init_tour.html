init_tour

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_recombination.c:33

目的

遺伝的クエリ最適化（GEQO）において、指定された遺伝子数（num_gene）に基づき、1から num_gene までの値を含むランダムな順列（ツアー）を生成する。

シグネチャまたは形状

void init_tour(PlannerInfo *root, Gene *tour, int num_gene)

入力と出力

  • **入力**:
  • root: クエリ最適化のコンテキスト情報。
  • tour: 生成された順列を格納する Gene 型の配列（呼び出し元で確保済みであること）。
  • num_gene: 生成する順列の長さ（遺伝子数）。
  • **出力**: なし（tour 配列が直接書き換えられる）。

前提条件と不変条件

  • tour 配列は少なくとも num_gene 個の要素を保持できるサイズであること。
  • num_gene が 0 以下の場合、配列への書き込みは行われない。

アルゴリズムまたは振る舞い

Fisher-Yatesシャッフルアルゴリズムの「inside-out」バリアントを用いて、1回のパスで順列を生成する。 1. num_gene > 0 の場合、tour[0] を 1 で初期化する。 2. i = 1 から num_gene - 1 までループし、以下の処理を行う。

  • geqo_randint(root, i, 0) を呼び出し、乱数 j を取得する。
  • i != j の場合、既存の tour[j] の値を tour[i] にコピーする。
  • tour[j] に i + 1 を代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_pool.c（初期個体群生成時）
  • **呼び出し先**: geqo_randint（乱数生成関数）

関連データ構造

  • Gene: 遺伝子を表す型。
  • PlannerInfo: クエリ最適化の計画情報構造体。

失敗モードと境界条件

  • num_gene が 0 の場合、関数は何も行わずに終了する。
  • i != j のチェックにより、未初期化の配列要素へのアクセスを回避している。

保守担当者向けメモ

  • 本関数は Fisher-Yates シャッフルの効率的な実装例である。
  • geqo_randint の戻り値の範囲に依存するため、乱数生成器の仕様変更時には注意が必要である。
