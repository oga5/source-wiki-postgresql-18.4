set_rel_pathlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/allpaths.c:468

目的

指定されたリレーション（RelOptInfo）に対して、利用可能なすべての実行パス（Access Paths）を生成し、コスト最小のパスを特定する。

シグネチャまたは形状

static void set_rel_pathlist(PlannerInfo *root, RelOptInfo *rel, Index rti, RangeTblEntry *rte)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: パスを生成する対象のリレーション情報。
  • rti: リレーションの範囲テーブルインデックス。
  • rte: リレーションの範囲テーブルエントリ。
  • **出力**: なし（副作用として rel->pathlist 等が更新される）。

前提条件と不変条件

  • IS_DUMMY_REL(rel) が真の場合、リレーションは空とみなされ、パス生成はスキップされる。
  • RTE_SUBQUERY, RTE_CTE, RTE_NAMEDTUPLESTORE, RTE_RESULT の各リレーション種別は、本関数呼び出し前の set_rel_size 段階で処理が完了している必要がある。

アルゴリズムまたは振る舞い

1. **分岐処理**: リレーションの種類（継承、外部テーブル、関数、VALUES等）に応じて適切なパス生成関数を呼び出す。 2. **プラグイン介入**: set_rel_pathlist_hook が定義されている場合、これを実行してパスの追加・削除・編集を許可する。 3. **並列パスの生成**: ベースリレーションであり、かつクエリ全体の最上位リレーションでない場合、generate_useful_gather_paths を呼び出して並列実行用の Gather パスを生成する。 4. **コスト評価**: set_cheapest(rel) を呼び出し、生成されたパス群からコスト最小のものを特定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c 内の set_base_rel_pathlists および継承リレーション処理箇所。
  • **呼び出し先**:
  • set_append_rel_pathlist, set_foreign_pathlist, set_tablesample_rel_pathlist, set_plain_rel_pathlist, set_function_pathlist, set_tablefunc_pathlist, set_values_pathlist。
  • set_rel_pathlist_hook（プラグイン用フック）。
  • generate_useful_gather_paths。
  • set_cheapest。

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • RelOptInfo: パス生成対象のリレーション情報。
  • RangeTblEntry: 範囲テーブルエントリ。

失敗モードと境界条件

  • 未知の rtekind が渡された場合、elog(ERROR) により処理を中断する。
  • 継承の子リレーションについては、Gather パスの生成をスキップする（親リレーション側で集約するため）。

保守担当者向けメモ

  • 本関数はパス生成のディスパッチャーとして機能する。新しいリレーションタイプを追加する場合は、switch 文にケースを追加すること。
  • set_rel_pathlist_hook の呼び出し順序は、Gather パスの生成前である必要がある点に注意（フックが並列パスを追加できるようにするため）。
