get_partition_exprs

定義

  • 種別: function
  • 定義箇所: src/include/utils/partcache.h:70

目的

パーティションキー構造体から、パーティション定義に使用されている式（expression）のリストを取得する。

シグネチャまたは形状

static inline List *get_partition_exprs(PartitionKey key)

入力と出力

  • **入力**: PartitionKey key - パーティションキー情報を保持する構造体へのポインタ。
  • **出力**: List * - パーティションキーに関連付けられた式のリスト。

前提条件と不変条件

  • key は有効な PartitionKey 構造体を指している必要がある。
  • パーティションキーはリレーションを開いている間は不変であり、本関数が返すリストの内容も同様に保持される。

アルゴリズムまたは振る舞い

PartitionKey 構造体内の partexprs フィールドを直接参照し、そのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/partition.c:268
  • src/backend/parser/parse_utilcmd.c:4294
  • src/backend/parser/parse_utilcmd.c:4446
  • **呼び出し先**:
  • なし（インライン関数として定義されており、構造体フィールドへの直接アクセスのみを行う）。

関連データ構造

  • PartitionKey: パーティションキーのメタデータを保持する構造体。
  • List: PostgreSQL のリスト構造（nodes/pg_list.h で定義）。

失敗モードと境界条件

  • key が NULL の場合、本関数はセグメンテーション違反を引き起こす可能性がある。呼び出し側で key が有効であることを保証する必要がある。

保守担当者向けメモ

  • 本関数は PartitionKeyData 構造体へのアクセスをカプセル化するインライン関数である。将来的に PartitionKey の内部構造が変更された場合、この関数を介することで影響範囲を限定できる。
