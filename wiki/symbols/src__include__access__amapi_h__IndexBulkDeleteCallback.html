IndexBulkDeleteCallback

定義

  • 種別: typedef
  • 定義箇所: src/include/access/amapi.h:133

目的

インデックスのバルク削除（VACUUM）処理において、特定のインデックスエントリを削除すべきかどうかを判定するためのコールバック関数を定義します。

シグネチャまたは形状

typedef bool (*IndexBulkDeleteCallback) (ItemPointer itup_ctid, void *callback_state);

※シグネチャは ambulkdelete_function の引数定義および各アクセスメソッドの実装から推論される一般的な形状です。

入力と出力

  • **入力**:
  • ItemPointer itup_ctid: 削除対象を判定するインデックスエントリが指すヒープタプルの TID。
  • void *callback_state: 判定に必要な状態情報（VACUUMのコンテキストなど）を保持するポインタ。
  • **出力**:
  • bool: true の場合、該当するインデックスエントリを削除します。false の場合は保持します。

前提条件と不変条件

  • インデックスアクセスメソッドの ambulkdelete メソッド内で呼び出されることを前提とします。
  • callback_state は、呼び出し元で適切に初期化された構造体へのポインタである必要があります。

アルゴリズムまたは振る舞い

1. インデックスアクセスメソッドは、インデックスをスキャンする際に各エントリに対してこのコールバックを呼び出します。 2. コールバック関数は、渡された itup_ctid が削除対象のヒープタプルであるかを判定します。 3. 戻り値に基づいて、アクセスメソッドはインデックス構造からのエントリ削除を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/ginvacuum.c
  • src/backend/access/gist/gistvacuum.c
  • src/backend/access/hash/hash.c
  • contrib/bloom/blvacuum.c
  • **呼び出し先**:
  • 呼び出し先に関する情報は提供されていません。

関連データ構造

  • IndexBulkDeleteResult: バルク削除処理の結果を格納する構造体。
  • IndexVacuumInfo: VACUUM処理のメタデータを保持する構造体。
  • ambulkdelete_function: このコールバックを引数として受け取るインデックスAMの関数ポインタ型。

失敗モードと境界条件

  • callback_state が NULL の場合、コールバック関数内で不正なメモリアクセスが発生する可能性があります。
  • インデックススキャン中にエラーが発生した場合、トランザクションのロールバックに伴い削除処理も中断されます。

保守担当者向けメモ

  • 本シンボルは IndexAmRoutine の ambulkdelete メソッドの引数として使用されます。
  • 新しいインデックスAMを実装する際は、このコールバックを適切に呼び出し、戻り値に応じてインデックスページ上のエントリを物理的に削除するロジックを実装してください。
