sizebitvec

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_intbig_gist.c:210

目的

ビットベクトル内のセットされているビット数（ハミング重み）を計算し、インデックスの密度やコスト評価の指標として提供します。

シグネチャまたは形状

int32 sizebitvec(BITVECP sign, int siglen)

入力と出力

  • **入力**:
  • sign (BITVECP): 対象となるビットベクトルの先頭ポインタ。
  • siglen (int): ビットベクトルの長さ（バイト単位）。
  • **出力**:
  • int32: ビットベクトル内で 1 に設定されているビットの総数。

前提条件と不変条件

  • sign は少なくとも siglen バイト以上の有効なメモリ領域を指している必要があります。
  • siglen は非負である必要があります。

アルゴリズムまたは振る舞い

pg_popcount を呼び出し、指定されたメモリ領域内のビット数をカウントして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_intbig_gist.c (内部のコスト計算ロジック)
  • contrib/hstore/hstore_gist.c
  • contrib/ltree/_ltree_gist.c
  • contrib/pg_trgm/trgm_gist.c
  • **呼び出し先**:
  • pg_popcount (PostgreSQL のビット操作ユーティリティ)

関連データ構造

  • BITVECP: ビットベクトルを表現するための型（通常は char * 等のポインタ型）。

失敗モードと境界条件

  • sign が NULL の場合、pg_popcount の実装に依存した未定義動作が発生する可能性があります。
  • siglen が 0 の場合、戻り値は 0 となります。

保守担当者向けメモ

  • 本関数は複数の contrib モジュール（hstore, ltree, pg_trgm 等）で共通して利用されています。変更を加える際は、これら他モジュールへの影響を考慮してください。
  • pg_popcount はハードウェアアクセラレーション（POPCNT 命令等）を利用する可能性があるため、非常に高速に動作します。
