blrescan

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blscan.c:44

目的

Bloom インデックススキャンの状態をリセットし、新しい検索条件を適用するために使用されます。

シグネチャまたは形状

void blrescan(IndexScanDesc scan, ScanKey scankey, int nscankeys,
              ScanKey orderbys, int norderbys)

入力と出力

  • **入力**:
  • scan: インデックススキャンの記述子。
  • scankey: 新しい検索条件（オプション）。
  • nscankeys: 検索キーの数。
  • orderbys: ソート順序（Bloom インデックスでは未使用）。
  • norderbys: ソートキーの数。
  • **出力**: なし（副作用としてスキャン状態を更新）。

前提条件と不変条件

  • scan->opaque は BloomScanOpaque 型としてキャスト可能であること。
  • scankey が提供された場合、scan->numberOfKeys と整合性があること。

アルゴリズムまたは振る舞い

1. スキャン記述子から BloomScanOpaque を取得します。 2. 既存の検索シグネチャ so->sign が存在する場合、メモリを解放 (pfree) し、ポインタを NULL に初期化します。これにより、次回の blgetbitmap 実行時に新しいシグネチャが再計算されます。 3. 新しい scankey が提供されている場合、scan->keyData を新しいキー情報で上書き更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックスアクセスメソッド（AM）インターフェースを通じて呼び出されます（contrib/bloom/blutils.c で amrescan に登録）。
  • **呼び出し先**: pfree（メモリ解放）。

関連データ構造

  • BloomScanOpaqueData: スキャン中の状態を保持する構造体。so->sign（検索シグネチャ）を管理します。
  • IndexScanDesc: PostgreSQL の標準的なインデックススキャン記述子。

失敗モードと境界条件

  • scankey が NULL の場合、キーのコピーは行われません。
  • scan->numberOfKeys が 0 の場合、キーのコピーは行われません。

保守担当者向けメモ

  • 本関数は、スキャン中にキャッシュされた検索シグネチャを破棄する役割を担います。これにより、再スキャン時に最新の検索条件に基づいたシグネチャ計算が強制されます。
  • orderbys 引数は現在の実装では使用されていません。
