PLpgSQL_expr

定義

  • 種別: struct
  • 定義箇所: src/pl/plpgsql/src/plpgsql.h:230

目的

PL/pgSQL 内の SQL クエリや式を保持し、SPI（Server Programming Interface）を通じて実行するための構造体です。

シグネチャまたは形状

typedef struct PLpgSQL_expr {
    char *query;
    RawParseMode parseMode;
    struct PLpgSQL_function *func;
    struct PLpgSQL_nsitem *ns;
    int target_param;
    bool target_is_local;
    SPIPlanPtr plan;
    Bitmapset *paramnos;
    Expr *expr_simple_expr;
    Oid expr_simple_type;
    int32 expr_simple_typmod;
    bool expr_simple_mutable;
    PLpgSQL_rwopt expr_rwopt;
    Param *expr_rw_param;
    CachedPlanSource *expr_simple_plansource;
    CachedPlan *expr_simple_plan;
    LocalTransactionId expr_simple_plan_lxid;
    ExprState *expr_simple_state;
    bool expr_simple_in_use;
    LocalTransactionId expr_simple_lxid;
} PLpgSQL_expr;

入力と出力

  • **入力**: 関数本体から抽出されたクエリ文字列（query）および名前空間情報（ns）。
  • **出力**: SPI を介した実行結果、または「単純な式（simple expression）」として評価された場合は expr_simple_state を通じた評価結果。

前提条件と不変条件

  • query は関数本体から逐語的に取得される。
  • plan や expr_simple_state などの実行関連フィールドは、初回解析・計画作成時までゼロまたは NULL で初期化される。
  • 単純な式の実行状態（expr_simple_state 等）は、expr_simple_lxid が現在のトランザクション ID（LXID）と一致する場合にのみ有効とみなされる。

アルゴリズムまたは振る舞い

1. **解析フェーズ**: plpgsql_parser_setup 等により、クエリ文字列と名前空間が設定される。 2. **計画作成**: 初回実行時に plan が生成され、paramnos に参照される変数のインデックス（dno）が記録される。 3. **最適化**:

  • 単純な式と判定された場合、expr_simple_expr に評価ツリーが保持され、高速なパスで実行される。
  • 読み書き可能な展開済みオブジェクト（expanded object）への代入がある場合、expr_rwopt を用いて最適化が行われる。

    • 4. **キャッシュ管理**: CachedPlan を保持し、トランザクションを跨ぐ再利用を管理する。

呼び出し元と呼び出し先

  • **呼び出し元**: pl_comp.c（解析時）、pl_exec.c（実行時）。
  • **呼び出し先**: SPI インターフェース、PostgreSQL の式評価器、キャッシュ管理システム。

関連データ構造

  • PLpgSQL_function: この式を保持する関数定義。
  • PLpgSQL_nsitem: 式から参照可能な名前空間チェーン。
  • SPIPlanPtr: SPI による実行計画。
  • ExprState: 単純な式を評価するための実行ツリー。

失敗モードと境界条件

  • expr_simple_lxid が現在の LXID と一致しない場合、expr_simple_state は無効なメモリを指す可能性があるため、再検証が必要。
  • target_param が -1 の場合、代入ターゲットとしての最適化は行われない。

保守担当者向けメモ

  • 本構造体はコンパイル時と実行時の両方で状態を保持するため、フィールドの有効期限（特に lxid によるキャッシュの整合性）に注意すること。
  • dno（datum number）による変数参照の整合性は、pl_comp.c の resolve_column_ref 等で保証されている。
