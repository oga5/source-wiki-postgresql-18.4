TransactionIdAbortTree

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:269

目的

指定されたトランザクションID（xid）およびそのサブトランザクション群（xids）のステータスを、一括して「アボート済み（TRANSACTION_STATUS_ABORTED）」に設定する。

シグネチャまたは形状

void TransactionIdAbortTree(TransactionId xid, int nxids, TransactionId *xids)

入力と出力

  • **入力**:
  • xid: 親トランザクションのID。
  • nxids: サブトランザクションの数。
  • xids: サブトランザクションIDの配列。
  • **出力**: なし（void）。

前提条件と不変条件

  • xid および xids 配列内の各IDは、有効なトランザクションIDであること。
  • 内部的に TransactionIdSetTreeStatus を呼び出すため、共有メモリ上のCLOG（Commit Log）への書き込み権限が必要。

アルゴリズムまたは振る舞い

1. TransactionIdSetTreeStatus を呼び出し、親トランザクション xid とサブトランザクション配列 xids のステータスを TRANSACTION_STATUS_ABORTED に更新する。 2. この際、ログシーケンス番号（LSN）として InvalidXLogRecPtr を渡すことで、アボート状態には特定のLSNを関連付けない。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xact.c
  • **呼び出し先**:
  • TransactionIdSetTreeStatus (内部関数)

関連データ構造

  • TransactionId: トランザクションを識別する32ビットの循環値。
  • CLOG (Commit Log): トランザクションのコミット/アボート状態を保持する永続化ストレージ。

失敗モードと境界条件

  • nxids が0の場合、親トランザクション xid のみがアボート状態に設定される。
  • 共有メモリやCLOGの書き込み時にエラーが発生した場合、PostgreSQLの標準的なエラーハンドリングによりトランザクションが中断される。

保守担当者向けメモ

  • 本関数は TransactionIdCommitTree と対になる関数であり、トランザクションツリーのステータス更新を一元管理する。
  • サブトランザクションの階層が深い場合、TransactionIdSetTreeStatus の処理コストに依存するため、呼び出し頻度とパフォーマンスに注意すること。
