pg_get_ruledef_ext

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ruleutils.c:577

目的

指定されたルール（Rule）の定義を SQL 文字列として逆コンパイル（deparse）し、整形オプション付きで取得します。

シグネチャまたは形状

Datum pg_get_ruledef_ext(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象となるルールの OID。
  • PG_GETARG_BOOL(1): 整形（pretty-print）を行うかどうかのフラグ。
  • **出力**:
  • Datum: ルール定義を表す text 型の文字列。ルールが見つからない場合は NULL を返します。

前提条件と不変条件

  • PG_FUNCTION_ARGS マクロを通じて呼び出される PostgreSQL の SQL 関数インターフェースに準拠しています。
  • 内部で GET_PRETTY_FLAGS マクロを使用して、入力されたブール値を内部的な整形フラグへ変換します。

アルゴリズムまたは振る舞い

1. 入力された pretty フラグを prettyFlags に変換します。 2. pg_get_ruledef_worker を呼び出し、指定された OID のルール定義を文字列として取得します。 3. pg_get_ruledef_worker が NULL を返した場合は、PG_RETURN_NULL() で終了します。 4. 取得した文字列を string_to_text で text 型に変換し、PG_RETURN_TEXT_P を通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（具体的な SQL 文脈はソースコード上では未定義）。
  • **呼び出し先**:
  • pg_get_ruledef_worker: ルール定義の逆コンパイルを実行する内部関数。
  • string_to_text: 文字列を text 型に変換するユーティリティ。

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • 指定された OID に対応するルールが存在しない場合、pg_get_ruledef_worker が NULL を返し、本関数も NULL を返します。

保守担当者向けメモ

  • 本関数は pg_get_ruledef の拡張版であり、整形オプションの制御を外部から受け取れるように設計されています。
  • 逆コンパイルのロジック変更が必要な場合は、pg_get_ruledef_worker を参照してください。
