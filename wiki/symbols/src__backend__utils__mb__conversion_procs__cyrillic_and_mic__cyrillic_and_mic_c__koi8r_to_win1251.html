koi8r_to_win1251

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:437

目的

KOI8-R エンコーディングから WIN1251 エンコーディングへ文字列を変換する PostgreSQL の変換プロシージャです。

シグネチャまたは形状

Datum koi8r_to_win1251(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得します。
  • src (第3引数): 変換元文字列（CSTRING）
  • dest (第4引数): 変換先バッファ（CSTRING）
  • len (第5引数): 文字列長（INT32）
  • noError (第6引数): エラーハンドリングフラグ（BOOL）
  • **出力**: 変換されたバイト数を PG_RETURN_INT32 で返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_KOI8R, PG_WIN1251) マクロにより、ソースエンコーディングが KOI8-R、宛先エンコーディングが WIN1251 であることが検証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認します。 3. local2local 関数を呼び出し、koi2win1251 変換テーブルを使用して変換を実行します。 4. 変換されたバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: local2local（変換処理の実行）。

関連データ構造

  • koi2win1251: KOI8-R から WIN1251 への変換用マッピングテーブル。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS によるエンコーディング不一致時のエラー発生。
  • local2local 内部での変換エラー（noError フラグの状態に依存）。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_ARGS を引数に取るため、SQL レベルの変換プロシージャとして機能します。直接呼び出すことは想定されていません。
  • 変換ロジックの変更が必要な場合は、koi2win1251 テーブルの定義を確認してください。
