目的

ECPG（Embedded SQL for C）プリプロセッサによる文字列リテラル処理の妥当性を検証するためのテスト実行エントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しない）
  • 出力: 成功時に 0 を返し、標準出力にSQLクエリの結果を表示する。

前提条件と不変条件

  • ECPGランタイムライブラリが利用可能であること。
  • テスト用データベース ecpg1_regression への接続が可能であること。

アルゴリズムまたは振る舞い

1. ECPGdebug を有効化し、デバッグ出力を stderr に設定する。 2. ECPGconnect を呼び出し、テスト用データベースに接続する。 3. ECPGdo を介して set standard_conforming_strings to on を実行し、文字列リテラルの解釈ルールを固定する。 4. 最初の ECPGdo で各種文字列リテラル（N'', E'', U&'', $$ 等）をクエリし、結果を s1 から s6 に格納する。 5. 取得した文字列を printf で標準出力に書き出す。 6. 二番目の ECPGdo でビット文字列および16進数文字列をクエリし、結果を s7 と s8 に格納する。 7. 取得した文字列を printf で標準出力に書き出す。 8. ECPGdisconnect でデータベース接続を終了する。

呼び出し元と呼び出し先

  • 呼び出し元: 不明（テストスイートの実行環境から呼び出される）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGdisconnect, printf

関連データ構造

  • s1〜s8: SQLクエリの結果を受け取るためのC言語文字列ポインタ変数。

失敗モードと境界条件

  • データベース接続失敗時やSQL実行エラー時の挙動は、ECPGライブラリのデフォルトの例外処理に依存する。
  • プリプロセッサによって生成されたコードであるため、手動での変更は推奨されない。
