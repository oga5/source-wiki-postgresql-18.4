LargeObjectExists

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_largeobject.c:166

目的

指定された OID を持つラージオブジェクトがシステム上に存在するかを確認します。

シグネチャまたは形状

bool LargeObjectExists(Oid loid)

入力と出力

  • **入力**: loid (Oid) - 存在を確認するラージオブジェクトの識別子。
  • **出力**: bool - オブジェクトが存在する場合は true、存在しない場合は false を返します。

前提条件と不変条件

  • 呼び出し元のトランザクション境界内で実行されます。
  • システムキャッシュを使用せず、カタログテーブルを直接走査するため、常に最新のカタログ状態を参照します。

アルゴリズムまたは振る舞い

内部的に LargeObjectExistsWithSnapshot(loid, NULL) を呼び出します。NULL スナップショットを渡すことで、現在のトランザクションのスナップショットを使用して pg_largeobject_metadata テーブルを走査し、対象の OID が存在するかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c (複数箇所)
  • src/backend/libpq/be-fsstubs.c
  • **呼び出し先**:
  • LargeObjectExistsWithSnapshot

関連データ構造

  • pg_largeobject_metadata: 存在確認の対象となるシステムカタログ。
  • Oid: ラージオブジェクトを一意に識別する型。

失敗モードと境界条件

  • 存在しない OID を指定した場合、エラーにはならず false を返します。
  • カタログテーブルへのアクセス権限やトランザクションの状態に依存します。

保守担当者向けメモ

  • 本関数はキャッシュを回避するため、頻繁な呼び出しはカタログへの直接アクセスを伴い、パフォーマンスに影響を与える可能性があります。
  • 特定のスナップショットを指定して存在確認を行う必要がある場合は、直接 LargeObjectExistsWithSnapshot を使用してください。
