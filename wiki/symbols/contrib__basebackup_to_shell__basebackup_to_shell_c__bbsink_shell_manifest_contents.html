bbsink_shell_manifest_contents

定義

  • 種別: function
  • 定義箇所: contrib/basebackup_to_shell/basebackup_to_shell.c:357

目的

バックアップマニフェストのデータチャンクを処理し、外部シェルコマンドへの転送および後続のシンクへのデータ転送を行う。

シグネチャまたは形状

static void bbsink_shell_manifest_contents(bbsink *sink, size_t len)

入力と出力

  • sink: bbsink 構造体へのポインタ。bbsink_shell 型としてキャストして使用される。
  • len: 転送対象となるデータチャンクのサイズ（バイト単位）。
  • 出力: なし（副作用として外部プロセスへのデータ書き込みおよび後続シンクへの転送を行う）。

前提条件と不変条件

  • sink は bbsink_shell 構造体として適切に初期化されている必要がある。
  • shell_send_data が内部で呼び出されるため、パイプが既に開かれている状態である必要がある。

アルゴリズムまたは振る舞い

1. 引数 sink を bbsink_shell 型にキャストする。 2. shell_send_data を呼び出し、指定されたサイズ len のデータを外部シェルコマンドのパイプへ書き込む。 3. bbsink_forward_manifest_contents を呼び出し、現在のデータチャンクをバックアップチェーン内の後続のシンクへ転送する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink フレームワーク（bbsink_shell_ops 構造体の .manifest_contents フィールド経由で呼び出される）。
  • 呼び出し先:
  • shell_send_data
  • bbsink_forward_manifest_contents

関連データ構造

  • bbsink: バックアップターゲットの抽象基底構造体。
  • bbsink_shell: bbsink を継承した、シェルコマンド実行状態を保持する拡張構造体。

失敗モードと境界条件

  • shell_send_data 内部でパイプへの書き込みエラーが発生した場合、バックアップ処理全体がエラーとして扱われる可能性がある。

保守担当者向けメモ

  • 本関数は bbsink インターフェースの一部であり、マニフェストデータのストリーム処理において「外部コマンドへの供給」と「チェーンの継続」という二重の責務を担っている。
