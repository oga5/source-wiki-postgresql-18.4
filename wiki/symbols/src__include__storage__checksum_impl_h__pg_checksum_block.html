pg_checksum_block

定義

  • 種別: function
  • 定義箇所: src/include/storage/checksum_impl.h:145

目的

データページの内容に基づき、整合性検証のためのチェックサム値を計算します。

シグネチャまたは形状

static uint32 pg_checksum_block(const PGChecksummablePage *page)

入力と出力

  • **入力**: PGChecksummablePage 構造体へのポインタ。ページデータを32列の行列として保持します。
  • **出力**: 計算された32ビットのチェックサム値。

前提条件と不変条件

  • sizeof(PGChecksummablePage) が BLCKSZ と一致している必要があります（Assert により検証）。
  • checksumBaseOffsets が初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: sums 配列を checksumBaseOffsets で初期化します。 2. **ブロック計算**: ページデータを N_SUMS（32）個の並列ハッシュとして処理します。各データ要素に対して CHECKSUM_COMP マクロを適用し、ハッシュ値を更新します。 3. **混合**: ゼロ値を用いて CHECKSUM_COMP を2回実行し、ビットの拡散を促進します。 4. **集約**: 32個のハッシュ値をXORで折り畳み、最終的な32ビットの値を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/storage/checksum_impl.h 内のコードから呼び出されています。
  • **呼び出し先**: CHECKSUM_COMP マクロ（計算の基本単位）。

関連データ構造

  • PGChecksummablePage: ページデータをSIMDベクトル化に適した行列形式で保持する共用体。
  • N_SUMS: 並列計算されるハッシュの数（32）。
  • checksumBaseOffsets: ハッシュ計算の初期オフセット値。

失敗モードと境界条件

  • BLCKSZ が sizeof(uint32) * N_SUMS の倍数でない場合、計算ループの範囲外アクセスや不整合が発生する可能性があります。

保守担当者向けメモ

  • 本関数はコンパイラによる自動ベクトル化を意図して設計されています。ループ構造を変更する際は、SIMD命令の効率を維持するように注意してください。
  • N_SUMS や checksumBaseOffsets の変更は、既存のデータベースページとの互換性を完全に破壊します。
