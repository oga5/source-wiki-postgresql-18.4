WalWriterMain

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/walwriter.c:87

目的

PostgreSQL の WAL Writer プロセスのメインエントリポイントとして、WAL バッファの定期的なフラッシュとディスクへの同期を行い、非同期コミットの永続化を保証する。

シグネチャまたは形状

void WalWriterMain(const void *startup_data, size_t startup_data_len)

入力と出力

  • **入力**:
  • startup_data: 使用されない（startup_data_len == 0 であることが前提）。
  • startup_data_len: 0 である必要がある。
  • **出力**: なし（プロセス終了までループを継続する）。

前提条件と不変条件

  • startup_data_len は 0 であること。
  • AuxiliaryProcessMainCommon() を経由して補助プロセスとして初期化されていること。
  • sigsetjmp を用いたエラーリカバリスタックが常に有効であること。

アルゴリズムまたは振る舞い

1. **初期化**: シグナルハンドラの設定、専用メモリコンテキストの作成、およびシグナルマスクの解除を行う。 2. **メインループ**:

  • ProcessMainLoopInterrupts() でシグナルを処理する。
  • XLogBackgroundFlush() を呼び出し、WAL の書き出しを試みる。
  • 書き込みが成功した場合はハイバネーションカウンタをリセットし、失敗した場合はカウンタを減算する。
  • カウンタが 0 に達した場合、HIBERNATE_FACTOR を用いて待機時間を延長する。

    • 3. **エラー処理**: sigsetjmp によりエラー発生を検知し、リソース解放と FlushErrorState() を経由して 1 秒待機後にループを再開する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c（B_WAL_WRITER プロセス起動時）。
  • **呼び出し先**: AuxiliaryProcessMainCommon, XLogBackgroundFlush, ProcessMainLoopInterrupts, WaitLatch（ループ内）, pgstat_report_wal 等。

関連データ構造

  • sigjmp_buf: エラーリカバリ用スタック。
  • MemoryContext: walwriter_context として、メモリリーク防止のために使用。
  • ProcGlobal->walwriterProc: バックエンドプロセスが WAL Writer を起床させるためのプロセス番号。

失敗モードと境界条件

  • **エラー発生時**: sigsetjmp により捕捉され、リソース（LWLocks, バッファ等）を解放した後に 1 秒間待機して再開する。
  • **アイドル状態**: left_till_hibernate が 0 になると、WalWriterDelay に HIBERNATE_FACTOR を乗じた時間まで待機時間を延長し、消費電力を抑制する。

保守担当者向けメモ

  • PG_TRY ではなく sigsetjmp を直接使用しているのは、エラーハンドラ自体がクラッシュするリスクを避けるためである。
  • WAL Writer は必須プロセスではないため、停止してもデータベースの整合性は保たれる。
  • ハイバネーションの挙動（HIBERNATE_FACTOR）は、低負荷時の CPU 使用率に直結する。
