check_table

定義

  • 種別: function
  • 定義箇所: contrib/isn/isn.c:68

目的

各識別子（ISBN, ISSN等）の範囲テーブルおよびインデックステーブルの整合性を検証する。デバッグビルド環境において、テーブル定義が正しい形式と順序に従っているかを確認するために使用される。

シグネチャまたは形状

bool check_table(const char *(*TABLE)[2], const unsigned TABLE_index[10][2])

入力と出力

  • **入力**:
  • TABLE: 範囲定義を保持する文字列配列のポインタ。
  • TABLE_index: 範囲のインデックスを保持する 10x2 の符号なし整数配列。
  • **出力**:
  • true: 検証成功。
  • false: 検証失敗（elog にて DEBUG1 レベルのログを出力）。

前提条件と不変条件

  • TABLE および TABLE_index が NULL の場合は常に true を返す。
  • TABLE 内の各エントリは、数字で開始し、同一のフォーマットと長さを持つ必要がある。
  • TABLE_index は、TABLE 内の範囲定義と整合する値（開始位置とオフセット）を保持していなければならない。

アルゴリズムまたは振る舞い

1. TABLE を順次走査し、各要素が数字で構成されているか、およびハイフン等の形式が一致しているかを確認する。 2. 範囲の開始値（a）と終了値（y）を追跡し、TABLE_index が示す範囲と TABLE のインデックスが一致するかを検証する。 3. 範囲の境界値が逆転している場合（y < x）や、インデックス値が期待値と異なる場合はエラーとみなす。 4. 検証中に不整合が検出された場合、invalidtable または invalidindex ラベルへジャンプし、エラーログを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: _PG_init（contrib/isn/isn.c 内）から、各識別子（EAN13, ISBN, ISMN, ISSN, UPC）のテーブル検証のために呼び出される。
  • **呼び出し先**: isdigit（標準ライブラリ）、elog（PostgreSQL エラーログ出力）。

関連データ構造

  • EAN13_range, ISBN_range, ISMN_range, ISSN_range, UPC_range
  • EAN13_index, ISBN_index, ISMN_index, ISSN_index, UPC_index

失敗モードと境界条件

  • **失敗モード**:
  • テーブル内の文字列が数字以外を含む、または形式が不一致の場合。
  • インデックステーブルの値が実際の範囲と一致しない場合。
  • 範囲の終了値が開始値より小さい場合。
  • **境界条件**:
  • TABLE が空、または NULL の場合は検証をスキップする。

保守担当者向けメモ

  • 本関数はデバッグ目的の整合性チェックであるため、運用環境ではパフォーマンスへの影響を考慮し、ISN_DEBUG マクロ等で制御される初期化処理の一部としてのみ実行されることが想定されている。
