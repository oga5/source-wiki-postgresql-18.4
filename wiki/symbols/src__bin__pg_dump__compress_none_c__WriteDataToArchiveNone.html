WriteDataToArchiveNone

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_none.c:48

目的

圧縮を行わないアーカイブデータ出力において、データを直接転送するためのラッパー関数です。

シグネチャまたは形状

static void WriteDataToArchiveNone(ArchiveHandle *AH, CompressorState *cs, const void *data, size_t dLen);

入力と出力

  • AH: アーカイブハンドルへのポインタ。
  • cs: 圧縮状態を保持する CompressorState 構造体へのポインタ。
  • data: 書き込むデータバッファへのポインタ。
  • dLen: 書き込むデータのバイト数。
  • 戻り値: なし。

前提条件と不変条件

  • cs->writeF が有効な関数ポインタとして初期化されている必要があります。
  • 本関数は圧縮処理を介さず、入力データをそのまま出力先へ転送します。

アルゴリズムまたは振る舞い

1. cs->writeF を呼び出し、引数として AH、data、dLen を渡します。 2. 内部的には fwrite 等の標準 I/O を用いたデータ転送が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: pg_dump のアーカイブ処理において、非圧縮モードが選択された際に CompressorState の writeData メンバとして登録され、間接的に呼び出されます。
  • 呼び出し先: cs->writeF（CompressorState 内の関数ポインタ）。

関連データ構造

  • ArchiveHandle: アーカイブ全体の状態を管理する構造体。
  • CompressorState: 圧縮アルゴリズムごとの関数ポインタや状態を保持する構造体。

失敗モードと境界条件

  • cs->writeF が実行するファイル I/O 操作が失敗した場合、pg_fatal を通じて致命的なエラーとして報告されます。

保守担当者向けメモ

  • 本関数は単なる転送ラッパーであり、状態の保持や複雑なロジックは含みません。
  • cs->writeData メンバへの代入を通じて利用されるため、インターフェースの変更時には CompressorState の定義と整合性を確認してください。
