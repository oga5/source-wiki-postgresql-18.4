TestDecodingData

定義

  • 種別: struct
  • 定義箇所: contrib/test_decoding/test_decoding.c:30

目的

論理デコーディング出力プラグインの設定および状態を保持するための構造体です。ユーザーが指定したオプション値を格納し、プラグインの各コールバック関数間で共有するために使用されます。

シグネチャまたは形状

typedef struct
{
	MemoryContext context;
	bool		include_xids;
	bool		include_timestamp;
	bool		skip_empty_xacts;
	bool		only_local;
} TestDecodingData;

入力と出力

  • **入力**: pg_decode_startup 内でユーザー指定のオプションを解析し、本構造体の各フィールドに値を設定します。
  • **出力**: プラグインの各コールバック関数（pg_decode_change 等）が、本構造体のフラグを参照して出力内容を制御します。

前提条件と不変条件

  • LogicalDecodingContext の output_plugin_private フィールドに格納され、プラグインのライフサイクルを通じて保持されます。
  • MemoryContext フィールドは、プラグインのメモリ管理に使用されます。

アルゴリズムまたは振る舞い

  • pg_decode_startup にて palloc0 を用いてメモリ確保および初期化が行われます。
  • 各コールバック関数は ctx->output_plugin_private をキャストして本構造体を取得し、include_xids や skip_empty_xacts などのフラグに基づいて、WAL デコード結果の出力形式やフィルタリング条件を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_decode_startup（初期化）、および pg_decode_change、pg_output_begin、pg_decode_commit_txn 等の各コールバック関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • LogicalDecodingContext: 本構造体を output_plugin_private として保持するコンテキスト。
  • TestDecodingTxnData: トランザクションごとの状態を追跡する構造体。

失敗モードと境界条件

  • メモリ確保失敗時: palloc0 が失敗した場合は PostgreSQL のエラーハンドリング機構により例外が発生します。
  • オプション未設定時: 各 bool フィールドは palloc0 によりデフォルトで false となります。

保守担当者向けメモ

  • 本構造体は contrib/test_decoding プラグイン専用です。設定項目を追加する場合は、pg_decode_startup 内のオプション解析ロジックも併せて更新してください。
