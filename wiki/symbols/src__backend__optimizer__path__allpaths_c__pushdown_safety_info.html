pushdown_safety_info

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/path/allpaths.c:60

目的

サブクエリへのクエリ条件（Qual）のプッシュダウンが安全かどうかを判定・管理するための情報を保持します。

シグネチャまたは形状

typedef struct pushdown_safety_info
{
    unsigned char *unsafeFlags; /* ターゲットリスト列がプッシュダウンに不適格な理由を示すビットマスク */
    bool        unsafeVolatile; /* volatile な Qual のプッシュダウンを禁止 */
    bool        unsafeLeaky;    /* leaky な Qual のプッシュダウンを禁止 */
} pushdown_safety_info;

入力と出力

本構造体は、サブクエリの最適化プロセスにおいて、条件のプッシュダウン可否を判定する関数（subquery_is_pushdown_safe 等）の引数として渡され、フラグの状態を更新・参照するために使用されます。

前提条件と不変条件

  • unsafeFlags は、ターゲットリストの各列に対する安全性をビットマスクで管理します。理由がない場合は 0 となります。
  • unsafeVolatile および unsafeLeaky は、それぞれ特定の性質を持つ Qual のプッシュダウンを制御するブール値です。

アルゴリズムまたは振る舞い

プランナがサブクエリへの条件プッシュダウンを検討する際、この構造体を使用して、対象となる式や列が安全かどうかを検証します。check_output_expressions などの関数が、この構造体の unsafeFlags を設定することで、プッシュダウンの可否を決定します。

呼び出し元と呼び出し先

  • **主な呼び出し元**: src/backend/optimizer/path/allpaths.c 内のサブクエリ最適化関連関数（subquery_is_pushdown_safe 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Query: サブクエリの解析ツリー。
  • pushdown_safe_type: プッシュダウンの安全性を定義する関連型。

失敗モードと境界条件

  • unsafeFlags が適切に初期化されていない場合、誤ったプッシュダウン判定が行われる可能性があります。
  • unsafeVolatile や unsafeLeaky が真の場合、たとえ他の条件が安全であってもプッシュダウンは抑制されます。

保守担当者向けメモ

  • unsafeFlags のビットマスク定義については、allpaths.c の 52 行目付近にあるコメントを参照してください。
  • 本構造体は subquery_is_pushdown_safe や check_output_expressions と密接に関連しているため、プッシュダウンロジックを変更する際はこれらの関数と併せて確認してください。
