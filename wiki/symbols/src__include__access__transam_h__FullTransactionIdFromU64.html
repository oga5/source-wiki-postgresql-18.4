FullTransactionIdFromU64

定義

  • 種別: function
  • 定義箇所: src/include/access/transam.h:80

目的

64ビットの符号なし整数（uint64）から FullTransactionId 構造体を生成するユーティリティ関数です。

シグネチャまたは形状

static inline FullTransactionId FullTransactionIdFromU64(uint64 value)

入力と出力

  • **入力**: uint64 value（変換対象となる64ビットのトランザクションID値）
  • **出力**: FullTransactionId（入力値を内部の value フィールドに保持した構造体）

前提条件と不変条件

  • 入力された uint64 値は、有効な FullTransactionId として解釈可能である必要があります。
  • 本関数はインライン関数として定義されており、呼び出し元で FullTransactionId 型のメモリレイアウトが確定している必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った uint64 値を、ローカル変数 result（FullTransactionId 型）の value フィールドに代入します。 2. 代入済みの result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • src/backend/access/transam/twophase.c
  • src/backend/storage/ipc/procarray.c
  • src/backend/utils/adt/xid.c
  • src/backend/utils/adt/xid8funcs.c
  • **呼び出し先**:
  • 呼び出し先に関する情報は提供されていません。

関連データ構造

  • FullTransactionId: エポックを含む64ビットのトランザクションIDを保持する構造体。

失敗モードと境界条件

  • 本関数自体は単純な代入操作であるため、関数内部での失敗モードはありません。
  • 入力値が FullTransactionId として無効な範囲（予約済みIDの不適切な組み合わせ等）である場合、呼び出し元での論理的な不整合を招く可能性があります。

保守担当者向けメモ

  • 本関数は src/include/access/transam.h 内でインライン定義されており、パフォーマンスが重視される箇所で頻繁に使用されます。
  • FullTransactionId の内部構造に変更を加える場合は、本関数および関連する U64FromFullTransactionId 等の変換ロジックとの整合性に注意してください。
