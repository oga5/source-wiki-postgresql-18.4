AlterCollation

定義

  • 種別: function
  • 定義箇所: src/backend/commands/collationcmds.c:423

目的

照合順序（Collation）オブジェクトのバージョン情報を更新する。

シグネチャまたは形状

ObjectAddress AlterCollation(AlterCollationStmt *stmt)

入力と出力

  • **入力**: AlterCollationStmt *stmt（更新対象の照合順序名を含む構文解析ツリー）
  • **出力**: ObjectAddress（更新された照合順序のオブジェクト識別子）

前提条件と不変条件

  • 対象の照合順序が DEFAULT_COLLATION_OID ではないこと。
  • 呼び出し元が対象照合順序の所有者であること。
  • 既存のバージョン情報（collversion）が NULL から非 NULL、またはその逆へ変化するような更新は許可されない。

アルゴリズムまたは振る舞い

1. pg_collation システムカタログを RowExclusiveLock で開く。 2. 指定された照合順序の OID を取得し、所有権を確認する。 3. システムキャッシュから対象のタプルをコピーし、現在の collversion を取得する。 4. プロバイダ（libc またはその他）に応じて、現在の環境における最新の照合順序バージョンを get_collation_actual_version で取得する。 5. 現在のバージョンと新しいバージョンを比較し、異なる場合にのみ heap_modify_tuple を用いて collversion 列を更新する。 6. CatalogTupleUpdate を実行してカタログを反映し、オブジェクト変更フックを呼び出す。 7. 最後にタプルを解放し、テーブルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し先**:

関連データ構造

  • AlterCollationStmt: SQL文の解析結果を保持する構造体。
  • Form_pg_collation: pg_collation カタログの行構造。

失敗モードと境界条件

  • **エラー**:
  • デフォルト照合順序のバージョン更新を試みた場合。
  • 所有者権限がない場合。
  • 指定された照合順序が見つからない場合。
  • バージョン情報の NULL/非 NULL 状態が不整合な変更となる場合。

保守担当者向けメモ

  • 本関数は pg_collation カタログの collversion 列のみを更新対象としています。
  • プロバイダごとのバージョン取得ロジックは get_collation_actual_version に依存しているため、プロバイダ追加時はそちらの確認が必要です。
