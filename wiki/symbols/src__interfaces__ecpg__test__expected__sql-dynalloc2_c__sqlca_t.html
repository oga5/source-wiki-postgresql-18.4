sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-dynalloc2.c:31

目的

ECPG（Embedded SQL for C）におけるSQL通信領域（SQL Communication Area）を定義し、SQL文の実行結果やエラー情報を保持する。

シグネチャまたは形状

struct sqlca_t
{
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: SQL文の実行結果（成功・失敗、警告、処理行数など）がECPGライブラリ経由で各フィールドに書き込まれる。
  • **出力**: アプリケーション側で sqlcode や sqlstate を参照し、エラーハンドリングや後続処理の分岐に使用する。

前提条件と不変条件

  • ECPGライブラリが提供する ECPGget_sqlca() を通じてインスタンスを取得することが一般的である。
  • sqlwarn の要素0は、他の要素に警告が含まれる場合に 'W' が設定される。

アルゴリズムまたは振る舞い

  • SQL文の実行ごとに内容が更新される。
  • sqlerrd[2] には、INSERT、UPDATE、DELETE 文の実行後に処理された行数が格納される。
  • sqlerrm 構造体には、エラー発生時のメッセージが格納される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/compatlib/informix.c、src/interfaces/ecpg/ecpglib/connect.c、src/interfaces/ecpg/ecpglib/data.c、src/interfaces/ecpg/ecpglib/descriptor.c 等のECPGライブラリ内部関数が、SQL実行状態の確認のために本構造体を参照・更新する。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはない。

関連データ構造

  • sqlda_t: 動的SQL記述子領域。
  • sqlvar_t: sqlda_t 内で使用される列定義構造体。

失敗モードと境界条件

  • sqlcode が 0 以外の場合、SQL文の実行に失敗または警告が発生していることを示す。
  • sqlwarn の各要素は、文字列の切り捨てや非致命的な通知が発生した際に 'W' が設定される。

保守担当者向けメモ

  • 本構造体はECPGの標準的な通信領域であるため、定義を変更する場合はライブラリ全体への影響を考慮すること。
  • sqlerrd や sqlwarn の各要素の意味は、PostgreSQLのECPG仕様に準拠している。
