array_subscript_handler

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arraysubs.c:539

目的

標準的なPostgreSQL配列型に対する添字操作（サブスクリプト）を処理するための、SubscriptRoutines 構造体へのポインタを返すハンドラ関数です。

シグネチャまたは形状

Datum array_subscript_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（標準的なPostgreSQL関数呼び出し規約に従う引数）。
  • **出力**: Datum 型として、静的に定義された SubscriptRoutines 構造体へのポインタを返します。

前提条件と不変条件

  • 本関数は、標準的な配列型に対してのみ使用されることを前提としています。
  • 返される SubscriptRoutines 構造体は静的（static const）であり、実行中に変更されることはありません。

アルゴリズムまたは振る舞い

1. 以下の設定を持つ SubscriptRoutines 構造体を静的に定義します。

  • transform: array_subscript_transform
  • exec_setup: array_exec_setup
  • fetch_strict: true（NULL入力に対してNULLを返す）
  • fetch_leakproof: true（不正な添字に対してNULLを返す）
  • store_leakproof: false（代入操作でエラーを投げる可能性がある）

    • 2. 定義した構造体のアドレスを PG_RETURN_POINTER マクロ経由で返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/catalog/pg_type.h: typsubscript 列のデフォルト値として参照。
  • src/backend/utils/cache/lsyscache.c: 配列型の判定時に参照。
  • src/backend/parser/parse_node.c および src/backend/commands/typecmds.c: 添字操作の解析や型定義の検証時に参照。
  • **呼び出し先**:
  • array_subscript_transform（構造体メンバとして参照）
  • array_exec_setup（構造体メンバとして参照）

関連データ構造

  • SubscriptRoutines: 添字操作の解析および実行に必要な関数ポインタを保持する構造体。

失敗モードと境界条件

  • 本関数自体は静的なポインタを返すのみであるため、関数実行時の失敗モードはありません。
  • ただし、このハンドラが提供する store_leakproof = false の設定により、代入操作時にエラーが発生する可能性があることが示唆されています。

保守担当者向けメモ

  • 本関数はSQLレベルで直接呼び出されることは想定されていません（typecmds.c 等で明示的な選択が禁止されています）。
  • raw_array_subscript_handler とは異なる標準配列専用のハンドラです。将来的に配列操作のロジックを分離する際は、このハンドラが参照する SubscriptRoutines の内容を維持する必要があります。
