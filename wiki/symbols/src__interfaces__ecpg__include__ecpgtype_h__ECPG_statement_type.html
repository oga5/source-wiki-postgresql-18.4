ECPG_statement_type

定義

  • 種別: enum
  • 定義箇所: src/interfaces/ecpg/include/ecpgtype.h:95

目的

ECPG（Embedded SQL for C）において、実行されるSQL文のコンテキストや実行形態を分類し、ランタイムライブラリ（ecpglib）が適切な処理を選択するための識別子として機能します。

シグネチャまたは形状

enum ECPG_statement_type
{
    ECPGst_normal,
    ECPGst_execute,
    ECPGst_exec_immediate,
    ECPGst_prepnormal,
    ECPGst_prepare,
    ECPGst_exec_with_exprlist
};

入力と出力

  • 入力: プリプロセッサがソースコード解析時に決定した実行形態。
  • 出力: ecpglib の関数（例: execute.c 内の処理）において、SQL文の実行パスを分岐させるための列挙値。

前提条件と不変条件

  • 本列挙型は、プリプロセッサが生成するCコード内で、SQL文の実行特性をランタイムライブラリへ伝達するメタデータとして使用されます。

アルゴリズムまたは振る舞い

  • ECPGst_normal: 通常のSQL文実行。
  • ECPGst_execute: 準備済みステートメントの実行。
  • ECPGst_exec_immediate: 即時実行されるSQL文。
  • ECPGst_prepnormal: プリペア処理を伴う通常実行。
  • ECPGst_prepare: SQL文の準備（PREPARE）。
  • ECPGst_exec_with_exprlist: 式リストを伴う実行。

呼び出し元と呼び出し先

  • 呼び出し元（プリプロセッサ側）: src/interfaces/ecpg/preproc/output.c の output_statement 関数。
  • 呼び出し先（ランタイムライブラリ側）: src/interfaces/ecpg/ecpglib/execute.c の実行関数群。
  • 参照元: src/interfaces/ecpg/ecpglib/ecpglib_extern.h にて外部変数として宣言。

関連データ構造

  • ECPGttype: ホスト変数の型定義。
  • ECPGdtype: SQL記述子の属性識別子。

失敗モードと境界条件

  • 該当なし（列挙型定義のため、本シンボル自体に実行時の失敗モードは存在しません）。

保守担当者向けメモ

  • 本列挙型に新しい実行形態を追加した場合は、必ず ecpglib 側の実行ロジック（execute.c 等）で対応する分岐処理を追加してください。
  • プリプロセッサの output_statement が生成するコードと、ライブラリ側の解釈が一致していることを確認してください。
