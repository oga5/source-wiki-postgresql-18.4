ExecForeignScanInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeForeignscan.c:417

目的

並列クエリ実行時、各ワーカープロセスにおいて外部テーブルスキャン（Foreign Scan）の初期化を行う。

シグネチャまたは形状

void ExecForeignScanInitializeWorker(ForeignScanState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 実行中の ForeignScanState 構造体。
  • pwcxt: 並列ワーカーのコンテキスト情報。
  • **出力**: なし（副作用としてFDW側の初期化処理を実行）。

前提条件と不変条件

  • node->fdwroutine が有効であること。
  • pwcxt->toc が有効な共有メモリテーブル（TOC）を指していること。

アルゴリズムまたは振る舞い

1. node->fdwroutine から InitializeWorkerForeignScan 関数ポインタを取得する。 2. 関数ポインタが定義されている場合、以下の手順で処理を実行する：

  • node からプランノードIDを取得する。
  • pwcxt->toc から当該プランノードIDに対応する共有メモリ上の coordinate（調整用データ）を検索する。
  • FDW固有の InitializeWorkerForeignScan 関数を呼び出し、node、pwcxt->toc、および coordinate を渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c 内の並列実行ロジックから呼び出される。
  • **呼び出し先**: node->fdwroutine->InitializeWorkerForeignScan（FDW実装依存の関数）。

関連データ構造

  • ForeignScanState: 外部スキャンの実行状態を保持する構造体。
  • FdwRoutine: 外部データソースごとの操作関数群を定義する構造体。
  • ParallelWorkerContext: 並列ワーカーの共有メモリ管理用コンテキスト。

失敗モードと境界条件

  • fdwroutine->InitializeWorkerForeignScan が定義されていない場合、本関数は何もしない。
  • shm_toc_lookup が指定された plan_node_id を見つけられない場合、shm_toc_lookup の仕様に従いエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、FDWが並列クエリをサポートする場合にのみ機能する。
  • 実際の初期化ロジックはすべて fdwroutine 経由で外部モジュールに委譲されているため、デバッグ時は対象のFDW実装を確認すること。
