quote_postgres

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:39

目的

SQL文の構築時に、ホスト変数の値をPostgreSQLが安全に解釈できる文字列形式に変換（クォート）します。

シグネチャまたは形状

static char *quote_postgres(char *arg, bool quote, int lineno);

入力と出力

  • **入力**:
  • arg: クォート対象の文字列（呼び出し元でメモリ確保されているもの）。
  • quote: クォート処理を行うかどうかのフラグ。
  • lineno: エラー発生時の行番号。
  • **出力**:
  • クォート済みの文字列（ecpg_alloc で確保された新しいメモリ領域）、または quote が false の場合は入力された arg そのもの。

前提条件と不変条件

  • quote が true の場合、関数内で ecpg_alloc を使用してメモリを確保し、入力された arg は ecpg_free で解放されます。
  • ターゲットデータベースの standard_conforming_strings 設定が不明なため、エスケープが必要な場合は常に E'' 構文（ESCAPE_STRING_SYNTAX）を使用します。

アルゴリズムまたは振る舞い

1. quote が false の場合、何もせず arg をそのまま返します。 2. quote が true の場合:

  • PQescapeString を使用して入力文字列をエスケープします。
  • エスケープ前後で長さが変わらない（特殊文字が含まれない）場合は、単一引用符 ' で囲みます。
  • 長さが変わる場合は、E を接頭辞として付与し、E'...' 形式で囲みます。
  • 処理完了後、元の arg を解放し、新しく確保した文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/execute.c 内の複数箇所（主にホスト変数の値をSQL文に埋め込む処理）。
  • **呼び出し先**: PQescapeString（libpq）、ecpg_alloc、ecpg_free。

関連データ構造

  • char *（文字列バッファ）

失敗モードと境界条件

  • ecpg_alloc がメモリ確保に失敗した場合、NULL を返します。
  • 入力文字列が NULL の場合の挙動については、本ソース excerpt 内では明示されていません。

保守担当者向けメモ

  • standard_conforming_strings 設定を考慮した安全なクォート処理を行うため、常に E'' 構文を優先する設計になっています。
  • ecpg_free(arg) が関数内で行われるため、呼び出し元は quote_postgres に渡した後の arg ポインタを再利用してはなりません。
