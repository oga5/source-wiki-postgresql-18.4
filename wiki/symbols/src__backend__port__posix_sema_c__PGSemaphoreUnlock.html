PGSemaphoreUnlock

定義

  • 種別: function
  • 定義箇所: src/backend/port/posix_sema.c:338

目的

POSIX セマフォを解放（アンロック）し、待機中のプロセスを起床させるための抽象化関数です。

シグネチャまたは形状

void PGSemaphoreUnlock(PGSemaphore sema)

入力と出力

  • **入力**: sema (型: PGSemaphore) - 解放対象のセマフォへのポインタ。
  • **出力**: なし（成功時）。失敗時は elog(FATAL, ...) によりプロセスを終了します。

前提条件と不変条件

  • sema は有効な PGSemaphore 構造体を参照している必要があります。
  • PG_SEM_REF(sema) マクロにより、内部の sem_t が正しく取得できる必要があります。

アルゴリズムまたは振る舞い

1. sem_post を呼び出し、セマフォの値をインクリメントします。 2. sem_post がシグナル割り込み（EINTR）により中断された場合、ループ内で再試行します。 3. sem_post がエラーを返した場合、FATAL レベルのログを出力してプロセスを停止します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/clog.c
  • src/backend/storage/ipc/procarray.c
  • src/backend/storage/lmgr/lwlock.c
  • その他、sysv_sema.c や win32_sema.c 等のプラットフォーム別実装内でも同名の関数が定義・利用されています。
  • **呼び出し先**:
  • sem_post (POSIX 標準ライブラリ関数)

関連データ構造

  • PGSemaphore: PGSemaphoreData 構造体へのポインタ。
  • PG_SEM_REF(x): PGSemaphore から内部の sem_t を抽出するマクロ。

失敗モードと境界条件

  • **シグナル割り込み**: sem_post が EINTR を返した場合、再試行により透過的に処理されます。
  • **システムエラー**: sem_post がその他のエラーを返した場合、回復不能なエラーとして FATAL で終了します。

保守担当者向けメモ

  • 本関数は POSIX セマフォ実装におけるラッパーです。プラットフォーム固有のセマフォ実装（System V や Windows）にも同名の関数が存在するため、修正時は対象のファイルを確認してください。
