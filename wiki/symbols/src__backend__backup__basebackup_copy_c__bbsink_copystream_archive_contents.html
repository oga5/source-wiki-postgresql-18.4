bbsink_copystream_archive_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_copy.c:182

目的

バックアップのアーカイブデータをクライアントへ転送し、必要に応じて進捗状況を報告する。

シグネチャまたは形状

static void bbsink_copystream_archive_contents(bbsink *sink, size_t len)

入力と出力

  • sink: bbsink_copystream 構造体へのポインタ。
  • len: 送信するデータのバイト数。
  • 出力: なし（副作用としてネットワーク経由でクライアントへデータを送信）。

前提条件と不変条件

  • sink は bbsink_copystream 型としてキャスト可能であること。
  • mysink->msgbuffer には、先頭にタイプバイトを付与するための十分な領域が確保されていること。

アルゴリズムまたは振る舞い

1. **データ送信**: mysink->send_to_client が真の場合、pq_putmessage を使用してタイプバイト 'd' を付与したデータをクライアントへ送信する。 2. **進捗チェックの判定**: 最後に進捗を確認した時点から PROGRESS_REPORT_BYTE_INTERVAL（64KB）以上のデータが転送されたかを確認する。 3. **進捗報告**: 閾値を超えている場合、現在時刻を取得し、前回の報告から PROGRESS_REPORT_MILLISECOND_THRESHOLD 以上経過しているか、またはシステム時刻が巻き戻った場合に、タイプバイト 'p' を付与した進捗メッセージを送信する。 4. **状態更新**: 進捗報告を行った場合、last_progress_report_time および bytes_done_at_last_time_check を更新する。

呼び出し元と呼び出し先

  • 呼び出し元: basebackup サブシステムのメインループ（bbsink_copystream_ops 構造体を通じて呼び出される）。
  • 呼び出し先: pq_putmessage, GetCurrentTimestamp, TimestampDifferenceMilliseconds, pq_beginmessage, pq_sendbyte, pq_sendint64, pq_endmessage, pq_flush_if_writable。

関連データ構造

  • bbsink_copystream: ストリーム状態や進捗管理用のタイマーを保持するメイン構造体。
  • bbsink_state: 転送済みバイト数などのバックアップ状態を保持する。

失敗モードと境界条件

  • PROGRESS_REPORT_BYTE_INTERVAL 未満のデータ転送では進捗報告はスキップされる。
  • システム時刻が巻き戻った場合、強制的に進捗報告が送信される。
  • pq_flush_if_writable により、ネットワーク書き込みの可否に応じたフラッシュが行われる。

保守担当者向けメモ

  • msgbuffer の先頭にタイプバイトを埋め込む際、アライメントを考慮したパディングが前提となっている点に注意すること。
  • 進捗報告の頻度制御は、システムコールを抑制するためのヒューリスティックな設計である。
