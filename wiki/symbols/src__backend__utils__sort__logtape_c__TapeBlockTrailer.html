TapeBlockTrailer

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/logtape.c:95

目的

論理テープを構成する各ブロックの末尾に配置され、ブロック間のチェーン関係およびデータの有効長を記録するためのメタデータ構造体です。

シグネチャまたは形状

typedef struct TapeBlockTrailer
{
    int64 prev; /* 前のブロック番号、または最初のブロックの場合は -1 */
    int64 next; /* 次のブロック番号、または最終ブロックの場合は有効バイト数（負の値） */
} TapeBlockTrailer;

入力と出力

本構造体はデータ構造であり、関数ではありません。物理ファイル上の各ブロックの末尾（BLCKSZ の境界）に配置されます。

前提条件と不変条件

  • 各ブロックの末尾に配置される必要があります。
  • TapeBlockPayloadSize（BLCKSZ - sizeof(TapeBlockTrailer)）によって、ブロック内のデータ領域サイズが定義されます。
  • next フィールドが負の値である場合、そのブロックが論理テープの末尾であることを示し、その値は有効なバイト数として解釈されます。

アルゴリズムまたは振る舞い

  • ブロックの読み書き時に、logtape.c 内のロジックによってブロック末尾のメモリ領域が本構造体としてキャストされ、チェーンの追跡やデータ長の判定に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数名は、提供されたソース抜粋からは特定できません。
  • logtape.c 内のブロック管理ロジック（ltsGetBlock, ltsWriteBlock 等）が、この構造体を通じてブロックの連結を制御していることが示唆されています。

関連データ構造

  • LogicalTape: 論理テープの状態を管理する構造体。
  • LogicalTapeSet: 複数の論理テープを管理する構造体。

失敗モードと境界条件

  • ブロックサイズ（BLCKSZ）が TapeBlockTrailer のサイズよりも小さい場合、構造体の配置が物理的に不可能となります。
  • next フィールドの負の値の解釈において、有効バイト数が正しく設定されていない場合、読み込み時にデータ破損や境界外アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は BLCKSZ 境界に依存しているため、メモリダンプやデバッグを行う際は、ブロックの末尾に配置されていることを考慮してください。
  • next フィールドが「次のブロック番号」と「最終ブロックの有効バイト数」という2つの役割を兼ねている点に注意してください。
