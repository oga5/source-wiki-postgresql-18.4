GlobalTransaction

定義

  • 種別: typedef (不透明ポインタ)
  • 定義箇所: src/include/access/twophase.h:26

目的

PostgreSQLの2フェーズコミット（2PC）において、準備済みトランザクション（Prepared Transaction）を共有メモリ上で抽象化・追跡するためのハンドル。

シグネチャまたは形状

typedef struct GlobalTransactionData *GlobalTransaction;

実体である GlobalTransactionData は src/backend/access/transam/twophase.c に隠蔽されています。

入力と出力

  • 入力: 共有メモリ上の準備済みトランザクション領域へのポインタ。
  • 出力: 準備済みトランザクションの状態管理および識別（GID）へのアクセスを提供。

前提条件と不変条件

  • max_prepared_xacts で指定された上限数以内で管理される必要があります。
  • 準備済みトランザクションは、コミットまたはアボートが完了するまでロックを保持し続けるため、システム全体のリソース消費に影響を与えます。

アルゴリズムまたは振る舞い

  • MarkAsPreparing() を通じて共有メモリ上のエントリが確保されます。
  • twophase.c 内の RemoveGXact() 等により、トランザクションのライフサイクル（準備から終了まで）が管理されます。
  • チェックポイント時には、この構造体を通じて状態がディスク（pg_twophase）へ永続化されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/twophase.c 内の各関数（RemoveGXact, MarkAsPreparingGuts 等）。
  • 呼び出し先: 外部からは直接操作せず、twophase.c が提供するインターフェースを介して操作されます。

関連データ構造

  • GlobalTransactionData: 本ポインタが指す実体（twophase.c で定義）。
  • max_prepared_xacts: 同時存在可能な準備済みトランザクションの最大数を制御するGUC変数。

失敗モードと境界条件

  • max_prepared_xacts を超えて準備済みトランザクションを作成しようとした場合、システムはエラーを返します。
  • 準備済みトランザクションが長期間放置されると、ロック競合やリソース枯渇を引き起こす可能性があります。

保守担当者向けメモ

  • 内部構造は twophase.c に隠蔽されているため、変更が必要な場合は同ファイルを確認してください。
  • リカバリ時の挙動を理解するには、twophase.c 冒頭の「Life track of state data」の解説を参照してください。
