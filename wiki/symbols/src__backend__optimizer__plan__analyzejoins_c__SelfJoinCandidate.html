SelfJoinCandidate

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:61

目的

自己結合排除（Self-Join Elimination）の対象となるリレーションを特定し、管理するための構造体です。

シグネチャまたは形状

typedef struct
{
	int			relid;
	Oid			reloid;
} SelfJoinCandidate;

入力と出力

本構造体は、クエリ内のリレーションを識別する以下のフィールドを保持します。

  • relid: プランナ内部で使用されるリレーションの識別子（int）。
  • reloid: システムカタログ上のリレーション（テーブル）のOID（Oid）。

前提条件と不変条件

  • 自己結合排除の候補として収集される際、relid と reloid のペアとして保持されます。
  • 複数の候補をソートして比較するために使用されます。

アルゴリズムまたは振る舞い

  • remove_useless_self_joins 関数内で、自己結合排除の候補をリスト化するために使用されます。
  • palloc を用いてメモリ確保され、候補リストのソート（qsort）時に、relid や reloid を基準とした比較処理の対象となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/plan/analyzejoins.c 内の自己結合排除処理ルーチン。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • PlannerInfo: クエリ全体のプランニング情報を管理する構造体。
  • RelOptInfo: プランナがリレーションを扱う際に使用する構造体。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • 候補リストが空の場合、自己結合排除は実行されません。

保守担当者向けメモ

  • 本構造体は自己結合排除の判定ロジックにおいて、リレーションの同一性を検証するためのキーとして機能します。
  • qsort を使用した比較関数（analyzejoins.c:2524-2525 付近）でキャストして使用されるため、比較ロジックを変更する際は、この構造体のフィールド定義と整合性を取る必要があります。
