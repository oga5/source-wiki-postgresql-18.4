hex_encode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:180

目的

バイナリデータを16進数文字列（HEX形式）に変換します。

シグネチャまたは形状

uint64 hex_encode(const char *src, size_t len, char *dst)

入力と出力

  • **入力**:
  • src: 変換対象のバイナリデータへのポインタ。
  • len: 変換対象データのバイト長。
  • dst: 変換後の文字列を格納するバッファへのポインタ（十分なサイズが確保されている必要がある）。
  • **出力**:
  • 変換後の文字列長（len * 2 を uint64 型で返す）。

前提条件と不変条件

  • dst バッファは、入力データの2倍のサイズ（len * 2）以上である必要があります。
  • hextbl が有効な16進数変換テーブルとして参照可能であること。

アルゴリズムまたは振る舞い

1. 入力バッファの開始位置から終了位置まで、1バイトずつループ処理を行います。 2. 各バイトを unsigned char として読み込み、hextbl を参照して対応する2文字の16進数表現を取得します。 3. memcpy を使用して、取得した2文字を dst バッファに書き込みます。 4. 処理したバイト数に応じて src ポインタと dst ポインタを進めます。 5. 最後に変換後の総バイト数（len * 2）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/backup_manifest.c
  • src/backend/utils/adt/varlena.c
  • src/bin/pg_combinebackup/write_manifest.c
  • **呼び出し先**:
  • memcpy（標準ライブラリ）

関連データ構造

  • hextbl: 16進数変換用の静的テーブル（本関数内で参照）。

失敗モードと境界条件

  • 本関数自体はメモリ確保を行わないため、dst バッファのサイズ不足によるバッファオーバーフローのリスクがあります。呼び出し側で事前に必要なサイズを計算し、確保する必要があります。

保守担当者向けメモ

  • 本関数は src/backend/utils/adt/encode.c 内の enclist に登録されており、binary_encode 経由でSQL関数から利用されます。
  • src/bin/pg_combinebackup/write_manifest.c にも同名の関数定義が存在しますが、本ページは src/backend/utils/adt/encode.c の実装を対象としています。
