win32_transfer_all_new_dbs

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/parallel.c:262

目的

Windows 環境において、データベースのデータ転送処理を別スレッドで実行するためのスレッドエントリーポイント関数です。

シグネチャまたは形状

DWORD win32_transfer_all_new_dbs(transfer_thread_arg *args)

入力と出力

  • **入力**: transfer_thread_arg *args - データベース転送に必要な情報（データベース配列、パス情報、テーブルスペース情報）を保持する構造体へのポインタ。
  • **出力**: DWORD - スレッド終了コードとして常に 0 を返します。

前提条件と不変条件

  • 引数 args は有効な transfer_thread_arg 構造体を指している必要があります。
  • 本関数は Windows の _beginthreadex から呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. 受け取った args 構造体の各メンバ（old_db_arr, new_db_arr, old_pgdata, new_pgdata, old_tablespace）を引数として transfer_all_new_dbs を呼び出します。 2. transfer_all_new_dbs の完了後、0 を返してスレッドを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/parallel.c 内の _beginthreadex 呼び出し箇所（250行目）。
  • **呼び出し先**: transfer_all_new_dbs。

関連データ構造

  • transfer_thread_arg: スレッド間で受け渡されるデータベース転送用の引数構造体。

失敗モードと境界条件

  • transfer_all_new_dbs 内部で発生したエラーは、当該関数の仕様に従って処理されます。本関数自体は戻り値として常に 0 を返します。

保守担当者向けメモ

  • 本関数は Windows 特有の実装です。Unix 系環境では fork を使用した別のアプローチが取られるため、プラットフォーム間の差異に注意してください。
  • スレッド終了時に 0 を返すことで、Windows のスレッド管理機構に対して正常終了を通知しています。
