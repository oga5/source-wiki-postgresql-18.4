jit_compile_expr

定義

  • 種別: function
  • 定義箇所: src/backend/jit/jit.c:150

目的

実行エンジンから要求された ExprState に対し、JIT コンパイルを試行して実行可能な形式へ変換するメインのエントリポイントです。

シグネチャまたは形状

bool jit_compile_expr(struct ExprState *state)

入力と出力

  • **入力**: state (struct ExprState *) - コンパイル対象の式状態。
  • **出力**: bool - コンパイルが成功した場合は true、それ以外（条件不適合やプロバイダーの初期化失敗を含む）は false を返します。

前提条件と不変条件

  • state->parent が存在すること（PlanState に関連付けられていない式は、メモリ管理上の理由から JIT 対象外となります）。
  • 実行環境の es_jit_flags に PGJIT_PERFORM および PGJIT_EXPR フラグが設定されていること。
  • jit_enabled GUC が有効であること（内部的に provider_init() が考慮します）。

アルゴリズムまたは振る舞い

1. **ガード条件の確認**: state->parent の有無を確認し、存在しない場合は false を返します。 2. **フラグチェック**: es_jit_flags を参照し、JIT 実行および式コンパイルが許可されているかを確認します。許可されていない場合は false を返します。 3. **プロバイダー初期化**: provider_init() を呼び出し、JIT プロバイダーが利用可能か確認します。 4. **委譲**: プロバイダーが初期化済みであれば、provider.compile_expr(state) を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExpr.c:904
  • **呼び出し先**: provider_init() および provider.compile_expr()（provider 構造体に格納された関数ポインタ）。

関連データ構造

  • ExprState: コンパイル対象の式状態。
  • PlanState: ExprState の親として参照される実行計画状態。
  • provider: ロードされた JIT プロバイダーの関数ポインタを保持する構造体。

失敗モードと境界条件

  • state->parent が NULL の場合、メモリリークやデバッグ時の挙動悪化を防ぐため、コンパイルを拒否します。
  • provider_init() が失敗した場合、コンパイルは行われず false が返されます。
  • es_jit_flags が不適合な場合、コンパイルは行われません。

保守担当者向けメモ

  • 本関数は LLVM 等の具体的なコンパイルロジックを持たず、あくまでプロバイダーへのディスパッチャーとして機能します。
  • provider_init() の呼び出しにより、初回実行時に共有ライブラリのロードが発生する可能性があります。
