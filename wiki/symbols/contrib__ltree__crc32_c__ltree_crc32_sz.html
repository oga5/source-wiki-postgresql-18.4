ltree_crc32_sz

定義

  • **種別**: 関数
  • **定義箇所**: contrib/ltree/crc32.c:22

目的

指定されたバイト列に対してCRC-32チェックサムを計算します。主にltreeモジュールのGiSTインデックスにおけるシグネチャ生成や、データの一貫性確保のために使用されます。

シグネチャまたは形状

unsigned int ltree_crc32_sz(const char *buf, int size);

入力と出力

  • **入力**:
  • buf: 計算対象の文字列バッファへのポインタ。
  • size: バッファのバイトサイズ。
  • **出力**:
  • 計算されたCRC-32値をunsigned int型として返却します。

前提条件と不変条件

  • DEFAULT_COLLATION_OIDに基づくロケール情報が内部的に保持されます。
  • インデックスのオンディスク形式と密接に関連しているため、アルゴリズムの変更は既存インデックスの破損を招く可能性があります。

アルゴリズムまたは振る舞い

1. **初期化**: INIT_TRADITIONAL_CRC32マクロでCRC状態を初期化します。 2. **ロケール判定**: ロケールがCロケール（ctype_is_c）の場合、各文字をpg_ascii_tolowerで小文字化しつつCRCを計算します。 3. **マルチバイト対応**: Cロケール以外の場合、pg_strfoldを使用してコードポイント単位で小文字化（正規化）を行い、その結果に対してCRCを計算します。 4. **終了**: FIN_TRADITIONAL_CRC32で最終値を確定させ、呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/_ltree_gist.c
  • contrib/ltree/ltree_gist.c
  • contrib/ltree/ltree_io.c
  • contrib/ltree/ltxtquery_io.c
  • **呼び出し先**:
  • pg_newlocale_from_collation
  • pg_ascii_tolower
  • pg_mblen_range
  • pg_strfold
  • CRC計算用マクロ群 (INIT_TRADITIONAL_CRC32, COMP_TRADITIONAL_CRC32, FIN_TRADITIONAL_CRC32)

関連データ構造

  • pg_crc32: CRC計算に使用される型。
  • pg_locale_t: ロケール情報を保持する構造体。

失敗モードと境界条件

  • sizeが0以下の場合、ループ処理は実行されず初期化されたCRC値が返されます。
  • マルチバイト文字の処理において、UNICODE_CASEMAP_BUFSZを超える変換結果が生じる場合の挙動は、使用するライブラリの仕様に依存します。

保守担当者向けメモ

  • 本関数はcontrib/ltree/crc32.hで定義されるcrc32マクロ経由でも利用されます。
  • ロケール依存の処理が含まれるため、ビルド環境やデータベースの照合順序設定によって計算結果が変化する可能性があります。
