dintdict_lexize

定義

  • 種別: function
  • 定義箇所: contrib/dict_int/dict_int.c:79

目的

PostgreSQL の全文検索エンジンにおいて、入力された整数トークンを辞書設定（最大長、符号の扱い）に基づいて正規化またはフィルタリングする。

シグネチャまたは形状

Datum dintdict_lexize(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): DictInt *（辞書設定構造体）
  • PG_GETARG_POINTER(1): char *（入力トークン文字列）
  • PG_GETARG_INT32(2): int（トークンの長さ）
  • **出力**:
  • Datum: TSLexeme * 型の配列へのポインタ。配列の第1要素に正規化後の文字列、第2要素に NULL（終端）が格納される。

前提条件と不変条件

  • palloc0 で確保された TSLexeme 配列は、呼び出し元の全文検索エンジン側で解放されることが前提。
  • 入力トークンが整数であることは呼び出し側で保証されている必要がある。

アルゴリズムまたは振る舞い

1. TSLexeme 配列（サイズ2）をメモリ確保し、終端を NULL に設定する。 2. d->absval が真かつトークンが符号（+ または -）で始まる場合、符号を除去して文字列をコピーする。それ以外はそのままコピーする。 3. 文字列長が d->maxlen を超える場合:

  • d->rejectlong が真なら、メモリを解放し、結果の第1要素を NULL に設定して除外する。
  • d->rejectlong が偽なら、d->maxlen で文字列を切り詰める。

    • 4. 処理結果を TSLexeme 配列に格納し、ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dict_int/dict_int.c:32: PG_FUNCTION_INFO_V1 マクロにより登録されている。
  • **呼び出し先**:
  • PostgreSQL のメモリ管理関数 (palloc0, pfree) および文字列操作関数 (pnstrdup) を使用。

関連データ構造

  • DictInt: 辞書の設定（maxlen, rejectlong, absval）を保持する構造体。
  • TSLexeme: 全文検索エンジンが期待するトークン結果構造体。

失敗モードと境界条件

  • maxlen を超えるトークンに対し rejectlong が設定されている場合、トークンは破棄され NULL が返される。
  • メモリ確保失敗時は palloc0 がエラーを発生させる。

保守担当者向けメモ

  • 本関数は TSLexeme 配列を動的に確保するため、メモリリークを防ぐために TSearch エンジン側のライフサイクル管理に依存している点に注意すること。
  • 整数値のバリデーションは本関数内では行わないため、入力データの妥当性は呼び出し側のトークナイザーに依存する。
