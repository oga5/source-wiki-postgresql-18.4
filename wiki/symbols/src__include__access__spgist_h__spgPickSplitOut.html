spgPickSplitOut

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist.h:117

目的

SP-GiSTインデックスにおいて、ノード分割（PickSplit）操作の結果を格納し、新しい内部タプル（inner tuple）の構成およびリーフタプルの再配置を定義するために使用されます。

シグネチャまたは形状

typedef struct spgPickSplitOut
{
    bool        hasPrefix;        /* 新しい内部タプルがプレフィックスを持つか */
    Datum       prefixDatum;      /* プレフィックスを持つ場合のその値 */

    int         nNodes;           /* 新しい内部タプルのノード数 */
    Datum      *nodeLabels;       /* 各ノードのラベル（ラベルなしの場合はNULL） */

    int        *mapTuplesToNodes; /* 各リーフタプルの割り当て先ノードインデックス */
    Datum      *leafTupleDatums;  /* 各新しいリーフタプルに格納するDatum */
} spgPickSplitOut;

入力と出力

本構造体は、SP-GiSTの演算子クラスが実装する picksplit サポート関数から出力されるデータ構造です。入力側である spgPickSplitIn と対になり、分割後のインデックス構造を決定します。

前提条件と不変条件

  • nNodes は作成されるノードの数を指定します。
  • nodeLabels は nNodes 分の配列である必要があります。
  • mapTuplesToNodes は、分割対象となる各リーフタプルがどのノードに移動するかを示すインデックス（0から nNodes - 1）を保持します。

アルゴリズムまたは振る舞い

1. hasPrefix が真の場合、prefixDatum に指定された値が内部タプルの共通プレフィックスとして設定されます。 2. nNodes に基づいて新しい内部タプルが作成され、各ノードには nodeLabels が割り当てられます。 3. mapTuplesToNodes を使用して、元のリーフタプルが新しいノードのいずれかに再配置されます。 4. leafTupleDatums には、再配置後の各リーフタプルに格納すべきデータが保持されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c 内の checkAllTheSame 関数や、インデックス挿入処理の過程で呼び出されます。
  • **呼び出し先**: 各データ型固有の演算子クラス実装（例: spgkdtreeproc.c, spgquadtreeproc.c, spgtextproc.c, geo_spgist.c, network_spgist.c, rangetypes_spgist.c）において、この構造体に値を書き込む形で利用されます。

関連データ構造

  • spgPickSplitIn: 分割対象の入力データを提供する構造体。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • nNodes が0以下の場合や、mapTuplesToNodes が範囲外のインデックスを参照する場合の挙動は、呼び出し側の実装に依存します。
  • メモリ割り当てに関する失敗モードについては、PostgreSQLの標準的なメモリ管理（palloc等）の制約を受けます。

保守担当者向けメモ

  • 本構造体はSP-GiSTのインデックス構造の根幹に関わるため、変更時は spgdoinsert.c 側の処理ロジックとの整合性に注意してください。
  • nodeLabels が NULL の場合、ラベルを持たないノードとして扱われます。
