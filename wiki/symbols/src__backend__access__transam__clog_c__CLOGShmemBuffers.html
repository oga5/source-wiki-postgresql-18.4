CLOGShmemBuffers

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:767

目的

CLOG（Commit Log）が使用する共有メモリバッファ数を決定する。

シグネチャまたは形状

int CLOGShmemBuffers(void)

入力と出力

  • 入力: なし（グローバル変数 transaction_buffers を参照）
  • 出力: 確保すべきバッファ数（int 型）

前提条件と不変条件

  • transaction_buffers が 0 の場合、自動チューニングが適用される。
  • 返り値は CLOG_MAX_ALLOWED_BUFFERS を超えない。
  • 返り値は最低 16 バッファ以上となるよう制約される。

アルゴリズムまたは振る舞い

1. transaction_buffers が 0 の場合、SimpleLruAutotuneBuffers(512, 1024) を呼び出し、共有バッファ数に基づいた値を算出する。 2. transaction_buffers が設定されている場合、16 と transaction_buffers の大きい方を選択し、さらに CLOG_MAX_ALLOWED_BUFFERS との最小値をとる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/clog.c: CLOGShmemSize 内での共有メモリサイズ計算時
  • src/backend/access/transam/clog.c: GUC パラメータ表示用
  • src/backend/access/transam/clog.c: CLOGShmemInit 内での初期化時
  • 呼び出し先:
  • SimpleLruAutotuneBuffers (条件付き)
  • Min, Max (マクロ)

関連データ構造

  • transaction_buffers: バッファ数を制御するグローバル変数。
  • CLOG_MAX_ALLOWED_BUFFERS: バッファ数の上限定数。

失敗モードと境界条件

  • transaction_buffers が極端に小さい値であっても、内部の Max(16, ...) により最低 16 バッファが保証される。

保守担当者向けメモ

  • 本関数は共有メモリ割り当ての初期段階で呼び出されるため、計算ロジックを変更する場合は CLOGShmemSize への影響を考慮すること。
