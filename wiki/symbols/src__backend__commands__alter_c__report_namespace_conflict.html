report_namespace_conflict

定義

  • 種別: function
  • 定義箇所: src/backend/commands/alter.c:110

目的

指定されたオブジェクトが既にスキーマ内に存在する場合に、重複エラー（ERRCODE_DUPLICATE_OBJECT）を報告する。

シグネチャまたは形状

static void report_namespace_conflict(Oid classId, const char *name, Oid nspOid);

入力と出力

  • classId (Oid): 対象オブジェクトのシステムカタログOID。
  • name (const char *): オブジェクト名。
  • nspOid (Oid): 対象スキーマのOID。
  • 出力: なし。エラー発生時に ereport(ERROR, ...) を呼び出し、処理を中断する。

前提条件と不変条件

  • nspOid は有効なOIDであること（Assert(OidIsValid(nspOid)) により検証）。
  • classId はサポートされているオブジェクト型（Conversion, StatisticExt, TSParser, TSDictionary, TSTemplate, TSConfig）のいずれかであること。

アルゴリズムまたは振る舞い

1. classId に基づき、対応するエラーメッセージのフォーマット文字列を選択する。 2. サポート外の classId が渡された場合は elog(ERROR) で異常終了する。 3. get_namespace_name(nspOid) を呼び出し、スキーマ名を取得する。 4. ereport を使用して、ERRCODE_DUPLICATE_OBJECT を伴うエラーを発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/alter.c 内の複数箇所（オブジェクトの変更処理中に名前の重複を検知した際）。
  • 呼び出し先: get_namespace_name（utils/lsyscache.h）。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • ConversionRelationId, StatisticExtRelationId, TSParserRelationId, TSDictionaryRelationId, TSTemplateRelationId, TSConfigRelationId: classId の判定に使用されるシステムカタログのOID定数。

失敗モードと境界条件

  • classId がサポート対象外の場合: elog(ERROR) によりサーバープロセスが停止する。
  • nspOid が無効な場合: Assert によりデバッグビルドで停止する。
  • スキーマ名が取得できない場合: get_namespace_name の挙動に依存する。

保守担当者向けメモ

  • 本関数は alter.c 内の汎用的な名前変更やスキーマ移動処理において、重複チェックの共通ロジックとして利用されている。
  • 新しいオブジェクト型を本関数でサポートする場合は、switch 文に新しい case を追加し、適切なエラーメッセージを定義する必要がある。
