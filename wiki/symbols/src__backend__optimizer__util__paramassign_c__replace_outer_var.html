replace_outer_var

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:120

目的

上位レベルのクエリで定義された Var を、実行時に参照可能な PARAM_EXEC 型のパラメータノードに置換します。

シグネチャまたは形状

Param *replace_outer_var(PlannerInfo *root, Var *var)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo 構造体。
  • var: 置換対象となる上位レベルの Var ノード。
  • **出力**:
  • Param: PARAM_EXEC として設定された新しいパラメータノード。

前提条件と不変条件

  • var->varlevelsup > 0 かつ var->varlevelsup < root->query_level であること（上位レベルの変数であること）。
  • assign_param_for_var を介して、対象変数が root->glob->paramExecTypes に登録済み、または登録可能であること。

アルゴリズムまたは振る舞い

1. assign_param_for_var(root, var) を呼び出し、対象変数を plan_params リストで検索または新規登録し、対応するパラメータIDを取得します。 2. makeNode(Param) で新しい Param ノードを生成します。 3. paramkind を PARAM_EXEC に設定します。 4. 取得したパラメータID、および元の Var の型情報（vartype, vartypmod, varcollid）と location を新しい Param ノードにコピーします。 5. 生成した Param ノードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/subselect.c:1985
  • **呼び出し先**:
  • assign_param_for_var (同ファイル内)
  • makeNode (マクロ)

関連データ構造

  • PlannerInfo: クエリのプランニング状態を保持する構造体。
  • Var: 置換対象の変数ノード。
  • Param: 生成される実行時パラメータノード。

失敗モードと境界条件

  • varlevelsup が現在のクエリレベルと一致またはそれ以下の場合、Assert により停止します。
  • assign_param_for_var が失敗した場合、その依存関数に準じたエラーが発生します。

保守担当者向けメモ

  • 本関数は Var 専用です。PlaceHolderVar に対しては replace_outer_placeholdervar を使用してください。
  • PARAM_EXEC ID はプラン全体で一意である必要があるため、assign_param_for_var による重複排除のロジックに依存しています。
