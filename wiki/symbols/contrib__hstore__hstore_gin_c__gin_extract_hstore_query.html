gin_extract_hstore_query

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_gin.c:81

目的

GIN 索引検索において、クエリ演算子（@>, ?, ?|, ?&）に基づき、インデックス検索に使用するキーのリストを抽出する。

シグネチャまたは形状

Datum gin_extract_hstore_query(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_DATUM(0): クエリ条件（hstore または text[]）。
  • PG_GETARG_POINTER(1): 抽出されたキーの数を格納する int32 ポインタ。
  • PG_GETARG_UINT16(2): 演算子の戦略番号（StrategyNumber）。
  • PG_GETARG_POINTER(6): 検索モードを制御する int32 ポインタ。
  • **出力**:
  • 戻り値: 抽出されたキーの Datum 配列。
  • 副作用: nentries にキー数、searchMode に必要に応じて GIN_SEARCH_MODE_ALL を設定。

前提条件と不変条件

  • strategy が HStoreContainsStrategyNumber の場合、gin_extract_hstore を呼び出してキーを抽出する。
  • strategy が HStoreExistsAllStrategyNumber でキーが空の場合、全件検索モード（GIN_SEARCH_MODE_ALL）へ遷移する。
  • インデックス項目には KEYFLAG を付与して識別する。

アルゴリズムまたは振る舞い

1. **Contains (@>)**: gin_extract_hstore を再利用してクエリ hstore からキーと値を抽出する。結果が空（{}）の場合は searchMode を GIN_SEARCH_MODE_ALL に設定する。 2. **Exists (?)**: 単一の text キーを抽出し、makeitem で KEYFLAG を付与して返す。 3. **ExistsAny/All (?|, ?&)**: text[] 配列を分解し、各要素を KEYFLAG 付きの text 項目に変換する。ExistsAll でキーが空の場合は GIN_SEARCH_MODE_ALL を設定する。 4. **エラー処理**: 未知の戦略番号が渡された場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GIN アクセスメソッド（直接の呼び出し元情報はソースコード上では明示されていない）。
  • **呼び出し先**: gin_extract_hstore, makeitem, deconstruct_array_builtin。

関連データ構造

  • StrategyNumber: 演算子の種類を識別。
  • KEYFLAG: インデックス項目をキーとして識別するためのフラグ。
  • GIN_SEARCH_MODE_ALL: 全件走査が必要な場合に指定する検索モード。

失敗モードと境界条件

  • 未知の戦略番号が指定された場合、エラーとなる。
  • contains {} のような空のクエリは、インデックス検索では解決できないため GIN_SEARCH_MODE_ALL が設定される。
  • ExistsAll でキーが空の場合も同様に全件検索となる。

保守担当者向けメモ

  • 本関数は hstore の GIN 索引におけるクエリ解析の入り口である。
  • インデックス項目に付与されるフラグ（KEYFLAG）は、ディスクフォーマットの互換性を維持するために重要である。
  • recheck の要否は本関数ではなく、別途 gin_consistent_hstore で判定される。
