GistPageGetDeleteXid

定義

  • 種別: function
  • 定義箇所: src/include/access/gist.h:217

目的

削除済みとしてマークされたGiSTインデックスページから、そのページが削除された際のトランザクションID（deleteXid）を取得します。

シグネチャまたは形状

static inline FullTransactionId GistPageGetDeleteXid(Page page)

入力と出力

  • **入力**: Page page（GiSTインデックスページへのポインタ）
  • **出力**: FullTransactionId（ページ削除時のトランザクションID。情報が存在しない場合は FirstNormalTransactionId を含むデフォルト値）

前提条件と不変条件

  • 対象の page は、GistPageIsDeleted(page) が真である（削除済み状態である）必要があります。

アルゴリズムまたは振る舞い

1. GistPageIsDeleted(page) を用いて、ページが削除済みであることをアサートします。 2. ページヘッダーの pd_lower 値をチェックし、GISTDeletedPageContents 構造体の deleteXid フィールドがページ内に格納されているかを確認します。 3. フィールドが存在する場合、PageGetContents(page) を通じて GISTDeletedPageContents にキャストし、deleteXid を返します。 4. フィールドが存在しない場合、FullTransactionIdFromEpochAndXid(0, FirstNormalTransactionId) を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistutil.c:867 (gistXLogPageReuse 内)
  • src/backend/access/gist/gistutil.c:904
  • **呼び出し先**:
  • GistPageIsDeleted (マクロ)
  • PageGetContents (マクロ)
  • FullTransactionIdFromEpochAndXid

関連データ構造

  • GISTDeletedPageContents: 削除済みページの内容を保持する構造体。
  • FullTransactionId: トランザクションIDを保持する型。
  • PageHeader: ページヘッダー構造体。

失敗モードと境界条件

  • GistPageIsDeleted(page) が偽の場合、アサーションエラーが発生します。
  • ページ内の空き領域（pd_lower）が不足しており、deleteXid フィールドが書き込まれていないと判断された場合は、デフォルトのトランザクションIDが返されます。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、GiSTインデックスのページ再利用処理において、削除されたページの追跡に使用されます。
  • ページレイアウトの変更や GISTDeletedPageContents の構造変更を行う際は、pd_lower による存在判定ロジックが正しく機能するか確認してください。
