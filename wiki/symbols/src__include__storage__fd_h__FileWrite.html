FileWrite

定義

  • 種別: function
  • 定義箇所: src/include/storage/fd.h:221

目的

仮想ファイル記述子（VFD）を介して、指定されたオフセット位置からファイルへデータを書き込むためのラッパー関数です。

シグネチャまたは形状

ssize_t FileWrite(File file, const void *buffer, size_t amount, off_t offset, uint32 wait_event_info);

入力と出力

  • **入力**:
  • file: 書き込み対象の File 型（VFD）。
  • buffer: 書き込むデータが格納されたメモリ領域。
  • amount: 書き込むバイト数。
  • offset: 書き込みを開始するファイル内のオフセット位置。
  • wait_event_info: I/O待機イベントの識別情報。
  • **出力**:
  • 成功時: 書き込まれたバイト数を返します。
  • 失敗時: -1 を返し、errno が設定されます。

前提条件と不変条件

  • file は有効な VFD である必要があります。
  • VFD管理システムにおいて、ファイルポインタの概念は存在しないため、offset の指定は必須です。

アルゴリズムまたは振る舞い

1. iovec 構造体を初期化し、buffer と amount を設定します。 2. 内部的に FileWriteV を呼び出し、単一のベクトルとして書き込み処理を実行します。 3. wait_event_info を使用して、I/O待機中のイベント追跡を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/rewriteheap.c
  • src/backend/backup/basebackup_server.c
  • src/backend/backup/walsummary.c
  • src/backend/storage/file/buffile.c
  • src/backend/storage/smgr/md.c
  • **呼び出し先**:
  • FileWriteV（src/backend/storage/file/fd.c 内で定義）

関連データ構造

  • File: VFDを表す整数型。
  • iovec: 書き込みデータのバッファ配置を定義する構造体。

失敗モードと境界条件

  • VFDの割り当て上限やOS側のファイル記述子制限により、書き込みが失敗する可能性があります。
  • offset がファイルサイズを超えている場合や、ディスク容量不足などのOSレベルのエラーが発生した場合は、FileWriteV の挙動に従い -1 を返します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、FileWriteV への単なるインターフェース変換層です。
  • PostgreSQL内では標準の write(2) を直接使用せず、必ず本関数または FileWriteV を使用してください。
  • wait_event_info には適切な待機イベント定数を指定し、監視ツールでの可視性を確保してください。
