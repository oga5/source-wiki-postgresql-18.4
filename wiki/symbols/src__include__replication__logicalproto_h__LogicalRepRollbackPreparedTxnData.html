LogicalRepRollbackPreparedTxnData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/logicalproto.h:173

目的

論理レプリケーションにおいて、準備済みトランザクション（Prepared Transaction）のロールバック情報を保持し、パブリッシャーからサブスクライバーへ伝達するためのデータ構造です。

シグネチャまたは形状

typedef struct LogicalRepRollbackPreparedTxnData
{
    XLogRecPtr    prepare_end_lsn;
    XLogRecPtr    rollback_end_lsn;
    TimestampTz   prepare_time;
    TimestampTz   rollback_time;
    TransactionId xid;
    char          gid[GIDSIZE];
} LogicalRepRollbackPreparedTxnData;

入力と出力

本構造体は、論理レプリケーションのワイヤプロトコルにおけるメッセージのデコード結果を格納するために使用されます。

  • **入力**: ストリームから読み取られたバイナリデータが、この構造体の各フィールドにマッピングされます。
  • **出力**: レプリケーションワーカーがこの構造体を参照し、ローカル側で対応する準備済みトランザクションのロールバックを実行します。

前提条件と不変条件

  • gid フィールドは GIDSIZE で定義された固定長配列であり、グローバルなトランザクション識別子を保持します。
  • prepare_end_lsn および rollback_end_lsn は、WAL上の位置を指す有効な XLogRecPtr である必要があります。

アルゴリズムまたは振る舞い

論理レプリケーションのワーカープロセスにおいて、パブリッシャーから送信されたロールバックメッセージを受信した際、この構造体にデータがデシリアライズされます。その後、ワーカーはこの情報を使用して、ローカルノード上の該当する準備済みトランザクションを特定し、ロールバック処理を適用します。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/backend/replication/logical/worker.c: 受信したロールバックデータを処理する際に本構造体を使用。
  • src/backend/replication/logical/proto.c: プロトコルデータの読み取り/書き込み処理において本構造体を参照。
  • **呼び出し先**: 構造体であるため、直接的な呼び出し先はありません。

関連データ構造

  • TransactionId: トランザクション識別子。
  • XLogRecPtr: WAL上の位置情報。
  • TimestampTz: タイムスタンプ情報。

失敗モードと境界条件

  • gid が GIDSIZE を超える場合、または不正な形式である場合、デシリアライズ時にエラーが発生する可能性があります。
  • 該当する xid や gid を持つ準備済みトランザクションがローカル側に存在しない場合、レプリケーションワーカー側で適用エラーが発生します。

保守担当者向けメモ

  • 本構造体は論理レプリケーションのワイヤプロトコルの一部です。フィールドの追加や変更を行う際は、プロトコルバージョン（LOGICALREP_PROTO_VERSION_NUM）の整合性に注意してください。
  • GIDSIZE の定義は src/include/replication/reorderbuffer.h 等の関連ヘッダーに依存しているため、変更時には影響範囲を確認してください。
