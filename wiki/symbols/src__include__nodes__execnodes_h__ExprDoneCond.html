ExprDoneCond

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/execnodes.h:320

目的

セットを返す関数（SRF: Set-Returning Function）の評価状態を追跡し、呼び出し元に対して現在の結果がセットの一部であるか、あるいはセットが終了したかを示すために使用されます。

シグネチャまたは形状

typedef enum {
    ExprSingleResult,
    ExprMultipleResult,
    ExprEndResult,
} ExprDoneCond;

入力と出力

  • **入力**: 式評価の実行状態。
  • **出力**: 以下のいずれかの状態値。
  • ExprSingleResult: 式がセットを返さない（単一値）。
  • ExprMultipleResult: 現在の結果がセットの要素である。
  • ExprEndResult: セットの要素がこれ以上存在しない。

前提条件と不変条件

  • SRFの評価を行うコンテキストにおいて、呼び出し元は本列挙型を使用して反復処理の継続可否を判断する必要があります。

アルゴリズムまたは振る舞い

  • ExprSingleResult は、通常のスカラ値を返す式で使用されます。
  • ExprMultipleResult と ExprEndResult は、ValuePerCall モードでのSRF評価において、反復的な結果取得の制御フローを管理するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（確認されているもの）**:
  • src/backend/executor/execSRF.c
  • src/backend/executor/nodeProjectSet.c
  • **呼び出し先**:
  • 本シンボルは列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • ExprState: 式評価の実行状態を保持する構造体。isDone フィールドで本型を使用。
  • ProjectSetState: elemdone フィールド（SRFごとの状態配列）として本型を使用。

失敗モードと境界条件

  • SRFの評価中に ExprEndResult を受け取った後、さらに評価を継続した場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本シンボルは実行時専用のステータス管理用であり、永続化やコピーの対象外です。
  • nodeProjectSet.c では、複数のSRFを扱うために ExprDoneCond の配列として利用されています。
