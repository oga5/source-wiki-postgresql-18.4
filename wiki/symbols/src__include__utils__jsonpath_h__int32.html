int32

定義

  • 種別: typedef（PostgreSQL の標準的な 32 ビット符号付き整数型）
  • 定義箇所: src/include/utils/jsonpath.h:151

目的

JSON Path のバイナリ表現において、構造体内のオフセットやインデックスを保持するために使用されます。

シグネチャまたは形状

struct {
    int32 left;
    int32 right;
} args;

入力と出力

  • 入力: JSON Path の各ノードにおけるオペランドの相対位置（オフセット）。
  • 出力: 該当するノードの左および右オペランドへのアクセス用インデックス。

前提条件と不変条件

  • int32 は 32 ビット符号付き整数として定義されている必要があります。
  • args 構造体内の値は、バイナリ形式の JsonPath 内における有効なオフセットを指している必要があります。

アルゴリズムまたは振る舞い

  • JsonPath のバイナリ構造内において、演算子ノードが持つ左右の引数（オペランド）の位置を特定するために使用されます。
  • 走査時には、これらの値を基に JsonPathItem のポインタを算出し、パスのリンクを辿ります。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは、特定の関数による直接的な呼び出し関係は特定できません。
  • 呼び出し先: 提供されたソース情報からは、特定の関数への呼び出し関係は特定できません。
  • 補足: contrib/amcheck 等の他モジュールで int32 型自体は広く使用されていますが、本ヘッダー内の args 構造体における使用箇所との直接的な関連性は確認されていません。

関連データ構造

  • JsonPath: 本型を含むバイナリ形式の JSON Path 構造体。
  • JsonPathItem: args を用いて走査される実行時抽象化構造体。

失敗モードと境界条件

  • オフセット値が JsonPath のバッファサイズを超える場合、メモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは PostgreSQL の標準的な int32 型のエイリアスです。
  • jsonpath.h 内の args 構造体で使用されている int32 は、バイナリ形式の不変性を維持するため、構造の変更には細心の注意が必要です。
