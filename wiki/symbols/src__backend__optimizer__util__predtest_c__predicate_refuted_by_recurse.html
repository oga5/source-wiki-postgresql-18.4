predicate_refuted_by_recurse

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/util/predtest.c:530

目的

与えられた条件（clause）が述語（predicate）を論理的に否定（refute）できるかを、AND/OR 構造を再帰的に分解して判定します。

シグネチャまたは形状

static bool predicate_refuted_by_recurse(Node *clause, Node *predicate, bool weak);

入力と出力

  • **入力**:
  • clause: 判定の前提となる条件式（Node）。RestrictInfo の場合は内部の clause が抽出されます。
  • predicate: 否定される対象の述語（Node）。
  • weak: 弱否定（weak refutation）を行うかどうかのフラグ。
  • **出力**:
  • bool: clause が predicate を否定できる場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • clause は NULL であってはなりません。
  • 入力される式は IMMUTABLE であることが前提です。
  • predicate_classify を用いて式を CLASS_AND, CLASS_OR, CLASS_ATOM に分類し、それぞれの論理規則に基づいて処理を行います。

アルゴリズムまたは振る舞い

1. **正規化**: clause が RestrictInfo の場合、内部の式を取り出します。 2. **分類**: predicate_classify を使用して clause と predicate の論理構造を特定します。 3. **再帰的判定**:

  • **AND/OR 構造**: 各要素に対して再帰的に predicate_refuted_by_recurse を呼び出し、論理包含関係（AND なら「いずれかが否定」、OR なら「すべてが否定」など）を評価します。
  • **NOT 演算子**: predicate が NOT を含む場合、extract_not_arg で引数を取り出し、predicate_implied_by_recurse を用いて論理包含をテストします。
  • **終端**: CLASS_ATOM に到達すると、演算子ベースの推論（operator_predicate_proof 等）へ委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • predicate_refuted_by（メインの外部インターフェース）
  • 同関数自身（再帰呼び出し）
  • **呼び出し先**:
  • predicate_classify
  • predicate_implied_by_recurse
  • extract_not_arg
  • iterate_begin / iterate_end（マクロによるイテレータ）

関連データ構造

  • PredClass: 式の分類（ATOM, AND, OR）。
  • PredIterInfoData: 式の走査状態を保持する構造体。
  • RestrictInfo: clause がラップされている場合に使用されるノード構造。

失敗モードと境界条件

  • **スタックオーバーフロー**: 非常に深い AND/OR 構造を持つ式ツリーを処理する場合、再帰の深さによりスタックオーバーフローが発生する可能性があります。
  • **キャッシュ**: 演算子の推論結果は OprProofCacheHash にキャッシュされます。キャッシュのフラッシュタイミング（InvalidateOprProofCacheCallBack）と整合しない場合、誤った判定を下すリスクがあります。

保守担当者向けメモ

  • 本関数は predicate_refuted_by の中核となる再帰エンジンです。論理規則の追加や変更を行う際は、AND/OR の組み合わせパターンが網羅されているか確認してください。
  • weak フラグの扱いは、単純な NOT 句の処理において predicate_implied_by_recurse を呼び出す際の挙動に影響します。
