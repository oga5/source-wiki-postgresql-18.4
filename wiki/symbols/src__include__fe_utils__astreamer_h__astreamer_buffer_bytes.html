astreamer_buffer_bytes

定義

  • 種別: static inline function
  • 定義箇所: src/include/fe_utils/astreamer.h:167

目的

入力データストリームから指定されたバイト数を読み取り、astreamer 構造体が保持する内部バッファ（bbs_buffer）へ転送する。

シグネチャまたは形状

static inline void astreamer_buffer_bytes(astreamer *streamer, const char **data, int *len, int nbytes)

入力と出力

  • streamer: 処理対象の astreamer 構造体へのポインタ。
  • data: 入力データポインタの参照。処理後、読み取ったバイト数分だけ前進する。
  • len: 残りのデータ長へのポインタ。処理後、読み取ったバイト数分だけ減少する。
  • nbytes: バッファへ転送するバイト数。

前提条件と不変条件

  • nbytes <= *len であること（アサーションにより検証）。
  • streamer->bbs_buffer は StringInfo 型として初期化済みであること。

アルゴリズムまたは振る舞い

1. appendBinaryStringInfo を使用し、*data が指す位置から nbytes 分のデータを streamer->bbs_buffer にコピーする。 2. 入力データポインタ *data を nbytes 分進める。 3. 残りデータ長 *len を nbytes 分減算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/fe_utils/astreamer_tar.c:249
  • src/include/fe_utils/astreamer.h:200 (内部ヘルパー関数経由)
  • src/include/fe_utils/astreamer.h:205 (内部ヘルパー関数経由)
  • **呼び出し先**:
  • appendBinaryStringInfo (lib/stringinfo.h)

関連データ構造

  • astreamer: ストリーム処理の基底構造体。bbs_buffer を保持する。
  • StringInfo: bbs_buffer の型。可変長文字列バッファ。

失敗モードと境界条件

  • nbytes が *len を超える場合、Assert が有効なビルドではプログラムが停止する。
  • メモリ不足時には appendBinaryStringInfo 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はインライン関数であり、頻繁に呼び出されるためオーバーヘッドを最小限に抑える設計となっている。
  • data と len をポインタ経由で更新するため、呼び出し側で状態の同期が自動的に行われる。
