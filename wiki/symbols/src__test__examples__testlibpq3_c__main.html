目的

PostgreSQL の libpq ライブラリを使用し、パラメータ化クエリ（PQexecParams）を用いたクエリ実行と、バイナリ形式でのデータ送受信を実証するサンプルプログラムのエントリポイントです。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。argv[1] が指定された場合、それを接続文字列（conninfo）として使用します。
  • **出力**:
  • 正常終了時は 0 を返します。
  • 接続失敗やクエリ実行エラー時は exit_nicely を呼び出し、プログラムを終了します。

前提条件と不変条件

  • 実行前にデータベース内に test1 テーブルが存在し、適切なスキーマが設定されている必要があります。
  • バイナリパラメータを送信する際は、ホストバイトオーダーからネットワークバイトオーダー（ビッグエンディアン）への変換（htonl）が必須です。

アルゴリズムまたは振る舞い

1. **接続**: PQconnectdb でデータベースに接続し、SET search_path で環境を初期化します。 2. **テキストパラメータの実行**: PQexecParams を使用し、パラメータをテキストとして送信しつつ、結果をバイナリ形式で要求します。 3. **バイナリパラメータの実行**: 整数値を htonl で変換し、paramFormats を 1（バイナリ）に設定して PQexecParams を実行します。 4. **結果処理**: 各クエリの結果を show_binary_results で解析・表示し、PQclear で結果セットを解放します。 5. **終了**: PQfinish で接続を閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のプロセス実行環境。
  • **呼び出し先**:
  • PQconnectdb, PQstatus, PQerrorMessage, PQexec, PQresultStatus, PQclear, PQexecParams, PQfinish (libpq API)
  • exit_nicely (同ファイル内のヘルパー関数)
  • show_binary_results (同ファイル内のヘルパー関数)
  • htonl (システムライブラリ)
  • 注記: ツールが提示した contrib/amcheck 等の main への言及は、本プログラムの main 関数とは無関係な別シンボルへの参照です。

関連データ構造

  • PGconn: データベース接続ハンドル。
  • PGresult: クエリ結果セット。
  • paramValues, paramLengths, paramFormats: PQexecParams に渡すパラメータ配列。

失敗モードと境界条件

  • 接続失敗時: PQstatus が CONNECTION_OK でない場合、エラーメッセージを出力して終了します。
  • クエリ実行失敗時: PQresultStatus が期待値（PGRES_COMMAND_OK または PGRES_TUPLES_OK）でない場合、エラーメッセージを出力して終了します。

保守担当者向けメモ

  • バイナリ I/O を扱う際は、strlen ではなく PQgetlength を使用してデータ長を特定してください。
  • test1 テーブルのスキーマ定義については、ソースコード内のコメントおよび関連する .sql ファイルを参照してください。
