ConditionVariableCancelSleep

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/condition_variable.c:229

目的

条件変数に対するプロセスの待機状態を解除し、待機リストから自身を安全に削除する。

シグネチャまたは形状

bool ConditionVariableCancelSleep(void)

入力と出力

  • **入力**: なし（スレッドローカル変数 cv_sleep_target を参照）。
  • **出力**: bool 型。プロセスが待機リストから削除された場合は false、既に起床済み（シグナル受信済み）であった場合は true を返す。

前提条件と不変条件

  • cv_sleep_target が現在のプロセスが待機中の条件変数を指していること。
  • 呼び出し後、cv_sleep_target は NULL にリセットされる。

アルゴリズムまたは振る舞い

1. cv_sleep_target が NULL の場合、何もせず false を返す。 2. 対象の条件変数のスピンロック（cv->mutex）を取得する。 3. 待機リスト（cv->wakeup）内に現在のプロセスが存在するか確認する。

  • 存在する場合: リストから削除し、signaled を false とする。
  • 存在しない場合: 既に起床処理が行われたとみなし、signaled を true とする。

    • 4. スピンロックを解放する。 5. cv_sleep_target を NULL にクリアする。 6. signaled の値を返す。

関連データ構造

  • ConditionVariable: 待機リストとスピンロックを保持する構造体。
  • cv_sleep_target: 現在のプロセスが待機中の条件変数を指すスレッドローカル変数。

失敗モードと境界条件

  • cv_sleep_target が NULL の場合、待機状態ではないと判断し即座に false を返す。
  • 既に起床済み（proclist に存在しない）の場合は true を返し、正常な終了フローとして扱う。

保守担当者向けメモ

  • 本関数は、条件変数の待機ループを終了する際のクリーンアップ処理として必須である。
  • proclist 操作はスピンロックで保護されているため、競合が発生しても整合性は保たれる。
