fmtlong

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-rfmtlong.c:26

目的

Informix 互換ライブラリの rfmtlong 関数の動作を検証するためのラッパー関数であり、指定された長整数とフォーマット文字列を用いて変換を行い、その結果を標準出力へ整形して出力する。

シグネチャまたは形状

static void fmtlong(long lng, const char *fmt)

入力と出力

  • **入力**:
  • lng: 変換対象の long 型整数。
  • fmt: 変換に使用するフォーマット文字列。
  • **出力**: なし（標準出力への書き出しのみ）。

前提条件と不変条件

  • rfmtlong が正常に変換を完了した場合、戻り値 r は 0 となる。
  • 変換結果を格納するバッファ buf は 30 バイトの固定長である。

アルゴリズムまたは振る舞い

1. rfmtlong(lng, fmt, buf) を呼び出し、変換を実行する。 2. 変換結果の戻り値 r を標準出力に表示する。 3. r == 0 の場合、変換後の文字列 buf、フォーマット文字列 fmt、および静的カウンタ i の値を整形して出力する。 4. r != 0 の場合、check_return(r) を呼び出し、エラーコードに応じた処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/interfaces/ecpg/test/expected/compat_informix-rfmtlong.c 内の各テストケース）。
  • **呼び出し先**:
  • rfmtlong (Informix 互換ライブラリ関数)
  • printf (標準ライブラリ)
  • check_return (同ファイル内のヘルパー関数)

関連データ構造

  • buf: 変換結果を保持する char[30] 型のバッファ。

失敗モードと境界条件

  • rfmtlong が 0 以外の値を返した場合、check_return 関数へ制御が移る。
  • buf のサイズ（30バイト）を超えるフォーマット変換が行われた場合の挙動については、本ソースコードからは明示されていない。

保守担当者向けメモ

  • 本関数はテスト用の期待値生成コードの一部である。rfmtlong の仕様変更やテストケースの追加時には、本関数が呼び出している rfmtlong の引数や出力形式との整合性に注意すること。
