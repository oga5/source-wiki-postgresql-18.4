SendNegotiateProtocolVersion

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/backend_startup.c:935

目的

クライアントが要求したプロトコルバージョンがサポート対象外である場合、サーバーがサポートする最新のプロトコルバージョンと、クライアントが送信した未知のオプションリストを通知するメッセージを構築・送信する。

シグネチャまたは形状

static void SendNegotiateProtocolVersion(List *unrecognized_protocol_options)

入力と出力

  • **入力**: unrecognized_protocol_options (クライアントから送られたが認識できなかったプロトコルオプションのリスト)
  • **出力**: なし（通信バッファへのメッセージ書き込みのみ）

前提条件と不変条件

  • pq_beginmessage および pq_endmessage を使用するため、通信層が初期化されている必要がある。
  • 本関数はメッセージをバッファに書き込むのみであり、即時のフラッシュは行わない。

アルゴリズムまたは振る舞い

1. PqMsg_NegotiateProtocolVersion 型のメッセージを開始する。 2. サーバーがサポートする現在の FrontendProtocol バージョンを書き込む。 3. unrecognized_protocol_options リストの長さを書き込む。 4. リスト内の各オプション文字列を順次メッセージバッファに書き込む。 5. メッセージを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/backend_startup.c:842 (スタートアップパケット処理中に呼び出される)
  • **呼び出し先**:
  • pq_beginmessage
  • pq_sendint32
  • pq_sendstring
  • pq_endmessage
  • list_length
  • lfirst

関連データ構造

  • List (PostgreSQLのリスト構造)
  • StringInfoData (メッセージ構築用バッファ)

失敗モードと境界条件

  • メモリ不足時や通信エラー発生時の挙動は、呼び出し先の pq_* 関数群の仕様に依存する。
  • unrecognized_protocol_options が空の場合、リストの長さとして 0 が送信される。

保守担当者向けメモ

  • 本関数はメッセージをフラッシュしないため、呼び出し側で後続のメッセージ送信またはフラッシュ処理が行われることが前提となっている。
  • FrontendProtocol はグローバルなプロトコルバージョン定数を参照している。
