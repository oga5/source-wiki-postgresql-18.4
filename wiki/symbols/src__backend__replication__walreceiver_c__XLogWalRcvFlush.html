XLogWalRcvFlush

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiver.c:984

目的

受信した WAL データをディスクへ物理的にフラッシュし、共有メモリ上の進捗情報を更新して、リカバリプロセスおよびプライマリサーバーへ通知を行う。

シグネチャまたは形状

static void XLogWalRcvFlush(bool dying, TimeLineID tli)

入力と出力

  • **入力**:
  • dying (bool): プロセス終了処理中であるかを示すフラグ。
  • tli (TimeLineID): 現在のタイムラインID。
  • **出力**: なし（副作用として共有メモリおよびディスク状態を更新）。

前提条件と不変条件

  • tli != 0 であること。
  • LogstreamResult.Flush < LogstreamResult.Write の場合にのみ、ディスクへのフラッシュ処理が実行される。

アルゴリズムまたは振る舞い

1. issue_xlog_fsync を呼び出し、書き込み済みの WAL データをディスクへ永続化する。 2. LogstreamResult.Flush を LogstreamResult.Write と同期させる。 3. WalRcv 共有メモリ構造体のミューテックスを確保し、flushedUpto および receivedTLI を更新する。 4. WakeupRecovery() を呼び出し、リカバリプロセスへ通知する。 5. カスケードレプリケーションが有効な場合、WalSndWakeup を呼び出し、ダウンストリームのスタンバイへ通知する。 6. プロセスタイトルを更新し、現在のストリーミング進捗を表示する。 7. dying が false の場合、XLogWalRcvSendReply および XLogWalRcvSendHSFeedback を呼び出し、プライマリへ受信状況とフィードバックを送信する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/walreceiver.c 内のストリーミングループおよび終了処理（行 490, 616, 789, 1051）。
  • **呼び出し先**:
  • issue_xlog_fsync
  • SpinLockAcquire / SpinLockRelease
  • WakeupRecovery
  • AllowCascadeReplication
  • WalSndWakeup
  • set_ps_display
  • XLogWalRcvSendReply
  • XLogWalRcvSendHSFeedback

関連データ構造

  • LogstreamResult: 受信およびフラッシュ済みの WAL 位置を保持するグローバル変数。
  • WalRcvData (WalRcv): 共有メモリ上のレプリケーション状態管理構造体。
  • recvFile, recvSegNo: 現在書き込み中の WAL ファイル情報。

失敗モードと境界条件

  • issue_xlog_fsync が失敗した場合、PostgreSQL の標準的なエラーハンドリングによりプロセスが終了する可能性がある。
  • dying が true の場合、プライマリへのフィードバック送信はスキップされる。

保守担当者向けメモ

  • 本関数は walreceiver プロセスのメインループから定期的に呼び出される。
  • 共有メモリへの書き込みはミューテックスで保護されているため、競合は適切に制御されている。
  • update_process_title が有効な場合、set_ps_display によるプロセスリストの更新が発生するため、頻繁な呼び出しには注意が必要。
