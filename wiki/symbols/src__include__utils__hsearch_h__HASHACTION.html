HASHACTION

定義

  • 種別: enum
  • 定義箇所: src/include/utils/hsearch.h:111

目的

hash_search() 関数において、ハッシュテーブルに対する操作の種類（検索、挿入、削除など）を指定するために使用されます。

シグネチャまたは形状

typedef enum
{
    HASH_FIND,
    HASH_ENTER,
    HASH_REMOVE,
    HASH_ENTER_NULL,
} HASHACTION;

入力と出力

  • 入力: hash_search() 関数の引数として渡されます。
  • 出力: なし（列挙型定数）。

前提条件と不変条件

  • hash_search() の呼び出し時に、有効な HASHACTION 値を指定する必要があります。

アルゴリズムまたは振る舞い

各列挙値は hash_search() 内で以下の動作を決定します：

  • HASH_FIND: キーに対応するエントリを検索し、存在すればそのポインタを返し、存在しなければ NULL を返します。
  • HASH_ENTER: キーに対応するエントリを検索し、存在すればそのポインタを返し、存在しなければ新しいエントリを作成して返します。
  • HASH_REMOVE: キーに対応するエントリを検索し、存在すれば削除して true（または成功を示す値）を返し、存在しなければ false を返します。
  • HASH_ENTER_NULL: HASH_ENTER と同様ですが、メモリ不足等の理由でエントリ作成ができない場合に NULL を返す可能性があります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/hash/dynahash.c 内の関数が hash_search() を呼び出す際に引数として使用しています。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数呼び出し先はありません。

関連データ構造

  • HTAB: ハッシュテーブルの管理構造体。
  • hash_search(): この列挙型を引数として受け取る主要な関数。

失敗モードと境界条件

  • 未定義の HASHACTION 値が渡された場合の挙動は、hash_search() の実装に依存します。

保守担当者向けメモ

  • 新しい操作を追加する場合は、hash_search() の実装（src/backend/utils/hash/dynahash.c）で switch 文のケースを追加する必要があります。
