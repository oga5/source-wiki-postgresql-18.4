charlt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/char.c:144

目的

PostgreSQLの1バイト内部型 "char" 同士を比較し、第1引数が第2引数より小さいかどうかを判定します。

シグネチャまたは形状

Datum charlt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの "char" 型の値を受け取ります。
  • **出力**: 第1引数が第2引数より小さい場合は true、それ以外は false を Datum 型（bool）として返します。

前提条件と不変条件

  • 比較対象は1バイトの "char" 型です。
  • 比較は符号なし8ビット整数 (uint8) として実行されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_CHAR マクロを使用して、引数から2つの1バイト文字を取得します。 2. 両方の値を uint8 にキャストします。 3. キャスト後の値を用いて大小比較を行い、結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**: PG_GETARG_CHAR および PG_RETURN_BOOL マクロを使用します。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • "char": 本関数が対象とする1バイトの内部型。

失敗モードと境界条件

  • 比較は符号なしとして行われるため、符号付き文字としての大小関係とは異なる結果になる場合があります。

保守担当者向けメモ

  • 本関数は uint8 として比較を行いますが、int4 との変換関数では int8（符号付き）として扱われるという歴史的な不整合が存在するため、修正や拡張の際は注意してください。
  • SQL標準の CHAR(n) 型（bpchar）とは異なる型を扱っている点に留意してください。
