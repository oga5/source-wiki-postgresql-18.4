NIImportOOAffixes

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/spell.c:1207

目的

Hunspell 形式の接辞（affix）ファイルを読み込み、IspellDict 構造体に複合語ルール、フラグ設定、および接辞エイリアスをインポートする。

シグネチャまたは形状

static void NIImportOOAffixes(IspellDict *Conf, const char *filename)

入力と出力

  • **入力**:
  • Conf: 辞書設定を保持する IspellDict 構造体へのポインタ。
  • filename: 読み込む接辞ファイルへのパス。
  • **出力**: なし（Conf 構造体の内部状態を更新）。

前提条件と不変条件

  • Conf は初期化済みであること。
  • 指定された filename が読み取り可能であること。
  • ファイル内のフラグ設定（COMPOUND... 等）は、接辞定義の前に記述されている必要がある（関数内でファイルを2回走査するため）。

アルゴリズムまたは振る舞い

1. **初期化**: Conf のフラグモードをデフォルト（FM_CHAR）に設定。 2. **1回目の走査**: ファイルを読み込み、COMPOUND 関連のフラグ設定および FLAG モード（long, num, default）を解析して Conf に格納。 3. **ソート**: 複合語フラグが複数ある場合、cmpcmdflag を用いてソート。 4. **2回目の走査**: ファイルを再度読み込み、AF（エイリアス）定義および PFX/SFX（接頭辞/接尾辞）ルールを解析。 5. **エイリアス処理**: AF 行を検出すると、Conf->AffixData を確保し、エイリアスリストを構築。 6. **接辞解析**: parse_ooaffentry を使用して各行をパースし、接辞の属性（isSuffix 等）を判定。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/spell.c 内の辞書読み込み処理から呼び出される（例: NIImportAffixes 等の文脈）。
  • **呼び出し先**: tsearch_readline_begin, tsearch_readline, addCompoundAffixFlagValue, parse_ooaffentry, lowerstr_ctx, palloc0, cpstrdup, qsort 等。

関連データ構造

  • IspellDict: 辞書の状態を保持するメイン構造体。
  • CompoundAffixFlag: 複合語フラグの定義構造体。
  • tsearch_readline_state: ファイル読み込み用状態管理構造体。

失敗モードと境界条件

  • ファイルが開けない場合、ERROR を発生させ処理を中断。
  • 不正な FLAG 値や、定義数を超過したエイリアス行が存在する場合、ERROR を発生。
  • sflag の長さが flagMode に適合しない場合、その行をスキップ。

保守担当者向けメモ

  • 本関数はファイルを2回走査する設計であるため、ファイルシステムへのアクセス負荷に注意すること。
  • COMPOUNDLAST と COMPOUNDEND は同義として扱われる。
  • AF（エイリアス）の処理において、最初の行で定義された数と実際の行数が一致しない場合はエラーとなる。
