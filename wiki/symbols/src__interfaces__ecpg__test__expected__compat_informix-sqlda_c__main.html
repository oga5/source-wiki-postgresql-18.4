目的

Informix 互換モードにおける ECPG の SQLDA（SQL Descriptor Area）機能の動作を検証する回帰テストのエントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（コマンドライン引数は受け取らない）
  • 出力: 成功時に 0 を返し、エラー発生時に exit(1) を呼び出します。標準出力には FETCH されたレコードの内容がダンプされます。

前提条件と不変条件

  • 実行環境において ecpg1_regression データベースへの接続が可能であること。
  • データベース内にテスト用テーブル t1 を作成・操作する権限があること。
  • sqlca.sqlcode を通じて各 SQL 文の実行結果が監視され、負の値（エラー）が返された場合は即座に終了します。

アルゴリズムまたは振る舞い

1. **環境設定**: ECPGdebug を有効化し、データベースへ接続します。 2. **テーブル構築**: t1 テーブルを作成し、初期データを挿入してコミットします。 3. **動的 SQL 実行（標準）**: PREPARE で SQL 文を準備し、カーソルを宣言します。FETCH をループ実行し、outp_sqlda を介して結果を取得・ダンプします。 4. **動的 SQL 実行（Informix 互換）**: Informix 特有の FETCH ... USING DESCRIPTOR 構文を用いたデータ取得を検証します。 5. **クリーンアップ**: 割り当てられたメモリ（outp_sqlda）を解放し、リソースを終了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGprepare, ECPGdeallocate, ECPGtrans, dump_sqlda, printf, malloc, free, strcpy, exit

関連データ構造

  • sqlda_t: 動的 SQL の記述子情報を保持するメイン構造体。
  • sqlvar_t: 列ごとの属性を保持する構造体。
  • sqlca_t: SQL 実行結果のステータスを保持するグローバル変数 sqlca の型。

失敗モードと境界条件

  • データベース接続失敗、テーブル作成失敗、または SQL 実行エラーが発生した場合、sqlca.sqlcode < 0 を検知して exit(1) で異常終了します。
  • FETCH 時に ECPG_NOT_FOUND が発生した場合はループを正常に終了します。

保守担当者向けメモ

  • 本関数はプリプロセッサによって生成されたテストコードです。直接編集せず、期待値の変更が必要な場合はソースとなる .pgc ファイルを修正してください。
  • dump_sqlda 関数による出力形式が、テストの期待値と一致していることを確認してください。
