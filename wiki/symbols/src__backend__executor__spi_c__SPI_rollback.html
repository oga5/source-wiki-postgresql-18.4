SPI_rollback

定義

  • 種別: function
  • 定義箇所: src/backend/executor/spi.c:412

目的

SPI（Server Programming Interface）経由で実行されたトランザクションをロールバックします。

シグネチャまたは形状

void SPI_rollback(void)

入力と出力

  • 入力: なし
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • SPI 接続が確立されているコンテキストで呼び出される必要があります。
  • トランザクション制御が許可されている環境（例: 非アトミックな手続き型言語のコンテキスト）である必要があります。

アルゴリズムまたは振る舞い

内部関数 _SPI_rollback(false) を呼び出し、現在のトランザクションをロールバックします。引数 false は、連鎖（chaining）を行わないことを示します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/pl/plperl/plperl.c
  • src/pl/plpgsql/src/pl_exec.c
  • src/pl/plpython/plpy_spi.c
  • src/pl/tcl/pltcl.c
  • 呼び出し先:
  • _SPI_rollback（内部関数）

関連データ構造

  • _SPI_stack: SPI 呼び出しのネストを管理する内部スタック。

失敗モードと境界条件

  • トランザクション制御が許可されていないコンテキストで呼び出された場合、PostgreSQL のエラー処理メカニズムにより例外が発生する可能性があります。

保守担当者向けメモ

  • 本関数は _SPI_rollback のラッパーです。トランザクション制御のロジック変更は _SPI_rollback を参照してください。
  • SPI_rollback_and_chain との対比において、連鎖の有無が唯一の差異です。
