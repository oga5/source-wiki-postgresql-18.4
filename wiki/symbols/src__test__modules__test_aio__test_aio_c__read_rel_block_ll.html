read_rel_block_ll

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:357

目的

PostgreSQL の非同期 I/O (AIO) サブシステムの動作を検証するためのテスト用関数です。指定されたリレーションのブロック範囲に対して、低レベルな AIO 読み込み操作を実行します。

シグネチャまたは形状

Datum read_rel_block_ll(PG_FUNCTION_ARGS)

SQL から呼び出し可能な PostgreSQL 関数として定義されています。

入力と出力

  • **入力 (PG_FUNCTION_ARGS)**:
  • relid (Oid): 対象リレーションの OID。
  • blkno (BlockNumber): 読み込み開始ブロック番号。
  • nblocks (int): 読み込むブロック数。
  • wait_complete (bool): 完了を待機するかどうか。
  • batchmode_enter / batchmode_exit (bool): AIO バッチモードの開始・終了制御。
  • call_smgrreleaseall (bool): smgrreleaseall() を呼び出すかどうか。
  • zero_on_error (bool): エラー時にゼロクリアを行うか。
  • **出力**: PG_RETURN_VOID() により値を返しません。

前提条件と不変条件

  • nblocks は 0 より大きく PG_IOV_MAX 以下である必要があります。
  • 対象リレーションは AccessExclusiveLock で開かれます。

アルゴリズムまたは振る舞い

1. create_toy_buffer を用いて指定ブロック数分のバッファを確保し、対応する BufferDesc を取得します。 2. pgaio_io_acquire で AIO ハンドルを取得し、バッファの状態に応じて StartLocalBufferIO または StartBufferIO を呼び出し I/O を開始します。 3. pgaio_io_register_callbacks で読み込みコールバックを登録します。 4. smgrstartreadv を呼び出し、ストレージマネージャ経由で非同期読み込みを発行します。 5. wait_complete が真の場合、pgaio_wref_wait で完了を待ち、結果がエラーであれば報告します。 6. 最後にバッファを解放し、リレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（PG_FUNCTION_INFO_V1 による登録あり）。
  • **呼び出し先**: relation_open, create_toy_buffer, pgaio_* API 群, smgrstartreadv, smgrreleaseall, ReleaseBuffer, relation_close 等。

関連データ構造

  • Buffer, BufferDesc: バッファ管理構造体。
  • PgAioHandle, PgAioWaitRef: AIO サブシステムの制御構造体。
  • SMgrRelation: ストレージマネージャのリレーション情報。

失敗モードと境界条件

  • nblocks が範囲外の場合、elog(ERROR) で停止します。
  • wait_complete が真の場合、I/O 結果が PGAIO_RS_OK でなければ pgaio_result_report によりエラーまたは警告が報告されます。

保守担当者向けメモ

  • 本関数はテスト専用であり、共有バッファの直接操作など本番環境では許容されないハックを含みます。
  • PG_IOV_MAX までのブロック数制限がある点に注意してください。
