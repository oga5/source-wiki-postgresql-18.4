overwrite

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlo64.c:113

目的

PostgreSQL のラージオブジェクト内の指定されたオフセット位置から、特定の長さ分を文字 'X' で上書きするテスト用関数です。

シグネチャまたは形状

static void overwrite(PGconn *conn, Oid lobjId, int64_t start, int len)

入力と出力

  • **入力**:
  • conn: PostgreSQL 接続オブジェクト (PGconn *)。
  • lobjId: 操作対象のラージオブジェクトの OID (Oid)。
  • start: 上書きを開始する 64 ビットオフセット位置 (int64_t)。
  • len: 上書きするバイト数 (int)。
  • **出力**: なし（副作用としてラージオブジェクトの内容を更新）。

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • lo_open が成功し、書き込みモード (INV_WRITE) でファイル記述子が取得できること。
  • lo_lseek64 により、指定された start 位置へシーク可能であること。

アルゴリズムまたは振る舞い

1. lo_open を使用してラージオブジェクトを書き込みモードで開く。 2. lo_lseek64 を呼び出し、ファイルポインタを start 位置へ移動させる。 3. 指定された len 分のメモリを確保し、すべて 'X' で埋める。 4. lo_write をループ内で呼び出し、バッファの内容をラージオブジェクトに書き込む。 5. 書き込み完了後、メモリを解放し lo_close でファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に明示的な呼び出し元の記述はありません。
  • **呼び出し先**:
  • lo_open, lo_lseek64, lo_write, lo_close (libpq ラージオブジェクト API)
  • malloc, free (標準ライブラリ)
  • fprintf (標準ライブラリ)
  • PQerrorMessage (libpq)

関連データ構造

  • PGconn: PostgreSQL 接続状態を保持する構造体。
  • Oid: ラージオブジェクトを一意に識別する型。

失敗モードと境界条件

  • lo_open が負の値を返した場合、エラーメッセージを出力する。
  • lo_lseek64 が失敗した場合、PQerrorMessage を含むエラーメッセージを出力する。
  • lo_write が 0 以下の値を返した場合、書き込み失敗としてループを中断し、エラーメッセージを出力する。

保守担当者向けメモ

  • 本関数は testlo64.c 内のテストロジックの一部であり、64 ビットオフセット API の動作検証を目的としています。
  • ツールが提示した他のファイル（contrib/ 配下等）にある overwrite という名称のシンボルは、本関数とは無関係な別個の識別子です。
