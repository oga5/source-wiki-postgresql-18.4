pg_compress_algorithm

定義

  • 種別: enum
  • 定義箇所: src/include/common/compression.h:21

目的

PostgreSQL全体で利用される圧縮アルゴリズムを識別するための列挙型です。ディスク上のデータ形式やバックエンドの圧縮処理における基盤となります。

シグネチャまたは形状

typedef enum pg_compress_algorithm
{
    PG_COMPRESSION_NONE,
    PG_COMPRESSION_GZIP,
    PG_COMPRESSION_LZ4,
    PG_COMPRESSION_ZSTD,
} pg_compress_algorithm;

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: 圧縮アルゴリズムの識別子（PG_COMPRESSION_NONE, PG_COMPRESSION_GZIP, PG_COMPRESSION_LZ4, PG_COMPRESSION_ZSTD）

前提条件と不変条件

  • **ディスク互換性**: 本列挙型の順序はディスク上のデータ形式に直接影響します。既存の値を変更したり、順序を入れ替えたりしてはなりません。

アルゴリズムまたは振る舞い

  • 指定された圧縮アルゴリズムを識別し、後続の圧縮ライブラリ（zlib, lz4, zstd）のAPI呼び出しや、圧縮設定の検証ロジックにおける判定基準として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • src/bin/pg_basebackup/walmethods.c
  • src/bin/pg_basebackup/walmethods.h
  • **呼び出し先**: 該当なし（本シンボルは型定義であり、関数ではないため）

関連データ構造

  • pg_compress_specification: 圧縮アルゴリズム、レベル、ワーカー数、オプションフラグを保持する構造体。

失敗モードと境界条件

  • **ビルド依存**: 特定のアルゴリズム（LZ4やZstd）は、ビルド時の構成（--with-lz4 等）に依存します。環境がサポートしていないアルゴリズムを選択した場合の挙動は、呼び出し側の検証ロジックに依存します。

保守担当者向けメモ

  • 本シンボルはフロントエンドとバックエンドの両方から参照される共有ヘッダーの一部です。
  • 特定のアルゴリズムを追加・変更する際は、ディスクフォーマットへの影響を考慮してください。
