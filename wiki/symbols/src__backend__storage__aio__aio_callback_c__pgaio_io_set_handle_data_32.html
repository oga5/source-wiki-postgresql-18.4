pgaio_io_set_handle_data_32

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_callback.c:139

目的

非同期I/Oハンドル（PgAioHandle）に関連付けられた共有メモリ領域に、32ビット単位の補助データを格納します。

シグネチャまたは形状

void pgaio_io_set_handle_data_32(PgAioHandle *ioh, uint32 *data, uint8 len)

入力と出力

  • **入力**:
  • ioh: 対象となる PgAioHandle 構造体へのポインタ。
  • data: 格納する uint32 型データの配列。
  • len: 格納するデータの要素数。
  • **出力**: なし（副作用として共有メモリ pgaio_ctl->handle_data を更新）。

前提条件と不変条件

  • ioh->state が PGAIO_HS_HANDED_OUT であること。
  • ioh->handle_data_len が 0 であること（1つのI/Oにつき1回のみ設定可能）。
  • len が PG_IOV_MAX 以下であること。
  • len が io_max_combine_limit 以下であること。

アルゴリズムまたは振る舞い

1. 引数で指定された len 分のデータを、ioh->iovec_off をオフセットとして pgaio_ctl->handle_data 配列にコピーします。 2. 最後に ioh->handle_data_len を len に更新し、データが設定済みであることを記録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c:1934
  • src/test/modules/test_aio/test_aio.c:411
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（メモリ操作のみ）。

関連データ構造

  • PgAioHandle: I/O操作の状態とメタデータを保持する構造体。
  • pgaio_ctl: 共有メモリ上の制御構造体。handle_data 配列を保持する。

失敗モードと境界条件

  • 前提条件（Assert）に違反した場合、デバッグビルドではアサーション失敗によりプロセスが停止します。
  • len が制限を超えた場合、アサーションにより保護されます。

保守担当者向けメモ

  • 本関数は共有メモリを直接操作するため、呼び出し時は適切な排他制御が行われていることを確認してください。
  • メモリ消費を抑える設計のため、一度設定したデータは上書きできません。
