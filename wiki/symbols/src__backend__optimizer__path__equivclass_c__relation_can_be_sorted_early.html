relation_can_be_sorted_early

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/equivclass.c:1076

目的

指定された等価クラス（EquivalenceClass）に基づいて、リレーションが早期にソート可能かどうかを判定します。

シグネチャまたは形状

bool relation_can_be_sorted_early(PlannerInfo *root, RelOptInfo *rel,
                                  EquivalenceClass *ec, bool require_parallel_safe)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 対象となるリレーション情報。
  • ec: 評価対象の等価クラス。
  • require_parallel_safe: 並列安全性の検証を要求するかどうか。
  • **出力**:
  • bool: 早期ソートが可能な場合は true、それ以外は false。

前提条件と不変条件

  • ec->ec_has_volatile が真の場合、揮発性関数を含むため早期ソートは不可とみなされます。
  • 早期ソートには、リレーションのターゲットリスト（rel->reltarget）に含まれる式、またはそこから計算可能な式が必要です。

アルゴリズムまたは振る舞い

1. **揮発性チェック**: ec_has_volatile が設定されている場合は直ちに false を返します。 2. **直接マッチング**: リレーションのターゲットリスト内の各式に対し、find_ec_member_matching_expr を使用して等価クラスのメンバーと直接一致するか確認します。

  • 一致した場合、集合を返す関数（SRF）を含まないこと、および必要に応じて並列安全であることを検証します。条件を満たせば true を返します。

    • 3. **計算可能なメンバーの探索**: 直接一致しない場合、find_computable_ec_member を使用して、ターゲットリストから計算可能なメンバーを探索します。 4. **最終検証**: 見つかったメンバーが SRF を含まないことを確認し、結果を返します。

呼び出し元と呼び出し先

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • RelOptInfo: リレーションの最適化情報。
  • EquivalenceClass: 等価な式の集合。
  • EquivalenceMember: 等価クラス内の個別の式。
  • PathTarget: リレーションが生成する式リスト。

失敗モードと境界条件

  • 揮発性関数を含む等価クラスは、常に早期ソート不可となります。
  • 集合を返す関数（SRF）を含む式は、早期計算ができないため除外されます。
  • require_parallel_safe が指定されている場合、並列安全でない式は除外されます。

保守担当者向けメモ

  • 本関数は、プランナがソート処理を早期に行うべきか判断する際の重要なフィルタです。
  • is_parallel_safe の呼び出しはコストが高いため、他の条件チェックの最後に行う設計となっています。
