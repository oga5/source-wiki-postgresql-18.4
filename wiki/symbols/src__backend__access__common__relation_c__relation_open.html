relation_open

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/relation.c:46

目的

OIDで指定されたリレーションをオープンし、リレーション記述子（Relation）を取得します。ロックの取得、リレーションキャッシュの検索、一時テーブルのフラグ更新、および統計情報の初期化を一括して行います。

シグネチャまたは形状

Relation relation_open(Oid relationId, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relationId (Oid): 開く対象のリレーションのOID。
  • lockmode (LOCKMODE): 取得するロックモード。NoLock を指定可能。
  • **出力**:
  • Relation: 成功時にリレーション記述子を返します。

前提条件と不変条件

  • lockmode は NoLock 以上かつ MAX_LOCKMODES 未満であること。
  • NoLock を指定する場合、呼び出し元が既に適切なロックを保持しているか、ブートストラップモードである必要があります。

アルゴリズムまたは振る舞い

1. **ロック取得**: lockmode が NoLock でない場合、LockRelationOid を呼び出して指定されたロックを取得します。 2. **キャッシュ検索**: RelationIdGetRelation を呼び出し、リレーションキャッシュから記述子を取得します。 3. **検証**: 取得した記述子が有効でない場合、エラーを発生させます。 4. **不変条件チェック**: NoLock 指定時は、ブートストラップモードでない限り、呼び出し元がロックを保持していることを CheckRelationLockedByMe で検証します。 5. **状態更新**: 一時リレーション（RelationUsesLocalBuffers）であれば、MyXactFlags に XACT_FLAGS_ACCESSEDTEMPNAMESPACE を設定します。 6. **統計初期化**: pgstat_init_relation を呼び出し、統計情報収集の準備を行います。 7. **戻り値**: 取得した Relation を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: LockRelationOid, RelationIdGetRelation, RelationIsValid, IsBootstrapProcessingMode, CheckRelationLockedByMe, RelationUsesLocalBuffers, pgstat_init_relation
  • **呼び出し元**: contrib/amcheck/verify_heapam.c, contrib/pageinspect/btreefuncs.c, contrib/pageinspect/hashfuncs.c, contrib/pageinspect/heapfuncs.c, contrib/pg_buffercache/pg_buffercache_pages.c, contrib/pg_freespacemap/pg_freespacemap.c, contrib/pg_prewarm/pg_prewarm.c, contrib/pg_surgery/heap_surgery.c, contrib/pg_visibility/pg_visibility.c など。

関連データ構造

  • Relation: リレーション記述子。
  • Oid: リレーションを一意に識別する型。
  • LOCKMODE: ロックの強さを定義する列挙型。

失敗モードと境界条件

  • 指定された relationId に対応するリレーションが存在しない場合、elog(ERROR, ...) により処理が中断されます。
  • lockmode が不正な値の場合、Assert によりデバッグビルドで停止します。

保守担当者向けメモ

  • 本関数はリレーションアクセスの標準的なボイラープレートです。特定のアクセス手法（ヒープやインデックス固有の処理）は含みません。
  • relation_close を呼び出してもロックは自動解除されません。ロックの解放が必要な場合は、relation_close の引数で明示的に指定する必要があります。
