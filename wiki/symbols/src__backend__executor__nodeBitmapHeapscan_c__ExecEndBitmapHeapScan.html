ExecEndBitmapHeapScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapHeapscan.c:266

目的

ビットマップヒープスキャン実行ノードに関連するリソースを解放し、並列実行時には統計情報を集約する。

シグネチャまたは形状

void ExecEndBitmapHeapScan(BitmapHeapScanState *node)

入力と出力

  • **入力**: BitmapHeapScanState *node（スキャン状態を保持する構造体）
  • **出力**: なし（副作用としてリソース解放および統計情報の更新を行う）

前提条件と不変条件

  • node は ExecInitBitmapHeapScan によって初期化済みであること。
  • 並列ワーカープロセスで実行される場合、node->sinstrument が有効であること。

アルゴリズムまたは振る舞い

1. **並列統計の集約**: 並列ワーカープロセスである場合、node->stats に記録された exact_pages および lossy_pages を共有メモリ上の sinstrument に加算する。 2. **サブプランの終了**: outerPlanState(node) を呼び出し、子ノード（インデックススキャン等）の終了処理を行う。 4. **メモリ解放**: node->tbm が存在する場合、tbm_free を呼び出してビットマップを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecEndNode から呼び出されることが確認されている。
  • **呼び出し先**: ExecEndNode（子ノード用）、tbm_end_iterate、table_endscan、tbm_free。

関連データ構造

  • BitmapHeapScanState: スキャンノードの実行状態を保持する主要構造体。
  • TableScanDesc: テーブルスキャンの状態記述子。
  • BitmapHeapScanInstrumentation: 並列実行時の統計情報保持構造体。

失敗モードと境界条件

  • scanDesc が NULL の場合、テーブルスキャンの終了処理はスキップされる。
  • node->tbm が NULL の場合、ビットマップの解放はスキップされる。
  • 並列ワーカーでない場合、統計情報の集約処理は実行されない。

保守担当者向けメモ

  • 本関数は EXPLAIN ANALYZE の統計情報収集において重要な役割を果たすため、並列実行時の統計加算ロジックを変更する際は注意が必要である。
  • tbm_end_iterate はイテレータが既に使い切られている（tbm_exhausted）場合には呼び出されない設計となっている。
