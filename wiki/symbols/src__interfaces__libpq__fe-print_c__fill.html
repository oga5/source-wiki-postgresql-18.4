fill

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-print.c:785

目的

指定された文字（filler）を、指定された回数だけ出力ストリーム（fp）に書き込むことで、パディングを行う。

シグネチャまたは形状

static void fill(int length, int max, char filler, FILE *fp)

入力と出力

  • length (int): 現在の長さ。
  • max (int): 目標とする最大長。
  • filler (char): 出力する文字。
  • fp (FILE *): 出力先のファイルストリーム。
  • 戻り値: なし。

前提条件と不変条件

  • max が length 以上である場合、max - length + 1 回の filler が出力される。
  • max が length 未満の場合、ループは実行されず何も出力されない。

アルゴリズムまたは振る舞い

1. max から length を減算し、出力すべき回数 count を算出する。 2. count が 0 以上である間、putc を使用して filler を fp に書き出し、count をデクリメントする。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース抜粋内には、本関数を呼び出している具体的な箇所は記載されていない。
  • 呼び出し先: 標準ライブラリ関数 putc を呼び出す。

関連データ構造

  • なし。

失敗モードと境界条件

  • fp が無効なストリームである場合、putc の動作に従い失敗する可能性がある。
  • count が負の値（max < length）の場合、ループは一度も実行されない。

保守担当者向けメモ

  • 本関数は fe-print.c 内の表形式出力処理における補助的なパディング機能である。
  • 現代のアプリケーションでは推奨されないレガシーな出力ロジックの一部であるため、変更時は PQprint 等の既存の出力関数への影響を考慮すること。
