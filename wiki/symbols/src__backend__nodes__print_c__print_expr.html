print_expr

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/print.c:324

目的

クエリ内の式（Node）を人間が読みやすい形式で標準出力にダンプする。主にデバッグ用途で使用される。

シグネチャまたは形状

void print_expr(const Node *expr, const List *rtable)

入力と出力

  • **入力**:
  • expr: 表示対象のノード（式）。
  • rtable: Var ノードの参照先を解決するためのレンジテーブル（List）。
  • **出力**: なし（標準出力へ直接書き出す）。

前提条件と不変条件

  • rtable は、expr 内に含まれる Var ノードの varno を解決するために十分な長さを持つ必要がある。
  • Var ノードの varno が INNER_VAR, OUTER_VAR, INDEX_VAR 以外の場合、rtable 内のインデックスとして有効であること（Assert で検証される）。

アルゴリズムまたは振る舞い

ノードの型に応じて以下の通り処理を行う： 1. **NULL**: <> を出力。 2. **Var**: varno に応じて INNER, OUTER, INDEX または rtable から取得したエイリアス名と属性名を表示。 3. **Const**: 値を文字列に変換して出力（NULL の場合は NULL と出力）。 4. **OpExpr**: 演算子名を挟んで左右の引数を再帰的に表示。単項演算子の場合は演算子を先頭に表示。 5. **FuncExpr**: 関数名と括弧で囲まれた引数リストを再帰的に表示。 6. **その他**: unknown expr と出力。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/print.c 内の他の出力関数（print_tl 等）から呼び出される。
  • **呼び出し先**:
  • print_expr（再帰呼び出し）
  • rt_fetch
  • get_rte_attribute_name
  • getTypeOutputInfo
  • OidOutputFunctionCall
  • get_opname
  • get_func_name
  • get_leftop / get_rightop

関連データ構造

  • Node: 表示対象の基底構造体。
  • List: rtable および関数引数リスト。
  • Var, Const, OpExpr, FuncExpr: IsA マクロで判定される具体的なノード型。
  • RangeTblEntry: Var の解決に使用される。

失敗モードと境界条件

  • expr が未知の型である場合、unknown expr と表示される。
  • opno や funcid が無効な場合、(invalid operator) や (invalid function) と表示される。
  • rtable が不適切な場合、Assert 失敗によりプロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数はデバッグ専用であり、本番環境のロジックには組み込まないこと。
  • rtable のコンテキストが正しく渡されていない場合、Var ノードの表示が正しく行われない。
  • pretty_format_node_dump 等の整形関数とは異なり、インデント制御は行わない。
