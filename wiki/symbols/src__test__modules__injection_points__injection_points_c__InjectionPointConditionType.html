InjectionPointConditionType

定義

  • 種別: enum
  • 定義箇所: src/test/modules/injection_points/injection_points.c:51

目的

インジェクションポイントのコールバックを実行する際の条件判定種別を定義します。

シグネチャまたは形状

typedef enum InjectionPointConditionType
{
    INJ_CONDITION_ALWAYS = 0,
    INJ_CONDITION_PID,
} InjectionPointConditionType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: INJ_CONDITION_ALWAYS（常に実行）、INJ_CONDITION_PID（PIDによる制限）のいずれか。

前提条件と不変条件

  • INJ_CONDITION_ALWAYS は常に真として評価されます。
  • INJ_CONDITION_PID は、特定のプロセスIDに関連付けられた条件評価を伴うことを示唆します。

アルゴリズムまたは振る舞い

インジェクションポイントの実行可否を決定する際のフラグとして使用されます。具体的な判定ロジックは、この型を参照する関数（injection_point_allowed 等）内で実装されます。

呼び出し元と呼び出し先

  • 呼び出し元: InjectionPointCondition 構造体（src/test/modules/injection_points/injection_points.c:60）内でメンバとして使用されています。
  • 呼び出し先: 提供されたソース excerpt には、この型を直接呼び出す関数や、この型が呼び出す関数に関する情報は含まれていません。

関連データ構造

  • InjectionPointCondition: この列挙型を保持する構造体。

失敗モードと境界条件

  • 範囲外の整数値がキャストされた場合の挙動は未定義です。

保守担当者向けメモ

  • 新しい条件判定ロジックを追加する場合は、この列挙型に新しいメンバを追加し、対応する判定ロジックを injection_point_allowed に実装してください。
