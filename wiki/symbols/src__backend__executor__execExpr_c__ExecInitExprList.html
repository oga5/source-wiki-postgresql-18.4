ExecInitExprList

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExpr.c:334

目的

式ツリーのリスト（List *nodes）を、実行時に評価可能な ExprState のリストへ一括変換します。

シグネチャまたは形状

List *ExecInitExprList(List *nodes, PlanState *parent)

入力と出力

  • **入力**:
  • nodes: 変換対象となる Expr ノードのリスト。
  • parent: 式の評価コンテキストを提供する PlanState ノード。
  • **出力**:
  • 変換された ExprState ノードを格納する List。

前提条件と不変条件

  • 入力 nodes は Expr ノードのリストである必要があります。
  • parent は式評価の実行環境として有効である必要があります。
  • 各 Expr ノードに対して ExecInitExpr が呼び出され、個別の ExprState が生成されます。

アルゴリズムまたは振る舞い

1. 空のリスト result を初期化します。 2. nodes リストを走査し、各要素 e に対して ExecInitExpr(e, parent) を呼び出します。 3. 生成された ExprState を lappend を用いて result リストに追加します。 4. 変換済みの ExprState リストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/executor/execExpr.c
  • src/backend/executor/execSRF.c
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeIndexscan.c
  • src/backend/executor/nodeSamplescan.c
  • src/backend/executor/nodeTableFuncscan.c
  • **呼び出し先**:
  • ExecInitExpr (各要素の変換のため)

関連データ構造

  • Expr: 解析・計画済みの式ツリーノード。
  • ExprState: 実行時に評価可能な形式にコンパイルされた式。
  • PlanState: 式評価の実行環境を保持する親ノード。

失敗モードと境界条件

  • nodes が NIL の場合、空のリスト NIL を返します。
  • ExecInitExpr が失敗した場合、その内部でエラーが発生し、処理が中断されます。

保守担当者向けメモ

  • 本関数は ExecInitExpr のラッパーとして機能します。リスト内の各式に対して個別にコンパイルを行うため、式ツリーの深さやリストの長さに応じたスタック消費に注意してください。
  • ExecInitExpr の仕様変更に伴い、本関数が生成する ExprState の整合性も維持される必要があります。
