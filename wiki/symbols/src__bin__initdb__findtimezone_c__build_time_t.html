build_time_t

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/findtimezone.c:189

目的

指定された年月日から、システム標準の time_t 型の値を生成します。主にタイムゾーンのサンプリング比較において、特定の基準日時を time_t 形式へ変換するために使用されます。

シグネチャまたは形状

static time_t build_time_t(int year, int month, int day)

入力と出力

  • **入力**:
  • year: 西暦年（例: 2026）
  • month: 月（1〜12）
  • day: 日（1〜31）
  • **出力**:
  • 指定された日時を mktime で変換した time_t 値。

前提条件と不変条件

  • struct tm 構造体は memset によりゼロ初期化されます。
  • tm_mon は month - 1、tm_year は year - 1900 として計算されます。
  • tm_isdst は -1 に設定されており、夏時間の有無を mktime が自動的に判断します。

アルゴリズムまたは振る舞い

1. struct tm をゼロ初期化します。 2. 引数で受け取った年月日から、tm 構造体の各フィールド（tm_mday, tm_mon, tm_year）を標準的な形式に変換して設定します。 3. tm_isdst を -1 に設定し、システム標準の mktime を呼び出して time_t を算出します。 4. 算出された time_t 値を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/findtimezone.c:374
  • src/bin/initdb/findtimezone.c:387
  • **呼び出し先**:
  • memset (標準ライブラリ)
  • mktime (標準ライブラリ)

関連データ構造

  • struct tm (C標準ライブラリ)

失敗モードと境界条件

  • mktime が指定された日時を表現できない場合、-1 が返される可能性があります。
  • 引数に不正な日付（例: 13月や存在しない日）が渡された場合、mktime の実装依存の挙動に従います。

保守担当者向けメモ

  • 本関数は initdb プロセス内でのタイムゾーン比較用として限定的に利用されています。
  • tm_year の計算において year - 1900 を行っているため、1900年以前の日付を扱う場合は注意が必要です。
