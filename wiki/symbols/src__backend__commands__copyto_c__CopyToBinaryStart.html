CopyToBinaryStart

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyto.c:313

目的

COPY TO 操作において、バイナリ形式での出力ストリームを開始するための初期化処理を行います。

シグネチャまたは形状

static void CopyToBinaryStart(CopyToState cstate, TupleDesc tupDesc)

入力と出力

  • **入力**:
  • cstate: COPY TO 操作の全状態を保持する CopyToState 構造体。
  • tupDesc: 出力対象となるタプルの記述子。
  • **出力**: なし（副作用として出力ストリームへバイナリヘッダーを書き込みます）。

前提条件と不変条件

  • cstate は有効な状態である必要があります。
  • 本関数はバイナリ形式の出力が選択された際に呼び出されます。

アルゴリズムまたは振る舞い

1. BinarySignature（11バイト）を出力ストリームに書き込みます。 2. フラグフィールドとして 0 を 32 ビット整数で書き込みます。 3. ヘッダー拡張がないことを示すため、0 を 32 ビット整数で書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyto.c 内の CopyToStart 関数ポインタ経由で呼び出されます。
  • **呼び出し先**:
  • CopySendData: バイナリシグネチャの送信に使用。
  • CopySendInt32: フラグおよびヘッダー拡張の送信に使用。

関連データ構造

  • CopyToState: COPY TO 操作の実行状態を管理する構造体。
  • TupleDesc: タプルの構造定義。

失敗モードと境界条件

  • 出力ストリームへの書き込みエラーが発生した場合、基盤となる CopySendData 等の関数がエラーを発生させます。

保守担当者向けメモ

  • 本関数はバイナリ形式の固定ヘッダーを書き込む役割を担います。将来的にバイナリフォーマットの仕様変更（フラグの追加やヘッダー拡張の導入）が必要な場合は、ここを修正してください。
