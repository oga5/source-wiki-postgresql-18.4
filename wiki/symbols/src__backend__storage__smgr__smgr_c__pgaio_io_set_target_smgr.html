pgaio_io_set_target_smgr

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/smgr.c:1037

目的

非同期 I/O 操作（PgAioHandle）のターゲットとして、特定のストレージマネージャ（SMgrRelationData）に関連するリレーション情報およびブロック範囲を設定する。

シグネチャまたは形状

void pgaio_io_set_target_smgr(PgAioHandle *ioh,
                              SMgrRelationData *smgr,
                              ForkNumber forknum,
                              BlockNumber blocknum,
                              int nblocks,
                              bool skip_fsync)

入力と出力

  • **入力**:
  • ioh: 設定対象の非同期 I/O ハンドル。
  • smgr: 対象となるリレーションの SMgrRelationData 構造体。
  • forknum: 操作対象のフォーク番号。
  • blocknum: 操作開始ブロック番号。
  • nblocks: 操作対象のブロック数。
  • skip_fsync: fsync をスキップするかどうかのフラグ。
  • **出力**: なし（ioh が保持する PgAioTargetData を更新する）。

前提条件と不変条件

  • ioh は有効な PgAioHandle であること。
  • 一時リレーション（SmgrIsTemp(smgr) が真）の場合、skip_fsync の値に関わらず、内部的に sd->smgr.skip_fsync は強制的に false にはならず、一時リレーションは常に fsync をスキップするよう制御される。

アルゴリズムまたは振る舞い

1. pgaio_io_get_target_smgr を介して ioh からターゲットデータ領域を取得する。 2. pgaio_io_set_target を呼び出し、ターゲットタイプを PGAIO_TID_SMGR に設定する。 3. smgr からリレーションのロケータ（rlocator）を抽出し、ターゲットデータに格納する。 4. 指定された forknum、blocknum、nblocks をターゲットデータにコピーする。 5. SmgrIsTemp(smgr) を用いて一時リレーションかどうかを判定し、is_temp フラグを設定する。 6. 一時リレーションではない場合に限り、引数 skip_fsync の値を sd->smgr.skip_fsync に反映する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/smgr/md.c:1022 からの呼び出しが確認されている。
  • **呼び出し先**: pgaio_io_get_target_data および pgaio_io_set_target を呼び出す。

関連データ構造

  • PgAioHandle: 非同期 I/O のハンドル。
  • SMgrRelationData: ストレージマネージャのリレーション情報。
  • PgAioTargetData: 非同期 I/O のターゲット詳細を保持する内部構造体。

失敗モードと境界条件

  • smgr が NULL の場合、ポインタ参照エラーが発生する可能性がある。
  • 一時リレーションに対する skip_fsync の指定は無視され、常に fsync がスキップされる（安全な不変条件）。

保守担当者向けメモ

  • 本関数は storage/aio.h で定義される非同期 I/O インフラの一部である。
  • smgr 構造体の変更が PgAioTargetData に正しく伝搬されるよう、rlocator のコピー箇所には注意すること。
