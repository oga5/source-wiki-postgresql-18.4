SupportRequestSelectivity

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/supportnodes.h:91

目的

特定の関数呼び出しに対して、プランナーがクエリ最適化時に使用するブール値の選択率（Selectivity）見積もりを要求するためのデータ構造です。

シグネチャまたは形状

typedef struct SupportRequestSelectivity
{
    NodeTag     type;
    /* 入力フィールド */
    struct PlannerInfo *root;
    Oid         funcid;
    List       *args;
    Oid         inputcollid;
    bool        is_join;
    int         varRelid;
    JoinType    jointype;
    struct SpecialJoinInfo *sjinfo;
    /* 出力フィールド */
    Selectivity selectivity;
} SupportRequestSelectivity;

入力と出力

  • **入力**: プランナーのインフラ情報 (root)、対象関数 (funcid)、簡略化済み引数 (args)、照合順序 (inputcollid)、および結合か制限かを示すフラグと関連情報 (is_join, varRelid, jointype, sjinfo)。
  • **出力**: 算出された選択率 (selectivity)。

前提条件と不変条件

  • サポート関数は、リクエストを処理できない場合、エラーを発生させずに NULL を返す必要があります。
  • 構造体自体は一時的なものであり、サポート関数内で永続的なメモリ確保を行う必要はありません。

アルゴリズムまたは振る舞い

1. プランナーが特定の関数に対して選択率の見積もりが必要と判断した際、本構造体を生成します。 2. 対象関数に紐付けられたサポート関数が本構造体を引数として呼び出されます。 3. サポート関数は、算出結果を selectivity フィールドに格納します。 4. 呼び出し元は、返された結果に基づきプランのコスト見積もりや最適化を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/plancat.c にて本構造体が生成・利用されています。
  • **呼び出し先**: src/backend/utils/adt/like_support.c および src/test/regress/regress.c にて、本構造体型であるかの判定およびキャストが行われています。

関連データ構造

  • PlannerInfo: プランナーのインフラ情報。
  • SpecialJoinInfo: 外部結合に関する情報。
  • Selectivity: 選択率を保持する型。

失敗モードと境界条件

  • サポート関数が処理不能な場合、NULL が返されることがプロトコルとして規定されています。

保守担当者向けメモ

  • 本構造体はプランナーだけでなく、実行エンジンからも利用される可能性があるため、プランナー専用のロジックと限定しないよう注意してください。
  • is_join が true の場合と false の場合で、参照すべきフィールド（varRelid や sjinfo）が異なります。
