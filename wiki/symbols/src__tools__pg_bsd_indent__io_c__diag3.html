diag3

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/io.c:572

目的

整形処理中に発生した警告またはエラーメッセージを、出力先（標準出力または標準エラー出力）に応じてフォーマットして出力する。

シグネチャまたは形状

void diag3(int level, const char *msg, int a)

入力と出力

  • **入力**:
  • level: メッセージの重大度（0の場合は「Warning」、それ以外は「Error」として扱う）。
  • msg: printf 形式のフォーマット文字列。
  • a: msg に埋め込むための整数値。
  • **出力**: なし（戻り値なし）。副作用として stdout または stderr へメッセージを出力し、エラー発生時はグローバル変数 found_err を 1 に設定する。

前提条件と不変条件

  • line_no（グローバル変数）が現在の行番号を保持していること。
  • output（グローバル変数）が有効なファイルポインタであること。

アルゴリズムまたは振る舞い

1. level が真であれば、グローバル変数 found_err を 1 に設定する。 2. output が stdout と等しい場合:

  • /**INDENT** [Warning|Error]@[line_no]: [msg] */ という形式で標準出力に書き出す。

    • 3. output が stdout 以外の場合:

  • [Warning|Error]@[line_no]: [msg] という形式で標準エラー出力に書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/tools/pg_bsd_indent/indent.c (複数箇所)
  • **呼び出し先**:
  • fprintf (標準ライブラリ)

関連データ構造

  • found_err: エラー発生状態を保持するグローバル変数。
  • line_no: 現在の処理行番号を保持するグローバル変数。
  • output: 出力先ストリームを保持するグローバル変数。

失敗モードと境界条件

  • msg に不正なフォーマット文字列が渡された場合、fprintf の挙動に依存する。
  • output が適切に初期化されていない場合、出力先が意図しないストリームになる可能性がある。

保守担当者向けメモ

  • 本関数は diag4 等の類似関数と共通のロジックを持つ。修正時は他の diag 系関数との整合性に注意すること。
  • level 引数の判定ロジック（0か否か）は、呼び出し側で一貫した値が渡されることを前提としている。
