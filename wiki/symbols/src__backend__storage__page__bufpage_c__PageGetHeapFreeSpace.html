PageGetHeapFreeSpace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/bufpage.c:989

目的

ヒープページにおいて、新しいタプルを挿入するために利用可能な空き容量を計算します。単なる物理的な空き領域だけでなく、ページ内のアイテムポインタ（ラインポインタ）の空き状況も考慮します。

シグネチャまたは形状

Size PageGetHeapFreeSpace(const PageData *page)

入力と出力

  • **入力**: page (対象となるページへのポインタ)
  • **出力**: Size 型のバイト数。挿入可能な空き容量を返します。

前提条件と不変条件

  • 入力 page は有効なページ構造である必要があります。
  • ページ内のアイテムポインタ数が MaxHeapTuplesPerPage に達している場合、物理的な空き領域があっても、未使用のラインポインタが存在しない限り、戻り値は 0 となります。

アルゴリズムまたは振る舞い

1. PageGetFreeSpace(page) を呼び出し、物理的な空き容量を取得します。 2. 物理的な空き容量が 0 より大きい場合、ページ内の現在のアイテムポインタ数（nline）を確認します。 3. nline が MaxHeapTuplesPerPage 以上の場合、ラインポインタの空き状況を判定します。

  • PageHasFreeLinePointers ヒントが有効な場合、実際に未使用のラインポインタがあるか走査します。未使用のポインタが見つからなければ、空き容量を 0 と見なします。
  • ヒントが無効な場合、PageAddItem の挙動に合わせて空き容量を 0 と見なします。

    • 4. 最終的な空き容量を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_xlog.c
  • src/backend/access/heap/hio.c
  • src/backend/access/heap/pruneheap.c
  • src/backend/access/heap/rewriteheap.c
  • **呼び出し先**:
  • PageGetFreeSpace
  • PageGetMaxOffsetNumber
  • PageHasFreeLinePointers
  • PageGetItemId
  • ItemIdIsUsed

関連データ構造

  • PageData: 操作対象のページ構造体。
  • ItemId: ページ内のアイテムポインタ。
  • MaxHeapTuplesPerPage: 1ページあたりの最大タプル数。

失敗モードと境界条件

  • ページが満杯（物理的空きがない、またはラインポインタが枯渇）の場合、戻り値は 0 となります。
  • PageHasFreeLinePointers のヒントが誤っている場合でも、本関数はページをダーティにできないため、ヒントに従った保守的な値を返します。

保守担当者向けメモ

  • 本関数は、ヒープページ特有の制約（ラインポインタの制限）を考慮した空き容量計算を行います。インデックスページ等には適用されません。
  • 内部で unconstify を使用して PageData をキャストしていますが、これは PageGetItemId のインターフェース要件によるものです。
