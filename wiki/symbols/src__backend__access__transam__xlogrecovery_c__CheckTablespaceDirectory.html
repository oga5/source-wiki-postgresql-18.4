CheckTablespaceDirectory

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogrecovery.c:2162

目的

リカバリ処理中に、pg_tblspc ディレクトリ内のエントリがすべてシンボリックリンクであることを検証し、データベースの整合性を確認します。

シグネチャまたは形状

static void CheckTablespaceDirectory(void)

入力と出力

  • **入力**: なし（PG_TBLSPC_DIR を直接参照）。
  • **出力**: なし。
  • **副作用**: 異常なエントリが検出された場合、ereport により WARNING または PANIC を発生させます。

前提条件と不変条件

  • PG_TBLSPC_DIR 内のすべてのディレクトリ・ファイルエントリは、シンボリックリンクである必要があります。
  • 数値のみで構成される名前のエントリは、テーブルスペースの OID と見なされます。

アルゴリズムまたは振る舞い

1. AllocateDir を使用して PG_TBLSPC_DIR を開きます。 2. ReadDir を用いてディレクトリ内の各エントリを走査します。 3. エントリ名が数値のみ（OID）である場合のみ検証対象とします。 4. get_dirent_type を呼び出し、対象がシンボリックリンク（PGFILETYPE_LNK）であるかを確認します。 5. シンボリックリンクでない場合、allow_in_place_tablespaces の設定値に応じて、WARNING または PANIC を発行してリカバリを中断または警告します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogrecovery.c 内のリカバリプロセス（詳細はソースコード上の明示的な呼び出し箇所として xlogrecovery.c:2264 が確認されています）。
  • **呼び出し先**: AllocateDir, ReadDir, get_dirent_type, ereport（いずれも PostgreSQL 内部のファイル操作およびエラー報告ユーティリティ）。

関連データ構造

  • PG_TBLSPC_DIR: テーブルスペースディレクトリのパス定数。
  • dirent: ディレクトリ内の各エントリ情報を保持する構造体。

失敗モードと境界条件

  • **失敗モード**: PG_TBLSPC_DIR 内にディレクトリやファイルが直接存在する場合（シンボリックリンクではない場合）、allow_in_place_tablespaces が false ならば PANIC でリカバリが停止します。
  • **境界条件**: allow_in_place_tablespaces が true の場合は、エラーではなく WARNING が発行されます。

保守担当者向けメモ

  • 本関数はリカバリの整合性チェックの一部です。テーブルスペースの配置に関する厳格なルールを強制するため、将来的に allow_in_place_tablespaces の挙動を変更する際は注意してください。
