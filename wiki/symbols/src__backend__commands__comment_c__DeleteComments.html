DeleteComments

定義

  • 種別: function
  • 定義箇所: src/backend/commands/comment.c:325

目的

指定されたデータベースオブジェクトに関連付けられたコメントをシステムカタログ pg_description から削除します。

シグネチャまたは形状

void DeleteComments(Oid oid, Oid classoid, int32 subid)

入力と出力

  • **入力**:
  • oid: 対象オブジェクトのOID。
  • classoid: 対象オブジェクトが属するカタログのOID。
  • subid: サブオブジェクトID（指定がない場合は0）。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • pg_description テーブルに対する RowExclusiveLock を取得します。
  • subid が 0 以外の場合、objoid、classoid、objsubid の3つのキーでインデックススキャンを行います。
  • subid が 0 の場合、objoid と classoid の2つのキーでインデックススキャンを行います。

アルゴリズムまたは振る舞い

1. ScanKeyInit を使用して、指定された oid、classoid、および subid に基づくスキャンキーを構築します。 2. table_open で pg_description を開きます。 3. systable_beginscan を用いて DescriptionObjIndexId インデックスを介したスキャンを開始します。 4. systable_getnext で一致するタプルを順次取得し、CatalogTupleDelete を呼び出して削除します。 5. systable_endscan と table_close を呼び出し、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/dependency.c:1333
  • **呼び出し先**:
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan
  • CatalogTupleDelete
  • ScanKeyInit

関連データ構造

  • pg_description: コメントを保持するシステムカタログ。
  • ScanKeyData: インデックススキャン用の検索条件構造体。
  • SysScanDesc: システムテーブルスキャンの状態管理構造体。

失敗モードと境界条件

  • 対象オブジェクトが存在しない場合、スキャンは何もヒットせず、削除処理は行われません。
  • subid が 0 の場合、そのオブジェクトに関連するすべてのサブオブジェクトのコメントも削除対象となります。

保守担当者向けメモ

  • 本関数は通常オブジェクト用です。共有オブジェクト（データベース、ロール等）を扱う場合は DeleteSharedComments を使用してください。
  • RowExclusiveLock を使用しているため、同時実行される他のカタログ操作とのデッドロックに注意してください。
