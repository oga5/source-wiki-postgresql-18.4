ginScanKeyAddHiddenEntry

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginscan.c:142

目的

GINインデックススキャンにおいて、検索条件の一部として明示的に指定されていないものの、インデックス構造上必要な「隠しエントリ（Hidden Entry）」を GinScanKey に追加します。

シグネチャまたは形状

static void ginScanKeyAddHiddenEntry(GinScanOpaque so, GinScanKey key, GinNullCategory queryCategory)

入力と出力

  • **入力**:
  • so: GINスキャンの状態を保持する GinScanOpaque 構造体。
  • key: エントリを追加対象とする GinScanKey 構造体。
  • queryCategory: 検索対象のカテゴリを示す GinNullCategory（例: GIN_CAT_EMPTY_ITEM や GIN_CAT_EMPTY_QUERY）。
  • **出力**: なし（key 構造体内の scanEntry 配列および nentries カウンタが更新されます）。

前提条件と不変条件

  • key->nentries が key->scanEntry 配列の許容範囲内であること。
  • so が有効なメモリコンテキストを保持していること。

アルゴリズムまたは振る舞い

1. 現在の key->nentries をインデックス i として取得し、key->nentries をインクリメントします。 2. ginFillScanEntry を呼び出し、以下の引数で新しい GinScanEntry を生成します。

  • attnum: key->attnum
  • strategy: InvalidStrategy（部分一致ではないため）
  • searchMode: key->searchMode
  • datum: (Datum) 0
  • queryCategory: 引数で受け取った queryCategory
  • isPartialMatch: false
  • extra_data: NULL

    • 3. 生成されたエントリを key->scanEntry[i] に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginscan.c 内の複数箇所（ginNewScanKey 等から、特定の検索モードや条件に応じて呼び出されます）。
  • **呼び出し先**:
  • ginFillScanEntry: 実際の GinScanEntry オブジェクトの構築に使用されます。

関連データ構造

  • GinScanOpaque: スキャン全体の状態管理。
  • GinScanKey: 個別のインデックス属性に対する検索キーの管理。
  • GinScanEntry: 検索キー内の個別のエントリ。
  • GinNullCategory: NULL値や空のアイテムを扱うためのカテゴリ定義。

失敗モードと境界条件

  • key->nentries が key->scanEntry 配列の最大サイズを超えた場合、メモリ破壊やクラッシュの可能性があります（本関数内でのチェックは明示されていません）。

保守担当者向けメモ

  • 本関数は、部分一致検索ではない特殊な検索条件（空のアイテムや空のクエリなど）をインデックススキャンに追加するために使用されます。
  • strategy に InvalidStrategy を渡している通り、通常のインデックス演算子による比較とは異なるパスで処理されるエントリを対象としています。
