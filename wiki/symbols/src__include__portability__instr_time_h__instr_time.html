instr_time

定義

  • 種別: struct
  • 定義箇所: src/include/portability/instr_time.h:69

目的

プラットフォーム固有のタイマーAPIを抽象化し、PostgreSQL全体で一貫した高精度な時間計測を行うための不透明なデータ型を提供します。

シグネチャまたは形状

typedef struct instr_time
{
    int64 ticks; /* プラットフォーム固有の単位 */
} instr_time;

入力と出力

  • 入力: INSTR_TIME_SET_CURRENT(t) マクロ等を通じて、プラットフォーム固有のタイマー値が ticks メンバに格納されます。
  • 出力: INSTR_TIME_GET_xxx マクロ群を使用し、ticks を秒、ミリ秒、マイクロ秒、ナノ秒などの単位に変換して取得します。

前提条件と不変条件

  • ticks メンバへの直接アクセスは推奨されません。提供されるマクロ（INSTR_TIME_ADD, INSTR_TIME_SUBTRACT 等）を介して操作する必要があります。

アルゴリズムまたは振る舞い

  • 内部的には int64 型のティック値を保持します。
  • Unix系環境では clock_gettime、Windows環境では QueryPerformanceCounter を利用して時刻を取得します。
  • マクロ引数は複数回評価される可能性があるため、副作用のある式を引数に渡さないよう注意が必要です。

呼び出し元と呼び出し先

  • 呼び出し元（参照例）:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • contrib/tsm_system_time/tsm_system_time.c
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/access/transam/xlog.c
  • 呼び出し先:
  • プラットフォーム固有のタイマー関数（pg_clock_gettime_ns または pg_query_performance_counter）が内部的に呼び出されます。

関連データ構造

  • src/include/portability/instr_time.h 内で定義される各種演算・変換マクロ。

失敗モードと境界条件

  • 内部メンバ ticks のオーバーフローについては、プラットフォーム固有のタイマーの仕様に依存します。

保守担当者向けメモ

  • 本構造体は不透明型として扱うこと。
  • 精度維持のため、変換は計測プロセスの最後に行うこと。
  • マクロ使用時の副作用に注意すること。
