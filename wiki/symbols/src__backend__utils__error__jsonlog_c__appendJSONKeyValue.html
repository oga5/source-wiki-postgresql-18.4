appendJSONKeyValue

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/jsonlog.c:42

目的

StringInfo バッファに対して、JSON 形式のキーと値のペアを追加します。

シグネチャまたは形状

static void appendJSONKeyValue(StringInfo buf, const char *key, const char *value, bool escape_value)

入力と出力

  • buf: 追加先の StringInfo バッファ（入力および出力）。
  • key: JSON のキー文字列（入力）。
  • value: JSON の値文字列（入力）。
  • escape_value: 値に対して escape_json を適用するかどうかを指定するフラグ（入力）。
  • 戻り値: なし。

前提条件と不変条件

  • key は NULL であってはなりません（Assert により検証）。
  • value が NULL の場合、関数は何も行わずに終了します。

アルゴリズムまたは振る舞い

1. バッファにカンマ , を追加します。 2. key を escape_json でエスケープしてバッファに追加します。 3. バッファにコロン : を追加します。 4. escape_value が true の場合、value を escape_json でエスケープして追加します。 5. escape_value が false の場合、value をそのまま appendStringInfoString で追加します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/error/jsonlog.c 内の複数の箇所（write_jsonlog 等）から、ログの各フィールドを出力するために呼び出されます。
  • 呼び出し先: escape_json（utils/json.h）、appendStringInfoChar、appendStringInfoString（lib/stringinfo.h）。

関連データ構造

  • StringInfo: 可変長文字列バッファ。

失敗モードと境界条件

  • key が NULL の場合、アサーション違反により異常終了します。
  • value が NULL の場合、何も出力されません。

保守担当者向けメモ

  • キーは常にエスケープされますが、値は escape_value フラグにより制御されます。数値や既にフォーマット済みの文字列など、エスケープが不要な場合はこのフラグを false に設定してください。
  • StringInfo を使用しているため、メモリ管理については呼び出し元のライフサイクルに依存します。
