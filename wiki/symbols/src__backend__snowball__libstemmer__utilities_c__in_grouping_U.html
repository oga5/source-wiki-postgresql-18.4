in_grouping_U

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/utilities.c:117

目的

UTF-8 エンコードされた文字列において、現在のカーソル位置にある文字が指定された文字集合（グループ）に含まれるかを判定し、必要に応じてカーソルを進める。

シグネチャまたは形状

int in_grouping_U(struct SN_env * z, const unsigned char * s, int min, int max, int repeat)

入力と出力

  • z: ステミング環境を保持する SN_env 構造体へのポインタ。
  • s: 文字集合をビットマップ形式で保持するバイト配列。
  • min, max: 判定対象となる文字コードの範囲。
  • repeat: 0 以外の場合、グループに一致する限りカーソルを連続して進める。
  • 戻り値:
  • 0: 一致しなかった場合。
  • 0 以外（正の整数）: 一致した文字のバイト長（カーソルを進めた量）。
  • -1: UTF-8 デコードに失敗した場合。

前提条件と不変条件

  • z->c は現在の処理位置（カーソル）を指す。
  • s は min から max までの範囲をカバーするビットマップである必要がある。
  • z->p（文字列バッファ）および z->l（文字列長）が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. get_utf8 を呼び出し、現在のカーソル位置から文字をデコードする。 2. デコード失敗時は -1 を返す。 3. 文字コード ch が min 未満または max を超える場合、グループ外とみなして直ちにその文字のバイト長を返す。 4. ch が範囲内の場合、s のビットマップを参照し、該当ビットが立っているか確認する。 5. ビットが立っていない（グループ外）場合は、その文字のバイト長を返す。 6. ビットが立っている（グループ内）場合、カーソル z->c を進め、repeat が真であれば 1 に戻り処理を繰り返す。 7. 最終的にグループ外の文字に到達したか、範囲外の文字に到達した時点で終了する。

呼び出し元と呼び出し先

  • 呼び出し先: get_utf8
  • 呼び出し元（直接的な証拠）:
  • stem_UTF_8_armenian.c
  • stem_UTF_8_basque.c
  • stem_UTF_8_catalan.c
  • stem_UTF_8_danish.c
  • stem_UTF_8_dutch.c

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • among: 辞書検索等で使用される構造体（本関数では直接使用しないが、モジュール内で関連）。

失敗モードと境界条件

  • get_utf8 が不正な UTF-8 シーケンスを検出した場合、-1 を返し処理を中断する。
  • repeat が 0 の場合、グループに一致してもカーソルは最大 1 文字分しか進まない。
  • ch が min から max の範囲外である場合、ビットマップ参照を行わずにグループ外と判定する。

保守担当者向けメモ

  • 本関数は UTF-8 専用である。ASCII 等の非 UTF-8 環境では in_grouping を使用すること。
  • ビットマップ s のインデックス計算 s[ch >> 3] & (0X1 << (ch & 0X7)) は、文字コードのオフセットを考慮した標準的な実装である。
