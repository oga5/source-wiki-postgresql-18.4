pgstat_gist_page

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:502

目的

GiSTインデックスの特定のページを走査し、統計情報（空き容量やタプル数）を収集して pgstattuple_type 構造体に反映させる。

シグネチャまたは形状

static void pgstat_gist_page(pgstattuple_type *stat, Relation rel, BlockNumber blkno, BufferAccessStrategy bstrategy)

入力と出力

  • **入力**:
  • stat: 統計情報を格納する pgstattuple_type 構造体へのポインタ。
  • rel: 対象の GiST インデックスリレーション。
  • blkno: 走査対象のブロック番号。
  • bstrategy: バッファアクセスの戦略。
  • **出力**: なし（stat 構造体のフィールドを直接更新する）。

前提条件と不変条件

  • 指定された blkno は有効な GiST インデックスページである必要がある。
  • 走査中は GIST_SHARE ロックが保持される。

アルゴリズムまたは振る舞い

1. ReadBufferExtended を使用して指定されたブロックを読み込み、GIST_SHARE モードでロックする。 2. PageIsNew でページが空か判定し、空であれば stat->free_space に BLCKSZ を加算する。 3. PageGetSpecialSize を確認し、GiST ページ構造として妥当か検証する。 4. GistPageIsLeaf を使用してリーフページかどうかを判定する。

  • リーフページの場合: pgstat_index_page を呼び出し、ページ内の全オフセットを走査して統計を更新する。
  • 非リーフページ（ルートまたは内部ノード）の場合: 統計更新は行わずスキップする。

    • 5. UnlockReleaseBuffer を呼び出し、バッファのロック解除と解放を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgstattuple/pgstattuple.c 内のインデックス走査ロジックから呼び出される（直接的な呼び出し元関数の詳細はソース抜粋からは特定不可）。
  • **呼び出し先**:
  • ReadBufferExtended, LockBuffer, BufferGetPage, UnlockReleaseBuffer (PostgreSQL ストレージマネージャ)
  • PageIsNew, PageGetSpecialSize, PageGetMaxOffsetNumber (PostgreSQL ページ管理)
  • GistPageIsLeaf (GiST アクセスメソッド)
  • pgstat_index_page (本モジュール内)

関連データ構造

  • pgstattuple_type: 統計情報の集計結果を保持する構造体。
  • Relation: インデックスリレーションのメタデータ。
  • Buffer: バッファマネージャが管理するページバッファ。
  • GISTPageOpaqueData: GiST ページ固有のヘッダ情報。

失敗モードと境界条件

  • ページが新規（PageIsNew）の場合、空き容量として BLCKSZ が加算される。
  • GiST ページ構造が期待と異なる場合（PageGetSpecialSize が不一致）、統計更新は行われず終了する。
  • 非リーフページについては、現在の実装では統計情報の集計対象外となる。

保守担当者向けメモ

  • 本関数は GiST インデックス専用の走査ロジックである。
  • pgstat_index_page を介してリーフノードのタプルをカウントしているため、GiST の内部構造変更時には GistPageIsLeaf の判定条件と併せて確認が必要である。
