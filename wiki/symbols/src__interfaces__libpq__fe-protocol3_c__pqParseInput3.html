pqParseInput3

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:69

目的

PostgreSQL プロトコル 3.0 におけるメインの入力解析ループであり、ネットワークバッファから受信した生のバイトストリームを解析し、メッセージタイプに応じて適切なハンドラへ振り分けます。

シグネチャまたは形状

void pqParseInput3(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn（接続状態を保持する構造体）。conn->inBuffer に格納された未処理のデータを使用します。
  • **出力**: なし（戻り値は void）。副作用として conn 内の asyncStatus、result、inCursor 等の状態が更新されます。

前提条件と不変条件

  • 非ブロッキング I/O を前提としており、バッファ内に完全なメッセージが存在しない場合は即座にリターンします。
  • メッセージ長が 4 バイト未満、または不正な長さである場合は handleSyncLoss を呼び出して同期喪失を処理します。
  • メッセージ解析中にメモリ不足が発生した場合は handleFatalError を呼び出し、接続を強制終了します。

アルゴリズムまたは振る舞い

1. **ループ処理**: バッファ内の完全なメッセージを消費し続けるまでループします。 2. **ヘッダー読み込み**: pqGetc でメッセージタイプ（1バイト）、pqGetInt でメッセージ長（4バイト）を読み取ります。 3. **検証**: メッセージ長が妥当か確認し、不足している場合は pqCheckInBufferSpace でバッファ拡張を試みた後、リターンします。 4. **ディスパッチ**:

  • NotificationResponse や NoticeResponse は状態に関わらず即時処理します。
  • IDLE 状態では ErrorResponse や ParameterStatus のみを処理し、それ以外は警告を出して破棄します。
  • BUSY 状態では CommandComplete、ErrorResponse、ReadyForQuery 等の主要なメッセージを処理し、状態を PGASYNC_READY へ遷移させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-connect.c および src/interfaces/libpq/fe-exec.c から呼び出されます。
  • **呼び出し先**: pqGetc, pqGetInt, handleSyncLoss, pqCheckInBufferSpace, handleFatalError, getNotify, pqGetErrorNotice3, getParameterStatus, pqInternalNotice, pqGets, pgHavePendingResult, PQmakeEmptyPGresult, libpq_append_conn_error, pqSaveErrorResult, getReadyForQuery, pqCommandQueueAdvance 等。

関連データ構造

  • PGconn: 接続状態、入力バッファ（inBuffer）、カーソル位置（inCursor）、非同期状態（asyncStatus）を管理します。
  • PGresult: 解析結果を格納する構造体。

失敗モードと境界条件

  • **バッファ不足**: メッセージ全体が読み込めていない場合、処理を中断してリターンします。
  • **不正なメッセージ長**: msgLength < 4 または巨大なメッセージ（VALID_LONG_MESSAGE_TYPE 以外）の場合、同期喪失として扱われます。
  • **メモリ不足**: バッファ拡張失敗時に致命的エラーとして接続を終了します。

保守担当者向けメモ

  • 本関数はプロトコル 3.0 専用です。
  • IDLE 状態での予期せぬメッセージ受信時は、接続状態を維持するために一部のメッセージを除き破棄する設計となっています。
  • pqParseInput3 がメッセージを完全に消費したことを示すために、各ハンドラは適切にカーソルを進める必要があります。
