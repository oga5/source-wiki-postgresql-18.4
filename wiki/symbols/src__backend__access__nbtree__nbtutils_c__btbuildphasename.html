btbuildphasename

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtutils.c:3727

目的

B-tree インデックス構築処理における現在の進捗フェーズ番号に対応する、人間が読み取り可能な名称（文字列）を返す。

シグネチャまたは形状

char *btbuildphasename(int64 phasenum)

入力と出力

  • **入力**: phasenum (int64) - インデックス構築のサブフェーズを示す数値。
  • **出力**: 対応するフェーズ名の文字列（char *）。該当するフェーズがない場合は NULL を返す。

前提条件と不変条件

  • phasenum は commands/progress.h 等で定義されたインデックス構築フェーズの定数と一致している必要がある。

アルゴリズムまたは振る舞い

switch 文を用いて phasenum を評価し、以下のいずれかの文字列を返す：

  • PROGRESS_CREATEIDX_SUBPHASE_INITIALIZE → "initializing"
  • PROGRESS_BTREE_PHASE_INDEXBUILD_TABLESCAN → "scanning table"
  • PROGRESS_BTREE_PHASE_PERFORMSORT_1 → "sorting live tuples"
  • PROGRESS_BTREE_PHASE_PERFORMSORT_2 → "sorting dead tuples"
  • PROGRESS_BTREE_PHASE_LEAF_LOAD → "loading tuples in tree"
  • 上記以外 → NULL

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtree.c において、amroutine->ambuildphasename に関数ポインタとして代入されている。
  • **呼び出し先**: 本関数は他の関数を呼び出さない。

関連データ構造

  • int64 (フェーズ番号)
  • commands/progress.h (フェーズ定数の定義元)

失敗モードと境界条件

  • 定義されていない phasenum が渡された場合、NULL を返す。

保守担当者向けメモ

  • 新しいインデックス構築フェーズが追加された場合は、本関数の switch 文にケースを追加すること。
  • 本関数は進捗報告のためのユーティリティであり、インデックスの物理的な構築ロジックには直接関与しない。
