UpdateSubscriptionRelState

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_subscription.c:320

目的

論理レプリケーションにおける特定のテーブルとサブスクリプションの紐付け状態（srsubstate）および LSN を更新します。

シグネチャまたは形状

void UpdateSubscriptionRelState(Oid subid, Oid relid, char state, XLogRecPtr sublsn, bool already_locked)

入力と出力

  • **入力**:
  • subid: 対象サブスクリプションの OID。
  • relid: 対象リレーションの OID。
  • state: 更新後の状態を示す文字コード。
  • sublsn: 更新する LSN（InvalidXLogRecPtr の場合は NULL として扱われる）。
  • already_locked: 呼び出し元で既に適切なロックが保持されているかを示すフラグ。
  • **出力**: なし（副作用としてシステムカタログ pg_subscription_rel を更新）。

前提条件と不変条件

  • 本関数はトランザクション内で呼び出される必要があります。
  • already_locked が true の場合、呼び出し元は SubscriptionRelRelationId に対して RowExclusiveLock を、かつ subid に対して AccessShareLock を保持している必要があります（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. **ロック取得**: already_locked が false の場合、subid に対して AccessShareLock を取得し、pg_subscription_rel テーブルを RowExclusiveLock で開きます。 2. **検索**: SUBSCRIPTIONRELMAP システムキャッシュを使用して、指定された subid と relid に一致するタプルをコピーとして取得します。存在しない場合はエラーを発生させます。 3. **更新**: 取得したタプルの srsubstate を state に、srsublsn を sublsn（有効な値の場合）に書き換えます。 4. **永続化**: CatalogTupleUpdate を呼び出し、システムカタログを更新します。 5. **終了**: table_close を呼び出し、リレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/tablesync.c 内の各所（論理レプリケーションのテーブル同期プロセス）。
  • **呼び出し先**:
  • table_open, table_close
  • SearchSysCacheCopy2
  • heap_modify_tuple
  • CatalogTupleUpdate
  • LockSharedObject

関連データ構造

  • pg_subscription_rel: 更新対象のシステムカタログ。
  • SubscriptionRelRelationId: pg_subscription_rel の OID。
  • SUBSCRIPTIONRELMAP: pg_subscription_rel を検索するためのシステムキャッシュ ID。

失敗モードと境界条件

  • 指定された subid と relid の組み合わせが pg_subscription_rel に存在しない場合、elog(ERROR) によりトランザクションが中断されます。
  • sublsn が InvalidXLogRecPtr の場合、カタログ上の srsublsn は NULL に設定されます。

保守担当者向けメモ

  • 本関数は pg_subscription_rel の状態遷移を直接制御するため、論理レプリケーションの同期進行に直結します。
  • already_locked フラグは、デッドロック回避やロックの重複取得を防ぐために重要です。呼び出し元でロックを管理する際は、アサーションで指定されたロックレベルが維持されていることを確認してください。
