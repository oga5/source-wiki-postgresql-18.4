cancel_parser_errposition_callback

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_node.c:155

目的

パーサーのエラー報告用コールバックをスタックから削除し、エラーコンテキストを以前の状態に復元します。

シグネチャまたは形状

void cancel_parser_errposition_callback(ParseCallbackState *pcbstate)

入力と出力

  • **入力**: pcbstate (ParseCallbackState *) - 登録時に使用したコールバック状態構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • pcbstate は、事前に setup_parser_errposition_callback によって正しく初期化および登録されている必要があります。
  • 本関数呼び出しにより、グローバル変数 error_context_stack が更新されます。

アルゴリズムまたは振る舞い

1. pcbstate 内に保持されている errcallback.previous（以前のコールバックスタックの先頭）を参照します。 2. グローバル変数 error_context_stack を、その previous 値で上書きすることで、現在のコールバックをスタックから取り除きます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c
  • src/backend/parser/parse_clause.c
  • src/backend/parser/parse_coerce.c
  • src/backend/parser/parse_func.c
  • src/backend/parser/parse_node.c
  • src/backend/parser/parse_oper.c
  • src/backend/parser/parse_relation.c
  • src/backend/parser/parse_type.c
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。グローバル変数 error_context_stack を直接操作します。

関連データ構造

  • ParseCallbackState: エラーコールバックの管理に使用される構造体。errcallback メンバを介してスタック構造を保持します。

失敗モードと境界条件

  • pcbstate が NULL である場合や、スタックが正しく構築されていない状態で呼び出された場合の動作は未定義です。

保守担当者向けメモ

  • 本関数は setup_parser_errposition_callback と対になるクリーンアップ処理です。解析処理の終了時やエラー発生後の後処理において、必ず呼び出される必要があります。
