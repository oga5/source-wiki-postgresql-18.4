create_plan

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/createplan.c:336

目的

プランナが選択した Path ツリーを実行エンジンが解釈可能な Plan ツリーへ変換するエントリポイントです。

シグネチャまたは形状

Plan *create_plan(PlannerInfo *root, Path *best_path)

入力と出力

  • **入力**:
  • root: クエリ全体の最適化情報を保持する PlannerInfo 構造体。
  • best_path: 最適と判断された Path ツリーのルートノード。
  • **出力**:
  • 変換後の実行可能な Plan ツリーのルートノード。

前提条件と不変条件

  • root->plan_params が NIL であること（現在のクエリレベルで未使用であること）。
  • 実行終了時に root->curOuterParams が NIL であること（すべての NestLoopParams がプランノードに割り当て済みであること）。

アルゴリズムまたは振る舞い

1. **初期化**: PlannerInfo 内のワークスペース（curOuterRels, curOuterParams）を初期化します。 2. **再帰的変換**: create_plan_recurse を呼び出し、CP_EXACT_TLIST フラグを指定して Path ツリーを Plan ツリーへ変換します。 3. **ラベル付け**: ModifyTable ノード以外の場合、apply_tlist_labeling を用いて最上位ノードのターゲットリストに列名等の装飾情報を適用します。 4. **初期化プランの結合**: SS_attach_initplans を呼び出し、クエリレベルの InitPlan を最上位ノードにアタッチします。 5. **検証とクリーンアップ**: NestLoopParams の割り当て漏れがないか確認し、root->plan_params をリセットして終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c (planner)
  • src/backend/optimizer/plan/createplan.c (再帰的なサブクエリ処理時)
  • **呼び出し先**:
  • create_plan_recurse
  • apply_tlist_labeling
  • SS_attach_initplans

関連データ構造

  • PlannerInfo: クエリの最適化状態を管理。
  • Path: 最適化されたパスツリーのノード。
  • Plan: 実行エンジンが使用する実行計画ツリーのノード。

失敗モードと境界条件

  • root->curOuterParams が NIL でない場合、NestLoopParams の割り当て失敗として elog(ERROR) を発生させます。

保守担当者向けメモ

  • 本関数は Path から Plan への変換の最上位層です。
  • ModifyTable ノードはターゲットリストの扱いが特殊であるため、ラベル付けの対象外となっています。
  • InitPlan のアタッチ位置については、SS_finalize_plan のコメントを参照してください。
