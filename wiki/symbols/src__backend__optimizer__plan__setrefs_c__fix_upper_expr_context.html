fix_upper_expr_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:76

目的

プランツリーの上位ノード（Upper Plan Nodes）における式ツリーの変数参照を解決・変換する際、再帰的な走査（fix_upper_expr_mutator）に渡すための状態保持用コンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    PlannerInfo *root;
    indexed_tlist *subplan_itlist;
    int         newvarno;
    int         rtoffset;
    NullingRelsMatch nrm_match;
    double      num_exec;
} fix_upper_expr_context;

入力と出力

本構造体は関数引数として渡されるコンテキストであり、直接的な入力・出力はありません。各フィールドは以下の役割を持ちます。

  • root: プラン全体の PlannerInfo へのポインタ。
  • subplan_itlist: 変数解決の参照先となるインデックス付きターゲットリスト。
  • newvarno: 変換後の変数に割り当てる新しい varno。
  • rtoffset: レンジテーブルのオフセット値。
  • nrm_match: NULL 許容関係の照合ルール。
  • num_exec: 推定実行回数。

前提条件と不変条件

  • subplan_itlist は、対象となるサブプランの出力が正しくインデックス化されている必要があります。
  • root は有効な PlannerInfo を指している必要があります。

アルゴリズムまたは振る舞い

fix_upper_expr_mutator 関数において、式ツリーを走査する際のパラメータとして利用されます。このコンテキストを介して、式内の Var ノードをサブプランの出力位置へマッピングし、必要に応じて varno や varattno を書き換えます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/setrefs.c 内の関数（fix_upper_expr_mutator 等）でインスタンス化され、使用されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しは行いません。

関連データ構造

  • PlannerInfo: プラン作成時のグローバル情報。
  • indexed_tlist: ターゲットリストの高速検索用構造体。
  • NullingRelsMatch: 外部結合時の NULL 許容関係を管理する列挙型。

失敗モードと境界条件

  • subplan_itlist が適切に初期化されていない場合、変数参照の解決に失敗し、プランの整合性が失われます。
  • newvarno や rtoffset の設定が不適切な場合、実行エンジンが誤ったタプルスロットを参照する可能性があります。

保守担当者向けメモ

  • 本構造体は fix_scan_expr_context や fix_join_expr_context と類似の目的を持ちますが、上位ノード特有の処理（num_exec 等）を保持しています。
  • 構造体のフィールドを追加・変更する場合は、fix_upper_expr_mutator 内での参照箇所をすべて確認してください。
