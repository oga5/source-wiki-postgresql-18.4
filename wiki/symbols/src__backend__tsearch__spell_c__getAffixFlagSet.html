getAffixFlagSet

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/spell.c:1169

目的

ISpell/Hunspell 辞書設定において、接辞フラグのエイリアス（数値）を実際の接辞文字列に解決する。

シグネチャまたは形状

const char *getAffixFlagSet(IspellDict *Conf, char *s)

入力と出力

  • **入力**:
  • Conf: IspellDict 構造体へのポインタ。
  • s: 解決対象の接辞フラグ文字列（数値の文字列表現またはフラグそのもの）。
  • **出力**:
  • Conf->useFlagAliases が有効な場合、s に対応する Conf->AffixData 内の文字列、または VoidString。
  • useFlagAliases が無効な場合、入力された s をそのまま返す。

前提条件と不変条件

  • Conf->useFlagAliases が真である場合、s は数値としてパース可能である必要がある。
  • Conf->AffixData は NIImportOOAffixes によって構築済みであること。

アルゴリズムまたは振る舞い

1. Conf->useFlagAliases が偽、または s が空文字の場合は、入力 s をそのまま返す。 2. useFlagAliases が真の場合、s を strtol で整数 curaffix に変換する。 3. 変換失敗時（s == end）や範囲外（errno == ERANGE）の場合はエラーを報告する。 4. curaffix が 0 より大きく Conf->nAffixData 未満であれば、Conf->AffixData[curaffix] を返す。 5. curaffix が Conf->nAffixData を超える場合はエラーを報告する。 6. 上記以外（curaffix == 0 等）の場合は VoidString を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/spell.c:1170 (定義内での再帰的参照の可能性あり)
  • src/backend/tsearch/spell.c:1398
  • **呼び出し先**:
  • strtol (標準ライブラリ)
  • ereport (PostgreSQL エラー報告マクロ)

関連データ構造

  • IspellDict: 辞書の状態と接辞データを保持する構造体。
  • AffixData: IspellDict 内の接辞文字列配列。

失敗モードと境界条件

  • **エラー**: s が数値として解釈できない場合、または curaffix が Conf->nAffixData を超える場合に ERRCODE_CONFIG_FILE_ERROR でエラー終了する。
  • **境界条件**: s が空文字の場合は変換を行わずそのまま返す。

保守担当者向けメモ

  • NIImportOOAffixes で空文字列が AffixData に追加されているため、インデックスのオフセット調整は不要である点に注意すること。
