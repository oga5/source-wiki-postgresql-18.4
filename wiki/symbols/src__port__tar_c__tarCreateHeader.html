tarCreateHeader

定義

  • 種別: function
  • 定義箇所: src/port/tar.c:142

目的

ファイルメタデータから、POSIX 準拠の 512 バイト tar ヘッダーブロックを生成します。

シグネチャまたは形状

enum tarError tarCreateHeader(char *h, const char *filename, const char *linktarget,
                              pgoff_t size, mode_t mode, uid_t uid, gid_t gid, time_t mtime);

入力と出力

  • **入力**:
  • h: 512 バイト以上の書き込み先バッファ。
  • filename: ファイル名（最大 99 バイト）。
  • linktarget: シンボリックリンク先（NULL 可、最大 99 バイト）。
  • size, mode, uid, gid, mtime: ファイル属性。
  • **出力**:
  • h: 構築された tar ヘッダーデータが格納される。
  • 戻り値: 成功時は TAR_OK、エラー時は TAR_NAME_TOO_LONG または TAR_SYMLINK_TOO_LONG。

前提条件と不変条件

  • filename および linktarget は 99 バイト以下であること。
  • h が指すメモリ領域は TAR_BLOCK_SIZE (512 バイト) 分の書き込みが可能であること。
  • チェックサムは、他の全フィールドが埋められた後に計算・挿入される。

アルゴリズムまたは振る舞い

1. filename と linktarget の長さを検証し、超過していればエラーを返す。 2. バッファ h を 512 バイト分ゼロクリアする。 3. ファイル名、モード、UID、GID、サイズ、更新時刻を各オフセットに書き込む。 4. linktarget が存在する場合、またはディレクトリの場合は、名前に末尾スラッシュを付与し、適切な typeflag を設定する。 5. ustar マジックナンバー、バージョン、ユーザー名/グループ名（固定値 "postgres"）を書き込む。 6. 最後に tarChecksum を呼び出し、計算結果をチェックサムフィールドに書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/bin/pg_basebackup/walmethods.c
  • src/bin/pg_dump/pg_backup_tar.c
  • src/fe_utils/astreamer_tar.c
  • **呼び出し先**:
  • tarChecksum (チェックサム計算用)
  • print_tar_number (数値フィールド変換用)

関連データ構造

  • enum tarError: 戻り値のステータスコード。
  • pgtar.h: TAR_BLOCK_SIZE や各フィールドのオフセット定数が定義されている。

失敗モードと境界条件

  • filename または linktarget が 99 バイトを超過する場合、それぞれ TAR_NAME_TOO_LONG または TAR_SYMLINK_TOO_LONG を返す。
  • シンボリックリンクやディレクトリの場合、サイズフィールドは 0 として扱われる。

保守担当者向けメモ

  • 本関数は POSIX tar 形式に準拠しており、ファイル名等の長さに制限がある。
  • ユーザー名およびグループ名としてハードコードされた "postgres" を変更する必要があるかは未検討。
  • チェックサム計算時には、チェックサムフィールド自体は「8 つのスペース」として扱われる点に注意すること。
