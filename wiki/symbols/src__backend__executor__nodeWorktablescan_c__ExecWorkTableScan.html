ExecWorkTableScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWorktablescan.c:80

目的

再帰的クエリ（WITH RECURSIVE）の実行中、RecursiveUnion ノードが保持する作業テーブル（tuplestore）からタプルを順次取得し、スキャンを実行する。

シグネチャまたは形状

static TupleTableSlot *ExecWorkTableScan(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（WorkTableScanState へのポインタとしてキャストされる）。
  • **出力**: TupleTableSlot *（スキャンされたタプル、またはスキャン終了時は NULL）。

前提条件と不変条件

  • RecursiveUnion ノードが投影（Projection）を行わないことを前提とする。
  • 初回呼び出し時に node->rustate が NULL である場合、ParamExecData を介して親の RecursiveUnionState を解決する。
  • ExecScan を呼び出す前に、スキャン型と投影情報の初期化が完了している必要がある。
  • 後方スキャン（Backward Scan）はサポートされない。

アルゴリズムまたは振る舞い

1. **遅延初期化**: 初回呼び出し時、estate->es_param_exec_vals から RecursiveUnionState を取得し、node->rustate に格納する。 2. **メタデータ設定**: ExecAssignScanType でスキャン対象の行型を確定し、ExecAssignScanProjectionInfo で投影情報を初期化する。 3. **スキャン実行**: ExecScan を呼び出し、WorkTableScanNext（タプル取得）および WorkTableScanRecheck（再チェック）を介してタプルを順次取得する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（ExecProcNode 経由で呼び出される）。
  • **呼び出し先**:
  • ExecScan（スキャン実行の共通基盤）
  • WorkTableScanNext（タプル取得の実務関数）
  • WorkTableScanRecheck（再チェック関数）
  • ExecAssignScanType / ExecAssignScanProjectionInfo（初期化用ユーティリティ）

関連データ構造

  • WorkTableScanState: 本ノードの実行状態を保持する構造体。
  • RecursiveUnionState: 親ノードの状態。rustate として参照される。
  • TupleTableSlot: タプルの格納に使用される。
  • ParamExecData: 親ノードの状態を特定するためのパラメータデータ。

失敗モードと境界条件

  • rustate の解決時に param->execPlan が NULL でない場合、または param->isnull が真である場合はアサーションエラーとなる。
  • RecursiveUnion ノードの初期化前に本ノードが呼び出される可能性があるため、遅延初期化が必須となっている。

保守担当者向けメモ

  • 本関数は RecursiveUnion ノードの tuplestore を唯一の読み手として消費する。
  • rustate の解決は、実行順序の依存関係を回避するために初回実行時まで遅延される設計である。
