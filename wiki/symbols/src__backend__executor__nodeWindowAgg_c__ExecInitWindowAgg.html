ExecInitWindowAgg

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/nodeWindowAgg.c:2478

目的

ウィンドウ関数を実行するための WindowAggState ノードを初期化し、実行に必要なメモリコンテキスト、式コンテキスト、タプルスロット、および集約状態管理構造を準備する。

シグネチャまたは形状

WindowAggState *ExecInitWindowAgg(WindowAgg *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の WindowAgg プランノード。
  • estate: クエリ実行状態を保持する EState。
  • eflags: 実行フラグ（EXEC_FLAG_BACKWARD および EXEC_FLAG_MARK はサポート外）。
  • **出力**:
  • 初期化済みの WindowAggState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと（Assert で検証）。
  • node->plan.qual が存在する場合、そのノードは topWindow である必要がある。
  • frameOptions に応じて、framehead_slot および frametail_slot が必要最小限の条件で初期化される。

アルゴリズムまたは振る舞い

1. WindowAggState ノードを割り当て、ExecWindowAgg を実行関数として設定。 2. パーティション管理用の partcontext および集約状態用の aggcontext を作成。 3. ExecInitQual を用いて qual および runCondition を初期化。 4. 子ノードを ExecInitNode で初期化。 5. TTSOpsMinimalTuple を用いてスキャン用および作業用のタプルスロットを初期化。 6. RANGE または GROUPS モードの場合、フレーム境界計算用のスロットを条件付きで作成。 7. execTuplesMatchPrepare を呼び出し、パーティションおよびソートキーの比較関数を準備。 8. 集約関数およびウィンドウ関数用の状態配列（perfunc, peragg）を割り当て、初期化の準備を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c（ExecInitNode 経由）。
  • **呼び出し先**: ExecInitNode, ExecAssignExprContext, AllocSetContextCreate, ExecInitQual, ExecCreateScanSlotFromOuterPlan, ExecInitExtraTupleSlot, ExecInitResultTupleSlotTL, ExecAssignProjectionInfo, execTuplesMatchPrepare, palloc0。

関連データ構造

  • WindowAggState: 実行時の状態を保持するメイン構造体。
  • WindowAgg: プランノード。
  • WindowStatePerFuncData, WindowStatePerAggData: 関数および集約ごとの実行状態。
  • TupleTableSlot: タプル保持用スロット。

失敗モードと境界条件

  • work_mem 不足時には、パーティションデータがディスクにスピルする可能性がある（tuplestore の挙動に依存）。
  • frameOptions の組み合わせにより、不要なスロットは NULL に初期化される。

保守担当者向けメモ

  • 本関数は WindowAgg ノードのセットアップのみを行う。実際の実行ロジックは ExecWindowAgg に委譲される。
  • partcontext はパーティションごとのメモリ管理に使用されるため、メモリリークを防ぐために ExecEndWindowAgg での適切な解放が必須である。
