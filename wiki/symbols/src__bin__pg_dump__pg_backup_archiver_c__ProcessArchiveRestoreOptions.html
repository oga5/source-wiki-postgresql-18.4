ProcessArchiveRestoreOptions

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:287

目的

アーカイブ内の各TOC（Table of Contents）エントリに対して、復元またはダンプの対象となるかを判定し、要求フラグ（reqs）を更新する。また、アーカイブ作成時にはセクション順序の整合性を検証する。

シグネチャまたは形状

void ProcessArchiveRestoreOptions(Archive *AHX)

入力と出力

  • **入力**: Archive *AHX （ArchiveHandle 構造体へのポインタ）
  • **出力**: なし（副作用として AH->toc 内の各 TocEntry の reqs フィールドを更新する）

前提条件と不変条件

  • AHX は有効な ArchiveHandle であること。
  • AH->toc が初期化済みであること。
  • ropt（復元オプション）が AH->public に設定されていること。

アルゴリズムまたは振る舞い

1. アーカイブのTOCリストを走査し、各エントリのセクション（SECTION_PRE_DATA, SECTION_DATA, SECTION_POST_DATA）を追跡する。 2. **順序検証**: 書き込みモード（archModeRead 以外）の場合、エントリのセクション順序が不正であれば警告を発行する。 3. **フラグ更新**: _tocEntryRequired 関数を呼び出し、現在のセクションとアーカイブ設定に基づいて te->reqs を設定する。 4. **厳密な名前チェック**: ropt->strict_names が有効な場合、StrictNamesCheck を実行して名前の整合性を検証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c:1237
  • src/bin/pg_dump/pg_restore.c:516
  • **呼び出し先**:
  • _tocEntryRequired（内部ヘルパー）
  • StrictNamesCheck（名前検証）
  • pg_log_warning / pg_fatal（ログ出力）

関連データ構造

  • ArchiveHandle: アーカイブ全体の状態を保持。
  • TocEntry: ダンプ対象オブジェクトのメタデータと依存関係を保持。
  • RestoreOptions: ユーザー指定のフィルタリング設定。

失敗モードと境界条件

  • **順序エラー**: 書き込みモードでセクション順序が不正な場合、警告が出力される。
  • **予期せぬセクション**: SECTION_NONE 以外の未知のセクションコードが検出された場合、pg_fatal により処理が中断される。

保守担当者向けメモ

  • 本関数は pg_dump と pg_restore の両方から呼び出される。
  • _tocEntryRequired のロジック変更は、本関数のマーキング結果に直接影響を与えるため注意すること。
