addCompiledLexeme

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_thesaurus.c:302

目的

シソーラス辞書のコンパイルプロセスにおいて、新しい語彙（TheLexeme）を配列に追加し、必要に応じてメモリ領域を拡張します。

シグネチャまたは形状

static TheLexeme *addCompiledLexeme(
    TheLexeme *newwrds, 
    int *nnw, 
    int *tnm, 
    TSLexeme *lexeme, 
    LexemeInfo *src, 
    uint16 tnvariant
)

入力と出力

  • **入力**:
  • newwrds: 現在の語彙配列へのポインタ。
  • nnw: 現在の要素数（ポインタ）。
  • tnm: 現在の配列容量（ポインタ）。
  • lexeme: 追加対象の語彙情報（TSLexeme）。
  • src: 関連付けられる LexemeInfo。
  • tnvariant: バリアント識別子。
  • **出力**: 拡張または更新された TheLexeme 配列のポインタを返します。

前提条件と不変条件

  • nnw が tnm に達した場合、repalloc により配列容量が2倍に拡張されます。
  • lexeme が有効な場合、その文字列が複製（pstrdup）され、tnvariant が設定されます。
  • lexeme が NULL の場合、語彙は NULL となり、tnvariant は 1 に固定されます。
  • entries->nextentry は常に NULL で初期化されます。

アルゴリズムまたは振る舞い

1. 現在の要素数 *nnw が容量 *tnm 以上であれば、repalloc で配列を2倍に拡張します。 2. 新しい要素の entries 領域を palloc で確保します。 3. lexeme の有無に応じて、語彙文字列のコピーまたは NULL 設定を行い、tnvariant を格納します。 4. src から idsubst および posinsubst をコピーします。 5. nextentry を NULL に設定します。 6. 要素数 *nnw をインクリメントし、配列ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/dict_thesaurus.c:404
  • src/backend/tsearch/dict_thesaurus.c:446
  • **呼び出し先**:
  • repalloc (PostgreSQLメモリ管理)
  • palloc (PostgreSQLメモリ管理)
  • pstrdup (PostgreSQL文字列複製)

関連データ構造

  • TheLexeme: シソーラスの語彙を保持する構造体。
  • LexemeInfo: 語彙の置換情報やバリアントを保持する構造体。
  • TSLexeme: 全文検索用の語彙構造体。

失敗モードと境界条件

  • メモリ不足時には palloc/repalloc がエラーを発生させます。
  • lexeme 引数が NULL の場合、語彙文字列は NULL として扱われ、tnvariant は 1 に強制されます。

保守担当者向けメモ

  • 本関数は thesaurus_init 内のコンパイル処理の一部として使用されます。
  • 配列の再割り当て（repalloc）が発生するため、呼び出し側で返り値のポインタを必ず更新する必要があります。
