get_useful_group_keys_orderings

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/pathkeys.c:466

目的

クエリの GROUP BY 句に対して、実行計画（Path）のソート順序を最大限に活用するための代替的なグループ化キーの順序リストを生成します。これにより、インクリメンタルソートなどを利用したコスト効率の高いグループ化が可能になります。

シグネチャまたは形状

List *get_useful_group_keys_orderings(PlannerInfo *root, Path *path)

入力と出力

  • **入力**:
  • root: 現在のクエリのプランナ情報。
  • path: 評価対象の実行計画パス。
  • **出力**:
  • GroupByOrdering 構造体のリスト。各要素は pathkeys と clauses のペアを保持します。

前提条件と不変条件

  • root->group_pathkeys および root->processed_groupClause が初期化されていること。
  • 生成される各 GroupByOrdering は、元のクエリで指定されたグループ化句と整合性（SortGroupClause の参照整合性など）を保つこと。

アルゴリズムまたは振る舞い

1. **初期化**: クエリで指定されたデフォルトの pathkeys と clauses を含むリストを生成します。 2. **早期リターン**: enable_group_by_reordering が無効な場合、または parse->groupingSets が存在する場合は、デフォルトの順序のみを返します。 3. **再順序付けの試行**: path がソート順序を持ち、かつその順序が現在の root->group_pathkeys に含まれていない場合、group_keys_reorder_by_pathkeys を呼び出して、パスの順序に適合するようにグループ化キーの順序を調整します。 4. **検証**: 調整後の順序が有効（n > 0）であり、かつ元の順序と異なる場合に限り、新しい GroupByOrdering をリストに追加します。 5. **整合性チェック**: USE_ASSERT_CHECKING が有効な場合、生成されたすべての順序が元のグループ化句と論理的に等価であることを検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c 内の複数箇所（planner.c:7386, 7467, 7719, 7776 など）。
  • **呼び出し先**:
  • group_keys_reorder_by_pathkeys
  • compare_pathkeys
  • pathkeys_contained_in
  • makeNode (メモリ確保)

関連データ構造

  • GroupByOrdering: グループ化の順序と対応する句を保持する構造体。
  • PathKey: 順序を表現する構造体。
  • SortGroupClause: GROUP BY 句の定義情報。

失敗モードと境界条件

  • enable_group_by_reordering がオフの場合、最適化は行われません。
  • groupingSets が存在する場合、複雑なロジックを避けるため再順序付けは行われません。
  • path->pathkeys が空、または既存のグループ化キー順序と包含関係にある場合は、追加の順序は生成されません。

保守担当者向けメモ

  • 本関数は PlannerInfo 内のグループ化情報を書き換えるのではなく、代替案のリストを生成する役割を担います。
  • USE_ASSERT_CHECKING ブロック内のアサーションは、pk_eclass->ec_sortref と tleSortGroupRef の一致を検証しており、正規化されたパスキーの整合性を維持する重要な箇所です。
