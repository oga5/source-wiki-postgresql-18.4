InitPgFdwOptions

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/option.c:238

目的

postgres_fdw で使用可能な全オプション（libpq 由来の接続オプションおよび FDW 固有オプション）を統合した静的リストを構築し、バックエンドプロセス内で共有可能な状態にする。

シグネチャまたは形状

static void InitPgFdwOptions(void)

入力と出力

  • **入力**: なし（内部で PQconndefaults() を呼び出し、静的配列 non_libpq_options を参照する）。
  • **出力**: なし（副作用として静的変数 postgres_fdw_options および libpq_options を初期化する）。

前提条件と不変条件

  • バックエンドプロセスごとに一度だけ実行される。
  • postgres_fdw_options が既に初期化されている場合、関数は即座にリターンする。
  • libpq_options は malloc で確保され、プロセス終了まで生存する。

アルゴリズムまたは振る舞い

1. **重複防止**: postgres_fdw_options が非 NULL であれば終了する。 2. **libpq オプション取得**: PQconndefaults() を呼び出し、利用可能な libpq オプションリストを取得する。 3. **フィルタリング**: libpq オプションのうち、デバッグ用（dispchar に 'D' を含む）、内部オーバーライド対象（fallback_application_name, client_encoding）、および OAuth 関連を除外する。 4. **コンテキスト割り当て**: user オプションや秘密情報（dispchar に '*' を含む）は UserMappingRelationId に、それ以外は ForeignServerRelationId に割り当てる。 5. **統合**: libpq オプションと non_libpq_options（FDW 固有オプション）を連結し、postgres_fdw_options に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/option.c 内のオプション検証関数（is_valid_option 等）から呼び出される。
  • **呼び出し先**: PQconndefaults()（libpq API）、malloc、memcpy、strcmp、strchr 等。

関連データ構造

  • PgFdwOption: オプション名、適用対象カタログOID、libpq 依存フラグを保持する構造体。
  • PQconninfoOption: libpq が提供する接続オプション構造体。
  • non_libpq_options: FDW 固有オプションを定義した静的配列。

失敗モードと境界条件

  • PQconndefaults() が NULL を返した場合（メモリ不足）、ERRCODE_FDW_OUT_OF_MEMORY でエラー終了する。
  • malloc が NULL を返した場合、ERRCODE_FDW_OUT_OF_MEMORY でエラー終了する。

保守担当者向けメモ

  • 新しい FDW 固有オプションを追加する場合は、non_libpq_options 配列に定義を追加すること。
  • libpq オプションのフィルタリング条件を変更する場合は、ループ内の if 文を更新すること。
  • 本関数は malloc を使用しているため、メモリリークを考慮する必要はない（プロセス寿命と同期するため）。
