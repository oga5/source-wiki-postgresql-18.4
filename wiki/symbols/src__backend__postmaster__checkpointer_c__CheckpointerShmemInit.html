CheckpointerShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/checkpointer.c:959

目的

チェックポインタープロセスが使用する共有メモリ領域（CheckpointerShmem）の割り当ておよび初期化を行う。

シグネチャまたは形状

void CheckpointerShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の CheckpointerShmemStruct を初期化）

前提条件と不変条件

  • ShmemInitStruct を介して共有メモリを確保する。
  • 初回実行時のみ、メモリ領域のゼロクリアおよびスピンロック、条件変数の初期化が行われる。
  • CompactCheckpointerRequestQueue が要求構造体のパディングバイトがゼロであることを前提としているため、MemSet によるゼロクリアが必須である。

アルゴリズムまたは振る舞い

1. CheckpointerShmemSize() を呼び出し、必要な共有メモリサイズを取得する。 2. ShmemInitStruct を使用して "Checkpointer Data" という名前で共有メモリを確保またはアタッチする。 3. 既に初期化済みでない場合（found が false の場合）:

  • 確保したメモリ領域全体を MemSet で 0 に初期化する。
  • ckpt_lck スピンロックを初期化する。
  • max_requests を NBuffers と MAX_CHECKPOINT_REQUESTS の小さい方に設定する。
  • start_cv および done_cv 条件変数を初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:327（システム起動時の共有メモリ初期化シーケンス）
  • 呼び出し先: CheckpointerShmemSize, ShmemInitStruct, MemSet, SpinLockInit, ConditionVariableInit

関連データ構造

  • CheckpointerShmemStruct: 共有メモリ上のメイン構造体。
  • NBuffers: 最大リクエスト数を決定するために参照されるバッファ数。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct の内部実装によりプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は PostgreSQL の起動初期段階で一度だけ実行されることを想定している。
  • max_requests の設定において NBuffers を参照しているため、バッファ管理設定との整合性に注意すること。
