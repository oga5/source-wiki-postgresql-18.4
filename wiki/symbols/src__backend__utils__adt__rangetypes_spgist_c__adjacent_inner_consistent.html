adjacent_inner_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_spgist.c:886

目的

SP-GiSTインデックスにおける隣接関係（Adjacent）の検索時、現在のノードから探索すべき子ノードを決定するために、前段のノードでの探索方向を考慮して不要な枝を刈り取る。

シグネチャまたは形状

static int adjacent_inner_consistent(TypeCacheEntry *typcache, const RangeBound *arg,
                                     const RangeBound *centroid, const RangeBound *prev)

入力と出力

  • **入力**:
  • typcache: 型キャッシュ情報。
  • arg: 検索対象の範囲境界。
  • centroid: 現在のノードの重心（分割基準）。
  • prev: 前段のノードの重心（存在しない場合は NULL）。
  • **出力**:
  • int: 探索すべき方向や状態を示す値。0 は探索不要（枝刈り）を意味する。

前提条件と不変条件

  • typcache は有効な範囲型キャッシュである必要がある。
  • centroid は現在の空間分割の基準点として機能する。

アルゴリズムまたは振る舞い

1. **前段の検証**: prev が存在する場合、前段での探索方向（prevcmp）と、現在の重心に対する実際の方向（cmp）を比較する。 2. **枝刈り**: 前段での期待される方向と実際の方向が一致しない場合（例: 前段で「左」へ行くべきだったが、現在は「右」に位置する場合）、探索を打ち切り 0 を返す。 3. **方向決定**: 枝刈りが発生しない場合、adjacent_cmp_bounds を呼び出し、現在の arg と centroid の比較結果を返す。

関連データ構造

  • RangeBound: 範囲の境界値を保持する構造体。
  • TypeCacheEntry: データ型のキャッシュ情報。

失敗モードと境界条件

  • prev が NULL の場合、前段の制約チェックをスキップし、単純に arg と centroid の比較結果を返す。
  • 枝刈り条件 (prevcmp < 0 && cmp >= 0) || (prevcmp > 0 && cmp < 0) に該当する場合、探索対象外として 0 を返す。

保守担当者向けメモ

  • 本関数による枝刈りは、現在の重心と前段の重心の比較に基づいているが、完全な最適化ではない。不要な探索が一部残る可能性があることがコード内のコメントで指摘されている。
