RecoveryPauseState

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlogrecovery.h:44

目的

WALリカバリプロセスにおける一時停止状態を管理・追跡するために使用されます。

シグネチャまたは形状

typedef enum RecoveryPauseState
{
    RECOVERY_NOT_PAUSED,        /* pause not requested */
    RECOVERY_PAUSE_REQUESTED,   /* pause requested, but not yet paused */
    RECOVERY_PAUSED,            /* recovery is paused */
} RecoveryPauseState;

入力と出力

  • 入力: 状態遷移のトリガー（外部からの停止要求など）。
  • 出力: 現在のリカバリ一時停止状態を示す列挙値。

前提条件と不変条件

  • リカバリプロセスが実行中であること。
  • 状態は RECOVERY_NOT_PAUSED から開始され、要求に応じて RECOVERY_PAUSE_REQUESTED を経由して RECOVERY_PAUSED へ遷移します。

アルゴリズムまたは振る舞い

  • RECOVERY_NOT_PAUSED: 通常のリカバリ実行状態。
  • RECOVERY_PAUSE_REQUESTED: 一時停止が指示されたが、まだ停止ポイントに到達していない状態。
  • RECOVERY_PAUSED: リカバリ処理が停止ポイントで待機している状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogrecovery.c 内のリカバリ制御ロジック。
  • 呼び出し先: GetRecoveryPauseState() 関数がこの型を返します。
  • 補足: 具体的な関数呼び出しの連鎖については、提供されたソース情報からは詳細なフローを特定できません。

関連データ構造

  • src/backend/access/transam/xlogrecovery.c 内の recoveryPauseState 変数。

失敗モードと境界条件

  • 状態遷移の整合性: リカバリが終了または異常終了した際に、状態が適切にリセットされる必要があります。

保守担当者向けメモ

  • 本シンボルはリカバリの制御フローに直結しています。状態遷移を追加・変更する場合は、xlogrecovery.c 内の関連する条件判定ロジックに影響がないか確認してください。
