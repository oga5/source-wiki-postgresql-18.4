scram_init

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-scram.c:239

目的

SASL SCRAM-SHA-256認証セッションを初期化し、認証状態を管理するための構造体 scram_state を生成・設定します。

シグネチャまたは形状

static void *scram_init(Port *port, const char *selected_mech, const char *shadow_pass)

入力と出力

  • **入力**:
  • port: 現在の接続ポート情報。
  • selected_mech: クライアントが選択したSASLメカニズム名。
  • shadow_pass: データベースに保存されているユーザーのSCRAMシークレット（またはNULL）。
  • **出力**:
  • 初期化された scram_state 構造体へのポインタ。

前提条件と不変条件

  • selected_mech が SCRAM-SHA-256 または SCRAM-SHA-256-PLUS（SSL使用時のみ）であること。それ以外はプロトコル違反としてエラーを発生させます。
  • ユーザーが存在しない、またはシークレットが無効な場合、認証は「モック」モード（doomed フラグが真）へ移行し、攻撃者に対する情報漏洩を防ぐため計算コストをかけて失敗をシミュレートします。

アルゴリズムまたは振る舞い

1. scram_state 構造体をメモリ確保し、初期状態 SCRAM_AUTH_INIT を設定します。 2. selected_mech を検証し、チャネルバインディングの有無を決定します。 3. shadow_pass が提供されている場合、parse_scram_secret を用いてシークレットを解析します。 4. 解析に失敗した場合、または shadow_pass が存在しない場合は、mock_scram_secret を呼び出してモック用のキーを生成し、doomed フラグを立てます。 5. 最終的に state ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は提供されていませんが、認証サブシステム（auth.c 等）からSASLメカニズム定義の一部として参照されることが示唆されています。
  • **呼び出し先**: palloc0, strcmp, ereport, get_password_type, parse_scram_secret, psprintf, mock_scram_secret。

関連データ構造

  • scram_state: 認証セッションの状態を保持する構造体。
  • Port: 接続情報を保持する構造体。

失敗モードと境界条件

  • **プロトコル違反**: サポートされていないメカニズムが選択された場合、ERRCODE_PROTOCOL_VIOLATION でエラー終了します。
  • **解析失敗**: シークレットの形式が不正な場合、ログに記録し、モック認証へ移行します。
  • **ユーザー不在**: shadow_pass がNULLの場合、モック認証へ移行します。

保守担当者向けメモ

  • 本関数はユーザーの存在を隠蔽するために、認証失敗時も成功時と同等の計算負荷をかける設計になっています。
  • チャネルバインディングのサポートは USE_SSL マクロおよび port->ssl_in_use に依存しています。
