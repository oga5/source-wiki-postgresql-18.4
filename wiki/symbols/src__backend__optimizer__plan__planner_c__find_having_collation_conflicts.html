find_having_collation_conflicts

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/planner.c:1458

目的

クエリの HAVING 句に含まれる各条件式を走査し、照合順序（collation）の競合が発生している箇所を特定する。

シグネチャまたは形状

Bitmapset *find_having_collation_conflicts(Query *parse, Index group_rtindex)

入力と出力

  • **入力**:
  • parse: 処理対象の Query 構造体。
  • group_rtindex: グループ化対象の範囲テーブルインデックス。
  • **出力**:
  • 競合が検出された HAVING 句のインデックスを保持する Bitmapset。競合がない場合は NULL を返す。

前提条件と不変条件

  • parse->havingQual が NULL の場合、直ちに NULL を返す。
  • 走査中、having_collation_ctx の ancestor_collids は各句の処理終了時に空（NIL）であることが保証される。

アルゴリズムまたは振る舞い

1. having_collation_ctx を初期化し、group_rtindex を設定する。 2. parse->havingQual をリストとして反復処理する。 3. 各句に対して having_collation_conflict_walker を呼び出し、競合を判定する。 4. 競合が検出された場合、その句のインデックスを Bitmapset に追加する。 5. 最終的な Bitmapset を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c 内の grouping_planner から呼び出される（行1116）。
  • **呼び出し先**: having_collation_conflict_walker を再帰的に呼び出す。

関連データ構造

  • Query: クエリツリー構造体。
  • having_collation_ctx: 競合判定のコンテキストを保持する構造体。
  • Bitmapset: 競合インデックスの集合を保持するビットマップ。

失敗モードと境界条件

  • HAVING 句が存在しない場合、安全に NULL を返す。
  • 競合が一つも存在しない場合、NULL を返す。

保守担当者向けメモ

  • 本関数は having_collation_conflict_walker と密接に連携している。照合順序の判定ロジックを変更する場合は、両方の関数を確認すること。
