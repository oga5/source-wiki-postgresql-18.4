r_consonant_pair

定義

  • **種別**: function
  • **定義箇所**: src/backend/snowball/libstemmer/stem_ISO_8859_1_norwegian.c:200

目的

単語の末尾において、特定の二重子音パターンを検出し、その一部を削除することで語幹を正規化します。

シグネチャまたは形状

static int r_consonant_pair(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境の状態を保持する構造体）
  • **出力**: 処理が成功した場合は 1、条件を満たさない場合やエラー時は 0 または負の値を返します。

前提条件と不変条件

  • z->I[1] で定義された領域（Region）境界より後ろの文字を対象とします。
  • 単語の末尾から前方に向かって処理を行います。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置から z->I[1] までの範囲を保護しつつ、末尾から前方へ検索を行います。 2. find_among_b を使用して、a_1 に定義された特定の二重子音パターンを末尾から検索します。 3. 条件に合致した場合、その直前の文字を削除対象（bra）としてマークします。 4. slice_del を呼び出し、マークされた文字を単語バッファから削除します。

呼び出し元と呼び出し先

  • **呼び出し元**: stem_ISO_8859_1_norwegian.c および stem_UTF_8_norwegian.c 内のステミング処理フローから呼び出されます。また、デンマーク語やスウェーデン語のステマーからも参照されています。
  • **呼び出し先**: find_among_b（パターンマッチング用）、slice_del（文字列操作用）。

関連データ構造

  • SN_env: ステミング環境の状態（バッファ、カーソル位置、領域境界など）を保持します。
  • a_1: 検索対象となる二重子音パターンを格納した among 構造体。

失敗モードと境界条件

  • カーソル位置が z->I[1] より前にある場合、処理を中断し 0 を返します。
  • 単語の末尾に該当するパターンが見つからない場合、0 を返します。
  • slice_del が失敗した場合、その戻り値（負の値）をそのまま伝播させます。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによって自動生成されています。ロジックの変更が必要な場合は、直接Cファイルを編集せず、元のSnowballソース（.sbl）を修正してください。
  • z->lb（境界）の操作は、mlimit2 を用いて一時的に変更し、処理後に復元する安全な設計になっています。
