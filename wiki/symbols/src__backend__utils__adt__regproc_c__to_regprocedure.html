to_regprocedure

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regproc.c:277

目的

関数名（引数リストを含む文字列）を対応するシステムカタログ pg_proc の OID に変換します。エラー発生時に例外をスローせず、NULL を返す安全な変換インターフェースを提供します。

シグネチャまたは形状

Datum to_regprocedure(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介して、関数名および引数型を含む文字列（例: my_schema.my_func(int4, text)）を受け取ります。
  • **出力**: 成功した場合は対応する regprocedure 型の OID を Datum として返します。解決できない場合は NULL を返します。

前提条件と不変条件

  • 入力文字列は regprocedurein 関数が解釈可能な形式である必要があります。
  • ErrorSaveContext を使用してエラーを捕捉するため、呼び出し元は NULL 戻り値を適切に処理する必要があります。

アルゴリズムまたは振る舞い

1. 入力された text 型の引数を C 文字列に変換します。 2. ErrorSaveContext を初期化し、エラー発生を抑制する準備を行います。 3. DirectInputFunctionCallSafe を呼び出し、regprocedurein を実行して文字列から OID への解決を試みます。 4. 解決に成功した場合はその OID を返し、失敗した場合は PG_RETURN_NULL() を実行して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（直接的な C 言語レベルの呼び出し元情報は提供されていません）。
  • **呼び出し先**:
  • text_to_cstring
  • regprocedurein (内部的な名前解決処理)
  • DirectInputFunctionCallSafe

関連データ構造

  • ErrorSaveContext: エラーをスローせずに捕捉するためのコンテキスト構造体。
  • regprocedure: 関数を識別するための OID 型。

失敗モードと境界条件

  • 入力文字列が不正な形式である場合、または該当する関数がシステムカタログに見つからない場合、DirectInputFunctionCallSafe が失敗し、本関数は NULL を返します。

保守担当者向けメモ

  • 本関数は regprocedurein のラッパーであり、エラーハンドリングの挙動を標準的な SQL 関数から安全な形式へ変換する役割を担っています。
  • 複雑な名前解決ロジックの詳細は regprocedurein およびその内部で呼び出される parseNameAndArgTypes に依存しています。
