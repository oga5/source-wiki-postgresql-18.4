heap_copytuple_with_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/heaptuple.c:803

目的

既存の HeapTuple（ソース）の内容を、指定された別の HeapTuple 構造体（デスティネーション）へコピーする。

シグネチャまたは形状

void heap_copytuple_with_tuple(HeapTuple src, HeapTuple dest)

入力と出力

  • **入力**:
  • src: コピー元の HeapTuple 構造体。
  • dest: コピー先の HeapTuple 構造体。
  • **出力**: なし（dest が指す構造体のフィールドおよび dest->t_data が指すメモリ領域が更新される）。

前提条件と不変条件

  • dest は呼び出し元によって事前にメモリ確保されている必要がある。
  • src が無効（HeapTupleIsValid が偽）または src->t_data が NULL の場合、dest->t_data は NULL に設定される。

アルゴリズムまたは振る舞い

1. src が無効またはデータを持たない場合、dest->t_data を NULL に設定して終了する。 2. src が有効な場合、以下のメタデータを src から dest へコピーする。

  • t_len（タプル長）
  • t_self（タプルの物理位置）
  • t_tableOid（テーブルのOID）

    • 3. src->t_len 分のメモリを palloc で確保し、dest->t_data に割り当てる。 4. src->t_data が指す実データを、確保した dest->t_data 領域へ memcpy で複製する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し関係は、提供された情報からは特定できません。
  • **呼び出し先**: palloc（メモリ確保）、memcpy（メモリコピー）、HeapTupleIsValid（マクロによる検証）。

関連データ構造

  • HeapTuple: タプルのメタデータと実データ（HeapTupleHeader）を保持する構造体。
  • HeapTupleHeader: タプルの物理的なヘッダーおよびデータ本体。

失敗モードと境界条件

  • palloc がメモリ不足に陥った場合、PostgreSQLの標準的なエラー処理によりアボートする可能性がある。
  • src が指すメモリ領域が不正な場合、memcpy 実行時にセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は dest 構造体自体のメモリは確保しないが、dest->t_data が指す実データ領域は動的に確保する。呼び出し元は、不要になった時点で pfree(dest->t_data) を呼び出してメモリを解放する責任がある。
