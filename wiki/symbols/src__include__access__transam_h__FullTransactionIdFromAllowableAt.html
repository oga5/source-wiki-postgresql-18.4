FullTransactionIdFromAllowableAt

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/access/transam.h:380

目的

32ビットの TransactionId (XID) と、現在のシステム状態を示す FullTransactionId (64ビット) を受け取り、そのXIDに対応する完全な64ビットの FullTransactionId を復元する。

シグネチャまたは形状

static inline FullTransactionId FullTransactionIdFromAllowableAt(FullTransactionId nextFullXid, TransactionId xid)

入力と出力

  • **入力**:
  • nextFullXid: 現在のシステムにおける最新の FullTransactionId。
  • xid: 復元対象の32ビット TransactionId。
  • **出力**:
  • 復元された64ビットの FullTransactionId。

前提条件と不変条件

  • xid が正規のトランザクションIDである場合、xid は nextFullXid が指すXIDよりも過去（または等しい）である必要がある（Assert による検証）。
  • 復元される64ビット値は、nextFullXid 以下である必要がある。

アルゴリズムまたは振る舞い

1. **特殊IDの判定**: TransactionIdIsNormal(xid) が偽（InvalidTransactionId 等）の場合、エポック0として即座に返す。 2. **エポックの特定**: nextFullXid からエポック（上位32ビット）を抽出する。 3. **エポックの調整**: xid が nextFullXid の下位32ビットよりも大きい場合、その xid は前のエポックに属すると判断し、エポックを1減算する。 4. **構築**: 決定したエポックと xid を組み合わせて FullTransactionId を生成する。

関連データ構造

  • FullTransactionId: 64ビットのトランザクション識別子。
  • TransactionId: 32ビットのトランザクション識別子。

失敗モードと境界条件

  • xid が nextFullXid よりも未来のIDである場合、Assert によりプログラムが停止する可能性がある。
  • エポックが0の状態で xid が nextFullXid を超えるような不正な入力が与えられた場合、エポックの減算がアンダーフローする可能性がある（Assert で保護されている）。

保守担当者向けメモ

  • unlikely() 分岐ヒントは、クラスタの寿命初期や特定のXID範囲で予測精度が変動する。平均的なレイテンシを優先した設計となっている。
  • 32ビットXIDのラップアラウンドを考慮し、エポックの境界を正しく判定するロジックであるため、変更時はトランザクションIDの循環特性に注意すること。
