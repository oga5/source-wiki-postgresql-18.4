ScalarMCVItem

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/analyze.c:1852

目的

統計情報生成プロセスにおいて、列内の値の出現頻度（Most Common Values: MCV）を追跡・集計するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	int			count;			/* # of duplicates */
	int			first;			/* values[] index of first occurrence */
} ScalarMCVItem;

入力と出力

  • 入力: analyze 処理中にサンプリングされたデータセットから抽出された値の出現回数およびインデックス。
  • 出力: pg_statistic カタログに格納される MCV 統計情報の計算元となる中間データ。

前提条件と不変条件

  • first は、対象となる値の配列内における最初の出現位置を指す必要があります。
  • count は、その値がサンプル内で何回出現したかを示す非負の整数です。

アルゴリズムまたは振る舞い

1. analyze 処理において、サンプリングされた行から MCV を特定するために ScalarMCVItem の配列が動的に割り当てられます。 2. qsort_interruptible を使用して、この構造体の配列をソートし、頻出値の順序を決定します。 3. ソート時、first メンバの値に基づいて比較が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/analyze.c 内の統計計算ロジック。
  • 呼び出し先: palloc（メモリ確保）、qsort_interruptible（ソート処理）。
  • 補足: 具体的な関数名はソース抜粋からは特定できません。

関連データ構造

  • TrackItem: 統計計算に関連する別の追跡用構造体（analyze.c 内で定義）。

失敗モードと境界条件

  • メモリ不足: palloc によるメモリ確保失敗時にエラーが発生する可能性があります。
  • 境界条件: num_mcv が 0 の場合、配列の割り当てやソート処理はスキップされます。

保守担当者向けメモ

  • 本構造体は analyze.c 内の統計計算専用です。他のモジュールから参照されることは想定されていません。
  • ソートアルゴリズムを変更する場合は、qsort_interruptible に渡される比較関数のロジック（first メンバの参照）に注意してください。
