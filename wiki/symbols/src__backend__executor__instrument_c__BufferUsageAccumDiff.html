BufferUsageAccumDiff

定義

  • 種別: function
  • 定義箇所: src/backend/executor/instrument.c:247

目的

2つの BufferUsage 構造体の差分（add - sub）を計算し、その結果を dst に加算します。主に、特定の処理区間におけるバッファアクセス統計や経過時間を抽出するために使用されます。

シグネチャまたは形状

void BufferUsageAccumDiff(BufferUsage *dst, const BufferUsage *add, const BufferUsage *sub);

入力と出力

  • **入力**:
  • dst: 差分結果を累積する先の BufferUsage 構造体へのポインタ。
  • add: 終了時点の統計値を持つ BufferUsage 構造体へのポインタ。
  • sub: 開始時点の統計値を持つ BufferUsage 構造体へのポインタ。
  • **出力**: なし（dst が指す構造体の各フィールドが更新されます）。

前提条件と不変条件

  • dst, add, sub は有効な BufferUsage 構造体を指している必要があります。
  • add の各統計値は、sub の各統計値以上である（または経過時間として妥当な差分である）ことが期待されます。

アルゴリズムまたは振る舞い

1. BufferUsage 構造体内の各カウンタ（shared_blks_hit, local_blks_read 等）について、add から sub を減算した値を dst の対応するフィールドに加算します。 2. 経過時間フィールド（shared_blk_read_time 等）については、INSTR_TIME_ACCUM_DIFF マクロを使用して、プラットフォーム固有のタイマー精度に基づいた差分計算と累積を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/commands/analyze.c
  • src/backend/commands/explain.c
  • src/backend/commands/explain_dr.c
  • src/backend/commands/prepare.c
  • src/backend/executor/instrument.c
  • **呼び出し先**:
  • INSTR_TIME_ACCUM_DIFF マクロ（経過時間の差分計算に使用）。

関連データ構造

  • BufferUsage: バッファアクセス統計（ヒット、読み込み、書き込み、ダーティ数）および関連する時間を保持する構造体。src/include/executor/instrument.h で定義されています。

失敗モードと境界条件

  • ポインタが NULL の場合、セグメンテーション違反が発生します。
  • カウンタがオーバーフローするような極端な値の差分については考慮されていません。

保守担当者向けメモ

  • 本関数は、グローバルな統計カウンタ（pgBufferUsage 等）から特定の処理区間の統計を分離するために不可欠です。
  • BufferUsage 構造体に新しいフィールドが追加された場合は、本関数にも必ず加算・減算ロジックを追加してください。
