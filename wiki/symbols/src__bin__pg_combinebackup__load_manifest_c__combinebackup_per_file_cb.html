combinebackup_per_file_cb

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/load_manifest.c:267

目的

バックアップマニフェストのパース処理中、各ファイルのエントリをメモリ上のハッシュテーブル（manifest_files_hash）に登録するために使用されるコールバック関数です。

シグネチャまたは形状

static void combinebackup_per_file_cb(
    JsonManifestParseContext *context,
    const char *pathname,
    uint64 size,
    pg_checksum_type checksum_type,
    int checksum_length,
    uint8 *checksum_payload
)

入力と出力

  • **入力**:
  • context: パース状態を保持するコンテキスト。private_data 経由で manifest_data 構造体にアクセスします。
  • pathname: ファイルパス。
  • size: ファイルサイズ。
  • checksum_type, checksum_length, checksum_payload: ファイルのチェックサム情報。
  • **出力**: なし（副作用としてハッシュテーブルを更新）。

前提条件と不変条件

  • context->private_data が有効な manifest_data 構造体を指していること。
  • ハッシュテーブル manifest->files が初期化済みであること。

アルゴリズムまたは振る舞い

1. manifest_files_insert を呼び出し、pathname をキーとしてハッシュテーブルにエントリを作成します。 2. 同一パスが既に存在する場合は pg_fatal を呼び出し、エラーとして処理を中断します。 3. 新規エントリに対して、引数で受け取ったファイルサイズおよびチェックサム情報を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/load_manifest.c 内のパース処理（JsonManifestParseContext の per_file_cb フィールドに設定され、パースエンジンから間接的に呼び出されます）。
  • **呼び出し先**:
  • manifest_files_insert（simplehash.h により生成された関数）
  • pg_fatal

関連データ構造

  • JsonManifestParseContext: パースの実行状態を管理。
  • manifest_data: マニフェスト全体のデータを保持する構造体。
  • manifest_file: ハッシュテーブルに格納される個々のファイル情報。

失敗モードと境界条件

  • **重複パス**: マニフェスト内に同一のファイルパスが複数存在する場合、pg_fatal によりプログラムが異常終了します。

保守担当者向けメモ

  • 本関数は simplehash.h を利用したハッシュテーブル操作に依存しています。ハッシュテーブルの型定義や関数名は load_manifest.c 内の SH_PREFIX マクロ等によって生成されているため、構造変更時は注意してください。
