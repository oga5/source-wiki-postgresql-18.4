plpgsql_peek2

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_scanner.c:470

目的

字句解析器の状態を変化させずに、後続の2つのトークンとその位置情報を取得する。

シグネチャまたは形状

void plpgsql_peek2(int *tok1_p, int *tok2_p, int *tok1_loc, int *tok2_loc, yyscan_t yyscanner)

入力と出力

  • **入力**:
  • yyscanner: 字句解析器の状態オブジェクト。
  • **出力**:
  • tok1_p, tok2_p: 取得したトークン種別を格納するポインタ。
  • tok1_loc, tok2_loc: トークンのソース位置（オフセット）を格納するポインタ（NULL指定可能）。

前提条件と不変条件

  • 呼び出し後、スキャナの内部状態（トークンスタック）は呼び出し前と同一の状態に復元される。
  • internal_yylex を使用してトークンを取得するため、スキャナが保持するトークンストリームの先頭2つが読み取られる。

アルゴリズムまたは振る舞い

1. internal_yylex を2回呼び出し、トークンと補助データ（TokenAuxData）を順次取得する。 2. 取得したトークン種別と位置情報を、引数で指定されたポインタへ書き出す。 3. push_back_token を使用し、取得した2番目のトークン、続いて1番目のトークンの順でスキャナのスタックへ戻す。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • internal_yylex: トークンの取得に使用。
  • push_back_token: 取得したトークンの復元に使用。
  • **呼び出し元**:
  • 提供されたソース情報には、本関数を直接呼び出している具体的な関数名は含まれていない。

関連データ構造

  • TokenAuxData: トークンの位置情報（lloc）を保持する構造体。
  • yyscan_t: スキャナの状態を管理する不透明なポインタ型。

失敗モードと境界条件

  • トークンストリームが2つ未満の場合、internal_yylex の挙動に依存する。
  • tok1_loc または tok2_loc が NULL の場合、位置情報の書き込みはスキップされる。

保守担当者向けメモ

  • 本関数は plpgsql_peek と同様の先読み機能を提供するが、一度に2トークン分を処理する。
  • push_back_token のスタック容量（MAX_PUSHBACKS）を超えるトークンを扱うことはできないため、本関数は最大2トークンまでの先読みに限定されている。
