mixedStruct

定義

  • 種別: union
  • 定義箇所: src/backend/utils/misc/help_config.c:30

目的

PostgreSQLのGUC（Grand Unified Configuration）変数を統一的に扱うための共用体です。異なる型のGUC構造体を同一のポインタ型でキャストして処理することを可能にします。

シグネチャまたは形状

typedef union {
    struct config_generic generic;
    struct config_bool _bool;
    struct config_real real;
    struct config_int integer;
    struct config_string string;
    struct config_enum _enum;
} mixedStruct;

入力と出力

  • 入力: GUC変数の配列（guc_vars）から取得された各要素を本構造体にキャストして使用します。
  • 出力: 該当しません（データ構造定義のため）。

前提条件と不変条件

  • 各GUC構造体（config_bool, config_int 等）の先頭メンバが config_generic である必要があります。このメモリレイアウトの共通性を利用して、型をまたいだキャストが行われます。

アルゴリズムまたは振る舞い

  • help_config.c 内で、GUC変数のリストを走査する際に、各変数を mixedStruct にキャストしてアクセスします。
  • displayStruct 関数や printMixedStruct 関数において、この共用体を通じて各変数のメンバにアクセスし、型に応じた表示制御や出力を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/help_config.c 内の GucInfoMain（GUC変数リストの走査時）。
  • 呼び出し先: 本構造体を引数として受け取る関数として displayStruct(mixedStruct *) および printMixedStruct(mixedStruct *) が定義されています。

関連データ構造

  • config_generic: 全GUC構造体の基底となる構造体。
  • config_bool, config_real, config_int, config_string, config_enum: 各種GUC変数の定義構造体。

失敗モードと境界条件

  • 不適切な型へのキャスト: generic メンバ以外の各型メンバにアクセスする際、実際の変数の型と一致していない場合に未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はハック的な手法であり、GUCシステムが新しい型を追加した場合は、この共用体にもメンバを追加する必要があります。
  • guc_tables.c で定義されたフラグと組み合わせて使用されるため、表示ロジックを変更する際は displayStruct の実装と併せて確認してください。
