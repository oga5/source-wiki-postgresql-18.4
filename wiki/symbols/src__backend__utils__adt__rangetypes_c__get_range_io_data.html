get_range_io_data

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes.c:318

目的

範囲型の入出力（I/O）処理に必要なメタデータおよび関数情報を取得し、キャッシュする。

シグネチャまたは形状

RangeIOData *get_range_io_data(FunctionCallInfo fcinfo, Oid rngtypid, IOFuncSelector func)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報（fn_extra をキャッシュとして利用）。
  • rngtypid: 対象となる範囲型の OID。
  • func: 実行する I/O 操作の種類（IOFunc_input, IOFunc_output, IOFunc_receive, IOFunc_send）。
  • **出力**:
  • RangeIOData*: 範囲型の要素型情報および I/O 関数情報を含む構造体。

前提条件と不変条件

  • fcinfo->flinfo->fn_extra をキャッシュ領域として使用する。
  • 呼び出し元は、rngtypid が有効な範囲型であることを保証する必要がある。
  • 要素型が特定できない場合、エラーが発生する。

アルゴリズムまたは振る舞い

1. fcinfo->flinfo->fn_extra にキャッシュが存在し、かつ rngtypid が一致する場合はそのキャッシュを返す。 2. キャッシュがない、または不一致の場合は以下を実行する：

  • MemoryContextAlloc を使用して RangeIOData 構造体を確保する。
  • lookup_type_cache を呼び出し、範囲型のメタデータを取得する。
  • get_type_io_data を使用して、要素型の I/O 関数情報を取得する。
  • fmgr_info_cxt を使用して、取得した I/O 関数を現在のメモリコンテキストに登録する。
  • 構築したキャッシュを fcinfo->flinfo->fn_extra に保存して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • range_in (src/backend/utils/adt/rangetypes.c:106)
  • range_out (src/backend/utils/adt/rangetypes.c:153)
  • range_recv (src/backend/utils/adt/rangetypes.c:192)
  • range_send (src/backend/utils/adt/rangetypes.c:275)
  • **呼び出し先**:
  • lookup_type_cache
  • get_type_io_data
  • fmgr_info_cxt
  • elog / ereport (エラーハンドリング)

関連データ構造

  • RangeIOData: 範囲型の I/O 処理に必要な情報を保持する構造体。
  • TypeCacheEntry: 型のメタデータを保持するキャッシュ構造体。
  • FunctionCallInfo: PostgreSQL の関数呼び出し規約におけるコンテキスト情報。

失敗モードと境界条件

  • rngtypid が範囲型ではない場合、elog(ERROR, ...) により処理が中断される。
  • 指定された I/O 関数（特にバイナリ送受信）が存在しない場合、ERRCODE_UNDEFINED_FUNCTION を伴うエラーが発生する。

保守担当者向けメモ

  • 本関数は fn_extra を利用してキャッシュを永続化するため、呼び出し元の関数は fcinfo のライフサイクルが適切であることを確認すること。
  • get_type_io_data は必要以上の情報を取得するが、実装の簡便さのために利用されている。
