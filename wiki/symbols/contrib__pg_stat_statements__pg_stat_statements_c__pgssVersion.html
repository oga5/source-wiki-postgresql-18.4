pgssVersion

定義

  • 種別: enum
  • 定義箇所: contrib/pg_stat_statements/pg_stat_statements.c:106

目的

pg_stat_statements 拡張機能の内部データ構造やファイル形式のバージョンを識別するために使用されます。

シグネチャまたは形状

typedef enum pgssVersion
{
	PGSS_V1_0 = 0,
	PGSS_V1_1,
	PGSS_V1_2,
	PGSS_V1_3,
	PGSS_V1_8,
	PGSS_V1_9,
	PGSS_V1_10,
	PGSS_V1_11,
	PGSS_V1_12,
} pgssVersion;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • 各列挙子は、拡張機能の進化に伴うスキーマやデータレイアウトの変更を表現します。
  • 値は 0 から始まり、順次インクリメントされます。

アルゴリズムまたは振る舞い

  • バージョン管理の識別子として機能し、統計データの読み込みや互換性チェックの際に参照されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pg_stat_statements/pg_stat_statements.c:360 (関数引数として使用)
  • contrib/pg_stat_statements/pg_stat_statements.c:1664 (関数引数として使用)
  • 呼び出し先: なし（列挙型定義のため）

関連データ構造

  • pgssSharedState: 共有メモリ上の状態管理において、バージョン情報の保持に使用される可能性があります。

失敗モードと境界条件

  • 定義されていないバージョン番号を指定した場合の挙動は、この列挙型自体ではなく、これを利用する関数側の実装に依存します。

保守担当者向けメモ

  • 新しいバージョンを追加する際は、既存の列挙子との整合性を保ち、末尾に新しい値を追加してください。
  • この列挙型を参照している関数（api_version 引数を持つもの）が、新しいバージョンを正しく処理できるか確認してください。
