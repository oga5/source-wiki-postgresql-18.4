SubscriptRoutines

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/subscripting.h:158

目的

PostgreSQLにおいて、配列以外のデータ型（JSONBやカスタム型など）がSQLの添字操作（expr[subscript]）をサポートするためのインターフェースを定義する。型固有の解析および実行ロジックをコアシステムに提供する役割を持つ。

シグネチャまたは形状

typedef struct SubscriptRoutines
{
    SubscriptTransform transform;    /* 解析関数 */
    SubscriptExecSetup exec_setup;   /* 式コンパイル関数 */
    bool        fetch_strict;        /* 取得操作の厳密性 */
    bool        fetch_leakproof;     /* 取得操作の漏洩防止特性 */
    bool        store_leakproof;     /* 代入操作の漏洩防止特性 */
} SubscriptRoutines;

入力と出力

  • **入力**: 添字操作を実装する各データ型が、本構造体のインスタンスを定義し、関数ポインタとプロパティフラグを設定する。
  • **出力**: システム（パーサー、オプティマイザ、エグゼキュータ）がこの構造体を参照し、型に応じた添字操作の解析や実行を行う。

前提条件と不変条件

  • transform および exec_setup は、対象のデータ型が添字操作をサポートするために必須の関数ポインタである。
  • fetch_strict、fetch_leakproof、store_leakproof の各フラグは、クエリ最適化やセキュリティ上の挙動を決定するため、実装する型に合わせて適切に設定する必要がある。

アルゴリズムまたは振る舞い

1. **解析フェーズ**: パーサーが添字構文を検出すると、対象型の transform 関数を呼び出し、添字式の検証と変換を行う。 2. **実行準備**: エグゼキュータの初期化時に exec_setup が呼び出され、実行に必要なメソッド（sbs_fetch, sbs_assign 等）が登録される。 3. **実行フェーズ**: 登録されたメソッドを通じて、コンテナ値と添字から結果が算出される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_node.c
  • src/backend/optimizer/util/clauses.c
  • src/backend/executor/execExpr.c
  • src/backend/utils/cache/lsyscache.c
  • **呼び出し先**:
  • 各データ型で実装された添字操作関数（contrib/hstore/hstore_subs.c、src/backend/utils/adt/arraysubs.c、src/backend/utils/adt/jsonbsubs.c 等の sbsroutines インスタンス）。

関連データ構造

  • SubscriptingRef: 添字操作を表すノード。
  • SubscriptExecSteps: 実行時に使用されるメソッド群。
  • nodes/primnodes.h: 本構造体が依存する基本ノード定義。

失敗モードと境界条件

  • 添字操作のメソッド実装において、fetch_strict 等のフラグ設定が不適切である場合、NULL伝播やセキュリティ上の情報漏洩が発生する可能性がある。
  • sbs_check_subscripts が呼び出される場合、その検証ロジックが sbs_fetch や sbs_assign の前処理として正しく機能することが求められる。

保守担当者向けメモ

  • 本構造体は新しいデータ型に添字機能を追加する際の「契約」となる。
  • 実装時は、fetch_leakproof や store_leakproof がクエリ最適化に与える影響を考慮すること。
  • 既存の実装例として src/backend/utils/adt/arraysubs.c を参照すること。
