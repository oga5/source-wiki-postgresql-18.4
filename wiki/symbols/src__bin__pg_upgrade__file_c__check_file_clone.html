check_file_clone

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/file.c:359

目的

実行環境がファイルクローン（reflink）によるデータ移行をサポートしているかを検証する診断関数です。

シグネチャまたは形状

void check_file_clone(void)

入力と出力

  • 入力: なし（グローバル変数 old_cluster および new_cluster を参照）
  • 出力: なし（成功時は正常終了、失敗時は pg_fatal によりプロセス終了）

前提条件と不変条件

  • old_cluster.pgdata および new_cluster.pgdata が有効なパスとして設定されていること。
  • 呼び出し元は、この関数が実行されることで一時ファイルが作成・削除されることを許容していること。

アルゴリズムまたは振る舞い

1. old_cluster 内の PG_VERSION ファイルをソースとし、new_cluster 内に PG_VERSION.clonetest という一時ファイルを作成します。 2. プラットフォームに応じて以下のいずれかの方法でクローンを試みます。

  • HAVE_COPYFILE かつ COPYFILE_CLONE_FORCE が定義されている場合: copyfile() を使用。
  • Linux かつ FICLONE が定義されている場合: ioctl(dest_fd, FICLONE, src_fd) を使用。

    • 3. 上記のいずれの条件も満たさない場合、またはクローン操作が失敗した場合は pg_fatal でエラーを報告します。 4. 最後に、作成した一時ファイルを unlink で削除します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/check.c:721
  • 呼び出し先: open, ioctl, close, unlink, pg_fatal（いずれも標準ライブラリまたは pg_upgrade ユーティリティ関数）

関連データ構造

  • old_cluster, new_cluster: pg_upgrade.h で定義されるクラスタ情報を保持する構造体。

失敗モードと境界条件

  • 権限不足やディスク容量不足によりファイル作成やクローンが失敗した場合、pg_fatal により移行処理全体が中断されます。
  • プラットフォームがファイルクローンをサポートしていない場合、即座にエラーとなります。

保守担当者向けメモ

  • 本関数は pg_upgrade の事前チェックフェーズで使用されます。
  • OS 固有の高速化機能（reflink）の可否を判定するため、新しいプラットフォームへの移植時には #ifdef ブロックの拡張が必要になる場合があります。
