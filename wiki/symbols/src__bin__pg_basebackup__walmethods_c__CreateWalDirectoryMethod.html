CreateWalDirectoryMethod

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/walmethods.c:639

目的

WAL（Write-Ahead Log）データをディレクトリ構造として永続化するための WalWriteMethod インスタンスを生成・初期化します。

シグネチャまたは形状

WalWriteMethod *CreateWalDirectoryMethod(const char *basedir,
                                         pg_compress_algorithm compression_algorithm,
                                         int compression_level,
                                         bool sync);

入力と出力

  • **入力**:
  • basedir: WAL ファイルを格納するベースディレクトリのパス。
  • compression_algorithm: 使用する圧縮アルゴリズム。
  • compression_level: 圧縮レベル。
  • sync: 書き込み時に fsync を実行するかどうかのフラグ。
  • **出力**:
  • 初期化された WalWriteMethod 構造体へのポインタを返します。

前提条件と不変条件

  • DirectoryMethodData 構造体分のメモリ確保が成功している必要があります。
  • basedir は有効なパス文字列である必要があります。

アルゴリズムまたは振る舞い

1. DirectoryMethodData 構造体用のメモリをゼロクリアして確保します。 2. WalDirectoryMethodOps をメソッド操作テーブルとして設定します。 3. 引数で受け取った圧縮設定および同期フラグを構造体に格納します。 4. clear_error を呼び出し、エラー状態を初期化します。 5. basedir を複製して保持します。 6. 構築された WalWriteMethod インターフェースを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • **呼び出し先**:
  • pg_malloc0 (メモリ確保)
  • clear_error (エラー状態の初期化)
  • pg_strdup (文字列の複製)

関連データ構造

  • WalWriteMethod: 抽象基底構造体。
  • DirectoryMethodData: ディレクトリ保存用メソッドの具体的なデータ構造。
  • WalDirectoryMethodOps: ディレクトリメソッド固有のコールバック関数テーブル。

失敗モードと境界条件

  • メモリ不足が発生した場合、pg_malloc0 によりプロセスが終了する可能性があります。
  • basedir が無効な場合、後続のファイル操作時にエラーが発生します。

保守担当者向けメモ

  • 本関数は WalWriteMethod のディレクトリ実装版コンストラクタです。tar メソッド版の CreateWalTarMethod と対になる設計となっています。
  • 圧縮アルゴリズムとして LZ4 がサポートされているのはこのディレクトリメソッドのみです。
