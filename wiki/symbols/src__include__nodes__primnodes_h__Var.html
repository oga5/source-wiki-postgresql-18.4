Var

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/primnodes.h:261

目的

クエリ処理の各段階（解析・計画・実行）において、テーブルの列やサブプランの出力を参照するための変数ノードを表現する。

シグネチャまたは形状

typedef struct Var {
    Expr        xpr;
    int         varno;
    AttrNumber  varattno;
    Oid         vartype;
    int32       vartypmod;
    Oid         varcollid;
    Bitmapset  *varnullingrels;
    Index       varlevelsup;
    VarReturningType varreturningtype;
    Index       varnosyn;
    AttrNumber  varattnosyn;
    ParseLoc    location;
} Var;

入力と出力

  • **入力**: クエリ内の列参照やサブクエリの出力。
  • **出力**: 実行時に評価される式の一部として、対応する属性値を提供。

前提条件と不変条件

  • varno が負の値（INNER_VAR, OUTER_VAR 等）の場合、物理テーブルではなく実行時のサブプラン出力を指す。
  • varattno が 0 の場合、そのテーブルの全列（whole-row Var）を指す。
  • vartype, vartypmod, varcollid はクエリのハッシュ化（Jumble）対象外。
  • varnosyn と varattnosyn は意味論的な等価性判定において無視される。

アルゴリズムまたは振る舞い

  • クエリの実行時、ExecEvalExpr 等の関数によって再帰的に走査され、指定された varno と varattno に基づいて値が抽出される。
  • varnullingrels は、外部結合によって値が NULL に置換される可能性のあるリレーションを追跡する。
  • varlevelsup が 0 より大きい場合、外側のクエリレベルを参照する相関サブクエリ変数として扱われる。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/deparse.c 等の外部データラッパーや、クエリ最適化・実行エンジン全般。
  • **呼び出し先**: 直接的な関数呼び出し関係はソース抜粋からは不明。

関連データ構造

  • Expr: 基底構造体。
  • AttrNumber: 属性番号の定義。
  • Bitmapset: 外部結合のNULL化対象リレーション管理に使用。
  • VarReturningType: 戻り値の型定義。

失敗モードと境界条件

  • location が -1 の場合、トークン位置が不明であることを示す。
  • varcollid が InvalidOid の場合、照合順序（Collation）が定義されていないことを示す。

保守担当者向けメモ

  • pg_node_attr マクロが付与されたフィールドは、クエリのハッシュ化や等価性判定において特別な扱いを受けるため、変更時は注意が必要。
  • 構文上の識別子（varnosyn, varattnosyn）と意味上の識別子（varno, varattno）が分離されている点に留意すること。
