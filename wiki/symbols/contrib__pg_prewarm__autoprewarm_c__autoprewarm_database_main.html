autoprewarm_database_main

定義

  • 種別: function
  • 定義箇所: contrib/pg_prewarm/autoprewarm.c:500

目的

特定のデータベースに対して、共有メモリ上のブロック情報に基づき、共有バッファへのデータのプリウォーム（事前読み込み）を実行するバックグラウンドワーカーのメイン処理を行う。

シグネチャまたは形状

void autoprewarm_database_main(Datum main_arg)

入力と出力

  • **入力**: Datum main_arg（ワーカー起動時の引数）。内部的には共有メモリ上の apw_state を介してデータベースOIDやプリウォーム対象のブロック範囲を取得する。
  • **出力**: なし（関数は void）。副作用として共有バッファへのデータ読み込みが行われる。

前提条件と不変条件

  • apw_init_shmem() が呼び出され、共有メモリ状態が初期化されていること。
  • プリウォーム対象のブロック情報が動的共有メモリ（DSM）に展開されていること。
  • 処理対象のデータベースに接続可能であること。
  • プリウォーム対象のブロックは、対象データベースまたはグローバルオブジェクトに属していること（Assert による検証）。

アルゴリズムまたは振る舞い

1. **初期化**: シグナルハンドラを設定し、共有メモリセグメントをアタッチしてデータベースへ接続する。 2. **ループ処理**: apw_state で指定された範囲のブロックを順次処理する。 3. **リレーションのオープン**: RelidByRelfilenumber を使用してリレーションを特定し、try_relation_open で開く。失敗した場合は該当リレーションのレコードをスキップする。 4. **フォークの検証**: 各ブロックのフォークが有効かを確認し、無効な場合はスキップする。 5. **プリウォーム実行**: read_stream_begin_relation を使用してストリームを作成し、read_stream_next_buffer を通じてバッファへ読み込む。 6. **安全装置**: have_free_buffer() をチェックし、空きバッファが枯渇した場合は処理を中断する。 7. **終了処理**: リレーションをクローズし、トランザクションをコミットして終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: autoprewarm_start_worker（contrib/pg_prewarm/autoprewarm.c 内でバックグラウンドワーカーとして登録・起動される）。
  • **呼び出し先**: apw_init_shmem, dsm_attach, BackgroundWorkerInitializeConnectionByOid, StartTransactionCommand, RelidByRelfilenumber, try_relation_open, read_stream_begin_relation, read_stream_next_buffer, ReleaseBuffer, read_stream_end, relation_close, CommitTransactionCommand, dsm_detach 等。

関連データ構造

  • BlockInfoRecord: プリウォーム対象ブロックのメタデータ。
  • AutoPrewarmSharedState: 共有メモリ上の制御構造体。
  • AutoPrewarmReadStreamData: ストリーム読み込み時のコンテキスト情報。

失敗モードと境界条件

  • **DSM アタッチ失敗**: 共有メモリセグメントの取得に失敗した場合、ERROR を発生させる。
  • **リレーションオープン失敗**: try_relation_open が失敗した場合、該当するリレーションに関連する全レコードをスキップする。
  • **バッファ枯渇**: have_free_buffer() が偽を返すと、プリウォーム処理を即座に停止する。
  • **無効なフォーク**: smgrexists で検証し、無効なフォークはスキップする。

保守担当者向けメモ

  • 本関数はバックグラウンドワーカーとして動作するため、シグナル処理（SIGTERM）が適切に行われるよう設計されている。
  • read_stream API を利用して効率的なブロック読み込みを行っているため、変更時はストリームのライフサイクル管理に注意すること。
  • プリウォームの進捗は apw_state->prewarmed_blocks に記録される。
