options

定義

  • 種別: struct
  • 定義箇所: contrib/oid2name/oid2name.c:29

目的

oid2name ユーティリティの実行時設定、データベース接続パラメータ、および検索対象のオブジェクトリストを保持するための構造体です。

シグネチャまたは形状

struct options
{
    eary       *tables;
    eary       *oids;
    eary       *filenumbers;

    bool        quiet;
    bool        systables;
    bool        indexes;
    bool        nodb;
    bool        extended;
    bool        tablespaces;

    char       *dbname;
    char       *hostname;
    char       *port;
    char       *username;
    const char *progname;
};

入力と出力

  • **入力**: プログラム起動時のコマンドライン引数から get_opts 関数を通じて値が設定されます。
  • **出力**: データベース接続処理（sql_conn）やクエリ実行処理（sql_exec_searchtables 等）において、接続先情報や検索条件として参照されます。

前提条件と不変条件

  • tables, oids, filenumbers は eary 型の動的配列であり、検索対象のリストを管理します。
  • 接続パラメータ（dbname, hostname 等）は、libpq の接続関数に渡される前に適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • プログラムの初期化フェーズでインスタンス化され、コマンドライン引数の解析結果を格納します。
  • 実行中、ユーザーが指定した検索条件（OID、テーブル名、ファイルノード）を eary 構造体に蓄積し、後の SQL クエリ生成時に IN 句の構築へ利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数が本構造体を初期化し、get_opts 関数へ渡して値を設定します。
  • **呼び出し先**: sql_conn や各 sql_exec_* 関数群が、接続情報や検索フラグを参照します。
  • **注記**: ツールによる contrib/amcheck や contrib/bloom 等の参照結果は、本構造体とは異なる文脈の同名シンボルであり、本ファイルとは無関係です。

関連データ構造

  • eary: 検索対象のリストを管理するために使用される動的配列構造体。

失敗モードと境界条件

  • 構造体内のポインタメンバ（dbname 等）が未初期化のまま接続処理に使用された場合、libpq 側で接続エラーが発生する可能性があります。
  • eary 構造体が適切に初期化されていない状態で要素を追加しようとすると、メモリ破壊やセグメンテーション違反を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は oid2name ユーティリティのグローバルな設定状態を保持するため、変更を加える際は get_opts による引数解析ロジックとの整合性に注意してください。
  • 検索対象の追加（tables, oids, filenumbers）は add_one_elt 等のヘルパー関数を介して行われるため、構造体メンバを直接操作する際は注意が必要です。
