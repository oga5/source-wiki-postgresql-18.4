leftmostvalue_oid

定義

  • 種別: function
  • 定義箇所: contrib/btree_gin/btree_gin.c:257

目的

OID（Object Identifier）型に対するGINインデックスの範囲検索において、インデックスの左端（最小値）となる値を返します。

シグネチャまたは形状

static Datum leftmostvalue_oid(void)

入力と出力

  • 入力: なし
  • 出力: Datum 型として 0（ObjectIdGetDatum(0)）を返します。

前提条件と不変条件

  • 本関数は、OID型の最小値として 0 が有効であるという前提で動作します。
  • GINインデックスの BTLess 系戦略において、スキャン開始点を決定するために使用されます。

アルゴリズムまたは振る舞い

1. ObjectIdGetDatum(0) を呼び出し、OIDの最小値である 0 を Datum 形式に変換します。 2. 変換された値を呼び出し元に返します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/btree_gin/btree_gin.c 内の GIN_SUPPORT(oid, false, leftmostvalue_oid, btoidcmp) マクロ定義により参照されています。
  • 呼び出し先: ObjectIdGetDatum（PostgreSQLの型変換マクロ）。

関連データ構造

  • Datum: PostgreSQLの汎用データ表現型。

失敗モードと境界条件

  • 境界条件: OID 0 はPostgreSQLにおいて無効なOIDとして扱われることがありますが、本関数ではインデックスの論理的な最小値として利用されます。

保守担当者向けメモ

  • 本関数は GIN_SUPPORT マクロを通じて、OID型に対するGINインデックスのサポート関数の一部として登録されます。
  • OID型の比較ロジックに変更が生じた場合、本関数が返す最小値が依然として適切であるか確認してください。
