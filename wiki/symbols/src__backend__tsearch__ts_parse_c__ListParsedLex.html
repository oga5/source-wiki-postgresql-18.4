ListParsedLex

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/ts_parse.c:35

目的

ParsedLex 構造体の要素を管理するための、単方向連結リストのヘッダー構造体です。

シグネチャまたは形状

typedef struct ListParsedLex
{
	ParsedLex  *head;
	ParsedLex  *tail;
} ListParsedLex;

入力と出力

  • 入力: ParsedLex 型のポインタをリストに追加または削除する操作の対象となります。
  • 出力: リストの先頭（head）および末尾（tail）へのポインタを保持します。

前提条件と不変条件

  • head が NULL の場合、リストは空とみなされます。
  • tail は常にリストの最後の要素を指すか、リストが空の場合は NULL となります。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータ保持用であり、操作は専用の関数（LPLAddTail, LPLRemoveHead）を通じて行われます。
  • LexizeData 構造体の一部として、処理中のトークンリスト（towork）や処理済み語彙リスト（waste）を管理するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/ts_parse.c 内の LexizeData 構造体定義および、リスト操作関数（LPLAddTail, LPLRemoveHead）が本構造体を引数として利用します。
  • 呼び出し先: 本構造体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ParsedLex: リストの各ノードとして格納されるデータ構造。
  • LexizeData: 本構造体をメンバとして保持し、辞書処理の状態を管理する構造体。

失敗モードと境界条件

  • リストが空の状態で LPLRemoveHead を呼び出した場合の挙動については、当該関数の実装に依存します。

保守担当者向けメモ

  • LexizeExec における辞書処理の過程で、トークンの追加・削除が頻繁に行われるため、リスト操作時のメモリ管理（palloc）には注意が必要です。
  • towork と waste の二つのリストがどのように状態遷移するかを追跡することで、辞書チェーンの挙動を理解できます。
