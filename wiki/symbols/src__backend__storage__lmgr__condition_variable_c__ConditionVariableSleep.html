ConditionVariableSleep

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/condition_variable.c:95

目的

条件変数が通知されるまで、現在のプロセスを無期限に待機させる。

シグネチャまたは形状

void ConditionVariableSleep(ConditionVariable *cv, uint32 wait_event_info)

入力と出力

  • **入力**:
  • cv: 待機対象となる ConditionVariable 構造体へのポインタ。
  • wait_event_info: 待機イベントの識別子（待機中の統計情報に使用）。
  • **出力**: なし（void）。

前提条件と不変条件

  • 呼び出し側は、事前に ConditionVariablePrepareToSleep を呼び出し、待機リストへの登録準備を完了している必要がある。
  • 偽の起床（Spurious Wakeup）が発生する可能性があるため、必ず while ループ内で条件を評価する文脈で使用すること。

アルゴリズムまたは振る舞い

1. 内部的に ConditionVariableTimedSleep を呼び出す。 2. タイムアウト値として -1（無期限）を指定する。 3. 待機中、プロセスは WaitLatch を介して待機状態となり、シグナルまたはブロードキャストによる通知を待つ。 4. 割り込み（シグナル）を受け取った場合、適切に応答する。

関連データ構造

  • ConditionVariable: 待機リストとスピンロックを保持する構造体。

失敗モードと境界条件

  • タイムアウトなしの待機であるため、通知が送られない限りプロセスは永久に待機し続ける可能性がある。
  • 割り込みが発生した場合、待機から復帰する。

保守担当者向けメモ

  • 本関数は ConditionVariableTimedSleep のラッパーである。
  • 待機終了後は、必ず ConditionVariableCancelSleep を呼び出して待機リストから自身を削除する必要がある。
