SpecialTags

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/wparser_def.c:563

目的

HTMLタグの解析中に特定のタグ（<script> や <style>）を検出し、それ以降のコンテンツを無視すべきかどうかを判定して TParser の ignore フラグを更新します。

シグネチャまたは形状

static void SpecialTags(TParser *prs)

入力と出力

  • **入力**: TParser *prs （解析状態を保持する構造体）
  • **出力**: なし（副作用として prs->ignore を更新）

前提条件と不変条件

  • prs->token に解析対象の文字列が格納されていること。
  • prs->state->lenchartoken にトークンの長さが正しく設定されていること。

アルゴリズムまたは振る舞い

1. prs->state->lenchartoken の値に基づき、トークン文字列を pg_strncasecmp で比較します。 2. <script> または <style> の開始タグが検出された場合、prs->ignore を true に設定します。 3. </script> または </style> の終了タグが検出された場合、prs->ignore を false に設定します。 4. 一致しない場合は何もしません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/wparser_def.c 内の状態遷移テーブル（Actions）から、特定の文字（> や空白）に遭遇した際に呼び出されます。
  • **呼び出し先**: pg_strncasecmp（PostgreSQL の大文字小文字を区別しない文字列比較関数）。

関連データ構造

  • TParser: 解析状態を管理するメイン構造体。ignore フラグおよび現在のトークン情報を保持します。

失敗モードと境界条件

  • トークン長が 6, 7, 8 以外の場合は何も行われません。
  • 文字列比較はケースインセンシティブ（大文字小文字を区別しない）で行われるため、<SCRIPT> 等も正しく判定されます。

保守担当者向けメモ

  • 新しいタグを無視対象に追加する場合は、switch 文のケースを拡張し、適切な長さと文字列比較を追加してください。
  • この関数は状態遷移テーブルから直接参照されているため、テーブル内の他のアクションとの整合性に注意してください。
