table_block_parallelscan_estimate

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableam.c:382

目的

ブロックベースの並列テーブルスキャンにおいて、共有メモリ上に確保すべき並列スキャン記述子（ParallelBlockTableScanDescData）のサイズを見積もる。

シグネチャまたは形状

Size table_block_parallelscan_estimate(Relation rel)

入力と出力

  • **入力**: rel (対象となるリレーションの Relation 構造体ポインタ)
  • **出力**: Size 型のバイト数（ParallelBlockTableScanDescData のサイズ）

前提条件と不変条件

  • 引数 rel は有効なリレーションを指している必要がある。
  • 本関数は常に固定値（sizeof(ParallelBlockTableScanDescData)）を返す。

アルゴリズムまたは振る舞い

1. ParallelBlockTableScanDescData 構造体のサイズを計算する。 2. 計算したサイズを呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c において、TableAmRoutine 構造体の .parallelscan_estimate フィールドに本関数が割り当てられている。
  • **呼び出し先**:
  • 本関数は内部で他の関数を呼び出さない。

関連データ構造

  • ParallelBlockTableScanDescData: 並列スキャン時に共有メモリ上で使用される制御構造体。
  • TableAmRoutine: テーブルアクセスメソッドのインターフェースを定義する構造体。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。引数が有効であれば常に成功する。

保守担当者向けメモ

  • 本関数は、ブロック単位の並列スキャンをサポートするアクセスメソッドにおいて、共有メモリのサイズ見積もりに使用される。
  • 将来的に ParallelBlockTableScanDescData の構造が変更された場合、本関数が返すサイズも自動的に追従する。
