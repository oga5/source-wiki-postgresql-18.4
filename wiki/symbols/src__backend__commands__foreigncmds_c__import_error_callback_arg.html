import_error_callback_arg

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/foreigncmds.c:43

目的

IMPORT FOREIGN SCHEMA コマンド実行時に発生するエラーを補足し、エラーメッセージに文脈（対象テーブル名や実行中のコマンド）を付与するためのエラーコールバック用引数構造体です。

シグネチャまたは形状

typedef struct
{
	char	   *tablename;
	char	   *cmd;
} import_error_callback_arg;

入力と出力

  • 入力: tablename（エラー発生時の対象テーブル名）、cmd（実行中のSQLコマンド文字列）。
  • 出力: なし（構造体として状態を保持）。

前提条件と不変条件

  • IMPORT FOREIGN SCHEMA 処理のコンテキスト内で使用されることを想定しています。
  • エラー発生時にこの構造体を参照するコールバック関数が適切に設定されている必要があります。

アルゴリズムまたは振る舞い

本構造体は、エラー発生時に呼び出されるエラーハンドラに対して、現在処理中のテーブル名とコマンド文字列を渡すためのコンテナとして機能します。これにより、ユーザーに対して「どのテーブルの処理中に、どのようなコマンドでエラーが発生したか」を通知可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/foreigncmds.c 内の ImportForeignSchema 関連処理。
  • 呼び出し先: エラー発生時に登録されたエラーコールバック関数（import_error_callback 等）から参照されます。

関連データ構造

  • ErrorContextCallback（PostgreSQLのエラーコールバック機構で使用される標準的な構造体）。

失敗モードと境界条件

  • tablename や cmd が NULL の場合、エラーメッセージ生成時に適切に処理される必要があります。
  • メモリ不足等により構造体自体が初期化できない場合、エラーハンドリングが機能しません。

保守担当者向けメモ

  • 本構造体は IMPORT FOREIGN SCHEMA のエラーハンドリング専用です。他のDDLコマンドで同様の機能が必要な場合は、汎用的なエラーコンテキスト構造体の導入を検討してください。
