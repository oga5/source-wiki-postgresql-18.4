BTreeTupleGetPointsToTID

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_nbtree.c:3581

目的

B-Tree インデックスのタプルから、そのタプルが指し示す先の TID（Tuple Identifier）を取得する。ピボットタプルとリーフタプルの両方に対して、適切な TID を抽出するために使用される。

シグネチャまたは形状

static inline ItemPointer BTreeTupleGetPointsToTID(IndexTuple itup)

入力と出力

  • **入力**: IndexTuple itup（検証対象のインデックスタプル）
  • **出力**: ItemPointer（タプルが指し示す先の TID へのポインタ）

前提条件と不変条件

  • itup は有効な IndexTuple であること。
  • !heapkeyspace な内部ページデータアイテムであっても、t_tid フィールドにダウンリンク情報が含まれていることを前提としている。

アルゴリズムまたは振る舞い

1. BTreeTupleIsPivot(itup) を呼び出し、対象がピボットタプルかどうかを判定する。 2. ピボットタプルでない場合（リーフタプル等）、BTreeTupleGetHeapTID(itup) を呼び出してヒープ TID を取得し、それを返す。 3. ピボットタプルの場合、itup->t_tid（ダウンリンクブロック情報）を直接返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_nbtree.c 内の各所（行 1338, 1385, 1466, 1573, 1611, 1623 など）。
  • **呼び出し先**:
  • BTreeTupleIsPivot
  • BTreeTupleGetHeapTID

関連データ構造

  • IndexTuple: B-Tree インデックスの各エントリを構成する構造体。
  • ItemPointer: TID を保持するデータ型。

失敗モードと境界条件

  • itup が NULL の場合や、不正なメモリ領域を指している場合の挙動は定義されていない。
  • !heapkeyspace の内部ページデータアイテムに対しては、BTreeTupleGetHeapTID がピボットタプルとして認識しない場合があるが、t_tid フィールドが正しく返されることで整合性が維持される設計となっている。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを考慮してインライン展開される。
  • ピボットタプルと非ピボットタプルで TID の取得元が異なるため、インデックス構造の変更時にはこの判定ロジックが最新の B-Tree 仕様と合致しているか確認が必要である。
