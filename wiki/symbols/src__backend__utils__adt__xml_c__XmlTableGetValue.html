XmlTableGetValue

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xml.c:4925

目的

XMLTABLE 演算子の実行において、現在の行コンテキストから指定された列の値を抽出・変換します。

シグネチャまたは形状

Datum XmlTableGetValue(TableFuncScanState *state, int colnum, Oid typid, int32 typmod, bool *isnull)

入力と出力

  • **入力**:
  • state: 実行状態を保持する TableFuncScanState 構造体。
  • colnum: 抽出対象の列インデックス。
  • typid: ターゲット列の型OID。
  • typmod: ターゲット列の型修飾子。
  • **出力**:
  • isnull: 値がNULLの場合に true が設定されるフラグ。
  • 戻り値: 変換後の Datum 値。

前提条件と不変条件

  • USE_LIBXML が定義されている環境でのみ有効。
  • GetXmlTableBuilderPrivateData により取得した xtCxt 内の xpathobj が XPATH_NODESET 型であり、かつ nodesetval が存在すること。
  • xtCxt->xpathscomp[colnum] がコンパイル済みXPath式として存在すること。

アルゴリズムまたは振る舞い

1. **コンテキスト設定**: xmlSetStructuredErrorFunc を用いてエラーハンドラを現在のコンテキストに設定。 2. **XPath評価**: 現在の行ノードを起点に、xmlXPathCompiledEval で列のXPath式を評価。 3. **型変換**: 評価結果の型（NODESET, STRING, BOOLEAN, NUMBER）に応じて以下の処理を行う:

  • NODESET: ターゲットが XMLOID ならノードをシリアライズして連結。それ以外でノード数が1より多ければエラー（ERRCODE_CARDINALITY_VIOLATION）。
  • STRING/BOOLEAN/NUMBER: 各型に応じた文字列変換を実行。

    • 4. **型キャスト**: 最終的な文字列を InputFunctionCall を通じてターゲットのPostgreSQL型へ変換。 5. **クリーンアップ**: PG_FINALLY ブロックにて xmlXPathFreeObject を呼び出し、XPathオブジェクトを解放。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/xml.c 内の XmlTableRoutine 構造体（.GetValue フィールド）を通じて呼び出される。
  • **呼び出し先**: GetXmlTableBuilderPrivateData, xmlSetStructuredErrorFunc, xmlXPathCompiledEval, xml_ereport, xml_xmlnodetoxmltype, xmlXPathCastNodeSetToString, xmlXPathCastBooleanToString, xmlXPathCastNumberToString, InputFunctionCall, xmlXPathFreeObject 等。

関連データ構造

  • TableFuncScanState: 実行状態を管理。
  • XmlTableBuilderData: XMLTABLEの内部状態（XPathコンテキストやコンパイル済み式）を保持。

失敗モードと境界条件

  • **XPathエラー**: xmlXPathCompiledEval が失敗、またはエラーが発生した場合は xml_ereport によりエラーを報告。
  • **カーディナリティ違反**: XMLOID 以外の型で複数のノードが返された場合、ERRCODE_CARDINALITY_VIOLATION でエラー終了。
  • **型変換失敗**: InputFunctionCall が失敗した場合、PostgreSQLの標準的なエラーハンドリングに従う。
  • **XMLサポート無効時**: NO_XML_SUPPORT() マクロによりエラーを報告。

保守担当者向けメモ

  • libxml2 のメモリ管理とPostgreSQLのメモリコンテキストの整合性に注意すること。
  • xmlXPathCast... 系関数は動的にメモリを確保するため、xml_pstrdup_and_free を通じて適切に解放またはコピーを行う必要がある。
