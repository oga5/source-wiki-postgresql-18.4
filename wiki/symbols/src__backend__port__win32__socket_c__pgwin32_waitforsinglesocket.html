pgwin32_waitforsinglesocket

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:180

目的

Windows環境において、単一のソケットに対する待機処理とPostgreSQLのシグナル監視を統合し、ブロッキング操作をエミュレートする。

シグネチャまたは形状

int pgwin32_waitforsinglesocket(SOCKET s, int what, int timeout)

入力と出力

  • **入力**:
  • s: 待機対象のソケットハンドル。
  • what: 監視するイベントの種類（FD_READ, FD_WRITE 等のビットマスク）。
  • timeout: 待機するタイムアウト時間（ミリ秒）。
  • **出力**:
  • 1: ソケットが準備完了状態になった場合。
  • 0: タイムアウト、シグナル割り込み、またはエラーが発生した場合。

前提条件と不変条件

  • 静的変数 waitevent は初回呼び出し時に作成され、以降の呼び出しで再利用される。
  • WSAEventSelect を使用してソケットとイベントを関連付けるが、関数終了前に必ず WSAEventSelect(s, NULL, 0) を呼び出し、関連付けを解除する。
  • UDPソケットの書き込み待機時には、高負荷時のロック問題を回避するため、0.1秒のタイムアウトを用いたループ処理を行う。

アルゴリズムまたは振る舞い

1. waitevent をリセットする。 2. WSAEventSelect でソケット s に what イベントを関連付ける。 3. pgwin32_signal_event（シグナル用）と waitevent（ソケット用）の2つを対象に WaitForMultipleObjectsEx を実行する。 4. UDPかつ FD_WRITE の場合は、タイムアウト時に WSASend を試行してソケット状態を確認するループを実行する。 5. 待機解除後、イベントの関連付けを解除する。 6. 戻り値に応じて、シグナル受信時は errno = EINTR、タイムアウト時は errno = EWOULDBLOCK を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/win32/socket.c 内のソケット関連関数（接続、読み込み、書き込み処理など）。
  • **呼び出し先**:
  • CreateEvent, ResetEvent, WSAEventSelect, WaitForMultipleObjectsEx, WSASend, WSAGetLastError (Win32 API)
  • isDataGram, TranslateSocketError, pgwin32_dispatch_queued_signals (内部関数)
  • ereport (PostgreSQLエラー報告)

関連データ構造

  • SOCKET: Win32ソケットハンドル。
  • HANDLE: イベントオブジェクト。
  • pgwin32_signal_event: PostgreSQLのシグナル通知用イベント。

失敗モードと境界条件

  • CreateEvent または ResetEvent の失敗: ereport(ERROR, ...) によりプロセスを停止させる。
  • WSAEventSelect の失敗: TranslateSocketError を呼び出し、0を返す。
  • 予期しない WaitForMultipleObjects の戻り値: ereport(ERROR, ...) によりプロセスを停止させる。
  • exceptfds はサポートされていない。

保守担当者向けメモ

  • UDP通信時の高負荷下でのロック問題に対するワークアラウンドが含まれている。
  • WSAEventSelect のデタッチ処理は、他のコードが同一ソケットに対してイベントを関連付ける可能性があるため必須である。
