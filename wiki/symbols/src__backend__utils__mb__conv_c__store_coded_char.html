store_coded_char

定義

  • 種別: static inline function
  • 定義箇所: src/backend/utils/mb/conv.c:352

目的

uint32 型のコードポイントを、ビッグエンディアン形式のバイトストリームとしてメモリ領域へ書き出します。

シグネチャまたは形状

unsigned char *store_coded_char(unsigned char *dest, uint32 code)

入力と出力

  • **入力**:
  • dest: 書き込み先のバッファポインタ。
  • code: 書き込む対象の 32 ビットコードポイント。
  • **出力**:
  • 書き込み後のバッファの次の位置を指すポインタを返します。

前提条件と不変条件

  • dest は、コードポイントのバイト数（最大 4 バイト）を格納するのに十分な領域を確保している必要があります。
  • コードポイントは、上位バイトから順に（ビッグエンディアンで）書き込まれます。

アルゴリズムまたは振る舞い

1. code の各バイトを上位から順に評価します。 2. code の各バイトが 0 でない場合、そのバイトを dest に書き込み、dest ポインタをインクリメントします。

  • 0xff000000 (第 4 バイト)
  • 0x00ff0000 (第 3 バイト)
  • 0x0000ff00 (第 2 バイト)
  • 0x000000ff (第 1 バイト)

    • 3. 最終的に更新された dest ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mb/conv.c 内の以下の箇所から呼び出されています。
  • UtfToLocal および LocalToUtf 変換処理において、変換後のコードをストリームへ出力する際に使用されます（行 639, 657, 669, 792, 804, 805, 818）。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • uint32: PostgreSQL で定義される 32 ビット符号なし整数型。

失敗モードと境界条件

  • バッファオーバーランのチェックは行われません。呼び出し側で十分なバッファサイズを保証する必要があります。
  • 0 のバイトは書き込まれません。例えば code が 0x00000041 ('A') の場合、1 バイトのみが書き込まれます。

保守担当者向けメモ

  • 本関数は inline 指定されており、変換処理のループ内でのオーバーヘッドを最小限に抑える設計となっています。
  • 変換ロジックの変更により、コードポイントのエンディアン形式やバイト長が変更される場合は、本関数のビットシフト処理と条件分岐を再検証してください。
