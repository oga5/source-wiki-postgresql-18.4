uint32

定義

  • 種別: typedef（PostgreSQL の標準的な符号なし 32 ビット整数型）
  • 定義箇所: src/include/utils/jsonpath.h:172（構造体メンバとしての使用例）

目的

JSON Path 式の範囲指定（anybounds）において、最小値（first）および最大値（last）を保持するために使用されます。

シグネチャまたは形状

struct {
    uint32 first;
    uint32 last;
} anybounds;

入力と出力

  • 入力: JSON Path の範囲指定における数値。
  • 出力: 構造体 anybounds のメンバとして格納された 32 ビット符号なし整数。

前提条件と不変条件

  • first および last は、JSON Path の範囲指定におけるインデックスまたは繰り返し回数を表します。
  • 負の値は許容されません。

アルゴリズムまたは振る舞い

  • 本シンボル自体は型定義であり、anybounds 構造体の一部として、JSON Path の評価時に範囲の境界値を保持する役割を担います。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数関係については、提供されたソース情報からは特定できません。
  • ツールによるヒューリスティックな検索結果では、contrib/amcheck や contrib/bloom など、PostgreSQL の他のモジュールでも広範に使用されていることが示唆されていますが、これらは本ヘッダーファイル内の uint32 と直接的な呼び出し関係にあるとは限りません。

関連データ構造

  • anybounds: uint32 型の first と last を保持する親構造体。

失敗モードと境界条件

  • 32 ビットの範囲（0 ～ 4,294,967,295）を超える値は格納できません。

保守担当者向けメモ

  • 本シンボルは PostgreSQL 全体で利用される基本型です。jsonpath.h 内では主に範囲指定の制約として機能します。
  • ツールによる参照情報は、プロジェクト全体での広範な利用を示しており、特定の関数呼び出しを指すものではありません。
