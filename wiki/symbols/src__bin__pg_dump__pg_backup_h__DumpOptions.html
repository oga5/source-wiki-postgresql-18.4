DumpOptions

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_dump/pg_backup.h:170

目的

pg_dump 実行時におけるダンプ処理の詳細な動作設定を保持するための構造体です。コマンドライン引数やデフォルト値から生成された設定を、バックアップ処理全体で共有するために使用されます。

シグネチャまたは形状

typedef struct _dumpOptions {
    ConnParams  cparams;
    int         binary_upgrade;
    int         dumpSections;
    bool        aclsSkip;
    const char *lockWaitTimeout;
    int         dump_inserts;
    /* ... 多数のフラグメンバ ... */
    bool        include_everything;
    bool        dumpSchema;
    bool        dumpData;
    bool        dumpStatistics;
    char       *restrict_key;
} DumpOptions;

入力と出力

  • **入力**: NewDumpOptions() により初期化され、コマンドライン引数や RestoreOptions からの値（dumpOptionsFromRestoreOptions）によって設定されます。
  • **出力**: SetArchiveOptions() を介して Archive ハンドルに適用され、ダンプ処理の各フェーズで参照されます。

前提条件と不変条件

  • include_everything が true の場合、特定の包含スイッチが指定されていない限り、データベース全体がダンプ対象となります。
  • dumpSchema, dumpData, dumpStatistics は、ユーザー指定のフラグから派生して算出される不変的な状態フラグです。

アルゴリズムまたは振る舞い

1. **初期化**: NewDumpOptions() でメモリ確保とデフォルト値の設定が行われます。 2. **構成**: ユーザー指定のオプション（no_comments, disable_triggers 等）が構造体に格納されます。 3. **適用**: SetArchiveOptions() を通じて、アーカイバがダンプ処理中に参照する設定として登録されます。 4. **利用**: flagInhAttrs() 等の関数内で、特定の属性をダンプするかどうかの判断基準として使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c: NewDumpOptions および InitDumpOptions の実装。
  • src/bin/pg_dump/common.c: flagInhAttrs 関数内でダンプオプションを参照。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • Archive: ダンプ処理のメインコンテキスト。
  • ConnParams: データベース接続パラメータ。
  • RestoreOptions: リストア時のオプション（dumpOptionsFromRestoreOptions で変換可能）。

失敗モードと境界条件

  • メモリ不足: NewDumpOptions 内で pg_malloc が失敗した場合、プロセスは終了します。
  • 整合性: dump_inserts が 0 の場合は COPY コマンドが使用され、それ以外の場合は指定された行数ごとの INSERT 文が生成されます。

保守担当者向けメモ

  • 本構造体は非常に多くのフラグを含んでおり、新しいコマンドラインオプションを追加する際は、適切なカテゴリ（ユーザー設定、フラグ、派生フラグ）に分類して追加してください。
  • dumpSchema 等の派生フラグは、ユーザー設定フラグの変更後に正しく再計算される必要があります。
