jsonb_contained

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_op.c:129

目的

SQL演算子 <@（包含関係）を実装し、第1引数の jsonb が第2引数の jsonb に含まれているかを判定します。これは jsonb_contains (@>) の交換法則（commutator）として機能します。

シグネチャまたは形状

Datum jsonb_contained(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの jsonb 型データを受け取ります。
  • **出力**: bool 型の Datum を返します。第1引数が第2引数に含まれる場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力される2つの jsonb オブジェクトのルート構造（オブジェクトか否か）が一致している必要があります。一致しない場合、包含関係は成立しないとみなされます。

アルゴリズムまたは振る舞い

1. PG_GETARG_JSONB_P を使用して2つの入力 jsonb を取得します。 2. JB_ROOT_IS_OBJECT マクロを用いて、両者のルートが共にオブジェクトであるか（あるいは共にオブジェクトでないか）を比較します。不一致の場合は直ちに false を返します。 3. JsonbIteratorInit を用いて、両方の jsonb 構造に対するイテレータを初期化します。 4. JsonbDeepContains を呼び出し、再帰的に包含関係を検証した結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）を介してSQL演算子 <@ から呼び出されます。
  • **呼び出し先**:
  • JsonbIteratorInit
  • JsonbDeepContains
  • PG_RETURN_BOOL

関連データ構造

  • Jsonb: jsonb 型の内部表現。
  • JsonbIterator: jsonb 構造を走査するためのイテレータ。

失敗モードと境界条件

  • ルート構造が異なる（一方がオブジェクトで他方が配列など）場合、包含関係は成立せず false を返します。
  • メモリ不足等のシステムエラーが発生した場合、PostgreSQLの標準的なエラーハンドリングにより処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は jsonb_contains の交換法則として定義されています。ロジックの変更時は jsonb_contains との整合性に注意してください。
  • JsonbDeepContains による再帰的な比較を行っているため、非常に深いネストを持つ jsonb データに対してはスタック消費量に留意する必要があります。
