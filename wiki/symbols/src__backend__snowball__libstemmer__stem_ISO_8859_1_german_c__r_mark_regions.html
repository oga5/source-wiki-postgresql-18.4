r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_german.c:237

目的

単語内の接尾辞削除の境界となる領域 R1 および R2 を特定し、SN_env 構造体の整数配列に格納する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z（ステミング環境。現在のカーソル位置 c、文字列長 l、および領域境界を保持する配列 I を含む）
  • **出力**: 成功時に 1、失敗時に 0 を返す。副作用として z->I[0]（R1の開始位置）、z->I[1]（R2の開始位置）、z->I[2]（一時的な境界位置）が更新される。

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。

アルゴリズムまたは振る舞い

1. z->I[0]（R1の開始位置）を、単語の先頭から3文字目以降の最初の母音と子音の並びの直後に設定する。 2. z->I[1]（R2の開始位置）を、R1の開始位置からさらに母音と子音の並びを探索した直後に設定する。 3. 母音・子音の判定には out_grouping（子音探索）および in_grouping（母音探索）を使用する。 4. 探索中に境界を超えた場合や条件を満たさない場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: german_ISO_8859_1_stem（直接の呼び出し関係はソースコードの制御フローより推論）。
  • **呼び出し先**: out_grouping, in_grouping（Snowball ランタイムのユーティリティ関数）。
  • 補足: 他の言語のステマー（Basque, Catalan, Danish 等）にも同名の関数が存在するが、本関数は stem_ISO_8859_1_german.c 内で定義されたもの。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。I 配列（整数値）を領域境界の保持に使用。

失敗モードと境界条件

  • 文字列長が不足している場合や、母音・子音のパターンが見つからない場合は 0 を返す。
  • z->c が z->l を超える場合、処理を中断する。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接編集せず元の Snowball スクリプトを修正すること。
  • z->I[0], z->I[1], z->I[2] の各インデックスが何を表すかは、Snowball のランタイム仕様に依存する。
