Nsrt

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_utils_num.h:24

目的

GiST 索引のソート処理において、数値型キーとそのインデックス（または識別子）をペアとして保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	int			i;
	GBT_NUMKEY *t;
} Nsrt;

入力と出力

  • i: 整数型の識別子またはインデックス値。
  • t: 数値型キーの内部表現である GBT_NUMKEY 構造体へのポインタ。

前提条件と不変条件

  • t は有効な GBT_NUMKEY 構造体を指している必要があります。
  • ソート操作のコンテキストにおいて、i は元のデータ位置や順序を追跡するために使用されます。

アルゴリズムまたは振る舞い

各データ型の実装（btree_bool.c, btree_float8.c 等）において、Nsrt 型のポインタをキャストし、内部の GBT_NUMKEY を取り出して比較や処理を行うために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のファイル内の関数が Nsrt を参照しています。
  • contrib/btree_gist/btree_bool.c
  • contrib/btree_gist/btree_cash.c
  • contrib/btree_gist/btree_date.c
  • contrib/btree_gist/btree_enum.c
  • contrib/btree_gist/btree_float4.c
  • contrib/btree_gist/btree_float8.c
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • GBT_NUMKEY: 数値型キーの内部表現。
  • 各種 *KEY 構造体（boolKEY, float8KEY 等）: Nsrt からキャストされた t が指す実際のデータ型。

失敗モードと境界条件

  • t が NULL の場合、キャスト後のアクセスでセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は、GiST 索引のソートサポート（qsort 等）において、キーと元のインデックスを紐付けるための汎用的なラッパーとして機能します。
  • 新しい数値型を btree_gist に追加する際は、既存の btree_*.c ファイルと同様に、Nsrt を介してキーへアクセスするパターンを踏襲してください。
