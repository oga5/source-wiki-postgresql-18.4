PGEventId

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-events.h:27

目的

libpq のイベント API において、接続（PGconn）や結果セット（PGresult）のライフサイクルイベントを識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	PGEVT_REGISTER,
	PGEVT_CONNRESET,
	PGEVT_CONNDESTROY,
	PGEVT_RESULTCREATE,
	PGEVT_RESULTCOPY,
	PGEVT_RESULTDESTROY
} PGEventId;

入力と出力

  • 入力: イベントハンドラ（PGEventProc）の引数 evtId として渡されます。
  • 出力: なし（列挙値としてイベントの種類を特定します）。

前提条件と不変条件

  • libpq 内部で発生する特定の状態遷移（接続の破棄や結果の生成など）と一対一で対応します。
  • イベントハンドラは、この値に基づいて適切な処理（メモリの確保や解放など）を分岐させる必要があります。

アルゴリズムまたは振る舞い

libpq 内部でイベントが発生した際、登録済みの各イベントハンドラに対してこの列挙値が渡されます。

  • PGEVT_REGISTER: ハンドラ登録時に発行。
  • PGEVT_CONNRESET / PGEVT_CONNDESTROY: 接続の再設定または破棄時に発行。
  • PGEVT_RESULTCREATE / PGEVT_RESULTCOPY / PGEVT_RESULTDESTROY: 結果セットの生成、コピー、破棄時に発行。

呼び出し元と呼び出し先

  • 呼び出し元: libpq 内部のイベント発行ロジック（詳細はソースコードの文脈からは未特定）。
  • 呼び出し先: 登録された PGEventProc 型の関数。

関連データ構造

  • PGEventProc: この列挙値を引数に取る関数ポインタ型。
  • PGconn, PGresult: イベントの対象となるデータ構造。

失敗モードと境界条件

  • イベントハンドラ内で PGEVT_RESULTDESTROY や PGEVT_CONNDESTROY を適切に処理しない場合、メモリリークが発生します。

保守担当者向けメモ

  • 新しいイベントタイプを追加する場合は、この列挙型を拡張し、対応するイベント発行箇所を libpq 内部に実装する必要があります。
  • 呼び出し元の具体的な関数名は、本ソース抜粋の範囲外のため特定されていません。
