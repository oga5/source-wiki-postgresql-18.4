ExecOpenIndices

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execIndexing.c:160

目的

テーブルへのタプル挿入や更新に伴い、対象リレーションに関連する全インデックスをオープンし、インデックス操作に必要な情報を ResultRelInfo にキャッシュする。

シグネチャまたは形状

void ExecOpenIndices(ResultRelInfo *resultRelInfo, bool speculative)

入力と出力

  • **入力**:
  • resultRelInfo: インデックス情報を格納する対象リレーションの構造体。
  • speculative: 投機的挿入（INSERT ... ON CONFLICT）を行うかを示すフラグ。
  • **出力**: なし（resultRelInfo の ri_IndexRelationDescs および ri_IndexRelationInfo フィールドを更新）。

前提条件と不変条件

  • resultRelInfo->ri_IndexRelationDescs は呼び出し時に NULL であること（二重呼び出しの禁止）。
  • 対象リレーションにインデックスが存在しない場合、関数は即座にリターンする。

アルゴリズムまたは振る舞い

1. リレーションのインデックスリストを取得し、インデックスが存在しない場合は終了する。 2. インデックス数分のメモリを確保し、ResultRelInfo に割り当てる。 3. 各インデックスに対して RowExclusiveLock を取得してオープンする。 4. BuildIndexInfo を呼び出し、インデックスのキー情報を抽出する。 5. speculative が真かつ一意性インデックス（排他制約を除く）の場合、BuildSpeculativeIndexInfo を呼び出し、投機的挿入に必要な情報を追加する。 6. オープンしたリレーションとインデックス情報を配列に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/indexing.c
  • src/backend/commands/copyfrom.c
  • src/backend/executor/execPartition.c
  • src/backend/executor/nodeModifyTable.c
  • **呼び出し先**:
  • RelationGetIndexList
  • index_open
  • BuildIndexInfo
  • BuildSpeculativeIndexInfo

関連データ構造

  • ResultRelInfo: インデックス情報を保持する実行時のリレーション情報。
  • Relation: インデックスの物理的記述子。
  • IndexInfo: インデックスの論理的属性情報。

失敗モードと境界条件

  • インデックスが存在しない場合、ri_NumIndices は 0 に設定され、処理はスキップされる。
  • ExecOpenIndices を同一の ResultRelInfo に対して二度呼び出すと、Assert により失敗する。

保守担当者向けメモ

  • 本関数で取得したロックは、後続の ExecCloseIndices で解放される必要がある。
  • インデックスが indisready でない場合でもオープン処理が行われる。これは最適化の対象外としている。
