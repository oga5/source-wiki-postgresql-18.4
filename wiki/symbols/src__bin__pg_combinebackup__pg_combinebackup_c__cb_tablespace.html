cb_tablespace

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:91

目的

バックアップ統合処理において、テーブルスペースの配置情報およびマッピング情報を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct cb_tablespace
{
	Oid			oid;
	bool		in_place;
	char		old_dir[MAXPGPATH];
	char		new_dir[MAXPGPATH];
	struct cb_tablespace *next;
} cb_tablespace;

入力と出力

  • oid: テーブルスペースの識別子。
  • in_place: テーブルスペースが元の場所に配置されるかを示すフラグ。
  • old_dir: 元のディレクトリパス。
  • new_dir: 新しいディレクトリパス。
  • next: 連結リストを構成するための次要素へのポインタ。

前提条件と不変条件

  • old_dir および new_dir は MAXPGPATH のサイズ制限に従います。
  • next ポインタにより、複数のテーブルスペースを単方向連結リストとして管理可能です。

アルゴリズムまたは振る舞い

  • pg_combinebackup 実行時に、既存のテーブルスペースを走査してリストを構築するために使用されます。
  • pg_malloc0 を用いてメモリ確保が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: scan_for_existing_tablespaces 関数内でリストの構築に使用されています。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • cb_options: プログラム全体のオプション設定を保持する構造体。
  • cb_tablespace_mapping: テーブルスペースのマッピング情報を管理する関連構造体。

失敗モードと境界条件

  • メモリ不足時に pg_malloc0 が失敗する可能性があります。
  • パス長が MAXPGPATH を超える場合、バッファオーバーフローのリスクがあるため、入力パスの検証が必要です。

保守担当者向けメモ

  • 本構造体は連結リストとして実装されているため、リストの走査や解放を行う際はメモリリークに注意してください。
  • scan_for_existing_tablespaces 関数が本構造体の主な生成元であるため、テーブルスペースの検出ロジックを変更する際は同関数を確認してください。
