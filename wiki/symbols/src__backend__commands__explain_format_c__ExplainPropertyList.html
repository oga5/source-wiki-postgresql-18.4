ExplainPropertyList

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:37

目的

EXPLAIN コマンドの出力において、文字列のリストを特定のフォーマット（TEXT, XML, JSON, YAML）に従って整形し、ExplainState が管理するバッファへ出力する。

シグネチャまたは形状

void ExplainPropertyList(const char *qlabel, List *data, ExplainState *es)

入力と出力

  • qlabel: プロパティのラベル名。
  • data: 出力対象の文字列を保持する List。
  • es: 出力状態を管理する ExplainState 構造体。
  • 戻り値: なし（es->str に結果が書き込まれる）。

前提条件と不変条件

  • es は初期化済みであること。
  • data リスト内の各要素は const char * としてキャスト可能であること。
  • es->format に基づいて適切なフォーマット処理が選択される。

アルゴリズムまたは振る舞い

es->format の値に応じて以下の処理を行う：

  • **TEXT**: qlabel を出力後、リスト要素をカンマ区切りで出力。
  • **XML**: qlabel をタグ名として開始し、各要素を <Item> タグで囲んで出力。
  • **JSON**: qlabel をキーとして、リストを JSON 配列 [...] 形式で出力。
  • **YAML**: qlabel を出力後、各要素を - を付与したリスト形式で出力。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c 内の各所（例: ExplainPrintPlan 等の実行計画出力処理）。
  • **呼び出し先**:
  • ExplainIndentText, ExplainXMLTag, ExplainJSONLineEnding, ExplainYAMLLineStarting（フォーマット制御）
  • appendStringInfo 系関数（バッファ書き込み）
  • escape_xml, escape_json, escape_yaml（エスケープ処理）

関連データ構造

  • ExplainState: 出力バッファやフォーマット状態を保持。
  • List: 出力対象の文字列リスト。

失敗モードと境界条件

  • data が空の場合、各フォーマットの仕様に従い、空のリストやタグが生成される。
  • XML 出力時、escape_xml によってメモリが動的に割り当てられ、処理後に pfree される。

保守担当者向けメモ

  • 新しいフォーマットを追加する場合、switch 文に新しい case を追加する必要がある。
  • data リスト内の要素が文字列以外である場合、本関数を直接呼び出すことはできない。
