arcp

定義

  • 種別: struct
  • 定義箇所: src/backend/regex/regexec.c:39

目的

正規表現エンジンにおいて、DFA（決定性有限オートマトン）の「アウトアーク（outarc）」を指し示すポインタとして機能します。

シグネチャまたは形状

struct arcp
{
    struct sset *ss;
    color       co;
};

入力と出力

  • **入力**: sset 構造体へのポインタおよび color 型の識別子。
  • **出力**: なし（構造体定義）。

前提条件と不変条件

  • ss は状態セット（sset）を指し、co はその状態遷移に関連する色（文字クラス）を表します。

アルゴリズムまたは振る舞い

本構造体は、DFA の状態遷移を管理するためのリンク情報として使用されます。rege_dfa.c において、マッチング中の状態遷移先を追跡するために、配列やポインタ経由で動的に確保・参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/regex/rege_dfa.c: MALLOC_ARRAY によるメモリ確保、および ap や lastap 変数としての利用。
  • src/backend/regex/regexec.c: vars 構造体内の ins（inarcs の連鎖）、inchain（ポインタベクトル）、incarea（ストレージ）として利用。
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、関数呼び出しは行いません。

関連データ構造

  • struct sset: 状態セットを管理する構造体。
  • struct vars: マッチングのコンテキストを保持する構造体。
  • color: 文字クラスを識別する型。

失敗モードと境界条件

  • incarea として固定長配列（FEWSTATES * 2 * FEWCOLORS）が定義されており、大規模な正規表現ではメモリ不足や境界外アクセスのリスクを伴うため、MALLOC_ARRAY による動的確保と併用されます。

保守担当者向けメモ

  • regexec.c 内の incarea は、小規模な正規表現に対する最適化としてスタックまたは構造体の一部として確保されます。大規模な場合は rege_dfa.c で動的に割り当てられるため、メモリ管理のライフサイクルに注意してください。
