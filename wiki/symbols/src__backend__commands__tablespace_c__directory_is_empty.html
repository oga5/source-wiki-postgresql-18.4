directory_is_empty

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tablespace.c:852

目的

指定されたディレクトリが空であるか（. および .. 以外のエントリを含まないか）を判定します。

シグネチャまたは形状

bool directory_is_empty(const char *path)

入力と出力

  • **入力**: path (検査対象のディレクトリパス)
  • **出力**: ディレクトリが空であれば true、それ以外（ファイルやサブディレクトリが存在する場合）は false を返します。

前提条件と不変条件

  • path で指定されたディレクトリは、呼び出し元によってアクセス可能である必要があります。
  • AllocateDir および ReadDir が正常に動作する環境である必要があります。

アルゴリズムまたは振る舞い

1. AllocateDir(path) を呼び出し、ディレクトリ記述子を取得します。 2. ReadDir を使用してディレクトリ内のエントリを順次読み取ります。 3. エントリ名が . または .. である場合はスキップします。 4. それ以外のエントリが見つかった場合、FreeDir を呼び出してリソースを解放し、false を返します。 5. ループ終了までエントリが見つからなければ、FreeDir を呼び出し true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c
  • src/backend/commands/tablespace.c
  • src/backend/utils/adt/misc.c
  • **呼び出し先**:
  • AllocateDir, ReadDir, FreeDir (PostgreSQL のディレクトリ操作 API)

関連データ構造

  • DIR (ディレクトリ記述子)
  • struct dirent (ディレクトリ内のエントリ情報)

失敗モードと境界条件

  • 指定された path が存在しない、または読み取り権限がない場合、AllocateDir がエラーを発生させる可能性があります。
  • ディレクトリが空でない場合、最初に見つかったエントリの時点で即座に false を返します。

保守担当者向けメモ

  • 本関数は src/include/commands/tablespace.h で宣言されており、モジュールを跨いで利用されています。
  • ディレクトリ内のエントリ走査は、. と .. を除外する標準的な実装です。
