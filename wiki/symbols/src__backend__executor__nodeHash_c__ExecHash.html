ExecHash

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:90

目的

Hash ノードに対する ExecProcNode 呼び出し規約を拒否し、エラーを発生させるためのスタブ関数です。

シグネチャまたは形状

static TupleTableSlot *ExecHash(PlanState *pstate)

入力と出力

  • **入力**: pstate (PlanState *) - 実行中のプランノードの状態。
  • **出力**: なし（常にエラーで終了するため、戻り値は到達しません）。

前提条件と不変条件

  • 本関数は Hash ノードの ExecProcNode フィールドに割り当てられますが、実行時に呼び出されることは想定されていません。

アルゴリズムまたは振る舞い

1. elog(ERROR, ...) を呼び出し、Hash ノードが ExecProcNode 呼び出し規約をサポートしていない旨を報告して処理を中断します。 2. NULL を返しますが、このコードパスには到達しません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c:383 にて、hashstate->ps.ExecProcNode に関数ポインタとして代入されています。
  • **呼び出し先**: elog（エラー報告用マクロ）。

関連データ構造

  • PlanState: プランノードの実行状態を保持する構造体。

失敗モードと境界条件

  • 本関数が呼び出された場合、常にエラーが発生します。これは Hash ノードが通常のタプル取得ノードではなく、ハッシュ結合の構築フェーズ専用のノードであることを示しています。

保守担当者向けメモ

  • Hash ノードは MultiExecHash を通じて直接呼び出される設計であり、標準的な ExecProcNode インターフェース経由での実行は意図されていません。この関数は、誤った呼び出しを検知するためのガードレールとして機能しています。
