relation_stats_argnum

定義

  • 種別: enum
  • 定義箇所: src/backend/statistics/relation_stats.c:37

目的

テーブル（リレーション）の統計情報を操作する関数において、引数のインデックスを識別するための列挙型です。

シグネチャまたは形状

enum relation_stats_argnum
{
    RELSCHEMA_ARG = 0,
    RELNAME_ARG,
    RELPAGES_ARG,
    RELTUPLES_ARG,
    RELALLVISIBLE_ARG,
    RELALLFROZEN_ARG,
    NUM_RELATION_STATS_ARGS
};

入力と出力

  • 入力: なし（定数定義）
  • 出力: 整数値（各列挙子は 0 から始まるインデックスを表す）

前提条件と不変条件

  • NUM_RELATION_STATS_ARGS は、この列挙型の要素数および引数配列のサイズとして機能します。
  • 各メンバは、pg_restore_relation_stats 等の関数が受け取る引数配列の順序と一致している必要があります。

アルゴリズムまたは振る舞い

  • SQL関数から渡される引数配列に対して、特定の統計項目（スキーマ名、リレーション名、ページ数、タプル数など）を特定するためのオフセットとして使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/statistics/relation_stats.c 内の統計更新関数群（pg_restore_relation_stats 等）。
  • 呼び出し先: 情報なし。

関連データ構造

  • relarginfo: 本列挙型と対応する統計カラム・データ型のマッピング定義に使用される構造体（ファイル内で定義）。

失敗モードと境界条件

  • 列挙子の順序を変更する場合、それを利用する関数側の引数処理ロジックも同時に更新する必要があります。

保守担当者向けメモ

  • 新しい統計項目を pg_class 更新対象に追加する場合は、この列挙型にメンバを追加し、NUM_RELATION_STATS_ARGS が正しくインクリメントされるようにしてください。
