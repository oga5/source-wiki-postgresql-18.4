JoinPathExtraData

目的

外部データラッパー（FDW）が結合パス（Join Path）を生成する際、結合に関する追加的な情報や制約を保持・伝達するために使用されます。

シグネチャまたは形状

typedef struct JoinPathExtraData JoinPathExtraData; （詳細は nodes/pathnodes.h を参照）

入力と出力

本シンボル自体はデータ構造の型定義です。GetForeignJoinPaths_function の引数として渡され、プランナからFDWへ結合関連のコンテキスト情報を提供します。

前提条件と不変条件

  • GetForeignJoinPaths コールバック関数において、プランナによって適切に初期化された状態で提供されます。

アルゴリズムまたは振る舞い

FDWの GetForeignJoinPaths 実装内で、結合のコスト見積もりやパス生成の判断材料として参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/optimizer/path/joinpath.c
  • src/backend/optimizer/path/costsize.c
  • contrib/postgres_fdw/postgres_fdw.c
  • **呼び出し先**:
  • 本シンボルはデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • RelOptInfo: 結合対象となるリレーションの情報を保持します。
  • PlannerInfo: クエリ全体の計画情報を保持します。
  • JoinType: 結合の種類（INNER, LEFT, SEMI等）を定義します。

失敗モードと境界条件

  • 本シンボル自体に失敗モードはありません。
  • GetForeignJoinPaths 内で本構造体が NULL でないことを前提とする実装が一般的です。

保守担当者向けメモ

  • 本構造体の定義は src/include/nodes/pathnodes.h に存在します。構造体のフィールド構成を変更する場合は、FDW API への影響範囲を考慮してください。
  • postgres_fdw 等の拡張モジュールで頻繁に参照されているため、変更時は contrib 配下のコードとの整合性に注意してください。
