HeapTupleGetDatum

定義

  • 種別: static inline function
  • 定義箇所: src/include/funcapi.h:229

目的

HeapTupleData 構造体から、PostgreSQL の関数呼び出しインターフェース（fmgr）で扱える Datum 型への変換を行う。

シグネチャまたは形状

static inline Datum HeapTupleGetDatum(const HeapTupleData *tuple)

入力と出力

  • **入力**: const HeapTupleData *tuple （変換対象のタプルへのポインタ）
  • **出力**: Datum （タプルヘッダーから抽出されたデータ表現）

前提条件と不変条件

  • 引数 tuple は有効な HeapTupleData 構造体を指している必要がある。
  • tuple->t_data が有効な HeapTupleHeader を保持していることを前提とする。

アルゴリズムまたは振る舞い

1. 入力された HeapTupleData 構造体の t_data メンバ（HeapTupleHeader）にアクセスする。 2. HeapTupleHeaderGetDatum マクロを呼び出し、タプルヘッダーを Datum 型に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/hstore/hstore_io.c
  • contrib/hstore/hstore_op.c
  • contrib/pageinspect/brinfuncs.c
  • contrib/pageinspect/btreefuncs.c
  • contrib/pageinspect/ginfuncs.c
  • **呼び出し先**:
  • HeapTupleHeaderGetDatum（マクロ）

関連データ構造

  • HeapTupleData: タプルの物理的な格納状態を管理する構造体。
  • Datum: PostgreSQL における汎用的なデータ表現型。

失敗モードと境界条件

  • tuple が NULL の場合、デリファレンスによりクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結している。
  • HeapTupleHeaderGetDatum マクロへの単なるラッパーであるため、将来的にタプルヘッダーの表現が変更された場合は、当該マクロの定義を確認すること。
