DeleteRelationTuple

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:1589

目的

指定された OID に対応するリレーションのメタデータをシステムカタログ pg_class から削除する。

シグネチャまたは形状

void DeleteRelationTuple(Oid relid)

入力と出力

  • **入力**: relid (削除対象リレーションの OID)
  • **出力**: なし（成功時）。失敗時はエラーをスローする。

前提条件と不変条件

  • 本関数はトランザクション内で呼び出される必要がある。
  • pg_class に対する RowExclusiveLock を取得して操作を行う。

アルゴリズムまたは振る舞い

1. table_open を使用して pg_class を RowExclusiveLock モードで開く。 2. SearchSysCache1 を用いて、指定された relid に対応する pg_class のタプルを検索する。 3. タプルが見つからない場合は elog(ERROR) を発生させる。 4. CatalogTupleDelete を呼び出し、pg_class から該当タプルを削除する。 5. ReleaseSysCache でキャッシュを解放し、table_close で pg_class を閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c (リレーション削除処理内)
  • src/backend/catalog/index.c (インデックス削除処理内)
  • **呼び出し先**:
  • table_open / table_close
  • SearchSysCache1 / ReleaseSysCache
  • CatalogTupleDelete

関連データ構造

  • Relation: pg_class を操作するためのリレーション記述子。
  • HeapTuple: pg_class から取得されるタプル構造体。
  • Oid: リレーションを一意に識別するオブジェクト ID。

失敗モードと境界条件

  • 指定された relid が pg_class に存在しない場合、SearchSysCache1 が無効なタプルを返し、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数はカタログ上のエントリのみを削除する。物理的なストレージの削除や、依存関係のチェックは呼び出し元で行う必要がある。
  • pg_class のロックレベルは RowExclusiveLock で固定されている。
