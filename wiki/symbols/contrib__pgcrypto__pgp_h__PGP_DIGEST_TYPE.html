PGP_DIGEST_TYPE

定義

  • 種別: enum
  • 定義箇所: contrib/pgcrypto/pgp.h:98

目的

OpenPGP (RFC 4880) 規格で定義されるハッシュアルゴリズム（ダイジェスト）を識別するための列挙型です。

シグネチャまたは形状

enum PGP_DIGEST_TYPE {
    PGP_DIGEST_MD5 = 1,
    PGP_DIGEST_SHA1 = 2,
    PGP_DIGEST_RIPEMD160 = 3,
    PGP_DIGEST_XSHA = 4,
    PGP_DIGEST_MD2 = 5,
    PGP_DIGEST_TIGER192 = 6,
    PGP_DIGEST_HAVAL5_160 = 7,
    PGP_DIGEST_SHA256 = 8,
    PGP_DIGEST_SHA384 = 9,
    PGP_DIGEST_SHA512 = 10
};

入力と出力

  • 入力: なし（定数定義）
  • 出力: PGP 処理におけるハッシュアルゴリズムの識別子として使用されます。

前提条件と不変条件

  • RFC 4880 に準拠した値が割り当てられています。
  • 一部の値（XSHA, MD2, TIGER192, HAVAL5_160）には obsolete（廃止）の注釈が付与されています。

アルゴリズムまたは振る舞い

  • 本列挙型は、PGP 暗号化・復号処理において使用するハッシュアルゴリズムを選択するための識別子を提供します。
  • PGP_DIGEST_SHA1 は must（必須）、PGP_DIGEST_MD5 は should（推奨）かつ deprecated（非推奨）として定義されています。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース excerpt には含まれていません。

関連データ構造

  • PGP_Context: 暗号化/復号セッションの状態を保持する構造体であり、本列挙型を使用してハッシュアルゴリズムを指定する可能性があります。

失敗モードと境界条件

  • 廃止されたアルゴリズム（obsolete）を指定した場合の挙動は、本定義のみでは不明です。

保守担当者向けメモ

  • RFC 4880 の仕様変更に伴い、新しいハッシュアルゴリズムの追加や、既存アルゴリズムの非推奨化を行う際は本定義を更新してください。
  • 実際の暗号処理の実装については px.h および pgp.c を参照してください。
