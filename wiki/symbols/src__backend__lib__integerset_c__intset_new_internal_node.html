intset_new_internal_node

定義

  • 種別: function
  • 定義箇所: src/backend/lib/integerset.c:314

目的

B-tree 構造における新しい内部ノード（intset_internal_node）をメモリ上に割り当て、初期化する。

シグネチャまたは形状

static intset_internal_node *intset_new_internal_node(IntegerSet *intset)

入力と出力

  • **入力**: IntegerSet *intset（ノードを割り当てるメモリコンテキストを保持する集合構造体）
  • **出力**: 初期化された intset_internal_node へのポインタ。

前提条件と不変条件

  • intset は有効な IntegerSet 構造体を指していること。
  • intset->context を通じてメモリ割り当てが行われる。
  • 戻り値のノードの level フィールドは、呼び出し元で適切に設定される必要がある（初期値は 0 に設定される）。

アルゴリズムまたは振る舞い

1. intset->context から intset_internal_node サイズのメモリを割り当てる。 2. 割り当てたメモリサイズを intset->mem_used に加算する。 3. ノードの level を 0 に、num_items を 0 に初期化する。 4. 初期化済みのノードポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/integerset.c:509
  • src/backend/lib/integerset.c:537
  • **呼び出し先**:
  • MemoryContextAlloc
  • GetMemoryChunkSpace

関連データ構造

  • IntegerSet: 集合の管理構造体。
  • intset_internal_node: B-tree の内部ノード構造体。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は内部ノードのメモリ確保と基本初期化のみを担当する。ノードの階層（level）や子ノードとのリンク設定は呼び出し元で行う必要がある。
