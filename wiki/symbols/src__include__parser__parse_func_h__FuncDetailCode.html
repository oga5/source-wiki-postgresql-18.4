FuncDetailCode

定義

  • 種別: enum
  • 定義箇所: src/include/parser/parse_func.h:22

目的

関数呼び出しや演算子解決の試行結果を示すステータスコードを定義します。パーサーがシステムカタログから関数を検索した際の状態を識別するために使用されます。

シグネチャまたは形状

typedef enum {
    FUNCDETAIL_NOTFOUND,
    FUNCDETAIL_MULTIPLE,
    FUNCDETAIL_NORMAL,
    FUNCDETAIL_PROCEDURE,
    FUNCDETAIL_AGGREGATE,
    FUNCDETAIL_WINDOWFUNC,
    FUNCDETAIL_COERCION
} FuncDetailCode;

入力と出力

本シンボルは列挙型であり、関数呼び出しの戻り値や変数として状態を保持するために使用されます。入力（関数検索処理）の結果として、上記いずれかの値が出力されます。

前提条件と不変条件

  • 検索対象の関数名および引数型が指定されている必要があります。
  • 検索結果が FUNCDETAIL_NORMAL 等の成功値である場合、対応する関数定義がシステムカタログ（pg_proc）に存在することが保証されます。

アルゴリズムまたは振る舞い

パーサーが関数解決を行う際、以下の状態を表現します：

  • FUNCDETAIL_NOTFOUND: 一致する関数なし。
  • FUNCDETAIL_MULTIPLE: 候補が複数あり、曖昧。
  • FUNCDETAIL_NORMAL: 通常の関数として解決。
  • FUNCDETAIL_PROCEDURE: プロシージャとして解決。
  • FUNCDETAIL_AGGREGATE: 集約関数として解決。
  • FUNCDETAIL_WINDOWFUNC: ウィンドウ関数として解決。
  • FUNCDETAIL_COERCION: 型変換要求として解決。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/catalog/pg_aggregate.c
  • src/backend/parser/parse_func.c
  • src/backend/parser/parse_oper.c
  • src/backend/utils/adt/ruleutils.c
  • **呼び出し先**:
  • 列挙型であるため、呼び出し先はありません。

関連データ構造

  • pg_proc: 関数定義が格納されるシステムカタログ。
  • ParseFuncOrColumn: 本列挙型を利用して結果を判定するメインエントリーポイント。

失敗モードと境界条件

  • FUNCDETAIL_NOTFOUND: 適切な関数が見つからない場合に発生します。
  • FUNCDETAIL_MULTIPLE: 引数の型適合性が同等で、最適な関数を一つに絞り込めない場合に発生します。

保守担当者向けメモ

  • 新しい関数タイプ（例: 新規のルーチン種別）が追加された場合は、本列挙型に値を追加し、各パーサー処理箇所でハンドリングを行う必要があります。
