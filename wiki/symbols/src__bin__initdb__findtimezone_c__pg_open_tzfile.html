pg_open_tzfile

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/findtimezone.c:64

目的

PostgreSQLのタイムゾーンデータベースディレクトリから、指定された名前のタイムゾーンファイルを読み取り専用で開くための低レベル関数です。

シグネチャまたは形状

int pg_open_tzfile(const char *name, char *canonname);

入力と出力

  • **入力**:
  • name: 開くタイムゾーンファイル名（相対パス）。
  • canonname: 呼び出し元が提供するバッファ。成功時に name がコピーされます（NULL の場合はスキップ）。
  • **出力**:
  • 成功時: ファイル記述子（int）。
  • 失敗時: -1。

前提条件と不変条件

  • pg_TZDIR() が有効なタイムゾーンディレクトリパスを返すこと。
  • name と pg_TZDIR() の結合パスが MAXPGPATH を超えないこと。

アルゴリズムまたは振る舞い

1. canonname が指定されていれば、name を canonname にコピーする。 2. pg_TZDIR() を取得し、fullname バッファに格納する。 3. fullname にディレクトリ区切り文字 / と name を連結する。 4. パス長が MAXPGPATH を超える場合は -1 を返す。 5. open() システムコールを O_RDONLY | PG_BINARY フラグで呼び出し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/timezone/localtime.c
  • src/timezone/pgtz.c
  • **呼び出し先**:
  • pg_TZDIR() (マクロまたは関数)
  • open() (システムコール)

関連データ構造

  • MAXPGPATH: パス名の最大長を規定する定数。
  • TZ_STRLEN_MAX: タイムゾーン文字列の最大長を規定する定数。

失敗モードと境界条件

  • パス長が MAXPGPATH を超える場合、ファイルを開かずに -1 を返す。
  • ファイルが存在しない、またはアクセス権限がない場合、open() の結果（-1）をそのまま返す。
  • errno は open() の結果に依存するが、本関数自体は errno を明示的に設定しない。

保守担当者向けメモ

  • 本関数は src/timezone/ 内のコードからも利用される共有ユーティリティです。
  • PG_BINARY フラグを使用しているため、Windows環境でのバイナリモードオープンが保証されています。
  • 呼び出し元によっては、本関数が失敗した際に errno が設定されない可能性があることを考慮する必要があります。
