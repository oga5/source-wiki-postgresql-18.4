help

定義

  • 種別: function
  • 定義箇所: src/bin/pg_config/pg_config.c:70

目的

pg_config コマンドの利用方法、サポートされているオプション一覧、およびプロジェクトの連絡先情報を標準出力に表示する。

シグネチャまたは形状

static void help(void)

入力と出力

  • 入力: なし（グローバル変数 progname および PACKAGE_BUGREPORT 等の定数を使用）
  • 出力: 標準出力（stdout）へのテキスト表示

前提条件と不変条件

  • progname が適切に初期化されていること。
  • info_items テーブルと本関数内の出力内容が同期していること（新しいオプション追加時は両方の修正が必要）。

アルゴリズムまたは振る舞い

1. プログラム名を含む使用方法の概要を表示する。 2. サポートされている全オプション（ディレクトリパスやビルド時のコンパイルフラグなど）の一覧を列挙する。 3. 引数なしで実行した場合の挙動について補足する。 4. バグ報告先およびプロジェクトのホームページ URL を表示する。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（引数解析時に --help または -? が指定された場合）。
  • 呼び出し先: printf（標準ライブラリ）。
  • 注記: ツールが提示した contrib/oid2name/oid2name.c 等の呼び出し元情報は、本シンボル（src/bin/pg_config/pg_config.c 内の help）とは異なる同名の別関数を指している可能性が高く、本シンボルの直接の呼び出し元とは断定できない。

関連データ構造

  • info_items[]: 本関数で表示されるオプション定義と対応する内部設定項目を保持するテーブル。

失敗モードと境界条件

  • 標準出力への書き込みが失敗した場合、printf の挙動に依存する。
  • 本関数自体は終了処理を行わないため、呼び出し側で適切な終了処理（exit 等）を行う必要がある。

保守担当者向けメモ

  • 本プログラムはシェルスクリプトからの移植版である。
  • オプションを追加する際は、info_items テーブルの定義と本関数の出力内容の整合性を維持すること。
  • リロケーション対応のため、パス情報はハードコードされず実行時の実行ファイル位置から算出される点に注意。
