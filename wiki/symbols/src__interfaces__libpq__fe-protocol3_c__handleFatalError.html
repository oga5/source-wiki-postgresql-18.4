handleFatalError

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:486

目的

回復不可能なエラーが発生した際に、接続状態をクリーンアップし、エラー結果を保存して接続を終了させる。

シグネチャまたは形状

static void handleFatalError(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（PostgreSQL 接続オブジェクトへのポインタ）
  • **出力**: なし（副作用として conn の状態を更新する）

前提条件と不変条件

  • conn は有効な PGconn オブジェクトを指している必要がある。
  • 本関数は、プロトコル 3.0 における致命的なエラー発生時に呼び出されることを前提としている。

アルゴリズムまたは振る舞い

1. pqSaveErrorResult(conn) を呼び出し、現在のエラーメッセージを PGresult として保存する。 2. conn->asyncStatus を PGASYNC_READY に設定し、PQgetResult の待機ループから脱出させる。 3. pqDropConnection(conn, true) を呼び出し、入力バッファを破棄して接続をクローズする。 4. conn->status を CONNECTION_BAD に更新し、バックエンドとの接続が失われたことを示す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c 内の各メッセージ解析関数（pqParseInput3 等）から、プロトコル違反や致命的なエラー発生時に呼び出される。
  • **呼び出し先**:
  • pqSaveErrorResult
  • pqDropConnection

関連データ構造

  • PGconn: 接続状態を保持する構造体。asyncStatus および status フィールドが本関数によって変更される。

失敗モードと境界条件

  • 本関数自体はエラーを返さないが、pqDropConnection の実行により、以降の通信操作は失敗するようになる。

保守担当者向けメモ

  • 本関数はプロトコル 3.0 専用のクリーンアップ処理である。
  • 接続を強制終了するため、呼び出し後は呼び出し元で速やかに処理を中断する必要がある。
