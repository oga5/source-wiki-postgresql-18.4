PgStatShared_HashEntry

定義

  • 種別: struct
  • 定義箇所: src/include/utils/pgstat_internal.h:65

目的

PostgreSQLの累積統計システムにおいて、共有メモリ上の統計データエントリを管理するためのハッシュテーブルエントリ構造体です。統計データの生存期間管理と、バックエンド間での安全な共有アクセスを提供します。

シグネチャまたは形状

typedef struct PgStatShared_HashEntry
{
    PgStat_HashKey key;             /* ハッシュキー */
    bool           dropped;         /* 削除フラグ */
    pg_atomic_uint32 refcount;      /* 参照カウンタ */
    pg_atomic_uint32 generation;    /* 再利用カウンタ */
    dsa_pointer    body;            /* 統計データ本体へのポインタ */
} PgStatShared_HashEntry;

入力と出力

  • **入力**: dshash（共有ハッシュテーブル）のキーとして PgStat_HashKey を保持します。
  • **出力**: body フィールドを通じて、統計種別ごとの具体的なデータ構造（PgStatShared_Common を含む）を指し示します。

前提条件と不変条件

  • refcount および generation はアトミック変数であり、操作には対象エントリを含む dshash パーティションの共有ロック以上が必要です。
  • dropped が true の場合、新たな参照を作成してはなりません。
  • refcount が 0 に達した時点で、エントリは解放対象となります。

アルゴリズムまたは振る舞い

  • **生存期間管理**: 統計エントリの生存期間は、ハッシュテーブルエントリ自体の生存期間とは独立しています。バックエンドがハッシュテーブルロックを保持せずに統計エントリを参照できるよう、refcount による管理が行われます。
  • **再利用**: pgstat_reinit_entry() が呼び出されるたびに generation がインクリメントされ、エントリの再利用回数が追跡されます。
  • **削除**: dropped フラグが設定されると、各バックエンドは参照を解放し、最終的にメモリが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c および pgstat_shmem.c 内の統計管理関数群が、この構造体を操作します。
  • **呼び出し先**: dshash（共有ハッシュテーブル）および dsa（動的共有メモリ）のインターフェースを利用してメモリ管理を行います。

関連データ構造

  • PgStat_HashKey: ハッシュテーブルの検索キー。
  • PgStatShared_Common: body が指す統計データの先頭に配置される共通構造体。
  • dshash: 本エントリを格納する共有ハッシュテーブル。

失敗モードと境界条件

  • refcount の不適切な管理（解放漏れ）は、共有メモリ上の統計エントリのリークを引き起こします。
  • dropped 状態のエントリに対して参照を維持し続けると、ガベージコレクションが阻害されます。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、ポインタには dsa_pointer を使用してください。
  • refcount は複数のバックエンドから同時に操作される可能性があるため、必ずアトミック操作関数を使用してください。
