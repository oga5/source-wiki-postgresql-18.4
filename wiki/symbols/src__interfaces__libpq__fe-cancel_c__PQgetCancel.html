PQgetCancel

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:367

目的

既存のデータベース接続 (PGconn) から、サーバーへのクエリキャンセル要求に必要な認証情報（PIDおよびキャンセルキー）を抽出し、PGcancel 構造体を生成する。

シグネチャまたは形状

PGcancel *PQgetCancel(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn（有効なデータベース接続ポインタ）
  • **出力**: 成功時は初期化された PGcancel 構造体へのポインタ、失敗時は NULL。

前提条件と不変条件

  • conn が NULL または conn->sock が PGINVALID_SOCKET の場合は NULL を返す。
  • サーバーからキャンセルキーを受信していない場合、全フィールドがゼロのダミー PGcancel オブジェクトを返す（このオブジェクトに対する PQcancel 呼び出しは失敗する）。
  • 成功時、返される PGcancel オブジェクトには、接続のネットワークアドレス、PID、キャンセルキー、および各種TCPパラメータがコピーされる。

アルゴリズムまたは振る舞い

1. 接続状態を確認し、無効な場合は NULL を返す。 2. キャンセルキーが未受信の場合、calloc でゼロ初期化された PGcancel を返して終了する。 3. キャンセルキーが存在する場合、必要なメモリ量を計算し malloc で領域を確保する。 4. 接続の raddr をコピーし、TCP関連パラメータ（keepalives 等）を pqParseIntParam を用いて解析・設定する。 5. CancelRequestPacket を構築し、ネットワークバイトオーダーに変換した backendPID、cancelAuthCode、およびパケット長を格納する。 6. 構築した PGcancel オブジェクトを返す。失敗時は確保したメモリを解放する。

  • memcpy
  • pg_hton32（ネットワークバイトオーダー変換）

関連データ構造

  • PGconn: 接続情報を保持する構造体。
  • PGcancel: キャンセル要求に必要な情報を保持する構造体。
  • CancelRequestPacket: サーバーへ送信するキャンセルパケットのレイアウト。

失敗モードと境界条件

  • malloc が失敗した場合、NULL を返す。
  • pqParseIntParam によるパラメータ解析に失敗した場合、確保したメモリを解放し NULL を返す。
  • サーバーがキャンセルキーを送信していない場合、ダミーオブジェクトを返すが、これは PQcancel 実行時に失敗する仕様である。

保守担当者向けメモ

  • 本関数は古いAPIであり、新規開発には PQcancelCreate を含む非同期API群の使用が推奨される。
  • 戻り値が NULL かどうかで「メモリ不足」と「キャンセル機能非対応」を区別できない仕様が残っている。
  • PGcancel オブジェクトのメモリ管理は呼び出し元に委ねられる。
