gettoken_query_standard

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery.c:285

目的

標準的な構文規則に従い、tsquery 文字列から次のトークン（演算子、オペランド、括弧、または終了）をスキャンして抽出する。

シグネチャまたは形状

ts_tokentype gettoken_query_standard(
    TSQueryParserState state, 
    int8 *operator, 
    int *lenval, 
    char **strval, 
    int16 *weight, 
    bool *prefix
)

入力と出力

  • **入力**:
  • state: 解析状態を保持する TSQueryParserState 構造体。
  • **出力**:
  • operator: 演算子（OP_NOT, OP_AND, OP_OR, OP_PHRASE）が格納される。
  • lenval, strval: オペランドの長さと文字列ポインタ。
  • weight, prefix: オペランドに付随する重みとプレフィックスフラグ。
  • 戻り値: トークン種別（PT_OPR, PT_VAL, PT_OPEN, PT_CLOSE, PT_END, PT_ERR）。

前提条件と不変条件

  • state->state が WAITFIRSTOPERAND または WAITOPERAND であること。
  • state->buf は解析対象の文字列を指していること。
  • 内部的に tsvector パーサー（state->valstate）を利用してオペランドを解析する。

アルゴリズムまたは振る舞い

1. state->state に基づき、現在の解析フェーズを分岐する。 2. **WAITFIRSTOPERAND / WAITOPERAND**:

  • !, (, : などの特殊文字を検出し、対応するトークンを返す。
  • それ以外の場合、gettoken_tsvector を呼び出してオペランドを抽出する。成功時は get_modifiers で修飾子を処理し、WAITOPERATOR へ遷移する。

    • 3. **WAITOPERATOR**:

  • &, |, PHRASE 演算子、または ) を検出し、対応するトークンを返す。
  • 演算子検出後は WAITOPERAND へ遷移する。

    • 4. 空白文字はスキップし、解析不能な文字は PT_ERR を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery.c 内で、TSQueryParserStateData の gettoken フィールドに代入され、パーサーから間接的に呼び出される。
  • **呼び出し先**:
  • gettoken_tsvector, get_modifiers, parse_phrase_operator, t_iseq, pg_mblen_cstr, ereturn。

関連データ構造

  • TSQueryParserStateData: 解析中の状態（バッファ、スタック、括弧の深さ等）を管理。
  • ts_tokentype: トークンの種類を定義する列挙型。

失敗モードと境界条件

  • **構文エラー**: 不正な文字や演算子の配置、括弧の不整合（count < 0）が発生した場合に PT_ERR を返す。
  • **ソフトエラー**: SOFT_ERROR_OCCURRED が真の場合、エラーコンテキストを介して PT_ERR を返す。
  • **終了条件**: 文字列終端に達した際、括弧の深さが 0 でなければ PT_ERR を返し、正常であれば PT_END を返す。

保守担当者向けメモ

  • 本関数は tsquery の標準的な構文解析を担当する。Web検索用やプレーンテキスト用のパーサーと並列して実装されている。
  • tsvector パーサーへの依存が強いため、オペランド解析の挙動を変更する場合は tsvector_parser.c の影響範囲を確認すること。
