pgstat_get_my_plan_id

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:1157

目的

現在のバックエンドプロセスに関連付けられたプランID（st_plan_id）を取得します。

シグネチャまたは形状

int64 pgstat_get_my_plan_id(void)

入力と出力

  • 入力: なし
  • 出力: 現在のバックエンドのプランID（int64）。バックエンドが初期化されていない場合は 0 を返します。

前提条件と不変条件

  • MyBEEntry が有効な共有メモリ上の PgBackendStatus 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. グローバル変数 MyBEEntry が NULL かどうかを確認し、NULL であれば 0 を返して終了します。 2. MyBEEntry が指す構造体から st_plan_id を読み取り、その値を返します。 3. 読み取り時にロックは取得されません。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execParallel.c:178 にて、pstmt->planId の設定に使用されています。
  • 呼び出し先: なし（直接的な関数呼び出しは含まれません）。

関連データ構造

  • PgBackendStatus: バックエンドの状態を保持する共有メモリ構造体。st_plan_id フィールドを含みます。
  • MyBEEntry: 現在のプロセスが使用する PgBackendStatus へのポインタ。

失敗モードと境界条件

  • MyBEEntry が初期化されていない（NULL）場合、関数は安全に 0 を返します。
  • 共有メモリ上の値に対するロックを行わないため、読み取り時に値が更新されている可能性がありますが、int64 の読み取りはアトミックであると想定されています。

保守担当者向けメモ

  • 本関数はロックを取得せずに st_plan_id を読み取ります。これは、他の統計情報読み取りと同様の楽観的な設計に基づいています。
