AppendPlainCommandOption

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/streamutil.c:745

目的

レプリケーションコマンドの文字列構築において、オプション名（値を持たないフラグ形式のオプション）をバッファに追加します。サーバーのバージョンに応じた構文（旧形式または括弧付きの新形式）を吸収する役割を持ちます。

シグネチャまたは形状

void AppendPlainCommandOption(PQExpBuffer buf, bool use_new_option_syntax, char *option_name)

入力と出力

  • **入力**:
  • buf: 構築中のコマンド文字列を保持する PQExpBuffer。
  • use_new_option_syntax: 新しい構文（括弧付き）を使用するかどうかを示すフラグ。
  • option_name: 追加するオプション名（文字列）。
  • **出力**: なし（buf の内容が更新されます）。

前提条件と不変条件

  • buf は初期化済みである必要があります。
  • use_new_option_syntax が true の場合、オプション間はカンマ , で区切られます。false の場合はスペース で区切られます。

アルゴリズムまたは振る舞い

1. バッファが空でなく、かつ末尾が ( でない場合、区切り文字を挿入します。

  • use_new_option_syntax が true なら ", " を追加。
  • false なら " " を追加。

    • 2. 指定された option_name を " %s" の形式でバッファに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c (複数箇所)
  • src/bin/pg_basebackup/streamutil.c (複数箇所)
  • **呼び出し先**:
  • appendPQExpBufferStr
  • appendPQExpBufferChar
  • appendPQExpBuffer

関連データ構造

  • PQExpBuffer: libpq が提供する動的文字列バッファ。

失敗モードと境界条件

  • buf がメモリ不足状態にある場合、libpq のバッファ操作関数内部でエラーが発生する可能性があります。
  • バッファの末尾が ( である場合、区切り文字は挿入されません。

保守担当者向けメモ

  • 本関数は、PostgreSQL 15以降で導入されたレプリケーションコマンドの新しいオプション構文に対応するための抽象化レイヤーです。新しいフラグオプションを追加する際は、本関数を利用することで構文の整合性が保たれます。
