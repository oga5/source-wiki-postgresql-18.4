pg_get_ruledef

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ruleutils.c:559

目的

指定されたルール（pg_rewrite カタログの OID）に対応する CREATE RULE 文の SQL 定義文字列を再構築して返却します。

シグネチャまたは形状

Datum pg_get_ruledef(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してルールオブジェクトの OID を受け取ります。
  • **出力**: 成功時はルール定義を表す text 型の Datum を返却します。ルールが見つからない場合は NULL を返却します。

前提条件と不変条件

  • PRETTYFLAG_INDENT が有効な状態で逆コンパイルが行われます。
  • 内部的に pg_get_ruledef_worker を呼び出し、カタログから定義を読み取ります。

アルゴリズムまたは振る舞い

1. 引数からルール OID を取得します。 2. インデント整形フラグ（PRETTYFLAG_INDENT）を初期化します。 3. pg_get_ruledef_worker を呼び出し、カタログ情報から SQL 文字列を生成します。 4. 結果が NULL の場合は PG_RETURN_NULL() を実行します。 5. 生成された文字列を string_to_text で text 型に変換し、PG_RETURN_TEXT_P を介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c: ルール定義のダンプ時に使用。
  • src/bin/psql/describe.c: psql の \d コマンド等でルール定義を表示する際に使用。
  • **呼び出し先**:
  • pg_get_ruledef_worker: 逆コンパイルの主要な内部処理を実行。
  • string_to_text: 文字列を PostgreSQL の text 型へ変換。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • PG_FUNCTION_ARGS: 関数呼び出し規約のためのマクロ。

失敗モードと境界条件

  • 指定された OID がシステムカタログに存在しない場合、pg_get_ruledef_worker が NULL を返し、本関数も NULL を返却します。

保守担当者向けメモ

  • 本関数はデフォルトでインデント付きの出力を生成します。インデントの有無を制御したい場合は、pg_get_ruledef_ext の利用を検討してください。
  • 逆コンパイルのロジックは ruleutils.c 内の pg_get_ruledef_worker に集約されているため、出力形式の変更が必要な場合はそちらを確認してください。
