Variable

定義

  • 種別: struct
  • 定義箇所: src/bin/pgbench/pgbench.c:324

目的

pgbench におけるスクリプト実行時の変数を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	char	   *name;			/* variable's name */
	char	   *svalue;			/* its value in string form, if known */
	PgBenchValue value;			/* actual variable's value */
} Variable;

入力と出力

  • 入力: 変数名（name）、文字列形式の値（svalue）、および PgBenchValue 型の内部値。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • name は変数の識別子として使用されます。
  • value フィールドは、pgbench のスクリプトエンジンで計算される実際の値を保持します。

アルゴリズムまたは振る舞い

本構造体は、pgbench 内で定義される変数の名前と値をペアとして管理します。svalue は必要に応じて文字列形式のキャッシュとして機能し、value は式評価エンジンが直接利用する型として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pgbench/pgbench.c 内の変数管理ロジック（vars 配列の要素として使用）。
  • 呼び出し先: なし（データ構造体のため）。

    • ※提供されたソース情報には、本構造体を直接操作する関数呼び出しの詳細は含まれていません。

関連データ構造

  • Variables: Variable の配列を管理する構造体（src/bin/pgbench/pgbench.c:336 に Variable *vars として定義）。
  • PgBenchValue: Variable 構造体内で値の保持に使用される型。

失敗モードと境界条件

  • メモリ割り当て: name や svalue は動的に割り当てられる可能性があるため、メモリ不足時のハンドリングが必要です。
  • 変数名の重複: pgbench の変数管理ロジックにおいて、同一名の変数が複数存在しないよう制御する必要があります。

保守担当者向けメモ

  • 本構造体は pgbench.c 内の変数管理の基盤です。変数の追加・検索・削除を行う関数群と併せて参照してください。
  • svalue と value の同期が保たれているか、あるいはどちらが優先されるかについては、変数更新ロジックを確認してください。
