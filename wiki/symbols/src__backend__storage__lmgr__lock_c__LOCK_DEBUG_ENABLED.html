LOCK_DEBUG_ENABLED

定義

  • 種別: inline static function
  • 定義箇所: src/backend/storage/lmgr/lock.c:358

目的

特定のロックタグ（LOCKTAG）に対してデバッグログ出力が有効であるかを判定します。

シグネチャまたは形状

inline static bool LOCK_DEBUG_ENABLED(const LOCKTAG *tag)

入力と出力

  • **入力**: const LOCKTAG *tag（判定対象のロックタグへのポインタ）
  • **出力**: bool（デバッグ出力が有効な場合は true、それ以外は false）

前提条件と不変条件

  • tag は有効な LOCKTAG 構造体を指している必要があります。
  • LockMethods 配列が初期化済みであり、tag->locktag_lockmethodid が有効なインデックスであることを前提としています。

アルゴリズムまたは振る舞い

以下のいずれかの条件を満たす場合に true を返します。 1. ロックメソッド固有のトレースフラグが有効であり、かつタグの locktag_field2 が設定された最小OID（Trace_lock_oidmin）以上である場合。 2. グローバルな Trace_lock_table が設定されており、かつタグの locktag_field2 がその値と一致する場合。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の箇所から呼び出されています（src/backend/storage/lmgr/lock.c 内）。
  • 372行目, 396行目, 713行目, 874行目, 2088行目
  • **呼び出し先**:
  • LockMethods 配列へのアクセス。
  • グローバル変数 Trace_lock_oidmin および Trace_lock_table への参照。

関連データ構造

  • LOCKTAG: ロック対象を識別する構造体。
  • LockMethods: ロックメソッドごとの属性を保持する配列。

失敗モードと境界条件

  • tag が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • tag->locktag_lockmethodid が範囲外の場合、LockMethods 配列へのアクセスで未定義の挙動を引き起こします。

保守担当者向けメモ

  • 本関数は LOCK_DEBUG が定義された環境でのデバッグログ出力を制御するために使用されます。
  • 動作を変更する場合は、Trace_lock_oidmin や Trace_lock_table といったグローバルなトレース設定との整合性に注意してください。
