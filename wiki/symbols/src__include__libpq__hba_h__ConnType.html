ConnType

定義

  • 種別: enum
  • 定義箇所: src/include/libpq/hba.h:58

目的

PostgreSQL のホストベース認証（HBA）設定において、クライアントの接続タイプを識別するために使用されます。

シグネチャまたは形状

typedef enum ConnType
{
	ctLocal,        /* Unixドメインソケット接続 */
	ctHost,         /* TCP/IP接続（SSL/GSSの有無を問わない） */
	ctHostSSL,      /* SSLを使用したTCP/IP接続 */
	ctHostNoSSL,    /* SSLを使用しないTCP/IP接続 */
	ctHostGSS,      /* GSSAPIを使用したTCP/IP接続 */
	ctHostNoGSS,    /* GSSAPIを使用しないTCP/IP接続 */
} ConnType;

入力と出力

  • 本列挙型は、pg_hba.conf の解析結果を保持するデータ構造の一部として使用されます。
  • 関数への入力として、特定の接続がどのタイプに該当するかを判定・指定するために利用されます。

前提条件と不変条件

  • pg_hba.conf の各行（HbaLine）は、必ず一つの ConnType を保持します。

アルゴリズムまたは振る舞い

  • 接続試行時、hba_getauthmethod() が現在の接続情報を評価し、設定ファイル内のルールと照合する際の条件分岐に使用されます。
  • SSL や GSSAPI の要件に基づき、接続が許可されるか、あるいは認証方式が適用されるかを決定する判断基準となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/libpq/hba.h:100 の HbaLine 構造体内でメンバ変数 conntype として参照されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数呼び出しの詳細は特定できません。

関連データ構造

  • HbaLine: pg_hba.conf の1行を表現する構造体であり、この中で ConnType が使用されます。

失敗モードと境界条件

  • 設定ファイル内で未定義の接続タイプが指定された場合、パース処理（parse_hba_line）においてエラーが発生する可能性があります。

保守担当者向けメモ

  • 新しい接続タイプを追加する場合は、hba.c 内のパースロジックおよび pg_hba.conf の構文解析処理を適宜更新してください。
