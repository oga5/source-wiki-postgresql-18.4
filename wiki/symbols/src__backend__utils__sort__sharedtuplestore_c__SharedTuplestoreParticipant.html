SharedTuplestoreParticipant

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/sharedtuplestore.c:50

目的

並列クエリ実行において、各バックエンドが書き込む一時ファイルの状態（読み取り位置や書き込み済みページ数）を個別に管理するための構造体です。

シグネチャまたは形状

typedef struct SharedTuplestoreParticipant
{
    LWLock      lock;           /* 読み取り位置更新用の排他制御 */
    BlockNumber read_page;      /* 次に読み取るページ番号 */
    BlockNumber npages;         /* 書き込み済みの総ページ数 */
    bool        writing;        /* アサーション用フラグ */
} SharedTuplestoreParticipant;

入力と出力

本構造体は SharedTuplestore 内の配列要素として保持されます。直接的な関数引数や戻り値としての利用ではなく、並列スキャン時に各参加者の状態を追跡するために使用されます。

前提条件と不変条件

  • 書き込み操作は各バックエンドが専用ファイルに対して行うため、書き込み時のロックは不要です。
  • lock は、並列スキャン時に read_page を更新する際の競合を防ぐために使用されます。

アルゴリズムまたは振る舞い

  • 並列スキャン時、各バックエンドは sts_parallel_scan_next を通じてこの構造体の read_page を参照・更新します。
  • 読み取り側は lock を取得し、現在の read_page が npages に達していないことを確認した上で、チャンクを読み出し、read_page をインクリメントします。

呼び出し元と呼び出し先

  • 呼び出し元: SharedTuplestore 構造体（participants 配列として保持）。
  • 呼び出し先: 構造体内部で LWLock を使用しており、sts_parallel_scan_next 等の関数から参照・操作されます。

関連データ構造

  • SharedTuplestore: 本構造体を配列として保持する親構造体。
  • SharedTuplestoreAccessor: 各バックエンドが操作時に使用するローカルな構造体。

失敗モードと境界条件

  • read_page が npages に到達した場合、その参加者のファイルからの読み取りは完了とみなされます。
  • 巨大なタプルがチャンクを跨ぐ場合、SharedTuplestoreChunk のヘッダー情報と組み合わせて読み取り位置を制御する必要があります。

保守担当者向けメモ

  • 本構造体は SharedTuplestore の一部として共有メモリ上に配置されます。
  • writing フラグはデバッグおよびアサーション専用であり、実行時のロジックには影響しません。
