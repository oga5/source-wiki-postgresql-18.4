readdir

定義

  • 種別: function
  • 定義箇所: src/port/dirent.c:77

目的

Windows環境において、ディレクトリ内の次のファイルエントリを読み取り、POSIX準拠の struct dirent 形式で提供する。

シグネチャまたは形状

struct dirent *readdir(DIR *d);

入力と出力

  • **入力**: DIR *d （ディレクトリ走査の状態を保持する構造体へのポインタ）
  • **出力**: 成功時は struct dirent へのポインタ、失敗または終端到達時は NULL を返す。

前提条件と不変条件

  • d は opendir によって初期化済みであること。
  • d->ret.d_ino および d->ret.d_reclen はWindows APIの制限により常に 0 に固定される。
  • d_name は MAX_PATH 長まで格納可能。

アルゴリズムまたは振る舞い

1. **初回呼び出し**: d->handle が INVALID_HANDLE_VALUE の場合、FindFirstFile を実行して検索を開始する。 2. **2回目以降**: FindNextFile を呼び出し、次のエントリを取得する。 3. **データ変換**: 取得した WIN32_FIND_DATA からファイル名と長さを d->ret にコピーする。 4. **ファイルタイプ判定**:

  • FILE_ATTRIBUTE_REPARSE_POINT かつ IO_REPARSE_TAG_MOUNT_POINT の場合: DT_LNK
  • FILE_ATTRIBUTE_DIRECTORY の場合: DT_DIR
  • それ以外: DT_REG

    • 5. **終了処理**: 成功時は &d->ret を返し、失敗時は errno を適切に設定（ファイルがない場合は 0）して NULL を返す。

関連データ構造

  • DIR: ディレクトリ走査の状態を管理する構造体。
  • struct dirent: POSIX互換のディレクトリ情報格納構造体。
  • WIN32_FIND_DATA: Windowsネイティブのファイル情報構造体。

失敗モードと境界条件

  • ディレクトリ内にファイルが存在しない場合、errno を 0 に設定して NULL を返す。
  • その他のエラー発生時は _dosmaperr を介して errno を設定し NULL を返す。

保守担当者向けメモ

  • 本関数はWindows専用の実装である。
  • リパースポイント（シンボリックリンクやジャンクション）の判定ロジックが含まれており、Windows特有のファイルシステム構造に依存している。
