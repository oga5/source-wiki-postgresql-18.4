oid_decrement

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:477

目的

B-tree インデックスのスキップスキャンにおいて、指定された Oid 型の値を 1 減算する処理を提供します。

シグネチャまたは形状

static Datum oid_decrement(Relation rel, Datum existing, bool *underflow)

入力と出力

  • **入力**:
  • rel: 対象となるリレーション（本関数内では未使用）。
  • existing: 減算対象の Oid 値（Datum 型）。
  • underflow: 減算結果がアンダーフローしたかを示すフラグへのポインタ。
  • **出力**:
  • 戻り値: 減算後の Oid 値（Datum 型）。
  • underflow: existing が InvalidOid の場合は true、それ以外は false が設定されます。

前提条件と不変条件

  • existing は Oid 型として解釈可能である必要があります。
  • InvalidOid を減算しようとした場合、結果は未定義となり、underflow が true に設定されます。

アルゴリズムまたは振る舞い

1. DatumGetObjectId を使用して existing を Oid に変換します。 2. 値が InvalidOid であるかを確認します。

  • InvalidOid の場合、underflow を true に設定し、0 を返します。

    • 3. それ以外の場合、underflow を false に設定し、値を 1 減算した結果を ObjectIdGetDatum で変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内の SkipSupport 構造体初期化処理（sksup->decrement = oid_decrement;）にて参照されています。
  • **呼び出し先**: DatumGetObjectId および ObjectIdGetDatum マクロを使用しています。

関連データ構造

  • SkipSupport: インデックススキップスキャン用の増減ロジックを保持する構造体。

失敗モードと境界条件

  • InvalidOid を入力した場合、減算は行われず underflow フラグが立てられます。

保守担当者向けメモ

  • 本関数は SkipSupport インターフェースの一部として実装されています。
  • rel 引数は現在の実装では使用されていませんが、将来的な拡張やインターフェースの整合性のために保持されています。
