CopyMultiInsertInfoSetupBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfrom.c:379

目的

COPY コマンド実行時におけるバルク挿入用のバッファを初期化し、指定されたリレーション情報（ResultRelInfo）と管理構造体（CopyMultiInsertInfo）を関連付けます。

シグネチャまたは形状

static inline void CopyMultiInsertInfoSetupBuffer(CopyMultiInsertInfo *miinfo, ResultRelInfo *rri)

入力と出力

  • **入力**:
  • miinfo: バルク挿入の状態を保持する CopyMultiInsertInfo 構造体へのポインタ。
  • rri: 挿入対象テーブルの情報を保持する ResultRelInfo 構造体へのポインタ。
  • **出力**: なし（副作用として rri および miinfo の内部状態を更新）。

前提条件と不変条件

  • rri は有効な ResultRelInfo を指している必要があります。
  • miinfo は初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. CopyMultiInsertBufferInit(rri) を呼び出し、新しいバルク挿入バッファを作成します。 2. 作成したバッファへのポインタを rri->ri_CopyMultiInsertBuffer に格納し、相互参照を確立します。 3. miinfo->multiInsertBuffers リストに作成したバッファを追加し、追跡対象として登録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c 内の複数箇所（行 380, 418, 1242 付近）から呼び出されます。
  • **呼び出し先**:
  • CopyMultiInsertBufferInit

関連データ構造

  • CopyMultiInsertInfo: バルク挿入全体の管理構造体。
  • ResultRelInfo: 挿入先リレーションのメタデータ。
  • CopyMultiInsertBuffer: 個別のテーブルに対するバルク挿入用バッファ。

失敗モードと境界条件

  • メモリ不足時には CopyMultiInsertBufferInit 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、COPY 処理のパフォーマンスに直結します。
  • rri と miinfo のリンク関係を維持することが重要であり、バッファのライフサイクル管理は CopyMultiInsertInfo のリスト構造に依存しています。
