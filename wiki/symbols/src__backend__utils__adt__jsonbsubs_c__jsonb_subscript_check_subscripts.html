jsonb_subscript_check_subscripts

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonbsubs.c:174

目的

jsonb 型の添字操作において、実行時に提供された添字（インデックス）を検証し、後続の取得（fetch）や代入（assign）処理で利用可能な形式（text 型のパス）へ正規化する。

シグネチャまたは形状

static bool jsonb_subscript_check_subscripts(ExprState *state, ExprEvalStep *op, ExprContext *econtext);

入力と出力

  • **入力**:
  • state: 式評価の状態管理構造体。
  • op: 現在の式評価ステップ。sbsref_subscript.state から SubscriptingRefState を取得する。
  • econtext: 式の評価コンテキスト。
  • **出力**:
  • 戻り値: 処理が正常に完了した場合は true。添字が NULL であり、かつ代入操作ではない場合は false を返し、op->resnull を true に設定する。

前提条件と不変条件

  • sbsrefstate->workspace は JsonbSubWorkspace 型としてキャスト可能であること。
  • 添字が INT4OID 型である場合、それは jsonb の配列インデックスとして扱われ、内部的に text 型のパスへ変換される。
  • 代入操作（isassignment）において添字が NULL の場合はエラー（ERRCODE_NULL_VALUE_NOT_ALLOWED）が発生する。

アルゴリズムまたは振る舞い

1. **配列期待値の判定**: 最初の添字が整数型の場合、workspace->expectArray を true に設定する。 2. **添字のループ処理**: すべての添字（numupper）を順次走査する。 3. **NULL チェック**: 添字が NULL の場合、代入操作であればエラーを投げ、参照操作であれば結果を NULL として終了する。 4. **型変換**: 添字が INT4OID（整数）である場合、int4out 関数を介して文字列に変換し、さらに CStringGetTextDatum で text 型に変換して workspace->index に格納する。それ以外の場合はそのまま格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonbsubs.c:390（jsonb_subscript_handler の sbs_check_subscripts フィールドに登録）。
  • **呼び出し先**:
  • DirectFunctionCall1（int4out を呼び出し）。
  • ereport（エラー発生時）。

関連データ構造

  • SubscriptingRefState: 添字操作の実行時状態を保持。
  • JsonbSubWorkspace: 添字の型情報や変換後のパスを保持するワークスペース。
  • ExprEvalStep: 式評価のステップ情報。

失敗モードと境界条件

  • **NULL 添字**: 代入操作時に添字が NULL の場合、ERRCODE_NULL_VALUE_NOT_ALLOWED エラーが発生する。
  • **型変換失敗**: 内部的な int4out 呼び出しが失敗した場合、PostgreSQL の標準的なエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数はスライス操作をサポートしていない。
  • workspace->index に格納される値は、後続の jsonb_subscript_fetch や jsonb_subscript_assign でパスとして利用されるため、型変換の整合性に注意すること。
