InitializeLatchWaitSet

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/latch.c:34

目的

WaitLatch() が使用するプロセス固有の WaitEventSet を初期化し、ラッチ待機およびポストマスター終了監視のためのイベントを登録します。

シグネチャまたは形状

void InitializeLatchWaitSet(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 LatchWaitSet を初期化）

前提条件と不変条件

  • LatchWaitSet が NULL であること（二重初期化の禁止）。
  • MyLatch が適切に設定されていること。

アルゴリズムまたは振る舞い

1. CreateWaitEventSet を呼び出し、最大2つのイベントを保持可能な LatchWaitSet を作成します。 2. AddWaitEventToSet を使用して、MyLatch を WL_LATCH_SET イベントとして登録します。 3. プロセスがポストマスター配下（IsUnderPostmaster が真）で動作している場合、WL_EXIT_ON_PM_DEATH イベントを追加登録し、ポストマスターの終了を監視可能にします。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/init/miscinit.c:132
  • src/backend/utils/init/miscinit.c:193
  • 呼び出し先:
  • CreateWaitEventSet
  • AddWaitEventToSet

関連データ構造

  • LatchWaitSet: 待機イベントを管理するグローバルな WaitEventSet。
  • MyLatch: 現在のプロセスに関連付けられたラッチ。
  • LatchWaitSetLatchPos, LatchWaitSetPostmasterDeathPos: イベントセット内の位置を保持する定数。

失敗モードと境界条件

  • LatchWaitSet が既に初期化されている場合、Assert により停止します。
  • AddWaitEventToSet が失敗した場合、内部でエラーが発生します。

保守担当者向けメモ

  • 本関数はプロセス初期化時に一度だけ呼び出されることを想定しています。
  • WaitLatch は本関数で設定されたイベントセットを再利用するため、イベントの追加順序や位置（latch_pos）は latch.c 内の他の関数と整合している必要があります。
