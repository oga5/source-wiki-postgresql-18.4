ginMergeItemPointers

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginpostinglist.c:377

目的

2つのソート済み ItemPointer 配列をマージし、重複を排除した新しい配列を生成する。

シグネチャまたは形状

ItemPointer ginMergeItemPointers(ItemPointerData *a, uint32 na,
                                 ItemPointerData *b, uint32 nb,
                                 int *nmerged)

入力と出力

  • **入力**:
  • a, b: マージ対象のソート済み ItemPointerData 配列。
  • na, nb: それぞれの配列の要素数。
  • **出力**:
  • 戻り値: マージおよび重複排除後の ItemPointerData 配列（pallocによりメモリ確保）。
  • nmerged: マージ後の要素数が格納されるポインタ。

前提条件と不変条件

  • 入力配列 a および b は、ginCompareItemPointers の順序に従ってソートされている必要がある。
  • 戻り値の配列は呼び出し元で pfree する必要がある。

アルゴリズムまたは振る舞い

1. (na + nb) 分のメモリを確保する。 2. 配列が重ならない場合（一方が他方より小さい場合）、memcpy を用いて連結する。 3. 重なる可能性がある場合、マージソートの要領で走査する：

  • ginCompareItemPointers を用いて比較し、小さい方を結果配列に追加する。
  • 値が等しい場合は重複とみなし、片方のみを結果に追加し、両方のポインタを進める。

    • 4. 片方の配列を走査し終えたら、残りの要素を結果配列にコピーする。 5. 最終的な要素数を nmerged に設定して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/gin/ginxlog.c
  • **呼び出し先**:
  • ginCompareItemPointers (比較用)
  • palloc (メモリ確保用)

関連データ構造

  • ItemPointerData: GINインデックスで管理されるTID（Tuple Identifier）。
  • GinPostingList: 本関数が扱うTIDリストの格納形式に関連する構造体。

失敗モードと境界条件

  • na または nb が 0 の場合、もう一方の配列をそのまま返す（連結処理が適用される）。
  • メモリ不足時は palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数はソート済みであることを前提としているため、入力配列の順序が崩れていると正しくマージされない。
  • ginCompareItemPointers の仕様変更が本関数の動作に直接影響する。
