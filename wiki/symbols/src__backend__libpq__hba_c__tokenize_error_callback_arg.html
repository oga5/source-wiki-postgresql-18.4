tokenize_error_callback_arg

定義

  • 種別: struct
  • 定義箇所: src/backend/libpq/hba.c:63

目的

設定ファイル（pg_hba.conf 等）の解析中にエラーが発生した際、エラーコンテキスト（ファイル名と行番号）をエラー報告関数へ伝達するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	const char *filename;
	int			linenum;
} tokenize_error_callback_arg;

入力と出力

  • **入力**: filename（解析対象のファイルパス）、linenum（エラー発生時の行番号）。
  • **出力**: なし（構造体として状態を保持）。

前提条件と不変条件

  • filename は解析対象のファイル名を示す有効な文字列ポインタである必要があります。
  • linenum は解析中の現在の行位置を正確に反映している必要があります。

アルゴリズムまたは振る舞い

解析処理において、エラー発生時に errcontext を設定するコールバック関数へこの構造体を渡すことで、ユーザーに対して「どのファイルの何行目でエラーが発生したか」という詳細な位置情報を提供します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c 内の解析処理（tokenize_auth_file 等）において、エラー報告用の引数としてインスタンス化されます。
  • **呼び出し先**: エラー報告関数（errcontext を設定する関数）が、この構造体のメンバを参照してエラーメッセージを構築します。

関連データ構造

  • HbaLine: 設定ファイルの各行を保持する構造体。
  • IdentLine: 識別マッピングファイルの各行を保持する構造体。

失敗モードと境界条件

  • filename が NULL の場合や、linenum が不適切な値（負の値など）である場合、エラー報告時に不正確な情報が出力される可能性があります。

保守担当者向けメモ

  • 本構造体は hba.c 内でのエラー報告の標準的な手段です。新しい解析関数を追加する際は、この構造体を用いてエラーコンテキストを適切に管理してください。
