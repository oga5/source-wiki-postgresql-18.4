ProcessingMode

定義

  • 種別: enum
  • 定義箇所: src/include/miscadmin.h:468

目的

PostgreSQL バックエンドプロセスの現在の実行フェーズを識別し、システム全体の動作状態を管理するために使用されます。

シグネチャまたは形状

typedef enum ProcessingMode
{
    BootstrapProcessing, /* テンプレートデータベースのブートストラップ作成時 */
    InitProcessing,      /* システム初期化時 */
    NormalProcessing     /* 通常処理時 */
} ProcessingMode;

入力と出力

  • 本シンボル自体は状態を示す列挙型です。
  • 外部変数 extern PGDLLIMPORT ProcessingMode Mode; を通じて、システム全体で現在のモードが参照・更新されます。

前提条件と不変条件

  • プロセス起動時に適切なモードへ設定される必要があります。
  • 実行フェーズの遷移は、システムの初期化フローに従って行われます。

アルゴリズムまたは振る舞い

  • システムの各サブシステムは、Mode 変数の値を参照することで、現在のフェーズに応じた制限や動作の切り替えを行います。
  • BootstrapProcessing はデータベース作成の初期段階、InitProcessing はプロセス起動時の準備段階、NormalProcessing は通常のクエリ処理段階を示します。

呼び出し元と呼び出し先

  • 呼び出し元（参照・設定箇所）:
  • src/backend/utils/init/miscinit.c:62: Mode 変数の初期化（InitProcessing への設定）。
  • 呼び出し先:
  • 提供されたソース情報からは、特定の関数呼び出し先は特定できません。

関連データ構造

  • BackendType: プロセスの役割を識別する列挙型（miscadmin.h で定義）。

失敗モードと境界条件

  • 適切なモード設定が行われない場合、初期化処理やクエリ実行において予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • Mode 変数は PGDLLIMPORT として宣言されており、globals.c 等で実体が定義されています。
  • 状態遷移のロジックを変更する際は、miscinit.c における初期化フローへの影響を考慮してください。
