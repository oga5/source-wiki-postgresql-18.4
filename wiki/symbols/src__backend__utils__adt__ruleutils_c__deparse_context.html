deparse_context

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/ruleutils.c:112

目的

クエリツリーや式ツリーを SQL 文字列へ逆コンパイル（deparse）する際、現在の処理状態、出力バッファ、名前空間、および整形設定を保持するために使用されるコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    StringInfo  buf;
    List       *namespaces;
    TupleDesc   resultDesc;
    List       *targetList;
    List       *windowClause;
    int         prettyFlags;
    int         wrapColumn;
    int         indentLevel;
    bool        varprefix;
    bool        colNamesVisible;
    bool        inGroupBy;
    bool        varInOrderBy;
    Bitmapset  *appendparents;
} deparse_context;

入力と出力

  • **入力**: 逆コンパイル処理の開始時に、初期化された StringInfo バッファや関連するクエリ情報と共に生成されます。
  • **出力**: 逆コンパイルの過程で buf メンバに SQL 文字列が逐次追記されます。

前提条件と不変条件

  • buf は有効な StringInfo を指している必要があります。
  • prettyFlags や indentLevel は、整形出力の有効・無効および現在のインデント状態を正しく反映している必要があります。

アルゴリズムまたは振る舞い

逆コンパイル関数（get_rule_expr 等）に引数として渡され、以下の制御を行います。

  • **状態保持**: 現在のクエリレベルにおける targetList や windowClause を参照し、式ツリーの解決を支援します。
  • **整形制御**: prettyFlags に基づき、SQL 生成時の改行やインデントを制御します。
  • **名前解決**: namespaces リストを通じて、RTE（RangeTblEntry）のエイリアスや列名の競合解決を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: ruleutils.c 内の各種逆コンパイル関数（get_select_query_def, get_insert_query_def, get_update_query_def, get_delete_query_def, get_merge_query_def, get_values_def, get_with_clause など）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • StringInfo: 出力文字列を保持するバッファ。
  • deparse_namespace: クエリ内の名前空間情報を保持するノード。
  • List: クエリレベルの構成要素（ターゲットリスト、ウィンドウ句など）を保持。
  • Bitmapset: appendparents として使用され、親リレーションの識別に使用。

失敗モードと境界条件

  • buf が未初期化の場合、文字列追記時にセグメンテーション違反が発生する可能性があります。
  • wrapColumn が -1 の場合、行の折り返し制限は無効となります。

保守担当者向けメモ

  • 本構造体は ruleutils.c 内の再帰的な逆コンパイル処理において中心的な役割を果たします。
  • 複雑なクエリの可読性を維持するため、prettyFlags や indentLevel の変更が後続の再帰呼び出しに与える影響に注意してください。
  • appendparents は UNION ALL などの継承関係を解決するために使用されます。
