TSAnyCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_cache.h:23

目的

PostgreSQLの全文検索（Tsearch）サブシステムにおける、キャッシュエントリの共通ヘッダー構造体です。各キャッシュエントリの先頭に配置することで、汎用的なキャッシュ管理を可能にします。

シグネチャまたは形状

typedef struct TSAnyCacheEntry
{
	Oid			objId;
	bool		isvalid;
} TSAnyCacheEntry;

入力と出力

  • **入力**: キャッシュ対象オブジェクトの Oid。
  • **出力**: キャッシュの有効性を示す isvalid フラグ。

前提条件と不変条件

  • 各キャッシュ構造体（TSParserCacheEntry 等）の最初のメンバとして配置される必要があります。
  • objId はハッシュテーブルのキーとして機能します。

アルゴリズムまたは振る舞い

  • **有効性管理**: システムカタログの更新を検知した際、InvalidateTSCacheCallBack によって関連するキャッシュエントリの isvalid が false に設定されます。
  • **汎用アクセス**: キャッシュ検索関数は、特定のキャッシュエントリを TSAnyCacheEntry* にキャストすることで、型に依存せずに有効性を確認したり、ハッシュテーブルを走査したりします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/ts_cache.c 内のキャッシュ走査ロジック（hash_seq_search を使用したループ処理など）から参照されます。
  • **呼び出し先**: 情報なし。

関連データ構造

  • TSParserCacheEntry
  • TSDictionaryCacheEntry
  • TSConfigCacheEntry
  • fmgr.h (関連する関数呼び出しインターフェース)

失敗モードと境界条件

  • isvalid が false の場合、キャッシュミスとして扱われ、カタログから情報を再読み込みする必要があります。
  • キャッシュ再構築時にメモリリークが発生しないよう、各キャッシュエントリが保持するメモリコンテキストの管理に注意が必要です。

保守担当者向けメモ

  • 本構造体は継承の基底として機能するため、メンバの順序変更はキャッシュ管理ロジック全体に影響を及ぼします。
  • キャッシュエントリはバックエンドの生存期間中保持されるため、動的メモリ確保を行う際は適切なメモリコンテキストを選択してください。
