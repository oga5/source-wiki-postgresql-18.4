printatt

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/printtup.c:422

目的

タプルの各属性（カラム）に関するメタデータおよび値を、デバッグ目的で標準出力へ整形表示する。

シグネチャまたは形状

static void printatt(unsigned attributeId, Form_pg_attribute attributeP, char *value)

入力と出力

  • **入力**:
  • attributeId: 属性のインデックス番号。
  • attributeP: 属性定義を保持する Form_pg_attribute 構造体へのポインタ。
  • value: 表示する属性値の文字列（NULL の場合は値の表示を省略）。
  • **出力**: なし（標準出力への printf による副作用のみ）。

前提条件と不変条件

  • attributeP は有効な Form_pg_attribute 構造体を指している必要がある。
  • value が NULL の場合、出力には値が含まれない。

アルゴリズムまたは振る舞い

1. printf を使用し、以下の情報をタブ区切りで標準出力に書き出す。

  • 属性番号 (attributeId)
  • 属性名 (attributeP->attname)
  • 属性値 (value が存在する場合のみ = "..." の形式で出力)
  • 型情報 (typeid, len, typmod, byval フラグ)

    • 2. byval フラグは true の場合に 't'、false の場合に 'f' として出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/printtup.c 内のデバッグ用関数（debugtup 等）から呼び出される。
  • **呼び出し先**: 標準ライブラリの printf を使用。

関連データ構造

  • Form_pg_attribute: PostgreSQL のシステムカタログ pg_attribute の行構造を保持する構造体。

失敗モードと境界条件

  • attributeP が無効なポインタである場合、セグメンテーション違反が発生する可能性がある。
  • value が NULL の場合、値の表示部分はスキップされるが、他のメタデータは正常に出力される。

保守担当者向けメモ

  • 本関数は主にデバッグ用であり、フロントエンドへの通信用（printtup）とは別系統の出力である。
  • 出力フォーマットを変更する際は、デバッグログの解析ツール等への影響を考慮すること。
