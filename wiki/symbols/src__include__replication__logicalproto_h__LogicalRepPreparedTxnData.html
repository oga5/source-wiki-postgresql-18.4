LogicalRepPreparedTxnData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/logicalproto.h:144

目的

論理レプリケーションにおいて、準備済みトランザクション（Prepared Transaction）の情報を保持し、パブリッシャーからサブスクライバーへ転送するためのデータ構造です。

シグネチャまたは形状

typedef struct LogicalRepPreparedTxnData
{
    XLogRecPtr    prepare_lsn;    /* 準備時のLSN */
    XLogRecPtr    end_lsn;        /* 終了時のLSN */
    TimestampTz   prepare_time;   /* 準備時刻 */
    TransactionId xid;            /* トランザクションID */
    char          gid[GIDSIZE];   /* グローバル識別子 */
} LogicalRepPreparedTxnData;

入力と出力

  • 本構造体は、論理レプリケーションのワイヤプロトコルにおける「準備済みトランザクション」メッセージのデコード結果として生成されます。
  • logicalrep_read_begin_prepare、logicalrep_read_prepare、logicalrep_read_stream_prepare などの関数を通じて、ストリームから読み取られたデータが本構造体に格納されます。

前提条件と不変条件

  • gid フィールドは GIDSIZE で定義された固定長配列です。
  • prepare_lsn および end_lsn は、PostgreSQLのWAL（Write-Ahead Log）上の有効な位置を指す必要があります。

アルゴリズムまたは振る舞い

  • パブリッシャー側で出力プラグインが準備済みトランザクションをエンコードし、サブスクライバー側でレプリケーションワーカーが本構造体を使用して状態を復元します。
  • apply_handle_prepare_internal 等の関数において、この構造体の情報に基づき、サブスクライバー側でのトランザクション準備処理が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元（読み取り側）**:
  • src/backend/replication/logical/proto.c: logicalrep_read_begin_prepare, logicalrep_read_prepare, logicalrep_read_stream_prepare
  • src/backend/replication/logical/worker.c: apply_handle_prepare_internal
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出し先はありません。

関連データ構造

  • XLogRecPtr: WAL上の位置を保持する型。
  • TransactionId: トランザクションを一意に識別する型。
  • TimestampTz: 時刻情報を保持する型。

失敗モードと境界条件

  • gid の長さが GIDSIZE を超える場合、またはストリームから読み取られたデータがプロトコル定義と一致しない場合、デコード処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は論理レプリケーションのプロトコルの一部であるため、フィールドの追加や変更を行う際は、プロトコルバージョン（LOGICALREP_PROTO_VERSION_NUM）の整合性に注意してください。
  • GIDSIZE の定義変更は本構造体のサイズに直接影響します。
