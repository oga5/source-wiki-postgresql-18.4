heap_page_prune_opt

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/pruneheap.c:192

目的

ヒープページ内の不要なタプルを削除し、HOT (Heap Only Tuple) チェーンを剪定することで、ページ内の空き領域を確保する。オンアクセス・プランニング（アクセス時の最適化）として機能し、競合を避けるために条件付きロックを用いる。

シグネチャまたは形状

void heap_page_prune_opt(Relation relation, Buffer buffer)

入力と出力

  • **入力**:
  • relation: 対象のリレーション。
  • buffer: 剪定対象のヒープページを含むバッファ。
  • **出力**: なし（副作用としてページ内容の変更および統計情報の更新を行う）。

前提条件と不変条件

  • リカバリモード中ではないこと（RecoveryInProgress() が真の場合は即時終了）。
  • ページヘッダーの pd_prune_xid が有効であり、かつ GlobalVisTestIsRemovableXid により削除可能なタプルが存在する可能性があると判断されること。
  • ページが満杯であるか、空き領域が設定された閾値（fill-factor または BLCKSZ / 10 の大きい方）を下回っていること。

アルゴリズムまたは振る舞い

1. **事前チェック**: ページヘッダーの pd_prune_xid を確認し、削除対象の可能性があるか判定する。 2. **ヒューリスティック判定**: ページが満杯か、空き領域が不足しているかを判定する。 3. **ロック取得**: ConditionalLockBufferForCleanup(buffer) を使用し、排他ロックの取得を試みる。取得できない場合は即時終了する。 4. **再チェックと実行**: ロック取得後、再度空き領域を確認し、条件を満たせば heap_page_prune_and_freeze を呼び出して剪定を実行する。 5. **統計更新**: 削除されたタプル数（presult.ndeleted - presult.nnewlpdead）を pgstat_update_heap_dead_tuples に報告する。 6. **終了処理**: バッファロックを解除する。FSM（空き領域マップ）の更新は行わない。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_handler.c
  • **呼び出し先**:
  • heap_page_prune_and_freeze
  • GlobalVisTestFor / GlobalVisTestIsRemovableXid
  • RelationGetTargetPageFreeSpace
  • ConditionalLockBufferForCleanup
  • pgstat_update_heap_dead_tuples

関連データ構造

  • Relation: リレーション情報。
  • Buffer: バッファ管理構造。
  • PageHeader: ページヘッダー（pd_prune_xid の参照に使用）。
  • GlobalVisState: 可視性判定状態。
  • PruneFreezeResult: 剪定結果を保持する構造体。

失敗モードと境界条件

  • **ロック競合**: ConditionalLockBufferForCleanup が失敗した場合、剪定を行わずに終了する。
  • **リカバリ中**: リカバリ中は WAL 記録ができないため、処理をスキップする。
  • **空き領域**: 判定時の空き領域計算はロックなしで行われるため、ヒューリスティックな見積もりとなる。

保守担当者向けメモ

  • 本関数はオンアクセス・プランニングの一部であり、パフォーマンスへの影響を最小限にするため、ロックの取得に条件付きロジックを使用している。
  • pgstat への報告は、VACUUM による削除と重複カウントを避けるため、LP_DEAD に設定された項目を除外して計算している。
  • FSM の更新を意図的に省略しているため、この処理で確保された空き領域は、主に当該ページに対する後続の更新処理で再利用されることを想定している。
