PrepareSortSupportComparisonShim

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sortsupport.c:67

目的

従来のSQL呼び出し可能な比較関数（BTORDER_PROC）を、SortSupport インターフェースで利用可能な形式に適合させるためのラッパー（Shim）を初期化します。

シグネチャまたは形状

void PrepareSortSupportComparisonShim(Oid cmpFunc, SortSupport ssup)

入力と出力

  • cmpFunc (Oid): ラップ対象となる比較関数のOID。
  • ssup (SortSupport): 初期化対象のソートサポート構造体。
  • 出力: なし（ssup 構造体の ssup_extra および comparator フィールドを更新）。

前提条件と不変条件

  • ssup は有効な SortSupportData 構造体へのポインタであること。
  • ssup->ssup_cxt には、Shim用のメモリ領域を割り当てるための有効なメモリコンテキストが設定されていること。

アルゴリズムまたは振る舞い

1. ssup->ssup_cxt 上に SortShimExtra 構造体用のメモリを確保します。 2. fmgr_info_cxt を使用して、指定された cmpFunc の関数情報を取得し、extra->flinfo に格納します。 3. InitFunctionCallInfoData を用いて、比較関数呼び出しに必要な fcinfo を初期化します。引数の数は2に固定され、NULLフラグは false に設定されます。 4. ssup->ssup_extra に確保した extra を割り当てます。 5. ssup->comparator に、Shim実行用の関数ポインタ comparison_shim を割り当てます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/gin/gininsert.c
  • src/backend/executor/nodeMergejoin.c
  • src/backend/utils/sort/sortsupport.c (内部のフォールバック処理)
  • src/backend/utils/sort/tuplesortvariants.c
  • 呼び出し先:
  • MemoryContextAlloc
  • fmgr_info_cxt
  • InitFunctionCallInfoData

関連データ構造

  • SortSupportData: ソートの比較ロジックを保持するメイン構造体。
  • SortShimExtra: Shimの実行に必要な FmgrInfo や FunctionCallInfoData を保持するコンテキスト構造体。

失敗モードと境界条件

  • メモリ不足時: MemoryContextAlloc がエラーをスローします。
  • 無効な cmpFunc: fmgr_info_cxt がシステムカタログ検索に失敗した場合、エラーをスローします。

保守担当者向けメモ

  • 本関数は、高速なソートサポート関数（BTSORTSUPPORT_PROC）が存在しない場合のフォールバックとして機能します。
  • ssup_extra に割り当てられたメモリは ssup_cxt に依存しているため、ssup のライフサイクル管理に注意してください。
