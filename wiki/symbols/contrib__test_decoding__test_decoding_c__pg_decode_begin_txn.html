pg_decode_begin_txn

定義

  • 種別: function
  • 定義箇所: contrib/test_decoding/test_decoding.c:289

目的

論理デコーディングにおけるトランザクション開始時のコールバック関数であり、トランザクションごとの状態管理用データ構造の初期化と、必要に応じた BEGIN メッセージの出力を行う。

シグネチャまたは形状

static void pg_decode_begin_txn(LogicalDecodingContext *ctx, ReorderBufferTXN *txn)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。output_plugin_private に TestDecodingData を保持。
  • txn: 開始されるトランザクション情報。
  • **出力**: なし（副作用として txn->output_plugin_private へのデータ割り当ておよび出力ストリームへの書き込みを行う）。

前提条件と不変条件

  • ctx->context は有効なメモリコンテキストであること。
  • txn->output_plugin_private は本関数によって TestDecodingTxnData 型のメモリ領域で初期化される。
  • txndata->xact_wrote_changes は初期状態で false に設定される。

アルゴリズムまたは振る舞い

1. ctx->context 上に TestDecodingTxnData 構造体をゼロクリアして割り当てる。 2. 割り当てた構造体を txn->output_plugin_private に格納する。 3. data->skip_empty_xacts が真の場合、トランザクション内の変更が確認されるまで BEGIN の出力を保留し、即座に終了する。 4. data->skip_empty_xacts が偽の場合、pg_output_begin を呼び出し、トランザクション開始情報を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの論理デコーディングエンジン（cb->begin_cb として登録された関数ポインタ経由）。
  • **呼び出し先**:
  • MemoryContextAllocZero（メモリ確保）
  • pg_output_begin（条件付きで呼び出し）

関連データ構造

  • LogicalDecodingContext: デコーディングの全体コンテキスト。
  • ReorderBufferTXN: トランザクションのメタデータ。
  • TestDecodingData: プラグイン全体の設定情報。
  • TestDecodingTxnData: トランザクションごとの状態（変更の有無など）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocZero がエラーを発生させる可能性がある。
  • skip_empty_xacts が有効な場合、トランザクション内で変更が発生しない限り BEGIN は出力されない。

保守担当者向けメモ

  • 本関数は _PG_output_plugin_init で begin_cb に登録される。
  • TestDecodingTxnData のメモリ管理は ctx->context に依存しているため、コンテキストの生存期間に注意すること。
