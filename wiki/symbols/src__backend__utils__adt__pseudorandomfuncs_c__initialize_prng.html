initialize_prng

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pseudorandomfuncs.c:33

目的

プロセス単位で保持される疑似乱数生成器（PRNG）の状態を初期化し、シードを設定します。

シグネチャまたは形状

static void initialize_prng(void)

入力と出力

  • 入力: なし（内部状態 prng_seed_set および prng_state を参照）
  • 出力: なし（副作用として prng_state を初期化し、prng_seed_set を true に更新）

前提条件と不変条件

  • prng_state はバックエンドプロセスごとに静的変数として保持されます。
  • prng_seed_set が true の場合、本関数は何も行いません。

アルゴリズムまたは振る舞い

1. prng_seed_set が false であることを確認します。 2. pg_prng_strong_seed(&prng_state) を呼び出し、高品質な乱数源によるシード設定を試みます。 3. 上記が失敗した場合、フォールバックとして現在の時刻（GetCurrentTimestamp()）とプロセスID（MyProcPid）を組み合わせた値で pg_prng_seed を実行します。 4. 処理完了後、prng_seed_set を true に設定します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/adt/pseudorandomfuncs.c 内の各乱数生成関数（drandom, drandom_normal, int4random, int8random, numeric_random 等）から、乱数生成の直前に呼び出されます。
  • 呼び出し先:
  • pg_prng_strong_seed
  • GetCurrentTimestamp
  • pg_prng_seed

関連データ構造

  • prng_state: 乱数生成器の状態を保持する構造体。
  • prng_seed_set: 初期化済みか否かを示す静的フラグ。

失敗モードと境界条件

  • pg_prng_strong_seed が失敗した場合、時刻とPIDに基づく低品質なシード生成へ自動的に切り替わります。

保守担当者向けメモ

  • 本関数はプロセス単位の初期化を行うため、セッションをまたぐ共有や並列クエリ間での同期は考慮されていません。
  • pg_prng ライブラリの仕様変更に伴い、シード生成ロジックを調整する必要がある場合は本関数を確認してください。
