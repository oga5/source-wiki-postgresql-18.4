init_ps_display

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/ps_status.c:268

目的

バックエンドプロセスのタイトル（ps コマンド等で表示される文字列）を初期化し、プログラム名、クラスタ名、およびプロセス種別を含む固定プレフィックスを設定します。

シグネチャまたは形状

void init_ps_display(const char *fixed_part)

入力と出力

  • **入力**: fixed_part（プロセス名の固定部分。NULL の場合は MyBackendType から自動生成される）。
  • **出力**: なし（副作用としてグローバルな ps_buffer 等を更新）。

前提条件と不変条件

  • save_ps_display_args() が事前に呼び出されていること。
  • IsUnderPostmaster が真であること（スタンドアローンバックエンドでは動作しない）。
  • PS_USE_NONE が定義されている環境では何もしない。

アルゴリズムまたは振る舞い

1. fixed_part が未指定の場合、GetBackendTypeDesc(MyBackendType) で補完する。 2. PS_USE_CLOBBER_ARGV 環境では、save_argv の各スロットを ps_buffer の終端（NUL）を指すように再配置する。 3. OSの仕様に応じて PROGRAM_NAME_PREFIX を決定し、cluster_name の有無に応じて ps_buffer に文字列を書き込む。 4. ps_buffer_cur_len および ps_buffer_fixed_size を更新する。 5. update_process_title の設定を一時的に強制有効化し、set_ps_display("") を呼び出して初期状態をOSに反映させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/auxprocess.c
  • src/backend/postmaster/bgworker.c
  • src/backend/postmaster/syslogger.c
  • src/backend/replication/logical/slotsync.c
  • src/backend/tcop/backend_startup.c
  • **呼び出し先**:
  • GetBackendTypeDesc
  • set_ps_display

関連データ構造

  • ps_buffer, ps_buffer_size, ps_buffer_cur_len, ps_buffer_fixed_size
  • save_argv, save_argc
  • cluster_name
  • MyBackendType

失敗モードと境界条件

  • save_ps_display_args() が未実行の場合、何もせず終了する。
  • ps_buffer が確保されていない場合（PS_USE_CLOBBER_ARGV 環境）、何もせず終了する。
  • cluster_name が空の場合、プレフィックスからクラスタ名を除外してフォーマットする。

保守担当者向けメモ

  • 本関数はOS固有のプロセス名変更メカニズム（setproctitle や argv 上書き）を抽象化する起点です。
  • PS_USE_CLOBBER_ARGV が有効な環境では、argv 領域のメモリレイアウトに直接介入するため、変更時は save_ps_display_args との整合性に注意してください。
