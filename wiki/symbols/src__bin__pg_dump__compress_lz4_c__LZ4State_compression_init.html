LZ4State_compression_init

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:101

目的

LZ4 圧縮セッションを開始するためのコンテキスト初期化およびバッファ割り当てを行う。

シグネチャまたは形状

static bool LZ4State_compression_init(LZ4State *state);

入力と出力

  • **入力**: LZ4State *state（圧縮設定 prefs を含む状態構造体へのポインタ）
  • **出力**: bool（成功時に true、失敗時に false を返す）

前提条件と不変条件

  • state->prefs が適切に設定されていること。
  • state->buflen は LZ4F_compressBound によって計算され、LZ4F_HEADER_SIZE_MAX 以上であることが保証される。

アルゴリズムまたは振る舞い

1. LZ4F_compressBound を用いて、DEFAULT_IO_BUFFER_SIZE に基づく必要なバッファサイズを計算し、state->buflen に格納する。 2. state->buflen が LZ4F_HEADER_SIZE_MAX 未満の場合は、最小値を確保するために値を更新する。 3. LZ4F_createCompressionContext を呼び出し、圧縮コンテキスト state->ctx を生成する。失敗時は state->errcode にエラーコードを格納して終了する。 4. pg_malloc を使用して state->buffer を割り当てる。 5. LZ4F_compressBegin を呼び出し、圧縮ヘッダーを生成して state->buffer に書き込む。 6. 成功時、書き込まれたヘッダーサイズを state->compressedlen に格納し、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_lz4.c:301
  • src/bin/pg_dump/compress_lz4.c:367
  • 同ファイル内の初期化ロジック（src/bin/pg_dump/compress_lz4.c:92 に言及あり）
  • **呼び出し先**:
  • LZ4F_compressBound, LZ4F_createCompressionContext, LZ4F_compressBegin, LZ4F_isError (liblz4 API)
  • pg_malloc (PostgreSQL ユーティリティ)

関連データ構造

  • LZ4State: 圧縮コンテキスト、設定、バッファ、エラーコードを保持する構造体。

失敗モードと境界条件

  • LZ4F_createCompressionContext または LZ4F_compressBegin がエラーを返した場合、state->errcode にエラー値が設定され、関数は false を返す。
  • メモリ割り当て失敗時は pg_malloc によりプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は LZ4Stream 系関数の遅延初期化プロセスの一部として呼び出される。
  • LZ4F_HEADER_SIZE_MAX のチェックは、圧縮開始時に必要な最小バッファサイズを保証するための重要な不変条件である。
