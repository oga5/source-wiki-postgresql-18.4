allocate_incremental_state

定義

  • 種別: function
  • 定義箇所: src/common/jsonapi.c:433

目的

JSONのインクリメンタル（非再帰的）解析に必要な状態管理構造体および解析スタックを初期化し、JsonLexContext に割り当てます。

シグネチャまたは形状

static bool allocate_incremental_state(JsonLexContext *lex)

入力と出力

  • **入力**: JsonLexContext *lex（解析コンテキストへのポインタ）
  • **出力**: bool（成功した場合は true、メモリ不足等で失敗した場合は false）

前提条件と不変条件

  • lex は有効な JsonLexContext を指している必要があります。
  • lex->lex_level が 0 であることを前提としています。
  • 成功時、lex->incremental が true に設定されます。

アルゴリズムまたは振る舞い

1. JsonIncrementalState および JsonParserStack に必要なメモリ領域（prediction、fnames、fnull）を確保します。 2. JSONAPI_USE_PQEXPBUFFER が定義されている環境では、メモリ確保の失敗を検出し、確保済みの領域を解放した上で lex->inc_state を failed_inc_oom に設定し、false を返します。 3. jsonapi_initStringInfo を呼び出し、部分トークン保持用のバッファを初期化します。 4. lex->pstack に確保した各領域を割り当て、スタックサイズを JS_STACK_CHUNK_SIZE に設定します。 5. lex->pstack->fnames[0] を NULL に初期化し、解析レベル 0 の状態を確定させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/jsonapi.c 内のインクリメンタル解析開始処理（行 515, 753 など）。
  • **呼び出し先**: ALLOC / ALLOC0（メモリ確保マクロ）、FREE（メモリ解放マクロ）、jsonapi_initStringInfo。

関連データ構造

  • JsonLexContext: 解析のメインコンテキスト。
  • JsonIncrementalState: インクリメンタル解析の状態保持用構造体。
  • JsonParserStack: 予測スタックおよび関連するメタデータ（fnames, fnull）を保持。

失敗モードと境界条件

  • メモリ確保失敗時、JSONAPI_USE_PQEXPBUFFER 定義下では全領域を解放し、false を返します。
  • lex->lex_level が 0 でない場合、Assert により停止します。

保守担当者向けメモ

  • 本関数は JsonLexContext のメモリ管理戦略（palloc vs malloc）に依存します。
  • fnames 配列の初期化は、freeJsonLexContext() が安全にクリーンアップを行うための重要な不変条件です。
