MemoryContextId

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/mcxtfuncs.c:37

目的

メモリコンテキスト（MemoryContext）と、SQLインターフェースを通じて一意に識別するための整数ID（context_id）を紐付けるための内部構造体です。

シグネチャまたは形状

typedef struct MemoryContextId
{
	MemoryContext context;
	int			context_id;
} MemoryContextId;

入力と出力

本構造体は、メモリコンテキストの走査中にハッシュテーブルのキーおよび値として使用されます。

  • 入力: MemoryContext ポインタおよび割り当てられた整数ID。
  • 出力: なし（データ保持用）。

前提条件と不変条件

  • context_id は、メモリコンテキストの階層構造を幅優先探索（BFS）で走査する過程で、一貫性を保つために割り当てられます。
  • ハッシュテーブル（context_id_lookup）の各エントリとして使用される際、context フィールドがキーとして機能します。

アルゴリズムまたは振る舞い

メモリコンテキストの統計情報を取得する際、この構造体を用いてコンテキストとIDの対応関係を保持します。これにより、SQLクエリの結果セットにおいて、親子関係のパスを安定して表現することが可能になります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/mcxtfuncs.c 内のメモリコンテキスト走査ロジック（pg_get_backend_memory_contexts 等）。
  • 呼び出し先: hash_search（ハッシュテーブル操作用）。

関連データ構造

  • MemoryContext: メモリ管理サブシステムにおけるコンテキストの基底型。
  • context_id_lookup: 本構造体を格納するために使用されるハッシュテーブル。

失敗モードと境界条件

  • ハッシュテーブルへの挿入や検索時にメモリ不足が発生した場合、hash_search の挙動に依存します。
  • 構造体自体のサイズは sizeof(MemoryContextId) としてハッシュテーブルの初期化時に使用されます。

保守担当者向けメモ

  • 本構造体は mcxtfuncs.c 内でのみ使用される内部的なマッピング用データです。
  • IDの割り当て順序（BFS）を変更する場合は、この構造体を利用する走査ロジック全体に影響が及ぶため注意してください。
