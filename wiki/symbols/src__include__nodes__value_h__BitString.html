BitString

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/value.h:71

目的

SQLクエリ内のビット文字列リテラルを構文解析ツリー（Parse Tree）内で保持するためのノード構造体です。

シグネチャまたは形状

typedef struct BitString
{
	pg_node_attr(special_read_write)

	NodeTag		type;
	char	   *bsval;
} BitString;

入力と出力

  • 入力: コンストラクタ makeBitString(char *str) を介して文字列ポインタを受け取り、ノードを生成します。
  • 出力: bsval フィールドを通じて、保持しているビット文字列の文字列表現を提供します。

前提条件と不変条件

  • type フィールドには T_BitString が設定される必要があります。
  • bsval は palloc されたメモリ領域を指す必要があります。

アルゴリズムまたは振る舞い

  • パーサーが字句解析でビット文字列リテラルを検出した際に生成されます。
  • pg_node_attr(special_read_write) 属性により、ノードの読み書きに関する特殊な処理が適用されます。

呼び出し元と呼び出し先

  • 呼び出し元（生成）: src/backend/nodes/value.c 内の makeBitString 関数。
  • 呼び出し先（シリアライズ/デシリアライズ）:
  • src/backend/nodes/outfuncs.c: _outBitString 関数がノードの出力処理を行います。
  • src/backend/nodes/readfuncs.c: ノードの読み込み処理で参照されます。
  • その他: src/include/nodes/parsenodes.h 等で構造体の一部として利用される可能性があります。

関連データ構造

  • NodeTag: ノードの型識別子。
  • List: 本ノードが格納されるコンテナ構造。

失敗モードと境界条件

  • bsval が NULL の場合、後続の処理でクラッシュする可能性があります。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • Float ノードと同様に、値を char* として保持する設計です。
  • src/backend/nodes/value.c に実装されている makeBitString 関数が、本構造体のメモリ割り当てと初期化の唯一の標準的な手段です。
