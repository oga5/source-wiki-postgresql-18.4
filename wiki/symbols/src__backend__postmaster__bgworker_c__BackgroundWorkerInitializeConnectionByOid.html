BackgroundWorkerInitializeConnectionByOid

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/bgworker.c:886

目的

バックグラウンドワーカーが特定のデータベースおよびユーザーとして接続を初期化し、実行環境をセットアップする。

シグネチャまたは形状

void BackgroundWorkerInitializeConnectionByOid(Oid dboid, Oid useroid, uint32 flags)

入力と出力

  • **入力**:
  • dboid (Oid): 接続先データベースのOID。
  • useroid (Oid): 接続に使用するロールのOID。
  • flags (uint32): 接続動作を制御するフラグ（BGWORKER_BYPASS_ALLOWCONN, BGWORKER_BYPASS_ROLELOGINCHECK）。
  • **出力**: なし（成功時はプロセス状態が更新される）。

前提条件と不変条件

  • MyBgworkerEntry が有効であること。
  • ワーカー登録時に BGWORKER_BACKEND_DATABASE_CONNECTION フラグが設定されていること（未設定の場合は FATAL エラー）。
  • 実行モードが InitProcessingMode であること。

アルゴリズムまたは振る舞い

1. flags に基づき、InitPostgres に渡す内部フラグ（INIT_PG_OVERRIDE_ALLOW_CONNS, INIT_PG_OVERRIDE_ROLE_LOGIN）を構築する。 2. worker->bgw_flags を確認し、データベース接続が許可されていない場合は FATAL で終了する。 3. InitPostgres を呼び出し、指定されたデータベースとユーザーでセッションを初期化する。 4. 現在の処理モードが InitProcessingMode であることを確認し、その後 SetProcessingMode(NormalProcessing) を呼び出して通常処理モードへ移行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c
  • src/backend/access/transam/parallel.c
  • src/backend/replication/logical/worker.c
  • src/test/modules/worker_spi/worker_spi.c
  • **呼び出し先**:
  • InitPostgres
  • IsInitProcessingMode
  • SetProcessingMode
  • ereport

関連データ構造

  • BackgroundWorker (グローバル変数 MyBgworkerEntry 経由で参照)

失敗モードと境界条件

  • 登録時にデータベース接続フラグがない場合: ERRCODE_PROGRAM_LIMIT_EXCEEDED で FATAL エラー。
  • 接続初期化後に InitProcessingMode ではない場合: ERROR で終了。

保守担当者向けメモ

  • InitPostgres を呼び出すため、この関数はワーカープロセスの初期化フェーズの最後で一度だけ呼び出されることを想定している。
  • flags の処理は、データベースの接続制限やログインチェックをバイパスするために使用される。
