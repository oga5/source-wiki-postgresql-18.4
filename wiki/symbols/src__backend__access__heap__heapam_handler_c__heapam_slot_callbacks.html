heapam_slot_callbacks

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_handler.c:69

目的

ヒープテーブルアクセス手法（AM）で使用される TupleTableSlot の操作コールバックセットを取得する。

シグネチャまたは形状

static const TupleTableSlotOps *heapam_slot_callbacks(Relation relation)

入力と出力

  • **入力**: Relation relation（対象となるリレーション）
  • **出力**: const TupleTableSlotOps *（ヒープタプルおよびバッファ管理用のスロット操作関数群へのポインタ）

前提条件と不変条件

  • 常に TTSOpsBufferHeapTuple を返す。
  • 入力された relation は本関数内では参照されていない。

アルゴリズムまたは振る舞い

  • 呼び出されると、ヒープストレージエンジンが使用する標準的なスロット操作定義である TTSOpsBufferHeapTuple を静的に返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_handler.c 内の heapam_methods 構造体の初期化において、.slot_callbacks フィールドに割り当てられている。
  • **呼び出し先**: なし。

関連データ構造

  • TupleTableSlotOps: スロット操作を抽象化する構造体。
  • TTSOpsBufferHeapTuple: ヒープAM用の具体的なスロット操作実装。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。常に固定の定数アドレスを返す。

保守担当者向けメモ

  • 本関数は TableAmRoutine の一部としてテーブルAMディスパッチ機構から利用される。
  • 将来的にヒープAMで異なるスロット形式が必要となった場合、relation の属性に基づいて動的に切り替えるロジックへの拡張が可能である。
