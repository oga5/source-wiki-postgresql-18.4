TM_Result

定義

  • 種別: enum
  • 定義箇所: src/include/access/tableam.h:71

目的

テーブルに対するタプル操作（更新、削除、ロック）の結果状態を表現し、呼び出し元に対して操作の成否や、競合が発生した際の詳細な理由を伝達するために使用されます。

シグネチャまたは形状

typedef enum TM_Result
{
    TM_Ok,
    TM_Invisible,
    TM_SelfModified,
    TM_Updated,
    TM_Deleted,
    TM_BeingModified,
    TM_WouldBlock
} TM_Result;

入力と出力

  • 本シンボルは列挙型であり、関数呼び出しの戻り値として使用されます。
  • table_update、table_delete、table_lock_tuple などのテーブルアクセス手法（Table AM）のルーチンが、操作結果としてこの値を返します。

前提条件と不変条件

  • TM_Ok は操作が成功したことを示します。
  • TM_WouldBlock は、ロック取得を待機しない設定で呼び出された場合にのみ返される可能性があります。

アルゴリズムまたは振る舞い

各列挙値は以下の状況を意味します：

  • TM_Ok: 操作成功。
  • TM_Invisible: 対象タプルが現在のスナップショットから不可視。
  • TM_SelfModified: 対象タプルが現在のバックエンドによって既に変更済み。
  • TM_Updated: 対象タプルが他トランザクションによって更新済み（移動を含む）。
  • TM_Deleted: 対象タプルが他トランザクションによって削除済み。
  • TM_BeingModified: 他セッションが現在変更中（待機しない設定時のみ）。
  • TM_WouldBlock: ロック取得不可（lock_tuple でのみ使用）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c 内の各関数（heap_lock_updated_tuple 等）や contrib/pgrowlocks/pgrowlocks.c などがこの結果を受け取り、後続の処理を分岐させています。
  • **呼び出し先**: 本シンボルは列挙型であるため、直接の呼び出し先はありません。

関連データ構造

  • TM_FailureData: 操作失敗時の詳細情報を保持するために併用されることがあります。

失敗モードと境界条件

  • 本シンボル自体は失敗を示すものではなく、操作結果の分類です。
  • 境界条件として、同時実行トランザクションによる競合（TM_Updated, TM_Deleted, TM_BeingModified）が発生した際、呼び出し側で適切な再試行やエラーハンドリングを行う必要があります。

保守担当者向けメモ

  • 新しい Table AM を実装する際は、各操作関数がこれらの値を適切に返すように設計してください。
  • TM_BeingModified のような状態は、呼び出し側の設定（待機するか否か）に依存するため、実装時には仕様を厳密に守る必要があります。
