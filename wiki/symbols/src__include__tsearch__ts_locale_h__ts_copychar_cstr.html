ts_copychar_cstr

定義

  • 種別: static inline function
  • 定義箇所: src/include/tsearch/ts_locale.h:49

目的

ヌル終端文字列（C文字列）として与えられたマルチバイト文字を、指定された宛先バッファへコピーし、コピーしたバイト数を返します。

シグネチャまたは形状

int ts_copychar_cstr(void *dest, const void *src)

入力と出力

  • **入力**:
  • dest: コピー先のメモリ領域へのポインタ。
  • src: コピー元のマルチバイト文字を含むヌル終端文字列へのポインタ。
  • **出力**:
  • 戻り値: コピーされたバイト数（int 型）。

前提条件と不変条件

  • src は有効なヌル終端文字列を指している必要があります。
  • dest は、コピーされるマルチバイト文字を格納するのに十分な領域を確保している必要があります。

アルゴリズムまたは振る舞い

1. pg_mblen_cstr((const char *) src) を呼び出し、src が指すマルチバイト文字のバイト長を算出します。 2. 算出したバイト長を引数として ts_copychar_with_len を呼び出し、実際のコピー処理を実行します。 3. ts_copychar_with_len の戻り値（コピーしたバイト数）をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/regis.c
  • src/backend/tsearch/spell.c
  • src/backend/utils/adt/tsquery.c
  • src/backend/utils/adt/tsvector_parser.c
  • **呼び出し先**:
  • pg_mblen_cstr
  • ts_copychar_with_len

関連データ構造

  • 特になし。

失敗モードと境界条件

  • src が指す文字が不正なマルチバイトシーケンスである場合、pg_mblen_cstr の挙動に依存します。
  • dest のバッファサイズが不足している場合、バッファオーバーランが発生する可能性があります。

保守担当者向けメモ

  • 本関数は COPYCHAR マクロとしても定義されています。
  • マルチバイト文字の長さを動的に判定するため、固定長文字処理よりもオーバーヘッドが発生します。パフォーマンスが重要なループ内での使用には注意してください。
