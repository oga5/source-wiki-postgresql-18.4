XLogInitNewTimeline

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/transam/xlog.c:5251

目的

アーカイブ・リカバリ完了後に新しいタイムラインへ切り替える際、WAL の初期状態を構築する。

シグネチャまたは形状

static void XLogInitNewTimeline(TimeLineID endTLI, XLogRecPtr endOfLog, TimeLineID newTLI)

入力と出力

  • **入力**:
  • endTLI: 旧タイムライン ID。
  • endOfLog: 旧タイムラインの終了位置（LSN）。
  • newTLI: 新しく開始するタイムライン ID。
  • **出力**: なし（副作用としてファイルシステム上に新しい WAL セグメントを作成する）。

前提条件と不変条件

  • endTLI != newTLI であること（アサーションにより強制）。
  • WAL の書き込みが許可されていない状態であること。

アルゴリズムまたは振る舞い

1. UpdateMinRecoveryPoint を呼び出し、最小リカバリポイントを更新する。 2. endOfLog を基に、旧タイムラインの最終セグメント番号と、新タイムラインの開始セグメント番号を算出する。 4. 切り替え位置がセグメント境界である場合、XLogFileInit を使用して新しいセグメントファイルを初期化する。 5. XLogArchiveCleanup を呼び出し、新しく作成したセグメントに関連する .ready または .done フラグが存在しないことを保証する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c 内のリカバリ終了処理フローから呼び出される。
  • **呼び出し先**:
  • UpdateMinRecoveryPoint
  • XLByteToPrevSeg, XLByteToSeg
  • XLogFileCopy
  • XLogFileInit
  • XLogFileName
  • XLogArchiveCleanup
  • close

関連データ構造

  • TimeLineID: タイムライン識別子。
  • XLogRecPtr: WAL 内の論理位置。
  • XLogSegNo: WAL セグメント番号。

失敗モードと境界条件

  • close(fd) が失敗した場合、ereport(ERROR, ...) により処理を中断し、エラーログを出力する。

保守担当者向けメモ

  • 本関数は WAL 書き込みが開始される前に実行されるため、ロックの考慮は不要である。
  • XLogFileInit と同様に、不正なファイルが配置されないよう慎重に扱う必要がある。
