SplitGUCList

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:764

目的

GUC（Grand Unified Configuration）リスト形式の文字列を解析し、個別の識別子またはファイル名の配列に分割する。

シグネチャまたは形状

bool SplitGUCList(char *rawstring, char separator, char ***namelist)

入力と出力

  • **入力**:
  • rawstring: 解析対象の文字列（破壊的に変更される）。
  • separator: リストの区切り文字（例: ,）。
  • **出力**:
  • namelist: 分割された文字列ポインタの配列へのポインタ。呼び出し元で解放が必要。
  • 戻り値: 成功時に true、構文エラー（引用符の不一致や空の識別子など）時に false を返す。

前提条件と不変条件

  • rawstring は pg_malloc で確保されたメモリ領域を指している必要がある。
  • 空の識別子は許可されない。
  • 引用符で囲まれた識別子内の二重引用符（""）は、単一の引用符として扱われる。

アルゴリズムまたは振る舞い

1. rawstring の長さに基づき、結果を格納するポインタ配列を pg_malloc で確保する。 2. 先頭の空白をスキップし、空文字列の場合は true を返して終了する。 3. 文字列を走査し、以下のいずれかの形式で識別子を抽出する：

  • **引用符あり**: " で囲まれた範囲。内部の "" を " に置換（collapse）する。
  • **引用符なし**: separator または空白文字が出現するまでを識別子とする。

    • 4. 抽出した識別子の直後の文字が separator であれば次の識別子へ進み、\0 であれば終了する。 5. 抽出した各識別子の末尾を \0 で終端し、namelist にポインタを格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c
  • src/backend/postmaster/postmaster.c
  • src/backend/storage/file/fd.c
  • src/backend/utils/adt/ruleutils.c
  • src/backend/utils/adt/varlena.c
  • src/bin/pg_dump/dumputils.c
  • **呼び出し先**:
  • pg_malloc（メモリ確保）
  • isspace, strchr, memmove, strlen（標準ライブラリ）

関連データ構造

  • namelist: char ** 型の動的配列。末尾は NULL で終端される。

失敗モードと境界条件

  • 引用符が閉じられていない場合、false を返す。
  • 引用符なしの識別子が空（例: ,, や先頭が区切り文字）の場合、false を返す。
  • 不明な文字や不正な構文に遭遇した場合、false を返す。

保守担当者向けメモ

  • 本関数は入力文字列 rawstring を直接書き換えるため、呼び出し元で元の文字列を保持する必要がある場合は事前にコピーを作成すること。
  • src/backend/utils/adt/varlena.c にも同名の関数が存在し、同様のロジックで実装されている。修正時は両方の整合性を確認すること。
