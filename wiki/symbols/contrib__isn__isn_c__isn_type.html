isn_type

定義

  • 種別: enum
  • 定義箇所: contrib/isn/isn.c:39

目的

PostgreSQL の contrib/isn モジュールにおいて、サポートされる国際標準番号（ISBN, ISSN, ISMN, EAN13, UPC）の識別子型を定義し、型変換や検証処理における分類を管理する。

シグネチャまたは形状

enum isn_type
{
    INVALID,
    ANY,
    EAN13,
    ISBN,
    ISMN,
    ISSN,
    UPC
};

入力と出力

本シンボルは列挙型定義であり、関数ではないため直接的な入出力はない。プログラム内では、識別子の種類を特定するための定数として使用される。

前提条件と不変条件

  • INVALID は初期化状態やエラー状態を示すために使用される。
  • ANY は特定の型に限定しない汎用的な処理を指示する際に使用される。

アルゴリズムまたは振る舞い

各識別子に対応する列挙子を持ち、ean2isn 関数などの内部処理において、入力された値がどの規格に準拠するか、あるいはどの規格として扱うかを判定するフラグとして機能する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/isn/isn.c 内の ean2isn 関数や、各型の入力関数（_in サフィックスを持つ関数群）において、型判定用の変数として参照されている。
  • **呼び出し先**: 本シンボルは列挙型定義であるため、呼び出し先は存在しない。

関連データ構造

  • ean13: 本モジュールで扱う内部ストレージ形式（uint64）。
  • contrib/isn/*.h 各ヘッダーファイル: 各識別子に対応する範囲テーブル定義。

失敗モードと境界条件

  • INVALID が選択された場合、後続の処理で型不一致や解析エラーとして扱われる可能性がある。

保守担当者向けメモ

  • 新しい識別子規格を追加する場合は、この列挙型に新しい要素を追加する必要がある。
  • ean2isn 等のロジックでこの列挙型がどのように判定に使用されているかを確認すること。
