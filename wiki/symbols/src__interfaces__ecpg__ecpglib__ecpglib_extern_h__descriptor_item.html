descriptor_item

定義

  • **種別**: struct
  • **定義箇所**: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:121

目的

SQL記述子（Descriptor）内の個々の項目情報を保持し、ホスト変数とデータベース間のデータ転送における属性管理を行うための構造体です。

シグネチャまたは形状

struct descriptor_item
{
    int            num;
    char          *data;
    int            indicator;
    int            length;
    int            precision;
    int            scale;
    int            type;
    bool           is_binary;
    int            data_len;
    struct descriptor_item *next;
};

入力と出力

  • **入力**: 構造体内の各フィールドは、SQL実行時のデータ型、精度、スケール、および実際のデータ値（data）を保持します。
  • **出力**: next ポインタにより、記述子内の項目リストとして後続の descriptor_item へ連結されます。

前提条件と不変条件

  • next フィールドにより、単方向リスト構造を形成します。
  • メモリ割り当てには、ECPGライブラリのメモリ管理ポリシー（ecpg_alloc 等）に従う必要があります。

アルゴリズムまたは振る舞い

  • SQL実行時に、データベースから取得した結果セットの各列属性を格納します。
  • store_input_from_desc 関数などを通じて、記述子内の情報をステートメントの入力パラメータとして利用します。
  • set_desc_attr 関数により、記述子項目の属性設定が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/ecpglib/descriptor.c: 記述子の属性設定および管理処理。
  • src/interfaces/ecpg/ecpglib/execute.c: SQL実行時の入力データ処理。
  • src/interfaces/ecpg/ecpglib/ecpglib_extern.h: descriptor 構造体内の items フィールドとして参照。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • struct descriptor: 記述子全体を管理する構造体（items フィールドで本構造体を保持）。
  • struct variable: ホスト変数とのマッピングに使用されます。

失敗モードと境界条件

  • メモリ不足時に ecpg_alloc が失敗する可能性があります。
  • リストの走査時に next ポインタが適切に終端（NULL）されている必要があります。

保守担当者向けメモ

  • 本構造体は ecpglib 内部の記述子管理の基盤です。リスト操作を行う際は、メモリリークを防ぐため ecpg_free による適切な解放を徹底してください。
