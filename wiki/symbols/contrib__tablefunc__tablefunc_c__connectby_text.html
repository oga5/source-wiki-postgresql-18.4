connectby_text

定義

  • 種別: function
  • 定義箇所: contrib/tablefunc/tablefunc.c:976

目的

階層データ（親子関係）を持つテーブルから、指定された開始点に基づき再帰的にツリー構造を抽出し、結果セットとして返却する。

シグネチャまたは形状

Datum connectby_text(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relname (text): 対象テーブル名
  • key_fld (text): 主キー列名
  • parent_key_fld (text): 親キー列名
  • start_with (text): 探索開始ノードのキー値
  • max_depth (int32): 探索の最大深さ
  • branch_delim (text, オプション): 階層パスの区切り文字（指定がない場合は ~ を使用）
  • **出力**:
  • Datum: 常に (Datum) 0 を返す。実際のデータは rsinfo->setResult に格納された Tuplestore を通じて返却される。

前提条件と不変条件

  • 呼び出し元が ReturnSetInfo をサポートし、SFRM_Materialize モードを許可している必要がある。
  • expectedDesc が提供されている必要がある。
  • 戻り値のタプル構造は validateConnectbyTupleDesc による検証を通過する必要がある。

アルゴリズムまたは振る舞い

1. **コンテキスト検証**: 呼び出し元が Tuplestore を受け取れるか確認し、エラー時は ERRCODE_FEATURE_NOT_SUPPORTED を送出する。 2. **引数処理**: 入力テキストを C 文字列に変換し、オプション引数 branch_delim を設定する。 3. **メモリ管理**: per_query_ctx に切り替え、タプル記述子をコピーしてメタデータを取得する。 4. **探索実行**: connectby 関数を呼び出し、再帰的な探索結果を Tuplestore に格納する。 5. **結果設定**: rsinfo->setResult に Tuplestore を、rsinfo->setDesc にタプル記述子を設定し、SFRM_Materialize モードを有効化する。 6. **終了**: メモリコンテキストを復元し、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数呼び出しインターフェース（PG_FUNCTION_INFO_V1 により登録）。
  • **呼び出し先**: connectby（実質的な探索処理を実行）、validateConnectbyTupleDesc、TupleDescGetAttInMetadata、CreateTupleDescCopy。

関連データ構造

  • ReturnSetInfo: 関数呼び出しの戻り値情報および Tuplestore の保持に使用。
  • TupleDesc: 返却されるタプルの構造定義。
  • AttInMetadata: 文字列からタプルへの変換用メタデータ。

失敗モードと境界条件

  • **モード不一致**: SFRM_Materialize モードが許可されていない場合、エラーを発生させる。
  • **メモリ不足**: Tuplestore 構築時にメモリ不足が発生した場合は PostgreSQL の標準的なエラー処理に従う。
  • **階層の深さ**: max_depth を超える階層や循環参照がある場合、connectby 内部の再帰処理に依存する。

保守担当者向けメモ

  • 本関数はラッパーであり、実際の再帰ロジックは connectby に委譲されている。
  • branch_delim のデフォルト値は ~ である。
  • SFRM_Materialize モードで動作するため、タプルは Tuplestore を介して返却される点に注意すること。
