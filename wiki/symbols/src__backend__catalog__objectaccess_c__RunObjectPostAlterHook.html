RunObjectPostAlterHook

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:91

目的

データベースオブジェクトの変更（ALTER）操作が完了した直後に、登録された外部フック関数を呼び出し、監査やセキュリティ拡張機能へ通知を行う。

シグネチャまたは形状

void RunObjectPostAlterHook(Oid classId, Oid objectId, int subId, Oid auxiliaryId, bool is_internal)

入力と出力

  • **入力**:
  • classId: 変更されたオブジェクトのカタログOID。
  • objectId: 変更されたオブジェクトのOID。
  • subId: サブオブジェクトID（列番号など）。
  • auxiliaryId: 変更に関連する補助的なOID。
  • is_internal: 内部的な変更かどうかを示すフラグ。
  • **出力**: なし（void）。

前提条件と不変条件

  • グローバル変数 object_access_hook が非NULLであること（Assertにより検証）。
  • フック関数が登録されていない状態で呼び出された場合、プログラムは異常終了する。

アルゴリズムまたは振る舞い

1. ObjectAccessPostAlter 構造体をスタック上に確保し、ゼロクリアする。 2. 引数 auxiliaryId および is_internal を構造体に格納する。 3. グローバル関数ポインタ object_access_hook を介して、イベントタイプ OAT_POST_ALTER を指定し、構築した構造体およびオブジェクト識別子を渡してフック関数を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し元は、提供された抜粋内では特定されていません。
  • **呼び出し先**: グローバル変数 object_access_hook に登録された関数。

関連データ構造

  • ObjectAccessPostAlter: 変更操作のメタデータを保持する構造体。
  • OAT_POST_ALTER: フックに渡されるイベントタイプ定数。

失敗モードと境界条件

  • object_access_hook が NULL の場合、Assert によりアサーション失敗が発生する。

保守担当者向けメモ

  • 本関数はフックの実行エントリーポイントであり、実際の処理はロードされたプラグイン側に依存します。
  • ...Str で終わる別関数（RunObjectPostAlterHookStr）が存在するため、OIDではなく名前ベースでの識別が必要な場合はそちらを検討してください。
