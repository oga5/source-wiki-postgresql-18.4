dblink_get_connections

定義

  • 種別: function
  • 定義箇所: contrib/dblink/dblink.c:1297

目的

現在開いているすべての dblink 接続の名前を配列として取得する。

シグネチャまたは形状

Datum dblink_get_connections(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL レベルからの呼び出し引数）。
  • **出力**: Datum 型。現在アクティブな接続名（text 型）の配列を返す。接続が存在しない場合は NULL を返す。

前提条件と不変条件

  • remoteConnHash が初期化されている必要がある。
  • 接続名として取得されるのは、remoteConn 構造体内の conn フィールドが NULL でない（接続が確立されている）もののみである。

アルゴリズムまたは振る舞い

1. remoteConnHash が存在する場合、ハッシュテーブルのイテレータ（HASH_SEQ_STATUS）を初期化する。 2. hash_seq_search を使用してハッシュテーブル内の全エントリを走査する。 3. 各エントリについて、rconn.conn が有効か確認し、有効であればその接続名（hentry->name）を accumArrayResult を用いて配列構築状態（ArrayBuildState）に蓄積する。 4. 走査終了後、蓄積されたデータがあれば makeArrayResult で配列を生成して返し、なければ PG_RETURN_NULL() を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(dblink_get_connections) マクロにより登録）。
  • **呼び出し先**:
  • hash_seq_init, hash_seq_search（ハッシュテーブル走査用）
  • accumArrayResult, makeArrayResult（配列構築用）
  • CStringGetTextDatum（文字列から Datum への変換用）

関連データ構造

  • remoteConnHash: 接続を管理するハッシュテーブル。
  • remoteConnHashEnt: ハッシュテーブル内の各エントリを保持する構造体。
  • ArrayBuildState: 戻り値の配列を構築するための一時的な状態保持構造体。

失敗モードと境界条件

  • remoteConnHash が初期化されていない場合、関数は何も蓄積せず NULL を返す。
  • 接続が一つも開かれていない場合、astate が NULL のままとなり、結果として NULL が返される。

保守担当者向けメモ

  • 本関数は CurrentMemoryContext を使用して配列を構築している。メモリコンテキストの生存期間に注意すること。
  • 接続名の取得元である remoteConnHashEnt の name フィールドは、ハッシュテーブル登録時に設定された接続名である。
