trivalue

定義

  • 種別: enum
  • 定義箇所: contrib/vacuumlo/vacuumlo.c:34

目的

三値論理（デフォルト、否定、肯定）の状態を表現するための列挙型です。主にパスワードプロンプトの表示制御などの設定値として利用されます。

シグネチャまたは形状

enum trivalue
{
	TRI_DEFAULT,
	TRI_NO,
	TRI_YES,
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: TRI_DEFAULT, TRI_NO, TRI_YES のいずれかの値

前提条件と不変条件

  • 常にこれら3つの値のいずれかをとることを想定しています。

アルゴリズムまたは振る舞い

  • TRI_DEFAULT: システムのデフォルト動作に従う状態。
  • TRI_NO: 明示的に無効化または否定する状態。
  • TRI_YES: 明示的に有効化または肯定する状態。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/vacuumlo/vacuumlo.c 内で pg_prompt 変数の型として参照されています。
  • 呼び出し先: なし（データ型定義のため）。
  • 注記: 他の PostgreSQL コンポーネント（pg_dump, psql 等）でも同名の型が定義・利用されていますが、本ファイルにおける直接的な呼び出し関係は上記変数への適用に限定されます。

関連データ構造

  • _param: vacuumlo ユーティリティの実行オプションを保持する構造体において、関連する設定値として利用される可能性があります。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 本シンボルは PostgreSQL の複数のツール間で共通の概念として定義されています。変更を加える際は、他のツール（pg_dump や psql など）との整合性に注意してください。
