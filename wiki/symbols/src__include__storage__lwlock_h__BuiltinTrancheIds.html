BuiltinTrancheIds

定義

  • 種別: enum
  • 定義箇所: src/include/storage/lwlock.h:181

目的

PostgreSQLの軽量ロック（LWLock）システムにおいて、システム全体で共有される標準的なロックグループ（トランチ: Tranche）を識別するためのIDを定義する。

シグネチャまたは形状

typedef enum BuiltinTrancheIds {
    LWTRANCHE_XACT_BUFFER = NUM_INDIVIDUAL_LWLOCKS,
    /* ... 各種トランチID ... */
    LWTRANCHE_FIRST_USER_DEFINED,
} BuiltinTrancheIds;

入力と出力

  • 入力: なし（列挙型定数）
  • 出力: LWLock 構造体の tranche フィールドに格納される識別子として機能する。

前提条件と不変条件

  • LWTRANCHE_XACT_BUFFER は NUM_INDIVIDUAL_LWLOCKS から開始される。
  • LWTRANCHE_FIRST_USER_DEFINED は列挙型の末尾に配置されており、これ以降のIDは動的に割り当てられるユーザー定義トランチのために予約されている。

アルゴリズムまたは振る舞い

  • LWLockの統計情報取得やデバッグ、およびロック競合時の管理において、特定のロックがどの機能グループに属するかを識別するために使用される。
  • LWLock 構造体は、この列挙型で定義されたいずれかのIDを保持することで、そのロックの用途（例: LWTRANCHE_WAL_INSERT や LWTRANCHE_BUFFER_MAPPING など）を明示する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/lwlock.c 内のロック管理ロジックが、これらのIDを参照してロックの初期化や統計処理を行う。
  • 呼び出し先: 情報なし。

関連データ構造

  • LWLock: この列挙型で定義されたIDを tranche フィールドに保持する。
  • NamedLWLockTranche: 動的に割り当てられるトランチを管理するための構造体。

失敗モードと境界条件

  • 境界条件: LWTRANCHE_FIRST_USER_DEFINED を超えるIDを静的に追加することはできない。動的なトランチ割り当てが必要な場合は、この列挙型を拡張するのではなく、動的トランチ管理機構を使用する必要がある。

保守担当者向けメモ

  • 新しいサブシステムで独自のLWLockが必要な場合、この列挙型に安易に追加せず、動的トランチ割り当ての利用を検討すること。
  • lwlock.c 内のロック名定義とこの列挙型の順序が一致している必要がある。
