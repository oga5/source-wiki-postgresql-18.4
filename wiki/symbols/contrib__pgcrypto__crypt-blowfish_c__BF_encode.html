BF_encode

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/crypt-blowfish.c:403

目的

バイナリデータを bcrypt アルゴリズム特有の Base64 エンコーディング形式（BF_itoa64 テーブルに基づく）に変換します。

シグネチャまたは形状

void BF_encode(char *dst, const BF_word *src, int size)

入力と出力

  • dst: エンコード結果を格納するバッファへのポインタ。
  • src: エンコード対象のバイナリデータ（BF_word 配列）。
  • size: エンコードするバイト数。

前提条件と不変条件

  • dst は、変換後の文字列を格納するために十分なメモリ領域を確保している必要があります。
  • BF_itoa64 テーブルがスコープ内で利用可能である必要があります。

アルゴリズムまたは振る舞い

1. 入力データを 3 バイト単位で処理し、6 ビットずつに分割して BF_itoa64 テーブルのインデックスとして使用します。 2. 3 バイト（24 ビット）を 4 文字の Base64 文字に変換します。 3. 入力データの末尾で 3 バイトに満たない場合、残りのビットを適切にシフトしてエンコードを完了します。 4. do-while ループにより、size バイト分がすべて処理されるまで継続します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/crypt-blowfish.c:747
  • contrib/pgcrypto/crypt-gensalt.c:124
  • contrib/pgcrypto/crypt-gensalt.c:183
  • **呼び出し先**:
  • 本関数内での明示的な関数呼び出しはありません。BF_itoa64 テーブルを参照します。

関連データ構造

  • BF_word: エンコード対象のデータを保持する型。
  • BF_itoa64: エンコードに使用される変換テーブル。

失敗モードと境界条件

  • size が 0 の場合、ループは実行されず何も出力されません。

保守担当者向けメモ

  • 本関数は bcrypt の仕様に基づいた特殊な Base64 エンコードを行っており、標準的な Base64 とは変換テーブルが異なります。
  • 移植性を考慮し、エンディアンに依存しないバイト単位のアクセスを行っています。
