pg_sha256_init

定義

  • 種別: function
  • 定義箇所: src/common/sha2.c:278

目的

SHA-256 ハッシュ計算のためのコンテキスト構造体を初期化し、計算開始可能な状態にリセットします。

シグネチャまたは形状

void pg_sha256_init(pg_sha256_ctx *context)

入力と出力

  • **入力**: pg_sha256_ctx *context（初期化対象のコンテキスト構造体へのポインタ）
  • **出力**: なし（副作用として context が書き換えられます）

前提条件と不変条件

  • context が NULL の場合、関数は何も行わず直ちに復帰します。
  • 正常な呼び出し後、context->state は SHA-256 の初期ハッシュ値で満たされ、context->buffer はゼロクリアされ、context->bitcount は 0 に設定されます。

アルゴリズムまたは振る舞い

1. context が NULL かどうかを確認し、該当する場合は終了します。 2. sha256_initial_hash_value を context->state にコピーします。 3. context->buffer を PG_SHA256_BLOCK_LENGTH 分だけゼロで埋めます。 4. context->bitcount を 0 にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash.c:117 にて呼び出しを確認しています。
  • **呼び出し先**: 標準ライブラリの memcpy および memset を使用します。

関連データ構造

  • pg_sha256_ctx: SHA-256 計算の状態を保持する構造体（src/common/sha2_int.h で定義）。

失敗モードと境界条件

  • context に NULL ポインタが渡された場合、安全に無視されます。

保守担当者向けメモ

  • 本関数は Aaron D. Gifford 氏による実装に基づいています。
  • コンテキストの初期化は、ハッシュ計算の更新（pg_sha256_update）や最終化（pg_sha256_final）の前に必ず実行する必要があります。
