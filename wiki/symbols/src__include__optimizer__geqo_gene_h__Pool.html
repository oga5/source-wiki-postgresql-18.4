Pool

定義

  • 種別: struct
  • 定義箇所: src/include/optimizer/geqo_gene.h:38

目的

遺伝的クエリ最適化（GEQO）において、遺伝的アルゴリズムの世代交代に使用される個体群（染色体の集合）を管理するための構造体です。

シグネチャまたは形状

typedef struct Pool
{
	Chromosome *data;
	int			size;
	int			string_length;
} Pool;

入力と出力

  • data: 染色体（Chromosome）の配列へのポインタ。
  • size: プール内に保持されている染色体の総数。
  • string_length: 各染色体が保持する遺伝子配列の長さ。

前提条件と不変条件

  • data は size 分の Chromosome 構造体を保持するメモリ領域を指している必要があります。
  • string_length は、GEQOの探索対象となるクエリの結合順序の長さに一致します。

アルゴリズムまたは振る舞い

本構造体はデータ保持に特化しており、以下の操作の対象となります：

  • geqo_pool.c による初期化、ソート、および解放。
  • geqo_misc.c による統計情報の算出（avg_pool）や内容の出力（print_pool, print_gen）。
  • 遺伝的アルゴリズムの交叉や突然変異の過程で、個体群の評価や選択の対象として参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/optimizer/geqo/geqo_main.c
  • src/backend/optimizer/geqo/geqo_misc.c
  • src/backend/optimizer/geqo/geqo_pool.c
  • src/backend/optimizer/geqo/geqo_selection.c
  • **呼び出し先**:
  • 本構造体はデータ構造定義であり、関数呼び出しは含みません。

関連データ構造

  • Chromosome: Pool 内の data 配列の各要素として格納される個体データ。

失敗モードと境界条件

  • size が 0 の場合、プールに対する操作（ソートや統計計算）は無効またはエラーとなる可能性があります。
  • メモリ割り当て（palloc）が失敗した場合、Pool の初期化は継続できません。

保守担当者向けメモ

  • Pool の管理ロジック（初期化・破棄）は src/include/optimizer/geqo_pool.h に定義されています。
  • Gene 型（int）の変更が Chromosome を介して本構造体に波及するため、変更時は関連するすべてのGEQOモジュールを確認してください。
