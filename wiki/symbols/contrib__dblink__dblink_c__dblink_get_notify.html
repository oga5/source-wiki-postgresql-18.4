dblink_get_notify

定義

  • 種別: function
  • 定義箇所: contrib/dblink/dblink.c:1898

目的

リモートデータベースから受信した非同期通知（NOTIFY）を取得し、PostgreSQL のセットを返す関数（SRF）として結果セットを生成する。

シグネチャまたは形状

Datum dblink_get_notify(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 引数なし（デフォルト接続を使用）または接続名（text 型）。
  • **出力**:
  • (notify_name text, be_pid int4, extra text) 形式のタプルセット。

前提条件と不変条件

  • dblink_init() が呼び出され、内部状態が初期化されていること。
  • InitMaterializedSRF により、結果を格納するための Tuplestore が準備されていること。

アルゴリズムまたは振る舞い

1. 接続名が指定されていれば dblink_get_named_conn で取得し、指定がなければデフォルト接続を使用する。 2. PQconsumeInput を呼び出し、リモートからの通知入力を読み込む。 3. PQnotifies をループで呼び出し、受信した通知を順次処理する。 4. 各通知の relname（通知名）、be_pid（送信元PID）、extra（ペイロード）を抽出し、tuplestore_putvalues を用いて結果セットに格納する。 5. PQfreemem で通知メモリを解放し、次の通知を読み込むために再度 PQconsumeInput を実行する。 6. 最終的に (Datum) 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(dblink_get_notify) マクロにより SQL インターフェースとして登録されている。
  • **呼び出し先**:
  • dblink_init
  • dblink_get_named_conn
  • InitMaterializedSRF
  • PQconsumeInput (libpq)
  • PQnotifies (libpq)
  • tuplestore_putvalues
  • PQfreemem (libpq)

関連データ構造

  • PGconn: libpq の接続オブジェクト。
  • PGnotify: libpq の通知構造体。
  • ReturnSetInfo: SRF の結果セット情報を保持する構造体。

失敗モードと境界条件

  • 接続が確立されていない場合、dblink_get_named_conn 等でエラーが発生する可能性がある。
  • 通知の各フィールド（relname, extra）が NULL の場合、適切に NULL フラグを立てて処理する。

保守担当者向けメモ

  • 本関数は Tuplestore を使用して結果をマテリアライズするため、大量の通知が溜まっている場合はメモリ消費に注意が必要である。
  • PQconsumeInput をループ内で呼び出すことで、通知キューを空になるまで処理する設計となっている。
