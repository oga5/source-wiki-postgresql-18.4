test_gb18030_page_multiple

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_escape/test_escape.c:179

目的

GB18030エンコーディングにおけるマルチバイト文字の境界条件およびメモリ安全性を検証するためのテストを実行する。

シグネチャまたは形状

static void test_gb18030_page_multiple(pe_test_config *tc)

入力と出力

  • 入力: pe_test_config *tc（データベース接続情報を含む設定構造体）
  • 出力: なし（内部で report_result を呼び出し、テスト結果を標準出力へ記録する）

前提条件と不変条件

  • tc->conn が有効なデータベース接続を保持していること。
  • input_len として 0x20000 バイトのメモリが確保可能であること。

アルゴリズムまたは振る舞い

1. **入力準備**: 0x20000 バイトのバッファを確保し、末尾に GB18030 のマルチバイト文字の開始バイトとして機能し得る 0xfe を配置する。 2. **テスト名生成**: PQExpBuffer を用いてテストケースの説明文字列を構築する。 3. **環境設定**: PQsetClientEncoding を呼び出し、クライアントエンコーディングを GB18030 に設定する。 4. **実行と検証**: PQescapeLiteral を使用してエスケープ処理を行い、その結果が NULL になるかどうかを report_result で検証する。 5. **クリーンアップ**: 使用したバッファおよび PQExpBuffer を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/test_escape/test_escape.c 内の main 関数（またはテスト実行ループ）から呼び出される。
  • 呼び出し先: pg_malloc, memset, createPQExpBuffer, appendPQExpBuffer, escapify, PQsetClientEncoding, report_result, PQescapeLiteral, destroyPQExpBuffer, pg_free。

関連データ構造

  • pe_test_config: テスト実行に必要な接続情報等を保持する構造体。
  • PQExpBuffer: テスト結果の報告用文字列を構築するために使用されるバッファ構造体。

失敗モードと境界条件

  • PQescapeLiteral が予期せず NULL を返す場合、またはメモリ確保に失敗した場合にテストが失敗する。
  • GB18030 のマルチバイト境界（末尾の 0xfe）が正しく処理されない場合、メモリ安全性の問題やエスケープの失敗が発生する可能性がある。

保守担当者向けメモ

  • 本関数は、GB18030 のようなマルチバイトエンコーディング特有の境界条件をテストするために設計されている。
  • input_len を 0x20000 としているのは、メモリ境界を意図的に跨ぐケースをシミュレートするためである。変更時はメモリ割り当て量に注意すること。
