sepgsql_attribute_setattr

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/relation.c:208

目的

リレーション内のカラムに対する属性変更操作において、SELinuxの db_column:{setattr} 権限を検証する。

シグネチャまたは形状

void sepgsql_attribute_setattr(Oid relOid, AttrNumber attnum)

入力と出力

  • **入力**:
  • relOid (Oid): 対象リレーションのOID。
  • attnum (AttrNumber): 属性変更対象のカラム番号。
  • **出力**: なし（権限チェック失敗時は例外をスロー）。

前提条件と不変条件

  • 対象リレーションが通常のテーブル（RELKIND_RELATION）またはパーティションテーブル（RELKIND_PARTITIONED_TABLE）であること。それ以外の場合は何もせず復帰する。

アルゴリズムまたは振る舞い

1. get_rel_relkind を呼び出し、対象リレーションの種類を確認する。 2. 対象がテーブルまたはパーティションテーブルでない場合、処理を終了する。 3. ObjectAddress 構造体に RelationRelationId、relOid、attnum を設定する。 4. getObjectIdentity を用いて、監査ログ用のオブジェクト識別名を取得する。 5. sepgsql_avc_check_perms を呼び出し、SEPG_CLASS_DB_COLUMN クラスに対する SEPG_DB_COLUMN__SETATTR 権限を検証する。 6. 取得した識別名を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:240
  • **呼び出し先**:
  • get_rel_relkind
  • getObjectIdentity
  • sepgsql_avc_check_perms
  • pfree

関連データ構造

  • ObjectAddress: 権限チェック対象のオブジェクトを特定するために使用。
  • RelationRelationId: システムカタログ pg_class を指す定数。

失敗モードと境界条件

  • 対象リレーションがテーブルやパーティションテーブル以外の場合、権限チェックはスキップされる。
  • SELinuxポリシーにより setattr 権限が拒否された場合、sepgsql_avc_check_perms 内でエラーが発生し、トランザクションが中断される可能性がある。

保守担当者向けメモ

  • 本関数は、カラムに対するメタデータ変更（ALTER TABLE ... ALTER COLUMN ... 等）のフックとして機能する。
  • インデックス等の特殊なリレーションに対する操作は、本関数ではなく sepgsql_index_modify 等の専用パスを経由する場合があるため注意すること。
