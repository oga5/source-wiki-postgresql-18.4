tarWrite

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_tar.c:528

目的

TARアーカイブ内の特定のメンバー（TAR_MEMBER）に対して、指定されたバッファの内容を書き込み、内部の書き込み位置情報を更新する。

シグネチャまたは形状

static size_t tarWrite(const void *buf, size_t len, TAR_MEMBER *th)

入力と出力

  • **入力**:
  • buf: 書き込むデータが格納されたメモリ領域。
  • len: 書き込むデータのバイト数。
  • th: 書き込み先となる TAR_MEMBER 構造体へのポインタ。
  • **出力**:
  • size_t: 実際に書き込まれたバイト数。

前提条件と不変条件

  • th->nFH は有効なファイルハンドルとして初期化されている必要がある。
  • th->pos は、書き込み成功後に書き込まれたバイト数分だけ正確にインクリメントされる。

アルゴリズムまたは振る舞い

1. fwrite を使用して、buf から len バイトのデータを th->nFH が指すファイルストリームへ書き込む。 2. 書き込みによって返されたバイト数を res に格納する。 3. th->pos に res を加算し、現在の書き込み位置を更新する。 4. res を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_tar.c 内の各所（データ書き込み、ヘッダー書き込み、スクリプト生成処理など）。
  • **呼び出し先**:
  • 標準ライブラリ関数 fwrite。

関連データ構造

  • TAR_MEMBER: TARアーカイブ内のメンバー情報を保持する構造体。ファイルハンドル（nFH）と現在の位置情報（pos）を含む。

失敗モードと境界条件

  • fwrite が len 未満の値を返した場合、書き込み処理は部分的に完了した状態となる。呼び出し元で戻り値を確認し、エラーハンドリングを行う必要がある（ソース内の呼び出し箇所では、戻り値と len を比較して検証している）。

保守担当者向けメモ

  • 本関数は pg_dump のTAR形式出力における低レベルな書き込みラッパーである。
  • 圧縮処理はサポートされていないため、直接ファイルストリームへの書き込みを行っている。
