ChooseConstraintName

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_constraint.c:512

目的

指定された名前空間において、他の制約名と重複しない一意な制約名を生成する。

シグネチャまたは形状

char *ChooseConstraintName(const char *name1, const char *name2,
                           const char *label, Oid namespaceid,
                           List *others)

入力と出力

  • **入力**:
  • name1, name2: 名前生成のベースとなる文字列（通常はテーブル名など）。
  • label: 制約の識別ラベル。
  • namespaceid: 制約が属する名前空間の OID。
  • others: 現在の処理で既に生成済み、または考慮すべき制約名のリスト。
  • **出力**:
  • 重複しない一意な制約名（palloc された文字列）。

前提条件と不変条件

  • pg_constraint カタログに対する AccessShareLock を内部で取得・解放する。
  • label が空文字列の場合、サフィックスとしてカウンタ値が付与される。
  • 生成された名前は、others リストに含まれておらず、かつ pg_constraint カタログ内に存在しないことが保証される。

アルゴリズムまたは振る舞い

1. pg_constraint テーブルを AccessShareLock で開く。 2. makeObjectName を使用して、name1, name2, label から候補名を生成する。 3. others リスト内に候補名が存在するか確認する。 4. リストに存在しない場合、ConstraintNameNspIndexId インデックスを使用して pg_constraint をスキャンし、カタログ内に同名の制約が存在するか確認する。 5. 重複が検出された場合、label にカウンタ（pass）を付与して名前を再生成し、重複がなくなるまで繰り返す。 6. 最後に table_close でロックを解放し、決定した名前を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/commands/tablecmds.c
  • src/backend/commands/typecmds.c
  • **呼び出し先**:
  • table_open / table_close
  • makeObjectName
  • systable_beginscan / systable_getnext / systable_endscan
  • ScanKeyInit

関連データ構造

  • Relation: pg_constraint カタログへのアクセスに使用。
  • SysScanDesc: カタログスキャン状態の保持。
  • ScanKeyData: インデックススキャンの検索条件。

失敗モードと境界条件

  • label が空文字列の場合、初期状態からカウンタが付与される。
  • others リストが空の場合、カタログ内の検索のみで一意性を判断する。
  • メモリ不足時は palloc 等によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は pg_constraint の名前空間内での一意性を保証する重要なユーティリティである。
  • ConstraintNameIsUsed や ConstraintNameExists とは異なるロジックで一意性を判定しているため、変更時は整合性に注意すること。
