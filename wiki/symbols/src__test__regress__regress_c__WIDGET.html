WIDGET

定義

  • 種別: struct
  • 定義箇所: src/test/regress/regress.c:163

目的

PostgreSQL の回帰テストスイートにおいて、カスタムデータ型として利用されるテスト用構造体です。

シグネチャまたは形状

typedef struct
{
	Point		center;
	double		radius;
} WIDGET;

入力と出力

本構造体はデータ型として定義されており、SQL レベルの関数（widget_in, widget_out 等）を通じて Datum 型との相互変換が行われます。

前提条件と不変条件

  • Point 型の center と double 型の radius を保持するメモリレイアウトを持ちます。
  • テスト用データ型であるため、本番環境での利用は想定されていません。

アルゴリズムまたは振る舞い

  • palloc を用いたメモリ確保の対象として使用されます。
  • SQL 関数内で PG_GETARG_POINTER を介してポインタとして取得され、内部フィールドの読み書きが行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/regress/regress.c 内の各テスト関数（widget_in, widget_out, pt_in_widget 等）。
  • 呼び出し先: 情報なし。

関連データ構造

  • Point: WIDGET の中心座標を保持するために使用される構造体。

失敗モードと境界条件

  • palloc によるメモリ確保失敗時には PostgreSQL のエラーハンドリング機構により elog(ERROR) が発生します。

保守担当者向けメモ

  • 本構造体は PostgreSQL の内部データ構造の挙動を検証するためのサンプルとして機能します。
  • 構造体の変更は、関連する SQL テスト関数（widget_in 等）の入出力ロジックに影響を与えるため、適宜修正が必要です。
