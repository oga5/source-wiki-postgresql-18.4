ExecEndGatherMerge

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:290

目的

GatherMerge 実行ノードの終了処理を行い、関連する子ノードのリソース解放および並列ワーカーの停止を管理する。

シグネチャまたは形状

void ExecEndGatherMerge(GatherMergeState *node)

入力と出力

  • 入力: GatherMergeState *node （終了対象となる GatherMerge ノードの状態構造体）
  • 出力: なし（void）

前提条件と不変条件

  • 引数 node は有効な GatherMergeState 構造体を指している必要がある。
  • 本関数は GatherMerge ノードの実行終了時に呼び出される。

アルゴリズムまたは振る舞い

1. outerPlanState(node) を介して子ノードの終了処理（ExecEndNode）を先に実行し、子ノードが保持するリソースを解放する。 2. ExecShutdownGatherMerge(node) を呼び出し、並列ワーカーの停止および関連する並列実行リソースのクリーンアップを行う。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execProcnode.c:636
  • 呼び出し先:
  • ExecEndNode （子ノードの終了）
  • ExecShutdownGatherMerge （並列リソースの解放）

関連データ構造

  • GatherMergeState: GatherMerge ノードの実行状態を保持する構造体。

失敗モードと境界条件

  • 供給されたソースコードからは、特定の失敗モードや境界条件に関する明示的な記述は確認できない。

保守担当者向けメモ

  • 本関数は GatherMerge ノードの終了シーケンスの起点である。子ノードのクリーンアップが先に行われる順序に注意すること。
  • ExecShutdownGatherMerge はワーカーの停止を担うため、並列実行環境の整合性を維持するために重要である。
