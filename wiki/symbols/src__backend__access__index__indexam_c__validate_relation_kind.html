validate_relation_kind

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/indexam.c:196

目的

指定されたリレーションがインデックスとして有効な種別（RELKIND_INDEX または RELKIND_PARTITIONED_INDEX）であるかを検証し、不適切な場合にエラーを報告する。

シグネチャまたは形状

static inline void validate_relation_kind(Relation r)

入力と出力

  • **入力**: Relation r（検証対象のリレーション記述子）
  • **出力**: なし（正常時は何も返さない）
  • **副作用**: 検証失敗時に ereport(ERROR, ...) を呼び出し、トランザクションを中断させる。

前提条件と不変条件

  • 入力される Relation r は有効なポインタである必要がある。
  • r->rd_rel が初期化されていることが前提となる。

アルゴリズムまたは振る舞い

1. r->rd_rel->relkind を参照し、その値が RELKIND_INDEX または RELKIND_PARTITIONED_INDEX のいずれかと一致するかを確認する。 2. いずれにも一致しない場合、ERRCODE_WRONG_OBJECT_TYPE エラーを発生させ、対象がインデックスではない旨のメッセージを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/index/indexam.c 内の複数の箇所で、インデックス操作の直前に呼び出されている。
  • **呼び出し先**: RelationGetRelationName（マクロ）、ereport（エラー報告関数）。
  • **注意**: 同名の関数が src/backend/access/sequence/sequence.c および src/backend/access/table/table.c にも存在し、それぞれのモジュールで同様の検証を行っている。

関連データ構造

  • Relation: リレーションのメタデータと状態を保持する構造体。
  • Form_pg_class: r->rd_rel が指す構造体で、リレーションの種別（relkind）を保持する。

失敗モードと境界条件

  • **失敗モード**: 対象リレーションがインデックスまたはパーティション化されたインデックスではない場合、ERROR レベルの例外が発生する。
  • **境界条件**: r が NULL の場合、あるいは r->rd_rel が未初期化の場合は未定義の動作（セグメンテーション違反等）を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、インデックスアクセス層のガードとして機能する。
  • 他のモジュール（sequence, table）にも同名の関数が存在するため、修正を行う際は各モジュールの責務範囲を混同しないよう注意すること。
