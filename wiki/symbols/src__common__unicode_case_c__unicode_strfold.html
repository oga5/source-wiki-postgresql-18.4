unicode_strfold

定義

  • 種別: function
  • 定義箇所: src/common/unicode_case.c:188

目的

Unicode 文字列に対してケースフォールディング（正規化のための比較用変換）を適用する。

シグネチャまたは形状

size_t unicode_strfold(char *dst, size_t dstsize, const char *src, ssize_t srclen, bool full);

入力と出力

  • dst: 変換後の文字列を格納するバッファ。
  • dstsize: dst バッファのサイズ（バイト単位）。0 の場合は必要なバッファサイズの計算のみを行う。
  • src: 変換対象の UTF-8 文字列。
  • srclen: 入力文字列の長さ（バイト単位）。-1 の場合はヌル終端文字列として扱う。
  • full: true の場合、文脈依存の特殊マッピング（Special Casing）を適用する。
  • 戻り値: 変換後の文字列に必要なバッファサイズ（バイト単位）。

前提条件と不変条件

  • 入力文字列 src は UTF-8 エンコーディングであること。
  • dstsize が 0 の場合、関数は変換を行わず、必要なサイズのみを返す。

アルゴリズムまたは振る舞い

内部関数 convert_case を呼び出し、CaseFold モードで文字列を走査する。各文字に対して casemap を実行し、full フラグに応じて check_special_conditions を介した特殊マッピングを適用する。変換結果は UTF-8 として dst に書き込まれる。

関連データ構造

  • CaseMapResult（変換結果の状態を示す列挙型）

失敗モードと境界条件

  • dstsize が変換後の文字列を格納するのに不十分な場合、バッファオーバーフローを防ぐための切り詰めやエラー処理は convert_case の実装に依存する。
  • 入力文字列が不正な UTF-8 シーケンスを含む場合の挙動については、本関数単体では定義されていない。

保守担当者向けメモ

  • 本関数は convert_case のラッパーであり、ケースフォールディング専用のインターフェースを提供する。
  • ギリシャ語の最終シグマ処理などの特殊ケースは check_special_conditions を通じて処理されるため、ロジックの変更時は unicode_case.c 内の関連テーブルおよび判定関数を確認すること。
