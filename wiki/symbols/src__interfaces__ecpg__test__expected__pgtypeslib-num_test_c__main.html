目的

ECPG（Embedded SQL for C）の pgtypes ライブラリにおける数値型（numeric）操作およびデータベース連携機能の回帰テストを実行するエントリポイント。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しない）
  • 出力: 成功時に 0 を返却。標準出力にテスト結果（数値演算結果や変換結果）を出力し、標準エラー出力にSQLトレースを出力する。

前提条件と不変条件

  • 実行環境において ecpg1_regression データベースへの接続が可能であること。
  • EXEC SQL 文が正しくプリプロセスされ、ECPGランタイムライブラリがリンクされていること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug でデバッグ出力を有効化し、データベースへ接続。テスト用テーブル test を作成。 2. **数値演算**: PGTYPESnumeric_from_asc 等を用いてメモリ上で加算・減算・乗算・除算を行い、結果を検証。 3. **DB連携**: INSERT 文で numeric 型変数をデータベースへ格納し、SELECT 文で再取得して整合性を確認。 4. **型変換**: PGTYPESnumeric_to_int や to_long を使用し、C言語の整数型への変換および境界値（INT_MIN, INT_MAX）の挙動を検証。 5. **終了処理**: トランザクションのロールバックを行い、データベース接続を切断して終了。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGtrans, ECPGdisconnect（ECPGランタイム）、PGTYPESnumeric_* 系関数（数値操作）、print_double（ヘルパー関数）、printf（標準出力）

関連データ構造

  • numeric: pgtypes ライブラリで定義される数値型構造体。
  • sqlca: SQL通信領域（エラーチェックに使用）。

失敗モードと境界条件

  • SQL操作でエラーが発生した場合、sqlprint() が呼び出され、エラー内容が標準エラー出力に表示される。
  • 数値変換において、INT_MIN や INT_MAX を超える値の変換境界をテストしている。

保守担当者向けメモ

  • 本関数はプリプロセッサ生成コードであるため、直接編集せず元の .pgc ファイルを修正すること。
  • print_double 関数はプラットフォーム依存の浮動小数点数表現を正規化するために使用されている。
