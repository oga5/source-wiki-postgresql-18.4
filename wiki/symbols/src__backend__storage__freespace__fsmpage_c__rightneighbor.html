rightneighbor

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/fsmpage.c:36

目的

FSM（空き領域マップ）ページ内のバイナリツリー構造において、現在のノードから右隣のノードへ移動するためのインデックス計算を行う。

シグネチャまたは形状

static int rightneighbor(int x)

入力と出力

  • **入力**: x (現在のノードインデックス)
  • **出力**: x の右隣に位置するノードのインデックス。

前提条件と不変条件

  • 入力 x は FSM ページ内の有効なノードインデックスであること。
  • ツリー構造の各レベルにおける左端ノードは 2^level - 1 で表される。

アルゴリズムまたは振る舞い

1. x をインクリメントして右方向へ移動する。 2. 移動後の x が現在のレベルの右端を超え、次のレベルの左端ノードに到達したかを判定する。

  • 判定式 ((x + 1) & x) == 0 を用いて、x + 1 が 2 のべき乗であるか（＝レベルの境界であるか）を確認する。

    • 3. 境界に達している場合、parentof(x) を呼び出して親ノードのインデックスへ補正する。 4. 計算されたノードインデックスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/freespace/fsmpage.c 内のコードから呼び出されている（例: nodeno = parentof(rightneighbor(nodeno));）。
  • **呼び出し先**: parentof マクロ。

関連データ構造

  • FSMPage (ページ内のバイナリツリー構造)

失敗モードと境界条件

  • ツリーの最右端ノードで呼び出された場合、ツリー構造の定義に従い親ノードへ遷移する。
  • 入力値が不正な場合、計算結果がツリーの範囲外となる可能性がある。

保守担当者向けメモ

  • 本関数は fsmpage.c 内のツリー走査ロジックの基盤となる。
  • ((x + 1) & x) == 0 は 2 のべき乗を判定するビット演算の定石であり、変更時はツリー構造の数学的定義との整合性に注意すること。
