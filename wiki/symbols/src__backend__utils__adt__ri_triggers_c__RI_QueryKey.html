RI_QueryKey

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/ri_triggers.c:142

目的

参照整合性（RI）制約の検証やアクション実行のためにキャッシュされたSPI（Server Programming Interface）プランを識別するためのキーとして使用されます。

シグネチャまたは形状

typedef struct RI_QueryKey
{
	Oid			constr_id;		/* pg_constraint エントリのOID */
	int32		constr_queryno; /* クエリタイプID (RI_PLAN_XXX) */
} RI_QueryKey;

入力と出力

本構造体はデータ構造であり、関数ではありません。ri_BuildQueryKey 関数によって、特定の制約IDとクエリ種別から本構造体が生成されます。

前提条件と不変条件

  • constr_id は pg_constraint システムカタログ内の有効な制約を指している必要があります。
  • constr_queryno は、ri_triggers.c 内で定義された RI_PLAN_XXX 定数群のいずれかである必要があります。

アルゴリズムまたは振る舞い

本構造体は、ハッシュテーブルのキーとして機能します。ri_FetchPreparedPlan などの関数において、このキーを用いてキャッシュ済みのSPIプランを検索し、クエリの再構築コストを削減するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ri_BuildQueryKey が本構造体を初期化します。また、RI_FKey_check 関連のトリガー関数（RI_FKey_check_ins 等）や、ri_PerformCheck などの内部関数が、プラン検索やハッシュ操作のために本構造体を生成・使用します。
  • **呼び出し先**: 本構造体自体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • RI_QueryHashEntry: 本構造体をキーとして保持するハッシュテーブルのエントリ。
  • SPIPlanPtr: 本構造体によって検索・管理される実行可能なクエリプラン。

失敗モードと境界条件

  • 無効な constr_id を指定した場合、キャッシュ検索が失敗し、プランの再生成が必要となります。
  • constr_queryno に未定義の値を設定した場合、ハッシュテーブル内での不一致や予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は ri_triggers.c 内のキャッシュ機構の根幹です。新しいRIアクションやクエリタイプを追加する際は、constr_queryno の定義と ri_BuildQueryKey のロジックを整合させる必要があります。
