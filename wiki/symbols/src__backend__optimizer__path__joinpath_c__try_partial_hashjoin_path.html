try_partial_hashjoin_path

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinpath.c:1298

目的

並列クエリ実行において、ハッシュ結合（Hash Join）を実行するための部分パス（Partial Path）を生成し、コスト評価を経て有効であれば結合リレーションのパスリストに追加する。

シグネチャまたは形状

static void try_partial_hashjoin_path(PlannerInfo *root,
                                      RelOptInfo *joinrel,
                                      Path *outer_path,
                                      Path *inner_path,
                                      List *hashclauses,
                                      JoinType jointype,
                                      JoinPathExtraData *extra,
                                      bool parallel_hash)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • joinrel: 結合対象のリレーション情報。
  • outer_path, inner_path: 結合の外部および内部パス。
  • hashclauses: ハッシュ結合に使用する句のリスト。
  • jointype: 結合タイプ（INNER, LEFT, SEMI 等）。
  • extra: 結合に必要なメタデータ（restrictlist 等）。
  • parallel_hash: パラレルハッシュを使用するかどうかのフラグ。
  • **出力**: なし（成功時に joinrel->partial_pathlist を更新）。

前提条件と不変条件

  • joinrel に横方向参照（Lateral references）が含まれていないこと（joinrel->lateral_relids が空であること）。
  • outer_path が外部リレーションに依存していないこと。
  • inner_path がパラメータ化されていないこと（パラメータ化された部分パスはサポート外）。

アルゴリズムまたは振る舞い

1. **検証**: 入力パスのパラメータ化状態をチェックし、条件を満たさない場合は即座に終了する。 2. **コスト見積もり**: initial_cost_hashjoin を呼び出し、結合コストの概算を行う。 3. **事前チェック**: add_partial_path_precheck を呼び出し、コストが既存のパスと比較して許容範囲内かを確認する。 4. **パス生成と登録**: create_hashjoin_path でパスノードを生成し、add_partial_path を通じて joinrel に登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinpath.c 内の結合パス生成ロジック（行2442, 2469付近）。
  • **呼び出し先**:
  • initial_cost_hashjoin
  • add_partial_path_precheck
  • create_hashjoin_path
  • add_partial_path

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • RelOptInfo: 結合リレーションのパスリストを保持。
  • JoinPathExtraData: 結合に必要な追加情報（restrictlist 等）。
  • JoinCostWorkspace: コスト計算結果を一時保持する構造体。

失敗モードと境界条件

  • inner_path がパラメータ化されている場合、処理を中断する。
  • add_partial_path_precheck でコストが劣ると判断された場合、パスの生成は行われない。

保守担当者向けメモ

  • 本関数は並列実行専用のパス生成ロジックである。通常のハッシュ結合パス生成（try_hashjoin_path）とは異なる制約（特にパラメータ化に関する制約）を持つため、変更時は注意が必要。
