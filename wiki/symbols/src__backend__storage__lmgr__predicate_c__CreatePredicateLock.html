CreatePredicateLock

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/predicate.c:2452

目的

指定されたトランザクション（sxact）とロック対象（targettag）を関連付け、述語ロック（SIREADロック）を共有メモリ上のハッシュテーブルに生成・登録します。

シグネチャまたは形状

static void CreatePredicateLock(const PREDICATELOCKTARGETTAG *targettag,
                                uint32 targettaghash,
                                SERIALIZABLEXACT *sxact)

入力と出力

  • **入力**:
  • targettag: ロック対象を識別するタグ。
  • targettaghash: targettag のハッシュ値。
  • sxact: ロックを保持する直列化可能トランザクションの構造体。
  • **出力**: なし（成功時は共有メモリ上のハッシュテーブルを更新）。

前提条件と不変条件

  • SerializablePredicateListLock（共有）および対象パーティションの LWLock（排他）が取得可能であること。
  • 並列モード時は sxact->perXactPredicateListLock（排他）が取得可能であること。
  • 共有メモリが不足していないこと（不足時はエラーをスロー）。

アルゴリズムまたは振る舞い

1. 対象パーティションの LWLock を排他モードで取得し、SerializablePredicateListLock を共有モードで取得する。並列モード時は sxact->perXactPredicateListLock も排他取得する。 2. PredicateLockTargetHash を検索し、対象が存在しない場合は新規作成する。 3. PredicateLockHash を検索し、トランザクションと対象のペアが存在しない場合は新規作成する。 4. 新規作成された場合、target->predicateLocks および sxact->predicateLocks の双方向リストにロックを登録し、commitSeqNo を InvalidSerCommitSeqNo に初期化する。 5. 取得したすべてのロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/predicate.c 内の複数箇所（PredicateLockAcquire 等から呼び出されることが示唆されている）。
  • **呼び出し先**:
  • PredicateLockHashPartitionLock
  • LWLockAcquire / LWLockRelease
  • hash_search_with_hash_value
  • dlist_init / dlist_push_tail
  • PredicateLockHashCodeFromTargetHashCode
  • ereport

関連データ構造

  • PREDICATELOCKTARGET: 述語ロックの対象（リレーション、ページ等）。
  • PREDICATELOCK: トランザクションと対象のペアを保持する構造体。
  • SERIALIZABLEXACT: 直列化可能トランザクションの状態。
  • PredicateLockTargetHash, PredicateLockHash: 共有メモリ上のハッシュテーブル。

失敗モードと境界条件

  • **メモリ不足**: 共有メモリ上のハッシュテーブルへの挿入時に領域が不足した場合、ERRCODE_OUT_OF_MEMORY でエラーを発生させ、max_pred_locks_per_transaction の引き上げを推奨するメッセージを出力する。

保守担当者向けメモ

  • 本関数はロックの生成のみを行い、競合判定は行わない。
  • 並列クエリ実行時には perXactPredicateListLock による保護が必須となるため、呼び出し時のモード判定に注意すること。
