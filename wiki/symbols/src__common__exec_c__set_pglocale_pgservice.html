set_pglocale_pgservice

定義

  • 種別: function
  • 定義箇所: src/common/exec.c:428

目的

実行ファイルの場所に基づいて、アプリケーションのロケール設定（NLS）および設定ディレクトリ（PGSYSCONFDIR）を初期化し、libpq 等のライブラリが参照可能な環境変数を設定する。

シグネチャまたは形状

void set_pglocale_pgservice(const char *argv0, const char *app)

入力と出力

  • argv0: プロセス起動時の argv[0]。実行ファイルのパス特定に使用。
  • app: アプリケーション名（メッセージドメイン名）。
  • 出力: なし（副作用として環境変数およびロケール設定を更新）。

前提条件と不変条件

  • argv0 は実行ファイルのパス解決のために find_my_exec に渡される。
  • app が "postgres" と等しくない場合のみ、setlocale(LC_ALL, "") が実行される。

アルゴリズムまたは振る舞い

1. **ロケール設定**: app が "postgres" でない場合、setlocale(LC_ALL, "") を呼び出し、環境変数に基づくロケールを適用する。 2. **パス特定**: find_my_exec(argv0, ...) を呼び出し、現在の実行ファイルの絶対パスを取得する。失敗した場合は処理を中断する。 3. **NLS設定** (ENABLE_NLS 定義時):

  • get_locale_path でロケールディレクトリを特定し、bindtextdomain および textdomain を設定する。
  • PGLOCALEDIR 環境変数が未設定の場合に設定する。

    • 4. **設定ディレクトリ設定**: PGSYSCONFDIR 環境変数が未設定の場合、get_etc_path で特定したパスを PGSYSCONFDIR に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: postmaster (src/backend/main/main.c) および initdb, pg_config, pg_basebackup 等の各種フロントエンドツール。
  • **呼び出し先**: find_my_exec, setlocale, get_locale_path, bindtextdomain, textdomain, setenv, getenv, get_etc_path。

関連データ構造

  • MAXPGPATH: パス文字列を格納するための固定長バッファサイズ。

失敗モードと境界条件

  • find_my_exec が失敗した場合、ロケールやディレクトリの設定は行われず、関数は早期リターンする。
  • PGSYSCONFDIR が既に環境変数として設定されている場合、この関数による上書きは行われない。

保守担当者向けメモ

  • 本関数はフロントエンドとバックエンドの両方から呼び出されるため、ロケール設定の副作用（特にバックエンドでの LC_ALL 設定回避）には注意が必要である。
  • ENABLE_NLS が定義されていない環境では、ロケール関連のパス設定処理はスキップされる。
