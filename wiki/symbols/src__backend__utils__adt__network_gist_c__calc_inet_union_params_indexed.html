calc_inet_union_params_indexed

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_gist.c:406

目的

複数の GistInetKey を含むインデックスエントリの集合から、それらを包含する統合的なパラメータ（アドレスファミリーの範囲、最小ネットマスク長、共通プレフィックス長）を計算します。

シグネチャまたは形状

static void calc_inet_union_params_indexed(GISTENTRY *ent,
                                           OffsetNumber *offsets, int noffsets,
                                           int *minfamily_p,
                                           int *maxfamily_p,
                                           int *minbits_p,
                                           int *commonbits_p)

入力と出力

  • **入力**:
  • ent: GiSTエントリの配列。
  • offsets: 処理対象となるエントリのインデックス番号配列。
  • noffsets: 処理対象の数。
  • **出力**:
  • minfamily_p, maxfamily_p: 含まれるアドレスファミリーの最小値と最大値。
  • minbits_p: 含まれるキーの最小ネットマスク長。
  • commonbits_p: 含まれる全キー間で共通するプレフィックス長。

前提条件と不変条件

  • noffsets > 0 であること（少なくとも1つのキーが存在すること）。
  • 異なるアドレスファミリー（IPv4とIPv6の混在など）が含まれる場合、minbits および commonbits は強制的に 0 に設定されます。

アルゴリズムまたは振る舞い

1. 最初のキーを使用して minfamily, maxfamily, minbits, commonbits を初期化します。 2. 残りのキーを順次走査し、各値を更新します。

  • minfamily/maxfamily: ファミリー番号の範囲を更新。
  • minbits: 最小値を保持。
  • commonbits: 最小値を保持しつつ、bitncommon 関数を用いてアドレス間の共通ビット数を再計算します。

    • 3. 走査終了後、minfamily != maxfamily であれば、minbits と commonbits を 0 にリセットして返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_gist.c:770 (inet_gist_picksplit 内)
  • src/backend/utils/adt/network_gist.c:780 (inet_gist_picksplit 内)
  • **呼び出し先**:
  • bitncommon (共通ビット数計算用)
  • DatumGetInetKeyP (マクロ)
  • gk_ip_family, gk_ip_minbits, gk_ip_commonbits, gk_ip_addr (アクセサマクロ)

関連データ構造

  • GistInetKey: ネットワーク型GiSTインデックスの内部キー構造体。

失敗モードと境界条件

  • noffsets が 0 の場合、Assert により停止します。
  • 異なるアドレスファミリーが混在する場合、インデックスの共通範囲は無効化（0）されます。

保守担当者向けメモ

  • 本関数は inet_gist_picksplit におけるノード分割後の代表値再計算に利用されています。
  • bitncommon によるビット比較は、アドレスファミリーが一致している場合にのみ有効な結果を返します。
