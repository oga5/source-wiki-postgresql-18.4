AddUserToTokenDacl

定義

  • 種別: function
  • 定義箇所: src/common/exec.c:518

目的

Windows環境において、指定されたアクセストークンのデフォルトDACL（随意アクセス制御リスト）に、現在のユーザーに対するフルアクセス権限（GENERIC_ALL）を持つACE（アクセス制御エントリ）を追加する。

シグネチャまたは形状

BOOL AddUserToTokenDacl(HANDLE hToken)

入力と出力

  • **入力**: hToken (Windowsのアクセストークンハンドル)
  • **出力**: 成功時に TRUE、失敗時に FALSE を返す。

前提条件と不変条件

  • hToken は有効なアクセストークンハンドルであること。
  • 呼び出し元は、Windows API（GetTokenInformation, LocalAlloc, InitializeAcl 等）が利用可能な環境であること。

アルゴリズムまたは振る舞い

1. **情報取得**: GetTokenInformation を使用して、トークンのデフォルトDACL情報を取得する。 2. **ユーザーSID取得**: GetTokenUser を呼び出し、現在のユーザーのSIDを取得する。 3. **ACL再構築**:

  • 新しいDACLに必要なサイズを計算し、メモリを確保する。
  • InitializeAcl で新しいACLを初期化する。
  • 既存のDACLからすべてのACEをコピーして新しいACLに追加する。
  • AddAccessAllowedAceEx を使用し、現在のユーザーSIDに対して OBJECT_INHERIT_ACE および GENERIC_ALL 権限を持つ新しいACEを追加する。

    • 4. **トークン更新**: SetTokenInformation を使用して、更新されたDACLをトークンに適用する。 5. **クリーンアップ**: 確保したメモリ（pTokenUser, pacl, ptdd）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_ctl/pg_ctl.c:1843
  • src/common/restricted_token.c:99
  • **呼び出し先**:
  • GetTokenUser (同ファイル内)
  • Windows API: GetTokenInformation, LocalAlloc, GetAclInformation, GetLengthSid, InitializeAcl, GetAce, AddAce, AddAccessAllowedAceEx, SetTokenInformation, LocalFree, GetLastError

関連データ構造

  • TOKEN_DEFAULT_DACL: トークンのデフォルトDACL情報を保持する構造体。
  • TOKEN_USER: ユーザーSIDを保持する構造体。
  • ACL_SIZE_INFORMATION: ACLのサイズ情報を保持する構造体。

失敗モードと境界条件

  • メモリ割り当て（LocalAlloc）失敗時は log_error を経由してエラーを報告し、FALSE を返す。
  • Windows API呼び出しが失敗した場合、GetLastError を用いてエラーをログ出力し、処理を中断して FALSE を返す。
  • GetTokenUser が失敗した場合、その関数内でエラーメッセージが出力されるため、そのまま終了する。

保守担当者向けメモ

  • 本関数はWindows専用のコードパスである。
  • log_error マクロを使用しているため、フロントエンドとバックエンドの両方でエラー出力が一貫している。
  • LocalAlloc で確保したメモリは、cleanup ラベル以降で確実に解放される設計となっている。
