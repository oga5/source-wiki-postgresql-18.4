plpgsql_param_ref

定義

  • **種別**: function
  • **定義箇所**: src/pl/plpgsql/src/pl_comp.c:1055

目的

SQL 式内の $n 形式のパラメータ参照を、PL/pgSQL の内部変数（Datum）へ解決するためのパーサーコールバック関数です。

シグネチャまたは形状

Node *plpgsql_param_ref(ParseState *pstate, ParamRef *pref)

入力と出力

  • **入力**:
  • pstate: PostgreSQL のパーサー状態。p_ref_hook_state に PLpgSQL_expr が格納されていることを前提とします。
  • pref: 解析対象のパラメータ参照ノード。number フィールドにパラメータ番号が含まれます。
  • **出力**:
  • 成功時: 解決されたパラメータを表す Param ノード（make_datum_param の戻り値）。
  • 失敗時: NULL（PL/pgSQL の名前空間で該当するパラメータが見つからない場合）。

前提条件と不変条件

  • pstate->p_ref_hook_state が有効な PLpgSQL_expr 構造体を指している必要があります。
  • pref->number は正の整数であり、$n という形式の文字列に変換可能である必要があります。

アルゴリズムまたは振る舞い

1. pref->number を基に、"$n" という形式の文字列を生成します。 2. plpgsql_ns_lookup を呼び出し、現在の式に関連付けられた名前空間（expr->ns）から当該パラメータ名を検索します。 3. 名前空間に存在しない場合は NULL を返します。 4. 名前空間に存在する場合、make_datum_param を呼び出し、対応する itemno を用いて Param ノードを生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のパーサー（pstate->p_paramref_hook に設定されたフック経由）。
  • **呼び出し先**:
  • plpgsql_ns_lookup: 名前空間からの検索。
  • make_datum_param: パラメータノードの生成。

関連データ構造

  • PLpgSQL_expr: 式のコンテキストと名前空間を保持。
  • PLpgSQL_nsitem: 名前空間内の項目情報。
  • ParamRef: パーサーが生成するパラメータ参照ノード。

失敗モードと境界条件

  • 指定された番号のパラメータが PL/pgSQL の名前空間に存在しない場合、NULL を返してパーサーに解決を委ねます。
  • pstate->p_ref_hook_state が正しく設定されていない場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は plpgsql_parser_setup 内で pstate->p_paramref_hook に割り当てられることで機能します。
  • $n 形式のパラメータ解決に特化しており、列参照の解決（plpgsql_pre_column_ref）とは別のパスで動作します。
