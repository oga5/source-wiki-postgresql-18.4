JsonTablePlanState

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/jsonpath_exec.c:177

目的

JSON_TABLE 関数実行時におけるプランツリーの状態管理を行うための構造体です。行パターンの評価、結果の保持、およびネストされたプランの走査を管理します。

シグネチャまたは形状

typedef struct JsonTablePlanState {
    JsonTablePlan *plan;
    JsonPath      *path;
    MemoryContext  mcxt;
    List          *args;
    JsonValueList  found;
    JsonValueListIterator iter;
    JsonTablePlanRowSource current;
    int            ordinal;
    struct JsonTablePlanState *nested;
    struct JsonTablePlanState *left;
    struct JsonTablePlanState *right;
    struct JsonTablePlanState *parent;
} JsonTablePlanState;

入力と出力

  • **入力**: JsonTablePlan（元のプラン定義）および PASSING 句で渡される引数リスト（args）。
  • **出力**: JsonValueList（jsonpath 評価結果のリスト）および JsonTablePlanRowSource（現在選択中の行）。

前提条件と不変条件

  • path、mcxt、args、found、iter の各フィールドは、JsonTablePathScan 型のプランにおいてのみ有効です。
  • nested、left、right、parent フィールドにより、プランツリーの階層構造が維持されます。

アルゴリズムまたは振る舞い

  • JsonTableInitPlan により初期化され、プランツリーの構築が行われます。
  • JsonTableResetRowPattern を介して行パターンのリセットが行われます。
  • JsonTablePlanScanNextRow により、次の行の走査が実行されます。
  • JsonTableResetNestedPlan により、ネストされたプランの状態がリセットされます。
  • JsonTableGetValue が呼び出される際、current フィールドに保持された行データが使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: JsonTableInitPlan、JsonTableResetRowPattern、JsonTablePlanScanNextRow、JsonTableResetNestedPlan 等の jsonpath_exec.c 内の関数。
  • **呼び出し先**: 構造体内部で JsonTablePlan や JsonPath へのポインタを保持し、再帰的に JsonTablePlanState を参照します。

関連データ構造

  • JsonTablePlan: 元となるプラン定義。
  • JsonValueList: マッチした要素のリスト。
  • JsonTablePlanRowSource: 現在の行ソース。
  • JsonTableExecContext: JSON_TABLE 実行時の全体的なコンテキスト。

失敗モードと境界条件

  • JsonTablePlanScanNextRow 等の関数実行時に、メモリコンテキスト（mcxt）の管理が不適切である場合、メモリリークや不正なメモリ参照が発生する可能性があります。
  • ネストされたプランの走査において、スタック深さの制限を超える深い階層構造を持つ場合、実行時エラーとなる可能性があります。

保守担当者向けメモ

  • 本構造体は JSON_TABLE 実行専用の状態管理用であり、他の jsonpath 実行ロジックとは独立しています。
  • nested、left、right、parent の各ポインタはプランツリーの構造を定義するため、プランの再構築や走査時にはこれらの整合性を維持する必要があります。
