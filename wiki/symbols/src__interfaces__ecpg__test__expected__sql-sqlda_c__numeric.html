numeric

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-sqlda.c:67

目的

ECPG（Embedded SQL for C）において、数値データを表現および管理するための構造体です。

シグネチャまたは形状

typedef struct
{
    int            ndigits;      /* digits[] 内の桁数 */
    int            weight;       /* 先頭桁の重み */
    int            rscale;       /* 結果のスケール */
    int            dscale;       /* 表示スケール */
    int            sign;         /* NUMERIC_POS, NUMERIC_NEG, または NUMERIC_NAN */
    NumericDigit  *buf;          /* digits[] 用に割り当てられたメモリ領域の先頭 */
    NumericDigit  *digits;       /* 10進数の桁データ */
} numeric;

入力と出力

  • 入力: 数値の各属性（スケール、符号、桁データ）を保持するフィールド。
  • 出力: digits フィールドを介した数値の各桁へのアクセス、および buf によるメモリ管理。

前提条件と不変条件

  • ndigits は 0 になる可能性があります。
  • buf は digits が指すメモリ領域の開始位置を保持する必要があります。
  • sign フィールドは NUMERIC_POS、NUMERIC_NEG、または NUMERIC_NAN のいずれかである必要があります。

アルゴリズムまたは振る舞い

本構造体は、可変長の数値データを保持するために、buf を通じて動的に割り当てられたメモリ領域を使用します。ndigits で示される桁数に基づき、digits 配列を走査することで数値の表現を構築します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内では、本構造体を利用する具体的な関数呼び出しの明示的な証拠はありません。
  • 呼び出し先: NumericDigit 型（定義元は pgtypes.h）に依存しています。

関連データ構造

  • NumericDigit: 数値の各桁を保持する型。
  • sqlda_t: ECPG における SQLDA 構造体。

失敗モードと境界条件

  • ndigits が 0 の場合、数値がゼロまたは未定義として扱われる可能性があります。
  • buf が適切に初期化または割り当てられていない場合、メモリ破壊や不正なアクセスが発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は pgtypes.h で定義される数値型に関連しています。
  • src/interfaces/ecpg/test/expected/sql-sqlda.c はテスト用コードであるため、本構造体の利用は主にテストケース内でのデータ表現に限定されています。
