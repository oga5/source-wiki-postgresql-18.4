GUCArrayDelete

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc.c:6571

目的

GUC（Grand Unified Configuration）設定値を含む文字列配列から、指定されたパラメータ名に一致するエントリを削除し、新しい配列を生成する。

シグネチャまたは形状

ArrayType *GUCArrayDelete(ArrayType *array, const char *name)

入力と出力

  • **入力**:
  • array: 削除対象の ArrayType（TEXT 型の配列を想定）。
  • name: 削除対象の GUC パラメータ名。
  • **出力**:
  • 指定されたパラメータを除外した新しい ArrayType を返す。入力配列が NULL の場合は NULL を返す。

前提条件と不変条件

  • name は NULL であってはならない（Assert(name) による検証）。
  • validate_option_array_item により、指定された name が有効な GUC パラメータであるか、および設定変更が許可されているかが事前に検証される。

アルゴリズムまたは振る舞い

1. find_option を使用して name を正規化し、古い GUC 名を現在の正式名称に変換する。 2. 入力 array が NULL の場合は直ちに NULL を返す。 3. 入力配列の各要素を走査し、name=value 形式の文字列を解析する。 4. 指定された name と一致するエントリ（name に = が続くもの）をスキップする。 5. 一致しないエントリは、construct_array_builtin または array_set を用いて新しい配列にコピーする。 6. 最終的に構築された新しい配列を返す。

関連データ構造

  • ArrayType: PostgreSQL の内部配列表現。
  • config_generic: GUC パラメータの定義構造体。

失敗モードと境界条件

  • array が NULL の場合、削除処理を行わず NULL を返す。
  • validate_option_array_item が失敗した場合、エラーが発生して処理が中断される。
  • 配列内の要素が NULL である場合、その要素は無視される。

保守担当者向けメモ

  • 本関数は TEXT 型の配列を前提としており、配列内の各要素が name=value という形式であることを期待している。
  • find_option による正規化を行っているため、古いパラメータ名が渡されても適切に処理される。
