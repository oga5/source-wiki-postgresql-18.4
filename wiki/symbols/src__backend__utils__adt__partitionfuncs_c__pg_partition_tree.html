pg_partition_tree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/partitionfuncs.c:61

目的

指定されたパーティションルートリレーションから、その配下にある全パーティションの階層構造を列挙するセット返却関数（SRF）です。

シグネチャまたは形状

pg_partition_tree(rootrelid Oid) RETURNS TABLE(relid Oid, parentid Oid, isleaf bool, level int)

入力と出力

  • **入力**: rootrelid (Oid) - パーティションツリーの起点となるリレーションのOID。
  • **出力**: 以下のカラムを持つ行セット。
  • relid: パーティションのOID。
  • parentid: 直近の親リレーションのOID（ルートの場合はNULL）。
  • isleaf: パーティションが子を持たないリーフノードであるかを示すフラグ。
  • level: ルートからの階層深度（ルートは0）。

前提条件と不変条件

  • 呼び出し元は FuncCallContext を使用して状態を保持する。
  • check_rel_can_be_partition により、対象リレーションがパーティションツリーの一部として有効である必要がある。
  • 初回呼び出し時に find_all_inheritors を用いて全継承メンバーを特定し、以降の呼び出しでそのリストを順次消費する。

アルゴリズムまたは振る舞い

1. **初回呼び出し時**:

  • SRF_FIRSTCALL_INIT でコンテキストを初期化。
  • check_rel_can_be_partition で検証。失敗時は終了。
  • find_all_inheritors で全パーティションのリストを取得し、funcctx->user_fctx に保存。

    • 2. **各呼び出し時**:

  • 保存されたリストから現在のインデックス（call_cntr）に対応する relid を取得。
  • get_partition_ancestors で祖先リストを取得し、parentid と level を算出。
  • RELKIND_HAS_PARTITIONS マクロを用いて isleaf を判定。
  • heap_form_tuple で行を構築し、SRF_RETURN_NEXT で返却。

    • 3. **終了時**:

  • リストを全て走査し終えたら SRF_RETURN_DONE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/bin/psql/describe.c
  • **呼び出し先**:
  • check_rel_can_be_partition
  • find_all_inheritors
  • get_partition_ancestors
  • get_rel_relkind
  • heap_form_tuple

関連データ構造

  • FuncCallContext: SRF の状態保持用構造体。
  • List: パーティションおよび祖先の保持に使用。
  • HeapTuple: 返却する行データの構築に使用。

失敗モードと境界条件

  • check_rel_can_be_partition が false を返した場合、結果を返さずに終了する。
  • 戻り値の型が期待される複合型でない場合、elog(ERROR) を発生させる。
  • 祖先リストが空（ルート自身）の場合、parentid は NULL となる。

保守担当者向けメモ

  • 本関数は SRF_IS_FIRSTCALL を用いた標準的な SRF 実装パターンに従っている。
  • メモリ管理は multi_call_memory_ctx を介して行われており、呼び出し間での状態維持が保証されている。
