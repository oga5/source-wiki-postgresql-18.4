CteState

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_cte.c:71

目的

WITH 句（CTE）の構文解析および意味解析プロセスにおいて、依存関係の解決や再帰の妥当性検証を行うための状態を保持する構造体です。ツリーウォーカー間で共有されるコンテキストとして機能します。

シグネチャまたは形状

typedef struct CteState
{
    ParseState *pstate;
    CteItem    *items;
    int         numitems;
    int         curitem;
    List       *innerwiths;
    int         selfrefcount;
    RecursionContext context;
} CteState;

入力と出力

  • **入力**: transformWithClause 等の解析関数において、ParseState や解析対象の WITH 句情報から初期化されます。
  • **出力**: 解析過程で items 配列内の各 CteItem が更新され、最終的に Query ノードへの変換や名前空間の構築に利用されます。

前提条件と不変条件

  • items は numitems 分の CteItem を保持する配列である必要があります。
  • innerwiths はネストされた WITH 句の可視性を追跡するために使用されます。

アルゴリズムまたは振る舞い

  • **依存関係構築**: makeDependencyGraph がツリーウォーカーを用いて各CTE間の依存関係を特定します。
  • **再帰検証**: checkWellFormedRecursion が RecursionContext を切り替えながら、再帰的CTEの構造がSQL標準に適合しているか（非再帰項と再帰項の分離など）を検証します。
  • **状態管理**: 解析の進行に伴い、curitem が現在処理中のCTEを指し示します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_cte.c 内の transformWithClause 等。
  • **呼び出し先**: makeDependencyGraph、makeDependencyGraphWalker、checkWellFormedRecursion、checkWellFormedRecursionWalker 等の静的関数がこの構造体を引数として受け取ります。
  • 注: src/backend/executor/nodeCtescan.c からの参照が示唆されていますが、直接的な呼び出し関係の詳細はソース抜粋からは不明です。

関連データ構造

  • ParseState: グローバルな解析状態。
  • CteItem: 個別のCTE情報。
  • RecursionContext: 再帰参照の可否を制御する列挙型。
  • List: innerwiths を保持するリスト構造。

失敗モードと境界条件

  • 再帰的CTEにおいて、不正な位置への再帰参照が含まれる場合、checkWellFormedRecursion によって検証エラーが発生します。
  • 重複するCTE名が存在する場合、transformWithClause の初期化フェーズで検出されます。

保守担当者向けメモ

  • checkWellFormedRecursionWalker は RecursionContext を動的に変更するため、状態遷移のロジックを変更する際は注意が必要です。
  • makeDependencyGraphWalker はネストされた WITH 句（innerwiths）を考慮して名前の可視性を追跡します。
