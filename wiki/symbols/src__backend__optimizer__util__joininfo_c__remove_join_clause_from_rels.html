remove_join_clause_from_rels

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/joininfo.c:160

目的

指定された結合条件（RestrictInfo）を、それが参照する各リレーションの joininfo リストから削除します。

シグネチャまたは形状

void remove_join_clause_from_rels(PlannerInfo *root, RestrictInfo *restrictinfo, Relids join_relids)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報構造体。
  • restrictinfo: 削除対象の結合条件。
  • join_relids: 条件が参照するリレーション ID の集合（Relids）。
  • **出力**: なし（副作用としてリレーションの joininfo リストが更新されます）。

前提条件と不変条件

  • restrictinfo は、join_relids に含まれる各ベースリレーションの joininfo リストに既に存在している必要があります。
  • 削除対象はベースリレーション（baserel）の joininfo リストに限定されます。

アルゴリズムまたは振る舞い

1. join_relids に含まれる各リレーション ID を順次取得します。 2. find_base_rel_ignore_join を使用して、対応する RelOptInfo を取得します。 3. リレーションが存在しない場合はスキップします。 4. list_member_ptr を用いて、対象の restrictinfo がリスト内に存在することをアサートします。 5. list_delete_ptr を使用し、ポインタ比較によってリストから該当する restrictinfo を削除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/analyzejoins.c: 結合解析プロセスにおいて、不要になった結合条件を削除するために呼び出されます。
  • **呼び出し先**:
  • bms_next_member: ビットマップセットの走査に使用。
  • find_base_rel_ignore_join: リレーションの検索に使用。
  • list_member_ptr / list_delete_ptr: リスト操作に使用。

関連データ構造

  • PlannerInfo: プランナの全体状態を保持。
  • RestrictInfo: 結合条件や制約を保持する構造体。
  • RelOptInfo: リレーションの最適化情報を保持し、joininfo リストを内包します。

失敗モードと境界条件

  • join_relids が空の場合、ループは実行されず何も行われません。
  • find_base_rel_ignore_join が NULL を返す場合（ベースリレーション以外が指定された場合など）、そのリレーションに対する削除処理はスキップされます。

保守担当者向けメモ

  • 本関数は add_join_clause_to_rels の逆操作です。
  • ポインタ比較による削除を行っているため、同一の RestrictInfo インスタンスが正しく管理されていることが前提となります。
