smgropen

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/smgr.c:239

目的

指定されたリレーション（RelFileLocator）に対応する SMgrRelation オブジェクトを取得または作成します。

シグネチャまたは形状

SMgrRelation smgropen(RelFileLocator rlocator, ProcNumber backend)

入力と出力

  • **入力**:
  • rlocator: 対象リレーションを識別する RelFileLocator。
  • backend: リレーションを所有するバックエンドプロセス番号。
  • **出力**:
  • 対象リレーションの物理ファイルハンドルを保持する SMgrRelation 構造体へのポインタ。

前提条件と不変条件

  • rlocator.relNumber が有効であること（Assert による検証）。
  • 割り込みを禁止した状態（HOLD_INTERRUPTS）で実行される。
  • SMgrRelationHash が未初期化の場合は、初回呼び出し時にハッシュテーブルが作成される。

アルゴリズムまたは振る舞い

1. HOLD_INTERRUPTS() を呼び出し、ストレージ操作中の割り込みを抑止する。 2. SMgrRelationHash が存在しない場合、ハッシュテーブルを初期化し、ピン留めされていないリレーションのリスト unpinned_relns を初期化する。 3. RelFileLocatorBackend をキーとして SMgrRelationHash を検索する。 4. エントリが存在しない場合（新規作成時）:

  • smgr_targblock を InvalidBlockNumber に初期化。
  • 各フォークの smgr_cached_nblocks を InvalidBlockNumber に初期化。
  • smgr_which を 0（md.c）に設定。
  • pincount を 0 に設定し、unpinned_relns リストに追加。
  • smgrsw[0].smgr_open(reln) を呼び出し、物理的なオープン処理を実行。

    • 5. RESUME_INTERRUPTS() を呼び出し、割り込みを許可して reln を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogprefetcher.c
  • src/backend/access/transam/xlogutils.c
  • src/backend/catalog/storage.c
  • src/backend/commands/dbcommands.c
  • src/backend/commands/sequence.c
  • **呼び出し先**:
  • hash_create, hash_search (utils/hsearch.h)
  • dlist_init, dlist_push_tail (lib/ilist.h)
  • smgrsw[].smgr_open (ストレージ実装、現在は md.c)

関連データ構造

  • SMgrRelation: 物理ファイルへのハンドルを保持する構造体。
  • SMgrRelationHash: SMgrRelation を管理するハッシュテーブル。
  • RelFileLocator: リレーションの物理的な識別子。
  • smgrsw: ストレージマネージャの関数ポインタスイッチ。

失敗モードと境界条件

  • メモリ不足によりハッシュテーブルの作成やエントリの追加が失敗した場合、PostgreSQL の標準的なエラーハンドリング（elog 等）により中断される可能性がある。
  • 割り込みが禁止されているため、この関数内での長時間待機やエラー発生時のクリーンアップには注意が必要。

保守担当者向けメモ

  • smgr_which は現在 0（md.c）に固定されている。将来的に他のストレージ実装が追加される場合は、このインデックス管理を拡張する必要がある。
  • HOLD_INTERRUPTS の範囲外で呼び出された場合の挙動については、本ソースコード内では考慮されていない。
