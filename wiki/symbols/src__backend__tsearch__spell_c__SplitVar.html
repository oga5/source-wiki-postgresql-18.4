SplitVar

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/spell.c:2332

目的

複合語分割処理において、単語の分割候補（ステミング結果のリスト）を保持するための連結リスト構造体です。

シグネチャまたは形状

typedef struct SplitVar
{
    int             nstem;      /* ステムの数 */
    int             lenstem;    /* ステムの総長 */
    char          **stem;       /* ステム文字列の配列 */
    struct SplitVar *next;      /* 次の候補へのポインタ */
} SplitVar;

入力と出力

  • 本構造体は SplitToVariants 関数内で生成・操作され、単語の分割候補リストとして出力されます。
  • CopyVar 関数により、既存の SplitVar 構造体の複製が作成されます。

前提条件と不変条件

  • next ポインタは、リストの終端では NULL である必要があります。
  • stem は nstem 個の文字列を保持する配列を指す必要があります。

アルゴリズムまたは振る舞い

  • 複合語の分割候補を再帰的に探索する過程で、各分割パターンをリストとして蓄積します。
  • AddStem 関数を通じて、新しいステムが現在の SplitVar 構造体に追加されます。
  • CopyVar を用いて、探索の分岐ごとに候補リストのコピーが生成されます。

呼び出し元と呼び出し先

  • **呼び出し元**: SplitToVariants が主に本構造体を生成・管理します。また、CopyVar や AddStem が操作を行います。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • IspellDict: 辞書の状態を保持する構造体。
  • SPNode: 辞書検索用 Trie 構造のノード。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗した場合、PostgreSQL のエラーハンドリングによりアボートされます。
  • 非常に長い単語や複雑な複合語ルールの場合、再帰的な処理によりスタックオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • SplitVar は SplitToVariants による複合語分割の核心的なデータ構造です。
  • リストの操作（特に CopyVar による複製）はメモリ消費に直結するため、大規模な辞書を扱う際は注意が必要です。
