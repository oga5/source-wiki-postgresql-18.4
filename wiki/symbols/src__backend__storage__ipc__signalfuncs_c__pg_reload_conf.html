pg_reload_conf

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/signalfuncs.c:287

目的

Postmasterプロセスに対して設定ファイルの再読み込み（SIGHUPシグナル）を指示するためのSQL関数インターフェースです。

シグネチャまたは形状

Datum pg_reload_conf(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL経由での呼び出し引数）
  • **出力**: Datum 型として bool を返却。成功時は true、失敗時は false。

前提条件と不変条件

  • PostmasterPid が有効なPostmasterプロセスのPIDを保持している必要があります。
  • 本関数はSQL関数として実行されるため、呼び出し元には適切な権限（通常はスーパーユーザー権限）が必要です。

アルゴリズムまたは振る舞い

1. kill(PostmasterPid, SIGHUP) を呼び出し、Postmasterプロセスに設定再読み込みを要求します。 2. kill が失敗した場合（戻り値が0以外）、WARNING レベルのログを ereport で出力し、false を返します。 3. kill が成功した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（具体的なSQL呼び出し元はソースコード上では特定されていません）。
  • **呼び出し先**:
  • kill() (システムコール)
  • ereport() (PostgreSQLエラー報告機構)

関連データ構造

  • PostmasterPid (グローバル変数: miscadmin.h で定義)

失敗モードと境界条件

  • kill システムコールが失敗した場合、警告ログが出力され、関数は false を返して終了します。
  • プロセスが存在しない、または権限不足によりシグナル送信が拒否された場合が主な失敗要因となります。

保守担当者向けメモ

  • 本関数は単に SIGHUP を送るのみであり、Postmaster側での再読み込みが正常に完了したかどうかまでは保証しません。
  • kill の戻り値チェックにより、シグナル配送の成否のみを判定しています。
