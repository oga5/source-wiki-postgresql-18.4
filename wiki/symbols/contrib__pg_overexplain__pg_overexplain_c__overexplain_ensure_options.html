overexplain_ensure_options

定義

  • 種別: function
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:89

目的

ExplainState に紐付けられた overexplain_options 構造体を取得、または存在しない場合に新規作成してアタッチします。

シグネチャまたは形状

static overexplain_options *overexplain_ensure_options(ExplainState *es)

入力と出力

  • **入力**: ExplainState *es （EXPLAIN コマンドの実行状態を保持する構造体）
  • **出力**: overexplain_options * （拡張モジュール固有のオプション状態を保持する構造体へのポインタ）

前提条件と不変条件

  • es_extension_id が初期化済みであること。
  • GetExplainExtensionState および SetExplainExtensionState を介して、ExplainState の拡張領域にアクセス可能であること。

アルゴリズムまたは振る舞い

1. GetExplainExtensionState を呼び出し、現在の ExplainState に overexplain_options が既にアタッチされているか確認します。 2. 既に存在する場合はそのポインタを返します。 3. 存在しない場合は palloc0 を使用して overexplain_options 構造体をゼロ初期化してメモリ確保し、SetExplainExtensionState を使用して ExplainState に登録した上で、そのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_overexplain/pg_overexplain.c 内のハンドラ関数（overexplain_debug_handler および overexplain_range_table_handler と推測される箇所）。
  • **呼び出し先**:
  • GetExplainExtensionState
  • SetExplainExtensionState
  • palloc0

関連データ構造

  • ExplainState: PostgreSQL の EXPLAIN 実行状態を管理する構造体。
  • overexplain_options: 本モジュールが ExplainState に保持する拡張オプションの状態構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • es が NULL の場合の挙動は、基盤となる GetExplainExtensionState の実装に依存します。

保守担当者向けメモ

  • 本関数は EXPLAIN のオプション解析フェーズで、拡張状態を遅延初期化するために使用されます。
  • ExplainState へのデータ保持手法として、他の EXPLAIN 拡張を作成する際の標準的なパターンとなっています。
