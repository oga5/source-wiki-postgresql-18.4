ShippableCacheKey

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/shippable.c:41

目的

postgres_fdw において、特定のデータベースオブジェクトがリモートサーバーへプッシュダウン可能（shippable）であるかを判定する際の、キャッシュ検索用キーとして機能します。

シグネチャまたは形状

typedef struct {
    Oid objid;    /* 関数、演算子、またはデータ型のOID */
    Oid classid;  /* オブジェクトが属するカタログのOID (pg_proc等) */
    Oid serverid; /* 対象となるFDWサーバーのOID */
} ShippableCacheKey;

入力と出力

  • **入力**: オブジェクト識別子（objid）、カタログ識別子（classid）、サーバー識別子（serverid）の3つの Oid。
  • **出力**: ハッシュテーブル検索のための構造体データ。

前提条件と不変条件

  • 構造体内にパディングバイトが含まれないことが前提とされています。
  • ハッシュテーブル（ShippableCacheHash）内で使用される際、この構造体はエントリの先頭に配置される必要があります。

アルゴリズムまたは振る舞い

  • ShippableCacheHash のキーサイズとして sizeof(ShippableCacheKey) が使用されます。
  • is_shippable 関数等の判定ロジックにおいて、特定のオブジェクトとサーバーの組み合わせに対する判定結果をキャッシュから取得・格納するためのインデックスとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/shippable.c 内のキャッシュ初期化および検索処理（InitializeShippableCache 等）から参照されます。
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • ShippableCacheEntry: キャッシュの値を保持する構造体。ShippableCacheKey をメンバとして含みます。
  • ShippableCacheHash: 本キーを使用して検索を行うハッシュテーブル。

失敗モードと境界条件

  • 構造体のメモリレイアウトがコンパイラによって変更された場合、ハッシュ検索の整合性が失われる可能性があります。
  • serverid が無効な場合、キャッシュのヒット率は低下しますが、システム上のエラーには直結しません。

保守担当者向けメモ

  • 本構造体はハッシュテーブルのキーとして直接メモリ比較されるため、メンバの順序変更やパディングの混入には細心の注意を払ってください。
  • 拡張機能の追加やカタログ定義の変更に伴い、キーの構成要素を増やす必要がある場合は、キャッシュの無効化ロジック（InvalidateShippableCacheCallback）への影響を再評価してください。
