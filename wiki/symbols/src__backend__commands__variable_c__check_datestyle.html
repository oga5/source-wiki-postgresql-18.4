check_datestyle

定義

  • 種別: function
  • 定義箇所: src/backend/commands/variable.c:51

目的

datestyle GUCパラメータの検証および正規化を行います。ユーザーが指定した日付形式の文字列を解析し、システムが許容する形式であるかを確認した上で、内部表現へ変換します。

シグネチャまたは形状

bool check_datestyle(char **newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定対象の文字列へのポインタ。検証および正規化後に標準化された文字列で上書きされます。
  • extra: 検証結果を格納する領域へのポインタ。
  • source: 設定のソース元（GucSource）。
  • **出力**:
  • 成功時は true を返し、newval に正規化された文字列を、extra に内部表現（int配列）を格納します。
  • 失敗時は false を返し、GUC_check_errdetail を通じてエラー詳細を報告します。

前提条件と不変条件

  • newval が指す文字列は、カンマ区切りの識別子リストとして解析可能である必要があります。
  • 矛盾するスタイル（例: ISO と SQL の同時指定）はエラーとして扱われます。
  • DEFAULT キーワードが含まれる場合、現在のシステム設定を再帰的に取得して解決します。

アルゴリズムまたは振る舞い

1. 入力文字列を pstrdup で複製し、SplitIdentifierString でトークンリストに分割します。 2. 各トークンを pg_strcasecmp 等で照合し、日付スタイル（ISO, SQL, POSTGRES, GERMAN）および日付順序（YMD, DMY, MDY）を特定します。 3. GERMAN 指定時はデフォルトで DMY 順序が適用されます。 4. DEFAULT 指定時は、現在設定されている datestyle のリセット値を再帰的に解析して適用します。 5. 全トークンの解析後、矛盾がないかを確認し、成功時は正規化された文字列を生成して newval に格納します。 6. 最後に、assign_datestyle で使用される int 配列（スタイルと順序のペア）を guc_malloc で確保し、extra に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: GUCサブシステム（src/backend/utils/misc/guc_tables.c でフックとして登録）。
  • **呼び出し先**: 内部で自身を再帰的に呼び出す場合があります。また、SplitIdentifierString, pstrdup, guc_malloc, guc_free などのユーティリティ関数を使用します。

関連データ構造

  • DateStyle / DateOrder: 内部的な日付形式および順序の定数。
  • GucSource: 設定の適用元を示す列挙型。

失敗モードと境界条件

  • **構文エラー**: 不明なキーワードや無効なリスト形式の場合、false を返します。
  • **競合エラー**: スタイルや順序の指定が矛盾している場合、false を返します。
  • **メモリ不足**: guc_malloc 等のメモリ確保に失敗した場合、false を返します。

保守担当者向けメモ

  • 本関数はテーブル駆動型ではないため、新しい日付形式を追加する際は if-else ブロックの拡張が必要です（ソース内のコメント参照）。
  • extra に割り当てられたメモリはGUCサブシステムによって管理されるため、呼び出し側での解放は不要ですが、check_datestyle 内でのエラー発生時には適切に guc_free を行う必要があります。
