PLy_commit

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:445

目的

Python コードから PostgreSQL のトランザクションをコミットするために使用されます。

シグネチャまたは形状

PyObject *PLy_commit(PyObject *self, PyObject *args)

入力と出力

  • 入力: self (Python オブジェクト), args (引数タプル)
  • 出力: 成功時は Py_None を返し、失敗時は Python 例外を発生させ NULL を返します。

前提条件と不変条件

  • PLy_current_execution_context() により現在の実行コンテキストが取得可能であること。
  • トランザクション終了時に exec_ctx->scratch_ctx がクリアされるため、関数内でポインタを NULL にリセットする不変条件を持ちます。

アルゴリズムまたは振る舞い

1. 現在のメモリコンテキストを保存します。 2. PG_TRY() ブロック内で SPI_commit() を呼び出し、トランザクションをコミットします。 3. 成功時は exec_ctx->scratch_ctx を NULL にリセットし、Py_RETURN_NONE を返します。 4. PG_CATCH() ブロック（エラー発生時）では以下を実行します。

  • メモリコンテキストを復元し、エラー情報を CopyErrorData() で取得後、FlushErrorState() でエラー状態をクリアします。
  • exec_ctx->scratch_ctx を NULL にリセットします。
  • PLy_spi_exceptions ハッシュテーブルから SQL エラーコードに対応する Python 例外を検索します。
  • PLy_spi_exception_set() を呼び出し、Python 側で適切な例外を発生させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_plpymodule.c にて plpy モジュールのメソッドとして登録されています。
  • 呼び出し先: SPI_commit, CopyErrorData, FlushErrorState, hash_search, PLy_spi_exception_set, FreeErrorData 等。

関連データ構造

  • PLyExecutionContext: 実行コンテキストの管理に使用。
  • PLy_spi_exceptions: エラーコードと Python 例外のマッピングを保持するハッシュテーブル。
  • ErrorData: PostgreSQL のエラー情報を保持する構造体。

失敗モードと境界条件

  • SPI_commit() が失敗した場合、PG_CATCH() ブロックへ遷移し、PostgreSQL のエラーが Python の例外（デフォルトは PLy_exc_spi_error）として再送出されます。
  • カスタムエラーコードの場合、SPIError へフォールバックします。

保守担当者向けメモ

  • PG_TRY / PG_CATCH ブロック内でのメモリ管理（MemoryContextSwitchTo）が重要です。エラー発生時にメモリコンテキストが正しく復元されないと、後続の処理に影響を与える可能性があります。
