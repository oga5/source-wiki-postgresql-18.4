vars

定義

  • 種別: struct
  • 定義箇所: src/backend/regex/regcomp.c:281

目的

正規表現のコンパイルプロセスにおいて、解析状態、NFA（非決定性有限オートマトン）、サブ式ツリー、およびエラー情報を一元管理するためのコンテキスト構造体です。

シグネチャまたは形状

struct vars {
    regex_t    *re;
    const chr  *now;
    const chr  *stop;
    int         err;
    int         cflags;
    int         lasttype;
    int         nexttype;
    chr         nextvalue;
    int         lexcon;
    int         nsubexp;
    struct subre **subs;
    size_t      nsubs;
    struct subre *sub10[10];
    struct nfa *nfa;
    struct colormap *cm;
    color       nlcolor;
    struct state *wordchrs;
    struct subre *tree;
    struct subre *treechain;
    struct subre *treefree;
    int         ntree;
    struct cvec *cv;
    struct cvec *cv2;
    struct subre *lacons;
    int         nlacons;
    size_t      spaceused;
};

入力と出力

  • **入力**: pg_regcomp() 呼び出し時に初期化され、コンパイル対象の正規表現パターンやフラグ、ロケール情報が反映されます。
  • **出力**: コンパイル結果として regex_t 構造体へのポインタ（re）を保持し、解析過程で生成された NFA やサブ式ツリーを構築します。

前提条件と不変条件

  • sub10[10] は、ほとんどの式で十分な初期サブ式ベクトルとして機能します。
  • lacons において、使用されるスロットは 1 から nlacons - 1 までです。
  • コンパイル中は err フィールドが 0 であることが正常状態を示します。

アルゴリズムまたは振る舞い

正規表現のコンパイルにおけるオーケストレーターとして機能します。字句解析（lexstart）から構文解析（parse）、NFA 構築、最適化、および最終的な圧縮に至るまで、各フェーズで共有される作業領域を提供します。メモリ管理や一時的なツリーノードの割り当て・解放もこの構造体を通じて行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_regcomp() が本構造体を初期化し、コンパイルの起点となります。
  • **呼び出し先**: regc_lex.c, regc_nfa.c, regc_color.c 等のモジュール内の関数が、この構造体を引数として受け取り、状態の更新やデータ構造の構築を行います。
  • **注意**: ツールが検出した contrib/postgres_fdw 等の vars は、本構造体とは異なる変数名（List *vars）であり、本シンボルとは無関係です。

関連データ構造

  • regex_t: 公開用の正規表現ハンドル。
  • subre: 正規表現の構造を表現するツリーノード。
  • nfa: 非決定性有限オートマトン。
  • colormap: 文字色マッピング。
  • cvec: 文字セットのインターフェース。

失敗モードと境界条件

  • err フィールドに非ゼロ値が設定された場合、コンパイルエラーが発生したことを示します。
  • spaceused はコンパイルに使用された概算メモリ量を追跡します。
  • sub10 を超えるサブ式が必要な場合、動的なメモリ割り当てが行われます。

保守担当者向けメモ

  • 本構造体は regcomp.c 内の多くの関数で共有されるため、フィールドの追加や変更は広範囲に影響します。
  • REG_DEBUG マクロを有効にすると、コンパイル過程で本構造体の状態をダンプ可能です。
  • contrib/postgres_fdw 等で同名の変数が存在しますが、これらは本構造体とは別物であるため、検索結果の混同に注意してください。
