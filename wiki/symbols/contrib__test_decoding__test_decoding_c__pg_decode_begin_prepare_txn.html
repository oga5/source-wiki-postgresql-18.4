pg_decode_begin_prepare_txn

定義

  • 種別: function
  • 定義箇所: contrib/test_decoding/test_decoding.c:349

目的

論理デコーディングにおける「準備済みトランザクション（Prepared Transaction）」の開始イベントを処理し、トランザクションごとの状態管理用データ構造を初期化する。

シグネチャまたは形状

static void pg_decode_begin_prepare_txn(LogicalDecodingContext *ctx, ReorderBufferTXN *txn)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • txn: 処理対象のトランザクション情報（ReorderBufferTXN）。
  • **出力**: なし（void）。副作用として txn->output_plugin_private に TestDecodingTxnData が割り当てられる。

前提条件と不変条件

  • ctx->output_plugin_private に TestDecodingData が格納されていること。
  • ctx->context が有効なメモリコンテキストであること。

アルゴリズムまたは振る舞い

1. ctx->context 上に TestDecodingTxnData 構造体をゼロクリアして確保する。 2. txndata->xact_wrote_changes を false に初期化する。 3. 確保した txndata を txn->output_plugin_private に保持させる。 4. data->skip_empty_xacts が有効な場合、即座に終了する（この場合、BEGIN の出力は後続の操作発生時まで遅延される）。 5. skip_empty_xacts が無効な場合、pg_output_begin を呼び出し、準備済みトランザクション開始の情報を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの論理デコーディングエンジン（_PG_output_plugin_init で begin_prepare_cb として登録される）。
  • **呼び出し先**:
  • MemoryContextAllocZero: メモリ確保。
  • pg_output_begin: トランザクション開始情報の出力（条件付き）。

関連データ構造

  • LogicalDecodingContext: デコーディングの実行コンテキスト。
  • ReorderBufferTXN: トランザクションの状態を保持する構造体。
  • TestDecodingData: プラグイン全体の設定情報。
  • TestDecodingTxnData: トランザクションごとの状態管理用構造体。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocZero がエラーを発生させる可能性がある。
  • skip_empty_xacts 設定により、トランザクション開始の出力タイミングが制御される。

保守担当者向けメモ

  • 本関数は begin_prepare_cb コールバックとして登録される。
  • txn->output_plugin_private に割り当てられたメモリは、トランザクション終了時に適切に解放される必要がある（関連する commit や rollback コールバックの確認を推奨）。
