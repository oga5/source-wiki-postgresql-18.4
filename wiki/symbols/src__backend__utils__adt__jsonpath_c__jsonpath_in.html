jsonpath_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonpath.c:97

目的

SQL文字列として提供されたJSONパス式を、PostgreSQL内部で利用可能な jsonpath 型のバイナリ表現へ変換します。

シグネチャまたは形状

Datum jsonpath_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して取得される、JSONパス式を表すC文字列。
  • **出力**: jsonPathFromCstring の戻り値である Datum 型（jsonpath 型のバイナリ表現）。

前提条件と不変条件

  • 入力文字列は有効なSQL/JSONパス構文に従っている必要があります。
  • fcinfo->context を介して、メモリ管理のためのコンテキスト情報が渡される必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) を使用して入力文字列を取得します。 2. strlen を用いて入力文字列の長さを算出します。 3. jsonPathFromCstring を呼び出し、文字列と長さを渡して解析およびバイナリ形式への変換を実行します。 4. jsonPathFromCstring の結果をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_jsontable.c:508 (DirectFunctionCall1経由)
  • **呼び出し先**:
  • jsonPathFromCstring (同ファイル内)

関連データ構造

  • JsonPath: jsonpath 型のメモリ上の構造体。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 入力文字列が不正な構文である場合、jsonPathFromCstring 内部でエラーが発生し、トランザクションがアボートされる可能性があります。
  • メモリ不足時には、PostgreSQLのメモリ管理機構によりエラーがスローされます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の解析ロジックは jsonPathFromCstring に依存しています。
  • 複雑なパス式の解析はスタックを消費するため、再帰的な処理を行う際は check_stack_depth() が適切に機能しているか確認してください。
