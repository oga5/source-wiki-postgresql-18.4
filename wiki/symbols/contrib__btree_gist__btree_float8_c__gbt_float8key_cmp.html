gbt_float8key_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float8.c:55

目的

GiSTインデックスにおける float8KEY 構造体同士の比較を行い、ソートやインデックス構築時の順序判定を行う。

シグネチャまたは形状

int gbt_float8key_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部で float8KEY にキャストされる。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • 0: a と b が等しい場合。
  • 1: a が b より大きい場合。
  • -1: a が b より小さい場合。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体を介して float8KEY を保持している必要がある。
  • float8KEY は lower および upper フィールドを持つ。

アルゴリズムまたは振る舞い

1. a と b を Nsrt 構造体経由で float8KEY 型にキャストする。 2. まず lower 値を比較する。

  • ia->lower != ib->lower の場合、その大小関係を返す。

    • 3. lower 値が等しい場合、upper 値を比較する。

  • ia->upper == ib->upper ならば 0 を返す。
  • ia->upper > ib->upper ならば 1 を返す。
  • それ以外（ia->upper < ib->upper）ならば -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_float8.c 内の tinfo 構造体定義（96行目）にて参照されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • float8KEY: float8 型の範囲を保持する構造体。
  • Nsrt: ソート処理で使用される中間構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動は定義されていない。
  • 浮動小数点数の比較において、NaN が含まれる場合の挙動は標準的な比較演算子に依存する。

保守担当者向けメモ

  • 本関数は btree_utils_num が提供するソートサポート機能の一部として利用される。
  • 比較ロジックは lower を優先し、同値の場合のみ upper を考慮する辞書順に近い順序付けとなっている。
