ExecInitSampleScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSamplescan.c:92

目的

TABLESAMPLE 句を用いたスキャン実行ノード（SampleScanState）を初期化し、サンプリング手法の準備と式評価のセットアップを行う。

シグネチャまたは形状

SampleScanState *ExecInitSampleScan(SampleScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の SampleScan プランノード。
  • estate: 実行状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの SampleScanState 構造体へのポインタ。

前提条件と不変条件

  • node の outerPlan および innerPlan は NULL であること（Assert による検証）。
  • TABLESAMPLE の REPEATABLE 句が指定されていない場合、乱数シードが pg_global_prng_state から生成される。
  • BeginSampleScan の実行は、パラメータ評価が必要なため本関数内では行われない（scanstate->begun = false）。

アルゴリズムまたは振る舞い

1. SampleScanState ノードを割り当て、ExecSampleScan を実行関数として登録する。 2. 式コンテキストの作成、スキャン対象リレーションのオープン、およびタプルスロットの初期化を行う。 3. TABLESAMPLE 句の引数式および REPEATABLE 式を初期化する。 4. REPEATABLE 句がない場合、再スキャン時にも一貫性を保つため、乱数シードを一度だけ生成する。 5. GetTsmRoutine を呼び出し、指定されたサンプリング手法（TsmRoutine）を取得する。 6. サンプリング手法が InitSampleScan を定義している場合、それを呼び出して初期化を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode から呼び出される（直接の呼び出し元は ExecInitSampleScan への参照として確認可能）。
  • **呼び出し先**:
  • ExecAssignExprContext, ExecOpenScanRelation, ExecInitScanTupleSlot, ExecInitResultTypeTL, ExecAssignScanProjectionInfo, ExecInitQual, ExecInitExprList, ExecInitExpr
  • pg_prng_uint32
  • GetTsmRoutine
  • tsm->InitSampleScan (サンプリング手法固有の初期化関数)

関連データ構造

  • SampleScanState: スキャン実行状態を保持する構造体。
  • SampleScan: プランノード。
  • TsmRoutine: サンプリング手法のインターフェース定義。
  • TableSampleClause: TABLESAMPLE 句のパラメータ。

失敗モードと境界条件

  • GetTsmRoutine が無効なハンドラを渡された場合、またはリレーションのオープンに失敗した場合はエラーとなる。
  • Assert 条件（子プランの不在）が満たされない場合、デバッグビルドではアボートする。

保守担当者向けメモ

  • 本関数は BeginSampleScan を呼び出さないため、実際のサンプリング開始は SampleNext 実行時の tablesample_init に委譲されている。
  • REPEATABLE 句の有無によるシード生成ロジックは、再スキャン（ReScan）時にも値が保持されるよう設計されている。
