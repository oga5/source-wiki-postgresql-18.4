ExceptionMap

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:43

目的

PostgreSQL の SQLSTATE エラーコードと、それに対応する Python の例外クラス名を紐付けるためのマッピング構造体です。

シグネチャまたは形状

typedef struct ExceptionMap
{
	char	   *name;       /* 例外の名称 */
	char	   *classname;  /* Python側でのクラス名 */
	int			sqlstate;   /* 対応するPostgreSQLのSQLSTATE */
} ExceptionMap;

入力と出力

本構造体はデータ保持用であり、直接的な関数入力や出力は持ちません。静的な配列 exception_map の要素として定義され、PL/Python モジュールの初期化時に参照されます。

前提条件と不変条件

  • sqlstate は PostgreSQL の有効なエラーコードである必要があります。
  • name および classname は、Python の例外クラスとして生成可能な文字列である必要があります。

アルゴリズムまたは振る舞い

PL/Python モジュールの初期化プロセスにおいて、この構造体の配列を走査し、各要素に基づいて Python の例外クラスを動的に生成して plpy.spiexceptions 名前空間に登録するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_plpymodule.c 内の exception_map 配列定義にて使用されています。
  • **呼び出し先**: 本構造体自体はデータ定義であり、関数を呼び出すことはありません。

関連データ構造

  • exception_map[]: ExceptionMap 型の静的配列。
  • PLy_spi_exceptions: SQLSTATE と Python 例外オブジェクトを紐付けるハッシュテーブル。

失敗モードと境界条件

  • 定義された sqlstate が PostgreSQL のシステムカタログと一致しない場合、例外の自動生成が意図通りに行われない可能性があります。
  • メモリ不足により例外クラスの生成に失敗した場合、Python インタプリタの初期化が中断される可能性があります。

保守担当者向けメモ

  • 本構造体は spiexceptions.h で定義される SQLSTATE 定義と密接に関連しています。
  • exception_map 配列を更新する際は、Python 側の例外階層との整合性に注意してください。
