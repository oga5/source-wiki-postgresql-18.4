DefineTSTemplate

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tsearchcmds.c:689

目的

全文検索テンプレート（Text Search Template）をシステムカタログ pg_ts_template に新規作成し、永続化する。

シグネチャまたは形状

ObjectAddress DefineTSTemplate(List *names, List *parameters)

入力と出力

  • **入力**:
  • names: テンプレート名を指定する List。
  • parameters: init および lexize 関数を指定する DefElem のリスト。
  • **出力**:
  • 作成されたテンプレートの ObjectAddress を返す。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • lexize メソッドの指定が必須。

アルゴリズムまたは振る舞い

1. 権限チェックを行い、スーパーユーザーでない場合はエラーを投げる。 2. names から名前空間とテンプレート名を解決する。 3. pg_ts_template カタログを RowExclusiveLock で開く。 4. parameters を走査し、init および lexize パラメータから対応する関数 OID を取得・設定する。未知のパラメータが指定された場合はエラーとする。 5. lexize メソッドが指定されていない場合はエラーを投げる。 6. heap_form_tuple でタプルを作成し、CatalogTupleInsert でカタログへ挿入する。 7. makeTSTemplateDependencies を呼び出し、依存関係を構築する。 8. InvokeObjectPostCreateHook を呼び出し、作成後のフック処理を実行する。 9. カタログを閉じて ObjectAddress を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:1431（SQL解析後の実行）
  • **呼び出し先**:
  • superuser, QualifiedNameGetCreationNamespace, table_open, GetNewOidWithIndex, get_ts_template_func, heap_form_tuple, CatalogTupleInsert, makeTSTemplateDependencies, InvokeObjectPostCreateHook, heap_freetuple, table_close

関連データ構造

  • pg_ts_template: テンプレート情報を保持するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • スーパーユーザー権限がない場合、ERRCODE_INSUFFICIENT_PRIVILEGE で失敗する。
  • 未知のパラメータが指定された場合、ERRCODE_SYNTAX_ERROR で失敗する。
  • lexize 関数が指定されていない場合、ERRCODE_INVALID_OBJECT_DEFINITION で失敗する。

保守担当者向けメモ

  • 本関数は DDL 処理の一部であり、カタログ操作と依存関係の構築を担う。
  • テンプレートの初期化関数（init）は省略可能だが、lexize は必須である点に注意。
