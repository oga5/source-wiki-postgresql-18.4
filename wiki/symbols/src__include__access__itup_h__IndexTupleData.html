IndexTupleData

定義

  • 種別: struct
  • 定義箇所: src/include/access/itup.h:35

目的

PostgreSQLのインデックスページ内に格納されるタプルの物理的な先頭構造を定義します。ヒープタプルへの参照（TID）と、タプルのサイズや属性状態を管理する制御情報を保持します。

シグネチャまたは形状

typedef struct IndexTupleData
{
    ItemPointerData t_tid;      /* ヒープタプルへの参照TID */
    unsigned short t_info;      /* タプルの各種情報 */
} IndexTupleData;

入力と出力

  • **入力**: 構造体自体はメモリ上のデータレイアウトを規定します。
  • **出力**: t_info ビットフィールドを通じて、タプルのサイズ、NULL値の有無、可変長属性の有無を外部（アクセス手法）へ提供します。

前提条件と不変条件

  • IndexTupleData の直後に、NULLビットマップ（存在する場合）が配置され、その後に属性値が MAXALIGN 境界で並びます。
  • t_info のビット構成:
  • 15ビット目: NULL値の有無
  • 14ビット目: 可変長属性の有無
  • 13ビット目: アクセス手法（AM）固有の定義
  • 12-0ビット目: タプル全体のサイズ

アルゴリズムまたは振る舞い

インデックスアクセス手法（AM）は、この構造体を基点としてポインタ演算を行い、後続するNULLビットマップや属性データにアクセスします。index_getattr などの関数がこの構造体を参照し、t_info のフラグに基づいて属性値のオフセットを計算します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck, contrib/pageinspect, src/backend/access/common/indextuple.c, src/backend/access/gin/, src/backend/access/gist/ 等のインデックス関連モジュールが、タプルの解析や構築のために本構造体を参照しています。
  • **呼び出し先**: 本構造体自体はデータ定義であり、関数を呼び出すことはありません。

関連データ構造

  • ItemPointerData: t_tid フィールドの型。
  • IndexAttributeBitMapData: NULL値管理に使用される関連構造体。

失敗モードと境界条件

  • t_info のサイズフィールド（13ビット）により、1つのインデックスエントリの最大サイズに制限がかかります。
  • 構造体直後のメモリレイアウトを誤って計算すると、属性値の読み取り時に不正なメモリ参照が発生します。

保守担当者向けメモ

  • 本構造体の直後に可変長データが続くため、sizeof(IndexTupleData) を使用したポインタ演算を行う際は、必ず MAXALIGN を考慮してください。
  • t_info の13ビット目はAM依存であるため、特定のAMで利用する際はその仕様を確認してください。
