InitArchiveFmt_Directory

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_directory.c:109

目的

ディレクトリ形式アーカイブの初期化を行い、ArchiveHandle 構造体にディレクトリ形式特有の関数ポインタ群を登録し、実行時コンテキスト（lclContext）を構築します。

シグネチャまたは形状

void InitArchiveFmt_Directory(ArchiveHandle *AH);

入力と出力

  • **入力**: ArchiveHandle *AH（アーカイブ処理のメインハンドル）
  • **出力**: なし（副作用として AH 内の関数ポインタ群および formatData メンバを更新）

前提条件と不変条件

  • AH->fSpec に有効なディレクトリパスが指定されていること。
  • archModeWrite モードの場合、指定されたディレクトリが存在するか、作成可能であること。
  • archModeRead モードの場合、指定ディレクトリ内に toc.dat が存在すること。

アルゴリズムまたは振る舞い

1. **関数ポインタの登録**: _ArchiveEntry, _StartData, _CloseArchive 等、ディレクトリ形式用の各操作関数を AH に割り当てます。 2. **コンテキストの初期化**: lclContext 構造体をメモリ確保し、AH->formatData に格納します。 3. **モード別処理**:

  • **書き込みモード**: create_or_open_dir を呼び出し、出力先ディレクトリを準備します。
  • **読み込みモード**: toc.dat を特定し、InitDiscoverCompressFileHandle で開きます。その後、ReadHead および ReadToc を実行してアーカイブのメタデータを読み込み、最後に toc.dat を閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_archiver.c:2478
  • **呼び出し先**: pg_malloc0, create_or_open_dir, setFilePath, InitDiscoverCompressFileHandle, ReadHead, ReadToc, EndCompressFileHandle, pg_fatal

関連データ構造

  • ArchiveHandle: アーカイブ処理のメイン構造体。
  • lclContext: ディレクトリ形式の実行時状態を保持する構造体。

失敗モードと境界条件

  • fSpec が空または未指定の場合、pg_fatal で異常終了します。
  • 読み込みモードで toc.dat が開けない場合、または toc.dat のクローズに失敗した場合、pg_fatal で異常終了します。

保守担当者向けメモ

  • 本関数は tar 形式と共通の TOC 構造を前提としています。変更時は pg_backup_tar.c との整合性に注意してください。
  • AH->format を一時的に archTar に切り替えて ReadHead を実行する特殊な手順が含まれています。
