assign_expr_collations

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_collate.c:176

目的

指定された式（Node *expr）に対して照合順序（Collation）の割り当てと解決を行うエントリーポイントです。

シグネチャまたは形状

void assign_expr_collations(ParseState *pstate, Node *expr)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。
  • expr: 照合順序を解決すべき対象の構文木ノード。
  • **出力**: なし（戻り値は void）。副作用として、対象ノードおよびその配下のノードに対して照合順序が決定・付与されます。

前提条件と不変条件

  • pstate は有効な解析状態を指している必要があります。
  • expr は解析済みの構文木ノードである必要があります。

アルゴリズムまたは振る舞い

1. assign_collations_context 構造体を初期化し、照合順序の解決状態（collation = InvalidOid, strength = COLLATE_NONE）をリセットします。 2. assign_collations_walker を呼び出し、対象の式ツリーを再帰的に走査します。 3. 走査過程で各ノードの照合順序をバブルアップさせ、競合の検出や適切な照合順序の決定を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/commands/copy.c
  • src/backend/commands/functioncmds.c
  • src/backend/commands/policy.c
  • src/backend/commands/prepare.c
  • src/backend/commands/publicationcmds.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • assign_collations_walker（直接の再帰的走査）

関連データ構造

  • ParseState: 解析状態を管理する構造体。
  • assign_collations_context: 走査中に照合順序の状態を保持するためのコンテキスト構造体。

失敗モードと境界条件

  • 複数の式間で異なる照合順序が競合した場合、assign_collations_walker 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は式単位の照合順序解決の起点です。クエリ全体を対象とする場合は assign_query_collations が使用されます。
  • 照合順序の解決は構文解析後の「後処理」として実装されており、メモリ効率を考慮して再帰的なウォーカーが採用されています。修正時は assign_collations_walker のロジックと整合性を保つ必要があります。
