GinChkVal

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int_bool.c:319

目的

GINインデックスの整合性チェック処理において、クエリツリーの各ノードと、それに対応するチェック結果（真偽値）を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    ITEM       *first;
    bool       *mapped_check;
} GinChkVal;

入力と出力

  • **入力**: GINインデックスの整合性チェック関数（gin_bool_consistent 等）から、checkval ポインタとしてキャストされて渡されます。
  • **出力**: 構造体内の mapped_check 配列を通じて、各クエリノードの評価結果が格納されます。

前提条件と不変条件

  • first はクエリツリーの先頭（ITEM 配列）を指している必要があります。
  • mapped_check は、対応するクエリツリーのノード数分以上のメモリ領域が確保されている必要があります。

アルゴリズムまたは振る舞い

GINインデックスの検索時に、インデックス内のキーがクエリ条件を満たすかを判定する際、この構造体を使用してクエリツリーの各要素の評価状態を追跡します。checkcondition_gin 等の関数内で、GinChkVal にキャストされたポインタを経由して、特定のノードの真偽値の読み書きが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_bool.c 内の gin_bool_consistent 等の関数。
  • src/backend/utils/adt/tsginidx.c 内の関連関数（PostgreSQLコアのテキスト検索用GINインデックス処理）。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わず、データ保持用として参照されます。

関連データ構造

  • ITEM: クエリツリーを構成する各ノード（値または演算子）。
  • QUERYTYPE: クエリ全体を保持する構造体。

失敗モードと境界条件

  • mapped_check が指すメモリ領域が不足している場合、バッファオーバーランが発生する可能性があります。
  • first が無効なメモリを指している場合、整合性チェック時にセグメンテーション違反が発生します。

保守担当者向けメモ

  • 本構造体は contrib/intarray だけでなく、src/backend/utils/adt/tsginidx.c でも同名で定義・利用されています。変更を加える際は、PostgreSQLコア側の実装への影響を考慮してください。
