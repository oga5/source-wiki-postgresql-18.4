ConditionVariableBroadcast

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/condition_variable.c:281

目的

条件変数（ConditionVariable）で待機しているすべてのプロセスを起床させます。

シグネチャまたは形状

void ConditionVariableBroadcast(ConditionVariable *cv)

入力と出力

  • **入力**: cv (操作対象の ConditionVariable 構造体へのポインタ)
  • **出力**: なし（副作用として待機中プロセスのラッチがセットされる）

前提条件と不変条件

  • 呼び出し側が cv_sleep_target を介して待機準備中である場合、本関数内で ConditionVariableCancelSleep() が自動的に呼び出され、待機状態が解除されます。
  • 待機リストの操作は cv->mutex（スピンロック）によって保護されます。
  • 呼び出し側プロセス自身が待機リストに含まれていないことがアサーションにより保証されます。

アルゴリズムまたは振る舞い

1. **待機準備の解除**: 呼び出し側が待機準備中であれば、ConditionVariableCancelSleep を呼び出してリストから自身を削除します。 2. **センチネルの配置**: 待機リストが空でない場合、最初のプロセスを起床対象として取り出します。リストにまだ他のプロセスが残っている場合、呼び出し側自身の cvWaitLink をリストの末尾に「センチネル（番兵）」として挿入します。 3. **初期起床**: 取り出した最初のプロセスに対して SetLatch を実行します。 4. **ループ処理**: センチネルがリスト内に存在する限り、リストの先頭からプロセスを順次取り出し、SetLatch を実行して起床させます。 5. **終了条件**: センチネルがリストから除去された時点で処理を終了します。他者によるシグナルでセンチネルが除去された場合でも、その時点ですべての待機者が処理済みであると判断します。

呼び出し元と呼び出し先

  • **呼び出し元**: nbtree.c, xlogrecovery.c, nodeBitmapHeapscan.c, checkpointer.c, walsummarizer.c, origin.c, slot.c 等の各モジュール。
  • **呼び出し先**: ConditionVariableCancelSleep, SpinLockAcquire, SpinLockRelease, proclist_contains, proclist_is_empty, proclist_pop_head_node, proclist_push_tail, SetLatch。

関連データ構造

  • ConditionVariable: 待機リストとスピンロックを保持する構造体。
  • PGPROC: プロセス情報を保持し、procLatch を通じて起床通知を受け取る構造体。
  • proclist: cvWaitLink を用いてプロセスを管理するリスト構造。

失敗モードと境界条件

  • **空のリスト**: 待機者がいない場合、何も行わず即座に復帰します。
  • **他者による割り込み**: 処理中に他者が ConditionVariableSignal 等を呼び出し、センチネルがリストから除去される可能性があります。この場合、余分に1プロセスを起床させる可能性がありますが、これは安全な挙動として許容されます。
  • **再帰的なキューイング**: 起床されたプロセスが即座に再待機する場合でも、センチネルを用いることで無限ループを回避します。

保守担当者向けメモ

  • 本関数は、待機リストを効率的に空にするために自身の cvWaitLink をセンチネルとして利用する巧妙なアルゴリズムを採用しています。
  • センチネルがリストから消滅した際の挙動は、意図的な「余分な起床」を含んでおり、デッドロックや起床漏れを防ぐための設計です。
  • CHECK_FOR_INTERRUPTS() が含まれる可能性があるため、呼び出し側は割り込みに対して安全である必要があります。
