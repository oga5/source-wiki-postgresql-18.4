ExecEndCteScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCtescan.c:287

目的

CTE（共通テーブル式）スキャンノードの実行終了処理を行い、CTEの結果を保持する Tuplestore のリソースを解放する。

シグネチャまたは形状

void ExecEndCteScan(CteScanState *node)

入力と出力

  • **入力**: CteScanState *node - 終了対象のCTEスキャンノードの状態構造体。
  • **出力**: なし。

前提条件と不変条件

  • node は初期化済みであること。
  • node->leader が自身（node）を指している場合、そのノードが Tuplestore の所有者（リーダー）であるとみなされる。

アルゴリズムまたは振る舞い

1. 引数 node がリーダーノードであるかを確認する（node->leader == node）。 2. リーダーである場合、tuplestore_end(node->cte_table) を呼び出して Tuplestore を破棄し、メモリ等のリソースを解放する。 3. node->cte_table を NULL に設定し、二重解放を防止する。 4. リーダーでない場合、何もしない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:676 にて呼び出されていることが確認されている。
  • **呼び出し先**: tuplestore_end（src/backend/utils/sort/tuplestore.c 内の関数）。

関連データ構造

  • CteScanState: CTEスキャンの実行状態を保持する構造体。leader ポインタおよび cte_table（Tuplestore へのポインタ）を含む。

失敗モードと境界条件

  • node が NULL の場合、あるいは node->cte_table が既に解放されている場合の動作については、本ソース抜粋からは判断できない。

保守担当者向けメモ

  • 本関数はリーダーノードのみが Tuplestore の解放責任を持つという設計に基づいている。共有されている Tuplestore を誤って早期解放しないよう注意すること。
