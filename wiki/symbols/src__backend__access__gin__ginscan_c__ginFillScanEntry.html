ginFillScanEntry

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginscan.c:56

目的

GINインデックススキャンにおいて、検索条件に対応する GinScanEntry を生成または再利用し、スキャン状態（GinScanOpaque）に登録します。

シグネチャまたは形状

GinScanEntry ginFillScanEntry(GinScanOpaque so, OffsetNumber attnum,
                              StrategyNumber strategy, int32 searchMode,
                              Datum queryKey, GinNullCategory queryCategory,
                              bool isPartialMatch, Pointer extra_data)

入力と出力

  • **入力**:
  • so: スキャン状態を保持する GinScanOpaque 構造体。
  • attnum, strategy, searchMode, queryKey, queryCategory, isPartialMatch, extra_data: 検索条件を定義する各属性。
  • **出力**:
  • 検索条件に合致する既存の、または新規作成された GinScanEntry へのポインタ。

前提条件と不変条件

  • extra_data が NULL でない場合、重複排除は行われません。
  • so->totalentries が 100 未満の場合のみ、既存エントリとの重複排除（ginCompareEntries による比較）が試行されます。

アルゴリズムまたは振る舞い

1. **重複排除**: extra_data が NULL かつ totalentries < 100 の場合、既存の so->entries を走査し、属性およびキーが一致するエントリがあればそれを返します。 2. **新規作成**: 一致するものがない場合、palloc で GinScanEntryData を確保し、引数で渡された各値を設定します。 3. **初期化**: 新規エントリのバッファ、ビットマップ、イテレータ等の検索状態を初期値（InvalidBuffer, NULL, false 等）で初期化します。 4. **登録**: so->entries 配列にエントリを追加します。配列の容量が不足している場合は repalloc で倍増させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginscan.c 内の ginNewScanKey（複数箇所）。
  • **呼び出し先**:
  • ginCompareEntries（ginutil.c 経由の可能性あり）
  • palloc / repalloc（メモリ管理）

関連データ構造

  • GinScanOpaque: スキャン全体の状態を管理。
  • GinScanEntry: 個別の検索キーの状態を保持。
  • GinScanKey: GinScanEntry を束ねる構造体。

失敗モードと境界条件

  • **メモリ不足**: palloc または repalloc が失敗した場合、PostgreSQL のメモリ管理機構によりエラーがスローされます。
  • **重複排除の制限**: 100個以上のエントリが存在する場合、重複排除ロジックをスキップして新規作成を行うため、メモリ使用量が増加する可能性があります。

保守担当者向けメモ

  • 重複排除の閾値（100）は任意に設定された値であり、パフォーマンスとメモリ効率のトレードオフに基づいています。
  • extra_data を持つエントリは、演算子クラスの内部動作が不明であるため、一律で重複排除の対象外としています。
