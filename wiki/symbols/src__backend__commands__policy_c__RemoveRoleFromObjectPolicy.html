RemoveRoleFromObjectPolicy

定義

  • 種別: function
  • 定義箇所: src/backend/commands/policy.c:415

目的

指定されたポリシー定義から特定のロール（roleid）を削除し、ポリシーの依存関係を更新します。ロールが削除された結果、ポリシーに関連付けられたロールが一つも残らなくなる場合は、ポリシー自体を削除すべきであることを呼び出し元に通知します。

シグネチャまたは形状

bool RemoveRoleFromObjectPolicy(Oid roleid, Oid classid, Oid policy_id)

入力と出力

  • **入力**:
  • roleid: 削除対象のロールOID。
  • classid: 対象オブジェクトのクラスOID（PolicyRelationId である必要がある）。
  • policy_id: 編集対象のポリシーOID。
  • **出力**:
  • bool: ポリシーを保持し続ける場合は true、ロールが残らずポリシーを削除すべき場合は false を返します。

前提条件と不変条件

  • classid は PolicyRelationId と一致しなければなりません（Assert で検証）。
  • 指定された policy_id に対応するポリシーが pg_policy カタログに存在する必要があります。存在しない場合はエラーをスローします。

アルゴリズムまたは振る舞い

1. pg_policy テーブルを RowExclusiveLock で開きます。 2. policy_id をキーにシステムスキャンを行い、対象のポリシー行を取得します。 3. 現在の polroles 配列から roleid を除外した新しいロールリストを構築します。 4. ロールが残る場合:

  • pg_policy の該当行を更新し、新しいロール配列を格納します。
  • 既存の共有依存関係を削除し、残ったロールに対して新しい共有依存関係を登録します。
  • InvokeObjectPostAlterHook を呼び出し、キャッシュ無効化（CacheInvalidateRelcacheByTuple）を実行して変更を反映させます。
  • true を返します。

    • 5. ロールが一つも残らない場合:

  • false を返します（ポリシーの削除は呼び出し元に委ねられます）。

    • 6. 最後にスキャンを終了し、テーブルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_shdepend.c:1421 からの呼び出しが確認されています。
  • **呼び出し先**: table_open, systable_beginscan, heap_modify_tuple, CatalogTupleUpdate, deleteSharedDependencyRecordsFor, recordSharedDependencyOn, CacheInvalidateRelcacheByTuple 等のカタログ操作関数群。

関連データ構造

  • pg_policy: ポリシー定義を保持するシステムカタログ。
  • RowSecurityDesc: RLS記述子（本関数は直接操作しませんが、キャッシュ無効化を通じて影響を与えます）。

失敗モードと境界条件

  • 指定された policy_id が存在しない場合、elog(ERROR) によりトランザクションが中断されます。
  • ロールが重複して登録されている場合、すべて削除されるまで処理が継続されます。
  • public ロール（ACL_ID_PUBLIC）に対する依存関係は登録されません。

保守担当者向けメモ

  • 本関数はポリシーの「定義」を修正するものであり、ポリシー自体の削除（DROP POLICY）は呼び出し元が本関数の戻り値 false を受けて実行する必要があります。
  • CommandCounterIncrement() が呼び出されるため、本関数の実行後にカタログの変更が可視化されます。
