generate_dependencies

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/dependencies.c:156

目的

関数従属性の計算に必要な属性の組み合わせを再帰的に生成するためのエントリポイントとして機能します。

シグネチャまたは形状

static void generate_dependencies(DependencyGenerator state)

入力と出力

  • **入力**: DependencyGenerator state（属性の組み合わせ生成状態を保持する構造体へのポインタ）
  • **出力**: なし（副作用として生成された組み合わせが処理されます）

前提条件と不変条件

  • state は DependencyGenerator_init によって初期化されている必要があります。
  • state->k は生成する組み合わせのサイズを保持しており、メモリ確保の基準として使用されます。

アルゴリズムまたは振る舞い

1. state->k のサイズで AttrNumber 型の配列 current をメモリ確保（palloc0）します。 2. generate_dependencies_recurse を呼び出し、属性の組み合わせ生成を開始します。 3. 再帰処理の完了後、確保した current 配列を解放（pfree）します。

呼び出し元と呼び出し先

  • **呼び出し元**: statext_dependencies_build（src/backend/statistics/dependencies.c:189）
  • **呼び出し先**: generate_dependencies_recurse

関連データ構造

  • DependencyGenerator: 属性の組み合わせ生成状態を管理する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc0 がエラーを発生させます。
  • state が NULL の場合: 構造体メンバへのアクセスによりクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の組み合わせ生成ロジックは generate_dependencies_recurse に委譲されています。
  • current 配列のメモリ管理は本関数内で完結しており、再帰呼び出し中にのみ有効な一時領域として使用されます。
