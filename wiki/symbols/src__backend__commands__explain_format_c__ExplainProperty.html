ExplainProperty

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:160

目的

EXPLAIN コマンドの実行計画における特定のプロパティ（キーと値のペア）を、現在の出力形式（TEXT, XML, JSON, YAML）に合わせて整形し、出力バッファへ書き込みます。

シグネチャまたは形状

static void ExplainProperty(const char *qlabel, const char *unit, const char *value, bool numeric, ExplainState *es);

入力と出力

  • qlabel: プロパティのラベル名。
  • unit: 単位（TEXT形式でのみ使用、不要な場合は NULL）。
  • value: 出力する値の文字列。
  • numeric: 値が数値として扱われるべきかを示すフラグ（JSON/YAMLでの引用符付与を制御）。
  • es: 出力状態を保持する ExplainState 構造体。
  • 出力: 戻り値なし。es->str にフォーマット済みの文字列が追加されます。

前提条件と不変条件

  • es は初期化済みであること。
  • es->format に基づいて適切な出力処理が行われるため、es のフォーマット状態と整合している必要があります。

アルゴリズムまたは振る舞い

es->format に基づき以下の処理を行います。

  • **TEXT**: ExplainIndentText でインデントし、ラベル: 値 [単位] の形式で出力。
  • **XML**: インデント後、qlabel をタグ名として escape_xml された value を囲むタグを出力。
  • **JSON**: ExplainJSONLineEnding で区切りを管理し、"ラベル": 値 の形式で出力。numeric が偽の場合は escape_json を適用。
  • **YAML**: ExplainYAMLLineStarting で行頭を処理し、ラベル: 値 の形式で出力。numeric が偽の場合は escape_yaml を適用。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/explain_format.c 内の各所（ExplainPropertyText, ExplainPropertyInteger 等のラッパー関数から呼び出される）。
  • 呼び出し先: ExplainIndentText, ExplainXMLTag, escape_xml, appendStringInfo, ExplainJSONLineEnding, escape_json, ExplainYAMLLineStarting, escape_yaml 等。

関連データ構造

  • ExplainState: 出力バッファ、フォーマット種別、インデントレベルを管理する構造体。

失敗モードと境界条件

  • value が NULL の場合、呼び出し側で適切に文字列化されている必要があります（例: "true" / "false"）。
  • XML形式では、タグ名（qlabel）に不正な文字が含まれる場合、ExplainXMLTag 内部でのサニタイズ処理に依存します。

保守担当者向けメモ

  • 新しいフォーマットを追加する場合は、本関数の switch 文にケースを追加する必要があります。
  • numeric フラグは JSON/YAML における型情報の保持に重要であるため、呼び出し側で正しく設定してください。
