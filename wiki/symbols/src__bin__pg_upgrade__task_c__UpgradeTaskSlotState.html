UpgradeTaskSlotState

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_upgrade/task.c:92

目的

pg_upgrade における並列タスク実行フレームワークにおいて、各接続スロットの現在の処理状態を管理するために使用されます。

シグネチャまたは形状

typedef enum UpgradeTaskSlotState
{
    FREE,
    CONNECTING,
    RUNNING_QUERIES,
} UpgradeTaskSlotState;

入力と出力

  • 本シンボル自体は状態値の列挙体であり、入力や出力は持ちません。
  • UpgradeTaskSlot 構造体の state メンバとして保持され、process_slot() 関数内で状態遷移の判断基準として利用されます。

前提条件と不変条件

  • 各スロットは常に FREE、CONNECTING、RUNNING_QUERIES のいずれかの状態をとります。
  • FREE 状態は、新しいデータベース処理のためにスロットが利用可能であることを示します。

アルゴリズムまたは振る舞い

process_slot() 内のステートマシンにおいて、以下の遷移を制御します。

  • FREE: 新しいデータベース接続を開始する準備状態。
  • CONNECTING: PQconnectPoll を用いて接続確立を待機する状態。
  • RUNNING_QUERIES: PQsendQuery を用いてクエリを実行し、結果を処理する状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/task.c 内の process_slot() 関数がこの状態を参照・更新します。
  • 呼び出し先: 該当なし（列挙型定義のため）。

関連データ構造

  • UpgradeTaskSlot: この列挙型をメンバとして保持する構造体。

失敗モードと境界条件

  • 状態遷移の整合性は process_slot() の実装に依存します。本シンボル自体に境界条件はありません。

保守担当者向けメモ

  • 新しい状態を追加する場合は、process_slot() 内のステートマシンロジックを更新し、すべての状態遷移が網羅されていることを確認してください。
