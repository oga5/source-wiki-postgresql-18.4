InitArchiveFmt_Custom

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_custom.c:104

目的

pg_dump のカスタム形式アーカイブを処理するための関数ポインタ群を初期化し、アーカイブのコンテキスト（lclContext）を割り当ててファイル操作の準備を行う。

シグネチャまたは形状

void InitArchiveFmt_Custom(ArchiveHandle *AH)

入力と出力

  • **入力**: ArchiveHandle *AH（アーカイブ操作の抽象ハンドル）
  • **出力**: なし（AH 構造体内の関数ポインタおよび formatData フィールドを更新）

前提条件と不変条件

  • AH は有効な ArchiveHandle 構造体へのポインタであること。
  • AH->mode に基づき、書き込み（archModeWrite）または読み込みのいずれかのモードで初期化される。

アルゴリズムまたは振る舞い

1. **関数ポインタの割り当て**: ArchiveHandle が保持する各種操作用ポインタ（_ArchiveEntry, _WriteData, _PrintTocData 等）をカスタム形式固有の実装へ設定する。 2. **並列処理設定**: 並列ダンプはサポートせず、並列リストア用の関数（_WorkerJobRestoreCustom）を割り当てる。 3. **コンテキスト初期化**: lclContext 構造体をメモリ確保し、AH->formatData に格納する。 4. **ファイルオープン**:

  • 書き込みモード時: AH->fSpec が指定されていればファイルを開き、なければ stdout を使用する。
  • 読み込みモード時: AH->fSpec が指定されていればファイルを開き、なければ stdin を使用する。

    • 5. **状態確認**: checkSeek を用いてファイルがシーク可能かを判定し、ctx->hasSeek に記録する。 6. **読み込み時処理**: ReadHead(AH) および ReadToc(AH) を呼び出し、ヘッダーと目次情報を読み込む。その後、データブロック開始位置を ctx->lastFilePos に保存する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_archiver.c:2470
  • **呼び出し先**:
  • pg_malloc0, fopen, checkSeek, ReadHead, ReadToc, _getFilePos
  • 各種カスタム形式関数（_ArchiveEntry, _StartData, _WriteData, _PrintTocData 等）

関連データ構造

  • ArchiveHandle: アーカイブ全体の管理構造体。
  • lclContext: カスタム形式固有のコンテキスト（ファイルハンドル、シーク可否、ファイル位置等）。

失敗モードと境界条件

  • ファイルオープン失敗時: pg_fatal を呼び出し、エラーメッセージを出力してプロセスを終了する。
  • fSpec が空文字列の場合、標準入出力が使用される。

保守担当者向けメモ

  • 本関数はカスタム形式のプラグインエントリポイントである。
  • ctx->hasSeek の値は、後のデータ読み込み時のパフォーマンス（シークによる直接アクセスか、順次スキャンか）に直結する。
  • Windows環境におけるファイルハンドル操作については、ソース内のコメントを参照のこと。
