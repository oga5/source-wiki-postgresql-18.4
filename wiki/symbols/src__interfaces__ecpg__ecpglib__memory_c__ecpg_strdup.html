ecpg_strdup

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/memory.c:46

目的

文字列を複製し、メモリ割り当て失敗時にECPG固有のエラー処理を統合して通知する。

シグネチャまたは形状

char *ecpg_strdup(const char *string, int lineno)

入力と出力

  • **入力**:
  • string: 複製対象のヌル終端文字列（NULL許容）。
  • lineno: エラー発生時に報告されるソースコードの行番号。
  • **出力**:
  • 成功時: 複製された文字列へのポインタ（呼び出し元で解放が必要）。
  • 失敗時または入力がNULLの場合: NULL。

前提条件と不変条件

  • 入力が NULL の場合、関数は何もせず NULL を返す。
  • 複製に成功した場合、返されるメモリ領域は malloc 等で確保されたヒープ領域である。

アルゴリズムまたは振る舞い

1. 入力 string が NULL かどうかを確認し、NULL ならばそのまま NULL を返す。 2. 標準の strdup を呼び出し、メモリを確保して文字列をコピーする。 3. strdup が失敗（NULL を返却）した場合:

  • ecpg_raise を呼び出し、ECPG_OUT_OF_MEMORY エラーと ECPG_SQLSTATE_ECPG_OUT_OF_MEMORY SQLSTATE を記録する。
  • NULL を返す。

    • 4. 成功した場合は、確保したメモリ領域へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/connect.c (データベース名、環境変数、ホスト名、ポート等の文字列複製)
  • src/interfaces/ecpg/ecpglib/descriptor.c (ロケール情報の保持)
  • **呼び出し先**:
  • strdup (標準ライブラリ)
  • ecpg_raise (エラーハンドリング)

関連データ構造

  • なし。

失敗モードと境界条件

  • **メモリ不足**: strdup が NULL を返した場合、ecpg_raise を経由してエラーが通知される。
  • **NULL 入力**: string が NULL の場合、安全に NULL を返す。

保守担当者向けメモ

  • 本関数は malloc のラッパーとしての側面を持ち、メモリ確保失敗時のエラーハンドリングを標準化している。
  • 確保されたメモリは、呼び出し側で適切に解放される必要がある。
