SlabGetNextFreeChunk

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/slab.c:270

目的

SLAB アロケータの特定のブロック内から、割り当て可能な空きチャンクを一つ取得する。

シグネチャまたは形状

static inline MemoryChunk *SlabGetNextFreeChunk(SlabContext *slab, SlabBlock *block)

入力と出力

  • **入力**:
  • slab: 対象の SlabContext 構造体へのポインタ。
  • block: チャンクを取得する対象の SlabBlock 構造体へのポインタ。
  • **出力**:
  • 割り当てられた MemoryChunk へのポインタ。

前提条件と不変条件

  • block->nfree > 0 であること（空きチャンクが少なくとも一つ存在すること）。
  • freehead が存在する場合、そのチャンクはブロック内の有効な範囲内にあること。

アルゴリズムまたは振る舞い

1. block->freehead が NULL でない場合（解放済みチャンクのリストが存在する場合）:

  • リストの先頭からチャンクを取り出し、block->freehead を次のチャンクへ更新する。
  • Valgrind 用にメモリ定義状態を更新する。

    • 2. block->freehead が NULL の場合:

  • block->nunused > 0 であることを確認し、未使用領域 (block->unused) からチャンクを取得する。
  • block->unused ポインタを slab->fullChunkSize 分だけ進め、block->nunused をデクリメントする。

    • 3. 最後に block->nfree をデクリメントし、取得したチャンクを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/slab.c 内の SlabAlloc 関連処理（行 562, 675 付近）。
  • **呼び出し先**:
  • SlabChunkGetPointer
  • SlabBlockGetChunk
  • SlabChunkMod
  • VALGRIND_MAKE_MEM_DEFINED（マクロ）

関連データ構造

  • SlabContext: アロケータの全体設定を保持。
  • SlabBlock: ブロック単位の管理構造体。
  • MemoryChunk: メモリチャンクのヘッダー構造。

失敗モードと境界条件

  • block->nfree が 0 の状態で呼び出された場合、Assert により停止する。
  • freehead が NULL かつ block->nunused が 0 の場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁に呼び出されるためインライン展開を意図している。
  • freehead リストの操作において、チャンク内のポインタを直接書き換えるため、メモリレイアウトの整合性に注意すること。
