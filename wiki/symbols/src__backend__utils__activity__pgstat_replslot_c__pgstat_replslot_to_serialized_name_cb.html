pgstat_replslot_to_serialized_name_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_replslot.c:188

目的

統計情報の永続化（シリアライズ）処理において、レプリケーションスロットの内部インデックス（objid）を、再起動後も識別可能な「スロット名」に変換するために使用されます。

シグネチャまたは形状

void pgstat_replslot_to_serialized_name_cb(
    const PgStat_HashKey *key, 
    const PgStatShared_Common *header, 
    NameData *name
)

入力と出力

  • **入力**:
  • key: 統計エントリを特定するためのハッシュキー。objid フィールドにレプリケーションスロットのインデックスが格納されています。
  • header: 統計エントリの共通ヘッダー（本関数内では未使用）。
  • **出力**:
  • name: 変換されたスロット名が格納される NameData 構造体へのポインタ。

前提条件と不変条件

  • 本関数は、データベースシャットダウン時の統計情報シリアライズ処理の最終段階で呼び出されることを前提としています。
  • 呼び出し時点でレプリケーションスロットの状態は変更されないことが保証されており、指定されたインデックスに対応するスロットが存在することが期待されます。

アルゴリズムまたは振る舞い

1. key->objid（スロットインデックス）を引数として ReplicationSlotName を呼び出します。 2. スロット名が取得できた場合、それを name に書き込みます。 3. スロット名が見つからない場合、elog(ERROR) を発生させ、処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c において、統計情報のシリアライズ用コールバックテーブル（.to_serialized_name）に登録されています。
  • **呼び出し先**: ReplicationSlotName（レプリケーションスロットのインデックスから名前を検索する関数）。

関連データ構造

  • PgStat_HashKey: 統計エントリの識別子。
  • NameData: PostgreSQLの固定長文字列型。
  • ReplicationSlot: レプリケーションスロットの管理構造体（ReplicationSlotName 経由で間接的に参照）。

失敗モードと境界条件

  • 指定された objid に対応するレプリケーションスロットが存在しない場合、エラーが発生し処理が停止します。これは、シャットダウン時の整合性が保たれていない異常系として扱われます。

保守担当者向けメモ

  • 本関数は、レプリケーションスロットのインデックスが再起動時に変動する性質を補完するための重要なブリッジです。
  • 統計情報の永続化ロジックを変更する際は、pgstat_replslot_from_serialized_name_cb との整合性に注意してください。
