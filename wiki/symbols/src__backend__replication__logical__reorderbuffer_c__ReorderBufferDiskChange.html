ReorderBufferDiskChange

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/reorderbuffer.c:190

目的

メモリ制限を超えたトランザクション変更（ReorderBufferChange）をディスクへシリアライズ（スプール）する際に使用される、ディスク書き込み用のヘッダー構造体です。

シグネチャまたは形状

typedef struct ReorderBufferDiskChange
{
    Size        size;           /* 変更データのサイズ */
    ReorderBufferChange change; /* 変更内容のメタデータ */
    /* 実際のデータがこの後に続く */
} ReorderBufferDiskChange;

入力と出力

  • **入力**: ReorderBufferChange 構造体および関連する可変長データ。
  • **出力**: ディスク上のファイルへ書き込まれるバイト列。構造体の直後に実際のデータが配置されるレイアウトをとる。

前提条件と不変条件

  • ReorderBufferDiskChange の直後に、size で指定された長さのデータがメモリ上に連続して配置されている必要がある。
  • ディスクからの読み出し時、sizeof(ReorderBufferDiskChange) をオフセットとして加算することで、後続のデータ領域へアクセスする。

アルゴリズムまたは振る舞い

1. ReorderBufferSerializeTXN 等の処理において、メモリ上の ReorderBufferChange をディスクへ書き出す際、本構造体をヘッダーとして付与する。 2. 書き込み時は rb->outbuf を介して、ヘッダーとデータ本体をシリアライズする。 3. 読み出し時は、ファイルから読み込んだバッファの先頭を ReorderBufferDiskChange* にキャストし、size フィールドを参照して後続のデータ長を特定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/reorderbuffer.c 内のシリアライズ処理（ReorderBufferSerializeTXN 等）から参照される。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはない。

関連データ構造

  • ReorderBufferChange: 本構造体に埋め込まれ、変更の論理的な内容を保持する。
  • ReorderBuffer: デコーディングセッションの状態を管理する構造体。outbuf を通じて本構造体の読み書きを行う。

失敗モードと境界条件

  • size フィールドが実際のデータ長と一致しない場合、ディスクからの復元時にメモリ破壊や不正なデータ読み込みが発生する可能性がある。
  • ディスク書き込みエラーが発生した場合、トランザクションのシリアライズが失敗し、論理デコーディングが中断される。

保守担当者向けメモ

  • 本構造体はディスクフォーマットの一部であるため、フィールドの追加や変更を行う際は、既存の .spill ファイルとの互換性に注意すること。
  • sizeof(ReorderBufferDiskChange) を使用してデータ開始位置を計算するロジックが複数箇所（4103, 4135, 4165, 4187 行目付近）に存在するため、構造体定義を変更する場合はこれらの箇所をすべて確認すること。
