DoCopy

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copy.c:61

目的

SQL COPY コマンドの実行を統括するメインエントリーポイント。権限チェック、オプション解析、および対象テーブルの準備を行い、データ転送処理へ橋渡しを行う。

シグネチャまたは形状

void DoCopy(ParseState *pstate, const CopyStmt *stmt, int stmt_location, int stmt_len, uint64 *processed)

入力と出力

  • **入力**:
  • pstate: パーサーの状態。
  • stmt: 構文解析済みの COPY 文構造体。
  • stmt_location, stmt_len: SQL 文のソース内位置情報。
  • **出力**:
  • processed: 処理された行数を格納するポインタ。

前提条件と不変条件

  • 外部ファイルやプログラムへのアクセスには、対応する権限（pg_read_server_files, pg_write_server_files, pg_execute_server_program）が必要。
  • COPY 対象がテーブルの場合、適切なロック（RowExclusiveLock または AccessShareLock）が取得される。
  • WHERE 句が存在する場合、生成列（generated columns）はサポートされない。

アルゴリズムまたは振る舞い

1. **権限検証**: ファイルやプログラムへのアクセス権を has_privs_of_role で確認し、不足時はエラーを投げる。 2. **テーブル準備**: table_openrv で対象リレーションを開き、ロックを取得。 3. **式変換**: WHERE 句がある場合、transformExpr 等を用いて式を変換・検証し、定数畳み込みと正規化を行う。 4. **属性マッピング**: CopyGetAttnums を呼び出し、対象列のビットマップを構築。 5. **RLS チェック**: check_enable_rls を実行。RLS が有効な場合、COPY FROM はエラーとし、それ以外はクエリベースの転送へ切り替える準備を行う。 6. **実行委譲**: 最終的に CopyFrom または DoCopyTo（ソースコード上の呼び出し先）へ処理を委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c 等のユーティリティ層。
  • **呼び出し先**: table_openrv, addRangeTableEntryForRelation, transformExpr, CopyGetAttnums, ExecCheckPermissions, check_enable_rls, CopyFrom, DoCopyTo 等。

関連データ構造

  • CopyStmt: COPY コマンドの構文解析結果。
  • ParseState: パーサーの状態管理。
  • Relation: 対象テーブルのリレーション情報。
  • Bitmapset: 処理対象列や WHERE 句で使用される属性の管理。

失敗モードと境界条件

  • **権限不足**: 必要なロール権限がない場合に ERRCODE_INSUFFICIENT_PRIVILEGE でエラー。
  • **生成列の参照**: WHERE 句に生成列が含まれる場合に ERRCODE_INVALID_COLUMN_REFERENCE でエラー。
  • **RLS 制限**: COPY FROM で RLS が有効な場合に ERRCODE_FEATURE_NOT_SUPPORTED でエラー。

保守担当者向けメモ

  • 本関数はセットアップ層であり、実際のデータ転送ロジックは copyfrom.c や copyto.c に依存している。
  • WHERE 句の処理は COPY FROM のフィルタリングに直結するため、変更時は copyfrom.c 側の処理との整合性に注意すること。
