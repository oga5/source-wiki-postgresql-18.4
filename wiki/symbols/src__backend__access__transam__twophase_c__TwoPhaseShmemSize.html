TwoPhaseShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/twophase.c:236

目的

2フェーズコミット（2PC）機能で使用する共有メモリ領域の総サイズを計算します。

シグネチャまたは形状

Size TwoPhaseShmemSize(void)

入力と出力

  • **入力**: なし（GUC変数 max_prepared_xacts を内部的に参照）。
  • **出力**: 必要な共有メモリサイズ（バイト単位の Size 型）。

前提条件と不変条件

  • max_prepared_xacts が設定されている必要があります。
  • 計算結果は、TwoPhaseStateData 構造体、準備済みトランザクションのポインタ配列、および GlobalTransactionData 構造体の配列を格納するために必要なメモリ量を反映します。

アルゴリズムまたは振る舞い

1. TwoPhaseStateData 構造体の固定部分のサイズを算出します。 2. max_prepared_xacts に基づき、GlobalTransaction（ポインタ）配列のサイズを加算します。 3. アライメント調整（MAXALIGN）を行います。 4. max_prepared_xacts に基づき、GlobalTransactionData 構造体配列のサイズを加算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipci.c: CreateSharedMemoryAndSemaphores 内で共有メモリの総サイズ計算時に呼び出されます。
  • **呼び出し先**:
  • add_size (マクロ/関数)
  • mul_size (マクロ/関数)
  • MAXALIGN (マクロ)

関連データ構造

  • TwoPhaseStateData: 2PCの共有メモリ状態を保持するメイン構造体。
  • GlobalTransactionData: 個々の準備済みトランザクションの状態を保持する構造体。

失敗モードと境界条件

  • max_prepared_xacts が 0 の場合、ポインタ配列やデータ構造体配列のサイズは 0 となり、固定ヘッダー部分のみのサイズが返されます。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトと密接に関連しています。TwoPhaseStateData 構造体の定義や、準備済みトランザクションの管理方法に変更を加える場合は、必ず本関数の計算ロジックを更新してください。
