test_slru_delete_all

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_slru.c:184

目的

テスト用SLRU（Simple Least Recently Used）ディレクトリ内の全セグメントファイルを削除する。

シグネチャまたは形状

Datum test_slru_delete_all(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（SQLインターフェース経由の引数）
  • 出力: void（PG_RETURN_VOID() を介して戻り値なし）

前提条件と不変条件

  • TestSlruCtl が SimpleLruInit によって初期化済みであること。
  • 本関数はSQL関数として公開されており、PostgreSQLのバックエンドプロセスから呼び出されることを前提とする。

アルゴリズムまたは振る舞い

1. SlruScanDirectory 関数を呼び出し、TestSlruCtl が管理するディレクトリをスキャンする。 2. スキャン時にコールバック関数 test_slru_scan_cb を使用して、ディレクトリ内の全セグメントファイルを削除する。 3. 処理完了後、PG_RETURN_VOID() を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: SQLインターフェース（PG_FUNCTION_INFO_V1(test_slru_delete_all) により登録）。
  • 呼び出し先: SlruScanDirectory（PostgreSQLコアのSLRU API）。

関連データ構造

  • TestSlruCtl: テスト用SLRUを制御する SlruCtlData 構造体。

失敗モードと境界条件

  • ディレクトリが存在しない、または権限不足により削除が失敗した場合、SlruScanDirectory 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は SlruScanDirCbDeleteAll のロジックを内部的に利用して削除を実行する。
  • 削除対象は TestSlruCtl に関連付けられたディレクトリ内のファイル群である。
