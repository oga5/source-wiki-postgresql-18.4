ExecInitIndexOnlyScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:527

目的

インデックスのみスキャン（Index-Only Scan）を実行するための IndexOnlyScanState ノードを初期化し、スキャンに必要なリソース（インデックスリレーション、スキャンキー、式コンテキスト等）を準備する。

シグネチャまたは形状

IndexOnlyScanState *ExecInitIndexOnlyScan(IndexOnlyScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 実行対象の IndexOnlyScan プランノード。
  • estate: 実行状態を保持する EState。
  • eflags: 実行フラグ（EXEC_FLAG_EXPLAIN_ONLY 等）。
  • **出力**: 初期化済みの IndexOnlyScanState 構造体へのポインタ。

前提条件と不変条件

  • node に基づく IndexOnlyScanState が作成され、ExecProcNode フィールドに ExecIndexOnlyScan が設定される。
  • ss.ss_currentScanDesc はヒープスキャンを行わないため NULL に初期化される。
  • ioss_TableSlot は、可視性再チェック時にヒープからタプルを取得するために割り当てられる。

アルゴリズムまたは振る舞い

1. **状態構造体の作成**: IndexOnlyScanState を割り当て、プランと実行状態をリンクする。 2. **スロットと式コンテキストの準備**: ExecInitScanTupleSlot でタプルスロットを、ExecAssignExprContext で式コンテキストを初期化する。 3. **インデックスのオープン**: index_open を使用して対象インデックスを開き、ロックを取得する。 4. **スキャンキーの構築**: ExecIndexBuildScanKeys を呼び出し、indexqual および indexorderby からスキャンキーを生成する。 5. **ランタイムキーの処理**: ランタイムキーが存在する場合、専用の ioss_RuntimeContext を作成する。 6. **CSTRING変換の最適化**: インデックス上の CSTRINGOID 型を NAMEOID として扱う必要がある場合、該当する属性番号を ioss_NameCStringAttNums に記録する。 7. **EXPLAIN対応**: eflags に EXEC_FLAG_EXPLAIN_ONLY が含まれる場合、インデックスオープン以降の処理をスキップして早期リターンする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode（直接の呼び出し元として確認）。
  • **呼び出し先**: ExecAssignExprContext, ExecOpenScanRelation, ExecTypeFromTL, ExecInitScanTupleSlot, ExecAllocTableSlot, ExecInitResultTypeTL, ExecAssignScanProjectionInfoWithVarno, ExecInitQual, index_open, ExecIndexBuildScanKeys, palloc 等。

関連データ構造

  • IndexOnlyScanState: 本ノードの実行状態を保持する主要構造体。
  • IndexOnlyScan: プランノード。
  • EState: 実行状態。
  • TupleTableSlot: タプルデータを保持するスロット。

失敗モードと境界条件

  • eflags に EXEC_FLAG_EXPLAIN_ONLY が指定されている場合、インデックスのオープンやキー構築を行わずに終了する。
  • インデックスの属性型が CSTRINGOID かつ NAMEOID である場合、namecount を計算して ioss_NameCStringAttNums を動的に割り当てる。

保守担当者向けメモ

  • StoreIndexTuple は、ioss_NameCStringAttNums を参照して cstring から name 型への変換を行うため、本関数での属性番号の記録は必須である。
  • インデックスのみスキャンは EvalPlanQual (EPQ) をサポートしていないため、再チェックが必要な場合はエラーとなる設計である。
