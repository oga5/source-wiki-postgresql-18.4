pqTraceOutput_CommandComplete

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/libpq/fe-trace.c:277

目的

PostgreSQL プロトコルの CommandComplete メッセージを解析し、人間が読み取れる形式でトレース出力ストリームに書き出します。

シグネチャまたは形状

static void pqTraceOutput_CommandComplete(FILE *f, const char *message, int *cursor);

入力と出力

  • **入力**:
  • f: 出力先の FILE ポインタ。
  • message: プロトコルメッセージのペイロードを含むバッファ。
  • cursor: 現在の解析位置を示すオフセット（関数内で更新される）。
  • **出力**: なし（副作用として f への書き出しを行う）。

前提条件と不変条件

  • f は書き込み可能な有効なストリームであること。
  • cursor は message 内の有効な読み取り開始位置を指していること。

アルゴリズムまたは振る舞い

1. f に対して "CommandComplete\t" という文字列を出力します。 2. pqTraceOutputString を呼び出し、message 内の cursor 位置から文字列（コマンド完了タグ）を読み取り、フォーマットして出力します。この際、引用符の付与は行われません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-trace.c:688（pqTraceOutputMessage 内のディスパッチ処理から呼び出されます）。
  • **呼び出し先**: pqTraceOutputString（文字列の抽出と出力用）。

関連データ構造

  • FILE: 標準ライブラリの出力ストリーム。
  • PGconn.Pfdebug: 本関数が書き出し先として利用するトレース用ファイルポインタ。

失敗モードと境界条件

  • message や cursor が不正な場合、pqTraceOutputString の動作に依存して読み取りエラーや不正な出力が発生する可能性があります。

保守担当者向けメモ

  • 本関数は CommandComplete メッセージ専用です。新しいプロトコルメッセージの追加時には、pqTraceOutputMessage の switch 文で適切なケースを追加してください。
  • regress モードでの出力整合性を維持するため、出力形式を変更する際は回帰テストへの影響を確認してください。
