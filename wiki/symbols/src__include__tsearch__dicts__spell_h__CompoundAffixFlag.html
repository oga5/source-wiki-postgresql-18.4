CompoundAffixFlag

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/dicts/spell.h:168

目的

Hunspell形式の辞書における複合語（Compound）処理用のフラグ定義を保持する。フラグ名とその型、および対応する内部値をマッピングするために使用される。

シグネチャまたは形状

typedef struct CompoundAffixFlag
{
    union
    {
        const char *s;
        uint32      i;
    }           flag;
    FlagMode    flagMode;
    uint32      value;
} CompoundAffixFlag;

入力と出力

  • 入力: 辞書ファイルから読み込まれたフラグ名（文字列または数値）と、そのモード（FlagMode）。
  • 出力: 辞書構築時に IspellDict 内の配列として格納され、検索時の複合語判定に使用される。

前提条件と不変条件

  • flagMode フィールドは、flag 共用体の解釈方法（FM_CHAR, FM_LONG, FM_NUM）を決定する。
  • bsearch 等の検索関数で使用されるため、配列は flagMode および flag の値に基づいてソートされている必要がある。

アルゴリズムまたは振る舞い

  • 辞書構築時、setCompoundAffixFlagValue を通じてフラグ値が設定される。
  • 辞書内の CompoundAffixFlags 配列としてメモリ確保され、必要に応じて tmpalloc 等でリサイズされる。
  • 検索時には、特定のフラグに対応する value を取得するために、配列に対してバイナリサーチが実行される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/spell.c 内の辞書構築および検索ロジック（setCompoundAffixFlagValue や検索関数など）。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • IspellDict: 辞書全体の状態を保持し、CompoundAffixFlag の配列を管理する。
  • FlagMode: フラグの型定義を保持する列挙型。

失敗モードと境界条件

  • メモリ不足: 辞書構築時に CompoundAffixFlags 配列の拡張が必要な際、メモリ確保に失敗する可能性がある。
  • 検索失敗: 未定義のフラグが指定された場合、バイナリサーチで該当するエントリが見つからない。

保守担当者向けメモ

  • bsearch_arg が利用できない環境のため、比較関数には FlagMode のコピーが含まれている点に注意すること。
  • IspellDict 内の mCompoundAffixFlag（確保済みサイズ）と nCompoundAffixFlag（現在数）の整合性を維持すること。
