VacuumParams

定義

  • 種別: struct
  • 定義箇所: src/include/commands/vacuum.h:217

目的

VACUUM および ANALYZE コマンドの実行時における動作オプション、凍結（Freeze）閾値、並列処理設定、およびリソース制御パラメータを保持する設定構造体です。

シグネチャまたは形状

typedef struct VacuumParams {
    bits32      options;
    int         freeze_min_age;
    int         freeze_table_age;
    int         multixact_freeze_min_age;
    int         multixact_freeze_table_age;
    bool        is_wraparound;
    int         log_min_duration;
    VacOptValue index_cleanup;
    VacOptValue truncate;
    Oid         toast_parent;
    double      max_eager_freeze_failure_rate;
    int         nworkers;
} VacuumParams;

入力と出力

  • **入力**: ExecVacuum 等のコマンド解析層で初期化され、vacuum() 関数や do_analyze_rel() 等の実行層へ渡されます。
  • **出力**: 本構造体自体は設定値の保持に特化しており、直接的な戻り値はありません。各処理関数がこの構造体の値を参照して、テーブル走査や統計収集の挙動を決定します。

前提条件と不変条件

  • freeze_min_age や multixact_freeze_min_age に -1 を指定した場合、システムデフォルト値が使用されます。
  • nworkers が 0 の場合、インデックス数に基づいた自動決定が行われます。-1 の場合は並列処理が無効化されます。
  • max_eager_freeze_failure_rate は、可視性マップ（Visibility Map）を使用するテーブルAMでのみ適用されます。

アルゴリズムまたは振る舞い

  • options フィールドは VACOPT_* 定数のビットマスクとして機能し、FULL や FREEZE などの基本動作を制御します。
  • index_cleanup および truncate は VacOptValue 型（列挙型）により、明示的な有効/無効/デフォルトの制御を行います。
  • toast_parent は、再帰的な処理において権限チェックを行う際に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/vacuum.c（vacuum()）、src/backend/commands/analyze.c（do_analyze_rel()）、src/backend/access/heap/vacuumlazy.c（heap_vacuum_rel()）、src/backend/commands/cluster.c などから参照・初期化されます。
  • **呼び出し先**: 本構造体はデータ保持用であり、関数を直接呼び出すことはありません。各処理関数が本構造体の値を読み取り、適切なサブシステム（ストレージ管理や統計収集）へ値を伝播させます。

関連データ構造

  • VacOptValue: index_cleanup や truncate の設定値に使用される列挙型。
  • ParallelVacuumState: 並列処理実行時の状態管理に使用される構造体。

失敗モードと境界条件

  • nworkers に負の値を設定した場合の挙動は、-1（無効化）として定義されています。
  • freeze_table_age 等の年齢パラメータが不適切な場合、意図しない頻度でのテーブルフルスキャンが発生する可能性があります。

保守担当者向けメモ

  • 新しい VACUUM オプションを追加する際は、本構造体にフィールドを追加し、ExecVacuum での解析ロジックを更新してください。
  • max_eager_freeze_failure_rate はテーブルAMの挙動に直結するため、変更時には可視性マップの整合性に注意してください。
