ltree_execute

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_op.c:19

目的

ポーランド記法（後置記法）で格納された ltxtquery のツリー構造を再帰的に走査し、論理演算（AND, OR, NOT）を評価して ltree パスとの照合結果を判定する。

シグネチャまたは形状

bool ltree_execute(ITEM *curitem, void *checkval, bool calcnot, bool (*chkcond) (void *checkval, ITEM *val))

入力と出力

  • curitem: 評価対象の ITEM（クエリ要素）。
  • checkval: 照合に必要なコンテキストデータ（CHKVAL 構造体など）。
  • calcnot: NOT 演算を評価するかどうかを制御するフラグ。
  • chkcond: VAL 型のノードに到達した際に呼び出されるコールバック関数。
  • 戻り値: 評価結果が真であれば true、偽であれば false を返す。

前提条件と不変条件

  • check_stack_depth() により、再帰呼び出しによるスタックオーバーフローを防止している。
  • curitem はポーランド記法で構成された ITEM 配列の有効なポインタであること。
  • curitem->left は、演算子ノードから左オペランドへのオフセットとして正しく機能すること。

アルゴリズムまたは振る舞い

1. VAL 型の場合: chkcond を呼び出し、その結果を返す。 2. ! (NOT) 演算子の場合: calcnot が真であれば、右側のノードを再帰評価し、その結果を反転させる。偽であれば true を返す。 3. & (AND) 演算子の場合: 左側のノードを再帰評価し、真であれば右側のノードを再帰評価してその結果を返す。偽であれば false を返す。 4. | (OR) 演算子の場合: 左側のノードを再帰評価し、真であれば true を返す。偽であれば右側のノードを再帰評価した結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/ltree/_ltree_gist.c
  • contrib/ltree/ltree_gist.c
  • contrib/ltree/ltxtquery_op.c（自身の再帰呼び出し）
  • 呼び出し先:
  • check_stack_depth()
  • chkcond（関数ポインタ経由で checkcondition_str 等が実行される）

関連データ構造

  • ITEM: ltxtquery を構成する要素（値または演算子）。
  • CHKVAL: 照合時に使用されるコンテキスト構造体。

失敗モードと境界条件

  • 非常に深いクエリ構造に対しては、スタックオーバーフローが発生する可能性がある（check_stack_depth() で検知）。
  • curitem が不正なメモリ領域を指している場合、セグメンテーション違反を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は再帰的であるため、スタック消費量に注意すること。
  • curitem + curitem->left や curitem + 1 といったポインタ演算は、ltxtquery のメモリレイアウトに強く依存しているため、構造変更時には注意が必要である。
