interpt_pp

定義

  • 種別: function
  • 定義箇所: src/test/regress/regress.c:93

目的

2つの PATH 型データを受け取り、それらが交差する最初の点を計算して返す。PostgreSQL 回帰テスト用の補助関数。

シグネチャまたは形状

Datum interpt_pp(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの PATH 型ポインタ (p1, p2) を受け取る。
  • **出力**: 交点が存在する場合は POINT 型の Datum を返す。交差しない場合は NULL を返す。

前提条件と不変条件

  • 入力される PATH 型データは有効である必要がある。
  • regress_lseg_construct を用いて PATH 内の各線分 (LSEG) を構築する。

アルゴリズムまたは振る舞い

1. p1 と p2 の各線分ペアに対して二重ループを実行する。 2. regress_lseg_construct で線分を生成し、lseg_intersect を呼び出して交差判定を行う。 3. 交差が確認された時点でループを終了する。 4. 交差が見つかった場合、lseg_interpt を呼び出して交点を計算し、その結果を返す。 5. 交差が見つからない場合は PG_RETURN_NULL() を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(interpt_pp) により登録）。
  • **呼び出し先**:
  • regress_lseg_construct
  • lseg_intersect (DirectFunctionCall2 経由)
  • lseg_interpt (DirectFunctionCall2 経由)

関連データ構造

  • PATH: 複数の点からなるパス構造体。
  • LSEG: 線分構造体。
  • POINT: 交点座標を保持する構造体。

失敗モードと境界条件

  • 交差する線分が存在しない場合、NULL を返す。
  • lseg_interpt が内部でエラーを発生させる可能性があるが、本関数内では交差が確認された後に呼び出されるため、理論上は到達不能なエラーパスとなる。

保守担当者向けメモ

  • 本関数は回帰テスト専用であり、本番環境での利用は想定されていない。
  • lseg_intersect および lseg_interpt は PostgreSQL のコア機能に依存しているため、コア側の仕様変更の影響を受ける可能性がある。
