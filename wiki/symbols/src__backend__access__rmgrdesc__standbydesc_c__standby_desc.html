standby_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/standbydesc.c:46

目的

WAL（Write-Ahead Log）のスタンバイ関連レコードを解析し、人間が読める形式のテキストとして StringInfo バッファへ出力する。

シグネチャまたは形状

void standby_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 解析結果を格納する StringInfo バッファ。
  • record: 解析対象のWALレコードを保持する XLogReaderState 構造体。
  • **出力**: なし（buf を通じて文字列が追記される）。

前提条件と不変条件

  • record は有効なWALレコードを指していること。
  • XLogRecGetInfo(record) で取得される情報ビットが、スタンバイ関連のレコードタイプ（XLOG_STANDBY_LOCK, XLOG_RUNNING_XACTS, XLOG_INVALIDATIONS）のいずれかであること。

アルゴリズムまたは振る舞い

1. XLogRecGetData を用いてレコードのデータ部を取得する。 2. XLogRecGetInfo から XLR_INFO_MASK を除外した info を取得し、以下の通り分岐する。

  • XLOG_STANDBY_LOCK: レコード内のロック情報をループで展開し、トランザクションID、データベースOID、リレーションOIDを buf に追記する。
  • XLOG_RUNNING_XACTS: standby_desc_running_xacts を呼び出し、実行中のトランザクション状態を出力する。
  • XLOG_INVALIDATIONS: standby_desc_invalidations を呼び出し、共有キャッシュ無効化メッセージの詳細を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h において、RM_STANDBY_ID リソースマネージャの記述子関数として登録されている。
  • **呼び出し先**:
  • standby_desc_running_xacts
  • standby_desc_invalidations
  • XLogRecGetData / XLogRecGetInfo（マクロ）
  • appendStringInfo

関連データ構造

  • XLogReaderState: WALレコード読み取り状態。
  • StringInfo: 文字列バッファ。
  • xl_standby_locks, xl_running_xacts, xl_invalidations: スタンバイ関連のWALレコード構造体。

失敗モードと境界条件

  • info が上記3つのいずれにも該当しない場合、関数は何も出力せずに終了する。
  • xlrec へのキャストは、info が各タイプと一致している前提で行われる。

保守担当者向けメモ

  • 新しいスタンバイ関連のWALレコードタイプを追加した場合は、必ず standby_desc 内の if-else 分岐にケースを追加すること。
  • standby_desc_invalidations は汎用的なユーティリティとして設計されているため、他モジュールからの利用も考慮されている。
