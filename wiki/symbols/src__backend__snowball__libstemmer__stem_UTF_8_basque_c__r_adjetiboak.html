r_adjetiboak

目的

バスク語の形容詞（adjetiboak）に関する接尾辞を単語末尾から検索し、削除または置換を行うことで語幹抽出を行う。

シグネチャまたは形状

static int r_adjetiboak(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （Snowball ステミング環境構造体）
  • 出力: int （処理が成功した場合は 1、適用されなかった場合や失敗した場合は 0 または負の整数）

前提条件と不変条件

  • z->c が単語内の現在位置を指していること。
  • find_among_b を使用するため、単語の末尾から前方に向かって処理が行われる。
  • 処理対象の単語が SN_env 構造体に保持されていること。

アルゴリズムまたは振る舞い

1. z->ket を現在位置 z->c に設定する。 2. 単語末尾の文字が形容詞の接尾辞テーブル a_2 に適合するかを find_among_b で判定する。 3. 一致した場合、z->bra を現在位置に設定し、among_var の値に応じて以下の処理を行う。

  • case 1: r_RV を呼び出し、RV領域内であれば slice_del で接尾辞を削除する。
  • case 2: slice_from_s を使用して接尾辞を別の文字列に置換する。

    • 4. 処理結果（成功またはエラーコード）を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_basque.c および src/backend/snowball/libstemmer/stem_ISO_8859_1_basque.c 内のメインステミング処理。
  • 呼び出し先: find_among_b, r_RV, slice_del, slice_from_s。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_2: 形容詞の接尾辞を定義した among 構造体のテーブル。

失敗モードと境界条件

  • find_among_b が一致しない場合、0 を返して終了する。
  • r_RV や slice_del 等の内部関数が負の値を返した場合、そのエラーコードを伝播させる。
  • 単語の境界条件（z->lb）に達した場合、処理を中断する。
