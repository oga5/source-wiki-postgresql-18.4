RewriteQuery

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteHandler.c:3946

目的

パーサが生成したクエリツリーに対し、ルール適用やターゲットリストの正規化を行い、実行可能なクエリツリーのリストを生成します。特に、WITH句内のデータ変更クエリの書き換えと、INSERT/UPDATE/DELETE/MERGE文のターゲットリスト調整を担います。

シグネチャまたは形状

List *RewriteQuery(Query *parsetree, List *rewrite_events, int orig_rt_length, int num_ctes_processed)

入力と出力

  • **入力**:
  • parsetree: 書き換え対象のクエリツリー。
  • rewrite_events: 再帰呼び出し時のループ検知用リスト。
  • orig_rt_length: 元のレンジテーブル長。
  • num_ctes_processed: 処理済みCTE数。
  • **出力**: 書き換えられたクエリツリーの List。

前提条件と不変条件

  • parsetree 内の全リレーションは、呼び出し前に AcquireRewriteLocks 等でロック済みである必要があります。
  • WITH句内のクエリ書き換えにおいて、複数のクエリを生成するルールや、条件付き DO INSTEAD ルールはサポートされていません。

アルゴリズムまたは振る舞い

1. **CTEの再帰的処理**: parsetree->cteList を走査し、データ変更クエリ（INSERT/UPDATE/DELETE/MERGE）が含まれる場合は RewriteQuery を再帰的に呼び出します。 2. **ターゲットリストの調整**: event が CMD_SELECT や CMD_UTILITY でない場合、resultRelation を取得し、rewriteTargetListIU を用いてターゲットリストを正規化します。 3. **INSERTの特殊処理**: VALUES 句を含むINSERTの場合、rewriteTargetListIU および rewriteValuesRTE を呼び出して値を補完します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/rewrite/rewriteHandler.c 内の各箇所（再帰呼び出しを含む）。
  • QueryRewrite（推測される上位のインターフェース）。
  • **呼び出し先**:
  • rewriteTargetListIU
  • rewriteValuesRTE
  • RewriteQuery（自身を再帰呼び出し）
  • table_open

関連データ構造

  • Query: 書き換え対象のクエリツリー構造体。
  • CommonTableExpr: WITH句の定義。
  • RangeTblEntry: レンジテーブルエントリ。

失敗モードと境界条件

  • **エラー**:
  • WITH句内のデータ変更クエリに対して、サポート外のルール（条件付き DO INSTEAD、DO ALSO、複数ステートメントの DO INSTEAD）が適用された場合に ereport(ERROR) を発行します。
  • VALUES RTEが複数見つかった場合に elog(ERROR) を発行します。
  • **境界条件**:
  • num_ctes_processed を用いて、CTEの重複書き換えを防止します。

保守担当者向けメモ

  • 本関数は rewriteHandler.c におけるクエリ書き換えの主要な再帰ポイントです。
  • WITH 句内のクエリ書き換えは、生成されるクエリが単一であることに強く依存しています。将来的に複雑なルールをサポートする場合、この制約の緩和が必要です。
  • orig_rt_length は、クエリのレンジテーブルが変更された際に、元の構造を維持・識別するために使用されます。
