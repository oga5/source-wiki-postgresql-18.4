build_index_value_desc

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/conflict.c:460

目的

論理レプリケーションの競合発生時に、指定されたインデックスに基づいてタプルのキー値を人間が読める形式の文字列として生成します。

シグネチャまたは形状

static char *build_index_value_desc(EState *estate, Relation localrel, TupleTableSlot *slot, Oid indexoid)

入力と出力

  • **入力**:
  • estate: 実行状態（EState）。
  • localrel: 対象となるローカルリレーション。
  • slot: キー値を抽出する対象のタプルスロット。
  • indexoid: 記述に使用するインデックスのOID。
  • **出力**:
  • インデックスキー値を表す文字列（char *）。slot が NULL の場合は NULL を返します。

前提条件と不変条件

  • indexoid で指定されたインデックスに対して、呼び出し元が RowExclusiveLock を保持している必要があります（Assert による検証あり）。
  • FormIndexDatum を使用するため、インデックス式が存在する場合に備えて estate の式コンテキストが適切に設定されます。

アルゴリズムまたは振る舞い

1. slot が NULL の場合は直ちに NULL を返します。 2. index_open を使用してインデックスを開きます。 3. slot が仮想タプル（Virtual Slot）である場合、table_slot_create と ExecCopySlot を用いてヒープタプルスロットへ変換します。 4. GetPerTupleExprContext を介して、式コンテキストの ecxt_scantuple を現在のスロットに設定します。 5. FormIndexDatum を呼び出し、インデックスのAM（アクセスメソッド）に対する「生」の入力値（values および isnull 配列）を抽出します。 6. BuildIndexValueDescription を呼び出し、抽出された値から説明文字列を生成します。 7. index_close でインデックスを閉じ、生成した文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/conflict.c 内の競合処理ロジック（行番号344および424付近）。
  • **呼び出し先**:
  • index_open, index_close
  • table_slot_create, ExecCopySlot
  • GetPerTupleExprContext
  • FormIndexDatum
  • BuildIndexValueDescription

関連データ構造

  • EState: 実行状態を保持。
  • TupleTableSlot: タプルデータへのアクセスに使用。
  • Relation: リレーション記述子。
  • INDEX_MAX_KEYS: インデックスキーの最大数。

失敗モードと境界条件

  • slot が NULL の場合、安全に NULL を返します。
  • TTS_IS_VIRTUAL(slot) が真の場合、メモリ割り当てを伴うスロットのコピーが発生します。
  • index_open および index_close は NoLock モードで呼び出されます。

保守担当者向けメモ

  • 本関数は FormIndexDatum に依存しており、入力スロットがヒープタプル形式であることを要求するため、仮想スロットの変換処理が含まれています。
  • 文字列生成には BuildIndexValueDescription を利用しているため、メモリ管理は当該関数の仕様に従います。
