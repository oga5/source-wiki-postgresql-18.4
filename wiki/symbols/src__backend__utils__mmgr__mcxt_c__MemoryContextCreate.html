MemoryContextCreate

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/mcxt.c:1105

目的

新しいメモリコンテキスト構造体を初期化し、メモリコンテキストの階層ツリーに登録する。

シグネチャまたは形状

void MemoryContextCreate(MemoryContext node,
                         NodeTag tag,
                         MemoryContextMethodID method_id,
                         MemoryContext parent,
                         const char *name)

入力と出力

  • **入力**:
  • node: 初期化対象のメモリコンテキスト構造体へのポインタ。
  • tag: コンテキストのノード型。
  • method_id: コンテキストの動作を定義するメソッドテーブルの識別子。
  • parent: 親コンテキスト（NULL可）。
  • name: コンテキストを識別するための名称。
  • **出力**: なし（node が指すメモリ領域が初期化される）。

前提条件と不変条件

  • クリティカルセクション内（CritSectionCount > 0）では呼び出し不可。
  • parent が指定されている場合、MemoryContextIsValid(parent) が真であること。
  • node と parent は同一ではないこと。

アルゴリズムまたは振る舞い

1. **検証**: クリティカルセクションの状態および親コンテキストの妥当性をアサートする。 2. **初期化**: node の各フィールド（型、メソッドテーブル、親ポインタ、名前等）を引数に基づき設定する。isReset を true に、mem_allocated を 0 に初期化する。 3. **ツリー登録**:

  • parent が存在する場合、node を parent の子リストの先頭に挿入し、allowInCritSection フラグを親から継承する。
  • parent が存在しない場合、node をリストの先頭とし、allowInCritSection を false に設定する。

    • 4. **デバッグ**: VALGRIND_CREATE_MEMPOOL を呼び出し、Valgrind 用のメモリプールを生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: aset.c、bump.c、generation.c、slab.c などの各メモリコンテキスト実装ファイルにおけるコンテキスト作成ルーチンから呼び出される。
  • **呼び出し先**: 外部関数への直接的な呼び出しはソース内に見当たらないが、Valgrind マクロ (VALGRIND_CREATE_MEMPOOL) を実行する。

関連データ構造

  • MemoryContext: メモリコンテキストのヘッダー構造体。
  • MemoryContextMethods: コンテキストの動作を定義するメソッドテーブル。
  • mcxt_methods: method_id に対応するメソッドテーブルを保持するグローバル配列。

失敗モードと境界条件

  • 本関数自体はエラーを返さない（呼び出し側は失敗しない前提で設計されている）。
  • parent が無効なポインタである場合、アサーションにより異常終了する可能性がある。

保守担当者向けメモ

  • 本関数はメモリコンテキストの初期化のみを担当する。メモリ領域自体の割り当ては呼び出し元で行う必要がある。
  • allowInCritSection の継承ロジックは、メモリ管理の安全性に直結するため、変更時は注意が必要である。
