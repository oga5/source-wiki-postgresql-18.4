SSLerrmessage

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-openssl.c:1481

目的

OpenSSL のエラーコード（unsigned long）を、人間が判読可能なエラーメッセージ文字列に変換する。

シグネチャまたは形状

static const char *SSLerrmessage(unsigned long ecode)

入力と出力

  • **入力**: ecode (OpenSSL のエラーコード)
  • **出力**: エラー内容を示す const char * 型の文字列。

前提条件と不変条件

  • ecode が 0 の場合、"no SSL error reported" を返す。
  • 静的バッファ errbuf を使用するため、戻り値の文字列は呼び出し元で即座に使用する必要がある。

アルゴリズムまたは振る舞い

1. ecode が 0 ならば固定メッセージを返す。 2. ERR_reason_error_string(ecode) を呼び出し、既知のエラー文字列があればそれを返す。 3. OpenSSL 3.0.0 以降でシステムエラーコードが含まれる場合（ERR_SYSTEM_ERROR マクロが定義されている環境）、strerror を用いてシステムエラーメッセージを取得する。 4. 上記で解決できない場合、snprintf を使用して "SSL error code %lu" という形式の文字列を errbuf に書き込み、そのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure-openssl.c 内の各所（ERR_get_error() の戻り値を引数として多数の箇所から参照されている）。
  • **呼び出し先**: ERR_reason_error_string, ERR_SYSTEM_ERROR, ERR_GET_REASON, strerror, snprintf。

関連データ構造

  • errbuf: 36バイトの静的 char 配列。エラーコードを文字列化する際の作業領域として使用。

失敗モードと境界条件

  • ERR_reason_error_string が NULL を返す場合、システムエラーの判定または数値フォーマットへのフォールバックが行われるため、常に有効な文字列ポインタが返される。
  • 静的バッファを使用しているため、マルチスレッド環境での同時呼び出しには注意が必要だが、PostgreSQL バックエンドの現在の実装コンテキストにおいては単一スレッドでのエラー報告を想定している。

保守担当者向けメモ

  • OpenSSL 3.0.0 以降でのシステムエラー対応のため、ERR_SYSTEM_ERROR マクロの有無による条件分岐が含まれている。
  • エラーメッセージの国際化（_() マクロ）が適用されている。
