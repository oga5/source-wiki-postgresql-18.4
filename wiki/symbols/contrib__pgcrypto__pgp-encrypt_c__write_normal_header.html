write_normal_header

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-encrypt.c:75

目的

OpenPGP パケットの標準ヘッダ（タグおよび可変長データ長）を構築し、指定された出力フィルタへ書き込みます。

シグネチャまたは形状

static int write_normal_header(PushFilter *dst, int tag, int len);

入力と出力

  • **入力**:
  • dst: 書き込み先の PushFilter オブジェクト。
  • tag: PGP パケットタグ。
  • len: パケットのデータ長。
  • **出力**:
  • int: pushf_write の戻り値（成功時は 0、失敗時はエラーコード）。

前提条件と不変条件

  • dst は有効な PushFilter インスタンスである必要があります。
  • len は render_newlen が処理可能な形式である必要があります。

アルゴリズムまたは振る舞い

1. 8バイトのスタック領域 hdr を確保します。 2. 0xC0 と tag のビット論理和を計算し、パケットタグとしてヘッダの先頭に格納します。 3. render_newlen を呼び出し、len を PGP 仕様（RFC 4880）に基づく可変長形式にエンコードしてヘッダの続きに書き込みます。 4. 構築したヘッダ全体を pushf_write を介して dst に書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-encrypt.c:550（PGP_PKT_SYMENCRYPTED_SESSKEY パケットの書き込み時）
  • **呼び出し先**:
  • render_newlen
  • pushf_write

関連データ構造

  • PushFilter: 暗号化パイプラインの出力先を抽象化した構造体。

失敗モードと境界条件

  • pushf_write が失敗した場合、そのエラーコードがそのまま返されます。
  • len が render_newlen の許容範囲を超えている場合、エンコード処理で予期せぬ動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は RFC 4880 の「New Packet Format」に基づいています。パケット形式の変更や仕様更新時には render_newlen と併せて確認してください。
