xl_btree_mark_page_halfdead

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtxlog.h:283

目的

B-treeインデックスにおいて、リーフページを「半死（half-dead）」状態にマークする操作をWAL（Write-Ahead Log）に記録するための構造体です。

シグネチャまたは形状

typedef struct xl_btree_mark_page_halfdead
{
    OffsetNumber poffset;
    BlockNumber leafblk;
    BlockNumber leftblk;
    BlockNumber rightblk;
    BlockNumber topparent;
} xl_btree_mark_page_halfdead;

入力と出力

  • **入力**: 削除対象のリーフページおよびその周辺の兄弟ページ、親ページ内のタプルオフセット情報。
  • **出力**: WALレコードのデータ部として書き出されるバイナリデータ。

前提条件と不変条件

  • poffset は親ページ内の削除対象タプルを指す有効なオフセットであること。
  • leafblk は削除対象のリーフブロック番号であること。
  • leftblk および rightblk は、対象リーフページの左右の兄弟ブロック番号を保持すること。

アルゴリズムまたは振る舞い

本構造体は、B-treeのページ削除プロセスの一部として、対象ページが削除準備状態（half-dead）に入ったことを記録します。REDO処理時には、この構造体に含まれるブロック情報を用いて、インデックスの物理的な整合性を再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtpage.c において、ページ削除処理中に本構造体がインスタンス化されます。
  • **呼び出し先**: src/backend/access/nbtree/nbtxlog.c および src/backend/access/rmgrdesc/nbtdesc.c が、WALレコードから本構造体を読み取り、REDO処理やログのデコードを行います。

関連データ構造

  • SizeOfBtreeMarkPageHalfDead: 本構造体のサイズを計算するためのマクロ（src/include/access/nbtxlog.h:294）。

失敗モードと境界条件

  • 兄弟ページが存在しない場合、leftblk には無効なブロック番号が設定される可能性があります。
  • 構造体のサイズ計算には offsetof が使用されており、フィールドの追加や順序変更を行う場合は SizeOfBtreeMarkPageHalfDead マクロの更新が必須です。

保守担当者向けメモ

  • 本構造体はB-treeの削除ロジックと密接に関連しています。REDO処理の冪等性を維持するため、nbtxlog.c 内の対応するREDOルーチンがLSNチェックを適切に行っているか確認してください。
