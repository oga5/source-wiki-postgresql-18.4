EventTriggerCollectCreateOpClass

定義

  • 種別: function
  • 定義箇所: src/backend/commands/event_trigger.c:1954

目的

演算子クラス（Operator Class）の作成コマンド情報をイベントトリガーの収集リストに追加する。

シグネチャまたは形状

void EventTriggerCollectCreateOpClass(CreateOpClassStmt *stmt, Oid opcoid, List *operators, List *procedures)

入力と出力

  • **入力**:
  • stmt: CreateOpClassStmt 構造体（パースツリー）。
  • opcoid: 作成された演算子クラスの OID。
  • operators: 演算子クラスに関連付けられた演算子のリスト。
  • procedures: 演算子クラスに関連付けられたサポートプロシージャのリスト。
  • **出力**: なし（副作用として currentEventTriggerState を更新）。

前提条件と不変条件

  • currentEventTriggerState が初期化されており、かつコマンド収集が禁止されていないこと。
  • 実行は currentEventTriggerState->cxt メモリコンテキスト内で行われる。

アルゴリズムまたは振る舞い

1. イベントトリガーの状態が未設定、または収集が禁止されている場合は即座に終了する。 2. currentEventTriggerState->cxt にメモリコンテキストを切り替える。 3. CollectedCommand 構造体を割り当て、以下の情報を格納する：

  • type: SCT_CreateOpClass
  • in_extension: 現在の拡張作成状態 (creating_extension)
  • d.createopc.address: OperatorClassRelationId と opcoid を設定
  • d.createopc.operators / procedures: 引数のリストを保持
  • parsetree: stmt を copyObject で複製して保持

    • 4. 作成した command を currentEventTriggerState->commandList の末尾に追加する。 5. 元のメモリコンテキストに復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/opclasscmds.c:715
  • **呼び出し先**:
  • MemoryContextSwitchTo
  • palloc0
  • ObjectAddressSet
  • copyObject
  • lappend

関連データ構造

  • CollectedCommand: イベントトリガーが追跡するコマンド情報を保持する構造体。
  • CreateOpClassStmt: 演算子クラス作成のパースツリー。
  • currentEventTriggerState: 現在のクエリ実行におけるイベントトリガーの状態を保持するグローバルな状態管理構造体。

失敗モードと境界条件

  • currentEventTriggerState が NULL の場合、何も行わずに正常終了する。
  • commandCollectionInhibited が真の場合、何も行わずに正常終了する。

保守担当者向けメモ

  • 本関数は opclasscmds.c から呼び出されることを前提としている。
  • copyObject を使用してパースツリーを保持しているため、メモリ消費量に注意すること。
