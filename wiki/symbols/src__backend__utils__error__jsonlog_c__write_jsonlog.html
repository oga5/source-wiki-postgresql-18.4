write_jsonlog

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/jsonlog.c:108

目的

ErrorData 構造体に格納されたエラー情報を JSON 形式の文字列にシリアライズし、ログ出力パイプラインへ送信する。

シグネチャまたは形状

void write_jsonlog(ErrorData *edata)

入力と出力

  • **入力**: ErrorData *edata（エラーの詳細情報を含む構造体）
  • **出力**: なし（内部で構築した JSON 文字列をログ出力先へ書き出す）

前提条件と不変条件

  • log_line_number はプロセスごとに管理され、PID が変更された際にリセットされる。
  • StringInfo バッファを使用して動的にメモリを確保するため、終了時に適切に解放される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: StringInfoData バッファを初期化し、JSON の開始文字 { を挿入する。 2. **メタデータ収集**: タイムスタンプ、ユーザー名、DB名、PID、セッションID、行番号、プロセス状態（PS）、トランザクション情報などを順次バッファに追加する。 3. **エラー詳細の追加**: ErrorData からメッセージ、詳細、ヒント、SQL ステート、コンテキストなどを抽出して JSON フィールドとして追加する。 4. **クエリ情報**: check_log_of_query の判定に基づき、実行中のクエリ文字列やカーソル位置を追加する。 5. **終了処理**: JSON の閉じ括弧 } を付与し、ログ出力先へ転送する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/error/elog.c:3404
  • **呼び出し先**: appendJSONKeyValue, appendJSONKeyValueFmt, escape_json, get_formatted_log_time, get_ps_display, get_formatted_start_time, GetTopTransactionIdIfAny, error_severity, unpack_sql_state, check_log_of_query, get_backend_type_for_log 等（ソース内の関数呼び出しに基づく）

関連データ構造

  • ErrorData: エラー情報の保持に使用。
  • StringInfoData: JSON 文字列構築用の可変長バッファ。
  • PGPROC: プロセス情報およびトランザクション情報の参照に使用。

失敗モードと境界条件

  • MyProcPort が NULL の場合、ユーザー名やホスト情報などの関連フィールドはスキップされる。
  • Log_error_verbosity 設定が PGERROR_VERBOSE 未満の場合、ファイル名や関数名などの詳細情報は出力されない。
  • MyProc が NULL の場合、VXID やリーダー PID は出力されない。

保守担当者向けメモ

  • appendJSONKeyValue はキーを常にエスケープするが、値は escape_value フラグで制御される。数値やフォーマット済み文字列など、エスケープ不要な場合はフラグ設定に注意すること。
  • log_line_number は静的変数であり、プロセス間での一貫性よりもプロセス内での連続性が優先される設計である。
