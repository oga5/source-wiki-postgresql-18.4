pg_create_physical_replication_slot

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:64

目的

SQLインターフェースを通じて、新しい物理レプリケーションスロットを作成します。

シグネチャまたは形状

Datum pg_create_physical_replication_slot(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • name (Name): スロット名。
  • immediately_reserve (bool): true の場合、作成時に現在のWAL位置（LSN）を予約します。
  • temporary (bool): true の場合、セッション終了時に自動削除される一時スロットとして作成します。
  • **出力**:
  • Datum (複合型): スロット名と、immediately_reserve が有効な場合は予約された restart_lsn を含む行を返します。

前提条件と不変条件

  • 呼び出し元は適切な権限を持っている必要があります（CheckSlotPermissions による検証）。
  • スロット作成に必要なシステム要件を満たしている必要があります（CheckSlotRequirements による検証）。
  • 戻り値の型は複合型（row type）である必要があります。

アルゴリズムまたは振る舞い

1. 戻り値の型が複合型であることを確認します。 2. 権限と要件のチェックを実行します。 3. create_physical_replication_slot を呼び出し、内部的なスロット作成処理を実行します。 4. 作成されたスロットの情報を取得し、結果行を構築します。 5. immediately_reserve が指定されている場合、MyReplicationSlot から restart_lsn を抽出して結果に含めます。 6. ReplicationSlotRelease を呼び出し、スロットのロックを解放します。 7. 構築した行データを返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SQLコマンド経由）。
  • **呼び出し先**:
  • CheckSlotPermissions
  • CheckSlotRequirements
  • create_physical_replication_slot
  • ReplicationSlotRelease
  • heap_form_tuple

関連データ構造

  • MyReplicationSlot: 現在のバックエンドが操作中のレプリケーションスロット。
  • TupleDesc, HeapTuple: 結果行の構築に使用。

失敗モードと境界条件

  • 戻り値の型が複合型でない場合、elog(ERROR) により処理が中断されます。
  • 権限チェックまたは要件チェックで失敗した場合、例外が発生します。
  • スロット作成処理（create_physical_replication_slot）内でエラーが発生した場合、トランザクションのロールバックに伴いクリーンアップが行われます。

保守担当者向けメモ

  • 本関数はSQLレイヤーと内部の slot.c ロジックを繋ぐブリッジです。
  • スロット作成後の ReplicationSlotRelease 呼び出しは必須であり、これを忘れると共有メモリ上のロックが解放されずデッドロックの原因となります。
