get_object_type

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/filter.c:121

目的

フィルタ設定ファイル内の文字列キーワードを、プログラム内部で利用可能な FilterObjectType 列挙型へ変換する。

シグネチャまたは形状

static bool get_object_type(const char *keyword, int size, FilterObjectType *objtype);

入力と出力

  • **入力**:
  • keyword: 比較対象の文字列。
  • size: keyword の長さ。
  • objtype: 変換結果を格納する FilterObjectType へのポインタ。
  • **出力**:
  • 成功時は true を返し、objtype に対応する列挙値を設定する。
  • 一致するキーワードがない場合は false を返す。

前提条件と不変条件

  • keyword は有効な文字列であること。
  • objtype は書き込み可能なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. is_keyword_str を用いて、入力された keyword を定義済みのオブジェクト型キーワード（table, schema, function 等）と順次比較する。 2. 一致するものが見つかった場合、対応する FilterObjectType を objtype に代入し、true を返す。 3. どのキーワードにも一致しない場合、何も変更せず false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/filter.c 内の filter_read_item から呼び出されていることが確認できる。
  • **呼び出し先**: is_keyword_str を使用している。
  • **注意**: ツールが提示したバックエンド側の get_object_type（src/include/catalog/objectaddress.h 等）は、シグネチャが異なる別関数であり、本シンボルとは直接の関係はない。

関連データ構造

  • FilterObjectType: フィルタ対象となるオブジェクトの種類を定義する列挙型。

失敗モードと境界条件

  • 未知のキーワードが渡された場合、false を返し、objtype は更新されない。

保守担当者向けメモ

  • 本関数は pg_dump のフィルタ処理専用である。バックエンド側の同名関数と混同しないよう注意すること。
  • 新しいオブジェクト型をフィルタに追加する場合は、本関数の if-else チェーンに新しいキーワードと列挙値のペアを追加する必要がある。
