find_compatible_trans

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepagg.c:456

目的

既存の集約遷移状態（AggTransInfo）を再利用可能か判定し、共有可能な既存の transno を検索します。

シグネチャまたは形状

int find_compatible_trans(PlannerInfo *root, Aggref *newagg, bool shareable,
                          Oid aggtransfn, Oid aggtranstype,
                          int transtypeLen, bool transtypeByVal,
                          Oid aggcombinefn,
                          Oid aggserialfn, Oid aggdeserialfn,
                          Datum initValue, bool initValueIsNull,
                          List *transnos)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • newagg: 検索対象の Aggref ノード。
  • shareable: 状態共有が許可されているかを示すフラグ。
  • aggtransfn, aggtranstype 等: 遷移関数、型、シリアライズ/デシリアライズ関数、初期値などの集約属性。
  • transnos: 検索対象となる既存の transno のリスト。
  • **出力**: 互換性のある既存の transno（整数）。見つからない場合は -1 を返します。

前提条件と不変条件

  • shareable が false の場合、検索を行わず即座に -1 を返します。
  • 遷移状態を共有するには、遷移関数、遷移型、シリアライズ/デシリアライズ関数、結合関数、および初期値がすべて一致する必要があります。

アルゴリズムまたは振る舞い

1. shareable が偽であれば -1 を返す。 2. transnos リスト内の各 transno について、root->aggtransinfos から対応する AggTransInfo を取得する。 3. 以下の属性を比較し、すべて一致するか確認する:

  • 遷移関数 (transfn_oid) と遷移型 (aggtranstype)。
  • シリアライズ/デシリアライズ関数 (serialfn_oid, deserialfn_oid)。
  • 結合関数 (combinefn_oid)。
  • 初期値 (initValue): initValueIsNull の状態および datumIsEqual による値の比較。

    • 4. すべての条件を満たす transno が見つかった時点でその値を返す。ループ終了まで見つからなければ -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepagg.c 内の集約前処理ロジック（preprocess_aggref 等）から呼び出されます。
  • **呼び出し先**: list_nth_node（マクロ）、datumIsEqual。

関連データ構造

  • AggTransInfo: 遷移状態の計算情報を保持する構造体。
  • PlannerInfo: プランニング中のグローバル情報を保持する構造体。
  • Aggref: クエリ内の集約関数呼び出しを表すノード。

失敗モードと境界条件

  • 互換性のある状態が存在しない場合、-1 を返します。
  • initValue が NULL かどうかで比較ロジックが分岐します。
  • 結合関数（combinefn）は、部分集約の有無が不明な段階であっても保守的に一致を要求します。

保守担当者向けメモ

  • 本関数はプランニングの初期段階で呼び出されます。将来的に実行時のオーバーヘッドを削減するため、ここで収集された情報は Agg ノードに保持される設計となっています。
  • 比較対象の関数 OID が InvalidOid である場合も、それらが一致している必要があります。
