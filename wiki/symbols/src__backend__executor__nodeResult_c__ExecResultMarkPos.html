ExecResultMarkPos

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeResult.c:145

目的

Result ノードにおいて、現在位置をマーク（保存）する操作を子ノードへ転送する。

シグネチャまたは形状

void ExecResultMarkPos(ResultState *node)

入力と出力

  • **入力**: ResultState *node（Result ノードの実行時状態構造体）
  • **出力**: なし（副作用として子ノードのマーク操作を実行、またはデバッグログを出力）

前提条件と不変条件

  • node は有効な ResultState 構造体を指している必要がある。
  • Result ノード自体は Mark/Restore 操作を保持・管理しない。

アルゴリズムまたは振る舞い

1. outerPlanState(node) を介して、Result ノードの子プラン状態を取得する。 2. 子プランが存在する場合、ExecMarkPos(outerPlan) を呼び出し、マーク操作を子ノードへ委譲する。 3. 子プランが存在しない場合、DEBUG2 レベルで「Result nodes do not support mark/restore」というメッセージをログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:352
  • **呼び出し先**: ExecMarkPos（子プランに対して）

関連データ構造

  • ResultState: Result ノードの実行時状態を保持する構造体。
  • PlanState: プランツリーの実行時状態を保持する基底構造体。

失敗モードと境界条件

  • 子ノードが存在しない場合、エラーではなくデバッグログの出力のみが行われる。

保守担当者向けメモ

  • Result ノードはデータソースを持たない場合があるため、マーク操作の可否は子ノードの存在に依存する。本関数は単なる転送役である。
