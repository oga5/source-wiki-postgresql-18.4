AlignedAllocRealloc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/alignedalloc.c:60

目的

アライメント制約を維持したまま、既存のアライメント調整済みメモリブロックを新しいサイズで再確保し、データを移行する。

シグネチャまたは形状

void *AlignedAllocRealloc(void *pointer, Size size, int flags)

入力と出力

  • **入力**:
  • pointer: 再確保対象となる、アライメント調整済みのメモリポインタ。
  • size: 要求する新しいメモリサイズ。
  • flags: メモリ確保時の動作を指定するフラグ。
  • **出力**: 再確保された新しいメモリブロックへのポインタ。失敗時は NULL またはメモリ確保失敗時の標準的なエラー処理へ遷移する。

前提条件と不変条件

  • pointer は palloc_aligned() によって確保された有効なメモリ領域を指していること。
  • アライメント値（alignto）は 2 のべき乗であること（Assert で検証）。
  • MEMORY_CONTEXT_CHECKING が有効な場合、再計算された old_size が元のチャンクの要求サイズ以上であることを検証する。

アルゴリズムまたは振る舞い

1. PointerGetMemoryChunk() を用いて、ポインタからメタデータ（MemoryChunk）を取得する。 2. MemoryChunkGetValue() でアライメント値を取得し、MemoryChunkGetBlock() で元の非アライメントメモリブロックを特定する。 3. GetMemoryChunkSpace() を用いて元のメモリブロックのサイズを算出し、アライメント調整用の余分なバイト数とヘッダーサイズを差し引いて old_size を決定する。 4. GetMemoryChunkContext() で元のメモリコンテキストを取得し、MemoryContextAllocAligned() を呼び出して新しい領域を確保する。 5. 確保に失敗した場合は MemoryContextAllocationFailure() を呼び出す。 6. VALGRIND_MAKE_MEM_DEFINED を使用して旧領域のアクセス権限を一時的に有効化し、memcpy() でデータを新しい領域へコピーする。 7. pfree() を呼び出し、旧メモリブロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MCTX_ALIGNED_REDIRECT_ID 構造体を通じて参照されている。
  • **呼び出し先**:
  • PointerGetMemoryChunk, MemoryChunkGetValue, MemoryChunkGetBlock, GetMemoryChunkSpace, GetMemoryChunkContext, MemoryContextAllocAligned, MemoryContextAllocationFailure, pfree, memcpy。

関連データ構造

  • MemoryChunk: アライメント調整されたポインタの直前に配置されるメタデータ構造体。

失敗モードと境界条件

  • **OOM (Out of Memory)**: MemoryContextAllocAligned が NULL を返した場合、MemoryContextAllocationFailure を経由してエラー処理を行う。
  • **サイズ不正確性**: GetMemoryChunkSpace が返す値にはアライメントのためのパディングが含まれるため、memcpy 時には元の要求サイズよりも多くのデータをコピーする可能性がある。

保守担当者向けメモ

  • 本関数は palloc_aligned で確保されたメモリに対するデコレータとして機能する。
  • old_size の算出において、コンテキストの実装が MemoryChunk をヘッダーとして使用していることを前提としている。
  • VALGRIND マクロによるメモリ状態の管理が必須であり、変更時にはアクセス権限の整合性に注意すること。
