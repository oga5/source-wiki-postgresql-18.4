PendingRelSync

定義

  • **種別**: struct
  • **定義箇所**: src/backend/catalog/storage.c:71

目的

トランザクション終了時に物理ストレージの同期（fsync）や WAL への書き出しが必要なリレーションを追跡するための管理構造体です。

シグネチャまたは形状

typedef struct PendingRelSync
{
    RelFileLocator rlocator; /* 同期対象のリレーション識別子 */
    bool           is_truncated; /* 切り詰め操作の有無 */
} PendingRelSync;

入力と出力

本構造体はデータ保持用であり、関数としての入力・出力はありません。ハッシュテーブル（pendingSyncHash）の要素として使用されます。

前提条件と不変条件

  • rlocator は対象リレーションを一意に特定する必要があります。
  • is_truncated は、当該トランザクション内でリレーションが切り詰められたか否かを示し、同期処理の挙動を制御します。

アルゴリズムまたは振る舞い

  • トランザクション内でリレーションに対して同期が必要な操作が行われた際、本構造体が生成されハッシュテーブルに登録されます。
  • トランザクション終了時に smgrDoPendingSyncs がハッシュテーブルを走査し、各 PendingRelSync エントリに基づいて物理的な同期処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/storage.c 内の AddPendingSync 等、同期予約を行う関数群から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • RelFileLocator: リレーションの物理的な場所を特定する識別子。
  • pendingSyncHash: 本構造体を要素として保持するハッシュテーブル。

失敗モードと境界条件

  • メモリ不足: ハッシュテーブルへの登録時にメモリ確保が失敗する可能性があります。
  • 整合性: トランザクションのアボート時に、予約された同期処理が適切に破棄または無視される必要があります。

保守担当者向けメモ

  • is_truncated フラグは、チェックポイントとの整合性や WAL 記録の最適化に直結するため、変更時には smgrDoPendingSyncs のロジックを再確認してください。
  • 本構造体は TopMemoryContext に関連付けられたハッシュテーブルで管理されることが想定されています。
