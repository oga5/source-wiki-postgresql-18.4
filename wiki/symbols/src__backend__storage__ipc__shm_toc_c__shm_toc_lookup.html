shm_toc_lookup

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_toc.c:231

目的

共有メモリセグメント内の目次（TOC）から、指定されたキーに対応するデータ構造の絶対アドレスを解決する。

シグネチャまたは形状

void *shm_toc_lookup(shm_toc *toc, uint64 key, bool noError);

入力と出力

  • **入力**:
  • toc: 検索対象の目次構造体へのポインタ。
  • key: 検索する64ビットの識別キー。
  • noError: 見つからなかった場合にエラーを発生させるか否か（true なら NULL を返し、false なら elog(ERROR) を発行する）。
  • **出力**:
  • キーに対応するデータ構造の絶対アドレス（void *）。見つからない場合は NULL。

前提条件と不変条件

  • toc は有効な shm_toc 構造体を指していること。
  • 共有メモリ内のデータ構造は、セグメント先頭からの相対オフセットとして管理されていること。
  • toc->toc_nentry の読み込みはアトミックであると仮定される。

アルゴリズムまたは振る舞い

1. toc->toc_nentry を読み込み、現在のエントリ数を取得する。 2. pg_read_barrier() を実行し、エントリ内容の読み込み前にメモリの整合性を確保する。 3. 登録されている全エントリを順次走査し、指定された key と一致するものを探す。 4. 一致した場合、toc のベースアドレスに当該エントリの相対オフセットを加算して返却する。 5. 一致しない場合、noError が false であればエラーを発生させ、true であれば NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/common/session.c
  • src/backend/access/gin/gininsert.c
  • **呼び出し先**:
  • elog (エラー発生時)

関連データ構造

  • shm_toc: 目次を保持する構造体。
  • shm_toc_entry: 目次内の各エントリ（キーとオフセットを保持）。

失敗モードと境界条件

  • **キー未検出**: noError が false の場合、elog(ERROR) によりプロセスが中断される。
  • **ロックフリー**: 本関数はロックを取得しないため、他のプロセスが shm_toc_insert を実行中の場合でも読み取りを行う。

保守担当者向けメモ

  • 本関数はロックフリーで動作するため、パフォーマンスに優れるが、エントリの追加と並行して実行される可能性がある。
  • 多数のキーを登録すると検索コストが増大するため、多数のポインタが必要な場合は、TOC には単一の管理構造体へのポインタのみを登録する設計を推奨する。
