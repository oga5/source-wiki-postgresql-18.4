test_multi_pipelines

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:467

目的

複数のパイプラインを連続してキューイングし、コマンドモードへ戻ることなく各パイプラインの結果を順次処理する挙動を検証する。

シグネチャまたは形状

static void test_multi_pipelines(PGconn *conn)

入力と出力

  • 入力: PGconn *conn （確立済みのデータベース接続）
  • 出力: なし（成功時は標準エラー出力に "ok" を出力し、失敗時は pg_fatal を通じてプロセスを終了する）

前提条件と不変条件

  • conn は有効な接続状態であること。
  • パイプラインモードの開始・終了および同期処理が正しく行われること。
  • 各パイプラインの処理において、PGRES_TUPLES_OK および PGRES_PIPELINE_SYNC が期待される順序で返されること。

アルゴリズムまたは振る舞い

1. PQenterPipelineMode でパイプラインモードを開始する。 2. 3つのパイプラインを順次送信する（PQsendQueryParams と PQpipelineSync / PQsendPipelineSync を使用）。 3. 各パイプラインの結果を PQgetResult で取得し、以下の検証を行う：

  • 各クエリの結果が PGRES_TUPLES_OK であること。
  • 各同期ポイントで PGRES_PIPELINE_SYNC が返されること。

    • 4. 最終的に PQexitPipelineMode を呼び出し、パイプラインモードを正常に終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/libpq_pipeline/libpq_pipeline.c:2362 （main 関数内と推測される）
  • 呼び出し先: libpq API（PQenterPipelineMode, PQsendQueryParams, PQpipelineSync, PQsendPipelineSync, PQgetResult, PQresultStatus, PQclear, PQexitPipelineMode, PQpipelineStatus, PQerrorMessage）、pg_fatal

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: クエリ実行結果を保持する構造体。

失敗モードと境界条件

  • PQenterPipelineMode が失敗した場合、致命的エラーとして終了する。
  • 各パイプライン送信や結果取得において、期待値と異なる結果（NULL や予期せぬステータス）が返された場合、エラーメッセージと共に終了する。
  • 同期前に PQexitPipelineMode を呼び出した際に成功してしまうなど、期待される制約が守られない場合にエラーとなる。

保守担当者向けメモ

  • 本関数は libpq のパイプラインモードにおける「入れ子」や「連続実行」のテストケースとして機能する。
  • PQsendPipelineSync を使用してフラッシュをスキップするケースが含まれており、パイプラインのバッファリング挙動の検証に利用されている。
