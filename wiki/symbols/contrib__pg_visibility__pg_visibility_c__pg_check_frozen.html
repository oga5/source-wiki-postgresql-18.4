pg_check_frozen

定義

  • 種別: function
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:323

目的

指定されたリレーションにおいて、可視性マップ（VM）の「all-frozen」ビットが立っているにもかかわらず、実際にはフリーズが必要なタプルが存在するページ（破損箇所）を特定し、そのTID（Tuple Identifier）を返す。

シグネチャまたは形状

Datum pg_check_frozen(PG_FUNCTION_ARGS)

  • PostgreSQLのSRF（Set-Returning Function）として実装されている。

入力と出力

  • **入力**: PG_GETARG_OID(0) により、対象リレーションのOIDを受け取る。
  • **出力**: 破損が検出されたタプルのTID（ItemPointer）を順次返す。

前提条件と不変条件

  • 呼び出し元は AccessShareLock を保持している必要がある（モジュール概要より）。
  • collect_corrupt_items を介してリレーションの relkind が検証される。
  • SRFのコンテキスト管理のため、初回呼び出し時に FuncCallContext が初期化される。

アルゴリズムまたは振る舞い

1. **初回呼び出し時**:

  • SRF_FIRSTCALL_INIT() でコンテキストを初期化。
  • collect_corrupt_items(relid, false, true) を呼び出し、リレーション全体を走査して破損タプルを収集する。第3引数 true はフリーズ状態のチェックを有効にするフラグである。
  • 収集結果を funcctx->user_fctx に格納する。

    • 2. **各呼び出し時**:

  • funcctx->user_fctx から収集済みデータ corrupt_items を取得する。
  • items->next が items->count に達するまで、保持しているTIDを順次 SRF_RETURN_NEXT で返す。

    • 3. **終了時**:

  • 全てのTIDを返した後、SRF_RETURN_DONE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（PG_FUNCTION_INFO_V1(pg_check_frozen) により登録）。
  • **呼び出し先**:
  • collect_corrupt_items（破損タプルの収集）
  • SRF_FIRSTCALL_INIT, SRF_PERCALL_SETUP, SRF_RETURN_NEXT, SRF_RETURN_DONE（SRF制御マクロ）
  • MemoryContextSwitchTo（メモリコンテキスト管理）

関連データ構造

  • corrupt_items: 収集された破損タプルのTIDリストと件数を保持する構造体。
  • FuncCallContext: SRFの呼び出し間で状態を保持するための構造体。

失敗モードと境界条件

  • 対象リレーションがサポート外の relkind である場合、collect_corrupt_items 内でエラーが発生する可能性がある。
  • 大規模なテーブルに対して実行した場合、全ページ走査によるI/O負荷が発生する。

保守担当者向けメモ

  • 本関数は collect_corrupt_items に処理を委譲しているため、ロジックの変更は主に collect_corrupt_items 側で行うこと。
  • 偽陽性を避けるため、内部では厳格なXID horizon計算が使用されている点に留意すること。
