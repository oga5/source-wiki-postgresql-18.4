isort_cmp

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:190

目的

lib/sort_template.h を利用したソート処理において、整数配列の要素を比較するための比較関数を提供します。

シグネチャまたは形状

static inline int isort_cmp(const void *a, const void *b, void *arg)

入力と出力

  • **入力**:
  • a, b: 比較対象となる int32 型の値へのポインタ。
  • arg: 昇順ソートを行うか否かを示す bool 型の値へのポインタ。
  • **出力**:
  • a < b の場合: 昇順なら -1、降順なら 1。
  • a > b の場合: 昇順なら 1、降順なら -1。
  • a == b の場合: 0。

前提条件と不変条件

  • arg が指す値が true の場合は昇順、false の場合は降順として比較を行います。
  • 入力ポインタ a および b は有効な int32 型のメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. a と b が指す値を int32 として取得します。 2. arg（bool値）を参照し、昇順または降順のロジックを選択します。 3. 選択された順序に基づき、aval と bval を比較して整数値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_tool.c 内のソート処理（ST_COMPARE マクロ経由）。
  • qunique_arg 関数（contrib/intarray/_int_tool.c:318）。
  • **呼び出し先**:
  • 特になし（純粋な比較ロジック）。

関連データ構造

  • int32: 比較対象のデータ型。

失敗モードと境界条件

  • arg が指すポインタが NULL の場合、未定義の動作を引き起こす可能性があります。
  • 比較対象のポインタが NULL の場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は lib/sort_template.h の要件に従ったシグネチャを持っています。ソート順序のロジックを変更する場合は、arg の解釈と戻り値の整合性に注意してください。
