cipher_info

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/pgp.c:59

目的

OpenPGP で使用される暗号アルゴリズムのメタデータ（名称、内部コード、鍵長、ブロック長）を保持し、アルゴリズムの識別と管理を行うための構造体です。

シグネチャまたは形状

struct cipher_info
{
	const char *name;      /* アルゴリズムの公開名称 */
	int			code;      /* 内部識別コード */
	const char *int_name;  /* 内部実装名 */
	int			key_len;   /* 鍵長（バイト） */
	int			block_len; /* ブロック長（バイト） */
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。cipher_list 配列の要素として定義され、get_cipher_info 等の関数を通じて参照されます。

前提条件と不変条件

  • name および int_name は、有効な文字列リテラルを指す必要があります。
  • code は、PGP プロトコルで定義された暗号アルゴリズム識別子と一致している必要があります。

アルゴリズムまたは振る舞い

本構造体自体はアルゴリズムを実装しません。pgcrypto モジュール内において、ユーザーが指定したアルゴリズム名から内部コードへの変換や、暗号化処理に必要な鍵長・ブロック長の照会を行うための静的なメタデータ定義として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp.c 内の cipher_list 配列定義、および get_cipher_info を呼び出す関数群（pgp_get_cipher_code、pgp_get_cipher_key_size、pgp_get_cipher_block_size 等）。
  • **呼び出し先**: 構造体であるため、直接の呼び出し先はありません。

関連データ構造

  • cipher_list: cipher_info 型の静的配列。サポートされる全暗号アルゴリズムの定義を保持します。

失敗モードと境界条件

  • get_cipher_info 等で存在しない code が指定された場合、検索結果として NULL が返される可能性があります。

保守担当者向けメモ

  • 新しい暗号アルゴリズムを追加する場合は、cipher_list 配列に本構造体のインスタンスを追加してください。
  • key_len や block_len は、px レイヤー（低レベル暗号ライブラリ）の実装と整合している必要があります。
