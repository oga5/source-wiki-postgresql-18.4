PartitionHashBound

定義

  • **種別**: struct
  • **定義箇所**: src/backend/partitioning/partbounds.c:49

目的

ハッシュパーティションにおける各パーティションの境界条件（モジュラスと剰余）および対応するインデックスを保持するために使用されます。

シグネチャまたは形状

typedef struct PartitionHashBound
{
	int			modulus;
	int			remainder;
	int			index;
} PartitionHashBound;

入力と出力

  • **入力**: ハッシュパーティションの定義情報から生成される modulus（除数）と remainder（剰余）。
  • **出力**: パーティションの識別子である index。

前提条件と不変条件

  • modulus は正の整数である必要があります。
  • remainder は 0 以上 modulus 未満である必要があります。
  • index はパーティション配列内の位置を指します。

アルゴリズムまたは振る舞い

  • ハッシュパーティションの作成時、create_hash_bounds 内でこの構造体の配列が生成されます。
  • qsort を用いて、modulus および remainder を基準にソートされ、パーティションの順序が決定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partbounds.c 内の create_hash_bounds 関数において、メモリ確保およびソート処理の対象として参照されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • PartitionBoundInfoData: ハッシュパーティションを含む境界情報の全体を管理する構造体。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • 不正な modulus や remainder の組み合わせが指定された場合、パーティションの検索やマッピングが正しく行われない可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/partitioning/partbounds.c 内のソート処理（qsort）において、PartitionHashBound 型のポインタとしてキャストされて比較されます。変更を加える際は、比較ロジックとの整合性に注意してください。
