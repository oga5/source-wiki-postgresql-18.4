nonword

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regcomp.c:1458

目的

正規表現のコンパイル過程において、非単語文字（non-word character）が先行（ahead）または後続（behind）する場合の NFA 遷移アークを生成する。

シグネチャまたは形状

static void nonword(struct vars *v, int dir, struct state *lp, struct state *rp)

入力と出力

  • v: コンパイル状態を保持する vars 構造体へのポインタ。
  • dir: 判定方向。AHEAD（前方）または BEHIND（後方）を指定。
  • lp: 遷移元の NFA 状態。
  • rp: 遷移先の NFA 状態。
  • 出力: なし（副作用として v->nfa にアークを追加）。

前提条件と不変条件

  • dir は AHEAD または BEHIND のいずれかであること（assert により検証）。
  • v->nfa および v->cm が初期化済みであること。

アルゴリズムまたは振る舞い

1. dir に基づき、アンカー文字（AHEAD なら $、BEHIND なら ^）を決定する。 2. newarc を呼び出し、決定したアンカーを用いて lp から rp への遷移アークを2つ追加する（カラー値 1 および 0）。 3. colorcomplement を呼び出し、v->wordchrs に含まれない文字に対する遷移を lp から rp 間に追加する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/regex/regcomp.c 内の複数箇所（行 903, 913, 921, 926, 938, 939）。
  • 呼び出し先: newarc（NFA アーク追加）、colorcomplement（補集合のカラー遷移追加）。

関連データ構造

  • struct vars: コンパイルコンテキスト。
  • struct state: NFA の状態ノード。
  • nfa: vars 内の NFA 構造体。
  • cm: vars 内のカラーマップ。
  • wordchrs: 単語構成文字の集合。

失敗モードと境界条件

  • dir に不正な値が渡された場合、assert によりプログラムが停止する可能性がある。
  • メモリ不足等の理由で newarc が失敗した場合の挙動は、newarc の実装に依存する。

保守担当者向けメモ

  • 本関数は \W（非単語文字）の境界判定を NFA 上で表現するために使用される。
  • \n（改行）に対する特別な処理は不要である旨がコメントに記載されている。
