GinBufferInit

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/gin/gininsert.c:1255

目的

GINインデックス構築時に使用する GinBuffer 構造体を初期化し、インデックスキーのソートに必要な比較関数（SortSupport）を準備する。

シグネチャまたは形状

GinBuffer *GinBufferInit(Relation index)

入力と出力

  • **入力**: index (Relation) - 初期化対象のGINインデックスリレーション。
  • **出力**: 初期化済みの GinBuffer 構造体へのポインタ。

前提条件と不変条件

  • index は有効なGINインデックスのリレーションであること。
  • インデックスの各キー属性に対して、比較関数（GIN_COMPARE_PROC）またはデフォルトのBツリー比較関数が定義されていること。

アルゴリズムまたは振る舞い

1. GinBuffer 構造体をメモリ確保し、maxitems を 64KB 分の ItemPointerData 数に設定する。 2. インデックスのキー属性数分だけ SortSupportData 配列を確保する。 3. 各キー属性についてループし、以下を実行する:

  • 照合順序（Collation）を設定する（未指定時は DEFAULT_COLLATION_OID を使用）。
  • インデックスの演算子クラスから GIN_COMPARE_PROC を取得する。
  • 取得できない場合は、型キャッシュからデフォルトのBツリー比較関数を検索する。
  • PrepareSortSupportComparisonShim を呼び出し、比較関数を SortSupport に登録する。

    • 4. 初期化された buffer を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gininsert.c:1666
  • src/backend/access/gin/gininsert.c:1861
  • **呼び出し先**:

  • palloc0
  • ereport (エラー時)

関連データ構造

  • GinBuffer: 構築中のキーとTIDリストを保持する構造体。
  • SortSupportData: ソート処理を最適化するための比較関数情報。
  • Relation: インデックスのメタデータ。

失敗モードと境界条件

  • インデックスキーの型に対して比較関数が特定できない場合、ERRCODE_UNDEFINED_FUNCTION でエラー終了する。
  • メモリ不足時は palloc0 がエラーを発生させる。

保守担当者向けメモ

  • maxitems の計算（64KB固定）はハードコードされているため、将来的に maintenance_work_mem を考慮した動的なサイズ調整を検討する余地がある。
  • SortSupport の設定は、インデックス構築時のソート効率に直結するため、比較関数のルックアップロジックを変更する際は注意が必要。
