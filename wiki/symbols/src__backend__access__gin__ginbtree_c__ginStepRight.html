ginStepRight

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbtree.c:176

目的

GINインデックスのB-tree構造において、現在のページから右側の兄弟ページへ移動し、ロックの受け渡しを行う。

シグネチャまたは形状

Buffer ginStepRight(Buffer buffer, Relation index, int lockmode)

入力と出力

  • **入力**:
  • buffer: 現在保持しているページのバッファ。
  • index: 対象のインデックスリレーション。
  • lockmode: 右兄弟ページ取得時に適用するロックモード（GIN_SHARE または GIN_EXCLUSIVE）。
  • **出力**:
  • 右兄弟ページのバッファ（指定された lockmode でロック済み）。

前提条件と不変条件

  • 入力 buffer は有効であり、呼び出し元でロックされている必要がある。
  • 移動先のページは、移動元のページと「リーフか否か」「データツリーか否か」の属性が一致していなければならない。一致しない場合はエラーとなる。

アルゴリズムまたは振る舞い

1. 現在のページ（buffer）の不透明領域（Opaque）から右兄弟のブロック番号（rightlink）を取得する。 2. ReadBuffer を使用して右兄弟ページを読み込む。 3. LockBuffer を使用して、指定された lockmode で右兄弟ページをロックする。 4. UnlockReleaseBuffer を使用して、元のページ（buffer）のロックを解除し、バッファを解放する。 5. 移動先のページ属性（リーフ属性およびデータ属性）を検証し、矛盾があれば elog(ERROR) を発生させる。 6. 移動先のバッファを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbtree.c (複数箇所)
  • src/backend/access/gin/ginget.c (複数箇所)
  • src/backend/access/gin/ginvacuum.c
  • src/backend/access/gin/ginxlog.c
  • **呼び出し先**:

関連データ構造

  • Buffer: PostgreSQLのバッファ管理構造体。
  • Relation: インデックスリレーションの記述子。
  • GinPageOpaqueData: GINページヘッダーの末尾にある不透明領域。rightlink を保持する。

失敗モードと境界条件

  • **型不一致**: 移動先のページが移動元と異なる種類（リーフ/非リーフ、エントリ/データ）である場合、elog(ERROR) により処理が中断される。
  • **無効なブロック番号**: rightlink が無効な場合、ReadBuffer 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はロックカップリング（Lock Coupling）のパターンに従い、現在のページを解放する前に次のページをロックする。
  • 呼び出し元は、本関数が現在のバッファを解放（ReleaseBuffer）することを認識しておく必要がある。
