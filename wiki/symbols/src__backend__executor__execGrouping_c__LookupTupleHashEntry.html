LookupTupleHashEntry

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execGrouping.c:300

目的

指定されたタプル（TupleTableSlot）に対応するハッシュエントリを、インメモリハッシュテーブルから検索または作成します。

シグネチャまたは形状

TupleHashEntry LookupTupleHashEntry(TupleHashTable hashtable, TupleTableSlot *slot, bool *isnew, uint32 *hash)

入力と出力

  • **入力**:
  • hashtable: 操作対象の TupleHashTable 構造体。
  • slot: 検索または挿入のキーとなるタプルを含む TupleTableSlot。
  • isnew: 新規エントリが作成された場合に true が設定される出力用ポインタ。
  • hash: 計算されたハッシュ値を格納する出力用ポインタ（NULL 指定可）。
  • **出力**:
  • 検索または作成された TupleHashEntry。

前提条件と不変条件

  • 呼び出し側は、ハッシュ計算および比較に必要なメモリコンテキストの管理（tempcxt のリセット等）を適切に行う必要があります。
  • 戻り値が NULL でない場合、entry->hash は local_hash と一致することが保証されます。

アルゴリズムまたは振る舞い

1. hashtable->tempcxt にメモリコンテキストを切り替え、短命なメモリ領域で処理を行います。 2. hashtable 内の inputslot、in_hash_expr、cur_eq_func を現在の slot および関連する関数で更新します。 3. TupleHashTableHash_internal を呼び出し、タプルのハッシュ値を計算します。 4. LookupTupleHashEntry_internal を呼び出し、ハッシュテーブルからエントリを検索または挿入します。 5. 計算されたハッシュ値を hash 引数に書き戻します。 6. 元のメモリコンテキストに復帰し、エントリを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeRecursiveunion.c
  • src/backend/executor/nodeSetOp.c
  • **呼び出し先**:
  • TupleHashTableHash_internal
  • LookupTupleHashEntry_internal
  • MemoryContextSwitchTo

関連データ構造

  • TupleHashTable: グループ化のためのハッシュテーブルを保持する主要構造体。
  • TupleHashEntry: ハッシュテーブル内の各エントリを表す構造体。
  • TupleTableSlot: 処理対象のタプルを保持するコンテナ。

失敗モードと境界条件

  • メモリ不足時には MemoryContext の割り当て失敗が発生する可能性があります。
  • hashtable が適切に初期化されていない場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本関数は tempcxt を使用するため、呼び出し側で tempcxt のリセットが適切に行われているか確認してください。
  • LookupTupleHashEntry_internal が新規エントリ作成時に tablecxt を使用してメモリを確保するため、メモリコンテキストの使い分けに注意してください。
