exec_command_C

定義

  • 種別: function
  • 定義箇所: src/bin/psql/command.c:604

目的

psql の \C コマンド（テーブル出力のタイトル設定）を実行します。

シグネチャまたは形状

static backslashResult exec_command_C(PsqlScanState scan_state, bool active_branch)

入力と出力

  • **入力**:
  • scan_state: コマンド引数を解析するためのスキャン状態。
  • active_branch: 現在の \if ブロックがアクティブかどうかを示すフラグ。
  • **出力**:
  • backslashResult: 実行結果を示す列挙型。成功時は PSQL_CMD_SKIP_LINE、失敗時は PSQL_CMD_ERROR を返します。

前提条件と不変条件

  • active_branch が true の場合、psql_scan_slash_option を介して引数を取得します。
  • active_branch が false の場合、引数の読み飛ばしのみを行います。

アルゴリズムまたは振る舞い

1. active_branch が真であれば、psql_scan_slash_option で引数（タイトル文字列）を取得します。 2. do_pset("title", opt, &pset.popt, pset.quiet) を呼び出し、グローバル設定 pset のタイトルを更新します。 3. 取得した引数メモリを解放します。 4. active_branch が偽であれば、ignore_slash_options を呼び出して引数を無視します。 5. 処理の成否に基づき、PSQL_CMD_SKIP_LINE または PSQL_CMD_ERROR を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c 内のコマンドディスパッチャ（exec_command）から呼び出されます。
  • **呼び出し先**:
  • psql_scan_slash_option
  • do_pset
  • free
  • ignore_slash_options

関連データ構造

  • PsqlScanState: 字句解析の状態管理。
  • backslashResult: コマンド実行後の制御状態。
  • pset: psql のグローバル設定構造体。

失敗モードと境界条件

  • do_pset が失敗した場合、success が false となり、戻り値として PSQL_CMD_ERROR が返されます。
  • 引数が指定されない場合、do_pset の挙動に依存します。

保守担当者向けメモ

  • 本関数は \C コマンド専用です。タイトル設定ロジックを変更する場合は do_pset の仕様を確認してください。
