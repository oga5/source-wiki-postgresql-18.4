AlterTypeNamespace_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/typecmds.c:4109

目的

データ型のスキーマ（名前空間）を変更する際、所有者権限の検証および配列型に対する制約チェックを行い、内部的な変更処理を実行します。

シグネチャまたは形状

Oid AlterTypeNamespace_oid(Oid typeOid, Oid nspOid, bool ignoreDependent, ObjectAddresses *objsMoved)

入力と出力

  • **入力**:
  • typeOid: 変更対象の型のOID。
  • nspOid: 移動先のスキーマのOID。
  • ignoreDependent: 配列型などの依存オブジェクトに対する処理を無視するかどうか。
  • objsMoved: 移動したオブジェクトを記録するリスト。
  • **出力**:
  • 成功時は InvalidOid または内部処理の結果を返します。

前提条件と不変条件

  • 呼び出し元は、対象の型に対して所有者権限を持っている必要があります（object_ownercheck による検証）。
  • 配列型を直接変更することは許可されていません。

アルゴリズムまたは振る舞い

1. object_ownercheck を呼び出し、現在のユーザーが対象型の所有者であることを確認します。所有者でない場合は aclcheck_error_type でエラーを発生させます。 2. get_element_type を使用して、対象が配列型であるかを確認します。配列型である場合、ignoreDependent が偽であればエラー（ERRCODE_WRONG_OBJECT_TYPE）を発生させます。真であれば InvalidOid を返して終了します。 3. 上記の検証を通過した後、AlterTypeNamespaceInternal を呼び出し、実際のカタログ更新処理（スキーマ変更）を実行します。

関連データ構造

  • ObjectAddresses: 移動対象のオブジェクトを追跡するために使用されます。

失敗モードと境界条件

  • **権限不足**: 呼び出し元が型の所有者でない場合、エラーが発生します。
  • **配列型の直接変更**: 配列型を直接変更しようとした場合、ignoreDependent が偽であればエラーが発生します。

保守担当者向けメモ

  • 本関数は AlterTypeNamespaceInternal のラッパーとして機能しており、主に権限チェックと配列型の保護を担当しています。
  • 依存関係の処理については ignoreDependent フラグの挙動に注意してください。
