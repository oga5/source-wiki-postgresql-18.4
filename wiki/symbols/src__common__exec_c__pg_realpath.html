pg_realpath

定義

  • 種別: function
  • 定義箇所: src/common/exec.c:281

目的

ファイルパスを絶対パスに正規化し、シンボリックリンクを解決する。プラットフォーム間の差異を吸収し、POSIX環境とWindows環境の両方で一貫したパス解決を提供する。

シグネチャまたは形状

static char *pg_realpath(const char *fname)

入力と出力

  • **入力**: fname (正規化対象のファイルパス文字列)
  • **出力**: 解決された絶対パスを保持する動的に割り当てられた文字列。失敗した場合は NULL を返す。

前提条件と不変条件

  • 呼び出し元は、返された文字列が不要になった際にメモリを解放する責任を負う。
  • Windows環境では、関数呼び出し前に errno がクリアされる。

アルゴリズムまたは振る舞い

1. **POSIX環境**: realpath(fname, NULL) を呼び出し、パスを正規化する。 2. **Windows環境**:

  • errno を 0 にリセットする。
  • _fullpath(NULL, fname, 0) を呼び出し、絶対パスを取得する。

    • 3. 解決されたパス文字列を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/exec.c 内の normalize_exec_path (行247) から呼び出される。
  • **呼び出し先**:
  • POSIX: realpath(3)
  • Windows: _fullpath()

関連データ構造

  • 特になし（標準の文字列ポインタを使用）。

失敗モードと境界条件

  • パスが存在しない場合やアクセス権限がない場合、realpath または _fullpath の仕様に従い NULL が返される。
  • Windows環境では、_fullpath が失敗した際に errno が設定される可能性がある。

保守担当者向けメモ

  • macOS環境では、より正確なパス解決のために _DARWIN_BETTER_REALPATH の定義が推奨されている。
  • Windowsの _fullpath はPOSIXの realpath と完全に同一ではない可能性があるため、挙動の差異に注意すること。
