BinaryUpgradeClassOidItem

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.c:97

目的

バイナリアップグレード処理において、特定のデータベースオブジェクト（テーブル、TOASTテーブル、およびそれらのインデックス）の OID とファイルノード（RelFileNumber）の対応関係を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	Oid			oid;			/* object OID */
	char		relkind;		/* object kind */
	RelFileNumber relfilenumber;	/* object filenode */
	Oid			toast_oid;		/* toast table OID */
	RelFileNumber toast_relfilenumber;	/* toast table filenode */
	Oid			toast_index_oid;	/* toast table index OID */
	RelFileNumber toast_index_relfilenumber;	/* toast table index filenode */
} BinaryUpgradeClassOidItem;

入力と出力

  • 本構造体はデータ保持用であり、関数としての入出力はありません。
  • pg_dump 実行時に、バイナリアップグレード用のメタデータとしてメモリ上に配列として構築されます。

前提条件と不変条件

  • binaryUpgradeClassOids という静的配列で管理されます。
  • bsearch() を用いた検索が想定されており、配列は OID 順にソートされている必要があります。

アルゴリズムまたは振る舞い

  • pg_dump 内で、バイナリアップグレード時に必要なオブジェクトの物理的な識別子を追跡します。
  • bsearch() の比較関数において、BinaryUpgradeClassOidItem 型のインスタンス同士を比較し、OID に基づく検索を可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_dump.c 内のバイナリアップグレード関連処理。
  • 呼び出し先: pg_malloc（メモリ確保時）、bsearch（検索時）。
  • 注: 具体的な呼び出し元の関数名はソース抜粋からは特定できません。

関連データ構造

  • Oid (PostgreSQL オブジェクト識別子)
  • RelFileNumber (ファイルノード識別子)
  • binaryUpgradeClassOids (本構造体を要素とする静的配列)

失敗モードと境界条件

  • メモリ不足時: pg_malloc によるメモリ確保失敗。
  • 検索失敗: bsearch で該当する OID が見つからない場合、NULL が返される可能性があります。

保守担当者向けメモ

  • 本構造体はバイナリアップグレード機能に特化しています。変更を加える際は、pg_dump.c 内の検索ロジックおよびソート順序が維持されていることを確認してください。
