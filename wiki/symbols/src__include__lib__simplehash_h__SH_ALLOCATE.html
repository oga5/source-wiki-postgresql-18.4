SH_ALLOCATE

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/lib/simplehash.h:411

目的

simplehash.h で生成されるハッシュテーブルにおいて、メモリの割り当てを抽象化し、カスタムアロケータまたはPostgreSQLのメモリコンテキストのいずれかを利用してメモリを確保する。

シグネチャまたは形状

void *SH_ALLOCATE(SH_TYPE *type, Size size)

入力と出力

  • **入力**:
  • type: ハッシュテーブルの制御構造体へのポインタ。
  • size: 確保するメモリのバイトサイズ。
  • **出力**:
  • 確保されたメモリ領域への void * ポインタ。

前提条件と不変条件

  • SH_RAW_ALLOCATOR マクロが定義されている場合、その関数が優先的に使用される。
  • SH_RAW_ALLOCATOR が未定義の場合、type->ctx に有効な MemoryContext が設定されている必要がある。

アルゴリズムまたは振る舞い

1. SH_RAW_ALLOCATOR が定義されている場合、SH_RAW_ALLOCATOR(size) を呼び出して結果を返す。 2. 定義されていない場合、MemoryContextAllocExtended を使用してメモリを確保する。

  • MCXT_ALLOC_HUGE フラグにより、巨大なメモリ割り当てを許可する。
  • MCXT_ALLOC_ZERO フラグにより、確保したメモリをゼロクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ハッシュテーブルの初期化時（src/include/lib/simplehash.h:464）
  • ハッシュテーブルの拡張時（src/include/lib/simplehash.h:509）
  • **呼び出し先**:
  • SH_RAW_ALLOCATOR（定義されている場合）
  • MemoryContextAllocExtended（utils/memutils.h）

関連データ構造

  • SH_TYPE: ハッシュテーブルの型定義。
  • MemoryContext: PostgreSQLのメモリ管理コンテキスト。

失敗モードと境界条件

  • メモリ不足時には、MemoryContextAllocExtended の仕様に従い、通常はエラーが発生して処理が中断される。
  • SH_RAW_ALLOCATOR を使用する場合、その実装に依存する。

保守担当者向けメモ

  • 本関数はマクロによるテンプレート生成の一部であり、インクルードされるたびに再定義される可能性がある。
  • SH_RAW_ALLOCATOR の有無によってメモリ管理の挙動が大きく変わるため、カスタムアロケータを導入する際は注意すること。
