find_unconnected_slot

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:158

目的

ParallelSlotArray 内で、現在使用されておらず、かつデータベース接続が確立されていないスロットを検索します。

シグネチャまたは形状

static int find_unconnected_slot(const ParallelSlotArray *sa)

入力と出力

  • **入力**: sa (const ParallelSlotArray *) - 検索対象のスロット配列。
  • **出力**:
  • 条件を満たすスロットのインデックス（int）。
  • 見つからない場合は -1。

前提条件と不変条件

  • sa は有効な ParallelSlotArray 構造体を指している必要があります。
  • スロットの inUse フラグが false であり、かつ connection メンバが NULL であることが検索条件です。

アルゴリズムまたは振る舞い

1. 配列 sa->slots をインデックス 0 から sa->numslots - 1 まで順次走査します。 2. 各スロットについて以下の条件を評価します。

  • inUse が真であればスキップします。
  • connection が NULL であれば、そのインデックスを即座に返します。

    • 3. すべてのスロットを確認しても条件に合致するものがない場合、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/fe_utils/parallel_slot.c 内の関数から呼び出されています（行 389 および 464）。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • ParallelSlotArray: スロットの配列と総数を保持する構造体。
  • ParallelSlot: 個別の接続状態（inUse, connection）を保持する構造体。

失敗モードと境界条件

  • sa->numslots が 0 の場合、ループは実行されず -1 を返します。
  • すべてのスロットが使用中（inUse == true）または接続済み（connection != NULL）である場合、-1 を返します。

保守担当者向けメモ

  • 本関数は static であり、parallel_slot.c 内でのみ使用されます。
  • 接続の再利用や新規接続の割り当てを行う際の、空きスロット探索ロジックの一部として機能します。
