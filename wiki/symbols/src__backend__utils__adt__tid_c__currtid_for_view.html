currtid_for_view

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:337

目的

ビュー（View）を介して参照されているベーステーブルのタプルに対し、指定された TID（Tuple Identifier）が指すタプルの最新バージョンを取得する。

シグネチャまたは形状

static ItemPointer currtid_for_view(Relation viewrel, ItemPointer tid)

入力と出力

  • **入力**:
  • viewrel: 対象となるビューのリレーション記述子。
  • tid: 検索対象の ItemPointer（物理タプル位置）。
  • **出力**:
  • 成功時: 最新のタプル位置を示す ItemPointer。
  • 失敗時: エラーをスローし、処理を中断する。

前提条件と不変条件

  • ビューのタプル記述子（TupleDesc）に ctid という名前の属性が存在し、かつその型が TIDOID であること。
  • ビューに CMD_SELECT イベントに対するルール（RewriteRule）が定義されていること。
  • ビューの定義が、ベーステーブルの ctid を直接参照する単純な構造であること。

アルゴリズムまたは振る舞い

1. ビューの属性を走査し、ctid 属性のインデックスを特定する。存在しない場合はエラー。 2. ビューに定義されたルールを走査し、CMD_SELECT ルールを検索する。 3. ルール内のクエリのターゲットリストから、ctid に対応する式を取得する。 4. 式が Var ノードであり、かつベーステーブルの ctid を指していることを確認する。 5. 該当するベーステーブルを table_open で開き、currtid_internal を呼び出して最新の TID を取得する。 6. 開いたリレーションを table_close で閉じ、結果を返す。 7. 上記の条件を満たさない場合、currtid が処理できないビューとしてエラーを投げる。

呼び出し元と呼び出し先

  • **呼び出し元**: currtid_byrelname（src/backend/utils/adt/tid.c:312）から呼び出される。
  • **呼び出し先**:
  • table_open / table_close
  • currtid_internal
  • RelationGetDescr, TupleDescAttr, rt_fetch, get_tle_by_resno 等のアクセサ関数。

関連データ構造

  • ItemPointer: タプルの物理位置（ブロック番号とオフセット）。
  • Relation: リレーション記述子。
  • RewriteRule: ビューの書き換えルール。
  • Var: クエリ内の変数ノード。

失敗モードと境界条件

  • ctid 属性がない、または型が異なる場合に ERRCODE_FEATURE_NOT_SUPPORTED をスロー。
  • ビューにルールが存在しない、または SELECT ルールが複数ある場合にエラーをスロー。
  • ビューの定義が複雑でベーステーブルの ctid を直接特定できない場合、elog(ERROR) により処理を中断する。

保守担当者向けメモ

  • 本関数はビューの定義を静的に解析するため、複雑なビューやルールには対応できない設計となっている。
  • table_open には AccessShareLock を使用している。
  • 呼び出し元および呼び出し先の情報は、ソースコード内の直接的な参照に基づいている。
