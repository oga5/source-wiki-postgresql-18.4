resolve_unique_index_expr

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:3200

目的

ON CONFLICT 句における一意性インデックスの推論（inference）のために、指定されたインデックス要素（InferClause）を意味解析し、InferenceElem のリストへ変換します。

シグネチャまたは形状

static List *resolve_unique_index_expr(ParseState *pstate, InferClause *infer, Relation heapRel);

入力と出力

  • **入力**:
  • pstate: 現在の構文解析状態。
  • infer: 解析対象の InferClause（インデックス要素のリストを含む）。
  • heapRel: 対象となるリレーション（使用箇所なし）。
  • **出力**: 変換済みの InferenceElem ノードを含む List。

前提条件と不変条件

  • infer->indexElems 内の各要素において、ASC/DESC 指定や NULLS FIRST/NULLS LAST 指定は許可されません。これらが含まれる場合はエラーとなります。
  • transformExpr を介して式が変換されるため、サブクエリ、集約関数、ウィンドウ関数、SRF はこのコンテキストでは拒否されます。

アルゴリズムまたは振る舞い

1. infer->indexElems を反復処理します。 2. 各 IndexElem に対して以下を実行します：

  • 式が未定義の場合、ColumnRef を生成して列参照として扱います。
  • transformExpr を呼び出し、EXPR_KIND_INDEX_EXPRESSION として式を変換します。
  • 照合順序（Collation）が指定されている場合は LookupCollation で解決し、指定がない場合は InvalidOid を設定します。
  • 演算子クラス（Opclass）が指定されている場合は get_opclass_oid で解決し、指定がない場合は InvalidOid を設定します。

    • 3. 変換結果を InferenceElem 構造体に格納し、リストに追加して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_clause.c 内の ON CONFLICT 句を処理する箇所（例: arbiterExpr の生成時）。
  • **呼び出し先**: transformExpr, LookupCollation, get_opclass_oid, makeNode, lfirst, lappend 等。

関連データ構造

  • InferClause: 構文解析器が生成する入力構造体。
  • InferenceElem: 変換後のインデックス推論要素。
  • ParseState: 構文解析の状態管理。

失敗モードと境界条件

  • **エラー**: ASC/DESC または NULLS FIRST/NULLS LAST が指定されている場合、ERRCODE_INVALID_COLUMN_REFERENCE でエラーを報告します。
  • **制約**: transformExpr により、インデックス式として不適切なノード（サブクエリ等）が含まれる場合は解析エラーとなります。

保守担当者向けメモ

  • 本関数は CREATE INDEX のインフラを再利用する ON CONFLICT 句の特性上、構文解析レベルでの厳格なチェックを行っています。
  • heapRel 引数は現在の実装では使用されていませんが、シグネチャに含まれています。
