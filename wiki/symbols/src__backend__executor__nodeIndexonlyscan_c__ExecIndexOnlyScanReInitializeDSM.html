ExecIndexOnlyScanReInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:812

目的

パラレルクエリ実行時において、インデックスのみスキャン（Index-Only Scan）のDSM（動的共有メモリ）状態を再初期化し、スキャンを最初からやり直せる状態にする。

シグネチャまたは形状

void ExecIndexOnlyScanReInitializeDSM(IndexOnlyScanState *node, ParallelContext *pcxt)

入力と出力

  • node: インデックスのみスキャンの実行状態を保持する IndexOnlyScanState 構造体へのポインタ。
  • pcxt: 並列実行コンテキスト ParallelContext へのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • node->ss.ps.plan->parallel_aware が真であること（アサーションにより強制）。

アルゴリズムまたは振る舞い

1. node->ss.ps.plan->parallel_aware が真であることを確認する。 2. node->ioss_ScanDesc を引数として index_parallelrescan を呼び出し、インデックススキャンの並列再スキャン処理を実行する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execParallel.c:988 にて呼び出されていることが確認されている。
  • 呼び出し先: index_parallelrescan（インデックスアクセスメソッドの並列再スキャン用API）。

関連データ構造

  • IndexOnlyScanState: インデックスのみスキャンの実行状態を管理する構造体。
  • ParallelContext: 並列クエリの実行コンテキスト。

失敗モードと境界条件

  • node が指す構造体が正しく初期化されていない場合、または parallel_aware でないプランに対して呼び出された場合はアサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は、並列実行中のスキャンをリセットする際に呼び出される。index_parallelrescan の呼び出しにより、インデックススキャンの内部状態が再初期化される点に注意すること。
