allow_star_schema_join

定義

  • 種別: static inline function
  • 定義箇所: src/backend/optimizer/path/joinpath.c:362

目的

スター型スキーマ（Star Schema）における結合条件を満たすかどうかを判定します。具体的には、内部リレーションのパラメータ化が外部リレーションによって部分的にのみ解決されるケースを識別するために使用されます。

シグネチャまたは形状

static inline bool allow_star_schema_join(PlannerInfo *root, Relids outerrelids, Relids inner_paramrels)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報（PlannerInfo）。
  • outerrelids: 外部リレーションの識別子セット（Relids）。
  • inner_paramrels: 内部リレーションが依存するパラメータリレーションのセット（Relids）。
  • **出力**:
  • bool: スター型スキーマ結合が許可される場合は true、それ以外は false。

前提条件と不変条件

  • inner_paramrels と outerrelids は Relids 型（ビットマップセット）である必要があります。
  • 内部リレーションが外部リレーションによって「一部のみ」パラメータ化されている状態（inner_paramrels と outerrelids が重なりを持ち、かつ inner_paramrels が outerrelids に完全には含まれない状態）を判定します。

アルゴリズムまたは振る舞い

以下の論理積が真となる場合に true を返します。 1. bms_overlap(inner_paramrels, outerrelids): 内部リレーションのパラメータと外部リレーションの間に共通の要素が存在する。 2. bms_nonempty_difference(inner_paramrels, outerrelids): 内部リレーションのパラメータのうち、外部リレーションに含まれない要素が少なくとも一つ存在する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinpath.c 内の結合パス生成ロジック（特に 885 行目付近）から、パラメータ化された結合の妥当性チェックとして呼び出されます。
  • **呼び出し先**: bms_overlap および bms_nonempty_difference（ビットマップセット操作関数）。

関連データ構造

  • Relids: リレーションの集合を表現するビットマップ。
  • PlannerInfo: クエリ全体のプランニング情報を保持する構造体。

失敗モードと境界条件

  • inner_paramrels が空集合の場合、bms_overlap が偽となるため、常に false を返します。
  • 内部リレーションのパラメータがすべて外部リレーションで解決される場合（bms_nonempty_difference が偽となる場合）、スター型スキーマとは見なされず false を返します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、オプティマイザの結合パス生成におけるコスト計算やパスの有効性判定の最適化パスで使用されます。
  • ロジックは単純な集合演算に基づいているため、変更時は Relids の定義やビットマップ操作のセマンティクスに注意してください。
