vacuum_open_relation

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/commands/vacuum.c:786

目的

VACUUM または ANALYZE 操作の対象となるリレーションをオープンし、適切なロックを取得する。リレーションが存在しない場合やロックが取得できない場合のログ出力も担当する。

シグネチャまたは形状

Relation vacuum_open_relation(Oid relid, RangeVar *relation, bits32 options, bool verbose, LOCKMODE lmode)

入力と出力

  • **入力**:
  • relid: 対象リレーションのOID。
  • relation: リレーション名情報（ログ出力用）。NULLの場合、ログ出力はスキップされる。
  • options: VACOPT_* ビットフラグ。
  • verbose: 詳細ログ出力フラグ。
  • lmode: 取得するロックモード。
  • **出力**:
  • 成功時: オープンされた Relation 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • options に VACOPT_VACUUM または VACOPT_ANALYZE のいずれかが含まれている必要がある（Assert で検証）。

アルゴリズムまたは振る舞い

1. **ロックとオープン**:

  • VACOPT_SKIP_LOCKED が指定されていない場合、try_relation_open を使用してリレーションをオープンする。
  • VACOPT_SKIP_LOCKED が指定されている場合、ConditionalLockRelationOid で非ブロック的にロックを試み、成功時のみ try_relation_open を実行する。

    • 2. **成功時の終了**: リレーションが取得できた場合は即座に返却する。 3. **失敗時の処理**:

  • relation が NULL の場合は何もせず NULL を返す。
  • 自動バキュームワーカー以外の場合は WARNING レベルで、自動バキュームワーカーの場合は verbose 指定があれば LOG レベルでエラーメッセージを報告する。
  • VACUUM と ANALYZE の両方のオプションがある場合、VACUUM に関するエラー報告を優先する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c
  • src/backend/commands/vacuum.c
  • **呼び出し先**:
  • try_relation_open
  • ConditionalLockRelationOid
  • AmAutoVacuumWorkerProcess
  • ereport

関連データ構造

  • Relation: オープンされたテーブルのメタデータ構造体。
  • RangeVar: リレーションの識別情報。
  • LOCKMODE: ロックの強度を指定する列挙型。

失敗モードと境界条件

  • **リレーション不在**: try_relation_open が失敗した場合、ERRCODE_UNDEFINED_TABLE を含むメッセージを報告する。
  • **ロック競合**: ConditionalLockRelationOid が失敗した場合、ERRCODE_LOCK_NOT_AVAILABLE を含むメッセージを報告する。
  • **情報不足**: relation 引数が NULL の場合、ログ出力を行わずに NULL を返す。

保守担当者向けメモ

  • 本関数はトランザクションの境界を管理しない。呼び出し元で適切なトランザクション制御が行われていることを前提としている。
  • VACOPT_SKIP_LOCKED 指定時のロック取得失敗と、リレーション自体の消失を区別してエラーメッセージを出し分けている点に注意。
