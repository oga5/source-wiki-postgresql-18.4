xl_xact_stats_items

定義

  • 種別: struct
  • 定義箇所: src/include/access/xact.h:295

目的

トランザクションに関連する統計情報の項目群をWAL（Write-Ahead Logging）に記録するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct xl_xact_stats_items
{
    int            nitems;
    xl_xact_stats_item items[FLEXIBLE_ARRAY_MEMBER];
} xl_xact_stats_items;

入力と出力

  • **入力**: トランザクション終了時に統計情報が収集され、本構造体に格納されます。
  • **出力**: WALレコードの一部としてシリアライズされ、リカバリ時やデコード時に読み取られます。

前提条件と不変条件

  • items フィールドは可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されています。
  • 構造体のサイズ計算には、定義済みのマクロ MinSizeOfXactStatsItems（offsetof(xl_xact_stats_items, items)）を使用する必要があります。
  • WALレコードのフラグ XINFO_HAS_DROPPED_STATS が設定されている場合にのみ、本構造体がWALデータ内に存在することが期待されます。

アルゴリズムまたは振る舞い

トランザクションのコミットまたはアボート処理において、統計情報の更新が必要な場合にこの構造体がインスタンス化されます。WALログの解析時には、データポインタを本構造体にキャストして nitems を参照し、後続の xl_xact_stats_item 配列を走査します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c: トランザクション終了処理（xl_dropped_stats 変数として使用）。
  • src/backend/access/rmgrdesc/xactdesc.c: WALレコードのデコード処理（xl_drops 変数として使用）。
  • **呼び出し先**:
  • 本構造体自体はデータコンテナであり、特定の関数を呼び出すことはありません。

関連データ構造

  • xl_xact_stats_item: 本構造体内の配列要素として保持される個別の統計項目。

失敗モードと境界条件

  • 可変長配列であるため、メモリレイアウトを直接操作する際は、必ず MinSizeOfXactStatsItems を用いたオフセット計算を行い、バッファオーバーランを回避する必要があります。

保守担当者向けメモ

  • 本構造体はWALレコードの一部であるため、構造の変更はWALの互換性に影響を与えます。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、C言語の標準的な構造体サイズ計算（sizeof）は意図した結果にならない点に注意してください。
