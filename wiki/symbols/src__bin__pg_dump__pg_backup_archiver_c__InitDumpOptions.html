InitDumpOptions

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:145

目的

DumpOptions 構造体を初期化し、ダンプ処理に必要なデフォルト値を設定する。

シグネチャまたは形状

void InitDumpOptions(DumpOptions *opts)

入力と出力

  • **入力**: DumpOptions *opts（初期化対象の構造体へのポインタ）
  • **出力**: なし（引数 opts が指すメモリ領域を直接更新する）

前提条件と不変条件

  • 引数 opts は有効なメモリ領域を指している必要がある。
  • memset により構造体全体がゼロクリアされるため、明示的に設定されないフィールドは 0 または NULL になる。

アルゴリズムまたは振る舞い

1. memset を使用して opts が指す領域を sizeof(DumpOptions) 分だけゼロクリアする。 2. 以下のデフォルト値を設定する:

  • include_everything = true
  • cparams.promptPassword = TRI_DEFAULT
  • dumpSections = DUMP_UNSECTIONED
  • dumpSchema = true
  • dumpData = true
  • dumpStatistics = false

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c:138
  • src/bin/pg_dump/pg_dump.c:564
  • **呼び出し先**:
  • memset (標準ライブラリ関数)

関連データ構造

  • DumpOptions: ダンプ処理の挙動を制御する設定構造体。

失敗モードと境界条件

  • 引数 opts が NULL の場合、memset 呼び出しによりセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数はダンプ設定の初期化を一元管理している。新しいダンプオプションを追加した際は、本関数で適切なデフォルト値が設定されているか確認すること。
