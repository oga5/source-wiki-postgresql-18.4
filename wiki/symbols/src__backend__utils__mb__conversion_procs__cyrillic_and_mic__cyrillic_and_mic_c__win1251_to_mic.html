win1251_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:373

目的

WIN1251 エンコーディングの文字列を PostgreSQL 内部のマルチバイト表現（MULE_INTERNAL）へ変換する。

シグネチャまたは形状

Datum win1251_to_mic(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取る。
  • src (CSTRING): 変換元の WIN1251 文字列。
  • dest (CSTRING): 変換結果を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 変換エラー発生時の挙動フラグ。
  • **出力**: PG_RETURN_INT32 により、変換されたバイト数を返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_WIN1251, PG_MULE_INTERNAL) により、ソースエンコーディングが WIN1251、宛先エンコーディングが MULE_INTERNAL であることが検証される。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. latin2mic_with_table 関数を呼び出し、変換テーブル win12512koi を使用して WIN1251 から MULE_INTERNAL への変換を実行する。 3. 変換後のバイト数を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: latin2mic_with_table（変換処理の実行）。

関連データ構造

  • win12512koi: WIN1251 から KOI8-R へのマッピングを定義する static const unsigned char 配列。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断される。
  • noError 引数の値に応じて、変換不可能な文字が含まれる場合の挙動が latin2mic_with_table 内で制御される。

保守担当者向けメモ

  • 本関数は latin2mic_with_table を利用しており、KOI8-R が MULE_INTERNAL のキリル文字表現と同一であることを前提としている。
  • 変換テーブル win12512koi の内容を変更する場合は、対応するエンコーディング定義との整合性に注意すること。
