jsonb_to_tsvector_worker

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/to_tsany.c:284

目的

Jsonb 型のデータを走査し、全文検索用の tsvector 型へ変換するための内部処理を実行します。

シグネチャまたは形状

static TSVector jsonb_to_tsvector_worker(Oid cfgId, Jsonb *jb, uint32 flags)

入力と出力

  • **入力**:
  • cfgId (Oid): 全文検索設定の識別子。
  • jb (Jsonb *): 変換対象の JSONB データ。
  • flags (uint32): JSONB の走査方法や変換ルールを制御するフラグ。
  • **出力**:
  • TSVector: 変換結果として生成された全文検索用ベクトル。

前提条件と不変条件

  • ParsedText 構造体 (prs) が初期化され、words が NULL、curwords が 0 に設定されている必要があります。
  • TSVectorBuildState 構造体を通じて、設定された cfgId が iterate_jsonb_values に渡される必要があります。

アルゴリズムまたは振る舞い

1. ParsedText および TSVectorBuildState を初期化し、解析コンテキストを構築します。 2. iterate_jsonb_values を呼び出し、指定された flags に基づいて jb を走査します。この際、各値に対して add_to_tsvector コールバックを実行し、トークンを抽出・蓄積します。 3. 蓄積された解析済みテキスト (prs) を make_tsvector に渡し、最終的な TSVector 構造体を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • jsonb_string_to_tsvector_byid
  • jsonb_string_to_tsvector
  • jsonb_to_tsvector_byid
  • jsonb_to_tsvector
  • **呼び出し先**:
  • iterate_jsonb_values
  • make_tsvector
  • add_to_tsvector (コールバックとして使用)

関連データ構造

  • TSVectorBuildState: 変換プロセス中の状態（解析済みテキストと設定ID）を保持します。
  • ParsedText: トークン化された単語列を保持する構造体。
  • Jsonb: 変換対象の入力データ構造。

失敗モードと境界条件

  • 入力 Jsonb が空の場合、空の TSVector が生成されます。
  • iterate_jsonb_values 内でのメモリ割り当て失敗や、make_tsvector での構築失敗が発生する可能性があります。

保守担当者向けメモ

  • 本関数は to_tsany.c 内の JSONB 変換ロジックの共通基盤です。JSONB の走査ロジックに変更を加える場合は、iterate_jsonb_values の引数や flags の定義を確認してください。
  • make_tsvector はメモリを動的に確保するため、呼び出し側での適切なメモリ管理（メモリコンテキストの生存期間）に注意してください。
