PullFilter

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/mbuf.c:180

目的

pgcrypto モジュールにおいて、データストリームを段階的に読み込み、変換（復号や解凍など）を適用するためのフィルタチェーンを構成する構造体です。

シグネチャまたは形状

struct PullFilter
{
    PullFilter *src;           /* 上流のフィルタ */
    const PullFilterOps *op;   /* フィルタ操作の関数テーブル */
    int         buflen;        /* バッファサイズ */
    uint8      *buf;           /* 内部データバッファ */
    int         pos;           /* 現在の読み込み位置 */
    void       *priv;          /* フィルタ固有のプライベートデータ */
};

入力と出力

  • **入力**: フィルタチェーンの構築時に src（上流フィルタ）が指定されます。
  • **出力**: pullf_read 等の関数を通じて、フィルタ処理済みのデータが uint8 **data_p として提供されます。

前提条件と不変条件

  • op（PullFilterOps）は、フィルタの初期化およびデータ読み込みに必要な関数ポインタを保持している必要があります。
  • priv は、各フィルタの実装が動的に割り当てるメモリ領域を指します。
  • palloc を使用してメモリ管理が行われるため、適切なタイミングで pullf_free を呼び出し、メモリリークを防ぐ必要があります。

アルゴリズムまたは振る舞い

1. **チェーン構造**: src ポインタを介してフィルタが連結され、データは末端から順次読み込まれます。 2. **読み込み処理**: pullf_read が呼び出されると、現在のフィルタの op->pull 関数が実行されます。 3. **再帰的取得**: データが不足している場合、src（上流）のフィルタを再帰的に呼び出し、必要なデータを取得します。 4. **バッファリング**: buf を使用して、フィルタ処理に必要な中間データを一時的に保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: pullf_create, pullf_read, pullf_read_max, pullf_read_fixed, pullf_free, pull_from_mbuf, pullf_create_mbuf_reader などが本構造体を操作します。
  • **呼び出し先**: op 構造体内の関数ポインタ（init, pull など）が実行時に呼び出されます。

関連データ構造

  • PullFilterOps: フィルタの振る舞いを定義する関数テーブル。
  • MBuf: データのソースとして利用されるメモリバッファ。
  • PushFilter: 書き込み処理用の対となるフィルタ構造体。

失敗モードと境界条件

  • **データ破損**: pullf_read_fixed 等で期待されるバイト数が読み込めない場合、PXE_PGP_CORRUPT_DATA エラーが発生する可能性があります。
  • **メモリ不足**: palloc 失敗時に PostgreSQL のエラーハンドリング機構により中断されます。

保守担当者向けメモ

  • 本構造体はポリモーフィックな設計です。新しいフィルタを追加する場合は、PullFilterOps を実装し、pullf_create を通じて初期化してください。
  • priv 領域の解放は、フィルタ固有のクリーンアップ処理が必要な場合があるため、pullf_free の実装に注意してください。
