checkcondition_gin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsginidx.c:182

目的

TS_execute_ternary のコールバック関数として、GINインデックス内のキー状態と tsquery のオペランドを照合し、検索条件を満たすかを判定します。

シグネチャまたは形状

TSTernaryValue checkcondition_gin(void *checkval, QueryOperand *val, ExecPhraseData *data);

入力と出力

  • **入力**:
  • checkval: GinChkVal 構造体へのポインタ。インデックス内のキーの存在状態を保持。
  • val: 評価対象の QueryOperand。
  • data: ExecPhraseData 構造体へのポインタ（位置情報検索用）。
  • **出力**:
  • TSTernaryValue: 評価結果（GIN_TRUE, GIN_FALSE, GIN_MAYBE のいずれか）。

前提条件と不変条件

  • GinTernaryValue と TSTernaryValue の値の割り当てが等価であることを前提としています。
  • gcv->map_item_operand を使用して、クエリ内のアイテム番号からインデックス内のエントリ番号へのマッピングを行います。

アルゴリズムまたは振る舞い

1. val からインデックス内のエントリ番号 j を特定します。 2. gcv->check[j] を参照し、インデックス内での該当キーの存在状態を取得します。 3. 取得した結果が GIN_TRUE であっても、以下のいずれかに該当する場合は GIN_MAYBE に昇格させます（再評価が必要なため）。

  • val->weight が 0 以外（重み付け条件がある）。
  • data が NULL ではない（位置情報が必要）。

    • 4. 最終的な TSTernaryValue を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: TS_execute_ternary（src/backend/utils/adt/tsginidx.c 内の整合性判定ロジックから間接的に呼び出される）。
  • **呼び出し先**: 外部からの直接的な呼び出し先情報は提供されていません。

関連データ構造

  • GinChkVal: インデックスの検索状態とマッピング情報を保持する構造体。
  • QueryOperand: tsquery のオペランド情報。
  • TSTernaryValue / GinTernaryValue: 評価結果を示す列挙型。

失敗モードと境界条件

  • data が提供されている場合や重み付けが指定されている場合、インデックスのみでの判定が不可能なため、常に GIN_MAYBE を返して再評価（recheck）を促す設計となっています。

保守担当者向けメモ

  • 本関数は TS_execute_ternary のコールバックとして機能します。判定ロジックを変更する際は、GinTernaryValue と TSTernaryValue の互換性が維持されていることを確認してください。
  • デバッグ時は、gcv->check 配列内の状態と、val->weight 等のクエリ属性が正しくマッピングされているかを確認してください。
