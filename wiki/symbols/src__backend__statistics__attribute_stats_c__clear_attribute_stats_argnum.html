clear_attribute_stats_argnum

定義

  • 種別: enum
  • 定義箇所: src/backend/statistics/attribute_stats.c:85

目的

pg_clear_attribute_stats 関数に渡される引数のインデックスを定義し、引数配列へのアクセスを型安全かつ可読性の高い形式で提供します。

シグネチャまたは形状

enum clear_attribute_stats_argnum
{
    C_ATTRELSCHEMA_ARG = 0,
    C_ATTRELNAME_ARG,
    C_ATTNAME_ARG,
    C_INHERITED_ARG,
    C_NUM_ATTRIBUTE_STATS_ARGS
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • C_ATTRELSCHEMA_ARG から C_INHERITED_ARG までの値は、統計削除関数が期待する引数の順序と一致している必要があります。
  • C_NUM_ATTRIBUTE_STATS_ARGS は、引数配列のサイズを定義する定数として機能します。

アルゴリズムまたは振る舞い

  • 統計削除処理において、特定の引数（スキーマ名、リレーション名、属性名、継承フラグ）を配列から取得する際のオフセットとして使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: pg_clear_attribute_stats 関数がこの列挙値を使用して引数配列にアクセスしていることが示唆されますが、本ソース excerpt 内に直接的な呼び出しコードの記述はありません。
  • 呼び出し先: なし。

関連データ構造

  • pg_clear_attribute_stats: 本列挙型を利用する主要な関数。

失敗モードと境界条件

  • 列挙値の順序が変更された場合、pg_clear_attribute_stats 側の引数解析ロジックと不整合が生じるリスクがあります。

保守担当者向けメモ

  • 引数を追加・削除する場合は、必ず C_NUM_ATTRIBUTE_STATS_ARGS の値が最新の要素数と一致するように更新してください。
