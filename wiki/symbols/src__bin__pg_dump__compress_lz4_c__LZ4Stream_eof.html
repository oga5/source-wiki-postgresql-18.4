LZ4Stream_eof

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:321

目的

LZ4 圧縮ストリームにおいて、読み込み対象のデータが完全に終了したかどうかを判定する。

シグネチャまたは形状

bool LZ4Stream_eof(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH（LZ4 圧縮ストリームの状態を保持するハンドル）
  • **出力**: bool（ストリームが EOF に達している場合は true、そうでない場合は false）

前提条件と不変条件

  • CFH は有効な CompressFileHandle を指していること。
  • CFH->private_data は LZ4State 構造体としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. CFH->private_data から LZ4State を取得する。 2. 以下の両方の条件を満たす場合にのみ true を返す。

  • state->overflowlen == 0: 解凍済みデータがオーバーフローバッファに残っていない。
  • feof(state->fp): 内部ファイルポインタが EOF に達している。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_lz4.c:632（条件判定で使用）
  • src/bin/pg_dump/compress_lz4.c:793（CFH->eof_func への関数ポインタ代入）
  • **呼び出し先**:
  • feof()（標準 C ライブラリ関数）

関連データ構造

  • CompressFileHandle: 圧縮ストリームの抽象化インターフェース。
  • LZ4State: LZ4 圧縮・解凍のコンテキストおよびバッファ状態を保持する構造体。

失敗モードと境界条件

  • CFH が NULL の場合、あるいは private_data が初期化されていない場合は未定義の動作となる。
  • オーバーフローバッファにデータが残っている間は、物理的なファイルが EOF であっても false を返す。

保守担当者向けメモ

  • 本関数は CompressFileHandle の eof_func インターフェースを実装している。
  • 読み込み処理において、解凍後のバッファと物理的なファイル終端の両方を考慮する必要があるため、単純な feof ではなく本関数を経由して判定を行うこと。
