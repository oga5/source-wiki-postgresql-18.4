SortItem

定義

  • 種別: struct
  • 定義箇所: src/include/statistics/extended_stats_internal.h:53

目的

拡張統計情報の構築過程において、複数列の値を保持し、ソートや比較を行うためのデータ単位として使用されます。

シグネチャまたは形状

typedef struct SortItem
{
	Datum	   *values;
	bool	   *isnull;
	int			count;
} SortItem;

入力と出力

  • values: 各列の値を格納する Datum 配列へのポインタ。
  • isnull: 各列の値が NULL かどうかを示す bool 配列へのポインタ。
  • count: 保持している列数（次元数）。

前提条件と不変条件

  • values および isnull 配列は、少なくとも count 分の要素を保持している必要があります。
  • 統計構築の文脈では、メモリレイアウトを連続させるために SortItem 自体の配列と、その各要素が指す values/isnull 配列がメモリ上で一括確保されるケースがあります（extended_stats.c 参照）。

アルゴリズムまたは振る舞い

  • 複数列の値を一括して保持し、MultiSortSupport を用いた比較処理の対象となります。
  • extended_stats.c 内の比較関数（multi_sort_compare_dim 等）において、特定の次元の値を比較するための引数として渡されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/statistics/dependencies.c: 統計構築時の配列管理。
  • src/backend/statistics/extended_stats.c: ソート処理および統計構築時のデータ保持。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。比較ロジックは utils/sortsupport.h に依存します。

関連データ構造

  • MultiSortSupport: 本構造体をソートするための比較ロジックを管理する構造体。
  • StatsBuildData: 統計構築の入力データセット。

失敗モードと境界条件

  • count が 0 の場合、values および isnull ポインタが指すメモリ領域の扱いに注意が必要です。
  • メモリ確保時に SortItem 配列とデータ配列を混在させる場合、ポインタ演算のオフセット計算が正しく行われないとメモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は統計構築時のメモリ効率を考慮して設計されています。構造体のメンバやメモリレイアウトを変更する場合は、extended_stats.c 内の SortItem 配列確保ロジック（sizeof(SortItem) を用いた計算箇所）との整合性を確認してください。
