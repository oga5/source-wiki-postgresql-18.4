AlterPublicationSchemas

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/publicationcmds.c:1360

目的

パブリケーションに関連付けられたスキーマの追加、削除、または置換（SET）を行う。

シグネチャまたは形状

static void AlterPublicationSchemas(AlterPublicationStmt *stmt, HeapTuple tup, List *schemaidlist)

入力と出力

  • **入力**:
  • stmt: 実行する ALTER PUBLICATION 文の構文解析ツリー。
  • tup: 対象パブリケーションの pg_publication カタログタプル。
  • schemaidlist: 操作対象となるスキーマのOIDリスト。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • schemaidlist が空で、かつアクションが AP_SetObjects 以外の場合、処理は即座に終了する。
  • 操作中、対象スキーマに対して LockSchemaList によるロックが保持され、同時実行によるスキーマ削除を防ぐ。

アルゴリズムまたは振る舞い

1. **ロックの取得**: LockSchemaList を呼び出し、対象スキーマを保護する。 2. **アクション別の分岐**:

  • AP_AddObjects:
  • 既存のパブリケーションに含まれるテーブルを走査し、列リスト（column list）が指定されているテーブルが存在する場合、スキーマの追加を禁止する（ERRCODE_INVALID_PARAMETER_VALUE を送出）。
  • PublicationAddSchemas を呼び出し、スキーマを追加する。
  • AP_DropObjects:
  • PublicationDropSchemas を呼び出し、指定されたスキーマを削除する。
  • AP_SetObjects:
  • 現在のパブリケーションに紐付くスキーマと schemaidlist の差分を計算し、不要なスキーマを削除（PublicationDropSchemas）した後、新しいスキーマリストを追加（PublicationAddSchemas）する。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の AlterPublication（推論されるディスパッチャ）。
  • **呼び出し先**:
  • LockSchemaList, GetPublicationRelations, SearchSysCache2, PublicationAddSchemas, PublicationDropSchemas, GetPublicationSchemas, list_difference_oid。

関連データ構造

  • AlterPublicationStmt: SQL文の解析結果を保持する構造体。
  • Form_pg_publication: pg_publication カタログの行データ。
  • PUBLICATIONRELMAP: パブリケーションとリレーションの関連付けを検索するためのシステムキャッシュ。

失敗モードと境界条件

  • **列リストとの競合**: パブリケーションに列リスト付きのテーブルが既に含まれている場合、スキーマの追加はエラーとなる。
  • **同時実行**: スキーマが削除される前にロックを取得することで整合性を維持する。

保守担当者向けメモ

  • 本関数は AP_SetObjects 時に既存スキーマとの差分を計算するが、追加分についてはカタログ更新時に重複をスキップするロジックとなっている。
  • PublicationAddSchemas および PublicationDropSchemas の呼び出し時に渡されるフラグ（false または true）は、カタログ更新の挙動を制御する。
