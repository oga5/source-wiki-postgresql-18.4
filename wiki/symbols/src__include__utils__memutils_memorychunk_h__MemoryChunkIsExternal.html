MemoryChunkIsExternal

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/memutils_memorychunk.h:203

目的

指定された MemoryChunk が「外部管理モード（External）」であるかどうかを判定します。

シグネチャまたは形状

static inline bool MemoryChunkIsExternal(MemoryChunk *chunk)

入力と出力

  • **入力**: MemoryChunk *chunk - 判定対象のメモリチャンクへのポインタ。
  • **出力**: bool - 外部管理モードであれば true、そうでなければ false を返します。

前提条件と不変条件

  • 外部管理モードのチャンクは、hdrmask の上位ビットに MEMORYCHUNK_MAGIC を保持している必要があります。
  • 本関数は、Assert を通じて hdrmask の整合性（外部フラグが立っている場合はマジックナンバーが正しいこと）を検証します。

アルゴリズムまたは振る舞い

1. chunk->hdrmask に対して HdrMaskIsExternal マクロを適用し、外部フラグの状態を確認します。 2. デバッグビルド時、外部フラグが立っている場合は HdrMaskCheckMagic を用いてマジックナンバーの妥当性を検証します。 3. 最終的に HdrMaskIsExternal の評価結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/alignedalloc.c
  • src/backend/utils/mmgr/aset.c
  • src/backend/utils/mmgr/bump.c
  • src/backend/utils/mmgr/generation.c
  • **呼び出し先**:
  • HdrMaskIsExternal (マクロ)
  • HdrMaskCheckMagic (マクロ、デバッグ時のみ)

関連データ構造

  • MemoryChunk: メモリチャンクのヘッダー構造体。
  • hdrmask: チャンクのメタデータを保持する 64 ビットフィールド。

失敗モードと境界条件

  • chunk が NULL の場合の挙動は定義されていません。
  • hdrmask が破損している場合、Assert によりプロセスが停止する可能性があります。

保守担当者向けメモ

  • 本関数は MemoryChunk のメタデータ解釈の基盤です。hdrmask のビットレイアウトを変更する場合は、HdrMaskIsExternal および HdrMaskCheckMagic の定義と整合性を取る必要があります。
