GetIntoRelEFlags

定義

  • 種別: function
  • 定義箇所: src/backend/commands/createas.c:374

目的

CREATE TABLE AS (CTAS) コマンドの実行時に、エグゼキュータへ渡すべき実行フラグを計算する。

シグネチャまたは形状

int GetIntoRelEFlags(IntoClause *intoClause)

入力と出力

  • **入力**: IntoClause *intoClause - CTAS コマンドの構文解析結果を含む構造体。
  • **出力**: int - エグゼキュータの動作を制御するビットフラグ。

前提条件と不変条件

  • intoClause は非 NULL である必要がある。

アルゴリズムまたは振る舞い

1. 初期フラグ値 0 を設定する。 2. intoClause->skipData が真（WITH NO DATA 指定時）であれば、EXEC_FLAG_WITH_NO_DATA フラグをビット単位の OR で追加する。 3. 計算されたフラグ値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/createas.c: ExecutorStart 呼び出し時に使用。
  • src/backend/commands/explain.c: EXPLAIN コマンド実行時に使用。
  • src/backend/commands/prepare.c: プリペアドステートメントの実行準備時に使用。
  • **呼び出し先**:
  • 提供されたソース excerpt 内には呼び出し先となる関数は含まれていない。

関連データ構造

  • IntoClause: src/nodes/parsenodes.h で定義される、INTO 句の情報を保持する構造体。
  • EXEC_FLAG_WITH_NO_DATA: エグゼキュータの動作を制御する定数。

失敗モードと境界条件

  • intoClause が NULL の場合、ポインタ参照エラーが発生する可能性がある（ソースコード上での明示的なチェックは存在しない）。

保守担当者向けメモ

  • 本関数は WITH NO DATA オプションの伝播に特化している。将来的に CTAS の実行フラグが増加した場合は、この関数を介してフラグの集約を行うこと。
