ItemPointerSetInvalid

定義

  • 種別: function
  • 定義箇所: src/include/storage/itemptr.h:183

目的

指定された ItemPointerData 構造体を「無効（Invalid）」な状態に初期化する。

シグネチャまたは形状

static inline void ItemPointerSetInvalid(ItemPointerData *pointer)

入力と出力

  • **入力**: pointer (ItemPointerData *) - 無効化対象のアイテムポインタへのポインタ。
  • **出力**: なし（副作用として入力ポインタの値を更新する）。

前提条件と不変条件

  • pointer は有効なメモリ領域を指している必要がある（PointerIsValid(pointer) アサーションによる検証）。

アルゴリズムまたは振る舞い

1. pointer が指すメモリ領域が有効かを確認する。 2. BlockIdSet を使用して、ブロック番号を InvalidBlockNumber に設定する。 3. ip_posid を InvalidOffsetNumber に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • contrib/hstore/hstore_io.c
  • src/backend/access/brin/brin_revmap.c
  • src/backend/access/brin/brin_xlog.c
  • src/backend/access/common/heaptuple.c
  • src/backend/access/gin/gindatapage.c
  • **呼び出し先**:
  • PointerIsValid (マクロ)
  • BlockIdSet (マクロ)

関連データ構造

  • ItemPointerData: タプル位置を保持する 6 バイトの構造体。
  • InvalidBlockNumber: 無効なブロック番号を示す定数。
  • InvalidOffsetNumber: 無効なオフセット番号を示す定数。

失敗モードと境界条件

  • pointer が NULL の場合、Assert が有効なビルドではプログラムが停止する。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスが重視されるパスで使用される。
  • ItemPointerData のメモリレイアウト（6バイト）に依存しているため、構造体定義を変更する際は本関数への影響を考慮すること。
