compare_path_costs

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/pathnode.c:71

目的

2つの Path ノードのコストを比較し、どちらが優れているか（あるいは等価か）を判定する。

シグネチャまたは形状

int compare_path_costs(Path *path1, Path *path2, CostSelector criterion)

入力と出力

  • **入力**:
  • path1, path2: 比較対象の Path 構造体へのポインタ。
  • criterion: 比較の優先順位を指定する CostSelector（STARTUP_COST またはそれ以外）。
  • **出力**:
  • -1: path1 が path2 より優れている場合。
  • +1: path2 が path1 より優れている場合。
  • 0: 両者が等価である場合。

前提条件と不変条件

  • path1 および path2 は有効な Path 構造体を指している必要がある。
  • disabled_nodes の値が異なる場合、その値が小さい方が常に優先される。

アルゴリズムまたは振る舞い

1. **無効ノード数の比較**: disabled_nodes を比較し、値が小さい方を優先する。 2. **コスト比較**:

  • criterion が STARTUP_COST の場合:
  • まず startup_cost を比較し、差があれば結果を返す。
  • startup_cost が等しい場合、total_cost を比較して結果を返す。
  • criterion がそれ以外（TOTAL_COST 等）の場合:
  • まず total_cost を比較し、差があれば結果を返す。
  • total_cost が等しい場合、startup_cost を比較して結果を返す。

    • 3. すべての比較項目が等しい場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/joinpath.c
  • src/backend/optimizer/path/pathkeys.c
  • src/backend/optimizer/util/pathnode.c（内部の複数箇所）
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、Path 構造体のメンバ（disabled_nodes, startup_cost, total_cost）を直接参照する。

関連データ構造

  • Path: コスト情報（startup_cost, total_cost）および無効ノード数（disabled_nodes）を保持する構造体。
  • CostSelector: 比較基準を定義する列挙型。

失敗モードと境界条件

  • path1 または path2 が NULL の場合の挙動は定義されていない（セグメンテーション違反の可能性がある）。
  • コスト値が完全に一致する場合、0 を返して「等価」とみなす。浮動小数点誤差を考慮した比較が必要な場合は、compare_path_costs_fuzzily を使用すること。

保守担当者向けメモ

  • 本関数は厳密な比較を行う。プランナの安定性のために曖昧さが必要な箇所では、compare_path_costs_fuzzily の使用を検討すること。
  • disabled_nodes の比較が最優先されるため、このロジックを変更する際は注意が必要である。
