FileBackupMethod

定義

  • 種別: enum
  • 定義箇所: src/include/backup/basebackup_incremental.h:22

目的

PostgreSQLの増分バックアップ処理において、対象ファイルを「完全転送」するか「増分転送」するかを区別するために使用される列挙型です。

シグネチャまたは形状

typedef enum
{
	BACK_UP_FILE_FULLY,
	BACK_UP_FILE_INCREMENTALLY
} FileBackupMethod;

入力と出力

  • 本シンボル自体は列挙型であり、関数ではありません。
  • GetFileBackupMethod() 関数の戻り値として使用され、バックアップ実行時の転送戦略を決定します。

前提条件と不変条件

  • BACK_UP_FILE_FULLY: ファイル全体をバックアップ対象とします。
  • BACK_UP_FILE_INCREMENTALLY: 前回のバックアップ以降に変更されたブロックのみをバックアップ対象とします。

アルゴリズムまたは振る舞い

バックアップ実行中、GetFileBackupMethod() が各ファイルの状態を評価し、この列挙値のいずれかを返します。呼び出し側はこの値に基づき、ファイル転送のロジックを切り替えます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • src/backend/backup/basebackup.c:1482
  • src/include/backup/basebackup_incremental.h:44 (GetFileBackupMethod の戻り値型として使用)
  • 呼び出し先:
  • 列挙型であるため、関数としての呼び出し先はありません。

関連データ構造

  • IncrementalBackupInfo: 増分バックアップのメタデータを保持する構造体。GetFileBackupMethod() の引数として使用されます。

失敗モードと境界条件

  • 該当する情報はありません。

保守担当者向けメモ

  • 本列挙型は basebackup_incremental.c で実装される判定ロジックの根幹を成します。新しいバックアップ手法を追加する場合は、この列挙型を拡張する必要があります。
