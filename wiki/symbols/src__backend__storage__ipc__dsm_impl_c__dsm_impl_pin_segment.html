dsm_impl_pin_segment

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm_impl.c:962

目的

動的共有メモリ（DSM）セグメントの生存期間を延長し、Postmasterプロセスにおいてセグメントハンドルを保持するための補助関数です。

シグネチャまたは形状

void dsm_impl_pin_segment(dsm_handle handle, void *impl_private, void **impl_private_pm_handle)

入力と出力

  • handle: 操作対象のDSMセグメント識別子。
  • impl_private: 実装固有のセグメント情報（ハンドル等）。
  • impl_private_pm_handle: 出力用。Postmasterプロセス内で複製されたハンドルを格納するポインタ。

前提条件と不変条件

  • dynamic_shared_memory_type に基づいて動作が分岐します。
  • Windows環境（USE_DSM_WINDOWS）において、IsUnderPostmaster が真である場合にのみハンドル複製処理が実行されます。

アルゴリズムまたは振る舞い

1. dynamic_shared_memory_type を確認します。 2. Windows環境かつPostmaster配下の場合:

  • DuplicateHandle を使用し、現在のプロセスから PostmasterHandle へセグメントハンドルを複製します。
  • 複製に失敗した場合はエラー（ereport(ERROR, ...)）を発生させます。
  • 複製したハンドルを *impl_private_pm_handle に格納します。このハンドルは、後に dsm_impl_unpin_segment が呼び出された際にクローズするために使用されます。

    • 3. その他の実装方式では、特段の処理を行いません。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/dsm.c
  • 呼び出し先: DuplicateHandle（Windows API）、ereport（PostgreSQLエラー報告）

関連データ構造

  • dsm_handle: DSMセグメントを識別する型。
  • dynamic_shared_memory_type: 現在有効なDSM実装方式を保持するGUC変数。

失敗モードと境界条件

  • Windows環境において DuplicateHandle が失敗した場合、GetLastError() を用いてエラー情報を取得し、ereport によりプロセスを終了させます。

保守担当者向けメモ

  • 本関数はWindows特有のカーネルオブジェクト生存期間管理を補完するためのものです。
  • 複製されたハンドルはPostmasterプロセス内でのみ有効であり、他のプロセスでは使用できません。
  • dsm_impl_unpin_segment と対になる設計であるため、修正時は両方の整合性を確認してください。
