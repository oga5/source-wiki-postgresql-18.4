ismn_in

定義

  • 種別: function
  • 定義箇所: contrib/isn/isn.c:1016

目的

入力された文字列を ISMN（国際標準音楽番号）として解析し、PostgreSQL 内部の ean13 型へ変換する。

シグネチャまたは形状

Datum ismn_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して受け取る文字列（const char *str）。
  • **出力**: 成功時は ean13 型の値を PG_RETURN_EAN13 マクロで返し、失敗時は PG_RETURN_NULL() を返す。

前提条件と不変条件

  • string2ean 関数が ISMN 形式の検証および変換を担う。
  • ean13 型の最下位ビットは、チェックデジットが不正な状態で入力されたことを示すフラグとして予約されている。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) で入力文字列を取得する。 2. string2ean 関数を呼び出し、文字列を ISMN 型として解析・検証する。 3. 解析に失敗した場合は NULL を返す。 4. 解析に成功した場合は、変換された ean13 値を返す。

呼び出し元と呼び出し先

  • **呼び出し先**: string2ean（contrib/isn/isn.c 内で定義）。
  • **呼び出し元**: PostgreSQL の型入力関数として、システムカタログ経由で呼び出される（直接的なコード上の呼び出し元はソースコード内に明示されていない）。

関連データ構造

  • ean13: 内部ストレージ形式（uint64）。
  • ISMN: string2ean に渡される識別子タイプ定数。

失敗モードと境界条件

  • 入力文字列が ISMN の形式（チェックデジット等）を満たさない場合、string2ean が false を返し、本関数は NULL を返す。
  • isn.weak 設定が有効な場合、チェックデジットの検証が緩和される可能性がある。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(ismn_in) により PostgreSQL の関数として登録されている。
  • ハイフン挿入等のフォーマット処理は string2ean および関連する _range テーブルのロジックに依存している。
