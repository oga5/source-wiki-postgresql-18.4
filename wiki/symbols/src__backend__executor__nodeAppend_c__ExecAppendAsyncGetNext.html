ExecAppendAsyncGetNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:928

目的

Append ノードにおいて、非同期サブプランからタプルを非同期的に取得し、結果を統合する。

シグネチャまたは形状

static bool ExecAppendAsyncGetNext(AppendState *node, TupleTableSlot **result)

入力と出力

  • **入力**:
  • node: AppendState 構造体へのポインタ。実行中の Append ノードの状態を保持。
  • result: TupleTableSlot へのポインタのポインタ。取得したタプルを格納する。
  • **出力**:
  • 戻り値: bool 型。タプルが取得できた場合、または処理が完了した場合は true、それ以外は false。
  • result: 成功時はタプルが格納され、失敗時は NULL が設定される。

前提条件と不変条件

  • node->as_nasyncremain > 0 であること（有効な非同期サブプランが存在しない状態で呼び出してはならない）。
  • 処理中に CHECK_FOR_INTERRUPTS() が実行される。

アルゴリズムまたは振る舞い

1. result を NULL に初期化する。 2. ExecAppendAsyncRequest を呼び出し、即座にタプルが取得可能か確認する。取得できた場合は true を返す。 3. 非同期サブプランが残っている間、以下のループを実行する:

  • CHECK_FOR_INTERRUPTS() を実行。
  • ExecAppendAsyncEventWait を呼び出し、非同期イベントの待機またはポーリングを行う。
  • 再度 ExecAppendAsyncRequest を呼び出し、タプルが取得できた場合は true を返す。

    • 4. 全ての同期サブプランが完了している場合、as_nasyncremain が 0 であることを確認し、result をクリアして true を返す。 5. 上記以外の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAppend.c 内の ExecAppend から呼び出される（ソース内の直接的な参照情報に基づく）。
  • **呼び出し先**:
  • ExecAppendAsyncRequest
  • ExecAppendAsyncEventWait
  • ExecClearTuple
  • CHECK_FOR_INTERRUPTS

関連データ構造

  • AppendState: Append ノードの実行状態を管理する構造体。as_nasyncremain（残りの非同期サブプラン数）や as_syncdone（同期サブプランの完了フラグ）を保持する。
  • TupleTableSlot: タプルを保持するためのスロット。

失敗モードと境界条件

  • 非同期サブプランが存在しない状態で呼び出された場合、Assert により停止する。
  • 同期サブプランと非同期サブプランの混在状況により、戻り値の bool が制御される。

保守担当者向けメモ

  • 本関数は postgres_fdw 等の非同期対応データラッパーと連携する。
  • 非同期処理の待機には WaitEventSet が利用されるため、イベントループの挙動を変更する際は ExecAppendAsyncEventWait の実装を確認すること。
