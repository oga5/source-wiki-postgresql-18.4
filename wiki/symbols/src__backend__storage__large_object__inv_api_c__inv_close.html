inv_close

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:298

目的

オープンされているラージオブジェクトの記述子（LargeObjectDesc）を解放し、リソースのクリーンアップを行います。

シグネチャまたは形状

void inv_close(LargeObjectDesc *obj_desc)

入力と出力

  • **入力**: obj_desc - 解放対象となる LargeObjectDesc 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • obj_desc は有効なポインタである必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. PointerIsValid(obj_desc) により、渡されたポインタが有効であることを確認します。 2. pfree(obj_desc) を呼び出し、メモリコンテキストから当該記述子を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-fsstubs.c 内の各関数（lo_close, lo_read, lo_write 等のバックエンド実装）から複数回参照されています。
  • **呼び出し先**:
  • pfree（メモリ解放関数）

関連データ構造

  • LargeObjectDesc: ラージオブジェクトのオープン状態を保持する構造体。

失敗モードと境界条件

  • obj_desc が NULL または無効なポインタである場合、Assert が有効なビルド環境ではアボートします。

保守担当者向けメモ

  • 本関数はメモリの解放のみを行います。ラージオブジェクトに関連するリレーションのクローズ処理（close_lo_relation 等）は、本関数の呼び出し元または別のライフサイクル管理プロセスで行う必要があります。
