AssertCheckExpandedRanges

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_minmax_multi.c:425

目的

ExpandedRange 配列の整合性を検証するためのデバッグ用アサーション関数です。USE_ASSERT_CHECKING が有効な場合にのみ実行され、各範囲の妥当性と範囲間の順序・非重複性を保証します。

シグネチャまたは形状

static void AssertCheckExpandedRanges(BrinDesc *bdesc, Oid colloid, AttrNumber attno, Form_pg_attribute attr, ExpandedRange *ranges, int nranges)

入力と出力

  • **入力**:
  • bdesc: BRIN インデックスの記述子。
  • colloid: 照合順序（Collation）ID。
  • attno: 属性番号。
  • attr: 属性の定義情報。
  • ranges: 検証対象の ExpandedRange 配列。
  • nranges: 配列内の要素数。
  • **出力**: なし（検証失敗時は Assert によりプログラムが停止）。

前提条件と不変条件

  • 各 ExpandedRange 要素において、collapsed が真の場合は minval == maxval であること。
  • 各 ExpandedRange 要素において、collapsed が偽の場合は minval < maxval であること。
  • 配列内のすべての範囲は、昇順にソートされ、かつ互いに重なりがないこと（先行する範囲の maxval < 後続の範囲の minval）。

アルゴリズムまたは振る舞い

1. minmax_multi_get_strategy_procinfo を使用し、対象属性の等価（BTEqualStrategyNumber）および小なり（BTLessStrategyNumber）比較関数を取得します。 2. 配列をループし、各範囲内で minval と maxval の関係が collapsed フラグと一致するかを検証します。 3. 隣接する範囲をループし、先行する範囲の maxval が後続の範囲の minval より小さいことを検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin_minmax_multi.c 内の複数箇所（行 1662, 1680, 2812, 2841）。
  • **呼び出し先**: minmax_multi_get_strategy_procinfo、FunctionCall2Coll。

関連データ構造

  • ExpandedRange: 範囲の境界値と状態を保持する構造体。
  • BrinDesc: BRIN インデックスのメタデータ。

失敗モードと境界条件

  • USE_ASSERT_CHECKING が定義されていない場合、本関数は何も行いません。
  • 比較関数が期待通りに動作しない場合や、データ構造が不変条件を満たさない場合に Assert がトリガーされます。

保守担当者向けメモ

  • 本関数はデバッグ専用です。本番環境のパフォーマンスには影響しません。
  • 比較関数（eq, lt）の取得には minmax_multi_get_strategy_procinfo を使用しており、インデックスの演算クラス定義に依存しています。
