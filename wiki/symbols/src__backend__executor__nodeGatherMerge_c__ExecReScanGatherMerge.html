ExecReScanGatherMerge

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:340

目的

GatherMerge ノードの再スキャン（Rescan）処理を実行し、並列ワーカーの停止、メモリの解放、および状態フラグのリセットを行う。

シグネチャまたは形状

void ExecReScanGatherMerge(GatherMergeState *node)

入力と出力

  • **入力**: GatherMergeState *node - 再スキャン対象の GatherMerge 実行状態構造体。
  • **出力**: なし（副作用として状態を更新）。

前提条件と不変条件

  • node は有効な GatherMergeState を指している必要がある。
  • 本関数呼び出しにより、次回の実行時に共有状態が再構築されるよう initialized および gm_initialized が false に設定される。

アルゴリズムまたは振る舞い

1. **ワーカーの停止**: ExecShutdownGatherMergeWorkers を呼び出し、既存の並列ワーカーを安全に終了させる。 2. **メモリ解放**: gather_merge_clear_tuples を呼び出し、未使用のタプルを解放してメモリリークを防ぐ。 3. **状態リセット**: node->initialized および node->gm_initialized を false に設定する。 4. **パラメータ更新**: rescan_param が設定されている場合、子ノードの chgParam にパラメータを追加し、リーダープロセスにおける行セットの変更を通知する。 5. **子ノードの再スキャン**: outerPlan->chgParam が空の場合のみ、ExecReScan を呼び出して子ノードの再スキャンを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:178
  • **呼び出し先**:
  • ExecShutdownGatherMergeWorkers
  • gather_merge_clear_tuples
  • ExecReScan (条件付き)

関連データ構造

  • GatherMergeState: 実行状態を保持する構造体。
  • GatherMerge: プランノード構造体。
  • PlanState: 子ノードの実行状態。

失敗モードと境界条件

  • rescan_param が設定されている場合、子ノードの ReScan は即座には実行されず、最初の ExecProcNode 呼び出し時に行われる。
  • 並列ワーカーの終了処理は、共有状態の再構築とローカル状態のリセットが分離されているため、呼び出し順序に依存する可能性がある。

保守担当者向けメモ

  • 本関数は GatherMerge の再スキャンを管理する。ReInitializeDSM（共有状態のリセット）と ReScan（ローカル状態のリセット）の責務分離に注意すること。
  • 並列ワーカーの終了とタプルバッファのクリアは、メモリリーク防止のために必須である。
