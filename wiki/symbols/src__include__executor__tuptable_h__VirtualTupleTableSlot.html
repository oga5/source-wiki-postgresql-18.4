VirtualTupleTableSlot

定義

  • 種別: struct
  • 定義箇所: src/include/executor/tuptable.h:248

目的

実行エンジンにおいて、物理的なタプルコピーを回避し、メモリ上の仮想的なタプルを保持するためのスロット構造体です。

シグネチャまたは形状

typedef struct VirtualTupleTableSlot
{
	pg_node_attr(abstract)

	TupleTableSlot base;

	char	   *data;			/* data for materialized slots */
} VirtualTupleTableSlot;

入力と出力

  • 入力: TupleTableSlot を基底として構成されます。
  • 出力: data フィールドにより、スロットがマテリアライズ（実体化）された際のデータを保持します。

前提条件と不変条件

  • TupleTableSlot を継承しており、実行エンジンのスロット操作インターフェースと互換性があります。
  • 仮想タプルとして運用される場合、下位ノードのメモリ領域を参照する可能性があるため、生存期間の管理が重要です。

アルゴリズムまたは振る舞い

  • 物理的なタプルを保持せず、必要に応じて属性値を抽出する遅延評価をサポートします。
  • ExecMaterializeSlot 等の操作を通じて、必要に応じて data フィールドにタプルを実体化（コピー）します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execTuples.c 内の TupleTableSlotOps 実装およびスロット操作関数から参照されます。
  • 呼び出し先: 具体的な関数呼び出しの連鎖については、提供されたソース範囲内では明示されていません。

関連データ構造

  • TupleTableSlot: 基底構造体。
  • TupleTableSlotOps: このスロット型の挙動を定義する関数ポインタテーブル。

失敗モードと境界条件

  • 仮想タプルとして使用する場合、参照先のメモリ領域が解放された後にアクセスすると不正なメモリ参照が発生します。

保守担当者向けメモ

  • src/backend/executor/execTuples.c にて TupleTableSlotOps の実装が定義されています。
  • base_slot_size として sizeof(VirtualTupleTableSlot) が使用されています。
