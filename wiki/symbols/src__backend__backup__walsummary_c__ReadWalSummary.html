ReadWalSummary

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummary.c:272

目的

WalSummaryIO 構造体を通じて、WAL要約ファイルからデータを読み取るための低レベルI/Oコールバック関数です。

シグネチャまたは形状

int ReadWalSummary(void *wal_summary_io, void *data, int length);

入力と出力

  • **入力**:
  • wal_summary_io: WalSummaryIO 構造体へのポインタ（void * として渡される）。
  • data: 読み取ったデータを格納するバッファ。
  • length: 読み取るバイト数。
  • **出力**:
  • 戻り値: 実際に読み取ったバイト数。

前提条件と不変条件

  • wal_summary_io は有効な WalSummaryIO 構造体を指している必要があります。
  • io->file はオープン済みのファイル記述子である必要があります。
  • 読み取り操作のたびに io->filepos が読み取ったバイト数分だけ更新されます。

アルゴリズムまたは振る舞い

1. FileRead を呼び出し、io->file から io->filepos を開始位置として length 分のデータを読み取ります。この際、WAIT_EVENT_WAL_SUMMARY_READ を待機イベントとして使用します。 2. 読み取り結果が負（エラー）の場合、ereport(ERROR, ...) を発行して処理を中断します。 3. 正常に読み取れた場合、io->filepos を読み取ったバイト数分進めます。 4. 読み取ったバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c:587 (CreateBlockRefTableReader への引数として使用)
  • src/backend/backup/walsummaryfuncs.c:103 (CreateBlockRefTableReader への引数として使用)
  • **呼び出し先**:
  • FileRead (PostgreSQLのファイルI/O抽象化レイヤー)
  • FilePathName (エラーメッセージ生成用)
  • ereport (エラー通知用)

関連データ構造

  • WalSummaryIO: ファイル記述子 (file) と現在の読み取り位置 (filepos) を保持する構造体。

失敗モードと境界条件

  • FileRead が負の値を返した場合、ファイル読み取りエラーとして ERROR レベルの例外が発生します。

保守担当者向けメモ

  • 本関数は BlockRefTableReader が使用するコールバックインターフェースに適合するように設計されています。
  • 読み取り位置の管理は呼び出し側ではなく本関数内で行われるため、連続した読み取り操作において io->filepos が正しく維持されることが重要です。
