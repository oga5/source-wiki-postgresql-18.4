WORKSTATE

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int_bool.c:32

目的

クエリ文字列の構文解析およびポーランド記法への変換プロセスにおいて、解析状態、トークンリスト、および一時的な作業領域を保持するために使用されるコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    char       *buf;        /* 解析対象のクエリ文字列バッファ */
    int32       state;      /* 現在の解析状態 */
    int32       count;      /* 処理中のカウント */
    struct Node *escontext; /* エラー報告用コンテキスト */
    NODE       *str;        /* 一時的な逆ポーランド記法リスト */
    int32       num;        /* str 内の要素数 */
} WORKSTATE;

入力と出力

本構造体は関数間で受け渡される状態保持オブジェクトとして機能します。

  • **入力**: gettoken 等の関数により、buf からトークンが読み取られ、str（NODE リスト）に蓄積されます。
  • **出力**: 最終的に makepol 等の処理を経て、クエリツリー構築のためのデータソースとなります。

前提条件と不変条件

  • str フィールドは、クエリ解析中の一時的なリスト構造を指す必要があります。
  • num は str に格納されたノード数と整合している必要があります。

アルゴリズムまたは振る舞い

クエリ解析のパイプラインにおいて、以下の役割を担います。 1. **トークン化**: gettoken が buf を走査し、state を更新しながらトークンを抽出します。 2. **リスト構築**: pushquery が str リストに演算子やオペランドを追加します。 3. **ツリー変換**: makepol が str リストを走査し、findoprnd を用いて演算子のオペランド位置を解決します。

呼び出し元と呼び出し先

  • **呼び出し元**: _int_bool.c 内の makepol、gettoken、pushquery、findoprnd 等の関数で引数として使用されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出す側（bqarr_in 等）でインスタンス化されます。

関連データ構造

  • NODE: str フィールドが保持するリストの要素型。
  • ITEM: makepol による変換後の最終的なクエリツリー形式。

失敗モードと境界条件

  • buf が不正な形式の場合、gettoken が解析エラーを発生させる可能性があります。
  • 再帰的な処理が含まれるため、スタック深さの制限に抵触する可能性があります（check_stack_depth() による保護が前提）。

保守担当者向けメモ

  • 本構造体は _int_bool.c 内のクエリ解析ロジックに密結合しています。フィールドの追加や変更を行う際は、makepol および findoprnd におけるリスト操作ロジックへの影響を考慮してください。
