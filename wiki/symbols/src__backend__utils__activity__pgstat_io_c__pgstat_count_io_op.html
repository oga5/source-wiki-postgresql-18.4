pgstat_count_io_op

定義

  • src/backend/utils/activity/pgstat_io.c:67

目的

バックエンドプロセスにおけるI/O操作の回数およびバイト数を、ローカルの統計蓄積領域（PendingIOStats）に記録します。

シグネチャまたは形状

void pgstat_count_io_op(IOObject io_object, IOContext io_context, IOOp io_op, uint32 cnt, uint64 bytes)

入力と出力

  • **入力**:
  • io_object: 対象のI/Oオブジェクト（テーブル、インデックス等）。
  • io_context: I/Oが発生したコンテキスト。
  • io_op: 実行されたI/O操作の種類。
  • cnt: 加算する操作回数。
  • bytes: 加算するデータ量（バイト単位）。
  • **出力**: なし（副作用として内部状態を更新）。

前提条件と不変条件

  • io_object は IOOBJECT_NUM_TYPES 未満であること。
  • io_context は IOCONTEXT_NUM_TYPES 未満であること。
  • io_op がバイト数を追跡する種類でない場合、bytes は 0 であること。
  • pgstat_tracks_io_op により、現在のバックエンドタイプにおいて当該操作が統計対象として妥当であると判定されること。

アルゴリズムまたは振る舞い

1. 引数の妥当性を Assert で検証します。 2. 静的変数 PendingIOStats の該当するカウンタ（回数およびバイト数）に値を加算します。 3. pgstat_count_backend_io_op を呼び出し、バックエンドごとの統計を更新します。 4. 統計が更新されたことを示すフラグ have_iostats および pgstat_report_fixed を true に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c（バッファヒットやリレーションI/O発生時）
  • src/backend/storage/buffer/localbuf.c（一時リレーションの退避時）
  • src/backend/utils/activity/pgstat_io.c（内部的な再帰呼び出し）
  • **呼び出し先**:
  • pgstat_count_backend_io_op（バックエンドごとの統計更新）
  • pgstat_is_ioop_tracked_in_bytes（検証用）
  • pgstat_tracks_io_op（検証用）

関連データ構造

  • PendingIOStats: バックエンドローカルで未フラッシュの統計を保持する構造体。

失敗モードと境界条件

  • 妥当性検証（Assert）に失敗した場合、デバッグビルドではプロセスが停止します。
  • 統計対象外の操作に対して誤った値を渡した場合、Assert により捕捉されます。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、オーバーヘッドを最小限に抑える設計となっています。
  • 新しいI/O操作やバックエンドタイプを追加する際は、pgstat_tracks_io_op の判定ロジックが適切に更新されているか確認してください。
