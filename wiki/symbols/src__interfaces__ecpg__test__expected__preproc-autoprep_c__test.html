目的

ECPG（Embedded SQL for C）における「自動プリペア（Automatic Prepare）」機能の動作を検証する回帰テスト関数。SQL文が明示的にプリペアされていない場合でも、ECPGライブラリが透過的にプリペア処理を行う挙動を確認する。

シグネチャまたは形状

static void test(void)

入力と出力

  • 入力: なし（関数内で宣言されたローカル変数およびデータベース接続を使用）
  • 出力: なし（標準出力および標準エラー出力へのログ出力、およびデータベース操作の実行）

前提条件と不変条件

  • データベース ecpg1_regression への接続が確立可能であること。
  • 実行環境において ECPGdebug(1, stderr) により詳細なデバッグログが標準エラー出力へ出力される。
  • sqlca 構造体を使用して各SQL実行後のエラーおよび警告を監視している。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGconnect でデータベースに接続。 2. **テーブル作成**: ECPGst_normal モードで create table T を実行。 3. **自動プリペア検証**: ECPGst_prepnormal モードを使用して INSERT 文を複数回実行し、ライブラリによる透過的なプリペア処理を検証。 4. **明示的プリペア検証**: ECPGprepare でステートメントを定義し、ECPGst_execute で実行。 5. **データ取得**: SELECT 文を ECPGst_prepnormal で実行し、結果を配列に格納して標準出力へ表示。 6. **カーソル操作**: DECLARE CURSOR および FETCH を使用した動的SQL操作の検証。 7. **終了処理**: CLOSE 文によるカーソル終了。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内では明示されていません。
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGprepare, ECPGprepared_statement, sqlprint, printf。

関連データ構造

  • sqlca: SQL通信領域（エラーコードおよび警告フラグの保持）。
  • item, ind: 取得結果およびインジケータを保持する配列。

失敗モードと境界条件

  • 各SQL実行後に sqlca.sqlcode < 0 をチェックし、エラー発生時は sqlprint() を呼び出して終了する。
  • sqlca.sqlwarn[0] == 'W' の場合、警告を出力する。
  • while ループによるカーソルフェッチにおいて、データが存在しない場合は break によりループを抜ける。

保守担当者向けメモ

  • 本関数はプリプロセッサが生成したコードであるため、手動編集は推奨されません。
  • test() 関数が複数回呼び出されることを想定した設計となっており、再実行時のキャッシュ整合性を確認する役割を持ちます。
