ComputeIndexAttrs

定義

  • 種別: function
  • 定義箇所: src/backend/commands/indexcmds.c:1871

目的

インデックス定義（IndexElem リスト）を解析し、インデックスの各列、式、および排他制約に必要な情報を計算して IndexInfo 構造体へ格納する。

シグネチャまたは形状

static void ComputeIndexAttrs(IndexInfo *indexInfo, Oid *typeOids, Oid *collationOids, Oid *opclassOids, Datum *opclassOptions, int16 *colOptions, const List *attList, const List *exclusionOpNames, Oid relId, const char *accessMethodName, Oid accessMethodId, bool amcanorder, bool isconstraint, bool iswithoutoverlaps, Oid ddl_userid, int ddl_sec_context, int *ddl_save_nestlevel)

入力と出力

  • **入力**:
  • indexInfo: 構築対象のインデックス情報。
  • attList: インデックス対象列または式のリスト (IndexElem)。
  • relId: 対象テーブルの OID。
  • exclusionOpNames: 排他演算子名のリスト（存在する場合）。
  • その他: アクセスメソッド情報、権限コンテキスト、制約フラグ等。
  • **出力**:
  • indexInfo の各フィールド（属性番号、式リスト、排他演算子情報等）を更新。
  • typeOids, collationOids, opclassOids 等の配列へ計算結果を書き込み。

前提条件と不変条件

  • exclusionOpNames が提供される場合、その長さは indexInfo->ii_NumIndexKeyAttrs と一致しなければならない。
  • インデックス式に含まれる関数は IMMUTABLE でなければならない。
  • インデックスに含まれる列（Included columns）には、照合順序、演算子クラス、ソート順序の指定は不可。

アルゴリズムまたは振る舞い

1. **排他制約の準備**: exclusionOpNames または iswithoutoverlaps が指定されている場合、IndexInfo 内の排他演算子用配列を確保する。 2. **権限昇格**: ddl_userid が有効な場合、GetUserIdAndSecContext を呼び出し現在のセキュリティコンテキストを保存する。 3. **属性リストの走査**: attList を順次処理する。

  • **単純な列**: SearchSysCacheAttName で属性情報を取得し、属性番号を記録。
  • **式**: exprType および exprCollation で型と照合順序を特定。式が可変関数を含む場合はエラーとする。

    • 4. **検証**: インデックスのキー列数を超えた位置にある「付加列（Included columns）」に対して、制約違反（照合順序や演算子クラスの指定など）がないかチェックする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/indexcmds.c 内の DefineIndex 等から複数回参照されている。
  • **呼び出し先**: SearchSysCacheAttName, GetUserIdAndSecContext, exprType, exprCollation, contain_mutable_functions_after_planning 等のユーティリティ関数を呼び出す。

関連データ構造

  • IndexInfo: インデックス構築に必要なメタデータを保持する主要構造体。
  • IndexElem: パーサが生成するインデックス要素の定義。
  • Form_pg_attribute: システムカタログ pg_attribute の行データ。

失敗モードと境界条件

  • **列未定義**: 指定された列名がテーブルに存在しない場合、ERRCODE_UNDEFINED_COLUMN でエラー。
  • **式制約違反**: インデックス式に IMMUTABLE 以外の関数が含まれる場合、ERRCODE_INVALID_OBJECT_DEFINITION でエラー。
  • **サポート外の操作**: 付加列に対して演算子クラスやソート順序を指定した場合、ERRCODE_INVALID_OBJECT_DEFINITION でエラー。
  • **式制限**: インデックス式内でサブクエリやウィンドウ関数を使用した場合、パーサ段階で拒否される前提。

保守担当者向けメモ

  • 本関数は DefineIndex の中核的な解析フェーズを担う。
  • iswithoutoverlaps の処理において、exclusionOpNames が NIL の場合でも配列の確保が必要になる点に注意。
  • 付加列（attn >= nkeycols）のバリデーションロジックは、将来的に新しいインデックス機能が追加される際に拡張が必要となる可能性がある。
