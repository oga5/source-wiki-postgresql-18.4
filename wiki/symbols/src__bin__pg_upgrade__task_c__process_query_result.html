process_query_result

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/task.c:206

目的

データベース接続に対して実行されたクエリの結果を処理し、必要に応じてコールバックを実行して結果リソースを解放する。

シグネチャまたは形状

static void process_query_result(const ClusterInfo *cluster, UpgradeTaskSlot *slot, const UpgradeTask *task)

入力と出力

  • **入力**:
  • cluster: 対象となるデータベースクラスタ情報。
  • slot: クエリを実行した現在のタスクスロット。
  • task: 実行中のタスク定義。
  • **出力**: なし（副作用としてクエリ結果の処理および接続状態の検証を行う）。

前提条件と不変条件

  • slot->conn は有効な libpq 接続であること。
  • slot->step_idx は task->steps の範囲内であること。

アルゴリズムまたは振る舞い

1. PQgetResult を呼び出し、現在の接続からクエリ結果を取得する。 2. 接続状態が CONNECTION_BAD であるか、結果ステータスが PGRES_TUPLES_OK または PGRES_COMMAND_OK 以外である場合、pg_fatal を呼び出して処理を中断する。 3. steps->process_cb が定義されている場合、そのコールバック関数を呼び出し、データベース情報とクエリ結果を渡す。 4. steps->free_result が真であれば、PQclear を呼び出して PGresult を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/task.c 内の process_slot 関数から呼び出される。
  • **呼び出し先**: libpq の PQgetResult, PQstatus, PQresultStatus, PQerrorMessage, PQclear を呼び出す。また、task->steps[slot->step_idx].process_cb に登録されたコールバック関数を実行する。

関連データ構造

  • ClusterInfo: クラスタ全体の情報を保持。
  • UpgradeTaskSlot: 現在のタスク実行状態（接続、ステップインデックス）を保持。
  • UpgradeTask: 実行すべきステップのリストを保持。
  • UpgradeTaskStep: 個別のクエリステップと処理用コールバックを保持。

失敗モードと境界条件

  • 接続が切断されている場合や、クエリ実行が失敗した場合は pg_fatal によりプログラムが終了する。
  • process_cb が NULL の場合、コールバックの実行はスキップされる（主に search_path 設定などの初期ステップで利用）。

保守担当者向けメモ

  • 本関数は libpq の非同期クエリ実行フローの一部として動作する。
  • steps->free_result のフラグ管理は、メモリリークを防ぐために重要である。コールバック側で結果を保持する必要がある場合は、このフラグを適切に設定すること。
