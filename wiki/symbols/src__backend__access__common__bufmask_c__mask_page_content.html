mask_page_content

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/bufmask.c:118

目的

WAL 整合性チェックにおいて、ページ内容を正規化するために使用されます。ページヘッダーを除く領域およびページ内のオフセット情報を MASK_MARKER（通常 0）で上書きし、比較可能な状態にします。

シグネチャまたは形状

void mask_page_content(Page page)

入力と出力

  • **入力**: Page page（操作対象のページへのポインタ）
  • **出力**: なし（引数で渡されたページをインプレースで書き換えます）

前提条件と不変条件

  • 引数 page は有効な Page 構造体である必要があります。
  • 本関数はテスト目的の正規化ツールであり、実稼働時のページ更新には使用できません。

アルゴリズムまたは振る舞い

1. ページヘッダー直後からページ末尾まで（BLCKSZ - SizeOfPageHeaderData）を MASK_MARKER で埋めます。 2. ページヘッダー内の pd_lower フィールドを MASK_MARKER で上書きします。 3. ページヘッダー内の pd_upper フィールドを MASK_MARKER で上書きします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginxlog.c:810
  • src/backend/access/hash/hash_xlog.c:1140
  • **呼び出し先**:
  • memset（標準ライブラリ関数）

関連データ構造

  • Page: ページデータへのポインタ型。
  • PageHeader: ページヘッダー構造体。pd_lower および pd_upper フィールドの操作に使用されます。

失敗モードと境界条件

  • page が NULL の場合、セグメンテーション違反が発生する可能性があります。
  • ページサイズが SizeOfPageHeaderData より小さい場合、memset のサイズ計算でアンダーフローが発生する可能性があります。

保守担当者向けメモ

  • MASK_MARKER は access/bufmask.h で定義されており、通常は 0 です。
  • 本関数はページ構造を初期化状態に近づけるため、比較対象間で値が異なっていても問題ない領域を強制的に一致させる役割を担います。
