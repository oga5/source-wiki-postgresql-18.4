pg_comp_crc32c_choose

定義

  • 種別: function
  • 定義箇所: src/port/pg_crc32c_sse42_choose.c:60

目的

実行時のCPU機能判定に基づき、最適なCRC-32C計算アルゴリズムを動的に選択し、関数ポインタ pg_comp_crc32c を適切な実装へ書き換えるディスパッチャーです。

シグネチャまたは形状

static pg_crc32c pg_comp_crc32c_choose(pg_crc32c crc, const void *data, size_t len);

入力と出力

  • **入力**:
  • crc: 計算の初期値（pg_crc32c 型）。
  • data: 計算対象のデータバッファ。
  • len: データの長さ。
  • **出力**:
  • 計算されたCRC-32C値（pg_crc32c 型）。

前提条件と不変条件

  • 初回呼び出し時にのみ実行されることを想定しています。
  • pg_comp_crc32c 関数ポインタが本関数を指している状態で呼び出される必要があります。
  • cpuid 命令が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: フォールバックとして pg_comp_crc32c_sb8 (Slicing-by-8) を設定します。 2. **CPU判定**: cpuid 命令を実行し、SSE 4.2のサポート状況を確認します。 3. **SSE 4.2対応時**: pg_comp_crc32c を pg_comp_crc32c_sse42 に更新します。 4. **AVX-512拡張判定**: OSXSAVE フラグおよび zmm_regs_available() の結果に基づき、AVX-512の利用可能性を検証します。条件を満たす場合、cpuid (leaf 7) を再実行し、VPCLMULQDQ および AVX512-VL のサポートを確認した上で、pg_comp_crc32c を pg_comp_crc32c_avx512 に更新します。 5. **実行**: 最終的に選択された関数ポインタを呼び出し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pg_comp_crc32c 関数ポインタの初期化時（src/port/pg_crc32c_sse42_choose.c:109）。
  • src/port/pg_crc32c_armv8_choose.c からの参照が確認されていますが、具体的な呼び出しフローは不明です。
  • **呼び出し先**:
  • zmm_regs_available()
  • pg_comp_crc32c_sb8 (フォールバック)
  • pg_comp_crc32c_sse42
  • pg_comp_crc32c_avx512

関連データ構造

  • pg_comp_crc32c: CRC-32C計算を行う関数ポインタ。

失敗モードと境界条件

  • cpuid 命令が利用できない環境では #error によりコンパイルが失敗します。
  • AVX-512の判定において、OSがZMMレジスタを有効化していない場合、ハードウェアが対応していてもAVX-512実装は選択されません。

保守担当者向けメモ

  • 本関数は初回呼び出し時に一度だけ実行されるイディオムです。
  • zmm_regs_available を呼び出す際は、必ず事前に OSXSAVE フラグを確認する順序を守る必要があります。
  • コンパイルオプション（USE_SSE42_CRC32C_WITH_RUNTIME_CHECK 等）により、選択可能なアルゴリズムが制限される場合があります。
