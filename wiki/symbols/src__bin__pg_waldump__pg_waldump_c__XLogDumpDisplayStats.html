XLogDumpDisplayStats

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/pg_waldump.c:625

目的

収集された WAL 統計情報（XLogStats）を整形し、リソースマネージャ（RMGR）ごとのレコード数、レコードサイズ、フルページイメージ（FPI）サイズ、およびそれらの合計値を標準出力に表示する。

シグネチャまたは形状

static void XLogDumpDisplayStats(XLogDumpConfig *config, XLogStats *stats)

入力と出力

  • **入力**:
  • config: 実行オプションを保持する XLogDumpConfig 構造体。stats_per_record フラグにより表示の粒度（RMGR単位か、レコードタイプ単位か）を制御する。
  • stats: 集計済みの統計データを含む XLogStats 構造体。
  • **出力**: なし（標準出力へのフォーマット済みテキスト出力のみ）。

前提条件と不変条件

  • stats->endptr が無効な場合（XLogRecPtrIsInvalid が真）、統計情報が未計算とみなされ、何も出力せずに終了する。
  • stats 内の rmgr_stats および record_stats は、呼び出し前に適切に集計されている必要がある。

アルゴリズムまたは振る舞い

1. **全体集計**: 全リソースマネージャの統計を走査し、列ごとの合計値（total_count, total_rec_len, total_fpi_len）を算出する。 2. **ヘッダー出力**: 統計対象の LSN 範囲と、各列のラベルを表示する。 3. **行出力**:

  • config->stats_per_record が偽の場合: リソースマネージャ単位で XLogDumpStatsRow を呼び出す。
  • config->stats_per_record が真の場合: リソースマネージャ内の各レコードタイプ（MAX_XLINFO_TYPES）を走査し、個別に XLogDumpStatsRow を呼び出す。

    • 4. **フッター出力**: 全体の合計行を表示し、レコードサイズと FPI サイズが全体に占める割合をパーセンテージで出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_waldump/pg_waldump.c:1306（main 関数内での統計表示処理）。
  • **呼び出し先**:
  • GetRmgrDesc: リソースマネージャの記述子を取得。
  • XLogDumpStatsRow: 各行のフォーマットと出力。
  • desc->rm_identify: レコードタイプ識別子の取得。
  • psprintf: 文字列の動的生成。

関連データ構造

  • XLogDumpConfig: 統計表示設定を保持。
  • XLogStats: 集計された WAL 統計データ。
  • RmgrDescData: リソースマネージャのメタデータ。

失敗モードと境界条件

  • RmgrIdIsValid で無効と判定された ID はスキップされる。
  • config->stats_per_record が真の場合、レコードタイプごとの統計が 0 のものは出力から除外される。
  • total_len が 0 の場合、パーセンテージ計算で 0 除算を回避するロジックが組み込まれている。

保守担当者向けメモ

  • 本関数は pg_waldump の出力フォーマットを定義しているため、表示形式を変更する場合は XLogDumpStatsRow との整合性に注意すること。
  • rm_identify コールバックが NULL を返す場合、UNKNOWN としてフォールバック表示する仕様となっている。
