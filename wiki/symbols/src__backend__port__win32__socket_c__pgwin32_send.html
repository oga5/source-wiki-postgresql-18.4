pgwin32_send

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:458

目的

Windows環境において、POSIX準拠の send() 関数をWinsock2 API上でエミュレートし、PostgreSQLのシグナル処理と統合されたソケット送信機能を提供します。

シグネチャまたは形状

int pgwin32_send(SOCKET s, const void *buf, int len, int flags);

入力と出力

  • **入力**:
  • s: 送信対象のソケットハンドル。
  • buf: 送信データバッファへのポインタ。
  • len: 送信データの長さ。
  • flags: Winsockの WSASend に渡されるフラグ。
  • **出力**:
  • 成功時: 送信されたバイト数（int）。
  • 失敗時: -1 を返し、errno を適切に設定します。

前提条件と不変条件

  • 呼び出しの冒頭で pgwin32_poll_signals() が実行され、保留中のシグナルがある場合は即座に中断されます。
  • pgwin32_noblock が真の場合、送信がブロックされる状況では EWOULDBLOCK を返します。
  • UDPソケットの場合、送信準備が整っていると報告されても送信に失敗する可能性があるため、内部でループ処理を行います。

アルゴリズムまたは振る舞い

1. pgwin32_poll_signals() を呼び出し、シグナル割り込みをチェックします。 2. WSASend を使用してデータの送信を試みます。 3. 送信が成功した場合、送信バイト数を返します。 4. WSAEWOULDBLOCK 以外のエラーが発生した場合は TranslateSocketError() を呼び出し、エラーコードを変換して終了します。 5. pgwin32_noblock が有効な場合、送信がブロックされるなら errno を EWOULDBLOCK に設定して終了します。 6. 上記以外（ブロックが必要な場合）は pgwin32_waitforsinglesocket を呼び出し、ソケットが書き込み可能になるかクローズされるまで待機してから再試行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて send マクロとして定義されており、PostgreSQLのネットワーク通信箇所から間接的に呼び出されます。
  • **呼び出し先**: pgwin32_poll_signals, WSASend, WSAGetLastError, TranslateSocketError, pgwin32_waitforsinglesocket。

関連データ構造

  • SOCKET (Win32型)
  • WSABUF (Winsock構造体)

失敗モードと境界条件

  • シグナルが保留中の場合、-1 を返します。
  • WSASend が SOCKET_ERROR を返し、かつ WSAEWOULDBLOCK ではない場合、TranslateSocketError() を経由して変換されたエラーを返します。
  • pgwin32_waitforsinglesocket が待機に失敗した場合（戻り値 0）、-1 を返します。

保守担当者向けメモ

  • 本関数は src/include/port/win32_port.h で send をオーバーライドするために使用されています。
  • UDP通信時の再試行ループは、Windowsのソケットイベント通知の特性を考慮した実装です。
  • exceptfds のサポートは含まれていません。
