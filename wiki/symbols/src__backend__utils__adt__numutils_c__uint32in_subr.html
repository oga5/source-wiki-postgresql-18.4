uint32in_subr

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numutils.c:897

目的

文字列入力を uint32 型の数値へ変換します。PostgreSQL の型システムに適合するエラーチェックとパースを行い、不正な構文や範囲外の値を適切に処理します。

シグネチャまたは形状

uint32 uint32in_subr(const char *s, char **endloc, const char *typname, Node *escontext)

入力と出力

  • **入力**:
  • s: 解析対象の文字列。
  • endloc: 解析終了位置を格納するポインタ（NULL の場合、文字列末尾に空白以外の文字があることをエラーとみなす）。
  • typname: エラーメッセージに使用する型名。
  • escontext: エラー報告用コンテキスト。
  • **出力**:
  • 変換された uint32 値を返します。エラー発生時は escontext を介して報告されます。

前提条件と不変条件

  • strtoul を使用して変換を行います。
  • PG_UINT32_MAX != ULONG_MAX の環境では、unsigned long から uint32 への変換時に値の範囲を再検証し、オーバーフローを検出します。
  • 負の入力値については、符号付き/符号なしの解釈で一致する場合に限り許容する後方互換性ロジックを含みます。

アルゴリズムまたは振る舞い

1. strtoul(s, &endptr, 0) を呼び出し、文字列を数値に変換します。 2. errno または endptr == s を確認し、構文エラーまたは範囲外エラーを escontext に報告します。 3. endloc が NULL の場合、数値の後に空白以外の文字が残っていないか検証します。 4. unsigned long が uint32 より大きい環境では、変換後の値が uint32 の範囲内に収まっているかをチェックし、範囲外であればエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/oid.c (oidin, oidvectorin, float8_oid 等)
  • src/backend/utils/adt/xid.c (xidin, cidin)
  • **呼び出し先**:
  • strtoul (標準ライブラリ)
  • ereturn (エラー報告マクロ)

関連データ構造

  • Node *escontext: エラーハンドリングのためのコンテキスト構造体。

失敗モードと境界条件

  • **構文エラー**: endptr == s の場合（数値として解析できない）。
  • **範囲外エラー**: errno == ERANGE の場合、または unsigned long から uint32 へのキャストで値が変化する場合。
  • **末尾の不正文字**: endloc が NULL で、数値の後に空白以外の文字が存在する場合。

保守担当者向けメモ

  • 本関数は strtoul をラップしており、プラットフォーム間の unsigned long サイズの違いを吸収する設計になっています。
  • エラー報告には ereport ではなく ereturn マクロを使用しているため、呼び出し元で escontext を適切に管理する必要があります。
