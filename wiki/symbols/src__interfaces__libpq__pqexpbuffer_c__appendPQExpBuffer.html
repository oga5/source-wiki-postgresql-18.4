appendPQExpBuffer

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.c:264

目的

PQExpBuffer 構造体に対して、printf 形式のフォーマット済み文字列を追加します。

シグネチャまたは形状

void appendPQExpBuffer(PQExpBuffer str, const char *fmt, ...)

入力と出力

  • **入力**:
  • str: 操作対象の PQExpBuffer 構造体へのポインタ。
  • fmt: printf 形式の書式指定文字列。
  • ...: 書式指定に対応する可変長引数。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • str が指すバッファが初期化済みであること。
  • バッファが「broken」状態（メモリ不足等で破壊された状態）である場合、本関数は何も行わず即座に復帰します。

アルゴリズムまたは振る舞い

1. PQExpBufferBroken マクロにより、バッファが既に失敗状態にあるかを確認し、該当する場合は終了します。 2. errno を保存し、do-while ループを開始します。 3. va_start で可変長引数を取得し、appendPQExpBufferVA を呼び出して文字列の追加を試みます。 4. appendPQExpBufferVA がバッファ不足により書き込みを完了できなかった場合（戻り値が false）、内部でバッファ拡張が行われた後にループが継続されます。 5. 書き込みが成功するまでループを繰り返し、最後に va_end を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c
  • src/bin/initdb/initdb.c
  • src/bin/pg_amcheck/pg_amcheck.c

    • (上記はツールによる発見に基づく参照箇所です)

  • **呼び出し先**:
  • appendPQExpBufferVA (直接の文字列追加処理)

関連データ構造

  • PQExpBuffer: 文字列バッファを管理する構造体。

失敗モードと境界条件

  • **メモリ不足**: appendPQExpBufferVA 内でメモリ確保に失敗した場合、バッファは「broken」状態に遷移します。以降、本関数は PQExpBufferBroken チェックにより安全に無視されます。
  • **フォーマットエラー**: fmt 引数が不正な場合、vsnprintf の挙動に依存します。

保守担当者向けメモ

  • 本関数は va_list を使用して appendPQExpBufferVA に処理を委譲するラッパーです。
  • errno を保存・復元することで、フォーマット処理中の副作用から呼び出し元の環境を保護しています。
