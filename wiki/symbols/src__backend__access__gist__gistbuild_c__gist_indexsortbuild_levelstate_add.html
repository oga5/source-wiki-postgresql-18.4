gist_indexsortbuild_levelstate_add

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuild.c:460

目的

GiSTインデックスのソート済み構築（Sorted Build）において、指定されたレベルの状態管理構造体へインデックス・タプルを追加します。

シグネチャまたは形状

static void gist_indexsortbuild_levelstate_add(GISTBuildState *state, GistSortedBuildLevelState *levelstate, IndexTuple itup)

入力と出力

  • state: 構築全体の状態を保持する GISTBuildState へのポインタ。
  • levelstate: 現在の階層の状態を保持する GistSortedBuildLevelState へのポインタ。
  • itup: 追加対象の IndexTuple。
  • 出力: なし（副作用として levelstate 内のページへのタプル追加、またはページ溢れ時のフラッシュ処理を行う）。

前提条件と不変条件

  • levelstate->pages は初期化済みであること。
  • itup のサイズがページ容量を超えないこと（ページ分割ロジックは本関数内では完結せず、フラッシュ処理に依存）。

アルゴリズムまたは振る舞い

1. 追加対象のタプルサイズとアイテム識別子の合計サイズを計算します。 2. 現在のページに十分な空き容量があるか確認します。 3. 空き容量が不足している場合:

  • 現在のページが GIST_SORTED_BUILD_PAGE_NUM に達している場合は、gist_indexsortbuild_levelstate_flush を呼び出してページをディスクへ書き出します。
  • そうでない場合は、次のページへインデックスを進めます。
  • 新しいページが未割り当てであれば palloc0 で確保し、gistinitpage で初期化します。

    • 4. 最後に gistfillbuffer を使用して、現在のページにタプルを書き込みます。

関連データ構造

  • GISTBuildState: インデックス構築全体のコンテキスト。
  • GistSortedBuildLevelState: ソート済み構築における特定の階層の状態（ページ配列、現在のページインデックス等）。
  • IndexTuple: GiSTインデックスに格納されるタプル形式。

失敗モードと境界条件

  • メモリ不足: palloc0 が失敗した場合、PostgreSQLのメモリ管理機構によりエラーがスローされます。
  • ページ容量超過: IndexTuple 自体が BLCKSZ を超えるような巨大なタプルの場合、gistfillbuffer 等でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はソート済み構築のボトムアップ処理の一部です。ページが満杯になった際のフラッシュ処理は gist_indexsortbuild_levelstate_flush に依存しているため、ページ管理ロジックを変更する際は両方の関数を確認してください。
