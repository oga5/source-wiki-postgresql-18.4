local_queue_fetch_range

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/local_source.c:125

目的

ローカルファイルシステム上のソースデータディレクトリから、指定されたファイル内の特定のオフセット範囲を読み込み、ターゲットディレクトリへ書き出します。

シグネチャまたは形状

static void local_queue_fetch_range(rewind_source *source, const char *path, off_t off, size_t len)

入力と出力

  • source: local_source 構造体へのポインタ（rewind_source としてキャスト）。
  • path: 読み込み対象ファイルの相対パス。
  • off: 読み込み開始オフセット。
  • len: 読み込むバイト数。
  • 出力: なし（副作用としてターゲットファイルへの書き込みを実行）。

前提条件と不変条件

  • source は local_source 型としてキャスト可能であること。
  • ターゲットディレクトリへの書き込み権限およびソースファイルへの読み込み権限があること。
  • 読み込み中にソースファイルが変更されないこと（変更された場合、整合性エラーが発生する）。

アルゴリズムまたは振る舞い

1. source からデータディレクトリのパスを取得し、対象ファイルのフルパスを構築します。 2. 対象ファイルを読み込み専用で開き、lseek で指定された off までシークします。 3. open_target_file を呼び出し、ターゲット側の書き込み準備を行います。 4. len に達するまで、PGIOAlignedBlock バッファを使用してデータをチャンク単位で読み込みます。 5. 読み込んだデータを write_target_range を通じてターゲットへ書き出します。 6. 全データの転送完了後、ソースファイルをクローズします。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_rewind/local_source.c 内の初期化処理にて、rewind_source インターフェースの関数ポインタとして登録されます。
  • 呼び出し先: open (システムコール), lseek (システムコール), open_target_file, read (システムコール), write_target_range, pg_fatal, close (システムコール)。

関連データ構造

  • rewind_source: ソース抽象化インターフェース。
  • local_source: ローカルディレクトリパスを保持する構造体。
  • PGIOAlignedBlock: I/O 操作用のアライメントされたバッファ。

失敗モードと境界条件

  • ファイルのオープン失敗、シーク失敗、読み込みエラー、クローズ失敗時に pg_fatal を呼び出し、プロセスを終了します。
  • 読み込み中に予期せぬ EOF に到達した場合、pg_fatal を呼び出します。

保守担当者向けメモ

  • 本関数は同期的に動作し、呼び出し時点で即座にファイル I/O を完了させます。
  • 非同期的なフェッチを行う他のソース実装との互換性を保つため、本関数は即時実行されます。
