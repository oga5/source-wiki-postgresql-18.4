pgaio_io_set_target

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_target.c:63

目的

非同期I/Oハンドル（PgAioHandle）に対して、I/O操作の対象となるリソース（ターゲット）を割り当てます。

シグネチャまたは形状

void pgaio_io_set_target(PgAioHandle *ioh, PgAioTargetID targetid)

入力と出力

  • **入力**:
  • ioh: 操作対象の PgAioHandle 構造体へのポインタ。
  • targetid: 割り当てるターゲットを識別する PgAioTargetID 型の値。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • ioh->state が PGAIO_HS_HANDED_OUT であること。
  • ioh->target が PGAIO_TID_INVALID であること（ターゲットが未設定であること）。
  • 上記条件は Assert により実行時に検証されます。

アルゴリズムまたは振る舞い

1. ioh の状態が適切か、ターゲットが未設定かを確認するアサーションを実行します。 2. 引数 targetid を ioh->target に代入し、ハンドルにターゲットを紐付けます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/smgr/smgr.c:1047（PGAIO_TID_SMGR を設定する箇所）
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行いません。

関連データ構造

  • PgAioHandle: I/O操作の状態を保持する構造体。
  • PgAioTargetID: ターゲットの種類を識別する列挙型または整数型。

失敗モードと境界条件

  • 前提条件を満たさない場合（既にターゲットが設定されている、または状態が不適切な場合）、アサーション失敗によりプロセスが異常終了する可能性があります。

保守担当者向けメモ

  • 本関数は pgaio_io_start_* 系の関数を呼び出す前の必須ステップです。
  • 新しいターゲットタイプを追加する場合は、PgAioTargetInfo 構造体および pgaio_target_info 配列への登録が必要です。
