am_leader_apply_worker

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/replication/worker_internal.h:340

目的

現在のプロセスが論理レプリケーションの「リーダー適用ワーカー（Leader Apply Worker）」であるかどうかを判定する。

シグネチャまたは形状

bool am_leader_apply_worker(void)

入力と出力

  • 入力: なし（グローバル変数 MyLogicalRepWorker を参照）
  • 出力: 現在のプロセスが WORKERTYPE_APPLY 型のワーカーとして動作している場合は true、それ以外は false を返す。

前提条件と不変条件

  • MyLogicalRepWorker が指すワーカー構造体が初期化済みであり、in_use フラグが true であること（Assert により検証）。

アルゴリズムまたは振る舞い

1. MyLogicalRepWorker->in_use を確認し、ワーカーが使用中であることを検証する。 2. MyLogicalRepWorker->type が WORKERTYPE_APPLY と一致するかを評価し、その結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/applyparallelworker.c
  • src/backend/replication/logical/launcher.c
  • src/backend/replication/logical/worker.c
  • 呼び出し先: なし（インライン関数であり、他の関数を呼び出さない）。

関連データ構造

  • LogicalRepWorker: ワーカーの状態を保持する構造体。MyLogicalRepWorker はこの型のグローバル変数。
  • LogicalRepWorkerType: ワーカーの種類を定義する列挙型（WORKERTYPE_APPLY を含む）。

失敗モードと境界条件

  • MyLogicalRepWorker が NULL である場合、または in_use が false である場合に Assert 違反が発生する。

保守担当者向けメモ

  • 本関数は並列適用ワーカーの制御ロジックにおいて、リーダーと並列ワーカーの役割を区別するために頻繁に使用される。
  • WORKERTYPE_APPLY は論理レプリケーションの適用ワーカーを指す。並列適用ワーカー（am_parallel_apply_worker）とは異なる役割である点に注意すること。
