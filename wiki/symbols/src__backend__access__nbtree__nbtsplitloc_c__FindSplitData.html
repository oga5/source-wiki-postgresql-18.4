FindSplitData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtsplitloc.c:40

目的

B-tree インデックスのページ分割処理において、分割計算のコンテキスト情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    Relation    rel;
    Page        origpage;
    IndexTuple  newitem;
    Size        newitemsz;
    bool        is_leaf;
    bool        is_rightmost;
    OffsetNumber newitemoff;
    int         leftspace;
    int         rightspace;
    int         olddataitemstotal;
    Size        minfirstrightsz;
    int         maxsplits;
    int         nsplits;
    SplitPoint *splits;
    int         interval;
} FindSplitData;

入力と出力

本構造体は関数間で受け渡される状態保持用オブジェクトであり、直接的な入力・出力関数はありません。各フィールドは、ページ分割のシミュレーションに必要なリレーション情報、ページ状態、および候補となる分割点のリストを保持します。

前提条件と不変条件

  • splits フィールドは、分割計算中に割り当てられた SplitPoint 配列を指す必要があります。
  • is_leaf および is_rightmost は、対象ページの特性を正確に反映している必要があります。

アルゴリズムまたは振る舞い

ページ分割の最適化プロセスにおいて、以下の役割を果たします。 1. **状態保持**: 挿入される newitem を含めた仮想的なページ状態を管理します。 2. **候補蓄積**: _bt_recsplitloc 等の関数により、有効な分割候補（SplitPoint）が splits 配列に蓄積されます。 3. **最適化**: 蓄積された候補に対し、空き容量の均衡化やサフィックス切り詰め効率に基づいた評価を行い、最適な分割位置を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: _bt_findsplitloc（本ファイル内のメインエントリポイント）にてインスタンス化されます。
  • **呼び出し先**: 本構造体を引数として受け取る関数群（_bt_recsplitloc, _bt_deltasortsplits, _bt_afternewitemoff, _bt_bestsplitloc, _bt_defaultinterval, _bt_strategy, _bt_interval_edges, _bt_split_penalty, _bt_split_lastleft, _bt_split_firstright）によって参照・更新されます。

関連データ構造

  • SplitPoint: 分割候補の各詳細情報を保持する構造体。
  • Relation, Page, IndexTuple: PostgreSQL のコアデータ構造。

失敗モードと境界条件

  • splits 配列のメモリ割り当てが不足する場合、あるいは nsplits が maxsplits を超過する場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本構造体は、ページ分割のシミュレーションを行うための「作業用メモリ」として機能します。
  • 内部ページとリーフページで評価基準（ペナルティ）が異なるため、is_leaf フラグに基づいた条件分岐が各計算関数内で適切に行われているか確認してください。
