pg_localeconv_copy_members

定義

  • 種別: function
  • 定義箇所: src/port/pg_localeconv_r.c:154

目的

lconv 構造体のメンバをソースから宛先へコピーし、文字列メンバについては動的にメモリを確保して複製することで、スレッドセーフなロケール情報の保持を支援します。

シグネチャまたは形状

static int pg_localeconv_copy_members(struct lconv *dst, struct lconv *src, int category);

入力と出力

  • **入力**:
  • dst: コピー先の lconv 構造体へのポインタ。
  • src: コピー元の lconv 構造体へのポインタ。
  • category: 対象とするロケールカテゴリ（LC_NUMERIC または LC_MONETARY）。
  • **出力**:
  • 成功時は 0 を返します。
  • メモリ確保に失敗した場合は -1 を返し、errno に ENOMEM を設定します。

前提条件と不変条件

  • table[] 配列が定義されており、各メンバのカテゴリと型（文字列か否か）が正しく設定されている必要があります。
  • dst は呼び出し元によって適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. table[] を走査し、指定された category に一致するメンバを特定します。 2. メンバが文字列型の場合:

  • src から該当文字列を取得し、strdup() で複製します。
  • 複製に失敗した場合は pg_localeconv_free(dst) を呼び出して既に確保済みのメモリを解放し、エラーを返します。
  • 成功した場合は dst の該当メンバにポインタを格納します。

    • 3. メンバが文字列型以外の場合:

  • 値を直接コピーします。

    • 4. 全ての対象メンバの処理が完了したら 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/pg_localeconv_r.c 内の複数の箇所から呼び出されています（例: LC_MONETARY や LC_NUMERIC の処理時）。
  • **呼び出し先**:
  • lconv_string_member() / lconv_char_member(): メンバへのアクセスに使用。
  • strdup(): 文字列の複製に使用。
  • pg_localeconv_free(): エラー発生時のメモリ解放に使用。

関連データ構造

  • struct lconv: 標準 C ライブラリのロケール情報構造体。
  • lconv_member_info: 各メンバの属性を保持するメタデータテーブル。

失敗モードと境界条件

  • strdup() がメモリ不足により NULL を返した場合、関数は即座に中断し、それまでに確保したメモリを解放した上でエラーを報告します。

保守担当者向けメモ

  • 本関数が確保した文字列メンバは動的メモリであるため、呼び出し元は必ず pg_localeconv_free() を使用して解放する必要があります。
  • table[] の定義を変更する際は、is_string フラグと category の整合性に注意してください。
