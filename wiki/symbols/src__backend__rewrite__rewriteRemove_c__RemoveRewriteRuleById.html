RemoveRewriteRuleById

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteRemove.c:32

目的

指定された OID に対応する書き換えルール（Rewrite Rule）をシステムカタログ pg_rewrite から削除し、関連するリレーションキャッシュを無効化する。

シグネチャまたは形状

void RemoveRewriteRuleById(Oid ruleOid)

入力と出力

  • **入力**: ruleOid (削除対象ルールの OID)
  • **出力**: なし（成功時はカタログ更新とキャッシュ無効化を実行、失敗時はエラーをスロー）

前提条件と不変条件

  • pg_rewrite カタログへのアクセス権限が必要。
  • ルールが適用されている対象テーブル（イベントリレーション）に対して AccessExclusiveLock を取得し、当該ルールに依存するクエリの実行を排他する必要がある。
  • システムカタログに対する操作制限（allowSystemTableMods）が適用される。

アルゴリズムまたは振る舞い

1. pg_rewrite を RowExclusiveLock で開く。 2. RewriteOidIndexId を用いて対象ルールのタプルを検索する。タプルが存在しない場合はエラーをスローする。 3. ルールが定義されている対象テーブル（ev_class）を AccessExclusiveLock で開く。 4. 対象テーブルがシステムカタログであり、かつ allowSystemTableMods が無効な場合はエラーをスローする。 5. CatalogTupleDelete を呼び出し、カタログからタプルを物理削除する。 6. CacheInvalidateRelcache を呼び出し、全バックエンドに対してリレーションキャッシュの無効化を通知する。 7. pg_rewrite を閉じ、対象テーブルのロックを保持したまま終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c:1412
  • **呼び出し先**:
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan
  • CatalogTupleDelete
  • CacheInvalidateRelcache
  • elog, ereport

関連データ構造

  • pg_rewrite: ルール定義を保持するシステムカタログ。
  • Relation: テーブルやカタログへのアクセスに使用する構造体。
  • HeapTuple: カタログ内のタプル表現。

失敗モードと境界条件

  • 指定した OID のルールが見つからない場合、elog(ERROR, ...) により処理が中断される。
  • システムカタログに対する削除操作で権限不足が発生した場合、ereport(ERROR, ...) により処理が中断される。
  • AccessExclusiveLock の取得により、対象テーブルに対する他の操作と競合し、デッドロックや待機が発生する可能性がある。

保守担当者向けメモ

  • 本関数はルール削除のライフサイクル終端を担う。
  • CacheInvalidateRelcache の呼び出しは必須であり、これを省略すると他のバックエンドで古いルール定義が参照され、整合性が損なわれる。
  • ロックの取得順序（pg_rewrite → event_relation）はデッドロック回避のため厳守すること。
