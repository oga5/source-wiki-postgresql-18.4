y_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgquadtreeproc.c:156

目的

2つの Point 構造体へのポインタを比較し、その y 座標値に基づいてソート順序を決定する。

シグネチャまたは形状

static int y_cmp(const void *a, const void *b, void *arg)

入力と出力

  • **入力**:
  • a, b: Point 構造体へのポインタを指すポインタ (Point **)。
  • arg: 使用されていない汎用引数。
  • **出力**:
  • 0: pa->y == pb->y の場合。
  • 1: pa->y > pb->y の場合。
  • -1: pa->y < pb->y の場合。

前提条件と不変条件

  • a および b は、有効な Point 構造体を指すポインタの配列要素である必要がある。

アルゴリズムまたは振る舞い

1. 引数 a および b を Point ** 型にキャストし、参照先の Point 構造体を取得する。 2. 取得した2つの点の y 座標を比較する。 3. 比較結果に基づき、標準的なソート用比較関数の規約（0, 1, -1）に従って値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgquadtreeproc.c:188: qsort の比較関数として使用。
  • src/backend/access/spgist/spgkdtreeproc.c:125: qsort の比較関数として使用（条件分岐による動的選択）。
  • **呼び出し先**: なし。

関連データ構造

  • Point: utils/geo_decls.h で定義される、2次元座標（x, y）を持つ構造体。

失敗モードと境界条件

  • a または b が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • y 座標が NaN の場合の挙動は、標準的な比較演算子に依存する。

保守担当者向けメモ

  • 本関数は qsort 用の比較関数として設計されている。
  • spgkdtreeproc.c からも参照されているため、変更時は両方のファイルへの影響を考慮すること。
