namelt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:165

目的

2つの name 型の値を受け取り、第1引数が第2引数よりも小さいかどうかを判定する比較関数です。

シグネチャまたは形状

Datum namelt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの name 型の引数を受け取ります。
  • **出力**: 第1引数が第2引数より小さい場合は true、それ以外の場合は false を Datum 型として返します。

前提条件と不変条件

  • name 型は NAMEDATALEN（通常64バイト）の固定長文字列として扱われます。
  • 比較には現在の照合順序（Collation）が適用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_NAME マクロを使用して、2つの入力引数 arg1 と arg2 を取得します。 2. PG_GET_COLLATION() で現在の照合順序を取得します。 3. namecmp 関数を呼び出し、引数と照合順序を渡して比較結果を得ます。 4. namecmp の戻り値が 0 未満（arg1 < arg2）であるかを判定し、その結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの比較演算子（<）から呼び出されます。
  • **呼び出し先**: namecmp 関数を呼び出します。

関連データ構造

  • Name: name 型の内部表現。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • namecmp の挙動に依存します。namecmp は C ロケールの場合に高速な strncmp を使用し、それ以外の場合は varstr_cmp を介して汎用的な比較を行います。
  • NAMEDATALEN を超える部分は比較において無視されます。

保守担当者向けメモ

  • 本関数は name 型の比較演算子の一部です。比較ロジックを変更する場合は namecmp を確認してください。
  • 照合順序のサポートは namecmp 経由で提供されるため、ロケール依存の挙動については varstr_cmp の仕様を参照してください。
