GetLatestCommitTsData

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/commit_ts.c:359

目的

最後にコミットされたトランザクションのID、コミット時刻、およびレプリケーション元ノードIDを取得します。

シグネチャまたは形状

TransactionId GetLatestCommitTsData(TimestampTz *ts, RepOriginId *nodeid)

入力と出力

  • **入力**:
  • ts: コミット時刻を格納する TimestampTz へのポインタ（NULL可）。
  • nodeid: レプリケーション元ノードIDを格納する RepOriginId へのポインタ（NULL可）。
  • **出力**:
  • 最後にコミットされたトランザクションID（TransactionId）。

前提条件と不変条件

  • CommitTsLock（共有ロック）を取得して共有メモリ上のデータにアクセスします。
  • 本機能が有効（commitTsShared->commitTsActive が真）である必要があります。

アルゴリズムまたは振る舞い

1. CommitTsLock を共有モードで取得します。 2. commitTsShared->commitTsActive を確認し、無効な場合は error_commit_ts_disabled() を呼び出してエラーを発生させます。 3. 共有メモリ上の commitTsShared から、最後にコミットされたトランザクションID、時刻、ノードIDを読み取ります。 4. 引数 ts および nodeid が NULL でない場合、読み取った値をそれぞれに代入します。 5. CommitTsLock を解放し、取得したトランザクションIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/commit_ts.c 内で参照されていることが確認されています。
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease（ロック管理）
  • error_commit_ts_disabled（エラーハンドリング）

関連データ構造

  • CommitTimestampShared: コミットタイムスタンプの共有メモリ状態を保持する構造体。
  • TransactionId: トランザクション識別子。
  • TimestampTz: PostgreSQLのタイムスタンプ型。
  • RepOriginId: レプリケーション元ノード識別子。

失敗モードと境界条件

  • track_commit_timestamp が無効な状態で呼び出された場合、エラーが発生します。
  • ts や nodeid に NULL を渡した場合、その値の書き込みはスキップされます。

保守担当者向けメモ

  • 本関数は共有メモリ上の最新情報を取得する軽量なAPIです。
  • 呼び出し元情報の詳細は、ソースコード内の静的解析結果に基づいています。
