mxact

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/multixact.c:3719

目的

MultiXact のメンバー情報を一時的に保持し、反復処理を行うための構造体です。

シグネチャまたは形状

typedef struct
{
    MultiXactMember *members;
    int            nmembers;
    int            iter;
} mxact;

入力と出力

  • members: MultiXactMember 型の配列へのポインタ。
  • nmembers: 配列に含まれるメンバーの総数。
  • iter: 現在の反復処理におけるインデックス（カーソル）。

前提条件と不変条件

  • members は palloc 等で適切にメモリ確保されている必要があります。
  • iter は 0 から nmembers - 1 の範囲で操作されることが想定されています。

アルゴリズムまたは振る舞い

本構造体は、特定の MultiXactId に関連付けられたメンバーリストをメモリ上に展開し、順次アクセスするために使用されます。palloc を用いて動的に確保されたインスタンスに対し、メンバー情報の読み込みと反復的な走査を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/multixact.c 内の関数（palloc によるインスタンス生成および user_fctx へのキャストによる参照を確認）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • MultiXactMember: メンバーの個別の状態（トランザクション ID とロック状態）を保持する構造体。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • 範囲外アクセス: iter が nmembers を超えてインクリメントされた場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は主に multixact.c 内の内部的な反復処理用として定義されています。
  • heapam.c 等の外部モジュールから直接参照されることは想定されておらず、あくまで multixact.c の実装詳細の一部です。
