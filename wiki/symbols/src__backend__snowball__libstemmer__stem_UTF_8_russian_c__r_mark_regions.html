r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_russian.c:355

目的

ロシア語のステミング処理において、単語内の語幹抽出に使用する領域（R1, R2）の境界位置を計算し、Snowball環境の整数配列 z->I に格納する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: z (Snowball実行環境へのポインタ)
  • **出力**: 常に 1 を返す。
  • **副作用**: z->I[0] および z->I[1] を更新する。

前提条件と不変条件

  • z は有効な SN_env 構造体であること。
  • g_v（母音グループ）が定義されていること。
  • 処理対象の文字列はUTF-8エンコーディングであること。

アルゴリズムまたは振る舞い

1. z->I[1]（R1の開始位置）と z->I[0]（R2の開始位置）を初期値として文字列長 z->l に設定する。 2. 母音と子音の切り替わりを検出し、領域を特定する：

  • out_grouping_U（子音）と in_grouping_U（母音）を交互に適用し、単語内の構造を走査する。
  • 最初の母音の直後の位置を z->I[1] に記録する。
  • さらにその後の母音の直後の位置を z->I[0] に記録する。

    • 3. マッチングに失敗した場合は、初期値（文字列末尾）を維持する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述なし。
  • **呼び出し先**: out_grouping_U, in_grouping_U（Snowballランタイム関数）。

関連データ構造

  • SN_env: ステミング処理の実行環境を保持する構造体。
  • g_v: 母音を定義するグループ構造体。

失敗モードと境界条件

  • 文字列が短い場合や母音を含まない場合、領域の境界は文字列の末尾に設定される。
  • out_grouping_U または in_grouping_U が負の値を返した場合、処理を中断し、初期値を維持して正常終了する。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、元のSnowballスクリプト（.sbl）を修正すること。
  • z->I[0] と z->I[1] は、後続の接尾辞削除処理において、削除可能な範囲を制限するために使用される。
