option_parse_int

定義

  • 種別: function
  • 定義箇所: src/fe_utils/option_utils.c:49

目的

コマンドライン引数として渡された文字列を整数に変換し、指定された範囲内にあるか検証する。エラー発生時には標準エラー出力へログを出力する。

シグネチャまたは形状

bool option_parse_int(const char *optarg, const char *optname, int min_range, int max_range, int *result);

入力と出力

  • **入力**:
  • optarg: 解析対象の文字列。
  • optname: エラーメッセージに使用するオプション名。
  • min_range, max_range: 許容される整数の範囲（境界値を含む）。
  • result: 変換後の値を格納するポインタ（NULL可）。
  • **出力**:
  • 成功時は true を返し、result に値を格納する。
  • 失敗時は false を返し、pg_log_error を通じてエラーを出力する。

前提条件と不変条件

  • optarg は有効な文字列であること。
  • 数値の後に空白以外の文字が続く場合は不正な値として扱われる。
  • errno は関数内で初期化される。

アルゴリズムまたは振る舞い

1. strtoint を使用して文字列を整数に変換する。 2. 変換後のポインタ位置から末尾まで、空白文字をスキップする。 3. 空白以外の文字が残っている場合、エラーとして false を返す。 4. errno が ERANGE であるか、値が min_range 未満または max_range 超過の場合、エラーとして false を返す。 5. 上記のいずれにも該当しない場合、result が NULL でなければ値を代入し、true を返す。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **不正な形式**: 数値の後に非空白文字が含まれる場合、エラーとなる。
  • **範囲外**: min_range 未満または max_range 超過の場合、エラーとなる。
  • **オーバーフロー**: strtoint が ERANGE を設定した場合、エラーとなる。

保守担当者向けメモ

  • 本関数は pg_log_error を使用するため、呼び出し側でログ出力先が適切に設定されている必要がある。
  • 数値の厳密な検証を行うため、数値の直後に単位などが付与される形式には対応していない。
