oauth_get_mechanisms

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-oauth.c:88

目的

認証メカニズムのリストを要求するクライアントに対し、サポートされているSASLメカニズム名（OAUTHBEARER）を通知する。

シグネチャまたは形状

static void oauth_get_mechanisms(Port *port, StringInfo buf)

入力と出力

  • **入力**:
  • port: 現在の接続状態を保持する Port 構造体へのポインタ（本関数内では未使用）。
  • buf: メカニズム名を格納するための StringInfo バッファ。
  • **出力**: なし（buf に文字列が追記される）。

前提条件と不変条件

  • buf は appendStringInfoString および appendStringInfoChar を呼び出し可能な状態であること。
  • サポートするメカニズムは OAUTHBEARER のみに固定されている。

アルゴリズムまたは振る舞い

1. OAUTHBEARER_NAME を buf に追加する。 2. 文字列の終端としてヌル文字（\0）を buf に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c 内の pg_be_oauth_mech 構造体（.get_mechanisms フィールド）から参照されている。
  • **呼び出し先**:
  • appendStringInfoString
  • appendStringInfoChar

関連データ構造

  • StringInfo: PostgreSQLの動的文字列バッファ。
  • Port: 接続情報を保持する構造体。

失敗モードと境界条件

  • buf のメモリ不足が発生した場合、appendStringInfo 系の関数内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は OAUTHBEARER 以外のメカニズムをサポートしていないため、将来的に他のOAuth関連メカニズムを追加する場合は本関数を拡張する必要がある。
