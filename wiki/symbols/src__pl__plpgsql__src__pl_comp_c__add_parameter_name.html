add_parameter_name

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:914

目的

PL/pgSQL 関数のコンパイル過程において、パラメータ名を名前空間（namespace）に追加する。

シグネチャまたは形状

static void add_parameter_name(PLpgSQL_nsitem_type itemtype, int itemno, const char *name)

入力と出力

  • **入力**:
  • itemtype: 追加するアイテムの型（PLpgSQL_nsitem_type）。
  • itemno: アイテムに関連付けられた識別番号。
  • name: 追加するパラメータ名。
  • **出力**: なし（void）。成功時は名前空間に登録され、失敗時はエラーを報告する。

前提条件と不変条件

  • 名前空間の重複チェックが必須である。
  • PL/pgSQL 内では、IN パラメータと OUT パラメータは同一の名前空間で管理されるため、重複は許容されない。

アルゴリズムまたは振る舞い

1. plpgsql_ns_lookup を呼び出し、指定された name が現在の名前空間（plpgsql_ns_top()）に既に存在するかを確認する。 2. 重複が見つかった場合、ERRCODE_INVALID_FUNCTION_DEFINITION エラーを発生させる。 3. 重複がない場合、plpgsql_ns_additem を呼び出して名前空間にアイテムを登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_comp.c 内の関数（362行目、366行目付近で確認）。
  • **呼び出し先**:
  • plpgsql_ns_top()
  • plpgsql_ns_lookup()
  • plpgsql_ns_additem()
  • ereport()

関連データ構造

  • PLpgSQL_nsitem_type: 名前空間アイテムの型定義。
  • plpgsql_ns_top() が返す名前空間スタック。

失敗モードと境界条件

  • **失敗モード**: 同一スコープ内で既に存在する名前を登録しようとした場合、ereport によりエラー終了する。
  • **境界条件**: functioncmds.c でのチェックをすり抜けた IN/OUT パラメータの競合も、本関数で捕捉される。

保守担当者向けメモ

  • 本関数は、PL/pgSQL の名前解決において重複を厳格に排除する役割を持つ。
  • plpgsql_ns_lookup の第2引数に true を渡しており、現在のスコープのみを対象とした検索を行っている。
