heap_delete

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam.c:2772

目的

ヒープテーブル内の指定されたタプルを論理的に削除する。

シグネチャまたは形状

TM_Result heap_delete(Relation relation, ItemPointer tid, CommandId cid, 
                      Snapshot crosscheck, bool wait, TM_FailureData *tmfd, 
                      bool changingPart)

入力と出力

  • **入力**:
  • relation: 対象のテーブルリレーション。
  • tid: 削除対象タプルの物理位置（TID）。
  • cid: 現在のコマンドID。
  • crosscheck: 可視性チェック用のスナップショット。
  • wait: 他のトランザクションによるロック競合時に待機するかどうか。
  • tmfd: 失敗時の詳細情報を格納する構造体。
  • changingPart: パーティション変更中かどうかのフラグ。
  • **出力**:
  • TM_Result: 削除操作の結果（成功、不可視、競合など）。

前提条件と不変条件

  • tid は有効であること。
  • TOAST 処理のために適切なスナップショットが利用可能であること。
  • 並列モード（Parallel Mode）実行中ではないこと。

アルゴリズムまたは振る舞い

1. 対象タプルを含むページをバッファに読み込み、排他ロックを取得する。 2. ページが「全可視（All-Visible）」であれば、可視性マップ（VM）をピン留めする。 3. HeapTupleSatisfiesUpdate を呼び出し、タプルの可視性と更新可能性を判定する。 4. タプルが他トランザクションによってロックされている場合（TM_BeingModified）:

  • wait が真であれば、heap_acquire_tuplock でタプルロックを取得し、競合するトランザクションの終了を待機する。
  • 待機後、タプルの状態が変化した可能性があるため、必要に応じて再試行（goto l1）を行う。

    • 5. 削除対象が不可視である場合はエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c: heap_delete インターフェース経由。
  • src/backend/access/heap/heapam.c: 内部的な再帰的呼び出しや関連処理。
  • **呼び出し先**:
  • HeapTupleSatisfiesUpdate, LockBuffer, visibilitymap_pin, heap_acquire_tuplock, MultiXactIdWait, XactLockTableWait 等。

関連データ構造

  • Relation: テーブル定義。
  • ItemPointer: タプルの物理位置。
  • TM_Result: 操作結果の列挙型。
  • Buffer: ページバッファ。

失敗モードと境界条件

  • **並列モードエラー**: 並列操作中に呼び出された場合、ERRCODE_INVALID_TRANSACTION_STATE でエラー。
  • **不可視タプル**: 削除対象が現在のスナップショットから不可視の場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー。
  • **競合**: 他のトランザクションがタプルをロックしている場合、wait が偽であれば競合を返す。

保守担当者向けメモ

  • 本関数は START_CRIT_SECTION() を含むクリティカルセクションの開始前段階のロジックを含みます。
  • goto l1 による再試行ループは、バッファロックの解放と再取得を伴うため、デッドロックや競合状態の回避において重要です。
  • ページが「全可視」から変化した場合の再試行ロジックは、I/O をまたいだロック保持を避けるための設計です。
