pgp_key_alloc

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubkey.c:37

目的

PGP 鍵情報を保持するための PGP_PubKey 構造体に対してメモリを割り当て、初期化を行います。

シグネチャまたは形状

int pgp_key_alloc(PGP_PubKey **pk_p)

入力と出力

  • **入力**: PGP_PubKey **pk_p （割り当てられた構造体へのポインタを格納するアドレス）
  • **出力**: int 型の戻り値。成功時は 0 を返します。

前提条件と不変条件

  • palloc0 を使用しているため、PostgreSQL のメモリコンテキスト内で実行される必要があります。
  • 割り当てられた構造体はゼロクリアされます。

アルゴリズムまたは振る舞い

1. palloc0 を呼び出し、PGP_PubKey 構造体サイズ分のメモリをゼロ初期化状態で確保します。 2. 確保したメモリのアドレスを引数 pk_p が指すポインタに代入します。 3. 0 を返して正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pubkey.c:163 にて呼び出されていることが確認されています。
  • **呼び出し先**: PostgreSQL のメモリ管理関数 palloc0 を呼び出します。

関連データ構造

  • PGP_PubKey: 本関数で割り当てられる主要な鍵情報構造体。

失敗モードと境界条件

  • palloc0 がメモリ不足によりエラーを発生させた場合、PostgreSQL の標準的なエラーハンドリング（elog 等）により処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数で割り当てたメモリは、対応する解放関数（pgp_key_free）を使用して適切に解放する必要があります。構造体のメンバに変更を加える際は、解放処理側も併せて更新してください。
