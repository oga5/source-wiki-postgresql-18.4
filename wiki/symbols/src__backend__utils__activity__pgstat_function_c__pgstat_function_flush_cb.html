pgstat_function_flush_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_function.c:192

目的

バックエンドローカルに蓄積された関数実行統計（呼び出し回数および実行時間）を、共有メモリ上の統計情報へマージ（フラッシュ）するために使用されるコールバック関数です。

シグネチャまたは形状

bool pgstat_function_flush_cb(PgStat_EntryRef *entry_ref, bool nowait)

入力と出力

  • **入力**:
  • entry_ref: 統計エントリへの参照。ローカルの保留中統計（pending）と共有メモリ上の統計（shared_stats）を含む。
  • nowait: ロック取得時に待機するかどうかを指定するフラグ。
  • **出力**:
  • bool: 統計のフラッシュに成功した場合は true、ロック取得に失敗した場合は false を返す。

前提条件と不変条件

  • entry_ref->pending は PgStat_FunctionCounts 型としてキャスト可能であること。
  • entry_ref->shared_stats は PgStatShared_Function 型としてキャスト可能であること。
  • localent（ローカル統計）には常にゼロではない内容が含まれていることが前提となっている。

アルゴリズムまたは振る舞い

1. entry_ref からローカル統計構造体と共有メモリ上の統計構造体を取得する。 2. pgstat_lock_entry を呼び出し、共有メモリ上の統計エントリをロックする。nowait が true でロック取得に失敗した場合は直ちに false を返す。 3. ローカルの numcalls、total_time、self_time を共有メモリ上の統計値に加算する。この際、時間はマイクロ秒単位に変換される。 4. pgstat_unlock_entry を呼び出してロックを解放する。 5. true を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 統計サブシステムのフラッシュ処理（src/backend/utils/activity/pgstat.c 内の .flush_pending_cb フィールド経由）。
  • **呼び出し先**:
  • pgstat_lock_entry / pgstat_unlock_entry（統計エントリの排他制御）
  • INSTR_TIME_GET_MICROSEC（時間計測値の単位変換）

関連データ構造

  • PgStat_EntryRef: 統計エントリの参照管理構造体。
  • PgStat_FunctionCounts: バックエンドローカルの関数統計保持構造体。
  • PgStatShared_Function: 共有メモリ上の関数統計保持構造体。

失敗モードと境界条件

  • nowait が true の場合、他プロセスによるロック競合が発生すると、統計の更新を行わずに false を返す。

保守担当者向けメモ

  • 本関数は統計収集フレームワークのコールバックとして登録されており、直接呼び出すよりも統計サブシステム経由での実行が想定されています。
  • 統計の加算処理はアトミックではなく、pgstat_lock_entry による排他制御に依存しているため、ロックの取得・解放順序を維持してください。
