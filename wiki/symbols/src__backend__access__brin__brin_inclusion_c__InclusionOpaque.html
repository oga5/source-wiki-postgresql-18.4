InclusionOpaque

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/brin/brin_inclusion.c:76

目的

BRIN（Block Range Index）の「包含（Inclusion）」演算子クラスにおいて、インデックス列ごとのメタデータおよび関数キャッシュを保持するために使用されます。

シグネチャまたは形状

typedef struct InclusionOpaque
{
	FmgrInfo	extra_procinfos[INCLUSION_MAX_PROCNUMS];
	bool		extra_proc_missing[INCLUSION_MAX_PROCNUMS];
	Oid			cached_subtype;
	FmgrInfo	strategy_procinfos[RTMaxStrategyNumber];
} InclusionOpaque;

入力と出力

本構造体は直接的な関数入出力ではなく、BRINインデックスの記述子（BrinDesc）内の oi_opaque フィールドを通じて保持される状態コンテナです。

前提条件と不変条件

  • brin_inclusion_opcinfo によって初期化され、システムカタログから取得した関数情報がキャッシュされます。
  • extra_procinfos および strategy_procinfos は、インデックス操作のオーバーヘッドを削減するために利用されます。

アルゴリズムまたは振る舞い

  • **関数キャッシュ**: inclusion_get_procinfo および inclusion_get_strategy_procinfo を介して、必要な関数情報を一度だけ取得し、以降の呼び出しで再利用します。
  • **状態管理**: インデックスの各列に対して、サブタイプ情報や演算子戦略に対応する関数ポインタを保持し、スキャンや値の追加時に参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • brin_inclusion_opcinfo（初期化時）
  • brin_inclusion_add_value（値の追加時）
  • brin_inclusion_consistent（検索時）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありませんが、保持している FmgrInfo を通じて関数マネージャが実行されます。

関連データ構造

  • BrinDesc: 本構造体を保持する上位のインデックス記述子。
  • FmgrInfo: PostgreSQLの関数マネージャ情報。

失敗モードと境界条件

  • extra_proc_missing 配列により、特定のサポート関数が存在しない場合に適切にフラグを立て、実行時のエラーや不要な再検索を回避します。
  • INCLUSION_MAX_PROCNUMS および RTMaxStrategyNumber の範囲外へのアクセスは、定義された配列サイズにより制限されています。

保守担当者向けメモ

  • inclusion_get_strategy_procinfo は minmax_get_strategy_procinfo と設計が共通しています。修正時は両方の整合性を確認してください。
  • INET 型のように、単一の型であっても内部的に結合できない値が存在する場合の挙動を考慮する必要があります。
