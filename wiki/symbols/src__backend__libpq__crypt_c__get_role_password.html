get_role_password

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/crypt.c:37

目的

システムカタログ pg_authid から指定されたロールのパスワードハッシュを取得し、パスワードの有効期限が切れていないかを検証する。

シグネチャまたは形状

char *get_role_password(const char *role, const char **logdetail);

入力と出力

  • **入力**:
  • role: 検索対象のロール名（文字列）。
  • logdetail: 認証失敗時の詳細メッセージを格納するためのポインタ。
  • **出力**:
  • 成功時: pg_authid から取得したパスワードハッシュ文字列（char *）。
  • 失敗時: NULL。このとき logdetail に失敗理由が設定される。

前提条件と不変条件

  • pg_authid システムカタログへのアクセスが可能であること。
  • 認証プロセス中であるため、データベース選択前の制限（TOAST制限等）が適用される。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して pg_authid からロール情報を取得する。 2. ロールが存在しない場合、またはパスワードが設定されていない（rolpassword が NULL）場合は NULL を返し、logdetail にエラーを設定する。 3. rolvaliduntil 属性を確認し、有効期限が設定されている場合は GetCurrentTimestamp() と比較する。 4. 現在時刻が有効期限を過ぎている場合、認証失敗として NULL を返し、logdetail に期限切れの旨を設定する。 5. 検証を通過した場合、取得したパスワード文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c（認証フロー内で使用）。
  • **呼び出し先**: SearchSysCache1, SysCacheGetAttr, ReleaseSysCache, TextDatumGetCString, GetCurrentTimestamp, psprintf。

関連データ構造

  • pg_authid: ロール情報を保持するシステムカタログ。
  • HeapTuple: システムカタログの行データ。
  • TimestampTz: パスワード有効期限の型。

失敗モードと境界条件

  • **ロール不在**: logdetail に「Role "..." does not exist.」を設定。
  • **パスワード未設定**: logdetail に「User "..." has no password assigned.」を設定。
  • **有効期限切れ**: logdetail に「User "..." has an expired password.」を設定。

保守担当者向けメモ

  • セキュリティ上の理由から、logdetail に設定されるエラーメッセージはサーバーログへの出力用であり、クライアントへ直接送信してはならない。
  • pg_authid のキャッシュを操作するため、ReleaseSysCache による適切な解放が必須である。
