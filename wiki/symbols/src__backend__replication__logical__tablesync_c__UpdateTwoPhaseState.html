UpdateTwoPhaseState

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/tablesync.c:1791

目的

指定されたサブスクリプションの二相コミット（two-phase）状態を pg_subscription カタログ上で更新する。

シグネチャまたは形状

void UpdateTwoPhaseState(Oid suboid, char new_state)

入力と出力

  • **入力**:
  • suboid (Oid): 更新対象のサブスクリプションのOID。
  • new_state (char): 設定する新しい状態。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • new_state は LOGICALREP_TWOPHASE_STATE_DISABLED、PENDING、または ENABLED のいずれかである必要がある（Assert による検証）。
  • 呼び出し元は適切なトランザクションコンテキストにいる必要がある。

アルゴリズムまたは振る舞い

1. pg_subscription カタログを RowExclusiveLock で開く。 2. 指定された suboid に対応するタプルをシステムキャッシュから検索し、コピーを取得する。 3. subtwophasestate カラムを new_state に更新するための値を準備する。 4. heap_modify_tuple を使用してタプルを更新し、CatalogTupleUpdate でカタログへ書き込む。 5. 取得したタプルを解放し、カタログを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • table_open, table_close
  • SearchSysCacheCopy1, heap_freetuple
  • heap_modify_tuple, CatalogTupleUpdate

関連データ構造

  • pg_subscription (システムカタログ)
  • Anum_pg_subscription_subtwophasestate (更新対象カラムのインデックス)

失敗モードと境界条件

  • suboid に対応するサブスクリプションがカタログに存在しない場合、elog(ERROR) によりトランザクションがアボートされる。

保守担当者向けメモ

  • 本関数は共有メモリ上の状態ではなく、永続的なカタログ情報を直接更新する点に注意すること。
  • subtwophasestate の変更は、論理レプリケーションの二相コミットの挙動に直接影響を与える。
