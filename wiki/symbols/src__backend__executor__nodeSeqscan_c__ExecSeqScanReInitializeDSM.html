ExecSeqScanReInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:382

目的

並列クエリ実行において、シーケンシャルスキャンの共有メモリ状態（DSM）を再初期化し、スキャンを最初からやり直せる状態にする。

シグネチャまたは形状

void ExecSeqScanReInitializeDSM(SeqScanState *node, ParallelContext *pcxt)

入力と出力

  • node: シーケンシャルスキャンの実行状態を保持する SeqScanState 構造体。
  • pcxt: 並列実行コンテキスト（本関数内では直接使用されていない）。
  • 戻り値: なし。

前提条件と不変条件

  • node->ss.ss_currentScanDesc が有効であり、かつその内部の rs_parallel が初期化されていること。
  • node->ss.ss_currentRelation がオープンされていること。

アルゴリズムまたは振る舞い

1. node から現在のスキャン記述子 (rs_parallel) を取得する。 2. table_parallelscan_reinitialize を呼び出し、対象リレーションとスキャン記述子を渡して、共有メモリ上の並列スキャン状態をリセットする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execParallel.c:978
  • 呼び出し先: table_parallelscan_reinitialize（テーブルアクセスメソッドのインターフェース）

関連データ構造

  • SeqScanState: シーケンシャルスキャンの実行状態。
  • ParallelTableScanDesc: 並列スキャン用の共有メモリ状態記述子。
  • ParallelContext: 並列実行コンテキスト。

失敗モードと境界条件

  • node が NULL の場合や、スキャン記述子が正しく設定されていない場合はクラッシュの可能性がある。

保守担当者向けメモ

  • 本関数は、並列クエリ実行中にスキャンを再開する必要がある場合に呼び出される。
  • pcxt 引数は現在の実装では使用されていないが、インターフェースの整合性のために保持されている。
