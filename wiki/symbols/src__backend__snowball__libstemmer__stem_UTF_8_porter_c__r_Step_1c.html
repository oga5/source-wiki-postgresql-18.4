r_Step_1c

目的

Porter Stemming アルゴリズムのステップ 1c を実行し、単語末尾の 'y' または 'Y' を 'i' に置換する。

シグネチャまたは形状

static int r_Step_1c(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: 成功時に 1、失敗または条件不一致時に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • 内部で out_grouping_b_U を使用して、置換対象の直前に母音が存在するかを確認する。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置（z->c）を z->ket に設定する。 2. 直前の文字が 'y' または 'Y' であるかを確認し、該当すればカーソルを一つ戻す。 3. z->bra を現在のカーソル位置に設定する。 4. out_grouping_b_U を用いて、直前に母音（g_v）が存在するかを判定する。 5. 条件を満たす場合、slice_from_s を呼び出し、対象文字を 'i' に置換する。

呼び出し元と呼び出し先

  • 呼び出し元: porter_UTF_8_stem（直接的または間接的）、および stem_UTF_8_porter.c 内のメイン処理フロー。
  • 呼び出し先: out_grouping_b_U（母音判定）、slice_from_s（文字列置換）。

関連データ構造

  • SN_env: ステミング処理中のカーソル位置、文字列バッファ、領域境界を管理する。
  • g_v: 母音グループを定義する定数。

失敗モードと境界条件

  • 文字列の先頭に達している場合や、直前に母音がない場合は 0 を返して終了する。
  • slice_from_s が失敗した場合は、そのエラーコードを伝播させる。
