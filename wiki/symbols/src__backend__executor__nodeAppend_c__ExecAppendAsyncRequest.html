ExecAppendAsyncRequest

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:977

目的

Append ノードにおいて、非同期実行が必要なサブプランに対してリクエストを発行し、既に取得済みの非同期結果があればそれを返却する。

シグネチャまたは形状

static bool ExecAppendAsyncRequest(AppendState *node, TupleTableSlot **result)

入力と出力

  • **入力**:
  • node: AppendState 構造体へのポインタ。
  • result: 取得したタプルを格納する TupleTableSlot へのポインタのポインタ。
  • **出力**:
  • 戻り値: タプルが取得できた場合は true、それ以外は false。
  • result: タプルが取得できた場合、そのスロットが設定される。

前提条件と不変条件

  • node->as_needrequest が空の場合、node->as_nasyncresults は 0 である必要がある（Assert による検証）。

アルゴリズムまたは振る舞い

1. **既存結果の確認**: node->as_nasyncresults > 0 の場合、蓄積された非同期結果から1つを取り出し、result にセットして true を返す。 2. **新規リクエストの発行**: node->as_needrequest にビットマップで示された非同期サブプランに対し、ExecAsyncRequest を呼び出してリクエストを発行する。処理後、as_needrequest はクリアされる。 3. **再確認**: リクエスト発行後に再度 node->as_nasyncresults > 0 を確認し、結果があればそれを返却する。 4. **終了**: 上記のいずれにも該当しない場合、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAppend.c 内の ExecAppend（推測される文脈に基づく）。
  • **呼び出し先**: ExecAsyncRequest。

関連データ構造

  • AppendState: Append ノードの実行状態を保持。
  • AsyncRequest: 非同期リクエストの定義。
  • Bitmapset: リクエストが必要なサブプランの管理に使用。

失敗モードと境界条件

  • 非同期サブプランが存在しない、またはリクエストが不要な場合は即座に false を返す。
  • 非同期結果が空かつリクエスト対象もない場合、タプルは返されない。

保守担当者向けメモ

  • 本関数は Append ノードの非同期実行ループの一部であり、postgres_fdw 等の非同期対応データラッパーと密接に関連している。
  • as_asyncresults 配列のインデックス操作は as_nasyncresults の値に依存しているため、変更時は整合性に注意すること。
