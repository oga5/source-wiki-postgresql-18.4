test_rls_hooks_permissive

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_rls_hooks/test_rls_hooks.c:43

目的

PostgreSQLの行レベルセキュリティ（RLS）フック機能のテストにおいて、許可的（Permissive）なRLSポリシーを動的に生成して返す。

シグネチャまたは形状

List *test_rls_hooks_permissive(CmdType cmdtype, Relation relation)

入力と出力

  • **入力**:
  • cmdtype: CmdType 型のコマンド種別。
  • relation: 対象となる Relation 構造体。
  • **出力**:
  • List *: 生成された RowSecurityPolicy を含むリスト。対象テーブル名が rls_test_permissive または rls_test_both でない場合は NIL を返す。

前提条件と不変条件

  • 対象テーブル名が rls_test_permissive または rls_test_both である必要がある。
  • ParseState を用いた構文解析が成功し、current_user = username という形式のポリシー式が構築される。
  • 生成されるポリシーは ACL_ID_PUBLIC ロールに対して適用される。

アルゴリズムまたは振る舞い

1. テーブル名を確認し、対象外であれば NIL を即座に返す。 2. make_parsestate で一時的な解析状態を作成し、対象リレーションを名前空間に追加する。 3. RowSecurityPolicy 構造体を割り当て、ポリシー名や適用ロール（ACL_ID_PUBLIC）を設定する。 4. current_user 関数呼び出しと username カラム参照を用いた A_Expr（current_user = username）を構築する。 5. transformWhereClause を呼び出し、式を解析して policy->qual に格納する。 6. assign_expr_collations で照合順序を確定させる。 7. with_check_qual に qual をコピーし、リストに格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_rls_hooks/test_rls_hooks.c: row_security_policy_hook_permissive グローバル変数への関数ポインタ代入。
  • **呼び出し先**:
  • make_parsestate, addRangeTableEntryForRelation, addNSItemToQuery
  • makeFuncCall, makeSimpleA_Expr
  • transformWhereClause, assign_expr_collations
  • palloc0, pstrdup, list_make1, copyObject

関連データ構造

  • RowSecurityPolicy: 生成されるポリシー情報を保持する構造体。
  • ParseState: 構文解析のコンテキストを保持する構造体。

失敗モードと境界条件

  • 対象テーブル名が一致しない場合、ポリシーを生成せず NIL を返す。
  • transformWhereClause 等の解析関数が失敗した場合、PostgreSQLの標準的なエラーハンドリングにより処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、実運用環境での使用は想定されていない。
  • restrictive フックと併用する場合、許可的ポリシーの挙動が全体のアクセス制御に与える影響に注意すること。
