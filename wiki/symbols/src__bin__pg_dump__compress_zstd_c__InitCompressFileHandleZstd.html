InitCompressFileHandleZstd

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_zstd.c:29

目的

pg_dump における Zstandard (Zstd) 圧縮を用いたファイルハンドルを初期化します。

シグネチャまたは形状

void InitCompressFileHandleZstd(CompressFileHandle *CFH, const pg_compress_specification compression_spec);

入力と出力

  • **入力**:
  • CFH: 初期化対象となる CompressFileHandle 構造体へのポインタ。
  • compression_spec: 圧縮設定を保持する pg_compress_specification 構造体。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • 本関数は USE_ZSTD マクロが定義されていない環境（Zstd ライブラリが利用不可なビルド）で呼び出されます。
  • 実行時に必ずエラーを発生させ、処理を中断します。

アルゴリズムまたは振る舞い

  • pg_fatal を呼び出し、「this build does not support compression with ZSTD」というメッセージを出力してプログラムを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_io.c:207
  • src/bin/pg_dump/compress_zstd.c:560
  • **呼び出し先**:
  • pg_fatal（PostgreSQL のエラーハンドリング関数）

関連データ構造

  • CompressFileHandle: 圧縮処理を行うためのファイルハンドル構造体。
  • pg_compress_specification: 圧縮アルゴリズムやレベルを指定する構造体。

失敗モードと境界条件

  • 本関数自体が、Zstd がサポートされていない環境における失敗モードそのものを表しています。呼び出された時点で常にエラー終了します。

保守担当者向けメモ

  • 本ファイルは USE_ZSTD マクロで囲まれており、Zstd が有効なビルドでは別の実装が提供される可能性があります。現在の実装はスタブとして機能しています。
