SharedFileSet

定義

  • 種別: struct
  • 定義箇所: src/include/storage/sharedfileset.h:26

目的

複数のバックエンド間で共有される一時ファイル群を管理し、プロセス間での協調的なアクセスと自動的なライフサイクル管理（クリーンアップ）を実現する。

シグネチャまたは形状

typedef struct SharedFileSet
{
	FileSet		fs;
	slock_t		mutex;
	int			refcnt;
} SharedFileSet;

入力と出力

  • **入力**: 共有メモリセグメント（dsm_segment）上に配置されることを前提とする。
  • **出力**: 参照カウントが 0 になった際、保持する全一時ファイルを物理的に削除する。

前提条件と不変条件

  • mutex は refcnt の更新を保護するために使用される。
  • refcnt は、現在このファイルセットにアタッチしているバックエンドの数を表す。
  • 本構造体は必ず共有メモリ（DSM）セグメントと組み合わせて使用される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: SharedFileSetInit により、参照カウントが 1 で初期化される。 2. **アタッチ**: SharedFileSetAttach が呼ばれると、スピンロックを使用して refcnt がインクリメントされる。 3. **デタッチ**: プロセス終了時等に SharedFileSetOnDetach が実行され、refcnt がデクリメントされる。 4. **クリーンアップ**: refcnt が 0 に達した時点で、FileSetDeleteAll が呼び出され、関連するすべてのファイルが削除される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/file/buffile.c および src/backend/utils/sort/logtape.c 等で利用されている。
  • **呼び出し先**: FileSet（基盤となるファイル管理）、slock_t（スピンロック）、および dsm_segment（共有メモリ管理）に依存する。

関連データ構造

  • FileSet: SharedFileSet が内包する、ローカルな一時ファイル管理用構造体。
  • dsm_segment: 共有メモリセグメント。ライフサイクル管理の基盤となる。
  • slock_t: 参照カウント保護用のスピンロック。

失敗モードと境界条件

  • 既に破棄された SharedFileSet に対してアタッチを試みた場合、エラーが発生する。
  • 参照カウントの不整合は、スピンロックの競合やデタッチ処理の欠落により発生する可能性がある。

保守担当者向けメモ

  • SharedFileSetOnDetach はエラー発生時のクリーンアップパスでも呼び出されるため、堅牢な実装が求められる。
  • ファイル操作の詳細は src/include/storage/fileset.h を参照すること。
