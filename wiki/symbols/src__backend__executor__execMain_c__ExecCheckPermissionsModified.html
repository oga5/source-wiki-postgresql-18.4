ExecCheckPermissionsModified

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execMain.c:754

目的

リレーションの特定の列に対するユーザーのアクセス権限を検証します。特に、更新対象の列（modifiedCols）が指定されている場合に、それらの列に対する権限チェックを行います。

シグネチャまたは形状

static bool ExecCheckPermissionsModified(Oid relOid, Oid userid, Bitmapset *modifiedCols, AclMode requiredPerms)

入力と出力

  • **入力**:
  • relOid: 対象リレーションの OID。
  • userid: 権限チェックを行うユーザーの OID。
  • modifiedCols: 更新対象の列を示すビットマップセット。
  • requiredPerms: 要求されるアクセス権限（AclMode）。
  • **出力**:
  • bool: すべての対象列に対して権限が許可されている場合は true、それ以外は false を返します。

前提条件と不変条件

  • modifiedCols が空の場合、リレーションのいずれかの列に対して requiredPerms があれば許可とみなされます。
  • modifiedCols に含まれるビット位置は FirstLowInvalidHeapAttributeNumber を加算して属性番号（attno）に変換されます。

アルゴリズムまたは振る舞い

1. modifiedCols が空の場合、pg_attribute_aclcheck_all を使用してリレーションのいずれかの列に対して権限があるか確認します。 2. modifiedCols が空でない場合、ビットマップセット内の各ビットを走査します。 3. 各ビットに対応する属性番号を計算し、pg_attribute_aclcheck を呼び出して個別の列に対する権限を検証します。 4. 権限チェックで ACLCHECK_OK 以外が返された場合、直ちに false を返します。 5. すべてのチェックを通過した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execMain.c 内の関数から呼び出されています（詳細はソースコード上の直接的な参照に基づきます）。
  • **呼び出し先**: bms_is_empty, bms_next_member, pg_attribute_aclcheck_all, pg_attribute_aclcheck, elog。

関連データ構造

  • Bitmapset: 更新対象列の管理に使用。
  • Oid: リレーションおよびユーザーの識別子。
  • AclMode: 権限の種類。

失敗モードと境界条件

  • attno が InvalidAttrNumber となる場合（行全体への参照など）、elog(ERROR, ...) によりエラーが発生します。
  • 権限チェックが失敗した場合は false を返し、呼び出し元で処理を中断します。

保守担当者向けメモ

  • 本関数は SELECT FOR UPDATE や UPDATE 文の権限検証において重要な役割を果たします。
  • whole-row update は現在未対応であり、エラーとして扱われます。
