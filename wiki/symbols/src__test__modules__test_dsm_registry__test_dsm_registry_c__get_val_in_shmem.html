get_val_in_shmem

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_dsm_registry/test_dsm_registry.c:64

目的

共有メモリ上に配置された TestDSMRegistryStruct から現在の値を読み取り、SQLインターフェースを通じて呼び出し元に返却する。

シグネチャまたは形状

Datum get_val_in_shmem(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQLの関数呼び出し規約に基づく引数）。
  • **出力**: Datum 型としてキャストされた int32 値（共有メモリ内の val）。

前提条件と不変条件

  • 共有メモリセグメントが既に初期化されていること。
  • tdr_state が有効な共有メモリ領域を指していること。

アルゴリズムまたは振る舞い

1. tdr_attach_shmem() を呼び出し、共有メモリへのアタッチとロックのトランシェ登録を行う。 2. tdr_state->lck に対して LW_SHARED モードで軽量ロック（LWLock）を取得する。 3. 共有メモリ上の tdr_state->val をローカル変数 ret にコピーする。 4. LWLockRelease を呼び出し、ロックを解放する。 5. PG_RETURN_INT32(ret) を用いて値を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • tdr_attach_shmem()
  • LWLockAcquire()
  • LWLockRelease()

関連データ構造

  • TestDSMRegistryStruct: 共有メモリ上のデータ構造（val と lck を保持）。

失敗モードと境界条件

  • 共有メモリがアタッチできない場合、tdr_attach_shmem 内でエラーが発生する可能性がある。
  • ロック取得時にデッドロックや競合が発生した場合の挙動は、PostgreSQLの LWLock 機構に依存する。

保守担当者向けメモ

  • 本関数は読み取り専用のアクセスを行うため、LW_SHARED ロックを使用している。書き込みを行う set_val_in_shmem との整合性に注意すること。
  • tdr_state は静的ポインタであり、本モジュール内で共有メモリ領域を追跡するために使用されている。
