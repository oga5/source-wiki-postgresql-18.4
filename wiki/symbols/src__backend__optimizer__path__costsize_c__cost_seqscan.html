cost_seqscan

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/costsize.c:294

目的

シーケンシャルスキャン（SeqScan）パスのコスト（startup_cost, total_cost）および行数（rows）を算出し、指定された Path 構造体に設定する。

シグネチャまたは形状

void cost_seqscan(Path *path, PlannerInfo *root, RelOptInfo *baserel, ParamPathInfo *param_info)

入力と出力

  • **入力**:
  • path: コスト計算結果を格納する対象の Path 構造体。
  • root: プランナのグローバル情報。
  • baserel: スキャン対象のベースリレーション情報。
  • param_info: パラメータ化されたパスの場合の行数情報（存在する場合）。
  • **出力**:
  • path の rows, startup_cost, total_cost, disabled_nodes フィールドが更新される。

前提条件と不変条件

  • baserel->relid > 0 であること。
  • baserel->rtekind == RTE_RELATION であること。

アルゴリズムまたは振る舞い

1. **行数設定**: param_info があればその行数を、なければ baserel->rows を path->rows に設定する。 2. **I/Oコスト計算**: テーブルスペースのページコストを取得し、baserel->pages を乗じて disk_run_cost を算出する。 3. **CPUコスト計算**:

  • 制限句（restrictinfo）の評価コストを算出。
  • cpu_tuple_cost と制限句のコストを合計し、タプル数分を乗算。
  • path->pathtarget の評価コストを startup_cost および total_cost に加算。

    • 4. **並列処理の調整**: path->parallel_workers > 0 の場合、CPUコストを並列数で除算し、行数を clamp_row_est で調整する。 5. **最終設定**: enable_seqscan フラグに基づき disabled_nodes を設定し、各コストを合計して total_cost を算出する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/pathnode.c:1001
  • **呼び出し先**:
  • get_tablespace_page_costs
  • get_restriction_qual_cost
  • get_parallel_divisor
  • clamp_row_est

関連データ構造

  • Path: コスト計算結果を保持する構造体。
  • PlannerInfo: プランナのコンテキスト情報。
  • RelOptInfo: リレーションの統計情報（ページ数、タプル数など）。

失敗モードと境界条件

  • 並列ワーカー数が設定されている場合、行数がゼロ除算にならないよう clamp_row_est を介して調整される。
  • param_info が指定されていない場合は、ベースリレーション全体の行数が使用される。

保守担当者向けメモ

  • 本関数はMackert & Lohmanのモデルに基づく近似計算を行う。
  • disk_run_cost はOSのプリフェッチを考慮し、並列実行時でも減衰させない保守的な設計となっている。
