Syn

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/dict_synonym.c:23

目的

全文検索における同義語辞書の各エントリを保持する構造体です。入力単語とそれに対応する置換後の単語（同義語）のペアを管理します。

シグネチャまたは形状

typedef struct
{
	char	   *in;
	char	   *out;
	int			outlen;
	uint16		flags;
} Syn;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • in: 検索対象となる入力文字列。
  • out: 置換後の出力文字列。
  • outlen: out 文字列の長さ。
  • flags: 辞書処理用の制御フラグ。

前提条件と不変条件

  • 辞書初期化時（dsynonym_init）にメモリ上に配列として構築され、qsort によって in 文字列をキーとしてソートされる必要があります。
  • 検索時（dsynonym_lexize）には、ソート済み配列に対して bsearch を用いてアクセスされます。

アルゴリズムまたは振る舞い

  • 辞書ファイルから読み込まれた単語ペアを保持します。
  • dsynonym_lexize において、入力単語が in と一致した場合、対応する out が置換候補として返されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/dict_synonym.c 内の辞書初期化および検索関数（dsynonym_init, dsynonym_lexize 等）。
  • 呼び出し先: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。
  • 補足: contrib/dict_xsyn/dict_xsyn.c でも同名の構造体が定義・利用されていますが、これらは別個の定義です。

関連データ構造

  • DictSyn: 辞書全体の状態（Syn 配列、サイズ、フラグ）を保持する構造体。

失敗モードと境界条件

  • メモリ不足: 辞書構築時に palloc が失敗する可能性があります。
  • 検索失敗: bsearch で一致する in が見つからない場合、置換は行われません。

保守担当者向けメモ

  • in および out 文字列は palloc によって確保されたメモリを指します。
  • contrib/dict_xsyn にも同名の構造体が存在するため、コードの修正やデバッグ時には対象ファイルが src/backend/tsearch/dict_synonym.c であることを確認してください。
