ForeignScanState

定義

  • 種別: typedef
  • 定義箇所: src/include/foreign/fdwapi.h:43
  • 実体定義: nodes/execnodes.h:2099

目的

外部データラッパー（FDW）によるスキャン実行時の状態を保持するための実行ノード構造体です。

シグネチャまたは形状

構造体の詳細なフィールド定義は nodes/execnodes.h に存在します。FDW 開発者は、この構造体の fdw_state フィールドを使用して、スキャン実行に必要なプライベートな状態情報を保持します。

入力と出力

  • 入力: FDW の各コールバック関数（BeginForeignScan, IterateForeignScan 等）の引数として渡されます。
  • 出力: 該当なし（構造体へのポインタとして状態を管理）。

前提条件と不変条件

  • BeginForeignScan 等の FDW API を通じてエンジンから提供されます。
  • fdw_state フィールドは、FDW 実装側で自由に利用可能な領域として提供されます。

アルゴリズムまたは振る舞い

クエリ実行フェーズにおいて、BeginForeignScan で初期化され、IterateForeignScan でタプル取得のコンテキストとして利用され、最終的に EndForeignScan でリソースが解放されるライフサイクルを持ちます。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の実行エンジン（ModifyTable ノードやスキャン実行ノード）。
  • 呼び出し先: FDW 実装（contrib/file_fdw や contrib/postgres_fdw など）の各コールバック関数が、この構造体を受け取って処理を行います。

関連データ構造

  • FdwRoutine: この構造体を利用するコールバック関数群を保持する構造体。
  • TupleTableSlot: IterateForeignScan がタプルを格納するために使用する構造体。

失敗モードと境界条件

  • fdw_state が適切に初期化されていない場合、後続のコールバック関数でメモリ不正アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは fdwapi.h で型定義されていますが、実際の構造体定義は nodes/execnodes.h にあります。
  • FDW 固有の状態を保持する場合は、必ず fdw_state フィールドを使用してください。
  • contrib/file_fdw の実装が、この構造体を利用した状態管理の典型的な参照実装となります。
