set_base_rel_sizes

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/allpaths.c:289

目的

クエリ内のすべてのベースリレーション（RELOPT_BASEREL）に対して、サイズ見積もりおよび並列実行可能性の判定を行う。

シグネチャまたは形状

static void set_base_rel_sizes(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（クエリ全体のプランナ情報）。
  • **出力**: なし（副作用として root->simple_rel_array 内の各 RelOptInfo が更新される）。

前提条件と不変条件

  • root->simple_rel_array が初期化されていること。
  • rel->relid == rti であること（アサーションによる検証）。

アルゴリズムまたは振る舞い

1. root->simple_rel_array を走査し、各リレーション（rti）を順次処理する。 2. rel が NULL または RELOPT_BASEREL 以外の場合はスキップする。 3. クエリ全体で並列実行が許可されている場合（root->glob->parallelModeOK）、set_rel_consider_parallel を呼び出し、当該リレーションの並列実行可否を判定する。 4. set_rel_size を呼び出し、リレーションのサイズ見積もりを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/allpaths.c 内のメイン処理フロー（make_one_rel 等から参照）。
  • **呼び出し先**:
  • set_rel_consider_parallel
  • set_rel_size

関連データ構造

  • PlannerInfo: プランナのグローバル状態を保持。
  • RelOptInfo: 個別のリレーションの最適化情報を保持。
  • RangeTblEntry: レンジテーブルエントリ。

失敗モードと境界条件

  • rel が NULL のスロットはスキップされる。
  • RELOPT_BASEREL 以外のリレーション（結合リレーション等）は処理対象外となる。

保守担当者向けメモ

  • 本関数はパス生成の初期段階で実行される。set_rel_size は継承親リレーションに対して set_append_rel_size を呼び出す可能性があるため、並列実行フラグの判定順序が重要である。
