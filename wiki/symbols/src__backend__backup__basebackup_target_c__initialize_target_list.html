initialize_target_list

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_target.c:171

目的

組み込みのバックアップターゲット定義をグローバルなターゲットリスト（BaseBackupTargetTypeList）に登録し、システムが利用可能な状態にする。

シグネチャまたは形状

static void initialize_target_list(void)

入力と出力

  • 入力: なし（内部の静的配列 builtin_backup_targets を参照）
  • 出力: なし（副作用として BaseBackupTargetTypeList を更新）

前提条件と不変条件

  • TopMemoryContext が有効であること。
  • builtin_backup_targets が name フィールドが NULL である要素で終端されていること。

アルゴリズムまたは振る舞い

1. 現在のメモリコンテキストを TopMemoryContext に切り替え、リストの永続性を確保する。 2. builtin_backup_targets 配列を走査し、各要素を lappend を用いて BaseBackupTargetTypeList に追加する。 3. 処理終了後、元のメモリコンテキストに復帰する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • BaseBackupGetTargetHandle (同ファイル内)
  • BaseBackupGetSink (同ファイル内)
  • 呼び出し先:
  • MemoryContextSwitchTo
  • lappend

関連データ構造

  • BaseBackupTargetType: ターゲットの定義を保持する構造体。
  • BaseBackupTargetTypeList: 登録されたターゲットを保持するリスト。
  • TopMemoryContext: リストのメモリ確保に使用されるコンテキスト。

失敗モードと境界条件

  • メモリ不足時には lappend 経由でエラーが発生する可能性がある。
  • 既に初期化済みの場合の再実行については、本ソースコードからは明示的なガード条件は確認できない。

保守担当者向けメモ

  • 本関数は初回アクセス時に呼び出される初期化ルーチンである。
  • 拡張モジュールがターゲットを追加する際は BaseBackupAddTarget を使用するため、本関数は組み込みターゲットの登録のみに専念する。
