pg_u_prop_uppercase

定義

  • 種別: function
  • 定義箇所: src/common/unicode_category.c:132

目的

指定された Unicode コードポイントが「大文字（Uppercase）」プロパティを持つかどうかを判定します。

シグネチャまたは形状

bool pg_u_prop_uppercase(pg_wchar code)

入力と出力

  • **入力**: pg_wchar code（判定対象の Unicode コードポイント）
  • **出力**: bool（大文字プロパティを持つ場合は true、それ以外は false）

前提条件と不変条件

  • Unicode 規格の安定性ポリシーに基づき、カテゴリおよびプロパティ値は固定されています。
  • 入力値は pg_wchar 型の有効なコードポイントであることを前提とします。

アルゴリズムまたは振る舞い

1. **ASCII 最適化**: 入力 code が 0x80 未満の場合、unicode_opt_ascii テーブルを直接参照し、PG_U_PROP_UPPERCASE ビットマスクを用いて定数時間で判定結果を返します。 2. **テーブル検索**: 0x80 以上のコードポイントに対しては、unicode_uppercase テーブルに対して range_search を実行し、該当範囲内にコードポイントが含まれるかを判定して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode/category_test.c:70
  • src/common/unicode_category.c 内の他の関数（内部ロジックの一部として使用）
  • **呼び出し先**:
  • range_search（src/common/unicode_category.c 内のユーティリティ）

関連データ構造

  • unicode_opt_ascii: ASCII 文字のプロパティを保持する静的テーブル。
  • unicode_uppercase: 大文字プロパティを持つコードポイント範囲を保持する静的テーブル。
  • PG_U_PROP_UPPERCASE: プロパティ判定用のビットマスク。

失敗モードと境界条件

  • 0x80 未満の範囲外（負の値など）が入力された場合の挙動については、本ソース抜粋からは明示されていません。
  • range_search が内部で失敗または範囲外を返す場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は unicode_category_table.h で定義された静的データに依存しています。テーブルの更新が必要な場合は、生成スクリプトを確認してください。
  • unicode_category.c 内の他の pg_u_prop_* 系関数と一貫した設計になっています。
