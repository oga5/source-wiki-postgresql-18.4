PgAioTargetData

定義

  • 種別: union
  • 定義箇所: src/include/storage/aio_types.h:61

目的

非同期I/O（AIO）操作の対象となる物理的なリレーションやブロック位置を特定するための情報を保持します。ワーカープロセスがI/O完了後にターゲットを再特定し、処理を継続するために使用されます。

シグネチャまたは形状

typedef union PgAioTargetData
{
    struct
    {
        RelFileLocator rlocator;    /* 物理リレーション識別子 */
        BlockNumber blockNum;       /* リレーション先頭からのブロック番号 */
        BlockNumber nblocks;
        ForkNumber  forkNum:8;      /* フォーク番号 */
        bool        is_temp:1;      /* 一時リレーションか否か */
        bool        skip_fsync:1;   /* fsyncをスキップするか否か */
    } smgr;
} PgAioTargetData;

入力と出力

  • **入力**: I/O発行時に smgr 構造体の各フィールド（rlocator, blockNum 等）が設定されます。
  • **出力**: I/O完了後、呼び出し元がこの構造体を参照して、対象となったリレーションやブロックに対する事後処理（レポートや完了通知）を行います。

前提条件と不変条件

  • forkNum は8ビット、is_temp および skip_fsync は1ビットのビットフィールドとして定義されており、メモリ効率が最適化されています。
  • rlocator および blockNum は、I/O操作の再特定に十分な情報を提供する必要があります。

アルゴリズムまたは振る舞い

本構造体はデータ保持専用であり、ロジックは含みません。I/O発行時に設定され、完了時に PgAioReturn 等の構造体を通じて参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • pgaio_result_report (src/backend/storage/aio/aio_callback.c)
  • buffer_readv_complete_one, buffer_readv_report (src/backend/storage/buffer/bufmgr.c)
  • md_readv_report (src/backend/storage/smgr/md.c)
  • smgr_aio_describe_identity (src/backend/storage/smgr/smgr.c)
  • **呼び出し先**:
  • pgaio_io_get_target_data を介してI/Oハンドルから本構造体を取得するケースが多数確認されています。

関連データ構造

  • RelFileLocator: リレーションの物理的な特定に使用。
  • BlockNumber: ブロック位置の指定に使用。
  • PgAioReturn: I/O完了後のエラーハンドリングにおいて、本構造体と組み合わせて使用されます。

失敗モードと境界条件

  • ビットフィールド（forkNum, is_temp, skip_fsync）のサイズ制約を超えた値を代入した場合の動作は未定義です。
  • rlocator が無効な場合、I/O完了後の再特定処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は union として定義されていますが、現状では smgr 構造体のみが定義されています。将来的に他のI/Oターゲット形式が追加される可能性があります。
  • ビットフィールドの管理はメモリ効率を優先しているため、フィールド追加時は全体のサイズとアライメントに注意してください。
