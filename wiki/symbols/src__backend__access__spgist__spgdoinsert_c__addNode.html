addNode

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgdoinsert.c:79

目的

既存のSP-GiSTインナータプルに新しいノードを追加し、更新されたインナータプルを生成する。

シグネチャまたは形状

static SpGistInnerTuple addNode(SpGistState *state, SpGistInnerTuple tuple, Datum label, int offset)

入力と出力

  • **入力**:
  • state: SP-GiSTのインデックス状態。
  • tuple: 追加対象の既存インナータプル。
  • label: 新規ノードに付与するラベル。
  • offset: ノードを挿入する位置（負の場合は末尾に追加）。
  • **出力**:
  • ノードが追加された新しい SpGistInnerTuple を返す。

前提条件と不変条件

  • offset が tuple->nNodes を超える場合、エラーが発生する。
  • 新しいタプルは、元のタプルのプレフィックスやデータ構造を維持しつつ、ノード数が1つ増加した状態で生成される。

アルゴリズムまたは振る舞い

1. offset が負の場合、tuple->nNodes に設定して末尾挿入とする。 2. tuple->nNodes + 1 個のノードを格納するためのメモリを確保する。 3. SGITITERATE マクロを使用して既存のノードを走査し、指定された offset を境に新しい配列へコピーする。 4. spgFormNodeTuple を呼び出し、指定された label で新しいノードを作成して配列の offset 位置に配置する。 5. spgFormInnerTuple を呼び出し、更新されたノード配列を持つ新しいインナータプルを構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c:1525 にて直接呼び出されている。
  • **呼び出し先**: spgFormNodeTuple および spgFormInnerTuple を呼び出す。
  • **注記**: src/include/access/spgist.h や spgtextproc.c 等に存在する addNode という名称の構造体メンバーやフィールドは、本関数とは別の識別子である可能性がある。

関連データ構造

  • SpGistInnerTuple: 操作対象のインナータプル構造。
  • SpGistNodeTuple: インナータプル内に格納されるノード構造。
  • SpGistState: インデックス操作のメタデータ。

失敗モードと境界条件

  • offset > tuple->nNodes: elog(ERROR) により処理が中断される。
  • メモリ不足: palloc が失敗した場合、PostgreSQLのメモリ管理機構によりエラーがスローされる。

保守担当者向けメモ

  • 本関数は新しいインナータプルを生成して返すため、呼び出し側で古いタプルとの差し替えが必要となる。
  • SGITITERATE マクロによる反復処理とメモリ確保を行っているため、ノード数が多い場合のパフォーマンスに留意すること。
