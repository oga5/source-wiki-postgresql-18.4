PatternInfoArray

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:48

目的

ユーザーがコマンドライン引数で指定したデータベース、スキーマ、リレーションなどのパターン情報を動的に保持・管理するための可変長配列構造体です。

シグネチャまたは形状

typedef struct PatternInfoArray
{
	PatternInfo *data;
	size_t		len;
} PatternInfoArray;

入力と出力

  • **入力**: PatternInfo 型の要素を格納するメモリ領域（data）と、その要素数（len）。
  • **出力**: パターンマッチング処理において、対象をフィルタリングするためのリストとして利用されます。

前提条件と不変条件

  • data は PatternInfo 型の配列を指すポインタである必要があります。
  • len は現在格納されている有効な PatternInfo 要素の数を示します。

アルゴリズムまたは振る舞い

  • extend_pattern_info_array 関数により、必要に応じてメモリ領域が拡張されます。
  • append_database_pattern、append_schema_pattern、append_relation_pattern 等の関数を介して、特定のパターン情報が配列に追加されます。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数およびパターン追加用のヘルパー関数群（append_database_pattern, append_schema_pattern, append_relation_pattern, append_heap_pattern, append_btree_pattern）。
  • **呼び出し先**: extend_pattern_info_array がメモリ管理のために呼び出されます。

関連データ構造

  • PatternInfo: 配列の各要素として格納される、個別のパターン定義を保持する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（extend_pattern_info_array 内）の挙動については、標準的なメモリ確保エラーに依存します。
  • len が 0 の場合、パターン指定がない状態として扱われます。

保守担当者向けメモ

  • 本構造体は pg_amcheck のコマンドライン引数解析において、ユーザー指定のフィルタリング条件を保持する中心的な役割を果たします。
  • PatternInfo の定義内容と併せて、メモリの再割り当てロジックが正しく実装されているか確認してください。
