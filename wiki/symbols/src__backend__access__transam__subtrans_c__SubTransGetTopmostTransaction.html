SubTransGetTopmostTransaction

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:162

目的

指定されたトランザクションID（XID）からサブトランザクションの親子関係を遡り、そのトランザクションが属する最上位のトランザクションIDを特定する。

シグネチャまたは形状

TransactionId SubTransGetTopmostTransaction(TransactionId xid)

入力と出力

  • **入力**: xid (調査対象のトランザクションID)
  • **出力**: TransactionId (対象XIDの最上位親ID。自身が最上位の場合は自身を返す)

前提条件と不変条件

  • xid は TransactionXmin 以上である必要がある（Assertにより検証）。
  • 親トランザクションIDは子トランザクションIDよりも先に割り当てられるという慣習に基づき、親XIDは子XIDより先行（TransactionIdPrecedes）していなければならない。

アルゴリズムまたは振る舞い

1. parentXid を入力 xid で初期化する。 2. parentXid が有効である限り、以下のループを実行する。

  • 現在の parentXid を previousXid に退避する。
  • parentXid が TransactionXmin より古い場合はループを終了する。
  • SubTransGetParent(parentXid) を呼び出し、次の親IDを取得する。
  • 取得した親IDが previousXid より先行していない場合、データ構造の破損とみなし ERROR を発生させる。

    • 3. 最後に有効であった previousXid を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/storage/ipc/procarray.c
  • src/backend/storage/lmgr/lmgr.c
  • src/backend/utils/time/snapmgr.c
  • **呼び出し先**:
  • SubTransGetParent (親IDの取得)
  • TransactionIdFollowsOrEquals, TransactionIdIsValid, TransactionIdPrecedes (XID比較マクロ)
  • elog (エラー報告)

関連データ構造

  • SUBTRANS: サブトランザクションの親子関係を保持するSLRUベースのログ。

失敗モードと境界条件

  • **データ破損**: 親XIDが子XIDより先行していない場合、無限ループを防止するために ERROR をスローする。
  • **範囲外**: TransactionXmin より古いXIDについては、探索を打ち切る。

保守担当者向けメモ

  • 本関数は pg_subtrans のエントリを辿るため、SLRUのページ読み込みが発生する可能性がある。
  • 循環参照や不正なポインタによる無限ループを防ぐためのチェックが組み込まれている。
  • 実行中のトランザクション情報のみを扱うため、クラッシュリカバリ後の再起動時にはデータがゼロクリアされる点に注意すること。
