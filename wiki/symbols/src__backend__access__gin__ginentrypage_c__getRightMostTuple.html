getRightMostTuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:234

目的

GINエントリーツリーのページ内において、物理的に最後に格納されているインデックスタプルを取得します。

シグネチャまたは形状

static IndexTuple getRightMostTuple(Page page)

入力と出力

  • **入力**: Page page（対象となるGINエントリーツリーのページ）
  • **出力**: IndexTuple（ページ内の最大オフセット番号に対応するインデックスタプルへのポインタ）

前提条件と不変条件

  • 入力される page は有効なGINエントリーツリーのページである必要があります。
  • ページ内に少なくとも1つ以上のアイテムが存在することを前提としています。

アルゴリズムまたは振る舞い

1. PageGetMaxOffsetNumber(page) を呼び出し、ページ内の最大オフセット番号（maxoff）を取得します。 2. PageGetItemId(page, maxoff) を使用して、そのオフセットに対応するアイテム識別子を取得します。 3. PageGetItem を介して、該当するアイテムを IndexTuple 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginentrypage.c 内の複数箇所（ページ分割や内部ノードの構築処理など）から呼び出されています。
  • **呼び出し先**:
  • PageGetMaxOffsetNumber
  • PageGetItemId
  • PageGetItem

    • (いずれもPostgreSQLの標準的なページ操作マクロ)

関連データ構造

  • IndexTuple: GINインデックスのキーおよびポスティングリストを保持する構造体。
  • Page: PostgreSQLの標準的なページレイアウト構造。

失敗モードと境界条件

  • ページが空の場合、PageGetMaxOffsetNumber は FirstOffsetNumber - 1 を返すため、無効なアイテムへのアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数はエントリーツリーの管理において、右端のキーを特定するために頻繁に使用されます。
  • GINエントリーツリーは静的な性質を持つため、この関数は主にページ分割時や内部ノードの更新時に、親ノードへ伝播させるためのキーを取得する目的で利用されます。
