ExecInitCheck

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExpr.c:314

目的

チェック制約（qual）を実行可能な ExprState 形式へ変換・コンパイルします。

シグネチャまたは形状

ExprState *ExecInitCheck(List *qual, PlanState *parent)

入力と出力

  • **入力**:
  • qual: 評価対象となる制約式のリスト。
  • parent: 式の評価に関連付けられる PlanState ノード。
  • **出力**:
  • コンパイル済みの ExprState 構造体へのポインタ。qual が空（NIL）の場合は NULL を返します。

前提条件と不変条件

  • qual が NIL でない場合、IsA(qual, List) が真である必要があります。
  • ExecQual とは異なり、NULL 結果によるショートサーキットを行わないため、AND 演算として評価される必要があります。

アルゴリズムまたは振る舞い

1. qual が空リストであれば NULL を即座に返します。 2. qual が複数の要素を持つ場合、make_ands_explicit を使用して暗黙的な AND リストを明示的な AND 式に変換します。 3. 変換後の式を ExecInitExpr に渡し、実行可能な命令配列（ExprEvalStep）を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExpr.c 内の各所（制約の準備時など）。
  • **呼び出し先**:
  • make_ands_explicit
  • ExecInitExpr
  • (注: 呼び出し元・呼び出し先の詳細な網羅的リストは提供されていません)

関連データ構造

  • ExprState: 式評価の実行状態を保持する構造体。
  • PlanState: 実行プランのノード状態。
  • List: 式のリスト。

失敗モードと境界条件

  • qual が NIL の場合、エラーにはならず NULL を返します。
  • 式ツリーが非常に深い場合、内部で呼び出される ExecInitExpr 経由でスタックオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数は制約チェック専用の初期化パスです。ExecQual と異なり、NULL 値をスキップしない（AND 演算として厳密に評価する）必要がある点に注意してください。
  • 変更を加える際は、src/backend/executor/execExpr.c 内の ExecInitExpr および ExecPrepareCheck との整合性を確認してください。
