vacuumLeafPage

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/spgist/spgvacuum.c:125

目的

SP-GiST インデックスのリーフページ内において、不要になったタプルを削除し、タプルチェーンの再構成および REDIRECT タプルの整理を行う。

シグネチャまたは形状

static void vacuumLeafPage(spgBulkDeleteState *bds, Relation index, Buffer buffer, bool forPending)

入力と出力

  • **入力**:
  • bds: VACUUM 実行状態を保持する spgBulkDeleteState 構造体。
  • index: 対象のインデックスリレーション。
  • buffer: 処理対象のリーフページを含むバッファ。
  • forPending: 保留リスト処理中かどうかを示すフラグ。
  • **出力**: なし（副作用としてページ内容の更新および統計情報の更新を行う）。

前提条件と不変条件

  • 処理対象のページはリーフページであること。
  • bds->callback が提供されており、ヒープタプルの削除可否を判定できること。
  • ページ内のタプルチェーンリンクに矛盾がないこと（矛盾がある場合は elog(ERROR) を発生させる）。

アルゴリズムまたは振る舞い

1. **スキャンと識別**: ページ内の全タプルを走査し、SPGIST_LIVE タプルについてはコールバックを用いて削除対象か判定する。SPGIST_REDIRECT タプルについては、必要に応じて spgAddPendingTID を呼び出し保留リストへ追加する。 2. **削除対象の確認**: 削除対象タプルがない場合は即座に終了する。 3. **操作計画の策定**: ページ内のタプルチェーンを解析し、以下の操作を決定する。

  • toPlaceholder: 削除対象タプルを PLACEHOLDER に置換。
  • moveSrc/moveDest: ライブタプルの位置移動。
  • chainSrc/chainDest: チェーンリンクの再接続。

    • 4. **WAL 記録準備**: 決定した操作を spgxlogVacuumLeaf 構造体に格納し、WAL ログ出力およびページ修正の準備を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • spgvacuumpage (同ファイル内)
  • **呼び出し先**:
  • bds->callback (ヒープタプルの削除判定)
  • spgAddPendingTID (保留リストへの TID 追加)
  • PageGetItem, PageGetItemId, PageGetMaxOffsetNumber (ページ操作)

関連データ構造

  • spgBulkDeleteState: VACUUM の統計情報やコールバック状態を保持。
  • spgxlogVacuumLeaf: ページ修正内容を WAL に記録するための構造体。
  • SpGistLeafTuple: リーフページ内のタプル構造。

失敗モードと境界条件

  • **エラー**: ページ内のタプルチェーンリンクに矛盾（循環や範囲外のオフセット）が検出された場合、elog(ERROR) により処理を中断する。
  • **境界条件**: 削除対象タプルが 0 個の場合、ページ修正を行わずに早期リターンする。

保守担当者向けメモ

  • 本関数は、WAL リプレイ時にも同じロジックでページを再現できるよう、操作内容を xlrec に集約する設計となっている。
  • REDIRECT タプルの処理において、bds->myXmin を用いた判定を行っている。これは同時実行中の REINDEX CONCURRENTLY 等との整合性を保つための重要なロジックである。
