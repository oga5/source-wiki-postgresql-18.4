pg_cancel_backend

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/signalfuncs.c:135

目的

指定されたバックエンドプロセスに対してクエリキャンセルシグナル（SIGINT）を送信するSQL関数です。

シグネチャまたは形状

Datum pg_cancel_backend(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT32(0) を介してターゲットとなるバックエンドプロセスのPIDを受け取ります。
  • **出力**: 成功した場合は true を返し、失敗した場合はエラーをスローします。

前提条件と不変条件

  • 呼び出し元は、ターゲットプロセスに対してキャンセル操作を行うための適切な権限（スーパーユーザー権限、pg_signal_backend ロール、またはターゲットプロセスの所有者権限）を有している必要があります。

アルゴリズムまたは振る舞い

1. pg_signal_backend を呼び出し、ターゲットPIDに対して SIGINT を送信します。 2. 返り値に基づき、以下の権限エラーを判定して ereport(ERROR, ...) を発行します。

  • SIGNAL_BACKEND_NOSUPERUSER: スーパーユーザー権限が必要な操作に対する権限不足。
  • SIGNAL_BACKEND_NOAUTOVAC: pg_signal_autovacuum_worker ロール権限が必要な操作に対する権限不足。
  • SIGNAL_BACKEND_NOPERMISSION: 一般的な権限不足。

    • 3. 成功時（SIGNAL_BACKEND_SUCCESS）は true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます。
  • **呼び出し先**: pg_signal_backend を呼び出します。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約で使用されるマクロ。

失敗モードと境界条件

  • ターゲットPIDが存在しない、または権限が不足している場合、ereport によりトランザクションが中断されます。
  • 権限チェックの詳細は pg_signal_backend の実装に依存します。

保守担当者向けメモ

  • 本関数は SIGINT を送信するラッパーです。プロセス終了を目的とする場合は pg_terminate_backend を参照してください。
  • 権限エラーのメッセージは、ユーザーがなぜキャンセルできなかったかを理解できるよう、特定のロール属性や権限要件を明示する設計になっています。
