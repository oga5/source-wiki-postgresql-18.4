btree_xlog_newroot

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtxlog.c:940

目的

B-tree インデックスにおいて、新しいルートページが作成された際の WAL リプレイ処理を行う。

シグネチャまたは形状

static void btree_xlog_newroot(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象の WAL レコード）
  • **出力**: なし（副作用としてバッファの更新およびメタページの更新を行う）

前提条件と不変条件

  • record から xl_btree_newroot 構造体が正しく取得できること。
  • リプレイ対象のバッファが XLogInitBufferForRedo により初期化可能であること。

アルゴリズムまたは振る舞い

1. XLogInitBufferForRedo で新しいルートページ用のバッファを初期化し、_bt_pageinit でページを初期化する。 2. ページヘッダー（BTPageOpaque）を設定する。BTP_ROOT フラグを立て、レベルを xlrec->level に設定する（レベルが 0 の場合は BTP_LEAF も設定）。 3. レベルが 0 より大きい場合、XLogRecGetBlockData からデータを取得し、_bt_restore_page でページ内容を復元する。その後、_bt_clear_incomplete_split を呼び出し、左側の子ページの不完全な分割フラグをクリアする。 4. ページの LSN を更新し、バッファをダーティとしてマークして解放する。 5. 最後に _bt_restore_meta を呼び出し、インデックスのメタページを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_redo（src/backend/access/nbtree/nbtxlog.c:1061）
  • **呼び出し先**:
  • XLogInitBufferForRedo, XLogRecGetData, XLogRecGetBlockData
  • _bt_pageinit, BTPageGetOpaque
  • _bt_restore_page, _bt_clear_incomplete_split, _bt_restore_meta
  • PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer

関連データ構造

  • xl_btree_newroot: WAL レコードの構造体。
  • BTPageOpaque: B-tree ページヘッダーのメタデータ。

失敗モードと境界条件

  • XLogInitBufferForRedo が失敗した場合、リカバリ処理は中断される。
  • xlrec->level が 0 の場合、ページはリーフとして扱われ、ページ復元処理（_bt_restore_page）はスキップされる。

保守担当者向けメモ

  • 本関数は物理的なページ再構築とメタページの更新を分離して処理している。
  • _bt_restore_meta に渡される引数 2 は、WAL レコード内のメタデータブロックのインデックスを指す。
