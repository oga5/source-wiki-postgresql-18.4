BumpBlockInit

定義

  • 種別: static inline function
  • 定義箇所: src/backend/utils/mmgr/bump.c:534

目的

バンプアロケータにおける新しいメモリブロックの初期化を行い、割り当て可能な領域の境界を設定する。

シグネチャまたは形状

static inline void BumpBlockInit(BumpContext *context, BumpBlock *block, Size blksize)

入力と出力

  • **入力**:
  • context: 初期化対象のブロックが所属する BumpContext へのポインタ。
  • block: 初期化対象の BumpBlock 構造体へのポインタ。
  • blksize: ブロック全体のサイズ（バイト単位）。
  • **出力**: なし（副作用として block の各フィールドを更新）。

前提条件と不変条件

  • block は、少なくとも Bump_BLOCKHDRSZ を含む十分なメモリ領域を確保済みであること。
  • MEMORY_CONTEXT_CHECKING が有効な場合、block->context に context が設定される。

アルゴリズムまたは振る舞い

1. MEMORY_CONTEXT_CHECKING が定義されている場合、block->context に context を代入する。 2. block->freeptr を、ブロックヘッダー直後のアドレス（block + Bump_BLOCKHDRSZ）に設定する。 3. block->endptr を、ブロックの末尾アドレス（block + blksize）に設定する。 4. VALGRIND_MAKE_MEM_NOACCESS を呼び出し、割り当てられていない領域（ヘッダー以降の全領域）を Valgrind に対してアクセス不可としてマークする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/bump.c:198
  • src/backend/utils/mmgr/bump.c:461
  • **呼び出し先**:
  • VALGRIND_MAKE_MEM_NOACCESS（マクロ）

関連データ構造

  • BumpContext: バンプアロケータのコンテキストメタデータ。
  • BumpBlock: メモリブロックの管理構造体。

失敗モードと境界条件

  • blksize が Bump_BLOCKHDRSZ 未満の場合、freeptr が endptr を超える可能性がある（呼び出し側で適切なサイズが保証されている必要がある）。

保守担当者向けメモ

  • 本関数は static inline であり、パフォーマンスが重視される割り当てパスで使用される。
  • Valgrind によるメモリチェックをサポートしているため、ブロックの境界変更時には必ず VALGRIND_MAKE_MEM_NOACCESS の引数が整合していることを確認すること。
