LookupFuncWithArgs

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_func.c:2205

目的

指定されたオブジェクト型（関数、プロシージャ、集約など）と引数リストに基づき、システムカタログから対応するオブジェクトのOIDを検索・検証する。

シグネチャまたは形状

Oid LookupFuncWithArgs(ObjectType objtype, ObjectWithArgs *func, bool missing_ok)

入力と出力

  • **入力**:
  • objtype: 検索対象のオブジェクト種別（OBJECT_AGGREGATE, OBJECT_FUNCTION, OBJECT_PROCEDURE, OBJECT_ROUTINE）。
  • func: オブジェクト名と引数型リストを含む ObjectWithArgs 構造体。
  • missing_ok: 見つからない場合にエラーを投げるか（false）、InvalidOid を返すか（true）。
  • **出力**:
  • 成功時: 特定されたオブジェクトの Oid。
  • 失敗時: InvalidOid（missing_ok が true の場合）、またはエラー発生。

前提条件と不変条件

  • objtype は OBJECT_AGGREGATE, OBJECT_FUNCTION, OBJECT_PROCEDURE, OBJECT_ROUTINE のいずれかであること。
  • 引数の数が FUNC_MAX_ARGS を超える場合はエラーとなる。

アルゴリズムまたは振る舞い

1. **引数型の解決**: func->objargs 内の各 TypeName を LookupTypeNameOid でOIDに変換する。 2. **一次検索**: LookupFuncNameInternal を呼び出し、入力引数のみに基づいた候補を検索する。 3. **プロシージャ/ルーチンの追加検索**: objtype がプロシージャまたはルーチンであり、かつパラメータモード指定がない場合、全引数を用いた再検索を行い、曖昧さがないか確認する。 4. **型検証**: 検索結果のOIDに対し、get_func_prokind を用いて objtype との整合性を検証する。不一致の場合はエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/functioncmds.c
  • src/backend/commands/opclasscmds.c
  • **呼び出し先**:
  • LookupTypeNameOid
  • LookupFuncNameInternal
  • get_func_prokind
  • func_signature_string

関連データ構造

  • ObjectWithArgs: 検索対象の関数名と引数リストを保持する構造体。
  • ObjectType: オブジェクトの種類を定義する列挙型。
  • FuncLookupError: 検索時のエラー状態を保持する列挙型。

失敗モードと境界条件

  • **引数過多**: FUNC_MAX_ARGS を超える引数は ERRCODE_TOO_MANY_ARGUMENTS エラー。
  • **型不一致**: 検索されたオブジェクトの種別が objtype と合致しない場合（例：関数として検索したがプロシージャだった場合）、ERRCODE_WRONG_OBJECT_TYPE エラー。
  • **曖昧な検索**: 複数の候補が見つかり、解決できない場合は InvalidOid を返すかエラーとなる。

保守担当者向けメモ

  • 本関数は、歴史的な経緯から OBJECT_FUNCTION が集約関数やウィンドウ関数を含むことを許容しているが、プロシージャについては厳密な型チェックを行う。この挙動は switch 文による検証ロジックに依存している。
