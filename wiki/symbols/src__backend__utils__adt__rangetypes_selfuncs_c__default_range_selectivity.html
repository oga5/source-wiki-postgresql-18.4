default_range_selectivity

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_selfuncs.c:66

目的

統計情報が利用できない、または計算が困難な場合に、範囲型演算子に対するデフォルトの選択率（Selectivity）を返します。

シグネチャまたは形状

static double default_range_selectivity(Oid operator)

入力と出力

  • **入力**: operator (Oid) - 評価対象の範囲演算子。
  • **出力**: double - 推定された選択率（0.0 から 1.0 の範囲）。

前提条件と不変条件

  • 渡される operator は範囲型に関連する演算子である必要があります。
  • 統計情報が不足している状況下でのフォールバックとして使用されます。

アルゴリズムまたは振る舞い

演算子の種類に応じて以下の固定値を返します。

  • && (Overlap): 0.01
  • @> (Contains) / <@ (Contained): 0.005
  • @> (Contains Element) / <@ (Element Contained): DEFAULT_RANGE_INEQ_SEL
  • 比較演算子（<, <=, >, >=, <<, >>, &<, &>）: DEFAULT_INEQ_SEL
  • 上記以外: 0.01

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の複数箇所（src/backend/utils/adt/rangetypes_selfuncs.c の 127, 135, 160, 221, 338 行目など）から、統計情報を用いた計算のフォールバックとして呼び出されます。
  • **呼び出し先**: 本関数は計算ロジックのみであり、他の関数を呼び出しません。

関連データ構造

  • Oid: PostgreSQL のシステムカタログ識別子。
  • DEFAULT_RANGE_INEQ_SEL, DEFAULT_INEQ_SEL: 選択率のデフォルト値を定義するマクロ（utils/selfuncs.h 等で定義）。

失敗モードと境界条件

  • 未知の演算子が渡された場合、デフォルト値として 0.01 を返します。

保守担当者向けメモ

  • 本関数はハードコードされた定数を返すため、統計情報の精度向上よりも、プランナの安全なフォールバックとしての役割が主です。
  • 新しい範囲演算子を追加した場合は、本関数の switch 文にも適切なデフォルト値を追記してください。
