IsSystemRelation

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/catalog.c:73

目的

指定されたリレーションがシステムカタログまたはTOASTテーブルであるかを判定します。

シグネチャまたは形状

bool IsSystemRelation(Relation relation)

入力と出力

  • **入力**: Relation relation（判定対象のリレーション記述子）
  • **出力**: bool（システムカタログまたはTOASTテーブルであれば true、それ以外は false）

前提条件と不変条件

  • 本関数はカタログアクセスを行いません。ロック取得中や再帰的な呼び出しにおいても安全に使用可能です。
  • 判定ロジックは、OIDの範囲比較やハードコードされたOIDリストに基づいています。

アルゴリズムまたは振る舞い

1. RelationGetRelid(relation) を呼び出し、対象リレーションのOIDを取得します。 2. 取得したOIDとリレーションのメタデータ（relation->rd_rel）を引数として IsSystemClass を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/index/genam.c
  • src/backend/catalog/catalog.c
  • src/backend/catalog/index.c
  • src/backend/commands/cluster.c
  • src/backend/commands/indexcmds.c
  • **呼び出し先**:
  • RelationGetRelid (マクロ)
  • IsSystemClass (同ファイル内の関数)

関連データ構造

  • Relation: リレーション記述子。本関数はこの構造体からOIDおよびリレーション情報を抽出します。

失敗モードと境界条件

  • relation がNULLである場合の動作は定義されていません（呼び出し元で有効なポインタが渡されることが前提です）。

保守担当者向けメモ

  • 本関数はシステムカタログの判定における標準的なインターフェースです。判定ロジックを変更する場合は、IsSystemClass の定義も併せて確認してください。
  • カタログアクセスを禁止するという設計方針を維持してください。
