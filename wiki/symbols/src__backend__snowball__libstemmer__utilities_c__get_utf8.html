get_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/utilities.c:71

目的

UTF-8 エンコードされた文字列から、現在のカーソル位置にある 1 文字をデコードし、そのコードポイントを抽出してカーソルを進める。

シグネチャまたは形状

static int get_utf8(const symbol * p, int c, int l, int * slot)

入力と出力

  • **入力**:
  • p: 文字列バッファへのポインタ。
  • c: 現在のカーソル位置（インデックス）。
  • l: バッファの長さ（終端位置）。
  • slot: デコードされたコードポイントを格納する整数型ポインタ。
  • **出力**:
  • 戻り値: 読み込んだバイト数（1〜4）。範囲外の場合は 0 を返す。
  • slot: デコード結果のコードポイント値。

前提条件と不変条件

  • c が l 未満であること。
  • slot は有効な整数領域を指していること。
  • 入力文字列 p は有効な UTF-8 シーケンスであると仮定される。

アルゴリズムまたは振る舞い

1. c >= l の場合、0 を返して終了する。 2. 最初のバイト b0 を読み込み、その値に応じて UTF-8 のバイト長を判定する。

  • b0 < 0xC0: 1バイト文字として slot に格納し、1 を返す。
  • b0 < 0xE0: 2バイト文字として後続バイトを結合し、2 を返す。
  • b0 < 0xF0: 3バイト文字として後続2バイトを結合し、3 を返す。
  • その他: 4バイト文字として後続3バイトを結合し、4 を返す。

    • 3. 各ステップで、バッファの終端 l に達した場合はその時点までの結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/utilities.c 内の 120 行目および 144 行目から呼び出されている。
  • **呼び出し先**: 本関数は標準的な算術演算およびビット演算のみを行い、他の関数を呼び出さない。

関連データ構造

  • symbol: 文字列を構成する型（header.h で定義）。
  • SN_env: ステミング環境を保持する構造体（本関数は直接受け取らないが、呼び出し元で z->p, z->c, z->l が使用される）。

失敗モードと境界条件

  • c >= l: 文字列の終端に達している場合、0 を返し slot は更新されない。

保守担当者向けメモ

  • 本関数は UTF-8 のデコードを行う低レベルユーティリティである。
  • _U サフィックスが付く関数群（in_grouping_U 等）と組み合わせて使用される。
  • 戻り値は読み込んだバイト数であり、文字数ではない点に注意すること。
