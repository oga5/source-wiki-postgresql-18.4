gbt_enum_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_enum.c:114

目的

GiST インデックス内の列挙型（ENUM）エントリから、インデックスキーをフェッチ（取得）します。

シグネチャまたは形状

Datum gbt_enum_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、gbt_num_fetch が返すフェッチされたインデックスキーを返します。

前提条件と不変条件

  • tinfo（gbtree_ninfo 構造体）が、ENUM 型の比較関数やサイズ情報を含む設定テーブルとして適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. 取得した entry と、ENUM 型用に定義された tinfo を引数として、btree_utils_num.c で定義されている汎用関数 gbt_num_fetch を呼び出します。 3. gbt_num_fetch の戻り値をそのまま PG_RETURN_POINTER を介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構から呼び出されます（直接的な呼び出し元関数の詳細はソースコード内に明示されていません）。
  • **呼び出し先**: gbt_num_fetch（contrib/btree_gist/btree_utils_num.c 内のユーティリティ関数）。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • tinfo: gbtree_ninfo 構造体。ENUM 型の GiST 操作に必要なメタデータを提供します。

失敗モードと境界条件

  • entry が NULL の場合、あるいは tinfo が正しく設定されていない場合、gbt_num_fetch の内部挙動に依存します。

保守担当者向けメモ

  • 本関数は ENUM 型専用のラッパーであり、実質的なロジックは gbt_num_fetch に委譲されています。
  • PG_FUNCTION_INFO_V1(gbt_enum_fetch) マクロにより、PostgreSQL の関数マネージャーに登録されています。
