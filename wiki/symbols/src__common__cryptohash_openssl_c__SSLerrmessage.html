SSLerrmessage

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash_openssl.c:102

目的

OpenSSLのエラーコードを、人間が読み取り可能なエラーメッセージ文字列に変換するラッパー関数です。

シグネチャまたは形状

static const char *SSLerrmessage(unsigned long ecode)

入力と出力

  • **入力**: ecode (unsigned long) - OpenSSLの ERR_get_error() 等で取得されるエラーコード。
  • **出力**: const char * - エラー内容を示す文字列。ecode が 0 の場合は NULL を返します。

前提条件と不変条件

  • ecode が 0 の場合、関数は直ちに NULL を返します。
  • OpenSSLの ERR_reason_error_string() を内部で呼び出しており、OpenSSLライブラリの初期化状態に依存します。

アルゴリズムまたは振る舞い

1. 引数 ecode が 0 かどうかを判定し、0 であれば NULL を返します。 2. 0 以外の場合、OpenSSLの ERR_reason_error_string(ecode) を呼び出し、その結果を返します。 3. ERR_reason_error_string が NULL を返す可能性があることを前提としており、呼び出し元で適切なフォールバック処理を行う設計となっています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure-openssl.c 内の複数の箇所から、OpenSSLエラー発生時にログ出力やエラー報告のために呼び出されています。
  • **呼び出し先**: OpenSSLライブラリの ERR_reason_error_string を呼び出します。

関連データ構造

  • なし（OpenSSLの unsigned long 型エラーコードを直接扱います）。

失敗モードと境界条件

  • ecode が有効なOpenSSLエラーコードではない場合、ERR_reason_error_string は NULL を返すことがあります。
  • 本関数自体はエラーコードの検証を行わず、OpenSSLのAPI結果をそのまま透過的に返します。

保守担当者向けメモ

  • 本関数は static であり、src/common/cryptohash_openssl.c 内でのみ定義されていますが、src/backend/libpq/be-secure-openssl.c から参照されています（ビルド構成上の依存関係に注意してください）。
  • OpenSSLのエラーキューが空の場合（ecode == 0）の挙動は明示的に定義されています。
