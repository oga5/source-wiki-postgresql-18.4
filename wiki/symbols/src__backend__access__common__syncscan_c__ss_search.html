ss_search

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/syncscan.c:190

目的

共有メモリ上のLRUリストから、指定されたリレーションのスキャン位置を検索、更新、または新規作成する。

シグネチャまたは形状

static BlockNumber ss_search(RelFileLocator relfilelocator, BlockNumber location, bool set)

入力と出力

  • **入力**:
  • relfilelocator: 検索対象のリレーション識別子。
  • location: set が真の場合に更新するブロック番号。
  • set: リレーションが見つかった場合に location で値を更新するかどうかを示すフラグ。
  • **出力**:
  • 検索または更新後のリレーションに対応する現在の BlockNumber。

前提条件と不変条件

  • scan_locations（共有メモリ上の構造体）が初期化済みであること。
  • 本関数を呼び出す前に、呼び出し元で SyncScanLock を排他獲得していること。

アルゴリズムまたは振る舞い

1. scan_locations->head からリストを走査し、RelFileLocatorEquals を用いて対象リレーションを検索する。 2. リストの末尾に到達しても見つからない場合、末尾の要素を再利用して対象リレーションを登録し、指定された location を設定する。 3. リレーションが見つかった場合、set が真であれば location を更新する。 4. 処理対象となったエントリをリストの先頭（head）へ移動させ、LRU順序を維持する。 5. 最終的な location を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ss_get_location (src/backend/access/common/syncscan.c:259)
  • ss_report_location (src/backend/access/common/syncscan.c:313)
  • **呼び出し先**:
  • RelFileLocatorEquals (マクロ)

関連データ構造

  • ss_scan_locations_t: スキャン位置を保持するメイン構造体。
  • ss_lru_item_t: LRUリストの各要素。

失敗モードと境界条件

  • リストが空の場合や、全要素が別リレーションで埋まっている場合は、末尾の要素が強制的に上書きされる。
  • set が偽の場合、既存の location 値は変更されず、検索とLRU順序の更新のみが行われる。

保守担当者向けメモ

  • 本関数は SyncScanLock の保護下で実行されることを前提としている。
  • リストの先頭への移動処理において、ポインタの付け替え（unlink/link）が正しく行われるよう注意すること。
