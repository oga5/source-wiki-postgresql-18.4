sepgsql_avc_check_perms_label

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:336

目的

SELinux のセキュリティラベルに基づき、主体（クライアント）から対象オブジェクトへのアクセス権限を検証し、必要に応じて監査ログの記録やエラー送出を行う。

シグネチャまたは形状

bool sepgsql_avc_check_perms_label(const char *tcontext, uint16 tclass, uint32 required, const char *audit_name, bool abort_on_violation)

入力と出力

  • **入力**:
  • tcontext: 対象オブジェクトのセキュリティラベル（NULL の場合は未ラベルとして扱う）。
  • tclass: オブジェクトクラス。
  • required: 要求される権限ビットマスク。
  • audit_name: 監査ログに記録する識別名。
  • abort_on_violation: 権限拒否時にエラーを発生させるかどうかのフラグ。
  • **出力**:
  • bool: 権限が許可された場合は true、拒否された場合は false。

前提条件と不変条件

  • sepgsql_get_client_label() により取得した主体ラベルを使用する。
  • ポリシー更新による不整合を防ぐため、sepgsql_avc_check_valid() をループ条件に含めた do-while 構造で実行される必要がある。
  • abort_on_violation が true で権限が拒否された場合、関数は戻り値を返さず ereport(ERROR, ...) により処理を中断する。

アルゴリズムまたは振る舞い

1. sepgsql_avc_check_valid() でキャッシュの有効性を確認する。 2. sepgsql_avc_lookup() を呼び出し、キャッシュから権限情報を取得する（tcontext が NULL の場合は sepgsql_avc_unlabeled() を使用）。 3. required 権限とキャッシュされた allowed 権限を比較し、denied（拒否された権限）を算出する。 4. 監査対象の権限 audited を決定する（デバッグモード時は全権限、通常時は auditdeny または auditallow に基づく）。 5. 権限拒否時、パーミッシブモードまたはドメインがパーミッシブであれば、キャッシュを更新して許可扱いとする。それ以外は result = false とする。 6. ポリシーが更新された場合はループを再試行する。 7. 監査が必要な場合、sepgsql_audit_log() を呼び出す。 8. 最後に abort_on_violation に基づき、必要に応じてエラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/database.c, label.c, proc.c, relation.c の各関数から参照されている。
  • **呼び出し先**: sepgsql_get_client_label, sepgsql_avc_check_valid, sepgsql_avc_lookup, sepgsql_avc_unlabeled, sepgsql_get_debug_audit, sepgsql_getenforce, sepgsql_get_mode, sepgsql_audit_log, ereport。

関連データ構造

  • avc_cache: 権限判定結果を保持する構造体。

失敗モードと境界条件

  • tcontext が NULL の場合、sepgsql_avc_unlabeled() が返すラベルで代替される。
  • sepgsql_getenforce() が無効（パーミッシブモード）の場合、権限違反があっても result は true となり、エラーは発生しない。
  • audit_name に SEPGSQL_AVC_NOAUDIT が指定された場合、監査ログは出力されない。

保守担当者向けメモ

  • 本関数はシステムコールを跨ぐ可能性があるため、必ず do-while ループ内で sepgsql_avc_check_valid() を使用すること。
  • 監査ログの出力条件は sepgsql_get_mode() が SEPGSQL_MODE_INTERNAL でない場合に限定されている点に注意。
