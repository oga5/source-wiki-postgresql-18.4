r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_portuguese.c:509

目的

ポルトガル語のステミング処理において、単語内の領域（RV, R1, R2）を特定し、接尾辞削除の条件判定に使用する境界位置を計算する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング対象の単語データおよび状態を保持する構造体）
  • **出力**: int（常に 1 を返す）
  • **副作用**: z->I[0]（R2の開始位置）、z->I[1]（R1の開始位置）、z->I[2]（RVの開始位置）を更新する。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • g_v（母音グループ）が定義されている環境下で実行される。

アルゴリズムまたは振る舞い

1. **初期化**: z->I[0], z->I[1], z->I[2] を単語の末尾位置 z->l で初期化する。 2. **RV領域の特定**:

  • 母音の直後の子音、または単語先頭から2番目の母音の直後を探し、その位置を z->I[2] に格納する。

    • 3. **R1/R2領域の特定**:

  • 母音と子音の遷移パターンに基づき、R1（z->I[1]）および R2（z->I[0]）の開始位置を順次計算する。
  • 領域の特定に失敗した場合は、初期値（単語末尾）が維持される。

呼び出し元と呼び出し先

  • **呼び出し元**: portuguese_UTF_8_stem（直接の呼び出し元であることは文脈から推測されるが、本ソース excerpt 内には明示的な呼び出し記述なし）。
  • **呼び出し先**: in_grouping_U, out_grouping_U, skip_utf8（いずれも Snowball ランタイム関数）。

関連データ構造

  • struct SN_env: ステミングの状態管理用構造体。I 配列に領域境界を保持する。

失敗モードと境界条件

  • 単語が短すぎる場合や、母音・子音のパターンが適合しない場合、領域境界は単語末尾に設定される。
  • UTF-8 文字列の走査において skip_utf8 が失敗した場合、処理を中断し lab0 へ遷移する。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。ロジックの変更は .sbl ソースファイルを修正して再生成すること。
  • z->I 配列のインデックス（0, 1, 2）は、後続の接尾辞削除関数で R2, R1, RV として参照されるため、インデックスの変更は厳禁。
