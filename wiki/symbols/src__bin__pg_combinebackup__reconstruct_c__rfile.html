rfile

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_combinebackup/reconstruct.c:37

目的

再構築プロセスにおいて、ディスク上の特定のファイルを読み込み、データブロックを供給するためのメタデータおよびファイル記述子を保持する。

シグネチャまたは形状

typedef struct rfile
{
	char	   *filename;
	int			fd;
	size_t		header_length;
	unsigned	num_blocks;
	BlockNumber *relative_block_numbers;
	unsigned	truncation_block_length;
	unsigned	num_blocks_read;
	off_t		highest_offset_read;
} rfile;

入力と出力

本構造体はデータ構造であり、関数ではないため直接的な入出力はない。ただし、make_incremental_rfile や make_rfile によって初期化され、read_block や read_bytes の引数として渡されることで、ファイルシステムからのデータ読み込みを制御する。

前提条件と不変条件

  • fd は有効なファイル記述子である必要がある。
  • relative_block_numbers は、再構築対象のファイルに関連付けられたブロック番号の配列を指す必要がある。

アルゴリズムまたは振る舞い

再構築計画に基づき、特定のバックアップファイルから必要なブロックを特定・抽出する。num_blocks_read および highest_offset_read を更新することで、読み込み進捗を追跡する。

呼び出し元と呼び出し先

  • **呼び出し元**: make_incremental_rfile, make_rfile, find_reconstructed_block_length, read_bytes, read_block, write_reconstructed_file 等が本構造体を操作する。
  • **呼び出し先**: 本構造体自体はデータ構造であり、特定の関数を呼び出すことはない。

関連データ構造

  • BlockNumber (storage/block.h): ブロック番号の管理に使用。
  • sourcemap: rfile のポインタ配列として定義され、出力ファイルの各ブロックの供給元を管理する。

失敗モードと境界条件

  • ファイルオープン失敗時や、読み込み中に fd が無効になった場合の挙動については、本構造体定義からは不明。
  • num_blocks が実際のファイルサイズと整合しない場合の動作は未定義。

保守担当者向けメモ

  • 本構造体は pg_combinebackup における再構築の基盤となる。
  • relative_block_numbers のメモリ管理（割り当てと解放）が適切に行われているか、各関数でのライフサイクルを確認すること。
