write_target_range

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:90

目的

ターゲットデータディレクトリ内のファイルに対し、指定されたオフセットからデータを書き込みます。

シグネチャまたは形状

void write_target_range(char *buf, off_t begin, size_t size)

入力と出力

  • buf: 書き込むデータが格納されたバッファへのポインタ。
  • begin: 書き込みを開始するターゲットファイル内のオフセット。
  • size: 書き込むデータのバイト数。
  • 戻り値: なし（成功時）。失敗時は pg_fatal によりプロセスが終了します。

前提条件と不変条件

  • dstfd（ターゲットファイルのファイル記述子）および dstpath（ファイルパス）が、事前に open_target_file 等によって適切に設定されている必要があります。
  • dry_run モードが有効な場合、実際の書き込みは行われず、進捗報告のみが更新されます。

アルゴリズムまたは振る舞い

1. fetch_done を更新し、progress_report(false) を呼び出して進捗を表示します。 2. dry_run が有効であれば直ちに終了します。 3. lseek を使用して、ターゲットファイルの指定された begin オフセットへシークします。失敗した場合は pg_fatal を発行します。 4. write システムコールをループ内で呼び出し、buf からターゲットファイルへデータを書き込みます。

  • 書き込みが中断された場合、残りのデータを書き込むまでループを継続します。
  • write がエラーを返し、かつ errno が設定されていない場合は ENOSPC（ディスク容量不足）とみなして pg_fatal を発行します。

    • 5. 書き込み完了後、ファイル記述子 dstfd は開いたまま保持されます。

関連データ構造

  • dstfd（グローバル変数、ファイル記述子）
  • dstpath（グローバル変数、ファイルパス）
  • fetch_done（グローバル変数、進捗管理用）
  • dry_run（グローバル変数、実行モード制御）

失敗モードと境界条件

  • lseek 失敗: ターゲットファイルへのシークができない場合、致命的エラーとして終了します。
  • write 失敗: 書き込みエラーが発生した場合、errno に基づくエラーメッセージと共に致命的エラーとして終了します。
  • ディスク容量不足: write がエラーを返し errno が 0 の場合、ENOSPC として扱われます。

保守担当者向けメモ

  • 本関数はファイル記述子を閉じないため、呼び出し側で適切なタイミングで close_target_file を呼び出す必要があります。
  • dry_run モードのチェックは必須であり、新しいロジックを追加する際もこの慣習を維持してください。
