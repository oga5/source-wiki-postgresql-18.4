getObjectIdentity

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaddress.c:4823

目的

指定された ObjectAddress に対応する、SQLで利用可能なオブジェクトの識別子（名前）を文字列として生成します。

シグネチャまたは形状

char *getObjectIdentity(const ObjectAddress *object, bool missing_ok)

入力と出力

  • **入力**:
  • object: 識別対象となる ObjectAddress 構造体へのポインタ。
  • missing_ok: オブジェクトが見つからない場合にエラーを投げるか（false）、NULL を返すか（true）を制御するフラグ。
  • **出力**:
  • オブジェクトを特定するための文字列（char *）。メモリは呼び出し元で管理する必要があります。

前提条件と不変条件

  • ObjectAddress 構造体は、有効な classId および objectId を保持している必要があります。
  • 内部的にシステムカタログを検索するため、適切なトランザクションコンテキスト内で呼び出される必要があります。

アルゴリズムまたは振る舞い

内部関数 getObjectIdentityParts を呼び出し、引数に NULL を渡すことで、オブジェクトの完全な識別子文字列を取得します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/database.c
  • contrib/sepgsql/dml.c
  • contrib/sepgsql/proc.c
  • contrib/sepgsql/relation.c
  • **呼び出し先**:
  • getObjectIdentityParts（同ファイル内の内部関数）

関連データ構造

  • ObjectAddress: オブジェクトのカタログOID、行OID、サブIDを保持する構造体。

失敗モードと境界条件

  • missing_ok が false の場合、オブジェクトが存在しないとエラーが発生します。
  • メモリ不足時にはメモリ割り当てエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は getObjectIdentityParts のラッパーです。識別子の生成ロジックを変更する場合は getObjectIdentityParts を参照してください。
  • sepgsql などの拡張モジュールで監査ログ生成のために頻繁に利用されています。
