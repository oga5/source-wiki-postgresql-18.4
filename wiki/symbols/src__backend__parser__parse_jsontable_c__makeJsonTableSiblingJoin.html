makeJsonTableSiblingJoin

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_jsontable.c:531

目的

JSON_TABLE 構文の解析において、ネストされたカラム構造を処理する際に、2つの JsonTablePlan ノードを結合するための JsonTableSiblingJoin ノードを生成します。

シグネチャまたは形状

JsonTablePlan *makeJsonTableSiblingJoin(JsonTablePlan *lplan, JsonTablePlan *rplan)

入力と出力

  • **入力**:
  • lplan: 結合の左側に配置される JsonTablePlan。
  • rplan: 結合の右側に配置される JsonTablePlan。
  • **出力**:
  • 結合された JsonTablePlan 構造体へのポインタ。

前提条件と不変条件

  • 入力される lplan および rplan は有効な JsonTablePlan である必要があります。
  • 生成されるノードの plan.type は T_JsonTableSiblingJoin に設定されます。

アルゴリズムまたは振る舞い

1. makeNode(JsonTableSiblingJoin) を使用して新しい結合ノードを割り当てます。 2. ノードの型を T_JsonTableSiblingJoin に設定します。 3. 引数として渡された lplan を join->lplan に、rplan を join->rplan にそれぞれ代入します。 4. 構築したノードを JsonTablePlan 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_jsontable.c 内の transformJsonTableNestedColumns 関数から呼び出されます（ソース内の参照情報に基づく）。
  • **呼び出し先**:
  • makeNode マクロ（nodes/makefuncs.h で定義）を呼び出し、ノードメモリを確保します。

関連データ構造

  • JsonTablePlan: JSON_TABLE の実行計画を構成する基底構造体。
  • JsonTableSiblingJoin: NESTED 句などで使用される、兄弟関係にあるパスを結合するためのノード構造体。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性があります。
  • 入力引数に NULL が渡された場合の挙動については、本ソースコード内では明示的なチェックは行われていません。

保守担当者向けメモ

  • 本関数は NESTED 句の再帰的な処理において、プランツリーを構築するために不可欠です。
  • JsonTableSiblingJoin ノードの構造変更が必要な場合は、本関数と実行エンジン側の対応箇所を併せて確認してください。
