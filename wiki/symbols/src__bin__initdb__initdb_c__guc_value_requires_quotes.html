guc_value_requires_quotes

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/initdb.c:643

目的

PostgreSQL の設定値（GUC値）が、postgresql.conf 等の構成ファイルに記述される際に引用符（クォート）を必要とするかどうかを判定する。

シグネチャまたは形状

static bool guc_value_requires_quotes(const char *guc_value)

入力と出力

  • **入力**: guc_value (文字列へのポインタ)
  • **出力**: bool 型。引用符が必要な場合は true、不要（識別子または数値として解釈可能）な場合は false を返す。

前提条件と不変条件

  • ctype.h のマクロは使用しない（過剰な文字を許容する可能性があるため）。
  • 空文字列は常に引用符が必要とみなされる。

アルゴリズムまたは振る舞い

以下の条件に基づき判定を行う： 1. **空文字列**: true を返す。 2. **識別子（英字開始）**: 先頭が英字で、かつ全体が英数字のみで構成されている場合は false。それ以外は true。 3. **数値（数字開始）**: 先頭が数字で、かつ数字の後に単位（英字）のみが続く場合は false。それ以外は true。 4. **その他**: 上記に該当しない場合は true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/initdb.c 内の replace_guc_value 関数（539行目）から呼び出されている。
  • **呼び出し先**: strchr, strspn, strlen などの標準 C ライブラリ関数。

関連データ構造

  • 特になし。内部で LETTERS および DIGITS マクロを定義して使用している。

失敗モードと境界条件

  • guc_value が NULL の場合の挙動は定義されていない。
  • 記号や特殊文字を含む文字列は、識別子や数値のルールに適合しないため、常に true（引用符が必要）と判定される。

保守担当者向けメモ

  • 本関数は initdb における設定ファイル生成時の文字列エスケープ処理の一部である。
  • 判定ロジックは厳格に定義されているため、新しい設定値の形式をサポートする場合は、識別子や数値の定義（LETTERS, DIGITS）との整合性に注意すること。
