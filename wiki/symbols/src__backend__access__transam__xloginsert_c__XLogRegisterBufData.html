XLogRegisterBufData

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xloginsert.c:404

目的

WALレコード構築中に、既に登録済みのバッファに関連付けられたデータチャンクを追加するために使用されます。

シグネチャまたは形状

void XLogRegisterBufData(uint8 block_id, const void *data, uint32 len)

入力と出力

  • block_id: データを関連付ける対象バッファの識別子。
  • data: 追加するデータへのポインタ。
  • len: 追加するデータのバイト長。
  • 出力: なし（副作用として内部状態を更新）。

前提条件と不変条件

  • XLogBeginInsert() が呼び出されていること（begininsert_called が真であること）。
  • 指定された block_id が XLogRegisterBuffer() によって既に登録済みであること。
  • 登録されるデータ長 len およびバッファごとの累積データ長が UINT16_MAX を超えないこと。
  • システム全体のデータセグメント数 num_rdatas が max_rdatas を超えないこと。

アルゴリズムまたは振る舞い

1. registered_buffers 配列から block_id に対応する構造体を取得し、登録済みであることを確認します。 2. 登録可能なデータセグメント数およびデータ長の制限（UINT16_MAX）をチェックし、超過している場合はエラーを報告します。 3. rdatas 配列から新しい XLogRecData エントリを確保し、データポインタと長さを設定します。 4. 指定されたバッファの rdata_tail を更新して、新しいエントリをチェーンの末尾に連結します。 5. バッファの累積データ長 rdata_len を更新します。

呼び出し元と呼び出し先

  • 呼び出し元: brin_pageops.c, gindatapage.c, ginentrypage.c, ginfast.c, gistxlog.c 等の各リソースマネージャ実装。
  • 呼び出し先: なし（内部的なエラー報告関数 elog/ereport を除く）。

関連データ構造

  • registered_buffer: バッファのメタデータとデータチェーンを保持する構造体。
  • XLogRecData: WALレコード構築用のデータチャンクを管理する連結リスト構造体。

失敗モードと境界条件

  • block_id が未登録の場合: ERROR を発生させます。
  • num_rdatas が max_rdatas を超える場合: ERROR を発生させます。
  • len または累積データ長が UINT16_MAX を超える場合: ERROR を発生させます。

保守担当者向けメモ

  • 本関数は XLogRegisterBuffer の後に呼び出す必要があります。
  • 登録されるデータは、XLogInsert が完了するまで有効である必要があります。
