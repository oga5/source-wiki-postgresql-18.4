ArchiveFormat

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/pg_backup.h:39

目的

PostgreSQL のバックアップおよびリストア処理において、サポートされるアーカイブ形式を識別するために使用される列挙型です。

シグネチャまたは形状

typedef enum _archiveFormat
{
	archUnknown = 0,
	archCustom = 1,
	archTar = 3,
	archNull = 4,
	archDirectory = 5,
} ArchiveFormat;

入力と出力

  • 本シンボル自体は定数セットであり、関数呼び出しの引数としてアーカイブ形式を指定する際に利用されます。

前提条件と不変条件

  • archUnknown は初期値や未設定状態を表します。
  • 各列挙値は、pg_backup_custom.c や pg_backup_tar.c 等の具体的なアーカイバ実装と一対一で対応します。

アルゴリズムまたは振る舞い

  • pg_dump および pg_restore において、ユーザー指定のフォーマット文字列を解析し、この列挙型に変換して処理を分岐させるための基盤となります。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/bin/pg_dump/pg_dump.c: parseArchiveFormat 関数内でのフォーマット解析。
  • src/bin/pg_dump/pg_backup_archiver.c: CreateArchive および OpenArchive 関数での形式指定。
  • **呼び出し先**:
  • 本シンボルはデータ型であるため、直接的な呼び出し先はありません。

関連データ構造

  • Archive: アーカイブのメインコンテキストを保持する構造体。
  • ArchiveHandle: アーカイバの内部実装で使用される構造体（ArchiveFormat をメンバとして保持）。

失敗モードと境界条件

  • 未定義の形式や archUnknown が渡された場合、アーカイバの初期化処理（OpenArchive / CreateArchive）においてエラーが発生する可能性があります。

保守担当者向けメモ

  • 新しいアーカイブ形式を追加する場合は、本列挙型に値を追加し、各アーカイバ実装（pg_backup_*.c）側で対応するロジックを実装する必要があります。
