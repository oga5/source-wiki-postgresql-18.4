GinSegmentInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/gininsert.c:2205

目的

GINインデックスの転置リスト（Posting List）を構成するセグメント情報を保持し、リスト構造として管理するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    dlist_node    node;           /* 連結リスト用ポインタ */
    GinPostingList *seg;          /* 転置リストセグメントへのポインタ */
} GinSegmentInfo;

入力と出力

本構造体はデータ保持用であり、関数ではないため入出力はありません。ただし、palloc を用いてメモリ確保され、dlist（二重連結リスト）の要素として操作されます。

前提条件と不変条件

  • node フィールドは dlist の一部として機能するため、リスト操作関数（dlist_push_tail 等）と組み合わせて使用する必要があります。
  • seg は有効な GinPostingList 構造体を指している必要があります。

アルゴリズムまたは振る舞い

  • dlist を介して複数のセグメントを順序立てて管理します。
  • dlist_container マクロを使用して、リストノードから親である GinSegmentInfo 構造体を取得するパターンで使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/gin/gininsert.c 内の関数群。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • dlist_node: dlist（PostgreSQLの二重連結リスト）のノード構造。
  • GinPostingList: GINインデックスの転置リストを表現する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc 呼び出し箇所でエラーが発生する可能性があります。
  • リスト操作: 空のリストに対して dlist_container を使用した場合や、不正なポインタを保持したままリスト操作を行った場合の動作は未定義です。

保守担当者向けメモ

  • 本構造体は gininsert.c 内の転置リスト構築ロジックにおいて、セグメントの動的な管理に使用されています。
  • dlist を使用しているため、リストの走査や要素の追加・削除を行う際は、PostgreSQLの lib/ilist.h で定義されている dlist APIの作法に従ってください。
