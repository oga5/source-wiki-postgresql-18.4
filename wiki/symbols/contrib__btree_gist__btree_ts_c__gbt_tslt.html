gbt_tslt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_ts.c:89

目的

GiSTインデックス内において、2つの Timestamp 値を比較し、左辺が右辺より小さい（less than）かどうかを判定します。

シグネチャまたは形状

static bool gbt_tslt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の Timestamp 値へのポインタ。
  • b: 比較対象の Timestamp 値へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では直接使用されません）。
  • **出力**:
  • bool: *a < *b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な Timestamp 型の値を指している必要があります。
  • 内部的に timestamp_lt 関数を呼び出すため、PostgreSQLのタイムスタンプ比較ルールに従います。

アルゴリズムまたは振る舞い

1. 引数 a と b を Timestamp 型のポインタとしてキャストします。 2. TimestampGetDatumFast マクロを使用して、各値を Datum 形式に変換します。 3. DirectFunctionCall2 を介して、PostgreSQLのコア関数 timestamp_lt を実行します。 4. 比較結果を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_ts.c 内の静的変数 tinfo の初期化において参照されています。
  • **呼び出し先**:
  • timestamp_lt (PostgreSQLコア関数)

関連データ構造

  • Timestamp: PostgreSQLのタイムスタンプ型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • timestamp_lt の仕様に基づき、タイムスタンプ値が範囲外である場合や無効な値である場合の挙動は、コアのタイムスタンプ処理に依存します。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する GiST 共通ロジックから呼び出されることを想定したアダプター関数です。
  • timestamptz 型の場合も、本ファイル内の変換処理を経て最終的に Timestamp として比較されるため、本関数が再利用されます。
