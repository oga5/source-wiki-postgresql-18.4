dshash_dump

定義

  • 種別: function
  • 定義箇所: src/backend/lib/dshash.c:777

目的

共有メモリ上のハッシュテーブル（dshash_table）の現在の状態をデバッグ目的で標準エラー出力（stderr）にダンプする。

シグネチャまたは形状

void dshash_dump(dshash_table *hash_table)

入力と出力

  • **入力**: hash_table - ダンプ対象の dshash_table 構造体へのポインタ。
  • **出力**: なし（副作用として stderr へハッシュテーブルの構造、パーティション情報、各バケット内のエントリ数が書き出される）。

前提条件と不変条件

  • hash_table->control->magic が DSHASH_MAGIC と一致していること。
  • 呼び出し元がパーティションロックを保持していないこと。

アルゴリズムまたは振る舞い

1. 全パーティションのロックを LW_SHARED モードで順次取得する。 2. ensure_valid_bucket_pointers を呼び出し、ローカルのバケットポインタを最新の状態に同期する。 3. ハッシュテーブルの総バケット数を出力する。 4. 各パーティションを走査し、以下の情報を出力する：

  • パーティション番号と、そのパーティションが管理するエントリ数。
  • 各バケット内の連結リストを走査し、バケットごとのエントリ数（キー数）をカウントして出力する。

    • 5. 最後に、取得した全パーティションのロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていませんが、src/include/lib/dshash.h で外部公開されています。
  • **呼び出し先**:
  • ensure_valid_bucket_pointers
  • LWLockAcquire / LWLockRelease
  • dsa_get_address
  • fprintf

関連データ構造

  • dshash_table: ハッシュテーブルの制御構造体。
  • dshash_table_item: バケット内の連結リストを構成するエントリ。
  • dshash_partition: パーティションごとの管理情報。

失敗モードと境界条件

  • ロック取得時にデッドロックが発生する可能性があるため、全パーティションのロック取得順序は固定されている。
  • 共有メモリ上のデータ構造が破損している場合、dsa_get_address によるポインタ解決時に予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は全パーティションのロックを保持するため、実行中は他のプロセスによるハッシュテーブル操作が完全にブロックされる。本番環境での使用は避けること。
