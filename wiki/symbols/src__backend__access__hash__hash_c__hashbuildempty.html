hashbuildempty

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:207

目的

空のハッシュインデックスを初期化し、初期化フォーク（initialization fork）内に構築します。

シグネチャまたは形状

void hashbuildempty(Relation index)

入力と出力

  • **入力**: index (Relation) - 初期化対象となるハッシュインデックスのリレーション。
  • **出力**: なし（void）。

前提条件と不変条件

  • index は有効なハッシュインデックスのリレーションである必要があります。

アルゴリズムまたは振る舞い

内部関数 _hash_init を呼び出し、指定されたリレーションに対して初期化処理を実行します。この際、バケット数 0 および INIT_FORKNUM を引数として渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hash.c 内の IndexAmRoutine 構造体（ambuildempty フィールド）を通じて、PostgreSQL のインデックス構築プロセスから呼び出されます。
  • **呼び出し先**: _hash_init（src/backend/access/hash/hashpage.c で定義されていると推測される内部関数）。

関連データ構造

  • Relation: PostgreSQL のリレーション記述子。

失敗モードと境界条件

  • リレーションの初期化に失敗した場合、基盤となるストレージ層の例外が発生する可能性があります。

保守担当者向けメモ

  • 本関数は hashhandler によって登録されるインデックス AM インターフェースの一部です。
  • 物理的なインデックスの初期化を担うため、変更時は _hash_init の仕様と整合性を保つ必要があります。
