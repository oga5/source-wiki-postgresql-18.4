end_heap_rewrite

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/rewriteheap.c:296

目的

テーブル再構築処理（VACUUM FULL や CLUSTER 等）の終了処理を行い、未解決のタプルをフラッシュし、リソースを解放します。

シグネチャまたは形状

void end_heap_rewrite(RewriteState state)

入力と出力

  • **入力**: RewriteState state（再構築中の状態を保持する構造体へのポインタ）
  • **出力**: なし（副作用としてディスクへの書き込みとメモリ解放を行う）

前提条件と不変条件

  • 呼び出し元は、対象テーブルに対して AccessExclusiveLock を保持している必要があります。
  • state は begin_heap_rewrite によって初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. **未解決タプルの処理**: rs_unresolved_tups ハッシュテーブルに残存するタプルを走査し、t_ctid を無効化した上で raw_heap_insert を用いて新しいヒープへ書き込みます。 2. **バッファのフラッシュ**: rs_buffer に保持されている最後のページをディスクへ書き出します。 3. **バルク書き込みの終了**: smgr_bulk_finish を呼び出し、バルク書き込み状態をクリーンアップします。 4. **論理デコーディングの終了**: logical_end_heap_rewrite を呼び出し、論理レプリケーション用のマッピング情報を確定させます。 5. **メモリ解放**: state->rs_cxt を削除し、再構築に関連するすべてのメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c:988
  • **呼び出し先**:
  • hash_seq_init / hash_seq_search
  • raw_heap_insert
  • smgr_bulk_write
  • smgr_bulk_finish
  • logical_end_heap_rewrite
  • MemoryContextDelete

関連データ構造

  • RewriteState: 再構築の状態を管理する不透明な構造体。
  • UnresolvedTup: 更新チェーン解決待ちのタプルデータ。

失敗モードと境界条件

  • メモリ不足: rs_unresolved_tups に大量のタプルが滞留している場合、メモリ消費が大きくなります。
  • ディスク書き込みエラー: smgr_bulk_write 実行時にファイルシステムエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は再構築の最終段階であり、ここで解放されるメモリコンテキストには再構築中に使用されたすべてのハッシュテーブルや一時データが含まれます。
  • 論理レプリケーションが有効な場合、logical_end_heap_rewrite によるマッピングファイルの同期がクラッシュリカバリの整合性に直結するため、変更時は注意が必要です。
