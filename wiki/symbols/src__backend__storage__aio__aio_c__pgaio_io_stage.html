pgaio_io_stage

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/storage/aio/aio.c:423

目的

取得済みの PgAioHandle に対して I/O 操作の内容を定義し、実行キューへの投入または同期実行を行う。

シグネチャまたは形状

void pgaio_io_stage(PgAioHandle *ioh, PgAioOp op)

入力と出力

  • **入力**:
  • ioh: 操作対象の PgAioHandle ポインタ。
  • op: 実行する I/O 操作（PgAioOp）。
  • **出力**: なし（副作用としてハンドルの状態遷移と I/O 発行を行う）。

前提条件と不変条件

  • ioh->state が PGAIO_HS_HANDED_OUT であること。
  • pgaio_my_backend->handed_out_io が対象の ioh と一致すること。
  • pgaio_io_has_target(ioh) が真であること。
  • 割り込みによる状態不整合を防ぐため、内部で HOLD_INTERRUPTS() を呼び出し、終了時に RESUME_INTERRUPTS() を呼び出す。

アルゴリズムまたは振る舞い

1. ioh->op に操作を設定し、状態を PGAIO_HS_DEFINED に更新する。 2. pgaio_my_backend->handed_out_io をクリアし、新たな I/O 取得を許可する。 3. pgaio_io_call_stage(ioh) を実行し、状態を PGAIO_HS_STAGED に更新する。 4. pgaio_io_needs_synchronous_execution(ioh) を判定する。

  • **非同期の場合**: pgaio_my_backend->staged_ios に追加する。in_batchmode が無効であれば pgaio_submit_staged() を呼び出し即時発行する。
  • **同期の場合**: pgaio_io_prepare_submit(ioh) を経て pgaio_io_perform_synchronously(ioh) を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio_io.c（pgaio_io_stage を介した readv/writev 操作など）
  • **呼び出し先**:
  • pgaio_io_update_state
  • pgaio_io_call_stage
  • pgaio_io_needs_synchronous_execution
  • pgaio_submit_staged
  • pgaio_io_prepare_submit
  • pgaio_io_perform_synchronously
  • pgaio_debug_io

関連データ構造

  • PgAioHandle: I/O 操作の状態を保持するハンドル。
  • PgAioOp: I/O 操作の種類を定義する列挙型または構造体。
  • pgaio_my_backend: 現在のバックエンドの AIO 状態を管理するグローバル構造体。

失敗モードと境界条件

  • pgaio_my_backend->num_staged_ios が PGAIO_SUBMIT_BATCH_SIZE を超える場合、アサーションエラーが発生する。
  • 割り込みが許可された状態で実行された場合、状態遷移の不整合を招く可能性がある（HOLD_INTERRUPTS で保護）。

保守担当者向けメモ

  • 本関数は状態遷移を伴うため、割り込み禁止区間内での実行が必須です。
  • in_batchmode の状態によって発行タイミングが制御されるため、バッチ処理の実装時にはデッドロックに注意してください。
