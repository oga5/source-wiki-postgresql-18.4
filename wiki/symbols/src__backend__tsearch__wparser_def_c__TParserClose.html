TParserClose

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/wparser_def.c:371

目的

TParser 構造体に関連付けられたメモリリソースを解放し、パーサーの状態をクリーンアップする。

シグネチャまたは形状

static void TParserClose(TParser *prs)

入力と出力

  • **入力**: TParser *prs （解放対象のパーサーインスタンスへのポインタ）
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • prs は有効な TParser 構造体へのポインタであること。
  • prs->state が指すスタック構造は、pfree によって安全に解放可能であること。

アルゴリズムまたは振る舞い

1. prs->state が指す TParserPosition スタックを辿り、各要素を pfree で順次解放する。 2. prs->wstr および prs->pgwstr が割り当てられている場合、それぞれ pfree で解放する。 3. WPARSER_TRACE が定義されている場合、標準エラー出力に終了ログを出力する。 4. 最後に prs 自体を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/wparser_def.c:1923 （直接的な呼び出しを確認）
  • **呼び出し先**:
  • pfree （PostgreSQL のメモリ解放関数）

関連データ構造

  • TParser: 解析状態を保持するメイン構造体。
  • TParserPosition: 解析状態のスタック要素。

失敗モードと境界条件

  • prs が NULL の場合、ポインタ参照によりクラッシュする可能性がある（ソースコード上ではガードなし）。
  • prs->wstr や prs->pgwstr が NULL の場合はスキップされるため、安全に処理される。

保守担当者向けメモ

  • 本関数はパーサーのライフサイクルの終端を担当する。新しいフィールドを TParser 構造体に追加した場合は、必ず本関数内で解放処理を追加すること。
