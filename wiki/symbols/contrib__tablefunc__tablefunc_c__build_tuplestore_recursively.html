build_tuplestore_recursively

定義

  • 種別: function
  • 定義箇所: contrib/tablefunc/tablefunc.c:1187

目的

階層データ探索（connectby）において、指定されたテーブルから親子関係を深さ優先探索で辿り、結果を Tuplestore に格納する。

シグネチャまたは形状

static void build_tuplestore_recursively(
    char *key_fld, char *parent_key_fld, char *relname, char *orderby_fld,
    char *branch_delim, char *start_with, char *branch, int level,
    int *serial, int max_depth, bool show_branch, bool show_serial,
    MemoryContext per_query_ctx, AttInMetadata *attinmeta, Tuplestorestate *tupstore
)

入力と出力

  • **入力**: 探索対象のテーブル名、キーフィールド名、開始ノード、再帰レベル、シリアル番号カウンタ、および Tuplestore への書き込み用メタデータ。
  • **出力**: なし（副作用として tupstore にタプルが追加される）。

前提条件と不変条件

  • max_depth > 0 の場合、level が max_depth を超えると探索を停止する。
  • level == 0 の場合、ルートノードを tupstore に格納してから再帰を開始する。
  • 戻り値のタプル記述子（tupdesc）と SPI 実行結果の記述子が互換性を持つ必要がある。

アルゴリズムまたは振る舞い

1. **SQL 生成**: 指定された start_with 値を基に、子ノードを取得する SELECT 文を構築する。 2. **ルート処理**: level == 0 の場合、開始ノードをタプル化して tupstore に格納する。 3. **データ取得**: SPI_execute を用いて子ノードを取得する。 4. **再帰的探索**: 取得した各行に対し、以下の処理を行う：

  • 循環参照の検出（現在のキーが既に branch に含まれている場合はエラー）。
  • パス（branch）の更新。
  • BuildTupleFromCStrings によるタプル構築と tuplestore_puttuple による格納。
  • 自身を再帰呼び出し（※ソースコード上、再帰呼び出しの記述はループ内の末尾に続く構造）。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/tablefunc/tablefunc.c 内の connectby_text および connectby_text_serial から呼び出される。
  • **呼び出し先**: SPI_execute, BuildTupleFromCStrings, tuplestore_puttuple, compatConnectbyTupleDescs 等の PostgreSQL 内部 API。

関連データ構造

  • Tuplestorestate: 構築されたタプルを保持する一時ストレージ。
  • AttInMetadata: タプル構築に必要な型情報。
  • StringInfoData: SQL 文およびパス文字列の動的構築に使用。

失敗モードと境界条件

  • **無限再帰**: 探索パス内に既に存在するキーが出現した場合、ERRCODE_INVALID_RECURSION でエラー終了する。
  • **深さ制限**: max_depth を超えた場合、探索を打ち切る。
  • **NULL 値**: キーフィールドが NULL の場合、探索対象から除外される。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、スタックオーバーフローのリスクがある。深い階層を持つデータに対しては注意が必要。
  • branch_delim を使用してパスを構築しているため、区切り文字の選択が重要。
  • SPI_execute を多用するため、パフォーマンスは対象テーブルのインデックス構成に強く依存する。
