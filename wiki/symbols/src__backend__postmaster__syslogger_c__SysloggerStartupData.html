SysloggerStartupData

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/syslogger.c:153

目的

Syslogger プロセスの起動時に、現在開いているログファイルのファイル記述子を保持および受け渡すためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	int			syslogFile;
	int			csvlogFile;
	int			jsonlogFile;
} SysloggerStartupData;

入力と出力

  • **入力**: 構造体の各フィールドは、ログ出力先となるファイル記述子（int）を保持します。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • 各フィールドは、有効なファイル記述子または無効な状態を示す値（通常は -1）を保持します。

アルゴリズムまたは振る舞い

  • Syslogger プロセス起動時に、親プロセスから引き継がれた、あるいは初期化されたログファイルのファイル記述子を格納します。
  • 構造体へのポインタを通じて、ログ書き込み処理に必要なファイル記述子を参照するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/syslogger.c 内の関数で、startup_data 変数として宣言および初期化されます（行598）。また、この構造体を参照するポインタ slsdata が同ファイル内の関数で使用されています（行186）。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SysLoggerMain: Syslogger プロセスのメインループ関数。
  • logfile_open: ログファイルを開く際に使用される関数。

失敗モードと境界条件

  • ファイル記述子が正しく初期化されていない場合、ログの書き込みが失敗する可能性があります。
  • ログローテーション等によりファイル記述子が更新された場合、本構造体の値と実際のファイル状態の整合性を維持する必要があります。

保守担当者向けメモ

  • 本構造体は syslogger.c 内でのみ使用されています。
  • ログ出力形式（stderr, csvlog, jsonlog）の追加や変更がある場合は、本構造体のフィールド構成を見直す必要があります。
