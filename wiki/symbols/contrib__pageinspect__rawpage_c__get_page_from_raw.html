get_page_from_raw

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/rawpage.c:217

目的

bytea 型として渡された生のページデータを、メモリ上で適切にアライメントされた Page 構造体に変換し、後続のページ解析処理で安全にアクセスできるようにする。

シグネチャまたは形状

Page get_page_from_raw(bytea *raw_page)

入力と出力

  • **入力**: bytea *raw_page（PostgreSQL のページイメージを含む可変長バイト配列）
  • **出力**: Page（メモリ上に確保された、BLCKSZ サイズのページデータへのポインタ）

前提条件と不変条件

  • 入力された raw_page のサイズは、PostgreSQL のシステム定数 BLCKSZ と完全に一致していなければならない。
  • 戻り値の Page は palloc によってメモリ確保されるため、呼び出し元で適切に管理（または解放）する必要がある。

アルゴリズムまたは振る舞い

1. VARSIZE_ANY_EXHDR を使用して、入力された bytea データの本体サイズを取得する。 2. 取得したサイズが BLCKSZ と一致するか検証する。不一致の場合は ERROR を発生させる。 3. palloc を用いて BLCKSZ 分のメモリを確保する。 4. VARDATA_ANY を用いて raw_page のデータ開始位置を特定し、確保したメモリ領域へ memcpy でコピーを行う。 5. コピーしたメモリ領域の先頭アドレスを Page として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/brinfuncs.c
  • contrib/pageinspect/btreefuncs.c
  • contrib/pageinspect/fsmfuncs.c
  • contrib/pageinspect/ginfuncs.c
  • contrib/pageinspect/gistfuncs.c
  • contrib/pageinspect/hashfuncs.c
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理関数)

関連データ構造

  • Page (PostgreSQL のページレイアウト定義)
  • bytea (PostgreSQL の可変長バイト配列型)

失敗モードと境界条件

  • **サイズ不一致**: 入力データのサイズが BLCKSZ 以外の場合、ERRCODE_INVALID_PARAMETER_VALUE を伴うエラーが発生する。

保守担当者向けメモ

  • 本関数は単なるコピーではなく、CPU アライメントを考慮したメモリ配置を行うための重要なステップである。ページ内の構造体に直接キャストしてアクセスする前に、必ず本関数を経由させること。
  • 権限管理については、本関数を呼び出す上位の SQL 関数側でスーパーユーザー権限の確認が行われている前提である。
