RawColumnDefault

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/heap.h:28

目的

テーブル作成や列定義の変更時に、未変換（解析前）の列デフォルト値式を一時的に保持するために使用されます。

シグネチャまたは形状

typedef struct RawColumnDefault
{
    AttrNumber  attnum;         /* デフォルト値を付与する属性番号 */
    Node       *raw_default;    /* デフォルト値（未変換の解析ツリー） */
    char        generated;      /* attgenerated 設定 */
} RawColumnDefault;

入力と出力

  • **入力**: palloc を用いてメモリ確保され、パーサーが生成した Node ツリーが raw_default に格納されます。
  • **出力**: 最終的に CookedConstraint 形式へ変換され、システムカタログ（pg_attrdef 等）へ永続化されるためのデータソースとなります。

前提条件と不変条件

  • raw_default はパーサーが生成した未変換の Node ツリーである必要があります。
  • 実行可能な形式（CookedConstraint）に変換される前の段階でリストとして管理されます。

アルゴリズムまたは振る舞い

1. DDL（CREATE TABLE 等）実行時、デフォルト値の定義が RawColumnDefault 構造体に格納されます。 2. これらはリスト形式で保持され、後続の処理で解析（cook）されます。 3. 解析完了後、CookedConstraint へ変換され、カタログ登録処理へ引き渡されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/catalog/heap.c: newColDefaults リストとして管理。
  • src/backend/commands/tablecmds.c: CREATE TABLE 等のコマンド実行時に palloc でインスタンス化され、リストに追加されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • Node: raw_default が保持する解析ツリーの基底型。
  • CookedConstraint: 本構造体のデータが変換された後の、解析済み制約情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足: palloc によるメモリ確保時に失敗する可能性があります。
  • 不正な式: raw_default に格納されたノードが解析不能な場合、後続の変換フェーズでエラーが発生します。

保守担当者向けメモ

  • 本構造体は「未変換」の情報を扱うため、ライフサイクルは短く、主に DDL 実行中の過渡的なデータ保持に限定されます。
  • generated フィールドは attgenerated の設定を反映しており、生成列（Generated Column）の処理と密接に関連しています。
