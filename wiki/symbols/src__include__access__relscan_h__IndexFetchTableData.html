IndexFetchTableData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/relscan.h:121

目的

インデックススキャンを通じてテーブルからタプルをフェッチする際の、アクセスメソッド（AM）非依存の基底状態を保持するために使用されます。

シグネチャまたは形状

typedef struct IndexFetchTableData
{
    Relation    rel;
} IndexFetchTableData;

入力と出力

  • **入力**: table_index_fetch_begin 関数等により、対象となる Relation を受け取って初期化されます。
  • **出力**: 構造体自体は状態保持用であり、具体的なタプル取得は table_index_fetch_tuple 等のインターフェースを通じて行われます。

前提条件と不変条件

  • 本構造体は、特定のテーブルアクセスメソッド（AM）によって、より大きな構造体の一部として埋め込まれることが想定されています。

アルゴリズムまたは振る舞い

  • 本構造体はスキャン実行時のコンテキストを保持します。
  • heapam 等の具体的なAM実装において、heapam_index_fetch_reset、heapam_index_fetch_end、heapam_index_fetch_tuple といった関数で操作されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/constraint.c (トリガー処理等)
  • src/backend/access/table/tableam.c (テーブルAMの抽象レイヤー)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。操作は tableam.h で定義された関数ポインタ経由で実行されます。

関連データ構造

  • Relation: スキャン対象のリレーション情報。
  • IndexScanDescData: インデックススキャン全体の管理構造体（xs_heapfetch メンバとして本構造体を参照する場合がある）。

失敗モードと境界条件

  • 構造体の初期化（index_fetch_begin）に失敗した場合、無効なポインタやリソースリークが発生する可能性があります。
  • 適切な end 関数（heapam_index_fetch_end 等）を呼び出さない場合、保持しているリソースが解放されない可能性があります。

保守担当者向けメモ

  • 本構造体は多くのAMで「継承」的に利用されるため、メンバの追加や変更は、これを利用するすべてのAM実装に影響を与える可能性があります。
  • tableam.h の index_fetch_begin インターフェース定義と整合性を保つ必要があります。
