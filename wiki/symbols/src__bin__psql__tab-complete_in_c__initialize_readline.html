initialize_readline

定義

  • 種別: function
  • 定義箇所: src/bin/psql/tab-complete.in.c:1494

目的

Readline ライブラリの補完機能を psql の対話型シェル向けに初期化および設定する。

シグネチャまたは形状

void initialize_readline(void)

入力と出力

  • 入力: なし（グローバルな pset 構造体および Readline のグローバル変数を参照）
  • 出力: なし（Readline のグローバル変数への副作用を通じて設定を適用）

前提条件と不変条件

  • pset.progname が初期化されていること。
  • Readline ライブラリがリンクされ、利用可能な状態であること。

アルゴリズムまたは振る舞い

1. rl_readline_name を pset.progname に設定する。 2. rl_attempted_completion_function に psql_completion を割り当て、補完ロジックを登録する。 3. USE_FILENAME_QUOTING_FUNCTIONS が定義されている場合、ファイル名のクォート/デクォート関数を登録する。 4. rl_basic_word_break_characters を WORD_BREAKS に設定する。 5. rl_completer_quote_characters を '（シングルクォート）に設定する。 6. HAVE_RL_FILENAME_QUOTE_CHARACTERS が定義されている場合、すべての文字をクォート対象として rl_filename_quote_characters を動的に生成・設定する。 7. completion_max_records を 1000 に制限する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/input.c:355
  • 呼び出し先: psql_completion（関数ポインタとして登録）、pg_malloc（メモリ確保）

関連データ構造

  • pset: psql の設定状態を保持するグローバル構造体。
  • Readline ライブラリのグローバル変数群（rl_...）。

失敗モードと境界条件

  • pg_malloc によるメモリ確保失敗時はプロセスが終了する可能性がある。
  • rl_completer_quote_characters の設定は、ライブラリ（特に libedit）の不整合により意図した通りに動作しない場合がある。

保守担当者向けメモ

  • 本関数は psql 起動時に一度だけ呼び出されることを想定している。
  • rl_filename_quote_characters の生成には pg_malloc を使用しているため、メモリリークの懸念はないが、設定の変更時には注意が必要。
  • rl_completion_query_items は環境によって定義されない場合があるため、補完候補数の制限には completion_max_records を使用している。
