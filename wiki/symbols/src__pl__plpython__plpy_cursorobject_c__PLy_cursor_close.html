PLy_cursor_close

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_cursorobject.c:498

目的

Python から呼び出されたカーソルオブジェクトを明示的にクローズし、関連する PostgreSQL ポータルを解放する。

シグネチャまたは形状

static PyObject *PLy_cursor_close(PyObject *self, PyObject *unused);

入力と出力

  • **入力**:
  • self: PLyCursorObject 型の Python オブジェクト。
  • unused: 使用されない引数（METH_NOARGS 呼び出し用）。
  • **出力**: 成功時は Py_None を返し、失敗時は NULL を返す。

前提条件と不変条件

  • self は PLyCursorObject 型としてキャスト可能であること。
  • カーソルが既にクローズされている場合、本関数は何もしない。

アルゴリズムまたは振る舞い

1. self を PLyCursorObject にキャストする。 2. cursor->closed フラグを確認し、未クローズの場合のみ処理を行う。 3. cursor->portalname を使用して、対応する PostgreSQL の Portal を取得する。 4. ポータルが無効な場合（アボートされたサブトランザクション内など）、PyExc_ValueError を発生させ NULL を返す。 5. 有効なポータルであれば、UnpinPortal を呼び出して参照カウントを減らし、SPI_cursor_close を実行してポータルを閉じる。 6. cursor->closed を true に設定する。 7. Py_RETURN_NONE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Python インタープリタ（PLyCursorObject のメソッドテーブル close 経由）。
  • **呼び出し先**: GetPortalByName, PortalIsValid, PLy_exception_set, UnpinPortal, SPI_cursor_close。

関連データ構造

  • PLyCursorObject: カーソルの状態（portalname, closed フラグ）を保持する構造体。

失敗モードと境界条件

  • **失敗**: アボートされたサブトランザクション内でクローズしようとした場合、ValueError が発生する。
  • **境界条件**: 既にクローズ済みのカーソルに対して呼び出された場合、安全に無視され Py_None が返される。

保守担当者向けメモ

  • 本関数は METH_NOARGS として登録されているため、引数の処理は最小限である。
  • SPI_cursor_close の呼び出しにより、ポータルに関連付けられたリソースが解放されるため、以降のカーソル操作は不可となる。
