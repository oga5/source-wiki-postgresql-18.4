pad_output

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/io.c:467

目的

出力ストリームにおいて、現在のカラム位置から指定されたターゲットカラムまで、タブやスペースを挿入して埋めることでインデントを調整します。

シグネチャまたは形状

static int pad_output(int current, int target)

入力と出力

  • **入力**:
  • current (int): 現在の出力カラム位置。
  • target (int): 到達すべき目標カラム位置。
  • **出力**:
  • 戻り値 (int): 処理後のカラム位置（通常は target）。

前提条件と不変条件

  • current が target 以上の場合、何も出力せず current を返します。
  • use_tabs が有効な場合、タブ文字の挿入が試行されます。
  • postgres_tab_rules が有効な場合、特定の条件下でタブの代わりにスペースが挿入される特殊なルールが適用されます。

アルゴリズムまたは振る舞い

1. current >= target であれば即座に終了します。 2. use_tabs が真の場合、tabsize に基づいてタブ文字（\t）を挿入します。この際、postgres_tab_rules に従い、特定の境界条件ではタブの代わりにスペースを出力します。 3. 残りの差分に対して、スペース（' '）を putc で出力します。 4. 最終的なカラム位置として target を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/io.c 内の dump_line 等から複数回呼び出されています。
  • **呼び出し先**: 標準ライブラリの putc を使用して output ストリームへ書き込みます。

関連データ構造

  • output: 出力先のファイルストリーム。
  • use_tabs: タブ使用の可否を制御するグローバル変数。
  • tabsize: タブの幅を定義する変数。
  • postgres_tab_rules: PostgreSQL 固有のタブ出力ルールを適用するかを決定するフラグ。

失敗モードと境界条件

  • current >= target の場合、出力は行われません。
  • postgres_tab_rules が有効な場合、target がタブ境界付近にあると、タブではなくスペースが優先的に選択される場合があります。

保守担当者向けメモ

  • 本関数は io.c 内のバッファ出力処理において、インデントの整合性を保つための基盤となります。
  • タブ計算ロジックには postgres_tab_rules が深く関与しているため、インデントの挙動を変更する際は indent_globs.h で定義された関連フラグの影響を確認してください。
