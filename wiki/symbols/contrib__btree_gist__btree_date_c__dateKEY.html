dateKEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_date.c:12

目的

PostgreSQL の btree_gist 拡張において、GiST インデックスのノード内に date 型の範囲（下限値と上限値）を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct
{
	DateADT		lower;
	DateADT		upper;
} dateKEY;

入力と出力

  • **入力**: DateADT 型の lower（範囲の下限）および upper（範囲の上限）。
  • **出力**: GiST インデックスのノードに格納される固定長の構造体データ。

前提条件と不変条件

  • GiST インデックスのリーフノードにおいては、通常 lower == upper となることが期待されます。
  • lower は upper 以下の値である必要があります（範囲の整合性）。

アルゴリズムまたは振る舞い

  • 本構造体は、btree_utils_num.c が提供する汎用的な数値処理ロジックと組み合わせて使用されます。
  • インデックス構築時には gbt_date_compress を通じて date 型の値が本構造体に変換され、検索時には gbt_date_consistent がこの構造体の lower および upper を参照してクエリ条件との合致を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_date.c 内の各サポート関数（gbt_datekey_cmp, gbt_date_compress, gbt_date_consistent 等）が、インデックスデータの読み書きや比較のために本構造体をキャストして利用しています。
  • **呼び出し先**: 直接的な関数呼び出しは行いません。データ保持のための型定義です。

関連データ構造

  • DateADT: PostgreSQL の日付型を表現する内部データ型。
  • gbtree_ninfo: btree_utils_num.h で定義される、数値型 GiST インデックスのメタデータ構造体。

失敗モードと境界条件

  • lower が upper を超えるような不正な範囲データが格納された場合、インデックス検索の結果が不正確になる可能性があります。
  • メモリ割り当て時（palloc）に失敗した場合は、PostgreSQL のエラーハンドリング機構により処理が中断されます。

保守担当者向けメモ

  • 本構造体は btree_utils_num.c のユーティリティ関数と密接に連携しています。変更を加える際は、btree_utils_num.h で定義されているインターフェースとの整合性に注意してください。
  • インデックスのノードサイズは sizeof(dateKEY) に依存するため、構造体のメンバ変更はインデックスの物理レイアウトに影響を与えます。
