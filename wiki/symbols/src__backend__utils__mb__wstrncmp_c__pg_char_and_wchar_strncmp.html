pg_char_and_wchar_strncmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/wstrncmp.c:54

目的

char 型の文字列と pg_wchar 型の文字列を、指定した長さ n まで比較します。型変換を伴う比較が必要な場面で使用されます。

シグネチャまたは形状

int pg_char_and_wchar_strncmp(const char *s1, const pg_wchar *s2, size_t n)

入力と出力

  • s1: 比較対象の char 型文字列へのポインタ。
  • s2: 比較対象の pg_wchar 型文字列へのポインタ。
  • n: 比較する最大文字数。
  • 戻り値: 比較結果。一致した場合は 0 を返し、不一致の場合は差分を返します。

前提条件と不変条件

  • n が 0 の場合、比較は行われず即座に 0 を返します。
  • s1 の各文字は unsigned char にキャストされた後、pg_wchar 型に昇格されてから s2 の要素と比較されます。

アルゴリズムまたは振る舞い

1. n が 0 であれば 0 を返します。 2. do-while ループ内で、s1 の現在の文字を pg_wchar に変換し、s2 の現在の要素と比較します。 3. 不一致が発生した場合、その差分（s1 の変換値 - s2 の値）を返します。 5. n 回の比較が完了するまで、ポインタをインクリメントしながら処理を継続します。

関連データ構造

  • pg_wchar: PostgreSQLの内部的なワイド文字型。mb/pg_wchar.h で定義されています。

失敗モードと境界条件

  • n が 0 の場合、比較は行われず 0 が返されます。

保守担当者向けメモ

  • 本関数は型が異なる配列を直接比較する特殊なケース用です。一般的な pg_wchar 配列同士の比較には pg_wchar_strncmp を使用してください。
  • s1 の文字を unsigned char にキャストしてから pg_wchar に昇格させる処理は、符号拡張による予期せぬ挙動を防ぐために重要です。
