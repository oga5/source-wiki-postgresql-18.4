precheck_tar_backup_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:880

目的

Tar形式のバックアップディレクトリ内に存在するファイルを検証し、後続の処理に必要なメタデータ（圧縮形式やテーブルスペースOID）を抽出してリストへ追加する。

シグネチャまたは形状

static void precheck_tar_backup_file(verifier_context *context, char *relpath, char *fullpath, SimplePtrList *tarfiles)

入力と出力

  • **入力**:
  • context: 検証コンテキスト（フォーマット確認に使用）。
  • relpath: ファイルの相対パス。
  • fullpath: ファイルの絶対パス。
  • tarfiles: 抽出した情報を格納するリスト。
  • **出力**: なし（副作用として tarfiles リストへの追加、またはエラー報告を行う）。

前提条件と不変条件

  • context->format が 't'（Tar形式）であること。
  • 対象ファイルが通常のファイル（S_ISREG）であること。
  • ファイル名が base.tar、pg_wal.tar、または <tablespaceoid>.tar（各圧縮拡張子付き）のいずれかの形式に従っていること。

アルゴリズムまたは振る舞い

1. stat を実行し、ファイル情報の取得と存在確認を行う。 2. ファイルが通常ファイルでない場合はエラーを報告して終了する。 3. ファイル名からテーブルスペースOIDを特定し、拡張子から圧縮アルゴリズム（None, Gzip, LZ4, Zstd）を判定する。 4. 判定できない形式や不正なOIDの場合はエラーを報告する。 5. pg_wal 関連ファイルであれば、検証対象外として処理を終了する。 6. 上記以外の場合、tar_file 構造体を割り当て、情報をリスト tarfiles に追加する。 7. show_progress が有効な場合、total_size を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_verifybackup/pg_verifybackup.c 内のディレクトリ走査処理から呼び出される。
  • **呼び出し先**: report_backup_error, pg_malloc, pstrdup, simple_ptr_list_append。

関連データ構造

  • verifier_context: 検証全体の状態管理。
  • tar_file: Tarファイルのメタデータ（パス、OID、圧縮形式）を保持する構造体。
  • SimplePtrList: tar_file を格納するリスト。

失敗モードと境界条件

  • stat 失敗: ファイルアクセス権限やパスの誤り。
  • 非通常ファイル: ディレクトリやデバイスファイルが混入している場合。
  • 命名規則違反: 期待される形式（base, pg_wal, OID）に一致しないファイル名。
  • 圧縮形式不明: サポートされていない拡張子。
  • OID範囲外: ファイル名から抽出した数値が OID_MAX を超える場合。

保守担当者向けメモ

  • 本関数はTar形式のバックアップ検証における「準備フェーズ」を担う。
  • WALファイルは pg_waldump で処理されるため、本関数ではリストに追加せずスキップする仕様となっている。
