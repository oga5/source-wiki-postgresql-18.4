PGEventResultDestroy

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-events.h:64

目的

PGresult オブジェクトが破棄される際に発生するイベント（PGEVT_RESULTDESTROY）において、関連付けられたカスタムデータをクリーンアップするための情報を受け渡すために使用されます。

シグネチャまたは形状

typedef struct
{
	PGresult   *result;
} PGEventResultDestroy;

入力と出力

  • **入力**: PGresult *result。破棄されようとしている PGresult オブジェクトへのポインタを保持します。
  • **出力**: なし（構造体としてイベントハンドラに渡されます）。

前提条件と不変条件

  • 本構造体は、PGEVT_RESULTDESTROY イベントが発生した際に PGEventProc に渡される引数の一部として利用されます。
  • result フィールドは、破棄対象の有効な PGresult オブジェクトを指している必要があります。

アルゴリズムまたは振る舞い

1. libpq 内部で PGresult の破棄処理が開始されます。 2. 登録済みのイベントハンドラ（PGEventProc）が呼び出され、イベント種別として PGEVT_RESULTDESTROY が通知されます。 3. ハンドラは、この構造体の result フィールドを参照し、PQresultInstanceData を用いて取得したカスタムデータの解放処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-exec.c 内で本構造体がインスタンス化されていることが確認されています。
  • **呼び出し先**: 呼び出し先に関する具体的な情報は提供されていません。

関連データ構造

  • PGresult: libpq のクエリ結果を保持する構造体。
  • PGEventProc: イベントハンドラの関数ポインタ型。
  • PGEventId: イベントの種類を識別する列挙型（PGEVT_RESULTDESTROY を含む）。

失敗モードと境界条件

  • イベントハンドラ内で result に対して不正な操作を行うと、libpq のメモリ管理と競合し、クラッシュやメモリ破壊を引き起こす可能性があります。
  • 破棄イベント内でカスタムデータの解放を忘れると、メモリリークが発生します。

保守担当者向けメモ

  • 本構造体は PGresult のライフサイクル管理に直結しています。変更を加える際は、libpq-events.c におけるイベント発行ロジックとの整合性に注意してください。
