BaseBackupGetTargetHandle

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_target.c:116

目的

ユーザーが指定したバックアップターゲット名に基づき、登録済みのターゲットリストから対応する定義を検索し、検証済みの BaseBackupTargetHandle を生成して返します。

シグネチャまたは形状

BaseBackupTargetHandle *BaseBackupGetTargetHandle(char *target, char *target_detail)

入力と出力

  • **入力**:
  • target: ターゲット名を指定する文字列。
  • target_detail: ターゲット固有の詳細設定文字列。
  • **出力**:
  • 成功時: 初期化および検証済みの BaseBackupTargetHandle 構造体へのポインタ。
  • 失敗時: ターゲットが見つからない場合、ERROR を発生させ処理を中断します。

前提条件と不変条件

  • ターゲットリスト BaseBackupTargetTypeList が未初期化の場合は、関数内で initialize_target_list() が自動的に呼び出されます。
  • 戻り値のハンドルに含まれる detail_arg は、対応するターゲット型の check_detail 関数によって検証済みであることが保証されます。

アルゴリズムまたは振る舞い

1. BaseBackupTargetTypeList が空であれば initialize_target_list() を実行し、ターゲットリストを構築します。 2. リストを走査し、target 引数と一致する名前を持つ BaseBackupTargetType を検索します。 3. 一致するターゲットが見つかった場合、palloc で BaseBackupTargetHandle を確保します。 4. 該当ターゲットの check_detail 関数を呼び出し、target_detail を検証して結果をハンドルに格納します。 5. ターゲットが見つからない場合は ERRCODE_FEATURE_NOT_SUPPORTED エラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:958
  • **呼び出し先**:
  • initialize_target_list()
  • ttype->check_detail() (ターゲット型ごとに定義された検証関数)
  • palloc()

関連データ構造

  • BaseBackupTargetType: ターゲットの定義（名前、検証関数など）を保持する構造体。
  • BaseBackupTargetHandle: 検索結果と検証済み詳細情報を保持する構造体。

失敗モードと境界条件

  • 指定された target が登録リストに存在しない場合、ereport(ERROR, ...) によりバックアップ処理全体が停止します。
  • check_detail 関数内でエラーが発生した場合、そのエラーがそのまま伝播します。

保守担当者向けメモ

  • 本関数はバックアップ開始の初期段階で呼ばれます。check_detail による検証は、バックアップ実行前に不正な設定を排除するために重要です。
  • ターゲットリストのメモリ管理は TopMemoryContext に依存しています。拡張モジュールでターゲットを追加する際は、メモリの生存期間に注意してください。
