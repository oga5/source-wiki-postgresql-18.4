get_gating_quals

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/createplan.c:1001

目的

クエリの制約条件（quals）から、実行時に評価される擬似定数（pseudoconstant）条件を抽出し、実行に適した順序に並べ替える。

シグネチャまたは形状

static List *get_gating_quals(PlannerInfo *root, List *quals)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報構造体。
  • quals: RestrictInfo 形式の制約リスト。
  • **出力**: 抽出された擬似定数条件のリスト（List *）。条件が存在しない場合は NIL を返す。

前提条件と不変条件

  • root->hasPseudoConstantQuals が偽の場合、処理を行わず直ちに NIL を返す。

アルゴリズムまたは振る舞い

1. root->hasPseudoConstantQuals を確認し、擬似定数が含まれない場合は終了する。 2. order_qual_clauses を呼び出し、RestrictInfo 形式のまま条件をコスト順に並べ替える。 3. extract_actual_clauses を呼び出し、並べ替えられたリストから擬似定数のみを抽出して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c 内の複数箇所（結合プランやスキャンプランの生成時）。
  • **呼び出し先**:
  • order_qual_clauses
  • extract_actual_clauses

関連データ構造

  • PlannerInfo: プランナの全体状態を保持する構造体。
  • RestrictInfo: 述語条件を保持する最適化用構造体。

失敗モードと境界条件

  • 擬似定数が含まれない場合、または root->hasPseudoConstantQuals が偽の場合は NIL を返す。

保守担当者向けメモ

  • 本関数は createplan.c 内で、プラン生成時にゲート（Result ノード等）として機能する条件を特定するために使用される。
  • 擬似定数の判定ロジックは PlannerInfo のフラグに依存しているため、このフラグの整合性が維持されていることが前提となる。
