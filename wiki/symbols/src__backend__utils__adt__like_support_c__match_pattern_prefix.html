match_pattern_prefix

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_support.c:240

目的

パターンマッチング演算子（LIKE 等）を含むクエリに対し、インデックススキャンが可能な条件式（>= や = 等）を導出することで、クエリ実行を最適化する。

シグネチャまたは形状

List *match_pattern_prefix(Node *leftop, Node *rightop, Pattern_Type ptype, Oid expr_coll, Oid opfamily, Oid indexcollation)

入力と出力

  • **入力**:
  • leftop: 比較対象の左辺ノード。
  • rightop: パターンを含む右辺ノード（Const 型である必要がある）。
  • ptype: パターンの種類（Pattern_Type）。
  • expr_coll, indexcollation: 照合順序（Collation）情報。
  • opfamily: インデックスの演算子族。
  • **出力**:
  • インデックススキャン用の条件式を含む List。条件が生成できない場合は NIL を返す。

前提条件と不変条件

  • rightop は Const 型かつ非 NULL であること。
  • 抽出された接頭辞とインデックスの照合順序が一致していること。
  • 非決定論的（nondeterministic）な照合順序を持つ場合、完全一致（Pattern_Prefix_Exact）以外はサポートされない。

アルゴリズムまたは振る舞い

1. **接頭辞抽出**: pattern_fixed_prefix を呼び出し、パターンから固定接頭辞を試行する。抽出不可の場合は NIL を返す。 2. **演算子選択**: 左辺のデータ型（TEXT, NAME, BPCHAR, BYTEA）および opfamily に基づき、適切な比較演算子（eq, lt, ge）を選択する。 3. **型変換**: 必要に応じて接頭辞定数をターゲットのデータ型に適合させる。 4. **条件生成**:

  • 完全一致の場合: eq 演算子を用いた条件式を生成する。
  • 部分一致の場合: インデックスが直接サポートしていれば pre 演算子を、そうでなければ範囲条件の構築を試みる（ソース excerpt はここで終了）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/like_support.c 内の textlike_support, texticlike_support 等のサポート関数から呼び出される。
  • **呼び出し先**:
  • pattern_fixed_prefix: 接頭辞の抽出。
  • exprType, get_collation_isdeterministic, op_in_opfamily, make_opclause: ユーティリティ関数。

関連データ構造

  • Pattern_Type: パターンマッチングの種類を定義する列挙型。
  • Pattern_Prefix_Status: 接頭辞抽出の状態（Pattern_Prefix_None, Pattern_Prefix_Exact 等）。
  • List: 生成された条件式を保持するノードリスト。

失敗モードと境界条件

  • rightop が定数でない、または NULL の場合は NIL を返す。
  • 演算子がインデックスの opfamily に含まれていない場合は NIL を返す。
  • 照合順序の不一致や、非決定論的照合順序における不適合なパターンは NIL を返す。

保守担当者向けメモ

  • 本関数は「損失のあるインデックススキャン」を可能にするための重要なフックである。
  • TEXT_PATTERN_BTREE_FAM_OID 等のレガシーな演算子族と、標準的な比較演算子の使い分けがロジックに含まれているため、新しいデータ型や演算子族を追加する際は注意が必要である。
