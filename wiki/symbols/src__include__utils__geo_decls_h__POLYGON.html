POLYGON

定義

  • 種別: struct
  • 定義箇所: src/include/utils/geo_decls.h:151

目的

PostgreSQLにおける多角形（Polygon）データを保持するためのC言語構造体。SQLレベルの幾何型とメモリ上の表現を橋渡しする。

シグネチャまたは形状

typedef struct {
    int32       vl_len_;    /* varlena header */
    int32       npts;       /* 頂点数 */
    BOX         boundbox;   /* 外接矩形 */
    Point       p[FLEXIBLE_ARRAY_MEMBER]; /* 頂点配列 */
} POLYGON;

入力と出力

  • 本構造体は varlena 型であり、PostgreSQLの関数マネージャを介して Datum から取得される。
  • アクセス時には PG_DETOAST_DATUM を用いたTOAST解除が必須である。

前提条件と不変条件

  • vl_len_ フィールドは直接操作してはならない。
  • p 配列は npts 個の Point を保持する。
  • boundbox は多角形全体を包含する最小の矩形として維持される必要がある。

アルゴリズムまたは振る舞い

  • src/backend/utils/adt/geo_ops.c 内の関数群により、多角形の包含判定、距離計算、円への変換、外接矩形の再計算などの操作が行われる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistproc.c（GiSTインデックス処理）、src/backend/utils/adt/geo_ops.c（幾何演算関数）。
  • **呼び出し先**: make_bound_box（外接矩形計算）、poly_to_circle、lseg_inside_poly、poly_contain_poly、dist_ppoly_internal、dist_cpoly_internal 等の幾何演算ルーチン。

関連データ構造

  • Point: 頂点座標を保持。
  • BOX: 外接矩形を保持。
  • CIRCLE: 多角形との演算対象となる円型。

失敗モードと境界条件

  • varlena 型であるため、TOAST解除を怠るとメモリ破損や不正なデータアクセスを引き起こす。
  • 頂点数が0または1の場合の挙動は、各演算関数の実装に依存する。

保守担当者向けメモ

  • 本構造体は可変長データ型であるため、メモリレイアウトの変更には細心の注意を払うこと。
  • 数値演算は FPeq 等の曖昧比較関数を用いて行われるが、推移律を満たさない可能性があるため、厳密な計算が必要なロジックでは注意が必要。
