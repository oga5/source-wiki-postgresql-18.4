inet_inclusion_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_selfuncs.c:896

目的

2つの inet 型アドレスの包含関係を判定するための比較関数。ネットワークアドレスの階層構造に基づいた順序付けを行い、選択率推定の計算を支援する。

シグネチャまたは形状

static int inet_inclusion_cmp(inet *left, inet *right, int opr_codenum)

入力と出力

  • **入力**:
  • left, right: 比較対象の inet 型ポインタ。
  • opr_codenum: 演算子の種類を示す数値コード。
  • **出力**:
  • int: 比較結果。0 は一致、負または正の値は順序関係を示す。

前提条件と不変条件

  • left および right は有効な inet 型構造体であること。
  • ip_family が一致する場合、ビット単位の比較が優先される。

アルゴリズムまたは振る舞い

1. **アドレスファミリーの比較**: ip_family が異なる場合、その差分を返して終了する。 2. **ビット単位の比較**: ip_family が同じ場合、両者のネットマスク長のうち短い方を基準として bitncmp を実行する。 3. **順序の確定**: bitncmp の結果が 0 でない場合はその値を返す。 4. **マスク長による補完**: ビット比較が一致した場合、inet_masklen_inclusion_cmp を呼び出し、マスク長と演算子コードに基づいた詳細な包含関係を判定して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_selfuncs.c 内の 645 行目および 652 行目（ヒストグラムの比較処理）。
  • **呼び出し先**:
  • bitncmp
  • inet_masklen_inclusion_cmp

関連データ構造

  • inet: ネットワークアドレスを保持する構造体。
  • ip_family, ip_addr, ip_bits: inet 構造体のフィールドにアクセスするためのマクロ。

失敗モードと境界条件

  • アドレスファミリーが異なる場合、ビット比較を行わずにファミリーの差分を返すため、論理的な順序はアドレス値そのものよりもファミリーの数値に依存する。

保守担当者向けメモ

  • 本関数は inet 型の統計推定における比較ロジックの基盤である。
  • inet_masklen_inclusion_cmp への依存があるため、包含関係の定義を変更する場合は両方の関数を確認すること。
