transfer_single_new_db

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/relfilenumber.c:500

目的

指定されたデータベース内のリレーションファイル群を、旧クラスタから新クラスタへ移行（転送または入れ替え）します。

シグネチャまたは形状

static void transfer_single_new_db(FileNameMap *maps, int size, char *old_tablespace)

入力と出力

  • maps: 移行対象となるファイルマッピングの配列。
  • size: maps 配列の要素数。
  • old_tablespace: 処理対象とするテーブルスペース名（NULLの場合は全テーブルスペースを対象とする）。
  • 戻り値: なし（void）。

前提条件と不変条件

  • old_cluster および new_cluster の制御データが初期化済みであること。
  • --swap モードの場合、可視性マップ（VM）の再構築が必要なバージョン間アップグレードはサポートされません（Assert により検証）。

アルゴリズムまたは振る舞い

1. **VM更新判定**: 旧クラスタと新クラスタのカタログバージョンを比較し、可視性マップの凍結ビット（frozen bit）追加が必要か判定します。 2. **モード分岐**:

  • --swap モードの場合: do_swap を呼び出し、ディレクトリ入れ替えを実行して終了します。
  • 通常モードの場合: maps 配列を走査し、指定されたテーブルスペースに一致するファイルに対して以下の処理を順次行います。
  • transfer_relfile を呼び出し、プライマリファイル、FSMファイル、VMファイルを個別に転送（コピー/リンク等）します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/relfilenumber.c:207（transfer_all_new_dbs からの呼び出しが確認されています）。
  • 呼び出し先: do_swap、transfer_relfile。

関連データ構造

  • FileNameMap: 旧ファイルと新ファイルのパスおよび属性を保持する構造体。

失敗モードと境界条件

  • old_tablespace が指定されている場合、一致しないテーブルスペースのファイルはスキップされます。
  • --swap モードにおいて、VMの再構築が必要な状況が検出された場合はアサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は pg_upgrade のデータ移行フェーズにおける主要なループ処理です。
  • transfer_relfile を介してファイル操作を行うため、ファイルシステムレベルの権限やディスク容量に依存します。
  • --swap モードは特殊なパスを通るため、修正時は do_swap 側のロジックとの整合性に注意してください。
