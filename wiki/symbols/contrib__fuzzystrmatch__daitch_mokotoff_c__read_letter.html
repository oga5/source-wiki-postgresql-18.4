read_letter

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/daitch_mokotoff.c:466

目的

入力文字列から音韻ルールに基づいたコード（dm_codes）を読み取ります。文字列を走査し、最長一致する音韻単位に対応するコードを特定します。

シグネチャまたは形状

static const dm_codes *read_letter(const char *str, int *ix)

入力と出力

  • **入力**:
  • str: 処理対象の文字列。
  • ix: 現在の読み取り位置を示すインデックスへのポインタ。
  • **出力**:
  • 成功時: 対応する音韻コードの構造体へのポインタを返し、ix を読み取った文字の末尾まで進めます。
  • 失敗時: NULL を返します。

前提条件と不変条件

  • str は有効な文字列である必要があります。
  • ix は str 内の有効なオフセットを指している必要があります。
  • read_valid_char が入力の正規化と有効性の確認を担います。

アルゴリズムまたは振る舞い

1. read_valid_char を使用して最初の文字を読み取ります。失敗した場合は NULL を返します。 2. letter_ テーブルから最初の文字に対応する音韻ルールを取得します。 3. 後続の文字を順次読み取り、現在の音韻ルールツリーを辿ります。 4. より長い一致が見つかるたびに codes を更新し、ix を進めます。 5. 一致するシーケンスが途切れるか、それ以上一致する文字がない場合、最後に確定した codes を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/fuzzystrmatch/daitch_mokotoff.c 内の daitch_mokotoff_coding 関数（530行目および546行目付近）。
  • **呼び出し先**:
  • read_valid_char: 文字の読み取りと検証に使用。

関連データ構造

  • dm_codes: 音韻コードを保持する構造体。
  • dm_letter: 音韻ルールを保持するツリー状の構造体。

失敗モードと境界条件

  • 入力文字列が空、または有効な文字を含まない場合、NULL を返します。
  • 音韻ルールテーブルに存在しない文字シーケンスに遭遇した場合、その時点までに確定したコードを返します。

保守担当者向けメモ

  • 本関数は daitch_mokotoff.h で定義されたルールテーブルに依存しています。ルールテーブルの構造変更は本関数の走査ロジックに影響を与える可能性があります。
  • ix の更新は、最長一致が確定したタイミングでのみ行われる点に注意してください。
