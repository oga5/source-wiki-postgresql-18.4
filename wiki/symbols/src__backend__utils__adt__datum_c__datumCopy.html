datumCopy

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datum.c:131

目的

指定された Datum の独立したコピーを現在のメモリコンテキスト上に作成します。拡張オブジェクト（Expanded Object）が含まれる場合は、フラット化してコピーします。

シグネチャまたは形状

Datum datumCopy(Datum value, bool typByVal, int typLen)

入力と出力

  • **入力**:
  • value: コピー対象の Datum。
  • typByVal: 値渡し（pass-by-value）か否かを示すフラグ。
  • typLen: 型の長さ（バイト数）。-1 は可変長（varlena）、-2 は C 文字列を意味します。
  • **出力**:
  • コピーされたデータを含む新しい Datum を返します。

前提条件と不変条件

  • typByVal および typLen は、対象となるデータ型に対して正確である必要があります。誤った値はメモリ破壊や不正なポインタ参照を引き起こします。
  • palloc を使用するため、呼び出し元は適切なメモリコンテキストがアクティブであることを保証する必要があります。

アルゴリズムまたは振る舞い

1. **値渡しの場合**: typByVal が真であれば、value をそのまま返します。 2. **可変長（varlena）の場合** (typLen == -1):

  • 拡張オブジェクト（VARATT_IS_EXTERNAL_EXPANDED）であれば、EOH_get_flat_size でサイズを算出し、EOH_flatten_into を用いてフラットなメモリ領域へコピーします。
  • 通常の varlena であれば、VARSIZE_ANY でサイズを特定し、palloc 後の領域へ memcpy します。

    • 3. **参照渡しの場合**: datumGetSize でサイズを算出し、palloc 後の領域へ memcpy します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/brin/brin_inclusion.c
  • src/backend/access/brin/brin_minmax.c
  • src/backend/access/brin/brin_minmax_multi.c
  • **呼び出し先**:
  • palloc, memcpy, VARSIZE_ANY, DatumGetPointer, PointerGetDatum, DatumGetEOHP, EOH_get_flat_size, EOH_flatten_into, datumGetSize

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • varlena: 可変長データ型の構造体。
  • ExpandedObjectHeader: 拡張オブジェクトのヘッダー。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • typByVal や typLen が実際のデータ型と一致しない場合、memcpy やポインタ操作により未定義動作が発生します。

保守担当者向けメモ

  • 本関数は TOAST ポインタをそのまま扱うことが基本ですが、拡張オブジェクトについてはフラット化を行います。これは呼び出し元がメモリ寿命を意識せずに済むようにするためです。
  • 比較操作（datumIsEqual 等）とは異なり、本関数はデータの物理的な複製を目的としています。
