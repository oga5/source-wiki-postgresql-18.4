run_ssl_passphrase_command

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-common.c:39

目的

ssl_passphrase_command 設定で指定された外部コマンドを実行し、SSL秘密鍵の復号に必要なパスフレーズを安全に取得してバッファに格納します。

シグネチャまたは形状

int run_ssl_passphrase_command(const char *prompt, bool is_server_start, char *buf, int size);

入力と出力

  • **入力**:
  • prompt: コマンドに渡すプロンプト文字列。
  • is_server_start: サーバー起動時かどうかを示すフラグ。エラー時のログレベル決定に使用。
  • buf: パスフレーズを格納するバッファ。
  • size: buf のサイズ。
  • **出力**:
  • 戻り値: 取得したパスフレーズの長さ（size_t）。エラー時は 0。
  • buf: 取得したパスフレーズ（末尾の改行は除去済み）。

前提条件と不変条件

  • prompt は非 NULL であること。
  • size は 0 より大きいこと。
  • 実行前に buf[0] は \0 で初期化されます。
  • エラー発生時には explicit_bzero を使用してバッファがクリアされます。

アルゴリズムまたは振る舞い

1. replace_percent_placeholders を用いて ssl_passphrase_command 文字列内のプレースホルダーを置換し、実行コマンドを生成します。 2. OpenPipeStream でコマンドを読み取りモードで実行します。 3. fgets で出力を buf に読み込みます。 4. ClosePipeStream でパイプを閉じ、終了ステータスを確認します。 5. pg_strip_crlf を呼び出し、取得した文字列から末尾の改行および復帰文字を除去します。 6. 最後に生成したコマンド文字列を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure-openssl.c 内の関数から呼び出されます。
  • **呼び出し先**: replace_percent_placeholders, OpenPipeStream, fgets, ferror, explicit_bzero, ClosePipeStream, wait_result_to_str, ereport, pfree, pg_strip_crlf。

関連データ構造

  • ssl_passphrase_command: 外部から参照される設定値（グローバル変数）。

失敗モードと境界条件

  • コマンドの実行失敗、読み取りエラー、またはパイプのクローズ失敗時には、is_server_start に応じて ERROR または LOG レベルで ereport を発行します。
  • サーバー起動時（is_server_start が true）のエラーは ERROR となり、プロセスを停止させます。
  • 失敗時は explicit_bzero でバッファを確実に消去します。

保守担当者向けメモ

  • 本関数は be-secure-openssl.c 等のライブラリ固有実装から利用される共通ユーティリティです。
  • 外部コマンド実行時のエラーハンドリングは、PostgreSQLのサーバー起動プロセスに影響を与える可能性があるため注意してください。
