fatal_system_new_handler

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_error.cpp:113

目的

C++のメモリ割り当て失敗（std::bad_alloc）発生時に、PostgreSQLの標準的なエラーハンドリング機構を介してプロセスを安全に終了させるためのカスタムハンドラです。

シグネチャまたは形状

static void fatal_system_new_handler(void);

入力と出力

  • 入力: なし
  • 出力: なし（ereport(FATAL, ...) によりプロセスが終了するため、呼び出し元へは復帰しません）

前提条件と不変条件

  • llvm_enter_fatal_on_oom() が呼び出され、本関数が std::set_new_handler を介してC++のメモリ不足ハンドラとして登録されている必要があります。

アルゴリズムまたは振る舞い

1. ereport(FATAL, ...) を呼び出します。 2. エラーコードとして ERRCODE_OUT_OF_MEMORY を指定します。 3. エラーメッセージとして "out of memory" を、詳細情報として "while in LLVM" を報告し、PostgreSQLのプロセスを終了させます。

呼び出し元と呼び出し先

  • 呼び出し元: C++のランタイム（メモリ割り当て失敗時）。src/backend/jit/llvm/llvmjit_error.cpp:59 にて std::set_new_handler に登録されます。
  • 呼び出し先: ereport（PostgreSQLのコアエラー報告関数）。

関連データ構造

  • なし

失敗モードと境界条件

  • 本関数は回復不能なメモリ不足状態を前提としています。
  • FATAL エラーとして報告されるため、トランザクションのロールバックではなく、プロセス全体の終了を伴います。

保守担当者向けメモ

  • 本関数は extern "C" の文脈で動作するC++関数です。
  • LLVM内部でのメモリ不足をPostgreSQLのプロセス管理と整合させるための防波堤として機能します。他のエラーハンドラと競合しないよう注意してください。
