l_struct_gep

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/jit/llvmjit_emit.h:107

目的

LLVM IR において、構造体の特定のフィールドを指すポインタを計算する GetElementPtr (GEP) 命令の生成を簡略化するラッパー関数です。

シグネチャまたは形状

LLVMValueRef l_struct_gep(LLVMBuilderRef b, LLVMTypeRef t, LLVMValueRef v, int32 idx, const char *name)

入力と出力

  • **入力**:
  • b: 命令を挿入する LLVMBuilderRef。
  • t: 対象となる構造体の LLVMTypeRef。
  • v: 構造体へのポインタを保持する LLVMValueRef。
  • idx: アクセスするフィールドのインデックス（int32）。
  • name: 命令に付与する名前（現在の実装では無視され、空文字列が使用される）。
  • **出力**:
  • 指定されたフィールドを指すポインタを表す LLVMValueRef。

前提条件と不変条件

  • b は有効な LLVMBuilderRef であること。
  • t は v が指す構造体の型と一致していること。
  • idx は構造体 t の有効なフィールドインデックスであること。

アルゴリズムまたは振る舞い

内部で LLVMBuildStructGEP2 を呼び出し、指定された構造体型 t とインデックス idx に基づいて、ポインタ演算命令を生成します。引数 name は使用されず、空文字列として渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit_deform.c
  • src/backend/jit/llvm/llvmjit_expr.c

    • (主にタプルスロットのフィールドアクセスや式評価のコード生成で使用)

  • **呼び出し先**:
  • LLVMBuildStructGEP2 (LLVM C API)

関連データ構造

  • LLVMBuilderRef: 命令生成器。
  • LLVMTypeRef: LLVM IR 上の型定義。
  • LLVMValueRef: LLVM IR 上の値（ポインタや定数）。
  • FIELDNO_... 定数: llvmjit.h で定義されるフィールド番号定数と組み合わせて使用されることが一般的です。

失敗モードと境界条件

  • idx が構造体の範囲外である場合、LLVM の内部 API によりエラーが発生する可能性があります。
  • v が構造体へのポインタではない場合、LLVM の型チェックにより失敗します。

保守担当者向けメモ

  • 本関数は LLVMBuildStructGEP2 の単なるラッパーであり、name 引数が無視される仕様です。将来的にデバッグ情報を強化する場合、この引数を活用する可能性があります。
  • USE_LLVM が定義されている環境でのみ利用可能です。
