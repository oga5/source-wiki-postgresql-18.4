pull_up_sublinks_jointree_recurse

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/prep/prepjointree.c:494

目的

ジョインツリーを再帰的に走査し、クエリ内のサブリンク（ANY/EXISTS）をジョイン構造へ引き上げるための前処理を行う。

シグネチャまたは形状

static Node *pull_up_sublinks_jointree_recurse(PlannerInfo *root, Node *jtnode, Relids *relids);

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • jtnode: 処理対象のジョインツリーノード（RangeTblRef, FromExpr, JoinExpr）。
  • relids: 処理されたノードが参照するリレーションIDを格納するための出力用ポインタ。
  • **出力**:
  • 変換後のジョインツリーノード（Node *）。

前提条件と不変条件

  • 再帰的な呼び出しを行うため、スタックオーバーフローを防ぐために check_stack_depth() を実行する。
  • JoinExpr の場合、JOIN_SEMI, JOIN_ANTI, JOIN_RIGHT_SEMI, JOIN_RIGHT_ANTI は入力として想定されていない。
  • 処理順序は厳密であり、この関数による変換後に後続の最適化ステップでジョインの平坦化や再配置が行われることを前提とする。

アルゴリズムまたは振る舞い

1. **RangeTblRef**: リレーションIDを抽出し、ノードをそのまま返す。 2. **FromExpr**:

  • 子ノードを再帰的に処理し、relids を収集する。
  • pull_up_sublinks_qual_recurse を呼び出し、quals 内のサブリンクをジョインへ変換する。

    • 3. **JoinExpr**:

  • ノードのコピーを作成し、左右の子ノードを再帰的に処理する。
  • jointype に応じて quals を処理する：
  • JOIN_INNER: 両側の子の relids を使用して quals を処理。
  • JOIN_LEFT/JOIN_RIGHT: それぞれ対応する側の relids を使用して quals を処理。
  • JOIN_FULL: quals の引き上げは行わない。
  • 最後に、自身の rtindex があれば relids に含める。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の pull_up_sublinks および本関数自身による再帰呼び出し。
  • **呼び出し先**: pull_up_sublinks_qual_recurse（クエリ条件の変換）、check_stack_depth（スタック保護）。

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • FromExpr, JoinExpr, RangeTblRef: ジョインツリーを構成するノード。
  • Relids: リレーションIDの集合。

失敗モードと境界条件

  • jtnode が未知の型である場合、elog(ERROR) を発生させる。
  • jointype が JOIN_INNER, JOIN_LEFT, JOIN_FULL, JOIN_RIGHT 以外の場合、elog(ERROR) を発生させる。
  • 深いジョインツリーに対してはスタックオーバーフローの可能性がある（check_stack_depth で検知）。

保守担当者向けメモ

  • 本関数は pull_up_sublinks の中核となる再帰処理である。
  • JoinExpr の処理において、jtlink を介して新しいジョイン構造を挿入する仕組みは、外結合の null 伝播を正しく扱うために重要である。
  • relids の計算において、JoinExpr の rtindex を含める必要があるのは、ジョインエイリアス変数がまだ崩されていないためである。
