hex2_to_uchar

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac8.c:58

目的

16進数2桁の文字列を、対応する1バイトの unsigned char 値に変換します。

シグネチャまたは形状

static inline unsigned char hex2_to_uchar(const unsigned char *ptr, bool *badhex)

入力と出力

  • **入力**:
  • ptr: 16進数文字（'0'-'9', 'a'-'f', 'A'-'F'）を指すポインタ。
  • badhex: 変換失敗時に true が設定されるフラグへのポインタ。
  • **出力**:
  • 変換後の unsigned char 値。失敗時は 0 を返します。

前提条件と不変条件

  • ptr は少なくとも2バイトの有効なメモリ領域を指している必要があります。
  • hexlookup 配列が有効な16進数変換テーブルとして存在している必要があります。

アルゴリズムまたは振る舞い

1. 1文字目を読み取り、hexlookup を用いて数値に変換します。 2. 変換結果を4ビット左シフトし、上位4ビットとして保持します。 3. ptr をインクリメントし、2文字目を同様に数値へ変換します。 4. 2文字目の数値を下位4ビットとして加算し、結果を返します。 5. 入力文字が127を超える場合、または hexlookup で負の値（無効な文字）が返された場合は、badhex を true に設定し、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/mac8.c 内の関数（134行目から155行目にかけて、MACアドレスの各バイトを解析する際に繰り返し呼び出されています）。
  • **呼び出し先**: 外部関数への直接的な呼び出しはありませんが、グローバルな hexlookup 配列を参照します。

関連データ構造

  • hexlookup: 16進数文字から数値へのマッピングを保持する配列（ソースファイル内で定義）。

失敗モードと境界条件

  • **無効な文字**: 16進数以外の文字が入力された場合、badhex が true に設定されます。
  • **範囲外の文字**: ASCII値が127を超える文字が入力された場合、エラーとして扱われます。

保守担当者向けメモ

  • 本関数は inline 指定されており、パフォーマンスを重視して設計されています。
  • 呼び出し元では、本関数が返す badhex フラグを確認し、true の場合に ereport() を発行してエラーを通知する設計となっています。
