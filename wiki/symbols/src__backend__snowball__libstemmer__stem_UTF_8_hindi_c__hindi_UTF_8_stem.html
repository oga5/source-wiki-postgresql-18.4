hindi_UTF_8_stem

定義

  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_hindi.c:302

目的

UTF-8でエンコードされたヒンディー語の単語に対し、末尾の接尾辞を削除して語幹を抽出する。

シグネチャまたは形状

int hindi_UTF_8_stem(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境を保持する構造体へのポインタ）
  • 出力: 処理が成功した場合は 1、失敗した場合は 0 または負の値を返す。

前提条件と不変条件

  • z は hindi_UTF_8_create_env によって初期化されている必要がある。
  • 入力文字列は UTF-8 エンコーディングであること。

アルゴリズムまたは振る舞い

1. skip_utf8 を呼び出し、入力文字列の先頭をスキップする。失敗時は 0 を返す。 2. ステミング環境の境界を更新し、末尾から検索を開始する準備を行う。 3. find_among_b を使用し、a_0 テーブル（132個の接尾辞パターン）と照合して末尾の接尾辞を検索する。 4. 一致した接尾辞を slice_del で削除する。削除に失敗した場合はそのエラーコードを返す。 5. 処理終了後、環境のカーソル位置を復元し 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの全文検索エンジン（詳細は提供されたソース範囲外のため不明）。
  • 呼び出し先: skip_utf8, find_among_b, slice_del（いずれも header.h で定義されたSnowballランタイム関数）。

関連データ構造

  • struct SN_env: ステミング処理の状態を保持する構造体。
  • a_0: ヒンディー語の接尾辞パターンを格納した among 構造体の配列。

失敗モードと境界条件

  • skip_utf8 が負の値を返した場合、処理を中断し 0 を返す。
  • find_among_b が一致する接尾辞を見つけられなかった場合、0 を返す。
  • slice_del が負の値を返した場合、その値をそのまま呼び出し元へ伝播させる。
