ExecBitmapHeapReInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapHeapscan.c:532

目的

並列ビットマップヒープスキャンにおいて、共有メモリ（DSM）上の状態を再初期化し、スキャンをリセットする。

シグネチャまたは形状

void ExecBitmapHeapReInitializeDSM(BitmapHeapScanState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: BitmapHeapScanState 構造体へのポインタ。
  • pcxt: ParallelContext 構造体へのポインタ（未使用）。
  • **出力**: なし（副作用として共有メモリの状態を更新）。

前提条件と不変条件

  • node->pstate が有効な ParallelBitmapHeapState を指していること。
  • node->ss.ps.state->es_query_dsa が共有メモリ領域（DSA）を提供していること。

アルゴリズムまたは振る舞い

1. node から DSA 領域を取得する。DSA が存在しない場合は何もしない。 2. pstate->state を BM_INITIAL にリセットする。 3. pstate->tbmiterator が有効な場合、tbm_free_shared_area を呼び出して共有メモリ上のイテレータを解放する。 4. pstate->tbmiterator を InvalidDsaPointer に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c:1007
  • **呼び出し先**:
  • tbm_free_shared_area (ビットマップイテレータの解放)

関連データ構造

  • BitmapHeapScanState: スキャンノードの実行状態を保持。
  • ParallelBitmapHeapState: 並列実行時の共有状態（pstate）。
  • dsa_area: 動的共有メモリ領域。

失敗モードと境界条件

  • dsa が NULL の場合、関数は即座にリターンし、何も変更しない。
  • tbmiterator が既に無効な場合、解放処理はスキップされる。

保守担当者向けメモ

  • 本関数は ExecReScan 等の文脈で、並列スキャンの再実行時に共有状態をクリーンアップするために使用される。
  • tbm_free_shared_area の呼び出しは、共有メモリリークを防ぐために必須である。
