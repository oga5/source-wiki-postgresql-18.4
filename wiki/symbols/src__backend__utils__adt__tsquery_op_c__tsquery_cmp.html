tsquery_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_op.c:214

目的

2つの tsquery 型の値の大小を比較し、その結果を整数で返します。

シグネチャまたは形状

Datum tsquery_cmp(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの tsquery 型引数を受け取ります。
  • **出力**: 比較結果を示す int32 型の値を返します（a < b なら負、a == b なら 0、a > b なら正）。

前提条件と不変条件

  • 入力された tsquery は、PG_GETARG_TSQUERY_COPY によりメモリ上にコピーされた状態で取得されます。
  • 比較処理終了後、PG_FREE_IF_COPY を用いて一時的に確保したメモリを解放する必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_TSQUERY_COPY を使用して、2つの入力引数 a および b を取得します。 2. CompareTSQ(a, b) を呼び出し、内部的なツリー構造に基づいた比較を実行します。 3. PG_FREE_IF_COPY を呼び出し、引数として受け取ったメモリ領域を適切に解放します。 4. PG_RETURN_INT32 を使用して、比較結果 res を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: CompareTSQ（比較ロジックの実行）、PG_GETARG_TSQUERY_COPY、PG_FREE_IF_COPY、PG_RETURN_INT32。
  • **呼び出し元**: ソースコード内に明示的な呼び出し元の記述はありません。

関連データ構造

  • TSQuery: 全文検索クエリを保持するデータ型。
  • QTNode: CompareTSQ 内で比較のために展開されるツリー構造（tsearch/ts_utils.h で定義）。

失敗モードと境界条件

  • 入力引数が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で適切に処理される必要があります。
  • メモリ不足時には、PG_GETARG_TSQUERY_COPY 等の内部処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は tsquery 型の比較演算子を実装するためのラッパーです。
  • 比較ロジックの詳細は CompareTSQ に依存しているため、比較アルゴリズムの変更が必要な場合は CompareTSQ を参照してください。
