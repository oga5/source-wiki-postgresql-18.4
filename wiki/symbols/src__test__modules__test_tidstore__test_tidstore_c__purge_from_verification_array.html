purge_from_verification_array

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_tidstore/test_tidstore.c:168

目的

検証用データ構造 items（ItemArray 型）から、指定されたブロック番号に関連付けられたすべての ItemPointer を削除し、配列を再構成する。

シグネチャまたは形状

static void purge_from_verification_array(BlockNumber blkno)

入力と出力

  • **入力**: blkno (削除対象の BlockNumber)
  • **出力**: なし（副作用として静的変数 items の内容を更新する）

前提条件と不変条件

  • items が初期化済みであること。
  • items.insert_tids 配列が有効な ItemPointer を保持していること。

アルゴリズムまたは振る舞い

1. items.num_tids を上限として配列を走査する。 2. 各要素のブロック番号が引数 blkno と一致しない場合、その要素を配列の先頭側（dst インデックス）へ詰め直す。 3. 走査終了後、items.num_tids を dst（残った要素数）に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_tidstore/test_tidstore.c:204（ソース内の特定の箇所から呼び出されていることが確認できる）
  • **呼び出し先**:
  • ItemPointerGetBlockNumber（マクロ）

関連データ構造

  • ItemArray: テストの整合性検証のために使用される、挿入済みデータを保持する構造体。

失敗モードと境界条件

  • items.num_tids が 0 の場合、ループは実行されず何も変更されない。
  • 指定した blkno が配列内に存在しない場合、配列の内容は変更されない。

保守担当者向けメモ

  • 本関数は TidStore のテストモジュール内でのみ使用される補助関数である。
  • items は静的変数であるため、本関数の実行はグローバルな検証状態に直接影響を与える。並行実行環境での使用は想定されていない。
