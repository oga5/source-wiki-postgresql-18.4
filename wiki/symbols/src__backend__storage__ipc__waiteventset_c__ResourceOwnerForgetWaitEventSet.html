ResourceOwnerForgetWaitEventSet

定義

  • 種別: static inline function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:227

目的

指定された WaitEventSet を ResourceOwner の管理対象から除外します。

シグネチャまたは形状

static inline void ResourceOwnerForgetWaitEventSet(ResourceOwner owner, WaitEventSet *set)

入力と出力

  • owner: リソースを所有している ResourceOwner オブジェクト。
  • set: 管理対象から外す WaitEventSet へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • owner が有効な ResourceOwner であること。
  • set が以前に ResourceOwnerRememberWaitEventSet を通じて当該 owner に登録されていること。

アルゴリズムまたは振る舞い

内部で ResourceOwnerForget を呼び出し、wait_event_set_resowner_desc を記述子として使用して、set を owner のリソースリストから削除します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/waiteventset.c 内の FreeWaitEventSet 関数（行485）から呼び出されています。
  • 呼び出し先: ResourceOwnerForget（utils/resowner.h で定義される汎用リソース管理関数）。

関連データ構造

  • ResourceOwner: リソースのライフサイクルを追跡する構造体。
  • WaitEventSet: 待機対象イベントを保持する構造体。
  • wait_event_set_resowner_desc: ResourceOwner がリソースを識別・解放するために使用する記述子。

失敗モードと境界条件

  • owner が NULL の場合、ResourceOwnerForget の実装に依存します。
  • set が owner に登録されていない場合、ResourceOwnerForget 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、主に FreeWaitEventSet によるリソース解放時のクリーンアップ処理として利用されます。
  • ResourceOwner による自動的なリソース解放メカニズムの一部であるため、変更時は utils/resowner.h のインターフェースとの整合性に注意してください。
