inet_gist_union

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_gist.c:504

目的

GiSTインデックスにおいて、複数のネットワーク型（INET / CIDR）キーを統合し、それらを包含する単一の親ノード用キーを生成します。

シグネチャまたは形状

Datum inet_gist_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GistEntryVector を受け取ります。これには統合対象となる複数の GISTENTRY が含まれます。
  • **出力**: 統合された結果を表す GistInetKey 構造体へのポインタを Datum として返します。

前提条件と不変条件

  • 入力された GistEntryVector は有効な GISTENTRY の配列である必要があります。
  • 統合されるキーのアドレスファミリー（IPv4/IPv6）が混在する場合、結果のファミリーは 0（未定義/混合）として扱われます。

アルゴリズムまたは振る舞い

1. calc_inet_union_params を呼び出し、入力された全キーから共通のプレフィックス長（commonbits）と最小ネットマスク長（minbits）、およびアドレスファミリーの範囲を算出します。 2. 算出されたファミリーが一致しない場合、ファミリー番号を 0 に設定します。 3. 最初のキーからベースとなるIPアドレスを取得します。 4. build_inet_union_key を使用し、算出したパラメータとアドレスを用いて新しい GistInetKey を構築して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの構築または更新プロセス（PostgreSQLのGiSTフレームワークから呼び出されます）。
  • **呼び出し先**:
  • calc_inet_union_params（パラメータ算出）
  • DatumGetInetKeyP（キーの抽出）
  • gk_ip_addr（アドレス取得）
  • build_inet_union_key（キー構築）

関連データ構造

  • GistEntryVector: 統合対象のキー群を保持する構造体。
  • GistInetKey: ネットワーク型GiSTインデックスの内部表現。
  • GISTENTRY: GiSTインデックスの各エントリを保持する汎用構造体。

失敗モードと境界条件

  • 入力ベクトルが空の場合の挙動については、本ソースコード内では明示的なガード句は確認できません。
  • アドレスファミリーが混在する場合、minfamily が 0 に強制されるため、以降の検索処理ではこの値が「混合」を意味するフラグとして扱われる必要があります。

保守担当者向けメモ

  • 本関数は calc_inet_union_params に依存しており、インデックスの枝刈り効率は同関数の計算精度に直結します。
  • GistInetKey は可変長データとして扱われるため、メモリ管理には注意が必要です。
