SPI_sql_row_to_xmlelement

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xml.c:4084

目的

SPI（Server Programming Interface）を通じて取得した行データを、XML要素形式に変換して StringInfo バッファへ書き出します。

シグネチャまたは形状

static void SPI_sql_row_to_xmlelement(
    uint64 rownum, 
    StringInfo result, 
    char *tablename, 
    bool nulls, 
    bool tableforest, 
    const char *targetns, 
    bool top_level
)

入力と出力

  • **入力**:
  • rownum: 処理対象の行番号。
  • result: XML文字列を蓄積する StringInfo バッファ。
  • tablename: 要素名に使用するテーブル名（NULLの場合はデフォルト値を使用）。
  • nulls: NULL値の属性を出力するかどうかのフラグ。
  • tableforest: XMLフォレスト形式（ルート要素あり）か、単一の行要素形式かを選択するフラグ。
  • targetns: 名前空間URI。
  • top_level: ルート要素の開始タグ生成を制御するフラグ。
  • **出力**: なし（result バッファを直接更新）。

前提条件と不変条件

  • SPI_tuptable が有効な状態である必要があります。
  • libxml2 の初期化が完了している必要があります。

アルゴリズムまたは振る舞い

1. **要素名の決定**: tablename が指定されていればそれをXML名に変換し、指定がなければ tableforest の値に応じて "row" または "table" を使用します。 2. **開始タグの生成**: tableforest が真であれば xmldata_root_element_start を呼び出し、偽であれば <row> を書き込みます。 3. **列データの反復処理**: SPI_tuptable の全列を走査します。

  • 各列名をXML名に変換します。
  • SPI_getbinval で値を取得し、NULLであれば nulls フラグに応じて xsi:nil="true" 属性を出力します。
  • 値が存在する場合は map_sql_value_to_xml_value でXML形式に変換して出力します。

    • 4. **終了タグの生成**: tableforest が真であれば xmldata_root_element_end を呼び出し、偽であれば </row> を書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/xml.c 内の複数箇所（行データからXMLを生成する関数群から呼び出されています）。
  • **呼び出し先**:
  • map_sql_identifier_to_xml_name
  • xmldata_root_element_start / xmldata_root_element_end
  • appendStringInfo / appendStringInfoString / appendStringInfoChar
  • SPI_fname / SPI_getbinval / SPI_gettypeid
  • map_sql_value_to_xml_value

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • SPI_tuptable: SPI実行結果を保持する構造体。

失敗モードと境界条件

  • SPI_tuptable が初期化されていない場合、メモリ参照エラーが発生する可能性があります。
  • map_sql_identifier_to_xml_name 等の内部関数がエラーを発生させた場合、PostgreSQLの例外処理機構（ereport）により処理が中断されます。

保守担当者向けメモ

  • 本関数は SPI_tuptable というグローバルなSPI実行結果に依存しています。
  • libxml2 のエラーハンドリングは、モジュール全体で定義されている PgXmlErrorContext を通じて行われます。
