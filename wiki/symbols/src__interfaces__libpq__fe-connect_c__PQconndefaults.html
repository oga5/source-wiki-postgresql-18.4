PQconndefaults

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-connect.c:2192

目的

libpqがサポートする接続オプションのデフォルト値リストを取得する。

シグネチャまたは形状

PQconninfoOption *PQconndefaults(void);

入力と出力

  • 入力: なし
  • 出力: PQconninfoOption 構造体の配列へのポインタ。配列の末尾は keyword フィールドが NULL である要素で示される。メモリ不足等の失敗時は NULL を返す。

前提条件と不変条件

  • 呼び出し元は、返された PQconninfoOption 配列を PQconninfoFree を用いて解放する責任がある。
  • 内部的に PQExpBuffer を使用して一時的なエラーバッファを構築する。

アルゴリズムまたは振る舞い

1. initPQExpBuffer で一時的なエラーバッファを初期化する。 2. conninfo_init を呼び出し、接続オプションの初期構造体配列を生成する。 3. conninfo_add_defaults を呼び出し、環境変数やシステム設定からデフォルト値を配列に充填する。 4. 充填に失敗した場合は PQconninfoFree でメモリを解放し、NULL を返す。 5. 成功した場合は、一時バッファを破棄し、構築された配列を返す。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • contrib/dblink/dblink.c
  • contrib/postgres_fdw/option.c
  • src/bin/pg_upgrade/server.c
  • src/bin/psql/command.c
  • 呼び出し先:
  • initPQExpBuffer / termPQExpBuffer
  • conninfo_init
  • conninfo_add_defaults
  • PQconninfoFree

関連データ構造

  • PQconninfoOption: 接続パラメータ（キーワード、環境変数名、デフォルト値、ヘルプテキスト等）を保持する構造体。

失敗モードと境界条件

  • メモリ不足: initPQExpBuffer の失敗時、または conninfo_init / conninfo_add_defaults の内部処理でメモリ確保に失敗した場合、NULL を返す。

保守担当者向けメモ

  • 本関数は malloc を使用してメモリを確保するため、呼び出し側でのメモリ管理（PQconninfoFree の呼び出し）が必須である。
  • conninfo_add_defaults に NULL を渡しているため、デフォルト値解決時のエラーメッセージは無視される仕様となっている。
