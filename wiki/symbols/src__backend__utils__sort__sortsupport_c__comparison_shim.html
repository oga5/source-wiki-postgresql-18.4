comparison_shim

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sortsupport.c:42

目的

従来のSQL呼び出し可能な比較関数（BTORDER_PROC）を、SortSupport インターフェース（ssup->comparator）に適合させるためのラッパー関数です。

シグネチャまたは形状

static int comparison_shim(Datum x, Datum y, SortSupport ssup);

入力と出力

  • **入力**:
  • x, y: 比較対象の Datum 値。
  • ssup: ソート状態を保持する SortSupport 構造体。ssup_extra フィールドに SortShimExtra 構造体が含まれている必要があります。
  • **出力**:
  • 比較結果を示す整数値（x < y なら負、x == y なら 0、x > y なら正）。

前提条件と不変条件

  • ssup->ssup_extra は SortShimExtra 型としてキャスト可能であること。
  • ssup_extra 内の fcinfo（関数呼び出し情報）は、事前に適切に初期化されていること。

アルゴリズムまたは振る舞い

1. ssup->ssup_extra から SortShimExtra を取得します。 2. 比較対象の x および y を fcinfo.args に設定します。 3. fcinfo.isnull を false にリセットします。 4. FunctionCallInvoke を呼び出し、従来の比較関数を実行します。 5. 実行結果が NULL であればエラーを発生させます。 6. 比較関数の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソート実行時に ssup->comparator を介して呼び出されます。
  • **呼び出し先**: FunctionCallInvoke（fmgr.h で定義される関数呼び出しインターフェース）。

関連データ構造

  • SortSupport: ソートの比較ロジックや状態を保持する構造体。
  • SortShimExtra: Shim 実行に必要な関数呼び出し情報（fcinfo）や関数情報（flinfo）を保持する構造体。

失敗モードと境界条件

  • 比較関数が NULL を返した場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は SortSupport の高速化パスが存在しない場合のフォールバックとして使用されます。
  • ssup_extra のメモリ管理は ssup_cxt 上で行う必要があるため、変更時はメモリリークに注意してください。
