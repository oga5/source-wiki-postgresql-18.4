spgChooseIn

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist.h:53

目的

SP-GiSTインデックスへの新しい値の挿入先を決定する際、現在の内部タプル（inner tuple）の状態を演算子クラスの choose 関数へ渡すための入力構造体です。

シグネチャまたは形状

typedef struct spgChooseIn
{
    Datum       datum;          /* インデックス対象の元のデータ */
    Datum       leafDatum;      /* リーフに格納される現在のデータ */
    int         level;          /* 現在のレベル（0から開始） */

    /* 現在の内部タプルに関するデータ */
    bool        allTheSame;     /* タプルが all-the-same マークされているか */
    bool        hasPrefix;      /* タプルがプレフィックスを持つか */
    Datum       prefixDatum;    /* プレフィックス値 */
    int         nNodes;         /* 内部タプル内のノード数 */
    Datum      *nodeLabels;     /* ノードラベル値（存在しない場合はNULL） */
} spgChooseIn;

入力と出力

  • **入力**: 挿入対象の値（datum, leafDatum）、現在の木構造の深さ（level）、および比較対象となる内部タプルの属性情報。
  • **出力**: 本構造体自体は入力専用です。この構造体を受け取った choose 関数は、挿入先を決定するための指示（降下、追加、分割など）を spgChooseOut 構造体に格納して返します。

前提条件と不変条件

  • level は 0 以上の整数であること。
  • nodeLabels は nNodes が 0 より大きい場合に有効な配列を指すこと。
  • hasPrefix が false の場合、prefixDatum の値は無視されます。

アルゴリズムまたは振る舞い

SP-GiSTの挿入プロセスにおいて、インデックスの各階層で呼び出されます。演算子クラスは本構造体の情報を参照し、以下のいずれかの判断を行います。 1. 既存のノードへ降下する。 2. 新しいノードを追加する。 3. 既存のタプルを分割する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c が挿入処理の過程で本構造体を構築します。また、各データ型の演算子クラス実装（spgkdtreeproc.c, spgquadtreeproc.c, spgtextproc.c, geo_spgist.c, network_spgist.c, rangetypes_spgist.c, spgist_name_ops.c）が、PG_GETARG_POINTER(0) を介して本構造体を受け取ります。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • spgChooseOut: spgChooseIn に対応する出力用構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • nNodes が 0 の場合、nodeLabels が NULL であることを想定した実装が必要です。
  • allTheSame が true の場合、ノードの選択ロジックは通常の分岐とは異なる挙動をとる必要があります。

保守担当者向けメモ

  • 本構造体はSP-GiSTのコアAPIの一部であり、変更は演算子クラスのすべての実装に影響を与えます。
  • traversalValue（検索時）のような状態引き継ぎ用ポインタは本構造体には含まれておらず、挿入時の決定は主に現在のタプル情報と挿入対象データに基づいて行われます。
