spgist_name_leaf_consistent

定義

  • 種別: function
  • 定義箇所: src/test/modules/spgist_name_ops/spgist_name_ops.c:398

目的

SP-GiSTインデックスのリーフノードにおいて、格納された値と検索条件（scankeys）を比較し、条件を満たすかどうかを判定する。

シグネチャまたは形状

Datum spgist_name_leaf_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgLeafConsistentIn 構造体を受け取る。これには現在のインデックスレベル、リーフデータ、再構築された値、および検索キーが含まれる。
  • **出力**: PG_RETURN_BOOL を介して、すべての検索条件を満たす場合は true、それ以外は false を返す。また、spgLeafConsistentOut を通じて、再構築された完全な name 値を leafValue として設定する。

前提条件と不変条件

  • in->reconstructedValue が存在する場合、そのサイズは level と一致する必要がある。
  • 復元される fullValue は NAMEDATALEN を超えてはならない。
  • 照合順序（Collation）は考慮せず、バイナリ比較を行う。

アルゴリズムまたは振る舞い

1. in->reconstructedValue と in->leafDatum を結合し、元の name 型の値を復元する。 2. 復元した値を out->leafValue に格納する。 3. in->nkeys 個の検索キーを順次走査する。 4. 各キーについて、memcmp を用いて復元値と検索条件値を比較する。 5. 比較結果（r）に基づき、BTLessStrategyNumber 等の戦略に従って論理判定を行う。 6. いずれかの条件で不一致となった場合、即座に false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSP-GiSTインデックス走査エンジン（直接的な関数呼び出しの記述はソース内にないが、SP-GiSTのインターフェースとして呼び出される）。
  • **呼び出し先**: palloc0, DatumGetTextPP, DatumGetName, NameStr, strlen, memcmp, elog（エラー時）。

関連データ構造

  • spgLeafConsistentIn: 検索入力パラメータ。
  • spgLeafConsistentOut: 検索結果出力パラメータ。
  • name: PostgreSQLの固定長文字列型。

失敗モードと境界条件

  • fullLen >= NAMEDATALEN: アサーション失敗により異常終了する可能性がある。
  • 未知の strategy 番号が渡された場合: elog(ERROR, ...) によりエラーを発生させる。

保守担当者向けメモ

  • 本関数は spgtextproc.c のロジックを name 型向けに移植したものである。
  • reconstructedValue が NULL の場合は level が 0 であることを前提としている。
  • 比較ロジックはバイナリ比較に依存しているため、ロケール依存のソート順が必要な場合には適さない。
