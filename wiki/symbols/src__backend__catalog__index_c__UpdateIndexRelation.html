UpdateIndexRelation

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/index.c:562

目的

システムカタログ pg_index にインデックスの定義情報を挿入する。

シグネチャまたは形状

static void UpdateIndexRelation(
    Oid indexoid, Oid heapoid, Oid parentIndexId,
    const IndexInfo *indexInfo,
    const Oid *collationOids, const Oid *opclassOids,
    const int16 *coloptions,
    bool primary, bool isexclusion, bool immediate,
    bool isvalid, bool isready
)

入力と出力

  • **入力**:
  • indexoid, heapoid: インデックスおよび対象テーブルのOID。
  • indexInfo: インデックスの属性数、ユニーク性、式、述語を含むメタデータ。
  • collationOids, opclassOids, coloptions: インデックスの照合順序、演算子クラス、列オプションの配列。
  • primary, isexclusion, immediate, isvalid, isready: インデックスの特性および状態フラグ。
  • **出力**: なし（副作用として pg_index カタログを更新）。

前提条件と不変条件

  • 呼び出し元は、適切なロック（RowExclusiveLock）を保持している必要がある。
  • indexInfo 内の属性数と、提供された配列（collationOids 等）の長さが整合していること。

アルゴリズムまたは振る舞い

1. indexInfo からインデックスキー、演算子クラス、オプション情報を int2vector および oidvector に変換する。 2. インデックス式（ii_Expressions）および述語（ii_Predicate）が存在する場合、nodeToString を用いてテキスト形式に変換する。 3. pg_index リレーションを RowExclusiveLock で開く。 4. 提供された引数および変換済みデータを用いて pg_index のタプルを構築する。 5. CatalogTupleInsert を呼び出し、カタログへタプルを挿入する。 6. リレーションを閉じ、タプルメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/index.c 内の index_create 等から呼び出される。
  • **呼び出し先**: buildint2vector, buildoidvector, nodeToString, make_ands_explicit, table_open, heap_form_tuple, CatalogTupleInsert, table_close, heap_freetuple 等。

関連データ構造

  • pg_index: システムカタログ。
  • IndexInfo: インデックスの論理構造を保持する構造体。

失敗モードと境界条件

  • メモリ不足による pfree やタプル構築時のエラー。
  • カタログ操作に伴うトランザクション制約やロック競合。

保守担当者向けメモ

  • pg_index のスキーマ変更（列の追加・削除）が発生した際は、values 配列のインデックスと nulls 配列の初期化を適切に更新する必要がある。
  • isvalid や isready などのフラグは、インデックスのライフサイクル管理において重要であるため、呼び出し側の状態遷移ロジックと整合させること。
