lrq_free

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogprefetcher.c:226

目的

LsnReadQueue 構造体に関連付けられたメモリ領域を解放する。

シグネチャまたは形状

static inline void lrq_free(LsnReadQueue *lrq)

入力と出力

  • **入力**: lrq (LsnReadQueue *) - 解放対象のキュー構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • lrq は palloc 等で割り当てられた有効なメモリ領域を指している必要がある。
  • 本関数呼び出し後、lrq が指していたメモリ領域へのアクセスは無効となる。

アルゴリズムまたは振る舞い

1. pfree(lrq) を呼び出し、指定された LsnReadQueue 構造体のメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogprefetcher.c 内の以下の箇所から呼び出されていることが確認されている。
  • prefetcher->streaming_read の解放時（行392および行996）。
  • **呼び出し先**:
  • pfree (PostgreSQL のメモリ管理関数)。

関連データ構造

  • LsnReadQueue: 先読み中のI/Oリクエストを管理する循環キュー構造体。

失敗モードと境界条件

  • lrq が NULL の場合、pfree の仕様に依存する（通常、pfree は NULL ポインタを許容する）。

保守担当者向けメモ

  • 本関数は static inline で定義されており、主に XLogPrefetcher 構造体の破棄や再構成プロセスの一部として利用される。
  • メモリリークを防ぐため、LsnReadQueue を保持する親構造体（XLogPrefetcher）のライフサイクル管理と整合させること。
