mbms_add_members

定義

  • src/backend/nodes/multibitmapset.c:70

目的

2つのマルチビットマップセット（List of Bitmapset）の和集合を計算し、左辺のリストを更新して返します。

シグネチャまたは形状

List *mbms_add_members(List *a, const List *b)

入力と出力

  • **入力**:
  • a: 更新対象となるマルチビットマップセット（List）。
  • b: 追加する要素を持つマルチビットマップセット（const List）。
  • **出力**:
  • 和集合の結果を保持する List ポインタ。a がインプレース更新されます。

前提条件と不変条件

  • a および b は Bitmapset を要素として持つリストである必要があります。
  • リスト内の各要素は NULL である可能性があります。
  • 負のインデックスは許可されません。

アルゴリズムまたは振る舞い

1. a の長さが b より短い場合、lappend(a, NULL) を用いて a を b と同じ長さになるまで動的に拡張します。 2. forboth マクロを使用して両方のリストを同時に走査します。 3. 各ペアの Bitmapset に対して bms_add_members を適用し、その結果を a の該当セルに格納します。 4. 更新されたリスト a を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepjointree.c
  • src/backend/optimizer/util/clauses.c
  • **呼び出し先**:
  • lappend
  • list_length
  • lfirst_node
  • bms_add_members
  • lfirst

関連データ構造

  • List: マルチビットマップセットのコンテナ。
  • Bitmapset: 各リスト要素が保持するビット集合。

失敗モードと境界条件

  • a が b より短い場合、自動的に拡張されます。
  • リスト内に NULL 要素が含まれる場合、lfirst_node の使用により適切に処理されます。

保守担当者向けメモ

  • 本関数はインプレース更新を行うため、呼び出し側は戻り値でリストポインタを更新する必要があります。
  • forboth は短い方のリストの長さに合わせて終了するため、拡張処理が必須となっています。
