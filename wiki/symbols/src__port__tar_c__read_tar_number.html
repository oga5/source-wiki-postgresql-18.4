read_tar_number

定義

  • 種別: function
  • 定義箇所: src/port/tar.c:57

目的

tar ヘッダー内の固定長フィールドから数値をデコードします。標準的な 8 進数表現と、GNU 拡張である base-256 形式の両方をサポートします。

シグネチャまたは形状

uint64 read_tar_number(const char *s, int len)

入力と出力

  • **入力**:
  • s: 数値データが格納されている文字列（バッファ）へのポインタ。
  • len: 読み取るフィールドの長さ（バイト数）。
  • **出力**:
  • uint64: デコードされた数値。

前提条件と不変条件

  • 入力文字列 s は、tar ヘッダー内の指定されたフィールドを指している必要があります。
  • base-256 形式の場合、先頭バイトが \200 である必要があります。

アルゴリズムまたは振る舞い

1. 先頭バイト *s が \200 であるかを確認します。 2. **base-256 形式の場合**: 先頭バイトを除いた残りの len - 1 バイトを処理し、各バイトを 8 ビットずつ左シフトして result に加算します。 3. **8 進数形式の場合**: len バイト分を走査し、文字が '0' から '7' の範囲内である限り、3 ビットずつ左シフトして数値を構築します。 4. 最終的な result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_tar.c: チェックサムおよびサイズフィールドの読み取りに使用。
  • src/fe_utils/astreamer_tar.c: サイズ、モード、UID、GID フィールドの読み取りに使用。
  • src/port/tar.c: isValidTarHeader 内でチェックサムの検証に使用。
  • **呼び出し先**:
  • 本関数は標準ライブラリ関数を呼び出さず、ポインタ演算とビット演算のみで構成されています。

関連データ構造

  • pgtar.h: tar ヘッダーのオフセット定義（TAR_OFFSET_CHECKSUM, TAR_OFFSET_SIZE 等）が本関数と組み合わせて使用されます。

失敗モードと境界条件

  • 8 進数形式において、範囲外の文字（'0'-'7' 以外）に遭遇した場合、その時点で読み取りを停止します。
  • 入力バッファが適切に終端されていない場合や、len がフィールドサイズと一致しない場合、誤った値が返される可能性があります。

保守担当者向けメモ

  • 本関数は GNU 拡張の base-256 形式をサポートしており、標準的な POSIX tar 実装との互換性を維持しています。
  • チェックサムフィールドの読み取り時には、ヘッダー内の特定のオフセットと長さを正しく指定する必要があります。
