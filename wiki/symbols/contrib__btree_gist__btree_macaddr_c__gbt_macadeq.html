gbt_macadeq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr.c:41

目的

macaddr 型の2つの値が等しいかどうかを判定し、GiSTインデックスの比較処理をサポートする。

シグネチャまたは形状

static bool gbt_macadeq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の macaddr 値へのポインタ。
  • flinfo: 関数呼び出し情報（PostgreSQLの関数マネージャ用）。
  • **出力**:
  • bool: a と b が等しい場合に true、それ以外の場合に false を返す。

前提条件と不変条件

  • 入力ポインタ a および b は有効な macaddr 型のデータである必要がある。
  • PostgreSQLの macaddr_eq 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. a と b を PointerGetDatum マクロで Datum 型に変換する。 2. DirectFunctionCall2 を介して、PostgreSQL標準の macaddr_eq 関数を呼び出す。 3. 戻り値を DatumGetBool で bool 型に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_macaddr.c 内の静的構造体（tinfo）から参照されている。
  • **呼び出し先**: PostgreSQL内部関数 macaddr_eq。

関連データ構造

  • macaddr: 比較対象となるPostgreSQLのネットワークMACアドレス型。

失敗モードと境界条件

  • a または b が NULL の場合、PostgreSQLの関数呼び出し規約に従い、予期せぬ動作や例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数は btree_utils_num が期待する比較インターフェースを macaddr 型に適合させるためのラッパーである。
  • macaddr 型の比較ロジックに変更があった場合、本関数を直接修正するのではなく、基底となる macaddr_eq の挙動を確認すること。
