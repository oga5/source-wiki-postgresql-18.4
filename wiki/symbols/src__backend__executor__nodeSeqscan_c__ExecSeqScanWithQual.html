ExecSeqScanWithQual

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:129

目的

フィルタ条件（Qual）が存在し、かつプロジェクション（射影）を必要としないシーケンシャルスキャンを実行する。

シグネチャまたは形状

static TupleTableSlot *ExecSeqScanWithQual(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（シーケンシャルスキャンノードの状態）
  • **出力**: フィルタ条件を通過した TupleTableSlot、またはスキャン終了時は NULL。

前提条件と不変条件

  • pstate->state->es_epq_active が NULL であること（EvalPlanQual実行中ではないこと）。
  • pstate->qual が NULL ではないこと。
  • pstate->ps_ProjInfo が NULL であること。

アルゴリズムまたは振る舞い

1. pstate を SeqScanState にキャストする。 2. ExecScanExtended を呼び出し、以下の引数を渡してスキャンを実行する。

  • アクセスメソッドとして SeqNext を使用。
  • 再チェックメソッドとして SeqRecheck を使用。
  • フィルタ条件として pstate->qual を適用。
  • プロジェクションは行わない（NULL を指定）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSeqscan.c 内の ExecInitSeqScan において、実行ノードの ExecProcNode 関数ポインタとして設定される。
  • **呼び出し先**: ExecScanExtended、SeqNext、SeqRecheck。

関連データ構造

  • SeqScanState: シーケンシャルスキャンの実行状態を保持する構造体。
  • PlanState: 実行計画ノードの共通状態。
  • TupleTableSlot: タプルを保持する実行時のデータコンテナ。

失敗モードと境界条件

  • テーブルの末尾に到達した場合は NULL を返す。
  • フィルタ条件に一致するタプルがない場合、スキャンが終了するまで ExecScanExtended 内でタプル取得と評価が繰り返される。

保守担当者向けメモ

  • 本関数は、クエリ実行時の最適化パスの一つである。条件評価のみが必要な場合に呼び出されるため、プロジェクションが必要な場合は ExecSeqScanWithQualProject 等の別関数が使用される。
