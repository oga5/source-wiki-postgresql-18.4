sql_exec_dumpalltables

定義

  • 種別: function
  • 定義箇所: contrib/oid2name/oid2name.c:468

目的

データベース内のテーブル、インデックス、シーケンス等のオブジェクト情報をシステムカタログから取得し、ファイルノードや名前などの詳細情報を標準出力に表示する。

シグネチャまたは形状

void sql_exec_dumpalltables(PGconn *conn, struct options *opts)

入力と出力

  • **入力**:
  • conn: データベース接続を保持する PGconn ポインタ。
  • opts: 検索条件や出力オプションを保持する struct options 構造体。
  • **出力**: なし（sql_exec を通じて標準出力へ結果を表示）。

前提条件と不変条件

  • conn は有効なデータベース接続である必要がある。
  • opts 構造体内のフラグ（extended, indexes, systables）に基づき、動的に SQL クエリが生成される。

アルゴリズムまたは振る舞い

1. opts の設定に基づき、pg_class, pg_namespace, pg_database, pg_tablespace を結合する SQL クエリ文字列を構築する。 2. opts->extended が真の場合、OID、スキーマ名、テーブルスペース名を追加で取得する。 3. opts->indexes や opts->systables のフラグに応じて、WHERE 句の relkind 条件およびシステムカタログの除外条件を調整する。 4. 構築したクエリを sql_exec 関数に渡し、実行および結果の整形出力を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/oid2name/oid2name.c 内の main 関数（637行目付近）。
  • **呼び出し先**: sql_exec（クエリ実行用）。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • struct options: 実行時の検索条件や表示オプションを保持する構造体。

失敗モードと境界条件

  • データベース接続が切断されている場合、sql_exec 内部でエラーが発生する可能性がある。
  • システムカタログへのアクセス権限がない場合、クエリ実行が失敗する。
  • todo バッファ（1024バイト）を超える長さのクエリが生成される可能性については、現在の実装では考慮されていない。

保守担当者向けメモ

  • クエリの動的生成において CppAsString2 マクロを使用しているため、relkind 定数の変更時には注意が必要。
  • opts->systables が偽の場合、pg_catalog や information_schema などのシステムスキーマが自動的に除外される仕様となっている。
