generate_nonunion_paths

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepunion.c:997

目的

INTERSECT および EXCEPT 集合演算ノードに対する実行パスを生成する。

シグネチャまたは形状

static RelOptInfo *generate_nonunion_paths(SetOperationStmt *op, PlannerInfo *root, List *refnames_tlist, List **pTargetList)

入力と出力

  • **入力**:
  • op: 処理対象の SetOperationStmt。
  • root: 現在のプランナ情報。
  • refnames_tlist: 参照名を含むターゲットリスト。
  • **出力**:
  • 戻り値: 集合演算の結果を表す RelOptInfo。
  • pTargetList: 生成されたターゲットリストが格納される。

前提条件と不変条件

  • root->tuple_fraction は一時的に 0.0 に設定され、処理後に復元される。
  • ターゲットリスト生成において型変換が不要であること（result_trivial_tlist が真であること）を Assert で検証する。

アルゴリズムまたは振る舞い

1. recurse_set_operations を再帰的に呼び出し、左右の子ノードのパスと関係情報を取得する。 2. generate_setop_tlist で出力ターゲットリストを生成する。 3. generate_setop_grouplist でグループ化のセマンティクスを特定し、ソートまたはハッシュが可能か判定する。不可の場合はエラーを投げる。 4. build_setop_child_paths を呼び出し、子ノードに対して必要なパス（ソート済みパスなど）を構築する。 5. INTERSECT の場合、ハッシュテーブルのサイズ最小化と効率化のため、グループ数の少ない方を左側に配置するよう入力を入れ替える。 6. UPPERREL_SETOP として結果リレーションを作成し、行数を見積もる。 7. 最後に create_setop_path を呼び出し、ハッシュまたはソートに基づくパスを作成する（ソース抜粋はここで終了）。

呼び出し元と呼び出し先

  • **呼び出し元**: recurse_set_operations (src/backend/optimizer/prep/prepunion.c:279)
  • **呼び出し先**: recurse_set_operations, generate_setop_tlist, generate_setop_grouplist, grouping_is_sortable, grouping_is_hashable, make_pathkeys_for_sortclauses, build_setop_child_paths, fetch_upper_rel, create_pathtarget, create_setop_path

関連データ構造

  • SetOperationStmt: 集合演算の定義。
  • RelOptInfo: プランニング中のリレーション情報。
  • Path: 実行パス。
  • SetOpCmd: 実行する集合演算の種類。

失敗モードと境界条件

  • ソートもハッシュもサポートされていないデータ型が含まれる場合、ERROR を発生させる。
  • 認識できない op->op が渡された場合、elog によりエラーとなる。

保守担当者向けメモ

  • 本関数は INTERSECT / EXCEPT 特有の最適化（入力の入れ替え等）を行う。
  • tuple_fraction の強制変更は副作用を伴うため、復元処理が正しく行われているか注意すること。
