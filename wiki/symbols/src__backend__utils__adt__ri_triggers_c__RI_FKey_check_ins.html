RI_FKey_check_ins

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ri_triggers.c:473

目的

テーブルへの挿入操作（INSERT）時に、外部キー制約の整合性を検証するためのトリガープロシージャです。

シグネチャまたは形状

Datum RI_FKey_check_ins(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（トリガー実行コンテキストを含む）。
  • **出力**: Datum 型。制約違反がない場合は PointerGetDatum(NULL) を返し、違反がある場合はエラーを発生させます。

前提条件と不変条件

  • 本関数は、RI_TRIGTYPE_INSERT イベントとしてトリガーマネージャから呼び出される必要があります。
  • 内部で ri_CheckTrigger を呼び出し、呼び出しタイミングとイベントが適切であるかを検証します。

アルゴリズムまたは振る舞い

1. ri_CheckTrigger を呼び出し、トリガーが正しいタイミング（INSERT）で実行されているかを確認します。 2. RI_FKey_check を呼び出し、実際の外部キー制約チェック処理を委譲します。この際、fcinfo->context を TriggerData にキャストして渡します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c: ALTER TABLE 等の制約追加処理において、既存データの検証のために呼び出されます。
  • **呼び出し先**:
  • ri_CheckTrigger: トリガーの実行環境検証。
  • RI_FKey_check: 外部キー制約チェックのメインロジック。

関連データ構造

  • TriggerData: トリガー実行時のコンテキスト情報。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • ri_CheckTrigger による検証で、トリガーの呼び出しタイミングやイベントが不適切と判断された場合、エラーが発生します。
  • RI_FKey_check 内部で外部キー制約違反が検出された場合、エラーが発生しトランザクションが中断されます。

保守担当者向けメモ

  • 本関数は RI_FKey_check のラッパーであり、UPDATE用トリガーである RI_FKey_check_upd と共通のロジックを共有しています。
  • 修正を行う際は、RI_FKey_check 側の影響範囲を考慮してください。
