LaunchParallelWorkers

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/transam/parallel.c:579

目的

リーダープロセスが指定された数のバックグラウンドワーカーを登録し、並列クエリ実行を開始させる。

シグネチャまたは形状

void LaunchParallelWorkers(ParallelContext *pcxt)

入力と出力

  • **入力**: ParallelContext *pcxt（並列実行コンテキスト。ワーカー数や共有メモリセグメント情報を含む）。
  • **出力**: なし（副作用として pcxt 内のワーカーハンドルや状態が更新される）。

前提条件と不変条件

  • pcxt->nworkers および pcxt->nworkers_to_launch が 0 より大きいこと。
  • pcxt->seg が有効な共有メモリセグメントを指していること。
  • 呼び出し元はロックグループリーダーである必要がある（BecomeLockGroupLeader() が内部で実行される）。

アルゴリズムまたは振る舞い

1. **初期化**: TopTransactionContext にメモリコンテキストを切り替え、ワーカー設定用の BackgroundWorker 構造体を初期化する。 2. **ワーカー設定**: エントリポイントを ParallelWorkerMain に固定し、共有メモリセグメントのハンドルを引数として設定する。 3. **登録ループ**: pcxt->nworkers_to_launch 回数分、RegisterDynamicBackgroundWorker を呼び出す。

  • 登録成功時: shm_mq（エラー用メッセージキュー）をワーカーハンドルに関連付け、nworkers_launched をインクリメントする。
  • 登録失敗時: max_worker_processes 制限等により登録が失敗した場合、以降のループでは登録を試行せず、該当するエラーキューを解放してリソースを整理する。

    • 4. **状態更新**: 起動したワーカー数に基づき、known_attached_workers 配列を割り当てる。

関連データ構造

  • ParallelContext: 並列実行の全体状態を管理する構造体。
  • BackgroundWorker: バックグラウンドワーカーの起動パラメータを保持する構造体。

失敗モードと境界条件

  • **登録失敗**: RegisterDynamicBackgroundWorker が失敗した場合、エラーを投げずに処理を継続する。これはシステム全体のワーカー上限に達した場合を許容するためであり、起動できなかったワーカー分のエラーキューは適切に解放される。
  • **ワーカー数 0**: nworkers または nworkers_to_launch が 0 の場合、何もせずに即時復帰する。

保守担当者向けメモ

  • 本関数はワーカーの「登録」を行うものであり、ワーカーが実際に起動・初期化されることを保証するものではない。
  • any_registrations_failed フラグは、一度登録に失敗した後に残りのスロットを適切にクリーンアップするために使用される。このロジックを変更する際は、エラーキューのデタッチ処理が漏れないよう注意すること。
