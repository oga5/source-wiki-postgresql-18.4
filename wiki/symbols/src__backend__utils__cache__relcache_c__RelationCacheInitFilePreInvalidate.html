RelationCacheInitFilePreInvalidate

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relcache.c:6859

目的

リレーションキャッシュの初期化ファイル（pg_internal.init）を削除し、古いキャッシュ状態を無効化する。

シグネチャまたは形状

void RelationCacheInitFilePreInvalidate(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてファイルシステム上の初期化ファイルを削除）

前提条件と不変条件

  • RelCacheInitLock を排他モードで取得する必要がある。
  • DatabasePath が設定されている場合、ローカルの初期化ファイルパスが構築される。

アルゴリズムまたは振る舞い

1. ローカル（base/<db>/pg_internal.init）および共有（global/pg_internal.init）の初期化ファイルパスを構築する。 2. RelCacheInitLock を排他モードで取得する。 3. unlink_initfile を呼び出し、各初期化ファイルを削除する。

  • ファイルが存在しない場合（ENOENT）は無視される。
  • それ以外のエラーが発生した場合は ERROR を発生させる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/twophase.c
  • src/backend/utils/cache/inval.c
  • 呼び出し先:
  • LWLockAcquire
  • unlink_initfile

関連データ構造

  • RELCACHE_INIT_FILENAME: 初期化ファイル名（pg_internal.init）
  • RelCacheInitLock: キャッシュ初期化ファイルへのアクセスを制御するロック

失敗モードと境界条件

  • ファイル削除時に ENOENT 以外のエラーが発生した場合、ERROR が発生しトランザクションが中断される可能性がある。
  • DatabasePath が未設定の場合、ローカル側の初期化ファイル削除はスキップされる。

保守担当者向けメモ

  • 本関数は RelCacheInitLock を保持したまま削除処理を行うため、デッドロックの可能性に注意すること。
  • inval.c からの呼び出しは、キャッシュ無効化メッセージに関連して発生する。
