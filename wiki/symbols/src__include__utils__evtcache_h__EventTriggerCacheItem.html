EventTriggerCacheItem

定義

  • 種別: struct
  • 定義箇所: src/include/utils/evtcache.h:29

目的

イベントトリガーの実行に必要な情報をメモリ上に保持し、SQL実行時のシステムカタログ検索オーバーヘッドを削減するためのキャッシュエントリ構造体です。

シグネチャまたは形状

typedef struct
{
	Oid			fnoid;			/* 呼び出される関数 */
	char		enabled;		/* SESSION_REPLICATION_ROLE_* に準拠 */
	Bitmapset  *tagset;			/* コマンドタグの集合、空の場合は NULL */
} EventTriggerCacheItem;

入力と出力

  • 本構造体は EventCacheLookup を通じて取得されるリストの要素として出力されます。
  • 入力として、システムカタログ pg_event_trigger から読み込まれた値が各フィールドに格納されます。

前提条件と不変条件

  • tagset が NULL の場合、そのトリガーは特定のコマンドタグに限定されず、すべての対象イベントで考慮されることを示唆します。
  • キャッシュ内のメモリを指しているため、呼び出し元で長期間保持する場合は、適切なメモリコンテキストへのコピーが必要です。

アルゴリズムまたは振る舞い

  • EventCacheLookup 呼び出し時に、キャッシュが未構築であれば BuildEventTriggerCache が実行され、本構造体のインスタンスが生成・格納されます。
  • filter_event_trigger 関数において、実行対象の CommandTag が tagset に含まれるかどうかの判定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/backend/commands/event_trigger.c: filter_event_trigger 内でトリガーのフィルタリングに使用。
  • src/backend/commands/event_trigger.c: リスト走査時に lfirst を介して参照。
  • **呼び出し先（生成）**:
  • src/backend/utils/cache/evtcache.c: palloc0 を使用してメモリを確保し、キャッシュ構築時にインスタンス化。

関連データ構造

  • Bitmapset: tagset フィールドで使用。
  • List: EventCacheLookup が返すリスト構造の保持に使用。

失敗モードと境界条件

  • キャッシュの無効化はシステムカタログ更新時に自動で行われますが、再構築は遅延評価されるため、無効化直後のアクセスでは再構築処理が走ります。
  • tagset が NULL の場合の挙動（全イベント対象）を考慮したロジックが必要です。

保守担当者向けメモ

  • 本構造体はキャッシュの一部であるため、直接的な変更は evtcache.c のキャッシュ管理ロジックに影響を与えます。
  • フィールドを追加する場合は、キャッシュ構築処理および無効化コールバックの整合性を確認してください。
