LWLockHandle

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/lmgr/lwlock.c:211

目的

現在プロセスが保持している軽量ロック（LWLock）の状態を追跡するためのハンドル構造体です。

シグネチャまたは形状

typedef struct LWLockHandle
{
	LWLock	   *lock;
	LWLockMode	mode;
} LWLockHandle;

入力と出力

  • lock: 保持対象の LWLock 構造体へのポインタ。
  • mode: 取得したロックのモード（LWLockMode 型）。

前提条件と不変条件

  • 本構造体は、プロセスが現在保持しているロックを管理するために使用されます。
  • 保持しているロックの数には MAX_SIMUL_LWLOCKS による上限が存在します。

アルゴリズムまたは振る舞い

  • プロセスがロックを取得した際、そのロックへの参照と取得モードを本構造体に記録することで、エラー発生時やトランザクション終了時に LWLockReleaseAll を通じて安全にロックを解放できるようにします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/lwlock.c 内の held_lwlocks 配列（static LWLockHandle held_lwlocks[MAX_SIMUL_LWLOCKS];）によって参照されています。
  • 呼び出し先: 提供されたソース情報からは、特定の関数による直接的な呼び出し関係は特定できません。

関連データ構造

  • LWLock: ロックの本体。
  • LWLockMode: ロックのモード（共有または排他）。

失敗モードと境界条件

  • 保持するロック数が MAX_SIMUL_LWLOCKS を超える場合、追跡が不可能となります。

保守担当者向けメモ

  • 本構造体は lwlock.c 内の静的配列 held_lwlocks で使用されており、プロセスごとのロック管理の基盤となっています。変更時は LWLockReleaseAll 等のクリーンアップ処理への影響を確認してください。
