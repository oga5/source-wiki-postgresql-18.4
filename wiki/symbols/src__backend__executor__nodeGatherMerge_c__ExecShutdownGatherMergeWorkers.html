ExecShutdownGatherMergeWorkers

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:322

目的

GatherMerge ノードに関連する並列ワーカーの実行を終了させ、関連するリーダー側のリソースを解放する。

シグネチャまたは形状

static void ExecShutdownGatherMergeWorkers(GatherMergeState *node)

入力と出力

  • **入力**: GatherMergeState *node - 終了対象の GatherMerge ノードの状態構造体。
  • **出力**: なし。

前提条件と不変条件

  • node が指す GatherMergeState は初期化済みであること。
  • node->pei が有効な場合、並列実行状態が保持されている。

アルゴリズムまたは振る舞い

1. node->pei が NULL でない場合、ExecParallelFinish(node->pei) を呼び出し、並列ワーカーの終了処理を行う。 2. node->reader 配列が存在する場合、pfree を用いてメモリを解放する。 3. node->reader ポインタを NULL に設定し、二重解放を防止する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeGatherMerge.c 内の ExecShutdownGatherMerge および ExecReScanGatherMerge から呼び出されることが確認されている。
  • **呼び出し先**:
  • ExecParallelFinish (並列実行終了処理)
  • pfree (メモリ解放)

関連データ構造

  • GatherMergeState: 並列クエリ実行ノードの状態を保持する構造体。
  • ParallelExecutorInfo (pei): 並列実行の制御情報。

失敗モードと境界条件

  • node->pei が NULL の場合、並列ワーカーの終了処理はスキップされる。
  • node->reader が NULL の場合、メモリ解放処理はスキップされる。

保守担当者向けメモ

  • 本関数は ExecShutdownGatherMerge の一部として、リソースのクリーンアップを担う。ワーカーの終了とリーダー側の管理用配列の解放を確実に行うため、ノードの終了処理フローにおいて適切に呼び出されていることを確認すること。
