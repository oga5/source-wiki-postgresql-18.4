xact_desc_assignment

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xactdesc.c:427

目的

トランザクションのサブトランザクション割り当て（subtransaction assignment）に関するWALレコードの内容を人間が読める形式に変換し、バッファへ出力する。

シグネチャまたは形状

static void xact_desc_assignment(StringInfo buf, xl_xact_assignment *xlrec)

入力と出力

  • buf (StringInfo): 解析結果を追記する出力バッファ。
  • xlrec (xl_xact_assignment*): 解析対象のサブトランザクション割り当て情報を含む構造体。
  • 戻り値: なし（buf を直接更新する）。

前提条件と不変条件

  • xlrec は有効な xl_xact_assignment 構造体を指している必要がある。
  • xlrec->nsubxacts は xlrec->xsub 配列の要素数と一致している必要がある。

アルゴリズムまたは振る舞い

1. buf に文字列 "subxacts:" を追加する。 2. xlrec->nsubxacts 回ループし、各サブトランザクションID (xlrec->xsub[i]) を " %u" の形式で buf に順次追記する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/rmgrdesc/xactdesc.c 内のメインディスパッチャ（xact_desc）から呼び出される。
  • 呼び出し先: appendStringInfoString および appendStringInfo（lib/stringinfo.h）。

関連データ構造

  • xl_xact_assignment: サブトランザクション割り当て情報を保持する構造体。
  • StringInfo: PostgreSQLの動的文字列バッファ。

失敗モードと境界条件

  • xlrec->nsubxacts が 0 の場合、"subxacts:" のみがバッファに追加される。
  • buf の容量が不足した場合、appendStringInfo の内部仕様に従いメモリ拡張が行われる。

保守担当者向けメモ

  • 本関数は pg_waldump 等のツールでWALをデコードする際に使用される。
  • サブトランザクションIDのリストを表示するシンプルなロジックであるため、構造体 xl_xact_assignment の定義が変更された場合は、ループ処理および出力フォーマットの整合性を確認すること。
