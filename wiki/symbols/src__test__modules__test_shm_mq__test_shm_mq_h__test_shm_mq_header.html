test_shm_mq_header

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_shm_mq/test_shm_mq.h:29

目的

PostgreSQLの共有メモリメッセージキュー（shm_mq）のテストにおいて、複数のバックグラウンドワーカー間で同期状態を管理するための制御構造体です。

シグネチャまたは形状

typedef struct
{
	slock_t		mutex;            /* 排他制御用スピンロック */
	int			workers_total;    /* 起動するワーカーの総数 */
	int			workers_attached; /* アタッチ済みのワーカー数 */
	int			workers_ready;    /* 準備完了状態のワーカー数 */
} test_shm_mq_header;

入力と出力

  • **入力**: 共有メモリ領域へのポインタとして配置され、各ワーカーおよびメインプロセスによって更新されます。
  • **出力**: 構造体内の各カウンタ値が、ワーカーのライフサイクル（アタッチ、準備完了）の進捗状況を反映します。

前提条件と不変条件

  • mutex フィールドは、構造体内のカウンタを更新する際の競合を防ぐために使用されます。
  • 共有メモリ上に配置されるため、プロセス間で一貫したメモリレイアウトが保証される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: test_shm_mq_setup 関数により、共有メモリセグメント内に確保されます。 2. **アタッチ通知**: 各ワーカーは起動後、mutex を取得して workers_attached をインクリメントします。 3. **準備完了通知**: ワーカーは処理の準備が整うと、mutex を取得して workers_ready をインクリメントします。 4. **監視**: メインプロセスは、この構造体のカウンタを監視することで、全ワーカーの準備完了を待機します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_shm_mq/setup.c および src/test/modules/test_shm_mq/worker.c 内の関数がこの構造体を参照・更新します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • slock_t: storage/spin.h で定義されるスピンロック。
  • shm_toc: 共有メモリ内のテーブル・オブ・コンテンツ（setup.c で本構造体の割り当てに使用）。

失敗モードと境界条件

  • mutex の取得に失敗した場合や、デッドロックが発生した場合、ワーカーの進捗が停止し、メインプロセスが待機状態から復帰できなくなる可能性があります。
  • 共有メモリの割り当てサイズが sizeof(test_shm_mq_header) 未満の場合、メモリ破壊が発生します。

保守担当者向けメモ

  • 本構造体は volatile 修飾子を伴って参照されることが多いため、コンパイラによる最適化でメモリ読み書きが省略されないよう注意してください。
  • カウンタの更新は必ず mutex による保護下で行う必要があります。
