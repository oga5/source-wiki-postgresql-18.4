ConditionVariableMinimallyPadded

定義

  • 種別: typedef (共用体)
  • 定義箇所: src/include/storage/condition_variable.h:39

目的

共有メモリ上での配置において、キャッシュラインの競合（False Sharing）を回避しつつ、ConditionVariable を配列として安全に管理するためにサイズを調整したラッパー構造を提供します。

シグネチャまたは形状

typedef union ConditionVariableMinimallyPadded
{
	ConditionVariable cv;
	char		pad[CV_MINIMAL_SIZE];
} ConditionVariableMinimallyPadded;

入力と出力

  • 入力: なし（データ構造の定義）
  • 出力: なし

前提条件と不変条件

  • CV_MINIMAL_SIZE で定義されたサイズが、内部の ConditionVariable 構造体を包含し、かつキャッシュライン境界を考慮したパディングとして機能している必要があります。

アルゴリズムまたは振る舞い

  • 本シンボルは、ConditionVariable を直接使用する代わりに、パディングを伴う共用体としてラップします。これにより、メモリ上に連続して配置された際に、各条件変数がキャッシュラインをまたがない、あるいは適切に分離されるように配置を制御します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/storage/buffer/buf_init.c: BufferIOCVArray の型定義およびメモリ確保に使用。
  • src/include/storage/buf_internals.h: BufferIOCVArray の外部参照宣言。
  • 呼び出し先:
  • ConditionVariable 構造体（メンバとして保持）。

関連データ構造

  • ConditionVariable: 本共用体がラップするメインの同期プリミティブ。
  • BufferIOCVArray: 本型を使用して定義される、バッファI/O待機用の条件変数配列。

失敗モードと境界条件

  • CV_MINIMAL_SIZE が sizeof(ConditionVariable) より小さい場合、コンパイルエラーまたはメモリ破壊の危険性があります。

保守担当者向けメモ

  • 本構造体は、主に BufferIOCVArray のような大規模な配列構造でキャッシュ効率を最適化するために使用されます。変更を加える際は、CV_MINIMAL_SIZE の定義とターゲットアーキテクチャのキャッシュラインサイズとの整合性を確認してください。
