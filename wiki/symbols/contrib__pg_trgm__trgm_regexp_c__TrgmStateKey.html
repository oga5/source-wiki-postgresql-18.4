TrgmStateKey

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_regexp.c:310

目的

正規表現から変換されたグラフ構造において、状態（State）を一意に識別するためのハッシュテーブルキーとして機能します。

シグネチャまたは形状

typedef struct
{
	TrgmPrefix	prefix;
	int			nstate;
} TrgmStateKey;

入力と出力

  • **入力**: TrgmPrefix（プレフィックス情報）および int（状態番号）。
  • **出力**: なし（構造体として保持される）。

前提条件と不変条件

  • TrgmState 構造体において、本構造体はハッシュテーブルのキーとして使用されるため、常に構造体の先頭フィールドとして配置される必要があります。

アルゴリズムまたは振る舞い

  • 正規表現の NFA をトリグラムベースのグラフへ変換する際、状態の重複を排除するためにハッシュテーブルのキーとして利用されます。
  • getState 関数等を通じて、特定のプレフィックスと状態番号の組み合わせに対応する TrgmState を検索・管理するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: addKey, addKeyToQueue, addArc, validArcLabel, getState などの関数が本構造体を引数として受け取ります。また、hashCtl.keysize の設定時に sizeof(TrgmStateKey) が参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • TrgmState: 本キーを使用して管理される状態本体。
  • TrgmPrefix: キーの一部を構成するプレフィックス情報。
  • TrgmNFA: 本構造体を利用してグラフを構築する作業用構造体。

失敗モードと境界条件

  • ハッシュテーブルのキーとして使用されるため、メモリ不足やハッシュ衝突時の挙動は utils/hsearch.h の実装に依存します。

保守担当者向けメモ

  • 本構造体を変更する場合は、TrgmState 構造体内の配置順序（先頭であること）を維持してください。
  • TrgmState のハッシュテーブル管理ロジックと密結合しているため、キーの構成要素を変更する際は関連する addKey 系関数および getState 関数の整合性を確認してください。
