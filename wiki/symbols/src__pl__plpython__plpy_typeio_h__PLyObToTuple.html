PLyObToTuple

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_typeio.h:105

目的

PythonオブジェクトからPostgreSQLの複合型（タプル）へ変換する際のメタデータおよび変換状態を保持する。

シグネチャまたは形状

typedef struct PLyObToTuple
{
    TupleDesc       recdesc;
    TypeCacheEntry *typentry;
    uint64          tupdescid;
    PLyObToDatum   *atts;
    int             natts;
    FmgrInfo        recinfunc;
} PLyObToTuple;

入力と出力

  • **入力**: 変換対象のPythonオブジェクト（変換実行時に使用）。
  • **出力**: PostgreSQLのタプル表現（Datum）。

前提条件と不変条件

  • atts および natts は初期化前には NULL または 0 である。
  • typentry および tupdescid は名前付き複合型の場合に設定される。
  • recdesc は RECORD 型を扱う場合に設定される。

アルゴリズムまたは振る舞い

1. 変換対象が名前付き複合型か RECORD 型かに応じて、型記述子（typentry または recdesc）を参照する。 2. 各カラムの変換情報を atts 配列に保持し、再帰的に変換を行う。 3. 必要に応じて record_in 関数（recinfunc）を使用して変換を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の関数呼び出しは特定できない。
  • **呼び出し先**: PLyObToDatum 構造体（各カラムの変換用）および PostgreSQL の record_in 関数。

関連データ構造

  • PLyObToDatum: 各カラムの変換情報を保持する構造体。
  • TupleDesc: タプルの構造定義。
  • TypeCacheEntry: 型情報のキャッシュ。
  • FmgrInfo: 関数マネージャ情報。

失敗モードと境界条件

  • atts が未設定の場合、変換処理は実行できない。
  • tupdescid が typcache 内の最新の識別子と一致しない場合、再初期化が必要となる可能性がある。

保守担当者向けメモ

  • 本構造体は plpy_typeio.c 内部での使用を前提としている。
  • 変換関数は再帰的に呼び出されるため、メモリコンテキストの管理に注意すること。
