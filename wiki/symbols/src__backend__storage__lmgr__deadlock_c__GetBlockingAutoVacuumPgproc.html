GetBlockingAutoVacuumPgproc

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/deadlock.c:289

目的

現在保持されているブロッキング・オートバキュームプロセスの情報を取得し、その保持状態をリセットします。

シグネチャまたは形状

PGPROC *GetBlockingAutoVacuumPgproc(void)

入力と出力

  • **入力**: なし
  • **出力**:
  • 戻り値: ブロッキング・オートバキュームプロセスを表す PGPROC 構造体へのポインタ。保持されていない場合は NULL を返します。

前提条件と不変条件

  • 本関数は、グローバル変数 blocking_autovacuum_proc の値を読み取った直後に、当該変数を NULL にクリアします。
  • 呼び出し後、以前の保持値はクリアされるため、本関数を複数回連続で呼び出した場合、2回目以降は NULL が返されます。

アルゴリズムまたは振る舞い

1. 静的変数 blocking_autovacuum_proc の値をローカル変数 ptr に退避します。 2. blocking_autovacuum_proc を NULL に設定します。 3. 退避した ptr を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/proc.c:1478 にて呼び出されています。
  • **呼び出し先**: 本関数内から他の関数を呼び出すことはありません。

関連データ構造

  • PGPROC: PostgreSQLのバックエンドプロセスを管理する構造体。storage/proc.h で定義されています。

失敗モードと境界条件

  • blocking_autovacuum_proc が設定されていない場合、戻り値として NULL を返します。これは正常な動作として扱われます。

保守担当者向けメモ

  • 本関数は blocking_autovacuum_proc の値を消費（取得してクリア）する役割を持ちます。この変数がいつ、どのように設定されるかについては、src/backend/storage/lmgr/deadlock.c 内の他の関数による更新ロジックを参照してください。
