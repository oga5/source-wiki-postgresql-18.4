TsmRoutine

定義

  • 種別: struct
  • 定義箇所: src/include/access/tsmapi.h:56

目的

PostgreSQL の TABLESAMPLE 句で使用されるサンプリングメソッドのインターフェースを定義し、プランナおよびエグゼキュータがサンプリングアルゴリズムを抽象的に利用できるようにする。

シグネチャまたは形状

typedef struct TsmRoutine
{
    NodeTag     type;
    List       *parameterTypes;
    bool        repeatable_across_queries;
    bool        repeatable_across_scans;
    SampleScanGetSampleSize_function SampleScanGetSampleSize;
    InitSampleScan_function InitSampleScan;
    BeginSampleScan_function BeginSampleScan;
    NextSampleBlock_function NextSampleBlock;
    NextSampleTuple_function NextSampleTuple;
    EndSampleScan_function EndSampleScan;
} TsmRoutine;

入力と出力

  • **入力**: TABLESAMPLE 句の引数型リスト (parameterTypes)、および各実行フェーズに対応する関数ポインタ。
  • **出力**: プランナおよびエグゼキュータに対するサンプリングロジックの提供。

前提条件と不変条件

  • 初期化には必ず makeNode(TsmRoutine) を使用すること。これにより将来的なフィールド追加時のメモリ安全性が保証される。
  • NextSampleBlock が NULL の場合、システムは通常のシーケンシャルスキャン動作へフォールバックする。

アルゴリズムまたは振る舞い

1. **プランニング**: SampleScanGetSampleSize を通じてサンプリング後の推定ページ数とタプル数を算出し、コスト見積もりに利用する。 2. **実行**: InitSampleScan で状態を初期化し、BeginSampleScan でスキャンを開始する。 3. **反復**: NextSampleBlock で対象ブロックを、NextSampleTuple で対象タプルを順次取得する。 4. **終了**: EndSampleScan によりリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/tablesample/tablesample.c (GetTsmRoutine 関数)
  • contrib/tsm_system_rows/tsm_system_rows.c
  • contrib/tsm_system_time/tsm_system_time.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/tablesample/bernoulli.c
  • src/backend/access/tablesample/system.c
  • **呼び出し先**: 構造体内の各関数ポインタ（SampleScanGetSampleSize 等）が、各サンプリングメソッドの実装へ委譲される。

関連データ構造

  • NodeTag: ノード型識別子。
  • List: 引数型リスト。
  • SampleScanState: エグゼキュータが保持するサンプリング状態（nodes/execnodes.h 依存）。

失敗モードと境界条件

  • InitSampleScan、NextSampleBlock、EndSampleScan は NULL を許容する。
  • BeginSampleScan、NextSampleTuple、SampleScanGetSampleSize は必須であり、これらが欠落している場合の動作は未定義。

保守担当者向けメモ

  • 本構造体は TABLESAMPLE の拡張性を担保する中心的な API である。
  • 新しいサンプリングメソッドを追加する際は、GetTsmRoutine を経由して本構造体を返すハンドラを実装すること。
