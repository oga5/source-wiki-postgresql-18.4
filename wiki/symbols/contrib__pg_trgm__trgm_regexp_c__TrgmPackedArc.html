TrgmPackedArc

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_regexp.c:429

目的

正規表現から変換されたトリグラム展開グラフにおいて、状態遷移を表現するためのコンパクトなデータ構造です。

シグネチャまたは形状

typedef struct
{
	int			targetState;	/* 遷移先状態のインデックス（0始まり） */
	int			colorTrgm;		/* 遷移条件となるカラー化トリグラムのインデックス */
} TrgmPackedArc;

入力と出力

本構造体自体はデータ保持用ですが、packGraph 関数等においてメモリ割り当ての単位として使用されます。

  • 入力: arcsCount * sizeof(TrgmPackedArc) バイトのメモリ確保。
  • 出力: グラフ走査時に targetState および colorTrgm を参照可能。

前提条件と不変条件

  • targetState は、対応する TrgmPackedGraph 内の有効な状態インデックスを指す必要があります。
  • colorTrgm は、グラフ構築時に定義されたトリグラムインデックスと整合している必要があります。

アルゴリズムまたは振る舞い

正規表現の NFA をトリグラムベースのグラフへ変換する過程で、メモリ効率を最適化するために使用されます。trigramsMatchGraph 関数によるマッチング判定時に、現在の状態から次の状態へ遷移可能かを判定するために走査されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_trgm/trgm_regexp.c 内のグラフ構築および走査ロジック（packGraph 等）。
  • 呼び出し先: 直接的な関数呼び出しは行わず、データ構造として参照されます。

関連データ構造

  • TrgmPackedGraph: 本構造体の配列を保持する上位のグラフ構造。
  • TrgmState: グラフ構築過程で使用される中間状態構造体。

失敗モードと境界条件

  • メモリ不足: MemoryContextAlloc による割り当て失敗時にエラーが発生する可能性があります。
  • インデックス範囲外: targetState がグラフの保持する状態数を超過する場合、走査時に未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は TrgmPackedGraph の一部としてメモリ上に連続して配置されることを前提としています。
  • 構造体の変更は、グラフのパッキングおよび走査ロジック全体に影響するため、メモリレイアウトの整合性に注意してください。
