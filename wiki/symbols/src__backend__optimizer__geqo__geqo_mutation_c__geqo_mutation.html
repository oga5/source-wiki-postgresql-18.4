geqo_mutation

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_mutation.c:42

目的

遺伝的アルゴリズム（GEQO）における探索空間の多様性を維持し、局所解への陥没を防ぐために、個体（ツアー）内の遺伝子をランダムに入れ替える突然変異操作を行います。

シグネチャまたは形状

void geqo_mutation(PlannerInfo *root, Gene *tour, int num_gene)

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報（乱数生成に使用）。
  • tour: 突然変異対象となる遺伝子の配列。
  • num_gene: 配列内の遺伝子数。
  • **出力**: なし（tour 配列の内容を直接書き換える）。

前提条件と不変条件

  • num_gene は正の整数であること。
  • tour は少なくとも num_gene 個の要素を持つ配列であること。
  • CX（Cycle Crossover）モードが有効な場合にのみ使用される（ソースコードの条件付きコンパイルによる）。

アルゴリズムまたは振る舞い

1. geqo_randint を使用し、最大 num_gene / 3 回の入れ替え回数（num_swaps）を決定します。 2. num_swaps が 0 になるまで以下のループを実行します。

  • geqo_randint を用いて、入れ替え対象のインデックス swap1 と swap2 を選択します。
  • swap1 と swap2 が同一の場合は、異なる値になるまで再選択します。
  • tour[swap1] と tour[swap2] の値を入れ替えます。

    • 3. 指定回数の入れ替えが完了した時点で終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_main.c 内で呼び出されていることが確認されています。
  • **呼び出し先**:
  • geqo_randint（乱数生成のため）。

関連データ構造

  • Gene: GEQOで使用される遺伝子型。
  • PlannerInfo: プランナのグローバルな状態を保持する構造体。

失敗モードと境界条件

  • num_gene が極端に小さい場合、num_swaps が 0 となり、突然変異が発生しない可能性があります。
  • swap1 と swap2 の選択において、インデックスが重複しないよう制御されています。

保守担当者向けメモ

  • 本関数は Darrell L. Whitley 氏による「Genitor」アルゴリズムに基づいています。
  • 遺伝子数の約 1/3 を入れ替えるヒューリスティックな設計となっており、収束速度と探索範囲のバランスを調整しています。
  • #if defined(CX) による条件付きコンパイル対象であるため、他の交叉アルゴリズムとの組み合わせには注意が必要です。
