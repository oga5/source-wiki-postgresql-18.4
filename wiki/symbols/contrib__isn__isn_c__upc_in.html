upc_in

定義

  • 種別: function
  • 定義箇所: contrib/isn/isn.c:1044

目的

PostgreSQL の UPC 型に対する入力関数であり、人間が読みやすい文字列形式の UPC を内部表現である ean13 型へ変換します。

シグネチャまたは形状

Datum upc_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して受け取った UPC 文字列。
  • **出力**: 成功時は ean13 型の Datum を返し、解析に失敗した場合は NULL を返します。

前提条件と不変条件

  • string2ean 関数が内部的に呼び出され、UPC 固有の検証ルール（チェックデジット等）が適用されます。
  • ean13 型の最下位ビットは、チェックデジットが不正な場合にフラグとして使用されます。

アルゴリズムまたは振る舞い

1. 引数から文字列を取得します。 2. string2ean 関数を呼び出し、対象を UPC 型として解析・検証します。 3. 検証が成功した場合、PG_RETURN_EAN13 マクロを使用して結果を返します。 4. 検証が失敗した場合、PG_RETURN_NULL() を呼び出し、無効な入力として処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の型システム（SQL レベルでの型変換時）。
  • **呼び出し先**: string2ean（文字列解析および検証用）。

関連データ構造

  • ean13: 内部ストレージ形式（uint64）。
  • UPC: string2ean に渡される識別子タイプ。

失敗モードと境界条件

  • 入力文字列が UPC の形式やチェックデジットの要件を満たさない場合、string2ean が false を返し、関数は NULL を返します。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(upc_in) により PostgreSQL の関数マネージャに登録されています。
  • 識別子ごとの検証ロジックを変更する場合は、string2ean または UPC.h で定義された範囲テーブルを確認してください。
