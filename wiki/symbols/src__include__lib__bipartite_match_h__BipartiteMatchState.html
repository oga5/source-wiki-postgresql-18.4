BipartiteMatchState

定義

  • 種別: struct
  • 定義箇所: src/include/lib/bipartite_match.h:27

目的

二部グラフにおける最大マッチング問題を解くための Hopcroft-Karp アルゴリズムの実行状態、入力データ、および計算結果を保持する構造体です。

シグネチャまたは形状

typedef struct BipartiteMatchState
{
    int         u_size;         /* U側のノード数 */
    int         v_size;         /* V側のノード数 */
    short     **adjacency;      /* 隣接リスト: adjacency[u] = [k, v1, ..., vk] */
    int         matching;       /* マッチングされたエッジ数 */
    short      *pair_uv;        /* UからVへのペアリング結果 */
    short      *pair_vu;        /* VからUへのペアリング結果 */
    short      *distance;       /* アルゴリズム内部用: 距離配列 */
    short      *queue;          /* アルゴリズム内部用: BFS用キュー */
} BipartiteMatchState;

入力と出力

  • **入力**: u_size, v_size（グラフのサイズ）、adjacency（各ノードの接続先リスト）。
  • **出力**: matching（最大マッチング数）、pair_uv および pair_vu（各ノードの対応関係）。

前提条件と不変条件

  • ノード数は SHRT_MAX 未満である必要があります（内部で short 型を使用しているため）。
  • adjacency リストの所有権は呼び出し元にあり、本構造体はそれを参照します。

アルゴリズムまたは振る舞い

Hopcroft-Karp アルゴリズムの実行状態を管理します。hk_breadth_search による増加パスの探索と、hk_depth_search によるマッチングの更新を反復し、増加パスが見つからなくなるまで計算を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/lib/bipartite_match.c 内の BipartiteMatch 関数で初期化され、src/backend/optimizer/plan/planner.c 等から利用されます。
  • **呼び出し先**: hk_breadth_search および hk_depth_search 関数が、計算状態を更新するために本構造体を参照します。

関連データ構造

  • BipartiteMatch(): 本構造体を生成・初期化する関数。
  • BipartiteMatchFree(): 本構造体が保持する作業領域を解放する関数。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc 経由）の挙動は PostgreSQL の標準的なエラー処理に従います。
  • ノード数が SHRT_MAX を超える場合、内部の short 型配列でオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • BipartiteMatch で割り当てられたメモリは、必ず BipartiteMatchFree で解放してください。
  • adjacency 配列のメモリ管理は呼び出し元が行うため、本構造体側では解放されません。
