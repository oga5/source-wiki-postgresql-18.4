variable_is_guc_list_quote

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:729

目的

特定の GUC（Grand Unified Configuration）パラメータが、リスト形式であり、かつダンプ時に個別の要素としてクォート処理が必要なものかどうかを判定します。

シグネチャまたは形状

bool variable_is_guc_list_quote(const char *name)

入力と出力

  • **入力**: name (判定対象の GUC パラメータ名)
  • **出力**: 対象がリストクォートを必要とするパラメータであれば true、そうでなければ false を返します。

前提条件と不変条件

  • パラメータ名の比較には pg_strcasecmp を使用しており、大文字小文字を区別しません。
  • 判定対象となるリストは以下の通りです:
  • local_preload_libraries
  • oauth_validator_libraries
  • search_path
  • session_preload_libraries
  • shared_preload_libraries
  • temp_tablespaces
  • unix_socket_directories

アルゴリズムまたは振る舞い

1. 引数 name を定義済みのリスト（上記）と大文字小文字を無視して比較します。 2. 一致するものがあれば true を返し、いずれにも該当しない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/dumputils.c (内部ロジック)
  • src/bin/pg_dump/pg_dump.c
  • src/backend/utils/misc/guc_tables.c (参照)
  • **呼び出し先**:
  • pg_strcasecmp (標準的な文字列比較関数)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • name が NULL の場合の挙動は定義されていません。
  • 新しい GUC パラメータが追加された場合、本関数に明示的に追加しない限り、デフォルトで false（クォート不要）と判定されます。

保守担当者向けメモ

  • 本関数は src/backend/utils/misc/guc.c で定義される GUC 変数の管理と同期している必要があります。リスト形式のパラメータが増減した際は、必ず本関数を更新してください。
