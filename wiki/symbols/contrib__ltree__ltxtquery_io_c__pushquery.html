pushquery

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_io.c:154

目的

クエリ解析中に生成された新しいノード（NODE）を、解析状態（QPRS_STATE）が保持する連結リストの先頭に追加します。

シグネチャまたは形状

bool pushquery(QPRS_STATE *state, int32 type, int32 val, int32 distance, int32 lenval, uint16 flag)

入力と出力

  • **入力**:
  • state: 解析状態を保持する構造体へのポインタ。
  • type: ノードの型（演算子またはオペランド）。
  • val: ノードの値。
  • distance: オペランド間の距離。
  • lenval: オペランドの長さ。
  • flag: ノードに関連付けられたフラグ。
  • **出力**: 成功した場合は true を返します。エラー発生時は ereturn マクロにより呼び出し元へ制御が戻ります。

前提条件と不変条件

  • distance は 0xffff 以下である必要があります。
  • lenval は 0xff 以下である必要があります。
  • 上記条件を満たさない場合、エラーが発生し関数は終了します。

アルゴリズムまたは振る舞い

1. palloc を使用して新しい NODE 構造体をメモリ上に確保します。 2. 引数として受け取った type, val, flag, distance, lenval をノードに格納します。 3. state->str が指す既存のリストを新しいノードの next ポインタに設定し、リストの先頭を新しいノードに更新します。 4. state->num（ノード数）をインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/ltxtquery_io.c 内の解析ロジック（makepol 等）から呼び出されます。
  • **呼び出し先**: palloc（PostgreSQL メモリ管理関数）を呼び出します。
  • **注記**: contrib/intarray/_int_bool.c にも同名の関数が存在しますが、シグネチャが異なるため、本ページで説明している ltxtquery_io.c の関数とは別の実装です。

関連データ構造

  • NODE: 解析中に一時的に使用される連結リスト形式のツリーノード。
  • QPRS_STATE: 解析中の状態（バッファ、演算子スタック、エラーコンテキスト）を保持する構造体。

失敗モードと境界条件

  • distance > 0xffff: ERRCODE_INVALID_PARAMETER_VALUE エラーを発生させます。
  • lenval > 0xff: ERRCODE_INVALID_PARAMETER_VALUE エラーを発生させます。

保守担当者向けメモ

  • 本関数は ltxtquery の構文解析におけるノード構築の基本単位です。
  • palloc を使用しているため、メモリリークの懸念はありません（PostgreSQL のメモリコンテキスト管理に依存）。
  • distance および lenval の制約は、ltxtquery の内部バイナリ形式の仕様に直結しています。
