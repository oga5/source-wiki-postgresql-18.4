can_coerce_type

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_coerce.c:556

目的

複数の入力型を対応するターゲット型へ変換可能かどうかを判定する。

シグネチャまたは形状

bool can_coerce_type(int nargs, const Oid *input_typeids, const Oid *target_typeids, CoercionContext ccontext)

入力と出力

  • nargs: 引数の数。
  • input_typeids: 入力型の OID 配列。
  • target_typeids: ターゲット型の OID 配列。
  • ccontext: 変換コンテキスト（COERCION_IMPLICIT 等）。
  • 戻り値: すべての引数で変換が可能であれば true、そうでなければ false。

前提条件と不変条件

  • input_typeids と target_typeids は少なくとも nargs 分の要素を持つ必要がある。
  • 多相型（Polymorphic types）が含まれる場合、最終的に check_generic_type_consistency による整合性チェックが行われる。

アルゴリズムまたは振る舞い

1. 各引数について以下の順序で判定を行う：

  • 型が同一、ターゲットが ANYOID、または入力が UNKNOWNOID の場合はスキップ。
  • ターゲットが多相型の場合はフラグを立ててスキップ。
  • find_coercion_pathway を呼び出し、pg_cast に基づく変換パスが存在するか確認。
  • RECORD 型と複合型の相互変換、または複合型配列の変換可能性をチェック。
  • typeInheritsFrom または typeIsOfTypedTable による継承関係をチェック。

    • 2. 全ての引数で変換可能と判定された後、多相型が含まれていた場合は check_generic_type_consistency で最終的な整合性を検証する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c、src/backend/parser/parse_clause.c、および src/backend/parser/parse_coerce.c 内の各所。
  • **呼び出し先**: find_coercion_pathway、check_generic_type_consistency、typeInheritsFrom、typeIsOfTypedTable、is_complex_array。

関連データ構造

  • Oid: 型識別子。
  • CoercionContext: 変換の許容範囲を定義する列挙型。

失敗モードと境界条件

  • いずれかの引数で変換パスが見つからず、かつ特殊な許容条件（継承や RECORD 等）にも該当しない場合、即座に false を返す。
  • 多相型の整合性チェックに失敗した場合、false を返す。

保守担当者向けメモ

  • RECORDARRAYOID と複合型配列の変換ロジックは現在 #ifdef NOT_USED で無効化されている。
  • find_coercion_pathway の結果に依存しているため、新しい型変換ルールを追加する場合は pg_cast カタログの更新が必要となる。
