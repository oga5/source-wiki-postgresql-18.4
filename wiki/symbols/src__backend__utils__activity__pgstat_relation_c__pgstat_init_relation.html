pgstat_init_relation

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:90

目的

リレーション（テーブルやインデックス）がオープンされた際に、そのリレーションが統計情報の追跡対象であるかを判定し、追跡の有効化または無効化を行う。

シグネチャまたは形状

void pgstat_init_relation(Relation rel)

入力と出力

  • **入力**: Relation rel（初期化対象のリレーション構造体）
  • **出力**: なし（副作用として rel 構造体のメンバ pgstat_enabled および pgstat_info を更新する）

前提条件と不変条件

  • rel は有効なリレーション構造体であること。
  • グローバル変数 pgstat_track_counts が統計収集の有効/無効を制御する。

アルゴリズムまたは振る舞い

1. **対象判定**: リレーションの種類（relkind）を確認する。ストレージを持つリレーションまたはパーティションテーブル以外は、統計追跡を無効化（pgstat_enabled = false, pgstat_info = NULL）して終了する。 2. **グローバル設定確認**: pgstat_track_counts が無効な場合、既存の統計情報があれば pgstat_unlink_relation でリンクを解除し、追跡を無効化して終了する。 3. **有効化**: 上記の条件に該当しない場合、rel->pgstat_enabled を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/relation.c:75
  • src/backend/access/common/relation.c:125
  • **呼び出し先**:
  • pgstat_unlink_relation（条件付きで呼び出される）

関連データ構造

  • Relation: リレーションのメタデータと統計状態を保持する構造体。
  • PgStat_TableStatus: リレーションの統計情報を保持する構造体（本関数はこれへの参照を管理する）。

失敗モードと境界条件

  • pgstat_track_counts が無効な場合、既に設定済みの統計情報とのリンクが解除される。
  • ストレージを持たないリレーションやパーティションテーブル以外の特殊なリレーションは、統計追跡の対象外として処理される。

保守担当者向けメモ

  • 本関数はリレーションのオープン時に呼ばれる初期化処理である。統計情報の紐付け（pgstat_assoc_relation）とは別の処理であることに注意すること。
  • pgstat_enabled フラグは、後続のDML操作等で統計を更新すべきかどうかの判定に使用される。
