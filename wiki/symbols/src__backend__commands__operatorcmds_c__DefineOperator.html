DefineOperator

定義

  • 種別: function
  • 定義箇所: src/backend/commands/operatorcmds.c:66

目的

SQLの CREATE OPERATOR 文を処理し、指定された属性に基づいてシステムカタログ pg_operator に新しい演算子を定義するためのメインエントリポイントとして機能します。

シグネチャまたは形状

ObjectAddress DefineOperator(List *names, List *parameters)

入力と出力

  • **入力**:
  • names: 演算子の名前（スキーマ修飾を含むリスト）。
  • parameters: 演算子の属性を定義する DefElem リスト（関数名、引数型、commutator、negator、推定関数など）。
  • **出力**:
  • ObjectAddress: 作成された演算子のオブジェクトアドレスを返します。

前提条件と不変条件

  • 演算子を作成する前に、関連する関数や型が定義されている必要があります。
  • 演算子の引数型（leftarg, rightarg）には SETOF を指定できません。
  • 演算子関数および引数型に対して、実行権限（EXECUTE）および使用権限（USAGE）が必要です。
  • 少なくとも一つの引数型（左または右）が指定されている必要があります。

アルゴリズムまたは振る舞い

1. **名前空間の解決**: QualifiedNameGetCreationNamespace を使用して、演算子を作成するスキーマを特定します。 2. **権限チェック**: 対象スキーマに対する CREATE 権限を検証します。 3. **パラメータ解析**: parameters リストを走査し、演算子の属性（関数名、型、commutator、negator、推定関数、ハッシュ/マージ可否）を抽出します。 4. **型解決**: typenameTypeId を用いて型名を OID に変換し、権限を検証します。 5. **関数解決**: LookupFuncName を使用して、指定された関数名と引数型から演算子の背後にある関数の OID を取得します。 6. **権限再検証**: 取得した関数に対する EXECUTE 権限および戻り値の型に対する USAGE 権限を検証します。 7. **推定関数の検証**: restrictionName が指定されている場合、ValidateRestrictionEstimator を呼び出して検証します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c（SQL解析の過程で呼び出されます）
  • **呼び出し先**:
  • QualifiedNameGetCreationNamespace
  • object_aclcheck
  • defGetTypeName / defGetQualifiedName / defGetBoolean
  • typenameTypeId
  • LookupFuncName
  • ValidateRestrictionEstimator（部分的な言及あり）

関連データ構造

  • List, DefElem, TypeName, ObjectAddress
  • pg_operator（システムカタログ）

失敗モードと境界条件

  • **エラー**:
  • 必要な権限がない場合（aclcheck_error）。
  • 必須の関数が指定されていない場合。
  • 引数型が指定されていない場合（両方欠落、または右引数のみ欠落した「後置演算子」の試行）。
  • 演算子属性が認識できない場合（警告を発行）。
  • **境界条件**:
  • 演算子の引数が1つ（単項）または2つ（二項）のケースを適切に処理します。

保守担当者向けメモ

  • 本関数は CREATE OPERATOR の構文解析と検証を担当します。実際のカタログ挿入処理は、この後の工程（ソースコードの続き）で呼び出される OperatorCreate 等に委譲されます。
  • 過去の互換性のために、一部の古いオプション（sort1, sort2 等）は canMerge として扱われます。
