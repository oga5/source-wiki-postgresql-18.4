NumLWLocksForNamedTranches

定義

  • src/backend/storage/lmgr/lwlock.c:416

目的

名前付きLWLockトランチ（Named LWLock Tranches）に割り当てられたLWLockの総数を計算し、共有メモリ割り当てサイズの算出などに利用する。

シグネチャまたは形状

static int NumLWLocksForNamedTranches(void)

入力と出力

  • **入力**: なし（グローバル変数 NamedLWLockTrancheRequests および NamedLWLockTrancheRequestArray を参照）。
  • **出力**: int 型の合計ロック数。

前提条件と不変条件

  • NamedLWLockTrancheRequestArray が初期化済みであること。
  • NamedLWLockTrancheRequests が現在のトランチ要求数を正しく保持していること。

アルゴリズムまたは振る舞い

1. カウンタ numLocks を 0 に初期化する。 2. NamedLWLockTrancheRequests 回のループを実行する。 3. 各イテレーションで、NamedLWLockTrancheRequestArray の各要素が持つ num_lwlocks を numLocks に加算する。 4. 最終的な合計値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/lwlock.c 内の LWLockShmemSize 等で、共有メモリの必要サイズを計算する際に呼び出される。
  • **呼び出し先**:
  • 直接的な関数呼び出しはない。グローバル配列 NamedLWLockTrancheRequestArray へのアクセスを行う。

関連データ構造

  • NamedLWLockTrancheRequest: 各トランチの要求情報を保持する構造体。num_lwlocks フィールドを持つ。

失敗モードと境界条件

  • NamedLWLockTrancheRequests が 0 の場合、ループは実行されず 0 を返す。
  • 配列の範囲外アクセスは、NamedLWLockTrancheRequests が正しく管理されている限り発生しない。

保守担当者向けメモ

  • 本関数は静的関数であり、lwlock.c 内でのみ使用される。
  • 新しいトランチタイプを追加する際は、NamedLWLockTrancheRequestArray の整合性が保たれていることを確認すること。
