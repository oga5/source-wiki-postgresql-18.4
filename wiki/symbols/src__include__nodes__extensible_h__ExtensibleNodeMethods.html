ExtensibleNodeMethods

定義

  • **種別**: struct
  • **定義箇所**: src/include/nodes/extensible.h:62

目的

拡張ノード（ExtensibleNode）のライフサイクルを制御するためのコールバック関数群を定義し、コアシステムが拡張機能によって定義されたノードをコピー、比較、シリアライズ、デシリアライズできるようにします。

シグネチャまたは形状

typedef struct ExtensibleNodeMethods
{
    const char *extnodename;
    Size        node_size;
    void        (*nodeCopy) (struct ExtensibleNode *newnode, const struct ExtensibleNode *oldnode);
    bool        (*nodeEqual) (const struct ExtensibleNode *a, const struct ExtensibleNode *b);
    void        (*nodeOut) (struct StringInfoData *str, const struct ExtensibleNode *node);
    void        (*nodeRead) (struct ExtensibleNode *node);
} ExtensibleNodeMethods;

入力と出力

  • **入力**: 拡張ノードの識別名（extnodename）、ノードのメモリサイズ（node_size）、および各操作（コピー、比較、入出力）を実行する関数ポインタ。
  • **出力**: なし（構造体として定義）。

前提条件と不変条件

  • extnodename は一意である必要があり、最大長は EXTNODENAME_MAX_LEN（64バイト）です。
  • 登録されたメソッドは、コアシステムがノードを処理する際に間接的に呼び出されます。
  • すべてのメソッドポインタは必須です。特に nodeRead は pg_strtok() を用いた手動パースの実装が求められます。

アルゴリズムまたは振る舞い

1. **登録**: 拡張機能が RegisterExtensibleNodeMethods を呼び出し、この構造体をハッシュテーブルに格納します。 2. **検索**: コアシステムはノードの extnodename をキーとして GetExtensibleNodeMethods を呼び出し、コールバックを取得します。 3. **実行**: コアシステムは取得したポインタを通じて、ノードのコピー、比較、シリアライズ（nodeOut）、デシリアライズ（nodeRead）を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/copyfuncs.c、equalfuncs.c、outfuncs.c、readfuncs.c が、ノード処理の過程でこの構造体のメソッドを参照します。
  • **呼び出し先**: 拡張機能側で実装された各関数ポインタが呼び出されます。

関連データ構造

  • ExtensibleNode: 本メソッド群によって制御される基底構造体。
  • StringInfoData: nodeOut でシリアライズ先として使用されます。

失敗モードと境界条件

  • 登録されていない extnodename を参照した場合、エラーが発生します（src/backend/nodes/extensible.c:114 参照）。
  • nodeRead におけるパース失敗は、システム全体のデシリアライズ処理の失敗に直結します。

保守担当者向けメモ

  • 本構造体は拡張機能のインターフェースであるため、変更時は既存の拡張機能との互換性に注意してください。
  • nodeRead の実装には pg_strtok() を使用した手動パースが必要である点に留意してください。
