tuplesort_reset

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesort.c:1018

目的

既存の Tuplesortstate を再利用可能な状態にリセットし、次のソートバッチを開始するための準備を行う。

シグネチャまたは形状

void tuplesort_reset(Tuplesortstate *state)

入力と出力

  • **入力**: state (Tuplesortstate *) - リセット対象のソート状態構造体。
  • **出力**: なし（副作用として state が更新される）。

前提条件と不変条件

  • state は有効な Tuplesortstate ポインタである必要がある。
  • 本関数呼び出しにより、現在のバッチに関連するメモリリソースが解放される。

アルゴリズムまたは振る舞い

1. tuplesort_updatemax(state) を呼び出し、現在のソート実行に関する統計情報を更新する。 2. tuplesort_free(state) を呼び出し、現在のバッチで使用していたメモリやリソースを解放する。 3. tuplesort_begin_batch(state) を呼び出し、新しいバッチのための初期化を行う。 4. state 内のポインタメンバ（lastReturnedTuple, slabMemoryBegin, slabMemoryEnd, slabFreeHead）を NULL に初期化し、スラブメモリ管理状態をクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeIncrementalSort.c (複数箇所)
  • src/backend/optimizer/path/costsize.c (コスト計算時の言及)
  • **呼び出し先**:
  • tuplesort_updatemax
  • tuplesort_free
  • tuplesort_begin_batch

関連データ構造

  • Tuplesortstate: ソート操作のプライベートな状態を保持するメイン構造体。

失敗モードと境界条件

  • state が NULL の場合の挙動については、本ソースコード断片からは定義されていない。

保守担当者向けメモ

  • 本関数は、IncrementalSort ノードのように、同一の Tuplesortstate を使い回して複数回のソートを行う場合に利用される。
  • tuplesort_free を内部で呼び出しているため、リセット後に再度ソートを開始する際は tuplesort_begin_batch による再初期化が必須となる。
