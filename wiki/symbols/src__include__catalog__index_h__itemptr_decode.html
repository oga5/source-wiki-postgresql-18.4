itemptr_decode

定義

  • 種別: function
  • 定義箇所: src/include/catalog/index.h:210

目的

int64 型としてエンコードされたタプル識別子（TID）を、元の ItemPointer 構造体へ復元する。

シグネチャまたは形状

static inline void itemptr_decode(ItemPointer itemptr, int64 encoded)

入力と出力

  • **入力**:
  • itemptr: 復元結果を格納する ItemPointer へのポインタ。
  • encoded: itemptr_encode によって変換された int64 値。
  • **出力**: なし（itemptr が指すメモリ領域が更新される）。

前提条件と不変条件

  • encoded 値は、itemptr_encode で生成された形式（上位ビットに BlockNumber、下位 16 ビットに OffsetNumber）に従っている必要がある。

アルゴリズムまたは振る舞い

1. encoded 値を右に 16 ビットシフトし、BlockNumber を抽出する。 2. encoded 値の下位 16 ビット（0xFFFF との論理積）を抽出し、OffsetNumber とする。 3. ItemPointerSet マクロを使用して、抽出したブロック番号とオフセット番号を itemptr に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_handler.c:1900 にて、Datum から変換された値のデコードに使用されている。
  • **呼び出し先**: ItemPointerSet マクロ。

関連データ構造

  • ItemPointer: タプルを特定するための物理的な位置情報（ブロック番号とオフセット番号）。
  • int64: ソート可能な形式に変換された中間表現。

失敗モードと境界条件

  • encoded 値が int64 の範囲を超えてエンコードされている場合、ビットシフトによりデータが欠落する可能性がある。
  • OffsetNumber が 16 ビット（65535）を超える値を持つ場合、下位 16 ビットのみが抽出される。

保守担当者向けメモ

  • 本関数は itemptr_encode と対になる処理である。インデックス検証時のバルク処理におけるパフォーマンス最適化を目的としているため、変更時はソート順序への影響を考慮すること。
