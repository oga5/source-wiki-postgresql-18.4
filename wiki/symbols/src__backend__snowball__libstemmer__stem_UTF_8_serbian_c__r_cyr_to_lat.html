r_cyr_to_lat

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/snowball/libstemmer/stem_UTF_8_serbian.c:4793

目的

入力テキスト内のキリル文字を対応するラテン文字へ変換し、後続のステミング処理のために文字コードを正規化する。

シグネチャまたは形状

static int r_cyr_to_lat(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境構造体。現在の文字列バッファおよびカーソル位置を含む）
  • **出力**: 成功時に 1 を返す。変換処理中にエラーが発生した場合は負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • 内部で使用される a_0 テーブル（among 構造体）が初期化されていること。

アルゴリズムまたは振る舞い

1. 文字列全体を走査し、find_among を用いて a_0 テーブル内のキリル文字パターンを検索する。 2. マッチしたキリル文字のシーケンスを、slice_from_s を用いて対応するラテン文字へ置換する。 3. マッチしない文字は skip_utf8 を用いてスキップし、次の文字へ進む。 4. 文字列の末尾に到達するまでこの処理を繰り返し、最終的にカーソル位置を初期位置へ戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_serbian.c 内のメイン処理（serbian_UTF_8_stem 等から間接的に呼び出される）。
  • **呼び出し先**: find_among（パターン検索）、slice_from_s（文字列置換）、skip_utf8（UTF-8文字スキップ）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • among: 変換ルールを定義する探索用テーブル（a_0）。

失敗モードと境界条件

  • slice_from_s が失敗した場合（メモリ不足やバッファ操作エラー）、その戻り値（負数）をそのまま呼び出し元へ伝播させる。
  • UTF-8 文字列の走査中に不正なシーケンスに遭遇した場合、skip_utf8 がエラーを返し、処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接Cファイルを編集せず、対応するSnowballソース（.sbl）を修正すること。
  • a_0 テーブルのサイズ（30）や各ケースの置換ルールは、セルビア語の文字変換仕様に依存している。
