astreamer_zstd_compressor_new

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_zstd.c:69

目的

Zstandard (zstd) 圧縮を行うストリーミング処理のインスタンスを生成し、パイプラインの構成要素として初期化します。

シグネチャまたは形状

astreamer *astreamer_zstd_compressor_new(astreamer *next, pg_compress_specification *compress)

入力と出力

  • **入力**:
  • next: 後続の処理を行う astreamer オブジェクト（必須）。
  • compress: 圧縮レベル、ワーカースレッド数、ロングディスタンスマッチング設定を含む pg_compress_specification 構造体。
  • **出力**:
  • 圧縮処理用に初期化された astreamer オブジェクトへのポインタ。

前提条件と不変条件

  • next は NULL であってはならない。
  • ビルド時に USE_ZSTD が定義されている必要がある。
  • pg_compress_specification に基づき、zstd コンテキスト (ZSTD_CCtx) が正常に作成・設定されること。

アルゴリズムまたは振る舞い

1. astreamer_zstd_frame 構造体を割り当て、astreamer_zstd_compressor_ops を操作関数として設定する。 2. astreamer 基底構造体の bbs_next に next を格納し、出力用バッファを初期化する。 3. ZSTD_createCCtx() で圧縮コンテキストを生成する。 4. compress 引数の設定値（圧縮レベル、ワーカースレッド数、ロングディスタンスマッチング）を ZSTD_CCtx_setParameter でコンテキストに適用する。 5. zstd 出力用バッファ (zstd_outBuf) を初期化し、基底構造体のバッファと紐付ける。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • **呼び出し先**:
  • palloc0
  • initStringInfo / enlargeStringInfo
  • ZSTD_createCCtx
  • ZSTD_CCtx_setParameter
  • pg_fatal (エラー発生時)

関連データ構造

  • astreamer_zstd_frame: 圧縮コンテキストとバッファを保持する内部構造体。
  • astreamer: ストリーマーの基底構造体。
  • pg_compress_specification: 圧縮設定を保持する構造体。

失敗モードと境界条件

  • USE_ZSTD が未定義の場合、pg_fatal を呼び出し異常終了する。
  • ZSTD_createCCtx が失敗した場合、pg_fatal で終了する。
  • 圧縮レベルやワーカースレッド数などのパラメータ設定が ZSTD_isError を返した場合、pg_fatal で終了する。

保守担当者向けメモ

  • 本関数は USE_ZSTD が定義されている環境でのみ動作します。
  • ZSTD_c_nbWorkers の設定は、使用している libzstd のバージョンやビルド時のスレッドサポート状況に依存するため、設定失敗時のエラーハンドリングが重要です。
