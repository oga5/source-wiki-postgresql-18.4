TidStore

定義

  • 種別: struct
  • 定義箇所: src/backend/access/common/tidstore.c:114

目的

ItemPointerData (TID) の集合をメモリ効率よく保持・検索するためのデータ構造です。主に VACUUM 処理等において、特定のブロックに含まれるタプルのオフセットを追跡するために使用されます。

シグネチャまたは形状

struct TidStore
{
    MemoryContext rt_context;
    union
    {
        local_ts_radix_tree *local;
        shared_ts_radix_tree *shared;
    } tree;
    dsa_area *area;
};

入力と出力

  • **入力**: TidStoreCreateLocal または TidStoreCreateShared を介して初期化されます。共有メモリ使用時は dsa_area が必要です。
  • **出力**: TidStoreIsMember による存在判定、または TidStoreIterateNext を用いた全 TID の反復取得が可能です。

前提条件と不変条件

  • TidStoreSetBlockOffsets を使用する場合、オフセットは昇順で指定する必要があります。
  • 同一ブロックに対する再登録は、以前の情報を上書きします。
  • 共有メモリ版を使用する場合、呼び出し側で TidStoreLockExclusive 等の適切なロック管理を行う必要があります。

アルゴリズムまたは振る舞い

  • **ストレージ**: 基盤となる lib/radixtree.h を利用し、BlockNumber をキーとしてオフセット情報を管理します。
  • **メモリ管理**: ローカルメモリ（MemoryContext）と共有メモリ（DSA）の両方に対応しています。
  • **最適化**: オフセット数が少ない場合は BlocktableEntry のヘッダー領域に直接格納し、多い場合はビットマップを使用することでメモリ効率を最適化しています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/vacuumlazy.c 等の VACUUM 処理で利用されます。
  • **呼び出し先**: lib/radixtree.h（ラディックツリー操作）、utils/dsa.h（共有メモリ管理）、storage/lwlock.h（ロック制御）を内部的に利用します。
  • ※具体的な関数呼び出しの全容については、提供されたソース excerpt には網羅されていません。

関連データ構造

  • BlocktableEntry: 各ブロックのオフセット情報を保持する構造体。
  • TidStoreIter: イテレーション状態を保持する構造体。
  • local_ts_radix_tree / shared_ts_radix_tree: 基盤となるラディックツリー型。

失敗モードと境界条件

  • 共有メモリ使用時に適切なロックを取得せずにアクセスした場合、競合が発生します。
  • オフセットの順序が昇順でない場合、期待される動作を保証できません。

保守担当者向けメモ

  • 本構造体は src/include/access/tidstore.h にも定義が存在します。
  • 共有メモリ版の TidStore は TidStoreAttach を通じて他プロセスから参照可能です。
