PQsetAuthDataHook

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth.c:1594

目的

認証データ取得用のフック関数をグローバルに設定し、認証プロセス中に外部から認証情報を注入可能にする。

シグネチャまたは形状

void PQsetAuthDataHook(PQauthDataHook_type hook);

入力と出力

  • **入力**: hook (PQauthDataHook_type 型) - 設定するフック関数ポインタ。
  • **出力**: なし。

前提条件と不変条件

  • 引数 hook が NULL の場合、内部的に PQdefaultAuthDataHook がデフォルトとして割り当てられる。
  • 呼び出し後、以降の認証処理において指定されたフックが使用される。

アルゴリズムまたは振る舞い

1. 引数 hook が有効（非NULL）であれば、それをグローバル変数 PQauthDataHook に代入する。 2. 引数 hook が NULL であれば、デフォルトの PQdefaultAuthDataHook を PQauthDataHook に代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/oauth_validator/oauth_hook_client.c:129 にて handle_auth_data を引数として呼び出されている証拠あり。
  • **呼び出し先**:
  • 直接的な呼び出し先はソースコード上には存在しない（フック関数を代入するのみ）。

関連データ構造

  • PQauthDataHook_type: 認証データ取得フックの型定義。
  • PQdefaultAuthDataHook: デフォルトの認証データ取得関数。

失敗モードと境界条件

  • 特筆すべき失敗モードなし。NULL を渡すことで安全にデフォルト状態へリセット可能。

保守担当者向けメモ

  • 本関数はグローバルな状態を変更するため、マルチスレッド環境下での呼び出し順序には注意が必要である。
  • 認証フックの詳細は fe-auth.h および libpq-fe.h を参照のこと。
