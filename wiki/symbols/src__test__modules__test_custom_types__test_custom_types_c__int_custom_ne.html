int_custom_ne

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_types/test_custom_types.c:125

目的

カスタム型に対する「等しくない」比較演算（<>）を実行し、真偽値を返す。

シグネチャまたは形状

Datum int_custom_ne(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの int32 型の値。
  • **出力**: PG_RETURN_BOOL を介して、2つの入力値が等しくない場合は true、等しい場合は false を返す。

前提条件と不変条件

  • 入力引数は int32 として解釈可能であること。
  • PostgreSQLの関数マネージャインターフェース（fmgr.h）の規約に従うこと。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32(0) および PG_GETARG_INT32(1) を使用して、2つの引数を int32 型として取得する。 2. C言語の不等価演算子 != を用いて比較を行う。 3. 比較結果を PG_RETURN_BOOL で PostgreSQL の Datum 形式に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの演算子実行エンジン（SQLレベルでカスタム型の <> 演算子が呼び出された際に実行される）。
  • **呼び出し先**: PG_GETARG_INT32 および PG_RETURN_BOOL マクロ（PostgreSQLの関数マネージャ）。
  • **直接の証拠**: src/test/modules/test_custom_types/test_custom_types.c:32 にて PG_FUNCTION_INFO_V1(int_custom_ne) が宣言されており、関数として登録されている。

関連データ構造

  • int32: 内部表現として使用される整数型。

失敗モードと境界条件

  • 入力が int32 の範囲外である場合、PG_GETARG_INT32 の挙動に依存する。
  • 比較対象の型が int32 以外である場合、型不整合によるエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はテストモジュールの一部であり、単純な int32 の比較ロジックを実装している。
  • 複雑なロジックは含まれていないため、変更時は他の比較関数（int_custom_eq 等）との整合性に注意すること。
