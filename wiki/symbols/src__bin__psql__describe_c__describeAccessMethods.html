describeAccessMethods

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:147

目的

データベース内のアクセスメソッド（Access Methods）の一覧をシステムカタログから取得し、整形して出力する。

シグネチャまたは形状

bool describeAccessMethods(const char *pattern, bool verbose)

入力と出力

  • **入力**:
  • pattern: フィルタリング対象のアクセスメソッド名（ワイルドカード指定可）。
  • verbose: 詳細表示フラグ。true の場合、ハンドラ情報と説明文が追加される。
  • **出力**:
  • 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • 接続先のサーバーバージョンが 9.6 以上であること。それ未満の場合はエラーを出力して終了する。
  • pset.sversion が正しく設定されていること。

アルゴリズムまたは振る舞い

1. サーバーバージョンが 9.6 未満であればエラーを報告し、処理を中断する。 2. PQExpBuffer を初期化し、pg_catalog.pg_am テーブルからアクセスメソッド名とタイプ（インデックスまたはテーブル）を取得するクエリを構築する。 3. verbose が指定されている場合、amhandler と obj_description をクエリに追加する。 4. validateSQLNamePattern を呼び出し、pattern 引数に基づいた WHERE 句を構築する。 5. PSQLexec を実行し、結果を printQuery を用いて表形式で出力する。 6. 使用したバッファと結果セットを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:1073
  • **呼び出し先**:
  • initPQExpBuffer, printfPQExpBuffer, appendPQExpBuffer, appendPQExpBufferStr, termPQExpBuffer (バッファ操作)
  • validateSQLNamePattern (パターン検証)
  • PSQLexec (クエリ実行)
  • printQuery (結果出力)
  • PQclear (結果解放)

関連データ構造

  • PQExpBufferData: SQL クエリ構築用バッファ。
  • PGresult: クエリ結果保持用。
  • printQueryOpt: 出力オプション設定。

失敗モードと境界条件

  • サーバーバージョンが 9.6 未満の場合、エラーメッセージを表示し true を返す。
  • validateSQLNamePattern が失敗した場合、バッファを解放して false を返す。
  • PSQLexec が失敗した場合、false を返す。

保守担当者向けメモ

  • 本関数は \dA メタコマンドの実装に使用される。
  • 新しいアクセスメソッドタイプが追加された場合は、CASE 文の条件分岐を更新する必要がある。
