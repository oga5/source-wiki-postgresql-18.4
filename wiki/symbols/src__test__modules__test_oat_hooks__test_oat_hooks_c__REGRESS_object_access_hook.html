REGRESS_object_access_hook

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_oat_hooks/test_oat_hooks.c:323

目的

PostgreSQLのオブジェクトアクセスフック機構を検証するためのテスト用関数です。特定のオブジェクト操作に対する権限チェックを介入させ、GUC設定に基づいたアクセス拒否や監査ログの記録を行います。

シグネチャまたは形状

static void REGRESS_object_access_hook(ObjectAccessType access, Oid classId, Oid objectId, int subId, void *arg)

入力と出力

  • **入力**:
  • access: ObjectAccessType 型の操作種別。
  • classId, objectId, subId: 操作対象のオブジェクトを特定する識別子。
  • arg: 操作固有の追加情報。
  • **出力**: なし（void）。ただし、権限拒否時には ereport(ERROR, ...) を通じて処理を中断します。

前提条件と不変条件

  • 本関数は object_access_hook グローバル変数に登録されることで機能します。
  • 処理の最後で必ず next_object_access_hook を呼び出し、フックチェーンを維持する必要があります。

アルゴリズムまたは振る舞い

1. audit_attempt を呼び出し、操作の試行を記録します。 2. REGRESS_deny_object_access が有効かつ現在のユーザーがスーパーユーザーでない場合、ereport(ERROR, ...) を発行して操作を拒否します。 3. 拒否されない場合、next_object_access_hook が存在すればそれを呼び出し、コアシステムの本来の処理を実行します。 4. 最後に audit_success を呼び出し、操作の成功を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLコアシステムのオブジェクトアクセスフックポイント（object_access_hook を介して呼び出される）。
  • **呼び出し先**:
  • audit_attempt, audit_success: 監査ログ出力用関数。
  • accesstype_to_string, accesstype_arg_to_string: 文字列変換用ヘルパー。
  • next_object_access_hook: 以前に登録されていたフック関数。

関連データ構造

  • ObjectAccessType: オブジェクト操作の種類を定義する列挙型。
  • next_object_access_hook: フックチェーンを保持するための関数ポインタ変数。

失敗モードと境界条件

  • **権限拒否**: REGRESS_deny_object_access が有効な場合、非スーパーユーザーによる操作は ERRCODE_INSUFFICIENT_PRIVILEGE で失敗します。
  • **フックチェーンの欠如**: next_object_access_hook が NULL の場合、チェーンの転送はスキップされます。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部です。本番環境のフック実装の参考にする際は、フックのオーバーヘッドや競合に十分注意してください。
  • 並列ワーカープロセスでの動作については、emit_audit_message 内の IsParallelWorker() チェックが重複出力を防ぐ役割を果たしています。
