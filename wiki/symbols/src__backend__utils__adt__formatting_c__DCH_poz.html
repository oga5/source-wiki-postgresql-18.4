DCH_poz

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/formatting.c:635

目的

日時データ型の書式化および解析処理において、フォーマット文字列内の各キーワード（例: YYYY, MM, DD 等）を識別するための列挙型です。

シグネチャまたは形状

typedef enum {
    DCH_A_D,
    DCH_A_M,
    /* ... (中略) ... */
    DCH_y,
    _DCH_last_
} DCH_poz;

入力と出力

  • **入力**: parse_format() 関数がフォーマット文字列を走査する際、この列挙型の各要素がトークンとして割り当てられます。
  • **出力**: 解析結果として FormatNode 構造体内に格納され、後続の DCH_to_char() や DCH_from_char() で利用されます。

前提条件と不変条件

  • DCH_FF1 から DCH_FF6 までのコードは連続している必要があります。
  • 新しいフォーマット指定子を追加する場合、DCH_keywords テーブルと本列挙型の両方を更新する必要があります。

アルゴリズムまたは振る舞い

  • parse_format() がフォーマット文字列を解析する際、DCH_keywords テーブルを参照して文字列を DCH_poz 型の識別子に変換します。
  • DCH_poz は、TO_CHAR 等の関数が入力データに対してどのような変換操作を行うかを決定するインデックスとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/formatting.c 内のフォーマット解析ロジック（parse_format 等）。
  • **呼び出し先**: 本シンボル自体は列挙型であり、直接的な関数呼び出しは行いません。
  • 補足: 具体的な呼び出し関係の詳細は、ソースコードの解析範囲内では限定的です。

関連データ構造

  • FormatNode: フォーマット文字列の解析結果を保持する構造体。
  • KeyWord: フォーマット指定子を定義するテーブル。

失敗モードと境界条件

  • _DCH_last_ は列挙型の終端を示すマーカーとして使用されます。
  • 未定義のフォーマット文字列が指定された場合、パーサーは適切なエラー処理を行う必要があります。

保守担当者向けメモ

  • 本列挙型は DCH_keywords テーブルと密接に結合しています。変更時は必ず両方の整合性を確認してください。
  • FFn 系のコードは連続性が保証されている必要があるため、順序の変更には注意が必要です。
