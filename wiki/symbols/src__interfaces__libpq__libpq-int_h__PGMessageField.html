PGMessageField

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-int.h:139

目的

PostgreSQLのバックエンドから送信されるエラーメッセージや通知メッセージを、個別のフィールド（コードと内容）に分解して保持するためのデータ構造です。

シグネチャまたは形状

typedef struct pgMessageField
{
	struct pgMessageField *next;	/* リストのリンク */
	char		code;			/* フィールドコード */
	char		contents[FLEXIBLE_ARRAY_MEMBER];	/* 値（null終端） */
} PGMessageField;

入力と出力

  • 入力: 構造体生成時に code と contents が割り当てられます。
  • 出力: next ポインタを介して、複数のフィールドが連結されたリストとして保持されます。

前提条件と不変条件

  • contents は FLEXIBLE_ARRAY_MEMBER を使用しており、構造体の末尾に可変長のデータが格納されます。
  • contents は null 終端されていることが期待されます。

アルゴリズムまたは振る舞い

  • リンクドリスト形式でメッセージフィールドを管理します。
  • fe-exec.c 等において、メッセージの解析時に動的にメモリを確保し、フィールド情報を格納するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-exec.c 内のメッセージ解析処理にて、PGMessageField 型のポインタが使用されています。
  • 呼び出し先: 構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • PGconn: errFields メンバを通じて、この構造体のリストを保持します。

失敗モードと境界条件

  • FLEXIBLE_ARRAY_MEMBER を含むため、メモリ確保時には offsetof(PGMessageField, contents) を用いた適切なサイズ計算が必要です。計算を誤るとバッファオーバーフローやメモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は libpq 内部専用です。アプリケーションコードから直接参照しないでください。
  • メモリ管理は libpq の内部関数（pqResultAlloc 等）に依存しているため、ライフサイクル管理には注意が必要です。
