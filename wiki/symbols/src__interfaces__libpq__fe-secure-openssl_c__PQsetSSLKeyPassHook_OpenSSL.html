PQsetSSLKeyPassHook_OpenSSL

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-openssl.c:1927

目的

OpenSSL を使用したセキュア接続において、クライアント証明書の秘密鍵パスワードを取得するためのコールバック関数（フック）をグローバルに設定する。

シグネチャまたは形状

void PQsetSSLKeyPassHook_OpenSSL(PQsslKeyPassHook_OpenSSL_type hook)

入力と出力

  • **入力**: hook (PQsslKeyPassHook_OpenSSL_type 型) - 秘密鍵のパスワードを取得するために呼び出される関数ポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 設定されたフックは、OpenSSL が秘密鍵の復号を必要とする際に PQssl_passwd_cb を通じて利用される。
  • グローバル変数 PQsslKeyPassHook が提供された hook で更新される。

アルゴリズムまたは振る舞い

1. 引数として受け取った hook を、モジュール内のグローバル変数 PQsslKeyPassHook に代入する。 2. これにより、以降の SSL 接続確立プロセスにおいて、秘密鍵のパスワード入力が必要な場合に当該フックが使用されるようになる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-secure-openssl.c:1928
  • src/interfaces/libpq/fe-secure.c:467
  • **呼び出し先**:
  • 情報なし（本関数は代入のみを行うため、直接的な関数呼び出しは含まれない）。

関連データ構造

  • PQsslKeyPassHook: 本関数によって更新されるグローバル変数。
  • PQsslKeyPassHook_OpenSSL_type: フック関数の型定義。

失敗モードと境界条件

  • 特筆すべき失敗モードなし（単純なポインタ代入のため）。

保守担当者向けメモ

  • 本関数は libpq-fe.h で公開されており、アプリケーション側からカスタムのパスワード取得ロジックを注入するために使用される。
  • スレッドセーフティに関する明示的なロックは本関数内には存在しないため、呼び出しタイミングには注意が必要である。
