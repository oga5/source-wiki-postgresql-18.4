normalize_exec_path

定義

  • 種別: function
  • 定義箇所: src/common/exec.c:240

目的

指定されたファイルパスを絶対パスに変換し、シンボリックリンクを解決して正規化する。

シグネチャまたは形状

static int normalize_exec_path(char *path);

入力と出力

  • **入力**: path (正規化対象のパス文字列へのポインタ。MAXPGPATH 以上のバッファサイズを想定)
  • **出力**: 成功時は 0 を返し、path バッファに正規化された絶対パスが格納される。失敗時は -1 を返す。

前提条件と不変条件

  • path は書き込み可能なバッファであること。
  • 内部で pg_realpath を使用しており、その結果が NULL でないことが成功の前提となる。

アルゴリズムまたは振る舞い

1. pg_realpath(path) を呼び出し、シンボリックリンクを解決した絶対パスを取得する。 2. 取得に失敗した場合は log_error を通じてエラーを報告し、-1 を返す。 3. 成功した場合は、結果を strlcpy を用いて元の path バッファにコピーし、一時的なメモリ領域を解放する。 4. WIN32 環境下では、さらに canonicalize_path(path) を呼び出し、パス区切り文字を \ から / に変換する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/exec.c 内の複数の箇所から呼び出されている（行72, 171, 181, 212）。
  • **呼び出し先**: pg_realpath, log_error, strlcpy, free, canonicalize_path (WIN32のみ)。

関連データ構造

  • MAXPGPATH: パス文字列の最大長を規定する定数。

失敗モードと境界条件

  • pg_realpath が NULL を返した場合、パスの解決に失敗したとみなされ、エラーログ出力後に -1 が返される。
  • 入力パスが長すぎる場合や、アクセス権限がない場合などに pg_realpath が失敗する可能性がある。

保守担当者向けメモ

  • 本関数は realpath(3) のラッパーである pg_realpath に依存している。プラットフォーム固有のパス解決挙動については pg_realpath の実装を参照すること。
  • Windows環境でのパス区切り文字の正規化は、POSIX環境との互換性を維持するために重要である。
