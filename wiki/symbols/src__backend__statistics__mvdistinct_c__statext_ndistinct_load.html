statext_ndistinct_load

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mvdistinct.c:147

目的

システムカタログから多変量NDistinct統計情報を読み込み、メモリ上の構造体として復元する。

シグネチャまたは形状

MVNDistinct *statext_ndistinct_load(Oid mvoid, bool inh)

入力と出力

  • **入力**:
  • mvoid (Oid): 読み込む統計オブジェクトのOID。
  • inh (bool): 継承された統計情報を含めるかどうか。
  • **出力**:
  • MVNDistinct *: 復元された多変量NDistinct統計情報構造体へのポインタ。

前提条件と不変条件

  • 指定された mvoid と inh に対応する統計情報が pg_statistic_ext_data システムカタログに存在すること。
  • 統計情報が既に構築済みであること（stxdndistinct 属性が NULL でないこと）。

アルゴリズムまたは振る舞い

1. STATEXTDATASTXOID システムキャッシュを検索し、対象の統計オブジェクトのタプルを取得する。 2. タプルが見つからない場合はエラーを投げる。 3. Anum_pg_statistic_ext_data_stxdndistinct 属性から ndist データを抽出する。 4. 抽出したデータが NULL の場合は、統計情報が未構築であるとしてエラーを投げる。 5. statext_ndistinct_deserialize を呼び出し、バイナリ形式のデータを MVNDistinct 構造体に変換する。 6. システムキャッシュのタプルを解放し、復元した構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/selfuncs.c:4343
  • **呼び出し先**:
  • SearchSysCache2
  • SysCacheGetAttr
  • statext_ndistinct_deserialize
  • ReleaseSysCache
  • elog

関連データ構造

  • MVNDistinct: 復元される多変量NDistinct統計情報の内部表現。
  • HeapTuple: システムカタログから取得されるタプル。

失敗モードと境界条件

  • SearchSysCache2 で対象が見つからない場合、elog(ERROR) により処理が中断される。
  • 統計情報が構築されていない（stxdndistinct が NULL）場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は pg_statistic_ext_data カタログの特定の属性に依存しているため、カタログスキーマの変更時には注意が必要である。
  • 戻り値の MVNDistinct 構造体はメモリ上に確保されるため、呼び出し側での適切なメモリ管理が求められる。
