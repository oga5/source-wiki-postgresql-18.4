deconstruct_expanded_record

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/expandedrecord.c:951

目的

展開されたレコード（ExpandedRecordHeader）内のタプルを分解し、個別のフィールド値（Datum）と NULL フラグの配列を生成・保持させる。

シグネチャまたは形状

void deconstruct_expanded_record(ExpandedRecordHeader *erh)

入力と出力

  • **入力**: erh - 操作対象の ExpandedRecordHeader 構造体へのポインタ。
  • **出力**: なし（副作用として erh 内部の dvalues および dnulls 配列が更新される）。

前提条件と不変条件

  • erh は有効な ExpandedRecordHeader を指している必要がある。
  • erh->hdr.eoh_context は、配列割り当てに使用可能なメモリコンテキストとして有効であること。
  • 既に ER_FLAG_DVALUES_VALID フラグが立っている場合、関数は即座に終了する。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: ER_FLAG_DVALUES_VALID が設定済みであれば何もせず終了する。 2. **メモリ確保**: erh->dvalues が未割り当て、またはフィールド数（nfields）が変化している場合、erh->hdr.eoh_context 上に Datum 配列と bool（isnull）配列を単一のチャンクとして確保する。 3. **分解処理**:

  • ER_FLAG_FVALUE_VALID が有効な場合、heap_deform_tuple を呼び出し、物理タプルから各フィールド値を抽出する。
  • 無効な場合（空のレコード）、全フィールドを NULL として初期化する。

    • 4. **状態更新**: ER_FLAG_DVALUES_VALID フラグを立て、分解済み状態であることを記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/expandedrecord.c 内の複数の内部関数。
  • src/pl/plpgsql/src/pl_exec.c（PL/pgSQL の実行エンジン）。
  • **呼び出し先**:
  • expanded_record_get_tupdesc: タプル記述子の取得。
  • MemoryContextAlloc: メモリ確保。
  • heap_deform_tuple: タプルの分解。

関連データ構造

  • ExpandedRecordHeader: レコードのメタデータ、フラグ、フィールド値を保持するメイン構造体。
  • TupleDesc: タプルの構造定義。
  • Datum: フィールド値の汎用表現。

失敗モードと境界条件

  • **メモリ不足**: MemoryContextAlloc が失敗した場合、PostgreSQL の標準的なエラーハンドリングにより例外が発生する。
  • **フィールド数不一致**: nfields が期待値と異なる場合、既存の配列を破棄して再確保する（古い配列のメモリリークについては、コンテキスト管理により許容される）。

保守担当者向けメモ

  • 本関数は遅延評価（Lazy Evaluation）を実装しており、フィールドアクセスが必要になるまでタプルの分解を回避する設計となっている。
  • dvalues と dnulls は単一のメモリチャンクとして確保されるため、個別に pfree しないこと。
  • ER_FLAG_DVALUES_VALID フラグの管理が分解状態の整合性を保証する唯一の手段であるため、他の関数でフィールドを直接操作する際は注意が必要。
