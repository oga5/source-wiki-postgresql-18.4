RelPathStr

定義

  • 種別: struct
  • 定義箇所: src/include/common/relpath.h:122

目的

リレーションファイル（テーブルやインデックスの実体）のパス文字列を、スタック上で安全に格納・保持するための固定長構造体です。

シグネチャまたは形状

typedef struct RelPathStr
{
	char		str[REL_PATH_STR_MAXLEN + 1];
} RelPathStr;

入力と出力

  • **入力**: 構造体自体はデータ保持用であり、直接的な入力引数は持ちません。
  • **出力**: str メンバを通じて、リレーションのフルパス文字列（ヌル終端）を提供します。

前提条件と不変条件

  • str メンバは REL_PATH_STR_MAXLEN + 1 バイトの固定長であり、パス文字列がこの長さを超えないことが保証される必要があります。
  • GetRelationPath 等の関数から返される際、スタックの有効範囲内で安全にアクセス可能です。

アルゴリズムまたは振る舞い

本構造体は、パス生成関数（GetRelationPath 等）の戻り値として利用されます。char* を直接返すのではなく本構造体を返すことで、呼び出し元がスタックの有効範囲を超えてポインタを参照するリスクをコンパイル時に抑制します。

呼び出し元と呼び出し先

  • **呼び出し元（利用箇所）**:
  • src/backend/access/transam/xlogutils.c
  • src/backend/backup/basebackup_incremental.c
  • src/backend/catalog/catalog.c
  • src/backend/catalog/storage.c
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/smgr/md.c
  • src/backend/storage/smgr/smgr.c
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • REL_PATH_STR_MAXLEN: パス文字列の最大長を定義する定数。
  • RelFileLocator: リレーションを特定するための識別子構造体。

失敗モードと境界条件

  • パス文字列が REL_PATH_STR_MAXLEN を超える場合、バッファオーバーフローの危険性があります。パス生成ロジック側でこの制限が守られていることが前提となります。

保守担当者向けメモ

  • 本構造体は src/common/relpath.c で生成されるパス文字列の受け皿として機能します。
  • パス生成ロジックに変更を加える際は、REL_PATH_STR_MAXLEN の算出根拠が維持されているか確認してください。
