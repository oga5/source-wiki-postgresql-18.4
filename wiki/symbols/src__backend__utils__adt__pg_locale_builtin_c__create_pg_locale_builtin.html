create_pg_locale_builtin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_builtin.c:122

目的

PostgreSQLの組み込み（builtin）ロケールプロバイダにおいて、指定された照合順序（Collation）OIDに基づき、ロケール情報を保持する pg_locale_t オブジェクトを初期化・生成する。

シグネチャまたは形状

pg_locale_t create_pg_locale_builtin(Oid collid, MemoryContext context)

入力と出力

  • **入力**:
  • collid: 照合順序のOID。DEFAULT_COLLATION_OID の場合はデータベースのデフォルトロケールを使用する。
  • context: ロケール構造体のメモリ割り当てに使用するメモリコンテキスト。
  • **出力**:
  • 初期化済みの pg_locale_t 構造体へのポインタ。

前提条件と不変条件

  • collid が DEFAULT_COLLATION_OID の場合、MyDatabaseId が有効である必要がある。
  • システムカタログ（pg_database または pg_collation）からロケール文字列が取得できない場合、エラーが発生する。
  • builtin_validate_locale によるロケール文字列の検証が成功しなければならない。

アルゴリズムまたは振る舞い

1. **ロケール文字列の取得**: collid が DEFAULT_COLLATION_OID ならば pg_database から、それ以外ならば pg_collation からロケール名（locstr）をシステムキャッシュ経由で取得する。 2. **検証**: builtin_validate_locale を呼び出し、現在のデータベースエンコーディングに対して locstr が妥当か確認する。 3. **構造体の割り当て**: context 上に pg_locale_struct をゼロ初期化して割り当てる。 4. **属性の設定**:

  • info.builtin.locale: locstr をコピーして設定。
  • info.builtin.casemap_full: locstr が "PG_UNICODE_FAST" と一致する場合に真。
  • provider: COLLPROVIDER_BUILTIN に固定。
  • deterministic: true に固定。
  • collate_is_c: true に固定。
  • ctype_is_c: locstr が "C" と一致する場合に真。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale.c 内のロケール管理関数（pg_locale.c:69, 1089, 1169 等）。
  • **呼び出し先**: SearchSysCache1, SysCacheGetAttrNotNull, ReleaseSysCache, TextDatumGetCString, builtin_validate_locale, MemoryContextAllocZero, MemoryContextStrdup。

関連データ構造

  • pg_locale_t (および struct pg_locale_struct): ロケール情報を保持する主要な構造体。
  • Oid: 照合順序やデータベースを識別する型。

失敗モードと境界条件

  • 指定された collid がシステムカタログに存在しない場合、elog(ERROR) により処理が中断される。
  • メモリ不足時には MemoryContextAllocZero 等がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は libc や ICU に依存しない組み込みロケール専用の初期化ロジックである。
  • PG_UNICODE_FAST ロケール名が指定された場合にのみ、高速なケースマッピングフラグが有効になる点に注意すること。
