WalRcvShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiverfuncs.c:54

目的

WALレシーバーの制御用共有メモリ領域を割り当て、初期化する。

シグネチャまたは形状

void WalRcvShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 WalRcv を介して共有メモリ領域を指し示す）

前提条件と不変条件

  • ShmemInitStruct を使用して共有メモリを確保するため、PostgreSQLの共有メモリ初期化フェーズで呼び出される必要がある。
  • 共有メモリが既に存在する場合（found が真）、初期化処理はスキップされる。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、"Wal Receiver Ctl" という名称で WalRcvShmemSize() 分の共有メモリを確保する。 2. 共有メモリが新規作成された場合（found が偽）のみ、以下の初期化を行う：

  • MemSet で領域をゼロクリアする。
  • walRcvState を WALRCV_STOPPED に設定する。
  • walRcvStoppedCV（条件変数）を初期化する。
  • mutex（スピンロック）を初期化する。
  • writtenUpto（アトミック変数）を 0 に初期化する。
  • procno を INVALID_PROC_NUMBER に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:332（PostgreSQLの共有メモリ初期化プロセスの一部として呼び出される）
  • 呼び出し先: ShmemInitStruct, MemSet, ConditionVariableInit, SpinLockInit, pg_atomic_init_u64

関連データ構造

  • WalRcvData: 共有メモリ上に配置されるWALレシーバーの制御構造体。
  • WalRcv: WalRcvData を指すグローバルポインタ。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct 内部でエラーが発生し、プロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数はシステム起動時に一度だけ実行されることを想定している。
  • 共有メモリのサイズ計算には WalRcvShmemSize() が使用されており、構造体の定義変更時にはこのサイズ計算ロジックとの整合性に注意すること。
