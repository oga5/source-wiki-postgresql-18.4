XLogRecStats

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogstats.h:21

目的

WAL（Write-Ahead Log）レコードの統計情報を集計するための基本単位として、レコード数、レコード本体のサイズ、およびフルページイメージ（FPI）のサイズを保持します。

シグネチャまたは形状

typedef struct XLogRecStats
{
    uint64      count;      /* レコード数 */
    uint64      rec_len;    /* レコード本体の合計サイズ */
    uint64      fpi_len;    /* FPIの合計サイズ */
} XLogRecStats;

入力と出力

本構造体はデータ保持用であり、直接的な入力・出力関数を持ちません。統計の更新は XLogRecStoreStats() 関数を介して行われ、XLogStats 構造体内の配列要素として集計値が加算されます。

前提条件と不変条件

  • 各フィールドは uint64 型であり、統計の累積値がオーバーフローしないことを前提としています。
  • XLogStats 構造体の一部として使用される際、rmgr_stats および record_stats 配列の要素として初期化される必要があります。

アルゴリズムまたは振る舞い

  • XLogRecGetLen() によって算出されたレコード長および FPI 長が、本構造体の rec_len および fpi_len に加算されます。
  • count は該当するレコードタイプまたはリソースマネージャが出現するたびにインクリメントされます。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/access/xlogstats.h 内で XLogStats 構造体のメンバとして定義されています。
  • 呼び出し先: 提供されたソース情報からは、本構造体自体が関数を呼び出す関係は確認できません。

関連データ構造

  • XLogStats: 本構造体を配列として保持し、全体の統計情報を管理する上位構造体。
  • RmgrId: リソースマネージャ単位の統計を分類するために使用される識別子。

失敗モードと境界条件

  • 統計対象となる WAL レコードの解析に失敗した場合、本構造体への加算処理は行われません。
  • XACT レコードなど、一部の特殊なレコードタイプでは xl_info のビットマスク操作が必要であり、誤った分類が行われる可能性があります。

保守担当者向けメモ

  • FRONTEND マクロの定義有無により、XLogStats 構造体内のフィールド構成が変化する可能性があるため、クライアントツール（pg_waldump 等）で使用する際は注意してください。
  • record_stats は [RM_MAX_ID + 1][MAX_XLINFO_TYPES] の二次元配列として定義されており、xl_info の上位 4 ビットをインデックスとして利用します。
