unaccent_lexize

定義

  • 種別: function
  • 定義箇所: contrib/unaccent/unaccent.c:378

目的

PostgreSQL のテキスト検索辞書として、入力文字列内のアクセント記号付き文字を、事前に構築された Trie 木のルールに基づいて置換する。

シグネチャまたは形状

Datum unaccent_lexize(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): TrieChar 型のルートノード（辞書データ）。
  • PG_GETARG_POINTER(1): 入力文字列のポインタ。
  • PG_GETARG_INT32(2): 入力文字列の長さ。
  • **出力**:
  • 置換が行われた場合は TSLexeme 構造体の配列（lexeme に置換後の文字列、flags に TSL_FILTER を設定）。
  • 置換が一度も行われなかった場合は NULL。

前提条件と不変条件

  • 入力文字列は有効なエンコーディングである必要がある。
  • rootTrie は initTrie によって適切に初期化されていること。
  • 置換が発生しない場合、関数は NULL を返す。

アルゴリズムまたは振る舞い

1. 入力文字列を先頭からスキャンし、findReplaceTo を用いて Trie 木から最長一致する置換ルールを検索する。 2. 置換ルールが見つかった場合、StringInfo バッファを初期化し、置換後の文字列を格納する。 3. 置換ルールが見つからない場合、現在の文字（マルチバイト文字を考慮）をそのままバッファに追加する。 4. 文字列の終端まで走査を繰り返し、最終的に置換後の文字列を TSLexeme として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のテキスト検索エンジン（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: findReplaceTo, initStringInfo, appendBinaryStringInfo, pg_mblen_range, palloc0。

関連データ構造

  • TrieChar: 置換ルールを保持する Trie 木のノード。
  • StringInfoData: 置換後の文字列を構築するための一時バッファ。
  • TSLexeme: テキスト検索エンジンに返却される結果構造体。

失敗モードと境界条件

  • 入力文字列が空、または長さが 0 の場合は処理を行わず NULL を返す。
  • 置換ルールが一つも適用されない場合、メモリ割り当てを行わず NULL を返す。

保守担当者向けメモ

  • Trie 木の検索はバイト単位で行われるため、マルチバイト文字の処理は pg_mblen_range に依存している。
  • buf.data が NULL の間は置換が発生していないことを示すフラグとして利用されており、最初の置換が発生した時点でバッファが初期化される効率的な設計となっている。
