DatumGetNumericCopy

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/numeric.h:66

目的

Datum 形式で渡された Numeric 値をデトースト（必要に応じてメモリ展開）し、そのコピーを生成して Numeric ポインタとして返します。

シグネチャまたは形状

static inline Numeric DatumGetNumericCopy(Datum X)

入力と出力

  • **入力**: Datum X （Numeric 型のデータを含む Datum）
  • **出力**: Numeric （デトーストされ、メモリ上にコピーされた数値データへのポインタ）

前提条件と不変条件

  • 入力 Datum X は有効な Numeric 型のデータである必要があります。
  • PG_DETOAST_DATUM_COPY マクロを使用するため、戻り値は呼び出し元で適切にメモリ管理（解放など）を行う必要があります。

アルゴリズムまたは振る舞い

1. 引数 X に対して PG_DETOAST_DATUM_COPY を実行します。 2. これにより、データが圧縮またはインライン化されている場合は展開され、メモリ上に独立したコピーが作成されます。 3. その結果を Numeric 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonb.c:2083
  • src/include/utils/numeric.h:79 （PG_GETARG_NUMERIC_COPY マクロ内）
  • **呼び出し先**:
  • PG_DETOAST_DATUM_COPY （PostgreSQLのコアマクロ）

関連データ構造

  • Numeric: NumericData 構造体へのポインタ。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 入力が有効な Numeric 形式でない場合、PG_DETOAST_DATUM_COPY の内部仕様に従いエラーが発生する可能性があります。
  • メモリ不足時には、palloc 関連の例外がスローされます。

保守担当者向けメモ

  • 本関数は、元のデータを変更せずに安全に操作するためのコピーを作成する際に使用してください。
  • 内部構造の詳細は src/backend/utils/adt/numeric.c に隠蔽されています。直接メンバにアクセスせず、提供されているインターフェースを介して操作してください。
