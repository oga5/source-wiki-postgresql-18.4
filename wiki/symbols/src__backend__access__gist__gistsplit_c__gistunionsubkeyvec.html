gistunionsubkeyvec

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistsplit.c:46

目的

GiSTインデックスのページ分割処理において、特定の分割対象から「don't-care（移動コストがゼロのタプル）」を除外した上で、残りのタプル群からユニオンキーを再計算する。

シグネチャまたは形状

static void gistunionsubkeyvec(GISTSTATE *giststate, IndexTuple *itvec, GistSplitUnion *gsvp)

入力と出力

  • **入力**:
  • giststate: GiSTインデックスの状態管理構造体。
  • itvec: インデックス内のタプル配列。
  • gsvp: 分割対象のタプルインデックスや属性情報、除外対象（dontcare）フラグを保持する GistSplitUnion 構造体。
  • **出力**: なし（副作用として gsvp 内のユニオンキーが更新される）。

前提条件と不変条件

  • gsvp->len は itvec から参照される有効な要素数であること。
  • gsvp->dontcare が存在する場合、そのフラグが立っているタプルはユニオン計算から除外される。

アルゴリズムまたは振る舞い

1. gsvp->len 分のメモリを確保し、一時的なタプル配列 cleanedItVec を作成する。 2. gsvp->entries を順に走査し、gsvp->dontcare フラグが立っていないタプルのみを cleanedItVec にコピーする。 3. gistMakeUnionItVec を呼び出し、抽出されたタプル群から gsvp->attr および gsvp->isnull に基づいてユニオンキーを計算・更新する。 4. 確保した cleanedItVec を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistsplit.c 内の gistSplitByKey 関数から複数回呼び出される。
  • **呼び出し先**: gistMakeUnionItVec（ユニオンキー計算用ユーティリティ）。

関連データ構造

  • GISTSTATE: GiSTインデックスの操作状態を保持。
  • IndexTuple: インデックス内の各レコード。
  • GistSplitUnion: 分割処理中の状態（対象タプル、属性、don't-careフラグ等）を管理。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性がある。
  • gsvp->len が0の場合、cleanedItVec は空となり、gistMakeUnionItVec は空の入力に対して処理を行う。

保守担当者向けメモ

  • 本関数は、don't-care タプルを動的に除外することで、インデックスの分割品質を最適化する役割を持つ。
  • gsvp->entries は1ベースのインデックスとして扱われているため、itvec へのアクセス時に -1 のオフセット調整が行われている点に注意すること。
