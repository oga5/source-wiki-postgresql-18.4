block_range_read_stream_cb

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/read_stream.c:161

目的

指定された範囲内のブロック番号を順次提供する、ReadStream 用のコールバック関数です。

シグネチャまたは形状

BlockNumber block_range_read_stream_cb(ReadStream *stream, void *callback_private_data, void *per_buffer_data);

入力と出力

  • **入力**:
  • stream: 呼び出し元の ReadStream オブジェクト。
  • callback_private_data: BlockRangeReadStreamPrivate 構造体へのポインタ（現在のブロック番号と終了条件を保持）。
  • per_buffer_data: 使用されない（NULL が渡されることを想定）。
  • **出力**:
  • 次に読み込むべき BlockNumber。範囲外に達した場合は InvalidBlockNumber を返す。

前提条件と不変条件

  • callback_private_data は BlockRangeReadStreamPrivate 型としてキャスト可能であること。
  • p->current_blocknum が p->last_exclusive に達するまで、呼び出しごとにインクリメントされる。

アルゴリズムまたは振る舞い

1. callback_private_data から現在のブロック範囲情報を取得する。 2. current_blocknum が last_exclusive 未満であるかを確認する。 3. 条件を満たす場合、現在のブロック番号を返し、内部のカウンタをインクリメントする。 4. 範囲外であれば InvalidBlockNumber を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の各モジュールから、ReadStream のコールバックとして参照されていることが確認されています。
  • contrib/amcheck/verify_heapam.c
  • contrib/pg_prewarm/pg_prewarm.c
  • contrib/pg_visibility/pg_visibility.c
  • src/backend/access/gist/gistvacuum.c
  • src/backend/access/nbtree/nbtree.c
  • **呼び出し先**: 外部関数を呼び出さない単純なアクセサです。

関連データ構造

  • ReadStream: 本関数が利用されるストリームの管理構造体。
  • BlockRangeReadStreamPrivate: callback_private_data として渡される、範囲管理用の内部構造体。

失敗モードと境界条件

  • current_blocknum が last_exclusive に到達した時点で、以降の呼び出しは常に InvalidBlockNumber を返します。

保守担当者向けメモ

  • 本関数は状態を持たない（per_buffer_data を使用しない）ため、バッチモードでの実行が安全であると各呼び出し元で判断されています。変更を加える際は、この前提が維持されるか確認してください。
