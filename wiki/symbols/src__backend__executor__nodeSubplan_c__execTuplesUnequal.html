execTuplesUnequal

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:657

目的

2つの TupleTableSlot が指定された列において等しくないかどうかを判定します。主にサブプラン実行時のハッシュテーブル照合において、不一致を早期に検出するために使用されます。

シグネチャまたは形状

bool execTuplesUnequal(TupleTableSlot *slot1,
                       TupleTableSlot *slot2,
                       int numCols,
                       AttrNumber *matchColIdx,
                       FmgrInfo *eqfunctions,
                       const Oid *collations,
                       MemoryContext evalContext)

入力と出力

  • **入力**:
  • slot1, slot2: 比較対象のタプルスロット。
  • numCols: 比較する列数。
  • matchColIdx: 比較対象の属性番号配列。
  • eqfunctions: 各列に対応する型固有の等価比較関数。
  • collations: 各列の照合順序（Collation）。
  • evalContext: 比較処理に使用する一時メモリコンテキスト。
  • **出力**:
  • bool: 少なくとも1つの列で値が一致しない（かつ両者がNULLでない）場合に true を返します。すべて一致する場合、またはNULLが含まれる場合は false を返します。

前提条件と不変条件

  • evalContext は呼び出しのたびに MemoryContextReset されます。
  • 比較処理は、指定された evalContext に切り替えて実行されます。

アルゴリズムまたは振る舞い

1. evalContext をリセットし、現在のメモリコンテキストを切り替えます。 2. numCols で指定された列を、最後尾（ソートキーの重要度が低い順）から先頭に向かって逆順に走査します。 3. 各列について slot_getattr を呼び出し、値を取得します。 4. いずれかのスロットで値がNULLの場合、その列の比較はスキップされます（NULLは等価性の証明を妨げるため）。 5. 両方の値が非NULLの場合、eqfunctions を用いて比較します。 6. 比較結果が false（等しくない）であれば、直ちにループを中断し true を返します。 7. すべての列を走査しても不一致が見つからなければ false を返します。 8. 最後に元のメモリコンテキストへ復帰します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSubplan.c 内のハッシュテーブル照合ロジック（hashtable->tableslot との比較など）から呼び出されます。
  • **呼び出し先**: slot_getattr、FunctionCall2Coll、MemoryContextReset、MemoryContextSwitchTo。

関連データ構造

  • TupleTableSlot: タプルを保持する実行時の構造体。
  • FmgrInfo: 関数呼び出し情報を保持する構造体。

失敗モードと境界条件

  • いずれかのスロットがNULLを含む場合、その列の比較はスキップされるため、NULLを含むタプル同士の比較では「等しくない」という判定は行われません。
  • numCols が0の場合、ループは実行されず false が返されます。

保守担当者向けメモ

  • 比較順序が最後尾からである理由は、ソート済み入力において不一致が早期に発見される可能性が高いためです。この最適化は、比較対象の列の特性に依存します。
