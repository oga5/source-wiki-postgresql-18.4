mul_size

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:214

目的

2つのサイズ値の乗算を行い、結果が整数オーバーフローを起こしていないか、およびシステムが許容する安全な上限値（SIZE_MAX / 2）を超えていないかを検証します。

シグネチャまたは形状

Size mul_size(Size s1, Size s2)

入力と出力

  • **入力**: s1 (Size), s2 (Size) - 乗算対象の2つのサイズ。
  • **出力**: 乗算結果（Size 型）。検証を通過した場合に返却されます。

前提条件と不変条件

  • s1 * s2 が SIZE_MAX を超えないこと。
  • s1 * s2 が SIZE_MAX / 2 を超えないこと（バックエンドとの整合性を保つための制約）。

アルゴリズムまたは振る舞い

1. pg_mul_size_overflow を使用して、s1 * s2 の計算結果がオーバーフローするかを確認します。 2. 計算結果が SIZE_MAX / 2 を超えているかを確認します。 3. 上記いずれかの条件に該当する場合、mul_size_error(s1, s2) を呼び出し、プロセスを終了させます。 4. 条件を満たす場合、計算結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_mul_size_overflow（オーバーフロー判定）、mul_size_error（エラー処理）。
  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c

    • ※上記はツールによる直接的な参照の発見に基づくものです。

関連データ構造

  • Size: メモリサイズを表す型（通常 size_t）。

失敗モードと境界条件

  • **オーバーフロー**: s1 * s2 が SIZE_MAX を超える場合、mul_size_error によりプロセスが終了します。
  • **上限超過**: 計算結果が SIZE_MAX / 2 を超える場合、物理メモリの有無に関わらずエラーとして扱われ、プロセスが終了します。

保守担当者向けメモ

  • 本関数は FRONTEND マクロが定義されている環境でコンパイルされる fe_memutils.c に含まれています。
  • SIZE_MAX / 2 という制約は、バックエンドのメモリ管理との互換性を維持するための設計上の決定事項です。
