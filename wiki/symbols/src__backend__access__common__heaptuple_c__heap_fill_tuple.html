heap_fill_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/heaptuple.c:400

目的

タプル記述子（TupleDesc）に基づき、提供された値配列からタプルのデータ領域を構築し、NULLビットマップおよび情報マスク（infomask）を更新する。

シグネチャまたは形状

void heap_fill_tuple(TupleDesc tupleDesc,
                     const Datum *values, const bool *isnull,
                     char *data, Size data_size,
                     uint16 *infomask, bits8 *bit)

入力と出力

  • tupleDesc: タプルの構造を定義する記述子。
  • values: 属性値の配列。
  • isnull: 各属性がNULLかどうかを示すフラグ配列。
  • data: 書き込み先のメモリ領域（ポインタ）。
  • data_size: 書き込むべきデータの期待サイズ。
  • infomask: タプルの状態を示すフラグ（更新される）。
  • bit: NULLビットマップの開始位置（NULLが存在しない場合は NULL を指定可能）。

前提条件と不変条件

  • infomask は、呼び出し前に HEAP_HASNULL, HEAP_HASVARWIDTH, HEAP_HASEXTERNAL フラグがクリアされる。
  • 処理終了時、書き込まれたデータの合計サイズは data_size と一致しなければならない（USE_ASSERT_CHECKING 有効時）。

アルゴリズムまたは振る舞い

1. bit が提供されている場合、NULLビットマップの初期位置とビットマスクをセットアップする。 2. infomask からNULL、可変長、外部データ関連のフラグをリセットする。 3. tupleDesc に含まれる各属性に対してループを実行する。 4. 各属性に対し fill_val を呼び出し、以下の処理を行う：

  • 属性値のメモリへの書き込み。
  • 必要に応じたアライメント調整。
  • NULLビットマップの更新。
  • infomask へのフラグ（NULL有無、可変長有無、外部データ有無）の反映。

    • 5. 最後に、書き込みに使用したデータ領域のサイズが期待値と一致するかを検証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_tuple.c
  • src/backend/access/common/heaptuple.c
  • src/backend/access/common/indextuple.c
  • src/backend/access/heap/heaptoast.c
  • **呼び出し先**:
  • fill_val（各属性の書き込み用ヘルパー）

関連データ構造

  • TupleDesc: タプルの属性定義を保持。
  • CompactAttribute: TupleDesc から取得される属性情報。
  • Datum: 属性値の汎用型。

失敗モードと境界条件

  • data_size が実際の書き込みサイズと一致しない場合、アサーションエラーが発生する。
  • values や isnull が NULL の場合、デフォルト値（NULL または true）として扱われる。

保守担当者向けメモ

  • 本関数はタプルの物理レイアウトを決定する重要なルーチンである。アライメントや圧縮処理は fill_val に委譲されているため、データ形式の変更時は fill_val の確認も必須である。
