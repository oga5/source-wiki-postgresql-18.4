pq_verify_peer_name_matches_certificate_name

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-common.c:86

目的

SSL/TLSサーバー証明書に含まれるDNS名（またはCommon Name）と、接続先ホスト名を照合し、証明書の妥当性を検証する。

シグネチャまたは形状

int pq_verify_peer_name_matches_certificate_name(PGconn *conn, const char *namedata, size_t namelen, char **store_name);

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体。
  • namedata: 証明書から抽出された名前の文字列（NULL終端の保証なし）。
  • namelen: namedata の長さ。
  • store_name: 処理した名前を格納するためのポインタ（呼び出し元で解放が必要）。
  • **出力**:
  • 成功時: 一致すれば 1、一致しなければ 0 を返す。
  • 失敗時: メモリ不足やホスト名未指定などのエラー発生時に -1 を返し、conn にエラー情報を格納する。

前提条件と不変条件

  • conn->connhost[conn->whichhost].host が有効なホスト名として設定されていること。
  • 証明書内の名前は、CVE-2009-4034対策のため、NULLバイトを含まないこと。

アルゴリズムまたは振る舞い

1. conn から接続先ホスト名を取得し、未指定の場合はエラーを返す。 2. namedata を malloc で複製し、NULL終端文字列を作成する。 3. 文字列内にNULLバイトが含まれているかを確認し、含まれる場合はエラーとして終了する。 4. pg_strcasecmp を用いてホスト名と大文字小文字を区別せずに完全一致比較を行う。 5. 一致しない場合、wildcard_certificate_match を呼び出し、ワイルドカードパターン（RFC 2818準拠）による照合を行う。 6. 最終的な結果を result に格納し、store_name に複製した文字列を渡して呼び出し元へ制御を戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure-openssl.c 内の関数から呼び出されることが確認されている。
  • **呼び出し先**:
  • libpq_append_conn_error (エラー発生時)
  • wildcard_certificate_match (ワイルドカード照合時)
  • malloc, free, memcpy, strlen, pg_strcasecmp (標準およびPostgreSQLユーティリティ関数)

関連データ構造

  • PGconn: 接続情報およびエラーメッセージ格納用。

失敗モードと境界条件

  • **ホスト名未指定**: conn 内のホスト名が空の場合、-1 を返す。
  • **メモリ不足**: malloc 失敗時に -1 を返す。
  • **不正な名前**: 証明書内の名前にNULLバイトが含まれる場合、セキュリティ上の理由で -1 を返す。

保守担当者向けメモ

  • 本関数は fe-secure-openssl.c 等のSSLバックエンド実装から呼び出される共通ユーティリティである。
  • ワイルドカードマッチングの規則はRFC 2818に準拠しており、ドットを跨ぐマッチングは許可されない。
  • store_name に格納されたメモリは、呼び出し元で適切に解放する必要がある。
