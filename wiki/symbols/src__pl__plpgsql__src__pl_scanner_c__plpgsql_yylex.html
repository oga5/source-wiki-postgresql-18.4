plpgsql_yylex

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_scanner.c:159

目的

PL/pgSQL の構文解析器（Bison）に対して、トークンを供給するメインの字句解析関数です。コアの字句解析結果に対し、PL/pgSQL 特有の識別子解決やキーワード判定を適用します。

シグネチャまたは形状

int plpgsql_yylex(YYSTYPE *yylvalp, YYLTYPE *yyllocp, yyscan_t yyscanner);

入力と出力

  • **入力**:
  • yylvalp: トークンの値を格納するポインタ。
  • yyllocp: トークンの位置情報を格納するポインタ。
  • yyscanner: スキャナの状態を保持する不透明なハンドル。
  • **出力**:
  • 戻り値: 識別されたトークンコード（int）。
  • 副作用: yylvalp および yyllocp への値の書き込み、および yyextra（スキャナ状態）の更新。

前提条件と不変条件

  • yyscanner が plpgsql_scanner_init によって適切に初期化されていること。
  • 識別子解決において、文の開始位置（AT_STMT_START）の判定には、直前に解析されたトークン（yyextra->plpgsql_yytoken）が使用されるため、解析順序の整合性が保たれていること。

アルゴリズムまたは振る舞い

1. internal_yylex を呼び出し、コアスキャナからトークンを取得する。 2. 取得したトークンが IDENT または PARAM の場合、後続のトークンを先読み（最大3つまで）し、ドット区切りの識別子（A.B.C や A.B）を構成できるか試行する。 3. 識別子が見つかった場合、plpgsql_parse_tripword や plpgsql_parse_dblword を用いて変数名（T_DATUM）か修飾名（T_CWORD）かを判定する。 4. 単一の識別子の場合、plpgsql_parse_word を用いて変数名との照合を行う。変数名でない場合は、非予約キーワードリスト（UnreservedPLKeywords）との照合を行い、一致すればキーワードトークンを返す。 5. 判定できなかった場合は T_WORD を返す。 6. 最終的なトークン情報（値、位置、長さ）を yylvalp 等に格納し、yyextra の状態を更新して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 構文解析器（pl_gram.y から生成されたコード）。
  • **呼び出し先**:
  • internal_yylex: トークンの取得。
  • plpgsql_parse_tripword, plpgsql_parse_dblword, plpgsql_parse_word: 識別子の解決。
  • ScanKeywordLookup, GetScanKeyword: キーワードの判定。
  • push_back_token: 先読みしたトークンの戻し。

関連データ構造

  • YYSTYPE, YYLTYPE: トークンの値と位置を保持する型。
  • TokenAuxData: internal_yylex が返すトークンの補助情報。
  • plpgsql_yy_extra_type: スキャナのグローバル状態（yyextra）。

失敗モードと境界条件

  • **先読みの限界**: A.B.C を超える構造はサポートされず、それ以降は通常のトークンとして処理される。
  • **変数名解決の抑制**: 文の開始位置において、代入演算子や [ が続く場合を除き、変数名解決をスキップすることで、予約語との衝突を回避するロジックが組み込まれている。
  • **プッシュバック**: 構文解析の都合でトークンを戻す際、最大 MAX_PUSHBACKS（4）の制限がある。

保守担当者向けメモ

  • 本関数は pl_gram.y と密接に結合しており、トークンコードの変更は両方のファイルに影響を与える。
  • plpgsql_parse_word の呼び出しは、変数名解決を行わない場合でも aux1.lval.word を初期化するために必須である。
