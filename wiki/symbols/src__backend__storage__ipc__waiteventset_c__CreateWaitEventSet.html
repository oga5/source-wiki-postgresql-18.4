CreateWaitEventSet

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:363

目的

複数のイベント（ソケット、ラッチ等）を同時に待機するための WaitEventSet 構造体を初期化し、OS 固有のイベント通知メカニズム（epoll, kqueue, poll, WaitForMultipleObjects）を準備する。

シグネチャまたは形状

WaitEventSet *CreateWaitEventSet(ResourceOwner resowner, int nevents)

入力と出力

  • **入力**:
  • resowner: リソース所有権を管理する ResourceOwner（NULL 可）。
  • nevents: 待機対象とするイベントの最大数。
  • **出力**:
  • 初期化済みの WaitEventSet 構造体へのポインタ。失敗時は elog(ERROR) によりプロセスが終了する。

前提条件と不変条件

  • nevents は正の整数であること。
  • resowner が指定された場合、ResourceOwnerEnlarge が呼び出され、リソース管理が有効化される。
  • メモリ割り当てには TopMemoryContext が使用される。

アルゴリズムまたは振る舞い

1. **メモリ計算**: WaitEventSet 本体、イベント配列、および OS 固有の待機用構造体（epoll_event, kevent, pollfd, HANDLE）に必要なメモリサイズを MAXALIGN を考慮して算出する。 2. **メモリ確保**: MemoryContextAllocZero を用いて一括確保し、各メンバへのポインタを割り当てる。 3. **リソース登録**: resowner が指定されていれば ResourceOwnerRememberWaitEventSet を呼び出し、ライフサイクル管理を登録する。 4. **OS 固有の初期化**:

  • epoll: epoll_create1 でファイル記述子を作成。
  • kqueue: kqueue でファイル記述子を作成し、FD_CLOEXEC を設定。
  • Win32: pgwin32_signal_event をハンドル配列の先頭に配置し、シグナル待機を保証する。

    • 5. **初期状態設定**: latch を NULL、exit_on_postmaster_death を false に初期化して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAppend.c
  • src/backend/libpq/pqcomm.c
  • src/backend/postmaster/postmaster.c
  • src/backend/postmaster/syslogger.c
  • src/backend/storage/ipc/latch.c
  • **呼び出し先**:
  • ResourceOwnerEnlarge, ResourceOwnerRememberWaitEventSet
  • MemoryContextAllocZero
  • epoll_create1, kqueue, fcntl（プラットフォーム依存）
  • AcquireExternalFD, ReleaseExternalFD

関連データ構造

  • WaitEventSet: 待機セットのメイン構造体。
  • WaitEvent: 監視対象イベントの情報を保持する構造体。
  • ResourceOwner: リソースの生存期間を管理するオブジェクト。

失敗モードと境界条件

  • **メモリ不足**: MemoryContextAllocZero が失敗した場合、PostgreSQL のメモリ管理規約に従いエラーとなる。
  • **FD 枯渇**: epoll_create1 や kqueue が失敗した場合、elog(ERROR) によりエラー終了する。
  • **リソース制限**: AcquireExternalFD が失敗した場合、システム全体の FD 制限に達していると判断されエラーとなる。

保守担当者向けメモ

  • 本関数はコンパイル時のフラグ（WAIT_USE_EPOLL 等）により実装が大きく分岐する。プラットフォーム固有の挙動を修正する際は、該当するプリプロセッサブロックを確認すること。
  • Windows 版では pgwin32_signal_event が配列の先頭（インデックス 0）に配置されるため、この順序を変更しないこと。
