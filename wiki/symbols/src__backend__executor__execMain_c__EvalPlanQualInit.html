EvalPlanQualInit

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execMain.c:2721

目的

EPQState（EvalPlanQual 実行状態）を初期化し、クエリ実行中に必要となるリソースの参照先を準備します。

シグネチャまたは形状

void EvalPlanQualInit(EPQState *epqstate, EState *parentestate,
                      Plan *subplan, List *auxrowmarks,
                      int epqParam, List *resultRelations)

入力と出力

  • **入力**:
  • epqstate: 初期化対象の EPQState 構造体へのポインタ。
  • parentestate: 親クエリの実行状態（EState）。
  • subplan: 実行対象のサブプラン。
  • auxrowmarks: 補助的な行マークのリスト。
  • epqParam: EPQ 実行用のパラメータ ID。
  • resultRelations: 更新対象のリレーションリスト。
  • **出力**: なし（epqstate を直接書き換える）。

前提条件と不変条件

  • parentestate が有効であること。
  • epqstate が指すメモリ領域が確保されていること。

アルゴリズムまたは振る舞い

1. parentestate からリレーションテーブルサイズを取得します。 2. epqstate に親の状態、パラメータ、結果リレーションを格納します。 3. palloc0 を使用して、リレーションテーブルサイズに応じた TupleTableSlot ポインタ配列（relsubs_slot）を動的に割り当てます。 4. サブプランと行マークを epqstate に保存します。 5. origslot、recheckestate、recheckplanstate などの実行関連フィールドを NULL に初期化し、EPQ 状態を「非アクティブ」に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeLockRows.c
  • src/backend/executor/nodeModifyTable.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • palloc0（メモリ割り当て）

関連データ構造

  • EPQState: EPQ 実行状態を保持する構造体。
  • EState: クエリ実行全体の実行状態。
  • Plan: 実行されるプランツリー。
  • TupleTableSlot: タプルを保持するためのスロット。

失敗モードと境界条件

  • メモリ不足時に palloc0 がエラーを発生させる可能性があります。
  • parentestate->es_range_table_size が 0 の場合、relsubs_slot に空のポインタ配列が割り当てられます。

保守担当者向けメモ

  • 本関数は EvalPlanQualBegin() よりも前に呼び出されることを前提としています。
  • relsubs_slot の割り当ては、EvalPlanQualBegin() のオーバーヘッドを避けるためにこの段階で行われます。
  • epqstate の各フィールドのライフサイクルについては src/include/nodes/execnodes.h の定義を参照してください。
