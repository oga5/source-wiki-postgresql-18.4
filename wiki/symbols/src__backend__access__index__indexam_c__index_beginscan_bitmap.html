index_beginscan_bitmap

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/index/indexam.c:288

目的

ビットマップスキャン（amgetbitmap を使用するスキャン）を開始するためのインデックススキャン記述子（IndexScanDesc）を初期化します。

シグネチャまたは形状

IndexScanDesc index_beginscan_bitmap(Relation indexRelation,
                                     Snapshot snapshot,
                                     IndexScanInstrumentation *instrument,
                                     int nkeys)

入力と出力

  • **入力**:
  • indexRelation: スキャン対象のインデックスリレーション。
  • snapshot: スキャンに使用する MVCC スナップショット。
  • instrument: 実行統計情報を記録するための構造体ポインタ。
  • nkeys: スキャンキーの数。
  • **出力**:
  • 初期化済みの IndexScanDesc 構造体へのポインタ。

前提条件と不変条件

  • snapshot は InvalidSnapshot であってはならない（Assert により検証）。

アルゴリズムまたは振る舞い

1. index_beginscan_internal を呼び出し、基本的なスキャン記述子を生成します。この際、nkeys、snapshot、およびビットマップスキャンであることを示すフラグが渡されます。 2. 生成された scan 構造体に対し、引数として受け取った snapshot および instrument を格納します。 3. 初期化された scan を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeBitmapIndexscan.c:324
  • **呼び出し先**:
  • index_beginscan_internal (同ファイル内)

関連データ構造

  • IndexScanDesc: インデックススキャンの状態を保持する構造体。
  • Relation: インデックスリレーションのメタデータ。
  • Snapshot: MVCC 制御のためのスナップショット。
  • IndexScanInstrumentation: スキャン実行時の統計情報。

失敗モードと境界条件

  • snapshot が InvalidSnapshot の場合、アサーション違反によりプロセスが停止します。
  • index_beginscan_internal 内部でメモリ確保やリソース取得に失敗した場合、そのエラーが伝播します。

保守担当者向けメモ

  • 本関数はビットマップスキャン専用の初期化ラッパーです。スキャン記述子の共通設定は index_beginscan_internal に依存しているため、共通の初期化ロジックを変更する際は注意してください。
