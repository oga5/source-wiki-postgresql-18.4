PGAsyncStatusType

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-int.h:213

目的

PostgreSQL クライアントライブラリ（libpq）におけるクエリ実行状態マシンのステータスを定義する。

シグネチャまたは形状

typedef enum {
    PGASYNC_IDLE,
    PGASYNC_BUSY,
    PGASYNC_READY,
    PGASYNC_READY_MORE,
    PGASYNC_COPY_IN,
    PGASYNC_COPY_OUT,
    PGASYNC_COPY_BOTH,
    PGASYNC_PIPELINE_IDLE
} PGAsyncStatusType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • PGconn 構造体の内部メンバ asyncStatus として保持され、接続ごとのクエリ進行状態を追跡する。

アルゴリズムまたは振る舞い

各列挙値は以下の状態を表す：

  • PGASYNC_IDLE: 待機中。
  • PGASYNC_BUSY: クエリ実行中。
  • PGASYNC_READY: クエリ完了、結果取得待ち。
  • PGASYNC_READY_MORE: クエリ完了、追加結果待ち。
  • PGASYNC_COPY_IN / COPY_OUT / COPY_BOTH: COPY データ転送中。
  • PGASYNC_PIPELINE_IDLE: パイプラインモードにおけるコマンド間の待機状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/libpq-int.h 内で PGconn 構造体のメンバとして参照されている。
  • 呼び出し先: 情報なし。

関連データ構造

  • PGconn: 本ステータスを保持するメインの接続構造体。

失敗モードと境界条件

  • 状態遷移の整合性は libpq 内部のプロトコル処理ロジック（fe-protocol3.c 等）に依存しており、本型定義自体には失敗モードは存在しない。

保守担当者向けメモ

  • 本シンボルは非公開 API であり、libpq-int.h を通じて libpq 内部でのみ使用される。
  • 状態の追加や変更を行う際は、fe-exec.c や fe-protocol3.c 等のステートマシン実装への影響を考慮すること。
