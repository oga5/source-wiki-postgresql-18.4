LWLock

定義

  • 種別: struct
  • 定義箇所: src/include/storage/lwlock.h:41

目的

共有メモリ上のデータ構造を保護するための、低オーバーヘッドな相互排他（Lightweight Lock）メカニズムを提供します。

シグネチャまたは形状

typedef struct LWLock
{
    uint16      tranche;        /* トランチID */
    pg_atomic_uint32 state;      /* 共有/排他ロックの状態 */
    proclist_head waiters;      /* 待機中のPGPROCリスト */
#ifdef LOCK_DEBUG
    pg_atomic_uint32 nwaiters;  /* 待機者数 */
    struct PGPROC *owner;       /* 最後に排他ロックを取得した所有者 */
#endif
} LWLock;

入力と出力

  • **入力**: ロック取得・解放操作の対象となる LWLock 構造体へのポインタ。
  • **出力**: アトミックな状態遷移の結果、および待機リストへのプロセス追加・削除。

前提条件と不変条件

  • LWLock は共有メモリ上に配置される必要があります。
  • state フィールドはアトミック操作（pg_atomic_uint32）によって管理され、競合状態を回避します。
  • LOCK_DEBUG が有効な場合、nwaiters と owner が追跡されますが、パフォーマンスに影響を与えます。

アルゴリズムまたは振る舞い

1. **取得**: LWLockAcquire は state に対してアトミックな compare-and-exchange を試みます。成功すれば即座にロックを獲得します。失敗した場合は、自身の PGPROC を waiters リストに追加し、OS レベルで待機します。 2. **解放**: LWLockRelease は state をアトミックに減算します。待機者が存在する場合、適切なプロセスを起床させます。 3. **変数待ち**: LWLockUpdateVar および LWLockWaitForVar を併用することで、ロックを保持したまま特定の共有変数の変更を待機するフローをサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_prewarm, contrib/pg_stat_statements, src/backend/access/nbtree, src/backend/access/transam/clog 等の各モジュールが、共有ハッシュテーブルや並列状態の保護のために本構造体を参照しています。
  • **呼び出し先**: port/atomics.h（アトミック操作）、storage/proclist_types.h（待機リスト管理）を利用します。

関連データ構造

  • LWLockPadded: キャッシュライン競合を避けるために LWLock をパディングした構造体。
  • PGPROC: 待機リスト（waiters）に含まれるプロセス情報。
  • MainLWLockArray: システム全体で共有される主要なロック配列。

失敗モードと境界条件

  • 競合が激しい場合、待機リストの管理によるオーバーヘッドが発生します。
  • LOCK_DEBUG 定義時は、デバッグ情報の更新によりパフォーマンスが低下します。
  • フロントエンドコードからはインクルードできません。

保守担当者向けメモ

  • LWLock は LWLockPadded を介して使用されることが一般的です。構造体のサイズを変更する際は、キャッシュライン境界（LWLOCK_PADDED_SIZE）への影響を考慮してください。
  • 待機プロセスリストの操作は proclist_types.h に依存しているため、リスト操作の変更時には関連性に注意してください。
