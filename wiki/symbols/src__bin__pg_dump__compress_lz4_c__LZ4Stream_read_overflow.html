LZ4Stream_read_overflow

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:408

目的

LZ4 解凍処理において、一度の解凍で得られたデータが要求サイズを超過した場合に保持される「オーバーフローバッファ」から、データを読み出し、バッファを更新する。

シグネチャまたは形状

static int LZ4Stream_read_overflow(LZ4State *state, void *ptr, int size, bool eol_flag)

入力と出力

  • state: LZ4State 構造体へのポインタ。オーバーフローバッファと現在の保持長を管理する。
  • ptr: 読み出したデータを格納する出力先バッファ。
  • size: 読み出しを要求する最大バイト数。
  • eol_flag: true の場合、改行文字 (\n) を検出した時点で読み出しを終了する。
  • **戻り値**: 実際に読み出されたバイト数。

前提条件と不変条件

  • state->overflowlen が 0 の場合、本関数は即座に 0 を返す。
  • 読み出し後、state->overflowlen は消費された分だけ減少し、残りのデータは state->overflowbuf の先頭へ memmove される。

アルゴリズムまたは振る舞い

1. state->overflowlen が 0 ならば 0 を返す。 2. 要求サイズ size と現在の保持量 state->overflowlen の小さい方を readlen とする。 3. eol_flag が有効かつ overflowbuf 内に改行文字が存在する場合、改行文字までの位置を readlen として再計算する。 4. memcpy により overflowbuf から ptr へデータをコピーする。 5. state->overflowlen を更新し、残存データがある場合は memmove でバッファを左詰めする。 6. 最終的な readlen を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_lz4.c 内のストリーム読み込み処理（LZ4Stream_read_internal 等）から呼び出される。
  • **呼び出し先**: 標準 C ライブラリの memchr, memcpy, memmove を使用。

関連データ構造

  • LZ4State: overflowbuf（バッファ本体）および overflowlen（現在の保持サイズ）を保持する構造体。

失敗モードと境界条件

  • overflowlen が 0 の場合、読み込みは行われない。
  • eol_flag が指定されていても改行文字が見つからない場合は、要求された readlen 分をすべて読み出す。
  • size が overflowlen より大きい場合、保持されている全データが返される。

保守担当者向けメモ

  • 本関数は、解凍後のデータが要求サイズを上回った際のバッファリングを担う。memmove を使用しているため、バッファサイズが極端に大きい場合はパフォーマンスに影響する可能性があるが、現在の用途では許容範囲内である。
