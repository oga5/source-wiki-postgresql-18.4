ginFillScanKey

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginscan.c:158

目的

GINインデックススキャンにおいて、個別のインデックス属性に対する検索キー（GinScanKey）を構築し、クエリ条件からGinScanEntryのリストを生成する。

シグネチャまたは形状

static void ginFillScanKey(
    GinScanOpaque so, 
    OffsetNumber attnum,
    StrategyNumber strategy, 
    int32 searchMode,
    Datum query, 
    uint32 nQueryValues,
    Datum *queryValues, 
    GinNullCategory *queryCategories,
    bool *partial_matches, 
    Pointer *extra_data
)

入力と出力

  • **入力**:
  • so: スキャン状態を保持する GinScanOpaque 構造体。
  • attnum, strategy, searchMode: 検索対象の属性番号、演算子戦略、検索モード。
  • query, nQueryValues, queryValues, queryCategories: クエリのデータ本体およびカテゴリ情報。
  • partial_matches, extra_data: 部分一致フラグおよび追加データ（オプション）。
  • **出力**: なし（so->keys 配列を更新し、GinScanKey を初期化する）。

前提条件と不変条件

  • so->keys 配列に十分な空きがあること。
  • ginstate が初期化済みであること。
  • nQueryValues 個の GinScanEntry を格納するために、key->scanEntry および key->entryRes に nQueryValues + 1 分のメモリが割り当てられる。

アルゴリズムまたは振る舞い

1. so->keys から新しい GinScanKey を取得し、基本情報を設定する。 2. ginInitConsistentFunction を呼び出し、一貫性チェック関数を初期化する。 3. nQueryValues 分のループを回し、ginFillScanEntry を用いて各検索キーに対応する GinScanEntry を生成・格納する。 4. 検索モードが GIN_SEARCH_MODE_INCLUDE_EMPTY または GIN_SEARCH_MODE_EVERYTHING の場合、ginScanKeyAddHiddenEntry を呼び出し、隠しエントリを追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginscan.c 内の ginNewScanKey 等から呼び出される（ソース内の直接的な参照箇所として line 378, line 461 が確認されている）。
  • **呼び出し先**:
  • ginInitConsistentFunction
  • ginFillScanEntry
  • ginScanKeyAddHiddenEntry

関連データ構造

  • GinScanOpaque: スキャン全体の状態を保持。
  • GinScanKey: 個別の属性に対する検索キー構造体。
  • GinScanEntry: 検索キー内の個別のエントリ。

失敗モードと境界条件

  • nQueryValues が 0 の場合でも、palloc により +1 された領域が確保される。
  • searchMode が GIN_SEARCH_MODE_ALL の場合、初期状態では excludeOnly が true に設定される。
  • 隠しエントリ用のスロットは 1 つのみ確保されるため、複数の隠しエントリを必要とする検索モードの追加には制限がある。

保守担当者向けメモ

  • key->scanEntry には「隠しエントリ」用に 1 つ余分なスロットが確保されている。
  • GIN_SEARCH_MODE_ALL の隠しエントリ処理は、この関数内ではなく後続の処理で行われる可能性があるため注意すること。
