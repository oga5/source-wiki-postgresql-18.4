getSpGistTupleDesc

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgutils.c:314

目的

SP-GiSTインデックスのキー列に対して、指定された型情報に基づいたタプル記述子（TupleDesc）を取得または生成します。

シグネチャまたは形状

TupleDesc getSpGistTupleDesc(Relation index, SpGistTypeDesc *keyType)

入力と出力

  • **入力**:
  • index: 対象となるインデックスのリレーション。
  • keyType: 目的の型情報（SpGistTypeDesc 構造体）。
  • **出力**:
  • TupleDesc: インデックスのキー列が keyType に適合するように調整されたタプル記述子。

前提条件と不変条件

  • keyType がインデックスの既存のキー列型と一致する場合、元のリレーション記述子がそのまま返されます。
  • 型が異なる場合、新しい TupleDesc がコピー作成され、キー列の属性（atttypid, attlen, attbyval, attalign, attstorage）が keyType の値で上書きされます。
  • 変更後の記述子に対しては、後続の列のキャッシュオフセットがリセットされ、populate_compact_attribute が呼び出されます。

アルゴリズムまたは振る舞い

1. インデックスの現在のキー列型と keyType->type を比較します。 2. 一致する場合は、RelationGetDescr(index) をそのまま返します。 3. 一致しない場合：

  • CreateTupleDescCopy で記述子を複製します。
  • 複製した記述子の spgKeyColumn 属性を keyType の値で更新します。
  • 圧縮メソッドを InvalidCompressionMethod、照合順序を InvalidOid に設定します。
  • spgFirstIncludeColumn 以降の列について、attcacheoff を -1 にリセットします。
  • populate_compact_attribute を呼び出し、属性の配置を再計算します。

    • 4. 生成した記述子を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgscan.c:333
  • src/backend/access/spgist/spgutils.c:364
  • **呼び出し先**:
  • RelationGetDescr
  • TupleDescAttr
  • CreateTupleDescCopy
  • TupleDescCompactAttr
  • populate_compact_attribute

関連データ構造

  • TupleDesc: PostgreSQLのタプル構造定義。
  • SpGistTypeDesc: SP-GiSTにおける型情報のラッパー構造体。
  • Relation: インデックスリレーション。

失敗モードと境界条件

  • メモリ不足時には CreateTupleDescCopy 等の内部関数によりエラーが発生する可能性があります。
  • spgKeyColumn が存在しない、または無効なインデックスリレーションが渡された場合の挙動は、TupleDescAttr の実装に依存します。

保守担当者向けメモ

  • 本関数は、インデックスの物理的な型と実行時の型が異なる場合に、動的に TupleDesc を適合させるために使用されます。
  • attcacheoff のリセット処理は、列の型変更に伴うオフセットの不整合を防ぐために必須です。変更を加える際は、populate_compact_attribute による再計算が正しく行われているか確認してください。
