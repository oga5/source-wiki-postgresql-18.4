TidRangeNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidrangescan.c:221

目的

TID（タプル識別子）の範囲指定に基づき、テーブルから次のタプルをスキャンして取得する。

シグネチャまたは形状

TupleTableSlot *TidRangeNext(TidRangeScanState *node)

入力と出力

  • **入力**: TidRangeScanState *node（スキャン状態を保持するノード構造体）
  • **出力**: TupleTableSlot *（取得したタプルを格納するスロット、またはスキャン終了時は空のスロット）

前提条件と不変条件

  • node->ss.ss_currentScanDesc は、スキャン開始前は NULL であるか、有効な TableScanDesc を指している必要がある。
  • node->trss_inScan フラグにより、初回実行時の範囲評価とスキャン開始処理が制御される。

アルゴリズムまたは振る舞い

1. **初回実行時の初期化**: trss_inScan が false の場合、TidRangeEval(node) を呼び出し、スキャン対象の TID 範囲（trss_mintid, trss_maxtid）を算出する。

  • TidRangeEval が false を返した場合はスキャンを終了する。
  • scandesc が未初期化であれば table_beginscan_tidrange を呼び出し、初期化済みであれば table_rescan_tidrange を呼び出して範囲を更新する。

    • 2. **タプル取得**: table_scan_getnextslot_tidrange を呼び出し、指定された方向（estate->es_direction）で次のタプルを取得する。 3. **終了処理**: タプルが取得できなかった場合、trss_inScan を false にリセットし、ExecClearTuple でスロットをクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン内のスキャン実行メソッドとして参照される（ExecScanAccessMtd 等）。
  • **呼び出し先**:
  • TidRangeEval: TID 範囲の算出。
  • table_beginscan_tidrange / table_rescan_tidrange: テーブルアクセス層によるスキャン開始・再開。
  • table_scan_getnextslot_tidrange: テーブルアクセス層によるタプル取得。
  • ExecClearTuple: スロットのクリア。

関連データ構造

  • TidRangeScanState: スキャンノードの状態を保持する構造体。
  • TableScanDesc: テーブルスキャンの記述子。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • TidRangeEval が範囲を空と判定した場合、即座に NULL を返す。
  • スキャン中にタプルが尽きた場合、trss_inScan をリセットし、空のスロットを返す。

保守担当者向けメモ

  • table_rescan_tidrange は TidRangeNext 内で呼び出されるまで待機する必要がある点に注意すること。
  • trss_inScan フラグの管理は、再スキャン（ReScan）時の範囲再計算を正しく行うために不可欠である。
