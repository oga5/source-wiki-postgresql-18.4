ChooseTablespace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fileset.c:185

目的

FileSet 内の特定のファイル名に対し、設定されたテーブルスペース群の中から配置先を決定する。

シグネチャまたは形状

static Oid ChooseTablespace(const FileSet *fileset, const char *name)

入力と出力

  • **入力**:
  • fileset: 配置先テーブルスペースのリストを保持する FileSet 構造体へのポインタ。
  • name: 配置先を決定するためのファイル名。
  • **出力**:
  • 選択されたテーブルスペースの Oid。

前提条件と不変条件

  • fileset->tablespaces に有効なテーブルスペースのリストが格納されていること。
  • fileset->ntablespaces が 0 より大きいこと（除算の除数となるため）。
  • 同一の name に対しては、常に同一のテーブルスペースが選択される。

アルゴリズムまたは振る舞い

1. name 文字列に対して hash_any を用いてハッシュ値を計算する。 2. 計算したハッシュ値を fileset->ntablespaces で剰余演算し、インデックスを算出する。 3. fileset->tablespaces 配列の該当インデックスにある Oid を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/fileset.c 内のファイル作成やパス解決を行う関数群（FileSetCreate や FileSetPath など）。
  • **呼び出し先**:
  • hash_any (ハッシュ計算用ユーティリティ)

関連データ構造

  • FileSet: テーブルスペースのリスト (tablespaces) とその数 (ntablespaces) を保持する構造体。

失敗モードと境界条件

  • fileset->ntablespaces が 0 の場合、ゼロ除算が発生する可能性がある。

保守担当者向けメモ

  • 本関数は static であり、fileset.c 内でのみ使用される。
  • テーブルスペースの分散アルゴリズムを変更する場合は、ハッシュ関数の選択や分散ロジックの整合性に注意すること。
