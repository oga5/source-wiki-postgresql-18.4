ExecInitTableFunctionResult

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execSRF.c:55

目的

テーブル関数（SRF: Set-Returning Function）の実行状態を初期化し、SetExprState 構造体を構築する。

シグネチャまたは形状

SetExprState *ExecInitTableFunctionResult(Expr *expr, ExprContext *econtext, PlanState *parent)

入力と出力

  • **入力**:
  • expr: 実行対象の式（通常は FuncExpr）。
  • econtext: 式評価用のコンテキスト。
  • parent: 親ノードの実行状態。
  • **出力**:
  • 初期化済みの SetExprState 構造体へのポインタ。

前提条件と不変条件

  • expr が FuncExpr である場合、その関数が集合を返すか否か（funcretset）に基づいて初期化が行われる。
  • FuncExpr 以外の場合、ExecInitExpr を用いて汎用的な式評価状態として初期化される。

アルゴリズムまたは振る舞い

1. SetExprState ノードを割り当て、funcReturnsSet を false に初期化する。 2. expr が FuncExpr 型か判定する。

  • **FuncExpr の場合**:
  • func->funcretset を state->funcReturnsSet に反映する。
  • ExecInitExprList で引数リストを初期化する。
  • init_sexpr を呼び出し、関数情報（OID等）のキャッシュやメモリコンテキストの設定を行う。
  • **それ以外の場合**:
  • ExecInitExpr を呼び出し、state->elidedFuncState に式評価状態を格納する。

    • 3. 構築した state を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeFunctionscan.c:349
  • **呼び出し先**:
  • makeNode
  • ExecInitExprList
  • init_sexpr
  • ExecInitExpr
  • IsA (マクロ)

関連データ構造

  • SetExprState: SRF の実行状態を保持する主要な構造体。
  • FuncExpr: 関数呼び出しを表すノード。
  • ExprContext: 式評価の実行コンテキスト。
  • PlanState: 実行プランのノード状態。

失敗モードと境界条件

  • expr が FuncExpr でない場合、集合を返す関数を内部に含む式はサポートされない（ExecInitExpr のパスへ進むため）。
  • init_sexpr 内部での関数情報取得失敗やメモリ不足時には、PostgreSQL の標準的なエラーハンドリング（elog / ereport）により中断される。

保守担当者向けメモ

  • FuncExpr 以外の式が渡された場合の挙動は、プランナによる定数畳み込みやインライン化の結果として発生する。このパスでは SRF の実行はサポートされない点に注意すること。
  • init_sexpr への引数として econtext->ecxt_per_query_memory が渡されており、関数情報のキャッシュはクエリ期間中保持される。
