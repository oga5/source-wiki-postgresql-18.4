unicode_to_utf8word

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c:62

目的

Unicode コードポイント（uint32）を、対応する UTF-8 エンコーディングのバイト列（uint32 に格納）へ変換します。

シグネチャまたは形状

static inline uint32 unicode_to_utf8word(uint32 c)

入力と出力

  • **入力**: c (Unicode コードポイント)
  • **出力**: UTF-8 エンコーディングされたバイト列を格納した uint32 値。

前提条件と不変条件

  • 入力 c は有効な Unicode コードポイントであることを前提とします。
  • 出力はビッグエンディアン形式でバイトが詰め込まれます（例: 4バイト文字は 0xF0... が最上位バイトになる）。

アルゴリズムまたは振る舞い

入力されたコードポイントの範囲に応じて、UTF-8 のマルチバイトシーケンスを構築します。 1. c <= 0x7F: 1バイト形式としてそのまま返します。 2. c <= 0x7FF: 2バイト形式に変換します。 3. c <= 0xFFFF: 3バイト形式に変換します。 4. その他（4バイト）: 4バイト形式に変換します。 各ステップで、ビットシフトとマスク処理を用いて UTF-8 のヘッダービット（0x80, 0xC0, 0xE0, 0xF0 等）を付与します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c:135（conv_18030_to_utf8 内での計算結果の変換）
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • なし。

失敗モードと境界条件

  • 入力が有効な Unicode 範囲を超えている場合の動作は定義されていません。
  • 戻り値の uint32 型に収まらない（5バイト以上を要する）コードポイントは考慮されていません。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを重視して実装されています。
  • UTF-8 の仕様変更や、より広範なコードポイント対応が必要な場合は、ビット演算ロジックの修正が必要です。
