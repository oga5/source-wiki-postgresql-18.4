transformCreateStmt

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_utilcmd.c:163

目的

CREATE TABLE（または CREATE FOREIGN TABLE）文の構文解析ツリーを、実行可能な一連のユーティリティコマンドリストへ変換する。

シグネチャまたは形状

List *transformCreateStmt(CreateStmt *stmt, const char *queryString)

入力と出力

  • **入力**:
  • stmt: 構文解析器から生成された CreateStmt ノード。
  • queryString: 元の SQL クエリ文字列。
  • **出力**: 実行すべきユーティリティコマンドの List。IF NOT EXISTS 指定時にテーブルが既に存在する場合は NIL を返す。

前提条件と不変条件

  • stmt->ofTypename と stmt->inhRelations は同時に真にはならない（文法レベルで保証）。
  • パーティションテーブルとして作成する場合、継承関係（inhRelations）とパーティション境界（partbound）の組み合わせが不正であればエラーとなる。

アルゴリズムまたは振る舞い

1. **初期化**: ParseState を作成し、CreateStmtContext を初期化して変換状態を保持する。 2. **名前空間の解決**: RangeVarGetAndCheckCreationNamespace を呼び出し、対象の名前空間の権限確認とロックを行う。 3. **存在確認**: IF NOT EXISTS が指定されている場合、対象が既に存在すれば拡張メンバーシップを確認した上で NOTICE を発行し終了する。 4. **要素の変換**: tableElts を走査し、ColumnDef、Constraint、TableLikeClause をそれぞれの変換関数（transformColumnDefinition 等）で処理する。 5. **制約の伝播**: NOT NULL 制約を ColumnDef に反映させる。 6. **インデックス生成**: transformIndexConstraints を呼び出し、主キーやユニーク制約等のインデックス定義を生成する。 7. **後続処理**: LIKE 句の処理や外部キー制約の処理へ続く（ソース抜粋範囲外）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c の ProcessUtility 等から呼び出される。
  • **呼び出し先**: make_parsestate, RangeVarGetAndCheckCreationNamespace, transformColumnDefinition, transformTableConstraint, transformTableLikeClause, transformIndexConstraints 等。

関連データ構造

  • CreateStmt: 入力となる構文解析ノード。
  • CreateStmtContext: 変換処理中に状態を保持する内部構造体。
  • ParseState: 解析状態を保持する構造体。

失敗モードと境界条件

  • 対象の名前空間に対する権限がない場合、エラーが発生する。
  • 継承とパーティションの組み合わせが不正な場合、エラーが発生する。
  • 未知のノードタイプが tableElts に含まれる場合、elog(ERROR) により中断される。

保守担当者向けメモ

  • 本関数は CREATE TABLE の変換の入り口であり、CreateStmtContext を介して状態を管理する。
  • alist（後処理コマンドリスト）の順序は歴史的な経緯があり、transformIndexConstraints の前後でリストを分離・統合する処理が含まれている点に注意すること。
