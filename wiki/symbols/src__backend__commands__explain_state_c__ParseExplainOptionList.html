ParseExplainOptionList

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_state.c:76

目的

EXPLAIN コマンドのオプションリスト（DefElem のリスト）を走査し、ExplainState 構造体の各フィールドを更新・検証します。

シグネチャまたは形状

void ParseExplainOptionList(ExplainState *es, List *options, ParseState *pstate)

入力と出力

  • **入力**:
  • es: 更新対象の ExplainState 構造体へのポインタ。
  • options: 解析対象の DefElem リスト。
  • pstate: エラー報告に使用する ParseState 構造体へのポインタ。
  • **出力**: なし（es を直接変更する）。

前提条件と不変条件

  • options リスト内の各要素は DefElem である必要があります。
  • ANALYZE オプションと依存関係にあるオプション（WAL, TIMING, SERIALIZE, SUMMARY）は、ANALYZE が有効でない場合に指定されるとエラーとなります。
  • GENERIC_PLAN と ANALYZE は同時に指定できません。

アルゴリズムまたは振る舞い

1. options リストを反復処理し、組み込みオプション（analyze, verbose, costs, buffers, wal, settings, generic_plan, timing, summary, memory, serialize, format）を ExplainState に反映します。 2. 未知のオプションについては ApplyExtensionExplainOption を呼び出し、拡張機能による処理を試みます。いずれも該当しない場合はエラーを報告します。 3. WAL, TIMING, SERIALIZE が指定された場合、ANALYZE との整合性をチェックします。 4. timing, buffers, summary が明示的に設定されていない場合、ANALYZE の値に基づいたデフォルト値を適用します。 5. GENERIC_PLAN と ANALYZE の競合をチェックします。 6. 最後に explain_validate_options_hook が設定されていれば、それを呼び出してプラグイン固有の検証を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c 内の EXPLAIN コマンド実行処理から呼び出されます。
  • **呼び出し先**: defGetBoolean, defGetString, ApplyExtensionExplainOption, ereport, parser_errposition, explain_validate_options_hook。

関連データ構造

  • ExplainState: EXPLAIN の実行状態を保持する構造体。
  • DefElem: EXPLAIN オプションの定義を保持する構造体。
  • ParseState: パーサーの状態を保持する構造体。

失敗モードと境界条件

  • 未知のオプションが指定された場合、ERRCODE_SYNTAX_ERROR でエラーとなります。
  • オプションに不正な値が指定された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなります。
  • 依存関係（ANALYZE 必須など）が満たされない場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなります。

保守担当者向けメモ

  • 新しい EXPLAIN オプションを追加する場合は、本関数の foreach ループ内に条件分岐を追加してください。
  • 拡張機能によるオプション追加は ApplyExtensionExplainOption 経由で行われるため、コアロジックを汚染せずに拡張可能です。
