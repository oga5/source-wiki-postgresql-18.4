XLogRecordSaveFPWs

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/pg_waldump.c:489

目的

WAL レコードに含まれるフルページイメージ（FPW）を抽出し、指定されたディレクトリに個別のファイルとして保存する。

シグネチャまたは形状

static void XLogRecordSaveFPWs(XLogReaderState *record, const char *savepath)

入力と出力

  • **入力**:
  • record: 解析済みの WAL レコード状態を保持する XLogReaderState 構造体へのポインタ。
  • savepath: FPW ファイルを保存するディレクトリパスを示す文字列。
  • **出力**: なし（成功時はファイルシステムへの書き出し、失敗時は pg_fatal によるプロセス終了）。

前提条件と不変条件

  • record は有効な XLogReaderState を指していること。
  • savepath は書き込み可能なディレクトリであること。
  • BLCKSZ（ページサイズ）は固定であると仮定される。

アルゴリズムまたは振る舞い

1. レコード内の全ブロック ID を走査する。 2. 各ブロックについて、以下の条件を満たすか確認する:

  • ブロック参照が存在する (XLogRecHasBlockRef)。
  • ブロックイメージが含まれている (XLogRecHasBlockImage)。

    • 3. RestoreBlockImage を用いてイメージを復元する。 4. XLogRecGetBlockTagExtended を用いて、対象ブロックの識別子（リレーション、フォーク、ブロック番号）を取得する。 5. タイムライン、LSN、リレーション情報、ブロック番号、フォーク名を組み合わせたファイル名を生成する。 6. 生成したファイル名でバイナリ書き込みモード (PG_BINARY_W) でファイルを開き、ページデータを書き出して閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_waldump/pg_waldump.c:1296（メインの読み込みループ内）。
  • **呼び出し先**:
  • XLogRecMaxBlockId, XLogRecHasBlockRef, XLogRecHasBlockImage, RestoreBlockImage, XLogRecGetBlockTagExtended（いずれも XLogReader インターフェース）。
  • pg_fatal, fopen, fwrite, fclose, snprintf, sprintf（標準ライブラリおよび PostgreSQL ユーティリティ）。

関連データ構造

  • XLogReaderState: WAL レコードの読み込み状態を管理する構造体。
  • RelFileLocator: リレーションの識別情報（spcOid, dbOid, relNumber）。
  • PGAlignedBlock: ページデータを保持するためのアライメントされたバッファ。

失敗モードと境界条件

  • **ファイル操作失敗**: 保存先ディレクトリへの書き込み権限がない、またはディスクフル等の理由で fopen, fwrite, fclose が失敗した場合、pg_fatal により終了する。
  • **無効なフォーク番号**: fork が MAX_FORKNUM を超える場合、pg_fatal により終了する。
  • **復元失敗**: RestoreBlockImage が失敗した場合、エラーメッセージを出力して終了する。

保守担当者向けメモ

  • 本関数は pg_waldump のフロントエンド処理の一部であり、BLCKSZ に依存している。
  • ファイル命名規則は TLI-LSN-LSN.spcOid.dbOid.relNumber.blk.fork の形式である。
  • forkNames 配列を使用してフォーク名を文字列化しているため、フォーク定義の変更時には注意が必要。
