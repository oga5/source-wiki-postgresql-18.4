PLy_notice

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:295

目的

Python スクリプトから plpy.notice() を呼び出した際に、PostgreSQL の NOTICE レベルのメッセージをログ出力するためのインターフェースを提供します。

シグネチャまたは形状

static PyObject *PLy_notice(PyObject *self, PyObject *args, PyObject *kw);

入力と出力

  • **入力**:
  • self: Python モジュールオブジェクト。
  • args: Python から渡された可変長引数（メッセージ内容など）。
  • kw: Python から渡されたキーワード引数。
  • **出力**:
  • PLy_output 関数の戻り値（通常は Py_None またはエラー発生時の NULL）。

前提条件と不変条件

  • 本関数は Python の C API 規約に従うメソッドとして定義されています。
  • 内部で呼び出される PLy_output が、PostgreSQL の ereport マクロを介してログ出力を行うため、PostgreSQL のエラー処理コンテキスト内で実行される必要があります。

アルゴリズムまたは振る舞い

1. Python から受け取った引数 args および kw をそのまま PLy_output 関数に渡します。 2. 第1引数として NOTICE（PostgreSQL のログレベル定数）を指定します。 3. PLy_output が Python の引数を解析し、PostgreSQL の ereport(NOTICE, ...) を実行してメッセージを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: Python インタプリタ（plpy.notice() メソッド経由）。
  • **呼び出し先**: PLy_output（src/pl/plpython/plpy_elog.c に定義）。
  • **登録**: src/pl/plpython/plpy_plpymodule.c 内の PLy_methods テーブルに {"notice", ...} として登録されています。

関連データ構造

  • PyObject: Python のオブジェクト型。
  • NOTICE: PostgreSQL のログレベルを示す列挙値。

失敗モードと境界条件

  • PLy_output 内部で引数の解析に失敗した場合、または PostgreSQL 側でエラーが発生した場合は、Python の例外として処理が戻ります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のログ出力ロジックは PLy_output に集約されています。ログ出力の挙動を変更する場合は PLy_output を参照してください。
  • PLy_methods テーブルへの登録時に METH_VARARGS | METH_KEYWORDS が指定されているため、引数の受け渡しには注意が必要です。
