rank_context

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/windowfuncs.c:24

目的

ウィンドウ関数（RANK、DENSE_RANK、PERCENT_RANK、CUME_DIST 等）の計算実行中、パーティション単位で現在の順位状態を保持するために使用されます。

シグネチャまたは形状

typedef struct rank_context
{
    int64 rank; /* 現在のランク */
} rank_context;

入力と出力

  • **入力**: WinGetPartitionLocalMemory を介して、実行エンジンからパーティションごとのメモリ領域として割り当てられます。
  • **出力**: 構造体内の rank メンバが更新され、後続の行処理における順位判定の基準として利用されます。

前提条件と不変条件

  • ウィンドウ関数の実行ライフサイクルにおいて、パーティションが切り替わるたびに WinGetPartitionLocalMemory を通じて初期化される必要があります。
  • rank 値は、各パーティションの先頭行から順次計算される過程で保持されます。

アルゴリズムまたは振る舞い

1. ウィンドウ関数実行時に WinGetPartitionLocalMemory を呼び出し、rank_context 構造体のメモリを確保します。 2. 各行の処理において、rank_up などのユーティリティ関数が現在の行と前の行を比較し、順位を更新すべきか判定します。 3. 判定結果に基づき、rank_context->rank の値をインクリメントまたは維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/windowfuncs.c 内の各ウィンドウ関数実装（window_rank、window_dense_rank、window_percent_rank、window_cume_dist 等）。
  • **呼び出し先**: windowapi.h で定義された WinGetPartitionLocalMemory を呼び出し、メモリを確保します。

関連データ構造

  • windowapi.h: ウィンドウ関数実行用API。
  • ntile_context: 同様にパーティションごとの状態を保持する構造体。

失敗モードと境界条件

  • パーティションの境界で状態がリセットされない場合、誤った順位が算出されます。
  • メモリ確保失敗時の挙動は WinGetPartitionLocalMemory の仕様に依存します。

保守担当者向けメモ

  • 本構造体は非常にシンプルですが、ウィンドウ関数の状態管理の基盤です。rank の型が int64 であるため、非常に大きなパーティションでの順位計算にも対応可能です。
  • 順位の更新ロジックを変更する場合は、この構造体を参照している各ウィンドウ関数および rank_up 関数のピア判定ロジックとの整合性に注意してください。
