dsm_op

定義

  • 種別: enum
  • 定義箇所: src/include/storage/dsm_impl.h:61

目的

動的共有メモリ（DSM）セグメントに対して実行可能な操作の種類を定義する。

シグネチャまたは形状

typedef enum
{
    DSM_OP_CREATE,
    DSM_OP_ATTACH,
    DSM_OP_DETACH,
    DSM_OP_DESTROY,
} dsm_op;

入力と出力

  • 本シンボル自体は列挙値であり、dsm_impl_op() 関数等の引数として渡されることで、実行すべき操作を指示する。

前提条件と不変条件

  • dsm_impl_op() を介してプラットフォーム固有の実装（POSIX, System V, Windows, mmap）へ渡される。
  • 各操作は、対応するプラットフォーム固有の関数（例: dsm_impl_posix）において、共有メモリのライフサイクル管理（作成、アタッチ、デタッチ、破棄）を制御するために使用される。

アルゴリズムまたは振る舞い

  • DSM_OP_CREATE: 新規共有メモリセグメントの作成を指示する。
  • DSM_OP_ATTACH: 既存の共有メモリセグメントへのアタッチを指示する。
  • DSM_OP_DETACH: 共有メモリセグメントからのデタッチを指示する。
  • DSM_OP_DESTROY: 共有メモリセグメントの破棄を指示する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/dsm_impl.c 内の dsm_impl_op() 関数が、上位層からの要求に応じて本列挙値を受け取り、各プラットフォーム固有の実装へディスパッチする。
  • 呼び出し先: 本シンボル自体は列挙型であり、直接的な呼び出し先は存在しない。

関連データ構造

  • dsm_handle: 共有メモリセグメントを識別するためのハンドル。
  • dsm_impl_op(): 本列挙型を引数として受け取るメインエントリーポイント。

失敗モードと境界条件

  • 不正な dsm_op 値が渡された場合の挙動については、本ソース excerpt には記載がない。

保守担当者向けメモ

  • 新しいプラットフォームを追加する際は、dsm_impl.c 内の各プラットフォーム固有関数（dsm_impl_posix 等）がこれらすべての列挙値を適切に処理していることを確認すること。
