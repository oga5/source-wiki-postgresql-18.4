defaultGetLocalPQExpBuffer

定義

  • 種別: function
  • 定義箇所: src/fe_utils/string_utils.c:41

目的

識別子のクォート処理等で使用される、再利用可能な静的 PQExpBuffer を提供します。

シグネチャまたは形状

static PQExpBuffer defaultGetLocalPQExpBuffer(void)

入力と出力

  • 入力: なし
  • 出力: PQExpBuffer 型のポインタ。関数内で保持されている静的バッファを返します。

前提条件と不変条件

  • 戻り値のバッファは静的領域に確保されるため、呼び出し元は戻り値を保持し続けることはできません。
  • 次回の呼び出しで内容が上書きされる可能性があるため、呼び出し直後に使用を完了するか、必要な場合は内容をコピーする必要があります。

アルゴリズムまたは振る舞い

1. 静的変数 id_return が初期化済みか確認します。 2. 初期化済み（id_return != NULL）の場合、resetPQExpBuffer() を呼び出してバッファの内容をクリアします。 3. 未初期化の場合、createPQExpBuffer() を呼び出して新しいバッファを生成し、id_return に格納します。 4. 最後に id_return を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/fe_utils/string_utils.c 内の関数ポインタ getLocalPQExpBuffer により参照されています。
  • 呼び出し先: resetPQExpBuffer および createPQExpBuffer を呼び出します。

関連データ構造

  • PQExpBuffer: PostgreSQLの文字列バッファ管理構造体。

失敗モードと境界条件

  • メモリ不足時: createPQExpBuffer() が失敗した場合、その内部実装に依存します（通常はエラー終了します）。
  • スレッド安全性: 本関数は静的変数を使用しているため、マルチスレッド環境では安全ではありません。

保守担当者向けメモ

  • 本関数は fmtId() などの識別子処理関数から利用される内部ユーティリティです。
  • スレッドセーフではないため、マルチスレッド環境での利用は避けてください。
