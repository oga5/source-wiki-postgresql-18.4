bbsink_throttle_archive_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_throttle.c:109

目的

バックアップのアーカイブコンテンツ転送時に、設定されたレート制限に従ってスロットリング（待機）を適用し、データを後続のシンクへ転送する。

シグネチャまたは形状

static void bbsink_throttle_archive_contents(bbsink *sink, size_t len)

入力と出力

  • sink: スロットリング機能を持つ bbsink オブジェクトへのポインタ。
  • len: 転送対象となるデータのバイト数。
  • 戻り値: なし。

前提条件と不変条件

  • sink は bbsink_throttle 型としてキャスト可能であること。
  • 本関数は bbsink インターフェースの一部として呼び出されること。

アルゴリズムまたは振る舞い

1. throttle() 関数を呼び出し、現在の転送量に基づいて必要であれば WaitLatch を用いた待機処理を実行する。 2. 待機完了後、bbsink_forward_archive_contents() を呼び出し、データをパイプライン内の次のシンクへ転送する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink インターフェースのメソッドテーブル（archive_contents）を通じて呼び出される。
  • 呼び出し先:
  • throttle(): 転送レートの計算と待機処理。
  • bbsink_forward_archive_contents(): 次のシンクへのデータ転送。

関連データ構造

  • bbsink: バックアップ処理のパイプラインを構成する基本構造体。
  • bbsink_throttle: bbsink を継承し、スロットリング状態を保持する構造体。

失敗モードと境界条件

  • 転送レートが設定されていない場合や、待機が不要な場合は即座に転送処理へ移行する。
  • 待機中、バックアッププロセスが停止要求（シグナル）を受け取った場合は CHECK_FOR_INTERRUPTS() により適切に割り込み処理が行われる。

保守担当者向けメモ

  • 本関数は「デコレータ」パターンで実装されており、シンクチェーンの間に挿入される。
  • 整数演算でスロットリングを行っているため、端数は次回の呼び出しに持ち越される。
