TclExceptionNameMap

定義

  • 種別: struct
  • 定義箇所: src/pl/tcl/pltcl.c:258

目的

Tcl の例外名（ラベル）と、それに対応する PostgreSQL の SQLSTATE エラーコードを関連付けるためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	const char *label;
	int			sqlerrstate;
} TclExceptionNameMap;

入力と出力

  • label: Tcl の例外名を表す文字列へのポインタ。
  • sqlerrstate: 対応する PostgreSQL のエラーコード（errcodes.h 等で定義される定数）。

前提条件と不変条件

  • label は NULL 終端文字列である必要があります。
  • この構造体は、主に静的な配列（exception_name_map）の要素として使用されることを想定しています。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用であり、実行時のロジックは持ちません。この構造体を用いた配列を検索することで、Tcl 側で発生したエラーを PostgreSQL のエラーハンドリングシステムが理解できる形式へ変換する際のルックアップテーブルとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/tcl/pltcl.c 内の exception_name_map 配列定義（264行目）で参照されています。
  • 呼び出し先: 本構造体はデータ型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • exception_name_map: 本構造体のインスタンスを格納する静的配列。
  • pltclerrcodes.h: SQLSTATE 定義が含まれるヘッダーファイル。

失敗モードと境界条件

  • 構造体定義自体に失敗モードはありません。
  • 検索時に一致する label が存在しない場合、エラーコードの変換が失敗する可能性があります。

保守担当者向けメモ

  • 新しい Tcl 例外を PostgreSQL のエラーコードにマッピングする必要がある場合は、exception_name_map 配列に本構造体のインスタンスを追加してください。
