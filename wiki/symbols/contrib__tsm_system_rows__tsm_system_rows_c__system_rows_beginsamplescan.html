system_rows_beginsamplescan

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_rows/tsm_system_rows.c:173

目的

SYSTEM_ROWS テーブルサンプルメソッドにおけるスキャン開始時の初期化処理を行います。パラメータの検証と、スキャン状態の初期設定を担います。

シグネチャまたは形状

static void system_rows_beginsamplescan(SampleScanState *node, Datum *params, int nparams, uint32 seed)

入力と出力

  • **入力**:
  • node: スキャン状態を保持する SampleScanState 構造体へのポインタ。
  • params: サンプルサイズ（行数）を含む Datum 配列。
  • nparams: パラメータ数。
  • seed: 乱数生成用のシード値。
  • **出力**: なし（副作用として node および node->tsm_state を更新）。

前提条件と不変条件

  • node->tsm_state には SystemRowsSamplerData 型の構造体が割り当てられている必要があります。
  • params[0] には、サンプリング対象とする行数（int64）が格納されている必要があります。

アルゴリズムまたは振る舞い

1. params[0] からサンプル行数 ntuples を取得し、負の値であればエラーを報告します。 2. node->tsm_state を SystemRowsSamplerData としてキャストし、以下の値を設定します。

  • seed: 引数のシード値を保存。
  • ntuples: 取得したサンプル行数を保存。
  • lt: InvalidOffsetNumber に初期化。
  • doneblocks: 0 に初期化。

    • 3. node->use_pagemode を true に設定し、ページ単位の可視性チェックを強制します。 4. nblocks、firstblock、step は、再スキャン時の一貫性を維持するため、この関数内では変更しません。

呼び出し元と呼び出し先

  • **呼び出し元**: tsm_system_rows_handler 内で TsmRoutine 構造体に登録され、クエリ実行エンジンから呼び出されます。
  • **呼び出し先**: ereport（エラー発生時）。

関連データ構造

  • SampleScanState: PostgreSQL のスキャン実行状態を保持する構造体。
  • SystemRowsSamplerData: SYSTEM_ROWS メソッド固有のスキャン状態を保持する構造体。

失敗モードと境界条件

  • ntuples < 0: ERRCODE_INVALID_TABLESAMPLE_ARGUMENT を発生させ、エラー終了します。

保守担当者向けメモ

  • 本関数は SYSTEM_ROWS の初期化フェーズを担当します。SystemRowsSamplerData のフィールドのうち、ブロック走査に関連する動的な値（nblocks 等）は、初回ブロック取得時に決定されるため、ここでは意図的に変更しないように注意してください。
