ResourceOwnerForgetCryptoHash

定義

  • 種別: static inline function
  • 定義箇所: src/common/cryptohash_openssl.c:95

目的

バックエンド環境において、指定された pg_cryptohash_ctx を ResourceOwner の管理対象から除外する。

シグネチャまたは形状

static inline void ResourceOwnerForgetCryptoHash(ResourceOwner owner, pg_cryptohash_ctx *ctx)

入力と出力

  • owner: 対象の ResourceOwner オブジェクト。
  • ctx: 管理を解除する pg_cryptohash_ctx 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • ResourceOwnerRememberCryptoHash によって、当該 ctx が既に owner に登録されている必要がある。

アルゴリズムまたは振る舞い

内部で ResourceOwnerForget を呼び出し、cryptohash_resowner_desc を識別子として ctx をリソース所有者管理リストから削除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/cryptohash_openssl.c 内のコンテキスト解放処理（例: pg_cryptohash_free 相当の処理フロー）。
  • 呼び出し先: ResourceOwnerForget（PostgreSQLコアのメモリ管理API）。

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算状態を保持する構造体。
  • ResourceOwner: トランザクション等のリソースライフサイクルを管理するオブジェクト。
  • cryptohash_resowner_desc: リソース所有者管理用の型定義（ResourceOwnerForget に渡される）。

失敗モードと境界条件

  • owner が無効な場合や、ctx が当該 owner に登録されていない場合の挙動は ResourceOwnerForget の仕様に依存する。

保守担当者向けメモ

  • 本関数はバックエンド専用のメモリ管理機構の一部である。フロントエンド（クライアントツール）ビルド時には使用されない。
