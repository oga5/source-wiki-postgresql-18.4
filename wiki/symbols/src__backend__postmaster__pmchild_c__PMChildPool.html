PMChildPool

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/pmchild.c:46

目的

PostgreSQL の Postmaster が管理する特定プロセスタイプ（バックエンド、ワーカ、補助プロセス等）向けに、予約された子プロセススロットのプールを管理します。

シグネチャまたは形状

typedef struct PMChildPool
{
    int         size;           /* このプロセスタイプ用に予約されたスロット数 */
    int         first_slotno;   /* このプールに属する最初のスロット番号 */
    dlist_head  freelist;       /* 現在未使用の PMChild エントリのリスト */
} PMChildPool;

入力と出力

  • **入力**: InitPostmasterChildSlots() による初期化時に、プロセスタイプごとのスロット数と開始インデックスが設定されます。
  • **出力**: AssignPostmasterChildSlot() 呼び出し時に、freelist から未使用の PMChild エントリが提供されます。

前提条件と不変条件

  • size は、そのプロセスタイプに対して割り当てられたスロットの総数を表します。
  • freelist は、現在稼働中でないスロットを保持する双方向リストとして維持されます。
  • first_slotno から始まる連続したスロット領域が、当該プールに割り当てられている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: InitPostmasterChildSlots が設定値に基づき各プールを構築し、freelist にスロットを登録します。 2. **割り当て**: 新規プロセス生成時、AssignPostmasterChildSlot が freelist からノードを取り出し、アクティブなプロセスとして追跡を開始します。 3. **解放**: プロセス終了時、ReleasePostmasterChildSlot が該当スロットを freelist に戻し、再利用可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/pmchild.c 内の InitPostmasterChildSlots、AssignPostmasterChildSlot、ReleasePostmasterChildSlot 等がこの構造体を操作します。
  • **呼び出し先**: dlist API（dlist_head 操作用）を利用します。
  • **注記**: 具体的な関数呼び出しの連鎖については、提供されたソース範囲外のため詳細な追跡は行っていません。

関連データ構造

  • PMChild: プール内の個々のスロットを表現する構造体。
  • pmchild_pools: PMChildPool 型の静的配列（src/backend/postmaster/pmchild.c:54）。
  • ActiveChildList: 稼働中の全子プロセスを管理する双方向リスト。

失敗モードと境界条件

  • **プール枯渇**: freelist が空の場合、スロットの割り当ては失敗します。
  • **範囲外アクセス**: first_slotno と size の管理が不整合な場合、スロット配列の境界外参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は Postmaster プロセス内のメモリ上でのみ有効です。
  • BACKEND_NUM_TYPES に基づく配列 pmchild_pools を介してアクセスされるため、プロセスタイプの追加時にはこの配列の整合性に注意してください。
