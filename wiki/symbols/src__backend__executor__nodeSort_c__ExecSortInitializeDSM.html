ExecSortInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSort.c:436

目的

並列クエリ実行時において、ソートノードの統計情報を保持するための共有メモリ領域を初期化し、共有メモリテーブル（TOC）に登録する。

シグネチャまたは形状

void ExecSortInitializeDSM(SortState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の SortState 構造体へのポインタ。
  • pcxt: 並列実行コンテキスト（ParallelContext）へのポインタ。
  • **出力**: なし（副作用として node->shared_info の設定および共有メモリへの書き込みを行う）。

前提条件と不変条件

  • node->ss.ps.instrument が有効であり、かつ pcxt->nworkers > 0 である場合にのみ処理が実行される。
  • 共有メモリ領域は shm_toc_allocate を介して確保され、初期状態はゼロクリアされる。

アルゴリズムまたは振る舞い

1. 計測（instrumentation）が不要、またはワーカーが存在しない場合は直ちに終了する。 2. SharedSortInfo 構造体と、ワーカー数分の TuplesortInstrumentation 配列を格納するために必要なサイズを計算する。 3. shm_toc_allocate を使用して共有メモリを確保し、node->shared_info に割り当てる。 4. 確保した領域を memset でゼロクリアする。 5. num_workers を設定し、shm_toc_insert を用いてプランノードIDをキーとして共有メモリテーブルに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c:521
  • **呼び出し先**:
  • shm_toc_allocate
  • shm_toc_insert
  • memset

関連データ構造

  • SortState: ソートノードの実行状態を保持する構造体。
  • ParallelContext: 並列クエリの実行コンテキスト。
  • SharedSortInfo: 並列ワーカー間で共有されるソート統計情報。
  • TuplesortInstrumentation: 各ワーカーのソート統計情報。

失敗モードと境界条件

  • pcxt->nworkers が 0 の場合、共有メモリの確保は行われない。
  • node->ss.ps.instrument が NULL の場合、処理はスキップされる。

保守担当者向けメモ

  • 本関数は並列クエリの統計収集に特化している。共有メモリのレイアウト変更時には SharedSortInfo の構造と offsetof の計算を整合させる必要がある。
