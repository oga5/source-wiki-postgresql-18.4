目的

Informix 互換モードにおける ECPG の DESCRIBE 文の動作を検証する回帰テスト用エントリポイントです。名前付き記述子（DESCRIPTOR）と SQL 記述子領域（SQLDA）の両方でメタデータが正しく取得できることを確認します。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しません）。
  • 出力: 成功時は終了コード 0 を返し、失敗時は exit(1) を呼び出します。標準出力および標準エラー出力（ECPGdebug によるトレース）にテスト結果を表示します。

前提条件と不変条件

  • データベース ecpg1_regression への接続が可能であること。
  • テスト用テーブル descr_t1 の作成およびデータ挿入が成功すること。
  • Informix 互換モードにおいて、DESCRIPTOR と SQLDA が別個のメカニズムとして扱われることを前提としています。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGconnect でデータベースに接続し、set datestyle を実行します。 2. **準備**: create table および insert 文を実行してテストデータを準備し、commit します。 3. **記述子確保**: ECPGallocate_desc を使用して "desc1" および "desc2" を確保します。 4. **ステートメント準備**: ECPGprepare で SQL 文を準備します。 5. **メタデータ取得**:

  • ECPGdescribe を呼び出し、名前付き記述子（"desc1", "desc2"）および sqlda_t 構造体（sqlda1, sqlda2, sqlda3）にメタデータを格納します。

    • 6. **検証**: 各記述子が NULL でないことを確認し、ECPGget_desc_header を用いて列数を取得・検証します。 7. **終了**: 各種リソースを解放し、テストを完了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムの実行時）。
  • 呼び出し先: ECPG ライブラリ関数（ECPGconnect, ECPGdo, ECPGtrans, ECPGallocate_desc, ECPGprepare, ECPGdescribe, ECPGget_desc_header 等）。

関連データ構造

  • sqlda_t: Informix 互換モードで使用される SQL 記述子構造体。
  • sqlca_t: ECPG のエラー情報を保持する構造体（sqlca.sqlcode を使用して各ステップの成否を判定）。

失敗モードと境界条件

  • 各 ECPG ライブラリ関数の呼び出し後、sqlca.sqlcode < 0 の場合に exit(1) を呼び出し、異常終了します。
  • sqlda 構造体が NULL の場合、エラーメッセージを出力して終了します。
