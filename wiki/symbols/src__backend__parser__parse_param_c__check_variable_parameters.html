check_variable_parameters

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_param.c:268

目的

クエリ解析の最終段階において、動的に解決されたパラメータの型が矛盾なく決定されているかを検証する。

シグネチャまたは形状

void check_variable_parameters(ParseState *pstate, Query *query)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。
  • query: 検証対象の Query 構造体へのポインタ。
  • **出力**: なし（検証失敗時はエラーをスローする）。

前提条件と不変条件

  • pstate->p_ref_hook_state は VarParamState 型としてキャスト可能であること。
  • VarParamState 内の numParams が正しく初期化されていること。

アルゴリズムまたは振る舞い

1. pstate から VarParamState を取得する。 2. numParams が 0 より大きい場合のみ、query_tree_walker を使用してクエリツリーを走査する。 3. 走査には check_parameter_resolution_walker を使用し、各パラメータの型解決状態を確認する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c:164
  • src/backend/parser/analyze.c:3159
  • **呼び出し先**:
  • query_tree_walker (PostgreSQL 内部 API)
  • check_parameter_resolution_walker (ツリー走査用コールバック)

関連データ構造

  • ParseState: 解析の全体状態を保持。
  • VarParamState: 可変パラメータの型情報を管理する内部構造体。
  • Query: 解析対象のクエリツリー。

失敗モードと境界条件

  • numParams が 0 の場合、検証処理はスキップされる。
  • check_parameter_resolution_walker 内で型が未解決（UNKNOWNOID 等）と判断された場合、エラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、setup_parse_variable_parameters によってフックが設定された場合にのみ有効に機能する。
  • query_tree_walker を介した再帰的な走査を行うため、クエリツリーの構造変更時には check_parameter_resolution_walker の実装と併せて確認が必要である。
