secure_read

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure.c:178

目的

クライアントとの通信において、暗号化方式（SSL/TLS、GSSAPI、または平文）を透過的に抽象化し、安全かつ一貫したデータ読み取り操作を提供する。

シグネチャまたは形状

ssize_t secure_read(Port *port, void *ptr, size_t len)

入力と出力

  • **入力**:
  • port: 通信状態および暗号化フラグを保持する Port 構造体へのポインタ。
  • ptr: 読み取ったデータを格納するバッファ。
  • len: 読み取るバイト数。
  • **出力**:
  • 成功時は読み取ったバイト数（ssize_t）、失敗時は -1 を返す。

前提条件と不変条件

  • port が有効な接続状態であること。
  • ブロッキングモードの場合、ソケットの準備ができるまで待機する。
  • 割り込み（シグナルやクエリキャンセル）が発生した場合は、適切に処理を再試行する。

アルゴリズムまたは振る舞い

1. ProcessClientReadInterrupt(false) を呼び出し、保留中の割り込みを処理する。 2. port の設定に基づき、以下のいずれかの読み取り関数を呼び出す。

  • SSL有効時: be_tls_read
  • GSSAPI有効時: be_gssapi_read
  • それ以外: secure_raw_read

    • 3. 読み取りが EWOULDBLOCK または EAGAIN で失敗し、かつブロッキングモードの場合:

  • WaitEventSet を使用してソケットの準備完了を待機する。
  • ポストマスターの終了を検知した場合は FATAL エラーで終了する。
  • 割り込み（WL_LATCH_SET）が発生した場合は ProcessClientReadInterrupt(true) を実行し、読み取りを再試行する。

    • 4. 最後に ProcessClientReadInterrupt(false) を呼び出し、読み取り完了後の割り込みを処理して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/pqcomm.c (複数箇所)
  • **呼び出し先**:
  • ProcessClientReadInterrupt
  • be_tls_read (条件付き)
  • be_gssapi_read (条件付き)
  • secure_raw_read (条件付き)
  • ModifyWaitEvent, WaitEventSetWait, ResetLatch (ブロッキング時)

関連データ構造

  • Port: 通信状態、暗号化フラグ、ソケット情報を保持。
  • WaitEventSet: I/O待機イベント管理用。

失敗モードと境界条件

  • **ポストマスターの終了**: WL_POSTMASTER_DEATH イベント発生時に FATAL エラーを発生させる。
  • **I/Oエラー**: EWOULDBLOCK/EAGAIN 以外のエラーは呼び出し元へそのまま伝播する。

保守担当者向けメモ

  • 本関数は retry ラベルを用いたループ構造を持ち、非同期的な割り込み処理を確実に実行する設計となっている。
  • WaitEventSet を使用した待機処理は、ポストマスターの死活監視とクエリキャンセル等の割り込みを統合するための重要なパターンである。
