pgstat_detach_shmem

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_shmem.c:264

目的

現在のバックエンドプロセスから統計情報共有メモリ領域を切り離し、関連するリソースを解放する。

シグネチャまたは形状

void pgstat_detach_shmem(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリのデタッチとローカル状態のクリアを行う）

前提条件と不変条件

  • pgStatLocal.dsa が有効（非NULL）であること。

アルゴリズムまたは振る舞い

1. pgstat_release_all_entry_refs(false) を呼び出し、共有統計情報へのローカル参照をすべて解放する。 2. dshash_detach を呼び出し、共有ハッシュテーブル pgStatLocal.shared_hash をデタッチし、ポインタを NULL に設定する。 3. dsa_detach を呼び出し、動的共有メモリ領域 pgStatLocal.dsa をデタッチする。 4. dsa_release_in_place を呼び出し、pgStatLocal.shmem->raw_dsa_area を手動で解放する（dsa_attach_in_place を使用したため、自動クリーンアップコールバックが登録されていないことへの対応）。 5. pgStatLocal.dsa を NULL に設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/activity/pgstat.c:626
  • 呼び出し先:
  • pgstat_release_all_entry_refs
  • dshash_detach
  • dsa_detach
  • dsa_release_in_place

関連データ構造

  • pgStatLocal (グローバルな統計制御構造体)
  • dshash (共有ハッシュテーブル)
  • dsa_area (動的共有メモリ領域)

失敗モードと境界条件

  • pgStatLocal.dsa が初期化されていない状態で呼び出された場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は、バックエンド終了時や統計システムの再初期化時に共有メモリリソースをクリーンアップするために使用される。
  • dsa_release_in_place の呼び出しは、dsa_detach が参照カウントを減らさない特殊なケース（dsa_attach_in_place 使用時）に対する明示的な解放処理である点に注意すること。
