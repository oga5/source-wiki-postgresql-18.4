BumpBlock

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/bump.c:64

目的

PostgreSQLのバンプアロケータにおいて、メモリ割り当ての物理的な単位となるブロックを管理するための構造体です。

シグネチャまたは形状

ソースコード上で前方宣言 typedef struct BumpBlock BumpBlock; が行われており、src/backend/utils/mmgr/bump.c 内で定義されています。具体的なメンバ変数の構成は提供された抜粋に含まれていません。

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。BumpBlockInit や BumpBlockFree などの関連関数を通じて、BumpContext と組み合わせて操作されます。

前提条件と不変条件

  • BumpContext が管理する dlist_head blocks リストの一部として保持されます。
  • BumpContext の KeeperBlock（コンテキスト自体を含むブロック）として使用される場合と、動的に確保されるブロックとして使用される場合があります。

アルゴリズムまたは振る舞い

  • バンプアロケータのメモリ確保単位として機能します。
  • 割り当て要求に対し、ブロック内の空き領域をポインタの移動（バンプ）によって切り出します。
  • 個別のメモリチャンクに対する pfree はサポートされず、BumpReset によるブロック単位の一括解放を前提とします。

呼び出し元と呼び出し先

  • 呼び出し元: BumpContext の管理ロジック（BumpAlloc 等）から参照されます。
  • 呼び出し先: BumpBlockInit, BumpBlockIsEmpty, BumpBlockFreeBytes, BumpBlockFree などのインライン関数が本構造体を引数として操作します。

関連データ構造

  • BumpContext: 本構造体をリストとして管理する親コンテキスト。
  • dlist_node (推測): lib/ilist.h を使用しているため、リスト管理用のノードが含まれていると推測されます。

失敗モードと境界条件

  • allocChunkLimit を超える要求が発生した場合、BumpAllocLarge を介して専用のブロックとして扱われます。
  • pfree を呼び出すと ERROR となる制限があります。

保守担当者向けメモ

  • Bump_BLOCKHDRSZ マクロにより、sizeof(BumpBlock) をアライメント調整したサイズがヘッダーサイズとして定義されています。
  • MEMORY_CONTEXT_CHECKING が有効な場合、デバッグ用のチャンクヘッダーが付与されるため、メモリレイアウトが変化することに注意してください。
