customer4

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:105

目的

ECPG（Embedded SQL for C）における構造体ホスト変数のデータマッピングを検証するためのテスト用データ構造。

シグネチャまたは形状

struct customer4 {
  struct varchar_4 { int len; char arr[50]; } name;
  int phone;
} custs4;

入力と出力

  • **入力**: SQL の SELECT 文の結果セットを本構造体のメンバ（name, phone）にバインドする。
  • **出力**: データベースから取得した値が本構造体の各メンバに格納される。

前提条件と不変条件

  • 構造体メンバの順序は、対応するデータベーステーブルの列定義と一致している必要がある。
  • name メンバは varchar 型として定義されており、len（長さ）と arr（文字列バッファ）のペアで構成される。

アルゴリズムまたは振る舞い

ECPG プリプロセッサによって生成されたコードにおいて、ECPGdo 関数を介して SQL 操作のホスト変数として使用される。ECPGdo 呼び出し時に、構造体全体のサイズ（sizeof(struct customer4)）がバインド情報として渡される。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • preproc-array_of_struct.c:238, 240: ECPGdo 関数による SQL 実行時のホスト変数として使用。
  • preproc-pointer_to_struct.c:115: malloc による動的メモリ確保の対象として使用。
  • preproc-pointer_to_struct.c:242, 244: ECPGdo 関数による SQL 実行時のホスト変数として使用。
  • **呼び出し先**: 不明（本シンボルはデータ構造であり、関数呼び出しを行わない）。

関連データ構造

  • varchar_4: customer4 内で定義されたネストされた構造体。
  • custs4: customer4 型のインスタンス変数。

失敗モードと境界条件

  • name メンバの arr は 50 バイト固定であるため、これを超える文字列がデータベースから返された場合、バッファオーバーフローや切り捨てが発生する可能性がある。
  • ECPGdo に渡されるサイズ情報と実際の構造体定義が不一致の場合、メモリレイアウトの不整合により実行時エラーが発生する。
