pg_hmac_init

定義

  • src/common/hmac.c:137

目的

HMAC 計算のためのコンテキストを初期化し、鍵の正規化および内部パディング（k_ipad）の適用を行う。

シグネチャまたは形状

int pg_hmac_init(pg_hmac_ctx *ctx, const uint8 *key, size_t len);

入力と出力

  • **入力**:
  • ctx: 初期化対象の pg_hmac_ctx 構造体へのポインタ。
  • key: HMAC 計算に使用する鍵データ。
  • len: 鍵のバイト長。
  • **出力**: 成功時に 0、失敗時に -1 を返す。

前提条件と不変条件

  • ctx が NULL の場合は -1 を返す。
  • ctx->k_ipad および ctx->k_opad は、ctx->block_size 分のメモリが確保されている必要がある。
  • 鍵長がブロックサイズを超える場合、内部で一時的なハッシュ計算が行われる。

アルゴリズムまたは振る舞い

1. k_ipad を HMAC_IPAD で、k_opad を HMAC_OPAD で初期化する。 2. 鍵長がブロックサイズを超える場合、鍵をハッシュ化して短縮する。 3. 鍵（または短縮された鍵）と k_ipad/k_opad をそれぞれ XOR する。 4. 内部ハッシュコンテキストを初期化し、k_ipad を更新して HMAC の最初のステップを準備する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-scram.c, src/common/scram-common.c, src/interfaces/libpq/fe-auth-scram.c 等から呼び出される。
  • **呼び出し先**: pg_cryptohash_create, pg_cryptohash_init, pg_cryptohash_update, pg_cryptohash_final, pg_cryptohash_free を使用して鍵の短縮やハッシュ計算を行う。

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する不透明な構造体。

失敗モードと境界条件

  • ctx が NULL: -1 を返す。
  • メモリ不足 (ALLOC 失敗): ctx->error に PG_HMAC_ERROR_OOM を設定し -1 を返す。
  • 内部ハッシュ計算失敗: ctx->error に PG_HMAC_ERROR_INTERNAL を設定し -1 を返す。

保守担当者向けメモ

  • 本関数はフォールバック実装であり、OpenSSL 等の外部ライブラリが利用できない環境で使用される。
  • メモリ管理は環境（バックエンドの palloc またはフロントエンドの malloc）に依存する。
  • 鍵の短縮処理で使用する shrinkbuf は、処理終了後に必ず解放される。
