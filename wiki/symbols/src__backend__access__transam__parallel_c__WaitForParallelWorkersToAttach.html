WaitForParallelWorkersToAttach

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/parallel.c:699

目的

並列クエリ実行において、起動されたすべてのワーカープロセスが共有メモリのメッセージキュー（エラーキュー）にアタッチし、初期化を完了するまでリーダープロセスを待機させる。

シグネチャまたは形状

void WaitForParallelWorkersToAttach(ParallelContext *pcxt)

入力と出力

  • **入力**: ParallelContext *pcxt（並列実行コンテキストへのポインタ）
  • **出力**: なし（完了時に復帰）

前提条件と不変条件

  • pcxt は有効な並列コンテキストであること。
  • リーダープロセスは、ワーカーがアタッチするまで定期的に CHECK_FOR_INTERRUPTS() を呼び出し、ワーカーからのエラー伝播を処理する必要がある。

アルゴリズムまたは振る舞い

1. pcxt->nworkers_launched が 0 の場合は即座に終了する。 2. 無限ループ内で以下を繰り返す：

  • CHECK_FOR_INTERRUPTS() を実行し、ワーカーからの保留中のメッセージやエラーを処理する。
  • 未アタッチの各ワーカーについて：
  • error_mqh が NULL の場合、ワーカーは既に終了済みとみなし、アタッチ済みとしてマークする。
  • GetBackgroundWorkerPid で状態を確認し、BGWH_STARTED であれば shm_mq_get_sender を通じてアタッチを確認する。
  • BGWH_STOPPED であり、かつアタッチが確認できない場合は、初期化失敗として ERROR を投げる。
  • 上記以外（未起動）の場合は、WaitLatch を使用してポストマスターからの通知を待機する。

    • 3. すべてのワーカーがアタッチ済み（nknown_attached_workers == nworkers_launched）になった時点でループを抜ける。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • **呼び出し先**:
  • CHECK_FOR_INTERRUPTS()
  • GetBackgroundWorkerPid()
  • shm_mq_get_queue()
  • shm_mq_get_sender()
  • WaitLatch()
  • ResetLatch()
  • ereport()

関連データ構造

  • ParallelContext: 並列実行の管理構造体。
  • BgwHandleStatus: バックグラウンドワーカーのステータス。
  • shm_mq: 共有メモリメッセージキュー。

失敗モードと境界条件

  • ワーカーがアタッチを完了する前に終了した場合、ereport(ERROR, ...) によりリーダープロセスでエラーが発生する。
  • WaitLatch 待機中にポストマスターが終了した場合、WL_EXIT_ON_PM_DEATH により待機が解除される。

保守担当者向けメモ

  • 本関数はリーダープロセスがワーカーの準備完了を同期するための重要なポイントである。
  • known_attached_workers 配列は、各ワーカーのアタッチ状態を追跡するために使用される。
  • shm_mq の送信者が設定されているかどうかが、ワーカーのアタッチ完了の判定基準となっている。
