JsonParseContext

定義

  • 種別: enum
  • 定義箇所: src/common/jsonapi.c:93

目的

JSON構文解析プロセスにおける現在の解析状態（コンテキスト）を追跡し、パーサーが次に期待するトークンや構造を管理するために使用されます。

シグネチャまたは形状

typedef enum {
    JSON_PARSE_VALUE,
    JSON_PARSE_STRING,
    JSON_PARSE_ARRAY_START,
    JSON_PARSE_ARRAY_NEXT,
    JSON_PARSE_OBJECT_START,
    JSON_PARSE_OBJECT_LABEL,
    JSON_PARSE_OBJECT_NEXT
} JsonParseContext;

入力と出力

  • 入力: 解析中のJSONストリームおよび現在の字句解析状態。
  • 出力: 解析の進行に伴い、状態遷移が発生します。

前提条件と不変条件

  • JSONの構文規則に従い、特定のトークンが出現した際に、対応するコンテキストへ遷移する必要があります。
  • 状態遷移の整合性は、パーサー（再帰下降またはテーブル駆動型）によって保証されます。

アルゴリズムまたは振る舞い

  • JSON_PARSE_VALUE: 値（オブジェクト、配列、文字列、数値、真偽値、null）を期待。
  • JSON_PARSE_STRING: フィールド名として文字列を期待。
  • JSON_PARSE_ARRAY_START: 配列の開始 [ を受け取り、要素または終了 ] を期待。
  • JSON_PARSE_ARRAY_NEXT: 配列要素を受け取り、区切り文字 , または終了 ] を期待。
  • JSON_PARSE_OBJECT_START: オブジェクトの開始 { を受け取り、ラベルまたは終了 } を期待。
  • JSON_PARSE_OBJECT_LABEL: オブジェクトのラベルを受け取り、値との区切り : を期待。
  • JSON_PARSE_OBJECT_NEXT: オブジェクトの値を受け取り、区切り文字 , または終了 } を期待。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/jsonapi.c 内の解析ロジック（report_parse_error や lex_expect など）。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数呼び出しは発生しません。

関連データ構造

  • JsonLexContext: 字句解析の状態を保持する構造体。
  • JsonParserStack: インクリメンタル解析時に使用される予測スタック。

失敗モードと境界条件

  • 予期しないトークンが入力された場合、report_parse_error を通じてエラーが報告されます。
  • 構文エラー発生時、現在の JsonParseContext の値がエラー詳細の生成に使用されます。

保守担当者向けメモ

  • 新しいJSON構文要素を追加する場合は、この列挙型に適切な状態を追加し、パーサーの遷移テーブルまたはロジックを更新してください。
