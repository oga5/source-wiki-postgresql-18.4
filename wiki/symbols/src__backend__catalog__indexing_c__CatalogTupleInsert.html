CatalogTupleInsert

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/indexing.c:232

目的

システムカタログへのタプル挿入と、それに伴うインデックスの整合性維持を統合的に実行する。

シグネチャまたは形状

void CatalogTupleInsert(Relation heapRel, HeapTuple tup)

入力と出力

  • **入力**:
  • heapRel: 挿入対象のシステムカタログリレーション記述子。
  • tup: 挿入するヒープタプル。
  • **出力**: なし（副作用としてカタログおよびインデックスを更新）。

前提条件と不変条件

  • システムカタログ特有の制約（NOT NULL等）が満たされている必要がある。
  • 式インデックス、部分インデックス、除外制約はサポートされていない。

アルゴリズムまたは振る舞い

1. CatalogTupleCheckConstraints を呼び出し、カタログ制約の検証を行う。 2. CatalogOpenIndexes を呼び出し、対象リレーションのインデックス情報を取得・初期化する。 3. simple_heap_insert を呼び出し、ヒープへタプルを挿入する。 4. CatalogIndexInsert を呼び出し、取得したインデックス情報に基づき全インデックスを更新する。 5. CatalogCloseIndexes を呼び出し、インデックス関連のリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: aclchk.c, heap.c, index.c, pg_aggregate.c, pg_attrdef.c, pg_cast.c 等のカタログ操作関数から参照されている。
  • **呼び出し先**:
  • CatalogTupleCheckConstraints
  • CatalogOpenIndexes
  • simple_heap_insert
  • CatalogIndexInsert
  • CatalogCloseIndexes

関連データ構造

  • Relation: ヒープリレーションの記述子。
  • HeapTuple: 挿入対象のタプルデータ。
  • CatalogIndexState: インデックス更新状態を保持する不透明な構造体。

失敗モードと境界条件

  • カタログ制約違反が発生した場合、CatalogTupleCheckConstraints によりエラーが送出される。
  • インデックス更新時にシステムカタログの制約（式インデックス等の非対応機能）に抵触する場合、CatalogIndexInsert 内の Assert により停止する可能性がある。

保守担当者向けメモ

  • 複数行を挿入する場合は、オーバーヘッドを削減するため CatalogTupleInsertWithInfo または CatalogTuplesMultiInsertWithInfo の使用を検討すること。
  • 本関数は単一タプルの挿入用であり、インデックスのオープン・クローズを毎回行うため、大量挿入時のパフォーマンスには注意が必要である。
