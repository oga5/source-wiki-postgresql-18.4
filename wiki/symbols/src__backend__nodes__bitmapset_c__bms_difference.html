bms_difference

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/bitmapset.c:345

目的

2つのビットマップセット a と b の差集合（a に含まれ、かつ b に含まれない要素の集合）を計算し、新しい Bitmapset として返します。

シグネチャまたは形状

Bitmapset *bms_difference(const Bitmapset *a, const Bitmapset *b)

入力と出力

  • **入力**:
  • a: 減算対象となる Bitmapset（const）。
  • b: 減算する要素を持つ Bitmapset（const）。
  • **出力**:
  • 差集合を表す新しい Bitmapset。結果が空集合となる場合は NULL を返します。

前提条件と不変条件

  • a および b は bms_is_valid_set を満たす有効なビットマップセットである必要があります。
  • 空集合は NULL ポインタとして表現されます。
  • 戻り値の Bitmapset は呼び出し元で管理（必要に応じて bms_free）する必要があります。

アルゴリズムまたは振る舞い

1. **早期リターン**: a が NULL なら NULL を返し、b が NULL なら a のコピーを返します。 2. **最適化**: bms_nonempty_difference を用いて、結果が空になるケースを事前に判定し、不要なメモリ割り当てを回避します。 3. **演算**: a のコピーを作成し、b の各ワードをビット単位で反転させて AND 演算を行います。 4. **トリミング**: a のワード数が b より多い場合はそのまま保持しますが、そうでない場合は演算後に末尾のゼロワードを削除し、nwords を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: 主にクエリ最適化フェーズで使用されます。
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/path/joinpath.c
  • src/backend/optimizer/plan/analyzejoins.c
  • src/backend/optimizer/plan/createplan.c
  • src/backend/optimizer/plan/initsplan.c
  • **呼び出し先**:
  • bms_is_valid_set (Assert内)
  • bms_copy
  • bms_nonempty_difference

関連データ構造

  • Bitmapset: 集合を保持する構造体。nwords（ワード数）と words（ビット配列）で構成されます。

失敗モードと境界条件

  • 入力が NULL の場合、適切に処理され NULL またはコピーが返されます。
  • 演算の結果、集合が空になる場合は NULL が返されます。
  • Assert により、無効なビットマップセットが渡された場合はプログラムが停止します。

保守担当者向けメモ

  • bms_nonempty_difference による事前チェックは、頻繁に発生する「結果が空」のケースに対するパフォーマンス最適化です。
  • result->nwords の更新ロジックは、メモリ効率を維持するために重要です。変更を加える際は、末尾のゼロワードが正しく処理されるか確認してください。
