PGTYPESnumeric_from_int

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/numeric.c:1308

目的

符号付き整数（signed int）を PostgreSQL の numeric 構造体に変換します。

シグネチャまたは形状

int PGTYPESnumeric_from_int(signed int int_val, numeric *var)

入力と出力

  • **入力**:
  • int_val: 変換対象の signed int 値。
  • var: 変換結果を格納する numeric 構造体へのポインタ。
  • **出力**:
  • 成功時は 0 を返します。失敗時の戻り値は PGTYPESnumeric_from_long の仕様に依存します。

前提条件と不変条件

  • var は有効な numeric 構造体を指している必要があります。
  • 内部で PGTYPESnumeric_from_long を呼び出すため、その関数が要求するメモリ管理や構造体の初期化条件に従います。

アルゴリズムまたは振る舞い

1. 引数 int_val を signed long 型の変数 long_int に暗黙的にキャストします。 2. PGTYPESnumeric_from_long(long_int, var) を呼び出し、実際の変換処理を委譲してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c
  • src/interfaces/ecpg/test/expected/sql-array.c
  • **呼び出し先**:
  • PGTYPESnumeric_from_long (同ファイル内の関数)

関連データ構造

  • numeric: 数値データを保持する構造体。

失敗モードと境界条件

  • 変換処理自体は PGTYPESnumeric_from_long に依存します。
  • int から long へのキャストは安全ですが、numeric 構造体への格納時にメモリ不足等のエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は PGTYPESnumeric_from_long の単なるラッパーです。将来的に int 特有の最適化が必要でない限り、この構造を維持してください。
  • 戻り値の詳細は PGTYPESnumeric_from_long の実装を確認してください。
