findLastCheckpoint

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/parsexlog.c:167

目的

ターゲットサーバーの WAL を逆方向に辿り、同期の起点となる最新のチェックポイントレコードを特定する。

シグネチャまたは形状

void findLastCheckpoint(const char *datadir, XLogRecPtr forkptr, int tliIndex,
                        XLogRecPtr *lastchkptrec, TimeLineID *lastchkpttli,
                        XLogRecPtr *lastchkptredo, const char *restoreCommand)

入力と出力

  • **入力**:
  • datadir: データディレクトリのパス。
  • forkptr: 探索を開始する WAL の LSN（分岐点）。
  • tliIndex: タイムラインインデックス。
  • restoreCommand: WAL アーカイブから復元するためのコマンド文字列。
  • **出力**:
  • lastchkptrec: 見つかったチェックポイントレコードの LSN。
  • lastchkpttli: チェックポイントのタイムライン ID。
  • lastchkptredo: チェックポイントの REDO LSN。

前提条件と不変条件

  • forkptr がページ境界にある場合、関数内で自動的にページヘッダー分（ロングまたはショート）が加算され、レコードの開始位置へ調整される。
  • XLogReader の初期化に失敗した場合は pg_fatal で終了する。

アルゴリズムまたは振る舞い

1. XLogReaderAllocate を使用し、SimpleXLogPageRead をコールバックとして WAL リーダーを初期化する。 2. forkptr から開始し、XLogReadRecord を用いて WAL を逆方向に走査する。 3. 走査中に新しい WAL セグメントに到達した場合は、keepwal_add_entry を呼び出し、該当ファイルを削除対象から除外する。 4. RM_XLOG_ID リソースマネージャかつ XLOG_CHECKPOINT_SHUTDOWN または XLOG_CHECKPOINT_ONLINE フラグを持つレコードを検索する。 5. チェックポイントレコードを発見した場合、その情報を出力引数に格納してループを終了する。 6. 最後に XLogReaderFree でリーダーを解放し、グローバルな xlogreadfd が開いている場合は閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/pg_rewind.c:462
  • **呼び出し先**:
  • XLogReaderAllocate, XLogBeginRead, XLogReadRecord, XLogRecGetInfo, XLogRecGetRmid, XLogRecGetData, XLogReaderFree, keepwal_add_entry, SimpleXLogPageRead（コールバックとして登録）

関連データ構造

  • XLogReaderState: WAL 読み込み状態を保持する構造体。
  • XLogPageReadPrivate: SimpleXLogPageRead で使用されるプライベートデータ。
  • CheckPoint: チェックポイントレコードのデータ構造。

失敗モードと境界条件

  • WAL レコードが見つからない場合、pg_fatal を呼び出しプロセスを終了する。
  • メモリ不足により XLogReaderAllocate が失敗した場合、pg_fatal で終了する。
  • 探索中に xlogreadfd が開いたままになる可能性があるため、終了時に明示的にクローズ処理を行う。

保守担当者向けメモ

  • 本関数は WAL を逆方向に辿るため、record->xl_prev を使用して LSN を更新する。
  • タイムラインの切り替えやセグメントの境界を跨ぐ処理が含まれているため、WAL の物理構造変更時には注意が必要。
