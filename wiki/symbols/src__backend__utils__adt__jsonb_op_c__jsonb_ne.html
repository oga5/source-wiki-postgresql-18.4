jsonb_ne

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_op.c:148

目的

2つの jsonb 型の値が等しくないかどうか（!= 演算子）を判定します。

シグネチャまたは形状

Datum jsonb_ne(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの jsonb ポインタ（jba, jbb）を受け取ります。
  • **出力**: bool 型の結果を Datum として返します。jba と jbb が等しくない場合に true、等しい場合に false を返します。

前提条件と不変条件

  • 入力された jsonb ポインタは PG_GETARG_JSONB_P マクロにより取得されます。
  • PG_FREE_IF_COPY により、必要に応じて入力引数のメモリ解放が行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_JSONB_P を使用して、2つの jsonb 構造体へのポインタを取得します。 2. compareJsonbContainers 関数を呼び出し、両者の root 構造を比較します。 3. 比較結果が 0（等価）でない場合、res を true とします。 4. PG_FREE_IF_COPY を呼び出して、コピーされた入力引数のメモリを解放します。 5. PG_RETURN_BOOL を使用して、判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの != 演算子から関数マネージャ（fmgr）を介して呼び出されます。
  • **呼び出し先**: compareJsonbContainers（jsonb 構造の比較を実行）。

関連データ構造

  • Jsonb: jsonb 型の内部表現。
  • JsonbContainer: Jsonb 構造体内の root フィールドが指すデータ構造。

失敗モードと境界条件

  • 入力が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で NULL が処理される必要があります。
  • compareJsonbContainers の仕様に依存し、構造が複雑な場合でも再帰的に比較が行われます。

保守担当者向けメモ

  • 本関数は jsonb 型の比較演算の一部であり、B-Tree インデックス等のソート順序を決定する jsonb_cmp と一貫した比較ロジックを維持する必要があります。
