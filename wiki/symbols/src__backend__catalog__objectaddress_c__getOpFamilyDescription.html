getOpFamilyDescription

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaddress.c:4177

目的

指定された演算子族（Operator Family）のOIDに基づき、その演算子族を識別するための人間が読める説明文を生成し、提供されたバッファに書き込みます。

シグネチャまたは形状

static void getOpFamilyDescription(StringInfo buffer, Oid opfid, bool missing_ok)

入力と出力

  • **入力**:
  • buffer: 説明文を格納する StringInfo 型のバッファ。
  • opfid: 対象となる演算子族のOID。
  • missing_ok: true の場合、対象が見つからなくてもエラーにせず終了する。false の場合はエラーを投げる。
  • **出力**: なし（buffer への書き込みによる副作用）。

前提条件と不変条件

  • opfid に対応する pg_opfamily カタログエントリが存在する必要があります（missing_ok が false の場合）。
  • 演算子族が参照するアクセスメソッド（pg_am）のエントリは常に存在する必要があります。

アルゴリズムまたは振る舞い

1. OPFAMILYOID システムキャッシュを使用して pg_opfamily から対象のタプルを取得します。見つからない場合は missing_ok に応じてエラーを投げるか終了します。 2. 取得したタプルからアクセスメソッドのOID（opfmethod）を抽出し、AMOID システムキャッシュから対応する pg_am タプルを取得します。見つからない場合はエラーを投げます。 3. OpfamilyIsVisible を用いて、演算子族が現在の検索パスで可視かどうかを判定します。 4. 可視でない場合は get_namespace_name で名前空間名を取得し、可視であれば NULL とします。 5. appendStringInfo を使用し、「operator family [名前] for access method [アクセスメソッド名]」という形式の文字列をバッファに追加します。 6. 使用したシステムキャッシュタプルを ReleaseSysCache で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c 内の複数箇所（オブジェクトアドレスの記述生成処理など）。
  • **呼び出し先**:
  • SearchSysCache1, ReleaseSysCache (システムキャッシュ操作)
  • GETSTRUCT (タプル構造体変換)
  • OpfamilyIsVisible (可視性判定)
  • get_namespace_name (名前空間名取得)
  • appendStringInfo (文字列操作)
  • quote_qualified_identifier (識別子整形)
  • elog (エラー報告)

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • pg_opfamily: 演算子族のカタログ定義。
  • pg_am: アクセスメソッドのカタログ定義。

失敗モードと境界条件

  • opfid が無効で missing_ok が false の場合、elog(ERROR) によりトランザクションが中断されます。
  • 演算子族に対応するアクセスメソッドがシステムカタログから削除されているなど、整合性が取れない場合は elog(ERROR) が発生します。

保守担当者向けメモ

  • 本関数は objectaddress.c 内のオブジェクト記述生成ロジックの一部です。新しいオブジェクトタイプを追加する際、演算子族に関連する記述が必要な場合は本関数が再利用されます。
