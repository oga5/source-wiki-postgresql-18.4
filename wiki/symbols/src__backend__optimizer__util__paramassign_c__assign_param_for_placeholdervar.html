assign_param_for_placeholdervar

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:149

目的

PlaceHolderVar を実行時パラメータ（PARAM_EXEC）として割り当て、サブクエリやネストループ結合間での値の受け渡しを可能にする。既存の割り当てがあれば再利用し、なければ新規 ID を発行する。

シグネチャまたは形状

int assign_param_for_placeholdervar(PlannerInfo *root, PlaceHolderVar *phv)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo。
  • phv: パラメータ化対象の PlaceHolderVar。
  • **出力**: 割り当てられた PARAM_EXEC ID（整数）。

前提条件と不変条件

  • phv->phlevelsup を参照し、対象の PlaceHolderVar が属するクエリレベルまで root を遡る必要がある。
  • 新規作成時、phv は copyObject され、phlevelsup が 0 に調整される。
  • root->glob->paramExecTypes には、割り当てられた ID に対応する型情報が保持される。

アルゴリズムまたは振る舞い

1. **レベル特定**: phv->phlevelsup に基づき、対象のクエリレベルまで root を移動させる。 2. **重複排除**: 対象レベルの root->plan_params を走査し、同一の phid を持つ PlaceHolderVar が既に登録されているか確認する。存在すればその paramId を返す。 3. **新規登録**:

  • phv を複製し、レベルを 0 に調整する。
  • PlannerParamItem を作成し、root->glob->paramExecTypes に phv->phexpr の型を追加する。
  • root->plan_params に新しい pitem を追加する。

    • 4. **返却**: 割り当てられた paramId を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/paramassign.c 内の replace_outer_placeholdervar からの呼び出しが確認されている。
  • **呼び出し先**: copyObject, IncrementVarSublevelsUp, makeNode, list_length, lappend_oid, lappend, exprType 等の PostgreSQL 内部ユーティリティ。

関連データ構造

  • PlannerInfo: クエリのプランニング状態を保持。
  • PlaceHolderVar: プレースホルダ式。
  • PlannerParamItem: パラメータ化されたアイテムと ID のマッピング。
  • root->glob->paramExecTypes: 実行時パラメータの型リスト。

失敗モードと境界条件

  • phv が既に存在する場合、重複して ID を発行せず既存の ID を再利用する（重複排除）。
  • phlevelsup が不正な場合、root の探索で予期せぬ動作をする可能性がある。

保守担当者向けメモ

  • 本関数は PlaceHolderVar の phid を比較して同一性を判定している。
  • root->plan_params はサブクエリやネストループの処理単位で管理されるため、異なるスコープ間での ID 共有は行われない点に注意すること。
