COMPAT_MODE

定義

  • 種別: enum
  • 定義箇所: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:17

目的

ECPGライブラリにおける動作モード（PostgreSQL、Informix、Oracle等）を切り替え、データベースとの互換性を制御するために使用されます。

シグネチャまたは形状

enum COMPAT_MODE
{
    ECPG_COMPAT_PGSQL = 0,
    ECPG_COMPAT_INFORMIX,
    ECPG_COMPAT_INFORMIX_SE,
    ECPG_COMPAT_ORACLE
};

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: ECPG_COMPAT_PGSQL, ECPG_COMPAT_INFORMIX, ECPG_COMPAT_INFORMIX_SE, ECPG_COMPAT_ORACLE のいずれかの値。

前提条件と不変条件

  • ライブラリ内部の関数（データ変換や接続管理など）において、互換性フラグとして引数に渡されることが前提となっています。

アルゴリズムまたは振る舞い

  • ecpglib 内の各関数が、この列挙値に基づいて特定のデータベース製品特有の挙動（データ型の解釈やエラー処理など）を選択的に実行します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/interfaces/ecpg/ecpglib/connect.c:263
  • src/interfaces/ecpg/ecpglib/data.c:46, 209
  • src/interfaces/ecpg/ecpglib/ecpglib_extern.h:70, 167, 181, 212, 216, 224, 227, 229, 231
  • 呼び出し先: 該当なし（列挙型のため）

関連データ構造

  • struct connection: 接続状態の一部として COMPAT_MODE を保持します。
  • struct statement: SQL実行の文脈としてこのモードを参照する場合があります。

失敗モードと境界条件

  • 定義外の整数値がキャストされて代入された場合、ライブラリ内の条件分岐で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しいデータベース製品の互換モードを追加する場合は、この列挙型に値を追加し、data.c や execute.c 等の各関数内で該当する条件分岐を更新する必要があります。
