pg_fatal_impl

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:74

目的

テスト実行中に致命的なエラーが発生した際、エラーメッセージを標準エラー出力に整形して出力し、プログラムを異常終了させるためのユーティリティ関数です。

シグネチャまたは形状

pg_noreturn static void pg_fatal_impl(int line, const char *fmt, ...);

入力と出力

  • **入力**:
  • line: エラーが発生したソースコードの行番号（通常 __LINE__ が渡される）。
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（関数は exit(1) により終了するため戻り値を持たない）。

前提条件と不変条件

  • fmt 文字列の末尾は改行文字（\n）であってはならない（Assert により検証される）。
  • progname グローバル変数が適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

1. fflush(stdout) を呼び出し、標準出力のバッファをフラッシュする。 2. progname と line を含むエラーメッセージのヘッダーを標準エラー出力 (stderr) に出力する。 3. 可変長引数を用いて、指定されたフォーマットでエラー詳細を stderr に出力する。 4. fmt の末尾に改行が含まれていないことを確認する。 5. 改行を出力してメッセージを完結させる。 6. exit(1) を呼び出し、プロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pg_fatal マクロ（src/test/modules/libpq_pipeline/libpq_pipeline.c:73 で定義）を介して、テスト内の各所（PQgetResult の結果チェック、クエリ送信失敗時、期待値との不一致時など）から呼び出される。
  • **呼び出し先**:
  • 標準ライブラリ関数: fflush, fprintf, vfprintf, strlen, exit。

関連データ構造

  • なし。

失敗モードと境界条件

  • fmt の末尾に改行文字が含まれている場合、Assert が有効なビルドではアサーション失敗が発生する。
  • progname が未設定の場合、出力されるエラーメッセージのヘッダーが不完全になる可能性がある。

保守担当者向けメモ

  • 本関数は pg_fatal マクロ経由で使用することが推奨されます。直接呼び出す場合は、行番号の管理に注意してください。
  • pg_noreturn 属性が付与されているため、コンパイラは本関数呼び出し後のコード到達不能を前提に最適化を行います。
