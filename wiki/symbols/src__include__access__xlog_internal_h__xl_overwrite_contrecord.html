xl_overwrite_contrecord

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/xlog_internal.h:293

目的

WAL（先行書き込みログ）のオーバーライト（上書き）操作が発生した際に、その詳細を記録するための構造体です。

シグネチャまたは形状

typedef struct xl_overwrite_contrecord
{
    XLogRecPtr    overwritten_lsn;
    TimestampTz   overwrite_time;
} xl_overwrite_contrecord;

入力と出力

  • **入力**: WAL書き込み時、overwritten_lsn（上書きされたLSN）と overwrite_time（上書き発生時刻）が設定されます。
  • **出力**: WAL読み込み時、XLogRecGetData を介してこの構造体へデータがコピーされ、リカバリ処理等で参照されます。

前提条件と不変条件

  • overwritten_lsn は有効なWALログシーケンス番号である必要があります。
  • overwrite_time はシステム時刻に基づいたタイムスタンプ値です。

アルゴリズムまたは振る舞い

WALのオーバーライトイベント発生時に、この構造体を介してログレコードとして書き込まれます。リカバリ時には、この構造体を通じて過去のオーバーライト情報を復元し、適切な処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元（書き込み）**: src/backend/access/transam/xlog.c
  • **呼び出し元（読み込み/解析）**:
  • src/backend/access/rmgrdesc/xlogdesc.c
  • src/backend/access/transam/xlogrecovery.c
  • **呼び出し先**: 直接的な関数呼び出し関係は本構造体定義には含まれません。

関連データ構造

  • XLogRecPtr: WALの論理位置を示す型。
  • TimestampTz: タイムスタンプ型。

失敗モードと境界条件

  • 構造体のサイズが sizeof(xl_overwrite_contrecord) と一致しない場合、memcpy によるデータ復元時に不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体はWALの物理フォーマットの一部であるため、フィールドの追加や変更を行う際は、関連する xlogdesc.c（ログ出力用）および xlogrecovery.c（リカバリ用）の memcpy 箇所を必ず更新してください。
