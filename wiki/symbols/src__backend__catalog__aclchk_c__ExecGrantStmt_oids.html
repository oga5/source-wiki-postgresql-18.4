ExecGrantStmt_oids

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/aclchk.c:601

目的

InternalGrant 構造体で指定されたオブジェクト型に基づき、適切な権限操作（GRANT または REVOKE）を実行するディスパッチャ関数です。

シグネチャまたは形状

static void ExecGrantStmt_oids(InternalGrant *istmt)

入力と出力

  • **入力**: InternalGrant *istmt（実行対象の権限操作情報を含む構造体）
  • **出力**: なし（副作用としてシステムカタログを更新）

前提条件と不変条件

  • istmt->objtype が有効なオブジェクト型である必要があります。
  • 呼び出し元で istmt が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. istmt->objtype を評価し、対象オブジェクトの種類に応じて以下のいずれかを呼び出します。

  • ExecGrant_Relation: テーブル、シーケンス用。
  • ExecGrant_Largeobject: ラージオブジェクト用。
  • ExecGrant_Parameter: パラメータACL用。
  • ExecGrant_common: データベース、型、FDW、外部サーバー、関数/プロシージャ、言語、スキーマ、テーブルスペース用。

    • 2. 未知の objtype が渡された場合は elog(ERROR) を発生させます。 3. 権限操作の実行後、EventTriggerSupportsObjectType が真であれば、EventTriggerCollectGrant を呼び出し、イベントトリガーに操作情報を通知します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/aclchk.c 内の複数箇所（ExecuteGrantStmt 等から参照）。
  • src/backend/tcop/utility.c（SQLコマンド実行フローの一部として）。
  • **呼び出し先**:
  • ExecGrant_Relation, ExecGrant_Largeobject, ExecGrant_Parameter, ExecGrant_common
  • EventTriggerSupportsObjectType, EventTriggerCollectGrant

関連データ構造

  • InternalGrant: 権限操作の対象や内容を保持する構造体。

失敗モードと境界条件

  • istmt->objtype がサポートされていない場合、elog(ERROR) によりトランザクションが中断されます。

保守担当者向けメモ

  • 新しいオブジェクト型を GRANT/REVOKE の対象として追加する場合は、本関数の switch 文に新しい case を追加する必要があります。
  • イベントトリガーの通知は、カタログ更新が完了した後に実行される設計となっています。
