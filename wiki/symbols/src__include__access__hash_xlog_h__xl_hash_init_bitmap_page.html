xl_hash_init_bitmap_page

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/hash_xlog.h:235

目的

ハッシュインデックスのビットマップページを初期化する際のWAL（Write-Ahead Logging）レコードのペイロード構造を定義します。

シグネチャまたは形状

typedef struct xl_hash_init_bitmap_page
{
    uint16      bmsize;
} xl_hash_init_bitmap_page;

入力と出力

  • **入力**: bmsize（ビットマップのサイズ）を保持します。
  • **出力**: なし（本構造体はWALレコードの一部として書き込まれます）。

前提条件と不変条件

  • bmsize はビットマップページの初期化に必要なサイズ情報を格納します。
  • WALレコードとして書き込まれる際、この構造体に続くデータが存在する可能性があります（offsetof を用いた計算がヘッダー内で参照されています）。

アルゴリズムまたは振る舞い

  • ハッシュインデックスのリカバリ処理において、ビットマップページを初期化する操作を記録するために使用されます。
  • リカバリ時、XLogRecGetData を通じてこの構造体が読み取られ、ビットマップページの初期化ロジックに渡されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/hash/hash_xlog.c:73
  • src/backend/access/hash/hashpage.c:465
  • src/backend/access/rmgrdesc/hashdesc.c:37
  • **呼び出し先**: 呼び出し先に関する情報は提供されていません。

関連データ構造

  • src/include/access/hash_xlog.h 内の他の xl_hash_* 構造体（WALレコード定義群）。

失敗モードと境界条件

  • 構造体のサイズや bmsize の値が不正な場合、リカバリ処理中のページ初期化が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は hash_redo 等のリカバリ処理において、WALレコードからキャストして使用されます。
  • src/include/access/hash_xlog.h:241 にある offsetof を用いた計算式は、この構造体を含むレコードのレイアウトを理解する上で重要です。
