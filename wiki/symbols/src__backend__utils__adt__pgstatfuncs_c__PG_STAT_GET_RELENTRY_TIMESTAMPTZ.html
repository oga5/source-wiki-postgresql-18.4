PG_STAT_GET_RELENTRY_TIMESTAMPTZ

定義

  • 種別: macro (関数定義用テンプレート)
  • 定義箇所: src/backend/utils/adt/pgstatfuncs.c:137

目的

統計情報システムから、テーブルに関連する特定の TimestampTz 型の統計項目を抽出して SQL レイヤーへ返却するための関数を生成するマクロです。

シグネチャまたは形状

PG_STAT_GET_RELENTRY_TIMESTAMPTZ(stat_name)
  • stat_name: PgStat_StatTabEntry 構造体に含まれる TimestampTz 型のフィールド名。

入力と出力

  • **入力**: SQL 関数呼び出し時の引数として Oid (テーブルのOID) を受け取ります。
  • **出力**: 指定された統計項目の値（TimestampTz）を Datum 型として返します。統計情報が存在しない場合は NULL を返します。

前提条件と不変条件

  • 呼び出し元は、対象の統計情報に対する適切なアクセス権限を有している必要があります。
  • pgstat_fetch_stat_tabentry(relid) を介して、共有メモリまたはローカルキャッシュから最新のスナップショットを取得できることが前提です。

アルゴリズムまたは振る舞い

1. 引数として渡された Oid を取得します。 2. pgstat_fetch_stat_tabentry を呼び出し、対象テーブルの統計エントリを取得します。 3. エントリが存在しない場合は PG_RETURN_NULL() を実行します。 4. エントリが存在する場合、マクロ引数で指定されたフィールド（last_analyze_time 等）の値を PG_RETURN_TIMESTAMPTZ() を用いて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（システムビュー経由の呼び出し）。
  • **呼び出し先**:
  • pgstat_fetch_stat_tabentry: 統計情報の取得。
  • PG_GETARG_OID: 引数の取得。
  • PG_RETURN_TIMESTAMPTZ: 結果の返却。
  • **具体的な使用例**:
  • last_autoanalyze_time
  • last_autovacuum_time
  • last_vacuum_time
  • lastscan

関連データ構造

  • PgStat_StatTabEntry: テーブル統計情報を保持する構造体。
  • TimestampTz: PostgreSQL のタイムスタンプ型。

失敗モードと境界条件

  • 指定された Oid に対応する統計エントリが共有メモリ上に存在しない場合、関数は NULL を返します。
  • 統計情報システムが初期化されていない、あるいは無効な状態である場合、期待される値が取得できない可能性があります。

保守担当者向けメモ

  • 本マクロは pgstatfuncs.c 内で統計取得関数を簡潔に定義するために使用されています。新しいタイムスタンプ型の統計項目を追加する際は、このマクロを呼び出すだけで実装が完了します。
  • 統計情報はスナップショットベースであるため、呼び出しタイミングによって値が更新される可能性がある点に留意してください。
