bernoulli_nextsampletuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/bernoulli.c:180

目的

TABLESAMPLE BERNOULLI メソッドにおいて、スキャン中のブロック内から次にサンプリング対象となるタプルのオフセット番号を決定する。

シグネチャまたは形状

static OffsetNumber bernoulli_nextsampletuple(SampleScanState *node, BlockNumber blockno, OffsetNumber maxoffset)

入力と出力

  • **入力**:
  • node: サンプリング状態を保持する SampleScanState 構造体。
  • blockno: 現在走査中のブロック番号。
  • maxoffset: 現在のブロック内の最大オフセット番号。
  • **出力**:
  • 選択されたタプルの OffsetNumber。ブロック内に該当タプルがない場合は InvalidOffsetNumber を返す。

前提条件と不変条件

  • node->tsm_state は BernoulliSamplerData 型としてキャスト可能であること。
  • sampler->seed および sampler->cutoff は初期化済みであること。
  • ハッシュ計算には hash_any を使用し、マシンアーキテクチャに依存しない結果を生成する。

アルゴリズムまたは振る舞い

1. sampler->lt を参照し、前回のスキャン位置から次のオフセットを特定する。 2. blockno、tupoffset、seed を hashinput 配列に格納する。 3. tupoffset から maxoffset までループし、各タプルに対してハッシュ値を計算する。 4. 計算されたハッシュ値が sampler->cutoff 未満であれば、そのタプルを選択対象としてループを終了する。 5. ブロックの末尾に達した場合は InvalidOffsetNumber を返し、それ以外の場合は選択された tupoffset を sampler->lt に保存して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/tablesample/bernoulli.c 内の tsm_bernoulli_handler において、TsmRoutine の NextSampleTuple フィールドに登録される。
  • **呼び出し先**: hash_any（ハッシュ計算用）。

関連データ構造

  • BernoulliSamplerData: スキャン中の状態（カットオフ値、シード値、現在のオフセット lt）を保持する。
  • SampleScanState: サンプリングスキャンの実行状態を管理する構造体。

失敗モードと境界条件

  • ブロック内にサンプリング条件を満たすタプルが存在しない場合、InvalidOffsetNumber を返す。
  • tupoffset が maxoffset を超えた場合、スキャン終了とみなして InvalidOffsetNumber を返す。

保守担当者向けメモ

  • 本関数はタプルごとにハッシュ計算を行うため、全ページを走査する。サンプリング率が高い場合のパフォーマンスについては use_pagemode の利用を検討すること。
  • ハッシュ計算の入力値はブロック内で一貫しているため、効率的な計算が可能となっている。
