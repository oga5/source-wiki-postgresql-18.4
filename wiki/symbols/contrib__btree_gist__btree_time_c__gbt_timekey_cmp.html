gbt_timekey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_time.c:96

目的

GiSTインデックスのソート処理において、2つの timeKEY 構造体を比較し、その順序関係を決定します。

シグネチャまたは形状

static int gbt_timekey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の timeKEY 構造体を保持する Nsrt 型へのポインタ。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されず、DirectFunctionCall2 経由で間接的に利用される可能性がある）。
  • **出力**:
  • int: a が b より小さい場合は負の数、等しい場合は 0、大きい場合は正の数を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体としてキャスト可能であり、その内部の t メンバが timeKEY 構造体として解釈可能である必要があります。
  • timeKEY 構造体の lower および upper フィールドには、有効な TimeADT 値が格納されている必要があります。

アルゴリズムまたは振る舞い

1. 入力された Nsrt ポインタから timeKEY 構造体を取り出します。 2. time_cmp 関数を呼び出し、両者の lower 値を比較します。 3. lower 値が異なる場合、その比較結果を返します。 4. lower 値が等しい場合、同様に upper 値を time_cmp で比較し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、contrib/btree_gist/btree_time.c:134 において参照されていることが確認できます。
  • **呼び出し先**:
  • time_cmp (PostgreSQLコアの time 型比較関数)
  • TimeADTGetDatumFast (マクロ)
  • DirectFunctionCall2 (PostgreSQLコアの関数呼び出しインターフェース)

関連データ構造

  • timeKEY: 時間範囲（lower, upper）を保持する構造体。
  • Nsrt: ソート処理時に使用される、キーとインデックス情報を保持するラッパー構造体。

失敗モードと境界条件

  • time_cmp が予期しない値を返す場合、その挙動は time_cmp の実装に依存します。
  • 入力ポインタが NULL の場合の動作は定義されていません。

保守担当者向けメモ

  • 本関数は btree_utils_num.c 等で提供される汎用的なソートロジックの一部として利用されます。
  • time_cmp は TimeADT 型の比較を行うため、タイムゾーンの考慮が必要な timetz 型を扱う場合は、事前に正規化された値が timeKEY に格納されていることを前提としています。
