pe_test_escape_func

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_escape/test_escape.c:41

目的

PostgreSQLの各種エスケープ関数をテストスイート内で統一的に扱うためのメタデータおよび実行インターフェースを定義する構造体です。

シグネチャまたは形状

typedef struct pe_test_escape_func
{
    const char *name;
    bool        reports_errors;
    bool        supports_only_valid;
    bool        supports_only_ascii_overlap;
    bool        supports_input_length;
    bool        (*escape) (PGconn *conn, PQExpBuffer target,
                           const char *unescaped, size_t unescaped_len,
                           PQExpBuffer escape_err);
} pe_test_escape_func;

入力と出力

  • **入力**: 構造体の各フィールドは、テスト対象となるエスケープ関数の特性（エラー報告能力、エンコーディング制約、長さ指定の可否）および関数ポインタ escape を保持します。
  • **出力**: escape 関数ポインタは、エスケープ処理の成否を bool で返し、結果を target バッファに格納します。

前提条件と不変条件

  • escape 関数ポインタは、PGconn 接続、出力用 PQExpBuffer、入力文字列とその長さ、およびエラー報告用 PQExpBuffer を引数に取ります。
  • supports_only_valid が true の場合、無効なエンコーディング入力に対しては --force-unsupported オプションがない限りテストがスキップされます。

アルゴリズムまたは振る舞い

この構造体は、pe_test_escape_funcs 配列内でテスト対象の関数群を定義するために使用されます。テスト実行時、test_one_vector_escape 関数がこの構造体を参照し、指定されたエスケープ関数を呼び出して結果を検証します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_escape/test_escape.c: pe_test_escape_funcs 配列の定義で使用。
  • test_one_vector_escape: テスト実行時にこの構造体を参照。
  • main ループ内: pe_test_escape_funcs 配列の要素として参照。
  • **呼び出し先**:
  • 構造体内の escape フィールドに代入された関数（escape_literal, escape_identifier 等）。

関連データ構造

  • pe_test_vector: テストケースとなる入力データ。
  • pe_test_config: テスト全体の構成設定。
  • PGconn, PQExpBuffer: libpq および PostgreSQL ユーティリティのデータ構造。

失敗モードと境界条件

  • reports_errors が true の場合、無効な入力に対してエスケープ関数がエラーを報告しないとテストが失敗します。
  • supports_only_ascii_overlap の制約に違反するエンコーディングが渡された場合、期待される挙動と一致しない可能性があります。

保守担当者向けメモ

  • 新しいエスケープ関数をテスト対象に追加する場合は、pe_test_escape_funcs 配列に本構造体のインスタンスを追加してください。
  • escape 関数ポインタの実装は、メモリ安全性を担保するため、NEVER_ACCESS_STR を用いた境界チェックと整合させる必要があります。
