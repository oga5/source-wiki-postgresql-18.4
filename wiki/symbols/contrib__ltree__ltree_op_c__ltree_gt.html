ltree_gt

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:122

目的

2つの ltree 型の値が「より大きい」関係にあるかを判定し、SQLの比較演算子 > をサポートする。

シグネチャまたは形状

Datum ltree_gt(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS マクロを通じて渡される2つの ltree 型の Datum。
  • 出力: bool 型（Datum として返却）。第1引数が第2引数より大きい場合に true、それ以外は false を返す。

前提条件と不変条件

  • RUNCMP マクロが定義されている環境で実行されること。
  • RUNCMP は内部的に ltree_compare を呼び出し、比較結果を res 変数に格納する前提となっている。

アルゴリズムまたは振る舞い

1. RUNCMP マクロを実行し、引数として渡された2つの ltree を比較する。 2. 比較結果 res が 0 より大きい（res > 0）かどうかを評価する。 3. PG_RETURN_BOOL を使用して、評価結果をブール値として呼び出し元へ返す。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の SQL エンジン（演算子 > の実行時）。
  • 呼び出し先: RUNCMP マクロ（内部で ltree_compare を呼び出す）。
  • 注: ソースコード上の明示的な呼び出し元・先情報は、提供された抜粋の範囲内では上記マクロ利用のみである。

関連データ構造

  • ltree: 階層構造を持つラベルパスを保持するデータ型。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • 入力が有効な ltree 構造体でない場合、ltree_compare の挙動に依存する。
  • 比較対象の ltree が NULL の場合、PostgreSQL の関数呼び出し規約に従い NULL が返される可能性がある。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(ltree_gt) により SQL 関数として登録されている。
  • 比較ロジックの変更が必要な場合は、ltree_compare を修正すること。本関数自体は res > 0 の判定のみを行う薄いラッパーである。
