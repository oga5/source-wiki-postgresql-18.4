in_range_float4_float8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/float.c:1102

目的

ウィンドウ関数のフレーム境界計算において、float4 型の値が base 値から指定された offset（float8）の範囲内にあるかどうかを判定します。

シグネチャまたは形状

Datum in_range_float4_float8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • val (float4): 比較対象の値。
  • base (float4): 基準となる値。
  • offset (float8): 許容範囲のオフセット。
  • sub (bool): true の場合 base - offset、false の場合 base + offset を計算。
  • less (bool): true の場合 val <= sum、false の場合 val >= sum を判定。
  • **出力**:
  • bool (Datum): 範囲内であれば true、そうでなければ false。

前提条件と不変条件

  • offset は非負である必要があります。負の値や NaN が渡された場合はエラーとなります。
  • NaN の扱いは float8_cmp_internal に準拠し、NaN は非 NaN よりも大きいものとして扱われます。

アルゴリズムまたは振る舞い

1. **入力検証**: offset が NaN または負の場合、ERRCODE_INVALID_PRECEDING_OR_FOLLOWING_SIZE でエラーを報告します。 2. **NaN の処理**:

  • val が NaN の場合: base も NaN なら true、そうでなければ !less を返します。
  • base が NaN の場合: less を返します。

    • 3. **無限大の特殊ケース**: offset と base が共に無限大で、かつ計算結果が NaN になるような組み合わせ（例: +inf - inf）の場合、すべての値が範囲内にあるとみなして true を返します。 4. **範囲判定**: sub フラグに基づき sum = base +/- offset を計算し、less フラグに応じて val <= sum または val >= sum を評価して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は特定されていません。SQLレベルのウィンドウ関数から呼び出されることが想定されます。
  • **呼び出し先**: ereport（エラー報告）、isnan、isinf（標準数学関数）、および PG_RETURN_BOOL 等のPostgreSQLマクロ。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約で使用される引数構造体。

失敗モードと境界条件

  • offset が負または NaN の場合: ERRCODE_INVALID_PRECEDING_OR_FOLLOWING_SIZE エラー。
  • base または offset が無限大の場合: FPUの挙動に依存し、適切に符号付き無限大として計算されます。

保守担当者向けメモ

  • 本関数は in_range_float8_float8 と類似のロジックを共有しています。修正時は両者の整合性を確認してください。
  • NaN の比較順序はPostgreSQLの標準的なソート順序に従っています。
