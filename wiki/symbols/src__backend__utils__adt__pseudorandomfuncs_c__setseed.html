setseed

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pseudorandomfuncs.c:61

目的

現在のバックエンドプロセスにおける疑似乱数生成器（PRNG）の状態を、指定されたシード値に基づいて初期化します。

シグネチャまたは形状

Datum setseed(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_FLOAT8(0) を介して float8 型のシード値を受け取ります。
  • **出力**: なし（PG_RETURN_VOID() を返却）。

前提条件と不変条件

  • シード値は -1 から 1 の範囲内（両端を含む）である必要があります。
  • NaN（非数）は許可されません。
  • 成功した場合、内部状態 prng_state が更新され、グローバルなフラグ prng_seed_set が true に設定されます。

アルゴリズムまたは振る舞い

1. 入力された seed が範囲外（-1 未満または 1 超）であるか、isnan(seed) であるかを検証します。条件に合致する場合、ERRCODE_INVALID_PARAMETER_VALUE エラーを発生させます。 2. 検証を通過した場合、pg_prng_fseed を呼び出し、prng_state を指定されたシードで初期化します。 3. prng_seed_set を true に設定し、乱数生成器が明示的にシードされたことを記録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/variable.c:672 から呼び出されます。
  • その他、SQLインターフェースを介してユーザーから直接呼び出される可能性があります。
  • **呼び出し先**:
  • pg_prng_fseed（common/pg_prng.h で定義された内部関数）を呼び出します。

関連データ構造

  • prng_state: プロセス単位で保持される乱数生成器の状態構造体。
  • prng_seed_set: シードが設定済みであることを示す静的フラグ。

失敗モードと境界条件

  • **範囲外エラー**: シード値が [-1, 1] の範囲外、または NaN の場合に ereport(ERROR, ...) が発生し、トランザクションが中断されます。

保守担当者向けメモ

  • 本関数はプロセス固有の prng_state を変更します。セッションをまたぐ共有や並列クエリ間での同期は行われないため、並列実行環境下での挙動に注意してください。
  • pg_prng_fseed の詳細な仕様については src/include/common/pg_prng.h を参照してください。
