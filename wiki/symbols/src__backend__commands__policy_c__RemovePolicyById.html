RemovePolicyById

定義

  • 種別: function
  • 定義箇所: src/backend/commands/policy.c:331

目的

指定されたOIDを持つ行レベルセキュリティ（RLS）ポリシーをシステムカタログ pg_policy から削除し、関連するリレーショナルキャッシュを無効化する。

シグネチャまたは形状

void RemovePolicyById(Oid policy_id)

入力と出力

  • **入力**: policy_id (削除対象ポリシーのOID)
  • **出力**: なし（成功時はカタログ更新とキャッシュ無効化を実行）

前提条件と不変条件

  • pg_policy カタログに対する RowExclusiveLock を保持する。
  • 対象ポリシーが属するテーブルに対して AccessExclusiveLock を取得する。
  • 削除対象のポリシーが存在しなければならない。
  • 対象テーブルがシステムカタログである場合、allowSystemTableMods が有効でない限りエラーとなる。

アルゴリズムまたは振る舞い

1. pg_policy カタログをスキャンし、指定された policy_id に一致するタプルを検索する。 2. タプルが見つからない場合はエラーを発生させる。 3. ポリシーが属するテーブル（polrelid）を AccessExclusiveLock モードでオープンする。 4. 対象がテーブルまたはパーティションテーブルであることを検証する。 5. システムテーブルへの変更制限を確認する。 6. CatalogTupleDelete を使用して pg_policy からタプルを削除する。 7. CacheInvalidateRelcache を呼び出し、対象テーブルのキャッシュを無効化する。 8. オープンしたリレーションとカタログをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c:1432 からの呼び出しが確認されている。
  • **呼び出し先**: table_open, systable_beginscan, systable_getnext, CatalogTupleDelete, CacheInvalidateRelcache, table_close 等のアクセスメソッドおよびカタログ操作関数を呼び出す。

関連データ構造

  • pg_policy: ポリシー定義を保持するシステムカタログ。
  • Form_pg_policy: pg_policy タプルの構造体。
  • Relation: テーブルおよびカタログへのアクセスに使用されるリレーション記述子。

失敗モードと境界条件

  • **ポリシー不在**: 指定したOIDのポリシーが存在しない場合、elog(ERROR) で終了する。
  • **オブジェクト型不一致**: 対象がテーブルまたはパーティションテーブルでない場合、ERRCODE_WRONG_OBJECT_TYPE でエラーとなる。
  • **権限不足**: システムカタログに対して変更を試みた場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーとなる。

保守担当者向けメモ

  • 本関数はポリシーの削除のみを担う。依存関係の整合性については、呼び出し元（dependency.c 等）が適切に管理していることを前提としている。
  • CacheInvalidateRelcache の呼び出しは、プランナが古いポリシー定義を参照し続けることを防ぐために必須である。
