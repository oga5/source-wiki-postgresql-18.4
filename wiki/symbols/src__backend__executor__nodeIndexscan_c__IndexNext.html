IndexNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:79

目的

インデックススキャンノードから次のタプルを順次取得し、必要に応じてインデックスの損失性（lossy）判定に基づく再チェックを行う。

シグネチャまたは形状

static TupleTableSlot *IndexNext(IndexScanState *node)

入力と出力

  • **入力**: IndexScanState *node（インデックススキャンの実行状態を保持する構造体）
  • **出力**: TupleTableSlot *（取得したタプル、またはスキャン終了時の空スロット）

前提条件と不変条件

  • node->ss.ps.state が有効な実行状態を指していること。
  • node->iss_ScanDesc が初期化されていない場合、本関数内で index_beginscan を用いて初期化される。
  • ランタイムキーが存在する場合、node->iss_RuntimeKeysReady が真であるか、キーが不要である必要がある。

アルゴリズムまたは振る舞い

1. **スキャン方向の決定**: ScanDirectionCombine を使用し、プランの方向と実行時の方向を統合する。 2. **スキャン記述子の初期化**: scandesc が未割り当ての場合、index_beginscan で開始し、必要に応じて index_rescan を実行する。 3. **タプル取得ループ**: index_getnext_slot を呼び出し、タプルを順次取得する。

  • CHECK_FOR_INTERRUPTS() を実行し、割り込みを監視する。
  • **再チェック**: scandesc->xs_recheck が真（損失性インデックス）の場合、ExecQualAndReset を用いてインデックス条件を再評価する。条件を満たさない場合はフィルタリングし、次のタプルへ進む。

    • 4. **終了処理**: スキャンが終了した場合は node->iss_ReachedEnd を true に設定し、ExecClearTuple でスロットをクリアして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecIndexScan（メインエントリーポイント）、および src/backend/executor/nodeIndexscan.c 内の実行メソッドテーブル。
  • **呼び出し先**: ScanDirectionCombine, index_beginscan, index_rescan, index_getnext_slot, CHECK_FOR_INTERRUPTS, ExecQualAndReset, InstrCountFiltered2, ExecClearTuple。

関連データ構造

  • IndexScanState: インデックススキャンの実行状態を管理。
  • IndexScanDesc: インデックスアクセスメソッドの走査状態。
  • TupleTableSlot: タプルデータを保持するコンテナ。
  • ExprContext: 式評価のためのコンテキスト。

失敗モードと境界条件

  • **スキャン終了**: インデックスから取得できるタプルがなくなった場合、ExecClearTuple を返し、iss_ReachedEnd を更新する。
  • **再チェック失敗**: インデックス条件の再評価で不一致となったタプルは、InstrCountFiltered2 でカウントされ、スキップされる。

保守担当者向けメモ

  • 本関数は通常のインデックススキャン用である。ORDER BY 句を伴う再ソートが必要な場合は IndexNextWithReorder が使用されるため、用途に応じて使い分けること。
  • 並列クエリ実行時、シリアル実行へフォールバックする際の初期化ロジックが含まれている点に注意。
