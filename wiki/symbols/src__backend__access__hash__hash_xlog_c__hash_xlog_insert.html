hash_xlog_insert

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash_xlog.c:124

目的

ハッシュインデックスへのタプル挿入操作をWALリプレイ時に再適用（Redo）する。

シグネチャまたは形状

static void hash_xlog_insert(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）
  • **出力**: なし（副作用としてバッファの状態を更新）

前提条件と不変条件

  • リプレイ処理中であるため、他のプロセスによる同時アクセスは発生しない前提で動作する。
  • 挿入先のデータページとメタページの更新は、それぞれ独立したバッファ操作として行われる。

アルゴリズムまたは振る舞い

1. WALレコードから xl_hash_insert 構造体を取得する。 2. **データページの更新**:

  • XLogReadBufferForRedo で対象ページを読み込む。
  • BLK_NEEDS_REDO の場合、PageAddItem を用いてタプルをページに追加する。
  • ページにLSNを書き込み、バッファをダーティにする。

    • 3. **メタページの更新**:

  • 2番目のブロック（メタページ）を読み込む。
  • BLK_NEEDS_REDO の場合、hashm_ntuples をインクリメントする。
  • ページにLSNを書き込み、バッファをダーティにする。

    • 4. 各バッファは処理終了後に UnlockReleaseBuffer で解放される。

呼び出し元と呼び出し先

  • **呼び出し元**: hash_redo（src/backend/access/hash/hash_xlog.c:1080 付近のディスパッチャより呼び出される）
  • **呼び出し先**: XLogRecGetData, XLogReadBufferForRedo, XLogRecGetBlockData, BufferGetPage, PageAddItem, PageSetLSN, MarkBufferDirty, BufferIsValid, UnlockReleaseBuffer, HashPageGetMeta, elog

関連データ構造

  • xl_hash_insert: WALレコード内に格納された挿入情報。
  • HashMetaPage: ハッシュインデックスのメタデータ構造体。

失敗モードと境界条件

  • PageAddItem が InvalidOffsetNumber を返した場合、PANIC レベルでログを出力し、リカバリを停止する。

保守担当者向けメモ

  • 本関数は冪等（idempotent）である必要がある。
  • 通常のインデックス挿入時とは異なり、メタページの更新時にデータページに対するロックを保持し続ける必要はない。
