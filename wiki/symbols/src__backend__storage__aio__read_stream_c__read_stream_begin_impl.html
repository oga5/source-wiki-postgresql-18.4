read_stream_begin_impl

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/read_stream.c:537

目的

ReadStream 構造体を初期化し、リレーションデータの効率的な先読み（Look-ahead）を行うための環境を構築する。

シグネチャまたは形状

static ReadStream *read_stream_begin_impl(
    int flags,
    BufferAccessStrategy strategy,
    Relation rel,
    SMgrRelation smgr,
    char persistence,
    ForkNumber forknum,
    ReadStreamBlockNumberCB callback,
    void *callback_private_data,
    size_t per_buffer_data_size
);

入力と出力

  • **入力**:
  • flags: ストリームの動作モード（READ_STREAM_MAINTENANCE, READ_STREAM_USE_BATCHING, READ_STREAM_SEQUENTIAL, READ_STREAM_FULL）。
  • strategy: バッファアクセス戦略。
  • rel, smgr: 対象リレーション情報。
  • callback: 次のブロック番号を取得するためのコールバック関数。
  • per_buffer_data_size: 各バッファに関連付ける付加データのサイズ。
  • **出力**: 初期化済みの ReadStream 構造体へのポインタ。

前提条件と不変条件

  • max_ios は PG_INT16_MAX を超えないこと。
  • max_pinned_buffers は GetAccessStrategyPinLimit および GetPinLimit（または GetLocalPinLimit）によって制限される。
  • queue_size は max_pinned_buffers + 1 であり、バッファの循環利用においてヘッドとテールの衝突を避けるための空きを確保する。

アルゴリズムまたは振る舞い

1. **I/O 並行数の決定**: テーブルスペース設定やカタログリレーションの判定に基づき、max_ios を算出する。 2. **バッファ制限の計算**: io_combine_limit を考慮し、ピン留め可能な最大バッファ数を決定する。 3. **メモリ確保**: ReadStream 本体、バッファ配列、InProgressIO 配列、および per_buffer_data を単一のメモリチャンクとして palloc で確保する。 4. **ストリーム設定**: 同期/非同期モードの判定、コールバックの登録、および初期先読み距離（distance）の設定を行う。READ_STREAM_FULL 指定時は初期距離を最大化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • read_stream_begin_relation
  • read_stream_begin_smgr_relation
  • **呼び出し先**:
  • get_tablespace_maintenance_io_concurrency / get_tablespace_io_concurrency
  • GetAccessStrategyPinLimit
  • GetLocalPinLimit / GetPinLimit
  • SmgrIsTemp
  • palloc

関連データ構造

  • ReadStream: ストリームの状態を保持するメイン構造体。
  • InProgressIO: 進行中の I/O 操作を管理する構造体。
  • BufferAccessStrategy: バッファ管理の制約。

失敗モードと境界条件

  • max_ios が 0 の場合、強制的に 1 に設定し、同期 I/O モードへ切り替える。
  • per_buffer_data_size が 0 の場合、関連データ領域は確保されない。
  • システムのバッファ数が極端に少ない場合、max_pinned_buffers を最低 1 に補正する。

保守担当者向けメモ

  • 本関数は ReadStream のメモリレイアウトを決定する重要な箇所である。queue_size や queue_overflow の計算を変更する場合は、メモリ確保時のオフセット計算と MAXALIGN の整合性に注意すること。
  • GUC 設定（io_combine_limit 等）はストリーム開始時にキャプチャされるため、ストリーム生存期間中の GUC 変更は影響しない。
