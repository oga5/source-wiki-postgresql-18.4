PgStat_StatInjFixedEntry

定義

  • 種別: struct
  • 定義箇所: src/test/modules/injection_points/injection_stats_fixed.c:27

目的

インジェクションポイント機能における「固定数統計（fixed-numbered statistics）」の各項目を保持するためのデータ構造です。共有メモリ上で累積された統計情報を格納し、スナップショット生成やリセット処理の単位として利用されます。

シグネチャまたは形状

typedef struct PgStat_StatInjFixedEntry
{
	PgStat_Counter numattach;	/* number of points attached */
	PgStat_Counter numdetach;	/* number of points detached */
	PgStat_Counter numrun;		/* number of points run */
	PgStat_Counter numcached;	/* number of points cached */
	PgStat_Counter numloaded;	/* number of points loaded */
	TimestampTz stat_reset_timestamp;
} PgStat_StatInjFixedEntry;

入力と出力

本構造体は直接的な関数ではありませんが、統計情報の集計結果を保持するコンテナとして機能します。

  • **入力**: 各種カウンタのインクリメントおよびリセット時のタイムスタンプ更新。
  • **出力**: SQL関数 injection_points_stats_fixed を通じた統計情報の可視化。

前提条件と不変条件

  • 各カウンタ（numattach 等）は PgStat_Counter 型であり、非負の整数として累積されます。
  • stat_reset_timestamp は統計がリセットされた時点の時刻を保持します。

アルゴリズムまたは振る舞い

  • **統計の保持**: インジェクションポイントのライフサイクル（attach, detach, run, cached, loaded）に関連するイベント数を個別に追跡します。
  • **スナップショット**: 共有メモリ上の値から reset_offset を差し引くことで、リセット以降の純粋な統計値を算出する際のデータソースとなります。
  • **リセット**: リセット処理時には、現在の統計値が reset_offset にコピーされ、以降の計算の基準値として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • injection_stats_fixed.c 内の統計更新処理（pgstat_report_inj_fixed）およびスナップショット生成処理（injection_points_stats_fixed）から参照されます。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • PgStatShared_InjectionPointFixed: 本構造体をメンバとして保持し、共有メモリ上でのロック管理と併せて統計を管理する構造体。

失敗モードと境界条件

  • **オーバーフロー**: PgStat_Counter の上限に達した場合の挙動は pgstat フレームワークの仕様に依存します。
  • **ロック競合**: 共有メモリ上の統計更新は LWLock で保護されていますが、高頻度な更新が発生する環境ではロック競合によるパフォーマンス低下の可能性があります。

保守担当者向けメモ

  • 本構造体は injection_stats_fixed.c 内でのみ使用される固定統計用のスキーマです。
  • 統計項目を追加・変更する場合は、本構造体の定義と併せて、スナップショット生成およびリセット処理のロジックを更新する必要があります。
