appendStringInfoStringQuoted

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/stringinfo_mb.c:33

目的

マルチバイト文字を考慮しつつ、文字列をシングルクォートで囲み、内部のシングルクォートをエスケープ（二重化）して StringInfo バッファに追加する。

シグネチャまたは形状

void appendStringInfoStringQuoted(StringInfo str, const char *s, int maxlen)

入力と出力

  • **入力**:
  • str: 追加先の StringInfo 構造体へのポインタ。
  • s: 追加対象のヌル終端文字列。
  • maxlen: 切り詰めるバイト数。負の値を指定すると切り詰めを行わない。
  • **出力**: なし（str が指すバッファを直接更新する）。

前提条件と不変条件

  • str は NULL であってはならない（Assert で検証）。
  • maxlen が指定された場合、マルチバイト文字の境界を壊さないよう pg_mbcliplen が適用される。
  • 処理終了後、バッファは必ず ' で閉じられる。

アルゴリズムまたは振る舞い

1. **切り詰め処理**: maxlen >= 0 かつ maxlen < strlen(s) の場合、pg_mbcliplen で安全な長さを算出し、一時メモリにコピーを作成する。 2. **開始**: バッファの先頭に ' を追加する。 3. **エスケープ**: 文字列内の ' を strchr で検索し、見つかるたびにその直前までの文字列と ' 自体をバッファに追加する。この際、' を二重化するために検索位置を調整する。 4. **終了**: 残りの文字列を追加する。切り詰めが発生した場合は ... を付与し、最後に ' を追加して閉じる。 5. **後処理**: 一時メモリを使用した場合、pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/params.c
  • src/backend/tcop/postgres.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:

  • pfree (メモリ解放)

関連データ構造

  • StringInfo: PostgreSQLの動的文字列バッファ管理構造体。

失敗モードと境界条件

  • SQLインジェクションに対する完全な保護機能ではないため、信頼されたコンテキストでの使用が推奨される。

保守担当者向けメモ

  • 本関数は common/stringinfo.c に依存せず、マルチバイト対応を完結させている。
  • maxlen はバイト単位である点に注意すること。
