pg_strong_random

定義

  • 種別: function
  • 定義箇所: src/port/pg_strong_random.c:63

目的

暗号学的に安全な乱数（CSPRNG）を生成し、認証用ソルトやクエリキャンセルキーなどの予測不可能性が求められる用途に提供する。

シグネチャまたは形状

bool pg_strong_random(void *buf, size_t len);

入力と出力

  • **入力**:
  • buf: 乱数を格納するバッファへのポインタ。
  • len: 生成する乱数のバイト数。
  • **出力**:
  • 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • 呼び出し元は、戻り値が false の場合に適切なエラーハンドリングを行う必要がある。
  • 本関数はPostgreSQLの起動初期段階でも呼び出される可能性があるため、elog() や palloc() といったバックエンド固有のインフラに依存してはならない。

アルゴリズムまたは振る舞い

OpenSSLビルド環境下では以下の手順で動作する： 1. NUM_RAND_POLL_RETRIES（8回）を上限として、RAND_status() でCSPRNGのシード状態を確認する。 2. シードが不十分な場合、RAND_poll() を呼び出してシードの補完を試みる。 3. 最終的に RAND_bytes(buf, len) を呼び出し、その結果（成功なら1）を関数の戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgcrypto.c
  • contrib/pgcrypto/pgp-encrypt.c
  • contrib/pgcrypto/pgp-pubenc.c
  • contrib/pgcrypto/pgp-s2k.c
  • contrib/pgcrypto/px-crypt.c
  • src/backend/access/transam/xlog.c
  • src/backend/libpq/auth-scram.c
  • **呼び出し先**:
  • RAND_status() (OpenSSL)
  • RAND_poll() (OpenSSL)
  • RAND_bytes() (OpenSSL)

関連データ構造

  • NUM_RAND_POLL_RETRIES: シード状態を確認するための試行回数定数（8）。

失敗モードと境界条件

  • OpenSSL環境において、RAND_poll() が規定回数試行してもシード生成に失敗した場合、RAND_bytes() が失敗し、本関数も false を返す。
  • 乱数生成の失敗はセキュリティ上の脆弱性に直結するため、呼び出し元での戻り値チェックが必須である。

保守担当者向けメモ

  • 本関数はプラットフォーム抽象化レイヤーの一部であり、Windows環境では CryptGenRandom、その他環境では /dev/urandom を使用するパスが存在する（ソースコードのビルド条件による）。
  • OpenSSL使用時のシード不足に対する RAND_poll() の挙動は、古いOpenSSLバージョンとの互換性を考慮している。
