ExecScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execScan.c:46

目的

スキャンノードにおけるタプル取得、フィルタリング（Qual）、およびプロジェクション処理を抽象化し、共通化する。

シグネチャまたは形状

TupleTableSlot *ExecScan(ScanState *node, ExecScanAccessMtd accessMtd, ExecScanRecheckMtd recheckMtd)

入力と出力

  • **入力**:
  • node: スキャンノードの状態を保持する ScanState 構造体。
  • accessMtd: ストレージ層からタプルを取得するためのコールバック関数。
  • recheckMtd: EPQ（EvalPlanQual）再チェック用のコールバック関数。
  • **出力**:
  • 処理済みの TupleTableSlot を返す。

前提条件と不変条件

  • node->ps.state->es_epq_active から現在の EPQ 状態を取得可能であること。
  • node->ps.qual および node->ps.ps_ProjInfo が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. node から es_epq_active、qual、projInfo を抽出する。 2. 抽出した情報を引数として ExecScanExtended を呼び出し、実際のタプル取得から射影までの処理を委譲する。 3. ExecScanExtended の戻り値をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nodeBitmapHeapscan.c
  • nodeCtescan.c
  • nodeForeignscan.c
  • その他、各スキャンノードの実装（nodeSeqscan.c 等）。
  • **呼び出し先**:
  • ExecScanExtended（内部で呼び出し）。

関連データ構造

  • ScanState: スキャンノードの実行状態を保持。
  • TupleTableSlot: 取得したタプルを格納するコンテナ。
  • EPQState: 同時実行制御（MVCC）下での再チェック状態。
  • ExprState: フィルタリング条件（Qual）。
  • ProjectionInfo: 射影処理に必要な情報。

失敗モードと境界条件

  • 提供されたソースコード上では、特定の失敗モードや境界条件に関する明示的な記述はない。

保守担当者向けメモ

  • 本関数は ExecScanExtended への単なるラッパーである。詳細なロジックは ExecScanExtended を参照すること。
  • EPQ 状態の管理は ExecScanFetch（execScan.h 定義）が担うため、EPQ 関連の不具合調査時はそちらを確認すること。
