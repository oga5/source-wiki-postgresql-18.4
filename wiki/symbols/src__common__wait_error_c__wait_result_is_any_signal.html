wait_result_is_any_signal

定義

  • 種別: function
  • 定義箇所: src/common/wait_error.c:120

目的

プロセス終了ステータスが、シグナルによる終了、またはシェルが定義する特定の終了コード（コマンド未検出や実行不可など）に該当するかを判定します。

シグネチャまたは形状

bool wait_result_is_any_signal(int exit_status, bool include_command_not_found)

入力と出力

  • exit_status (int): wait(2) や waitpid(2) から返されるプロセス終了ステータス。
  • include_command_not_found (bool): true の場合、終了コード 126（実行不可）および 127（コマンド未検出）をシグナル相当の異常終了とみなします。
  • 戻り値 (bool): 条件に合致すれば true、それ以外は false を返します。

前提条件と不変条件

  • exit_status は <sys/wait.h> のマクロ（WIFSIGNALED 等）で解釈可能な形式である必要があります。
  • POSIX シェルの慣習に基づき、シグナル終了時は 128 + シグナル番号 が終了コードとして扱われることを前提としています。

アルゴリズムまたは振る舞い

1. WIFSIGNALED(exit_status) が真であれば、シグナルにより終了したと判断し true を返します。 2. WIFEXITED(exit_status) が真の場合、終了コードを評価します。

  • include_command_not_found が true の場合、終了コードが 125 を超えていれば true を返します。
  • include_command_not_found が false の場合、終了コードが 128 を超えていれば true を返します。

    • 3. 上記いずれにも該当しない場合は false を返します。

関連データ構造

  • 特になし（標準的な int 型の終了ステータスを使用）。

失敗モードと境界条件

  • Windows 環境など、WIFSIGNALED 等のマクロがプラットフォーム依存の挙動を示す場合、その実装に依存します。
  • include_command_not_found のフラグ設定により、126 や 127 といったシェル特有の終了コードの扱いが変化します。

保守担当者向けメモ

  • 本関数はシェル経由のプロセス実行（system(3) 等）を想定した判定ロジックです。直接実行されたプロセスの終了コードを評価する際には、フラグの指定に注意してください。
