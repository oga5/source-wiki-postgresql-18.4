TransactionIdLatest

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:344

目的

メインのトランザクションIDと、それに付随するサブトランザクションIDのリストの中から、論理的に最も新しい（最新の）トランザクションIDを特定する。

シグネチャまたは形状

TransactionId TransactionIdLatest(TransactionId mainxid, int nxids, const TransactionId *xids)

入力と出力

  • **入力**:
  • mainxid: メイントランザクションのID。
  • nxids: xids 配列に含まれるサブトランザクションIDの数。
  • xids: サブトランザクションIDの配列。
  • **出力**:
  • TransactionId: 入力されたID群の中で、TransactionIdPrecedes に基づいて最も新しいと判定されたID。

前提条件と不変条件

  • トランザクションIDは32ビットの循環値であるため、比較には必ず TransactionIdPrecedes を使用する必要がある。
  • xids 配列がソートされているという仮定は置かず、すべての要素を走査する。

アルゴリズムまたは振る舞い

1. result を mainxid で初期化する。 2. xids 配列を末尾から先頭に向かって走査する（nxids からデクリメント）。 3. 各要素 xids[nxids] と現在の result を TransactionIdPrecedes(result, xids[nxids]) で比較する。 4. xids[nxids] の方が新しい場合、result を更新する。 5. 全ての走査終了後、最終的な result を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c: latestXid の取得に使用。
  • src/backend/access/transam/xact.c: latestXid や max_xid の取得に使用。
  • src/backend/storage/ipc/procarray.c: max_xid の取得に使用。
  • **呼び出し先**:
  • TransactionIdPrecedes (マクロまたは関数)

関連データ構造

  • TransactionId: トランザクションを識別する32ビット整数型。

失敗モードと境界条件

  • nxids が 0 の場合、ループは実行されず mainxid がそのまま返される。
  • トランザクションIDのラップアラウンドが発生している場合、TransactionIdPrecedes の仕様に依存する。

保守担当者向けメモ

  • 本関数はパフォーマンスクリティカルな箇所で呼び出されることがあるが、現在の実装は配列の全走査を行うため、配列サイズが大きい場合は注意が必要である。
  • xids 配列がソート済みであるという前提を置かないことで、堅牢性を維持している。
