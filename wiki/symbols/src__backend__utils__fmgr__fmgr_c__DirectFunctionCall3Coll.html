DirectFunctionCall3Coll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:833

目的

3つの引数を取る関数を、FmgrInfo構造体の初期化を介さずに直接呼び出すための軽量なラッパー関数です。

シグネチャまたは形状

Datum DirectFunctionCall3Coll(PGFunction func, Oid collation, Datum arg1, Datum arg2, Datum arg3)

入力と出力

  • **入力**:
  • func: 呼び出す関数のポインタ（PGFunction）。
  • collation: 関数実行時に使用する照合順序（Oid）。
  • arg1, arg2, arg3: 関数に渡す3つの引数（Datum）。
  • **出力**:
  • 関数の実行結果（Datum）。

前提条件と不変条件

  • 呼び出される関数は、fcinfo構造体を通じて引数を受け取る標準的なPostgreSQLの関数インターフェースに準拠している必要があります。
  • 呼び出し元は、戻り値がNULLにならないことを期待しています。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFO(fcinfo, 3) マクロを使用して、スタック上に3引数用のFunctionCallInfoBaseDataを確保します。 2. InitFunctionCallInfoData を呼び出し、fcinfo を初期化します。 3. 提供された3つの引数（arg1, arg2, arg3）をfcinfo->args配列に格納し、isnullフラグをfalseに設定します。 4. 関数ポインタ func を fcinfo を引数として実行します。 5. 実行後の fcinfo->isnull を確認し、関数がNULLを返した場合は elog(ERROR) を発生させます。 6. 正常終了時に結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/extension.c (複数箇所)
  • **呼び出し先**:
  • 引数として渡された func (関数ポインタ)

関連データ構造

  • FunctionCallInfoBaseData: 関数呼び出しのコンテキストを保持する構造体。
  • Datum: PostgreSQLの汎用データ型。
  • PGFunction: 関数ポインタ型。

失敗モードと境界条件

  • **NULL戻り値**: 呼び出された関数が fcinfo->isnull を true に設定して戻った場合、ERROR レベルのログを出力して処理を中断します。

保守担当者向けメモ

  • 本関数は FmgrInfo を必要としない単純な関数呼び出しに最適化されています。ポリモーフィック関数など、FmgrInfo の情報が必要な関数には使用できません。
  • src/include/fmgr.h にて、InvalidOid を照合順序として渡す簡易マクロが定義されています。
