PLpgSQL_resolve_option

定義

  • 種別: enum
  • 定義箇所: src/pl/plpgsql/src/plpgsql.h:184

目的

PL/pgSQL において、識別子が変数名とテーブル列名のどちらを指すか曖昧な場合の解決方針を定義する。

シグネチャまたは形状

typedef enum PLpgSQL_resolve_option
{
    PLPGSQL_RESOLVE_ERROR,
    PLPGSQL_RESOLVE_VARIABLE,
    PLPGSQL_RESOLVE_COLUMN,
} PLpgSQL_resolve_option;

入力と出力

本シンボルは列挙型であり、入力や出力を持つ関数ではない。設定値として使用される。

前提条件と不変条件

  • 識別子の解決時に、PL/pgSQL 変数とテーブル列の優先順位を決定するために使用される。

アルゴリズムまたは振る舞い

  • PLPGSQL_RESOLVE_ERROR: 曖昧な場合にエラーを発生させる。
  • PLPGSQL_RESOLVE_VARIABLE: テーブル列よりも PL/pgSQL 変数を優先する。
  • PLPGSQL_RESOLVE_COLUMN: PL/pgSQL 変数よりもテーブル列を優先する。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/plpgsql.h:984 にて PLpgSQL_resolve_option resolve_option; として構造体メンバで使用されている。
  • 呼び出し先: 本シンボル自体は列挙型定義であり、直接的な呼び出し先は存在しない。

関連データ構造

  • PLpgSQL_execstate などの実行時状態を保持する構造体内で、解決オプションを保持するために使用される可能性がある（ソースコード上の参照箇所より）。

失敗モードと境界条件

  • PLPGSQL_RESOLVE_ERROR が設定されている場合、名前の衝突が発生すると実行時またはコンパイル時にエラーがスローされる。

保守担当者向けメモ

  • 本列挙型は、PL/pgSQL の名前解決ロジックの挙動を制御する重要な設定値である。変更を加える際は、名前解決を行うコンパイラ（pl_comp.c）および実行エンジン（pl_exec.c）への影響を考慮すること。
