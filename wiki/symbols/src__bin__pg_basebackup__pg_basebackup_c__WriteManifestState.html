WriteManifestState

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:73

目的

バックアップマニフェストの書き込み処理において、対象ファイル名とファイルポインタを保持するために使用されます。

シグネチャまたは形状

typedef struct WriteManifestState
{
	char		filename[MAXPGPATH];
	FILE	   *file;
} WriteManifestState;

入力と出力

本構造体は、バックアップ処理中にマニフェストファイルの状態を管理するためのコンテナとして機能します。

前提条件と不変条件

  • filename は MAXPGPATH の長さを超えないパスである必要があります。
  • file は書き込み用にオープンされた有効な FILE ポインタである必要があります。

アルゴリズムまたは振る舞い

バックアップストリームの処理過程において、マニフェスト情報の書き込み先を追跡します。具体的には、コールバック関数内で callback_data として渡され、ファイル操作のコンテキストとして利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_basebackup.c 内の関数（1701行目でインスタンス化、1720行目で callback_data としてキャストして参照）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • WriteDataCallback: 本構造体をコンテキストとして利用する可能性のあるコールバック型。

失敗モードと境界条件

  • file が NULL の状態でファイル操作を行うとクラッシュの原因となります。
  • filename が MAXPGPATH を超える場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本構造体は pg_basebackup におけるバックアップマニフェスト生成パイプラインの一部です。
  • astreamer インターフェースを介したデータ処理フローの中で、マニフェストの書き込みが必要な箇所でのみ使用されます。
