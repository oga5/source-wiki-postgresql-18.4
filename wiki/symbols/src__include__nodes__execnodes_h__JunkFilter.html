JunkFilter

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/execnodes.h:410

目的

クエリ実行時に、不要な属性（ジャンク属性）をタプルから除去し、必要な属性のみで構成されるタプルを生成するためのフィルタリング構造体です。

シグネチャまたは形状

typedef struct JunkFilter
{
	NodeTag		type;
	List	   *jf_targetList;
	TupleDesc	jf_cleanTupType;
	AttrNumber *jf_cleanMap;
	TupleTableSlot *jf_resultSlot;
} JunkFilter;

入力と出力

  • **入力**: フィルタリング対象のタプル（TupleTableSlot 経由で供給される）。
  • **出力**: jf_cleanTupType に基づく形式に変換され、jf_resultSlot に格納されたクリーンなタプル。

前提条件と不変条件

  • jf_resultSlot は、フィルタリング後のタプルを保持するために初期化されている必要があります。
  • jf_cleanMap は、元のタプルから新しいタプルへの属性マッピングを定義する配列である必要があります。

アルゴリズムまたは振る舞い

1. jf_targetList を用いて、元のタプルから必要な属性を特定します。 2. jf_cleanMap に基づき、元のタプルから値を抽出し、新しいタプル形式（jf_cleanTupType）へ変換します。 3. 変換後のタプルを jf_resultSlot に格納して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExpr.c および src/backend/executor/execJunk.c 内の関数群が本構造体の生成および操作を行っています。
  • **呼び出し先**: 具体的な関数呼び出しの連鎖については、提供されたソース範囲内では明示されていません。

関連データ構造

  • TupleTableSlot: フィルタリング前後のタプルを保持するために使用されます。
  • TupleDesc: 変換後のタプルの構造を定義します。
  • List: ターゲットリスト（jf_targetList）を保持します。

失敗モードと境界条件

  • jf_cleanMap が正しく初期化されていない場合、属性の抽出時に不正なメモリ参照が発生する可能性があります。
  • フィルタリング対象のタプルが jf_cleanTupType と互換性のない構造である場合、変換処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は makeNode(JunkFilter) を使用して生成されます。
  • ExecFindJunkAttribute 関数により、特定のジャンク属性を検索することが可能です。
  • 実行時専用の構造体であるため、永続化やコピーを目的とした使用は避けてください。
