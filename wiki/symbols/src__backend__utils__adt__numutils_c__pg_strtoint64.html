pg_strtoint64

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numutils.c:643

目的

文字列を int64 型の整数値へ変換します。不正な形式やオーバーフローが発生した場合は、ereport を介してエラーを報告します。

シグネチャまたは形状

int64 pg_strtoint64(const char *s)

入力と出力

  • **入力**: s (解析対象の文字列へのポインタ)
  • **出力**: 変換後の int64 値。エラー時は呼び出し元へ制御が戻らない（ereport による中断）。

前提条件と不変条件

  • 入力文字列 s は NUL 終端されている必要があります。
  • 内部的に pg_strtoint64_safe を呼び出しており、エラーコンテキスト（escontext）を NULL として渡すため、エラー発生時は即座に ereport が発行されます。

アルゴリズムまたは振る舞い

1. 内部関数 pg_strtoint64_safe(s, NULL) を呼び出します。 2. pg_strtoint64_safe は、単純な10進数解析を試行し、失敗した場合は基数指定（0x, 0o, 0b）やアンダースコアを含む複雑な構文を解析します。 3. 変換結果をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copy.c:479
  • **呼び出し先**:
  • pg_strtoint64_safe (同一ファイル内)

関連データ構造

  • int64 (PostgreSQL の 64 ビット整数型)

失敗モードと境界条件

  • **不正な形式**: 数値として解釈できない文字列が渡された場合、ereport によりエラーが発生します。
  • **オーバーフロー**: int64 の範囲を超える値が渡された場合、ereport によりエラーが発生します。

保守担当者向けメモ

  • 本関数は pg_strtoint64_safe のラッパーです。エラーハンドリングの挙動を変更する場合は、pg_strtoint64_safe 側の実装を確認してください。
  • パフォーマンスが重要な箇所では、本関数が内部で行う ereport のコストを考慮してください。
