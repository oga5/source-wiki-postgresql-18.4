ExecCustomScanReInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCustom.c:189

目的

並列クエリ実行時において、カスタムスキャンノードが使用する共有メモリ（DSM）領域を再初期化します。

シグネチャまたは形状

void ExecCustomScanReInitializeDSM(CustomScanState *node, ParallelContext *pcxt)

入力と出力

  • node: カスタムスキャンの実行状態を保持する CustomScanState 構造体へのポインタ。
  • pcxt: 並列クエリのコンテキストを保持する ParallelContext 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • node->methods が有効な CustomExecMethods を指していること。
  • pcxt->toc に、現在のプランノード ID (plan_node_id) に対応する共有メモリ領域が登録されていること。

アルゴリズムまたは振る舞い

1. node->methods からカスタムスキャン固有のメソッドテーブルを取得します。 2. 拡張機能側で ReInitializeDSMCustomScan コールバックが定義されているか確認します。 3. 定義されている場合、pcxt->toc から現在のプランノード ID をキーとして共有メモリ上の座標（coordinate）を検索します。 4. 取得した座標を引数として、拡張機能側の ReInitializeDSMCustomScan を呼び出し、共有メモリの状態をリセットします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execParallel.c 内の並列実行処理から呼び出されます。
  • 呼び出し先: 拡張機能が提供する CustomExecMethods 内の ReInitializeDSMCustomScan 関数。

関連データ構造

  • CustomScanState: カスタムスキャンの実行状態を管理する構造体。
  • CustomExecMethods: 拡張機能が実装するコールバック関数群。
  • ParallelContext: 並列クエリの共有メモリ管理用コンテキスト。

失敗モードと境界条件

  • methods->ReInitializeDSMCustomScan が定義されていない場合、本関数は何もしません。
  • shm_toc_lookup が指定された ID を見つけられない場合、shm_toc_lookup の仕様に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は、並列クエリの再実行（ReScan）時に共有メモリの状態をクリーンな状態に戻すためのブリッジです。
  • 拡張機能側で並列処理をサポートする場合、このコールバックの実装が必須となります。
