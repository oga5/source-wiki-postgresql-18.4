pgstat_progress_incr_param

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_progress.c:68

目的

現在実行中のバックエンドプロセスに関連付けられた進捗パラメータ（st_progress_param）の値を、指定した増分値で加算更新します。

シグネチャまたは形状

void pgstat_progress_incr_param(int index, int64 incr)

入力と出力

  • index (int): 更新対象のパラメータインデックス（0〜19の範囲）。
  • incr (int64): 加算する値。
  • 出力: なし（共有メモリ上の状態を直接更新）。

前提条件と不変条件

  • index は 0 以上 PGSTAT_NUM_PROGRESS_PARAM 未満である必要があります（Assert により検証）。
  • pgstat_track_activities が有効である必要があります。無効な場合、関数は何も行わずにリターンします。
  • MyBEEntry が有効（NULLでない）である必要があります。

アルゴリズムまたは振る舞い

1. MyBEEntry を取得し、パラメータの有効性を確認します。 2. PGSTAT_BEGIN_WRITE_ACTIVITY マクロを使用して、共有メモリ上の進捗状態への書き込みロック（またはメモリバリア）を確保します。 3. st_progress_param[index] に対して incr を加算します。 4. PGSTAT_END_WRITE_ACTIVITY マクロを使用して書き込みを完了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/vacuum.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/utils/activity/backend_progress.c（pgstat_progress_parallel_incr_param 内）
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行いませんが、PGSTAT_BEGIN_WRITE_ACTIVITY および PGSTAT_END_WRITE_ACTIVITY マクロを介して共有メモリ操作を行います。

関連データ構造

  • PgBackendStatus: 共有メモリ上に保持されるバックエンドの状態構造体。st_progress_param 配列を保持します。
  • MyBEEntry: 現在のバックエンドプロセスの PgBackendStatus へのポインタ。

失敗モードと境界条件

  • index が範囲外の場合、アサーション違反によりプロセスが停止する可能性があります（デバッグビルド時）。
  • pgstat_track_activities が無効な場合、更新は無視されます。
  • MyBEEntry が NULL の場合、更新は行われません。

保守担当者向けメモ

  • 本関数はアトミックな加算操作を提供しますが、複数のパラメータを同時に更新する必要がある場合は、整合性を保つために pgstat_progress_update_multi_param の使用を検討してください。
  • パラレルワーカーからの呼び出しについては、直接本関数を呼ぶのではなく、メッセージングを介する pgstat_progress_parallel_incr_param を使用する設計となっています。
