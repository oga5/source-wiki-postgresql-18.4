intset_union_elem

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_op.c:379

目的

既存の整数配列に対して単一の整数要素を追加し、重複排除とソートを行った新しい配列を生成する。

シグネチャまたは形状

Datum intset_union_elem(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_ARRAYTYPE_P(0): 対象となる整数配列（ArrayType）。
  • PG_GETARG_INT32(1): 追加する整数値。
  • **出力**:
  • PG_RETURN_POINTER: 要素が追加・ソート・重複排除された新しい ArrayType へのポインタ。

前提条件と不変条件

  • 入力配列は _int.h で定義されたマクロによる検証が可能であること。
  • 処理の過程で PG_FREE_IF_COPY が呼び出されるため、入力配列のメモリ管理はPostgreSQLの関数呼び出し規約に従う必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_ARRAYTYPE_P を用いて入力配列を取得する。 2. intarray_add_elem を呼び出し、指定された整数を配列に追加する。 3. PG_FREE_IF_COPY により、必要に応じて入力配列のコピーを解放する。 4. QSORT マクロを使用して、結果の配列をソートする。 5. _int_unique を呼び出して配列内の重複要素を排除し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • intarray_add_elem
  • QSORT (マクロ)
  • _int_unique

関連データ構造

  • ArrayType: PostgreSQLの配列データ型。

失敗モードと境界条件

  • 入力配列が不正な形式である場合、intarray_add_elem 等の内部処理でエラーが発生する可能性がある。
  • メモリ不足時にはPostgreSQLのメモリ管理機構によりエラーがスローされる。

保守担当者向けメモ

  • 本関数は、配列の正規化（ソートと重複排除）を確実に行うため、_int_unique の戻り値を直接返す設計となっている。
  • QSORT マクロは _int.h で定義されており、配列のソート順序を決定する。変更時は _int.h の定義を確認すること。
