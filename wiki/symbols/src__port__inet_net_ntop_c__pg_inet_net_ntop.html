pg_inet_net_ntop

定義

  • 種別: function
  • 定義箇所: src/port/inet_net_ntop.c:76

目的

ネットワークアドレス（IPv4/IPv6）を、バイナリ形式から人間が読み取り可能なプレゼンテーション形式（文字列）へ変換します。システム標準の inet_net_ntop が存在しない、あるいは挙動が異なる環境での移植性を提供します。

シグネチャまたは形状

char *pg_inet_net_ntop(int af, const void *src, int bits, char *dst, size_t size);

入力と出力

  • af: アドレスファミリ（PGSQL_AF_INET, PGSQL_AF_INET6, またはシステム標準の AF_INET, AF_INET6）。
  • src: 変換対象のネットワークアドレス（ネットワークバイトオーダー）。
  • bits: CIDR 表記用のビット数。
  • dst: 変換結果を格納するバッファ。
  • size: dst バッファのサイズ。
  • 戻り値: 成功時は dst へのポインタ、失敗時は NULL を返し errno を設定します。

前提条件と不変条件

  • PGSQL_AF_INET とシステム側の AF_INET は同一であると仮定されます。
  • PGSQL_AF_INET6 とシステム側の AF_INET6 は異なる可能性があるため、両方の定数を考慮したディスパッチを行います。

アルゴリズムまたは振る舞い

1. af 引数に基づき、switch 文で適切な変換関数へディスパッチします。

  • PGSQL_AF_INET: inet_net_ntop_ipv4 を呼び出します。
  • PGSQL_AF_INET6 または AF_INET6: inet_net_ntop_ipv6 を呼び出します。

    • 2. サポート外の af が指定された場合は errno に EAFNOSUPPORT を設定し NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network.c
  • src/interfaces/libpq/fe-connect.c
  • src/interfaces/libpq/fe-secure-common.c
  • **呼び出し先**:
  • inet_net_ntop_ipv4
  • inet_net_ntop_ipv6

関連データ構造

  • PGSQL_AF_INET, PGSQL_AF_INET6: PostgreSQL 内部で定義されたアドレスファミリ定数。

失敗モードと境界条件

  • バッファサイズ不足時やビット数が範囲外の場合、内部の変換関数が NULL を返し errno（EMSGSIZE, EINVAL 等）をセットします。
  • 未知のアドレスファミリが指定された場合、EAFNOSUPPORT が返されます。

保守担当者向けメモ

  • 本関数は PGSQL_AF_INET とシステム側の AF_INET の定数差異を吸収する役割を持ちます。
  • IPv6 のゼロ圧縮ロジック（inet_net_ntop_ipv6）に変更を加える際は、最長ゼロランの計算ロジックに注意してください。
  • sprintf の戻り値の扱いに SPRINTF マクロが使用されている点に留意してください。
