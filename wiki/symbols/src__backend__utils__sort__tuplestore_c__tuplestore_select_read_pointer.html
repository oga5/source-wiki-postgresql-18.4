tuplestore_select_read_pointer

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/sort/tuplestore.c:506

目的

Tuplestorestate 内で管理される複数の読み取りポインタのうち、アクティブなものを切り替える。ディスクへ退避された状態（TSS_READFILE）において、物理的なファイルシーク位置を論理的な読み取り位置と同期させる。

シグネチャまたは形状

void tuplestore_select_read_pointer(Tuplestorestate *state, int ptr)

入力と出力

  • **入力**:
  • state: Tuplestorestate 構造体へのポインタ。
  • ptr: 切り替え先の読み取りポインタインデックス。
  • **出力**: なし（副作用として state->activeptr および内部のファイルシーク位置を更新）。

前提条件と不変条件

  • ptr は 0 以上 state->readptrcount 未満であること（Assert で検証）。
  • 既に ptr がアクティブな場合は何もしない。
  • TSS_READFILE 状態の場合、BufFile のシーク操作が成功する必要がある。

アルゴリズムまたは振る舞い

1. **アクティブチェック**: 指定された ptr が現在のアクティブポインタと一致すれば即座に終了する。 2. **状態別処理**:

  • TSS_INMEM または TSS_WRITEFILE: 状態遷移は発生せず、アクティブポインタの更新のみを行う。
  • TSS_READFILE:
  • 現在のアクティブポインタが EOF に達していない場合、現在の BufFile 位置を oldptr->file と oldptr->offset に保存する。
  • 新しい readptr が EOF に達している場合は、書き込み位置（writepos_file, writepos_offset）へシークする。
  • それ以外の場合は、readptr に保存された位置へ BufFileSeek を実行する。

    • 3. **更新**: state->activeptr を ptr に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nodeCtescan.c: ExecCteScan 等の CTE スキャン処理。
  • nodeNamedtuplestorescan.c: 名前付きタプルストアスキャン処理。
  • nodeWindowAgg.c: ウィンドウ関数処理におけるバッファ操作。
  • **呼び出し先**:
  • BufFileTell: 現在のファイル位置の取得。
  • BufFileSeek: 指定位置へのファイルシーク。
  • ereport: ファイルシーク失敗時のエラー報告。

関連データ構造

  • Tuplestorestate: タプルストアの全体状態を保持。
  • TSReadPointer: 個別の読み取り位置（ファイル位置、オフセット、EOF 到達フラグ）を保持。

失敗モードと境界条件

  • ptr が範囲外の場合: Assert によりアボート。
  • BufFileSeek 失敗時: ERROR を発生させ、トランザクションを中断する。
  • TSS_READFILE 以外の無効な状態: elog(ERROR) により異常終了。

保守担当者向けメモ

  • 本関数は、読み取りポインタを切り替える際に物理的なファイルポインタを追従させるための重要な同期ポイントである。
  • TSS_READFILE 状態でのみファイル操作が発生するため、メモリ内での操作（TSS_INMEM）時にはオーバーヘッドは最小限に抑えられている。
