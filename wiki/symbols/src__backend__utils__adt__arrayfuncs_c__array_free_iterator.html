array_free_iterator

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:4764

目的

配列走査のために割り当てられた ArrayIterator 構造体およびその内部で動的に確保されたメモリを解放します。

シグネチャまたは形状

void array_free_iterator(ArrayIterator iterator)

入力と出力

  • **入力**: iterator (ArrayIterator) - 解放対象のイテレータへのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • iterator は array_create_iterator 等によって正当に割り当てられたメモリ領域を指している必要があります。
  • iterator が NULL の場合、本関数はセグメンテーション違反を引き起こす可能性があります（ソースコード上にNULLチェックは存在しません）。

アルゴリズムまたは振る舞い

1. iterator->slice_ndim が 0 より大きい場合、スライス操作用のバッファである iterator->slice_values および iterator->slice_nulls を pfree で解放します。 2. 最後に、イテレータ本体である iterator 構造体自身を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/array_userfuncs.c:1461
  • src/backend/utils/adt/array_userfuncs.c:1601
  • src/backend/utils/adt/array_userfuncs.c:1990
  • **呼び出し先**:
  • pfree (メモリ解放関数)

関連データ構造

  • ArrayIterator: 配列要素を走査するための状態を保持する構造体。

失敗モードと境界条件

  • iterator が指すメモリが既に解放済みである場合（二重解放）、pfree の内部実装によりエラーが発生します。
  • iterator が NULL の場合、未定義の動作となります。

保守担当者向けメモ

  • 本関数は ArrayIterator のライフサイクル管理の終端を担います。イテレータを生成した側で必ず本関数を呼び出し、メモリリークを防ぐ必要があります。
