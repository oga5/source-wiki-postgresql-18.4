Zstd_read_internal

定義

  • src/bin/pg_dump/compress_zstd.c:262

目的

Zstandard 圧縮されたアーカイブデータからデータを読み込み、解凍してバッファへ展開する内部関数です。

シグネチャまたは形状

static size_t Zstd_read_internal(void *ptr, size_t size, CompressFileHandle *CFH, bool exit_on_error)

入力と出力

  • **入力**:
  • ptr: 解凍後のデータを格納する出力バッファ。
  • size: 読み込むバイト数。
  • CFH: 圧縮ファイルハンドル（ZstdCompressorState を保持）。
  • exit_on_error: エラー発生時に pg_fatal を呼び出すかどうかのフラグ。
  • **出力**: 実際に解凍・書き込みが行われたバイト数を size_t で返します。エラー時は -1 を返します。

前提条件と不変条件

  • CFH->private_data は ZstdCompressorState 型である必要があります。
  • 初回呼び出し時に zstdcs->dstream が NULL の場合、内部でメモリ確保とストリームの初期化が行われます。
  • input->pos <= input->size および input->size <= ZSTD_DStreamInSize() が常に維持されます。

アルゴリズムまたは振る舞い

1. 初回実行時、ZSTD_createDStream を用いて解凍ストリームを初期化します。 2. 出力バッファ（ptr）のサイズを設定します。 3. ループ内で以下を繰り返します：

  • 入力バッファが空の場合、fread で圧縮データをファイルから読み込みます。
  • ZSTD_decompressStream を呼び出し、圧縮データを解凍して出力バッファへ転送します。
  • 出力バッファが満杯になるか、解凍ストリームが終了（EOF）するまで処理を継続します。

    • 4. 最終的に書き込まれたバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_zstd.c 内の読み込み関連関数（Zstd_read 等）。
  • **呼び出し先**:
  • ZSTD_createDStream, ZSTD_decompressStream, ZSTD_isError, ZSTD_getErrorName (Zstd ライブラリ)
  • fread, ferror (標準ライブラリ)
  • pg_malloc0, pg_fatal (PostgreSQL ユーティリティ)

関連データ構造

  • ZstdCompressorState: 圧縮・解凍のコンテキストを保持する構造体。
  • CompressFileHandle: 圧縮処理の抽象化ハンドル。
  • ZSTD_inBuffer, ZSTD_outBuffer: Zstd ライブラリの入出力バッファ構造体。

失敗モードと境界条件

  • ZSTD_createDStream が失敗した場合、exit_on_error が真であれば pg_fatal を呼び出し、偽であれば -1 を返します。
  • fread でエラーが発生した場合、同様にエラーハンドリングを行います。
  • ZSTD_decompressStream がエラーを返した場合、ZSTD_getErrorName を用いてエラーメッセージを出力し、終了または -1 を返します。
  • 入力データが尽きた場合（cnt == 0）、ループを抜けて処理を終了します。

保守担当者向けメモ

  • 本関数は USE_ZSTD マクロが定義されている環境でのみ有効です。
  • バッファの動的リサイズや EOF 判定を含むため、修正時は ZSTD_inBuffer の状態管理に注意してください。
