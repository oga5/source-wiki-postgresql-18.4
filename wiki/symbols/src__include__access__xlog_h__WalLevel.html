WalLevel

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlog.h:72

目的

PostgreSQLの先行書き込みログ（WAL）が記録する情報の詳細度を制御し、システムの運用モード（最小限のログ、レプリケーション、論理デコード）を決定するために使用されます。

シグネチャまたは形状

typedef enum WalLevel
{
    WAL_LEVEL_MINIMAL = 0,
    WAL_LEVEL_REPLICA,
    WAL_LEVEL_LOGICAL,
} WalLevel;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: WAL_LEVEL_MINIMAL（0）、WAL_LEVEL_REPLICA、WAL_LEVEL_LOGICAL のいずれかの値。

前提条件と不変条件

  • 値は 0 から順に割り当てられており、比較演算が可能です。
  • システムの動作モードを決定する重要な設定値であり、実行中に変更する場合は再起動が必要となる場合があります。

アルゴリズムまたは振る舞い

  • WAL_LEVEL_MINIMAL: 最小限のログ出力。クラッシュリカバリには十分ですが、ストリーミングレプリケーションやアーカイブには不十分です。
  • WAL_LEVEL_REPLICA: レプリケーションに必要な情報を追加で記録します。
  • WAL_LEVEL_LOGICAL: 論理デコードに必要な情報を追加で記録します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c および src/bin/pg_controldata/pg_controldata.c で参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • XLogRecPtr: WAL 内の位置を示す構造体。
  • ArchiveMode: アーカイブ設定に関連する列挙型。

失敗モードと境界条件

  • 範囲外の値が設定された場合の挙動については、本ソース excerpt に記載がありません。

保守担当者向けメモ

  • 本シンボルはシステム全体の WAL 動作を規定するため、変更時はバイナリ互換性および各レプリケーション機能への影響を十分に考慮してください。
  • GetActiveWalLevelOnStandby() 関数などが本型を利用して状態を判定しています。
