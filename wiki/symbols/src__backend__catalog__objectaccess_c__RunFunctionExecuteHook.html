RunFunctionExecuteHook

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:138

目的

関数実行イベントを外部のフック関数に通知し、セキュリティ拡張や監査ログプラグインによる介入を可能にする。

シグネチャまたは形状

void RunFunctionExecuteHook(Oid objectId)

入力と出力

  • **入力**: objectId (実行対象となる関数のOID)
  • **出力**: なし（副作用としてフック関数を実行）

前提条件と不変条件

  • グローバル変数 object_access_hook が非NULLであること（Assertにより検証）。

アルゴリズムまたは振る舞い

1. object_access_hook がNULLでないことを確認する。 2. 登録されたフック関数を以下の引数で呼び出す:

  • イベントタイプ: OAT_FUNCTION_EXECUTE
  • クラスID: ProcedureRelationId
  • オブジェクトID: objectId
  • その他の引数: 0 および NULL

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は、提供された抜粋からは特定できません。
  • **呼び出し先**: グローバル関数ポインタ object_access_hook が指す任意の関数。

関連データ構造

  • OAT_FUNCTION_EXECUTE: フックに渡されるイベントタイプ定数。
  • ProcedureRelationId: pg_proc カタログを識別するOID。

失敗モードと境界条件

  • object_access_hook がNULLの状態で呼び出された場合、アサーションエラーが発生しプロセスが停止する。

保守担当者向けメモ

  • 本関数はフックの実行エントリーポイントです。実際のロジックはロードされたプラグイン側に依存します。
  • 名前ベースの識別が必要な場合は、対となる RunFunctionExecuteHookStr の使用を検討してください。
