htsv_get_valid_status

定義

  • 種別: static inline function
  • 定義箇所: src/backend/access/heap/pruneheap.c:959

目的

整数値として保持されているタプルの可視性ステータスを、型安全な HTSV_Result 型へ変換するキャスト関数です。

シグネチャまたは形状

static inline HTSV_Result htsv_get_valid_status(int status)

入力と出力

  • **入力**: int status （タプルの可視性ステータスを示す整数値）
  • **出力**: HTSV_Result （変換後の列挙型値）

前提条件と不変条件

  • 入力 status は、HEAPTUPLE_DEAD 以上かつ HEAPTUPLE_DELETE_IN_PROGRESS 以下の範囲内である必要があります。この条件は Assert によって実行時に検証されます。

アルゴリズムまたは振る舞い

1. 入力された status が有効な範囲内にあるかアサーションで確認します。 2. 入力値を HTSV_Result 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/pruneheap.c 内の剪定処理ロジック（例: 1080行目付近の switch 文）から呼び出されています。
  • **呼び出し先**: なし（標準的な型変換のみを実行）。

関連データ構造

  • HTSV_Result: タプルの可視性状態を表す列挙型。
  • PruneState: 剪定処理中の状態を保持する構造体（本関数は PruneState 内の htsv 配列の要素を処理する際に利用されます）。

失敗モードと境界条件

  • **失敗モード**: アサーションが有効なビルドにおいて、範囲外の status が渡された場合、プロセスは異常終了（アボート）します。
  • **境界条件**: HEAPTUPLE_DEAD および HEAPTUPLE_DELETE_IN_PROGRESS は許容される境界値です。

保守担当者向けメモ

  • 本関数は static inline で定義されており、コンパイル時にインライン展開されることを意図しています。
  • 可視性ステータスの定義に変更があった場合は、アサーションの範囲チェックを更新する必要があります。
