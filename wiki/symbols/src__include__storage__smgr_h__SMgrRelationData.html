SMgrRelationData

定義

  • 種別: struct
  • 定義箇所: src/include/storage/smgr.h:35

目的

物理的なリレーション（テーブルやインデックス等）に対する I/O 操作を抽象化し、ストレージマネージャ層で管理するためのキャッシュされたハンドル構造体です。

シグネチャまたは形状

typedef struct SMgrRelationData
{
    RelFileLocatorBackend smgr_rlocator;
    BlockNumber smgr_targblock;
    BlockNumber smgr_cached_nblocks[MAX_FORKNUM + 1];
    int         smgr_which;
    int         md_num_open_segs[MAX_FORKNUM + 1];
    struct _MdfdVec *md_seg_fds[MAX_FORKNUM + 1];
    int         pincount;
    dlist_node  node;
} SMgrRelationData;

入力と出力

  • **入力**: RelFileLocatorBackend（ハッシュテーブルの検索キーとして使用）。
  • **出力**: 物理ストレージへのアクセスに必要なファイル記述子およびキャッシュされたリレーションサイズ情報。

前提条件と不変条件

  • smgr_rlocator は構造体の先頭に配置され、ハッシュテーブルの検索キーとして機能します。
  • smgr_cached_nblocks はキャッシュフラッシュイベント時に InvalidBlockNumber にリセットされます。
  • pincount が 0 の場合、node は未ピン留めリレーションのリストにリンクされます。
  • smgr_which 以降のフィールドは smgr.c およびそのサブモジュール専用のプライベート領域であり、外部からの直接操作は禁止されています。

アルゴリズムまたは振る舞い

  • リレーションの物理識別子を保持し、ストレージマネージャ（md.c 等）へのディスパッチ先を smgr_which で管理します。
  • トランザクション終了時に pincount が 0 のリレーションは自動的にクローズされます。
  • md_seg_fds を通じて、各フォーク（データ、FSM、VM等）ごとのオープン済みセグメントを管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: bufmgr.h（PrefetchSharedBuffer, FlushRelationsAllBuffers 等）、bulk_write.h（smgr_bulk_start_smgr）、smgr.c（ハッシュテーブル管理）。
  • **呼び出し先**: 構造体内部の _MdfdVec を介して物理ファイルシステム操作（md.c）へ接続されます。

関連データ構造

  • RelFileLocatorBackend: リレーションの物理識別子。
  • _MdfdVec: md.c で使用されるファイルセグメント管理構造体。
  • dlist_node: 未ピン留めリレーションのリスト管理用。

失敗モードと境界条件

  • キャッシュ無効化の仕組みがフォーク拡張に対して存在しないため、smgr_cached_nblocks はリカバリ時以外は信頼性が低い場合があります。
  • smgr.c の関数はリエントラントではなく、elog(ERROR) 等の割り込みによる smgrreleaseall 実行時に状態が変化する可能性があります。

保守担当者向けメモ

  • smgr_which 以降のフィールドはプライベートであるため、外部モジュールからは直接アクセスしないでください。
  • SMgrRelation（本構造体へのポインタ）を長期的に保持する必要がある場合は、必ず smgrpin() を使用してトランザクション終了時の自動クローズを回避してください。
