PruneState

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/heap/pruneheap.c:33

目的

ヒープページの剪定（Pruning）および凍結（Freezing）処理中における作業状態を保持する構造体です。削除対象のタプル、HOTチェーンの構成、可視性判定結果、および処理後の統計情報を集約します。

シグネチャまたは形状

typedef struct {
    /* 入力引数 */
    GlobalVisState *vistest;
    bool        mark_unused_now;
    bool        freeze;
    struct VacuumCutoffs *cutoffs;

    /* ページ変更計画 */
    TransactionId new_prune_xid;
    TransactionId latest_xid_removed;
    int         nredirected, ndead, nunused, nfrozen;
    OffsetNumber redirected[MaxHeapTuplesPerPage * 2];
    OffsetNumber nowdead[MaxHeapTuplesPerPage];
    OffsetNumber nowunused[MaxHeapTuplesPerPage];
    HeapTupleFreeze frozen[MaxHeapTuplesPerPage];

    /* HOTチェーン処理用作業領域 */
    int         nroot_items;
    OffsetNumber root_items[MaxHeapTuplesPerPage];
    int         nheaponly_items;
    OffsetNumber heaponly_items[MaxHeapTuplesPerPage];
    bool        processed[MaxHeapTuplesPerPage + 1];
    int8        htsv[MaxHeapTuplesPerPage + 1];
    HeapPageFreeze pagefrz;

    /* 処理結果統計 */
    int         ndeleted;
    int         live_tuples;
    int         recently_dead_tuples;
    bool        hastup;
    int         lpdead_items;
    OffsetNumber *deadoffsets;
    bool        all_visible;
    bool        all_frozen;
    TransactionId visibility_cutoff_xid;
} PruneState;

入力と出力

  • **入力**: heap_page_prune_and_freeze 呼び出し時に初期化され、可視性判定用の GlobalVisState や凍結用の VacuumCutoffs を受け取ります。
  • **出力**: 処理完了後、ndeleted や all_visible などの統計情報が呼び出し元の PruneFreezeResult へコピーされます。

前提条件と不変条件

  • processed および htsv 配列は MaxHeapTuplesPerPage + 1 のサイズを持ち、FirstOffsetNumber（1）を直接インデックスとして使用します。
  • htsv 配列において、値 -1 は可視性が未計算であることを示します。
  • all_frozen は all_visible が真である場合にのみ有効とみなされます。

アルゴリズムまたは振る舞い

1. **初期化**: ページ内の全タプルの可視性を一度だけ判定し htsv に格納します。 2. **走査**: root_items（リダイレクト先や非HOTタプル）と heaponly_items を分類し、HOTチェーンを辿ります。 3. **記録**: 削除やリダイレクトが必要なオフセットを各配列に蓄積します。 4. **実行**: 蓄積された計画に基づき、物理的なページ変更を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**: heap_page_prune_and_freeze が本構造体を管理します。
  • **呼び出し先**: heap_prune_satisfies_vacuum、heap_prune_record_prunable、heap_prune_record_redirect 等の heap_prune_record_* 系関数群が、本構造体のフィールドを更新するために使用します。

関連データ構造

  • GlobalVisState: 可視性判定に使用。
  • HeapTupleFreeze: 凍結対象タプルの情報を保持。
  • HeapPageFreeze: ページ全体の凍結状態を管理。

失敗モードと境界条件

  • MaxHeapTuplesPerPage を超えるオフセット操作は発生しない設計ですが、配列サイズが固定であるため、ページ内のタプル数が上限を超えないことが前提となります。
  • deadoffsets は呼び出し元のメモリ領域を指すポインタであるため、生存期間の管理に注意が必要です。

保守担当者向けメモ

  • 本構造体は剪定処理の「一時的な作業台」です。処理終了後に呼び出し元へ結果を渡す役割も兼ねているため、フィールドの追加時には heap_page_prune_and_freeze でのコピー処理も併せて確認してください。
  • htsv 配列のインデックス操作は、FirstOffsetNumber を考慮した設計になっているため、オフセットのオフセット（-1）を行わないよう注意してください。
