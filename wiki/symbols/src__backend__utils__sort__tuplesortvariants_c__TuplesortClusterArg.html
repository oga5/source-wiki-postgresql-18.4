TuplesortClusterArg

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:111

目的

テーブルのクラスタリング（CLUSTERコマンド等）実行時に、ソート処理へ必要なメタデータを提供するための引数構造体です。

シグネチャまたは形状

typedef struct {
    TupleDesc   tupDesc;        /* タプルの記述子 */
    IndexInfo  *indexInfo;      /* 参照するインデックス情報 */
    EState     *estate;         /* インデックス式評価用の実行状態 */
} TuplesortClusterArg;

入力と出力

  • 本構造体自体は、tuplesort_begin_cluster 等の初期化関数内でメモリ確保（palloc0）され、ソート状態（Tuplesortstate）の補助引数として保持されます。
  • 出力として、ソート処理中の比較やタプル変換時に、indexInfo や estate が参照されます。

前提条件と不変条件

  • Tuplesortstate の base->arg フィールドにキャストして格納されることが前提です。
  • estate はインデックス式を評価するために有効な状態である必要があります。

アルゴリズムまたは振る舞い

  • ソート処理において、クラスタリング対象のインデックスに基づいたタプルの比較や変換を行うためのコンテキストを提供します。
  • tuplesortvariants.c 内の各関数（comparetup_cluster 等）において、base->arg から本構造体を復元し、インデックス情報や式評価機能を利用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesortvariants.c 内の tuplesort_begin_cluster（初期化時）および、ソート処理中の各関数（comparetup_cluster 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、保持する estate を通じて式評価関数が呼び出されます。

関連データ構造

  • TupleDesc: タプルの構造定義。
  • IndexInfo: クラスタリングに使用するインデックスのメタデータ。
  • EState: クエリ実行状態。
  • Tuplesortstate: 本構造体を内部に保持するソート処理のメイン状態構造体。

失敗モードと境界条件

  • メモリ不足時に palloc0 が失敗する可能性があります。
  • estate が適切に初期化されていない場合、インデックス式の評価時にクラッシュする可能性があります。

保守担当者向けメモ

  • 本構造体は tuplesortvariants.c 内のクラスタリング専用の引数です。新しいソート対象型を追加する際は、同様のパターンで専用の Arg 構造体を定義してください。
  • base->arg からのキャストに依存しているため、型安全性の確保には注意が必要です。
