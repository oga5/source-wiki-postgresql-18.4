oid_increment

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:493

目的

B-tree インデックスのスキップスキャンにおいて、指定された Oid 型の値を 1 増加させるための補助関数です。

シグネチャまたは形状

static Datum oid_increment(Relation rel, Datum existing, bool *overflow);

入力と出力

  • **入力**:
  • rel: 対象のリレーション（本関数内では未使用）。
  • existing: 増加対象の Oid 値（Datum 型）。
  • overflow: オーバーフロー発生時に状態を書き込むためのブール値ポインタ。
  • **出力**:
  • 戻り値: 増加後の Oid 値（Datum 型）。オーバーフロー時は (Datum) 0 を返す。

前提条件と不変条件

  • 入力 existing は有効な Oid として解釈可能であること。
  • overflow ポインタは書き込み可能なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. existing を Oid 型に変換します。 2. 値が OID_MAX と等しい場合、*overflow を true に設定し、戻り値として (Datum) 0 を返します。 3. それ以外の場合、*overflow を false に設定し、値を 1 増加させた結果を Datum 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内の sksup->increment への代入処理（src/backend/access/nbtree/nbtcompare.c:515）にて参照されています。
  • **呼び出し先**: DatumGetObjectId および ObjectIdGetDatum マクロを使用します。

関連データ構造

  • SkipSupport 構造体（sksup）の increment フィールドに本関数が登録されます。

失敗モードと境界条件

  • existing が OID_MAX の場合、これ以上増加できないためオーバーフローとして扱われます。この際、戻り値は未定義となります。

保守担当者向けメモ

  • 本関数は SkipSupport インターフェースの一部として実装されています。新しいデータ型を追加する際は、同様のパターンで increment 関数を定義し、対応する skipsupport 関数に登録してください。
