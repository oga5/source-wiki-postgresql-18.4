ltree_same

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_gist.c:130

目的

2つの ltree_gist インデックスエントリが論理的に同一であるかを判定し、GiST インデックスの重複排除や比較処理をサポートします。

シグネチャまたは形状

Datum ltree_same(PG_FUNCTION_ARGS)
  • 引数: PG_FUNCTION_ARGS (内部的に ltree_gist *a, ltree_gist *b, bool *result を取得)
  • 戻り値: Datum (結果を格納した bool *result へのポインタ)

入力と出力

  • 入力: 比較対象となる2つの ltree_gist 構造体へのポインタ。
  • 出力: result 引数が指すメモリ領域に、両者が同一であれば true、そうでなければ false を書き込みます。

前提条件と不変条件

  • LTREE_GET_SIGLEN() により取得されるシグネチャ長が、比較対象の ltree_gist 構造体と整合している必要があります。
  • ltree_gist 構造体は、LTG_ISONENODE マクロで単一ノードか否かを判定可能である必要があります。

アルゴリズムまたは振る舞い

1. **単一ノード判定**: 両者の LTG_ISONENODE 状態が異なる場合、直ちに false を返します。 2. **単一ノード比較**: 両者が単一ノードの場合、ISEQ マクロを用いてノード内容を比較します。 3. **複数ノード比較**:

  • LTG_ISALLTRUE（全ビットセット状態）が一致しない場合は false を返します。
  • LTG_LNODE（左ノード）および LTG_RNODE（右ノード）が ISEQ で一致することを確認します。
  • LTG_ISALLTRUE が偽の場合、シグネチャ（ビットベクトル）の各バイトを LOOPBYTE マクロで走査し、全ビットが一致するかを検証します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/ltree/ltree_gist.c:84 にて PG_FUNCTION_INFO_V1 マクロにより登録されています。
  • 呼び出し先: ISEQ マクロ、LTG_* 系アクセサマクロ、LOOPBYTE マクロ。

関連データ構造

  • ltree_gist: ltree 型の GiST インデックス用内部表現。
  • LtreeSignature: シグネチャ（ビットマップ）を保持する構造。

失敗モードと境界条件

  • siglen の不一致や、不正なメモリレイアウトを持つ ltree_gist ポインタが渡された場合の動作は未定義です。
  • LTG_ISALLTRUE が真の場合、ビットベクトルの比較はスキップされます。

保守担当者向けメモ

  • 本関数は GiST インデックスの same メソッドとして機能します。新しい演算子やインデックス構造を追加する際は、ltree_gist の構造変更が本関数の比較ロジックに影響を与えないか確認してください。
