ExecAppendAsyncBegin

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:876

目的

Append ノードにおける非同期サブプラン実行の初期化を行う。有効な非同期サブプランを特定し、各サブプランに対して非同期リクエストを発行する。

シグネチャまたは形状

static void ExecAppendAsyncBegin(AppendState *node)

入力と出力

  • **入力**: AppendState *node - Append ノードの実行時状態を保持する構造体。
  • **出力**: なし（副作用として node 内の状態変数を更新し、非同期リクエストを発行する）。

前提条件と不変条件

  • 順方向スキャンであること（ScanDirectionIsForward）。
  • 少なくとも1つのサブプランが存在すること（as_nplans > 0）。
  • 少なくとも1つの非同期サブプランが存在すること（as_nasyncplans > 0）。

アルゴリズムまたは振る舞い

1. **サブプランの特定**: まだ有効なサブプランが特定されていない場合、ExecFindMatchingSubPlans を呼び出して特定し、classify_matching_subplans で分類を行う。 2. **状態変数の初期化**:

  • as_syncdone: 有効なサブプランが空であれば true に設定。
  • as_nasyncremain: 有効な非同期サブプランの数を設定。

    • 3. **リクエスト発行**: 有効な非同期サブプランが存在する場合、各サブプランに対して ExecAsyncRequest を呼び出し、非同期処理を開始する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から ExecAppend 内で呼び出されていることが確認できる。
  • **呼び出し先**: ExecFindMatchingSubPlans, classify_matching_subplans, bms_next_member, ExecAsyncRequest。

関連データ構造

  • AppendState: Append ノードの実行状態を管理する主要構造体。
  • AsyncRequest: 非同期リクエストの情報を保持する構造体。

失敗モードと境界条件

  • 有効な非同期サブプランが存在しない場合、処理は即座に終了する。
  • 逆方向スキャンが指定されている場合はアサーションエラーとなる。

保守担当者向けメモ

  • 本関数は非同期実行のセットアップのみを担当する。タプルの取得は ExecAppendAsyncGetNext 等の関連関数が担う。
  • as_valid_subplans が未確定の場合にのみ初期化ロジックが走るため、複数回呼び出された場合の再入可能性に注意すること。
