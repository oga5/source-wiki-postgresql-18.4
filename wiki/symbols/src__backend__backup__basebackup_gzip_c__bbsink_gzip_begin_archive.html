bbsink_gzip_begin_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_gzip.c:113

目的

gzip圧縮を行うバックアップシンクの初期化およびアーカイブ開始処理を行います。zlibのストリームを準備し、後続のシンクへアーカイブ開始を通知します。

シグネチャまたは形状

static void bbsink_gzip_begin_archive(bbsink *sink, const char *archive_name)

入力と出力

  • **入力**:
  • sink: bbsink_gzip 構造体へのポインタ。
  • archive_name: アーカイブのファイル名。
  • **出力**: なし（副作用として sink 内の z_stream を初期化し、後続シンクの begin_archive を呼び出す）。

前提条件と不変条件

  • sink は bbsink_gzip 型としてキャスト可能であること。
  • sink->bbs_next が存在すること（Assertにより検証）。
  • HAVE_LIBZ が定義されている環境であること。

アルゴリズムまたは振る舞い

1. z_stream 構造体をゼロクリアし、カスタムメモリ割り当て関数（gzip_palloc, gzip_pfree）を割り当てます。 2. 後続シンクのバッファを z_stream の出力先として設定します。 3. deflateInit2 を呼び出し、gzipヘッダーを生成する設定（window bits: 15+16）で圧縮ストリームを初期化します。初期化失敗時はエラーを報告します。 4. archive_name に .gz を付与した文字列を生成します。 5. 後続シンクの bbsink_begin_archive を呼び出し、処理を委譲します。 6. 生成した一時的なファイル名文字列を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink インターフェースのメソッドテーブル（begin_archive）を通じて呼び出されます。
  • **呼び出し先**:
  • deflateInit2 (zlib)
  • bbsink_begin_archive (後続のシンク)
  • psprintf, pfree (PostgreSQLメモリ管理)

関連データ構造

  • bbsink_gzip: 圧縮状態（z_stream）や圧縮レベルを保持する構造体。
  • bbsink: バックアップシンクの基底インターフェース。

失敗モードと境界条件

  • deflateInit2 が Z_OK 以外を返した場合、ERRCODE_INTERNAL_ERROR でエラー終了します。
  • sink->bbs_next が NULL の場合、Assert によりデバッグビルドで停止します。

保守担当者向けメモ

  • deflateInit2 の第4引数 15 + 16 は、zlib形式ではなくgzip形式のヘッダーを生成するために必須です。
  • 本関数はマニフェストファイル以外のアーカイブ開始時に呼び出されます。
