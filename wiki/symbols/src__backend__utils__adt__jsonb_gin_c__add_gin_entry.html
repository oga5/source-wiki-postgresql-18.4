add_gin_entry

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_gin.c:171

目的

GINインデックス構築時に、抽出されたキー（Datum）を動的に拡張可能なバッファへ追加し、そのインデックス番号を返却します。

シグネチャまたは形状

int add_gin_entry(GinEntries *entries, Datum entry)

入力と出力

  • **入力**:
  • entries: GinEntries 構造体へのポインタ。バッファの状態（現在の要素数、確保済みサイズ、データ配列）を保持します。
  • entry: インデックスに追加する Datum 型のキー値。
  • **出力**:
  • int: 追加されたキーが格納されたバッファ内のインデックス番号（0から始まるオフセット）。

前提条件と不変条件

  • entries は初期化済みである必要があります。
  • entries->count は常に現在の有効な要素数を示し、entries->allocated は entries->buf が保持可能な最大要素数を示します。

アルゴリズムまたは振る舞い

1. 現在の entries->count を戻り値用の id として保持します。 2. バッファの空き容量が不足している場合（count >= allocated）:

  • 既にバッファが確保されている場合は、allocated を2倍に拡張します。
  • 未確保の場合は、初期サイズとして8要素分を確保します。
  • repalloc または palloc を使用してメモリを再確保します。

    • 3. entries->buf の entries->count 番目に entry を格納し、count をインクリメントします。 4. 保持しておいた id を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonb_gin.c 内の各箇所（gin_extract_jsonb 等のインデックス抽出処理、および extract_jsp_query 等のクエリ解析処理から呼び出されます）。
  • **呼び出し先**:
  • repalloc / palloc (PostgreSQL メモリ管理関数)

関連データ構造

  • GinEntries: インデックスキーを一時的に保持するための構造体。buf（Datum配列）、count、allocated を含みます。

失敗モードと境界条件

  • メモリ不足時には palloc/repalloc がエラーを発生させ、処理が中断されます。
  • entries が NULL の場合、ポインタ参照エラーが発生します。

保守担当者向けメモ

  • 本関数は GinEntries の動的拡張を担うユーティリティです。バッファの拡張戦略（2倍のサイズ変更）は、要素数が増加する際の再割り当てコストを抑えるための標準的な実装です。
