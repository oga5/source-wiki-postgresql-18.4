recordDependencyOnOwner

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_shdepend.c:167

目的

データベースオブジェクトと、その所有者（ロール）との間の共有依存関係を pg_shdepend カタログに記録します。

シグネチャまたは形状

void recordDependencyOnOwner(Oid classId, Oid objectId, Oid owner)

入力と出力

  • **入力**:
  • classId: オブジェクトが属するシステムカタログの OID。
  • objectId: 対象オブジェクトの OID。
  • owner: 所有者ロールの OID。
  • **出力**: なし（void）。

前提条件と不変条件

  • objectSubId は常に 0 として扱われます（オブジェクト全体に対する所有権を想定）。
  • 依存先として AuthIdRelationId（pg_authid）が固定的に使用されます。

アルゴリズムまたは振る舞い

1. myself 構造体に、引数で受け取った classId と objectId を設定します。 2. referenced 構造体に、所有者ロール（AuthIdRelationId および owner）を設定します。 3. recordSharedDependencyOn を呼び出し、SHARED_DEPENDENCY_OWNER 型の依存関係としてカタログへ挿入します。

関連データ構造

  • ObjectAddress: オブジェクトを識別するための構造体。
  • pg_shdepend: 共有依存関係を保持するシステムカタログ。

失敗モードと境界条件

  • カタログへの挿入時にロック競合や権限エラーが発生する可能性があります。
  • 存在しない objectId や owner を指定した場合の動作は、基盤となる recordSharedDependencyOn の実装に依存します。

保守担当者向けメモ

  • 本関数は所有権の記録専用のラッパーです。所有権の変更が必要な場合は changeDependencyOnOwner を使用してください。
  • pg_shdepend は共有オブジェクトを跨ぐ依存関係を管理するため、データベース間で一貫性を保つ必要があります。
