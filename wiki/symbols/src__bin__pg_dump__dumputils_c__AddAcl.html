AddAcl

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:657

目的

ACL（アクセス制御リスト）の文字列バッファに対して、権限キーワードおよびオプションのサブネームを追記する。

シグネチャまたは形状

static void AddAcl(PQExpBuffer aclbuf, const char *keyword, const char *subname)

入力と出力

  • aclbuf (入出力): 権限文字列を格納する PQExpBuffer。
  • keyword (入力): 追加する権限キーワード。
  • subname (入力): オプションのサブネーム（NULL の場合は追加されない）。
  • 戻り値: なし。

前提条件と不変条件

  • aclbuf は初期化済みであること。
  • keyword は有効な文字列ポインタであること。

アルゴリズムまたは振る舞い

1. aclbuf が空でない場合、区切り文字としてカンマ（,）を追記する。 2. keyword を aclbuf に追記する。 3. subname が指定されている場合、括弧で囲んだ形式 (subname) を aclbuf に追記する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/dumputils.c 内の複数箇所（行470, 475）。
  • 呼び出し先: appendPQExpBufferChar、appendPQExpBufferStr、appendPQExpBuffer（いずれも libpq の PQExpBuffer 操作関数）。

関連データ構造

  • PQExpBuffer: PostgreSQL の動的文字列バッファ構造体。

失敗モードと境界条件

  • aclbuf が NULL の場合、プログラムはクラッシュする可能性がある。
  • keyword が NULL の場合、未定義動作となる。

保守担当者向けメモ

  • 本関数は static であり、dumputils.c 内でのみ使用される。
  • ACL 文字列のフォーマット変更が必要な場合は、本関数の追記ロジックを調整すること。
