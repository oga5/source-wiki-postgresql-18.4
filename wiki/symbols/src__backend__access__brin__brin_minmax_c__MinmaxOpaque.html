MinmaxOpaque

定義

  • 種別: struct
  • 定義箇所: src/backend/access/brin/brin_minmax.c:23

目的

BRIN（Block Range Index）の最小値・最大値演算クラスにおいて、比較関数（戦略関数）の情報をキャッシュし、SysCache検索のオーバーヘッドを削減するために使用される構造体です。

シグネチャまたは形状

typedef struct MinmaxOpaque
{
	Oid			cached_subtype;
	FmgrInfo	strategy_procinfos[BTMaxStrategyNumber];
} MinmaxOpaque;

入力と出力

  • **入力**: brin_minmax_opcinfo 関数内で初期化され、BrinOpcInfo の oi_opaque フィールドに格納されます。
  • **出力**: 比較処理時に minmax_get_strategy_procinfo を介して、キャッシュされた FmgrInfo が取得されます。

前提条件と不変条件

  • cached_subtype は、現在キャッシュされているデータ型（subtype）の OID を保持します。
  • strategy_procinfos 配列は、B-tree の戦略番号（BTMaxStrategyNumber）に対応する関数情報を保持します。

アルゴリズムまたは振る舞い

1. **初期化**: brin_minmax_opcinfo が呼び出される際、この構造体用にメモリが割り当てられます。 2. **キャッシュ**: minmax_get_strategy_procinfo が呼び出されると、指定された subtype が cached_subtype と一致するか確認します。一致しない場合、SysCache を検索して strategy_procinfos を更新します。 3. **利用**: 最小値・最大値の比較や更新を行う際、この構造体から取得した FmgrInfo を使用して関数呼び出しを行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • brin_minmax_opcinfo (初期化)
  • minmax_get_strategy_procinfo (キャッシュ参照・更新)
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、直接関数を呼び出しません。保持している FmgrInfo を通じて、PostgreSQL の関数マネージャが各戦略関数を呼び出します。

関連データ構造

  • BrinOpcInfo: 本構造体を oi_opaque として保持する親構造体。
  • FmgrInfo: 関数呼び出しを高速化するための情報構造体。

失敗モードと境界条件

  • subtype が変更された場合、キャッシュの再構築が必要です。
  • BTMaxStrategyNumber を超えるインデックスアクセスはサポートされません。

保守担当者向けメモ

  • minmax_get_strategy_procinfo は brin_inclusion.c の実装と設計が共通化されています。修正時は両方のファイルを確認してください。
  • bv_allnulls フラグが立っている範囲では、本構造体による比較処理はスキップされます。
