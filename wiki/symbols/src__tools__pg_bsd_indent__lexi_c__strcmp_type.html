strcmp_type

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/lexi.c:141

目的

bsearch 関数で使用するための、文字列比較用の比較関数を提供します。

シグネチャまたは形状

static int strcmp_type(const void *e1, const void *e2)

入力と出力

  • **入力**:
  • e1: 比較対象の文字列（const char *）。
  • e2: typenames 配列内の要素へのポインタ（const char * const *）。
  • **出力**:
  • strcmp の結果をそのまま返します（第1引数が小さい場合は負、等しい場合は0、大きい場合は正の整数）。

前提条件と不変条件

  • e2 は typenames 配列の要素（文字列へのポインタ）を指している必要があります。
  • e1 は有効なヌル終端文字列である必要があります。

アルゴリズムまたは振る舞い

1. e1 を文字列として扱います。 2. e2 を文字列へのポインタとして間接参照し、比較対象の文字列を取得します。 3. strcmp を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/lexi.c 内の bsearch 呼び出し（typenames 配列の検索時）。
  • **呼び出し先**: string.h の strcmp。

関連データ構造

  • typenames: ユーザー定義の型名を保持する動的配列。本関数はこの配列の検索に使用されます。

失敗モードと境界条件

  • e1 または e2 が NULL の場合、strcmp の仕様に従い未定義動作となります。
  • e2 が指す先が有効な文字列ポインタでない場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は bsearch のコールバックとして設計されています。typenames 配列の構造や検索ロジックを変更する際は、この比較関数の引数型との整合性に注意してください。
