trivalue

定義

  • 種別: enum
  • 定義箇所: src/include/fe_utils/connect_utils.h:17

目的

パスワードプロンプトの表示や接続設定の再利用など、3つの状態（デフォルト、無効、有効）を表現するための列挙型です。

シグネチャまたは形状

enum trivalue
{
    TRI_DEFAULT,
    TRI_NO,
    TRI_YES
};

入力と出力

  • 入力: なし（列挙定数）
  • 出力: TRI_DEFAULT（0）、TRI_NO（1）、TRI_YES（2）のいずれかの整数値。

前提条件と不変条件

  • 接続処理において、パスワードプロンプトの制御や接続パラメータの再利用判断に使用されます。
  • TRI_DEFAULT は、システムが定義する標準的な挙動を選択することを意味します。

アルゴリズムまたは振る舞い

  • 接続ロジック（connectDatabase 等）において、この値が TRI_YES である場合、対話的なパスワード入力プロンプトが実行されます。
  • 接続失敗時の再試行ロジックや、接続パラメータの再利用判断において、条件分岐のフラグとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • contrib/vacuumlo/vacuumlo.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_dump/connectdb.c / connectdb.h
  • src/bin/pg_dump/pg_backup.h / pg_backup_db.c
  • src/bin/pg_dump/pg_dumpall.c
  • src/bin/psql/command.c / settings.h
  • 呼び出し先: なし（列挙型のため）。

関連データ構造

  • ConnParams: 接続パラメータを保持する構造体。この中でパスワードプロンプトの制御用として利用されます。

失敗モードと境界条件

  • 該当なし。列挙型であるため、定義外の値が代入された場合の挙動は呼び出し側の実装に依存します。

保守担当者向けメモ

  • 本シンボルは libpq を利用する複数のフロントエンドツールで共通して使用されています。
  • 状態の追加や変更を行う際は、参照しているすべてのツール（psql, pg_dump 等）のロジックに影響を与えるため注意してください。
