ReplSlotSyncWorkerMain

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/slotsync.c:1453

目的

物理スタンバイサーバー上でバックグラウンドワーカーとして動作し、プライマリサーバーから論理レプリケーションスロットの情報を定期的に取得・同期するメインループを実行します。

シグネチャまたは形状

void ReplSlotSyncWorkerMain(const void *startup_data, size_t startup_data_len)

入力と出力

  • **入力**:
  • startup_data: 使用されません（startup_data_len == 0 であることが前提）。
  • startup_data_len: 0 である必要があります。
  • **出力**: なし（プロセス終了またはエラー発生までループを継続）。

前提条件と不変条件

  • SlotSyncCtx が初期化済みであること。
  • InitProcess および BaseInit が呼び出され、バックエンドとしての初期化が完了していること。
  • 実行プロセスは B_SLOTSYNC_WORKER として識別されること。

アルゴリズムまたは振る舞い

1. **初期化**: シグナルハンドラの設定、例外スタックの構築、共有メモリへの登録を行います。 2. **接続**: primary_conninfo を使用してプライマリサーバーへ接続し、validate_remote_info でカスケードスタンバイでないこと等の検証を行います。 3. **同期ループ**:

  • 設定変更の確認（ConfigReloadPending）。
  • synchronize_slots を呼び出し、プライマリからスロット情報を取得してローカルへ反映。
  • wait_for_slot_activity を呼び出し、同期結果に応じて待機時間を調整。

    • 4. **終了**: 停止要求またはエラー発生時に proc_exit を経由して安全に終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/launch_backend.c: バックグラウンドワーカーとして起動時に呼び出されます。
  • **呼び出し先**:
  • synchronize_slots: スロット同期の主要ロジック。
  • walrcv_connect: プライマリへの接続。
  • validate_remote_info: リモート情報の検証。
  • wait_for_slot_activity: 同期後の待機処理。
  • InitPostgres: データベース接続の確立。

関連データ構造

  • SlotSyncCtx: 同期プロセスの状態を管理する共有メモリ構造体。
  • WalReceiverConn: プライマリとの通信用接続ハンドル。

失敗モードと境界条件

  • **接続失敗**: プライマリへの接続が確立できない場合、ERROR を発行して終了します。
  • **例外発生**: sigsetjmp によりエラーを捕捉し、ログ出力後に proc_exit でクリーンアップして終了します。
  • **割り込み**: CHECK_FOR_INTERRUPTS() により、シグナル受信時に適切に処理を中断します。

保守担当者向けメモ

  • 本関数は PG_TRY を使用せず、手動で sigsetjmp を管理しています。エラーハンドリングを追加する際は、スタックの整合性に注意してください。
  • libpqwalreceiver を動的にロードするため、通信プロトコルの変更時には関連する walrcv_* 関数群の挙動を確認してください。
