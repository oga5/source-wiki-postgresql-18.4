pgunlink

定義

  • 種別: function
  • 定義箇所: src/port/dirmod.c:118

目的

Windows 環境において、ファイルまたはジャンクションポイント（シンボリックリンクの代替）の削除を、共有違反を考慮したリトライロジック付きで実行する。

シグネチャまたは形状

int pgunlink(const char *path);

入力と出力

  • **入力**: path (削除対象のファイルまたはディレクトリパス)
  • **出力**: 成功時は 0、失敗時は -1 を返す。

前提条件と不変条件

  • Windows 環境下で動作することを前提とする。
  • EACCES エラーが発生した場合、対象がジャンクションポイントである可能性があるため、lstat を用いて判定を行う。
  • 削除保留状態（STATUS_DELETE_PENDING）にあるファイルに対しては、即座にエラーを返さず、ファイルが実際に削除されるまで待機する。

アルゴリズムまたは振る舞い

1. unlink(path) を試行し、成功すれば終了する。 2. EACCES 以外のエラーであれば即座に -1 を返す。 3. EACCES の場合、lstat で対象がジャンクションポイント（S_ISLNK）かどうかを判定する。 4. 削除保留状態であれば is_lnk を false に設定し、それ以外は lstat の結果に従う。 5. is_lnk が真なら rmdir(path)、偽なら unlink(path) を実行するループに入る。 6. 共有違反等で失敗した場合、pg_usleep を用いて 100 ミリ秒間隔で最大 10 秒間（100 回）リトライを行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port.h にて unlink マクロとして定義されており、ファイル削除を行う各モジュールから呼び出される。
  • **呼び出し先**: unlink, rmdir, lstat, lstat_error_was_status_delete_pending, pg_usleep。

関連データ構造

  • struct stat: ファイル属性の取得に使用。

失敗モードと境界条件

  • 10 秒（100 回のリトライ）を超えても共有違反が解消されない場合は -1 を返す。
  • EACCES 以外のエラーが発生した場合は、リトライを行わず即座に -1 を返す。
  • 実行中にファイル種別が変更される可能性については考慮していない（設計上の許容範囲）。

保守担当者向けメモ

  • 本関数は Windows 特有の共有違反を回避するためのラッパーである。
  • lstat_error_was_status_delete_pending は、削除保留中のファイルを正しく扱うための重要なハックであるため、変更時は注意が必要。
  • ジャンクションポイントの判定は lstat に依存している。
