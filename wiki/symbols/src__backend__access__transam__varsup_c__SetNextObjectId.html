SetNextObjectId

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:622

目的

共有メモリ上のOIDカウンタを強制的に指定された値へ更新します。主に initdb 実行時の初期化プロセスで使用されます。

シグネチャまたは形状

static void SetNextObjectId(Oid nextOid)

入力と出力

  • **入力**: nextOid (設定する次回のOID値)
  • **出力**: なし（成功時）。エラー発生時は elog(ERROR) によりプロセスが中断されます。

前提条件と不変条件

  • 本関数は initdb 実行中（IsPostmasterEnvironment が偽の状態）のみ呼び出し可能です。
  • 設定しようとする nextOid は、現在の TransamVariables->nextOid よりも大きい値である必要があります。

アルゴリズムまたは振る舞い

1. **環境チェック**: IsPostmasterEnvironment が真の場合、initdb 以外での呼び出しとみなしてエラーを発生させます。 2. **排他制御**: OidGenLock を取得します。 3. **妥当性検証**: 現在の nextOid が引数 nextOid を既に超えている場合、更新不可能としてエラーを発生させます。 4. **状態更新**: TransamVariables->nextOid を引数の値で更新し、TransamVariables->oidCount を 0 にリセットします。 5. **ロック解放**: OidGenLock を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/varsup.c 内のコードから呼び出されることが確認されています。
  • **呼び出し先**: LWLockAcquire および LWLockRelease を使用して OidGenLock を操作します。また、エラー発生時に elog を呼び出します。

関連データ構造

  • TransamVariables: 共有メモリ上のグローバルなOID管理状態を保持する構造体。
  • OidGenLock: OID生成処理を保護するための軽量ロック（LWLock）。

失敗モードと境界条件

  • **Postmaster環境下での実行**: initdb 完了後に本関数を呼び出すと、elog(ERROR) により強制終了します。
  • **OIDの逆行**: 現在のOIDカウンタより小さい値を指定した場合、elog(ERROR) により強制終了します。

保守担当者向けメモ

  • 本関数は initdb 中の特殊な操作を目的としており、通常の運用環境では使用されません。
  • oidCount を 0 にリセットすることで、次回のOID払い出し時に必ずWALへの書き込み（OIDプリフェッチ）が発生するよう強制しています。
