read_stream_look_ahead

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/read_stream.c:428

目的

ストリームの先読み距離（distance）を維持するために、コールバック関数からブロック番号を補充し、必要に応じて非同期 I/O を開始する。

シグネチャまたは形状

static void read_stream_look_ahead(ReadStream *stream)

入力と出力

  • **入力**: ReadStream *stream（ストリームの状態を保持する構造体）
  • **出力**: なし（副作用として stream 構造体の内部状態および I/O 状態を更新）

前提条件と不変条件

  • stream->pinned_buffers > 0 または stream->distance == 0 であること（関数終了時の不変条件）。
  • batch_mode が有効な場合、pgaio_enter_batchmode() および pgaio_exit_batchmode() が適切に呼び出されること。

アルゴリズムまたは振る舞い

1. **バッチモード開始**: stream->batch_mode が真であれば pgaio_enter_batchmode() を実行。 2. **ブロック補充ループ**: 進行中の I/O 数およびピン留め済みバッファ数が目標距離に達するまで以下を繰り返す。

  • コールバックから次の blocknum を取得。InvalidBlockNumber の場合はストリーム終了とみなし、distance を 0 に設定して終了。
  • 取得したブロックが既存の pending_read と連続していれば、pending_read_nblocks をインクリメントしてマージ。
  • 連続していない場合、既存の pending_read を read_stream_start_pending_read() で実行し、新しい pending_read を開始。

    • 3. **I/O 実行判定**: ループ終了後、io_combine_limit に達しているか、距離制限を満たしている場合に read_stream_start_pending_read() を呼び出し、保留中の I/O を発行。 4. **バッチモード終了**: pgaio_exit_batchmode() を実行。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/read_stream.c:429 (自身)
  • src/backend/storage/aio/read_stream.c:889
  • src/backend/storage/aio/read_stream.c:991
  • **呼び出し先**:
  • pgaio_enter_batchmode() / pgaio_exit_batchmode()
  • read_stream_start_pending_read()
  • get_per_buffer_data()
  • read_stream_get_block()
  • read_stream_unget_block()

関連データ構造

  • ReadStream: ストリームの制御状態（distance, pending_read_nblocks, ios_in_progress 等）を保持。

失敗モードと境界条件

  • **ストリーム終了**: コールバックが InvalidBlockNumber を返すと、distance を 0 に設定し、それ以上の先読みを停止する。
  • **I/O 制限**: max_ios に達した場合、または io_combine_limit に達した場合は、現在の保留分を処理し、それ以上の追加を一時停止する。
  • **バッファ制限**: pinned_buffers が目標距離に達した場合、新たな I/O は発行されない。

保守担当者向けメモ

  • 本関数は、I/O 提出コストを償却するために複数のブロックをマージする役割を持つ。
  • pending_read のマージロジックは、buffer_index の循環計算を含んでおり、変更時は queue_size との整合性に注意すること。
  • Assert 文により、関数終了時に必ず何らかのバッファがピン留めされているか、ストリームが終了していることが保証されている。
