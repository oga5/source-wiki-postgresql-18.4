euc_tw_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:83

目的

EUC_TW エンコーディングから MULE_INTERNAL (MIC) エンコーディングへ文字列を変換する、PostgreSQLの変換プロシージャです。

シグネチャまたは形状

Datum euc_tw_to_mic(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • src (CSTRING): 変換元の EUC_TW 文字列。
  • dest (CSTRING): 変換後のデータを格納するバッファ。
  • len (INT32): 入力文字列のバイト長。
  • noError (BOOL): 変換エラー発生時の挙動制御フラグ。
  • **出力**: Datum 型として、変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_EUC_TW, PG_MULE_INTERNAL) により、エンコーディング変換の引数が妥当である必要があります。
  • noError が true の場合、変換エラーが発生しても例外を投げず、処理を中断してそこまでの変換バイト数を返します。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの妥当性を検証します。 3. 内部関数 euc_tw2mic を呼び出し、実際のバイト変換ロジックを実行します。 4. 変換されたバイト数を PG_RETURN_INT32 で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1(euc_tw_to_mic) により登録）。
  • **呼び出し先**: euc_tw2mic（実際の変換ロジック）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約で使用される引数構造体。

失敗モードと境界条件

  • noError が false の場合、不正な文字シーケンスが検出されると例外が発生します。
  • noError が true の場合、不正なシーケンスで処理を中断し、正常に変換できたバイト数のみを返します。

保守担当者向けメモ

  • 本関数はインターフェース層であり、具体的な変換ロジックは euc_tw2mic に依存しています。
  • 変換テーブルや詳細な計算式については、euc_tw2mic の実装を参照してください。
