SPIExecuteOptions

定義

  • 種別: struct
  • 定義箇所: src/include/executor/spi.h:46

目的

SPI（Server Programming Interface）を介して SQL コマンドやプリペアドプランを実行する際の、実行パラメータおよびオプションをカプセル化するために使用されます。

シグネチャまたは形状

typedef struct SPIExecuteOptions
{
	ParamListInfo params;       /* パラメータリスト */
	bool		read_only;      /* 読み取り専用フラグ */
	bool		allow_nonatomic; /* 非アトミック実行の許可 */
	bool		must_return_tuples; /* タプル返却の強制 */
	uint64		tcount;         /* 処理対象のタプル数制限 */
	DestReceiver *dest;         /* 結果の出力先 */
	ResourceOwner owner;        /* リソース所有者 */
} SPIExecuteOptions;

入力と出力

  • **入力**: 実行エンジンに対して、パラメータ値（params）、実行制約（read_only, tcount）、および実行モード（allow_nonatomic）を渡します。
  • **出力**: 実行結果の転送先（dest）や、リソース管理のための所有者情報（owner）を保持します。

前提条件と不変条件

  • SPI_execute_plan などの関数に渡される際、構造体は適切に初期化されている必要があります。
  • dest が指定されている場合、実行結果はそのレシーバーへ送られます。

アルゴリズムまたは振る舞い

SPI 実行関数（_SPI_execute_plan 等）は、この構造体を参照して実行計画の挙動を制御します。特に、read_only フラグはトランザクションの書き込み制限を決定し、tcount は実行時に処理されるタプル数の上限を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/spi.c 内の _SPI_execute_plan 関数が、この構造体を引数として受け取ります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、実行エンジンがこの構造体のフィールドを参照して処理を分岐させます。

関連データ構造

  • ParamListInfo: パラメータの受け渡しに使用。
  • DestReceiver: 実行結果の出力先を定義。
  • ResourceOwner: リソース管理に使用。
  • SPIPlanPtr: 実行対象となるプリペアドプラン。

失敗モードと境界条件

  • tcount に 0 を指定した場合の挙動は、実行関数の実装に依存します。
  • owner が正しく設定されていない場合、リソースの解放漏れや不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/executor/spi.h で定義されており、SPI の API 変更に伴いフィールドが追加・変更される可能性があります。
  • src/backend/executor/spi.c 内の _SPI_execute_plan が主要な利用者であるため、変更時は同ファイルの整合性を確認してください。
