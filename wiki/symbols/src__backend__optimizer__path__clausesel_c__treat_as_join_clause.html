treat_as_join_clause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/clausesel.c:585

目的

与えられた条件式（clause）が、現在のプランナの文脈において「制限（Restriction）」として扱うべきか、「結合（Join）」として扱うべきかを判定するヘルパー関数です。

シグネチャまたは形状

static inline bool treat_as_join_clause(PlannerInfo *root, Node *clause, RestrictInfo *rinfo, int varRelid, SpecialJoinInfo *sjinfo)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • clause: 評価対象の式。
  • rinfo: 関連する RestrictInfo（存在する場合）。
  • varRelid: 呼び出し元が強制するリレーションID（0以外の場合は制限モードを強制）。
  • sjinfo: 結合情報（SpecialJoinInfo）。
  • **出力**:
  • bool: true の場合、結合条件として扱います。false の場合は制限条件として扱います。

前提条件と不変条件

  • varRelid が 0 以外の場合、強制的に制限モード（false）を返します。
  • sjinfo が NULL の場合、スキャンノードで評価される制限条件とみなされ、false を返します。

アルゴリズムまたは振る舞い

1. **強制モードの確認**: varRelid != 0 ならば false を返します。 2. **結合情報の確認**: sjinfo == NULL ならば false を返します。 3. **結合判定**: 上記以外の場合、複数のベースリレーションが関与しているかを判定します。

  • rinfo が提供されている場合: rinfo->num_base_rels > 1 を評価します。
  • rinfo がない場合: NumRelids(root, clause) > 1 を評価します。
  • 外部結合（OJ）による遅延条件であっても、ベースリレーションの数のみをカウントし、制限条件として扱うよう誘導します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/clausesel.c 内の複数箇所（836行目、872行目、883行目付近）。
  • **呼び出し先**:
  • NumRelids（rinfo がない場合）。
  • **補足**: src/backend/utils/adt/selfuncs.c:2240 に言及がありますが、直接の呼び出し関係についてはソースコードから明示的なフローは確認できません。

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • RestrictInfo: 条件式のキャッシュおよびメタデータ。
  • SpecialJoinInfo: 結合の特性を保持する構造体。

失敗モードと境界条件

  • rinfo が NULL の場合、NumRelids を使用して式を解析するため、式が複雑な場合は計算コストが増加する可能性があります。
  • 外部結合によって注入される NULL の可能性については、本関数ではなく制限条件の推定器側で対処することが意図されています。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを考慮して最適化されています。
  • 結合条件か制限条件かの判定ロジックを変更する際は、num_base_rels のカウント方針が拡張統計や外部結合の扱いに影響を与える点に注意してください。
