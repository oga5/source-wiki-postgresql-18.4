ReplicationOriginNameForLogicalRep

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/worker.c:420

目的

論理レプリケーションで使用されるレプリケーションオリジン（Replication Origin）の名称を生成します。

シグネチャまたは形状

void ReplicationOriginNameForLogicalRep(Oid suboid, Oid relid, char *originname, Size szoriginname)

入力と出力

  • **入力**:
  • suboid (Oid): サブスクリプションのOID。
  • relid (Oid): テーブルのOID（テーブル同期ワーカーの場合）。InvalidOid を指定可能。
  • szoriginname (Size): originname バッファのサイズ。
  • **出力**:
  • originname (char *): 生成された名称が格納されるバッファ。

前提条件と不変条件

  • originname は、生成される文字列（pg_ + OID + 必要に応じて _ + OID）を格納するのに十分なサイズである必要があります。

アルゴリズムまたは振る舞い

1. relid が有効なOIDである場合、テーブル同期ワーカー用の名称として "pg_%u_%u"（サブスクリプションOIDとテーブルOID）を生成します。 2. relid が InvalidOid である場合、通常のワーカー用の名称として "pg_%u"（サブスクリプションOIDのみ）を生成します。 3. 生成された文字列は snprintf を使用して originname に書き込まれます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/backend/replication/logical/tablesync.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • snprintf（標準ライブラリ関数）

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • originname バッファが短すぎる場合、snprintf の仕様に従い文字列が切り詰められます。

保守担当者向けメモ

  • 本関数は論理レプリケーションのオリジン名の一貫性を保つために使用されます。名称フォーマットを変更する場合は、関連するすべての呼び出し箇所で整合性が取れているか確認してください。
