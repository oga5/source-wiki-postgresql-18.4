PGPROC

定義

  • 種別: struct
  • 定義箇所: src/include/storage/proc.h:176

目的

PostgreSQLの各バックエンドプロセスが共有メモリ上で保持するプロセス制御構造体です。バックエンドのライフサイクル管理、ロック待ち行列の管理、およびトランザクション状態の共有を担います。

シグネチャまたは形状

共有メモリ上に配置される大規模な構造体であり、プロセス状態（PID、XID、待機中のロック情報）、同期プリミティブ（セマフォ、ラッチ）、およびロック管理用のリスト構造を保持します。

入力と出力

  • **入力**: InitProcess() による初期化時に ProcGlobal->freeProcs から割り当てられ、トランザクション開始・終了時に xid や statusFlags が更新されます。
  • **出力**: ProcGlobal を通じた全バックエンドの状態集約や、ロックマネージャによる待機キューの参照に使用されます。

前提条件と不変条件

  • 一部のフィールドは ProcArrayLock 等の保持が必須ですが、一部はロックなしでアクセス可能です。
  • xid や statusFlags 等のミラーリングされたフィールドは、ProcGlobal 配列と整合性を保つ必要があります。
  • vxid はアトミックに代入できないため、コード上では procNumber と lxid を個別に扱う必要があります。

アルゴリズムまたは振る舞い

  • **プロセス待機**: ProcSleep() 呼び出し時、対象の PGPROC がロックの待機キューにリンクされ、セマフォで待機状態に入ります。
  • **状態共有**: ProcGlobal 内のミラー配列を通じて、他のバックエンドが効率的にトランザクション状態を参照可能です。
  • **グループ処理**: procArrayGroupMember や clogGroupMember 等のフラグを用い、グループ単位でのXIDクリアやステータス更新をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/clog.c（グループ更新処理）、src/backend/access/heap/heapam_visibility.c（可視性判定）、src/backend/access/nbtree/nbtxlog.c（競合テスト）等から参照されます。
  • **呼び出し先**: storage/latch.h（ラッチ制御）、storage/lock.h（ロック管理）、lib/ilist.h（リスト管理）等の機能を利用します。

関連データ構造

  • PROC_HDR: 全 PGPROC を管理するグローバル構造体。
  • ProcGlobal: プロセス管理情報のグローバル変数。
  • LOCK, PROCLOCK: ロック管理用構造体。
  • dlist_node, proclist_node: リスト管理用構造体。

失敗モードと境界条件

  • PGPROC の配列がオーバーフローする場合、グループ更新等の機能が制限される可能性があります。
  • 準備されたトランザクション（prepared xact）では pid が 0 になるなど、状態によってフィールドの解釈が異なります。

保守担当者向けメモ

  • PGPROC のフィールドには、ロックなしでアクセス可能なものと、必須なものが混在しています。特にミラーリングされたフィールドへのアクセスには注意が必要です。
  • pgxactoff を用いたインデックスアクセスはキャッシュ効率を最適化するための設計です。変更時は ProcArray 周りの設計ドキュメントを併読してください。
