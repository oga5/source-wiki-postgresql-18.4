QueryOperand

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_type.h:158

目的

tsquery 内の検索対象となる語彙（オペランド）の属性を保持する構造体です。検索時の重み付け、プレフィックス検索の有無、および語彙の識別情報を管理します。

シグネチャまたは形状

typedef struct {
    QueryItemType type;
    uint8         weight;
    bool          prefix;
    int32         valcrc;
    uint32        length:12,
                  distance:20;
} QueryOperand;

入力と出力

  • **入力**: tsquery の構築時または解析時に、語彙のメタデータ（重み、プレフィックスフラグ、CRC値、文字列長、距離）が本構造体に格納されます。
  • **出力**: 検索実行時、checkcondition 系の関数（tsginidx.c, tsgistidx.c 等）において、検索条件の評価に使用されます。

前提条件と不変条件

  • weight はビットマスクとして機能します（A: 1<<3, B: 1<<2, C: 1<<1, D: 1<<0）。値が 0 の場合はすべての重みが許可されます。
  • length および distance はビットフィールドとして定義されており、合計 32 ビットに収められています。
  • valcrc は語彙の識別に使用されますが、比較演算の都合上 int32 型として扱われています。

アルゴリズムまたは振る舞い

  • tsquery 内のノードとして、演算子（QueryOperator）と混在して配置されます。
  • 検索エンジンは、この構造体の weight ビットマスクと prefix フラグを参照し、インデックス内の WordEntry と照合して一致判定を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_selfuncs.c
  • src/backend/tsearch/wparser_def.c
  • src/backend/utils/adt/tsginidx.c
  • src/backend/utils/adt/tsgistidx.c
  • src/backend/utils/adt/tsquery.c
  • **呼び出し先**: 本構造体はデータ保持用であり、直接的な関数呼び出し先は定義されていません。各検索エンジン（GIN/GiST）の checkcondition 関数が本構造体を引数として受け取り、評価を行います。

関連データ構造

  • QueryItem: QueryOperand と QueryOperator を包含する共用体。
  • WordEntry: tsvector 内の語彙メタデータ。QueryOperand の値と相関関係を持つ必要があります。

失敗モードと境界条件

  • valcrc の比較において、符号付き整数としての比較が行われるため、CRC 計算時の型変換に注意が必要です。
  • length は 12 ビット（最大 4095）、distance は 20 ビット（最大 1,048,575）の制限があります。

保守担当者向けメモ

  • valcrc は本来 pg_crc32 が適切ですが、既存コードとの互換性のため int32 が使用されています。変更には広範囲な修正が必要です。
  • QueryItem 共用体の一部としてメモリ配置されるため、構造体のサイズ変更は tsquery 全体のバイナリレイアウトに影響を与えます。
