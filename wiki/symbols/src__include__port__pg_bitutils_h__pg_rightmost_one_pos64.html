pg_rightmost_one_pos64

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/pg_bitutils.h:144

目的

64ビット整数において、最下位ビット（LSB）が立っている位置（0から始まるインデックス）を特定する。

シグネチャまたは形状

int pg_rightmost_one_pos64(uint64 word)

入力と出力

  • **入力**: uint64 word（対象となる64ビット整数）
  • **出力**: int（LSBの位置。0〜63の範囲）

前提条件と不変条件

  • word != 0 であること。0が渡された場合、アサーションエラーが発生するか、未定義の動作となる。

アルゴリズムまたは振る舞い

環境に応じて以下のいずれかの手法を選択する： 1. **コンパイラ組み込み関数**: HAVE__BUILTIN_CTZ が定義されている場合、__builtin_ctzl または __builtin_ctzll を使用する。 2. **MSVC組み込み関数**: Windows環境（AMD64/ARM64）では _BitScanForward64 を使用する。 3. **フォールバック**: 上記が利用できない場合、8ビットずつシフトしながらループし、pg_rightmost_one_pos ルックアップテーブルを用いて位置を算出する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/method_worker.c:173
  • src/include/nodes/bitmapset.h:83（bmw_rightmost_one_pos マクロ経由）
  • **呼び出し先**:
  • pg_rightmost_one_pos（フォールバック時のみ使用されるルックアップテーブル）

関連データ構造

  • pg_rightmost_one_pos: フォールバック実装で使用される、8ビット値に対するLSB位置を保持するルックアップテーブル。

失敗モードと境界条件

  • word が 0 の場合、Assert(word != 0) によりデバッグビルドでは停止する。
  • 組み込み関数が利用できない環境で、かつ uint64 と同じサイズの整数型が特定できない場合はコンパイルエラーとなる。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスが重視される箇所で使用される。
  • 新しいプラットフォームで最適化を追加する場合は、pg_bitutils.c にあるルックアップテーブルとの整合性およびコンパイラ固有の組み込み関数のサポート状況を確認すること。
