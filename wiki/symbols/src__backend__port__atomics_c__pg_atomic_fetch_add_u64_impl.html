pg_atomic_fetch_add_u64_impl

定義

  • 種別: function
  • 定義箇所: src/backend/port/atomics.c:61

目的

ハードウェアによるネイティブな 64 ビットアトミック加算がサポートされていない環境において、スピンロックを用いてアトミックな加算操作と古い値の取得をシミュレートします。

シグネチャまたは形状

uint64 pg_atomic_fetch_add_u64_impl(volatile pg_atomic_uint64 *ptr, int64 add_)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint64 構造体へのポインタ。
  • add_: 加算する値（int64）。
  • **出力**:
  • 加算前の値（uint64）。

前提条件と不変条件

  • システムが PG_HAVE_ATOMIC_U64_SIMULATION を定義している環境でのみ使用されます。
  • ptr が指す構造体内の sema（スピンロック）が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. SpinLockAcquire を呼び出し、ptr->sema を用いて排他ロックを取得します。 2. 現在の ptr->value を oldval に退避します。 3. ptr->value に add_ を加算します。 4. SpinLockRelease を呼び出し、ptr->sema を解放します。 5. 退避しておいた oldval を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics.h および src/include/port/atomics/generic.h 経由で、アトミック操作を要求する各モジュールから呼び出されます。
  • **呼び出し先**: SpinLockAcquire および SpinLockRelease（src/include/storage/spin.h で定義）。

関連データ構造

  • pg_atomic_uint64: アトミック操作対象の値を保持する構造体。スピンロック（slock_t）と値（uint64）を含みます。

失敗モードと境界条件

  • スピンロックの取得に失敗した場合や、デッドロックが発生する状況下では処理が停止する可能性があります。
  • add_ による加算結果が 64 ビットの範囲を超える場合、オーバーフローが発生します。

保守担当者向けメモ

  • 本関数はパフォーマンスが重要なパスではオーバーヘッドが大きいため、可能な限りネイティブなアトミック命令が利用される環境を優先してください。
  • スピンロックによる排他制御を行っているため、ロックの保持時間は最小限に留める必要があります。
