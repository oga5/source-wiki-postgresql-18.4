bbsink_gzip_end_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_gzip.c:224

目的

gzip圧縮ストリームを終了させ、残存する圧縮データをフラッシュして後続のシンクへ転送する。

シグネチャまたは形状

static void bbsink_gzip_end_archive(bbsink *sink);

入力と出力

  • **入力**: bbsink *sink （bbsink_gzip 構造体としてキャストして使用）
  • **出力**: なし（副作用として後続のシンクへデータが転送される）

前提条件と不変条件

  • sink は bbsink_gzip 型のインスタンスであること。
  • deflateInit2 等による圧縮準備が完了していること。
  • bbsink_gzip_archive_contents によるデータ処理が終了していること。

アルゴリズムまたは振る舞い

1. z_stream の入力バッファを空に設定する。 2. deflate を Z_FINISH モードで呼び出し、内部バッファに残っている圧縮データをすべて出力バッファへ書き出す。 3. 出力バッファが満杯になるか、圧縮が完了するまでループする。 4. 出力バッファにデータがある場合、bbsink_archive_contents を呼び出して後続のシンクへ転送する。 5. 最後に bbsink_forward_end_archive を呼び出し、アーカイブの終了を後続のシンクへ通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink 構造体の .end_archive フィールドを通じて呼び出される（直接的な呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**:
  • deflate (zlib)
  • bbsink_archive_contents
  • bbsink_forward_end_archive

関連データ構造

  • bbsink_gzip: 圧縮状態（z_stream）やバッファ情報を保持する構造体。
  • z_stream: zlib の圧縮状態管理構造体。

失敗モードと境界条件

  • deflate が Z_STREAM_ERROR を返した場合、elog(ERROR) により処理を中断する。

保守担当者向けメモ

  • 本関数は HAVE_LIBZ が定義されている環境でのみ有効です。
  • Z_FINISH を使用しているため、呼び出し後は圧縮ストリームを再開できません。
