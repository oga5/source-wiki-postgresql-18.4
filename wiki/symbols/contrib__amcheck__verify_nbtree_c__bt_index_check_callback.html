bt_index_check_callback

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_nbtree.c:311

目的

B-Tree インデックスのメタデータ検証および構造的整合性チェックを実行するコールバック関数です。

シグネチャまたは形状

static void bt_index_check_callback(Relation indrel, Relation heaprel, void *state, bool readonly)

入力と出力

  • indrel: 検証対象のインデックスリレーション。
  • heaprel: インデックスが属するヒープリレーション（NULL の場合あり）。
  • state: BTCallbackState 構造体へのポインタ。
  • readonly: 読み取り専用モードで実行するかを示すフラグ。
  • 出力: なし。検証失敗時は ereport(ERROR, ...) を通じて例外を発生させます。

前提条件と不変条件

  • インデックスのメインフォーク（MAIN_FORKNUM）が存在すること。
  • メタページから取得した heapkeyspace および allequalimage の値が、現在の nbtree バージョンと整合していること。
  • allequalimage が真の場合、インデックスが重複排除（deduplication）に対して安全であると正しくマークされていること。

アルゴリズムまたは振る舞い

1. **存在確認**: smgrexists を使用してインデックスのメインフォークを確認し、存在しなければエラーを報告します。 2. **メタデータ検証**: _bt_metaversion を呼び出し、インデックスのメタデータを抽出・検証します。 3. **互換性チェック**: allequalimage が設定されている場合、_bt_allequalimage を用いてインデックスの属性が重複排除に適しているかを確認します。INTERVAL 型など、過去のバージョンで問題があったケースではエラーを発生させます。 4. **構造検証**: bt_check_every_level を呼び出し、インデックスの全レベルを走査して論理的な整合性を検証します。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内の直接的な呼び出し元は明示されていませんが、BTCallbackState を引数として渡す検証プロセスの一部として使用されます。
  • 呼び出し先: smgrexists, _bt_metaversion, _bt_allequalimage, IndexRelationGetNumberOfKeyAttributes, bt_check_every_level。

関連データ構造

  • BTCallbackState: 検証に必要な状態（heapallindexed, rootdescend, checkunique 等）を保持します。
  • Relation: PostgreSQL のリレーション記述子。

失敗モードと境界条件

  • インデックスのメインフォーク欠損時に ERRCODE_INDEX_CORRUPTED を発生させます。
  • メタページの equalimage フィールドがサポート外のバージョンで設定されている場合、または重複排除の安全性が誤って示されている場合にエラーを発生させます。

保守担当者向けメモ

  • 本関数は bt_check_every_level を呼び出すためのラッパーとして機能します。
  • readonly フラグは、インデックスのロック状態に応じた検証の厳密さを制御するために bt_check_every_level へ伝播されます。
