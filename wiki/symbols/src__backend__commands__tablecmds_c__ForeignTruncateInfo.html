ForeignTruncateInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/tablecmds.c:346

目的

外部テーブルに対する TRUNCATE 操作において、特定の外部サーバーに関連付けられたリレーションの情報を保持するために使用されます。

シグネチャまたは形状

typedef struct ForeignTruncateInfo
{
    Oid         serverid;
    List       *rels;
} ForeignTruncateInfo;

入力と出力

  • serverid: 外部サーバーを識別する Oid。
  • rels: 当該サーバーに関連するリレーション（テーブル）のリスト。

前提条件と不変条件

  • TRUNCATE コマンド実行時に、外部テーブルが対象に含まれる場合にインスタンス化されます。
  • 外部サーバー単位でグループ化されたリレーション情報を管理します。

アルゴリズムまたは振る舞い

  • TRUNCATE 処理の過程で、外部テーブルをサーバーごとに分類し、一括して処理するためのデータ構造として機能します。
  • ハッシュテーブル等のエントリサイズ計算（sizeof(ForeignTruncateInfo)）に使用されることがあります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/tablecmds.c 内の TRUNCATE 関連処理（2163行目、2259行目付近）。
  • 呼び出し先: 具体的な関数呼び出し関係はソース抜粋からは特定できません。

関連データ構造

  • List: PostgreSQL の汎用リスト構造。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • 外部サーバーに関連付けられていないテーブルや、空のリストが渡された場合の挙動については、本ソース抜粋からは不明です。

保守担当者向けメモ

  • 本構造体は tablecmds.c 内の TRUNCATE ロジックにおいて、外部テーブルの管理を効率化するために利用されています。
  • 外部データラッパー（FDW）に関連する処理フローの一部であるため、変更時は foreign/fdwapi.h 等の関連定義との整合性に注意してください。
