BackgroundWorkerHandle

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/bgworker.c:102

目的

バックグラウンドワーカーのライフサイクル管理および状態追跡を行うためのハンドル構造体です。ワーカーの登録後、そのワーカーを識別し、起動待機や終了指示を行うための参照として使用されます。

シグネチャまたは形状

struct BackgroundWorkerHandle
{
    int         slot;       /* 共有メモリ上のスロット番号 */
    uint64      generation; /* スロットの再利用を識別するための世代番号 */
};

入力と出力

  • 本構造体は、RegisterDynamicBackgroundWorker 等の関数を通じて生成・取得されます。
  • 呼び出し元は、このハンドルを介してワーカーの PID 取得や終了指示を行います。

前提条件と不変条件

  • slot は BackgroundWorkerArray 内の有効なインデックスを指す必要があります。
  • generation は、スロットが再利用された際に古いハンドルによる誤操作を防ぐための不変条件として機能します。

アルゴリズムまたは振る舞い

  • 本構造体は、Postmasterとバックエンドプロセス間で共有されるワーカー状態を安全に参照するために使用されます。
  • generation を照合することで、ハンドルが指すスロットが別のワーカーに割り当て直されていないかを確認します。

呼び出し元と呼び出し先

  • **主な呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c
  • src/backend/postmaster/bgworker.c (内部関数)
  • src/backend/replication/logical/launcher.c
  • src/backend/storage/ipc/shm_mq.c
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、関数を呼び出すことはありません。本構造体を引数に取る関数として GetBackgroundWorkerPid, WaitForBackgroundWorkerStartup, WaitForBackgroundWorkerShutdown, TerminateBackgroundWorker が存在します。

関連データ構造

  • BackgroundWorkerArray: 共有メモリ上のワーカー状態配列。
  • BackgroundWorkerSlot: 各ワーカーの詳細状態を保持する構造体。

失敗モードと境界条件

  • 無効なハンドル（初期化されていない、または既に解放されたスロットを指すもの）を操作した場合の挙動は未定義です。
  • 共有メモリ上のスロットが再利用された場合、generation の不一致により操作が拒否される可能性があります。

保守担当者向けメモ

  • 本構造体は palloc を用いて動的に割り当てられることが一般的です。
  • 共有メモリのロックレスプロトコルと密接に関連しているため、ハンドルを介した操作時には bgworker.c 内の同期ロジックを遵守してください。
