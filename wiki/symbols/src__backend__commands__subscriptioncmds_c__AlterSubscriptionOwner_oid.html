AlterSubscriptionOwner_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/subscriptioncmds.c:2065

目的

指定された OID を持つサブスクリプションの所有者を変更します。

シグネチャまたは形状

void AlterSubscriptionOwner_oid(Oid subid, Oid newOwnerId)

入力と出力

  • subid (Oid): 所有者を変更する対象のサブスクリプションの OID。
  • newOwnerId (Oid): 新しい所有者となるロールの OID。
  • 出力: なし（成功時はカタログを更新し、失敗時はエラーをスロー）。

前提条件と不変条件

  • pg_subscription システムカタログに対する RowExclusiveLock を保持する必要があります。
  • 指定された subid に対応するサブスクリプションがシステムカタログ上に存在しなければなりません。

アルゴリズムまたは振る舞い

1. table_open を呼び出し、pg_subscription テーブルを RowExclusiveLock モードで開きます。 2. SearchSysCacheCopy1 を使用して、対象サブスクリプションのタプルをキャッシュから取得します。 3. タプルが存在しない場合は ERRCODE_UNDEFINED_OBJECT エラーを発生させます。 4. AlterSubscriptionOwner_internal を呼び出し、実際の所有者変更処理（カタログ更新および依存関係の管理）を実行します。 5. heap_freetuple で取得したタプルを解放します。 6. table_close でテーブルを閉じます。

呼び出し元と呼び出し先

関連データ構造

  • HeapTuple: システムカタログの行データ。
  • Relation: pg_subscription テーブルへのアクセス用ハンドル。

失敗モードと境界条件

  • 指定した subid が存在しない場合、ereport(ERROR, ...) により処理が中断されます。
  • ロック取得やカタログ更新時にデータベースの制約に抵触した場合、PostgreSQL の標準的なエラーメカニズムにより中断されます。

保守担当者向けメモ

  • 本関数は AlterSubscriptionOwner_internal のラッパーとして機能しており、主に OID からタプルを検索する役割を担います。
  • 権限チェックや所有者変更のロジックを変更する場合は、AlterSubscriptionOwner_internal を参照してください。
