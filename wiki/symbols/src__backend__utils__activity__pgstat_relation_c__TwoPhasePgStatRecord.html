TwoPhasePgStatRecord

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:30

目的

2フェーズコミット（2PC）において、準備されたトランザクション（PREPARED XACT）に関連するリレーションの統計情報を永続化および復元するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct TwoPhasePgStatRecord
{
    PgStat_Counter tuples_inserted;
    PgStat_Counter tuples_updated;
    PgStat_Counter tuples_deleted;
    PgStat_Counter inserted_pre_truncdrop;
    PgStat_Counter updated_pre_truncdrop;
    PgStat_Counter deleted_pre_truncdrop;
    Oid            id;
    bool           shared;
    bool           truncdropped;
} TwoPhasePgStatRecord;

入力と出力

  • **入力**: トランザクション実行中に蓄積されたリレーションの統計カウンタ値およびリレーション識別子。
  • **出力**: 2PCの準備フェーズにおいて、共有メモリまたはディスクへ書き出すためのシリアライズされた統計データ。

前提条件と不変条件

  • id は対象リレーションの OID を保持する必要があります。
  • truncdropped が true の場合、*_pre_truncdrop フィールドに TRUNCATE/DROP 実行前の統計値が保持されている必要があります。

アルゴリズムまたは振る舞い

1. **保存**: トランザクションの準備時に、現在の統計状態を本構造体にコピーし、2PC状態の一部として保存します。 2. **復元**: 準備されたトランザクションがコミットまたはロールバックされる際、本構造体から統計情報を読み出し、システム全体の統計へ反映させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_relation.c 内の AtPrepare_PgStat_Relations（保存時）および PostPrepare_PgStat_Relations（復元時）から参照されています。
  • **呼び出し先**: 直接的な関数呼び出し先はありません（データ保持用の構造体です）。

関連データ構造

  • PgStat_Counter: 統計カウンタの型定義。
  • PgStat_TableStatus: 実行中のリレーション統計を保持する構造体。

失敗モードと境界条件

  • 共有カタログ（shared が true）と通常のリレーションで統計の扱いが異なります。
  • TRUNCATE や DROP が発生したトランザクションでは、リセット前の値を *_pre_truncdrop フィールドで保持しないと統計の不整合が生じます。

保守担当者向けメモ

  • 本構造体は 2PC の状態管理に直結しているため、フィールドの追加や変更を行う際は AtPrepare_PgStat_Relations および PostPrepare_PgStat_Relations でのシリアライズ/デシリアライズ処理を必ず更新してください。
