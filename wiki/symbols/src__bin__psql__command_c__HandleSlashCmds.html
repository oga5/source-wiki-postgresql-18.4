HandleSlashCmds

定義

  • 種別: function
  • 定義箇所: src/bin/psql/command.c:230

目的

psql の対話型ターミナルにおいて、ユーザーが入力したバックスラッシュコマンド（\ で始まるコマンド）を解析し、適切な実行関数へディスパッチして結果を制御する。

シグネチャまたは形状

backslashResult HandleSlashCmds(PsqlScanState scan_state,
                                ConditionalStack cstack,
                                PQExpBuffer query_buf,
                                PQExpBuffer previous_buf)

入力と出力

  • **入力**:
  • scan_state: 字句解析器の状態。
  • cstack: 条件付き実行（\if 等）のスタック。
  • query_buf, previous_buf: クエリバッファ。
  • **出力**:
  • backslashResult: コマンドの実行結果（PSQL_CMD_SEND, PSQL_CMD_TERMINATE, PSQL_CMD_ERROR 等）。

前提条件と不変条件

  • scan_state および cstack は NULL であってはならない（Assert による検証）。
  • 制限モード（restricted）が有効な場合、\unrestrict 以外のコマンドは実行できない。

アルゴリズムまたは振る舞い

1. psql_scan_slash_command を使用してコマンド名を取得する。 2. 制限モードを確認し、許可されていない場合はエラーを返す。 3. exec_command を呼び出し、コマンド名に基づいた実行関数へ処理を委譲する。 4. コマンドが不明（PSQL_CMD_UNKNOWN）な場合はエラーログを出力する。 5. 正常終了時、残りの引数を読み飛ばす（psql_scan_slash_option を使用）。エラー発生時は行の残りを破棄する。 6. psql_scan_slash_command_end で末尾の \\ を処理し、メモリを解放する。 7. pset.queryFout をフラッシュして出力を確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/mainloop.c
  • src/bin/psql/startup.c
  • **呼び出し先**:
  • exec_command（コマンド実行の振り分け）
  • psql_scan_slash_command / psql_scan_slash_option（字句解析）
  • conditional_stack_push / conditional_stack_pop（条件スタック操作）
  • pg_log_error / pg_log_warning（ログ出力）

関連データ構造

  • backslashResult: コマンド実行状態を示す列挙型。
  • PsqlScanState: 字句解析器の状態保持構造体。
  • ConditionalStack: \if 等の条件分岐管理用スタック。
  • PQExpBuffer: クエリ文字列バッファ。

失敗モードと境界条件

  • **制限モード違反**: restricted フラグが真の状態で \unrestrict 以外のコマンドが入力された場合、PSQL_CMD_ERROR を返す。
  • **コマンド不明**: exec_command が PSQL_CMD_UNKNOWN を返した場合、エラーメッセージを表示し PSQL_CMD_ERROR に変換する。
  • **引数過多**: 有効なコマンドの後に引数が存在する場合、警告を出力して破棄する。

保守担当者向けメモ

  • 新しいコマンドを追加する場合は exec_command を更新すること。
  • \if ブロック内の処理は cstack の状態に依存するため、引数の読み飛ばし処理（conditional_stack_push 等）の整合性に注意が必要。
  • pset.queryFout のフラッシュは、コマンドが外部ファイルやストリームへ出力を行う可能性があるため必須である。
