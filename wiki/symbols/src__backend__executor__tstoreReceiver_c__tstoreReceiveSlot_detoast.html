tstoreReceiveSlot_detoast

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tstoreReceiver.c:135

目的

実行エンジンから受け取ったタプル内の外部 TOAST データを強制的に展開（detoast）し、その結果を Tuplestore に格納する。

シグネチャまたは形状

static bool tstoreReceiveSlot_detoast(TupleTableSlot *slot, DestReceiver *self)

入力と出力

  • **入力**:
  • slot: 格納対象のタプルを含む TupleTableSlot。
  • self: TStoreState にキャスト可能な DestReceiver オブジェクト。
  • **出力**:
  • 常に true を返す。

前提条件と不変条件

  • slot は slot_getallattrs により属性が完全に分解されている必要がある。
  • TStoreState の cxt（メモリコンテキスト）が適切に設定されていること。
  • detoast 処理とタプル変換（tupmap）の同時使用はサポートされていない（SetTuplestoreDestReceiverParams 内の Assert により保証）。

アルゴリズムまたは振る舞い

1. slot_getallattrs(slot) を呼び出し、タプルを完全に分解する。 2. タプルの各属性を走査し、外部 TOAST データ（VARATT_IS_EXTERNAL）が存在する場合は detoast_external_attr で展開する。 3. 展開した値は myState->tofree に一時保存し、後で解放できるようにする。 4. 展開後の値または元の値を myState->outvalues 配列に格納する。 5. MemoryContextSwitchTo を使用して myState->cxt に切り替え、tuplestore_putvalues で Tuplestore に書き込む。 6. 書き込み完了後、一時的に確保した展開済みデータを pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（DestReceiver インターフェース経由）。
  • **呼び出し先**:
  • slot_getallattrs
  • detoast_external_attr
  • tuplestore_putvalues
  • MemoryContextSwitchTo
  • pfree

関連データ構造

  • TStoreState: 格納先 Tuplestore やメモリコンテキスト、一時バッファを保持する構造体。
  • TupleTableSlot: 処理対象のタプルを保持する。
  • DestReceiver: 汎用的なデータ受信インターフェース。

失敗モードと境界条件

  • メモリ不足: detoast 処理や tuplestore_putvalues 実行時にメモリ不足が発生した場合、エラーがスローされる可能性がある。
  • 外部データなし: 属性がすべてインラインの場合、展開処理はスキップされ、そのまま格納される。

保守担当者向けメモ

  • 本関数は tstoreStartupReceiver 内で、TOAST データの展開が必要な場合に receiveSlot として選択される。
  • myState->tofree 配列のサイズは、タプルの属性数に基づいているため、メモリ管理には注意が必要。
