pg_get_keywords

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/misc.c:417

目的

PostgreSQLの予約語および非予約語の一覧を、そのカテゴリやラベル付けの可否情報とともにSQLテーブル形式で返却する集合を返す関数（SRF）です。

シグネチャまたは形状

Datum pg_get_keywords(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数）。
  • **出力**: Datum（行形式のタプルを順次返却）。各行はキーワード名、カテゴリコード、ラベル可否フラグ、カテゴリ説明、ラベル説明の5列で構成されます。

前提条件と不変条件

  • 呼び出し元はSRFとして本関数を呼び出す必要があります。
  • 最初の呼び出し時に FuncCallContext が初期化され、以降の呼び出しで状態が維持されます。
  • 返却される型は複合型（row type）である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時、FuncCallContext を作成し、結果のタプル記述子（TupleDesc）を取得・保存します。 2. **反復処理**: ScanKeywords 配列の要素数分だけループします。 3. **データ構築**:

  • キーワード名を取得します。
  • ScanKeywordCategories を参照し、キーワードの種別（予約語、非予約語など）を判定してカテゴリコードと説明を割り当てます。
  • ScanKeywordBareLabel を参照し、ラベルとして使用可能か（true/false）を判定します。

    • 4. **返却**: BuildTupleFromCStrings でタプルを構築し、SRF_RETURN_NEXT で返却します。全要素の処理完了後、SRF_RETURN_DONE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SQLクエリからの直接呼び出し）。
  • **呼び出し先**:
  • GetScanKeyword（キーワード取得）
  • BuildTupleFromCStrings（タプル構築）
  • SRF マクロ群（集合返却制御）
  • ※具体的な呼び出し元コードの詳細は提供されていません。

関連データ構造

  • FuncCallContext: SRFの呼び出し間で状態を保持するための構造体。
  • ScanKeywords: キーワード定義を保持するグローバル配列。
  • ScanKeywordCategories: 各キーワードのカテゴリを保持する配列。
  • ScanKeywordBareLabel: 各キーワードのラベル可否を保持する配列。

失敗モードと境界条件

  • 戻り値の型が複合型でない場合、elog(ERROR) によりエラーが発生します。
  • 内部的な配列インデックスは ScanKeywords.num_keywords によって安全に管理されています。

保守担当者向けメモ

  • unconstify を使用してキーワード文字列の定数性を外していますが、これは GetScanKeyword の仕様に基づくものです。
  • 新しいキーワードカテゴリを追加する場合は、switch 文のケースを拡張する必要があります。
