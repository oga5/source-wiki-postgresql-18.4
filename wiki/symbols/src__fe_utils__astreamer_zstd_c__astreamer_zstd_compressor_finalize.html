astreamer_zstd_compressor_finalize

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_zstd.c:190

目的

Zstandard圧縮ストリームの終端処理を行い、残存する圧縮データをフラッシュして後続のストリーマーへ転送する。

シグネチャまたは形状

static void astreamer_zstd_compressor_finalize(astreamer *streamer)

入力と出力

  • **入力**: streamer (圧縮処理を行う astreamer_zstd_frame 構造体へのポインタ)
  • **出力**: なし（副作用として後続ストリーマーへデータを転送し、終了処理を伝播させる）

前提条件と不変条件

  • streamer は astreamer_zstd_frame 型としてキャスト可能であること。
  • mystreamer->cctx（zstd圧縮コンテキスト）が初期化済みであること。
  • mystreamer->base.bbs_next が有効な後続ストリーマーを指していること。

アルゴリズムまたは振る舞い

1. ZSTD_e_end モードで ZSTD_compressStream2 を繰り返し呼び出し、圧縮ストリームの終端処理を行う。 2. 出力バッファ（zstd_outBuf）の空き容量が不足する場合、現在のバッファ内容を astreamer_content を介して後続ストリーマーへ転送し、バッファをリセットする。 3. 圧縮処理でエラーが発生した場合は pg_fatal を呼び出し、エラーメッセージを出力してプロセスを終了する。 4. 全ての圧縮データがフラッシュされた後、バッファに残存するデータがあれば後続ストリーマーへ転送する。 5. 最後に astreamer_finalize を呼び出し、後続ストリーマーの終了処理を起動する。

呼び出し元と呼び出し先

  • **呼び出し元**: 構造体 astreamer_ops の .finalize フィールド経由で呼び出される（直接の呼び出し元コードはソース抜粋内では特定不可）。
  • **呼び出し先**:
  • astreamer_content (後続ストリーマーへのデータ転送)
  • astreamer_finalize (後続ストリーマーの終了処理)
  • ZSTD_compressStream2 (libzstdによる圧縮フラッシュ)
  • pg_fatal (エラー発生時)

関連データ構造

  • astreamer_zstd_frame: 圧縮コンテキストとバッファを保持するメイン構造体。
  • ZSTD_inBuffer / ZSTD_outBuffer: libzstd の入出力バッファ構造体。

失敗モードと境界条件

  • ZSTD_compressStream2 がエラーを返した場合、pg_fatal により異常終了する。
  • 出力バッファの容量が不足するたびに転送とリセットが行われるため、バッファサイズが極端に小さい場合でも処理は継続される。

保守担当者向けメモ

  • 本関数は USE_ZSTD が定義されている環境でのみコンパイルされる。
  • ZSTD_compressBound(0) を使用してフラッシュに必要なバッファサイズを計算している点に注意すること。
