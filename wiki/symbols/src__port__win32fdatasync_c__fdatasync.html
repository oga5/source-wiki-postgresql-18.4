fdatasync

定義

  • 種別: function
  • 定義箇所: src/port/win32fdatasync.c:22

目的

Windows環境において、POSIX標準の fdatasync() をエミュレートし、ファイル記述子に関連付けられたデータを物理ストレージへフラッシュする。メタデータの更新を最小限に抑えることを意図している。

シグネチャまたは形状

int fdatasync(int fd);

入力と出力

  • **入力**: fd (int) - 同期対象のファイル記述子。
  • **出力**: 成功時は 0 を返す。失敗時は -1 を返し、errno を適切に設定する。

前提条件と不変条件

  • initialize_ntdll() が成功し、必要なネイティブAPI関数ポインタがロードされている必要がある。
  • fd は有効なファイル記述子であり、_get_osfhandle() によって有効なWindowsハンドルへ変換可能であること。

アルゴリズムまたは振る舞い

1. _get_osfhandle(fd) を呼び出し、Cランタイムのファイル記述子からWindowsハンドルを取得する。無効な場合は errno を EBADF に設定して終了する。 2. initialize_ntdll() を呼び出し、ネイティブAPIの準備を行う。 3. pg_NtFlushBuffersFileEx を呼び出し、FLUSH_FLAGS_FILE_DATA_SYNC_ONLY フラグを指定してデータのみの同期を要求する。 4. NT_SUCCESS マクロで結果を判定し、成功時は 0 を返す。 5. 失敗時は pg_RtlNtStatusToDosError で NTSTATUS をDOSエラーコードに変換し、_dosmaperr を介して errno を設定した上で -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c
  • src/backend/storage/file/fd.c
  • src/bin/pg_test_fsync/pg_test_fsync.c
  • **呼び出し先**:
  • _get_osfhandle (Cランタイム)
  • initialize_ntdll
  • pg_NtFlushBuffersFileEx
  • pg_RtlNtStatusToDosError
  • _dosmaperr

関連データ構造

  • IO_STATUS_BLOCK: pg_NtFlushBuffersFileEx の実行結果を受け取るために使用。
  • HANDLE: Windowsネイティブのファイルハンドル。

失敗モードと境界条件

  • _get_osfhandle が INVALID_HANDLE_VALUE を返した場合、EBADF を設定して失敗する。
  • initialize_ntdll が失敗した場合、関数は -1 を返す。
  • pg_NtFlushBuffersFileEx が失敗した場合、対応するWindowsエラーコードが errno にマッピングされる。

保守担当者向けメモ

  • 本関数は ntdll.dll を直接操作する低レベルな実装である。
  • ファイルシステムの実装によっては、データのみの同期を要求してもメタデータが更新される可能性がある。
  • initialize_ntdll() は初回実行時にのみ重い処理が発生するよう設計されている。
