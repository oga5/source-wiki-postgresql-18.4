BufferUsageAdd

定義

  • 種別: function
  • 定義箇所: src/backend/executor/instrument.c:225

目的

2つの BufferUsage 構造体の値を加算し、累積統計を更新します。

シグネチャまたは形状

static void BufferUsageAdd(BufferUsage *dst, const BufferUsage *add)

入力と出力

  • **入力**:
  • dst: 加算先となる BufferUsage 構造体へのポインタ。
  • add: 加算元となる BufferUsage 構造体へのポインタ（変更不可）。
  • **出力**: なし（dst が指す構造体が直接更新されます）。

前提条件と不変条件

  • dst および add は有効な BufferUsage 構造体を指している必要があります。
  • INSTR_TIME_ADD マクロが定義されている環境である必要があります。

アルゴリズムまたは振る舞い

1. add の各メンバ（shared_blks_hit, shared_blks_read, shared_blks_dirtied, shared_blks_written, local_blks_hit, local_blks_read, local_blks_dirtied, local_blks_written, temp_blks_read, temp_blks_written）を dst の対応するメンバに加算します。 2. INSTR_TIME_ADD マクロを使用して、以下の時間計測値を dst に加算します。

  • shared_blk_read_time, shared_blk_write_time
  • local_blk_read_time, local_blk_write_time
  • temp_blk_read_time, temp_blk_write_time

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/instrument.c 内の統計集約処理（InstrAccumParallelQuery 等）から呼び出されています。
  • **呼び出し先**:
  • INSTR_TIME_ADD マクロ（時間加算用）。

関連データ構造

  • BufferUsage: バッファアクセス統計および WAL 生成量を保持する構造体。

失敗モードと境界条件

  • ポインタが NULL の場合、セグメンテーション違反が発生します。
  • カウンタが各型の最大値を超えた場合、オーバーフローが発生します。

保守担当者向けメモ

  • 本関数は静的関数であり、instrument.c 内でのみ使用されます。
  • BufferUsage 構造体のメンバが追加・変更された場合は、本関数にも反映が必要です。
