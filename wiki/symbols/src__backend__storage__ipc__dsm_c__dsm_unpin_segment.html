dsm_unpin_segment

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm.c:987

目的

指定された動的共有メモリ（DSM）セグメントの「ピン留め（pin）」を解除し、参照カウントをデクリメントする。参照カウントがゼロに達した場合は、セグメントの破棄とリソース解放を行う。

シグネチャまたは形状

void dsm_unpin_segment(dsm_handle handle)

入力と出力

  • **入力**: handle (dsm_handle) - 解除対象のDSMセグメントハンドル。
  • **出力**: なし。

前提条件と不変条件

  • 対象セグメントは既にピン留めされている必要がある。
  • DynamicSharedMemoryControlLock を排他モードで取得して制御領域を操作する。
  • 呼び出し時点で、呼び出し元プロセスは当該セグメントをマッピングしていないことが前提となる（マッピングが存在する場合、参照カウントが1より大きいため破棄されない）。

アルゴリズムまたは振る舞い

1. DynamicSharedMemoryControlLock を取得し、制御領域から指定された handle に一致するスロットを検索する。 2. 対象スロットが見つからない、または pinned フラグが立っていない場合はエラーを投げる。 3. dsm_impl_unpin_segment を呼び出し、実装固有のピン解除処理を行う（メイン領域のDSMを除く）。 4. 参照カウント（refcnt）をデクリメントし、pinned フラグを false に設定する。 5. 参照カウントが1（未使用状態）になった場合、セグメントの破棄フラグを立てる。 6. ロックを解放後、破棄フラグが立っていれば dsm_impl_op を用いてセグメントを破棄し、メイン領域の場合は FreePageManager を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/dsm.c (内部的なセグメント管理処理)
  • src/backend/utils/mmgr/dsa.c (動的共有領域の管理)
  • **呼び出し先**:
  • dsm_impl_unpin_segment
  • dsm_impl_op (破棄時)
  • FreePageManagerPut (メイン領域の破棄時)
  • LWLockAcquire / LWLockRelease

関連データ構造

  • dsm_control: DSM制御領域全体を管理する構造体。
  • dsm_control_item: 各セグメントの状態（参照カウント、ピン状態、ハンドル等）を保持する構造体。

失敗モードと境界条件

  • **エラー**:
  • 指定したハンドルが不明な場合（elog(ERROR)）。
  • ピン留めされていないセグメントに対して呼び出された場合（elog(ERROR)）。
  • **境界条件**:
  • 参照カウントが1の状態でピン解除を行うと、セグメントの破棄処理が実行される。

保守担当者向けメモ

  • 本関数は、トランザクション終了時やリソース所有者のクリーンアップと密接に関連している。
  • dsm_impl_unpin_segment はロック保持中に実行されるため、実装側で重い処理や新たなロック取得を行わないよう注意が必要である。
  • 破棄処理（destroy）はロック解放後に実行される設計であり、これはデッドロックを回避しつつ、エラー発生時の安全性を確保するためである。
