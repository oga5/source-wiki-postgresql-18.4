px

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_px.c:48

目的

遺伝的アルゴリズム（GEQO）において、2つの親個体（tour1, tour2）から「Position Crossover (PX)」を用いて子個体（offspring）を生成する。親個体の遺伝子位置を優先的に継承し、残りの空きを補完することで解の多様性を確保する。

シグネチャまたは形状

void px(PlannerInfo *root, Gene *tour1, Gene *tour2, Gene *offspring, int num_gene, City *city_table)

入力と出力

  • **入力**:
  • root: プランナ情報。
  • tour1, tour2: 交叉対象となる親個体の遺伝子配列。
  • num_gene: 遺伝子の総数。
  • city_table: 遺伝子の使用状況を追跡するための作業用配列。
  • **出力**:
  • offspring: 生成された子個体の遺伝子配列（関数内で直接書き込まれる）。

前提条件と不変条件

  • city_table は、関数呼び出し時に num_gene 分の領域が確保されている必要がある。
  • num_gene は正の整数であること。

アルゴリズムまたは振る舞い

1. **初期化**: city_table をリセットし、すべての都市を未使用状態にする。 2. **親1からの継承**: geqo_randint を使用し、全遺伝子の 1/3 から 2/3 の範囲でランダムに決定された個数の位置を tour1 から offspring へ直接コピーし、使用済みとしてマークする。 3. **親2による補完**: offspring の空き位置に対し、tour2 の順序を走査する。まだ使用されていない都市であれば offspring に配置し、使用済みであればスキップする。 4. **終了**: offspring の全位置が埋まった時点で処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内には直接の呼び出し元に関する記述はありません。
  • **呼び出し先**: geqo_randint（乱数生成関数）を呼び出します。
  • **注意**: ツールが検出した contrib/pgcrypto 関連の px.h 等への参照は、本関数 px とは異なるシンボル（暗号化関連）を指している可能性が高く、本関数の呼び出し元とは見なせません。

関連データ構造

  • Gene: 遺伝子を表す型。
  • City: 遺伝子の使用状況（used フラグ）を保持する構造体。

失敗モードと境界条件

  • num_gene が 0 または負の場合、ループ処理が正しく動作しない可能性がある。
  • city_table のサイズが num_gene 未満の場合、メモリ破壊が発生する。

保守担当者向けメモ

  • 本関数は #if defined(PX) で囲まれており、コンパイル設定に依存します。
  • Darrell L. Whitley氏による「Genitor」アルゴリズムの実装をベースとしています。
  • 順序の保持よりも「位置」の保持を重視するアルゴリズムです。
