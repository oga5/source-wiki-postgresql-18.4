sync_target_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:316

目的

ターゲットデータディレクトリ内の変更をディスクへフラッシュし、ファイルシステムの整合性を保証します。

シグネチャまたは形状

void sync_target_dir(void);

入力と出力

  • 入力: なし（グローバル変数 do_sync, dry_run, datadir_target, PG_VERSION_NUM, sync_method を参照）
  • 出力: なし（副作用としてディスクへの書き込みを行う）

前提条件と不変条件

  • do_sync が真であり、かつ dry_run が偽である場合にのみ実行されます。
  • ターゲットディレクトリのパスおよび同期メソッドが適切に設定されている必要があります。

アルゴリズムまたは振る舞い

1. do_sync が偽、または dry_run が真の場合は、何もせず直ちに終了します。 2. 上記条件を満たさない場合、sync_pgdata を呼び出し、ターゲットディレクトリ（datadir_target）内の全ファイルを指定された sync_method でディスクへフラッシュします。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_rewind/pg_rewind.c:527
  • 呼び出し先: sync_pgdata (PostgreSQL 共通ユーティリティ)

関連データ構造

  • datadir_target (ターゲットディレクトリパス)
  • sync_method (同期方式)

失敗モードと境界条件

  • do_sync が無効な場合、同期処理はスキップされます。
  • dry_run モード時は、ファイルシステムへの物理的な書き込みは行われません。

保守担当者向けメモ

  • 本関数は pg_rewind の終了処理の最終段階で呼び出されることを想定しています。
  • sync_pgdata は common/file_utils.h で定義されている外部関数であり、OS レベルの同期を担います。
