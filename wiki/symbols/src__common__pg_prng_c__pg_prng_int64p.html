pg_prng_int64p

定義

  • 種別: function
  • 定義箇所: src/common/pg_prng.c:181

目的

擬似乱数生成器の状態から、正の符号付き64ビット整数（int64）を生成する。

シグネチャまたは形状

int64 pg_prng_int64p(pg_prng_state *state)

入力と出力

  • **入力**: pg_prng_state *state（乱数生成器の状態を保持する構造体へのポインタ）
  • **出力**: int64 型の正の乱数値（0 以上 INT64_MAX 以下）

前提条件と不変条件

  • state は初期化済みであること。
  • 内部で使用される xoroshiro128ss の特性上、状態がすべて0である場合は常に0を返す。

アルゴリズムまたは振る舞い

1. 内部関数 xoroshiro128ss(state) を呼び出し、64ビットの乱数値を生成する。 2. 生成された値に対してビットマスク 0x7FFFFFFFFFFFFFFF を適用し、最上位ビット（符号ビット）を強制的に0にする。 3. 結果を int64 型にキャストして返す。これにより、常に正の範囲の整数が保証される。

呼び出し元と呼び出し先

  • **呼び出し先**: xoroshiro128ss
  • **呼び出し元**: 提供されたソース excerpt 内には具体的な呼び出し元の記述なし。

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • **境界条件**: 生成される値は 0 から 9223372036854775807 (INT64_MAX) の範囲に収まる。
  • **失敗モード**: state が NULL の場合、未定義の動作を引き起こす可能性がある（ソースコード上に明示的なガードなし）。

保守担当者向けメモ

  • 本関数は暗号学的に安全ではない。パスワードや暗号鍵の生成には使用しないこと。
  • 内部で xoroshiro128** アルゴリズムを使用しており、高速な乱数生成を目的としている。
