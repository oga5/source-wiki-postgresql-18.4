ArrayMetaState

定義

  • 種別: struct
  • 定義箇所: src/include/utils/array.h:236

目的

配列操作関数において、配列要素の型情報や入出力関数、および実行時のメタデータをキャッシュし、効率的なアクセスを提供するための構造体です。

シグネチャまたは形状

typedef struct ArrayMetaState
{
	Oid			element_type;
	int16		typlen;
	bool		typbyval;
	char		typalign;
	char		typdelim;
	Oid			typioparam;
	Oid			typiofunc;
	FmgrInfo	proc;
} ArrayMetaState;

入力と出力

  • 本構造体自体は入力として関数（DatumGetExpandedArrayX 等）に渡され、配列のメタデータを提供します。
  • FmgrInfo proc フィールドは、配列要素の操作に必要な関数呼び出し情報を保持します。

前提条件と不変条件

  • 配列要素の型に関する情報（長さ、アライメント、値渡し可否など）が正しく設定されている必要があります。
  • proc フィールドは、使用前に適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • 配列の反復処理や展開済み配列の操作において、型情報を毎回システムカタログから検索するオーバーヘッドを避けるために使用されます。
  • fcinfo->flinfo->fn_extra を通じて関数呼び出し間でキャッシュとして保持されることがあります。

呼び出し元と呼び出し先

  • **呼び出し元（確認されたもの）**:
  • src/backend/utils/adt/array_expanded.c: DatumGetExpandedArrayX 等で利用。
  • src/backend/utils/adt/array_userfuncs.c: 関数呼び出しの fn_extra として利用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すわけではありませんが、保持する FmgrInfo proc を通じて要素操作関数が呼び出されます。

関連データ構造

  • ArrayType: 標準的なフラット配列のヘッダー。
  • ExpandedArrayHeader: 展開済み配列のヘッダー。
  • FmgrInfo: 関数マネージャ情報。

失敗モードと境界条件

  • キャッシュされたメタデータと実際の配列要素の型が一致しない場合、不正なメモリアクセスや型変換エラーが発生する可能性があります。
  • fn_extra が初期化されていない状態でアクセスすると、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体は array_userfuncs.c 等で fn_extra として再利用されるため、メモリ管理（palloc 等）のライフサイクルに注意してください。
  • 新しい配列操作関数を追加する際は、型情報の検索コストを削減するために本構造体の利用を検討してください。
