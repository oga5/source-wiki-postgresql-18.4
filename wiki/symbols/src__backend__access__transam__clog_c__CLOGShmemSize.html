CLOGShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:780

目的

PostgreSQL の共有メモリ領域において、CLOG（Commit Log）管理のために必要な合計サイズを計算する。

シグネチャまたは形状

Size CLOGShmemSize(void)

入力と出力

  • **入力**: なし。
  • **出力**: Size 型のバイト数。CLOG のバッファ管理に必要な共有メモリの総量を返す。

前提条件と不変条件

  • CLOGShmemBuffers() が返すバッファ数と、CLOG_LSNS_PER_PAGE（ページあたりの LSN 数）に基づいて計算される。
  • 共有メモリの割り当てが行われる前に呼び出される必要がある。

アルゴリズムまたは振る舞い

SimpleLruShmemSize 関数を呼び出し、CLOG 用の SLRU（Simple LRU）構造体に必要な共有メモリサイズを算出する。引数として、現在の設定に基づくバッファ数とページあたりの LSN 数を渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c:124（共有メモリの初期化プロセスの一部として呼び出される）。
  • **呼び出し先**: SimpleLruShmemSize（src/backend/access/transam/slru.c 内の関数）。

関連データ構造

  • SlruCtlData: SLRU の制御構造体。
  • CLOGShmemBuffers(): 必要なバッファ数を決定する関数。

失敗モードと境界条件

  • 共有メモリの計算に必要なパラメータが正しく設定されていない場合、誤ったサイズが返される可能性がある。
  • 呼び出し元で add_size を用いて他の共有メモリ領域と加算される際、オーバーフローが発生する可能性がある。

保守担当者向けメモ

  • 本関数は src/include/access/clog.h で公開されている。
  • CLOG のバッファ数やページ構成を変更する場合は、本関数が参照する定数や関数が適切か確認すること。
