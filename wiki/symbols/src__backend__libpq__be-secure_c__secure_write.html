secure_write

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure.c:304

目的

クライアントとの通信において、暗号化方式（SSL/TLS、GSSAPI）または平文のいずれか適切な手段を選択し、データを安全に書き込むための抽象化インターフェースを提供します。

シグネチャまたは形状

ssize_t secure_write(Port *port, const void *ptr, size_t len)

入力と出力

  • **入力**:
  • port: 通信状態や暗号化フラグを保持する Port 構造体へのポインタ。
  • ptr: 書き込むデータが格納されたバッファ。
  • len: 書き込むデータのバイト数。
  • **出力**:
  • 成功時は書き込んだバイト数（ssize_t）、失敗時は -1 を返します。

前提条件と不変条件

  • port が有効な接続状態であること。
  • 割り込み処理（ProcessClientWriteInterrupt）が呼び出しの前後で適切に実行されること。
  • 非ブロッキングモード（port->noblock が偽）の場合、ソケットが書き込み可能になるまで待機するループが保証されます。

アルゴリズムまたは振る舞い

1. **割り込みチェック**: ProcessClientWriteInterrupt(false) を呼び出し、保留中の割り込みを処理します。 2. **書き込み実行**: port の設定に基づき、以下のいずれかの関数を呼び出します。

  • SSL有効時: be_tls_write
  • GSSAPI有効時: be_gssapi_write
  • それ以外: secure_raw_write

    • 3. **ブロッキング制御**: 書き込みが EWOULDBLOCK または EAGAIN で失敗し、かつ非ブロッキングモードでない場合、WaitEventSet を使用してソケットが書き込み可能になるまで待機します。

  • 待機中にポストマスターの終了が検知された場合は FATAL エラーを発生させます。
  • 割り込み（WL_LATCH_SET）が発生した場合は、ラッチをリセットし割り込み処理を実行してから再試行します。

    • 4. **終了処理**: 最後に再度 ProcessClientWriteInterrupt(false) を呼び出し、書き込み完了後の割り込みを処理して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/pqcomm.c:1374
  • src/backend/tcop/backend_startup.c:604
  • src/backend/tcop/backend_startup.c:667
  • **呼び出し先**:
  • be_tls_write (SSL有効時)
  • be_gssapi_write (GSSAPI有効時)
  • secure_raw_write (平文時)
  • ProcessClientWriteInterrupt
  • WaitEventSetWait

関連データ構造

  • Port: 通信状態、暗号化フラグ（ssl_in_use, gss）、ブロッキング設定（noblock）を保持。
  • WaitEventSet: I/O待機イベント管理用。

失敗モードと境界条件

  • **書き込みエラー**: n < 0 かつ EWOULDBLOCK/EAGAIN 以外の場合、呼び出し元へエラーが伝播します。
  • **ポストマスター終了**: 待機中にポストマスターが終了した場合、FATAL エラーによりプロセスが終了します。
  • **割り込み**: 待機中に割り込みが発生した場合、再試行ループにより処理が継続されます。

保守担当者向けメモ

  • 本関数は be-secure-openssl.c や be-secure-gssapi.c と密接に連携しています。具体的な暗号化ロジックの変更は各実装ファイルを確認してください。
  • retry ループ内での WaitEventSet の利用は、PostgreSQLの非同期的な割り込み処理を正しく扱うための重要なパターンです。修正時は secure_read との整合性に注意してください。
