logicalrep_read_commit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/proto.c:97

目的

論理レプリケーションの COMMIT メッセージをネットワークバッファからデシリアライズし、トランザクションのコミット情報を構造体に格納する。

シグネチャまたは形状

void logicalrep_read_commit(StringInfo in, LogicalRepCommitData *commit_data)

入力と出力

  • **入力**:
  • in: 受信したメッセージデータを含む StringInfo バッファ。
  • commit_data: デシリアライズされたコミット情報を格納するための LogicalRepCommitData 構造体へのポインタ。
  • **出力**: なし（commit_data を介して値を返却）。

前提条件と不変条件

  • in バッファは、プロトコル定義に従った COMMIT メッセージの形式で構成されている必要がある。
  • StringInfo の読み込み順序は、対応する logicalrep_write_commit の書き込み順序と完全に一致しなければならない。

アルゴリズムまたは振る舞い

1. バッファから 1 バイトのフラグを読み込む。現在フラグは使用されておらず、0 以外の場合はエラーを発生させる。 2. バッファから 8 バイトの整数（int64）を 3 回連続で読み込み、それぞれ commit_lsn、end_lsn、committime として commit_data に代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/worker.c:1014（サブスクライバー側のメッセージ処理ループ）。
  • **呼び出し先**: pq_getmsgbyte および pq_getmsgint64（libpq/pqformat.h で定義されたバッファ読み込み関数）。

関連データ構造

  • LogicalRepCommitData: コミットの LSN およびタイムスタンプを保持する構造体。
  • StringInfo: ネットワーク通信用の可変長バイトバッファ。

失敗モードと境界条件

  • flags が 0 以外の場合、elog(ERROR) により処理が中断される。
  • バッファの残量が期待されるデータサイズ（フラグ 1 バイト + 8 バイト整数 × 3）に満たない場合、pq_getmsg* 関数群によってエラーが発生する。

保守担当者向けメモ

  • 本関数はプロトコルの一部であるため、メッセージ形式を変更する場合は logicalproto.h の定義および logicalrep_write_commit との同期を必ず確認すること。
