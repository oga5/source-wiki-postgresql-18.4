pg_itoa

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numutils.c:1041

目的

符号付き16ビット整数（int16）を文字列形式に変換します。

シグネチャまたは形状

int pg_itoa(int16 i, char *a);

入力と出力

  • **入力**:
  • i: 変換対象の int16 値。
  • a: 変換後の文字列を格納するバッファへのポインタ。
  • **出力**:
  • 戻り値: 書き込まれた文字数（int）。

前提条件と不変条件

  • 引数 a は、変換結果を格納するために十分なサイズのメモリ領域を指している必要があります。
  • 本関数は NUL 終端文字を書き込みません。

アルゴリズムまたは振る舞い

1. 引数 i を int32 型にキャストします。 2. pg_ltoa を呼び出し、その結果をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/int.c
  • src/backend/utils/sort/logtape.c
  • **呼び出し先**:
  • pg_ltoa

関連データ構造

  • なし

失敗モードと境界条件

  • バッファ a が不足している場合、バッファオーバーフローが発生します。
  • int16 の範囲（-32768〜32767）のすべての値に対して変換可能です。

保守担当者向けメモ

  • 本関数は pg_ltoa への単なるラッパーです。
  • 文字列化のパフォーマンスを最適化するため、内部では DIGIT_TABLE を利用した効率的な変換が行われています。
  • 呼び出し側で NUL 終端が必要な場合は、戻り値を使用して手動で追加する必要があります。
