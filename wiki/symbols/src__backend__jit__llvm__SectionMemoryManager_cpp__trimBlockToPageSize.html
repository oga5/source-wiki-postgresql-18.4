trimBlockToPageSize

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/SectionMemoryManager.cpp:306

目的

与えられたメモリブロックを、OSのページ境界に整合するように切り詰め（トリミング）、ページ単位でアライメントされた新しいメモリブロックを返します。

シグネチャまたは形状

static sys::MemoryBlock trimBlockToPageSize(sys::MemoryBlock M)

入力と出力

  • **入力**: sys::MemoryBlock M （トリミング対象のメモリブロック）
  • **出力**: sys::MemoryBlock （ページ境界に調整された新しいメモリブロック）

前提条件と不変条件

  • sys::Process::getPageSizeEstimate() により取得したページサイズが、計算の基点となります。
  • 戻り値のベースアドレスはページサイズで割り切れること（base % PageSize == 0）。
  • 戻り値のサイズはページサイズの倍数であること（size % PageSize == 0）。
  • 戻り値のメモリ範囲は、入力された M の範囲内に収まること。

アルゴリズムまたは振る舞い

1. システムのページサイズを取得します。 2. 入力ブロックの開始アドレスから、次のページ境界までのオフセット（StartOverlap）を計算します。 3. 入力ブロックのサイズから StartOverlap を減算し、さらにページサイズで切り捨てて、ページ境界に収まる最大サイズ（TrimmedSize）を算出します。 4. 計算された新しいベースアドレスとサイズを用いて sys::MemoryBlock を生成し、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/SectionMemoryManager.cpp:339 にて、空きメモリ領域の調整に使用されています。
  • **呼び出し先**: sys::Process::getPageSizeEstimate() を呼び出し、OSのページサイズを取得します。

関連データ構造

  • sys::MemoryBlock: LLVMのメモリ管理用構造体。ベースアドレスとサイズを保持します。

失敗モードと境界条件

  • 入力ブロックのサイズがページサイズより小さい場合、TrimmedSize は 0 になる可能性があります。
  • アサーションにより、戻り値がページ境界に整合していない場合は実行時エラーとなります。

保守担当者向けメモ

  • 本関数はLLVMのバックポート実装の一部です。メモリ保護（mprotect等）を適用する前に、メモリ領域をページ単位に厳密に合わせる必要がある箇所で使用されます。
  • sys::Process::getPageSizeEstimate() は静的変数としてキャッシュされるため、呼び出しごとのオーバーヘッドは最小限です。
