linear_rand

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_selection.c:87

目的

遺伝的クエリ最適化（GEQO）において、適応度順にソートされた個体プールから、指定されたバイアス（選択圧）に基づき、適応度の高い個体が選ばれやすくなるような線形確率分布に従ったインデックスを生成する。

シグネチャまたは形状

static int linear_rand(PlannerInfo *root, int pool_size, double bias)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体へのポインタ（乱数生成器 geqo_rand に渡される）。
  • pool_size: 選択対象となる個体プールのサイズ。
  • bias: 選択の重み付けを制御するバイアス値。
  • **出力**:
  • 0 以上 pool_size 未満の整数（選択された個体のインデックス）。

前提条件と不変条件

  • プールは適応度順にソートされていることを前提とする。
  • geqo_rand は 0.0 以上 1.0 未満の値を返すことを期待する。
  • 戻り値のインデックスは常に 0 <= index < pool_size の範囲内である。

アルゴリズムまたは振る舞い

1. geqo_rand(root) を用いて乱数を取得し、線形確率密度関数を解くための計算を行う。 2. sqrtval = (bias * bias) - 4.0 * (bias - 1.0) * geqo_rand(root) を計算する。 3. sqrtval が正の場合、平方根を求め、その結果を用いて index を算出する。 4. 浮動小数点演算の丸め誤差や geqo_rand の出力値により、期待される範囲外の値が生成される可能性があるため、index が 0 未満または pool_size 以上となる場合は、範囲内に収まるまで再試行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_selection.c 内の geqo_selection 関数から、親個体（momma, daddy）を選択するために2回呼び出される。
  • **呼び出し先**: 乱数生成のために geqo_rand を呼び出す。また、math.h の sqrt 関数を使用する。

関連データ構造

  • PlannerInfo: クエリ最適化の情報を保持する構造体。

失敗モードと境界条件

  • bias が 1.0 に近い場合、計算式が不安定になる可能性がある。
  • geqo_rand が 1.0 を返した場合や、丸め誤差により sqrt の引数が負になる可能性があるが、ループ内のガード処理（sqrtval > 0.0 および index の範囲チェック）により回避される。

保守担当者向けメモ

  • 本関数は geqo_selection の内部関数として設計されている。
  • 選択スキームの数学的根拠は Darrell L. Whitley の「Genitor」アルゴリズムに基づいている。
