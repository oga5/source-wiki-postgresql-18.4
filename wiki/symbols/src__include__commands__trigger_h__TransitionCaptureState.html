TransitionCaptureState

定義

  • 種別: struct
  • 定義箇所: src/include/commands/trigger.h:56

目的

SQLの REFERENCING 句で指定される「遷移テーブル（Transition Table）」をサポートするため、DML操作（INSERT/UPDATE/DELETE/COPY）における変更前後のタプル状態を保持・管理します。

シグネチャまたは形状

typedef struct TransitionCaptureState
{
    bool        tcs_delete_old_table;
    bool        tcs_update_old_table;
    bool        tcs_update_new_table;
    bool        tcs_insert_new_table;
    TupleTableSlot *tcs_original_insert_tuple;
    struct AfterTriggersTableData *tcs_insert_private;
    struct AfterTriggersTableData *tcs_update_private;
    struct AfterTriggersTableData *tcs_delete_private;
} TransitionCaptureState;

入力と出力

  • **入力**: DML実行時に TriggerDesc 等から生成され、トリガー実行エンジンへ渡されます。
  • **出力**: 遷移テーブルへのタプル蓄積状態を保持し、必要に応じて tcs_original_insert_tuple を介して元のタプルスロットを提供します。

前提条件と不変条件

  • 遷移テーブルを必要とするトリガーが存在する場合にのみ有効化されます。
  • tcs_..._table フラグは、対応する操作で遷移テーブルが定義されているかを示します。

アルゴリズムまたは振る舞い

  • **タプル蓄積**: 実行エンジン（nodeModifyTable.c等）がトリガーを呼び出す際、変更されたタプルを tcs_..._private が指す AfterTriggersTableData（内部的なタプルストア）へ蓄積します。
  • **最適化**: INSERT および COPY 操作において、ルーティング済みのタプルを再変換するコストを避けるため、tcs_original_insert_tuple を通じて直接スロットを参照するバイパス処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/trigger.c 内のトリガー実行関数群（ExecBR..., ExecAR... 等）から参照・操作されます。
  • **呼び出し先**: 内部的に AfterTriggersTableData を操作します。
  • ※具体的な関数呼び出しの連鎖については、提供されたソース範囲内では詳細なコールグラフは定義されていません。

関連データ構造

  • AfterTriggersTableData: 遷移テーブルの物理的な格納先（非公開データ）。
  • TupleTableSlot: タプルを保持する実行時のスロット。
  • TriggerDesc: 本構造体を生成するための元となるトリガー記述子。

失敗モードと境界条件

  • 遷移テーブルが定義されていない操作において、フラグが誤って有効化された場合の挙動については本構造体単体では規定されていません。
  • tcs_original_insert_tuple が設定されていない場合、通常のタプル変換処理が優先されます。

保守担当者向けメモ

  • 本構造体は src/backend/commands/trigger.c で生成・管理されます。
  • tcs_..._private フィールドは不透明なポインタとして扱われており、具体的なデータストアの実装は AfterTriggersTableData に依存しています。
  • 遷移テーブルのサポートは REFERENCING 句の処理と密接に関連しているため、変更時は nodeModifyTable.c 側の処理との整合性に注意してください。
