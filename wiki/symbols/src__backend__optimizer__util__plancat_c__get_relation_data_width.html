get_relation_data_width

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/plancat.c:1241

目的

指定されたリレーションの平均データ幅（バイト単位）を計算し、プランナのコスト見積もりに提供する。

シグネチャまたは形状

int32 get_relation_data_width(Oid relid, int32 *attr_widths)

入力と出力

  • **入力**:
  • relid: 対象リレーションの OID。
  • attr_widths: 各属性の幅を格納する配列（または NULL）。
  • **出力**:
  • リレーションの平均行幅を int32 で返す。

前提条件と不変条件

  • 対象のリレーションは、呼び出し元ですでに適切なロックが取得されている必要がある（関数内では NoLock を使用して開くため）。

アルゴリズムまたは振る舞い

1. table_open(relid, NoLock) を呼び出し、リレーション記述子を取得する。 2. get_rel_data_width を呼び出し、実際の幅計算を実行する。 3. table_close(relation, NoLock) を呼び出し、リレーション記述子を解放する。 4. 計算結果を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/costsize.c: wholerow_width の計算に使用。
  • src/backend/optimizer/plan/planner.c: rel->reltarget->width の設定に使用。
  • **呼び出し先**:
  • table_open / table_close (テーブルアクセス層)
  • get_rel_data_width (同一ファイル内の内部関数)

関連データ構造

  • Relation: リレーションの物理的・論理的特性を保持する構造体。

失敗モードと境界条件

  • relid が無効な場合、table_open がエラーを発生させる可能性がある。
  • attr_widths が NULL の場合、get_rel_data_width は属性ごとの幅を個別に記録せず、合計幅のみを計算する。

保守担当者向けメモ

  • 本関数は get_rel_data_width のラッパーであり、OID からリレーション記述子を解決する責務を持つ。
  • ロック管理は呼び出し元に依存しているため、本関数単体でロックの整合性を保証しない点に注意すること。
