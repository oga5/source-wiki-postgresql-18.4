pgaio_worker_submission_queue_insert

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_worker.c:180

目的

共有メモリ上のI/O投入用リングバッファ（PgAioWorkerSubmissionQueue）に、新しいI/OリクエストのIDを挿入する。

シグネチャまたは形状

static bool pgaio_worker_submission_queue_insert(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh （挿入対象のI/Oハンドル）
  • **出力**: bool （挿入に成功した場合は true、キューが満杯で挿入できなかった場合は false）

前提条件と不変条件

  • io_worker_submission_queue が初期化されていること。
  • キューのサイズは2のべき乗であることを前提としたビット演算（& (queue->size - 1)）が行われる。

アルゴリズムまたは振る舞い

1. グローバル変数 io_worker_submission_queue を参照する。 2. 現在の head 位置から次の挿入位置 new_head を計算する。 3. new_head が tail と一致する場合、キューが満杯と判断し false を返す。 4. 満杯でない場合、pgaio_io_get_id(ioh) で取得したIDを queue->sqes[queue->head] に格納する。 5. head を new_head に更新し、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/method_worker.c 内のロジックから呼び出される（直接の証拠として、同ファイル257行目での呼び出しが確認されている）。
  • **呼び出し先**:
  • pgaio_io_get_id(ioh) （I/Oハンドルから識別子を取得する関数）
  • pgaio_debug （キュー満杯時のログ出力）

関連データ構造

  • PgAioWorkerSubmissionQueue: 共有メモリ上のリングバッファ構造体。
  • PgAioHandle: I/Oリクエストを保持するハンドル。

失敗モードと境界条件

  • **キュー満杯**: new_head == queue->tail となった場合、挿入は行われず false が返される。この際、DEBUG3 レベルでログが出力される。

保守担当者向けメモ

  • 本関数はロックを取得しないため、呼び出し側で AioWorkerSubmissionQueueLock による排他制御が行われている必要がある。
  • キューのサイズが2のべき乗でない場合、インデックス計算が正しく動作しないため注意が必要。
