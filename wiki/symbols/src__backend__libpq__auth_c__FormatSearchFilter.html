FormatSearchFilter

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth.c:2412

目的

LDAP認証において、設定された検索フィルタ文字列内のプレースホルダー（ユーザー名）を実際のユーザー名に置換し、検索用文字列を生成する。

シグネチャまたは形状

static char *FormatSearchFilter(const char *pattern, const char *user_name)

入力と出力

  • **入力**:
  • pattern: 検索フィルタのテンプレート文字列。
  • user_name: 置換対象となるユーザー名。
  • **出力**:
  • プレースホルダーが置換された新しい文字列（StringInfoData の data フィールド）。

前提条件と不変条件

  • pattern および user_name は有効なヌル終端文字列であること。
  • LPH_USERNAME 定数で定義されたプレースホルダーが pattern 内に含まれる場合、それを user_name に置換する。

アルゴリズムまたは振る舞い

1. initStringInfo を使用して出力用のバッファを初期化する。 2. pattern を一文字ずつ走査する。 3. 現在の pattern 位置が LPH_USERNAME と一致する場合、user_name を出力バッファに追記し、pattern のポインタを LPH_USERNAME_LEN 分進める。 4. 一致しない場合は、現在の文字をそのまま出力バッファに追記し、pattern を1文字進める。 5. 走査完了後、構築された文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth.c:2552 (CheckLDAPAuth 内で呼び出される)
  • **呼び出し先**:
  • initStringInfo
  • appendStringInfoString
  • appendStringInfoChar
  • strncmp

関連データ構造

  • StringInfoData: 文字列構築のための動的バッファ構造体。

失敗モードと境界条件

  • pattern が空の場合、空の文字列が返される。
  • pattern 内にプレースホルダーが存在しない場合、元の文字列がそのままコピーされる。
  • メモリ不足が発生した場合、StringInfo 関連関数内でエラー（elog 等）が発生する可能性がある。

保守担当者向けメモ

  • 本関数は現在 LPH_USERNAME という単一のプレースホルダーのみをサポートしている。将来的に他のプレースホルダーを追加する場合は、while ループ内の条件分岐を拡張する必要がある。
