hash_combine64

定義

  • 種別: function
  • 定義箇所: src/include/common/hashfn.h:79

目的

既存の64ビットハッシュ値 a に別の64ビット値 b を混合し、新しいハッシュ値を生成します。ハッシュテーブルのキー生成や、複数の値を組み合わせたハッシュ値の算出に使用されます。

シグネチャまたは形状

static inline uint64 hash_combine64(uint64 a, uint64 b)

入力と出力

  • **入力**:
  • a: 既存のハッシュ値（uint64）
  • b: 混合対象の値（uint64）
  • **出力**: 混合後の新しいハッシュ値（uint64）

前提条件と不変条件

  • 入力されるハッシュ値は、ビットが均等に分散していることが期待されます。
  • 本関数はインライン関数として定義されており、呼び出し側のコンテキストで展開されます。

アルゴリズムまたは振る舞い

以下のビット演算を用いてハッシュ値を混合します。定数 0x49a0f4dd15e5a8e3 は64ビットのランダムデータとして使用されます。 1. b に定数を加算し、a のビットシフト（左54ビット、右7ビット）と加算・XOR演算を組み合わせることで、ビットの混合を行います。 2. 演算式: a ^= b + 0x49a0f4dd15e5a8e3 + (a << 54) + (a >> 7); 3. 最終的な a を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/partitioning/partbounds.c (複数箇所: パーティション境界のハッシュ計算)
  • src/backend/utils/resowner/resowner.c (リソース所有者のハッシュ計算)
  • **呼び出し先**:
  • 外部関数への依存はありません。

関連データ構造

  • uint64: PostgreSQLで定義される64ビット符号なし整数型。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。入力値が uint64 の範囲内であれば常に計算可能です。

保守担当者向けメモ

  • 本関数はインライン化されているため、頻繁に呼び出されるループ内での使用に適しています。
  • 変更を加える際は、ハッシュの衝突耐性に影響を与えるため、定数やビットシフト量の変更には注意が必要です。
