AtCCI_RelationMap

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relmapper.c:503

目的

トランザクション内の CommandCounterIncrement (CCI) 発生時に、保留中（pending）のマッピング更新をアクティブな状態へ移行させる。

シグネチャまたは形状

void AtCCI_RelationMap(void)

入力と出力

  • 入力: なし（グローバルな pending_shared_updates および pending_local_updates を参照）
  • 出力: なし（副作用としてグローバルな active_ 構造体を更新し、pending_ 構造体をリセットする）

前提条件と不変条件

  • pending_shared_updates および pending_local_updates に格納された更新内容が、現在のトランザクションの整合性を維持していること。
  • 本関数は CommandCounterIncrement のタイミングで呼び出されることを前提とする。

アルゴリズムまたは振る舞い

1. pending_shared_updates に保留中の更新がある場合、merge_map_updates を呼び出して active_shared_updates に統合し、pending_shared_updates.num_mappings を 0 にリセットする。 2. 同様に、pending_local_updates に保留中の更新がある場合、merge_map_updates を呼び出して active_local_updates に統合し、pending_local_updates.num_mappings を 0 にリセットする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c:1586
  • 呼び出し先: merge_map_updates（内部関数）

関連データ構造

  • pending_shared_updates / pending_local_updates: トランザクション内で予約された未適用のマッピング変更。
  • active_shared_updates / active_local_updates: 現在のトランザクションで有効化されたマッピング変更。

失敗モードと境界条件

  • 保留中の更新がない場合、関数は何も行わず即座に復帰する。
  • 統合処理（merge_map_updates）の内部的な失敗モードについては、本関数のソース範囲外のため不明。

保守担当者向けメモ

  • 本関数はトランザクションの進行に伴うマッピングの可視性を制御する重要なポイントである。
  • merge_map_updates の第3引数に true を渡しており、更新の統合時に既存の値を上書きまたは追加する挙動をとる。
