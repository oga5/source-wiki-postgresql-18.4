entryLocateLeafEntry

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:345

目的

GINエントリーツリーのリーフページ内において、指定されたキーに対応する位置（オフセット）を二分探索により特定する。

シグネチャまたは形状

static bool entryLocateLeafEntry(GinBtree btree, GinBtreeStack *stack)

入力と出力

  • **入力**:
  • btree: 検索対象のGIN B-tree構造体。
  • stack: 検索対象のページバッファと、結果のオフセットを格納するスタック構造体。
  • **出力**:
  • 戻り値: キーがページ内に存在する場合は true、存在しない場合は false。
  • stack->off: 検索結果のオフセット値。キーが見つかった場合はその位置、見つからなかった場合は挿入すべき位置が設定される。

前提条件と不変条件

  • stack->buffer が指すページは、GINエントリーツリーのリーフページであること（GinPageIsLeaf および !GinPageIsData のアサーションにより検証）。

アルゴリズムまたは振る舞い

1. btree->fullScan が真の場合、FirstOffsetNumber を返して終了する。 2. ページ内の全タプルに対して、ginCompareAttEntries を用いて二分探索を行う。 3. キーが一致した場合、そのオフセットを stack->off に格納し true を返す。 4. キーが一致しない場合、探索終了時の high 位置（挿入に適した位置）を stack->off に格納し false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginentrypage.c 内の btree->findItem 関数ポインタとして設定されている。
  • **呼び出し先**: ginCompareAttEntries（キー比較）、PageGetMaxOffsetNumber、PageGetItem 等のページ操作マクロ。

関連データ構造

  • GinBtree: GINインデックスのB-tree操作状態を保持する構造体。
  • GinBtreeStack: 探索中のページとオフセットを保持するスタック構造体。
  • IndexTuple: ページ内に格納されているインデックスタプル。

失敗モードと境界条件

  • ページが空の場合: high < low となり、FirstOffsetNumber を返して false となる。
  • キーがページ内のどのタプルよりも小さい、または大きい場合: 二分探索の結果、適切な挿入位置が stack->off に設定される。

保守担当者向けメモ

  • 本関数はエントリーツリー専用であり、データページ（ポスティングリスト）の探索には使用しないこと。
  • 削除操作を考慮しない設計のため、ページ内のキーは常に昇順で維持されている前提で動作する。
