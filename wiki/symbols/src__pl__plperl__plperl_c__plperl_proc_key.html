plperl_proc_key

定義

  • 種別: struct
  • 定義箇所: src/pl/plperl/plperl.c:151

目的

PL/Perl 関数をキャッシュするためのハッシュテーブル（plperl_proc_hash）において、関数を一意に識別するためのキーとして使用されます。

シグネチャまたは形状

typedef struct plperl_proc_key
{
    Oid         proc_id;    /* 関数 OID */
    Oid         is_trigger; /* トリガー関数かどうかのフラグ */
    Oid         user_id;    /* 呼び出しユーザーの OID、または 0 */
} plperl_proc_key;

入力と出力

本構造体はデータ構造のキーとして機能します。入力として proc_id、is_trigger、user_id を保持し、ハッシュテーブル検索時の照合に使用されます。

前提条件と不変条件

  • is_trigger は論理値として機能しますが、構造体内のパディングを排除してメモリレイアウトを最適化するため、bool ではなく Oid 型として宣言されています。
  • ハッシュテーブルのキーとして使用される際、構造体全体が比較対象となるため、メモリ上の配置が固定されている必要があります。

アルゴリズムまたは振る舞い

  • plperl_proc_hash の初期化時（hash_ctl.keysize）に、この構造体のサイズがキーのサイズとして指定されます。
  • 関数実行時、現在の実行コンテキストから生成された plperl_proc_key を用いてハッシュテーブルを検索し、キャッシュされた関数記述子（plperl_proc_desc）の有無を確認します。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plperl/plperl.c 内の関数キャッシュ管理ロジック（compile_plperl_function 等）から、ハッシュ検索のキーとしてインスタンス化されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • plperl_proc_desc: 本キーに対応する値としてハッシュテーブルに格納される、関数のメタデータ構造体。
  • plperl_proc_ptr: キャッシュエントリを指すポインタ型。

失敗モードと境界条件

  • 構造体内の各フィールドが正しく初期化されていない場合、ハッシュ検索が失敗し、関数が再コンパイルされる可能性があります。
  • is_trigger に Oid を使用しているため、論理値として扱う際の比較ロジックが適切である必要があります。

保守担当者向けメモ

  • 構造体のサイズ変更やフィールドの追加を行う際は、plperl_proc_hash のキーサイズ定義と整合性が取れていることを確認してください。
  • is_trigger フィールドは bool ではなく Oid であるため、コード内で比較する際は注意が必要です。
