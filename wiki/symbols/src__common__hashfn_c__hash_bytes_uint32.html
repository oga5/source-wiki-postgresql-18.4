hash_bytes_uint32

定義

  • 種別: function
  • 定義箇所: src/common/hashfn.c:609

目的

32ビットの整数値（uint32）を、ハッシュテーブル等で使用するための32ビットのハッシュ値に変換する。

シグネチャまたは形状

uint32 hash_bytes_uint32(uint32 k)

入力と出力

  • **入力**: k (ハッシュ化対象の uint32 値)
  • **出力**: uint32 型のハッシュ値

前提条件と不変条件

  • 本関数は elog(ERROR) を投げない設計であり、クリティカルなパスでも安全に使用可能である。
  • 内部で使用される final マクロにより、入力値に対してビット攪拌が行われる。

アルゴリズムまたは振る舞い

1. 内部状態 a, b, c を定数 0x9e3779b9、データサイズ（sizeof(uint32)）、および固定値 3923095 を用いて初期化する。 2. 入力値 k を a に加算する。 3. final(a, b, c) マクロを適用し、攪拌された状態から最終的なハッシュ値を算出する。 4. 最終的なハッシュ値として c を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/utils/adt/json.c
  • src/common/hashfn.c (内部的なラッパーとして)
  • src/include/common/hashfn.h (インライン関数経由)
  • **呼び出し先**:
  • final マクロ（src/common/hashfn.c 内で定義）

関連データ構造

  • uint32: PostgreSQL の標準的な32ビット符号なし整数型。

失敗モードと境界条件

  • 本関数は入力値に対して常に決定論的なハッシュ値を返す。
  • 境界条件（オーバーフロー等）は uint32 の算術演算の仕様に従う。

保守担当者向けメモ

  • 本関数は hash_bytes の単一値版としての利便性を提供する。
  • mix や final マクロを変更する場合は、ハッシュの分布特性（アバランチ効果）に影響を与えるため注意が必要である。
