test_rls_hooks_restrictive

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_rls_hooks/test_rls_hooks.c:111

目的

PostgreSQLの行レベルセキュリティ（RLS）フックを介して、特定のテーブル（rls_test_restrictive または rls_test_both）に対して「制約的（Restrictive）」なポリシーを動的に生成・注入する。

シグネチャまたは形状

List *test_rls_hooks_restrictive(CmdType cmdtype, Relation relation)

入力と出力

  • **入力**:
  • cmdtype: 実行されるコマンドの型（CmdType）。
  • relation: ポリシーを適用する対象のリレーション（Relation）。
  • **出力**:
  • List *: 生成された RowSecurityPolicy 構造体を含むリスト。対象外のテーブルの場合は NIL を返す。

前提条件と不変条件

  • 対象テーブル名が rls_test_restrictive または rls_test_both である必要がある。
  • palloc0 によりメモリが確保されるため、PostgreSQLのメモリコンテキスト内で実行される必要がある。
  • transformWhereClause を使用するため、有効な ParseState が構築されている必要がある。

アルゴリズムまたは振る舞い

1. **対象判定**: RelationGetRelationName でテーブル名を確認し、一致しない場合は即座に NIL を返す。 2. **解析状態構築**: make_parsestate で一時的な解析状態を作成し、対象リレーションを名前空間に追加する。 3. **ポリシー定義**: RowSecurityPolicy 構造体を初期化し、ポリシー名、適用対象ロール（ACL_ID_PUBLIC）、コマンド種別（*）を設定する。 4. **式構築**: current_user = supervisor という条件式を FuncCall および ColumnRef を用いて構築する。 5. **変換と照合**: transformWhereClause を呼び出して式を解析し、照合順序（Collation）を割り当てる。 6. **戻り値**: 生成したポリシーをリストに格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_rls_hooks/test_rls_hooks.c 内の初期化処理にて、row_security_policy_hook_restrictive グローバル変数に本関数が代入されている。
  • **呼び出し先**:
  • make_parsestate, addRangeTableEntryForRelation, addNSItemToQuery (パーサー関連)
  • makeFuncCall, makeSimpleA_Expr (ノード構築)
  • transformWhereClause, assign_expr_collations (式変換)
  • palloc0, pstrdup, list_make1 (メモリ・リスト操作)

関連データ構造

  • RowSecurityPolicy: RLSポリシーの定義を保持する構造体。
  • ParseState: クエリ解析中の状態を保持する構造体。
  • Relation: データベースリレーションのメタデータ。

失敗モードと境界条件

  • 対象外のテーブルが渡された場合、NIL を返しポリシーを適用しない。
  • transformWhereClause が失敗した場合、PostgreSQLの標準的なエラーハンドリング（elog 等）により処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、実運用環境での使用は想定されていない。
  • restrictive フックとして動作するため、この関数が返すポリシーは他の許可的ポリシーと論理積（AND）で結合される。許可的ポリシーが存在しない場合、データへのアクセスが拒否される点に注意すること。
