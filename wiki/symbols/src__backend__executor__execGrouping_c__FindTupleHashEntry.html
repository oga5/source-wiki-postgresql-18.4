FindTupleHashEntry

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execGrouping.c:387

目的

指定されたタプルスロットをキーとして、既存のハッシュテーブルから対応するエントリを検索します。

シグネチャまたは形状

TupleHashEntry FindTupleHashEntry(TupleHashTable hashtable, TupleTableSlot *slot,
                                  ExprState *eqcomp,
                                  ExprState *hashexpr)

入力と出力

  • **入力**:
  • hashtable: 検索対象の TupleHashTable 構造体。
  • slot: 検索キーとなるタプルを含む TupleTableSlot。
  • eqcomp: 等価比較用の ExprState。
  • hashexpr: ハッシュ計算用の ExprState。
  • **出力**:
  • 見つかった場合は対応する TupleHashEntry、見つからない場合は NULL を返します。

前提条件と不変条件

  • hashtable は初期化済みである必要があります。
  • ハッシュ計算および比較処理は、メモリリークを防ぐため hashtable->tempcxt 上で実行されます。

アルゴリズムまたは振る舞い

1. hashtable->tempcxt にメモリコンテキストを切り替えます。 2. ハッシュ計算および比較関数が使用する inputslot、in_hash_expr、cur_eq_func を hashtable に設定します。 3. tuplehash_lookup を呼び出し、ハッシュテーブルを検索します。この際、key に NULL を渡すことで、inputslot を参照するよう指示します。 4. 元のメモリコンテキストに復帰し、検索結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execGrouping.c:160 (言及あり)
  • src/backend/executor/nodeSubplan.c:153
  • **呼び出し先**:
  • tuplehash_lookup (マクロ経由)
  • MemoryContextSwitchTo

関連データ構造

  • TupleHashTable: グループ化のためのハッシュテーブルを保持する主要構造体。
  • TupleHashEntry: ハッシュテーブル内の各エントリ。
  • TupleTableSlot: タプルを保持する実行エンジン共通のコンテナ。
  • ExprState: 式評価の状態を保持する構造体。

失敗モードと境界条件

  • hashtable が適切に初期化されていない場合、メモリコンテキストの切り替えやハッシュテーブル操作で不正なアクセスが発生する可能性があります。
  • 検索対象のタプルが NULL の場合の挙動は、tuplehash_lookup の実装に依存します。

保守担当者向けメモ

  • 本関数は検索のみを行い、エントリの作成は行いません。エントリの作成が必要な場合は LookupTupleHashEntry を検討してください。
  • tempcxt の切り替えを行っているため、呼び出し側で tempcxt のリセットが適切に行われているか確認してください。
