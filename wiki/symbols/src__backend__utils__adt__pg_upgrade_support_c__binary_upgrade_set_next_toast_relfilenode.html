binary_upgrade_set_next_toast_relfilenode

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/pg_upgrade_support.c:151

目的

pg_upgrade 実行時において、次に作成される TOAST テーブルの物理ファイル番号（RelFileNode）を強制的に指定するために使用されます。

シグネチャまたは形状

Datum binary_upgrade_set_next_toast_relfilenode(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して、設定対象となる RelFileNumber（OID 型として渡される）を受け取ります。
  • **出力**: なし（PG_RETURN_VOID()）。

前提条件と不変条件

  • CHECK_IS_BINARY_UPGRADE マクロにより、サーバーが binary_upgrade モードで起動されていることが必須条件です。このモードでない場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. 引数から RelFileNumber を取得します。 2. CHECK_IS_BINARY_UPGRADE を実行し、アップグレードモードであることを検証します。 3. グローバル変数 binary_upgrade_next_toast_pg_class_relfilenumber に、受け取った relfilenumber を代入します。 4. 処理を完了し、void を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c:5754 において、SQL コマンドとして呼び出されています。
  • **呼び出し先**:
  • 提供されたソースコード内では、特定の関数呼び出しは行われていません。

関連データ構造

  • binary_upgrade_next_toast_pg_class_relfilenumber: この関数が値を設定するグローバル変数（src/include/catalog/binary_upgrade.h で宣言）。

失敗モードと境界条件

  • binary_upgrade モード以外で呼び出された場合、CHECK_IS_BINARY_UPGRADE マクロによりエラーがスローされます。

保守担当者向けメモ

  • 本関数は pg_upgrade ツールによるカタログ操作を支援するための特殊な管理用インターフェースです。
  • 誤用はシステムカタログの整合性を破壊する可能性があるため、通常の運用環境では使用しないでください。
