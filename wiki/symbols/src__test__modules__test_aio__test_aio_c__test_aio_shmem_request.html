test_aio_shmem_request

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:58

目的

PostgreSQL の共有メモリ領域を確保し、非同期 I/O テスト用モジュールが必要とするエラー状態管理構造体（InjIoErrorState）のための領域を割り当てる。

シグネチャまたは形状

static void test_aio_shmem_request(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリの要求を行う）

前提条件と不変条件

  • prev_shmem_request_hook が定義されている場合、本関数内で呼び出される必要がある。
  • 本関数は PostgreSQL の共有メモリ初期化プロセスの一部として実行されることを前提とする。

アルゴリズムまたは振る舞い

1. 既存の共有メモリ要求フック（prev_shmem_request_hook）が存在する場合、それを実行する。 2. RequestAddinShmemSpace を呼び出し、InjIoErrorState 構造体のサイズ分だけ共有メモリ領域を要求する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/test_aio/test_aio.c 内の初期化処理（shmem_request_hook への代入を通じて実行される）。
  • 呼び出し先: prev_shmem_request_hook（存在する場合）、RequestAddinShmemSpace。

関連データ構造

  • InjIoErrorState: 本関数によってメモリが確保される、エラー注入状態を保持するための構造体。

失敗モードと境界条件

  • 共有メモリの要求が失敗した場合、PostgreSQL の共有メモリ初期化メカニズムによってエラーが報告される。

保守担当者向けメモ

  • 本関数は shmem_request_hook に登録されることを意図している。
  • 既存のフックを連鎖させる設計となっているため、他のモジュールとの競合を避けるために prev_shmem_request_hook の呼び出しを維持すること。
