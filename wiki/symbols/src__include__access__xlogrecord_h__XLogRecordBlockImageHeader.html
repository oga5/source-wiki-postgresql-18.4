XLogRecordBlockImageHeader

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogrecord.h:141

目的

WALレコード内にフルページイメージ（FPI）が含まれる際、そのイメージのサイズ、ホール（ゼロ埋め領域）の位置、および圧縮状態などのメタデータを保持するために使用されます。

シグネチャまたは形状

typedef struct XLogRecordBlockImageHeader
{
    uint16      length;         /* ページイメージのバイト数 */
    uint16      hole_offset;    /* ホール開始位置までのバイト数 */
    uint8       bimg_info;      /* フラグビット */
} XLogRecordBlockImageHeader;

入力と出力

  • **入力**: WALレコード構築時に、XLogRecordBlockHeader に続くデータとして書き込まれます。
  • **出力**: XLogReader によるWAL解析時に、フルページイメージの復元およびデコードのための制御情報として読み取られます。

前提条件と不変条件

  • XLogRecordBlockHeader 内のフラグで BKPBLOCK_HAS_IMAGE が設定されている場合にのみ、本ヘッダーが後続します。
  • bimg_info に BKPIMAGE_HAS_HOLE および BKPIMAGE_COMPRESSED() の両方が設定されている場合、本ヘッダーの直後に XLogRecordBlockCompressHeader が続きます。

アルゴリズムまたは振る舞い

1. WALレコード構築時、ページイメージからホールを除去し、必要に応じて圧縮を適用した後のサイズを length に格納します。 2. hole_offset には、ページ内のホール開始位置を記録します。 3. bimg_info を用いて、イメージの圧縮状態やホールの有無をWAL解析側に伝達します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xloginsert.c において、WALレコード生成時に本構造体のインスタンスが操作されます。
  • **呼び出し先**: 特定の関数呼び出し関係はソースから直接特定できませんが、XLogReader を含むWAL解析処理全般で参照されます。

関連データ構造

  • XLogRecordBlockHeader: 本ヘッダーの直前に配置されるブロックメタデータ。
  • XLogRecordBlockCompressHeader: BKPIMAGE_COMPRESSED が有効な場合に後続する圧縮情報ヘッダー。

失敗モードと境界条件

  • 本構造体自体は固定長ですが、後続するデータ（イメージ本体や圧縮ヘッダー）は可変長です。解析時には bimg_info のフラグに基づき、動的にオフセットを計算する必要があります。

保守担当者向けメモ

  • bimg_info の上位ビットはリソースマネージャが自由に使用可能です。
  • メモリ上で操作する際は、アライメントに注意してください。offsetof(XLogRecordBlockImageHeader, bimg_info) + sizeof(uint8) のような計算式が、構造体内のフィールドアクセスやオフセット計算に利用されています。
