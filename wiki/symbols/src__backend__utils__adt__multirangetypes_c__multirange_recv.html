multirange_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes.c:336

目的

バイナリ形式の入力ストリームからマルチレンジ型（MultirangeType）の値を構築します。

シグネチャまたは形状

Datum multirange_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • PG_GETARG_POINTER(0): StringInfo 形式の入力バッファ。
  • PG_GETARG_OID(1): マルチレンジ型の OID。
  • PG_GETARG_INT32(2): 型修飾子（typmod）。
  • **出力**:
  • Datum: 構築された MultirangeType へのポインタ。

前提条件と不変条件

  • get_multirange_io_data を介して、対象の型に対する適切な I/O キャッシュが取得可能であること。
  • 入力バッファは PostgreSQL のバイナリプロトコルに従った形式であること。
  • 構築されたマルチレンジは、内部的に make_multirange を経由することで、ソート済みかつ非重複という不変条件が保証されます。

アルゴリズムまたは振る舞い

1. get_multirange_io_data を呼び出し、型情報と受信関数を取得します。 2. バッファからレンジの個数（range_count）を読み取り、レンジ格納用の配列を確保します。 3. 各レンジについて以下の処理を繰り返します。

  • バッファからレンジのバイナリ長とデータを読み取ります。
  • ReceiveFunctionCall を呼び出し、個々のレンジ型をデシリアライズします。

    • 4. make_multirange を呼び出し、読み取ったレンジの配列から正規化された MultirangeType を生成します。 5. 最後に pq_getmsgend でバッファの終端を確認し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（SQL レベルのバイナリ入力処理）。
  • **呼び出し先**:
  • get_multirange_io_data
  • pq_getmsgint, pq_getmsgbytes, pq_getmsgend
  • ReceiveFunctionCall
  • make_multirange

関連データ構造

  • MultirangeType: マルチレンジのメモリ上の表現。
  • MultirangeIOData: I/O 処理中に使用されるキャッシュ構造体。
  • StringInfo: 入力バッファの管理構造体。
  • RangeType: 個々のレンジ要素。

失敗モードと境界条件

  • range_count が不正な値の場合、palloc_array によるメモリ確保時にエラーが発生する可能性があります。
  • ReceiveFunctionCall が失敗した場合、例外がスローされます。
  • バッファの読み取り位置が不整合な場合、pq_getmsgend 等でエラーが発生します。

保守担当者向けメモ

  • 本関数は MultirangeType のバイナリデシリアライズの入り口です。
  • tmpbuf を使用して個々のレンジを一時的に保持する設計となっており、ループ終了後に pfree で解放されます。
  • レンジの正規化ロジックは make_multirange 内にカプセル化されているため、本関数側で個別にソートやマージを行う必要はありません。
