get_val

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_io.c:95

目的

hstore 文字列形式のパースにおいて、キーまたは値のトークンを抽出する。

シグネチャまたは形状

static bool get_val(HSParser *state, bool ignoreeq, bool *escaped)

入力と出力

  • **入力**:
  • state: パース状態を保持する HSParser 構造体へのポインタ。
  • ignoreeq: = を区切り文字として扱うか否かを示すフラグ。
  • escaped: エスケープされた値であるかを格納する出力用ポインタ。
  • **出力**:
  • 戻り値: トークンの抽出に成功した場合は true、終端に達した場合は false。
  • state->word: 抽出された文字列が格納されるバッファ。
  • escaped: ダブルクォートで囲まれていた場合に true が設定される。

前提条件と不変条件

  • state->ptr は解析対象文字列の現在位置を指している必要がある。
  • state->word には palloc されたメモリが割り当てられ、state->wordlen でサイズが管理される。
  • RESIZEPRSBUF マクロにより、必要に応じてバッファサイズが拡張される。

アルゴリズムまたは振る舞い

有限オートマトンを用いて文字列を走査する。 1. GV_WAITVAL: トークンの開始を待機。" であればエスケープモードへ、\ であればエスケープ文字処理へ、それ以外は通常値として読み込みを開始。 2. GV_INVAL: 通常の値の読み込み。ignoreeq が false の場合の = や、空白文字、終端文字でトークンを終了する。 3. GV_INESCVAL: ダブルクォート内の読み込み。" が出現するまで文字をバッファに格納する。 4. GV_WAITESCIN / GV_WAITESCESCIN: エスケープ文字（\）直後の処理。次の文字をそのままバッファにコピーする。

  • 処理中、state->ptr を進めながらトークンを構築し、終了条件を満たした時点で true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_hstore（contrib/hstore/hstore_io.c 内の 233 行目および 284 行目から呼び出しを確認）。
  • **呼び出し先**: palloc（メモリ確保）、scanner_isspace（空白判定）、elog（エラー報告）、および PRSSYNTAXERROR, PRSEOF, RESIZEPRSBUF などのマクロ。

関連データ構造

  • HSParser: パース中の状態（現在のポインタ、バッファ、長さ等）を保持する構造体。

失敗モードと境界条件

  • PRSSYNTAXERROR: 不適切な位置での = 出現時に発生。
  • PRSEOF: エスケープ処理中に文字列が予期せず終了した場合に発生。
  • elog(ERROR): 未定義の内部状態（st）に遷移した場合に発生。
  • 文字列終端（\0）に達した際の挙動は、現在の状態に応じてトークン終了またはパース終了として扱われる。

保守担当者向けメモ

  • state->word のメモリ管理は HSParser の呼び出し元に依存する可能性があるため、メモリリークや二重解放に注意すること。
  • ignoreeq フラグの切り替えにより、hstore のキーと値のパースルールを柔軟に変更している。
