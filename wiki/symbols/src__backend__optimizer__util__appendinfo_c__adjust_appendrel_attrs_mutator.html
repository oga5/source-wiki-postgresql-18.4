adjust_appendrel_attrs_mutator

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:218

目的

クエリ最適化プロセスにおいて、親リレーションに対して定義された式ツリー内の Var ノードを、対応する子リレーション（継承テーブルやパーティション）の属性へ再帰的に変換・置換します。

シグネチャまたは形状

static Node *adjust_appendrel_attrs_mutator(Node *node, adjust_appendrel_attrs_context *context)

入力と出力

  • **入力**:
  • node: 変換対象の式ツリーノード。
  • context: AppendRelInfo 配列およびプランナのコンテキスト情報を含む adjust_appendrel_attrs_context 構造体。
  • **出力**: 変換後の Node ツリー。変換が不要な場合は元のノードが返されます。

前提条件と不変条件

  • varlevelsup == 0 である（サブクエリを跨がない）Var ノードのみが変換対象です。
  • ROWID_VAR プレースホルダーは、リーフリレーションへの変換が可能な場合にのみ具体的な変数へ置換されます。
  • 属性の不一致（型や照合順序の不一致）が検出された場合、elog(ERROR) により処理が中断されます。

アルゴリズムまたは振る舞い

1. **ノード走査**: expression_tree_mutator を介して式ツリーを再帰的に走査します。 2. **Var ノードの変換**:

  • varno が AppendRelInfo の parent_relid と一致する場合、child_relid に書き換えます。
  • varattno > 0 の場合、translated_vars を参照して対応する子テーブルの属性へ置換します。
  • varattno == 0（全列参照）の場合、子テーブルの型に合わせて ConvertRowtypeExpr を生成するか、RowExpr を構築します。

    • 3. **ROWID_VAR 処理**: ROWID_VAR を検出した場合、ターゲットとなるリーフリレーションを特定し、対応する変数へ変換を試みます。 4. **メタデータ保持**: 変換後の Var に対して、元の varnullingrels や varreturningtype を適切にマージ・継承します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • adjust_appendrel_attrs (直接の呼び出し)
  • expression_tree_mutator (再帰的な走査による間接呼び出し)
  • RestrictInfo 関連の処理（clause, orclause の変換）
  • **呼び出し先**:
  • copyObject, list_nth, bms_add_members, makeNode, rt_fetch 等のノード操作関数。
  • get_rel_name (エラー報告用)。

関連データ構造

  • AppendRelInfo: 親子関係の変換情報を保持。
  • adjust_appendrel_attrs_context: 変換に必要なコンテキスト情報。
  • Var, RowExpr, ConvertRowtypeExpr: 変換対象および生成されるノード。

失敗モードと境界条件

  • **属性不在**: varattno が translated_vars の範囲外である場合、または対応する変換先が NULL の場合にエラーとなります。
  • **型不一致**: varreturningtype や varnullingrels を非 Var ノードに適用しようとした場合にエラーとなります。
  • **複数リーフ**: ROWID_VAR の変換時に複数のリーフリレーションが特定された場合、エラーとなります。

保守担当者向けメモ

  • 本関数は expression_tree_mutator を利用した再帰的な変換を行うため、新しいノードタイプが追加された場合は nodeFuncs.h 等の定義と整合性を取る必要があります。
  • varnullingrels のマージ処理は、パーティション分割された結合（partitionwise joining）において重要です。将来的に非 Var ノードへの変換が必要となった場合は、PlaceHolderVar を用いたラップ等の設計変更を検討してください。
