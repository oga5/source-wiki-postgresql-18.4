link

定義

  • 種別: function
  • 定義箇所: src/port/win32link.c:17

目的

Windows環境において、POSIX準拠の link() 関数をエミュレートし、ファイルシステムレベルでハードリンクを作成する。

シグネチャまたは形状

int link(const char *src, const char *dst);

入力と出力

  • **入力**:
  • src: 既存のファイルパス（ANSI文字列）。
  • dst: 作成するハードリンクのパス（ANSI文字列）。
  • **出力**:
  • 成功時: 0
  • 失敗時: -1（errno が適切に設定される）

前提条件と不変条件

  • Windows APIの CreateHardLinkA を使用するため、ハードリンクは同一ボリューム内でのみ作成可能である。
  • 入力パスはマルチバイト文字（ANSI）として扱われる。

アルゴリズムまたは振る舞い

1. Windows APIの CreateHardLinkA(dst, src, NULL) を呼び出す。 2. 戻り値が0以外（成功）の場合、0 を返す。 3. 戻り値が0（失敗）の場合、GetLastError() でエラーコードを取得し、_dosmaperr() を介してPOSIX準拠の errno に変換した上で、-1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内には、本関数を呼び出している具体的な箇所は記載されていない。
  • **呼び出し先**:
  • CreateHardLinkA (Windows API)
  • GetLastError (Windows API)
  • _dosmaperr (PostgreSQL内部の互換性関数)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • CreateHardLinkA が失敗した場合、_dosmaperr によりWindows固有のエラーコードがPOSIXの errno にマッピングされる。
  • 同一ボリューム外へのリンク作成など、OSの制約に抵触する場合は失敗する。

保守担当者向けメモ

  • 本関数は CreateHardLinkA を使用しているため、Unicodeパスを扱う必要がある場合は CreateHardLinkW を使用する別の実装が必要となる可能性がある。
  • c.h に依存しており、PostgreSQLのプラットフォーム抽象化レイヤーの一部として機能する。
