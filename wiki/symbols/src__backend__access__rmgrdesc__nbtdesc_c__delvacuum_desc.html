delvacuum_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/nbtdesc.c:195

目的

B-treeインデックスの VACUUM および DELETE 操作に関連するWALレコードのペイロードを解析し、削除されたオフセットや更新されたTIDリストを人間が読める形式で StringInfo バッファに出力する。

シグネチャまたは形状

static void delvacuum_desc(StringInfo buf, char *block_data, uint16 ndeleted, uint16 nupdated)

入力と出力

  • buf: 出力先の StringInfo バッファ。
  • block_data: 解析対象のWALブロックデータ（char *）。
  • ndeleted: 削除されたオフセットの数。
  • nupdated: 更新されたオフセットの数。
  • 出力: 戻り値なし。buf に整形された文字列が追記される。

前提条件と不変条件

  • block_data は、ndeleted 個の OffsetNumber、続いて nupdated 個の OffsetNumber、最後に nupdated 個の xl_btree_update 構造体が連続して配置されている物理レイアウトである必要がある。
  • updates->ndeletedtids は常に 0 より大きい必要がある（Assert による検証）。

アルゴリズムまたは振る舞い

1. block_data の先頭から ndeleted 個の OffsetNumber を読み取り、array_desc を用いて「deleted: [...]」形式で出力する。 2. 続いて nupdated 個の OffsetNumber を読み取り、更新対象のオフセットとして保持する。 3. 更新対象の各オフセットに対し、対応する xl_btree_update 構造体を走査する。 4. 各更新オブジェクトについて、オフセット番号と、削除されたTID（ptids）のリストを「{ off: X, nptids: Y, ptids: [...] }」形式で出力する。 5. 可変長データであるため、各 xl_btree_update の処理後にポインタを SizeOfBtreeUpdate + ndeletedtids * sizeof(uint16) 分だけ進める。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/rmgrdesc/nbtdesc.c 内の btree_desc 関数（WALレコードの解析中に呼び出される）。
  • 呼び出し先: appendStringInfo 系関数、array_desc。

関連データ構造

  • xl_btree_update: B-tree更新操作を記述する構造体。
  • OffsetNumber: インデックスページ内のオフセット位置。
  • StringInfo: PostgreSQLの文字列バッファ構造体。

失敗モードと境界条件

  • block_data のポインタ演算が物理レイアウトと一致しない場合、メモリ破壊や誤った値の出力が発生する。
  • ndeleted や nupdated が実際のデータ量と乖離している場合、不正なメモリ領域を参照する可能性がある。

保守担当者向けメモ

  • 本関数はWALの可視化専用であり、インデックスの物理操作には影響しない。
  • xl_btree_update の構造体定義（nbtxlog.h）に変更があった場合、ポインタ演算のオフセット計算を必ず見直すこと。
  • 可読性を優先して物理データ構造とは異なる形式で出力しているため、デバッグ時には nbtxlog.h の定義と照らし合わせること。
