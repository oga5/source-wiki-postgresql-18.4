euc_jp2mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c:408

目的

EUC_JP形式の文字列を、PostgreSQL内部表現であるMULE_INTERNAL（MIC）形式へ変換する。

シグネチャまたは形状

static int euc_jp2mic(const unsigned char *euc, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • euc: 変換元EUC_JP文字列へのポインタ。
  • p: 変換先バッファへのポインタ。
  • len: 変換対象のバイト長。
  • noError: 変換エラー時にエラーを報告せず中断するかどうかを指定するフラグ。
  • **出力**:
  • 戻り値: 変換処理したバイト数。
  • p: 変換後のMIC文字列（ヌル終端済み）。

前提条件と不変条件

  • euc は有効なEUC_JPバイト列であること。
  • p は十分なサイズのバッファを指していること。
  • 変換処理中、len が0になるまでループが継続される。

アルゴリズムまたは振る舞い

1. 入力文字列を1文字ずつ走査する。 2. ASCII文字（高ビットが立っていない場合）はそのままコピーする。 3. マルチバイト文字の場合、pg_encoding_verifymbchar で妥当性を検証する。 4. EUC_JPの制御コードに基づき、以下の通りMICへ変換する：

  • SS2 (0x8E): LC_JISX0201K を付与し、後続1バイトをコピー。
  • SS3 (0x8F): LC_JISX0212 を付与し、後続2バイトをコピー。
  • その他: LC_JISX0208 を付与し、2バイトをコピー。

    • 5. 最後に p の末尾にヌル文字を書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mb/conversion_procs/euc_jp_and_sjis/euc_jp_and_sjis.c 内の euc_jp_to_mic 関数から呼び出される。
  • **呼び出し先**:
  • pg_encoding_verifymbchar: 文字列の妥当性検証。
  • report_invalid_encoding: 不正なエンコーディング検出時のエラー報告。

関連データ構造

  • PG_EUC_JP: エンコーディング識別子。
  • LC_JISX0201K, LC_JISX0212, LC_JISX0208: MIC形式における文字セット識別子。

失敗モードと境界条件

  • c1 == 0 の場合、ASCII終端とみなされ、noError が偽であればエラーを報告する。
  • pg_encoding_verifymbchar が負の値を返した場合、不正なマルチバイト列として noError に応じてエラー報告または中断を行う。

保守担当者向けメモ

  • 本関数は euc_jp_to_mic からの呼び出しを前提としており、直接的な外部インターフェースではない。
  • 変換ロジックはEUC_JPのバイト構造に強く依存しているため、エンコーディング仕様の変更時には SS2/SS3 の処理を確認すること。
