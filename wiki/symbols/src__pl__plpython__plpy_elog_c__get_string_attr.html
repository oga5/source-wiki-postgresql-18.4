get_string_attr

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:579

目的

Python オブジェクトから指定された属性名の値を文字列として取得し、PostgreSQL のメモリコンテキストにコピーして格納します。

シグネチャまたは形状

static void get_string_attr(PyObject *obj, char *attrname, char **str)

入力と出力

  • **入力**:
  • obj: 属性を取得する対象の PyObject。
  • attrname: 取得する属性名の文字列。
  • str: 取得した文字列を格納するための char* へのポインタ。
  • **出力**: なし（str が指す先に pstrdup された文字列が書き込まれます）。

前提条件と不変条件

  • obj は有効な Python オブジェクトである必要があります。
  • str は呼び出し元で初期化されたポインタである必要があります。
  • 取得された属性値が None の場合、str は更新されません。

アルゴリズムまたは振る舞い

1. PyObject_GetAttrString を使用して、obj から attrname に対応する属性値を取得します。 2. 取得した値が NULL でなく、かつ Py_None でない場合、PLyUnicode_AsString を介して Python 文字列を C 文字列に変換します。 3. 変換された文字列を pstrdup で PostgreSQL のメモリコンテキストに複製し、*str に代入します。 4. 最後に Py_XDECREF を呼び出し、取得した属性値の参照カウントを適切に解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_elog.c 内の各所（detail, hint, schema_name 等の属性抽出時に使用）。
  • **呼び出し先**: PyObject_GetAttrString, PLyUnicode_AsString, pstrdup, Py_XDECREF。

関連データ構造

  • PyObject: Python の汎用オブジェクト型。

失敗モードと境界条件

  • 属性が存在しない場合や取得に失敗した場合、PyObject_GetAttrString は NULL を返しますが、本関数は Py_XDECREF を通じて安全に処理を終了します。
  • 属性値が None の場合は何も行われません。

保守担当者向けメモ

  • 本関数は PostgreSQL のメモリ管理（palloc/pstrdup）と Python の参照カウント管理を橋渡しするユーティリティです。メモリリークを防ぐため、Py_XDECREF の呼び出しが必須となっています。
