SockAddr

定義

  • **種別**: struct
  • **定義箇所**: src/include/libpq/pqcomm.h:30

目的

PostgreSQLの通信において、IPv4、IPv6、およびUnixドメインソケットを含む多様なネットワークアドレスを汎用的に保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	struct sockaddr_storage addr;
	socklen_t	salen;
} SockAddr;

入力と出力

  • **入力**: ネットワーク接続に関連するアドレス情報（sockaddr_storage）およびその長さ（socklen_t）。
  • **出力**: 構造体として保持されたアドレス情報。

前提条件と不変条件

  • addr フィールドは、任意のソケットアドレス型を格納できる十分なサイズを持つ struct sockaddr_storage である必要があります。
  • salen は、addr に格納された有効なアドレスデータのバイト長と一致している必要があります。

アルゴリズムまたは振る舞い

本構造体はデータ保持に特化しており、直接的なアルゴリズムは持ちません。通信処理において、接続先（リモート）や接続元（ローカル）のアドレスを型安全に受け渡し、ソケット関連のシステムコール（bind, connect, accept 等）へ渡すためのコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元（参照例）**:
  • src/backend/libpq/auth.c: クライアントおよびローカルアドレスの保持に使用。
  • src/backend/libpq/hba.c: ホストベース認証（HBA）におけるアドレス比較に使用。
  • src/include/libpq/libpq-be.h: バックエンドの接続状態（Port構造体等）の一部として保持。
  • src/interfaces/libpq/fe-cancel.c: キャンセル要求時のリモートアドレス保持に使用。
  • **呼び出し先**: 本構造体自体はデータ構造であり、特定の関数を呼び出すことはありません。

関連データ構造

  • struct sockaddr_storage: SockAddr の内部でアドレスを保持するために使用される汎用ソケットアドレス構造体。
  • Port (定義は libpq-be.h 等): 通信セッションを管理する構造体で、SockAddr をメンバとして保持します。

失敗モードと境界条件

  • **サイズ制限**: sockaddr_storage の容量を超えるアドレス情報は格納できません。
  • **不整合**: salen の値が実際の addr の内容と一致しない場合、ソケット関連のシステムコールでエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は libpq のフロントエンド・バックエンド双方で共有されるため、変更時は両方の互換性に注意してください。
  • sys/socket.h 等のシステムヘッダーに依存しているため、プラットフォーム固有のソケット定義の差異が sockaddr_storage の挙動に影響を与える可能性があります。
