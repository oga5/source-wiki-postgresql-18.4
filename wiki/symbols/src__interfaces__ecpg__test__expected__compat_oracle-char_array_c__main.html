目的

ECPG（Embedded SQL for C）における文字配列の取り扱いおよびSQLDAを用いた動的SQL実行の挙動を検証する回帰テスト用プログラム。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム実行時にデータベース ecpg1_regression へ接続）。
  • 出力: 標準出力へテスト結果（文字列およびインジケータ値）を出力。エラー発生時は標準エラー出力へデバッグ情報を出力し、異常終了する。

前提条件と不変条件

  • データベース ecpg1_regression が利用可能であること。
  • ECPGランタイムライブラリが正しくリンクされていること。
  • sqlca.sqlwarn[0] == 'W' のチェックにより、データ切り詰め等の警告を捕捉する。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、データベースへ接続。 2. **テーブル作成**: strdbase テーブルを作成し、様々な長さの文字列データを挿入。 3. **静的フェッチ**: カーソル C を宣言し、varchar 型フィールドを固定長 char 配列へフェッチ。この際、バッファサイズとインジケータ変数を用いてパディングや切り詰めを検証。 4. **ループ処理**: 100回を上限としてフェッチを繰り返し、結果を標準出力へ表示。 5. **終了処理**: カーソルを閉じ、テーブルを削除し、トランザクションをコミットして接続を切断。

呼び出し元と呼び出し先

  • 呼び出し元: OS（エントリポイント）。
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGtrans, warn, printf, strncpy, memset, exit。
  • 補足: 呼び出し元・呼び出し先の詳細な依存関係グラフは、本ソースコードの範囲内ではこれらライブラリ関数およびECPGプリプロセッサ生成関数への呼び出しに限定される。

関連データ構造

  • sqlda_t: 動的SQLの結果保持用構造体。
  • sqlvar_t: sqlda_t 内のカラム情報保持用構造体。
  • sqlca: SQL通信領域（エラーおよび警告の確認に使用）。

失敗モードと境界条件

  • sqlca.sqlcode < 0: SQL実行エラー発生時に exit(1) で終了。
  • sqlca.sqlcode == ECPG_NOT_FOUND: フェッチループの終了条件として使用。
  • 文字列の長さ: shortstr（5バイト）および bigstr（11バイト）のバッファサイズを超えるデータ取得時の挙動を検証。
