CatalogIdMapEntry

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/common.c:57

目的

CatalogId（OIDとテーブルOIDのペア）をキーとして、対応する DumpableObject や所有する拡張機能情報を高速に検索・管理するためのハッシュテーブル要素です。

シグネチャまたは形状

typedef struct _catalogIdMapEntry
{
	CatalogId	catId;			/* 索引付けされた CatalogId */
	uint32		status;			/* ハッシュステータス */
	uint32		hashval;		/* CatalogId のハッシュコード */
	DumpableObject *dobj;		/* 関連付けられた DumpableObject（存在する場合） */
	ExtensionInfo *ext;			/* 所有する拡張機能（存在する場合） */
} CatalogIdMapEntry;

入力と出力

  • 本構造体は lib/simplehash.h を用いたハッシュテーブルの要素として使用されます。
  • 入力: CatalogId をキーとして登録・検索されます。
  • 出力: 検索成功時に DumpableObject へのポインタを返します。

前提条件と不変条件

  • simplehash.h の仕様に基づき、ハッシュテーブル内では直接オブジェクトを保持せず、ポインタを介して管理されます。
  • メモリ配置が動的に変更される可能性があるため、本構造体へのポインタの生存期間には注意が必要です。

アルゴリズムまたは振る舞い

  • pg_dump のスキーマ収集プロセスにおいて、AssignDumpId 等を通じてオブジェクトが生成されるたびに、この構造体がハッシュテーブルに登録されます。
  • findObjectByCatalogId 等の関数により、カタログIDから対応するダンプ対象オブジェクトを効率的に特定するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/common.c 内のハッシュテーブル操作関数群（findObjectByCatalogId 等）から参照されます。
  • 呼び出し先: 構造体内部で DumpableObject および ExtensionInfo を保持します。

関連データ構造

  • CatalogId: 検索キーとなる識別子。
  • DumpableObject: ダンプ対象となるデータベースオブジェクトの抽象。
  • ExtensionInfo: 拡張機能に関する情報。
  • catalogIdHash: 本構造体を要素として管理するハッシュテーブル。

失敗モードと境界条件

  • 該当する CatalogId がハッシュテーブルに存在しない場合、検索関数は NULL を返します。
  • ハッシュテーブルの動的リサイズに伴い、本構造体へのポインタが無効化される可能性があるため、保持し続けることは推奨されません。

保守担当者向けメモ

  • 本構造体は simplehash.h を使用しています。ハッシュテーブルの定義（SH_ELEMENT_TYPE）として CatalogIdMapEntry が指定されています。
  • pg_dump のスキーマ収集ロジックにおいて、オブジェクト間の依存関係や継承関係を解決する際の基盤となります。
