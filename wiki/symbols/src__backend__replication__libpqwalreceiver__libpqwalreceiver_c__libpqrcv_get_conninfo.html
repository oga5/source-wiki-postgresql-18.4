libpqrcv_get_conninfo

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:347

目的

アクティブなレプリケーション接続（WalReceiverConn）から、セキュリティ上の機密情報を難読化した接続文字列を生成して返します。

シグネチャまたは形状

char *libpqrcv_get_conninfo(WalReceiverConn *conn)

入力と出力

  • **入力**: WalReceiverConn *conn（有効なストリーミング接続状態を保持する構造体）
  • **出力**: 接続文字列を含む char *（メモリは呼び出し元で解放が必要）。エラーまたはバッファ破損時は NULL を返す。

前提条件と不変条件

  • conn->streamConn が NULL であってはならない（Assert により検証）。
  • PQconninfo がメモリ不足等で失敗した場合は ereport(ERROR) を発行する。

アルゴリズムまたは振る舞い

1. PQconninfo を使用して現在の接続オプションを取得する。 2. 取得したオプションを順次走査し、以下の条件に該当するものを除外する。

  • デバッグ用オプション（dispchar に 'D' を含む）。
  • 値が空または NULL のオプション。

    • 3. セキュリティ機密オプション（dispchar に '*' を含む）は、値を ******** に置換する。 4. 処理済みのオプションをスペース区切りで PQExpBuffer に連結する。 5. PQconninfoFree で一時的なオプション配列を解放し、バッファの内容を複製して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PQWalReceiverFunctions 構造体を通じて、WALレシーバープロセスから間接的に参照される。
  • **呼び出し先**:
  • libpq API: PQconninfo, PQconninfoFree
  • PQExpBuffer API: initPQExpBuffer, appendPQExpBuffer, termPQExpBuffer, PQExpBufferDataBroken

関連データ構造

  • WalReceiverConn: 接続状態を保持する内部構造体。
  • PQconninfoOption: libpq が提供する接続オプションの構造体。
  • PQExpBufferData: 文字列構築用バッファ。

失敗モードと境界条件

  • PQconninfo が NULL を返した場合、ERRCODE_OUT_OF_MEMORY でエラー終了する。
  • PQExpBuffer の構築中にメモリ不足等で破損（broken）が発生した場合、NULL を返す。

保守担当者向けメモ

  • 本関数は libpq の PQconninfo が返す dispchar フラグに依存して機密情報を制御している。新しい接続オプションが追加された際は、dispchar の設定が適切か確認すること。
