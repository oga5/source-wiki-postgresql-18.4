pg_tzenumerate_end

定義

  • 種別: function
  • 定義箇所: src/timezone/pgtz.c:413

目的

タイムゾーン列挙処理で使用した pg_tzenum 構造体および関連するリソースを解放し、メモリリークを防ぐ。

シグネチャまたは形状

void pg_tzenumerate_end(pg_tzenum *dir)

入力と出力

  • **入力**: dir (pg_tzenum *) - 列挙状態を保持する構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • dir は pg_tzenumerate_start によって初期化された有効なポインタであること。
  • dir->depth は現在のディレクトリ走査の深さを表し、0 以上の値を持つ。

アルゴリズムまたは振る舞い

1. dir->depth が 0 以上である間、以下の処理を繰り返す。

  • FreeDir を呼び出し、現在の深さのディレクトリ記述子を閉じる。
  • pfree を呼び出し、現在の深さのディレクトリパス文字列を解放する。
  • dir->depth をデクリメントする。

    • 2. 最後に pfree(dir) を呼び出し、pg_tzenum 構造体自体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/datetime.c:5390
  • **呼び出し先**:
  • FreeDir (ディレクトリ記述子の解放)
  • pfree (メモリの解放)

関連データ構造

  • pg_tzenum: タイムゾーン列挙のためのイテレータ状態を保持する構造体。

失敗モードと境界条件

  • dir が NULL の場合、pfree 等のメモリ管理関数内でクラッシュする可能性がある。
  • 既に解放済みのポインタを渡した場合の挙動は未定義。

保守担当者向けメモ

  • 本関数は pg_tzenumerate_start で割り当てられたメモリを完全にクリーンアップする責任を持つ。構造体定義に変更を加えた場合は、本関数の解放ロジックも同期させること。
