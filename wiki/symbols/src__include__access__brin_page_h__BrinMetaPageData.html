BrinMetaPageData

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_page.h:64

目的

BRIN（Block Range Index）インデックスの先頭ページ（ブロック0）に格納されるメタデータ構造体です。インデックスのバージョン管理および範囲設定の保持を目的とします。

シグネチャまたは形状

typedef struct BrinMetaPageData
{
	uint32		brinMagic;
	uint32		brinVersion;
	BlockNumber pagesPerRange;
	BlockNumber lastRevmapPage;
} BrinMetaPageData;

入力と出力

  • 入力: ページバッファから PageGetContents を介して取得されるポインタ。
  • 出力: インデックスの構成情報（マジックナンバー、バージョン、範囲あたりのページ数、リバースマップの最終ページ番号）。

前提条件と不変条件

  • BRINインデックスのブロック0に配置される必要があります。
  • PageGetContents を使用してページ内のデータ領域からアクセスされます。

アルゴリズムまたは振る舞い

  • インデックスの初期化時や読み込み時に、この構造体を通じてインデックスのメタ情報が検証・参照されます。
  • brinMagic により、対象ページがBRINインデックスのメタページであることを識別します。
  • pagesPerRange は、インデックスが管理する各範囲のブロック数を定義します。
  • lastRevmapPage は、リバースマップの末尾ページを追跡するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/pageinspect/brinfuncs.c
  • src/backend/access/brin/brin.c
  • src/backend/access/brin/brin_pageops.c
  • src/backend/access/brin/brin_revmap.c
  • 呼び出し先:
  • 実行時の呼び出し先情報は提供されていません。

関連データ構造

  • BlockNumber: storage/block.h で定義されるブロック番号型。
  • Page: ページ全体を扱うための抽象データ型。

失敗モードと境界条件

  • brinMagic が期待値と一致しない場合、メタページとして正しく認識されません。
  • lastRevmapPage の値が不正な場合、リバースマップの走査や更新処理で境界エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は可変長ではありませんが、ページ内の配置については PageGetContents を経由する標準的なBRINページ操作に従ってください。
  • brinVersion の変更時には、関連する brin_xlog.h 等のログ記録ロジックとの整合性に注意してください。
