bms_equal

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/nodes/bitmapset.c:141

目的

2つの Bitmapset が論理的に等価であるか（同じ要素集合を持つか）を判定します。両者が NULL（空集合）である場合も等価とみなします。

シグネチャまたは形状

bool bms_equal(const Bitmapset *a, const Bitmapset *b)

入力と出力

  • **入力**: 比較対象となる2つの Bitmapset へのポインタ（a, b）。NULL を許容します。
  • **出力**: 両者が等価であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力される Bitmapset ポインタは、bms_is_valid_set による検証を通過する有効な構造体である必要があります。
  • 空集合は NULL ポインタとして表現されます。

アルゴリズムまたは振る舞い

1. **NULLチェック**:

  • 両方が NULL ならば true を返す。
  • 片方のみが NULL ならば false を返す。

    • 2. **構造比較**:

  • nwords（保持するワード数）が異なる場合は false を返す。

    • 3. **ビット比較**:

  • do-while ループを用いて、各ワード（words[i]）を順次比較する。
  • 1つでも異なるワードがあれば false を返し、すべて一致すれば true を返す。

  • **呼び出し先**:
  • bms_is_valid_set（アサーション内）

関連データ構造

  • Bitmapset: 集合を表現する構造体。nwords と words 配列を保持。

失敗モードと境界条件

  • 入力が NULL の場合、適切に処理され true または false が返されます。
  • nwords が異なる場合、ビット内容を走査することなく早期に false を返します。

保守担当者向けメモ

  • 本関数は Bitmapset の論理的な等価性を保証します。
  • src/backend/nodes/equalfuncs.c などでノードの等価性判定の一部として利用されています。
  • bms_is_valid_set による検証はデバッグビルド時のアサーションとして機能します。
