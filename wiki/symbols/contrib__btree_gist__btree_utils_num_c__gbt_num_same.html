gbt_num_same

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_num.c:211

目的

2つの GiST インデックスキー（数値範囲）が同一であるかを判定します。

シグネチャまたは形状

bool gbt_num_same(const GBT_NUMKEY *a, const GBT_NUMKEY *b, const gbtree_ninfo *tinfo, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の GBT_NUMKEY 配列。
  • tinfo: データ型固有の比較関数やサイズ情報を含む gbtree_ninfo 構造体。
  • flinfo: 関数呼び出し情報（FmgrInfo）。
  • **出力**:
  • bool: 両者の下限値および上限値が等しい場合に true を返します。

前提条件と不変条件

  • a および b は、tinfo->size で指定されたサイズに基づき、下限値と上限値が格納されている必要があります。
  • tinfo->f_eq は、各データ型の等価性を判定する有効な関数ポインタである必要があります。

アルゴリズムまたは振る舞い

1. 入力された GBT_NUMKEY 配列から、tinfo->size をオフセットとして下限値（lower）と上限値（upper）のポインタを抽出します。 2. tinfo->f_eq を使用して、a の下限値と b の下限値が等しいかを確認します。 3. 同様に、a の上限値と b の上限値が等しいかを確認します。 4. 両方の条件が満たされた場合のみ true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_bool.c, btree_cash.c, btree_date.c, btree_enum.c, btree_float4.c, btree_float8.c, btree_inet.c, btree_int2.c, btree_int4.c, btree_int8.c, btree_interval.c, btree_macaddr.c 等の各データ型実装ファイル。
  • **呼び出し先**: tinfo 構造体内に保持されている f_eq 関数。

関連データ構造

  • GBT_NUMKEY: 数値範囲を保持するキー構造。
  • gbtree_ninfo: データ型ごとの比較関数（f_eq 等）とサイズ情報を保持する設定構造体。
  • GBT_NUMKEY_R: 内部的な範囲表現（lower, upper ポインタのペア）。

失敗モードと境界条件

  • tinfo が指す関数ポインタが不正な場合、セグメンテーション違反が発生する可能性があります。
  • 入力キーのメモリレイアウトが tinfo->size と一致しない場合、誤った比較結果を返す可能性があります。

保守担当者向けメモ

  • 本関数は GiST インデックスの same メソッドとして利用されます。
  • 新しい数値型を追加する際は、gbtree_ninfo の f_eq に適切な比較関数を割り当てることで本関数が自動的に対応可能です。
