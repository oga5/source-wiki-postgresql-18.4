CreateCachedPlan

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/plancache.c:182

目的

クエリの実行計画をキャッシュするための CachedPlanSource 構造体を初期化し、メモリ上に確保する。

シグネチャまたは形状

CachedPlanSource *CreateCachedPlan(RawStmt *raw_parse_tree, const char *query_string, CommandTag commandTag)

入力と出力

  • **入力**:
  • raw_parse_tree: 解析対象の生パースツリー。
  • query_string: 対応する SQL クエリ文字列（必須）。
  • commandTag: コマンドの種類を示すタグ。
  • **出力**:
  • 初期化済みの CachedPlanSource 構造体へのポインタ。

前提条件と不変条件

  • query_string は NULL であってはならない。
  • 呼び出し元のメモリコンテキストは一時的なものと想定され、生成された CachedPlanSource は専用の AllocSetContextCreate で作成された長寿命なコンテキスト内に配置される。

アルゴリズムまたは振る舞い

1. CachedPlanSource 専用のメモリコンテキストを作成する。 2. 新しいコンテキストへ切り替え、palloc0 で CachedPlanSource 構造体を割り当てる。 3. raw_parse_tree をコピーし、query_string を複製して構造体に格納する。 4. 構造体の各フィールドを初期化する（is_valid や is_complete は false に設定される）。 5. 呼び出し元のメモリコンテキストへ戻り、作成した構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/prepare.c
  • src/backend/executor/functions.c
  • src/backend/executor/spi.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • copyObject (パースツリーの複製)
  • AllocSetContextCreate / MemoryContextSwitchTo (メモリ管理)
  • palloc0 / pstrdup (メモリ確保)

関連データ構造

  • CachedPlanSource: クエリのソースや解析状態を保持するメイン構造体。
  • RawStmt: 入力されるパースツリーの型。

失敗モードと境界条件

  • メモリ不足時には palloc 等によりエラーが発生する可能性がある。
  • query_string が NULL の場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は計画キャッシュ作成の「第1段階」である。完了には CompleteCachedPlan の呼び出しが必要。
  • 作成された CachedPlanSource は、エラー発生時に自動的にクリーンアップされるよう、呼び出し元のコンテキストを親として作成される。
