r_verb_suffix

目的

ルーマニア語のステミング処理において、動詞の活用語尾を削除し、語幹を正規化する。

シグネチャまたは形状

static int r_verb_suffix(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowball ステミング環境構造体。z->p に文字列、z->c にカーソル位置、z->I[2] に領域境界情報を含む）
  • 出力: int（成功時は 1、失敗時は 0、メモリ不足等のエラー時は負の値を返す）

前提条件と不変条件

  • z->I[2]（領域境界）が設定されていること。
  • z->c が z->I[2] 以上であること（領域内での処理）。

アルゴリズムまたは振る舞い

1. z->I[2] を境界として、後方検索（find_among_b）によりテーブル a_5 から動詞語尾を検索する。 2. マッチした場合、z->bra と z->ket を設定し、以下のいずれかの処理を行う。

  • ケース1: 語尾の直前が母音（g_v）であるか、または直前が 'u' であることを確認した上で slice_del を実行する。
  • ケース2: 条件なしで slice_del を実行する。

    • 3. 処理終了後、一時的に変更した z->lb を復元する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内では明示的な呼び出し元は特定されていません。
  • 呼び出し先: find_among_b（ユーティリティ）、out_grouping_b_U（ユーティリティ）、slice_del（ユーティリティ）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_5: 動詞語尾のマッチングに使用される among 構造体のテーブル。
  • g_v: 母音グループを定義するビットマップまたは配列。

失敗モードと境界条件

  • z->c < z->I[2]: 処理対象領域外のため即時終了（0 を返す）。
  • find_among_b がマッチしない場合: 処理を中断し 0 を返す。
  • slice_del が負の値を返した場合: エラーとしてその値をそのまま伝播させる。
