pgwin32_poll_signals

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:156

目的

Windows環境において、保留中のシグナルを検知・処理し、ソケット操作中にシグナル割り込みが発生したことを呼び出し元へ通知します。

シグネチャまたは形状

static int pgwin32_poll_signals(void)

入力と出力

  • **入力**: なし
  • **出力**:
  • 1: 保留中のシグナルが処理された場合。
  • 0: 保留中のシグナルがない場合。

前提条件と不変条件

  • UNBLOCKED_SIGNAL_QUEUE() マクロにより、シグナルキューの状態を判定します。
  • シグナルが処理された場合、副作用として errno が EINTR に設定されます。

アルゴリズムまたは振る舞い

1. UNBLOCKED_SIGNAL_QUEUE() を評価し、保留中のシグナルが存在するか確認します。 2. シグナルが存在する場合、pgwin32_dispatch_queued_signals() を呼び出してシグナルをディスパッチします。 3. errno を EINTR に設定し、1 を返します。 4. シグナルが存在しない場合は、何もせず 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/win32/socket.c 内の各ソケット関数（recv, send, select 等のラッパー関数）から、操作の冒頭で呼び出されます。
  • **呼び出し先**: pgwin32_dispatch_queued_signals()（src/backend/port/win32/signal.c で定義されていると推測される関数）。

関連データ構造

  • UNBLOCKED_SIGNAL_QUEUE(): シグナルキューの状態を判定するマクロ。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • シグナルが処理された場合、呼び出し元は errno が EINTR に設定されていることを前提に、操作の再試行や中断を行う必要があります。

保守担当者向けメモ

  • 本関数は、Windowsのソケット操作においてPOSIX準拠のシグナル割り込みをエミュレートするための重要なフックです。
  • pgwin32_dispatch_queued_signals の詳細な挙動については src/backend/port/win32/signal.c を参照してください。
