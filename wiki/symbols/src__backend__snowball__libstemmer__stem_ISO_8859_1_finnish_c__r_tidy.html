r_tidy

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_finnish.c:574

目的

フィンランド語のステミング処理の最終段階として、単語の語尾に残る不要な文字や二重母音の調整を行い、正規化された語幹を確定させる。

シグネチャまたは形状

static int r_tidy(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: 成功時に 1、処理対象外または条件不一致時に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z->I[1] に格納された境界値が、処理対象の有効範囲として設定されていること。
  • SN_env 構造体が初期化済みであり、r_mark_regions 等によって領域が定義されていること。

アルゴリズムまたは振る舞い

1. **領域制限**: z->I[1] を境界として処理範囲を制限する。 2. **語尾調整**: 以下の順序でパターンマッチングと削除を試みる。

  • r_LONG に合致する場合、その直前の文字を削除。
  • 母音（g_AEI）と子音（g_C）の並びを検出し、該当する場合は削除。
  • 'j' で終わる二重母音（'oj', 'uj'）を検出し、削除。
  • 'oj' の並びを検出し、削除。

    • 3. **最終正規化**: 最後に、母音（g_V1）と子音（g_C）の並びを逆方向に検索し、一致する部分を削除して語幹を確定させる。

呼び出し元と呼び出し先

  • 呼び出し元: finnish_ISO_8859_1_stem（src/backend/snowball/libstemmer/stem_ISO_8859_1_finnish.c:710）
  • 呼び出し先: r_LONG, slice_del, in_grouping_b, slice_to, eq_v_b

関連データ構造

  • struct SN_env: ステミングアルゴリズムの実行状態（現在のカーソル位置、境界、文字列バッファ等）を保持する。
  • g_AEI, g_C, g_V1: 文字グループを定義する静的データ。

失敗モードと境界条件

  • z->c が z->lb 以下になる場合、またはバッファの境界を超える場合は処理を中断し 0 を返す。
  • slice_to や slice_del がメモリ不足等のエラーを返した場合、負の値を伝播させる。

保守担当者向けメモ

  • ISO-8859-1版とUTF-8版の両方で同名の関数が存在するが、文字コードの扱いに依存するため、修正時は両方の整合性に注意が必要である。
