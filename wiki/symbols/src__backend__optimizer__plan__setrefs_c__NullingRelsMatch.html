NullingRelsMatch

定義

  • 種別: enum
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:34

目的

外部結合等において、NULLを生成するリレーション（nullingrels）の照合ルールを定義するために使用されます。

シグネチャまたは形状

typedef enum
{
    NRM_EQUAL,      /* expect exact match of nullingrels */
    NRM_SUBSET,     /* actual Var may have a subset of input */
    NRM_SUPERSET,   /* actual Var may have a superset of input */
} NullingRelsMatch;

入力と出力

本シンボルは列挙型であり、関数への引数として渡されることで、後続の式変換処理における比較ロジックを制御します。

前提条件と不変条件

  • NRM_EQUAL: 比較対象の nullingrels が完全に一致することを期待します。
  • NRM_SUBSET: 実際の Var が持つ nullingrels が、入力のサブセットであることを許容します。
  • NRM_SUPERSET: 実際の Var が持つ nullingrels が、入力のスーパーセットであることを許容します。

アルゴリズムまたは振る舞い

プランツリーの後処理において、式内の変数が参照するリレーションの nullingrels 属性を検証する際、この列挙値に基づいて一致判定の厳密さを切り替えます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/setrefs.c 内の複数の関数（fix_scan_expr や fix_join_expr 等のコンテキストを扱う関数）から引数として参照されています。
  • **呼び出し先**: 本シンボル自体は列挙型であり、関数を呼び出すことはありません。

関連データ構造

  • fix_scan_expr_context
  • fix_join_expr_context
  • fix_upper_expr_context
  • fix_windowagg_cond_context

    • （これらコンテキスト構造体のメンバとして保持されます）

失敗モードと境界条件

  • 適切な NullingRelsMatch モードを選択しない場合、式変換時に nullingrels の不整合が発生し、プランの正当性が損なわれる可能性があります。

保守担当者向けメモ

  • 本シンボルは setrefs.c における式変換の厳密さを制御する重要なフラグです。新しい式変換ロジックを追加する際は、既存の NRM_* モードが要件を満たすか確認してください。
