ApplyLauncherGetWorkerStartTime

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/launcher.c:1058

目的

指定されたサブスクリプションID（subid）に対応する論理レプリケーションワーカーの、最後に起動を試みた時刻を取得します。

シグネチャまたは形状

static TimestampTz ApplyLauncherGetWorkerStartTime(Oid subid)

入力と出力

  • **入力**: subid (Oid) - 情報を取得するサブスクリプションの識別子。
  • **出力**: TimestampTz - 最後に起動を試みた時刻。該当するエントリが存在しない場合は 0 を返します。

前提条件と不変条件

  • 共有メモリ上のハッシュテーブル last_start_times が初期化されている必要があります。
  • 本関数内で logicalrep_launcher_attach_dshmem() を呼び出し、動的共有メモリへのアタッチを保証しています。

アルゴリズムまたは振る舞い

1. logicalrep_launcher_attach_dshmem() を呼び出し、共有メモリへのアクセスを確保します。 2. dshash_find を使用して、共有ハッシュテーブル last_start_times から subid に一致するエントリを検索します。 3. エントリが見つからない場合は 0 を返します。 4. エントリが見つかった場合は、その last_start_time フィールドを読み取り、dshash_release_lock でロックを解放した後に時刻を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/launcher.c:1204 にて、サブスクリプションの再起動判定ロジックから呼び出されています。
  • **呼び出し先**:
  • logicalrep_launcher_attach_dshmem()
  • dshash_find
  • dshash_release_lock

関連データ構造

  • LauncherLastStartTimesEntry: 共有ハッシュテーブル last_start_times 内の各エントリを保持する構造体。last_start_time フィールドを含みます。

失敗モードと境界条件

  • 指定した subid がハッシュテーブル内に存在しない場合、エラーにはならず 0 を返します。
  • 共有メモリへのアタッチに失敗した場合の挙動は、logicalrep_launcher_attach_dshmem の実装に依存します。

保守担当者向けメモ

  • 本関数は last_start_times ハッシュテーブルの読み取り専用アクセスを提供します。
  • 共有メモリ上のロックを適切に解放するため、dshash_release_lock の呼び出しを忘れないようにしてください。
