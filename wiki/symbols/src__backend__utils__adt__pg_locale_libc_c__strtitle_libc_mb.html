strtitle_libc_mb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:283

目的

マルチバイトエンコーディング環境において、OSのlibcロケール機能を利用して文字列をタイトルケース（単語の先頭を大文字、それ以外を小文字）に変換する。

シグネチャまたは形状

static size_t strtitle_libc_mb(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: 変換結果を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換元の文字列。
  • srclen: src の長さ（負の場合は strlen(src) が使用される）。
  • locale: ロケール情報を含む pg_locale_t 構造体。
  • **出力**: 変換後の文字列のバイト長を返す。

前提条件と不変条件

  • srclen が INT_MAX / sizeof(wchar_t) を超える場合、メモリ不足エラー（ERRCODE_OUT_OF_MEMORY）が発生する。
  • 変換処理には palloc を使用してヒープメモリを確保する。

アルゴリズムまたは振る舞い

1. char2wchar を使用して、入力文字列をワイド文字（wchar_t）配列に変換する。 2. ワイド文字配列を走査し、直前の文字が英数字（iswalnum_l）であれば towlower_l で小文字に、そうでなければ towupper_l で大文字に変換する。 3. 変換後のワイド文字配列を wchar2char を使用して元のエンコーディングに戻す。 4. destsize が十分な大きさであれば、結果を dest にコピーし、NUL終端する。 5. 確保したワークスペースと結果用バッファを pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale_libc.c 内の関数から参照されている。
  • **呼び出し先**: strlen, palloc, char2wchar, towlower_l, towupper_l, iswalnum_l, pg_database_encoding_max_length, wchar2char, memcpy, pfree, ereport。

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。内部の locale_t 型のハンドルを使用する。

失敗モードと境界条件

  • 入力文字列が非常に長い場合、メモリ確保に失敗する可能性がある。
  • destsize が変換後の文字列長より小さい場合、コピーは行われない。

保守担当者向けメモ

  • 本関数はマルチバイト対応版である。シングルバイト版（strtitle_libc_sb）とは処理経路が異なるため注意すること。
  • pg_database_encoding_max_length() を用いて変換後の最大サイズを算出しているため、エンコーディングごとの特性に依存する。
