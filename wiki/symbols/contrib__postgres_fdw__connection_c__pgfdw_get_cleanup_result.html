pgfdw_get_cleanup_result

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/connection.c:1615

目的

リモートの PostgreSQL サーバーから送信される残りのクエリ結果を、タイムアウト制限付きで安全に読み取り、破棄または取得する。

シグネチャまたは形状

bool pgfdw_get_cleanup_result(PGconn *conn, TimestampTz endtime,
                              TimestampTz retrycanceltime,
                              PGresult **result,
                              bool *timed_out)

入力と出力

  • **入力**:
  • conn: リモート接続用の PGconn オブジェクト。
  • endtime: 処理を打ち切る絶対時刻。
  • retrycanceltime: キャンセル要求を再送する時刻。
  • **出力**:
  • result: 最後に受信した PGresult を格納（成功時）。
  • timed_out: タイムアウトが発生したかどうかを示すフラグ。
  • **戻り値**: 失敗（接続エラーまたはタイムアウト）した場合は true、成功した場合は false を返す。

前提条件と不変条件

  • PG_TRY ブロック内で実行され、エラー発生時には last_res が確実に解放される。
  • 処理中に CHECK_FOR_INTERRUPTS() が呼び出され、割り込みが許容される。
  • pgfdw_we_cleanup_result が未初期化の場合、初回実行時に WaitEventExtensionNew で初期化される。

アルゴリズムまたは振る舞い

1. PQisBusy が真の間、ループして入力を待機する。 2. 現在時刻が endtime を超えた場合、timed_out を設定して終了する。 3. 現在時刻が retrycanceltime を超えた場合、libpqsrv_cancel を呼び出し、再送間隔を指数的に増加させる。 4. WaitLatchOrSocket を使用して、ソケットの読み取り準備ができるまで待機する。 5. ソケットからデータが読み込める場合、PQconsumeInput で入力を消費する。 6. PQgetResult が NULL を返すまで結果を読み込み、直前の結果を PQclear で解放しながら最新の結果を保持する。 7. 終了時に failed が真であれば保持していた結果を解放し、偽であれば result に格納して呼び出し元へ渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/connection.c 内の複数箇所（行1486, 1578付近）で、クリーンアップ処理の一部として呼び出される。
  • **呼び出し先**: PQisBusy, GetCurrentTimestamp, libpqsrv_cancel, TimestampTzPlusMilliseconds, TimestampDifferenceMilliseconds, WaitLatchOrSocket, ResetLatch, CHECK_FOR_INTERRUPTS, PQconsumeInput, PQgetResult, PQclear。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: クエリ結果を保持する構造体。
  • TimestampTz: PostgreSQL のタイムスタンプ型。

失敗モードと境界条件

  • PQconsumeInput が失敗した場合、接続エラーとして failed = true を返す。
  • endtime に達した場合、timed_out = true となり、処理を中断する。
  • 割り込み（CHECK_FOR_INTERRUPTS）が発生した場合、PG_CATCH ブロックにより last_res が解放され、例外が再スローされる。

保守担当者向けメモ

  • 本関数は postgres_fdw における接続のクリーンアップ状態を管理する重要な関数である。
  • retrycanceltime の更新ロジックは、キャンセル要求の過度な発行を防ぐための指数バックオフを採用している。
  • pgfdw_we_cleanup_result はグローバルな待機イベント変数であり、競合や初期化順序に注意すること。
