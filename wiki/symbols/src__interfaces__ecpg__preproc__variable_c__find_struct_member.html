find_struct_member

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/variable.c:24

目的

構造体や共用体のメンバを再帰的に探索し、指定された名前のメンバに対応する変数オブジェクトを生成・返却する。

シグネチャまたは形状

struct variable *find_struct_member(const char *name, char *str, struct ECPGstruct_member *members, int brace_level)

入力と出力

  • **入力**:
  • name: 検索対象の変数名。
  • str: 探索対象のメンバ名を含む文字列（strpbrkで解析される）。
  • members: 探索対象の構造体メンバリスト。
  • brace_level: 変数のスコープレベル。
  • **出力**:
  • 成功時: メンバに対応する新しい variable 構造体へのポインタ。
  • 失敗時: NULL、または mmfatal によるプロセス終了。

前提条件と不変条件

  • members リストが有効であること。
  • 探索対象の文字列 str は、. や ->、[ を含む可能性がある。
  • 構造体の階層が深い場合、再帰的に呼び出される。

アルゴリズムまたは振る舞い

1. strpbrk を使用して、次に続く区切り文字（., -, [）を特定する。 2. members リストを走査し、現在の str と一致するメンバ名を探す。 3. 一致した場合:

  • 区切り文字がない場合（末尾）: メンバの型に応じて new_variable を呼び出し、結果を返す。
  • 区切り文字がある場合:
  • [（配列）の場合: インデックス部分をスキップし、次の階層へ進む。
  • . または -> の場合: メンバの型が配列か構造体かに応じて、適切なメンバリストを引数に再帰呼び出しを行う。

    • 4. 探索中に不正な形式が検出された場合、mmfatal を呼び出しエラー終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/variable.c 内の find_struct_member 自身（再帰呼び出し）。
  • 外部からの呼び出し元は、提供されたソース抜粋からは特定不可。
  • **呼び出し先**:
  • new_variable: 変数エントリの生成。
  • ECPGmake_array_type, ECPGmake_struct_type, ECPGmake_simple_type: 型定義の生成。
  • mmfatal: エラー発生時のプロセス終了。

関連データ構造

  • ECPGstruct_member: 構造体のメンバ情報を保持するリスト。
  • variable: プリプロセッサが管理する変数オブジェクト。
  • ECPGtype: 変数の型情報。

失敗モードと境界条件

  • メンバ名が見つからない場合、NULL を返す。
  • 構造体定義と一致しない不正なアクセス（例: 構造体ではない変数へのメンバアクセス）が発生した場合、mmfatal によりエラー終了する。
  • 多次元配列や複雑なポインタ構造の解決において、型定義が不整合な場合はエラーとなる。

保守担当者向けメモ

  • 本関数は再帰的であり、brace_level の整合性が重要です。
  • mmfatal が呼び出されるとプリプロセッサ全体が停止するため、構文解析の厳格なチェックポイントとして機能しています。
  • 配列のインデックス処理（[...]）は単に読み飛ばす実装となっており、インデックス値自体の検証は行われません。
