XLogWalRcvClose

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiver.c:1039

目的

現在オープンされているWALセグメントファイルをフラッシュしてクローズし、アーカイブ処理のための通知または強制完了フラグを作成する。

シグネチャまたは形状

static void XLogWalRcvClose(XLogRecPtr recptr, TimeLineID tli)

入力と出力

  • **入力**:
  • recptr: 現在のWAL書き込み位置を示す XLogRecPtr。
  • tli: 現在のタイムラインID。
  • **出力**: なし（副作用としてファイル操作およびアーカイブ通知を実行）。

前提条件と不変条件

  • recvFile が 0 以上であること。
  • recptr が現在の recvSegNo の範囲外であること（セグメント切り替え時であること）。
  • tli が 0 ではないこと。

アルゴリズムまたは振る舞い

1. XLogWalRcvFlush(false, tli) を呼び出し、現在のファイルをディスクへフラッシュする。 2. XLogFileName を使用して、現在のセグメントファイル名を生成する。 3. close(recvFile) を実行し、失敗した場合は PANIC で終了する。 4. アーカイブモードに応じて以下の処理を行う:

  • ARCHIVE_MODE_ALWAYS 以外の場合: XLogArchiveForceDone を呼び出し、.done ファイルを作成してアーカイブを抑制する。
  • ARCHIVE_MODE_ALWAYS の場合: XLogArchiveNotify を呼び出し、アーカイブを通知する。

    • 5. recvFile を -1 にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walreceiver.c 内のストリーミング処理ループ（904行目、975行目）。
  • **呼び出し先**: XLogWalRcvFlush、XLogFileName、close、XLogArchiveForceDone、XLogArchiveNotify。

関連データ構造

  • recvFile: 現在オープン中のファイルディスクリプタ。
  • recvFileTLI: 現在のタイムラインID。
  • recvSegNo: 現在のWALセグメント番号。
  • wal_segment_size: WALセグメントのサイズ。

失敗モードと境界条件

  • close() が失敗した場合、PANIC が発生しプロセスが異常終了する。
  • アーカイブモードの設定により、.done ファイルの作成または通知のいずれかが選択される。

保守担当者向けメモ

  • 本関数はセグメント切り替え時にのみ呼び出されることを想定している。
  • XLogFileClose() と異なり、リカバリプロセスが直後に読み取ることを前提としているため、OSキャッシュの解放（posix_fadvise 等）は行わない。
