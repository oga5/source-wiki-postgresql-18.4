pg_wcsformat

定義

  • 種別: function
  • 定義箇所: src/fe_utils/mbprint.c:293

目的

マルチバイト文字列を端末表示用に整形し、制御文字のエスケープやタブ展開を行いながら lineptr 構造体へ格納します。

シグネチャまたは形状

void pg_wcsformat(const unsigned char *pwcs, size_t len, int encoding, struct lineptr *lines, int count)

入力と出力

  • pwcs: 整形対象の文字列（マルチバイト対応）。
  • len: 入力文字列のバイト長。
  • encoding: 文字エンコーディングID。
  • lines: 整形後のデータと幅を格納する lineptr 構造体の配列。
  • count: lines 配列の要素数。
  • 出力: lines が指すメモリ領域に整形済み文字列が書き込まれます。

前提条件と不変条件

  • pg_wcssize で事前に計算されたサイズに基づき、十分なメモリが lines に確保されている必要があります。
  • pg_wcssize と本関数は、タブ幅やエスケープルール等のロジックが完全に一致している必要があります。

アルゴリズムまたは振る舞い

1. 入力文字列を走査し、PQmblen で文字境界を、PQdsplen で表示幅を特定します。 2. シングルバイト文字の場合:

  • \n: 改行処理を行い、次の lineptr へ移行します。
  • \r: \r という文字列に変換します。
  • \t: 8カラム境界までスペースで埋めます。
  • 制御文字 (w < 0): \xHH 形式にエスケープします。
  • その他: そのまま出力します。

    • 3. マルチバイト文字の場合:

  • 制御文字 (w < 0): UTF-8 の場合は \uXXXX 形式に変換します。
  • その他: バイト列をそのままコピーし、表示幅を加算します。

    • 4. 最後に lines 配列の終端を NULL でマークします。

呼び出し元と呼び出し先

  • 呼び出し元: src/fe_utils/print.c 内の各関数（テーブル出力処理など）。
  • 呼び出し先: PQmblen, PQdsplen（libpq）、utf8_to_unicode。

関連データ構造

  • lineptr: 整形後の文字列ポインタと表示幅を保持する構造体（src/include/fe_utils/mbprint.h で定義）。

失敗モードと境界条件

  • count を超える行数が必要になった場合、exit(1) を呼び出し異常終了します。

保守担当者向けメモ

  • pg_wcssize との同期が必須です。ロジック変更時は必ず両方の関数を確認してください。
  • 現在、マルチバイトの制御文字エスケープは UTF-8 に特化しています。他エンコーディングへの対応が必要な場合は \u???? の箇所を拡張してください。
