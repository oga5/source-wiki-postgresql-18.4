handle_args

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:145

目的

コマンドライン引数を解析し、pg_test_fsync ツールの実行パラメータ（テスト対象ファイル名、テストあたりの秒数）を設定する。

シグネチャまたは形状

static void handle_args(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc: 引数の数。
  • argv: 引数文字列の配列。
  • オプション: -f または --filename (ファイル名)、-s または --secs-per-test (秒数)。
  • **出力**: なし（副作用としてグローバル変数 filename および secs_per_test を更新し、標準出力へ設定情報を表示する）。

前提条件と不変条件

  • secs-per-test に指定される値は 1 から UINT_MAX の範囲内である必要がある。
  • 引数解析中にエラーが発生した場合、または --help / --version が指定された場合は、プログラムを終了する。

アルゴリズムまたは振る舞い

1. --help または --version が指定されているか確認し、該当する場合は情報を表示して終了する。 2. getopt_long を使用してコマンドライン引数をループ処理する。

  • -f: filename グローバル変数に値をコピーする。
  • -s: 指定された文字列を unsigned int に変換し、範囲チェックを行う。

    • 3. 未知のオプションや引数の過剰指定がある場合はエラーメッセージを表示して終了する。 4. 最後に、設定されたテスト秒数を表示し、プラットフォームごとの Direct I/O サポート状況（O_DIRECT または F_NOCACHE）を判定して出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_test_fsync/pg_test_fsync.c 内の main 関数（行110）。
  • **呼び出し先**: getopt_long, pg_strdup, strtoul, pg_log_error, pg_fatal, printf, puts, exit 等の標準および PostgreSQL ユーティリティ関数。

関連データ構造

  • struct option long_options[]: getopt_long で使用されるコマンドラインオプションの定義。

失敗モードと境界条件

  • 引数に不正な数値や範囲外の秒数が指定された場合、pg_fatal または pg_log_error を経由して終了する。
  • 引数の数が多すぎる場合、エラーを出力して終了する。

保守担当者向けメモ

  • 本関数は pg_test_fsync 固有の引数解析を行っており、pg_test_timing にも同名の関数が存在するが、実装は異なる点に注意すること。
