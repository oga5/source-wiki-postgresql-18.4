WalSndErrorCleanup

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walsender.c:343

目的

WAL 送信プロセス（walsender）においてエラーが発生した際、共有リソースの解放、状態の初期化、および必要に応じたプロセスの終了処理を行う。

シグネチャまたは形状

void WalSndErrorCleanup(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ状態の更新およびリソース解放を行う）

前提条件と不変条件

  • 本関数は、walsender プロセスがエラー状態に陥った際に呼び出されることを想定している。
  • xlogreader が有効な場合、そのファイル記述子が適切に管理されている必要がある。

アルゴリズムまたは振る舞い

1. **リソース解放**: LWLockReleaseAll で保持している全軽量ロックを解放し、ConditionVariableCancelSleep で待機状態を解除する。 2. **統計・非同期処理のクリーンアップ**: pgstat_report_wait_end および pgaio_error_cleanup を実行。 3. **WAL 読み込み終了**: xlogreader が存在し、ファイルが開かれている場合は wal_segment_close で閉じる。 4. **スロット管理**: MyReplicationSlot が存在する場合は ReplicationSlotRelease で解放し、ReplicationSlotCleanup(false) を実行。 5. **状態リセット**: replication_active を false に設定。 6. **トランザクションリソースの解放**: トランザクションブロック外であれば ReleaseAuxProcessResources(false) を実行。 7. **終了判定**: got_STOPPING または got_SIGUSR2 がセットされている場合は proc_exit(0) でプロセスを終了する。 8. **状態遷移**: 最後に WalSndSetState(WALSNDSTATE_STARTUP) を呼び出し、状態を初期状態へ戻す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c:4456
  • src/backend/replication/walsender.c:344 (自己再帰的な呼び出しではなく、定義箇所への参照)
  • **呼び出し先**:
  • LWLockReleaseAll, ConditionVariableCancelSleep, pgstat_report_wait_end, pgaio_error_cleanup, wal_segment_close, ReplicationSlotRelease, ReplicationSlotCleanup, ReleaseAuxProcessResources, proc_exit, WalSndSetState

関連データ構造

  • MyReplicationSlot: 現在のプロセスが使用しているレプリケーションスロット。
  • xlogreader: WAL 読み込みに使用されるリーダー構造体。
  • got_STOPPING, got_SIGUSR2: プロセスの終了要求を示すフラグ。

失敗モードと境界条件

  • xlogreader が NULL の場合や、ファイル記述子が既に閉じられている場合は安全にスキップされる。
  • トランザクションが進行中の場合、ReleaseAuxProcessResources はスキップされる。

保守担当者向けメモ

  • 本関数は walsender のエラーハンドリングの要であり、論理レプリケーションの slotsync.c など、他のコンポーネントでも同様のクリーンアップロジックの参考として言及されている。
  • 状態遷移の順序（特に proc_exit のタイミング）は、プロセス終了時の整合性に直結するため慎重に扱うこと。
