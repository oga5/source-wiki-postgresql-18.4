GetDefaultTablespace

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tablespace.c:1142

目的

現在のデフォルトテーブルスペースの OID を取得します。

シグネチャまたは形状

Oid GetDefaultTablespace(char relpersistence, bool partitioned)

入力と出力

  • **入力**:
  • relpersistence: リレーションの永続性（RELPERSISTENCE_TEMP 等）。
  • partitioned: パーティションテーブルかどうかを示すフラグ。
  • **出力**:
  • 成功時: 該当するテーブルスペースの Oid。
  • 該当なし/デフォルト設定時: InvalidOid。

前提条件と不変条件

  • relpersistence が RELPERSISTENCE_TEMP の場合、一時テーブルスペースの準備と取得が優先されます。
  • default_tablespace GUC 変数が空文字列または NULL の場合、常に InvalidOid を返します。

アルゴリズムまたは振る舞い

1. 一時テーブルの場合、PrepareTempTablespaces() を呼び出し、GetNextTempTableSpace() の結果を返します。 2. 一時テーブル以外の場合、default_tablespace GUC 変数の値を get_tablespace_oid() で検索します。 3. 検索結果が現在のデータベースのデフォルトテーブルスペース（MyDatabaseTableSpace）と一致する場合:

  • partitioned が真であればエラー（ERRCODE_FEATURE_NOT_SUPPORTED）を報告します。
  • それ以外の場合は InvalidOid を返します。

    • 4. 検索されたテーブルスペースが存在しない場合、エラーにはせず InvalidOid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/indexcmds.c:790
  • src/backend/commands/matview.c:305
  • src/backend/commands/tablecmds.c:905
  • **呼び出し先**:
  • PrepareTempTablespaces()
  • GetNextTempTableSpace()
  • get_tablespace_oid()

関連データ構造

  • Oid (戻り値型)
  • default_tablespace (GUC 変数)
  • MyDatabaseTableSpace (グローバル変数)

失敗モードと境界条件

  • default_tablespace に指定された名前のテーブルスペースが存在しない場合、エラーを発生させず InvalidOid を返します。
  • パーティションテーブル作成時に default_tablespace がデータベースのデフォルトと一致する場合、機能制限としてエラーを投げます。

保守担当者向けメモ

  • この関数はテーブルスペースのルックアップ結果をキャッシュしません。これは、GUC 設定後にテーブルスペースが削除されたケースを正しく扱うためです。
