ReadEndOfStreamingResult

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:697

目的

ストリーミング終了時にサーバーから返される結果セットを解析し、次のタイムラインIDと、そのタイムラインが開始されるWAL位置（LSN）を抽出します。

シグネチャまたは形状

static bool ReadEndOfStreamingResult(PGresult *res, XLogRecPtr *startpos, uint32 *timeline)

入力と出力

  • **入力**:
  • res: サーバーから受け取った PGresult ポインタ。1行2列のデータ（next_tli と next_tli_startpos）を含むことが期待されます。
  • startpos: 解析したWAL開始位置を格納する XLogRecPtr へのポインタ。
  • timeline: 解析したタイムラインIDを格納する uint32 へのポインタ。
  • **出力**:
  • 成功時に true、解析失敗やフォーマット不一致時に false を返します。

前提条件と不変条件

  • res は PQnfields(res) == 2 かつ PQntuples(res) == 1 である必要があります。
  • next_tli_startpos は "%X/%X" 形式の文字列である必要があります。

アルゴリズムまたは振る舞い

1. PQnfields と PQntuples を使用して、結果セットが期待される構造（1行2列）であるか検証します。不一致の場合はエラーログを出力し false を返します。 2. 1列目の値を atoi で timeline に格納します。 3. 2列目の値を sscanf で "%X/%X" 形式として解析し、上位32ビットと下位32ビットを結合して startpos に格納します。 4. 解析に成功した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/receivelog.c:617 にて呼び出されています。
  • **呼び出し先**: libpq の関数群（PQnfields, PQntuples, PQgetvalue）および標準ライブラリ（atoi, sscanf）を呼び出します。

関連データ構造

  • PGresult: libpq が提供するクエリ結果保持構造体。
  • XLogRecPtr: WAL上の位置を保持する64ビット整数型。

失敗モードと境界条件

  • 結果セットの行数や列数が期待値と異なる場合、エラーログを出力して false を返します。
  • next_tli_startpos の文字列が "%X/%X" 形式で解析できない場合、エラーログを出力して false を返します。

保守担当者向けメモ

  • 本関数はタイムライン切り替え時のメタデータ取得に特化しています。将来的にサーバー側のプロトコルで結果セットの形式が変更された場合、この解析ロジックの更新が必要です。
