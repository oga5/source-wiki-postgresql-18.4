CMPTRGM_SIGNED

定義

  • 種別: static inline function
  • 定義箇所: contrib/pg_trgm/trgm_op.c:194

目的

トリグラム（3文字の連続）を signed char として比較するための比較関数です。システム環境の文字符号の扱いに応じて、CMPTRGM 関数ポインタ経由で呼び出される比較ロジックの一部として機能します。

シグネチャまたは形状

int CMPTRGM_SIGNED(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つのトリグラムへのポインタ a および b。
  • **出力**: 比較結果を示す整数。a < b なら負、a > b なら正、a == b なら 0 を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、少なくとも3バイト以上のメモリ領域を指している必要があります。
  • 内部で signed char としてキャストして比較を行うため、プラットフォームの char 型が符号付きであるかどうかに依存せず、一貫した符号付き比較を提供します。

アルゴリズムまたは振る舞い

1. 内部マクロ CMPPCHAR_S を使用し、トリグラムの各バイト（0番目、1番目、2番目）を順次比較します。 2. 0番目のバイトが等しくない場合はその結果を返し、等しい場合は1番目のバイトを比較します。 3. 1番目のバイトも等しい場合は、最後に2番目のバイトを比較した結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_op.c 内の CMPTRGM_CHOOSE 関数により、システム設定に基づいて CMPTRGM 関数ポインタに代入されます。
  • **呼び出し先**: CMPCHAR マクロ（trgm.h で定義されていると推測される）を呼び出します。

関連データ構造

  • TRGM: トリグラムを保持する型。

失敗モードと境界条件

  • 入力ポインタが NULL または3バイト未満の領域を指している場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本関数は CMPTRGM_UNSIGNED と対をなします。比較ロジックを変更する場合は、両方の関数に一貫性を持たせる必要があります。
  • 直接呼び出すのではなく、必ず CMPTRGM 関数ポインタを経由して使用してください。
