gbt_oidlt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_oid.c:48

目的

Oid 型の値同士を比較し、第一引数が第二引数より小さいかどうかを判定する。

シグネチャまたは形状

static bool gbt_oidlt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の Oid 型値へのポインタ。
  • b: 比較対象の Oid 型値へのポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a < *b が真であれば true、それ以外は false を返す。

前提条件と不変条件

  • a および b は、有効な Oid 型の値を示すポインタである必要がある。

アルゴリズムまたは振る舞い

1. a が指す値を Oid 型としてキャストし、デリファレンスする。 2. b が指す値を Oid 型としてキャストし、デリファレンスする。 3. 両者の値を比較演算子 < で比較し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_oid.c 内の静的配列（tinfo 構造体初期化時など）から参照されている。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子型。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は Oid 型専用の比較ロジックであり、GiSTインデックスのサポート関数として利用される。
  • btree_utils_num.c で定義される汎用的な数値比較ロジックの一部として機能する。
