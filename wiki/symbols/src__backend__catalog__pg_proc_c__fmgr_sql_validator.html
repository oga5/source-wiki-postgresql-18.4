fmgr_sql_validator

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_proc.c:831

目的

SQL言語で定義された関数の構文解析、型チェック、および整合性検証を行うバリデータです。CREATE FUNCTION 実行時や関数定義の更新時に、関数本体が正当であるかを検証します。

シグネチャまたは形状

Datum fmgr_sql_validator(PG_FUNCTION_ARGS)
  • PG_FUNCTION_ARGS を介して対象関数の OID を受け取ります。

入力と出力

  • **入力**: fcinfo（関数呼び出し情報）。内部で PG_GETARG_OID(0) を使用して検証対象の関数 OID を取得します。
  • **出力**: PG_RETURN_VOID()。検証成功時は正常終了し、失敗時は ereport(ERROR, ...) により例外をスローします。

前提条件と不変条件

  • CheckFunctionValidatorAccess によるアクセス権限チェックを通過する必要があります。
  • pg_proc システムカタログ上に該当する OID が存在しなければなりません。
  • 戻り値型および引数型に、RECORD、VOID、または多相型（polymorphic）以外の擬似型（pseudotype）が含まれている場合、エラーとなります。

アルゴリズムまたは振る舞い

1. **初期チェック**: 対象関数のメタデータを SearchSysCache1 で取得し、引数および戻り値の型が SQL 関数として許容されるか確認します。 2. **バリデーションの制御**: グローバル変数 check_function_bodies が真の場合のみ、関数本体の解析を実行します。 3. **解析処理**:

  • prosqlbody 属性が存在する場合は、格納されたノードツリーを再構築し、pg_rewrite_query で書き換えルールを適用します。
  • prosqlbody がない場合は、prosrc 文字列を pg_parse_query で構文解析します。
  • 多相型引数がない場合は、pg_analyze_and_rewrite_withcb を用いて完全な意味解析と書き換えを行い、check_sql_fn_statements および check_sql_fn_retval で戻り値の整合性を検証します。

    • 4. **エラーハンドリング**: 解析中に発生したエラーは sql_function_parse_error_callback を通じて、元の SQL 文のコンテキストにマッピングされます。

呼び出し元と呼び出し先

  • **呼び出し元**: ProcedureCreate（src/backend/catalog/pg_proc.c）から間接的に呼び出されることが示唆されています。
  • **呼び出し先**: CheckFunctionValidatorAccess, SearchSysCache1, pg_parse_query, pg_analyze_and_rewrite_withcb, check_sql_fn_statements, check_sql_fn_retval 等。

関連データ構造

  • Form_pg_proc: pg_proc カタログの行構造体。
  • parse_error_callback_arg: エラー発生時のコンテキスト保持用構造体。
  • ErrorContextCallback: エラーハンドリングのスタック管理用。

失敗モードと境界条件

  • **キャッシュルックアップ失敗**: 指定された OID が pg_proc に存在しない場合、elog(ERROR) で停止します。
  • **型制約違反**: 許可されていない擬似型が使用されている場合、ERRCODE_INVALID_FUNCTION_DEFINITION でエラーとなります。
  • **構文・意味エラー**: SQL 文法違反や型不整合がある場合、解析フェーズでエラーが報告されます。

保守担当者向けメモ

  • 多相型引数を持つ関数は、実行時まで型が確定しないため、バリデーション時の完全な型チェックがスキップされます。
  • prosqlbody の有無により解析パスが分岐するため、修正時は両方のパスの整合性に注意してください。
  • エラー位置の変換ロジック（function_parse_error_transpose 関連）は複雑であり、デバッグ時は error_context_stack の状態を確認してください。
