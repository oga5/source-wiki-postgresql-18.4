pg_sha512_update

定義

  • 種別: function
  • 定義箇所: src/common/sha2.c:801

目的

SHA-512 ハッシュ計算において、入力データストリームをコンテキストに供給し、内部状態を逐次更新する。

シグネチャまたは形状

void pg_sha512_update(pg_sha512_ctx *context, const uint8 *data, size_t len)

入力と出力

  • context: 更新対象の pg_sha512_ctx 構造体へのポインタ。
  • data: ハッシュ計算対象の入力データバイト列。
  • len: 入力データの長さ（バイト単位）。
  • 出力: なし（context 内部の状態が更新される）。

前提条件と不変条件

  • len が 0 の場合、関数は何もせず即座に復帰する。
  • context->bitcount は、処理された総ビット数を保持し、ADDINC128 マクロによって適切にインクリメントされる。
  • コンテキスト内のバッファは PG_SHA512_BLOCK_LENGTH（128バイト）の境界で処理される。

アルゴリズムまたは振る舞い

1. **バッファの埋め合わせ**: コンテキスト内の既存バッファに空きがある場合、入力データでバッファを埋める。バッファが満杯になった時点で SHA512_Transform を呼び出し、中間ハッシュ値を更新する。 2. **ブロック単位の処理**: 入力データが PG_SHA512_BLOCK_LENGTH を超える場合、残りのデータをブロック単位で SHA512_Transform に渡し、効率的に処理する。 3. **残余データの保持**: 処理しきれなかった端数のデータは、次回の update または final 呼び出しのために context->buffer にコピーされる。 4. **状態更新**: 各ステップで bitcount を更新し、データ長を追跡する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/common/cryptohash.c:159
  • src/common/sha2.c:946
  • 呼び出し先:
  • SHA512_Transform (ブロック単位の圧縮関数)
  • ADDINC128 (内部マクロ)
  • memcpy (標準ライブラリ)

関連データ構造

  • pg_sha512_ctx: SHA-512 の計算状態（ハッシュ値、ビットカウンタ、バッファ）を保持する構造体。sha2_int.h で定義。

失敗モードと境界条件

  • len == 0: 正常に無視される。
  • 入力データがブロック境界に満たない場合: バッファに蓄積され、即座に処理は行われない。
  • メモリ境界: memcpy を使用しているため、data ポインタが有効なメモリ領域を指している必要がある。

保守担当者向けメモ

  • 本関数は Aaron D. Gifford 氏の実装に基づいている。
  • SHA2_UNROLL_TRANSFORM が定義されている場合、変換ループの最適化が行われる可能性がある。
  • エンディアン変換が必要な環境では、REVERSE64 等のマクロが SHA512_Transform 内で利用される。
