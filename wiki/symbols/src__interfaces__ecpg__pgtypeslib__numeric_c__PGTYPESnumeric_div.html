PGTYPESnumeric_div

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/numeric.c:1052

目的

2つの numeric 型変数の除算を行い、結果を第3引数に格納する。

シグネチャまたは形状

int PGTYPESnumeric_div(numeric *var1, numeric *var2, numeric *result)

入力と出力

  • **入力**:
  • var1: 被除数（numeric 構造体へのポインタ）
  • var2: 除数（numeric 構造体へのポインタ）
  • **出力**:
  • result: 計算結果を格納する numeric 構造体へのポインタ
  • 戻り値: 成功時に 0、失敗時に -1 を返す。

前提条件と不変条件

  • var2 がゼロの場合、errno に PGTYPES_NUM_DIVIDE_ZERO を設定し -1 を返す。
  • result が指す構造体のバッファは、計算成功時に内部で再割り当てされる。
  • select_div_scale を用いて除算の精度（rscale）が自動決定される。

アルゴリズムまたは振る舞い

1. 除数 var2 がゼロでないか確認する。 2. 結果の符号、重み、桁数を算出し、result 用のバッファを確保する。 3. 被除数 var1 と除数 var2 のコピーを作成し、内部演算用の形式に変換する。 4. 筆算アルゴリズムに基づき、各桁の商を guess（0〜9）として推定する。 5. 推定した商に対応する除数の倍数を divisor[guess] としてキャッシュし、cmp_abs で比較しながら減算を繰り返す。 6. 最終的な商を result の digits 配列に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:

関連データ構造

  • numeric: 数値の符号、重み、桁数、バッファを保持する構造体。
  • NumericDigit: 数値の各桁を保持する型。

失敗モードと境界条件

  • **ゼロ除算**: var2 の桁数が0の場合、エラーとして処理される。
  • **メモリ不足**: digitbuf_alloc が NULL を返した場合、処理を中断し -1 を返す。
  • **精度**: res_ndigits が 0 以下になる場合は 1 桁として計算される。

保守担当者向けメモ

  • result のバッファは digitbuf_free で解放後に再割り当てされるため、呼び出し元で result が指すメモリのライフサイクルに注意すること。
  • 内部で divisor 配列（サイズ10）を使用して商の候補をキャッシュしている。
