Tuplestorestate

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/tuplestore.c:103

目的

ソートを伴わない一時的なタプル格納のための状態管理構造体。メモリ制限内でのタプル保持と、制限超過時のディスク（BufFile）への退避を自動的に制御する。

シグネチャまたは形状

  • status: 現在の格納状態（TupStoreStatus）。
  • memtuples: メモリ内タプルポインタ配列。
  • readptrs: 読み取り位置を管理する TSReadPointer 配列。
  • myfile: 一時ファイルへのポインタ（BufFile）。
  • copytup, writetup, readtup: タプル操作のための関数ポインタ。
  • その他、メモリ使用量（availMem, allowedMem）、タプル数、ファイル位置情報などを保持。

入力と出力

  • 入力: tuplestore_begin_heap 等の初期化関数により生成され、tuplestore_puttupleslot を通じてタプルを受け取る。
  • 出力: tuplestore_gettupleslot を通じてメモリまたはディスクからタプルを返す。

前提条件と不変条件

  • copytup でコピーされるタプルは、単一の palloc チャンクで表現される「フラット」な形式である必要がある。
  • availMem は、メモリに格納されたタプルの実使用量と同期して増減されなければならない。
  • memtuples 配列は TSS_INMEM 状態でのみ使用される。

アルゴリズムまたは振る舞い

1. **メモリ格納**: 初期状態ではタプルを memtuples 配列に保持する。 2. **ディスク退避**: メモリ制限（allowedMem）を超過すると、全タプルを myfile へ書き出し、TSS_WRITEFILE 状態へ移行する。 3. **読み取り**: 読み取りポインタごとに独立した位置を管理し、ディスク状態の場合は BufFileSeek を使用して対象位置へ移動する。 4. **トリミング**: tuplestore_trim により、不要になった古いタプルをメモリから解放する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）: contrib/amcheck/verify_heapam.c, contrib/dblink/dblink.c, contrib/tablefunc/tablefunc.c 等の拡張モジュールで利用されている。
  • 呼び出し先: BufFile API（storage/buffile.h）およびメモリ管理関数（utils/memutils.h）を呼び出す。

関連データ構造

  • TupStoreStatus: 格納状態の列挙型。
  • TSReadPointer: 読み取り位置を管理する構造体。
  • BufFile: 一時ファイル操作用構造体。

失敗モードと境界条件

  • メモリ不足: allowedMem を超えるとディスクへ退避するが、ディスク容量が不足した場合はエラーとなる。
  • tuplestore_trim 実行時: 呼び出し元が保持しているタプルポインタが無効化される可能性がある。

保守担当者向けメモ

  • 本構造体は tuplesort.c の設計を参考にしている。将来的にヒープタプル以外のオブジェクトを扱う場合、関数ポインタ（copytup 等）を適切に設定することで拡張可能である。
  • memtupdeleted が正の値の場合、memtuples の先頭部分は無効なポインタであり、アクセスしないよう注意が必要。
