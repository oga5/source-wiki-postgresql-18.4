mac8KEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_macaddr8.c:12

目的

PostgreSQL の macaddr8 型の値を GiST インデックス内で範囲（下限・上限）として保持し、B-tree ライクな検索を可能にするためのデータ構造です。

シグネチャまたは形状

typedef struct {
    macaddr8 lower;
    macaddr8 upper;
} mac8KEY;

※構造体のサイズは sizeof(gbtreekey16) と等しくなるよう設計されています。

入力と出力

  • 入力: GiST インデックスの構築時や検索時に、macaddr8 型のデータが本構造体の lower および upper フィールドに格納されます。
  • 出力: インデックスの比較関数やペナルティ計算関数において、範囲の境界値として参照されます。

前提条件と不変条件

  • lower は範囲の下限、upper は範囲の上限を表します。
  • インデックスの整合性を保つため、lower は upper 以下である必要があります。

アルゴリズムまたは振る舞い

  • GiST インデックスの圧縮処理（gbt_macad8_compress）において、データは mac8KEY 形式に変換されます。
  • 検索時には gbt_macad8_consistent がこの構造体を読み取り、クエリ条件との一致判定を行います。
  • インデックスのバランス調整（gbt_macad8_penalty）では、本構造体の値を uint64 に変換して距離計算が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/btree_gist/btree_macaddr8.c 内の各 GiST サポート関数（gbt_macad8_compress, gbt_macad8_consistent, gbt_macad8_union, gbt_macad8_penalty 等）。
  • 呼び出し先: 直接的な関数呼び出しではなく、データ構造として各サポート関数から参照されます。

関連データ構造

  • macaddr8: PostgreSQL のネットワークアドレス型。
  • gbtreekey16: 本構造体がサイズを合わせる対象となる GiST 共通のキー構造体。

失敗モードと境界条件

  • lower と upper が逆転した場合、インデックスの検索結果が不正になる可能性があります。
  • macaddr8 から uint64 への変換時にオーバーフローや精度欠損が発生しないよう、mac8_2_uint64 関数による適切な変換が前提となります。

保守担当者向けメモ

  • 本構造体は btree_utils_num.c で提供される汎用数値処理関数と密接に連携しています。挙動の変更が必要な場合は、まず btree_utils_num.c のロジックを確認してください。
  • 構造体のサイズ変更を行う際は、gbtreekey16 との整合性に注意してください。
