GinBufferKeyEquals

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gininsert.c:1339

目的

GinBuffer に保持されているキーと、対象の GinTuple が持つキーが等しいかどうかを判定します。

シグネチャまたは形状

static bool GinBufferKeyEquals(GinBuffer *buffer, GinTuple *tup)

入力と出力

  • **入力**:
  • buffer: 比較対象のキー情報を持つ GinBuffer 構造体へのポインタ。
  • tup: 比較対象の GinTuple 構造体へのポインタ。
  • **出力**:
  • bool: 両者のキーが一致する場合は true、それ以外は false を返します。

前提条件と不変条件

  • AssertCheckGinBuffer(buffer) により、バッファの状態が妥当であることが検証されます。
  • tup と buffer の属性番号、型情報（typbyval, typlen）が一致している必要があります。

アルゴリズムまたは振る舞い

1. 属性番号が一致しない場合、直ちに false を返します。 2. カテゴリ（GIN_CAT_NORM_KEY 等）が一致しない場合、直ちに false を返します。 3. カテゴリが GIN_CAT_NORM_KEY 以外（NULLや空キー）であれば、カテゴリの一致をもって等価とみなします。 4. GIN_CAT_NORM_KEY の場合、buffer->typbyval に応じて tup からキー値を抽出し、ApplySortComparator を用いて比較を行います。比較結果が 0 であれば等価と判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gininsert.c 内のコードから参照されています（行 1330, 1625）。
  • **呼び出し先**: ApplySortComparator を呼び出します。

関連データ構造

  • GinBuffer: 比較の基準となるキーや型情報、比較関数（ssup）を保持します。
  • GinTuple: 比較対象のタプルデータおよびメタデータを保持します。

失敗モードと境界条件

  • 属性番号やカテゴリの不一致は、比較を行わずに false を返します。
  • typbyval が真の場合、tup->data を Datum として直接キャストして読み取ります。

保守担当者向けメモ

  • 本関数は GinBuffer 内の ssup（ソートサポート情報）を利用して比較を行います。型定義の変更や比較ロジックの拡張時には、GinBuffer の初期化処理との整合性に注意してください。
