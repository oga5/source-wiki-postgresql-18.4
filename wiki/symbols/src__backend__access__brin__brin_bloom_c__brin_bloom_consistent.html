brin_bloom_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_bloom.c:592

目的

BRINインデックスのページ範囲が、指定されたスキャンキー（検索条件）と整合するか（検索対象となり得るか）を判定する。

シグネチャまたは形状

Datum brin_bloom_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): BrinDesc *（インデックス記述子）
  • PG_GETARG_POINTER(1): BrinValues *（ページ範囲の要約データ）
  • PG_GETARG_POINTER(2): ScanKey *（検索条件の配列）
  • PG_GETARG_INT32(3): int（検索条件の数）
  • **出力**:
  • bool: ページ範囲が検索条件を満たす可能性がある場合は true、確実に含まれない場合は false を返す。

前提条件と不変条件

  • column->bv_values[0] に格納されたブルームフィルタはデトースト（PG_DETOAST_DATUM）可能であること。
  • スキャンキーに SK_ISNULL フラグが含まれていないこと（bringetbitmap 側で処理済みである前提）。
  • 検索戦略として BloomEqualStrategyNumber のみがサポートされている。

アルゴリズムまたは振る舞い

1. ページ範囲の要約からブルームフィルタを取得する。 2. すべてのスキャンキーについて以下のループを実行する：

  • 検索条件の戦略が BloomEqualStrategyNumber であることを確認する。
  • 検索値に対して型固有のハッシュ関数（PROCNUM_HASH）を適用し、uint32 のハッシュ値を生成する。
  • bloom_contains_value を呼び出し、フィルタ内にハッシュ値が存在するか判定する。
  • 判定結果が false であれば、そのページ範囲は検索対象外としてループを終了する。

    • 3. すべてのキーで true と判定された場合のみ、最終的に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース excerpt 内には明示的な呼び出し元の記述なし。
  • **呼び出し先**:
  • bloom_get_procinfo: ハッシュ関数の取得。
  • FunctionCall1Coll: ハッシュ関数の実行。
  • bloom_contains_value: ブルームフィルタによる包含判定。

関連データ構造

  • BrinDesc: インデックスのメタデータ。
  • BrinValues: ページ範囲の要約値。
  • BloomFilter: 包含判定に使用されるブルームフィルタ構造体。
  • ScanKey: 検索条件を保持する構造体。

失敗モードと境界条件

  • **エラー**: BloomEqualStrategyNumber 以外の戦略が渡された場合、elog(ERROR) により処理が中断される。
  • **境界条件**: フィルタ内のビットがすべて立っている場合、誤判定（False Positive）により実際には値が含まれていなくても true を返す可能性がある。

保守担当者向けメモ

  • 本関数は「Less Hashing, Same Performance」論文に基づくハッシュ戦略を採用している。
  • 検索条件の追加や変更を行う際は、switch 文の戦略判定ロジックを拡張する必要がある。
