HeapTupleFreeze

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam.h:139

目的

VACUUM 処理において、ヒープページ内のタプルを凍結（Freeze）するための計画および状態を保持する構造体です。トランザクション ID の周回問題を防ぐため、古いタプルの情報を更新する際に使用されます。

シグネチャまたは形状

typedef struct HeapTupleFreeze
{
    TransactionId xmax;
    uint16        t_infomask2;
    uint16        t_infomask;
    uint8         frzflags;
    uint8         checkflags;
    OffsetNumber  offset;
} HeapTupleFreeze;

入力と出力

本構造体はデータ保持用であり、関数ではありません。VACUUM 処理中に HeapTupleFreeze の配列が構築され、ページ内の各タプルに対する凍結操作の指示として利用されます。

前提条件と不変条件

  • offset は、対象となるタプルが格納されているページ内の有効な OffsetNumber を指している必要があります。
  • 凍結処理の実行前には、対象タプルの現在の状態（xmax や infomask）が適切に評価されている必要があります。

アルゴリズムまたは振る舞い

1. VACUUM プロセスがページ内のタプルをスキャンし、凍結が必要なタプルを特定します。 2. 特定されたタプルごとに HeapTupleFreeze 構造体を生成し、新しい xmax や infomask の値を設定します。 3. 構築された計画（配列）を heap_freeze_prepared_tuples 等の関数に渡し、実際のページ上のタプルに対して変更を適用します。 4. 変更内容は WAL に記録され、永続化されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/pruneheap.c および src/backend/access/heap/heapam.c 内の VACUUM 関連ロジック（heap_page_prune_and_freeze など）から、凍結計画の作成および実行時に参照されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • HeapPageFreeze: ページ全体の凍結状態を管理する構造体。
  • PruneFreezeResult: 剪定および凍結処理の結果を保持する構造体。
  • Buffer: タプルが格納されているバッファ。

失敗モードと境界条件

  • MaxHeapTuplesPerPage を超える数のタプルを一度に凍結しようとした場合、配列のオーバーフローが発生する可能性があります。
  • 凍結対象のタプルが既に削除されている、あるいは無効なオフセットを指している場合、整合性エラーを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/access/heap/pruneheap.c におけるページ剪定ロジックと密接に結合しています。変更を加える際は、heap_log_freeze_eq 等の WAL 記録ロジックへの影響を確認してください。
