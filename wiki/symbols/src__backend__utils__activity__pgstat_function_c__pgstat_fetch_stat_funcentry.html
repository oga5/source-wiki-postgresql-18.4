pgstat_fetch_stat_funcentry

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_function.c:238

目的

指定された関数OIDに対応する統計エントリを取得します。

シグネチャまたは形状

PgStat_StatFuncEntry *pgstat_fetch_stat_funcentry(Oid func_id)

入力と出力

  • **入力**: func_id (対象関数のOID)
  • **出力**: 該当する関数の統計情報を含む PgStat_StatFuncEntry 構造体へのポインタ。見つからない場合は NULL を返します。

前提条件と不変条件

  • 統計サブシステムが初期化されている必要があります。
  • MyDatabaseId が有効なデータベースコンテキストを指している必要があります。

アルゴリズムまたは振る舞い

1. 統計サブシステムの汎用関数 pgstat_fetch_entry を呼び出します。 2. その際、統計種別として PGSTAT_KIND_FUNCTION を指定し、現在のデータベースID (MyDatabaseId) と引数の func_id を渡します。 3. 取得した結果を PgStat_StatFuncEntry * 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c (統計情報表示用関数内)
  • **呼び出し先**:
  • pgstat_fetch_entry (統計サブシステムの汎用取得関数)

関連データ構造

  • PgStat_StatFuncEntry: 関数統計を保持する構造体。
  • Oid: 関数を一意に識別するオブジェクトID。

失敗モードと境界条件

  • 指定した func_id に対応する統計エントリが存在しない場合、NULL を返します。
  • 統計収集が無効な場合や、対象関数が一度も実行されていない場合はエントリが存在しない可能性があります。

保守担当者向けメモ

  • 本関数は pgstat_fetch_entry のラッパーです。統計情報の取得ロジックに変更がある場合は、pgstat_fetch_entry の仕様を確認してください。
