astreamer_tar_archiver

定義

  • 種別: struct
  • 定義箇所: src/fe_utils/astreamer_tar.c:39

目的

tarアーカイブの生成および修正を行うためのストリーマー状態を保持する構造体です。

シグネチャまたは形状

typedef struct astreamer_tar_archiver
{
    astreamer   base;
    bool        rearchive_member;
} astreamer_tar_archiver;

入力と出力

  • 入力: astreamer インターフェースを介した構造化チャンク。
  • 出力: astreamer インターフェースを介した、ヘッダーやパディングが補完・再構築されたtarデータ。

前提条件と不変条件

  • base メンバにより astreamer インターフェースと互換性を持つ必要があります。
  • rearchive_member は、メンバーの再アーカイブが必要かどうかを制御するフラグとして機能します。

アルゴリズムまたは振る舞い

  • astreamer_tar_archiver_content を通じてチャンクを受け取ります。
  • ヘッダーが空（サイズ0）の場合、tarCreateHeader を使用してヘッダーを再構築し、後続のストリーマーへ転送します。
  • 必要に応じてパディングの調整を行い、有効なtar形式を維持します。

呼び出し元と呼び出し先

  • 呼び出し元: astreamer_tar_archiver_new（インスタンス生成時）、astreamer_tar_archiver_content（データ処理時）、astreamer_tar_archiver_finalize（終了処理時）、astreamer_tar_archiver_free（メモリ解放時）。
  • 呼び出し先: astreamer インターフェースで定義された後続のストリーマー。

関連データ構造

  • astreamer: 本構造体が継承する基底ストリーマー型。

失敗モードと境界条件

  • pax 拡張フォーマットはサポートされておらず、検出時にエラーとなります。
  • 不正なtarヘッダーが渡された場合、astreamer_tar_header 内の検証により pg_fatal が呼び出され、プロセスが終了します。

保守担当者向けメモ

  • 本構造体は palloc0 を使用してメモリ確保されます。
  • サーバーが生成する不完全なtarアーカイブ（終端のNULブロック欠落など）を補完するパイプラインの一部として設計されています。
