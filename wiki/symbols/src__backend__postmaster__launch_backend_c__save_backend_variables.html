save_backend_variables

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/launch_backend.c:729

目的

EXEC_BACKEND モードにおいて、親プロセス（Postmaster）から子プロセスへ引き継ぐべき実行環境や共有メモリの状態を BackendParameters 構造体にシリアライズします。

シグネチャまたは形状

static bool save_backend_variables(BackendParameters *param,
                                   int child_slot, 
                                   ClientSocket *client_sock,
#ifdef WIN32
                                   HANDLE childProcess, 
                                   pid_t childPid,
#endif
                                   const void *startup_data, 
                                   size_t startup_data_len);

入力と出力

  • **入力**:
  • param: 状態を格納する BackendParameters 構造体へのポインタ。
  • child_slot: 子プロセスのスロット番号。
  • client_sock: クライアント接続用ソケット情報。
  • childProcess/childPid: (Windowsのみ) 子プロセスのハンドルとPID。
  • startup_data/startup_data_len: 子プロセスへ渡す追加の初期化データ。
  • **出力**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • param は呼び出し元で確保された有効なメモリ領域である必要があります。
  • EXEC_BACKEND モードが有効な環境でのみ使用されることを想定しています。

アルゴリズムまたは振る舞い

1. **ソケット情報の保存**: client_sock を param にコピーし、write_inheritable_socket を用いて子プロセスへ継承可能な形式に変換します。 2. **環境変数の保存**: データディレクトリ、共有メモリセグメント情報、ロック、プロセスのグローバル状態（ProcGlobal, PMSignalState 等）を param にコピーします。 3. **プラットフォーム固有処理**:

  • Windows: write_duplicated_handle を使用してシグナルリスナーパイプを複製します。
  • Unix: postmaster_alive_fds をコピーします。

    • 4. **メタデータの保存**: 実行パス、パッケージライブラリパス、および startup_data をコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c 内のプロセス起動処理（156, 328, 499 行付近）。
  • **呼び出し先**: write_inheritable_socket、write_duplicated_handle（Windowsのみ）、pgwin32_create_signal_listener（Windowsのみ）。

関連データ構造

  • BackendParameters: プロセス間で受け渡される状態情報のコンテナ。
  • ClientSocket: クライアント接続を管理する構造体。

失敗モードと境界条件

  • write_inheritable_socket または write_duplicated_handle が失敗した場合、関数は false を返し、呼び出し元でエラーログが出力されます。
  • startup_data_len が param->startup_data の許容サイズを超える場合の挙動については、本ソース excerpt からは明示されていません。

保守担当者向けメモ

  • 本関数は EXEC_BACKEND モードにおけるプロセス間通信の要です。新しいグローバル変数を追加した際は、本関数で BackendParameters に適切にコピーされているか確認してください。
  • Windows環境でのハンドル複製処理は特に慎重な管理が必要です。
