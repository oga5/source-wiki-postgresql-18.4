SimpleOidList

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/simple_list.h:26

目的

PostgreSQL のフロントエンドツール（pg_dump 等）において、OID（Object Identifier）のリストを管理するための軽量な単方向連結リスト構造を提供します。

シグネチャまたは形状

typedef struct SimpleOidList
{
    SimpleOidListCell *head;
    SimpleOidListCell *tail;
} SimpleOidList;

入力と出力

  • **入力**: リストの先頭（head）および末尾（tail）を保持する管理構造体です。
  • **出力**: 連結リストの末尾への高速な追加や、リスト全体の走査を可能にするためのポインタ群を保持します。

前提条件と不変条件

  • リストが空の場合、head および tail は NULL である必要があります。
  • pg_malloc を使用して動的に確保されるため、使用後は対応する破棄関数によるメモリ解放が必須です。

アルゴリズムまたは振る舞い

  • **追加**: リストの末尾に新しいセルを連結し、tail ポインタを更新します。
  • **検索**: リストを線形探索し、指定された OID が存在するかを確認します。
  • **破棄**: リストを走査し、各セルを pg_free で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c において、schema_include_oids、schema_exclude_oids、table_include_oids 等の静的変数として定義・利用されています。
  • **呼び出し先**: 構造体内部で SimpleOidListCell を参照しています。

関連データ構造

  • SimpleOidListCell: リストの各要素を構成するセル構造体。

失敗モードと境界条件

  • メモリ不足時には pg_malloc がエラーを発生させます。
  • 空のリストに対して検索や破棄を行う際は、head が NULL であることを適切に処理する必要があります。

保守担当者向けメモ

  • バックエンドの複雑な List 構造体とは異なり、機能が最小限に絞られています。複雑な操作が必要な場合は別のデータ構造を検討してください。
  • pg_dump 等のフロントエンドコードで頻繁に使用されるため、変更時は影響範囲に注意してください。
