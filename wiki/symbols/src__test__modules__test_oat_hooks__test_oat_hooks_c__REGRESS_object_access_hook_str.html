REGRESS_object_access_hook_str

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_oat_hooks/test_oat_hooks.c:275

目的

PostgreSQLのオブジェクトアクセスフック機構をテストするための関数であり、特定のオブジェクト操作（主にパラメータ設定やALTER SYSTEM）に対する権限チェックの介入と監査ログの出力を行います。

シグネチャまたは形状

static void REGRESS_object_access_hook_str(ObjectAccessType access, Oid classId, const char *objName, int subId, void *arg)

入力と出力

  • **入力**:
  • access: 操作の種類（ObjectAccessType）。
  • classId: 操作対象のカタログOID。
  • objName: 操作対象のオブジェクト名。
  • subId: 操作のサブタイプ（ACLフラグ等）。
  • arg: 追加の引数。
  • **出力**: なし（void）。ただし、権限チェックに失敗した場合は ereport(ERROR, ...) により処理を中断します。

前提条件と不変条件

  • モジュール初期化時に object_access_hook_str フックに本関数が登録されている必要があります。
  • 処理の前後で audit_attempt および audit_success が呼び出され、監査ログが記録されます。
  • フックチェーンを維持するため、next_object_access_hook_str が存在する場合は必ず呼び出されます。

アルゴリズムまたは振る舞い

1. audit_attempt を呼び出し、操作の試行を記録します。 2. next_object_access_hook_str が設定されている場合、元のフック関数を呼び出してシステム本来の処理を継続します。 3. access が OAT_POST_ALTER の場合、subId に基づいて以下の権限チェックを行います。

  • ACL_SET および ACL_ALTER_SYSTEM の両方が含まれる場合、REGRESS_deny_set_variable が有効かつ非スーパーユーザーであればエラーを発生させます。
  • ACL_SET のみが含まれる場合、REGRESS_deny_set_variable が有効かつ非スーパーユーザーであればエラーを発生させます。
  • ACL_ALTER_SYSTEM のみが含まれる場合、REGRESS_deny_alter_system が有効かつ非スーパーユーザーであればエラーを発生させます。

    • 4. 最後に audit_success を呼び出し、操作の成功を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLコアシステムのオブジェクトアクセスフック呼び出しポイント（object_access_hook_str 経由）。
  • **呼び出し先**:
  • audit_attempt / audit_success（監査ログ出力）
  • next_object_access_hook_str（登録されていた以前のフック関数）
  • accesstype_to_string（ログ用文字列変換）
  • superuser_arg（権限判定）

関連データ構造

  • ObjectAccessType: オブジェクト操作の種類を定義する列挙型。
  • next_object_access_hook_str: 以前に登録されていたフック関数を保持するグローバル変数。

失敗モードと境界条件

  • 非スーパーユーザーが制限された操作（SET や ALTER SYSTEM）を実行しようとした場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了します。
  • subId が想定外の値（ACL_SET や ACL_ALTER_SYSTEM 以外）である場合、elog(ERROR, ...) により処理が停止します。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部です。本番環境への適用を検討する場合は、フックの連鎖（next_*）が正しく維持されていることを確認してください。
  • subId のビットマスク判定は、PostgreSQLのカタログ定義に依存しているため、コア側の変更時には整合性を確認する必要があります。
