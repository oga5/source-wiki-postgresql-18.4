xpath_workspace

定義

  • 種別: struct
  • 定義箇所: contrib/xml2/xpath.c:36

目的

libxml2 を用いた XPath 評価プロセスにおいて、XML ドキュメント、XPath コンテキスト、および評価結果オブジェクトを一元管理するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct
{
	xmlDocPtr	doctree;
	xmlXPathContextPtr ctxt;
	xmlXPathObjectPtr res;
} xpath_workspace;

入力と出力

本構造体は関数間で受け渡される管理オブジェクトであり、直接的な入力・出力はありません。各フィールドは libxml2 のポインタ型を保持します。

前提条件と不変条件

  • libxml2 のメモリ領域を保持するため、PostgreSQL のメモリコンテキストとは独立しています。
  • 使用後は cleanup_workspace を呼び出し、明示的にリソースを解放する必要があります。

アルゴリズムまたは振る舞い

  • doctree: パースされた XML ドキュメントツリーを保持します。
  • ctxt: XPath 式を評価するためのコンテキストを保持します。
  • res: XPath 評価の結果として得られた xmlXPathObject を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: pgxml_xpath 内でローカル変数として宣言され、初期化されます。また、cleanup_workspace の引数として渡されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • xmlDocPtr (libxml2)
  • xmlXPathContextPtr (libxml2)
  • xmlXPathObjectPtr (libxml2)

失敗モードと境界条件

  • cleanup_workspace による解放を怠ると、PostgreSQL プロセス内でメモリリークが発生します。
  • libxml2 の各ポインタが NULL の状態で操作を行うと、セグメンテーション違反を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は contrib/xml2 モジュールにおけるリソース管理の要です。pgxml_xpath 等の関数で利用される際、エラー発生時には PG_CATCH ブロック内で確実に cleanup_workspace が呼ばれる設計になっていることを確認してください。
