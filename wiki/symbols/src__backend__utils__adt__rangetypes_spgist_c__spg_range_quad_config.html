spg_range_quad_config

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/rangetypes_spgist.c:59

目的

範囲型（Range Types）に対するSP-GiSTインデックスの構成設定を定義します。

シグネチャまたは形状

Datum spg_range_quad_config(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SP-GiSTの構成インターフェース経由で渡される引数）。
  • **出力**: void（spgConfigOut 構造体への書き込みを通じて設定を反映）。

前提条件と不変条件

  • spgConfigOut 構造体へのポインタが有効であること。

アルゴリズムまたは振る舞い

SP-GiSTインデックスの動作パラメータを以下のように設定します。 1. prefixType を ANYRANGEOID に設定し、範囲型をプレフィックスとして使用します。 2. labelType を VOIDOID に設定し、ノードラベルを使用しないことを指定します。 3. canReturnData を true に設定し、インデックスから元のデータを取得可能にします。 4. longValuesOK を false に設定し、長い値を許容しない構成とします。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックスの初期化プロセス（PostgreSQLのインデックスアクセス手法の基盤）。
  • **呼び出し先**: なし。

関連データ構造

  • spgConfigOut: SP-GiSTインデックスの構成情報を保持する構造体。
  • ANYRANGEOID: 範囲型のOID。
  • VOIDOID: ラベル不要を示すOID。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。

保守担当者向けメモ

  • 本関数はSP-GiSTの構成定義の一部であり、インデックスの構造（四分木）を決定づける重要な設定です。変更時は spg_range_quad_picksplit や spg_range_quad_inner_consistent との整合性に注意してください。
