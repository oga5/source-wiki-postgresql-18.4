cached_re_str

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/regexp.c:102

目的

正規表現のコンパイル結果と、それに関連するメタデータを保持し、キャッシュとして再利用するための構造体です。

シグネチャまたは形状

typedef struct cached_re_str
{
	MemoryContext cre_context;	/* この正規表現用のメモリコンテキスト */
	char	   *cre_pat;		/* 元の正規表現文字列（null終端ではない） */
	int			cre_pat_len;	/* 元の正規表現の長さ（バイト単位） */
	int			cre_flags;		/* コンパイルフラグ（extended, icase等） */
	Oid			cre_collation;	/* 使用する照合順序 */
	regex_t		cre_re;			/* コンパイル済み正規表現オブジェクト */
} cached_re_str;

入力と出力

本構造体はデータ保持用であり、関数ではありません。RE_compile_and_cache 関数によって初期化・管理されます。

前提条件と不変条件

  • cre_pat は null 終端されていません。
  • コンパイルされた正規表現 cre_re は、cre_context で指定されたメモリコンテキスト内で管理されます。

アルゴリズムまたは振る舞い

  • RE_compile_and_cache が管理する「自己組織化リスト」の一部として機能します。
  • キャッシュヒット時、リストの先頭へ移動させることで LRU（Least Recently Used）に近い挙動を実現します。
  • キャッシュミス時には、古いエントリを追い出し、新しいコンパイル結果を格納します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/regexp.c 内の re_array（静的配列）によって保持・参照されます。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数呼び出しは行いません。

関連データ構造

  • regex_t: regex/regex.h で定義される、POSIX 正規表現エンジンのコンパイル済み形式。
  • MemoryContext: utils/memutils.h で定義されるメモリ管理単位。
  • re_array: cached_re_str 型の静的配列。キャッシュの格納先。

失敗モードと境界条件

  • MAX_CACHED_RES（デフォルト32）を超える数の正規表現が使用される場合、最も古いエントリが破棄されます。
  • cre_pat が null 終端されていないため、文字列操作時には必ず cre_pat_len を使用する必要があります。

保守担当者向けメモ

  • 本構造体は RegexpCacheMemoryContext と密接に関連しています。キャッシュエントリの破棄や更新時には、メモリリークを防ぐため cre_context の適切なリセットまたは解放が必要です。
