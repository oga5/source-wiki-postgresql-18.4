heap_redo

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_xlog.c:1180

目的

RM_HEAP_ID に属するWALレコードを解析し、ヒープページに対する物理的なリプレイ処理をディスパッチする。

シグネチャまたは形状

void heap_redo(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード状態）
  • **出力**: なし（副作用としてヒープページの状態を更新）

前提条件と不変条件

  • record は有効な XLogReaderState を指している必要がある。
  • 本関数で処理される操作はMVCCデータを上書きしないため、Hot Standbyにおけるコンフリクト処理は不要である（heap2_redo との対比）。

アルゴリズムまたは振る舞い

1. XLogRecGetInfo を使用してレコードのオペコードを取得し、XLR_INFO_MASK を除去する。 2. XLOG_HEAP_OPMASK に基づき、以下のいずれかの処理へ分岐する：

  • XLOG_HEAP_INSERT: heap_xlog_insert を呼び出す。
  • XLOG_HEAP_DELETE: heap_xlog_delete を呼び出す。
  • XLOG_HEAP_UPDATE: heap_xlog_update (false) を呼び出す。
  • XLOG_HEAP_TRUNCATE: 何も行わない（論理デコード用レコードのため）。
  • XLOG_HEAP_HOT_UPDATE: heap_xlog_update (true) を呼び出す。
  • XLOG_HEAP_CONFIRM: heap_xlog_confirm を呼び出す。
  • XLOG_HEAP_LOCK: heap_xlog_lock を呼び出す。
  • XLOG_HEAP_INPLACE: heap_xlog_inplace を呼び出す。

    • 3. 未知のオペコードが指定された場合は elog(PANIC) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h にて RM_HEAP_ID のリソースマネージャ関数として登録されており、WALリプレイプロセスから呼び出される。
  • **呼び出し先**: heap_xlog_insert, heap_xlog_delete, heap_xlog_update, heap_xlog_confirm, heap_xlog_lock, heap_xlog_inplace。

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態を保持する構造体。
  • XLOG_HEAP_OPMASK: ヒープ操作を識別するためのビットマスク。

失敗モードと境界条件

  • 未知のオペコードが渡された場合、PANIC レベルのログを出力してプロセスを停止する。

保守担当者向けメモ

  • 本関数は物理的なページ操作のみを扱う。論理的な整合性チェックはWAL生成時に完了している前提である。
  • TRUNCATE は物理的なリプレイとしては不要（SMGRレコードで処理済み）だが、論理デコードのためにレコード自体は存在する。
