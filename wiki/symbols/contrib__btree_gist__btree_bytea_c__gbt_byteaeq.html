gbt_byteaeq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bytea.c:39

目的

GiSTインデックスにおいて、2つの bytea 型の値が等しいかどうかを判定します。

シグネチャまたは形状

bool gbt_byteaeq(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の bytea データへのポインタ。
  • collation: 照合順序（本関数では使用されません）。
  • flinfo: 関数呼び出し情報（本関数では使用されません）。
  • **出力**:
  • bool: a と b が等しい場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は有効な bytea データ構造を指している必要があります。
  • PostgreSQLの標準関数 byteaeq が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. a と b を PointerGetDatum を用いて Datum 型に変換します。 2. DirectFunctionCall2 を介して PostgreSQL の標準関数 byteaeq を呼び出します。 3. byteaeq の戻り値（bool）を DatumGetBool で取得し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: byteaeq (PostgreSQL標準の比較関数)
  • **呼び出し元**: ソースコード内の直接的な参照として contrib/btree_gist/btree_bytea.c:78 で使用されています。

関連データ構造

  • bytea: PostgreSQLの可変長バイナリデータ型。

失敗モードと境界条件

  • byteaeq が内部で発生させるエラー（メモリ不足や不正なデータ形式など）がそのまま伝播します。
  • a または b が NULL の場合の挙動は、byteaeq の実装仕様に依存します。

保守担当者向けメモ

  • 本関数は byteaeq への単なるラッパーです。ロジックの変更が必要な場合は、基盤となる byteaeq の仕様を確認してください。
  • collation および flinfo 引数はシグネチャ合わせのために存在しますが、現在の実装では使用されていません。
