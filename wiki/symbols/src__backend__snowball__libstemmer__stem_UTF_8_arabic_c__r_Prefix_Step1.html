r_Prefix_Step1

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_arabic.c:1046

目的

アラビア語のステミング処理において、特定の接頭辞パターンを検索し、一致した場合はそれを置換することで語幹抽出を行う。

シグネチャまたは形状

static int r_Prefix_Step1(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境の状態を保持する構造体）。
  • **出力**: int 型。成功時は 1、失敗または一致なしの場合は 0、内部エラー発生時は負の値を返す。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • z->p に UTF-8 エンコードされた文字列が格納されていること。
  • 処理対象の文字列長が 3 文字を超えている必要がある（len_utf8(z->p) <= 3 の場合は処理を中断する）。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c を開始点（bra）に設定する。 2. 文字列の特定のバイト位置（z->c + 3）に対してビット演算による事前チェックを行い、一致しない場合は終了する。 3. find_among を使用して、定義されたテーブル a_4（5要素）から接頭辞を検索する。 4. 一致した場合、その範囲を ket に設定し、among_var の値（1〜4）に応じて以下の処理を行う：

  • case 1～4: slice_from_s を呼び出し、該当する接頭辞を定義済みの定数文字列（s_54～s_57）で置換する。

    • 5. 成功時に 1 を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_arabic.c:1605 にて呼び出されている。
  • **呼び出し先**: find_among（検索用）、len_utf8（文字数判定）、slice_from_s（文字列置換）。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。
  • a_4: among 構造体の配列（接頭辞の検索テーブル）。

失敗モードと境界条件

  • 文字列長が 3 以下の場合、置換を行わずに 0 を返す。
  • find_among が一致するパターンを見つけられない場合、0 を返す。
  • slice_from_s が内部エラー（負の値）を返した場合、その値をそのまま呼び出し元へ伝播させる。

保守担当者向けメモ

  • UTF-8 のマルチバイト文字を扱うため、カーソル操作には注意が必要である。
