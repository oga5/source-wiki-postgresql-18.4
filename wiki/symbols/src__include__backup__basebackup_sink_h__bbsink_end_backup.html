bbsink_end_backup

定義

  • 種別: function
  • 定義箇所: src/include/backup/basebackup_sink.h:254

目的

バックアップ処理の最終段階において、シンクチェーンに対してバックアップの終了を通知し、必要な後処理を実行する。

シグネチャまたは形状

static inline void bbsink_end_backup(bbsink *sink, XLogRecPtr endptr, TimeLineID endtli)

入力と出力

  • **入力**:
  • sink: 操作対象の bbsink 構造体へのポインタ。
  • endptr: バックアップ終了時点の WAL 位置 (XLogRecPtr)。
  • endtli: バックアップ終了時点のタイムライン ID (TimeLineID)。
  • **出力**: なし（void）。

前提条件と不変条件

  • sink は NULL であってはならない。
  • sink->bbs_state->tablespace_num と sink->bbs_state->tablespaces のリスト長が一致している必要がある。

アルゴリズムまたは振る舞い

1. 引数として渡された sink が有効であること、およびバックアップ状態の整合性を Assert で検証する。 2. sink->bbs_ops->end_backup コールバック関数を呼び出し、具体的な終了処理を委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c:652
  • src/backend/backup/basebackup_sink.c:114（チェーン内の次段シンクへの転送として呼び出し）
  • **呼び出し先**:
  • sink->bbs_ops->end_backup（bbsink_ops 構造体で定義された関数ポインタ）

関連データ構造

  • bbsink: シンクの基本構造体。
  • bbsink_ops: 各種バックアップイベントに対するコールバック関数テーブル。
  • bbsink_state: バックアップ全体の進捗やメタデータを保持する共有状態。

失敗モードと境界条件

  • sink が NULL の場合、または bbs_state のテーブルスペース情報に不整合がある場合、Assert によりプログラムが停止する可能性がある。

保守担当者向けメモ

  • 本関数はインライン関数であり、シンクチェーンの各段階で bbs_ops->end_backup を呼び出すためのラッパーとして機能する。
  • 新しいシンク型を実装する際は、end_backup コールバックを適切に定義し、必要に応じて bbsink_forward_end_backup 等を用いてチェーンの次段へ処理を伝播させること。
