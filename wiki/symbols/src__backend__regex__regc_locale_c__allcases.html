allcases

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_locale.c:716

目的

指定された文字 c に対し、その文字自身を含むすべてのケース（小文字および大文字）の等価集合を cvec 構造体として生成する。

シグネチャまたは形状

static struct cvec *allcases(struct vars *v, chr c)

入力と出力

  • **入力**:
  • v: 正規表現コンパイル時のコンテキストを保持する vars 構造体。
  • c: 処理対象の文字（chr 型）。
  • **出力**:
  • struct cvec *: 文字 c の小文字変換結果と大文字変換結果を格納した文字ベクトル。

前提条件と不変条件

  • pg_wc_tolower および pg_wc_toupper が現在のロケール設定に基づいて正しく動作すること。
  • getcvec がメモリ確保に成功すること。

アルゴリズムまたは振る舞い

1. pg_wc_tolower(c) および pg_wc_toupper(c) を呼び出し、対象文字の小文字表現と大文字表現を取得する。 2. getcvec(v, 2, 0) を呼び出し、最大2文字を保持できる cvec を確保する。 3. 小文字表現を cvec に追加する。 4. 大文字表現が小文字表現と異なる場合のみ、大文字表現を cvec に追加する。 5. 生成した cvec を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_locale.c:523
  • src/backend/regex/regcomp.c:1926
  • **呼び出し先**:
  • pg_wc_tolower (PostgreSQLロケール関数)
  • pg_wc_toupper (PostgreSQLロケール関数)
  • getcvec (cvec操作関数)
  • addchr (cvec操作関数)

関連データ構造

  • struct vars: 正規表現コンパイルのコンテキスト。
  • struct cvec: 文字の集合を保持する構造体。
  • chr: 文字コードを表す型。

失敗モードと境界条件

  • getcvec がメモリ不足により NULL を返した場合、後続の addchr でクラッシュする可能性がある（呼び出し元でのエラーハンドリングに依存）。
  • 小文字と大文字が同一の文字（記号や数字など）の場合、cvec には1文字のみが格納される。

保守担当者向けメモ

  • 本関数は regcomp.c からも参照されるため、変更時は正規表現コンパイルプロセスへの影響を確認すること。
  • pg_wc_* 関数群はロケール依存であるため、データベースのエンコーディング設定が結果に直接影響する。
