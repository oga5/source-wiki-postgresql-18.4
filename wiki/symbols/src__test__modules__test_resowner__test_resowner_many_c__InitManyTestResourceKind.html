InitManyTestResourceKind

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_resowner/test_resowner_many.c:95

目的

テスト用リソース種別（ManyTestResourceKind）の初期化を行い、リソース管理に必要なメタデータおよび統計カウンタをセットアップします。

シグネチャまたは形状

static void InitManyTestResourceKind(ManyTestResourceKind *kind, char *name,
                                     ResourceReleasePhase phase, uint32 priority)

入力と出力

  • **入力**:
  • kind: 初期化対象の ManyTestResourceKind 構造体へのポインタ。
  • name: リソース種別の名称。
  • phase: ResourceReleasePhase（解放フェーズ）。
  • priority: 解放優先順位（uint32）。
  • **出力**: なし（kind が指す構造体を直接更新）。

前提条件と不変条件

  • kind は有効なメモリ領域を指している必要があります。
  • kind->desc 内のコールバック関数として ReleaseManyTestResource および PrintManyTest が静的に割り当てられます。

アルゴリズムまたは振る舞い

1. 引数 name, phase, priority を kind->desc に設定します。 2. kind->desc の ReleaseResource に ReleaseManyTestResource を、DebugPrint に PrintManyTest を割り当てます。 3. 統計カウンタ（nremembered, nforgotten, nreleased, nleaked）をすべて 0 に初期化します。 4. dlist_init を呼び出し、リソース管理用の侵入型リストを初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**: test_resowner_many 関数（src/test/modules/test_resowner/test_resowner_many.c:232, 240）。
  • **呼び出し先**: dlist_init（lib/ilist.h）。

関連データ構造

  • ManyTestResourceKind: 本関数で初期化される構造体。
  • ResourceReleasePhase: utils/resowner.h で定義される解放フェーズの列挙型。
  • dlist: lib/ilist.h で定義される侵入型リスト。

失敗モードと境界条件

  • kind が NULL の場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数はテストモジュール専用の初期化ロジックです。ResourceOwner のコールバックインターフェース（ReleaseResource, DebugPrint）を固定的に割り当てているため、これらコールバック関数のシグネチャ変更時には本関数の修正が必要です。
