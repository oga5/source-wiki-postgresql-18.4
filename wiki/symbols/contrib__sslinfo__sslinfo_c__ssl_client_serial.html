ssl_client_serial

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:115

目的

現在のセッションで使用されているクライアント証明書のシリアル番号を、PostgreSQLの numeric 型として取得する。

シグネチャまたは形状

Datum ssl_client_serial(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (SQL関数インターフェース経由)
  • **出力**: Datum (PostgreSQLの numeric 型としてシリアル番号を返す。証明書が存在しない、または無効な場合は NULL)

前提条件と不変条件

  • MyProcPort が初期化されていること。
  • SSL接続が確立されており、かつクライアント証明書が検証済みであること（ssl_in_use および peer_cert_valid が真であること）。

アルゴリズムまたは振る舞い

1. MyProcPort を参照し、SSLが使用されていないか証明書が無効な場合は PG_RETURN_NULL() を返す。 2. be_tls_get_peer_serial を呼び出し、証明書のシリアル番号を10進数文字列として decimal バッファに格納する。 3. バッファが空であれば PG_RETURN_NULL() を返す。 4. numeric_in 関数を DirectFunctionCall3 で呼び出し、文字列を numeric 型の Datum に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • be_tls_get_peer_serial (証明書シリアル番号の取得)
  • numeric_in (文字列から numeric 型への変換)

関連データ構造

  • Port (グローバル変数 MyProcPort を介してアクセス)

失敗モードと境界条件

  • SSL接続が確立されていない場合や、クライアント証明書が提供されていない場合は NULL を返す。
  • be_tls_get_peer_serial がシリアル番号を取得できない場合、空の文字列として扱われ NULL を返す。
  • numeric_in への変換失敗時は、PostgreSQLの標準的なエラーハンドリング（ereport(ERROR, ...)）によりトランザクションが中断される可能性がある。

保守担当者向けメモ

  • 本関数は MyProcPort に強く依存している。
  • be_tls_get_peer_serial は src/backend/libpq/be-secure-openssl.c 等で実装されている低レイヤー関数であり、OpenSSLの証明書構造体からシリアル番号を抽出する。
  • 戻り値の型は numeric であるため、シリアル番号が非常に長い場合や特殊な形式の場合の変換挙動については numeric_in の仕様に準拠する。
