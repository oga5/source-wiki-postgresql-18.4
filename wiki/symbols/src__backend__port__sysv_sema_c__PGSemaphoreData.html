PGSemaphoreData

定義

  • 種別: struct
  • 定義箇所: src/backend/port/sysv_sema.c:30

目的

PostgreSQL のセマフォ抽象化において、System V セマフォセット内の特定のセマフォを識別・参照するためのデータ構造です。

シグネチャまたは形状

typedef struct PGSemaphoreData
{
    int semId;    /* semaphore set identifier */
    int semNum;   /* semaphore number within set */
} PGSemaphoreData;

入力と出力

本構造体はデータ保持を目的としており、関数としての入力・出力はありません。各フィールドは以下の値を保持します。

  • semId: カーネルから割り当てられたセマフォセットの識別子。
  • semNum: セマフォセット内における特定のセマフォのインデックス。

前提条件と不変条件

  • semId は、semget システムコール等で取得された有効な System V セマフォセット ID である必要があります。
  • semNum は、確保されたセマフォセットの範囲内（0 から SEMAS_PER_SET - 1）である必要があります。

アルゴリズムまたは振る舞い

本構造体自体は状態を保持するコンテナです。この構造体を利用して、PGSemaphoreLock や PGSemaphoreUnlock などの関数が semop システムコールを呼び出し、プロセス間同期を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/sysv_sema.c 内の PGSemaphoreCreate や PGSemaphoreReset 等が、共有メモリ上の PGSemaphoreData 配列を操作します。また、src/backend/port/posix_sema.c でも同名の構造体が定義・利用されています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • PGSemaphore: PGSemaphoreData へのポインタとして定義される共通インターフェース型。
  • sharedSemas: 共有メモリ上に配置される PGSemaphoreData の配列。

失敗モードと境界条件

  • 共有メモリ上の配列サイズ（maxSharedSemas）を超えてアクセスした場合、メモリ破壊や未定義動作を引き起こす可能性があります。
  • semId が無効な場合、後続の semop 呼び出しが失敗します。

保守担当者向けメモ

  • 本構造体は sysv_sema.c と posix_sema.c の両方で定義されています。プラットフォーム間で共有されるインターフェースであるため、変更時は両方の実装への影響を考慮してください。
  • 各セマフォセットの末尾には識別用のセマフォが確保されており、semNum の管理には注意が必要です。
