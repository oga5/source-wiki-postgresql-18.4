printXheaderWidthType

定義

  • 種別: enum
  • 定義箇所: src/include/fe_utils/print.h:69

目的

拡張表示（Expanded format）におけるヘッダー行の幅を制御するための列挙型です。

シグネチャまたは形状

typedef enum printXheaderWidthType
{
    PRINT_XHEADER_FULL,
    PRINT_XHEADER_COLUMN,
    PRINT_XHEADER_PAGE,
    PRINT_XHEADER_EXACT_WIDTH,
} printXheaderWidthType;

入力と出力

  • 入力: なし（列挙定数）
  • 出力: printTableOpt 構造体内の expanded_header_width_type フィールドとして保持されます。

前提条件と不変条件

  • PRINT_XHEADER_FULL がデフォルトの挙動として想定されています。

アルゴリズムまたは振る舞い

  • PRINT_XHEADER_FULL: ヘッダー行を切り詰めずに出力します。
  • PRINT_XHEADER_COLUMN: 最初のカラムの上部にのみヘッダー行を出力します。
  • PRINT_XHEADER_PAGE: 端末の幅を超えないようにヘッダー行を制限します。
  • PRINT_XHEADER_EXACT_WIDTH: 指定された正確な幅に合わせてヘッダー行を調整します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/fe_utils/print.h:116 にて printTableOpt 構造体のメンバとして参照されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数呼び出し先は特定できません。

関連データ構造

  • printTableOpt: この列挙型をメンバとして保持し、出力設定を管理します。

失敗モードと境界条件

  • PRINT_XHEADER_PAGE を使用する場合、端末幅の取得に依存します。端末幅が正しく取得できない場合の挙動については、本ソース excerpt に記載がありません。

保守担当者向けメモ

  • 本シンボルは printTable による出力整形ロジックの挙動を決定します。新しいヘッダー幅制御を追加する場合は、print.c 側の実装で各列挙値に対する条件分岐を更新する必要があります。
