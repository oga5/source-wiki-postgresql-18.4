analyzeCTETargetList

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_cte.c:570

目的

共通テーブル式（CTE）の出力列に関するメタデータ（名前、型、照合順序）を確定させ、CommonTableExpr 構造体に格納する。

シグネチャまたは形状

void analyzeCTETargetList(ParseState *pstate, CommonTableExpr *cte, List *tlist)

入力と出力

  • **入力**:
  • pstate: パーサーの状態管理構造体。
  • cte: 解析対象の CommonTableExpr 構造体。
  • tlist: CTEの出力に対応する TargetEntry のリスト。
  • **出力**: なし（副作用として cte 引数のフィールドを更新する）。

前提条件と不変条件

  • cte->ctecolnames が NIL であること（未初期化状態であること）。
  • tlist 内の TargetEntry は、resjunk でないものが順序通りに並んでいること。

アルゴリズムまたは振る舞い

1. **メタデータ初期化**: cte->aliascolnames を基に列名を初期化する。 2. **列情報の抽出**: tlist を走査し、各列の型（Oid）、型修飾子（typmod）、照合順序（collation）を抽出する。 3. **エイリアス補完**: aliascolnames が不足している場合、TargetEntry の名前を使用して ctecolnames を拡張する。 4. **再帰CTEの特殊処理**: cte->cterecursive が真かつ型が UNKNOWN の場合、型を TEXTOID に強制し、照合順序が未設定ならデフォルトを適用する。 5. **結果格納**: 抽出した情報を cte の各フィールド（ctecoltypes, ctecoltypmods, ctecolcollations）に保存する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c:2432
  • src/backend/parser/parse_cte.c:349
  • **呼び出し先**:
  • exprType, exprTypmod, exprCollation（式情報の取得）
  • copyObject, lappend, lappend_oid, lappend_int（リスト操作）
  • ereport（エラー報告）

関連データ構造

  • CommonTableExpr: CTEの定義を保持する構造体。
  • TargetEntry: クエリの出力列を表現する構造体。
  • ParseState: 解析中の状態を保持する構造体。

失敗モードと境界条件

  • **列数不一致**: tlist 内の有効な列数が、指定されたエイリアス数よりも少ない場合、ERRCODE_INVALID_COLUMN_REFERENCE エラーを発生させる。

保守担当者向けメモ

  • 本関数は再帰的CTEの非再帰項に対して呼び出されることを想定しており、未知の型（UNKNOWN）を解決する役割を担う。
  • SQL標準に従い、エイリアスリストが提供されている場合はそれが優先されるが、不足している場合は自動的に列名が補完される仕様である。
