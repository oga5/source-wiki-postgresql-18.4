find_window_functions

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/clauses.c:228

目的

式ツリー（clause）を走査し、含まれるウィンドウ関数を抽出して、ウィンドウ参照（winref）ごとに分類されたリスト構造を構築します。

シグネチャまたは形状

WindowFuncLists *find_window_functions(Node *clause, Index maxWinRef)

入力と出力

  • **入力**:
  • clause: 走査対象の式ツリー（Node *）。
  • maxWinRef: 考慮すべきウィンドウ参照の最大値（Index）。
  • **出力**:
  • WindowFuncLists *: ウィンドウ関数が格納された構造体へのポインタ。

前提条件と不変条件

  • maxWinRef は、式ツリー内に存在する可能性のあるウィンドウ参照の最大値以上である必要があります。
  • 戻り値の windowFuncs 配列は maxWinRef + 1 のサイズで確保されます。

アルゴリズムまたは振る舞い

1. WindowFuncLists 構造体をメモリ確保し、初期化します。 2. windowFuncs 配列を maxWinRef + 1 個の要素で確保し、ゼロ初期化します。 3. find_window_functions_walker を呼び出し、式ツリーを再帰的に走査します。 4. 走査中に見つかったウィンドウ関数を、対応する winref インデックスに基づいて windowFuncs リストに分類・格納します。 5. 構築した WindowFuncLists を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c: クエリのプランニングプロセスにおいて、ターゲットリスト内のウィンドウ関数を特定するために呼び出されます。
  • **呼び出し先**:
  • find_window_functions_walker: 式ツリーの再帰的な走査を実行します。
  • palloc / palloc0: メモリ確保を行います。

関連データ構造

  • WindowFuncLists: ウィンドウ関数のリストと参照情報を保持する構造体。
  • WindowFunc: 式ツリー内のウィンドウ関数ノード。

失敗モードと境界条件

  • メモリ不足時には palloc によりエラーが発生します。
  • maxWinRef が実際のウィンドウ参照数より小さい場合、配列外アクセスのリスクがあるため、呼び出し側で適切な値を指定する必要があります。

保守担当者向けメモ

  • 本関数は find_window_functions_walker と密結合しています。走査ロジックを変更する場合は、両方の整合性を確認してください。
  • WindowFuncLists のメモリレイアウトは maxWinRef に依存するため、プランナのコンテキストにおけるウィンドウ参照の管理状況を考慮する必要があります。
