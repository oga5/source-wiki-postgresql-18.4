LockViewRecurse_context

定義

  • **種別**: struct
  • **定義箇所**: src/backend/commands/lockcmds.c:167

目的

ビューに対する再帰的なロック処理を行う際、現在のロック状態、対象ビューの識別子、および循環参照を防ぐための祖先ビューのリストを保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    LOCKMODE    lockmode;        /* 使用するロックモード */
    bool        nowait;          /* NOWAIT モードの有無 */
    Oid         check_as_user;   /* 権限チェックに使用するユーザーOID */
    Oid         viewoid;         /* ロック対象ビューのOID */
    List       *ancestor_views;  /* 祖先ビューのOIDリスト */
} LockViewRecurse_context;

入力と出力

本構造体は関数への引数として渡されるコンテキストオブジェクトとして機能します。

  • **入力**: ロック実行時に設定される lockmode, nowait, check_as_user, viewoid。
  • **出力**: 再帰走査中に更新される ancestor_views（循環参照防止用）。

前提条件と不変条件

  • ancestor_views は、再帰的なビュー走査において既にロック済みのビューを追跡するために使用されます。
  • check_as_user は、ビューの security_invoker プロパティに応じて、ビュー所有者または現在のユーザーの権限を検証するために使用されます。

アルゴリズムまたは振る舞い

LockViewRecurse 関数がビューのクエリ定義を走査する際、この構造体を介してロックパラメータを伝播させます。LockViewRecurse_walker はこのコンテキストを参照し、クエリ内の RangeTblEntry を特定して再帰的にロックを適用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/lockcmds.c 内の LockViewRecurse 関数がこの構造体を初期化して使用します。
  • **呼び出し先**: LockViewRecurse_walker 関数がこの構造体を引数として受け取ります。

関連データ構造

  • LOCKMODE: ロックの強度を定義する列挙型。
  • List: PostgreSQL のリスト構造（ancestor_views の保持に使用）。
  • RangeTblEntry: ビューのクエリ定義に含まれるリレーション情報。

失敗モードと境界条件

  • nowait が true の場合、ロックが即座に取得できないとエラーが発生します。
  • ビューの循環参照が発生した場合、ancestor_views を用いたチェックにより無限再帰が防止されます。

保守担当者向けメモ

  • 本構造体は src/backend/commands/lockcmds.c 内のビュー再帰ロック専用です。
  • ancestor_views リストの管理は LockViewRecurse 内で行われるため、このリストのメモリ管理（palloc 等）については LockViewRecurse の実装を参照してください。
