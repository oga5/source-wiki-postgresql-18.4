r_R2

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_KOI8_R_russian.c:380

目的

Snowballアルゴリズムにおける「R2領域」の境界条件を判定し、現在のカーソル位置がR2領域内にあるかを確認する。

シグネチャまたは形状

static int r_R2(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （ステミング環境の状態を保持する構造体）
  • **出力**: int 型。現在のカーソル位置 z->c が、事前に計算されたR2領域の開始位置 z->I[0] 以降であれば真（1）、そうでなければ偽（0）を返す。

前提条件と不変条件

  • z->I[0] には、r_mark_regions 関数によって計算されたR2領域の開始オフセットが格納されている必要がある。
  • z->c は現在の処理対象文字列内のカーソル位置を示す。

アルゴリズムまたは振る舞い

1. ステミング環境 z のインデックス配列 I の0番目（R2領域の開始位置）と、現在のカーソル位置 c を比較する。 2. z->I[0] <= z->c の評価結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 本ファイル（stem_KOI8_R_russian.c）内のステミングルール適用関数から呼び出されることが想定される。また、他の言語実装（stem_ISO_8859_1_basque.c, stem_ISO_8859_1_catalan.c, stem_ISO_8859_1_dutch.c 等）でも同名の関数が定義・利用されている。
  • **呼び出し先**: 外部関数呼び出しなし。

関連データ構造

  • struct SN_env: Snowballランタイムの共通環境構造体。I（整数配列）および c（カーソル位置）メンバを使用。

失敗モードと境界条件

  • z が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • z->I[0] が適切に初期化されていない場合、誤った判定結果を返す。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによって自動生成されたコードである。直接編集せず、元のSnowballスクリプト（.sbl）を修正すること。
  • I[0] がR2領域を指すという規約は、Snowballランタイム全体で共有されている。
