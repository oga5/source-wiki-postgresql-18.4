PLyObject_ToJsonbValue

定義

  • **種別**: 関数
  • **定義箇所**: contrib/jsonb_plpython/jsonb_plpython.c:409

目的

Python オブジェクトを PostgreSQL の JsonbValue 構造体に変換し、必要に応じて JsonbParseState を介して JSONB 構造へ組み込む。

シグネチャまたは形状

JsonbValue *PLyObject_ToJsonbValue(PyObject *obj, JsonbParseState **jsonb_state, bool is_elem)

入力と出力

  • **入力**:
  • obj: 変換対象の Python オブジェクト。
  • jsonb_state: JSONB 構築状態を保持するポインタ。NULL の場合は単一のスカラ値として処理される。
  • is_elem: 配列の要素として追加するか（true）、オブジェクトの値として追加するか（false）を指定するフラグ。
  • **出力**: 変換された JsonbValue 構造体へのポインタ。

前提条件と不変条件

  • jsonb_state が提供されている場合、関数終了時に pushJsonbValue が呼び出され、構築状態が更新される。
  • PyBool_Check は PyNumber_Check よりも先に評価される必要がある（Python の仕様上、bool は number のサブクラスであるため）。

アルゴリズムまたは振る舞い

1. **型判定と委譲**:

  • Python のシーケンス型であれば PLySequence_ToJsonbValue を、マッピング型であれば PLyMapping_ToJsonbValue を呼び出す。

    • 2. **スカラ変換**:

  • None は jbvNull に変換。
  • 文字列は PLyUnicode_ToJsonbValue を使用。
  • 真偽値は jbvBool に変換。
  • 数値は PLyNumber_ToJsonbValue を使用。

    • 3. **エラー処理**: サポートされていない型が渡された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーを報告する。 4. **構造への組み込み**: jsonb_state が有効な場合、pushJsonbValue を用いて is_elem に応じたトークン（WJB_ELEM または WJB_VALUE）で結果を状態にプッシュする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/jsonb_plpython/jsonb_plpython.c 内の各所（PLySequence_ToJsonbValue や PLyMapping_ToJsonbValue 等の再帰呼び出しを含む）。
  • **呼び出し先**:
  • PLySequence_ToJsonbValue, PLyMapping_ToJsonbValue, PLyUnicode_ToJsonbValue, PLyNumber_ToJsonbValue, pushJsonbValue。

関連データ構造

  • JsonbValue: PostgreSQL の JSONB 要素を表現する構造体。
  • JsonbParseState: JSONB 構築中の状態を保持する構造体。
  • PyObject: Python 側のオブジェクト。

失敗モードと境界条件

  • **未サポート型**: Python の型が変換対象外の場合、PostgreSQL エラーとして処理される。
  • **メモリ不足**: palloc によるメモリ確保失敗時は PostgreSQL のエラーハンドリング機構により中断される。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、複雑なネスト構造を持つ Python オブジェクトの変換時にスタックオーバーフローのリスクがないか留意すること。
  • PyNumber_Check の挙動に依存しているため、Python の型階層の変更には注意が必要。
