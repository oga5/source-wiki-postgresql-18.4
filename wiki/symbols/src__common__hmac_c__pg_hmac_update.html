pg_hmac_update

定義

  • 種別: function
  • 定義箇所: src/common/hmac.c:222

目的

HMAC 計算の進行中に、認証対象となるメッセージデータを逐次的にハッシュコンテキストへ追加する。

シグネチャまたは形状

int pg_hmac_update(pg_hmac_ctx *ctx, const uint8 *data, size_t len);

入力と出力

  • ctx: HMAC コンテキストへのポインタ。
  • data: 追加するデータバッファへのポインタ。
  • len: 追加するデータのバイト長。
  • 戻り値: 成功時は 0、失敗時は -1 を返す。

前提条件と不変条件

  • ctx は pg_hmac_create および pg_hmac_init によって初期化済みであること。
  • ctx が NULL の場合、関数は即座に -1 を返す。

アルゴリズムまたは振る舞い

1. ctx が有効か確認する。 2. 内部の pg_cryptohash コンテキストに対し、pg_cryptohash_update を呼び出してデータを渡す。 3. pg_cryptohash_update がエラーを返した場合、ctx->error に PG_HMAC_ERROR_INTERNAL を設定し、ctx->errreason に詳細なエラー情報を格納した上で -1 を返す。 4. 正常終了時は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-scram.c において、SCRAM 認証プロセスのデータ処理に使用されていることが確認されている。
  • **呼び出し先**: pg_cryptohash_update および pg_cryptohash_error を呼び出す。

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する不透明な構造体。

失敗モードと境界条件

  • ctx が NULL の場合: -1 を返す。
  • 内部ハッシュ計算（pg_cryptohash_update）が失敗した場合: -1 を返し、コンテキスト内にエラー状態を記録する。

保守担当者向けメモ

  • 本関数は OpenSSL 等の外部ライブラリを使用しないフォールバック実装の一部である。
  • 内部ハッシュ関数のエラーハンドリングは pg_cryptohash インターフェースに依存している。
