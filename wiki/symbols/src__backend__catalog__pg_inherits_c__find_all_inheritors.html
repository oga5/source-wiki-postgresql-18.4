find_all_inheritors

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_inherits.c:254

目的

指定されたリレーション（テーブル）から継承階層を再帰的に辿り、直接的および間接的なすべての子孫リレーションのリストを取得する。

シグネチャまたは形状

List *find_all_inheritors(Oid parentrelId, LOCKMODE lockmode, List **numparents)

入力と出力

  • **入力**:
  • parentrelId: 起点となる親リレーションの OID。
  • lockmode: 探索中に各リレーションに対して取得するロックモード。
  • numparents: 各子孫が持つ親の数を格納するリストへのポインタ（不要な場合は NULL）。
  • **出力**:
  • 継承階層に含まれるすべてのリレーション OID を含む List*。

前提条件と不変条件

  • 呼び出し元は、対象のリレーションに対して適切なロックを保持している必要がある。
  • 継承グラフに循環は存在しないものと仮定する。

アルゴリズムまたは振る舞い

1. seen_rels ハッシュテーブルを初期化し、訪問済みリレーションを追跡する。 2. rels_list（結果リスト）と rel_numparents（各リレーションの親数リスト）を初期化する。 3. rels_list をキューとして扱い、各リレーションに対して find_inheritance_children を呼び出し、直接の子を取得する。 4. 未訪問の子であれば rels_list に追加し、既に訪問済みであれば rel_numparents の該当カウンタをインクリメントする。 5. 全階層の走査完了後、ハッシュテーブルを破棄し、結果リストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/dml.c
  • src/backend/catalog/pg_publication.c
  • src/backend/commands/analyze.c
  • src/backend/commands/cluster.c
  • src/backend/commands/indexcmds.c
  • **呼び出し先**:
  • find_inheritance_children (直接の子を取得)
  • hash_create, hash_search, hash_destroy (訪問済み管理)
  • list_make1_oid, lappend_oid, list_length 等 (リスト操作)

関連データ構造

  • SeenRelsEntry: ハッシュテーブルの各エントリ。リレーションの OID と、rel_numparents リスト内のインデックスを保持する。

失敗モードと境界条件

  • 継承関係が存在しない場合、起点となるリレーションのみを含むリストを返す。
  • numparents が NULL の場合、内部で生成した rel_numparents リストはメモリリークを防ぐために解放される。

保守担当者向けメモ

  • 探索順序は幅優先探索（BFS）に相当する。
  • rel_numparents のインデックス管理は rels_list の長さと同期しているため、リスト操作時の整合性に注意すること。
