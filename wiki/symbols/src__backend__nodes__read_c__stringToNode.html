stringToNode

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/read.c:89

目的

シリアライズされた文字列形式のPostgreSQL内部ノード構造（Nodeツリー）を、メモリ上のオブジェクトへ復元する公開インターフェースです。

シグネチャまたは形状

void *stringToNode(const char *str);

入力と出力

  • **入力**: str (ノード構造を表現するシリアライズ済み文字列)
  • **出力**: 復元された Node ツリーのルートポインタ（void *）

前提条件と不変条件

  • 入力文字列は outfuncs.c によって生成された形式に従っている必要があります。
  • 内部的に pg_strtok_ptr を静的変数として使用するため、再帰的なノード読み込みは可能ですが、マルチスレッド環境での同時実行には適していません。

アルゴリズムまたは振る舞い

1. 引数として受け取った文字列 str を stringToNodeInternal へ渡します。 2. 第2引数に false を指定して呼び出し、ノードの読み込みを開始します。 3. 内部で pg_strtok を用いてトークンを切り出し、nodeRead を再帰的に呼び出すことでノードツリーを再構築します。 4. 最終的に再構築されたノードのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tupdesc.c
  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/catalog/pg_proc.c
  • src/backend/catalog/pg_type.c
  • src/backend/commands/policy.c
  • **呼び出し先**: stringToNodeInternal

関連データ構造

  • Node: 復元対象となるPostgreSQLの内部ノード構造。
  • List, Bitmapset: nodeRead を通じて再帰的に構築される主要なデータ構造。

失敗モードと境界条件

  • 入力文字列が不正な形式である場合、nodeRead 経由でエラーが発生する可能性があります。
  • エスケープされた文字が含まれる場合、適切に debackslash 処理が行われないと復元に失敗します。

保守担当者向けメモ

  • 本関数は stringToNodeInternal のラッパーです。詳細な解析ロジックは nodeRead および readfuncs.c に依存しています。
  • pg_strtok_ptr が静的変数であるため、呼び出しの並行性には注意してください。
