pgstat_reset_wait_event_storage

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/wait_event.c:360

目的

バックエンドプロセスの待機イベント情報を保持するポインタを、共有メモリ上の領域からローカル変数へリセットします。

シグネチャまたは形状

void pgstat_reset_wait_event_storage(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 my_wait_event_info の参照先を更新）

前提条件と不変条件

  • my_wait_event_info は、本関数の呼び出し後、ローカル変数 local_my_wait_event_info を指すようになります。

アルゴリズムまたは振る舞い

1. グローバルポインタ my_wait_event_info に、ローカル変数 local_my_wait_event_info のアドレスを代入します。 2. これにより、以降の待機イベント報告は共有メモリではなくローカル変数に対して行われるようになります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/proc.c 内のプロセス終了処理に関連する箇所（行995および1064）。
  • 呼び出し先: なし。

関連データ構造

  • my_wait_event_info: 現在の待機状態を保持するポインタ。
  • local_my_wait_event_info: 共有メモリ初期化前や終了後に使用されるローカルの待機イベント格納領域。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。

保守担当者向けメモ

  • 本関数は、バックエンドプロセスが共有メモリへのアクセスを終了する際（プロセスの終了処理など）に呼び出されます。
  • 共有メモリ上の領域を指している状態で本関数を呼び出さないと、プロセス終了後に共有メモリ上の無効な領域を参照するリスクがあるため、呼び出しタイミングには注意が必要です。
