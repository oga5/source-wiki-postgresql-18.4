heap_page_items

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/heapfuncs.c:129

目的

PostgreSQL のヒープページ（テーブルデータ）を低レベルで解析し、ページ内の各行ポインタおよびタプルヘッダ情報を SQL レベルで抽出・可視化します。

シグネチャまたは形状

Datum heap_page_items(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型の生ページデータ（PG_GETARG_BYTEA_P(0)）。
  • **出力**: ページ内の各アイテムに対応する行情報を含む複合型（SRF: Set Returning Function）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 入力ページサイズが SizeOfPageHeaderData 以上であること。
  • ページ内の各アイテム（行ポインタ）は、ページ境界内に収まっている必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: SRF_IS_FIRSTCALL でページヘッダを検証し、heap_page_items_state を初期化して走査範囲（max_calls）を決定。 2. **走査**: offset をインクリメントしながらページ内の各行ポインタを順次処理。 3. **バリデーション**: 行ポインタの lp_offset や lp_len がページ範囲内か、t_hoff が妥当かをチェックし、破損ページでもクラッシュを防ぐ。 4. **抽出**:

  • 行ポインタ情報（オフセット、フラグ、長さ）を取得。
  • タプルヘッダ情報（Xmin, Xmax, CommandId, ctid, infomask 等）を抽出。
  • HEAP_HASNULL フラグがある場合は NULL ビットマップをテキスト化。
  • HEAP_HASOID_OLD フラグがある場合は OID を取得。
  • タプルの実データを bytea として抽出。

    • 5. **返却**: heap_form_tuple で構築した結果を SRF_RETURN_NEXT で返し、全走査完了後に SRF_RETURN_DONE を発行。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 マクロによる登録）。
  • **呼び出し先**: superuser(), PageGetMaxOffsetNumber, PageGetItemId, PageGetItem, bits_to_text, HeapTupleHeaderGetOidOld, heap_form_tuple 等。

関連データ構造

  • heap_page_items_state: SRF 呼び出し間でページ走査の状態（現在のオフセット、ページポインタ、タプル記述子）を保持。
  • HeapTupleHeaderData: タプルのヘッダ情報。
  • ItemIdData: 行ポインタ。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合、エラーを報告。
  • **ページサイズ異常**: 入力ページが SizeOfPageHeaderData 未満の場合、エラーを報告。
  • **破損データ**: ページが破損している場合、バリデーションチェックにより一部フィールドを NULL に設定し、クラッシュを回避して可能な限りの情報を返す。

保守担当者向けメモ

  • 本関数は破損したページを読み込む可能性があるため、メモリ破壊を防ぐための境界チェックが厳格に行われています。
  • values[14] および nulls[14] は、返される複合型のカラム数に対応しています。スキーマ変更時はこの配列サイズとインデックスの整合性に注意してください。
