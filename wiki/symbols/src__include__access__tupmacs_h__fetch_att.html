fetch_att

定義

  • 種別: function
  • 定義箇所: src/include/access/tupmacs.h:52

目的

タプル内のデータ領域から、属性の定義（値渡しフラグおよび長さ）に基づいて Datum を安全に抽出する。

シグネチャまたは形状

static inline Datum fetch_att(const void *T, bool attbyval, int attlen)

入力と出力

  • **入力**:
  • T: 属性値が格納されているメモリ領域へのポインタ。
  • attbyval: 属性が値渡し（true）か参照渡し（false）かを示すフラグ。
  • attlen: 属性のバイト長。
  • **出力**:
  • 抽出された Datum 値。

前提条件と不変条件

  • attbyval が true の場合、attlen は sizeof(char)、sizeof(int16)、sizeof(int32)、または sizeof(Datum)（64bit環境のみ）のいずれかである必要がある。これら以外の場合はエラーとなる。

アルゴリズムまたは振る舞い

1. attbyval が true の場合:

  • attlen に応じて T が指す値を適切な型にキャストし、CharGetDatum、Int16GetDatum、Int32GetDatum 等のマクロを用いて Datum に変換して返す。
  • サポートされていない長さの場合は elog(ERROR) を発生させる。

    • 2. attbyval が false の場合:

  • T をそのまま PointerGetDatum に渡し、ポインタ値を Datum として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_minmax_multi.c
  • src/backend/catalog/pg_type.c
  • src/backend/executor/execExprInterp.c
  • src/backend/statistics/mcv.c
  • src/backend/utils/adt/arrayfuncs.c
  • **呼び出し先**:
  • CharGetDatum, Int16GetDatum, Int32GetDatum, PointerGetDatum などの Datum 変換マクロ。
  • elog（エラー発生時）。

関連データ構造

  • Datum: PostgreSQL における汎用的なデータ表現型。

失敗モードと境界条件

  • attbyval が true で、かつ attlen が定義されたサイズ（1, 2, 4, 8）のいずれにも該当しない場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は tupmacs.h 内のインライン関数であり、パフォーマンスが重視されるタプル走査のホットパスで使用される。
  • attbyval が false の場合はポインタをそのまま返すため、呼び出し側でデータの有効期間を管理する必要がある。
