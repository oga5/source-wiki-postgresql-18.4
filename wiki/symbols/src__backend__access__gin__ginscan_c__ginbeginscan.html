ginbeginscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginscan.c:24

目的

GINインデックススキャンを開始するための状態管理構造体（GinScanOpaque）を初期化し、スキャンに必要なメモリコンテキストを準備する。

シグネチャまたは形状

IndexScanDesc ginbeginscan(Relation rel, int nkeys, int norderbys)

入力と出力

  • **入力**:
  • rel: スキャン対象のインデックスリレーション。
  • nkeys: スキャンキーの数。
  • norderbys: ORDER BY句の数。
  • **出力**:
  • 初期化済みの IndexScanDesc 構造体。

前提条件と不変条件

  • norderbys は必ず 0 であること（GINインデックスはORDER BY演算子をサポートしないため、Assertにより強制される）。

アルゴリズムまたは振る舞い

1. RelationGetIndexScan を呼び出し、標準的なインデックススキャン記述子を生成する。 2. GinScanOpaqueData 構造体用のメモリを確保し、初期化する。 3. スキャン用のメモリコンテキストとして tempCtx（一時作業用）と keyCtx（スキャンキー管理用）の2つを作成する。 4. initGinState を呼び出し、GinScanOpaque 内の ginstate を初期化する。 5. 生成した GinScanOpaque を scan->opaque に格納し、スキャン記述子を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginutil.c 内の amroutine->ambeginscan への代入を通じて、インデックスアクセスメソッドのインターフェースとして呼び出される。
  • **呼び出し先**:
  • RelationGetIndexScan
  • palloc
  • AllocSetContextCreate
  • initGinState

関連データ構造

  • IndexScanDesc: インデックススキャンの共通状態。
  • GinScanOpaqueData: GIN固有のスキャン状態（キー配列、メモリコンテキスト、GinState を保持）。

失敗モードと境界条件

  • norderbys != 0 の場合、アサーション違反により異常終了する。
  • メモリ不足時には palloc または AllocSetContextCreate がエラーを発生させる。

保守担当者向けメモ

  • keyCtx はスキャンキーの生存期間を管理しており、ginrescan 時にリセットされる設計となっている。
  • 本関数はインデックススキャンのライフサイクルの起点であり、終了時には ginendscan によるリソース解放が必須である。
