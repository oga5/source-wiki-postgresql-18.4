provider_init

定義

  • 種別: function
  • 定義箇所: src/backend/jit/jit.c:66

目的

JITプロバイダーの共有ライブラリを動的にロードし、プロバイダー関数ポインタを初期化する。

シグネチャまたは形状

static bool provider_init(void);

入力と出力

  • **入力**: なし（GUC変数 jit_enabled および jit_provider を参照）
  • **出力**:
  • true: プロバイダーのロードおよび初期化に成功した場合。
  • false: JITが無効、プロバイダーが見つからない、またはロードに失敗した場合。

前提条件と不変条件

  • jit_enabled が有効であること。
  • pkglib_path に指定されたパスに共有ライブラリが存在すること。
  • 一度ロードに失敗した場合は provider_failed_loading が true となり、以降の呼び出しでは即座に false を返す。

アルゴリズムまたは振る舞い

1. jit_enabled が無効であれば false を返す。 2. 既にロード済み（provider_successfully_loaded）であれば true を返す。 3. ロード失敗履歴（provider_failed_loading）がある場合は false を返す。 4. pkglib_path と jit_provider から共有ライブラリのパスを構築し、pg_file_exists で存在を確認する。存在しない場合は provider_failed_loading を true に設定して false を返す。 5. load_external_function を使用して _PG_jit_provider_init シンボルをロードする。 6. ロードした関数を呼び出し、provider 構造体を初期化する。 7. 成功フラグを更新し、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/jit.c 内の関数（pg_jit_available 等から呼び出されることが示唆されている）。
  • **呼び出し先**:
  • load_external_function（PostgreSQLの動的ロード機能）
  • _PG_jit_provider_init（ロードされた共有ライブラリ内の初期化関数）

関連データ構造

  • provider: ロードされたプロバイダーの関数ポインタを保持する構造体。
  • JitProviderInit: プロバイダー初期化関数の型定義。

失敗モードと境界条件

  • **ライブラリ不在**: 指定されたパスにファイルが存在しない場合、provider_failed_loading をセットし、以降の再試行を抑制する。
  • **ロード失敗**: load_external_function がエラーを発生させた場合、ユーザーに通知される。

保守担当者向けメモ

  • 本関数は一度だけ実行されることを想定した設計である。
  • provider_failed_loading フラグにより、パフォーマンス低下を防ぐための再試行抑止ロジックが組み込まれている。
  • 共有ライブラリのロード失敗時は、依存関係の問題等で load_external_function がエラーを投げる可能性がある。
