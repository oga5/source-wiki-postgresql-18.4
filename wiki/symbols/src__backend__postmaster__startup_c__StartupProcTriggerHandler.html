StartupProcTriggerHandler

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:92

目的

スタンバイモードからプライマリへの昇格要求（SIGUSR2）を処理し、リカバリプロセスを中断または終了させるためのトリガーを引く。

シグネチャまたは形状

static void StartupProcTriggerHandler(SIGNAL_ARGS)
  • SIGNAL_ARGS: シグナルハンドラ用の標準引数。

入力と出力

  • 入力: なし（シグナルハンドラとして呼び出される）。
  • 出力: なし（副作用としてグローバル変数を更新し、リカバリ処理を再開させる）。

前提条件と不変条件

  • 本関数は SIGUSR2 シグナルを受信した際に呼び出されることを前提とする。
  • promote_signaled フラグが true に設定されることで、リカバリ処理中のループが昇格を認識する。

アルゴリズムまたは振る舞い

1. グローバル変数 promote_signaled を true に設定する。 2. WakeupRecovery() を呼び出し、リカバリ処理の待機状態を解除して昇格処理へ移行させる。

呼び出し元と呼び出し先

  • 呼び出し元: SIGUSR2 シグナル受信時のOSシグナルディスパッチャ。
  • 呼び出し先: WakeupRecovery()。

関連データ構造

  • promote_signaled: 昇格要求の状態を保持するグローバルフラグ。

失敗モードと境界条件

  • シグナルハンドラ内であるため、非同期安全（async-signal-safe）な操作のみに限定されている。

保守担当者向けメモ

  • 本関数は pqsignal(SIGUSR2, StartupProcTriggerHandler) によってシグナルハンドラとして登録される。
  • 昇格要求の確認には IsPromoteSignaled() が使用されるため、本関数によるフラグ設定と対になるロジックを確認すること。
