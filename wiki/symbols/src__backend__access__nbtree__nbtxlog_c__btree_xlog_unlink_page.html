btree_xlog_unlink_page

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtxlog.c:801

目的

B-tree インデックスにおけるページ削除（unlink）操作の WAL リプレイを実行し、物理的なページ構造の再構築とリンクの修正を行う。

シグネチャまたは形状

static void btree_xlog_unlink_page(uint8 info, XLogReaderState *record)

入力と出力

  • info: WAL レコードのサブタイプ（メタページ更新の有無を判定）。
  • record: リプレイ対象の WAL レコードデータ。
  • 出力: なし（副作用としてバッファの内容を更新）。

前提条件と不変条件

  • xl_btree_unlink_page 構造体に含まれる leftsib、rightsib、level 等の情報が有効であること。
  • ページ削除の対象がリーフページ（level == 0）またはレベル1の場合、leaftopparent は無効であること。
  • ページロックは leftsib -> target -> rightsib の順序で取得されること。

アルゴリズムまたは振る舞い

1. **左兄弟のリンク修正**: leftsib が存在する場合、その btpo_next を rightsib に更新する。 2. **ターゲットページの初期化**: ターゲットページを空の削除済みページとして再初期化し、btpo_prev/next を設定して削除フラグ（safexid）を付与する。 3. **右兄弟のリンク修正**: rightbuf の btpo_prev を leftsib に更新する。 4. **兄弟ページのロック解放**: 修正完了後、leftbuf および rightbuf を解放する。 5. **親ページの更新（オプション）**: WAL レコードにブロック参照（ID: 3）が含まれる場合、削除されたサブツリーの親ページを再構築し、ダミーのハイキー（leaftopparent を含む）を追加する。 6. **メタページ更新**: info が XLOG_BTREE_UNLINK_PAGE_META の場合、_bt_restore_meta を呼び出してメタページを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_redo（src/backend/access/nbtree/nbtxlog.c 内でディスパッチされる）。
  • **呼び出し先**:
  • XLogRecGetData, XLogReadBufferForRedo, XLogInitBufferForRedo, XLogRecHasBlockRef
  • _bt_pageinit, _bt_restore_meta
  • BufferGetPage, BTPageGetOpaque, BTPageSetDeleted, PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer
  • PageAddItem, BTreeTupleSetTopParent

関連データ構造

  • xl_btree_unlink_page: WAL レコードのデータ構造。
  • BTPageOpaque: B-tree ページヘッダーのメタデータ。
  • Buffer: PostgreSQL のバッファ管理構造。

失敗モードと境界条件

  • PageAddItem が失敗した場合、elog(ERROR) を発生させる。
  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、そのバッファに対する更新はスキップされる。

保守担当者向けメモ

  • 本関数はリカバリ時の物理的なページ状態復元に特化している。
  • ページロックの順序はデッドロック回避のため厳密に守る必要がある。
  • leaftopparent を含むダミーハイキーの追加は、削除対象がリーフページそのものではなく、その親ページである場合にのみ発生する。
