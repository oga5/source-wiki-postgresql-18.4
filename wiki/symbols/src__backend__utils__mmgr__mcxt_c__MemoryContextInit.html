MemoryContextInit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/mcxt.c:344

目的

PostgreSQL のメモリ管理システム全体を初期化し、ルートとなるメモリコンテキストおよびエラー処理用コンテキストを構築する。

シグネチャまたは形状

void MemoryContextInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数を初期化）

前提条件と不変条件

  • TopMemoryContext が NULL であること（二重初期化の防止）。
  • 本関数はメモリ管理システムが利用可能になる前の、システム起動の極めて初期段階で呼び出される必要がある。

アルゴリズムまたは振る舞い

1. **ルートコンテキストの作成**: AllocSetContextCreate を使用して TopMemoryContext を生成する。これはすべてのメモリコンテキストの親となる。 2. **デフォルトコンテキストの設定**: CurrentMemoryContext を TopMemoryContext に設定する。 3. **エラーコンテキストの作成**: ErrorContext を生成する。これは常に 8KB のメモリを保持し、クリティカルセクション内での割り当てを許可するように設定される。 4. **完了通知**: ErrorContext の初期化をもってメモリ管理システムが稼働可能とみなされる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/main/main.c:114
  • 呼び出し先: AllocSetContextCreate, MemoryContextAllowInCriticalSection

関連データ構造

  • TopMemoryContext (グローバル変数)
  • CurrentMemoryContext (グローバル変数)
  • ErrorContext (グローバル変数)
  • AllocSetContext (コンテキストの実装型)

失敗モードと境界条件

  • TopMemoryContext が既に初期化されている場合、Assert により停止する。
  • ErrorContext の初期化は本関数の最後に行う必要がある。これは elog.c が ErrorContext が非 NULL であることを前提としてメモリ管理を行うためである。

保守担当者向けメモ

  • 本関数はシステム起動時に一度だけ実行されることを想定している。
  • ErrorContext は PANIC 発生時など、他のメモリが枯渇した状況でもエラー報告を行えるよう設計されているため、設定変更には細心の注意を払うこと。
