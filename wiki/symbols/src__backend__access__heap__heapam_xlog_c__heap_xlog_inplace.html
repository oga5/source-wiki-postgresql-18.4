heap_xlog_inplace

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_xlog.c:1133

目的

WALリプレイ時に、ヒープページ上のタプルデータをインプレース（直接）更新し、関連する無効化メッセージを処理する。

シグネチャまたは形状

static void heap_xlog_inplace(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファの更新および無効化メッセージの処理を行う）。

前提条件と不変条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返す場合、対象ページはロックされており、更新可能であること。
  • 更新対象のタプル長（oldlen）とWALレコード内のデータ長（newlen）が一致していること。一致しない場合は PANIC を発生させる。

アルゴリズムまたは振る舞い

1. XLogReadBufferForRedo を呼び出し、対象バッファを取得する。 2. BLK_NEEDS_REDO が返された場合:

  • ページ内の指定されたオフセット（xlrec->offnum）からタプルヘッダーを取得する。
  • タプルのデータ部分（t_hoff 以降）を、WALレコード内のデータで memcpy により上書きする。
  • PageSetLSN でページのLSNを更新し、MarkBufferDirty でバッファをダーティにする。

    • 3. バッファが有効であれば、ロックを解除しバッファを解放する。 4. 最後に ProcessCommittedInvalidationMessages を呼び出し、WALレコードに含まれる無効化メッセージを処理する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_xlog.c 内のディスパッチャ（heap2_redo 等から呼び出されることが示唆されている）。
  • **呼び出し先**: XLogReadBufferForRedo, PageGetMaxOffsetNumber, PageGetItemId, PageGetItem, ItemIdGetLength, PageSetLSN, MarkBufferDirty, BufferIsValid, UnlockReleaseBuffer, ProcessCommittedInvalidationMessages。

関連データ構造

  • xl_heap_inplace: WALレコードのデータ構造体。
  • XLogReaderState: WALレコード読み取り状態。
  • Buffer, Page, ItemId, HeapTupleHeader: ヒープページ操作に関連する型。

失敗モードと境界条件

  • PageGetMaxOffsetNumber(page) < offnum または !ItemIdIsNormal(lp) の場合: PANIC を発生させる。
  • oldlen != newlen の場合: PANIC を発生させる。

保守担当者向けメモ

  • 本関数は物理的なデータ更新と論理的な無効化メッセージ処理を混在させている。
  • ProcessCommittedInvalidationMessages はバッファのロック状態に関わらず実行される。
  • 新しいWALレコードタイプを追加する際は、heap_mask での適切なマスク処理を検討すること。
