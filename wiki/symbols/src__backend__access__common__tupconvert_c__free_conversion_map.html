free_conversion_map

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tupconvert.c:298

目的

TupleConversionMap 構造体およびその内部で確保されたワークスペース用メモリを解放する。

シグネチャまたは形状

void free_conversion_map(TupleConversionMap *map)

入力と出力

  • 入力: TupleConversionMap *map （解放対象の変換マップへのポインタ）
  • 出力: なし

前提条件と不変条件

  • 引数 map は有効な TupleConversionMap 構造体を指している必要がある。
  • 構造体内の indesc および outdesc は本関数では解放されない（これらは本モジュールの管理外である）。

アルゴリズムまたは振る舞い

1. free_attrmap(map->attrMap) を呼び出し、属性マッピング情報を解放する。 2. pfree() を使用して、変換作業用ワークスペースである invalues, inisnull, outvalues, outisnull の各配列を順次解放する。 3. 最後に map 構造体自体を pfree() で解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/analyze.c:1608
  • src/backend/executor/tstoreReceiver.c:218
  • 呼び出し先:
  • free_attrmap
  • pfree

関連データ構造

  • TupleConversionMap: 変換に必要な属性マッピング情報とワークスペースを保持する構造体。

失敗モードと境界条件

  • map が NULL の場合、pfree の仕様に依存する（通常はセグメンテーション違反を引き起こす可能性があるため、呼び出し側で NULL チェックを行うことが推奨される）。

保守担当者向けメモ

  • 本関数は TupleConversionMap のメモリ管理のみを担当する。TupleDesc 等の参照先オブジェクトは解放しないため、メモリリークを防ぐには呼び出し側で適切な生存期間管理を行うこと。
