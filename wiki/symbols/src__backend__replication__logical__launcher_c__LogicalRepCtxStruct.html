LogicalRepCtxStruct

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/launcher.c:56

目的

論理レプリケーションのランチャープロセスおよびワーカーの状態を共有メモリ上で管理するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct LogicalRepCtxStruct
{
    pid_t       launcher_pid;
    dsa_handle  last_start_dsa;
    dshash_table_handle last_start_dsh;
    LogicalRepWorker workers[FLEXIBLE_ARRAY_MEMBER];
} LogicalRepCtxStruct;

入力と出力

  • 入力: 共有メモリ領域の確保時に sizeof(LogicalRepCtxStruct) を用いてメモリサイズが決定されます。
  • 出力: 構造体内の各フィールドを通じて、ランチャープロセスの PID、ワーカーの起動時刻管理用ハッシュテーブル、およびワーカーの配列状態が保持されます。

前提条件と不変条件

  • workers フィールドは FLEXIBLE_ARRAY_MEMBER を使用しており、構造体の末尾に可変長のワーカー配列が配置されます。
  • 共有メモリ上に配置される前提の構造体であり、アクセスには適切なロック（LogicalRepWorkerLock）が必要です。

アルゴリズムまたは振る舞い

  • ランチャープロセスの PID を保持し、監視対象のワーカー群を管理します。
  • last_start_dsa および last_start_dsh を介して、サブスクリプションごとのワーカー起動時刻を共有ハッシュテーブルで管理し、再起動間隔の制御を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/launcher.c 内の共有メモリ初期化処理（ApplyLauncherShmemInit）およびサイズ計算処理にて参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • LogicalRepWorker: workers 配列の要素として含まれるワーカーの内部状態構造体。
  • LauncherLastStartTimesEntry: last_start_dsh ハッシュテーブルで管理されるエントリ構造体。

失敗モードと境界条件

  • 共有メモリの確保サイズが不足している場合、LogicalRepCtx へのキャストやアクセス時にメモリ破壊が発生する可能性があります。
  • FLEXIBLE_ARRAY_MEMBER を含むため、構造体自体のサイズ計算には注意が必要です。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、フィールドの追加や変更を行う際は、共有メモリのレイアウトおよびアライメントに十分注意してください。
  • LogicalRepCtx グローバルポインタを通じてアクセスされます。
