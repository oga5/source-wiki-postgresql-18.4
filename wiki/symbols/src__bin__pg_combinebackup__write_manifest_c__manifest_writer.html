manifest_writer

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_combinebackup/write_manifest.c:27

目的

バックアップマニフェストファイルを生成・書き出す際の状態を保持する構造体です。ファイル記述子、バッファリング用のメモリ領域、およびマニフェスト全体の整合性を保証するためのチェックサムコンテキストを管理します。

シグネチャまたは形状

struct manifest_writer
{
    char        pathname[MAXPGPATH];
    int         fd;
    StringInfoData buf;
    bool        first_file;
    bool        still_checksumming;
    pg_checksum_context manifest_ctx;
};

入力と出力

  • **入力**: create_manifest_writer() によって初期化され、add_file_to_manifest() を通じてファイル情報が逐次追加されます。
  • **出力**: flush_manifest() を介してディスクへ書き出され、最終的に finalize_manifest() によってマニフェスト全体のチェックサムが計算・記録されます。

前提条件と不変条件

  • buf は StringInfoData 型であり、マニフェストの JSON 内容を一時的に保持します。
  • manifest_ctx は、マニフェストファイル全体の整合性を保護するための SHA-256 等のチェックサム計算状態を保持します。
  • first_file フラグは、JSON 配列の区切り文字（カンマ）を制御するために使用されます。

アルゴリズムまたは振る舞い

1. **バッファリング**: add_file_to_manifest で追加されたデータは buf に蓄積されます。 2. **フラッシュ**: バッファが一定サイズ（128KB）に達すると flush_manifest が呼び出され、ディスクへ書き出されると同時に manifest_ctx が更新されます。 3. **終了処理**: finalize_manifest により、残りのバッファ内容の書き出し、WAL レンジの追記、および最終的なチェックサムの計算が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_combinebackup/pg_combinebackup.c
  • src/bin/pg_combinebackup/backup_label.c
  • **呼び出し先**:
  • flush_manifest (内部関数)
  • create_manifest_writer (初期化)
  • add_file_to_manifest (データ追加)
  • finalize_manifest (終了処理)

関連データ構造

  • StringInfoData: マニフェストの JSON 文字列を構築するための可変長バッファ。
  • pg_checksum_context: マニフェスト全体の整合性保護に使用するチェックサム計算コンテキスト。

失敗モードと境界条件

  • pathname が MAXPGPATH を超える場合、パスの切り詰めやエラーが発生する可能性があります。
  • fd が無効な場合、ディスクへの書き込み（flush_manifest）が失敗します。
  • 非 UTF-8 のファイルパスが渡された場合、Encoded-Path として 16 進数エンコードされる必要があります。

保守担当者向けメモ

  • 本構造体はフロントエンドツール用であり、バックエンドの AddFileToBackupManifest とは異なるバッファリング戦略をとっています。
  • flush_manifest を呼び出す際は、必ず manifest_ctx への反映がディスク書き出しと同期していることを確認してください。
