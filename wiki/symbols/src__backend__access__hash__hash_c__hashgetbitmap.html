hashgetbitmap

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:341

目的

ハッシュインデックススキャンにおいて、条件に一致するすべてのタプルを一度に取得し、TIDBitmap に格納する。

シグネチャまたは形状

int64 hashgetbitmap(IndexScanDesc scan, TIDBitmap *tbm)

入力と出力

  • **入力**:
  • scan: インデックススキャンの状態を保持する IndexScanDesc 構造体。
  • tbm: 取得したタプルの TID を格納するための TIDBitmap。
  • **出力**:
  • int64: ビットマップに追加されたタプルの総数。

前提条件と不変条件

  • scan は有効なハッシュインデックススキャン状態を指している必要がある。
  • _hash_first および _hash_next が、scan->ignore_killed_tuples が真の場合に不要なインデックスエントリを排除する処理を担うため、本関数内での追加処理は不要である。

アルゴリズムまたは振る舞い

1. _hash_first を呼び出し、スキャンを開始して最初のタプルを検索する。 2. タプルが見つかる限りループを継続する。

  • 現在のインデックススキャン位置から heapTid を取得する。
  • tbm_add_tuples を使用して、取得した heapTid を TIDBitmap に追加する。
  • カウンタ ntids をインクリメントする。
  • _hash_next を呼び出し、次のタプルへ移動する。

    • 3. すべてのタプルを走査した後、ntids を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hash.c 内の amroutine 構造体（amgetbitmap コールバックとして登録）。
  • **呼び出し先**:
  • _hash_first
  • _hash_next
  • tbm_add_tuples

関連データ構造

  • IndexScanDesc: スキャン状態を管理する標準的な構造体。
  • HashScanOpaque: ハッシュインデックス固有のスキャン状態（scan->opaque）。
  • HashScanPosItem: スキャン位置におけるアイテム情報。
  • TIDBitmap: タプル ID の集合を保持するビットマップ構造。

失敗モードと境界条件

  • スキャン対象のインデックスが空の場合、ループは実行されず 0 が返される。
  • _hash_first または _hash_next が失敗（タプルなし）した場合はループが終了し、それまでに蓄積された ntids が返される。

保守担当者向けメモ

  • 本関数は、ハッシュインデックスの「損失あり（Lossy）」という特性を考慮し、インデックスエントリをビットマップに集約する役割を担う。
  • _hash_first および _hash_next への依存が強いため、スキャンロジックの変更時はこれらの関数との整合性に注意すること。
