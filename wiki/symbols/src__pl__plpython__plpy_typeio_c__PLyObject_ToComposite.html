PLyObject_ToComposite

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_typeio.c:943

目的

PythonオブジェクトをPostgreSQLの複合型（Composite Type）に対応する Datum へ変換します。

シグネチャまたは形状

static Datum PLyObject_ToComposite(PLyObToDatum *arg, PyObject *plrv, bool *isnull, bool inarray)

入力と出力

  • **入力**:
  • arg: 変換設定とキャッシュ情報を保持する PLyObToDatum 構造体。
  • plrv: 変換対象のPythonオブジェクト。
  • isnull: NULL値を通知するための出力用フラグ。
  • inarray: 配列の一部として処理されているかを示すフラグ。
  • **出力**: 変換された Datum 値。

前提条件と不変条件

  • plrv が Py_None の場合、isnull を true に設定し、0 を返します。
  • 複合型（RECORDOID 以外）の場合、arg->u.tuple.tupdescid とキャッシュされた tupDesc_identifier を比較し、不一致があれば PLy_output_setup_tuple を呼び出してキャッシュを更新します。
  • 変換完了後、取得した TupleDesc は ReleaseTupleDesc により解放されます。

アルゴリズムまたは振る舞い

1. **NULLチェック**: plrv が Py_None なら即座に終了します。 2. **文字列変換**: plrv が PyUnicode 型であれば、PLyUnicode_ToComposite を呼び出して処理を委譲します。 3. **型記述子の取得**:

  • RECORDOID 以外: lookup_rowtype_tupdesc で型情報を取得し、キャッシュの整合性を確認・更新します。
  • RECORDOID: arg->u.tuple.recdesc を使用し、必要に応じて取得およびピン留めを行います。

    • 4. **変換実行**: Pythonオブジェクトの型に応じて以下のいずれかを呼び出します。

  • PySequence_Check: PLySequence_ToComposite
  • PyMapping_Check: PLyMapping_ToComposite
  • その他: PLyGenericObject_ToComposite

    • 5. **後処理**: ReleaseTupleDesc を呼び出し、戻り値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_typeio.c 内の変換セットアップ処理（arg->func に代入される形で参照）。
  • **呼び出し先**: PLyUnicode_ToComposite, lookup_rowtype_tupdesc, PLy_output_setup_tuple, PinTupleDesc, PLySequence_ToComposite, PLyMapping_ToComposite, PLyGenericObject_ToComposite, ReleaseTupleDesc。

関連データ構造

  • PLyObToDatum: 変換ロジックと型情報を保持する構造体。
  • TupleDesc: PostgreSQLの行型記述子。

失敗モードと境界条件

  • plrv が適切な型（シーケンス、マッピング、または属性アクセス可能なオブジェクト）でない場合、各変換関数内でエラーが発生する可能性があります。
  • RECORDOID の場合、recdesc が未設定であれば動的に取得を試みます。

保守担当者向けメモ

  • 本関数は再帰的な型変換の一部として機能します。メモリコンテキストの管理については、呼び出し元の PLy_output_convert 等の最上位レベルで制御されていることを前提としています。
  • RECORDOID は実行時に型が決定するため、キャッシュの扱いに注意してください。
