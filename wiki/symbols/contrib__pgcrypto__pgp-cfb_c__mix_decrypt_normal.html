mix_decrypt_normal

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-cfb.c:104

目的

PGP CFB（Cipher Feedback）モードにおける、通常の復号処理（再同期なし）を実行します。暗号化された入力データと、事前に計算されたフィードバックレジスタの暗号化結果を XOR 演算し、平文を生成します。

シグネチャまたは形状

static int mix_decrypt_normal(PGP_CFB *ctx, const uint8 *data, int len, uint8 *dst)

入力と出力

  • ctx: 暗号化コンテキスト（PGP_CFB 構造体）。現在の処理位置 pos および暗号化済みフィードバックバッファ fre を保持。
  • data: 復号対象の入力データ（暗号文）。
  • len: 処理するデータの長さ。
  • dst: 復号結果（平文）を格納するバッファ。
  • 戻り値: 処理したバイト数（len を返す）。

前提条件と不変条件

  • ctx->fre には、現在のブロックに対する暗号化済みのフィードバック値が格納されている必要があります。
  • ctx->pos は、現在のブロック内でのオフセットを指しており、処理中に更新されます。

アルゴリズムまたは振る舞い

1. ctx->pos から ctx->pos + len までの範囲でループを実行します。 2. 入力データ data の各バイトを ctx->encbuf に保存します。 3. ctx->fre[i]（暗号化済みフィードバック）と ctx->encbuf[i]（入力データ）の XOR を取り、結果を dst に書き込みます。 4. 処理終了後、ctx->pos を len 分進めます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-cfb.c 内の pgp_cfb_decrypt 等から、mix_data 関数ポインタを介して間接的に呼び出されます（ctx->resync が偽の場合）。
  • 呼び出し先: 本関数内から直接呼び出される関数はありません。

関連データ構造

  • PGP_CFB: 暗号化コンテキスト。pos（現在の位置）、encbuf（入力バッファ）、fre（暗号化済みフィードバックレジスタ）を管理します。

失敗モードと境界条件

  • len が ctx->pos との合計でバッファサイズを超過する場合の安全性については、本関数単体ではチェックを行っていません。呼び出し元での適切な長さ管理が必要です。

保守担当者向けメモ

  • 本関数は「通常」の復号用です。古い PGP 仕様で必要な特殊な再同期処理が必要な場合は、mix_decrypt_resync が使用されます。
  • ctx->pos の更新は本関数で行われますが、ブロックの境界判定や ctx->fre の再計算は呼び出し元の cfb_process が担います。
