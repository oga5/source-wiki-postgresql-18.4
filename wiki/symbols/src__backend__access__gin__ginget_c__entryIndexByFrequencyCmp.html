entryIndexByFrequencyCmp

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginget.c:489

目的

GINインデックススキャンにおいて、検索キー（GinScanKey）に含まれる各エントリーを、推定される結果件数（predictNumberResult）に基づいて昇順にソートするための比較関数です。

シグネチャまたは形状

static int entryIndexByFrequencyCmp(const void *a1, const void *a2, void *arg)

入力と出力

  • **入力**:
  • a1, a2: ソート対象となるエントリーインデックス（int *）へのポインタ。
  • arg: 比較対象のエントリーを保持する GinScanKey 構造体へのポインタ。
  • **出力**:
  • n1 < n2 の場合 -1
  • n1 == n2 の場合 0
  • n1 > n2 の場合 1

前提条件と不変条件

  • arg は GinScanKey 型としてキャスト可能であること。
  • a1 および a2 が指すインデックス値は、key->scanEntry 配列の有効な範囲内であること。
  • 各 scanEntry の predictNumberResult フィールドが初期化済みであること。

アルゴリズムまたは振る舞い

1. arg から GinScanKey を取得します。 2. a1 と a2 が指す整数値をインデックスとして、key->scanEntry から対応するエントリーの predictNumberResult を参照します。 3. 取得した2つの頻度値を比較し、昇順となるように整数値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginget.c:559（qsort の比較関数として使用）。
  • **呼び出し先**: なし（標準的な比較ロジックのみ）。

関連データ構造

  • GinScanKey: GINスキャンにおける検索キーの状態を保持する構造体。
  • GinScanEntry: 個別の検索エントリーの統計情報（predictNumberResult を含む）を保持する構造体。

失敗モードと境界条件

  • arg が NULL の場合、または a1, a2 が指すインデックスが範囲外の場合はセグメンテーションフォールトが発生します。

保守担当者向けメモ

  • 本関数は qsort で使用されるため、比較ロジックの変更はソート順序に直接影響します。
  • 頻度の低いエントリーを先に処理することで検索効率を最適化する目的があるため、このソート順序はパフォーマンスに直結します。
