libpqsrv_connect_prepare

定義

  • 種別: function
  • 定義箇所: src/include/libpq/libpq-be-fe-helpers.h:130

目的

PostgreSQL バックエンドプロセスにおいて、外部データベース接続に使用するファイル記述子（FD）を安全に確保し、バックエンドの FD 管理制限を遵守させる。

シグネチャまたは形状

static inline void libpqsrv_connect_prepare(void)

入力と出力

  • 入力: なし
  • 出力: なし（成功時は正常終了、失敗時は ereport(ERROR, ...) により例外をスロー）

前提条件と不変条件

  • PGconn オブジェクトが 1 つの長寿命な FD を消費することを前提としている。
  • AcquireExternalFD() を呼び出すことで、バックエンドの FD 制限を遵守し、必要に応じて仮想ファイル記述子（VFD）をクローズして空きを確保する。

アルゴリズムまたは振る舞い

1. AcquireExternalFD() を呼び出し、外部接続用の FD 確保を試みる。 2. 確保に失敗した場合、ereport(ERROR) を発行し、接続確立失敗の旨を通知する。 3. エラーメッセージには、サーバーの max_files_per_process 設定や ulimit -n の引き上げを推奨するヒントが含まれる（OS に応じてメッセージが分岐する）。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/include/libpq/libpq-be-fe-helpers.h 内の接続関連関数（libpqsrv_connect 等）から呼び出される。
  • 呼び出し先:
  • AcquireExternalFD() (バックエンドの FD 管理関数)
  • ereport() (エラー報告用マクロ)

関連データ構造

  • なし（本関数は特定のデータ構造を操作せず、システムリソースの予約のみを行う）。

失敗モードと境界条件

  • **FD 不足**: AcquireExternalFD() が偽を返した場合、ERRCODE_SQLCLIENT_UNABLE_TO_ESTABLISH_SQLCONNECTION エラーが発生し、処理が中断される。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル経由でインライン展開される。
  • ereport 内で #if を使用できないため、WIN32 の有無で条件分岐を行っている。変更時はこの構造を維持すること。
