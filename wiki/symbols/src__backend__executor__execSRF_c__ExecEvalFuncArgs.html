ExecEvalFuncArgs

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execSRF.c:833

目的

関数呼び出しに必要な引数リストを評価し、FunctionCallInfo 構造体に格納する。

シグネチャまたは形状

static void ExecEvalFuncArgs(FunctionCallInfo fcinfo, List *argList, ExprContext *econtext)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報を保持する FunctionCallInfo 構造体。
  • argList: 評価対象の式（ExprState）のリスト。
  • econtext: 式評価に使用するコンテキスト（ExprContext）。
  • **出力**: なし（fcinfo->args 配列を直接更新する）。

前提条件と不変条件

  • argList 内の各要素は ExprState 型としてキャスト可能であること。
  • 評価された引数の総数が fcinfo->nargs と一致すること（Assert により検証）。

アルゴリズムまたは振る舞い

1. argList を順次走査する。 2. 各 ExprState に対して ExecEvalExpr を呼び出し、式を評価する。 3. 評価結果（値および NULL フラグ）を fcinfo->args[i] に格納する。 4. 全ての引数処理終了後、処理した引数の数が fcinfo->nargs と等しいことを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execSRF.c 内の関数（ExecMakeFunctionResultSet 等から呼び出される）。
  • **呼び出し先**:
  • ExecEvalExpr: 式の評価を実行する。

関連データ構造

  • FunctionCallInfo: 関数呼び出しの引数や戻り値を管理する構造体。
  • ExprState: 評価対象の式ノード。
  • ExprContext: 式評価時の実行コンテキスト。

失敗モードと境界条件

  • argList の要素数と fcinfo->nargs が一致しない場合、Assert が失敗する（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は execSRF.c 内で SRF 実行時の引数準備を担うユーティリティである。
  • ExecEvalExpr を介して式を評価するため、メモリコンテキストの管理は呼び出し元（econtext の設定）に依存する。
