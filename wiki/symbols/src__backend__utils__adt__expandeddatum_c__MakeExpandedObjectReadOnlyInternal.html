MakeExpandedObjectReadOnlyInternal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandeddatum.c:94

目的

展開済みオブジェクト（Expanded Object）の読み取り書き込み可能（R/W）なポインタを、読み取り専用（R/O）のポインタへ安全に変換します。

シグネチャまたは形状

Datum MakeExpandedObjectReadOnlyInternal(Datum d)

入力と出力

  • **入力**: Datum d（展開済みオブジェクトのポインタを含む Datum）。
  • **出力**: 変換後の Datum。入力が既に R/O であるか、展開済みオブジェクトでない場合は入力値をそのまま返します。

前提条件と不変条件

  • 入力 d が展開済みオブジェクトのポインタである場合、そのオブジェクトは ExpandedObjectHeader を保持している必要があります。
  • 変換後は、元の R/W ポインタを介した変更操作は行われません。

アルゴリズムまたは振る舞い

1. VARATT_IS_EXTERNAL_EXPANDED_RW マクロを使用して、入力 d が R/W 展開済みオブジェクトポインタであるかを確認します。 2. R/W でない場合は、入力 d をそのまま返します。 3. R/W である場合、DatumGetEOHP(d) を呼び出して ExpandedObjectHeader を取得します。 4. EOHPGetRODatum(eohptr) を呼び出し、ヘッダー内にあらかじめ保持されている読み取り専用ポインタを返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c
  • src/backend/jit/llvm/llvmjit_expr.c
  • src/include/utils/expandeddatum.h
  • **呼び出し先**:
  • DatumGetEOHP
  • EOHPGetRODatum

関連データ構造

  • ExpandedObjectHeader: 展開済みオブジェクトの共通ヘッダー。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • 入力が展開済みオブジェクトではない場合、または既に R/O である場合は、副作用なく入力値を返します。
  • DatumGetEOHP が無効なポインタを指している場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は、実行時式評価（execExprInterp.c）や JIT コンパイルされた式において、値の不変性を保証するために使用されます。
  • R/W から R/O への変換は、オブジェクトのメモリコンテキストを維持したままポインタの参照先を切り替える軽量な操作です。
