report_clusters_compatible

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/check.c:772

目的

アップグレード対象の旧クラスタと新クラスタの互換性検証が完了したことをユーザーに通知し、実行モードに応じて後続処理または終了処理を行う。

シグネチャまたは形状

void report_clusters_compatible(void)

入力と出力

  • 入力: なし（グローバル変数 user_opts を参照）
  • 出力: なし（標準出力へのログ出力およびプロセス終了）

前提条件と不変条件

  • 本関数が呼び出される時点で、クラスタ間の互換性チェックが正常に完了している必要がある。

アルゴリズムまたは振る舞い

1. user_opts.check が真の場合（チェックのみのモード）:

  • 「*Clusters are compatible*」というメッセージをログ出力する。
  • stop_postmaster(false) を呼び出し、新クラスタのポストマスターを停止する。
  • cleanup_output_dirs() を実行して一時ディレクトリをクリーンアップする。
  • exit(0) でプロセスを正常終了する。

    • 2. user_opts.check が偽の場合（実際のアップグレード実行モード）:

  • 以降の処理で失敗した場合に initdb の再実行が必要であることを示す警告メッセージをログ出力する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_upgrade/pg_upgrade.c:143
  • 呼び出し先:
  • stop_postmaster
  • cleanup_output_dirs
  • pg_log

関連データ構造

  • user_opts: pg_upgrade の実行オプションを保持するグローバル構造体。

失敗モードと境界条件

  • user_opts.check が真の場合、本関数は呼び出し元に戻らずプロセスを終了させる。

保守担当者向けメモ

  • 本関数は pg_upgrade の検証フェーズの最終段階で呼び出されます。チェックモードと実行モードで挙動が大きく異なるため、変更時は両方のパスを確認してください。
