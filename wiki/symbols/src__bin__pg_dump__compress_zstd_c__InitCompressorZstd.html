InitCompressorZstd

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_zstd.c:23

目的

CompressorState を Zstandard (Zstd) 圧縮用に初期化するためのエントリポイントです。

シグネチャまたは形状

void InitCompressorZstd(CompressorState *cs, const pg_compress_specification compression_spec)

入力と出力

  • **入力**:
  • cs: 初期化対象となる CompressorState 構造体へのポインタ。
  • compression_spec: 圧縮設定を保持する pg_compress_specification 構造体。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • 本関数は USE_ZSTD マクロが定義されていない環境（Zstd ライブラリが利用不可なビルド）で呼び出されます。

アルゴリズムまたは振る舞い

  • 呼び出された際、常に pg_fatal を実行してエラーメッセージを出力し、プログラムを終了させます。Zstd 圧縮がサポートされていないビルドであることを通知します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_io.c:139 からの呼び出しが確認されています。
  • **呼び出し先**: pg_fatal（PostgreSQL のエラー報告関数）。

関連データ構造

  • CompressorState: 圧縮処理の状態を管理する構造体。
  • pg_compress_specification: 圧縮アルゴリズムやレベル等の設定を保持する構造体。

失敗モードと境界条件

  • Zstd がサポートされていない環境では、本関数が呼び出されると必ず失敗（pg_fatal によるプロセス終了）します。

保守担当者向けメモ

  • 本関数はスタブとして実装されています。Zstd を有効にするには、ビルド環境で USE_ZSTD を定義し、適切なライブラリをリンクする必要があります。
