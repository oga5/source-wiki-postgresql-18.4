QUERYTYPE

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int.h:147

目的

PostgreSQL の intarray モジュールにおいて、ブール検索クエリを保持するための可変長構造体です。ポーランド記法で表現されたクエリを格納し、インデックスや配列データとの照合に使用されます。

シグネチャまたは形状

typedef struct QUERYTYPE
{
    int32       vl_len_;    /* varlena header */
    int32       size;       /* ITEMの数 */
    ITEM        items[FLEXIBLE_ARRAY_MEMBER];
} QUERYTYPE;

入力と出力

  • **入力**: PostgreSQL の Datum として受け取ったクエリデータ（DatumGetQueryTypeP マクロ等で変換）。
  • **出力**: 構造体内の items 配列に格納された ITEM の集合。

前提条件と不変条件

  • vl_len_ は PostgreSQL の可変長データ（varlena）ヘッダーであり、直接操作してはなりません。
  • items は FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリ確保時には HDRSIZEQT マクロや適切な計算式を用いてサイズを算出する必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、_int_bool.c で実装されるブール検索ロジックの基盤となります。
  • signconsistent や execconsistent などの関数に渡され、配列データとの整合性チェックや、GIN インデックスを用いた検索実行に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: _int_bool.c 内の signconsistent, execconsistent, gin_bool_consistent 等が本構造体を引数として受け取ります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ITEM: クエリを構成する個々の要素。
  • ArrayType: execconsistent 等で比較対象となる PostgreSQL の配列型。
  • BITVECP: signconsistent で使用されるビットベクトル。

失敗モードと境界条件

  • 可変長構造体であるため、メモリ確保が不適切な場合、バッファオーバーランや不正なメモリ参照が発生する可能性があります。
  • Datum から変換する際は、PG_DETOAST_DATUM を介して適切にトースト解除を行う必要があります。

保守担当者向けメモ

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズは固定ではありません。メモリ割り当てやポインタ演算を行う際は、必ず HDRSIZEQT マクロを利用してください。
  • _int_bool.c にある関連関数群が本構造体の主な利用者です。変更を加える際は、これらの関数への影響を確認してください。
