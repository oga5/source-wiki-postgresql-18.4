before_shmem_exit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipc.c:343

目的

プロセス終了シーケンスの初期段階（共有メモリのデタッチ前）に実行されるクリーンアップ関数を登録する。

シグネチャまたは形状

void before_shmem_exit(pg_on_exit_callback function, Datum arg);

入力と出力

  • function: 終了時に呼び出されるコールバック関数へのポインタ。
  • arg: コールバック関数に渡される Datum 型の引数。
  • 戻り値: なし。

前提条件と不変条件

  • 登録可能なコールバックの最大数は MAX_ON_EXITS に制限されている。
  • コールバックは登録された順序とは逆（LIFO）に実行される。

アルゴリズムまたは振る舞い

1. 現在の登録数が MAX_ON_EXITS に達していないか確認する。達している場合は FATAL エラーを発生させる。 2. 指定された function と arg を before_shmem_exit_list の現在のインデックス位置に格納する。 3. before_shmem_exit_index をインクリメントする。 4. atexit_callback_setup フラグを確認し、未設定であれば atexit(atexit_callback) を呼び出してプロセス終了時のバックストップを有効化し、フラグを true に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c
  • src/backend/access/transam/parallel.c
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xlog.c
  • src/backend/catalog/namespace.c
  • src/backend/commands/async.c
  • src/backend/postmaster/auxprocess.c
  • **呼び出し先**:
  • atexit()（標準ライブラリ関数）
  • ereport()（エラー発生時）

関連データ構造

  • before_shmem_exit_list: コールバック関数と引数を保持する配列。
  • before_shmem_exit_index: 次の登録位置を示すインデックス。
  • atexit_callback_setup: atexit が既に設定済みかを示すフラグ。

失敗モードと境界条件

  • before_shmem_exit_index >= MAX_ON_EXITS: 登録スロットが不足している場合、FATAL エラーによりプロセスが終了する。

保守担当者向けメモ

  • 本関数で登録されたコールバックは、shmem_exit() が呼び出された際、動的共有メモリ（DSM）のシャットダウンや on_shmem_exit リストの処理よりも前に実行される。
  • コールバック内でエラーが発生した場合の再帰防止策として、実行前にリストから削除される設計となっている。
