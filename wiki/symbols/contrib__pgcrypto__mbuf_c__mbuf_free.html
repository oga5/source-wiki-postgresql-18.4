mbuf_free

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/mbuf.c:61

目的

MBuf 構造体によって管理されているメモリバッファを安全に解放します。

シグネチャまたは形状

int mbuf_free(MBuf *mbuf);

入力と出力

  • **入力**: mbuf (MBuf *) - 解放対象のバッファ構造体へのポインタ。
  • **出力**: int - 常に 0 を返します。

前提条件と不変条件

  • mbuf は有効なポインタである必要があります。
  • mbuf->data が指すメモリ領域は、PostgreSQL のメモリコンテキスト内で palloc されたものである必要があります。

アルゴリズムまたは振る舞い

1. mbuf->own_data が真の場合:

  • px_memset を使用して、バッファ内のデータ領域（mbuf->data から mbuf->buf_end まで）をゼロクリアします。
  • pfree を使用して、データ領域のメモリを解放します。

    • 2. pfree を使用して、mbuf 構造体自体のメモリを解放します。 3. 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pgsql.c 内の各関数（pgp_sym_encrypt_bytea 等の処理フロー）から、一時的なバッファを破棄するために呼び出されます。
  • **呼び出し先**: px_memset (メモリのゼロクリア用)、pfree (PostgreSQL のメモリ解放関数)。

関連データ構造

  • MBuf: 本関数が管理・解放を行う対象の構造体。

失敗モードと境界条件

  • mbuf が NULL の場合、pfree 呼び出し時にセグメンテーション違反が発生する可能性があります。
  • own_data が偽の場合、データ領域の解放はスキップされます。

保守担当者向けメモ

  • 本関数は pfree を使用しているため、PostgreSQL のメモリコンテキストの生存期間内で呼び出す必要があります。
  • データの機密性を保持するため、解放前に px_memset でバッファをゼロクリアする実装になっています。
