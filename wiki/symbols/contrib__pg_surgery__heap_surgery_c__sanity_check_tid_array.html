sanity_check_tid_array

定義

  • 種別: function
  • 定義箇所: contrib/pg_surgery/heap_surgery.c:372

目的

SQLから渡されたTID（タプル識別子）配列の妥当性を検証し、配列内の要素数を取得する。

シグネチャまたは形状

static void sanity_check_tid_array(ArrayType *ta, int *ntids)

入力と出力

  • **入力**:
  • ta: 検証対象の ArrayType ポインタ。
  • ntids: 要素数を格納するための整数ポインタ。
  • **出力**:
  • なし（ntids を介して要素数を返却）。検証失敗時はエラーをスローする。

前提条件と不変条件

  • 入力配列 ta は ArrayType 構造体として有効であること。
  • 配列内に NULL が含まれていないこと。
  • 配列は空であるか、1次元配列であること。

アルゴリズムまたは振る舞い

1. ARR_HASNULL(ta) および array_contains_nulls(ta) を確認し、NULL が含まれる場合は ERRCODE_NULL_VALUE_NOT_ALLOWED でエラーを報告する。 2. ARR_NDIM(ta) が 1 より大きい場合、ERRCODE_DATA_EXCEPTION でエラーを報告する。 3. ArrayGetNItems を使用して配列の総要素数を計算し、ntids に代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: heap_force_common（heap_surgery.c:104）から呼び出される。
  • **呼び出し先**: ArrayGetNItems（PostgreSQL 内部関数）を呼び出す。

関連データ構造

  • ArrayType: PostgreSQL の配列型を表現する構造体。

失敗モードと境界条件

  • **NULL 値**: 配列内に NULL が存在する場合、エラーとなる。
  • **多次元配列**: 2次元以上の配列が渡された場合、エラーとなる。

保守担当者向けメモ

  • 本関数は heap_force_kill および heap_force_freeze の前処理として使用される。
  • 配列の次元数や NULL の有無という基本的な制約を強制する役割を持つ。
