check_root

定義

  • 種別: function
  • 定義箇所: src/backend/main/main.c:442

目的

PostgreSQL サーバープロセスが特権ユーザー（root または Windows の管理者）として実行されることを防ぎ、セキュリティ上のリスクを回避する。

シグネチャまたは形状

static void check_root(const char *progname)

入力と出力

  • 入力: progname (実行ファイル名を示す文字列)
  • 出力: なし（正常時は復帰、異常時はエラーメッセージを標準エラー出力に出力し exit(1) でプロセスを終了する）

前提条件と不変条件

  • 非 Windows 環境では、実効ユーザー ID (geteuid) が 0 でないこと。
  • 非 Windows 環境では、実ユーザー ID (getuid) と実効ユーザー ID (geteuid) が一致していること。
  • Windows 環境では、pgwin32_is_admin() が偽を返すこと。

アルゴリズムまたは振る舞い

1. **プラットフォーム判定**: WIN32 マクロの有無により処理を分岐する。 2. **非 Windows 環境**:

  • geteuid() が 0（root）の場合、警告を出力して終了する。
  • getuid() != geteuid() の場合、setuid プログラムとしての実行を検知し、警告を出力して終了する。

    • 3. **Windows 環境**:

  • pgwin32_is_admin() を呼び出し、管理者権限で実行されている場合は警告を出力して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/main/main.c 内の main() 関数（195行目付近）。
  • 呼び出し先: geteuid(), getuid(), write_stderr(), exit(), pgwin32_is_admin()。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • root 権限または管理者権限で起動された場合、即座にプロセスを終了する。
  • 実ユーザー ID と実効ユーザー ID が不一致の場合、セキュリティ上の懸念からプロセスを終了する。

保守担当者向けメモ

  • 本関数は main() の初期化フェーズで呼び出される。
  • Windows 環境における管理者権限チェックは pgwin32_is_admin() に依存しているため、プラットフォーム固有の権限管理仕様に変更があった場合は同関数の実装を確認すること。
