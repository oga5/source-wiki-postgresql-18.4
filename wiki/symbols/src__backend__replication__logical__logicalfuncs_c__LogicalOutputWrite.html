LogicalOutputWrite

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logicalfuncs.c:61

目的

論理デコーディングの出力プラグインによって生成されたデータを、SQLインターフェースの戻り値である tuplestore に書き込むためのコールバック関数です。

シグネチャまたは形状

static void LogicalOutputWrite(LogicalDecodingContext *ctx, XLogRecPtr lsn, TransactionId xid, bool last_write);

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。ctx->out に出力データが格納されている。
  • lsn: 書き込むレコードのログシーケンス番号。
  • xid: 書き込むレコードに関連するトランザクションID。
  • last_write: 書き込みが最後かどうかを示すフラグ（本関数内では未使用）。
  • **出力**: なし（副作用として ctx->output_writer_private が指す DecodingOutputState 内の tuplestore にデータが追加される）。

前提条件と不変条件

  • ctx->out->len が MaxAllocSize - VARHDRSZ を超えてはならない。
  • テキスト出力の場合、ctx->out->data はデータベースのエンコーディングと一致している必要がある。

アルゴリズムまたは振る舞い

1. 出力データのサイズが許容範囲内か確認し、超過している場合はエラーを投げる。 2. ctx->output_writer_private から DecodingOutputState を取得する。 3. lsn と xid を Datum に変換する。 4. テキスト出力の場合は、データベースエンコーディングの妥当性を検証する。 5. ctx->out のデータを text 型（または bytea）の Datum に変換する。 6. tuplestore_putvalues を使用して、lsn、xid、および変換したデータ本体を tuplestore に格納する。 7. DecodingOutputState 内の returned_rows カウンタをインクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/logicalfuncs.c 内の論理デコーディングコンテキスト初期化処理（CreateDecodingContext への引数として登録される）。
  • **呼び出し先**:
  • tuplestore_putvalues
  • cstring_to_text_with_len
  • pg_verify_mbstr（テキスト出力時のみ）

関連データ構造

  • LogicalDecodingContext: デコーディングの実行状態を保持する構造体。
  • DecodingOutputState: 出力プラグインから受け取ったデータを tuplestore に変換・蓄積するための状態管理構造体。

失敗モードと境界条件

  • ctx->out->len が MaxAllocSize - VARHDRSZ を超える場合、elog(ERROR) により処理が中断される。
  • テキスト出力時にエンコーディングの検証に失敗した場合、Assert により異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は LogicalDecodingContext の write コールバックとして登録されることを前提としています。
  • cstring_to_text_with_len を使用して bytea データを処理する実装は、コード内のコメントにある通り、型変換の便宜上の手法です。
