FreePageManagerLargestContiguous

定義

  • 種別: static function
  • 定義箇所: src/backend/utils/mmgr/freepage.c:323

目的

FreePageManager が管理するメモリ領域において、現在利用可能な最大の連続ページ数を算出します。

シグネチャまたは形状

static Size FreePageManagerLargestContiguous(FreePageManager *fpm)

入力と出力

  • **入力**: fpm (管理対象の FreePageManager 構造体へのポインタ)
  • **出力**: 最大の連続ページ数を示す Size 型の値。

前提条件と不変条件

  • fpm は初期化済みであること。
  • fpm_segment_base(fpm) により、管理対象メモリのベースアドレスが正しく取得できること。

アルゴリズムまたは振る舞い

1. fpm->freelist の末尾（最大サイズのスパンを保持するリスト）を確認します。 2. 末尾のリストが空でない場合、そのリスト内の全スパンを走査し、candidate->npages の最大値を求めます。 3. 末尾のリストが空である場合、より小さいサイズのフリーリストを降順に検索し、最初に見つかった空でないリストのインデックス（f + 1）を最大連続ページ数として返します。 4. いずれのフリーリストも空の場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/freepage.c 内の Assert 文および fpm->contiguous_pages の更新処理（行 245, 371, 416）。
  • **呼び出し先**:
  • fpm_segment_base (マクロ)
  • relptr_is_null (マクロ)
  • relptr_access (マクロ)

関連データ構造

  • FreePageManager: メモリ管理のメイン構造体。
  • FreePageSpanLeader: 空き領域の先頭に配置されるヘッダ。

失敗モードと境界条件

  • 管理対象のメモリ領域が完全に枯渇している場合、0 を返します。
  • fpm->freelist がすべて空の場合、ループは f > 0 の条件で終了し、初期値の 0 が返されます。

保守担当者向けメモ

  • 本関数は fpm->contiguous_pages の整合性チェックや更新に使用されます。
  • FPM_NUM_FREELISTS に依存したループ処理を行うため、フリーリストの構成を変更する際は本関数のロジックも再検討が必要です。
