GetPubPartitionOptionRelations

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_publication.c:309

目的

論理レプリケーションにおいて、指定されたリレーション（テーブル）をパブリケーションに追加する際、パーティション階層を考慮して対象となるリレーションのリストを解決する。

シグネチャまたは形状

List *GetPubPartitionOptionRelations(List *result, PublicationPartOpt pub_partopt, Oid relid)

入力と出力

  • **入力**:
  • result: 既存のリレーションOIDリスト。
  • pub_partopt: パーティションの扱いを指定する列挙型（PUBLICATION_PART_ROOT, PUBLICATION_PART_ALL, PUBLICATION_PART_LEAF）。
  • relid: 対象となるリレーションのOID。
  • **出力**:
  • パーティション階層を解決した後のリレーションOIDリスト（List *）。

前提条件と不変条件

  • relid がパーティション化されたテーブルである場合、pub_partopt の値に応じて適切な階層のOIDがリストに追加される。
  • pub_partopt が PUBLICATION_PART_ROOT の場合、パーティション階層に関わらず relid 自体がリストに追加される。

アルゴリズムまたは振る舞い

1. relid がパーティション化されたテーブルであり、かつ pub_partopt が PUBLICATION_PART_ROOT 以外の場合：

  • find_all_inheritors を使用して全パーティションを取得する。
  • pub_partopt == PUBLICATION_PART_ALL の場合、全パーティションを result に結合する。
  • pub_partopt == PUBLICATION_PART_LEAF の場合、パーティションのうち、さらにパーティション化されていないもの（リーフパーティション）のみを result に追加する。

    • 2. 上記以外（パーティション化されていないテーブル、または PUBLICATION_PART_ROOT 指定時）の場合：

  • relid をそのまま result に追加する。

    • 3. 最終的な result リストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c 内の複数箇所（パブリケーション関連の処理）。
  • src/backend/commands/publicationcmds.c（CREATE PUBLICATION 等のDDLコマンド実行時）。
  • **呼び出し先**:
  • get_rel_relkind: リレーションの種類判定。
  • find_all_inheritors: パーティション階層の取得。
  • list_concat, lappend_oid: リスト操作。

関連データ構造

  • PublicationPartOpt: パブリケーションにおけるパーティションの公開オプションを定義する列挙型。
  • List: PostgreSQLの汎用リスト構造。

失敗モードと境界条件

  • pub_partopt に定義外の値が渡された場合、Assert(false) により停止する可能性がある。
  • find_all_inheritors は NoLock モードで呼び出される。

保守担当者向けメモ

  • 本関数は pubviaroot 設定に基づいたパブリケーションの挙動を制御する中核的なロジックである。
  • パーティション階層の変更や新しいパーティション種別の導入時には、find_all_inheritors の結果に対するフィルタリングロジックが適切か確認すること。
