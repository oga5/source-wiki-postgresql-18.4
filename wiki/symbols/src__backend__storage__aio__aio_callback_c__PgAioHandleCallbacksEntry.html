PgAioHandleCallbacksEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/aio/aio_callback.c:28

目的

非同期I/O（AIO）完了時に実行されるコールバック関数群とその識別用名称を保持するための構造体です。

シグネチャまたは形状

typedef struct PgAioHandleCallbacksEntry
{
	const PgAioHandleCallbacks *const cb;
	const char *const name;
} PgAioHandleCallbacksEntry;

入力と出力

  • **入力**:
  • cb: PgAioHandleCallbacks 構造体への定数ポインタ。
  • name: コールバックを識別するための文字列定数。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • cb および name は定数ポインタであり、初期化後に変更することはできません。
  • 本構造体は、aio_handle_cbs 配列の要素として静的に定義されることを前提としています。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持に特化しており、特定のアルゴリズムは持ちません。この構造体を参照する関数（pgaio_io_call_complete_shared 等）が、cb_id をインデックスとして aio_handle_cbs 配列から本構造体を取得し、格納されているコールバック関数を順次実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio_callback.c 内の各関数（pgaio_io_register_callbacks、pgaio_io_call_stage、pgaio_io_call_complete_shared、pgaio_io_call_complete_local など）が、aio_handle_cbs 配列を介して本構造体を参照しています。
  • **呼び出し先**:
  • 本構造体は関数ポインタ（cb）を保持しており、これを通じて実際のコールバック処理が実行されます。

関連データ構造

  • PgAioHandleCallbacks: 実際のコールバック関数ポインタを定義する構造体。
  • aio_handle_cbs: 本構造体のインスタンスを保持する静的配列。

失敗モードと境界条件

  • cb_id が aio_handle_cbs の範囲外である場合、不正なメモリ参照が発生する可能性があります。
  • cb が NULL である場合、呼び出し側のロジックで適切にハンドリングされない限り、実行時にクラッシュする可能性があります。

保守担当者向けメモ

  • 本構造体は aio_handle_cbs 配列の要素としてのみ使用されます。新しいコールバックタイプを追加する場合は、aio_handle_cbs に新しいエントリを追加してください。
  • name フィールドはデバッグやログ出力に使用されることを想定しています。
