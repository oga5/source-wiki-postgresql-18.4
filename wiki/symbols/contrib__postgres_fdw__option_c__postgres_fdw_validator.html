postgres_fdw_validator

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/option.c:70

目的

postgres_fdw における外部サーバー、外部テーブル、およびユーザーマッピングのオプション定義が、システムカタログの制約および型定義に適合しているかを検証する SQL レベルの検証関数です。

シグネチャまたは形状

Datum postgres_fdw_validator(PG_FUNCTION_ARGS)
  • 入力: PG_FUNCTION_ARGS（options_list と catalog OID を含む）
  • 出力: void（検証成功時）

入力と出力

  • **入力**:
  • PG_GETARG_DATUM(0): untransformRelOptions でリスト化されたオプションのリスト。
  • PG_GETARG_OID(1): オプションが適用される対象のカタログ OID。
  • **出力**: 成功時は PG_RETURN_VOID()。検証失敗時は ereport(ERROR, ...) によりトランザクションを中断。

前提条件と不変条件

  • InitPgFdwOptions() が呼び出され、内部オプションリストが初期化されていること。
  • 各オプションは、指定された catalog OID に対して許可されている必要がある。

アルゴリズムまたは振る舞い

1. untransformRelOptions で入力オプションをリスト化する。 2. InitPgFdwOptions() を呼び出し、検証用オプションリストを構築する。 3. リスト内の各オプションに対し、is_valid_option で存在確認を行う。不明なオプションの場合は ClosestMatchState を用いて類似の有効なオプションをヒントとして提示しエラー終了する。 4. オプション名に応じて以下の検証を行う：

  • **Boolean型**: defGetBoolean を使用。
  • **浮動小数点型**: parse_real を使用し、0以上であることを確認。
  • **整数型**: parse_int を使用し、0より大きいことを確認。
  • **拡張リスト**: ExtractExtensionList で構文チェックを行う。
  • **権限チェック**: password_required や sslcert/sslkey は、非スーパーユーザーによる設定を制限する。
  • **文字列型**: analyze_sampling など、許可された値のセットに含まれるかを確認。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの CREATE/ALTER 文実行時に PostgreSQL コアから呼び出される（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: InitPgFdwOptions, is_valid_option, defGetBoolean, defGetString, parse_real, parse_int, ExtractExtensionList, superuser 等。

関連データ構造

  • DefElem: オプションのキーと値を保持する構造体。
  • PgFdwOption: postgres_fdw がサポートするオプションの定義情報。
  • ClosestMatchState: 不正なオプション名に対する修正候補を算出するための構造体。

失敗モードと境界条件

  • **ERRCODE_FDW_INVALID_OPTION_NAME**: 未定義のオプションが指定された場合。
  • **ERRCODE_INVALID_PARAMETER_VALUE**: 値が型定義（数値範囲等）を満たさない場合。
  • **ERRCODE_INSUFFICIENT_PRIVILEGE**: スーパーユーザー権限が必要なオプションを一般ユーザーが設定しようとした場合。

保守担当者向けメモ

  • 新しいオプションを追加する場合は、postgres_fdw_options 配列への追加と、本関数内での検証ロジックの追加が必要です。
  • password_required や sslcert などの権限チェックは、catalog OID を用いて適用対象を厳密に区別しています。
