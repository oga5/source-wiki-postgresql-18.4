spgLeafConsistentOut

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist.h:183

目的

SP-GiSTインデックスのリーフノードにおける検索結果を保持し、呼び出し元へ返すための出力構造体です。

シグネチャまたは形状

typedef struct spgLeafConsistentOut
{
    Datum       leafValue;        /* 再構築された元のデータ（存在する場合） */
    bool        recheck;          /* 演算子の再チェックが必要な場合に真 */
    bool        recheckDistances; /* 距離の再チェックが必要な場合に真 */
    double     *distances;        /* 関連する距離情報 */
} spgLeafConsistentOut;

入力と出力

本構造体は、spgLeafConsistent サポート関数が検索結果を格納するために使用します。呼び出し元（インデックススキャン実行部）は、この構造体を通じてリーフノードの適合判定結果および再チェックの要否を受け取ります。

前提条件と不変条件

  • recheck が true の場合、インデックススキャン実行側はヒープから実際のタプルを取得し、演算子を再評価する必要があります。
  • distances が設定される場合、そのメモリ領域は適切に確保されている必要があります。

アルゴリズムまたは振る舞い

SP-GiSTの検索フローにおいて、spgLeafConsistent 関数がリーフノードに到達した際に、検索条件との照合を行います。その結果、適合するデータがあれば leafValue に格納し、必要に応じて recheck フラグを立てて呼び出し元へ制御を戻します。

  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • spgLeafConsistentIn: 対応する入力構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • distances ポインタが適切に初期化されていない場合、メモリ破壊や不正な参照が発生する可能性があります。
  • recheck フラグの誤設定は、検索結果の不整合（偽陽性または偽陰性）を招く恐れがあります。

保守担当者向けメモ

  • 本構造体はSP-GiSTの演算子クラス実装において必須のインターフェースです。変更を加える際は、すべての関連するサポート関数（spgLeafConsistent）の実装に影響が及ぶことに留意してください。
