WalRcvStreaming

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiverfuncs.c:126

目的

WALレシーバープロセスが現在ストリーミング中、またはストリーミング開始・再開の準備中であるかを判定し、リカバリプロセスに状態を通知する。

シグネチャまたは形状

bool WalRcvStreaming(void)

入力と出力

  • **入力**: なし（共有メモリ上の WalRcv 構造体を参照）
  • **出力**: bool 型。WALレシーバーが稼働中（WALRCV_STREAMING）、開始中（WALRCV_STARTING）、または再開中（WALRCV_RESTARTING）であれば true を返し、それ以外（WALRCV_STOPPED 等）であれば false を返す。

前提条件と不変条件

  • 共有メモリ上の WalRcv 構造体へのアクセスは、mutex スピンロックによって保護される。
  • 起動から一定時間（WALRCV_STARTUP_TIMEOUT）経過しても WALRCV_STARTING 状態が継続している場合、本関数は自動的に状態を WALRCV_STOPPED へ遷移させる。

アルゴリズムまたは振る舞い

1. 共有メモリの WalRcv 構造体から現在の状態と開始時刻をスピンロック保護下で取得する。 2. 状態が WALRCV_STARTING であり、かつ WALRCV_STARTUP_TIMEOUT を超過している場合：

  • 再度スピンロックを取得し、状態を WALRCV_STOPPED に更新する。
  • 状態遷移が発生した場合は walRcvStoppedCV をブロードキャストし、待機中のプロセスに通知する。

    • 3. 最終的な状態が WALRCV_STREAMING、WALRCV_STARTING、または WALRCV_RESTARTING のいずれかであれば true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c (リカバリプロセスにおける状態確認およびアサーション)
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease
  • ConditionVariableBroadcast

関連データ構造

  • WalRcvData: 共有メモリ上のレシーバー状態管理構造体。
  • WalRcvState: レシーバーの状態（WALRCV_STARTING, WALRCV_STREAMING, WALRCV_RESTARTING, WALRCV_STOPPED 等）。

失敗モードと境界条件

  • **タイムアウト**: 起動処理が長引いた場合、本関数が自律的に停止状態へ遷移させる。
  • **競合**: スピンロックにより、状態取得と更新時のデータ競合は回避される。

保守担当者向けメモ

  • 本関数は単なる状態確認だけでなく、タイムアウト監視という副作用を持つ。
  • WALRCV_STARTUP_TIMEOUT を超えた場合に WALRCV_STOPPED へ遷移させる処理は、WALレシーバーが起動に失敗した際にリカバリプロセスが永久待機に陥ることを防ぐ重要な安全装置である。
