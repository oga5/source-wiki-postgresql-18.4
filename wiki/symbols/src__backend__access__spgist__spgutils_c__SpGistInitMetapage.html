SpGistInitMetapage

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgutils.c:731

目的

SP-GiSTインデックスのメタページを初期化し、デフォルト値の設定と構造の整合性を確保します。

シグネチャまたは形状

void SpGistInitMetapage(Page page)

入力と出力

  • **入力**: page (初期化対象となるメモリ上のページバッファ)
  • **出力**: なし（引数として渡された page が直接書き換えられます）

前提条件と不変条件

  • page は有効なメモリ領域を指している必要があります。
  • メタページとして初期化されるため、ページヘッダーのフラグが SPGIST_META に設定されます。

アルゴリズムまたは振る舞い

1. SpGistInitPage を呼び出し、ページを SPGIST_META 型として初期化します。 2. SpGistPageGetMeta を介してメタデータ領域を取得し、ゼロクリアします。 3. magicNumber に SPGIST_MAGIC_NUMBER を設定します。 4. lastUsedPages キャッシュを走査し、すべての blkno を InvalidBlockNumber で初期化します。 5. ページヘッダーの pd_lower をメタデータ領域の直後に設定します。これは、WALログによるページ圧縮時にメタデータが消失するのを防ぐために必須の処理です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spginsert.c:99
  • src/backend/access/spgist/spginsert.c:163
  • **呼び出し先**:
  • SpGistInitPage
  • SpGistPageGetMeta

関連データ構造

  • SpGistMetaPageData: メタページに格納される構造体。
  • PageHeader: PostgreSQLの標準的なページヘッダー構造。

失敗モードと境界条件

  • 渡された page が不正なポインタである場合、メモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • pd_lower の設定は、WALログの圧縮処理と密接に関係しています。この計算ロジックを変更する場合は、ページレイアウトの整合性に十分注意してください。
