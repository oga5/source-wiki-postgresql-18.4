JsonbDeepContains

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_util.c:1074

目的

2つの Jsonb コンテナ（オブジェクトまたは配列）を走査し、右辺（mContained）の構造が左辺（val）に完全に含まれているかを再帰的に判定します。

シグネチャまたは形状

bool JsonbDeepContains(JsonbIterator **val, JsonbIterator **mContained)

入力と出力

  • **入力**:
  • val: 左辺の JsonbIterator へのポインタ。
  • mContained: 右辺（含まれるべき対象）の JsonbIterator へのポインタ。
  • **出力**:
  • bool: 右辺の全要素が左辺に存在し、かつ構造が一致する場合に true を返します。

前提条件と不変条件

  • check_stack_depth() により、過度に複雑な構造によるスタックオーバーフローを防止します。
  • オブジェクトのキーは正規化（重複排除）されていることを前提とします。
  • val と mContained は、比較対象のコンテナを指すイテレータである必要があります。

アルゴリズムまたは振る舞い

1. 両方のイテレータから次のトークンを取得し、型が一致するか確認します。 2. **オブジェクトの場合**:

  • 左辺のペア数が右辺より少ない場合は即座に false を返します。
  • 右辺の各キーに対して左辺から値を検索し、見つからない場合は false を返します。
  • 値がスカラー型の場合は equalsJsonbScalarValue で比較し、コンテナ型（オブジェクト・配列）の場合は自身を再帰的に呼び出します。

    • 3. **配列の場合**:

  • スカラー値の場合は findJsonbValueFromContainer を使用して検索します。
  • ネストされた構造の場合は、再帰的に包含関係を検証します。

    • 4. 右辺の走査が完了した時点で true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonb_op.c: JSONB 演算子（@> 等）の実装から呼び出されます。
  • src/backend/utils/adt/jsonb_util.c: 自身の再帰呼び出し。
  • **呼び出し先**:
  • JsonbIteratorNext: イテレータの進捗管理。
  • getKeyJsonValueFromContainer: オブジェクトのキー検索。
  • findJsonbValueFromContainer: 配列の要素検索。
  • JsonbIteratorInit: ネストされた構造の初期化。
  • equalsJsonbScalarValue: スカラー値の比較。

関連データ構造

  • JsonbIterator: コンテナ走査用の状態保持構造体。
  • JsonbValue: JSONB の各ノード（スカラー、オブジェクト、配列、バイナリ）を表現する構造体。

失敗モードと境界条件

  • **型不一致**: オブジェクトと配列の比較など、構造が異なる場合は false を返します。
  • **キー不在**: オブジェクトにおいて右辺のキーが左辺に存在しない場合は false を返します。
  • **スカラー包含**: スカラーが配列を含むことはできないため、その場合は false を返します。
  • **スタックオーバーフロー**: 再帰が深すぎる場合、check_stack_depth() によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は再帰的であるため、スタック消費量に注意してください。
  • オブジェクトのキー重複排除が前提となっているため、この前提が崩れるような変更を加える場合は注意が必要です。
  • 包含関係の判定において、右辺の全要素が左辺に存在する必要があるという「包含」の定義を厳守してください。
