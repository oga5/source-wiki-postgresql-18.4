cvec

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regguts.h:286

目的

正規表現エンジンにおいて、文字（chr）の集合や範囲（range）を動的に保持・管理するためのデータ構造です。

シグネチャまたは形状

struct cvec
{
    int         nchrs;          /* 保持している文字数 */
    int         chrspace;       /* chrs[] に割り当てられた容量 */
    chr        *chrs;           /* 文字ベクトルのポインタ */
    int         nranges;        /* 範囲（chrのペア）の数 */
    int         rangespace;     /* ranges[] に割り当てられた容量 */
    chr        *ranges;         /* 範囲ベクトルのポインタ */
    int         cclasscode;     /* クラスコード（enum classes 値または -1） */
};

入力と出力

  • 本構造体自体は、正規表現コンパイル時の文字クラスや範囲の集合を表現するコンテナとして機能します。
  • regc_cvec.c 等の関数を通じて、メモリ確保や初期化が行われます。

前提条件と不変条件

  • chrs および ranges は、それぞれ chrspace および rangespace で示される容量を超えない範囲で使用される必要があります。
  • cclasscode が有効な場合、特定の文字クラスを指し示します。

アルゴリズムまたは振る舞い

  • 正規表現のコンパイル過程で、文字の集合を動的に構築するために使用されます。
  • regc_cvec.c の newcvec 関数では、struct cvec 本体と chrs 配列を連続したメモリ領域として確保する実装が見られます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/regex/regc_color.c: subcolorcvec 関数内で cvec メンバーの処理に使用。
  • src/backend/regex/regc_cvec.c: newcvec（確保）および clearcvec（初期化）関数で使用。
  • **呼び出し先**:
  • 構造体内部で chr 型のポインタを保持しており、メモリ管理関数（MALLOC 等）と組み合わせて使用されます。

関連データ構造

  • chr: 正規表現エンジンで使用される文字型。
  • colormap: 文字を「色」に変換する際に、この cvec が保持する文字集合が参照されます。

失敗モードと境界条件

  • メモリ確保失敗時（MALLOC が NULL を返す場合）の挙動は、呼び出し側の実装に依存します。
  • chrspace や rangespace を超えるアクセスは、バッファオーバーフローを引き起こす可能性があります。

保守担当者向けメモ

  • cvec は動的にサイズ変更される可能性があるため、ポインタの生存期間とメモリの解放タイミングに注意してください。
  • regc_cvec.c にあるメモリ確保ロジックは、構造体と配列を単一の MALLOC で確保する最適化を行っているため、構造体のレイアウト変更時には注意が必要です。
