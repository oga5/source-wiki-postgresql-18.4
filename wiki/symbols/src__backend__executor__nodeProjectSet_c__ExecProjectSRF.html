ExecProjectSRF

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeProjectSet.c:138

目的

ターゲットリスト内の集合を返す関数（SRF）および通常の式を評価し、結果を TupleTableSlot に格納する。SRF が複数の行を生成する場合、状態を保持して反復的な評価を制御する。

シグネチャまたは形状

static TupleTableSlot *ExecProjectSRF(ProjectSetState *node, bool continuing)

入力と出力

  • **入力**:
  • node: 実行状態を保持する ProjectSetState 構造体。
  • continuing: 前回の入力タプルに対する SRF の継続評価中であるかを示すフラグ。
  • **出力**:
  • 構築されたタプルを含む TupleTableSlot。すべての SRF が終了した場合は NULL を返す。

前提条件と不変条件

  • ターゲットリスト内に少なくとも1つの SRF が存在すること（Assert(hassrf) により検証）。
  • econtext->ecxt_per_tuple_memory がタプル単位の評価コンテキストとして適切に設定されていること。

アルゴリズムまたは振る舞い

1. resultSlot をクリアし、評価用メモリコンテキストへ切り替える。 2. pending_srf_tuples を false に初期化する。 3. ターゲットリストの各要素をループ処理する:

  • continuing が真かつ SRF が終了状態（ExprEndResult）の場合、その列を NULL として扱う。
  • SetExprState（SRF）の場合、ExecMakeFunctionResultSet を呼び出し、結果が ExprMultipleResult であれば pending_srf_tuples を true に設定する。
  • SRF 以外の場合、ExecEvalExpr で通常評価を行う。

    • 4. 評価結果が残っている場合（hasresult が真）、ExecStoreVirtualTuple でスロットを確定させ、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeProjectSet.c 内の ExecProjectSet 関数（continuing 引数に true または false を指定して呼び出し）。
  • **呼び出し先**:
  • ExecMakeFunctionResultSet（SRF 評価用）
  • ExecEvalExpr（通常式評価用）
  • MemoryContextSwitchTo（メモリコンテキスト管理）
  • ExecClearTuple, ExecStoreVirtualTuple（タプル操作）

関連データ構造

  • ProjectSetState: ノードの実行状態を保持。
  • TupleTableSlot: 出力タプルの格納先。
  • ExprContext: 式評価のコンテキスト。
  • SetExprState: SRF の評価状態。

失敗モードと境界条件

  • SRF が含まれないターゲットリストに対しては Assert により停止する。
  • すべての SRF が ExprEndResult を返した場合、行は生成されず NULL が返される。
  • continuing モードにおいて、SRF が終了した列は NULL で埋められる。

保守担当者向けメモ

  • 本関数は ProjectSet ノードの核心であり、SRF の反復評価とメモリ管理（argcontext）の整合性が重要である。
  • pending_srf_tuples フラグの状態遷移が、呼び出し元である ExecProjectSet のロジックと密接に結合している点に注意すること。
