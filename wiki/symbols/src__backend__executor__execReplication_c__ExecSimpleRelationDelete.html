ExecSimpleRelationDelete

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execReplication.c:733

目的

論理レプリケーションにおいて、指定された行を対象テーブルから削除する。トリガーの実行やレプリカアイデンティティの検証を含む、レプリケーション適用プロセス向けの簡易的な削除インターフェースを提供する。

シグネチャまたは形状

void ExecSimpleRelationDelete(ResultRelInfo *resultRelInfo, EState *estate, EPQState *epqstate, TupleTableSlot *searchslot)

入力と出力

  • **入力**:
  • resultRelInfo: 対象リレーションの実行時情報。
  • estate: 実行状態。
  • epqstate: EPQ（EvalPlanQual）状態。
  • searchslot: 削除対象のタプル（TIDを含む）を保持するスロット。
  • **出力**: なし（副作用としてテーブルから行を削除）。

前提条件と不変条件

  • searchslot には有効な TID (tts_tid) が格納されている必要がある。
  • 対象テーブルが CMD_DELETE 操作を許可するレプリカアイデンティティ要件を満たしていること。

アルゴリズムまたは振る舞い

1. **検証**: CheckCmdReplicaIdentity を呼び出し、テーブルが削除操作を許可しているか確認する。 2. **BEFORE トリガー**: ri_TrigDesc に trig_delete_before_row が存在する場合、ExecBRDeleteTriggers を実行する。戻り値が false の場合、削除処理をスキップする。 3. **削除実行**: スキップされない場合、simple_table_tuple_delete を呼び出し、指定された TID の行を削除する。 4. **AFTER トリガー**: 削除成功後、ExecARDeleteTriggers を実行して削除後のトリガー処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c (論理レプリケーション適用ワーカー)
  • **呼び出し先**:
  • CheckCmdReplicaIdentity
  • ExecBRDeleteTriggers
  • simple_table_tuple_delete
  • ExecARDeleteTriggers

関連データ構造

  • ResultRelInfo: 実行対象のリレーション情報。
  • EState: 実行状態全般。
  • TupleTableSlot: タプルデータと TID の保持に使用。

失敗モードと境界条件

  • レプリカアイデンティティの検証に失敗した場合、エラーが発生する。
  • simple_table_tuple_delete 実行時に同時実行制御による競合が発生した場合、呼び出し元での再試行ロジックが必要となる（本関数自体は再試行を内包しない）。

保守担当者向けメモ

  • 本関数は論理レプリケーション専用の簡易インターフェースである。通常の SQL 実行パスとは異なり、トリガーの実行順序や競合処理がレプリケーションの仕様に依存している点に注意すること。
  • simple_table_tuple_delete は estate->es_snapshot を使用するため、スナップショットの整合性に注意が必要。
