bootstrap_signals

定義

  • 種別: function
  • 定義箇所: src/backend/bootstrap/bootstrap.c:412

目的

ブートストラップモード実行時におけるシグナルハンドリングをデフォルト状態（SIG_DFL）に設定し、異常発生時にプロセスが即座に終了するように強制する。

シグネチャまたは形状

static void bootstrap_signals(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてプロセスのシグナルハンドラ設定を変更する）

前提条件と不変条件

  • IsUnderPostmaster が false であること（Postmaster配下ではないスタンドアロンプロセスであること）。

アルゴリズムまたは振る舞い

1. Assert により、Postmaster配下で実行されていないことを検証する。 2. pqsignal を使用し、以下のシグナルに対するハンドラを SIG_DFL（デフォルトの終了動作）に明示的に設定する。

  • SIGHUP
  • SIGINT
  • SIGTERM
  • SIGQUIT

呼び出し元と呼び出し先

  • 呼び出し元: BootstrapModeMain (src/backend/bootstrap/bootstrap.c:363)
  • 呼び出し先: pqsignal（シグナルハンドラ設定用ユーティリティ）

関連データ構造

  • なし

失敗モードと境界条件

  • IsUnderPostmaster が真の場合、Assert によりアボートする。

保守担当者向けメモ

  • 本関数はブートストラップモード特有の単純な終了動作を保証するためのものであり、通常のバックエンドプロセスのような複雑なシグナル処理は意図的に排除されている。
