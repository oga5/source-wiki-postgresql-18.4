wait_for_connection_state

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:119

目的

指定されたプロセスID（procpid）のバックエンドが、特定の接続状態（state）または待機イベント（event）に到達するまでポーリングを行い、待機する。

シグネチャまたは形状

static void wait_for_connection_state(int line, PGconn *monitorConn, int procpid, char *state, char *event);

入力と出力

  • line: エラー発生時に報告するソースコード行番号。
  • monitorConn: 監視対象のデータベース接続（PGconn）。
  • procpid: 監視対象のバックエンドプロセスID。
  • state: 監視する pg_stat_activity.state（NULL可）。
  • event: 監視する pg_stat_activity.wait_event（NULL可）。
  • 戻り値: なし（条件を満たした時点で制御を戻す）。

前提条件と不変条件

  • state と event のいずれか一方が必ず指定され、もう一方は NULL でなければならない（Assert((state == NULL) ^ (event == NULL))）。
  • monitorConn は有効なデータベース接続であること。

アルゴリズムまたは振る舞い

1. procpid を文字列に変換する。 2. state が指定されていれば pg_stat_activity の state を、event が指定されていれば wait_event を対象にクエリを実行する。 3. クエリ結果のカウントが "0" 以外（＝対象の状態に該当する行が存在する）になるまで、10ms間隔（pg_usleep(10000)）でループを繰り返す。 4. 条件を満たした時点で PQclear を行い、メモリを解放して終了する。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: クエリ結果を保持する構造体。

失敗モードと境界条件

  • PQresultStatus が PGRES_TUPLES_OK 以外を返した場合、pg_fatal_impl を呼び出しテストを中断する。
  • クエリ結果の行数または列数が期待値（1）と異なる場合、pg_fatal_impl を呼び出しテストを中断する。

保守担当者向けメモ

  • 本関数はテスト専用のユーティリティであり、pg_stat_activity ビューに依存している。データベースの負荷状況によっては、ポーリング間隔やタイムアウトの考慮が必要になる可能性がある。
