pqTraceOutput_Close

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:269

目的

PostgreSQL プロトコルにおける Close メッセージの内容をデバッグ用トレース出力ストリームへ書き出す。

シグネチャまたは形状

static void pqTraceOutput_Close(FILE *f, const char *message, int *cursor);

入力と出力

  • f: 出力先の FILE ストリーム。
  • message: 解析対象のメッセージデータを含むバッファ。
  • cursor: メッセージ内の現在位置を指すポインタ。関数実行後に読み込み分だけ進められる。
  • 戻り値: なし。

前提条件と不変条件

  • cursor が指す位置から、プロトコル定義に従った Close メッセージ（型バイトおよび文字列）が読み取り可能であること。
  • f が有効な書き込み可能なストリームであること。

アルゴリズムまたは振る舞い

1. ストリーム f に "Close" というラベルとタブ文字を出力する。 2. pqTraceOutputByte1 を呼び出し、メッセージ内の型バイトを読み取って出力する。 3. pqTraceOutputString を呼び出し、メッセージ内の文字列フィールドを読み取って出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-trace.c 内のメッセージ解析ルーチン（pqTraceOutputMessage 等のディスパッチャから呼び出されることが想定される）。
  • 呼び出し先: pqTraceOutputByte1, pqTraceOutputString。

関連データ構造

  • FILE: 標準ライブラリの出力ストリーム。
  • PGconn: 本関数が間接的に利用するトレース設定（Pfdebug）を保持する構造体。

失敗モードと境界条件

  • message バッファの終端に達している場合、内部で呼び出される pqTraceOutputByte1 や pqTraceOutputString の挙動に依存する。
  • f が NULL の場合、fprintf によりセグメンテーションフォールトが発生する可能性がある。

保守担当者向けメモ

  • 本関数はプロトコル v3 の Close メッセージ専用である。
  • メッセージフォーマットに変更があった場合は、pqTraceOutputByte1 および pqTraceOutputString の呼び出し順序や引数を確認すること。
