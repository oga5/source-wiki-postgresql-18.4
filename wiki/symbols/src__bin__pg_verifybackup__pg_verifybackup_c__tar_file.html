tar_file

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:52

目的

バックアップディレクトリ内の Tar アーカイブファイルを識別し、そのメタデータ（相対パス、テーブルスペース OID、圧縮アルゴリズム）を保持するために使用されます。

シグネチャまたは形状

typedef struct tar_file
{
    char       *relpath;
    Oid         tblspc_oid;
    pg_compress_algorithm compress_algorithm;
} tar_file;

入力と出力

  • 入力: pg_malloc を介したメモリ割り当て時に各フィールドが初期化されます。
  • 出力: 構造体ポインタとして、バックアップ検証プロセス中のファイル追跡に使用されます。

前提条件と不変条件

  • relpath は有効な文字列ポインタである必要があります。
  • tblspc_oid は PostgreSQL のテーブルスペース識別子を保持します。

アルゴリズムまたは振る舞い

本構造体は、pg_verifybackup における Tar 形式バックアップの検証プロセスにおいて、処理対象となるアーカイブファイルの属性を管理します。検証の過程で、リスト構造の要素として格納され、各アーカイブの検証時に参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_verifybackup/pg_verifybackup.c 内の検証ロジック（verify_tar_backup 等）において、メモリ確保およびリスト操作を通じて参照されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • pg_compress_algorithm: 圧縮アルゴリズムを定義する列挙型。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • メモリ不足時に pg_malloc が失敗した場合、プログラムは終了します。
  • relpath が適切に設定されていない場合、ファイルパスの解決に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は pg_verifybackup.c 内で定義されていますが、Tar 形式の検証ロジック全体に関わるため、変更時は astreamer_verify.c との整合性に注意してください。
