check_safe_enum_use

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:62

目的

未コミットの列挙型（ENUM）値がSQL操作で使用されることを防ぐガード関数です。ロールバック後にインデックスが破損するリスクを回避するため、SQLレベルで値を返す全ての箇所で呼び出されます。

シグネチャまたは形状

static void check_safe_enum_use(HeapTuple enumval_tup)

入力と出力

  • **入力**: enumval_tup (検証対象の pg_enum システムカタログのタプル)
  • **出力**: なし（安全な場合は正常終了、未コミットで危険な場合は ERROR を発生させトランザクションを中断）

前提条件と不変条件

  • 入力タプルは pg_enum システムカタログの有効な行であること。
  • 列挙値がコミット済みであるか、あるいは EnumUncommitted マクロによって安全と判定される状態であること。

アルゴリズムまたは振る舞い

1. **コミット済み判定（高速パス）**: タプルのヘッダー情報からコミット済みであることが示されている場合、直ちに終了します。 2. **トランザクション状態確認**: xmin を直接確認し、進行中でなくコミット済みであれば終了します。 3. **未コミット判定**: EnumUncommitted マクロにより、当該OIDが未コミットの列挙値として登録されていないか確認します。登録されていなければ安全とみなします。 4. **例外発生**: 上記のいずれにも該当しない場合、ERRCODE_UNSAFE_NEW_ENUM_VALUE_USAGE エラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/enum.c 内の列挙型操作関数（enum_in, enum_out 等の関連処理）から複数回呼び出されています。
  • src/backend/catalog/pg_enum.c からの参照が示唆されています。
  • **呼び出し先**:
  • HeapTupleHeaderXminCommitted, HeapTupleHeaderGetXmin, TransactionIdIsInProgress, TransactionIdDidCommit (トランザクション状態確認)
  • EnumUncommitted (カタログ状態確認)
  • ereport (エラー通知)

関連データ構造

  • HeapTuple: pg_enum の行データ。
  • Form_pg_enum: pg_enum カタログの構造体定義。

失敗モードと境界条件

  • **失敗モード**: 未コミットの列挙値が使用された場合、ERROR が発生し処理が停止します。
  • **境界条件**: トランザクション内で ADD VALUE された直後の値は、コミットされるまで本関数によって使用が制限されます。

保守担当者向けメモ

  • 本関数はインデックスの整合性保護に直結するため、変更時はトランザクションの可視性ルールに十分注意してください。
  • 現在はエラーを投げるのみですが、将来的に安全な条件を絞り込むための追加テストを実装する余地があります。
