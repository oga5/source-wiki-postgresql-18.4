pg_hmac_update

定義

  • 種別: function
  • 定義箇所: src/common/hmac_openssl.c:216

目的

HMAC計算において、入力データをストリームとして処理し、内部状態を更新する。

シグネチャまたは形状

int pg_hmac_update(pg_hmac_ctx *ctx, const uint8 *data, size_t len)

入力と出力

  • ctx: HMAC計算状態を保持する pg_hmac_ctx 構造体へのポインタ。
  • data: 更新対象のデータバッファ。
  • len: data のバイト長。
  • 戻り値: 成功時に 0、失敗時に -1 を返す。

前提条件と不変条件

  • ctx は pg_hmac_create によって初期化済みであること。
  • ctx が NULL の場合、関数は即座に -1 を返す。

アルゴリズムまたは振る舞い

1. OpenSSLの HMAC_Update 関数を呼び出し、提供された data と len を現在のコンテキストに適用する。 2. HMAC_Update の戻り値が 1（成功）以外の場合、OpenSSLのエラーキューからエラー情報を取得する。 3. エラー発生時は SSLerrmessage を用いてエラー理由を ctx->errreason に格納し、ctx->error を PG_HMAC_ERROR_OPENSSL に設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/libpq/auth-scram.c (SCRAM認証プロセス内)
  • src/common/hmac.c (抽象化レイヤー経由)
  • 呼び出し先:
  • HMAC_Update (OpenSSL)
  • ERR_get_error (OpenSSL)
  • SSLerrmessage (内部関数)

関連データ構造

  • pg_hmac_ctx: HMAC計算の状態を保持する構造体。

失敗モードと境界条件

  • ctx が NULL の場合: -1 を返す。
  • OpenSSL内部でエラーが発生した場合: -1 を返し、ctx 内にエラー理由が記録される。

保守担当者向けメモ

  • 本関数はOpenSSLの HMAC_Update をラップしたものである。
  • エラー発生時の詳細な診断情報は ctx->errreason を通じて取得可能である。
