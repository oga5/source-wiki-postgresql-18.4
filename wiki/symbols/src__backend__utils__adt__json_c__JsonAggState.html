JsonAggState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/json.c:78

目的

json_agg および json_object_agg 集約関数を実行する際、遷移関数間で共有される状態を保持するために使用されます。

シグネチャまたは形状

typedef struct JsonAggState
{
	StringInfo	str;
	JsonTypeCategory key_category;
	Oid			key_output_func;
	JsonTypeCategory val_category;
	Oid			val_output_func;
	JsonUniqueBuilderState unique_check;
} JsonAggState;

入力と出力

  • **入力**: 集約関数の遷移関数（json_agg_transfn 等）において、PG_GETARG_POINTER(0) を介して取得されます。
  • **出力**: 最終的に json_agg_finalfn 等により、StringInfo に蓄積されたデータが text 型として返却されます。

前提条件と不変条件

  • JsonAggState のライフサイクルは集約実行期間に依存します。
  • MemoryContext の切り替えが頻繁に行われるため、メモリ管理には注意が必要です。
  • unique_check（JsonUniqueBuilderState）は、キー重複チェックが必要な場合に適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: palloc を用いてメモリを確保し、集約の初期状態を構築します。 2. **状態更新**: 遷移関数が呼び出されるたびに、str（StringInfo）へJSON要素を追加し、必要に応じて unique_check を更新します。 3. **最終化**: finalfn が str に格納されたバッファを読み取り、最終的なJSON文字列を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: json_agg_transfn, json_object_agg_transfn, json_agg_finalfn, json_object_agg_finalfn 等の集約関数実装から参照されます。
  • **呼び出し先**: palloc によるメモリ確保、および JsonUniqueBuilderState を介したキー重複チェック機能を利用します。

関連データ構造

  • StringInfo: JSON文字列を構築するための動的バッファ。
  • JsonTypeCategory: 型変換のカテゴリ判定に使用。
  • JsonUniqueBuilderState: JSONオブジェクト構築時のキー重複検証用状態。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • キー重複: unique_check が有効な場合、重複キーが検出されるとエラーをスローする可能性があります。
  • ポインタ無効化: 集約処理中のメモリコンテキスト管理が不適切な場合、不正なポインタ参照が発生するリスクがあります。

保守担当者向けメモ

  • JsonAggState は集約の実行期間中のみ有効です。メモリリークを防ぐため、MemoryContext のライフサイクルを厳密に管理してください。
  • unique_check を使用する際は、ハッシュテーブルへのキーコピー処理がバッファの再配置を考慮して設計されている点に留意してください。
