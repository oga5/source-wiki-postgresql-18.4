pg_be_scram_build_secret

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-scram.c:482

目的

新規パスワードから、PostgreSQLのSASL SCRAM-SHA-256認証で使用されるSCRAMシークレット文字列を生成します。

シグネチャまたは形状

char *pg_be_scram_build_secret(const char *password)

入力と出力

  • **入力**: password (認証に使用する平文パスワード)
  • **出力**: SCRAMシークレット形式の文字列（成功時）。失敗時はエラーを報告し、呼び出し元へは戻りません。

前提条件と不変条件

  • pg_strong_random が利用可能であること。
  • scram_sha_256_iterations が適切に設定されていること。
  • メモリ割り当てが成功すること。

アルゴリズムまたは振る舞い

1. **正規化**: pg_saslprep を用いてパスワードを正規化します。UTF-8として無効な場合や禁止文字が含まれる場合は、元のパスワードをそのまま使用します。 2. **ソルト生成**: pg_strong_random を使用して SCRAM_DEFAULT_SALT_LEN バイトのランダムなソルトを生成します。生成に失敗した場合は ERROR を発生させます。 3. **シークレット構築**: scram_build_secret を呼び出し、SHA-256アルゴリズム、生成したソルト、設定された反復回数を用いてシークレットを生成します。 4. **クリーンアップ**: 正規化に使用した一時的なパスワードメモリを解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/crypt.c:145
  • **呼び出し先**: pg_saslprep, pg_strong_random, scram_build_secret, pfree

関連データ構造

  • SCRAM_DEFAULT_SALT_LEN (定数)
  • PG_SHA256 (アルゴリズム識別子)
  • scram_sha_256_iterations (反復回数設定)

失敗モードと境界条件

  • **乱数生成失敗**: pg_strong_random が失敗した場合、ereport(ERROR, ...) によりトランザクションが中断されます。
  • **メモリ不足**: palloc 等の内部関数がメモリ不足に陥った場合、PostgreSQLの標準的なエラー処理に従い中断されます。

保守担当者向けメモ

  • 本関数は libpq/scram.h で宣言されており、外部モジュールから利用可能です。
  • パスワードの正規化失敗時に元のパスワードをそのまま使用する挙動は、非UTF-8環境への寛容性を維持するための意図的な設計です。
