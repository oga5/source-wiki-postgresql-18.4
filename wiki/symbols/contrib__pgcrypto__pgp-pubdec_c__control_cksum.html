control_cksum

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubdec.c:73

目的

復号されたPGPセッション鍵メッセージの整合性を、チェックサムを用いて検証します。

シグネチャまたは形状

static int control_cksum(uint8 *msg, int msglen)

入力と出力

  • **入力**:
  • msg: 復号されたデータを含むバイト配列。
  • msglen: msg の長さ。
  • **出力**:
  • 成功時: 0
  • 失敗時: PXE_PGP_WRONG_KEY

前提条件と不変条件

  • msglen が 3 未満の場合、チェックサム計算が不可能なためエラーを返します。
  • メッセージの末尾 2 バイトがチェックサム値として格納されていることを前提とします。

アルゴリズムまたは振る舞い

1. msglen が 3 未満であれば PXE_PGP_WRONG_KEY を返します。 2. メッセージの 2 番目のバイト（インデックス 1）から末尾の 2 バイト前までを合計し、下位 16 ビットを抽出して my_cksum とします。 3. メッセージの末尾 2 バイトを結合して got_cksum を算出します。 4. my_cksum と got_cksum を比較し、一致しない場合は px_debug でログを出力した上で PXE_PGP_WRONG_KEY を返します。 5. 一致した場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pubdec.c 内の pgp_parse_pubenc_sesskey から呼び出されます。
  • **呼び出し先**: px_debug（エラーログ出力用）。

関連データ構造

  • uint8: バイトデータ型。
  • PXE_PGP_WRONG_KEY: px.h で定義されるエラーコード。

失敗モードと境界条件

  • msglen < 3: チェックサム検証に必要な最小長を満たさないため失敗します。
  • チェックサム不一致: 復号された鍵データが破損しているか、誤った鍵で復号されたと判断され、PXE_PGP_WRONG_KEY が返されます。

保守担当者向けメモ

  • 本関数はPGPのセッション鍵パケットの仕様に基づいた単純な加算チェックサムを使用しています。
  • 復号失敗時のエラーコードは、パディングエラーやチェックサム不一致を区別せず一律に PXE_PGP_WRONG_KEY を返す仕様です。
