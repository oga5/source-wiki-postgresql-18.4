pg_utf_to_local_combined

定義

  • 種別: struct
  • 定義箇所: src/include/mb/pg_wchar.h:493

目的

UTF-8からローカルエンコーディングへの変換において、複数のUTF-8コードポイントを単一のローカルコードへマッピングするための変換テーブル用データ構造です。

シグネチャまたは形状

typedef struct
{
	uint32		utf1;			/* UTF-8 code 1 */
	uint32		utf2;			/* UTF-8 code 2 */
	uint32		code;			/* local code */
} pg_utf_to_local_combined;

入力と出力

  • 入力: 変換対象となる2つのUTF-8コード（utf1, utf2）。
  • 出力: 対応するローカルエンコーディングのコード（code）。

前提条件と不変条件

  • 本構造体は、変換テーブルの要素として使用されます。
  • 変換テーブルは、src/backend/utils/mb/conv.c 内の関数（compare3 等）によってソートされた状態で保持されることが想定されています。

アルゴリズムまたは振る舞い

  • src/backend/utils/mb/conv.c において、バイナリサーチ等の検索アルゴリズムを用いて、特定のUTF-8コードペアに対応するローカルコードを特定するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/utils/mb/conv.c: 変換テーブルの走査および比較処理で使用。
  • src/include/mb/pg_wchar.h: 関数プロトタイプの引数として定義。
  • 呼び出し先:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • pg_mb_radix_tree: UTF-8変換の高速化に利用される基数木構造。

失敗モードと境界条件

  • 検索対象のUTF-8コードペアがテーブル内に存在しない場合、変換処理は失敗またはデフォルト値へのフォールバックを行う必要があります。

保守担当者向けメモ

  • 本構造体は、主に src/backend/utils/mb/conv.c で実装される変換ロジックと密接に関連しています。変更を加える際は、対応する変換テーブルのソート順序および検索ロジックに影響がないか確認してください。
