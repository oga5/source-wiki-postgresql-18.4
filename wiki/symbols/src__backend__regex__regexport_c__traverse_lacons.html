traverse_lacons

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:92

目的

コンパイル済みの正規表現（NFA）において、LACON（Lookaround Constraints）を透過的に処理し、実質的な遷移先のみを抽出するための内部ヘルパー関数です。

シグネチャまたは形状

static void traverse_lacons(struct cnfa *cnfa, int st, int *arcs_count, regex_arc_t *arcs, int arcs_len);

入力と出力

  • cnfa: 探索対象の正規表現 NFA 構造体。
  • st: 現在探索中の状態インデックス。
  • arcs_count: 検出された有効なアークの総数を保持するポインタ（入出力）。
  • arcs: 検出されたアーク情報を格納するバッファ（出力）。
  • arcs_len: arcs バッファのサイズ。

前提条件と不変条件

  • check_stack_depth() により、再帰呼び出しによるスタックオーバーフローを防止しています。
  • LACON アークは、正規表現のコンパイル時に除去されるべきループ構造を持たないことが想定されています。
  • LACON アークが直接 cnfa->post（終了状態）へ遷移することはアサートにより禁止されています。

アルゴリズムまたは振る舞い

1. 指定された状態 st から始まる各アークを走査します。 2. カラーコードが cnfa->ncolors 未満の場合、通常の遷移とみなし、arcs_count をインクリメントして、バッファに空きがあればアーク情報を記録します。 3. カラーコードが cnfa->ncolors 以上の場合、LACON アークと判断し、その遷移先に対して再帰的に traverse_lacons を呼び出します。 4. このプロセスにより、LACON をスキップした実質的な遷移先のみが抽出されます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_reg_getnumoutarcs および pg_reg_getoutarcs から呼び出されます。
  • **呼び出し先**: 自身を再帰的に呼び出します。

関連データ構造

  • struct cnfa: 正規表現の NFA 構造体。
  • struct carc: 状態遷移アークの定義。
  • regex_arc_t: 公開用の遷移情報構造体。

失敗モードと境界条件

  • **スタックオーバーフロー**: 極端に深い LACON 構造を持つ正規表現に対しては、check_stack_depth() が実行時エラーを発生させる可能性があります。
  • **バッファオーバーフロー**: arcs_len を超えるアークが存在する場合、arcs への書き込みは行われませんが、arcs_count は正しくカウントされます。

保守担当者向けメモ

  • 本関数は LACON を「常に満足されるもの」として扱う設計です。
  • 再帰関数であるため、スタック消費量に注意してください。
  • Assert(ca->to != cnfa->post) は、LACON が終了状態へ直接到達しないことを保証する重要な不変条件です。
