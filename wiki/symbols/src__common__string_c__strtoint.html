strtoint

定義

  • 種別: function
  • 定義箇所: src/common/string.c:49

目的

文字列を int 型へ安全に変換します。標準の strtol をラップし、変換結果が int 型の範囲内に収まるかを検証する機能を提供します。

シグネチャまたは形状

int strtoint(const char *pg_restrict str, char **pg_restrict endptr, int base);

入力と出力

  • **入力**:
  • str: 変換対象の文字列。
  • endptr: 変換終了位置を格納するポインタ（NULL 指定可能）。
  • base: 基数（例: 10）。
  • **出力**:
  • 変換された int 値を返します。

前提条件と不変条件

  • strtol の挙動に依存します。
  • 変換結果が int 型の範囲外である場合、errno が ERANGE に設定されます。

アルゴリズムまたは振る舞い

1. strtol を呼び出し、結果を long 型の変数 val に格納します。 2. val を int 型にキャストし、元の val と比較することで範囲チェックを行います。 3. 範囲外（オーバーフロー/アンダーフロー）の場合、errno に ERANGE を設定します。 4. 最終的に int にキャストした値を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: strtol（標準 C ライブラリ関数）。
  • **呼び出し元**:
  • contrib/pgcrypto/crypt-sha.c
  • src/backend/commands/tsearchcmds.c
  • src/backend/nodes/read.c
  • src/backend/utils/adt/datetime.c

    • （上記はツールによるヒューリスティックな検出結果であり、網羅的ではありません）

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 変換結果が int 型の表現範囲を超えた場合、errno が ERANGE に設定されます。
  • strtol の仕様に基づき、無効な入力に対する挙動や endptr の更新が行われます。

保守担当者向けメモ

  • 本関数は PostgreSQL のバックエンドおよびフロントエンドの両方から利用されます。
  • errno を介したエラー伝播を行うため、呼び出し元は変換後に errno を確認する必要があります。
