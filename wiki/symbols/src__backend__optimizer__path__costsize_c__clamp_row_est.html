clamp_row_est

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/costsize.c:212

目的

クエリプランナにおける行数推定値（nrows）を妥当な範囲に調整し、コスト計算時の異常値やゼロ除算を防止する。

シグネチャまたは形状

double clamp_row_est(double nrows)

入力と出力

  • **入力**: double nrows（推定行数）
  • **出力**: double（調整済みの行数）

前提条件と不変条件

  • 出力値は必ず 1.0 以上となる。
  • 出力値は MAXIMUM_ROWCOUNT を超えない。
  • 出力値は整数値（rint による丸め）として返される。

アルゴリズムまたは振る舞い

1. nrows が MAXIMUM_ROWCOUNT を超える場合、または NaN である場合、MAXIMUM_ROWCOUNT に設定する。 2. nrows が 1.0 以下の場合、1.0 に設定する。 3. 上記以外の場合、rint() を使用して nrows を最も近い整数に丸める。 4. 調整後の値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/postgres_fdw.c
  • contrib/tsm_system_rows/tsm_system_rows.c
  • contrib/tsm_system_time/tsm_system_time.c
  • **呼び出し先**:
  • math.h の isnan() および rint() を利用。

関連データ構造

  • 特になし（プリミティブな double 型を操作する）。

失敗モードと境界条件

  • **NaN**: isnan() チェックにより MAXIMUM_ROWCOUNT に置換される。
  • **ゼロまたは負数**: 1.0 にクランプされるため、ゼロ除算が回避される。
  • **過大な値**: MAXIMUM_ROWCOUNT を上限として制限される。

保守担当者向けメモ

  • 本関数は EXPLAIN 出力の見た目を整える役割も兼ねている。
  • コスト計算の安定性を維持するため、行数推定値を使用する箇所では必ず本関数を通すことが推奨される。
