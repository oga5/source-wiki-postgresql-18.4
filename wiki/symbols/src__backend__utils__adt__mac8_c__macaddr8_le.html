macaddr8_le

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac8.c:346

目的

2つの macaddr8 型の値が「小なりまたは等しい（less than or equal to）」関係にあるかを判定する。

シグネチャまたは形状

Datum macaddr8_le(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの macaddr8 型のポインタ。
  • **出力**: bool 型の Datum。第1引数が第2引数以下であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力される macaddr8 ポインタは有効なメモリ領域を指している必要がある。
  • 比較ロジックは macaddr8_cmp_internal に依存しており、同関数が8バイトのMACアドレスを正しく比較できることが前提となる。

アルゴリズムまたは振る舞い

1. PG_GETARG_MACADDR8_P を使用して、引数から2つの macaddr8 構造体へのポインタを取得する。 2. macaddr8_cmp_internal を呼び出し、2つの値を比較する。 3. 比較結果が 0 以下（<= 0）であるかを判定し、その結果を PG_RETURN_BOOL で返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_macaddr8.c:49（DirectFunctionCall2 を介した呼び出し）
  • **呼び出し先**:
  • macaddr8_cmp_internal

関連データ構造

  • macaddr8: 8バイト（EUI-64）のMACアドレスを保持する構造体。

失敗モードと境界条件

  • 引数が NULL の場合、PG_GETARG_MACADDR8_P の実装に依存した動作となる（通常はエラーまたは例外）。
  • 比較対象のMACアドレスが同一である場合、macaddr8_cmp_internal は 0 を返し、本関数は true を返す。

保守担当者向けメモ

  • 本関数は比較演算子としての標準的な実装であり、macaddr8_cmp_internal の比較ロジック（hibits および lobits マクロによるビット単位の比較）に変更があった場合、その影響を直接受ける。
