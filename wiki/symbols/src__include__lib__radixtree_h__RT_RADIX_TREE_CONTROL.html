RT_RADIX_TREE_CONTROL

定義

  • 種別: struct
  • 定義箇所: src/include/lib/radixtree.h:686

目的

適応型基数木（ART）のメタデータおよび制御情報を保持する構造体です。木のルートノード、統計情報、および共有メモリ環境下での同期制御を管理します。

シグネチャまたは形状

typedef struct RT_RADIX_TREE_CONTROL
{
#ifdef RT_SHMEM
    RT_HANDLE   handle;
    uint32      magic;
    LWLock      lock;
#endif

    RT_PTR_ALLOC root;
    uint64      max_val;
    int64       num_keys;
    int         start_shift;

#ifdef RT_DEBUG
    int64       num_nodes[RT_NUM_SIZE_CLASSES];
    int64       num_leaves;
#endif
} RT_RADIX_TREE_CONTROL;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入出力はありません。ただし、RT_CREATE 等の操作を通じてメモリ確保時に初期化され、木の操作に伴い num_keys や root ポインタが更新されます。

前提条件と不変条件

  • RT_SHMEM が定義されている場合、LWLock を用いた排他制御が必須となります。
  • RT_DEBUG が定義されている場合のみ、統計情報（num_nodes, num_leaves）が保持されます。

アルゴリズムまたは振る舞い

  • **共有メモリ管理**: RT_SHMEM 定義時は dsa_allocate0 を介して確保され、dsa_get_address でアクセスされます。
  • **ローカルメモリ管理**: palloc0 を介して確保されます。
  • **階層管理**: root を起点として、キーのビット分割に基づいたノード探索を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/lib/radixtree.h 内の木生成・初期化ロジック（dsa_allocate0 や palloc0 を使用する箇所）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • RT_RADIX_TREE: 本構造体を保持するメインの基数木構造体。
  • RT_HANDLE, LWLock: 共有メモリ環境下での管理用構造体。
  • RT_PTR_ALLOC: ノードへのポインタ型。

失敗モードと境界条件

  • メモリ不足時: dsa_allocate0 または palloc0 が失敗し、制御構造体の確保ができません。
  • 共有メモリ環境下でのロック競合: LWLock の取得に失敗した場合、操作がブロックされる可能性があります。

保守担当者向けメモ

  • 本構造体はマクロテンプレートの一部として展開されるため、RT_RADIX_TREE_CONTROL という名前はコンパイル時に RT_MAKE_NAME マクロによって生成されます。
  • 構造体のメンバ構成は RT_SHMEM や RT_DEBUG の定義有無によって動的に変化します。
