parse_output_parameters

定義

  • 種別: function
  • 定義箇所: src/backend/replication/pgoutput/pgoutput.c:289

目的

論理レプリケーションの出力プラグイン（pgoutput）の初期化時に、クライアントから提供されたオプションリストを解析し、プラグインのセッション状態（PGOutputData）を構成する。

シグネチャまたは形状

static void parse_output_parameters(List *options, PGOutputData *data)

入力と出力

  • **入力**:
  • options: DefElem 構造体のリスト。各要素は name=value 形式のオプション。
  • data: 設定を格納する PGOutputData 構造体へのポインタ。
  • **出力**: なし（data 構造体の各フィールドを直接更新する）。

前提条件と不変条件

  • data が指す構造体は、呼び出し前にメモリ確保済みであること。
  • options リスト内の各要素は DefElem 型であること。
  • proto_version および publication_names は必須オプションであり、指定がない場合はエラーとなる。

アルゴリズムまたは振る舞い

1. data のデフォルト値を初期化する（binary=false, streaming=OFF, messages=false, two_phase=false）。 2. options リストを走査し、各 DefElem の名前（defname）に基づいて以下のパラメータを解析・適用する。

  • proto_version: 数値変換し data->protocol_version に格納。
  • publication_names: SplitIdentifierString を使用してパブリケーション名のリストを解析。
  • binary, messages, two_phase: defGetBoolean で真偽値を判定。
  • streaming: defGetStreamingMode でストリーミングモードを判定。
  • origin: LOGICALREP_ORIGIN_NONE または ANY を判定し data->publish_no_origin を設定。

    • 3. 各オプションが重複して指定された場合はエラーを報告する。 4. 未知のオプションが指定された場合はエラーを報告する。 5. 必須オプションの欠落を確認し、存在しない場合はエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: pgoutput_startup（src/backend/replication/pgoutput/pgoutput.c:490 付近）。
  • **呼び出し先**: defGetBoolean, defGetStreamingMode, defGetString, SplitIdentifierString, strVal, ereport, elog 等のユーティリティ関数。

関連データ構造

  • PGOutputData: プラグインのセッション状態を保持する構造体。
  • DefElem: パーサーから渡されるオプション定義構造体。

失敗モードと境界条件

  • **エラー発生時**: ereport(ERROR, ...) によりトランザクションを中断する。
  • 必須オプション（proto_version, publication_names）の欠落。
  • オプションの重複指定。
  • 不正な値（範囲外の数値、不正な構文、未定義のオプション名）。
  • **境界条件**: proto_version が PG_UINT32_MAX を超える場合はエラーとなる。

保守担当者向けメモ

  • 新しいオプションを追加する場合は、foreach ループ内の if-else ブロックに条件を追加し、重複チェック用のフラグ変数を関数冒頭で定義すること。
  • SplitIdentifierString は入力文字列を破壊的に変更するため、pstrdup を介してコピーを渡している点に注意。
