CatalogCacheInitializeCache

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/cache/catcache.c:1125

目的

システムカタログキャッシュ（CatCache）の初期化を行い、カタログのタプル記述子（TupleDesc）の永続化および検索キーの関数情報を設定する。

シグネチャまたは形状

static void CatalogCacheInitializeCache(CatCache *cache)

入力と出力

  • **入力**: CatCache *cache（初期化対象のキャッシュ構造体）
  • **出力**: なし（副作用として cache 構造体の各フィールドが更新される）

前提条件と不変条件

  • CacheMemoryContext が初期化済みであること。
  • cache->cc_reloid が有効なカタログリレーションを指していること。
  • キャッシュキーの属性は NOT NULL であること。
  • システム属性（attnum < 0）はキャッシュキーとしてサポートされない。

アルゴリズムまたは振る舞い

1. table_open を使用して対象カタログを AccessShareLock で開く。 2. CacheMemoryContext にメモリコンテキストを切り替え、タプル記述子をコピーして cache->cc_tupdesc に格納する。 3. リレーション名と共有フラグ（relisshared）をキャッシュ構造体に保存する。 4. table_close でリレーションを閉じる。 5. 各キーに対して以下の処理をループ実行する：

  • GetCCHashEqFuncs を呼び出し、ハッシュ関数および等価関数を取得する。
  • fmgr_info_cxt を使用して等価関数の情報を CacheMemoryContext に登録する。
  • ScanKeyData（cc_skey）の各フィールド（属性番号、戦略、照合順序など）を初期化する。

    • 6. 最後に cache->cc_tupdesc を設定し、初期化完了状態とする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/catcache.c 内の ConditionalCatalogCacheInitializeCache など、キャッシュの遅延初期化を行う関数から呼び出される。
  • **呼び出し先**: table_open, table_close, CreateTupleDescCopyConstr, GetCCHashEqFuncs, fmgr_info_cxt 等。

関連データ構造

  • CatCache: キャッシュ管理構造体。
  • TupleDesc: タプルの構造定義。
  • ScanKeyData: キャッシュ検索に使用されるスキャンキー。
  • MemoryContext: メモリ割り当ての管理。

失敗モードと境界条件

  • システム属性（attnum < 0）がキーとして指定された場合、FATAL エラーで停止する。
  • CacheMemoryContext が NULL の場合、Assert により停止する。

保守担当者向けメモ

  • 本関数はキャッシュの初回アクセス時に実行される。
  • cc_skey の初期化において、照合順序は常に C_COLLATION_OID が強制される点に注意。
  • CACHE_elog マクロによるデバッグ出力が含まれており、DEBUG2 レベルで動作確認が可能。
