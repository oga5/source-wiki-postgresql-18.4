ginPlaceToPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbtree.c:336

目的

GINインデックスのページに対して、新しいタプルやダウンリンクの挿入を試みます。ページ容量が不足している場合は分割（Split）を実行し、必要に応じて親ページの更新をトリガーします。

シグネチャまたは形状

static bool ginPlaceToPage(GinBtree btree, GinBtreeStack *stack,
                           void *insertdata, BlockNumber updateblkno,
                           Buffer childbuf, GinStatsData *buildStats)

入力と出力

  • **入力**:
  • btree: GIN B-treeの操作コンテキスト。
  • stack: 挿入対象ページを含むツリーパス情報。
  • insertdata: 挿入するデータ。
  • updateblkno: 更新対象のブロック番号（リーフ以外の場合）。
  • childbuf: 分割時に更新が必要な子ページのバッファ。
  • buildStats: インデックス構築時の統計情報（任意）。
  • **出力**:
  • 成功時に true を返します。

前提条件と不変条件

  • 関数実行中は一時的なメモリコンテキストが作成され、終了時に解放されます。
  • ページがリーフの場合、childbuf は無効であり、updateblkno は InvalidBlockNumber である必要があります。
  • リーフ以外の場合、childbuf は有効であり、updateblkno も有効である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: 一時メモリコンテキストを作成し、挿入対象ページの属性（データ/エントリ、リーフ/非リーフ）を判定します。 2. **挿入判定**: btree->beginPlaceToPage を呼び出し、挿入可能か（GPTP_INSERT）、分割が必要か（GPTP_SPLIT）、あるいは何もしないか（GPTP_NO_WORK）を決定します。 3. **挿入処理**:

  • 挿入可能な場合、btree->execPlaceToPage を実行します。
  • 非リーフへの挿入時は、子ページの GIN_INCOMPLETE_SPLIT フラグを解除します。
  • WALが必要な場合は、挿入内容を記録します。

    • 4. **分割処理**:

  • 分割が必要な場合、新しいバッファを確保し、ページ内容を再構築します。
  • ルートページを分割する場合は、新しいルートを作成してツリーの高さを増やします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbtree.c 内の再帰的なツリー操作処理（行736, 825付近）。
  • **呼び出し先**:
  • btree->beginPlaceToPage（関数ポインタ）
  • btree->execPlaceToPage（関数ポインタ）
  • GinNewBuffer
  • XLogInsert 等のWAL関連関数

関連データ構造

  • GinBtree: GIN B-treeの操作を抽象化した構造体。
  • GinBtreeStack: ページへのパスを保持するスタック構造。
  • GinStatsData: インデックス構築時の統計データ。

失敗モードと境界条件

  • ページ容量不足による分割発生時、ルートページが分割されるとツリーの高さが増加します。
  • WAL記録が必要な環境では、START_CRIT_SECTION() 内で処理が完結する必要があります。
  • ページ分割処理はアトミックではないため、中断された場合は GIN_INCOMPLETE_SPLIT フラグにより状態が保持されます。

保守担当者向けメモ

  • 本関数は一時メモリコンテキストを使用するため、関数内で割り当てたメモリの生存期間管理に注意してください。
  • beginPlaceToPage および execPlaceToPage は GinBtree 構造体経由で呼び出されるため、具体的な実装はエントリツリーとデータツリーで異なります。
