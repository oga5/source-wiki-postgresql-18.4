ReplicationSlotsShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slot.c:203

目的

レプリケーションスロット管理用の共有メモリ領域を割り当て、初期化する。

シグネチャまたは形状

void ReplicationSlotsShmemInit(void)

入力と出力

  • 入力: なし（グローバル変数 max_replication_slots を参照）
  • 出力: なし（副作用として共有メモリ上の ReplicationSlotCtl を初期化）

前提条件と不変条件

  • max_replication_slots が 0 の場合、本関数は何も行わず即座に復帰する。
  • 共有メモリの割り当ては ShmemInitStruct を介して行われ、初回実行時のみ初期化処理が実行される。

アルゴリズムまたは振る舞い

1. max_replication_slots が 0 であれば終了する。 2. ShmemInitStruct を呼び出し、ReplicationSlot Ctl という名称で共有メモリを確保する。 3. 初回実行時（found が false の場合）:

  • ReplicationSlotsShmemSize() で算出されたサイズ分、メモリを 0 クリアする。
  • max_replication_slots の数だけループし、各スロットに対して以下の初期化を行う:
  • SpinLockInit によるスロット用ミューテックスの初期化。
  • LWLockInitialize による LWTRANCHE_REPLICATION_SLOT_IO トランシェを用いた I/O 進捗ロックの初期化。
  • ConditionVariableInit によるアクティブ状態監視用条件変数の初期化。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:329 (システム初期化プロセスの一部として呼び出される)
  • 呼び出し先: ShmemInitStruct, ReplicationSlotsShmemSize, MemSet, SpinLockInit, LWLockInitialize, ConditionVariableInit

関連データ構造

  • ReplicationSlotCtlData: 共有メモリ上の制御構造体。
  • ReplicationSlot: 個別のスロット状態を保持する構造体。

失敗モードと境界条件

  • max_replication_slots が 0 の場合、共有メモリの確保は行われない。
  • 共有メモリの確保に失敗した場合、ShmemInitStruct の内部実装に従いエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は PostgreSQL の起動シーケンスにおいて共有メモリをセットアップする重要な役割を担う。
  • スロットのロック階層（slot->mutex や LWLock）の初期化順序は、後続の排他制御の正当性に直結するため注意すること。
