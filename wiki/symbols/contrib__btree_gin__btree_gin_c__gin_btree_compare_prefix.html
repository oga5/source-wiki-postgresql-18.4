gin_btree_compare_prefix

定義

  • 種別: function
  • 定義箇所: contrib/btree_gin/btree_gin.c:108

目的

GINインデックスにおいて、インデックス内の値とクエリ値を比較し、スキャンを継続すべきか、あるいは条件を満たすかを判定する比較関数です。B-treeの比較ロジックをGINのインターフェースに適合させます。

シグネチャまたは形状

static Datum gin_btree_compare_prefix(FunctionCallInfo fcinfo)

入力と出力

  • **入力**: FunctionCallInfo 構造体（fcinfo）。内部から以下の値を取得します。
  • fcinfo->arg[0] (Datum a): 比較対象のインデックス値。
  • fcinfo->arg[1] (Datum b): 比較対象のインデックス値。
  • fcinfo->arg[3] (QueryInfo* data): 検索戦略とクエリ値を保持する構造体。
  • **出力**: int32 型の Datum。
  • 0: 一致（条件を満たす）。
  • 1: 不一致（スキャン終了）。
  • -1: スキャン継続（BTGreaterStrategyNumber の場合）。

前提条件と不変条件

  • data->typecmp には、対象データ型に対応するB-tree比較関数が設定されている必要があります。
  • data->strategy は access/stratnum.h で定義された有効なB-tree戦略番号である必要があります。

アルゴリズムまたは振る舞い

1. data->typecmp を呼び出し、インデックス値とクエリ値（または戦略に応じた比較基準値）を比較します。 2. data->strategy に基づき、以下の判定を行います。

  • BTLessStrategyNumber / BTLessEqualStrategyNumber: 比較結果が条件を満たせば 0、そうでなければ 1 を返します。
  • BTEqualStrategyNumber: 比較結果が 0 であれば 0、そうでなければ 1 を返します。
  • BTGreaterEqualStrategyNumber: 比較結果が条件を満たせば 0、そうでなければ 1 を返します。
  • BTGreaterStrategyNumber: 比較結果が条件を満たせば 0、一致する場合は -1（継続）、それ以外は 1 を返します。

    • 3. 認識できない戦略番号が渡された場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は contrib/btree_gin/btree_gin.c:203 です。
  • **呼び出し先**: data->typecmp に格納された関数ポインタ（各データ型の比較関数）を呼び出します。

関連データ構造

  • QueryInfo: 検索戦略とクエリ値を保持する構造体。

失敗モードと境界条件

  • 未定義の strategy が渡された場合、ERROR レベルで処理が中断されます。
  • typecmp 関数が予期しない値を返した場合の挙動は、その関数に依存します。

保守担当者向けメモ

  • 本関数は GIN_SUPPORT マクロを通じて各データ型に適用されます。新しいデータ型を追加する際は、適切な比較関数が QueryInfo に設定されていることを確認してください。
