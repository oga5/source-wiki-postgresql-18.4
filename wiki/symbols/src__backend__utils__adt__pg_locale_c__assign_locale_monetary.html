assign_locale_monetary

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:371

目的

lc_monetary 設定が変更された際に、キャッシュされた通貨フォーマット情報を無効化する GUC（Grand Unified Configuration）の割り当てフック関数です。

シグネチャまたは形状

void assign_locale_monetary(const char *newval, void *extra)

入力と出力

  • newval: 設定される新しいロケール文字列（本関数内では使用されません）。
  • extra: GUC システムから渡される追加データ（本関数内では使用されません）。
  • 戻り値: なし。

前提条件と不変条件

  • 本関数は lc_monetary 設定値の変更時に呼び出されることを前提としています。
  • 実行後、CurrentLocaleConvValid が false に設定される不変条件を持ちます。

アルゴリズムまたは振る舞い

1. CurrentLocaleConvValid 変数に false を代入します。 2. これにより、次回通貨フォーマット情報が必要になった際に、システムが最新のロケール設定に基づいて情報を再取得（再計算）するように強制します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/guc_tables.c 内の GUC 設定テーブルから、lc_monetary の変更時にコールバックとして呼び出されます。
  • 呼び出し先: なし。

関連データ構造

  • CurrentLocaleConvValid: 通貨フォーマット情報の有効性を示すグローバルなフラグ変数。

失敗モードと境界条件

  • 特になし。本関数は常に成功し、副作用としてフラグの更新のみを行います。

保守担当者向けメモ

  • 本関数は単なるフラグの無効化を行う軽量なフックです。通貨フォーマット情報の実際の再取得は、PGLC_localeconv() 等の利用側で行われます。
  • lc_monetary 以外のロケール設定（lc_numeric や lc_time）にはそれぞれ専用の割り当て関数が存在するため、混同しないよう注意してください。
