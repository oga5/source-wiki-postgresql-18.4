remoteDep

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/pg_shdepend.c:600

目的

特定のデータベース（dbOid）に関連付けられた共有依存関係の発生回数（count）を追跡・集計するために使用されます。

シグネチャまたは形状

typedef struct
{
    Oid         dbOid;
    int         count;
} remoteDep;

入力と出力

  • 入力: palloc を介してメモリが割り当てられ、dbOid と count が設定されます。
  • 出力: 共有依存関係の集計リストの一部として、リスト内の要素（lfirst(cell)）として参照されます。

前提条件と不変条件

  • dbOid は有効なデータベースの OID である必要があります。
  • count は、特定のデータベース内における共有オブジェクトへの依存関係の数を示します。

アルゴリズムまたは振る舞い

共有依存関係の検索や処理において、データベースごとの依存関係数を保持する一時的なデータ構造として機能します。リスト操作を通じて、特定のデータベースに関連する依存関係の集計や比較に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_shdepend.c 内の関数（773行目、794行目、838行目付近）で、リストの走査やメモリ確保時に参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • List (PostgreSQLのリスト構造): remoteDep のインスタンスを格納するために使用されます。

失敗モードと境界条件

  • メモリ不足: palloc 呼び出し時にメモリが不足した場合、PostgreSQL の標準的なエラー処理により中断されます。
  • データベース OID の不整合: 存在しないデータベース OID が設定された場合、後続の依存関係チェックで不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は pg_shdepend.c 内のローカルな集計処理専用です。
  • 共有依存関係の集計ロジックを変更する際は、count のインクリメントおよびリスト管理の整合性に注意してください。
