btbuildempty

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtree.c:178

目的

B-tree インデックスの初期化用フォーク（initialization fork）において、空のインデックス構造を構築する。

シグネチャまたは形状

void btbuildempty(Relation index)

入力と出力

  • **入力**: index (構築対象のインデックスリレーション)
  • **出力**: なし（副作用としてストレージ上にメタページが書き込まれる）

前提条件と不変条件

  • index は有効なリレーションであること。
  • smgr_bulk_start_rel が正常に開始できること。

アルゴリズムまたは振る舞い

1. _bt_allequalimage を呼び出し、インデックスが等価イメージ（equal image）をサポートするかを判定する。 2. smgr_bulk_start_rel を使用して、初期化フォークに対するバルク書き込みセッションを開始する。 3. smgr_bulk_get_buf でメタページ用のバッファを取得する。 4. _bt_initmetapage を呼び出し、取得したバッファをメタページとして初期化する。 5. smgr_bulk_write を使用して、メタページを BTREE_METAPAGE（インデックスの先頭ページ）として書き込む。 6. smgr_bulk_finish を呼び出し、バルク書き込みセッションを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtree.c 内の IndexAmRoutine 構造体（ambuildempty フィールド）を通じて呼び出される。
  • **呼び出し先**:
  • _bt_allequalimage
  • smgr_bulk_start_rel
  • smgr_bulk_get_buf
  • _bt_initmetapage
  • smgr_bulk_write
  • smgr_bulk_finish

関連データ構造

  • Relation: インデックスのメタデータを含む構造体。
  • BulkWriteState: バルク書き込みの進行状態を管理する構造体。
  • BulkWriteBuffer: 書き込み対象のページバッファ。

失敗モードと境界条件

  • ストレージ層（smgr）の書き込みエラーが発生した場合、例外がスローされる可能性がある。
  • メモリ不足やリソース制限により、バルク書き込みの初期化が失敗する場合がある。

保守担当者向けメモ

  • 本関数はインデックスの初期化時にのみ使用される。
  • _bt_initmetapage に渡される引数は、インデックスの物理的な整合性に直結するため、変更時は注意が必要である。
