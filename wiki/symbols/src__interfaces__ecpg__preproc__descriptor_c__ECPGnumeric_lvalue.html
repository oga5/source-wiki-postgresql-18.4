ECPGnumeric_lvalue

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:44

目的

ECPGプリプロセスにおいて、指定された変数が数値型であることを検証し、有効な場合にその変数名をコード出力ストリームへ書き出します。

シグネチャまたは形状

static void ECPGnumeric_lvalue(char *name)

入力と出力

  • **入力**: name (変数名を示す文字列)
  • **出力**: なし（副作用として base_yyout への書き出し、またはエラー報告を行う）

前提条件と不変条件

  • find_variable(name) が呼び出し可能であること。
  • base_yyout が有効な出力ストリームとして初期化されていること。

アルゴリズムまたは振る舞い

1. find_variable(name) を呼び出し、変数情報を取得します。 2. 変数の型（v->type->type）を判定します。 3. 数値型（short, int, long, long long の各符号付き/符号なし、および const）であれば、fputs(name, base_yyout) を実行して変数名をコード出力ストリームに書き出します。 4. 上記以外の型である場合、mmerror を通じて PARSE_ERROR を発生させ、エラーメッセージを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c:164
  • src/interfaces/ecpg/preproc/descriptor.c:216
  • **呼び出し先**:
  • find_variable (変数検索)
  • mmerror (エラー報告)
  • fputs (標準ライブラリ関数)

関連データ構造

  • struct variable: 変数の型情報を保持する構造体。
  • base_yyout: コード生成先の出力ストリーム。

失敗モードと境界条件

  • **失敗モード**: 指定された name が find_variable で見つからない場合、v が NULL となりセグメンテーションフォールトを引き起こす可能性があります。
  • **境界条件**: 数値型以外の変数が渡された場合、mmerror によりプリプロセスが停止します。

保守担当者向けメモ

  • 本関数は GET/SET DESCRIPTOR 文における変数代入の型チェックを担っています。
  • 数値型として許可する列挙型（ECPGt_*）の追加・変更が必要な場合は、switch 文を更新してください。
