config_enum_entry

定義

  • 種別: struct
  • 定義箇所: src/include/utils/guc.h:173

目的

GUC（Grand Unified Configuration）システムにおいて、列挙型（Enum）として定義される設定パラメータの各選択肢を表現するために使用されます。

シグネチャまたは形状

struct config_enum_entry
{
	const char *name;  /* 選択肢の名称 */
	int			val;   /* 内部的な整数値 */
	bool		hidden; /* ユーザーへの表示を隠すか否か */
};

入力と出力

  • 入力: 設定パラメータの定義時に、この構造体の配列として選択肢リストが提供されます。
  • 出力: GUCシステムは、ユーザーが指定した文字列（name）をこのリストと照合し、対応する整数値（val）を内部状態として保持します。

前提条件と不変条件

  • name は一意である必要があります。
  • 配列の終端は、通常 name が NULL であるエントリによって示されます。

アルゴリズムまたは振る舞い

  • ユーザーが設定値を変更する際、GUCシステムはこの構造体のリストを走査し、入力文字列と一致する name を検索します。
  • 一致した場合、対応する val が設定値として適用されます。
  • hidden が true の場合、設定値の一覧表示やヘルプ出力においてその選択肢は除外されます。

呼び出し元と呼び出し先

  • 呼び出し元（定義側）:

  • contrib/pgcrypto/pgcrypto.c
  • src/backend/access/rmgrdesc/xlogdesc.c

  • src/backend/storage/aio/aio.c

  • 呼び出し先: GUCシステムの内部ロジック（guc.c）が、設定値の検証および変換のためにこの構造体を参照します。

関連データ構造

  • GucEnumAssignHook: 列挙型変数の値が変更された際に呼び出されるフック関数。

失敗モードと境界条件

  • 未定義の文字列が指定された場合、GUCシステムはエラーを報告し、設定変更を拒否します。
  • hidden フラグの誤設定により、有効な設定値がユーザーから見えなくなる可能性があります。

保守担当者向けメモ

  • 新しい列挙型パラメータを追加する際は、必ず NULL 終端のエントリを配列の最後に追加してください。
  • hidden 属性は、非推奨のオプションや内部テスト用オプションを隠すために使用されます。
