ExecInitInterpreter

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExprInterp.c:2935

目的

式評価インタープリタで使用されるディスパッチテーブルおよび逆引きテーブルの初期化を行います。

シグネチャまたは形状

static void ExecInitInterpreter(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルなディスパッチテーブルを構築する）

前提条件と不変条件

  • EEO_USE_COMPUTED_GOTO が定義されている環境でのみ実行されます。
  • dispatch_table が NULL の場合にのみ初期化処理が実行されます。

アルゴリズムまたは振る舞い

1. dispatch_table が未初期化の場合、ExecInterpExpr(NULL, NULL, NULL) を呼び出してディスパッチテーブルを取得します。 2. 全ての命令（EEOP_LAST まで）を走査し、reverse_dispatch_table に命令コードと対応するオペコードを格納します。 3. reverse_dispatch_table を dispatch_compare_ptr を用いてソートし、bsearch() による検索を可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execExprInterp.c 内の複数箇所から参照されています。
  • 呼び出し先: ExecInterpExpr（ディスパッチテーブル取得のため）、qsort（標準ライブラリ）。

関連データ構造

  • ExprEvalOp: 実行可能な命令の列挙型。
  • ExprEvalOpLookup: 命令とオペコードの対応を保持する構造体。
  • dispatch_table: 命令のジャンプ先アドレスを保持するグローバル配列。
  • reverse_dispatch_table: オペコードから命令を逆引きするための配列。

失敗モードと境界条件

  • EEO_USE_COMPUTED_GOTO が定義されていない環境では、本関数は何も行いません。
  • dispatch_table が既に初期化されている場合は再実行されません。

保守担当者向けメモ

  • 本関数は式評価のホットパスにおけるディスパッチ効率を最適化するための準備処理です。
  • reverse_dispatch_table のソート順序や検索アルゴリズムを変更する場合は、dispatch_compare_ptr の実装と整合性を取る必要があります。
