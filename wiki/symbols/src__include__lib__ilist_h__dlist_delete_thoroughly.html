dlist_delete_thoroughly

定義

  • 種別: static inline function
  • 定義箇所: src/include/lib/ilist.h:415

目的

二重連結リストから特定のノードを削除し、そのノードのリンクポインタを NULL に初期化することで、リストから完全に切り離された状態を保証します。

シグネチャまたは形状

static inline void dlist_delete_thoroughly(dlist_node *node)

入力と出力

  • **入力**: dlist_node *node - リストから削除対象となるノードへのポインタ。
  • **出力**: なし（副作用としてノードのリンクを更新）。

前提条件と不変条件

  • node は有効な二重連結リストに含まれている必要があります。
  • node がリストの先頭や末尾である場合、リストの head 構造体側が適切に更新されている必要があります（本関数はノード間の接続のみを操作します）。

アルゴリズムまたは振る舞い

1. node の前後のノード（prev および next）のポインタを直接書き換え、node をリストから除外します。 2. node->next を NULL に設定します。 3. node->prev を NULL に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/syncrep.c
  • src/backend/storage/lmgr/predicate.c
  • src/include/lib/ilist.h（内部的な再帰的利用）
  • **呼び出し先**:
  • 本関数はポインタ操作のみを行うため、他の関数を呼び出しません。

関連データ構造

  • dlist_node: 二重連結リストのリンクを保持する構造体。

失敗モードと境界条件

  • node がリストに含まれていない場合や、既に NULL に初期化されている場合に呼び出すと、不正なメモリアクセス（セグメンテーション違反）が発生する可能性があります。
  • リストの head が指すポインタが更新されないため、リストの先頭ノードを削除する際は、呼び出し側で head の更新を別途行う必要があります。

保守担当者向けメモ

  • dlist_delete と異なり、削除後にノードのポインタを NULL にクリアするため、削除済みノードへの誤ったアクセスを早期に検出したい場合に適しています。
  • インライン関数として定義されているため、ヘッダーファイル内で完結しています。
