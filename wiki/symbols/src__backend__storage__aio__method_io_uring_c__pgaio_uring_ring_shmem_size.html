pgaio_uring_ring_shmem_size

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_io_uring.c:241

目的

io_uring のリング構造体に必要な共有メモリサイズを算出する。

シグネチャまたは形状

size_t pgaio_uring_ring_shmem_size(void)

入力と出力

  • 入力: なし（グローバルな pgaio_uring_caps および pgaio_uring_procs() を参照）
  • 出力: 必要な共有メモリの合計サイズ（バイト単位）

前提条件と不変条件

  • pgaio_uring_caps.mem_init_size が 0 より大きい場合にのみ、正のサイズが計算される。
  • 算出されるサイズには、ページ境界へのアライメント用としてシステムページサイズ分が含まれる。

アルゴリズムまたは振る舞い

1. pgaio_uring_caps.mem_init_size が 0 より大きいか確認する。 2. 0 より大きい場合、システムページサイズ（_SC_PAGESIZE）を初期値として確保する。 3. 全バックエンド数（pgaio_uring_procs()）と、バックエンドごとの必要メモリサイズ（pgaio_uring_caps.mem_init_size）を乗算し、その結果を合計サイズに加算して返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/aio/method_io_uring.c 内の pgaio_uring_shmem_size（直接的な呼び出しを確認）
  • その他、共有メモリ確保に関連する初期化処理から参照されている。
  • 呼び出し先:
  • add_size, mul_size（PostgreSQL の安全な算術演算関数）
  • sysconf(_SC_PAGESIZE)（システムコール）
  • pgaio_uring_procs()（関数）

関連データ構造

  • PgAioUringCaps: io_uring の機能およびメモリ要件を保持する構造体。

失敗モードと境界条件

  • pgaio_uring_caps.mem_init_size が 0 の場合、戻り値は 0 となる。
  • 算術演算においてオーバーフローが発生した場合、add_size および mul_size の内部仕様に従いエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数で算出されるメモリは、HugePage を使用する場合でも HugePage 境界へのアライメントは不要である。
  • ページ境界へのアライメントが必要なため、将来的に sysconf の呼び出し箇所を変更する場合は注意すること。
