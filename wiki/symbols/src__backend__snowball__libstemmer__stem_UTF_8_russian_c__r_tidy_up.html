r_tidy_up

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_russian.c:534

目的

ロシア語のステミング処理における最終的な語尾調整（tidy up）を行う。

シグネチャまたは形状

static int r_tidy_up(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowballの実行環境構造体）
  • 出力: 成功時に 1、マッチしなかった場合に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • find_among_b を用いて、単語末尾から a_7 配列に定義されたパターンを後方検索する。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置に設定し、a_7 を用いて後方検索を実行する。 2. マッチした among_var の値に応じて以下の処理を行う：

  • **case 1**: マッチした接尾辞を削除し、さらに特定の文字列（s_6 および s_7）が連続して存在する場合、それらも削除する。
  • **case 2**: 特定の文字列（s_8）が直前に存在する場合、マッチした接尾辞を削除する。
  • **case 3**: マッチした接尾辞を単に削除する。

    • 3. 削除処理（slice_del）の戻り値が負の場合は、その値をそのまま返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_russian.c および src/backend/snowball/libstemmer/stem_KOI8_R_russian.c 内のメイン処理から呼び出される。
  • 呼び出し先: find_among_b, slice_del, eq_s_b（いずれもSnowballランタイム関数）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_7: 調整対象となる接尾辞パターンを格納した among 構造体の配列。

失敗モードと境界条件

  • find_among_b がマッチしなかった場合は 0 を返す。
  • slice_del がメモリ不足やバッファ操作エラーにより負の値を返した場合、そのエラーコードを伝播させる。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接Cファイルを編集せず、元のSnowballスクリプト（.sbl）を修正すること。
  • a_7 配列および s_6, s_7, s_8 などの定数文字列は、ロシア語のステミングルールに基づいている。
