fill_ident_view

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/hbafuncs.c:539

目的

pg_ident.conf ファイルの内容をパースし、その結果を Tuplestore に格納して SQL ビューとして参照可能にする。

シグネチャまたは形状

static void fill_ident_view(Tuplestorestate *tuple_store, TupleDesc tupdesc)

入力と出力

  • **入力**:
  • tuple_store: 結果を蓄積するための Tuplestorestate へのポインタ。
  • tupdesc: 出力行の構造を定義する TupleDesc。
  • **出力**: なし（tuple_store を直接更新する）。

前提条件と不変条件

  • IdentFileName で指定されたファイルがシステム上に存在し、読み取り可能であること。
  • Tuplestore は呼び出し元で適切に初期化されていること。

アルゴリズムまたは振る舞い

1. open_auth_file を使用して pg_ident.conf を開く（失敗時はエラーを投げる）。 2. tokenize_auth_file を呼び出し、ファイル内容をトークンリストに変換する。 3. AllocSetContextCreate で一時的なメモリコンテキストを作成し、パース処理中のメモリを管理する。 4. トークンリストを走査し、各行に対して以下の処理を行う：

  • parse_ident_line で行を構造体に変換（エラーがある行はスキップ）。
  • エラーがない行については、マッピング番号（map_number）をインクリメントする。
  • fill_ident_line を呼び出し、パース結果またはエラー情報を Tuplestore に書き込む。

    • 5. 最後にメモリコンテキストを破棄し、ファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_ident_file_mappings 関数（src/backend/utils/adt/hbafuncs.c:607）。
  • **呼び出し先**:
  • open_auth_file
  • tokenize_auth_file
  • parse_ident_line
  • fill_ident_line
  • free_auth_file

関連データ構造

  • Tuplestorestate: 結果保持用。
  • TokenizedAuthLine: トークン化された行データ。
  • IdentLine: パース済みのマッピング定義。

失敗モードと境界条件

  • ファイルが開けない場合、open_auth_file によりエラーが発生し、関数は中断される。
  • パースエラーが発生した場合、tok_line->err_msg が設定され、fill_ident_line を通じてビューの error カラムに情報が記録される。

保守担当者向けメモ

  • 本関数はメモリ管理のために MemoryContext を明示的に切り替えている。パース処理に関連するメモリリークを防ぐため、identcxt の作成・削除のフローを変更する際は注意が必要。
  • map_number はエラーのない行に対してのみインクリメントされる仕様である。
