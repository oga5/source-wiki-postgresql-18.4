dfa_backref

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:505

目的

正規表現における後方参照（backreference）の文字列を特定し、指定された範囲内で最適な一致長を算出する。

シグネチャまたは形状

static chr *dfa_backref(struct vars *v, struct dfa *d, chr *start, chr *min, chr *max, bool shortest)

入力と出力

  • **入力**:
  • v: 正規表現の実行状態を保持する vars 構造体。
  • d: DFA 実行状態を保持する struct dfa。
  • start: マッチングを開始する位置。
  • min, max: マッチング終了位置の許容範囲。
  • shortest: 最短一致を優先するかどうかを示すフラグ。
  • **出力**: マッチングが成功した場合は終了位置の chr * を返し、失敗した場合は NULL を返す。

前提条件と不変条件

  • 呼び出し元は、後方参照対象の文字列が有効であることを確認済みであること。
  • v->pmatch[n].rm_so が -1 の場合、本関数は直ちに NULL を返す。

アルゴリズムまたは振る舞い

1. v->pmatch から参照対象の文字列（brstring）と長さ（brlen）を取得する。 2. brlen が 0 の場合、min == start かつ backmin <= backmax であれば start を返す。 3. min および max を brlen に基づく繰り返し回数（minreps, maxreps）に変換し、DFA 定義の backmin/backmax で制約を適用する。 4. shortest が真かつ minreps == 0 の場合は start を返す。 5. brstring と現在の位置 p を v->g->compare を用いて繰り返し比較し、一致する限り p を進める。 6. shortest が真の場合は、minreps に達した時点で比較を終了する。 7. 最終的な繰り返し回数が minreps 以上であれば、到達した位置 p を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/rege_dfa.c 内の各所（longest や shortest 等のメインループから呼び出される）。
  • **呼び出し先**: v->g->compare（関数ポインタ経由で文字列比較を実行）。

関連データ構造

  • struct vars: 正規表現実行時のコンテキスト。
  • struct dfa: DFA 実行状態。
  • struct dfa 内の backno, backmin, backmax: 後方参照の制約情報。

失敗モードと境界条件

  • 後方参照が未設定（rm_so == -1）の場合は NULL を返す。
  • brlen が 0 の場合、制約を満たさなければ NULL を返す。
  • maxreps < minreps の場合は NULL を返す。
  • 文字列比較が一致しない、または繰り返し回数が minreps に満たない場合は NULL を返す。

保守担当者向けメモ

  • lacon() と同様に、再帰的な呼び出しや複雑な制約評価を伴う可能性があるため、スタック使用量に注意すること。
  • v->g->compare は実行時のロケールや文字セットに依存するため、比較ロジックの変更時は注意が必要。
