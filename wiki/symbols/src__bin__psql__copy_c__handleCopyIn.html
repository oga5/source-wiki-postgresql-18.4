handleCopyIn

定義

  • 種別: function
  • 定義箇所: src/bin/psql/copy.c:510

目的

ローカルのストリーム（ファイル、パイプ、または標準入力）からデータを読み込み、libpq の COPY プロトコルを使用してサーバーへ送信します。

シグネチャまたは形状

bool handleCopyIn(PGconn *conn, FILE *copystream, bool isbinary, PGresult **res)

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル。
  • copystream: 読み込み元のファイルストリーム。
  • isbinary: バイナリモードかテキストモードかを指定するフラグ。
  • **出力**:
  • res: 処理結果を格納する PGresult へのポインタ。
  • 戻り値: 転送が正常に完了した場合は true、エラーまたはユーザーによる中断が発生した場合は false。

前提条件と不変条件

  • sigsetjmp を使用して、割り込み（Ctrl+C）発生時の復帰ポイントを確立します。
  • テキストモードでは、入力ストリームの終端を示す \. シーケンスを検知する必要があります。
  • バッファリング処理において、EOF マーカーが分割されないよう、常に 5 バイト以上の空き領域を確保して読み込みを行います。

アルゴリズムまたは振る舞い

1. **割り込み設定**: sigsetjmp を設定し、割り込み時に PQputCopyEnd を呼び出して転送を終了し、クリーンアップへ移行するようにします。 2. **対話モード判定**: isatty を使用して対話型入力か判定し、必要に応じてプロンプトを表示します。 3. **データ転送**:

  • **バイナリモード**: fread でバッファへ読み込み、PQputCopyData でサーバーへ送信するループを実行します。
  • **テキストモード**: fgets で行単位に読み込みます。入力が \. で終わる行の場合、それを EOF とみなして転送を終了します。

    • 4. **終了処理**: 転送完了後、またはエラー発生時にリソースを解放し、ステータスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/common.c:971
  • **呼び出し先**:
  • libpq 関数群 (PQputCopyData, PQputCopyEnd, PQprotocolVersion)
  • 標準ライブラリ (fread, fgets, isatty, puts, fputs, fflush, ferror)
  • その他 (get_prompt, _ (国際化マクロ))

関連データ構造

  • PGconn: libpq の接続オブジェクト。
  • PGresult: libpq の結果オブジェクト。
  • pset: psql のグローバル設定構造体（pset.quiet, pset.cur_cmd_source, pset.lineno 等を参照）。

失敗モードと境界条件

  • **ユーザー中断**: sigsetjmp によりキャッチされ、PQputCopyEnd を経由して安全に終了します。
  • **読み込みエラー**: ferror でストリームエラーを検知した場合、false を返します。
  • **転送エラー**: PQputCopyData が 0 以下の値を返した場合、転送を中断します。
  • **EOF マーカー**: テキストモードにおいて、\. が行頭にある場合のみ EOF として処理されます。

保守担当者向けメモ

  • sigint_interrupt_enabled フラグの切り替えにより、sigsetjmp の有効範囲を制御しています。
  • テキストモードでの \. 判定ロジックは、SQL スクリプト内にインライン化されたデータと対話型入力を区別するために重要です。
  • プロトコルバージョン 2 以前の古い libpq との互換性のためのチェックが含まれています。
