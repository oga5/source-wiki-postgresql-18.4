TM_FailureData

定義

  • 種別: struct
  • 定義箇所: src/include/access/tableam.h:146

目的

テーブル操作（更新、削除、ロック）が失敗した際に、その原因となったタプルの詳細情報や、操作の追跡状態を呼び出し元へ伝達するために使用されます。

シグネチャまたは形状

typedef struct TM_FailureData
{
	ItemPointerData ctid;
	TransactionId xmax;
	CommandId	cmax;
	bool		traversed;
} TM_FailureData;

入力と出力

本構造体は関数引数としてポインタ経由で渡され、操作結果に応じて以下のフィールドが更新されます。

  • ctid: 失敗の原因となったタプルの物理位置。
  • xmax: タプルをロックまたは削除したトランザクションID。
  • cmax: タプルを削除したコマンドID。
  • traversed: 失敗の過程でリレーションの再スキャンや移動が発生したかを示すフラグ。

前提条件と不変条件

  • 呼び出し側は、本構造体を初期化した上でポインタを渡す必要があります。
  • ctid は有効な ItemPointerData であることが期待されます。

アルゴリズムまたは振る舞い

テーブルアクセス手法（Table AM）の操作関数（tuple_update や tuple_delete 等）が、MVCCの競合やロックの衝突により操作を完了できない場合に、この構造体に詳細情報を格納して呼び出し元へ返します。呼び出し元はこの情報を基に、再試行やエラーハンドリングを決定します。

呼び出し元と呼び出し先

  • **呼び出し元（確認されているもの）**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/table/tableam.c
  • src/backend/commands/trigger.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • TM_Result: 操作結果を示す列挙型。
  • ItemPointerData: タプルの物理位置を指す構造体。
  • TransactionId, CommandId: トランザクションおよびコマンドの識別子。

失敗モードと境界条件

  • xmax が InvalidTransactionId の場合、タプルはロックされていないか、削除されていないことを示します。
  • traversed が true の場合、呼び出し元はタプルの位置が変更された可能性があることを考慮する必要があります。

保守担当者向けメモ

  • 本構造体は Table AM のインターフェースにおいて、エラー時の詳細なフィードバックを標準化するために不可欠です。
  • 新しいストレージエンジンを実装する際は、TM_Result と組み合わせて、この構造体に正確な情報を設定するようにしてください。
