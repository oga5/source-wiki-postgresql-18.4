PQfireResultCreateEvents

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/libpq-events.c:184

目的

PGresult オブジェクトに対して、登録済みのイベントプロシージャへ PGEVT_RESULTCREATE イベントを通知する。

シグネチャまたは形状

int PQfireResultCreateEvents(PGconn *conn, PGresult *res);

入力と出力

  • **入力**:
  • conn: 関連する PGconn オブジェクトへのポインタ。
  • res: イベントを通知する対象の PGresult オブジェクトへのポインタ。
  • **出力**:
  • int: すべてのイベントプロシージャが正常に処理を完了した場合は true (1)、一つでも失敗した場合は false (0) を返す。

前提条件と不変条件

  • res が NULL の場合、関数は直ちに false を返す。
  • 各イベントプロシージャは、res->events[i].resultInitialized フラグにより、既にイベントが発火済みかどうかが管理される。

アルゴリズムまたは振る舞い

1. res が有効か確認する。 2. res に登録されているイベント配列を順次走査する。 3. resultInitialized が未設定のイベントに対してのみ、以下の処理を行う:

  • PGEventResultCreate 構造体を初期化し、conn と res を格納する。
  • PGEVT_RESULTCREATE イベント ID を指定して、登録されたイベントプロシージャを呼び出す。
  • プロシージャが成功（非ゼロ）を返した場合、resultInitialized を true に設定する。
  • プロシージャが失敗（ゼロ）を返した場合、戻り値の result を false に更新する。

    • 4. 全てのイベント処理終了後、最終的な成功状態を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-exec.c:2231
  • **呼び出し先**: res->events[i].proc（登録されたイベントプロシージャ）

関連データ構造

  • PGconn: 接続状態を保持する構造体。
  • PGresult: クエリ結果を保持する構造体。
  • PGEventResultCreate: イベントプロシージャに渡される引数構造体。

失敗モードと境界条件

  • res が NULL の場合は false を返す。
  • 登録されたイベントプロシージャのいずれかが false を返した場合、この関数は false を返す（ただし、他のプロシージャの呼び出しは継続される）。

保守担当者向けメモ

  • 本関数は PGresult の作成時に呼び出されることを想定している。
  • resultInitialized フラグにより、同一の PGresult に対してイベントが重複して発火することを防いでいる。
