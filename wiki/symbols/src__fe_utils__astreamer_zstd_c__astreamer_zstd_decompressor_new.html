astreamer_zstd_decompressor_new

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_zstd.c:258

目的

Zstandard形式のデータを伸長するための astreamer インスタンスを生成し、パイプラインに組み込む。

シグネチャまたは形状

astreamer *astreamer_zstd_decompressor_new(astreamer *next)

入力と出力

  • **入力**: next (後続の処理を行う astreamer オブジェクトへのポインタ)
  • **出力**: 伸長処理を行う astreamer オブジェクトへのポインタ。USE_ZSTD が未定義の場合は pg_fatal によりプロセスが終了する。

前提条件と不変条件

  • next は NULL であってはならない。
  • USE_ZSTD が定義されている環境でのみ動作する。
  • 内部の bbs_buffer は ZSTD_DStreamOutSize() に基づいて初期化される。

アルゴリズムまたは振る舞い

1. astreamer_zstd_frame 構造体用のメモリを確保し、astreamer_zstd_decompressor_ops を操作関数として設定する。 2. bbs_next に next を格納し、bbs_buffer を初期化・拡張する。 3. ZSTD_createDCtx() を呼び出し、伸長コンテキストを作成する。失敗した場合は pg_fatal を発行する。 4. ZSTD_outBuffer 構造体を bbs_buffer のデータ領域とサイズで初期化する。 5. 初期化済みの astreamer オブジェクトを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c:1272
  • src/bin/pg_verifybackup/pg_verifybackup.c:1312
  • **呼び出し先**:
  • palloc0
  • initStringInfo / enlargeStringInfo
  • ZSTD_createDCtx
  • ZSTD_DStreamOutSize
  • pg_fatal

関連データ構造

  • astreamer: ストリーミング処理の基底構造体。
  • astreamer_zstd_frame: Zstd圧縮・伸長コンテキストとバッファを保持する内部構造体。
  • astreamer_ops: ストリーマーの操作関数テーブル。

失敗モードと境界条件

  • next が NULL の場合、Assert により異常終了する。
  • ZSTD_createDCtx() が NULL を返した場合、pg_fatal によりエラー終了する。
  • USE_ZSTD が定義されていないビルドでは、呼び出し時に pg_fatal が発生する。

保守担当者向けメモ

  • 本関数は astreamer パイプラインの伸長ステップを構築する。
  • bbs_buffer のサイズ管理は ZSTD_DStreamOutSize() に依存しているため、zstdライブラリの更新時にはこのバッファサイズ要件を確認すること。
