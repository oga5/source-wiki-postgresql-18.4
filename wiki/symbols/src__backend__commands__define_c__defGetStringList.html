defGetStringList

定義

  • 種別: function
  • 定義箇所: src/backend/commands/define.c:342

目的

DefElem 構造体の arg フィールドから、文字列のリスト（List）を抽出および検証します。

シグネチャまたは形状

List *defGetStringList(DefElem *def)

入力と出力

  • **入力**: DefElem *def （解析対象の定義要素）
  • **出力**: List * （文字列ノードを含むリスト）
  • **戻り値**: 成功時は def->arg が指すリストを返します。

前提条件と不変条件

  • def->arg が NULL であってはなりません（NULL の場合はエラーを報告します）。
  • def->arg は T_List 型のノードである必要があります。
  • リスト内の各要素は String 型である必要があります。

アルゴリズムまたは振る舞い

1. def->arg が NULL の場合、ERRCODE_SYNTAX_ERROR でエラーを報告します。 2. nodeTag(def->arg) を確認し、T_List でない場合はエラーを報告します。 3. リスト内の各要素を走査し、すべての要素が String 型であることを確認します。 4. 検証に成功した場合、リストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には具体的な呼び出し元の記述はありません。
  • **呼び出し先**:
  • ereport / elog（エラー報告）
  • nodeTag（ノード型判定）
  • IsA（ノード型チェック）
  • lfirst（リスト要素の取得）

関連データ構造

  • DefElem: SQL コマンドのオプション引数を保持する構造体。
  • List: PostgreSQL の汎用リスト構造。
  • String: 文字列値を保持するノード型。

失敗モードと境界条件

  • def->arg が NULL の場合: ereport により構文エラーを発生させます。
  • def->arg が T_List ではない場合: elog によりエラーを発生させます。
  • リスト内に String 型以外のノードが含まれる場合: elog によりエラーを発生させます。

保守担当者向けメモ

  • 本関数は、SQL コマンドのオプションとして文字列のリストを期待する箇所で利用されます。
  • リスト内の要素がすべて文字列であることを保証するため、型チェックを厳格に行っています。
