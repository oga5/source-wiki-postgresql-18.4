bt_index_block_validate

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/btreefuncs.c:224

目的

B-treeインデックスの指定されたブロックに対するアクセスが、セキュリティおよび整合性の観点から許可されているかを検証します。

シグネチャまたは形状

static void bt_index_block_validate(Relation rel, int64 blkno)

入力と出力

  • **入力**:
  • rel: 検証対象のリレーション（Relation）。
  • blkno: 検証対象のブロック番号（int64）。
  • **出力**: なし。検証に失敗した場合はエラーを報告し、処理を中断します。

前提条件と不変条件

  • 対象リレーションはB-treeインデックスである必要があります。
  • 他のセッションのローカル一時テーブルへのアクセスは禁止されています。
  • メタページ（ブロック0）への直接アクセスは許可されません。

アルゴリズムまたは振る舞い

1. **型検証**: IS_INDEX および IS_BTREE マクロを用いて、対象がB-treeインデックスであることを確認します。不適合な場合は ERRCODE_WRONG_OBJECT_TYPE でエラーを発生させます。 2. **一時テーブル制限**: RELATION_IS_OTHER_TEMP を確認し、他セッションの一時テーブルであれば ERRCODE_FEATURE_NOT_SUPPORTED でエラーを発生させます。 3. **メタページ保護**: blkno が0の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを発生させます。 4. **範囲チェック**: check_relation_block_range を呼び出し、指定されたブロック番号がリレーションの有効範囲内にあるかを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/btreefuncs.c 内の各関数（bt_page_stats、bt_multi_page_stats、bt_page_items 等）から、インデックスページ操作の直前に呼び出されます。
  • **呼び出し先**:
  • check_relation_block_range（同一ファイル内）。

関連データ構造

  • Relation: PostgreSQLのシステムカタログにおけるリレーション定義。

失敗モードと境界条件

  • **エラー発生**:
  • インデックスではない、またはB-treeではないリレーションが渡された場合。
  • 他セッションの一時テーブルを指定した場合。
  • メタページ（ブロック0）を指定した場合。
  • ブロック番号がリレーションの物理的なサイズを超過している場合。

保守担当者向けメモ

  • 本関数は、pageinspect モジュールが提供する各SQL関数が安全にインデックスへアクセスするためのゲートキーパーとして機能します。
  • 新たなB-tree検査関数を追加する際は、必ず本関数による検証を先行させてください。
