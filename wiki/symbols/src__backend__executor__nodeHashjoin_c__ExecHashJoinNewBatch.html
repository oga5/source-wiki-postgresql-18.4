ExecHashJoinNewBatch

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHashjoin.c:1129

目的

ハイブリッド・ハッシュ結合において、現在のバッチ処理を終了し、次のバッチへ移行するための準備を行う。メモリ内に収まらないタプルをディスクから再ロードし、ハッシュテーブルを再構築する。

シグネチャまたは形状

static bool ExecHashJoinNewBatch(HashJoinState *hjstate)

入力と出力

  • **入力**: hjstate (HashJoinState) - 結合ノードの実行状態を保持する構造体。
  • **出力**: bool - 次のバッチが存在し、処理が継続可能な場合は true、全バッチが完了した場合は false を返す。

前提条件と不変条件

  • hjstate->hj_HashTable が初期化されていること。
  • 呼び出し時点で hashtable->curbatch が現在のバッチ番号を保持していること。
  • 外部・内部リレーションのバッチファイルが hashtable 内に適切に管理されていること。

アルゴリズムまたは振る舞い

1. **クリーンアップ**: 現在のバッチ（curbatch）の外部ファイルが不要であればクローズする。最初のバッチ終了時はスキュー最適化状態をリセットする。 2. **バッチスキップ判定**: 両側のバッチが空であるか、結合の種類（外部結合等）やバッチ数変更のルールに基づき、処理不要なバッチをスキップしてファイルをクローズする。 3. **終了判定**: 全バッチを走査し終えた場合は false を返す。 4. **ハッシュテーブル再構築**: ExecHashTableReset でテーブルを初期化し、新しい内部バッチファイルを読み込んで ExecHashTableInsert でタプルを再挿入する。 5. **ファイル準備**: 外部バッチファイルが存在する場合は、読み込みのために先頭へシークする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHashjoin.c 内の結合実行ロジック（ExecHashJoin 等）から呼び出される。
  • **呼び出し先**: BufFileClose, BufFileSeek, ExecHashTableReset, ExecHashJoinGetSavedTuple, ExecHashTableInsert 等。

関連データ構造

  • HashJoinState: 結合ノードの実行状態。
  • HashJoinTable: ハッシュテーブルおよびバッチファイル管理構造体。
  • BufFile: ディスク上のバッチファイル操作用インターフェース。

失敗モードと境界条件

  • **ファイル操作エラー**: BufFileSeek 失敗時に ereport(ERROR, ...) を発行し、処理を中断する。
  • **バッチ数変更**: 実行中に nbatch が増加した場合、再スキャンが必要となるロジックが組み込まれている。

保守担当者向けメモ

  • 本関数はハイブリッド・ハッシュ結合のステートマシンの一部です。デバッグ時は hj_JoinState の遷移と curbatch のインクリメント状況を追跡してください。
  • スキュー最適化の無効化やバッチスキップのルールは、外部結合（Left/Full/Right）の正確性に直結するため、変更時は注意が必要です。
