gbt_numeric_same

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:135

目的

GiST インデックスにおいて、2つの numeric 型のキーが同一であるかを判定するサポート関数です。

シグネチャまたは形状

Datum gbt_numeric_same(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_DATUM(0): 比較対象の第1キー (d1)
  • PG_GETARG_DATUM(1): 比較対象の第2キー (d2)
  • PG_GETARG_POINTER(2): 結果を格納する bool 型へのポインタ (result)
  • **出力**: result ポインタを介して、両者が同一であれば true、そうでなければ false を返します。

前提条件と不変条件

  • tinfo（gbtree_vinfo 構造体）が numeric 型の比較ロジックを保持している必要があります。
  • fcinfo->flinfo を通じて関数呼び出し情報が正しく提供されている必要があります。

アルゴリズムまたは振る舞い

1. d1 と d2 を取得します。 2. btree_utils_var.c で定義された汎用関数 gbt_var_same を呼び出します。 3. gbt_var_same に tinfo および関数呼び出し情報を渡し、比較結果を result に格納します。 4. 最終的に result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構（直接の呼び出し元はソースコード上では明示されていません）。
  • **呼び出し先**: gbt_var_same（contrib/btree_gist/btree_utils_var.c 内の関数）。

関連データ構造

  • gbtree_vinfo (tinfo): numeric 型の比較関数群を保持する設定情報構造体。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • d1 または d2 が無効なデータである場合、gbt_var_same の内部実装に依存します。
  • 比較対象が numeric 型として不正な形式である場合、基盤となる numeric 比較ロジックでエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_var.c の汎用ロジックを呼び出すアダプターです。ロジックの変更が必要な場合は、まず gbt_var_same の仕様を確認してください。
