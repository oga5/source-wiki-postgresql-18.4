PartClauseInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/partitioning/partprune.c:63

目的

パーティションキーに対するクエリの比較条件（WHERE 句など）を保持し、パーティションプルーニングのステップ生成プロセスで利用するための構造体です。

シグネチャまたは形状

typedef struct PartClauseInfo
{
	int			keyno;			/* パーティションキー番号 (0 to partnatts - 1) */
	Oid			opno;			/* パーティションキーと式を比較する演算子 */
	bool		op_is_ne;		/* 演算子が <> かどうか */
	Expr	   *expr;			/* 比較対象の式 */
	Oid			cmpfn;			/* 式とパーティションキーを比較する関数 */
	int			op_strategy;	/* 演算子を識別するB-tree戦略 */
} PartClauseInfo;

入力と出力

本構造体は関数 match_clause_to_partition_key 等によって生成され、PartClauseInfo **pc として出力されます。また、gen_partprune_steps_internal 等の関数内で、条件式をプルーニングステップへ変換するための入力として参照されます。

前提条件と不変条件

  • keyno は有効なパーティションキーの範囲内（0 から partnatts - 1）である必要があります。
  • opno および cmpfn は、対象となるパーティションキーの型と expr の型の間で比較可能な有効な OID である必要があります。

アルゴリズムまたは振る舞い

クエリの RestrictInfo リストを走査する過程で、パーティションキーに関連する条件が抽出されると、その属性情報が PartClauseInfo に格納されます。この情報は、後のステップで PartitionPruneStepOp を構築する際の基礎データとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partprune.c 内の match_clause_to_partition_key や gen_partprune_steps_internal 等の関数が、条件解析時に本構造体をインスタンス化します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。この構造体のフィールド値は、プルーニングステップの構築ロジックによって読み取られます。

関連データ構造

  • PartitionPruneStepOp: 本構造体の情報を用いて生成される、比較演算を行うためのプルーニングステップ。
  • GeneratePruningStepsContext: ステップ生成中の状態を保持する構造体。

失敗モードと境界条件

  • expr が評価不能な場合や、パーティションキーと型が一致しない演算子である場合、PartClauseInfo の生成に失敗する（または NULL が返される）可能性があります。
  • op_is_ne が true の場合、不等価演算子特有のプルーニングロジックが適用されます。

保守担当者向けメモ

  • 本構造体は partprune.c 内でのみ使用される内部的なデータ保持用構造体です。
  • op_strategy は access/nbtree.h で定義される戦略定数と整合している必要があります。
  • 構造体のフィールド追加や変更を行う際は、gen_partprune_steps_internal における条件解析ロジックへの影響を確認してください。
