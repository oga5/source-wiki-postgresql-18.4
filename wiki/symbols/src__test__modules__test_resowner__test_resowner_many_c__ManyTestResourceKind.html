ManyTestResourceKind

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_resowner/test_resowner_many.c:28

目的

ResourceOwner 機構のストレステストにおいて、特定のリソース種別に関連する統計情報および現在保持しているリソースのリストを管理するために使用されます。

シグネチャまたは形状

typedef struct
{
	ResourceOwnerDesc desc;
	int			nremembered;
	int			nforgotten;
	int			nreleased;
	int			nleaked;

	dlist_head	current_resources;
} ManyTestResourceKind;

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは以下の通りです：

  • desc: リソースの記述子（ResourceOwnerDesc）。
  • nremembered: 登録されたリソースの総数。
  • nforgotten: 明示的に破棄されたリソースの数。
  • nreleased: ResourceOwnerRelease により解放されたリソースの数。
  • nleaked: 解放されずに残存したリソースの数。
  • current_resources: 現在アクティブなリソースを追跡するための侵入型リスト（dlist_head）。

前提条件と不変条件

  • current_resources は lib/ilist.h の dlist API を用いて管理される必要があります。
  • 各フィールドのカウンタは、リソースのライフサイクル（登録・破棄・解放）に応じて整合性を保つ必要があります。

アルゴリズムまたは振る舞い

本構造体自体は状態保持に特化しており、具体的な振る舞いは InitManyTestResourceKind や RememberManyTestResources などの操作関数によって定義されます。リソースのライフサイクル全体を通じて、ResourceOwner のコールバックと同期した統計情報の更新を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: test_resowner_many 関数内でインスタンス化され、InitManyTestResourceKind、RememberManyTestResources、ForgetManyTestResources、GetTotalResourceCount 等の関数に引数として渡されます。
  • **呼び出し先**: 本構造体自体はデータ構造であるため、直接の呼び出し先はありません。

関連データ構造

  • ResourceOwnerDesc: desc フィールドで使用。
  • dlist_head: current_resources フィールドで使用。
  • ManyTestResource: current_resources リスト内に格納される個別のリソースインスタンス。

失敗モードと境界条件

  • リソースの登録数と解放数の不一致が発生した場合、nleaked フィールドが正しくカウントされない可能性があります。
  • dlist の操作において、リストの整合性が失われた場合、current_resources の走査時にメモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はテスト専用のモジュールで使用されるため、本番環境のコードには影響しません。
  • nremembered 等のカウンタは、テストの成功判定（アサーション）において重要な役割を果たすため、変更時は関連するテストロジックの更新が必要です。
