DSMRegistryEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/dsm_registry.c:43

目的

名前付き動的共有メモリ（DSM）セグメントをレジストリ内で管理するためのメタデータ構造体です。

シグネチャまたは形状

typedef struct DSMRegistryEntry
{
	char		name[64];
	dsm_handle	handle;
	size_t		size;
} DSMRegistryEntry;

入力と出力

本構造体は、共有メモリ上のハッシュテーブルにおけるエントリとして機能します。

  • name: セグメントを識別する一意のキー（最大64バイト）。
  • handle: 関連付けられたDSMセグメントのハンドル。
  • size: セグメントのサイズ。

前提条件と不変条件

  • セグメント名は64バイト未満である必要があります。
  • 一度作成されたセグメントのサイズは変更できません。

アルゴリズムまたは振る舞い

本構造体は dshash（共有ハッシュテーブル）の要素として格納されます。GetNamedDSMSegment 関数において、指定された名前でハッシュテーブルを検索し、エントリが存在しない場合は新規作成、存在する場合は既存のハンドルを取得するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/dsm_registry.c 内の関数（GetNamedDSMSegment 等）が本構造体を操作します。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • DSMRegistryCtxStruct: レジストリ全体のメタデータを保持する構造体。
  • dshash: 本構造体を格納する共有ハッシュテーブルの実装。

失敗モードと境界条件

  • name が64バイト以上の場合、レジストリへの登録が制限される可能性があります。
  • 共有メモリ上のハッシュテーブルが満杯の場合、新しいエントリの追加に失敗する可能性があります。

保守担当者向けメモ

  • offsetof(DSMRegistryEntry, handle) や sizeof(DSMRegistryEntry) がコード内で参照されており、構造体のレイアウト変更時には注意が必要です。
  • 本構造体は共有メモリ上に配置されるため、アライメントやパディングに留意してください。
