run_testspec

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolationtester.c:393

目的

提供されたテスト仕様（TestSpec）に基づき、実行すべきテスト順列（Permutation）の実行戦略を決定し、テスト実行を開始する。

シグネチャまたは形状

static void run_testspec(TestSpec *testspec)

入力と出力

  • **入力**: TestSpec *testspec - 実行対象のテスト仕様構造体。
  • **出力**: なし（戻り値 void）。

前提条件と不変条件

  • 引数 testspec は有効なポインタであること。
  • testspec 内の permutations メンバが初期化されていること。

アルゴリズムまたは振る舞い

1. testspec->permutations が存在（非NULL）する場合、run_named_permutations(testspec) を呼び出し、仕様内で明示的に定義された順列を実行する。 2. testspec->permutations が存在しない場合、run_all_permutations(testspec) を呼び出し、自動生成された全順列を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/isolation/isolationtester.c 内のテスト実行フロー（main 関数等からの呼び出し）。
  • **呼び出し先**:
  • run_named_permutations
  • run_all_permutations

関連データ構造

  • TestSpec: テスト仕様全体を保持する構造体。

失敗モードと境界条件

  • testspec が NULL の場合、セグメンテーションフォールトが発生する可能性がある。
  • 順列が定義されていない場合、run_all_permutations が全順列を生成するが、その過程でメモリ不足や論理的な不整合が発生する可能性がある。

保守担当者向けメモ

  • 本関数はテスト実行のディスパッチャーとして機能する。テスト実行ロジックの追加や変更を行う際は、この分岐条件が適切に機能しているか確認すること。
