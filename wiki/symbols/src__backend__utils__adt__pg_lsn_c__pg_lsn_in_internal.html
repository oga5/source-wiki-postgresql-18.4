pg_lsn_in_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:28

目的

文字列形式（XX/XXXXXXXX）のログシーケンス番号（LSN）を解析し、内部表現である64ビット整数（XLogRecPtr）へ変換します。

シグネチャまたは形状

XLogRecPtr pg_lsn_in_internal(const char *str, bool *have_error);

入力と出力

  • **入力**:
  • str: 解析対象の文字列（XX/XXXXXXXX 形式）。
  • have_error: 解析の成否を格納するブール値へのポインタ。
  • **出力**:
  • 成功時: 解析された XLogRecPtr（uint64）。
  • 失敗時: InvalidXLogRecPtr を返し、have_error に true を設定。

前提条件と不変条件

  • have_error は NULL であってはなりません（Assert により検証）。
  • 入力文字列は XX/XXXXXXXX の形式に従う必要があり、各コンポーネントは16進数で構成されます。
  • 各コンポーネントの長さは MAXPG_LSNCOMPONENT 以下である必要があります。

アルゴリズムまたは振る舞い

1. have_error を false に初期化します。 2. strspn を使用して、/ を境界として前半と後半の16進数文字列の長さを検証します。 3. 形式が不正な場合（長さが0、上限超過、または区切り文字が / でない場合）、have_error を true にして InvalidXLogRecPtr を返します。 4. strtoul を使用して前半（id）と後半（off）を16進数としてデコードします。 5. id を32ビット左シフトし、off とビット論理和をとることで64ビットの XLogRecPtr を構築して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c:4871
  • src/backend/utils/adt/pg_lsn.c:69（pg_lsn_in 関数内）
  • **呼び出し先**:
  • 標準Cライブラリ関数: strspn, strtoul

関連データ構造

  • XLogRecPtr: LSNを保持する64ビット整数型。
  • InvalidXLogRecPtr: 無効なLSNを示す定数。

失敗モードと境界条件

  • 入力文字列が XX/XXXXXXXX 形式に適合しない場合、エラーとして扱われます。
  • 各コンポーネントが MAXPG_LSNCOMPONENT を超える場合、解析は失敗します。
  • strtoul の結果がオーバーフローする場合の挙動は、標準ライブラリの仕様に依存します。

保守担当者向けメモ

  • 本関数は pg_lsn 型の入力処理のコアロジックです。
  • strtoul を使用しているため、入力文字列の妥当性は strspn による事前のチェックに強く依存しています。
  • 算術演算が必要な場合は、numeric 型を介する他の関数群（pg_lsn_mi 等）が利用される設計となっています。
