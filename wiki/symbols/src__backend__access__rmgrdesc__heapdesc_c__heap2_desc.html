heap2_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/heapdesc.c:263

目的

ヒープの補助操作（XLOG_HEAP2_*）に関連するWALレコードを解析し、人間が読める形式の文字列に変換して StringInfo バッファへ出力する。主に pg_waldump 等のツールで利用される。

シグネチャまたは形状

void heap2_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 解析結果を格納する StringInfo オブジェクト。
  • record: 解析対象のWALレコード状態を保持する XLogReaderState へのポインタ。
  • **出力**: なし（buf を通じて文字列が書き込まれる）。

前提条件と不変条件

  • record は有効なWALレコードを指している必要がある。
  • XLogRecGetInfo を用いて取得したレコード情報から、ヒープ操作タイプを正しく識別できること。

アルゴリズムまたは振る舞い

1. XLogRecGetData でレコードのデータ部を取得し、XLogRecGetInfo で操作タイプを特定する。 2. 操作タイプ（info）に基づき、以下のいずれかの処理を実行する：

  • **PRUNE系操作 (PRUNE_ON_ACCESS, PRUNE_VACUUM_SCAN, PRUNE_VACUUM_CLEANUP)**:
  • xl_heap_prune 構造体を読み取り、必要に応じて conflict_xid やカタログフラグを出力。
  • ブロックデータが存在する場合、heap_xlog_deserialize_prune_and_freeze を呼び出して詳細なオフセット配列や凍結計画を抽出。
  • array_desc を用いて、計画、リダイレクト、デッド、未使用の各オフセット情報を整形して出力。
  • **VISIBLE**: xl_heap_visible 構造体からスナップショットの競合ホライゾンとフラグを出力。
  • **MULTI_INSERT**: xl_heap_multi_insert 構造体からタプル数とフラグを出力。ブロックデータがある場合はオフセット配列を出力。
  • **LOCK_UPDATED**: xl_heap_lock_updated 構造体からXMAX、オフセット、情報ビット、フラグを出力。
  • **NEW_CID**: xl_heap_new_cid 構造体からリレーション情報、TID、コマンドID（cmin/cmax/combo）を出力。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されたリソースマネージャリストを通じて、WAL解析ツールから呼び出される。
  • **呼び出し先**:
  • heap_xlog_deserialize_prune_and_freeze
  • array_desc
  • infobits_desc
  • XLogRecGetData, XLogRecGetInfo, XLogRecHasBlockData, XLogRecGetBlockData (WAL解析用API)

関連データ構造

  • xl_heap_prune, xl_heap_visible, xl_heap_multi_insert, xl_heap_lock_updated, xl_heap_new_cid
  • xlhp_freeze_plan
  • StringInfo, XLogReaderState

失敗モードと境界条件

  • レコードタイプが XLOG_HEAP2_* のいずれにも該当しない場合、何も出力されない。
  • ブロックデータが存在しない場合、詳細なオフセット解析はスキップされる。

保守担当者向けメモ

  • 新しい XLOG_HEAP2_* タイプを追加した場合は、本関数の if-else 分岐に処理を追加すること。
  • heap_xlog_deserialize_prune_and_freeze はRedoロジックと共通化されているため、解析ロジックを変更する際は影響範囲に注意すること。
