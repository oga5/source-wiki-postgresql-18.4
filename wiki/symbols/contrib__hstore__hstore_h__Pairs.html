Pairs

定義

  • 種別: struct
  • 定義箇所: contrib/hstore/hstore.h:161

目的

hstore 型のキーと値のペアをメモリ上で扱うための「展開済み（decompressed）」表現を提供します。

シグネチャまたは形状

typedef struct
{
	char	   *key;
	char	   *val;
	size_t		keylen;
	size_t		vallen;
	bool		isnull;
	bool		needfree;
} Pairs;

入力と出力

  • 本構造体は、hstore の構築や変換処理において、キーと値のペアを保持する中間データとして使用されます。
  • key および val は文字列データへのポインタ、keylen および vallen はその長さを保持します。

前提条件と不変条件

  • isnull が true の場合、値は NULL として扱われます。
  • needfree は、val が指すメモリ領域を pfree する必要があるかどうかを示します。

アルゴリズムまたは振る舞い

  • hstore 構築時の中間表現として機能します。
  • hstoreUniquePairs 関数により、Pairs 配列内の重複排除が行われます。
  • hstorePairs 関数により、Pairs 配列から HStore 構造体への変換が行われます。
  • hstoreArrayToPairs 関数により、ArrayType から Pairs への変換が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/hstore/hstore_io.c において、palloc を用いて Pairs の配列が動的に確保され、データ処理に使用されています。
  • 呼び出し先: hstoreUniquePairs, hstorePairs, hstoreArrayToPairs などの関数が本構造体を引数として受け取ります。

関連データ構造

  • HStore: hstore 型のメイン構造体。
  • HEntry: hstore 内の各キー・値のメタデータを保持する構造体。

失敗モードと境界条件

  • Pairs 配列の長さは、MaxAllocSize および INT_MAX/40 の制約を受けます。

保守担当者向けメモ

  • 本構造体は hstore の物理レイアウト（HStore 構造体）とは異なり、メモリ上で操作しやすい形式です。
  • hstore_io.c 内で repalloc を用いて配列サイズが拡張されるケースがあるため、ポインタの取り扱いに注意してください。
