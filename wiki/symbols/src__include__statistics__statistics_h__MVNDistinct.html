MVNDistinct

定義

  • 種別: struct
  • 定義箇所: src/include/statistics/statistics.h:34

目的

PostgreSQLの拡張統計情報において、複数列の組み合わせに対する「NDistinct（個体数）」統計をメモリ内で保持するための構造体です。

シグネチャまたは形状

typedef struct MVNDistinct
{
    uint32      magic;          /* マジック定数マーカー */
    uint32      type;           /* NDistinctの型 (BASIC) */
    uint32      nitems;         /* 統計に含まれるアイテム数 */
    MVNDistinctItem items[FLEXIBLE_ARRAY_MEMBER]; /* 可変長配列 */
} MVNDistinct;

入力と出力

  • **入力**: カタログからロードされた bytea 形式のデータが、デシリアライズ処理を経て本構造体に格納されます。
  • **出力**: プランナがクエリの選択率を推定する際、本構造体のデータが参照されます。

前提条件と不変条件

  • FLEXIBLE_ARRAY_MEMBER を使用した可変長構造体であるため、メモリ確保時には offsetof(MVNDistinct, items) を用いたサイズ計算が必須です。
  • magic フィールドは、シリアライズされたデータの整合性チェックに利用されます。

アルゴリズムまたは振る舞い

  • ANALYZE 処理時に構築され、統計情報としてカタログに永続化されます。
  • プランナは statext_ndistinct_serialize 等の関数を通じて、本構造体のシリアライズやデシリアライズを行い、クエリ最適化に利用します。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/backend/statistics/extended_stats.c (プランナによる統計利用)
  • src/backend/statistics/mvdistinct.c (シリアライズ・デシリアライズ処理)
  • **呼び出し先**:
  • 本構造体は MVNDistinctItem を要素として保持します。

関連データ構造

  • MVNDistinctItem: MVNDistinct が保持する個々の統計アイテムの型。

失敗モードと境界条件

  • STATS_MAX_DIMENSIONS（デフォルト8）を超える列数を持つ統計情報は扱えません。
  • メモリ確保時に構造体のサイズ計算を誤ると、バッファオーバーランや不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 新しい統計型を追加する場合は、magic フィールドに一意な値を定義してください。
  • 可変長構造体であるため、ポインタ操作やメモリ確保時には FLEXIBLE_ARRAY_MEMBER の特性を考慮した実装を行ってください。
