check_amoptsproc_signature

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/amvalidate.c:191

目的

アクセスメソッドのオプション（amoptions）を処理するサポート関数のシグネチャが、期待される要件を満たしているかを検証します。

シグネチャまたは形状

bool check_amoptsproc_signature(Oid funcid)

入力と出力

  • **入力**: funcid (Oid) - 検証対象の関数のOID。
  • **出力**: bool - シグネチャが要件に適合していれば true、そうでなければ false。

前提条件と不変条件

  • check_amproc_signature を内部的に呼び出し、以下の制約を強制します。
  • 戻り値の型: VOIDOID
  • 引数の数: 1個
  • 引数の型: INTERNALOID

アルゴリズムまたは振る舞い

1. check_amproc_signature を呼び出し、指定された funcid の関数定義をシステムカタログから取得します。 2. 戻り値の型が void であることを確認します。 3. 引数が1つであり、その型が internal であることを確認します。 4. 検証結果を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blvalidate.c
  • src/backend/access/brin/brin_validate.c
  • src/backend/access/gin/ginvalidate.c
  • src/backend/access/gist/gistvalidate.c
  • src/backend/access/hash/hashvalidate.c
  • src/backend/access/nbtree/nbtvalidate.c
  • src/backend/access/spgist/spgvalidate.c
  • **呼び出し先**:
  • check_amproc_signature (同ファイル内)

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • funcid がシステムカタログに存在しない場合、check_amproc_signature 経由でエラーが発生する可能性があります。
  • 引数の型や数が期待値と異なる場合、check_amproc_signature は false を返します。

保守担当者向けメモ

  • 本関数は、各アクセスメソッドのバリデーション処理において、オプション解析用関数の整合性を担保するために共通利用されています。シグネチャの要件変更が必要な場合は、check_amproc_signature への引数を確認してください。
