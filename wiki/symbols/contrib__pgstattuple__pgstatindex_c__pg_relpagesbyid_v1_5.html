pg_relpagesbyid_v1_5

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstatindex.c:441

目的

指定されたリレーションID（OID）に基づき、対象リレーションの総ページ数を取得して返すSQL関数です。

シグネチャまたは形状

Datum pg_relpagesbyid_v1_5(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してリレーションのOIDを受け取ります。
  • **出力**: int64 型として、リレーションの総ページ数を返します。

前提条件と不変条件

  • 対象リレーションに対して AccessShareLock を取得します。
  • 呼び出し元はPostgreSQLの関数呼び出し規約（PG_FUNCTION_ARGS）に従う必要があります。

アルゴリズムまたは振る舞い

1. relation_open を呼び出し、指定されたOIDのリレーションを AccessShareLock モードで開きます。 2. pg_relpages_impl 関数を呼び出し、開いたリレーションのページ数を計算します。 3. 取得したページ数を PG_RETURN_INT64 マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されることを想定しています（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**:
  • relation_open (PostgreSQLコア関数)
  • pg_relpages_impl (同ファイル内の実装関数)

関連データ構造

  • Relation: リレーション情報を保持する構造体。

失敗モードと境界条件

  • 指定されたOIDが存在しない場合、relation_open によりエラーが発生します。
  • 適切な権限がない場合、ロック取得時に失敗する可能性があります。

保守担当者向けメモ

  • 本関数は pg_relpagesbyid のバージョン1.5であり、セキュリティ強化のため REVOKE EXECUTE ON PUBLIC を前提としています。
  • 内部ロジックは pg_relpages_impl に集約されているため、統計計算ロジックの変更はそちらを確認してください。
