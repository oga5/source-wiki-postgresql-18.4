GucStack

定義

  • 種別: struct
  • 定義箇所: src/include/utils/guc_tables.h:120

目的

SET LOCAL コマンド等による一時的な設定変更が発生した際、変更前の値を保持し、トランザクション終了時やスコープ終了時に元の状態へ復元するためのスタック構造を提供します。

シグネチャまたは形状

typedef struct guc_stack
{
    struct guc_stack *prev;
    int              nest_level;
    GucStackState    state;
    GucSource        source;
    GucContext       scontext;
    GucContext       masked_scontext;
    Oid              srole;
    Oid              masked_srole;
    config_var_value prior;
    config_var_value masked;
} GucStack;

入力と出力

  • **入力**: 設定変更時に MemoryContextAllocZero を介してメモリが割り当てられ、現在の設定値（prior）やコンテキスト情報が格納されます。
  • **出力**: スタックの先頭（prev）を辿ることで、過去の階層的な設定状態を復元可能です。

前提条件と不変条件

  • masked 値のソースは PGC_S_SESSION であることが前提とされており、ソース情報の保持は省略されています。
  • prev ポインタにより、単方向連結リストとして構成されます。

アルゴリズムまたは振る舞い

1. **プッシュ**: 設定値が変更される際、現在の値を prior に保存し、新しいスタック要素を生成して既存のスタックに連結します。 2. **管理**: nest_level により、どのネスト深さで変更が行われたかを追跡します。 3. **ポップ**: 変更が取り消される際、prev を参照してスタックを巻き戻し、prior に保存されていた値を現在の設定値として復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc.c 内の複数の関数（行248, 457, 710, 749, 2136, 2280, 3770, 3868, 3966）で、設定変数の変更管理のために参照されています。
  • **呼び出し先**: MemoryContextAllocZero を使用してメモリを確保する処理が確認されています。

関連データ構造

  • config_var_value: 設定値を保持する共用体。
  • GucStackState: スタックの状態（enum）を定義。
  • GucSource, GucContext: 設定の適用元やコンテキストを定義する列挙型。

失敗モードと境界条件

  • メモリ不足時に MemoryContextAllocZero が失敗する可能性があります。
  • スタックが空の状態でポップ操作を行った場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • GucStack は config_generic 構造体の一部として管理されるスタック情報のノードです。
  • masked フィールドは GUC_SET_LOCAL エントリにおいて、一時的に隠蔽される値を保持するために使用されます。
  • 構造体のメモリ管理は TopTransactionContext 等の適切なメモリコンテキストで行われているか確認してください。
