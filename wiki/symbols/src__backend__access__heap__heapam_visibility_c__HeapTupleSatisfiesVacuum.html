HeapTupleSatisfiesVacuum

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_visibility.c:1170

目的

VACUUM処理において、特定のヒープタプルが物理削除可能かどうかを判定します。

シグネチャまたは形状

HTSV_Result HeapTupleSatisfiesVacuum(HeapTuple htup, TransactionId OldestXmin, Buffer buffer)

入力と出力

  • **入力**:
  • htup: 判定対象のヒープタプル。
  • OldestXmin: 現在実行中のトランザクションの中で最も古いXmin（これより古いトランザクションは存在しない）。
  • buffer: タプルが格納されているバッファ。
  • **出力**:
  • HTSV_Result: 可視性判定結果（HEAPTUPLE_DEAD, HEAPTUPLE_RECENTLY_DEAD 等）。

前提条件と不変条件

  • 呼び出し元は、対象のバッファに対して少なくとも共有ロック（Shared Lock）を保持している必要があります。
  • HEAPTUPLE_RECENTLY_DEAD と判定された場合、内部で算出された dead_after トランザクションIDが有効であることが保証されます。

アルゴリズムまたは振る舞い

1. HeapTupleSatisfiesVacuumHorizon を呼び出し、タプルの基本的な可視性状態と、そのタプルが削除可能になる境界となるトランザクションID（dead_after）を取得します。 2. 結果が HEAPTUPLE_RECENTLY_DEAD の場合、dead_after が OldestXmin よりも古い（TransactionIdPrecedes）かを確認します。 3. dead_after が OldestXmin より古い場合、そのタプルはどのトランザクションからも参照されないと見なし、結果を HEAPTUPLE_DEAD に昇格させます。 4. それ以外の場合は、HeapTupleSatisfiesVacuumHorizon の結果をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_visibility/pg_visibility.c
  • contrib/pgstattuple/pgstatapprox.c
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_handler.c
  • **呼び出し先**:
  • HeapTupleSatisfiesVacuumHorizon

関連データ構造

  • HeapTuple: 判定対象のタプル構造体。
  • HTSV_Result: 可視性判定結果を示す列挙型。
  • TransactionId: トランザクション識別子。

失敗モードと境界条件

  • HeapTupleSatisfiesVacuumHorizon が HEAPTUPLE_RECENTLY_DEAD 以外を返した場合、dead_after が InvalidTransactionId であることをアサートで検証します。
  • 呼び出し元で認識できない戻り値が返された場合、エラー（elog(ERROR, ...)）が発生する実装が各所に存在します。

保守担当者向けメモ

  • 本関数は、HeapTupleSatisfiesVacuumHorizon の結果を基に、VACUUMの基準となる OldestXmin との比較を行うラッパーとしての役割を果たします。
  • 可視性判定のロジックを変更する際は、heapam_visibility.c 内の他の判定関数との整合性に注意してください。
