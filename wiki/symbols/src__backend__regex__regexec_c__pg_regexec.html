pg_regexec

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexec.c:184

目的

コンパイル済みの正規表現（NFA）を対象文字列に適用し、マッチングの成否およびサブマッチの境界を特定する正規表現実行エンジンのメインエントリポイント。

シグネチャまたは形状

int pg_regexec(regex_t *re, const chr *string, size_t len, size_t search_start, rm_detail_t *details, size_t nmatch, regmatch_t pmatch[], int flags);

入力と出力

  • **入力**:
  • re: コンパイル済み正規表現構造体。
  • string, len: 対象文字列とその長さ。
  • search_start: 検索開始位置のオフセット。
  • details: 追加の詳細情報（REG_EXPECT 使用時に必須）。
  • nmatch, pmatch: 結果を格納する配列とそのサイズ。
  • flags: 実行フラグ（eflags）。
  • **出力**: 成功時は REG_OKAY (0)、失敗時はエラーコード（REG_NOMATCH, REG_INVARG, REG_ESPACE 等）を返す。

前提条件と不変条件

  • re が NULL でなく、re->re_magic が REMAGIC であること。
  • re->re_csize が sizeof(chr) と一致すること。
  • search_start が len 以下であること。
  • REG_EXPECT フラグが指定されている場合、details は NULL であってはならない。

アルゴリズムまたは振る舞い

1. **初期化**: ロケール依存のサポートをセットアップし、vars 構造体（マッチング状態）を初期化する。 2. **メモリ確保**: バックリファレンスの有無に応じて、サブマッチ結果を保持するための作業領域（pmatch）をスタックまたはヒープ上に確保する。 3. **DFA 準備**: 正規表現のツリー構造に基づき、必要な DFA 状態管理用のメモリを確保・初期化する。 4. **探索実行**:

  • バックリファレンスを含む場合は cfind を呼び出す。
  • それ以外の場合は find を呼び出し、DFA ベースの高速なマッチングを行う。

    • 5. **結果格納**: マッチ成功時、必要に応じて作業領域から呼び出し元の pmatch 配列へ結果をコピーする。 6. **クリーンアップ**: 確保した DFA 領域やメモリを解放し、終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c, src/backend/tsearch/spell.c, src/backend/utils/adt/regexp.c, src/backend/utils/adt/varlena.c, src/test/modules/test_regex/test_regex.c など。
  • **呼び出し先**: find, cfind, zapallsubs, freedfa, pg_set_regex_collation 等。

関連データ構造

  • vars: マッチング中のコンテキスト（文字列ポインタ、DFA キャッシュ、エラーコード等）を保持。
  • dfa: 遅延評価 DFA の状態管理構造体。
  • regmatch_t: マッチしたサブ式の開始・終了位置を格納する構造体。

失敗モードと境界条件

  • re->re_csize 不一致: REG_MIXED を返す。
  • search_start > len: REG_NOMATCH を返す。
  • メモリ不足: REG_ESPACE を返す。
  • 無効な引数: REG_INVARG を返す。
  • マッチ失敗: REG_NOMATCH を返す。

保守担当者向けメモ

  • 本関数は struct vars を通じてマッチング状態を管理する。
  • subdfas や ladfas などの DFA 関連メモリは、cleanup ラベル以降で確実に解放される設計となっている。
  • バックリファレンスがない場合は find が呼ばれ、ある場合は cfind が呼ばれるため、性能特性が大きく異なる点に注意すること。
