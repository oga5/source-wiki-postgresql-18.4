ActiveSnapshotElt

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/time/snapmgr.c:172

目的

アクティブなスナップショットを管理するスタック構造の各要素を表現します。ActiveSnapshot スタックにおいて、現在有効なスナップショットとそのネストレベルを保持するために使用されます。

シグネチャまたは形状

typedef struct ActiveSnapshotElt
{
	Snapshot	as_snap;
	int			as_level;
	struct ActiveSnapshotElt *as_next;
} ActiveSnapshotElt;

入力と出力

  • as_snap: 保持するスナップショットへのポインタ。
  • as_level: スナップショットがプッシュされた際のトランザクションネストレベル。
  • as_next: スタック内の次の要素へのポインタ。

前提条件と不変条件

  • TopTransactionContext 上に割り当てられることが想定されています。
  • スタックの先頭は ActiveSnapshot 静的変数によって管理されます。

アルゴリズムまたは振る舞い

  • PushActiveSnapshot 時に新しい要素が生成され、スタックの先頭に追加されます。
  • PopActiveSnapshot 時にスタックから取り出され、メモリが解放されます。
  • スタック構造により、クエリ実行コンテキストにおけるスナップショットの階層的な管理を実現します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/time/snapmgr.c 内の PushActiveSnapshot 等のスタック操作関数。
  • 呼び出し先: 情報なし。

関連データ構造

  • SnapshotData: as_snap が指す実体。
  • ActiveSnapshot: 本構造体のリストを管理するスタックのヘッドポインタ。

失敗モードと境界条件

  • メモリ不足時: MemoryContextAlloc が失敗した場合、システムはエラーを発生させます。
  • スタックアンダーフロー: PopActiveSnapshot が空のスタックに対して呼ばれた場合の挙動は、snapmgr.c の実装に依存します。

保守担当者向けメモ

  • 本構造体は TopTransactionContext に依存しているため、トランザクション終了時に適切にクリーンアップされない場合、メモリリークの原因となります。
  • as_level はサブトランザクションの境界を判定するために使用されます。
