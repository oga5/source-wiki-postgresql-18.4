FileSetOpen

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fileset.c:118

目的

FileSet 内の指定された名前の一時ファイルをオープンし、ファイル記述子（File）を返します。

シグネチャまたは形状

File FileSetOpen(FileSet *fileset, const char *name, int mode)

入力と出力

  • **入力**:
  • fileset: 対象となる FileSet 構造体へのポインタ。
  • name: オープンするファイルの名前。
  • mode: ファイルオープン時のモード（fd.c の PathNameOpenTemporaryFile に準拠）。
  • **出力**:
  • File: オープンされたファイルの記述子。

前提条件と不変条件

  • fileset は FileSetInit によって初期化済みである必要があります。
  • 指定された name のファイルが、FileSet が管理するディレクトリ内に存在している必要があります。

アルゴリズムまたは振る舞い

1. FilePath を呼び出し、fileset の情報と name を組み合わせて、対象ファイルの物理パスを path バッファに生成します。 2. 生成されたパスを引数として PathNameOpenTemporaryFile を呼び出し、ファイルをオープンします。 3. 取得したファイル記述子を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/buffile.c:316
  • **呼び出し先**:
  • FilePath (同ファイル内)
  • PathNameOpenTemporaryFile (src/backend/storage/file/fd.c で定義)

関連データ構造

  • FileSet: 一時ファイルの集合を管理する構造体。
  • File: fd.c で管理されるファイル記述子型。

失敗モードと境界条件

  • PathNameOpenTemporaryFile が失敗した場合（ファイルが存在しない、権限不足、ディスクフルなど）、その挙動に従いエラーまたは無効な記述子が返されます。

保守担当者向けメモ

  • 本関数は FileSet 内のパス解決と低レベルなファイルオープンを仲介するラッパーです。
  • ファイルの存在確認やオープン失敗時のエラーハンドリングは PathNameOpenTemporaryFile の仕様に依存します。
