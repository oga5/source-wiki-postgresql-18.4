eclass

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_locale.c:499

目的

正規表現における等価クラス（equivalence class）に対応する文字集合（cvec）を生成します。

シグネチャまたは形状

struct cvec *eclass(struct vars *v, chr c, int cases)

入力と出力

  • **入力**:
  • v: コンテキストを保持する vars 構造体。
  • c: 等価クラスを表す照合要素（chr 型）。
  • cases: 大文字小文字を区別しないマッチングを行うかを示すフラグ（非ゼロで有効）。
  • **出力**:
  • 生成された文字集合を保持する struct cvec *。

前提条件と不変条件

  • cases が偽の場合、戻り値の cvec は少なくとも1つの文字（c）を保持します。
  • getcvec が返す cvec は NULL でないことが保証されます（assert による検証）。

アルゴリズムまたは振る舞い

1. **テスト用偽装**: v->cflags に REG_FAKE が設定されており、かつ c == 'x' の場合、'x', 'y' を含む cvec を生成します。cases が真であれば 'X', 'Y' も追加します。 2. **通常処理**:

  • cases が真の場合、allcases(v, c) を呼び出し、指定された文字の全ケース（大文字・小文字等）を含む集合を返します。
  • cases が偽の場合、c のみを含む cvec を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は、提供された情報からは特定できません。
  • **呼び出し先**:
  • getcvec: cvec 構造体のメモリ確保に使用。
  • addchr: cvec への文字追加に使用。
  • allcases: cases が真の場合に呼び出される。
  • **注意**: ツールが提示した EquivalenceClass 関連の呼び出し元（src/backend/optimizer/... 等）は、PostgreSQLのクエリ最適化器における概念であり、本関数（正規表現エンジン）とは別のものです。

関連データ構造

  • struct vars: 正規表現コンパイルのコンテキスト。
  • struct cvec: 文字集合を保持する構造体。
  • chr: 文字コードを表す型。

失敗モードと境界条件

  • getcvec がメモリ不足等で失敗した場合、assert によりプログラムが停止する可能性があります。

保守担当者向けメモ

  • 本関数は正規表現エンジン内部の処理であり、クエリ最適化器の EquivalenceClass とは無関係です。
  • REG_FAKE による分岐はテスト目的の特殊な挙動であるため、本番環境のロケール処理には影響しません。
