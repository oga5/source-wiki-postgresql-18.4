inetKEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_inet.c:12

目的

PostgreSQL の inet 型を GiST インデックス内で表現するための境界値構造体です。inet 型の値を double 型のスカラ値に変換し、インデックス内での範囲比較を可能にします。

シグネチャまたは形状

typedef struct inetkey
{
	double		lower;
	double		upper;
} inetKEY;

入力と出力

  • 入力: inet 型の値（gbt_inet_compress 等を通じて変換される）。
  • 出力: double 型の lower（下限）および upper（上限）値。

前提条件と不変条件

  • リーフノードにおいては、lower == upper となるよう設計されています。
  • inet 型を double に変換して扱うため、非常に大きなネットワーク範囲や複雑なサブネットマスクでは精度に限界があります。

アルゴリズムまたは振る舞い

  • gbt_inet_compress 関数により、inet 型の値が convert_network_to_scalar を経由して inetKEY に変換され、インデックスに格納されます。
  • インデックス検索時には、クエリ値も同様に double に変換され、gbt_num_consistent を介して inetKEY の範囲と比較されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/btree_gist/btree_inet.c 内の各サポート関数（gbt_inet_compress, gbt_inet_consistent, gbt_inet_union, gbt_inet_penalty, gbt_inet_picksplit 等）が本構造体をキャストして利用しています。
  • 呼び出し先: 直接的な関数呼び出しはありませんが、btree_utils_num.c 内の汎用数値処理関数へデータとして渡されます。

関連データ構造

  • gbtree_ninfo: inet 型の比較ロジックを定義する構造体。
  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • GiST インデックスによる検索結果は「不正確」である可能性があるため、エンジン側での再チェック（recheck）が必須となります。

保守担当者向けメモ

  • 本構造体は btree_gist モジュールにおける inet 型専用の変換インターフェースの一部です。変更時は btree_utils_num.c との整合性に注意してください。
