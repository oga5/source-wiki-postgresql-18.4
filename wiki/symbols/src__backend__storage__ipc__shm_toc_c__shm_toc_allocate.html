shm_toc_allocate

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_toc.c:87

目的

共有メモリセグメント内の「目次（TOC）」管理領域において、指定されたサイズのメモリ領域を動的に割り当てます。

シグネチャまたは形状

void *shm_toc_allocate(shm_toc *toc, Size nbytes)

入力と出力

  • **入力**:
  • toc: 割り当て対象の shm_toc 構造体へのポインタ。
  • nbytes: 割り当てるバイト数。内部で BUFFERALIGN によりアライメント調整されます。
  • **出力**:
  • 割り当てられたメモリ領域の先頭アドレス（void *）。

前提条件と不変条件

  • toc は有効な shm_toc 構造体を指している必要があります。
  • 割り当ては、セグメントの末尾から前方へ向かって行われます。これにより、セグメント先頭から成長する toc_entry 配列との衝突を回避します。
  • 内部状態の更新には toc->toc_mutex による排他制御が必須です。

アルゴリズムまたは振る舞い

1. nbytes を BUFFERALIGN でアライメント調整します。 2. toc_mutex を取得し、現在の TOC 状態（合計サイズ、割り当て済みサイズ、エントリ数）を読み取ります。 3. 現在の TOC 構造体サイズと割り当て済みサイズの合計に nbytes を加えた値が、セグメントの総容量を超えていないか確認します。 4. 容量不足の場合はエラー（ERRCODE_OUT_OF_MEMORY）を報告し、ロックを解放して終了します。 5. 容量に余裕がある場合、toc_allocated_bytes を更新し、ロックを解放します。 6. セグメントの末尾から割り当て分を差し引いたオフセット位置を計算し、そのアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/common/session.c
  • src/backend/access/gin/gininsert.c
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease（排他制御）
  • ereport（エラー報告）

関連データ構造

  • shm_toc: 共有メモリセグメントの管理ヘッダー。
  • shm_toc_entry: TOC 内で管理される各領域のメタデータ。

失敗モードと境界条件

  • **メモリ不足**: 割り当て要求がセグメントの残存容量を超える場合、ereport によりエラーが発生します。
  • **アライメント**: BUFFERALIGN を使用してアライメントを保証していますが、アトミック操作に必要な最小アライメント要件については、BUFFERALIGN が十分であるという前提に基づいています。

保守担当者向けメモ

  • 本関数はロックフリー検索（shm_toc_lookup）と共存するため、割り当て時の整合性確保にはスピンロックが不可欠です。
  • 割り当て方向が「末尾から前方」であるため、toc_entry 配列の拡張と衝突しないよう、常に toc_bytes の計算結果を検証する必要があります。
