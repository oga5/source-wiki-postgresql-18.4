iso_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:341

目的

ISO-8859-5 エンコーディングの文字列を PostgreSQL 内部のマルチバイト表現（MULE_INTERNAL）へ変換する。

シグネチャまたは形状

Datum iso_to_mic(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して以下の引数を受け取る。
  • src (CSTRING): 変換元文字列。
  • dest (CSTRING): 変換先バッファ。
  • len (INT32): 変換対象の長さ。
  • noError (BOOL): エラーハンドリングフラグ。
  • **出力**: Datum 型として変換後のバイト数を返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_ISO_8859_5, PG_MULE_INTERNAL) により、ソースエンコーディングが PG_ISO_8859_5 であり、宛先が PG_MULE_INTERNAL であることが保証される。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. latin2mic_with_table 関数を呼び出し、変換テーブル iso2koi を用いて ISO-8859-5 から MULE_INTERNAL への変換を実行する。 3. 変換されたバイト数を PG_RETURN_INT32 で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1(iso_to_mic) により登録）。
  • **呼び出し先**: latin2mic_with_table（変換処理の実行）。

関連データ構造

  • iso2koi: ISO-8859-5 から KOI8-R へのマッピングを定義する static const unsigned char 配列。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断される。
  • noError フラグの状態に応じて、変換中の不正な文字に対する挙動が latin2mic_with_table 内で制御される。

保守担当者向けメモ

  • 本関数は latin2mic_with_table を利用しており、KOI8-R が MULE_INTERNAL のキリル文字表現と同一であることを前提としている。マッピングテーブル iso2koi を変更する際は、この前提との整合性に注意すること。
