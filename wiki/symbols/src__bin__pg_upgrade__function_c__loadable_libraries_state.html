loadable_libraries_state

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/function.c:48

目的

旧クラスタから収集した共有ライブラリ情報のクエリ結果を一時的に保持し、ライブラリの検証プロセスを管理するための状態構造体です。

シグネチャまたは形状

struct loadable_libraries_state
{
	PGresult  **ress;			/* 各データベースのクエリ結果 */
	int			totaltups;		/* 全結果に含まれるタプル総数 */
};

入力と出力

  • **入力**: get_loadable_libraries 関数において、各データベースから取得された PGresult ポインタの配列が ress に格納されます。
  • **出力**: 収集されたライブラリ情報は、後続の検証フェーズで totaltups を用いて反復処理されます。

前提条件と不変条件

  • ress は、対象となるデータベース数に応じた PGresult ポインタの配列を指す必要があります。
  • totaltups は、保持しているすべての PGresult 内のタプル数の合計と一致している必要があります。

アルゴリズムまたは振る舞い

1. get_loadable_libraries 内で初期化され、各データベースに対するクエリ結果を保持します。 2. 収集フェーズ終了後、保持されたデータはライブラリ名の抽出および検証に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/function.c:62 (キャストによる参照)
  • src/bin/pg_upgrade/function.c:85 (ローカル変数としての宣言)
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PGresult: libpq のクエリ結果構造体。ress メンバを通じて参照されます。

失敗モードと境界条件

  • メモリ割り当て失敗: ress 配列の確保に失敗した場合、pg_upgrade は致命的なエラーとして終了します。
  • クエリ結果の欠如: データベースからライブラリ情報が取得できない場合、ress が空または totaltups が 0 となる可能性があります。

保守担当者向けメモ

  • 本構造体は get_loadable_libraries の実行期間中のみ有効な一時的なコンテナです。
  • ress に格納された PGresult は、使用後に適切に解放される必要があります。
