ExtensionSiblingCache

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/extension.c:135

目的

拡張機能に関連する型（Type）の検索結果をキャッシュし、同一拡張機能内のオブジェクト参照を効率化するためのデータ構造です。

シグネチャまたは形状

typedef struct ExtensionSiblingCache
{
    struct ExtensionSiblingCache *next; /* リストのリンク */
    Oid         reqfuncoid;             /* 要求元の関数OID */
    const char *typname;                /* 型名 */
    bool        valid;                  /* エントリの有効性 */
    uint32      exthash;                /* 所有拡張機能OIDのハッシュ値 */
    Oid         typeoid;                /* 型名に関連付けられたOID */
} ExtensionSiblingCache;

入力と出力

  • 入力: 検索キーとして reqfuncoid（要求元関数）および typname（型名）を使用します。
  • 出力: キャッシュヒット時に typeoid を提供します。

前提条件と不変条件

  • ext_sibling_list を先頭とする連結リストとして管理されます。
  • valid フラグにより、キャッシュエントリが現在有効であるかどうかが制御されます。

アルゴリズムまたは振る舞い

  • 拡張機能のインストールや更新プロセスにおいて、型情報の検索を高速化するために使用されます。
  • 関連する拡張機能の変更が発生した際、キャッシュエントリを無効化する処理が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/extension.c 内の拡張機能管理ロジック。
  • 呼び出し先: 提供されたソース情報からは、特定の関数呼び出し関係は特定できません。

関連データ構造

  • ext_sibling_list: ExtensionSiblingCache 構造体の連結リストのヘッドを保持する静的変数。

失敗モードと境界条件

  • キャッシュの無効化タイミングが不適切な場合、古い型情報が参照される可能性があります。
  • メモリ割り当て失敗時の挙動については、ソースコードの当該箇所からは明示されていません。

保守担当者向けメモ

  • 本構造体は src/backend/commands/extension.c 内でのみ使用されるプライベートなキャッシュ機構です。
  • キャッシュの無効化ロジックが、拡張機能の依存関係変更と正しく同期されているかを確認してください。
