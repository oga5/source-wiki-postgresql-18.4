InitExecPartitionPruneContexts

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execPartition.c:2340

目的

実行時パーティションプルーニングにおいて、初期段階で除外されなかったサブプランのインデックスを再マッピングし、各パーティション階層の present_parts ビットマップを再構築する。また、必要に応じて実行時プルーニング用のコンテキストを初期化する。

シグネチャまたは形状

static void InitExecPartitionPruneContexts(
    PartitionPruneState *prunestate,
    PlanState *parent_plan,
    Bitmapset *initially_valid_subplans,
    int n_total_subplans
)

入力と出力

  • prunestate: 更新対象の PartitionPruneState 構造体。
  • parent_plan: 親となる PlanState。EState の取得に使用。
  • initially_valid_subplans: 初期プルーニング後に有効と判定されたサブプランのビットマップ。
  • n_total_subplans: 全サブプランの総数。
  • 出力: なし（prunestate 内部のデータ構造を直接更新）。

前提条件と不変条件

  • prunestate->do_exec_prune が真であること。
  • parent_plan が NULL でないこと。
  • initially_valid_subplans に含まれるインデックスは n_total_subplans 未満であること。

アルゴリズムまたは振る舞い

1. **マッピング生成**: 初期プルーニングでサブプランが除外された場合、古いサブプランインデックスから新しいインデックスへの変換配列を作成する。 2. **コンテキスト初期化**: 各パーティション階層を走査し、exec_pruning_steps が存在する場合は InitPartitionPruneContext を呼び出して実行時プルーニングコンテキストを初期化する。 3. **ビットマップ再構築**:

  • 各パーティションの present_parts をクリアし、有効なサブプランに基づいて再計算する。
  • サブパーティションが存在する場合、その present_parts が空でないことを確認して親のビットマップに含める。

    • 4. **その他サブプランの更新**: prunestate->other_subplans を新しいインデックス体系に合わせて更新する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execPartition.c 内の複数箇所（詳細はソース内の InitExecPartitionPruneContexts への参照を確認のこと）。
  • 呼び出し先: InitPartitionPruneContext, PartitionDirectoryLookup, RelationGetPartitionKey, bms_next_member, bms_add_member, bms_free, palloc0, pfree。

関連データ構造

  • PartitionPruneState: プルーニング状態を保持するメイン構造体。
  • PartitionPruningData / PartitionedRelPruningData: パーティション階層ごとのプルーニング情報。
  • PlanState: 実行計画の状態。

失敗モードと境界条件

  • initially_valid_subplans が全サブプランを含む場合、マッピングの修正処理はスキップされる。
  • パーティションが完全にプルーニングされた場合、対応するインデックスは -1 となり、present_parts から除外される。

保守担当者向けメモ

  • 本関数は PartitionPruneState の内部状態を破壊的に更新するため、呼び出しタイミングに注意すること。
  • present_parts の再計算は、階層の深い順（ボトムアップ）に行われるため、サブパーティションのプルーニング結果が親に正しく伝播するよう設計されている。
