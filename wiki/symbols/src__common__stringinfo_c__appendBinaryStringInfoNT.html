appendBinaryStringInfoNT

定義

  • 種別: function
  • 定義箇所: src/common/stringinfo.c:306

目的

StringInfo バッファに対して、ヌル終端を強制せずにバイナリデータを追加する。

シグネチャまたは形状

void appendBinaryStringInfoNT(StringInfo str, const void *data, int datalen)

入力と出力

  • str: 操作対象の StringInfo オブジェクト（入力および更新）。
  • data: 追加するバイナリデータのポインタ（入力）。
  • datalen: 追加するデータのバイト数（入力）。
  • 戻り値: なし。

前提条件と不変条件

  • str は NULL であってはならない（Assert により検証）。
  • str が指すバッファは、datalen 分のデータを格納するために必要なメモリを確保可能であること。

アルゴリズムまたは振る舞い

1. Assert を用いて str が有効であることを確認する。 2. enlargeStringInfo を呼び出し、現在のバッファ容量が不足している場合はメモリを再確保（拡張）する。 3. memcpy を使用して、str->data の現在の末尾位置に data から datalen バイトをコピーする。 4. str->len を datalen 分増加させる。 5. ヌル終端文字（\0）の書き込みは行わない。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/libpq/pqformat.c
  • src/backend/utils/mb/stringinfo_mb.c
  • 呼び出し先:
  • enlargeStringInfo (内部でバッファ拡張を実行)
  • memcpy (標準ライブラリ)

関連データ構造

  • StringInfoData: StringInfo が指す構造体。data（バッファ）、len（現在のデータ長）、maxlen（最大容量）を保持する。

失敗モードと境界条件

  • str が NULL の場合、アサーション有効時にプログラムが停止する。
  • メモリ不足が発生した場合、enlargeStringInfo 経由でメモリ割り当てエラーが発生する可能性がある。
  • ヌル終端を保証しないため、本関数で操作したバッファを C 言語の文字列関数（strlen 等）に渡すと、バッファ外参照を引き起こすリスクがある。

保守担当者向けメモ

  • 本関数は「No-Terminator」の略称通り、ヌル終端を付与しない。バイナリデータや、後続の処理で明示的に終端を管理するデータに対して使用すること。
  • ヌル終端が必要な場合は appendBinaryStringInfo を使用すること。
