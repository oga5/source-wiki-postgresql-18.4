hashint8

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:83

目的

int64 型の値をハッシュ値に変換します。hashint4 や hashint2 と論理的に等価な入力に対して互換性のあるハッシュ値を生成し、型を跨いだハッシュ結合をサポートすることを目的としています。

シグネチャまたは形状

Datum hashint8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して渡される int64 型のデータ。
  • **出力**: Datum 型として返される 32 ビットのハッシュ値。

前提条件と不変条件

  • 入力値が int64 型であること。
  • hashint4 および hashint2 との互換性を維持するため、符号に応じたビット操作を行うこと。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT64(0) で入力値を取得します。 2. 64ビット値を上位32ビット（hihalf）と下位32ビット（lohalf）に分割します。 3. 入力値が正またはゼロの場合は lohalf と hihalf を XOR し、負の場合は lohalf と hihalf のビット反転値を XOR します。 4. 得られた値を hash_uint32 に渡し、最終的なハッシュ値を計算して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/date.c
  • src/backend/utils/adt/pg_lsn.c
  • src/backend/utils/adt/timestamp.c
  • src/backend/utils/adt/xid.c
  • **呼び出し先**: hash_uint32

関連データ構造

  • Datum (PostgreSQLの汎用データ型)
  • int64, uint32 (C言語の固定長整数型)

失敗モードと境界条件

  • 負の数に対してはビット反転処理を行うことで、正の数との一貫性を保っています。
  • int64 の全範囲の値を処理可能です。

保守担当者向けメモ

  • 本関数は hashint4 および hashint2 との互換性が必須です。アルゴリズムを変更する際は、これらとのハッシュ値の衝突や不整合が発生しないよう注意してください。
  • hashint8extended が別途存在するため、ハッシュ結合のシード値が必要な場合はそちらを使用してください。
