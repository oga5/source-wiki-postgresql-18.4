ReleasePostmasterChildSlot

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pmchild.c:235

目的

終了した子プロセスのスロットを解放し、管理リストから削除して再利用可能な状態に戻す。

シグネチャまたは形状

bool ReleasePostmasterChildSlot(PMChild *pmchild)

入力と出力

  • **入力**: pmchild (解放対象の PMChild 構造体へのポインタ)
  • **出力**: bool (成功時に true を返す)

前提条件と不変条件

  • pmchild は ActiveChildList に登録済みであること。
  • B_DEAD_END_BACKEND 型以外の場合、pmchild->child_slot が対応する PMChildPool の範囲内にあること。

アルゴリズムまたは振る舞い

1. dlist_delete を呼び出し、ActiveChildList から対象の pmchild を削除する。 2. bkend_type が B_DEAD_END_BACKEND の場合、メモリを pfree で解放して終了する。 3. それ以外の場合:

  • B_WAL_SENDER 型であれば B_BACKEND プールを、それ以外は自身の型に対応するプールを選択する。
  • スロット番号がプールの範囲内にあるか検証し、範囲外であればエラーを投げる。
  • dlist_push_head を使用して、対象スロットをプールの freelist に戻す。
  • MarkPostmasterChildSlotUnassigned を呼び出し、共有メモリ上の状態を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c 内の各プロセス終了処理（Startup, BgWriter, Checkpointer, WalWriter, WalReceiver, WalSummarizer, AutoVacLauncher, PgArch, SysLogger, SlotSyncWorker 等）。
  • **呼び出し先**: dlist_delete, pfree, dlist_push_head, MarkPostmasterChildSlotUnassigned。

関連データ構造

  • PMChild: 子プロセスのメタデータを保持する構造体。
  • PMChildPool: プロセスタイプごとのスロット管理用プール。
  • ActiveChildList: 稼働中の全子プロセスを保持するリスト。

失敗モードと境界条件

  • pmchild->child_slot が所属するプールの範囲外である場合、elog(ERROR) によりプロセスが異常終了する。

保守担当者向けメモ

  • WAL Sender は起動時に通常のバックエンドとして扱われ、B_BACKEND プールを共有する仕様であるため、解放時も同様に B_BACKEND プールへ戻される。
