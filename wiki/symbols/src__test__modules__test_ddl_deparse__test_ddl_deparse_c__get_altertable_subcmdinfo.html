get_altertable_subcmdinfo

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_ddl_deparse/test_ddl_deparse.c:85

目的

CollectedCommand 構造体に含まれる ALTER TABLE コマンドのサブコマンドリストを走査し、各操作の種類を文字列として抽出してセット値（SRF）として返す。

シグネチャまたは形状

Datum get_altertable_subcmdinfo(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して CollectedCommand 構造体を受け取る。
  • **出力**: tuplestore に格納されたサブコマンド情報のタプルセットを Datum として返す。

前提条件と不変条件

  • 入力された CollectedCommand の type が SCT_AlterTable であること。
  • cmd->d.alterTable.subcmds が空（NIL）ではないこと。

アルゴリズムまたは振る舞い

1. InitMaterializedSRF を呼び出し、結果セットを初期化する。 2. cmd->d.alterTable.subcmds リストを foreach で反復処理する。 3. 各要素を CollectedATSubcmd としてキャストし、内部の AlterTableCmd の subtype を参照する。 4. switch 文を用いて AlterTableType を人間が読み取り可能な文字列（例: "ADD COLUMN", "SET STORAGE" 等）にマッピングする。 5. tuplestore_putvalues を使用して、変換した文字列をタプルとして構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(get_altertable_subcmdinfo) マクロにより、PostgreSQL の関数インターフェースとして登録されている。具体的な SQL クエリからの呼び出しについては、ソースコード内に直接的な記述はない。
  • **呼び出し先**: InitMaterializedSRF（SRF 初期化）、lfirst（リスト走査）、castNode（ノードキャスト）、tuplestore_putvalues（結果格納）、elog（エラー報告）。

関連データ構造

  • CollectedCommand: DDL 操作を保持する構造体。
  • CollectedATSubcmd: ALTER TABLE のサブコマンドを保持する構造体。
  • AlterTableCmd: PostgreSQL 内部の ALTER TABLE コマンドノード。
  • ReturnSetInfo: SRF の結果情報を管理する構造体。

失敗モードと境界条件

  • cmd->type が SCT_AlterTable 以外の場合、elog(ERROR) により処理を中断する。
  • subcmds が NIL の場合、elog(ERROR) により処理を中断する。
  • 未定義の AlterTableType に遭遇した場合、文字列は "unrecognized" となる。

保守担当者向けメモ

  • 本関数は ALTER TABLE のサブコマンドを網羅的にマッピングしている。新しい AlterTableType が PostgreSQL 本体に追加された際は、本関数の switch 文にもケースを追加する必要がある。
  • テスト専用モジュールであるため、プロダクション環境での利用は想定されていない。
