decimal

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-outofscope.c:56

目的

ECPG（Embedded SQL for C）において、数値データを保持および操作するための構造体定義です。

シグネチャまたは形状

typedef struct
{
	int			ndigits;
	int			weight;
	int			rscale;
	int			dscale;
	int			sign;
	NumericDigit digits[DECSIZE];
} decimal;

入力と出力

  • **入力**: 構造体の各メンバ（ndigits, weight, rscale, dscale, sign, digits）を通じて数値の構成要素を保持します。
  • **出力**: ECPGのSQL操作において、データベースの数値型データとC言語の変数間でのデータ交換に使用されます。

前提条件と不変条件

  • digits 配列のサイズは DECSIZE で規定されています。
  • ndigits は digits[] 内の有効な桁数を示し、0になる可能性があります。
  • sign は NUMERIC_POS（正）、NUMERIC_NEG（負）、または NUMERIC_NAN（非数）のいずれかの状態を保持する必要があります。

アルゴリズムまたは振る舞い

本構造体は、PostgreSQLの数値型（NUMERIC）をC言語環境で表現するためのデータコンテナとして機能します。ECPGプリプロセッサによって生成されたコード内で、データベースからフェッチされた数値データの格納先として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内での具体的な呼び出し元情報は確認できません。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すものではありません。

関連データ構造

  • NumericDigit: digits 配列の要素型として使用されます。
  • DECSIZE: digits 配列の長さを定義する定数です。

失敗モードと境界条件

  • DECSIZE を超える桁数の数値を扱おうとした場合、バッファオーバーフローやデータ切り捨てのリスクがあります。
  • sign に不正な値が設定された場合、数値の正負判定や演算結果が未定義となります。

保守担当者向けメモ

  • 本構造体は src/interfaces/ecpg/test/expected/preproc-outofscope.c 内で定義されていますが、ECPGのランタイムライブラリ（pgtypes.h 等）で定義されている標準的な decimal 型との整合性に注意してください。
  • 構造体のメモリレイアウトを変更する場合は、関連するECPGプリプロセッサの生成コードおよびテストケースへの影響を考慮してください。
