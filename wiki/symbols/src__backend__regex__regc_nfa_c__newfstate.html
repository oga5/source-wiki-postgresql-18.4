newfstate

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:211

目的

指定されたフラグ値を保持する新しいNFA（非決定性有限オートマトン）状態を割り当てます。

シグネチャまたは形状

static struct state *newfstate(struct nfa *nfa, int flag)

入力と出力

  • **入力**:
  • nfa: 状態を追加する対象のNFA構造体へのポインタ。
  • flag: 新しい状態に設定するフラグ値（char型として格納）。
  • **出力**:
  • 成功時: 新しく作成された struct state へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • newstate(nfa) がメモリ割り当てに成功している必要があります。
  • 戻り値が NULL でない場合、その状態の flag フィールドには引数 flag が設定されています。

アルゴリズムまたは振る舞い

1. newstate(nfa) を呼び出し、NFA内に新しい状態を生成します。 2. newstate が正常に状態を返した場合、その状態の flag メンバに引数 flag をキャストして代入します。 3. 生成された状態へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_nfa.c: newnfa 関数内（nfa->post および nfa->pre の初期化時）。
  • **呼び出し先**:
  • newstate (同ファイル内)

関連データ構造

  • struct nfa: NFAの全体構造を管理する構造体。
  • struct state: NFAの各状態を表現する構造体。

失敗モードと境界条件

  • newstate がメモリ不足等で失敗した場合、newfstate も同様に NULL を返します。

保守担当者向けメモ

  • 本関数は、特定のフラグを持つ状態を生成するためのラッパーです。状態のメモリ管理は newstate に依存しており、個別の解放は行わず、最終的に freenfa で一括解放される設計です。
