pg_lsn_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:101

目的

pg_lsn 型の値を、クライアントとの通信に適したバイナリ形式（bytea）に変換して送信する。

シグネチャまたは形状

Datum pg_lsn_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_LSN(0) を介して XLogRecPtr（64ビット整数）を受け取る。
  • **出力**: bytea 型のデータとして、ネットワークバイトオーダー（ビッグエンディアン）に変換された64ビット整数を返却する。

前提条件と不変条件

  • libpq/pqformat.h で定義された StringInfo インターフェースを介してメモリ管理を行う。
  • 入力された lsn は XLogRecPtr 型として正当である必要がある。

アルゴリズムまたは振る舞い

1. pq_begintypsend(&buf) を呼び出し、バイナリ送信用のバッファを初期化する。 2. pq_sendint64(&buf, lsn) を呼び出し、LSNの内部表現である64ビット整数をバッファに書き込む。 3. pq_endtypsend(&buf) を呼び出し、バッファの内容を bytea 形式に変換して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）経由で呼び出される。具体的な呼び出し元コードの詳細は提供されていない。
  • **呼び出し先**:
  • pq_begintypsend
  • pq_sendint64
  • pq_endtypsend

関連データ構造

  • XLogRecPtr: LSNの内部表現（uint64）。
  • StringInfoData: バイナリデータ構築用の一時バッファ。

失敗モードと境界条件

  • メモリ不足時、pq_begintypsend または pq_endtypsend 内部でエラーが発生する可能性がある。
  • 入力値が NULL の場合、PG_GETARG_LSN のマクロ定義に従い、呼び出し側で適切に処理される必要がある。

保守担当者向けメモ

  • 本関数は pg_lsn 型のバイナリプロトコルを定義する。プロトコル変更時には pq_sendint64 の使用箇所に注意すること。
  • XLogRecPtr は64ビット整数であるため、プラットフォーム間のエンディアン差異は pq_sendint64 が吸収する。
