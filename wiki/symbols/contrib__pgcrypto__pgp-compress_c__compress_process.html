compress_process

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-compress.c:106

目的

PGPデータストリームの圧縮フィルタ処理において、入力データを zlib の deflate アルゴリズムを用いて圧縮し、後続のフィルタへ書き出す。

シグネチャまたは形状

static int compress_process(PushFilter *next, void *priv, const uint8 *data, int len)

入力と出力

  • **入力**:
  • next: 後続の PushFilter オブジェクト。
  • priv: 内部状態を保持する ZipStat 構造体へのポインタ。
  • data: 圧縮対象の入力データバッファ。
  • len: 入力データの長さ。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は負の pgcrypto エラーコード（PXE_PGP_COMPRESSION_ERROR または pushf_write の戻り値）を返す。

前提条件と不変条件

  • priv は ZipStat 構造体として初期化済みであること。
  • zlib ライブラリが利用可能であること。

アルゴリズムまたは振る舞い

1. st->stream に入力データ (data, len) を設定する。 2. avail_in が 0 になるまでループし、以下の処理を繰り返す。

  • deflate() を Z_NO_FLUSH モードで呼び出し、圧縮を実行する。
  • 圧縮処理が Z_OK 以外を返した場合は PXE_PGP_COMPRESSION_ERROR を返す。
  • 圧縮されたデータが生成された場合、pushf_write() を介して後続のフィルタへ書き出す。
  • 書き込み処理が失敗した場合は、そのエラーコードをそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: pgp-compress.c 内のフィルタ構造体初期化処理（compress_init 等で使用される関数ポインタ経由）。
  • **呼び出し先**:
  • deflate (zlib)
  • pushf_write (フィルタパイプラインの書き込み関数)

関連データ構造

  • ZipStat: 圧縮状態（z_stream および内部バッファ）を保持する構造体。
  • PushFilter: フィルタパイプラインの抽象インターフェース。

失敗モードと境界条件

  • deflate() が Z_OK 以外の値を返した場合、圧縮エラーとして終了する。
  • pushf_write() が負の値を返した場合、そのエラーを伝播させる。
  • 入力データが空の場合、ループは実行されず 0 を返す。

保守担当者向けメモ

  • 本関数は Z_NO_FLUSH を使用しているため、ストリームの終了時には別途 compress_flush を呼び出して残りのバッファをフラッシュする必要がある。
  • zlib のメモリ確保には PostgreSQL の palloc をラップした関数が使用されているため、メモリ管理は ZipStat のライフサイクルに依存する。
