port_bio_ctrl

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-openssl.c:954

目的

OpenSSL の BIO インターフェースに対する制御コマンドを処理し、PostgreSQL の Port 構造体と OpenSSL 間の状態同期を行う。

シグネチャまたは形状

long port_bio_ctrl(BIO *h, int cmd, long num, void *ptr)

入力と出力

  • **入力**:
  • h: 操作対象の BIO オブジェクト。
  • cmd: 実行する制御コマンド（BIO_CTRL_EOF, BIO_CTRL_FLUSH 等）。
  • num, ptr: コマンド固有の引数。
  • **出力**:
  • long: コマンドの実行結果。成功時は 1 または状態値、デフォルトは 0。

前提条件と不変条件

  • BIO_get_data(h) を通じて、有効な Port 構造体へのポインタが取得可能であること。
  • BIO メソッドとして登録され、OpenSSL ライブラリから呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. BIO_get_data(h) から Port 構造体を取得する。 2. cmd に基づき分岐する:

  • BIO_CTRL_EOF: port->last_read_was_eof の値を返し、EOF 状態を OpenSSL に通知する。
  • BIO_CTRL_FLUSH: 常に 1 を返し、フラッシュ要求を正常終了させる。
  • その他: 0 を返し、未サポートのコマンドであることを示す。

呼び出し元と呼び出し先

  • **呼び出し元**: OpenSSL ライブラリ内部（BIO_ctrl 経由）。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。
  • **参照**: src/backend/libpq/be-secure-openssl.c:1000 にて BIO_meth_set_ctrl を介してメソッドテーブルに登録される。

関連データ構造

  • BIO: OpenSSL の I/O 抽象化構造体。
  • Port: PostgreSQL の接続状態を保持する構造体。

失敗モードと境界条件

  • BIO_CTRL_EOF 以外の未知のコマンドが渡された場合、0 を返して無視する。
  • BIO_get_data(h) が NULL を返す場合の挙動については、本ソースコード内では明示的なチェックは行われていない。

保守担当者向けメモ

  • BIO_CTRL_EOF の処理は、OpenSSL の非互換な仕様変更（Issue #8208）に対応するための回避策である。将来的な OpenSSL のアップデート時に挙動を確認すること。
