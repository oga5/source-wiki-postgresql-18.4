pg_buffercache_numa_pages

定義

  • 種別: function
  • 定義箇所: contrib/pg_buffercache/pg_buffercache_pages.c:304

目的

共有バッファキャッシュ内の各バッファが配置されているメモリページと、それに対応する NUMA ノード情報を取得し、SRF（Set-Returning Function）として提供する。

シグネチャまたは形状

Datum pg_buffercache_numa_pages(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL インターフェース経由の呼び出し引数）。
  • **出力**: bufferid (int4), os_page_num (int8), numa_node (int4) を含むタプルを順次返す。

前提条件と不変条件

  • OS が move_pages(2) システムコールをサポートしている必要がある。
  • pg_numa_init() が成功すること。
  • データベースのブロックサイズ（BLCKSZ）と OS のページサイズが 2 のべき乗の関係にあり、一方が他方を割り切れること。
  • 呼び出し元は SRF_IS_FIRSTCALL() を用いて初回呼び出しを判定し、コンテキストを保持する必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: pg_numa_init() を呼び出し、NUMA サポートを確認する。 2. **メモリマッピング**: 全バッファ領域をカバーする OS ページ範囲を特定し、pg_numa_query_pages() を用いて全ページの NUMA ノード情報を一括取得する。 3. **コンテキスト作成**: FuncCallContext を初期化し、後続の呼び出しのために BufferCacheNumaContext を確保する。 4. **スキャン**: 全バッファ（NBuffers）を順次走査する。各バッファヘッダーを LockBufHdr でロックして状態を読み取り、対応するメモリページの NUMA ノード情報を記録する。 5. **中断対応**: ループ内で CHECK_FOR_INTERRUPTS() を実行し、クエリの中断要求に応答する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: pg_numa_init, pg_get_shmem_pagesize, pg_numa_query_pages, BufferGetBlock, GetBufferDescriptor, LockBufHdr, UnlockBufHdr 等。

関連データ構造

  • BufferCacheNumaContext: 複数回の呼び出し間で状態を保持するためのコンテキスト構造体。
  • BufferCacheNumaRec: 各バッファの NUMA 情報を格納するレコード構造体。
  • BufferDesc: バッファヘッダー情報。

失敗モードと境界条件

  • **初期化失敗**: pg_numa_init() が失敗した場合、ERROR をスローする。
  • **型不一致**: 戻り値の型が期待される行型と異なる場合、ERROR をスローする。
  • **NUMA 問い合わせ失敗**: pg_numa_query_pages() が失敗した場合、ERROR をスローする。
  • **一貫性**: バッファヘッダーのロックは個別に取得されるため、関数全体としてシステム全体の一貫したスナップショットは保証されない。

保守担当者向けメモ

  • 本関数は move_pages(2) を使用するため、OS のカーネルサポートに強く依存する。
  • NBuffers が大きい場合、メモリ消費量とロック競合に注意が必要。
  • firstNumaTouch フラグにより、初回アクセス時にメモリページへのタッチが行われる。
