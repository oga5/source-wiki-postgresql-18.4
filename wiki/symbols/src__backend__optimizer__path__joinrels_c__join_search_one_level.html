join_search_one_level

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/path/joinrels.c:72

目的

動的計画法を用いた結合探索において、指定された結合レベル（level）で可能なすべての結合リレーションを生成・探索する。

シグネチャまたは形状

void join_search_one_level(PlannerInfo *root, int level)

入力と出力

  • **入力**:
  • root: クエリ全体のプランニング情報を保持する PlannerInfo 構造体。
  • level: 現在探索対象としている結合レベル（結合するリレーションの総数）。
  • **出力**: なし（副作用として root->join_rel_level[level] に新しい RelOptInfo が追加される）。

前提条件と不変条件

  • root->join_rel_level[level] は、呼び出し時に NIL である必要がある。
  • root->join_cur_level は、実行中に現在の level に設定される。

アルゴリズムまたは振る舞い

1. **左/右結合の探索**: level-1 のリレーションと初期リレーション（joinrels[1]）を結合する。結合条件や制約がある場合は make_rels_by_clause_joins を、ない場合は make_rels_by_clauseless_joins を呼び出す。 2. **Bushy Plan の探索**: k と level-k（2 <= k <= level-2）の組み合わせで、結合条件または制約が存在する場合に make_join_rel を呼び出し、結合リレーションを構築する。 3. **最終手段（Cartesian Product）**: 上記の探索で joinrels[level] が空の場合、level-1 と初期リレーションの間で強制的にデカルト積を生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/allpaths.c
  • **呼び出し先**: make_rels_by_clause_joins, make_rels_by_clauseless_joins, make_join_rel, has_join_restriction, have_relevant_joinclause, have_join_order_restriction

関連データ構造

  • PlannerInfo: クエリ全体のプランニング状態を保持。
  • RelOptInfo: 結合対象となるリレーションの情報を保持。

失敗モードと境界条件

  • 特殊な結合（SpecialJoinInfo）や LATERAL 参照が存在する場合、特定のレベルで結合が生成されないことがある。その場合、関数はエラーを返さず、後続のレベルでの探索に委ねる。
  • 結合条件がない場合、デカルト積が生成される。

保守担当者向けメモ

  • 本関数は結合探索のメインループを制御する。パフォーマンス上の理由から、Bushy Plan の探索は結合条件や制約が存在する場合のみに制限されている。
  • level == 2 の場合、対称性を考慮して探索範囲を最適化している。
