test_status_ok

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress.c:302

目的

テストケースが成功したことを記録し、その結果を標準出力へ報告する。

シグネチャまたは形状

static void test_status_ok(const char *testname, double runtime, bool parallel)

入力と出力

  • **入力**:
  • testname: 実行されたテストの名前（文字列）。
  • runtime: テストの実行時間（ミリ秒単位の浮動小数点数）。
  • parallel: 並列実行されていたかどうかを示すフラグ。
  • **出力**: なし（void）。内部でグローバルな成功カウンタを更新し、ログを出力する。

前提条件と不変条件

  • グローバル変数 success_count がこの関数によってインクリメントされる。
  • test_status_print を呼び出すことで、TAP 形式等の出力が行われる。

アルゴリズムまたは振る舞い

1. グローバル変数 success_count を 1 増やす。 2. test_status_print を呼び出し、成功ステータス（true）と引数で受け取ったテスト名、実行時間、並列フラグを渡して結果を表示する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/regress/pg_regress.c:1827
  • src/test/regress/pg_regress.c:1907
  • **呼び出し先**:
  • test_status_print (同ファイル内)

関連データ構造

  • success_count: テストの成功数を保持するグローバル変数。

失敗モードと境界条件

  • 本関数自体は成功を記録する処理であるため、関数内部での失敗モードは定義されていない。
  • testname に NULL が渡された場合の挙動は test_status_print の実装に依存する。

保守担当者向けメモ

  • 本関数はテストの成功報告を抽象化するラッパーである。成功時の出力フォーマットを変更する場合は、呼び出し先の test_status_print を参照すること。
