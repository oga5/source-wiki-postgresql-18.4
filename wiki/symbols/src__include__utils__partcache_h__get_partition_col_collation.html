get_partition_col_collation

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/partcache.h:97

目的

パーティションキーの特定の列に対応する照合順序（Collation）の OID を取得します。

シグネチャまたは形状

Oid get_partition_col_collation(PartitionKey key, int col)

入力と出力

  • **入力**:
  • key: パーティションキー情報を保持する PartitionKey 構造体へのポインタ。
  • col: 照合順序を取得したい列のインデックス（0から始まる整数）。
  • **出力**:
  • 指定された列の照合順序を表す Oid。

前提条件と不変条件

  • key は有効な PartitionKey 構造体を指している必要があります。
  • パーティションキーはリレーションが開かれている間は不変であるため、取得されるポインタおよびデータは安全に参照可能です。

アルゴリズムまたは振る舞い

key 構造体内の partcollation 配列から、指定されたインデックス col に対応する要素を直接参照して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_utilcmd.c (行 4368 および 4523) にて、パーティション定義の解析時に呼び出されています。
  • **呼び出し先**:
  • 本関数はインライン関数であり、他の関数を呼び出しません。

関連データ構造

  • PartitionKey: パーティションキーのメタデータを保持する構造体。本関数はこの構造体内の partcollation フィールドに依存しています。

失敗モードと境界条件

  • col が key に定義された列数を超えている場合、メモリ上の不正な領域を参照する可能性があります（境界チェックは本関数内では行われていません）。

保守担当者向けメモ

  • 本関数は PartitionKey 構造体への直接アクセスをカプセル化するインライン関数群の一つです。将来的に PartitionKeyData の構造が変更された場合、この抽象化レイヤーを通じて修正を行う必要があります。
