ClientSocket

定義

  • **種別**: struct
  • **定義箇所**: src/include/libpq/libpq-be.h:248

目的

クライアント接続の初期段階において、ネットワークソケットのファイル記述子とリモートアドレス情報を保持するための軽量なデータ構造です。

シグネチャまたは形状

typedef struct ClientSocket
{
    pgsocket    sock;           /* File descriptor */
    SockAddr    raddr;          /* remote addr (client) */
} ClientSocket;

入力と出力

  • **入力**: 接続確立時にソケット記述子およびリモートアドレス情報が格納されます。
  • **出力**: 認証プロセスやバックエンドプロセスの初期化時に、この構造体の内容が Port 構造体へコピーまたは引き継がれます。

前提条件と不変条件

  • sock は有効なソケットファイル記述子である必要があります。
  • raddr は接続元クライアントの有効なアドレス情報を保持している必要があります。

アルゴリズムまたは振る舞い

1. **接続受入**: AcceptConnection 等により、新規接続のソケットとアドレスが本構造体に格納されます。 2. **伝播**: launch_backend.c 等のプロセス起動処理において、この構造体はメモリコピー（memcpy）を通じてバックエンドプロセスへ渡されます。 3. **昇格**: バックエンドプロセス内で、この構造体の情報はより詳細な接続状態を管理する Port 構造体へ統合されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照・操作）**:
  • src/backend/libpq/pqcomm.c: pq_init, AcceptConnection
  • src/backend/postmaster/launch_backend.c: プロセス起動時の引数および MyClientSocket へのコピー処理
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • Port: クライアント接続の全情報を保持する中心的な構造体。ClientSocket の内容は Port に引き継がれます。
  • SockAddr: raddr フィールドで使用されるアドレス情報構造体。
  • pgsocket: sock フィールドで使用されるソケット記述子の型。

失敗モードと境界条件

  • ソケットの初期化失敗や、アドレス情報の取得失敗時には、本構造体が不完全な状態で保持される可能性があります。
  • プロセス間での受け渡し時にメモリ確保（palloc）が失敗した場合、接続処理が中断されます。

保守担当者向けメモ

  • 本構造体は接続の「初期状態」を保持するものです。接続確立後の詳細な通信状態については Port 構造体を参照してください。
  • launch_backend.c での memcpy によるコピーが行われているため、構造体のサイズ変更やフィールド追加を行う際は、バイナリ互換性に注意してください。
