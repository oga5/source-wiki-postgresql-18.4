add_preprocessor_define

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/ecpg.c:89

目的

コマンドライン引数で指定されたプリプロセッサ定義（-D オプション）を内部の定義リストに追加し、プリプロセス時に参照可能にする。

シグネチャまたは形状

static void add_preprocessor_define(char *define);

入力と出力

  • **入力**: define (文字列)。NAME または NAME=VALUE の形式。
  • **出力**: なし（副作用としてグローバルな定義リスト defines を更新）。

前提条件と不変条件

  • define 引数は mm_strdup によって複製され、呼び出し元のメモリ管理から独立する。
  • defines グローバル変数が定義リストの先頭を保持する。
  • cmdvalue は明示的に解放されない前提でメモリ管理される。

アルゴリズムまたは振る舞い

1. 引数 define を複製し、作業用メモリを確保する。 2. 文字列内に = が含まれるか確認する。

  • 含まれる場合: = を区切り文字として名前と値を分割し、名前の末尾の空白を除去する。
  • 含まれない場合: 値を "1" として定義する。

    • 3. struct _defines 型の新しいノードを割り当て、名前と値を格納する。 4. 新しいノードをグローバルな defines リストの先頭に挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/ecpg.c 内の main 関数（コマンドライン引数解析時）。
  • **呼び出し先**: mm_strdup, mm_alloc, strchr（いずれも標準またはユーティリティ関数）。

関連データ構造

  • struct _defines: プリプロセッサ定義を保持する連結リスト構造体。name, value, cmdvalue, used, next を含む。

失敗モードと境界条件

  • = が含まれない場合、値は自動的に "1" となる。
  • 名前と = の間に空白がある場合、その空白は除去される。
  • メモリ割り当て失敗時の挙動は mm_alloc に依存する。

保守担当者向けメモ

  • cmdvalue は解放されない設計であるため、メモリリークを懸念する必要はない。
  • defines リストはグローバル変数であるため、複数ファイル処理時のリセット処理が適切に行われているか注意すること。
