systable_endscan_ordered

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:756

目的

順序付きシステムテーブルスキャンの終了処理を行い、関連するリソースを解放する。

シグネチャまたは形状

void systable_endscan_ordered(SysScanDesc sysscan)

入力と出力

  • **入力**: sysscan (SysScanDesc) - 終了対象のスキャン記述子。
  • **出力**: なし。

前提条件と不変条件

  • sysscan は有効なポインタであること。
  • sysscan->irel が存在すること（Assert により検証）。

アルゴリズムまたは振る舞い

1. sysscan->slot が存在する場合、ExecDropSingleTupleTableSlot を呼び出してスロットを破棄し、ポインタを NULL にリセットする。 2. index_endscan を呼び出し、インデックススキャンを終了する。 3. sysscan->snapshot が存在する場合、UnregisterSnapshot を呼び出してスナップショットの参照カウントを減らす。 4. CheckXidAlive が有効なトランザクションIDである場合、bsysscan フラグを false にリセットする。 5. pfree(sysscan) を呼び出し、スキャン記述子メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • src/backend/access/common/toast_internals.c
  • src/backend/access/heap/heaptoast.c
  • src/backend/storage/large_object/inv_api.c
  • src/backend/utils/adt/enum.c
  • src/backend/utils/cache/evtcache.c
  • **呼び出し先**:
  • ExecDropSingleTupleTableSlot
  • index_endscan
  • UnregisterSnapshot
  • pfree

関連データ構造

  • SysScanDesc: スキャン状態を保持する構造体。

失敗モードと境界条件

  • sysscan->irel が NULL の場合、アサーション失敗により異常終了する。
  • sysscan が指すメモリが既に解放済みである場合、二重解放や不正なメモリ参照が発生する可能性がある。

保守担当者向けメモ

  • bsysscan フラグのリセットは、xact.c で定義されているトランザクション状態管理に関連しているため、変更時は該当箇所のコメントを参照すること。
