rbt_delete

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:694

目的

赤黒木から指定されたノードを削除し、木構造の平衡を維持する。

シグネチャまたは形状

void rbt_delete(RBTree *rbt, RBTNode *node)

入力と出力

  • **入力**:
  • rbt: 削除対象のノードが属する RBTree 構造体へのポインタ。
  • node: 削除対象の RBTNode 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は rbt に含まれている必要がある。
  • 削除後、赤黒木の不変条件（各ノードの色、ルートから葉までの黒ノード数の一致など）が維持される。

アルゴリズムまたは振る舞い

1. 内部関数 rbt_delete_node(rbt, node) を呼び出し、物理的なノードの削除を実行する。 2. 削除に伴う平衡の崩れを修正するため、内部的に rbt_delete_fixup を用いて回転および色の再割り当てを行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/rbtree.c 内の内部処理。
  • src/test/modules/test_rbtree/test_rbtree.c（テストコード）。
  • **呼び出し先**:
  • rbt_delete_node（直接的な処理の委譲）。

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTNode: 木の各ノードのヘッダー。

失敗モードと境界条件

  • node が RBTNIL（番兵）である場合や、木に含まれないノードを指定した場合の挙動については、呼び出し側で適切に管理する必要がある。
  • 物理的なメモリ解放は本関数内では行われず、呼び出し側が提供するコールバック関数を通じて管理される。

保守担当者向けメモ

  • 本関数は rbt_delete_node のラッパーとして機能している。削除ロジックの詳細は rbt_delete_node および rbt_delete_fixup を参照すること。
