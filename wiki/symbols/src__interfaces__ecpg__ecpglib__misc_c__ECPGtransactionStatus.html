ECPGtransactionStatus

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/misc.c:144

目的

指定された接続名に対応するデータベース接続の現在のトランザクション状態を取得します。

シグネチャまたは形状

PGTransactionStatusType ECPGtransactionStatus(const char *connection_name)

入力と出力

  • **入力**: connection_name (接続を識別する文字列)
  • **出力**: PGTransactionStatusType 型のトランザクション状態値。

前提条件と不変条件

  • 内部で ecpg_get_connection を呼び出し、接続の検索を行います。
  • 接続が見つからない場合、状態は不明とみなされます。

アルゴリズムまたは振る舞い

1. ecpg_get_connection(connection_name) を呼び出し、対応する connection 構造体を取得します。 2. 接続が存在しない場合、PQTRANS_UNKNOWN を返します。 3. 接続が存在する場合、その内部の connection メンバを使用して libpq の PQtransactionStatus を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: ecpg_get_connection (内部関数), PQtransactionStatus (libpq関数)
  • **呼び出し元**: 提供されたソースコード内には、本関数を呼び出している具体的な関数名は記載されていません。

関連データ構造

  • connection: src/interfaces/ecpg/ecpglib/ecpglib_extern.h 等で定義される接続管理構造体。
  • PGTransactionStatusType: libpq で定義されるトランザクション状態の列挙型。

失敗モードと境界条件

  • connection_name に対応する接続が存在しない場合、エラーや例外を発生させるのではなく、PQTRANS_UNKNOWN を返して正常に終了します。

保守担当者向けメモ

  • 本関数は libpq の PQtransactionStatus のラッパーです。ECPG 固有の接続管理層を介して状態を問い合わせるため、マルチスレッド環境下での接続の有効性に依存します。
