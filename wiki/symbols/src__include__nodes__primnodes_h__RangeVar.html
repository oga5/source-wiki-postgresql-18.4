RangeVar

定義

  • **種別**: struct
  • **定義箇所**: src/include/nodes/primnodes.h:72

目的

クエリ内のテーブル、シーケンス、またはビューなどのリレーションを識別するための構造体です。解析段階において、FROM句やターゲットリスト等で指定されたリレーションの場所（カタログ、スキーマ、名前）を保持します。

シグネチャまたは形状

typedef struct RangeVar {
    NodeTag     type;
    char       *catalogname;    /* カタログ名 (NULL可) */
    char       *schemaname;     /* スキーマ名 (NULL可) */
    char       *relname;        /* リレーション名 */
    bool        inh;            /* 継承を含めるか */
    char        relpersistence; /* 永続性 (pg_class.h参照) */
    Alias      *alias;          /* テーブル/列のエイリアス */
    ParseLoc    location;       /* トークン位置 (-1で不明) */
} RangeVar;

入力と出力

  • **入力**: SQLクエリの解析時にパーサーによって生成されます。
  • **出力**: 解析ツリーの一部として後続の変換処理（transform）へ渡され、最終的にリレーションのOID解決や計画作成に利用されます。

前提条件と不変条件

  • relname は必須です。
  • catalogname および schemaname は、指定がない場合は NULL となります。
  • location はソースコード上の位置を指し、不明な場合は -1 が設定されます。

アルゴリズムまたは振る舞い

クエリの解析過程で、ユーザーが指定したリレーション名を保持し、必要に応じてスキーマ検索パスを用いた解決を行うための基盤となります。inh フラグが true の場合、継承関係にある子テーブルも対象に含まれます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/dblink/dblink.c、contrib/pageinspect/btreefuncs.c、contrib/pgstattuple/pgstatindex.c 等の拡張モジュールで、リレーションを特定するために利用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Alias: テーブルや列の別名を定義する構造体。
  • NodeTag: PostgreSQLのノードシステムで型を識別するためのタグ。

失敗モードと境界条件

  • relname が NULL の場合、不正な状態となります。
  • location が -1 の場合、エラーメッセージ等でクエリ内の正確な位置を表示できません。

保守担当者向けメモ

  • 本構造体は primnodes.h に定義されていますが、主に解析段階（parsenodes.h 関連）で生成・利用される点に注意してください。
  • relpersistence フィールドは pg_class.h の定数と連動しているため、永続性に関する変更時は該当ヘッダーとの整合性を確認してください。
