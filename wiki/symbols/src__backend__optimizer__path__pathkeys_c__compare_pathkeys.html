compare_pathkeys

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/pathkeys.c:303

目的

2つのパスキーリスト（List *keys1, List *keys2）を比較し、それらの包含関係や等価性を判定します。

シグネチャまたは形状

PathKeysComparison compare_pathkeys(List *keys1, List *keys2)

入力と出力

  • **入力**: 比較対象となる2つの List（PathKey 構造体のリスト）。
  • **出力**: PathKeysComparison 型の列挙値。
  • PATHKEYS_EQUAL: 両リストが完全に一致。
  • PATHKEYS_DIFFERENT: 両リストが一致せず、包含関係もない。
  • PATHKEYS_BETTER1: keys1 が keys2 を含み、かつ keys1 の方が長い。
  • PATHKEYS_BETTER2: keys2 が keys1 を含み、かつ keys2 の方が長い。

前提条件と不変条件

  • 入力されるリスト内の各 PathKey は、正規化（Canonical）済みである必要があります。これにより、ポインタ比較（==）による高速な判定が可能です。

アルゴリズムまたは振る舞い

1. **同一性チェック**: keys1 == keys2 の場合、直ちに PATHKEYS_EQUAL を返します。 2. **順次比較**: forboth マクロを用いて両リストを先頭から順に走査します。 3. **不一致判定**: 各要素のポインタを比較し、一致しない場合は直ちに PATHKEYS_DIFFERENT を返します。 4. **長さの判定**: ループ終了後、リストの長さを比較します。

  • keys1 が残っていれば keys1 の方が長いため PATHKEYS_BETTER1 を返します。
  • keys2 が残っていれば keys2 の方が長いため PATHKEYS_BETTER2 を返します。
  • 両方とも終了していれば PATHKEYS_EQUAL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/pathkeys.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/util/pathnode.c
  • **呼び出し先**:
  • lfirst（リスト要素取得マクロ）
  • forboth（リスト走査マクロ）

関連データ構造

  • PathKey: 順序を表現する構造体。
  • List: PostgreSQL の汎用リスト構造。
  • PathKeysComparison: 比較結果を保持する列挙型。

失敗モードと境界条件

  • 両方のリストが NIL（空）の場合、PATHKEYS_EQUAL を返します。
  • 一方のリストが空で他方が空でない場合、空でない方が長いと判定されます。

保守担当者向けメモ

  • 本関数はポインタ比較に依存しているため、比較対象の PathKey が必ず正規化されていることを保証してください。
  • 包含関係の判定において、リストの先頭からの完全一致を前提としています。部分的な順序の一致を確認したい場合は、他の関数（pathkeys_contained_in 等）の利用を検討してください。
