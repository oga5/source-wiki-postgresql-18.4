convert_tuples_by_position

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tupconvert.c:58

目的

2つのタプル記述子（TupleDesc）間で、属性の物理的な位置に基づいた変換を行うための TupleConversionMap を生成する。

シグネチャまたは形状

TupleConversionMap *convert_tuples_by_position(TupleDesc indesc, TupleDesc outdesc, const char *msg)

入力と出力

  • **入力**:
  • indesc: 入力タプルの記述子。
  • outdesc: 出力タプルの記述子。
  • msg: 互換性エラー発生時に使用されるエラーメッセージの接頭辞。
  • **出力**:
  • 変換が必要な場合は初期化済みの TupleConversionMap へのポインタ。
  • 変換が不要（物理的に同一）な場合は NULL。

前提条件と不変条件

  • build_attrmap_by_position を呼び出し、属性の互換性を検証する。
  • 変換マップが生成される場合、outvalues/outisnull および invalues/inisnull 用のワークスペースが palloc により確保される。
  • invalues[0] および inisnull[0] は、NULL 値を表現するために固定で初期化される。

アルゴリズムまたは振る舞い

1. build_attrmap_by_position を実行し、属性マッピングを取得する。 2. マッピングが NULL であれば、変換不要と判断して即座に NULL を返す。 3. 変換が必要な場合、TupleConversionMap 構造体をメモリ確保し、入力・出力の記述子とマッピング情報を格納する。 4. 出力側および入力側の属性数に基づき、Datum 配列と bool 配列（NULLフラグ用）をワークスペースとして確保する。 5. 入力側の NULL ワークスペースのインデックス 0 を NULL 値として初期化し、マップを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/tstoreReceiver.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • build_attrmap_by_position (属性マッピングの構築)
  • palloc (メモリ確保)

関連データ構造

  • TupleConversionMap: 変換に必要なマッピング情報とワークスペースを保持する構造体。
  • TupleDesc: タプルの構造定義。
  • AttrMap: 属性間の対応関係を定義する構造体。

失敗モードと境界条件

  • build_attrmap_by_position が互換性エラーを検出した場合、内部でエラーが発生し処理が中断される可能性がある。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数が NULL を返す場合、呼び出し側で変換処理をスキップすることで最適化が可能である。
  • 確保されたメモリは free_conversion_map で解放する必要がある。
  • TupleDesc 自体の生存期間管理は本関数では行わないため、呼び出し側で適切に管理すること。
