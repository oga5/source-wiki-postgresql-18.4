pg_hmac_final

定義

  • src/common/hmac_openssl.c:241

目的

HMAC 計算の最終ステップを実行し、計算された MAC 値をバッファに書き出す。

シグネチャまたは形状

int pg_hmac_final(pg_hmac_ctx *ctx, uint8 *dest, size_t len);

入力と出力

  • ctx: HMAC コンテキストへのポインタ。
  • dest: 計算結果を格納するバッファ。
  • len: dest バッファのサイズ。
  • 戻り値: 成功時に 0、失敗時に -1 を返す。

前提条件と不変条件

  • ctx が NULL であってはならない。
  • len は、選択されたハッシュアルゴリズム（MD5, SHA系）の出力長以上である必要がある。

アルゴリズムまたは振る舞い

1. ctx が NULL の場合、-1 を返す。 2. ctx->type に基づき、len が各ハッシュアルゴリズムの最小必要サイズ（MD5_DIGEST_LENGTH 等）を満たしているか検証する。満たさない場合は ctx->error に PG_HMAC_ERROR_DEST_LEN を設定し、-1 を返す。 3. OpenSSL の HMAC_Final を呼び出し、MAC 値を dest に生成する。 4. HMAC_Final が失敗（戻り値が 0 以下）した場合、ERR_get_error() で OpenSSL のエラーを取得し、SSLerrmessage を経由して ctx->errreason に格納し、ctx->error に PG_HMAC_ERROR_OPENSSL を設定して -1 を返す。 5. 成功時は 0 を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/libpq/auth-scram.c
  • src/common/hmac.c
  • src/common/scram-common.c
  • src/interfaces/libpq/fe-auth-scram.c
  • 呼び出し先:
  • HMAC_Final (OpenSSL)
  • SSLerrmessage
  • ERR_get_error (OpenSSL)

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する構造体。

失敗モードと境界条件

  • ctx が NULL の場合: -1 を返す。
  • バッファサイズ不足: len がアルゴリズムの出力長より小さい場合、PG_HMAC_ERROR_DEST_LEN を設定し -1 を返す。
  • OpenSSL 内部エラー: HMAC_Final が失敗した場合、PG_HMAC_ERROR_OPENSSL を設定し -1 を返す。

保守担当者向けメモ

  • 本関数は OpenSSL の HMAC_Final をラップするインターフェースである。
  • エラー発生時は ctx->error および ctx->errreason に詳細が記録されるため、呼び出し側でこれらを参照可能である。
