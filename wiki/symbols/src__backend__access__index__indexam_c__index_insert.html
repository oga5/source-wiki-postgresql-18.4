index_insert

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/indexam.c:212

目的

インデックスリレーションに対して新しいタプルを挿入するための汎用インターフェースを提供します。特定のインデックスアクセス手法（AM）の実装詳細を隠蔽し、実行エンジンが統一的にインデックス更新を行えるようにします。

シグネチャまたは形状

bool index_insert(Relation indexRelation,
                  Datum *values,
                  bool *isnull,
                  ItemPointer heap_t_ctid,
                  Relation heapRelation,
                  IndexUniqueCheck checkUnique,
                  bool indexUnchanged,
                  IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • indexRelation: 挿入対象のインデックスリレーション。
  • values, isnull: 挿入するキー値とそのNULLフラグの配列。
  • heap_t_ctid: 挿入対象のヒープタプルを指すTID。
  • heapRelation: 対応するヒープリレーション。
  • checkUnique: 一意性制約チェックの要否（IndexUniqueCheck）。
  • indexUnchanged: インデックス値が変更されていないかを示すフラグ。
  • indexInfo: インデックスに関するメタ情報。
  • **出力**:
  • bool: 挿入処理の結果（通常はAMの実装に依存）。

前提条件と不変条件

  • RELATION_CHECKS マクロにより、リレーションが適切にオープンされていることが検証されます。
  • CHECK_REL_PROCEDURE(aminsert) により、対象のインデックスAMが aminsert 関数を実装していることが必須となります。
  • インデックスが述語ロック（predicate locks）をサポートしていない場合、シリアライザブルな競合チェック（CheckForSerializableConflictIn）が実行されます。

アルゴリズムまたは振る舞い

1. リレーションの整合性チェックおよびAM固有の挿入関数が定義されているかを確認します。 2. インデックスが述語ロックをサポートしていない場合、シリアライザブルな競合の有無をチェックします。 3. indexRelation->rd_indam->aminsert を呼び出し、実際の挿入処理を各インデックス型（B-tree, GIN等）のAM実装に委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/toast_internals.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/catalog/indexing.c
  • src/backend/commands/constraint.c
  • src/backend/executor/execIndexing.c
  • **呼び出し先**:
  • indexRelation->rd_indam->aminsert（各インデックスAMの実装関数）
  • CheckForSerializableConflictIn（述語ロックチェック時）

関連データ構造

  • Relation: インデックスおよびヒープリレーションの構造体。
  • IndexInfo: インデックスの属性や制約情報を含む構造体。
  • IndexUniqueCheck: 一意性制約のチェックレベルを定義する列挙型。

失敗モードと境界条件

  • CHECK_REL_PROCEDURE により、AMが aminsert を実装していない場合はエラーが発生します。
  • RELATION_CHECKS により、不正なリレーション状態での呼び出しはエラーとなります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の挿入ロジックは rd_indam（IndexAmRoutine）経由で呼び出される各AMの実装に依存します。
  • 挿入処理に伴うキャッシュの初期化やクリーンアップが必要な場合、呼び出し元で適切に管理する必要があります（例: constraint.c での言及）。
