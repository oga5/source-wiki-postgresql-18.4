ecpg_register_prepared_stmt

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:58

目的

ECPGにおいて、指定されたステートメントをプリペアドステートメントとして接続オブジェクトのリストに登録します。

シグネチャまたは形状

bool ecpg_register_prepared_stmt(struct statement *stmt)

入力と出力

  • **入力**: stmt (登録対象のステートメント構造体へのポインタ)
  • **出力**: 登録に成功した場合は true、メモリ確保失敗や既存ステートメントの解放失敗時は false を返します。

前提条件と不変条件

  • stmt->connection が有効な接続オブジェクトを指している必要があります。
  • 同名のプリペアドステートメントが既に存在する場合、本関数内で deallocate_one を呼び出し、既存の登録を破棄してから新しい登録を行います。

アルゴリズムまたは振る舞い

1. ecpg_find_prepared_statement を使用し、同名のステートメントが既に存在するか確認します。存在する場合は deallocate_one で解放を試みます。 2. prepared_statement 構造体および内部の statement 構造体用にメモリを確保します。 3. 入力された stmt の情報を新しい prepared_statement にコピーし、prepared フラグを true に設定します。 4. 接続オブジェクト (con->prep_stmts) が保持するプリペアドステートメントのリストの先頭に、新しいエントリを挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/execute.c
  • **呼び出し先**:
  • ecpg_find_prepared_statement
  • deallocate_one
  • ecpg_alloc / ecpg_free
  • ecpg_strdup

関連データ構造

  • struct statement: 実行されるSQL文の情報を保持。
  • struct prepared_statement: プリペアドステートメントの管理用構造体。
  • struct connection: データベース接続状態を保持。

失敗モードと境界条件

  • ecpg_alloc によるメモリ確保が失敗した場合、false を返します。
  • 既存の同名ステートメントの解放 (deallocate_one) が失敗した場合、登録処理を中断し false を返します。

保守担当者向けメモ

  • 本関数は接続ごとのリスト管理を行っています。リストの操作は先頭挿入 (con->prep_stmts = this) のみであるため、順序は最新の登録が先頭になります。
  • ecpg_alloc で確保したメモリは deallocate_one 等で適切に解放される必要があるため、メモリリークには注意してください。
