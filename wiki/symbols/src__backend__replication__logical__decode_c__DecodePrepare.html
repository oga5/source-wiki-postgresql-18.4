DecodePrepare

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/decode.c:762

目的

2フェーズコミット（2PC）における PREPARE TRANSACTION WAL レコードを解析し、トランザクションの準備状態を ReorderBuffer に登録・反映する。

シグネチャまたは形状

static void DecodePrepare(LogicalDecodingContext *ctx, XLogRecordBuffer *buf, xl_xact_parsed_prepare *parsed)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • buf: 解析対象の WAL レコードバッファ。
  • parsed: PREPARE レコードから抽出されたトランザクション情報（XID、GID、サブトランザクションリスト等）。
  • **出力**: なし（副作用として ReorderBuffer および ctx の状態を更新）。

前提条件と不変条件

  • ReorderBufferRememberPrepareInfo が成功すること。
  • スナップショットが SNAPBUILD_CONSISTENT 状態に達していること（未達の場合はスキップされる）。

アルゴリズムまたは振る舞い

1. **準備情報の保存**: ReorderBufferRememberPrepareInfo を呼び出し、後続の COMMIT PREPARED で使用するための情報を記録する。失敗した場合は即座に終了する。 2. **整合性チェック**: スナップショットが整合状態にない場合、ReorderBufferSkipPrepare を呼び出し、当該トランザクションをスキップする。 3. **フィルタリング**: DecodeTXNNeedSkip により、対象外のデータベースやオリジンのトランザクションであればスキップし、キャッシュ無効化処理のみを行う。 4. **サブトランザクションの登録**: parsed->subxacts に含まれる全てのサブトランザクションを ReorderBufferCommitChild で登録する。 5. **準備の実行**: ReorderBufferPrepare を呼び出し、トランザクションを準備状態へ移行させる。 6. **統計更新**: UpdateDecodingStats を呼び出し、デコーディング統計を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c 内のレコード処理ループ（LogicalDecodingProcessRecord からのディスパッチ）。
  • **呼び出し先**:
  • ReorderBufferRememberPrepareInfo, ReorderBufferSkipPrepare, ReorderBufferInvalidate, ReorderBufferCommitChild, ReorderBufferPrepare
  • SnapBuildCurrentState, DecodeTXNNeedSkip, UpdateDecodingStats
  • XLogRecGetOrigin

関連データ構造

  • LogicalDecodingContext: デコーディング全体のコンテキスト。
  • XLogRecordBuffer: WAL レコードの物理データ。
  • xl_xact_parsed_prepare: PREPARE レコードの解析済み構造体。
  • ReorderBuffer: 変更を蓄積するバッファ。
  • SnapBuild: スナップショット構築状態。

失敗モードと境界条件

  • **整合性未達**: スナップショットが SNAPBUILD_CONSISTENT 未満の場合、処理はスキップされる。
  • **フィルタリング**: DecodeTXNNeedSkip が真を返す場合、トランザクションはスキップされ、キャッシュ無効化のみが実行される。
  • **情報保存失敗**: ReorderBufferRememberPrepareInfo が失敗した場合、処理は中断される。

保守担当者向けメモ

  • 本関数は COMMIT PREPARED 処理（ReorderBufferFinishPrepared）と密接に関連している。
  • DecodeCommit と異なり、コミット前であるため ReorderBufferForget を呼び出してはならない（restart_lsn の計算に影響するため）。
  • 統計情報の更新タイミングは、デコーディングの中断時に情報を失わないよう、準備・コミット・アボートの各フェーズで行われる。
