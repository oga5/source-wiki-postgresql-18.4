StartupPacketTimeoutHandler

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/backend_startup.c:975

目的

バックエンドプロセスの起動時におけるスタートアップパケット受信処理が、規定の制限時間を超過した場合にプロセスを即座に終了させるためのシグナルハンドラです。

シグネチャまたは形状

static void StartupPacketTimeoutHandler(void);

入力と出力

  • 入力: なし
  • 出力: なし（関数は戻り値を返さず、プロセスを終了させます）

前提条件と不変条件

  • 本関数は RegisterTimeout(STARTUP_PACKET_TIMEOUT, ...) を介してタイムアウトイベントに登録されることを前提としています。
  • 共有メモリへのアクセスが許可されていない初期化フェーズで呼び出されるため、安全な終了手段として _exit(1) を使用します。

アルゴリズムまたは振る舞い

1. 呼び出された際、即座に _exit(1) を実行します。 2. これにより、接続を試みるクライアントがスタートアップパケットの送信を遅延または停止させた場合に、バックエンドプロセスがゾンビ化したりリソースを占有し続けたりすることを防ぎます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/tcop/backend_startup.c: RegisterTimeout によりタイムアウト発生時に呼び出されます。
  • src/backend/libpq/be-secure-openssl.c: タイムアウト発生時の直接的な言及があります。
  • 呼び出し先:
  • _exit(2) (システムコール): プロセスを即時終了させます。

関連データ構造

  • STARTUP_PACKET_TIMEOUT: タイムアウトの種類を識別する定数。

失敗モードと境界条件

  • 本関数自体が失敗することはありません。_exit(1) はプロセスを強制終了させるため、呼び出し後の制御フローは存在しません。

保守担当者向けメモ

  • 本関数は proc_exit ではなく _exit を使用しています。これは、初期化中の不完全な状態でのクリーンアップ処理（共有メモリの解放など）が予期せぬ副作用を引き起こすのを避けるためです。この設計方針を変更する場合は注意してください。
