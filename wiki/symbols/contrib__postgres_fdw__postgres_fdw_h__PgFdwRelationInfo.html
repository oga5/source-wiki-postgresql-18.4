PgFdwRelationInfo

定義

  • **種別**: struct
  • **定義箇所**: contrib/postgres_fdw/postgres_fdw.h:30

目的

PostgreSQL の外部データラッパー（postgres_fdw）において、プランナがリモートテーブルに対する最適化を行う際に必要な情報を一元管理するための構造体です。クエリのプッシュダウン可否、コスト見積もり、およびリモート実行に必要なメタデータを保持します。

シグネチャまたは形状

typedef struct PgFdwRelationInfo {
    bool        pushdown_safe;
    List       *remote_conds;
    List       *local_conds;
    List       *final_remote_exprs;
    Bitmapset  *attrs_used;
    bool        qp_is_pushdown_safe;
    QualCost    local_conds_cost;
    Selectivity local_conds_sel;
    Selectivity joinclause_sel;
    double      rows;
    int         width;
    int         disabled_nodes;
    Cost        startup_cost;
    Cost        total_cost;
    double      retrieved_rows;
    Cost        rel_startup_cost;
    Cost        rel_total_cost;
    bool        use_remote_estimate;
    Cost        fdw_startup_cost;
    Cost        fdw_tuple_cost;
    List       *shippable_extensions;
    bool        async_capable;
    ForeignTable *table;
    ForeignServer *server;
    UserMapping *user;
    int         fetch_size;
    char       *relation_name;
    RelOptInfo *outerrel;
    RelOptInfo *innerrel;
    JoinType    jointype;
    List       *joinclauses;
    UpperRelationKind stage;
    List       *grouped_tlist;
    bool        make_outerrel_subquery;
    bool        make_innerrel_subquery;
    Relids      lower_subquery_rels;
    Relids      hidden_subquery_rels;
    int         relation_index;
} PgFdwRelationInfo;

入力と出力

本構造体は、プランナが作成する RelOptInfo の fdw_private フィールドに格納されます。プランニングの過程で各フィールドが初期化・更新され、最終的に deparse.c 等のモジュールがこの情報を読み取ってリモート SQL を生成します。

前提条件と不変条件

  • pushdown_safe は、そのリレーションがリモートで処理可能かを示す重要なフラグであり、単純な外部スキャンでは常に true です。
  • remote_conds および local_conds に格納されるリストは、効率的なコスト見積もりのために RestrictInfo ラッパーを含む必要があります。

アルゴリズムまたは振る舞い

1. **プランニング**: プランナが本構造体を参照し、リモートで実行可能な条件式を特定します。 2. **デパース**: deparse.c が本構造体の情報を元に、ローカルのクエリツリーをリモートサーバー用の SQL 文字列へ変換します。 3. **実行**: connection.c が本構造体の接続情報（server, user 等）を用いてリモート接続を管理し、クエリを送信します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/deparse.c 内の複数の関数（deparseSelectStmt, deparseDirectUpdateSql 等）が、RelOptInfo から fdw_private をキャストして本構造体を取得しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • RelOptInfo: プランナが管理するリレーション情報。本構造体は fdw_private としてここに付加されます。
  • ForeignTable, ForeignServer, UserMapping: リモート接続に必要なカタログ情報。
  • PgFdwConnState: リモート接続のライフサイクルを管理する構造体。

失敗モードと境界条件

  • use_remote_estimate が有効な場合、user フィールドが設定されている必要があります。
  • join や upper リレーションの場合、relation_name は含まれるベーステーブルや結合・集約タイプを識別するために使用されます。

保守担当者向けメモ

  • 本構造体は非常に巨大ですが、最適化の各段階で必要な情報を一元管理するために設計されています。
  • 新機能を追加する際は、既存のフィールドで要件を満たせるか確認し、不足している場合は本構造体への追加を検討してください。
  • pushdown_safe フラグの管理は、クエリのプッシュダウン判断において最も重要です。
