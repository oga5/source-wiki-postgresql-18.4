ShmemAllocNoError

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:171

目的

共有メモリから指定されたサイズのメモリ領域を割り当てます。割り当て失敗時にエラーを発生させず、呼び出し元に制御を戻すためのラッパー関数です。

シグネチャまたは形状

void *ShmemAllocNoError(Size size);

入力と出力

  • **入力**: size (Size型) - 割り当てるメモリのバイト数。
  • **出力**: 割り当てられたメモリ領域へのポインタ。失敗した場合は NULL を返します。

前提条件と不変条件

  • 共有メモリ管理システムが初期化済みである必要があります。
  • 割り当てられる領域は、システムが規定する境界（キャッシュライン境界等）にアライメントされます。

アルゴリズムまたは振る舞い

1. 内部で ShmemAllocRaw(size, &allocated_size) を呼び出します。 2. ShmemAllocRaw が返すポインタをそのまま戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し先**: ShmemAllocRaw
  • **呼び出し元**:
  • src/backend/storage/ipc/shmem.c:349 (構造体フィールドへの関数ポインタ代入として参照)
  • その他、共有メモリの動的割り当てを必要とするモジュール（詳細はソースコード全体での検索が必要）。

関連データ構造

  • Size: メモリサイズを表す型。

失敗モードと境界条件

  • 共有メモリの空き容量が不足している場合、ShmemAllocRaw の仕様に従い NULL を返します。
  • 割り当てサイズが 0 の場合の挙動は ShmemAllocRaw の実装に依存します。

保守担当者向けメモ

  • 本関数は ShmemAllocRaw の単なるラッパーです。エラーハンドリングのロジックを変更する場合は、呼び出し先である ShmemAllocRaw の仕様を確認してください。
  • 共有メモリは一度割り当てると解放できないため、本関数を呼び出す際はメモリリークが発生しないよう注意が必要です。
