ExecutorRun

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execMain.c:296

目的

クエリ実行エンジンにおけるメインの実行フェーズを制御します。プランツリーの走査を開始し、指定された方向と回数に従ってタプルの処理を行います。

シグネチャまたは形状

void ExecutorRun(QueryDesc *queryDesc, ScanDirection direction, uint64 count)

入力と出力

  • **入力**:
  • queryDesc: 実行対象のクエリ情報（プランツリーや実行状態を含む）。
  • direction: スキャン方向（ForwardScanDirection 等）。
  • count: 処理するタプル数（0 は無制限を意味する）。
  • **出力**: なし（副作用としてクエリ結果が DestReceiver に送出される）。

前提条件と不変条件

  • ExecutorStart が呼び出され、queryDesc が初期化済みであること。
  • 実行中に ExecutorRun_hook が設定されている場合、標準処理の代わりにフックが優先的に実行されます。

アルゴリズムまたは振る舞い

1. ExecutorRun_hook が定義されているか確認します。 2. フックが存在する場合はそれを呼び出し、存在しない場合は standard_ExecutorRun を呼び出します。 3. standard_ExecutorRun は内部で ExecutePlan を呼び出し、プランノードを再帰的に走査してタプルを処理します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyto.c
  • src/backend/commands/createas.c
  • src/backend/commands/explain.c
  • src/backend/commands/extension.c
  • src/backend/commands/matview.c
  • src/backend/commands/portalcmds.c
  • **呼び出し先**:
  • ExecutorRun_hook (存在する場合)
  • standard_ExecutorRun

関連データ構造

  • QueryDesc: 実行に必要なプランや状態を保持する構造体。
  • ScanDirection: スキャン方向を定義する列挙型。

失敗モードと境界条件

  • count に 0 を指定した場合、クエリが完了するまで実行が継続されます。
  • フックが設定されている場合、その実装に依存した失敗モードが発生する可能性があります。

保守担当者向けメモ

  • 本関数は実行エンジンの主要なエントリポイントの一つです。拡張機能（auto_explain や pg_stat_statements など）がフックを通じて実行時間を計測したり、ネスト深度を追跡したりするために利用されています。
  • 処理の詳細は standard_ExecutorRun に委譲されているため、標準的な実行フローを追う場合はそちらを参照してください。
