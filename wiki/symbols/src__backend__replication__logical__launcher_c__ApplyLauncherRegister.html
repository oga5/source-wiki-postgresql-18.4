ApplyLauncherRegister

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/launcher.c:927

目的

論理レプリケーションのランチャープロセスをバックグラウンドワーカーとしてPostgreSQLのポストマスターに登録する。

シグネチャまたは形状

void ApplyLauncherRegister(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてポストマスターのバックグラウンドワーカーリストに登録）

前提条件と不変条件

  • max_logical_replication_workers が 0 より大きいこと。
  • IsBinaryUpgrade が偽であること（バイナリアップグレード中は登録されない）。
  • データベースのリカバリが完了していること（BgWorkerStart_RecoveryFinished）。

アルゴリズムまたは振る舞い

1. max_logical_replication_workers および IsBinaryUpgrade をチェックし、条件を満たさない場合は早期リターンする。 2. BackgroundWorker 構造体をゼロクリアし、以下の設定を行う：

  • bgw_flags: BGWORKER_SHMEM_ACCESS および BGWORKER_BACKEND_DATABASE_CONNECTION を設定。
  • bgw_start_time: BgWorkerStart_RecoveryFinished。
  • bgw_library_name: "postgres"。
  • bgw_function_name: "ApplyLauncherMain"。
  • bgw_name / bgw_type: "logical replication launcher"。
  • bgw_restart_time: 5秒。

    • 3. RegisterBackgroundWorker(&bgw) を呼び出し、ポストマスターに登録する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/postmaster.c（直接の証拠あり）
  • 呼び出し先: RegisterBackgroundWorker（直接の証拠あり）

関連データ構造

  • BackgroundWorker: バックグラウンドワーカーの起動設定を保持する構造体。

失敗モードと境界条件

  • max_logical_replication_workers が 0 の場合、ランチャーは起動しない。
  • バイナリアップグレード実行中は、レプリケーションオリジンの競合を防ぐためランチャーは登録されない。

保守担当者向けメモ

  • 本関数はポストマスターの起動プロセスの一部として呼び出されます。
  • bgw_function_name に指定されている ApplyLauncherMain が、実際にランチャーのメインループを実行する関数です。
