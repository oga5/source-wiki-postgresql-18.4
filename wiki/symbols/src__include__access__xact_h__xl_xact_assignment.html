xl_xact_assignment

定義

  • 種別: struct
  • 定義箇所: src/include/access/xact.h:218

目的

トランザクションID（XID）の割り当て情報をWAL（Write-Ahead Logging）に記録するための構造体です。主にトップレベルのトランザクションIDと、それに紐づくサブトランザクションIDのリストを保持します。

シグネチャまたは形状

typedef struct xl_xact_assignment
{
    TransactionId xtop;
    int           nsubxacts;
    TransactionId xsub[FLEXIBLE_ARRAY_MEMBER];
} xl_xact_assignment;

入力と出力

  • **入力**: WALレコード生成時に、現在のトップレベルXIDおよびサブトランザクションIDの配列が本構造体に格納されます。
  • **出力**: WALリカバリ時、XLogRecGetData を通じて本構造体が読み出され、トランザクション状態の再現に使用されます。

前提条件と不変条件

  • xsub フィールドは FLEXIBLE_ARRAY_MEMBER として定義されており、可変長データを含みます。
  • 構造体の最小サイズは MinSizeOfXactAssignment（offsetof(xl_xact_assignment, xsub)）で定義されます。

アルゴリズムまたは振る舞い

1. トランザクション処理中にサブトランザクションが割り当てられた際、その情報を記録するために使用されます。 2. xact_desc_assignment 関数により、WALのデバッグ出力や解析が行われます。 3. リカバリ時には、WALレコードからキャストされた xl_xact_assignment ポインタを通じて、記録されたXID群が参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c: トランザクション管理ロジック内でレコードの初期化に使用。
  • **呼び出し先**:
  • src/backend/access/rmgrdesc/xactdesc.c: xact_desc_assignment 関数が本構造体を引数として受け取り、内容を記述します。

関連データ構造

  • TransactionId: トランザクション識別子。
  • MinSizeOfXactAssignment: 本構造体の可変長部分を除いた最小サイズを定義するマクロ。

失敗モードと境界条件

  • 可変長配列を含むため、直接的なメモリコピーやサイズ計算を行う際は MinSizeOfXactAssignment を考慮する必要があります。
  • 構造体へのアクセスは、必ず XLogRecGetData 等で取得した有効なWALレコードポインタを経由して行う必要があります。

保守担当者向けメモ

  • 本構造体はWALレコードの一部としてシリアライズされるため、フィールドの追加や変更を行う際は、xactdesc.c 側の解析ロジックも併せて更新してください。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリレイアウトには十分注意してください。
