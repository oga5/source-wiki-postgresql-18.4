AlterOpFamilyAdd

定義

  • 種別: function
  • 定義箇所: src/backend/commands/opclasscmds.c:880

目的

ALTER OPERATOR FAMILY ... ADD コマンドの実行時、演算子やサポート関数を既存の演算子ファミリーに追加する。

シグネチャまたは形状

static void AlterOpFamilyAdd(AlterOpFamilyStmt *stmt, Oid amoid, Oid opfamilyoid,
                             int maxOpNumber, int maxProcNumber, int optsProcNumber,
                             List *items)

入力と出力

  • **入力**:
  • stmt: 解析された ALTER OPERATOR FAMILY 文の構造体。
  • amoid: 対象アクセスメソッドのOID。
  • opfamilyoid: 対象演算子ファミリーのOID。
  • maxOpNumber, maxProcNumber, optsProcNumber: AM固有の制約値。
  • items: 追加対象の演算子や関数の定義リスト。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • 呼び出し元でスーパーユーザー権限のチェックが完了していること。
  • items 内の各項目は、指定されたAMの許容範囲（maxOpNumber 等）内であること。
  • STORAGE 句の指定は不可（エラーとなる）。

アルゴリズムまたは振る舞い

1. items リストを走査し、各項目が演算子か関数かを判定する。 2. 演算子の場合は LookupOperWithArgs、関数の場合は LookupFuncWithArgs を用いてOIDを解決する。 3. OpFamilyMember 構造体を生成し、ref_is_hard = false（ソフト依存）として情報を格納する。 4. assignOperTypes または assignProcTypes を呼び出し、型情報の検証と割り当てを行う。 5. AM固有の調整が必要な場合、amroutine->amadjustmembers を実行する。 6. storeOperators および storeProcedures を呼び出し、pg_amop および pg_amproc カタログへ永続化する。 7. EventTriggerCollectAlterOpFam を呼び出し、DDLイベントをトリガーする。

呼び出し元と呼び出し先

  • **呼び出し元**: AlterOpFamily（src/backend/commands/opclasscmds.c 内）。
  • **呼び出し先**: GetIndexAmRoutineByAmId, LookupOperWithArgs, LookupFuncWithArgs, get_opfamily_oid, assignOperTypes, assignProcTypes, addFamilyMember, storeOperators, storeProcedures, EventTriggerCollectAlterOpFam。

関連データ構造

  • AlterOpFamilyStmt: コマンド解析結果。
  • OpFamilyMember: 追加対象の演算子や関数の情報を保持する内部構造体。
  • IndexAmRoutine: アクセスメソッドのAPIルーチン。

失敗モードと境界条件

  • **エラー**:
  • 演算子番号や関数番号が範囲外の場合。
  • 演算子引数型が指定されていない場合。
  • STORAGE 句が含まれている場合。
  • 演算子や関数が解決できない場合。

保守担当者向けメモ

  • 本関数は「ソフト」な依存関係を作成する。これは CREATE OPERATOR CLASS で作成される「ハード」な依存関係とは異なる。
  • NOT_USED ブロック内の所有権チェックは、スーパーユーザーチェックが上位で行われているため省略されている。
  • 新しいAMを追加する際は、amadjustmembers によるAM固有の検証ロジックが適切に実装されているか確認すること。
