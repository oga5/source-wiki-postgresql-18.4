bbsink_progress_end_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_progress.c:113

目的

バックアップにおけるアーカイブ（テーブルスペース単位）の終了を処理し、進捗統計の更新と状態管理を行う。

シグネチャまたは形状

static void bbsink_progress_end_archive(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - 現在の進捗追跡用シンクへのポインタ。
  • **出力**: なし（副作用として pgstat 統計および bbsink_state を更新）。

前提条件と不変条件

  • sink->bbs_state が有効であること。
  • アーカイブの終了はテーブルスペースの終了と等価であると見なされる。
  • WALが含まれる場合、最後のテーブルスペース完了が最後のアーカイブ完了より先に行われる可能性があるため、tablespace_num が総数を超えないようガードされている。

アルゴリズムまたは振る舞い

1. 現在のテーブルスペース番号が総数未満である場合、pgstat_progress_update_param を呼び出し、PROGRESS_BASEBACKUP_TBLSPC_STREAMED をインクリメントして統計を更新する。 2. bbsink_forward_end_archive(sink) を呼び出し、後続のシンクへ処理を委譲する。 3. sink->bbs_state->tablespace_num をインクリメントし、現在のテーブルスペース位置を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_progress_ops 構造体（src/backend/backup/basebackup_progress.c:46）を介して、バックアップのデータ転送パイプラインから呼び出される。
  • **呼び出し先**: bbsink_forward_end_archive（後続のシンクへの転送）、pgstat_progress_update_param（統計更新）。

関連データ構造

  • bbsink: バックアップのデータ転送パイプラインを構成する構造体。
  • bbsink_state: 転送状態（テーブルスペース番号やリスト）を保持する共有オブジェクト。

失敗モードと境界条件

  • WAL転送が含まれるバックアップにおいて、テーブルスペースのストリーム数が総数を超えないようガード条件が適用されている。

保守担当者向けメモ

  • 本関数は bbsink チェーンの最外層で実行されることを想定している。
  • bbsink_state は共有オブジェクトであるため、この関数での tablespace_num 更新はバックアップ全体の進捗状態に影響を与える。
