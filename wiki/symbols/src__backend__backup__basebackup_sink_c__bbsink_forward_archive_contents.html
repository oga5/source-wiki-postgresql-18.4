bbsink_forward_archive_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_sink.c:53

目的

bbsink（バックアップシンク）チェーンにおいて、アーカイブコンテンツの処理を現在のシンクから後続のシンク（bbs_next）へ転送する。

シグネチャまたは形状

void bbsink_forward_archive_contents(bbsink *sink, size_t len)

入力と出力

  • sink: 現在の bbsink オブジェクトへのポインタ。
  • len: 転送するデータのバイト長。
  • 戻り値: なし。

前提条件と不変条件

  • sink->bbs_next が NULL であってはならない。
  • 現在のシンクのバッファ（bbs_buffer および bbs_buffer_length）が、後続のシンクのバッファと完全に一致している必要がある。

アルゴリズムまたは振る舞い

1. sink->bbs_next が存在することをアサーションで検証する。 2. 現在のシンクと後続のシンク間でバッファおよびバッファ長が共有されていることをアサーションで検証する。 3. bbsink_archive_contents を呼び出し、後続のシンクへ処理を委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/basebackup_to_shell/basebackup_to_shell.c
  • src/backend/backup/basebackup_progress.c
  • src/backend/backup/basebackup_server.c
  • src/backend/backup/basebackup_throttle.c
  • **呼び出し先**:
  • bbsink_archive_contents（bbs_next に対して実行）

関連データ構造

  • bbsink: バックアップデータの転送・処理を行うための抽象インターフェース構造体。

失敗モードと境界条件

  • sink->bbs_next が NULL の場合、アサーション失敗により異常終了する。
  • バッファの不整合（bbs_buffer または bbs_buffer_length が一致しない）がある場合、アサーション失敗により異常終了する。

保守担当者向けメモ

  • 本関数は、バッファのコピーを発生させずにデータを次段へ渡すためのヘルパーである。
  • 独自の bbsink 実装において、転送のみを行うコールバックとしてそのまま割り当てることが可能である。
