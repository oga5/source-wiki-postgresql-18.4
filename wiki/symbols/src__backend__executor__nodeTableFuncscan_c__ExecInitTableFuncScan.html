ExecInitTableFuncScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTableFuncscan.c:110

目的

TableFuncScan ノードの実行時状態（TableFuncScanState）を構築し、XMLTABLE や JSON_TABLE 実行のための初期化を行う。

シグネチャまたは形状

TableFuncScanState *ExecInitTableFuncScan(TableFuncScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 実行対象の TableFuncScan プランノード。
  • estate: 実行状態を保持する EState。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの TableFuncScanState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_MARK を含めてはならない。
  • TableFuncScan ノードは子プラン（outerPlan / innerPlan）を持たないこと。
  • 現在サポートされている関数型は TFT_XMLTABLE または TFT_JSON_TABLE のみ。

アルゴリズムまたは振る舞い

1. TableFuncScanState を割り当て、ExecProcNode 関数ポインタを ExecTableFuncScan に設定する。 2. 式評価コンテキスト（ExprContext）を作成し、スキャンタプルスロットを初期化する。 3. BuildDescFromLists を使用して出力タプルの記述子（TupleDesc）を構築する。 4. 射影（Projection）およびフィルタ（Qual）の式ツリーを初期化する。 5. 関数型に応じて XmlTableRoutine または JsonbTableRoutine を選択し、routine フィールドに格納する。 6. メモリ管理用の perTableCxt を作成し、各式（docexpr, rowexpr, colexprs 等）を初期化する。 7. 各列の入力関数（in_functions）と型情報（typioparams）を getTypeInputInfo を用いて取得・設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内のノード初期化処理から呼び出される。
  • **呼び出し先**: ExecAssignExprContext, BuildDescFromLists, ExecInitScanTupleSlot, ExecInitQual, AllocSetContextCreate, ExecInitExpr, getTypeInputInfo, fmgr_info 等。

関連データ構造

  • TableFuncScanState: 本ノードの実行時状態を保持する主要構造体。
  • TableFuncScan: プランノード。
  • TableFunc: テーブル関数の定義情報。
  • FmgrInfo: 各列の入力関数情報を保持。

失敗モードと境界条件

  • サポート外のフラグが渡された場合、Assert により停止する。
  • 子プランが存在する場合、Assert により停止する。
  • メモリ不足時には AllocSetContextCreate や palloc がエラーを発生させる。

保守担当者向けメモ

  • 本ノードはマテリアライズ型スキャンであるため、perTableCxt のメモリ管理が重要。
  • opaque ポインタは実行時に解析エンジンが使用するため、初期化時点では NULL に設定される。
  • tupdesc の属性数に基づいて in_functions 配列が動的に割り当てられるため、メモリリークに注意すること。
