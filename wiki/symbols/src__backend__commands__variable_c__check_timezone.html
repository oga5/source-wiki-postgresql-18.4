check_timezone

定義

  • 種別: function
  • 定義箇所: src/backend/commands/variable.c:260

目的

GUC（Grand Unified Configuration）サブシステムにおいて、TimeZone パラメータに設定された値が有効なタイムゾーン名、数値（時間オフセット）、または INTERVAL 形式であるかを検証し、適用可能な形式へ変換します。

シグネチャまたは形状

bool check_timezone(char **newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定しようとしているタイムゾーン文字列へのポインタ。
  • source: 設定のソース（GucSource）。
  • **出力**:
  • 戻り値: 検証に成功した場合は true、失敗した場合は false。
  • extra: 検証成功時、適用フェーズ（assign_timezone）で使用される pg_tz * 構造体へのポインタが格納されます。

前提条件と不変条件

  • newval が指す文字列は、タイムゾーン名、数値（時間単位）、または INTERVAL 形式のいずれかである必要があります。
  • 閏秒を含むタイムゾーンはサポートされません。
  • INTERVAL 形式の場合、月や日の単位を含むことはできません。

アルゴリズムまたは振る舞い

1. **形式判定**: 文字列が INTERVAL で始まるかを確認します。

  • INTERVAL の場合: SQL標準に従い解析し、月・日の指定がないことを確認後、オフセットを計算して pg_tzset_offset で pg_tz を取得します。
  • 数値の場合: strtod で解析し、時間オフセットとして pg_tzset_offset を呼び出します。
  • それ以外の場合: タイムゾーン名とみなし pg_tzset で解決を試みます。

    • 2. **検証**: 解決された pg_tz が存在し、かつ pg_tz_acceptable（閏秒を含まないこと）を満たすか確認します。 3. **データ保存**: 成功時、guc_malloc を用いて pg_tz * を extra に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: GUCサブシステム（src/backend/utils/misc/guc_tables.c 等のフック経由）。
  • **呼び出し先**: pg_tzset, pg_tzset_offset, pg_tz_acceptable, guc_malloc, DatumGetIntervalP 等。

関連データ構造

  • pg_tz: タイムゾーン情報を保持する内部構造体。
  • Interval: INTERVAL 形式解析時に使用される構造体。

失敗モードと境界条件

  • **解析失敗**: 不正な形式や未知のタイムゾーン名が指定された場合、false を返します。
  • **制約違反**: 閏秒を含むタイムゾーンや、INTERVAL で月・日を指定した場合は GUC_check_errdetail を通じてエラー詳細を報告し、false を返します。
  • **メモリ不足**: guc_malloc が失敗した場合、false を返します。

保守担当者向けメモ

  • 本関数で割り当てた extra のメモリは、GUCサブシステムによって管理されます。
  • INTERVAL 解析時に interval_in を直接呼び出しているため、将来的なインターフェース変更には注意が必要です。
