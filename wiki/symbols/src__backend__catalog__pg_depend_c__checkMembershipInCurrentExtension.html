checkMembershipInCurrentExtension

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:259

目的

現在実行中の拡張機能（Extension）の作成プロセスにおいて、指定されたオブジェクトがその拡張機能のメンバーであるかを確認し、メンバーでない場合にエラーを報告する。

シグネチャまたは形状

void checkMembershipInCurrentExtension(const ObjectAddress *object)

入力と出力

  • **入力**: object (ObjectAddressへのポインタ) - チェック対象のデータベースオブジェクト。
  • **出力**: なし。正常時は何も返さず終了する。

前提条件と不変条件

  • object->objectSubId == 0 であること（拡張機能のメンバーはオブジェクト全体である必要があるため）。
  • creating_extension が真である環境下で呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. objectSubId が 0 であることをアサートする。 2. creating_extension が真の場合、getExtensionOfObject を呼び出し、対象オブジェクトが現在どの拡張機能に属しているか（oldext）を特定する。 3. oldext が CurrentExtensionObject（現在作成中の拡張機能）と一致すれば、正常として処理を終了する。 4. 一致しない場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーを発生させる。エラーメッセージには、対象オブジェクトの記述と拡張機能名が含まれる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_collation.c
  • src/backend/commands/createas.c
  • src/backend/commands/foreigncmds.c
  • src/backend/commands/schemacmds.c
  • src/backend/commands/sequence.c
  • src/backend/parser/parse_utilcmd.c
  • **呼び出し先**:
  • getExtensionOfObject
  • getObjectDescription
  • get_extension_name
  • ereport

関連データ構造

  • ObjectAddress: オブジェクトを識別するための構造体（classId, objectId, objectSubId を保持）。
  • CurrentExtensionObject: 現在処理中の拡張機能の OID を保持するグローバル変数。

失敗モードと境界条件

  • 対象オブジェクトが現在作成中の拡張機能に属していない場合、ERROR レベルで処理が中断される。
  • objectSubId が 0 でない場合、アサーション違反によりプロセスが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は recordDependencyOnCurrentExtension と同様の条件をチェックするが、エラーメッセージの文言を区別するために独立して実装されている。
  • 拡張機能の作成中に CREATE ... IF NOT EXISTS を使用して既存オブジェクトをスキップする際、そのオブジェクトが既に当該拡張機能によって所有されていることを保証するために使用される。
