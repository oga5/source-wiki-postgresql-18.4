set_dl_error

定義

  • 種別: function
  • 定義箇所: src/port/win32dlopen.c:20

目的

Windows環境における動的リンク操作で発生した直近のエラーコードをシステムから取得し、人間が読める形式の文字列として静的バッファに格納する。

シグネチャまたは形状

static void set_dl_error(void)

入力と出力

  • 入力: なし（内部で GetLastError() を呼び出す）
  • 出力: なし（副作用として静的変数 last_dyn_error を更新する）

前提条件と不変条件

  • Windows APIの GetLastError() が有効なエラー状態を保持していること。
  • last_dyn_error は静的バッファであるため、本関数はスレッドセーフではない。

アルゴリズムまたは振る舞い

1. GetLastError() を呼び出し、現在のスレッドの直近のエラーコードを取得する。 2. FormatMessage を使用して、システムからエラーコードに対応するメッセージ文字列を取得し、last_dyn_error に書き込む。 3. FormatMessage が失敗した場合（戻り値が0）、フォールバックとして snprintf を使用し、エラーコードを含む「unknown error %lu」という文字列を last_dyn_error に書き込む。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/win32dlopen.c 内の dlopen、dlsym、dlclose 関数から、操作失敗時に呼び出される。
  • 呼び出し先: Windows API (GetLastError, FormatMessage)。

関連データ構造

  • last_dyn_error: エラーメッセージを保持する静的バッファ（本ファイル内で定義）。

失敗モードと境界条件

  • FormatMessage によるメッセージ取得に失敗した場合、数値のみの汎用エラーメッセージに切り替わる。
  • バッファサイズ制限（sizeof(last_dyn_error) - 1）により、長いエラーメッセージは切り詰められる可能性がある。

保守担当者向けメモ

  • 本関数は dlerror() が返すエラーメッセージのソースを更新する役割を持つ。
  • スレッドセーフではないため、マルチスレッド環境で動的ロードを行う場合は注意が必要である。
  • Windowsのシステムエラーダイアログを抑制する設定と併用されることを前提としている。
