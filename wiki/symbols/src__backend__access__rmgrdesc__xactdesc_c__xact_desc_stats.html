xact_desc_stats

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xactdesc.c:311

目的

トランザクションに関連する統計情報（削除されたオブジェクト等）をWALデコード用の文字列バッファへ整形して追加します。

シグネチャまたは形状

static void xact_desc_stats(StringInfo buf, const char *label, int ndropped, xl_xact_stats_item *dropped_stats)

入力と出力

  • buf: 出力先の StringInfo バッファ。
  • label: 統計情報の前に付与するラベル文字列。
  • ndropped: 統計アイテムの数。
  • dropped_stats: 統計情報の配列（xl_xact_stats_item 型）。
  • 出力: なし（buf を直接更新）。

前提条件と不変条件

  • ndropped が 0 より大きい場合のみ、バッファへの書き込みが行われます。
  • dropped_stats 配列は少なくとも ndropped 個の要素を持つ必要があります。

アルゴリズムまたは振る舞い

1. ndropped > 0 の場合、label を含むヘッダーを buf に追加します。 2. dropped_stats 配列をループ処理します。 3. 各アイテムの objid_hi と objid_lo を結合して 64 ビットの objid を生成します。 4. kind（種別）、dboid（データベースOID）、objid を buf に追記します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/rmgrdesc/xactdesc.c 内の各デコード関数（xact_desc から呼び出される処理の一部）から、parsed.nstats や parsed.abortstats 等を引数として呼び出されます。
  • 呼び出し先:
  • appendStringInfo（文字列バッファへの書き込み）。

関連データ構造

  • StringInfo: PostgreSQLの動的文字列バッファ。
  • xl_xact_stats_item: 統計情報を保持する構造体。

失敗モードと境界条件

  • ndropped が 0 の場合、関数は何も行わず即座に復帰します。
  • objid の生成において、objid_hi と objid_lo のビット演算による結合が行われます。

保守担当者向けメモ

  • 本関数は pg_waldump 等のツールでWALを可読形式にするために使用されます。
  • xl_xact_stats_item 構造体の定義変更時には、本関数でのパースロジックも整合性を確認してください。
