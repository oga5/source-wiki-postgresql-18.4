macaddr8

定義

  • 種別: struct
  • 定義箇所: src/include/utils/inet.h:107

目的

PostgreSQLにおいて、8バイトのMACアドレスを格納するための固定長データ構造を提供します。

シグネチャまたは形状

typedef struct macaddr8
{
	unsigned char a;
	unsigned char b;
	unsigned char c;
	unsigned char d;
	unsigned char e;
	unsigned char f;
	unsigned char g;
	unsigned char h;
} macaddr8;

入力と出力

  • 本構造体は固定長（8バイト）のデータ型です。
  • PostgreSQLの関数マネージャ（fmgr）を介して、Datum 型との間でポインタとして変換されます。

前提条件と不変条件

  • 8個の unsigned char フィールドで構成される固定長データ型であり、inet 型のような可変長ヘッダー（varlena）は持ちません。

アルゴリズムまたは振る舞い

  • ネットワークアドレスをデータベースのタプルとして格納・処理するために使用されます。
  • src/backend/access/brin/brin_minmax_multi.c 等において、各バイトを個別に扱うことで距離計算や比較操作が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/btree_gin/btree_gin.c
  • contrib/btree_gist/btree_macaddr8.c
  • src/backend/access/brin/brin_minmax_multi.c
  • src/backend/utils/adt/mac8.c
  • **呼び出し先**:
  • 本構造体自体はデータ格納用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • macaddr: 6バイトのMACアドレス用構造体。
  • inet: ネットワークアドレス用の可変長構造体。

失敗モードと境界条件

  • 固定長データ型であるため、メモリ確保やサイズ計算において可変長型のような複雑なヘッダー管理は不要ですが、ポインタ経由で操作する際は適切なアライメントとメモリ確保（palloc0等）が必要です。

保守担当者向けメモ

  • macaddr8 は固定長データ型として設計されています。inet 型と混同しないよう注意してください。
  • 構造体の各フィールド（a〜h）は、ネットワークバイトオーダーまたは特定の比較ロジック（mac8_2_uint64等）で解釈されることを前提としています。
