ExecInitAppend

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:108

目的

Append ノードの実行時状態（AppendState）を初期化し、子プランの準備および実行に必要なデータ構造を構築する。

シグネチャまたは形状

AppendState *ExecInitAppend(Append *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Append プランノード。
  • estate: クエリ実行状態を保持する EState。
  • eflags: 実行フラグ（EXEC_FLAG_MARK はサポート外）。
  • **出力**: 初期化済みの AppendState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_MARK を含めてはならない。
  • パーティションプルーニングが有効な場合、node->part_prune_index >= 0 であること。
  • 子プランが存在する場合、nplans > 0 であること。

アルゴリズムまたは振る舞い

1. AppendState を割り当て、ExecAppend を実行関数として設定する。 2. パーティションプルーニングが有効な場合、ExecInitPartitionExecPruning を呼び出し、実行時に有効なサブプランを特定する。無効な場合は全サブプランを有効とみなす。 3. 各有効なサブプランに対して ExecInitNode を再帰的に呼び出し、結果を appendplanstates 配列に格納する。 4. 子プラン間で共通のタプルスロット形式を特定し、ExecInitResultTupleSlotTL で結果スロットを初期化する。 5. 非同期実行が可能なサブプラン（initNode->async_capable）がある場合、AsyncRequest 構造体を割り当て、非同期処理用のイベント管理構造を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c (182行目)
  • **呼び出し先**:
  • ExecInitPartitionExecPruning
  • ExecInitNode
  • ExecGetCommonSlotOps
  • ExecInitResultTupleSlotTL
  • classify_matching_subplans (非同期プランが存在する場合)

関連データ構造

  • AppendState: Append ノードの実行時状態を保持する主要構造体。
  • Append: プランツリー上のノード定義。
  • PlanState: 子プランの実行状態。
  • AsyncRequest: 非同期実行時のリクエスト管理構造体。

失敗モードと境界条件

  • eflags に EXEC_FLAG_MARK が含まれる場合、Assert により停止する。
  • 子プランが一つも存在しない場合、nplans の計算や bms_add_range の呼び出しにおいて境界条件に注意が必要（ソース上は nplans > 0 を前提としている）。

保守担当者向けメモ

  • 本関数は Append ノードの初期化を担うため、子プランの追加や非同期実行の仕様変更時には ExecInitNode の呼び出しループおよび AsyncRequest の初期化ロジックを同期させる必要がある。
  • ExecGetCommonSlotOps によるスロット最適化は、子プランの出力形式が統一されている場合にのみ有効である。
