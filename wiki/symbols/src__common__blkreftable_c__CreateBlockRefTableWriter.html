CreateBlockRefTableWriter

定義

  • 種別: function
  • 定義箇所: src/common/blkreftable.c:803

目的

ブロック参照テーブルをディスクやストリームへ書き出すための BlockRefTableWriter 構造体を初期化する。

シグネチャまたは形状

BlockRefTableWriter *CreateBlockRefTableWriter(io_callback_fn write_callback, void *write_callback_arg)

入力と出力

  • **入力**:
  • write_callback: 書き込み処理を実行するコールバック関数。
  • write_callback_arg: コールバック関数に渡される引数。
  • **出力**:
  • 初期化済みの BlockRefTableWriter 構造体へのポインタ。

前提条件と不変条件

  • palloc0 によりメモリがゼロクリアされた状態で割り当てられる。
  • BLOCKREFTABLE_MAGIC 定数が書き込みバッファの先頭に書き込まれる。
  • CRC32C チェックサム計算が初期化される。

アルゴリズムまたは振る舞い

1. BlockRefTableWriter 構造体用にメモリを確保する。 2. 引数として受け取ったコールバック関数と引数を構造体の buffer メンバに格納する。 3. INIT_CRC32C を呼び出し、CRC 計算の状態を初期化する。 4. BlockRefTableWrite を呼び出し、マジックナンバー（BLOCKREFTABLE_MAGIC）を書き込みバッファへ出力する。 5. 初期化された構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース excerpt には具体的な呼び出し元の記述はありません。
  • **呼び出し先**:
  • palloc0
  • INIT_CRC32C
  • BlockRefTableWrite

関連データ構造

  • BlockRefTableWriter: 書き込み状態を保持する構造体。
  • io_callback_fn: I/O を抽象化するための関数ポインタ型。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる可能性がある。
  • write_callback が失敗した場合の挙動は、コールバックの実装に依存する。

保守担当者向けメモ

  • 本関数は BlockRefTable のシリアライズ処理の開始点となる。
  • write_callback を介した抽象化により、ファイル出力だけでなくメモリバッファへの書き出しにも対応可能である。
