contains_required_value

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_bool.c:361

目的

クエリツリー（ポーランド記法）を走査し、そのクエリが「必須の値（required value）」を含んでいるかどうかを判定する。これは、インデックススキャンにおいてクエリが特定の条件を満たすために必要な値を持っているかを判断するために使用される。

シグネチャまたは形状

static bool contains_required_value(ITEM *curitem)

入力と出力

  • **入力**: ITEM *curitem - 評価対象のクエリノードへのポインタ。
  • **出力**: bool - 必須の値が含まれていると判断された場合は true、そうでない場合は false。

前提条件と不変条件

  • check_stack_depth() が呼び出され、再帰によるスタックオーバーフローが防止されている。
  • 入力される ITEM は、makepol によって構築された有効なポーランド記法ツリーの一部であること。

アルゴリズムまたは振る舞い

再帰的にツリーを走査し、以下のルールで判定を行う：

1. **VAL (値)**: 必須の値とみなされ true を返す。 2. **! (NOT演算子)**: 必須ではないとみなされ false を返す（ネストされたNOTの解析は行わない）。 3. **& (AND演算子)**: 左オペランド（curitem + curitem->left）または右オペランド（curitem - 1）のいずれかが必須の値を含んでいれば true を返す。 4. **| (OR演算子)**: 左オペランドと右オペランドの両方が必須の値を含んでいる場合にのみ true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_bool.c 内の query_has_required_values から呼び出される。
  • 自身を再帰的に呼び出す。
  • **呼び出し先**:
  • check_stack_depth() (PostgreSQL 内部関数)

関連データ構造

  • ITEM: クエリツリーの各ノードを保持する構造体。type（値か演算子か）、val（演算子の種類）、left（左オペランドへのオフセット）を持つ。

失敗モードと境界条件

  • **スタックオーバーフロー**: 深いネストを持つクエリに対しては check_stack_depth() によりエラーが発生する可能性がある。
  • **NOT演算子の扱い**: ! 演算子の配下にある値は、論理的に必須であってもこの関数では false と判定される。

保守担当者向けメモ

  • 本関数はヒューリスティックな判定を行っている。特に ! 演算子配下の解析を省略しているため、より厳密な判定が必要な場合はロジックの拡張を検討すること。
  • curitem + curitem->left および curitem - 1 によるポインタ演算は、ポーランド記法配列の構造に強く依存しているため、ITEM 構造体のメモリレイアウトを変更する際は注意が必要である。
