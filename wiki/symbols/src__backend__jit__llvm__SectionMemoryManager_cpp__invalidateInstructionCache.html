invalidateInstructionCache

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/SectionMemoryManager.cpp:352

目的

JITコンパイルされたコードが格納されているメモリ領域に対し、CPUの命令キャッシュを無効化（フラッシュ）し、メモリ上の最新の機械語命令が実行されるように同期する。

シグネチャまたは形状

void invalidateInstructionCache();

入力と出力

  • 入力: なし（クラスメンバ関数として CodeMem.PendingMem を参照）
  • 出力: なし（副作用としてCPUキャッシュの状態を更新）

前提条件と不変条件

  • CodeMem.PendingMem に格納されているメモリブロックが、有効な実行可能領域として確保済みであること。

アルゴリズムまたは振る舞い

1. CodeMem.PendingMem リスト内の各 sys::MemoryBlock を順次走査する。 2. 各ブロックのベースアドレスとサイズを取得する。 3. sys::Memory::InvalidateInstructionCache を呼び出し、該当領域の命令キャッシュを無効化する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/jit/llvm/SectionMemoryManager.cpp:301 (finalizeMemory 内から呼び出される)
  • src/include/jit/SectionMemoryManager.h:178 (仮想関数宣言)
  • 呼び出し先:
  • sys::Memory::InvalidateInstructionCache (LLVMのOS抽象化レイヤー)

関連データ構造

  • CodeMem: SectionMemoryManager クラスが保持するコードセクション管理構造。
  • sys::MemoryBlock: LLVMのメモリブロック表現。

失敗モードと境界条件

  • PendingMem が空の場合、何も処理を行わず正常終了する。
  • sys::Memory::InvalidateInstructionCache の内部実装に依存し、OSやアーキテクチャごとのキャッシュ同期制約を受ける。

保守担当者向けメモ

  • 本関数は finalizeMemory の最終工程で実行され、JIT生成コードの実行直前の整合性を保証する。
  • LLVMのバックポート実装であるため、将来的なLLVMのAPI変更やJITLinkへの移行時には、このキャッシュ同期ロジックの必要性を再評価すること。
