tarPrintf

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_tar.c:953

目的

フォーマット文字列と可変長引数を使用して生成した文字列を、指定されたTARメンバー（TAR_MEMBER）へ書き込む。

シグネチャまたは形状

int tarPrintf(TAR_MEMBER *th, const char *fmt, ...);

入力と出力

  • **入力**:
  • th: 書き込み先となる TAR_MEMBER 構造体へのポインタ。
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**:
  • 戻り値: tarWrite が返した書き込みバイト数（int）。

前提条件と不変条件

  • th は有効な TAR_MEMBER 構造体を指している必要がある。
  • pg_malloc によるメモリ確保が成功することを前提としている。

アルゴリズムまたは振る舞い

1. 初期バッファサイズ（128バイト）を確保する。 2. pvsnprintf を使用して、現在のバッファサイズでフォーマットを試みる。 3. バッファサイズが不足している場合（cnt >= len）、必要なサイズ cnt に拡張して再試行する。 4. フォーマットが完了したら、tarWrite を呼び出してバッファの内容をアーカイブへ書き込む。 5. 確保した作業用バッファを解放し、書き込みバイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_tar.c 内の各所（アーカイブのメタデータやTOC情報の書き込み時に利用）。
  • **呼び出し先**:
  • pg_malloc
  • pvsnprintf
  • tarWrite
  • free

関連データ構造

  • TAR_MEMBER: TARアーカイブ内の個々のファイルを管理する構造体。

失敗モードと境界条件

  • メモリ不足時: pg_malloc が失敗した場合、プログラムは終了する可能性がある。
  • 書き込み失敗時: tarWrite がエラーを返した場合、その結果がそのまま呼び出し元に伝播する。

保守担当者向けメモ

  • 本関数は pg_attribute_printf(2, 3) により、コンパイル時にフォーマット文字列の型チェックが行われるよう設計されている。
  • バッファサイズが不足した際の再試行ロジックは、pvsnprintf の戻り値に基づいて動的に拡張されるため、非常に長い文字列の出力にも対応可能である。
