pg_cryptohash_create

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash_openssl.c:121

目的

OpenSSLの EVP_MD_CTX をラップした暗号学的ハッシュ計算用コンテキストを生成・初期化する。

シグネチャまたは形状

pg_cryptohash_ctx *pg_cryptohash_create(pg_cryptohash_type type)

入力と出力

  • **入力**: pg_cryptohash_type type（使用するハッシュアルゴリズムの種別）
  • **出力**: 成功時は確保された pg_cryptohash_ctx 構造体へのポインタ、失敗時は NULL。

前提条件と不変条件

  • バックエンド環境では、ResourceOwner がコンテキストを保持するための十分な空き容量を確保している必要がある。
  • EVP_MD_CTX_create() の呼び出し前に ERR_clear_error() を実行し、OpenSSLのエラーキューをクリアする。

アルゴリズムまたは振る舞い

1. **リソース確保**: バックエンド環境であれば ResourceOwnerEnlarge を呼び出し、コンテキスト用のメモリを確保する。 2. **初期化**: 構造体をゼロクリアし、指定された type とエラー状態を初期化する。 3. **OpenSSLコンテキスト生成**: EVP_MD_CTX_create() を実行する。 4. **エラー処理**: 生成に失敗した場合はメモリを解放し、バックエンドであれば ereport(ERROR, ...) を発行し、フロントエンドであれば NULL を返す。 5. **リソース登録**: バックエンド環境であれば、ResourceOwnerRememberCryptoHash を呼び出し、トランザクション終了時の自動解放を保証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/libpq/auth-scram.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/checksum_helper.c
  • src/common/cryptohash.c
  • **呼び出し先**:
  • ResourceOwnerEnlarge (バックエンドのみ)
  • ALLOC / FREE (メモリ管理マクロ)
  • ERR_clear_error (OpenSSL)
  • EVP_MD_CTX_create (OpenSSL)
  • explicit_bzero
  • ResourceOwnerRememberCryptoHash (バックエンドのみ)

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態を保持する構造体。
  • pg_cryptohash_type: ハッシュアルゴリズムの列挙型。

失敗モードと境界条件

  • メモリ不足時: ALLOC が NULL を返すか、EVP_MD_CTX_create が失敗する。バックエンドではエラーとして報告され、フロントエンドでは NULL が返される。
  • OpenSSLエラー: EVP_MD_CTX_create が失敗した場合、explicit_bzero でコンテキストをクリアしてから解放する。

保守担当者向けメモ

  • バックエンド環境では ResourceOwner による自動クリーンアップが必須であるため、ResourceOwner 関連の呼び出しを削除しないこと。
  • フロントエンドとバックエンドでエラー時の挙動が異なるため、修正時は両方のビルド環境を考慮すること。
