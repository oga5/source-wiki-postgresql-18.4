ECPGsetconn

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/connect.c:194

目的

指定された名前のデータベース接続を現在のスレッドにおけるアクティブな接続として設定します。

シグネチャまたは形状

bool ECPGsetconn(int lineno, const char *connection_name)

入力と出力

  • **入力**:
  • lineno: 呼び出し元のソースコード行番号（エラー報告用）。
  • connection_name: 切り替え対象となる接続の識別名。
  • **出力**:
  • bool: 成功した場合は true、接続が見つからない、または初期化に失敗した場合は false を返します。

前提条件と不変条件

  • ecpg_get_connection を介して、指定された connection_name に対応する connection 構造体が取得可能である必要があります。
  • actual_connection_key（スレッドローカルストレージのキー）が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. ecpg_get_connection(connection_name) を呼び出し、対象の接続構造体を取得します。 2. ecpg_init を呼び出し、取得した接続の初期化状態を確認します。初期化に失敗した場合は false を返して終了します。 3. pthread_setspecific を使用し、actual_connection_key に対象の接続構造体を格納することで、現在のスレッドにおけるカレント接続として設定します。 4. 成功時に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/test/expected/connect-test2.c
  • src/interfaces/ecpg/test/expected/connect-test4.c
  • **呼び出し先**:
  • ecpg_get_connection (接続構造体の取得)
  • ecpg_init (接続の初期化確認)
  • pthread_setspecific (スレッドローカルストレージへの保存)

関連データ構造

  • connection: 接続情報を保持する構造体。
  • actual_connection_key: スレッドごとのカレント接続を追跡するための pthread_key_t。

失敗モードと境界条件

  • connection_name に一致する接続が存在しない場合、ecpg_get_connection が NULL を返し、続く ecpg_init が失敗して false が返されます。
  • ecpg_init が内部的なエラーを検出した場合、本関数は false を返します。

保守担当者向けメモ

  • 本関数はスレッドローカルな接続状態を更新するため、マルチスレッド環境下での接続切り替えにおいて重要な役割を果たします。
  • pthread 関連の依存関係があるため、プラットフォーム固有の挙動（Windows環境での ecpg-pthread-win32.h 等）に注意してください。
