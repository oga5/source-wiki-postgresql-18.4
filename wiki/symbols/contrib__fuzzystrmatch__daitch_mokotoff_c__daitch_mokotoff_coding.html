daitch_mokotoff_coding

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/daitch_mokotoff.c:518

目的

入力文字列に対して Daitch-Mokotoff Soundex アルゴリズムを適用し、音韻的な類似性に基づくコードを生成して ArrayBuildState に蓄積する。

シグネチャまたは形状

static bool daitch_mokotoff_coding(const char *word, ArrayBuildState *soundex)

入力と出力

  • **入力**:
  • word: 処理対象の文字列。
  • soundex: 生成されたコードを格納するための ArrayBuildState 構造体。
  • **出力**:
  • 成功した場合は true、入力文字列にエンコード可能な文字が含まれない場合は false を返す。

前提条件と不変条件

  • word は UTF-8 から正規化された ASCII 文字列であることが期待される。
  • 処理中に dm_node を用いた動的なツリー構築が行われる。
  • 最終的に生成されるコードは 6 桁の Soundex コードとして出力される。

アルゴリズムまたは振る舞い

1. **初期化**: read_letter を使用して最初の音韻単位を読み取り、開始ノードを生成する。 2. **ループ処理**: 入力文字列が尽きるか、すべてのコードが 6 桁に達するまで繰り返す。

  • read_letter で次の音韻単位を取得。
  • update_leaves を呼び出し、現在の音韻状態（dm_node）を更新・分岐させる。

    • 3. **結果の蓄積**: ループ終了後、残存するノードを走査し、cstring_to_text_with_len を経て accumArrayResult により soundex 配列へ追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/fuzzystrmatch/daitch_mokotoff.c 内の daitch_mokotoff 関数から呼び出される。
  • **呼び出し先**:
  • read_letter: 音韻単位の読み取り。
  • update_leaves: ノード状態の更新。
  • palloc_object: メモリ確保。
  • cstring_to_text_with_len, accumArrayResult: PostgreSQL 配列への変換と格納。

関連データ構造

  • dm_node: 音韻コード生成過程の状態を保持するツリー構造体。
  • ArrayBuildState: PostgreSQL の配列構築用構造体。
  • dm_codes: 音韻ルールを保持する構造体。

失敗モードと境界条件

  • 入力文字列にエンコード可能な文字が含まれない場合、read_letter が失敗し、関数は false を返す。
  • 入力文字列の長さや音韻分岐の数に応じて、メモリ消費量や処理時間が増大する可能性がある。

保守担当者向けメモ

  • 本関数は daitch_mokotoff.h で定義された音韻ルールに依存している。ルール変更時は生成スクリプトを確認すること。
  • dm_node の children および next ポインタによるツリー構造の管理がロジックの核心であるため、デバッグ時はノードの遷移状態に注意すること。
