sql_exec_dumpalltbspc

定義

  • 種別: function
  • 定義箇所: contrib/oid2name/oid2name.c:570

目的

データベース内の全テーブルスペース情報を取得し、OID とテーブルスペース名を標準出力に表示する。

シグネチャまたは形状

void sql_exec_dumpalltbspc(PGconn *conn, struct options *opts)

入力と出力

  • **入力**:
  • conn: データベース接続を保持する PGconn ポインタ。
  • opts: 実行時オプションを保持する struct options ポインタ（quiet フラグを参照）。
  • **出力**: なし（関数内で sql_exec を呼び出し、結果を標準出力へ書き出す）。

前提条件と不変条件

  • conn は有効なデータベース接続状態であること。
  • opts は初期化済みであること。

アルゴリズムまたは振る舞い

1. pg_catalog.pg_tablespace システムカタログから oid と spcname を選択する SQL クエリ文字列を生成する。 2. 生成したクエリを sql_exec 関数に渡し、実行および結果の整形出力を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/oid2name/oid2name.c:613（main 関数内と推測される箇所）から呼び出される。
  • **呼び出し先**: sql_exec 関数を呼び出す。

関連データ構造

  • PGconn: libpq のデータベース接続ハンドル。
  • struct options: 実行時の設定（quiet モードなど）を保持する構造体。

失敗モードと境界条件

  • sql_exec 内部でのクエリ実行失敗（権限不足や接続切断など）が発生した場合、その挙動は sql_exec の実装に依存する。
  • todo バッファ（1024バイト）を超えるクエリは生成されないため、バッファオーバーフローの懸念はない。

保守担当者向けメモ

  • 本関数は pg_tablespace を直接参照するため、PostgreSQL のカタログスキーマに変更があった場合はクエリの更新が必要となる可能性がある。
