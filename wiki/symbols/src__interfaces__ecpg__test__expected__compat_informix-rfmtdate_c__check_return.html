check_return

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-rfmtdate.c:164

目的

ECPG Informix 互換ライブラリの関数が返したエラーコードを判別し、対応するエラー定数名を標準出力へ出力するヘルパー関数です。

シグネチャまたは形状

static void check_return(int ret)

入力と出力

  • **入力**: int ret（変換関数から返されたエラーコード）
  • **出力**: なし（標準出力への文字列書き出しのみ）

前提条件と不変条件

  • 入力される ret は、Informix 互換ライブラリが定義するエラーコード（ECPG_INFORMIX_*）であることを想定しています。

アルゴリズムまたは振る舞い

1. switch 文を用いて ret の値を評価します。 2. 既知のエラーコード（ECPG_INFORMIX_ENOTDMY, ECPG_INFORMIX_ENOSHORTDATE, ECPG_INFORMIX_BAD_DAY, ECPG_INFORMIX_BAD_MONTH）に一致する場合、その定数名を括弧付きで出力します。 3. いずれにも一致しない場合、default 句にて (unknown ret: %d) という形式で数値を表示します。 4. 最後に改行を出力して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/test/expected/compat_informix-rfmtdate.c 内のテスト関数群。
  • src/interfaces/ecpg/test/expected/compat_informix-rfmtlong.c 内のテスト関数群。
  • **呼び出し先**: 標準ライブラリの printf 関数。

関連データ構造

  • pgtypes_error.h および ecpg_informix.h で定義されているエラー定数群。

失敗モードと境界条件

  • 既知のエラーコード以外の値が渡された場合、default 句によりその数値がそのまま出力されます。

保守担当者向けメモ

  • 本関数は回帰テストの期待値確認用です。新しいエラーコードが追加された場合は、switch 文に case を追加する必要があります。
