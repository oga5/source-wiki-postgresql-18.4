gbt_macad8key_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr8.c:59

目的

GiST インデックスの構築時におけるソート処理などで使用される、mac8KEY 構造体同士の比較を行う。

シグネチャまたは形状

int gbt_macad8key_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の mac8KEY 構造体を指すポインタ（Nsrt 構造体経由で渡される）。
  • flinfo: 関数呼び出し情報（本関数内では直接使用されていない）。
  • **出力**:
  • 比較結果を int で返す（a < b なら負、a == b なら 0、a > b なら正）。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体としてキャスト可能であり、そのメンバ t が mac8KEY 型であることを前提とする。
  • macaddr8_cmp 関数がシステムに登録されている必要がある。

アルゴリズムまたは振る舞い

1. a と b の lower（下限値）を macaddr8_cmp を用いて比較する。 2. lower の比較結果が 0（等しい）でない場合、その結果を返す。 3. lower が等しい場合、upper（上限値）を macaddr8_cmp で比較し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の静的解析によれば、contrib/btree_gist/btree_macaddr8.c 内のソート関連処理（gbt_macad8key_cmp が登録されている箇所）から参照されている。
  • **呼び出し先**: macaddr8_cmp（DirectFunctionCall2 経由）。

関連データ構造

  • mac8KEY: macaddr8 型の範囲（下限・上限）を保持する構造体。
  • Nsrt: ソート処理時に使用される中間データ構造。

失敗モードと境界条件

  • macaddr8_cmp が予期せぬエラーを発生させた場合、DirectFunctionCall2 の挙動に従い例外が発生する可能性がある。
  • 入力ポインタが NULL の場合の挙動は定義されていない。

保守担当者向けメモ

  • 本関数は macaddr8 型の範囲比較を辞書順（まず下限、次に上限）で行う。
  • btree_utils_num.c 等の汎用ユーティリティではなく、macaddr8 特有の比較関数を直接呼び出している点に注意すること。
