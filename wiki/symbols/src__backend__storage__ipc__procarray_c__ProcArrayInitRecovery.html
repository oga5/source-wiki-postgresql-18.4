ProcArrayInitRecovery

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procarray.c:1022

目的

リカバリ処理（ホットスタンバイ）におけるトランザクションID管理環境を初期化する。

シグネチャまたは形状

void ProcArrayInitRecovery(TransactionId initializedUptoXID)

入力と出力

  • **入力**: initializedUptoXID (サブトランザクション管理が初期化済みの最新トランザクションID)
  • **出力**: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • standbyState が STANDBY_INITIALIZED であること。
  • initializedUptoXID が正常なトランザクションID（TransactionIdIsNormal）であること。

アルゴリズムまたは振る舞い

1. latestObservedXid を引数 initializedUptoXID で初期化する。 2. TransactionIdRetreat を呼び出し、latestObservedXid を1つ前の値に減算する。これにより、以降の RecordKnownAssignedTransactionIds や ProcArrayApplyRecoveryInfo において、この時点からトランザクションIDの追跡を開始できるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c:5839
  • **呼び出し先**: TransactionIdIsNormal (マクロ), TransactionIdRetreat (マクロ)

関連データ構造

  • latestObservedXid (グローバル変数)

失敗モードと境界条件

  • standbyState が STANDBY_INITIALIZED でない場合、アサーションエラーが発生する。
  • initializedUptoXID が正常なトランザクションIDでない場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は、ホットスタンバイ時のトランザクション追跡（KnownAssignedXids）の起点となる。
  • latestObservedXid の減算処理は、リカバリ開始時点の整合性を保つために重要である。
