cb_tablespace_mapping

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:61

目的

バックアップ統合処理において、テーブルスペースのパスを元の場所（old_dir）から新しい場所（new_dir）へ再マッピングするための情報を保持する。

シグネチャまたは形状

typedef struct cb_tablespace_mapping
{
	char		old_dir[MAXPGPATH];
	char		new_dir[MAXPGPATH];
	struct cb_tablespace_mapping *next;
} cb_tablespace_mapping;

入力と出力

  • **入力**: 構造体内の old_dir および new_dir フィールドにパス文字列を格納する。
  • **出力**: 連結リストの next ポインタを介して、複数のマッピング定義を保持する。

前提条件と不変条件

  • old_dir および new_dir は MAXPGPATH バイト以内のパスである必要がある。
  • リストの終端は next ポインタが NULL であることで示される。

アルゴリズムまたは振る舞い

  • 本構造体は、コマンドライン引数等で指定されたテーブルスペースの再配置ルールを管理する連結リストのノードとして機能する。
  • pg_malloc0 を使用してメモリ確保が行われる（src/bin/pg_combinebackup/pg_combinebackup.c:448）。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_combinebackup.c 内の関数（add_tablespace_mapping 等と推測されるが、ソース本文からは明示的な呼び出し関係の詳細は不明）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • cb_options: 実行設定を保持する構造体であり、このリストの先頭ポインタ（tsmappings）を保持する。

失敗モードと境界条件

  • メモリ不足時には pg_malloc0 によりプログラムが終了する可能性がある。
  • パス長が MAXPGPATH を超える場合、バッファオーバーフローのリスクがある（入力値の検証が必要）。

保守担当者向けメモ

  • 本構造体は連結リストとして実装されているため、リストの走査時には next ポインタのヌルチェックを確実に行うこと。
  • old_dir と new_dir の比較や操作を行う際は、PostgreSQL の標準的なパス操作ユーティリティを使用すること。
