AlterTablePass

定義

  • 種別: enum
  • 定義箇所: src/backend/commands/tablecmds.c:148

目的

ALTER TABLE コマンドの実行において、複数の変更操作を論理的な順序（フェーズ）で処理するための列挙型です。

シグネチャまたは形状

typedef enum AlterTablePass
{
    AT_PASS_UNSET = -1,
    AT_PASS_DROP,
    AT_PASS_ALTER_TYPE,
    AT_PASS_ADD_COL,
    AT_PASS_SET_EXPRESSION,
    AT_PASS_OLD_INDEX,
    AT_PASS_OLD_CONSTR,
    AT_PASS_ADD_CONSTR,
    AT_PASS_COL_ATTRS,
    AT_PASS_ADD_INDEXCONSTR,
    AT_PASS_ADD_INDEX,
    AT_PASS_ADD_OTHERCONSTR,
    AT_PASS_MISC,
} AlterTablePass;

入力と出力

  • 入力: ALTER TABLE 実行時の各サブコマンドの種別。
  • 出力: 処理順序を決定するためのフェーズ識別子。

前提条件と不変条件

  • AT_PASS_UNSET は初期値として使用され、そのまま実行フェーズに到達するとエラーとなります。
  • DROP 系操作が先頭に、ADD 系操作が後続するように定義されており、カタログ更新の整合性を維持します。

アルゴリズムまたは振る舞い

ALTER TABLE の Phase 2（カタログ更新）において、キューイングされたコマンドをこの列挙型の順序に従って反復処理します。これにより、依存関係のある操作（例：列の削除後に制約を追加する等）が正しい順序で実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: ATRewriteCatalogs および RebuildConstraintComment 等の tablecmds.c 内の関数が、現在の処理フェーズを追跡するために使用しています。
  • 呼び出し先: 直接的な関数呼び出しはありません（列挙型のため）。

関連データ構造

  • AlteredTableInfo: ALTER TABLE 実行中のテーブルごとの状態を保持する構造体。
  • AlterTableCmd: 個別の ALTER TABLE サブコマンドを保持するパースツリー構造体。

失敗モードと境界条件

  • AT_PASS_UNSET が処理フェーズで明示的に指定された場合、エラーが発生します。
  • 定義されていないフェーズ値が渡された場合の挙動は、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • 新しい ALTER TABLE サブコマンドを追加する際は、適切な実行順序を考慮して本列挙型に新しいフェーズを追加するか、既存のフェーズに割り当てる必要があります。
  • AT_NUM_PASSES を用いたループ処理が tablecmds.c 内に存在するため、列挙型の順序変更には注意してください。
