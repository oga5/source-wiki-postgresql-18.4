create_output_directory

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:726

目的

バックアップの出力先ディレクトリを準備し、必要に応じて作成または既存ディレクトリの検証を行う。

シグネチャまたは形状

static void create_output_directory(char *dirname, cb_options *opt)

入力と出力

  • dirname: 作成または検証対象のディレクトリパス。
  • opt: 実行オプションを保持する cb_options 構造体へのポインタ。ドライラン設定などを参照する。
  • 戻り値: なし（成功時）。失敗時は pg_fatal によりプログラムを終了する。

前提条件と不変条件

  • pg_check_dir を用いて対象パスの状態を判定する。
  • 既存ディレクトリの場合、空でないものはエラーとして扱う。

アルゴリズムまたは振る舞い

1. pg_check_dir(dirname) の結果に基づき分岐する。

  • 0 (ディレクトリが存在しない): opt->dry_run が真ならログ出力のみ行い終了。偽なら pg_mkdir_p でディレクトリを作成し、remember_to_cleanup_directory に登録する。
  • 1 (ディレクトリが存在する): 既存ディレクトリとして remember_to_cleanup_directory に登録する。
  • 2, 3, 4 (空でないディレクトリ): pg_fatal で異常終了する。
  • -1 (アクセス不可): pg_fatal で異常終了する。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（src/bin/pg_combinebackup/pg_combinebackup.c:330, 333）。
  • 呼び出し先: pg_check_dir, pg_log_debug, pg_mkdir_p, pg_fatal, remember_to_cleanup_directory。

関連データ構造

  • cb_options: 実行時の設定（ドライランフラグ等）を保持。

失敗モードと境界条件

  • ディレクトリが空でない場合、またはアクセス権限がない場合は pg_fatal により処理が中断される。

保守担当者向けメモ

  • 本関数は remember_to_cleanup_directory を呼び出し、作成または使用するディレクトリをクリーンアップ管理対象として登録する。
  • pg_mkdir_p の作成モードには pg_dir_create_mode が使用される。
