PageIsEmpty

定義

  • 種別: function
  • 定義箇所: src/include/storage/bufpage.h:224

目的

指定されたページが空（データが格納されていない状態）であるかどうかを判定します。

シグネチャまたは形状

static inline bool PageIsEmpty(const PageData *page)

入力と出力

  • **入力**: page (const PageData *) - 検査対象のページへのポインタ。
  • **出力**: bool - ページが空であれば true、そうでなければ false を返します。

前提条件と不変条件

  • page は有効な PageData 構造体を指している必要があります。
  • ページヘッダー内の pd_lower は、ページ内の空き領域の開始位置（ラインポインタ配列の末尾）を示します。

アルゴリズムまたは振る舞い

ページヘッダーの pd_lower 値が SizeOfPageHeaderData 以下であるかどうかを比較します。pd_lower がヘッダーサイズ以下であることは、ページ内にラインポインタが一つも配置されていない（＝データが格納されていない）ことを意味します。

  • **呼び出し先**: 呼び出し先に関する情報は提供されていません。

関連データ構造

  • PageData: ページの実体。
  • PageHeaderData: ページのメタデータを保持する構造体。pd_lower フィールドを含みます。

失敗モードと境界条件

  • page が NULL の場合、未定義の動作を引き起こす可能性があります。
  • ページが初期化されていない（PageInit が呼ばれていない）場合、pd_lower の値が不定であるため、本関数の結果は信頼できません。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスが重視されるパスで使用されます。
  • PageIsNew と組み合わせて使用されるケースが多く、ページの状態遷移を確認する際の標準的なチェックとして機能します。
