raw_parser

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parser.c:41

目的

SQL文字列を解析し、意味解析前の「生の構文木（Raw Parse Tree）」を生成するメインエントリーポイントです。

シグネチャまたは形状

List *raw_parser(const char *str, RawParseMode mode)

入力と出力

  • **入力**:
  • str: 解析対象のSQL文字列。
  • mode: 解析モード（RawParseMode列挙型）。デフォルトのSQL解析のほか、型名やPL/pgSQL式などの特定モードを指定可能。
  • **出力**:
  • 成功時: 生成された構文木のリスト（List *）。
  • 失敗時: NIL。

前提条件と不変条件

  • 本関数はテーブルアクセスを行ってはなりません（トランザクションアボート状態でも動作する必要があるため）。
  • mode に応じて、字句解析器の先読みトークンが適切に初期化されます。

アルゴリズムまたは振る舞い

1. scanner_init を呼び出し、字句解析器を初期化します。 2. mode に基づき、base_yy_extra_type 構造体の先読み情報を設定します。 3. parser_init を呼び出し、Bison構文解析器を初期化します。 4. base_yyparse を実行し、構文木を構築します。 5. scanner_finish を呼び出し、メモリを解放します。 6. 解析結果（yyextra.parsetree）を返します。エラー発生時は NIL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/executor/spi.c
  • src/backend/parser/parse_type.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • scanner_init, parser_init, base_yyparse, scanner_finish（いずれも内部的な解析補助関数）。

関連データ構造

  • base_yy_extra_type: 解析状態（先読みトークンや構文木）を保持する構造体。
  • List: 構文木のリスト構造。
  • RawParseMode: 解析モードを定義する列挙型。

失敗モードと境界条件

  • base_yyparse が非ゼロの値を返した場合（構文エラー等）、関数は NIL を返します。
  • 空のクエリの場合、parsetree は NIL となる可能性があります。

保守担当者向けメモ

  • 本関数は字句解析（Flex）と構文解析（Bison）の橋渡し役です。
  • SQL文法をLALR(1)に収めるため、base_yylex でのトークン置換や先読み処理が組み込まれています。
  • エラー発生時は scanner_errposition を使用してSQL内の位置を報告する設計となっています。
