Interval

定義

  • 種別: struct
  • 定義箇所: src/include/datatype/timestamp.h:47

目的

PostgreSQLにおける期間（時間間隔）データを保持するための構造体です。月や日の長さが可変であるという日時計算特有の複雑さを吸収し、期間を構成する各単位を分離して管理します。

シグネチャまたは形状

typedef struct
{
    TimeOffset  time;   /* 日、月、年以外のすべての時間単位 */
    int32       day;    /* 日 */
    int32       month;  /* 月および年 */
} Interval;

入力と出力

本構造体はデータ保持用であり、関数ではありません。入力として各フィールドに値を格納し、出力として日時演算処理（src/backend/utils/adt/timestamp.c 等）へ期間情報を提供します。

前提条件と不変条件

  • time フィールドは TimeOffset 型（マイクロ秒単位）で保持されます。
  • day および month フィールドは int32 型です。
  • 期間の計算において、月や日の可変性を考慮した分離保持が行われます。

アルゴリズムまたは振る舞い

本構造体は、期間を「月・年」「日」「その他の時間単位」の3つのコンポーネントに分解して保持します。これにより、加算や減算などの日時演算時に、各単位の性質に応じた適切な調整が可能となります。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）の例**:
  • contrib/btree_gin/btree_gin.c: メモリ確保時に使用。
  • contrib/btree_gist/btree_interval.c: インデックス操作や比較演算（intr2num 等）で使用。
  • **呼び出し先**:
  • 本構造体はデータ定義であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Timestamp / TimestampTz: 本構造体を用いた演算対象となる絶対時間。
  • TimeOffset: Interval.time フィールドの型。

失敗モードと境界条件

  • INTERVALSIZE と sizeof(Interval) の不一致に関する懸念が contrib/btree_gist/btree_interval.c 内で言及されていますが、現在の定義では解消済みとされています。

保守担当者向けメモ

  • 本構造体はディスク保存用データ型として使用されます。フィールドの順序や型サイズを変更する場合は、バイナリ互換性に注意してください。
  • 演算ロジックについては src/backend/utils/adt/timestamp.c を参照してください。
