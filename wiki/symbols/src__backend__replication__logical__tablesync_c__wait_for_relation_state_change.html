wait_for_relation_state_change

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/tablesync.c:182

目的

論理レプリケーションにおいて、指定されたテーブルの同期状態が期待される値に遷移するまで待機する。

シグネチャまたは形状

static bool wait_for_relation_state_change(Oid relid, char expected_state)

入力と出力

  • **入力**:
  • relid: 監視対象のテーブルの OID。
  • expected_state: 待機する目標状態（char 型）。
  • **出力**:
  • bool: 状態が expected_state に到達した場合は true、同期ワーカーの終了や状態不明により待機を継続できなくなった場合は false を返す。

前提条件と不変条件

  • 呼び出し元は論理レプリケーションワーカーのコンテキスト（MyLogicalRepWorker が有効）である必要がある。
  • GetSubscriptionRelState を通じて pg_subscription_rel カタログの現在の状態を定期的に再取得する。

アルゴリズムまたは振る舞い

1. 無限ループ内で CHECK_FOR_INTERRUPTS() を実行し、割り込みを処理する。 2. InvalidateCatalogSnapshot() を呼び出し、最新のカタログ情報を取得可能にする。 3. GetSubscriptionRelState で現在の状態を確認する。

  • 状態が SUBREL_STATE_UNKNOWN の場合はループを抜けて false を返す。
  • 状態が expected_state と一致した場合は true を返す。

    • 4. LogicalRepWorkerLock を取得し、対象テーブルの同期ワーカーが現在も稼働中か確認する。ワーカーが存在しない場合はループを抜けて false を返す。 5. WaitLatch を使用して 1000ms 待機し、ResetLatch でラッチをリセットした後、再度ループの先頭に戻る。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/tablesync.c:589 (推測される箇所)
  • **呼び出し先**:
  • GetSubscriptionRelState
  • logicalrep_worker_find
  • WaitLatch
  • ResetLatch
  • CHECK_FOR_INTERRUPTS
  • InvalidateCatalogSnapshot
  • LWLockAcquire / LWLockRelease

関連データ構造

  • LogicalRepWorker: ワーカーの状態管理に使用。
  • SUBREL_STATE_*: 同期状態を表す定数群。

失敗モードと境界条件

  • **同期ワーカーの消失**: 待機中に同期ワーカーが停止した場合、logicalrep_worker_find が失敗し、関数は false を返して終了する。
  • **状態不明**: GetSubscriptionRelState が SUBREL_STATE_UNKNOWN を返した場合、待機を中断する。
  • **割り込み**: CHECK_FOR_INTERRUPTS() により、プロセスへのシグナル送信時に処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は WaitLatch を用いたポーリングループであるため、状態遷移の頻度と待機時間のバランスに注意すること。
  • LogicalRepWorkerLock の取得は共有ロック (LW_SHARED) であり、ワーカーの存在確認のみを行う。
