binaryheap_allocate

定義

  • 種別: function
  • 定義箇所: src/common/binaryheap.c:38

目的

指定された容量と比較関数を用いて、バイナリヒープ構造体を動的に確保および初期化します。

シグネチャまたは形状

binaryheap *binaryheap_allocate(int capacity, binaryheap_comparator compare, void *arg);

入力と出力

  • **入力**:
  • capacity: ヒープに格納可能な最大要素数。
  • compare: 要素の順序を決定するための比較関数。
  • arg: 比較関数に渡される任意のユーザーデータ。
  • **出力**:
  • 初期化済みの binaryheap 構造体へのポインタ。

前提条件と不変条件

  • palloc を使用するため、メモリコンテキストが有効である必要があります。
  • 確保されたヒープは、初期状態で bh_size が 0、bh_has_heap_property が true に設定されます。

アルゴリズムまたは振る舞い

1. offsetof(binaryheap, bh_nodes) と capacity 分のノード領域を合わせたサイズを計算します。 2. palloc を用いて必要なメモリ領域を確保します。 3. 構造体の各フィールド（容量、比較関数、ユーザー引数）を初期化します。 4. ヒープの現在の要素数を 0 に設定し、ヒーププロパティが維持されている状態として初期化を完了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeGatherMerge.c
  • src/backend/executor/nodeMergeAppend.c
  • src/backend/postmaster/pgarch.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/storage/buffer/bufmgr.c
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_dump/pg_dump_sort.c
  • **呼び出し先**:
  • palloc (メモリ確保用)

関連データ構造

  • binaryheap: ヒープの状態を保持する構造体。
  • bh_node_type: ヒープに格納される要素の型。
  • binaryheap_comparator: 比較関数の型定義。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQL のエラーハンドリング機構により例外が発生します。
  • capacity に 0 以下の値を指定した場合の挙動は、palloc の仕様および後続のヒープ操作に依存します。

保守担当者向けメモ

  • 本関数で確保したメモリは、必ず binaryheap_free を呼び出して解放してください。
  • 比較関数 binaryheap_comparator は、実行環境（バックエンドかフロントエンドか）に応じて適切なキャストを行う必要があります。
