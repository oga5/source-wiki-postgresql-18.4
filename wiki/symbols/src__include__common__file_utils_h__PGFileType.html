PGFileType

定義

  • 種別: enum
  • 定義箇所: src/include/common/file_utils.h:18

目的

ファイルシステム上のエントリ種別（通常ファイル、ディレクトリ、シンボリックリンク等）を識別し、PostgreSQLのファイル操作ユーティリティ全体で共通の型として扱うために使用されます。

シグネチャまたは形状

typedef enum PGFileType
{
	PGFILETYPE_ERROR,
	PGFILETYPE_UNKNOWN,
	PGFILETYPE_REG,
	PGFILETYPE_DIR,
	PGFILETYPE_LNK,
} PGFileType;

入力と出力

  • 本シンボルは列挙型であり、関数への引数や戻り値としてファイル種別を伝達するために使用されます。
  • 具体的な値の生成元として、get_dirent_type 等の関数がファイルシステムから取得した情報をこの型に変換して返します。

前提条件と不変条件

  • PGFILETYPE_ERROR は、ファイル種別の取得に失敗した際のエラー状態を示します。
  • PGFILETYPE_UNKNOWN は、ファイルシステムから取得した種別が定義されたカテゴリに該当しない場合に使用されます。

アルゴリズムまたは振る舞い

  • ファイルシステム走査時、dirent 構造体から得られた情報を基に、各プラットフォーム固有のファイル種別を本列挙型のいずれかにマッピングします。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/transam/xlog.c
  • src/backend/replication/logical/snapbuild.c
  • src/backend/replication/slot.c
  • src/backend/storage/file/copydir.c
  • src/backend/storage/file/fd.c
  • src/backend/utils/misc/conffiles.c
  • src/bin/pg_combinebackup/pg_combinebackup.c
  • src/common/file_utils.c
  • **呼び出し先**:
  • 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • dirent 構造体（<dirent.h>）: ファイルシステムエントリのメタデータ取得元。

失敗モードと境界条件

  • ファイルシステムへのアクセス権限不足やパスの不整合により、種別判定が失敗した場合は PGFILETYPE_ERROR が返されることが想定されます。

保守担当者向けメモ

  • 本列挙型は src/common/file_utils.c で実装されるファイル操作ロジックと密接に関連しています。
  • 新しいファイル種別をサポートする必要が生じた場合は、本列挙型の定義と、それを変換する get_dirent_type 等の関数を併せて更新してください。
