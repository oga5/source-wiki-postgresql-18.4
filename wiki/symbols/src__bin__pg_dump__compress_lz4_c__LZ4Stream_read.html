LZ4Stream_read

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:609

目的

LZ4 圧縮されたファイルストリームからデータを読み込み、解凍してバッファへ格納する。

シグネチャまたは形状

static size_t LZ4Stream_read(void *ptr, size_t size, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • ptr: 解凍後のデータを格納するバッファへのポインタ。
  • size: 読み込むデータのバイト数。
  • CFH: 圧縮ファイルハンドル（LZ4State を保持）。
  • **出力**:
  • 実際に読み込まれたバイト数を size_t 型で返す。

前提条件と不変条件

  • CFH->private_data は有効な LZ4State 構造体へのポインタであること。
  • LZ4Stream_read_internal が負の値を返した場合、エラーが発生したものとみなされる。

アルゴリズムまたは振る舞い

1. CFH->private_data から LZ4State を取得する。 2. LZ4Stream_read_internal を呼び出し、解凍処理を実行する（引数 false はオーバーフローバッファの利用を示唆）。 3. 戻り値が負の場合、LZ4Stream_get_error を用いてエラーメッセージを取得し、pg_fatal でプログラムを終了させる。 4. 正常終了時は、解凍されたバイト数を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_lz4.c:789（CFH->read_func への代入として参照）。
  • **呼び出し先**:
  • LZ4Stream_read_internal
  • LZ4Stream_get_error
  • pg_fatal

関連データ構造

  • CompressFileHandle: 圧縮ストリームの抽象化インターフェース。
  • LZ4State: LZ4 圧縮・解凍のコンテキストおよびバッファ状態を保持する構造体。

失敗モードと境界条件

  • LZ4Stream_read_internal が負の値を返した場合、pg_fatal によりプロセスが強制終了する。

保守担当者向けメモ

  • 本関数は CompressFileHandle の read_func インターフェースとして登録される。
  • 内部処理の詳細は LZ4Stream_read_internal に依存しているため、解凍ロジックの変更時は同関数を確認すること。
