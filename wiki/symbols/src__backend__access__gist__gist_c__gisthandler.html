gisthandler

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:58

目的

PostgreSQL の GiST (Generalized Search Tree) インデックスアクセスメソッドをシステムに登録するためのハンドラ関数です。

シグネチャまたは形状

Datum gisthandler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (PostgreSQL の関数呼び出し規約に基づく引数)
  • **出力**: Datum 型として IndexAmRoutine 構造体へのポインタを返す。

前提条件と不変条件

  • makeNode(IndexAmRoutine) を使用してメモリを確保し、GiST インデックスに必要な各種コールバック関数およびフラグを初期化する必要がある。

アルゴリズムまたは振る舞い

1. IndexAmRoutine 構造体を生成する。 2. GiST の特性を示すフラグ（amcanorderbyop = true, amcanmulticol = true 等）を設定する。 3. GiST の各操作に対応する関数ポインタ（gistbuild, gistinsert, gistbeginscan 等）を構造体の各フィールドに割り当てる。 4. 最後に PG_RETURN_POINTER マクロを使用して、設定済みの IndexAmRoutine を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックス管理システム（インデックス作成時やロード時に呼び出される）。
  • **呼び出し先**: gistbuild, gistinsert, gistbulkdelete, gistvacuumcleanup, gistcanreturn, gistcostestimate, gistoptions, gistproperty, gistvalidate, gistadjustmembers, gistbeginscan, gistrescan, gistgettuple, gistgetbitmap, gistendscan, gisttranslatecmptype などの各 GiST 実装関数。

関連データ構造

  • IndexAmRoutine: インデックスアクセスメソッドのインターフェースを定義する構造体。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性がある。
  • 必要なコールバック関数が未定義（NULL）の場合、システム側で適切に処理される必要がある（一部のフィールドは明示的に NULL に設定されている）。

保守担当者向けメモ

  • 新しい GiST 機能を追加する際は、本関数内の IndexAmRoutine フィールドに適切な関数ポインタを追加または更新すること。
  • amparallelvacuumoptions には VACUUM_OPTION_PARALLEL_BULKDEL および VACUUM_OPTION_PARALLEL_COND_CLEANUP が設定されており、並列バキュームに対応している点に注意。
