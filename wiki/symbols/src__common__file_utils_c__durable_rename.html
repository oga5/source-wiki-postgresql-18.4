durable_rename

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:482

目的

ファイルシステム上でのファイル移動（リネーム）操作において、クラッシュ耐性を保証するために必要なメタデータおよびデータ内容の同期（fsync）を伴う安全なリネームを実行する。

シグネチャまたは形状

int durable_rename(const char *oldfile, const char *newfile)

入力と出力

  • **入力**:
  • oldfile: 移動元のファイルパス（const char *）
  • newfile: 移動先のファイルパス（const char *）
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は -1 を返す。

前提条件と不変条件

  • oldfile は存在し、アクセス可能であること。
  • newfile が既に存在する場合、そのファイルも同期対象として扱われる。
  • OSが fsync をサポートしている環境であること。

アルゴリズムまたは振る舞い

1. **事前同期**: fsync_fname(oldfile) を呼び出し、移動元ファイルの永続化を保証する。 2. **ターゲット同期**: newfile が存在する場合、それをオープンして fsync を実行し、既存の移動先ファイルを永続化する。 3. **リネーム実行**: rename(oldfile, newfile) を実行する。 4. **事後同期**:

  • fsync_fname(newfile) を呼び出し、リネーム後のファイル内容を永続化する。
  • fsync_parent_path(newfile) を呼び出し、親ディレクトリのメタデータを更新してリネーム操作を永続化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/basic_archive/basic_archive.c
  • contrib/pg_prewarm/autoprewarm.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/backend/access/transam/timeline.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogarchive.c
  • **呼び出し先**:
  • fsync_fname
  • fsync_parent_path
  • 標準ライブラリ: open, fsync, close, rename

関連データ構造

  • 特になし（標準的なファイルパス文字列とファイルディスクリプタを使用）。

失敗モードと境界条件

  • oldfile の同期失敗、または newfile のオープン・同期失敗時に -1 を返す。
  • rename システムコールが失敗した場合、エラーログを出力して -1 を返す。
  • newfile の事後同期（ファイルまたは親ディレクトリ）に失敗した場合、-1 を返す。
  • newfile が存在しない場合の open エラー（ENOENT）は正常系として処理される。

保守担当者向けメモ

  • 本関数はフロントエンドツールおよびバックエンドの両方から利用される可能性があるため、FRONTEND マクロの定義状況に注意すること。
  • fsync_parent_path はプラットフォームによってはディレクトリの同期をサポートしていない場合があり、その場合はエラーを無視する設計となっている。
