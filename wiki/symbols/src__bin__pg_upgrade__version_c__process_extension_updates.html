process_extension_updates

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/version.c:165

目的

アップグレード後に更新が必要な拡張機能（Extension）のリストを抽出し、それらを更新するためのSQLスクリプトを生成する。

シグネチャまたは形状

static void process_extension_updates(DbInfo *dbinfo, PGresult *res, void *arg)

入力と出力

  • dbinfo: データベース情報を含む DbInfo 構造体。
  • res: 拡張機能のクエリ結果を含む PGresult。
  • arg: UpgradeTaskReport 構造体へのポインタ。
  • 出力: 戻り値なし。副作用として、指定されたパスに ALTER EXTENSION ... UPDATE; を含むSQLファイルを作成または追記する。

前提条件と不変条件

  • process_extension_updates は UpgradeTaskProcessCB 型と互換性がある必要がある（AssertVariableIsOfType による検証）。
  • arg は UpgradeTaskReport 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. クエリ結果 (res) のタプル数が0であれば即座に終了する。 2. report->file が未オープンであれば、report->path を書き込みモードで開く。 3. appendPsqlMetaConnect を使用して、対象データベースへの接続コマンドをファイルに書き込む。 4. クエリ結果の各行を走査し、拡張機能名を取得して ALTER EXTENSION <name> UPDATE; 文をファイルに書き出す。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/version.c 内の upgrade_task_add_step にコールバックとして登録される。
  • 呼び出し先: PQntuples, PQfnumber, fopen_priv, pg_fatal, initPQExpBuffer, appendPsqlMetaConnect, termPQExpBuffer, fprintf, quote_identifier, PQgetvalue。

関連データ構造

  • DbInfo: データベースの識別情報。
  • UpgradeTaskReport: 出力先ファイルパスやファイルハンドルを保持する構造体。
  • PGresult: libpq のクエリ結果セット。

失敗モードと境界条件

  • report->file のオープンに失敗した場合、pg_fatal によりプロセスが異常終了する。
  • クエリ結果が空の場合、ファイル生成や書き込みは行われない。

保守担当者向けメモ

  • 本関数は UpgradeTask フレームワークの一部として動作する。
  • quote_identifier を使用してSQLインジェクションを防いでいるため、拡張機能名の処理は安全である。
  • ファイル操作は fopen_priv を介して行われるため、権限管理に注意すること。
