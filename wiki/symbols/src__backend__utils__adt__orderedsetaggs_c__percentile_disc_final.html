percentile_disc_final

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/orderedsetaggs.c:426

目的

離散的パーセンタイル（percentile_disc）集約関数の最終計算を実行し、ソートされたデータセットから指定されたパーセンタイル値に対応する値を抽出する。

シグネチャまたは形状

Datum percentile_disc_final(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • fcinfo: PG_FUNCTION_ARGS。第0引数に OSAPerGroupState*、第1引数に float8 型のパーセンタイル値（0〜1）をとる。
  • **出力**:
  • Datum: 指定されたパーセンタイルに対応する値。入力がNULLまたはデータが存在しない場合はNULLを返す。

前提条件と不変条件

  • AggCheckCallContext により、集約コンテキストで呼び出されていることを検証する。
  • パーセンタイル値は 0 から 1 の範囲内である必要がある（範囲外はエラー）。
  • OSAPerGroupState 内の number_of_rows が 0 の場合、結果は NULL となる。

アルゴリズムまたは振る舞い

1. **ソートの実行**: osastate->sort_done が偽であれば tuplesort_performsort を呼び出し、真であれば tuplesort_rescan を呼び出してソート済み状態を確保する。 2. **順位の算出**: rownum = ceil(percentile * number_of_rows) を計算し、対象となる行番号を特定する。 3. **行のスキップ**: tuplesort_skiptuples を使用して、対象行の直前までカーソルを進める。 4. **値の抽出**: tuplesort_getdatum を使用して対象行の値を取得し、結果として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の集約実行エンジン（nodeAgg.c 等から間接的に呼び出される）。
  • **呼び出し先**: tuplesort_performsort, tuplesort_rescan, tuplesort_skiptuples, tuplesort_getdatum。

関連データ構造

  • OSAPerGroupState: グループごとの集約状態を保持し、tuplesort オブジェクトを管理する。

失敗モードと境界条件

  • **エラー**: パーセンタイル値が 0 未満、1 超、または NaN の場合に ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE を送出。
  • **エラー**: tuplesort_skiptuples または tuplesort_getdatum が行の取得に失敗した場合、elog(ERROR) を送出。
  • **境界**: 入力データがすべて NULL の場合、または number_of_rows が 0 の場合は NULL を返す。

保守担当者向けメモ

  • 本関数は tuplesort を利用して順序を確定させるため、メモリ消費量は work_mem に依存する。
  • rownum の計算において、ceil を使用して離散的な順位を決定するロジックは、percentile_disc の定義に直結している。
