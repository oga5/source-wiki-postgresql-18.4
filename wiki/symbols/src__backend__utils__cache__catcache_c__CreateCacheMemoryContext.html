CreateCacheMemoryContext

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/catcache.c:718

目的

キャッシュ関連のデータ構造を保持するための専用メモリコンテキスト（CacheMemoryContext）を初期化する。

シグネチャまたは形状

void CreateCacheMemoryContext(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 CacheMemoryContext を初期化する）

前提条件と不変条件

  • CacheMemoryContext が未初期化（NULL）である場合にのみ作成される。
  • TopMemoryContext が既に存在している必要がある。

アルゴリズムまたは振る舞い

1. グローバル変数 CacheMemoryContext が NULL であるかを確認する。 2. NULL の場合、AllocSetContextCreate を呼び出し、TopMemoryContext を親として CacheMemoryContext という名称のメモリコンテキストを作成する。 3. 作成したコンテキストを CacheMemoryContext に代入する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/cache/attoptcache.c
  • src/backend/utils/cache/catcache.c
  • src/backend/utils/cache/evtcache.c
  • src/backend/utils/cache/relcache.c
  • src/backend/utils/cache/relfilenumbermap.c
  • src/backend/utils/cache/spccache.c
  • src/backend/utils/cache/ts_cache.c
  • 呼び出し先: AllocSetContextCreate（メモリコンテキスト作成用）

関連データ構造

  • CacheMemoryContext（グローバル変数）
  • TopMemoryContext（親メモリコンテキスト）

失敗モードと境界条件

  • TopMemoryContext が初期化されていない状態で呼び出された場合、メモリ管理システムによりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は冪等性を持ち、複数回呼び出されても CacheMemoryContext が一度作成されていれば再作成は行われない。
  • キャッシュのライフサイクル管理において、メモリコンテキストの生存期間を制御する重要な起点となる。
