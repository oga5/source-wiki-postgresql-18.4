PLyObject_ToBytea

定義

  • **種別**: 関数
  • **定義箇所**: src/pl/plpython/plpy_typeio.c:899

目的

PythonオブジェクトをPostgreSQLの bytea 型（Datum）へ変換します。

シグネチャまたは形状

static Datum PLyObject_ToBytea(PLyObToDatum *arg, PyObject *plrv, bool *isnull, bool inarray)

入力と出力

  • **入力**:
  • arg: 変換メタデータ構造体（未使用）。
  • plrv: 変換対象の PyObject。
  • isnull: NULLフラグを格納するポインタ。
  • inarray: 配列内での変換かを示すフラグ（未使用）。
  • **出力**:
  • 戻り値: 変換後の bytea データを保持する Datum。
  • isnull: plrv が Py_None の場合は true、それ以外は false が設定される。

前提条件と不変条件

  • plrv が Py_None の場合、関数は即座に (Datum) 0 を返し、isnull を true に設定します。
  • PyObject_Bytes によるバイト表現の生成に失敗した場合、PLy_elog(ERROR, ...) を呼び出し、処理を中断します。

アルゴリズムまたは振る舞い

1. plrv が Py_None かを確認し、該当すればNULLとして処理を終了します。 2. PyObject_Bytes(plrv) を呼び出し、Pythonオブジェクトのバイト表現を取得します。 3. PG_TRY ブロック内で以下を実行します：

  • PyBytes_AsString と PyBytes_Size を用いて、バイト列のポインタとサイズを取得します。
  • palloc を使用して、VARHDRSZ（ヘッダーサイズ）を加算したメモリ領域を確保します。
  • SET_VARSIZE でサイズを記録し、memcpy でデータをコピーします。
  • PointerGetDatum で bytea 構造体を Datum に変換します。

    • 4. PG_FINALLY ブロックにて、一時的に生成した plrv_so の参照カウントを Py_XDECREF で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_typeio.c 内の変換セットアップ処理（arg->func = PLyObject_ToBytea;）にて参照されています。
  • **呼び出し先**: PyObject_Bytes, PyBytes_AsString, PyBytes_Size, palloc, SET_VARSIZE, memcpy, PointerGetDatum, PG_TRY, PG_FINALLY, Py_XDECREF, PLy_elog。

関連データ構造

  • PLyObToDatum: 変換ロジックを保持する構造体。
  • bytea: PostgreSQLの可変長バイト列型。

失敗モードと境界条件

  • PyObject_Bytes が失敗した場合、PostgreSQLの ERROR レベルで例外が発生します。
  • メモリ不足時は palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は PG_TRY を使用してリソース解放を保証しています。修正を加える際は、Py_XDECREF が確実に実行されるよう注意してください。
  • arg および inarray 引数は現在の実装では使用されていません。
