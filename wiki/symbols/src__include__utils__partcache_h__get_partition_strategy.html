get_partition_strategy

定義

  • 種別: function (インライン関数)
  • 定義箇所: src/include/utils/partcache.h:58

目的

指定されたパーティションキー構造体から、そのテーブルが採用しているパーティション戦略（RANGE, LIST, HASH）を判定して返します。

シグネチャまたは形状

static inline int get_partition_strategy(PartitionKey key)

入力と出力

  • **入力**: PartitionKey key - パーティションテーブルのメタデータを保持する構造体へのポインタ。
  • **出力**: int - パーティション戦略を表す整数値。

前提条件と不変条件

  • 引数 key は有効な PartitionKey 構造体を指している必要があります。
  • パーティションキーはリレーションが開かれている間は不変であり、本関数はキャッシュされた値を直接参照します。

アルゴリズムまたは振る舞い

1. 引数として受け取った PartitionKey 構造体の strategy フィールドにアクセスします。 2. 保持されている戦略値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_utilcmd.c:4292 にて、key から戦略を取得する用途で呼び出されています。
  • **呼び出し先**: 本関数はインライン関数であり、他の関数を呼び出しません。

関連データ構造

  • PartitionKeyData: パーティションキーの全情報を保持する構造体。本関数はこの構造体の strategy メンバを参照します。

失敗モードと境界条件

  • key が NULL の場合、セグメンテーション違反が発生します。呼び出し側で PartitionKey が構築済みであることを保証する必要があります。

保守担当者向けメモ

  • 本関数は PartitionKeyData 構造体への直接アクセスをカプセル化する抽象化レイヤーです。将来的に PartitionKey の内部構造が変更された場合、この関数を修正することで影響範囲を局所化できます。
