foreign_loc_cxt

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/deparse.c:88

目的

式ツリーのデパース（SQL構築）処理において、照合順序（Collation）の伝播状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct foreign_loc_cxt
{
    Oid            collation;        /* 現在の照合順序のOID */
    FDWCollateState state;           /* 現在の照合順序選択の状態 */
} foreign_loc_cxt;

入力と出力

  • **入力**: 構造体の各フィールドは、式ツリーの走査過程で照合順序の整合性を検証するために使用されます。
  • **出力**: 走査中のノードにおける照合順序の状態を保持し、後続の式評価やSQL構築の判断材料として利用されます。

前提条件と不変条件

  • collation フィールドには、有効な照合順序の OID または InvalidOid が格納されます。
  • state フィールドは FDWCollateState 型であり、照合順序の解決状況（未決定、決定済みなど）を追跡します。

アルゴリズムまたは振る舞い

式ツリーの再帰的な走査（foreign_expr_walker 等）において、親ノードから子ノードへ照合順序のコンテキストを渡すために使用されます。これにより、リモートサーバーで実行する際に照合順序の不一致が発生しないよう、式全体の整合性を検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/deparse.c 内の式走査関数（foreign_expr_walker 等）において、ローカル変数としてインスタンス化されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • FDWCollateState: foreign_loc_cxt 内で照合順序の状態を管理するために使用される列挙型。
  • deparse_expr_cxt: SQL構築全体のコンテキストを保持する構造体。

失敗モードと境界条件

  • 照合順序が競合する場合、foreign_expr_walker 等の走査関数が false を返し、リモート実行不可と判定されます。
  • 構造体内の collation が InvalidOid の場合、照合順序に依存しない式として扱われます。

保守担当者向けメモ

  • 本構造体は、リモートサーバーとローカルサーバー間での照合順序の不一致を防ぐための重要な役割を担っています。
  • 照合順序の解決ロジックを変更する場合は、FDWCollateState の定義と併せて本構造体の利用箇所を確認してください。
