NewExplainState

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_state.c:60

目的

EXPLAIN コマンドの実行状態を保持する ExplainState 構造体をメモリ上に確保し、初期化を行うコンストラクタ関数。

シグネチャまたは形状

ExplainState *NewExplainState(void)

入力と出力

  • **入力**: なし
  • **出力**: 初期化済みの ExplainState 構造体へのポインタ。

前提条件と不変条件

  • palloc0 を使用してメモリを確保するため、PostgreSQL のメモリコンテキストが有効であること。
  • 返される ExplainState は、costs フィールドが true に設定され、出力用の StringInfo バッファが準備された状態であること。

アルゴリズムまたは振る舞い

1. palloc0 を呼び出し、ExplainState 構造体サイズ分のメモリをゼロクリアして確保する。 2. es->costs を true に設定する。 3. makeStringInfo() を呼び出し、出力用バッファを初期化して es->str に格納する。 4. 初期化済みのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/auto_explain/auto_explain.c:394
  • src/backend/commands/explain.c:179
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • makeStringInfo (出力バッファ生成)

関連データ構造

  • ExplainState: 本関数が生成・初期化する主要な状態管理構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させ、処理が中断される。

保守担当者向けメモ

  • 本関数で設定されるデフォルト値は costs = true のみであり、他のフィールドは palloc0 によるゼロ初期化に依存している。将来的にデフォルト値の変更が必要な場合は、ここを修正すること。
