XLogSource

定義

  • 種別: enum
  • 定義箇所: src/backend/access/transam/xlogrecovery.c:211

目的

WAL（Write-Ahead Log）の読み込み元ソースを識別するための列挙型です。リカバリプロセスにおいて、どの手段でWALデータを取得すべきかを指定・追跡するために使用されます。

シグネチャまたは形状

typedef enum
{
    XLOG_FROM_ANY = 0,      /* 任意のソースから読み込みを要求 */
    XLOG_FROM_ARCHIVE,      /* restore_command を使用して復元 */
    XLOG_FROM_PG_WAL,       /* pg_wal ディレクトリ内の既存ファイル */
    XLOG_FROM_STREAM,       /* プライマリからのストリーミング */
} XLogSource;

入力と出力

  • 本シンボル自体は列挙型であり、関数ではありません。
  • 主に XLogFileReadAnyTLI などの関数において、読み込み元を指定する引数として入力されます。
  • currentSource や readSource といった静的変数に格納され、現在の読み込み状態を出力（保持）します。

前提条件と不変条件

  • XLOG_FROM_ANY は、読み込み元を限定しない初期状態や要求状態として使用されます。
  • リカバリ処理中、currentSource は現在WALを読み込んでいるソースを保持し、状態遷移に応じて更新されます。

アルゴリズムまたは振る舞い

  • リカバリプロセスにおいて、WALの読み込み元を切り替える際の識別子として機能します。
  • XLogFileReadAnyTLI 関数は、この列挙値に基づいて、アーカイブ、pg_wal、ストリーミングのいずれからデータを取得するかを決定します。

呼び出し元と呼び出し先

  • **参照元**:
  • src/backend/access/transam/xlogrecovery.c:
  • readSource, currentSource, XLogReceiptSource の型定義として使用。
  • XLogFileReadAnyTLI の引数として使用。
  • oldSource 変数の型として使用。
  • **呼び出し先**:
  • 列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • XLogRecoveryCtlData: リカバリ状態を保持する共有メモリ構造体（関連する状態管理に使用）。

失敗モードと境界条件

  • 該当なし（列挙型のため）。

保守担当者向けメモ

  • 新しいWAL取得ソースを追加する場合は、本列挙型に値を追加し、XLogFileReadAnyTLI 等の読み込みロジックを更新する必要があります。
  • currentSource の値がどのように遷移するかは、xlogrecovery.c 内のリカバリループの挙動に直結するため、デバッグ時にはこの変数の追跡が有効です。
