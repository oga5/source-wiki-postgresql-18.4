be_gssapi_get_princ

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-gssapi.c:768

目的

GSSAPI認証済み接続に関連付けられたプリンシパル名（文字列）を取得します。

シグネチャまたは形状

const char *be_gssapi_get_princ(Port *port);

入力と出力

  • **入力**: Port *port（接続情報を保持する構造体へのポインタ）
  • **出力**: 成功した場合はプリンシパル名を示す const char * を返し、失敗または未認証の場合は NULL を返します。

前提条件と不変条件

  • port が有効なポインタであること。
  • port->gss が初期化されている（GSSAPIセッションが確立されている）場合にのみ、有効なプリンシパル名が返されます。

アルゴリズムまたは振る舞い

1. port が NULL であるか、または port->gss が NULL であるかを確認します。 2. いずれかが真であれば NULL を返します。 3. 上記以外の場合、port->gss->princ に格納されているプリンシパル名へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/backend_status.c:425
  • src/backend/utils/init/postinit.c:291
  • **呼び出し先**:
  • 本関数は他の関数を呼び出しません。

関連データ構造

  • Port: 接続状態を管理する構造体。gss メンバを通じてGSSAPIコンテキスト情報にアクセスします。

失敗モードと境界条件

  • port が NULL の場合、安全に NULL を返します。
  • GSSAPI認証が完了していない、またはGSSAPIを使用していない接続に対して呼び出された場合、port->gss が NULL であるため NULL を返します。

保守担当者向けメモ

  • 本関数は単なるアクセサであり、内部状態の変更は行いません。
  • 返される文字列の生存期間は、対応する Port 構造体の生存期間に依存します。
