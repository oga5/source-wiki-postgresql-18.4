ltree_ge

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:115

目的

2つの ltree 型の値が「以上（greater than or equal to）」の関係にあるかを判定し、SQL レベルで利用可能な真偽値を返します。

シグネチャまたは形状

Datum ltree_ge(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの ltree 型の Datum。
  • **出力**: bool 型（Datum 形式）。第1引数が第2引数以上であれば true、そうでなければ false を返します。

前提条件と不変条件

  • RUNCMP マクロが定義されている環境で実行される必要があります。
  • RUNCMP は内部的に ltree_compare を呼び出し、比較結果を res 変数に格納することを前提としています。

アルゴリズムまたは振る舞い

1. RUNCMP マクロを実行し、引数として渡された2つの ltree を比較します。 2. 比較結果 res が 0 以上（res >= 0）であるかを確認します。 3. PG_RETURN_BOOL を使用して、比較結果が真であれば true を、偽であれば false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の SQL エンジン（演算子 >= の実行時）。
  • **呼び出し先**: RUNCMP マクロ（内部で ltree_compare を呼び出す）。
  • **補足**: ソースコード上では PG_FUNCTION_INFO_V1(ltree_ge) による登録が確認されています。

関連データ構造

  • ltree: 比較対象となるラベルパス構造体。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • 入力値が ltree として不正な形式である場合、PostgreSQL のメモリ管理または型チェック機構によりエラーが発生する可能性があります。
  • RUNCMP マクロの依存先である ltree_compare が予期せぬ値を返した場合の挙動は、比較ロジックに依存します。

保守担当者向けメモ

  • 本関数は ltree_op.c 内の他の比較演算子（ltree_lt, ltree_gt 等）と同様のパターンで実装されています。ロジックを変更する場合は、ltree_compare の仕様と整合性を保つ必要があります。
