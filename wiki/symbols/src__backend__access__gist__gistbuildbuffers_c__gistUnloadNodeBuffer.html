gistUnloadNodeBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuildbuffers.c:245

目的

メモリ上に保持されている GiST ノードバッファの内容を一時ファイルへ退避（スワップアウト）し、メモリを解放する。

シグネチャまたは形状

static void gistUnloadNodeBuffer(GISTBuildBuffers *gfbb, GISTNodeBuffer *nodeBuffer)

入力と出力

  • **入力**:
  • gfbb: バッファリングビルド全体の管理構造体。
  • nodeBuffer: 退避対象のノードバッファ。
  • **出力**: なし（副作用として一時ファイルへの書き込みとメモリ解放を行う）。

前提条件と不変条件

  • nodeBuffer->pageBuffer が存在する場合のみ書き込み処理が実行される。
  • gistBuffersGetFreeBlock を介して一時ファイル内の有効なブロック番号が割り当てられる必要がある。

アルゴリズムまたは振る舞い

1. nodeBuffer->pageBuffer が非 NULL かを確認する。 2. gistBuffersGetFreeBlock(gfbb) を呼び出し、一時ファイル内の書き込み先ブロック番号を取得する。 3. WriteTempFileBlock を使用して、メモリ上のページデータを一時ファイルへ書き出す。 4. pfree を呼び出してメモリ上のページバッファを解放し、nodeBuffer->pageBuffer を NULL に設定する。 5. nodeBuffer->pageBlocknum に書き込み先のブロック番号を保存し、ディスク上の位置を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuildbuffers.c 内の gistUnloadNodeBuffers などから呼び出される。
  • **呼び出し先**:
  • gistBuffersGetFreeBlock
  • WriteTempFileBlock
  • pfree

関連データ構造

  • GISTBuildBuffers: ビルド全体の状態を保持する構造体。
  • GISTNodeBuffer: 個別のノードバッファの状態（メモリ上のページやディスク上のブロック番号）を保持する構造体。

失敗モードと境界条件

  • WriteTempFileBlock 内部でエラーが発生した場合、PostgreSQL のエラーハンドリング機構により elog(ERROR) が発行され、処理が中断される。

保守担当者向けメモ

  • 本関数はメモリ不足時に呼び出される重要なパスである。
  • nodeBuffer->pageBuffer を解放した後は、再度アクセスする際に gistLoadNodeBuffer を用いてディスクから読み戻す必要がある点に注意すること。
