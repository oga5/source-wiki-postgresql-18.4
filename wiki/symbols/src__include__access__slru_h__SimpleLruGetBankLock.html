SimpleLruGetBankLock

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/access/slru.h:174

目的

指定されたページ番号（pageno）に対応する SLRU バンクの軽量ロック（LWLock）を取得する。

シグネチャまたは形状

static inline LWLock *SimpleLruGetBankLock(SlruCtl ctl, int64 pageno)

入力と出力

  • **入力**:
  • ctl: SLRU 制御構造体（SlruCtl）。
  • pageno: ターゲットとなるページ番号（int64）。
  • **出力**:
  • 該当するバンクの LWLock へのポインタ。

前提条件と不変条件

  • ctl が有効な SlruCtl 構造体を指していること。
  • ctl->nbanks が 0 より大きいこと（剰余演算に使用されるため）。

アルゴリズムまたは振る舞い

1. pageno を ctl->nbanks で除算した剰余を計算し、対象のバンク番号（bankno）を特定する。 2. ctl->shared 内の bank_locks 配列から、該当するインデックスのロック構造体のアドレスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/clog.c: ページ読み込み、状態確認、ロック解放等の各処理で頻繁に使用。
  • src/backend/access/transam/commit_ts.c: ページ読み込みおよびロック解放処理で使用。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない（インライン展開される）。

関連データ構造

  • SlruCtl: SLRU インスタンスの制御構造体。
  • SlruSharedData: 共有メモリ上の状態管理構造体（bank_locks を保持）。
  • LWLock: PostgreSQL の軽量ロック構造体。

失敗モードと境界条件

  • ctl->nbanks が 0 の場合、ゼロ除算が発生する可能性がある。
  • pageno が負の値である場合、剰余演算の結果が負になる可能性があるため、呼び出し側で適切なページ番号が保証されている必要がある。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、インライン関数として定義されている。
  • バンク単位のロックは、SLRU の共有状態に対する競合を最小化するために使用される。I/O 実行時にはこのロックを解放する設計となっているため、ロックの保持期間には注意すること。
