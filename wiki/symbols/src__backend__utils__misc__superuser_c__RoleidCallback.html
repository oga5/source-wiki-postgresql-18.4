RoleidCallback

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/superuser.c:102

目的

システムカタログ pg_authid の変更を検知した際に、プロセスローカルなスーパーユーザー権限判定キャッシュを無効化する。

シグネチャまたは形状

static void RoleidCallback(Datum arg, int cacheid, uint32 hashvalue)

入力と出力

  • **入力**:
  • arg: コールバック登録時に渡されたデータ（本関数では使用されない）。
  • cacheid: 無効化対象のキャッシュID。
  • hashvalue: 無効化対象のハッシュ値。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数は CacheRegisterSyscacheCallback を介してシステムキャッシュの無効化イベントに登録されることを前提とする。
  • 実行後、静的変数 last_roleid が InvalidOid にリセットされる不変条件を持つ。

アルゴリズムまたは振る舞い

1. pg_authid の変更イベントが発生すると、本関数が呼び出される。 2. 静的変数 last_roleid に InvalidOid を代入する。これにより、次回の権限判定時にキャッシュが強制的に再構築される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/syscache.c のキャッシュ無効化メカニズムから、AUTHOID キャッシュの変更時に呼び出される。
  • **呼び出し先**: 直接的な関数呼び出しはない。

関連データ構造

  • last_roleid: 本関数によってリセットされるプロセスローカルなキャッシュ変数。

失敗モードと境界条件

  • 本関数自体はエラーを発生させない。
  • キャッシュの無効化はプロセスローカルであるため、他のバックエンドプロセスには影響を与えない。

保守担当者向けメモ

  • 本関数は superuser_arg が初回実行時に CacheRegisterSyscacheCallback を通じて登録するコールバックである。
  • ロールのスーパーユーザー権限が変更された可能性がある場合にのみ実行されるため、頻繁な呼び出しは発生しない。
