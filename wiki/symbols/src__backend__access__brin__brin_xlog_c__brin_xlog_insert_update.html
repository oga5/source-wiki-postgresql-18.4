brin_xlog_insert_update

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_xlog.c:45

目的

BRINインデックスにおける挿入および更新操作のWALリプレイを実行し、インデックスページへのタプル追加とREVMAP（範囲マップ）の更新を統合的に管理する。

シグネチャまたは形状

static void brin_xlog_insert_update(XLogReaderState *record, xl_brin_insert *xlrec)

入力と出力

  • **入力**:
  • record: リプレイ対象のWALレコード状態。
  • xlrec: 挿入/更新操作の詳細情報を含む構造体（ヒープブロック番号、オフセット番号、範囲あたりのページ数など）。
  • **出力**: なし（副作用としてバッファの更新とディスクへの反映を行う）。

前提条件と不変条件

  • xlrec->heapBlk と挿入される BrinTuple の bt_blkno が一致していること。
  • REVMAPの更新時には、対象のインデックスページが有効なバッファとして取得可能であること。

アルゴリズムまたは振る舞い

1. **ページ初期化**: XLOG_BRIN_INIT_PAGE フラグがある場合、brin_page_init を呼び出しページを初期化する。 2. **インデックスページ更新**:

  • XLogReadBufferForRedo により対象ページを取得。
  • BLK_NEEDS_REDO の場合、PageAddItem を用いてタプルをページに追加し、LSNを更新してバッファをダーティにする。

    • 3. **REVMAP更新**:

  • インデックスページに対応するREVMAPページを読み込む。
  • brinSetHeapBlockItemptr を呼び出し、ヒープブロックとインデックス内のタプル位置を関連付ける。
  • LSNを更新し、バッファをダーティにする。

    • 4. **終了処理**: 各ステップで取得したバッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • brin_xlog_insert (src/backend/access/brin/brin_xlog.c:128)
  • brin_xlog_update (src/backend/access/brin/brin_xlog.c:160)
  • **呼び出し先**:
  • XLogInitBufferForRedo, XLogReadBufferForRedo, BufferGetPage, brin_page_init, PageAddItem, PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer, brinSetHeapBlockItemptr 等。

関連データ構造

  • xl_brin_insert: WALレコードのデータ構造。
  • BrinTuple: インデックスに格納されるタプル形式。
  • Buffer: PostgreSQLのバッファ管理構造。

失敗モードと境界条件

  • PageGetMaxOffsetNumber(page) + 1 < offnum: ページ内のオフセット番号が不正な場合に PANIC を発生させる。
  • PageAddItem が InvalidOffsetNumber を返した場合、タプル追加失敗として PANIC を発生させる。

保守担当者向けメモ

  • 本関数はREVMAPの更新を伴うため、インデックスページとREVMAPページの整合性を維持する責任がある。
  • FSM（空き領域マップ）の更新は本関数内では行われない（ソースコード内の XXX コメント参照）。
