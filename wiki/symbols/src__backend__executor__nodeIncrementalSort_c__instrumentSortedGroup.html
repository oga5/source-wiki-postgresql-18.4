instrumentSortedGroup

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:126

目的

インクリメンタルソートの実行統計（EXPLAIN ANALYZE 用）を収集し、指定された IncrementalSortGroupInfo 構造体に集計結果を記録する。

シグネチャまたは形状

static void instrumentSortedGroup(IncrementalSortGroupInfo *groupInfo, Tuplesortstate *sortState)

入力と出力

  • **入力**:
  • groupInfo: 統計情報を格納する IncrementalSortGroupInfo 構造体へのポインタ。
  • sortState: 統計取得元の Tuplesortstate 構造体へのポインタ。
  • **出力**: なし（groupInfo の各フィールドが更新される）。

前提条件と不変条件

  • groupInfo および sortState は有効なメモリ領域を指している必要がある。
  • sortState は tuplesort_get_stats が呼び出し可能な状態であること。

アルゴリズムまたは振る舞い

1. groupInfo->groupCount をインクリメントする。 2. tuplesort_get_stats を呼び出し、現在のソート状態から TuplesortInstrumentation を取得する。 3. sort_instr.spaceType に基づき、メモリまたはディスクの使用量を判定する。

  • SORT_SPACE_TYPE_DISK: totalDiskSpaceUsed に加算し、maxDiskSpaceUsed を更新する。
  • SORT_SPACE_TYPE_MEMORY: totalMemorySpaceUsed に加算し、maxMemorySpaceUsed を更新する。

    • 4. groupInfo->sortMethods に対して、今回のソートで使用されたメソッド（sort_instr.sortMethod）をビットマスクとして追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeIncrementalSort.c 内の複数箇所（node->shared_info または node->incsort_info を引数として呼び出される）。
  • **呼び出し先**:
  • tuplesort_get_stats（src/backend/utils/sort/tuplesort.c で定義）。

関連データ構造

  • IncrementalSortGroupInfo: 実行統計を保持する構造体。
  • Tuplesortstate: ソート処理の状態を保持する構造体。
  • TuplesortInstrumentation: ソートの統計情報（使用領域、メソッド等）を保持する構造体。

失敗モードと境界条件

  • sortState が NULL の場合、tuplesort_get_stats 内でクラッシュする可能性がある。
  • spaceType が定義外の値である場合、統計情報の更新がスキップされる。

保守担当者向けメモ

  • 本関数は EXPLAIN ANALYZE の出力精度に直結する。統計情報の項目を追加する場合は、IncrementalSortGroupInfo 構造体の定義と併せて修正すること。
  • パラレルクエリ環境下では、共有メモリ上の統計情報に対して本関数が呼び出されるため、スレッドセーフなアクセスが保証されているか確認が必要。
