extract_or_clause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/orclauses.c:155

目的

結合条件に含まれるOR条件を解析し、特定のベースリレーション（RelOptInfo）のみを参照する制限条件（RestrictInfo）を再帰的に抽出・再構築する。

シグネチャまたは形状

static Expr *extract_or_clause(RestrictInfo *or_rinfo, RelOptInfo *rel)

入力と出力

  • **入力**:
  • or_rinfo: 抽出対象のOR条件を含む RestrictInfo。
  • rel: 制限条件の適用対象となるベースリレーション。
  • **出力**:
  • 抽出に成功した場合、対象リレーションのみに依存する Expr ツリー。
  • 抽出できない場合、NULL。

前提条件と不変条件

  • or_rinfo->orclause は is_orclause を満たす必要がある。
  • 抽出された条件は、元の RestrictInfo から剥離され、新しい Expr として構築される。
  • AND/ORのフラット構造を維持するため、OR直下にORが配置されないよう調整される。

アルゴリズムまたは振る舞い

1. 入力されたOR条件の各アーム（引数）を走査する。 2. 各アームがAND条件の場合、その内部の各要素を評価する。

  • ネストされたOR条件であれば再帰的に extract_or_clause を呼び出す。
  • is_safe_restriction_clause_for が真であれば、その条件を抽出リストに追加する。

    • 3. 各アームが単一の条件であれば、同様に安全性を確認して抽出する。 4. いずれかのアームから条件を抽出できなかった場合、そのOR条件全体は利用不可と判断し NULL を返す。 5. 抽出された条件群を make_ands_explicit で結合し、最終的に make_orclause でラップして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/orclauses.c 内の関数（自身を再帰呼び出しする箇所を含む）。
  • **呼び出し先**:
  • is_orclause, is_andclause, restriction_is_or_clause, is_safe_restriction_clause_for
  • make_ands_explicit, make_orclause
  • extract_or_clause (再帰)

関連データ構造

  • RestrictInfo: 制限条件を保持する構造体。
  • RelOptInfo: ベースリレーションの情報を保持する構造体。
  • BoolExpr: OR/AND条件を表現するノード。

失敗モードと境界条件

  • OR条件のいずれかのアームから条件を抽出できない場合、関数は NULL を返す。
  • 抽出された条件が1つのみで、かつそれがORノードである場合、AND/ORのフラット化のためにサブ条件が展開される。

保守担当者向けメモ

  • 本関数は RestrictInfo にキャッシュされた選択率を再利用せず、新しい Expr を構築する。これは結合条件と制限条件で選択率の計算ロジックが異なる可能性があるためである。
  • ネストされたOR条件を処理するため、再帰呼び出しが必須となっている。
