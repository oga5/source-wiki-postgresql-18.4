LogicalTapeImport

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/logtape.c:608

目的

並列ソートにおいて、ワーカープロセスが作成した物理ファイルをリーダープロセスの LogicalTapeSet に統合し、論理テープとしてインポートする。

シグネチャまたは形状

LogicalTape *LogicalTapeImport(LogicalTapeSet *lts, int worker, TapeShare *shared)

入力と出力

  • **入力**:
  • lts: 統合先の LogicalTapeSet 構造体。
  • worker: インポート対象のワーカーID。
  • shared: ワーカーのテープ情報（開始ブロック番号等）を保持する TapeShare 構造体。
  • **出力**:
  • インポートされた状態を反映した LogicalTape 構造体へのポインタ。

前提条件と不変条件

  • lts が有効な LogicalTapeSet であること。
  • shared に格納された firstblocknumber が、インポート対象のファイルと整合していること。

アルゴリズムまたは振る舞い

1. ltsCreateTape を呼び出し、新しい LogicalTape 構造体を確保する。 2. ワーカーのファイル名（worker ID）から BufFile を読み取り専用で開く。 3. lts->pfile が未設定であればそれを新しいファイルで初期化し、既に存在する場合は BufFileAppend を使用して既存のファイルセットに連結する。 4. 連結時のブロックオフセットを lt->offsetBlockNumber に記録する。 5. インポートしたファイルのサイズに基づき、lts->nBlocksAllocated および lts->nBlocksWritten を更新する。 6. 連結によって生じた「穴（hole）」のブロック数を計算し、lts->nHoleBlocks に加算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/sort/tuplesort.c:3118
  • src/backend/utils/sort/logtape.c:546（内部的なループ処理）
  • **呼び出し先**:
  • ltsCreateTape
  • BufFileOpenFileSet
  • BufFileSize
  • BufFileAppend

関連データ構造

  • LogicalTapeSet: テープセット全体を管理。
  • LogicalTape: 個別の論理テープ状態。
  • TapeShare: ワーカー間の共有情報。
  • BufFile: 物理ファイルI/Oの抽象化。

失敗モードと境界条件

  • ファイルのオープン失敗（BufFileOpenFileSet）は、基盤となる BufFile のエラーハンドリングに依存する。
  • MaxAllocSize を超えるファイルサイズの場合、lt->max_size は MaxAllocSize に制限される。

保守担当者向けメモ

  • 本関数は並列ソートにおける物理ファイルの連結を担うため、ブロック番号のオフセット計算（offsetBlockNumber）が正確に行われていることが重要である。
  • lts->nHoleBlocks の更新は、後の物理ブロック数計算の整合性に直結するため、変更時は注意が必要。
