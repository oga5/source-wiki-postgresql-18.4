printTextRule

定義

  • 種別: enum
  • 定義箇所: src/include/fe_utils/print.h:52

目的

テーブル形式の出力において、行の種類（ヘッダー、データ、境界線など）を識別し、適切な罫線描画文字を選択するためのコンテキストを提供します。

シグネチャまたは形状

typedef enum printTextRule
{
    PRINT_RULE_TOP,
    PRINT_RULE_MIDDLE,
    PRINT_RULE_BOTTOM,
    PRINT_RULE_DATA,
} printTextRule;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: printTextLineFormat 構造体の配列インデックスとして使用されます。

前提条件と不変条件

  • printTextLineFormat 構造体の lrule 配列（サイズ4）のインデックスとして使用されるため、値は常に 0 から 3 の範囲内である必要があります。

アルゴリズムまたは振る舞い

  • PRINT_RULE_TOP: テーブルの最上部水平線に使用されます。
  • PRINT_RULE_MIDDLE: データ行間の水平線に使用されます。
  • PRINT_RULE_BOTTOM: テーブルの最下部水平線に使用されます。
  • PRINT_RULE_DATA: データ行自体に使用されます（この場合、水平線は使用されません）。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な使用箇所）:
  • src/fe_utils/print.c:594
  • src/fe_utils/print.c:1230
  • src/fe_utils/print.c:1589
  • 呼び出し先: なし（列挙型のため）

関連データ構造

  • printTextLineFormat: lrule メンバがこの列挙型をインデックスとして保持します。

失敗モードと境界条件

  • 配列のインデックスとして使用されるため、範囲外の値が渡された場合、メモリ破壊や未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しい行タイプを追加する場合は、printTextLineFormat の lrule 配列サイズと整合性を取る必要があります。
  • src/fe_utils/print.c 内の各関数で、この列挙値がどのように描画ロジックに渡されているかを確認してください。
