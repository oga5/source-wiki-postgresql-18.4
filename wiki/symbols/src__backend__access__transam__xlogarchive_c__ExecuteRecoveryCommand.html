ExecuteRecoveryCommand

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:294

目的

リカバリ処理やアーカイブ管理に関連する外部シェルコマンド（recovery_end_command や archive_cleanup_command 等）を安全に構築・実行します。

シグネチャまたは形状

void ExecuteRecoveryCommand(const char *command, const char *commandName,
                           bool failOnSignal, uint32 wait_event_info)

入力と出力

  • command: 実行するコマンド文字列（%r プレースホルダーを含む）。
  • commandName: ログ出力に使用するコマンドの識別名。
  • failOnSignal: true の場合、コマンドがシグナルによって終了した際に FATAL エラーを発生させる。
  • wait_event_info: 待機イベントの識別子。
  • 出力: なし（副作用として外部コマンドを実行）。

前提条件と不変条件

  • command および commandName は非 NULL であること。
  • 実行環境において system() 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. GetOldestRestartPoint を呼び出し、最新の再起動ポイントの情報を取得する。 2. XLogFileName を用いて、再起動ポイントに対応するWALファイル名を生成する。 3. replace_percent_placeholders を使用し、コマンド文字列内の %r を上記WALファイル名に置換して実行コマンドを構築する。 4. fflush(NULL) を実行してバッファをフラッシュする。 5. pgstat_report_wait_start を呼び出し、待機イベントを開始する。 6. system() を呼び出し、構築したコマンドを実行する。 7. pgstat_report_wait_end を呼び出し、待機イベントを終了する。 8. コマンドの終了コード rc を確認し、0 以外の場合は wait_result_to_str を用いてエラーログを出力する。failOnSignal が有効かつシグナルによる終了であれば FATAL として処理を中断する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xlog.c: recoveryEndCommand および archiveCleanupCommand の実行時に呼び出される。
  • 呼び出し先:
  • GetOldestRestartPoint, XLByteToSeg, XLogFileName: 再起動ポイント情報の取得とファイル名生成。
  • replace_percent_placeholders: コマンド文字列の置換。
  • pgstat_report_wait_start/end: 統計情報の更新。
  • system: 外部コマンドの実行。
  • wait_result_is_any_signal, wait_result_to_str: 終了コードの解析。

関連データ構造

  • XLogSegNo, XLogRecPtr, TimeLineID: WALのセグメント番号やポインタ、タイムラインIDを管理する型。

失敗モードと境界条件

  • コマンド実行が失敗（rc != 0）した場合、警告または致命的エラーが記録される。
  • コマンドがシグナルで中断された場合、failOnSignal が true であればリカバリプロセス全体が FATAL で停止する。

保守担当者向けメモ

  • 本関数は system() を使用するため、シェルインジェクションのリスクを考慮し、command 引数には信頼できる設定値のみを渡すこと。
  • replace_percent_placeholders による置換ロジックは common/percentrepl.h に依存している。
