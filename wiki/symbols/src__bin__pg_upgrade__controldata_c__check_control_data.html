check_control_data

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/controldata.c:697

目的

PostgreSQLのアップグレード前後における、新旧クラスタの物理的な互換性を検証します。

シグネチャまたは形状

void check_control_data(ControlData *oldctrl, ControlData *newctrl)

入力と出力

  • 入力:
  • oldctrl: 旧クラスタの制御データ構造体へのポインタ。
  • newctrl: 新クラスタの制御データ構造体へのポインタ。
  • 出力: なし（検証失敗時は pg_fatal を呼び出しプロセスを終了します）。

前提条件と不変条件

  • 比較対象の各パラメータ（アライメント、ブロックサイズ、WAL設定等）は、0であってはならず、かつ新旧で一致している必要があります。
  • チェックサムバージョンについては、両クラスタで有効/無効の状態が一致しているか、あるいは両方とも有効な場合はバージョンが一致している必要があります。

アルゴリズムまたは振る舞い

1. 以下の項目について、値が0であるか、または新旧で不一致がある場合にエラーを報告します。

  • アライメント (align)
  • ブロックサイズ (blocksz)
  • 最大リレーションセグメントサイズ (largesz)
  • WALブロックサイズ (walsz)
  • WALセグメントサイズ (walseg)
  • 最大識別子長 (ident)
  • 最大インデックス列数 (index)
  • 最大TOASTチャンクサイズ (toast)

    • 2. large_object チャンクサイズは、旧クラスタで0でない場合に限り比較を行います。 3. 日付/時刻の格納型 (date_is_int) が一致することを確認します。 4. データチェックサムのバージョン (data_checksum_version) を比較し、一方が有効で他方が無効な場合、または両方有効でバージョンが異なる場合にエラーを報告します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/check.c:910
  • 呼び出し先: pg_fatal（エラー発生時）

関連データ構造

  • ControlData: クラスタの制御情報を保持する構造体（src/bin/pg_upgrade/pg_upgrade.h で定義）。

失敗モードと境界条件

  • 各パラメータが0の場合、または新旧で値が異なる場合は pg_fatal により処理が中断されます。
  • 特にアライメントの不一致は、32bit/64bit環境の混在を示唆するエラーとして報告されます。

保守担当者向けメモ

  • float8_pass_by_value は本関数では比較されませんが、別のチェック関数 (check_for_isn_and_int8_passing_mismatch) で使用されます。
  • 将来的にチェックサムの有無が異なるクラスタ間でのアップグレードを許可する可能性があるため、その際はロジックの修正が必要です。
