detoast_attr_slice

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/detoast.c:204

目的

TOASTされたデータ、圧縮データ、間接参照、または展開済みオブジェクトを含む可変長データ（varlena）から、指定されたオフセットと長さの範囲（スライス）を効率的に抽出して復元する。

シグネチャまたは形状

struct varlena *detoast_attr_slice(struct varlena *attr, int32 sliceoffset, int32 slicelength)

入力と出力

  • **入力**:
  • attr: 処理対象の varlena 構造体ポインタ。
  • sliceoffset: 抽出を開始するオフセット（0以上）。
  • slicelength: 抽出する長さ（負の値は末尾までを意味する）。
  • **出力**:
  • 抽出されたデータを含む、新しく palloc された varlena 構造体。

前提条件と不変条件

  • sliceoffset が負の場合はエラーとなる。
  • 処理の過程で生成された一時的な中間データは、最終的に pfree される。
  • 外部データ（VARATT_EXTERNAL）が入力された場合、再帰的または段階的な解決を経て、最終的に非外部形式のデータに対してスライス処理が行われる。

アルゴリズムまたは振る舞い

1. **スライス範囲の計算**: slicelimit（オフセット+長さ）を算出する。整数オーバーフロー時は全データを取得する。 2. **形式に応じた解決**:

  • **外部データ**: toast_fetch_datum_slice を使用して必要な範囲を取得。圧縮されている場合は、解凍に必要な範囲を計算して取得する。
  • **間接参照**: 参照先に対して再帰的に detoast_attr_slice を呼び出す。
  • **展開済みデータ**: detoast_external_attr でフラット化する。

    • 3. **解凍**: 圧縮されている場合は toast_decompress_datum_slice または toast_decompress_datum で展開する。 4. **抽出**: 最終的なデータから memcpy を用いて指定範囲をコピーし、新しい varlena として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/detoast.c（再帰呼び出し）
  • src/backend/utils/fmgr/fmgr.c:1860
  • **呼び出し先**:
  • toast_fetch_datum_slice, toast_fetch_datum
  • detoast_external_attr
  • toast_decompress_datum_slice, toast_decompress_datum
  • pg_add_s32_overflow, pglz_maximum_compressed_size

関連データ構造

  • struct varlena: PostgreSQLの可変長データ型。
  • struct varatt_external: 外部TOASTデータのポインタ構造。
  • struct varatt_indirect: 間接参照データの構造。

失敗モードと境界条件

  • sliceoffset < 0: elog(ERROR) を発生させる。
  • sliceoffset がデータサイズを超える場合: 0バイトのデータを返す。
  • slicelength が大きすぎる場合: データの末尾までを抽出する。
  • 圧縮データ（特にLZ4）の場合、APIの制約により全データを取得する必要がある場合がある。

保守担当者向けメモ

  • 本関数はメモリを割り当てるため、呼び出し元での pfree が必須である。
  • VARATT_EXTERNAL_GET_POINTER マクロによるアライメント処理が重要である。
