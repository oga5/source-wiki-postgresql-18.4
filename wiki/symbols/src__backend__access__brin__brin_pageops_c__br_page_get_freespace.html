br_page_get_freespace

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_pageops.c:919

目的

BRINインデックスページにおいて、挿入可能な空き領域のサイズを安全に取得します。ページの状態を考慮し、退避中や非正規ページに対しては空き領域を0と見なすことで、不適切な書き込みを防止します。

シグネチャまたは形状

static Size br_page_get_freespace(Page page)

入力と出力

  • **入力**: Page page（対象となるBRINインデックスページ）
  • **出力**: Size（ページ内の利用可能な空き領域のバイト数。条件を満たさない場合は0）

前提条件と不変条件

  • 入力される page は、PostgreSQLの標準的なページ構造に従っている必要があります。
  • ページが BRIN_EVACUATE_PAGE フラグを保持している場合、または正規のBRINページ（BRIN_IS_REGULAR_PAGE）でない場合は、常に0を返します。

アルゴリズムまたは振る舞い

1. BRIN_IS_REGULAR_PAGE(page) マクロにより、対象ページが正規のBRINインデックスページであるかを確認します。 2. BrinPageFlags(page) を参照し、BRIN_EVACUATE_PAGE フラグが立っていないか確認します。 3. 上記の条件を両方満たす場合、PageGetFreeSpace(page) を呼び出して実際の空き領域サイズを返します。 4. 条件を満たさない場合は、即座に0を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin_pageops.c 内の各所（brin_doinsert やページ内の空き領域確認を行うロジックなど）から、ページへの書き込み可否を判断するために呼び出されます。
  • **呼び出し先**: PageGetFreeSpace（PostgreSQLの標準的なページ管理マクロ）。

関連データ構造

  • Page: PostgreSQLの標準的なページデータ型。
  • BRIN_EVACUATE_PAGE: ページ退避中であることを示すフラグ。

失敗モードと境界条件

  • **退避中のページ**: BRIN_EVACUATE_PAGE フラグが設定されているページに対しては、空き領域があっても0を返すことで、新たなタプルの挿入を抑制します。
  • **非正規ページ**: メタページやその他の特殊なページに対しては、常に0を返し、誤った書き込みを防ぎます。

保守担当者向けメモ

  • 本関数は、FSM（空き領域マップ）から取得したページが実際に書き込み可能かどうかを最終確認するガードとして機能します。
  • ページレイアウトやフラグ定義に変更を加える際は、本関数の条件判定ロジックが最新の仕様と整合しているか確認してください。
