pqTraceOutput_ErrorResponse

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:339

目的

PostgreSQL プロトコルにおける ErrorResponse メッセージの内容をデバッグ用トレース出力としてフォーマットし、指定されたストリームへ書き出します。

シグネチャまたは形状

static void pqTraceOutput_ErrorResponse(FILE *f, const char *message, int *cursor, bool regress)

入力と出力

  • f: 出力先のファイルポインタ。
  • message: 解析対象のメッセージデータを含むバッファ。
  • cursor: メッセージ内の現在位置を指すポインタ（更新される）。
  • regress: 回帰テストモードのフラグ。true の場合、タイムスタンプや可変フィールドなどの非決定的な情報を抑制します。
  • 戻り値: なし（void）。

前提条件と不変条件

  • f は有効な書き込み可能なファイルポインタである必要があります。
  • cursor はメッセージバッファ内の有効なオフセットを指している必要があります。

アルゴリズムまたは振る舞い

内部的に pqTraceOutputNR 関数を呼び出し、メッセージタイプ名として "ErrorResponse" を指定して共通の出力処理を実行します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-trace.c 内のメッセージディスパッチャ（pqTraceOutputMessage 等から間接的に呼び出されることが想定されます）。
  • 呼び出し先: pqTraceOutputNR。

関連データ構造

  • FILE: 標準ライブラリのファイルストリーム構造体。

失敗モードと境界条件

  • f が NULL の場合、標準ライブラリのファイル操作関数と同様に未定義の動作を引き起こす可能性があります。
  • メッセージバッファの終端を超えて cursor が進むような不正なメッセージデータが渡された場合、pqTraceOutputNR 内部での境界チェックに依存します。

保守担当者向けメモ

  • 本関数は単なるラッパーです。出力形式の変更や詳細なフィールド解析が必要な場合は、pqTraceOutputNR の実装を確認するか、本関数内で個別のフィールドデコード処理を追加してください。
