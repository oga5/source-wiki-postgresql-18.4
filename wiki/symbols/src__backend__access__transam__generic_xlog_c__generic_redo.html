generic_redo

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/generic_xlog.c:477

目的

リカバリ処理時に、WALレコードに記録されたページ差分（デルタ）を適用し、ページの状態を再現する。

シグネチャまたは形状

void generic_redo(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（解析対象のWALレコード）
  • **出力**: なし（副作用としてバッファの更新を行う）

前提条件と不変条件

  • record 内のブロックID数が MAX_GENERIC_XLOG_PAGES を超えないこと。
  • ページ内の pd_lower から pd_upper までの領域（ホール）は、WAL適用後にゼロクリアされる必要がある。

アルゴリズムまたは振る舞い

1. record に含まれる各ブロックを順次処理する。 2. XLogReadBufferForRedo を呼び出し、対象バッファを読み込む。 3. BLK_NEEDS_REDO が返された場合：

  • XLogRecGetBlockData でデルタを取得し、applyPageRedo でページに適用する。
  • ページヘッダーの pd_lower から pd_upper までの領域を memset でゼロクリアする。
  • ページのLSNをレコードの終了位置（record->EndRecPtr）に更新し、バッファをダーティにする。

    • 4. 全ブロックの処理完了後、ロックされていた全てのバッファを UnlockReleaseBuffer で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: リカバリプロセス（src/include/access/rmgrlist.h にて RM_GENERIC_ID のリソースマネージャとして登録されており、リカバリ時に呼び出される）。
  • **呼び出し先**: XLogRecMaxBlockId, XLogRecHasBlockRef, XLogReadBufferForRedo, BufferGetPage, XLogRecGetBlockData, applyPageRedo, PageSetLSN, MarkBufferDirty, BufferIsValid, UnlockReleaseBuffer。

関連データ構造

  • XLogReaderState: WALレコードの読み込み状態を保持。
  • Buffer: ページを保持するバッファ。
  • PageHeader: ページ構造のヘッダー情報。

失敗モードと境界条件

  • XLogRecMaxBlockId(record) >= MAX_GENERIC_XLOG_PAGES の場合、アサーションエラーが発生する。
  • 対象ブロックが存在しない場合はスキップされる。

保守担当者向けメモ

  • 本関数はリソースマネージャのRedoルーチンとして機能する。
  • ページ内のホール（pd_lower〜pd_upper）のゼロクリアは、異なる環境間でのページ再構築時にゴミデータによる不一致を防ぐために必須である。
