apply_tlist_labeling

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/tlist.c:326

目的

ソースターゲットリスト (src_tlist) からメタデータ情報を抽出し、対応する宛先ターゲットリスト (dest_tlist) の各 TargetEntry にコピーして適用する。

シグネチャまたは形状

void apply_tlist_labeling(List *dest_tlist, List *src_tlist)

入力と出力

  • **入力**:
  • dest_tlist: ラベルを適用されるターゲットリスト。
  • src_tlist: ラベルのコピー元となるターゲットリスト。
  • **出力**: なし（dest_tlist を直接変更する副作用を持つ）。

前提条件と不変条件

  • dest_tlist と src_tlist のリスト長は等しくなければならない（Assert により検証）。
  • 両リスト内の各 TargetEntry は、同じ順序で対応する resno を持たなければならない（Assert により検証）。

アルゴリズムまたは振る舞い

1. forboth マクロを使用して、dest_tlist と src_tlist を同時に走査する。 2. 各ペアの TargetEntry について、以下のメタデータを src_tle から dest_tle へコピーする。

  • resname (列名)
  • ressortgroupref (ソート/グループ化参照)
  • resorigtbl (元のテーブル OID)
  • resorigcol (元の列番号)
  • resjunk (ジャンクフラグ)

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c
  • src/backend/optimizer/plan/setrefs.c
  • **呼び出し先**:
  • なし（PostgreSQL 内部のリスト操作関数 lfirst を使用）。

関連データ構造

  • TargetEntry: ターゲットリストの各要素を構成する構造体。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • リスト長が一致しない場合、アサーションエラーが発生する。
  • resno が一致しない場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は、プランナが生成した新しいターゲットリストに対して、元のクエリで定義されたメタデータ（列名やソート情報など）を再適用するために使用される。
  • resno の一致を前提としているため、リストの順序を入れ替えるような操作の直後に呼び出す場合は注意が必要である。
