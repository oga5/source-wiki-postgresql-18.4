stmtCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:28

目的

ECPGの自動プリペア機能において、キャッシュされたSQLステートメントのメタデータと実行状態を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	int			lineno;
	char		stmtID[STMTID_SIZE];
	char	   *ecpgQuery;
	long		execs;
	const char *connection;
} stmtCacheEntry;

入力と出力

本構造体はデータ保持用であり、直接的な関数入出力は持ちません。各フィールドは以下の通りです。

  • lineno: 定義行番号。
  • stmtID: ステートメント識別子。
  • ecpgQuery: キャッシュ対象のSQLクエリ文字列。
  • execs: 実行回数（キャッシュ置換アルゴリズムの指標として使用）。
  • connection: 関連付けられた接続名。

前提条件と不変条件

  • stmtCacheEntries 配列の一部として管理されます。
  • execs フィールドは、キャッシュの置換対象を決定する際のLRUに近い指標として機能します。

アルゴリズムまたは振る舞い

  • ecpg_auto_prepare がクエリをハッシュ化し、キャッシュミス時に本構造体のインスタンスを生成して stmtCacheEntries に追加します。
  • キャッシュヒット時は execs がカウントアップされます。
  • キャッシュが満杯の場合、execs が最も少ないエントリが置換対象となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/ecpglib/prepare.c 内のキャッシュ管理関数（ecpg_auto_prepare 等）が本構造体をインスタンス化および参照します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • stmtCacheEntries: 本構造体の配列（ハッシュテーブルとして機能）。
  • prepared_statement: 接続オブジェクトにリンクされるプリペアドステートメントの管理構造体。

失敗モードと境界条件

  • メモリ不足: ecpg_alloc による確保失敗時にエラーが発生する可能性があります。
  • キャッシュ衝突: バケットあたりのエントリ数（stmtCacheEntPerBucket = 8）を超過した場合、置換アルゴリズムが動作します。

保守担当者向けメモ

  • ecpg_alloc / ecpg_free を使用してメモリ管理を行っているため、キャッシュ破棄時のメモリリークに注意してください。
  • stmtID のサイズは STMTID_SIZE 定数に依存します。
