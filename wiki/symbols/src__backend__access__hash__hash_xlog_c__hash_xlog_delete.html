hash_xlog_delete

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash_xlog.c:860

目的

ハッシュインデックスにおけるタプル削除操作のWAL（Write-Ahead Log）リプレイを実行し、バッファ上のページ状態を復元する。

シグネチャまたは形状

static void hash_xlog_delete(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファの状態を更新）。

前提条件と不変条件

  • リプレイ対象のページに対して、スキャンとの競合を防ぐためのクリーンアップロック（Cleanup Lock）を保持する必要がある。
  • xldata->is_primary_bucket_page が真の場合、対象ページ自体にクリーンアップロックをかける。偽の場合は、まずプライマリバケットページにクリーンアップロックを確保してから対象ページを読み込む。

アルゴリズムまたは振る舞い

1. WALレコードから xl_hash_delete 構造体を取得する。 2. XLogReadBufferForRedoExtended を使用して、必要なページ（プライマリバケットおよび対象ページ）をバッファに読み込み、クリーンアップロックを確保する。 3. XLogRedoAction が BLK_NEEDS_REDO の場合、以下の処理を行う：

  • XLogRecGetBlockData で削除対象のオフセット番号リストを取得する。
  • PageIndexMultiDelete を呼び出し、ページ上の該当タプルを削除する。
  • xldata->clear_dead_marking が真であれば、ページヘッダーの hasho_flag から LH_PAGE_HAS_DEAD_TUPLES フラグを解除する。
  • ページのLSNを更新し、MarkBufferDirty を呼び出してバッファをダーティ状態にする。

    • 4. 最後に、取得したすべてのバッファのロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: hash_redo（src/backend/access/hash/hash_xlog.c 内のディスパッチャ）。
  • **呼び出し先**:
  • XLogRecGetData, XLogRecGetBlockData
  • XLogReadBufferForRedoExtended, XLogReadBufferForRedo
  • BufferGetPage, PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer
  • PageIndexMultiDelete, HashPageGetOpaque

関連データ構造

  • xl_hash_delete: 削除操作のメタデータ（プライマリページフラグ、デッドタプルクリアフラグ等）を保持する構造体。
  • HashPageOpaque: ハッシュインデックスページのメタデータ領域。

失敗モードと境界条件

  • action が BLK_NEEDS_REDO 以外の場合、ページ更新はスキップされる。
  • 削除対象のオフセットリストが空（len == 0）の場合、PageIndexMultiDelete は実行されない。

保守担当者向けメモ

  • 本関数は冪等（idempotent）である必要がある。
  • クリーンアップロックの確保は、リプレイ中にスキャンが開始されたり、進行中のスキャンが不整合なデータを読み取ったりすることを防ぐために必須である。
