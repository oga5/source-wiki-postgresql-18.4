test_re_compile

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_regex/test_regex.c:159

目的

SQLから提供された正規表現パターン文字列を、PostgreSQLの正規表現エンジンが処理可能な形式（regex_t）へコンパイルする。

シグネチャまたは形状

static void test_re_compile(text *text_re, int cflags, Oid collation, regex_t *result_re)

入力と出力

  • **入力**:
  • text_re: コンパイル対象の正規表現パターン（text型）。
  • cflags: コンパイルオプション（フラグ）。
  • collation: 正規表現の照合順序。
  • **出力**:
  • result_re: コンパイル結果を格納する regex_t 構造体へのポインタ。
  • **戻り値**: なし（成功時は result_re を更新、失敗時はエラーをスロー）。

前提条件と不変条件

  • text_re は有効な text 型データであること。
  • result_re は呼び出し元によって確保された有効なメモリ領域であること。
  • 内部で pg_mb2wchar_with_len を使用するため、入力文字列は現在のデータベースエンコーディングに従っている必要がある。

アルゴリズムまたは振る舞い

1. 入力された text 型のパターン文字列を、pg_mb2wchar_with_len を用いてワイド文字（pg_wchar）配列に変換する。 2. 変換したワイド文字配列を引数として pg_regcomp を呼び出し、正規表現をコンパイルする。 3. 一時的に確保したワイド文字配列のメモリを pfree で解放する。 4. pg_regcomp の戻り値が REG_OKAY でない場合、pg_regerror でエラーメッセージを取得し、ereport(ERROR, ...) を通じて例外を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_regex/test_regex.c 内の test_regex 関数（101行目付近）。
  • **呼び出し先**:
  • pg_mb2wchar_with_len
  • pg_regcomp
  • pg_regerror
  • palloc / pfree
  • ereport

関連データ構造

  • regex_t: PostgreSQL の正規表現エンジンが使用するコンパイル済みパターン構造体。
  • pg_wchar: ワイド文字型。

失敗モードと境界条件

  • **コンパイル失敗**: pg_regcomp が REG_OKAY 以外を返した場合、ERRCODE_INVALID_REGULAR_EXPRESSION エラーとして処理を中断する。
  • **メモリ不足**: palloc が失敗した場合は PostgreSQL のメモリ管理機構によりエラーがスローされる。

保守担当者向けメモ

  • 本関数はマルチバイト文字の処理を明示的に行っているため、エンコーディング関連の修正を行う際は pg_mb2wchar_with_len の挙動に注意すること。
  • コンパイル失敗時に pg_regfree を呼び出す必要はない（pg_regcomp が失敗した場合は内部でクリーンアップ済みであるため）。
