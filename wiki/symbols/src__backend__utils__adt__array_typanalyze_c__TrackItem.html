TrackItem

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/array_typanalyze.c:68

目的

Lossy Counting アルゴリズムにおいて、配列要素の頻度情報を追跡するためのハッシュテーブルエントリとして使用されます。

シグネチャまたは形状

typedef struct {
    Datum       key;            /* 要素値 */
    int         frequency;      /* 観測された頻度 (f) */
    int         delta;          /* 誤差許容値 (delta) */
    int         last_container; /* 重複排除用コンテナID */
} TrackItem;

入力と出力

本構造体はデータ保持用であり、関数としての入出力はありません。compute_array_stats 内でハッシュテーブルの要素として生成・更新されます。

前提条件と不変条件

  • frequency および delta は、Manku & Motwani の Lossy Counting アルゴリズムの定義に従って管理されます。
  • last_container は、同一配列内での要素の重複カウントを防ぐために使用されます。

アルゴリズムまたは振る舞い

  • ANALYZE 処理中に配列要素をハッシュテーブルに登録・更新する際に使用されます。
  • prune_element_hashtable 関数により、メモリ使用量を抑えるための剪定対象となります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_typanalyze.c 内の compute_array_stats が本構造体をハッシュテーブルエントリとして利用します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な呼び出し先はありません。
  • **注記**: src/backend/commands/analyze.c および src/backend/tsearch/ts_typanalyze.c にも同名の構造体定義が存在しますが、これらは別個の定義であり、本ページで説明している array_typanalyze.c のものとは異なります。

関連データ構造

  • ArrayAnalyzeExtraData: 本構造体を利用する統計解析処理のコンテキストを保持します。

失敗モードと境界条件

  • メモリ不足時: palloc 失敗によりエラーが発生する可能性があります。
  • 統計精度の限界: Lossy Counting の性質上、厳密な頻度ではなく近似値が保持されます。

保守担当者向けメモ

  • 本構造体は array_typanalyze.c 専用の定義です。他ファイル（ts_typanalyze.c 等）の同名構造体と混同しないよう注意してください。
  • 頻度計算のロジックを変更する場合は、Lossy Counting のアルゴリズム要件（delta の更新ルール等）を再確認してください。
