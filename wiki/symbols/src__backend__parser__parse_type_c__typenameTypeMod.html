typenameTypeMod

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_type.c:331

目的

TypeName 構造体に含まれる型修飾子（typmod）のリストを解析・検証し、データベース内部で利用可能な int32 型の数値へ変換します。

シグネチャまたは形状

int32 typenameTypeMod(ParseState *pstate, const TypeName *typeName, Type typ)

入力と出力

  • **入力**:
  • pstate: パーサーの状態管理構造体。
  • typeName: 解析対象の型名情報。
  • typ: 対象型のシステムカタログ（pg_type）タプル。
  • **出力**:
  • 変換された int32 型の型修飾子値。

前提条件と不変条件

  • typ は有効な pg_type タプルである必要があります。
  • typeName->typmods が空の場合、typeName->typemod がそのまま返されます。
  • 型修飾子を処理する場合、対象の型は「シェル型（未定義型）」であってはなりません。

アルゴリズムまたは振る舞い

1. typeName->typmods が空であれば、即座に typeName->typemod を返します。 2. typ がシェル型であるか、または typmodin 関数を持たない場合はエラーを報告します。 3. typmods リスト内の各ノード（定数または識別子）を文字列に変換し、cstring 配列を作成します。 4. construct_array_builtin を用いて、作成した配列を CSTRINGOID 型の配列に変換します。 5. typ から取得した typmodin 関数を呼び出し、配列を渡して最終的な int32 値を得ます。 6. 処理中にエラーが発生した場合、parser_errposition を用いて SQL 文中の位置を報告します。 7. 最後に確保したメモリ（datums および arrtypmod）を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_type.c 内の型解決処理（例: typenameTypeIdAndMod 等の内部フロー）。
  • **呼び出し先**:
  • TypeNameToString: 型名文字列の取得。
  • setup_parser_errposition_callback / cancel_parser_errposition_callback: エラー位置報告の管理。
  • OidFunctionCall1: 型固有の typmodin 関数の実行。

関連データ構造

  • TypeName: SQL 文中の型指定を保持する構造体。
  • Form_pg_type: pg_type システムカタログの構造体。
  • ParseState: 構文解析中の状態を保持する構造体。

失敗モードと境界条件

  • **シェル型エラー**: typisdefined が偽の場合、型修飾子を指定できないためエラーとなります。
  • **typmodin 未定義**: 型が型修飾子をサポートしていない（typmodin が InvalidOid）場合、エラーとなります。
  • **構文エラー**: 型修飾子が単純な定数または識別子でない場合、エラーとなります。

保守担当者向けメモ

  • 本関数が返す値は、pg_type の typmodin 関数に依存します。
  • palloc で確保したメモリは関数終了時に必ず pfree される設計となっています。
  • エラー発生時の位置報告には pstate が必須であるため、呼び出し時には適切な ParseState を渡す必要があります。
