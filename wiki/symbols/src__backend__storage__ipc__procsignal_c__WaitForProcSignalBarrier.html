WaitForProcSignalBarrier

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procsignal.c:424

目的

全バックエンドプロセスが指定されたバリア生成番号（generation）に到達したことを保証するまで待機する。

シグネチャまたは形状

void WaitForProcSignalBarrier(uint64 generation)

入力と出力

  • **入力**: generation (待機対象のバリア生成番号)
  • **出力**: なし（完了時に復帰）

前提条件と不変条件

  • generation は、現在のグローバルバリア生成番号（ProcSignal->psh_barrierGeneration）以下であること。
  • 呼び出し後にメモリバリア（pg_memory_barrier()）が挿入され、本関数以降の処理と本関数内での読み取り操作の順序が保証される。

アルゴリズムまたは振る舞い

1. 全ての ProcSignalSlot を逆順に走査する。 2. 各スロットの pss_barrierGeneration が指定された generation に達するまで、ConditionVariableTimedSleep を用いて待機する。 3. 待機中、5秒ごとにタイムアウトが発生した場合は、対象プロセスのPIDと共にログを出力する。 4. 各スロットの更新を確認後、ConditionVariableCancelSleep() を呼び出して待機を終了する。 5. 最後にメモリバリアを実行し、呼び出し元へ復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c
  • src/backend/commands/tablespace.c
  • **呼び出し先**:
  • pg_atomic_read_u64 / pg_atomic_read_u32
  • ConditionVariableTimedSleep
  • ConditionVariableCancelSleep
  • pg_memory_barrier

関連データ構造

  • ProcSignalHeader (共有メモリ上のヘッダー)
  • ProcSignalSlot (各プロセスのシグナル状態を保持)

失敗モードと境界条件

  • 5秒経過しても対象プロセスがバリアを吸収しない場合、LOG レベルで警告が出力される。
  • pss_barrierCheckMask ではなく pss_barrierGeneration を基準に判定を行うことで、バリア吸収の正確性を担保している。

保守担当者向けメモ

  • 本関数は全プロセスを待機させるため、呼び出し頻度が高いとシステム全体のパフォーマンスに影響を与える可能性がある。
  • pss_barrierGeneration の読み取りはロックなしで行われるため、後続の共有状態へのアクセスには必ずメモリバリアが必要である。
