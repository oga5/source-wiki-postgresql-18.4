init_local_reloptions

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/reloptions.c:744

目的

リレーションオプションを管理するための local_relopts 構造体を初期化し、オプションリストとバリデーターリストを空に設定するとともに、オプション構造体のサイズを登録します。

シグネチャまたは形状

void init_local_reloptions(local_relopts *relopts, Size relopt_struct_size)

入力と出力

  • **入力**:
  • relopts: 初期化対象となる local_relopts 構造体へのポインタ。
  • relopt_struct_size: オプションを格納する構造体のサイズ（Size 型）。
  • **出力**: なし（relopts が指すメモリ領域を直接更新します）。

前提条件と不変条件

  • relopts は有効なメモリ領域を指している必要があります。
  • 実行後、relopts->options および relopts->validators は NIL（空リスト）となります。

アルゴリズムまたは振る舞い

1. relopts->options に NIL を代入します。 2. relopts->validators に NIL を代入します。 3. relopts->relopt_struct_size に引数 relopt_struct_size を代入します。

  • **呼び出し先**: 本関数内での関数呼び出しはありません。

関連データ構造

  • local_relopts: オプションのリスト、バリデーターのリスト、および構造体サイズを保持する構造体。

失敗モードと境界条件

  • relopts が NULL の場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は、各アクセスメソッドが独自のオプションを定義する際の初期化ステップとして使用されます。新しいオプションを追加する際は、この初期化後に add_local_reloption 等を用いてリストを構築してください。
