sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-sqljson_jsontable.c:32

目的

ECPG（Embedded SQL for C）アプリケーションにおいて、SQL 文の実行結果やエラー診断情報を保持するための通信領域（SQL Communication Area）を定義する。

シグネチャまたは形状

struct sqlca_t {
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: ECPG ライブラリ関数（ECPGdo 等）が SQL 文の実行後に、その結果やエラー状態を本構造体の各メンバに書き込む。
  • **出力**: アプリケーション側で sqlcode や sqlstate を参照し、エラーハンドリングや処理の分岐に使用する。

前提条件と不変条件

  • sqlwarn[0] は、他の警告フラグが一つでも設定されている場合に 'W' となる。
  • sqlerrd[2] は、INSERT、UPDATE、DELETE 文の実行後に処理された行数を保持する。

アルゴリズムまたは振る舞い

SQL 文の実行結果に基づき、以下の情報を更新する：

  • sqlcode: 実行結果のステータスコード。
  • sqlerrm: エラーメッセージの長さと内容。
  • sqlwarn: 実行中に発生した警告（切り捨てなど）。
  • sqlstate: SQL 標準準拠のエラーコード。

呼び出し元と呼び出し先

  • **呼び出し元（参照/更新）**:
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/data.c
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • src/interfaces/ecpg/compatlib/informix.c
  • **呼び出し先**: 情報なし。

関連データ構造

  • sqlda_t（SQL 記述子領域）

失敗モードと境界条件

  • sqlerrmc は SQLERRMC_LEN で定義されたバッファサイズを超えるエラーメッセージを保持できない。
  • sqlwarn の各要素は特定の警告条件（文字列の切り捨て等）に対応しており、条件を満たさない場合は空または初期値となる。

保守担当者向けメモ

  • 本構造体は ECPG のコア機能であり、sqlca.h を通じて提供される。
  • テストコード内では、sqlprint() 関数を使用して sqlca の内容を標準エラー出力にダンプすることが推奨される。
