CopyToTextLikeOneRow

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyto.c:263

目的

COPY TO 操作において、単一のタプルをテキストまたはCSV形式で出力バッファへ書き出す。

シグネチャまたは形状

static pg_attribute_always_inline void CopyToTextLikeOneRow(CopyToState cstate, TupleTableSlot *slot, bool is_csv)

入力と出力

  • **入力**:
  • cstate: COPY TO 操作の現在の状態を保持する構造体。
  • slot: 出力対象のデータを含むタプルスロット。
  • is_csv: CSV形式として処理するか（true）、テキスト形式として処理するか（false）を指定するフラグ。
  • **出力**: なし（副作用として cstate を通じて出力先へデータが書き込まれる）。

前提条件と不変条件

  • cstate->attnumlist に基づき、指定された列のみが処理される。
  • cstate->out_functions に各列の出力関数が設定されている必要がある。
  • slot は有効なタプルデータを含んでいる必要がある。

アルゴリズムまたは振る舞い

1. cstate->attnumlist を反復し、各列の値を順次処理する。 2. 2列目以降の各値の前に、cstate->opts.delim[0] で指定された区切り文字を挿入する。 3. 値が NULL の場合、cstate->opts.null_print_client を出力する。 4. 値が非 NULL の場合、OutputFunctionCall で文字列化する。

  • is_csv が true の場合、CopyAttributeOutCSV を呼び出し、必要に応じてクォート処理を行う。
  • is_csv が false の場合、CopyAttributeOutText を呼び出し、テキスト形式のエスケープ処理を行う。

    • 5. 全列の処理終了後、CopySendTextLikeEndOfRow を呼び出し、行末処理（改行の送信等）を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyto.c 内の COPY TO 実行ロジック（テキストおよびCSV出力パス）。
  • **呼び出し先**:
  • CopySendChar
  • CopySendString
  • OutputFunctionCall
  • CopyAttributeOutCSV
  • CopyAttributeOutText
  • CopySendTextLikeEndOfRow

関連データ構造

  • CopyToStateData (CopyToState): 操作全体の状態管理。
  • TupleTableSlot: 処理対象のタプルデータ。
  • FmgrInfo: 列ごとの出力関数情報。

失敗モードと境界条件

  • cstate->opts.delim が空の場合の挙動は呼び出し側の設定に依存する。
  • OutputFunctionCall 内でエラーが発生した場合、処理は中断される。

保守担当者向けメモ

  • pg_attribute_always_inline が指定されており、パフォーマンス重視のインライン関数として実装されている。
  • CSVとテキスト形式の分岐は is_csv 引数で行われるため、両形式の共通ロジックに変更を加える際は注意が必要。
