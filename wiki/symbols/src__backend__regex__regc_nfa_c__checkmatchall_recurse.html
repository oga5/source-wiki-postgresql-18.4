checkmatchall_recurse

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:3276

目的

NFA（非決定性有限オートマトン）において、特定の状態から終了状態（nfa->post）までのパスが、すべての可能な入力に対してマッチするかどうかを再帰的に判定する。

シグネチャまたは形状

static bool checkmatchall_recurse(struct nfa *nfa, struct state *s, bool **haspaths)

入力と出力

  • **入力**:
  • nfa: 対象のNFA構造体。
  • s: 探索を開始する現在の状態。
  • haspaths: 各状態から終了状態までのパス長（ステップ数）の到達可能性を保持する配列の配列。
  • **出力**:
  • bool: パスが条件を満たせば true、そうでなければ false を返す。

前提条件と不変条件

  • s->tmp は探索開始時に NULL であること（訪問済みフラグとして使用）。
  • haspaths 配列は、各状態のインデックス no に対応する bool 配列を格納できること。
  • DUPINF 定数に基づいたパス長の管理が行われること。

アルゴリズムまたは振る舞い

1. **スタック保護**: STACK_TOO_DEEP マクロにより再帰の深さをチェックし、過度な場合は false を返す。 2. **状態管理**: s->tmp を設定して訪問済みとし、haspath 配列を割り当てる。 3. **遷移走査**: s->outs にある各遷移（RAINBOW カラーのみ対象）を辿る。

  • nfa->post に到達した場合、haspath[0] を true にする。
  • 既に訪問済みの状態へ遷移した場合はループとみなし、長さ1のループであればパス長を更新し、それ以外は false を返す。
  • 未訪問の状態であれば再帰的に呼び出し、結果をマージする。

    • 4. **パス長計算**: 遷移先のパス長情報（nexthaspath）を基に、現在の状態からのパス長を計算・統合する。 5. **終了処理**: s->tmp を NULL に戻し、計算した haspath を haspaths[s->no] に格納して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_nfa.c 内の checkmatchall_recurse（再帰呼び出し）および、同ファイル内の上位関数（checkmatchall_recurse を呼び出すロジックが存在）。
  • **呼び出し先**: checkmatchall_recurse（再帰）、MALLOC、memset、assert、INTERRUPT。

関連データ構造

  • struct nfa: NFA 全体を管理する構造体。
  • struct state: NFA の状態。tmp フィールドを再帰探索中の訪問フラグとして利用。
  • struct arc: 状態間の遷移。co（カラー）と to（遷移先）を持つ。

失敗モードと境界条件

  • **スタックオーバーフロー**: STACK_TOO_DEEP により安全に中断し false を返す。
  • **メモリ不足**: MALLOC が失敗した場合、false を返す。
  • **循環参照**: 状態間のループが検出された場合、長さ1のループは許容されるが、それ以外は false となる。
  • **パス長不整合**: nexthaspath の情報が DUPINF の制約を満たさない場合、false を返す。

保守担当者向けメモ

  • 本関数は regc_nfa.c 内で再帰的にグラフを走査する。スタックオーバーフロー対策として STACK_TOO_DEEP が挿入されている。
  • haspaths 配列は、各状態から終了状態までのパス長情報をキャッシュする役割を持つ。
  • RAINBOW 以外の擬似カラー遷移は無視される仕様である。
