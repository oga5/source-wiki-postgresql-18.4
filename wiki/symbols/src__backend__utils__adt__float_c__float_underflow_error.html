float_underflow_error

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/float.c:93

目的

浮動小数点演算においてアンダーフローが発生した際、PostgreSQLの標準的なエラーメカニズムを通じて例外を報告する。

シグネチャまたは形状

pg_noinline void float_underflow_error(void)

入力と出力

  • 入力: なし
  • 出力: なし（関数は正常に復帰せず、ereport によりエラーを発生させる）

前提条件と不変条件

  • 本関数は ereport(ERROR, ...) を呼び出すため、呼び出し元はトランザクションのロールバックやエラーハンドリングが適切に行われる環境にある必要がある。
  • pg_noinline 修飾により、コンパイラによるインライン化が抑制されている。

アルゴリズムまたは振る舞い

1. ereport マクロを呼び出す。 2. エラーレベルを ERROR に設定する。 3. エラーコードとして ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE を指定する。 4. エラーメッセージとして "value out of range: underflow" を出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/float.c 内の複数の算術演算関数（例: 行番号 1204, 1460, 1479, 1617, 1625, 1632, 1674, 1679, 1712, 1745, 2643 など）。
  • 呼び出し先: PostgreSQL のエラー報告サブシステム（ereport）。

関連データ構造

  • なし

失敗モードと境界条件

  • 本関数自体がエラー報告を行うための手段であるため、本関数内での失敗は PostgreSQL のバックエンドエラー処理に委ねられる。

保守担当者向けメモ

  • 本関数は、プラットフォーム間で異なる浮動小数点演算の挙動を統一し、SQL標準に準拠したエラー報告を行うために使用される。
  • 類似の関数として float_overflow_error や float_zero_divide_error が存在し、これらと一貫性を保つ必要がある。
