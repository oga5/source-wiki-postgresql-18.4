be_tls_destroy

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-openssl.c:429

目的

PostgreSQL バックエンドで使用される TLS 通信用の OpenSSL コンテキストを破棄し、関連する状態をリセットします。

シグネチャまたは形状

void be_tls_destroy(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • SSL_context が有効な場合は SSL_CTX_free が呼び出されます。
  • 実行後、SSL_context は NULL に設定され、ssl_loaded_verify_locations フラグは false にリセットされます。

アルゴリズムまたは振る舞い

1. グローバル変数 SSL_context が非 NULL であるかを確認します。 2. SSL_context が存在する場合、SSL_CTX_free を呼び出して OpenSSL のコンテキストリソースを解放します。 3. SSL_context を NULL に設定します。 4. 証明書の検証場所が読み込まれたことを示す ssl_loaded_verify_locations を false に設定します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/libpq/be-secure.c:91
  • 呼び出し先:
  • SSL_CTX_free (OpenSSL API)

関連データ構造

  • SSL_context (OpenSSL の SSL_CTX 構造体へのポインタ)
  • ssl_loaded_verify_locations (証明書検証場所の読み込み状態を示すフラグ)

失敗モードと境界条件

  • SSL_context が既に NULL の場合、関数は何もせず安全に終了します。

保守担当者向けメモ

  • 本関数は TLS 設定の破棄を行うため、サーバーのシャットダウン時や設定リロード時に呼び出されることを想定しています。
  • SSL_CTX_free は OpenSSL の内部的なリソース解放を行うため、呼び出し後に SSL_context を再利用しないよう注意してください。
