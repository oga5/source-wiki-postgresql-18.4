AttoptCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/attoptcache.c:38

目的

PostgreSQLの属性オプション（attoptions）をメモリ上でキャッシュするためのハッシュテーブルエントリ構造体です。システムカタログ pg_attribute へのアクセス頻度を抑え、クエリ実行時のオーバーヘッドを削減します。

シグネチャまたは形状

typedef struct {
    AttoptCacheKey key;      /* 検索キー - 構造体の先頭である必要がある */
    AttributeOpts *opts;     /* オプションデータ、存在しない場合は NULL */
} AttoptCacheEntry;

入力と出力

  • **入力**: ハッシュテーブル検索時のキーとして AttoptCacheKey を使用します。
  • **出力**: opts フィールドを通じて AttributeOpts 構造体（解析済みの属性オプション）を提供します。

前提条件と不変条件

  • key フィールドは構造体の先頭に配置される必要があります（ハッシュテーブル操作の制約）。
  • opts は CacheMemoryContext 上に割り当てられます。
  • キャッシュエントリの無効化時には、opts が指すメモリ領域を適切に解放（pfree）する必要があります。

アルゴリズムまたは振る舞い

  • AttoptCacheHash ハッシュテーブルの要素として使用されます。
  • get_attribute_options() がキャッシュミスを検出した際、hash_search を用いて新規エントリが挿入されます。
  • InvalidateAttoptCacheCallback() が呼び出されると、該当するエントリがハッシュテーブルから削除され、メモリが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: get_attribute_options() がキャッシュの検索・挿入時に本構造体を参照します。また、InvalidateAttoptCacheCallback() が無効化処理時に参照します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • AttoptCacheKey: キャッシュ検索用のキー構造体。
  • AttributeOpts: opts フィールドが指す、解析済みのオプションデータ構造体。
  • AttoptCacheHash: 本構造体を格納するハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時: CacheMemoryContext での割り当てに失敗する可能性があります。
  • キャッシュミス時: opts は NULL として保持されるか、あるいはエントリ自体が生成されません（実装依存）。
  • 無効化の遅延: pg_attribute の更新通知が遅延した場合、古いオプション情報が返される可能性があります。

保守担当者向けメモ

  • opts フィールドは pfree 可能な領域を指しているため、キャッシュから取得したデータを呼び出し元で長期間保持する場合は、適切なコピーを作成してください。
  • ハッシュテーブルの entrysize は sizeof(AttoptCacheEntry) で初期化されています。構造体を拡張する際は注意してください。
