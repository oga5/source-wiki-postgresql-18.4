size_pretty_unit

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/dbsize.c:38

目的

バイト単位の数値を人間が読みやすい形式（kB, MB, GBなど）へ変換、またはその逆変換を行う際に使用する単位定義を保持します。

シグネチャまたは形状

struct size_pretty_unit
{
    const char *name;       /* 単位名 (bytes, kB, MB, GB等) */
    uint32      limit;      /* 単位変換時の丸め判定用上限値 */
    bool        round;      /* 半端な値の丸めを行うかどうかのフラグ */
    uint8       unitbits;   /* 単位のバイト数（2のべき乗） */
};

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。size_pretty_units 配列の要素として定義され、変換ロジックのパラメータとして参照されます。

前提条件と不変条件

  • unitbits は、当該単位が何バイトに相当するかを (1 << unitbits) で表現します。
  • limit は、変換後の数値が特定の単位に適合するかを判定するための閾値として機能します。

アルゴリズムまたは振る舞い

  • pg_size_pretty 関数がこの構造体の配列を参照し、入力されたバイト数に対して適切な単位を選択します。
  • round フラグが真の場合、変換後の数値に対して半端な値の丸め処理が適用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/dbsize.c 内の size_pretty_units[] 配列定義、および pg_size_pretty 等の変換関数。
  • 呼び出し先: 情報なし。

関連データ構造

  • size_pretty_units: size_pretty_unit 型の要素を持つ静的配列。変換ロジックのマスターデータとして機能します。

失敗モードと境界条件

  • 単位の定義が不足している場合、pg_size_bytes 等の逆変換関数でエラーが発生する可能性があります。
  • unitbits の値が不適切な場合、ビットシフト演算の結果が予期しないサイズを算出する可能性があります。

保守担当者向けメモ

  • 新しい単位を追加する場合は、size_pretty_units 配列を更新するだけでなく、関連するエラーメッセージやドキュメントとの整合性を確認してください。
