GenericXLogFinish

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/generic_xlog.c:336

目的

構築中の汎用WALレコードを確定させ、変更内容をバッファに適用し、必要に応じてWALレコードを書き込みます。

シグネチャまたは形状

XLogRecPtr GenericXLogFinish(GenericXLogState *state)

入力と出力

  • **入力**: state (GenericXLogState *) - GenericXLogStart で初期化され、バッファ登録済みの状態オブジェクト。
  • **出力**: XLogRecPtr - 書き込まれたWALレコードのLSN。ログ出力が不要な（Unloggedな）場合は InvalidXLogRecPtr を返します。

前提条件と不変条件

  • state は有効なポインタである必要があります。
  • 呼び出し後、state が指すメモリ領域は pfree により解放されるため、呼び出し元で再利用してはなりません。
  • ログ出力対象の場合、内部で START_CRIT_SECTION() が呼び出されるため、エラー発生時に中断できないクリティカルセクションとして動作します。

アルゴリズムまたは振る舞い

1. **ログ出力の判定**: state->isLogged に基づき処理を分岐します。 2. **ログ出力ありの場合**:

  • XLogBeginInsert() を呼び出し、クリティカルセクションを開始します。
  • 各バッファに対し、差分（デルタ）が必要な場合は computeDelta を実行します。
  • ページイメージをバッファに適用し、pd_lower から pd_upper までの「ホール」領域をゼロクリアして物理的な不一致を防ぎます。
  • MarkBufferDirty でバッファをダーティにし、XLogRegisterBuffer / XLogRegisterBufData でWALレコードに登録します。
  • XLogInsert でWALを書き込み、各バッファのLSNを更新します。

    • 3. **ログ出力なしの場合**:

  • 差分計算を行わず、バッファへイメージを直接コピーしてダーティにします。

    • 4. **終了処理**: state をメモリから解放し、LSNを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/bloom/blinsert.c、contrib/bloom/blutils.c、contrib/bloom/blvacuum.c など。
  • **呼び出し先**: computeDelta、XLogBeginInsert、XLogInsert、XLogRegisterBuffer、XLogRegisterBufData、MarkBufferDirty、BufferGetPage、PageSetLSN、pfree。

関連データ構造

  • GenericXLogState: 構築中のWALレコードの状態を保持する構造体。
  • GenericXLogPageData: 各バッファの変更状態を保持する構造体。

失敗モードと境界条件

  • BufferIsInvalid なバッファはスキップされます。
  • GENERIC_XLOG_FULL_IMAGE フラグが設定されている場合、差分計算をスキップしページ全体を記録します。
  • クリティカルセクション内でのエラー発生時は、PostgreSQLの標準的なエラーハンドリングによりプロセスがアボートされます。

保守担当者向けメモ

  • ページ内の「ホール」領域のゼロクリアは、異なる環境間でのページ再構築時の不一致を防ぐための必須処理です。
  • state は本関数の最後で必ず解放されるため、呼び出し元で GenericXLogFinish を呼び出した後に state を参照しないよう注意してください。
