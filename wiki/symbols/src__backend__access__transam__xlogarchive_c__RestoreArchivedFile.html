RestoreArchivedFile

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:53

目的

アーカイブされたWALファイルを restore_command を用いて取得し、ローカルの pg_wal ディレクトリへ復元する。

シグネチャまたは形状

bool RestoreArchivedFile(char *path, const char *xlogfname,
                         const char *recovername, off_t expectedSize,
                         bool cleanupEnabled)

入力と出力

  • **入力**:
  • path: 復元先パスを格納するバッファ。
  • xlogfname: アーカイブから取得するWALファイル名。
  • recovername: ローカルに配置する際のファイル名。
  • expectedSize: 期待されるファイルサイズ（0の場合はチェックしない）。
  • cleanupEnabled: アーカイブのクリーンアップを許可するかどうかのフラグ。
  • **出力**:
  • 成功時に true を返し、path に復元されたファイルのフルパスを格納する。失敗時は false または FATAL エラーでプロセスを終了する。

前提条件と不変条件

  • ArchiveRecoveryRequested が真であること。
  • recoveryRestoreCommand が設定されていること。
  • 復元先ディレクトリ（XLOGDIR）への書き込み権限があること。

アルゴリズムまたは振る舞い

1. **初期チェック**: アーカイブリカバリが要求されていない、または restore_command が空の場合は即座に終了する。 2. **既存ファイル排除**: recovername と同名のファイルが XLOGDIR に存在する場合、それを削除する。 3. **コマンド構築**: BuildRestoreCommand を使用して、アーカイブからファイルを取得するためのシェルコマンドを生成する。 4. **実行**: PreRestoreCommand() を呼び出し、system() でコマンドを実行後、PostRestoreCommand() を呼び出す。 5. **検証**: コマンド終了後、ファイルが存在し、かつ expectedSize が指定されている場合はサイズが一致するかを確認する。

  • スタンバイモードでサイズが小さい場合は DEBUG1 レベルでログ出力し、それ以外は FATAL エラーとする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/timeline.c
  • src/backend/access/transam/xlogrecovery.c
  • src/bin/pg_rewind/parsexlog.c
  • **呼び出し先**:
  • BuildRestoreCommand
  • PreRestoreCommand / PostRestoreCommand
  • GetOldestRestartPoint (cleanupEnabled時)
  • system()

関連データ構造

  • XLogSegNo, XLogRecPtr, TimeLineID (WAL管理用)
  • struct stat (ファイルサイズ確認用)

失敗モードと境界条件

  • stat() や unlink() の失敗: FATAL エラーとして報告される。
  • system() の戻り値が0以外: 失敗として扱われる。
  • ファイルサイズ不一致: スタンバイモード以外では FATAL エラー。

保守担当者向けメモ

  • system() を呼び出すため、シグナルハンドラとの競合を避ける目的で PreRestoreCommand/PostRestoreCommand が必須となっている。
  • Windows環境でのファイルロック競合を避けるため、既存ファイルの削除処理には注意が必要。
  • expectedSize のチェックは、アーカイブからの転送が不完全な状態を検知するために重要である。
