LogicalRepWorker

定義

  • 種別: struct
  • 定義箇所: src/include/replication/worker_internal.h:37

目的

論理レプリケーションにおける各ワーカー（Apply, Tablesync, Parallel Apply）の実行状態、ライフサイクル、および通信情報を保持するための共有構造体です。

シグネチャまたは形状

typedef struct LogicalRepWorker {
    LogicalRepWorkerType type;
    TimestampTz launch_time;
    bool        in_use;
    uint16      generation;
    PGPROC     *proc;
    Oid         dbid;
    Oid         userid;
    Oid         subid;
    Oid         relid;
    char        relstate;
    XLogRecPtr  relstate_lsn;
    slock_t     relmutex;
    FileSet    *stream_fileset;
    pid_t       leader_pid;
    bool        parallel_apply;
    XLogRecPtr  last_lsn;
    TimestampTz last_send_time;
    TimestampTz last_recv_time;
    XLogRecPtr  reply_lsn;
    TimestampTz reply_time;
} LogicalRepWorker;

入力と出力

  • **入力**: ワーカー起動時にサブスクリプション情報（subid）、接続先情報（dbid, userid）、およびワーカー種別が設定されます。
  • **出力**: ワーカーの実行状態、統計情報（last_lsn 等）、および並列適用時のストリーミング用ファイルセット（stream_fileset）が管理されます。

前提条件と不変条件

  • in_use フラグにより、スロットが現在使用中か解放済みかが管理されます。
  • generation はスロットが新しいワーカーによって再利用されるたびにインクリメントされます。
  • proc はワーカーが実行中でない場合 NULL となります。
  • relmutex はテーブル同期状態（relstate）の更新時に使用されます。

アルゴリズムまたは振る舞い

  • **ワーカー管理**: logicalrep_worker_launch によりスロットが割り当てられ、ワーカーのライフサイクルが管理されます。
  • **ストリーミング**: トランザクションのストリーミングが発生した際、またはリーダーワーカーが並列ワーカーへ変更を送信する際に stream_fileset が初期化されます。このファイルセットはワーカー終了時に削除されます。
  • **並列適用**: 並列適用ワーカーとして動作する場合、leader_pid にリーダーワーカーの PID が格納されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/launcher.c 内の関数群（logicalrep_worker_cleanup 等）から参照・操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出し先はソースコードの文脈からは特定できません。

関連データ構造

  • LogicalRepWorkerType: ワーカーの種類を定義する列挙型。
  • FileSet: ストリーミングトランザクションの永続化に使用されるファイルセット。
  • PGPROC: ワーカープロセスのプロセス情報。

失敗モードと境界条件

  • stream_fileset の初期化は、最初のストリーミングトランザクション到着時、またはリーダーワーカーのタイムアウト時に行われます。
  • proc が NULL の場合、ワーカーは実行中ではありません。

保守担当者向けメモ

  • MyLogicalRepWorker グローバル変数が各ワーカープロセスで現在実行中の情報を指すために使用されます。
  • stream_fileset の管理は、トランザクションのコミット順序や整合性に直結するため、変更時は注意が必要です。
