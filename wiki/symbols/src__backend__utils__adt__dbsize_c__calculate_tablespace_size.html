calculate_tablespace_size

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/dbsize.c:210

目的

指定されたテーブルスペースの物理ディスクサイズをバイト単位で計算する。

シグネチャまたは形状

int64 calculate_tablespace_size(Oid tblspcOid)

入力と出力

  • **入力**: tblspcOid (対象テーブルスペースのOID)
  • **出力**: int64 (合計サイズ（バイト単位）。失敗時は -1 を返す)

前提条件と不変条件

  • 実行ユーザーは、対象テーブルスペースに対して CREATE 権限を持つか、pg_read_all_stats ロールの権限を有している必要がある（ただし、現在のデータベースのデフォルトテーブルスペースは例外）。
  • 権限がない場合、aclcheck_error を呼び出しエラーを発生させる。

アルゴリズムまたは振る舞い

1. 権限チェックを行い、不適格な場合はエラーを報告する。 2. tblspcOid に基づき、対象テーブルスペースのディレクトリパスを特定する。 3. AllocateDir を使用してディレクトリを開き、ReadDir で内容を反復走査する。 4. 各エントリに対して stat() を実行し、ファイルサイズを合計する。

  • ディレクトリの場合は db_dir_size を再帰的に呼び出して加算する。
  • ループ内では CHECK_FOR_INTERRUPTS() を呼び出し、クエリのキャンセル要求を処理する。

    • 5. ディレクトリ走査終了後、FreeDir でリソースを解放し、合計値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/dbsize.c:295 (pg_tablespace_size_oid)
  • src/backend/utils/adt/dbsize.c:310 (pg_tablespace_size_name)
  • **呼び出し先**:
  • has_privs_of_role, object_aclcheck, aclcheck_error, get_tablespace_name
  • AllocateDir, ReadDir, FreeDir
  • stat, db_dir_size, CHECK_FOR_INTERRUPTS

関連データ構造

  • Oid: テーブルスペース識別子。
  • DIR: ディレクトリ操作用ハンドル。
  • struct dirent: ディレクトリ内のエントリ情報。
  • struct stat: ファイル属性情報。

失敗モードと境界条件

  • AllocateDir が失敗した場合、-1 を返す。
  • stat() が ENOENT で失敗した場合は無視して続行する。
  • その他の stat() エラーは ereport(ERROR, ...) により処理を中断する。
  • . および .. ディレクトリは走査対象から除外される。

保守担当者向けメモ

  • 本関数はファイルシステムを直接走査するため、大規模なテーブルスペースでは処理に時間がかかる可能性がある。
  • PG_TBLSPC_DIR 以下のディレクトリ構造変更時には、パス生成ロジックの修正が必要となる。
