HTSV_Result

定義

  • 種別: enum
  • 定義箇所: src/include/access/heapam.h:122

目的

ヒープテーブル内のタプルに対する可視性判定の結果を表現し、VACUUM 処理やスキャン時のタプル状態判断の基準として使用されます。

シグネチャまたは形状

typedef enum {
    HEAPTUPLE_DEAD,
    HEAPTUPLE_LIVE,
    HEAPTUPLE_RECENTLY_DEAD,
    HEAPTUPLE_INSERT_IN_PROGRESS,
    HEAPTUPLE_DELETE_IN_PROGRESS,
} HTSV_Result;

入力と出力

  • 入力: 該当なし（列挙型定義）
  • 出力: HeapTupleSatisfiesVacuum 等の可視性判定関数が、タプルの現在の状態を呼び出し元へ返すために使用します。

前提条件と不変条件

  • 各列挙値は、タプルのトランザクション状態および削除フラグに基づいた排他的な状態を表します。
  • HEAPTUPLE_DEAD は、タプルが不要であり削除可能であることを示します。

アルゴリズムまたは振る舞い

  • HEAPTUPLE_DEAD: タプルは無効であり、削除可能です。
  • HEAPTUPLE_LIVE: タプルは有効（コミット済み、削除者なし）です。
  • HEAPTUPLE_RECENTLY_DEAD: タプルは無効ですが、まだ削除可能ではありません。
  • HEAPTUPLE_INSERT_IN_PROGRESS: 挿入したトランザクションが実行中です。
  • HEAPTUPLE_DELETE_IN_PROGRESS: 削除したトランザクションが実行中です。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_visibility.c
  • src/backend/access/heap/pruneheap.c
  • contrib/pg_visibility/pg_visibility.c
  • 呼び出し先: 該当なし（列挙型のため）。

関連データ構造

  • HeapTupleSatisfiesVacuum: この列挙型を戻り値として使用する主要な関数。
  • PruneState: pruneheap.c 内でこの状態を保持・処理するために参照されます。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • pruneheap.c 内では、最適化のために int8[] 型としてこの値を扱うケースがあるため、型変換時の整合性に注意してください。
  • 新しいタプル状態を追加する場合は、可視性判定ロジック全体への影響を考慮する必要があります。
