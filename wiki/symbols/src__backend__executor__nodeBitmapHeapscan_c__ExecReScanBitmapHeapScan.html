ExecReScanBitmapHeapScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapHeapscan.c:225

目的

BitmapHeapScanState ノードの状態をリセットし、スキャンを最初から再実行できるように準備する。

シグネチャまたは形状

void ExecReScanBitmapHeapScan(BitmapHeapScanState *node)

入力と出力

  • **入力**: BitmapHeapScanState *node（リセット対象のスキャン状態構造体）
  • **出力**: なし（副作用として node の内部状態が更新される）

前提条件と不変条件

  • node は有効な BitmapHeapScanState 構造体を指している必要がある。
  • outerPlanState(node) が存在すること。

アルゴリズムまたは振る舞い

1. **スキャン記述子のクリーンアップ**: ss.ss_currentScanDesc が存在する場合、イテレータが終了していなければ tbm_end_iterate を呼び出し、table_rescan を実行してページピンを解放する。 2. **ビットマップの解放**: node->tbm が存在する場合、tbm_free でメモリを解放し、ポインタを NULL に設定する。 3. **状態の初期化**: node->initialized を false に、node->recheck を true にリセットする。 4. **スキャンノードの再スキャン**: ExecScanReScan を呼び出し、スキャン共通の状態をリセットする。 5. **サブプランの再スキャン**: outerPlan->chgParam が NULL の場合、ExecReScan を呼び出して子ノード（インデックススキャン等）を再実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c 内の ExecReScan 経由で呼び出されることが示唆されている。
  • **呼び出し先**: tbm_end_iterate, table_rescan, tbm_free, ExecScanReScan, ExecReScan（すべてソースコード内の直接的な呼び出し）。

関連データ構造

  • BitmapHeapScanState: ビットマップヒープスキャンの実行状態を保持する主要構造体。
  • TIDBitmap: ヒープから読み取るタプルIDの集合。
  • TableScanDesc: テーブルアクセスメソッドのスキャン記述子。

失敗モードと境界条件

  • scan が NULL の場合、スキャン記述子に関連するクリーンアップ処理はスキップされる。
  • node->tbm が NULL の場合、ビットマップの解放処理はスキップされる。

保守担当者向けメモ

  • 本関数は ExecReScan の一部として呼び出される。サブプランのパラメータが変更されていない場合のみ子ノードの再スキャンが行われる点に注意すること。
  • 並列実行時の共有状態の再初期化については、本関数内では直接扱われていないため、必要に応じて関連する DSM 初期化関数との整合性を確認すること。
