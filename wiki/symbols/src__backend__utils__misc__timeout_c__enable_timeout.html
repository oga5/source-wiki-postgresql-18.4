enable_timeout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:157

目的

指定されたタイムアウトイベントを有効化し、期限順に管理されるアクティブなタイムアウトリストへ登録する。

シグネチャまたは形状

static void enable_timeout(TimeoutId id, TimestampTz now, TimestampTz fin_time, int interval_in_ms)

入力と出力

  • id: 登録対象のタイムアウト識別子。
  • now: 現在のタイムスタンプ。
  • fin_time: タイムアウトが発生する期限時刻。
  • interval_in_ms: 繰り返し実行される場合のインターバル（ミリ秒）。
  • 出力: なし（副作用として active_timeouts 配列を更新）。

前提条件と不変条件

  • all_timeouts_initialized が真であること。
  • 指定された id に対応するハンドラ関数が登録済みであること。
  • 呼び出し側で適切な排他制御（disable_alarm 等によるシグナル抑制）が行われていること。

アルゴリズムまたは振る舞い

1. 指定された id が既にアクティブな場合、remove_timeout_index を呼び出してリストから一時的に削除する。 2. active_timeouts リストを走査し、fin_time（期限）の昇順、期限が同一の場合は id の昇順となる挿入位置を特定する。 3. all_timeouts[id] の状態（indicator, start_time, fin_time, interval_in_ms）を更新する。 4. insert_timeout を呼び出し、特定した位置にタイムアウトを挿入する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/timeout.c 内の enable_timeout_after、enable_timeout_every、および内部の再スケジュール処理（reschedule_timeouts 等）。
  • 呼び出し先: remove_timeout_index, find_active_timeout, insert_timeout。

関連データ構造

  • TimeoutId: タイムアウトを識別する列挙型。
  • all_timeouts: 全タイムアウト設定を保持する静的配列。
  • active_timeouts: 有効なタイムアウトを期限順に保持する配列。

失敗モードと境界条件

  • all_timeouts_initialized が偽の場合、Assert により異常終了する。
  • 存在しない id を指定した場合、all_timeouts[id].timeout_handler が NULL であれば Assert により異常終了する。

保守担当者向けメモ

  • 本関数は active_timeouts のソート順序を維持する責任を持つ。リストの挿入位置決定ロジックを変更する場合は、fin_time と id による優先順位付けの整合性に注意すること。
  • シグナルハンドラとの競合を防ぐため、本関数の呼び出し前後でシグナルが適切に制御されていることを確認すること。
