CompactAttribute

定義

  • 種別: struct
  • 定義箇所: src/include/access/tupdesc.h:68

目的

タプル変形（Deformation）処理のパフォーマンスを最適化するための、FormData_pg_attribute の軽量かつ高速アクセス可能なキャッシュ用構造体です。

シグネチャまたは形状

typedef struct CompactAttribute
{
	int32		attcacheoff;    /* タプルの固定オフセット、または -1 */
	int16		attlen;         /* 属性長（バイト）、-1=可変長、-2=cstring */
	bool		attbyval;       /* 値渡しフラグ */
	bool		attispackable;  /* ストレージが TYPSTORAGE_PLAIN 以外か */
	bool		atthasmissing;  /* 欠損値保持フラグ */
	bool		attisdropped;   /* 削除済み属性フラグ */
	bool		attgenerated;   /* 生成列フラグ */
	char		attnullability; /* NOT NULL 制約の状態 */
	uint8		attalignby;     /* アライメント要件（バイト） */
} CompactAttribute;

入力と出力

本構造体は TupleDesc 内に配列として保持されます。実行エンジンは TupleDescCompactAttr() を介して特定の属性情報を取得し、タプルのバイト列解析に利用します。

前提条件と不変条件

  • FormData_pg_attribute（カタログ定義）が変更された場合、必ず populate_compact_attribute() を呼び出して本構造体を再同期させる必要があります。
  • 構造体サイズは 16 バイトに厳密に制限されており、拡張には性能への影響を考慮した慎重な設計が求められます。

アルゴリズムまたは振る舞い

タプル変形時に、属性の型情報やアライメント、オフセットを高速に参照するために使用されます。特に attcacheoff が設定されている場合、タプル内の特定属性へのオフセット計算を省略可能です。

呼び出し元と呼び出し先

  • **呼び出し元（参照先）**:
  • contrib/amcheck/verify_heapam.c
  • contrib/pageinspect/heapfuncs.c
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/access/brin/brin_inclusion.c
  • src/backend/access/brin/brin_tuple.c
  • src/backend/access/common/attmap.c
  • src/backend/access/common/heaptuple.c
  • **呼び出し先**:
  • 構造体定義のため、特定の関数呼び出しは行いません。

関連データ構造

  • TupleDescData / TupleDesc: 本構造体を保持する親構造体。
  • FormData_pg_attribute: カタログ上の属性定義元。

失敗モードと境界条件

  • attcacheoff が -1 の場合、オフセットは固定ではなく計算が必要であることを示します。
  • attlen が -1 または -2 の場合、可変長データとして特別な処理が必要です。

保守担当者向けメモ

  • 本構造体はタプル変形の性能に直結するため、フィールドの追加や変更は慎重に行ってください。
  • FormData_pg_attribute との整合性が崩れると、タプル解析時に深刻なデータ破損や誤った読み取りが発生します。更新時は必ず populate_compact_attribute() のロジックを確認してください。
