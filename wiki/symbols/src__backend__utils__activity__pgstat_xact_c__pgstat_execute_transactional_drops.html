pgstat_execute_transactional_drops

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_xact.c:313

目的

WALリプレイ中またはトランザクション終了処理において、統計情報の削除を物理的に実行する。

シグネチャまたは形状

void pgstat_execute_transactional_drops(int ndrops, struct xl_xact_stats_item *items, bool is_redo)

入力と出力

  • ndrops: 処理対象となる統計アイテムの数。
  • items: 削除対象の統計情報を保持する xl_xact_stats_item 構造体の配列。
  • is_redo: WALリプレイ中であるかを示すフラグ（関数内で使用されていない）。
  • 戻り値: なし。

前提条件と不変条件

  • ndrops が 0 の場合、関数は即座に終了する。
  • items 内の各アイテムは、objid_hi と objid_lo を結合して 64 ビットの objid として解釈される。

アルゴリズムまたは振る舞い

1. 指定された ndrops 分のアイテムをループで走査する。 2. 各アイテムに対して pgstat_drop_entry を呼び出し、統計エントリの削除を試みる。 3. pgstat_drop_entry が失敗（false を返却）した場合、not_freed_count をインクリメントする。 4. ループ終了後、not_freed_count が 1 以上であれば、pgstat_request_entry_refs_gc() を呼び出し、参照されなくなったエントリのガベージコレクションを要求する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c (2相コミット処理)
  • src/backend/access/transam/xact.c (トランザクション終了処理)
  • **呼び出し先**:
  • pgstat_drop_entry
  • pgstat_request_entry_refs_gc

関連データ構造

  • xl_xact_stats_item: 削除対象の統計情報を記述する構造体。

失敗モードと境界条件

  • pgstat_drop_entry が失敗した場合、即座にエラーとはならず、カウンタを保持して最後にガベージコレクションをトリガーする。

保守担当者向けメモ

  • is_redo 引数は現在の実装では使用されていない。
  • objid の再構築ロジック（hi と lo のビットシフト）は、統計アイテムのシリアライズ形式に依存している。
