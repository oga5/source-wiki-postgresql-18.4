locate_var_of_level_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/var.c:52

目的

式ツリーの走査中に、特定のクエリレベルに属する変数の位置情報を保持するためのコンテキスト構造体です。主にエラー報告や変数の特定処理において、対象変数のパース位置とサブクエリの階層深さを追跡するために使用されます。

シグネチャまたは形状

typedef struct
{
	int			var_location;
	int			sublevels_up;
} locate_var_of_level_context;

入力と出力

  • var_location: 走査中に発見された変数のパース位置（出力として格納）。
  • sublevels_up: 探索対象とするクエリの階層深さ（入力として設定）。

前提条件と不変条件

  • sublevels_up は、走査対象の式ツリーに対して、どの階層の変数を探索するかを定義する非負の整数である必要があります。

アルゴリズムまたは振る舞い

本構造体は expression_tree_walker を利用した走査処理において、状態保持用オブジェクトとして渡されます。走査関数は、現在のノードが指定された sublevels_up に合致する変数であるかを判定し、合致した場合にその変数の位置情報を var_location に記録します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/util/var.c 内の locate_var_of_level 関数が本構造体をインスタンス化し、走査処理に利用しています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Var: 式ツリー内の変数ノード。
  • PlaceHolderVar: オプティマイザの文脈で Var と同様に扱われるノード。

失敗モードと境界条件

  • 該当するレベルの変数が式ツリー内に存在しない場合、var_location は初期値（通常は -1 等）のままとなります。

保守担当者向けメモ

  • 本構造体は locate_var_of_level 関数専用のコンテキストとして設計されています。
  • sublevels_up の管理はサブクエリのスコープ判定に直結するため、走査ロジックを変更する際は、この値の増減が正しく行われているかを確認してください。
