stats_fill_fcinfo_from_arg_pairs

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/stat_utils.c:292

目的

可変長引数（名前/値のペア）として渡された統計更新用の引数リストを解析し、内部的な FunctionCallInfo 構造体へマッピングします。

シグネチャまたは形状

bool stats_fill_fcinfo_from_arg_pairs(FunctionCallInfo pairs_fcinfo,
                                      FunctionCallInfo positional_fcinfo,
                                      struct StatsArgInfo *arginfo)

入力と出力

  • **入力**:
  • pairs_fcinfo: 可変長引数を含む FunctionCallInfo。
  • positional_fcinfo: マッピング先となる FunctionCallInfo。
  • arginfo: 引数名と型情報の定義配列。
  • **出力**:
  • bool: マッピングが正常に完了した場合は true、型不一致等で一部の引数がスキップされた場合は false を返します。

前提条件と不変条件

  • pairs_fcinfo に含まれる引数は、名前と値のペアであるため、合計数が偶数である必要があります。
  • arginfo 配列は argname が NULL で終端されている必要があります。

アルゴリズムまたは振る舞い

1. positional_fcinfo の全引数を初期化（value = 0, isnull = true）します。 2. extract_variadic_args を用いて可変長引数を抽出します。引数が奇数の場合は ERROR を発行します。 3. 引数リストを2つずつ走査し、以下の処理を行います。

  • 名前部分が TEXT 型でない場合、または NULL の場合は ERROR を発行します。
  • 名前が "version" の場合は無視します（将来の互換性用）。
  • get_arg_by_name を使用して引数名を arginfo と照合します。
  • stats_check_arg_type で型整合性を確認し、成功すれば positional_fcinfo に値を代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/attribute_stats.c:1025
  • src/backend/statistics/relation_stats.c:235
  • **呼び出し先**:
  • extract_variadic_args
  • get_arg_by_name
  • stats_check_arg_type
  • ereport

関連データ構造

  • FunctionCallInfo: PostgreSQLの関数呼び出し情報構造体。
  • StatsArgInfo: 統計更新関数の引数定義を保持する構造体。

失敗モードと境界条件

  • **ERROR**: 引数数が奇数の場合、名前が NULL の場合、または名前が TEXT 型でない場合に発生します。
  • **戻り値 false**: get_arg_by_name で引数名が見つからない、または stats_check_arg_type で型チェックに失敗した場合に発生します。

保守担当者向けメモ

  • "version" 引数は現在無視されますが、将来的に古い統計情報の解釈に使用される可能性があるため、この挙動を変更する際は注意してください。
  • 本関数はバリデーション層の一部であり、計算ロジックは含みません。
