float8in_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/float.c:394

目的

文字列形式の数値を double 型（float8）に変換します。プラットフォーム固有の strtod() の挙動差異を吸収し、PostgreSQL が期待する標準的な浮動小数点表現（NaN, Infinity 等を含む）を解析します。

シグネチャまたは形状

float8 float8in_internal(char *num, char **endptr_p, const char *type_name, const char *orig_string, struct Node *escontext)

入力と出力

  • **入力**:
  • num: 解析対象の文字列。
  • endptr_p: 解析終了位置を格納するポインタ（NULL可）。
  • type_name: エラーメッセージに使用する型名。
  • orig_string: エラーメッセージに使用する元の入力文字列全体。
  • escontext: エラー報告用コンテキスト。
  • **出力**:
  • 変換後の float8 値。

前提条件と不変条件

  • 入力文字列の先頭の空白は自動的にスキップされます。
  • 空文字列はエラーとして扱われます。
  • strtod() が失敗した場合、NaN や inf などの特殊な文字列パターンを個別に判定します。
  • endptr_p が NULL の場合、文字列の末尾まで解析が完了していない（余分な文字がある）場合はエラーとなります。

アルゴリズムまたは振る舞い

1. 先頭の空白をスキップします。 2. strtod() を呼び出し、数値を変換します。 3. strtod() が失敗（endptr == num または errno != 0）した場合、以下の特殊値を判定します：

  • NaN, Infinity (および +/- 符号付き), inf (および +/- 符号付き)。

    • 4. ERANGE エラーが発生した場合、値が 0.0 か、あるいは HUGE_VAL を超える極端な値であるかを確認し、範囲外であればエラーを報告します。 5. 末尾の空白をスキップします。 6. endptr_p が NULL で、かつ文字列の末尾に未解析の文字が残っている場合はエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/float.c (内部の型変換関数)
  • src/backend/utils/adt/geo_ops.c
  • src/backend/utils/adt/jsonpath_exec.c
  • **呼び出し先**:
  • strtod() (標準Cライブラリ)
  • pg_strncasecmp()
  • get_float8_nan(), get_float8_infinity()
  • ereturn() (エラー報告用マクロ)

関連データ構造

  • float8 (PostgreSQLの8バイト浮動小数点型)
  • Node (エラーコンテキスト用)

失敗モードと境界条件

  • **無効な構文**: 数値として解釈できない文字列が渡された場合、ERRCODE_INVALID_TEXT_REPRESENTATION でエラーを返します。
  • **範囲外**: 数値が double の表現範囲を超えている場合、ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE でエラーを返します。
  • **空文字列**: 入力が空または空白のみの場合、エラーとなります。

保守担当者向けメモ

  • 本関数はプラットフォーム間の strtod() の差異を吸収する重要なレイヤーです。新しい特殊な数値表現を追加する場合は、pg_strncasecmp による判定ロジックを拡張してください。
  • escontext を使用しているため、呼び出し元でエラーハンドリングが適切に行われることを前提としています。
