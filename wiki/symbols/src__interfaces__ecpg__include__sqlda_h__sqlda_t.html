sqlda_t

定義

  • 種別: typedef (構造体へのエイリアス)
  • 定義箇所: src/interfaces/ecpg/include/sqlda.h:8

目的

ECPG（Embedded SQL for C）において、動的 SQL の実行時に列情報やデータ値を保持するための「SQL Descriptor Area (SQLDA)」構造体に対する、モード非依存の汎用的な型名を提供します。

シグネチャまたは形状

typedef struct sqlda_compat sqlda_t;

※注: プリプロセッサ定義 _ECPG_INFORMIX_H の有無により、sqlda-compat.h または sqlda-native.h で定義される構造体に解決されます。

入力と出力

  • 入力: 動的 SQL 文の実行結果や、バインド変数として渡されるデータ構造のコンテナとして機能します。
  • 出力: データベースから取得した列のメタデータ（型、サイズ、値へのポインタ等）を保持し、アプリケーションへ提供します。

前提条件と不変条件

  • コンパイル時に _ECPG_INFORMIX_H の定義状態が決定されている必要があります。
  • 実行時には、malloc 等で適切にメモリが確保され、memset 等で初期化されていることが期待されます。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、実行時のロジックは持ちません。コンパイル時の条件分岐により、Informix 互換モードまたは PostgreSQL ネイティブモードのいずれかの構造体定義を指し示すスイッチとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/interfaces/ecpg/test/expected/compat_informix-describe.c
  • src/interfaces/ecpg/test/expected/compat_informix-sqlda.c
  • src/interfaces/ecpg/test/expected/compat_oracle-char_array.c
  • 呼び出し先:
  • src/interfaces/ecpg/ecpglib/sqlda.c（メモリ確保やデータ変換の実装において本型が利用されます）

関連データ構造

  • sqlvar_t: SQLDA 内の個々の列情報を保持する構造体。
  • sqlda-compat.h: Informix 互換モード時の構造体定義元。
  • sqlda-native.h: PostgreSQL ネイティブモード時の構造体定義元。

失敗モードと境界条件

  • メモリ割り当て失敗: malloc による確保に失敗した場合、NULL ポインタが返される可能性があります。
  • モード不一致: ビルド設定と実行環境のモードが一致していない場合、データ構造の解釈に不整合が生じる可能性があります。
