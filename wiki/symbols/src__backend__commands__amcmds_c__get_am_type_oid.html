get_am_type_oid

定義

  • src/backend/commands/amcmds.c:128

目的

指定された名前のアクセスメソッド（AM）をシステムカタログ pg_am から検索し、その OID を取得する。また、オプションで AM のタイプ（インデックス用かテーブル用か）の検証を行う。

シグネチャまたは形状

Oid get_am_type_oid(const char *amname, char amtype, bool missing_ok)

入力と出力

  • **入力**:
  • amname: 検索対象のアクセスメソッド名。
  • amtype: 期待する AM タイプ（AMTYPE_INDEX, AMTYPE_TABLE など）。'\0' を指定すると型チェックをスキップする。
  • missing_ok: true の場合、対象が存在しなくてもエラーにせず InvalidOid を返す。
  • **出力**:
  • 成功時: 見つかったアクセスメソッドの Oid。
  • 失敗時: missing_ok が false で対象が存在しない場合、または型が一致しない場合にエラーを発生させる。

前提条件と不変条件

  • syscache（AMNAME）が初期化されている必要がある。
  • amtype が指定されている場合、pg_am 内の amtype カラムと一致しなければならない。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して AMNAME キャッシュから amname に該当するタプルを検索する。 2. タプルが見つかった場合:

  • amtype が '\0' でない場合、amform->amtype と比較し、不一致であれば ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラーを投げる。
  • 一致する場合、amform->oid を取得し、ReleaseSysCache でタプルを解放する。

    • 3. タプルが見つからず、かつ missing_ok が false の場合、ERRCODE_UNDEFINED_OBJECT でエラーを投げる。 4. 最終的に取得した OID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/amcmds.c 内の各ヘルパー関数（get_index_am_oid, get_table_am_oid, get_am_oid など）から呼び出される。
  • **呼び出し先**:
  • SearchSysCache1, HeapTupleIsValid, GETSTRUCT, ReleaseSysCache, ereport, get_am_type_string 等の PostgreSQL 内部 API。

関連データ構造

  • Form_pg_am: pg_am システムカタログの行構造体。
  • HeapTuple: システムカタログから取得したタプル。

失敗モードと境界条件

  • 指定した名前の AM が存在しない場合、missing_ok が false ならエラー。
  • 指定した AM のタイプが期待値と異なる場合、エラー。
  • amname が無効な場合、システムカタログの検索結果に基づきエラーとなる。

保守担当者向けメモ

  • 本関数は pg_am の検索ロジックを抽象化しており、新しい AM 関連のコマンドを追加する際は、直接 syscache を叩くのではなく本関数を利用することが推奨される。
  • amtype の検証ロジックは pg_am のスキーマ定義に依存しているため、カタログ定義の変更時には注意が必要。
