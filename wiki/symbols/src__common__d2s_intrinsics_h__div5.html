div5

定義

  • 種別: function
  • 定義箇所: src/common/d2s_intrinsics.h:150

目的

浮動小数点数から文字列への変換（Ryuアルゴリズム）において、定数 5 による除算を高速に行うためのプリミティブ関数です。

シグネチャまたは形状

static inline uint64 div5(const uint64 x)

入力と出力

  • **入力**: x (uint64) - 除算対象の 64 ビット符号なし整数。
  • **出力**: x / 5 の結果を返す 64 ビット符号なし整数。

前提条件と不変条件

  • 本関数は、コンパイル時に RYU_32_BIT_PLATFORM が定義されている環境において、除算命令を回避するために最適化されます。
  • 入力値 x は 64 ビット符号なし整数の範囲内である必要があります。

アルゴリズムまたは振る舞い

定数乗算によるビットシフト演算を利用して除算をエミュレートします。具体的には、x にマジックナンバー 0xCCCCCCCCCCCCCCCD を乗算し、その上位 64 ビット（umulh で取得）をさらに 2 ビット右シフトすることで、除算結果を算出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/d2s.c:81
  • src/common/d2s.c:413
  • **呼び出し先**:
  • umulh (64 ビット整数の積の上位 64 ビットを返す関数)

関連データ構造

  • uint64 (PostgreSQL の標準的な 64 ビット符号なし整数型)

失敗モードと境界条件

  • 64 ビットを超える値のオーバーフローについては、Ryu アルゴリズムの前提範囲内で使用されることを想定しています。
  • マジックナンバー 0xCCCCCCCCCCCCCCCD は 5 で割るための定数であり、この値を変更する場合は数学的な正当性の検証が必要です。

保守担当者向けメモ

  • 本関数は Clang x64 が生成するコードと同等のマジックナンバーを使用しています。
  • umulh の実装がプラットフォーム（64 ビット環境か否か）に依存するため、移植性を確保しつつパフォーマンスを維持する設計となっています。
