uuid_internal_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_uuid.c:30

目的

2つの pg_uuid_t 型の値の大小関係を、バイト列の比較に基づいて判定する。

シグネチャまたは形状

static int uuid_internal_cmp(const pg_uuid_t *arg1, const pg_uuid_t *arg2)

入力と出力

  • **入力**: 比較対象となる pg_uuid_t 型へのポインタ arg1 および arg2。
  • **出力**: memcmp の結果を返す。
  • 0: 両者が等しい場合。
  • 0 未満: arg1 が arg2 より小さい場合。
  • 0 より大きい: arg1 が arg2 より大きい場合。

前提条件と不変条件

  • arg1 および arg2 は有効な pg_uuid_t 構造体を指している必要がある。
  • UUID の比較は UUID_LEN バイトのメモリ領域に対する辞書順（バイト単位）の比較として定義される。

アルゴリズムまたは振る舞い

arg1->data と arg2->data を memcmp 関数を用いて UUID_LEN バイト分比較し、その結果をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_uuid.c 内の比較演算子関数（gbt_uuidgt, gbt_uuidge, gbt_uuideq, gbt_uuidle, gbt_uuidlt）から呼び出される。
  • 同ファイル内の gbt_uuidkey_cmp から、uuidKEY 構造体の範囲比較のために呼び出される。
  • src/backend/utils/adt/uuid.c 内の比較関数からも呼び出される。
  • **呼び出し先**: memcmp（標準 C ライブラリ）。

関連データ構造

  • pg_uuid_t: UUID を保持する構造体。
  • uuidKEY: btree_gist において UUID の範囲を保持するために使用される構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作を引き起こす可能性がある。
  • UUID_LEN は固定長であるため、バッファオーバーフローの懸念はない。

保守担当者向けメモ

  • 本関数は src/backend/utils/adt/uuid.c と contrib/btree_gist/btree_uuid.c の両方で定義されている。変更時は両方の実装の一貫性に注意すること。
  • バイト列比較はエンディアンに依存する可能性があるが、PostgreSQL の UUID 型はネットワークバイトオーダー（ビッグエンディアン）で格納されるため、プラットフォーム間で一貫した順序が保証される。
