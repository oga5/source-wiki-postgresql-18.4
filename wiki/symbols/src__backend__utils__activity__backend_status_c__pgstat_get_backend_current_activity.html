pgstat_get_backend_current_activity

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:995

目的

指定されたプロセスID（PID）を持つバックエンドプロセスの現在のクエリ文字列を取得します。

シグネチャまたは形状

const char *pgstat_get_backend_current_activity(int pid, bool checkUser);

入力と出力

  • **入力**:
  • pid: 検索対象のバックエンドプロセスID。
  • checkUser: 呼び出し元がスーパーユーザーでない場合に、対象バックエンドの所有者と現在のユーザーを照合するかどうかを指定するフラグ。
  • **出力**:
  • 成功時: 現在実行中のクエリ文字列（const char *）。
  • 失敗時: 以下のいずれかのエラーメッセージ文字列を返します。
  • "<insufficient privilege>": 権限不足。
  • "<command string not enabled>": クエリ文字列が有効ではない。
  • "<backend information not available>": 指定されたPIDが見つからない。

前提条件と不変条件

  • BackendStatusArray が初期化済みであること。
  • 共有メモリ上のデータ読み取りには、st_changecount を用いた楽観的並行制御プロトコルが適用されます。

アルゴリズムまたは振る舞い

1. BackendStatusArray を走査し、指定された pid と一致する st_procpid を持つエントリを検索します。 2. 検索中、各エントリに対して以下の非ブロッキング読み取りプロトコルを実行します。

  • pgstat_begin_read_activity で st_changecount を記録。
  • データを読み取り、pgstat_end_read_activity で再度 st_changecount を記録。
  • pgstat_read_activity_complete で読み取りの一貫性を検証。不一致の場合はリトライします。

    • 3. 一致するエントリが見つかった場合:

  • checkUser が true かつ非スーパーユーザーであれば、所有者権限を確認します。
  • クエリ文字列が空であれば専用のメッセージを返します。
  • それ以外の場合は pgstat_clip_activity を介してクエリ文字列を返します。

    • 4. 全エントリを走査しても見つからない場合は、エラーメッセージを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/deadlock.c:1128
  • **呼び出し先**:
  • pgstat_begin_read_activity (マクロ)
  • pgstat_end_read_activity (マクロ)
  • pgstat_read_activity_complete
  • CHECK_FOR_INTERRUPTS
  • superuser
  • GetUserId
  • pgstat_clip_activity

関連データ構造

  • PgBackendStatus: バックエンドの状態を保持する共有メモリ上の構造体。
  • BackendStatusArray: 全バックエンドのステータスを格納する共有メモリ上の配列。

失敗モードと境界条件

  • 指定したPIDが存在しない場合、"<backend information not available>" を返します。
  • 権限チェックに失敗した場合、"<insufficient privilege>" を返します。
  • 読み取り中に共有メモリ上のデータが更新され続けた場合、CHECK_FOR_INTERRUPTS() を呼び出しつつリトライを繰り返します。

保守担当者向けメモ

  • 本関数は st_activity_raw を直接参照するため、マルチバイト文字の切り詰めについては pgstat_clip_activity の実装に依存しています。
  • 戻り値の文字列はメモリリークの可能性がありますが、現状の設計では許容されています。
