CheckBSDAuth

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth.c:2173

目的

BSD認証システムを使用して、クライアントから提供されたパスワードを検証し、認証を完了させる。

シグネチャまたは形状

static int CheckBSDAuth(Port *port, char *user)

入力と出力

  • **入力**:
  • port: 接続状態を保持する Port 構造体へのポインタ。
  • user: 認証対象のユーザー名。
  • **出力**:
  • STATUS_OK: 認証成功。
  • STATUS_ERROR: 認証失敗。
  • STATUS_EOF: クライアントからのパスワード受信失敗。

前提条件と不変条件

  • sendAuthRequest を通じてクライアントにパスワード要求が送信済みであること。
  • auth_userokay は内部でパスワード文字列をゼロクリアする仕様である。

アルゴリズムまたは振る舞い

1. sendAuthRequest を呼び出し、クライアントへ AUTH_REQ_PASSWORD を送信する。 2. recv_password_packet でクライアントからパスワード文字列を受信する。受信失敗時は STATUS_EOF を返す。 3. auth_userokay を呼び出し、BSD認証システム（auth-postgresql サービス）に対してユーザー名とパスワードを照合する。 4. 受信したパスワード文字列を pfree で解放する。 5. 認証結果が偽であれば STATUS_ERROR を返す。 6. 認証成功時は set_authn_id を呼び出して認証済みIDを確定させ、STATUS_OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c 内の認証ディスパッチャー（ClientAuthentication 等）。
  • **呼び出し先**:
  • sendAuthRequest
  • recv_password_packet
  • auth_userokay
  • pfree
  • set_authn_id

関連データ構造

  • Port: 接続情報を保持する構造体。

失敗モードと境界条件

  • クライアントがパスワードを送信せずに切断した場合、STATUS_EOF を返す。
  • BSD認証システムによる検証が拒否された場合、STATUS_ERROR を返す。

保守担当者向けメモ

  • 本関数は auth_userokay を利用して外部認証を行うため、BSD認証環境が正しく設定されている必要がある。
  • auth_userokay がパスワード文字列を破壊的に変更するため、呼び出し後のパスワード文字列の再利用は不可。
