heap_toast_delete

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heaptoast.c:42

目的

タプル削除時に、そのタプルに関連付けられたTOASTテーブル内の外部データをクリーンアップ（カスケード削除）します。

シグネチャまたは形状

void heap_toast_delete(Relation rel, HeapTuple oldtup, bool is_speculative)

入力と出力

  • rel: 対象リレーションの記述子。
  • oldtup: 削除対象のヒープタプル。
  • is_speculative: 推測的挿入（speculative insertion）に関連する削除かどうかを示すフラグ。
  • 戻り値: なし。

前提条件と不変条件

  • rel は通常のテーブル（RELKIND_RELATION）またはマテリアライズドビュー（RELKIND_MATVIEW）である必要があります。TOASTテーブル自体に対して再帰的に呼び出すことは禁止されています。
  • tupleDesc->natts は MaxHeapAttributeNumber 以下である必要があります。

アルゴリズムまたは振る舞い

1. リレーションの種類を検証し、アサーションで確認します。 2. heap_deform_tuple を使用して、削除対象タプルを属性値（toast_values）とNULLフラグ（toast_isnull）の配列に分解します。 3. 分解したデータを用いて toast_delete_external を呼び出し、外部保存されているデータを削除します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heapam.c 内のタプル削除処理（heap_delete 等）から呼び出されます。
  • 呼び出し先: heap_deform_tuple（ヒープタプルの分解）、toast_delete_external（TOASTテーブルの実際の削除処理）。

関連データ構造

  • Relation: リレーションのメタデータ。
  • HeapTuple: 削除対象のタプルデータ。
  • TupleDesc: タプルの属性定義。

失敗モードと境界条件

  • TOASTテーブルに対して本関数を呼び出した場合、アサーション違反により停止します。
  • タプル内に可変長属性（varlena）が存在しない場合でも処理は実行されますが、実質的な外部データ削除は行われません。

保守担当者向けメモ

  • heap_deform_tuple を使用することで、属性数に対して線形時間（O(N)）での処理を実現しています。
  • 本関数はタプル削除のライフサイクル管理において重要な役割を担っており、外部参照のリークを防ぐために必須です。
