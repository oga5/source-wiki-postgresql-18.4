ginRedoClearIncompleteSplit

定義

  • src/backend/access/gin/ginxlog.c:24

目的

GINインデックスのページにおいて、不完全な分割（incomplete split）状態を示すフラグを解除し、WALリプレイ時にページの状態を整合させる。

シグネチャまたは形状

static void ginRedoClearIncompleteSplit(XLogReaderState *record, uint8 block_id)

入力と出力

  • record: 処理対象のWALレコードを保持する XLogReaderState へのポインタ。
  • block_id: 対象ページを特定するためのブロックID。
  • 戻り値: なし。

前提条件と不変条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返した場合のみ、ページ内容の更新が行われる。
  • 処理対象のページは GinPageGetOpaque マクロを通じてGIN固有のページヘッダにアクセス可能である必要がある。

アルゴリズムまたは振る舞い

1. XLogReadBufferForRedo を呼び出し、指定された block_id のバッファを取得する。 2. リプレイが必要な場合（BLK_NEEDS_REDO）、以下の操作を行う：

  • ページヘッダのフラグから GIN_INCOMPLETE_SPLIT ビットをクリアする。
  • ページのLSNをWALレコードの終了位置（record->EndRecPtr）に更新する。
  • MarkBufferDirty を呼び出し、バッファをダーティ状態にする。

    • 3. バッファが有効であれば、UnlockReleaseBuffer を呼び出してロックを解放し、バッファを解放する。

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態を管理する構造体。
  • Buffer: PostgreSQLのバッファ管理用ハンドル。
  • Page: ページデータへのポインタ。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO 以外を返した場合、ページ更新はスキップされる。
  • バッファが無効な場合は UnlockReleaseBuffer は呼び出されない。

保守担当者向けメモ

  • 本関数はGINインデックスのWALリプレイ処理の一部であり、ページ分割の完了を記録する際に使用される。
  • GIN_INCOMPLETE_SPLIT フラグの操作は、GINのB-tree構造の整合性を維持するために不可欠である。
