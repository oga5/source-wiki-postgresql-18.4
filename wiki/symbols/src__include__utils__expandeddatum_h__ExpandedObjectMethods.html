ExpandedObjectMethods

定義

  • 種別: struct
  • 定義箇所: src/include/utils/expandeddatum.h:74

目的

展開済み（Expanded）データオブジェクトを、ディスク保存用のフラットな varlena 形式へ変換するためのメソッドテーブルを定義します。

シグネチャまたは形状

typedef struct ExpandedObjectMethods
{
    EOM_get_flat_size_method get_flat_size;
    EOM_flatten_into_method flatten_into;
} ExpandedObjectMethods;

入力と出力

  • get_flat_size: 展開済みオブジェクトからフラット化後のサイズを算出します。
  • flatten_into: 展開済みオブジェクトの内容を、指定されたバッファへフラットな varlena 形式として書き出します。

前提条件と不変条件

  • ExpandedObjectHeader を持つすべての展開済みデータ型は、この構造体のインスタンスを保持し、型固有の変換ロジックを提供する必要があります。

アルゴリズムまたは振る舞い

本構造体は関数ポインタの集合体であり、以下の処理を抽象化します。 1. **サイズ計算**: get_flat_size を呼び出し、変換に必要なメモリ量を決定します。 2. **変換**: flatten_into を呼び出し、メモリ上の複雑なデータ構造をディスク保存に適した連続したメモリ領域へシリアライズします。

呼び出し元と呼び出し先

  • **呼び出し元**: 展開済みオブジェクトを永続化またはフラット化する必要がある上位レイヤー（src/backend/utils/adt/expandeddatum.c 等）。
  • **呼び出し先**: 各データ型（配列、レコード等）で実装された具体的な変換関数（EA_methods や ER_methods 等）。

関連データ構造

  • ExpandedObjectHeader: 本構造体をメンバとして保持し、展開済みオブジェクトの共通ヘッダとして機能します。

失敗モードと境界条件

  • 呼び出し元は、get_flat_size で算出されたサイズ以上のバッファを flatten_into に提供する必要があります。
  • 変換処理中にメモリ不足が発生した場合、メモリコンテキストの制約に従いエラーがスローされます。

保守担当者向けメモ

  • 新しい展開済みデータ型を追加する際は、本構造体のインスタンスを静的に定義し、各メソッドを実装してください。
  • src/backend/utils/adt/array_expanded.c および src/backend/utils/adt/expandedrecord.c が実装の参考になります。
