pg_visibility_map_summary

定義

  • 種別: function
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:268

目的

指定されたリレーションの可視性マップ（VM）を全走査し、all-visible および all-frozen とマークされているブロックの総数を集計して返します。

シグネチャまたは形状

Datum pg_visibility_map_summary(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してリレーションの OID を受け取ります。
  • **出力**: all_visible（全可視ブロック数）と all_frozen（全凍結ブロック数）の2つのフィールドを持つ行（HeapTuple）を返します。

前提条件と不変条件

  • 対象リレーションは AccessShareLock で開かれます。
  • check_relation_relkind(rel) により、可視性マップを持つリレーション種別である必要があります。
  • 実行中に CHECK_FOR_INTERRUPTS() が呼び出され、割り込み可能な状態が維持されます。

アルゴリズムまたは振る舞い

1. 指定されたリレーションを AccessShareLock でオープンします。 2. リレーションの全ブロック数（nblocks）を取得します。 3. 0 から nblocks - 1 までループし、visibilitymap_get_status を用いて各ブロックの VM ビット状態を取得します。 4. VISIBILITYMAP_ALL_VISIBLE および VISIBILITYMAP_ALL_FROZEN ビットを確認し、それぞれのカウンタをインクリメントします。 5. 最後にバッファを解放し、リレーションをクローズした後、集計結果を heap_form_tuple で構築して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(pg_visibility_map_summary) マクロにより、PostgreSQL の関数インターフェースとして登録されています。
  • **呼び出し先**:
  • relation_open, relation_close
  • check_relation_relkind
  • RelationGetNumberOfBlocks
  • visibilitymap_get_status
  • ReleaseBuffer
  • get_call_result_type
  • heap_form_tuple
  • PG_RETURN_DATUM

関連データ構造

  • Relation: 対象リレーションのメタデータ。
  • Buffer: VM ページへのアクセスに使用されるバッファ。
  • TupleDesc: 返却する行型の定義。

失敗モードと境界条件

  • 可視性マップを持たないリレーションが指定された場合、check_relation_relkind によりエラーが発生します。
  • 戻り値の型が複合型でない場合、elog(ERROR) が発生します。
  • 巨大なテーブルに対して実行した場合、全ページ走査による I/O 負荷が発生します。

保守担当者向けメモ

  • 本関数は全ブロックを走査するため、テーブルサイズに比例した実行時間がかかります。
  • vmbuffer はループ内で再利用され、最後に解放されます。
