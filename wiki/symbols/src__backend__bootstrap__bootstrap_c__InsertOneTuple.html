InsertOneTuple

定義

  • 種別: function
  • 定義箇所: src/backend/bootstrap/bootstrap.c:628

目的

ブートストラップモードにおいて、現在開いているシステムカタログ（boot_reldesc）に対して、メモリ上に構築された属性値から新しいタプルを挿入する。

シグネチャまたは形状

void InsertOneTuple(void)

入力と出力

  • **入力**: グローバル変数 numattr（属性数）、attrtypes（属性型配列）、values（属性値配列）、Nulls（NULLフラグ配列）、および boot_reldesc（操作対象のリレーション記述子）。
  • **出力**: なし（副作用としてリレーションへのタプル挿入と Nulls 配列のリセットを行う）。

前提条件と不変条件

  • boot_reldesc が有効なリレーション記述子として開かれていること。
  • numattr および attrtypes が、挿入対象のリレーション定義と整合していること。

アルゴリズムまたは振る舞い

1. DEBUG4 レベルで挿入開始をログ出力する。 2. CreateTupleDesc を使用して、現在の属性定義から一時的な TupleDesc を作成する。 3. heap_form_tuple を呼び出し、values と Nulls から HeapTuple を生成する。 4. 作成した TupleDesc を解放する（attrtypes 自体は解放しない）。 5. simple_heap_insert を使用して、boot_reldesc にタプルを挿入する。 6. heap_freetuple でタプルメモリを解放する。 7. 次のタプル挿入に備え、Nulls 配列の全要素を false にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/bootstrap/bootstrap.c 内のBKIコマンド解析処理から呼び出される。
  • **呼び出し先**: CreateTupleDesc, heap_form_tuple, pfree, simple_heap_insert, heap_freetuple, elog。

関連データ構造

  • boot_reldesc: 操作対象のリレーション記述子。
  • HeapTuple: 挿入されるタプル構造体。
  • TupleDesc: タプルのスキーマ定義。

失敗モードと境界条件

  • numattr が不正な場合、CreateTupleDesc や heap_form_tuple 内でエラーが発生する可能性がある。
  • boot_reldesc が初期化されていない場合、simple_heap_insert でクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は Nulls 配列を再利用するため、呼び出し後に状態がリセットされる点に注意すること。
  • attrtypes は pfree されないため、メモリリークの懸念がある場合は呼び出し元のライフサイクルを確認すること。
