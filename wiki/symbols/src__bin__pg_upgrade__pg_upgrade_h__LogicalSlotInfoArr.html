LogicalSlotInfoArr

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.h:171

目的

論理レプリケーションスロット情報の配列を保持し、アップグレードプロセスにおいてデータベース単位でスロット情報を管理するために使用されます。

シグネチャまたは形状

typedef struct
{
	int			nslots;			/* 論理スロット情報の数 */
	LogicalSlotInfo *slots;		/* 論理スロット情報の配列 */
} LogicalSlotInfoArr;

入力と出力

  • 入力: nslots（スロット数）、slots（LogicalSlotInfo 構造体のポインタ配列）。
  • 出力: なし（データコンテナとして機能）。

前提条件と不変条件

  • slots が指すメモリ領域は、nslots で指定された要素数分だけ有効である必要があります。

アルゴリズムまたは振る舞い

  • データベースごとの論理スロット情報を一括して保持するコンテナとして機能します。
  • info.c 等のモジュールにおいて、スロット情報の収集や表示処理の引数として渡されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/bin/pg_upgrade/check.c:2183
  • src/bin/pg_upgrade/function.c:118
  • src/bin/pg_upgrade/pg_upgrade.c:984
  • src/bin/pg_upgrade/pg_upgrade.h:203
  • 呼び出し先:
  • src/bin/pg_upgrade/info.c:31（print_slot_infos 関数へ引数として渡される）

関連データ構造

  • LogicalSlotInfo: 配列の各要素として格納される個別の論理スロット情報。
  • DbInfo: LogicalSlotInfoArr をメンバとして保持し、データベース単位の情報を管理する構造体。

失敗モードと境界条件

  • nslots が 0 の場合、slots は NULL または空の配列として扱われる必要があります。メモリ割り当て失敗時の挙動については、本構造体自体は関与せず、呼び出し側のメモリ管理ロジックに依存します。

保守担当者向けメモ

  • 本構造体は DbInfo 構造体の一部として組み込まれています。スロット情報の追加や削除を行う際は、nslots の更新と slots 配列の再割り当てが整合していることを確認してください。
