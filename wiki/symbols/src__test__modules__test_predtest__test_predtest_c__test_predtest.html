test_predtest

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_predtest/test_predtest.c:31

目的

PostgreSQL オプティマイザの述語証明ロジック（predicate_implied_by および predicate_refuted_by）の正当性を、実行時のデータセットを用いて検証する。

シグネチャまたは形状

Datum test_predtest(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: SQL クエリ文字列（text 型）。クエリは 2 つのブール値カラムを返す SELECT 文である必要がある。
  • **出力**: 8 つのブール値を含むタプル（検証結果）。

前提条件と不変条件

  • 入力クエリは SELECT 文であり、正確に 2 つのブール値カラムを返す必要がある。
  • make_ands_implicit による正規化は行うが、canonicalize_qual は適用されないため、単純な形式の式が想定される。
  • 強反駁（strong refutation）が成立する場合、弱反駁（weak refutation）も必ず成立しなければならない。

アルゴリズムまたは振る舞い

1. **クエリ実行**: SPI_execute_plan を用いてクエリを実行し、結果セットから論理的な含意・反駁がデータ上で成立しているかを判定する。 2. **実測値の収集**: 各行の値を走査し、s_i_holds（強含意）、w_i_holds（弱含意）、s_r_holds（強反駁）、w_r_holds（弱反駁）の各フラグを更新する。 3. **証明ロジックの実行**: SPI_plan_get_cached_plan からクエリツリーを抽出し、predicate_implied_by および predicate_refuted_by を呼び出す。 4. **比較と警告**: 証明関数の結果と実測値フラグを比較し、矛盾がある場合は elog(WARNING, ...) を発行する。 5. **結果返却**: 検証結果を構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出される（PG_FUNCTION_INFO_V1 により登録）。
  • **呼び出し先**: SPI_connect, SPI_prepare, SPI_execute_plan, SPI_getbinval, SPI_plan_get_cached_plan, make_ands_implicit, predicate_implied_by, predicate_refuted_by, elog。

関連データ構造

  • SPIPlanPtr: クエリ実行プラン。
  • CachedPlan: キャッシュされたプラン。
  • PlannedStmt, Plan: クエリツリー構造。
  • Expr: 評価対象の式。

失敗モードと境界条件

  • **エラー**: SPI_prepare の失敗、クエリが SELECT ではない場合、またはカラム数が 2 以外、あるいは型が BOOLOID 以外の場合に elog(ERROR) を発行する。
  • **警告**: 証明関数の結果と実測値が矛盾する場合、または論理的にあり得ない証明結果（強反駁が真で弱反駁が偽など）が得られた場合に elog(WARNING) を発行する。

保守担当者向けメモ

  • 本関数はクエリ結果の全行を走査するため、大規模データセットでの実行はパフォーマンスに影響する。
  • 警告は証明ロジックのバグを示唆する可能性がある。
  • make_ands_implicit を使用しているが、RestrictInfo ノードが含まれる場合でも predtest.c は適切に処理するため、現状のままで問題ない。
