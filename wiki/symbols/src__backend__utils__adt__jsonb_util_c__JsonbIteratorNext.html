JsonbIteratorNext

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/jsonb_util.c:858

目的

JsonbIterator を使用して、jsonb コンテナ（配列またはオブジェクト）の要素を順次走査し、各要素の型と値を JsonbValue 構造体に抽出します。

シグネチャまたは形状

JsonbIteratorToken JsonbIteratorNext(JsonbIterator **it, JsonbValue *val, bool skipNested)

入力と出力

  • **入力**:
  • it: イテレータへのポインタ。走査中にネストされたコンテナへ潜る際、内部で更新される場合があります。
  • val: 抽出された要素を格納する JsonbValue 構造体へのポインタ。
  • skipNested: true の場合、ネストされたコンテナを再帰的に走査せず、コンテナ全体を一つの値として扱います。
  • **出力**:
  • 戻り値: JsonbIteratorToken 型のトークン（WJB_BEGIN_ARRAY, WJB_ELEM, WJB_END_ARRAY, WJB_BEGIN_OBJECT, WJB_KEY, WJB_VALUE, WJB_END_OBJECT, WJB_DONE など）。
  • val: 走査された要素の情報が書き込まれます。

前提条件と不変条件

  • *it が NULL の場合、val->type を jbvNull に設定し WJB_DONE を返します。
  • オブジェクトのキー走査時、val の型が jbvString でない場合はエラーとなります。
  • ネストされたコンテナを処理する際、内部で iteratorFromContainer が呼び出され、新しいイテレータが palloc されます。

アルゴリズムまたは振る舞い

1. 現在のイテレータ状態（JBI_ARRAY_START, JBI_ARRAY_ELEM, JBI_OBJECT_START, JBI_OBJECT_KEY, JBI_OBJECT_VALUE）に基づき処理を分岐します。 2. 配列またはオブジェクトの開始時に、対応するトークンを返し、状態を要素走査へ遷移させます。 3. 要素走査中、fillJsonbValue を用いて現在の要素を val に格納します。 4. skipNested が false かつ要素がコンテナである場合、iteratorFromContainer を呼び出して再帰的に内部へ潜ります。 5. コンテナの末尾に達した場合、freeAndGetParent を呼び出して親のイテレータへ戻り、終了トークン（WJB_END_ARRAY / WJB_END_OBJECT）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/jsonb_plperl/jsonb_plperl.c および contrib/jsonb_plpython/jsonb_plpython.c での利用が確認されています。
  • **呼び出し先**: fillJsonbValue, iteratorFromContainer, freeAndGetParent, getJsonbOffset, elog。

関連データ構造

  • JsonbIterator: 走査状態を保持する構造体。
  • JsonbValue: JSONB の各要素を表現する構造体。
  • JsonbIteratorToken: 走査の進捗を示す列挙型。

失敗モードと境界条件

  • **エラー**: オブジェクトのキーが文字列型でない場合、elog(ERROR) により中断されます。
  • **状態異常**: 定義されていないイテレータ状態に陥った場合、elog(ERROR) が発生します。
  • **メモリ**: ネストされたコンテナの走査時にイテレータが動的に確保されるため、深いネスト構造ではメモリ消費に注意が必要です。

保守担当者向けメモ

  • 本関数は再帰的な構造を扱うため、スタックオーバーフローを避ける設計になっています。
  • skipNested フラグの扱いは、呼び出し側が構造の深さを制御する重要なインターフェースです。
  • JBE_ADVANCE_OFFSET マクロを使用してバイナリデータのオフセットを更新しているため、バイナリ構造の変更時にはこのマクロの整合性を確認してください。
