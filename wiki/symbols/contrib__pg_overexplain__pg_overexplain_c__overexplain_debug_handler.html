overexplain_debug_handler

定義

  • 種別: function
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:108

目的

EXPLAIN コマンドの debug オプションを解析し、その値を ExplainState 内の拡張オプション構造体に反映させるためのハンドラ関数です。

シグネチャまたは形状

static void overexplain_debug_handler(ExplainState *es, DefElem *opt, ParseState *pstate)

入力と出力

  • **入力**:
  • es: ExplainState 構造体へのポインタ。
  • opt: 解析対象の DefElem（debug オプションの定義）。
  • pstate: ParseState 構造体へのポインタ（本関数内では未使用）。
  • **出力**: なし（副作用として es 内の状態を更新）。

前提条件と不変条件

  • es は有効な ExplainState である必要があります。
  • overexplain_ensure_options(es) が呼び出されることで、ExplainState に overexplain_options がアタッチされていることが保証されます。

アルゴリズムまたは振る舞い

1. overexplain_ensure_options(es) を呼び出し、現在の ExplainState に関連付けられた overexplain_options 構造体を取得します。 2. defGetBoolean(opt) を使用して、debug オプションに指定された真偽値を取得します。 3. 取得した値を options->debug に代入します。

呼び出し元と呼び出し先

  • **呼び出し元**: RegisterExtensionExplainOption への登録を通じて、EXPLAIN コマンドのオプション解析プロセスから呼び出されます。
  • **呼び出し先**:
  • overexplain_ensure_options: 拡張状態の取得および初期化。
  • defGetBoolean: オプション値の解析。

関連データ構造

  • ExplainState: PostgreSQL の EXPLAIN 実行状態を保持する構造体。
  • overexplain_options: 本拡張モジュールが ExplainState に保持するカスタムオプション状態。
  • DefElem: SQL 文のオプション定義を保持する構造体。

失敗モードと境界条件

  • opt が debug オプションとして期待される形式でない場合、defGetBoolean がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は EXPLAIN のオプション登録インターフェースに適合するように設計されています。将来的に debug オプションの挙動を変更する場合は、このハンドラと overexplain_options 構造体の両方を更新してください。
