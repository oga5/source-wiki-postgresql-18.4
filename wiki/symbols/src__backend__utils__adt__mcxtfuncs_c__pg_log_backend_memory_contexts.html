pg_log_backend_memory_contexts

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mcxtfuncs.c:263

目的

指定されたプロセスID（PID）を持つPostgreSQLバックエンドまたは補助プロセスに対し、メモリコンテキストのログ出力を要求するシグナル（PROCSIG_LOG_MEMORY_CONTEXT）を送信します。

シグネチャまたは形状

Datum pg_log_backend_memory_contexts(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT32(0) を通じて対象プロセスのPIDを受け取ります。
  • **出力**: 成功した場合は true、失敗または対象プロセスが見つからない場合は false を返します。

前提条件と不変条件

  • 対象PIDが有効なPostgreSQLサーバープロセスである必要があります。
  • セキュリティ上の理由から、通常はスーパーユーザー権限が必要です。
  • シグナル送信時にプロセスが終了している可能性があるため、関数はエラーで中断せず、警告を発して false を返す設計になっています。

アルゴリズムまたは振る舞い

1. BackendPidGetProc(pid) および AuxiliaryPidGetProc(pid) を使用して、指定されたPIDに対応する PGPROC 構造体を検索します。 2. プロセスが見つからない場合、WARNING を発行し false を返します。 3. プロセスが見つかった場合、GetNumberFromPGProc でプロセス番号を取得します。 4. SendProcSignal を呼び出し、対象プロセスに PROCSIG_LOG_MEMORY_CONTEXT シグナルを送信します。 5. シグナル送信に失敗した場合は WARNING を発行し false を返します。成功した場合は true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されることを想定しています。
  • **呼び出し先**:
  • BackendPidGetProc / AuxiliaryPidGetProc (プロセス検索)
  • GetNumberFromPGProc (プロセス番号取得)
  • SendProcSignal (シグナル送信)
  • ereport (ログ出力)

関連データ構造

  • PGPROC: プロセス情報を保持する構造体。
  • PROCSIG_LOG_MEMORY_CONTEXT: 送信されるシグナル種別。

失敗モードと境界条件

  • **PID無効**: 指定されたPIDが現在稼働中のバックエンドまたは補助プロセスと一致しない場合、警告を出して終了します。
  • **競合状態**: シグナル送信直前にプロセスが終了した場合、SendProcSignal が失敗する可能性がありますが、これも警告として処理されます。

保守担当者向けメモ

  • 本関数はDoS攻撃を防ぐため、実行権限の制限が適切に適用されていることを確認してください。
  • SendProcSignal の戻り値チェックにより、ループ処理内での実行時にも全体のクエリが中断されないよう配慮されています。
