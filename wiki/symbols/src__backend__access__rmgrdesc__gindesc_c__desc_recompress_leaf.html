desc_recompress_leaf

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/gindesc.c:20

目的

GINインデックスのリーフページ再圧縮操作に関するWALレコードを解析し、人間が読める形式の文字列としてバッファに出力する。

シグネチャまたは形状

static void desc_recompress_leaf(StringInfo buf, ginxlogRecompressDataLeaf *insertData)

入力と出力

  • **入力**:
  • buf: 出力先となる StringInfo バッファ。
  • insertData: 解析対象の ginxlogRecompressDataLeaf 構造体へのポインタ。
  • **出力**: なし（buf に解析結果が追記される）。

前提条件と不変条件

  • insertData は、WALレコードのデータ部から適切にキャストされた有効なポインタであること。
  • insertData の直後に、nactions 分の可変長データがバイナリ形式で配置されていること。

アルゴリズムまたは振る舞い

1. insertData->nactions を読み取り、アクションの総数をバッファに書き出す。 2. insertData の直後から始まるバイナリデータ領域をポインタ演算で走査する。 3. 各アクションについて以下の処理を繰り返す:

  • セグメント番号 (a_segno) とアクション種別 (a_action) を取得。
  • a_action が GIN_SEGMENT_INSERT または GIN_SEGMENT_REPLACE の場合、SizeOfGinPostingList を用いてポスティングリストのサイズを計算し、ポインタをスキップする。
  • a_action が GIN_SEGMENT_ADDITEMS の場合、追加されるアイテム数 (nitems) を読み取り、アイテムデータ分ポインタをスキップする。
  • a_action に応じて、対応する文字列（add, delete, insert, replace）を buf に追記する。

    • 4. 未知の a_action に遭遇した場合はエラーメッセージを出力して処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/rmgrdesc/gindesc.c 内の gin_desc 関数（複数箇所から呼び出される）。
  • **呼び出し先**: appendStringInfo（文字列構築用）、SizeOfGinPostingList（サイズ計算用）。

関連データ構造

  • ginxlogRecompressDataLeaf: 再圧縮操作のメタデータを含む構造体。
  • StringInfo: PostgreSQLの可変長文字列バッファ。
  • GinPostingList: GINインデックスのポスティングリスト構造体。

失敗モードと境界条件

  • 未知の a_action 値が渡された場合、unknown action を出力して処理を中断する。
  • WALレコードのバイナリレイアウトが期待される構造と一致しない場合、ポインタ演算によりメモリ破壊や不正な読み取りが発生する可能性がある。

保守担当者向けメモ

  • 本関数はWALレコードのバイナリレイアウトに強く依存している。ginxlogRecompressDataLeaf や関連する構造体の定義が変更された場合、本関数のポインタ演算ロジックも同期して更新する必要がある。
  • walbuf のインクリメント処理は、各アクションのデータ構造と厳密に一致させること。
