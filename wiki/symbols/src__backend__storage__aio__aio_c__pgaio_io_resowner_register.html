pgaio_io_resowner_register

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio.c:408

目的

非同期I/Oハンドル（PgAioHandle）を特定のResourceOwnerに関連付け、トランザクション境界でのリソース管理を可能にする。

シグネチャまたは形状

static void pgaio_io_resowner_register(PgAioHandle *ioh, struct ResourceOwnerData *resowner)

入力と出力

  • **入力**:
  • ioh: 登録対象のPgAioHandle構造体へのポインタ。
  • resowner: 関連付けるResourceOwnerData構造体へのポインタ。
  • **出力**: なし（副作用としてiohの状態を更新）。

前提条件と不変条件

  • ioh->resowner が NULL であること（既に他のリソースオーナーに登録されていないこと）。
  • resowner が有効なポインタであること。

アルゴリズムまたは振る舞い

1. ioh->resowner が空であることをアサートで確認する。 2. resowner が有効であることをアサートで確認する。 3. ResourceOwnerRememberAioHandle を呼び出し、指定されたリソースオーナーの管理リストに ioh->resowner_node を追加する。 4. ioh->resowner フィールドに resowner を代入し、関連付けを完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/aio.c:220（pgaio_io_acquire 内での呼び出しが確認されている）。
  • **呼び出し先**: ResourceOwnerRememberAioHandle（utils/resowner.h で定義されるリソース管理関数）。

関連データ構造

  • PgAioHandle: I/Oハンドルの状態を保持する構造体。
  • ResourceOwnerData: トランザクションやサブトランザクションに関連付けられたリソースを追跡する構造体。

失敗モードと境界条件

  • ioh->resowner が既に設定されている場合、アサート違反によりプロセスが停止する。
  • resowner が NULL の場合、アサート違反によりプロセスが停止する。

保守担当者向けメモ

  • 本関数は static であり、aio.c 内のI/Oハンドル取得処理からのみ呼び出されることを想定している。
  • リソースオーナーによるクリーンアップ処理（AtEOXact_Aio 等）と整合性を保つため、登録時は必ず ioh->resowner_node を介して管理リストに追加する必要がある。
