infoname

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_regex/test_regex.c:621

目的

正規表現エンジンのフラグやメタ情報に関連付けられたビット値と、それに対応するテキスト表現を保持するための構造体です。

シグネチャまたは形状

struct infoname
{
    int         bit;
    const char *text;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • bit: 整数型のフラグ値。
  • text: 対応するフラグの名称を示す文字列リテラル。

前提条件と不変条件

  • text フィールドは静的な文字列リテラルを指すことが想定されています。
  • 本構造体の配列（infonames[]）は、正規表現のフラグ解析や結果出力の際に参照されます。

アルゴリズムまたは振る舞い

本構造体自体はロジックを持ちませんが、infonames[] 配列の要素として、ビットフラグから人間が可読なテキストへの変換、またはその逆の検索処理に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/test/modules/test_regex/test_regex.c:626: infonames[] 配列の初期化に使用。
  • src/test/modules/test_regex/test_regex.c:643: infonames[] を走査するループ内でポインタ変数 inf として参照。
  • 呼び出し先:
  • 情報なし。

関連データ構造

  • infonames[]: infoname 型の静的配列。

失敗モードと境界条件

  • 構造体定義自体に失敗モードはありません。
  • infonames[] を走査する際、終端条件が正しく設定されていない場合、メモリ上の不正な領域を参照するリスクがあります。

保守担当者向けメモ

  • 本構造体は、正規表現のテスト結果におけるフラグ表示の統一を目的としています。新しいフラグを追加する場合は、infonames[] 配列にもエントリを追加してください。
