has_join_restriction

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinrels.c:1178

目的

指定されたリレーション（RelOptInfo）が、クエリ内の外部結合や LATERAL 参照、プレースホルダ式によって課される結合順序の制約を受けているかどうかを判定する。

シグネチャまたは形状

static bool has_join_restriction(PlannerInfo *root, RelOptInfo *rel)

入力と出力

  • **入力**:
  • root: クエリ全体のプランナ情報。
  • rel: 評価対象のリレーション情報。
  • **出力**:
  • bool: 制約が存在し、結合順序が制限される場合は true、それ以外は false。

前提条件と不変条件

  • rel->relids は、対象リレーションに含まれるベースリレーションの集合を表すビットマップセットであること。
  • root->placeholder_list および root->join_info_list が適切に初期化されていること。

アルゴリズムまたは振る舞い

以下のいずれかの条件を満たす場合に true を返す： 1. **LATERAL 参照**: rel が lateral_relids または lateral_referencers を保持している場合。 2. **プレースホルダ制約**: root->placeholder_list 内のいずれかの PlaceHolderInfo について、rel がその評価位置（ph_eval_at）の真部分集合である場合。 3. **外部結合制約**: root->join_info_list 内の SpecialJoinInfo（FULL JOIN を除く）について、以下の条件を満たす場合：

  • rel が結合の左辺または右辺と一部重複しているが、結合全体を完全には含んでいない（つまり、結合をまたぐような中途半端なリレーションである）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinrels.c 内の関数（行96, 171付近で参照を確認）。
  • **呼び出し先**:
  • bms_is_subset, bms_equal, bms_overlap などのビットマップセット操作関数。
  • lfirst マクロによるリスト走査。

関連データ構造

  • RelOptInfo: 結合対象のリレーション情報。
  • PlaceHolderInfo: プレースホルダ式の評価位置情報。
  • SpecialJoinInfo: 外部結合等の結合制約情報。

失敗モードと境界条件

  • FULL JOIN はこの関数の制約チェックから除外される（別のメカニズムで順序が保護されるため）。
  • rel が空のリレーションセットである場合や、制約リストが空の場合は false を返す。

保守担当者向けメモ

  • 本関数は、オプティマイザが結合順序を決定する際の正当性検証の要となる。
  • SpecialJoinInfo の min_lefthand / min_righthand との包含関係チェックは、結合の「分割不可性」を保証するために重要である。
