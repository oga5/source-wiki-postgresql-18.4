BlockSamplerData

定義

  • **種別**: struct
  • **定義箇所**: src/include/utils/sampling.h:28

目的

リレーションのブロックサンプリングを行う際の状態を保持するための構造体です。クエリ最適化や統計情報収集（ANALYZE）において、全ブロックを走査せずに指定したサンプルサイズを効率的に抽出するために使用されます。

シグネチャまたは形状

typedef struct
{
	BlockNumber N;				/* number of blocks, known in advance */
	int			n;				/* desired sample size */
	BlockNumber t;				/* current block number */
	int			m;				/* blocks selected so far */
	pg_prng_state randstate;	/* random generator state */
} BlockSamplerData;

入力と出力

  • **入力**: N（全ブロック数）、n（目標サンプルサイズ）、および乱数生成器の状態（randstate）。
  • **出力**: サンプリング過程における現在の走査位置（t）および選択済みブロック数（m）の更新。

前提条件と不変条件

  • N は事前に判明している必要があります。
  • pg_prng_state を介して乱数生成器の状態が適切に管理されている必要があります。
  • 浮動小数点演算を含むため、極端なケースでの丸め誤差に対する安全策がアルゴリズム内で講じられています。

アルゴリズムまたは振る舞い

クヌースの「アルゴリズムS」に基づき、以下の手順で動作します。 1. BlockSampler_Init で全ブロック数とサンプルサイズを初期化します。 2. BlockSampler_Next を呼び出すたびに、残りのブロック数と選択すべきサンプル数に基づき、確率的にブロックをスキップまたは選択します。 3. 全データを走査することなく、ランダムなサンプルを取得します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/analyze.c において、BlockSamplerData 型の変数が宣言および利用されています。
  • **呼び出し先**: 乱数生成のために common/pg_prng.h の機能を利用します。
  • **注記**: 上記以外の具体的な呼び出し元・呼び出し先関係については、提供されたソース情報からは網羅的ではありません。

関連データ構造

  • BlockSampler: BlockSamplerData へのポインタ型。
  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • サンプルサイズが全ブロック数を超える場合や、0以下の場合の挙動については、実装側の BlockSampler_Next 等のロジックに依存します。
  • 浮動小数点演算の精度限界により、極端な入力値に対しては丸め誤差が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は ANALYZE コマンドや外部データラッパー（FDW）で利用されます。
  • anl_ で始まる古いAPIとの混同を避け、新規開発では本構造体と関連関数（BlockSampler_Init/Next）を使用してください。
  • 実装の詳細は src/backend/utils/misc/sampling.c を参照してください。
