qsort_tuple_int32_compare

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesort.c:541

目的

int32 型のキーを持つタプル同士を比較し、ソート順序を決定する。

シグネチャまたは形状

static pg_attribute_always_inline int
qsort_tuple_int32_compare(SortTuple *a, SortTuple *b, Tuplesortstate *state)

入力と出力

  • **入力**:
  • a, b: 比較対象の SortTuple 構造体へのポインタ。
  • state: ソート状態を保持する Tuplesortstate 構造体へのポインタ。
  • **出力**:
  • a < b の場合は負の整数、a > b の場合は正の整数、等しい場合は 0 を返す。

前提条件と不変条件

  • state->base.sortKeys[0] に有効なソートキー設定が存在すること。
  • a および b は有効な SortTuple を指していること。

アルゴリズムまたは振る舞い

1. ApplyInt32SortComparator を呼び出し、第1ソートキー（datum1）に基づいて比較を行う。 2. 比較結果が 0 でない（順序が確定した）場合は、その値を即座に返す。 3. 比較結果が 0 であり、かつ state->base.onlyKey が存在する場合（単一キーソートの場合）、これ以上の比較は不要と判断し 0 を返す。 4. それ以外の場合（複数キーソートの場合）、state->base.comparetup_tiebreak を呼び出し、タイブレーク（同点決勝）処理を実行して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesort.c 内のソート処理（ST_COMPARE マクロ経由）。
  • **呼び出し先**:
  • ApplyInt32SortComparator
  • state->base.comparetup_tiebreak（関数ポインタ経由）

関連データ構造

  • SortTuple: ソート対象のタプル表現。
  • Tuplesortstate: ソート操作の全体状態を保持する構造体。

失敗モードと境界条件

  • a または b が NULL の場合、あるいは state が不正な場合は未定義の動作となる。
  • ApplyInt32SortComparator が内部で発生させるエラーについては、呼び出し元の例外処理に依存する。

保守担当者向けメモ

  • 本関数は pg_attribute_always_inline が指定されており、インライン展開されることを前提としている。
  • lib/sort_template.h を利用したソート処理の一部として使用されるため、変更時はテンプレート側の制約に注意すること。
