remove_rel_from_restrictinfo

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:673

目的

結合削除に伴い、RestrictInfo 構造体内に保持されているリレーション ID セットから、不要になったリレーション（relid および ojrelid）を削除し、整合性を維持します。

シグネチャまたは形状

void remove_rel_from_restrictinfo(RestrictInfo *rinfo, int relid, int ojrelid)

入力と出力

  • rinfo: 修正対象の RestrictInfo 構造体へのポインタ。
  • relid: 削除対象のリレーション ID。
  • ojrelid: 削除対象の外部結合リレーション ID。
  • 出力: なし（rinfo の各ビットマップセットが直接更新されます）。

前提条件と不変条件

  • RestrictInfo が共有されている可能性を考慮し、処理前に各ビットマップセット（clause_relids, required_relids 等）を bms_copy で複製して独立性を確保します。
  • OR 句を含む場合、再帰的にサブ句に対しても同様の削除処理を行います。

アルゴリズムまたは振る舞い

1. RestrictInfo が保持する以下のビットマップセットから、指定された relid および ojrelid を削除します。

  • clause_relids, required_relids, incompatible_relids, outer_relids, left_relids, right_relids

    • 2. restriction_is_or_clause(rinfo) が真の場合、OR 句の各引数（AND 句または RestrictInfo）に対して再帰的に本関数を呼び出し、内部の制約情報も更新します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/plan/analyzejoins.c 内の結合削除処理（remove_useless_joins 等）から呼び出されます。
  • 呼び出し先: bms_copy, bms_del_member, restriction_is_or_clause, is_orclause, is_andclause, castNode 等のノード操作関数。

関連データ構造

  • RestrictInfo: 結合条件やフィルタ条件を保持するプランナの主要構造体。
  • Bitmapset: リレーション ID の集合を管理するために使用。

失敗モードと境界条件

  • rinfo が NULL の場合の挙動は定義されていません。
  • OR 句内の構造が想定外（AND 句または RestrictInfo 以外）である場合、castNode 等でエラーが発生する可能性があります。

保守担当者向けメモ

  • initsplan.c が RestrictInfo を共有する可能性があるため、bms_copy による複製は必須です。
  • 結合削除後のクエリ整合性を保つため、RestrictInfo 内の全リレーションセットを網羅的に更新する必要があります。追加のフィールドが RestrictInfo に導入された場合は、本関数での更新漏れに注意してください。
