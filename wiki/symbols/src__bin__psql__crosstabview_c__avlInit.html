avlInit

定義

  • 種別: function
  • 定義箇所: src/bin/psql/crosstabview.c:437

目的

avl_tree 構造体を初期化し、空の木として使用可能な状態にする。

シグネチャまたは形状

static void avlInit(avl_tree *tree)

入力と出力

  • 入力: tree (avl_tree *) - 初期化対象の AVL 木構造体へのポインタ。
  • 出力: なし（副作用として tree の各メンバを初期化）。

前提条件と不変条件

  • tree は有効なメモリ領域を指している必要がある。
  • pg_malloc0 により tree->end 用のメモリが確保されるため、呼び出し後に avlFree 等で適切にメモリを解放する必要がある。

アルゴリズムまたは振る舞い

1. pg_malloc0 を使用して avl_node 構造体分のメモリを確保し、tree->end（番兵ノード）として割り当てる。 2. 番兵ノードの左右の子ポインタを自身（tree->end）に向けることで、空の木を表現する。 3. tree->count を 0 に設定する。 4. tree->root を番兵ノード（tree->end）に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/crosstabview.c 内の PrintResultInCrosstab 関数から、行用および列用の avl_tree を初期化するために呼び出される。
  • 呼び出し先: pg_malloc0（メモリ確保）。

関連データ構造

  • avl_tree: 木のルート、ノード数、番兵ノードを保持する。
  • avl_node: 木の各ノード。

失敗モードと境界条件

  • メモリ不足時: pg_malloc0 が失敗した場合、PostgreSQL の標準的なエラー処理によりプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は挿入専用の AVL 木を初期化する。削除や検索機能は想定されていないため、この初期化ロジックは挿入処理と対で管理すること。
