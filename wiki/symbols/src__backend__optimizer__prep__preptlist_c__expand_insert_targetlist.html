expand_insert_targetlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/preptlist.c:381

目的

INSERT文のターゲットリスト（tlist）を、対象リレーションの物理的な属性定義と整合するように補完・整列します。

シグネチャまたは形状

static List *expand_insert_targetlist(PlannerInfo *root, List *tlist, Relation rel)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • tlist: パーサから渡されたターゲットリスト。
  • rel: 挿入対象のリレーション（Relation）。
  • **出力**:
  • 欠落した列にNULLが補完され、物理属性順に並べ替えられた新しいターゲットリスト（List *）。

前提条件と不変条件

  • 入力されたターゲットリストの各エントリは、リライタによって既に正しい順序で並べられていることが期待されます。
  • 物理的な属性順序（rel->rd_att）に従い、欠落している属性に対しては適切なNULL値を生成します。
  • 最終的なリストにおいて、resjunk 属性が真であるエントリは、非ジャンク列の後に配置される必要があります。

アルゴリズムまたは振る舞い

1. リレーションの属性数（numattrs）を走査し、各属性に対してターゲットリストのエントリを照合します。 2. 既存のターゲットリストに該当する属性が存在する場合、それを再利用します。 3. 存在しない場合、以下のルールでNULL値を生成します：

  • **削除済み列**: INT4型のNULLを生成。
  • **生成列**: 基底型のNULLを生成（ドメイン制約は適用しない）。
  • **通常列**: coerce_null_to_domain を使用してドメイン制約を考慮したNULLを生成。

    • 4. 最後に、残存するジャンク列（resjunk == true）をリストの末尾に追加し、resno を属性数以降の値に再割り当てします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/preptlist.c: preprocess_targetlist 内で呼び出されます。
  • **呼び出し先**:
  • RelationGetNumberOfAttributes, TupleDescAttr, makeConst, getBaseTypeAndTypmod, coerce_null_to_domain, eval_const_expressions, makeTargetEntry, flatCopyTargetEntry, lappend 等のPostgreSQL内部ユーティリティ関数。

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • TargetEntry: ターゲットリストの各要素を保持する構造体。
  • Relation: リレーションのメタデータ（rd_att を介して属性情報にアクセス）。

失敗モードと境界条件

  • resjunk ではないエントリが属性順序と一致しない場合、elog(ERROR, "targetlist is not sorted correctly") を発生させます。
  • 削除済み列や生成列に対するNULL生成処理において、型情報の不整合が発生しないよう注意深く処理されます。

保守担当者向けメモ

  • 本関数は INSERT 文の物理的な整合性を保証する重要な役割を担っています。
  • MERGE 文のサポートに伴い、ターゲットリストの構築が複雑化しているため、変更時は resjunk 列の扱いと resno の再割り当てロジックに注意してください。
  • src/backend/executor/nodeModifyTable.c の処理と整合性を保つ必要があります。
