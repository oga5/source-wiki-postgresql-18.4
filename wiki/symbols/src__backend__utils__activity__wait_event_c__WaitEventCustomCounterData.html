WaitEventCustomCounterData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/activity/wait_event.c:84

目的

カスタム待機イベントを動的に登録する際、各イベントに割り当てる一意な識別子（ID）を管理するためのカウンタ構造体です。

シグネチャまたは形状

typedef struct WaitEventCustomCounterData
{
    int         nextId;         /* 次に割り当てるID */
    slock_t     mutex;          /* カウンタを保護するスピンロック */
} WaitEventCustomCounterData;

入力と出力

  • **入力**: 構造体内の nextId は、新しいカスタムイベントが要求されるたびにインクリメントされます。
  • **出力**: 割り当てられた nextId の値が、新しいカスタム待機イベントの識別子として使用されます。

前提条件と不変条件

  • 共有メモリ上に配置される必要があります。
  • mutex フィールドは、nextId へのアクセスを排他制御するために使用されます。

アルゴリズムまたは振る舞い

1. カスタム待機イベントの登録要求が発生した際、mutex を用いて排他ロックを取得します。 2. 現在の nextId を取得し、次回の割り当てのために値をインクリメントします。 3. ロックを解放し、取得したIDを呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: WaitEventCustomNew がこの構造体を使用してIDを生成します。
  • **呼び出し先**: 共有メモリ初期化時に ShmemInitStruct を通じて初期化されます。

関連データ構造

  • WaitEventCustomEntryByInfo / WaitEventCustomEntryByName: カスタムイベントの情報を保持するハッシュテーブルエントリ。

失敗モードと境界条件

  • **ID枯渇**: カスタムイベントの登録数には上限（WAIT_EVENT_CUSTOM_HASH_MAX_SIZE = 128）があり、これを超えた場合の挙動は本構造体単体ではなく、登録関数側の制御に依存します。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、プロセス間での同期が必須です。mutex の使用を省略したり、ロック範囲を誤ったりしないよう注意してください。
