TrgmState

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_regexp.c:334

目的

正規表現をトリグラムベースのグラフへ変換する過程において、NFA（非決定性有限オートマトン）の各状態を管理するための構造体です。

シグネチャまたは形状

typedef struct TrgmState
{
    TrgmStateKey stateKey;      /* ハッシュテーブルキー: 最初のフィールドである必要がある */
    List        *arcs;          /* この状態からの遷移（TrgmArc）のリスト */
    List        *enterKeys;     /* この状態へ入るためのキーのリスト */
    int          flags;         /* 状態の属性フラグ */
    int          snumber;       /* 状態番号 */
    struct TrgmState *parent;   /* 親状態へのポインタ */
    int          tentFlags;     /* 一時的なフラグ */
    struct TrgmState *tentParent; /* 一時的な親状態へのポインタ */
} TrgmState;

入力と出力

  • 入力: TrgmNFA 構築プロセスにおいて、getState 関数を介してキー（TrgmStateKey）に基づき生成または取得されます。
  • 出力: グラフ構築の各段階で、processState や addArcs などの関数によって状態遷移情報（arcs）や接続情報が更新されます。

前提条件と不変条件

  • stateKey は構造体の先頭に配置される必要があります。
  • TrgmNFA のコンテキスト内で管理され、ハッシュテーブルによる状態の重複排除が行われることを前提としています。

アルゴリズムまたは振る舞い

  • 正規表現の NFA を走査する際、各状態を TrgmState として表現し、トリグラムをラベルとする遷移（TrgmArc）を構築します。
  • parent および tentParent フィールドを用いて、グラフの階層構造や探索経路を保持します。
  • processState 関数により、特定の状態から到達可能な遷移が計算され、グラフが展開されます。

呼び出し元と呼び出し先

  • 呼び出し元: processState, addKey, addArcs, addArc, getState（いずれも contrib/pg_trgm/trgm_regexp.c 内）。
  • 呼び出し先: 構造体内部で TrgmState 型のポインタを保持しており、グラフのノード間を相互参照します。

関連データ構造

  • TrgmStateKey: ハッシュテーブルのキーとして使用。
  • TrgmArc: 状態間の遷移を定義。
  • TrgmNFA: 本構造体を管理する上位の作業用構造体。

失敗モードと境界条件

  • メモリ不足時や、正規表現が複雑すぎて MAX_EXPANDED_STATES を超える場合、グラフ構築が中断される可能性があります。

保守担当者向けメモ

  • TrgmState は TrgmNFA の構築過程でのみ使用される一時的な構造体です。最終的なインデックス検索には TrgmPackedGraph が使用されるため、本構造体の変更が最終的なマッチングロジックに与える影響を考慮してください。
