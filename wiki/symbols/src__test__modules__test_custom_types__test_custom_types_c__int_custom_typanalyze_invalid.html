int_custom_typanalyze_invalid

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_types/test_custom_types.c:95

目的

PostgreSQLのANALYZE処理において、統計情報収集の挙動を意図的に不正な状態へ誘導するためのテスト用コールバック関数です。

シグネチャまたは形状

Datum int_custom_typanalyze_invalid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して VacAttrStats 構造体へのポインタを受け取ります。
  • **出力**: PG_RETURN_BOOL(true) により、処理が完了したことを示す真値を返します。

前提条件と不変条件

  • 引数として渡される stats ポインタは有効である必要があります。
  • stats->attstattarget が負の値である場合、デフォルトの統計ターゲット値で上書きされます。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して VacAttrStats 構造体を取得します。 2. attstattarget が負の場合、default_statistics_target を代入します。 3. stats->minrows を固定値 300 に設定します。 4. stats->compute_stats に静的関数 int_custom_invalid_stats を割り当てます。これにより、後の統計収集フェーズで意図的に不正な統計情報が生成されるように制御します。 5. true を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのANALYZE処理プロセス（直接的な呼び出し箇所はソースコード上では明示されていませんが、PG_FUNCTION_INFO_V1 マクロにより関数として登録されています）。
  • **呼び出し先**: int_custom_invalid_stats（stats->compute_stats への代入を通じて間接的に参照）。

関連データ構造

  • VacAttrStats: commands/vacuum.h で定義される、統計情報収集のための構造体。

失敗モードと境界条件

  • 本関数自体はエラーを発生させませんが、設定された int_custom_invalid_stats が実行されることで、統計情報の妥当性フラグ（stats_valid）が false に設定されるという「意図的な失敗」を引き起こします。

保守担当者向けメモ

  • 本関数はテスト専用です。実運用環境のコードには使用しないでください。
  • int_custom_invalid_stats と密結合しているため、統計収集ロジックのテストを変更する際は両方の整合性を確認してください。
