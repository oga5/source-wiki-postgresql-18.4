g_int_picksplit

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_gist.c:441

目的

GiST インデックスのページ分割処理において、Guttman のアルゴリズムに基づき、インデックスエントリを2つのグループに効率的に分割する。

シグネチャまたは形状

Datum g_int_picksplit(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GistEntryVector（分割対象のエントリ群）および GIST_SPLITVEC（分割結果を格納する構造体）を受け取る。
  • **出力**: v->spl_left, v->spl_right に分割されたエントリのインデックスを格納し、v->spl_ldatum, v->spl_rdatum に各グループの統合された境界値を設定する。

前提条件と不変条件

  • entryvec には有効なインデックスエントリが含まれている必要がある。
  • 分割後の各グループは、元のインデックスのバランスを維持するために、無駄な領域（waste）を最小化するように選択される。

アルゴリズムまたは振る舞い

1. **シードの選択**: 全エントリのペアを走査し、union と intersection の差分（size_waste）が最大となるペアを2つのグループの初期シードとして選択する。 2. **コスト計算**: 各エントリについて、どちらのシードグループに追加した際の拡大コストが小さいかを計算し、costvector に格納してソートする。 3. **グループ分け**: ソートされた順序に従い、各エントリを「拡大コストが最小となるグループ」へ順次割り当てる。 4. **結果の格納**: 最終的なグループ分けを GIST_SPLITVEC に書き込み、各グループの統合値を spl_ldatum / spl_rdatum として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構（直接の呼び出し元はソースコード上では明示されていないが、PG_FUNCTION_INFO_V1 マクロにより GiST インターフェースとして登録されている）。
  • **呼び出し先**: inner_int_union, inner_int_inter, rt__int_size, copy_intArrayType, qsort, comparecost 等のヘルパー関数。

関連データ構造

  • GistEntryVector: 分割対象のエントリを保持。
  • GIST_SPLITVEC: 分割結果（左/右のインデックス配列と境界値）を保持。
  • SPLITCOST: 各エントリの分割コストを計算・保持するための構造体。

失敗モードと境界条件

  • エントリ数が少ない場合や、シードが適切に選択できない場合はデフォルト値（1, 2）を使用する。
  • メモリ不足時には palloc によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は Guttman の多項式時間分割アルゴリズムを実装している。
  • WISH_F マクロを使用して、グループ間のバランスを調整するための微小なペナルティを付与している。
  • 非常にスパースなデータセットに対しては、本関数による分割よりも gist__intbig_ops の利用が推奨される。
