verify_btree_slot_handler

定義

  • 種別: function
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:1119

目的

B-tree 索引の検証用に実行された SQL クエリの結果（PGresult）を処理し、破損の検出または実行エラーの報告を行う。

シグネチャまたは形状

static bool verify_btree_slot_handler(PGresult *res, PGconn *conn, void *context)

入力と出力

  • **入力**:
  • res: 実行された SQL クエリの結果。
  • conn: データベース接続。
  • context: RelationInfo 構造体へのポインタ。
  • **出力**:
  • bool: should_processing_continue(res) の戻り値。処理を継続すべきか否かを示す。

前提条件と不変条件

  • context は RelationInfo 型としてキャスト可能であること。
  • 呼び出し元は、ParallelSlotSetHandler を介して本関数をコールバックとして登録していること。

アルゴリズムまたは振る舞い

1. PQresultStatus(res) が PGRES_TUPLES_OK の場合:

  • 取得された行数（ntups）を確認する。1行を超える場合は、amcheck と pg_amcheck のバージョン不整合の可能性があるため警告を出力する。

    • 2. それ以外（エラー発生時）の場合:

  • PQerrorMessage(conn) を取得し、indent_lines で整形して標準出力に表示する。
  • グローバル変数 all_checks_pass を false に設定する。

    • 3. 終了処理:

  • RelationInfo 内の sql, nspname, relname を解放（FREE_AND_SET_NULL）する。
  • should_processing_continue を呼び出し、後続の処理可否を判定して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_amcheck/pg_amcheck.c 内の ParallelSlotSetHandler（並列実行のイベントループ内）。
  • **呼び出し先**: indent_lines, should_processing_continue, pg_log_warning, pg_log_warning_detail, pg_log_warning_hint。

関連データ構造

  • RelationInfo: チェック対象のリレーション情報（OID、名前、SQL クエリ等）を保持。
  • PGresult, PGconn: libpq のデータ構造。

失敗モードと境界条件

  • **バージョン不整合**: amcheck 関数が予期しない行数を返した場合、警告メッセージを表示する。
  • **クエリ失敗**: データベース側でエラーが発生した場合、エラーメッセージを整形して出力し、検証失敗としてマークする。
  • **メモリ管理**: RelationInfo 内の文字列フィールドは処理後に解放されるため、再利用は不可。

保守担当者向けメモ

  • 本関数は並列実行のイベントループ内で動作するため、標準出力への書き込み順序や stderr への出力タイミング（progress_since_last_stderr の制御）に注意すること。
  • amcheck 拡張機能が投げるエラーを捕捉する設計であるため、サーバー側の仕様変更が本関数のエラーハンドリングに影響を与える可能性がある。
