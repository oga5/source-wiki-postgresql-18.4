TimeZoneAbbrevTable

定義

  • 種別: struct
  • 定義箇所: src/include/utils/datetime.h:215

目的

タイムゾーンの省略形（abbreviation）情報を保持し、動的にロード可能なテーブルとして管理するための構造体です。

シグネチャまたは形状

typedef struct TimeZoneAbbrevTable
{
    Size        tblsize;        /* テーブル全体のバイトサイズ */
    int         numabbrevs;     /* abbrevs[] 配列の要素数 */
    datetkn     abbrevs[FLEXIBLE_ARRAY_MEMBER]; /* 可変長配列 */
    /* 必要に応じて DynamicZoneAbbrev が後続する */
} TimeZoneAbbrevTable;

入力と出力

  • 入力: タイムゾーン定義ファイルから読み込まれたデータ。
  • 出力: InstallTimeZoneAbbrevs を介してシステムのアクティブなタイムゾーンテーブルとして設定される。

前提条件と不変条件

  • abbrevs 配列は datetkn 型の要素を保持する。
  • 構造体の末尾に DynamicZoneAbbrev が配置される可能性があるため、メモリ確保時には tblsize を適切に計算する必要がある。

アルゴリズムまたは振る舞い

  • src/backend/commands/variable.c にて、ファイルから読み込まれたデータを guc_malloc で確保し、本構造体として構築する。
  • src/backend/utils/adt/datetime.c 内の FetchDynamicTimeZone 関数において、特定のトークンに対応するタイムゾーン情報を検索するために参照される。
  • InstallTimeZoneAbbrevs 関数により、システム全体で利用可能なアクティブなテーブルとしてインストールされる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/variable.c（テーブルのロードおよびインストール）、src/backend/utils/adt/datetime.c（テーブルの保持および検索）。
  • 呼び出し先: datetkn 型（要素として使用）、DynamicZoneAbbrev（後続データとして使用）。

関連データ構造

  • datetkn: タイムゾーンの省略形トークンを保持する構造体。
  • DynamicZoneAbbrev: TimeZoneAbbrevTable の直後に配置され得る動的なタイムゾーン定義。
  • pg_tz: FetchDynamicTimeZone で返されるタイムゾーン情報。

失敗モードと境界条件

  • メモリ確保失敗: guc_malloc を使用しているため、システムメモリ不足時に失敗する可能性がある。
  • テーブルサイズ計算: offsetof(TimeZoneAbbrevTable, abbrevs) を使用したサイズ計算が必須であり、誤ったサイズ指定はメモリ破壊を招く。

保守担当者向けメモ

  • 本構造体は guc_malloc で確保される前提で設計されているため、ライフサイクル管理には注意すること。
  • abbrevs は可変長配列であるため、構造体自体をスタック上に確保してはならない。
  • タイムゾーン定義の更新時には、InstallTimeZoneAbbrevs を通じて適切にアクティブなテーブルを差し替える必要がある。
