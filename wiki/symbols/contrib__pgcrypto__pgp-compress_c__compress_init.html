compress_init

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-compress.c:69

目的

PGP圧縮フィルタ（PushFilter）の初期化を行い、zlib ライブラリを用いた圧縮ストリームの状態を構築します。

シグネチャまたは形状

static int compress_init(PushFilter *next, void *init_arg, void **priv_p);

入力と出力

  • **入力**:
  • next: 後続のフィルタへのポインタ。
  • init_arg: PGP_Context 構造体へのポインタ（圧縮アルゴリズムとレベルを取得）。
  • priv_p: 初期化された ZipStat 構造体を格納するための出力用ポインタ。
  • **出力**:
  • 成功時: ZIP_IN_BLOCK を返却。
  • 失敗時: PXE_PGP_UNSUPPORTED_COMPR または PXE_PGP_COMPRESSION_ERROR を返却。

前提条件と不変条件

  • init_arg が指す PGP_Context の compress_algo は、PGP_COMPR_ZLIB または PGP_COMPR_ZIP である必要があります。
  • zlib がインストールされている環境である必要があります（未インストール時はスタブ動作）。

アルゴリズムまたは振る舞い

1. 圧縮アルゴリズムの妥当性を検証します。 2. ZipStat 構造体をメモリ確保し、zlib 用のメモリ割り当て関数（z_alloc, z_free）を割り当てます。 3. アルゴリズムに応じて deflateInit2（ZIP用）または deflateInit（ZLIB用）を呼び出し、z_stream を初期化します。 4. 初期化に成功した場合、priv_p に ZipStat を格納し、ZIP_IN_BLOCK を返します。 5. 失敗した場合はメモリを解放し、エラーコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、pgp_compress_filter 等のフィルタ生成関数から利用されることが想定されます。
  • **呼び出し先**: palloc0, pfree, deflateInit, deflateInit2（いずれも zlib および PostgreSQL 内部関数）。

関連データ構造

  • ZipStat: 圧縮処理の状態を保持する内部構造体。
  • PGP_Context: 圧縮アルゴリズムやレベルを保持するコンテキスト。
  • PushFilter: フィルタパイプラインの構成要素。

失敗モードと境界条件

  • PGP_COMPR_ZLIB / PGP_COMPR_ZIP 以外のアルゴリズムが指定された場合、PXE_PGP_UNSUPPORTED_COMPR を返します。
  • deflateInit / deflateInit2 が Z_OK 以外を返した場合、PXE_PGP_COMPRESSION_ERROR を返します。

保守担当者向けメモ

  • z_alloc および z_free を使用することで、zlib が PostgreSQL のメモリ管理機構（palloc/pfree）を利用するように強制しており、メモリリークを防止しています。
  • ZIP形式（PGP_COMPR_ZIP）の初期化には deflateInit2 を使用し、ウィンドウサイズ等のパラメータを調整しています。
