get_actual_clauses

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:459

目的

RestrictInfo ノードのリストから、プラン作成時に必要となる生の式（Expr）のリストを抽出します。

シグネチャまたは形状

List *get_actual_clauses(List *restrictinfo_list)

入力と出力

  • **入力**: restrictinfo_list (List *) - RestrictInfo ノードのリスト。
  • **出力**: List * - 抽出された clause（Expr）のリスト。

前提条件と不変条件

  • 入力リスト内の各 RestrictInfo ノードにおいて、pseudoconstant が false であること。
  • 入力リスト内の各 RestrictInfo ノードにおいて、rinfo_is_constant_true が false であること。
  • 上記の条件は Assert によって検証されます。

アルゴリズムまたは振る舞い

1. 空のリスト result を初期化します。 2. 入力された restrictinfo_list を走査します。 3. 各要素（RestrictInfo）から clause フィールドを取り出し、result リストに追加します。 4. 最終的な result リストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c (複数箇所: get_loc_restrictinfo の結果、indexquals、path_mergeclauses、path_hashclauses の処理)
  • **呼び出し先**:
  • lfirst_node (マクロ)
  • rinfo_is_constant_true (インライン関数)
  • lappend (リスト操作)

関連データ構造

  • RestrictInfo: プランナが使用する制約条件のラッパー構造体。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • 入力リストが NIL の場合、空のリストを返します。
  • Assert 条件（pseudoconstant や定数 true）に違反するノードが含まれている場合、デバッグビルドではアボートします。

保守担当者向けメモ

  • 本関数は、プランナが最適化の最終段階で RestrictInfo のメタデータが不要な「生の式」を必要とする際に使用されます。
  • pseudoconstant や定数 true の条件を排除する設計であるため、これらを含むリストを渡す必要がある場合は別の抽出関数を検討してください。
