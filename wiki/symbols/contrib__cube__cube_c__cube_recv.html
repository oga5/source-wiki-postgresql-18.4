cube_recv

定義

  • 種別: function
  • 定義箇所: contrib/cube/cube.c:355

目的

cube 型のバイナリ形式の入力ハンドラとして機能し、ネットワークやストレージから受信したバイナリデータから NDBOX 構造体を復元します。

シグネチャまたは形状

Datum cube_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して StringInfo バッファを受け取ります。
  • **出力**: NDBOX 構造体へのポインタを Datum 型として返します。

前提条件と不変条件

  • 入力バッファは libpq のバイナリプロトコル形式に従っている必要があります。
  • header に含まれる次元数（nitems）は CUBE_MAX_DIM を超えてはなりません。

アルゴリズムまたは振る舞い

1. pq_getmsgint を使用してバッファからヘッダー情報を読み取ります。 2. ヘッダーから次元数（nitems）を抽出し、上限チェックを行います。 3. POINT_BIT が設定されていない場合、座標数は次元数の2倍（下限と上限のペア）として計算されます。 4. palloc を用いて必要なメモリを確保し、SET_VARSIZE で PostgreSQL の可変長データ構造として初期化します。 5. ループ処理により、pq_getmsgfloat8 を使用してバッファから各座標値を cube->x 配列に読み込みます。 6. 構築した NDBOX を PG_RETURN_NDBOX_P マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の型システム（バイナリ入力処理時）。直接的な呼び出し元コードは提供されていません。
  • **呼び出し先**: pq_getmsgint, pq_getmsgfloat8, palloc, SET_VARSIZE, PG_RETURN_NDBOX_P などの PostgreSQL 内部 API。

関連データ構造

  • NDBOX: contrib/cube/cubedata.h で定義される多次元ボックス構造体。
  • StringInfo: libpq/pqformat.h で定義されるバイナリデータ保持用構造体。

失敗モードと境界条件

  • **次元数超過**: nitems > CUBE_MAX_DIM の場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーを報告します。
  • **メモリ不足**: palloc が失敗した場合は PostgreSQL の標準的なメモリ管理機構によりエラーがスローされます。

保守担当者向けメモ

  • 本関数は cube_send と対になるバイナリ入出力処理です。
  • POINT_BIT の有無によって座標配列のサイズ計算が異なるため、データ構造の変更時は cubedata.h の定義と整合性を確認してください。
