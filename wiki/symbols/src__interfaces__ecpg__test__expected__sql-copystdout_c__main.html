目的

ECPG（Embedded SQL for C）における COPY TO STDOUT コマンドの動作を検証するための回帰テスト用エントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しません）。
  • 出力:
  • 標準出力: COPY TO STDOUT コマンドによるテーブルデータおよび sqlca.sqlcode の値。
  • 戻り値: 成功時に 0 を返します。

前提条件と不変条件

  • ECPGランタイムライブラリが利用可能であること。
  • ecpg1_regression データベースへの接続が成功すること。
  • 各SQL操作後に sqlca.sqlcode を確認し、負の値（エラー）の場合は sqlprint で診断情報を出力する不変のパターンが適用されています。

アルゴリズムまたは振る舞い

1. ECPGdebug を呼び出し、デバッグ出力を有効化します。 2. ECPGconnect でデータベースに接続します。 3. ECPGdo を使用して以下のSQL操作を順次実行します。

  • create table foo ( a int , b varchar )
  • insert into foo ... （3回実行）
  • copy foo to stdout with delimiter ','

    • 4. printf を使用して、実行後の sqlca.sqlcode を標準出力へ書き出します。 5. ECPGdisconnect でデータベース接続を終了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムの実行時）。
  • 呼び出し先:
  • ECPGdebug, ECPGconnect, ECPGdo, ECPGdisconnect（ECPGランタイムライブラリ）
  • sqlprint（エラー診断出力）
  • printf（標準出力）

関連データ構造

  • sqlca: SQL通信領域（SQL Communication Area）。実行結果のステータスコードを保持します。

失敗モードと境界条件

  • 各SQL実行時に sqlca.sqlcode が負の値となった場合、sqlprint が呼び出され、エラー情報が標準エラー出力に書き出されます。
  • COPY コマンド実行時にデータが標準出力へ流し込まれるため、テスト環境ではその出力内容が検証対象となります。
