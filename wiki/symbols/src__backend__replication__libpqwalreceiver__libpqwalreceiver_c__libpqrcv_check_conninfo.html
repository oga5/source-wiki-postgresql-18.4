libpqrcv_check_conninfo

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:292

目的

提供された接続文字列（conninfo）の構文を検証し、必要に応じてパスワードが含まれているかをチェックする。

シグネチャまたは形状

static void libpqrcv_check_conninfo(const char *conninfo, bool must_use_password)

入力と出力

  • **入力**:
  • conninfo (const char *): 検証対象の接続文字列。
  • must_use_password (bool): パスワードの指定を必須とするかどうかのフラグ。
  • **出力**: なし。検証に失敗した場合は ereport(ERROR, ...) を通じて例外を発生させる。

前提条件と不変条件

  • conninfo は PQconninfoParse で解析可能な形式であること。
  • must_use_password が true の場合、接続文字列内に空ではない password オプションが含まれている必要がある。

アルゴリズムまたは振る舞い

1. PQconninfoParse を使用して接続文字列を解析する。解析に失敗した場合はエラーを報告する。 2. must_use_password が true の場合、解析されたオプションリストを走査し、password キーワードが存在し、かつその値が空文字列ではないことを確認する。 3. 条件を満たさない場合、ERRCODE_S_R_E_PROHIBITED_SQL_STATEMENT_ATTEMPTED エラーを発生させる。 4. 最後に PQconninfoFree を呼び出し、解析に使用したメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c 内の libpqrcv_connect 関数から呼び出される。
  • PQWalReceiverFunctions 構造体の .walrcv_check_conninfo フィールドとして登録されている。
  • **呼び出し先**:
  • libpq ライブラリの PQconninfoParse、PQfreemem、PQconninfoFree。

関連データ構造

  • PQconninfoOption: libpq が提供する接続オプションの構造体。

失敗モードと境界条件

  • **構文エラー**: 接続文字列が不正な場合、ERRCODE_SYNTAX_ERROR で終了する。
  • **パスワード欠落**: must_use_password が true でパスワードが指定されていない場合、ERRCODE_S_R_E_PROHIBITED_SQL_STATEMENT_ATTEMPTED で終了する。
  • **メモリ不足**: PQconninfoParse が失敗した場合、適切なエラーメッセージを生成して終了する。

保守担当者向けメモ

  • 本関数は libpq のメモリ管理ルールに従い、PQconninfoParse で割り当てられたメモリを確実に解放する必要がある。エラー発生時も PQconninfoFree を呼び出すパスを通すこと。
