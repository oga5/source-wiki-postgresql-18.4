GetWaitEventCustomIdentifier

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/wait_event.c:275

目的

指定された待機イベントID（wait_event_info）に対応するカスタム待機イベント名を、共有メモリ上のハッシュテーブルから取得して返します。

シグネチャまたは形状

static const char *GetWaitEventCustomIdentifier(uint32 wait_event_info)

入力と出力

  • **入力**: wait_event_info (uint32) - 待機イベントを識別する整数値。
  • **出力**: const char * - 待機イベント名。組み込みの PG_WAIT_EXTENSION の場合は "Extension" を返します。

前提条件と不変条件

  • 呼び出し前に WaitEventCustomLock が初期化されている必要があります。
  • 検索対象の待機イベントがカスタムイベントである場合、共有メモリ上の WaitEventCustomHashByInfo ハッシュテーブルに登録済みである必要があります。

アルゴリズムまたは振る舞い

1. wait_event_info が PG_WAIT_EXTENSION と一致するか確認し、一致すれば "Extension" を即座に返します。 2. それ以外の場合、WaitEventCustomLock を共有モード（LW_SHARED）で取得します。 3. WaitEventCustomHashByInfo ハッシュテーブルを HASH_FIND モードで検索します。 4. LWLockRelease でロックを解放します。 5. エントリが見つかった場合はその wait_event_name を返します。見つからない場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/wait_event.c 内の関数（行454など）から参照されています。
  • **呼び出し先**: LWLockAcquire, hash_search, LWLockRelease, elog を呼び出します。

関連データ構造

  • WaitEventCustomEntryByInfo: ハッシュテーブルの検索結果として使用される構造体。
  • WaitEventCustomHashByInfo: カスタム待機イベントを保持する共有メモリ上のハッシュテーブル。
  • WaitEventCustomLock: 検索時の排他制御に使用される LWLock。

失敗モードと境界条件

  • 指定された wait_event_info がハッシュテーブル内に存在しない場合、elog(ERROR) によりエラー終了します。

保守担当者向けメモ

  • 本関数は共有メモリ上のハッシュテーブルを読み取るため、必ず WaitEventCustomLock による保護下で実行されます。
  • 組み込みイベントとカスタムイベントの境界を意識した実装となっており、PG_WAIT_EXTENSION はハードコードされた例外として扱われています。
