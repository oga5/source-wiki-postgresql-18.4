sha1_result

定義

  • 種別: function
  • 定義箇所: src/common/sha1.c:275

目的

SHA-1 ハッシュ計算の最終結果（メッセージダイジェスト）を、コンテキストから指定されたバッファへ書き出します。

シグネチャまたは形状

static void sha1_result(uint8 *digest0, pg_sha1_ctx *ctx)

入力と出力

  • digest0 (入力/出力): 20バイトのダイジェスト値を格納するためのバッファへのポインタ。
  • ctx (入力): SHA-1 の内部状態を保持する pg_sha1_ctx 構造体。

前提条件と不変条件

  • ctx は pg_sha1_init および pg_sha1_update によって適切に初期化・更新されている必要があります。
  • digest0 は少なくとも20バイトのメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

  • プラットフォームのエンディアン設定（WORDS_BIGENDIAN）に基づき、内部状態 ctx->h.b8 からダイジェストバッファへバイト順序を調整してコピーします。
  • ビッグエンディアン環境では memmove を使用し、リトルエンディアン環境ではバイト順序を入れ替えて（各4バイト単位で反転させて）書き出します。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/sha1.c 内の pg_sha1_final から呼び出されます。
  • 呼び出し先: 直接的な関数呼び出しはありませんが、memmove を利用します。

関連データ構造

  • pg_sha1_ctx: SHA-1 の計算状態を保持する構造体（sha1_int.h で定義）。

失敗モードと境界条件

  • digest0 が NULL または十分な領域を確保していない場合、メモリ破壊が発生します。本関数自体には境界チェックは含まれていません。

保守担当者向けメモ

  • 本実装は RFC 3174 に準拠しています。エンディアン変換ロジックは移植性に直結するため、プラットフォーム固有の動作を変更する際は注意してください。
