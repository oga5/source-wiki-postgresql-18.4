AtProcExit_Twophase

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/twophase.c:293

目的

プロセス終了時に、2フェーズコミットに関連するトランザクション状態のクリーンアップ（アボート処理）を実行する。

シグネチャまたは形状

static void AtProcExit_Twophase(int code, Datum arg);

入力と出力

  • code: 終了コード（before_shmem_exit の規約に従う）。
  • arg: 補助引数（未使用）。
  • 戻り値: なし。

前提条件と不変条件

  • 本関数は before_shmem_exit のコールバックとして登録されることを前提としている。
  • 内部で呼び出される AtAbort_Twophase() の不変条件に依存する。

アルゴリズムまたは振る舞い

1. AtAbort_Twophase() を呼び出し、2フェーズコミットに関連するアボート処理を実行する。 2. 処理完了後、呼び出し元へ制御を戻す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c:381 (before_shmem_exit への登録)
  • src/backend/access/transam/twophase.c:559 (before_shmem_exit への登録)
  • **呼び出し先**:
  • AtAbort_Twophase()

関連データ構造

  • 特になし（AtAbort_Twophase が使用する内部状態に依存）。

失敗モードと境界条件

  • AtAbort_Twophase() の失敗モードに準拠する。

保守担当者向けメモ

  • 本関数は AtAbort_Twophase のラッパーとして機能している。ロジックの変更が必要な場合は、主に AtAbort_Twophase 側を確認すること。
