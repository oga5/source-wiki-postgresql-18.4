HashBuildState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/hash/hash.c:38

目的

ハッシュインデックス構築プロセス中の状態を保持するための構造体です。インデックス作成時のタプル数カウントや、メモリ不足時に使用されるソート用スプールの管理を行います。

シグネチャまたは形状

typedef struct {
    HSpool     *spool;        /* NULL if not using spooling */
    double      indtuples;    /* # tuples accepted into index */
    Relation    heapRel;      /* heap relation descriptor */
} HashBuildState;

入力と出力

本構造体は関数間で状態を共有するために使用されます。

  • 入力: hashbuild 関数内で初期化され、hashbuildCallback 等のコールバック関数へ渡されます。
  • 出力: 構築完了後の indtuples が統計情報として利用されます。

前提条件と不変条件

  • spool が NULL の場合、ソート処理は行われません。
  • indtuples はインデックスに正常に挿入されたタプルの総数を保持します。

アルゴリズムまたは振る舞い

1. hashbuild 関数内でインスタンスが生成されます。 2. インデックス構築中、テーブルから読み込まれたタプルが hashbuildCallback を経由して処理される際、indtuples がインクリメントされます。 3. メモリ制限を超過する場合、spool を使用して一時的なソート処理が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/hash/hash.c 内の hashbuild 関数で宣言および初期化されます。
  • 呼び出し先: hashbuildCallback 関数へ引数として渡され、状態更新に使用されます。

関連データ構造

  • HSpool: ソートが必要な場合に利用されるスプール構造体。
  • Relation: PostgreSQL のリレーション記述子。

失敗モードと境界条件

  • メモリ不足時には spool が有効化されますが、その際のディスク容量やソート制限については hashsort.c 等の依存先に依存します。

保守担当者向けメモ

  • 本構造体は hashbuild の実行コンテキストに特化しています。
  • indtuples は VACUUM や統計情報更新の基礎となるため、構築プロセスでの正確なカウントが重要です。
