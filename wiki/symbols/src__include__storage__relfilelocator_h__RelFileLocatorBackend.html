RelFileLocatorBackend

定義

  • 種別: struct
  • 定義箇所: src/include/storage/relfilelocator.h:73

目的

リレーション（テーブルやインデックス等）の物理的な格納場所を特定する識別子に、バックエンドプロセス番号を付与した構造体です。主に一時リレーションなど、バックエンド固有のファイルを識別するために使用されます。

シグネチャまたは形状

typedef struct RelFileLocatorBackend
{
	RelFileLocator locator;
	ProcNumber	backend;
} RelFileLocatorBackend;

入力と出力

  • **入力**: RelFileLocator（リレーション識別子）および ProcNumber（プロセス番号）。
  • **出力**: なし（データ構造定義）。

前提条件と不変条件

  • backend フィールドが INVALID_PROC_NUMBER の場合、共有リレーションとして扱われます。
  • それ以外の値の場合、バックエンドローカルな一時リレーションとして扱われます。

アルゴリズムまたは振る舞い

本構造体はデータ保持のみを目的としており、直接的な実行ロジックは持ちません。ストレージマネージャ（smgr）やバッファマネージャが、物理ファイルへのアクセスパスを決定する際のキーとして利用します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/catalog/catalog.c
  • src/backend/catalog/storage.c
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/smgr/md.c
  • src/backend/storage/smgr/smgr.c
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、関数呼び出しの概念は適用されません。

関連データ構造

  • RelFileLocator: 物理アクセスに必要な最小限の識別子（spcOid, dbOid, relNumber）。
  • ProcNumber: バックエンドプロセスを識別するための型。

失敗モードと境界条件

  • RelFileLocator 内の各 OID が正しく解決されていない場合、ファイルシステム上のパス解決に失敗します。
  • backend フィールドの不適切な設定は、共有リレーションと一時リレーションの誤認を招く可能性があります。

保守担当者向けメモ

  • 本構造体はハッシュテーブルのキーとして利用される可能性があるため、メンバの追加や変更を行う際はパディングが発生しないよう注意してください。
  • RelFileLocator と同様、正規化された OID を使用することが前提です。
