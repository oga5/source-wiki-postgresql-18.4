LOCALLOCKTAG

定義

  • 種別: struct
  • 定義箇所: src/include/storage/lock.h:409

目的

バックエンドプロセスがローカルで保持するロック状態（LOCALLOCK）を識別するためのタグです。共有メモリ上のロックテーブルへのアクセス頻度を低減するためのキャッシュキーとして機能します。

シグネチャまたは形状

typedef struct LOCALLOCKTAG
{
    LOCKTAG     lock;           /* ロック対象の識別子 */
    LOCKMODE    mode;           /* このテーブルエントリのロックモード */
} LOCALLOCKTAG;

入力と出力

  • **入力**: LOCKTAG（ロック対象オブジェクト）および LOCKMODE（要求するロックモード）の組み合わせ。
  • **出力**: ロックマネージャのローカルハッシュテーブルにおける一意なキーとして使用されます。

前提条件と不変条件

  • LOCKTAG はロック対象を一意に識別する構造体であり、本構造体の一部として埋め込まれます。
  • 本構造体は、バックエンドごとのローカルロックキャッシュの検索キーとして使用されるため、値の整合性が維持されている必要があります。

アルゴリズムまたは振る舞い

  • LockAcquire などの関数において、共有メモリ上のロック状態を参照する前に、このタグを用いてローカルキャッシュを検索します。
  • キャッシュヒットした場合は共有メモリへのアクセスを省略し、キャッシュミスした場合は共有メモリ上のロック情報を取得してローカルキャッシュを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/lock.c 内のロック管理関数（LockAcquire 等）において、ローカルキャッシュの検索や挿入のキーとして使用されています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • LOCKTAG: ロック対象オブジェクトの識別子。
  • LOCALLOCK: 本タグをキーとして保持する、各バックエンドのローカルロック管理構造体。
  • LOCKMODE: ロックの強さや競合ルールを定義する列挙型。

失敗モードと境界条件

  • 本構造体はメモリ上のデータ構造であるため、直接的な失敗モードはありません。ただし、ハッシュテーブルのキーとして使用される際、LOCKTAG の内容が不適切に初期化されていると、誤ったキャッシュエントリを参照する可能性があります。

保守担当者向けメモ

  • LOCALLOCKTAG はローカルキャッシュのキーであるため、構造体サイズやフィールドの変更は、ハッシュテーブルの検索ロジック（src/backend/storage/lmgr/lock.c）に直接影響を与えます。変更時はハッシュ関数の整合性に注意してください。
