hstore_in

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_io.c:479

目的

文字列形式（key=>value）の入力を受け取り、PostgreSQL の内部表現である HStore 構造体に変換する SQL 入力関数です。

シグネチャまたは形状

Datum hstore_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して取得される char * 型の文字列。
  • **出力**: HStore * 型のポインタを Datum として返却。パース失敗時は NULL を返却。

前提条件と不変条件

  • fcinfo->context からエラーコンテキスト（escontext）を取得し、パース処理に渡す必要があります。
  • 入力文字列は hstore 型の有効な構文に従っている必要があります。

アルゴリズムまたは振る舞い

1. HSParser 構造体を初期化し、入力文字列とエラーコンテキストを設定します。 2. parse_hstore() を呼び出し、文字列をトークン化して Pairs 配列に格納します。失敗した場合は PG_RETURN_NULL() で終了します。 3. hstoreUniquePairs() を呼び出し、重複キーの排除とソートを行い、最終的なデータ長（buflen）を算出します。 4. hstorePairs() を呼び出し、メモリを確保して HStore 構造体を構築します。 5. 構築された HStore 構造体を PG_RETURN_POINTER() で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: parse_hstore, hstoreUniquePairs, hstorePairs。

関連データ構造

  • HSParser: パース中の状態を保持する構造体。
  • HStore: hstore 型の内部バイナリ表現。
  • Pairs: キーと値のペアを保持する配列。

失敗モードと境界条件

  • 入力文字列のパースに失敗した場合、parse_hstore が false を返し、関数は NULL を返却します。
  • キーや値が HSTORE_MAX_KEY_LEN 等の制限を超過する場合、parse_hstore 内でエラー処理が行われます。

保守担当者向けメモ

  • 本関数は hstore 型の入力インターフェースの入り口です。パースロジックの変更は parse_hstore を、メモリレイアウトの変更は hstorePairs を参照してください。
  • escontext を使用したソフトエラー処理が組み込まれているため、パースエラー時に即座に elog(ERROR) を発生させない設計となっています。
