PartitionPruneContext

定義

  • 種別: struct
  • 定義箇所: src/include/partitioning/partprune.h:49

目的

パーティションプルーニング（不要なパーティションの除外）を実行する際に必要なメタデータ、評価コンテキスト、および実行時状態を集約するために使用されます。

シグネチャまたは形状

typedef struct PartitionPruneContext
{
	char		strategy;
	int			partnatts;
	int			nparts;
	PartitionBoundInfo boundinfo;
	Oid		   *partcollation;
	FmgrInfo   *partsupfunc;
	FmgrInfo   *stepcmpfuncs;
	MemoryContext ppccontext;
	PlanState  *planstate;
	ExprContext *exprcontext;
	ExprState **exprstates;
} PartitionPruneContext;

入力と出力

  • **入力**: 初期化時にパーティションの戦略、境界情報（boundinfo）、比較関数（partsupfunc, stepcmpfuncs）、および式評価用のコンテキスト（exprcontext, exprstates）が設定されます。
  • **出力**: 実行時、get_matching_partitions 等の関数を通じて、クエリ条件に合致するパーティションの集合（Bitmapset）を導出するための情報源となります。

前提条件と不変条件

  • PruneCxtStateIdx マクロによる配列インデックス計算において、ステップとキー番号が連続して配置されている必要があります。
  • 実行時プルーニングを行う場合、planstate および exprcontext が適切に設定されている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: InitPartitionPruneContext により、プルーニングに必要なメタデータと実行状態が構造体に格納されます。 2. **評価**: get_matching_hash_bounds、get_matching_list_bounds、get_matching_range_bounds 等の内部関数が、本構造体の情報を参照して各プルーニングステップを評価します。 3. **抽出**: perform_pruning_base_step および perform_pruning_combine_step を経て、最終的な対象パーティションが決定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execPartition.c（InitPartitionPruneContext 等による初期化）、src/backend/partitioning/partprune.c（プルーニング処理の開始点）。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありませんが、partprune.c 内の各プルーニングロジック（get_matching_... 系関数）が本構造体を引数として受け取ります。

関連データ構造

  • PartitionBoundInfo: パーティションの境界定義。
  • PlanState, ExprState, ExprContext: PostgreSQL の実行ノードおよび式評価用コンテキスト。
  • FmgrInfo: パーティションキーの比較やハッシュ計算を行うための関数情報。

失敗モードと境界条件

  • exprstates や partsupfunc が適切に初期化されていない場合、式評価や比較処理で予期せぬ動作を引き起こす可能性があります。
  • パーティション戦略（strategy）と boundinfo の不整合は、プルーニング結果の誤判定を招く恐れがあります。

保守担当者向けメモ

  • 本構造体はプランニング時と実行時の両方で使用される可能性があるため、planstate の有無を確認して利用フェーズを判別してください。
  • PruneCxtStateIdx マクロの計算順序は実装上の重要な不変条件です。構造体メンバーの順序を変更する際は注意が必要です。
