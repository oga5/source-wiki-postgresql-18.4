init_uncommitted_enum_types

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_enum.c:254

目的

トランザクション内で作成された未コミットの列挙型を追跡するためのハッシュテーブル uncommitted_enum_types を初期化します。

シグネチャまたは形状

static void init_uncommitted_enum_types(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 uncommitted_enum_types を初期化）

前提条件と不変条件

  • TopTransactionContext が有効である必要があります。
  • 本関数は、ハッシュテーブルが未初期化の状態で呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を設定し、キーとエントリのサイズを Oid 型のサイズに指定します。 2. メモリコンテキストとして TopTransactionContext を割り当てます。 3. hash_create を呼び出し、"Uncommitted enum types" という名前で初期サイズ 32 のハッシュテーブルを作成します。 4. 作成したハッシュテーブルを uncommitted_enum_types に格納します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_enum.c 内の複数箇所から参照されています（直接的な呼び出し元はソースコード内の init_uncommitted_enum_types() の実行箇所に依存します）。
  • 呼び出し先: hash_create（PostgreSQL 内部のハッシュテーブル管理関数）。

関連データ構造

  • uncommitted_enum_types: 未コミットの列挙型 OID を保持するハッシュテーブル。
  • TopTransactionContext: ハッシュテーブルのメモリ割り当てに使用されるコンテキスト。

失敗モードと境界条件

  • メモリ不足時には hash_create がエラーを発生させます。
  • 既に初期化済みの状態で再呼び出しされた場合の挙動については、本ソース抜粋からは判断できません。

保守担当者向けメモ

  • 本関数はトランザクション単位での列挙型整合性維持のために不可欠です。
  • ハッシュテーブルのサイズ（32）は初期値であり、必要に応じて動的に拡張されます。
