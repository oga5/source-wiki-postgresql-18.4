miss

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:785

目的

DFA（決定性有限オートマトン）の状態遷移キャッシュにおいて、指定された文字色に対する遷移先が見つからない（キャッシュミス）場合に、NFA（非決定性有限オートマトン）の遷移を計算して新しい状態セットを生成・キャッシュする。

シグネチャまたは形状

static struct sset *miss(struct vars *v, struct dfa *d, struct sset *css, color co, chr *cp, chr *start)

入力と出力

  • **入力**:
  • v: 正規表現の実行コンテキスト。
  • d: 現在の DFA 構造体。
  • css: 現在の状態セット（キャッシュエントリ）。
  • co: 入力文字から変換された色（クラス）。
  • cp: 現在の入力文字位置。
  • start: マッチング開始位置。
  • **出力**:
  • 遷移先の struct sset *。キャッシュミス時に新しい状態セットが作成された場合はそのポインタを、到達不能な場合は NULL を返す。

前提条件と不変条件

  • css->outs[co] が NULL である（キャッシュミスしている）ことが前提。
  • INTERRUPT(v->re) により、処理中に操作キャンセルがチェックされる。
  • LACON（先読み/後読み制約）を含む遷移の場合、キャッシュへのリンクは作成されない。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: css->outs[co] が存在すればそれを返す。 2. **NFA 遷移計算**: 現在の状態セットから、指定された色 co で到達可能な NFA 状態をビットマップ d->work に展開する。 3. **LACON 評価**: HASLACONS フラグが有効な場合、lacon() を呼び出して制約を評価し、到達可能な状態を再帰的に追加する。 4. **キャッシュ検索**: 計算した状態セットが既存のキャッシュにあるかハッシュ値で確認する。 5. **キャッシュ登録**: 存在しない場合、getvacant() で新しいキャッシュエントリを確保し、状態ビットマップとフラグ（POSTSTATE, NOPROGRESS）を保存する。 6. **リンク作成**: LACON が関与していない場合に限り、css->outs[co] に新しい状態セットへのリンクを張る。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し元は本抜粋からは特定不可。DFA 実行ループ（longest や shortest）から遷移処理の一部として呼び出される。
  • **呼び出し先**: INTERRUPT, ISBSET, BSET, lacon, HASH, HIT, getvacant, ISERR, FDEBUG。

関連データ構造

  • struct dfa: DFA の実行状態を保持。
  • struct sset: DFA の状態セット（キャッシュエントリ）。
  • struct cnfa: コンパイル済みの NFA。

失敗モードと境界条件

  • gotstate が偽の場合（どの状態にも遷移できない場合）、NULL を返す。
  • getvacant() が失敗した場合、NULL を返す。
  • lacon() 評価中にエラーが発生した場合、NULL を返す。
  • LACON が関与する遷移はキャッシュリンクを作成しないため、毎回 miss() が再実行される。

保守担当者向けメモ

  • miss() はキャッシュミス時だけでなく、LACON の再評価が必要な場合にも呼び出される設計となっている。
  • sawlacons フラグが立っている場合、遷移のキャッシュリンクを作成しないことで、コンテキスト依存の制約を確実に再評価させる仕組みである。
