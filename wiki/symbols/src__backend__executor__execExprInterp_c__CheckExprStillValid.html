CheckExprStillValid

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExprInterp.c:2314

目的

式評価の実行コンテキスト（ExprContext）に含まれるタプルスロットが、現在の式ステップ（ExprState）で想定されている型定義と互換性を維持しているかを確認します。

シグネチャまたは形状

void CheckExprStillValid(ExprState *state, ExprContext *econtext)

入力と出力

  • **入力**:
  • state: 評価対象の式ステップリストを保持する ExprState 構造体。
  • econtext: 実行コンテキスト。内部の各タプルスロット（inner, outer, scan, old, new）を参照するために使用。
  • **出力**: なし（検証失敗時は CheckVarSlotCompatibility を通じてエラーが発生する可能性があります）。

前提条件と不変条件

  • econtext は有効なポインタである必要があります。
  • state->steps に格納された各ステップのオペコードが、変数参照（EEOP_*_VAR）を含む場合、対応する econtext 内のタプルスロットが初期化されている必要があります。

アルゴリズムまたは振る舞い

1. econtext から ecxt_innertuple、ecxt_outertuple、ecxt_scantuple、ecxt_oldtuple、ecxt_newtuple を取得します。 2. state->steps に含まれるすべてのステップを順次走査します。 3. 各ステップのオペコード（ExecEvalStepOp）を判定します。 4. 変数参照系のオペコード（EEOP_INNER_VAR, EEOP_OUTER_VAR, EEOP_SCAN_VAR, EEOP_OLD_VAR, EEOP_NEW_VAR）に該当する場合、CheckVarSlotCompatibility を呼び出し、対象スロットの属性型と式の期待する型が一致するかを検証します。 5. 変数参照以外のオペコードは無視されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c:2301
  • src/backend/jit/llvm/llvmjit_expr.c:2993
  • **呼び出し先**:
  • ExecEvalStepOp
  • CheckVarSlotCompatibility

関連データ構造

  • ExprState: 式の実行ステップリストを管理する構造体。
  • ExprContext: 式評価時の実行環境（タプルスロット等）を保持する構造体。
  • ExprEvalStep: 式の個々の演算ステップ。

失敗モードと境界条件

  • state->steps_len が 0 の場合、ループは実行されず何も行われません。
  • CheckVarSlotCompatibility の内部で型不一致が検出された場合、エラーが発生し処理が中断されます。

保守担当者向けメモ

  • 本関数は、式評価中にタプルの構造が変更された可能性をチェックするために使用されます。
  • 新しい変数参照オペコードを ExprEvalOp に追加した場合は、本関数の switch 文にも対応するケースを追加する必要があります。
