hstore_from_text

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_io.c:566

目的

単一のキーと値のペアから hstore 型の値を生成します。

シグネチャまたは形状

Datum hstore_from_text(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: キーとなる text 型データ。
  • 第2引数: 値となる text 型データ（NULL許容）。
  • **出力**:
  • 構築された HStore 構造体へのポインタを Datum として返却。第1引数が NULL の場合は NULL を返却。

前提条件と不変条件

  • 第1引数（キー）が NULL の場合、関数は即座に NULL を返却します。
  • キーおよび値の長さは、hstoreCheckKeyLen および hstoreCheckValLen によって制限値内に収まっている必要があります。

アルゴリズムまたは振る舞い

1. 第1引数が NULL か確認し、該当すれば NULL を返却。 2. Pairs 構造体を初期化し、第1引数からキーのデータと長さを取得。 3. 第2引数が NULL の場合、値を NULL としてマークし、長さを 0 に設定。 4. 第2引数が NULL でない場合、値のデータと長さを取得。 5. hstorePairs 関数を呼び出し、単一のペアから HStore 構造体を生成。 6. 生成された HStore 構造体を返却。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • PG_FUNCTION_INFO_V1(hstore_from_text) によるマクロ定義。
  • HSTORE_POLLUTE(hstore_from_text, tconvert) による登録。
  • contrib/hstore/hstore_subs.c 内のコメントで、単一要素の hstore を構築する例として言及あり。
  • **呼び出し先**:
  • PG_GETARG_TEXT_PP: 引数の取得。
  • hstoreCheckKeyLen / hstoreCheckValLen: 長さの検証。
  • hstorePairs: HStore 構造体の構築。

関連データ構造

  • Pairs: キーと値のペアを保持する一時的な構造体。
  • HStore: 最終的な hstore 型の内部表現。

失敗モードと境界条件

  • 第1引数が NULL の場合は NULL を返却する。
  • キーまたは値が hstore の許容する最大長を超過した場合、hstoreCheckKeyLen または hstoreCheckValLen 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は単一ペアの変換に特化しています。複数のペアを扱う場合は hstore_from_arrays 等の利用を検討してください。
  • hstorePairs に渡すメモリサイズは、キーと値の長さの合計値に基づいています。
