mbuf_create

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/mbuf.c:110

目的

可変長メモリバッファ（MBuf）構造体を初期化し、指定されたサイズのメモリ領域を確保する。

シグネチャまたは形状

MBuf *mbuf_create(int len)

入力と出力

  • **入力**: len (確保するバッファの初期サイズ。0を指定した場合はデフォルトの8192バイトが使用される)
  • **出力**: 初期化された MBuf 構造体へのポインタ。

前提条件と不変条件

  • PostgreSQL のメモリコンテキスト内で実行されることを前提とする（palloc を使用）。
  • 戻り値の MBuf は、使用後に mbuf_free 等で適切に解放する必要がある。
  • no_write は false、own_data は true に初期化される。

アルゴリズムまたは振る舞い

1. len が 0 の場合、デフォルト値 8192 を設定する。 2. palloc を使用して MBuf 構造体本体を確保する。 3. 指定されたサイズ len 分のメモリを palloc で確保し、mbuf->data に割り当てる。 4. バッファの終端 (buf_end)、データ終端 (data_end)、読み込み位置 (read_pos) を初期化する。 5. 状態フラグを初期化し、構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c:398
  • contrib/pgcrypto/pgp-pgsql.c:477
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理関数)

関連データ構造

  • MBuf: 本関数によって生成されるデータ保持用構造体。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQL のエラーハンドリング機構により例外が発生する。

保守担当者向けメモ

  • 本関数は palloc を使用しているため、呼び出し側でメモリリークが発生しないよう注意すること。
  • len に負の値を渡した場合の挙動については、ソースコード上での明示的なチェックは存在しない。
