uuid_le

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/uuid.c:218

目的

2つの uuid 型の値が「小なりまたは等しい（less than or equal to）」という関係にあるかを判定します。

シグネチャまたは形状

Datum uuid_le(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの pg_uuid_t 型のポインタを受け取ります。
  • **出力**: Datum 型として、第1引数が第2引数以下であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力される2つの pg_uuid_t ポインタは有効である必要があります。
  • uuid_internal_cmp が提供する比較ロジックに基づき、16バイトのバイナリ値として比較が行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_UUID_P マクロを使用して、引数から pg_uuid_t 構造体へのポインタを取得します。 2. uuid_internal_cmp 関数を呼び出し、2つの UUID を比較します。 3. 比較結果が 0 以下（<= 0）であるかを判定します。 4. PG_RETURN_BOOL を使用して、判定結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_minmax_multi.c:2062 において、DirectFunctionCall2 を介して呼び出されていることが確認されています。
  • **呼び出し先**:
  • uuid_internal_cmp: 実際のバイナリ比較を実行するために呼び出されます。

関連データ構造

  • pg_uuid_t: UUID を保持する 16 バイトの構造体。

失敗モードと境界条件

  • 引数が NULL の場合、PG_GETARG_UUID_P の挙動に従い、呼び出し元で適切に処理される必要があります（通常は SQL レベルで NULL 伝播が発生します）。
  • 比較はバイナリレベルで行われるため、UUID のバージョンや形式に関わらず、メモリ上のバイト列として辞書順比較が行われます。

保守担当者向けメモ

  • 本関数は SQL 演算子 <= の実装として機能します。
  • 比較ロジックを変更する場合は、uuid_internal_cmp を参照してください。
