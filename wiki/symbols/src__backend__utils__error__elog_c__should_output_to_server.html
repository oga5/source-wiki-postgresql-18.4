should_output_to_server

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:235

目的

指定されたエラーレベル（elevel）が、現在のサーバーログ出力設定（log_min_messages）に基づいてログに出力されるべきかどうかを判定する。

シグネチャまたは形状

static inline bool should_output_to_server(int elevel)

入力と出力

  • **入力**: elevel (int) - 判定対象のエラーレベル。
  • **出力**: bool - ログ出力対象であれば true、そうでなければ false を返す。

前提条件と不変条件

  • log_min_messages グローバル変数が適切に設定されていること。
  • is_log_level_output 関数が利用可能であること。

アルゴリズムまたは振る舞い

内部的に is_log_level_output(elevel, log_min_messages) を呼び出し、その結果をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/error/elog.c 内の複数箇所（エラー報告処理の判定ロジックで使用）。
  • **呼び出し先**:
  • is_log_level_output

関連データ構造

  • log_min_messages (GUC変数): ログ出力の最小深刻度を決定する設定値。

失敗モードと境界条件

  • elevel に無効な値が渡された場合の挙動は is_log_level_output の実装に依存する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを考慮して elog.c 内でのみ使用されることを想定している。
  • ログ出力の条件を変更する場合は、本関数ではなく is_log_level_output を修正すること。
