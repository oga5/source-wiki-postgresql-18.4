ltree_lt

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:94

目的

2つの ltree 型の値が「より小さい（less than）」という関係にあるかを判定し、SQL の比較演算子 < の実装として機能します。

シグネチャまたは形状

Datum ltree_lt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される2つの ltree 型の Datum。
  • **出力**: bool 型の Datum。第1引数が第2引数より小さい場合は true、それ以外は false を返します。

前提条件と不変条件

  • RUNCMP マクロが定義されている環境で実行される必要があります。
  • RUNCMP は内部的に ltree_compare を呼び出し、比較結果を res 変数に格納することを前提としています。

アルゴリズムまたは振る舞い

1. RUNCMP マクロを実行し、引数として渡された2つの ltree を比較します。 2. 比較結果 res が 0 未満（res < 0）であるかを確認します。 3. PG_RETURN_BOOL を使用して、比較結果が真であれば true を、偽であれば false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の SQL エンジン（演算子 < の実行時）。
  • **呼び出し先**: RUNCMP マクロ（内部で ltree_compare を呼び出す）。
  • 注: ソースコード上の直接的な呼び出し元として PG_FUNCTION_INFO_V1(ltree_lt) が存在します。

関連データ構造

  • ltree: 比較対象となる階層ラベルパス構造体。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • 引数が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で適切に処理される必要があります（通常は NULL が返されます）。
  • ltree 構造が不正な場合、ltree_compare 内部で予期せぬ動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は ltree_op.c 内の他の比較関数（ltree_le, ltree_eq 等）と同一のパターンで実装されています。ロジックを変更する場合は、ltree_compare の仕様と整合性を保つ必要があります。
