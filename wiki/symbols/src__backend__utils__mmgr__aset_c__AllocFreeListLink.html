AllocFreeListLink

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/aset.c:122

目的

解放されたメモリチャンクをフリーリスト上で管理するためのリンク構造体です。メモリ割り当て要求に対して再利用可能なチャンクを効率的に追跡するために使用されます。

シグネチャまたは形状

typedef struct AllocFreeListLink
{
	MemoryChunk *next;
} AllocFreeListLink;

入力と出力

  • 入力: MemoryChunk へのポインタを保持します。
  • 出力: フリーリスト内の次のチャンクへの参照を提供します。

前提条件と不変条件

  • 割り当てサイズは常に sizeof(AllocFreeListLink) 以上である必要があります。
  • sizeof(AllocFreeListLink) は 1 << ALLOC_MINBITS 以下であるという静的アサーションが定義されています。

アルゴリズムまたは振る舞い

  • チャンクが解放される際、そのチャンクのメモリ領域を再利用して AllocFreeListLink を配置します。
  • GetFreeListLink マクロ（src/backend/utils/mmgr/aset.c:133）を介して、チャンクヘッダー直後の領域を本構造体としてキャストし、リストの操作を行います。
  • Valgrind 使用時には、リンク操作の前後でメモリのアクセス権限（DEFINED / NOACCESS）を明示的に制御し、不正なアクセスを防止します。

呼び出し元と呼び出し先

  • 呼び出し元: AllocSetFree および AllocSetAlloc 内のフリーリスト操作ロジックから直接参照されます。
  • 呼び出し先: MemoryChunk 型のポインタを介してリスト構造を形成します。

関連データ構造

  • MemoryChunk: 本構造体が指し示す対象。
  • AllocSetFreeList: 本構造体を用いて管理されるフリーリストの集合。

失敗モードと境界条件

  • 割り当てサイズが sizeof(AllocFreeListLink) 未満の場合、構造体を配置できずメモリ破壊を招く可能性があります（静的アサーションによりコンパイル時に検証されます）。

保守担当者向けメモ

  • 本構造体はチャンクのメモリ領域をオーバーレイして使用するため、チャンクがフリーリストにある間のみ有効です。
  • VALGRIND_MAKE_MEM_DEFINED 等のデバッグ用マクロが aset.c 内の各所で呼び出されているため、構造体のサイズ変更時にはこれらの箇所も整合性を確認してください。
