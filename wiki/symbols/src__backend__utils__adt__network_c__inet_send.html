inet_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network.c:289

目的

inet 型のデータを PostgreSQL のバイナリ転送形式（bytea）に変換します。

シグネチャまたは形状

Datum inet_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して inet 型のポインタを受け取ります。
  • **出力**: bytea 型の Datum を返します。

前提条件と不変条件

  • 入力される inet 型のデータは、PG_GETARG_INET_PP(0) マクロによって正しく取得可能である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_INET_PP(0) を使用して、入力された inet 構造体へのポインタを取得します。 2. network_send(addr, false) を呼び出し、inet データをバイナリ形式に変換します。この際、第2引数に false を渡すことで、cidr ではなく inet としての処理を指示します。 3. PG_RETURN_BYTEA_P マクロを使用して、変換されたバイナリデータを返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: network_send
  • **呼び出し元**: 提供されたソース excerpt 内には明示的な呼び出し元は記載されていません。

関連データ構造

  • inet: ネットワークアドレスを保持する内部構造体。
  • bytea: PostgreSQL の可変長バイナリデータ型。

失敗モードと境界条件

  • PG_GETARG_INET_PP が無効なポインタを返した場合や、メモリ不足が発生した場合にはエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は inet 型専用のラッパーであり、実際のバイナリ変換ロジックは network_send に集約されています。
  • cidr 型のバイナリ変換が必要な場合は、別途 cidr_send が使用されます。
