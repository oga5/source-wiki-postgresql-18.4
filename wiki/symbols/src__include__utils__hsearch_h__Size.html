Size

定義

  • 種別: typedef
  • 定義箇所: src/include/utils/hsearch.h:21

目的

メモリサイズやデータ長を表すための汎用的な型定義です。

シグネチャまたは形状

typedef size_t Size; // 推定される基底型

※ソース excerpt 上では Size 型そのものの定義式は明示されていませんが、HashValueFunc の引数として Size keysize の形式で使用されています。

入力と出力

  • 入力: 該当なし（型定義のため）
  • 出力: 該当なし

前提条件と不変条件

  • size_t と互換性のある整数型であることが期待されます。
  • メモリ割り当てサイズやデータ長を表現するために使用されます。

アルゴリズムまたは振る舞い

  • 本シンボルは型エイリアスであり、実行時のアルゴリズムは持ちません。
  • hsearch.h 内では、ハッシュ関数の引数（キーサイズ）や、ハッシュテーブルの制御構造体におけるサイズ指定に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • contrib/amcheck/verify_nbtree.c
  • contrib/bloom/bloom.h
  • contrib/cube/cube.c および cubedata.h
  • contrib/file_fdw/file_fdw.c
  • contrib/pg_buffercache/pg_buffercache_pages.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • 呼び出し先:
  • 情報なし（型定義のため）

関連データ構造

  • HashValueFunc: ハッシュ関数定義においてキーサイズを指定するために使用。
  • HASHCTL: ハッシュテーブル作成時のサイズ制御に使用される構造体。

失敗モードと境界条件

  • 型のオーバーフロー: 割り当てられるメモリサイズが Size 型の最大値を超える場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは PostgreSQL 全体で広く利用される基本的な型定義です。変更を行う際は、システム全体のメモリ管理および型定義への影響を考慮してください。
