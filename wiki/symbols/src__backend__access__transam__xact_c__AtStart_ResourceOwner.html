AtStart_ResourceOwner

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xact.c:1225

目的

トランザクションの開始時に、トップレベルのリソースオーナー（ResourceOwner）を生成し、現在のトランザクション状態およびグローバルなリソース管理ポインタに割り当てる。

シグネチャまたは形状

static void AtStart_ResourceOwner(void)

入力と出力

  • 入力: なし（CurrentTransactionState を参照）
  • 出力: なし（副作用としてグローバル変数および TransactionState を更新）

前提条件と不変条件

  • TopTransactionResourceOwner が NULL であること（既にリソースオーナーが存在しない状態であること）。

アルゴリズムまたは振る舞い

1. 現在のトランザクション状態（CurrentTransactionState）を取得する。 2. ResourceOwnerCreate を呼び出し、名前を "TopTransaction" としてトップレベルのリソースオーナーを生成する。 3. 生成したリソースオーナーを以下の変数に設定する。

  • s->curTransactionOwner（現在のトランザクション状態）
  • TopTransactionResourceOwner（グローバルなトップレベルリソースオーナー）
  • CurTransactionResourceOwner（現在のリソースオーナー）
  • CurrentResourceOwner（現在アクティブなリソースオーナー）

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c 内のトランザクション開始処理（StartTransaction 等から参照される）。
  • 呼び出し先: ResourceOwnerCreate（utils/resowner.h で定義）。

関連データ構造

  • TransactionStateData: トランザクション状態を保持する構造体。
  • ResourceOwner: リソース（メモリ、ロック、ファイル等）の所有権を管理する構造体。

失敗モードと境界条件

  • TopTransactionResourceOwner が既に設定されている場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数はトランザクション開始時の初期化の一部です。リソースリークを防ぐため、ここで作成されたリソースオーナーはトランザクション終了時に適切に解放される必要があります。
  • CurrentResourceOwner への代入は、この関数以降の処理が正しいリソース管理コンテキストで実行されることを保証します。
