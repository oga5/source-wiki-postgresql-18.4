pgaio_io_start_writev

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_io.c:90

目的

非同期I/O操作として書き込み（writev）を開始するためのエントリーポイントを提供します。

シグネチャまたは形状

void pgaio_io_start_writev(PgAioHandle *ioh, int fd, int iovcnt, uint64 offset)

入力と出力

  • **入力**:
  • ioh: 操作対象の PgAioHandle 構造体へのポインタ。
  • fd: 書き込み対象のファイル記述子。
  • iovcnt: 書き込み対象の iovec の数。
  • offset: 書き込みを開始するファイル内のオフセット。
  • **出力**: なし（void）。

前提条件と不変条件

  • pgaio_io_before_start(ioh) により、I/O開始前のガード条件（割り込み禁止状態の確認など）が強制されます。

アルゴリズムまたは振る舞い

1. pgaio_io_before_start を呼び出し、I/O開始の妥当性を検証します。 2. 引数として渡された fd、offset、iovcnt を ioh->op_data.write 構造体に格納します。 3. pgaio_io_stage を呼び出し、操作種別 PGAIO_OP_WRITEV を指定してI/Oサブシステムへ処理を登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**:
  • pgaio_io_before_start
  • pgaio_io_stage

関連データ構造

  • PgAioHandle: I/O操作の状態を保持するハンドル。
  • PGAIO_OP_WRITEV: 書き込み操作を示す列挙値。

失敗モードと境界条件

  • pgaio_io_before_start 内の不変条件（Assert(!INTERRUPTS_CAN_BE_PROCESSED())）に違反した場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数はI/Oの「準備」と「実行」の境界を定義しています。
  • ioh に書き込まれたパラメータは、後続の pgaio_io_stage を経て、各メソッド（io_uringや同期実行など）の実装に引き継がれます。
