ExecParallelInitializeDSMContext

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/execParallel.c:119

目的

並列クエリ実行における動的共有メモリ（DSM）の初期化プロセスにおいて、実行プランのノード走査および状態管理に必要なコンテキスト情報を保持します。

シグネチャまたは形状

typedef struct ExecParallelInitializeDSMContext
{
    ParallelContext *pcxt;
    SharedExecutorInstrumentation *instrumentation;
    int            nnodes;
} ExecParallelInitializeDSMContext;

入力と出力

本構造体は関数引数として渡され、以下のフィールドを介して情報を伝播します。

  • **入力**: pcxt（並列コンテキスト）、instrumentation（共有計測情報）、nnodes（ノード数）。
  • **出力**: 初期化処理中に各プランノードがDSM領域を確保・設定するための状態保持に使用されます。

前提条件と不変条件

  • pcxt は有効な ParallelContext を指している必要があります。
  • nnodes はプランツリー内の並列実行に関与するノード数と一致している必要があります。

アルゴリズムまたは振る舞い

1. リーダープロセスが ExecParallelInitializeDSM を呼び出す際、本構造体を初期化してプランツリーを再帰的に走査します。 2. 各プランノードは、このコンテキストを参照して自身のDSM領域を確保し、必要な初期化を実行します。 3. 実行終了まで、このコンテキストを通じて共有メモリ上の各データ構造へのアクセスが管理されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c 内の ExecParallelInitializeDSM 等から参照・生成されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、プランノードの初期化関数群（ExecParallelInitializeDSM 等）によって操作されます。

関連データ構造

  • ParallelContext: 並列実行全体の管理構造体。
  • SharedExecutorInstrumentation: ワーカープロセスからの統計情報を集約するための共有メモリ領域。

失敗モードと境界条件

  • pcxt が NULL の場合、または nnodes が不適切な値である場合、初期化処理が異常終了する可能性があります。
  • 共有メモリの確保に失敗した場合、本コンテキストを用いた初期化は中断されます。

保守担当者向けメモ

  • 新しい並列対応プランノードを追加する際は、ExecParallelInitializeDSM 内で本コンテキストを適切に処理するロジックを追加してください。
  • nnodes のカウントがプランツリーの構造と整合していることを常に確認してください。
