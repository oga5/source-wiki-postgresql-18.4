get_default_partition_oid

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/partition.c:314

目的

指定されたパーティション化テーブル（親テーブル）に関連付けられているデフォルトパーティションのOIDを取得します。

シグネチャまたは形状

Oid get_default_partition_oid(Oid parentId)

入力と出力

  • **入力**: parentId (Oid) - 調査対象のパーティション化テーブルのOID。
  • **出力**: Oid - デフォルトパーティションのOID。デフォルトパーティションが存在しない場合は InvalidOid を返します。

前提条件と不変条件

  • parentId は pg_partitioned_table システムカタログに存在する有効なパーティション化テーブルのOIDである必要があります。

アルゴリズムまたは振る舞い

1. PARTRELID システムキャッシュを使用して、parentId に対応する pg_partitioned_table のタプルを検索します。 2. タプルが見つかった場合、GETSTRUCT を介して Form_pg_partitioned_table 構造体へキャストし、partdefid フィールドからデフォルトパーティションのOIDを抽出します。 3. 取得したタプルを ReleaseSysCache で解放します。 4. タプルが見つからない場合、またはデフォルトパーティションが設定されていない場合は InvalidOid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c:1832
  • src/backend/partitioning/partdesc.c:293
  • **呼び出し先**:
  • SearchSysCache1 (システムキャッシュ検索)
  • ReleaseSysCache (キャッシュ解放)

関連データ構造

  • Form_pg_partitioned_table: pg_partitioned_table カタログの行構造体。partdefid フィールドを保持します。

失敗モードと境界条件

  • 指定された parentId がパーティション化テーブルでない場合、システムキャッシュからタプルが取得できず、InvalidOid が返されます。
  • メモリ不足等のシステムエラーが発生した場合、SearchSysCache1 がエラーをスローする可能性があります。

保守担当者向けメモ

  • 本関数は読み取り専用の操作です。デフォルトパーティションの更新が必要な場合は update_default_partition_oid を使用してください。
  • PARTRELID キャッシュを使用しているため、カタログの更新直後には最新の状態が反映されない可能性があります（キャッシュの整合性はシステムキャッシュの仕組みに依存します）。
