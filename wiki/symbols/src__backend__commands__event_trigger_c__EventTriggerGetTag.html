EventTriggerGetTag

定義

  • 種別: function
  • 定義箇所: src/backend/commands/event_trigger.c:623

目的

イベントトリガーの実行コンテキストに基づき、対応するコマンドタグ（CommandTag）を取得します。

シグネチャまたは形状

static CommandTag EventTriggerGetTag(Node *parsetree, EventTriggerEvent event)

入力と出力

  • **入力**:
  • parsetree: 解析対象の構文木（Node *）。
  • event: イベントの種類（EventTriggerEvent）。
  • **出力**:
  • 該当する CommandTag を返します。

前提条件と不変条件

  • event が EVT_Login の場合、構文木の内容に関わらず CMDTAG_LOGIN を返します。
  • それ以外の場合は、CreateCommandTag(parsetree) を呼び出してタグを生成します。

アルゴリズムまたは振る舞い

1. event 引数が EVT_Login であるかを確認します。 2. EVT_Login であれば、即座に CMDTAG_LOGIN を返します。 3. EVT_Login でなければ、CreateCommandTag 関数に parsetree を渡し、その戻り値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/event_trigger.c 内の複数箇所（行624, 665, 689）から呼び出されています。
  • **呼び出し先**:
  • CreateCommandTag（event が EVT_Login ではない場合）。

関連データ構造

  • CommandTag: コマンドの種類を識別する列挙型。
  • EventTriggerEvent: イベントの種類を定義する列挙型。
  • Node: PostgreSQLの構文木ノード。

失敗モードと境界条件

  • parsetree が CreateCommandTag で処理できない形式である場合、その関数の仕様に従った挙動となります。

保守担当者向けメモ

  • 本関数は静的関数であり、event_trigger.c 内でのみ使用されます。
  • EVT_Login 以外のイベントタイプが追加された場合、必要に応じて条件分岐の拡張を検討してください。
