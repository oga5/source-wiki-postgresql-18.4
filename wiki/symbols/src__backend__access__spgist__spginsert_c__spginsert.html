spginsert

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spginsert.c:182

目的

SP-GiSTインデックスへの単一タプル挿入を行うための外部インターフェースです。並行挿入による競合を検知し、成功するまで挿入処理を再試行する役割を担います。

シグネチャまたは形状

bool spginsert(Relation index, Datum *values, bool *isnull,
               ItemPointer ht_ctid, Relation heapRel,
               IndexUniqueCheck checkUnique,
               bool indexUnchanged,
               IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • index: 挿入対象のインデックスリレーション。
  • values, isnull: 挿入するタプルの属性値とNULLフラグ。
  • ht_ctid: 挿入対象タプルのヒープ内位置。
  • その他: heapRel, checkUnique, indexUnchanged, indexInfo（本関数内では主にインターフェース互換のために受け取ります）。
  • **出力**:
  • bool: 常に false を返します（一意性チェックを行わないため）。

前提条件と不変条件

  • 挿入処理中に一時的なメモリ消費を抑制するため、専用のメモリコンテキスト（insertCtx）が作成されます。
  • 競合発生時には insertCtx がリセットされ、状態が再初期化されます。

アルゴリズムまたは振る舞い

1. 挿入用の一時メモリコンテキストを作成し、現在のコンテキストを切り替えます。 2. initSpGistState を呼び出し、インデックスの状態を初期化します。 3. spgdoinsert が成功を返すまでループを実行します。

  • 競合が発生した場合（spgdoinsert が false を返した場合）、insertCtx をリセットし、initSpGistState を再実行してリトライします。

    • 4. 挿入完了後、SpGistUpdateMetaPage を呼び出してメタページを更新します。 5. 一時メモリコンテキストを削除し、元のコンテキストへ戻ります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgutils.c 内で amroutine->aminsert として登録されており、インデックスアクセスメソッドのインターフェース経由で呼び出されます。
  • **呼び出し先**:
  • initSpGistState
  • spgdoinsert
  • SpGistUpdateMetaPage
  • メモリ管理関数 (AllocSetContextCreate, MemoryContextSwitchTo, MemoryContextReset, MemoryContextDelete)

関連データ構造

  • SpGistState: インデックス操作の状態を保持する構造体。
  • MemoryContext: 挿入処理中のメモリ管理に使用。

失敗モードと境界条件

  • 並行挿入による競合が発生した場合、spgdoinsert が false を返すことで再試行ループが駆動されます。
  • 本関数自体は一意性チェックを行わないため、常に false を返します。

保守担当者向けメモ

  • spgdoinsert が失敗した際のリトライ処理において、initSpGistState の再実行は必須です。
  • 挿入処理はメモリを消費しやすいため、insertCtx のライフサイクル管理が重要です。デバッグ時にはメモリリークの有無を確認してください。
