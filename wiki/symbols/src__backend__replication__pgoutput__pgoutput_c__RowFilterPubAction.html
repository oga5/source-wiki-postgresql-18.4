RowFilterPubAction

定義

  • 種別: enum
  • 定義箇所: src/backend/replication/pgoutput/pgoutput.c:99

目的

論理レプリケーションにおける行フィルタリングの結果、UPDATE操作をどの操作（INSERT、UPDATE、またはDELETE）としてサブスクライバーへ送信すべきかを識別するために使用されます。

シグネチャまたは形状

enum RowFilterPubAction
{
	PUBACTION_INSERT,
	PUBACTION_UPDATE,
	PUBACTION_DELETE,
};

入力と出力

  • 入力: 行フィルタの評価結果（pgoutput_row_filter 内での判定）。
  • 出力: 該当する列挙値。

前提条件と不変条件

  • pgoutput_row_filter において、UPDATE操作がフィルタ条件によって変換される際に使用されます。
  • 常に PUBACTION_INSERT、PUBACTION_UPDATE、PUBACTION_DELETE のいずれかの値をとります。

アルゴリズムまたは振る舞い

論理デコーディング中にUPDATEイベントが発生した際、行フィルタの評価結果に基づいて、元のUPDATEをINSERTやDELETEに変換して送信するための制御フラグとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: pgoutput_change 内の行フィルタ処理ロジック。
  • 呼び出し先: 情報なし。

関連データ構造

  • RelationSyncEntry: パブリケーション設定や行フィルタ情報を保持する構造体。

失敗モードと境界条件

  • 情報なし。

保守担当者向けメモ

  • UPDATE操作の変換ロジックはサブスクライバー側のデータ整合性に直結するため、この列挙値の追加や変更を行う際は、pgoutput_change における送信処理への影響を十分に確認してください。
