internal_load_library

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/dfmgr.c:188

目的

指定された共有ライブラリをプロセス空間にロードし、ABI 互換性の検証および初期化処理を行う。ロード済みライブラリのリストを管理し、重複ロードを防止する。

シグネチャまたは形状

static void *internal_load_library(const char *libname)

入力と出力

  • **入力**: libname (ロード対象のライブラリパス)
  • **出力**: ロードされたライブラリのハンドル（void *）を返す。失敗時はエラーを報告しプロセスを中断する。

前提条件と不変条件

  • ライブラリは PG_MODULE_MAGIC マクロを使用して ABI 互換性を宣言している必要がある。
  • ロード済みライブラリは file_list および file_tail によって連結リストとして管理される。

アルゴリズムまたは振る舞い

1. **重複確認**: file_list を走査し、指定された libname が既にロード済みか確認する。 2. **パス解決**: 未ロードの場合、stat を用いてファイル情報を取得し、inode 等の比較により別名（シンボリックリンク等）による重複ロードを防止する。 3. **ロード**: dlopen を使用してライブラリをロードする。 4. **ABI 検証**: PG_MAGIC_FUNCTION_NAME_STRING シンボルを検索し、Pg_magic_struct を用いてバージョンおよび ABI 互換性をチェックする。不一致の場合は incompatible_module_error を呼び出し、ライブラリをクローズする。 5. **初期化**: _PG_init シンボルが存在する場合、それを実行する。 6. **登録**: 成功したライブラリ情報を DynamicFileList に追加し、ハンドルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/fmgr/dfmgr.c 内の load_file 等から呼び出される。src/backend/postmaster/pgarch.c からの参照も確認されている。
  • **呼び出し先**: dlopen, dlsym, dlclose, stat, malloc, free, incompatible_module_error, ereport 等。

関連データ構造

  • DynamicFileList: ロード済みライブラリを保持する連結リスト構造体。
  • Pg_magic_struct: ABI 互換性チェックに使用される構造体。

失敗モードと境界条件

  • **メモリ不足**: malloc 失敗時に ERRCODE_OUT_OF_MEMORY でエラー報告。
  • **アクセス権限・ファイル不在**: stat 失敗時にエラー報告。
  • **ロード失敗**: dlopen 失敗時に dlerror を含むエラー報告。
  • **ABI 不一致**: PG_MODULE_MAGIC が存在しない、または構造体が一致しない場合にエラー報告。

保守担当者向けメモ

  • 本関数は一度ロードしたライブラリをアンロードする仕組みを持たない。
  • Windows 環境では dlfcn.h 互換の抽象化層が使用される。
  • _PG_init の実行はライブラリロード直後に行われるため、初期化処理の副作用に注意すること。
