push_back_token

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_scanner.c:384

目的

字句解析器（スキャナ）において、一度読み取ったトークンをスタックに戻し、次回の解析時に再利用可能にする。

シグネチャまたは形状

static void push_back_token(int token, TokenAuxData *auxdata, yyscan_t yyscanner)

入力と出力

  • token: 戻すトークンの識別子（整数値）。
  • auxdata: トークンに関連付けられた補助データ（TokenAuxData 構造体）。
  • yyscanner: スキャナの状態を保持する yyscan_t 型のコンテキスト。
  • 出力: なし（副作用としてスキャナ状態を更新）。

前提条件と不変条件

  • yyextra（yyscanner から取得されるスキャナ状態）が初期化されていること。
  • スタック内のトークン数が MAX_PUSHBACKS（4）を超えていないこと。

アルゴリズムまたは振る舞い

1. 現在のプッシュバック数（yyextra->num_pushbacks）が MAX_PUSHBACKS に達しているか確認する。 2. 達している場合は elog(ERROR) を発生させる。 3. 達していない場合、指定された token と auxdata を yyextra 内のプッシュバック用スタック配列の末尾に格納する。 4. yyextra->num_pushbacks をインクリメントする。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/pl_scanner.c 内の複数の箇所（plpgsql_yylex 等の字句解析ロジック）から、先読み後の復帰処理として呼び出される。
  • 呼び出し先: elog（エラー発生時）。

関連データ構造

  • TokenAuxData: トークンに付随する補助情報を保持する構造体。
  • plpgsql_yy_extra_type: スキャナの状態を保持する構造体。プッシュバック用スタック（pushback_token 配列および pushback_auxdata 配列）とカウンタ（num_pushbacks）を含む。

失敗モードと境界条件

  • MAX_PUSHBACKS（4）を超えるトークンを戻そうとした場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は plpgsql_yylex が複雑な文法解析のために先読みを行う際の補助機能である。
  • MAX_PUSHBACKS の制限は、PL/pgSQL の文法解析における先読みの深さに依存している。この制限を変更する場合は、pl_scanner.c 内の定義を確認すること。
