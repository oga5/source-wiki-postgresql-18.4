ForgetManyTestResources

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_resowner/test_resowner_many.c:143

目的

ResourceOwner に登録されているテスト用リソースを明示的に破棄し、内部管理リストと ResourceOwner の状態を同期させる。

シグネチャまたは形状

static void ForgetManyTestResources(ResourceOwner owner, ManyTestResourceKind *kinds, int nkinds, int nresources)

入力と出力

  • **入力**:
  • owner: 対象の ResourceOwner インスタンス。
  • kinds: リソースの種類を保持する ManyTestResourceKind 配列。
  • nkinds: kinds 配列の要素数。
  • nresources: 破棄するリソースの総数。
  • **出力**: なし（副作用としてリソースの解放とメモリの解放を行う）。

前提条件と不変条件

  • GetTotalResourceCount で算出される総リソース数が、要求された nresources 以上であること。
  • kinds 配列内の各リソースリストが dlist で正しく管理されていること。

アルゴリズムまたは振る舞い

1. GetTotalResourceCount を呼び出し、現在の総リソース数が nresources を満たしているか確認する（不足時は PANIC）。 2. nresources 回のループを実行し、以下の処理を行う：

  • kinds 配列を順次走査し、リソースが登録されている最初のリストを見つける。
  • ResourceOwnerForget を呼び出し、ResourceOwner から当該リソースを削除する。
  • 内部管理用の dlist から該当要素を削除し、pfree でメモリを解放する。
  • kinds[kind_idx].nforgotten をインクリメントする。

    • 3. 破棄対象が見つからない場合は ERROR を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: test_resowner_many（src/test/modules/test_resowner/test_resowner_many.c:264, 270）
  • **呼び出し先**:
  • GetTotalResourceCount
  • ResourceOwnerForget
  • dlist_is_empty, dlist_head_element, dlist_delete
  • pfree

関連データ構造

  • ManyTestResourceKind: リソースの分類と管理状態を保持。
  • ManyTestResource: 個別のリソースインスタンス。
  • ResourceOwner: PostgreSQLのリソース所有者管理構造体。

失敗モードと境界条件

  • ntotal < nresources: 破棄対象が不足している場合、PANIC で異常終了する。
  • !found: 内部リストと ResourceOwner の整合性が取れず、破棄対象が見つからない場合、ERROR で異常終了する。

保守担当者向けメモ

  • 本関数は ResourceOwner の Forget ロジックを検証するためのテスト用ユーティリティである。
  • kind_idx を循環させてリソースを選択しているため、特定の種類のリストに偏らずに解放が行われる設計となっている。
