PLy_pop_execution_context

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_main.c:407

目的

現在実行中の Python 実行コンテキストをスタックから取り出し、関連するメモリリソースを解放します。

シグネチャまたは形状

static void PLy_pop_execution_context(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてスタックの更新とメモリ解放を行う）

前提条件と不変条件

  • PLy_execution_contexts スタックが空ではないこと。
  • 呼び出し時に、解放対象のコンテキストがスタックの先頭にあること。

アルゴリズムまたは振る舞い

1. グローバル変数 PLy_execution_contexts から現在のコンテキストを取得します。 2. コンテキストが NULL の場合、エラーを発生させます。 3. スタックの先頭を次のコンテキスト（context->next）に更新します。 4. コンテキスト内に scratch_ctx が存在する場合、MemoryContextDelete を呼び出してメモリを解放します。 5. コンテキスト構造体自体を pfree で解放します。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_main.c 内の各ハンドラ（関数呼び出し、トリガー、DO ブロック実行の終了処理）。
  • 呼び出し先: MemoryContextDelete（PostgreSQL メモリ管理）、pfree（PostgreSQL メモリ管理）、elog（エラー報告）。

関連データ構造

  • PLyExecutionContext: 実行中の手続きやメモリコンテキストを保持するスタック構造体。

失敗モードと境界条件

  • スタックが空の状態で呼び出された場合、elog(ERROR, ...) により PostgreSQL のエラーとして処理されます。

保守担当者向けメモ

  • scratch_ctx は遅延生成される可能性があるため、本関数での削除はメモリリーク防止に不可欠です。
  • SPI_finish() は本関数ではなく plpy_exec.c 側で管理されているため、呼び出し順序に注意してください。
