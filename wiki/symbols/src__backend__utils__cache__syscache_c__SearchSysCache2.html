SearchSysCache2

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/syscache.c:231

目的

2つのキーを使用してシステムカタログキャッシュを検索し、該当するタプルを取得します。

シグネチャまたは形状

HeapTuple SearchSysCache2(int cacheId, Datum key1, Datum key2)

入力と出力

  • **入力**:
  • cacheId: 検索対象のシステムキャッシュを識別するID。
  • key1, key2: 検索に使用する2つのキー値（Datum型）。
  • **出力**:
  • 成功時: キャッシュされたタプルへのポインタ（HeapTuple）。
  • 失敗時: 該当なしの場合は NULL。

前提条件と不変条件

  • cacheId は有効な範囲内（0 以上 SysCacheSize 未満）であること。
  • 指定された cacheId に対応するキャッシュが初期化済みであること。
  • 指定されたキャッシュのキー数が2つであること（cc_nkeys == 2）。

アルゴリズムまたは振る舞い

1. cacheId が有効かつ対応するキャッシュが存在することを確認するアサーションを実行します。 2. 指定されたキャッシュのキー数が2であることを検証します。 3. catcache.c が提供する SearchCatCache2 を呼び出し、キャッシュ内のハッシュテーブルからタプルを検索して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/dml.c
  • src/backend/catalog/aclchk.c
  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/catalog/objectaddress.c
  • **呼び出し先**:
  • SearchCatCache2 (内部的な検索実行関数)

関連データ構造

  • SysCache[]: システムキャッシュを保持する CatCache 構造体の配列。
  • HeapTuple: 検索結果として返されるタプル構造体。

失敗モードと境界条件

  • cacheId が不正な場合、アサーション違反によりプロセスが異常終了する可能性があります。
  • 指定されたキーに一致するタプルが存在しない場合、NULL を返します。
  • 返されたタプルはキャッシュ内部のポインタであるため、呼び出し側で直接変更してはなりません。

保守担当者向けメモ

  • 本関数は SearchSysCache* シリーズの一つであり、キー数に応じた検索を行います。
  • キャッシュの定義は syscache_info.h に依存しています。
  • タプルの変更が必要な場合は、本関数ではなく SearchSysCacheCopy を使用してください。
