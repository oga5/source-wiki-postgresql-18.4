cleanup_objects_atexit

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:176

目的

プログラム終了時に、パブリッシャー側で作成された不要な論理レプリケーション用オブジェクト（パブリケーションおよびレプリケーションスロット）を削除し、必要に応じてサブスクライバーの停止を行う。

シグネチャまたは形状

static void cleanup_objects_atexit(void)

入力と出力

  • 入力: なし（グローバル変数 success, recovery_ended, num_dbs, dbinfos, standby_running, subscriber_dir を参照）
  • 出力: なし（副作用としてパブリッシャーへの接続とオブジェクト削除、サブスクライバーの停止を実行）

前提条件と不変条件

  • atexit() を介して登録されることを想定している。
  • success が真の場合、本関数は即座に終了する。
  • recovery_ended が真の場合、物理レプリカとしての再利用が不可能であることを示す警告を出力する。

アルゴリズムまたは振る舞い

1. success が真であれば何もせず終了する。 2. recovery_ended が真であれば、物理レプリカの再作成が必要である旨の警告を表示する。 3. num_dbs 分だけループし、各データベースに対して以下の処理を行う：

  • パブリッシャーへの接続を試みる。
  • 接続成功時：made_publication または made_replslot が真であれば、それぞれ drop_publication および drop_replication_slot を呼び出して削除する。
  • 接続失敗時：削除できなかったオブジェクトについて、手動削除を促す警告を表示する。

    • 4. standby_running が真であれば、stop_standby_server を呼び出してサブスクライバーを停止する。

呼び出し元と呼び出し先

  • 呼び出し元: atexit(cleanup_objects_atexit) によりプログラム終了時に自動的に呼び出される。
  • 呼び出し先: connect_database, drop_publication, drop_replication_slot, disconnect_database, pg_log_warning, pg_log_warning_hint, stop_standby_server

関連データ構造

  • LogicalRepInfo: 各データベースのパブリケーションおよびスロット作成状況を保持する構造体。
  • dbinfos: LogicalRepInfo を管理するグローバルなリスト。

失敗モードと境界条件

  • パブリッシャーへの接続が確立できない場合、オブジェクトの自動削除は行われず、ユーザーに対して手動でのクリーンアップを求める警告が出力される。
  • 既にリカバリが終了している場合、物理レプリカの再構築が必要であるという警告が優先的に表示される。

保守担当者向けメモ

  • 本関数は atexit ハンドラであるため、実行環境の制約（シグナル安全性やメモリ確保の制限など）に注意すること。
  • 接続失敗時の警告メッセージは、ユーザーが手動で WAL ファイルの保持を防ぐために重要であるため、変更時は注意が必要。
