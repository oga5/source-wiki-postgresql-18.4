SpGistLastUsedPage

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist_private.h:99

目的

SP-GiSTインデックスにおいて、各バックエンドが最後に使用したページ情報をキャッシュし、メタページへのアクセス競合を低減するためのデータ構造です。

シグネチャまたは形状

typedef struct SpGistLastUsedPage
{
    BlockNumber blkno;      /* ブロック番号、または InvalidBlockNumber */
    int         freeSpace;  /* ページの空き容量（陳腐化の可能性あり） */
} SpGistLastUsedPage;

入力と出力

本構造体は単体で関数として機能するものではなく、SpGistLUPCache などの上位構造体の一部として保持されます。

前提条件と不変条件

  • blkno が InvalidBlockNumber の場合、そのキャッシュエントリは無効とみなされます。
  • freeSpace フィールドは、ページ更新後に即座に同期されない場合があるため、常に最新の空き容量を保証するものではありません。

アルゴリズムまたは振る舞い

SP-GiSTの挿入処理において、メタページを毎回参照する代わりに、このキャッシュを参照して適切な挿入先ページを特定します。これにより、メタページに対するロック競合を緩和します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/spgist/spgutils.c 内の関数が、キャッシュの読み取りおよび更新のために本構造体を参照しています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SpGistLUPCache: 本構造体の配列を保持し、バックエンドごとのキャッシュ状態を管理します。

失敗モードと境界条件

  • freeSpace の値が実際のページ状態と乖離している場合、挿入処理が一時的に非効率になる可能性がありますが、インデックスの整合性が損なわれることはありません。

保守担当者向けメモ

  • freeSpace はあくまでヒューリスティックな値であるため、利用時には必ずページの実態を確認するロジックを併用してください。
  • 同時実行制御下でのキャッシュの有効期限や更新タイミングについては、src/backend/access/spgist/spgutils.c の実装を参照してください。
