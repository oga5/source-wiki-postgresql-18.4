dropmsgstrings

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/tablecmds.c:245

目的

DROP コマンド実行時、対象オブジェクトの存在確認や削除処理に関連するエラーメッセージおよびヒントメッセージを保持するための構造体です。

シグネチャまたは形状

struct dropmsgstrings
{
	char		kind;
	int			nonexistent_code;
	const char *nonexistent_msg;
	const char *skipping_msg;
	const char *nota_msg;
	const char *drophint_msg;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは、特定のオブジェクト種別（kind）に対応するエラーコードやメッセージ文字列を格納します。

前提条件と不変条件

  • src/backend/commands/tablecmds.c:255 にて、この構造体の配列 dropmsgstringarray が静的に定義されています。
  • 各フィールドは、DROP コマンドの実行中に発生する可能性のあるエラー状態（オブジェクトが存在しない、型が異なる等）を報告するために使用されます。

アルゴリズムまたは振る舞い

本構造体自体はロジックを持ちませんが、DROP コマンド処理において、対象オブジェクトの検索結果に基づき、適切なエラーメッセージを選択するためのルックアップテーブルの一部として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/tablecmds.c 内の DROP 関連処理（1465 行目、1513 行目、1514 行目など）で、rentry や wentry といったポインタを通じて参照されます。
  • 呼び出し先: 情報なし。

関連データ構造

  • dropmsgstringarray (同ファイル内の静的配列)

失敗モードと境界条件

  • 構造体内の各メッセージポインタが NULL でないことが前提となります。
  • kind フィールドが、検索対象のオブジェクト種別と一致している必要があります。

保守担当者向けメモ

  • 本構造体は DROP コマンドのメッセージの一貫性を保つために使用されています。新しいオブジェクト種別に対して DROP 処理を追加する際は、dropmsgstringarray へのエントリ追加を検討してください。
