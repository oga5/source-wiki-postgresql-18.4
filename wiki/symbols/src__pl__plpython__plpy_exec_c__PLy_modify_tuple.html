PLy_modify_tuple

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_exec.c:919

目的

トリガー実行時に Python 辞書（TD["new"]）の内容を基に、PostgreSQL の行データ（HeapTuple）を更新する。

シグネチャまたは形状

HeapTuple PLy_modify_tuple(PLyProcedure *proc, PyObject *pltd, TriggerData *tdata, HeapTuple otup)

入力と出力

  • **入力**:
  • proc: 現在のプロシージャ情報。
  • pltd: トリガーデータを含む Python 辞書オブジェクト。
  • tdata: PostgreSQL の TriggerData 構造体。
  • otup: 更新対象の元の行データ。
  • **出力**:
  • 更新された新しい HeapTuple を返す。

前提条件と不変条件

  • pltd 内の "new" キーが辞書型として存在すること。
  • 更新対象の列がシステム属性や生成列（generated column）ではないこと。
  • proc->result が型変換のために適切に初期化されていること。

アルゴリズムまたは振る舞い

1. error_context_stack を設定し、エラー発生時のコンテキストを保持する。 2. PG_TRY ブロック内で以下を実行:

  • pltd から "new" 辞書を取得し、そのキー（列名）を列挙する。
  • 各キーに対応する列番号を SPI_fnumber で特定し、属性の妥当性を検証する。
  • PLy_output_convert を使用して Python オブジェクトを PostgreSQL の Datum に変換する。
  • heap_modify_tuple を呼び出し、変更を反映した新しいタプルを生成する。

    • 3. PG_CATCH ブロックで、割り当てたメモリ（modvalues 等）の解放と Python オブジェクトの参照カウント減少を行い、エラーを再送出する。 4. 正常終了時にメモリを解放し、元のエラーコンテキストを復元してタプルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_exec.c 内のトリガー実行ロジック（PLy_exec_trigger 等）から呼び出される。
  • **呼び出し先**: SPI_fnumber, heap_modify_tuple, PLy_output_convert, PyDict_GetItemString, PyDict_Keys 等の PostgreSQL SPI および Python C API 関数。

関連データ構造

  • PLyProcedure: プロシージャの実行状態と型変換情報を保持。
  • TriggerData: トリガー実行時のコンテキスト情報。
  • HeapTuple: PostgreSQL の行データ表現。

失敗モードと境界条件

  • "new" キーが辞書内に存在しない場合、または辞書型でない場合は ERROR を送出。
  • 存在しない列名やシステム属性、生成列へのアクセスを試みた場合は ERROR を送出。
  • Python インタプリタの整合性が失われていると判断された場合（plval が NULL）、FATAL で異常終了。

保守担当者向けメモ

  • PG_TRY/PG_CATCH ブロック内でのメモリ管理は、palloc0 で確保した領域を PG_CATCH 側でも確実に解放する設計となっている。
  • Py_INCREF/Py_DECREF による参照カウント管理が必須であり、修正時にはリークに注意すること。
