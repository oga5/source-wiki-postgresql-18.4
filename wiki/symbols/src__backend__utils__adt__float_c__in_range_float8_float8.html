in_range_float8_float8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/float.c:1026

目的

ウィンドウ関数のフレーム境界計算において、float8 型の値が指定された範囲内（base から offset の範囲内）にあるかを判定する。

シグネチャまたは形状

Datum in_range_float8_float8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • val (float8): 判定対象の値。
  • base (float8): 範囲の基準値。
  • offset (float8): 基準値からのオフセット量。
  • sub (bool): true なら base - offset、false なら base + offset を境界とする。
  • less (bool): true なら val <= 境界、false なら val >= 境界 を判定する。
  • **出力**:
  • bool: 判定結果を Datum として返す。

前提条件と不変条件

  • offset は負数であってはならず、NaN であってもならない（エラーとなる）。
  • NaN の比較は float8_cmp_internal に準拠し、NaN は非 NaN 値よりも大きいものとして扱われる。

アルゴリズムまたは振る舞い

1. **エラーチェック**: offset が負または NaN の場合、ERRCODE_INVALID_PRECEDING_OR_FOLLOWING_SIZE でエラーを報告する。 2. **NaN の処理**:

  • val が NaN の場合: base も NaN なら true、それ以外は !less の結果を返す。
  • base が NaN の場合: less の結果を返す。

    • 3. **無限大の特殊処理**: offset と base が共に無限大であり、かつ sub の条件によって計算結果が NaN になる場合（例: +inf - +inf）、すべての値が範囲内にあるとみなして true を返す。 4. **範囲判定**: base +/- offset を計算し、less フラグに応じて val との比較結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/float.c 内で定義。
  • numeric.c および timestamp.c のコメント内で、ウィンドウ関数の境界ロジックの参照先として言及されている。
  • **呼び出し先**:
  • isnan(), isinf() (標準数学ライブラリ)
  • ereport() (PostgreSQL エラー報告)
  • PG_GETARG_*, PG_RETURN_BOOL (PostgreSQL 関数マネージャマクロ)

関連データ構造

  • float8 (PostgreSQL の 8 バイト浮動小数点型)

失敗モードと境界条件

  • offset が負または NaN の場合、SQL レベルのエラーが発生する。
  • base や offset が無限大の場合、浮動小数点演算の仕様に従い、適切に符号付き無限大として比較される。

保守担当者向けメモ

  • 本関数はウィンドウ関数のフレーム境界定義に使用される。ロジックを変更する場合は、numeric.c や timestamp.c で行われている同様の境界判定との整合性に注意すること。
