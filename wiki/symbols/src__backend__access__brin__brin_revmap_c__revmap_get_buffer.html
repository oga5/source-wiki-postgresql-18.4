revmap_get_buffer

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/brin/brin_revmap.c:462

目的

指定されたヒープブロック番号に対応するBRINリバースレンジマップ（revmap）のページバッファを取得またはキャッシュから読み込みます。

シグネチャまたは形状

static Buffer revmap_get_buffer(BrinRevmap *revmap, BlockNumber heapBlk)

入力と出力

  • **入力**:
  • revmap: BrinRevmap 構造体へのポインタ。現在のバッファキャッシュ状態を保持。
  • heapBlk: 対象となるヒープのブロック番号。
  • **出力**:
  • Buffer: 対象のrevmapページを保持するバッファ。

前提条件と不変条件

  • revmap_get_blkno が有効なブロック番号を返すこと。
  • 取得した mapBlk がメタページ（BRIN_METAPAGE_BLKNO）ではなく、かつ revmap->rm_lastRevmapPage 以下であること。

アルゴリズムまたは振る舞い

1. revmap_get_blkno を呼び出し、ヒープブロック番号から対応する物理的なrevmapブロック番号（mapBlk）を算出する。 2. mapBlk が InvalidBlockNumber の場合はエラーを発生させる。 3. revmap->rm_currBuf が既に目的の mapBlk を保持しているか確認する。

  • 保持していない場合、現在の rm_currBuf が有効であれば ReleaseBuffer で解放する。
  • ReadBuffer を使用して新しい mapBlk を読み込み、rm_currBuf に格納する。

    • 4. 最後に rm_currBuf を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_revmap.c:138 (推測される直接の呼び出し)
  • **呼び出し先**:
  • revmap_get_blkno
  • BufferGetBlockNumber
  • ReleaseBuffer
  • ReadBuffer
  • elog

関連データ構造

  • BrinRevmap: revmapのアクセス状態とキャッシュされたバッファを管理する構造体。

失敗モードと境界条件

  • revmap_get_blkno が InvalidBlockNumber を返した場合、elog(ERROR) により処理が中断される。
  • mapBlk がメタページ番号と一致する場合、または rm_lastRevmapPage を超える場合は Assert により停止する。

保守担当者向けメモ

  • 本関数は BrinRevmap 構造体内のバッファキャッシュを更新する副作用を持つ。
  • キャッシュの排他制御については、呼び出し元が適切なロックを保持していることを前提としている。
