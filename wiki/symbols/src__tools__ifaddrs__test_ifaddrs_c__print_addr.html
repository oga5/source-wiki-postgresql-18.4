print_addr

定義

  • 種別: function
  • 定義箇所: src/tools/ifaddrs/test_ifaddrs.c:17

目的

ネットワークインターフェースの sockaddr 構造体を、人間が読める数値形式のIPアドレス文字列に変換して標準出力へ表示する。

シグネチャまたは形状

static void print_addr(struct sockaddr *addr)

入力と出力

  • **入力**: struct sockaddr *addr （変換対象のアドレス構造体へのポインタ）
  • **出力**: なし（標準出力への書き出しのみ）

前提条件と不変条件

  • addr は有効な sockaddr 構造体を指している必要がある。
  • getnameinfo が利用可能な環境であること。

アルゴリズムまたは振る舞い

1. addr->sa_family を確認し、アドレスファミリー（AF_INET または AF_INET6）に応じて適切な構造体サイズを決定する。 2. getnameinfo を NI_NUMERICHOST フラグ付きで呼び出し、バイナリ形式のアドレスを文字列に変換する。 3. 変換が成功した場合は文字列を標準出力に表示する。 4. 変換が失敗した場合は、[unknown: family %d] という形式でエラー情報を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/ifaddrs/test_ifaddrs.c 内の callback 関数（インターフェースアドレスおよびマスクの表示時に使用）。
  • **呼び出し先**: getnameinfo（標準ライブラリ関数）、printf（標準ライブラリ関数）。

関連データ構造

  • struct sockaddr: ソケットアドレスの汎用構造体。
  • struct sockaddr_in: IPv4用構造体。
  • struct sockaddr_in6: IPv6用構造体。
  • struct sockaddr_storage: アドレスファミリーが不明な場合に使用される汎用格納用構造体。

失敗モードと境界条件

  • getnameinfo が 0 以外の値を返した場合、変換失敗として [unknown: family %d] を出力する。
  • sa_family が AF_INET でも AF_INET6 でもない場合、sizeof(struct sockaddr_storage) をサイズとして使用する。

保守担当者向けメモ

  • 本関数は test_ifaddrs.c 専用のヘルパーであり、PostgreSQL バックエンド本体には含まれない。
  • getnameinfo を使用しているため、IPv4/IPv6 の両方に対応している。
