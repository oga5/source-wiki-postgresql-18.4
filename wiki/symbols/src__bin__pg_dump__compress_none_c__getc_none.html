getc_none

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_none.c:125

目的

非圧縮モードにおけるアーカイブファイルからの1文字読み込み処理を提供します。

シグネチャまたは形状

int getc_none(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH（非圧縮処理用のファイルハンドル構造体）。private_data メンバに FILE* が格納されていることを前提とします。
  • **出力**: 読み込んだ文字（int 型）。

前提条件と不変条件

  • CFH->private_data は有効な FILE* を指している必要があります。
  • 本関数は pg_dump の圧縮抽象化レイヤーの一部として呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. CFH->private_data から FILE* を取得します。 2. fgetc(fp) を実行してストリームから1文字読み込みます。 3. 読み込み結果が EOF の場合、feof(fp) を確認します。

  • feof が偽（エラー発生）の場合、pg_fatal を呼び出し致命的エラーとして終了します。
  • feof が真（終端到達）の場合、pg_fatal を呼び出しエラーとして終了します。

    • 4. 正常時は読み込んだ文字を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_none.c 内で、CFH->getc_func に関数ポインタとして代入されています。
  • **呼び出し先**: 標準ライブラリの fgetc および feof。エラー時には pg_fatal を呼び出します。

関連データ構造

  • CompressFileHandle: 非圧縮時のファイル操作関数ポインタを保持する構造体。

失敗モードと境界条件

  • ファイル読み込みエラーまたは予期せぬ EOF に遭遇した場合、pg_fatal によりプロセスが終了します。

保守担当者向けメモ

  • 本関数は FILE* を直接操作する単純なラッパーです。
  • エラーハンドリングは pg_fatal に依存しており、呼び出し側でエラーを回復することはできません。
