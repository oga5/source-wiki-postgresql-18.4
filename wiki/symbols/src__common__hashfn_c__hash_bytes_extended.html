hash_bytes_extended

定義

  • 種別: function
  • 定義箇所: src/common/hashfn.c:371

目的

可変長バイト列に対して、64bitのシード値を用いたハッシュ計算を行い、衝突の少ない uint64 型のハッシュ値を生成します。

シグネチャまたは形状

uint64 hash_bytes_extended(const unsigned char *k, int keylen, uint64 seed)

入力と出力

  • **入力**:
  • k: ハッシュ対象のバイト列へのポインタ。
  • keylen: バイト列の長さ。
  • seed: 初期状態を攪拌するための64bitシード値。
  • **出力**:
  • 計算された uint64 型のハッシュ値。

前提条件と不変条件

  • seed が 0 の場合、シードによる攪拌はスキップされます。
  • seed が 0 でない場合、内部状態 a, b, c の初期化後にシードを組み込み、mix マクロで攪拌します。
  • メモリのアライメント（UINT32_ALIGN_MASK）に応じて、ワード単位の高速アクセスまたはバイト単位のアクセスが選択されます。

アルゴリズムまたは振る舞い

1. **初期化**: 内部状態 a, b, c を定数と keylen で初期化します。 2. **シード適用**: seed が非ゼロの場合、上位32bitと下位32bitを a と b に加算し、mix マクロを適用します。 3. **データ処理**:

  • 12バイト単位で mix マクロを適用し、データを攪拌します。
  • 残りの11バイト以下のデータは、エンディアン（WORDS_BIGENDIAN）を考慮して a, b, c に加算します。

    • 4. **最終化**: 最終的なハッシュ値を算出します。

関連データ構造

  • uint32, uint64: PostgreSQL の標準的な固定長整数型。
  • UINT32_ALIGN_MASK: アライメントチェック用定数。

失敗モードと境界条件

  • keylen が 0 の場合、初期化された状態から最終化処理が行われます。
  • アライメントが不適切な場合、WORDS_BIGENDIAN の定義有無に応じたバイト単位の処理パスが実行されます。
  • 本関数はエラーを投げない設計です。

保守担当者向けメモ

  • mix マクロを変更する際は、ハッシュの分布特性（アバランチ効果）に十分注意してください。
  • 64bit ハッシュが必要な場合にのみ本関数を使用してください。32bit で十分な場合は hash_bytes を検討してください。
