SERIALIZABLEXACT

定義

  • 種別: struct
  • 定義箇所: src/include/storage/predicate_internals.h:58

目的

PostgreSQLのSSI（Serializable Snapshot Isolation）において、直列化可能トランザクションの状態を共有メモリ上で管理するための構造体です。トランザクション間の依存関係（読み書き競合）やコミット順序を追跡し、直列化失敗の判定を可能にします。

シグネチャまたは形状

typedef struct SERIALIZABLEXACT {
    VirtualTransactionId vxid;
    SerCommitSeqNo prepareSeqNo;
    SerCommitSeqNo commitSeqNo;
    union {
        SerCommitSeqNo earliestOutConflictCommit;
        SerCommitSeqNo lastCommitBeforeSnapshot;
    } SeqNo;
    dlist_head outConflicts;
    dlist_head inConflicts;
    dlist_head predicateLocks;
    dlist_node finishedLink;
    dlist_node xactLink;
    LWLock perXactPredicateListLock;
    dlist_head possibleUnsafeConflicts;
    TransactionId topXid;
    TransactionId finishedBefore;
    TransactionId xmin;
    uint32 flags;
    int pid;
    int pgprocno;
} SERIALIZABLEXACT;

入力と出力

  • 入力: トランザクション開始時に割り当てられ、実行中のプロセス情報（vxid, pid, pgprocno）やスナップショット情報（xmin）を保持します。
  • 出力: トランザクション終了時に finishedLink を通じて FinishedSerializableTransactions リストへ移動し、クリーンアップされます。

前提条件と不変条件

  • perXactPredicateListLock は、並列クエリ実行時に同一セッション内のワーカー間での predicateLocks リストへのアクセスを保護します。
  • flags（SXACT_FLAG_*）は、トランザクションのコミット可否や読み取り専用トランザクションの安全性判定に不可欠な状態を保持します。

アルゴリズムまたは振る舞い

1. **コミット順序の追跡**: prepareSeqNo と commitSeqNo を使用してコミット順序を記録します。厳密な順序が不明な場合は、安全側に倒して競合とみなす設計です。 2. **競合管理**: outConflicts（読み取り競合）および inConflicts（書き込み競合）リストにより、トランザクション間の依存関係をグラフとして保持します。 3. **ライフサイクル**: トランザクション開始時に生成され、終了時に ReleasePredXact を介して関連するロックや競合リストと共に解放・再利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/predicate.c 内の CreatePredXact 等で生成・管理されます。
  • 呼び出し先: RWConflictExists, SetRWConflict, FlagSxactUnsafe などの関数が本構造体を引数として操作します。

関連データ構造

  • PREDICATELOCK: predicateLocks リストに格納される述語ロック。
  • RWConflictData: outConflicts / inConflicts リストを構成する競合情報。
  • SerCommitSeqNo: コミット順序を追跡する単調増加シーケンス番号。

失敗モードと境界条件

  • finishedBefore が invalid の場合はトランザクションが実行中であることを示します。
  • 共有メモリ上の構造体であるため、アクセスには適切なロック（perXactPredicateListLock 等）が必要です。
  • LOCALPREDICATELOCK（ローカルキャッシュ）との同期不整合が発生する可能性がある点に注意してください。

保守担当者向けメモ

  • PREDICATELOCKTARGETTAG の変更時は、ハッシュ計算時のパディングやアライメントに注意が必要です。
  • flags フィールドはSSIの挙動を左右するため、変更時は関連するフラグ定義との整合性を確認してください。
