scram_init

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:96

目的

SCRAM-SHA-256 認証セッションを開始するための状態構造体（fe_scram_state）を初期化し、パスワードの正規化を行う。

シグネチャまたは形状

static void *scram_init(PGconn *conn, const char *password, const char *sasl_mechanism)

入力と出力

  • **入力**:
  • conn: PostgreSQL 接続オブジェクト。
  • password: 認証に使用する平文パスワード。
  • sasl_mechanism: 使用する SASL メカニズム名。
  • **出力**:
  • 成功時: 初期化された fe_scram_state 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • sasl_mechanism は NULL であってはならない。
  • 内部状態の state は FE_SCRAM_INIT に設定される。
  • ハッシュアルゴリズムとして PG_SHA256 が固定的に割り当てられる。

アルゴリズムまたは振る舞い

1. fe_scram_state 構造体をメモリ確保し、ゼロクリアする。 2. 接続情報、初期ステート、ハッシュタイプを構造体に格納する。 3. sasl_mechanism を複製して保持する。 4. password が提供されている場合、pg_saslprep を用いて正規化を試みる。

  • 正規化が成功した場合はその結果を、失敗した場合は元のパスワードを複製して保持する。
  • メモリ確保に失敗した場合は、確保済みのリソースを解放して NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-scram.c 内で参照されているが、具体的な呼び出しフローの詳細は提供されていない。
  • **呼び出し先**: malloc, memset, strdup, free, pg_saslprep。

関連データ構造

  • fe_scram_state: 認証セッション中の状態を保持する構造体。
  • PGconn: PostgreSQL 接続オブジェクト。

失敗モードと境界条件

  • メモリ不足（malloc や strdup の失敗）が発生した場合、NULL を返す。
  • pg_saslprep が SASLPREP_OOM を返した場合、エラーとして処理される。
  • パスワードが NULL の場合、正規化処理はスキップされる。

保守担当者向けメモ

  • 本関数は libpq の認証サブシステムの一部であり、fe-auth.c から SASL 認証フローの初期化として利用される。
  • password の正規化において、pg_saslprep が失敗した場合はフォールバックとして元のパスワードをそのまま使用する設計となっている。
