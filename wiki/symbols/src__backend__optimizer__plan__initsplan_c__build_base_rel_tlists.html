build_base_rel_tlists

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/initsplan.c:246

目的

クエリの最終的なターゲットリストおよび HAVING 句で使用される変数を抽出し、それらがどのベースリレーションで必要かをプランナの内部構造に登録します。

シグネチャまたは形状

void build_base_rel_tlists(PlannerInfo *root, List *final_tlist)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報を保持する PlannerInfo 構造体。
  • final_tlist: クエリの最終的なターゲットリスト。
  • **出力**: なし（副作用として root 内の各リレーションの属性必要情報を更新）。

前提条件と不変条件

  • root->parse が有効な Query 構造体を指していること。
  • final_tlist が TargetEntry ノードのリストであること。

アルゴリズムまたは振る舞い

1. final_tlist から pull_var_clause を使用して、集約関数、ウィンドウ関数、プレースホルダ変数を含むすべての変数リストを抽出します。 2. 抽出された変数が存在する場合、add_vars_to_targetlist を呼び出し、各変数がどのリレーションで必要かを root に記録します。 3. クエリに HAVING 句が存在する場合、同様に havingQual から変数を抽出し、add_vars_to_targetlist を介してリレーションの必要属性リストへ追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planmain.c: root->processed_tlist を引数として呼び出し。
  • src/backend/optimizer/util/appendinfo.c: 属性追加後の再計算時に呼び出し。
  • **呼び出し先**:
  • pull_var_clause: 変数抽出に使用。
  • add_vars_to_targetlist: 抽出した変数をリレーションの必要属性として登録。
  • bms_make_singleton: ビットマップセットの生成に使用。
  • list_free: 抽出した一時リストの解放。

関連データ構造

  • PlannerInfo: プランナのメイン状態管理構造体。
  • Query: 解析済みのクエリツリー。
  • TargetEntry: ターゲットリストの各要素。

失敗モードと境界条件

  • final_tlist が空の場合、変数の抽出は行われません。
  • HAVING 句が存在しない場合、その処理はスキップされます。
  • 変数が含まれない場合、add_vars_to_targetlist は呼び出されません。

保守担当者向けメモ

  • 本関数は、プランナがパス探索を開始する前に、必要な変数が確実に各リレーションの attr_needed に反映されることを保証します。
  • HAVING 句の処理において、Aggref は許可されますが WindowFunc は含まれないという制約があります。
