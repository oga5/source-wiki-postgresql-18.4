CopyGetAttnums

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copy.c:984

目的

COPY コマンドの対象となるテーブルの属性番号（attnum）リストを生成・検証します。

シグネチャまたは形状

List *CopyGetAttnums(TupleDesc tupDesc, Relation rel, List *attnamelist)

入力と出力

  • **入力**:
  • tupDesc: 対象テーブルのタプル記述子。
  • rel: 対象のリレーション（エラー報告用）。
  • attnamelist: ユーザーが指定した列名のリスト（NIL の場合は全列が対象）。
  • **出力**:
  • 処理対象となる属性番号の整数リスト（List *）。

前提条件と不変条件

  • attnamelist が NIL の場合、削除済み列（attisdropped）および生成列（attgenerated）を除外したデフォルトの列リストが生成されます。
  • attnamelist が指定されている場合、各列名がリレーション内に存在し、かつ生成列でないことが検証されます。

アルゴリズムまたは振る舞い

1. **デフォルト生成**: attnamelist が NIL の場合、tupDesc を走査し、有効な列の属性番号を順次リストに追加します。 2. **ユーザー指定リストの検証**:

  • 指定された各列名をリレーションの属性から検索します。
  • 削除済み列はスキップされます。
  • 生成列が指定された場合は ERRCODE_INVALID_COLUMN_REFERENCE でエラーを報告します。
  • 列が見つからない場合は ERRCODE_UNDEFINED_COLUMN でエラーを報告します。
  • 重複する列名が指定された場合は ERRCODE_DUPLICATE_COLUMN でエラーを報告します。

    • 3. **結果の返却**: 検証済みの属性番号リストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copy.c (メイン処理)
  • src/backend/commands/copyfrom.c (COPY FROM 関連処理)
  • src/backend/commands/copyto.c (COPY TO 関連処理)
  • **呼び出し先**:
  • ereport (エラー報告)
  • lappend_int, list_member_int (リスト操作)
  • TupleDescCompactAttr, TupleDescAttr (属性アクセス)
  • namestrcmp (名前比較)

関連データ構造

  • TupleDesc: タプルの構造定義。
  • Relation: テーブルのリレーション情報。
  • List: 属性番号を保持する PostgreSQL のリスト構造。

失敗モードと境界条件

  • **列未検出**: 指定された列名がテーブルに存在しない場合、エラーを発生させます。
  • **生成列の指定**: COPY 操作で生成列を明示的に指定した場合、エラーを発生させます。
  • **重複指定**: 同じ列を複数回指定した場合、エラーを発生させます。

保守担当者向けメモ

  • 本関数は COPY コマンドの列選択ロジックの要です。生成列の扱いに関するルール変更や、新しい属性タイプへの対応を行う際は、この検証ロジックが整合しているか確認してください。
