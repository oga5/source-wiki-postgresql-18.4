HashScanPosData

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash.h:109

目的

ハッシュインデックスのスキャン中における、現在のページ位置およびそのページ内のタプル情報を保持するための構造体です。

シグネチャまたは形状

typedef struct HashScanPosData
{
    Buffer      buf;
    BlockNumber currPage;
    BlockNumber nextPage;
    BlockNumber prevPage;
    int         firstItem;
    int         lastItem;
    int         itemIndex;
    HashScanPosItem items[MaxIndexTuplesPerPage];
} HashScanPosData;

入力と出力

本構造体は関数への引数や戻り値として直接渡されるのではなく、スキャン状態を管理する HashScanOpaqueData の一部として保持されます。

前提条件と不変条件

  • items 配列は常にインデックス順（indexoffset の昇順）で格納されます。
  • items 配列の末尾に配置される必要があるため、構造体の定義順序は厳守されます。
  • buf が有効な場合、そのバッファはピン留めされている必要があります。

アルゴリズムまたは振る舞い

  • **スキャン方向**: 前方スキャン時は items を順次参照し、後方スキャン時は items 配列の末尾から前方へ向かって充填・参照を行います。
  • **カーソル管理**: itemIndex は、呼び出し元に最後に返されたエントリを示すカーソルとして機能します。
  • **ページ遷移**: currPage を中心に、nextPage（オーバーフローページ）や prevPage（オーバーフローまたはバケットページ）を追跡し、インデックスの物理的な走査を制御します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/access/hash.h 内で HashScanOpaqueData のメンバ currPos として参照されています。
  • 呼び出し先: 具体的な関数レベルの呼び出し関係は、提供されたソース情報からは特定できません。

関連データ構造

  • HashScanOpaqueData: 本構造体を保持し、スキャン全体の状態を管理する親構造体。
  • HashScanPosItem: items 配列の各要素を構成するデータ構造。
  • Buffer: storage/bufmgr.h で定義されるバッファ管理用型。

失敗モードと境界条件

  • MaxIndexTuplesPerPage を超えるタプルは保持できません。
  • buf が無効な状態でのアクセスは未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は HashScanOpaqueData の一部として使用されるため、メモリレイアウトの変更には注意が必要です。
  • 後方スキャン時の items 配列の充填ロジックは、firstItem と lastItem の管理に依存しているため、変更時はこのインデックス制御ロジックの整合性を確認してください。
