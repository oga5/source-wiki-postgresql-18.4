Gzip_write

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:297

目的

CompressFileHandle を介して、GZIP 圧縮されたファイルストリームへデータを書き込みます。

シグネチャまたは形状

static void Gzip_write(const void *ptr, size_t size, CompressFileHandle *CFH)

入力と出力

  • ptr: 書き込むデータへのポインタ。
  • size: 書き込むデータのバイト数。
  • CFH: 圧縮ファイルハンドル。内部の private_data に gzFile を保持している必要があります。
  • 出力: なし（成功時）。失敗時は pg_fatal によりプロセスを終了します。

前提条件と不変条件

  • CFH->private_data は有効な gzFile ハンドルとしてキャスト可能であること。
  • zlib ライブラリが利用可能であること。

アルゴリズムまたは振る舞い

1. CFH->private_data から gzFile ハンドルを取得します。 2. gzwrite を呼び出し、指定されたサイズ分データを圧縮して書き出します。 3. 書き込み結果が要求サイズと一致しない場合、gzerror を使用してエラー詳細を取得します。 4. エラーが Z_ERRNO の場合は strerror(errno) を、それ以外の場合は zlib のエラーメッセージを用いて pg_fatal を呼び出し、プログラムを異常終了させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/compress_gzip.c 内の InitCompressFileHandleGzip において、CFH->write_func に関数ポインタとして代入されます。
  • 呼び出し先: zlib の gzwrite および gzerror を呼び出します。

関連データ構造

  • CompressFileHandle: 圧縮 I/O の抽象化インターフェース。
  • gzFile: zlib が提供する GZIP ファイル操作用ハンドル。

失敗モードと境界条件

  • 書き込みバイト数が要求サイズと異なる場合、致命的エラーとして処理されます。
  • zlib 内部でエラーが発生した場合、適切なエラーメッセージと共に pg_fatal が実行されます。

保守担当者向けメモ

  • 本関数は CompressFileHandle の関数ポインタ経由で呼び出されることを前提としています。
  • gzwrite の戻り値が size と一致しない場合のみをエラーと判定するシンプルな実装です。
