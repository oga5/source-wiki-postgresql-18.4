SPI_connect

定義

  • 種別: function
  • 定義箇所: src/backend/executor/spi.c:93

目的

SPI（Server Programming Interface）セッションを開始し、SQL クエリ実行のための実行コンテキストをスタックに構築します。

シグネチャまたは形状

int SPI_connect(void)

入力と出力

  • 入力: なし
  • 出力: int 型のステータスコード（成功時は SPI_OK_CONNECT を返すと想定されるが、実装上は SPI_connect_ext(0) の戻り値に依存）

前提条件と不変条件

  • 本関数は SPI_connect_ext(0) を呼び出すラッパーです。
  • 呼び出しにより、内部スタック _SPI_stack に新しい実行コンテキストがプッシュされます。

アルゴリズムまたは振る舞い

1. 引数 0 を指定して SPI_connect_ext を呼び出します。 2. SPI_connect_ext 内で、新しいメモリコンテキスト（procCxt, execCxt）が作成され、SPI 呼び出しのネストを管理する内部スタックに保存されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/dblink/dblink.c
  • contrib/spi/refint.c
  • contrib/tablefunc/tablefunc.c
  • contrib/xml2/xpath.c
  • src/backend/commands/matview.c
  • src/backend/utils/adt/ri_triggers.c
  • 呼び出し先: SPI_connect_ext

関連データ構造

  • _SPI_stack: SPI 呼び出しのネストを管理する内部スタック。
  • MemoryContext: 実行コンテキスト管理に使用されるメモリ領域。

失敗モードと境界条件

  • 内部スタックのオーバーフローやメモリ不足が発生した場合、SPI_connect_ext の仕様に従いエラーがスローされる可能性があります。

保守担当者向けメモ

  • 本関数は SPI_connect_ext(0) の単なるエイリアスです。SPI セッションの接続オプションを変更する必要がある場合は SPI_connect_ext を直接参照してください。
  • SPI_finish と対で使用し、リソースのリークを防ぐ必要があります。
