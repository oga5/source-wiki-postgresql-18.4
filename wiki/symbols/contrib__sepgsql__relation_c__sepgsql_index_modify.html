sepgsql_index_modify

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/relation.c:761

目的

インデックスに対する操作を、親テーブルに対する db_table:{setattr} 権限チェックへ転送する内部関数です。SELinuxにおいてインデックスは独立したオブジェクトではなく、親テーブルの属性の一部として扱われるため、この関数が仲介役となります。

シグネチャまたは形状

static void sepgsql_index_modify(Oid indexOid)
  • indexOid: 操作対象となるインデックスの OID。

入力と出力

  • **入力**: indexOid (Oid型)
  • **出力**: なし（戻り値なし）。権限チェックに失敗した場合は例外が発生します。

前提条件と不変条件

  • インデックスは親テーブルの属性の一部として扱われるため、呼び出し元はインデックスに対する操作を親テーブルの属性変更として検証する必要があります。
  • pg_index システムカタログへのアクセス権が必要です。

アルゴリズムまたは振る舞い

1. pg_index システムカタログを AccessShareLock モードで開きます。 2. sepgsql_relation_setattr_extra を呼び出し、以下の情報を渡して権限チェックを実行します。

  • カタログリレーション情報
  • インデックスの OID を特定するためのインデックス ID (IndexRelidIndexId)
  • 対象の indexOid
  • pg_index の属性番号 (Anum_pg_index_indrelid, Anum_pg_index_indexrelid)

    • 3. 最後に table_close を呼び出し、カタログを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/relation.c 内の各所（インデックスに関連するリレーション操作フックから呼び出されます）。
  • **呼び出し先**:
  • table_open / table_close (PostgreSQL アクセス層)
  • sepgsql_relation_setattr_extra (同ファイル内の権限チェック関数)

関連データ構造

  • pg_index (システムカタログ)
  • IndexRelationId (カタログの OID)
  • IndexRelidIndexId (インデックス検索用インデックスの OID)

失敗モードと境界条件

  • table_open が失敗した場合、PostgreSQL の標準的なエラー処理によりトランザクションが中断されます。
  • sepgsql_relation_setattr_extra 内で SELinux の権限チェック（db_table:{setattr}）が拒否された場合、エラーが発生します。

保守担当者向けメモ

  • 本関数はインデックス操作を親テーブルの権限チェックにリダイレクトする役割に特化しています。インデックス固有のセキュリティラベル管理は存在しないため、ロジック変更時は親テーブルの setattr 権限要件との整合性に注意してください。
