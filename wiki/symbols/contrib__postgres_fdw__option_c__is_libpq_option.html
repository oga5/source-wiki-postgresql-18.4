is_libpq_option

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/option.c:401

目的

指定されたキーワードが、postgres_fdw で管理されているオプションのうち、libpq に渡すべき接続パラメータであるかを判定する。

シグネチャまたは形状

static bool is_libpq_option(const char *keyword)

入力と出力

  • **入力**: keyword (判定対象のオプション名文字列)
  • **出力**: true (対象が libpq オプションである場合)、false (それ以外の場合)

前提条件と不変条件

  • 静的変数 postgres_fdw_options が初期化済みであること（Assert により検証）。
  • postgres_fdw_options は PgFdwOption 構造体の配列であり、末尾は keyword が NULL の要素で終端されていること。

アルゴリズムまたは振る舞い

1. 初期化済みの postgres_fdw_options 配列を先頭から順に走査する。 2. 各要素について、is_libpq_opt フラグが真であり、かつ keyword が引数と一致するかを比較する。 3. 一致する要素が見つかった場合は直ちに true を返し、配列の終端まで見つからなかった場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/option.c 内の他の関数から参照されている（直接的な呼び出し箇所として option.c:437 が確認されている）。
  • **呼び出し先**: 標準 C ライブラリの strcmp を使用。

関連データ構造

  • PgFdwOption: オプション名（keyword）と libpq 依存フラグ（is_libpq_opt）を保持する構造体。

失敗モードと境界条件

  • postgres_fdw_options が初期化されていない状態で呼び出された場合、Assert により異常終了する。
  • 引数 keyword が NULL の場合の挙動は未定義（strcmp に依存）。

保守担当者向けメモ

  • 本関数は postgres_fdw_options 配列の線形探索を行うため、オプション数が増大した場合は性能に影響する可能性があるが、現状のオプション数では許容範囲内である。
  • 新しい libpq オプションを追加する場合は、postgres_fdw_options 配列の定義を更新するだけで本関数は自動的に対応する。
