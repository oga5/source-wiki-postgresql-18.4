pg_be_sasl_mech

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/sasl.h:45

目的

PostgreSQLバックエンドにおけるSASL認証メカニズムを抽象化し、SCRAM等の具体的な認証方式をプラグインとして実装するためのインターフェースを定義します。

シグネチャまたは形状

typedef struct pg_be_sasl_mech
{
    void (*get_mechanisms) (Port *port, StringInfo buf);
    void *(*init) (Port *port, const char *mech, const char *shadow_pass);
    int (*exchange) (void *state, const char *input, int inputlen, char **output, int *outputlen, const char **logdetail);
    int max_message_length;
} pg_be_sasl_mech;

入力と出力

  • get_mechanisms: Port を入力とし、StringInfo バッファにサポートするメカニズム名を追記します。
  • init: Port、メカニズム名、パスワード情報を入力とし、認証セッション用の不透明な状態（void *）を返します。
  • exchange: 状態、クライアントからの入力データ（input）、およびその長さ（inputlen）を入力とし、サーバー側の応答（output）、その長さ（outputlen）、およびログ詳細（logdetail）を出力します。

前提条件と不変条件

  • exchange の input はヌル終端されていますが、SASL仕様に基づき埋め込みNULLを含む可能性があるため、必ず inputlen を参照する必要があります。
  • shadow_pass が NULL の場合でも、ユーザー列挙攻撃を防ぐため、認証失敗時と同様の挙動を維持しなければなりません。
  • init が返す状態ポインタは、他のコールバックの第一引数としてそのまま渡されます。

アルゴリズムまたは振る舞い

1. **初期化**: init によりメカニズム固有の状態が生成されます。 2. **交換**: exchange が呼び出され、戻り値が PG_SASL_EXCHANGE_CONTINUE である限り、クライアントとのメッセージ交換を繰り返します。 3. **終了**: PG_SASL_EXCHANGE_SUCCESS または FAILURE が返されると認証プロセスが完了します。

呼び出し元と呼び出し先

  • **呼び出し元**: CheckSASLAuth 関数（src/backend/libpq/auth-sasl.c）が、認証フローの制御において本構造体の各コールバックを呼び出します。
  • **呼び出し先**: 具体的な実装として pg_be_scram_mech（src/backend/libpq/auth-scram.c）や pg_be_oauth_mech（src/backend/libpq/auth-oauth.c）が定義されています。

関連データ構造

  • Port: 接続状態を保持する構造体。
  • StringInfo: 認証メッセージのバッファ操作に使用。

失敗モードと境界条件

  • exchange において、認証が失敗した場合は PG_SASL_EXCHANGE_FAILURE を返します。
  • max_message_length を超えるクライアントからのメッセージは制限されます。
  • inputlen が -1 の場合、クライアントからの初期応答がないことを示します。

保守担当者向けメモ

  • 本構造体はインターフェース定義のみであり、具体的な認証ロジックは含まれていません。
  • output バッファは palloc で確保する必要があります。交換が終了し、送信すべきデータがない場合は NULL を設定してください。
