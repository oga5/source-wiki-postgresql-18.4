function_parse_error_transpose

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_proc.c:1022

目的

関数定義内の構文エラー位置を、実行中のクエリ文字列（CREATE FUNCTION 文など）上の対応する位置へ変換し、エラー報告の可読性を向上させる。

シグネチャまたは形状

bool function_parse_error_transpose(const char *prosrc);

入力と出力

  • **入力**: prosrc (関数本体のソースコード文字列)
  • **出力**: 変換に成功した場合は true、変換対象のエラー位置情報が存在しない、または変換不能な場合は false を返す。

前提条件と不変条件

  • 現在の実行コンテキストでエラーが発生していること。
  • geterrposition() または getinternalerrposition() が有効なエラー位置（正の値）を保持していること。

アルゴリズムまたは振る舞い

1. 現在のエラー位置（origerrposition）を取得する。いずれも取得できない場合は false を返す。 2. ActivePortal が存在し、かつアクティブな状態であれば、その sourceText を取得する。 3. match_prosrc_to_query を呼び出し、関数本体のソースコードが元のクエリ内のどこに位置するかを特定し、新しいエラー位置（newerrposition）を算出する。 4. newerrposition が有効な場合、errposition() でエラー位置を更新し、内部エラー情報をクリアする。 5. 変換に失敗した場合は、エラー位置を内部エラーとして再設定し、prosrc を内部クエリとして報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_proc.c:1004
  • src/pl/plpgsql/src/pl_comp.c:896
  • **呼び出し先**:
  • geterrposition(), getinternalerrposition()
  • match_prosrc_to_query()
  • errposition(), internalerrposition(), internalerrquery()

関連データ構造

  • ActivePortal (グローバル変数)

失敗モードと境界条件

  • ActivePortal が存在しない場合（論理レプリケーションワーカーなど）、変換は行われず、エラー位置は内部エラーとして扱われる。
  • match_prosrc_to_query が位置を特定できない場合、元のエラー位置がそのまま内部エラーとして報告される。

保守担当者向けメモ

  • 本関数はエラー報告のコンテキストを調整するハック的な実装を含む。
  • 変換処理は ActivePortal に依存しているため、非対話的なバックグラウンド処理での挙動に注意すること。
