stats_check_required_arg

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/stat_utils.c:39

目的

統計情報の更新関数に渡された特定の引数が NULL でないことを検証し、必須引数の欠落による不正な処理を防ぎます。

シグネチャまたは形状

void stats_check_required_arg(FunctionCallInfo fcinfo, struct StatsArgInfo *arginfo, int argnum)

入力と出力

  • fcinfo: 呼び出し元の関数情報（FunctionCallInfo）。
  • arginfo: 引数のメタデータを含む構造体配列。
  • argnum: 検証対象となる引数のインデックス。
  • 戻り値: なし（検証成功時）。検証失敗時はエラーをスローして終了します。

前提条件と不変条件

  • argnum は arginfo 配列の有効なインデックスである必要があります。
  • fcinfo は有効な関数呼び出しコンテキストを指している必要があります。

アルゴリズムまたは振る舞い

1. PG_ARGISNULL(argnum) を使用して、指定されたインデックスの引数が NULL かどうかを判定します。 2. 引数が NULL の場合、ereport(ERROR, ...) を呼び出し、ERRCODE_INVALID_PARAMETER_VALUE エラーを発生させます。 3. エラーメッセージには、arginfo[argnum].argname から取得した引数名が含まれます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/attribute_stats.c (複数箇所)
  • src/backend/statistics/relation_stats.c (複数箇所)
  • **呼び出し先**:
  • PG_ARGISNULL (マクロ)
  • ereport (マクロ)

関連データ構造

  • StatsArgInfo: 引数の名前や属性情報を保持する構造体。
  • FunctionCallInfo: PostgreSQLの関数呼び出し情報を保持する構造体。

失敗モードと境界条件

  • 指定された引数が NULL の場合、即座にトランザクションを中断させるエラーが発生します。
  • argnum が不正な値である場合の動作については、本ソースコード内では明示的なガードは行われていません。

保守担当者向けメモ

  • 本関数は統計情報のバリデーション層において、必須引数の存在確認を行うための標準的な手段です。
  • エラーメッセージの形式を統一するため、新規の統計関数を追加する際も本関数を再利用してください。
