hash_any

定義

  • 種別: function
  • 定義箇所: src/include/common/hashfn.h:30

目的

任意のバイト列に対して標準的なハッシュ計算を行い、その結果を Datum 型として返します。

シグネチャまたは形状

static inline Datum hash_any(const unsigned char *k, int keylen)

入力と出力

  • **入力**:
  • k: ハッシュ対象のバイト列へのポインタ。
  • keylen: バイト列の長さ。
  • **出力**:
  • Datum: 計算されたハッシュ値を保持する Datum 型の値。

前提条件と不変条件

  • 入力されたバイト列 k は keylen 分のメモリ領域として有効である必要があります。
  • 返されるハッシュ値は、ハッシュテーブルのインデックス計算に適した、32ビット範囲で均一に分布する値であることが期待されます。

アルゴリズムまたは振る舞い

1. 内部で hash_bytes(k, keylen) を呼び出し、バイト列に対するハッシュ計算を実行します。 2. 計算された uint32 型のハッシュ値を UInt32GetDatum マクロを介して Datum 型に変換し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: hash_bytes（src/common/hashfn.c で定義）
  • **呼び出し元**:
  • contrib/citext/citext.c
  • contrib/hstore/hstore_op.c
  • contrib/ltree/ltree_op.c
  • contrib/sepgsql/uavc.c
  • src/backend/access/hash/hashfunc.c

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • uint32: ハッシュ計算の内部で使用される符号なし32ビット整数。

失敗モードと境界条件

  • keylen が負の値である場合の挙動は、内部の hash_bytes の実装に依存します。
  • k が NULL ポインタである場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスを重視する箇所での利用に適しています。
  • ハッシュ衝突攻撃への耐性が必要な場合は、シード値を指定可能な hash_any_extended の使用を検討してください。
  • 実装の詳細は src/common/hashfn.c にある hash_bytes を参照してください。
