be_lo_create

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-fsstubs.c:261

目的

SQLインターフェースを介して、指定されたOIDまたはシステムが割り当てたOIDで新しいラージオブジェクト（Large Object）を作成します。

シグネチャまたは形状

Datum be_lo_create(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る Oid（PG_GETARG_OID(0)）。これが0の場合、システムが新しいOIDを生成します。
  • **出力**: 作成されたラージオブジェクトの Oid を Datum 型として返します。

前提条件と不変条件

  • 読み取り専用トランザクション内では実行できません（PreventCommandIfReadOnly によるチェック）。
  • 実行後、lo_cleanup_needed フラグが true に設定され、トランザクション終了時のクリーンアップ対象となります。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID(0) で引数のOIDを取得します。 2. PreventCommandIfReadOnly を呼び出し、読み取り専用モードでないことを確認します。 3. lo_cleanup_needed を true に設定します。 4. inv_create(lobjId) を呼び出し、物理的なラージオブジェクト作成処理を実行します。 5. 作成されたOIDを PG_RETURN_OID で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_ARGS を使用しているため）。具体的なSQL文や関数名はソースコード上では明示されていません。
  • **呼び出し先**: PreventCommandIfReadOnly、inv_create（src/backend/storage/large_object/inv_api.c に定義）。

関連データ構造

  • Oid: ラージオブジェクトを一意に識別する型。
  • lo_cleanup_needed: be-fsstubs.c 内のグローバル変数。トランザクション終了時のクリーンアップ要否を管理します。

失敗モードと境界条件

  • 読み取り専用トランザクションで呼び出された場合、エラーが発生します。
  • inv_create 内部でのデータベース制約違反や権限不足により失敗する可能性があります。

保守担当者向けメモ

  • 本関数はSQLから直接呼び出されるラッパーです。ロジックの本体は inv_create に依存しているため、ラージオブジェクトのメタデータ管理に変更がある場合は inv_api.c を確認してください。
