pgtypes_fmt_replace

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/common.c:29

目的

多様なデータ型を文字列に変換し、指定されたバッファへ安全に書き込むための汎用的な変換関数です。

シグネチャまたは形状

int pgtypes_fmt_replace(union un_fmt_comb replace_val, int replace_type, char **output, int *pstr_len)

入力と出力

  • **入力**:
  • replace_val: 変換対象の値を保持する共用体。
  • replace_type: 変換対象の型を示す識別子（PGTYPES_TYPE_*）。
  • output: 書き込み先バッファのポインタへのポインタ。
  • pstr_len: 書き込み可能な残りバイト数へのポインタ。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は -1 またはメモリ確保失敗時に ENOMEM を返します。
  • output と pstr_len は、書き込み量に応じて更新されます。

前提条件と不変条件

  • pstr_len は常に書き込み可能な残りのバッファサイズを正確に保持している必要があります。
  • 数値型変換時には pgtypes_alloc を使用して一時バッファを確保し、処理後に必ず解放します。
  • PGTYPES_TYPE_STRING_MALLOCED が指定された場合、入力された文字列ポインタは関数内で free されます。

アルゴリズムまたは振る舞い

1. replace_type に基づき分岐します。 2. 文字列型の場合、長さを確認し、バッファに収まる場合に memcpy でコピーします。 3. 文字型の場合、1文字と終端文字を書き込みます。 4. 数値型の場合:

  • pgtypes_alloc で一時バッファを確保します。
  • snprintf を用いて型に応じた書式で文字列化します。
  • バッファの残容量を確認し、十分な場合に strcpy でコピーします。
  • 一時バッファを解放します。

    • 5. バッファ不足や変換エラー時は -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/pgtypeslib/timestamp.c での呼び出しが確認されています。
  • **呼び出し先**: pgtypes_alloc（メモリ確保）、free（メモリ解放）、snprintf、memcpy、strcpy、strlen を呼び出します。

関連データ構造

  • union un_fmt_comb: 変換対象の値を保持する共用体。

失敗モードと境界条件

  • バッファサイズ不足（pstr_len が不足）: -1 を返します。
  • メモリ確保失敗: ENOMEM を返します。
  • snprintf による変換失敗: -1 を返します。

保守担当者向けメモ

  • 新しい型を追加する際は、必ず pstr_len の減算とバッファオーバーフローチェックを実装してください。
  • Windows環境でのメモリ解放の整合性を保つため、free の使用箇所には注意を払ってください。
