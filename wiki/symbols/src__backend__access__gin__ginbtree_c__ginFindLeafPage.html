ginFindLeafPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbtree.c:82

目的

GINインデックスのルートからリーフページまで降下し、対象のキーを格納すべきリーフページを特定して、そのパスをスタック構造として返します。

シグネチャまたは形状

GinBtreeStack *ginFindLeafPage(GinBtree btree, bool searchMode, bool rootConflictCheck)

入力と出力

  • **入力**:
  • btree: GIN B-tree操作用のコンテキスト構造体。
  • searchMode: 検索モード（trueならパスを保持せず、falseなら更新用にスタックを構築）。
  • rootConflictCheck: ルートページでシリアライザブル競合チェックを行うかどうかのフラグ。
  • **出力**:
  • 成功時: リーフページへのパスを保持する GinBtreeStack 構造体へのポインタ。

前提条件と不変条件

  • btree 引数は有効なインデックス情報を含んでいる必要があります。
  • searchMode が false（更新モード）の場合、処理中に遭遇した不完全な分割（GIN_INCOMPLETE_SPLIT）は、ginFinishOldSplit によって修復されます。
  • ページ移動時には ginTraverseLock によるロックカップリングが適用されます。

アルゴリズムまたは振る舞い

1. ルートページから開始し、GinBtreeStack を初期化します。 2. rootConflictCheck が有効な場合、CheckForSerializableConflictIn を実行します。 3. ループ内で以下を繰り返します：

  • ginTraverseLock でページをロックします。
  • 更新モードの場合、GinPageIsIncompleteSplit を確認し、必要に応じて ginFinishOldSplit で修復します。
  • btree->isMoveRight を用いて、必要に応じて右兄弟ページへ移動します（ginStepRight を使用）。
  • リーフページに到達した場合は、そのスタックを返します。
  • 非リーフページの場合、btree->findChildPage で子ページを特定し、searchMode に応じてスタックを更新（または再利用）して降下を続けます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c
  • src/backend/access/gin/ginget.c
  • src/backend/access/gin/gininsert.c
  • **呼び出し先**:
  • ginTraverseLock, ginFinishOldSplit, ginStepRight, ReadBuffer, ReleaseAndReadBuffer, LockBuffer, BufferGetPage, CheckForSerializableConflictIn, palloc

関連データ構造

  • GinBtreeStack: ツリー降下時のパス情報（ブロック番号、バッファ、親へのポインタ）を保持。
  • GinBtree: GIN B-treeの操作に必要なメソッドやインデックス情報を含む構造体。

失敗モードと境界条件

  • child が InvalidBlockNumber になることはアサーションで禁止されています。
  • 右端のページ（rightlink == InvalidBlockNumber）に到達した際は、右への移動を停止します。
  • searchMode が true の場合、パスの履歴は保持されず、メモリ消費が抑えられます。

保守担当者向けメモ

  • 本関数はロックカップリングを伴うため、デッドロックを避けるためのロック順序（ルートからリーフへ）が厳守されています。
  • ginFinishOldSplit の呼び出しは、並行処理によるインデックスの不整合を防ぐために不可欠です。
