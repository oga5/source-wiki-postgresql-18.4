WalUsage

定義

  • 種別: struct
  • 定義箇所: src/include/executor/instrument.h:51

目的

クエリ実行に伴うWAL（Write-Ahead Logging）生成量の統計情報を保持・管理するために使用されます。

シグネチャまたは形状

typedef struct WalUsage
{
	int64		wal_records;		/* 生成されたWALレコード数 */
	int64		wal_fpi;			/* 生成されたWALフルページイメージ数 */
	uint64		wal_bytes;			/* 生成されたWALレコードの合計サイズ */
	int64		wal_buffers_full;	/* WALバッファが満杯になった回数 */
} WalUsage;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。実行エンジンが計測を行う際、この構造体の各フィールドが累積的に更新されます。

前提条件と不変条件

  • 各フィールドはリセットされず、常に増加し続ける累積カウンタです。
  • 特定の処理区間の統計を得るには、処理開始時と終了時の値の差分を計算する必要があります。

アルゴリズムまたは振る舞い

実行エンジンは、ノードの開始時および終了時にグローバルなWAL統計を取得し、その差分を本構造体に加算することで、特定のプランノードが消費したWALリソースを定量化します。

呼び出し元と呼び出し先

  • **呼び出し元（参照・操作の証拠）**:
  • contrib/pg_stat_statements/pg_stat_statements.c: 統計情報の集計およびリセットに使用。
  • src/backend/access/brin/brin.c: パラレルクエリ実行時のワーカー間での統計共有および領域確保に使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な呼び出し先はありません。

関連データ構造

  • Instrumentation: プランノードごとの統計情報を管理するメイン構造体。
  • BufferUsage: 同様にディスクI/O統計を保持する構造体。

失敗モードと境界条件

  • カウンタは int64 または uint64 型であり、極めて長時間の実行や大量のWAL生成によりオーバーフローする可能性がありますが、通常のクエリ実行範囲では考慮されていません。

保守担当者向けメモ

  • 本構造体は Instrumentation 構造体の一部として利用されることが一般的です。
  • パラレルクエリ環境下では、各ワーカーの WalUsage を集約するロジックが必要となるため、brin.c 等の並列処理実装を参考にしてください。
