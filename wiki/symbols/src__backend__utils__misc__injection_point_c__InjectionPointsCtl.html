InjectionPointsCtl

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/misc/injection_point.c:83

目的

PostgreSQL の実行コード中に任意のコードを挿入する「インジェクションポイント」機能を管理するための、共有メモリ上の制御構造体です。

シグネチャまたは形状

typedef struct InjectionPointsCtl
{
	pg_atomic_uint32 max_inuse;
	InjectionPointEntry entries[MAX_INJECTION_POINTS];
} InjectionPointsCtl;

入力と出力

  • **入力**: 共有メモリ領域の確保時に InjectionPointShmemSize および InjectionPointShmemInit を通じて初期化されます。
  • **出力**: 登録されたインジェクションポイントのリスト（entries）および現在使用中の最大インデックス（max_inuse）を保持します。

前提条件と不変条件

  • ビルド時に --enable-injection-points が指定されている必要があります。
  • 共有メモリ上に配置され、プロセス間で共有されます。
  • entries 配列は MAX_INJECTION_POINTS 個の InjectionPointEntry を保持します。

アルゴリズムまたは振る舞い

  • 共有メモリ内でのインジェクションポイントの登録情報を管理します。
  • InjectionPointAttach 呼び出し時に、この構造体内の空きスロットへ情報が書き込まれます。
  • 読み取り時には、ロックフリーなアクセスを可能にするための世代管理カウンタ（InjectionPointEntry 内）と併用されます。

呼び出し元と呼び出し先

  • **参照元**:
  • src/backend/utils/misc/injection_point.c: ActiveInjectionPoints ポインタとして定義・参照。
  • src/backend/postmaster/launch_backend.c: ActiveInjectionPoints として参照。
  • src/include/utils/injection_point.h: extern 宣言。
  • **呼び出し先**: 構造体定義のため、直接的な関数呼び出しはありません。

関連データ構造

  • InjectionPointEntry: entries 配列の各要素として使用される、個別のインジェクションポイント定義。

失敗モードと境界条件

  • MAX_INJECTION_POINTS を超える登録はできません。
  • 共有メモリの初期化に失敗した場合、本構造体へのアクセスはできません。

保守担当者向けメモ

  • 本構造体はデバッグおよびテスト専用です。本番環境での使用は想定されていません。
  • ActiveInjectionPoints グローバル変数を通じてアクセスされます。
  • 共有メモリのサイズ計算（InjectionPointShmemSize）および初期化（InjectionPointShmemInit）と密接に関連しています。
