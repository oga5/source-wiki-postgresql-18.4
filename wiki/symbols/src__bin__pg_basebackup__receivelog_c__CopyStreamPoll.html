CopyStreamPoll

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:876

目的

PostgreSQLサーバーとの接続ソケットおよび指定された停止用ソケットを監視し、データ受信の準備が整うか、タイムアウトまたは停止シグナルが発生するまで待機する。

シグネチャまたは形状

static int CopyStreamPoll(PGconn *conn, long timeout_ms, pgsocket stop_socket)

入力と出力

  • **入力**:
  • conn: サーバー接続を保持する PGconn オブジェクト。
  • timeout_ms: 待機するミリ秒数（負の値の場合は無期限）。
  • stop_socket: 監視対象に追加する停止用ソケット（PGINVALID_SOCKET で無効化可能）。
  • **出力**:
  • 1: 接続ソケットにデータが到着。
  • 0: タイムアウト、停止ソケットへの入力、またはシグナル割り込み（EINTR）が発生。
  • -1: エラー発生（ソケットが無効、または select システムコール失敗）。

前提条件と不変条件

  • conn は有効な接続状態である必要がある。
  • stop_socket が指定されている場合、select の監視対象に含まれる。

アルゴリズムまたは振る舞い

1. PQsocket を用いて接続ソケットを取得する。 2. fd_set を初期化し、接続ソケットと（有効であれば）停止ソケットを監視対象に追加する。 3. timeout_ms に基づき select システムコールのタイムアウト値を設定する。 4. select を実行し、監視対象のソケット状態を判定する。 5. EINTR が発生した場合はエラーとみなさず 0 を返す。 6. 接続ソケットにイベントがある場合は 1 を返し、それ以外（タイムアウトや停止ソケットのイベント）は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/receivelog.c 内のストリーミング受信ループ（HandleCopyStream 等から推測される）。
  • **呼び出し先**: PQsocket (libpq), select, FD_ZERO, FD_SET, FD_ISSET, Max, pg_log_error。

関連データ構造

  • PGconn: libpqの接続ハンドル。
  • fd_set: select システムコールで使用するファイル記述子集合。
  • pgsocket: ソケット記述子型。

失敗モードと境界条件

  • PQsocket が負の値を返す場合、エラーログを出力し -1 を返す。
  • select が負の値を返し、かつ errno が EINTR 以外の場合、エラーログを出力し -1 を返す。
  • timeout_ms が負の場合、timeoutptr は NULL となり、無期限待機となる。

保守担当者向けメモ

  • 本関数は select を直接使用しているため、監視対象のソケット数が FD_SETSIZE を超えないことを前提としている。
  • stop_socket は、ストリーミングの停止条件を外部から通知するために使用される。
