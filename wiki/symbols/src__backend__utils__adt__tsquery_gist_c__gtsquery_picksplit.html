gtsquery_picksplit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_gist.c:166

目的

GiST インデックスページが溢れた際に、エントリを2つのグループに分割してツリーのバランスと検索効率を最適化する。

シグネチャまたは形状

Datum gtsquery_picksplit(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *entryvec（分割対象のエントリ群）
  • PG_GETARG_POINTER(1): GIST_SPLITVEC *v（分割結果を格納する構造体）
  • **出力**:
  • PG_RETURN_POINTER(v): 分割後の左右のオフセットと、それぞれのシグネチャ（spl_ldatum, spl_rdatum）が設定された GIST_SPLITVEC。

前提条件と不変条件

  • entryvec には少なくとも2つのエントリが存在する必要がある。
  • 分割後の各グループは、hemdist（ハミング距離）に基づき、オーバーラップを最小化するように振り分けられる。

アルゴリズムまたは振る舞い

1. **シード選択**: 全エントリのペア間でハミング距離（hemdist）を計算し、最も距離が遠い2つのエントリを初期シード（seed_1, seed_2）として選択する。 2. **コスト計算**: 各エントリについて、どちらのシードに近いかを計算し、その差分をコストとして算出する。 3. **ソート**: コストに基づいて全エントリをソートする。 4. **振り分け**: ソート順に従い、各エントリを左右のグループに割り当てる。割り当ての際、WISH_F マクロを用いてグループのバランスを調整する。 5. **結果格納**: 左右のグループのシグネチャを TSQuerySignGetDatum で変換し、v に格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの管理機構（直接的な呼び出し元情報はソースコード内に明示されていない）。
  • **呼び出し先**: hemdist, qsort, comparecost, TSQuerySignGetDatum, palloc。

関連データ構造

  • GistEntryVector: 分割対象のインデックスエントリを保持。
  • GIST_SPLITVEC: 分割結果（左右のオフセット配列とシグネチャ）を保持。
  • SPLITCOST: 分割コスト計算用の一時構造体。
  • TSQuerySign: tsquery を表現するビットマップ。

失敗モードと境界条件

  • シードが選択できない場合（seed_1 == 0 || seed_2 == 0）、デフォルトで最初の2つのエントリをシードとして使用する。
  • メモリ不足時は palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数はハミング距離に基づくヒューリスティックな分割を行う。インデックスの検索効率は hemdist の精度に依存する。
  • WISH_F によるバランス調整の重み付け（0.05）は、インデックスの木構造の偏りを防ぐための定数である。
