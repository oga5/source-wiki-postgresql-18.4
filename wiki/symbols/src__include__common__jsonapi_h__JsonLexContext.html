JsonLexContext

定義

  • 種別: struct
  • 定義箇所: src/include/common/jsonapi.h:100

目的

JSON 解析プロセスにおける現在の状態、入力バッファ、字句解析の進捗、およびエラー情報を保持するメイン構造体です。

シグネチャまたは形状

typedef struct JsonLexContext
{
	const char *input;
	size_t		input_length;
	int			input_encoding;
	const char *token_start;
	const char *token_terminator;
	const char *prev_token_terminator;
	bool		incremental;
	JsonTokenType token_type;
	int			lex_level;
	bits32		flags;
	int			line_number;
	const char *line_start;
	JsonParserStack *pstack;
	JsonIncrementalState *inc_state;
	bool		need_escapes;
	struct jsonapi_StrValType *strval;
	struct jsonapi_StrValType *errormsg;
} JsonLexContext;

入力と出力

  • **入力**: input ポインタと input_length により解析対象の文字列を指定します。
  • **出力**: 解析結果は token_type に格納され、解析中のエラーメッセージは errormsg に保持されます。

前提条件と不変条件

  • line_number は 1 から開始されます。
  • need_escapes が true の場合のみ、strval が使用されます。
  • 解析の各段階で JsonSemAction コールバック関数と連携して動作します。

アルゴリズムまたは振る舞い

  • 入力ストリームから JSON トークンを順次切り出し、解析レベル（lex_level）を追跡します。
  • incremental フラグが有効な場合、断片的なデータに対するストリーミング解析をサポートします。
  • need_escapes が有効な場合、文字列のデコード処理を行い、メモリ消費量が増加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/json.c および jsonb.c、jsonfuncs.c 内の各関数がこの構造体をインスタンス化して使用しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、具体的な解析ロジックは jsonapi.c 内の関数群（pg_parse_json 等）によって操作されます。

関連データ構造

  • JsonTokenType: トークンの種類を識別する列挙型。
  • JsonParserStack: 解析中のネスト構造を管理するスタック。
  • JsonIncrementalState: 増分解析時の状態管理用構造体。
  • JsonSemAction: 解析イベントに応じたコールバック関数のテーブル。

失敗モードと境界条件

  • 解析エラーが発生した場合、errormsg に詳細が格納されます。
  • need_escapes を true に設定するとメモリ消費が増大するため、不要な場合は false を推奨します。
  • コールバックに渡される文字列トークンの解放責任は、setJsonLexContextOwnsTokens の設定によりパーサー側に委譲可能です。

保守担当者向けメモ

  • 本構造体はバックエンドとフロントエンド（libpq）の両方で利用されるため、変更時は両環境への影響を考慮してください。
  • 長期実行プロセスでは、トークンの所有権管理を適切に行いメモリリークを防止してください。
