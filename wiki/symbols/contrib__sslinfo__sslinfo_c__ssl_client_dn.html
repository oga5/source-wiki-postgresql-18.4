ssl_client_dn

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:300

目的

現在のセッションにおけるSSLクライアント証明書のサブジェクトDN（Distinguished Name）をテキスト形式で取得する。

シグネチャまたは形状

Datum ssl_client_dn(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数インターフェース経由の引数）
  • **出力**: Datum（クライアント証明書のサブジェクトDNを表す text 型。証明書が存在しない、または無効な場合は NULL）

前提条件と不変条件

  • MyProcPort が有効であり、SSL接続が確立されていること。
  • クライアント証明書が提示され、検証済みであること。

アルゴリズムまたは振る舞い

1. MyProcPort->ssl_in_use および MyProcPort->peer_cert_valid を確認し、SSLが使用されていないか証明書が無効な場合は NULL を返す。 2. be_tls_get_peer_subject_name を呼び出し、現在のポートからサブジェクトDNを subject バッファ（サイズ NAMEDATALEN）に取得する。 3. subject が空文字であれば NULL を返す。 4. 取得した文字列を cstring_to_text でPostgreSQLの text 型に変換し、PG_RETURN_TEXT_P で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロによる登録）。
  • **呼び出し先**: be_tls_get_peer_subject_name（証明書情報の抽出）。

関連データ構造

  • Port (MyProcPort): セッションのSSL状態を保持する構造体。
  • NAMEDATALEN: バッファサイズの定数。

失敗モードと境界条件

  • SSL接続が確立されていない、または証明書が検証できない場合は NULL を返す。
  • 証明書のサブジェクトDNが空の場合、NULL を返す。
  • be_tls_get_peer_subject_name 内部でエラーが発生した場合、PostgreSQLの標準的なエラーハンドリング（ereport(ERROR, ...)）によりトランザクションが中断される可能性がある。

保守担当者向けメモ

  • 本関数は ssl_issuer_dn と共通のロジックを持つ可能性がある（ソース内のコメント参照）。
  • NAMEDATALEN を超える長さのDNを持つ証明書の場合、be_tls_get_peer_subject_name の挙動に依存する。
