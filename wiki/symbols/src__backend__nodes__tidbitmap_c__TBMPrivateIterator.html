TBMPrivateIterator

定義

  • 種別: struct
  • 定義箇所: src/backend/nodes/tidbitmap.c:169

目的

TIDBitmap を反復処理する際の進捗状況を追跡するためのプライベートな構造体です。

シグネチャまたは形状

struct TBMPrivateIterator
{
    TIDBitmap  *tbm;            /* 反復対象のTIDBitmap */
    int         spageptr;       /* 次のspagesインデックス */
    int         schunkptr;      /* 次のschunksインデックス */
    int         schunkbit;      /* 現在のschunk内で次に確認するビット位置 */
};

入力と出力

  • **入力**: tbm_begin_iterate 等の関数により、対象となる TIDBitmap へのポインタが設定されます。
  • **出力**: tbm_private_iterate を通じて、反復処理の結果（TBMIterateResult）を生成するための内部状態として利用されます。

前提条件と不変条件

  • 反復処理中は、対象の TIDBitmap が読み取り専用の状態であることが想定されます。
  • 構造体は呼び出し側のメモリコンテキストで生成され、反復処理終了まで保持される必要があります。

アルゴリズムまたは振る舞い

  • spageptr、schunkptr、schunkbit を用いて、ビットマップ内のページおよびチャンクの走査位置を管理します。
  • tbm_private_iterate はこの構造体を参照して、次に処理すべきタプル識別子（TID）を特定します。
  • tbm_end_private_iterate により、反復処理の終了処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: tbm_begin_iterate（構造体の生成）、tbm_private_iterate（状態の更新）、tbm_end_private_iterate（終了処理）。また、src/include/access/gin_private.h 等で参照されています。
  • **呼び出し先**: 直接的な関数呼び出しは構造体定義には含まれませんが、反復処理の過程で TIDBitmap 内部のページ配列やチャンク配列にアクセスします。

関連データ構造

  • TIDBitmap: 反復対象となるメインのビットマップ構造体。
  • TBMIterateResult: 反復処理によって返される結果構造体。

失敗モードと境界条件

  • 構造体が適切に初期化されていない状態で tbm_private_iterate を呼び出した場合の動作は未定義です。
  • 反復処理中に TIDBitmap が変更された場合、整合性は保証されません。

保守担当者向けメモ

  • 本構造体はプライベートなインターフェースであり、tidbitmap.c の外部から直接メンバを操作することは推奨されません。
  • 構造体のメモリ割り当ては呼び出し側の責任で行われるため、ライフサイクル管理に注意してください。
