FileSetDeleteAll

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fileset.c:149

目的

FileSet に関連付けられたすべてのディレクトリを、設定されたすべてのテーブルスペースから削除し、リソースを解放する。

シグネチャまたは形状

void FileSetDeleteAll(FileSet *fileset)

入力と出力

  • **入力**: FileSet *fileset - 削除対象のファイルセットを指すポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • fileset は FileSetInit によって初期化済みであること。
  • FileSet はトランザクション終了時に自動削除されないため、呼び出し側が明示的に本関数を呼び出す必要がある。

アルゴリズムまたは振る舞い

1. fileset->ntablespaces で保持されているテーブルスペースの数だけループを実行する。 2. 各テーブルスペースに対して FileSetPath を呼び出し、対象となるディレクトリの物理パスを dirpath に生成する。 3. PathNameDeleteTemporaryDir(dirpath) を呼び出し、該当ディレクトリを削除する。 4. エラー発生時も処理を中断せず、必要に応じてログを出力する（エラークリーンアップパスでの利用を想定しているため）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/launcher.c
  • src/backend/storage/file/sharedfileset.c
  • **呼び出し先**:
  • FileSetPath
  • PathNameDeleteTemporaryDir

関連データ構造

  • FileSet: 一時ファイルの集合を管理する構造体。ntablespaces（テーブルスペース数）および tablespaces[]（テーブルスペース配列）を保持する。

失敗モードと境界条件

  • I/O エラーが発生した場合、関数自体は失敗しないが、ログメッセージが生成される可能性がある。
  • 削除対象のディレクトリが存在しない場合やアクセス権限がない場合の挙動は、内部で呼び出される PathNameDeleteTemporaryDir の実装に依存する。

保守担当者向けメモ

  • 本関数はエラークリーンアップパスでの使用を前提としており、例外を投げずに処理を完遂する設計となっている。
  • FileSet の生存期間中、テーブルスペース情報は一貫性が保たれるため、ループ処理中にテーブルスペース構成が変更される懸念はない。
