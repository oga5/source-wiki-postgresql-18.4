jsonb_subscript_transform

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonbsubs.c:42

目的

jsonb 型に対する添字操作（data[key] や data[index]）の構文解析フェーズにおいて、指定された添字式を検証し、適切な型（int4 または text）へ変換して SubscriptingRef ノードに格納する。

シグネチャまたは形状

static void jsonb_subscript_transform(SubscriptingRef *sbsref, List *indirection, ParseState *pstate, bool isSlice, bool isAssignment)

入力と出力

  • **入力**:
  • sbsref: 添字操作情報を保持する SubscriptingRef ノード。
  • indirection: 添字式のリスト。
  • pstate: パーサーの状態。
  • isSlice: スライス操作か否かを示すフラグ。
  • isAssignment: 代入操作か否かを示すフラグ。
  • **出力**: なし（sbsref の refupperindexpr および refrestype を更新）。

前提条件と不変条件

  • jsonb の添字操作はスライスをサポートしない。isSlice が真の場合、エラーを発生させる。
  • 添字式は int4 または text のいずれか一方にのみ暗黙的にキャスト可能である必要がある。両方にキャスト可能な場合は曖昧さ回避のためエラーとなる。
  • 変換後の refrestype は常に JSONBOID となる。

アルゴリズムまたは振る舞い

1. indirection リストを走査し、各添字式を transformExpr で解析する。 2. スライス指定（isSlice）がある場合は即座にエラーを投げる。 3. 添字式の型を can_coerce_type で確認し、int4 または text への変換可能性を判定する。 4. 変換可能な型が特定できない、または複数候補が存在する場合はエラーを投げる。 5. coerce_type を用いて添字式を確定した型へ変換し、upperIndexpr リストに追加する。 6. 最終的に sbsref->refupperindexpr に変換済みリストを格納し、refrestype を JSONBOID に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonbsubs.c:405（jsonb_subscript_handler 構造体内の関数ポインタとして参照）。
  • **呼び出し先**:
  • transformExpr, exprType, can_coerce_type, coerce_type, ereport, lappend 等のパーサーおよびリスト操作関数。

関連データ構造

  • SubscriptingRef: 添字操作を表現するノード。
  • A_Indices: 構文解析中の添字情報を保持する構造体。

失敗モードと境界条件

  • **スライス操作**: isSlice が真の場合、ERRCODE_DATATYPE_MISMATCH でエラー。
  • **型変換の曖昧性**: int4 と text の両方に変換可能な式が渡された場合、ERRCODE_DATATYPE_MISMATCH でエラー。
  • **型変換不能**: どちらの型にも変換できない式が渡された場合、ERRCODE_DATATYPE_MISMATCH でエラー。

保守担当者向けメモ

  • 本関数は jsonb の添字構文のバリデーションを担う。将来的に jsonpath 等のサポートを追加する場合、型変換ロジックの拡張が必要となる可能性がある。
  • isSlice のチェックは indirection の走査中に行われるが、SubscriptingRef の仕様変更時には注意が必要。
