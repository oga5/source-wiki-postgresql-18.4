ParseAbortRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xactdesc.c:140

目的

トランザクションのアボート（Abort）に関連するWALレコードのバイナリデータを解析し、構造体 xl_xact_parsed_abort に展開する。主に pg_waldump やログ出力機能において、可変長のWALレコードを人間が読める形式に変換する前処理として使用される。

シグネチャまたは形状

void ParseAbortRecord(uint8 info, xl_xact_abort *xlrec, xl_xact_parsed_abort *parsed)

入力と出力

  • **入力**:
  • info: WALレコードのフラグ（XLOG_XACT_HAS_INFO 等）。
  • xlrec: 解析対象の xl_xact_abort 構造体へのポインタ。
  • **出力**:
  • parsed: 解析結果が格納される xl_xact_parsed_abort 構造体。

前提条件と不変条件

  • parsed は呼び出し元で確保されたメモリ領域を指している必要がある。
  • xlrec は有効なWALレコードの先頭を指していること。
  • info フラグに基づき、data ポインタをオフセット計算で進めることで可変長データを順次読み取る。

アルゴリズムまたは振る舞い

1. parsed 構造体をゼロクリアし、基本情報をコピーする。 2. info フラグを確認し、以下のオプション情報が存在する場合は順次解析して parsed に格納する：

  • XLOG_XACT_HAS_INFO: 追加情報フラグ。
  • XACT_XINFO_HAS_DBINFO: データベースおよびテーブルスペース情報。
  • XACT_XINFO_HAS_SUBXACTS: サブトランザクションIDリスト。
  • XACT_XINFO_HAS_RELFILELOCATORS: 関連するリレーション情報。
  • XACT_XINFO_HAS_DROPPED_STATS: 削除された統計情報。
  • XACT_XINFO_HAS_TWOPHASE: 2フェーズコミット関連情報（GIDを含む）。
  • XACT_XINFO_HAS_ORIGIN: レプリケーションオリジン情報。

    • 3. アライメントが保証されないデータ（origin 等）に対しては、memcpy を使用してスタック上の変数にコピーしてから抽出する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/rmgrdesc/xactdesc.c (内部の記述子ルーチン)
  • src/backend/access/transam/xact.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/replication/logical/decode.c
  • **呼び出し先**:
  • 標準ライブラリ関数: memset, memcpy, strlcpy, strlen

関連データ構造

  • xl_xact_abort: WALレコードの基本構造。
  • xl_xact_parsed_abort: 解析後の情報を保持する構造体。
  • xl_xact_xinfo, xl_xact_dbinfo, xl_xact_subxacts, xl_xact_relfilelocators, xl_xact_stats_items, xl_xact_twophase, xl_xact_origin: WALレコード内の各セクションに対応するデータ構造。

失敗モードと境界条件

  • WALレコードの構造が期待されるフラグと一致しない場合、不正なメモリ読み取りが発生する可能性がある。
  • XACT_XINFO_HAS_ORIGIN 以降のデータはアライメントが保証されないため、直接ポインタキャストを行わず memcpy を介する必要がある。

保守担当者向けメモ

  • 本関数はバックエンドとフロントエンド（pg_waldump）で共有されるコードであるため、依存関係の追加には注意が必要。
  • WALレコードのパースはオフセット計算に依存しており、構造体の定義変更時には data ポインタの進め方を慎重に確認すること。
