getNotify

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:1634

目的

サーバーから送信された非同期通知（NotifyResponse）メッセージを解析し、PGnotify 構造体として接続オブジェクトの通知リストに追加します。

シグネチャまたは形状

static int getNotify(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn（現在の接続状態を保持する構造体）
  • **出力**: 成功時は 0、バッファ不足等で読み込みが中断された場合は EOF を返します。メモリ不足等の致命的エラー時は handleFatalError を経由して接続を終了し、0 を返します。

前提条件と不変条件

  • conn->inBuffer に NotifyResponse メッセージの本体が格納されている必要があります。
  • PGnotify 構造体は、通知名（relname）と追加情報（extra）を同一メモリブロック内に保持するよう設計されています。

アルゴリズムまたは振る舞い

1. pqGetInt を使用してバックエンドのプロセスID（be_pid）を読み取ります。 2. pqGets を使用して通知名を取得し、一時的に svname にコピーします。 3. 再度 pqGets を使用して追加情報を取得します。 4. malloc を行い、PGnotify 構造体と通知名、追加情報を格納するための連続したメモリ領域を確保します。 5. 確保した領域に各データをコピーし、conn->notifyHead および conn->notifyTail を更新してリストの末尾に連結します。 6. 一時領域 svname を解放して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c 内のメッセージ解析ループ（pqParseInput3 等）から、NotifyResponse メッセージタイプを受信した際に呼び出されます。
  • **呼び出し先**: pqGetInt, pqGets（バッファ読み込み）、handleFatalError（致命的エラー時）、malloc, free, strdup, strcpy, strlen（標準ライブラリ）。

関連データ構造

  • PGconn: 接続状態および通知リスト（notifyHead, notifyTail）を保持します。
  • PGnotify: 通知情報を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: malloc または strdup が失敗した場合、libpq_append_conn_error でエラーを記録し、handleFatalError を呼び出して接続を強制終了します。
  • **バッファ不足**: pqGetInt や pqGets がデータ不足を検知した場合、EOF を返して呼び出し元に制御を戻します。

保守担当者向けメモ

  • 本関数は NotifyResponse メッセージの解析に特化しています。
  • PGnotify 構造体のメモリレイアウトは、relname と extra を構造体の直後に配置することで、単一の free 呼び出しで解放できるように最適化されています。この構造を変更する際は注意してください。
