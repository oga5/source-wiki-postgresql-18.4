SyncRepStandbyData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/syncrep.h:42

目的

プライマリサーバーにおいて、各WAL送信プロセス（WalSender）の現在のレプリケーション進捗状況を保持し、同期レプリケーションの待機判定を行うためのデータ構造です。

シグネチャまたは形状

typedef struct SyncRepStandbyData
{
	pid_t		pid;
	XLogRecPtr	write;
	XLogRecPtr	flush;
	XLogRecPtr	apply;
	int			sync_standby_priority;
	int			walsnd_index;
	bool		is_me;
} SyncRepStandbyData;

入力と出力

本構造体は、共有メモリ上の WalSnd 構造体からコピーされた情報を保持します。

  • **入力**: WalSnd 構造体の各フィールド（PID、LSN進捗、優先度、インデックス）。
  • **出力**: 同期レプリケーションの判定ロジック（SyncRepReleaseWaiters 等）において、スタンバイの進捗状況として参照されます。

前提条件と不変条件

  • write, flush, apply は XLogRecPtr 型であり、スタンバイ側から報告されたWALの到達地点を示します。
  • walsnd_index は、共有メモリ内の WalSnd 配列におけるインデックスと一致している必要があります。

アルゴリズムまたは振る舞い

  • SyncRepGetCandidateStandbys 関数によって、全WalSenderの状態が本構造体の配列として取得されます。
  • 取得された配列は、同期スタンバイの選択アルゴリズム（Priorityベース等）に従ってソートされ、待機中のバックエンドを解放すべきかどうかの判定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/syncrep.c 内の関数群（SyncRepGetCandidateStandbys 等）が、この構造体のインスタンスを生成・操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • WalSnd: 共有メモリ上の実体。本構造体はここから情報をコピーします。
  • SyncRepConfigData: 同期レプリケーションの設定（待機数やメソッド）を保持する構造体。

失敗モードと境界条件

  • max_wal_senders の数を超えるスタンバイデータは扱えません。
  • SyncRepGetCandidateStandbys でのメモリ割り当て（palloc）が失敗した場合、処理が中断されます。

保守担当者向けメモ

  • 本構造体は WalSnd のスナップショットであるため、更新のタイミングによっては最新の進捗と乖離が生じる可能性があります。
  • is_me フラグは、現在のプロセス自身がスタンバイであるかを識別するために使用されます。
