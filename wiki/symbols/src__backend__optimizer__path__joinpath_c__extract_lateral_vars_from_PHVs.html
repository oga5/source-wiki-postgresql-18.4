extract_lateral_vars_from_PHVs

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinpath.c:583

目的

Memoize ノードのキャッシュキーとして使用するための、PlaceHolderVar (PHV) に含まれる横断的（LATERAL）な参照変数や式を抽出する。

シグネチャまたは形状

List *extract_lateral_vars_from_PHVs(PlannerInfo *root, Relids innerrelids)

入力と出力

  • **入力**:
  • root: クエリ全体のプランナ情報。
  • innerrelids: 評価対象となるリレーションの集合。
  • **出力**:
  • キャッシュキーとして利用可能な Var、PlaceHolderVar、または式（Expr）のリスト。

前提条件と不変条件

  • クエリ内に LATERAL RTE が存在しない場合、または innerrelids が複数のリレーションを含む場合（結合リレーション上での Memoize をサポートしないため）、即座に NIL を返す。
  • 抽出対象の PHV は、ph_lateral が設定されており、かつ ph_eval_at が innerrelids と一致している必要がある。

アルゴリズムまたは振る舞い

1. root->hasLateralRTEs を確認し、なければ終了。 2. innerrelids が単一リレーションでない場合は終了。 3. root->placeholder_list を走査し、条件を満たす PlaceHolderInfo を特定する。 4. PHV の式が innerrelids を参照しない場合、その式全体をキャッシュキーとしてリストに追加する。 5. PHV の式が innerrelids を参照する場合、式から Var および PlaceHolderVar を抽出する。

  • Var の場合: varno が phinfo->ph_lateral に含まれるものを抽出。
  • PlaceHolderVar の場合: その ph_eval_at が phinfo->ph_lateral のサブセットであるものを抽出。

    • 6. 抽出したノードのリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinpath.c:704 (文脈上、Memoize パス生成処理の一部として呼び出される)。
  • **呼び出し先**: pull_varnos, pull_vars_of_level, find_placeholder_info, bms_equal, bms_overlap, bms_is_member, bms_is_subset, lappend, list_free 等のノード操作・ビットマップ操作関数。

関連データ構造

  • PlannerInfo: クエリ全体のプランナ情報。
  • PlaceHolderInfo: プレースホルダの評価位置や横断的参照情報を保持。
  • Relids: リレーション ID の集合（ビットマップ）。
  • Var, PlaceHolderVar: 式に含まれるノード型。

失敗モードと境界条件

  • innerrelids が BMS_MULTIPLE の場合、キャッシュキー抽出を行わず NIL を返す。
  • 抽出対象のノードが Var でも PlaceHolderVar でもない場合、Assert(false) により停止する。

保守担当者向けメモ

  • 本関数は Memoize ノードの最適化に直結する。キャッシュキーの選択ロジック（式全体を使うか、個別の変数を使うか）は、キャッシュ効率に影響を与えるため、変更時は注意が必要である。
