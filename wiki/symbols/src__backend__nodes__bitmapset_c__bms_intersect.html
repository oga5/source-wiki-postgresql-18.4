bms_intersect

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/bitmapset.c:291

目的

2つの Bitmapset の積集合（共通部分）を計算し、新しい Bitmapset として返します。

シグネチャまたは形状

Bitmapset *bms_intersect(const Bitmapset *a, const Bitmapset *b)

入力と出力

  • **入力**:
  • a, b: 比較対象の Bitmapset へのポインタ。
  • **出力**:
  • 両方の集合に含まれる要素のみを持つ新しい Bitmapset。
  • 共通要素がない場合、または入力のいずれかが NULL の場合は NULL を返します。

前提条件と不変条件

  • 入力される Bitmapset は bms_is_valid_set を満たす有効なセットである必要があります。
  • 空集合は NULL ポインタとして表現されます。
  • 戻り値として新しいメモリ領域が割り当てられるため、呼び出し元は結果を適切に管理（必要に応じて bms_free）する必要があります。

アルゴリズムまたは振る舞い

1. 入力のいずれかが NULL の場合、直ちに NULL を返します。 2. 入力のうち要素数が少ない方を bms_copy で複製し、結果セットのベースとします。 3. 複製したセットの各ワードに対し、もう一方のセットの対応するワードとのビット単位の論理積（AND）をとります。 4. 演算の結果、全ビットがゼロになった場合は pfree でメモリを解放し NULL を返します。 5. ゼロ以外のワードが存在する場合、末尾のゼロワードを切り詰めて（nwords を更新して）結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c

  • src/backend/optimizer/path/joinpath.c

  • **呼び出し先**:

  • bms_copy
  • pfree

関連データ構造

  • Bitmapset: 集合を表現する構造体。nwords（ワード数）と words（ビット配列）を保持。

失敗モードと境界条件

  • 入力が NULL の場合、安全に NULL を返します。
  • 共通要素が全くない場合、結果セットは生成されず NULL が返されます。
  • メモリ不足時には palloc（bms_copy 経由）によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は入力集合を破壊しません。入力集合を再利用して効率化したい場合は、bms_int_members の利用を検討してください。
  • 戻り値が NULL になる可能性があるため、呼び出し側では必ず戻り値の NULL チェックを行ってください。
