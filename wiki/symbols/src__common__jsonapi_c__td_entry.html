td_entry

定義

  • 種別: struct
  • 定義箇所: src/common/jsonapi.c:233

目的

JSONのインクリメンタル（非再帰的）構文解析において、パーサーの遷移テーブル（td_parser_table）の各要素を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	size_t		len;
	char	   *prod;
} td_entry;

入力と出力

  • 入力: 構造体の各フィールド（len および prod）に値を設定して初期化します。
  • 出力: 構文解析の遷移テーブルの要素として、特定の非終端記号に対する生成規則（プロダクション）を表現します。

前提条件と不変条件

  • prod は、解析対象の生成規則を表す文字列を指す必要があります。
  • len は、prod が指す文字列の長さを保持する必要があります。

アルゴリズムまたは振る舞い

本構造体は、テーブル駆動型パーサーの遷移状態を定義します。pg_parse_json_incremental が実行される際、現在のスタック状態と入力トークンに基づき、この td_entry が参照されて次の解析アクション（予測）が決定されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/common/jsonapi.c 内の td_parser_table 定義にて使用。
  • push_prediction 関数（引数として利用）。
  • pg_parse_json_incremental の内部ロジック（goal の初期化や entry 変数への代入）。
  • 呼び出し先: 情報なし。

関連データ構造

  • JsonParserStack: インクリメンタル解析時に td_entry をスタックに積むために使用されます。
  • td_parser_table: td_entry 型の要素を格納する二次元配列。

失敗モードと境界条件

  • prod が NULL または不正な文字列を指している場合、パーサーは予期しない動作を引き起こす可能性があります。
  • len が実際の prod の長さと一致しない場合、文字列操作時にバッファオーバーランや誤った解析結果を招くリスクがあります。

保守担当者向けメモ

  • 本構造体は src/common/jsonapi.c 内のテーブル駆動型パーサー専用です。変更を加える際は、td_parser_table の整合性と、push_prediction におけるスタック操作への影響を十分に確認してください。
