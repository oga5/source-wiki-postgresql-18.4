anytime_typmodin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/date.c:50

目的

TIME および TIMETZ 型の型修飾子（typmod）を解析し、妥当性を検証する。

シグネチャまたは形状

int32 anytime_typmodin(bool istz, ArrayType *ta)

入力と出力

  • **入力**:
  • istz (bool): タイムゾーン付き（TIMETZ）か否か（TIME）を示すフラグ。
  • ta (ArrayType*): SQLパーサから渡される型修飾子の配列。
  • **出力**:
  • int32: 検証済みの型修飾子値。

前提条件と不変条件

  • ta は ArrayGetIntegerTypmods によって整数配列として解釈可能であること。
  • TIME 型の定義上、型修飾子は1つ（精度）のみ許容される。

アルゴリズムまたは振る舞い

1. ArrayGetIntegerTypmods を呼び出し、入力配列から整数値のリストを取得する。 2. 取得した要素数 n が1であることを確認する。1以外の場合は ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告する。 3. 取得した単一の修飾子値を anytime_typmod_check に渡し、精度制限の妥当性を検証してその結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/date.c:1627 (TIME 型の型修飾子処理)
  • src/backend/utils/adt/date.c:2452 (TIMETZ 型の型修飾子処理)
  • **呼び出し先**:
  • ArrayGetIntegerTypmods
  • anytime_typmod_check
  • ereport (エラー発生時)

関連データ構造

  • ArrayType: SQLの型修飾子を保持する配列構造体。

失敗モードと境界条件

  • n != 1: 期待される修飾子の数が異なる場合、ERROR を発生させ処理を中断する。
  • anytime_typmod_check が許容しない値（精度範囲外など）が渡された場合、同関数内でエラーが発生する。

保守担当者向けメモ

  • 本関数は TIME および TIMETZ の両方で共通の型修飾子解析ロジックとして機能する。
  • 文法上、正しい数の修飾子が渡されることが期待されているため、エラーメッセージの詳細は簡潔に留めている。
