comparecost

定義

  • 種別: function
  • 定義箇所: contrib/ltree/_ltree_gist.c:237

目的

GiSTインデックス構築時の分割コスト計算において、SPLITCOST 構造体の配列をコスト値に基づいて昇順にソートするための比較関数を提供します。

シグネチャまたは形状

static int comparecost(const void *a, const void *b)

入力と出力

  • 入力: const void *a, const void *b （SPLITCOST 構造体へのポインタとしてキャストされる）
  • 出力: int 型。a のコストが b より小さければ負、等しければ 0、大きければ正の値を返します。

前提条件と不変条件

  • 引数 a および b は、SPLITCOST 構造体へのポインタとしてキャスト可能である必要があります。
  • SPLITCOST 構造体には cost メンバが存在することが前提です。

アルゴリズムまたは振る舞い

1. 引数 a と b を const SPLITCOST * にキャストします。 2. a->cost から b->cost を減算した結果を返します。これにより、標準的な qsort 関数による昇順ソートを可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/ltree/_ltree_gist.c 内の qsort 呼び出し（322行目）で使用されています。
  • 類似の用途として、contrib/hstore/hstore_gist.c、contrib/intarray/_int_gist.c、contrib/intarray/_intbig_gist.c、contrib/pg_trgm/trgm_gist.c、および src/backend/utils/adt/tsgistidx.c においても同名の関数が定義・利用されています。

関連データ構造

  • SPLITCOST: コスト値を保持する構造体。

失敗モードと境界条件

  • a または b が NULL の場合、あるいは SPLITCOST 構造体以外のメモリ領域を指している場合は未定義の動作となります。

保守担当者向けメモ

  • 本関数は PostgreSQL の複数の拡張モジュールで重複して定義されています。共通ライブラリへの移行の検討対象となり得ますが、現状は各モジュール内で独立して定義されています。
