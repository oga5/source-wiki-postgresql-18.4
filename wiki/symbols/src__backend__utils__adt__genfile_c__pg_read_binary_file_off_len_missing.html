pg_read_binary_file_off_len_missing

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:363

目的

サーバー上のファイルを指定したオフセットと長さで読み込み、バイナリデータ（bytea）として返します。ファイルが存在しない場合にエラーを発生させるか否かを制御するフラグを受け取ります。

シグネチャまたは形状

Datum pg_read_binary_file_off_len_missing(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): ファイルパス（text型）。
  • PG_GETARG_INT64(1): 読み込み開始オフセット（int64）。
  • PG_GETARG_INT64(2): 読み込むバイト数（int64）。
  • PG_GETARG_BOOL(3): ファイル不在時にエラーとしない場合はtrueを指定。
  • **出力**:
  • 成功時: 読み込んだ内容を格納した bytea 型の Datum。
  • ファイル不在時（missing_ok が true の場合）: NULL。

前提条件と不変条件

  • 呼び出し元は PostgreSQL の関数呼び出し規約に従う必要があります。
  • 実行ユーザーは pg_read_server_files ロールの権限を有している必要があります（内部の pg_read_binary_file_common による検証）。

アルゴリズムまたは振る舞い

1. 引数からファイルパス、オフセット、読み込みサイズ、missing_ok フラグを取得します。 2. pg_read_binary_file_common を呼び出し、ファイルシステムからの読み込みを実行します。 3. pg_read_binary_file_common が NULL を返した場合（ファイル不在かつ missing_ok が true の場合）、PG_RETURN_NULL() で終了します。 4. 読み込みに成功した場合、結果を PG_RETURN_BYTEA_P を通じて返します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_read_binary_file_common
  • **呼び出し元**: SQLインターフェース（pg_proc.h で定義されたSQL関数経由）。具体的なSQLレベルの呼び出し元情報はソースコードからは特定できません。

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • text: PostgreSQL の可変長文字列型。
  • bytea: PostgreSQL のバイナリデータ型。

失敗モードと境界条件

  • **権限不足**: pg_read_server_files ロールがない場合、pg_read_binary_file_common 内でエラーが発生します。
  • **ファイル不在**: missing_ok が false の場合、ファイルが存在しないとエラーが発生します。
  • **パスの不正**: convert_and_check_filename の制約により、データディレクトリ外へのアクセスはエラーとなります。

保守担当者向けメモ

  • 本関数は pg_read_binary_file_off_len のバリエーションであり、missing_ok 引数を追加したものです。
  • 権限チェックやファイル操作のロジックは pg_read_binary_file_common に集約されているため、共通の挙動を変更する場合はそちらを確認してください。
