xl_seq_rec

定義

  • 種別: struct
  • 定義箇所: src/include/commands/sequence.h:48

目的

シーケンスの状態変更をWAL（Write-Ahead Logging）に記録するための構造体です。クラッシュリカバリ時にシーケンスの値を復元するために使用されます。

シグネチャまたは形状

typedef struct xl_seq_rec
{
    RelFileLocator locator;
    /* SEQUENCE TUPLE DATA FOLLOWS AT THE END */
} xl_seq_rec;

入力と出力

  • **入力**: RelFileLocator 型の locator フィールドにより、対象となるシーケンスリレーションを特定します。
  • **出力**: 構造体の直後にシーケンスのタプルデータが続く形式でWALレコードとして書き出されます。

前提条件と不変条件

  • 構造体の直後にシーケンスのタプルデータが配置されている必要があります。
  • WAL書き込み時には XLogRegisterData を使用して、本構造体とそれに続くタプルデータが適切に登録される必要があります。

アルゴリズムまたは振る舞い

1. シーケンスの更新時、xl_seq_rec を初期化し、対象リレーションの RelFileLocator を設定します。 2. XLogRegisterData を用いて xl_seq_rec 構造体自体をWALバッファに登録します。 3. 構造体の直後に、更新されたシーケンスのタプルデータを連結して書き込みます。 4. リカバリ時には、XLogRecGetData で本構造体を取得し、その直後のメモリ領域からタプルデータを読み取ってシーケンスの状態を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/sequence.c 内のシーケンス更新処理（nextval 等）において、WALログ生成時に使用されます。
  • **呼び出し先**: src/backend/access/rmgrdesc/seqdesc.c において、WALレコードのデコードおよび記述に使用されます。

関連データ構造

  • RelFileLocator: シーケンスの物理的な位置を特定するために使用されます。
  • FormData_pg_sequence_data: 本構造体の直後に格納される実際のシーケンスデータ構造です。

失敗モードと境界条件

  • WALレコードの長さが sizeof(xl_seq_rec) 未満である場合、データ読み取り時に不正なメモリ参照が発生する可能性があります。
  • XLogRecGetDataLen を用いて、構造体サイズを超えた分のデータ長を正確に計算する必要があります。

保守担当者向けメモ

  • 本構造体は可変長データ（タプルデータ）を伴うため、構造体サイズのみを読み取って終了しないよう注意してください。
  • src/backend/commands/sequence.c の 1925 行目付近で、構造体サイズをオフセットとして使用してデータにアクセスしている点に留意してください。
