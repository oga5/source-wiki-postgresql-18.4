enum_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:108

目的

列挙型（ENUM）のテキスト形式の入力を受け取り、対応するシステムカタログ pg_enum 上の OID に変換します。

シグネチャまたは形状

Datum enum_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_CSTRING(0): 列挙型のラベル名（文字列）。
  • PG_GETARG_OID(1): 対象となる列挙型の型 OID。
  • **出力**:
  • Datum: 変換後の列挙値の OID を返します。
  • **エラー**: 入力文字列が NAMEDATALEN を超える場合、または pg_enum に該当するラベルが存在しない場合はエラーを発生させます。

前提条件と不変条件

  • 入力されたラベル名は NAMEDATALEN 未満である必要があります。
  • pg_enum システムカタログに、指定された型 OID とラベル名の組み合わせが存在する必要があります。

アルゴリズムまたは振る舞い

1. 入力文字列の長さを検証し、制限を超える場合はエラーを返します。 2. SearchSysCache2 を使用して ENUMTYPOIDNAME キャッシュから対象の列挙値を検索します。 3. 検索結果が有効でない場合はエラーを返します。 4. check_safe_enum_use を呼び出し、未コミットの列挙値が使用されていないか安全性を検証します。 5. pg_enum のタプルから OID を抽出し、キャッシュを解放した後にその OID を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルでの列挙型入力処理において呼び出されます。
  • **呼び出し先**:
  • SearchSysCache2
  • check_safe_enum_use
  • ReleaseSysCache

関連データ構造

  • pg_enum: 列挙型の定義を保持するシステムカタログ。
  • HeapTuple: システムキャッシュから取得されるタプル構造。
  • Form_pg_enum: pg_enum のタプル構造体。

失敗モードと境界条件

  • **入力長超過**: strlen(name) >= NAMEDATALEN の場合、ERRCODE_INVALID_TEXT_REPRESENTATION で失敗します。
  • **値の不一致**: 指定された型に該当するラベルが存在しない場合、ERRCODE_INVALID_TEXT_REPRESENTATION で失敗します。
  • **安全性違反**: check_safe_enum_use による検証で、未コミットの列挙値が検出された場合の挙動は当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は pg_enum.oid を直接返します。この OID はバイナリアップグレード時に維持される必要があるため、変更には注意が必要です。
  • check_safe_enum_use の呼び出しは、インデックス破損を防ぐための重要なガードです。
