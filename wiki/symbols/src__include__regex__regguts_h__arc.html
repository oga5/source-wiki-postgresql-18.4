arc

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regguts.h:303

目的

正規表現のコンパイル過程で構築されるNFA（非決定性有限オートマトン）において、状態間の遷移を表現するためのデータ構造です。

シグネチャまたは形状

struct arc {
    int           type;
    color         co;
    struct state *from;
    struct state *to;
    struct arc   *outchain;
    struct arc   *outchainRev;
    struct arc   *inchain;
    struct arc   *inchainRev;
    struct arc   *colorchain;
    struct arc   *colorchainRev;
};

入力と出力

  • **入力**: NFA構築時に from（始点）および to（終点）の状態、遷移条件となる color が設定されます。
  • **出力**: 状態遷移のリンクとして、outchain（始点からの出力リスト）および inchain（終点への入力リスト）を形成します。

前提条件と不変条件

  • type が 0 の場合、そのアークは解放済み（free）とみなされます。
  • co が RAINBOW の場合、colorchain および colorchainRev フィールドは使用されません。
  • outchain は freechain としても再利用されます。

アルゴリズムまたは振る舞い

  • NFAの状態遷移グラフを構築・管理します。
  • 双方向リスト（outchain/outchainRev および inchain/inchainRev）により、特定の状態に関連するすべてのアークを高速に辿ることが可能です。
  • colorchain を通じて、特定の「色（文字クラス）」に関連付けられたすべてのアークを効率的に管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regcomp.c においてNFA構築時にインスタンス化されます。
  • **呼び出し先**: 構造体内部で struct state を参照しています。
  • **注記**: contrib/pg_trgm/trgm_regexp.c に TrgmPackedArc という類似の構造体が存在しますが、本 struct arc との直接的な呼び出し関係はソースコードから明示されていません。

関連データ構造

  • struct state: アークの始点および終点となる状態。
  • color: 遷移条件を抽象化した文字クラス。
  • struct nfa: アークと状態を保持するコンパイル時のメイン構造体。

失敗モードと境界条件

  • type の管理不備により、解放済みアークへのアクセスが発生する可能性があります。
  • メモリ確保失敗時の挙動については、本構造体単体では定義されていません。

保守担当者向けメモ

  • outchain と freechain は同一のポインタを指すマクロ定義であるため、メモリ管理時には注意が必要です。
  • colorchainRev は colorchain の逆方向リンクとして機能しますが、freechain に対応する freechainRev は存在しません。
