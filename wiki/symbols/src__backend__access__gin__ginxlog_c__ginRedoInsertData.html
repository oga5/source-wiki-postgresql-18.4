ginRedoInsertData

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginxlog.c:318

目的

GINインデックスのデータページ（リーフまたは内部ノード）に対する挿入操作をWALリプレイ時に再適用します。

シグネチャまたは形状

static void ginRedoInsertData(Buffer buffer, bool isLeaf, BlockNumber rightblkno, void *rdata)

入力と出力

  • buffer: 操作対象のページバッファ。
  • isLeaf: ページがリーフページであるかを示すフラグ。
  • rightblkno: ページ分割が発生した場合の右側ブロック番号。
  • rdata: WALレコードのデータ部（isLeaf に応じて ginxlogRecompressDataLeaf または ginxlogInsertDataInternal にキャストされる）。
  • 出力: なし（バッファの内容を直接更新）。

前提条件と不変条件

  • buffer は有効なGINデータページであること。
  • isLeaf が true の場合、ページはリーフページであること（Assert(GinPageIsLeaf(page))）。
  • isLeaf が false の場合、ページは内部ノードであること（Assert(!GinPageIsLeaf(page))）。

アルゴリズムまたは振る舞い

1. BufferGetPage を使用してバッファからページを取得します。 2. isLeaf が true の場合：

  • rdata を ginxlogRecompressDataLeaf として扱い、ginRedoRecompress を呼び出してポスティングリストの再圧縮を行います。

    • 3. isLeaf が false の場合：

  • rdata を ginxlogInsertDataInternal として扱います。
  • ページ分割後のリンクを更新するため、指定されたオフセットの PostingItem を取得し、rightblkno を設定します。
  • GinDataPageAddPostingItem を使用して、新しいアイテムをページに追加します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/gin/ginxlog.c 内のWALリプレイ処理（gin_redo 等から間接的に呼び出される）。
  • 呼び出し先: ginRedoRecompress, GinDataPageGetPostingItem, PostingItemSetBlockNumber, GinDataPageAddPostingItem。

関連データ構造

  • ginxlogRecompressDataLeaf: リーフページ再圧縮用データ構造。
  • ginxlogInsertDataInternal: 内部ノード挿入用データ構造。
  • PostingItem: GINデータページ内のポスティングアイテム構造体。

失敗モードと境界条件

  • ページタイプ（リーフ/内部）と isLeaf フラグの不一致が発生した場合、Assert により停止します。
  • rdata が不正な形式である場合、メモリ破壊や不正なページ更新を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はGINインデックスのデータページ構造に強く依存しています。ページレイアウトの変更時には ginxlog.h の定義と併せて確認が必要です。
  • ginRedoRecompress は複雑なロジックを含むため、リーフページの更新時は特に注意してください。
