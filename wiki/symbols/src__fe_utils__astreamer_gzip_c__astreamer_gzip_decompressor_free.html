astreamer_gzip_decompressor_free

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_gzip.c:363

目的

astreamer_gzip_decompressor 構造体に関連付けられたリソースを解放し、解凍処理を終了させる。

シグネチャまたは形状

static void astreamer_gzip_decompressor_free(astreamer *streamer);

入力と出力

  • **入力**: astreamer *streamer - 解放対象の astreamer_gzip_decompressor 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 streamer は astreamer_gzip_decompressor 型としてキャスト可能であること。
  • streamer->bbs_next が指す後続の astreamer は、本関数によって解放される。
  • streamer->zstream は inflateInit2 等で初期化済みであること。

アルゴリズムまたは振る舞い

1. 引数の streamer を astreamer_gzip_decompressor 型へキャストする。 2. astreamer_free(streamer->bbs_next) を呼び出し、パイプラインの後続ストリーマーを再帰的に解放する。 3. inflateEnd(&mystreamer->zstream) を呼び出し、zlib の解凍状態を破棄する。 4. pfree(streamer->bbs_buffer.data) を呼び出し、内部バッファを解放する。 5. pfree(streamer) を呼び出し、ストリーマー構造体自体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/fe_utils/astreamer_gzip.c 内の astreamer_gzip_decompressor 構造体の .free メンバとして参照されている。
  • **呼び出し先**:
  • astreamer_free
  • inflateEnd (zlib)
  • pfree

関連データ構造

  • astreamer: 基底となるストリーミングインターフェース構造体。
  • astreamer_gzip_decompressor: 解凍器固有の構造体。z_stream を保持する。

失敗モードと境界条件

  • streamer が NULL の場合、pfree 等でクラッシュする可能性がある（ソースコード上に明示的な NULL チェックは存在しない）。
  • bbs_next が NULL の場合、astreamer_free の実装依存となる。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの free コールバックとして設計されている。
  • zlib のリソース解放順序（inflateEnd）とメモリ解放順序（pfree）は、構造体の依存関係に従っている。
