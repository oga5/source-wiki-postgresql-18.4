inet_spg_inner_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_spgist.c:238

目的

SP-GiSTインデックスの内部ノードにおいて、検索条件（スキャンキー）に基づき、探索を継続すべき子ノードを特定する。

シグネチャまたは形状

Datum inet_spg_inner_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgInnerConsistentIn（検索条件、ノード情報）および spgInnerConsistentOut（結果格納用）を受け取る。
  • **出力**: out->nodeNumbers に探索対象となる子ノードのインデックスを格納し、out->nNodes にその数を設定する。戻り値は void。

前提条件と不変条件

  • in->hasPrefix が偽の場合、in->nNodes は 2 である必要がある。
  • in->hasPrefix が真かつ in->allTheSame が偽の場合、in->nNodes は 4 である必要がある。
  • in->allTheSame が真の場合、すべてのノードを探索対象とする。

アルゴリズムまたは振る舞い

1. **プレフィックスなしの場合**:

  • 2ノード構成（IPv4/IPv6の分離）を前提とし、スキャンキーの戦略（StrategyNumber）に応じてビット演算で which ビットマップを生成する。
  • 不等号演算子（<, <=, >, >=）に基づき、対象外となるファミリのノードを枝刈りする。

    • 2. **プレフィックスありの場合**:

  • 4ノード構成を前提とし、inet_spg_consistent_bitmap を呼び出して、ネットワーク演算子に基づいた詳細な枝刈りを行う。

    • 3. **共通値の場合**:

  • すべてのノードを探索対象とする。

    • 4. **結果の生成**:

  • which ビットマップに基づき、対象ノードのインデックスを out->nodeNumbers 配列にコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックスの検索処理（詳細はソースコード外のインデックスアクセスメソッド層）。
  • **呼び出し先**:
  • inet_spg_consistent_bitmap（プレフィックスがある場合のノード判定）
  • palloc（結果配列のメモリ確保）

関連データ構造

  • spgInnerConsistentIn: SP-GiSTの内部ノード整合性チェック用入力構造体。
  • spgInnerConsistentOut: 整合性チェック結果出力構造体。
  • inet: ネットワーク型データ。

失敗モードと境界条件

  • in->hasPrefix が偽かつ in->allTheSame が真となるケースは、Assert により禁止されている。
  • 探索対象ノードが 32 を超える場合は想定されていない。

保守担当者向けメモ

  • 本関数は、ネットワーク型のインデックス検索における「枝刈り」の要です。演算子ごとの which ビットマップ制御ロジックを変更する際は、inet_spg_consistent_bitmap との整合性に注意してください。
  • 呼び出し元に関する情報は、PostgreSQLのSP-GiSTアクセスメソッドの内部仕様に依存しており、本ソースコード内では明示的な呼び出し元関数は特定されていません。
