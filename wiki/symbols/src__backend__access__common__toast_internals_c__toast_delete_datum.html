toast_delete_datum

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/common/toast_internals.c:384

目的

外部ストレージ（TOASTテーブル）に保存されている巨大な属性値（チャンク）を、指定された valueid に基づいて物理的に削除する。

シグネチャまたは形状

void toast_delete_datum(Relation rel, Datum value, bool is_speculative)

入力と出力

  • **入力**:
  • rel: 親テーブルのリレーション（TOASTテーブルの特定に使用）。
  • value: 削除対象のデータ（varlena 形式の外部ポインタを含む）。
  • is_speculative: 推測的挿入（speculative insertion）の成否に関連するフラグ。
  • **出力**: なし（副作用としてTOASTテーブルからレコードが削除される）。

前提条件と不変条件

  • value が VARATT_IS_EXTERNAL_ONDISK であること。外部ディスク上に存在しないデータの場合、本関数は何もせず即座に復帰する。
  • トランザクション内で呼び出されること（get_toast_snapshot() が有効なスナップショットを返す必要がある）。

アルゴリズムまたは振る舞い

1. value から va_toastrelid および va_valueid を抽出する。 2. table_open で対象のTOASTテーブルを RowExclusiveLock で開く。 3. toast_open_indexes を使用して、TOASTテーブルのインデックス群を特定・オープンする。 4. ScanKeyInit で va_valueid をキーとして設定する。 5. systable_beginscan_ordered を用いて、該当する全チャンクをスキャンする。 6. スキャンで見つかった各タプルに対し、is_speculative が真であれば heap_abort_speculative を、偽であれば simple_heap_delete を実行して削除する。 7. スキャンを終了し、インデックスおよびTOASTテーブルをクローズする（ロックはトランザクション終了まで保持される）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/table/toast_helper.c (複数箇所)
  • **呼び出し先**:
  • table_open, table_close
  • toast_open_indexes, toast_close_indexes
  • ScanKeyInit
  • systable_beginscan_ordered, systable_getnext_ordered, systable_endscan_ordered
  • get_toast_snapshot
  • heap_abort_speculative または simple_heap_delete

関連データ構造

  • varatt_external: 外部保存されたデータのポインタ構造体。
  • Relation: テーブルおよびインデックスの管理構造体。
  • SysScanDesc: システムテーブルスキャンの状態管理。

失敗モードと境界条件

  • 外部ディスク上に存在しないデータが渡された場合、処理はスキップされる。
  • ロック取得やスキャン中にエラーが発生した場合、トランザクションの異常終了に伴いロールバックされる。

保守担当者向けメモ

  • 本関数はTOASTテーブルの物理的な削除を担うため、インデックスの整合性を維持するために toast_open_indexes で取得した有効なインデックスを使用している。
  • 削除後もロックを保持することで、同時実行される REINDEX 等の操作との競合を制御している。
