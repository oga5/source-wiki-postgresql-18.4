copy_intArrayType

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:282

目的

既存の ArrayType 構造体の内容を複製し、新しいメモリ領域にコピーした ArrayType を生成します。

シグネチャまたは形状

ArrayType *copy_intArrayType(ArrayType *a)

入力と出力

  • **入力**: a (コピー対象の ArrayType ポインタ)
  • **出力**: 複製された内容を持つ新しい ArrayType ポインタ

前提条件と不変条件

  • 入力 a は ARRNELEMS マクロで要素数が取得可能な有効な整数配列である必要があります。
  • 戻り値は palloc によってメモリ確保されているため、呼び出し側で適切に管理（pfree 等）する必要があります。

アルゴリズムまたは振る舞い

1. 入力配列 a の要素数 n を ARRNELEMS(a) で取得します。 2. new_intArrayType(n) を呼び出し、要素数 n に見合う新しい ArrayType 構造体をメモリ上に確保します。 3. memcpy を使用し、入力配列のデータポインタ (ARRPTR(a)) から新しい配列のデータポインタ (ARRPTR(r)) へ、n * sizeof(int32) バイト分をコピーします。 4. 複製された新しい配列ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c: datum_l および datum_r の生成時に呼び出し。
  • contrib/intarray/_int_tool.c: 内部処理にて呼び出し。
  • **呼び出し先**:
  • new_intArrayType (メモリ確保)
  • memcpy (標準ライブラリ)

関連データ構造

  • ArrayType: PostgreSQL の標準的な配列構造体。

失敗モードと境界条件

  • メモリ不足時には palloc (内部の new_intArrayType 経由) がエラーを発生させます。
  • 入力 a が無効なポインタや不正な配列構造である場合、ARRNELEMS や ARRPTR マクロの実行時に未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は単純なメモリコピーを行うため、配列のソート状態やメタデータはそのまま維持されます。
  • new_intArrayType が返す構造体の初期化状態に依存するため、new_intArrayType の実装変更時には本関数の動作も再確認してください。
