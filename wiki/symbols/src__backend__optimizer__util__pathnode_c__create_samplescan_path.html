create_samplescan_path

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/pathnode.c:1010

目的

テーブルスキャンにおけるサンプリング実行計画（T_SampleScan）を表す Path ノードを生成し、そのコストを算出する。

シグネチャまたは形状

Path *create_samplescan_path(PlannerInfo *root, RelOptInfo *rel, Relids required_outer)

入力と出力

  • **入力**:
  • root: クエリ全体のプランナ情報。
  • rel: 対象となるリレーション情報（RelOptInfo）。
  • required_outer: このパスが依存する外部リレーションの集合。
  • **出力**:
  • 初期化およびコスト計算済みの Path 構造体へのポインタ。

前提条件と不変条件

  • rel が有効な RelOptInfo であること。
  • get_baserel_parampathinfo により、パラメータ化されたパス情報が適切に取得されること。

アルゴリズムまたは振る舞い

1. makeNode(Path) を使用して新しい Path ノードを割り当てる。 2. pathtype を T_SampleScan に設定する。 3. parent（リレーション）、pathtarget（ターゲットリスト）を rel から継承する。 4. get_baserel_parampathinfo を呼び出し、param_info を設定する。 5. 並列実行に関するフラグ（parallel_aware = false、parallel_safe = rel->consider_parallel、parallel_workers = 0）を初期化する。 6. pathkeys を NIL に設定する（サンプルスキャンは順序を保証しないため）。 7. cost_samplescan を呼び出し、コスト計算を実行して pathnode に格納する。 8. 完成した pathnode を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c:879
  • src/backend/optimizer/util/pathnode.c:4256
  • **呼び出し先**:
  • makeNode (マクロ)
  • get_baserel_parampathinfo
  • cost_samplescan

関連データ構造

  • Path: 生成されるノードの基本構造。
  • PlannerInfo: プランナのグローバル状態。
  • RelOptInfo: スキャン対象のリレーション情報。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性がある。
  • cost_samplescan の内部実装に依存するが、コスト計算に必要な情報が rel に不足している場合は予期せぬ動作となる可能性がある。

保守担当者向けメモ

  • 本関数は T_SampleScan 用のファクトリである。並列実行には対応していない（parallel_workers = 0）。
  • pathkeys が NIL である点は、このパスがソート順序を保持しないことを明示している。変更が必要な場合は、コスト計算ロジックとの整合性に注意すること。
