PLy_function_build_args

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_exec.c:432

目的

PostgreSQL の関数呼び出し情報（FunctionCallInfo）から、Python 関数実行に必要な引数リストおよびグローバル辞書を構築する。

シグネチャまたは形状

static PyObject *PLy_function_build_args(FunctionCallInfo fcinfo, PLyProcedure *proc)

入力と出力

  • **入力**:
  • fcinfo: PostgreSQL の関数呼び出し情報。
  • proc: 実行対象の PL/Python プロシージャ情報。
  • **出力**:
  • 成功時: 引数を格納した PyList オブジェクト。
  • 失敗時: NULL（または PG_RE_THROW による例外伝播）。

前提条件と不変条件

  • proc->nargs に基づいて Python リストが割り当てられる。
  • PG_TRY ブロック内で実行されるため、エラー発生時には PG_CATCH を経由してリソースが解放される。

アルゴリズムまたは振る舞い

1. PyList_New で引数リストを初期化する。 2. proc->nargs 分ループし、各引数を PLy_input_convert で PostgreSQL の Datum から Python オブジェクトへ変換する。 3. 変換後のオブジェクトが NULL の場合は Py_None を代入する。 4. 変換したオブジェクトを Python リストに格納する。 5. proc->argnames が定義されている場合、対応する名前で proc->globals 辞書に引数を登録する。 6. 処理中にエラーが発生した場合は PG_CATCH ブロックで Py_XDECREF を呼び出し、メモリリークを防いだ上で例外を再スローする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_exec.c 内の関数実行ロジック（PLy_exec_function 等）。
  • **呼び出し先**: PLy_input_convert（型変換）、PyList_New / PyList_SetItem / PyDict_SetItemString（Python C API）。

関連データ構造

  • FunctionCallInfo: PostgreSQL 側の関数引数情報。
  • PLyProcedure: PL/Python プロシージャのメタデータ（引数名、型情報、グローバル辞書など）。

失敗モードと境界条件

  • PyList_New が失敗した場合は即座に NULL を返す。
  • PyList_SetItem や PyDict_SetItemString が失敗した場合は PLy_elog(ERROR, ...) を呼び出し、PostgreSQL のエラーメカニズムへ移行する。

保守担当者向けメモ

  • 本関数は PG_TRY ブロック内で Python オブジェクトを操作するため、Py_DECREF の漏れは致命的なメモリリークを招く。修正時は PG_CATCH ブロックの整合性を必ず確認すること。
  • proc->globals への書き込みは、再帰呼び出し時に PLy_global_args_push/pop によって保護されている前提で動作する。
