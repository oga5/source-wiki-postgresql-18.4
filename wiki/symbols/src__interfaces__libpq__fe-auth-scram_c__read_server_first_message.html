read_server_first_message

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:606

目的

SCRAM 認証プロセスにおいて、サーバーから送信された最初のメッセージを解析し、認証に必要なパラメータ（nonce、salt、反復回数）を抽出して fe_scram_state に格納する。

シグネチャまたは形状

static bool read_server_first_message(fe_scram_state *state, char *input);

入力と出力

  • **入力**:
  • state: 認証状態を保持する fe_scram_state 構造体へのポインタ。
  • input: サーバーから受信したメッセージ文字列（破壊的に変更される）。
  • **出力**:
  • 成功時は true を返し、state 内の各フィールドが更新される。
  • 失敗時は false を返し、conn->errorMessage にエラー詳細が格納される。

前提条件と不変条件

  • input は read_attr_value による解析に適した形式であること。
  • state->client_nonce が初期化済みであること。
  • 成功時、state->server_first_message、state->nonce、state->salt、state->iterations が適切に設定される。

アルゴリズムまたは振る舞い

1. input を state->server_first_message にコピーする。 2. read_attr_value を使用して、nonce (r)、salt (s)、反復回数 (i) を順次抽出する。 3. 抽出した nonce が、クライアントが送信した nonce で始まっているかを確認する（timingsafe_bcmp を使用）。 4. salt を Base64 デコードし、state->salt に格納する。 5. 反復回数を文字列から整数に変換し、1 以上であることを検証する。 6. メッセージの末尾に余分なデータが存在しないことを確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-scram.c 内の認証フロー（scram_exchange 等）から呼び出される。
  • **呼び出し先**: read_attr_value、pg_b64_dec_len、pg_b64_decode、timingsafe_bcmp、libpq_append_conn_error。

関連データ構造

  • fe_scram_state: 認証セッションの状態を保持する構造体。
  • PGconn: 接続情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足（strdup や malloc の失敗）。
  • read_attr_value による属性抽出失敗。
  • サーバー側 nonce とクライアント側 nonce の不一致。
  • Base64 デコード失敗（不正な salt）。
  • 反復回数が 1 未満、または数値として不正。
  • メッセージ末尾に予期しないデータが存在する場合。

保守担当者向けメモ

  • input 文字列は read_attr_value によって内部的に書き換えられるため、呼び出し後の再利用は避けること。
  • チャネルバインディングが有効な場合、この関数で取得したパラメータを用いて後続の calculate_client_proof が実行される。
