ParallelTransState

定義

  • 種別: enum
  • 定義箇所: src/include/replication/worker_internal.h:103

目的

論理レプリケーションにおける並列適用ワーカーのトランザクション実行状態を管理・追跡するために使用されます。

シグネチャまたは形状

typedef enum ParallelTransState
{
	PARALLEL_TRANS_UNKNOWN,
	PARALLEL_TRANS_STARTED,
	PARALLEL_TRANS_FINISHED,
} ParallelTransState;

入力と出力

  • 本シンボル自体は列挙型であり、状態遷移の値を保持します。
  • pa_get_xact_state 関数等の戻り値や、構造体メンバとして状態の受け渡しに使用されます。

前提条件と不変条件

  • トランザクションのコミット順序を制御するために使用されます。
  • 状態は UNKNOWN から始まり、処理の進行に応じて STARTED、FINISHED へと遷移することが想定されています。

アルゴリズムまたは振る舞い

  • リーダーワーカーと並列ワーカー間でのトランザクション同期において、現在のトランザクションがどの段階にあるかを識別します。
  • 並列適用ワーカーのライフサイクル管理において、トランザクションの完了状態を判定する指標となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/applyparallelworker.c 内の関数群（pa_get_xact_state 等）で状態の取得や判定に使用されています。
  • 呼び出し先: 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • ParallelApplyWorkerShared: 並列適用ワーカー間で共有される構造体であり、本状態をメンバとして保持します。
  • ParallelApplyWorkerInfo: 並列ワーカーの管理情報として本状態を参照します。

失敗モードと境界条件

  • 状態遷移が予期しない順序で行われた場合、トランザクションのコミット順序の整合性が損なわれる可能性があります。
  • PARALLEL_TRANS_UNKNOWN が継続する場合の挙動については、ソースコード上の明示的な記述が不足しています。

保守担当者向けメモ

  • 本列挙型は、並列適用ワーカーの同期ロジックにおいて重要な役割を果たします。状態遷移ロジックを変更する際は、リーダーワーカーと並列ワーカー間のコミット順序整合性に十分注意してください。
