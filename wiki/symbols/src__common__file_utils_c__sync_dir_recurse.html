sync_dir_recurse

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:235

目的

指定されたディレクトリツリーに対して、選択された同期手法（syncfs または fsync）を用いて永続化処理を再帰的に実行します。

シグネチャまたは形状

void sync_dir_recurse(const char *dir, DataDirSyncMethod sync_method)

入力と出力

  • dir: 同期対象のディレクトリパス（const char *）。
  • sync_method: 同期に使用する手法（DataDirSyncMethod 型）。
  • 出力: なし（void）。成功時は処理を完了し、失敗時はエラーログを出力してプロセスを終了する場合があります。

前提条件と不変条件

  • sync_method が DATA_DIR_SYNC_METHOD_SYNCFS の場合、プラットフォームで HAVE_SYNCFS が定義されている必要があります。
  • sync_method が DATA_DIR_SYNC_METHOD_FSYNC の場合、ディレクトリ内の各ファイルに対して fsync が実行されます。

アルゴリズムまたは振る舞い

1. sync_method に基づき分岐します。 2. DATA_DIR_SYNC_METHOD_SYNCFS の場合:

  • HAVE_SYNCFS が定義されていれば do_syncfs(dir) を呼び出します。
  • 未定義の場合はエラーを出力し、exit(EXIT_FAILURE) で終了します。

    • 3. DATA_DIR_SYNC_METHOD_FSYNC の場合:

  • PG_FLUSH_DATA_WORKS が定義されていれば、walkdir を用いて pre_sync_fname を各エントリに適用し、カーネルへフラッシュのヒントを与えます。
  • walkdir を用いて fsync_fname を各エントリに適用し、物理的な同期を完了させます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_dump/pg_backup_directory.c
  • 呼び出し先:
  • do_syncfs
  • walkdir
  • pre_sync_fname (条件付き)
  • fsync_fname

関連データ構造

  • DataDirSyncMethod: 同期手法を定義する列挙型。

失敗モードと境界条件

  • syncfs がサポートされていない環境で DATA_DIR_SYNC_METHOD_SYNCFS を指定すると、プログラムが異常終了します。
  • walkdir 実行中にアクセス権限やファイルシステムエラーが発生した場合、その挙動は walkdir の実装に依存します。

保守担当者向けメモ

  • 本関数は FRONTEND マクロが定義されている環境でコンパイルされます。
  • fsync 処理は、OSのクラッシュ耐性を確保するためにディレクトリのメタデータ同期を含めた厳格な順序で行われる必要があります。
