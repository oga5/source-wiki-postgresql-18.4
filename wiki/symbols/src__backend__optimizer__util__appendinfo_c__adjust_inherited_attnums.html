adjust_inherited_attnums

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:651

目的

親リレーションの属性番号リストを、対応する子リレーションの属性番号リストへ変換する。

シグネチャまたは形状

List *adjust_inherited_attnums(List *attnums, AppendRelInfo *context)

入力と出力

  • **入力**:
  • attnums: 変換対象となる親リレーションの属性番号（AttrNumber）のリスト。
  • context: 親子関係の変換情報を含む AppendRelInfo 構造体。
  • **出力**:
  • 変換後の子リレーションにおける属性番号のリスト。

前提条件と不変条件

  • context->parent_reloid が有効な OID であること（継承ケースであることを前提とする）。
  • context->translated_vars に、親の属性に対応する Var ノードが格納されていること。

アルゴリズムまたは振る舞い

1. 入力された attnums リストを走査する。 2. 各属性番号について、context->translated_vars を参照し、対応する Var ノードを取得する。 3. 取得した Var ノードから varattno を抽出し、結果リストに追加する。 4. 属性番号が範囲外である場合、または対応するノードが Var 型でない場合はエラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/appendinfo.c 内の関数から呼び出される（詳細はソースコード上の直接的な参照箇所を参照）。
  • **呼び出し先**: list_nth, lfirst_int, lappend_int, get_rel_name, elog 等のユーティリティ関数。

関連データ構造

  • AppendRelInfo: 変換に必要な translated_vars を保持する主要な構造体。
  • Var: 属性情報を保持するノード。

失敗モードと境界条件

  • parentattno <= 0: 属性番号が不正な場合、エラーを投げる。
  • parentattno > list_length(context->translated_vars): 属性番号が定義範囲外の場合、エラーを投げる。
  • childvar == NULL または !IsA(childvar, Var): 変換先が有効な Var ノードでない場合、エラーを投げる。

保守担当者向けメモ

  • 本関数は継承テーブルの属性変換専用であり、UNION ALL のようなケースには使用しないこと。
  • 属性の不一致はプランナの想定とカタログ定義の乖離を示すため、厳格なエラーチェックを行っている。
