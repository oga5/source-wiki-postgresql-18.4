fix_windowagg_cond_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:86

目的

WindowAgg ノードの条件式（runcondition）に含まれる変数参照を、サブプランの出力タプルに対するインデックスへ変換する際のコンテキスト情報を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    PlannerInfo *root;
    indexed_tlist *subplan_itlist;
    int newvarno;
} fix_windowagg_cond_context;

入力と出力

本構造体は関数への引数として渡されるコンテキストオブジェクトです。

  • root: プランナのグローバル情報へのポインタ。
  • subplan_itlist: 変数解決に使用される、サブプランのターゲットリストをインデックス化した構造体。
  • newvarno: 変換後の変数参照に使用される新しい varno（通常は INNER_VAR 等）。

前提条件と不変条件

  • subplan_itlist は、対象となるサブプランの出力に基づき適切に初期化されている必要があります。
  • root は有効な PlannerInfo 構造体を指している必要があります。

アルゴリズムまたは振る舞い

set_plan_references の処理過程において、WindowAgg ノードの条件式を走査する際に、この構造体を介して式ツリー内の Var ノードを物理的な位置参照へ書き換えます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/plan/setrefs.c:3477 にてインスタンス化され、同ファイル内の関数（3444 行目付近）へ渡されます。
  • 呼び出し先: 構造体自体はデータ保持用であり、関数呼び出しの引数として利用されます。

関連データ構造

  • PlannerInfo: プラン作成全体の情報を保持。
  • indexed_tlist: サブプランのターゲットリストを高速検索するための構造体。

失敗モードと境界条件

  • subplan_itlist が不適切な場合、式内の変数参照の解決に失敗し、プランの実行時に不正なタプルアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は WindowAgg ノード特有の条件式処理に特化しています。他のノードの式変換コンテキスト（fix_scan_expr_context 等）と混同しないよう注意してください。
