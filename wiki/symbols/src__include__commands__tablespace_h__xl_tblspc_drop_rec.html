xl_tblspc_drop_rec

定義

  • 種別: struct
  • 定義箇所: src/include/commands/tablespace.h:36

目的

テーブルスペースの削除操作をWAL（Write-Ahead Logging）に記録するための物理レコード構造体です。

シグネチャまたは形状

typedef struct xl_tblspc_drop_rec
{
    Oid    ts_id;
} xl_tblspc_drop_rec;

入力と出力

  • **入力**: 削除対象のテーブルスペースを識別する Oid 型の ts_id を保持します。
  • **出力**: WALログへの書き込みデータとして使用されます。

前提条件と不変条件

  • ts_id は削除されるテーブルスペースの有効なOIDである必要があります。
  • WALリプレイ時には、この構造体に含まれる ts_id を基に物理的なディレクトリ削除処理が再現されます。

アルゴリズムまたは振る舞い

1. **記録**: テーブルスペース削除コマンド実行時に、削除対象のOIDを本構造体に格納し、XLogRegisterData を介してWALレコードとして登録します。 2. **リプレイ**: クラッシュリカバリ時、XLogRecGetData を通じてこの構造体を読み取り、対応するテーブルスペースの物理ディレクトリを削除する処理へ渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablespace.c 内のテーブルスペース削除処理（DropTableSpace 等）から、WAL記録のために使用されます。
  • **呼び出し先**: src/backend/access/rmgrdesc/tblspcdesc.c において、WALレコードのデコードおよび説明文生成のために参照されます。

関連データ構造

  • xl_tblspc_create_rec: 対となるテーブルスペース作成時のWALレコード構造体。

失敗モードと境界条件

  • 構造体自体には失敗モードはありませんが、リプレイ時に ts_id に対応するディレクトリが存在しない場合や、ファイルシステム権限により削除が拒否される場合はリカバリ処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体はWALの物理フォーマットの一部であるため、フィールドの追加や変更を行う際は、WALの互換性を維持するために tblspcdesc.c のデコードロジックも併せて更新する必要があります。
