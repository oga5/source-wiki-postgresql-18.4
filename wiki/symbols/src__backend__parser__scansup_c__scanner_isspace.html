scanner_isspace

定義

  • 種別: function
  • 定義箇所: src/backend/parser/scansup.c:116

目的

PostgreSQL の字句解析器（Flex）が定義する空白文字のリストと完全に一致する判定処理を提供し、ロケール依存の標準 isspace() を避けて一貫した字句解析を保証する。

シグネチャまたは形状

bool scanner_isspace(char ch)

入力と出力

  • **入力**: char ch（判定対象の文字）
  • **出力**: bool（対象文字が字句解析器の定義する空白文字であれば true、そうでなければ false）

前提条件と不変条件

  • 字句解析器の定義ファイル src/backend/parser/scan.l 内の {space} 定義と常に同期している必要がある。

アルゴリズムまたは振る舞い

引数 ch が以下のいずれかの文字と一致するかを判定する。

  • 空白 (' ')
  • 水平タブ ('\t')
  • 改行 ('\n')
  • 復帰 ('\r')
  • 垂直タブ ('\v')
  • 改ページ ('\f')

いずれかに該当する場合に true を返し、それ以外は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_io.c
  • src/backend/commands/schemacmds.c
  • src/backend/nodes/queryjumblefuncs.c
  • src/backend/parser/parser.c
  • src/backend/utils/adt/arrayfuncs.c
  • **呼び出し先**:
  • 特になし（組み込みの比較演算のみを使用）。

関連データ構造

  • なし

失敗モードと境界条件

  • 負の文字コードが渡された場合の挙動は、呼び出し側で (unsigned char) にキャストする等の適切な処理が行われていることを前提とする。

保守担当者向けメモ

  • scan.l で空白の定義が変更された場合、本関数も必ず更新すること。
  • ロケール設定に依存しない判定が必要な箇所でのみ使用すること。
