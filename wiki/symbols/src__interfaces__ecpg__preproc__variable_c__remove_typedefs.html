remove_typedefs

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/variable.c:263

目的

指定されたスコープ（brace_level）以上の階層で定義された typedef 情報をシンボルテーブルから削除し、関連するメモリを解放します。

シグネチャまたは形状

void remove_typedefs(int brace_level)

入力と出力

  • **入力**: brace_level (int) - 削除対象となるスコープの深さ。
  • **出力**: なし（副作用としてグローバルな types リストを更新）。

前提条件と不変条件

  • グローバル変数 types が typedef のリンクリストを保持していること。
  • brace_level は、各 typedef エントリが保持する brace_level と比較され、等しいかそれ以上のものが削除対象となります。

アルゴリズムまたは振る舞い

1. types リストを先頭から走査します。 2. 各エントリの brace_level が引数の brace_level 以上である場合、リストから当該ノードを切り離します。 3. 構造体（ECPGt_struct）または共用体（ECPGt_union）の場合は、ECPGfree_struct_member を呼び出してメンバリストを解放します。 4. 型情報（type_storage, type_str, type_dimension, type_index, type_sizeof）、型構造体自体、名前、およびノード本体を順次 free します。 5. 削除対象でない場合は、ポインタを保持して走査を継続します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に明示的な呼び出し箇所は提供されていません。
  • **呼び出し先**: ECPGfree_struct_member（構造体/共用体の場合）、free（標準ライブラリ）。

関連データ構造

  • typedefs: typedef 情報を保持するリンクリスト構造体。
  • ECPGtype: 型情報を保持する構造体。

失敗モードと境界条件

  • types が空の場合、ループは実行されず正常に終了します。
  • 構造体メンバの解放時に ECPGfree_struct_member が適切に処理を行う必要があります。

保守担当者向けメモ

  • 本関数は types リストを直接操作するため、リストの整合性を維持するために prev ポインタの更新が重要です。
  • 構造体メンバの解放ロジックは type.c の実装に依存しているため、型定義の変更時には注意が必要です。
