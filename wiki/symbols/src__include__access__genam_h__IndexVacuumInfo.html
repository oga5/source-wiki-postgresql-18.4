IndexVacuumInfo

定義

  • 種別: struct
  • 定義箇所: src/include/access/genam.h:67

目的

インデックスの保守操作（VACUUM）において、対象インデックスおよび関連するヒープリレーションの情報、ならびに実行時の制御パラメータを保持するために使用されます。

シグネチャまたは形状

typedef struct IndexVacuumInfo
{
    Relation    index;              /* the index being vacuumed */
    Relation    heaprel;            /* the heap relation the index belongs to */
    bool        analyze_only;       /* ANALYZE (without any actual vacuum) */
    bool        report_progress;    /* emit progress.h status reports */
    bool        estimated_count;    /* num_heap_tuples is an estimate */
    int         message_level;      /* ereport level for progress messages */
    double      num_heap_tuples;    /* tuples remaining in heap */
    BufferAccessStrategy strategy;  /* access strategy for reads */
} IndexVacuumInfo;

入力と出力

本構造体は、インデックスAM（アクセス手法）の保守関数（amvacuumcleanup や ambulkdelete 等）への入力引数として渡されます。戻り値としては直接使用されず、処理結果は別途 IndexBulkDeleteResult 構造体に格納されます。

前提条件と不変条件

  • index および heaprel は、処理開始前に適切にオープンされた Relation である必要があります。
  • strategy は、バッファ管理のための BufferAccessStrategy を指す必要があります。

アルゴリズムまたは振る舞い

インデックスの不要領域回収（VACUUM）プロセスにおいて、以下の制御を行います。 1. **実行モードの指定**: analyze_only フラグにより、実際の削除処理を行わず統計情報の収集のみを行うかを選択します。 2. **進捗報告**: report_progress が真の場合、message_level に基づいて進捗状況をログ出力します。 3. **統計情報の活用**: num_heap_tuples を用いて、ヒープ内の残存タプル数に基づいた最適化を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: 主に VACUUM コマンド実行時に、各インデックスAMの保守ルーチンを呼び出す上位レイヤー（vacuum.c 等）から生成・設定されます。
  • **呼び出し先**: 以下の各インデックスAM実装が本構造体を受け取ります。
  • contrib/bloom: blbulkdelete, blvacuumcleanup
  • src/backend/access/brin: brinbulkdelete, brinvacuumcleanup
  • src/backend/access/gin: ginbulkdelete, ginvacuumcleanup
  • src/backend/access/gist: gistvacuumscan, gistvacuum_delete_empty_pages, gistdeletepage

関連データ構造

  • Relation: 対象となるインデックスおよびヒープのキャッシュ情報。
  • BufferAccessStrategy: バッファ読み込み時の戦略。
  • IndexBulkDeleteResult: 本構造体を用いた処理の結果として返される統計情報。

失敗モードと境界条件

  • index や heaprel が無効なポインタである場合、セグメンテーション違反が発生します。
  • num_heap_tuples に負の値が設定された場合の挙動は、各AMの実装に依存します。

保守担当者向けメモ

  • 本構造体は genam.h で定義されており、インデックスAMのインターフェースの一部です。変更を加える際は、すべてのAM実装（bloom, brin, gin, gist 等）への影響を確認してください。
