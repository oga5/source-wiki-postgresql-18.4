CreateEmptyBlockRefTable

定義

  • 種別: function
  • 定義箇所: src/common/blkreftable.c:234

目的

変更されたブロックを追跡するための空の BlockRefTable 構造体を初期化し、メモリを割り当てる。

シグネチャまたは形状

BlockRefTable *CreateEmptyBlockRefTable(void)

入力と出力

  • **入力**: なし
  • **出力**: 初期化された BlockRefTable 構造体へのポインタ。

前提条件と不変条件

  • 呼び出し元は、返された BlockRefTable 構造体のメモリを管理する責任を負う。
  • FRONTEND 定義の有無により、ハッシュテーブルの初期化方法（メモリコンテキストの指定）が分岐する。

アルゴリズムまたは振る舞い

1. palloc を使用して BlockRefTable 構造体分のメモリを確保する。 2. ハッシュテーブルの初期サイズを 4096 エントリと想定して blockreftable_create を呼び出す。 3. バックエンド環境（FRONTEND 未定義）の場合、現在のメモリコンテキスト (CurrentMemoryContext) を構造体に保持する。 4. 初期化済みの構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c:571
  • src/backend/postmaster/walsummarizer.c:920
  • **呼び出し先**:
  • palloc (メモリ確保)
  • blockreftable_create (ハッシュテーブル初期化)

関連データ構造

  • BlockRefTable: 本関数が生成するメイン構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • ハッシュテーブルの初期サイズ（4096）は、空のデータベースであっても数百のリレーションフォークが存在することを考慮した経験的な値である。
