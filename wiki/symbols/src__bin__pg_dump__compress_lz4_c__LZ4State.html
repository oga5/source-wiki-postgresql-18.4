LZ4State

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:39

目的

pg_dump および pg_restore における LZ4 圧縮・解凍処理のコンテキスト、ストリーム状態、およびバッファを保持する構造体です。

シグネチャまたは形状

typedef struct LZ4State
{
    FILE                       *fp;
    LZ4F_preferences_t          prefs;
    LZ4F_compressionContext_t   ctx;
    LZ4F_decompressionContext_t dtx;
    bool                        inited;
    bool                        compressing;
    bool                        needs_header_flush;
    size_t                      buflen;
    char                       *buffer;
    size_t                      overflowalloclen;
    size_t                      overflowlen;
    char                       *overflowbuf;
    size_t                      compressedlen;
    size_t                      errcode;
} LZ4State;

入力と出力

  • **入力**: InitCompressorLZ4 や InitCompressFileHandleLZ4 等の初期化関数を通じて割り当てられ、各ストリーム操作関数へ渡されます。
  • **出力**: 圧縮・解凍の進捗、エラーコード、およびバッファリングされたデータが本構造体内に保持されます。

前提条件と不変条件

  • inited フラグにより、ストリームの遅延初期化状態を管理します。
  • compressing フラグにより、圧縮操作と解凍操作を区別します。
  • 解凍時、一度の LZ4F_decompress で得られたデータが要求サイズを超えた場合、残余データは overflowbuf に保持されます。

アルゴリズムまたは振る舞い

  • **ストリーム管理**: fp を介してファイルストリームを追跡します。
  • **バッファリング**: 圧縮・解凍時のチャンク処理および、未消費データを保持するオーバーフローバッファを管理します。
  • **エラー追跡**: errcode フィールドにより、各操作におけるエラー状態を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: InitCompressorLZ4, InitCompressFileHandleLZ4, LZ4Stream_read_internal, LZ4Stream_write, LZ4State_compression_init, LZ4Stream_read_overflow 等の関数から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • LZ4F_preferences_t (LZ4 圧縮設定)
  • LZ4F_compressionContext_t (圧縮コンテキスト)
  • LZ4F_decompressionContext_t (解凍コンテキスト)
  • CompressorState / CompressFileHandle (本構造体を private_data として保持する上位インターフェース)

失敗モードと境界条件

  • errcode フィールドによりエラーを記録します。
  • overflowbuf の管理において、メモリ割り当てやバッファサイズの境界条件が発生する可能性があります。

保守担当者向けメモ

  • USE_LZ4 が定義されていない環境では、本構造体を利用する初期化関数はスタブとして動作します。
  • LZ4Stream 系関数は、最初の読み書き時にコンテキストを生成する遅延初期化を採用しています。
