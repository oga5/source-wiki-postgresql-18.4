PG_STAT_GET_DBENTRY_FLOAT8_MS

定義

  • 種別: macro (関数定義用テンプレート)
  • 定義箇所: src/backend/utils/adt/pgstatfuncs.c:1211

目的

データベース統計情報（PgStat_StatDBEntry）から、ミリ秒単位の浮動小数点数（float8）で保持されている特定の統計項目を抽出し、SQLの Datum 型として返却する関数を生成するためのテンプレートです。

シグネチャまたは形状

PG_STAT_GET_DBENTRY_FLOAT8_MS(stat)
  • stat: 抽出対象となる統計フィールド名（例: blk_read_time, session_time 等）。

入力と出力

  • **入力**: 呼び出し時に指定された統計フィールド名。
  • **出力**: Datum 型（SQLの float8 に対応）。

前提条件と不変条件

  • 呼び出し元は、pgstat_fetch_stat_dbentry() を介してデータベース統計情報が取得可能である必要があります。
  • 統計情報システムが初期化されており、対象のデータベース統計が存在することが前提です。

アルゴリズムまたは振る舞い

1. 指定された統計フィールド名に基づき、関数を定義します。 2. 実行時、pgstat_fetch_stat_dbentry() を呼び出して現在のデータベース統計スナップショットを取得します。 3. 取得した構造体から、引数 stat で指定されたフィールドの値を参照します。 4. PG_RETURN_FLOAT8() マクロを使用して、値を Datum 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c 内で、以下の統計取得関数を定義するために使用されています。
  • pg_stat_get_db_blk_read_time
  • pg_stat_get_db_blk_write_time
  • pg_stat_get_db_idle_in_transaction_time
  • pg_stat_get_db_session_time
  • **呼び出し先**:
  • pgstat_fetch_stat_dbentry()（統計情報の取得）

関連データ構造

  • PgStat_StatDBEntry: データベースごとの統計情報を保持する構造体。

失敗モードと境界条件

  • 統計情報が取得できない場合（例: データベースIDが無効な場合）、pgstat_fetch_stat_dbentry() が NULL を返す可能性があり、その場合はヌルポインタ参照が発生する可能性があります。
  • 統計データがまだ収集されていない場合、初期値（通常は0）が返されます。

保守担当者向けメモ

  • 本マクロは pgstatfuncs.c 内での定型的な統計取得処理を簡素化するためのものです。新しい統計項目を追加する際は、既存のパターンに従ってください。
  • 統計情報はスナップショットベースであるため、関数呼び出しのタイミングによって値が変動する可能性がある点に留意してください。
