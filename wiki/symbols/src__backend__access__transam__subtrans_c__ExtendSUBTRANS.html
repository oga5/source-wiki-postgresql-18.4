ExtendSUBTRANS

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/subtrans.c:378

目的

新しいトランザクション ID (XID) がサブトランザクションログ（SUBTRANS）の新しいページを必要とする際に、そのページをゼロクリアして初期化します。

シグネチャまたは形状

void ExtendSUBTRANS(TransactionId newestXact)

入力と出力

  • **入力**: newestXact (新しく割り当てられたトランザクション ID)
  • **出力**: なし（副作用として SUBTRANS のページが初期化される）

前提条件と不変条件

  • newestXact がページ境界（TransactionIdToEntry(newestXact) == 0）にあるか、または FirstNormalTransactionId である場合にのみ処理が実行されます。
  • ページ操作には SubTransCtl に関連付けられた排他ロック（LW_EXCLUSIVE）が必要です。

アルゴリズムまたは振る舞い

1. 指定された newestXact がページ内の最初の XID であるかを確認します。条件を満たさない場合は即座に終了します。 2. TransactionIdToPage を使用して、対象となるページ番号を算出します。 3. SimpleLruGetBankLock を呼び出し、対象ページに対する排他ロックを取得します。 4. ZeroSUBTRANSPage を呼び出し、該当ページをゼロクリアします。 5. ロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/subtrans.c (内部的なページ拡張時)
  • src/backend/access/transam/varsup.c
  • src/backend/storage/ipc/procarray.c
  • **呼び出し先**:
  • TransactionIdToEntry
  • TransactionIdEquals
  • TransactionIdToPage
  • SimpleLruGetBankLock
  • LWLockAcquire / LWLockRelease
  • ZeroSUBTRANSPage

関連データ構造

  • SubTransCtl: SLRU 管理のための共有メモリ制御構造体。

失敗モードと境界条件

  • **ラップアラウンド**: FirstNormalTransactionId の直後の処理において、ページゼロの初期化が正しく行われるよう考慮されています。
  • **ロック競合**: ページに対する排他ロック取得時に他のプロセスと競合する可能性があります。

保守担当者向けメモ

  • 本関数はクラッシュリカバリを目的としておらず、再起動時には SUBTRANS のデータはゼロクリアされる前提で設計されています。
  • ページ境界の判定ロジックは TransactionIdToEntry に依存しているため、SLRU のページサイズ変更時には注意が必要です。
