hstore_from_array

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_io.c:718

目的

テキスト型の配列（1次元または2次元）を hstore 構造体に変換します。

シグネチャまたは形状

Datum hstore_from_array(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して TEXT 型の ArrayType を受け取ります。
  • **出力**: hstore 構造体へのポインタを Datum として返します。

前提条件と不変条件

  • 入力配列の要素型は TEXTOID である必要があります。
  • 1次元配列の場合、要素数は偶数である必要があります。
  • 2次元配列の場合、第2次元のサイズは 2 である必要があります。
  • キーに NULL を含めることはできません。

アルゴリズムまたは振る舞い

1. 配列の次元数を確認し、不正な場合はエラーを報告します。 2. deconstruct_array_builtin を使用して配列を Datum 配列に分解します。 3. ペアの数を計算し、メモリを確保します。 4. 各要素を走査し、キーと値のペアを構築します。値が NULL の場合は isnull フラグを立てます。 5. hstoreUniquePairs を呼び出し、キーの重複排除とソートを行います。 6. hstorePairs を呼び出し、最終的な hstore 構造体を生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(hstore_from_array) マクロにより、PostgreSQL の関数マネージャから呼び出されます。
  • **呼び出し先**: deconstruct_array_builtin, hstoreCheckKeyLen, hstoreCheckValLen, hstoreUniquePairs, hstorePairs を呼び出します。

関連データ構造

  • ArrayType: 入力データ構造。
  • Pairs: キーと値のペアを保持する中間構造体。
  • HStore: 最終的な出力データ構造。

失敗モードと境界条件

  • **エラー**:
  • 配列の次元が 0, 1, 2 以外の場合。
  • 1次元配列の要素数が奇数の場合。
  • 2次元配列の第2次元が 2 以外の場合。
  • キーが NULL の場合。
  • ペア数が MaxAllocSize を超える場合。

保守担当者向けメモ

  • hstoreUniquePairs 実行時に buflen が計算されます。
  • hstoreCheckKeyLen および hstoreCheckValLen により、各要素のサイズ制限が検証されます。
