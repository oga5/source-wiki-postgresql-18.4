ExecInitBitmapOr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapOr.c:55

目的

BitmapOr ノードの初期化を行い、保持するすべてのサブプランに対して再帰的に初期化処理を実行する。

シグネチャまたは形状

BitmapOrState *ExecInitBitmapOr(BitmapOr *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の BitmapOr プランノード。
  • estate: 実行状態を管理する EState オブジェクト。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの BitmapOrState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと（アサーションで検証）。
  • BitmapOr ノードは ExecProcNode を呼び出さないため、式コンテキストやタプルスロットの初期化は行わない。

アルゴリズムまたは振る舞い

1. BitmapOrState 構造体を割り当て、プランノードおよび実行状態をセットアップする。 2. BitmapOr ノードが保持するサブプランの数 (nplans) を取得し、サブプラン状態を格納するための配列を確保する。 3. node->bitmapplans リストを走査し、各サブプランに対して ExecInitNode を再帰的に呼び出す。 4. 初期化した各サブプランの状態を bitmapplanstates 配列に保存する。 5. ps.ExecProcNode に ExecBitmapOr を割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode から呼び出される。
  • **呼び出し先**: 各サブプランに対して ExecInitNode を呼び出す。

関連データ構造

  • BitmapOr: プランツリー上のノード定義。
  • BitmapOrState: 実行時の状態を保持する構造体。
  • PlanState: サブプランの実行状態を保持する配列。

失敗モードと境界条件

  • eflags にサポート外のフラグが指定された場合、アサーション違反により停止する。
  • メモリ不足時は palloc0 がエラーを発生させる。

保守担当者向けメモ

  • 本関数は ExecProcNode をサポートしないため、ps.ExecProcNode にはエラーを投げるスタブ関数 ExecBitmapOr が設定される。
  • サブプランの初期化順序はリストの順序に従う。拡張時は ExecInitNode の再帰呼び出しが正しく行われているか確認すること。
