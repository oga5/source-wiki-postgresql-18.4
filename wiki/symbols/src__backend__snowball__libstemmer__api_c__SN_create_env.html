SN_create_env

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/api.c:3

目的

Snowball ステミングアルゴリズムを実行するための環境構造体 SN_env を動的に確保し、初期化します。

シグネチャまたは形状

struct SN_env * SN_create_env(int S_size, int I_size)

入力と出力

  • **入力**:
  • S_size: 確保する文字列ポインタ配列 S の要素数。
  • I_size: 確保する整数配列 I の要素数。
  • **出力**:
  • 成功時: 初期化された SN_env 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • SN_env 構造体の各フィールド（p, S, I）は、メモリ確保に成功した場合にのみ有効な値が割り当てられます。
  • 確保された環境は、使用後に必ず SN_close_env で解放する必要があります。

アルゴリズムまたは振る舞い

1. calloc を使用して SN_env 構造体本体をゼロ初期化して確保します。 2. create_s() を呼び出し、メインの文字列バッファ p を生成します。 3. S_size が指定されている場合、S 配列を確保し、各要素に対して create_s() を呼び出して個別の文字列バッファを生成します。 4. I_size が指定されている場合、整数配列 I を確保します。 5. 上記のいずれかのステップでメモリ確保に失敗した場合、error ラベルへ遷移し、SN_close_env を呼び出して確保済みのメモリを解放した後に NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 各言語固有のステミング環境生成関数（例: basque_ISO_8859_1_create_env, english_ISO_8859_1_create_env 等）。
  • **呼び出し先**: calloc (標準ライブラリ), create_s (内部ユーティリティ), SN_close_env (エラー時のクリーンアップ用)。

関連データ構造

  • SN_env: ステミング処理の実行状態を保持する構造体。
  • symbol: header.h で定義される文字型。

失敗モードと境界条件

  • メモリ不足により calloc または create_s が NULL を返した場合、関数は NULL を返します。
  • S_size や I_size が 0 の場合、対応する配列の確保はスキップされます。

保守担当者向けメモ

  • SN_env 構造体内の p はメインの文字列バッファ、S はアルゴリズム内で一時的に利用される文字列配列を指します。
  • header.h で定義される symbol 型のサイズがシステム全体のメモリ配置に影響を与えるため、変更時は注意してください。
