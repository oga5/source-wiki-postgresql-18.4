pgaio_io_release

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio.c:239

目的

現在バックエンドに割り当てられている非同期I/Oハンドル（PgAioHandle）を解放し、再利用可能な状態に戻す。

シグネチャまたは形状

void pgaio_io_release(PgAioHandle *ioh)

入力と出力

  • **入力**: ioh - 解放対象の PgAioHandle 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • ioh は、呼び出し元バックエンドの handed_out_io と一致している必要がある。
  • ioh->state は PGAIO_HS_HANDED_OUT であること。
  • ioh->resowner が設定されていること。

アルゴリズムまたは振る舞い

1. 引数 ioh が現在のバックエンドの handed_out_io と一致するか確認する。 2. 一致する場合、pgaio_my_backend->handed_out_io を NULL にリセットする。 3. pgaio_io_reclaim(ioh) を呼び出し、ハンドルを解放する。 4. 割り込みによる再帰的な解放を防ぐため、チェックから pgaio_io_reclaim 呼び出しまでの間、割り込み処理が発生しないことが重要である。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio.c
  • src/backend/storage/buffer/bufmgr.c:1874
  • src/test/modules/test_aio/test_aio.c:623, 655
  • **呼び出し先**:
  • pgaio_io_reclaim (内部関数)

関連データ構造

  • PgAioHandle: I/O操作の状態を保持するハンドル構造体。
  • pgaio_my_backend: 現在のバックエンドのAIO状態を管理するグローバル変数。

失敗モードと境界条件

  • ioh が現在のバックエンドの handed_out_io と一致しない場合、elog(ERROR) によりエラーが発生する。

保守担当者向けメモ

  • 本関数は状態遷移を伴うため、呼び出し時には割り込みが禁止されているか、割り込みが発生しない文脈であることを確認すること。
  • pgaio_io_reclaim への引き渡し前に handed_out_io をクリアする順序は、競合を防ぐために必須である。
