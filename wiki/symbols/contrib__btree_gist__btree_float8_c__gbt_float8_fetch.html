gbt_float8_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float8.c:129

目的

GiSTインデックス内の float8 型データに対して、インデックスエントリから元の値をフェッチ（取得）します。

シグネチャまたは形状

Datum gbt_float8_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、インデックスエントリから抽出された float8 値を返します。

前提条件と不変条件

  • tinfo（gbtree_ninfo 構造体）が contrib/btree_gist/btree_float8.c 内で適切に初期化されている必要があります。
  • 入力される GISTENTRY は、GiSTインデックスの有効なエントリである必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. gbt_num_fetch 関数を呼び出し、entry と float8 型用の設定オブジェクト tinfo を渡します。 3. gbt_num_fetch の戻り値をそのまま PG_RETURN_POINTER を介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックス管理システムから呼び出されます（直接的な呼び出し元コードは提供されていません）。
  • **呼び出し先**: gbt_num_fetch（btree_utils_num.c で定義された汎用数値フェッチ関数）。

関連データ構造

  • GISTENTRY: GiSTインデックスのエントリを保持する構造体。
  • tinfo: float8 型の比較や距離計算の定義を含む gbtree_ninfo 構造体。

失敗モードと境界条件

  • entry が NULL の場合、あるいは GISTENTRY の内容が不正な場合は gbt_num_fetch 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する汎用的なフェッチロジックのラッパーです。
  • float8 型固有のロジックは tinfo に集約されているため、本関数の実装自体に変更が必要になることは稀です。
