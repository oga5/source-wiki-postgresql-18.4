CreatePartitionDirectory

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partdesc.c:422

目的

クエリ実行中に複数のリレーションに対するパーティション記述子を一括管理するための PartitionDirectory 構造体を初期化・生成します。

シグネチャまたは形状

PartitionDirectory CreatePartitionDirectory(MemoryContext mcxt, bool omit_detached)

入力と出力

  • **入力**:
  • mcxt: 構造体および内部ハッシュテーブルの割り当てに使用するメモリコンテキスト。
  • omit_detached: デタッチ済みパーティションを記述子から除外するかどうかを指定するフラグ。
  • **出力**:
  • 初期化された PartitionDirectory 構造体へのポインタ。

前提条件と不変条件

  • 指定された mcxt が有効である必要があります。
  • 内部ハッシュテーブルは mcxt 上に作成され、キーとして Oid を使用します。

アルゴリズムまたは振る舞い

1. mcxt をカレントメモリコンテキストに切り替えます。 2. PartitionDirectoryData 構造体を割り当て、引数で受け取った mcxt と omit_detached を格納します。 3. hash_create を呼び出し、サイズ 256 のハッシュテーブルを生成します。このハッシュテーブルは HASH_ELEM、HASH_BLOBS、HASH_CONTEXT フラグで構成されます。 4. 元のメモリコンテキストを復元し、生成した PartitionDirectory を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execPartition.c
  • src/backend/optimizer/util/plancat.c
  • **呼び出し先**:
  • MemoryContextSwitchTo
  • palloc
  • hash_create

関連データ構造

  • PartitionDirectoryData: 本関数で生成されるメインの管理構造体。
  • PartitionDirectoryEntry: ハッシュテーブルの各エントリの型。

失敗モードと境界条件

  • メモリ不足時には palloc または hash_create 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数で作成された PartitionDirectory は、クエリ実行終了後に DestroyPartitionDirectory を用いて適切に解放する必要があります。
  • omit_detached フラグは、パーティションの動的な再構築が必要な場合にキャッシュの挙動を制御する重要なパラメータです。
