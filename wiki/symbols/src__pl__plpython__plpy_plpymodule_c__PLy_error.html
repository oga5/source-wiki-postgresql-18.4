PLy_error

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:307

目的

Python スクリプトから呼び出された際に、PostgreSQL の ERROR レベルでログ出力（およびエラー報告）を行うためのインターフェースを提供します。

シグネチャまたは形状

static PyObject *PLy_error(PyObject *self, PyObject *args, PyObject *kw)

入力と出力

  • **入力**:
  • self: Python モジュールオブジェクト。
  • args: Python から渡された可変長引数（ログメッセージの構成要素）。
  • kw: Python から渡されたキーワード引数。
  • **出力**:
  • PLy_output の戻り値（通常は PyObject* 型の None またはエラーを示す NULL）。

前提条件と不変条件

  • 本関数は Python の C API 規約に従う必要があります。
  • 内部で呼び出される PLy_output が PostgreSQL の ereport(ERROR, ...) を実行するため、本関数が正常に終了することはありません（PostgreSQL のエラー処理メカニズムにより、現在のトランザクションが中断されます）。

アルゴリズムまたは振る舞い

1. 引数として受け取った self, args, kw をそのまま PLy_output 関数へ渡します。 2. PLy_output の第一引数に ERROR を指定することで、PostgreSQL のエラー報告システムを起動します。

呼び出し元と呼び出し先

  • **呼び出し元**: Python インタプリタ（plpy.error() メソッド経由）。
  • **呼び出し先**: PLy_output（src/pl/plpython/plpy_elog.c に定義）。

関連データ構造

  • PyObject: Python のオブジェクト型。
  • ERROR: PostgreSQL のログレベル定数。

失敗モードと境界条件

  • PLy_output 内部で引数の解析に失敗した場合、Python の例外が設定され NULL が返されます。
  • PostgreSQL 側でエラーが発生した場合、ereport により制御が呼び出し元（Python 側）へ戻らず、PostgreSQL のエラーハンドラへジャンプします。

保守担当者向けメモ

  • 本関数は plpy モジュールのメソッドテーブル（PLy_methods）に登録されており、plpy.error として公開されています。
  • ログ出力のロジックは PLy_output に集約されているため、動作変更が必要な場合は PLy_output を参照してください。
