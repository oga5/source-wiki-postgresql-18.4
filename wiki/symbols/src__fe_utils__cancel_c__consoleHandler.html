consoleHandler

定義

  • 種別: function
  • 定義箇所: src/fe_utils/cancel.c:194

目的

Windows環境において、コンソールからの割り込みイベント（Ctrl+C や Ctrl+Break）を捕捉し、実行中のデータベースクエリに対してキャンセル要求を送信する。

シグネチャまたは形状

static BOOL WINAPI consoleHandler(DWORD dwCtrlType)

入力と出力

  • **入力**: dwCtrlType (DWORD型) - 発生したコンソールイベントの種類。
  • **出力**: BOOL - イベントを処理した場合は TRUE、それ以外は FALSE を返す。

前提条件と不変条件

  • Windows環境でのみ動作する。
  • cancelConnLock（CRITICAL_SECTION）が初期化されている必要がある。
  • cancelConn が有効な場合、PQcancel を呼び出すための排他制御が保証されている必要がある。

アルゴリズムまたは振る舞い

1. dwCtrlType が CTRL_C_EVENT または CTRL_BREAK_EVENT であるかを確認する。 2. 該当する場合、CancelRequested フラグを true に設定する。 3. 登録されている cancel_callback があれば実行する。 4. cancelConnLock を用いて排他制御を行い、cancelConn が存在すれば PQcancel を呼び出してバックエンドへキャンセル信号を送信する。 5. PQcancel の成否に応じて、cancel_sent_msg または cancel_not_sent_msg を write_stderr で出力する。 6. 処理終了後、ロックを解放して TRUE を返す。 7. 上記以外のイベントに対しては FALSE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Windowsのコンソールイベントハンドラとして OS から呼び出される。src/fe_utils/cancel.c 内の setup_cancel_handler 等で SetConsoleCtrlHandler に登録される。
  • **呼び出し先**:
  • cancel_callback（登録されている場合）
  • EnterCriticalSection / LeaveCriticalSection
  • PQcancel
  • write_stderr

関連データ構造

  • cancelConn: キャンセル対象の接続を保持するグローバル変数。
  • cancelConnLock: 排他制御用の CRITICAL_SECTION。
  • CancelRequested: キャンセル要求状態を示すフラグ。
  • cancel_callback: 割り込み時に実行されるコールバック関数ポインタ。

失敗モードと境界条件

  • cancelConn が NULL の場合、キャンセル要求の送信はスキップされる。
  • PQcancel が失敗した場合、エラーメッセージが errbuf に格納され、write_stderr を通じて出力される。
  • 処理対象外のイベント（CTRL_CLOSE_EVENT 等）が渡された場合は何もせず FALSE を返す。

保守担当者向けメモ

  • 本関数は Windows のコンソールイベントハンドラとして動作するため、スレッドセーフな実装が必須である。
  • write_stderr はシグナルハンドラ内でも安全に動作するように設計されている。
  • CancelRequested フラグは自動リセットされないため、呼び出し側で適切に管理すること。
