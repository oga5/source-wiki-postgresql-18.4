HeapTupleFields

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/htup_details.h:122

目的

ヒープタプルのヘッダーにおいて、タプルのライフサイクル管理に必要なトランザクション情報およびコマンド情報を保持する構造体です。

シグネチャまたは形状

typedef struct HeapTupleFields
{
    TransactionId t_xmin;       /* 挿入トランザクションID */
    TransactionId t_xmax;       /* 削除またはロックトランザクションID */

    union
    {
        CommandId   t_cid;      /* 挿入/削除コマンドID */
        TransactionId t_xvac;   /* 旧形式の VACUUM FULL トランザクションID */
    }           t_field3;
} HeapTupleFields;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。各フィールドは HeapTupleHeaderData の一部としてメモリ上に配置され、アクセサ関数（HeapTupleHeaderGetXmin 等）を通じて読み書きされます。

前提条件と不変条件

  • t_xmin および t_xmax は TransactionId 型であり、PostgreSQLのトランザクション可視性ルールに従います。
  • t_field3 は共用体であり、コンテキストに応じて t_cid または t_xvac として解釈されます。

アルゴリズムまたは振る舞い

  • タプルの挿入時や更新時に、対応するトランザクションIDやコマンドIDが本構造体の各フィールドに書き込まれます。
  • t_field3 の解釈は、タプルのフラグ（t_infomask）の状態に依存します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/rewriteheap.c 等で構造体のサイズ参照や初期化に使用されています。
  • **呼び出し先**: 提供されたソース情報には、本構造体を直接操作する具体的な関数呼び出しの記述はありません。

関連データ構造

  • HeapTupleHeaderData: 本構造体を含む、タプルの物理ヘッダー構造。
  • TransactionId: トランザクション識別子。
  • CommandId: コマンド識別子。

失敗モードと境界条件

  • 共用体 t_field3 の誤った解釈（t_cid を期待している箇所で t_xvac を読み取る等）は、データ破損や誤った可視性判定を招く可能性があります。

保守担当者向けメモ

  • 本構造体は HeapTupleHeaderData の一部として頻繁にアクセスされるため、パディングやアライメントが最適化されています。
  • 構造体定義を変更する場合は、MinimalTupleData とのメモリレイアウトの互換性に注意してください。
