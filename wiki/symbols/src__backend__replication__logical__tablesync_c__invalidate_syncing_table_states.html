invalidate_syncing_table_states

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/tablesync.c:279

目的

論理レプリケーションにおけるテーブル同期状態のキャッシュを無効化し、再構築が必要であることを示すために使用されます。

シグネチャまたは形状

void invalidate_syncing_table_states(Datum arg, int cacheid, uint32 hashvalue)

入力と出力

  • arg (Datum): キャッシュ無効化のコールバック引数。
  • cacheid (int): 無効化対象のキャッシュID。
  • hashvalue (uint32): 無効化対象のハッシュ値。
  • 戻り値: なし。

前提条件と不変条件

  • 本関数は、システムカタログの変更に伴うキャッシュ無効化コールバックとして登録されることを想定しています。
  • 実行後、グローバル変数 table_states_validity が SYNC_TABLE_STATE_NEEDS_REBUILD に設定されます。

アルゴリズムまたは振る舞い

1. 引数として受け取った arg, cacheid, hashvalue を使用せず、グローバルな状態フラグ table_states_validity を SYNC_TABLE_STATE_NEEDS_REBUILD に更新します。 2. これにより、次回テーブル同期状態を参照する際に、最新のカタログ情報に基づいた再構築が強制されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/applyparallelworker.c:981
  • src/backend/replication/logical/worker.c:4812
  • 呼び出し先:
  • 本関数内での直接的な関数呼び出しはありません。

関連データ構造

  • table_states_validity: 同期テーブル状態の有効性を管理するグローバル変数。

失敗モードと境界条件

  • 本関数自体は単純な代入操作であるため、失敗モードはありません。
  • キャッシュ無効化のトリガーとなるカタログ更新イベントが正しく通知されない場合、同期状態が古いまま維持される可能性があります。

保守担当者向けメモ

  • 本関数は SysCache の無効化コールバックとして登録されることを前提としています。
  • table_states_validity の状態遷移については、tablesync.c 内の他の状態管理ロジックと整合性を保つ必要があります。
