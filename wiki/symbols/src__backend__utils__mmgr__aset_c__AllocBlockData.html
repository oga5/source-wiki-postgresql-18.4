AllocBlockData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/aset.c:181

目的

AllocSet において malloc() で確保されたメモリブロックのヘッダーとして機能し、ブロック内の利用可能なメモリ領域を管理する。

シグネチャまたは形状

typedef struct AllocBlockData
{
    AllocSet    aset;           /* このブロックを所有するaset */
    AllocBlock  prev;           /* ブロックリスト内の前のブロック */
    AllocBlock  next;           /* ブロックリスト内の次のブロック */
    char       *freeptr;        /* ブロック内の空き領域の開始位置 */
    char       *endptr;         /* ブロック内の領域の終了位置 */
} AllocBlockData;

入力と出力

  • 本構造体はメモリ管理の内部データ構造であり、直接的な関数引数や戻り値としては扱われない。
  • ALLOC_BLOCKHDRSZ マクロにより、本構造体のサイズ（MAXALIGN 済み）が計算され、ブロック割り当て時のオフセットとして利用される。

前提条件と不変条件

  • freeptr は常に endptr 以下であること。
  • aset フィールドは、当該ブロックが属する有効な AllocSet を指していること。

アルゴリズムまたは振る舞い

  • AllocSet 内のブロックリストを双方向連結リストとして維持する。
  • freeptr を更新することで、ブロック内の空き領域を順次切り出してチャンクとして提供する。
  • freeptr が endptr に達した場合、そのブロックは満杯とみなされる。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数関係については、提供されたソース情報からは特定できません。

関連データ構造

  • AllocSet: このブロックを所有するメモリコンテキスト。
  • AllocBlock: AllocBlockData へのポインタ型（typedef struct AllocBlockData *AllocBlock）。

失敗モードと境界条件

  • ブロック内の空き領域が要求サイズに満たない場合、新しいブロックの割り当てが必要となる。
  • AllocBlockData 自体のメモリ確保に失敗した場合は、システム全体としてメモリ不足（OOM）となる。

保守担当者向けメモ

  • 本構造体は aset.c 内のメモリ割り当てロジックの基盤です。構造体のフィールドを変更する場合は、ALLOC_BLOCKHDRSZ の計算およびブロック管理ロジックへの影響を考慮してください。
