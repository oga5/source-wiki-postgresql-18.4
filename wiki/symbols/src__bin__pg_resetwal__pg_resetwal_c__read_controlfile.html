read_controlfile

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:585

目的

データディレクトリ内の pg_control ファイルを読み込み、その整合性を検証してグローバル変数 ControlFile に格納する。

シグネチャまたは形状

static bool read_controlfile(void)

入力と出力

  • **入力**: ファイルシステム上の XLOG_CONTROL_FILE（pg_control）。
  • **出力**:
  • 成功時: true を返し、ControlFile 構造体に内容をコピーする。
  • 失敗時: false を返す（ファイルが破損している、またはバージョン不一致の場合）。ファイル自体が開けない場合はエラーを出力してプロセスを終了する。

前提条件と不変条件

  • pg_malloc を使用して PG_CONTROL_FILE_SIZE 分のメモリを確保する。
  • 読み込まれたデータは ControlFileData 構造体として解釈され、pg_control_version が現在の PG_CONTROL_VERSION と一致する必要がある。

アルゴリズムまたは振る舞い

1. pg_control を読み込み専用で開く。失敗時はエラー終了する。 2. ファイルの内容をバッファに読み込む。 3. バージョンチェックを行い、一致しない場合は false を返す。 4. CRC32C を計算し、ファイル内の CRC 値と比較する。不一致の場合は警告を発し、グローバル変数 guessed を true に設定する。 5. ControlFile 構造体にバッファの内容をコピーする。 6. IsValidWalSegSize を用いて WAL セグメントサイズを検証する。無効な場合は警告を発し false を返す。 7. すべての検証を通過した場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数 (src/bin/pg_resetwal/pg_resetwal.c:407)。
  • **呼び出し先**: open, read, close, pg_malloc, pg_log_error, pg_fatal, pg_log_warning, INIT_CRC32C, COMP_CRC32C, FIN_CRC32C, EQ_CRC32C, IsValidWalSegSize。

関連データ構造

  • ControlFileData: 制御情報の定義元。
  • ControlFile: 読み込み結果を保持するグローバル変数。

失敗モードと境界条件

  • **ファイル不在**: ENOENT の場合、ヒントを表示して終了する。
  • **CRC 不一致**: 警告を表示し、guessed フラグを立てて処理を続行する。
  • **バージョン不一致**: 警告を表示し、false を返す。
  • **無効な WAL セグメントサイズ**: 警告を表示し、false を返す。

保守担当者向けメモ

  • 本関数は pg_resetwal の初期化フェーズで重要であり、ここで false が返されると GuessControlValues による推測処理へ移行するフローとなっている。
