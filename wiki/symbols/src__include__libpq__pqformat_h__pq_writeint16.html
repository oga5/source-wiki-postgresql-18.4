pq_writeint16

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/libpq/pqformat.h:59

目的

16ビット整数（uint16）をネットワークバイトオーダー（ビッグエンディアン）に変換し、通信バッファ StringInfoData に書き込みます。

シグネチャまたは形状

void pq_writeint16(StringInfoData *pg_restrict buf, uint16 i)

入力と出力

  • **入力**:
  • buf: 書き込み先となる StringInfoData 構造体へのポインタ。
  • i: 書き込む対象の16ビット符号なし整数。
  • **出力**: なし（戻り値なし）。副作用として buf の内容が更新されます。

前提条件と不変条件

  • buf が指すバッファには、書き込むデータ（2バイト）を格納するための十分な空き容量（buf->maxlen - buf->len >= sizeof(uint16)）が確保されている必要があります。
  • この条件を満たさない場合、Assert によりプログラムが停止します。

アルゴリズムまたは振る舞い

1. pg_hton16 を使用して、入力値 i をホストバイトオーダーからネットワークバイトオーダーへ変換します。 2. バッファの現在位置（buf->data + buf->len）に、変換後の値を memcpy でコピーします。 3. バッファのデータ長（buf->len）を2バイト分インクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printtup.c（複数箇所）
  • **呼び出し先**:
  • pg_hton16（バイトオーダー変換用）
  • memcpy（メモリコピー用）
  • Assert（デバッグ時の境界チェック用）

関連データ構造

  • StringInfoData: 通信データのバッファを管理する構造体（lib/stringinfo.h で定義）。

失敗モードと境界条件

  • バッファの容量不足が発生した場合、アサーションが有効なビルドでは実行時エラー（Assert 失敗）となります。
  • バッファの容量チェックは行わないため、呼び出し側で事前に十分な容量を確保しておく必要があります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを優先しています。
  • バッファの動的な拡張が必要な場合は、本関数ではなく pq_sendint16 を使用してください。
