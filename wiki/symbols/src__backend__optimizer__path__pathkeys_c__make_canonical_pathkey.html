make_canonical_pathkey

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/pathkeys.c:55

目的

指定された条件（等価クラス、演算子族、比較型、NULL順序）に基づき、正規化された PathKey を生成または既存のインスタンスを返却します。これにより、パスキーの比較をポインタ比較に簡略化します。

シグネチャまたは形状

PathKey *make_canonical_pathkey(PlannerInfo *root,
                                EquivalenceClass *eclass, Oid opfamily,
                                CompareType cmptype, bool nulls_first)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • eclass: 順序の基準となる等価クラス。
  • opfamily: 演算子族の OID。
  • cmptype: 比較の種類。
  • nulls_first: NULL を先頭に配置するかどうか。
  • **出力**: 正規化された PathKey 構造体へのポインタ。

前提条件と不変条件

  • root->ec_merging_done が真であること（等価クラスの統合が完了している必要がある）。
  • 渡された eclass が統合済みでない場合、ec_merged を辿って最上位の等価クラスが使用される。

アルゴリズムまたは振る舞い

1. root->ec_merging_done を確認し、偽であればエラーを投げる。 2. eclass が統合済みであれば、ec_merged を辿り正規の等価クラスを取得する。 3. root->canon_pathkeys リストを走査し、引数と完全に一致する既存の PathKey があればそれを返す。 4. 一致するものがない場合、root->planner_cxt メモリコンテキストに切り替え、新しい PathKey ノードを割り当てる。 5. 新しい PathKey に各フィールドを設定し、root->canon_pathkeys リストに追加して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/pathkeys.c 内の複数箇所（make_pathkey_from_sortop 等）。
  • contrib/postgres_fdw/postgres_fdw.c:1001。
  • **呼び出し先**:
  • MemoryContextSwitchTo
  • makeNode
  • lappend

関連データ構造

  • PathKey: 本関数が生成・管理する主要な構造体。
  • EquivalenceClass: 順序の等価性を定義する構造体。
  • PlannerInfo: プランナのコンテキスト情報。

失敗モードと境界条件

  • ec_merging_done が偽の場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は、プランナが保持する canon_pathkeys リストを介して一意性を保証します。
  • メモリ管理は root->planner_cxt に依存しているため、GEQO 等の特殊なプランニングコンテキスト下でも適切に動作するよう設計されています。
