SimpleLruAutotuneBuffers

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/slru.c:230

目的

システム全体の共有バッファ数（NBuffers）に基づき、SLRU（Simple LRU）サブシステムが使用するバッファ数を自動調整して算出する。

シグネチャまたは形状

int SimpleLruAutotuneBuffers(int divisor, int max)

入力と出力

  • **入力**:
  • divisor: バッファ数を決定するための除数。
  • max: バッファ数の上限値。
  • **出力**:
  • SLRU_BANK_SIZE の倍数に切り捨てられた、計算後のバッファ数。

前提条件と不変条件

  • 戻り値は常に SLRU_BANK_SIZE 以上である。
  • 戻り値は max を超えない。
  • NBuffers（共有バッファ総数）が定義されている環境で使用される。

アルゴリズムまたは振る舞い

1. NBuffers / divisor を計算し、SLRU_BANK_SIZE の倍数に切り捨てる。 2. 上記の値と SLRU_BANK_SIZE の大きい方を選択する。 3. max から max % SLRU_BANK_SIZE を引いた値（max を SLRU_BANK_SIZE の倍数に切り捨てた値）と比較し、小さい方を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/clog.c
  • src/backend/access/transam/commit_ts.c
  • src/backend/access/transam/subtrans.c
  • **呼び出し先**:
  • Min および Max マクロ（標準的な比較関数）。

関連データ構造

  • SLRU_BANK_SIZE: バッファの銀行（Bank）分割単位。

失敗モードと境界条件

  • divisor が 0 の場合、除算エラーが発生する可能性がある（呼び出し元での保証が必要）。
  • max が SLRU_BANK_SIZE 未満の場合、計算結果は SLRU_BANK_SIZE に固定される。

保守担当者向けメモ

  • 本関数は、各 SLRU サブシステム（CLOG, CommitTS, SubTrans）が共有メモリを初期化する際に、メモリ割り当て量を決定するために利用される。
  • SLRU_BANK_SIZE の倍数に調整するロジックは、銀行単位のロック管理と整合性を保つために必須である。
