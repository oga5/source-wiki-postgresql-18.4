add_partial_path

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/pathnode.c:797

目的

リレーションの候補パスリスト（partial_pathlist）に対し、新しい並列実行可能なパスを追加し、コストやパスキーに基づいて不要なパスを削除・選別する。

シグネチャまたは形状

void add_partial_path(RelOptInfo *parent_rel, Path *new_path)

入力と出力

  • **入力**:
  • parent_rel: 対象となるリレーション情報（RelOptInfo）。
  • new_path: 追加を試みる並列実行可能なパス（Path）。
  • **出力**: なし（parent_rel->partial_pathlist を直接更新）。

前提条件と不変条件

  • new_path->parallel_safe が真であること。
  • parent_rel->consider_parallel が真であること。
  • 割り込みチェック（CHECK_FOR_INTERRUPTS）が実行される。

アルゴリズムまたは振る舞い

1. parent_rel->partial_pathlist 内の既存パスを順次走査する。 2. compare_pathkeys を用いて、新しいパスと既存パスのパスキーを比較する。 3. パスキーが互換性を持つ場合、コストと disabled_nodes を比較し、支配関係（どちらが優れているか）を判定する。

  • 新しいパスが既存パスを支配する場合、既存パスを削除しメモリを解放する。
  • 既存パスが新しいパスを支配する場合、新しいパスの追加を中止し、メモリを解放する。

    • 4. 支配関係がない場合、コスト順を維持するように挿入位置を特定する。 5. 最終的に支配されなかった場合、partial_pathlist に新しいパスを挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/path/joinpath.c
  • **呼び出し先**:
  • compare_pathkeys
  • pfree
  • list_insert_nth
  • foreach_delete_current
  • foreach_current_index

関連データ構造

  • RelOptInfo: パスリストを保持するリレーション情報。
  • Path: 実行パスの情報を格納する構造体。

失敗モードと境界条件

  • new_path が既存のパスによって支配される場合、追加されずに破棄される。
  • パスキーが異なるパス同士は、互いに支配関係を持たないため両方保持される。
  • コストがほぼ等しい場合、パスキーの優劣によってどちらを残すかが決定される。

保守担当者向けメモ

  • STD_FUZZ_FACTOR を用いた曖昧なコスト比較を行っており、微小なコスト差によるプランの不安定化を抑制している。
  • disabled_nodes の値が異なる場合は、コスト比較よりも優先して評価される。
  • pfree によるメモリ解放を行うため、new_path はこの関数に渡した後は呼び出し元で参照してはならない。
