find_struct

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/variable.c:125

目的

構造体や共用体、またはそれらを含む配列のメンバを解決するために、親となる変数エントリを特定し、再帰的なメンバ探索を開始する。

シグネチャまたは形状

static struct variable *find_struct(const char *name, char *next, char *end)

入力と出力

  • **入力**:
  • name: 検索対象の変数名（親構造体名）。
  • next: 構造体メンバ演算子（. または ->）の位置を示すポインタ。
  • end: 検索範囲の終端を示すポインタ。
  • **出力**:
  • 成功時: 該当するメンバの variable 構造体へのポインタ。
  • 失敗時: mmfatal によりプロセスを終了する。

前提条件と不変条件

  • name は find_variable で解決可能な有効な変数名であること。
  • next が指す文字が '-'（ポインタアクセス）かそれ以外（直接アクセス）かによって、型チェックのロジックが分岐する。

アルゴリズムまたは振る舞い

1. next が指す位置を一時的に終端文字で置き換え、find_variable(name) を呼び出して親変数を取得する。 2. next が '-' の場合:

  • 親変数が ECPGt_array であり、かつその要素が構造体または共用体であることを検証する。
  • find_struct_member を呼び出し、メンバを探索する。

    • 3. next が '-' 以外の場合:

  • next == end の場合、親変数が直接構造体または共用体であることを検証し、find_struct_member を呼び出す。
  • それ以外の場合、親変数が配列であり、かつその要素が構造体または共用体であることを検証した上で find_struct_member を呼び出す。

    • 4. 処理の最後に *next を元の文字に戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: find_variable（src/interfaces/ecpg/preproc/variable.c 内の 227 行目および 252 行目）。
  • **呼び出し先**: find_variable, find_struct_member, mmfatal。

関連データ構造

  • struct variable: 変数情報を保持する構造体。
  • ECPGtype: 変数の型情報（ECPGt_struct, ECPGt_union, ECPGt_array 等）。

失敗モードと境界条件

  • 親変数が期待される型（構造体、共用体、配列）でない場合、mmfatal によりエラーメッセージを出力して終了する。
  • 構造体メンバの解決に失敗した場合、find_struct_member の挙動に従いエラーとなる。

保守担当者向けメモ

  • 本関数は find_variable からの再帰的な呼び出しの一部として機能する。
  • mmfatal を使用しているため、エラー発生時はプリプロセッサ全体が停止する点に注意すること。
  • brace_level を find_struct_member に引き継いでいるため、スコープの整合性が維持されている必要がある。
