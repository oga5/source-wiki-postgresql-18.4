be_lo_creat

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-fsstubs.c:248

目的

新しいラージオブジェクト（Large Object）を作成し、そのオブジェクト識別子（Oid）を返します。

シグネチャまたは形状

Datum be_lo_creat(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQLインターフェース経由の引数）
  • **出力**: Datum 型としてラージオブジェクトの Oid を返す。

前提条件と不変条件

  • 読み取り専用トランザクション内では実行できません（PreventCommandIfReadOnly によるチェック）。
  • 実行により、現在のトランザクションでクリーンアップが必要であることを示すフラグ lo_cleanup_needed が true に設定されます。

アルゴリズムまたは振る舞い

1. PreventCommandIfReadOnly を呼び出し、読み取り専用モードでないことを確認します。 2. lo_cleanup_needed を true に設定し、トランザクション終了時のクリーンアップ対象としてマークします。 3. inv_create(InvalidOid) を呼び出し、新しいラージオブジェクトを物理的に作成します。 4. 作成されたラージオブジェクトの Oid を PG_RETURN_OID マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_ARGS を介した呼び出し）。
  • **呼び出し先**:
  • PreventCommandIfReadOnly
  • inv_create（src/backend/storage/large_object/inv_api.c 内の関数）

関連データ構造

  • Oid: ラージオブジェクトを一意に識別する型。
  • lo_cleanup_needed: トランザクション終了時のクリーンアップ処理を制御するグローバル変数。

失敗モードと境界条件

  • 読み取り専用トランザクションで呼び出された場合、エラーが発生します。
  • inv_create が失敗した場合（ストレージ不足や権限エラー等）、その内部で例外がスローされます。

保守担当者向けメモ

  • 本関数はSQLから呼び出されるラージオブジェクト作成のフロントエンドです。
  • トランザクション終了時のリソース解放は AtEOXact_LargeObject が担当するため、本関数では作成フラグの管理のみを行います。
