test_status_print

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress.c:279

目的

テストの実行結果（成功または失敗）を TAP (Test Anything Protocol) 形式で標準出力へ整形して出力する。

シグネチャまたは形状

static void test_status_print(bool ok, const char *testname, double runtime, bool parallel)

入力と出力

  • **入力**:
  • ok: テストが成功した場合は true、失敗した場合は false。
  • testname: テストケース名。
  • runtime: テストの実行時間（ミリ秒）。
  • parallel: 並列実行された場合は true、単一実行の場合は false。
  • **出力**: なし（emit_tap_output を介して標準出力へ書き出す）。

前提条件と不変条件

  • fail_count および success_count がグローバル変数として定義されており、これらを加算して現在のテスト番号を算出する。
  • TESTNAME_WIDTH 定数が定義されており、テスト名表示の幅が固定されていること。

アルゴリズムまたは振る舞い

1. fail_count と success_count の合計から現在のテスト番号を算出する。 2. ok が false の場合、TAP 形式の "not ok" 文字列を生成する。 3. parallel フラグに基づき、並列テストには +、単一テストには - の接頭辞を付与する。 4. テスト番号を 5 桁にパディングし、テスト名と実行時間を整形して emit_tap_output を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/regress/pg_regress.c:307 (成功時)
  • src/test/regress/pg_regress.c:326 (失敗時)
  • **呼び出し先**:
  • emit_tap_output

関連データ構造

  • fail_count (グローバル変数)
  • success_count (グローバル変数)

失敗モードと境界条件

  • テスト番号が 9999 を超える場合、5 桁のパディングが崩れる可能性がある。
  • testname が TESTNAME_WIDTH を超える場合、出力の整列が乱れる可能性がある。

保守担当者向けメモ

  • TAP 出力フォーマットの変更は、emit_tap_output の呼び出し引数およびフォーマット文字列を修正することで行う。
  • テスト番号のパディングやテスト名の整列ロジックは、TAP ハーネスの解析仕様に依存しているため、変更時は注意が必要である。
