system_time_nextsampleblock

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_time/tsm_system_time.c:216

目的

TABLESAMPLE SYSTEM_TIME メソッドにおいて、指定された時間制限内でスキャンすべき次のブロック番号を決定する。

シグネチャまたは形状

static BlockNumber system_time_nextsampleblock(SampleScanState *node, BlockNumber nblocks)

入力と出力

  • **入力**:
  • node: サンプリング状態を保持する SampleScanState 構造体。
  • nblocks: 対象リレーションの現在の総ブロック数。
  • **出力**:
  • 次にスキャンすべき BlockNumber。スキャン終了時または時間切れの場合は InvalidBlockNumber を返す。

前提条件と不変条件

  • node->tsm_state は SystemTimeSamplerData 型としてキャスト可能であること。
  • 初回呼び出し時に sampler->step が 0 の場合、乱数生成器を用いて開始ブロックとステップサイズが初期化される。
  • ブロック選択には線形合同法的なアプローチが用いられ、step は nblocks と互いに素な値が維持される。

アルゴリズムまたは振る舞い

1. **初期化**: スキャン開始時（doneblocks == 0）に、クエリ内で初回であれば乱数シードを用いて開始ブロックとステップサイズを計算し、開始時刻を記録する。 2. **終了判定**:

  • 全ブロックを走査済み（doneblocks > nblocks）の場合、終了する。
  • 現在時刻と開始時刻の差分が指定されたミリ秒（sampler->millis）を超えた場合、終了する。

    • 3. **ブロック選択**: lb = (lb + step) % nblocks の計算を繰り返し、有効なブロック番号を選択する。リレーションのサイズが変動した場合に備え、nblocks 未満になるまでループする。 4. **更新**: 選択したブロック番号を返し、doneblocks をインクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**: tsm_system_time_handler 内で TsmRoutine の NextSampleBlock コールバックとして登録されている。
  • **呼び出し先**: 乱数生成ユーティリティ（sampler_random_init_state, sampler_random_fract, random_relative_prime）および時間計測マクロ（INSTR_TIME_SET_CURRENT 等）。

関連データ構造

  • SystemTimeSamplerData: サンプリングの進捗、時間制限、乱数状態、ブロック選択アルゴリズムのパラメータを保持する。
  • SampleScanState: PostgreSQL のテーブルスキャン状態を管理する構造体。

失敗モードと境界条件

  • nblocks == 0: リレーションが空の場合、即座に InvalidBlockNumber を返す。
  • 時間切れ: sampler->millis に達した時点でスキャンを打ち切る。
  • ブロック数変動: スキャン中にリレーションサイズが変化した場合、do-while ループにより有効なブロック番号を再選択する。

保守担当者向けメモ

  • 本関数は非決定的な動作（repeatable_across_queries = false）を行うため、デバッグ時は実行ごとの結果の差異に留意すること。
  • step の計算には random_relative_prime を使用しており、リレーション全体を網羅的に走査する設計となっている。
