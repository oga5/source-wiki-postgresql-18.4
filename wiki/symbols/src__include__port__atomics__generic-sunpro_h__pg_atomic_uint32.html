pg_atomic_uint32

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/generic-sunpro.h:48

目的

Solaris (SunPro) 環境において、アトミック操作の対象となる 32 ビット符号なし整数値を保持するための構造体です。

シグネチャまたは形状

typedef struct pg_atomic_uint32
{
    volatile uint32 value;
} pg_atomic_uint32;

入力と出力

  • 入力: uint32 型の値を保持する構造体。
  • 出力: なし（データ構造定義）。

前提条件と不変条件

  • value フィールドは volatile 修飾されており、コンパイラによる最適化で値がキャッシュされることを防ぎます。
  • 本構造体は、Solaris のシステム関数（atomic_cas_32 等）が期待するメモリ配置を確保するために使用されます。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行い、具体的なアトミック操作（比較交換や交換など）は pg_atomic_compare_exchange_u32_impl などの関数が本構造体のポインタを引数として受け取り、Solaris の <atomic.h> を介して実行します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/commands/vacuum.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/nodes/tidbitmap.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/pgarch.c
  • src/backend/storage/buffer/freelist.c
  • src/backend/storage/ipc/procsignal.c
  • 呼び出し先: 構造体定義のため該当なし。

関連データ構造

  • pg_atomic_uint64: 64 ビット版のアトミック操作用構造体。

失敗モードと境界条件

  • 本構造体は 32 ビットの値を保持するため、それ以上の範囲の数値は扱えません。
  • アライメントに関する厳密な要件は、コンパイラのバージョンやプラットフォームの仕様に依存します。

保守担当者向けメモ

  • Solaris 環境専用の実装です。
  • pg_atomic_uint64 と異なり、本構造体には明示的なアライメント属性が付与されていません。必要に応じて検証してください。
