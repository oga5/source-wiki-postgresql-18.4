pg_strtok

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/read.c:152

目的

シリアライズされた文字列から、ノードツリー構築に必要なトークンを順次切り出す字句解析器。

シグネチャまたは形状

const char *pg_strtok(int *length)

入力と出力

  • **入力**: length（切り出したトークンの長さを格納する整数へのポインタ）。
  • **出力**: トークンの開始位置を指す const char *。トークンがない場合は NULL を返す。

前提条件と不変条件

  • 静的変数 pg_strtok_ptr が解析対象の文字列を指している必要がある。
  • pg_strtok_ptr を介して状態を保持するため、マルチスレッド環境での同時実行には適さない。

アルゴリズムまたは振る舞い

1. pg_strtok_ptr を開始位置とし、空白・改行・タブをスキップする。 2. 文字列終端に達した場合は NULL を返し、length に 0 を設定する。 3. トークンが (, ), {, } のいずれかであれば、1文字をトークンとして切り出す。 4. それ以外の場合は、区切り文字（空白、改行、タブ、括弧類）が出現するまで読み進める。

  • バックスラッシュ \ が出現した場合、直後の文字をエスケープ対象としてスキップする。

    • 5. 切り出したトークンが <> の場合は、長さ 0 の空トークンとして扱う。 6. pg_strtok_ptr を更新し、トークンの開始位置を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/read.c 内の nodeRead 関数などから呼び出される。
  • **呼び出し先**: 直接的な関数呼び出しはないが、静的変数 pg_strtok_ptr を参照・更新する。

関連データ構造

  • pg_strtok_ptr（静的変数）: 現在の解析位置を保持する。

失敗モードと境界条件

  • トークンが存在しない場合、NULL を返し length を 0 に設定する。
  • バックスラッシュで終わる文字列や、エスケープされた文字の扱いについては、debackslash 関数による後続の処理が必要である。

保守担当者向けメモ

  • 本関数はバックスラッシュを保持したままトークンを切り出すため、値の復元時には必ず debackslash を通すこと。
  • pg_strtok_ptr はグローバルな静的変数であるため、再帰的な呼び出しを行う際は、呼び出し側で現在のポインタを保存・復元する処理が必要となる。
