r_substantive

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_estonian.c:1221

目的

エストニア語のステミング処理において、名詞に関連する複数の接尾辞削除ルールを順次適用する。

シグネチャまたは形状

static int r_substantive(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体へのポインタ（現在の単語状態を保持）。
  • 出力: 成功時に 1 を返し、内部のルール関数が失敗（負の値）を返した場合はその値を伝播する。

前提条件と不変条件

  • SN_env 構造体が初期化済みであること。
  • 各ルール適用前後で、現在のカーソル位置 z->c を保存・復元し、各ルールが独立して単語の末尾から評価されるように制御されている。

アルゴリズムまたは振る舞い

以下の順序で名詞関連の接尾辞削除ルールを試行する。各ルールは SN_env の状態を更新する可能性がある。 1. r_special_noun_endings: 特殊な名詞語尾の処理。 2. r_case_ending: 格変化語尾の処理。 3. r_plural_three_first_cases: 複数形の最初の3格の処理。 4. r_degrees: 比較級・最上級の処理。 5. r_i_plural: i 複数形の処理。 6. r_nu: nu 接尾辞の処理。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_estonian.c 内のメインステミングパイプライン（行1396付近）。
  • 呼び出し先: r_special_noun_endings, r_case_ending, r_plural_three_first_cases, r_degrees, r_i_plural, r_nu。

関連データ構造

  • struct SN_env: Snowball ランタイム環境を保持する構造体。

失敗モードと境界条件

  • 各サブ関数が負の値を返した場合、即座にその値を呼び出し元へ返す。
  • 各ルール適用時にカーソル位置が不適切であれば、ルールは適用されず次のルールへ進む。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接編集せず元の .sbl ファイルを修正すること。
  • 各ルール適用時に m1 から m6 までのローカル変数でカーソル位置を保存・復元しており、各ルールが単語の同一位置から開始されるよう設計されている。
