pg_create_logical_replication_slot

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:168

目的

論理レプリケーションスロットをSQLインターフェース経由で作成し、作成されたスロット名と現在の確定済みLSN（confirmed_flush）を返却します。

シグネチャまたは形状

Datum pg_create_logical_replication_slot(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • slot_name (name): スロット名
  • plugin (name): 使用する論理デコーディングプラグイン名
  • temporary (bool): 一時スロットか否か
  • two_phase (bool): 2フェーズコミットをサポートするか否か
  • failover (bool): フェイルオーバーを有効にするか否か
  • **出力**:
  • (slot_name, lsn) を含む複合型（Row type）の Datum

前提条件と不変条件

  • 呼び出し元は適切な権限（CheckSlotPermissions）を有していること。
  • 論理デコーディングに必要な構成要件（CheckLogicalDecodingRequirements）が満たされていること。
  • 戻り値の型が複合型（TYPEFUNC_COMPOSITE）であること。

アルゴリズムまたは振る舞い

1. get_call_result_type で戻り値の型を検証します。 2. 権限と論理デコーディングの要件を確認します。 3. create_logical_replication_slot を呼び出し、内部的なスロット作成処理を実行します。 4. 作成されたスロットの情報を MyReplicationSlot から取得し、結果用タプルを構築します。 5. 一時スロットでない場合は ReplicationSlotPersist を呼び出し、ディスクへ永続化します。 6. ReplicationSlotRelease を呼び出し、スロットのロックを解放して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQLコマンド（SELECT pg_create_logical_replication_slot(...)）
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_upgrade/pg_upgrade.c
  • **呼び出し先**:
  • create_logical_replication_slot
  • ReplicationSlotPersist
  • ReplicationSlotRelease
  • CheckSlotPermissions
  • CheckLogicalDecodingRequirements

関連データ構造

  • MyReplicationSlot: 現在のバックエンドが操作中のレプリケーションスロット。
  • TupleDesc, HeapTuple: SQL関数が返す行形式の構築に使用。

失敗モードと境界条件

  • 戻り値の型が複合型でない場合、elog(ERROR) を発生させます。
  • 権限不足や論理デコーディングの要件を満たさない場合、エラーが発生します。
  • スロット作成処理中にエラーが発生した場合、トランザクションのロールバックに伴い自動的にクリーンアップされます。

保守担当者向けメモ

  • 本関数はSQLレイヤーと内部の slot.c を繋ぐブリッジです。
  • スロットの永続化は temporary フラグに基づいて条件分岐するため、変更時は ReplicationSlotPersist の呼び出し条件に注意してください。
