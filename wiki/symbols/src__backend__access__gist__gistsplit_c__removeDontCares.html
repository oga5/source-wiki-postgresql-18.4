removeDontCares

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistsplit.c:166

目的

GiSTインデックスのページ分割処理において、特定の列で分割ペナルティが発生しない（don't-care）タプルを、現在の分割対象リストから除外するために使用されます。

シグネチャまたは形状

static void removeDontCares(OffsetNumber *a, int *len, const bool *dontcare)

入力と出力

  • a (入出力): OffsetNumber の配列。処理対象のタプルインデックスを保持。
  • len (入出力): 配列 a の要素数。関数終了時に除外後の要素数に更新される。
  • dontcare (入力): bool 型の配列。各タプルが「don't-care」であるかを示すフラグ。

前提条件と不変条件

  • dontcare 配列は、a に含まれるすべての OffsetNumber をインデックスとして参照可能である必要があります。
  • len は配列 a の有効な長さを正しく指している必要があります。

アルゴリズムまたは振る舞い

1. 配列 a を先頭から走査します。 2. 各要素 ai について、dontcare[ai] が false である場合のみ、その要素を配列の現在の書き込み位置（curwpos）に保持します。 3. dontcare[ai] が true の場合、その要素はスキップされ、最終的な長さ len がデクリメントされます。 4. 最終的に、dontcare が false の要素のみが配列の先頭側に詰められ、len が更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistsplit.c 内の以下の箇所から呼び出されます。
  • gistSplitByKey 内で、左側のページ分割結果（sv->spl_left）から don't-care タプルを除去する際。
  • gistSplitByKey 内で、右側のページ分割結果（sv->spl_right）から don't-care タプルを除去する際。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • OffsetNumber: PostgreSQLのページ内タプル位置を指す識別子。
  • GistEntryVector: 本関数が操作する分割対象タプルのリスト構造に関連します。

失敗モードと境界条件

  • len が 0 の場合、ループは実行されず、何も変更されません。
  • dontcare 配列の範囲外アクセスに対する保護は本関数内には存在しません。

保守担当者向けメモ

  • 本関数はインプレース（in-place）で配列を書き換えます。呼び出し元で len を更新するため、呼び出し後の len の値は必ず最新の要素数となります。
  • マルチカラムGiSTインデックスの再帰的分割において、不要なタプルを効率的にフィルタリングする重要なユーティリティです。
