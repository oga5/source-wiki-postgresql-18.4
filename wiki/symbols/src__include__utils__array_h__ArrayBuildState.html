ArrayBuildState

定義

  • 種別: struct
  • 定義箇所: src/include/utils/array.h:187

目的

PostgreSQLにおいて、配列を逐次的に構築する際の作業用状態を保持するための構造体です。

シグネチャまたは形状

typedef struct ArrayBuildState
{
	MemoryContext mcontext;
	Datum	   *dvalues;
	bool	   *dnulls;
	int			alen;
	int			nelems;
	Oid			element_type;
	int16		typlen;
	bool		typbyval;
	char		typalign;
	bool		private_cxt;
} ArrayBuildState;

入力と出力

  • **入力**: 配列の要素となる Datum 値およびその NULL 状態。
  • **出力**: 最終的に makeArrayResult 等の関数へ渡され、構築された ArrayType を生成するための基盤となります。

前提条件と不変条件

  • mcontext は、構築中に使用される一時的なメモリ領域を指します。
  • dvalues と dnulls は、少なくとも nelems 分の有効なエントリを保持し、alen 分のメモリが割り当てられている必要があります。
  • typlen, typbyval, typalign は、格納される element_type に対応する適切な型情報である必要があります。

アルゴリズムまたは振る舞い

1. initArrayResult 等で構造体を初期化し、メモリコンテキストと型情報を設定します。 2. accumArrayResult を繰り返し呼び出し、dvalues 配列に要素を追加して nelems をインクリメントします。必要に応じて alen を拡張します。 3. 最終的に makeArrayResult を呼び出し、蓄積されたデータからフラットな ArrayType を構築します。

関連データ構造

  • ArrayType: 最終的な出力形式となる標準的なフラット配列構造体。
  • ArrayBuildStateArr, ArrayBuildStateAny: 関連する構築用構造体。

失敗モードと境界条件

  • メモリ不足: mcontext におけるメモリ割り当て失敗時にエラーが発生する可能性があります。
  • 型不一致: 構築時に指定した element_type と異なる型の Datum を追加した場合の挙動は、呼び出し側の責任に依存します。

保守担当者向けメモ

  • 本構造体は、配列構築の各ステップで動的にメモリを確保・拡張するため、大量の要素を扱う場合はメモリ消費量に注意してください。
  • private_cxt が true の場合、構築終了後に MemoryContextDelete を用いて明示的にメモリを解放する必要があります。
