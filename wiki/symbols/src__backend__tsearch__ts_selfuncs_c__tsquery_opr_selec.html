tsquery_opr_selec

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_selfuncs.c:279

目的

tsquery の木構造を再帰的に走査し、統計情報（MCELEM）に基づいて全文検索演算子（@@）の選択率を推定する。

シグネチャまたは形状

Selectivity tsquery_opr_selec(QueryItem *item, char *operand, TextFreq *lookup, int length, float4 minfreq)

入力と出力

  • **入力**:
  • item: 評価対象の QueryItem（ノード）。
  • operand: tsquery のオペランド文字列。
  • lookup: 統計情報（TextFreq 配列）。
  • length: lookup 配列の要素数。
  • minfreq: 最小頻度の閾値。
  • **出力**:
  • Selectivity（double 型）: 推定された選択率（0.0 から 1.0 の範囲）。

前提条件と不変条件

  • 再帰関数であるため、スタックオーバーフローを防ぐために check_stack_depth() を呼び出す。
  • 計算された選択率は CLAMP_PROBABILITY マクロにより 0.0〜1.0 の範囲に収められる。
  • プレフィックス検索（word:*）の場合、統計情報が 100 件未満であればデフォルト値（DEFAULT_TS_MATCH_SEL * 4）を使用する。

アルゴリズムまたは振る舞い

1. **ノード種別の判定**:

  • QI_VAL（語彙）の場合:
  • プレフィックス検索（prefix フラグあり）: lookup 内の全要素を走査し、一致する要素の頻度を確率的に結合（s1 + s2 - s1*s2）して算出。
  • 通常検索: bsearch で lookup を検索。見つかればその頻度を、見つからなければ minfreq / 2 を返す。
  • 演算子の場合:
  • OP_NOT: 1.0 - 子ノードの選択率 を返す。
  • OP_AND / OP_PHRASE: 左ノードの選択率 * 右ノードの選択率 を返す。
  • OP_OR: s1 + s2 - (s1 * s2) を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_selfuncs.c 内の tsmatchsel および自身（再帰呼び出し）。
  • **呼び出し先**:
  • check_stack_depth()
  • bsearch()（標準ライブラリ）
  • compare_lexeme_textfreq（比較関数）
  • 自身（再帰）

関連データ構造

  • QueryItem: tsquery のノード構造。
  • TextFreq: 語彙と頻度を保持する構造体。
  • LexemeKey: 検索用のキー構造体。

失敗モードと境界条件

  • lookup が NULL の場合、デフォルト値（DEFAULT_TS_MATCH_SEL）を返す。
  • 未知の演算子が渡された場合、elog(ERROR) を発生させる。
  • プレフィックス検索において統計情報が不足している場合、精度が低下する可能性がある。

保守担当者向けメモ

  • プレフィックス検索の近似計算は、MCELEM が全語彙を代表しているという仮定に基づいている。
  • tsquery の構造が非常に深い場合、スタック深さに注意が必要。
