RelationMapUpdateMap

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relmapper.c:324

目的

OID と物理ファイル番号（RelFileNumber）の対応関係を更新します。ブートストラップモードでは永続的なマップを直接更新し、通常モードではトランザクションのコミット時に反映されるよう変更を予約または即時適用します。

シグネチャまたは形状

void RelationMapUpdateMap(Oid relationId, RelFileNumber fileNumber, bool shared, bool immediate)

入力と出力

  • relationId: 更新対象のリレーション OID。
  • fileNumber: 新しい物理ファイル番号。
  • shared: 共有カタログ（shared_map）を対象とする場合は true、ローカルカタログ（local_map）の場合は false。
  • immediate: true の場合、現在のトランザクション内で即座に有効化（active_updates へ格納）。false の場合、コミット時まで保留（pending_updates へ格納）。

前提条件と不変条件

  • サブトランザクション内での呼び出しは禁止されています（ERROR を発生）。
  • 並列モード（Parallel Mode）での呼び出しは禁止されています（ERROR を発生）。
  • 内部で apply_map_update を呼び出し、指定されたマップ構造体に対して変更を適用します。

アルゴリズムまたは振る舞い

1. **モード判定**: IsBootstrapProcessingMode() の場合、shared_map または local_map を直接更新します。 2. **通常モード**:

  • トランザクションのネストレベルが 1 より大きい場合、または並列モードの場合はエラーを発生させます。
  • immediate が true ならば、active_shared_updates または active_local_updates を更新対象とします。
  • immediate が false ならば、pending_shared_updates または pending_local_updates を更新対象とします。

    • 3. **適用**: 選択されたマップ構造体に対し、apply_map_update を実行してマッピングを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/cluster.c: cluster 操作に伴うマッピング更新。
  • src/backend/utils/cache/relcache.c: リレーションキャッシュの更新処理。
  • **呼び出し先**:
  • apply_map_update (内部関数)
  • elog (エラー発生時)
  • GetCurrentTransactionNestLevel / IsInParallelMode (状態チェック)

関連データ構造

  • RelMapFile: マップ情報を保持する構造体。
  • shared_map, local_map: 永続的なマップデータ。
  • active_shared_updates, active_local_updates: トランザクション内で即時有効化された更新。
  • pending_shared_updates, pending_local_updates: コミット待ちの保留中更新。

失敗モードと境界条件

  • サブトランザクション内での呼び出しは ERROR となります。
  • 並列ワーカー内での呼び出しは ERROR となります。
  • 内部の apply_map_update が失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数はマッピング変更の入り口であり、アトミックな更新を保証するための重要なゲートウェイです。
  • 将来的にサブトランザクションや並列モードでの更新をサポートする場合、pending 状態の管理構造を拡張する必要があります。
