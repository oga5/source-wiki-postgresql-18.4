process_directory_recursively

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:832

目的

バックアップディレクトリを再帰的に走査し、各ファイルに対して適切な統合処理（コピーまたは増分再構築）を実行し、出力ディレクトリ構造を構築する。

シグネチャまたは形状

static void process_directory_recursively(
    Oid tsoid,
    char *input_directory,
    char *output_directory,
    char *relative_path,
    int n_prior_backups,
    char **prior_backup_dirs,
    manifest_data **manifests,
    manifest_writer *mwriter,
    cb_options *opt
)

入力と出力

  • **入力**:
  • tsoid: テーブルスペースOID（指定時のみ）。
  • input_directory, output_directory: 入出力のルートパス。
  • relative_path: 現在走査中の相対パス。
  • n_prior_backups, prior_backup_dirs: 統合対象の先行バックアップ情報。
  • manifests, mwriter: マニフェストデータおよび書き込み用コンテキスト。
  • opt: 実行オプション（ドライラン設定等）。
  • **出力**: なし（副作用としてディレクトリ作成およびファイル処理を実行）。

前提条件と不変条件

  • relative_path が NULL の場合、ルートディレクトリとして扱われる。
  • pg_tblspc ディレクトリ内のテーブルスペースOIDに対応するエントリは、シンボリックリンクまたはディレクトリであればスキップされる。
  • pg_wal ディレクトリ配下はマニフェストに含まれないため、チェックサム計算の対象外となる。

アルゴリズムまたは振る舞い

1. **分類**: relative_path に基づき、is_pg_tblspc, is_pg_wal, is_incremental_dir フラグを設定し、処理対象を識別する。 2. **パス構築**: 入出力パスおよびマニフェスト用プレフィックスを生成する。 3. **ディレクトリ作成**: ルート以外では、出力先にディレクトリを新規作成する。 4. **走査**: opendir/readdir を使用してディレクトリを走査する。

  • . および .. は無視する。
  • ディレクトリであれば、相対パスを更新して自身を再帰呼び出しする。
  • 通常ファイル（PGFILETYPE_REG）以外は警告を出してスキップする。
  • 通常ファイルについては、後続の処理（ソースコード抜粋範囲外）へ引き継ぐ。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • main (データディレクトリの走査開始時)
  • process_directory_recursively (自身を再帰呼び出し)
  • **呼び出し先**:
  • get_dirent_type (ファイル種別判定)
  • parse_oid (テーブルスペースOID判定)
  • pg_log_debug, pg_log_warning, pg_fatal (ログ出力)
  • mkdir (ディレクトリ作成)

関連データ構造

  • cb_options: 実行設定を保持。
  • manifest_data, manifest_writer: バックアップマニフェストの管理用。

失敗モードと境界条件

  • ディレクトリのオープン失敗 (opendir が NULL) は pg_fatal で終了する。
  • ディレクトリ作成失敗 (mkdir が -1) は pg_fatal で終了する。
  • 不明なファイル種別 (PGFILETYPE_ERROR) は exit(1) で終了する。
  • シンボリックリンクや特殊ファイルは警告を出してスキップする。

保守担当者向けメモ

  • is_incremental_dir の判定ロジックは、増分再構築が必要な場所を限定するための重要な保護層である。
  • pg_wal 関連の処理はマニフェスト管理外であるため、他のファイル処理と混同しないよう注意すること。
