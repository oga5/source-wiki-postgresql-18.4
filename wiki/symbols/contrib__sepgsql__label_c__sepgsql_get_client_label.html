sepgsql_get_client_label

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/label.c:79

目的

現在のクライアントプロセスのセキュリティコンテキスト（ラベル）を取得する。権限チェックやオブジェクト作成時のコンテキスト計算において、優先順位に基づいた有効なラベルを特定するために使用される。

シグネチャまたは形状

char *sepgsql_get_client_label(void)

入力と出力

  • 入力: なし
  • 出力: 現在有効なセキュリティラベルを表す char * 型の文字列。

前提条件と不変条件

  • client_label_peer は常に初期化済みであること（Assert(client_label_peer != NULL) により保証）。
  • ラベルの優先順位は以下の通り固定されている:

    • 1. client_label_func (信頼されたプロシージャによる一時的な上書き) 2. client_label_pending (未コミットの変更) 3. client_label_committed (コミット済みの変更) 4. client_label_peer (セッション開始時のデフォルト値)

アルゴリズムまたは振る舞い

1. client_label_func が設定されていれば、それを優先して返す。 2. client_label_pending リストが存在する場合、その末尾（最新）の要素を確認し、ラベルが設定されていればそれを返す。 3. client_label_pending が空の場合、client_label_committed が設定されていればそれを返す。 4. 上記のいずれも該当しない場合、初期値である client_label_peer を返す。

関連データ構造

  • pending_label: 未コミットのラベル変更を保持するリスト構造。
  • client_label_func, client_label_pending, client_label_committed, client_label_peer: 本関数が参照する静的変数群。

失敗モードと境界条件

  • client_label_peer が未設定の場合、アサーション違反によりプロセスが停止する可能性がある。
  • client_label_pending リストが空であっても、client_label_committed または client_label_peer がフォールバックとして機能するため、戻り値が NULL になることは想定されていない。

保守担当者向けメモ

  • 本関数は、トランザクションの状態や信頼されたプロシージャの実行状況に応じて動的にラベルを解決する。ラベルの優先順位を変更する場合は、この関数の条件分岐ロジックを更新する必要がある。
  • client_label_pending のメモリ管理は sepgsql_xact_callback 等と密接に関連しているため、変更時はトランザクション整合性に注意すること。
