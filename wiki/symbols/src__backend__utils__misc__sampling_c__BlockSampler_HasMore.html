BlockSampler_HasMore

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/sampling.c:57

目的

ブロックサンプリングにおいて、まだ抽出対象のブロックが存在するかどうかを判定する。

シグネチャまたは形状

bool BlockSampler_HasMore(BlockSampler bs)

入力と出力

  • **入力**: BlockSampler bs（サンプリング状態を保持する構造体へのポインタ）
  • **出力**: bool 型。抽出可能なブロックが残っている場合は true、そうでない場合は false を返す。

前提条件と不変条件

  • 引数 bs は初期化済みであること。
  • bs->t（現在の走査位置）および bs->m（抽出済み数）が、それぞれ bs->N（全ブロック数）および bs->n（目標抽出数）に対して適切に管理されていること。

アルゴリズムまたは振る舞い

以下の論理積が真である場合に true を返す。 1. bs->t < bs->N: まだ全ブロックを走査し終えていないこと。 2. bs->m < bs->n: まだ目標とする抽出数に達していないこと。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c:1162
  • src/backend/utils/misc/sampling.c:71
  • **呼び出し先**:

関連データ構造

  • BlockSamplerData（BlockSampler 型の基底構造体）: サンプリングの進捗状態（t, m）およびパラメータ（N, n）を保持する。

失敗モードと境界条件

  • bs が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • bs->N や bs->n が負の値や不正な値で初期化されている場合の挙動は定義されていない。
