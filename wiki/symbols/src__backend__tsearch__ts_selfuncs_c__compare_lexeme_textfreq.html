compare_lexeme_textfreq

定義

  • 種別: static function
  • 定義箇所: src/backend/tsearch/ts_selfuncs.c:435

目的

LexemeKey 構造体と TextFreq 構造体の比較を行い、bsearch 等の検索アルゴリズムで語彙（Lexeme）を特定するための比較関数として機能します。

シグネチャまたは形状

static int compare_lexeme_textfreq(const void *e1, const void *e2)

入力と出力

  • **入力**:
  • e1: LexemeKey へのポインタ（検索対象のキー）。
  • e2: TextFreq へのポインタ（比較対象の統計データ）。
  • **出力**:
  • 0: 両者が一致する場合。
  • 正の整数: e1 が e2 より大きい場合。
  • 負の整数: e1 が e2 より小さい場合。

前提条件と不変条件

  • e1 は LexemeKey 型、e2 は TextFreq 型としてキャスト可能であること。
  • e2 が指す TextFreq 構造体内の element は有効な PostgreSQL の可変長データ（varlena）であること。

アルゴリズムまたは振る舞い

1. LexemeKey の長さ (length) と TextFreq の要素の長さ (VARSIZE_ANY_EXHDR) を比較します。 2. 長さが異なる場合、その時点で大小関係を確定し、比較を終了します。 3. 長さが等しい場合のみ、strncmp を用いてバイト単位の文字列比較を行い、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/ts_selfuncs.c 内の検索処理（bsearch を使用する箇所）から参照されています。
  • **呼び出し先**: strncmp（標準ライブラリ関数）。

関連データ構造

  • LexemeKey: 検索対象の語彙とその長さを保持する構造体。
  • TextFreq: 統計情報として保持される語彙要素とその頻度を保持する構造体。

失敗モードと境界条件

  • e1 または e2 が NULL の場合の動作は定義されていません。
  • TextFreq 内の element が不正なメモリ領域を指している場合、VARSIZE_ANY_EXHDR または strncmp でクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は bsearch のコールバックとして設計されています。比較の効率化のため、文字列比較の前に長さのチェックを優先する最適化が行われています。
