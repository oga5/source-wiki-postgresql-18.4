pg_atomic_uint64

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/arch-ppc.h:37

目的

PowerPC アーキテクチャにおいて、アトミック操作が可能な 64 ビット符号なし整数値を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct pg_atomic_uint64
{
    volatile uint64 value pg_attribute_aligned(8);
} pg_atomic_uint64;

入力と出力

  • 入力: 構造体内の value メンバに対して、アトミック操作関数（pg_atomic_compare_exchange_u64_impl 等）が読み書きを行います。
  • 出力: 保持している 64 ビット値。

前提条件と不変条件

  • 64 ビットモードでのみ定義されます。
  • pg_attribute_aligned(8) により、メモリ上で 8 バイト境界に配置されることが保証されます。
  • volatile 修飾子により、コンパイラによる最適化で値の読み書きが省略されないよう制御されます。

アルゴリズムまたは振る舞い

本構造体は、PowerPC の lwarx/stwcx. 命令を用いたアトミック操作の対象として使用されます。メモリ上の値に対して、予約付きロードと条件付きストアを行うことで、マルチスレッド環境下での整合性を維持します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な使用例）:
  • src/backend/access/transam/xlog.c: insertingAt, unloggedLSN, logInsertResult 等の変数として使用。
  • src/backend/access/transam/xlogprefetcher.c: reset_time, prefetch, hit 等のメンバとして使用。
  • 呼び出し先: 本構造体自体はデータ保持用であり、操作は src/include/port/atomics/generic.h を通じて提供される実装関数群によって行われます。

関連データ構造

  • pg_atomic_uint32: 32 ビット版のアトミック整数構造体。
  • pg_atomic_flag: フラグ用のアトミック構造体。

失敗モードと境界条件

  • 64 ビットモード以外でコンパイルされた場合、本構造体は利用できません。
  • アトミック操作時に stwcx. 命令が失敗した場合、予約が解除されたとみなされ、再試行ループが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は PowerPC アーキテクチャ特有の実装です。他アーキテクチャとの互換性を維持するため、上位の generic.h インターフェースを介して操作してください。
  • 8 バイト境界へのアライメントは、PowerPC における単一コピーアトミック性の保証に不可欠です。構造体の定義を変更する際は注意してください。
