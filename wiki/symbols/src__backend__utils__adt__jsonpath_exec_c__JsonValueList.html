JsonValueList

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonpath_exec.c:147

目的

jsonpath クエリの実行過程でマッチした JsonbValue の集合を保持・管理するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct JsonValueList
{
	JsonbValue *singleton;
	List	   *list;
} JsonValueList;

入力と出力

  • **入力**: jsonpath の実行エンジン（executeItem 等）から、マッチした JsonbValue が追加されます。
  • **出力**: 実行結果として、単一の JsonbValue（singleton）または複数の JsonbValue を含む List が格納されます。

前提条件と不変条件

  • singleton が非 NULL の場合、それは単一の結果を指します。
  • 複数の結果が存在する場合、list に JsonbValue のポインタが格納されます。

アルゴリズムまたは振る舞い

  • jsonpath の実行エンジンにおいて、クエリにマッチした要素を蓄積するために使用されます。
  • 実行結果が単一か複数かに応じて、メモリ効率を考慮した二段構えの保持形式をとります。

呼び出し元と呼び出し先

  • **呼び出し元**: executeJsonPath および executeItem 関連の関数群が、マッチ結果を格納する引数として本構造体を渡します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、具体的な操作は JsonValueList を引数に取る各実行関数（executeItem 等）によって行われます。

関連データ構造

  • JsonbValue: リスト内に格納される個々の JSON 要素。
  • List: PostgreSQL の標準的なリスト構造。複数のマッチ結果を保持するために使用されます。

失敗モードと境界条件

  • メモリ不足時、List への追加処理でエラーが発生する可能性があります。
  • 呼び出し元で JsonValueList が適切に初期化されていない場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は jsonpath_exec.c 内の再帰的な実行ロジックにおいて頻繁に受け渡されます。
  • lax モードにおける配列の自動アンラップ処理など、複数の値が生成されるケースでのリスト管理に注意してください。
