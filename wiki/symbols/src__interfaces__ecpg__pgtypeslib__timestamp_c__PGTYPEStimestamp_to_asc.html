PGTYPEStimestamp_to_asc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/timestamp.c:267

目的

timestamp 型の値を、ISO形式（USE_ISO_DATES）に基づいた文字列に変換します。

シグネチャまたは形状

char *PGTYPEStimestamp_to_asc(timestamp tstamp)

入力と出力

  • **入力**: timestamp tstamp（マイクロ秒単位の int64 値）。
  • **出力**: 変換された文字列を指す char *。呼び出し元は、返されたメモリを解放する責任があります。失敗した場合は NULL を返します。

前提条件と不変条件

  • tstamp が有限値であるか、特殊値（infinity 等）であるかによって処理が分岐します。
  • 内部的に DateStyle は 1（ISO形式）に固定されています。

アルゴリズムまたは振る舞い

1. TIMESTAMP_NOT_FINITE(tstamp) を判定し、特殊値であれば EncodeSpecialTimestamp を呼び出して文字列化します。 2. 有限値であれば timestamp2tm を呼び出し、timestamp を struct tm および小数秒 fsec_t に分解します。 3. 分解に成功した場合、EncodeDateTime を呼び出して buf に文字列を書き込みます。 4. 最後に pgtypes_strdup(buf) を使用して、ヒープ上に確保した文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c
  • **呼び出し先**:
  • TIMESTAMP_NOT_FINITE (マクロ)
  • EncodeSpecialTimestamp
  • timestamp2tm
  • EncodeDateTime
  • pgtypes_strdup

関連データ構造

  • timestamp: 内部的に int64 として扱われる時刻型。
  • struct tm: C標準の日時構造体。
  • fsec_t: 小数秒を保持する型。

失敗モードと境界条件

  • timestamp2tm が失敗した場合、errno に PGTYPES_TS_BAD_TIMESTAMP を設定し、NULL を返します。
  • メモリ不足により pgtypes_strdup が失敗した場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 現在 DateStyle が 1 にハードコードされています。将来的にオプション化を検討する余地があります。
  • timestamp2tm の戻り値が 0 以外の場合にエラーとして扱われます。
