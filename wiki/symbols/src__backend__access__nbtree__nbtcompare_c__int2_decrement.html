int2_decrement

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:155

目的

B-tree インデックスのスキップスキャンにおいて、int16 型の値を 1 減算する処理を提供します。

シグネチャまたは形状

static Datum int2_decrement(Relation rel, Datum existing, bool *underflow)

入力と出力

  • **入力**:
  • rel: 対象となるリレーション（未使用）。
  • existing: 減算対象の int16 値（Datum 型）。
  • underflow: アンダーフロー発生時に状態を書き込むためのポインタ。
  • **出力**:
  • 戻り値: 減算後の Datum 値。アンダーフロー時は 0。
  • *underflow: 減算結果が int16 の範囲を下回る場合に true、それ以外は false。

前提条件と不変条件

  • existing は int16 として解釈可能であること。
  • underflow ポインタは有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. existing を int16 に変換します。 2. 値が PG_INT16_MIN と等しい場合、アンダーフローと判断し、*underflow を true に設定して 0 を返します。 3. それ以外の場合、*underflow を false に設定し、値を 1 減算した結果を Datum 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内で、SkipSupport 構造体の decrement フィールドに本関数が登録されています（sksup->decrement = int2_decrement;）。
  • **呼び出し先**: PostgreSQL の型変換マクロ（DatumGetInt16, Int16GetDatum）を使用しています。

関連データ構造

  • SkipSupport: インデックススキップスキャン用の増減ロジックを保持する構造体。

失敗モードと境界条件

  • **アンダーフロー**: 入力値が PG_INT16_MIN の場合、減算は行われず underflow フラグが立てられます。

保守担当者向けメモ

  • 本関数は nbtcompare.c 内の他の型（int4, int8 等）の減算関数と同様のパターンに従っています。新しい整数型を追加する際は、この実装をテンプレートとして利用してください。
