WALOpenSegment

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogreader.h:45

目的

読み込み中の WAL セグメントの状態を保持し、セグメントファイルへのアクセスを管理するために使用される構造体です。

シグネチャまたは形状

typedef struct WALOpenSegment
{
    int         ws_file;    /* セグメントファイル記述子 */
    XLogSegNo   ws_segno;   /* セグメント番号 */
    TimeLineID  ws_tli;     /* 現在開いているファイルのタイムラインID */
} WALOpenSegment;

入力と出力

本構造体はデータ保持用であり、直接的な関数入出力は持ちません。各フィールドは、WAL 読み込み処理において現在開いているセグメントを識別するために参照されます。

前提条件と不変条件

  • ws_file が有効なファイル記述子である場合、そのファイルは ws_segno および ws_tli に対応する WAL セグメントである必要があります。

アルゴリズムまたは振る舞い

WAL の読み込み処理において、現在開いているセグメントのメタデータとファイル記述子を保持します。読み込み対象のセグメントが切り替わる際、この構造体の内容が更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogreader.c: WALOpenSegmentInit 関数内で初期化に使用。
  • src/backend/access/transam/xlogutils.c: エラー情報 (errinfo) の一部として参照。
  • src/bin/pg_waldump/pg_waldump.c: エラー報告時のセグメント情報として参照。
  • **呼び出し先**:
  • 本構造体はデータ構造であり、特定の関数を呼び出すことはありません。

関連データ構造

  • XLogReaderState: WAL リーダーのメイン状態構造体。この内部で WALOpenSegment が利用されます。
  • WALSegmentContext: WALOpenSegment の初期化時に使用されるコンテキスト情報。

失敗モードと境界条件

  • ws_file が -1 の場合、セグメントが開かれていないことを示します。
  • 存在しないセグメント番号や無効なタイムラインIDが設定された場合、後続の読み込み処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は xlogreader.h で定義されており、バックエンドとフロントエンド（pg_waldump 等）の両方で共有されます。変更時は両方の互換性に注意してください。
  • XLogReaderState 構造体内の wre_seg フィールドなどで利用されています。
