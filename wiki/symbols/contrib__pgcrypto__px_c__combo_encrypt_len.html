combo_encrypt_len

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px.c:169

目的

暗号化処理後の出力バッファに必要な最大サイズを算出します。

シグネチャまたは形状

static unsigned combo_encrypt_len(PX_Combo *cx, unsigned dlen)

入力と出力

  • **入力**:
  • cx: PX_Combo 構造体へのポインタ。
  • dlen: 暗号化対象となるデータの長さ（バイト単位）。
  • **出力**:
  • 暗号化後のデータ長として dlen + 512 を返します。

前提条件と不変条件

  • dlen は暗号化対象のデータ長です。
  • 戻り値は、パディング等を含めた暗号化後の最大バッファサイズを保証する値として計算されます。

アルゴリズムまたは振る舞い

入力されたデータ長 dlen に対して、固定値 512 を加算した値を返します。これは、暗号化およびパディング処理によって増加する可能性のあるバッファサイズを安全に見積もるための静的な計算です。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/px.c:325 にて、PX_Combo 構造体の関数ポインタ encrypt_len に本関数が代入されています。
  • **呼び出し先**: 本関数自体は他の関数を呼び出しません。

関連データ構造

  • PX_Combo: 暗号化アルゴリズムとパディング方式を抽象化する構造体。本関数はこの構造体のメンバとして利用されます。

失敗モードと境界条件

  • dlen が非常に大きい場合、unsigned 型の範囲内でオーバーフローする可能性があります。

保守担当者向けメモ

  • 本関数は固定のバッファ増分（512バイト）を返します。将来的にアルゴリズムやパディング方式に応じて動的な計算が必要になった場合は、この実装を見直す必要があります。
