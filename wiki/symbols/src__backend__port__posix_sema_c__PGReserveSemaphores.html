PGReserveSemaphores

定義

  • 種別: function
  • 定義箇所: src/backend/port/posix_sema.c:195

目的

PostgreSQL のセマフォサブシステムを初期化し、プロセス間同期に必要なリソースを確保します。

シグネチャまたは形状

void PGReserveSemaphores(int maxSemas)

入力と出力

  • **入力**: maxSemas (int) - 確保するセマフォの最大数。
  • **出力**: なし（副作用としてグローバル変数の設定とメモリ確保を行う）。

前提条件と不変条件

  • DataDir が有効であり、stat() システムコールが成功すること。
  • USE_NAMED_POSIX_SEMAPHORES の定義有無により、メモリ確保先（ヒープまたは共有メモリ）が分岐する。
  • 共有メモリを使用する場合、スピンロックが未準備であるため ShmemAllocUnlocked() を使用する必要がある。

アルゴリズムまたは振る舞い

1. DataDir の inode 番号を取得し、セマフォキー検索のシードとして設定する（衝突回避およびクラッシュ後のクリーンアップ用）。 2. USE_NAMED_POSIX_SEMAPHORES が定義されている場合、maxSemas 分の sem_t * をヒープ領域に確保する。 3. 未定義の場合、ShmemAllocUnlocked を使用して共有メモリ上にセマフォ領域を確保する。 4. グローバル変数 numSems を 0 に、maxSems を maxSemas に、nextSemKey を inode 番号に初期化する。 5. on_shmem_exit を登録し、プロセス終了時に ReleaseSemaphores が実行されるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c 内の PGReserveSemaphores(numSemas); からの呼び出しが確認されている。
  • **呼び出し先**: stat, malloc (または ShmemAllocUnlocked), on_shmem_exit, ereport, elog。

関連データ構造

  • PGSemaphore (共有メモリ上のセマフォ配列)
  • mySemPointers (名前付きセマフォ使用時のポインタ配列)
  • statbuf (データディレクトリの inode 取得用)

失敗モードと境界条件

  • stat(DataDir, ...) が失敗した場合: FATAL エラーでプロセスを終了する。
  • malloc が失敗した場合: PANIC エラーでプロセスを終了する。

保守担当者向けメモ

  • 本関数は EXEC_BACKEND 環境下では名前付きセマフォと共存できない制約がある。
  • nextSemKey の初期化に inode を使用することで、同一ディレクトリ内の古いセマフォリソースの検出とクリーンアップを容易にしている。
