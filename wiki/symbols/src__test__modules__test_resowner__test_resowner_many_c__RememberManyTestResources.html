RememberManyTestResources

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_resowner/test_resowner_many.c:115

目的

指定された数のテスト用リソースを生成し、ResourceOwner に登録および内部リストへの追跡を行う。

シグネチャまたは形状

static void RememberManyTestResources(ResourceOwner owner,
                                      ManyTestResourceKind *kinds, int nkinds,
                                      int nresources)

入力と出力

  • **入力**:
  • owner: リソースを登録する対象の ResourceOwner。
  • kinds: リソースの種類を定義する ManyTestResourceKind の配列。
  • nkinds: kinds 配列の要素数。
  • nresources: 生成および登録するリソースの総数。
  • **出力**: なし（副作用として owner および kinds の状態を更新）。

前提条件と不変条件

  • nresources 分のリソースが palloc によりメモリ確保される。
  • kinds 配列内の各リソース種別に対して、リソースが循環的に割り当てられる。
  • 各リソースは ResourceOwner に登録される前に ResourceOwnerEnlarge が呼び出され、容量が確保される。

アルゴリズムまたは振る舞い

1. nresources 回ループし、以下の処理を繰り返す。 2. ManyTestResource 構造体をメモリ確保し、現在の kind を割り当てる。 3. dlist_node_init でリストノードを初期化する。 4. ResourceOwnerEnlarge を呼び出し、ResourceOwnerRemember でリソースを登録する。 5. 対応する kind の nremembered カウンタをインクリメントする。 6. dlist_push_tail を使用し、当該 kind の current_resources リストの末尾にリソースを追加する。 7. DEBUG1 レベルでログを出力する。 8. kind_idx を更新し、次のリソースには次の種類の kind を割り当てる（循環）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_resowner/test_resowner_many.c 内の test_resowner_many 関数（252行目および257行目）。
  • **呼び出し先**:
  • palloc
  • dlist_node_init
  • ResourceOwnerEnlarge
  • ResourceOwnerRemember
  • dlist_push_tail
  • elog

関連データ構造

  • ResourceOwner: PostgreSQLのリソース所有者管理構造体。
  • ManyTestResourceKind: リソースの種別と管理用リストを保持する構造体。
  • ManyTestResource: 個別のリソースインスタンスを保持する構造体。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQLの標準的なエラーハンドリングによりアボートする。
  • nresources が 0 の場合、ループは実行されず何も行われない。
  • nkinds が 0 の場合、kinds[kind_idx] へのアクセスで境界外参照が発生する可能性がある（呼び出し元で nkinds > 0 が保証されている必要がある）。

保守担当者向けメモ

  • 本関数はテストモジュール専用であり、リソースの登録と追跡の整合性を検証するために dlist を併用している。
  • ResourceOwner への登録と内部リストへの追加がアトミックではないため、テスト実行中の並行処理には注意が必要。
