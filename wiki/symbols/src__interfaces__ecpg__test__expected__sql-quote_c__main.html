目的

ECPG（Embedded SQL for C）における文字列リテラルのエスケープ処理および引用符の扱いを検証する回帰テストのエントリポイント。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム引数は使用しない）
  • 出力: 標準出力へテスト結果（standard_conforming_strings の設定値およびテーブル内のデータ）を出力。終了コードは成功時に 0、エラー発生時に 1 を返す。

前提条件と不変条件

  • データベース ecpg1_regression への接続が可能であること。
  • ECPGdo を介した SQL 文の実行結果が sqlca.sqlcode に反映されること。
  • sqlca.sqlwarn[0] == 'W' のチェックにより、SQL 警告が標準出力に転送されること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug でデバッグを有効化し、データベースへ接続。 2. **環境設定**: standard_conforming_strings を off に設定し、テーブルを作成。 3. **テスト実行（off時）**: 文字列リテラルおよびエスケープ文字列定数（E'...'）を挿入。 4. **環境設定**: standard_conforming_strings を on に切り替え、同様の挿入を実行。 5. **検証**: カーソル C を使用してテーブル内の全データをフェッチし、結果を標準出力へ表示。 6. **終了処理**: トランザクションのロールバックおよびテーブル削除を実行し、接続を解除。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGsetcommit, ECPGdo, ECPGtrans, sqlprint, printf, exit

関連データ構造

  • sqlca: SQL 通信領域（エラーコードや警告の確認に使用）。
  • var: char[25] 型のバッファ（設定値の取得に使用）。
  • i, loopcount: 整数型の制御変数。

失敗モードと境界条件

  • sqlca.sqlcode < 0 となる SQL エラーが発生した場合、exit(1) により即座に終了する。
  • ECPG_NOT_FOUND が発生した場合は、フェッチループを正常に終了する。

保守担当者向けメモ

  • 本関数は ecpg プリプロセッサによって生成されたコードであるため、直接編集しないこと。
  • standard_conforming_strings の切り替えによるエスケープ挙動の差異を検証するテストであるため、修正時は E'...' 構文の解釈に注意すること。
