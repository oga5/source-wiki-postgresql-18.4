subcolor

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:336

目的

指定された文字 c に対して、必要に応じて新しいサブカラーを割り当て、カラーマップを更新する。

シグネチャまたは形状

static color subcolor(struct colormap *cm, chr c)

入力と出力

  • **入力**:
  • cm: カラーマップ構造体へのポインタ。
  • c: 彩色対象の文字コード。
  • **出力**:
  • 割り当てられた color 値。エラー発生時は COLORLESS を返す。

前提条件と不変条件

  • c <= MAX_SIMPLE_CHR であること（アサーションにより検証）。
  • cm は初期化済みであること。

アルゴリズムまたは振る舞い

1. cm->locolormap から文字 c の現在の色 co を取得する。 2. newsub(cm, co) を呼び出し、新しいサブカラー sco を取得する。 3. エラーチェックを行い、CISERR() が真であれば COLORLESS を返す。 4. co == sco の場合、既に適切なサブカラーが割り当て済みであるため、そのまま co を返す。 5. 上記以外の場合、以下の更新を行う：

  • 元の色 co の文字カウント nschrs を減らす。
  • 新しい色 sco の文字カウントを増やし、必要に応じて firstchr を設定する。
  • cm->locolormap の該当箇所を新しい色 sco で更新する。

    • 6. 最終的に sco を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、subcolor 自体は正規表現コンパイルプロセスの一部として呼び出されることが示唆されているが、具体的な関数名は提供されていない。
  • **呼び出し先**:
  • newsub: 新しいサブカラーの割り当てに使用。
  • CISERR: エラー状態の確認に使用。

関連データ構造

  • colormap: 文字と色のマッピングを保持する主要構造体。
  • colordesc: 色ごとの属性（文字数や最初の文字など）を保持する構造体。

失敗モードと境界条件

  • newsub が失敗した場合、COLORLESS を返す。
  • c が MAX_SIMPLE_CHR を超える場合、アサーション違反となる。

保守担当者向けメモ

  • 本関数は locolormap（低位文字用マップ）を操作する。高位文字用には subcolorhi が同様の役割を果たす。
  • cm->cd[co].nschrs の更新は、カラーマップの整合性を保つために不可欠である。
