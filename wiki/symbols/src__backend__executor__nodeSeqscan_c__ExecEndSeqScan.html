ExecEndSeqScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:288

目的

シーケンシャルスキャン実行ノードに関連するリソースを解放し、スキャン処理を終了する。

シグネチャまたは形状

void ExecEndSeqScan(SeqScanState *node)

入力と出力

  • **入力**: SeqScanState *node - 終了対象となるシーケンシャルスキャンノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は有効な SeqScanState 構造体を指している必要がある。
  • node->ss.ss_currentScanDesc が初期化されている場合、そのスキャン記述子は table_endscan を呼び出すことで適切にクローズされる。

アルゴリズムまたは振る舞い

1. node から現在のスキャン記述子 (TableScanDesc) を取得する。 2. スキャン記述子が NULL でない場合、table_endscan(scanDesc) を呼び出してテーブルアクセスメソッド層のスキャンを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c 内の実行ノード終了処理から呼び出されることが確認されている。
  • **呼び出し先**:
  • table_endscan (テーブルアクセスメソッドの終了関数)。

関連データ構造

  • SeqScanState: シーケンシャルスキャンの実行状態を保持する構造体。
  • TableScanDesc: テーブルアクセスメソッドによるスキャン状態を保持する構造体。

失敗モードと境界条件

  • scanDesc が NULL の場合、table_endscan は呼び出されず、関数は安全に終了する。

保守担当者向けメモ

  • 本関数はスキャンノードのライフサイクル管理の一部であり、ExecInitSeqScan で確保されたリソースの解放を担う。
  • 並列クエリ実行時のDSM（動的共有メモリ）解放ロジックは本関数には含まれていないため、必要に応じて別途管理されている点に注意すること。
