AllTablesyncsReady

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/tablesync.c:1766

目的

サブスクリプションに関連付けられたすべてのテーブルが、論理レプリケーションの同期プロセスにおいて READY 状態にあるかどうかを判定します。

シグネチャまたは形状

bool AllTablesyncsReady(void)

入力と出力

  • **入力**: なし。

前提条件と不変条件

  • FetchTableStates を呼び出し、最新の同期状態情報を取得する必要があります。
  • 内部でトランザクションが開始された場合、関数終了前に CommitTransactionCommand を呼び出してコミットする必要があります。

アルゴリズムまたは振る舞い

1. FetchTableStates を呼び出し、サブスクリプションのテーブル状態を更新します。この際、必要に応じてトランザクションを開始します。 2. トランザクションが開始された場合は、CommitTransactionCommand を実行し、統計情報をレポートします。 3. 以下の条件の両方を満たす場合にのみ true を返します。

  • サブスクリプションにテーブルが存在する (has_subrels が true)。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/applyparallelworker.c:312
  • src/backend/replication/logical/tablesync.c:666
  • src/backend/replication/logical/worker.c:4615
  • **呼び出し先**:
  • FetchTableStates
  • CommitTransactionCommand
  • pgstat_report_stat

関連データ構造

  • table_states_not_ready: 同期が完了していないテーブルを保持するリスト（グローバル変数またはモジュール内変数）。

失敗モードと境界条件

  • サブスクリプションにテーブルが一つも登録されていない場合、has_subrels が false となり、結果は false となります。
  • FetchTableStates の実行中にエラーが発生した場合、トランザクションの整合性に影響を与える可能性があります。

保守担当者向けメモ

  • 本関数は、論理レプリケーションの適用ワーカーが同期フェーズからメインのストリーミングフェーズへ移行可能かを判断する重要なゲートキーパーです。
  • table_states_not_ready の管理状態が正確であることを前提としています。同期状態の更新ロジックに変更を加える際は、この判定条件が正しく機能するか確認してください。
