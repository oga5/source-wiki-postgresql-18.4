VarString

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/varlena.c:50

目的

PostgreSQLの可変長データ型（text、bytea等）の内部表現を定義する型エイリアスです。

シグネチャまたは形状

typedef struct varlena VarString;

※ varlena 構造体の詳細なメンバ構成は本ソース抜粋には含まれていません。

入力と出力

  • 入力: Datum 型のデータ（TOAST圧縮されている可能性がある）。
  • 出力: VarString * 型のポインタ。

前提条件と不変条件

  • VarString を扱う際は、TOAST（圧縮・外部化）されたデータを適切に展開する必要があります。
  • 直接アクセスする前に PG_DETOAST_DATUM または PG_DETOAST_DATUM_PACKED マクロを介してポインタを取得することが推奨されます。

アルゴリズムまたは振る舞い

  • 本シンボル自体は型定義であり、具体的なロジックは持ちません。
  • DatumGetVarStringP および DatumGetVarStringPP マクロを通じて、Datum から VarString への変換が行われます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/adt/varlena.c 内の文字列操作関数（比較、連結、検索処理など）。
  • 呼び出し先:
  • PG_DETOAST_DATUM / PG_DETOAST_DATUM_PACKED: TOASTデータの展開処理。

関連データ構造

  • varlena: 本シンボルの実体である構造体。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • TOAST展開を伴わない不正なメモリ領域へのアクセスは、セグメンテーションフォールトやデータ破損の原因となります。

保守担当者向けメモ

  • varlena 構造体の定義は src/include/utils/varlena.h に依存しています。
  • VarString を引数に取る関数を実装する際は、必ず PG_DETOAST_DATUM マクロを使用して、データがメモリ上で正しく展開されていることを確認してください。
