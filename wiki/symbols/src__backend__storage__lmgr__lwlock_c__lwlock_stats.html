lwlock_stats

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/lmgr/lwlock.c:256

目的

軽量ロック（LWLock）の利用状況および競合に関する統計情報を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct lwlock_stats
{
	lwlock_stats_key key;
	int			sh_acquire_count;
	int			ex_acquire_count;
	int			block_count;
	int			dequeue_self_count;
	int			spin_delay_count;
} lwlock_stats;

入力と出力

  • **入力**: lwlock_stats_key をキーとしてハッシュテーブルに格納されます。
  • **出力**: 各フィールドはロックの取得回数や待機回数の累積値として更新されます。

前提条件と不変条件

  • lwlock_stats はハッシュテーブルのエントリとして管理されます。
  • 各フィールドは、対応するロック操作が発生するたびにインクリメントされます。

アルゴリズムまたは振る舞い

  • sh_acquire_count: 共有モードでのロック獲得回数を記録します。
  • ex_acquire_count: 排他モードでのロック獲得回数を記録します。
  • block_count: ロック獲得時に競合が発生し、待機が必要となった回数を記録します。
  • dequeue_self_count: 待機キューから自身を解除した回数を記録します。
  • spin_delay_count: ロック状態の確認時にスピンロックの遅延が発生した回数を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: get_lwlock_stats_entry 関数により、特定の LWLock に対応する統計エントリが取得されます。また、ロックの取得・解放処理（LWLockAcquire 等）の内部で統計更新のために参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • lwlock_stats_key: 統計エントリを識別するためのキー構造体。
  • LWLock: 統計対象となる軽量ロック本体。

失敗モードと境界条件

  • ハッシュテーブルのメモリ不足により、統計エントリの作成が失敗する可能性があります。
  • 統計情報の更新はアトミックではない可能性があるため、高負荷時には正確な値からわずかに乖離する可能性があります。

保守担当者向けメモ

  • 本構造体は lwlock.c 内の統計収集機能で使用されます。デバッグやパフォーマンス分析の目的でフィールドを追加・変更する場合は、ハッシュテーブルの初期化処理（ctl.entrysize の設定箇所）との整合性に注意してください。
