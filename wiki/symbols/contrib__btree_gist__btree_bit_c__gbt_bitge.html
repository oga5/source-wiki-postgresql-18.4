gbt_bitge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:33

目的

BIT 型および VARBIT 型のデータに対して「以上（greater than or equal to）」の比較を行い、その結果をブール値で返す。

シグネチャまたは形状

bool gbt_bitge(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のビット列データへのポインタ。
  • collation: 照合順序（本関数では使用されない）。
  • flinfo: 関数呼び出し情報（本関数では使用されない）。
  • **出力**:
  • bool: a >= b であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、PostgreSQL の BIT/VARBIT 型として有効なデータ構造を指している必要がある。
  • 内部で bitge 関数を呼び出すため、PostgreSQL の標準的なビット列比較規則に従う。

アルゴリズムまたは振る舞い

1. a と b を PointerGetDatum を用いて Datum 型に変換する。 2. DirectFunctionCall2 を介して PostgreSQL 標準の bitge 関数を呼び出す。 3. bitge の戻り値を DatumGetBool で変換し、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_bit.c 内の tinfo 構造体（gbtree_vinfo）で参照されている。
  • **呼び出し先**: PostgreSQL 標準の bitge 関数。

関連データ構造

  • gbtree_vinfo: btree_gist モジュールがインデックス操作に使用する設定テーブル。本関数はこのテーブルの一部として登録されている。

失敗モードと境界条件

  • 入力が NULL の場合、DirectFunctionCall2 の挙動に依存する（通常はエラーまたは予期せぬ動作を引き起こす可能性がある）。
  • ビット列の長さが異なる場合、標準の bitge 関数の比較ロジックに従う。

保守担当者向けメモ

  • 本関数は bitge をラップするだけの単純なアダプターである。
  • collation や flinfo 引数はシグネチャ合わせのために存在しており、現在の実装では利用されていない。
