pg_pwritev

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/port/pg_iovec.h:92

目的

プラットフォーム間で一貫したベクトル書き込み（pwritev）操作を提供するための抽象化レイヤーです。OS が pwritev をサポートしている場合は直接呼び出し、未サポートの場合は pg_pwrite を用いたループによるエミュレーションを行います。

シグネチャまたは形状

ssize_t pg_pwritev(int fd, const struct iovec *iov, int iovcnt, off_t offset);

入力と出力

  • **入力**:
  • fd: 書き込み対象のファイル記述子。
  • iov: 書き込みデータを含む iovec 構造体の配列。
  • iovcnt: iov 配列内の要素数。
  • offset: 書き込みを開始するファイル内のオフセット位置。
  • **出力**:
  • 成功時: 書き込まれた合計バイト数を ssize_t で返します。
  • 失敗時: -1 を返します。

前提条件と不変条件

  • iovcnt が 1 の場合、カーネルのオーバーヘッドを避けるため単一バッファの書き込みとして処理されます。
  • エミュレーション環境では、各 pg_pwrite 呼び出し後に offset が書き込みバイト数分だけ進められます。
  • Windows 環境では、この関数を実行するとファイルポインタが移動する副作用があることに注意が必要です。

アルゴリズムまたは振る舞い

1. **ネイティブサポート時 (HAVE_DECL_PWRITEV が真)**:

  • iovcnt == 1 ならば pwrite を呼び出します。
  • それ以外の場合は pwritev を呼び出します。

    • 2. **エミュレーション時**:

  • iov 配列を順次走査し、各要素に対して pg_pwrite を実行します。
  • 部分的な書き込みが発生した場合は、その時点までの合計バイト数を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元 (直接的な証拠)**:
  • src/backend/storage/aio/aio_io.c:135
  • src/backend/storage/file/fd.c:2296
  • src/common/file_utils.c:655 (ラッパー関数)
  • src/common/file_utils.c:675
  • **呼び出し先**:
  • pwrite (ネイティブサポート時)
  • pwritev (ネイティブサポート時)
  • pg_pwrite (エミュレーション時)

関連データ構造

  • struct iovec: 複数のバッファを保持する構造体。
  • PG_IOV_MAX: スタック上に iovec 配列を確保する際の安全な上限値。

失敗モードと境界条件

  • pg_pwrite が負の値を返した場合、処理を中断します。
  • 部分的な書き込みが発生した場合、残りのバッファを処理せずにその時点までの合計値を返します。

保守担当者向けメモ

  • PG_IOV_MAX を超える iovec 配列を扱う場合は、スタックオーバーフローを避けるためヒープメモリの使用を検討してください。
  • エミュレーション環境での挙動は pg_pwrite の実装に依存します。
