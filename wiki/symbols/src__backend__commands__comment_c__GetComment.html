GetComment

定義

  • 種別: function
  • 定義箇所: src/backend/commands/comment.c:409

目的

指定されたデータベースオブジェクトに関連付けられたコメント文字列を取得します。該当するコメントが存在しない場合は NULL を返します。

シグネチャまたは形状

char *GetComment(Oid oid, Oid classoid, int32 subid)

入力と出力

  • **入力**:
  • oid: 対象オブジェクトのOID。
  • classoid: 対象オブジェクトが属するカタログのOID。
  • subid: サブオブジェクトID（列など）。オブジェクト全体の場合は通常 0。
  • **出力**:
  • 成功時: コメント内容を保持する文字列（char *）。
  • 失敗時/未設定時: NULL。

前提条件と不変条件

  • pg_description システムカタログに対する読み取り権限が必要です。
  • 内部的に AccessShareLock を使用して pg_description を開きます。
  • 同一の oid, classoid, subid の組み合わせに対して、コメントは最大で1つのみ存在すると仮定しています。

アルゴリズムまたは振る舞い

1. pg_description のインデックス（DescriptionObjIndexId）を使用して、指定された oid, classoid, subid に一致するタプルを検索するためのスキャンキーを初期化します。 2. table_open で pg_description を AccessShareLock モードで開きます。 3. systable_beginscan を開始し、一致するタプルを走査します。 4. タプルが見つかった場合、heap_getattr を用いて description フィールドの値を取得し、TextDatumGetCString で文字列に変換します。 5. 最初のマッチが見つかった時点でループを終了し、スキャンをクローズします。 6. 最後に table_close を実行し、取得した文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c (インデックスや制約のコメント取得)
  • src/backend/parser/parse_utilcmd.c (属性や制約のコメント取得)
  • **呼び出し先**:
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan
  • heap_getattr, TextDatumGetCString

関連データ構造

  • pg_description: コメントを格納するシステムカタログ。
  • ScanKeyData: インデックススキャン用の検索条件構造体。
  • SysScanDesc: システムテーブルスキャンの状態管理構造体。

失敗モードと境界条件

  • 対象オブジェクトに対応するコメントが存在しない場合、NULL を返します。
  • 複数のマッチが存在する場合、最初の1つのみを取得し、残りは無視します（単一のコメントのみを想定）。

保守担当者向けメモ

  • 本関数は pg_description（通常オブジェクト用）のみを対象としています。共有オブジェクト（データベース、ロール等）のコメント取得には GetSharedComment（本ソース内の別関数）を使用する必要があります。
  • 戻り値の文字列はメモリ上に確保されるため、呼び出し側で適切に解放する必要があります。
