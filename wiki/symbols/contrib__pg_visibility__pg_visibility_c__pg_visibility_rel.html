pg_visibility_rel

定義

  • 種別: function
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:223

目的

指定されたリレーションの各ブロックにおける可視性マップ（VM）のビット状態（all-visible, all-frozen等）をSQLから取得するためのセット返却関数（SRF）です。

シグネチャまたは形状

Datum pg_visibility_rel(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象リレーションのOID。
  • **出力**:
  • SRF_RETURN_NEXT: 以下の4列からなるタプルを返却します。
  • blkno (int8): ブロック番号
  • all_visible (bool): VMのall-visibleビット状態
  • all_frozen (bool): VMのall-frozenビット状態
  • pd_all_visible (bool): ページヘッダーのPD_ALL_VISIBLEビット状態

前提条件と不変条件

  • SRF_IS_FIRSTCALL 時に collect_visibility_data を呼び出し、対象リレーションのVM情報をメモリ上に収集します。
  • 収集されたデータは FuncCallContext の user_fctx に保持されます。
  • リレーションの種別検証は collect_visibility_data 内で行われます。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時に collect_visibility_data を実行し、リレーションの全ブロックのVM状態を vbits 構造体に格納します。 2. **反復処理**: info->next が info->count に達するまで、各ブロックのビット情報を順次読み取ります。 3. **ビット変換**: vbits 内のビットフラグを bool 値に変換し、heap_form_tuple でタプルを構築します。 4. **終了**: 全ブロックの走査が完了すると SRF_RETURN_DONE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**:
  • collect_visibility_data (データ収集)
  • pg_visibility_tupdesc (タプル定義取得)
  • heap_form_tuple (タプル構築)

関連データ構造

  • vbits: 収集したVM情報を保持する構造体。
  • FuncCallContext: SRFの実行状態を管理するコンテキスト。

失敗モードと境界条件

  • 対象リレーションが存在しない場合や、アクセス権限がない場合はエラーとなります。
  • 巨大なリレーションに対して実行した場合、全ブロックの走査に伴うI/O負荷が発生します。

保守担当者向けメモ

  • 本関数は AccessShareLock を使用してリレーションにアクセスします。
  • 実行中にVMビットが変更される可能性があることを考慮した設計となっています。
