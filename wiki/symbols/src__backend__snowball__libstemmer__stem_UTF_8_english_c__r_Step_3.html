r_Step_3

定義

  • **定義箇所**: src/backend/snowball/libstemmer/stem_UTF_8_english.c:745

目的

Porterステミングアルゴリズムの第3ステップを実行し、特定の接尾辞パターンに基づいて単語の末尾を置換または削除する。

シグネチャまたは形状

static int r_Step_3(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （Snowballの実行環境状態）
  • **出力**: int （成功時に 1、失敗または条件不適合時に 0、エラー発生時に負の値を返す）

前提条件と不変条件

  • z->c（現在のカーソル位置）が有効な範囲にあること。
  • r_R1 関数による領域判定が適用されること。

アルゴリズムまたは振る舞い

1. **事前チェック**: 現在のカーソル位置から後方へ向かい、特定の文字コード条件（a_6 テーブルに基づく）を満たすか確認する。 2. **領域判定**: r_R1(z) を呼び出し、接尾辞削除が許可される領域内であることを確認する。 3. **パターンマッチング**: find_among_b を使用して、後方から接尾辞を検索する。 4. **置換・削除**: マッチしたパターンに応じて以下の処理を行う：

  • case 1-4: slice_from_s を用いて特定の文字列へ置換する。
  • case 5: slice_del を用いて接尾辞を削除する。
  • case 6: r_R2(z) を確認後、slice_del を実行する。

関連データ構造

  • struct SN_env: ステミング処理の環境状態を保持する構造体。
  • a_6: ステップ3で使用される接尾辞検索テーブル。

失敗モードと境界条件

  • find_among_b がマッチしない場合、0 を返す。
  • r_R1 または r_R2 のチェックで条件を満たさない場合、その戻り値（0 または負の値）を返す。
  • slice_from_s や slice_del がメモリ操作等で失敗した場合、負の値を返す。
