SetupGOutput

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:89

目的

psql において、クエリ結果の出力先（\g コマンド等で指定されたファイルやパイプ）が未オープンである場合に、それを適切にオープンして準備する。

シグネチャまたは形状

bool SetupGOutput(FILE **gfile_fout, bool *is_pipe)

入力と出力

  • **入力**:
  • gfile_fout: 出力先ファイルポインタへのポインタ。
  • is_pipe: 出力先がパイプであるかを示すフラグへのポインタ。
  • **出力**:
  • 戻り値: 成功した場合は true、失敗した場合は false を返す。
  • gfile_fout: オープンに成功した場合、開かれたファイルストリームが格納される。
  • is_pipe: オープンに成功した場合、パイプであれば true、ファイルであれば false が格納される。

前提条件と不変条件

  • グローバル変数 pset.gfname が出力先パスを保持している必要がある。
  • *gfile_fout が NULL の場合にのみ、新規オープン処理が実行される。

アルゴリズムまたは振る舞い

1. pset.gfname が設定されており、かつ *gfile_fout が NULL であるかを確認する。 2. 条件を満たす場合、openQueryOutputFile を呼び出して出力先を開く。 3. openQueryOutputFile が成功し、かつ出力先がパイプ (*is_pipe が true) である場合、disable_sigpipe_trap() を呼び出して SIGPIPE シグナルのトラップを無効化する。 4. 処理が成功した場合は true を、失敗した場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/common.c 内の複数箇所（1918行目、1952行目、2141行目付近）。
  • **呼び出し先**:
  • openQueryOutputFile
  • disable_sigpipe_trap

関連データ構造

  • pset: psql のグローバル設定を保持する構造体（settings.h で定義）。pset.gfname を参照する。

失敗モードと境界条件

  • openQueryOutputFile が失敗した場合、この関数は false を返し、出力先のオープンは行われない。
  • pset.gfname が NULL の場合、何もせず true を返す。

保守担当者向けメモ

  • 本関数は \g コマンド実行時の出力先管理を担う。パイプライン処理やファイル出力の切り替え時に呼び出されるため、シグナルハンドリング（SIGPIPE）の整合性に注意すること。
