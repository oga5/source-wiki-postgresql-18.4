UserContext

定義

  • 種別: struct
  • 定義箇所: src/include/utils/usercontext.h:15

目的

PostgreSQL のバックエンド処理において、一時的に別のデータベースユーザー（ロール）へ権限を切り替える際、元のユーザーID、セキュリティコンテキスト、および GUC（設定パラメータ）のネストレベルを保持するために使用されます。

シグネチャまたは形状

typedef struct UserContext
{
	Oid			save_userid;
	int			save_sec_context;
	int			save_nestlevel;
} UserContext;

入力と出力

  • 本構造体は、SwitchToUntrustedUser() 関数によって現在の状態を保存するために初期化されます。
  • RestoreUserContext() 関数に渡されることで、保存された状態を復元するために使用されます。

前提条件と不変条件

  • SwitchToUntrustedUser() を呼び出した後は、必ず対応する RestoreUserContext() を呼び出して状態を復元する必要があります。
  • 復元を怠ると、セッションの権限や GUC 設定が意図しない状態のまま残るリスクがあります。
  • GUC のネストレベルが -1 の場合、セキュリティ制限が不要（相互に切り替え可能）な状態であることを示します。

アルゴリズムまたは振る舞い

1. **保存**: SwitchToUntrustedUser 実行時に、現在のユーザーID、セキュリティコンテキスト、GUC ネストレベルが本構造体に格納されます。 2. **復元**: RestoreUserContext 実行時に、本構造体に保存された値を用いて GUC の変更をロールバックし、元のユーザーIDとコンテキストを再適用します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のソースファイルで UserContext 型の変数が宣言・使用されています。
  • src/backend/commands/tablecmds.c
  • src/backend/replication/logical/tablesync.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。操作関数として SwitchToUntrustedUser および RestoreUserContext が src/backend/utils/init/usercontext.c で定義されています。

関連データ構造

  • src/include/utils/acl.h: 権限チェック（member_can_set_role）に使用されます。
  • src/include/utils/guc.h: GUC ネストレベルの管理に使用されます。

失敗モードと境界条件

  • 適切な復元処理が行われない場合、セッションのセキュリティコンテキストが不正な状態で固定される可能性があります。

保守担当者向けメモ

  • 本機能は「信頼できないユーザー」としてコードを実行する際のセキュリティ保護を目的としています。
  • 構造体の各フィールドは、権限切り替えの前後で状態を整合させるために不可欠です。変更を加える際は、usercontext.c の実装と整合性を保ってください。
