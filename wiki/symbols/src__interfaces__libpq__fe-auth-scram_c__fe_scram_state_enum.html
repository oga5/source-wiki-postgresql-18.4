fe_scram_state_enum

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:44

目的

SCRAM-SHA-256 認証プロセスにおけるクライアント側の現在の進行状態を管理するために使用されます。

シグネチャまたは形状

typedef enum
{
	FE_SCRAM_INIT,
	FE_SCRAM_NONCE_SENT,
	FE_SCRAM_PROOF_SENT,
	FE_SCRAM_FINISHED,
} fe_scram_state_enum;

入力と出力

本シンボル自体は状態値の列挙体です。scram_exchange 関数において、現在の状態を保持する変数として利用されます。

前提条件と不変条件

  • 認証フローは FE_SCRAM_INIT から開始され、順次遷移します。
  • 状態遷移は scram_exchange 関数内のロジックによって制御されます。

アルゴリズムまたは振る舞い

scram_exchange 関数における状態遷移の役割は以下の通りです：

  • FE_SCRAM_INIT: 初期化完了後、クライアントの nonce を含む最初のメッセージを送信する準備状態。
  • FE_SCRAM_NONCE_SENT: サーバーからの salt/nonce を受信・解析し、クライアント証明を構築して送信した後の状態。
  • FE_SCRAM_PROOF_SENT: サーバーからの署名を検証し、認証の成否を確定させる最終段階。
  • FE_SCRAM_FINISHED: 認証プロセスが正常に完了した状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-auth-scram.c 内の scram_exchange 関数が、この列挙型を用いて状態遷移を管理しています。
  • 呼び出し先: 本シンボルはデータ型であり、関数呼び出しの対象ではありません。

関連データ構造

  • fe_scram_state: 認証セッション中の状態を保持する構造体であり、この列挙型の値をメンバとして保持します。

失敗モードと境界条件

  • 状態遷移の順序が不正な場合、認証プロセスは失敗します。
  • サーバーからの予期せぬメッセージ受信や解析エラーが発生した場合、状態遷移は中断されます。

保守担当者向けメモ

  • 本列挙型は libpq の SCRAM 認証ステートマシンの基盤です。新しい認証ステップを追加する場合は、この列挙型と scram_exchange のロジックを同期させる必要があります。
