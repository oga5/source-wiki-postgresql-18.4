rbt_populate

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_rbtree/test_rbtree.c:126

目的

指定されたサイズとステップ値に基づき、赤黒木（RBTree）に対してランダムな順序でキーを挿入し、木を構築するテスト用ユーティリティ関数です。

シグネチャまたは形状

static void rbt_populate(RBTree *tree, int size, int step)

入力と出力

  • **入力**:
  • tree: 操作対象の RBTree 構造体へのポインタ。
  • size: 挿入するノードの総数。
  • step: キー値の倍率を指定する整数。
  • **出力**: なし（void）。

前提条件と不変条件

  • GetPermutation(size) がメモリを割り当てるため、呼び出し元で permutation 配列のメモリ管理を考慮する必要はありません（本関数内で pfree されます）。
  • 挿入されるキーは step * permutation[i] として計算されます。

アルゴリズムまたは振る舞い

1. GetPermutation を使用して 0 から size-1 までの順列を生成します。 2. ループ内で各値を step 倍し、rbt_insert を用いて木に挿入します。この際、重複がないことを期待し、isNew が false の場合はエラーを投げます。 3. size > 0 の場合、最初の値を再度挿入し、重複キーの処理が期待通り（isNew が false になること）に動作するかを確認します。 4. 最後に permutation 配列を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_rbtree/test_rbtree.c 内の各テスト関数（test_rb_tree 等）から、異なる size や step を指定して呼び出されます。
  • **呼び出し先**:
  • GetPermutation
  • rbt_insert
  • elog
  • pfree

関連データ構造

  • RBTree: PostgreSQL の赤黒木ライブラリのメイン構造体。
  • IntRBTreeNode: キーを保持するテスト用ノード構造体。

失敗モードと境界条件

  • rbt_insert が期待に反して isNew を返した場合、elog(ERROR, ...) によりテストが中断されます。
  • size が 0 の場合、再挿入のロジックはスキップされます。

保守担当者向けメモ

  • 本関数はテスト専用です。実運用コードでの使用は想定されていません。
  • 乱数生成器 pg_global_prng_state に依存しているため、実行ごとに挿入順序が変化します。
