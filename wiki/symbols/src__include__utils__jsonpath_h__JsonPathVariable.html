JsonPathVariable

定義

  • 種別: struct
  • 定義箇所: src/include/utils/jsonpath.h:287

目的

SQL/JSON Path 評価時に使用される外部変数の情報を保持するための構造体です。

シグネチャまたは形状

typedef struct JsonPathVariable
{
	char	   *name;
	int			namelen;
	Oid			typid;
	int32		typmod;
	Datum		value;
	bool		isnull;
} JsonPathVariable;

入力と出力

本構造体は、JSON Path 実行エンジンに対して変数名、型情報、および値を渡すためのコンテナとして機能します。

前提条件と不変条件

  • namelen は strlen(name) の値を保持するキャッシュとして機能します。
  • value は Datum 型であり、typid で指定された型に従って解釈される必要があります。

アルゴリズムまたは振る舞い

JSON Path の実行コンテキストにおいて、外部から提供された変数を識別し、その値を評価エンジンが利用できるようにします。namelen を保持することで、GetJsonPathVar() 等の関数内での文字列長計算コストを削減します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExpr.c
  • src/backend/utils/adt/jsonpath_exec.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。
  • **注記**: 呼び出し元・呼び出し先の詳細な関数レベルの依存関係は、提供されたソース excerpt からは特定できません。

関連データ構造

  • src/include/nodes/execnodes.h: passing_values 句における JsonPathVariable エントリとして参照されます。

失敗モードと境界条件

  • name が NULL の場合や、typid が不適切な場合は、評価時に型不整合やクラッシュを招く可能性があります。
  • isnull フラグが正しく設定されていない場合、value の内容が誤って解釈される可能性があります。

保守担当者向けメモ

  • namelen はキャッシュ目的であるため、name を変更した場合は必ず同期して更新する必要があります。
  • 本構造体は palloc を用いて動的に確保されるケースが多いため、メモリ管理には注意してください。
