pg_is_ascii

定義

  • 種別: function
  • 定義箇所: src/common/string.c:131

目的

与えられた文字列が ASCII 文字のみで構成されているかを判定します。

シグネチャまたは形状

bool pg_is_ascii(const char *str)

入力と出力

  • **入力**: str (NULL 終端された文字列へのポインタ)
  • **出力**: 文字列内のすべての文字が ASCII（7ビット）範囲内であれば true、それ以外（8ビット目がセットされている文字を含む）であれば false を返します。

前提条件と不変条件

  • str は有効な NULL 終端文字列である必要があります。
  • 内部的に IS_HIGHBIT_SET マクロを使用して各文字のビット状態を評価します。

アルゴリズムまたは振る舞い

1. 文字列の先頭から NULL 文字 (\0) に到達するまで順次走査します。 2. 各文字に対して IS_HIGHBIT_SET を適用し、8ビット目がセットされているかを確認します。 3. 8ビット目がセットされている文字が見つかった時点で直ちに false を返します。 4. 文字列の終端まで走査を完了した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c
  • src/backend/commands/collationcmds.c
  • src/backend/utils/adt/pg_locale.c
  • src/bin/initdb/initdb.c
  • src/common/saslprep.c
  • **呼び出し先**:
  • 提供されたソース情報には、本関数が呼び出す具体的な関数は含まれていません（IS_HIGHBIT_SET マクロを使用）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 空文字列 ("") が渡された場合、ループは実行されず true を返します。
  • NULL ポインタが渡された場合の挙動は定義されていません（セグメンテーション違反の可能性があります）。

保守担当者向けメモ

  • 本関数は PostgreSQL のバックエンドおよびフロントエンドの両方から利用されるため、依存関係を最小限に抑えています。
  • 非 ASCII 文字の検出は、ロケール設定やエンコーディングの検証において重要な役割を果たします。
