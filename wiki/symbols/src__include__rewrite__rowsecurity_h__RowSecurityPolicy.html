RowSecurityPolicy

定義

  • 種別: struct
  • 定義箇所: src/include/rewrite/rowsecurity.h:20

目的

PostgreSQLの行レベルセキュリティ（RLS）において、特定のコマンドやロールに対して適用される個別のセキュリティポリシー定義を保持します。

シグネチャまたは形状

typedef struct RowSecurityPolicy
{
	char	   *policy_name;	/* ポリシー名 */
	char		polcmd;			/* 対象コマンドタイプ */
	ArrayType  *roles;			/* 対象ロールの配列 */
	bool		permissive;		/* 許可的(true)か制限的(false)か */
	Expr	   *qual;			/* 行フィルタリング用式 */
	Expr	   *with_check_qual;	/* 行挿入/更新制限用式 */
	bool		hassublinks;	/* 式にサブリンクが含まれるか */
} RowSecurityPolicy;

入力と出力

本構造体はデータ保持用であり、関数ではありません。クエリ書き換えフェーズにおいて、リレーションキャッシュから読み込まれたポリシー情報が本構造体に格納され、get_row_security_policies() 等を通じてクエリツリーへの条件注入に利用されます。

前提条件と不変条件

  • permissive が true の場合は許可的ポリシー（OR結合）として、false の場合は制限的ポリシー（AND結合）として扱われます。
  • qual および with_check_qual は、クエリ実行時に評価可能な Expr である必要があります。

アルゴリズムまたは振る舞い

クエリ書き換え時、rewriteHandler がリレーションのポリシーを走査する際、本構造体の qual や with_check_qual が抽出されます。これらはクエリの securityQuals や withCheckOptions リストに結合され、最終的な実行計画に反映されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/policy.c（ポリシー定義の生成時）、src/backend/rewrite/rowsecurity.c（ポリシーリストの走査・ソート・適用時）。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • RowSecurityDesc: 本構造体のリストを保持するコンテナ。
  • Expr: ポリシー条件式を保持するノード。
  • ArrayType: 対象ロールの管理に使用。

失敗モードと境界条件

  • hassublinks フラグが正しく設定されていない場合、クエリ最適化においてサブリンクの評価が適切に行われない可能性があります。
  • 複雑なポリシーやサブリンクを含む場合、クエリ性能に影響を与える可能性があります。

保守担当者向けメモ

  • ポリシーの結合ロジック（許可的 vs 制限的）は src/backend/rewrite/rowsecurity.c に実装されています。変更時はこの結合順序に注意してください。
  • 拡張機能から動的にポリシーを追加する場合は、row_security_policy_hook を介した利用が推奨されます。
