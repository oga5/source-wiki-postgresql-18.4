to_ascii_enc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ascii.c:137

目的

SQL関数 to_ascii のバックエンド実装として、指定されたエンコーディングコードに基づき、入力テキストをASCII近似表現に変換します。

シグネチャまたは形状

Datum to_ascii_enc(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_P_COPY(0): 変換対象の text 型データ。
  • PG_GETARG_INT32(1): エンコーディングを示す整数コード。
  • **出力**:
  • PG_RETURN_TEXT_P: 変換後の text 型データ。

前提条件と不変条件

  • 入力されたエンコーディングコード enc は、PG_VALID_ENCODING(enc) マクロにより妥当性が検証されなければなりません。

アルゴリズムまたは振る舞い

1. 入力された text データをコピーして取得します。 2. 指定されたエンコーディングコードが有効か確認し、無効な場合は ERRCODE_UNDEFINED_OBJECT でエラーを報告します。 3. encode_to_ascii 関数を呼び出し、エンコーディングに基づいた変換処理を実行します。 4. 変換結果を text 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（具体的なC言語レベルの呼び出し元はソースコードから特定できません）。
  • **呼び出し先**: encode_to_ascii を呼び出します。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • PG_VALID_ENCODING(enc) が偽を返す場合、ereport(ERROR, ...) により処理が中断されます。

保守担当者向けメモ

  • 本関数は encode_to_ascii へのラッパーです。変換ロジックの詳細については encode_to_ascii を参照してください。
  • PG_GETARG_TEXT_P_COPY を使用しているため、入力データのメモリ管理は本関数が責任を持ちます。
