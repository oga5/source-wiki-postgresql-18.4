JsonUniqueStackEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/json.c:52

目的

JSON解析時におけるネストされたオブジェクト構造を追跡し、キーの重複チェックを行うためのスタック要素を保持します。

シグネチャまたは形状

typedef struct JsonUniqueStackEntry
{
	struct JsonUniqueStackEntry *parent;
	int			object_id;
} JsonUniqueStackEntry;

入力と出力

  • 入力: parent ポインタにより、スタックの親要素（上位の階層）を参照します。
  • 出力: object_id により、現在の階層におけるオブジェクトを一意に識別します。

前提条件と不変条件

  • parent は、スタックの最上位（ルート）である場合 NULL となります。
  • object_id は、JSON解析プロセスにおいて各オブジェクト階層を区別するために使用されます。

アルゴリズムまたは振る舞い

JSONの構文解析中にオブジェクトがネストされる際、この構造体がスタックとして連結されます。これにより、解析器は現在の階層を把握し、キーの重複チェックを行う際に適切なスコープを維持します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/json.c 内の解析処理（1757, 1775, 1790 行付近）で、スタック操作のために参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • JsonUniqueParsingState: JSON解析時のキー重複チェック用コンテキスト。本構造体はこのコンテキスト内でスタックとして管理されます。

失敗モードと境界条件

  • メモリ不足: スタックの深さが極端に深い場合、メモリ割り当てに失敗する可能性があります。
  • スタックアンダーフロー: parent ポインタを辿る際に NULL 参照を行わないよう、解析ロジック側でのガードが必要です。

保守担当者向けメモ

  • 本構造体は JsonUniqueParsingState の一部として使用されます。スタックの操作ロジックを変更する場合は、json.c 内の解析ループにおけるポインタの整合性に注意してください。
