GinInitMetabuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginutil.c:360

目的

GINインデックスのメタページを初期化し、インデックスの管理に必要な初期状態を設定します。

シグネチャまたは形状

void GinInitMetabuffer(Buffer b)

入力と出力

  • **入力**: b (初期化対象のバッファ)
  • **出力**: なし（バッファの内容がメタページとして上書きされます）

前提条件と不変条件

  • b は有効なバッファである必要があります。
  • GinInitPage を呼び出し、ページタイプを GIN_META に設定します。

アルゴリズムまたは振る舞い

1. GinInitPage を呼び出し、ページヘッダーを GIN_META として初期化します。 2. GinPageGetMeta を介してメタデータ領域を取得します。 3. メタデータの各フィールド（head, tail, nPendingPages, nTotalPages 等）を初期値（主に0または InvalidBlockNumber）でリセットします。 4. ginVersion を GIN_CURRENT_VERSION に設定します。 5. ページヘッダーの pd_lower をメタデータ構造体の直後に設定します。これにより、WAL圧縮時にメタデータが消失するのを防ぎます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gininsert.c
  • src/backend/access/gin/ginxlog.c
  • **呼び出し先**:
  • GinInitPage
  • BufferGetPage
  • BufferGetPageSize
  • GinPageGetMeta

関連データ構造

  • GinMetaPageData: メタページに格納される構造体。
  • Buffer: PostgreSQLのバッファ管理構造体。
  • PageHeader: ページレイアウトを制御するヘッダー構造体。

失敗モードと境界条件

  • バッファが適切にピン留めされていない場合、BufferGetPage 等でエラーが発生する可能性があります。

保守担当者向けメモ

  • pd_lower の手動設定は、WAL圧縮によるメタデータ消失を防ぐための重要な実装です。このロジックを変更する際は、WALの復元プロセスへの影響を考慮してください。
