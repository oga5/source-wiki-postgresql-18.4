truncate_identifier

定義

  • 種別: function
  • 定義箇所: src/backend/parser/scansup.c:92

目的

SQL 識別子をデータベースの制限長（NAMEDATALEN - 1 バイト）に切り詰め、必要に応じてユーザーに警告を通知する。

シグネチャまたは形状

void truncate_identifier(char *ident, int len, bool warn)

入力と出力

  • ident: 操作対象の文字列（インプレースで変更される）。
  • len: 文字列の長さ。
  • warn: 切り詰めが発生した際に NOTICE レベルの警告を発行するかどうかを指定するフラグ。
  • 出力: なし（副作用として ident が書き換えられる）。

前提条件と不変条件

  • ident は書き込み可能なメモリ領域を指している必要がある。
  • マルチバイト文字の境界を破壊しないよう、内部で pg_mbcliplen を使用する。

アルゴリズムまたは振る舞い

1. len が NAMEDATALEN 以上であるかを確認する。 2. 以上の場合、pg_mbcliplen を用いて NAMEDATALEN - 1 バイト以内に収まるよう長さを再計算する。 3. warn が true の場合、ereport を使用して切り詰めが発生した旨を NOTICE として出力する。 4. 再計算された長さの位置にヌル終端文字 \0 を挿入し、文字列を切り詰める。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/dblink/dblink.c
  • src/backend/parser/parser.c
  • src/backend/parser/scansup.c
  • src/backend/utils/adt/varlena.c
  • src/backend/utils/misc/guc.c
  • 呼び出し先:
  • pg_mbcliplen (マルチバイト文字の切り詰め計算)
  • ereport (警告の出力)

関連データ構造

  • NAMEDATALEN: 識別子の最大長を定義する定数（src/include/pg_config_manual.h で定義）。

失敗モードと境界条件

  • len が NAMEDATALEN 未満の場合、何も行われない。

保守担当者向けメモ

  • 本関数はインプレース操作を行うため、呼び出し側で渡すバッファが十分なサイズを持っていることを確認すること。
  • NAMEDATALEN はシステム全体で共有される定数であるため、変更時は影響範囲に注意が必要である。
