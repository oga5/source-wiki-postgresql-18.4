GetScanKeyword

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/common/kwlookup.h:38

目的

ScanKeywordList 構造体で管理されるキーワードリストから、指定されたインデックス n に対応するキーワード文字列を取得します。

シグネチャまたは形状

static inline const char *GetScanKeyword(int n, const ScanKeywordList *keywords)

入力と出力

  • **入力**:
  • n: 取得対象のキーワードのインデックス（整数）。
  • keywords: 検索対象の ScanKeywordList 構造体へのポインタ。
  • **出力**:
  • 指定されたインデックスに対応するキーワード文字列への const char * ポインタ。

前提条件と不変条件

  • keywords は有効な ScanKeywordList 構造体を指している必要があります。
  • インデックス n は、keywords が保持する有効なキーワード範囲内である必要があります。
  • ScanKeywordList は src/tools/gen_keywordlist.pl によって生成された静的データ構造であることを前提としています。

アルゴリズムまたは振る舞い

1. keywords->kw_string（キーワード文字列が連結されたベースアドレス）を取得します。 2. keywords->kw_offsets[n]（指定インデックスのオフセット値）を取得します。 3. ベースアドレスにオフセットを加算し、該当する文字列の先頭アドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/misc.c:447
  • src/common/kwlookup.c:70
  • src/interfaces/ecpg/preproc/c_keywords.c:60
  • src/pl/plpgsql/src/pl_scanner.c:250, 292
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（ポインタ演算のみ）。

関連データ構造

  • ScanKeywordList: キーワードの文字列データ、オフセット配列、ハッシュ関数を保持する構造体。

失敗モードと境界条件

  • インデックス n が範囲外の場合、メモリ上の不正なアドレスを参照する可能性があります。本関数自体には境界チェックは含まれていません。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスを重視して設計されています。
  • kw_offsets 配列の整合性は、ビルド時に生成される ScanKeywordList の内容に依存します。変更を加える際は src/tools/gen_keywordlist.pl の出力を確認してください。
