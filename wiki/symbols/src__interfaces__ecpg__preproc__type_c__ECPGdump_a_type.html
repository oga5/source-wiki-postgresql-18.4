ECPGdump_a_type

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/type.c:217

目的

ECPGプリプロセッサにおいて、C言語の変数型情報を解析し、ECPGライブラリが実行時に解釈可能な形式でCソースコードへ出力（ダンプ）するメインのエントリポイントです。

シグネチャまたは形状

void ECPGdump_a_type(FILE *o, const char *name, struct ECPGtype *type, const int brace_level,
                     const char *ind_name, struct ECPGtype *ind_type, const int ind_brace_level,
                     const char *prefix, const char *ind_prefix,
                     char *arr_str_size, const char *struct_sizeof,
                     const char *ind_struct_sizeof)

入力と出力

  • **入力**: 出力先ファイルストリーム o、変数名 name、型情報 type、スコープ深さ brace_level、および対応するインジケータ変数情報（ind_name, ind_type 等）。
  • **出力**: なし（副作用として o に型記述子コードを書き出す）。

前提条件と不変条件

  • brace_level >= 0 の場合、find_variable を用いてスコープ内の変数定義と型の一致が検証されます。
  • indicator_set が有効な場合、インジケータ変数の型が期待される形式（配列や構造体など）と一致している必要があります。

アルゴリズムまたは振る舞い

1. **型検証**: ECPGt_descriptor 等の特殊型を除き、現在のスコープ内の変数定義と型情報を照合し、不一致があればエラーまたは警告を報告します。 2. **型別ディスパッチ**: type->type に基づき処理を分岐します。

  • ECPGt_array: 配列要素の型に応じて再帰的に ECPGdump_a_struct または ECPGdump_a_simple を呼び出します。ネストされた配列はエラーとなります。
  • ECPGt_struct: ECPGdump_a_struct を呼び出します。
  • ECPGt_union: サポートされていないためエラーを返します。
  • ECPGt_char_variable / ECPGt_descriptor / その他: ECPGdump_a_simple を呼び出し、型情報を出力します。

    • 3. **インジケータ処理**: インジケータ変数が指定されている場合、対応するダンプ関数を呼び出し、NULL値等の状態管理情報を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c
  • src/interfaces/ecpg/preproc/type.c (再帰呼び出しの可能性あり)
  • src/interfaces/ecpg/preproc/variable.c
  • **呼び出し先**:
  • find_variable
  • mmerror / mmfatal / base_yyerror
  • ECPGdump_a_struct
  • ECPGdump_a_simple
  • mm_strdup / free

関連データ構造

  • ECPGtype: 変数の型情報を保持する構造体。
  • variable: スコープ内の変数定義を保持する構造体。

失敗モードと境界条件

  • **ネストされた配列**: ECPGt_array の要素がさらに ECPGt_array である場合、PARSE_ERROR を発生させます。
  • **型不一致**: 変数定義と型情報が一致しない場合、ET_ERROR を発生させます。
  • **インジケータ不整合**: indicator_set が有効で、インジケータ変数の型が期待される構造（配列や構造体）と合致しない場合、mmfatal により処理を中断します。

保守担当者向けメモ

  • ECPGdump_a_simple 内の fprintf はECPGライブラリの実行時動作に直結するため、出力フォーマットの変更には細心の注意が必要です。
  • メモリ管理は mm_strdup 等のラッパー経由で行われており、free の漏れがないか確認してください。
