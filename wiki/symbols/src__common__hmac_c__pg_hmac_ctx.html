pg_hmac_ctx

定義

  • 種別: struct
  • 定義箇所: src/common/hmac.c:50

目的

RFC 2104 に準拠した HMAC 計算の状態を保持するための不透明な構造体です。内部ハッシュコンテキスト、パディングデータ、およびエラー状態を管理します。

シグネチャまたは形状

struct pg_hmac_ctx {
    pg_cryptohash_ctx *hash;
    pg_cryptohash_type type;
    pg_hmac_errno error;
    const char *errreason;
    int         block_size;
    int         digest_size;
    uint8       k_ipad[PG_SHA512_BLOCK_LENGTH];
    uint8       k_opad[PG_SHA512_BLOCK_LENGTH];
};

入力と出力

  • **入力**: pg_hmac_create により生成され、pg_hmac_init で鍵が設定されます。
  • **出力**: pg_hmac_final を通じて最終的な HMAC 値が生成されます。

前提条件と不変条件

  • k_ipad および k_opad は、サポートされる最大ブロックサイズ（PG_SHA512_BLOCK_LENGTH）に合わせて確保されています。
  • メモリ管理は環境に依存します（バックエンドでは palloc、フロントエンドでは malloc を使用）。

アルゴリズムまたは振る舞い

1. **初期化**: 鍵がブロックサイズを超える場合はハッシュ化して短縮し、HMAC_IPAD と XOR して内部ハッシュコンテキストを初期化します。 2. **更新**: データを内部の pg_cryptohash コンテキストへ逐次渡します。 3. **終了**: 内部ハッシュを確定後、k_opad と XOR した値と組み合わせて再度ハッシュ化し、最終結果を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-scram.c（pg_hmac_create を使用）。
  • **呼び出し先**: pg_cryptohash インターフェース（pg_cryptohash_ctx を介してハッシュ計算を実行）。

関連データ構造

  • pg_cryptohash_ctx: 実際のハッシュ計算を行う基盤構造体。
  • pg_hmac_errno: エラーコードを保持する列挙型。

失敗モードと境界条件

  • メモリ不足時や内部ハッシュ関数のエラー発生時に、error フィールドおよび errreason に状態が記録されます。
  • フロントエンド環境では、呼び出し側が pg_hmac_free を呼び出してメモリを解放する責任があります。

保守担当者向けメモ

  • 本実装は OpenSSL 等の外部ライブラリがない環境向けのフォールバックです。
  • k_ipad と k_opad のサイズは固定（PG_SHA512_BLOCK_LENGTH）であり、実装の簡素化のためにメモリを多めに確保しています。
