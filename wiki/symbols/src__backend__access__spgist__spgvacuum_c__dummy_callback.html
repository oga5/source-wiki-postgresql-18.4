dummy_callback

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgvacuum.c:969

目的

SP-GiST インデックスの VACUUM 処理において、タプル削除の判定を意図的に行わないためのプレースホルダー（ダミー）コールバック関数です。

シグネチャまたは形状

static bool dummy_callback(ItemPointer itemptr, void *state);

入力と出力

  • **入力**:
  • itemptr: 検査対象のタプルを指す ItemPointer。
  • state: 呼び出し元の状態を保持する汎用ポインタ。
  • **出力**:
  • 常に false を返します。

前提条件と不変条件

  • 本関数は常に false を返すため、このコールバックが適用される範囲では、いかなるタプルも削除対象として判定されません。

アルゴリズムまたは振る舞い

  • 呼び出された際、引数の内容に関わらず即座に false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgvacuum.c 内の処理において、spgBulkDeleteState 構造体の callback フィールドに代入される形で参照されています（同ファイル 1000 行目）。
  • **呼び出し先**: なし。

関連データ構造

  • spgBulkDeleteState: VACUUM 実行中の状態を保持する構造体であり、本関数はこの構造体のメンバとして設定されます。

失敗モードと境界条件

  • 本関数自体は単純な定数値を返すのみであり、失敗モードは存在しません。

保守担当者向けメモ

  • 本関数は、特定の条件下でコールバック関数を要求するインターフェースに対し、削除処理を無効化したい場合に利用されます。ロジックを変更する際は、この関数が「削除しない」という意図で利用されている箇所に影響がないか確認してください。
