Var

定義

  • 種別: struct
  • 定義箇所: src/include/executor/execExpr.h:345

目的

クエリ実行エンジンにおいて、プランツリー内の変数（Var ノード）を保持し、特に「行全体（whole-row）」の参照を評価するためのコンテキストを管理します。

シグネチャまたは形状

struct {
    Var        *var;            /* プランツリー内の元のVarノード */
    bool        first;          /* 初回実行時の初期化フラグ */
    bool        slow;           /* NULL値の実行時チェックが必要か */
    TupleDesc   tupdesc;        /* 結果タプルのディスクリプタ */
    JunkFilter *junkFilter;     /* 不要な列を除去するためのフィルタ */
} wholerow;

入力と出力

  • 入力: プランツリー上の Var ノード情報。
  • 出力: 評価対象の行データ（tupdesc に基づくタプル）および junkFilter によるフィルタリング結果。

前提条件と不変条件

  • var は有効なプランツリーの Var ノードを指している必要があります。
  • tupdesc は、評価対象となるタプルの構造と一致している必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: first フラグに基づき、初回アクセス時に必要なリソース（tupdesc や junkFilter）のセットアップを行います。 2. **評価**: slow フラグが有効な場合、実行時にNULL値のチェックを行い、安全な値の取得を試みます。 3. **フィルタリング**: junkFilter を使用して、結果から不要な列（resjunk列）を除去します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/postgres_fdw/deparse.c において、外部テーブルの変数参照として言及されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数呼び出し関係は特定できません。

関連データ構造

  • Var: プランツリー上の変数定義。
  • TupleDesc: タプルの型情報。
  • JunkFilter: 不要な列を除去するための構造体。

失敗モードと境界条件

  • var が NULL の場合、または tupdesc が不整合な場合、実行時に予期せぬ動作を引き起こす可能性があります。
  • slow モードでのNULLチェックは、パフォーマンスに影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体は ExprEvalStep の一部として使用されることを想定しています。
  • メモリレイアウトはキャッシュラインを意識する必要があるため、メンバの追加・変更時はサイズに注意してください。
