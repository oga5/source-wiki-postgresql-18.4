process_subquery_nestloop_params

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:526

目的

サブクエリが必要とする外部パラメータを、ネストループ結合のパラメータ（NestLoopParam）として PlannerInfo に登録し、実行時に上位レベルから値を供給できるようにする。

シグネチャまたは形状

void process_subquery_nestloop_params(PlannerInfo *root, List *subplan_params)

入力と出力

  • root: プランナーのグローバル情報を保持する PlannerInfo 構造体へのポインタ。
  • subplan_params: PlannerParamItem のリスト。サブクエリが依存するパラメータの定義を含む。
  • 出力: なし（副作用として root->curOuterParams を更新する）。

前提条件と不変条件

  • subplan_params 内の各項目は Var または PlaceHolderVar である必要がある。
  • パラメータは root->curOuterRels（現在のネストループの外側関係）に含まれている必要がある。
  • root->curOuterParams に既に同一の paramno を持つパラメータが存在する場合、その値（paramval）は既存のものと一致していなければならない。

アルゴリズムまたは振る舞い

1. subplan_params リストを反復処理する。 2. 各項目が Var の場合、その varno が root->curOuterRels に含まれるか確認し、含まれない場合はエラーを投げる。 3. 各項目が PlaceHolderVar の場合、その評価位置（ph_eval_at）が root->curOuterRels のサブセットであることを確認し、満たさない場合はエラーを投げる。 4. root->curOuterParams を走査し、対象の paramno が既に存在するか確認する。

  • 存在する場合: Assert で値の一致を確認し、何もしない。
  • 存在しない場合: 新規に NestLoopParam ノードを作成し、paramval をコピーして root->curOuterParams に追加する。

    • 5. 上記以外の型が渡された場合はエラーを投げる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/plan/createplan.c 内のサブクエリ処理箇所。
  • 呼び出し先: lfirst_node, IsA, bms_is_member, bms_is_subset, find_placeholder_info, makeNode, copyObject, lappend, elog 等の内部マクロおよび関数。

関連データ構造

  • PlannerInfo: プラン作成中の状態管理。
  • PlannerParamItem: サブクエリのパラメータ定義。
  • NestLoopParam: ネストループ結合で受け渡されるパラメータの定義。
  • Var, PlaceHolderVar: パラメータの元となる式。

失敗モードと境界条件

  • root->curOuterRels に含まれない変数が渡された場合: ERROR を発生させる。
  • PlaceHolderVar の評価位置が root->curOuterRels に収まらない場合: ERROR を発生させる。
  • 予期しない型のノードが subplan_params に含まれる場合: ERROR を発生させる。

保守担当者向けメモ

  • 本関数は、サブクエリが相関サブクエリとして実行される際に必要なパラメータを curOuterParams に集約する役割を持つ。
  • Assert(equal(var, nlp->paramval)) は、同一のパラメータIDに対して異なる値が割り当てられる矛盾を検出するための重要な不変条件チェックである。
