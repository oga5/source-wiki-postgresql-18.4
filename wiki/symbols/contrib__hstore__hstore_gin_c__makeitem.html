makeitem

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_gin.c:27

目的

hstore のキーや値を GIN 索引用の text 形式に変換し、識別用のフラグを付与する。

シグネチャまたは形状

static text *makeitem(char *str, int len, char flag)

入力と出力

  • **入力**:
  • str: 変換対象の文字列データ。
  • len: 文字列の長さ。
  • flag: 項目を識別するためのプレフィックス文字（KEYFLAG, VALFLAG, NULLFLAG など）。
  • **出力**:
  • フラグが先頭に付与された text 型のポインタ。

前提条件と不変条件

  • palloc を使用してメモリを確保するため、PostgreSQL のメモリコンテキスト内で呼び出される必要がある。
  • len が 0 の場合、str が NULL であっても flag のみを保持する text 構造体が生成される。

アルゴリズムまたは振る舞い

1. VARHDRSZ + len + 1 バイトのメモリを確保する。 2. SET_VARSIZE マクロで確保した領域のサイズを記録する。 3. 確保した領域の先頭 1 バイトに flag を書き込む。 4. str が存在し len > 0 の場合、フラグの直後に str をコピーする。 5. 生成した text ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • gin_extract_hstore (キー、値、NULLの抽出時)
  • gin_extract_hstore_query (クエリ条件のキー抽出時)
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理)
  • memcpy (標準ライブラリ)

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • KEYFLAG, VALFLAG, NULLFLAG: hstore_gin.c 内で定義される識別用フラグ。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • len が負の値である場合、memcpy の動作は保証されない。

保守担当者向けメモ

  • 本関数が付与するフラグ（K, V, N）は、PostgreSQL 9.1 以前からのディスクフォーマット互換性を維持するために不可欠である。変更時はインデックスの互換性に注意すること。
