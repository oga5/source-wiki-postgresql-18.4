SharedFileSetDeleteAll

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/sharedfileset.c:82

目的

共有ファイルセットに関連付けられたすべての物理ファイルおよびディレクトリを削除します。

シグネチャまたは形状

void SharedFileSetDeleteAll(SharedFileSet *fileset)

入力と出力

  • **入力**: SharedFileSet *fileset - 削除対象の共有ファイルセットへのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 fileset は有効な SharedFileSet 構造体を指している必要があります。
  • 内部的に保持する FileSet 構造体（fileset->fs）が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った SharedFileSet 構造体内部の fs メンバ（FileSet 型）を参照します。 2. FileSetDeleteAll 関数を呼び出し、当該ファイルセットに関連するディレクトリ構造とすべてのファイルを物理的に削除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHashjoin.c:1745
  • src/backend/storage/file/sharedfileset.c（SharedFileSetOnDetach からの呼び出しが想定される）
  • **呼び出し先**:
  • FileSetDeleteAll (src/backend/storage/file/fileset.c)

関連データ構造

  • SharedFileSet: 共有ファイルセットを管理する構造体。
  • FileSet: SharedFileSet が内部で利用する、ファイルセットの基本的なディレクトリ管理機能を提供する構造体。

失敗モードと境界条件

  • FileSetDeleteAll の振る舞いに依存します。ファイルシステム上の権限不足や、既に削除済みのディレクトリに対する操作が発生した場合の挙動は FileSetDeleteAll の実装に準拠します。

保守担当者向けメモ

  • 本関数は物理的な削除を実行する低レイヤーのラッパーです。参照カウンタの管理は行わないため、呼び出し側で適切なライフサイクル管理（SharedFileSetOnDetach 等）が行われていることを確認してください。
