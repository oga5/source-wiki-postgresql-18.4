DefineType

定義

  • 種別: function
  • 定義箇所: src/backend/commands/typecmds.c:151

目的

PostgreSQLにおいて、新しい基本データ型（Base Type）を定義・登録します。

シグネチャまたは形状

ObjectAddress DefineType(ParseState *pstate, List *names, List *parameters)

入力と出力

  • **入力**:
  • pstate: パーサーの状態。
  • names: 型の名前（スキーマ修飾を含むリスト）。
  • parameters: 型の属性を定義する DefElem のリスト。
  • **出力**:
  • ObjectAddress: 作成された型のオブジェクトアドレスを返します。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要です。
  • 型の作成には、事前にシェル型（Shell Type）が存在している必要があります（パラメータが空の場合を除く）。

アルゴリズムまたは振る舞い

1. **権限チェック**: superuser() を呼び出し、スーパーユーザーでない場合はエラーを発生させます。 2. **名前解決**: QualifiedNameGetCreationNamespace を使用して、型名と対象スキーマを特定します。 3. **存在確認**: GetSysCacheOid2 を使用して、指定された名前の型が既に存在するか確認します。 4. **シェル型作成**: parameters が空の場合、TypeShellMake を呼び出してシェル型を作成し、終了します。 5. **パラメータ解析**: parameters リストを走査し、like, internallength, input, output などの各属性を対応する変数に格納します。 6. **エラーハンドリング**: 既に存在する型がシェル型でない場合、またはパラメータ指定時にシェル型が存在しない場合はエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:1415
  • **呼び出し先**:
  • QualifiedNameGetCreationNamespace

関連データ構造

  • ObjectAddress: 作成されたオブジェクトの識別情報。
  • DefElem: SQLコマンドで指定されたパラメータの保持に使用。
  • ParseState: パーサーの状態管理。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合に ERRCODE_INSUFFICIENT_PRIVILEGE で失敗します。
  • **重複定義**: 既に存在する型を再定義しようとした場合、ERRCODE_DUPLICATE_OBJECT で失敗します。
  • **シェル型欠如**: パラメータ付きの CREATE TYPE 実行時にシェル型が存在しない場合、ERRCODE_DUPLICATE_OBJECT で失敗します。

保守担当者向けメモ

  • 本関数は、C言語レベルの表現パラメータがシステムに影響を与えるリスクを考慮し、スーパーユーザー限定の操作として設計されています。
  • NOT_USED セクションのコードは、現在のスーパーユーザーチェックが有効である限り不要です。
  • パラメータ解析時に不明なキーワードが指定された場合、歴史的な互換性のために ERROR ではなく WARNING を発行します。
