FreeDecodingContext

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logical.c:678

目的

論理デコーディングに使用される LogicalDecodingContext 構造体に関連するリソースを解放し、メモリおよび関連するサブシステムの状態をクリーンアップします。

シグネチャまたは形状

void FreeDecodingContext(LogicalDecodingContext *ctx)

入力と出力

  • **入力**: ctx - 解放対象の LogicalDecodingContext へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • ctx は有効な LogicalDecodingContext 構造体を指している必要があります。
  • 内部の各コンポーネント（reorder、snapshot_builder、reader、context）は、初期化時に割り当てられた状態を維持している必要があります。

アルゴリズムまたは振る舞い

1. ctx->callbacks.shutdown_cb が定義されている場合、shutdown_cb_wrapper(ctx) を呼び出してプラグインのシャットダウン処理を実行します。 2. ReorderBufferFree(ctx->reorder) を呼び出し、再順序付けバッファを解放します。 3. FreeSnapshotBuilder(ctx->snapshot_builder) を呼び出し、スナップショット構築器を解放します。 4. XLogReaderFree(ctx->reader) を呼び出し、WAL読み取り器を解放します。 5. MemoryContextDelete(ctx->context) を呼び出し、コンテキストに関連付けられたメモリ領域をすべて解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/logical.c (エラーハンドリングや終了処理)
  • src/backend/replication/logical/logicalfuncs.c
  • src/backend/replication/slotfuncs.c
  • src/backend/replication/walsender.c
  • **呼び出し先**:
  • shutdown_cb_wrapper
  • ReorderBufferFree
  • FreeSnapshotBuilder
  • XLogReaderFree
  • MemoryContextDelete

関連データ構造

  • LogicalDecodingContext: 本関数が管理・解放する主要な状態保持構造体。

失敗モードと境界条件

  • ctx が NULL の場合、ポインタ参照によるクラッシュが発生する可能性があります（ソースコード上でのチェックは行われていません）。
  • 各サブシステムの解放関数（ReorderBufferFree 等）が内部で保持するリソースの整合性に依存します。

保守担当者向けメモ

  • 本関数はコンテキストの破棄を担うため、呼び出し後に ctx ポインタを参照しないよう注意してください。
  • プラグインの shutdown_cb がエラーを発生させた場合、その後のリソース解放処理がスキップされる可能性があるため、プラグイン側の実装には注意が必要です。
