initGinState

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginutil.c:101

目的

GIN インデックスの実行時状態を保持する GinState 構造体を初期化し、インデックスの演算クラス（OpClass）で定義されたサポート関数や型情報をキャッシュする。

シグネチャまたは形状

void initGinState(GinState *state, Relation index)

入力と出力

  • **入力**:
  • state: 初期化対象となる GinState 構造体へのポインタ。
  • index: 対象となる GIN インデックスの Relation。
  • **出力**: なし（state が指すメモリ領域が更新される）。

前提条件と不変条件

  • state は呼び出し元で確保された有効なメモリ領域であること。
  • CurrentMemoryContext は、キャッシュされる FmgrInfo 構造体の生存期間に適したコンテキストであること。
  • インデックスの OpClass は、必須のサポート関数（extractValue, extractQuery および consistent または triConsistent）を提供していること。

アルゴリズムまたは振る舞い

1. state 構造体をゼロクリアする。 2. インデックスの属性ごとにループし、以下を実行する：

  • 属性ごとの TupleDesc を構築する。
  • GIN_COMPARE_PROC を検索し、存在しない場合は型キャッシュからデフォルトの B-tree 比較関数を取得する。
  • GIN_EXTRACTVALUE_PROC および GIN_EXTRACTQUERY_PROC を取得しキャッシュする。
  • GIN_TRICONSISTENT_PROC または GIN_CONSISTENT_PROC のいずれかが存在することを確認し、キャッシュする。
  • GIN_COMPARE_PARTIAL_PROC の有無を確認し、部分一致機能の可否をフラグ設定する。
  • インデックスの照合順序（Collation）を決定し、supportCollation に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • src/backend/access/gin/ginfast.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/gin/ginscan.c
  • src/backend/access/gin/ginvacuum.c
  • **呼び出し先**:
  • fmgr_info_copy: サポート関数のキャッシュ用。
  • index_getprocinfo / index_getprocid: OpClass サポート関数の取得用。
  • lookup_type_cache: 比較関数の検索用。

関連データ構造

  • GinState: GIN インデックスの実行時状態を保持する主要構造体。
  • Relation: インデックスのカタログ情報。
  • FmgrInfo: 関数呼び出しのためのキャッシュ情報。

失敗モードと境界条件

  • 必要なサポート関数（consistent 等）が OpClass に定義されていない場合、elog(ERROR) により処理を中断する。
  • 比較関数が特定できない場合、ereport(ERROR) により処理を中断する。

保守担当者向けメモ

  • 本関数で割り当てられるメモリは CurrentMemoryContext に依存するため、呼び出し元のコンテキストが短命な場合は注意が必要である。
  • ginExtractEntries 等のキー操作関数は、ここでキャッシュされた FmgrInfo を利用して実行時のオーバーヘッドを削減している。
