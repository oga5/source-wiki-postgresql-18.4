rbt_leftmost

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:234

目的

赤黒木（RBTree）内の最小値を持つノードを検索し、そのポインタを返します。

シグネチャまたは形状

RBTNode *rbt_leftmost(RBTree *rbt)

入力と出力

  • **入力**: rbt (赤黒木を管理する RBTree 構造体へのポインタ)
  • **出力**: 木の中で最も左側（最小値）にある RBTNode へのポインタ。木が空の場合は NULL を返します。

前提条件と不変条件

  • rbt は初期化済みである必要があります。
  • 木の走査には静的な番兵 RBTNIL が使用されます。

アルゴリズムまたは振る舞い

1. ルートノードから開始し、左の子ノードを RBTNIL に到達するまで辿ります。 2. 辿った経路の最後にあるノード（左端のノード）を保持します。 3. 保持したノードが RBTNIL でない場合、そのノードを返します。 4. 木が空（ルートが RBTNIL）の場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_rbtree/test_rbtree.c 内のテストコードにて、操作の正当性確認のために呼び出されています。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありませんが、RBTNIL（静的番兵）を参照してノードの終端判定を行います。

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTNode: 木の各ノード。
  • RBTNIL: 葉ノードを表す静的な番兵。

失敗モードと境界条件

  • **空の木**: rbt->root が RBTNIL である場合、NULL を返します。
  • **単一ノードの木**: ルートノード自身が左端ノードとして返されます。

保守担当者向けメモ

  • 本関数は、木構造の最小値探索を効率的に行うための標準的な実装です。
  • 戻り値が NULL か RBTNIL 以外のノードであるかを呼び出し側で適切に処理してください。
