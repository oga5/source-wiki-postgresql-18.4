FlushPendingNulls

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:376

目的

JumbleState 内に蓄積された「保留中の NULL 値の数」をハッシュバッファへ書き出し、カウンタをリセットします。

シグネチャまたは形状

static pg_attribute_always_inline void FlushPendingNulls(JumbleState *jstate)

入力と出力

  • **入力**: jstate (JumbleState *) - クエリのフィンガープリント状態を保持する構造体。
  • **出力**: なし（副作用として jstate の内部状態を更新）。

前提条件と不変条件

  • jstate->pending_nulls > 0 であること（アサーションにより強制）。

アルゴリズムまたは振る舞い

1. jstate->pending_nulls の値を 4 バイトのデータとして AppendJumbleInternal を呼び出し、ハッシュバッファへ書き込みます。 2. jstate->pending_nulls を 0 にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/queryjumblefuncs.c 内の各ノード走査関数（行 216, 300, 323, 337, 351, 365）。
  • **呼び出し先**: AppendJumbleInternal。

関連データ構造

  • JumbleState: クエリのフィンガープリント計算に必要なバッファや pending_nulls カウンタを保持する構造体。

失敗モードと境界条件

  • pending_nulls が 0 の状態で呼び出された場合、アサーション違反により停止します。

保守担当者向けメモ

  • pg_attribute_always_inline が指定されており、パフォーマンス向上のためインライン展開が推奨されています。
  • AppendJumbleInternal を介してハッシュバッファを更新するため、バッファの溢れが発生した場合は AppendJumbleInternal 内部のロジックに従いハッシュ化が実行されます。
