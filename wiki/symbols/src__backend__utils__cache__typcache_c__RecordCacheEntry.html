RecordCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/typcache.c:172

目的

匿名レコード型（record型）の構造情報をキャッシュし、システムカタログへのアクセスを最小化するために使用されます。

シグネチャまたは形状

typedef struct RecordCacheEntry
{
    TupleDesc   tupdesc;
} RecordCacheEntry;

入力と出力

  • 入力: RecordCacheHash ハッシュテーブルの検索キーとして使用されます。
  • 出力: 保持している TupleDesc を通じて、レコード型の列定義や型情報を提供します。

前提条件と不変条件

  • RecordCacheEntry は RecordCacheHash ハッシュテーブル内で管理されます。
  • tupdesc は、対応するレコード型の構造を正確に記述している必要があります。

アルゴリズムまたは振る舞い

  • RecordCacheHash ハッシュテーブルの要素として格納されます。
  • キャッシュ検索時、hash_search を用いてエントリの取得や生成が行われます。
  • ハッシュテーブルの構築時には sizeof(RecordCacheEntry) がエントリサイズとして指定されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/typcache.c 内のハッシュ関数、マッチ関数、および RecordCacheHash を操作する関数群（hash_search を含む）。
  • 呼び出し先: 構造体内部で TupleDesc を保持します。

関連データ構造

  • TupleDesc: レコード型の詳細な構造を定義するデータ構造。
  • RecordCacheHash: RecordCacheEntry を格納するハッシュテーブル。
  • SharedRecordTableEntry: 共有メモリ版のレコードキャッシュエントリ（関連する概念としてソース内で言及あり）。

失敗モードと境界条件

  • ハッシュテーブル検索時にエントリが見つからない場合、キャッシュミスとして処理されます。
  • メモリ不足時には hash_search が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体はローカルバックエンド用のキャッシュエントリです。並列クエリ等で利用される共有メモリ版の SharedRecordTableEntry と混同しないよう注意してください。
  • TupleDesc のライフサイクル管理（参照カウント等）が適切に行われているか確認してください。
