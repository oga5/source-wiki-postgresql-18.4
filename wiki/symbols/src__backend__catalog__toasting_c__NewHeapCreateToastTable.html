NewHeapCreateToastTable

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/toasting.c:63

目的

指定されたヒープリレーションに対して、TOAST（The Oversized-Attribute Storage Technique）テーブルを生成および関連付けます。

シグネチャまたは形状

void NewHeapCreateToastTable(Oid relOid, Datum reloptions, LOCKMODE lockmode, Oid OIDOldToast)

入力と出力

  • **入力**:
  • relOid: TOASTテーブルを作成する対象リレーションのOID。
  • reloptions: TOASTテーブルに適用するストレージオプション。
  • lockmode: テーブル作成時に取得するロックモード。
  • OIDOldToast: 既存のTOASTテーブルOID（置換対象がある場合）。
  • **出力**: なし（void）。

前提条件と不変条件

  • 対象リレーションがTOASTテーブルを必要とする状態である必要があります。
  • 内部的に CommandCounterIncrement() が実行されるため、呼び出し後にカタログの変更が可視化されます。

アルゴリズムまたは振る舞い

1. CheckAndCreateToastTable を呼び出し、TOASTテーブル作成の要件判定および構築処理を委譲します。 2. 内部的に create_toast_table を経由して、TOASTテーブル本体とB-treeインデックスの作成、システムカタログ（pg_class）の更新、および依存関係（DEPENDENCY_INTERNAL）の登録を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/cluster.c: cluster コマンド実行時に呼び出されます。
  • **呼び出し先**:
  • CheckAndCreateToastTable（直接の呼び出し先）。

関連データ構造

  • Oid: リレーション識別子。
  • Datum: ストレージオプションの保持に使用。
  • LOCKMODE: ロック制御用。

失敗モードと境界条件

  • 対象がシステムカタログや共有テーブルなど、TOASTをサポートしないリレーションである場合、CheckAndCreateToastTable 内の判定ロジックにより生成がスキップされる可能性があります。

保守担当者向けメモ

  • 本関数は NewRelationCreateToastTable 等と同様の公開インターフェースですが、主に既存テーブルの再構築（CLUSTER等）の文脈で使用されます。
  • OIDOldToast が指定されている場合、既存のTOASTテーブルとの置き換え処理が行われます。
