hash_create_memory

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAgg.c:1999

目的

ハッシュ集約処理に必要な3種類のメモリコンテキスト（式評価用、メタデータ用、ハッシュテーブル用）を初期化し、AggState に割り当てる。

シグネチャまたは形状

static void hash_create_memory(AggState *aggstate)

入力と出力

  • **入力**: AggState *aggstate（集約実行状態を保持する構造体）
  • **出力**: なし（副作用として aggstate の各コンテキストフィールドを初期化）

前提条件と不変条件

  • aggstate->ss.ps.state->es_query_cxt が有効なメモリコンテキストであること。
  • work_mem グローバル変数が設定されていること。

アルゴリズムまたは振る舞い

1. **式評価コンテキスト (hashcontext)**: CreateWorkExprContext を使用して作成。参照渡しされる遷移値や AggCheckCallContext の戻り値に使用される。 2. **メタコンテキスト (hash_metacxt)**: AllocSetContextCreate を使用して作成。ハッシュテーブルのバケット配列（TupleHashEntryData）の管理に使用される。 3. **テーブルコンテキスト (hash_tablecxt)**: BumpContextCreate を使用して作成。ハッシュエントリ（グループキーと集約データ）を保持する。

  • ブロックサイズは work_mem に基づいて計算され、ALLOCSET_DEFAULT_MINSIZE から ALLOCSET_DEFAULT_MAXSIZE の範囲内に制限される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAgg.c 内のハッシュ集約初期化処理から呼び出される。
  • **呼び出し先**: CreateWorkExprContext, AllocSetContextCreate, BumpContextCreate, pg_prevpower2_size_t 等のメモリ管理関数。

関連データ構造

  • AggState: 集約実行状態を保持するメイン構造体。
  • MemoryContext: PostgreSQLのメモリ管理単位。

失敗モードと境界条件

  • work_mem が極端に小さい場合、maxBlockSize は ALLOCSET_DEFAULT_INITSIZE に切り詰められる。
  • メモリ不足時は、PostgreSQLの標準的なメモリ割り当てエラー（palloc 関連）が発生する。

保守担当者向けメモ

  • hash_tablecxt に BumpContext を採用しているのは、ハッシュテーブル全体がリセットされるまで個別のエントリが解放されないためであり、メモリ効率と速度を最適化している。
  • work_mem に応じた動的なサイズ計算を行っているため、メモリ使用量の急激な変動を抑制する設計となっている。
