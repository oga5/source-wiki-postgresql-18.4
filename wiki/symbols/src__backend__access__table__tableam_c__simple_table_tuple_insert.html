simple_table_tuple_insert

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableam.c:276

目的

テーブルへのタプル挿入処理を簡略化するラッパー関数です。複雑な引数を省略し、現在のトランザクションIDを使用して挿入を実行します。

シグネチャまたは形状

void simple_table_tuple_insert(Relation rel, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • rel: 挿入対象のテーブルリレーション。
  • slot: 挿入するタプルデータを含む TupleTableSlot。
  • **出力**: なし（成功時）。エラー発生時は elog(ERROR) を発行します。

前提条件と不変条件

  • トランザクションの整合性が保証されている環境で使用する必要があります。
  • 内部で GetCurrentCommandId(true) を呼び出すため、有効なトランザクションコンテキストが必要です。

アルゴリズムまたは振る舞い

1. GetCurrentCommandId(true) を呼び出し、現在のコマンドIDを取得します。 2. table_tuple_insert を呼び出し、以下の引数を渡して挿入を実行します。

  • rel: 対象リレーション。
  • slot: 挿入データ。
  • cid: 取得した現在のコマンドID。
  • 0: ti_options（オプションなし）。
  • NULL: bistate（バルクインサート状態は指定なし）。

    • 3. table_tuple_insert の結果コード（TM_Result）を評価し、エラーが発生した場合は即座に elog(ERROR) を発行して処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execReplication.c:600
  • **呼び出し先**:
  • GetCurrentCommandId
  • table_tuple_insert

関連データ構造

  • Relation: テーブルリレーション構造体。
  • TupleTableSlot: タプルデータを保持するスロット構造体。

失敗モードと境界条件

  • table_tuple_insert 内部でエラーが発生した場合、本関数は elog(ERROR) により呼び出し元へ戻らずにトランザクションをアボートさせる可能性があります。

保守担当者向けメモ

  • 本関数はエラーハンドリングを簡略化しているため、トランザクションの整合性が保証されている環境でのみ使用してください。
  • table_tuple_insert は relation->rd_tableam を介したディスパッチを行うため、具体的な挙動は各テーブルアクセスメソッドの実装に依存します。
