finish_nodeitem

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_io.c:599

目的

解析中の ltree または lquery におけるラベル（nodeitem）の妥当性を検証し、最終的なバイト長を確定させる。

シグネチャまたは形状

static bool finish_nodeitem(nodeitem *lptr, const char *ptr, bool is_lquery, int pos, struct Node *escontext);

入力と出力

  • **入力**:
  • lptr: 検証対象の nodeitem 構造体へのポインタ。
  • ptr: 現在の解析位置を示すポインタ。
  • is_lquery: lquery 型として処理するかを示すフラグ。
  • pos: 現在の文字位置（エラー報告用）。
  • escontext: エラー報告用コンテキスト。
  • **出力**:
  • 成功時は true を返し、lptr->len を更新する。
  • 失敗時は escontext を通じてエラーを報告し、false を返す。

前提条件と不変条件

  • lptr は有効な nodeitem 構造体を指している必要がある。
  • is_lquery が true の場合、lptr にはフラグ文字（@, *, %）が含まれている可能性がある。

アルゴリズムまたは振る舞い

1. **フラグの除外**: is_lquery が true の場合、ラベル末尾のフラグ文字（@, *, %）を遡って確認し、それらを lptr->wlen および pos から減算する。 2. **長さ計算**: ptr と lptr->start の差分から、ラベルのバイト長 lptr->len を算出する。 3. **バリデーション**:

  • ラベルが空（len == 0）の場合は構文エラーとして報告する。
  • ラベル長（wlen）が LTREE_LABEL_MAX_CHARS を超える場合は、名前が長すぎる旨のエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/ltree_io.c 内の parse_ltree および parse_lquery 関数から複数回呼び出される。
  • **呼び出し先**: ereturn マクロ（エラー報告用）。

関連データ構造

  • nodeitem: 解析中の一時的なラベル情報を保持する構造体。

失敗モードと境界条件

  • **空ラベル**: ラベルが空の場合、ERRCODE_SYNTAX_ERROR を発生させる。
  • **長大ラベル**: ラベル長が LTREE_LABEL_MAX_CHARS を超える場合、ERRCODE_NAME_TOO_LONG を発生させる。

保守担当者向けメモ

  • 本関数は ltree と lquery の両方の解析で共通利用されるため、フラグ処理の変更が両者に影響を与える点に注意すること。
