PgAioWorkerControl

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/aio/method_worker.c:70

目的

非同期I/O（AIO）ワーカープロセス方式において、全ワーカーの稼働状況を管理するための共有メモリ上の制御構造体です。

シグネチャまたは形状

typedef struct PgAioWorkerControl
{
    uint64      idle_worker_mask;
    PgAioWorkerSlot workers[FLEXIBLE_ARRAY_MEMBER];
} PgAioWorkerControl;

入力と出力

  • **入力**: ワーカーのアイドル状態の変化（idle_worker_mask の更新）。
  • **出力**: 稼働中の全ワーカーの状態（workers 配列）へのアクセス。

前提条件と不変条件

  • 共有メモリ上に配置される構造体です。
  • workers 配列は FLEXIBLE_ARRAY_MEMBER を使用しており、実行時に割り当てられるサイズに依存します。

アルゴリズムまたは振る舞い

  • idle_worker_mask を用いて、現在アイドル状態にあるワーカーをビットマスク形式で保持します。
  • クライアントプロセスは、この構造体を参照してアイドル状態のワーカーを選択し、I/Oリクエストを割り当てます。

呼び出し元と呼び出し先

  • **参照元**: src/backend/storage/aio/method_worker.c:100（静的変数 io_worker_control による参照）。
  • **呼び出し先**: 構造体内部の PgAioWorkerSlot を介して各ワーカーの制御が行われます。
  • ※具体的な関数呼び出しの連鎖については、提供されたソース excerpt には含まれていません。

関連データ構造

  • PgAioWorkerSlot: 各ワーカーの個別の状態を保持する構造体。
  • PgAioWorkerSubmissionQueue: クライアントとワーカー間で共有されるI/O投入用リングバッファ。

失敗モードと境界条件

  • idle_worker_mask のビット数（64ビット）を超えるワーカー数には対応していません。
  • 共有メモリの初期化失敗時、本構造体へのアクセスは無効となります。

保守担当者向けメモ

  • offsetof(PgAioWorkerControl, workers) を使用して、可変長配列のオフセットを計算するロジックが src/backend/storage/aio/method_worker.c:116 に存在します。メモリレイアウト変更時は注意してください。
