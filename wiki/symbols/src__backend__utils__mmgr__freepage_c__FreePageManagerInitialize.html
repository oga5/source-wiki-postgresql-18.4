FreePageManagerInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/freepage.c:182

目的

FreePageManager 構造体を初期化し、共有メモリ等の絶対アドレスが固定されない環境でページ管理を開始するための準備を行う。

シグネチャまたは形状

void FreePageManagerInitialize(FreePageManager *fpm, char *base)

入力と出力

  • **入力**:
  • fpm: 初期化対象の FreePageManager 構造体へのポインタ。
  • base: 管理対象メモリ領域のベースアドレス。相対ポインタ（relptr）の計算基準として使用される。
  • **出力**: なし（fpm が指すメモリ領域が直接更新される）。

前提条件と不変条件

  • fpm は書き込み可能なメモリ領域を指していること。
  • base は管理対象となるメモリ領域の開始位置として有効であること。

アルゴリズムまたは振る舞い

1. fpm 自身の自己参照ポインタを relptr として格納。 2. B-tree のルートおよび再利用リストを空（NULL）に初期化。 3. B-tree の深さ、再利用カウント、シングルトンページ情報をゼロクリア。 4. 連続ページ数のキャッシュを無効化（contiguous_pages_dirty = true）。 5. FPM_EXTRA_ASSERTS が有効な場合、空きページ数をゼロに設定。 6. すべてのフリーリスト（FPM_NUM_FREELISTS）を空に初期化。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/dsm.c
  • src/backend/utils/mmgr/dsa.c
  • **呼び出し先**:
  • relptr_store（マクロ）

関連データ構造

  • FreePageManager: 管理対象のメタデータを保持するメイン構造体。
  • FreePageSpanLeader: フリーリストのノードとして機能するヘッダ。
  • FreePageBtree: メモリ断片化を管理するための B-tree 構造。

失敗モードと境界条件

  • 本関数自体はメモリ確保を行わないため、直接的な失敗モードはない。ただし、渡される fpm ポインタが不正な場合、メモリ破壊を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は管理対象領域のメタデータを初期化するのみであり、実際のメモリ領域の割り当てや解放は FreePageManagerGet および FreePageManagerPut が担当する。
  • relptr を使用しているため、base アドレスの指定が正しくない場合、以降のすべてのメモリ管理操作が誤ったアドレスを参照することになる点に注意すること。
