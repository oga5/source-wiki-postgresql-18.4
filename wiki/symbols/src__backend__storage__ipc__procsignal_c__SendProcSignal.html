SendProcSignal

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procsignal.c:284

目的

指定されたプロセス（PID）に対して、共有メモリを介したシグナル通知（SIGUSR1）を行う。

シグネチャまたは形状

int SendProcSignal(pid_t pid, ProcSignalReason reason, ProcNumber procNumber);

入力と出力

  • **入力**:
  • pid: 送信先プロセスのプロセスID。
  • reason: ProcSignalReason 型の通知理由。
  • procNumber: 送信先プロセスのインデックス（不明な場合は INVALID_PROC_NUMBER を指定）。
  • **出力**:
  • 成功時: kill() の戻り値（通常は 0）。
  • 失敗時: -1（errno に ESRCH が設定される）。

前提条件と不変条件

  • procNumber が指定されている場合、その値は NumProcSignalSlots 未満である必要がある。
  • 共有メモリ上の ProcSignalSlot 内のフラグ操作は、pss_mutex スピンロックによって保護される。
  • pss_pid の確認は、スピンロック取得前後でアトミックに行われる。

アルゴリズムまたは振る舞い

1. procNumber が有効な場合、該当するスロットを直接参照する。 2. procNumber が INVALID_PROC_NUMBER の場合、全スロットを後ろから前へ走査し、指定された pid と一致するスロットを検索する。 3. 一致するスロットが見つかった場合：

  • pss_mutex を取得し、対象プロセスの pss_signalFlags[reason] を true に設定する。
  • pss_mutex を解放する。
  • kill(pid, SIGUSR1) を実行して対象プロセスにシグナルを送信する。

    • 4. 対象が見つからない場合は errno を ESRCH に設定し -1 を返す。

関連データ構造

  • ProcSignalSlot: プロセスごとのシグナル状態を保持する構造体。
  • ProcSignalHeader: 全スロットを管理する共有メモリ上のヘッダー。

失敗モードと境界条件

  • 指定された pid に対応するスロットが存在しない場合、ESRCH を返し失敗する。
  • procNumber が INVALID_PROC_NUMBER の場合、全スロットを走査するため、スロット数が多い場合にオーバーヘッドが発生する。

保守担当者向けメモ

  • SIGUSR1 は多重化されているため、この関数は「通知フラグを立てる」ことと「シグナルを送る」ことのペアで機能する。
  • 補助プロセスはスロット配列の後方に配置される傾向があるため、INVALID_PROC_NUMBER 指定時は後ろからの走査が最適化として機能する。
