ExecInterpExpr

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/execExprInterp.c:467

目的

PostgreSQL の式評価エンジンにおけるメインループとして、コンパイル済みの ExprEvalStep 命令リストを順次解釈・実行する。

シグネチャまたは形状

static Datum ExecInterpExpr(ExprState *state, ExprContext *econtext, bool *isnull)

入力と出力

  • **入力**:
  • state: 実行対象の式状態（命令リストを保持）。
  • econtext: 実行コンテキスト（入力タプルやメモリ状態を保持）。
  • isnull: 結果が NULL かどうかを格納するフラグへのポインタ。
  • **出力**:
  • 評価結果の Datum を返す。

前提条件と不変条件

  • state が NULL の場合、ディスパッチテーブルのアドレスを返す（EEO_USE_COMPUTED_GOTO 定義時）。
  • state が非 NULL の場合、state->steps に基づいて命令を実行する。
  • dispatch_table のサイズは EEOP_LAST + 1 と一致しなければならない（StaticAssertDecl による検証）。

アルゴリズムまたは振る舞い

1. **初期化**: ExprContext から各スロット（inner, outer, scan 等）を取得し、実行状態をセットアップする。 2. **ディスパッチ**:

  • EEO_USE_COMPUTED_GOTO が有効な場合、EEO_DISPATCH() マクロにより直接スレッド化された命令へジャンプする。
  • 無効な場合、EEO_SWITCH() マクロによる標準的な switch 文で命令を分岐する。

    • 3. **実行**: 各 ExprEvalStep の opcode に応じて処理を実行し、結果を op->resvalue に格納する。 4. **遷移**: EEO_NEXT() マクロで次のステップへ進むか、EEO_JUMP() で特定のステップへ制御を移す。 5. **終了**: EEOP_DONE_RETURN 命令に到達すると、結果を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c 内の state->evalfunc_private に割り当てられる。
  • 外部からの直接的な呼び出し元情報は提供されていない。
  • **呼び出し先**:
  • CheckOpSlotCompatibility 等の検証関数。
  • slot_getsomeattrs 等のタプル操作関数。
  • 各 EEOP_* 命令に対応するコードブロック。

関連データ構造

  • ExprState: 実行する命令リストと結果格納先を保持。
  • ExprEvalStep: 実行単位となる命令構造体。
  • ExprContext: 実行時のタプルスロットやメモリコンテキストを保持。
  • ExprEvalOp: 命令の種類を定義する列挙型。

失敗モードと境界条件

  • state が NULL の場合、ディスパッチテーブルを返す特殊な動作を行う。
  • isnull が NULL の場合、EEOP_DONE_NO_RETURN 命令でアサーションエラーが発生する可能性がある。
  • CheckOpSlotCompatibility 等の検証関数により、スロットと命令の不整合が検出されるとエラーとなる。

保守担当者向けメモ

  • 本関数は PostgreSQL のホットパスであるため、命令の追加時はインライン化や分岐予測効率を考慮すること。
  • EEO_USE_COMPUTED_GOTO を無効にすると switch 文ベースのデバッグが容易になる。
  • 命令の順序は ExprEvalOp 列挙型と完全に一致させる必要がある。
