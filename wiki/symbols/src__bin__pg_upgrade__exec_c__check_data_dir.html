check_data_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/exec.c:340

目的

PostgreSQL クラスタのデータディレクトリ配下に、アップグレードに必要な必須サブディレクトリが存在するかを検証する。

シグネチャまたは形状

static void check_data_dir(ClusterInfo *cluster);

入力と出力

  • **入力**: ClusterInfo *cluster（検証対象のクラスタ情報構造体。pgdata パスおよびメジャーバージョン情報を含む）
  • **出力**: なし（検証失敗時は内部でエラー処理が実行される）

前提条件と不変条件

  • 引数 cluster が有効な ClusterInfo 構造体を指していること。
  • cluster->pgdata にデータディレクトリのパスが格納されていること。

アルゴリズムまたは振る舞い

1. get_major_server_version(cluster) を呼び出し、対象クラスタのメジャーバージョンを取得して cluster->major_version に格納する。 2. 共通の必須ディレクトリ（base, global, pg_multixact, pg_subtrans, pg_tblspc, pg_twophase）に対して check_single_dir を実行し、存在を確認する。 3. バージョン依存のディレクトリ検証を行う:

  • PostgreSQL 9.6 以下の場合: pg_xlog および pg_clog を検証する。
  • PostgreSQL 10 以降の場合: pg_wal および pg_xact を検証する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/exec.c 内の verify_directories 関数から、旧クラスタおよび新クラスタの検証のために呼び出される。
  • **呼び出し先**:
  • get_major_server_version
  • check_single_dir

関連データ構造

  • ClusterInfo: pg_upgrade.h で定義される、クラスタのパスやバージョン情報を保持する構造体。

失敗モードと境界条件

  • 指定されたディレクトリが存在しない場合、check_single_dir 経由でエラーが発生する。
  • バージョン判定（GET_MAJOR_VERSION）に基づき、古いディレクトリ名と新しいディレクトリ名のいずれか一方が正しく存在することを確認する。

保守担当者向けメモ

  • 本関数は PostgreSQL のバージョンアップに伴うディレクトリ名の変更（pg_xlog → pg_wal 等）を吸収する役割を持つ。将来的にディレクトリ構成が変更された場合は、この条件分岐を更新する必要がある。
