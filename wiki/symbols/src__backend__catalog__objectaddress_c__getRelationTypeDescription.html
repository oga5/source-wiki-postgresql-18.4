getRelationTypeDescription

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaddress.c:4686

目的

指定されたリレーション（テーブル、インデックス等）のOIDに基づき、そのオブジェクトの種類を表す文字列を生成し、提供された StringInfo バッファに追記します。

シグネチャまたは形状

void getRelationTypeDescription(StringInfo buffer, Oid relid, int32 objectSubId, bool missing_ok)

入力と出力

  • **入力**:
  • buffer: 出力先となる StringInfo 構造体へのポインタ。
  • relid: 対象リレーションのOID。
  • objectSubId: 列番号（0以外の場合、説明に " column" が付加される）。
  • missing_ok: true の場合、リレーションが見つからなくてもエラーにせず "relation" と出力する。
  • **出力**: なし（buffer を直接更新）。

前提条件と不変条件

  • relid に対応する pg_class エントリがシステムキャッシュ（RELOID）から取得可能であること。
  • missing_ok が false の場合、キャッシュ検索失敗は elog(ERROR) を引き起こします。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して pg_class からリレーション情報を取得します。 2. キャッシュミス時: missing_ok が false ならエラー終了。true なら "relation" をバッファに追記して終了します。 3. キャッシュヒット時: relkind フィールドを評価し、対応する文字列（"table", "index", "sequence", "view" 等）をバッファに追記します。 4. objectSubId が 0 でない場合、バッファに " column" を追記します。 5. ReleaseSysCache を呼び出し、取得したタプルを解放します。

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • Form_pg_class: pg_class システムカタログの行構造体。

失敗モードと境界条件

  • **キャッシュ検索失敗**: missing_ok が false の場合、elog(ERROR) によりトランザクションが中断されます。
  • **未知の relkind**: switch 文の default 節により、定義外の relkind が渡された場合は安全策として "relation" と出力されます。

保守担当者向けメモ

  • 新しい relkind が追加された場合は、本関数の switch 文にもケースを追加してください。
  • 本関数は pg_class のキャッシュに依存しているため、カタログの整合性が保たれている環境での実行を前提としています。
