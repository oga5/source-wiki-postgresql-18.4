JitInstrumentation

定義

  • 種別: struct
  • 定義箇所: src/include/jit/jit.h:27

目的

JITコンパイル処理の各フェーズ（コード生成、最適化、インライン化、コード出力等）にかかるコストを計測・集計するための構造体です。

シグネチャまたは形状

typedef struct JitInstrumentation
{
    size_t      created_functions;
    instr_time  generation_counter;
    instr_time  deform_counter;
    instr_time  inlining_counter;
    instr_time  optimization_counter;
    instr_time  emission_counter;
} JitInstrumentation;

入力と出力

  • 入力: JITコンパイルの実行過程で各カウンタが更新されます。
  • 出力: InstrJitAgg 関数等を通じて集計され、ExplainPrintJIT 等で統計情報として出力されます。

前提条件と不変条件

  • deform_counter は generation_counter に含まれる値として扱われます。
  • instr_time 型（executor/instrument.h 由来）を使用して時間を保持します。

アルゴリズムまたは振る舞い

JITコンパイルの各段階で経過時間を累積します。

  • created_functions: 生成された関数数。
  • generation_counter: コード生成全体の累積時間。
  • deform_counter: タプルデフォームの累積時間。
  • inlining_counter: インライン化の累積時間。
  • optimization_counter: 最適化の累積時間。
  • emission_counter: コード出力の累積時間。

呼び出し元と呼び出し先

  • 呼び出し元（参照・操作）:
  • src/backend/jit/jit.c: InstrJitAgg による集計。
  • src/backend/commands/explain.c: ExplainPrintJIT による統計表示。
  • src/backend/executor/execParallel.c: パラレルクエリ実行時のワーカー間集計。
  • contrib/pg_stat_statements/pg_stat_statements.c: 統計情報の記録。
  • 呼び出し先:
  • executor/instrument.h で定義された時間計測機能を利用します。

関連データ構造

  • SharedJitInstrumentation: 並列実行時に共有される計測データ構造。
  • instr_time: 時間計測用の基底型。

失敗モードと境界条件

  • 並列クエリ実行時、ワーカー数に応じたメモリ確保（sizeof(JitInstrumentation) * nworkers）が必要です。
  • 初期化されていない場合、ゼロ初期化（{0}）が必要です。

保守担当者向けメモ

  • 本構造体はプロバイダ非依存です。LLVM固有の計測を追加する場合は、llvmjit.h 側の構造体拡張を検討してください。
  • generation_counter と他のカウンタの包含関係（deform_counter が含まれる等）を維持するように更新してください。
