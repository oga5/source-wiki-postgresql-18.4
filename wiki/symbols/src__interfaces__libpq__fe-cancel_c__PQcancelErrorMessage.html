PQcancelErrorMessage

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:324

目的

キャンセル接続に関連する直近のエラーメッセージを取得します。

シグネチャまたは形状

char *PQcancelErrorMessage(const PGcancelConn *cancelConn);

入力と出力

  • **入力**: cancelConn (キャンセル接続オブジェクトへのポインタ)
  • **出力**: エラーメッセージを含む文字列へのポインタを返します。

前提条件と不変条件

  • cancelConn は有効な PGcancelConn 構造体を指している必要があります。
  • 内部的に PGcancelConn が保持する conn 構造体に対して PQerrorMessage を呼び出します。

アルゴリズムまたは振る舞い

1. 引数として受け取った cancelConn 構造体内部の conn メンバーを参照します。 2. PQerrorMessage を呼び出し、その結果をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/libpq/libpq-be-fe-helpers.h
  • src/test/isolation/isolationtester.c
  • src/test/modules/libpq_pipeline/libpq_pipeline.c
  • **呼び出し先**:
  • PQerrorMessage (libpq 内部関数)

関連データ構造

  • PGcancelConn: キャンセル要求の状態を管理する構造体。

失敗モードと境界条件

  • cancelConn が NULL の場合、内部の PQerrorMessage の挙動に依存します（通常はセグメンテーション違反等の未定義動作を引き起こす可能性があります）。

保守担当者向けメモ

  • 本関数は PQerrorMessage のラッパーです。エラーメッセージの取得ロジックを変更する場合は、libpq 全体のエラーハンドリング方針に従う必要があります。
