ClearOrSaveResult

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:557

目的

クエリ実行結果（PGresult）を評価し、エラー情報として保持すべきか、あるいは不要なリソースとして解放すべきかを判断する。

シグネチャまたは形状

static void ClearOrSaveResult(PGresult *result)

入力と出力

  • **入力**: PGresult *result（処理対象のクエリ結果ポインタ）
  • **出力**: なし（副作用として pset.last_error_result の更新または PQclear によるメモリ解放を行う）

前提条件と不変条件

  • result が NULL の場合、本関数は何もしない。
  • pset.last_error_result は、エラー発生時に最新のエラー結果を保持するために使用される。

アルゴリズムまたは振る舞い

1. result が NULL でないことを確認する。 2. PQresultStatus(result) を評価する。

  • PGRES_NONFATAL_ERROR または PGRES_FATAL_ERROR の場合:
  • 既存の pset.last_error_result を PQclear で解放する。
  • pset.last_error_result に現在の result を代入し、エラー情報を保持する。
  • それ以外の場合:
  • PQclear(result) を呼び出し、リソースを即座に解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/common.c 内の複数の箇所（ExecQueryAndProcessResults 等のクエリ処理ループ内）から呼び出されている。
  • **呼び出し先**: PQresultStatus（libpq）, PQclear（libpq）。

関連データ構造

  • PGresult: libpq が提供するクエリ結果構造体。
  • pset: psql のグローバル設定を保持する構造体（src/bin/psql/settings.h で定義）。last_error_result フィールドを介してエラー状態を管理する。

失敗モードと境界条件

  • result が NULL の場合は安全に無視される。
  • PQclear は NULL ポインタを渡しても安全な関数であるため、pset.last_error_result が初期状態で NULL であっても問題なく動作する。

保守担当者向けメモ

  • 本関数はエラー結果の「最新のもの」のみを保持する設計である。複数のエラーが発生した場合、古いエラー結果は PQclear によって破棄される。
  • pset.last_error_result の所有権が本関数に移転する点に注意すること。
