ExecInitWorkTableScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWorktablescan.c:129

目的

再帰的クエリ（WITH RECURSIVE）における作業テーブルスキャンノードの初期化を行い、実行に必要な状態構造体（WorkTableScanState）を構築する。

シグネチャまたは形状

WorkTableScanState *ExecInitWorkTableScan(WorkTableScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の WorkTableScan プランノード。
  • estate: 実行状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの WorkTableScanState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK を含めることはできない（アサーションにより強制）。
  • WorkTableScan ノードは子プランを持たない（outerPlan および innerPlan が NULL であること）。
  • rustate（RecursiveUnionState への参照）は、初期化時点では NULL に設定され、初回実行時に遅延解決される。

アルゴリズムまたは振る舞い

1. WorkTableScanState 構造体を割り当て、プランノードと実行状態を関連付ける。 2. ExecProcNode 関数ポインタを ExecWorkTableScan に設定する。 3. ExecAssignExprContext を呼び出し、式評価用のコンテキストを作成する。 4. ExecInitResultTypeTL を呼び出し、結果のタプル型を初期化する。 5. ExecInitScanTupleSlot を呼び出し、スキャン用のタプルスロットを TTSOpsMinimalTuple 形式で初期化する。 6. ExecInitQual を呼び出し、プランノードの制約（qual）を初期化する。 7. プロジェクション情報の初期化は行わず、実行時の ExecWorkTableScan に委ねる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode から呼び出されることが確認されている。
  • **呼び出し先**: makeNode, ExecAssignExprContext, ExecInitResultTypeTL, ExecInitScanTupleSlot, ExecInitQual などの実行エンジン共通ユーティリティ関数を呼び出す。

関連データ構造

  • WorkTableScan: プランノード。
  • WorkTableScanState: 実行状態を保持するノード。
  • RecursiveUnionState: rustate フィールドを通じて後続の実行フェーズで参照される。

失敗モードと境界条件

  • 不正な eflags が渡された場合、アサーション違反により停止する。
  • 子プランが存在する場合、アサーション違反により停止する。

保守担当者向けメモ

  • rustate の解決が遅延されるのは、RecursiveUnion ノードとの初期化順序依存を回避するためである。
  • プロジェクション情報の初期化を意図的にスキップしているため、変更を加える際は ExecWorkTableScan 側のロジックと整合性を確認すること。
