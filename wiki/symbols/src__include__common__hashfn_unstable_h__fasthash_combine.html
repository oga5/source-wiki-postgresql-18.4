fasthash_combine

定義

  • 種別: function
  • 定義箇所: src/include/common/hashfn_unstable.h:127

目的

ハッシュ計算の進行状況を保持する fasthash_state 内の蓄積値（accum）を、現在のハッシュ値（hash）に混合し、状態を更新する。

シグネチャまたは形状

static inline void fasthash_combine(fasthash_state *hs)

入力と出力

  • **入力**: fasthash_state *hs（ハッシュ計算状態へのポインタ）
  • **出力**: なし（hs が指す構造体の hash フィールドを直接更新する）

前提条件と不変条件

  • hs は fasthash_init() によって初期化済みであること。
  • 本関数は「不安定」なハッシュアルゴリズムの一部であり、プラットフォームやバージョン間で結果が一致する保証はない。永続化データには使用しないこと。

アルゴリズムまたは振る舞い

1. fasthash_mix(hs->accum, 0) を呼び出し、蓄積された値に対して混合処理を行う。 2. 現在の hs->hash と上記の結果を XOR 演算する。 3. その結果に定数 0x880355f21e6d1965 を乗算し、hs->hash を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/namespace.c:262
  • src/include/common/hashfn_unstable.h:210, 274
  • その他、ヘッダー内のドキュメント例として言及あり。
  • **呼び出し先**:
  • fasthash_mix

関連データ構造

  • fasthash_state: ハッシュ計算の進行状況（hash, accum 等）を保持する構造体。

失敗モードと境界条件

  • hs が NULL の場合、未定義の動作を引き起こす可能性がある。
  • 本関数はプラットフォームのエンディアンに依存する。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスが重視される箇所で使用される。
  • 変更を加える際は、fasthash アルゴリズム全体の整合性および SMHasher テストへの影響を考慮すること。
