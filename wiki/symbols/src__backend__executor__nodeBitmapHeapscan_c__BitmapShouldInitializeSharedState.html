BitmapShouldInitializeSharedState

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapHeapscan.c:419

目的

並列ビットマップヒープスキャンにおいて、共有状態の初期化が必要かどうかを判定し、必要に応じて初期化プロセスを同期させる。

シグネチャまたは形状

static bool BitmapShouldInitializeSharedState(ParallelBitmapHeapState *pstate)

入力と出力

  • **入力**: ParallelBitmapHeapState *pstate（並列実行時の共有状態構造体）
  • **出力**: bool 型。呼び出し元が共有状態の初期化を担当すべき場合は true、それ以外（既に初期化済み、または他プロセスが初期化中）の場合は false を返す。

前提条件と不変条件

  • pstate は有効な共有メモリ上の構造体を指していること。
  • pstate->mutex を用いた排他制御により、pstate->state の遷移が保護されていること。

アルゴリズムまたは振る舞い

1. 無限ループ内で pstate->mutex を取得し、現在の pstate->state を確認する。 2. 状態が BM_INITIAL であれば、BM_INPROGRESS に更新してループを抜け、true を返す（初期化担当の確定）。 3. 状態が BM_INPROGRESS の場合、ConditionVariableSleep を呼び出し、リーダープロセスによる状態更新を待機する。 4. 状態がそれ以外（初期化完了など）であればループを抜け、false を返す。 5. ループ終了後、ConditionVariableCancelSleep を呼び出して待機状態を解除する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeBitmapHeapscan.c 内の並列スキャン実行ロジック（BitmapTableScanSetup 等からの参照が確認されている）。
  • **呼び出し先**: SpinLockAcquire, SpinLockRelease, ConditionVariableSleep, ConditionVariableCancelSleep。

関連データ構造

  • ParallelBitmapHeapState: 並列ビットマップスキャンの共有状態を保持する構造体。
  • SharedBitmapState: BM_INITIAL や BM_INPROGRESS などの状態値を定義する列挙型。

失敗モードと境界条件

  • 共有メモリの競合が発生した場合、ConditionVariable による待機が発生し、デッドロックを避けるために適切なリーダープロセスの進行が前提となる。

保守担当者向けメモ

  • 本関数は並列ワーカー間の同期の要です。ConditionVariable の使用箇所であるため、待機イベント WAIT_EVENT_PARALLEL_BITMAP_SCAN の発生状況を監視する際は注意してください。
