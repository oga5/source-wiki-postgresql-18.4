compare3

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conv.c:319

目的

bsearch 関数で使用される比較関数であり、2つの結合文字（combined characters）のコードポイントを比較して順序を決定します。

シグネチャまたは形状

static int compare3(const void *p1, const void *p2)

入力と出力

  • **入力**:
  • p1: 比較対象のキーとなるデータ（uint32 の配列を指すポインタ）。
  • p2: 比較対象の pg_utf_to_local_combined 構造体へのポインタ。
  • **出力**:
  • 1: p1 が p2 より大きい場合。
  • 0: p1 と p2 が等しい場合。
  • -1: p1 が p2 より小さい場合。

前提条件と不変条件

  • p1 は少なくとも2つの uint32 要素を持つ配列を指している必要があります。
  • p2 は pg_utf_to_local_combined 型の構造体としてキャスト可能である必要があります。

アルゴリズムまたは振る舞い

1. p1 から2つの uint32 値（s1, s2）を読み取ります。 2. p2 が指す構造体から2つの uint32 値（d1, d2）を読み取ります。 3. 辞書順比較を行います。まず s1 と d1 を比較し、等しい場合は s2 と d2 を比較します。 4. 比較結果に基づき、1, 0, -1 のいずれかを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mb/conv.c 内の bsearch を呼び出す箇所（例: 635行目付近）。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • pg_utf_to_local_combined: 結合文字の変換テーブルで使用される構造体。utf1 および utf2 フィールドを持ちます。

失敗モードと境界条件

  • 入力ポインタが NULL である場合の動作は定義されていません。
  • p1 が指すメモリ領域が uint32 2つ分未満である場合、メモリ破壊や不正な読み取りが発生する可能性があります。

保守担当者向けメモ

  • 本関数は bsearch のコールバックとして設計されています。比較ロジックを変更する場合は、pg_utf_to_local_combined 構造体の定義と整合性が取れていることを確認してください。
