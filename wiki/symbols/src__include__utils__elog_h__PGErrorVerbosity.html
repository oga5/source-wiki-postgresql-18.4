PGErrorVerbosity

定義

  • 種別: enum
  • 定義箇所: src/include/utils/elog.h:483

目的

エラーメッセージの出力詳細度（冗長性）を制御するための列挙型です。

シグネチャまたは形状

typedef enum
{
    PGERROR_TERSE,    /* 単一行のエラーメッセージ */
    PGERROR_DEFAULT,  /* 推奨されるスタイル */
    PGERROR_VERBOSE,  /* すべての詳細情報 */
} PGErrorVerbosity;

入力と出力

  • 入力: なし（列挙値の定義）
  • 出力: PGERROR_TERSE, PGERROR_DEFAULT, PGERROR_VERBOSE のいずれかの値

前提条件と不変条件

  • PGERROR_DEFAULT が標準的な推奨スタイルとして扱われます。

アルゴリズムまたは振る舞い

  • PGERROR_TERSE: エラーメッセージを単一行に制限します。
  • PGERROR_DEFAULT: PostgreSQL の標準的なエラー報告スタイルを提供します。
  • PGERROR_VERBOSE: エラーに関するすべての利用可能な情報を出力します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な情報は、提供されたソース抜粋からは確認できません。

関連データ構造

  • ErrorData: エラー発生時に収集される情報を保持する構造体（src/include/utils/elog.h で定義）。

失敗モードと境界条件

  • 該当なし（列挙型の定義のため）。

保守担当者向けメモ

  • 本シンボルはエラー報告の出力形式を決定する設定値として利用されます。
  • ログ出力先（stderr, syslog, CSV, JSON 等）の抽象化レイヤーにおいて、メッセージのフィルタリングやフォーマット決定の基準として参照される可能性があります。
