InitializeLWLocks

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lwlock.c:501

目的

共有メモリ上に配置されるすべての軽量ロック（LWLock）を初期化し、名前付きトランチ（Tranche）の情報を共有メモリへコピーして設定する。

シグネチャまたは形状

static void InitializeLWLocks(void)

入力と出力

  • 入力: なし（グローバル変数 MainLWLockArray および NamedLWLockTrancheRequestArray を参照）
  • 出力: なし（共有メモリ上の MainLWLockArray および NamedLWLockTrancheArray を更新）

前提条件と不変条件

  • 共有メモリが既に割り当てられていること。
  • MainLWLockArray が初期化可能な状態であること。
  • NamedLWLockTrancheRequests が 0 以上の値であること。

アルゴリズムまたは振る舞い

1. **固定 LWLock の初期化**: MainLWLockArray 内の個別のロック、バッファマッピング用ロック、ロックマネージャ用ロック、述語ロックマネージャ用ロックを、それぞれのトランチ ID を指定して LWLockInitialize で初期化する。 2. **名前付きトランチの処理**: NamedLWLockTrancheRequests > 0 の場合、以下の処理を行う。

  • NamedLWLockTrancheArray を MainLWLockArray の末尾に配置する。
  • 各リクエストからトランチ名とロック数を取得し、共有メモリ上の領域にコピーする。
  • LWLockNewTrancheId() を呼び出して一意なトランチ ID を割り当てる。
  • 指定された数の LWLock を、割り当てたトランチ ID で初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/lwlock.c 内の初期化プロセス（CreateLWLocks 等からの呼び出しが示唆される）。
  • 呼び出し先: LWLockInitialize（各ロックの初期化）、LWLockNewTrancheId（トランチ ID の生成）、NumLWLocksForNamedTranches。

関連データ構造

  • MainLWLockArray: 全 LWLock を保持する共有メモリ上の配列。
  • LWLockPadded: パディングされた LWLock 構造体。
  • NamedLWLockTranche: 名前付きトランチのメタデータ構造体。
  • NamedLWLockTrancheRequest: 初期化リクエストを保持する構造体。

失敗モードと境界条件

  • NamedLWLockTrancheRequests が 0 の場合、名前付きトランチの初期化はスキップされる。
  • メモリ配置計算において、MainLWLockArray のサイズが不足している場合は未定義動作となる可能性がある。

保守担当者向けメモ

  • 本関数はシステム起動時の共有メモリ初期化フェーズで実行される。
  • MainLWLockArray のオフセット計算（BUFFER_MAPPING_LWLOCK_OFFSET 等）は、lwlock.h で定義された定数に依存しているため、構造変更時は注意が必要。
