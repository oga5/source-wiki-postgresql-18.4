unaccent_dict

定義

  • 種別: function
  • 定義箇所: contrib/unaccent/unaccent.c:439

目的

SQLから直接呼び出し可能なアクセント除去用のラッパー関数であり、指定されたテキスト検索辞書を使用して入力文字列のアクセント除去処理を実行します。

シグネチャまたは形状

Datum unaccent_dict(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 引数が1つの場合: 対象文字列（暗黙的に "unaccent" 辞書を使用）。
  • 引数が2つの場合: 第1引数に辞書OID、第2引数に対象文字列。
  • **出力**: text 型のアクセント除去済み文字列を返します。

前提条件と不変条件

  • 引数が1つの場合、関数が属するスキーマ内に "unaccent" という名称のテキスト検索辞書が存在する必要があります。
  • 辞書キャッシュから取得した lexize 関数が正常に動作し、結果を TSLexeme 構造体として返却できること。

アルゴリズムまたは振る舞い

1. **辞書特定**: 引数の数に応じて辞書OIDを決定します。1引数の場合は現在のスキーマから "unaccent" 辞書を検索します。 2. **キャッシュ参照**: lookup_ts_dictionary_cache を使用して辞書エントリを取得します。 3. **変換実行**: 辞書内の lexize 関数を呼び出し、入力文字列を処理します。 4. **結果処理**:

  • res == NULL または res->lexeme == NULL の場合: 変換不要とみなし、元の文字列のコピーを返します。
  • それ以外の場合: res->lexeme を text 型に変換して返却し、メモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • get_func_namespace, GetSysCacheOid2, lookup_ts_dictionary_cache (PostgreSQL内部API)
  • dict->lexize (辞書固有の変換関数)

関連データ構造

  • TSDictionaryCacheEntry: 辞書キャッシュエントリ。
  • TSLexeme: 変換結果を保持する構造体。

失敗モードと境界条件

  • 指定された辞書が見つからない場合、ERRCODE_UNDEFINED_OBJECT エラーを発生させます。
  • メモリ不足時はPostgreSQLの標準的なエラーハンドリングに従います。

保守担当者向けメモ

  • 本関数は unaccent 辞書をSQLから利用するためのインターフェースです。辞書の実装ロジックそのものは unaccent_lexize に依存しています。
  • PG_FREE_IF_COPY を使用して、入力文字列のメモリ管理を適切に行っています。
