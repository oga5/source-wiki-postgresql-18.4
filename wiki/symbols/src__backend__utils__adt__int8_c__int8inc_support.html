int8inc_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int8.c:825

目的

ウィンドウ関数における単調性（monotonicity）の情報をプランナに提供し、最適化を支援します。

シグネチャまたは形状

Datum int8inc_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して SupportRequestWFuncMonotonic 構造体へのポインタを受け取ります。
  • **出力**: SupportRequestWFuncMonotonic 構造体の monotonic フィールドを更新し、そのポインタを Datum として返します。要求が対象外の場合は NULL を返します。

前提条件と不変条件

  • 入力された rawreq が SupportRequestWFuncMonotonic 型であることを前提としています。
  • ウィンドウ句の ORDER BY 句の有無およびフレームオプション（FRAMEOPTION_START_UNBOUNDED_PRECEDING 等）に基づいて単調性を判定します。

アルゴリズムまたは振る舞い

1. 入力された rawreq が SupportRequestWFuncMonotonic かどうかを IsA マクロで確認します。 2. ORDER BY 句が存在しない場合、すべての行がピア（同等）とみなされ、MONOTONICFUNC_BOTH（増加かつ減少）としてマークします。 3. ORDER BY 句が存在する場合、フレームオプションを評価します。

  • FRAMEOPTION_START_UNBOUNDED_PRECEDING が設定されていれば、値は減少しないため MONOTONICFUNC_INCREASING を付与します。
  • FRAMEOPTION_END_UNBOUNDED_FOLLOWING が設定されていれば、値は減少しないため MONOTONICFUNC_DECREASING を付与します。

    • 4. 判定結果を req->monotonic に格納し、構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、プランナのサポート関数呼び出し機構から呼び出されることが示唆されています。
  • **呼び出し先**: PostgreSQLのノード型判定マクロ IsA を使用しています。

関連データ構造

  • SupportRequestWFuncMonotonic: ウィンドウ関数の単調性情報を保持する構造体。
  • WindowClause: ウィンドウ句の定義（フレームオプションやソート順を含む）。

失敗モードと境界条件

  • rawreq が SupportRequestWFuncMonotonic 以外の型である場合、関数は単に NULL を返します。
  • ORDER BY 句がない場合、単調性は MONOTONICFUNC_BOTH と判定されます。

保守担当者向けメモ

  • 本関数はプランナの最適化パスの一部であり、ウィンドウ関数の実行計画生成時にのみ関与します。
  • 判定ロジックは WindowClause のフレーム定義に強く依存しているため、フレーム定義の仕様変更時には本関数のロジックも再検証が必要です。
