Size

定義

  • 種別: typedef
  • 定義箇所: src/include/utils/expandeddatum.h:70

目的

メモリ上のデータサイズやオフセット、バッファ長などを表現するための汎用的なサイズ型です。

シグネチャまたは形状

typedef size_t Size;

※C言語の標準的な size_t に準拠した型定義です。

入力と出力

  • 入力: 該当なし（型定義のため）
  • 出力: 該当なし

前提条件と不変条件

  • 非負の整数値であることを前提とします。
  • メモリ割り当てやデータ構造のサイズ計算において、プラットフォームのポインタサイズに依存した最大値（SIZE_MAX）を上限とします。

アルゴリズムまたは振る舞い

  • メモリ内のデータサイズ計算、バッファの確保、およびデータ構造のオフセット計算に使用されます。
  • expandeddatum.h 内では、特に EOM_flatten_into_method の引数として、展開済みオブジェクトをフラット化する際に割り当てられたサイズを渡すために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL 全体で広範に使用されています。src/include/utils/expandeddatum.h 内では EOM_flatten_into_method の定義で参照されています。
  • 呼び出し先: 該当なし（基本型のため）。

関連データ構造

  • ExpandedObjectHeader: 展開済みオブジェクトのヘッダ構造体。
  • EOM_flatten_into_method: Size を引数に取るメソッド定義。

失敗モードと境界条件

  • 整数オーバーフロー: サイズ計算時に Size 型の最大値を超過する場合、メモリ割り当てエラーや不正なメモリ操作を引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは PostgreSQL 全体で共通の型定義であるため、変更時はシステム全体への影響を考慮してください。
  • expandeddatum.h においては、メモリ管理やフラット化処理の際のサイズ指定として不可欠な型です。
