gtsvector_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsgistidx.c:324

目的

GiST インデックスにおいて、tsvector 型のインデックスキーと tsquery 条件を照合し、検索条件に合致する可能性があるかを判定します。

シグネチャまたは形状

Datum gtsvector_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • fcinfo (PG_FUNCTION_ARGS): 関数呼び出し情報。
  • entry (GISTENTRY*): インデックスエントリ。
  • query (TSQuery): 検索対象のクエリ。
  • recheck (bool*): 呼び出し元へ返す再チェックフラグ。
  • **出力**:
  • Datum (bool): 検索条件に合致する可能性がある場合は true、そうでない場合は false。

前提条件と不変条件

  • 本関数が返す結果は常に「不正確（inexact）」であるため、*recheck は常に true に設定されます。
  • query が空の場合、常に false を返します。

アルゴリズムまたは振る舞い

1. *recheck を true に設定します。 2. query が空であれば false を返します。 3. インデックスキーの種類に応じて処理を分岐します：

  • **ビットベクトル形式 (ISSIGNKEY)**:
  • ISALLTRUE が真であれば true を返します。
  • それ以外の場合、TS_execute を呼び出し、コールバック関数 checkcondition_bit を用いてビットレベルの照合を行います。
  • **配列形式 (リーフページ)**:
  • CHKVAL 構造体に配列情報を格納し、TS_execute を呼び出し、コールバック関数 checkcondition_arr を用いて配列要素の照合を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックスアクセスメソッド（詳細はソースコード上の明示的な呼び出し元情報なし）。
  • **呼び出し先**:
  • TS_execute: クエリ実行エンジン。
  • checkcondition_bit: ビットベクトル用照合コールバック。
  • checkcondition_arr: 配列用照合コールバック。

関連データ構造

  • SignTSVector: インデックスキーの構造体。
  • GISTENTRY: GiST インデックスエントリ。
  • TSQuery: 全文検索クエリ。
  • CHKVAL: リーフページでの照合用一時構造体。

失敗モードと境界条件

  • query->size が 0 の場合、検索は行われず false が返されます。
  • インデックスキーがビットベクトルか配列かによって、照合ロジックが完全に分離されます。

保守担当者向けメモ

  • 本関数は GiST インデックスの「不正確」な性質を体現しており、インデックスによるフィルタリング後にヒープ上の実データに対する再チェックが必須です。
  • TS_EXEC_PHRASE_NO_POS フラグが TS_execute に渡されており、位置情報は考慮されません。
