ExecInitGenerated

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeModifyTable.c:429

目的

テーブルの生成列（Generated Columns）を計算するための式状態を初期化し、ResultRelInfo に格納する。

シグネチャまたは形状

void ExecInitGenerated(ResultRelInfo *resultRelInfo, EState *estate, CmdType cmdtype)

入力と出力

  • **入力**:
  • resultRelInfo: 対象リレーションの実行時情報。
  • estate: クエリ実行状態。
  • cmdtype: 実行するコマンドの種類（CMD_INSERT または CMD_UPDATE）。
  • **出力**: なし（副作用として resultRelInfo のフィールドを更新）。

前提条件と不変条件

  • 対象リレーションに生成列（格納型または仮想型）が存在しない場合、関数は即座に終了する。
  • resultRelInfo の生成列関連フィールド（ri_GeneratedExprsI/U 等）が既に初期化されている場合はアサーションエラーとなる。
  • estate->es_query_cxt のメモリコンテキストでデータ構造が構築される。

アルゴリズムまたは振る舞い

1. **初期チェック**: テーブルに生成列が存在するか確認し、なければ終了。 2. **最適化判断**: CMD_UPDATE の場合、BEFORE ROW UPDATE トリガーが存在せず、かつ更新対象列が生成列の依存先に含まれない場合は計算をスキップする。 3. **式準備**:

  • 各列を走査し、生成列であれば build_column_default で式ツリーを取得。
  • ExecPrepareExpr を用いて式を評価可能な状態に変換。

    • 4. **状態保存**:

  • CMD_UPDATE なら ri_GeneratedExprsU および ri_NumGeneratedNeededU に保存。
  • それ以外（INSERT等）なら ri_GeneratedExprsI および ri_NumGeneratedNeededI に保存。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execUtils.c: ExecInitResultRelInfo 等から CMD_UPDATE 指定で呼び出される。
  • src/backend/executor/nodeModifyTable.c: ExecInitModifyTable 内で cmdtype に応じて呼び出される。
  • **呼び出し先**:
  • RelationGetDescr, TupleDescAttr, build_column_default, pull_varattnos, bms_overlap, ExecPrepareExpr, MemoryContextSwitchTo 等。

関連データ構造

  • ResultRelInfo: 実行中のリレーションに関する状態を保持。
  • EState: クエリ全体の実行状態。
  • ExprState: 評価準備済みの式。

失敗モードと境界条件

  • 生成列の式ツリーが見つからない場合、elog(ERROR) を発生させる。
  • 生成列が一つも必要ないと判断された場合、割り当てたメモリは解放される。

保守担当者向けメモ

  • 本関数は INSERT と UPDATE で異なるフィールドを使用するため、呼び出し時の cmdtype に注意すること。
  • UPDATE 時の最適化（updatedCols によるスキップ）は、トリガーの有無に強く依存するため、トリガーロジックを変更する際は本関数の条件分岐も再確認が必要。
