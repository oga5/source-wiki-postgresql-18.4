read_inheritable_socket

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/launch_backend.c:865

目的

Windows環境において、親プロセスから引き継がれたソケット情報を復元し、子プロセス内で利用可能なソケットハンドルを作成します。

シグネチャまたは形状

static void read_inheritable_socket(SOCKET *dest, InheritableSocket *src)

入力と出力

  • **入力**:
  • dest: 復元されたソケットハンドルを格納するポインタ。
  • src: ソケット情報（InheritableSocket構造体）へのポインタ。
  • **出力**: なし（dest経由で結果を返却）。

前提条件と不変条件

  • src->origsocket が有効なソケットである場合、src->wsainfo には WSADuplicateSocket で取得された有効なプロトコル情報が含まれている必要があります。
  • 本関数はWindows環境（EXEC_BACKENDモード）での実行を前提としています。

アルゴリズムまたは振る舞い

1. src->origsocket が PGINVALID_SOCKET または 0 の場合、ソケットは実体を持たないと判断し、dest にそのまま値を代入して終了します。 2. 有効なソケットの場合、WSASocket を呼び出し、src->wsainfo を用いて新しいソケットハンドルを作成します。 3. WSASocket が失敗した場合は write_stderr でエラーを出力し、プロセスを終了（exit(1)）します。 4. 成功した場合、dest に新しいソケットハンドルを格納します。 5. 重複した参照を防ぐため、closesocket を呼び出して元の src->origsocket をクローズします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c:1003（read_inheritable_socket(&MyClientSocket->sock, &param->inh_sock);）
  • **呼び出し先**: WSASocket（Windows API）、write_stderr、exit、closesocket（Windows API）。
  • 注: src/backend/postmaster/launch_backend.c:719 にマクロ定義が存在しますが、これは本関数とは別の実装（非Windows用）として定義されています。

関連データ構造

  • SOCKET: Windowsソケットハンドル型。
  • InheritableSocket: ソケットの複製情報（origsocket および WSAPROTOCOL_INFO）を保持する構造体。

失敗モードと境界条件

  • WSASocket が INVALID_SOCKET を返した場合、エラーメッセージを出力してプロセスが異常終了します。
  • src->origsocket が無効な値である場合、ソケット作成処理はスキップされます。

保守担当者向けメモ

  • 本関数は EXEC_BACKEND モードにおけるソケットの引き継ぎ処理の核心部です。Windows環境特有のソケット複製メカニズムに依存しているため、修正時は WSADuplicateSocket との整合性に注意してください。
