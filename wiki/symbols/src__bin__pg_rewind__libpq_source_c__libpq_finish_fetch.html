libpq_finish_fetch

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/libpq_source.c:419

目的

libpq_source インターフェースにおいて、キューイングされているすべてのファイル取得リクエストを強制的に実行し、処理を完了させるために使用されます。

シグネチャまたは形状

static void libpq_finish_fetch(rewind_source *source);

入力と出力

  • **入力**: rewind_source *source - libpq_source 構造体としてキャスト可能なソースインターフェースへのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 source は、libpq_source として初期化されている必要があります。
  • process_queued_fetch_requests が内部で呼び出されるため、接続状態が有効である必要があります。

アルゴリズムまたは振る舞い

1. 受け取った rewind_source ポインタを libpq_source 型にキャストします。 2. process_queued_fetch_requests を呼び出し、蓄積されたリクエストキューを処理します。

呼び出し元と呼び出し先

  • **呼び出し元**: rewind_source インターフェースのメソッドとして登録されており、pg_rewind のメイン処理フローから呼び出されます。
  • **呼び出し先**: process_queued_fetch_requests を直接呼び出します。

関連データ構造

  • rewind_source: 抽象インターフェース構造体。
  • libpq_source: rewind_source を継承した、libpq 接続とリクエストキューを保持する具象構造体。

失敗モードと境界条件

  • キューが空の場合の挙動は process_queued_fetch_requests の実装に依存します。
  • リモートサーバーとの通信エラーが発生した場合、process_queued_fetch_requests 内でエラーが処理されます。

保守担当者向けメモ

  • 本関数は rewind_source インターフェースの finish_fetch 関数ポインタとして割り当てられています（src/bin/pg_rewind/libpq_source.c:93）。
  • 接続の所有権は本関数では管理されません。接続のクローズが必要な場合は、呼び出し元で別途管理する必要があります。
