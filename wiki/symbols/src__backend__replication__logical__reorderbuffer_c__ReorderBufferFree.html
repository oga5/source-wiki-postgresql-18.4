ReorderBufferFree

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/reorderbuffer.c:415

目的

ReorderBuffer 構造体が保持するメモリリソースを解放し、ディスク上に残存するシリアライズされたトランザクションデータをクリーンアップする。

シグネチャまたは形状

void ReorderBufferFree(ReorderBuffer *rb)

入力と出力

  • **入力**: rb (ReorderBuffer*) - 解放対象の再オーダーバッファへのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • rb は有効な ReorderBuffer 構造体を指している必要がある。
  • 本関数を呼び出すと、rb が保持していたメモリコンテキストが破棄されるため、以降 rb を参照することはできない。

アルゴリズムまたは振る舞い

1. rb->context に格納されているメモリコンテキストを取得する。 2. MemoryContextDelete を呼び出し、バッファに関連付けられたすべてのメモリを解放する。 3. ReorderBufferCleanupSerializedTXNs を呼び出し、現在のレプリケーションスロット名に基づいて、ディスク上に残存するシリアライズされたトランザクションファイルを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/logical.c:684 にて呼び出しを確認。
  • **呼び出し先**:
  • MemoryContextDelete (PostgreSQL メモリ管理 API)
  • ReorderBufferCleanupSerializedTXNs (同ファイル内の内部関数)

関連データ構造

  • ReorderBuffer: 論理デコーディングのメイン状態構造体。
  • MemoryContext: メモリ管理単位。

失敗モードと境界条件

  • rb が NULL の場合、メモリコンテキストの参照でクラッシュする可能性がある。
  • レプリケーションスロット名が取得できない場合、ReorderBufferCleanupSerializedTXNs の実行に失敗する可能性がある（MyReplicationSlot の状態に依存）。

保守担当者向けメモ

  • 本関数はリソースの完全な破棄を行うため、呼び出し後に ReorderBuffer を再利用することはできない。
  • ディスク上のクリーンアップはレプリケーションスロット名に依存しているため、スロットが正しく設定されている環境で実行されることを前提としている。
