pqsignal

定義

  • 種別: function
  • 定義箇所: src/port/pqsignal.c:122

目的

プラットフォーム間で一貫した信頼性の高いシグナルハンドリングを提供し、OS 固有のシグナル動作の差異を抽象化する。

シグネチャまたは形状

void pqsignal(int signo, pqsigfunc func);

入力と出力

  • **入力**:
  • signo: 処理対象のシグナル番号。
  • func: 登録するシグナルハンドラ関数（または SIG_IGN, SIG_DFL）。
  • **出力**: なし（void）。

前提条件と不変条件

  • signo は 0 より大きく PG_NSIG 未満であること（Assert による検証）。
  • func が SIG_IGN または SIG_DFL 以外の場合、内部配列 pqsignal_handlers に格納される。

アルゴリズムまたは振る舞い

1. func がデフォルト動作（SIG_IGN, SIG_DFL）でない場合、pqsignal_handlers[signo] に func を保存し、実際のハンドラとして wrapper_handler を指定する。 2. Unix 環境では sigaction を使用し、SA_RESTART フラグを付与してシグナルハンドラを登録する。SIGCHLD の場合は SA_NOCLDSTOP も付与する。 3. Windows 環境（かつフロントエンド）では、標準の signal() 関数へ処理を転送する。 4. 登録失敗時は Assert(false) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c
  • src/backend/access/transam/parallel.c
  • src/backend/bootstrap/bootstrap.c
  • （その他、PostgreSQL バックエンドおよびフロントエンドの各所）
  • **呼び出し先**:
  • wrapper_handler（シグナル発生時の仲介関数）
  • sigaction（Unix 系 OS のシステムコール）
  • signal（Windows 環境での標準ライブラリ関数）

関連データ構造

  • pqsignal_handlers[]: 登録されたシグナルハンドラを保持する内部配列。
  • struct sigaction: Unix 環境でのシグナル設定構造体。

失敗モードと境界条件

  • signo が範囲外の場合、Assert により停止する。
  • sigaction または signal の呼び出しが失敗した場合、Assert(false) により停止する。

保守担当者向けメモ

  • wrapper_handler 内で MyProcPid を用いたプロセス検証が行われるため、本関数で登録するハンドラは安全に実行されることが期待される。
  • Windows 環境におけるフロントエンドとバックエンドの挙動差異に注意すること。
