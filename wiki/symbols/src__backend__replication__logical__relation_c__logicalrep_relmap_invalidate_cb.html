logicalrep_relmap_invalidate_cb

定義

  • **種別**: function
  • **定義箇所**: src/backend/replication/logical/relation.c:63

目的

ローカルの relcache（リレーションキャッシュ）の変更通知を受け取り、論理レプリケーションで使用しているリレーションマッピングキャッシュを無効化する。

シグネチャまたは形状

static void logicalrep_relmap_invalidate_cb(Datum arg, Oid reloid);

入力と出力

  • **入力**:
  • arg: Datum 型の引数（未使用）。
  • reloid: 無効化対象のテーブルの Oid。InvalidOid の場合は全キャッシュエントリを対象とする。
  • **出力**: なし（副作用としてキャッシュエントリのフラグを更新）。

前提条件と不変条件

  • LogicalRepRelMap が初期化済みであること（未初期化の場合は即座にリターンする）。
  • 本関数は relcache の無効化コールバックとして登録されていること。

アルゴリズムまたは振る舞い

1. LogicalRepRelMap が NULL の場合は処理を終了する。 2. reloid が InvalidOid でない場合:

  • キャッシュ内の全エントリを走査する。
  • localreloid が指定された reloid と一致するエントリを探し、localrelvalid を false に設定して走査を終了する。

    • 3. reloid が InvalidOid の場合:

  • キャッシュ内の全エントリを走査し、すべての localrelvalid を false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: relcache の無効化メカニズム（CacheRegisterRelcacheCallback を介して登録）。
  • **呼び出し先**: hash_seq_init, hash_seq_search, hash_seq_term（ハッシュテーブル走査用ユーティリティ）。

関連データ構造

  • LogicalRepRelMap: リレーションマッピングを保持するハッシュテーブル。
  • LogicalRepRelMapEntry: キャッシュエントリ構造体。localrelvalid フラグを保持する。

失敗モードと境界条件

  • LogicalRepRelMap が未初期化の場合、安全に無視される。
  • reloid が InvalidOid の場合、キャッシュ全体が強制的に無効化される。
