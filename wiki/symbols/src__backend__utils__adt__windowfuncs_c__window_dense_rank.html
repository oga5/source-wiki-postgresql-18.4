window_dense_rank

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/windowfuncs.c:199

目的

SQL標準の DENSE_RANK() ウィンドウ関数を実装し、現在の行が属するパーティション内での順位を算出します。

シグネチャまたは形状

Datum window_dense_rank(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (ウィンドウ関数実行用API経由で WindowObject を取得)
  • **出力**: Datum (現在の行の DENSE_RANK 値を int64 として返却)

前提条件と不変条件

  • WinGetPartitionLocalMemory を使用して、パーティションごとに rank_context 構造体を確保・保持していること。
  • rank_up 関数により、現在の行が前の行と異なる順位グループ（ピアではない）であるかどうかが判定されていること。

アルゴリズムまたは振る舞い

1. PG_WINDOW_OBJECT() を呼び出し、現在のウィンドウ操作コンテキストを取得します。 2. rank_up(winobj) を呼び出し、現在の行が前の行とピア（同順位）でないかを確認します。 3. WinGetPartitionLocalMemory を介して、パーティション固有の rank_context を取得します。 4. rank_up が真（up == true）の場合、context->rank をインクリメントします。 5. 更新された context->rank を PG_RETURN_INT64 で返却します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • rank_up: ピア判定に使用。
  • WinGetPartitionLocalMemory: パーティションごとの状態保持に使用。
  • **呼び出し元**:
  • PostgreSQLのウィンドウ関数実行エンジン（src/backend/executor/nodeWindowAgg.c 等から間接的に呼び出される）。
  • 詳細は提供されたソースからは不明です。

関連データ構造

  • rank_context: パーティションごとの計算状態（ランク値）を保持する構造体。
  • WindowObject: ウィンドウ関数実行用APIのハンドル。

失敗モードと境界条件

  • パーティションの先頭行では rank_up が真となり、ランクが初期化される前提で動作します。
  • メモリ確保失敗時の挙動は WinGetPartitionLocalMemory の仕様に依存します。

保守担当者向けメモ

  • 本関数は window_dense_rank_support 関数と対になっており、オプティマイザがフレーム最適化を行う際に参照されます。
  • rank_up のロジックは RANK や DENSE_RANK で共通化されているため、変更時は影響範囲に注意してください。
