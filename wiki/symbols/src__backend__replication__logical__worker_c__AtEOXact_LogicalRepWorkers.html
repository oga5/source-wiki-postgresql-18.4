AtEOXact_LogicalRepWorkers

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/worker.c:5151

目的

トランザクション終了時（End of Transaction）に、コミット待ち状態にある論理レプリケーションワーカーを起床させる。

シグネチャまたは形状

void AtEOXact_LogicalRepWorkers(bool isCommit)

入力と出力

  • **入力**:
  • isCommit (bool): トランザクションがコミットされたかどうかを示すフラグ。
  • **出力**: なし（副作用としてワーカーの起床とメモリの解放を行う）。

前提条件と不変条件

  • on_commit_wakeup_workers_subids リストが、起床対象のサブスクリプションIDを保持していること。
  • LogicalRepWorkerLock を介してワーカーリストへの安全なアクセスが保証されていること。
  • トランザクション終了処理の一環として呼び出されること。

アルゴリズムまたは振る舞い

1. isCommit が真かつ on_commit_wakeup_workers_subids が空でない場合、以下の処理を行う。

  • LogicalRepWorkerLock を共有ロック（LW_SHARED）で取得する。
  • on_commit_wakeup_workers_subids に格納された各サブスクリプションIDに対し、logicalrep_workers_find を用いて関連するワーカーリストを取得する。
  • 各ワーカーに対して logicalrep_worker_wakeup_ptr を呼び出し、起床シグナルを送る。
  • ロックを解放する。

    • 2. 最後に、on_commit_wakeup_workers_subids を NIL にリセットし、メモリを解放する（リスト自体のメモリはトランザクション終了時に自動的に回収される）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c (トランザクション終了処理の各所から isCommit の値を切り替えて呼び出される)
  • **呼び出し先**:
  • logicalrep_workers_find
  • logicalrep_worker_wakeup_ptr
  • LWLockAcquire / LWLockRelease

関連データ構造

  • on_commit_wakeup_workers_subids: 起床対象のサブスクリプションIDを保持するリスト。
  • LogicalRepWorker: 論理レプリケーションワーカーの情報を保持する構造体。
  • LogicalRepWorkerLock: ワーカーリスト保護用の軽量ロック。

失敗モードと境界条件

  • isCommit が偽の場合、ワーカーの起床処理はスキップされる。
  • on_commit_wakeup_workers_subids が空（NIL）の場合、ロック取得等の処理は行われない。

保守担当者向けメモ

  • 本関数はトランザクション終了時のフックとして機能する。on_commit_wakeup_workers_subids の管理がトランザクションのライフサイクルと正しく同期していることを確認すること。
  • LogicalRepWorkerLock の取得は共有ロックであるため、ワーカーリストの構造変更を伴う操作と競合しないよう注意が必要である。
