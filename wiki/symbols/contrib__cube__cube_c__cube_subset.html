cube_subset

定義

  • 種別: function
  • 定義箇所: contrib/cube/cube.c:246

目的

既存の NDBOX（多次元ボックス）から、指定された次元インデックスのリストに基づいて新しい NDBOX を抽出します。

シグネチャまたは形状

Datum cube_subset(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_NDBOX_P(0): 抽出元の NDBOX。
  • PG_GETARG_ARRAYTYPE_P(1): 抽出する次元を指定する int32 型の配列。
  • **出力**:
  • PG_RETURN_NDBOX_P(result): 抽出された次元で構成される新しい NDBOX。

前提条件と不変条件

  • 入力配列に NULL を含めることはできません。
  • 指定された次元数は CUBE_MAX_DIM を超えてはなりません。
  • 配列内の各インデックスは、元の NDBOX の次元範囲（1 から DIM(c) まで）内である必要があります。

アルゴリズムまたは振る舞い

1. 入力配列の妥当性を検証し、次元数を取得します。 2. 元の NDBOX がポイント型（IS_POINT）かどうかを判定し、結果用のメモリを確保します。 3. 指定されたインデックスごとにループし、元の NDBOX から対応する座標値をコピーします。

  • ポイント型でない場合、下限座標と上限座標の両方がコピーされます。

    • 4. 確保したメモリを NDBOX として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数インターフェース（PG_FUNCTION_INFO_V1(cube_subset) により登録）。
  • **呼び出し先**: PostgreSQL のメモリ管理関数（palloc0）、エラー報告関数（ereport）、および配列操作マクロ（ARR_DATA_PTR, ARRNELEMS）。

関連データ構造

  • NDBOX: 多次元ボックスを表現する構造体。
  • ArrayType: PostgreSQL の配列型。

失敗モードと境界条件

  • **エラー**:
  • 配列に NULL が含まれる場合（ERRCODE_ARRAY_ELEMENT_ERROR）。
  • 指定された次元数が CUBE_MAX_DIM を超える場合（ERRCODE_PROGRAM_LIMIT_EXCEEDED）。
  • 配列内のインデックスが 1 未満、または元の次元数を超える場合（ERRCODE_ARRAY_ELEMENT_ERROR）。

保守担当者向けメモ

  • NDBOX の内部構造（x 配列）への直接アクセスを行っているため、cubedata.h の定義変更時には注意が必要です。
  • IS_POINT マクロによる条件分岐が、メモリ確保サイズと座標コピーの両方に影響を与える点に留意してください。
