RelationGetPartitionDesc

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partdesc.c:70

目的

リレーション（パーティションテーブル）に対応するパーティション記述子（PartitionDesc）を取得します。キャッシュの有効性を判定し、必要に応じて再構築を行うことで、パーティション情報の効率的な提供と一貫性の維持を担います。

シグネチャまたは形状

PartitionDesc RelationGetPartitionDesc(Relation rel, bool omit_detached)

入力と出力

  • **入力**:
  • rel: 対象となるリレーション（RELKIND_PARTITIONED_TABLE であること）。
  • omit_detached: デタッチ済みパーティションを除外するかどうかを指定するフラグ。
  • **出力**:
  • パーティション構造を記述する PartitionDesc 構造体へのポインタ。

前提条件と不変条件

  • rel->rd_rel->relkind が RELKIND_PARTITIONED_TABLE である必要があります（Assert による検証あり）。
  • キャッシュされた記述子を使用する場合、その記述子が現在のトランザクションスナップショットと整合している必要があります。

アルゴリズムまたは振る舞い

1. **全パーティション取得時**: omit_detached が false かつ rel->rd_partdesc が存在する場合、デタッチ済みパーティションが存在しないか、アクティブなスナップショットがない限り、キャッシュされた rel->rd_partdesc を返します。 2. **デタッチ済み除外時**: omit_detached が true の場合、rel->rd_partdesc_nodetached が存在し、かつ保存された xmin が現在のスナップショットにおいて進行中でない（XidInMVCCSnapshot が偽）場合に限り、キャッシュされた記述子を返します。 3. **再構築**: 上記のキャッシュ条件を満たさない場合、RelationBuildPartitionDesc(rel, omit_detached) を呼び出して記述子を再構築し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/tablecmds.c
  • src/backend/commands/trigger.c
  • **呼び出し先**:
  • RelationBuildPartitionDesc（キャッシュミス時）
  • ActiveSnapshotSet, GetActiveSnapshot, XidInMVCCSnapshot（キャッシュ判定時）

関連データ構造

  • PartitionDesc: パーティションの OID 配列や境界情報を保持する構造体。
  • Relation: リレーションキャッシュ（relcache）エントリ。

失敗モードと境界条件

  • rel がパーティションテーブルでない場合、アサーション違反により異常終了します。
  • 並行 DDL（DETACH CONCURRENTLY 等）によりカタログ状態が変化した場合、キャッシュの無効化と再構築が行われます。

保守担当者向けメモ

  • 本関数は relcache の rd_partdesc および rd_partdesc_nodetached を活用して構築コストを削減します。
  • omit_detached が true の場合のキャッシュ判定には xmin を使用しており、スナップショットの整合性が重要です。
  • 構築ロジックの詳細は RelationBuildPartitionDesc に委譲されています。
