pgstat_fetch_stat_subscription

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_subscription.c:89

目的

指定されたサブスクリプションID（subid）に対応する統計情報スナップショットを取得する。

シグネチャまたは形状

PgStat_StatSubEntry *pgstat_fetch_stat_subscription(Oid subid)

入力と出力

  • **入力**: subid (Oid) - 統計情報を取得する対象のサブスクリプション識別子。
  • **出力**: PgStat_StatSubEntry * - 共有メモリ上の統計情報エントリへのポインタ。

前提条件と不変条件

  • 統計サブシステムが初期化されていること。
  • 戻り値は共有メモリ上のデータ構造を指すため、呼び出し元での生存期間や排他制御については pgstat_fetch_entry の仕様に依存する。

アルゴリズムまたは振る舞い

内部的に pgstat_fetch_entry を呼び出し、PGSTAT_KIND_SUBSCRIPTION 型の統計エントリを検索して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c:2196 (SQL関数経由での統計参照)
  • **呼び出し先**:
  • pgstat_fetch_entry (統計サブシステムの汎用取得関数)

関連データ構造

  • PgStat_StatSubEntry: サブスクリプション統計情報を保持する構造体。
  • PGSTAT_KIND_SUBSCRIPTION: 統計情報の種類を示す識別子。

失敗モードと境界条件

  • 指定した subid に対応するエントリが存在しない場合、pgstat_fetch_entry の仕様に従い NULL または適切なデフォルト値が返される可能性がある。

保守担当者向けメモ

  • 本関数は単なるラッパーであるため、統計取得ロジックの変更は pgstat_fetch_entry 側の仕様を確認すること。
  • 戻り値の型キャストは PgStat_StatSubEntry * に固定されているため、構造体の定義変更時には整合性に注意が必要。
