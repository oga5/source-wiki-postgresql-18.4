strupper_libc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:142

目的

OSのlibcロケール機能を利用して、文字列を大文字に変換するインターフェースを提供します。

シグネチャまたは形状

size_t strupper_libc(char *dst, size_t dstsize, const char *src, ssize_t srclen, pg_locale_t locale);

入力と出力

  • **入力**:
  • dst: 変換後の文字列を格納するバッファ。
  • dstsize: dst バッファのサイズ。
  • src: 変換対象のソース文字列。
  • srclen: src の長さ（バイト単位）。
  • locale: 使用するロケールオブジェクト。
  • **出力**:
  • 変換後の文字列の長さ（バイト単位）を size_t で返します。

前提条件と不変条件

  • データベースのエンコーディング設定に基づき、適切な変換処理（シングルバイトまたはマルチバイト）が選択されます。

アルゴリズムまたは振る舞い

1. pg_database_encoding_max_length() を呼び出し、データベースのエンコーディングの最大文字長を確認します。 2. 最大長が1バイトを超える場合は strupper_libc_mb を呼び出し、マルチバイト対応の変換を行います。 3. 最大長が1バイト以下の場合は strupper_libc_sb を呼び出し、シングルバイト用の高速な変換を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c: ロケール抽象化層から呼び出されます。
  • **呼び出し先**:
  • strupper_libc_mb (条件分岐による)
  • strupper_libc_sb (条件分岐による)

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。

失敗モードと境界条件

  • データベースのエンコーディングがマルチバイトかシングルバイトかによって、内部的に呼び出す関数が切り替わります。具体的なエラーハンドリングの詳細は各下位関数（_mb / _sb）に依存します。

保守担当者向けメモ

  • 本関数は単なるディスパッチャーです。ロケール依存の変換ロジックを修正する場合は、strupper_libc_mb または strupper_libc_sb を確認してください。
