gbt_num_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_num.c:262

目的

GiST インデックスにおいて、検索クエリとインデックス内の数値範囲（GBT_NUMKEY_R）を比較し、指定された B-tree 演算子に基づいて一致するかを判定します。

シグネチャまたは形状

bool gbt_num_consistent(
    const GBT_NUMKEY_R *key,
    const void *query,
    const StrategyNumber *strategy,
    bool is_leaf,
    const gbtree_ninfo *tinfo,
    FmgrInfo *flinfo
)

入力と出力

  • **入力**:
  • key: 比較対象となるインデックス内の数値範囲。
  • query: 検索条件となる値。
  • strategy: 適用する B-tree 演算子（<, <=, =, >=, > など）。
  • is_leaf: 現在のノードがリーフノードかどうかを示すフラグ。
  • tinfo: データ型固有の比較関数（f_gt, f_lt, f_eq 等）を保持する構造体。
  • flinfo: 関数呼び出し情報。
  • **出力**:
  • 条件を満たす場合は true、それ以外は false を返します。

前提条件と不変条件

  • tinfo には、対象データ型に対応する適切な比較関数が設定されている必要があります。
  • is_leaf の値に応じて、内部ノードとリーフノードで異なる比較ロジックが適用されます。

アルゴリズムまたは振る舞い

strategy に基づき以下の判定を行います：

  • **BTLessEqualStrategyNumber**: query >= key->lower を判定。
  • **BTLessStrategyNumber**: リーフなら query > key->lower、内部ノードなら query >= key->lower を判定。
  • **BTEqualStrategyNumber**: リーフなら query == key->lower、内部ノードなら key->lower <= query <= key->upper を判定。
  • **BTGreaterStrategyNumber**: リーフなら query < key->upper、内部ノードなら query <= key->upper を判定。
  • **BTGreaterEqualStrategyNumber**: query <= key->upper を判定。
  • **BtreeGistNotEqualStrategyNumber**: !(query == key->lower && query == key->upper) を判定。
  • **デフォルト**: false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_bool.c, btree_cash.c, btree_date.c, btree_enum.c, btree_float4.c, btree_float8.c, btree_inet.c, btree_int2.c, btree_int4.c, btree_int8.c, btree_interval.c, btree_macaddr.c 等の各データ型実装ファイル。
  • **呼び出し先**: tinfo 構造体に含まれる関数ポインタ（f_ge, f_gt, f_eq, f_le, f_lt）。

関連データ構造

  • GBT_NUMKEY_R: インデックス内の数値範囲を保持する構造体。
  • gbtree_ninfo: 各データ型固有の比較関数を保持する設定構造体。

失敗モードと境界条件

  • 未定義の strategy が渡された場合、false を返します。
  • is_leaf が正しく設定されていない場合、内部ノードとリーフノードで不適切な比較が行われる可能性があります。

保守担当者向けメモ

  • 新しいデータ型を追加する際は、gbtree_ninfo を適切に構成して本関数を呼び出すことで、GiST インデックスの検索ロジックを再利用可能です。
  • 内部ノードとリーフノードで比較ロジックが異なるため、修正時は is_leaf の条件分岐に注意してください。
