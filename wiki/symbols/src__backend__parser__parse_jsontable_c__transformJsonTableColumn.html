transformJsonTableColumn

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_jsontable.c:396

目的

JSON_TABLE 構文における個別のカラム定義（JsonTableColumn）を、実行時に評価可能な JsonFuncExpr ノードへ変換します。

シグネチャまたは形状

JsonFuncExpr *transformJsonTableColumn(JsonTableColumn *jtc, Node *contextItemExpr, List *passingArgs)

入力と出力

  • **入力**:
  • jtc: 変換対象のカラム定義。
  • contextItemExpr: JSONパスの評価対象となる式。
  • passingArgs: PASSING 句で指定された引数リスト。
  • **出力**:
  • 構築された JsonFuncExpr 構造体へのポインタ。

前提条件と不変条件

  • jtc->name は NULL であってはならない（Assert により検証）。
  • jtc->coltype に応じて、JSON_VALUE_OP、JSON_EXISTS_OP、または JSON_QUERY_OP が適切に設定される。

アルゴリズムまたは振る舞い

1. jtc->coltype に基づき、JsonFuncExpr の演算子（op）を決定します。 2. カラム名を jfexpr->column_name にコピーします（エラー出力用）。 3. contextItemExpr を JsonValueExpr としてラップし、jfexpr->context_item に設定します。 4. jtc->pathspec が存在する場合はそれを使用し、存在しない場合は $. にカラム名をエスケープして結合したデフォルトパスを生成します。 5. passingArgs、型情報、ON EMPTY/ON ERROR 句、およびその他の属性（quotes, wrapper 等）を jtc から jfexpr へ転送します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_jsontable.c 内の transformJsonTableColumns から呼び出されます。
  • **呼び出し先**: makeNode, makeJsonValueExpr, makeJsonFormat, initStringInfo, appendStringInfoString, escape_json, makeStringConst, pstrdup 等のユーティリティ関数。

関連データ構造

  • JsonTableColumn: 入力となる構文解析木ノード。
  • JsonFuncExpr: 出力となる実行計画用ノード。

失敗モードと境界条件

  • jtc->name が NULL の場合、アサーション違反により異常終了します。
  • メモリ不足時には pstrdup や makeNode 等の内部関数がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は JSON_TABLE の各カラムを個別に処理する役割を担います。NESTED 句の再帰的な処理は transformJsonTableNestedColumns が担当するため、本関数は単一カラムの変換ロジックに集中しています。
