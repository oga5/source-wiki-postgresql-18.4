predicate_implied_by_recurse

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/predtest.c:289

目的

与えられた clause（条件）が predicate（述語）を論理的に含意するかを、AND/OR 構造を再帰的に分解して判定する。

シグネチャまたは形状

static bool predicate_implied_by_recurse(Node *clause, Node *predicate, bool weak)

入力と出力

  • **入力**:
  • clause: 含意の根拠となる式（RestrictInfo の場合は内部の clause が抽出される）。
  • predicate: 含意されるべき対象の式。
  • weak: 判定の厳密さを制御するフラグ。
  • **出力**:
  • bool: clause が predicate を含意する場合に true を返す。

前提条件と不変条件

  • 入力される clause は NULL であってはならない。
  • predicate_classify を用いて式を CLASS_AND, CLASS_OR, CLASS_ATOM に分類し、その論理構造に基づいて処理を行う。

アルゴリズムまたは振る舞い

論理構造の組み合わせに応じて再帰的に判定を行う：

  • **AND-clause**: 構成要素のいずれかが predicate を含意すれば全体として含意するとみなす（CLASS_ATOM との比較時など）。
  • **OR-clause**: 全ての構成要素が predicate を含意する場合にのみ、全体として含意するとみなす（CLASS_AND との比較時など）。
  • **再帰**: iterate_begin / iterate_end マクロを使用して、AND/OR 構造の各要素に対して自身を再帰的に呼び出す。
  • **終端**: CLASS_ATOM 同士の比較に到達した際、演算子ベースの推論（operator_predicate_proof 等）へ委譲される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • predicate_implied_by（直接の呼び出し元）
  • 自身を再帰的に呼び出す（src/backend/optimizer/util/predtest.c 内の各所）。
  • **呼び出し先**:
  • predicate_classify
  • predicate_implied_by_recurse（再帰呼び出し）
  • iterate_begin / iterate_end（イテレータマクロ）

関連データ構造

  • PredIterInfoData: AND/OR 構造の走査状態を保持する。
  • PredClass: 式の論理クラス（CLASS_AND, CLASS_OR, CLASS_ATOM）。
  • RestrictInfo: 入力ノードがラップされている場合に使用される。

失敗モードと境界条件

  • **再帰の深さ**: 非常に深い式ツリーを入力した場合、スタックオーバーフローのリスクがある。
  • **非IMMUTABLE関数**: IMMUTABLE ではない関数や演算子が含まれる場合、最適化の前提が崩れ、誤った推論結果を返す可能性がある。

保守担当者向けメモ

  • 本関数は predicate_refuted_by_recurse と対になる構造を持つ。
  • iterate_begin マクロ内のイテレータ変数は、呼び出し側のスコープで定義される必要があるため、修正時はマクロの展開内容に注意すること。
