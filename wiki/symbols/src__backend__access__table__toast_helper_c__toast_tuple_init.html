toast_tuple_init

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/toast_helper.c:40

目的

TOAST処理対象となるタプルの各属性をスキャンし、圧縮や外部化の必要性を判定して ToastTupleContext を初期化します。

シグネチャまたは形状

void toast_tuple_init(ToastTupleContext *ttc)

入力と出力

  • **入力**: ToastTupleContext *ttc（呼び出し側で適切に初期化されたコンテキスト構造体）。
  • **出力**: なし（引数 ttc の内部状態を更新）。

前提条件と不変条件

  • 呼び出し前に ttc->ttc_rel や ttc->ttc_values 等の必要なフィールドが設定されている必要があります。
  • ttc->ttc_attr 配列は、タプルの属性数分確保されている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: ttc->ttc_flags をリセットし、全属性の tai_colflags を初期化します。 2. **UPDATE処理**: ttc->ttc_oldvalues が存在する場合、旧値と新値を比較します。

  • 外部ストレージ上の値が変更された場合、TOASTCOL_NEEDS_DELETE_OLD を設定し、後続の削除処理を予約します。
  • 変更がない場合は TOASTCOL_IGNORE を設定し、処理対象から除外します。

    • 3. **属性スキャン**: 各属性に対し以下の処理を行います。

  • **NULL値**: TOASTCOL_IGNORE を設定。
  • **可変長データ (varlena)**:
  • TYPSTORAGE_PLAIN の場合は TOASTCOL_IGNORE を設定。
  • 外部化されているデータ（VARATT_IS_EXTERNAL）は detoast_external_attr 等で展開し、tai_oldexternal に保持。必要に応じて TOASTCOL_NEEDS_FREE を設定。
  • 属性サイズを tai_size に記録。
  • **固定長データ**: TOASTCOL_IGNORE を設定。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heaptoast.c:157
  • **呼び出し先**:
  • detoast_attr / detoast_external_attr (varlenaの展開)
  • VARATT_IS_EXTERNAL_ONDISK / VARATT_IS_EXTERNAL (マクロによる属性判定)
  • memcmp (旧値との比較)

関連データ構造

  • ToastTupleContext: TOAST処理の全体状態を保持。
  • ToastAttrInfo: 各属性ごとのTOAST状態（tai_colflags 等）。
  • TupleDesc: タプルの属性定義。

失敗モードと境界条件

  • attlen == -1 以外の属性は常に TOASTCOL_IGNORE となり、TOAST処理の対象外となります。
  • ttc_oldvalues が NULL の場合は INSERT 処理として振る舞い、旧値の削除判定はスキップされます。

保守担当者向けメモ

  • 本関数は ToastTupleContext の各フィールドの初期化に強く依存しています。構造体定義の変更時は toast_helper.h を参照してください。
  • TOASTCOL_NEEDS_DELETE_OLD フラグが設定された属性は、後の toast_tuple_cleanup で物理削除されるため、フラグの伝播漏れに注意してください。
