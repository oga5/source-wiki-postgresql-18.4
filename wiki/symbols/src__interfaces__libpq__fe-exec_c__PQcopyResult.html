PQcopyResult

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-exec.c:318

目的

既存の PGresult オブジェクトの内容を複製し、新しい PGresult オブジェクトを作成します。

シグネチャまたは形状

PGresult *PQcopyResult(const PGresult *src, int flags);

入力と出力

  • **入力**:
  • src: 複製元の PGresult オブジェクト。
  • flags: 複製する内容を指定するビットフラグ（PG_COPYRES_ATTRS, PG_COPYRES_TUPLES, PG_COPYRES_NOTICEHOOKS, PG_COPYRES_EVENTS）。
  • **出力**: 複製された新しい PGresult オブジェクト。メモリ不足や処理失敗時は NULL を返します。

前提条件と不変条件

  • src が NULL の場合、関数は直ちに NULL を返します。
  • 複製された結果は、PQclear を用いて呼び出し元で解放する必要があります。

アルゴリズムまたは振る舞い

1. PQmakeEmptyPGresult を使用して空の PGresult を初期化します。 2. クライアントエンコーディングとコマンドステータスをコピーします。 3. flags に応じて以下の要素を順次コピーします：

  • 属性情報 (PG_COPYRES_ATTRS): PQsetResultAttrs を使用。
  • タプルデータ (PG_COPYRES_TUPLES): PQsetvalue を使用して全行・全列をコピー。
  • 通知フック (PG_COPYRES_NOTICEHOOKS): ポインタをコピー。
  • イベント (PG_COPYRES_EVENTS): dupEvents を使用して複製。

    • 4. 最後に PGEVT_RESULTCOPY イベントをトリガーし、登録されているイベントハンドラを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-exec.c 内のコードから参照されています。
  • **呼び出し先**: PQmakeEmptyPGresult, PQsetResultAttrs, PQsetvalue, PQclear, dupEvents を呼び出します。

関連データ構造

  • PGresult: クエリ結果を保持する主要な構造体。

失敗モードと境界条件

  • src が NULL の場合は NULL を返します。
  • PQmakeEmptyPGresult や PQsetResultAttrs、PQsetvalue、dupEvents が失敗した場合、それまでに確保したメモリを PQclear で解放し、NULL を返します。

保守担当者向けメモ

  • cmdStatus のコピーは無条件に行われますが、これが常に有用であるかはソースコード上のコメントで疑問視されています。
  • イベントコピー時に src->events[i].resultInitialized が真であるもののみイベントハンドラが実行されます。
