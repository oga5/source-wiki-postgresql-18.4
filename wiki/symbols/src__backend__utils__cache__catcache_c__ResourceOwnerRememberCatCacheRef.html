ResourceOwnerRememberCatCacheRef

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/catcache.c:158

目的

キャッシュされたシステムカタログタプルの参照を ResourceOwner に登録し、リソースの追跡と適切な解放を保証する。

シグネチャまたは形状

static inline void ResourceOwnerRememberCatCacheRef(ResourceOwner owner, HeapTuple tuple)

入力と出力

  • **入力**:
  • owner: 参照を保持する ResourceOwner オブジェクト。
  • tuple: キャッシュされた HeapTuple。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • ResourceOwner が有効であること。
  • catcache_resowner_desc がリソース追跡用の記述子として適切に初期化されていること。

アルゴリズムまたは振る舞い

1. PointerGetDatum(tuple) を呼び出し、HeapTuple を Datum 型に変換する。 2. ResourceOwnerRemember を呼び出し、指定された owner に対して catcache_resowner_desc を用いて当該タプルの参照を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/catcache.c 内のキャッシュエントリ生成および検索処理（CurrentResourceOwner を引数として渡すケースが確認されている）。
  • **呼び出し先**:
  • ResourceOwnerRemember（リソース追跡の基盤関数）。

関連データ構造

  • ResourceOwner: リソースの所有権を管理する構造体。
  • HeapTuple: キャッシュ対象のタプルデータ。
  • catcache_resowner_desc: キャッシュエントリのリソース追跡用記述子。

失敗モードと境界条件

  • owner が NULL の場合、ResourceOwnerRemember の内部実装に依存する（通常はクラッシュまたはエラー）。
  • メモリ不足等の理由で ResourceOwner への登録に失敗した場合、ResourceOwnerRemember がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを考慮してホットパスで利用される。
  • ResourceOwner の仕組みを介して参照カウントを管理しているため、本関数で登録した参照は、対応する解放処理（ResourceOwnerForget 等）が適切に行われない場合、メモリリークの原因となる。
