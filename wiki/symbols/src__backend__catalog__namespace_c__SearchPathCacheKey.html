SearchPathCacheKey

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/namespace.c:167

目的

検索パス（search_path）のキャッシュを検索または生成するためのキーとして使用されます。セッションの検索パス設定と現在のロール（ユーザー）の組み合わせを一意に識別します。

シグネチャまたは形状

typedef struct SearchPathCacheKey
{
	const char *searchPath;
	Oid			roleid;
} SearchPathCacheKey;

入力と出力

  • **入力**: searchPath（文字列）、roleid（Oid）。
  • **出力**: 本構造体自体が、キャッシュルックアップのためのハッシュキーとして機能します。

前提条件と不変条件

  • searchPath は、セッションの search_path GUC 変数の値を保持します。
  • roleid は、現在のセッションユーザーまたは実行ロールの OID を保持します。

アルゴリズムまたは振る舞い

  • spcachekey_hash 関数により、本構造体をハッシュ化してキャッシュのインデックスとして利用します。
  • spcachekey_equal 関数により、2つのキーが同一の検索パスとロールを持つか判定します。
  • lib/simplehash.h を利用したハッシュテーブルのキー型（SH_KEY_TYPE）として定義されています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/namespace.c 内の検索パスキャッシュ管理ロジック（cachekey 変数として初期化される箇所）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • SearchPathCacheEntry: キャッシュの値を保持する構造体。
  • lib/simplehash.h: 本構造体を利用したハッシュテーブルの実装元。

失敗モードと境界条件

  • searchPath が NULL の場合や、roleid が無効な Oid である場合の挙動については、本構造体の定義からは特定できません。

保守担当者向けメモ

  • 本構造体は namespace.c 内のキャッシュ機構に密結合しています。キャッシュのキー定義を変更する場合は、spcachekey_hash および spcachekey_equal の実装も併せて更新する必要があります。
