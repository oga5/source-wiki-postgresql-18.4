numeric

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/include/pgtypes_numeric.h:18

目的

ECPG（Embedded SQL for C）において、PostgreSQLのNUMERIC型をC言語プログラム内で扱うための動的メモリ管理型構造体です。

シグネチャまたは形状

typedef struct {
    int           ndigits;    /* digits[]内の桁数（0も可） */
    int           weight;     /* 先頭桁の重み */
    int           rscale;     /* 結果のスケール */
    int           dscale;     /* 表示スケール */
    int           sign;       /* NUMERIC_POS, NUMERIC_NEG, または NUMERIC_NAN */
    NumericDigit *buf;        /* digits[]用に確保された領域の先頭ポインタ */
    NumericDigit *digits;     /* 10進数桁データ */
} numeric;

入力と出力

  • **入力**: PGTYPESnumeric_from_asc 等の関数を介して文字列や他の数値型から変換されたデータを受け取ります。
  • **出力**: PGTYPESnumeric_to_asc 等の関数を介して文字列や他の数値型へ変換されます。

前提条件と不変条件

  • numeric 構造体は動的メモリ確保を前提としています。
  • ライフサイクル管理のため、PGTYPESnumeric_new() で確保し、PGTYPESnumeric_free() で解放する必要があります。
  • 構造体のメンバを直接操作することは推奨されず、提供される公開API（PGTYPESnumeric_*）を介して操作する必要があります。

アルゴリズムまたは振る舞い

  • decimal 型とは異なり、動的にサイズが変化する高精度数値を保持します。
  • sign メンバにより、正・負・NaN（非数）の状態を識別します。
  • buf ポインタは、digits 配列のために動的に確保されたメモリ領域を管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: ツールによる解析では、contrib/ 配下のコード（jsonb_plperl.c 等）で numeric 型への言及が見られますが、これらが本ヘッダーで定義された numeric 構造体を直接参照しているか、あるいは PostgreSQL 内部の同名型を参照しているかの区別は、提供されたソース情報からは確定できません。
  • **呼び出し先**: 本構造体はデータ定義であり、具体的な操作は src/interfaces/ecpg/pgtypeslib/numeric.c 等の実装側に依存します。

関連データ構造

  • NumericDigit: digits 配列の要素型。
  • decimal: 固定長配列（DECSIZE=30）を持つ静的メモリ管理型の構造体。

失敗モードと境界条件

  • メモリ不足時: PGTYPESnumeric_new() による動的確保が失敗する可能性があります。
  • 精度制限: numeric は動的サイズですが、計算結果がシステムのリソース制限を超える可能性があります。

保守担当者向けメモ

  • decimal 型との主な違いはメモリ管理戦略です。スタック上に配置したい場合は decimal を、可変長精度が必要な場合は numeric を選択してください。
  • 構造体のメンバを直接変更すると、メモリリークや不正なメモリ参照を引き起こす可能性があるため、必ず API を経由してください。
