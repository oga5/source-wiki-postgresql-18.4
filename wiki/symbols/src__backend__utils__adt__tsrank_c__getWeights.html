getWeights

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsrank.c:404

目的

全文検索のランキング計算で使用する重み配列（A, B, C, D）を検証し、正規化された浮動小数点配列として取得する。

シグネチャまたは形状

static void getWeights(ArrayType *win, float *ws)

入力と出力

  • **入力**:
  • win: 重み設定を含む ArrayType 構造体へのポインタ。
  • ws: 重みを格納するための float 型配列（呼び出し元で確保済み）。
  • **出力**:
  • なし（ws 配列を直接更新する）。

前提条件と不変条件

  • win は NULL であってはならない。
  • win は1次元配列である必要がある。
  • win の要素数は NUM_WEIGHTS 以上である必要がある。
  • win の各要素は NULL を含んではならない。
  • 各重み値は 0.0 以上 1.0 以下である必要がある。

アルゴリズムまたは振る舞い

1. win が1次元配列であるか、要素数が NUM_WEIGHTS 以上か、NULL を含まないかを検証し、違反があればエラーを報告する。 2. win のデータポインタから各重みを取得する。 3. 各重みについて、値が 0 以上であればその値を採用し、負であれば default_weights の対応する値を採用する。 4. 最終的な重みが 1.0 を超える場合はエラーを報告する。

  • **呼び出し先**:
  • array_contains_nulls (PostgreSQL 内部関数)

関連データ構造

  • ArrayType: PostgreSQL の配列型内部表現。
  • default_weights: tsrank.c 内で定義されているデフォルトの重み配列。

失敗モードと境界条件

  • **ERRCODE_ARRAY_SUBSCRIPT_ERROR**: win が1次元でない、または要素数が不足している場合に発生。
  • **ERRCODE_NULL_VALUE_NOT_ALLOWED**: win に NULL が含まれている場合に発生。
  • **ERRCODE_INVALID_PARAMETER_VALUE**: 重みが 1.0 を超える場合に発生。

保守担当者向けメモ

  • 本関数は ts_rank および ts_rank_cd の引数処理の共通ロジックとして機能している。
  • NUM_WEIGHTS は通常 4（A, B, C, D）を指す。
  • 重みの範囲チェック（0.0〜1.0）は厳密に行われるため、変更時は注意が必要。
