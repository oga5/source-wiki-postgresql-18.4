gin_trgm_consistent

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_gin.c:169

目的

GINインデックスにおいて、検索条件とインデックスキーの整合性を判定する。トリグラムインデックスによる絞り込み結果が、実際の検索条件を満たすかを確認する。

シグネチャまたは形状

Datum gin_trgm_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • check (bool *): インデックスキーの存在フラグ配列。
  • strategy (StrategyNumber): 検索演算子の種類。
  • nkeys (int32): 検索キーの数。
  • extra_data (Pointer *): 正規表現検索用などの追加データ。
  • recheck (bool *): 呼び出し元へ返す再チェックフラグ。
  • **出力**:
  • Datum (bool): 整合性判定結果。

前提条件と不変条件

  • *recheck は常に true に設定される。これはトリグラムインデックスが偽陽性を含むため、検索エンジン側での再検証が必須であることを示す。
  • SimilarityStrategyNumber 系の演算では、トリグラムの出現数から類似度の上限値を計算し、閾値と比較する。

アルゴリズムまたは振る舞い

1. *recheck を true に設定。 2. strategy に応じて分岐:

  • **類似度検索 (Similarity, WordSimilarity, StrictWordSimilarity)**: ntrue（一致したトリグラム数）をカウントし、ntrue / nkeys が閾値以上か判定。
  • **等価・LIKE検索 (Equal, Like, ILike)**: すべてのトリグラムが check 配列に含まれているかを確認。
  • **正規表現検索 (RegExp, RegExpICase)**: nkeys < 1 の場合は全件走査（true）とし、それ以外は trigramsMatchGraph を用いてグラフ構造との整合性を判定。

    • 3. 認識できない strategy の場合はエラーをスロー。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGINインデックスエンジン（直接的な呼び出し元関数名はソース内に明示なし）。
  • **呼び出し先**: index_strategy_get_limit（類似度閾値取得）、trigramsMatchGraph（正規表現判定）、elog（エラー報告）。

関連データ構造

  • StrategyNumber: 検索演算子の識別子。
  • TrgmPackedGraph: 正規表現検索時に extra_data として渡されるグラフ構造。

失敗モードと境界条件

  • ILikeStrategyNumber または RegExpICaseStrategyNumber で IGNORECASE が定義されていない場合、エラーをスローする。
  • strategy が未定義の場合、エラーをスローする。
  • nkeys == 0 の場合、類似度検索では false を返す。

保守担当者向けメモ

  • 本関数は常に recheck = true を返すため、インデックスのヒットは「候補」として扱われる。
  • 類似度計算のロジックは、DIVUNION 定義の有無に関わらず、トリグラムの出現数に基づく上限値判定として共通化されている。
