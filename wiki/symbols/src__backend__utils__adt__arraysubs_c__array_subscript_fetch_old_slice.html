array_subscript_fetch_old_slice

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arraysubs.c:439

目的

配列の更新操作（代入）において、更新前のスライス値を保持するために使用される関数です。

シグネチャまたは形状

static void array_subscript_fetch_old_slice(ExprState *state, ExprEvalStep *op, ExprContext *econtext)

入力と出力

  • **入力**:
  • state: 式評価の状態。
  • op: 現在の式評価ステップ。op->resvalue に現在の配列値、op->resnull にその NULL 状態が格納されている。
  • econtext: 式の評価コンテキスト。
  • **出力**:
  • なし（副作用として sbsrefstate->prevvalue および sbsrefstate->prevnull を更新）。

前提条件と不変条件

  • op->d.sbsref.state から取得される ArraySubWorkspace が初期化済みであること。
  • sbsrefstate->numupper、workspace->upperindex、workspace->lowerindex 等の添字情報が有効であること。

アルゴリズムまたは振る舞い

1. op->resnull が真の場合、配列全体が NULL であると判断し、sbsrefstate->prevnull を true に設定して終了する。 2. op->resnull が偽の場合、array_get_slice を呼び出し、現在の配列から指定された範囲のスライスを取得する。 3. 取得したスライス値を sbsrefstate->prevvalue に格納し、sbsrefstate->prevnull を false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arraysubs.c 内の SubscriptingRef ハンドラ設定箇所（methods->sbs_fetch_old への代入）。
  • **呼び出し先**:
  • array_get_slice（utils/array.h で定義される配列スライス取得関数）。

関連データ構造

  • ExprState, ExprEvalStep: 式評価エンジンで使用される構造体。
  • SubscriptingRefState: 添字操作の実行状態を保持する構造体。
  • ArraySubWorkspace: 添字の境界値や要素の型情報（長さ、アライメント等）を保持する構造体。

失敗モードと境界条件

  • 入力配列が NULL の場合は NULL を返す（失敗ではない）。
  • array_get_slice 内部で発生するメモリ不足や、不正な添字範囲によるエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、代入操作の実行計画において「更新前の値」を保存する必要がある場合にのみ呼び出されます。
  • ArraySubWorkspace の各フィールド（refattrlength 等）は、対象となる配列の型情報に基づいて正しく設定されている必要があります。
