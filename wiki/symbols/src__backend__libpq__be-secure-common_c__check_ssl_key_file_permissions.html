check_ssl_key_file_permissions

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-common.c:113

目的

SSL秘密鍵ファイルの所有権およびパーミッションが、PostgreSQLのセキュリティ要件を満たしているかを確認する。

シグネチャまたは形状

bool check_ssl_key_file_permissions(const char *ssl_key_file, bool isServerStart)

入力と出力

  • **入力**:
  • ssl_key_file: 検証対象のファイルパス。
  • isServerStart: サーバー起動時の呼び出しかどうかを示すフラグ。true の場合、エラー発生時に FATAL レベルで報告される。
  • **出力**:
  • bool: 検証に成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • PostgreSQLがroot権限で動作していないことを前提とする。
  • Unix系環境以外（Windows等）では、パーミッションチェックはスキップされる。

アルゴリズムまたは振る舞い

1. stat() を使用してファイルのメタデータを取得し、アクセス不能な場合はエラーを報告する。 2. S_ISREG を使用し、対象が通常ファイルであることを確認する。 3. Unix系環境において、以下の所有権および権限チェックを行う：

  • ファイルの所有者がデータベース実行ユーザーまたはroot以外である場合は拒否する。
  • データベース実行ユーザーが所有者の場合、0600（所有者のみ読み書き可）より緩い権限は拒否する。
  • rootが所有者の場合、0640（所有者読み書き、グループ読み取り可）より緩い権限は拒否する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-secure-openssl.c (直接の呼び出しを確認)
  • **呼び出し先**:
  • stat() (システムコール)
  • geteuid() (システムコール)
  • ereport() (PostgreSQLエラー報告マクロ)

関連データ構造

  • struct stat: ファイル属性を保持するために使用。

失敗モードと境界条件

  • ファイルが存在しない、またはアクセス権がない場合、FATAL または LOG レベルでエラーを報告し false を返す。
  • 通常ファイルでない場合、設定エラーとして報告し false を返す。
  • 所有権やパーミッションが要件を満たさない場合、エラーを報告し false を返す。

保守担当者向けメモ

  • src/interfaces/libpq/fe-secure-openssl.c に類似のチェックロジックが存在するため、修正時は整合性に注意すること。
  • Windows環境ではUnixスタイルのパーミッションチェックが機能しないため、本関数ではスキップされる設計となっている。
