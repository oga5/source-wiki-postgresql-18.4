make_join_rel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinrels.c:695

目的

2つのリレーション（RelOptInfo）を結合するための新しい結合リレーションを構築し、利用可能なパスを生成する。

シグネチャまたは形状

RelOptInfo *make_join_rel(PlannerInfo *root, RelOptInfo *rel1, RelOptInfo *rel2)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel1, rel2: 結合対象となるリレーション。
  • **出力**:
  • 結合後の RelOptInfo 構造体。結合が論理的に不正な場合は NULL を返す。

前提条件と不変条件

  • rel1 と rel2 のリレーション識別子（relids）は重複してはならない（Assert による検証）。

アルゴリズムまたは振る舞い

1. **正当性検証**: join_is_legal を呼び出し、結合順序や制約（外部結合等）に基づき結合の可否を判定する。 2. **識別子の正規化**: add_outer_joins_to_relids を使用し、外部結合の制約を考慮した正規化された relids を生成する。 3. **順序調整**: 必要に応じて rel1 と rel2 を入れ替える。 4. **結合情報の準備**: 内部結合の場合は init_dummy_sjinfo でダミーの SpecialJoinInfo を作成する。 5. **リレーションの構築**: build_join_rel を呼び出し、結合リレーションの取得または生成を行う。 6. **早期終了**: 結合結果が空（is_dummy_rel）と判定された場合は、パス生成を行わずに終了する。 7. **パス生成**: populate_joinrel_with_paths を呼び出し、結合リレーションに対して実行可能なパスを追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_eval.c
  • src/backend/optimizer/path/joinrels.c（内部の探索ループ等）
  • **呼び出し先**:
  • join_is_legal, add_outer_joins_to_relids, init_dummy_sjinfo, build_join_rel, is_dummy_rel, populate_joinrel_with_paths

関連データ構造

  • RelOptInfo: 結合対象および結果となるリレーション情報。
  • SpecialJoinInfo: 結合の制約（外部結合のタイプや順序制約）を保持する構造体。

失敗モードと境界条件

  • join_is_legal が false を返した場合、結合は行われず NULL が返される。
  • 結合結果が is_dummy_rel と判定された場合、パス生成はスキップされる。

保守担当者向けメモ

  • 本関数は対称性を持つため、rel1 と rel2 の順序は内部で必要に応じて調整される。
  • 結合制約の複雑なロジックは join_is_legal に委譲されているため、結合順序の制約を変更する場合は同関数を確認すること。
