ObjectPropertyType

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/objectaddress.c:97

目的

PostgreSQLのシステムカタログテーブルに関するメタデータを一元管理し、オブジェクトの識別、権限チェック、およびカタログ操作を抽象化するための定義構造体です。

シグネチャまたは形状

typedef struct {
    const char *class_descr;
    Oid         class_oid;
    Oid         oid_index_oid;
    int         oid_catcache_id;
    int         name_catcache_id;
    AttrNumber  attnum_oid;
    AttrNumber  attnum_name;
    AttrNumber  attnum_namespace;
    AttrNumber  attnum_owner;
    AttrNumber  attnum_acl;
    ObjectType  objtype;
    bool        is_nsp_name_unique;
} ObjectPropertyType;

入力と出力

本構造体はデータ保持用であり、関数ではありません。get_object_property_data(Oid class_id) 等の関数を通じて、特定のカタログOIDに対応するメタデータが取得されます。

前提条件と不変条件

  • ObjectProperty 配列（静的定義）の各要素として使用されます。
  • 各フィールドは、対応するシステムカタログのスキーマ定義と一致している必要があります。

アルゴリズムまたは振る舞い

  • システムカタログのOIDから、そのカタログのインデックス、キャッシュID、および主要な属性番号（OID、名前、名前空間、所有者、ACL）をルックアップするために使用されます。
  • is_nsp_name_unique フラグは、名前空間と名前の組み合わせ（または名前のみ）がオブジェクトを一意に特定できるかを判定するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: get_object_property_data 関数および src/backend/catalog/objectaddress.c 内のオブジェクト操作関数群。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出しません。

関連データ構造

  • ObjectAddress: 本構造体で定義されたメタデータを用いて操作される対象。
  • ObjectProperty: 本構造体のインスタンスを保持する静的配列。

失敗モードと境界条件

  • 存在しない class_id に対して get_object_property_data を呼び出した場合、適切なメタデータが返されない可能性があります（呼び出し側のハンドリングに依存）。

保守担当者向けメモ

  • 新しいオブジェクトタイプを追加する場合は、ObjectProperty 配列への定義追加と、get_object_address 内の switch 文へのケース追加が必要です。
  • 属性番号（attnum_*）の変更は、対応するシステムカタログの定義変更と同期させる必要があります。
