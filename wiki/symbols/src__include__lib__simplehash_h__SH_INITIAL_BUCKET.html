SH_INITIAL_BUCKET

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/lib/simplehash.h:356

目的

ハッシュ値から、ハッシュテーブル内の探索を開始すべき「最適位置（初期バケット）」を算出します。

シグネチャまたは形状

static inline uint32 SH_INITIAL_BUCKET(SH_TYPE * tb, uint32 hash)

入力と出力

  • **入力**:
  • tb: ハッシュテーブル構造体へのポインタ。
  • hash: 計算済みのハッシュ値。
  • **出力**:
  • uint32: ハッシュテーブルのバケット配列におけるインデックス。

前提条件と不変条件

  • tb->sizemask が、ハッシュテーブルのサイズ（2のべき乗）に基づいて適切に設定されている必要があります。
  • hash は、ハッシュテーブルのサイズに対して有効な範囲内である必要はありません（本関数内でマスク処理されます）。

アルゴリズムまたは振る舞い

ハッシュ値に対して tb->sizemask をビット単位の論理積（AND）演算することで、ハッシュテーブルのサイズ内に収まるインデックスを算出します。これは、ハッシュテーブルのサイズが常に2のべき乗であることを前提とした高速な剰余演算です。

呼び出し元と呼び出し先

  • **呼び出し元**: simplehash.h 内の各操作関数（SH_INSERT, SH_LOOKUP 等の内部実装）から頻繁に呼び出されます。具体的には、行番号 550, 573, 642, 680, 802, 860, 900, 932, 963, 1102 での利用が確認されています。
  • **呼び出し先**: 外部関数は呼び出しません。

関連データ構造

  • SH_TYPE: 生成されるハッシュテーブルの構造体型。sizemask メンバを保持します。

失敗モードと境界条件

  • tb が NULL の場合、セグメンテーション違反が発生します。
  • ハッシュテーブルが初期化されていない場合、sizemask が不正な値である可能性があり、予期しないインデックスを返す場合があります。

保守担当者向けメモ

  • 本関数は simplehash.h のテンプレートマクロによって生成されるコードの一部です。
  • 性能向上のため、インライン展開されることが前提となっています。
  • sizemask を使用しているため、ハッシュテーブルのサイズが2のべき乗であるという不変条件が崩れると正しく動作しません。
