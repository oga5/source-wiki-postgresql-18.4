目的

dropdb コマンドのエントリポイントとして、コマンドライン引数の解析、データベース接続の確立、および DROP DATABASE SQL 文の生成と実行を制御します。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。データベース名および接続オプション（ホスト、ポート、ユーザー名等）を受け取ります。
  • **出力**:
  • 正常終了時は 0 を返します。
  • 引数エラーやデータベース削除失敗時は 1 を返して終了します。

前提条件と不変条件

  • 実行環境において PostgreSQL クライアントライブラリが利用可能であること。
  • 削除対象のデータベース名が引数として必須です。
  • 削除対象のデータベース自体には接続できないため、接続先として別のメンテナンス用データベース（デフォルトは postgres）が必要です。

アルゴリズムまたは振る舞い

1. **初期化**: ログ設定、ロケール設定、およびヘルプ/バージョンオプションの処理を行います。 2. **引数解析**: getopt_long を使用し、接続パラメータや削除オプション（--if-exists, --force 等）を解析します。 3. **検証**: データベース名が指定されているか確認します。--interactive 指定時はユーザーに削除の確認を求めます。 4. **接続**: connectMaintenanceDatabase を呼び出し、サーバーへ接続します。postgres データベースを削除しようとしている場合は、自動的に template1 へ接続先を切り替えます。 5. **SQL 生成**: fmtIdEnc でデータベース名をエスケープし、DROP DATABASE 文を構築します。 6. **実行**: PQexec で SQL を送信し、結果を確認します。成功した場合は接続を閉じて終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（プログラムのエントリポイント）。
  • **呼び出し先**:
  • getopt_long, pg_logging_init, connectMaintenanceDatabase, initPQExpBuffer, appendPQExpBuffer, PQexec, PQfinish 等の PostgreSQL フロントエンドライブラリ関数。
  • yesno_prompt（対話的確認用）。
  • **注記**: ツールによる検索結果に含まれる contrib/amcheck 等の main は、本シンボルとは無関係な別の関数です。

関連データ構造

  • struct option: コマンドライン引数の定義に使用。
  • ConnParams: 接続パラメータを保持。
  • PQExpBufferData: SQL 文構築用のバッファ。
  • PGconn, PGresult: libpq の接続および実行結果オブジェクト。

失敗モードと境界条件

  • **引数不足**: データベース名が指定されていない場合、エラーを出力して終了します。
  • **接続失敗**: メンテナンス用データベースへの接続に失敗した場合、エラーを出力します。
  • **SQL 実行失敗**: DROP DATABASE がサーバー側で拒否された場合（権限不足やデータベース使用中など）、エラーメッセージを出力して終了します。

保守担当者向けメモ

  • 新しいオプションを追加する場合は、long_options 配列と getopt_long の switch 文を更新してください。
  • データベース名の処理には必ず fmtIdEnc を使用し、SQL インジェクションを防止してください。
