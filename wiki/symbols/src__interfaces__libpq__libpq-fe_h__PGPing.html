PGPing

定義

  • **種別**: enum
  • **定義箇所**: src/interfaces/libpq/libpq-fe.h:174

目的

データベースサーバーへの接続可能性を診断した結果を示すステータスコードを定義します。

シグネチャまたは形状

typedef enum {
    PQPING_OK,          /* サーバーは接続を受け入れている */
    PQPING_REJECT,      /* サーバーは稼働中だが接続を拒否している */
    PQPING_NO_RESPONSE, /* 接続を確立できなかった */
    PQPING_NO_ATTEMPT   /* パラメータ不正等により接続を試行しなかった */
} PGPing;

入力と出力

  • **出力**: PQping および PQpingParams 関数の戻り値として使用されます。

前提条件と不変条件

  • 列挙型の順序は、外部システムやプロトコルとの互換性に影響するため変更してはなりません。

アルゴリズムまたは振る舞い

  • サーバーの生存確認および接続可否の判定結果を、上記4つの状態のいずれかとして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/scripts/pg_isready.c
  • src/test/regress/pg_regress.c
  • src/interfaces/libpq/fe-connect.c (内部関数 internal_ping 等)
  • **呼び出し先**:
  • 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • PGconn: 接続管理に使用される不透明構造体。

失敗モードと境界条件

  • PQPING_NO_ATTEMPT: 接続文字列やパラメータが不正な場合に発生します。
  • PQPING_NO_RESPONSE: ネットワーク到達不能やサーバー停止時に発生します。

保守担当者向けメモ

  • 列挙値の追加は可能ですが、既存の順序を変更するとバイナリ互換性や外部ツールとの連携に支障をきたす可能性があるため注意してください。
