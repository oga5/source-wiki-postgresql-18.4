AllocBlock

定義

  • 種別: typedef (構造体 AllocBlockData へのポインタ)
  • 定義箇所: src/backend/utils/mmgr/aset.c:107

目的

AllocSet におけるメモリ管理の基本単位を指し示すための型です。malloc() によって確保された大きなメモリブロックを抽象化し、その内部を小さなチャンクに分割して管理するために使用されます。

シグネチャまたは形状

typedef struct AllocBlockData *AllocBlock;

入力と出力

  • 本シンボル自体は型定義であり、関数ではありません。
  • 関連する操作（AllocSetAlloc 等）において、メモリブロックのヘッダーとして機能し、ブロックリストのリンクや空き領域の管理に使用されます。

前提条件と不変条件

  • AllocBlock は AllocSetContext が保持するブロックリストの一部として管理されます。
  • 巨大な割り当て（ALLOC_CHUNK_LIMIT 超）の場合、そのチャンク専用の AllocBlock として扱われます。

アルゴリズムまたは振る舞い

  • AllocSet 内でメモリブロックのリストを構成し、next および prev ポインタを通じてブロック間を相互参照します。
  • メモリ確保時には、現在のブロックの空き領域が不足した場合に新しい AllocBlock が生成されます。
  • メモリ解放時には、ブロック単位での free() またはフリーリストへのチャンク返却が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: AllocSetAlloc、AllocSetFree、AllocSetReset 等の aset.c 内の関数が、メモリブロックの操作や走査のためにこの型を使用しています。
  • **呼び出し先**: 本シンボルは型定義であるため、直接的な呼び出し先はありません。ただし、AllocBlockData 構造体のメンバ（next, prev）を介して他のブロックを参照します。

関連データ構造

  • AllocBlockData: AllocBlock が指す実体となる構造体。
  • AllocSetContext: メモリコンテキストのヘッダー。ブロックリストの先頭を保持します。

失敗モードと境界条件

  • メモリ不足時には malloc() が失敗し、AllocBlock の生成ができません。
  • 巨大な割り当ての場合、ブロック全体がそのチャンク専用となり、通常のフリーリスト管理とは異なる振る舞いをします。

保守担当者向けメモ

  • AllocBlock は aset.c におけるメモリ管理の根幹です。AllocBlockData の構造変更は、メモリレイアウトとアライメントに直接影響するため注意してください。
  • AllocBlock を取得する際は、chunk ポインタから ALLOC_BLOCKHDRSZ を減算する計算（aset.c:216）が頻出するため、この計算ロジックの整合性を維持してください。
