mic_to_latin4

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/latin_and_mic/latin_and_mic.c:125

目的

MULE_INTERNAL (MIC) エンコーディングから LATIN4 エンコーディングへの文字コード変換を行うための PostgreSQL 用変換プロシージャです。

シグネチャまたは形状

Datum mic_to_latin4(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • src (CSTRING): 変換元バッファ
  • dest (CSTRING): 変換先バッファ
  • len (INT32): 変換対象のバイト長
  • noError (BOOL): エラー発生時の挙動フラグ
  • **出力**: PG_RETURN_INT32 を介して、変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_MULE_INTERNAL, PG_LATIN4) により、呼び出し時のエンコーディングIDが MULE_INTERNAL から LATIN4 への変換として正しいことが検証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認します。 3. mb/pg_wchar.h で定義された mic2latin 関数を呼び出し、LC_ISO8859_4（LATIN4）を指定して変換を実行します。 4. 変換結果のバイト数を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の変換フレームワーク（PG_FUNCTION_INFO_V1(mic_to_latin4) により登録）。
  • **呼び出し先**: mic2latin（mb/pg_wchar.h で定義された変換ヘルパー関数）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約で使用される引数構造体。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラー処理機構により処理が中断されます。
  • noError フラグの値に応じて、変換中の不正な文字に対する挙動が mic2latin 内で制御されます。

保守担当者向けメモ

  • 本関数はボイラープレート的なラッパーです。実際の変換ロジックは mic2latin に依存しているため、変換アルゴリズムの変更が必要な場合は src/backend/utils/mb/wchar.c を参照してください。
