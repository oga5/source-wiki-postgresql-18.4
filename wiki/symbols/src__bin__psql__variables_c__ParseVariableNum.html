ParseVariableNum

定義

  • 種別: function
  • 定義箇所: src/bin/psql/variables.c:157

目的

文字列形式の変数値から整数値への安全な変換と検証を行います。

シグネチャまたは形状

bool ParseVariableNum(const char *value, const char *name, int *result)

入力と出力

  • **入力**:
  • value: 変換対象の文字列（NULL の場合は空文字列として扱われる）。
  • name: エラーログ出力時に使用する変数名（NULL 指定可能）。
  • result: 変換結果を格納する整数型ポインタ。
  • **出力**:
  • 変換が成功した場合は true を返し、result に値を格納する。
  • 失敗した場合は false を返し、result は変更しない。

前提条件と不変条件

  • value が NULL の場合、内部的に空文字列として処理されます。
  • 変換後の値は int 型の範囲内に収まる必要があります。

アルゴリズムまたは振る舞い

1. value が NULL ならば空文字列に置換します。 2. strtol を使用して文字列を数値に変換します。 3. 以下の条件をすべて満たす場合に成功とみなします。

  • errno が 0 であること。
  • 文字列全体が数値として解釈されていること（*end == '\0'）。
  • 空文字列ではないこと（end != value）。
  • 変換結果が int 型の範囲内であること。

    • 4. 失敗時は、name が指定されていれば pg_log_error を通じてエラーメッセージを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c: pager_min_lines の設定時。
  • src/bin/psql/startup.c: FETCH_COUNT, HISTSIZE, IGNOREEOF などの設定時。
  • **呼び出し先**:
  • strtol (標準ライブラリ)
  • pg_log_error (共通ログ出力)

関連データ構造

  • なし（プリミティブ型のみを使用）。

失敗モードと境界条件

  • **変換失敗**: 数値以外の文字が含まれる場合や、空文字列の場合、または int 型の範囲を超える値の場合は false を返します。
  • **エラーログ**: name が NULL の場合、変換失敗時のエラーメッセージ出力は抑制されます。

保守担当者向けメモ

  • 本関数は psql の設定変数（文字列）を内部の整数型変数へ同期させる際に使用されます。
  • 変換失敗時に result を書き換えないため、呼び出し側はデフォルト値を保持したまま安全に処理を継続できます。
