get_opclass

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_utilcmd.c:2210

目的

指定されたインデックス演算子クラス（opclass）の完全修飾名（スキーマ名と演算子クラス名）を取得します。

シグネチャまたは形状

List *get_opclass(Oid opclass, Oid actual_datatype)

入力と出力

  • **入力**:
  • opclass (Oid): 検索対象の演算子クラスのOID。
  • actual_datatype (Oid): 演算子クラスが適用される対象のデータ型。
  • **出力**:
  • List *: 演算子クラスがデフォルトでない場合、list_make2 で作成された (namespace_name, opclass_name) のリストを返します。デフォルトの場合は NIL を返します。

前提条件と不変条件

  • CLAOID システムキャッシュを使用して opclass を検索します。
  • 検索対象の opclass がシステムカタログ pg_opclass に存在しない場合、elog(ERROR) により処理を中断します。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を用いて pg_opclass から演算子クラス情報を取得します。 2. GetDefaultOpClass を呼び出し、指定されたデータ型とアクセスメソッドにおけるデフォルトの演算子クラスと、引数の opclass を比較します。 3. デフォルトと一致しない場合、get_namespace_name でスキーマ名を取得し、演算子クラス名と共にリストとして返します。 4. 最後に ReleaseSysCache を呼び出し、キャッシュエントリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_utilcmd.c 内の transformIndexConstraints（1933行目付近）から呼び出されます。
  • **呼び出し先**: SearchSysCache1, HeapTupleIsValid, GETSTRUCT, GetDefaultOpClass, get_namespace_name, pstrdup, NameStr, list_make2, makeString, ReleaseSysCache。

関連データ構造

  • Form_pg_opclass: pg_opclass システムカタログの行構造体。
  • HeapTuple: システムキャッシュから取得されるタプル。

失敗モードと境界条件

  • 指定された opclass OID が無効な場合、キャッシュルックアップ失敗としてエラーが発生します。
  • 演算子クラスが対象データ型のデフォルトである場合、戻り値は NIL となります。

保守担当者向けメモ

  • 本関数は、インデックス定義の生成時に演算子クラスを明示的に指定する必要があるかどうかを判定するために使用されます。
  • スキーマ修飾名は常に付与される仕様です。
