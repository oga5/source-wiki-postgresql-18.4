pg_realloc

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:70

目的

標準ライブラリの realloc をラップし、メモリ確保失敗時にプロセスを終了させることで、フロントエンドアプリケーションにおける安全なメモリ再確保を実現します。

シグネチャまたは形状

void *pg_realloc(void *ptr, size_t size)

入力と出力

  • **入力**:
  • ptr: 再確保対象のメモリ領域へのポインタ（NULL 可）。
  • size: 要求する新しいサイズ（バイト単位）。
  • **出力**:
  • 再確保されたメモリ領域へのポインタ。成功時は必ず有効なポインタを返します。

前提条件と不変条件

  • ptr が NULL かつ size が 0 の場合、移植性の観点から size を 1 に補正して処理します。
  • メモリ確保に失敗した場合、関数は呼び出し元へ戻らず、stderr にエラーメッセージを出力して exit(EXIT_FAILURE) を実行します。

アルゴリズムまたは振る舞い

1. ptr が NULL で size が 0 の場合、size を 1 に設定します。 2. 標準の realloc(ptr, size) を呼び出します。 3. 戻り値が NULL の場合、エラーメッセージを出力しプロセスを終了します。 4. 成功した場合は、確保されたメモリ領域へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/oid2name/oid2name.c
  • src/bin/initdb/initdb.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_basebackup/pg_recvlogical.c
  • src/bin/pg_dump/compress_lz4.c
  • src/bin/pg_dump/parallel.c
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_dump/pg_dump.c
  • **呼び出し先**:
  • realloc (標準ライブラリ)
  • fprintf (標準ライブラリ)
  • exit (標準ライブラリ)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **メモリ不足**: realloc が NULL を返した場合、プロセスは即座に終了します。
  • **サイズ 0**: ptr が NULL の場合に size が 0 であれば、malloc(1) と同等の挙動をとります。

保守担当者向けメモ

  • 本関数は FRONTEND マクロが定義されている環境でのみ使用可能です。
  • バックエンドの repalloc とは異なり、メモリ不足時にエラーを返さずプロセスを終了させる設計である点に注意してください。
