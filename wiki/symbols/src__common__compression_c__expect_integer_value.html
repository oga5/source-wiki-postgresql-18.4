expect_integer_value

定義

  • 種別: function
  • 定義箇所: src/common/compression.c:274

目的

圧縮設定オプションの文字列値を整数値に変換し、検証を行う。

シグネチャまたは形状

static int expect_integer_value(char *keyword, char *value, pg_compress_specification *result);

入力と出力

  • **入力**:
  • keyword: オプション名（エラーメッセージ用）。
  • value: パース対象の文字列。
  • result: エラー発生時にエラーメッセージを格納する pg_compress_specification 構造体へのポインタ。
  • **出力**:
  • 成功時: 変換された整数値。
  • 失敗時: -1。

前提条件と不変条件

  • result は有効な pg_compress_specification 構造体を指している必要がある。
  • value が NULL の場合、または整数として解釈できない文字列（空文字や非数値を含む）の場合、エラーとして扱われる。

アルゴリズムまたは振る舞い

1. value が NULL かどうかを確認し、NULL ならば result->parse_error にエラーメッセージをセットして -1 を返す。 2. strtol を使用して value を基数 10 の整数に変換する。 3. 変換後のポインタが入力開始位置と一致するか、または変換後の文字列に余分な文字が含まれている場合は、整数として不正と判断し、result->parse_error にエラーメッセージをセットして -1 を返す。 4. 上記のいずれにも該当しない場合、変換された整数値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/compression.c 内の parse_compress_specification 関数（level および workers オプションのパース時に呼び出し）。
  • **呼び出し先**:
  • strtol (標準ライブラリ)
  • psprintf (PostgreSQL ユーティリティ)

関連データ構造

  • pg_compress_specification: 圧縮設定を保持する構造体。本関数はエラー発生時にこの構造体の parse_error フィールドを更新する。

失敗モードと境界条件

  • value が NULL: エラーメッセージを生成し -1 を返す。
  • value が数値以外の文字を含む: エラーメッセージを生成し -1 を返す。
  • value が空文字: エラーメッセージを生成し -1 を返す。

保守担当者向けメモ

  • 本関数は parse_error フィールドを更新するため、呼び出し側はエラー発生時に result->parse_error を確認し、必要に応じてメモリ解放（pfree）を行う必要がある。
