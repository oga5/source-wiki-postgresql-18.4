FinalizeIncrementalManifest

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_incremental.c:226

目的

増分バックアップ用マニフェストの解析処理を終了し、関連するメモリリソースを解放する。

シグネチャまたは形状

void FinalizeIncrementalManifest(IncrementalBackupInfo *ib)

入力と出力

  • **入力**: ib (IncrementalBackupInfo *) - 増分バックアップのメタデータおよび解析状態を保持する構造体。
  • **出力**: なし（副作用としてメモリ解放と解析状態のクリーンアップを行う）。

前提条件と不変条件

  • ib は有効な IncrementalBackupInfo 構造体を指していること。
  • ib->mcxt に適切なメモリコンテキストが設定されていること。
  • 本関数呼び出し前に、マニフェストデータの断片が AppendIncrementalManifestData 等を通じて蓄積されていること。

アルゴリズムまたは振る舞い

1. ib->mcxt にメモリコンテキストを切り替える。 2. json_parse_manifest_incremental_chunk を呼び出し、バッファに残っている最後のマニフェストチャンクを解析する（is_last=true を指定）。 3. ib->buf.data を解放し、ポインタを NULL に設定する。 4. json_parse_manifest_incremental_shutdown を呼び出し、ib->inc_state（JSONパーサーの状態）を解放する。 5. 元のメモリコンテキストに切り替えて戻る。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/walsender.c:701
  • **呼び出し先**:
  • MemoryContextSwitchTo
  • json_parse_manifest_incremental_chunk
  • pfree
  • json_parse_manifest_incremental_shutdown

関連データ構造

  • IncrementalBackupInfo: 増分バックアップの解析状態を保持する主要構造体。

失敗モードと境界条件

  • ib が NULL の場合、メモリ参照エラーが発生する可能性がある。
  • マニフェストの解析中に不正なJSON形式が検出された場合、json_parse_manifest_incremental_chunk 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、マニフェスト解析の最終フェーズで必ず呼び出す必要がある。
  • ib->buf.data の解放後に再度マニフェストデータを追加しようとするとクラッシュするため、呼び出し順序には注意すること。
