llvm_leave_fatal_on_oom

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_error.cpp:69

目的

LLVMのJITコンパイル処理終了時に、一時的に差し替えていたC++およびLLVMのエラーハンドラを元の状態へ復元する。

シグネチャまたは形状

void llvm_leave_fatal_on_oom(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として静的変数の更新とグローバルなハンドラ設定の変更を行う）

前提条件と不変条件

  • llvm_enter_fatal_on_oom() によってハンドラが差し替えられていること。
  • fatal_new_handler_depth が正の整数であること。

アルゴリズムまたは振る舞い

1. 静的変数 fatal_new_handler_depth をデクリメントする。 2. ネストレベルが 0 に達した場合（最外層の呼び出し終了時）:

  • std::set_new_handler を使用して、保存されていた old_new_handler を復元する。
  • llvm::remove_bad_alloc_error_handler() を呼び出し、LLVMのメモリ不足ハンドラを解除する。
  • llvm::remove_fatal_error_handler() を呼び出し、LLVMの致命的エラーハンドラを解除する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/jit/llvm/llvmjit.c:306
  • src/backend/jit/llvm/llvmjit_expr.c:2970, 2998
  • 呼び出し先:
  • std::set_new_handler
  • llvm::remove_bad_alloc_error_handler
  • llvm::remove_fatal_error_handler

関連データ構造

  • fatal_new_handler_depth: ネストレベルを追跡する静的変数。
  • old_new_handler: 復元対象となる元の new_handler を保持する変数。

失敗モードと境界条件

  • fatal_new_handler_depth が 0 の状態で呼び出された場合、負の値になる可能性がある（本関数単体ではチェックを行わない）。
  • LLVMのハンドラ削除関数が内部で例外を投げる可能性については、LLVMの仕様に依存する。

保守担当者向けメモ

  • 本関数は llvm_enter_fatal_on_oom と対になる。ネストされた呼び出しをサポートするため、カウンタによる管理を行っている。
  • エラー発生時のリカバリには llvm_reset_after_error() が使用されるため、異常終了時には本関数が呼ばれない可能性がある点に注意すること。
