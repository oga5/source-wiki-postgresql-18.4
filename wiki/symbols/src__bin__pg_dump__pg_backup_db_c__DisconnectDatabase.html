DisconnectDatabase

定義

  • **種別**: 関数
  • **定義箇所**: src/bin/pg_dump/pg_backup_db.c:163

目的

データベース接続を安全に終了し、関連するリソースを解放します。

シグネチャまたは形状

void DisconnectDatabase(Archive *AHX)

入力と出力

  • **入力**: Archive *AHX（ArchiveHandle 構造体へのポインタとしてキャストして使用）
  • **出力**: なし（void）

前提条件と不変条件

  • AHX が指す ArchiveHandle 構造体は有効である必要があります。
  • 接続が既に閉じられている（AH->connection が NULL）場合、関数は何もせず即座に復帰します。

アルゴリズムまたは振る舞い

1. AH->connection が NULL の場合は終了します。 2. AH->connCancel が存在する場合、現在のトランザクション状態が PQTRANS_ACTIVE であれば PQcancel を呼び出し、クエリのキャンセルを試みます。 3. set_archive_cancel_info(AH, NULL) を呼び出し、シグナルハンドラによる以降のキャンセル要求を無効化します。 4. PQfinish(AH->connection) を呼び出して libpq の接続を破棄します。 5. AH->connection を NULL に設定し、接続が閉じられたことを記録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/parallel.c (複数箇所)
  • src/bin/pg_dump/pg_backup_archiver.c (複数箇所)
  • **呼び出し先**:
  • PQcancel (libpq)
  • set_archive_cancel_info
  • PQfinish (libpq)
  • PQtransactionStatus (libpq)

関連データ構造

  • ArchiveHandle (src/bin/pg_dump/pg_backup_archiver.h): 接続情報（connection）およびキャンセル情報（connCancel）を保持する主要構造体。

失敗モードと境界条件

  • AH->connection が既に NULL の場合、安全に無視されます。
  • PQcancel の実行結果は無視されます。これは主に pg_fatal 等の異常終了時に、可能な限りクリーンアップを試みるための設計です。

保守担当者向けメモ

  • 本関数は ArchiveHandle を介して libpq の接続状態を管理します。接続終了後に AH->connection を NULL にリセットすることで、二重解放を防いでいます。
  • キャンセル処理はベストエフォートであり、エラーが発生してもプロセス全体の終了処理には影響しません。
