bool_accum_inv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:361

目的

集約関数における逆遷移関数（inverse transition function）として、bool_accum で蓄積された状態から特定の値を減算し、ウィンドウ関数やインクリメンタルな集約をサポートします。

シグネチャまたは形状

Datum bool_accum_inv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: BoolAggState へのポインタ（集約状態）。
  • 第2引数: 減算対象のブール値。
  • **出力**: 更新された BoolAggState へのポインタを Datum として返します。

前提条件と不変条件

  • 第1引数（状態）が NULL である場合、エラーが発生します（bool_accum によって事前に初期化されていることが前提）。
  • 第2引数が NULL でない場合のみ、状態の更新が行われます。

アルゴリズムまたは振る舞い

1. PG_ARGISNULL(0) を確認し、状態ポインタを取得します。 2. 状態が NULL の場合は elog(ERROR) を送出します。 3. 第2引数が非 NULL の場合：

  • state->aggcount をデクリメントします。
  • 第2引数の値が true であれば、state->aggtrue をデクリメントします。

    • 4. 更新された状態ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの集約実行エンジン（ウィンドウ関数等の逆集約処理）。
  • **呼び出し先**: elog（エラー発生時）。

関連データ構造

  • BoolAggState: 集約計算中に保持される内部状態構造体。aggcount（総数）と aggtrue（真の数）を管理します。

失敗モードと境界条件

  • 状態が初期化されていない（NULL）状態で呼び出された場合、エラーとなります。
  • 第2引数が NULL の場合は何も行わず、状態をそのまま返します。

保守担当者向けメモ

  • 本関数は bool_accum と対になる設計です。集約状態の整合性を維持するため、bool_accum で加算したロジックと逆の操作を正確に行う必要があります。
