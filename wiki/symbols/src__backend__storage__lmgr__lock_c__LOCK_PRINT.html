LOCK_PRINT

定義

  • 種別: function (inline static)
  • 定義箇所: src/backend/storage/lmgr/lock.c:369

目的

ロックオブジェクトの状態をデバッグログとして出力する。

シグネチャまたは形状

void LOCK_PRINT(const char *where, const LOCK *lock, LOCKMODE type)

入力と出力

  • where: ログメッセージの先頭に付与する識別文字列。
  • lock: 出力対象の LOCK 構造体へのポインタ。
  • type: 出力するロックモード。
  • 出力: elog(LOG, ...) を介して、ロックのタグ情報、要求数、保持数、待機プロセス数などを標準ログに出力する。

前提条件と不変条件

  • LOCK_DEBUG_ENABLED(&lock->tag) が真である場合にのみログが出力される。
  • lock ポインタは有効な LOCK 構造体を指している必要がある。

アルゴリズムまたは振る舞い

1. LOCK_DEBUG_ENABLED マクロにより、対象のロックタグがデバッグ対象かを確認する。 2. デバッグ有効時、lock 構造体の各フィールド（tag, grantMask, requested 配列, granted 配列, waitProcs のカウント）をフォーマットしてログ出力する。 3. ロックモード名は LockMethods 配列から取得される。

呼び出し元と呼び出し先

  • 呼び出し元: LockHasWaiters, LockAcquire, GrantLock, UnGrantLock, CleanUpLock, LockRelease, LockReleaseAll など、ロック管理の各主要関数からデバッグ目的で呼び出される。
  • 呼び出し先: LOCK_DEBUG_ENABLED, dclist_count, elog。

関連データ構造

  • LOCK: ロック状態を保持する主要構造体。
  • LOCKMODE: ロックのモードを定義する列挙型。
  • LockMethods: ロックメソッドの定義テーブル。

失敗モードと境界条件

  • lock が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • LOCK_DEBUG が定義されていない環境では、マクロにより空の式 ((void) 0) に置換され、何も実行されない。

保守担当者向けメモ

  • 本関数はデバッグ専用である。本番環境で詳細なロック状態を追跡したい場合は、LOCK_DEBUG を有効にして再コンパイルする必要がある。
  • requested および granted 配列のインデックスは 1 から 7 までを想定している。
