定義

  • 定義箇所: src/backend/storage/lmgr/s_lock.c:250

目的

スピンロック（slock_t）の動作を検証するためのスタンドアローンテストプログラムです。S_LOCK_TEST マクロが定義されている場合にコンパイルされ、ロックの初期化、取得、解放、およびスタック検出の挙動を確認します。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（実行時引数は受け取らない）
  • 出力: 成功時は終了コード 1 を返して終了（意図的なスタック検出による終了を含む）。標準出力にテストの進捗および失敗時のエラーメッセージを出力する。

前提条件と不変条件

  • pg_global_prng_state が初期化可能であること。
  • test_lock 構造体のパディング領域（pad1, pad2）が、ロック操作前後で破壊されていないこと（メモリ破壊のチェック）。

アルゴリズムまたは振る舞い

1. **初期化**: 乱数生成器をシードし、テスト用ロックを S_INIT_LOCK で初期化する。 2. **基本操作テスト**:

  • S_LOCK_FREE による初期状態の確認。
  • S_LOCK によるロック取得と S_LOCK_FREE による状態変化の確認。
  • S_UNLOCK によるロック解放と S_LOCK_FREE による状態復帰の確認。
  • 再度のロック取得と状態確認。

    • 3. **スタック検出テスト**:

  • s_lock を呼び出し、意図的にスタック（デッドロック状態）を発生させる。
  • 成功すれば NUM_DELAYS 回の試行後に「stuck spinlock」メッセージを表示して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（スタンドアローン実行時）
  • 呼び出し先: pg_prng_seed, S_INIT_LOCK, S_LOCK_FREE, S_LOCK, S_UNLOCK, s_lock, printf, fflush
  • 注記: ツールが提示した contrib/amcheck 等の main への言及は、本関数の呼び出し元ではなく、同名の別シンボルを指す可能性が高い。

関連データ構造

  • test_lock: テスト用の静的構造体（slock_t を含む）。
  • pg_global_prng_state: 乱数生成器の状態。

失敗モードと境界条件

  • **メモリ破壊**: pad1 または pad2 の値が 0x44 から変更された場合、データ型のサイズ不整合としてエラー終了する。
  • **ロック状態異常**: S_LOCK_FREE が期待通りの値を返さない場合（初期化直後、ロック後、解放後）、エラー終了する。
  • **スタック検出**: 最後の s_lock 呼び出しでロックが取得できない場合、プログラムは「stuck spinlock」メッセージを出力して終了する（これはテストの正常な終了フローである）。

保守担当者向けメモ

  • 本関数は S_LOCK_TEST マクロが定義されている環境でのみ有効です。
  • s_lock の実装変更時には、このテストを実行してスピンロックの基本挙動とスタック検出ロジックが正しく動作することを確認してください。
