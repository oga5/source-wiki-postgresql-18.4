目的

ECPG（Embedded SQL for C）における JSON_TABLE 構文の動作検証を行うための回帰テスト用エントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は受け取らない）
  • 出力: 正常終了時に 0 を返却。標準出力に Found foo=<値> を出力。

前提条件と不変条件

  • データベース ecpg1_regression への接続が成功していること。
  • ホスト変数 foo（int 型）が宣言されており、SQL クエリの結果を格納するバッファとして機能すること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、ECPGconnect でデータベースに接続。ECPGsetcommit でコミットモードを有効化。 2. **クエリ実行1**: JSON_TABLE を使用して JSON 文字列から値を抽出し、ホスト変数 foo にバインドして取得。結果を printf で出力。 3. **クエリ実行2**: ネストされた JSON_TABLE 構造を使用して値を抽出し、再度 foo にバインドして取得。結果を printf で出力。 4. **終了処理**: ECPGdisconnect で接続を終了。 5. **エラーハンドリング**: 各ステップで sqlca.sqlcode を確認し、負の値（エラー）の場合は sqlprint() を呼び出して診断情報を出力。

呼び出し元と呼び出し先

  • 呼び出し元: 不明（テスト実行環境から直接呼び出される想定）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGsetcommit, ECPGdo, ECPGdisconnect, sqlprint, printf

関連データ構造

  • sqlca: SQL 通信領域。実行結果のステータス保持に使用。
  • foo: int 型のホスト変数。

失敗モードと境界条件

  • データベース接続失敗時、または SQL クエリ実行失敗時に sqlca.sqlcode < 0 となり、sqlprint() がエラー情報を出力して処理を継続する。
