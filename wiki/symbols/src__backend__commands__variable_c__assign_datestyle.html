assign_datestyle

定義

  • 種別: function
  • 定義箇所: src/backend/commands/variable.c:243

目的

GUC（Grand Unified Configuration）サブシステムにおいて、検証済みの datestyle 設定値をシステム内部の状態へ反映させるための assign_hook です。

シグネチャまたは形状

void assign_datestyle(const char *newval, void *extra)

入力と出力

  • newval: 設定される文字列値（本関数内では未使用）。
  • extra: check_datestyle によって事前に検証・準備された、2つの整数値を含む配列へのポインタ。
  • 出力: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • extra 引数は、少なくとも2つの整数要素を持つ配列を指している必要があります。
  • 本関数は、GUCフレームワークにより check_datestyle が成功した後に呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. extra 引数を int * 型のポインタ myextra にキャストします。 2. myextra[0] の値をグローバル変数 DateStyle に代入します。 3. myextra[1] の値をグローバル変数 DateOrder に代入します。

呼び出し元と呼び出し先

  • 呼び出し元: GUCサブシステム（src/backend/utils/misc/guc_tables.c でフックとして登録）。
  • 呼び出し先: なし。

関連データ構造

  • DateStyle（グローバル変数）: 日付の表示形式を保持。
  • DateOrder（グローバル変数）: 日付の順序（DMY, MDY, YMD）を保持。

失敗モードと境界条件

  • extra が NULL の場合、ポインタのデリファレンスによりクラッシュします。GUCフレームワークの仕様上、検証済みデータが渡されることが期待されています。

保守担当者向けメモ

  • 本関数は単なる値のコピーを行うため、複雑なロジックは含まれていません。検証ロジックについては check_datestyle を参照してください。
  • extra 配列のインデックス（0と1）は、check_datestyle での格納順序と密結合しているため、変更時は両方の関数を同時に更新する必要があります。
