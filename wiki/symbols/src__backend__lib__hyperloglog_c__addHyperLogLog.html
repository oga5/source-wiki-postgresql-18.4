addHyperLogLog

定義

  • 種別: function
  • 定義箇所: src/backend/lib/hyperloglog.c:166

目的

HyperLogLog 状態構造体に対し、入力ハッシュ値に基づいてレジスタを更新し、集合の濃度推定のためのデータを蓄積します。

シグネチャまたは形状

void addHyperLogLog(hyperLogLogState *cState, uint32 hash)

入力と出力

  • **入力**:
  • cState: 更新対象の hyperLogLogState 構造体へのポインタ。
  • hash: 集合に追加する要素のハッシュ値（uint32）。
  • **出力**: なし（cState が指す内部レジスタ配列が更新されます）。

前提条件と不変条件

  • cState は initHyperLogLog によって適切に初期化されている必要があります。
  • 入力 hash は一様に分布していることが、推定精度の維持に不可欠です。
  • cState->registerWidth は 4 から 16 の範囲内である必要があります。

アルゴリズムまたは振る舞い

1. **インデックス算出**: hash の上位 registerWidth ビットを抽出し、レジスタ配列のインデックスとして使用します。 2. **ランク算出**: hash の残りのビット（32 - registerWidth ビット）に対して rho 関数を適用し、最初のビットが立つ位置（ランク）を計算します。 3. **レジスタ更新**: 算出したインデックスに対応するレジスタ値と、計算されたランクを比較し、大きい方の値をレジスタに保持します（Max 操作）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/utils/adt/mac.c
  • src/backend/utils/adt/network.c
  • src/backend/utils/adt/numeric.c

  • src/backend/utils/adt/varlena.c
  • **呼び出し先**:
  • rho (補助関数)

関連データ構造

  • hyperLogLogState: レジスタ配列 (hashesArr) およびレジスタ幅 (registerWidth) を保持する構造体。

失敗モードと境界条件

  • registerWidth が不正な値の場合、インデックス算出時に予期せぬ動作を引き起こす可能性があります。
  • 入力ハッシュ値が偏っている場合、推定精度が著しく低下します。

保守担当者向けメモ

  • 本関数は Hideaki Ohno 氏の C++ 実装に基づいています。
  • rho 関数によるビット操作が正しく機能するよう、registerWidth の整合性に注意してください。
