BaseBackupGetSink

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_target.c:162

目的

指定されたバックアップターゲットに基づき、バックアップデータを処理するための bbsink パイプラインを構築・生成します。

シグネチャまたは形状

bbsink *BaseBackupGetSink(BaseBackupTargetHandle *handle, bbsink *next_sink)

入力と出力

  • **入力**:
  • handle: 検証済みのバックアップターゲット情報を保持する BaseBackupTargetHandle へのポインタ。
  • next_sink: パイプラインの後続に接続される bbsink オブジェクト。
  • **出力**:
  • 構築された bbsink オブジェクトへのポインタ。

前提条件と不変条件

  • handle は BaseBackupGetTargetHandle() 等によって事前に検証および初期化されている必要があります。
  • handle->type および handle->type->get_sink が有効な関数ポインタを保持している必要があります。

アルゴリズムまたは振る舞い

1. handle が保持するターゲット定義（BaseBackupTargetType）内の get_sink 関数を呼び出します。 2. その際、引数として next_sink およびターゲット固有の詳細情報である handle->detail_arg を渡します。 3. ターゲット固有のロジックによって生成された bbsink オブジェクトを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:1036 において、バックアップ処理のセットアップ時に呼び出されています。
  • **呼び出し先**: handle->type->get_sink 関数（ターゲットごとに実装が異なります）。

関連データ構造

  • bbsink: バックアップデータの転送先となるシンクインターフェース。
  • BaseBackupTargetHandle: ターゲットの型情報と詳細引数を保持する構造体。
  • BaseBackupTargetType: ターゲットの振る舞い（get_sink 等）を定義する構造体。

失敗モードと境界条件

  • handle が NULL の場合、または handle->type が不正な場合、セグメンテーションフォールトが発生する可能性があります。
  • ターゲット固有の get_sink 実装内でメモリ不足等のエラーが発生した場合、その実装に従って例外がスローされます。

保守担当者向けメモ

  • 本関数は bbsink チェーンの構築における抽象化レイヤーです。新しいバックアップターゲットを追加する場合は、BaseBackupTargetType を定義し、適切な get_sink 関数を実装してください。
