CopyMultiInsertBuffer

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/copyfrom.c:78

目的

COPY FROM コマンド実行時、ターゲットテーブルへのデータ挿入を効率化するためのバルク挿入用バッファとして機能します。

シグネチャまたは形状

typedef struct CopyMultiInsertBuffer
{
    TupleTableSlot *slots[MAX_BUFFERED_TUPLES];
    ResultRelInfo *resultRelInfo;
    BulkInsertState bistate;
    int            nused;
    uint64         linenos[MAX_BUFFERED_TUPLES];
} CopyMultiInsertBuffer;

入力と出力

  • **入力**: COPY ストリームから読み込まれたタプル（slots に格納）および対応する行番号（linenos）。
  • **出力**: 閾値に達した際に table_multi_insert を介してテーブルへ書き出されます。

前提条件と不変条件

  • nused は現在バッファされているタプル数を示し、MAX_BUFFERED_TUPLES を超えることはありません。
  • resultRelInfo は挿入対象のテーブル情報を保持します。
  • bistate は通常のテーブルの場合は BulkInsertState を保持しますが、外部テーブル（foreign table）の場合は NULL となります。
  • BEFORE トリガーや INSTEAD OF トリガーが存在する場合、バルク挿入は無効化されます。

アルゴリズムまたは振る舞い

1. COPY メインループからタプルを受け取り、slots 配列に蓄積します。 2. nused をインクリメントし、対応する入力ストリーム上の行番号を linenos に記録します。 3. バッファが満杯になった場合、または COPY 処理の終了時に CopyMultiInsertBufferFlush が呼び出され、蓄積されたタプルが一括してテーブルへ挿入されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c 内の COPY 実行ロジック（CopyMultiInsertInfoSetupBuffer 等による初期化および挿入処理）。
  • **呼び出し先**: table_multi_insert（データ挿入時）。
  • 注: 呼び出し関係の詳細は、ソースコード内の静的参照に基づいています。

関連データ構造

  • CopyMultiInsertInfo: 複数の CopyMultiInsertBuffer を管理する上位構造体。
  • TupleTableSlot: 挿入対象のタプルを保持するコンテナ。
  • ResultRelInfo: 挿入先テーブルのメタデータ。
  • BulkInsertState: バルク挿入の最適化状態。

失敗モードと境界条件

  • MAX_BUFFERED_TUPLES を超える挿入は発生しません（バッファフル時にフラッシュされるため）。
  • 外部テーブルに対しては bistate が NULL となるため、これに依存する処理ではヌルチェックが必要です。

保守担当者向けメモ

  • 本構造体は copycontext メモリコンテキスト内で管理されることを想定しています。
  • パーティションテーブルの場合、各パーティションごとに ResultRelInfo が動的に割り当てられるため、バッファのライフサイクル管理には注意してください。
