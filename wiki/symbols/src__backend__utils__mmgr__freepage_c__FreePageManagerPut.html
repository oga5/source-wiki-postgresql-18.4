FreePageManagerPut

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/mmgr/freepage.c:378

目的

指定されたページ範囲をメモリ管理システムに解放し、必要に応じて隣接する空き領域とのマージおよび B-tree 構造の整理を行う。

シグネチャまたは形状

void FreePageManagerPut(FreePageManager *fpm, Size first_page, Size npages)

入力と出力

  • **入力**:
  • fpm: 管理対象の FreePageManager 構造体へのポインタ。
  • first_page: 解放を開始するページ番号。
  • npages: 解放するページ数。
  • **出力**: なし（副作用として fpm が更新される）。

前提条件と不変条件

  • npages > 0 であること。
  • FPM_EXTRA_ASSERTS が有効な場合、解放後の fpm->free_pages は sum_free_pages(fpm) と一致し、fpm->contiguous_pages は FreePageManagerLargestContiguous(fpm) と一致しなければならない。

アルゴリズムまたは振る舞い

1. FreePageManagerPutInternal を呼び出し、指定された範囲を管理対象に追加する。 2. 追加した範囲が既存の空き領域と隣接している場合、FreePageBtreeCleanup を呼び出して B-tree の整理を行う。 3. 管理対象の最大連続ページ数（fpm->contiguous_pages）を更新する。 4. FreePageManagerUpdateLargest を呼び出し、必要に応じて最大連続ページ数を再計算する。 5. デバッグビルド時（FPM_EXTRA_ASSERTS）は、整合性チェックを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/dsm.c
  • src/backend/utils/mmgr/dsa.c
  • **呼び出し先**:
  • FreePageManagerPutInternal
  • FreePageBtreeCleanup
  • FreePageManagerUpdateLargest
  • FreePageManagerLargestContiguous (デバッグ時のみ)

関連データ構造

  • FreePageManager: メモリ管理のメイン構造体。

失敗モードと境界条件

  • 管理対象領域が枯渇している場合、メタデータ用ページの確保に失敗する可能性がある（再帰的な制約）。
  • npages が 0 の場合は Assert により停止する。

保守担当者向けメモ

  • 本関数は、管理対象のメモリ領域自体をメタデータ保持に利用する設計であるため、メモリ不足時には注意が必要。
  • FPM_EXTRA_ASSERTS を有効にすると整合性チェックが実行されるが、パフォーマンスに影響を与えるため本番環境での利用には注意すること。
