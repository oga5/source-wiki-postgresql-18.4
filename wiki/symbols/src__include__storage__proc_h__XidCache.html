XidCache

定義

  • 種別: struct
  • 定義箇所: src/include/storage/proc.h:49

目的

サブトランザクションのトランザクションID（XID）をキャッシュし、プロセス単位で高速に参照可能にするためのデータ構造です。

シグネチャまたは形状

struct XidCache
{
    TransactionId xids[PGPROC_MAX_CACHED_SUBXIDS];
};

入力と出力

  • **入力**: TransactionId 型の配列としてサブトランザクションIDを保持します。
  • **出力**: プロセスが保持するサブトランザクションIDのリストを提供します。

前提条件と不変条件

  • PGPROC_MAX_CACHED_SUBXIDS で定義された固定長配列として実装されています。
  • プロセス制御構造体（PGPROC）の一部として埋め込まれ、共有メモリ上で管理されます。

アルゴリズムまたは振る舞い

  • サブトランザクションの発生時に、当該プロセスの XidCache に XID が格納されます。
  • ProcArray 等の処理において、特定のプロセスが実行中のサブトランザクションを効率的に特定するために参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/procarray.c において、キャッシュの計測（カウンタ更新）およびデバッグ出力（elog 等）のために参照されています。
  • **呼び出し先**: 直接的な関数呼び出し関係はソースから特定できません。

関連データ構造

  • PGPROC: XidCache をメンバとして保持するプロセス制御構造体。
  • TransactionId: キャッシュされる識別子の型。

失敗モードと境界条件

  • キャッシュ容量（PGPROC_MAX_CACHED_SUBXIDS）を超過した場合の挙動については、本構造体の定義からは不明です。

保守担当者向けメモ

  • src/backend/storage/ipc/procarray.c にて、このキャッシュに関連する計測用カウンタが定義されています。
  • PGPROC 構造体の一部として利用されるため、アクセス時には適切なロック（ProcArrayLock 等）の取得が必要となる場合があります。
