findDependencyLoops

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump_sort.c:758

目的

オブジェクトの依存関係グラフにおける循環参照を特定し、修復関数へ委譲することで、トポロジカルソートの失敗を解消する。

シグネチャまたは形状

static void findDependencyLoops(DumpableObject **objs, int nObjs, int totObjs)

入力と出力

  • **入力**:
  • objs: ソート対象の DumpableObject ポインタ配列。
  • nObjs: ソート対象オブジェクトの数。
  • totObjs: 全オブジェクト数（ワークスペース確保用）。
  • **出力**: なし（副作用として依存関係グラフを修復する）。

前提条件と不変条件

  • getMaxDumpId() が有効なダンプIDの最大値を返すこと。
  • objs 配列内の各オブジェクトは dumpId を保持していること。
  • 少なくとも1つの循環参照が発見され、修復される必要がある（修復されない場合は致命的エラーとなる）。

アルゴリズムまたは振る舞い

1. ダンプIDをインデックスとする processed（処理済みフラグ）、searchFailed（探索失敗キャッシュ）、および workspace（ループ保持用）を初期化する。 2. 各オブジェクトに対して findLoop を呼び出し、循環参照を探索する。 3. ループが検出された場合（looplen > 0）:

  • repairDependencyLoop を呼び出し、依存関係を修復する。
  • 修復されたループ内のオブジェクトを processed にマークする。

    • 4. ループが検出されなかった場合、当該オブジェクトを processed にマークする。 5. 処理終了後、ループが一つも修復されなかった場合は pg_fatal で異常終了する。 6. 最後に確保したメモリ領域を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump_sort.c:576（TopoSort の失敗後に呼び出される）。
  • **呼び出し先**:
  • findLoop: 循環参照の探索。
  • repairDependencyLoop: 循環参照の修復。
  • pg_malloc0, pg_malloc, free: メモリ管理。
  • getMaxDumpId: ダンプIDの最大値取得。
  • pg_fatal: 異常終了処理。

関連データ構造

  • DumpableObject: ダンプ対象のデータベースオブジェクト。
  • processed[]: 処理済みオブジェクトを追跡するブール配列。
  • searchFailed[]: 探索済みのパスをキャッシュし、計算量を削減する配列。
  • workspace[]: 検出されたループを一時的に格納する配列。

失敗モードと境界条件

  • **循環参照の未検出**: 循環参照が存在するにもかかわらず findLoop が検出に失敗した場合、fixedloop フラグが false のままとなり pg_fatal が発生する。
  • **メモリ不足**: pg_malloc 呼び出し時にメモリ不足が発生した場合、システム標準のメモリ確保エラーとなる。

保守担当者向けメモ

  • searchFailed 配列は、探索開始点ごとの再初期化を避けるため、開始点のダンプIDを値として保持する設計になっている。この最適化ロジックを変更する際は注意が必要。
  • 循環参照の修復ロジックは repairDependencyLoop に委譲されているため、特定のオブジェクトタイプ（ビューや制約など）固有の修復ルールを追加・修正する場合はそちらを参照すること。
