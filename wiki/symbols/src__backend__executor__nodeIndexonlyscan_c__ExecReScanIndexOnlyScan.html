ExecReScanIndexOnlyScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:363

目的

インデックスのみスキャン（Index-Only Scan）ノードの状態をリセットし、スキャンを最初から再実行できるように準備する。

シグネチャまたは形状

void ExecReScanIndexOnlyScan(IndexOnlyScanState *node)

入力と出力

  • **入力**: IndexOnlyScanState *node - 再スキャン対象となるインデックスのみスキャンノードの状態構造体。
  • **出力**: なし（副作用として node の内部状態が更新される）。

前提条件と不変条件

  • node は初期化済みであること。
  • ランタイムキーが存在する場合、すべてのランタイムキーが再計算されることを前提としている。

アルゴリズムまたは振る舞い

1. **ランタイムキーの再計算**: ioss_NumRuntimeKeys が 0 でない場合、ioss_RuntimeContext をリセットしてメモリリークを防ぎ、ExecIndexEvalRuntimeKeys を呼び出して新しいキー値を計算する。その後、ioss_RuntimeKeysReady を true に設定する。 2. **インデックススキャンのリセット**: ioss_ScanDesc が存在する場合、index_rescan を呼び出し、スキャンキーおよびオーダーバイキーを再設定する。 3. **スキャン状態の更新**: ExecScanReScan を呼び出し、共通のスキャン状態（ss）をリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:186 からの呼び出しが確認されている。
  • **呼び出し先**: ResetExprContext, ExecIndexEvalRuntimeKeys, index_rescan, ExecScanReScan を呼び出す。

関連データ構造

  • IndexOnlyScanState: インデックスのみスキャンの実行状態を保持する構造体。
  • ExprContext: ランタイムキー計算に使用される式コンテキスト。

失敗モードと境界条件

  • ioss_ScanDesc が NULL の場合、インデックススキャンのリセット処理はスキップされる。

保守担当者向けメモ

  • 本関数は IndexOnlyScan の再実行時に呼び出される。ランタイムキーの計算ロジックは、すべてのキーを毎回再計算する設計となっているため、変更時にはこの前提に注意すること。
