hash_stats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/hash/dynahash.c:884

目的

ハッシュテーブルの統計情報を標準エラー出力（stderr）に書き出すためのデバッグ用関数です。

シグネチャまたは形状

void hash_stats(const char *where, HTAB *hashp)

入力と出力

  • where: 出力メッセージの先頭に付与されるラベル文字列。
  • hashp: 統計情報を取得する対象のハッシュテーブル（HTAB）へのポインタ。
  • 出力: 戻り値なし。HASH_STATISTICS マクロが定義されている場合に限り、stderr へ統計情報を出力します。

前提条件と不変条件

  • hashp は有効な HTAB 構造体を指している必要があります。
  • コンパイル時に HASH_STATISTICS マクロが定義されていない場合、この関数は何も行いません。

アルゴリズムまたは振る舞い

1. HASH_STATISTICS が定義されている場合、以下の情報を stderr に出力します。

  • 指定されたラベル where に関連付けられた、対象ハッシュテーブルのアクセス数と衝突数。
  • 対象ハッシュテーブルの現在のエントリ数（hash_get_num_entries を使用）、キーサイズ、最大バケット数、セグメント数。
  • システム全体での総アクセス数、総衝突数、および総拡張回数。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/hash/dynahash.c 内で参照されています。
  • 呼び出し先: hash_get_num_entries を呼び出します。

関連データ構造

  • HTAB: ハッシュテーブルの制御構造体。
  • HASHHDR: HTAB 内で統計情報（アクセス数、衝突数、キーサイズ等）を保持する共有ヘッダー構造体。

失敗モードと境界条件

  • hashp が NULL の場合、hashp->hctl へのアクセスによりセグメンテーション違反が発生する可能性があります。
  • HASH_STATISTICS が定義されていない環境では、呼び出しても副作用は発生しません。

保守担当者向けメモ

  • 本関数はデバッグおよびパフォーマンス分析専用です。本番環境での利用を想定したものではありません。
  • hash_accesses、hash_collisions、hash_expansions はグローバル変数として定義されていると推測されます。
