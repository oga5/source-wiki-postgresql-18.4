bbsink_server_end_manifest

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_server.c:286

目的

バックアップマニフェストファイルの書き込み処理を終了し、一時ファイルから正式なファイル名へアトミックにリネームすることで、バックアップの完了を確定させる。

シグネチャまたは形状

static void bbsink_server_end_manifest(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - サーバー保存用シンクのインスタンス。
  • **出力**: なし（副作用としてファイルシステム操作を実行）。

前提条件と不変条件

  • sink は bbsink_server 型としてキャスト可能であること。
  • mysink->file が有効なファイル記述子を保持していること。
  • backup_manifest.tmp が指定されたパスに存在すること。

アルゴリズムまたは振る舞い

1. FileClose を呼び出し、現在開いている一時ファイル（backup_manifest.tmp）をクローズする。 2. mysink->file を 0 にリセットする。 3. psprintf を使用して一時ファイル名と確定後のファイル名（backup_manifest）のパスを生成する。 4. durable_rename を実行し、一時ファイルを正式なファイル名へリネームする。この際、内部的に fsync が行われる。 5. pfree でパス文字列のメモリを解放する。 6. bbsink_forward_end_manifest(sink) を呼び出し、後続のシンクへ処理を転送する。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_server_ops 構造体の .end_manifest フィールドを通じて呼び出される（直接的な呼び出し元はバックアップのメインフローに依存）。
  • **呼び出し先**:
  • FileClose
  • psprintf
  • durable_rename
  • pfree
  • bbsink_forward_end_manifest

関連データ構造

  • bbsink_server: ローカルファイル保存用の状態を保持する構造体。
  • bbsink: 基底インターフェース構造体。

失敗モードと境界条件

  • durable_rename が失敗した場合、ERROR レベルで処理が中断される。
  • fsync 失敗時はリカバリ不能な状態を避けるため、PANIC ではなく ERROR が発生する。

保守担当者向けメモ

  • 本関数は durable_rename を使用しているため、マニフェストファイルが存在すればバックアップが正常に完了したと判断できる設計になっている。
  • durable_rename 内部で fsync が行われるため、本関数内で明示的な FileSync は不要である。
