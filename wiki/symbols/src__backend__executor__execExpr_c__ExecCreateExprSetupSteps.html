ExecCreateExprSetupSteps

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExpr.c:2882

目的

式評価に必要なセットアップステップ（タプルスロットのデフォーム等）を特定し、ExprState に追加する。

シグネチャまたは形状

static void ExecCreateExprSetupSteps(ExprState *state, Node *node)

入力と出力

  • state: 構築対象の ExprState 構造体へのポインタ。
  • node: 評価対象の式ツリー（Node）。
  • 出力: なし（state 構造体内の命令配列が更新される）。

前提条件と不変条件

  • state は初期化済みであること。
  • node は有効な式ツリーであること。

アルゴリズムまたは振る舞い

1. ExprSetupInfo 構造体をゼロ初期化し、必要な情報を収集するための準備を行う。 2. expr_setup_walker を呼び出し、式ツリーを走査して評価に必要なセットアップ要件（タプルデフォーム等）を info に記録する。 3. ExecPushExprSetupSteps を呼び出し、収集された info に基づいて state に具体的な命令ステップを追加する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execExpr.c 内の複数の初期化関数（ExecInitExpr, ExecInitQual, ExecInitCheck, ExecInitExprList, ExecPrepareCheck など）。
  • 呼び出し先: expr_setup_walker（走査用）、ExecPushExprSetupSteps（ステップ追加用）。

関連データ構造

  • ExprState: 式評価の状態を保持するメイン構造体。
  • ExprSetupInfo: 式の評価に必要なタプルスロットのデフォーム情報等を保持する構造体。

失敗モードと境界条件

  • 式ツリーが極端に深い場合、expr_setup_walker の再帰呼び出しによりスタックオーバーフローが発生する可能性がある（check_stack_depth() による保護が前提）。

保守担当者向けメモ

  • 本関数は式評価の「コンパイル」フェーズの一部であり、実行時のオーバーヘッドを削減するための前処理を行う。
  • ExprSetupInfo の内容を変更する場合は、expr_setup_walker と ExecPushExprSetupSteps の両方の整合性を確認すること。
