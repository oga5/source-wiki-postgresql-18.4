relation_close

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/relation.c:204

目的

リレーション記述子を解放し、必要に応じて関連するロックを解除します。

シグネチャまたは形状

void relation_close(Relation relation, LOCKMODE lockmode)

入力と出力

  • relation: 解放対象のリレーション記述子。
  • lockmode: 解放するロックのモード。NoLock を指定した場合、ロックは解除されません。

前提条件と不変条件

  • lockmode は NoLock 以上かつ MAX_LOCKMODES 未満である必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. 指定された relation からロック識別子（LockRelId）を取得します。 2. RelationClose(relation) を呼び出し、リレーションキャッシュの参照カウントを減らします。 3. lockmode が NoLock でない場合、UnlockRelationId を呼び出して指定されたモードのロックを解除します。

呼び出し元と呼び出し先

  • **呼び出し先**: RelationClose（リレーションキャッシュの解放）、UnlockRelationId（ロックの解除）。
  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • contrib/dblink/dblink.c
  • contrib/pageinspect/btreefuncs.c

    • （上記はツールによる直接的な参照の証拠です）

関連データ構造

  • Relation: リレーション記述子。
  • LOCKMODE: ロックの強度を指定する列挙型。
  • LockRelId: ロック対象を特定するための識別子。

失敗モードと境界条件

  • lockmode に不正な値が渡された場合、アサーションエラーが発生します。
  • relation が無効なポインタである場合、RelationClose 内部で未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はロックの自動解除を行いません。lockmode を指定しない場合、ロックはトランザクション終了時まで保持されます。
  • NoLock を指定して呼び出す場合、呼び出し側が適切なロックを保持していることを保証する必要があります。
