hash_xlog_split_complete

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash_xlog.c:441

目的

ハッシュインデックスの分割（Split）処理完了時に記録されたWALレコードをリプレイし、旧バケットおよび新バケットのページフラグを更新して整合性を復元する。

シグネチャまたは形状

static void hash_xlog_split_complete(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファの内容を更新）。

前提条件と不変条件

  • record には xl_hash_split_complete 構造体が含まれている必要がある。
  • リプレイ対象のバッファ（旧バケットおよび新バケット）が正常に読み込めること。
  • ページフラグの更新は、フルページイメージから復元された場合であっても実行される（フラグがイメージに含まれないため）。

アルゴリズムまたは振る舞い

1. WALレコードから xl_hash_split_complete データを取得する。 2. **旧バケットの処理**:

  • XLogReadBufferForRedo を呼び出し、旧バケットのページを取得する。
  • ページが更新対象（BLK_NEEDS_REDO または BLK_RESTORED）であれば、hasho_flag を xlrec->old_bucket_flag に更新する。
  • ページにLSNを書き込み、バッファをダーティにする。
  • バッファを解放する。

    • 3. **新バケットの処理**:

  • 同様に XLogReadBufferForRedo で新バケットのページを取得する。
  • ページが更新対象であれば、hasho_flag を xlrec->new_bucket_flag に更新する。
  • ページにLSNを書き込み、バッファをダーティにする。
  • バッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: hash_redo（src/backend/access/hash/hash_xlog.c:1092 付近で、WALレコード種別に基づきディスパッチされる）。
  • **呼び出し先**:
  • XLogRecGetData
  • XLogReadBufferForRedo
  • BufferGetPage
  • HashPageGetOpaque
  • PageSetLSN
  • MarkBufferDirty
  • BufferIsValid
  • UnlockReleaseBuffer

関連データ構造

  • xl_hash_split_complete: 分割完了時のフラグ情報を含むWALレコード構造体。
  • HashPageOpaque: ハッシュインデックスページのメタデータ構造体（hasho_flag を保持）。

失敗モードと境界条件

  • XLogReadBufferForRedo がバッファを取得できない場合、その後の処理はスキップされる。
  • ページがフルページイメージとして復元された場合でも、フラグの更新は必須であるため、条件分岐により適切に処理される。

保守担当者向けメモ

  • 本関数は冪等である。既に適用済みの変更に対して再度実行されても、ページフラグが同じ値で上書きされるだけであり、副作用はない。
  • old_bucket_flag および new_bucket_flag は、分割後のバケット状態を正しく反映させるために不可欠である。
