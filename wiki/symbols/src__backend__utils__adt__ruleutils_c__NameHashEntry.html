NameHashEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/ruleutils.c:318

目的

クエリの逆コンパイル（deparse）処理において、生成されるエイリアス名の一意性を管理するためのハッシュテーブル用エントリ構造体です。

シグネチャまたは形状

typedef struct {
    char        name[NAMEDATALEN]; /* ハッシュキー --- 構造体の先頭である必要がある */
    int         counter;           /* その名前に対して使用された最大の追加値 */
} NameHashEntry;

入力と出力

  • **入力**: ハッシュテーブルのキーとして name を使用します。
  • **出力**: counter フィールドにより、同一名が複数回出現した際に付与すべき連番を保持します。

前提条件と不変条件

  • name フィールドは構造体の先頭に配置されている必要があります（ハッシュテーブルのキーとして機能するため）。
  • counter は、特定の名前が重複した際に、次に割り当てるべき一意な識別子を生成するために使用されます。

アルゴリズムまたは振る舞い

1. hash_search を使用して、指定された名前の NameHashEntry を検索または作成します。 2. 同一の名前が既に存在する場合、counter を参照して新しいエイリアス名を生成します。 3. 新しい名前が生成されるたびに、対応するエントリの counter が更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/ruleutils.c 内のエイリアス生成ロジック（hash_search を介して呼び出されます）。
  • **呼び出し先**: PostgreSQL のハッシュテーブル管理関数 hash_search を利用します。

関連データ構造

  • names_hash: NameHashEntry を格納するために使用されるハッシュテーブル。

失敗モードと境界条件

  • name の長さが NAMEDATALEN を超えることはできません。
  • ハッシュテーブルのメモリ不足時には hash_search がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本構造体は ruleutils.c 内のローカルな名前空間管理に特化しています。
  • hash_ctl.entrysize に sizeof(NameHashEntry) を設定してハッシュテーブルを初期化する箇所と整合性を保つ必要があります。
