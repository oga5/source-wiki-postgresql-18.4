rangeTableEntry_used_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteManip.c:996

目的

クエリツリーの走査中に、特定の範囲テーブルエントリ（RTE）が使用されているかどうかを追跡するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	int			rt_index;
	int			sublevels_up;
} rangeTableEntry_used_context;

入力と出力

  • rt_index: 検索対象とする範囲テーブルのインデックス。
  • sublevels_up: 現在の走査位置におけるネストレベル（サブクエリの深さ）。

前提条件と不変条件

  • sublevels_up は、ツリー走査の過程でサブクエリノードを通過するたびに適切にインクリメントおよびデクリメントされる必要があります。

アルゴリズムまたは振る舞い

本構造体は、expression_tree_walker 等を用いたツリー走査において、特定の rt_index を持つ Var ノードが現在のネストレベル（sublevels_up）で参照されているかを判定するための状態保持に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/rewrite/rewriteManip.c:1071 にてインスタンス化が確認されています。
  • 呼び出し先: 構造体定義のため、関数呼び出しの対象ではありません。

関連データ構造

  • Var: クエリツリー内の変数参照ノード。
  • Query: 走査対象となるクエリツリーのルートノード。

失敗モードと境界条件

  • 構造体自体の失敗モードはありません。
  • 走査ロジックにおいて sublevels_up の管理が不正確な場合、誤ったネストレベルの Var を検出する可能性があります。

保守担当者向けメモ

  • 本構造体は rewriteManip.c 内の特定の走査関数で使用される補助的なデータ構造です。
  • 走査ロジックを拡張する際は、sublevels_up の整合性が保たれていることを確認してください。
