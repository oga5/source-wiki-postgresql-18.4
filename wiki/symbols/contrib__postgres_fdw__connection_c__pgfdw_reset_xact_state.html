pgfdw_reset_xact_state

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/connection.c:1372

目的

リモートサーバーとの接続状態（ConnCacheEntry）におけるトランザクション深度をリセットし、必要に応じて接続の破棄を行うことで、ローカルのトランザクション終了とリモート側の状態を同期させる。

シグネチャまたは形状

static void pgfdw_reset_xact_state(ConnCacheEntry *entry, bool toplevel)

入力と出力

  • entry: 更新対象の接続キャッシュエントリへのポインタ。
  • toplevel: 現在のトランザクションがトップレベルであるかを示すフラグ。true の場合はトランザクション全体、false の場合はサブトランザクションの終了を意味する。
  • 戻り値: なし。

前提条件と不変条件

  • entry は有効な ConnCacheEntry を指している必要がある。
  • toplevel が true の場合、entry->xact_depth は 0 にリセットされる。
  • toplevel が false の場合、entry->xact_depth はデクリメントされる。

アルゴリズムまたは振る舞い

1. toplevel が true の場合:

  • entry->xact_depth を 0 に設定する。
  • 以下のいずれかの条件に該当する場合、接続が正常なアイドル状態ではないと判断し、disconnect_pg_server(entry) を呼び出して接続を破棄する。
  • PQstatus(entry->conn) が CONNECTION_OK ではない。
  • PQtransactionStatus(entry->conn) が PQTRANS_IDLE ではない。
  • entry->changing_xact_state が真である。
  • entry->invalidated が真である。
  • entry->keep_connections が偽である。

    • 2. toplevel が false の場合:

  • entry->xact_depth を 1 減算する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/postgres_fdw/connection.c 内の複数の箇所（pgfdw_xact_callback や pgfdw_subxact_callback 等のトランザクションイベントフックに関連する箇所）から呼び出される。
  • 呼び出し先: disconnect_pg_server（条件合致時）、PQstatus、PQtransactionStatus（libpq 関数）。

関連データ構造

  • ConnCacheEntry: 接続状態、トランザクション深度、接続の有効性フラグを保持する構造体。

失敗モードと境界条件

  • entry->conn が既に閉じられている、または無効な状態である場合、PQstatus 等の libpq 関数がエラーを返す可能性がある。
  • xact_depth が 0 の状態で toplevel が false として呼び出された場合の挙動については、ソースコード上での明示的なガードは確認できない。

保守担当者向けメモ

  • 本関数は、ローカルのトランザクション終了時にリモート接続をクリーンな状態に戻すための重要なフックである。
  • 接続破棄の条件（PQtransactionStatus 等）は、リモートサーバーとの通信整合性を維持するために厳密に管理されている。変更時は postgres_fdw のトランザクション管理ロジックへの影響を考慮すること。
