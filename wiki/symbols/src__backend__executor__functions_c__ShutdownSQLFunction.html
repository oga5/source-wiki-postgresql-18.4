ShutdownSQLFunction

定義

  • 種別: function
  • 定義箇所: src/backend/executor/functions.c:1966

目的

SQL関数の実行状態（SQLFunctionCache）を安全に終了させ、関連するリソースを解放する。

シグネチャまたは形状

static void ShutdownSQLFunction(Datum arg)
  • arg: SQLFunctionCachePtr を Datum 型にキャストしたもの。

入力と出力

  • **入力**: arg（SQLFunctionCache へのポインタ）。
  • **出力**: なし（副作用としてリソース解放を行う）。

前提条件と不変条件

  • arg は有効な SQLFunctionCache 構造体を指している必要がある。
  • 呼び出し後、fcache->eslist、fcache->tstore、fcache->cplan は NULL にリセットされる。
  • fcache->shutdown_reg は false に設定される。

アルゴリズムまたは振る舞い

1. fcache->eslist を辿り、実行中のクエリ（F_EXEC_RUN 状態）に対して postquel_end を呼び出して終了処理を行う。

  • 非読み取り専用関数の場合、postquel_end の前後でアクティブスナップショットのプッシュ/ポップを行う。

    • 2. fcache->tstore が存在する場合、tuplestore_end で解放する。 3. fcache->cplan が存在する場合、ReleaseCachedPlan でプランキャッシュを解放する。 4. fcache->shutdown_reg を false に更新し、登録解除済みであることを示す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照箇所から、SQL関数の実行終了時やエラー発生時のクリーンアップ処理として呼び出されることが示唆されている。
  • **呼び出し先**: postquel_end、PushActiveSnapshot、PopActiveSnapshot、tuplestore_end、ReleaseCachedPlan。

関連データ構造

  • SQLFunctionCache: 関数実行状態を保持する主要構造体。
  • execution_state: 関数内の各クエリの実行状態を管理するリスト。

失敗モードと境界条件

  • fcache が NULL の場合、あるいは fcache 内の各ポインタが既に解放済み（NULL）である場合は、各条件分岐により安全にスキップされる。

保守担当者向けメモ

  • 本関数は execUtils によってコールバックとして登録されることを想定している。
  • 実行中のクエリを終了させる際、スナップショットの管理が正しく行われているか注意すること。
