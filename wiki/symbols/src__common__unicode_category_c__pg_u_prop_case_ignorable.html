pg_u_prop_case_ignorable

定義

  • src/common/unicode_category.c:158

目的

指定された Unicode コードポイントが、大文字・小文字の変換において無視されるべき文字（Case-ignorable）であるかを判定します。

シグネチャまたは形状

bool pg_u_prop_case_ignorable(pg_wchar code)

入力と出力

  • **入力**: pg_wchar code（判定対象の Unicode コードポイント）
  • **出力**: bool（該当する場合は true、それ以外は false）

前提条件と不変条件

  • 入力コードポイントは Unicode 規格に基づきます。
  • 内部テーブル（unicode_opt_ascii および unicode_case_ignorable）は固定されており、実行時に変更されません。

アルゴリズムまたは振る舞い

1. **ASCII 最適化**: code が 0x80 未満の場合、unicode_opt_ascii テーブルを直接参照し、PG_U_PROP_CASE_IGNORABLE ビットマスクを用いて判定結果を返します。 2. **範囲検索**: code が 0x80 以上の場合、unicode_case_ignorable テーブルに対して range_search を実行し、該当する範囲内にコードポイントが含まれるかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode/category_test.c
  • src/common/unicode_case.c
  • **呼び出し先**:
  • range_search（src/common/unicode_category.c 内のユーティリティ）

関連データ構造

  • unicode_opt_ascii: ASCII 文字用のプロパティテーブル。
  • unicode_case_ignorable: 非 ASCII 文字用のプロパティ範囲テーブル。

失敗モードと境界条件

  • code が有効な Unicode コードポイント範囲外である場合の挙動は、range_search の実装に依存します。

保守担当者向けメモ

  • 本関数は Unicode 規格の安定性ポリシーに基づいています。プロパティ定義の変更が必要な場合は、unicode_category_table.h を生成するスクリプトを確認してください。
