JsonPathCountVarsCallback

定義

  • 種別: typedef
  • 定義箇所: src/backend/utils/adt/jsonpath_exec.c:91

目的

JSON パス実行コンテキストにおいて、クエリ内で使用される変数の数をカウントするためのコールバック関数型です。

シグネチャまたは形状

typedef int (*JsonPathCountVarsCallback) (void *vars);

入力と出力

  • **入力**: void *vars - 変数リストまたは変数管理構造体へのポインタ。
  • **出力**: int - 変数の総数。

前提条件と不変条件

  • vars ポインタは、呼び出し元で定義された特定の変数管理構造体を指している必要があります。
  • 返り値は、指定されたコンテキスト内で有効な変数の個数と一致しなければなりません。

アルゴリズムまたは振る舞い

  • この関数ポインタは、JsonPathExecContext 構造体の一部として保持されます。
  • 実行エンジンが変数の数を特定する必要がある際に呼び出されます。具体的なカウントロジックは、この型を実装する関数に委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonpath_exec.c 内の関数（行 259, 678, 3198 付近）で、JsonPathExecContext の初期化や変数処理に関連して参照されています。
  • **呼び出し先**: この型を実装する具体的な関数が呼び出されます（ソースコード上での具体的な実装関数名は本抜粋からは特定できません）。

関連データ構造

  • JsonPathExecContext: このコールバックをメンバとして保持する実行状態管理構造体。

失敗モードと境界条件

  • vars が NULL の場合にコールバックがどのように振る舞うかは、実装側の定義に依存します。
  • 変数リストが空の場合、0 を返すことが期待されます。

保守担当者向けメモ

  • 本シンボルは関数ポインタ型であり、実際のロジックは代入される関数側に存在します。
  • JsonPathExecContext.vars の定義に関連して使用されるため、変数の動的な解決が必要な場合にこのコールバックが適切に設定されているか確認してください。
