guc_realloc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc.c:651

目的

GUCMemoryContext 内でのメモリ再割り当て（または新規割り当て）を行い、メモリ不足時には指定されたエラーレベルで報告を行うためのラッパー関数です。

シグネチャまたは形状

void *guc_realloc(int elevel, void *old, size_t size)

入力と出力

  • **入力**:
  • elevel: メモリ不足時に使用するエラーレベル（ereport 用）。
  • old: 再割り当て対象の既存メモリポインタ（NULL の場合は新規割り当てとして動作）。
  • size: 要求するメモリサイズ（バイト単位）。
  • **出力**:
  • 成功時: 割り当てられたメモリ領域へのポインタ。
  • 失敗時: elevel が FATAL 等であればプロセス終了、それ以外であればエラー報告後に制御が戻る可能性がある（data == NULL の場合）。

前提条件と不変条件

  • old が NULL でない場合、そのメモリブロックは GUCMemoryContext に属している必要がある（Assert による検証）。
  • 割り当て処理には MCXT_ALLOC_NO_OOM フラグが使用されるため、標準的な repalloc とは異なり、即座にエラー終了せず NULL を返す可能性がある。

アルゴリズムまたは振る舞い

1. old が NULL でない場合、repalloc_extended を用いて既存領域を拡張する。 2. old が NULL の場合、MemoryContextAllocExtended を用いて新規にメモリを割り当てる。 3. いずれの場合も GUCMemoryContext を対象とする。 4. 割り当て結果が NULL の場合、elevel を用いて ereport を発行し、メモリ不足エラーを通知する。 5. 成功したポインタを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc.c 内の動的なメモリ拡張が必要な箇所（例: 5734 行目付近）。
  • **呼び出し先**:
  • repalloc_extended
  • MemoryContextAllocExtended
  • ereport

関連データ構造

  • GUCMemoryContext: GUC システム専用のメモリコンテキスト。

失敗モードと境界条件

  • old が GUCMemoryContext 以外のメモリ領域を指している場合、Assert が有効なビルドではアボートする。
  • メモリ割り当てが失敗した場合、ereport が呼び出される。elevel の設定値によっては、呼び出し元へ制御が戻らずに終了する。

保守担当者向けメモ

  • 本関数は realloc(3) に似た挙動をしますが、old == NULL のケースを許容する点が異なります。
  • repalloc とは異なり、メモリ不足時に即座にエラー終了せず NULL を返す可能性があるため、呼び出し元で戻り値のチェックが必要です。
