gbt_tseq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_ts.c:67

目的

2つの Timestamp 値が等しいかどうかを判定し、GiSTインデックスの比較処理をサポートします。

シグネチャまたは形状

static bool gbt_tseq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Timestamp 型へのポインタ。
  • flinfo: 関数呼び出し情報（使用されません）。
  • **出力**:
  • bool: a と b が等しい場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は有効な Timestamp 型のメモリ領域を指している必要があります。
  • 内部的に timestamp_eq を呼び出すため、PostgreSQLの標準的なタイムスタンプ比較ルールに従います。

アルゴリズムまたは振る舞い

1. 入力された void * ポインタを Timestamp * にキャストします。 2. TimestampGetDatumFast マクロを使用して、各値を Datum 形式に変換します。 3. DirectFunctionCall2 を介して PostgreSQL のコア関数 timestamp_eq を実行し、その結果を bool に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_ts.c 内の静的データ構造（tinfo 等の初期化時）から参照されています。
  • **呼び出し先**: timestamp_eq（PostgreSQLコア関数）。

関連データ構造

  • Timestamp: PostgreSQLのタイムスタンプ型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。
  • timestamp_eq が内部的にエラーを発生させる場合（無効なタイムスタンプ値など）、その例外が伝播します。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける数値比較ユーティリティの一部として機能します。
  • timestamptz 型を扱う場合も、本関数は内部的にGMTへ正規化された timestamp 値を前提として動作します。
