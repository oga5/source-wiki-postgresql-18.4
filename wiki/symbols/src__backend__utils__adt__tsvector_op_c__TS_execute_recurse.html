TS_execute_recurse

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:1882

目的

全文検索クエリ（tsquery）のツリー構造を再帰的に走査し、tsvector に対してマッチング評価を行う。

シグネチャまたは形状

static TSTernaryValue TS_execute_recurse(
    QueryItem *curitem, 
    void *arg, 
    uint32 flags, 
    TSExecuteCallback chkcond
)

入力と出力

  • **入力**:
  • curitem: 評価対象の QueryItem（クエリツリーのノード）。
  • arg: コールバック関数に渡すコンテキストデータ。
  • flags: 実行オプション（TS_EXEC_SKIP_NOT や TS_EXEC_PHRASE_NO_POS など）。
  • chkcond: 語彙の照合を行うコールバック関数。
  • **出力**:
  • TSTernaryValue 型（TS_YES, TS_NO, TS_MAYBE のいずれか）を返す。

前提条件と不変条件

  • 再帰呼び出しを行うため、スタックオーバーフローを防ぐための check_stack_depth() が呼び出しの冒頭で実行される。
  • 割り込みチェックのため CHECK_FOR_INTERRUPTS() が実行される。

アルゴリズムまたは振る舞い

クエリツリーのノードタイプに応じて以下の処理を行う：

  • **QI_VAL**: chkcond コールバックを呼び出し、語彙の一致を判定する。
  • **OP_NOT**: 子ノードを評価し、結果を反転させる（TS_EXEC_SKIP_NOT フラグがある場合は TS_YES を返す）。
  • **OP_AND**: 左ノードを評価し、TS_NO であれば即座に TS_NO を返す。そうでなければ右ノードを評価し、論理積の結果を返す。
  • **OP_OR**: 左ノードを評価し、TS_YES であれば即座に TS_YES を返す。そうでなければ右ノードを評価し、論理和の結果を返す。
  • **OP_PHRASE**: TS_phrase_execute を呼び出し、フレーズ検索を実行する。TS_MAYBE が返された場合、TS_EXEC_PHRASE_NO_POS フラグの有無に応じて TS_NO または TS_MAYBE に変換する。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の TS_execute および TS_execute_recurse 自身による再帰呼び出し。
  • **呼び出し先**: chkcond コールバック、TS_phrase_execute。

関連データ構造

  • QueryItem: tsquery の各要素（演算子またはオペランド）。
  • TSTernaryValue: 3値論理（真・偽・不明）を表現する列挙型。
  • TSExecuteCallback: 語彙照合用の関数ポインタ型。

失敗モードと境界条件

  • 未知の演算子（default ケース）に遭遇した場合、elog(ERROR) によりエラーを発生させる。
  • 再帰の深さがスタック制限を超えた場合、check_stack_depth() によりエラーが発生する。

保守担当者向けメモ

  • OP_PHRASE の TS_MAYBE 変換ロジックは、過去の古い実装との互換性を維持するために現在の形式となっている。
  • 本関数はクエリツリーを再帰的に処理するため、複雑なクエリに対するスタック消費量に注意が必要である。
