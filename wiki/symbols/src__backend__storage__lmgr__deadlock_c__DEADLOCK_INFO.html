DEADLOCK_INFO

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/lmgr/deadlock.c:72

目的

デッドロック発生時に、エラーレポートやログ出力へ詳細情報を記録するためのメタデータ構造体です。

シグネチャまたは形状

typedef struct
{
	LOCKTAG		locktag;		/* ID of awaited lock object */
	LOCKMODE	lockmode;		/* type of lock we're waiting for */
	int			pid;			/* PID of blocked backend */
} DEADLOCK_INFO;

入力と出力

  • 本構造体は、デッドロック検出プロセスにおいて、ロック待ち状態にある各バックエンドの情報を保持するために使用されます。
  • 出力先として、デッドロック発生時のエラーメッセージ生成プロセスに利用されます。

前提条件と不変条件

  • deadlockDetails 配列として MaxBackends 分の領域が確保されている必要があります。
  • 各フィールドは、ロック待ちグラフの探索中に特定された、競合しているロックとプロセスの状態を反映します。

アルゴリズムまたは振る舞い

  • デッドロック検出アルゴリズムの実行中、循環参照に関与する各プロセスの情報を deadlockDetails 配列に格納します。
  • 検出されたデッドロックの報告時、この構造体を通じてどのプロセスがどのロックを待機しているかの詳細が収集されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/deadlock.c 内のデッドロック検出および報告ロジック（DeadLockCheckRecurse や DeadLockReport 等）から参照されます。
  • 呼び出し先: 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • LOCKTAG: 待機対象のロックを識別するタグ。
  • LOCKMODE: 待機しているロックのモード。
  • deadlockDetails: 本構造体の配列を保持する静的変数。

失敗モードと境界条件

  • MaxBackends を超えるプロセス数に対しては、配列の境界外アクセスが発生しないよう、deadlockDetails の初期化サイズに依存します。

保守担当者向けメモ

  • 本構造体は deadlock.c 内の静的配列 deadlockDetails を通じて管理されます。
  • メモリ割り当ては InitDeadLockChecking で行われるため、構造体の変更時は初期化ロジックとの整合性に注意してください。
