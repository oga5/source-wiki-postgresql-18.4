group_keys_reorder_by_pathkeys

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/pathkeys.c:369

目的

入力パスの順序（pathkeys）と GROUP BY 句の順序を照合し、一致するキーをリストの先頭に並べ替えることで、グループ化処理の効率化を図る。

シグネチャまたは形状

int group_keys_reorder_by_pathkeys(List *pathkeys, List **group_pathkeys, List **group_clauses, int num_groupby_pathkeys)

入力と出力

  • **入力**:
  • pathkeys: 入力パスが提供する順序リスト。
  • group_pathkeys: GROUP BY 句に関連するパスキーのリスト（ポインタ経由で更新）。
  • group_clauses: GROUP BY 句のリスト（ポインタ経由で更新）。
  • num_groupby_pathkeys: 処理対象とする group_pathkeys の先頭からの要素数。
  • **出力**:
  • 戻り値 (int): 一致したグループ化キーの数。
  • group_pathkeys, group_clauses: 一致した要素が先頭に移動され、残りの要素が後ろに連結されたリストに更新される。

前提条件と不変条件

  • pathkeys および *group_pathkeys が NIL でないこと。
  • pathkey->pk_eclass->ec_sortref が有効（0以外）であること。
  • sgc->sortop が有効な OID であること（Assert による検証）。

アルゴリズムまたは振る舞い

1. group_pathkeys から num_groupby_pathkeys 分のリストをコピーし、検索用の一時リストを作成する。 2. pathkeys を順に走査し、以下の条件をすべて満たす場合に一致とみなす：

  • 現在のインデックスが num_groupby_pathkeys 以内である。
  • pathkey が一時リストに含まれる。
  • pathkey に対応する SortGroupClause が group_clauses 内に存在する。

    • 3. 一致した pathkey と SortGroupClause を新しいリストに追加する。 4. 一致しないキーが出現した時点で走査を中断する。 5. 最後に、一致したリストと元のリストを list_concat_unique_ptr で結合し、ポインタを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/pathkeys.c:506 (文脈上の直接的な呼び出し)
  • **呼び出し先**:
  • list_copy_head, list_length, lappend, list_concat_unique_ptr, list_free (リスト操作)
  • get_sortgroupref_clause_noerr (グループ化句の取得)
  • list_member_ptr (ポインタ比較による検索)

関連データ構造

  • PathKey: 順序を表現する構造体。
  • SortGroupClause: GROUP BY 句の定義を保持する構造体。
  • EquivalenceClass: PathKey が参照する等価クラス。

失敗モードと境界条件

  • pathkeys または *group_pathkeys が NIL の場合、何もせず 0 を返す。
  • pathkey が grouping_pathkeys に含まれない、または ec_sortref が 0 の場合、その時点で走査を中断する。
  • get_sortgroupref_clause_noerr が NULL を返した場合、グループ化句がパスキーをカバーしていないと判断し、走査を中断する。

保守担当者向けメモ

  • 本関数は、集約関数を含むパスキーとグループ化キーを分離して処理するために、num_groupby_pathkeys を用いて検索範囲を制限している。
  • list_concat_unique_ptr を使用しているため、リストの結合時に重複が排除される点に注意すること。
