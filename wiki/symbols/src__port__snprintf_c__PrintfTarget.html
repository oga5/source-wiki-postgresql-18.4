PrintfTarget

定義

  • 種別: struct
  • 定義箇所: src/port/snprintf.c:125

目的

PostgreSQL の printf ファミリー実装において、出力先（メモリバッファまたはストリーム）と現在の処理状態を一元管理するために使用される構造体です。

シグネチャまたは形状

typedef struct {
    char       *bufptr;     /* 次のバッファ出力位置 */
    char       *bufstart;   /* バッファの開始位置 */
    char       *bufend;     /* バッファの終端（NULL の場合はバッファ十分とみなす） */
    FILE       *stream;     /* 出力先ストリーム（NULL の場合はメモリ出力） */
    int         nchars;     /* 送信済みまたは破棄された文字数 */
    bool        failed;     /* 失敗フラグ（errno が設定される） */
} PrintfTarget;

入力と出力

  • 入力: dopr() 等の内部関数において、フォーマット処理のターゲットとして参照されます。
  • 出力: 変換された文字列が bufptr を通じてメモリに書き込まれるか、stream を通じてファイルへ出力されます。

前提条件と不変条件

  • bufend == NULL の場合、sprintf のようにバッファサイズが十分であると仮定して動作します。
  • stream が指定されている場合、バッファが一杯になると flushbuffer() が呼び出され、内容がフラッシュされます。

アルゴリズムまたは振る舞い

1. **初期化**: 各ラッパー関数（pg_snprintf 等）でインスタンス化されます。 2. **書き込み**: dopr_outch() を介して文字が書き込まれ、nchars が更新されます。 3. **バッファ管理**: ストリーム出力時、バッファの限界に達すると flushbuffer() が実行され、fwrite を用いて出力されます。 4. **エラー処理**: 処理中にエラーが発生した場合、failed フラグが true に設定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_vsnprintf, pg_snprintf, pg_printf 等のラッパー関数。
  • **呼び出し先**: flushbuffer(), dopr(), fmtptr(), fmtchar() 等の内部関数が、この構造体を引数として受け取ります。

関連データ構造

  • PrintfArgValue: %n$ 指定子による引数解析時に使用されるデータ構造。

失敗モードと境界条件

  • failed メンバが true の場合、処理は失敗とみなされます。
  • bufend が NULL の場合、バッファオーバーランのチェックが行われないため、呼び出し側で十分な領域を確保する必要があります。

保守担当者向けメモ

  • 本構造体は src/port/snprintf.c 内でのみ使用される内部的な管理構造体です。
  • 浮動小数点数の出力補正など、プラットフォーム間の差異を吸収する処理において、この構造体の状態が正しく維持されることが重要です。
