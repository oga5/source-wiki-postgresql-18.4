char_increment

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:575

目的

B-tree インデックスのスキップスキャンにおいて、char 型の値をインクリメントし、次の走査対象値を算出する。

シグネチャまたは形状

static Datum char_increment(Relation rel, Datum existing, bool *overflow)

入力と出力

  • **入力**:
  • rel: 対象となるリレーション（未使用）。
  • existing: インクリメント対象の char 型値（Datum）。
  • overflow: オーバーフロー状態を格納するフラグへのポインタ。
  • **出力**:
  • 戻り値: インクリメント後の Datum 値。オーバーフロー時は (Datum) 0。
  • *overflow: インクリメントが最大値を超えた場合は true、それ以外は false。

前提条件と不変条件

  • existing は有効な char 型の値として解釈可能であること。
  • overflow ポインタは有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. existing を uint8 型に変換する。 2. 値が UCHAR_MAX と等しい場合：

  • *overflow を true に設定する。
  • (Datum) 0 を返す。

    • 3. それ以外の場合：

  • *overflow を false に設定する。
  • 値に 1 を加算し、CharGetDatum を介して Datum 型として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内の SkipSupport 構造体初期化処理（sksup->increment = char_increment;）。
  • **呼び出し先**: 直接的な関数呼び出しはなし。マクロ UInt8GetDatum および CharGetDatum を使用。

関連データ構造

  • Relation: PostgreSQL のリレーション構造体。
  • Datum: PostgreSQL の汎用データ表現型。
  • SkipSupport: インデックススキップスキャン用の補助構造体。

失敗モードと境界条件

  • **オーバーフロー**: 入力値が UCHAR_MAX (255) の場合、これ以上インクリメントできないため overflow が true となり、戻り値は未定義となる。

保守担当者向けメモ

  • 本関数は char 型専用のインクリメントロジックである。他の整数型については、同様の命名規則を持つ int2_increment 等が別途定義されている。
  • rel 引数は現在の実装では使用されていないが、将来的な拡張性のためにシグネチャに含まれている。
