sigpipe_info

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/fe-secure.c:57

目的

POSIX環境において、ネットワーク書き込み時に発生する SIGPIPE シグナルを一時的に制御・管理するための状態保持構造体です。

シグネチャまたは形状

struct sigpipe_info
{
    sigset_t    oldsigmask;
    bool        sigpipe_pending;
    bool        got_epipe;
};

入力と出力

本構造体は関数ではなくデータ型です。DECLARE_SIGPIPE_INFO(spinfo) マクロを通じてスタック上にインスタンス化され、シグナルマスクの保存およびシグナル発生状態の追跡に使用されます。

前提条件と不変条件

  • Windows環境では SIGPIPE が存在しないため、本構造体に関連する処理は no-op となります。
  • POSIX環境では、ネットワーク書き込み操作の前後でシグナルマスクを適切に操作するために使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: DECLARE_SIGPIPE_INFO により構造体が確保されます。 2. **保護**: pq_block_sigpipe 等のユーティリティ関数により、現在のシグナルマスクが oldsigmask に保存され、SIGPIPE がブロックされます。 3. **追跡**: 書き込み操作中に発生したシグナルやエラー（EPIPE）の状態が sigpipe_pending および got_epipe に記録されます。 4. **復元**: pq_reset_sigpipe 等により、保存されていた oldsigmask を用いてシグナル状態が復元されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-secure.c 内の pqsecure_raw_write 等、ネットワーク書き込みを行う関数。
  • 呼び出し先: 直接的な関数呼び出し先はありません（データ構造のため）。

関連データ構造

  • PGconn: libpqの接続状態を保持する構造体。本構造体は PGconn が管理する通信処理の文脈で使用されます。

失敗モードと境界条件

  • シグナルマスクの保存・復元に失敗した場合、プロセスのシグナル処理状態が意図せず変更される可能性があります。
  • SIGPIPE が発生した際、got_epipe フラグが適切にセットされない場合、書き込みエラーの検知が遅延する可能性があります。

保守担当者向けメモ

  • 本構造体は fe-secure.c 内の DISABLE_SIGPIPE / RESTORE_SIGPIPE マクロと密接に関連しています。
  • 修正を行う際は、スレッドセーフなシグナル操作が維持されているか確認してください。
