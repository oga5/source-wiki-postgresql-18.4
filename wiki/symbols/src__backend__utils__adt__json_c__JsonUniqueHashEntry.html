JsonUniqueHashEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/json.c:44

目的

JSONオブジェクトの解析時、同一階層内におけるキーの重複を検出するためのハッシュテーブル用エントリ構造体です。

シグネチャまたは形状

typedef struct JsonUniqueHashEntry
{
	const char *key;
	int			key_len;
	int			object_id;
} JsonUniqueHashEntry;

入力と出力

  • **入力**: ハッシュテーブルのキーとして文字列（key）とその長さ（key_len）、および所属するオブジェクトの識別子（object_id）を保持します。
  • **出力**: 該当なし（データ保持用構造体）。

前提条件と不変条件

  • ハッシュテーブルの keysize および entrysize として sizeof(JsonUniqueHashEntry) が指定される必要があります。
  • key はハッシュテーブルの生存期間中、有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

  • JSON解析中にキーが出現するたびに、この構造体を用いてハッシュテーブルを検索・挿入します。
  • object_id を含めることで、ネストされたJSON構造において、異なる階層のオブジェクト間でのキー重複誤検知を防ぎます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/json.c 内のJSON解析処理（json_validate 等に関連するロジック）。
  • **呼び出し先**: 該当なし（データ構造体のため）。

関連データ構造

  • JsonUniqueCheckState: 本構造体を利用するハッシュテーブルのコンテキスト。

失敗モードと境界条件

  • メモリ不足: ハッシュテーブルへの挿入時にメモリ不足が発生する可能性があります。
  • キーの不一致: key_len が正しく設定されていない場合、ハッシュ検索が正しく機能しません。

保守担当者向けメモ

  • 本構造体は src/backend/utils/adt/json.c 内のハッシュテーブル操作（hash_search 等）で直接キャストされて使用されます。
  • key フィールドは const char * ですが、ハッシュテーブルに格納される際は、バッファの再配置によるポインタ無効化を避けるため、必要に応じて文字列のコピーが管理されている点に注意してください。
