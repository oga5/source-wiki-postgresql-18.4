AutoVacuumShmemStruct

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/autovacuum.c:293

目的

PostgreSQL の自動バキューム（Autovacuum）システムにおける共有メモリ上の制御構造体です。ランチャープロセスとワーカープロセスの間で、プロセスの稼働状況、シグナル状態、および作業キューを共有するために使用されます。

シグネチャまたは形状

typedef struct
{
	sig_atomic_t av_signal[AutoVacNumSignals];
	pid_t		av_launcherpid;
	dclist_head av_freeWorkers;
	dlist_head	av_runningWorkers;
	WorkerInfo	av_startingWorker;
	AutoVacuumWorkItem av_workItems[NUM_WORKITEMS];
	pg_atomic_uint32 av_nworkersForBalance;
} AutoVacuumShmemStruct;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。共有メモリを介して、ランチャーおよびワーカープロセス間で状態の読み書きが行われます。

前提条件と不変条件

  • AutoVacuumShmem ポインタを介してアクセスされます。
  • 共有メモリ領域の初期化時に AutoVacuumShmemInit を通じて確保される必要があります。
  • 構造体内のリスト操作やカウンタ更新には、適切なロック（AutovacuumLock 等）の取得が前提となります。

アルゴリズムまたは振る舞い

  • **プロセス管理**: av_freeWorkers および av_runningWorkers を用いて、現在稼働中または待機中のワーカーの状態を追跡します。
  • **シグナル伝達**: av_signal 配列により、プロセス間での非同期通知を管理します。
  • **負荷分散**: av_nworkersForBalance を使用して、ワーカー間でバキュームコスト制限を動的に調整します。
  • **作業キュー**: av_workItems を通じて、バキューム対象の作業項目を保持します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/autovacuum.c 内の共有メモリ初期化処理およびワーカー管理関数。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • WorkerInfoData: 個々のワーカーの状態を保持する構造体。
  • AutoVacuumWorkItem: バキューム作業項目を定義する構造体。
  • dlist_head / dclist_head: リスト管理用のユーティリティ構造体。

失敗モードと境界条件

  • 共有メモリの確保失敗（AutoVacuumShmemInit 時）によりシステムが起動しない可能性があります。
  • NUM_WORKITEMS を超える作業項目は保持できません。
  • 共有メモリ上のロック競合が発生した場合、パフォーマンス低下やデッドロックのリスクがあります。

保守担当者向けメモ

  • 本構造体は src/backend/postmaster/autovacuum.c 内の静的変数 AutoVacuumShmem を通じて参照されます。
  • 構造体のサイズ変更を行う際は、共有メモリの再計算が必要となるため、AutoVacuumShmemInit 内のサイズ計算ロジックと整合性を取ってください。
