AtEOXact_Snapshot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/snapmgr.c:1013

目的

トランザクション終了時（End of Transaction）におけるスナップショット管理状態のクリーンアップおよびリソース解放を行います。

シグネチャまたは形状

void AtEOXact_Snapshot(bool isCommit, bool resetXmin)

入力と出力

  • isCommit (bool): トランザクションがコミットされる場合は true。この場合、残存するスナップショットに対して警告ログが出力されます。
  • resetXmin (bool): SnapshotResetXmin() を呼び出して MyProc->xmin を再計算するかどうかを指定します。
  • 出力: なし（void）。

前提条件と不変条件

  • TopTransactionContext がメモリ解放を担うため、スナップショット構造体自体の明示的なメモリ解放（pfree等）は行われません。
  • RegisteredSnapshots（ペアリングヒープ）および ActiveSnapshot（スタック）の状態をリセットし、整合性を保ちます。

アルゴリズムまたは振る舞い

1. **トランザクションスナップショットの除去**: FirstXactSnapshot が存在する場合、RegisteredSnapshots から削除します。 2. **エクスポート済みスナップショットの処理**: exportedSnapshots リストが存在する場合、対応するファイルを unlink で削除し、RegisteredSnapshots からも削除します。 3. **カタログスナップショットの無効化**: InvalidateCatalogSnapshot() を呼び出します。 4. **コミット時の検証**: isCommit が true の場合、RegisteredSnapshots が空でない、または ActiveSnapshot が残存している場合に警告ログを出力します。 5. **状態のリセット**: ActiveSnapshot を NULL にし、RegisteredSnapshots ヒープをリセットします。また、CurrentSnapshot、SecondarySnapshot、FirstSnapshotSet を初期化します。 6. **Xminの更新**: resetXmin が true の場合、SnapshotResetXmin() を実行してシステム全体の xmin を更新します。

失敗モードと境界条件

  • unlink 失敗時: WARNING ログを出力しますが、トランザクション自体は中断しません。
  • コミット時にスナップショットが残存している場合: WARNING ログを出力します。
  • resetXmin が false の場合: MyProc->xmin が 0 であることをアサートで検証します。

保守担当者向けメモ

  • 本関数はメモリの明示的な解放を行いません。メモリは TopTransactionContext の破棄に伴い自動的に解放される設計です。
  • ProcArrayEndTransaction() は本関数の呼び出し前に実行されることが想定されているため、xmin の管理順序には注意が必要です。
