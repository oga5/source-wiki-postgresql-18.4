test_is_full

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_tidstore/test_tidstore.c:333

目的

TidStore インスタンスが空の状態（tidstore_empty_size）を超えてメモリを使用しているかどうかを判定し、その結果を SQL インターフェースへ返します。

シグネチャまたは形状

Datum test_is_full(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL インターフェース経由の引数）。
  • **出力**: Datum 型として bool 値を返します（PG_RETURN_BOOL を使用）。TidStoreMemoryUsage(tidstore) が tidstore_empty_size より大きい場合に true を返します。

前提条件と不変条件

  • check_tidstore_available() が内部で呼び出されており、テスト対象の tidstore が初期化済みであることが前提です。

アルゴリズムまたは振る舞い

1. check_tidstore_available() を呼び出し、tidstore が利用可能か確認します。 2. TidStoreMemoryUsage(tidstore) を呼び出し、現在のメモリ使用量を取得します。 3. 取得した使用量が tidstore_empty_size を超えているかを比較し、その結果を is_full に格納します。 4. PG_RETURN_BOOL を使用して、判定結果を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(test_is_full) により登録）。
  • **呼び出し先**:
  • check_tidstore_available()
  • TidStoreMemoryUsage()

関連データ構造

  • tidstore: テスト対象の TidStore インスタンス。
  • tidstore_empty_size: 比較基準となる空状態のメモリサイズ。

失敗モードと境界条件

  • check_tidstore_available() が失敗した場合（tidstore が未初期化など）、エラーが発生し処理が中断されます。

保守担当者向けメモ

  • 本関数は TidStore のメモリ計測機能の正常性を検証するためのテスト関数です。
  • tidstore_empty_size はモジュール内のグローバル変数として定義されている前提です。
