GetDatabaseTuple

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:104

目的

指定されたデータベース名（dbname）に対応する pg_database システムカタログの行を検索し、そのタプルを取得します。

シグネチャまたは形状

static HeapTuple GetDatabaseTuple(const char *dbname)

入力と出力

  • **入力**: dbname (データベース名を示す文字列)
  • **出力**: 該当するデータベースの HeapTuple。見つからない場合は無効なタプル（NULL または HeapTupleIsValid が偽となる値）を返します。

前提条件と不変条件

  • table_open および systable_beginscan を使用するため、呼び出し元は適切なトランザクションコンテキストにある必要があります。
  • 戻り値のタプルはメモリコピーされているため、呼び出し元で不要になった際に heap_freetuple 等で解放する必要があります。

アルゴリズムまたは振る舞い

1. pg_database の datname 列を対象としたスキャンキーを作成します。 2. table_open で pg_database リレーションを AccessShareLock モードで開きます。 3. systable_beginscan を使用してスキャンを開始します。この際、criticalSharedRelcachesBuilt フラグの状態に応じてインデックススキャンまたはヒープスキャンを選択します。 4. systable_getnext でタプルを取得します。 5. タプルが有効な場合、heap_copytuple を使用してメモリ上にコピーを作成します。 6. systable_endscan および table_close を呼び出し、リソースを解放してコピーしたタプルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/init/postinit.c 内の関数（例: InitPostgres 等の初期化プロセス）。
  • **呼び出し先**: ScanKeyInit, table_open, systable_beginscan, systable_getnext, heap_copytuple, systable_endscan, table_close。

関連データ構造

  • HeapTuple: データベースの行データ。
  • Relation: pg_database システムカタログへのアクセス用。
  • ScanKeyData: 検索条件（datname）の保持に使用。

失敗モードと境界条件

  • 指定された dbname が存在しない場合、HeapTupleIsValid が偽となるタプルが返されます。
  • システムカタログへのアクセス権限がない場合や、カタログキャッシュが構築されていない段階での呼び出しには注意が必要です（criticalSharedRelcachesBuilt の状態に依存）。

保守担当者向けメモ

  • 本関数は pg_database を直接参照するため、カタログのスキーマ変更時には Anum_pg_database_datname や DatabaseRelationId の整合性に注意してください。
  • heap_copytuple によるメモリ確保が行われるため、呼び出し側でのメモリリークを防ぐため、取得したタプルのライフサイクル管理を徹底してください。
