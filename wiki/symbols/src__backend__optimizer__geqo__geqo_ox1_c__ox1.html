ox1

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_ox1.c:48

目的

遺伝的クエリ最適化（GEQO）において、2つの親個体から順序交叉（Order Crossover: OX1）を用いて新しい子個体を生成する。

シグネチャまたは形状

void ox1(PlannerInfo *root, Gene *tour1, Gene *tour2, Gene *offspring, int num_gene, City *city_table)

入力と出力

  • **入力**:
  • root: クエリ最適化のプランナ情報。
  • tour1, tour2: 交叉の元となる親個体の遺伝子配列。
  • num_gene: 遺伝子の総数。
  • city_table: 遺伝子の使用状況を追跡するための作業用配列。
  • **出力**:
  • offspring: 生成された子個体の遺伝子配列（引数経由で書き込み）。

前提条件と不変条件

  • city_table は num_gene 以上のサイズを持ち、関数内で初期化される。
  • num_gene を法とする循環的なインデックス操作が行われるため、配列の境界条件が維持される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: city_table の全 used フラグを 0 にリセットする。 2. **範囲選択**: geqo_randint を使用して、tour1 からコピーする部分列の開始点 left と終了点 right を決定する（left > right の場合は入れ替える）。 3. **親1のコピー**: tour1 の left から right までの遺伝子を offspring にコピーし、対応する city_table の used を 1 に設定する。 4. **親2の補完**: right の次の位置から開始し、tour2 を走査する。offspring に含まれていない（city_table が 0 の）遺伝子を順次 offspring に埋め込み、used を 1 に更新する。この処理を offspring が埋まるまで循環的に繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_main.c:213 にて呼び出されている。
  • **呼び出し先**: geqo_randint（乱数生成）を呼び出す。

関連データ構造

  • Gene: 遺伝子を表す型。
  • City: used フラグを保持する構造体。
  • PlannerInfo: クエリ最適化のコンテキスト情報。

失敗モードと境界条件

  • left と right が同じ場合、1つの遺伝子のみがコピーされる。
  • num_gene が 0 または 1 の場合の挙動については、geqo_randint の引数範囲に依存する。
  • city_table のインデックスは 1 から num_gene まで使用されるため、呼び出し側で十分なメモリが確保されている必要がある。

保守担当者向けメモ

  • 本関数は #if defined(OX1) で囲まれており、コンパイル時に有効化されている必要がある。
  • city_table のインデックス操作は num_gene を法とする循環参照であるため、修正時はインデックスのオーバーフローに注意すること。
