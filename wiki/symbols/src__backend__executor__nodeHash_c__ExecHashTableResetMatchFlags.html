ExecHashTableResetMatchFlags

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:2354

目的

ハッシュ結合の実行中、ハッシュテーブル内の全タプルに対して設定されている「マッチ済み（match）」フラグをリセットし、再スキャンや複数回結合の準備を行う。

シグネチャまたは形状

void ExecHashTableResetMatchFlags(HashJoinTable hashtable)

入力と出力

  • **入力**: hashtable (HashJoinTable) - 操作対象のハッシュ結合テーブル構造体。
  • **出力**: なし（副作用として hashtable 内部のタプル状態を更新）。

前提条件と不変条件

  • hashtable は ExecHashTableCreate 等によって初期化済みであること。
  • hashtable が保持するバケットおよびスキューバケットのタプルチェーンが有効であること。

アルゴリズムまたは振る舞い

1. **メインバケットのリセット**: hashtable->nbuckets 分のバケットを順次走査し、各バケット内の全タプルに対して HeapTupleHeaderClearMatch を実行してマッチフラグをクリアする。 2. **スキューバケットのリセット**: hashtable->nSkewBuckets 分のスキューバケットを順次走査し、同様に各タプルのマッチフラグをクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHashjoin.c:1519 (結合処理の再実行時など)
  • **呼び出し先**: HeapTupleHeaderClearMatch (マクロ)

関連データ構造

  • HashJoinTable: ハッシュ結合の全状態を保持する主要構造体。
  • HashJoinTuple: ハッシュテーブルに格納されるタプルのラッパー構造。

失敗モードと境界条件

  • hashtable が NULL の場合、あるいは初期化されていない場合はセグメンテーション違反が発生する可能性がある。
  • スキューバケットが存在しない場合（nSkewBuckets == 0）、スキューバケットのループはスキップされる。

保守担当者向けメモ

  • 本関数は、ハッシュ結合のプローブフェーズにおいて、外部リレーションの再スキャンが必要な場合に呼び出される。
  • HeapTupleHeaderClearMatch はタプルヘッダーのビットを直接操作するため、並列実行環境下での整合性については HashJoinTable のロック状態に依存する。
