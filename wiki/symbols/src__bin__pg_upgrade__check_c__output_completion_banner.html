output_completion_banner

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/check.c:812

目的

pg_upgrade プロセスの完了時に、ユーザーに対して統計情報の更新を促すコマンドと、旧クラスタの削除スクリプトに関する案内を表示する。

シグネチャまたは形状

void output_completion_banner(char *deletion_script_file_name)

入力と出力

  • **入力**:
  • deletion_script_file_name: 旧クラスタ削除用スクリプトのパス（char *）。NULL の場合、自動生成不可の旨が表示される。
  • **出力**: なし（void）。標準出力（pg_log経由）へメッセージを出力する。

前提条件と不変条件

  • os_info グローバル構造体が初期化されており、ユーザー指定の有無やユーザー名が参照可能であること。
  • new_cluster グローバル構造体が設定されており、新クラスタのバイナリディレクトリ（bindir）が取得可能であること。

アルゴリズムまたは振る舞い

1. PQExpBufferData を初期化し、ユーザー指定がある場合は接続用オプション文字列（-U <user>）を構築する。 2. vacuumdb を使用した統計情報更新コマンドの案内をログ出力する。 3. deletion_script_file_name が提供されている場合、そのスクリプトを実行して旧クラスタを削除するよう案内する。 4. deletion_script_file_name が NULL の場合、自動削除スクリプトが作成できなかった理由（テーブルスペースの配置等）を説明し、手動削除を促す。 5. 最後にバッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/pg_upgrade.c:238
  • **呼び出し先**:
  • initPQExpBuffer, appendPQExpBufferStr, appendShellString, appendPQExpBufferChar, termPQExpBuffer (libpq/fe_utils)
  • pg_log (pg_upgrade ユーティリティ関数)

関連データ構造

  • PQExpBufferData: 接続文字列構築用バッファ。
  • os_info: ユーザー情報保持用グローバル構造体。
  • new_cluster: 新クラスタ情報保持用グローバル構造体。

失敗モードと境界条件

  • deletion_script_file_name が NULL の場合、エラーではなく「手動削除が必要」という案内へ正常に分岐する。
  • ユーザー名にシェル特殊文字が含まれる場合、appendShellString により適切にエスケープされる。

保守担当者向けメモ

  • 本関数は pg_upgrade の最終段階で呼び出される。
  • 統計情報更新コマンドの引数や、削除スクリプトの案内文を変更する場合は、pg_upgrade.c のフローと整合性を取ること。
