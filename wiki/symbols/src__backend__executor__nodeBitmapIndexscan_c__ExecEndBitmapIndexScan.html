ExecEndBitmapIndexScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapIndexscan.c:174

目的

BitmapIndexScan ノードの実行を終了し、関連するインデックススキャンリソースを解放する。

シグネチャまたは形状

void ExecEndBitmapIndexScan(BitmapIndexScanState *node)

入力と出力

  • **入力**: BitmapIndexScanState *node （終了対象のビットマップインデックススキャン状態構造体）
  • **出力**: なし

前提条件と不変条件

  • node は初期化済みであること。
  • 並列クエリ実行時、呼び出し元は node->biss_SharedInfo を通じて統計情報の集約が行われることを期待する。

アルゴリズムまたは振る舞い

1. **統計情報の集約**: 並列ワーカープロセスで実行されている場合、node->biss_SharedInfo に格納された統計情報（nsearches）を共有メモリ上の対応するスロットへ加算する。 2. **スキャンの終了**: node->biss_ScanDesc が存在する場合、index_endscan を呼び出してインデックススキャンを終了する。 3. **リソースの解放**: node->biss_RelationDesc が存在する場合、index_close を呼び出してインデックスリレーションをクローズする（ロックは取得しない）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:648 からの呼び出しが確認されている。
  • **呼び出し先**:
  • index_endscan (インデックスアクセスメソッド)
  • index_close (インデックスアクセスメソッド)

関連データ構造

  • BitmapIndexScanState: スキャンノードの状態を保持する構造体。
  • IndexScanDesc: インデックススキャンの記述子。
  • IndexScanInstrumentation: 並列実行時の統計情報構造体。

失敗モードと境界条件

  • indexScanDesc または indexRelationDesc が NULL の場合、それぞれのクローズ処理はスキップされる。
  • 並列ワーカーとして動作していない場合、統計情報の共有メモリへの書き戻しは行われない。

保守担当者向けメモ

  • 本関数は ExecProcNode インターフェースの一部ではないため、直接呼び出さないこと。
  • 並列クエリ実行時の統計情報は memcpy ではなく加算（+=）で更新される点に注意すること。これは再スキャン時に統計情報がリセットされる仕様に対応するためである。
