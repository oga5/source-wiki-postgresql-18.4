pg_atomic_compare_exchange_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic-sunpro.h:71

目的

Solaris 環境において、pg_atomic_uint32 型の変数に対するアトミックな比較交換（CAS）操作を提供します。

シグネチャまたは形状

static inline bool pg_atomic_compare_exchange_u32_impl(volatile pg_atomic_uint32 *ptr, uint32 *expected, uint32 newval)

入力と出力

  • ptr: 操作対象となる pg_atomic_uint32 構造体へのポインタ。
  • expected: 期待値を保持する変数へのポインタ。操作後、現在の値で更新されます。
  • newval: 比較が成功した場合に書き込まれる新しい値。
  • 戻り値: 比較が成功（*expected と現在の値が一致）した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • ptr は有効な pg_atomic_uint32 構造体を指している必要があります。
  • Solaris の atomic_cas_32 を利用するため、コンパイル環境で <atomic.h> が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. Solaris のシステム関数 atomic_cas_32(&ptr->value, *expected, newval) を呼び出し、現在の値を読み取ると同時に比較交換を試みます。 2. 戻り値 current と元の *expected を比較し、一致すれば true を、不一致であれば false を ret に格納します。 3. *expected を current（操作直前の値）で更新します。 4. ret を返します。

呼び出し元と呼び出し先

  • 呼び出し先: Solaris システム関数 atomic_cas_32。
  • 呼び出し元: src/include/port/atomics.h および src/include/port/atomics/generic.h 内の汎用アトミック操作関数から呼び出されます。

関連データ構造

  • pg_atomic_uint32: アトミック操作対象の値を保持する構造体。

失敗モードと境界条件

  • 期待値と現在の値が一致しない場合、操作は失敗し、*expected が現在の値で更新された上で false が返されます。

保守担当者向けメモ

  • Solaris の atomic_cas は不一致時に現在の値を返す仕様であるため、PostgreSQL の期待する「期待値を更新する」動作と整合しています。
  • 本関数は Solaris 環境専用の実装です。
