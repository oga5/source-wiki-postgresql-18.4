fail_lo_xact

定義

  • 種別: function
  • 定義箇所: src/bin/psql/large_obj.c:120

目的

ラージオブジェクト操作中にエラーが発生した際、必要に応じてトランザクションをロールバックし、操作失敗を示す値を返却する。

シグネチャまたは形状

static bool fail_lo_xact(const char *operation, bool own_transaction)

入力と出力

  • **入力**:
  • operation: 実行しようとしていた操作名（ログ出力用）。
  • own_transaction: 本関数がトランザクションを管理しているかを示すフラグ。
  • **出力**:
  • 常に false を返却する。

前提条件と不変条件

  • pset.autocommit が有効であり、かつ own_transaction が真である場合にのみ、データベースに対して ROLLBACK コマンドが発行される。

アルゴリズムまたは振る舞い

1. own_transaction が真かつ pset.autocommit が有効な場合、PSQLexec("ROLLBACK") を実行してトランザクションを破棄する。 2. PQclear を呼び出し、ROLLBACK 実行時に生成された PGresult を解放する。 3. 呼び出し元がエラー処理を継続できるよう、常に false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • do_lo_export (src/bin/psql/large_obj.c:158)
  • do_lo_import (src/bin/psql/large_obj.c:193, 205, 214)
  • do_lo_unlink (src/bin/psql/large_obj.c:255)
  • **呼び出し先**:
  • PSQLexec
  • PQclear

関連データ構造

  • pset: settings.h で定義されるグローバルな設定構造体。autocommit 状態の判定に使用。
  • PGresult: libpq の結果保持構造体。

失敗モードと境界条件

  • PSQLexec が内部で失敗した場合の挙動については、本関数のソースコード内では明示的なエラーハンドリングを行っていない。

保守担当者向けメモ

  • 本関数は常に false を返すため、呼び出し側ではこの戻り値をそのまま関数の終了値として利用することが想定されている。
  • operation 引数は現在の実装では使用されていないが、将来的なログ出力の拡張のために保持されている可能性がある。
