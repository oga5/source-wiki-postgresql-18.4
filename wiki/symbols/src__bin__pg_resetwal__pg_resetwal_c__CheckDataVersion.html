CheckDataVersion

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:544

目的

データディレクトリ内の PG_VERSION ファイルを読み込み、その内容が実行中の pg_resetwal プログラムのメジャーバージョン（PG_MAJORVERSION）と一致するかを検証する。

シグネチャまたは形状

static void CheckDataVersion(void)

入力と出力

  • **入力**: カレントディレクトリにある PG_VERSION ファイル。
  • **出力**: なし（成功時）。不一致や読み込み失敗時はエラーメッセージを出力し、exit(1) を呼び出してプロセスを終了する。

前提条件と不変条件

  • カレントディレクトリに PG_VERSION ファイルが存在すること。
  • PG_MAJORVERSION マクロが定義されていること。

アルゴリズムまたは振る舞い

1. PG_VERSION ファイルを読み取りモードで開く。失敗した場合は致命的エラーとして終了する。 2. ファイルの最初の行を読み込み、末尾の改行コード（CR/LF）を除去する。 3. 読み取ったバージョン文字列と PG_MAJORVERSION を比較する。 4. 一致しない場合、エラー詳細を出力してプログラムを終了する。 5. 一致する場合、ファイルを閉じて正常に復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_resetwal/pg_resetwal.c 内の main 関数（385行目付近）。
  • **呼び出し先**: pg_fatal（エラー終了用）、pg_strip_crlf（文字列整形用）、pg_log_error / pg_log_error_detail（ログ出力用）、標準ライブラリの fopen, fgets, fclose。

関連データ構造

  • なし。

失敗モードと境界条件

  • **ファイル不在/アクセス不可**: fopen が失敗し、pg_fatal により終了する。
  • **空ファイル**: fgets が失敗し、pg_fatal により終了する。
  • **バージョン不一致**: strcmp で不一致が検出された場合、エラーメッセージを表示して exit(1) で終了する。

保守担当者向けメモ
