RmgrNotFound

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/rmgr.c:90

目的

未知のWALリソースマネージャID（RmgrId）が参照された際に、エラーを報告して処理を停止させる。

シグネチャまたは形状

void RmgrNotFound(RmgrId rmid)

入力と出力

  • **入力**: rmid (未知のWALレコードID)
  • **出力**: なし（関数内で ereport(ERROR, ...) を呼び出し、呼び出し元の実行を中断する）

前提条件と不変条件

  • 本関数は、WALレコードの解析中に RmgrTable 内で対応するリソースマネージャが見つからなかった場合に呼び出される。
  • rmid は uint8 型の範囲内である必要がある。

アルゴリズムまたは振る舞い

1. ereport を使用して ERROR レベルのログを出力する。 2. エラーメッセージには、該当する rmid が登録されていない旨を表示する。 3. errhint として、必要な拡張モジュールが shared_preload_libraries に含まれていない可能性があることを示唆する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/xlog_internal.h:379（WALレコード解析時のディスパッチ処理において、IDが範囲外または未登録の場合に呼び出される）。
  • **呼び出し先**: ereport（PostgreSQLのエラー報告メカニズム）。

関連データ構造

  • RmgrId: リソースマネージャを識別するID型。
  • RmgrTable: リソースマネージャの定義を保持するグローバル配列。

失敗モードと境界条件

  • 本関数自体がエラーを発生させるため、呼び出し元はリカバリ処理を継続できなくなる。
  • 拡張機能が shared_preload_libraries に正しく設定されていない場合、リカバリ中にこの関数が呼び出される可能性がある。

保守担当者向けメモ

  • 本関数はリカバリの安全性を担保するための最終防衛線である。
  • エラーメッセージのヒントにある通り、カスタムリソースマネージャの登録漏れが主な原因となるため、デバッグ時には shared_preload_libraries の設定を確認すること。
