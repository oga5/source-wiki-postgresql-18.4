PG_init_t

定義

  • 種別: typedef
  • 定義箇所: src/backend/utils/fmgr/dfmgr.c:32

目的

動的にロードされた共有ライブラリ内の初期化関数（_PG_init）を指すための関数ポインタ型を定義します。

シグネチャまたは形状

typedef void (*PG_init_t) (void);

入力と出力

  • 入力: なし（void）
  • 出力: なし（void）

前提条件と不変条件

  • 共有ライブラリ内に _PG_init という名前のシンボルが存在する場合にのみ使用されます。
  • この型へのキャストは、dlsym によって取得されたシンボルアドレスに対して行われます。

アルゴリズムまたは振る舞い

共有ライブラリのロード時、dlsym を介して _PG_init シンボルを検索し、この型にキャストして関数ポインタとして保持します。その後、必要に応じてこのポインタを介して初期化関数が実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/fmgr/dfmgr.c 内のライブラリロード処理（internal_load_library 等の文脈）。
  • 呼び出し先: 共有ライブラリ内で定義された _PG_init 関数。

関連データ構造

  • DynamicFileList: ロード済みライブラリを管理する構造体。

失敗モードと境界条件

  • ライブラリ内に _PG_init シンボルが存在しない場合、dlsym は NULL を返します。
  • キャスト後の関数ポインタが NULL の状態で呼び出そうとするとセグメンテーション違反が発生するため、呼び出し前に必ず存在確認が必要です。

保守担当者向けメモ

  • この型は PostgreSQL の拡張モジュールにおける初期化処理の標準的なインターフェースです。
  • Windows 環境など、プラットフォーム固有の動的リンク処理においても、この関数ポインタ型を介した呼び出しが抽象化されています。
