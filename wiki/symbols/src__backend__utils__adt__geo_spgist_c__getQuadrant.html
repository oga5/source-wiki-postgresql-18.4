getQuadrant

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_spgist.c:129

目的

2つの矩形（BOX）を比較し、基準となる矩形（centroid）に対する対象矩形（inBox）の相対的な位置関係を4次元空間上の象限（0〜15）として特定します。

シグネチャまたは形状

uint8 getQuadrant(BOX *centroid, BOX *inBox)

入力と出力

  • **入力**:
  • centroid: 基準となる矩形（重心）。
  • inBox: 判定対象の矩形。
  • **出力**:
  • uint8: 4ビットのビットマスク値（0〜15）。各ビットは x_low, x_high, y_low, y_high の各次元が基準値より大きいかどうかを表す。

前提条件と不変条件

  • centroid および inBox は有効な BOX 構造体へのポインタである必要があります。
  • 戻り値の各ビットは以下の対応関係を持ちます：
  • ビット3 (0x8): inBox->low.x > centroid->low.x
  • ビット2 (0x4): inBox->high.x > centroid->high.x
  • ビット1 (0x2): inBox->low.y > centroid->low.y
  • ビット0 (0x1): inBox->high.y > centroid->high.y

アルゴリズムまたは振る舞い

1. quadrant 変数を 0 で初期化します。 2. inBox の各座標（low.x, high.x, low.y, high.y）を centroid の対応する座標と比較します。 3. inBox の値が centroid より大きい場合、対応するビットを立てます。 4. 最終的なビットマスクを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/geo_spgist.c: spg_box_quad_choose および spg_box_quad_picksplit 内で呼び出されています。
  • src/backend/access/spgist/spgquadtreeproc.c: 同名の関数が Point 型を引数として呼び出されています（本シンボルとは引数型が異なります）。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • BOX: 幾何学的な矩形を表す構造体（utils/geo_decls.h で定義）。

失敗モードと境界条件

  • centroid と inBox の座標が等しい場合、対応するビットは立ちません（0が返されます）。
  • 浮動小数点数の比較精度については、呼び出し側の文脈に依存します。

保守担当者向けメモ

  • 本関数は BOX 型専用です。Point 型を扱う同名の関数が spgquadtreeproc.c に存在するため、修正時には対象ファイルを確認してください。
  • 戻り値のビットマスクは、SP-GiSTの四分木におけるノード選択のインデックスとして利用されます。
