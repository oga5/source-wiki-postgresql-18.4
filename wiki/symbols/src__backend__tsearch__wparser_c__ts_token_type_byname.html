ts_token_type_byname

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/wparser.c:124

目的

指定された名前の全文検索パーサーがサポートするトークン型の一覧を、SQLから取得するためのSet-Returning Function (SRF) です。

シグネチャまたは形状

Datum ts_token_type_byname(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): パーサー名を指定する text 型の引数。
  • **出力**:
  • SRFとして、トークン型情報を含むタプルを順次返します。

前提条件と不変条件

  • FuncCallContext を使用して呼び出し間の状態を保持します。
  • 初回呼び出し時に get_ts_parser_oid を用いてパーサーのOIDを解決する必要があります。

アルゴリズムまたは振る舞い

1. **初回呼び出し時 (SRF_IS_FIRSTCALL)**:

  • SRF_FIRSTCALL_INIT() でコンテキストを初期化します。
  • 入力されたパーサー名から get_ts_parser_oid でOIDを取得します。
  • tt_setup_firstcall を呼び出し、キャッシュからパーサー定義をロードしてトークンリストをメモリ上に展開します。

    • 2. **各呼び出し時**:

  • SRF_PERCALL_SETUP() でコンテキストを再取得します。
  • tt_process_call を呼び出し、メモリ上のリストから次のトークン型情報を取得します。
  • データが残っている場合は SRF_RETURN_NEXT で返し、終了時は SRF_RETURN_DONE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（外部からの呼び出し）。
  • **呼び出し先**:
  • get_ts_parser_oid
  • tt_setup_firstcall
  • tt_process_call

関連データ構造

  • FuncCallContext: SRFの状態保持に使用。
  • TSParserCacheEntry (間接的): tt_setup_firstcall 内でキャッシュから参照される。

失敗モードと境界条件

  • 指定されたパーサー名が存在しない場合、get_ts_parser_oid がエラーを発生させる可能性があります。
  • SRFを使用しているため、大量のトークン型を保持するパーサーの場合、メモリ消費量が増大する可能性があります。

保守担当者向けメモ

  • 本関数はSQLインターフェース層であり、実際のロジックは tt_setup_firstcall および tt_process_call に委譲されています。
  • 動作の詳細は src/backend/tsearch/wparser.c 内の関連ヘルパー関数を参照してください。
