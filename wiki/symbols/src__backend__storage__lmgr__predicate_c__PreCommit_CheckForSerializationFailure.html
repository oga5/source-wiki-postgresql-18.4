PreCommit_CheckForSerializationFailure

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/predicate.c:4702

目的

トランザクションのコミット直前に、直列化可能スナップショット分離（SSI）の観点から競合を検証し、直列化不可能な依存関係（サイクル等）が存在する場合はトランザクションを失敗させる。

シグネチャまたは形状

void PreCommit_CheckForSerializationFailure(void)

入力と出力

  • 入力: なし（グローバル変数 MySerializableXact を参照）
  • 出力: なし（成功時は副作用として状態更新、失敗時は ERROR をスロー）

前提条件と不変条件

  • MySerializableXact が InvalidSerializableXact でないこと。
  • 現在のトランザクション分離レベルが SERIALIZABLE であること（IsolationIsSerializable() のアサーション）。

アルゴリズムまたは振る舞い

1. SerializableXactHashLock を排他モードで取得する。 2. 自トランザクションが既に DOOMED 状態であればエラーを発生させる。 3. 自トランザクションの inConflicts リストを走査し、未コミットかつ未 DOOMED な競合先（nearConflict）を特定する。 4. さらにその先の依存関係（farConflict）を辿り、サイクルを形成する可能性があるか確認する。 5. 競合先が準備済み（prepared）であれば、自トランザクションをエラー終了させる。 6. それ以外の場合、競合先のトランザクションに SXACT_FLAG_DOOMED を付与してロールバックを強制する。 7. 最後に、自トランザクションのコミット順序番号を更新し、SXACT_FLAG_PREPARED フラグを立ててロックを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c（PreCommit_CheckForSerializationFailure() として呼び出し）
  • 呼び出し先: LWLockAcquire, LWLockRelease, ereport 等の内部ユーティリティ。

関連データ構造

  • SERIALIZABLEXACT: 直列化可能トランザクションの状態管理構造体。
  • RWConflictData: トランザクション間の読み書き依存関係を保持する構造体。
  • SerializableXactHashLock: 共有メモリ上の直列化可能トランザクション情報を保護するロック。

失敗モードと境界条件

  • ERRCODE_T_R_SERIALIZATION_FAILURE: 読み書き依存関係により直列化が不可能と判断された場合に発生。
  • 準備済みトランザクションとの競合: 相手が準備済みの場合、自らがエラーを投げる必要がある。
  • 部分解放中: SxactIsPartiallyReleased が真の場合は DOOMED チェックをスキップする。

保守担当者向けメモ

  • 本関数はコミットの最終段階で実行されるため、ここでのエラーはトランザクションのロールバックを誘発する。
  • SerializableXactHashLock を保持したまま複雑なリスト走査を行うため、競合が多い環境ではパフォーマンスへの影響に注意が必要。
