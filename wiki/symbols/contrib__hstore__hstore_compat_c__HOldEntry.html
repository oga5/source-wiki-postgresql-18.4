HOldEntry

定義

  • 種別: struct
  • 定義箇所: contrib/hstore/hstore_compat.c:99

目的

PostgreSQL の hstore 型における旧形式（hstore-old）のデータエントリを表現するための構造体です。新形式へのアップグレード処理において、旧形式データの読み込みと解析に使用されます。

シグネチャまたは形状

typedef struct {
    uint16 keylen;
    uint16 vallen;
    uint32 valisnull:1,
           pos:31;
} HOldEntry;

入力と出力

本構造体はデータ保持用であり、関数ではありません。hstoreUpgrade 等の処理において、ARRPTR(hs) からキャストされたポインタを通じて旧形式の hstore エントリ配列としてアクセスされます。

前提条件と不変条件

  • sizeof(HOldEntry) が 2 * sizeof(HEntry) と等しいことが、StaticAssertStmt により検証されています。
  • 旧形式と新形式のエントリサイズが一致していることが、本構造体を利用した変換処理の前提となります。

アルゴリズムまたは振る舞い

旧形式の hstore データ構造をメモリ上で表現します。hstoreUpgrade 関数内で、旧形式のデータが検出された際に、この構造体を用いて各エントリのキー長、値長、値の有無、および位置情報を読み取り、新形式（HEntry）へ変換するためのマッピング元として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_compat.c 内の hstoreUpgrade 関数が、旧形式の hstore データを処理する際に本構造体へキャストして参照します。
  • **呼び出し先**: 本構造体はデータ構造定義であるため、直接関数を呼び出すことはありません。

関連データ構造

  • HEntry: 現在の hstore 形式で使用されるエントリ構造体。
  • HStore: hstore 型のメイン構造体。

失敗モードと境界条件

  • プラットフォームの制約により sizeof(HOldEntry) が 2 * sizeof(HEntry) と一致しない場合、コンパイル時にアサーションエラーが発生します。
  • 旧形式と新形式の判別が曖昧なデータについては、コンパイル時の設定に基づいて解釈されます。

保守担当者向けメモ

  • 本構造体は旧形式との互換性維持専用です。新しい機能追加時には使用しないでください。
  • 変換ロジックを変更する際は、hstore_compat.c 内の StaticAssertStmt が維持されていることを確認してください。
