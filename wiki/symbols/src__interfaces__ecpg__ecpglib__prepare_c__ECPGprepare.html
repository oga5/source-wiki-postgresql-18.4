ECPGprepare

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:216

目的

ECPGにおける EXEC SQL PREPARE 文のフロントエンド実装であり、指定されたSQL文をデータベース接続上でプリペアドステートメントとして準備します。

シグネチャまたは形状

bool ECPGprepare(int lineno, const char *connection_name, const bool questionmarks, const char *name, const char *variable);

入力と出力

  • **入力**:
  • lineno: 呼び出し元の行番号（エラー報告用）。
  • connection_name: 接続名（NULLの場合はデフォルト接続）。
  • questionmarks: プレースホルダの処理フラグ（現在は未使用）。
  • name: プリペアドステートメントの識別子。
  • variable: 準備するSQL文字列。
  • **出力**:
  • bool: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • 呼び出し前に ecpg_init による接続の初期化が必要です。
  • 指定された name が既に存在する場合、既存のステートメントを破棄してから再準備を試みます。

アルゴリズムまたは振る舞い

1. ecpg_get_connection を使用して接続オブジェクトを取得します。 2. ecpg_init で接続状態を検証します。 3. ecpg_find_prepared_statement を呼び出し、同名のステートメントが既に存在するか確認します。 4. 既存のステートメントが存在する場合、deallocate_one を呼び出してバックエンドから破棄します。 5. prepare_common を呼び出し、SQL文字列の解析とバックエンドへの PREPARE コマンド送信を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/prepare.c (内部の自動プリペア処理)
  • プリプロセッサが生成したCコード（src/interfaces/ecpg/preproc/output.c 経由）
  • 各種テストコード（compat_informix-describe.c 等）
  • **呼び出し先**:
  • ecpg_get_connection
  • ecpg_init
  • ecpg_find_prepared_statement
  • deallocate_one
  • prepare_common

関連データ構造

  • connection: データベース接続情報を保持する構造体。
  • prepared_statement: プリペアドステートメントの管理用構造体。

失敗モードと境界条件

  • 接続が見つからない、または初期化に失敗した場合は false を返します。
  • 既存のステートメントの破棄（deallocate_one）に失敗した場合、処理を中断し false を返します。
  • prepare_common が失敗した場合、その結果をそのまま返します。

保守担当者向けメモ

  • questionmarks 引数は現在使用されておらず、コンパイラの警告を抑制するためにキャストされています。
  • deallocate_one 実行時のメモリ管理には注意が必要です。
  • プリペアドステートメントのリスト管理は接続オブジェクトに紐付いています。
