GetConfFilesInDir

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/conffiles.c:69

目的

指定されたディレクトリ内から .conf 拡張子を持つファイルを検索し、絶対パスのリストとして返します。

シグネチャまたは形状

char **GetConfFilesInDir(const char *includedir, const char *calling_file, int elevel, int *num_filenames, char **err_msg)

入力と出力

  • **入力**:
  • includedir: 検索対象のディレクトリパス。
  • calling_file: 基準となるファイルパス（AbsoluteConfigLocation で使用）。
  • elevel: ereport に渡すエラーレベル。
  • num_filenames: 検出されたファイル数を格納するポインタ（出力）。
  • err_msg: エラー発生時に詳細メッセージを格納するポインタ（出力）。
  • **出力**:
  • 成功時: palloc された絶対パスの文字列配列（char **）。
  • 失敗時: NULL。

前提条件と不変条件

  • includedir が空白のみ、または空文字列である場合はエラーとなります。
  • 返される配列は palloc されているため、呼び出し元で適切に解放する必要があります。
  • 検出されたファイルはアルファベット順にソートされます。

アルゴリズムまたは振る舞い

1. AbsoluteConfigLocation を用いて includedir を絶対パスに変換します。 2. AllocateDir でディレクトリを開き、ReadDir でエントリを走査します。 3. 以下の条件を満たすファイルのみを収集対象とします。

  • 名前が . で始まらない。
  • 拡張子が .conf である。
  • ディレクトリではない。

    • 4. 収集したパスを palloc された配列に格納し、必要に応じて repalloc で拡張します。 5. pg_qsort_strcmp を使用してファイル名順にソートします。 6. FreeDir でディレクトリを閉じ、変換後のディレクトリパスを pfree して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c:834
  • **呼び出し先**:
  • AbsoluteConfigLocation
  • AllocateDir, ReadDir, FreeDir
  • get_dirent_type
  • join_path_components, canonicalize_path
  • qsort (比較関数: pg_qsort_strcmp)
  • palloc, pfree, pstrdup, repalloc
  • ereport, psprintf

関連データ構造

  • DIR (ディレクトリストリーム)
  • struct dirent (ディレクトリエントリ)
  • PGFileType (ファイルタイプ判定用)

失敗モードと境界条件

  • **ディレクトリ名が空**: ereport を発行し NULL を返します。
  • **ディレクトリオープン失敗**: err_msg にエラー内容を格納し NULL を返します。
  • **ファイル統計情報取得失敗**: pfree でメモリを解放し、err_msg を設定して NULL を返します。
  • **隠しファイル・非.confファイル**: 無視されます。

保守担当者向けメモ

  • 本関数は palloc を使用しているため、呼び出し側でのメモリ管理に注意してください。
  • 無限ループを防ぐため、ディレクトリ名が空でないことのチェックが含まれています。
