gbt_bool_penalty

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bool.c:136

目的

GiSTインデックスにおいて、新しいブール値を挿入する際にインデックスツリーのどのノードを選択するのが最適か（ペナルティが最小か）を計算します。

シグネチャまたは形状

Datum gbt_bool_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): 既存のインデックスエントリ（boolKEY 構造体へのポインタ）。
  • PG_GETARG_POINTER(1): 挿入対象の新しいエントリ（boolKEY 構造体へのポインタ）。
  • PG_GETARG_POINTER(2): 計算結果を格納する float 型へのポインタ。
  • **出力**: PG_RETURN_POINTER(result) を通じて、計算されたペナルティ値（float）を返します。

前提条件と不変条件

  • 入力される boolKEY は、lower および upper フィールドを持つ有効な構造体である必要があります。
  • 内部的に penalty_num を呼び出すため、bool 型が数値（0/1）として比較可能であることを前提としています。

アルゴリズムまたは振る舞い

1. 引数から既存エントリ（origentry）と新規エントリ（newentry）のキーを取得します。 2. penalty_num 関数を呼び出し、既存の範囲と新しい値の差分に基づいてペナルティ値を計算します。 3. 計算された値を result ポインタに書き込み、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックスエンジン（直接の呼び出し元はソースコード上では明示されていませんが、GiSTのインデックス挿入プロセスから呼び出されます）。
  • **呼び出し先**: penalty_num（btree_utils_num.h で定義された共通ユーティリティ関数）。

関連データ構造

  • boolKEY: bool 型の範囲を保持する構造体。
  • GISTENTRY: GiSTインデックスの各エントリを保持する標準的なPostgreSQL構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。
  • boolKEY 構造体のキャストが正しく行われない場合、メモリ上の不正な領域を参照する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.h の penalty_num を利用する薄いラッパーです。ロジックの変更が必要な場合は、まず共通ユーティリティ側の影響範囲を確認してください。
  • bool 型は実質的に 0 または 1 の数値として扱われるため、この実装は他の数値型（int4 等）のペナルティ計算と論理的に同等です。
