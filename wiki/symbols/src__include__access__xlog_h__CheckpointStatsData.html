CheckpointStatsData

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlog.h:161

目的

チェックポイント処理の実行時間、書き込み量、および WAL セグメント管理に関する統計情報を保持するための構造体です。

シグネチャまたは形状

typedef struct CheckpointStatsData
{
    TimestampTz ckpt_start_t;       /* チェックポイント開始時刻 */
    TimestampTz ckpt_write_t;       /* バッファ書き込み開始時刻 */
    TimestampTz ckpt_sync_t;        /* fsync開始時刻 */
    TimestampTz ckpt_sync_end_t;    /* fsync終了時刻 */
    TimestampTz ckpt_end_t;         /* チェックポイント終了時刻 */

    int         ckpt_bufs_written;  /* 書き込まれたバッファ数 */
    int         ckpt_slru_written;  /* 書き込まれたSLRUバッファ数 */

    int         ckpt_segs_added;    /* 新規作成されたWALセグメント数 */
    int         ckpt_segs_removed;  /* 削除されたWALセグメント数 */
    int         ckpt_segs_recycled; /* 再利用されたWALセグメント数 */

    int         ckpt_sync_rels;     /* 同期されたリレーション数 */
    uint64      ckpt_longest_sync;  /* 単一リレーションの最長同期時間 */
    uint64      ckpt_agg_sync_time; /* 全同期時間の合計 */
} CheckpointStatsData;

入力と出力

  • 本構造体は、チェックポイント処理の各フェーズで更新される状態保持用データです。
  • 関数への引数として渡されるのではなく、グローバルな統計状態として利用されます。

前提条件と不変条件

  • ckpt_agg_sync_time は、個々の同期時間の総和であり、チェックポイント全体の同期フェーズの経過時間と一致するとは限りません。

アルゴリズムまたは振る舞い

  • チェックポイントの開始から終了までの各段階（バッファ書き込み、fsync、セグメント管理）において、対応するフィールドにタイムスタンプやカウンタが記録されます。
  • ckpt_longest_sync および ckpt_agg_sync_time は、リレーションの同期処理中に計測・集計されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c 内のチェックポイント実行ロジック。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • TimestampTz: タイムスタンプ管理に使用。
  • CheckpointStats: src/backend/access/transam/xlog.c で定義される、本構造体型のグローバル変数。

失敗モードと境界条件

  • 統計情報の更新はチェックポイント処理の進行に依存します。チェックポイントが中断された場合、一部のフィールドが未更新または不完全な値となる可能性があります。

保守担当者向けメモ

  • 本構造体は PGDLLIMPORT として公開されており、外部モジュールから参照可能です。
  • 統計フィールドを追加・変更する場合は、xlog.c 内の統計更新ロジックとの整合性に注意してください。
