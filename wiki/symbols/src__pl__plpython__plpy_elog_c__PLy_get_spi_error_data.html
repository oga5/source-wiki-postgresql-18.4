PLy_get_spi_error_data

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:388

目的

Python 例外オブジェクトから、PostgreSQL の SPI（Server Programming Interface）エラーに関連する詳細情報（SQLSTATE、エラーメッセージ、ヒント、クエリ位置など）を抽出します。

シグネチャまたは形状

static void PLy_get_spi_error_data(
    PyObject *exc, 
    int *sqlerrcode, char **detail, char **hint, char **query, int *position,
    char **schema_name, char **table_name, char **column_name,
    char **datatype_name, char **constraint_name
)

入力と出力

  • **入力**:
  • exc: 情報を抽出する対象の Python 例外オブジェクト。
  • **出力**:
  • 引数として渡されたポインタ（sqlerrcode, detail, hint 等）に、抽出した値が格納されます。

前提条件と不変条件

  • exc は有効な Python オブジェクトである必要があります。
  • 抽出された文字列ポインタは、Python オブジェクトの生存期間に依存します。

アルゴリズムまたは振る舞い

1. exc オブジェクトから "spidata" 属性を取得します。 2. spidata が存在する場合、PyArg_ParseTuple を使用してタプルから各エラーフィールドを解析・抽出します。 3. spidata が存在しない場合（Python コードから直接 SPI 例外が投げられた場合など）、PLy_get_sqlerrcode を呼び出して sqlerrcode のみを設定します。 4. 最後に spidata オブジェクトの参照カウントを減らします（Py_XDECREF）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_elog.c 内の例外処理ロジックから呼び出されます。
  • **呼び出し先**:
  • PyObject_GetAttrString
  • PyArg_ParseTuple
  • PLy_get_sqlerrcode
  • Py_XDECREF

関連データ構造

  • PyObject: Python の例外オブジェクト。
  • spidata: Python 例外に付随するエラー詳細情報のタプル。

失敗モードと境界条件

  • spidata 属性が存在しない場合、sqlerrcode 以外の出力引数は更新されません。
  • PyArg_ParseTuple の解析に失敗した場合の挙動は、Python の例外状態に依存します。

保守担当者向けメモ

  • 本関数は spidata 属性の構造に強く依存しています。Python 側の例外クラス定義に変更を加える際は、PyArg_ParseTuple のフォーマット文字列と引数順序を同期させる必要があります。
