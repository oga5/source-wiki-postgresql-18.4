pltcl_proc_desc

定義

  • 種別: struct
  • 定義箇所: src/pl/tcl/pltcl.c:144

目的

PostgreSQL の関数定義（pg_proc）を Tcl プロシージャとしてコンパイルした結果を保持し、実行時に再利用するためのキャッシュ用データ構造です。

シグネチャまたは形状

typedef struct pltcl_proc_desc {
    char *user_proname;
    char *internal_proname;
    MemoryContext fn_cxt;
    unsigned long fn_refcount;
    TransactionId fn_xmin;
    ItemPointerData fn_tid;
    bool fn_readonly;
    bool lanpltrusted;
    pltcl_interp_desc *interp_desc;
    Oid result_typid;
    FmgrInfo result_in_func;
    Oid result_typioparam;
    bool fn_retisset;
    bool fn_retistuple;
    bool fn_retisdomain;
    void *domain_info;
    int nargs;
    FmgrInfo *arg_out_func;
    bool *arg_is_rowtype;
} pltcl_proc_desc;

入力と出力

  • **入力**: コンパイル対象の関数情報（OID、トリガー情報、ユーザーID）。
  • **出力**: Tcl インタプリタ上で実行可能なプロシージャ名および、PostgreSQL とのデータ変換に必要な型情報や関数ポインタ。

前提条件と不変条件

  • fn_cxt は、このプロシージャに関連するメモリを管理する専用のメモリコンテキストです。
  • fn_refcount は、現在アクティブな参照数を保持し、不要になった際の解放を制御します。
  • fn_xmin および fn_tid は、pg_proc 行の更新を検知し、キャッシュの妥当性を検証するために使用されます。

アルゴリズムまたは振る舞い

1. **キャッシュ生成**: compile_pltcl_function により、関数定義が初めて呼び出された際に生成されます。 2. **検証**: 実行時に pg_proc の xmin や TID を比較し、定義が変更されている場合は再コンパイルまたは無効化が行われます。 3. **実行管理**: ユーザーIDごとに異なるインタプリタ（interp_desc）を保持し、セキュリティ境界を維持します。 4. **メモリ管理**: 関数固有のメモリコンテキスト（fn_cxt）を使用し、実行終了後のクリーンアップを確実に行います。

呼び出し元と呼び出し先

  • **呼び出し元**: compile_pltcl_function（生成元）、および pltcl_call_handler 等の実行ハンドラ。
  • **呼び出し先**: 構造体内部で保持する FmgrInfo を介して、PostgreSQL の型変換関数を呼び出します。
  • ※具体的な呼び出し関係の詳細は、ソースコード内の参照箇所（src/pl/tcl/pltcl.c:290, 805, 1062 等）から推測されますが、完全なコールグラフは提供されていません。

関連データ構造

  • pltcl_interp_desc: 実行に使用する Tcl インタプリタの状態。
  • FmgrInfo: PostgreSQL の関数マネージャ情報。
  • pltcl_proc_key: キャッシュ検索用のキー構造体。

失敗モードと境界条件

  • internal_proname が NULL の場合、当該プロシージャは削除済みとみなされます。
  • pg_proc の行が更新された場合、既存のキャッシュは無効となり、再コンパイルが必要です。
  • メモリ不足時には fn_cxt の割り当てや管理においてエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はハッシュテーブルの直接の要素ではなく、pltcl_proc_key を介して間接的に参照されます。これは、異なるユーザーIDに対して同一の関数OIDでも個別のキャッシュエントリを持つ必要があるためです。
  • fn_cxt のメモリリークを防ぐため、構造体の破棄時には必ず関連するメモリコンテキストを適切にリセットしてください。
