plpgsql_getdiag_kindname

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_funcs.c:299

目的

GET DIAGNOSTICS 文で使用される診断項目の識別子（列挙型）を、デバッグやログ出力に適した人間可読な文字列に変換します。

シグネチャまたは形状

const char *plpgsql_getdiag_kindname(PLpgSQL_getdiag_kind kind)

入力と出力

  • **入力**: PLpgSQL_getdiag_kind 型の列挙値 kind。
  • **出力**: 対応する診断項目名を表す const char * 型の文字列。

前提条件と不変条件

  • 入力された kind が定義済みの列挙値のいずれかである場合、対応する定数文字列を返します。
  • 定義されていない値が渡された場合、文字列 "unknown" を返します。

アルゴリズムまたは振る舞い

switch 文を用いて kind を評価し、以下のマッピングに従って文字列を返します。

  • ROW_COUNT
  • PG_ROUTINE_OID
  • PG_CONTEXT
  • PG_EXCEPTION_CONTEXT
  • PG_EXCEPTION_DETAIL
  • PG_EXCEPTION_HINT
  • RETURNED_SQLSTATE
  • COLUMN_NAME
  • CONSTRAINT_NAME
  • PG_DATATYPE_NAME
  • MESSAGE_TEXT
  • TABLE_NAME
  • SCHEMA_NAME

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpgsql/src/pl_funcs.c:1586 (直接の呼び出しを確認)
  • **呼び出し先**:
  • 本関数は内部で他の関数を呼び出しません。

関連データ構造

  • PLpgSQL_getdiag_kind: 診断項目の種類を定義する列挙型（plpgsql.h で定義）。

失敗モードと境界条件

  • 未定義の kind 値が渡された場合、プログラムの異常終了ではなく "unknown" を返す安全な設計となっています。

保守担当者向けメモ

  • 新しい診断項目が PLpgSQL_getdiag_kind に追加された場合は、本関数の switch 文にも必ず case を追加してください。
