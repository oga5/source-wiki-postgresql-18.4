byteanlike

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:346

目的

バイナリデータ（bytea）に対して NOT LIKE 演算を実行し、パターンに一致しない場合に真を返します。

シグネチャまたは形状

Datum byteanlike(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BYTEA_PP(0): 検索対象の bytea データ。
  • PG_GETARG_BYTEA_PP(1): パターンとして使用する bytea データ。
  • **出力**:
  • PG_RETURN_BOOL(result): パターンマッチングの結果が LIKE_TRUE ではない場合に true を返します。

前提条件と不変条件

  • 入力データは bytea 型として適切に展開（detoast）されている必要があります。
  • SB_MatchText 関数がシングルバイトのパターンマッチングを正しく処理できる環境である必要があります。

アルゴリズムまたは振る舞い

1. VARDATA_ANY および VARSIZE_ANY_EXHDR を使用して、対象文字列とパターンのデータポインタおよび長さを取得します。 2. SB_MatchText を呼び出し、対象文字列とパターンを比較します。 3. SB_MatchText の戻り値が LIKE_TRUE ではない（一致しない）場合、結果を true とします。 4. 最終的なブール値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL 演算子 NOT LIKE（bytea 型用）から呼び出されます。
  • **呼び出し先**:
  • SB_MatchText: 実際のパターンマッチングアルゴリズムを実行します。
  • PG_GETARG_BYTEA_PP, VARDATA_ANY, VARSIZE_ANY_EXHDR, PG_RETURN_BOOL: PostgreSQL の関数インターフェースマクロ。

関連データ構造

  • bytea: PostgreSQL の可変長バイナリデータ型。

失敗モードと境界条件

  • 入力データが NULL の場合、PG_GETARG_BYTEA_PP マクロの挙動に従い、呼び出し元で NULL 処理が行われない限りエラーとなる可能性があります。
  • SB_MatchText の戻り値が LIKE_TRUE 以外（LIKE_FALSE または LIKE_STRANGE）の場合、本関数は true を返します。

保守担当者向けメモ

  • 本関数は bytea 型専用の NOT LIKE 実装です。
  • SB_MatchText は like_match.c をインクルードすることで生成される関数であり、このファイル内のマクロ定義に依存しています。修正時は like_match.c のロジックと整合性を確認してください。
