termPQExpBuffer

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.c:128

目的

動的に確保された PQExpBuffer 構造体のメモリを解放し、バッファを安全な空状態にリセットする。

シグネチャまたは形状

void termPQExpBuffer(PQExpBuffer str)

入力と出力

  • **入力**: str (PQExpBuffer型) - 解放および初期化対象のバッファ構造体へのポインタ。
  • **出力**: なし（副作用として str が指す構造体の内容が更新される）。

前提条件と不変条件

  • str は有効な PQExpBuffer 構造体を指している必要がある。
  • str->data が oom_buffer（メモリ不足時に割り当てられる静的バッファ）を指している場合、free() は呼び出されない。

アルゴリズムまたは振る舞い

1. str->data が oom_buffer と等しくない場合、free(str->data) を実行して動的メモリを解放する。 2. str->data を oom_buffer_ptr（静的な空文字列）で再初期化する。 3. str->maxlen を 0 に設定する。 4. str->len を 0 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c
  • src/bin/initdb/initdb.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • **呼び出し先**:
  • free() (標準ライブラリ)

関連データ構造

  • PQExpBufferData: 本関数が操作する対象の構造体。
  • oom_buffer: メモリ不足時に使用される静的バッファ。

失敗モードと境界条件

  • str が NULL の場合、プログラムはクラッシュする可能性がある（本関数内でのチェックは存在しない）。
  • str->data が既に解放済みである場合、二重解放（double free）が発生する可能性がある。

保守担当者向けメモ

  • 本関数はメモリの解放を行うため、呼び出し側で destroyPQExpBuffer の代わりに使用されることがある。
  • oom_buffer を参照しているため、メモリ不足状態からの復帰やクリーンアップにおいて一貫性が保たれている。
