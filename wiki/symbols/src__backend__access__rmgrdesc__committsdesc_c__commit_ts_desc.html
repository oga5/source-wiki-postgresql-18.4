commit_ts_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/committsdesc.c:20

目的

Commit Timestamp機能に関連するWALレコードの内容を解析し、人間が読み取り可能な形式に変換してバッファへ出力する。

シグネチャまたは形状

void commit_ts_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 出力先となる StringInfo バッファ。
  • record: 解析対象のWALレコード状態を保持する XLogReaderState 構造体。
  • **出力**: なし（buf に解析結果が追記される）。

前提条件と不変条件

  • record は有効なWALレコードを指していること。
  • XLogRecGetInfo で取得される情報ビットから XLR_INFO_MASK が除去された値が、COMMIT_TS_ZEROPAGE または COMMIT_TS_TRUNCATE のいずれかであること。

アルゴリズムまたは振る舞い

1. XLogRecGetData を用いてレコードの生データポインタを取得する。 2. XLogRecGetInfo から操作タイプ（info）を特定する。 3. info が COMMIT_TS_ZEROPAGE の場合:

  • レコードデータからページ番号（int64）を読み取り、buf に追加する。

    • 4. info が COMMIT_TS_TRUNCATE の場合:

  • レコードデータを xl_commit_ts_truncate 構造体にキャストする。
  • 構造体内の pageno と oldestXid を buf に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されているリソースマネージャインターフェース（PG_RMGR）を通じて、WAL解析ツール（pg_waldump等）から呼び出される。
  • **呼び出し先**: XLogRecGetData、XLogRecGetInfo、appendStringInfo、memcpy。

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • XLogReaderState: WALレコード読み取り状態。
  • xl_commit_ts_truncate: COMMIT_TS_TRUNCATE レコードのペイロード構造体。

失敗モードと境界条件

  • info が定義された2つの値以外である場合、何も出力されず関数が終了する。
  • レコードデータが構造体の期待するサイズを満たさない場合、メモリの不正アクセスが発生する可能性がある（呼び出し元でレコードの整合性が保証されていることが前提）。

保守担当者向けメモ

  • 本関数はWALの表示のみを目的としており、更新ロジックは含まない。
  • 新しいWAL操作タイプが commit_ts に追加された場合は、本関数の if-else 文を拡張する必要がある。
