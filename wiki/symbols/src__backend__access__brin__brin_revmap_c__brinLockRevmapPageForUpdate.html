brinLockRevmapPageForUpdate

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/brin/brin_revmap.c:133

目的

指定されたヒープブロックに対応するBRINリバースレンジマップ（revmap）のページをバッファに読み込み、排他ロックを取得して更新可能な状態にする。

シグネチャまたは形状

Buffer brinLockRevmapPageForUpdate(BrinRevmap *revmap, BlockNumber heapBlk)

入力と出力

  • **入力**:
  • revmap: BrinRevmap 構造体へのポインタ。
  • heapBlk: 対象となるヒープブロック番号。
  • **出力**:
  • Buffer: ロックが取得されたrevmapページのバッファハンドル。

前提条件と不変条件

  • revmap は brinRevmapInitialize によって適切に初期化されている必要がある。
  • 呼び出し元は、この関数が返すバッファに対して、後続の処理で適切にロックの解放（UnlockReleaseBuffer等）を行う責任を負う。

アルゴリズムまたは振る舞い

1. revmap_get_buffer を呼び出し、指定されたヒープブロックに対応するrevmapページをバッファに読み込む。 2. 取得したバッファに対して LockBuffer を呼び出し、BUFFER_LOCK_EXCLUSIVE（排他ロック）を適用する。 3. ロック済みのバッファハンドルを呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_pageops.c (複数箇所)
  • src/backend/access/brin/brin_revmap.c:350
  • **呼び出し先**:
  • revmap_get_buffer
  • LockBuffer

関連データ構造

  • BrinRevmap: revmapのアクセス状態を保持する構造体。
  • Buffer: PostgreSQLのバッファ管理用ハンドル。

失敗モードと境界条件

  • revmap_get_buffer が内部でエラーを発生させた場合、本関数もそのエラーを伝播させる。
  • 指定されたヒープブロックがインデックスの範囲外である場合、revmap_get_buffer の挙動に依存してエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は排他ロックを取得するため、デッドロックを避けるため呼び出し順序に注意すること。
  • BrinRevmap 構造体はスレッドセーフではないため、単一バックエンド内での使用を前提としている。
