suppress_redundant_updates_trigger

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/trigfuncs.c:27

目的

UPDATE 操作において、更新前後のタプルが物理的に同一である場合に更新処理をスキップ（抑制）し、不要なインデックス更新やWAL書き込みを回避する。

シグネチャまたは形状

Datum suppress_redundant_updates_trigger(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（トリガー実行コンテキスト TriggerData を含む）。
  • **出力**: Datum 型。更新を続行する場合は新しいタプル、更新を抑制する場合は NULL を返す。

前提条件と不変条件

  • BEFORE UPDATE かつ FOR EACH ROW のトリガーとして呼び出される必要がある。
  • 上記条件を満たさない場合、ERRCODE_E_R_I_E_TRIGGER_PROTOCOL_VIOLATED エラーが発生する。

アルゴリズムまたは振る舞い

1. **検証**: fcinfo から TriggerData を取得し、トリガーの呼び出し条件（イベント種別、タイミング、行単位実行）を検証する。 2. **比較**: 更新前（tg_trigtuple）と更新後（tg_newtuple）のタプルを以下の項目で比較する。

  • タプル長 (t_len)
  • ヘッダーオフセット (t_hoff)
  • 属性数 (natts)
  • トランザクションマスク（HEAP_XACT_MASK を除外した t_infomask）
  • タプルペイロード（memcmp によるバイト単位の比較）

    • 3. **決定**: 全てが一致する場合、戻り値を NULL に設定し、更新を中止する。不一致の場合は更新を続行する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のトリガー実行エンジン（src/backend/commands/trigger.c 等から間接的に呼び出される）。
  • **呼び出し先**:
  • ereport: エラー発生時に使用。
  • memcmp: タプルペイロードの比較に使用。
  • PointerGetDatum: 戻り値の変換に使用。

関連データ構造

  • TriggerData: commands/trigger.h で定義。トリガー実行時のコンテキスト情報。
  • HeapTuple: access/htup_details.h で定義。タプルの物理構造。

失敗モードと境界条件

  • **プロトコル違反**: UPDATE 以外のイベントや、BEFORE 以外のタイミング、あるいは文単位のトリガーとして呼び出された場合にエラーとなる。
  • **物理的同一性**: 本関数は「物理的な」同一性を判定する。論理的に等価であっても、NULLビットマップやヘッダー情報の差異により物理表現が異なる場合は更新が実行される。

保守担当者向けメモ

  • memcmp を使用しているため高速だが、タプルの物理レイアウトに強く依存する。
  • 呼び出し元情報の詳細は、PostgreSQL のトリガー実行エンジン（src/backend/commands/trigger.c）のソースコードを参照のこと。
