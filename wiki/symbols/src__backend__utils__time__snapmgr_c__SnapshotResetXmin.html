SnapshotResetXmin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/snapmgr.c:934

目的

保持されているスナップショット群に基づき、現在のバックエンドの MyProc->xmin および TransactionXmin を再計算し、VACUUMが削除可能な行の境界を適切に更新する。

シグネチャまたは形状

static void SnapshotResetXmin(void)

入力と出力

  • 入力: なし（グローバルな ActiveSnapshot および RegisteredSnapshots を参照）
  • 出力: なし（副作用として MyProc->xmin および TransactionXmin を更新）

前提条件と不変条件

  • ActiveSnapshot が存在する場合、この関数は即座に終了する（アクティブなスナップショットがある間はXminを後退させないため）。
  • RegisteredSnapshots はペアリングヒープとして管理されており、その先頭要素が最も古い xmin を持つスナップショットであると仮定される。

アルゴリズムまたは振る舞い

1. ActiveSnapshot が非NULLであれば、何もせず終了する。 2. RegisteredSnapshots が空の場合、MyProc->xmin と TransactionXmin を InvalidTransactionId にリセットする。 3. RegisteredSnapshots が空でない場合、ヒープ内の最小の xmin を取得する。 4. 現在の MyProc->xmin が取得した minSnapshot->xmin よりも古い（TransactionIdPrecedes が真）場合、MyProc->xmin および TransactionXmin をその値に更新する。

呼び出し元と呼び出し先

  • 呼び出し元: snapmgr.c 内の複数の箇所（スナップショットの登録解除やライフサイクル管理に関連する処理）から呼び出される。
  • 呼び出し先: pairingheap_is_empty, pairingheap_first, pairingheap_container, TransactionIdPrecedes を使用。

関連データ構造

  • RegisteredSnapshots: 登録済みスナップショットを管理するペアリングヒープ。
  • ActiveSnapshot: 現在アクティブなスナップショットのスタック。
  • MyProc: 現在のバックエンドのプロセス情報（xmin を保持）。
  • SnapshotData: スナップショットの構造体。

失敗モードと境界条件

  • RegisteredSnapshots が空の場合、xmin は InvalidTransactionId に設定される。
  • ActiveSnapshot が存在する場合、他のスナップショットが登録されていても xmin の更新はスキップされる。

保守担当者向けメモ

  • 本関数はVACUUMの進行に直接影響を与えるため、スナップショットの解放処理と密接に連携している。
  • TransactionIdPrecedes による比較は、xmin が進む（より新しい値になる）場合のみ更新を行う安全策である。
