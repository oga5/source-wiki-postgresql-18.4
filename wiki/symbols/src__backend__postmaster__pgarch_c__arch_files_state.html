arch_files_state

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/pgarch.c:124

目的

WALアーカイバプロセスにおいて、ディレクトリ走査によって取得したアーカイブ対象ファイルの一時的な管理と、優先度付きキューによる処理順序の制御を行うための状態構造体です。

シグネチャまたは形状

struct arch_files_state
{
    binaryheap *arch_heap;
    int         arch_files_size;
    char       *arch_files[NUM_FILES_PER_DIRECTORY_SCAN];
    char        arch_filenames[NUM_FILES_PER_DIRECTORY_SCAN][MAX_XFN_CHARS + 1];
};

入力と出力

  • 入力: pg_wal/archive_status ディレクトリ内の .ready ファイル群。
  • 出力: arch_heap に格納された優先度順のファイルリスト、および arch_files 配列による処理対象の管理。

前提条件と不変条件

  • arch_files および arch_filenames は NUM_FILES_PER_DIRECTORY_SCAN で定義された固定長配列です。
  • arch_files_size は、現在処理対象となっている有効なエントリ数を保持します。

アルゴリズムまたは振る舞い

1. ディレクトリ走査時にファイル名を arch_filenames に格納します。 2. arch_files 配列は arch_filenames 内の各エントリを指し示します。 3. binaryheap を使用して、タイムライン履歴ファイルなどを優先したアーカイブ順序を決定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/pgarch.c 内で静的変数として定義され、palloc を介してメモリ確保が行われます。
  • 呼び出し先: 具体的な関数呼び出し関係は提供されたソース情報からは特定できません。

関連データ構造

  • binaryheap: アーカイブ対象ファイルの優先度付きキューとして使用されます。
  • NUM_FILES_PER_DIRECTORY_SCAN: 配列の最大サイズを規定する定数。

失敗モードと境界条件

  • NUM_FILES_PER_DIRECTORY_SCAN を超えるファイルがディレクトリ内に存在する場合の挙動については、本構造体の定義からは明示されていません。

保守担当者向けメモ

  • 本構造体は pgarch.c 内の静的変数として管理されています。メモリ割り当ては palloc を使用しているため、プロセスのライフサイクルに応じたメモリ管理に注意してください。
