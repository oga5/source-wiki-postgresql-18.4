Vector8

定義

  • 種別: typedef
  • 定義箇所: src/include/port/simd.h:33

目的

プラットフォーム非依存な SIMD 操作を実現するための抽象化されたベクトル型です。ハードウェア固有の SIMD 命令セット（x86_64 の SSE2 や ARM64 の NEON など）を透過的に利用可能にします。

シグネチャまたは形状

typedef __m128i Vector8;

※上記は x86_64 環境における定義例です。プラットフォームに応じて適切な SIMD 型にマッピングされます。

入力と出力

  • 入力: SIMD レジスタとしてメモリからロードされたデータ。
  • 出力: ベクトル演算の結果を保持する型。

前提条件と不変条件

  • Vector8 のサイズはプラットフォーム依存です。サイズに依存する処理を行う場合は、ハードコードせず sizeof(Vector8) を使用する必要があります。
  • コンパイル時に USE_SSE2、USE_NEON、または USE_NO_SIMD のいずれかの機能フラグが適用されます。

アルゴリズムまたは振る舞い

  • SIMD 命令セットを利用可能な環境では、ハードウェア並列処理を直接活用します。
  • SIMD 非対応環境では、ビット演算や標準的な整数演算によるフォールバック実装が提供されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/utils/adt/json.c: JSON 処理における高速な文字列検索ループ内で sizeof(Vector8) が使用されています。
  • src/common/jsonapi.c: pg_lfind8 と組み合わせて、バッファ内の特定文字検索に使用されています。
  • 呼び出し先:
  • 本シンボル自体は型定義であり、関数ではないため呼び出し先はありません。ただし、vector8_load や vector8_has などの関連関数が本型を操作対象として利用します。

関連データ構造

  • Vector32: 32 ビット要素を扱うための対となる抽象型。
  • pg_bitutils.h: 低レイヤーのデータ処理において併用されるビット操作ユーティリティ。

失敗モードと境界条件

  • SIMD が利用できない環境では、一部の複雑な演算が制限される場合があります。
  • Vector8 のサイズが 2 のべき乗であることを前提としたコード（json.c 等）が存在するため、移植先プラットフォームでサイズが異なる場合は注意が必要です。

保守担当者向けメモ

  • USE_ASSERT_CHECKING を有効にすると、SIMD 演算の正当性を逐次処理で検証するチェックが挿入されます。開発時のデバッグには有用ですが、本番環境では無効にしてください。
  • Vector32 と異なり、Vector8 は多くのプラットフォームでサポートされていますが、実装の詳細は simd.h 内の #if ディレクティブによる分岐に依存しています。
