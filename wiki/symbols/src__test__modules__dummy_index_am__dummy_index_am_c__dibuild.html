dibuild

定義

  • 種別: function
  • 定義箇所: src/test/modules/dummy_index_am/dummy_index_am.c:136

目的

PostgreSQLのインデックス・アクセスメソッド（AM）におけるインデックス構築処理のスタブ実装です。実際のデータ構築は行わず、構築結果を報告するインターフェースの動作を実証します。

シグネチャまたは形状

IndexBuildResult *dibuild(Relation heap, Relation index, IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • heap: 対象となるヒープリレーション。
  • index: 構築対象のインデックスリレーション。
  • indexInfo: インデックスのメタ情報。
  • **出力**:
  • IndexBuildResult *: 構築結果を格納した構造体。heap_tuples および index_tuples を 0 に初期化して返します。

前提条件と不変条件

  • palloc を使用して IndexBuildResult 構造体分のメモリを確保します。
  • 実際のタプルスキャンやインデックス作成は行われません。

アルゴリズムまたは振る舞い

1. IndexBuildResult 構造体用のメモリを割り当てます。 2. heap_tuples を 0 に設定し、スキャンされたタプルがないことを示します。 3. index_tuples を 0 に設定し、作成されたインデックスエントリがないことを示します。 4. 構築結果を格納したポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのインデックス構築プロセス（IndexAmRoutine を介して呼び出されます）。
  • **呼び出し先**: palloc（メモリ確保）。

関連データ構造

  • IndexBuildResult: インデックス構築の結果を保持する構造体。
  • IndexAmRoutine: 本関数が登録されるAMインターフェース構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 本関数は常に成功を前提としたスタブとして動作します。

保守担当者向けメモ

  • 本関数は dummy_index_am のテンプレートの一部です。実際のインデックスAMを実装する際は、この関数内で適切なインデックス構築ロジックを実装してください。
  • dihandler 関数内で amroutine->ambuild = dibuild; として登録されています。
