ParentMapEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gist/gistbuild.c:1509

目的

GiSTインデックス構築時において、子ブロック番号から親ブロック番号へのマッピングを保持し、インデックス構造の親子関係を追跡するために使用されます。

シグネチャまたは形状

typedef struct {
    BlockNumber childblkno;  /* ハッシュキー */
    BlockNumber parentblkno;
} ParentMapEntry;

入力と出力

  • **入力:** ハッシュテーブルのキーとして childblkno を使用します。
  • **出力:** 対応する parentblkno を保持します。

前提条件と不変条件

  • gistbuild 処理中のハッシュテーブル（buildstate->parentMap）内でエントリとして管理されます。
  • ハッシュテーブルの各エントリサイズは sizeof(ParentMapEntry) として初期化される必要があります。

アルゴリズムまたは振る舞い

  • hash_search 関数を通じて、特定の childblkno に対応する親ブロック情報を検索または挿入するために使用されます。
  • インデックス構築プロセスにおいて、子ノードがどの親ノードに属しているかを特定するルックアップテーブルの要素として機能します。

呼び出し元と呼び出し先

  • **呼び出し元:** src/backend/access/gist/gistbuild.c 内の関数群（hash_search を呼び出す箇所）。
  • **呼び出し先:** 情報なし。

関連データ構造

  • GISTBuildState: この構造体が保持する parentMap（ハッシュテーブル）の要素として使用されます。

失敗モードと境界条件

  • ハッシュテーブルのメモリ不足や、検索対象の childblkno が存在しない場合の hash_search の挙動に依存します。

保守担当者向けメモ

  • 本構造体は GiST のバッファリング構築アルゴリズムに関連しています。ハッシュテーブルのキーとして childblkno が使用されているため、変更時はハッシュ関数の整合性に注意してください。
