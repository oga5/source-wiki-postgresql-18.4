hash_string

定義

  • 種別: function
  • 定義箇所: src/include/common/hashfn_unstable.h:390

目的

NUL終端文字列から、メモリ内ハッシュテーブル等で使用するための高速な32ビットハッシュ値を生成します。

シグネチャまたは形状

static inline uint32 hash_string(const char *s)

入力と出力

  • **入力**: const char *s（ハッシュ対象のNUL終端文字列）
  • **出力**: uint32（計算されたハッシュ値）

前提条件と不変条件

  • 入力文字列 s は有効なNUL終端文字列である必要があります。
  • 本関数は「不安定」なハッシュアルゴリズム（fasthash）を使用しており、プラットフォームやPostgreSQLのバージョン間で結果が一致する保証はありません。永続化データには使用しないでください。

アルゴリズムまたは振る舞い

1. fasthash_init を呼び出し、ハッシュ状態 hs をシード値 0 で初期化します。 2. fasthash_accum_cstring を呼び出し、文字列 s をハッシュ状態に蓄積します。この際、文字列の長さが計測され、戻り値として保持されます。 3. fasthash_final32 を呼び出し、蓄積されたデータと文字列長（tweak値として使用）を組み合わせて最終的な32ビットハッシュ値を算出・返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: fasthash_init, fasthash_accum_cstring, fasthash_final32
  • **呼び出し元**: 以下のマクロ定義を通じて利用されています。
  • src/bin/pg_combinebackup/load_manifest.c:51
  • src/bin/pg_dump/pg_dumpall.c:48
  • src/bin/pg_rewind/filemap.c:46, 78
  • src/bin/pg_verifybackup/pg_verifybackup.h:51

関連データ構造

  • fasthash_state: ハッシュ計算の進行状況を保持するために内部で使用されます。

失敗モードと境界条件

  • 入力文字列が非常に長い場合、fasthash_accum_cstring の処理時間に比例して計算コストが増大します。
  • 本関数はプラットフォームのエンディアンの影響を受けるため、異なるアーキテクチャ間でのハッシュ値の互換性はありません。

保守担当者向けメモ

  • 本関数は fasthash アルゴリズムのラッパーです。アルゴリズムの変更や調整が必要な場合は、fasthash 関連の基盤関数を確認してください。
  • SMHasher テストをパスするため、文字列長を最終的なハッシュ計算の tweak として利用する設計になっています。この仕様を変更する際は注意してください。
