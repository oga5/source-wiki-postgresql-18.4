PredIterInfoData

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/predtest.c:59

目的

述語（Predicate）の論理構造（AND/OR）を抽象化し、式ツリーを走査するためのイテレータ機構を提供します。

シグネチャまたは形状

typedef struct PredIterInfoData
{
    void       *state;
    List       *state_list;
    void        (*startup_fn) (Node *clause, PredIterInfo info);
    Node       *(*next_fn) (PredIterInfo info);
    void        (*cleanup_fn) (PredIterInfo info);
} PredIterInfoData;

入力と出力

  • **入力**: startup_fn を介して対象の Node *clause を受け取ります。
  • **出力**: next_fn を介して、走査対象となる次の Node * を順次返します。

前提条件と不変条件

  • startup_fn はイテレーション開始前に呼び出される必要があります。
  • cleanup_fn はイテレーション終了後にリソースを解放するために呼び出される必要があります。
  • state および state_list は、特定のノード型に応じた走査状態を保持するために使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: startup_fn により、対象となる述語式の構造に基づいたイテレータ状態を構築します。 2. **走査**: next_fn を繰り返し呼び出すことで、論理構造（AND/OR 等）に含まれる個々の節を順次取得します。 3. **終了処理**: cleanup_fn を呼び出し、イテレーション中に確保したリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/predtest.c 内の関数（predicate_implied_by_recurse や predicate_refuted_by_recurse 等）において、ローカル変数としてインスタンス化され使用されています。
  • **呼び出し先**: 構造体内の関数ポインタ（startup_fn, next_fn, cleanup_fn）が、具体的なノード型に応じた処理関数を指し示します。

関連データ構造

  • PredIterInfo: PredIterInfoData へのポインタ型（typedef struct PredIterInfoData *PredIterInfo;）。

失敗モードと境界条件

  • 複雑な AND/OR 構造を持つ式を処理する場合、再帰的な走査によりスタックオーバーフローが発生する可能性があります。
  • startup_fn 等の関数ポインタが適切に初期化されていない状態で next_fn 等を呼び出した場合の挙動は未定義です。

保守担当者向けメモ

  • 本構造体は predicate_classify によって分類されたノード型に対して、統一的なインターフェースを提供します。新しい論理ノード型を追加する場合は、対応する startup_fn, next_fn, cleanup_fn を実装する必要があります。
