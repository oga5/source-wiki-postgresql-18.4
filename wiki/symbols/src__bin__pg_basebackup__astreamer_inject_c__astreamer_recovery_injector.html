astreamer_recovery_injector

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/astreamer_inject.c:18

目的

pg_basebackup が生成するアーカイブストリームに対し、リカバリ設定（recovery.conf または postgresql.auto.conf）を動的に挿入・編集するための状態保持用構造体です。

シグネチャまたは形状

typedef struct astreamer_recovery_injector
{
    astreamer    base;
    bool         skip_file;
    bool         is_recovery_guc_supported;
    bool         is_postgresql_auto_conf;
    bool         found_postgresql_auto_conf;
    PQExpBuffer  recoveryconfcontents;
    astreamer_member member;
} astreamer_recovery_injector;

入力と出力

本構造体は astreamer フレームワークのパイプラインの一部として機能します。

  • **入力**: アーカイブストリーム内の各メンバー情報（astreamer_member）およびファイルコンテンツ。
  • **出力**: 処理対象のリカバリ設定ファイルがスキップされたストリーム、または最終的に注入されたリカバリ設定を含むストリーム。

前提条件と不変条件

  • astreamer フレームワークの基底構造体 base を先頭に保持する必要があります。
  • is_recovery_guc_supported フラグにより、PostgreSQL 12 以降（postgresql.auto.conf）とそれ以前（recovery.conf）の処理ロジックが分岐します。

アルゴリズムまたは振る舞い

1. **初期化**: astreamer_recovery_injector_new によりインスタンス化されます。 2. **コンテンツ処理**:

  • ASTREAMER_MEMBER_HEADER 受信時、ファイルパスを検査し、リカバリ設定ファイルであれば skip_file を真に設定します。
  • ASTREAMER_MEMBER_TRAILER 受信時、postgresql.auto.conf であれば保持していた設定内容を追記します。
  • ASTREAMER_ARCHIVE_TRAILER 受信時、未注入のリカバリ設定や standby.signal をストリームに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_recovery_injector_new（初期化時）、およびストリーム処理関数群（astreamer_recovery_injector_content 等）。
  • **呼び出し先**: astreamer_inject_file（リカバリ設定の注入時）。
  • 補足: 呼び出し関係の詳細は、提供されたソースコードの断片からは限定的であり、完全なコールグラフは不明です。

関連データ構造

  • astreamer: 本構造体が継承する基底ストリーマー。
  • astreamer_member: ストリーム内のファイル情報を保持する構造体。
  • PQExpBuffer: リカバリ設定の内容をバッファリングするために使用。

失敗モードと境界条件

  • **バージョン依存**: is_recovery_guc_supported の設定が誤っている場合、誤った設定ファイル名で処理が行われる可能性があります。
  • **ストリームの整合性**: 本インジェクタはヘッダーを直接操作しないため、後続のストリーマーがアーカイブ形式（tar等）を正しく再パッケージ化できない場合、ストリームが破損する可能性があります。

保守担当者向けメモ

  • astreamer_inject_file 内で使用される UID/GID は固定値です。
  • recovery.conf から postgresql.auto.conf への移行に伴うロジック分岐は is_recovery_guc_supported に依存しているため、変更時は注意してください。
