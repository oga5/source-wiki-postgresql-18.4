pg_strtoint32_safe

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numutils.c:388

目的

文字列を int32 型の整数値へ安全に変換します。エラー発生時に ereport で即座に中断するのではなく、escontext を介してエラー情報を伝播させることで、呼び出し元での制御を可能にします。

シグネチャまたは形状

int32 pg_strtoint32_safe(const char *s, Node *escontext);

入力と出力

  • **入力**:
  • s: 解析対象の文字列（NUL終端）。
  • escontext: エラー情報を格納するための Node 構造体（ErrorSaveContext 等）。
  • **出力**:
  • 変換後の int32 値。
  • **戻り値**: 成功時は変換された整数値。失敗時は escontext にエラーが記録され、呼び出し元へ制御が戻ります。

前提条件と不変条件

  • 入力文字列 s は有効なポインタである必要があります。
  • 変換結果が int32 の範囲（PG_INT32_MIN ～ PG_INT32_MAX）に収まる必要があります。
  • アンダースコア（_）は可読性のためのみに使用され、数値計算には影響しません。

アルゴリズムまたは振る舞い

1. **高速パス**: まず、単純な10進数（符号付き）として解析を試みます。 2. **低速パス**: 高速パスで解析できない場合（基数指定 0x, 0o, 0b やアンダースコアを含む場合、または先頭の空白など）、slow ラベルへジャンプし、詳細な構文解析を行います。 3. **基数処理**:

  • 0x/0X: 16進数
  • 0o/0O: 8進数
  • 0b/0B: 2進数
  • その他: 10進数

    • 4. **エラーチェック**: 各桁の処理ごとにオーバーフローをチェックし、不正な構文や範囲外の値が検出された場合は out_of_range または invalid_syntax ラベルへ遷移します。

呼び出し元と呼び出し先

  • **呼び出し元**:

関連データ構造

  • Node: エラーコンテキストの保持に使用。
  • hexlookup: 16進数文字から数値への変換用テーブル。

失敗モードと境界条件

  • **オーバーフロー**: int32 の範囲を超える値が入力された場合、out_of_range へ遷移します。
  • **不正な構文**: 数値として解釈できない文字や、アンダースコアの不適切な配置（例: 1__0）がある場合、invalid_syntax へ遷移します。
  • **空の入力**: 適切な数値を含まない文字列はエラーとして扱われます。

保守担当者向けメモ

  • 本関数はパフォーマンスを重視しており、単純な10進数ケースを高速パスで処理します。修正を行う際は、高速パスと低速パスの両方でオーバーフローチェックが漏れなく行われていることを確認してください。
  • escontext の利用により、エラー発生時の挙動が呼び出し元に依存するため、変更時は ErrorSaveContext の仕様を確認してください。
