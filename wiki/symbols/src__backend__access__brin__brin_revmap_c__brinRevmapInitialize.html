brinRevmapInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_revmap.c:69

目的

BRINインデックスのrevmap（リバースレンジマップ）アクセス状態を初期化し、メタページから現在のインデックス構成情報を読み込んでキャッシュする。

シグネチャまたは形状

BrinRevmap *brinRevmapInitialize(Relation idxrel, BlockNumber *pagesPerRange)

入力と出力

  • **入力**:
  • idxrel: 対象となるBRINインデックスのリレーション。
  • pagesPerRange: メタページから読み取った pagesPerRange を格納するための出力用ポインタ。
  • **出力**:
  • 初期化された BrinRevmap 構造体へのポインタ。

前提条件と不変条件

  • idxrel は有効なBRINインデックスであること。
  • メタページ（BRIN_METAPAGE_BLKNO）がインデックス内に存在すること。

アルゴリズムまたは振る舞い

1. BRIN_METAPAGE_BLKNO を読み込み、共有ロックを取得する。 2. メタページの内容（BrinMetaPageData）から pagesPerRange および lastRevmapPage を取得する。 3. BrinRevmap 構造体をメモリ確保し、以下の値を設定する:

  • rm_irel: インデックスリレーション。
  • rm_pagesPerRange: メタページから取得した値。
  • rm_lastRevmapPage: メタページから取得した値。
  • rm_metaBuf: メタページのバッファ。
  • rm_currBuf: InvalidBuffer で初期化。

    • 4. 出力引数 pagesPerRange にメタページから取得した値を書き込む。 5. メタページのロックを解除し、作成した BrinRevmap を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c (複数箇所)
  • src/backend/access/brin/brin_revmap.c:339
  • **呼び出し先**:
  • ReadBuffer
  • LockBuffer
  • BufferGetPage
  • PageGetContents
  • palloc

関連データ構造

  • BrinRevmap: revmapのアクセス状態を保持する構造体。
  • BrinMetaPageData: インデックスのメタデータ（pagesPerRange 等を保持）。

失敗モードと境界条件

  • メタページが読み込めない場合、ReadBuffer によりエラーが発生する。
  • メモリ不足時、palloc によりエラーが発生する。

保守担当者向けメモ

  • 本関数が返す BrinRevmap 構造体は、単一バックエンド内での効率化を目的としたキャッシュであり、スレッドセーフではない。
  • rm_metaBuf は保持されるが、呼び出し元で適切に終了処理（brinRevmapTerminate 等）が行われる必要がある。
