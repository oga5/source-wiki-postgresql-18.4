date_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/date.c:184

目的

DateADT 型の内部表現（エポックからの経過日数）を、PostgreSQL の DateStyle 設定に基づいた人間が読み取り可能な文字列形式に変換する。

シグネチャまたは形状

Datum date_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して DateADT 型の値を受け取る。
  • **出力**: Datum 型として、変換された日付文字列（cstring）を返す。

前提条件と不変条件

  • 入力値が有限の日付である場合、j2date を用いてユリウス日（POSTGRES_EPOCH_JDATE 加算済み）から年・月・日へ変換される。
  • 無限大（Infinity）などの特殊な日付値については、DATE_NOT_FINITE マクロで判定され、専用の処理が行われる。

アルゴリズムまたは振る舞い

1. PG_GETARG_DATEADT(0) で入力値を取得する。 2. DATE_NOT_FINITE マクロで特殊値か判定する。

  • 特殊値の場合: EncodeSpecialDate を呼び出し、buf に文字列を格納する。
  • 通常値の場合: j2date を用いて内部表現を pg_tm 構造体に分解し、EncodeDateOnly を呼び出して DateStyle に従った文字列を buf に生成する。

    • 3. pstrdup(buf) でメモリを確保し、結果をコピーする。 4. PG_RETURN_CSTRING で結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c:5140 (直接的な呼び出し)
  • src/backend/utils/adt/json.c:324 (JSON 変換時の参照)
  • **呼び出し先**:
  • EncodeSpecialDate
  • j2date
  • EncodeDateOnly
  • pstrdup

関連データ構造

  • DateADT: int32 で定義される日付の内部表現。
  • pg_tm: 日付フィールド（年・月・日）を保持する構造体。

失敗モードと境界条件

  • MAXDATELEN を超える文字列は生成されない前提で buf が確保されている。
  • pstrdup によるメモリ割り当てが失敗した場合、PostgreSQL のエラーハンドリング機構により例外が発生する。

保守担当者向けメモ

  • 本関数は DateStyle に依存するため、出力形式はグローバル設定によって変化する。
  • DATE_NOT_FINITE の判定ロジックや j2date の変換ロジックを変更する場合は、src/backend/utils/adt/datetime.c の関連ルーチンとの整合性に注意すること。
