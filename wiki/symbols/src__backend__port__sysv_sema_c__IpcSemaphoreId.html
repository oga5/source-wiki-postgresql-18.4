IpcSemaphoreId

定義

  • 種別: typedef int
  • 定義箇所: src/backend/port/sysv_sema.c:46

目的

System V セマフォ API の semget(2) システムコールによって返されるセマフォ識別子を保持するために使用されます。

シグネチャまたは形状

int 型のエイリアスです。

入力と出力

  • **入力**: semget システムコールの戻り値として生成されます。
  • **出力**: セマフォセットに対する操作（初期化、削除、値の取得、PID の取得）を行う各関数の引数として渡されます。

前提条件と不変条件

  • 本シンボルは、OS のカーネルパラメータ（SEMMNI 等）の制限を受ける System V セマフォセットを指し示します。
  • 識別されたセマフォセットには、PostgreSQL が内部的に使用する識別用セマフォが含まれている必要があります。

アルゴリズムまたは振る舞い

本シンボル自体は識別子を保持する型であり、具体的な振る舞いは semop や semctl を呼び出す関数群に依存します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • InternalIpcSemaphoreCreate
  • IpcSemaphoreInitialize
  • IpcSemaphoreKill
  • IpcSemaphoreGetValue
  • IpcSemaphoreGetLastPID
  • IpcSemaphoreCreate
  • 静的変数 mySemaSets（取得済みのセマフォセット ID 配列）
  • **呼び出し先**:
  • 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • mySemaSets: 取得済みの IpcSemaphoreId を格納する配列。
  • PGSemaphoreData: セマフォセット ID とセット内のインデックスを保持する構造体。

失敗モードと境界条件

  • semget が失敗した場合、無効な ID が返される可能性があります。
  • カーネルのセマフォセット制限に達した場合、新しい ID の割り当てが失敗します。

保守担当者向けメモ

  • 本シンボルは System V セマフォ実装に特化した型です。POSIX セマフォや Win32 セマフォの実装では使用されません。
  • semget の戻り値が負の値をとる場合や、システムコールエラー時の挙動については、各関数（InternalIpcSemaphoreCreate 等）の実装を確認してください。
