AllocSetFreeIndex

定義

  • 種別: static inline function
  • 定義箇所: src/backend/utils/mmgr/aset.c:276

目的

メモリ割り当て要求サイズに基づき、AllocSet における適切なフリーリストのインデックスを計算する。

シグネチャまたは形状

static inline int AllocSetFreeIndex(Size size)

入力と出力

  • **入力**: Size size（要求されたメモリサイズ）
  • **出力**: int（フリーリストのインデックス。0 以上 ALLOCSET_NUM_FREELISTS 未満）

前提条件と不変条件

  • ALLOC_CHUNK_LIMIT は 64kB 未満である必要がある（StaticAssertDecl による検証）。
  • 計算されたインデックスは常に ALLOCSET_NUM_FREELISTS 未満であること（Assert による検証）。

アルゴリズムまたは振る舞い

1. size が (1 << ALLOC_MINBITS) 以下の場合、インデックスは 0 を返す。 2. それ以外の場合、size に基づいて ceil(log2(size >> ALLOC_MINBITS)) に相当する値を計算する。

  • HAVE_BITSCAN_REVERSE が定義されている環境では、pg_leftmost_one_pos32 を使用して高速に算出する。
  • 未定義の場合は、pg_leftmost_one_pos テーブルを用いたビット操作により算出する。

    • 3. この計算は、メモリ割り当てのホットスポットであるため、最適化が施されている。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/aset.c:850
  • src/backend/utils/mmgr/aset.c:1000
  • **呼び出し先**:
  • pg_leftmost_one_pos32 (環境依存)
  • pg_leftmost_one_pos (テーブル参照)

関連データ構造

  • AllocSetContext: 本関数がインデックスを決定するフリーリストを保持する構造体。
  • ALLOCSET_NUM_FREELISTS: インデックスの最大範囲を規定する定数。

失敗モードと境界条件

  • size が ALLOC_CHUNK_LIMIT を超えるような巨大な値の場合、本関数の前提（16ビット範囲内）を逸脱する可能性があるため注意が必要。

保守担当者向けメモ

  • 本関数は AllocSet におけるメモリ割り当てのパフォーマンスを左右するクリティカルパスである。
  • ビットスキャン命令の有無によって実装が分岐するため、変更時は両方のパスの整合性を確認すること。
