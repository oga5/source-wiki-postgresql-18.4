convert_testexpr_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/subselect.c:42

目的

サブクエリのテスト式（testexpr）を変換する際、式ツリーの走査中に保持すべき文脈情報を格納するために使用されます。

シグネチャまたは形状

typedef struct convert_testexpr_context
{
	PlannerInfo *root;
	List	   *subst_nodes;	/* Nodes to substitute for Params */
} convert_testexpr_context;

入力と出力

本構造体は関数呼び出しの引数として渡されるコンテキストオブジェクトです。

  • root: 現在のクエリレベルの PlannerInfo 構造体へのポインタ。
  • subst_nodes: Param ノードに置換されるべきノードのリスト。

前提条件と不変条件

  • root は有効な PlannerInfo を指している必要があります。
  • subst_nodes は、式ツリー内の特定のノードを Param に置き換えるためのマッピング情報として機能します。

アルゴリズムまたは振る舞い

サブクエリの最適化プロセスにおいて、式ツリーを再帰的に探索する際の「状態保持」の役割を果たします。具体的には、親クエリの条件式に含まれるノードをサブクエリの出力と接続するために、Param への置換処理を行う際の参照先として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/subselect.c 内の関数（convert_testexpr 等）でインスタンス化および参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • PlannerInfo: クエリ全体の計画生成に関する情報を保持する構造体。
  • List: PostgreSQL の汎用リスト構造。subst_nodes の保持に使用されます。

失敗モードと境界条件

  • root が NULL の場合、または subst_nodes が適切に初期化されていない状態で走査が行われた場合、セグメンテーションフォールトや不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は subselect.c 内の局所的な変換処理に特化しています。
  • subst_nodes のリスト構造がどのように構築され、どのタイミングで消費されるかについては、convert_testexpr 関数の実装詳細を参照してください。
