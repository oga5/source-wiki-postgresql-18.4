JumbleState

定義

  • **種別**: struct
  • **定義箇所**: src/include/nodes/queryjumble.h:38

目的

クエリの構造的同一性を識別するためのフィンガープリント（Query ID）生成プロセスにおいて、作業状態を保持するために使用されます。

シグネチャまたは形状

typedef struct JumbleState
{
    unsigned char *jumble;
    Size          jumble_len;
    LocationLen   *clocations;
    int           clocations_buf_size;
    int           clocations_count;
    int           highest_extern_param_id;
    bool          has_squashed_lists;
    unsigned int  pending_nulls;
#ifdef USE_ASSERT_CHECKING
    Size          total_jumble_len;
#endif
} JumbleState;

入力と出力

  • **入力**: クエリツリーの走査過程でノード情報や定数の位置情報が書き込まれます。
  • **出力**: 最終的に計算されたハッシュ値（Query ID）および、定数を除去して正規化するための位置情報配列（clocations）を提供します。

前提条件と不変条件

  • clocations 配列は、クエリ内の定数リテラルをマスクするために使用されます。
  • pending_nulls は、NULL ノードの連続を効率的に処理するために使用され、後続の追加処理や最終ハッシュ計算の前にフラッシュされます。
  • has_squashed_lists が真の場合、パラメータ番号の再割り当てが行われます。

アルゴリズムまたは振る舞い

1. クエリツリーの再帰的な走査中に、ノードの型や構造を jumble バッファに書き込みます。 2. 定数リテラルを検出した際、その位置と長さを clocations に記録します。 3. pending_nulls を用いて NULL ノードの連続を遅延処理し、バッファへの書き込みを最適化します。 4. 最終的なバッファ内容に基づき、クエリのフィンガープリントを計算します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_stat_statements/pg_stat_statements.c（pgss_post_parse_analyze 等）、src/backend/commands/explain.c、src/backend/commands/createas.c などから参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、具体的な関数呼び出しの詳細はソース抜粋からは特定できません。

関連データ構造

  • LocationLen: 定数の位置と長さを保持する構造体。
  • Query: nodes/parsenodes.h で定義される解析対象のクエリツリー。

失敗モードと境界条件

  • has_squashed_lists が存在する場合、クエリ提供のパラメータ番号を無視して再割り当てを行うことで、リスト圧縮による番号の欠落を回避します。
  • USE_ASSERT_CHECKING が有効な場合、total_jumble_len を用いてバッファへの追加バイト数の整合性を検証します。

保守担当者向けメモ

  • pending_nulls はハッシュ化の効率化のための遅延処理機構です。
  • squashed フラグは、リスト構造が圧縮された際にパラメータ番号の再割り当てが必要であることを示します。
