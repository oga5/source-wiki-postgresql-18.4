exec_command_bind_named

定義

  • 種別: function
  • 定義箇所: src/bin/psql/command.c:555

目的

準備されたステートメント（prepared statement）の名前と、それに紐付けるパラメータを psql のグローバル設定（pset）に設定します。

シグネチャまたは形状

static backslashResult exec_command_bind_named(PsqlScanState scan_state, bool active_branch, const char *cmd)

入力と出力

  • **入力**:
  • scan_state: コマンド引数を読み取るためのスキャン状態。
  • active_branch: 現在の条件分岐（\if）がアクティブかどうかを示すフラグ。
  • cmd: 実行されたコマンド名。
  • **出力**:
  • 戻り値: backslashResult 型。成功時は PSQL_CMD_SKIP_LINE、引数不足等のエラー時は PSQL_CMD_ERROR を返します。

前提条件と不変条件

  • active_branch が真の場合、pset の状態を更新します。
  • 準備されたステートメント名は必須の引数です。

アルゴリズムまたは振る舞い

1. active_branch が偽の場合、ignore_slash_options を呼び出して引数を読み飛ばします。 2. active_branch が真の場合:

  • clean_extended_state を呼び出し、既存の拡張クエリ状態をクリアします。
  • 最初の引数として準備されたステートメント名を取得します。取得できない場合はエラーをログ出力し、PSQL_CMD_ERROR を返します。
  • 取得できた場合、pset.stmtName に格納し、pset.send_mode を PSQL_SEND_EXTENDED_QUERY_PREPARED に設定します。
  • 続く引数をループで取得し、pset.bind_params 配列に格納します。必要に応じて pg_realloc_array でメモリを拡張します。
  • 最終的なパラメータ数を pset.bind_nparams に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c 内のディスパッチャ（exec_command）から呼び出されます。
  • **呼び出し先**: psql_scan_slash_option（引数取得）、clean_extended_state（状態初期化）、pg_log_error（エラー出力）、pg_realloc_array（メモリ確保）、ignore_slash_options（非アクティブ時のスキップ）。

関連データ構造

  • PsqlScanState: コマンド解析状態。
  • backslashResult: コマンド実行結果の列挙型。
  • pset: psql のグローバル設定構造体（stmtName, send_mode, bind_params, bind_nparams を保持）。

失敗モードと境界条件

  • 必須のステートメント名が指定されていない場合、エラーメッセージを出力して PSQL_CMD_ERROR を返します。
  • パラメータのメモリ確保に失敗した場合、pg_realloc_array が内部でエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は \bind コマンドの実装の一部です。
  • pset.bind_params のメモリ管理は pg_realloc_array に依存しているため、メモリリークや不正なポインタ参照が発生しないよう注意してください。
