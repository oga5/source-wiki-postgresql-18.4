set_option

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/args.c:260

目的

コマンドライン引数または設定ファイルから渡されたオプション文字列を解析し、対応する内部変数や動作設定に反映させる。

シグネチャまたは形状

void set_option(char *arg)

入力と出力

  • **入力**: arg - オプション文字列（例: "-lp"）。先頭の - は関数内でスキップされる。
  • **出力**: なし（副作用としてグローバル変数やプログラム状態を更新）。

前提条件と不変条件

  • pro テーブル（struct pro の配列）が定義されており、各オプションが適切に登録されていること。
  • option_source 変数に現在の解析元（ファイル名やコマンドライン）が格納されていること。

アルゴリズムまたは振る舞い

1. arg の先頭文字をスキップし、pro テーブルを走査して一致するオプションを検索する。 2. eqin 関数を用いて引数文字列とオプション名の前方一致を確認する。 3. 一致した場合、p_type に応じて以下の処理を行う：

  • PRO_SPECIAL: p_special の値に基づき、型定義の追加、バージョン表示、標準入出力の切り替えなどの特殊処理を実行する。
  • PRO_BOOL: p_special が OFF か否かに応じて、対応する変数を false または true に設定する。
  • PRO_INT: 引数が数値であることを確認し、atoi で変換して対応する変数に代入する。

    • 4. 一致するオプションが見つからない場合、または引数が必要なオプションで値が欠落している場合は errx を呼び出し、エラーメッセージを出力して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: scan_profile（プロファイルファイル解析時）およびコマンドライン引数解析ルーチン。
  • **呼び出し先**: eqin（検索用）、add_typename、add_typedefs_from_file、errx（エラー時）、printf/exit（バージョン表示時）。
  • **注記**: ツールが提示した src/backend/... や src/bin/pg_dump/... 等の呼び出し元情報は、PostgreSQL 本体のデータベース関連コードであり、本ツール（pg_bsd_indent）の set_option とは無関係な別シンボルへの参照である可能性が高い。

関連データ構造

  • struct pro: オプションの定義テーブル。名前、型、デフォルト値、対象変数のポインタを保持する。
  • ps (parser_state): 解析状態を保持するグローバル構造体。

失敗モードと境界条件

  • **不明なオプション**: pro テーブルに存在しないオプションが渡された場合、errx により終了する。
  • **引数不足**: PRO_INT や KEY など、値を必要とするオプションに対して値が提供されない場合、errx により終了する。
  • **内部エラー**: p_type や p_special が定義外の値を持つ場合、内部エラーとして終了する。

保守担当者向けメモ

  • pro[] テーブルの順序は重要であり、長い名前のオプションを短い名前のオプションより先に定義する必要がある（eqin の制約）。
  • 新しいオプションを追加する場合は、pro[] テーブルへのエントリ追加と、必要に応じて PRO_SPECIAL の switch 文へのケース追加を行う。
