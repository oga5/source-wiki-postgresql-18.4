pgstat_report_subscription_error

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_subscription.c:26

目的

論理レプリケーションのサブスクリプションにおいて発生したエラー（適用エラーまたは同期エラー）の発生回数をカウントし、バックエンドローカルの統計情報へ反映させる。

シグネチャまたは形状

void pgstat_report_subscription_error(Oid subid, bool is_apply_error)

入力と出力

  • subid (Oid): エラーが発生したサブスクリプションの識別子。
  • is_apply_error (bool): true の場合は適用エラー（Apply Error）、false の場合は同期エラー（Sync Error）としてカウントする。
  • 戻り値: なし。

前提条件と不変条件

  • pgstat_prep_pending_entry を介して、指定された subid に対応する統計エントリが取得可能であること。

アルゴリズムまたは振る舞い

1. pgstat_prep_pending_entry を呼び出し、PGSTAT_KIND_SUBSCRIPTION 種別の統計エントリ参照を取得する。 2. 取得したエントリの pending 構造体へアクセスする。 3. is_apply_error が true であれば pending->apply_error_count をインクリメントする。 4. is_apply_error が false であれば pending->sync_error_count をインクリメントする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/tablesync.c:1702
  • src/backend/replication/logical/worker.c:4532
  • src/backend/replication/logical/worker.c:4853
  • 呼び出し先:
  • pgstat_prep_pending_entry (統計サブシステム基盤関数)

関連データ構造

  • PgStat_BackendSubEntry: バックエンドローカルで保持されるサブスクリプション統計構造体。
  • PgStat_EntryRef: 統計エントリへの参照を保持する構造体。

失敗モードと境界条件

  • pgstat_prep_pending_entry が失敗した場合の挙動については、本ソースコード内では明示的なエラーハンドリングは行われていない。

保守担当者向けメモ

  • 本関数はバックエンドローカルの統計を更新するのみであり、共有メモリへの反映は後続のフラッシュ処理（pgstat_subscription_flush_cb）に依存する。
  • 統計フィールドの追加や変更を行う際は、PgStat_BackendSubEntry 構造体の定義と整合性を確認すること。
