ValidateJoinEstimator

定義

  • 種別: function
  • 定義箇所: src/backend/commands/operatorcmds.c:330

目的

結合推定関数（Join Estimator）の妥当性を検証し、その OID を取得します。

シグネチャまたは形状

Oid ValidateJoinEstimator(List *joinName)

入力と出力

  • **入力**: joinName (結合推定関数名を表すリスト)
  • **出力**: 検証済みの関数 OID (Oid)
  • **副作用**: 検証失敗時に ereport(ERROR) を発行します。

前提条件と不変条件

  • 推定関数は float8 型を戻り値として定義されている必要があります。
  • 5引数形式（Postgres 8.4以降の推奨）または4引数形式のシグネチャを持つ必要があります。
  • 非組み込み関数を指定する場合、実行にはスーパーユーザー権限が必要です。

アルゴリズムまたは振る舞い

1. LookupFuncName を使用し、5引数および4引数のシグネチャで関数を検索します。 2. 両方のシグネチャが存在する場合は ERRCODE_AMBIGUOUS_FUNCTION でエラーを報告します。 3. どちらも存在しない場合はエラーを報告します。 4. get_func_rettype を呼び出し、戻り値が FLOAT8OID であることを確認します。 5. FirstGenbkiObjectId を基準に組み込み関数か判定し、非組み込み関数の場合は superuser() チェックを行います。 6. 組み込み関数の場合は object_aclcheck を用いて ACL_EXECUTE 権限を検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/operatorcmds.c 内の DefineOperator 等から呼び出されます。
  • **呼び出し先**: LookupFuncName, get_func_rettype, superuser, object_aclcheck, aclcheck_error 等。

関連データ構造

  • List: 関数名を保持するリスト。
  • Oid: システムカタログ上の識別子。

失敗モードと境界条件

  • **ERRCODE_AMBIGUOUS_FUNCTION**: 4引数と5引数の両方の関数が定義されている場合。
  • **ERRCODE_INVALID_OBJECT_DEFINITION**: 戻り値が float8 でない場合。
  • **ERRCODE_INSUFFICIENT_PRIVILEGE**: 非組み込み関数に対してスーパーユーザー権限がない場合、または実行権限がない場合。

保守担当者向けメモ

  • 本関数は ValidateRestrictionEstimator と同様の権限チェックロジックを共有しています。
  • 8.4以前の古い4引数形式を許容していますが、将来的に廃止される可能性があるため注意してください。
