sepgsql_schema_post_create

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/schema.c:35

目的

新規作成されたスキーマに対し、SELinux ポリシーに基づいたデフォルトのセキュリティラベルを計算し、適用する。

シグネチャまたは形状

void sepgsql_schema_post_create(Oid namespaceId)

入力と出力

  • **入力**: namespaceId (作成されたスキーマの OID)
  • **出力**: なし（副作用としてシステムカタログ pg_seclabel にラベルを永続化する）

前提条件と不変条件

  • namespaceId に対応するスキーマが pg_namespace カタログに存在すること。
  • 一時スキーマ（pg_temp_ や pg_toast_temp_ で始まる名前）は、それぞれ pg_temp または pg_toast_temp としてラベル計算の対象となる。

アルゴリズムまたは振る舞い

1. pg_namespace をスキャンし、指定された namespaceId のスキーマ名を取得する。 2. 現在のデータベースのセキュリティラベルを sepgsql_get_label で取得する。 3. sepgsql_compute_create を呼び出し、クライアントのラベルとデータベースのラベルから、新しいスキーマ用のセキュリティラベルを計算する。 4. sepgsql_avc_check_perms_label を呼び出し、db_schema:{create} 権限を検証する（違反時はエラーとなる）。 5. SetSecurityLabel を使用して、計算されたラベルを対象スキーマに適用する。 6. 使用したメモリ（ラベル文字列）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:117
  • **呼び出し先**:
  • table_open, systable_beginscan, systable_getnext, systable_endscan, table_close (カタログ操作)
  • sepgsql_get_label, sepgsql_compute_create, sepgsql_avc_check_perms_label (SELinux/AVC 関連)
  • SetSecurityLabel (セキュリティラベル適用)

関連データ構造

  • Form_pg_namespace: スキーマのカタログ情報を保持。
  • ObjectAddress: ラベルを適用する対象オブジェクトを特定するために使用。

失敗モードと境界条件

  • namespaceId に対応するタプルが見つからない場合、elog(ERROR) によりトランザクションが中断される。
  • SELinux ポリシーにより db_schema:{create} が許可されていない場合、sepgsql_avc_check_perms_label がエラーを発生させる。

保守担当者向けメモ

  • 本関数はスキーマ作成後のフックとして動作する。
  • 一時スキーマの特殊処理（pg_temp_ 等）は、将来的に libselinux の更新により変更される可能性がある（ソース内の XXX コメント参照）。
