CatCache

定義

  • 種別: struct
  • 定義箇所: src/include/utils/catcache.h:44

目的

システムカタログへのアクセスを高速化するため、メモリ上にシステムテーブルの行（タプル）を保持するキャッシュ構造体。インデックススキャンを回避し、存在確認および部分一致検索を効率化する。

シグネチャまたは形状

typedef struct catcache {
    int id;
    int cc_nbuckets;
    TupleDesc cc_tupdesc;
    dlist_head *cc_bucket;
    CCHashFN cc_hashfunc[CATCACHE_MAXKEYS];
    CCFastEqualFN cc_fastequal[CATCACHE_MAXKEYS];
    int cc_keyno[CATCACHE_MAXKEYS];
    int cc_nkeys;
    int cc_ntup;
    int cc_nlist;
    int cc_nlbuckets;
    dlist_head *cc_lbucket;
    const char *cc_relname;
    Oid cc_reloid;
    Oid cc_indexoid;
    bool cc_relisshared;
    slist_node cc_next;
    ScanKeyData cc_skey[CATCACHE_MAXKEYS];
    /* CATCACHE_STATS 有効時のみ統計用メンバが存在 */
} CatCache;

入力と出力

  • **入力**: 検索キー（ScanKeyData）、ハッシュ関数、等価比較関数。
  • **出力**: キャッシュヒット時は CatCTup（タプル）または CatCList（リスト）、ミス時はシステムテーブルからの読み込みを経てキャッシュへ登録。

前提条件と不変条件

  • 対応するシステムテーブルのインデックスが一意（Unique）であることを前提とする。
  • キャッシュは CacheMemoryContext 上で管理される。
  • 参照カウントの管理が必須であり、ReleaseCatCache の呼び出し漏れはメモリリークの原因となる。

アルゴリズムまたは振る舞い

1. **検索**: ハッシュ値からバケットを特定し、エントリを探索。 2. **ヒット**: 参照カウントをインクリメントして返却。 3. **ミス**: システムテーブルをインデックススキャンし、取得したタプルをキャッシュに登録。 4. **無効化**: システムテーブル更新時に CatCacheInvalidate を呼び出し、該当エントリを「dead」とマーク。参照カウントがゼロになった時点で解放。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/catcache.c 内の SearchCatCacheInternal、SearchCatCacheMiss 等が本構造体を参照・操作する。
  • **呼び出し先**: access/htup.h（HeapTuple）、access/skey.h（ScanKeyData）、lib/ilist.h（dlist）、utils/relcache.h（TupleDesc）の各機能を利用。

関連データ構造

  • CatCTup: 個別のキャッシュエントリ（正・負のキャッシュを保持）。
  • CatCList: 部分キー検索の結果リスト。
  • CatCacheHeader: キャッシュ全体の管理ヘッダ。

失敗モードと境界条件

  • **キャッシュミス**: 存在しないオブジェクトに対する検索は「負のキャッシュ」として記録され、再検索を防止する。
  • **メモリ管理**: 参照カウントの不整合により、不要なエントリがメモリに残留する可能性がある。

保守担当者向けメモ

  • CATCACHE_STATS を定義してコンパイルすると、統計情報メンバが構造体末尾に追加される。ABI 互換性を維持するため、これらは必ず構造体の末尾に配置すること。
  • キャッシュの無効化ロジックは src/backend/utils/cache/inval.c に依存している。
