EnumItem

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/typcache.c:147

目的

列挙型（Enum）の各値の識別子と、そのソート順序を保持するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct
{
	Oid			enum_oid;		/* OID of one enum value */
	float4		sort_order;		/* its sort position */
} EnumItem;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではないため入出力はありません。
  • find_enumitem 関数等の検索処理において、特定の Oid を持つ EnumItem を取得するために利用されます。

前提条件と不変条件

  • sort_order は列挙型のソート順を決定するために使用されます。
  • enum_oid は pg_enum システムカタログ内の特定の列挙値に対応する OID です。

アルゴリズムまたは振る舞い

  • typcache.c 内で、列挙型の値をソートする際に qsort の比較対象として利用されます。
  • TypeCacheEnumData 構造体内の配列として保持され、列挙型のキャッシュ構築時にシステムカタログから読み込まれた値が格納されます。

呼び出し元と呼び出し先

  • 呼び出し元: find_enumitem、および列挙型キャッシュの構築・ソートを行う typcache.c 内の各関数。
  • 呼び出し先: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • TypeCacheEnumData: 列挙型キャッシュの情報を保持する構造体であり、EnumItem の配列をメンバとして持ちます。

失敗モードと境界条件

  • FLEXIBLE_ARRAY_MEMBER を用いた配列として定義される場合があり、メモリ確保時に sizeof(EnumItem) を用いた計算が必要です。
  • メモリ不足時には palloc や repalloc がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 列挙型のキャッシュ更新時には、memcpy を用いて EnumItem の配列全体がコピーされるため、構造体のサイズ変更には注意が必要です。
  • sort_order は float4 型であるため、列挙値の順序変更時には浮動小数点数としての比較特性に留意してください。
