HashOptions

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/hash.h:269

目的

ハッシュインデックスの作成時や管理時に指定可能なオプション（リレーションオプション）を保持するための構造体です。

シグネチャまたは形状

typedef struct HashOptions
{
    int32       varlena_header_;    /* varlena header (do not touch directly!) */
    int         fillfactor;         /* page fill factor in percent (0..100) */
} HashOptions;

入力と出力

  • **入力**: インデックス作成時の WITH 句などで指定されたパラメータが、本構造体のフィールドにマッピングされます。
  • **出力**: fillfactor フィールドは、インデックスページの充填率を制御するために使用されます。

前提条件と不変条件

  • fillfactor は 0 から 100 の範囲である必要があります。
  • varlena_header_ は PostgreSQL の可変長データ型ヘッダーであり、直接操作してはなりません。

アルゴリズムまたは振る舞い

  • 本構造体は、リレーションのオプションとして格納されます。
  • src/include/access/hash.h 内のマクロ定義により、リレーションの rd_options から HashOptions へキャストされ、fillfactor が参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hashutil.c: インデックスオプションの定義および初期化において参照されています。
  • src/include/access/hash.h: マクロを通じてリレーションのオプションから fillfactor を取得する際に参照されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • Relation: インデックスのリレーション情報を保持する構造体。rd_options フィールドを通じて本構造体を参照します。

失敗モードと境界条件

  • fillfactor が 0〜100 の範囲外である場合、インデックス操作の整合性に影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体は varlena 型として扱われるため、メモリレイアウトやサイズ計算（sizeof(HashOptions)）を行う際は、PostgreSQL の可変長データ型の規約に従う必要があります。
