r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_finnish.c:279

目的

単語内の母音と子音の並びを解析し、ステミング（語幹抽出）処理の境界となる領域 R1 および R2 の開始位置を特定します。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowball 実行環境）。z->l（文字列長）および z->c（現在のカーソル位置）を使用。
  • 出力: 成功時に 1、失敗時に 0 を返します。
  • 副作用: z->I[1] に領域 R1 の開始位置、z->I[0] に領域 R2 の開始位置を格納します。

前提条件と不変条件

  • SN_env 構造体が初期化されていること。
  • g_V1（母音グループ）が定義されていること。
  • 領域 R1 は R2 よりも前（または同じ位置）に配置される必要があります。

アルゴリズムまたは振る舞い

1. z->I[1]（R1）と z->I[0]（R2）を文字列の末尾 z->l で初期化します。 2. out_grouping_U を使用して、最初の母音以外の文字をスキップし、続く in_grouping_U で最初の母音を検索します。見つかった場合、その直後の位置を z->I[1]（R1）に設定します。 3. 同様に、R1 以降で再度母音以外の文字をスキップし、続く母音を検索します。見つかった場合、その直後の位置を z->I[0]（R2）に設定します。 4. 検索の過程で母音が見つからない場合は 0 を返し、正常終了時は 1 を返します。

呼び出し元と呼び出し先

  • 呼び出し元: finnish_UTF_8_stem（ファイル概要より）。
  • 呼び出し先: out_grouping_U, in_grouping_U（Snowball ランタイム関数）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。I 配列は整数インデックスを格納します。
  • g_V1: 母音を定義するビットマップ配列。

失敗モードと境界条件

  • 文字列内に母音が存在しない場合、領域の更新は行われず 0 を返します。
  • 文字列が短く、R1 や R2 を定義するための母音・子音の並びが不十分な場合、処理は中断されます。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによって自動生成されています。ロジックの変更が必要な場合は、直接編集せず Snowball ソースファイルを修正してください。
  • z->I[0] は R2、z->I[1] は R1 の開始位置を保持します。
