CopyGetInt16

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfromparse.c:378

目的

バイナリ形式の COPY FROM 入力ストリームから、ネットワークバイトオーダーで格納された 16 ビット整数を読み込み、ホストのバイトオーダーに変換して取得する。

シグネチャまたは形状

static inline bool CopyGetInt16(CopyFromState cstate, int16 *val)

入力と出力

  • **入力**:
  • cstate: COPY FROM 操作の状態を保持する CopyFromState 構造体。
  • val: 読み込んだ値を格納するための int16 型へのポインタ。
  • **出力**:
  • 戻り値: 正常に読み込めた場合は true、読み込みに失敗した場合は false。
  • val: 成功時に変換後の 16 ビット整数値が格納される。

前提条件と不変条件

  • 入力データはネットワークバイトオーダー（ビッグエンディアン）で提供されている必要がある。
  • CopyReadBinaryData が正常に動作する状態であること。

アルゴリズムまたは振る舞い

1. CopyReadBinaryData を呼び出し、入力ストリームから 2 バイト（sizeof(uint16)）を読み込む。 2. 読み込みバイト数が 2 バイトに満たない場合、val を 0 に初期化し、false を返して終了する。 3. 読み込めた場合、pg_ntoh16 を使用してネットワークバイトオーダーからホストバイトオーダーへ変換する。 4. 変換後の値を int16 にキャストして val に格納し、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfromparse.c:1101（fld_count の読み込みに使用）。
  • **呼び出し先**: CopyReadBinaryData。

関連データ構造

  • CopyFromState: COPY FROM 操作のコンテキストを保持する構造体。

失敗モードと境界条件

  • 入力ストリームが 2 バイト未満で終了した場合、読み込み失敗として false を返す。
  • 読み込みエラー発生時、val は 0 に設定される。

保守担当者向けメモ

  • 本関数は static inline で定義されており、バイナリ形式のパース処理において頻繁に呼び出されるため、インライン展開によるオーバーヘッド削減が意図されている。
  • バイトオーダー変換には PostgreSQL 標準の pg_ntoh16 を使用しているため、プラットフォーム依存のエンディアン差異は吸収される。
