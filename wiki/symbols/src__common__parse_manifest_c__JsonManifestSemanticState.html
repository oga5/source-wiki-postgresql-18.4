JsonManifestSemanticState

定義

  • 種別: enum
  • 定義箇所: src/common/parse_manifest.c:22

目的

バックアップマニフェスト（JSON形式）を解析する際、現在の解析フェーズ（どのJSONオブジェクトやフィールドを処理中か）を追跡するための状態マシンとして機能します。

シグネチャまたは形状

typedef enum {
    JM_EXPECT_TOPLEVEL_START,
    JM_EXPECT_TOPLEVEL_END,
    JM_EXPECT_TOPLEVEL_FIELD,
    JM_EXPECT_VERSION_VALUE,
    JM_EXPECT_SYSTEM_IDENTIFIER_VALUE,
    JM_EXPECT_FILES_START,
    JM_EXPECT_FILES_NEXT,
    JM_EXPECT_THIS_FILE_FIELD,
    JM_EXPECT_THIS_FILE_VALUE,
    JM_EXPECT_WAL_RANGES_START,
    JM_EXPECT_WAL_RANGES_NEXT,
    JM_EXPECT_THIS_WAL_RANGE_FIELD,
    JM_EXPECT_THIS_WAL_RANGE_VALUE,
    JM_EXPECT_MANIFEST_CHECKSUM_VALUE,
    JM_EXPECT_EOF,
} JsonManifestSemanticState;

入力と出力

  • 入力: JSONパーサーからのイベント通知に基づき、状態遷移が発生します。
  • 出力: 現在の状態値が JsonManifestParseState 構造体に保持され、後続の解析処理の分岐条件として使用されます。

前提条件と不変条件

  • マニフェストの解析順序は、この列挙型で定義された遷移フローに従う必要があります。
  • 状態遷移は、src/common/jsonapi.h を介したストリーミング解析のコールバック関数内で行われます。

アルゴリズムまたは振る舞い

JSONの構造を走査する際、現在の状態に応じて期待される次のトークンやフィールドを検証します。例えば、JM_EXPECT_FILES_START の状態では配列の開始を待ち、JM_EXPECT_THIS_FILE_FIELD ではファイルオブジェクト内のキーを処理します。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/parse_manifest.c 内の各JSONハンドラ関数（json_manifest_object_start 等）。
  • 呼び出し先: 特定の関数ではなく、状態変数として JsonManifestParseState 構造体内で参照・更新されます。

関連データ構造

  • JsonManifestParseState: この列挙型をメンバとして保持し、解析中の内部状態を管理します。

失敗モードと境界条件

  • 期待される状態と異なるJSON構造が入力された場合、解析処理は失敗します。
  • 供給されたソース情報には、エラー発生時の具体的な遷移先やエラーハンドリングの詳細は含まれていません。

保守担当者向けメモ

  • 新しいマニフェストフィールドを追加する場合は、この列挙型に適切な状態を追加し、対応するハンドラ関数での遷移ロジックを更新してください。
