ConstraintSetParentConstraint

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_constraint.c:1123

目的

制約の継承関係を設定または解除し、pg_constraint カタログのメタデータおよび pg_depend の依存関係を更新する。

シグネチャまたは形状

void ConstraintSetParentConstraint(Oid childConstrId, Oid parentConstrId, Oid childTableId)

入力と出力

  • childConstrId (Oid): 継承関係を変更する対象の制約OID。
  • parentConstrId (Oid): 親となる制約のOID（解除時は InvalidOid）。
  • childTableId (Oid): 子制約が属するテーブルのOID。
  • 出力: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • pg_constraint テーブルに対する RowExclusiveLock が保持されていること。
  • parentConstrId が有効な場合、対象の制約が既に親を持っていないこと（coninhcount == 0 かつ conparentid == InvalidOid）。
  • parentConstrId が InvalidOid の場合、対象の制約の継承カウントが 0 であること。

アルゴリズムまたは振る舞い

1. pg_constraint をスキャンし、childConstrId に該当するタプルを取得する。 2. **継承設定時 (parentConstrId が有効)**:

  • conislocal を false に設定し、coninhcount をインクリメントする。
  • conparentid を parentConstrId に更新する。
  • CatalogTupleUpdate でカタログを更新する。
  • 親制約への依存関係 (DEPENDENCY_PARTITION_PRI) および子テーブルへの依存関係 (DEPENDENCY_PARTITION_SEC) を pg_depend に記録する。

    • 3. **継承解除時 (parentConstrId が InvalidOid)**:

  • coninhcount をデクリメントし、conislocal を true に設定、conparentid を InvalidOid にリセットする。
  • CatalogTupleUpdate でカタログを更新する。
  • 関連する DEPENDENCY_PARTITION_PRI および DEPENDENCY_PARTITION_SEC の依存関係レコードを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/indexcmds.c および src/backend/commands/tablecmds.c 内の複数箇所（パーティションテーブルの制約操作に関連）。
  • **呼び出し先**: table_open, SearchSysCache1, heap_copytuple, CatalogTupleUpdate, recordDependencyOn, deleteDependencyRecordsForClass, ReleaseSysCache, table_close 等。

関連データ構造

  • Form_pg_constraint: pg_constraint カタログの行構造体。
  • ObjectAddress: 依存関係管理に使用されるオブジェクト識別子。

失敗モードと境界条件

  • childConstrId が存在しない場合、elog(ERROR) で停止する。
  • 既に親を持つ制約に対して親を設定しようとした場合、elog(ERROR) で停止する。
  • coninhcount がオーバーフローする場合、ereport(ERROR) で停止する。

保守担当者向けメモ

  • 本関数はパーティション継承における制約の管理を担う。依存関係の追加・削除は pg_depend の特定のクラス（DEPENDENCY_PARTITION_PRI/SEC）に対して行われるため、変更時は依存関係の整合性に注意すること。
