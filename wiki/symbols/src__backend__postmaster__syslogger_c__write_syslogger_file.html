write_syslogger_file

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/syslogger.c:1092

目的

ログデータを適切な出力先（標準ログ、CSVログ、またはJSONログ）のファイルへ書き込む。

シグネチャまたは形状

void write_syslogger_file(const char *buffer, int count, int destination)

入力と出力

  • buffer: 書き込むデータを含む文字列バッファ。
  • count: 書き込むバイト数。
  • destination: 書き込み先を指定するビットマスク（LOG_DESTINATION_CSVLOG, LOG_DESTINATION_JSONLOG 等）。
  • 出力: 指定されたファイルストリームへの書き込み。戻り値なし。

前提条件と不変条件

  • syslogFile は常にオープンされている必要がある。
  • 構造化ログ（CSV/JSON）のファイルポインタが NULL の場合、データはフォールバックとして syslogFile に書き込まれる。
  • 再帰的な呼び出しを避けるため、ファイルオープンに失敗しても動的な再オープンは行わない。

アルゴリズムまたは振る舞い

1. destination フラグに基づき、書き込み先のファイルポインタ（csvlogFile, jsonlogFile, または syslogFile）を選択する。 2. fwrite を使用してバッファの内容をファイルへ書き出す。 3. 書き込みバイト数が count と一致しない場合、write_stderr を使用してエラーを報告する。この際、無限ループを避けるため ereport は使用しない。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/syslogger.c 内のパイプ処理関数群、および src/backend/utils/error/ 配下の csvlog.c, elog.c, jsonlog.c。
  • 呼び出し先: fwrite（標準ライブラリ）、write_stderr（エラー報告用）。

関連データ構造

  • csvlogFile, jsonlogFile, syslogFile: ログ書き込みに使用される FILE ポインタ。

失敗モードと境界条件

  • 書き込み失敗時: fwrite の戻り値が count と一致しない場合、write_stderr を通じてエラーメッセージを出力する。
  • 構造化ログ未オープン時: 構造化ログのファイルポインタが NULL の場合、自動的に標準ログファイルへ転送される。

保守担当者向けメモ

  • 本関数内での ereport 使用は、ログ出力の再帰呼び出しを引き起こすため厳禁である。
  • ログファイルのオープン状態は動的に変化する可能性があるため、書き込み先選択時の NULL チェックは必須である。
