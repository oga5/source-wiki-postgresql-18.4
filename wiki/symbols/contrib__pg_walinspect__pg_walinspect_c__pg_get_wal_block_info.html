pg_get_wal_block_info

定義

  • 種別: function
  • 定義箇所: contrib/pg_walinspect/pg_walinspect.c:414

目的

指定されたLSN範囲内のWALレコードに含まれるブロック参照を詳細に展開し、SQLインターフェースを通じてテーブル形式で提供する。

シグネチャまたは形状

Datum pg_get_wal_block_info(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • start_lsn (LSN): 読み取り開始位置。
  • end_lsn (LSN): 読み取り終了位置。
  • show_data (bool): ブロックデータを含めるかどうか。
  • **出力**:
  • PG_RETURN_VOID(): SRF（Set Returning Function）として tuplestore に結果を格納して終了する。

前提条件と不変条件

  • ValidateInputLSNs により、入力されたLSN範囲が妥当であること。
  • InitMaterializedSRF により、結果を格納するための tuplestore が初期化されていること。
  • メモリ管理のため、ループ内での処理には一時的な MemoryContext (tmp_cxt) を使用し、各レコード処理後にリセットを行うこと。

アルゴリズムまたは振る舞い

1. ValidateInputLSNs で入力LSNを検証し、InitXLogReaderState でWAL読み取り状態を初期化する。 2. ReadNextXLogRecord を使用して、end_lsn に達するまでWALレコードを順次デコードする。 3. 各レコードに対し、XLogRecHasAnyBlockRefs でブロック参照の有無を確認する。 4. ブロック参照が存在する場合、GetWALBlockInfo を呼び出して情報を展開し、結果セットに追加する。 5. ループ終了後、一時メモリコンテキストの解放、xlogreader の解放を行い、関数を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • ValidateInputLSNs, InitMaterializedSRF, InitXLogReaderState
  • ReadNextXLogRecord, XLogRecHasAnyBlockRefs, GetWALBlockInfo
  • MemoryContextSwitchTo, MemoryContextReset, MemoryContextDelete, XLogReaderFree

関連データ構造

  • XLogReaderState: WAL読み取りの内部状態を保持する構造体。
  • FunctionCallInfo (fcinfo): SRFとしての関数呼び出し情報。

失敗モードと境界条件

  • ReadNextXLogRecord が失敗するか、end_lsn を超えた場合にループを終了する。
  • CHECK_FOR_INTERRUPTS() がループ内に配置されており、ユーザーによるクエリ中断要求に応答する。
  • メモリ不足時は AllocSetContextCreate 等がエラーを発生させる。

保守担当者向けメモ

  • 本関数は tuplestore を使用するSRFであるため、メモリリークを防ぐために tmp_cxt の切り替えとリセットが必須である。
  • GetWALBlockInfo の実装変更時は、本関数での呼び出し引数との整合性に注意すること。
