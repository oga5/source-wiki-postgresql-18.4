wchareq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:57

目的

2つのマルチバイト文字（または文字列）が等しいかどうかを、バイト単位の比較によって高速に判定します。

シグネチャまたは形状

static inline int wchareq(const char *p1, int p1len, const char *p2, int p2len)

入力と出力

  • **入力**:
  • p1, p2: 比較対象の文字列へのポインタ。
  • p1len, p2len: 各文字列の長さ（バイト数）。
  • **出力**:
  • 1: 両者が等しい場合。
  • 0: 両者が異なる場合。

前提条件と不変条件

  • p1 および p2 は、それぞれの長さ p1len, p2len を超えてアクセスされないことが前提です。
  • マルチバイト文字の長さ判定には pg_mblen_with_len を使用します。

アルゴリズムまたは振る舞い

1. **先頭バイト比較**: 最初のバイト *p1 と *p2 を比較し、異なる場合は即座に 0 を返します。 2. **文字長判定**: pg_mblen_with_len を用いて両者の文字長を計算し、一致しない場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/like.c 内の CHAREQ マクロ（#define CHAREQ(p1, p1len, p2, p2len) wchareq((p1), (p1len), (p2), (p2len))）経由で利用されます。
  • **呼び出し先**:
  • pg_mblen_with_len（マルチバイト文字の長さ計算）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • p1len または p2len が 0 の場合、pg_mblen_with_len の挙動に依存します。
  • 文字列の長さが異なる場合、比較は行われず 0 が返されます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、like.c 内でのパターンマッチング処理の最適化パスとして機能します。
  • CHAREQ マクロを介して呼び出されるため、直接呼び出すよりもマクロ経由での利用が推奨されます。
