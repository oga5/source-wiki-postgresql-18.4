hstore_exists_all

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:195

目的

指定された hstore 内に、入力されたテキスト配列のすべてのキーが存在するかどうかを判定する。

シグネチャまたは形状

Datum hstore_exists_all(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_HSTORE_P(0): 検索対象の HStore 構造体。
  • PG_GETARG_ARRAYTYPE_P(1): 検索するキーを含む ArrayType。
  • **出力**:
  • PG_RETURN_BOOL(res): すべてのキーが存在すれば true、一つでも欠けていれば false を返す。

前提条件と不変条件

  • hstore 内部のデータは、キーの長さと内容に基づいてソートされている必要がある。
  • hstoreArrayToPairs によって生成される key_pairs は、昇順にソートされている必要がある。

アルゴリズムまたは振る舞い

1. 入力配列を hstoreArrayToPairs を用いて Pairs 構造体の配列に変換する。 2. key_pairs がソート済みである特性を利用し、hstoreFindKey を呼び出して各キーを検索する。 3. 検索の際、lowbound 変数を使用して検索範囲を絞り込む。各検索は前回の検索結果の直後から開始される。 4. いずれかのキーが見つからない場合（idx < 0）、即座にループを終了し false を返す。 5. すべてのキーが見つかった場合、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数呼び出しインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • hstoreArrayToPairs (ユーティリティ関数)
  • hstoreFindKey (バイナリサーチによる検索関数)

関連データ構造

  • HStore: 検索対象のハッシュマップ構造。
  • Pairs: キーと値のペアを保持する構造体。
  • ArrayType: 入力キーの配列。

失敗モードと境界条件

  • 入力配列が空の場合、ループは実行されず true が返される。
  • hstoreFindKey がキーを見つけられない場合、res は false に設定され、検索は中断される。

保守担当者向けメモ

  • 本関数は hstore のソート済み特性に強く依存している。hstoreFindKey の lowbound 引数による最適化を維持するため、検索順序の変更には注意が必要。
