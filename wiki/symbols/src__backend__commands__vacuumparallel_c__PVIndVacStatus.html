PVIndVacStatus

定義

  • 種別: enum
  • 定義箇所: src/backend/commands/vacuumparallel.c:125

目的

並列 VACUUM 処理における各インデックスの現在の処理状態を追跡・管理するために使用されます。

シグネチャまたは形状

typedef enum PVIndVacStatus
{
    PARALLEL_INDVAC_STATUS_INITIAL = 0,
    PARALLEL_INDVAC_STATUS_NEED_BULKDELETE,
    PARALLEL_INDVAC_STATUS_NEED_CLEANUP,
    PARALLEL_INDVAC_STATUS_COMPLETED,
} PVIndVacStatus;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • インデックス処理の進行に伴い、INITIAL → NEED_BULKDELETE → NEED_CLEANUP → COMPLETED の順序で状態が遷移することが想定されています。

アルゴリズムまたは振る舞い

  • PARALLEL_INDVAC_STATUS_INITIAL: インデックス処理が未開始の状態。
  • PARALLEL_INDVAC_STATUS_NEED_BULKDELETE: インデックスのバルク削除フェーズが必要な状態。
  • PARALLEL_INDVAC_STATUS_NEED_CLEANUP: インデックスのクリーンアップフェーズが必要な状態。
  • PARALLEL_INDVAC_STATUS_COMPLETED: インデックスの全処理が完了した状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/vacuumparallel.c 内の並列処理ロジック（parallel_vacuum_process_all_indexes 等）で、インデックスのステータス管理に使用されています。
  • 呼び出し先: なし（列挙型のため）。

関連データ構造

  • PVIndStats: 各インデックスの統計情報を保持する構造体で、このステータスをメンバとして含む可能性があります。

失敗モードと境界条件

  • 状態遷移の整合性が崩れた場合、並列ワーカー間でインデックスの二重処理や処理漏れが発生する可能性があります。

保守担当者向けメモ

  • 本列挙型は vacuumparallel.c 内の共有メモリ管理と密接に関連しています。新しいフェーズを追加する場合は、リーダーおよびワーカープロセスのステートマシンロジックを更新する必要があります。
