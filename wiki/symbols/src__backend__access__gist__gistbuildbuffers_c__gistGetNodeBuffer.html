gistGetNodeBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuildbuffers.c:112

目的

GiST インデックス構築プロセスにおいて、指定されたブロック番号に対応するノードバッファ（GISTNodeBuffer）を取得または新規作成します。

シグネチャまたは形状

GISTNodeBuffer *gistGetNodeBuffer(GISTBuildBuffers *gfbb, GISTSTATE *giststate, BlockNumber nodeBlocknum, int level)

入力と出力

  • **入力**:
  • gfbb: ビルドバッファ全体の管理構造体。
  • giststate: GiST インデックスの状態情報。
  • nodeBlocknum: 対象となるノードのブロック番号。
  • level: ノードが属するツリーのレベル。
  • **出力**:
  • 指定されたブロックに対応する GISTNodeBuffer 構造体へのポインタ。

前提条件と不変条件

  • gfbb->nodeBuffersTab ハッシュテーブルが初期化されていること。
  • 新規作成時は gfbb->context メモリコンテキスト上でメモリが割り当てられる。
  • level が現在の gfbb->buffersOnLevelsLen を超える場合、buffersOnLevels 配列が自動的に拡張される。

アルゴリズムまたは振る舞い

1. gfbb->nodeBuffersTab ハッシュテーブルを nodeBlocknum をキーとして検索する。 2. 既に存在する場合はそのバッファを返す。 3. 存在しない場合（新規作成時）:

  • gfbb->context に切り替え、GISTNodeBuffer を初期化する。
  • blocksCount を 0、pageBlocknum を InvalidBlockNumber に設定する。
  • 必要に応じて gfbb->buffersOnLevels 配列を拡張し、新しいバッファを該当レベルのリストの先頭に追加する（分割時の効率を考慮した配置）。

    • 4. 取得したバッファを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuild.c:1018
  • src/backend/access/gist/gistbuildbuffers.c:612
  • **呼び出し先**:
  • hash_search (ハッシュテーブル操作)
  • MemoryContextSwitchTo (メモリ管理)
  • repalloc (メモリ再割り当て)
  • lcons (リスト操作)

関連データ構造

  • GISTBuildBuffers: ビルド全体の管理構造体。
  • GISTNodeBuffer: 個別のノードバッファ状態を保持する構造体。
  • GISTSTATE: GiST インデックスのメタデータ。

失敗モードと境界条件

  • メモリ不足時は repalloc 等によりエラーが発生する可能性がある。
  • level が負の値や異常に大きい値である場合の挙動は、呼び出し側の整合性に依存する。

保守担当者向けメモ

  • 新規作成されたバッファがリストの先頭に追加されるのは、ページ分割時に新しいページを優先的にフラッシュしてキャッシュ効率を高めるための設計である。
  • 本関数は hash_search の HASH_ENTER アクションを使用しており、存在しない場合は自動的にエントリが作成される。
