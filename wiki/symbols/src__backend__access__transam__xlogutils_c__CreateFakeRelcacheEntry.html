CreateFakeRelcacheEntry

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogutils.c:570

目的

リカバリ処理中やWAL同期中に、物理的なストレージ操作（ブロック読み込み等）を行うために必要な、最小限の情報を保持する一時的なリレーション構造体（Relation）を作成します。

シグネチャまたは形状

Relation CreateFakeRelcacheEntry(RelFileLocator rlocator)

入力と出力

  • **入力**: rlocator (RelFileLocator) - 対象リレーションの物理的な位置情報。
  • **出力**: Relation - 初期化された偽のリレーション構造体。

前提条件と不変条件

  • 本関数はリカバリ中、またはWALスキップファイルの同期中にのみ使用されます。
  • 作成されるリレーションは常に永続的（RELPERSISTENCE_PERMANENT）として扱われます。
  • 一時リレーション（Temp rels）の処理には対応していません。

アルゴリズムまたは振る舞い

1. FakeRelCacheEntryData 構造体分のメモリを palloc0 で確保し、Relation 型としてキャストします。 2. rd_locator に引数の rlocator を設定します。 3. rd_backend を INVALID_PROC_NUMBER に設定します。 4. リレーション名を rlocator.relNumber に基づく文字列として生成します。 5. rd_lockInfo に dbId と relId を設定し、ダミーのロック識別子を構築します。 6. smgropen を呼び出し、非ピン留め状態の rd_smgr を設定して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_xlog.c (複数箇所)
  • src/backend/catalog/storage.c (複数箇所)
  • **呼び出し先**:
  • palloc0
  • smgropen
  • sprintf

関連データ構造

  • Relation: 返却される構造体。
  • FakeRelCacheEntryData: 内部的に使用される、Relation とその補助情報を保持する構造体。
  • RelFileLocator: リレーションの物理位置を特定する識別子。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • リレーション名が不明なため、relfilenumber を代用しており、デバッグ目的以外での名前の正確性は保証されません。

保守担当者向けメモ

  • 本関数で作成される構造体は、完全なリレーションキャッシュの代用ではありません。物理的なブロック読み込みという非常に限定的な用途のために設計されています。
  • リカバリ中であることを前提としているため、ロック競合は発生しない設計ですが、必要に応じて rd_lockInfo が設定されています。
