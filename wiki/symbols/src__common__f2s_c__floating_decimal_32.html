floating_decimal_32

定義

  • 種別: struct
  • 定義箇所: src/common/f2s.c:215

目的

Ryuアルゴリズムを用いた単精度浮動小数点数の変換過程において、数値を表現する仮数（mantissa）と指数（exponent）を保持するための中間データ構造です。

シグネチャまたは形状

typedef struct floating_decimal_32
{
	uint32		mantissa;
	int32		exponent;
} floating_decimal_32;

入力と出力

本構造体は関数間のデータ受け渡しに使用されます。

  • **入力**: float 型のビット表現から変換された仮数および指数。
  • **出力**: to_chars や to_chars_f などのフォーマット関数へ渡され、最終的な文字列変換の基盤となります。

前提条件と不変条件

  • mantissa は uint32 型であり、Ryuアルゴリズムによって計算された有効数字を保持します。
  • exponent は int32 型であり、10進数表現における指数の値を保持します。

アルゴリズムまたは振る舞い

単精度浮動小数点数を m * 10^e の形式に分解した結果を格納します。この構造体は、浮動小数点数のビット解析から文字列へのフォーマット変換に至るパイプラインにおいて、中間状態を保持する役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/f2s.c 内の変換ロジック（f2d 等）が本構造体をインスタンス化します。
  • **呼び出し先**: to_chars および to_chars_f が本構造体を引数として受け取り、文字列バッファへの書き出しを行います。

関連データ構造

  • float_to_shortest_decimal_bufn: 本構造体を利用して変換を行うメインエントリポイント。
  • d2s.c: 倍精度浮動小数点数版のRyu実装（類似の構造体が存在する可能性があります）。

失敗モードと境界条件

  • 特殊値（NaN, Infinity, 0）が入力された場合、本構造体の各フィールドにはそれらを表現するための特定の数値が格納されます。

保守担当者向けメモ

  • 本構造体は src/common/f2s.c 内でのみ使用されます。
  • 32ビット環境と64ビット環境の両方で動作するよう設計されているため、フィールドの型定義（uint32, int32）を変更する際は注意が必要です。
