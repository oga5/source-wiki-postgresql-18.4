InitCompressorLZ4

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:279

目的

pg_dump におけるアーカイブ書き込み処理のために、CompressorState を LZ4 圧縮用に初期化し、関数ポインタおよび圧縮設定を割り当てる。

シグネチャまたは形状

void InitCompressorLZ4(CompressorState *cs, const pg_compress_specification compression_spec)

入力と出力

  • **入力**:
  • cs: 初期化対象の CompressorState 構造体へのポインタ。
  • compression_spec: 圧縮レベル等の設定を含む pg_compress_specification 構造体。
  • **出力**: なし（cs の内部状態を更新する）。

前提条件と不変条件

  • cs は有効な CompressorState 構造体を指している必要がある。
  • cs->readF が設定されている場合、読み込み専用モードとみなされ、LZ4 コンテキストの初期化はスキップされる。

アルゴリズムまたは振る舞い

1. cs の関数ポインタ（readData, writeData, end）を LZ4 用の各関数（ReadDataFromArchiveLZ4, WriteDataToArchiveLZ4, EndCompressorLZ4）に設定する。 2. compression_spec を cs に保存する。 3. cs->readF が存在しない場合（書き込みモード時）:

  • LZ4State 構造体をメモリ確保し、cs->private_data に格納する。
  • compression_spec.level が 0 以上であれば、state->prefs.compressionLevel に反映する。
  • LZ4State_compression_init を呼び出し、LZ4 圧縮コンテキストを初期化する。失敗時は pg_fatal で終了する。
  • state->needs_header_flush を true に設定し、ヘッダーが未書き込みであることを記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_io.c（InitCompressorLZ4(cs, compression_spec);）
  • **呼び出し先**:
  • pg_malloc0
  • LZ4State_compression_init
  • LZ4F_getErrorName
  • pg_fatal

関連データ構造

  • CompressorState: 圧縮処理の抽象化状態を保持する構造体。
  • LZ4State: LZ4 圧縮・解凍のコンテキストおよびバッファ状態を保持する構造体。
  • pg_compress_specification: 圧縮レベル等の設定値。

失敗モードと境界条件

  • LZ4State_compression_init が失敗した場合、LZ4F_getErrorName を用いてエラーメッセージを出力し、pg_fatal によりプロセスが異常終了する。
  • cs->readF が設定されている場合は、LZ4 固有のコンテキスト初期化は行われない。

保守担当者向けメモ

  • 本関数は書き込み用コンテキストの初期化を担う。読み込み用コンテキストは LZ4Stream_read_internal 等の遅延初期化メカニズムに依存している。
  • USE_LZ4 が定義されていない環境での挙動については、ソースコード内の条件付きコンパイル指示を確認すること。
