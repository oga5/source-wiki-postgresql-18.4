gist_page_items_bytea

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/gistfuncs.c:129

目的

GiSTインデックスの物理ページを走査し、ページ内の各インデックス項目（タプル）をデコードせずにバイナリデータ（bytea）として抽出して返す。

シグネチャまたは形状

Datum gist_page_items_bytea(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型の引数（GiSTインデックスのページデータ）。
  • **出力**: tuplestore を介したセット（各行は itemoffset (int2), ctid (tid), itemlen (int4), dead (bool), data (bytea) の5列で構成される）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 入力された bytea は verify_gist_page によって GiST ページとして有効であると検証される必要がある。
  • ページが新規（PageIsNew）の場合は NULL を返す。

アルゴリズムまたは振る舞い

1. スーパーユーザー権限を確認し、エラー時は例外を投げる。 2. InitMaterializedSRF で結果セットの初期化を行う。 3. verify_gist_page でページを検証し、削除済みページであれば通知を出力する。 4. ページ内の各アイテムを FirstOffsetNumber から maxoff までループ処理する。 5. 各アイテムについて以下の情報を取得・構築する:

  • オフセット番号
  • タプルの TID
  • タプルのサイズ
  • アイテムが削除済みかどうかのフラグ
  • タプルのバイナリデータ（palloc でメモリ確保しコピー）

    • 6. tuplestore_putvalues を使用して、構築した行を結果セットに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**: verify_gist_page, PageGetMaxOffsetNumber, PageGetItemId, PageGetItem, IndexTupleSize, tuplestore_putvalues 等。

関連データ構造

  • IndexTuple: インデックス内の各項目を表現する構造体。
  • Page: ページヘッダーおよびアイテムポインタを含むメモリ上のページ表現。
  • ReturnSetInfo: SQL関数がセットを返すための制御構造体。

失敗モードと境界条件

  • スーパーユーザーでない場合: ERRCODE_INSUFFICIENT_PRIVILEGE でエラー。
  • 無効な ItemId を検出した場合: elog(ERROR) で処理を中断。
  • 削除済みページ: NOTICE を出力し、アイテムの走査をスキップする。

保守担当者向けメモ

  • 本関数は gist_page_items と異なり、インデックスの定義（OID）を必要とせず、生のバイナリをダンプする。
  • palloc を使用してタプルデータをコピーしているため、メモリ消費量はページ内のアイテム数とサイズに比例する。
