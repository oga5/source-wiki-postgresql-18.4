expand_array

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_expanded.c:49

目的

配列 Datum を、メモリ上で効率的に操作可能な「展開済み配列（Expanded Array）」形式へ変換します。

シグネチャまたは形状

Datum expand_array(Datum arraydatum, MemoryContext parentcontext, ArrayMetaState *metacache)

入力と出力

  • **入力**:
  • arraydatum: 変換対象の配列 Datum。
  • parentcontext: 展開済みオブジェクトを割り当てるための親メモリコンテキスト。
  • metacache: 配列の型情報（要素型、長さ、アライメント等）を保持するキャッシュ構造体（省略可）。
  • **出力**:
  • 展開済み配列への読み書き可能な Datum ポインタを返します。

前提条件と不変条件

  • parentcontext は有効なメモリコンテキストである必要があります。
  • 戻り値の ExpandedArrayHeader は EA_MAGIC で初期化され、EA_methods が設定されます。
  • 入力が既に展開済み配列である場合、可能な限りコピーを回避して再利用されます。

アルゴリズムまたは振る舞い

1. **メモリ確保**: parentcontext の下に専用のメモリコンテキストを作成し、ExpandedArrayHeader を初期化します。 2. **最適化パス**: 入力が既に展開済み配列の場合、そのメタデータを再利用します。特に要素が値渡し（pass-by-value）かつ分解済みであれば、その構造をコピーして即座に返します。 3. **フラットコピー**: 上記以外の場合、ソース配列を DatumGetArrayTypePCopy でコピーし、フラットな形式として保持します。 4. **メタデータ設定**: 配列の次元、境界、要素型情報を設定します。metacache が提供されていればそれを利用し、なければ get_typlenbyvalalign で型情報を取得・キャッシュします。 5. **遅延評価**: この時点では要素の分解（dvalues 等の生成）は行わず、フラットな配列へのポインタのみを保持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/array_expanded.c (内部ヘルパー)
  • src/backend/utils/adt/arrayfuncs.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • AllocSetContextCreate, MemoryContextAlloc, MemoryContextSwitchTo
  • EOH_init_header, EOHPGetRWDatum
  • DatumGetArrayTypePCopy, get_typlenbyvalalign
  • copy_byval_expanded_array (条件付き)

関連データ構造

  • ExpandedArrayHeader: 展開済み配列のヘッダー。
  • ArrayMetaState: 配列の型情報を保持するキャッシュ。
  • ArrayType: フラットな配列形式。
  • EA_methods: 展開済みオブジェクトのメソッドテーブル。

失敗モードと境界条件

  • メモリ不足時には MemoryContext 関連関数によりエラーが発生します。
  • metacache が NULL の場合、内部で一時的なキャッシュを使用して型情報のルックアップを行います。
  • TOAST された配列を扱う際、一時的にメモリリークのリスクがありますが、最小限に抑えられています。

保守担当者向けメモ

  • 本関数は、配列に対する連続的な変更操作を行う前の準備として呼び出されます。
  • dvalues や dnulls は遅延評価されるため、アクセスが必要になった時点で deconstruct_expanded_array が呼び出される設計になっています。
  • パフォーマンス向上のため、ループ内では metacache を活用することが推奨されます。
