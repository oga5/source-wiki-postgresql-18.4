EndReplicationCommand

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/dest.c:204

目的

レプリケーションコマンドの完了をクライアントに通知する。EndCommand の簡略化されたバージョンとして機能する。

シグネチャまたは形状

void EndReplicationCommand(const char *commandTag)

入力と出力

  • **入力**: commandTag (完了したコマンドを示す文字列)
  • **出力**: なし（副作用としてクライアントへメッセージを送信）

前提条件と不変条件

  • commandTag は有効な文字列ポインタであること。
  • 呼び出し時点で、通信プロトコル層がメッセージ送信可能な状態にあること。

アルゴリズムまたは振る舞い

1. pq_putmessage を呼び出し、PqMsg_CommandComplete メッセージを構築する。 2. 引数として渡された commandTag とその長さをメッセージペイロードとして送信する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c 内の各所（例: START_STREAMING コマンド処理後など）。
  • **呼び出し先**: pq_putmessage（libpq プロトコル層）。

関連データ構造

  • PqMsg_CommandComplete: 通信プロトコル上のメッセージタイプ識別子。

失敗モードと境界条件

  • commandTag が NULL の場合、strlen でセグメンテーション違反が発生する可能性がある。
  • 通信エラーが発生した場合、pq_putmessage 内部でエラーが報告される。

保守担当者向けメモ

  • 本関数は EndCommand の軽量版であり、レプリケーション特有の完了通知を行う。
  • walsender.c から頻繁に呼び出されるため、変更時はレプリケーションプロトコルの互換性に注意すること。
