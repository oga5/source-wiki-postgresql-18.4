defGetInt64

定義

  • 種別: function
  • 定義箇所: src/backend/commands/define.c:172

目的

DefElem 構造体の arg フィールドから 64 ビット整数（int64）を抽出・変換する。

シグネチャまたは形状

int64 defGetInt64(DefElem *def)

入力と出力

  • **入力**: DefElem *def（解析対象の定義要素）
  • **出力**: 抽出された int64 値
  • **戻り値**: 成功時は変換された整数値。失敗時はエラーにより処理を中断（ereport）。

前提条件と不変条件

  • def は非 NULL であること。
  • def->arg が NULL の場合、または数値型（T_Integer または T_Float）以外の場合はエラーとなる。

アルゴリズムまたは振る舞い

1. def->arg が NULL の場合、ERRCODE_SYNTAX_ERROR でエラーを報告する。 2. nodeTag(def->arg) を判定する：

  • T_Integer: intVal() マクロを使用して値を抽出し、int64 にキャストして返す。
  • T_Float: int8in 関数を呼び出し、文字列形式の数値を int64 に変換して返す。これは int4 の範囲を超える大きな数値を扱うための処理である。

    • 3. 上記以外の場合、ERRCODE_SYNTAX_ERROR でエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/backend/commands/copy.c
  • src/backend/commands/sequence.c
  • **呼び出し先**:
  • intVal (マクロ)
  • nodeTag (マクロ)
  • DirectFunctionCall1
  • int8in (PostgreSQL 内部関数)
  • ereport

関連データ構造

  • DefElem: SQL コマンドのオプション引数を保持するノード。
  • Node: DefElem->arg の型。

失敗モードと境界条件

  • def->arg が NULL の場合: ERRCODE_SYNTAX_ERROR を発生させる。
  • def->arg が数値型でない場合: ERRCODE_SYNTAX_ERROR を発生させる。
  • T_Float で渡された値が int8 として不正な形式の場合: int8in 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は int4 範囲外の数値を Float として受け取るパーサーの挙動を補完している。
  • 新しい数値型ノードが追加された場合は、switch 文の拡張を検討すること。
