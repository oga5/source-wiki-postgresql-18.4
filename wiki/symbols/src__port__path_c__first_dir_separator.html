first_dir_separator

定義

  • 種別: function
  • 定義箇所: src/port/path.c:109

目的

与えられたファイルパス文字列内で、最初に出現するディレクトリ区切り文字の位置を特定します。

シグネチャまたは形状

char *first_dir_separator(const char *filename);

入力と出力

  • **入力**: filename (ディレクトリパスを含む文字列へのポインタ)
  • **出力**: 最初に見つかったディレクトリ区切り文字へのポインタ。区切り文字が見つからない場合は NULL を返します。

前提条件と不変条件

  • filename はヌル終端文字列である必要があります。
  • skip_drive を使用してドライブレター部分をスキップした後の文字列から検索を開始します。
  • IS_DIR_SEP マクロにより、プラットフォーム固有の区切り文字（Unix系では /、Windowsでは / または \）が判定されます。

アルゴリズムまたは振る舞い

1. skip_drive(filename) を呼び出し、ドライブレター（存在する場合）をスキップした位置から走査を開始します。 2. 文字列の終端に達するまで、各文字に対して IS_DIR_SEP を評価します。 3. 区切り文字が見つかった時点で、その位置へのポインタを char * にキャストして返します。 4. 文字列の最後まで区切り文字が見つからなかった場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/extension.c
  • src/backend/utils/fmgr/dfmgr.c
  • src/backend/utils/init/miscinit.c
  • src/common/exec.c
  • **呼び出し先**:
  • skip_drive (同ファイル内)
  • IS_DIR_SEP (マクロ)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • filename が NULL の場合の動作は定義されていません（セグメンテーション違反の可能性があります）。
  • ドライブレターのみ、または区切り文字を含まないパスが渡された場合、NULL を返します。

保守担当者向けメモ

  • unconstify を使用して const char * を char * にキャストして返しているため、呼び出し側で返り値の内容を書き換える際は注意してください。
  • Windows 環境におけるマルチバイト文字（Shift-JIS 等）の \（0x5C）問題については、src/port/path.c 全体の設計方針に従い、必要に応じて pg_wchar.h の関数と併用してください。
