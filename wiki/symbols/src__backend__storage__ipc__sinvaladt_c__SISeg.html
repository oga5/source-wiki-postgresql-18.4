SISeg

定義

  • **種別**: struct
  • **定義箇所**: src/backend/storage/ipc/sinvaladt.c:165

目的

共有メモリ上で、全バックエンド間におけるキャッシュ無効化メッセージの伝播と、各プロセスの読み取り進捗を管理するためのメインデータ構造です。

シグネチャまたは形状

typedef struct SISeg {
    int minMsgNum;
    int maxMsgNum;
    int nextThreshold;
    slock_t msgnumLock;
    SharedInvalidationMessage buffer[MAXNUMMESSAGES];
    int numProcs;
    int *pgprocnos;
    ProcState procState[FLEXIBLE_ARRAY_MEMBER];
} SISeg;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。共有メモリを介して、SIInsertDataEntries（書き込み）および SIGetDataEntries（読み取り）の対象となります。

前提条件と不変条件

  • msgnumLock は maxMsgNum へのアクセスを保護し、マルチプロセッサ環境でのメモリ順序付けを保証します。
  • procState は pgprocno をインデックスとして使用します。
  • pgprocnos 配列は、現在使用中のスロットを効率的にスキャンするために保持される密な配列です。

アルゴリズムまたは振る舞い

  • **循環バッファ管理**: buffer にメッセージを蓄積し、minMsgNum から maxMsgNum までの範囲でメッセージを管理します。
  • **クリーンアップ**: nextThreshold に達すると SICleanupQueue が呼び出され、全バックエンドが読み終えた古いメッセージを解放して領域を再利用します。
  • **進捗追跡**: 各バックエンドの ProcState を通じて、個別の読み取り進捗を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: shmInvalBuffer を介して src/backend/storage/ipc/sinvaladt.c 内の各関数（SIInsertDataEntries, SIGetDataEntries, SICleanupQueue 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • SharedInvalidationMessage: buffer 内に格納される個別の無効化メッセージ。
  • ProcState: 各バックエンドの進捗状態を保持する構造体。

失敗モードと境界条件

  • **バッファ溢れ**: メッセージの蓄積がバッファ容量を超えた場合、SICleanupQueue による強制的なクリーンアップが行われます。
  • **リセット状態**: 処理が極端に遅延し、バッファから溢れたバックエンドは resetState フラグが立てられ、キャッシュの再構築を強制されます。

保守担当者向けメモ

  • pgprocnos は ProcArrayStruct と重複していますが、ProcArrayLock の競合を避けるために独立して保持されています。
  • SIGetDataEntries は共有ロックを取得しますが、これは自身の ProcState 更新を許可するための設計であり、読み取り専用ではありません。
