SnapBuildCurrentState

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/snapbuild.c:276

目的

スナップショット構築プロセスにおける現在の状態（SnapBuildState）を取得します。

シグネチャまたは形状

SnapBuildState SnapBuildCurrentState(SnapBuild *builder)

入力と出力

  • **入力**: builder (SnapBuild *) - スナップショット構築の状態を保持する構造体へのポインタ。
  • **出力**: SnapBuildState - 現在の構築フェーズ（START, BUILDING_SNAPSHOT, FULL_SNAPSHOT, CONSISTENT のいずれか）。

前提条件と不変条件

  • builder は有効な SnapBuild 構造体を指している必要があります。

アルゴリズムまたは振る舞い

builder 構造体内の state メンバを直接参照し、その値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/decode.c: WALデコード処理において、現在の状態が SNAPBUILD_FULL_SNAPSHOT や SNAPBUILD_CONSISTENT に到達しているかを確認するために使用。
  • src/backend/replication/logical/logical.c: 論理レプリケーションコンテキストにおいて、状態が SNAPBUILD_CONSISTENT であるかを確認するために使用。
  • src/backend/replication/logical/reorderbuffer.c: トランザクション処理において、状態が SNAPBUILD_CONSISTENT に達しているかを確認するために使用。
  • **呼び出し先**: なし。

関連データ構造

  • SnapBuild: スナップショット構築のメイン状態構造体。
  • SnapBuildState: 構築プロセスの段階を定義する列挙型。

失敗モードと境界条件

  • builder が NULL の場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は単なるゲッターであり、副作用はありません。
  • 状態遷移の判定ロジック（SNAPBUILD_FULL_SNAPSHOT や SNAPBUILD_CONSISTENT との比較）は呼び出し元に依存しているため、状態遷移の定義を変更する際は呼び出し元の条件式も併せて確認してください。
