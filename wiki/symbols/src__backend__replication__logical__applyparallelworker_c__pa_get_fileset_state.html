pa_get_fileset_state

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/applyparallelworker.c:1533

目的

並列適用ワーカーの共有メモリ状態から、現在のファイルセット状態（PartialFileSetState）を取得する。

シグネチャまたは形状

PartialFileSetState pa_get_fileset_state(void)

入力と出力

  • 入力: なし（グローバルな共有メモリ状態を参照）
  • 出力: PartialFileSetState 型の値（現在のファイルセットの状態）

前提条件と不変条件

  • 呼び出し元が並列適用ワーカー（am_parallel_apply_worker() が真）であることを前提とする。
  • MyParallelShared が初期化済みであること。

アルゴリズムまたは振る舞い

1. am_parallel_apply_worker() を用いて、呼び出し元が並列適用ワーカーであることを検証する。 2. MyParallelShared->mutex を用いてスピンロックを獲得する。 3. 共有メモリ上の MyParallelShared->fileset_state をローカル変数にコピーする。 4. スピンロックを解放し、コピーした状態を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/applyparallelworker.c 内の以下の箇所から呼び出されることが確認されている。
  • 646行目
  • 662行目
  • 683行目
  • 呼び出し先:
  • am_parallel_apply_worker()
  • SpinLockAcquire()
  • SpinLockRelease()

関連データ構造

  • PartialFileSetState: ファイルセットの状態を保持する列挙型または型。
  • MyParallelShared: 並列適用ワーカーが共有するメモリ領域を指すポインタ。

失敗モードと境界条件

  • am_parallel_apply_worker() が偽の場合、アサーションエラーによりプロセスが停止する。

保守担当者向けメモ

  • 本関数は共有メモリへのアクセスを同期するためにスピンロックを使用している。高頻度で呼び出される場合、ロック競合の可能性を考慮すること。
