make_expanded_record_from_exprecord

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/expandedrecord.c:328

目的

既存の展開されたレコード（ExpandedRecordHeader）を基に、新しいメモリコンテキスト上でその構造を複製した新しい展開レコードを作成します。

シグネチャまたは形状

ExpandedRecordHeader *make_expanded_record_from_exprecord(ExpandedRecordHeader *olderh, MemoryContext parentcontext)

入力と出力

  • **入力**:
  • olderh: 複製元の ExpandedRecordHeader へのポインタ。
  • parentcontext: 新しいオブジェクトを割り当てるための親メモリコンテキスト。
  • **出力**:
  • 新規作成された ExpandedRecordHeader へのポインタ。

前提条件と不変条件

  • olderh は有効な ExpandedRecordHeader であること。
  • 新規作成されたレコードは、初期状態では論理的に空（ER_FLAG_DVALUES_VALID および ER_FLAG_FVALUE_VALID が未設定）です。
  • tupdesc の参照カウント管理は、tdrefcount >= 0 の場合に MemoryContextRegisterResetCallback を用いて自動的に行われます。

アルゴリズムまたは振る舞い

1. parentcontext の下に新しいメモリコンテキスト（objcxt）を作成します。 2. ExpandedRecordHeader と、フィールド用の dvalues/dnulls 配列を保持するために必要なメモリを objcxt 上に一括確保します。 3. ヘッダーを初期化し、er_magic や nfields などのメタデータを設定します。 4. 複製元の型情報（er_decltypeid, er_typeid, er_typmod, er_tupdesc_id）および ER_FLAG_IS_DOMAIN フラグをコピーします。 5. tupdesc の扱いを決定します：

  • tdrefcount >= 0 の場合: 参照カウントをインクリメントし、コンテキストリセット時にデクリメントするコールバックを登録します。
  • ER_FLAG_TUPDESC_ALLOCED が設定されている場合: CreateTupleDescCopy を使用して新しいコピーを作成します。
  • それ以外の場合: ポインタをそのまま共有します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpgsql/src/pl_exec.c (複数箇所でレコードの複製に使用)
  • **呼び出し先**:
  • expanded_record_get_tupdesc
  • AllocSetContextCreate
  • MemoryContextAlloc
  • EOH_init_header
  • MemoryContextRegisterResetCallback
  • ER_mc_callback
  • CreateTupleDescCopy
  • MemoryContextSwitchTo

関連データ構造

  • ExpandedRecordHeader: 展開されたレコードのメイン構造体。
  • TupleDesc: タプルの記述子。
  • MemoryContext: メモリ管理コンテキスト。

失敗モードと境界条件

  • メモリ不足時には palloc 関連関数によりエラーが発生します。
  • tupdesc のコピーが必要な場合、メモリコンテキストの切り替えが正しく行われないとメモリリークや不正なメモリ参照を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は、既存のレコードのメタデータと型情報を引き継ぎつつ、独立したメモリ管理単位を作成するために使用されます。
  • ER_FLAG_DVALUES_VALID 等が設定されないため、呼び出し側は必要に応じてフィールド値を再構築する必要があります。
