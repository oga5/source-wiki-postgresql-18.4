find_reconstructed_block_length

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/reconstruct.c:438

目的

指定された rfile 構造体が保持するブロック情報に基づき、再構築後のファイルに必要なブロック数を算出します。

シグネチャまたは形状

static unsigned find_reconstructed_block_length(rfile *s);

入力と出力

  • **入力**: rfile *s（再構築対象のソースファイル情報）
  • **出力**: unsigned（再構築後のファイルが持つべきブロック数）

前提条件と不変条件

  • 引数 s は有効な rfile 構造体を指している必要があります。
  • s->relative_block_numbers 配列には、各ブロックの相対的なブロック番号が格納されている必要があります。

アルゴリズムまたは振る舞い

1. 初期値として s->truncation_block_length を block_length に設定します。 2. s->num_blocks 分だけループし、各ブロックの相対番号 s->relative_block_numbers[i] を確認します。 3. 現在の block_length よりも大きいブロック番号が見つかった場合、その番号に 1 を加えた値を新しい block_length とします。 4. 最終的な block_length を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_combinebackup/reconstruct.c:131
  • **呼び出し先**:
  • 提供されたソースコード内では、本関数が他の関数を呼び出す記述はありません。

関連データ構造

  • rfile: 再構築に使用するソースファイルのメタデータ（ブロック数、相対ブロック番号配列、切り詰め長など）を保持する構造体。

失敗モードと境界条件

  • s->num_blocks が 0 の場合、初期値である s->truncation_block_length がそのまま返されます。
  • relative_block_numbers 内の最大値が truncation_block_length 未満の場合、truncation_block_length が結果となります。

保守担当者向けメモ

  • 本関数は、増分バックアップからファイルを再構築する際のファイルサイズ決定ロジックの一部です。rfile 構造体の定義変更時には、本関数のループ処理が正しくブロック番号を参照しているか確認してください。
