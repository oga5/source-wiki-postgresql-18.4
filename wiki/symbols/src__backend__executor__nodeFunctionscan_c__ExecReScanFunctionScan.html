ExecReScanFunctionScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeFunctionscan.c:555

目的

FunctionScan ノードの再スキャン処理を実行し、関数の出力結果（Tuplestore）の再利用または再計算を制御する。

シグネチャまたは形状

void ExecReScanFunctionScan(FunctionScanState *node)

入力と出力

  • **入力**: FunctionScanState *node（実行中の関数スキャンノードの状態）
  • **出力**: なし（副作用としてノード内部状態をリセット）

前提条件と不変条件

  • node は初期化済みであること。
  • node->ss.ps.chgParam に変更されたパラメータが設定されている場合、それに応じて関数の再計算が必要となる。

アルゴリズムまたは振る舞い

1. **スロットのクリア**: ノードの ps_ResultTupleSlot および各関数ごとの func_slot を ExecClearTuple でクリアする。 2. **共通スキャンリセット**: ExecScanReScan(&node->ss) を呼び出し、スキャン共通の再スキャン処理を行う。 3. **パラメータ変更の判定**: chgParam が存在する場合、各関数のパラメータ（rtfunc->funcparams）と照合する。変更がある場合は tuplestore_end で既存の Tuplestore を破棄し、rowcount を -1 にリセットして再計算を強制する。 4. **序数リセット**: node->ordinal を 0 にリセットする。 5. **再スキャン実行**: 破棄されなかった Tuplestore に対して tuplestore_rescan を呼び出し、読み取り位置を先頭に戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c（ExecReScan 経由での呼び出しが確認されている）。
  • **呼び出し先**:
  • ExecClearTuple
  • ExecScanReScan
  • tuplestore_end
  • tuplestore_rescan
  • bms_overlap

関連データ構造

  • FunctionScanState: ノードの実行状態を保持。
  • FunctionScan: 実行計画ノード。
  • FunctionScanPerFuncState: 各関数の Tuplestore や状態を管理。

失敗モードと境界条件

  • chgParam が NULL の場合は再計算を行わず、既存の Tuplestore を再利用する。
  • Tuplestore がまだ生成されていない（NULL）場合、その関数に対する tuplestore_end や tuplestore_rescan はスキップされる。

保守担当者向けメモ

  • 本関数は FunctionScan の再実行ロジックの中核であり、パラメータ変更時の再計算判定が重要である。
  • 関数の揮発性（Volatility）に基づく再計算の自動化については、現状の設計では明示的に条件分岐されていない（ソース内の XXX コメント参照）。
