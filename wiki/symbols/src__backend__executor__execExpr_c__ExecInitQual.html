ExecInitQual

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExpr.c:228

目的

クエリの制約条件（Qual）を評価するための ExprState を構築・コンパイルします。実行時に ExecQual() から呼び出されることを想定した命令配列を生成します。

シグネチャまたは形状

ExprState *ExecInitQual(List *qual, PlanState *parent)

入力と出力

  • **入力**:
  • qual: 評価対象となる式リスト（暗黙のAND形式）。
  • parent: この式を保持する親の PlanState。
  • **出力**:
  • 構築された ExprState 構造体へのポインタ。qual が NIL の場合は NULL を返します。

前提条件と不変条件

  • qual が List 型であること。
  • 戻り値の ExprState は EEO_FLAG_IS_QUAL フラグが設定され、ExecQual() での利用に最適化されます。
  • 評価結果が NULL の場合、ExecQual() は false を返す必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: qual が空であれば即座に NULL を返します。ExprState を作成し、EEO_FLAG_IS_QUAL を設定します。 2. **セットアップ**: ExecCreateExprSetupSteps を呼び出し、必要なタプルデフォーム等の準備ステップを挿入します。 3. **命令生成**: 各式ノードに対して ExecInitExprRec を呼び出し、評価結果を ExprState の resvalue/resnull に格納します。 4. **評価ステップの追加**: EEOP_QUAL 命令を生成し、式が偽または NULL と判定された場合にショートサーキット（短絡評価）するためのジャンプ先を管理します。 5. **完了処理**: 最後に EEOP_DONE_RETURN を追加し、ExecReadyExpr を呼び出して実行準備を完了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c
  • src/backend/executor/execExpr.c
  • src/backend/executor/execPartition.c
  • **呼び出し先**:
  • ExecCreateExprSetupSteps
  • ExecInitExprRec
  • ExprEvalPushStep
  • ExecReadyExpr
  • makeNode

関連データ構造

  • ExprState: 式評価の状態を保持するメイン構造体。
  • ExprEvalStep: 実行時の命令単位。
  • PlanState: 親ノードの情報。

失敗モードと境界条件

  • qual が NIL の場合、関数は NULL を返します。
  • 式ツリーが非常に深い場合、ExecInitExprRec の再帰呼び出しによりスタックオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数は ExecQual() のホットパスを最適化するため、BOOL_AND よりも単純な EEOP_QUAL 命令を使用しています。
  • EEOP_QUAL のジャンプ先（jumpdone）は、すべての式ノードのコンパイル後に確定されるため、adjust_jumps リストを用いて後から更新しています。
  • execExprInterp.c のインタプリタと命令コードの整合性を維持してください。
