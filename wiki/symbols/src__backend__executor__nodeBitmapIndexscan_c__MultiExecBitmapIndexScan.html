MultiExecBitmapIndexScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapIndexscan.c:48

目的

インデックスをスキャンして条件に合致するタプルID（TID）を抽出し、TIDBitmap を構築して返却する。

シグネチャまたは形状

Node *MultiExecBitmapIndexScan(BitmapIndexScanState *node)

入力と出力

  • **入力**: BitmapIndexScanState *node（スキャン対象のインデックス情報や実行状態を保持するノード）。
  • **出力**: Node *（構築された TIDBitmap へのポインタ）。

前提条件と不変条件

  • ExecProcNode インターフェースには対応していないため、直接呼び出す必要がある。
  • tbm_create で生成されるビットマップは work_mem の制限を受ける。
  • 並列クエリ実行時は、統計情報が共有メモリ（DSM）を介して管理される。

アルゴリズムまたは振る舞い

1. **計測開始**: node->ss.ps.instrument が存在する場合、実行時間を計測する。 2. **ランタイムキー処理**: biss_RuntimeKeysReady が偽であれば ExecReScan を呼び出し、インデックススキャンキーを準備する。 3. **ビットマップ準備**: node->biss_result が存在すればそれを再利用し、なければ tbm_create で新規作成する。 4. **スキャン実行**: index_getbitmap を用いてインデックスからTIDを抽出し、ビットマップに挿入する。 5. **反復処理**: 配列キー（ArrayKeys）が存在する場合、ExecIndexAdvanceArrayKeys でキーを更新し、index_rescan を呼び出してすべての条件を満たすまでスキャンを繰り返す。 6. **終了処理**: CHECK_FOR_INTERRUPTS() を呼び出し、計測を停止してビットマップを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:529（直接的な呼び出しを確認）。
  • **呼び出し先**:
  • ExecReScan
  • tbm_create
  • index_getbitmap
  • ExecIndexAdvanceArrayKeys
  • index_rescan
  • InstrStartNode / InstrStopNode
  • CHECK_FOR_INTERRUPTS

関連データ構造

  • BitmapIndexScanState: スキャンノードの実行状態。
  • TIDBitmap: 抽出されたTIDを保持するビットマップ構造。
  • IndexScanDesc: インデックススキャンの記述子。

失敗モードと境界条件

  • work_mem の制限を超えた場合、ビットマップ構築時にメモリ不足が発生する可能性がある。
  • 配列キーが空の場合、スキャンは実行されない。
  • 割り込みが発生した場合、CHECK_FOR_INTERRUPTS() により処理が中断される。

保守担当者向けメモ

  • 本関数は ExecProcNode インターフェースを実装していないため、呼び出し側は必ず MultiExecBitmapIndexScan を直接使用すること。
  • node->biss_result が設定されている場合、既存のビットマップに対して論理和（OR）をとる挙動となるため、呼び出し側の意図を確認すること。
