getlexeme

定義

  • 種別: function
  • 定義箇所: contrib/ltree/lquery_op.c:24

目的

文字列（またはバッファ）から、アンダースコア（_）で区切られた次の有効なレキシーム（トークン）を抽出します。

シグネチャまたは形状

static char *getlexeme(char *start, char *end, int *len)

入力と出力

  • **入力**:
  • start: 走査を開始するバッファのポインタ。
  • end: バッファの終端ポインタ。
  • len: 抽出されたレキシームの長さを格納するための出力用ポインタ。
  • **出力**:
  • 戻り値: 抽出されたレキシームの開始位置を指すポインタ。これ以上レキシームが存在しない場合は NULL を返します。
  • *len: 抽出されたレキシームのバイト長が設定されます。

前提条件と不変条件

  • start から end までの範囲は有効なメモリ領域である必要があります。
  • pg_mblen_range を使用してマルチバイト文字を考慮した走査を行います。

アルゴリズムまたは振る舞い

1. **スキップ**: start ポインタから開始し、アンダースコア（_）が連続する間はポインタを進めます。 2. **終端判定**: スキップ後のポインタが end に達している場合は NULL を返します。 3. **抽出**: アンダースコア以外の文字が続く間、ポインタを進めてレキシームの範囲を特定します。 4. **結果格納**: 特定したレキシームの長さを len に書き込み、開始位置を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/lquery_op.c 内のループ処理において、クエリやツリーの各要素を順次取得するために呼び出されています。
  • **呼び出し先**:
  • t_iseq: 文字が指定された文字（_）と一致するか判定するマクロ。
  • pg_mblen_range: マルチバイト文字の長さを取得する関数。

関連データ構造

  • ltree および lquery 型の内部表現（ラベルの区切りとしてアンダースコアを使用する構造）。

失敗モードと境界条件

  • start >= end の場合、またはアンダースコアのみで構成されている場合は NULL を返します。

保守担当者向けメモ

  • 本関数は ltree 関連の演算子実装において、文字列をトークン化する基本的なユーティリティです。
  • t_iseq マクロの定義および pg_mblen_range の挙動が、ロケール設定やエンコーディングに依存することに注意してください。
