initStringInfo

定義

  • 種別: function
  • 定義箇所: src/common/stringinfo.c:96

目的

StringInfoData 構造体を初期化し、デフォルトサイズのバッファを割り当てることで、可変長文字列操作のための準備を行う。

シグネチャまたは形状

void initStringInfo(StringInfo str)

入力と出力

  • **入力**: str (初期化対象となる StringInfo 構造体へのポインタ)
  • **出力**: なし（副作用として str が指す構造体のメンバが初期化され、メモリが割り当てられる）

前提条件と不変条件

  • 引数 str は有効な StringInfoData 構造体を指している必要がある。
  • 内部で palloc（またはフロントエンド環境での malloc）を使用するため、メモリ割り当てが可能な状態であること。

アルゴリズムまたは振る舞い

1. STRINGINFO_DEFAULT_SIZE を指定して内部関数 initStringInfoInternal を呼び出す。 2. initStringInfoInternal は、指定されたサイズのメモリを確保し、StringInfoData の data、len、maxlen、cursor を適切に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/cube/cube.c
  • contrib/dblink/dblink.c
  • contrib/hstore/hstore_io.c

    • (上記はソースコード内の直接的な参照に基づく)

  • **呼び出し先**:
  • initStringInfoInternal (同一ファイル内の内部関数)

関連データ構造

  • StringInfoData: バッファの実体（data, len, maxlen, cursor を保持）。
  • StringInfo: StringInfoData へのポインタ型。

失敗モードと境界条件

  • メモリ不足時には、palloc の仕様に従いエラーが発生する（バックエンドでは elog(ERROR)、フロントエンドでは malloc の失敗として処理される）。

保守担当者向けメモ

  • 本関数はデフォルトサイズでの初期化を行うラッパーである。特定の初期サイズが必要な場合は initStringInfoExt を使用すること。
  • StringInfo のメモリ管理は palloc/pfree に依存しているため、メモリコンテキストの生存期間に注意が必要である。
