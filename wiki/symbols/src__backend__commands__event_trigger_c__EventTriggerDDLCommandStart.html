EventTriggerDDLCommandStart

定義

  • 種別: function
  • 定義箇所: src/backend/commands/event_trigger.c:724

目的

DDLコマンドの実行開始時に、登録されているイベントトリガーを特定し、該当するトリガー関数を呼び出すためのエントリーポイント。

シグネチャまたは形状

void EventTriggerDDLCommandStart(Node *parsetree)

入力と出力

  • **入力**: parsetree (実行されるDDLコマンドの構文木)
  • **出力**: なし（副作用としてトリガー関数の実行およびコマンドカウンタの更新を行う）

前提条件と不変条件

  • スタンドアロンモード（!IsUnderPostmaster）またはイベントトリガー機能が無効（!event_triggers）な場合、本関数は即座にリターンする。
  • トリガー実行は専用のメモリコンテキストで行われる。

アルゴリズムまたは振る舞い

1. **ガードチェック**: スタンドアロンモードまたはイベントトリガー無効時は処理をスキップする。 2. **セットアップ**: EventTriggerCommonSetup を呼び出し、ddl_command_start イベントに関連付けられたトリガーリストを取得する。 3. **実行**: リストが空でなければ EventTriggerInvoke を呼び出し、各トリガー関数を実行する。 4. **後処理**:

  • 取得したトリガーリストのメモリを解放する。
  • CommandCounterIncrement() を呼び出し、トリガー実行による変更をメインのDDLコマンドから可視化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c (DDLコマンド実行のメインフロー)
  • **呼び出し先**:
  • EventTriggerCommonSetup
  • EventTriggerInvoke
  • list_free
  • CommandCounterIncrement

関連データ構造

  • Node (入力構文木)
  • EventTriggerData (トリガー実行コンテキスト)
  • List (トリガーリスト)

失敗モードと境界条件

  • **スタンドアロンモード**: 安全のためイベントトリガーは完全に無効化される。
  • **インデックス破損**: pg_event_trigger のインデックスが破損している場合、キャッシュ構築が失敗する可能性があるため、スタンドアロンモードでのリカバリが想定されている。

保守担当者向けメモ

  • 本関数はDDL実行のライフサイクルにおける重要なフックである。
  • 再帰的なDDL実行が発生する場合、スタック深さのチェックが重要となる。
  • CommandCounterIncrement の呼び出しは、トリガーが作成・変更したオブジェクトを後続の処理で確実に参照させるために不可欠である。
