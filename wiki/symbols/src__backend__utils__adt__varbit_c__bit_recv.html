bit_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:330

目的

外部バイナリ形式のデータストリームからビット列（BIT / BIT VARYING）を読み込み、PostgreSQL内部の VarBit 構造体に変換します。

シグネチャまたは形状

Datum bit_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して StringInfo バッファ（データソース）および int32 型の atttypmod（ビット長制約）を受け取ります。
  • **出力**: 変換された VarBit 構造体へのポインタを Datum として返します。

前提条件と不変条件

  • bitlen は 0 以上 VARBITMAXLEN 以下である必要があります。
  • atttypmod が正の値である場合、入力された bitlen と一致しなければなりません。
  • 処理終了時、VARBIT_PAD マクロにより、最後のバイトの未使用ビットがゼロクリアされている必要があります。

アルゴリズムまたは振る舞い

1. pq_getmsgint を使用して、バッファからビット長（bitlen）を読み取ります。 2. 長さの妥当性を検証し、制約違反がある場合はエラーを報告します。 3. palloc を用いて必要なメモリ領域を確保し、SET_VARSIZE および VARBITLEN を設定します。 4. pq_copymsgbytes を使用して、バッファからビットデータを VarBit 構造体のデータ領域へコピーします。 5. VARBIT_PAD を呼び出し、パディングビットをゼロ化します。 6. PG_RETURN_VARBIT_P を介して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 不明（PostgreSQLのバイナリ入力プロトコル経由で呼び出されることが想定されます）。
  • **呼び出し先**: pq_getmsgint, ereport, palloc, pq_copymsgbytes, VARBIT_PAD 等のユーティリティ関数。

関連データ構造

  • VarBit: ビット列を保持する構造体。
  • StringInfo: バイナリデータ転送用のバッファ構造体。

失敗モードと境界条件

  • **長さ不正**: bitlen が負または VARBITMAXLEN を超える場合、ERRCODE_INVALID_BINARY_REPRESENTATION でエラーとなります。
  • **型修飾子不一致**: atttypmod が指定されており、かつ bitlen と一致しない場合、ERRCODE_STRING_DATA_LENGTH_MISMATCH でエラーとなります。

保守担当者向けメモ

  • VARBIT_PAD によるゼロパディングは必須です。この不変条件が維持されない場合、比較演算（bit_cmp）の結果に悪影響を及ぼします。
  • bit_dat は最上位バイト（MSB）から配置されるため、ビット操作時はバイト境界を跨ぐシフト処理に注意してください。
