ExecInitSubPlanExpr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExpr.c:2820

目的

式評価のコンパイルフェーズにおいて、サブプラン（SubPlan）を評価するための命令ステップを生成し、ExprState に登録する。

シグネチャまたは形状

static void ExecInitSubPlanExpr(SubPlan *subplan, ExprState *state, Datum *resv, bool *resnull)

入力と出力

  • **入力**:
  • subplan: 初期化対象の SubPlan ノード。
  • state: 式評価の状態を保持する ExprState。
  • resv, resnull: 式評価結果を格納するためのポインタ。
  • **出力**: なし（state 内の命令リストおよび state->parent->subPlan リストを更新する）。

前提条件と不変条件

  • state->parent が存在すること（存在しない場合はエラー）。
  • subplan->parParam（パラメータIDリスト）と subplan->args（引数式リスト）の長さが一致していること。

アルゴリズムまたは振る舞い

1. **引数評価ステップの生成**: subplan->args に含まれる各式を ExecInitExprRec で再帰的にコンパイルする。 2. **パラメータ設定**: 各引数の評価結果を EEOP_PARAM_SET 命令を用いてパラメータへ反映させる。これにより、サブプラン実行時に必要な引数が確実に渡される。 3. **サブプランの初期化**: ExecInitSubPlan を呼び出し、SubPlanState を生成する。 4. **親への登録**: 生成された SubPlanState を state->parent->subPlan リストに追加する。 5. **実行ステップの追加**: EEOP_SUBPLAN 命令を ExprEvalPushStep で state に追加し、実行時にサブプランが評価されるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecInitExprRec（ソースコード内の文脈より、式ツリー走査中にサブプランノードに遭遇した際に呼び出される）。
  • **呼び出し先**:
  • ExecInitExprRec: 引数式のコンパイル用。
  • ExecInitSubPlan: サブプランの実行状態初期化用。
  • ExprEvalPushStep: 命令の登録用。

関連データ構造

  • SubPlan: 計画段階のサブプラン情報。
  • ExprState: 式評価の実行状態。
  • SubPlanState: サブプランの実行時状態。
  • ExprEvalStep: 式評価の命令単位。

失敗モードと境界条件

  • state->parent が NULL の場合、elog(ERROR) により処理を中断する。

保守担当者向けメモ

  • 本関数は式評価の「コンパイル」のみを行い、実行は行わない。
  • EEOP_PARAM_SET を使用することで、パラメータのポインタ値の安定性に依存せず、評価結果を安全に引き渡す設計となっている。
  • state->parent->subPlan への追加は、プランツリー全体でのサブプラン管理のために必須である。
