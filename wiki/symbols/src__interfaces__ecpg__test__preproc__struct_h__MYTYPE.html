MYTYPE

定義

  • 種別: typedef struct
  • 定義箇所: src/interfaces/ecpg/test/preproc/struct.h:9

目的

ECPG（Embedded SQL C Preprocessor）のテストにおいて、SQLのホスト変数として利用される構造体の型エイリアスを提供します。

シグネチャまたは形状

typedef struct mytype MYTYPE;

※ struct mytype の別名として定義されています。

入力と出力

  • 入力: なし（データ構造定義）
  • 出力: なし

前提条件と不変条件

  • ECPGプリプロセッサがホスト変数として認識可能な構造体である必要があります。
  • 構造体メンバーには整数、文字列、浮動小数点数が含まれることが想定されています。

アルゴリズムまたは振る舞い

本シンボル自体はデータ定義であり、実行時のアルゴリズムは持ちません。ECPGプリプロセッサがこの型定義を読み込み、SQL文とC言語の構造体メンバーをバインドするためのメタデータとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/interfaces/ecpg/test/expected/preproc-outofscope.c: get_var1 関数の引数型、malloc によるメモリ確保、および ECPGt_ 型変換マクロのキャスト先として参照されています。
  • 呼び出し先:
  • 情報なし（本シンボルは型定義であり、関数呼び出しを行いません）。

関連データ構造

  • struct mytype: 本型の実体となる構造体定義。
  • MYNULLTYPE: MYTYPE の各フィールドに対応するNULLインジケータを管理するための対となる構造体。

失敗モードと境界条件

  • 構造体のメモリレイアウトがECPGの期待する型マッピングと一致しない場合、プリプロセッサによるバインド処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルはプロダクションコードではなく、ECPGのプリプロセッサの挙動を検証するためのテスト用定義です。
  • 構造体のメンバー構成を変更する場合は、関連する .pgc テストファイル内の EXEC SQL 文との整合性を確認してください。
