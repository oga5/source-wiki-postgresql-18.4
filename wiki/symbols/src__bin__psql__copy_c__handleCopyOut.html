handleCopyOut

定義

  • 種別: function
  • 定義箇所: src/bin/psql/copy.c:433

目的

データベースサーバーから COPY TO プロトコルを通じてデータを受信し、指定されたローカルストリーム（ファイルまたはパイプ）へ書き出します。

シグネチャまたは形状

bool handleCopyOut(PGconn *conn, FILE *copystream, PGresult **res);

入力と出力

  • **入力**:
  • conn: データベース接続（PGconn）。
  • copystream: データ書き出し先のファイルポインタ（FILE）。
  • res: 最終的なコマンド実行結果を格納するための PGresult ポインタのポインタ。
  • **出力**:
  • 戻り値: 処理が正常に完了した場合は true、エラーが発生した場合は false。
  • res: サーバーから返された最終的な PGresult オブジェクト。

前提条件と不変条件

  • conn は COPY_OUT モードで初期化されている必要があります。
  • 処理中、サーバーから受信したデータバッファは PQfreemem で解放されます。

アルゴリズムまたは振る舞い

1. PQgetCopyData をループで呼び出し、サーバーからデータを受信します。 2. 受信したデータが有効な場合、fwrite を使用して copystream に書き出します。書き込み失敗時はエラーを記録し、残りのデータを読み込み続けます。 3. PQgetCopyData が終了コード（-1）を返すとループを抜けます。 4. ストリームのフラッシュ（fflush）を行い、書き込みエラーを確認します。 5. PQgetResult を呼び出し、サーバー側のコマンド完了ステータスを取得・検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/common.c 内の関数から呼び出されています。
  • **呼び出し先**: libpq の PQgetCopyData, PQfreemem, PQgetResult, PQerrorMessage, PQresultStatus を呼び出します。

関連データ構造

  • PGconn: libpq の接続オブジェクト。
  • PGresult: libpq のクエリ結果オブジェクト。
  • FILE: 標準 C ライブラリのストリーム。

失敗モードと境界条件

  • **書き込みエラー**: fwrite または fflush が失敗した場合、エラーログを出力し false を返します。
  • **転送エラー**: PQgetCopyData が -2（転送失敗）を返した場合、エラーメッセージを出力し false を返します。
  • **コマンド失敗**: PQgetResult のステータスが PGRES_COMMAND_OK でない場合、エラーとして扱われます。

保守担当者向けメモ

  • 本関数は COPY プロトコルの終了処理を確実に行うため、エラー発生時も PQgetCopyData の読み込みループを継続する設計になっています。
  • libpq が内部的に COPY_OUT 状態を維持している場合でも、本関数は PQgetResult を呼び出して正常な状態へ戻すことを試みます。
