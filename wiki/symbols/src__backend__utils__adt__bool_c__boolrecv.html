boolrecv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:174

目的

PostgreSQLのバイナリ通信プロトコルにおいて、外部から送信されたバイナリ形式のデータを boolean 型へ変換します。

シグネチャまたは形状

Datum boolrecv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して取得される StringInfo 型のバッファ。
  • **出力**: PG_RETURN_BOOL マクロを介して、変換された bool 値（Datum）を返します。

前提条件と不変条件

  • 入力バッファは libpq のバイナリプロトコルに従った形式である必要があります。
  • pq_getmsgbyte を使用するため、バッファ内に少なくとも1バイトのデータが存在することが前提となります。

アルゴリズムまたは振る舞い

1. pq_getmsgbyte を呼び出し、バッファから1バイトの整数値（ext）を読み取ります。 2. 読み取った値が 0 でない場合は true、0 の場合は false と判定します。 3. 判定結果を PG_RETURN_BOOL を通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）経由で、バイナリ形式のデータを受け取る際に呼び出されます。具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**: pq_getmsgbyte（libpq/pqformat.h）を呼び出します。

関連データ構造

  • StringInfo: PostgreSQLのバイナリ通信用データバッファ構造体。
  • Datum: PostgreSQL内部での汎用データ表現。

失敗モードと境界条件

  • バッファが空の場合、pq_getmsgbyte の内部仕様に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は boolsend と対になるバイナリI/O関数です。
  • 1バイトの値をそのまま真偽値として扱うシンプルな実装ですが、プロトコル仕様の変更時には pq_getmsgbyte の呼び出し部分に注意してください。
