xl_heap_insert

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam_xlog.h:160

目的

ヒープテーブルへのタプル挿入操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。リカバリ時に該当ページへの挿入を再現するために使用されます。

シグネチャまたは形状

typedef struct xl_heap_insert
{
    OffsetNumber offnum;        /* 挿入されたタプルのオフセット */
    uint8        flags;         /* 操作フラグ */
    /* xl_heap_header & TUPLE DATA がバックアップブロック0に続く */
} xl_heap_insert;

入力と出力

  • **入力**: 挿入対象のオフセット番号およびフラグ情報。
  • **出力**: WALレコードの一部としてシリアライズされたデータ。

前提条件と不変条件

  • 挿入されるタプルのヘッダー情報（xl_heap_header）およびタプルデータ本体は、WALレコードのバックアップブロック0に格納されている必要があります。
  • SizeOfHeapInsert マクロにより、構造体のサイズが定義されています。

アルゴリズムまたは振る舞い

1. ヒープへの挿入処理（heapam.c）において、本構造体が初期化されます。 2. XLogInsert を介してWALレコードとして書き込まれます。 3. リカバリ時、heap_redo 関数が XLogRecGetData を通じて本構造体を読み取り、指定されたオフセット位置にタプルを再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c: WALレコードの構築時に使用。
  • src/backend/replication/logical/decode.c: 論理デコード時に使用。
  • **呼び出し先**:
  • src/backend/access/heap/heapam_xlog.c: REDO処理にて本構造体を参照。
  • src/backend/access/rmgrdesc/heapdesc.c: WAL内容のデコード（可読化）に使用。

関連データ構造

  • xl_heap_header: タプルのメタデータ。
  • OffsetNumber: ページ内のタプル位置を特定する型。

失敗モードと境界条件

  • WALレコードのデータが破損している場合、XLogRecGetData によるキャスト後のフィールドアクセスで不正なメモリ参照が発生する可能性があります。
  • 8ビットのフラグ領域が定義されており、これを超えるフラグ追加は構造体の変更を伴います。

保守担当者向けメモ

  • 本構造体は可変長のタプルデータを伴うため、XLogRecGetData で取得したポインタの直後にタプルデータが配置されている前提で処理されています。
  • xl_heap_insert と xl_heap_multi_insert でフラグ値が共有されているため、フラグの追加・変更時は両方の実装への影響を確認してください。
