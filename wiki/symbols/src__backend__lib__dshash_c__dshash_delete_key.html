dshash_delete_key

定義

  • 種別: function
  • 定義箇所: src/backend/lib/dshash.c:502

目的

共有ハッシュテーブルから指定されたキーに対応するエントリを削除し、成功したかどうかを返す。

シグネチャまたは形状

bool dshash_delete_key(dshash_table *hash_table, const void *key)

入力と出力

  • **入力**:
  • hash_table: 操作対象の dshash_table 構造体へのポインタ。
  • key: 削除対象のキー。
  • **出力**:
  • bool: キーが見つかり、削除に成功した場合は true、見つからなかった場合は false を返す。

前提条件と不変条件

  • hash_table->control->magic が DSHASH_MAGIC と一致していること。
  • 呼び出し元がパーティションロックを保持していないこと（ASSERT_NO_PARTITION_LOCKS_HELD_BY_ME による検証）。

アルゴリズムまたは振る舞い

1. キーからハッシュ値を計算し、対応するパーティションを特定する。 2. 特定したパーティションの LWLock を排他モード（LW_EXCLUSIVE）で取得する。 3. ensure_valid_bucket_pointers を呼び出し、他プロセスによるリサイズの影響を反映させる。 4. delete_key_from_bucket を呼び出し、バケットからキーを検索・削除する。 5. 削除に成功した場合、該当パーティションの count をデクリメントする。 6. ロックを解放し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/launcher.c:1089
  • src/backend/utils/cache/typcache.c:3023
  • **呼び出し先**:
  • hash_key (ハッシュ計算)
  • PARTITION_LOCK (ロック取得)
  • ensure_valid_bucket_pointers (バケットポインタの整合性確保)
  • delete_key_from_bucket (削除処理)
  • LWLockAcquire / LWLockRelease (ロック制御)

関連データ構造

  • dshash_table: ハッシュテーブルの状態を管理する構造体。
  • dshash_table_control: 共有メモリ上の制御ブロック。

失敗モードと境界条件

  • キーが存在しない場合、false を返し、状態変更は行われない。
  • 削除時にパーティションの count が 0 以下になることは、不変条件としてアサートされている。

保守担当者向けメモ

  • 本関数は単一パーティションのロックを取得するため、高頻度な削除操作は特定のパーティションに競合を集中させる可能性がある。
  • 内部で ensure_valid_bucket_pointers を呼び出しているため、リサイズ中の他プロセスとの同期が保証されている。
