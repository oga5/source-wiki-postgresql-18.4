intset_leaf_node

定義

  • 種別: struct
  • 定義箇所: src/backend/lib/integerset.c:134

目的

B-tree 構造におけるリーフノードとして、圧縮された整数集合のデータを保持します。

シグネチャまたは形状

ソースコード上では typedef struct intset_leaf_node intset_leaf_node; として定義されています。構造体内部には、右側の兄弟ノードを指す next ポインタや、leaf_item 型の要素を保持するメンバが含まれます。

入力と出力

  • 入力: intset_new_leaf_node 関数により、MemoryContext からメモリが割り当てられます。
  • 出力: B-tree のリーフとして、検索やイテレーション時に leaf_item 形式のデータを提供します。

前提条件と不変条件

  • IntegerSet の一部として、昇順で追加される整数を保持します。
  • リーフノード間は next ポインタによって連結され、順次走査が可能です。

アルゴリズムまたは振る舞い

  • intset_flush_buffered_values により、バッファからフラッシュされたデータが本構造体に格納されます。
  • 検索時には、intset_is_member がルートからこのリーフノードまで辿り、simple8b_contains を用いて圧縮データ内の値を判定します。
  • イテレーション時には、iter_node として現在のリーフノードが追跡され、順次デコードが行われます。

呼び出し元と呼び出し先

  • 呼び出し元: intset_new_leaf_node（メモリ割り当て）、intset_add_member 関連のフラッシュ処理、および intset_is_member 等の検索処理。
  • 呼び出し先: MemoryContextAlloc（メモリ確保）。

関連データ構造

  • IntegerSet: 本ノードを管理するルート構造体。
  • leaf_item: リーフノード内に格納される最小単位のデータ構造。
  • intset_node: B-tree のノードを抽象化する型。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc が失敗する可能性があります。
  • 順序制約（昇順）に違反した挿入が行われた場合の動作は保証されません。
