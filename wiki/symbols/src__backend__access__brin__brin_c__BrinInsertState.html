BrinInsertState

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/brin/brin.c:192

目的

BRIN インデックスへの挿入操作において、複数のタプル挿入にまたがって保持すべき実行時状態を管理するために使用されます。

シグネチャまたは形状

typedef struct BrinInsertState
{
    BrinRevmap *bis_rmAccess;      /* Revmap へのアクセス用ハンドル */
    BrinDesc   *bis_desc;          /* インデックス記述子 */
    BlockNumber bis_pages_per_range; /* 範囲ごとのページ数 */
} BrinInsertState;

入力と出力

  • **入力**: initialize_brin_insertstate 関数により、対象のインデックスリレーション (idxRel) と IndexInfo を基に初期化されます。
  • **出力**: 挿入処理中のキャッシュとして IndexInfo->ii_AmCache に格納され、後続の挿入処理やクリーンアップ処理で参照されます。

前提条件と不変条件

  • IndexInfo->ii_AmCache を介して保持されるため、挿入操作のライフサイクルを通じて有効である必要があります。
  • 範囲要約が未作成（Revmap が NULL）の場合、挿入処理は本構造体を使用して適切に終了（スキップ）される必要があります。

アルゴリズムまたは振る舞い

  • 挿入操作の開始時に initialize_brin_insertstate でメモリ確保および初期化が行われます。
  • 複数のタプル挿入処理において、Revmap へのアクセスやインデックスのメタデータ（ページ範囲設定など）を再利用することで効率化を図ります。
  • 全ての挿入完了後、コールバックを通じて本構造体のクリーンアップが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: initialize_brin_insertstate が本構造体を生成します。また、brininsert 処理内で IndexInfo->ii_AmCache からキャストして取得されます。
  • **呼び出し先**: 構造体内部の bis_rmAccess を通じて BrinRevmap 関連の操作が行われます。

関連データ構造

  • BrinRevmap: ページ範囲とインデックス位置の対応管理に使用。
  • BrinDesc: インデックスの属性や演算クラス情報を含む記述子。
  • IndexInfo: PostgreSQL のインデックス情報構造体（ii_AmCache フィールドで本構造体を保持）。

失敗モードと境界条件

  • メモリ不足時には palloc0 によりエラーが発生する可能性があります。
  • ii_AmCache が正しく初期化されていない状態でアクセスした場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は brininsert のパフォーマンス最適化のためのキャッシュです。
  • 範囲要約の更新ロジック（brin_doupdate 等）を変更する際は、本構造体が保持する bis_rmAccess の状態整合性に注意してください。
