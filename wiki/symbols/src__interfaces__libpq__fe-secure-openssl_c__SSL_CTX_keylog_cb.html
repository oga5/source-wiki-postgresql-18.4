SSL_CTX_keylog_cb

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-openssl.c:701

目的

TLSハンドシェイク中に生成されるキーログ情報を、指定されたファイルへ追記出力するためのコールバック関数です。

シグネチャまたは形状

static void SSL_CTX_keylog_cb(const SSL *ssl, const char *line)

入力と出力

  • **入力**:
  • ssl: 関連付けられた SSL オブジェクト。
  • line: OpenSSLから渡されるNUL終端されたキーログ文字列。
  • **出力**: なし（戻り値は void）。副作用としてファイルシステムへの書き込みを行います。

前提条件と不変条件

  • SSL_get_app_data(ssl) を通じて、有効な PGconn 構造体が取得できる必要があります。
  • conn->sslkeylogfile に書き込み先のファイルパスが設定されている必要があります。

アルゴリズムまたは振る舞い

1. SSL_get_app_data を呼び出し、ssl オブジェクトから PGconn を取得します。conn が NULL の場合は即座に終了します。 2. conn->sslkeylogfile で指定されたパスを O_WRONLY | O_APPEND | O_CREAT モードで開き、ファイル記述子を取得します。 3. ファイルオープンに失敗した場合は、標準エラー出力に警告メッセージを表示して終了します。 4. line 文字列をファイルに書き込みます。書き込み成功後、改行コード \n を追記します。 5. 書き込み中にエラーが発生した場合は、標準エラー出力に警告を表示します。 6. 最後にファイル記述子を close します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure-openssl.c 内の SSL_CTX_set_keylog_callback に引数として渡されています。
  • **呼び出し先**: OpenSSL API (SSL_get_app_data) および標準ライブラリ関数 (open, write, strlen, close, fprintf) を使用しています。

関連データ構造

  • SSL (OpenSSL構造体)
  • PGconn (libpqの接続状態を保持する構造体)

失敗モードと境界条件

  • conn が NULL の場合、何もせず正常終了します。
  • ファイルのオープン失敗（権限不足やパス不正など）時は、標準エラー出力に警告を出力し、キーログの記録を断念します。
  • 書き込み失敗時も同様に警告を出力し、処理を終了します。

保守担当者向けメモ

  • 本関数は SSL_CTX_set_keylog_callback を介してOpenSSLに登録されるコールバックです。
  • ファイル操作は毎回 open/close を行うため、頻繁なキーログ出力が発生する環境ではI/O負荷に留意してください。
