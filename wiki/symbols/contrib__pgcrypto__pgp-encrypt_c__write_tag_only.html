write_tag_only

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-encrypt.c:67

目的

OpenPGP パケットのタグバイトを生成し、指定された出力フィルタへ書き込む。

シグネチャまたは形状

static int write_tag_only(PushFilter *dst, int tag);

入力と出力

  • **入力**:
  • dst: 書き込み先の PushFilter オブジェクト。
  • tag: 書き込む PGP パケットタグ（整数値）。
  • **出力**:
  • int: pushf_write の戻り値（成功時は 0、失敗時はエラーコード）。

前提条件と不変条件

  • tag は 0x3F 以下の値であること（0xC0 | tag のビット演算により、パケットタグの形式が決定されるため）。
  • dst は有効な PushFilter インスタンスであること。

アルゴリズムまたは振る舞い

1. 0xC0（新形式パケットタグのビットマスク）と tag 引数の論理和をとり、1バイトのヘッダを構築する。 2. 構築した1バイトのヘッダを pushf_write を介して dst フィルタへ書き出す。 3. 書き込み操作の結果をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-encrypt.c 内の各箇所（リテラルデータパケット、圧縮データパケット、その他パケットの書き込み処理）。
  • **呼び出し先**:
  • pushf_write（mbuf.h で定義されるフィルタ書き込み関数）。

関連データ構造

  • PushFilter: 暗号化パイプラインを構成するフィルタ構造体。

失敗モードと境界条件

  • pushf_write が失敗した場合、そのエラーコードがそのまま返される。
  • tag が大きすぎる場合、意図しないビット構成のヘッダが生成される可能性がある。

保守担当者向けメモ

  • 本関数は RFC 4880 で定義される「新形式パケット（New Packet Format）」のタグバイト生成に特化している。
  • 呼び出し元で PGP_PKT_LITERAL_DATA や PGP_PKT_COMPRESSED_DATA などの定数がタグとして渡されている点に注意すること。
