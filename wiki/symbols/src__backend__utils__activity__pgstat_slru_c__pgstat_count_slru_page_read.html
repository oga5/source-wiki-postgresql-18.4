pgstat_count_slru_page_read

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_slru.c:76

目的

指定されたSLRU（Simple Least Recently Used）キャッシュにおけるページ読み込み回数の統計カウンタをインクリメントします。

シグネチャまたは形状

void pgstat_count_slru_page_read(int slru_idx)

入力と出力

  • **入力**: slru_idx (int) - 統計対象となるSLRUのインデックス。
  • **出力**: なし。副作用として、ローカルメモリ上の統計カウンタが更新されます。

前提条件と不変条件

  • slru_idx は有効なSLRUインデックスである必要があります。
  • 本関数は pending_SLRUStats（各バックエンドのローカル統計領域）を直接操作するため、呼び出し元はスレッドセーフな環境下で実行される必要があります。

アルゴリズムまたは振る舞い

1. get_slru_entry(slru_idx) を呼び出し、指定されたインデックスに対応する統計エントリを取得します。 2. 取得したエントリの blocks_read メンバを 1 加算します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/slru.c:584
  • **呼び出し先**: get_slru_entry（インライン化されている可能性あり）

関連データ構造

  • pending_SLRUStats: 各バックエンドが保持するローカルな統計カウンタ配列。
  • PgStatShared_SLRU: 共有メモリ上の統計データ構造（本関数が更新するローカル統計の同期先）。

失敗モードと境界条件

  • 無効な slru_idx が渡された場合、get_slru_entry の実装に依存してセグメンテーションフォールトやメモリ破壊が発生する可能性があります。

保守担当者向けメモ

  • 本関数は統計収集のホットパスに含まれるため、オーバーヘッドを最小限に抑える設計となっています。
  • 共有メモリへの反映は、本関数が直接行うのではなく、後続の pgstat_slru_flush_cb によって行われます。
