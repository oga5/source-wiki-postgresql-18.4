minmax_get_strategy_procinfo

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_minmax.c:260

目的

BRINインデックスの最小値・最大値演算において、特定の属性とサブタイプに対応する戦略関数（比較演算子）の FmgrInfo を取得・キャッシュします。

シグネチャまたは形状

FmgrInfo *minmax_get_strategy_procinfo(BrinDesc *bdesc, uint16 attno, Oid subtype, uint16 strategynum)

入力と出力

  • **入力**:
  • bdesc: インデックスの記述子。
  • attno: 属性番号。
  • subtype: 比較対象のデータ型（Oid）。
  • strategynum: 実行する戦略番号（1〜BTMaxStrategyNumber）。
  • **出力**:
  • 指定された戦略に対応する FmgrInfo 構造体へのポインタ。

前提条件と不変条件

  • strategynum は 1 から BTMaxStrategyNumber の範囲内であること。
  • bdesc 内の oi_opaque は MinmaxOpaque 型としてキャスト可能であること。
  • 指定された戦略に対応する演算子がシステムカタログ（pg_amop）に存在すること。

アルゴリズムまたは振る舞い

1. bdesc から MinmaxOpaque を取得します。 2. subtype がキャッシュされているものと異なる場合、既存のキャッシュをすべて無効化（fn_oid を InvalidOid に設定）し、新しい subtype を記録します。 3. 指定された strategynum のキャッシュが空（InvalidOid）であれば、SearchSysCache4 を使用して pg_amop から演算子を検索します。 4. 演算子が見つからない場合はエラーを発生させます。 5. 検索結果から得られた演算子IDを fmgr_info_cxt を用いて bdesc->bd_context にキャッシュし、そのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin_minmax.c 内の各関数（brin_minmax_add_value, brin_minmax_consistent 等）から頻繁に呼び出されます。
  • **呼び出し先**: SearchSysCache4, SysCacheGetAttrNotNull, ReleaseSysCache, fmgr_info_cxt, get_opcode 等のユーティリティ関数。

関連データ構造

  • MinmaxOpaque: 演算クラスのキャッシュを保持する構造体。
  • BrinDesc: インデックスのメタデータ構造体。
  • FmgrInfo: 関数呼び出し情報を保持する構造体。

失敗モードと境界条件

  • 指定された strategynum や subtype に対応する演算子が pg_amop に存在しない場合、elog(ERROR) により処理が中断されます。
  • subtype が変更されるたびにキャッシュの再構築が発生します。

保守担当者向けメモ

  • inclusion_get_strategy_procinfo と設計が共通化されています。修正時は両方のファイルを確認してください。
  • キャッシュの無効化ロジックは subtype の変更に依存しているため、複数のサブタイプを頻繁に切り替える環境では注意が必要です。
