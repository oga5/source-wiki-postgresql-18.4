GISTIntArrayBigOptions

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int.h:84

目的

intarray モジュールにおける GiST インデックスのオプション設定を保持するための構造体です。特に、インデックスのシグネチャ長を動的に構成するために使用されます。

シグネチャまたは形状

typedef struct
{
	int32		vl_len_;		/* varlena header */
	int			siglen;			/* signature length in bytes */
} GISTIntArrayBigOptions;

入力と出力

  • **入力**: reloptions を介して設定される siglen 値。
  • **出力**: インデックス作成時や検索時に参照されるシグネチャ長（バイト単位）。

前提条件と不変条件

  • vl_len_ は可変長データ（varlena）のヘッダーであり、直接操作してはなりません。
  • siglen はインデックスのビットベクトル長を決定する重要なパラメータです。

アルゴリズムまたは振る舞い

本構造体は PostgreSQL の reloptions 機構と連携し、インデックスのメタデータとしてメモリ上に配置されます。_intbig_gist.c において、init_local_reloptions を通じてメモリ確保が行われ、インデックスの構成パラメータとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_intbig_gist.c: init_local_reloptions による初期化および offsetof を用いたメンバ参照。
  • **呼び出し先**:
  • contrib/intarray/_int.h: PG_GET_OPCLASS_OPTIONS() マクロを通じて、インデックス操作時に siglen が取得されます。

関連データ構造

  • GISTIntArrayOptions: 通常の GiST インデックス用オプション構造体。
  • GISTTYPE: GiST インデックスのキー構造体。

失敗モードと境界条件

  • siglen の値が不正な場合、インデックスのシグネチャ生成や照合ロジックにおいて予期せぬ動作を引き起こす可能性があります。
  • メモリ確保時に sizeof(GISTIntArrayBigOptions) を正しく指定しない場合、バッファオーバーランや不正なメモリ参照が発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を含む構造体と同様に、メモリレイアウトに依存するため、メンバの追加や変更を行う際は reloptions の初期化ロジック（_intbig_gist.c）との整合性に注意してください。
