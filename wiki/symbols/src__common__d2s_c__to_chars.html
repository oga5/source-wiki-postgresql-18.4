to_chars

定義

  • 種別: function
  • 定義箇所: src/common/d2s.c:786

目的

floating_decimal_64 構造体で表現された数値を、最短の 10 進数文字列としてバッファへ書き出す。

シグネチャまたは形状

static inline int to_chars(floating_decimal_64 v, const bool sign, char *const result)

入力と出力

  • **入力**:
  • v: 変換対象の数値（仮数と指数を含む）。
  • sign: 負数かどうかを示すフラグ。
  • result: 文字列を書き込むためのバッファ。
  • **出力**:
  • 戻り値: 書き込まれた文字数（int）。

前提条件と不変条件

  • result バッファは、変換結果を格納するために十分なサイズを確保していること。
  • v.exponent および v.mantissa は、Ryu アルゴリズムによって計算された有効な値であること。

アルゴリズムまたは振る舞い

1. **初期化**: 符号が負であれば result の先頭に '-' を書き込む。 2. **フォーマット判定**: 指数 exp が -4 以上 15 未満の場合、固定小数点表記として to_chars_df を呼び出し終了する。 3. **末尾ゼロの除去**: v.exponent が 0 の場合、div10 を用いて仮数の末尾にある不要なゼロを除去し、表示長を調整する。 4. **文字列化**: DIGIT_TABLE を参照し、数値を高速に文字列へ変換する。

  • 64 ビットの仮数が大きい場合は 8 桁ずつ分割して処理する。
  • 32 ビット範囲内では 4 桁、2 桁、1 桁の順で最適化された変換を行う。

    • 5. **小数点と指数**: 必要に応じて小数点 . を挿入し、科学的表記（e 表記）を付与する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/d2s.c および src/common/f2s.c 内の変換関数から呼び出される。
  • **呼び出し先**: to_chars_df, div10, div1e8。

関連データ構造

  • floating_decimal_64: 変換対象の数値を保持する構造体。
  • DIGIT_TABLE: 2 桁の数値を文字列に変換するためのルックアップテーブル。

失敗モードと境界条件

  • exp が閾値外の場合、科学的表記が強制される。
  • olength が 1 より大きい場合にのみ小数点が付与される。
  • 128 ビット演算が利用できない環境では、RYU_ONLY_64_BIT_OPS マクロによりパフォーマンスに影響が出る可能性がある。

保守担当者向けメモ

  • to_chars_df との整合性および DOUBLE_SHORTEST_DECIMAL_LEN の値は、固定小数点表記の閾値に依存しているため、変更時は注意が必要。
  • memcpy を使用した高速化が行われているため、バッファオーバーランが発生しないよう、書き込み位置の計算には細心の注意を払うこと。
