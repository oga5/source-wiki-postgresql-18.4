ua_page_items

定義

  • 種別: struct
  • 定義箇所: contrib/pageinspect/btreefuncs.c:91

目的

B-treeインデックスのページ内タプルを走査・出力する際のコンテキスト情報を保持するための構造体です。

シグネチャまたは形状

typedef struct ua_page_items
{
	Page		page;
	OffsetNumber offset;
	bool		leafpage;
	bool		rightmost;
	TupleDesc	tupd;
} ua_page_items;

入力と出力

本構造体は関数間で状態を共有するために使用されます。

  • 入力: palloc によりメモリ確保され、各フィールドが初期化されます。
  • 出力: bt_page_print_tuples 等の関数へ引数として渡され、タプル抽出処理の制御に使用されます。

前提条件と不変条件

  • page は有効なPostgreSQLのページポインタである必要があります。
  • tupd は対象となるインデックスのタプル記述子（TupleDesc）を指している必要があります。

アルゴリズムまたは振る舞い

SRF（Set Returning Function）の実装において、現在の走査位置（offset）やページの種類（leafpage）、ページ内の位置（rightmost）を追跡するために使用されます。これにより、ページ内の各アイテムを順次処理する際のステートフルな管理が可能となります。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pageinspect/btreefuncs.c 内のタプル抽出関数（bt_page_items_internal 等）にて、palloc を介してインスタンス化されます。
  • 呼び出し先: bt_page_print_tuples 関数が本構造体を引数として受け取り、タプルの解析と出力を行います。

関連データ構造

  • Page: ページデータへのポインタ。
  • OffsetNumber: ページ内のアイテムオフセット。
  • TupleDesc: インデックスの行構造を定義する記述子。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • 不正なポインタ: page や tupd が初期化されていない状態で bt_page_print_tuples に渡された場合、セグメンテーション違反や予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は bt_page_items 関連の関数群で共有されています。構造体のフィールドを変更する場合は、これらを参照するすべての関数（bt_page_print_tuples 等）の整合性を確認してください。
