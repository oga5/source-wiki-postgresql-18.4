bbsink_throttle_manifest_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_throttle.c:120

目的

バックアップマニフェストの内容を転送する際に、設定された転送レート制限（スロットリング）を適用し、後続のシンクへデータを転送する。

シグネチャまたは形状

static void bbsink_throttle_manifest_contents(bbsink *sink, size_t len)

入力と出力

  • sink: スロットリング機能を持つ bbsink 構造体へのポインタ。
  • len: 転送対象となるデータのバイト数。
  • 戻り値: なし。

前提条件と不変条件

  • sink は bbsink_throttle 型としてキャスト可能であること。
  • 本関数は bbsink インターフェースの一部として呼び出されること。

アルゴリズムまたは振る舞い

1. throttle() 関数を呼び出し、現在の累積転送量に基づいた待機時間（必要に応じて WaitLatch を使用）を計算・実行する。 2. bbsink_forward_manifest_contents() を呼び出し、データをパイプライン内の次のシンクへ透過的に転送する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink インターフェースのメソッドテーブル（.manifest_contents）を通じて呼び出される。
  • 呼び出し先:
  • throttle(): 転送レート調整用。
  • bbsink_forward_manifest_contents(): 次のシンクへのデータ転送用。

関連データ構造

  • bbsink: バックアップ処理のパイプラインを構成する基本構造体。
  • bbsink_throttle: bbsink を継承し、スロットリング状態を保持する構造体。

失敗モードと境界条件

  • 転送レートが制限値を超えた場合、throttle() 内で WaitLatch による待機が発生する。
  • 待機中、バックアッププロセスが停止要求（シグナル）を受け取った場合は CHECK_FOR_INTERRUPTS() により適切に割り込み処理が行われる。

保守担当者向けメモ

  • 本関数は「デコレータ」パターンで実装されており、マニフェスト転送処理の前後でレート制御を挟み込む役割を持つ。
  • 整数演算によるレート計算を行っているため、端数は内部的に次回の呼び出しへ持ち越される。
