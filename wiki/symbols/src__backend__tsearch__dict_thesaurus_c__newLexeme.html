newLexeme

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_thesaurus.c:71

目的

シソーラス辞書構造体（DictThesaurus）に新しい語彙（TheLexeme）を追加し、置換ルール情報を初期化します。

シグネチャまたは形状

static void newLexeme(DictThesaurus *d, char *b, char *e, uint32 idsubst, uint16 posinsubst)

入力と出力

  • **入力**:
  • d: 操作対象の DictThesaurus 構造体へのポインタ。
  • b, e: 追加する語彙文字列の開始位置と終了位置（ポインタ）。
  • idsubst: 置換先の識別子。
  • posinsubst: 置換先フレーズ内での位置。
  • **出力**: なし（d の内部状態を更新）。

前提条件と不変条件

  • d->wrds は palloc または repalloc によって管理される動的配列である必要があります。
  • b から e までの範囲は有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. **領域確保**: d->nwrds が d->ntwrds に達した場合、d->wrds 配列のサイズを倍増させます（初期値は16）。 2. **語彙追加**: d->wrds の末尾に新しい TheLexeme エントリを割り当て、d->nwrds をインクリメントします。 3. **文字列コピー**: b から e までの文字列を palloc した領域にコピーし、ヌル終端を付与します。 4. **メタデータ設定**: LexemeInfo 構造体を割り当て、idsubst および posinsubst を格納します。nextentry は NULL で初期化されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/dict_thesaurus.c:225
  • src/backend/tsearch/dict_thesaurus.c:230
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理)
  • repalloc (PostgreSQL メモリ管理)
  • memcpy (標準ライブラリ)

関連データ構造

  • DictThesaurus: 辞書全体を管理する構造体。
  • TheLexeme: 個別の語彙とそれに紐づく情報を保持する構造体。
  • LexemeInfo: 置換ルールに関連する詳細情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc / repalloc がエラーを発生させ、処理が中断されます。
  • b >= e の場合: 空文字列として処理されます。

保守担当者向けメモ

  • 本関数は DictThesaurus の動的配列を直接操作するため、呼び出し元で d->nwrds と d->ntwrds の整合性が保たれていることを確認してください。
  • LexemeInfo は単一の割り当てのみを行っており、リスト構造の拡張は考慮されていません。
