DictSubState

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_public.h:150

目的

シソーラスのような複雑な辞書処理において、辞書関数呼び出し間での状態を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	bool		isend;			/* in: テキストの終端に達したことを示す */
	bool		getnext;		/* out: 辞書が次の語彙を要求しているか */
	void	   *private_state;	/* 呼び出し間で保持する内部状態 */
} DictSubState;

入力と出力

  • **入力**: isend（呼び出し元がテキストの終端を通知するために設定）。
  • **出力**: getnext（辞書が処理を継続するために次の語彙を要求しているかを示すフラグ）。

前提条件と不変条件

  • private_state は、getnext が true となる呼び出し間で状態を維持するために使用されます。

アルゴリズムまたは振る舞い

辞書処理（lexize 等）において、一度の呼び出しで完結しない複雑な変換処理を行う際、この構造体を通じて呼び出し元と辞書間で状態の受け渡しを行います。辞書側は getnext を設定することで、呼び出し元に対して次の入力を要求できます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/tsearch/dict.c
  • src/backend/tsearch/dict_thesaurus.c
  • src/backend/tsearch/ts_parse.c
  • **呼び出し先**:
  • 辞書の実装（dict_thesaurus.c 等）がこの構造体を介して状態を管理します。

関連データ構造

  • TSLexeme: 辞書による単語分割結果を保持する構造体。

失敗モードと境界条件

  • private_state が適切に初期化されていない場合、または呼び出し間で正しく管理されていない場合、辞書処理の整合性が失われる可能性があります。

保守担当者向けメモ

  • 本構造体は、主に PostgreSQL 内部の全文検索モジュール間での状態共有を目的としています。
  • private_state のメモリ管理は、各辞書の実装に依存するため、メモリリークが発生しないよう注意してください。
