parse_sync_method

定義

  • 種別: function
  • 定義箇所: src/fe_utils/option_utils.c:89

目的

コマンドライン引数として与えられた文字列を、PostgreSQLのデータディレクトリ同期メソッドを表す列挙型 DataDirSyncMethod に変換し、検証を行う。

シグネチャまたは形状

bool parse_sync_method(const char *optarg, DataDirSyncMethod *sync_method)

入力と出力

  • **入力**:
  • optarg: 解析対象の文字列（例: "fsync", "syncfs"）。
  • sync_method: 変換結果を格納するための DataDirSyncMethod 型へのポインタ。
  • **出力**:
  • 成功時は true を返し、sync_method に値を設定する。
  • 失敗時は false を返し、エラーメッセージをログ出力する。

前提条件と不変条件

  • optarg は有効な文字列ポインタであること。
  • sync_method は書き込み可能なメモリ領域を指していること。
  • syncfs メソッドのサポートは、コンパイル時のマクロ HAVE_SYNCFS に依存する。

アルゴリズムまたは振る舞い

1. optarg が "fsync" と一致する場合、DATA_DIR_SYNC_METHOD_FSYNC を設定する。 2. optarg が "syncfs" と一致する場合:

  • HAVE_SYNCFS が定義されていれば DATA_DIR_SYNC_METHOD_SYNCFS を設定する。
  • 定義されていない場合、エラーをログ出力し false を返す。

    • 3. いずれにも一致しない場合、"unrecognized sync method" というエラーをログ出力し false を返す。 4. 成功時は true を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • initdb (src/bin/initdb/initdb.c)
  • pg_basebackup (src/bin/pg_basebackup/pg_basebackup.c)
  • pg_checksums (src/bin/pg_checksums/pg_checksums.c)
  • pg_combinebackup (src/bin/pg_combinebackup/pg_combinebackup.c)
  • pg_dump (src/bin/pg_dump/pg_dump.c)
  • pg_rewind (src/bin/pg_rewind/pg_rewind.c)
  • pg_upgrade (src/bin/pg_upgrade/option.c)
  • **呼び出し先**:
  • pg_log_error (エラー報告用)

関連データ構造

  • DataDirSyncMethod: 同期メソッドを定義する列挙型。

失敗モードと境界条件

  • 未知の文字列が渡された場合、pg_log_error を経由してエラーを報告し false を返す。
  • syncfs がサポートされていない環境で "syncfs" が指定された場合、エラーを報告し false を返す。

保守担当者向けメモ

  • 新しい同期メソッドを追加する場合は、本関数の条件分岐と DataDirSyncMethod 列挙型の定義を更新する必要がある。
  • プラットフォーム依存のメソッドを追加する際は、適切なコンパイル時マクロでガードすること。
