jsonb_9_4_check_applicable

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/version.c:20

目的

PostgreSQL 9.4ベータ版の期間中に発生したJSONBデータ型の内部ストレージ形式の変更を検知し、アップグレード対象のクラスターがその影響を受けるかどうかを判定します。

シグネチャまたは形状

bool jsonb_9_4_check_applicable(ClusterInfo *cluster)

入力と出力

  • **入力**: ClusterInfo *cluster （アップグレード対象のクラスター情報構造体へのポインタ）
  • **出力**: bool 型。条件に合致する場合は true、それ以外は false を返します。

前提条件と不変条件

  • cluster 引数は有効な ClusterInfo 構造体を指している必要があります。
  • cluster->major_version および cluster->controldata.cat_ver が正しく初期化されていることを前提としています。

アルゴリズムまたは振る舞い

1. cluster->major_version が 9.4（GET_MAJOR_VERSION で 904 と評価）であるかを確認します。 2. cluster->controldata.cat_ver が JSONB_FORMAT_CHANGE_CAT_VER 定数より小さい値であるかを確認します。 3. 上記の両条件を満たす場合、JSONB形式の変更が必要であると判断し true を返します。それ以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c:266 にて、.version_hook として参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。マクロ GET_MAJOR_VERSION を使用しています。

関連データ構造

  • ClusterInfo: pg_upgrade.h で定義される、クラスターのバージョン情報や制御データ（controldata）を保持する構造体。

失敗モードと境界条件

  • cluster ポインタが NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • cat_ver が未定義または不正な値である場合、誤った判定を下す可能性があります。

保守担当者向けメモ

  • 本関数は特定のバージョン（9.4ベータ）に依存したロジックです。将来的にサポート対象バージョンから9.4が除外された場合、本関数の削除を検討してください。
