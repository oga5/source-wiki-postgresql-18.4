ECPGnoticeReceiver

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/connect.c:207

目的

データベースサーバーから送信された通知（NOTICE）を捕捉し、ECPGの sqlca（SQL Communication Area）構造体にマッピングしてアプリケーションへ伝播させる。

シグネチャまたは形状

static void ECPGnoticeReceiver(void *arg, const PGresult *result)

入力と出力

  • **入力**:
  • arg: 使用されない引数。
  • result: 通知内容を含む PGresult オブジェクト。
  • **出力**: なし（戻り値は void）。副作用として sqlca 構造体の更新およびログ出力を行う。

前提条件と不変条件

  • ECPGget_sqlca() が有効な sqlca_t 構造体を返すこと。
  • result から取得した SQLSTATE が NULL の場合、内部エラーとして扱う。
  • SQLSTATE が "00" で始まる（成功を示す）通知は無視される。

アルゴリズムまたは振る舞い

1. PQresultErrorField を使用して SQLSTATE とメッセージを取得する。 2. ECPGget_sqlca() で現在の sqlca を取得し、失敗時はログを出力して終了する。 3. SQLSTATE が "00" で始まる場合は処理を中断する。 4. メッセージをログに出力する。 5. SQLSTATE に応じて sqlcode をマッピングする（例: INVALID_CURSOR_NAME は ECPG_WARNING_UNKNOWN_PORTAL に変換）。 6. sqlca の sqlstate、sqlcode、sqlwarn、および sqlerrm を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: libpq の PQsetNoticeReceiver を介して登録され、サーバーからの通知時に呼び出される（src/interfaces/ecpg/ecpglib/connect.c:675 で登録）。
  • **呼び出し先**:
  • PQresultErrorField (libpq)
  • ECPGget_sqlca
  • ecpg_log
  • ecpg_gettext
  • strncpy, strcmp, strncmp, strlen (標準ライブラリ)

関連データ構造

  • sqlca_t: 通知情報が書き込まれる通信領域。
  • PGresult: 通知情報を含む libpq の構造体。

失敗モードと境界条件

  • ECPGget_sqlca() が NULL を返した場合、メモリ不足としてログを出力し処理を中断する。
  • SQLSTATE やメッセージが取得できない場合、デフォルト値（ECPG_SQLSTATE_ECPG_INTERNAL_ERROR や "empty message text"）で補完する。
  • sqlerrmc へのコピー時にバッファオーバーフローを防ぐため、末尾のヌル終端を保証する。

保守担当者向けメモ

  • 本関数は libpq の通知受信コールバックとして動作するため、スレッドセーフな実装が求められる。
  • sqlca への書き込みは、後続の SQL 操作で上書きされる可能性がある点に注意すること。
