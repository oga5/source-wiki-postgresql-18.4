PermutationStepBlockerType

定義

  • 種別: enum
  • 定義箇所: src/test/isolation/isolationtester.h:42

目的

分離レベルテストにおいて、順列（Permutation）内の各ステップが実行される際に適用される「ブロック条件」の種類を定義します。

シグネチャまたは形状

typedef enum
{
	PSB_ONCE,
	PSB_OTHER_STEP,
	PSB_NUM_NOTICES,
} PermutationStepBlockerType;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力は持ちません。
  • PermutationStepBlocker 構造体の blocktype メンバとして使用されます。

前提条件と不変条件

  • テスト仕様の解析後、実行エンジンが check_testspec() を実行するまで、この型を用いた条件設定は未解決の状態である可能性があります。

アルゴリズムまたは振る舞い

各列挙値は以下のブロック振る舞いを表します：

  • PSB_ONCE: ステップの実行を一度だけ待機させます。
  • PSB_OTHER_STEP: 指定された別のステップが完了するまで待機します。
  • PSB_NUM_NOTICES: 指定された別のセッションから特定の数の通知（Notice）が届くまで待機します。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/isolation/isolationtester.h 内の PermutationStepBlocker 構造体定義で参照されています。
  • 呼び出し先: 情報なし（本シンボルは型定義であり、関数呼び出しは行いません）。

関連データ構造

  • PermutationStepBlocker: この列挙型を保持する構造体。
  • PermutationStep: PermutationStepBlocker を保持する構造体。

失敗モードと境界条件

  • 新しいブロック条件を追加する場合、本列挙型を拡張する必要があります。

保守担当者向けメモ

  • テスト仕様の拡張に伴い、新しい待機条件が必要になった場合は本列挙型に値を追加してください。
