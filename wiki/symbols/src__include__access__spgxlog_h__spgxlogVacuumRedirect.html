spgxlogVacuumRedirect

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgxlog.h:238

目的

SP-GiST インデックスのバキューム処理において、リダイレクトタプルをプレースホルダに変換するための WAL レコード構造を定義します。

シグネチャまたは形状

typedef struct spgxlogVacuumRedirect
{
    uint16      nToPlaceholder;
    OffsetNumber firstPlaceholder;
    TransactionId snapshotConflictHorizon;
    bool        isCatalogRel;
    OffsetNumber offsets[FLEXIBLE_ARRAY_MEMBER];
} spgxlogVacuumRedirect;

入力と出力

  • **入力**: プレースホルダへ変換対象となるリダイレクトタプルの数、開始オフセット、スナップショット競合解消のためのトランザクションID、およびカタログリレーションかどうかのフラグ。
  • **出力**: offsets 配列に格納された、変換対象となるリダイレクトタプルのオフセット番号。

前提条件と不変条件

  • offsets は FLEXIBLE_ARRAY_MEMBER として定義されており、構造体の末尾に可変長で配置されます。
  • 構造体の固定部分のサイズは SizeOfSpgxlogVacuumRedirect マクロで計算可能です。

アルゴリズムまたは振る舞い

1. バキューム処理中にリダイレクトタプルをプレースホルダへ変換する際、この構造体に情報を詰め込み WAL に書き出します。 2. リカバリ（Redo）時には、この構造体を参照して対象ページ上の指定されたオフセットにあるタプルをプレースホルダへ更新します。 3. isCatalogRel フラグは、スタンバイ側での論理デコーディング中にリカバリ競合を適切に処理するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgvacuum.c にて、バキューム処理中にこの構造体がインスタンス化されます。
  • **呼び出し先**: src/backend/access/spgist/spgxlog.c の Redo 処理にて、WAL レコードからキャストされて読み取られます。また、src/backend/access/rmgrdesc/spgdesc.c にてログ出力（記述）のために参照されます。

関連データ構造

  • OffsetNumber: ページ内のタプル位置を特定するために使用されます。
  • TransactionId: スナップショットの整合性確認に使用されます。

失敗モードと境界条件

  • 可変長配列 offsets を扱う際は、アライメントされていないデータとして扱う必要があるため、直接的な構造体アクセスではなく memcpy 等を用いた安全なコピーが推奨されます。

保守担当者向けメモ

  • SizeOfSpgxlogVacuumRedirect マクロ（src/include/access/spgxlog.h:250）は、可変長部分を除いた固定サイズの計算に使用されます。ポインタ演算を行う際は必ずこのマクロを基準としてください。
