macaddr8_ge

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac8.c:364

目的

2つの macaddr8 型の値が「以上（greater than or equal to）」の関係にあるかを判定する。

シグネチャまたは形状

Datum macaddr8_ge(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの macaddr8 型ポインタを受け取る。
  • **出力**: Datum 型として、第1引数が第2引数以上であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力される macaddr8 ポインタは有効であること。
  • 比較ロジックは macaddr8_cmp_internal の結果に依存する。

アルゴリズムまたは振る舞い

1. PG_GETARG_MACADDR8_P マクロを使用して、引数から2つの macaddr8 構造体へのポインタを取得する。 2. macaddr8_cmp_internal を呼び出し、2つの値を比較する。 3. 比較結果が 0 以上（>= 0）である場合に真を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_macaddr8.c:37 において、B-tree GiST インデックスの比較処理から呼び出される。
  • **呼び出し先**: macaddr8_cmp_internal を呼び出す。

関連データ構造

  • macaddr8: 8バイト（EUI-64）MACアドレスを保持するデータ型。

失敗モードと境界条件

  • 引数が NULL の場合、PG_GETARG_MACADDR8_P の仕様に従いエラーとなる可能性がある。
  • 比較は macaddr8_cmp_internal の実装（hibits および lobits マクロによるビット比較）に依存するため、構造体のメモリレイアウトが正しく維持されている必要がある。

保守担当者向けメモ

  • 本関数は SQL 演算子 >= の実装として機能する。
  • 比較ロジックの変更が必要な場合は、macaddr8_cmp_internal を修正すること。
