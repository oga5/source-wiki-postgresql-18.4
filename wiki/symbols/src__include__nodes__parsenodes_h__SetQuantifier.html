SetQuantifier

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/parsenodes.h:60

目的

SQLクエリにおける集合演算の重複排除ルール（ALL または DISTINCT）を表現するために使用されます。

シグネチャまたは形状

typedef enum SetQuantifier
{
	SET_QUANTIFIER_DEFAULT,
	SET_QUANTIFIER_ALL,
	SET_QUANTIFIER_DISTINCT,
} SetQuantifier;

入力と出力

  • 入力: SQLの構文解析時に、SELECT 句や集合演算子に関連する構文要素から設定されます。
  • 出力: 解析器（analyze.c）が生成するクエリツリー内のノードにおいて、重複排除の挙動を決定するフラグとして利用されます。

前提条件と不変条件

  • SET_QUANTIFIER_DEFAULT は、明示的な指定がない場合の初期状態として使用されます。
  • SET_QUANTIFIER_ALL および SET_QUANTIFIER_DISTINCT は、SQL標準の ALL および DISTINCT キーワードに対応します。

アルゴリズムまたは振る舞い

パーサーがSQL文を読み込む際、重複排除の指定に応じて本列挙型のいずれかの値をノードに割り当てます。この値は、後のクエリ最適化や実行計画生成の過程で、重複行を排除するかどうかの判断基準となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/gram.y（パーサー）がこの列挙型を使用してパースツリーを構築します。
  • 呼び出し先: 提供されたソース情報には、本シンボルを直接参照または呼び出している具体的な関数名は含まれていません。

関連データ構造

  • Query: 本列挙型が保持されるクエリツリーのルートノード構造体。
  • parsenodes.h 内の各種ノード構造体（SELECT 句や集合演算を伴うノード）。

失敗モードと境界条件

  • 該当なし。本シンボルは列挙型であり、値の範囲は定義された3つの定数に限定されます。

保守担当者向けメモ

  • 本シンボルは src/include/nodes/parsenodes.h で定義されており、SQLの構文解析結果を保持する重要な役割を担います。
  • 新たな集合演算の挙動を追加する場合、この列挙型を拡張する必要があります。
