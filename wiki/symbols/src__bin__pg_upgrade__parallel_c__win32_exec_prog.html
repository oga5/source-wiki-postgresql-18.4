win32_exec_prog

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/parallel.c:152

目的

Windows環境において、_beginthreadex を介して実行されるスレッド内で外部コマンドを実行し、その終了ステータスをスレッドの戻り値として提供する。

シグネチャまたは形状

DWORD win32_exec_prog(exec_thread_arg *args)

入力と出力

  • **入力**: exec_thread_arg *args（実行するコマンド文字列 cmd、ログファイルパス log_file、オプションログファイルパス opt_log_file を含む構造体へのポインタ）。
  • **出力**: DWORD 型。exec_prog が成功した場合は 0、失敗した場合は 1 を返す。

前提条件と不変条件

  • 引数 args は有効なメモリ領域を指している必要がある。
  • exec_prog が呼び出し可能な状態であること。

アルゴリズムまたは振る舞い

1. 引数として渡された args 構造体からログファイルパスおよびコマンド文字列を取得する。 2. exec_prog を呼び出し、外部コマンドを実行する。この際、ログ出力とコマンド実行の同期フラグを有効にする。 3. exec_prog の戻り値（成功時 true）を論理反転させ、成功時に 0、失敗時に 1 となるように変換する。 4. 変換した値を戻り値としてスレッドを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/parallel.c 内の _beginthreadex 呼び出し箇所（140行目付近）。
  • **呼び出し先**: exec_prog（src/bin/pg_upgrade/exec.c で定義される外部コマンド実行関数）。

関連データ構造

  • exec_thread_arg: スレッド間でコマンド実行に必要な引数を保持する構造体。

失敗モードと境界条件

  • exec_prog が失敗した場合、本関数は 1 を返す。
  • スレッド実行中のエラーハンドリングは exec_prog の実装に依存する。

保守担当者向けメモ

  • 本関数は Windows 専用の実装であるため、#ifdef WIN32 ブロック内で使用されることを前提としている。
  • スレッドの戻り値は _beginthreadex の仕様に従い DWORD 型である必要がある。
