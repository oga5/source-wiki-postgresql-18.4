PLy_output_setup_tuple

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_typeio.c:214

目的

タプル（複合型）をPythonオブジェクトからPostgreSQLの Datum へ変換するための変換構造体（PLyObToDatum）を初期化または更新します。

シグネチャまたは形状

void PLy_output_setup_tuple(PLyObToDatum *arg, TupleDesc desc, PLyProcedure *proc)

入力と出力

  • **入力**:
  • arg: 変換情報を保持する PLyObToDatum 構造体へのポインタ。
  • desc: 変換対象のタプル構造を記述する TupleDesc。
  • proc: 現在実行中のPL/Pythonプロシージャ情報。
  • **出力**: なし（arg が指す構造体を直接更新）。

前提条件と不変条件

  • arg->func が PLyObject_ToComposite であること（Assert で検証済み）。
  • arg が指すメモリコンテキスト（arg->mcxt）が有効であること。

アルゴリズムまたは振る舞い

1. **メタデータ保存**: 匿名レコード型（RECORDOID かつ typmod < 0）の場合、desc を arg->u.tuple.recdesc に保存します。 2. **属性配列の管理**: desc->natts と現在の arg->u.tuple.natts を比較し、不一致があれば既存の属性配列を解放し、新しいサイズで再割り当てします。 3. **再帰的セットアップ**: 各属性に対してループ処理を行い、以下の条件を満たす場合に PLy_output_setup_func を呼び出して再帰的に変換関数を構築します。

  • 属性が削除済み（attisdropped）でないこと。
  • 既存の変換設定と型情報（typoid, typmod）が一致しないこと。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_exec.c
  • src/pl/plpython/plpy_typeio.c（再帰的な呼び出しを含む）
  • **呼び出し先**:
  • PLy_output_setup_func（各属性の変換設定用）
  • pfree / MemoryContextAllocZero（メモリ管理）

関連データ構造

  • PLyObToDatum: 変換ロジックと型情報を保持する構造体。
  • TupleDesc: タプルの列定義情報。
  • PLyProcedure: プロシージャの実行状態を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: MemoryContextAllocZero が失敗した場合、PostgreSQLのメモリ管理機構によりエラーが発生します。
  • **型不一致**: 属性の型が変更された場合、既存の変換設定は破棄され、再構築されます。
  • **削除済み列**: attisdropped が真の列はスキップされ、変換対象から除外されます。

保守担当者向けメモ

  • 本関数は、変換の重複作業を避けるために既存の設定を再利用する設計になっています。
  • 匿名レコード型を扱う際は recdesc の保持が必須となるため、呼び出し元で適切な TupleDesc が渡されていることを確認してください。
