updateInitAclDependencies

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_shdepend.c:511

目的

オブジェクトの初期アクセス制御リスト（Initial ACL）に関連する共有依存関係を更新します。

シグネチャまたは形状

void updateInitAclDependencies(Oid classId, Oid objectId, int32 objsubId,
                               int noldmembers, Oid *oldmembers,
                               int nnewmembers, Oid *newmembers)

入力と出力

  • **入力**:
  • classId: 対象オブジェクトのカタログOID。
  • objectId: 対象オブジェクトのOID。
  • objsubId: 列番号（列単位の依存の場合）。
  • noldmembers, oldmembers: 変更前のメンバー（ロール）の数と配列。
  • nnewmembers, newmembers: 変更後のメンバー（ロール）の数と配列。
  • **出力**: なし（副作用として pg_shdepend カタログを更新）。

前提条件と不変条件

  • SHARED_DEPENDENCY_INITACL 型の依存関係として処理されます。
  • 所有者（Owner）情報は本関数では参照されません。

アルゴリズムまたは振る舞い

内部関数 updateAclDependenciesWorker を呼び出し、指定されたオブジェクトとメンバーリストの差分に基づき、pg_shdepend 内の初期ACL依存関係レコードを挿入または削除します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c 内の各所（updateInitAclDependencies を直接呼び出し）。
  • **呼び出し先**: updateAclDependenciesWorker（同ファイル内の内部関数）。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • SHARED_DEPENDENCY_INITACL: 共有依存関係のタイプを示す列挙値。

失敗モードと境界条件

  • pg_shdepend カタログへのアクセス権限やロック競合が発生した場合、基盤となるテーブルアクセス関数がエラーをスローします。

保守担当者向けメモ

  • 本関数は pg_shdepend.c 内のラッパーであり、実処理は updateAclDependenciesWorker に委譲されています。
  • 依存関係の変更は、oldmembers と newmembers の比較によって決定されるため、呼び出し側で正確なメンバーリストを渡す必要があります。
