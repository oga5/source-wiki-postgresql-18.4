hashline_number

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/output.c:91

目的

ECPGプリプロセッサが生成するCコード内に、元のソースコードのファイル名と行番号を同期させるための #line ディレクティブ文字列を生成します。

シグネチャまたは形状

char *hashline_number(void)

入力と出力

  • **入力**: なし（グローバル変数 input_filename および base_yylineno を参照）。
  • **出力**: #line <行番号> "<ファイル名>"\n という形式の文字列（char *）。デバッグモード等の条件により出力が抑制される場合は空文字列 "" を返します。

前提条件と不変条件

  • input_filename が有効であること。
  • loc_alloc を使用してメモリを動的に確保するため、呼び出し元で適切に管理（解放）する必要があります。

アルゴリズムまたは振る舞い

1. デバッグモード（YYDEBUG かつ base_yydebug が有効）でないことを確認します。 2. 条件を満たす場合、loc_alloc を用いて必要なバッファサイズを計算し、メモリを確保します。 3. sprintf で #line ディレクティブの基本構造を書き込みます。 4. input_filename を走査し、ファイルパスに含まれる \ または " をエスケープ（\\ または \"）してバッファにコピーします。 5. 最後に閉じ引用符と改行を付加して文字列を完成させ、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/output.c 内のコード（ツールによる直接的な参照確認あり）。
  • **呼び出し先**: loc_alloc（メモリ確保）、sprintf、strlen、strcat 等の標準ライブラリ関数。

関連データ構造

  • input_filename: 現在処理中のソースファイル名（グローバル変数）。
  • base_yylineno: 現在のソースコードの行番号（グローバル変数）。

失敗モードと境界条件

  • input_filename が未設定（NULL）の場合、空文字列を返します。
  • ファイルパスに \ や " が含まれる場合、正しくエスケープ処理が行われます。
  • メモリ確保失敗時の挙動については、loc_alloc の実装に依存します。

保守担当者向けメモ

  • #line ディレクティブの生成ロジックは、生成されたCコードのデバッグにおいて重要です。
  • ファイルパス内の特殊文字エスケープは、C言語の文字列リテラルとして正しく解釈されるよう慎重に設計されています。修正時はエスケープ処理の整合性に注意してください。
