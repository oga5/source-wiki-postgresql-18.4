des_cipher

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/crypt-des.c:616

目的

DESアルゴリズムを用いて、指定された回数（count）の暗号化処理を実行します。

シグネチャまたは形状

static int des_cipher(const char *in, char *out, long salt, int count)

入力と出力

  • in: 暗号化対象の入力データ（8バイト）。
  • out: 暗号化結果を格納するバッファ（8バイト）。
  • salt: DESの初期化に使用するソルト値。
  • count: 暗号化の反復回数。
  • 戻り値: 成功時は0、失敗時はdo_desが返すエラーコード。

前提条件と不変条件

  • 入力データおよび出力データは、メモリ上のアライメントを考慮し、内部でmemcpyを用いてuint32配列へコピーされます。
  • 初回呼び出し時にdes_initialisedフラグが偽であれば、des_init()が自動的に実行されます。

アルゴリズムまたは振る舞い

1. des_init()を呼び出し、必要に応じてマスクテーブルを初期化します。 2. setup_salt(salt)により、指定されたソルトでDESの内部状態を構成します。 3. 入力データをuint32配列にコピーし、pg_ntoh32を用いてネットワークバイトオーダーからホストバイトオーダーへ変換します。 4. do_desを呼び出し、Feistelネットワークによる暗号化ラウンドを実行します。 5. do_desの出力をpg_hton32で変換し、結果をoutバッファにコピーして返します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/crypt-des.c 内のコード（des_cipher自身を含む）。
  • 呼び出し先: des_init, setup_salt, do_des, pg_ntoh32, pg_hton32, memcpy。

関連データ構造

  • uint32 buffer[2]: 8バイトのデータを処理するための内部バッファ。

失敗モードと境界条件

  • do_desの実行結果が0以外の場合、その値をそのまま戻り値として返します。
  • 入力データが8バイト未満の場合、memcpyによるメモリ破壊の可能性があるため、呼び出し側で適切なバッファサイズを保証する必要があります。

保守担当者向けメモ

  • 本関数はDESアルゴリズムのラッパーであり、現代の基準では脆弱です。互換性維持以外の目的で使用しないでください。
  • do_des内部でCHECK_FOR_INTERRUPTS()が呼び出されるため、長時間実行される処理中にクエリのキャンセルが可能です。
