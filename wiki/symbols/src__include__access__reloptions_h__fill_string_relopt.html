fill_string_relopt

定義

  • 種別: typedef
  • 定義箇所: src/include/access/reloptions.h:134

目的

リレーションオプションとして定義された文字列型の値を、指定されたメモリ領域（構造体など）に格納するためのコールバック関数の型定義です。

シグネチャまたは形状

typedef Size (*fill_string_relopt) (const char *value, void *ptr);

入力と出力

  • **入力**:
  • value: 格納対象となる文字列（const char *）。
  • ptr: 値を書き込む先のメモリ領域へのポインタ（void *）。
  • **出力**:
  • Size: 書き込んだ、あるいは処理したバイト数を返します。

前提条件と不変条件

  • ptr が指すメモリ領域は、呼び出し元によって適切に確保されている必要があります。
  • 文字列の検証は、この関数が呼び出される前に validate_string_relopt 等を通じて完了していることが期待されます。

アルゴリズムまたは振る舞い

  • 指定された文字列 value を、ptr が指す構造体内の適切なオフセット位置へコピーまたは変換して格納します。
  • 具体的な格納ロジックは、この型を実装する各関数に委ねられます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c 内のオプション処理関数（reloptions.c:1067, reloptions.c:1132）から参照されています。
  • **呼び出し先**: この型自体は関数ポインタ型であり、具体的な実装関数が実行時に呼び出されます。

関連データ構造

  • relopt_string: 文字列オプションを定義する構造体で使用されます。
  • relopt_gen: オプションの共通ヘッダー構造体。

失敗モードと境界条件

  • ptr が NULL の場合や、書き込み先のバッファサイズを超える文字列が渡された場合の挙動は、実装関数に依存します。

保守担当者向けメモ

  • 本シンボルは reloptions フレームワークにおける文字列オプションの拡張性を担保しています。新しい文字列オプションを追加する際は、このシグネチャに従った関数を定義し、relopt_string 構造体に登録してください。
