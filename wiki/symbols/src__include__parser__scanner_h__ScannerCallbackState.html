ScannerCallbackState

定義

  • 種別: struct
  • 定義箇所: src/include/parser/scanner.h:124

目的

字句解析中にエラーが発生した際、エラーメッセージにソースコード上の位置情報を付与するためのコールバック状態を保持します。

シグネチャまたは形状

typedef struct ScannerCallbackState
{
	core_yyscan_t yyscanner;
	int			location;
	ErrorContextCallback errcallback;
} ScannerCallbackState;

入力と出力

  • **入力**: setup_scanner_errposition_callback を通じて、現在のスキャナインスタンス（yyscanner）とエラー発生位置（location）が設定されます。
  • **出力**: エラー発生時に errcallback が呼び出され、PostgreSQLのエラーコンテキストスタックに位置情報が登録されます。

前提条件と不変条件

  • yyscanner は scanner_init によって初期化された有効なスキャナインスタンスである必要があります。
  • location はソース文字列の先頭からのバイトオフセットとして保持されます。

アルゴリズムまたは振る舞い

1. setup_scanner_errposition_callback を呼び出し、構造体を初期化してエラーコンテキストスタックに登録します。 2. 字句解析中にエラーが発生すると、登録されたコールバックが実行され、location を基にエラー位置が報告されます。 3. 解析終了後、cancel_scanner_errposition_callback を呼び出し、エラーコンテキストスタックからコールバックを削除してリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parser.c において、パーサーの実行中にインスタンスが生成・利用されています。
  • **呼び出し先**: setup_scanner_errposition_callback および cancel_scanner_errposition_callback が本構造体を操作します。

関連データ構造

  • core_yyscan_t: スキャナインスタンスを指す不透明なポインタ。
  • ErrorContextCallback: PostgreSQLのエラー処理機構で使用されるコールバック構造体。

失敗モードと境界条件

  • スキャナが初期化されていない状態で本構造体を使用した場合、未定義の動作を引き起こす可能性があります。
  • location が解析対象文字列の範囲外である場合、エラー位置の報告が不正確になる可能性があります。

保守担当者向けメモ

  • 本構造体は、字句解析器のライフサイクルと密接に関連しています。エラーハンドリングのロジックを変更する場合は、setup および cancel 関数の実装と整合性を保つ必要があります。
