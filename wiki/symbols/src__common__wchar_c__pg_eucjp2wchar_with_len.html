pg_eucjp2wchar_with_len

定義

  • 種別: function
  • 定義箇所: src/common/wchar.c:183

目的

EUC-JP エンコーディングの文字列を、PostgreSQL 内部で使用されるワイド文字形式（pg_wchar）へ変換します。

シグネチャまたは形状

int pg_eucjp2wchar_with_len(const unsigned char *from, pg_wchar *to, int len)

入力と出力

  • from: 変換元となる EUC-JP エンコーディングの文字列（unsigned char *）。
  • to: 変換後のワイド文字を格納するバッファ（pg_wchar *）。
  • len: 変換対象となる入力文字列のバイト長。
  • 戻り値: 変換されたワイド文字の数。

前提条件と不変条件

  • 入力文字列 from は EUC-JP エンコーディングとして妥当である必要があります。
  • to バッファは、入力文字列から生成されるワイド文字を格納するのに十分なサイズを確保している必要があります。

アルゴリズムまたは振る舞い

内部的に pg_euc2wchar_with_len を呼び出し、EUC-JP 形式のバイトシーケンスをワイド文字へ変換します。

呼び出し元と呼び出し先

  • 呼び出し先: pg_euc2wchar_with_len
  • 呼び出し元: src/common/wchar.c 内の pg_wchar_table において、PG_EUC_JP および PG_EUC_JIS_2004 エンコーディングの変換関数として登録されています。

関連データ構造

  • pg_wchar: PostgreSQL 内部でワイド文字を表現するための型。
  • pg_wchar_table: 各エンコーディングの操作関数を保持するグローバルな関数テーブル。

失敗モードと境界条件

  • 入力文字列が不正なバイトシーケンスを含む場合の挙動については、呼び出し先である pg_euc2wchar_with_len の仕様に依存します。

保守担当者向けメモ

  • 本関数は pg_euc2wchar_with_len への単なるラッパーです。EUC-JP 固有の変換ロジックが必要になった場合は、この関数内で直接実装するか、呼び出し先を変更する必要があります。
