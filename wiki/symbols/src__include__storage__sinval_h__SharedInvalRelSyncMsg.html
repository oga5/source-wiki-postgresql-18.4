SharedInvalRelSyncMsg

定義

  • 種別: struct
  • 定義箇所: src/include/storage/sinval.h:116

目的

リレーション同期キャッシュ（RelationSyncCache）の無効化を通知するためのメッセージ構造体です。共有メモリ上の無効化キューを通じて、複数のバックエンド間でリレーションの同期状態を整合させるために使用されます。

シグネチャまたは形状

typedef struct
{
	int8		id;				/* type field --- must be first */
	Oid			dbId;			/* database ID */
	Oid			relid;			/* relation ID, or 0 if whole RelationSyncCache */
} SharedInvalRelSyncMsg;

入力と出力

  • **入力**: id（メッセージ識別子）、dbId（対象データベースのOID）、relid（対象リレーションのOID）。
  • **出力**: なし（本構造体はデータ保持用であり、関数ではない）。

前提条件と不変条件

  • id フィールドは構造体の先頭に配置されなければなりません。
  • relid が 0 の場合、特定のテーブルではなく RelationSyncCache 全体の無効化を意味します。

アルゴリズムまたは振る舞い

本構造体は SharedInvalidationMessage 共用体の一部として、共有メモリ上の無効化キューに格納されます。バックエンドがリレーションの同期に関連する変更を行った際、このメッセージがキューに投入され、受信側バックエンドがこれを読み取ることで自身のキャッシュを無効化します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、具体的な呼び出し元関数は特定できません。
  • **呼び出し先**: 提供されたソース情報からは、具体的な呼び出し先関数は特定できません。

関連データ構造

  • SharedInvalidationMessage: 本構造体を包含する共用体。
  • Oid: PostgreSQLのオブジェクト識別子型。

失敗モードと境界条件

  • relid に 0 を指定した場合の挙動は、キャッシュ全体への影響を及ぼすため、広範囲なキャッシュ再ロードを誘発する可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/storage/sinval.h で定義されており、キャッシュ無効化メカニズムのプロトコルの一部です。変更時は ReceiveSharedInvalidMessages 等の処理ロジックとの整合性に注意してください。
