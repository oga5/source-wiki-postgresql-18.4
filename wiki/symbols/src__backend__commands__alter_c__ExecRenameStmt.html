ExecRenameStmt

定義

  • 種別: function
  • 定義箇所: src/backend/commands/alter.c:372

目的

SQLの ALTER ... RENAME ステートメントを処理し、対象オブジェクトの名前変更を実行するためのメインエントリポイントです。

シグネチャまたは形状

ObjectAddress ExecRenameStmt(RenameStmt *stmt)

入力と出力

  • **入力**: RenameStmt *stmt（パーサーが生成した名前変更操作の構造体）
  • **出力**: ObjectAddress（変更されたオブジェクトの識別情報）

前提条件と不変条件

  • stmt->renameType に基づき、適切なサブコマンド関数へディスパッチされます。
  • 汎用的なオブジェクト型（OBJECT_FUNCTION 等）の場合、get_object_address により対象オブジェクトのロックとOID取得が行われます。

アルゴリズムまたは振る舞い

1. stmt->renameType を評価し、オブジェクト型に応じて以下のいずれかの処理を行います。

  • **専用関数への委譲**: RenameConstraint, RenameDatabase, RenameRelation 等、オブジェクト固有の関数を呼び出します。
  • **汎用的なカタログ更新**: OBJECT_FUNCTION や OBJECT_TYPE など、多くのオブジェクト型に対しては以下の手順を実行します。
  • get_object_address で対象オブジェクトのOIDとカタログ情報を取得（AccessExclusiveLock を使用）。
  • table_open で対象カタログを開く。
  • AlterObjectRename_internal を呼び出し、カタログ内の名前属性を更新。
  • table_close でカタログを閉じる。

    • 2. 未知の renameType が渡された場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（パーサーからのユーティリティコマンド実行フロー）
  • **呼び出し先**:
  • RenameConstraint, RenameDatabase, RenameRole, RenameSchema, RenameTableSpace, RenameRelation, renameatt, RenameRewriteRule, renametrig, rename_policy, RenameType
  • get_object_address, table_open, AlterObjectRename_internal, table_close

関連データ構造

  • RenameStmt: src/include/nodes/parsenodes.h で定義される、名前変更操作を保持するノード。
  • ObjectAddress: オブジェクトを特定するためのOIDとクラスIDのペア。

失敗モードと境界条件

  • サポートされていない renameType が指定された場合、エラーが発生します。
  • get_object_address が対象オブジェクトを見つけられない場合、または権限不足の場合はエラーが発生します。

保守担当者向けメモ

  • 新しいオブジェクト型を RENAME に対応させる場合、switch 文にケースを追加してください。
  • 物理的なストレージや複雑な依存関係を伴うオブジェクト（テーブル等）は、本関数内で直接更新せず、専用の Rename... 関数を介して処理する必要があります。
