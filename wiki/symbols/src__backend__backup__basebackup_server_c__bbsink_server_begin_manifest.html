bbsink_server_begin_manifest

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_server.c:227

目的

バックアップマニフェストファイルを書き出すための準備として、一時ファイル backup_manifest.tmp を作成し、後続のシンク処理へ制御を転送する。

シグネチャまたは形状

static void bbsink_server_begin_manifest(bbsink *sink)

入力と出力

  • **入力**: bbsink *sink （bbsink_server 構造体としてキャストして使用）
  • **出力**: なし（副作用としてファイルシステム上に一時ファイルを作成）

前提条件と不変条件

  • mysink->file が 0 であること（ファイルが未オープンであること）。
  • mysink->pathname に有効なディレクトリパスが格納されていること。

アルゴリズムまたは振る舞い

1. mysink->pathname を基に backup_manifest.tmp のパス文字列を生成する。 2. PathNameOpenFile を使用し、O_CREAT | O_EXCL | O_WRONLY | PG_BINARY モードで一時ファイルを作成する。 3. ファイル作成に失敗した場合は ereport(ERROR, ...) で処理を中断する。 4. 生成したパス文字列のメモリを解放する。 5. bbsink_forward_begin_manifest(sink) を呼び出し、後続のシンク処理へマニフェスト開始イベントを伝播させる。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_server_ops 構造体を通じて、バックアップ処理のメインフローから間接的に呼び出される。
  • **呼び出し先**: bbsink_forward_begin_manifest（シンクチェーンの次段へ処理を転送）。

関連データ構造

  • bbsink_server: ローカルファイル保存用の状態を保持する構造体。file（ファイル記述子）および pathname（保存先ディレクトリ）を管理する。

失敗モードと境界条件

  • **ファイル作成失敗**: 指定されたパスへの書き込み権限がない、または同名のファイルが既に存在する場合などに ereport によりエラーが発生する。

保守担当者向けメモ

  • 本関数はマニフェストを一時ファイルとして作成する。最終的なファイル名へのリネームは bbsink_server_end_manifest で行われるため、本関数単体ではバックアップの完了は保証されない。
  • Assert により、既にファイルがオープンされている状態での二重呼び出しを防止している。
