CreateRestrictedProcess

定義

  • 種別: function
  • 定義箇所: src/common/restricted_token.c:44

目的

Windows環境において、管理者権限やパワーユーザー権限を剥奪した「制限付きトークン」を生成し、そのトークンを用いて新しいプロセスを安全に起動する。

シグネチャまたは形状

HANDLE CreateRestrictedProcess(char *cmd, PROCESS_INFORMATION *processInfo)

入力と出力

  • **入力**:
  • cmd: 実行するコマンドライン文字列。
  • processInfo: 新しく生成されたプロセスのハンドルやIDを格納するための PROCESS_INFORMATION 構造体へのポインタ。
  • **出力**:
  • 成功時: 生成された制限付きトークンの HANDLE を返す。
  • 失敗時: 0 を返す。

前提条件と不変条件

  • Windows環境での実行を前提とする（WIN32 マクロ環境下）。
  • 呼び出し元は FRONTEND マクロが定義されている必要がある。
  • processInfo は有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. OpenProcessToken を使用して現在のプロセスのトークンを取得する。 2. AllocateAndInitializeSid を用いて、管理者（DOMAIN_ALIAS_RID_ADMINS）およびパワーユーザー（DOMAIN_ALIAS_RID_POWER_USERS）のSIDを生成する。 3. CreateRestrictedToken を呼び出し、DISABLE_MAX_PRIVILEGE フラグを付与して制限付きトークンを作成する。 4. 生成したSIDとトークンハンドルを解放・クローズする。 5. 非Cygwin環境の場合、AddUserToTokenDacl を呼び出してトークンのDACLを調整する。 6. CreateProcessAsUser を使用し、制限付きトークンを用いて CREATE_SUSPENDED 状態でプロセスを生成する。 7. ResumeThread を呼び出して子プロセスの実行を開始し、トークンハンドルを返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_ctl/pg_ctl.c
  • src/test/regress/pg_regress.c
  • src/common/restricted_token.c (内部の get_restricted_token からの呼び出しが示唆されている)
  • **呼び出し先**:
  • Windows API: OpenProcessToken, AllocateAndInitializeSid, CreateRestrictedToken, FreeSid, CloseHandle, CreateProcessAsUser, ResumeThread
  • 内部関数: AddUserToTokenDacl (非Cygwin環境時)

関連データ構造

  • PROCESS_INFORMATION: プロセスおよびメインスレッドのハンドルとIDを保持。
  • SID_IDENTIFIER_AUTHORITY: SIDの識別子。
  • SID_AND_ATTRIBUTES: 制限対象のSIDリスト。

失敗モードと境界条件

  • トークンのオープン、SIDの割り当て、制限付きトークンの作成、またはプロセスの生成に失敗した場合、pg_log_error でエラーコードを出力し 0 を返す。
  • CreateProcessAsUser 失敗時は、子プロセスは起動されない。

保守担当者向けメモ

  • 本関数はプロセスを再帰的に再起動する仕組みの一部であるため、デバッグ時にはプロセスツリーの深さに注意すること。
  • __CYGWIN__ 環境では AddUserToTokenDacl がスキップされる点に留意すること。
