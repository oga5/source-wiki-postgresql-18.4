test_regex

定義

  • **種別**: function
  • **定義箇所**: src/test/modules/test_regex/test_regex.c:78

目的

PostgreSQL の正規表現エンジンの動作を検証するための Set Returning Function (SRF) です。正規表現のコンパイル情報およびマッチング結果の詳細を text[] 配列として返します。

シグネチャまたは形状

Datum test_regex(PG_FUNCTION_ARGS)

SQL から test_regex(pattern text, string text, flags text) として呼び出されます。

入力と出力

  • **入力**:
  • pattern (text): 正規表現パターン。
  • string (text): マッチング対象の文字列。
  • flags (text): テスト用のフラグ（g, i, n 等）。
  • **出力**: SETOF text[]。最初の行は正規表現のメタ情報、以降の行は各マッチング結果を返します。

前提条件と不変条件

  • SRF_IS_FIRSTCALL 時のメモリコンテキストは funcctx->multi_call_memory_ctx に切り替えられ、マッチング状態が保持されます。
  • matchctx->elems および matchctx->nulls は、マッチング結果構築のために事前に確保されます。

アルゴリズムまたは振る舞い

1. **初回呼び出し時**:

  • parse_test_flags でフラグを解析し、test_re_compile でパターンをコンパイルします。
  • setup_test_matches で対象文字列全体のマッチングを実行し、結果を test_regex_ctx に格納します。
  • build_test_info_result を呼び出し、正規表現のメタ情報を最初の行として返します。

    • 2. **2回目以降の呼び出し**:

  • test_regex_ctx から次のマッチング結果を build_test_match_result で構築して返します。
  • 全てのマッチングが完了すると SRF_RETURN_DONE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（直接的な C 言語レベルの呼び出し元はソース内に明示されていません）。
  • **呼び出し先**:
  • parse_test_flags
  • test_re_compile
  • setup_test_matches
  • build_test_info_result
  • pg_regfree
  • build_test_match_result

関連データ構造

  • FuncCallContext: SRF の状態管理に使用。
  • test_regex_ctx: 複数回の呼び出し間でマッチング位置やフラグを保持。
  • test_re_flags: コンパイルおよび実行オプション。
  • regex_t: コンパイルされた正規表現パターン。

失敗モードと境界条件

  • 正規表現のコンパイル失敗や、メモリ確保失敗時には PostgreSQL のエラーハンドリング機構により例外が発生します。
  • マッチング結果が存在しない場合、初回呼び出しでメタ情報を返した直後に SRF_RETURN_DONE が実行されます。

保守担当者向けメモ

  • 本関数は Tcl の正規表現テストスイートの挙動を模倣しています。
  • pg_regfree は初回呼び出しの最後に実行され、コンパイルされたパターンを解放します。
  • マルチバイト文字の扱いは test_re_compile 内の pg_wchar 変換に依存しています。
