find_placeholders_in_jointree

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:186

目的

クエリのジョイントリーを走査し、PlaceHolderVar (PHV) を特定して、対応する PlaceHolderInfo を初期化・登録するために使用されます。

シグネチャまたは形状

void find_placeholders_in_jointree(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root - クエリ全体のプランニング情報。
  • **出力**: なし（副作用として root 内の PlaceHolderInfo 構造体が構築されます）。

前提条件と不変条件

  • root->placeholdersFrozen が false であること（PHVのセットが凍結される前に実行される必要がある）。
  • root->parse->jointree が存在し、FromExpr 型であること。

アルゴリズムまたは振る舞い

1. root->glob->lastPHId を確認し、クエリ内に PlaceHolderVar が存在しない場合は即座に終了します。 2. root->parse->jointree を開始点として、find_placeholders_recurse を呼び出し、ジョイントリーを再帰的に走査します。 3. 走査を通じて各PHVを検出し、クエリ全体で一意な PlaceHolderInfo を生成・管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planmain.c:187
  • **呼び出し先**: find_placeholders_recurse

関連データ構造

  • PlannerInfo: クエリの最適化状態を保持する主要構造体。
  • PlaceHolderVar: 結合によるNULL生成の影響を表現するノード。
  • PlaceHolderInfo: PHVの評価レベルや依存関係を保持する情報構造体。

失敗モードと境界条件

  • root->placeholdersFrozen が true の場合に呼び出されるとアサーションエラーが発生します。
  • root->parse->jointree が FromExpr 以外である場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は最適化プロセスの初期段階で実行される必要があります。
  • root->glob->lastPHId が 0 の場合は処理をスキップする最適化が行われています。
