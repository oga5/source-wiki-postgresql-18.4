InitializeLogRepWorker

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/worker.c:4658

目的

論理レプリケーションワーカー（適用ワーカーまたはテーブル同期ワーカー）の実行環境を初期化し、サブスクリプション設定を読み込んで適用処理を開始可能な状態にする。

シグネチャまたは形状

void InitializeLogRepWorker(void)

入力と出力

  • **入力**: グローバル変数 MyLogicalRepWorker（ワーカーの識別情報）を参照。
  • **出力**: なし（副作用としてプロセス状態を更新）。

前提条件と不変条件

  • MyLogicalRepWorker が初期化済みであること。
  • トランザクションコンテキスト内で呼び出される必要がある。

アルゴリズムまたは振る舞い

1. **セッション設定**: session_replication_role を replica に、search_path を空に設定し、セキュリティを確保する。 2. **接続確立**: BackgroundWorkerInitializeConnectionByOid を呼び出し、対象データベースへ接続する。 3. **サブスクリプションのロード**: ApplyContext メモリコンテキストを作成し、SubscriptionRelationId をロックしてサブスクリプション情報を取得する。 4. **検証**: サブスクリプションが存在しない、または無効化されている場合はエラーログを出力してプロセスを終了する。 5. **同期設定**: synchronous_commit をサブスクリプション設定に合わせて適用する。 6. **コールバック登録**: SUBSCRIPTIONOID および AUTHOID の変更を監視するシステムキャッシュコールバックを登録する。 7. **終了処理の登録**: before_shmem_exit を使用して、シャットダウン時にオリジン状態をリセットする replorigin_reset を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/backend/replication/logical/worker.c（自身を含む）
  • **呼び出し先**:
  • BackgroundWorkerInitializeConnectionByOid
  • GetSubscription
  • CacheRegisterSyscacheCallback
  • before_shmem_exit
  • proc_exit / apply_worker_exit（エラー時）

関連データ構造

  • MyLogicalRepWorker: ワーカーのメタデータ。
  • MySubscription: ロードされたサブスクリプション情報。
  • ApplyContext: 適用処理用のメモリコンテキスト。

失敗モードと境界条件

  • **サブスクリプション不在**: 起動中にサブスクリプションが削除された場合、ログを出力して proc_exit(0) で正常終了する。
  • **サブスクリプション無効**: enabled フラグが偽の場合、ログを出力して apply_worker_exit() で終了する。

保守担当者向けメモ

  • 本関数はワーカーの起動直後に実行されるため、ここでの設定変更は後続のすべての適用処理に影響する。
  • replorigin_reset の登録は、不完全なトランザクションのコミットによるデータ損失を防ぐために不可欠である。
