fsm_get_child

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/freespace.c:544

目的

FSM（Free Space Map）の木構造において、指定された親ノードの論理アドレスとスロット番号から、対応する子ノードの論理アドレス（FSMAddress）を算出する。

シグネチャまたは形状

static FSMAddress fsm_get_child(FSMAddress parent, uint16 slot)

入力と出力

  • **入力**:
  • parent: 親ノードの論理アドレス（FSMAddress 構造体）。
  • slot: 親ページ内での子ノードの位置を示すインデックス（uint16）。
  • **出力**:
  • 子ノードの論理アドレス（FSMAddress 構造体）。

前提条件と不変条件

  • parent.level が FSM_BOTTOM_LEVEL より大きいこと（Assert により検証）。

アルゴリズムまたは振る舞い

1. 親ノードのレベルから 1 を減算し、子ノードのレベルを決定する。 2. 親ノードの論理ページ番号に SlotsPerFSMPage を乗算し、指定された slot を加算することで、子ノードの論理ページ番号を算出する。 3. 算出されたレベルとページ番号を持つ FSMAddress を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/freespace/freespace.c 内の検索およびバキューム処理（fsm_search や fsm_vacuum_page 等）から呼び出される。
  • **呼び出し先**:
  • 本関数は計算のみを行うため、他の関数を呼び出さない。

関連データ構造

  • FSMAddress: FSM の木構造上の位置（レベルと論理ページ番号）を保持する構造体。

失敗モードと境界条件

  • parent.level が FSM_BOTTOM_LEVEL の場合、アサーション違反によりプログラムが停止する可能性がある。

保守担当者向けメモ

  • 本関数は FSM の木構造のナビゲーションにおける基本的な算術変換を行う。
  • SlotsPerFSMPage は src/include/storage/fsm_internals.h で定義されている定数に依存している。
