get_raw_page_fork_1_9

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/rawpage.c:99

目的

指定されたリレーション、フォーク、およびブロック番号に対応するデータベースページを物理ストレージから読み出し、その内容を bytea 形式で返します。

シグネチャまたは形状

Datum get_raw_page_fork_1_9(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relname (text): 対象リレーション名。
  • forkname (text): 対象フォーク名（例: 'main', 'fsm'）。
  • blkno (int64): 読み出すブロック番号。
  • **出力**:
  • bytea: 読み出されたページイメージ。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要です。
  • blkno は 0 以上かつ MaxBlockNumber 以下である必要があります。

アルゴリズムまたは振る舞い

1. forkname を ForkNumber 型に変換します。 2. blkno の範囲チェックを行い、範囲外の場合はエラーを報告します。 3. get_raw_page_internal を呼び出し、共有バッファから該当ページを取得します。 4. 取得したページデータを bytea 形式として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: forkname_to_number, text_to_cstring, get_raw_page_internal。

関連データ構造

  • ForkNumber: フォークの種類を識別する列挙型。
  • bytea: PostgreSQL の可変長バイナリデータ型。

失敗モードと境界条件

  • blkno が 0 未満または MaxBlockNumber を超える場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなります。
  • 無効なフォーク名が指定された場合、forkname_to_number がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は pageinspect 拡張機能の SQL API の一部です。
  • 物理データへの直接アクセスを伴うため、セキュリティ上の制約（スーパーユーザー権限）が適用されています。
  • get_raw_page_internal が共有バッファの読み込みを担うため、ページ取得のロジック変更時は同関数を確認してください。
