autoprewarm_start_worker

定義

  • 種別: function
  • 定義箇所: contrib/pg_prewarm/autoprewarm.c:814

目的

PostgreSQL の自動プリウォーム用リーダーワーカーを明示的に起動するための SQL 呼び出し可能な関数です。

シグネチャまたは形状

Datum autoprewarm_start_worker(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS (SQL インターフェース経由)
  • 出力: PG_RETURN_VOID() (成功時)
  • 失敗時: ereport(ERROR, ...) により例外をスロー

前提条件と不変条件

  • GUC 変数 autoprewarm が有効であること。
  • 既にリーダーワーカーが起動中（apw_state->bgworker_pid != InvalidPid）でないこと。

アルゴリズムまたは振る舞い

1. autoprewarm GUC が無効な場合、エラーを報告して終了。 2. apw_init_shmem() を呼び出し、共有メモリ状態を初期化。 3. apw_state->lock を排他ロックし、現在のリーダーワーカーの PID を確認。 4. 既に PID が存在する場合、エラーを報告して終了。 5. apw_start_leader_worker() を呼び出し、リーダーワーカーを起動。 6. 正常終了。

呼び出し元と呼び出し先

  • 呼び出し元: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先:
  • apw_init_shmem()
  • LWLockAcquire / LWLockRelease
  • apw_start_leader_worker()
  • ereport

関連データ構造

  • AutoPrewarmSharedState (共有メモリ上の制御構造体)

失敗モードと境界条件

  • autoprewarm が無効な場合: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー。
  • 既にワーカーが稼働中の場合: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー。

保守担当者向けメモ

  • 本関数は SQL からのトリガーを想定しています。バックグラウンドワーカーのライフサイクル管理については postmaster/bgworker.h の仕様に従います。
