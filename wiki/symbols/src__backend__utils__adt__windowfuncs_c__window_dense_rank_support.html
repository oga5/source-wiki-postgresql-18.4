window_dense_rank_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/windowfuncs.c:219

目的

DENSE_RANK ウィンドウ関数のクエリ最適化をサポートします。オプティマイザに対し、関数の単調性情報や、実行時の計算コストを削減するためのフレーム指定のヒントを提供します。

シグネチャまたは形状

Datum window_dense_rank_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して Node *rawreq を受け取ります。
  • **出力**: SupportRequestWFuncMonotonic または SupportRequestOptimizeWindowClause の要求に応じて、更新された要求構造体へのポインタを Datum として返します。該当しない場合は NULL を返します。

前提条件と不変条件

  • rawreq は SupportRequestWFuncMonotonic または SupportRequestOptimizeWindowClause のいずれかである必要があります。

アルゴリズムまたは振る舞い

1. **単調性の通知**: SupportRequestWFuncMonotonic が要求された場合、dense_rank() が単調増加関数であることを示し、req->monotonic に MONOTONICFUNC_INCREASING を設定します。 2. **フレーム最適化**: SupportRequestOptimizeWindowClause が要求された場合、フレームオプションを ROWS モード（FRAMEOPTION_ROWS）に設定します。これにより、デフォルトの RANGE モードで必要となる「ピア行（同順位行）」のチェックを回避し、実行時の計算コストを削減します。

  • 設定値: FRAMEOPTION_NONDEFAULT | FRAMEOPTION_ROWS | FRAMEOPTION_START_UNBOUNDED_PRECEDING | FRAMEOPTION_END_CURRENT_ROW

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のクエリプランナ（オプティマイザ）が、ウィンドウ関数の最適化フェーズで呼び出します。
  • **呼び出し先**: 呼び出し先に関する情報は提供されていません。

関連データ構造

  • SupportRequestWFuncMonotonic: 単調性情報を伝達するための構造体。
  • SupportRequestOptimizeWindowClause: ウィンドウ句のフレーム最適化情報を伝達するための構造体。

失敗モードと境界条件

  • 認識されない要求タイプが渡された場合、何も変更せずに NULL を返します。

保守担当者向けメモ

  • 本関数は row_number のサポート関数と同様のフレーム最適化戦略を採用しています。
  • RANGE モードから ROWS モードへの変更は、ピアチェックを省略するための重要な最適化です。変更を加える際は、DENSE_RANK の順位計算ロジックに影響がないか確認してください。
