table_openrv_extended

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/table.c:102

目的

RangeVar で指定されたリレーションをオープンし、それがテーブルであることを検証した上で Relation 構造体を返します。

シグネチャまたは形状

Relation table_openrv_extended(const RangeVar *relation, LOCKMODE lockmode, bool missing_ok)

入力と出力

  • **入力**:
  • relation: オープン対象を指定する RangeVar 構造体。
  • lockmode: リレーション取得時に適用するロックモード。
  • missing_ok: true の場合、リレーションが存在しなくてもエラーにせず NULL を返す。
  • **出力**:
  • 成功時: オープンされた Relation 構造体へのポインタ。
  • 失敗時: missing_ok が true でリレーションが存在しない場合は NULL。それ以外はエラーが発生する。

前提条件と不変条件

  • オープンされたリレーションがインデックスや複合型ではないことを保証します。

アルゴリズムまたは振る舞い

1. relation_openrv_extended を呼び出し、低レベルなリレーションオープン処理を実行します。 2. リレーションが取得できた場合（r が非 NULL）、validate_relation_kind(r) を呼び出して、対象がテーブルとして有効なリレーション種別であることを検証します。 3. 検証を通過した Relation ポインタを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • relation_openrv_extended (低レベルなリレーションオープン)
  • validate_relation_kind (リレーション種別の検証)
  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c:1442
  • src/backend/parser/parse_relation.c:1441
  • その他、src/include/access/table.h を通じた外部参照。

関連データ構造

  • Relation: オープンされたリレーションを保持する構造体。
  • RangeVar: リレーションの識別情報（スキーマ名、テーブル名等）を保持する構造体。

失敗モードと境界条件

  • validate_relation_kind の検証に失敗した場合、ereport(ERROR) が発行され処理が中断されます。
  • missing_ok が false の状態で対象リレーションが存在しない場合、relation_openrv_extended 内でエラーが発生します。

保守担当者向けメモ

  • 本関数は relation_openrv_extended のラッパーであり、テーブルアクセス専用のガードレール（validate_relation_kind）を適用する役割を担います。
  • 実際のデータ操作ロジックは含まれておらず、テーブルアクセスの入り口として機能します。
