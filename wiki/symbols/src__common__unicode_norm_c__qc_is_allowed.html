qc_is_allowed

定義

  • 種別: function
  • 定義箇所: src/common/unicode_norm.c:592

目的

指定された Unicode 正規化形式（NFC または NFKC）に基づき、特定の文字が正規化済みであるか、あるいは正規化処理が必要かを判定するためのクイックチェック値を取得する。

シグネチャまたは形状

static UnicodeNormalizationQC qc_is_allowed(UnicodeNormalizationForm form, pg_wchar ch)

入力と出力

  • **入力**:
  • form: UnicodeNormalizationForm 型。UNICODE_NFC または UNICODE_NFKC を指定する。
  • ch: pg_wchar 型。判定対象の文字コード。
  • **出力**:
  • UnicodeNormalizationQC 型の値を返す。これは文字の正規化状態（YES, NO, MAYBE 等）を示す。

前提条件と不変条件

  • form には UNICODE_NFC または UNICODE_NFKC のみが許可される。それ以外の値が渡された場合、Assert(false) により停止する。

アルゴリズムまたは振る舞い

1. 指定された form に応じて、対応する静的ハッシュテーブル（UnicodeNormInfo_NFC_QC または UnicodeNormInfo_NFKC_QC）を選択する。 2. qc_hash_lookup を呼び出し、対象文字 ch の正規化プロパティを検索する。 3. プロパティが見つかった場合は、そのエントリ内の quickcheck 値を返す。 4. プロパティが見つからない（ハッシュテーブルに存在しない）場合は、正規化が不要であることを示す UNICODE_NORM_QC_YES を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/unicode_norm.c 内の正規化処理フローから呼び出される。
  • **呼び出し先**: qc_hash_lookup を呼び出す。

関連データ構造

  • UnicodeNormalizationForm: 正規化形式を定義する列挙型。
  • UnicodeNormalizationQC: クイックチェックの結果を保持する列挙型。
  • pg_unicode_normprops: 文字の正規化プロパティを保持する構造体。

失敗モードと境界条件

  • UNICODE_NFD や UNICODE_NFKD が form として渡された場合、アサーション違反となる。
  • ハッシュテーブル検索で対象が見つからない場合、デフォルトで「正規化済み（YES）」とみなす安全側の挙動をとる。

保守担当者向けメモ

  • 本関数はバックエンドでの高速化を目的としており、フロントエンドでの利用についてはソース内の FRONTEND マクロ等の条件分岐に注意すること。
  • NFD/NFKD のクイックチェックは本関数ではサポートされていないため、これらが必要な場合は別のロジックを検討する必要がある。
