SharedInvalSmgrMsg

定義

  • 種別: struct
  • 定義箇所: src/include/storage/sinval.h:88

目的

ストレージマネージャ（smgr）に関連するキャッシュ無効化メッセージを保持するためのデータ構造です。共有メモリ上の無効化キューを介して、バックエンド間でリレーションの物理的な状態変更を通知するために使用されます。

シグネチャまたは形状

typedef struct
{
	int8		id;
	int8		backend_hi;
	uint16		backend_lo;
	RelFileLocator rlocator;
} SharedInvalSmgrMsg;

入力と出力

  • **入力**: 構造体の各フィールド（id はメッセージ種別、backend_hi/lo は一時リレーションの場合のバックエンド識別子、rlocator は対象リレーションの物理識別子）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • id フィールドは構造体の先頭に配置され、メッセージの型識別子として機能する必要があります。
  • フィールドレイアウトは、全体で16バイトにパッキングされるように設計されています。

アルゴリズムまたは振る舞い

本構造体は、SendSharedInvalidMessages を通じて共有メモリキューに書き込まれ、他のバックエンドによって ReceiveSharedInvalidMessages で読み取られます。受信側は id を確認し、rlocator に基づいてストレージマネージャのキャッシュを無効化します。他の無効化メッセージと異なり、ストレージマネージャの無効化は即時送信される特性があります。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元情報は提供されていません。
  • **呼び出し先**: 本構造体は SharedInvalidationMessage 共用体の一部として利用されます。

関連データ構造

  • SharedInvalidationMessage: 本構造体を含む、すべての無効化メッセージを包含する共用体。
  • RelFileLocator: リレーションの物理的な場所を特定するための構造体。

失敗モードと境界条件

  • メッセージキューの溢れ: 共有メモリ上のキューが満杯になった場合、PROCSIG_CATCHUP_INTERRUPT が発生し、強制的な同期処理がトリガーされます。
  • 一時リレーションの識別: backend_hi および backend_lo は、一時リレーションを扱う際にバックエンドを識別するために使用されます。

保守担当者向けメモ

  • 本構造体は16バイトにパッキングされるよう設計されているため、フィールドの追加や変更を行う際はメモリレイアウトに注意してください。
  • smgr の無効化はトランザクションコミットを待たずに即時送信されるため、他のカタログ無効化メッセージとの処理順序やタイミングに依存関係がないか確認が必要です。
