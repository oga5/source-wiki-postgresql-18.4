do_init

定義

  • 種別: function
  • 定義箇所: src/bin/pg_ctl/pg_ctl.c:901

目的

データベースクラスターの初期化を行うために、外部コマンド initdb を実行する。

シグネチャまたは形状

static void do_init(void)

入力と出力

  • **入力**: グローバル変数 exec_path, pgdata_opt, post_opts, silent_mode を参照。
  • **出力**: なし（成功時は終了、失敗時は標準エラー出力へメッセージを出力し exit(1) を実行）。

前提条件と不変条件

  • exec_path が未設定の場合、find_other_exec_or_die を用いて initdb のパスを特定する。
  • pgdata_opt および post_opts が NULL の場合は空文字列として扱う。

アルゴリズムまたは振る舞い

1. initdb の実行パスを特定する。 2. silent_mode の設定に基づき、initdb 実行コマンド文字列を構築する（サイレントモード時は出力を DEVNULL へリダイレクトする）。 3. fflush(NULL) を呼び出し、バッファをフラッシュする。 4. system(cmd) を呼び出し、外部コマンドとして initdb を実行する。 5. コマンドの戻り値が 0 以外の場合、エラーメッセージを出力してプロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_ctl/pg_ctl.c 内のメイン処理フロー（main 関数等からの呼び出しが示唆されている）。
  • **呼び出し先**: find_other_exec_or_die, psprintf, fflush, system, write_stderr, exit。
  • **注記**: src/interfaces/libpq/ に存在する PQinitSSL の引数 do_init は、本関数とは無関係な変数名である。

関連データ構造

  • なし。

失敗モードと境界条件

  • initdb コマンドが見つからない場合、find_other_exec_or_die によりプロセスが終了する。
  • system(cmd) が 0 以外の値を返した場合（コマンド実行失敗）、エラーメッセージを出力して exit(1) を実行する。

保守担当者向けメモ

  • 本関数は system() を使用してシェル経由でコマンドを実行するため、引数に渡されるパスやオプションのサニタイズ状況に注意すること。
  • initdb の実行結果に依存するため、環境変数や権限設定が適切であることを前提としている。
