BackendTypeMask

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/postmaster.c:138

目的

PostgreSQLのバックエンドプロセスの種類（BackendType）をビットマスクとして管理し、複数のプロセス種別を効率的に表現・操作するために使用されます。

シグネチャまたは形状

typedef struct
{
	uint32		mask;
} BackendTypeMask;

入力と出力

  • **入力**: uint32 型のビットフィールド。
  • **出力**: BackendTypeMask 構造体。

前提条件と不変条件

  • mask フィールドは、BACKEND_NUM_TYPES に基づくビット演算によって操作されることが想定されています。
  • BTYPE_MASK_ALL は全ビットがセットされた状態、BTYPE_MASK_NONE は全ビットがクリアされた状態を保持します。

アルゴリズムまたは振る舞い

以下のインライン関数を通じて、ビットマスクの操作が行われます。

  • **生成**: 単一の BackendType からマスクを作成。
  • **追加**: btmask_add_n により、複数の BackendType を既存のマスクに追加。
  • **削除**: btmask_del により、特定の BackendType をマスクから除去。
  • **判定**: btmask_contains により、特定の BackendType がマスクに含まれているかを判定。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c 内の各インライン関数（btmask_add_n, btmask_del, btmask_contains 等）から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • BackendType (列挙型): マスクの各ビットに対応するプロセス種別の定義。
  • BTYPE_MASK_ALL, BTYPE_MASK_NONE: BackendTypeMask 型の静的定数。

失敗モードと境界条件

  • mask のビット幅（uint32）を超える BackendType を指定した場合の挙動は未定義です。
  • 呼び出し元と呼び出し先の関係性については、提供されたソース範囲内では上記インライン関数による操作に限定されています。

保守担当者向けメモ

  • 本構造体は postmaster.c 内で定義されており、プロセス管理のフィルタリング処理において頻繁に使用されます。
  • 新しい BackendType を追加する際は、BackendTypeMask を操作する各インライン関数が正しくビット演算を処理できるか確認してください。
