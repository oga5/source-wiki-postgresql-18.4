compatible_oper_opid

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_oper.c:486

目的

指定された名前と引数型に適合する二項演算子を検索し、そのシステムカタログ上の OID を返します。

シグネチャまたは形状

Oid compatible_oper_opid(List *op, Oid arg1, Oid arg2, bool noError)

入力と出力

  • **入力**:
  • op: 演算子名（List 形式）。
  • arg1, arg2: 演算子の左辺および右辺の型 OID。
  • noError: 検索失敗時にエラーを投げるか（false）、InvalidOid を返すか（true）を制御するフラグ。
  • **出力**:
  • 適合する演算子の Oid。見つからない場合は InvalidOid。

前提条件と不変条件

  • compatible_oper を介してシステムカタログ（pg_operator）のキャッシュを操作します。
  • 成功時には ReleaseSysCache を呼び出し、取得したキャッシュエントリを解放します。

アルゴリズムまたは振る舞い

1. compatible_oper を呼び出し、指定された引数型に適合する演算子のキャッシュエントリ（Operator）を取得します。 2. エントリが取得できた場合、oprid() を使用してその OID を抽出します。 3. ReleaseSysCache でキャッシュエントリを解放します。 4. 取得した OID を返します。エントリが見つからなかった場合は InvalidOid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/indexcmds.c: インデックス作成時の演算子解決に使用。
  • src/backend/parser/parse_clause.c: ORDER BY 句などのソート演算子解決に使用。
  • **呼び出し先**:
  • compatible_oper: 演算子の検索を実行。
  • oprid: Operator 構造体から OID を抽出。
  • ReleaseSysCache: システムキャッシュの参照カウントを減少。

関連データ構造

  • Oid: PostgreSQL のシステムオブジェクト識別子。
  • Operator: pg_operator システムカタログの行を指すキャッシュハンドル。

失敗モードと境界条件

  • 適合する演算子が存在しない場合、noError が false であれば compatible_oper 内部でエラーが発生し、処理が中断されます。noError が true の場合は InvalidOid が返されます。

保守担当者向けメモ

  • 本関数は compatible_oper のラッパーであり、演算子の OID のみが必要な場合に利用されます。
  • 演算子解決のロジック変更は compatible_oper 側で行う必要があります。
