PLy_plan_dealloc

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_planobject.c:90

目的

Pythonオブジェクト PLyPlanObject が破棄される際、関連するPostgreSQL側のSPI実行計画およびメモリコンテキストを安全に解放する。

シグネチャまたは形状

static void PLy_plan_dealloc(PLyPlanObject *self)

入力と出力

  • 入力: PLyPlanObject *self（破棄対象のPL/Python計画オブジェクト）
  • 出力: なし（void）

前提条件と不変条件

  • self は有効な PLyPlanObject 構造体へのポインタであること。
  • Python 3.8以降では、PyTypeObject の参照カウント管理のため、関数内で型オブジェクトの参照を一時的に保持する。

アルゴリズムまたは振る舞い

1. Python 3.8以上の場合、Py_TYPE(self) を取得し型オブジェクトへの参照を保持する。 2. self->plan が存在する場合、SPI_freeplan を呼び出してSPI実行計画を解放し、ポインタを NULL に設定する。 3. self->mcxt が存在する場合、MemoryContextDelete を呼び出して関連するメモリコンテキストを破棄し、ポインタを NULL に設定する。 4. PyObject_Free(self) を呼び出し、Pythonオブジェクト自体のメモリを解放する。 5. Python 3.8以上の場合、保持していた型オブジェクトの参照カウントを Py_DECREF で減らす。

呼び出し元と呼び出し先

  • 呼び出し元: Pythonのガベージコレクション機構（Py_tp_dealloc スロット経由）。
  • 呼び出し先: SPI_freeplan（PostgreSQL SPI）、MemoryContextDelete（PostgreSQLメモリ管理）、PyObject_Free（Python C API）。

関連データ構造

  • PLyPlanObject: 実行計画、SPI計画、メモリコンテキストを保持する構造体。

失敗モードと境界条件

  • self が NULL の場合の挙動は定義されていない。
  • self->plan や self->mcxt が既に解放済み（NULL）である場合、条件分岐により安全にスキップされる。

保守担当者向けメモ

  • 本関数はPythonのC APIにおける tp_dealloc スロットとして使用される。
  • Python 3.8以降の PyTypeObject 参照カウント管理に関する条件付きコンパイルが含まれており、将来的なPythonバージョンアップ時にもこのパターンを維持する必要がある。
