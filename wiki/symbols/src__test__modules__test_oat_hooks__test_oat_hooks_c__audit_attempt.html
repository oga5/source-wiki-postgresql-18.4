audit_attempt

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_oat_hooks/test_oat_hooks.c:257

目的

データベース操作の試行（attempt）を監査ログとして記録する。

シグネチャまたは形状

static void audit_attempt(const char *hook, char *action, char *objName)

入力と出力

  • **入力**:
  • hook: 監査対象のフック名を示す文字列。
  • action: 実行しようとしている操作内容を示す文字列。
  • objName: 操作対象のオブジェクト名。
  • **出力**: なし（void）。

前提条件と不変条件

  • emit_audit_message が利用可能であること。
  • 本関数は、PostgreSQLのオブジェクトアクセスフック等の介入ポイントから呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. 引数として受け取った hook、action、objName をそのまま emit_audit_message 関数へ渡す。 2. emit_audit_message 内で "attempting" というステータスを付与し、NOTICE レベルのメッセージとして出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • REGRESS_object_access_hook_str (278行目)
  • REGRESS_object_access_hook (326行目)
  • REGRESS_exec_check_perms (352行目)
  • REGRESS_utility_command (391行目)
  • **呼び出し先**:
  • emit_audit_message

関連データ構造

  • 特になし。

失敗モードと境界条件

  • objName に NULL が渡された場合、emit_audit_message の実装に依存する。
  • 並列ワーカープロセス内から呼び出された場合、emit_audit_message 内の IsParallelWorker() チェックによりログ出力が抑制される可能性がある。

保守担当者向けメモ

  • 本関数はテストモジュール専用であり、監査ログのフォーマットを統一するために使用される。変更時は emit_audit_message との整合性に注意すること。
