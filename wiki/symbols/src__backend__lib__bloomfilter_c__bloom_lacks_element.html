bloom_lacks_element

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bloomfilter.c:156

目的

指定された要素がブルームフィルタ内に存在しないことを判定します。偽陰性（要素が存在するのに存在しないと判定されること）は発生しません。

シグネチャまたは形状

bool bloom_lacks_element(bloom_filter *filter, unsigned char *elem, size_t len)

入力と出力

  • filter: 判定対象の bloom_filter 構造体へのポインタ。
  • elem: 判定対象のデータ要素。
  • len: elem のバイト長。
  • 戻り値: 要素がフィルタ内に存在しないと確信できる場合は true、存在する可能性がある場合は false を返します。

前提条件と不変条件

  • filter は bloom_create によって適切に初期化されている必要があります。
  • MAX_HASH_FUNCS（10）以下のハッシュ関数が使用されます。
  • ビットセットのサイズは 2 のべき乗である必要があります。

アルゴリズムまたは振る舞い

1. k_hashes を呼び出し、要素から k 個のハッシュ値を生成します。 2. 各ハッシュ値について、ビットセット上の対応するビット位置を計算します。 3. hashes[i] >> 3 でバイトオフセットを、hashes[i] & 7 でビットオフセットを特定します。 4. 該当するビットが 0 であれば、その要素はフィルタに追加されていないと判断し、即座に true を返します。 5. すべてのハッシュ位置のビットが 1 であれば、要素が存在する可能性があると判断し、false を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/amcheck/verify_nbtree.c
  • src/backend/utils/adt/acl.c
  • src/test/modules/test_bloomfilter/test_bloomfilter.c
  • 呼び出し先:
  • k_hashes (同ファイル内)

関連データ構造

  • bloom_filter: フィルタの状態（ビットセット、ハッシュ関数数、シード値）を保持する構造体。

失敗モードと境界条件

  • フィルタが空の場合、常に true を返します。
  • 偽陽性（False Positive）が発生する可能性があり、その場合、実際には要素が含まれていなくても false を返すことがあります。

保守担当者向けメモ

  • 本関数はビット演算による高速な判定を行っています。ビットセットのサイズが 2 のべき乗であることを前提としたマスク処理に依存しているため、構造体の変更時には注意してください。
