KeepFileRestoredFromArchive

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:357

目的

アーカイブからリストアされたファイルを pg_wal ディレクトリへ配置し、アーカイブ状態の更新および関連プロセスへの通知を行う。

シグネチャまたは形状

void KeepFileRestoredFromArchive(const char *path, const char *xlogfname)

入力と出力

  • path: リストアされた一時ファイルのフルパス。
  • xlogfname: pg_wal 内での対象ファイル名。
  • 出力: なし（成功時はファイルを移動し、失敗時は ereport によりエラーを発生させる）。

前提条件と不変条件

  • path で指定されたファイルは既にリストア済みであること。
  • pg_wal ディレクトリへの書き込み権限があること。
  • Windows 環境では、ファイルが他プロセス（walsender 等）によって開かれている可能性があるため、リネーム処理が競合を回避するように設計されている。

アルゴリズムまたは振る舞い

1. **既存ファイルの削除**: pg_wal 内に同名のファイルが存在する場合、それを削除する。

  • Windows では、他プロセスによるハンドル保持を考慮し、一時的な .deleted 拡張子へリネームしてから削除を行う。

    • 2. **ファイルの配置**: durable_rename を使用して、リストアされたファイルを pg_wal 内の目的のパスへ移動する。 3. **アーカイブ状態の更新**:

  • ARCHIVE_MODE_ALWAYS でない場合、XLogArchiveForceDone を呼び出し、アーカイブ済みとしてマークする。
  • それ以外の場合は XLogArchiveNotify を呼び出し、アーカイブ対象として通知する。

    • 4. **通知**:

  • ファイルが置換された場合、WalSndRqstFileReload を呼び出し、walsender にファイルのリロードを要求する。
  • WalSndWakeup を呼び出し、新しい WAL が到着したことを walsender に通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/timeline.c
  • src/backend/access/transam/xlogrecovery.c
  • **呼び出し先**:
  • stat, rename, unlink (システムコール)
  • durable_rename
  • XLogArchiveForceDone / XLogArchiveNotify
  • WalSndRqstFileReload
  • WalSndWakeup

関連データ構造

  • XLogArchiveMode: アーカイブモードのグローバル設定。
  • MAXPGPATH: パス名の最大長。

失敗モードと境界条件

  • **リネーム失敗**: pg_wal 内の既存ファイルのリネームや削除に失敗した場合、ERROR または FATAL で終了する。
  • **ファイル移動失敗**: durable_rename が失敗した場合、ERROR を発生させる。
  • **Windows 特有の競合**: 他プロセスがファイルを保持している場合、リネームが失敗する可能性があるが、.deleted 拡張子による一意な名前付けで回避を試みる。

保守担当者向けメモ

  • 本関数は WAL セグメント以外のファイル（タイムライン履歴ファイル等）に対しても呼び出される可能性があるため、WalSnd 関連の通知は汎用的に行われる。
  • Windows における .deleted ファイルの生成は、ファイルハンドルがクローズされるまでファイルシステム上に残る挙動を考慮したものである。
