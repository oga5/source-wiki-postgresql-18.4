unicode_normalize

定義

  • 種別: function
  • 定義箇所: src/common/unicode_norm.c:402

目的

入力された Unicode 文字列を指定された正規化形式（NFC, NFD, NFKC, NFKD）に変換する。

シグネチャまたは形状

pg_wchar *unicode_normalize(UnicodeNormalizationForm form, const pg_wchar *input)

入力と出力

  • **入力**:
  • form: 適用する正規化形式（UnicodeNormalizationForm 型）。
  • input: 正規化対象の pg_wchar 型のヌル終端文字列。
  • **出力**:
  • 正規化された新しい pg_wchar 文字列を返す。メモリ不足時や分解後のサイズが MaxAllocSize を超える場合は NULL を返す。

前提条件と不変条件

  • input は有効な pg_wchar 文字列であること。
  • 内部処理で ALLOC（環境により palloc または malloc）を使用する。
  • 呼び出し元は、返されたメモリを適切に解放する責任を持つ。

アルゴリズムまたは振る舞い

1. **分解フェーズ**: 入力文字列の各文字を get_decomposed_size で計算し、必要なバッファを確保後、decompose_code を用いて再帰的に分解する。 2. **正規順序適用**: 結合クラス（get_canonical_class）に基づき、隣接する非スターター文字を交換して正規順序に並べ替える。 3. **結合フェーズ**: recompose が必要な形式（NFC/NFKC）の場合、recompose_code を使用して分解された文字を可能な限り合成する。 4. **メモリ管理**: 処理中に確保した一時的な分解用バッファは、結合完了後に FREE される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/varlena.c
  • src/common/saslprep.c
  • src/common/unicode/norm_test.c
  • **呼び出し先**:
  • get_decomposed_size, decompose_code, get_canonical_class, recompose_code 等の内部ユーティリティ関数。

関連データ構造

  • UnicodeNormalizationForm: 正規化形式を定義する列挙型。
  • pg_wchar: PostgreSQL で使用されるワイド文字型。

失敗モードと境界条件

  • **メモリ不足**: ALLOC が失敗した場合、NULL を返す。
  • **サイズ超過**: 分解後の文字列長が MaxAllocSize を超える場合、フロントエンドでは NULL を返し、バックエンドではエラーを発生させる（palloc 経由）。
  • **分解不要**: 入力文字列が既に分解済み、あるいは分解対象がない場合はそのままの文字列（またはそのコピー）を返す。

保守担当者向けメモ

  • FRONTEND マクロによりメモリ割り当て戦略が切り替わるため、修正時は両環境での動作を確認すること。
  • クイックチェック（unicode_is_normalized_quickcheck）は本関数とは別に存在し、本関数は常に正規化処理を実行する。
  • ハングル文字はテーブルを使用せずアルゴリズム的に処理されるため、ハングル関連の修正時は recompose_code 等のロジックを確認すること。
