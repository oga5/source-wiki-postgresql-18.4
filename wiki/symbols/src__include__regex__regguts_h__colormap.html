colormap

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regguts.h:236

目的

正規表現エンジンにおいて、個々の文字コードを内部的な抽象単位である「色（color）」に変換するためのマッピングテーブルを保持する構造体です。

シグネチャまたは形状

struct colormap
{
    int magic;
    struct vars *v;
    size_t ncds;
    size_t max;
    color free;
    struct colordesc *cd;
    color *locolormap;
    int classbits[NUM_CCLASSES];
    int numcmranges;
    colormaprange *cmranges;
    color *hicolormap;
    int maxarrayrows;
    int hiarrayrows;
    int hiarraycols;
    struct colordesc cdspace[NINLINECDS];
};

入力と出力

  • **入力**: 正規表現のコンパイルおよび実行時に、文字コード（chr）を引数として受け取り、対応する color を返します。
  • **出力**: locolormap（低位文字用）または hicolormap（高位文字用）を参照し、文字クラスを識別する color 値を出力します。

前提条件と不変条件

  • magic フィールドは CMMAGIC (0x876) である必要があります。
  • WHITE (0) はデフォルトの色として予約されています。
  • cdspace はメモリ確保を節約するために、NINLINECDS（10個）までの colordesc をインラインで保持します。

アルゴリズムまたは振る舞い

  • **マッピング**: MAX_SIMPLE_CHR 以下の文字は locolormap 配列で直接インデックス参照されます。それより大きい文字は、cmranges（範囲）および hicolormap（2次元配列）を用いて動的に解決されます。
  • **メモリ管理**: cd ポインタは通常 cdspace を指しますが、必要に応じて動的に割り当てられた領域を指すよう切り替わります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_color.c 内の各関数（initcm, freecm, pg_reg_getcolor, maxcolor, newcolor, freecolor, pseudocolor, subcolor, subcolorhi）から参照・操作されます。
  • **呼び出し先**: 構造体内部で struct colordesc および colormaprange を保持・参照します。

関連データ構造

  • struct colordesc: 各色の詳細情報を保持する構造体。
  • struct colormaprange: 高位文字の範囲マッピングを定義する構造体。
  • struct vars: コンパイルエラー報告のために保持されるコンテキスト。

失敗モードと境界条件

  • max フィールドは現在使用中の最大色番号を追跡しており、これを超えるアクセスは境界外参照となる可能性があります。
  • hicolormap は2次元配列として管理されるため、hiarrayrows および hiarraycols の範囲内でのアクセスが必須です。

保守担当者向けメモ

  • NINLINECDS を超える数の色が必要になった場合、cd ポインタが動的メモリを指すようになるため、メモリリークを防ぐための適切な解放処理（freecm）が重要です。
  • classbits はロケール依存の文字クラス判定に使用されるため、ロケール変更時の整合性に注意してください。
