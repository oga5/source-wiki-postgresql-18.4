tablespaceinfo

定義

  • 種別: struct
  • 定義箇所: src/include/backup/basebackup.h:28

目的

バックアップ処理において、対象となるテーブルスペース（または PGDATA）のメタデータおよび物理的な配置情報を管理するために使用されます。

シグネチャまたは形状

typedef struct
{
	Oid			oid;			/* テーブルスペースのOID */
	char	   *path;			/* テーブルスペースディレクトリへのフルパス */
	char	   *rpath;			/* PGDATA内の相対パス。PGDATA自身の場合はNULL */
	int64		size;			/* 送信時の合計サイズ。不明な場合は-1 */
} tablespaceinfo;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。バックアップ処理の過程で、各テーブルスペースの情報を格納するコンテナとして機能します。

前提条件と不変条件

  • path が NULL の場合、慣習として PGDATA ディレクトリ自身を指しているものとして扱われます。
  • size が -1 の場合、サイズが未確定であることを示します。

アルゴリズムまたは振る舞い

バックアップ実行時、この構造体はバックアップ対象のディレクトリ構造を特定するために利用されます。basebackup.c 等のロジックにおいて、リスト形式で保持され、各テーブルスペースの物理的な場所を追跡するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/backup/basebackup.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • BaseBackupCmd: nodes/replnodes.h で定義されるバックアップ要求構造体。
  • bbsink: バックアップデータの出力先を抽象化するインターフェース。

失敗モードと境界条件

  • path や rpath のメモリ管理は呼び出し側（palloc 等）に依存します。
  • size が -1 の状態でサイズ計算を試みるロジックがある場合、適切な初期化が行われていないと誤った値が参照される可能性があります。

保守担当者向けメモ

  • path が NULL の場合の挙動（PGDATA を指す）はコード全体の慣習であるため、この構造体を利用するロジックを追加する際は必ずこの条件を考慮してください。
  • 構造体のフィールド追加や変更を行う際は、xlog.c および xlogrecovery.c でのメモリ確保（palloc）箇所との整合性に注意してください。
