r_remove_plural_suffix

定義

  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_tamil.c:958

目的

タミル語の単語末尾から複数形の接尾辞を除去し、語幹を正規化する。

シグネチャまたは形状

static int r_remove_plural_suffix(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体。処理対象の文字列を保持）
  • 出力: int（成功時に 1、失敗またはマッチしない場合に 0、エラー時に負の値を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 処理対象の文字列は UTF-8 エンコーディングである。
  • 関数実行前後で z->lb および z->c のポインタ操作により、文字列の走査範囲が一時的に変更されるが、終了時には z->c が z->lb にリセットされる。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置を保存し、文字列の末尾から前方へ向かって走査を開始する。 2. 文字列の長さが十分か（z->c - 8 > z->lb）および末尾のバイトが 141 であるかを確認する。 3. find_among_b を使用して、定義済みの接尾辞パターン配列 a_13 と照合する。 4. マッチした場合、among_var の値に応じて以下のいずれかの処理を行う：

  • case 1: a_12 配列との照合を行い、マッチすれば s_17 に置換、マッチしなければ s_18 に置換する。
  • case 2: s_19 に置換する。
  • case 3: s_20 に置換する。
  • case 4: 該当部分を削除する。

    • 5. 処理完了後、カーソル位置を初期位置に戻し、成功を示す 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_tamil.c 内のメイン処理フロー（行1393付近）。
  • 呼び出し先: find_among_b（パターン照合）、slice_from_s（文字列置換）、slice_del（文字列削除）。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。
  • a_13, a_12: 接尾辞パターンを保持する among 構造体の配列。

失敗モードと境界条件

  • 文字列が短すぎる場合や、末尾が特定のバイトパターンに一致しない場合は 0 を返して終了する。
  • slice_from_s または slice_del が内部エラー（負の値）を返した場合、その値をそのまま呼び出し元へ伝播させる。

保守担当者向けメモ
