entryGetLeftMostPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:445

目的

GINエントリーツリーの内部ノードページから、最も左側の子ページ（ダウンリンク）のブロック番号を取得します。

シグネチャまたは形状

static BlockNumber entryGetLeftMostPage(GinBtree btree, Page page)

入力と出力

  • **入力**:
  • btree: GIN B-tree操作用のコンテキスト構造体。
  • page: 調査対象のページ（Page型）。
  • **出力**:
  • ページ内の最初のアイテムから抽出された BlockNumber（子ページへのポインタ）。

前提条件と不変条件

  • 渡される page はリーフページであってはならない（!GinPageIsLeaf(page)）。
  • 渡される page はデータページであってはならない（!GinPageIsData(page)）。
  • ページには少なくとも1つのアイテムが存在する必要がある（PageGetMaxOffsetNumber(page) >= FirstOffsetNumber）。

アルゴリズムまたは振る舞い

1. ページ内の最初のアイテム（FirstOffsetNumber）を PageGetItem を用いて取得します。 2. 取得した IndexTuple から GinGetDownlink マクロを使用して、そのタプルが指し示す子ページのブロック番号を抽出して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginentrypage.c 内で、btree->getLeftMostChild コールバックとして設定されています。
  • **呼び出し先**: PageGetItem、PageGetItemId、GinGetDownlink などのPostgreSQLのページ操作マクロを呼び出します。

関連データ構造

  • GinBtree: GINインデックスのB-tree操作を抽象化する構造体。
  • IndexTuple: GINエントリーツリーのノードに格納されるインデックスタプル。

失敗モードと境界条件

  • ページが空の場合、Assert によりプログラムが停止します。
  • ページがリーフページまたはデータページである場合、Assert によりプログラムが停止します。

保守担当者向けメモ

  • 本関数はエントリーツリーの構造に特化しており、B-treeの走査において左端のパスを辿る際に使用されます。
  • GinGetDownlink は IndexTuple の末尾付近に格納されたブロック番号を読み取るため、タプルの形式が変更された場合は注意が必要です。
