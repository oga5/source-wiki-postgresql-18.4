Vsrt

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_utils_var.c:16

目的

GiSTインデックスの分割処理（picksplit）において、可変長キーとそれに関連するインデックス項目をソートおよび管理するための補助構造体です。

シグネチャまたは形状

typedef struct
{
	int			i;          /* インデックスまたはオフセット */
	GBT_VARKEY *t;          /* 可変長キーへのポインタ */
} Vsrt;

入力と出力

本構造体は関数引数として直接渡されるものではなく、gbt_var_picksplit 関数内でメモリ確保され、ソート対象の配列として利用されます。

前提条件と不変条件

  • t メンバは有効な GBT_VARKEY を指している必要があります。
  • palloc を用いて (maxoff + 1) 個分の領域が確保されることが前提となります。

アルゴリズムまたは振る舞い

GiSTインデックスの分割時に、エントリをソートしてバランスを維持するために使用されます。具体的には、gbt_var_picksplit 内で qsort_arg を用いたソート処理の対象となり、各エントリの比較において t メンバが参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_utils_var.c 内の gbt_var_picksplit 関数。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、メンバの t は gbt_var_key_readable 等のユーティリティ関数に渡されます。

関連データ構造

  • GBT_VARKEY: 可変長キーの内部表現。
  • GBT_VARKEY_R: gbt_var_key_readable を通じて取得される、読み取り可能なキー構造。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗した場合、PostgreSQL のエラーハンドリングによりアボートされます。
  • maxoff が 0 の場合: 確保される配列サイズが 0 になる可能性があるため、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は gbt_var_picksplit の内部的なソート処理専用です。
  • Vsrt 配列の要素数は maxoff + 1 であり、インデックス操作時の境界値に注意してください。
