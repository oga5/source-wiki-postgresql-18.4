build_attrmap_by_position

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/attmap.c:74

目的

2つの TupleDesc（indesc と outdesc）間で、列の物理的な位置に基づいて属性番号の対応表（AttrMap）を生成します。

シグネチャまたは形状

AttrMap *build_attrmap_by_position(TupleDesc indesc, TupleDesc outdesc, const char *msg);

入力と出力

  • **入力**:
  • indesc: 入力側のタプル記述子。
  • outdesc: 出力側のタプル記述子。
  • msg: エラー発生時に報告されるメッセージ文字列。
  • **出力**:
  • 変換が必要な場合は AttrMap* を返します。
  • 変換が不要（1対1の対応）と判定された場合は NULL を返します。

前提条件と不変条件

  • outdesc の各列に対し、indesc 内の対応する非削除列が型（atttypid および atttypmod）において一致している必要があります。
  • attnums 配列のインデックスは出力側の列位置に対応し、値は入力側の列番号（1始まり）です。値が 0 の場合は削除済み列を指します。

アルゴリズムまたは振る舞い

1. outdesc の列数に基づいて AttrMap を確保します。 2. outdesc を走査し、削除済み列をスキップしながら indesc の非削除列と順次照合します。 3. 型の不一致が検出された場合、または列数に過不足がある場合は ERROR を発生させます。 4. 全ての列の照合完了後、check_attrmap_match を呼び出し、マッピングが不要な場合はメモリを解放して NULL を返します。 5. 変換が必要な場合のみ、構築した AttrMap を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tupconvert.c:68
  • **呼び出し先**:
  • make_attrmap (メモリ確保)
  • TupleDescAttr / TupleDescCompactAttr (属性アクセス)
  • ereport (エラー報告)
  • check_attrmap_match (最適化判定)
  • free_attrmap (メモリ解放)

関連データ構造

  • AttrMap: 属性番号の対応関係を保持する構造体。
  • TupleDesc: タプルの構造を定義する記述子。
  • Form_pg_attribute: 個々の属性の定義情報。

失敗モードと境界条件

  • **型不一致**: atttypid が異なる、または atttypmod が一致しない場合に ERRCODE_DATATYPE_MISMATCH でエラーとなります。
  • **列数不一致**: 入力側と出力側で非削除列の数が一致しない場合にエラーとなります。

保守担当者向けメモ

  • 本関数は変換の最適化を考慮しており、check_attrmap_match が真を返す場合は NULL を返すことで、呼び出し側での不要な変換処理を回避します。
  • msg 引数はエラー発生時のコンテキストを特定するために使用されます。
