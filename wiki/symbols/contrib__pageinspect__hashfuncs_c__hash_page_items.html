hash_page_items

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/hashfuncs.c:299

目的

ハッシュインデックスのバケットページまたはオーバーフローページ内の全アイテム（タプル）を列挙し、オフセット番号、TID、およびハッシュキーを抽出して返す SRF（Set Returning Function）です。

シグネチャまたは形状

Datum hash_page_items(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型のページデータ（PG_GETARG_BYTEA_P(0)）。
  • **出力**: 以下のカラムを持つ行セットを返します。
  • itemoffset (int4): アイテムのオフセット番号
  • ctid (tid): タプルの物理位置
  • hashkey (int8): インデックスのハッシュキー値

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要です（superuser() チェック）。
  • 入力ページは verify_hash_page により、ハッシュインデックスのバケットページまたはオーバーフローページであることが検証されます。
  • 戻り値の型はコンポジット型である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: SRF_IS_FIRSTCALL() 内でメモリコンテキストを切り替え、ページ検証を行い、user_args 構造体にページポインタと初期オフセットを格納します。 2. **反復処理**: SRF_PERCALL_SETUP() を使用して、ページ内の最大オフセット数までループします。 3. **アイテム抽出**: PageGetItemId と PageGetItem を用いてインデックスアイテムを取得し、_hash_get_indextuple_hashkey でハッシュキーを計算します。 4. **行構築**: heap_form_tuple を用いて結果を行形式に変換し、SRF_RETURN_NEXT で返却します。 5. **終了**: 全アイテムの走査完了後、SRF_RETURN_DONE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（例: SELECT * FROM hash_page_items(...)）。
  • **呼び出し先**:
  • verify_hash_page（ページ検証）
  • _hash_get_indextuple_hashkey（ハッシュキー抽出）
  • SRF 関連マクロ（SRF_FIRSTCALL_INIT, SRF_RETURN_NEXT 等）

関連データ構造

  • user_args: 走査中のページ状態と現在のオフセットを保持する構造体。
  • FuncCallContext: SRF の実行状態を管理する PostgreSQL 標準の構造体。
  • IndexTuple: ハッシュインデックス内のアイテム構造。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了します。
  • **ページ検証エラー**: verify_hash_page が不正なページタイプや破損を検知した場合、エラーを投げます。
  • **型不一致**: 戻り値がコンポジット型でない場合、エラーを投げます。
  • **アイテム不正**: ItemIdIsValid が偽の場合、エラーを投げます。

保守担当者向けメモ

  • 本関数は contrib/pageinspect の他の関数と同様、物理ページを直接操作するため、インデックスの破損時には予期せぬ動作をする可能性があります。
  • user_args 構造体は本ファイル内で定義されており、SRF の user_fctx を通じて状態を維持しています。
