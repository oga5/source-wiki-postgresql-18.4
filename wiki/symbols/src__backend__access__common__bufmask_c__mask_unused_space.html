mask_unused_space

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/bufmask.c:70

目的

WAL 整合性チェックにおいて、ページ内の未使用領域（pd_lower から pd_upper の間）を MASK_MARKER で埋めることで、比較時にゴミデータによる誤検知を防ぐ。

シグネチャまたは形状

void mask_unused_space(Page page)

入力と出力

  • **入力**: Page page（操作対象のページポインタ）
  • **出力**: なし（引数 page をインプレースで書き換える）

前提条件と不変条件

  • page は有効な PageHeader を保持している必要がある。
  • 以下の整合性条件を満たさない場合、elog(ERROR) が発生する:
  • pd_lower <= pd_upper
  • pd_special >= pd_upper
  • pd_lower >= SizeOfPageHeaderData
  • pd_special <= BLCKSZ

アルゴリズムまたは振る舞い

1. ページヘッダーから pd_lower、pd_upper、pd_special を取得する。 2. ページ構造の整合性を検証する。 3. memset を使用し、page + pd_lower から pd_upper - pd_lower バイト分を MASK_MARKER（通常は 0）で上書きする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_xlog.c
  • src/backend/access/gin/ginxlog.c
  • src/backend/access/gist/gistxlog.c
  • src/backend/access/hash/hash_xlog.c
  • src/backend/access/heap/heapam_xlog.c
  • src/backend/access/nbtree/nbtxlog.c
  • src/backend/access/spgist/spgxlog.c
  • src/backend/access/transam/generic_xlog.c
  • src/backend/commands/sequence.c
  • **呼び出し先**: memset（標準ライブラリ）、elog（PostgreSQL エラー報告）

関連データ構造

  • Page: ページ全体を指すポインタ型。
  • PageHeader: ページヘッダー構造体。pd_lower、pd_upper、pd_special フィールドを参照。

失敗モードと境界条件

  • ページ構造が破損している場合（前提条件の不変条件違反）、ERROR レベルでログを出力し処理を中断する。
  • pd_lower と pd_upper が等しい場合、memset のサイズが 0 となり、実質的な操作は行われない。

保守担当者向けメモ

  • 本関数はテスト目的の正規化ツールであり、実稼働時のページ更新には使用しないこと。
  • MASK_MARKER は access/bufmask.h で定義されており、比較対象間で差異を無視したい領域を統一するために使用される。
