flush_ps_display

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/ps_status.c:487

目的

現在のプロセス状態（ps_buffer）をOSのプロセス管理機能へ反映させ、外部ツール（psコマンドやProcess Explorer等）からバックエンドの稼働状況を可視化可能にする。

シグネチャまたは形状

static void flush_ps_display(void)

入力と出力

  • 入力: グローバル変数 ps_buffer（現在の表示文字列）、ps_buffer_cur_len、last_status_len、MyProcPid を参照。
  • 出力: なし（副作用としてOSのプロセス表示領域または名前付きオブジェクトを更新）。

前提条件と不変条件

  • ps_buffer には有効なプロセス状態文字列が格納されていること。
  • プラットフォームごとに定義された PS_USE_... マクロのいずれかが有効であること。

アルゴリズムまたは振る舞い

プラットフォームに応じて以下のいずれかの処理を実行する： 1. **PS_USE_SETPROCTITLE / PS_USE_SETPROCTITLE_FAST**: setproctitle APIを呼び出し、プロセス名を更新する。 2. **PS_USE_CLOBBER_ARGV**: argv 領域を直接書き換える。前回の文字列長（last_status_len）が現在の長さより長い場合、末尾を PS_PADDING で埋めて古い文字列の残骸を消去する。その後、last_status_len を更新する。 3. **PS_USE_WIN32**: pgident(PID): 文字列 という名前のイベントオブジェクトを作成する。既存のハンドルがある場合は CloseHandle で解放してから再作成する。

呼び出し元と呼び出し先

  • 呼び出し元: 同一ファイル内の set_ps_display_with_len および set_ps_display_suffix から呼び出される。
  • 呼び出し先: OS固有のAPI（setproctitle, setproctitle_fast, CloseHandle, CreateEvent）およびメモリ操作関数（MemSet）。

関連データ構造

  • ps_buffer: 表示文字列を保持するバッファ。
  • ps_buffer_cur_len: 現在の文字列長。
  • last_status_len: 前回の文字列長（PS_USE_CLOBBER_ARGV 使用時）。
  • MyProcPid: 現在のプロセスID（Windows用）。

失敗モードと境界条件

  • PS_USE_WIN32 環境下で CreateEvent が失敗した場合の戻り値チェックは含まれていない。
  • argv を直接書き換える環境では、確保されたメモリ領域を超えて書き込まないよう ps_buffer のサイズ管理に依存している。

保守担当者向けメモ

  • 本関数はOS依存の条件分岐が多いため、プラットフォーム固有の挙動を修正する際は #ifdef ブロック内の実装を確認すること。
  • PS_USE_CLOBBER_ARGV を使用する環境では、last_status_len の管理が不整合を起こすとプロセス名表示にゴミが残る可能性があるため注意が必要。
