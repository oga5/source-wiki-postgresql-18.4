IndexScanInstrumentation

定義

  • 種別: struct
  • 定義箇所: src/include/access/genam.h:35

目的

インデックススキャン実行時の統計情報を保持するための構造体です。主にインデックスの検索回数を記録するために使用されます。

シグネチャまたは形状

typedef struct IndexScanInstrumentation
{
    uint64      nsearches;
} IndexScanInstrumentation;

入力と出力

  • **入力**: pgstat_count_index_scan 関数呼び出しを通じて nsearches がインクリメントされます。
  • **出力**: 実行計画の統計情報として EXPLAIN 出力や、並列クエリ実行時のワーカー間での情報集約に使用されます。

前提条件と不変条件

  • 並列スキャン時に共有メモリへコピーされるため、構造体内にポインタを含めてはなりません。

アルゴリズムまたは振る舞い

インデックススキャンが実行されるたびに、対応する nsearches カウンタが加算されます。並列クエリ実行時には、各ワーカーが保持する本構造体のインスタンスが共有メモリ上に確保され、最終的に集約されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/index/indexam.c
  • src/backend/commands/explain.c
  • src/backend/executor/nodeBitmapIndexscan.c
  • src/backend/executor/nodeIndexonlyscan.c
  • **呼び出し先**:
  • pgstat_count_index_scan（インクリメント処理）

関連データ構造

  • ParallelIndexScanDesc: 並列スキャン時に本構造体を含む情報を管理します。
  • SharedIndexScanInstrumentation: 並列実行時に共有される統計情報構造体。

失敗モードと境界条件

  • カウンタ nsearches は uint64 型であるため、極めて長期間の稼働においてもオーバーフローのリスクは極めて低いです。

保守担当者向けメモ

  • 本構造体は並列処理の制約を受けるため、フィールドを追加する際はポインタ型を含めないよう注意してください。
  • 統計情報の正確性は pgstat_count_index_scan が適切なタイミングで呼び出されているかに依存します。
