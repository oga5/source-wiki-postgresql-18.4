r_mark_nUn

定義

  • **種別**: function
  • **定義箇所**: src/backend/snowball/libstemmer/stem_UTF_8_turkish.c:701

目的

トルコ語のステミング処理において、特定の接尾辞（"-nUn" 形式）を識別し、必要に応じて削除またはマークを行う。

シグネチャまたは形状

static int r_mark_nUn(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （Snowball ステミング環境へのポインタ）
  • **出力**: int 型。成功時は 1、失敗または条件不一致時は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 内部で r_check_vowel_harmony を呼び出すため、母音調和のルールが適用される。

アルゴリズムまたは振る舞い

1. r_check_vowel_harmony(z) を実行し、母音調和の検証を行う。失敗した場合はその結果を返す。 2. カーソル位置 z->c がバッファの境界内にあるか確認し、直前の文字が 'n' (ASCII 110) であるかをチェックする。 3. a_3 配列（4要素）を使用して、後方検索 find_among_b を実行し、対象の接尾辞パターンを探索する。 4. マッチした場合、r_mark_suffix_with_optional_n_consonant(z) を呼び出し、接尾辞のマーク処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/snowball/libstemmer/stem_UTF_8_turkish.c:1237
  • src/backend/snowball/libstemmer/stem_UTF_8_turkish.c:1659
  • **呼び出し先**:
  • r_check_vowel_harmony
  • find_among_b
  • r_mark_suffix_with_optional_n_consonant

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_3: 接尾辞パターンマッチング用 among 構造体配列。

失敗モードと境界条件

  • r_check_vowel_harmony が 0 以下を返した場合、即座に終了する。
  • カーソル位置がバッファの先頭（z->lb）に達している場合、または直前の文字が 'n' でない場合は 0 を返す。
  • find_among_b でパターンが見つからない場合は 0 を返す。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。直接編集せず、元の .sbl ファイルを修正すること。
  • z->c や z->p への直接アクセスは Snowball ランタイムの仕様に従うこと。
