descriptor_destructor

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/descriptor.c:27

目的

スレッド終了時に、当該スレッドに関連付けられたすべての SQL 記述子を解放し、メモリリークを防止する。

シグネチャまたは形状

static void descriptor_destructor(void *arg)

入力と出力

  • **入力**: void *arg - スレッドローカルストレージに保持されている記述子リストへのポインタ。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • 本関数は、スレッド終了時に pthread のキー破棄メカニズムを通じて呼び出されることを前提としている。
  • arg は descriptor_deallocate_all が処理可能な有効な記述子リストのポインタである必要がある。

アルゴリズムまたは振る舞い

1. 受け取った引数 arg を descriptor_deallocate_all 関数に渡す。 2. descriptor_deallocate_all が、当該スレッドに関連付けられた全記述子のメモリ解放を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: スレッド終了時に pthread ライブラリ（pthread_key_create で登録されたデストラクタとして呼び出される）。
  • **呼び出し先**: descriptor_deallocate_all。

関連データ構造

  • descriptor: スレッドローカルストレージで管理される SQL 記述子構造体。
  • descriptor_key: スレッドごとの記述子リストを保持するために使用される pthread_key_t。

失敗モードと境界条件

  • arg が NULL の場合、descriptor_deallocate_all の内部実装に依存する（通常は安全に無視される）。

保守担当者向けメモ

  • 本関数は pthread_key_create の引数として登録されており、スレッド終了時のクリーンアップを担う重要な役割を持つ。
  • 記述子の管理ロジックに変更を加える際は、descriptor_deallocate_all の動作と整合性が取れていることを確認すること。
