k_hashes

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bloomfilter.c:249

目的

ブルームフィルタにおいて、指定された要素に対して k 個のハッシュ値を生成し、ビットセット上のインデックスを算出する。

シグネチャまたは形状

static void k_hashes(bloom_filter *filter, uint32 *hashes, unsigned char *elem, size_t len)

入力と出力

  • **入力**:
  • filter: フィルタの状態を保持する bloom_filter 構造体へのポインタ。
  • elem: ハッシュ化対象のデータ。
  • len: elem のバイト長。
  • **出力**:
  • hashes: 生成された k 個のハッシュ値を格納する uint32 配列。

前提条件と不変条件

  • filter->seed を用いてハッシュ計算が行われる。
  • filter->m はビットセットのサイズ（2のべき乗）であり、mod_m を通じてハッシュ値がこの範囲内に収まることが保証される。
  • hashes 配列は、少なくとも filter->k_hash_funcs 分の要素を保持できるサイズである必要がある。

アルゴリズムまたは振る舞い

1. hash_any_extended を使用して 64 ビットのハッシュ値を生成し、それを上位 32 ビットと下位 32 ビットの 2 つの独立したハッシュ値（x, y）に分割する。 2. mod_m を適用して初期のハッシュ値を正規化する。 3. 拡張ダブルハッシュ法を用いて、k 個のハッシュ値を生成する。

  • 1 番目のハッシュ値は x。
  • 2 番目以降は x = mod_m(x + y, m) および y = mod_m(y + i, m) の漸化式により算出する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • bloom_add_element (src/backend/lib/bloomfilter.c:140)
  • bloom_lacks_element (src/backend/lib/bloomfilter.c:162)
  • **呼び出し先**:
  • hash_any_extended (外部関数)
  • mod_m (内部関数)

関連データ構造

  • bloom_filter: フィルタのシード値、ハッシュ関数数 k、ビットセットサイズ m を保持する構造体。

失敗モードと境界条件

  • filter->k_hash_funcs が 0 の場合、ループは実行されず hashes 配列は更新されない。
  • len が 0 の場合、hash_any_extended の挙動に依存する。

保守担当者向けメモ

  • 本関数は拡張ダブルハッシュ法を採用しており、ビットセットサイズが 2 のべき乗であっても衝突を最小限に抑える設計となっている。
  • hashes 配列のメモリ確保は呼び出し元で行われるため、バッファオーバーランを防ぐために filter->k_hash_funcs の値が適切であることを確認すること。
