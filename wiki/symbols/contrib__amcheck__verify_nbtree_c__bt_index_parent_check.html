bt_index_parent_check

定義

  • 種別: function
  • 定義箇所: contrib/amcheck/verify_nbtree.c:283

目的

B-Tree インデックスの論理的な整合性を検証するための SQL 呼び出し可能なエントリポイントです。特に、インデックスの親ページと子ページの包含関係（ダウンリンクの整合性）の検証を有効にして実行します。

シグネチャまたは形状

Datum bt_index_parent_check(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • indrelid (OID): 検証対象のインデックスの OID。
  • heapallindexed (boolean, オプション): ヒープとインデックスの完全同期を検証するかどうか。
  • rootdescend (boolean, オプション): ルートからの降下検証を行うかどうか。
  • checkunique (boolean, オプション): 一意性制約の重複チェックを行うかどうか。
  • **出力**: void (検証成功時)。エラーが発生した場合は例外がスローされます。

前提条件と不変条件

  • parentcheck フラグが true に設定され、親子の整合性チェックが強制されます。
  • ShareLock を使用して対象インデックスをロックし、検証中の構造変化を防ぎます。

アルゴリズムまたは振る舞い

1. BTCallbackState 構造体を初期化し、parentcheck を true に設定します。 2. SQL 引数から heapallindexed、rootdescend、checkunique の各フラグを読み取り、状態に反映します。 3. amcheck_lock_relation_and_check を呼び出し、bt_index_check_callback をコールバックとして指定してインデックスの検証を開始します。 4. 検証完了後、PG_RETURN_VOID() を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQL インターフェース経由で呼び出されます（pg_amcheck 等のツールから参照）。
  • **呼び出し先**:
  • amcheck_lock_relation_and_check (共通ユーティリティ)
  • bt_index_check_callback (検証ロジックの実行)

関連データ構造

  • BTCallbackState: 検証の動作オプションを保持する構造体。

失敗モードと境界条件

  • インデックスの破損が検出された場合、エラーがスローされます。
  • ShareLock が取得できない場合、検証は失敗します。
  • 引数の数が不足している場合、デフォルト値（false）が使用されます。

保守担当者向けメモ

  • 本関数は bt_index_check と類似していますが、親子の整合性チェックを明示的に有効化する点が異なります。
  • 詳細なエラーが発生した際、ユーザーに対して本関数での再試行を促すヒントが提示されることがあります。
