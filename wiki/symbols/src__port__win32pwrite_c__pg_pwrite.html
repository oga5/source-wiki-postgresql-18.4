pg_pwrite

定義

  • 種別: function
  • 定義箇所: src/port/win32pwrite.c:19

目的

Windows環境において、POSIX標準の pwrite(2) システムコールをエミュレートし、指定されたオフセット位置へアトミックにデータを書き込む。

シグネチャまたは形状

ssize_t pg_pwrite(int fd, const void *buf, size_t size, off_t offset)

入力と出力

  • **入力**:
  • fd: 書き込み対象のファイル記述子。
  • buf: 書き込むデータへのポインタ。
  • size: 書き込むバイト数（最大1GBに制限）。
  • offset: 書き込みを開始するファイル内のオフセット位置。
  • **出力**:
  • 成功時: 書き込まれたバイト数を返す。
  • 失敗時: -1 を返し、errno を設定する。

前提条件と不変条件

  • fd は有効なファイル記述子である必要がある。
  • 書き込みサイズは内部で Min(size, 1024 * 1024 * 1024) により最大1GBに制限される。
  • OVERLAPPED 構造体を使用するが、非同期I/Oではなく同期的な書き込みとして動作する。

アルゴリズムまたは振る舞い

1. _get_osfhandle(fd) を呼び出し、Cランタイムのファイル記述子からWindowsのネイティブハンドルを取得する。 2. ハンドルが無効な場合は errno を EBADF に設定し -1 を返す。 3. OVERLAPPED 構造体の Offset メンバに指定された offset を設定する。 4. WriteFile APIを呼び出し、データを書き込む。 5. WriteFile が失敗した場合は _dosmaperr を使用してWindowsエラーコードをPOSIXの errno に変換し、-1 を返す。 6. 成功時は書き込まれたバイト数を返す。

関連データ構造

  • OVERLAPPED: Windows APIの非同期I/O構造体。本関数ではオフセット指定のために利用。
  • HANDLE: Windowsのファイルハンドル。

失敗モードと境界条件

  • fd が無効な場合: errno が EBADF に設定される。
  • WriteFile が失敗した場合: GetLastError() を通じて取得したエラーコードが _dosmaperr によりPOSIX互換の errno に変換される。
  • 書き込みサイズが1GBを超える場合: 1GBに切り詰められる。

保守担当者向けメモ

  • 本関数は OVERLAPPED 構造体を使用しているが、ファイルポインタを移動させる副作用がある点に注意すること。
  • Windows環境専用の実装であり、他のプラットフォームには影響しない。
