SPLITCOST

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int_gist.c:422

目的

GiST インデックスのページ分割処理（picksplit）において、各エントリを分割した際のコストを保持し、最適な分割点を選択するための計算用データ構造です。

シグネチャまたは形状

typedef struct
{
	OffsetNumber pos;
	float		cost;
} SPLITCOST;

入力と出力

  • pos: 分割対象となるエントリのオフセット番号（OffsetNumber）。
  • cost: そのエントリを分割した際に発生するコスト（float）。

前提条件と不変条件

  • g_int_picksplit 関数内で、インデックスページ内の全エントリに対してこの構造体の配列が確保されます。
  • qsort を用いて cost に基づくソートが行われるため、比較関数 comparecost がこの構造体に対して定義されています。

アルゴリズムまたは振る舞い

1. g_int_picksplit 内で palloc を使用して SPLITCOST の配列（costvector）が確保されます。 2. 各エントリの分割コストが計算され、pos と cost が格納されます。 3. qsort と comparecost を使用して、コストの昇順または降順で並び替えられ、最も効率的な分割位置が決定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_int_gist.c 内の g_int_picksplit 関数。
  • **呼び出し先**: qsort（標準ライブラリ）および comparecost（同ファイル内の比較関数）。
  • **その他**: contrib/hstore/hstore_gist.c でも同名の構造体が定義・利用されています。

関連データ構造

  • OffsetNumber: PostgreSQL のインデックスページ内のエントリ位置を示す型。
  • comparecost: SPLITCOST 配列をソートするための比較関数。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗した場合、PostgreSQL のエラーハンドリングにより処理が中断されます。
  • maxoff が 0 の場合: palloc のサイズ計算で境界条件が発生する可能性があります。

保守担当者向けメモ

  • contrib/hstore/hstore_gist.c にも同一の定義が存在します。修正を行う際は、両方のファイルで整合性が保たれているか確認してください。
  • cost の計算ロジックは g_int_picksplit 内に依存しているため、コスト評価基準を変更する場合は g_int_picksplit の実装を確認してください。
