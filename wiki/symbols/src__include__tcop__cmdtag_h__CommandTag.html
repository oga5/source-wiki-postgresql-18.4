CommandTag

定義

  • 種別: enum
  • 定義箇所: src/include/tcop/cmdtag.h:22

目的

PostgreSQLにおいて実行されたSQLコマンドの種類を識別するための列挙型です。クライアント・サーバー間のプロトコルにおけるコマンド完了通知（CommandComplete）の基盤として機能します。

シグネチャまたは形状

typedef enum CommandTag
{
#include "tcop/cmdtaglist.h"
} CommandTag;

入力と出力

  • **入力**: tcop/cmdtaglist.h 内の PG_CMDTAG マクロ定義に基づき、全SQLコマンドが列挙子として展開されます。
  • **出力**: SQLコマンドを識別する列挙値。

前提条件と不変条件

  • 新しいSQLコマンドを追加する場合は、tcop/cmdtaglist.h を編集する必要があります。
  • 列挙値は cmdtag.c 内の属性テーブル（tag_behavior）と整合している必要があります。

アルゴリズムまたは振る舞い

  • cmdtaglist.h をインクルードすることで、SQLコマンドの網羅的なリストを生成します。
  • 実行されたコマンドの識別子として使用され、QueryCompletion 構造体を通じてクライアントへ送信される文字列の生成に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/commands/event_trigger.c: イベントトリガーのフィルタリングやタグの変換に使用。
  • src/backend/tcop/cmdtag.c: コマンド名の取得、行数表示の可否判定、イベントトリガー対応状況の確認に使用。
  • **呼び出し先**:
  • src/include/tcop/cmdtaglist.h: 列挙子の定義元として直接インクルード。

関連データ構造

  • QueryCompletion: CommandTag と処理行数を保持し、コマンド完了状態を管理する構造体。

失敗モードと境界条件

  • 該当する列挙値が存在しない場合、GetCommandTagName 等の関数で予期せぬ動作を引き起こす可能性があります。
  • INSERT コマンドのタグには、歴史的な理由により "0" を含む特殊なロジックが存在します。

保守担当者向けメモ

  • 新しいSQLコマンドを追加する際は、cmdtaglist.h を更新してください。列挙型と属性テーブルへの反映が自動化されています。
  • CommandTag を文字列に変換する際は、src/backend/tcop/cmdtag.c の関数群を利用してください。
