process_rel_infos

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:563

目的

データベースクラスタから取得したリレーション（テーブル、インデックス等）のメタデータ結果セットを解析し、DbInfo 構造体内の RelInfo 配列へ格納する。

シグネチャまたは形状

static void process_rel_infos(DbInfo *dbinfo, PGresult *res, void *arg)

入力と出力

  • **入力**:
  • dbinfo: 更新対象のデータベース情報を保持する DbInfo 構造体へのポインタ。
  • res: システムカタログ（pg_class 等）から取得したクエリ結果セット。
  • arg: 未使用。
  • **出力**:
  • dbinfo->rel_arr: 解析された RelInfo の配列と件数が格納される。

前提条件と不変条件

  • res は get_rel_infos_query() が生成するクエリの結果セットであること。
  • AssertVariableIsOfType(&process_rel_infos, UpgradeTaskProcessCB) により、UpgradeTaskProcessCB 型との整合性が保証されている。
  • dbinfo は有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. PQntuples で結果セットの行数を取得し、RelInfo 配列をメモリ確保する。 2. 各行を走査し、OID、テーブル/インデックスの関連情報、名前空間、テーブルスペース情報を抽出する。 3. **メモリ最適化**: nspname（名前空間）および tablespace（テーブルスペース）の文字列について、直前の行と値が一致する場合はポインタを再利用し、メモリ消費を抑制する。 4. reltablespace が 0 の場合は、デフォルトの dbinfo->db_tablespace を使用する。 5. 最終的に dbinfo->rel_arr に結果を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/info.c 内の get_db_rel_and_slot_infos から参照されている（UpgradeTask のコールバックとして登録）。
  • **呼び出し先**: pg_malloc, PQntuples, PQfnumber, atooid, PQgetvalue, strcmp, pg_strdup 等のユーティリティ関数。

関連データ構造

  • DbInfo: データベース情報を保持。
  • RelInfo: 個別のリレーション情報を保持。
  • PGresult: libpq のクエリ結果構造体。

失敗モードと境界条件

  • res が空の場合、ntups は 0 となり、空の RelInfo 配列が生成される。
  • reltablespace が 0 の場合、明示的なテーブルスペースパスではなくデータベースデフォルトが適用される。
  • メモリ確保失敗時は pg_malloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数はメモリ効率を考慮し、文字列の重複排除を行っている。新しいフィールドを追加する際は、同様のポインタ再利用ロジックが必要か検討すること。
  • UpgradeTaskProcessCB 型のコールバックとして実装されているため、シグネチャを変更する場合は関連するタスク定義も更新が必要。
