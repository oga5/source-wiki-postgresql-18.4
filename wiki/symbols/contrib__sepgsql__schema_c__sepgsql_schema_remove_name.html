sepgsql_schema_remove_name

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/schema.c:222

目的

指定されたスキーマからオブジェクトが削除される際に、SELinux ポリシーに基づいた権限チェックを実行します。

シグネチャまたは形状

void sepgsql_schema_remove_name(Oid namespaceId)

入力と出力

  • **入力**: namespaceId (Oid) - 対象となるスキーマの OID。
  • **出力**: なし。権限チェックに失敗した場合はエラーを発生させます。

前提条件と不変条件

  • namespaceId は有効な pg_namespace カタログの OID である必要があります。
  • 内部で呼び出される check_schema_perms により、権限違反時にはトランザクションを中断するエラーが送出されます。

アルゴリズムまたは振る舞い

1. check_schema_perms 関数を呼び出します。 2. その際、権限種別として SEPG_DB_SCHEMA__REMOVE_NAME を指定します。 3. 第3引数に true を渡すことで、権限がない場合にエラーを投げる（abort_on_violation）設定で検証を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/proc.c:278
  • contrib/sepgsql/relation.c:680
  • **呼び出し先**:
  • check_schema_perms (内部ユーティリティ関数)

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • namespaceId がシステムカタログ上に存在しない場合、check_schema_perms 内部の処理に依存します。
  • SELinux ポリシーで db_schema:remove_name 権限が許可されていない場合、ereport(ERROR, ...) により処理が停止します。

保守担当者向けメモ

  • 本関数は、スキーマ内のオブジェクト（テーブルやプロシージャ等）が削除される際のフックとして機能します。
  • 権限チェックのロジックを変更する場合は、check_schema_perms の実装を確認してください。
