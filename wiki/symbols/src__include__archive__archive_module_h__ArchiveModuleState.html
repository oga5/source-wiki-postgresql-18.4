ArchiveModuleState

定義

  • 種別: struct
  • 定義箇所: src/include/archive/archive_module.h:20

目的

アーカイブモジュールが実行時に保持すべき状態を格納するためのコンテナです。モジュール固有のコンテキストを各コールバック関数間で共有するために使用されます。

シグネチャまたは形状

typedef struct ArchiveModuleState
{
    void *private_data;
} ArchiveModuleState;

入力と出力

  • **入力**: private_data メンバに任意のデータ構造へのポインタを格納可能。
  • **出力**: 各アーカイブモジュールのコールバック関数（ArchiveFileCB 等）に対し、引数として本構造体へのポインタが渡されます。

前提条件と不変条件

  • private_data は、モジュールが自身のコンテキストを保持するために使用する唯一の場所として設計されています。
  • モジュールは、コールバック内で private_data を適切な型にキャストして使用する必要があります。

アルゴリズムまたは振る舞い

コアシステムはアーカイブ処理のライフサイクルにおいて本構造体を管理します。モジュールがロードされた際、コアシステム側でメモリが割り当てられ、以降のアーカイブ操作においてそのインスタンスが各コールバックへ引き渡されます。

呼び出し元と呼び出し先

  • **呼び出し元（生成）**: src/backend/postmaster/pgarch.c において palloc0 を用いてインスタンス化されます。
  • **呼び出し先（利用）**: 以下のモジュール内のコールバック関数が、引数として本構造体を受け取ります。
  • contrib/basic_archive/basic_archive.c
  • src/backend/archive/shell_archive.c

関連データ構造

  • ArchiveModuleCallbacks: モジュールが実装する関数ポインタを保持する構造体。本状態構造体と組み合わせて使用されます。

失敗モードと境界条件

  • private_data が初期化されていない、または不正なポインタを指している場合、コールバック関数内での参照時にセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は拡張性を考慮し、void * による汎用的なデータ保持を前提としています。モジュール固有のデータ構造を定義し、そのポインタを private_data に格納する運用が推奨されます。
