BuildRelationExtStatistics

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/extended_stats.c:110

目的

ANALYZE 処理の一環として、対象リレーションに対して定義された拡張統計情報（Extended Statistics）を構築し、計算結果をシステムカタログ pg_statistic_ext_data に保存する。

シグネチャまたは形状

void BuildRelationExtStatistics(Relation onerel, bool inh, double totalrows,
                               int numrows, HeapTuple *rows,
                               int natts, VacAttrStats **vacattrstats)

入力と出力

  • **入力**:
  • onerel: 対象リレーションのハンドル。
  • inh: 継承テーブルを含めるかどうかのフラグ。
  • totalrows: テーブルの総行数。
  • numrows, rows: サンプル行数およびそのデータ。
  • natts, vacattrstats: 列数および各列の統計情報。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • natts が 0 の場合、処理は即座に終了する。
  • 統計ターゲット（stattarget）が 0 に設定されている統計オブジェクトは構築をスキップする。
  • 統計構築用のメモリコンテキスト（AllocSetContextCreate）が作成され、各オブジェクトの処理後にリセットされる。

アルゴリズムまたは振る舞い

1. fetch_statentries_for_relation を呼び出し、対象リレーションに関連付けられた統計オブジェクトのリストを取得する。 2. 各統計オブジェクトに対し、以下の手順を実行する：

  • lookup_var_attr_stats で必要な属性情報を収集する。失敗した場合は警告を出力してスキップする。
  • statext_compute_stattarget で統計の精度を決定する。
  • make_build_data で統計構築用のデータを準備する。
  • 統計の種類（NDistinct, Dependencies, MCV, Expressions）に応じて、対応するビルド関数（statext_ndistinct_build 等）を呼び出す。
  • statext_store を呼び出し、計算結果をカタログに書き込む。

    • 3. 処理の進捗状況を pgstat_progress_update_param で報告する。 4. 最後にメモリコンテキストを解放し、リソースをクリーンアップする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/analyze.c:613 (ANALYZE 実行時)
  • **呼び出し先**:
  • fetch_statentries_for_relation
  • statext_compute_stattarget
  • make_build_data
  • statext_ndistinct_build, statext_dependencies_build, statext_mcv_build, compute_expr_stats
  • statext_store

関連データ構造

  • StatExtEntry: 統計オブジェクトの定義を保持する構造体。
  • VacAttrStats: 列ごとの統計情報。
  • StatsBuildData: 統計構築に必要な入力データセット。

失敗モードと境界条件

  • lookup_var_attr_stats が統計構築に必要な列情報を取得できない場合、自動バキューム実行中を除き警告（WARNING）を発行する。
  • 統計ターゲットが 0 の場合は再構築を行わない。
  • 式統計（Expressions）が要求されているにもかかわらず式が存在しない場合、エラー（ERROR）を発生させる。

保守担当者向けメモ

  • 本関数は ANALYZE のメインループから呼び出されるため、メモリリークを防ぐための MemoryContext 管理が重要である。
  • 新しい統計種別を追加する場合は、foreach(lc2, stat->types) ループ内に新しい条件分岐を追加し、対応するビルド関数を呼び出す必要がある。
