pgstat_flush_io

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_io.c:174

目的

バックエンドプロセスで蓄積されたI/O統計情報を、共有メモリ上のグローバルな統計領域へ同期（フラッシュ）するためのインターフェースを提供します。

シグネチャまたは形状

void pgstat_flush_io(bool nowait)

入力と出力

  • **入力**:
  • nowait (bool): true の場合、共有メモリへの書き込み時にロックの競合が発生しても待機せず、非ブロッキングで処理を試みます。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • 本関数は pgstat_io_flush_cb を呼び出すラッパーです。
  • 共有メモリへの書き込み時には、統計サブシステムの排他制御（LWLock）が内部的に使用されます。

アルゴリズムまたは振る舞い

1. 引数 nowait をそのまま pgstat_io_flush_cb に渡して呼び出します。 2. pgstat_io_flush_cb 内で、ローカルの PendingIOStats に蓄積された統計が共有メモリ上の PgStat_BktypeIO 構造体に加算されます。 3. 加算完了後、ローカルの統計変数はリセットされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/walsender.c
  • src/backend/utils/activity/pgstat_bgwriter.c
  • src/backend/utils/activity/pgstat_checkpointer.c
  • src/backend/utils/activity/pgstat_relation.c
  • src/backend/utils/activity/pgstat_wal.c
  • **呼び出し先**:
  • pgstat_io_flush_cb

関連データ構造

  • PendingIOStats: バックエンドローカルで未フラッシュの統計を保持する静的変数。
  • PgStat_BktypeIO: 共有メモリ上に配置されるグローバルなI/O統計構造体。

失敗モードと境界条件

  • nowait が true の場合、ロックが取得できない状況では統計のフラッシュがスキップされる可能性があります（pgstat_io_flush_cb の実装に依存）。

保守担当者向けメモ

  • 本関数は統計サブシステムの公開APIの一部です。I/O統計のフラッシュタイミングを変更する際は、呼び出し元の各プロセス（bgwriter, checkpointer等）の負荷特性を考慮してください。
