quote_literal_cstr

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/quote.c:102

目的

C言語の文字列（const char *）をSQLの文字列リテラルとして安全に使用できるよう、単一引用符で囲み、内部の特殊文字をエスケープした文字列を生成します。

シグネチャまたは形状

char *quote_literal_cstr(const char *rawstr)

入力と出力

  • **入力**: rawstr (エスケープ対象のC文字列)
  • **出力**: エスケープ処理済みの新しいC文字列（pallocによりメモリ確保された領域）。呼び出し元で解放する必要があります。

前提条件と不変条件

  • 入力文字列 rawstr はNULL終端されている必要があります。
  • 戻り値は、最悪のケース（全文字が引用符の場合）を想定した十分なバッファサイズを確保して生成されます。

アルゴリズムまたは振る舞い

1. 入力文字列の長さを取得します。 2. palloc を使用して、変換後の文字列を格納するためのバッファを確保します。サイズは (len * 2) + 3 + 1 バイト（引用符の二重化、外側の引用符、およびNULL終端用）です。 3. quote_literal_internal を呼び出し、エスケープ処理を実行します。 4. 戻り値の末尾にNULL終端文字を付与して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/spi/refint.c
  • contrib/tablefunc/tablefunc.c
  • contrib/test_decoding/test_decoding.c
  • **呼び出し先**:
  • quote_literal_internal (内部ヘルパー)
  • palloc (メモリ確保)
  • strlen (標準Cライブラリ)

関連データ構造

  • 特になし（標準的なC文字列および palloc による動的メモリを使用）。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 空文字列が入力された場合、単一引用符2つ（''）が返されます。

保守担当者向けメモ

  • 本関数は standard_conforming_strings の設定に関わらず、常に安全なエスケープを行うよう設計されています。
  • メモリ確保量は最悪のケースを想定しているため、非常に長い文字列を扱う場合はメモリ消費量に注意してください。
