目的

トルコ語のステミング処理において、単語末尾の「s」で始まる接尾辞を、母音調和の条件に基づき条件付きでマーク（または除去）する。

シグネチャまたは形状

static int r_mark_suffix_with_optional_s_consonant(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowball ステミング環境構造体）
  • 出力: int（成功時に 1、失敗または条件不一致時に 0 を返す）

前提条件と不変条件

  • z->c（カーソル位置）が単語の末尾付近にあること。
  • g_vowel グループ（母音）の定義が環境内に存在すること。

アルゴリズムまたは振る舞い

1. **末尾の「s」確認**: 現在のカーソル位置の直前が「s」であるかを確認する。 2. **条件付きマッチング**:

  • 「s」の直前に母音が存在する場合、その「s」を接尾辞としてマークし成功（1）を返す。
  • 「s」の直前に母音がない場合、さらに別の接尾辞パターン（子音スキップと母音の組み合わせ）を試行する。

    • 3. **バックトラッキング**: マッチングに失敗した場合は、カーソル位置を元の状態に復元し、0 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_turkish.c:670
  • 呼び出し先: in_grouping_b_U（母音判定）、skip_b_utf8（UTF-8文字の逆方向スキップ）

関連データ構造

  • struct SN_env: ステミング実行時の文字列バッファ、カーソル位置、境界情報を保持する。
  • g_vowel: 母音を定義するグループ構造体。

失敗モードと境界条件

  • 単語の先頭に達した場合（z->c <= z->lb）、マッチングは失敗する。
  • UTF-8 文字列のデコードに失敗した場合、skip_b_utf8 が負の値を返し、関数は 0 を返す。
