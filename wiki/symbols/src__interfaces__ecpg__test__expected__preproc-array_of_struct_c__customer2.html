customer2

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:62

目的

ECPG（Embedded SQL for C）における構造体の配列をホスト変数として使用するためのデータ構造定義です。データベースのテーブル行と C 言語のメモリレイアウトをマッピングする役割を担います。

シグネチャまたは形状

typedef struct {
    struct varchar_2 { int len; char arr[50]; } name;
    int phone;
} customer2;

入力と出力

  • **入力**: SQL クエリの結果セット（name 列および phone 列）。
  • **出力**: データベースからフェッチされたデータが格納される C 言語の構造体インスタンス。

前提条件と不変条件

  • 構造体内の name メンバは、ECPG が認識可能な varchar 型として定義されています。
  • 構造体のメンバ順序は、対応する SQL 文のフェッチ対象列と一致している必要があります。

アルゴリズムまたは振る舞い

  • ECPG プリプロセッサによって生成されるコードにおいて、SQL 文の実行結果をこの構造体の配列にバインドするために使用されます。
  • ECPGdo 関数呼び出しにおいて、ECPGt_varchar および ECPGt_int 型として指定され、データベースの列データが各メンバにコピーされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c 内の main 関数にて、customer2 custs2[10] として宣言され、データフェッチのバッファとして使用されています。
  • **呼び出し先**: ECPGdo 関数（SQL 操作の実行時にこの構造体のメンバアドレスが引数として渡されます）。
  • **その他**: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c でも同名の構造体が定義されており、ポインタ経由のメモリ割り当てに使用されています。

関連データ構造

  • varchar_2: customer2 構造体内部で定義された、文字列格納用のネストされた構造体。

失敗モードと境界条件

  • name メンバの arr[50] を超える長さの文字列がデータベースから返された場合、バッファオーバーフローや切り捨てが発生する可能性があります。
  • 構造体の配列サイズ（例: custs2[10]）を超える行数がフェッチされた場合、メモリ破壊や未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は ECPG のプリプロセッサが生成するテストコードの一部です。手動での変更は行わず、プリプロセッサの生成ロジックやテストケースの要件変更に合わせて調整してください。
  • 構造体定義を変更した場合は、ECPGdo 呼び出し時の型指定（ECPGt_varchar 等）およびサイズ指定（sizeof(customer2)）が整合していることを確認してください。
