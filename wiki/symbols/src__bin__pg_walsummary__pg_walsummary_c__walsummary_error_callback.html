walsummary_error_callback

定義

  • 種別: function
  • 定義箇所: src/bin/pg_walsummary/pg_walsummary.c:229

目的

BlockRefTableReader 実行中に発生したエラーをログに記録し、プログラムを異常終了させるためのエラーハンドラです。

シグネチャまたは形状

void walsummary_error_callback(void *callback_arg, char *fmt, ...)

入力と出力

  • **入力**:
  • callback_arg: void * 型。コールバック間で共有されるコンテキスト（本関数内では使用されません）。
  • fmt: エラーメッセージのフォーマット文字列。
  • ...: 可変長引数。
  • **出力**: なし（関数内で exit(1) を呼び出し、プロセスを終了します）。

前提条件と不変条件

  • pg_log_generic_v を使用するため、呼び出し前にログ出力の初期化が完了している必要があります。
  • 本関数が呼び出された場合、プログラムは必ず終了します。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化します。 2. pg_log_generic_v を呼び出し、PG_LOG_ERROR および PG_LOG_PRIMARY レベルでエラーメッセージを標準エラー出力等へ出力します。 3. va_end で引数リストを終了します。 4. exit(1) を実行し、ステータスコード 1 でプログラムを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_walsummary/pg_walsummary.c 内の BlockRefTableReader を初期化する箇所（CreateBlockRefTableReader への引数として渡されます）。
  • **呼び出し先**:
  • pg_log_generic_v (PostgreSQL ログ出力ユーティリティ)
  • exit (標準ライブラリ)

関連データ構造

  • なし。

失敗モードと境界条件

  • 本関数自体はエラーハンドラであるため、呼び出された時点で既にエラー状態にあります。
  • fmt が NULL である場合や、可変長引数がフォーマットと一致しない場合の挙動は pg_log_generic_v の実装に依存します。

保守担当者向けメモ

  • 本関数は BlockRefTableReader のエラーコールバックインターフェースに適合するように設計されています。
  • callback_arg は将来的な拡張のために用意されていますが、現在は使用されていません。
