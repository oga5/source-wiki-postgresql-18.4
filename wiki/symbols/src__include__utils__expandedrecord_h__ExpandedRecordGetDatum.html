ExpandedRecordGetDatum

定義

  • 種別: function
  • 定義箇所: src/include/utils/expandedrecord.h:142

目的

展開されたレコード（ExpandedRecordHeader）を、PostgreSQLの実行エンジンで扱える Datum 型に変換して取得する。

シグネチャまたは形状

static inline Datum ExpandedRecordGetDatum(const ExpandedRecordHeader *erh)

入力と出力

  • **入力**: erh - 変換対象の ExpandedRecordHeader 構造体へのポインタ。
  • **出力**: 展開されたレコードを指す Datum 値。

前提条件と不変条件

  • erh は有効な ExpandedRecordHeader を指している必要がある。
  • 本関数は EOHPGetRWDatum を呼び出すため、対象の展開されたオブジェクトが読み書き可能な状態であることを前提とする。

アルゴリズムまたは振る舞い

内部的に EOHPGetRWDatum を呼び出し、ExpandedRecordHeader 内のヘッダー情報（hdr）から、展開されたオブジェクトの読み書き可能な Datum 表現を抽出して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/expandedrecord.h: PG_RETURN_EXPANDED_RECORD マクロ内。
  • src/pl/plpgsql/src/pl_exec.c: PL/pgSQLの実行処理において、レコード値の取得や削除（DeleteExpandedObject）時に使用。
  • **呼び出し先**:
  • EOHPGetRWDatum（utils/expandeddatum.h で定義される基底関数）。

関連データ構造

  • ExpandedRecordHeader: 展開されたレコードのメイン構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • erh が NULL の場合、未定義の動作を引き起こす可能性がある。
  • 呼び出し先である EOHPGetRWDatum の制約に従い、メモリコンテキストの整合性が保たれている必要がある。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスを重視している。
  • 読み取り専用の Datum を取得したい場合は、対となる ExpandedRecordGetRODatum を使用すること。
