dsa_unpin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:993

目的

指定された動的共有メモリ領域（dsa_area）のピン留め状態を解除し、参照カウントを減らします。

シグネチャまたは形状

void dsa_unpin(dsa_area *area)

入力と出力

  • **入力**: area (dsa_area *) - 操作対象のDSA領域へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • area->control->refcnt が 1 より大きいこと（アサーションにより検証）。
  • area->control->pinned が true であること。条件を満たさない場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. DSA_AREA_LOCK(area) を排他モードで取得します。 2. 領域が現在ピン留めされているかを確認し、されていない場合はエラーをスローしてロックを解放します。 3. area->control->pinned を false に設定します。 4. area->control->refcnt をデクリメントします。 5. DSA_AREA_LOCK(area) を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定されていません。
  • **呼び出し先**: LWLockAcquire、LWLockRelease、elog を使用します。

関連データ構造

  • dsa_area: DSA領域の状態を保持する構造体。
  • dsa_area_control: 共有メモリ領域の制御メタデータ（refcnt および pinned フラグを含む）。

失敗モードと境界条件

  • area->control->pinned が false の状態で呼び出された場合、elog(ERROR, "dsa_area not pinned") により処理が中断されます。
  • refcnt が 1 以下の状態で呼び出された場合、アサーション違反が発生します。

保守担当者向けメモ

  • 本関数は dsa_pin と対になる操作です。
  • ロック取得順序の規約（DSA_AREA_LOCK）に従っているため、他の領域操作とのデッドロックに注意してください。
