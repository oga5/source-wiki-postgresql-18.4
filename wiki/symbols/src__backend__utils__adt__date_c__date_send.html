date_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/date.c:231

目的

DATE 型の値を PostgreSQL のバイナリ転送形式に変換します。

シグネチャまたは形状

Datum date_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して DateADT 型の値を受け取ります。
  • **出力**: bytea 型（バイナリデータ）を返します。

前提条件と不変条件

  • 入力値は PG_GETARG_DATEADT(0) を介して取得可能であること。
  • pq_begintypsend および pq_endtypsend を用いたバイナリプロトコルシーケンスに従うこと。

アルゴリズムまたは振る舞い

1. pq_begintypsend を呼び出し、バイナリ送信用のバッファを初期化します。 2. PG_GETARG_DATEADT(0) で取得した DateADT（内部表現: int32）を pq_sendint32 を使用してバッファに書き込みます。 3. pq_endtypsend を呼び出してバッファを bytea 型に変換し、その結果を PG_RETURN_BYTEA_P で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 不明（PostgreSQL の型システムにより、DATE 型のバイナリ送信時に呼び出されます）。
  • **呼び出し先**:
  • pq_begintypsend
  • pq_sendint32
  • pq_endtypsend

関連データ構造

  • DateADT: int32 で表現される日付（エポックからの経過日数）。
  • StringInfoData: バイナリ送信用のバッファ構造体。

失敗モードと境界条件

  • メモリ不足時、pq_begintypsend 等の内部関数によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は DATE 型のバイナリプロトコル対応を担います。DateADT の内部表現（int32）に変更を加える場合は、本関数の送信ロジックも整合性を確認してください。
