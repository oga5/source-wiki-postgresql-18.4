LogLogicalMessage

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/message.c:42

目的

論理デコーディングにおいて、任意のバイナリデータをWAL（Write-Ahead Log）に書き込み、レプリケーションストリームへカスタムイベントを挿入する。

シグネチャまたは形状

XLogRecPtr LogLogicalMessage(const char *prefix, const char *message, size_t size, bool transactional, bool flush)

入力と出力

  • **入力**:
  • prefix: メッセージの識別用プレフィックス（末尾にNULL文字を含む）。
  • message: 書き込むバイナリデータ。
  • size: メッセージのサイズ。
  • transactional: トランザクションの一部として記録するかどうか。
  • flush: 非トランザクション時に即時ディスク書き込みを行うかどうか。
  • **出力**:
  • XLogRecPtr: 書き込まれたWALレコードのLSN（Log Sequence Number）。

前提条件と不変条件

  • transactional が真の場合、現在のトランザクション状態であること（IsTransactionState()）が必須。
  • プレフィックスには、プラグイン間での衝突を防ぐための識別子が期待される。

アルゴリズムまたは振る舞い

1. **トランザクション管理**: transactional が真の場合、現在のトランザクションIDを割り当てる。 2. **レコード構築**: xl_logical_message 構造体にデータベースID、トランザクション属性、サイズ情報を格納する。 3. **WAL挿入**: XLogBeginInsert を呼び出し、構造体、プレフィックス、メッセージ本体をWALバッファに登録する。 4. **フラッシュ**: transactional が偽かつ flush が真の場合、XLogFlush を呼び出し、ディスクへの永続化を保証する。 5. **戻り値**: 挿入されたレコードのLSNを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/logicalfuncs.c:376
  • **呼び出し先**: XLogBeginInsert, XLogRegisterData, XLogSetRecordFlags, XLogInsert, XLogFlush（条件付き）, GetCurrentTransactionId（条件付き）。

関連データ構造

  • xl_logical_message: WALに記録される論理メッセージのヘッダ構造体。

失敗モードと境界条件

  • トランザクション外で transactional を真に設定した場合、アサーション違反が発生する。

保守担当者向けメモ

  • logicalmsg_redo は論理デコーディング用であり、物理的なリプレイは行わない（NOOP）。
  • プレフィックスの末尾にNULL文字を含めることが logicalmsg_desc の処理において重要である。
