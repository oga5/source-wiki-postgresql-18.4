目的

ECPG（Embedded SQL for C）における SQLDA（SQL Descriptor Area）インターフェースの動作を検証する回帰テストの実行関数。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しない）
  • 出力: 成功時に 0 を返し、エラー発生時に exit(1) を呼び出す。

前提条件と不変条件

  • ECPG ライブラリが初期化されていること。
  • データベース ecpg1_regression への接続が可能であること。
  • テスト用テーブル t1 が作成可能であること。

アルゴリズムまたは振る舞い

1. **環境設定**: ECPGdebug を有効にし、データベースへ接続。 2. **テーブル構築**: t1 テーブルを作成し、テストデータを挿入してコミット。 3. **動的 SQL 実行 (カーソル)**:

  • PREPARE 文で SQL を準備し、カーソルを宣言。
  • FETCH 時に ECPGt_sqlda 型を指定し、結果を outp_sqlda に格納。
  • dump_sqlda を呼び出し、取得したデータを標準出力へダンプ。
  • カーソルを閉じ、ステートメントを解放。

    • 4. **メモリ管理**: outp_sqlda に割り当てられたメモリを free で解放。 5. **終了処理**: データベース接続を終了。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGprepare, ECPGdeallocate, dump_sqlda, printf, exit, free

関連データ構造

  • sqlda_t: SQLDA 構造体。
  • sqlvar_t: SQLDA 内の列情報保持用構造体。
  • sqlca: ECPG のエラー情報を保持するグローバル変数。

失敗モードと境界条件

  • 各 ECPG 実行関数（ECPGdo 等）の実行後、sqlca.sqlcode が負の値（エラー）である場合に exit(1) で終了する。
  • FETCH 時に ECPG_NOT_FOUND が発生した場合はループを抜ける。

保守担当者向けメモ

  • 本関数は ECPG プリプロセッサによって生成されたコードであるため、直接修正する場合はプリプロセッサの仕様を考慮すること。
  • sqlda_t の定義は環境（Informix 互換モードの有無）により切り替わるため注意が必要。
