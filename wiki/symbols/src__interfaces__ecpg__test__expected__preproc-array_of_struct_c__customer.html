customer

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:33

目的

ECPG（Embedded SQL for C）における構造体の配列をホスト変数として使用し、データベースとの間でデータをやり取りするためのデータ構造。

シグネチャまたは形状

typedef struct {
  struct varchar_1 { int len; char arr[50]; } name;
  int phone;
} customer;

入力と出力

  • **入力**: データベースからフェッチされた行データ（name および phone）。
  • **出力**: ECPGdo 関数を通じてデータベースへ渡される、またはデータベースから受け取るホスト変数としての構造体インスタンス。

前提条件と不変条件

  • 構造体のメンバ順序は、対応するデータベーステーブルの列定義と一致している必要がある。
  • ECPGdo 呼び出し時に渡される sizeof(customer) は、コンパイル時の構造体サイズと一致しなければならない。

アルゴリズムまたは振る舞い

  • ECPG プリプロセッサによって C 言語のホスト変数として展開される。
  • ECPGdo 関数において、ECPGt_varchar 型（name メンバ用）および ECPGt_int 型（phone メンバ用）としてバインドされ、SQL の結果セットを格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c 内の main 関数（配列 custs1 の要素として使用）。
  • **呼び出し先**: ECPGdo 関数（SQL 操作の実行時に本構造体のアドレスが渡される）。
  • その他、src/interfaces/ecpg/test/expected/compat_informix-test_informix2.c および src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c からの参照が確認されている。

関連データ構造

  • varchar_1: customer 構造体内の name メンバとして定義されたネスト構造体。
  • cust_ind: customer 構造体に対応するインジケータ変数の構造体。

失敗モードと境界条件

  • name メンバの arr[50] を超える文字列がデータベースから返された場合、バッファオーバーフローやデータ切り捨てが発生する可能性がある。
  • 構造体のメモリレイアウトがプリプロセッサの期待と異なる場合、ECPGdo によるデータバインドが失敗する。
