read_stream_start_pending_read

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/read_stream.c:229

目的

蓄積されたブロック読み込み要求（pending_read）を、バッファマネージャの StartReadBuffers を用いた実際の I/O 操作へ変換し、必要に応じて先読みアドバイスを発行する。

シグネチャまたは形状

static bool read_stream_start_pending_read(ReadStream *stream)

入力と出力

  • **入力**: ReadStream *stream（ストリームの状態を保持する構造体）
  • **出力**: bool（I/O が開始されたか、またはバッファが利用可能であれば true、それ以外は false）

前提条件と不変条件

  • stream->pending_read_nblocks > 0 であること。
  • stream->pending_read_nblocks <= stream->io_combine_limit であること。
  • stream->pinned_buffers + stream->pending_read_nblocks <= stream->max_pinned_buffers であること。

アルゴリズムまたは振る舞い

1. **アドバイス発行**: ストリームの設定に基づき、シーケンシャルアクセスであれば READ_BUFFERS_ISSUE_ADVICE フラグを付与する。 2. **リソース制限の適用**: バックエンドのピン制限（GetAdditionalPinLimit 等）に基づき、読み込み可能なブロック数を調整する。制限を超える場合は読み込み数を縮小する。 3. **I/O 発行**: StartReadBuffers を呼び出し、バッファを割り当てて I/O を開始する。 4. **状態更新**:

  • I/O が即時完了しない場合は ios_in_progress をインクリメントし、WaitReadBuffers が必要であることを記録する。
  • 完了した場合は先読み距離（distance）を減衰させる。

    • 5. **後方処理**: 読み込み範囲がキューの末尾を超える場合、循環キューとして機能するようにバッファをコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/read_stream.c 内の各関数（read_stream_next_buffer 等から呼び出される）。
  • **呼び出し先**: StartReadBuffers（バッファマネージャ）、GetAdditionalPinLimit / GetAdditionalLocalPinLimit。

関連データ構造

  • ReadStream: ストリームの制御状態を保持するメイン構造体。
  • InProgressIO: 進行中の I/O 操作を管理する構造体。

失敗モードと境界条件

  • **バッファ制限**: buffer_limit が 0 かつ pinned_buffers が 0 の場合、進捗を保証するために buffer_limit を 1 に強制する。
  • **キューのラップアラウンド**: overflow が発生した場合、バッファをキューの先頭へコピーする処理が行われる。

保守担当者向けメモ

  • 本関数は StartReadBuffers の戻り値に基づいて need_wait を判定し、非同期 I/O の完了待ちが必要かどうかを管理している。
  • forwarded_buffers の管理は、前回の読み込みで分割された処理を正しく継続するために不可欠である。
