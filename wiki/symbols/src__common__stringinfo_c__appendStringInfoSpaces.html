appendStringInfoSpaces

定義

  • 種別: function
  • 定義箇所: src/common/stringinfo.c:259

目的

StringInfo バッファの末尾に指定された数のスペース文字（' '）を追加し、バッファを適切に拡張およびヌル終端する。

シグネチャまたは形状

void appendStringInfoSpaces(StringInfo str, int count)

入力と出力

  • **入力**:
  • str: 操作対象の StringInfo 構造体へのポインタ。
  • count: 追加するスペースの数（整数）。
  • **出力**: なし（str が指すバッファの内容が更新される）。

前提条件と不変条件

  • str は初期化済みであること。
  • count が負の値の場合、何も行われない。
  • 実行後、str->len は count 分増加し、str->data の末尾には常にヌル文字（'\0'）が配置される。

アルゴリズムまたは振る舞い

1. count が 0 より大きいかを確認する。 2. enlargeStringInfo を呼び出し、追加するスペース分を格納するための十分な容量をバッファに確保する。 3. memset を使用して、現在のバッファ末尾（str->data[str->len]）から count 分のスペースを書き込む。 4. str->len を count 分加算し、新しい末尾位置にヌル終端文字を書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c
  • src/backend/commands/explain_format.c

    • (上記ファイル内の各所から、インデント生成等の目的で呼び出されている)

  • **呼び出し先**:
  • enlargeStringInfo (バッファ拡張用)
  • memset (標準ライブラリ)

関連データ構造

  • StringInfoData (およびそのポインタ型 StringInfo): 文字列バッファ、現在長、最大容量を保持する構造体。

失敗モードと境界条件

  • count <= 0: 何も実行されず、副作用も発生しない。
  • メモリ不足時: enlargeStringInfo 内部で palloc / repalloc が失敗した場合、PostgreSQL のエラーハンドリング機構により例外が発生する。

保守担当者向けメモ

  • 本関数は StringInfo のヌル終端を維持する設計である。
  • 頻繁な呼び出しが発生する場合、事前に enlargeStringInfo でバッファを確保しておくことで、再確保のオーバーヘッドを抑制可能である。
