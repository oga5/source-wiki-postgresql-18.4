raw_array_subscript_handler

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arraysubs.c:566

目的

ヘッダーを持たない単純な配列構造（raw配列）に対する添字操作（サブスクリプト）を処理するための、SubscriptRoutines 構造体へのポインタを返します。

シグネチャまたは形状

Datum raw_array_subscript_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（標準的なPostgreSQL関数呼び出し規約）。
  • **出力**: SubscriptRoutines 構造体へのポインタを Datum 型として返します。

前提条件と不変条件

  • 本関数は静的な SubscriptRoutines 構造体を保持しており、呼び出しごとに同一の定義を返します。
  • fetch 操作は厳密（strict）であり、入力が NULL の場合は NULL を返します。
  • fetch 操作はリークプルーフ（leakproof）であり、不正な添字に対してはエラーを投げず NULL を返します。
  • store（代入）操作はリークプルーフではなく、不正な添字に対してはエラーを発生させます。

アルゴリズムまたは振る舞い

1. SubscriptRoutines 構造体を以下の設定で初期化します。

  • transform: array_subscript_transform
  • exec_setup: array_exec_setup
  • fetch_strict: true
  • fetch_leakproof: true
  • store_leakproof: false

    • 2. 初期化した構造体へのポインタを PG_RETURN_POINTER マクロを介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/typecmds.c:504 にて、後方互換性のために利用可能である旨の言及があります。
  • **呼び出し先**: 構造体定義内で array_subscript_transform および array_exec_setup を参照しています。

関連データ構造

  • SubscriptRoutines: 添字操作の動作を定義する構造体。
  • ArraySubWorkspace: 実行時に添字の境界値や型情報を保持するために使用されます。

失敗モードと境界条件

  • 本関数自体はポインタを返すのみですが、返された SubscriptRoutines を使用した代入操作において、不正な添字が指定された場合はエラーが発生します。

保守担当者向けメモ

  • 本関数はヘッダーを持たない単純な配列構造を扱うためのものです。将来的に標準配列のハンドラと分離される可能性があります。
  • 代入操作における NULL の扱いは、固定長配列と可変長配列（varlena）で挙動が異なるため注意が必要です。
