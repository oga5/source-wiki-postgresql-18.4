ItemPointerData

定義

  • 種別: struct
  • 定義箇所: contrib/pageinspect/brinfuncs.c:386

目的

データベース内の特定のタプルを指し示すための物理的な位置情報（TID: Tuple Identifier）を保持する構造体です。

シグネチャまたは形状

ソースコード上では ItemPointerData 型として定義されています。内部には ip_posid を含む構成要素が存在することが、contrib/amcheck/verify_nbtree.c のオフセット計算から示唆されています。

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。ただし、brinfuncs.c 内ではリビジョンマップのデータを走査する際、ItemPointerData *tids としてメモリ上に展開されます。

前提条件と不変条件

  • contrib/amcheck/verify_nbtree.c の実装に基づくと、ip_posid は構造体の先頭からのオフセットとして期待される位置に配置されている必要があります。

アルゴリズムまたは振る舞い

  • brinfuncs.c においては、リビジョンマップページから抽出された TID の配列を保持するために使用されます。
  • brin_revmap_data 関数内で、ページ内の TID 配列を反復処理するためのポインタとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pageinspect/brinfuncs.c 内の brin_revmap_data 関数が、リビジョンマップの解析時に本構造体の配列を割り当てて使用します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。
  • **その他**: amcheck モジュール（verify_gin.c, verify_nbtree.c）や bloom インデックスなど、PostgreSQL の複数のモジュールから参照されています。

関連データ構造

  • contrib/pageinspect/brinfuncs.c 内の無名構造体:

    • ``c struct { ItemPointerData *tids; int idx; } *state; ` この構造体は、brin_revmap_data の実行コンテキストとして ItemPointerData` の配列を管理します。

失敗モードと境界条件

  • メモリ割り当て失敗: palloc を使用して ItemPointerData の配列を確保する際、メモリ不足が発生する可能性があります。
  • 範囲外アクセス: brin_revmap_data における idx の管理が不適切な場合、tids 配列の境界を超えてアクセスするリスクがあります。

保守担当者向けメモ

  • 本構造体は PostgreSQL のコア定義（include/storage/itemptr.h 等）に依存する共通構造体です。pageinspect 側で定義を変更することは避けてください。
  • brinfuncs.c での利用は、あくまで物理ページ上の TID 配列を SQL 結果として出力するための「読み取り専用」のバッファとしての役割に限定されています。
