XLogFileName

定義

  • 種別: function
  • 定義箇所: src/include/access/xlog_internal.h:165

目的

WAL（先行書き込みログ）セグメントのファイル名を、タイムラインIDとセグメント番号から生成します。

シグネチャまたは形状

static inline void XLogFileName(char *fname, TimeLineID tli, XLogSegNo logSegNo, int wal_segsz_bytes)

入力と出力

  • **入力**:
  • fname: ファイル名を格納するバッファ（MAXFNAMELEN 以上のサイズが必要）。
  • tli: タイムラインID (TimeLineID)。
  • logSegNo: WALセグメント番号 (XLogSegNo)。
  • wal_segsz_bytes: WALセグメントサイズ（バイト単位）。
  • **出力**:
  • fname バッファに、%08X%08X%08X 形式の16進数文字列が書き込まれます。

前提条件と不変条件

  • wal_segsz_bytes は、WALセグメントの物理サイズを正しく反映している必要があります。
  • XLogSegmentsPerXLogId マクロが、渡された wal_segsz_bytes に基づいて正しくセグメント数を計算できる必要があります。

アルゴリズムまたは振る舞い

1. logSegNo を XLogSegmentsPerXLogId(wal_segsz_bytes) で除算し、上位のセグメントIDを算出します。 2. logSegNo を XLogSegmentsPerXLogId(wal_segsz_bytes) で剰余演算し、下位のセグメントIDを算出します。 3. snprintf を使用し、タイムラインIDと算出した2つのセグメントIDを連結して、24文字の16進数文字列（8文字×3）を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c (多数の箇所でWALファイル操作時に使用)
  • src/backend/access/transam/xlogarchive.c:144
  • **呼び出し先**:
  • snprintf (標準Cライブラリ)
  • XLogSegmentsPerXLogId (マクロ)

関連データ構造

  • TimeLineID: タイムラインを識別する型。
  • XLogSegNo: WALセグメント番号を保持する型。

失敗モードと境界条件

  • fname バッファが MAXFNAMELEN 未満の場合、バッファオーバーフローの可能性があります。
  • wal_segsz_bytes が不正な値（0など）の場合、XLogSegmentsPerXLogId 内でゼロ除算が発生する可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、フロントエンドツール（pg_receivewal 等）からも利用されます。変更時は互換性に注意してください。
  • セグメントサイズは動的に変更可能なパラメータであるため、ハードコードせず必ず引数から渡す設計になっています。
