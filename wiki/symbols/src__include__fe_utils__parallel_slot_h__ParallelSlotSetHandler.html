ParallelSlotSetHandler

定義

  • 種別: static inline function
  • 定義箇所: src/include/fe_utils/parallel_slot.h:46

目的

指定された ParallelSlot に対して、クエリ結果を処理するためのコールバック関数（ParallelSlotResultHandler）と、その実行時に渡されるコンテキストデータを登録します。

シグネチャまたは形状

static inline void
ParallelSlotSetHandler(ParallelSlot *slot, ParallelSlotResultHandler handler,
                       void *context)

入力と出力

  • **入力**:
  • slot: 処理対象の ParallelSlot 構造体へのポインタ。
  • handler: クエリ結果を受け取る関数ポインタ（ParallelSlotResultHandler 型）。
  • context: ハンドラ実行時に引数として渡される任意のデータへのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • slot は有効な ParallelSlot 構造体を指している必要があります。
  • 登録された handler は、後続の ParallelSlotsWaitCompletion 呼び出し時に、対応する接続で結果が利用可能になった際に実行されます。

アルゴリズムまたは振る舞い

1. 引数として受け取った handler を slot->handler に代入します。 2. 引数として受け取った context を slot->handler_context に代入します。 3. これにより、イベントループ（select_loop）が結果を検知した際、このハンドラが指定されたコンテキストと共に呼び出されるようになります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c
  • **呼び出し先**:
  • 直接的な関数呼び出しは行いません（ポインタの代入のみ）。

関連データ構造

  • ParallelSlot: 接続状態とハンドラ情報を保持する構造体。
  • ParallelSlotResultHandler: クエリ結果を処理するための関数ポインタ型。

失敗モードと境界条件

  • slot が NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • ハンドラ内で PQclear を実行するかどうかは、各ハンドラの実装に依存します。メモリリークを防ぐため、ハンドラの実装方針と整合性を取る必要があります。
