x_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgkdtreeproc.c:84

目的

SortedPoint 構造体の配列を x 座標に基づいて昇順にソートするための比較関数を提供します。

シグネチャまたは形状

int x_cmp(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つの SortedPoint 構造体へのポインタ (a, b)。
  • **出力**:
  • a の x 座標が b と等しい場合は 0。
  • a の x 座標が b より大きい場合は 1。
  • a の x 座標が b より小さい場合は -1。

前提条件と不変条件

  • 入力ポインタ a および b は、SortedPoint 構造体への有効なポインタである必要があります。
  • SortedPoint 構造体内の p フィールド（Point 型）が初期化されていることが前提です。

アルゴリズムまたは振る舞い

1. 引数 a および b を SortedPoint * 型にキャストします。 2. 各構造体が保持する Point 型の x 座標を比較します。 3. x 座標の大小関係に基づき、標準的な比較結果（0, 1, -1）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgkdtreeproc.c:125（spg_kd_picksplit 内でソート関数として指定）
  • src/backend/access/spgist/spgquadtreeproc.c:186（qsort の引数として使用）
  • **呼び出し先**: なし。

関連データ構造

  • SortedPoint: spgkdtreeproc.c で定義される、ソート処理用の補助構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作（セグメンテーション違反）が発生します。
  • x 座標が NaN である場合の挙動は、標準的な浮動小数点比較の仕様に依存します。

保守担当者向けメモ

  • 本関数は qsort 互換のシグネチャを持っています。
  • spgkdtreeproc.c 内の y_cmp と対になる関数です。
  • spgquadtreeproc.c からも参照されていますが、同ファイル内には同名の関数定義が存在する可能性があるため、リンク時の競合や意図しない参照に注意してください。
