ExecCustomScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCustom.c:113

目的

実行エンジンから呼び出され、拡張機能によって定義されたカスタムスキャンロジックを実行してタプルを取得するメインエントリポイントです。

シグネチャまたは形状

TupleTableSlot *ExecCustomScan(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate （実行中のカスタムスキャンノードの状態）
  • **出力**: TupleTableSlot * （取得されたタプルを格納するスロット、またはスキャン終了時は空のスロット）

前提条件と不変条件

  • pstate は CustomScanState 型にキャスト可能であること。
  • node->methods->ExecCustomScan が NULL でないこと（Assert により検証）。
  • 割り込み要求（CHECK_FOR_INTERRUPTS）が処理可能であること。

アルゴリズムまたは振る舞い

1. pstate を CustomScanState にキャストします。 2. CHECK_FOR_INTERRUPTS() を呼び出し、クエリのキャンセルや停止要求を確認します。 3. 拡張機能が提供する CustomExecMethods 内の ExecCustomScan 関数ポインタを呼び出し、実際のタプル取得処理を委譲します。 4. 拡張機能から返された TupleTableSlot をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（ExecProcNode 経由で呼び出される）。
  • **呼び出し先**: node->methods->ExecCustomScan （拡張機能側で実装された関数）。

関連データ構造

  • CustomScanState: カスタムスキャンの実行状態を保持する構造体。
  • CustomExecMethods: 拡張機能が実装すべきコールバック関数のテーブル。

失敗モードと境界条件

  • node->methods->ExecCustomScan が未定義（NULL）の場合、Assert によりデバッグビルド環境で異常終了します。
  • 拡張機能側の実装が NULL を返した場合、呼び出し元にそのまま伝播します。

保守担当者向けメモ

  • 本関数は「薄いラッパー」であり、実際のロジックはすべて拡張機能側の CustomExecMethods に依存します。
  • 拡張機能側で ExecCustomScan を実装する際は、必ず TupleTableSlot を適切に管理してください。
