query_tree_walker_impl

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/nodes/nodeFuncs.c:2693

目的

Query ノードツリーを再帰的に走査し、指定されたコールバック関数を各ノードに対して適用する。

シグネチャまたは形状

bool query_tree_walker_impl(Query *query, tree_walker_callback walker, void *context, int flags)

入力と出力

  • **入力**:
  • query: 走査対象の Query 構造体へのポインタ。
  • walker: 各ノードで呼び出されるコールバック関数。
  • context: コールバックに渡される任意のデータ。
  • flags: 走査の挙動を制御するビットフラグ（QTW_EXAMINE_SORTGROUP, QTW_IGNORE_CTE_SUBQUERIES, QTW_IGNORE_RANGE_TABLE 等）。
  • **出力**:
  • bool: コールバックが true を返した場合は走査を中断し true を返す。最後まで走査した場合は false を返す。

前提条件と不変条件

  • query は NULL であってはならず、かつ Query 型である必要がある（Assert による検証）。
  • UtilityStmt は本関数では走査されない。

アルゴリズムまたは振る舞い

1. Query 構造体の各フィールド（targetList, jointree, havingQual 等）に対して WALK() マクロを介してコールバックを適用する。 2. flags に QTW_EXAMINE_SORTGROUP が含まれる場合、groupClause, windowClause, sortClause, distinctClause を走査する。含まれない場合でも、windowClause 内の startOffset と endOffset は走査対象となる。 3. flags に QTW_IGNORE_CTE_SUBQUERIES が含まれていない場合、cteList を走査する。 4. flags に QTW_IGNORE_RANGE_TABLE が含まれていない場合、range_table_walker を呼び出して範囲テーブルを走査する。 5. groupingSets および rowMarks は本関数では走査されない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/nodes/nodeFuncs.h 内のラッパーマクロ等から呼び出される。
  • **呼び出し先**:
  • walker (コールバック関数)
  • range_table_walker (範囲テーブル走査用)
  • WALK() マクロ（内部的な再帰呼び出し）

関連データ構造

  • Query: 走査対象となるクエリツリーのルート構造体。
  • WindowClause: windowClause 走査時に参照される構造体。

失敗モードと境界条件

  • コールバック関数が true を返すと、その時点で走査が停止し、呼び出し元へ true が伝播する。
  • groupingSets や rowMarks は明示的にスキップされるため、これらが必要な場合は呼び出し側で個別に処理する必要がある。

保守担当者向けメモ

  • WALK() マクロは再帰的な走査を行うため、スタックオーバーフローのリスクがある。
  • 新しい Query フィールドを追加した場合は、本関数の走査対象リストに適切に追加する必要がある。
  • groupingSets や rowMarks の走査が必要な場合は、本関数を修正するのではなく、呼び出し側で処理を実装すること。
