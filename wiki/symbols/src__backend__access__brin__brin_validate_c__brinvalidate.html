brinvalidate

定義

  • src/backend/access/brin/brin_validate.c:36

目的

BRIN（Block Range Index）アクセスメソッドにおける演算子クラス（opclass）および演算子族（opfamily）の定義が、PostgreSQLの仕様に適合しているかを検証する。

シグネチャまたは形状

bool brinvalidate(Oid opclassoid)

入力と出力

  • **入力**: opclassoid (検証対象の演算子クラスのOID)
  • **出力**: 検証に成功した場合は true、失敗した場合は false を返す。検証エラーは ereport(INFO, ...) を通じてログに出力される。

前提条件と不変条件

  • opclassoid に対応する演算子クラスがシステムカタログ pg_opclass に存在すること。
  • BRINは ORDER BY 演算子をサポートしないため、これを含む定義は無効とみなされる。

アルゴリズムまたは振る舞い

1. pg_opclass および pg_opfamily から対象の演算子クラスと演算子族の情報を取得する。 2. 演算子族に属するすべてのサポート関数を走査し、check_amproc_signature を用いて必須の関数（opcinfo, addvalue, consistent, union 等）のシグネチャを検証する。 3. 演算子族に属するすべての演算子を走査し、ストラテジー番号（1-63）の妥当性、ORDER BY 句の有無、および check_amop_signature による型適合性を検証する。 4. identify_opfamily_groups を使用して、データ型ごとの演算子・関数のグループ整合性を確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c (AMルーチン amvalidate への登録経由)
  • **呼び出し先**: check_amproc_signature, check_amop_signature, identify_opfamily_groups, SearchSysCache1, SearchSysCacheList1 等のシステムカタログ操作関数。

関連データ構造

  • Form_pg_opclass, Form_pg_amproc, Form_pg_amop (システムカタログの構造体)
  • OpFamilyOpFuncGroup (演算子族のグループ化情報)

失敗モードと境界条件

  • 必須のサポート関数が欠落している、またはシグネチャが不一致の場合。
  • 許可されていないストラテジー番号（1未満または63超）が使用されている場合。
  • BRINでサポートされない ORDER BY 演算子が含まれている場合。
  • キャッシュルックアップが失敗した場合（elog(ERROR) で中断）。

保守担当者向けメモ

  • 本関数は CREATE OPERATOR CLASS 実行時にのみ呼び出される。
  • 演算子族レベルでのチェックと演算子クラスレベルでのチェックが混在しているため、エラーメッセージが重複する可能性がある。
  • 必須ではないオプションのサポート関数については、番号の範囲チェックのみを行い、シグネチャ検証はスキップされる。
