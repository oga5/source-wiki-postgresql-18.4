XLogRecordPageWithFreeSpace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/freespace.c:210

目的

WAL リカバリ処理中に、特定のヒープページの空き容量情報を Free Space Map (FSM) に記録する。

シグネチャまたは形状

void XLogRecordPageWithFreeSpace(RelFileLocator rlocator, BlockNumber heapBlk, Size spaceAvail)

入力と出力

  • **入力**:
  • rlocator: 対象リレーションの識別子。
  • heapBlk: 空き容量を更新する対象のヒープブロック番号。
  • spaceAvail: 記録する空き容量（バイト単位）。
  • **出力**: なし（副作用として FSM ページを更新）。

前提条件と不変条件

  • fsm_space_avail_to_cat により、spaceAvail は 256 段階のカテゴリ値に変換される。
  • FSM ページが存在しない場合は、XLogReadBufferExtended を用いてゼロ初期化されたページとして拡張される。
  • リカバリ中のため、MarkBufferDirtyHint ではなく MarkBufferDirty を使用して変更を確実に永続化する。

アルゴリズムまたは振る舞い

1. fsm_get_location を呼び出し、指定された heapBlk に対応する FSM 内の論理アドレスとスロットを特定する。 2. fsm_logical_to_physical で論理アドレスを物理ブロック番号へ変換する。 3. XLogReadBufferExtended で対象の FSM ページを読み込み、排他ロックを取得する。 4. ページが新規作成された場合は PageInit で初期化する。 5. fsm_set_avail を呼び出し、該当スロットの空き容量カテゴリを更新する。 6. 更新が発生した場合は MarkBufferDirty でバッファをダーティにし、ロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_xlog.c (複数の箇所で呼び出しを確認)
  • **呼び出し先**:
  • fsm_space_avail_to_cat
  • fsm_get_location
  • fsm_logical_to_physical
  • XLogReadBufferExtended
  • LockBuffer / UnlockReleaseBuffer
  • BufferGetPage / PageIsNew / PageInit
  • fsm_set_avail
  • MarkBufferDirty

関連データ構造

  • RelFileLocator: リレーションの物理的な場所を特定する構造体。
  • FSMAddress: FSM 木構造上の論理アドレス。
  • Buffer: バッファマネージャが管理するページバッファ。

失敗モードと境界条件

  • 対象の FSM ページが物理的に存在しない場合、XLogReadBufferExtended によりゼロ初期化されたページが新規作成される。
  • fsm_set_avail が変更なしと判断した場合（カテゴリ値が同じ場合）、バッファはダーティ化されない。

保守担当者向けメモ

  • 本関数はリカバリ専用の処理である。通常の実行時における FSM 更新には RecordPageWithFreeSpace を使用すること。
  • リカバリ中に MarkBufferDirty を使用するのは、チェックサム有効時に MarkBufferDirtyHint が変更を反映しない可能性があるためである。
