CheckSelectLocking

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:3402

目的

SQLの行ロック句（FOR UPDATE 等）が指定されたクエリに対し、そのクエリ構造がロック操作と両立可能かを検証し、不適切な場合にエラーを報告する。

シグネチャまたは形状

void CheckSelectLocking(Query *qry, LockClauseStrength strength)

入力と出力

  • **入力**:
  • qry: 検証対象の Query 構造体。
  • strength: 指定されたロックの強度（LockClauseStrength 型）。
  • **出力**: なし（正常時は終了）。エラー時は ereport(ERROR, ...) により処理を中断する。

前提条件と不変条件

  • strength は LCS_NONE 以外であること（Assert で検証）。

アルゴリズムまたは振る舞い

クエリ構造が以下のいずれかを含む場合、行ロックは論理的に適用できないためエラーを発生させる。 1. setOperations が存在（UNION/INTERSECT/EXCEPT）。 2. distinctClause が存在。 3. groupClause または groupingSets が存在。 4. havingQual が存在。 5. hasAggs が真（集約関数を含む）。 6. hasWindowFuncs が真（ウィンドウ関数を含む）。 7. hasTargetSRFs が真（ターゲットリストに集合を返す関数を含む）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c:2657
  • src/backend/parser/analyze.c:3476
  • **呼び出し先**:
  • LCS_asString(strength): ロック強度を文字列に変換するユーティリティ。
  • ereport(...): エラー報告用マクロ。

関連データ構造

  • Query: 解析済みのクエリツリー。
  • LockClauseStrength: ロックの強度を定義する列挙型。

失敗モードと境界条件

  • strength に LCS_NONE が渡された場合、アサーション違反によりデバッグビルドでは停止する。
  • 上記「アルゴリズム」に記載したクエリ構造の制約に抵触する場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。

保守担当者向けメモ

  • 本関数は、SQLの行ロック句がサポートされていない複雑なクエリ構造を排除するためのガードレールとして機能する。
  • 新しいクエリ構造や機能が追加され、それらがロック操作と競合する可能性がある場合は、本関数のチェックリストに条件を追加する必要がある。
