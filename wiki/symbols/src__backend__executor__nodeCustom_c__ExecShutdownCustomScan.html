ExecShutdownCustomScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCustom.c:220

目的

カスタムスキャン実行ノードの終了処理において、拡張機能側で定義されたシャットダウンロジックを呼び出すためのインターフェースを提供します。

シグネチャまたは形状

void ExecShutdownCustomScan(CustomScanState *node)

入力と出力

  • **入力**: CustomScanState *node - 実行中のカスタムスキャンノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は有効な CustomScanState 構造体を指している必要があります。
  • node->methods が初期化されており、拡張機能によって提供された CustomExecMethods 構造体が参照可能である必要があります。

アルゴリズムまたは振る舞い

1. node->methods から CustomExecMethods 構造体を取得します。 2. methods->ShutdownCustomScan が定義されている（NULLではない）場合、その関数ポインタを呼び出し、node を引数として渡します。 3. 定義されていない場合は何も行わず正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:809
  • **呼び出し先**: 拡張機能が CustomExecMethods 内で定義した ShutdownCustomScan コールバック関数。

関連データ構造

  • CustomScanState: カスタムスキャンの実行状態を保持する構造体。
  • CustomExecMethods: 拡張機能が実装するコールバック関数のテーブル。

失敗モードと境界条件

  • node->methods が NULL の場合、または node 自体が NULL の場合はセグメンテーション違反が発生する可能性があります。
  • methods->ShutdownCustomScan が NULL の場合、関数呼び出しはスキップされ、安全に終了します。

保守担当者向けメモ

  • 本関数は「薄いラッパー」です。実際の終了処理ロジックは拡張機能側に依存します。
  • 並列クエリ実行時、DSM（動的共有メモリ）の解放やクリーンアップが必要な場合は、拡張機能側で ShutdownCustomScan を適切に実装してください。
