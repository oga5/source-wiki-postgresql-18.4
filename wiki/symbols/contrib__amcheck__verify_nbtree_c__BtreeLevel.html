BtreeLevel

定義

  • 種別: struct
  • 定義箇所: contrib/amcheck/verify_nbtree.c:140

目的

B-Tree インデックスの検証プロセスにおいて、走査対象となる特定の階層（レベル）のメタデータを保持するために使用されます。

シグネチャまたは形状

typedef struct BtreeLevel
{
	uint32		level;           /* レベル番号（0はリーフページ） */
	BlockNumber leftmost;        /* 当該レベルの左端ブロック番号 */
	bool		istruerootlevel; /* メタページで真のルートと報告されているか */
} BtreeLevel;

入力と出力

本構造体はデータ保持用であり、関数ではありません。検証プロセスにおいて、現在の走査レベルを特定するための入力として各関数に渡されます。

前提条件と不変条件

  • level が 0 の場合、そのページはリーフページであることを示します。
  • leftmost は、そのレベルの走査を開始する起点となるブロック番号を指します。

アルゴリズムまたは振る舞い

本構造体は、bt_check_every_level におけるインデックス走査の制御状態を管理するために利用されます。各レベルの走査開始時に、この構造体を通じてレベル番号と開始位置が引き継がれます。

呼び出し元と呼び出し先

  • **呼び出し元**: bt_check_every_level および bt_check_level_from_leftmost がこの構造体を生成・利用しています。
  • **呼び出し先**: bt_check_level_from_leftmost は引数として BtreeLevel を受け取り、そのレベルの走査後に次のレベル（下位レベル）の BtreeLevel を返します。

関連データ構造

  • BtreeCheckState: 検証全体の状態を保持する構造体であり、BtreeLevel と組み合わせて使用されます。

失敗モードと境界条件

  • インデックス構造が破損している場合、leftmost に無効なブロック番号が格納される可能性があります。
  • istruerootlevel の判定はメタページの情報に依存するため、メタページ自体が破損している場合は誤ったフラグが設定される可能性があります。

保守担当者向けメモ

  • 本構造体は verify_nbtree.c 内でのみ使用される内部的な管理用データ構造です。
  • bt_check_level_from_leftmost の戻り値として次のレベルの情報を伝播させる設計となっているため、構造体のフィールドを変更する際は、この走査ロジックへの影響を考慮してください。
