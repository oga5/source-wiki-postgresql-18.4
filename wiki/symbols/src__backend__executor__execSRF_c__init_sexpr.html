init_sexpr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execSRF.c:695

目的

集合を返す関数（SRF）の実行状態を管理する SetExprState ノードを、初回使用時に初期化します。

シグネチャまたは形状

static void init_sexpr(Oid foid, Oid input_collation, Expr *node,
                       SetExprState *sexpr, PlanState *parent,
                       MemoryContext sexprCxt, bool allowSRF, bool needDescForSRF)

入力と出力

  • **入力**:
  • foid: 実行対象関数の OID。
  • input_collation: 入力照合順序。
  • node: 式ノード。
  • sexpr: 初期化対象の SetExprState 構造体。
  • parent: 親の PlanState（エラー報告用）。
  • sexprCxt: 状態保持用のメモリコンテキスト。
  • allowSRF: SRF の呼び出しを許可するかどうかのフラグ。
  • needDescForSRF: 戻り値のタプル記述子（TupleDesc）が必要かどうかのフラグ。
  • **出力**: なし（sexpr 構造体の各フィールドを更新）。

前提条件と不変条件

  • sexpr->args が設定済みであること。
  • sexpr->func.fn_retset と sexpr->funcReturnsSet が一致していること（Assert による検証）。

アルゴリズムまたは振る舞い

1. **権限チェック**: object_aclcheck を呼び出し、関数実行権限を確認。失敗時はエラーを投げる。 2. **引数数チェック**: 引数が FUNC_MAX_ARGS を超えていないか検証。 3. **関数情報設定**: fmgr_info_cxt を用いて関数情報をキャッシュし、fmgr_info_set_expr で式情報を関連付ける。 4. **パラメータ構造体初期化**: FunctionCallInfo を割り当て、InitFunctionCallInfoData で初期化。 5. **SRF 許可チェック**: 関数が集合を返すにもかかわらず allowSRF が false の場合、エラーを報告。 6. **タプル記述子の準備**: needDescForSRF が真かつ関数が集合を返す場合、get_expr_result_type で型情報を取得し、sexpr->funcResultDesc を設定する。

  • 複合型（Composite）: CreateTupleDescCopy でコピーを作成。
  • スカラー型: CreateTemplateTupleDesc で単一列の記述子を作成。
  • RECORD 型: funcResultDesc を NULL に設定。

    • 7. **状態リセット**: funcResultStore、funcResultSlot を NULL に、shutdown_reg を false に初期化。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execSRF.c 内の関数（行 81, 462, 470 など）。
  • **呼び出し先**: object_aclcheck, InvokeFunctionExecuteHook, fmgr_info_cxt, fmgr_info_set_expr, get_expr_result_type, CreateTupleDescCopy, CreateTemplateTupleDesc 等。

関連データ構造

  • SetExprState: SRF の実行状態を保持する主要構造体。
  • FunctionCallInfo: 関数呼び出しに必要な引数やメタデータ。
  • TupleDesc: 関数が返す行の構造定義。

失敗モードと境界条件

  • **権限不足**: ACLCHECK_OK 以外の場合、aclcheck_error を呼び出し終了。
  • **引数過多**: FUNC_MAX_ARGS を超える場合、ERRCODE_TOO_MANY_ARGUMENTS でエラー。
  • **SRF 禁止コンテキスト**: SRF を許可しない箇所で SRF を呼び出した場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー。

保守担当者向けメモ

  • 本関数は SetExprState の初期化を担うため、新しい実行状態フィールドを追加した場合は、本関数の末尾で適切に初期化されているか確認してください。
  • needDescForSRF が true の場合、メモリコンテキストの切り替え（MemoryContextSwitchTo）が発生するため、メモリリークやコンテキストの不整合に注意してください。
