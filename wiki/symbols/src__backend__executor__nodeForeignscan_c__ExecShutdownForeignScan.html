ExecShutdownForeignScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeForeignscan.c:440

目的

外部テーブルスキャン実行ノードに関連付けられたFDW（Foreign Data Wrapper）リソースのシャットダウン処理をトリガーします。

シグネチャまたは形状

void ExecShutdownForeignScan(ForeignScanState *node)

入力と出力

  • **入力**: ForeignScanState *node - 実行中の外部スキャンノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は有効な ForeignScanState を指している必要があります。
  • node->fdwroutine が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. node から FdwRoutine を取得します。 2. fdwroutine->ShutdownForeignScan が定義されている場合、その関数ポインタを呼び出し、FDW固有のシャットダウン処理を実行します。 3. 定義されていない場合は何も行いません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:806 にて呼び出されています。
  • **呼び出し先**: node->fdwroutine->ShutdownForeignScan（FDW実装依存の関数）。

関連データ構造

  • ForeignScanState: 外部スキャンノードの実行状態を保持する構造体。
  • FdwRoutine: 外部データソースへのアクセスに必要な関数ポインタ群を保持する構造体。

失敗モードと境界条件

  • fdwroutine が NULL の場合、あるいは ShutdownForeignScan 関数ポインタが NULL の場合、呼び出し元で適切に保護されている必要があります（本関数内では fdwroutine->ShutdownForeignScan の存在チェックのみ行っています）。

保守担当者向けメモ

  • 本関数は ExecEndForeignScan とは異なり、クエリ終了時だけでなく、並列クエリのワーカー終了時など、リソースの早期解放が必要なタイミングで呼び出されることを想定しています。
  • 実際のクリーンアップロジックはすべて fdwroutine 経由で外部モジュールに委譲されています。
