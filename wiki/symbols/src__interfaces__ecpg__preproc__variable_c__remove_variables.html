remove_variables

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/variable.c:297

目的

指定されたスコープ（brace_level）以上の深さで宣言された変数を、グローバルな変数リストおよびカーソルに関連付けられた引数リストから削除し、メモリを解放します。

シグネチャまたは形状

void remove_variables(int brace_level)

入力と出力

  • **入力**: brace_level (int) - 削除対象となるスコープの閾値。
  • **出力**: なし（副作用としてグローバル変数リスト allvariables およびカーソル内の引数リストが更新される）。

前提条件と不変条件

  • allvariables は、ECPGプリプロセッサで管理される全変数を保持するリンクリストであること。
  • 削除対象の変数がカーソル（cur）の argsinsert または argsresult に含まれている場合、それらの参照も適切に削除されること。

アルゴリズムまたは振る舞い

1. allvariables リストを走査し、各変数の brace_level が指定された値以上であるかを確認する。 2. 該当する場合、まず全カーソル（cur）を走査し、当該変数を参照している argsinsert および argsresult リストから該当エントリを検索・削除（free）する。 3. その後、allvariables リストから当該変数のノードを切り離し、ECPGfree_type を用いて型情報を解放し、変数名および変数構造体自体を free する。 4. 該当しない場合は、次のノードへ進む。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し箇所は提供されていませんが、preproc_extern.h で宣言されており、スコープ終了時に呼び出される設計です。
  • **呼び出し先**: ECPGfree_type（型情報の解放）、free（メモリ解放）。

関連データ構造

  • allvariables: 登録済み変数を保持するリンクリスト。
  • struct variable: 変数の属性（brace_level, type, name 等）を保持する構造体。
  • struct cursor: カーソル情報を保持し、引数リスト（argsinsert, argsresult）を管理する。

失敗モードと境界条件

  • allvariables が空の場合、ループは即座に終了する。
  • 削除対象の変数がどのカーソルにも参照されていない場合、カーソル走査はスキップされる。
  • brace_level が現在の最大スコープより小さい場合、該当するすべての変数が削除される。

保守担当者向けメモ

  • 本関数は brace_level に基づくスコープ管理の要です。カーソル内の引数リストを再帰的に走査して参照を削除する処理が含まれているため、カーソル関連のデータ構造に変更を加える際は本関数のロジックとの整合性に注意してください。
