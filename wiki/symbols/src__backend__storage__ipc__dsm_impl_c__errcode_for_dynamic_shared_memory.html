errcode_for_dynamic_shared_memory

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/ipc/dsm_impl.c:1046

目的

動的共有メモリ（DSM）操作中に発生したエラーに対し、PostgreSQLの標準的なエラーコードを割り当てるためのヘルパー関数です。

シグネチャまたは形状

static int errcode_for_dynamic_shared_memory(void)

入力と出力

  • **入力**: なし（グローバル変数 errno を参照）
  • **出力**: int 型のPostgreSQLエラーコード（ERRCODE_OUT_OF_MEMORY または errcode_for_file_access() が返す値）

前提条件と不変条件

  • 本関数は、共有メモリ操作に関連するシステムコールが失敗した直後に呼び出されることを前提としています。
  • errno が適切に設定されている必要があります。

アルゴリズムまたは振る舞い

1. 現在の errno を確認します。 2. errno が EFBIG（ファイルサイズ制限超過）または ENOMEM（メモリ不足）である場合、ERRCODE_OUT_OF_MEMORY を返します。 3. それ以外の場合は、errcode_for_file_access() を呼び出し、ファイルアクセスに関連する一般的なエラーコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/dsm_impl.c 内の各DSM実装関数（dsm_impl_posix, dsm_impl_mmap 等の操作に関連する箇所）から、エラー発生時に呼び出されます。
  • **呼び出し先**: errcode() および errcode_for_file_access() を呼び出します。

関連データ構造

  • なし

失敗モードと境界条件

  • errno の値に依存します。EFBIG や ENOMEM 以外のエラーが発生した場合は、ファイルシステム関連のエラーコードへフォールバックします。

保守担当者向けメモ

  • 本関数は static であり、dsm_impl.c 内でのみ使用されます。
  • 新たなDSM実装を追加し、特定のエラーハンドリングが必要になった場合は、この条件分岐を拡張してください。
