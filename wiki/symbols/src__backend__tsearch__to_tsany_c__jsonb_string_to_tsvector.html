jsonb_string_to_tsvector

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/to_tsany.c:313

目的

JSONB型の値から文字列要素のみを抽出し、現在の全文検索設定に基づいて tsvector 型へ変換します。

シグネチャまたは形状

Datum jsonb_string_to_tsvector(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される Jsonb オブジェクト。
  • **出力**: Datum 型として返される tsvector 構造体。

前提条件と不変条件

  • getTSCurrentConfig(true) により、現在のセッションにおける全文検索設定（Config OID）が取得可能であること。
  • 入力された Jsonb オブジェクトは、関数終了時に PG_FREE_IF_COPY によって適切にメモリ解放される。

アルゴリズムまたは振る舞い

1. getTSCurrentConfig(true) を呼び出し、現在の全文検索設定の OID を取得します。 2. jsonb_to_tsvector_worker を呼び出し、jtiString モードを指定して JSONB 内の文字列値を走査・変換します。 3. 入力された Jsonb オブジェクトがコピーである場合はメモリを解放します。 4. 生成された TSVector を PG_RETURN_TSVECTOR マクロを介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出されます（具体的な SQL 文脈はソースコード上では未定義）。
  • **呼び出し先**:
  • getTSCurrentConfig
  • jsonb_to_tsvector_worker

関連データ構造

  • Jsonb: 入力データ構造。
  • TSVector: 出力データ構造。

失敗モードと境界条件

  • getTSCurrentConfig が設定を見つけられない場合、例外が発生する可能性があります。
  • 入力された Jsonb が空、あるいは文字列を含まない場合、空の tsvector が生成されます。

保守担当者向けメモ

  • 本関数は jsonb_to_tsvector_worker のラッパーであり、特定の抽出モード（jtiString）を固定して利用しています。
  • JSONB 構造の走査ロジックに変更を加える場合は、jsonb_to_tsvector_worker の実装を確認してください。
