parse_analyze_withcb

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:185

目的

生の解析ツリー（RawStmt）を、オプティマイザーやエグゼキューターが処理可能なクエリツリー（Query）へ変換するセマンティック解析のエントリーポイントです。コールバック関数を用いた柔軟なパーサー設定をサポートします。

シグネチャまたは形状

Query *parse_analyze_withcb(RawStmt *parseTree, const char *sourceText,
                            ParserSetupHook parserSetup,
                            void *parserSetupArg,
                            QueryEnvironment *queryEnv)

入力と出力

  • **入力**:
  • parseTree: 構文解析済みの生の解析ツリー。
  • sourceText: SQL 文のソース文字列（必須）。
  • parserSetup: ParseState を初期化するためのコールバック関数。
  • parserSetupArg: コールバックに渡す引数。
  • queryEnv: クエリ実行環境。
  • **出力**:
  • 変換された Query 構造体へのポインタ。

前提条件と不変条件

  • sourceText は NULL であってはならない。
  • parserSetup コールバックは、ParseState の初期化を正しく行う必要がある。

アルゴリズムまたは振る舞い

1. make_parsestate を呼び出し、ParseState を初期化する。 2. sourceText と queryEnv を ParseState に設定する。 3. parserSetup コールバックを実行し、パーサーの状態を構成する。 4. transformTopLevelStmt を呼び出し、解析ツリーをクエリツリーへ変換する。 5. IsQueryIdEnabled() が真の場合、JumbleQuery を呼び出してクエリ ID を生成する。 6. post_parse_analyze_hook が定義されていれば、解析後のフック処理を実行する。 7. free_parsestate で ParseState を解放する。 8. pgstat_report_query_id を呼び出し、クエリ ID を統計システムへ報告する。 9. 完成した Query を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c:776
  • **呼び出し先**:
  • make_parsestate
  • parserSetup (コールバック)
  • transformTopLevelStmt
  • JumbleQuery
  • post_parse_analyze_hook (フック)
  • free_parsestate
  • pgstat_report_query_id

関連データ構造

  • RawStmt: 入力となる解析ツリー。
  • ParseState: 解析中の状態を保持する構造体。
  • Query: 出力となるクエリツリー。
  • JumbleState: クエリのハッシュ化（Jumble）状態。

失敗モードと境界条件

  • sourceText が NULL の場合、Assert により異常終了する。
  • parserSetup コールバック内でのエラーは、PostgreSQL のエラーハンドリング機構により上位へ伝播する。

保守担当者向けメモ

  • 本関数は parse_analyze_fixedparams や parse_analyze_varparams と同様の解析パイプラインを共有していますが、コールバックによる設定の注入が可能です。
  • post_parse_analyze_hook は外部モジュールが解析結果に介入するための重要な接点です。変更時はこのフックのシグネチャとの整合性に注意してください。
