DropSetting

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_db_role_setting.c:169

目的

システムカタログ pg_db_role_setting から、指定されたデータベースまたはロールに関連付けられた設定エントリを削除します。

シグネチャまたは形状

void DropSetting(Oid databaseid, Oid roleid)

入力と出力

  • **入力**:
  • databaseid: 削除対象のデータベースOID。InvalidOid を指定可能。
  • roleid: 削除対象のロールOID。InvalidOid を指定可能。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • RowExclusiveLock を使用して pg_db_role_setting カタログをロックします。
  • databaseid または roleid の少なくとも一方が有効なOIDである必要があります（両方が InvalidOid の場合、検索条件が空となり意図しない動作となる可能性があります）。

アルゴリズムまたは振る舞い

1. pg_db_role_setting カタログを RowExclusiveLock モードで開きます。 2. databaseid および roleid が有効な場合、それぞれを検索キー（setdatabase, setrole）として設定します。 3. table_beginscan_catalog を使用してカタログをスキャンします。 4. スキャンでヒットしたすべてのタプルに対し、CatalogTupleDelete を実行して削除します。 5. スキャンを終了し、カタログを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c: データベース削除時に呼び出し。
  • src/backend/commands/user.c: ロール削除時に呼び出し。
  • **呼び出し先**:
  • table_open, table_close: カタログの開閉。
  • ScanKeyInit: 検索キーの初期化。
  • table_beginscan_catalog, heap_getnext, table_endscan: カタログスキャン。
  • CatalogTupleDelete: タプルの削除。

関連データ構造

  • pg_db_role_setting: 操作対象のシステムカタログ。
  • ScanKeyData: スキャン条件を保持する構造体。

失敗モードと境界条件

  • databaseid と roleid の両方に InvalidOid を渡した場合の挙動については、ソースコード上での明示的なガードは存在しません。
  • カタログ操作中にロック競合が発生した場合、標準的な PostgreSQL のロックメカニズムに従います。

保守担当者向けメモ

  • 本関数は pg_db_role_setting のエントリを物理的に削除するのみです。設定の適用やメモリ上のキャッシュ整合性については、呼び出し側または GUC 管理モジュールが関与します。
  • setconfig フィールドは TOAST テーブルを使用するため、削除処理は TOAST データのクリーンアップを伴う可能性があります。
