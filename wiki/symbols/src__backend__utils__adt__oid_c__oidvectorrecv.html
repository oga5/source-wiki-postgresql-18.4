oidvectorrecv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:211

目的

外部バイナリ形式のデータを oidvector 型へ変換します。

シグネチャまたは形状

Datum oidvectorrecv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して StringInfo バッファ（バイナリデータ）を受け取ります。
  • **出力**: 変換された oidvector へのポインタを Datum として返します。

前提条件と不変条件

  • array_recv を呼び出すために、InitFunctionCallInfoData を使用して一時的な関数呼び出し情報を構築する必要があります。
  • 変換後の oidvector は以下の制約を満たす必要があります。
  • 1次元配列であること。
  • NULL要素を含まないこと。
  • 要素型が OIDOID であること。
  • インデックスの下限が 0 であること。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFO を使用して、array_recv 呼び出し用の関数呼び出し情報を初期化します。 2. array_recv を呼び出し、バイナリデータを汎用配列としてデコードします。 3. デコードされた結果に対して、oidvector としての妥当性チェック（次元数、NULL有無、要素型、下限インデックス）を行います。 4. チェックに失敗した場合は ERRCODE_INVALID_BINARY_REPRESENTATION エラーを発生させます。 5. 妥当な場合は、変換された oidvector ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での明示的な呼び出し元は確認されていません。
  • **呼び出し先**: array_recv（src/backend/utils/adt/arrayfuncs.c に依存）。

関連データ構造

  • oidvector: Oid の固定長配列を表現する構造体。
  • StringInfo: バイナリ通信用のメッセージフォーマット。
  • FunctionCallInfoData: 関数呼び出し情報を保持する構造体。

失敗モードと境界条件

  • array_recv が失敗した場合、または妥当性チェック（次元数、NULL、型、下限）に違反した場合は ereport(ERROR, ...) により処理が中断されます。

保守担当者向けメモ

  • oidvector は歴史的な経緯により、一般的な配列型とは異なり「インデックスの下限が0」であるという特殊な制約があります。
  • array_recv が fcinfo->flinfo->fn_extra を使用してデータをキャッシュするため、単なる DirectFunctionCall3 ではなく、自身の flinfo を渡す実装になっています。
