write_inheritable_socket

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/launch_backend.c:844

目的

Windows環境（EXEC_BACKENDモード）において、親プロセスが保持するソケットを子プロセスへ継承可能にするため、ソケット情報を複製して格納します。

シグネチャまたは形状

static bool write_inheritable_socket(InheritableSocket *dest, SOCKET src, pid_t childpid)

入力と出力

  • **入力**:
  • dest: 複製されたソケット情報および元ソケットIDを格納する InheritableSocket 構造体へのポインタ。
  • src: 複製対象の SOCKET ハンドル。
  • childpid: ソケットを継承する対象の子プロセスのプロセスID。
  • **出力**:
  • 成功時は true、失敗時は false を返します。

前提条件と不変条件

  • src が 0 または PGINVALID_SOCKET の場合、複製処理はスキップされ、dest->origsocket に値が代入された後 true が返されます。
  • WSADuplicateSocket が失敗した場合、エラーログ（LOGレベル）が出力されます。

アルゴリズムまたは振る舞い

1. dest->origsocket に src を保存します。 2. src が有効なソケットである場合、WSADuplicateSocket を呼び出し、指定された childpid に対してソケット情報を複製します。 3. 複製に成功すれば true を返し、失敗した場合は WSAGetLastError() を用いてエラー詳細をログ出力し、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c 内のプロセス起動処理（save_backend_variables 等）から呼び出されます。
  • **呼び出し先**: Windows API の WSADuplicateSocket および WSAGetLastError を呼び出します。

関連データ構造

  • InheritableSocket: ソケットの複製情報（WSAPROTOCOL_INFO）と元のソケットIDを保持する構造体。

失敗モードと境界条件

  • WSADuplicateSocket が失敗した場合、プロセス起動処理が中断される可能性があります。
  • src が無効な値である場合、複製は行われず、単なるIDのコピーとして扱われます。

保守担当者向けメモ

  • 本関数は EXEC_BACKEND モード（主にWindows環境）でのみ意味を持ちます。Unix系環境ではマクロ定義により単純な代入操作に置き換えられます。
  • WSADuplicateSocket の呼び出しには対象プロセスのIDが必要であり、プロセス生成のタイミングと整合している必要があります。
