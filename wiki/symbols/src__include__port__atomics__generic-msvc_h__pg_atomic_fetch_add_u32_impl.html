pg_atomic_fetch_add_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic-msvc.h:67

目的

MSVC環境において、32ビット符号なし整数に対してアトミックな加算を行い、加算前の値を返す。

シグネチャまたは形状

static inline uint32 pg_atomic_fetch_add_u32_impl(volatile pg_atomic_uint32 *ptr, int32 add_)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint32 構造体への揮発性ポインタ。
  • add_: 加算する値（int32）。
  • **出力**: 加算操作が行われる直前の ptr->value の値。

前提条件と不変条件

  • ptr は有効な pg_atomic_uint32 構造体を指している必要がある。
  • 本関数は atomics.h を経由して呼び出されることを前提としている。

アルゴリズムまたは振る舞い

1. Windows API の InterlockedExchangeAdd を呼び出す。 2. ptr->value に対して add_ をアトミックに加算する。 3. 加算前の値を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics.h:369 にて直接呼び出されている。また、src/include/port/atomics/generic.h 内の複数の関数から間接的に利用されている。
  • **呼び出し先**: Windows の InterlockedExchangeAdd（<intrin.h> で定義）。

関連データ構造

  • pg_atomic_uint32: アトミック操作対象の値を保持する構造体。

失敗モードと境界条件

  • InterlockedExchangeAdd はメモリバリアを内包する。
  • 整数オーバーフロー時の挙動は、標準的な符号付き整数の加算ルールに従う。

保守担当者向けメモ

  • 本関数は MSVC ビルド専用の実装である。
  • Interlocked ファミリー関数はメモリバリアを内包するため、呼び出し側で追加のバリアが必要かどうかは、PostgreSQL のアトミック操作のメモリモデル要件に従うこと。
