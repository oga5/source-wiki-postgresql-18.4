backup_manifest_info

定義

  • **種別**: struct
  • **定義箇所**: src/include/backup/backup_manifest.h:27

目的

バックアップマニフェストの構築および管理に必要なコンテキスト情報を保持する構造体です。メモリ消費を抑えるため、マニフェストの内容を一時ファイル（BufFile）へ逐次書き出す仕組みを提供します。

シグネチャまたは形状

typedef struct backup_manifest_info
{
    BufFile    *buffile;            /* マニフェスト書き出し用の一時ファイル */
    pg_checksum_type checksum_type; /* マニフェスト全体のチェックサムアルゴリズム */
    pg_cryptohash_ctx *manifest_ctx; /* ハッシュ計算用コンテキスト */
    uint64      manifest_size;      /* マニフェストの現在のサイズ */
    bool        force_encode;       /* エンコーディング強制フラグ */
    bool        first_file;         /* JSON配列の先頭要素判定用フラグ */
} backup_manifest_info;

入力と出力

  • **入力**: InitializeBackupManifest を通じて初期化され、AddFileToBackupManifest や AddWALInfoToBackupManifest を介してファイルやWALのメタデータが入力されます。
  • **出力**: SendBackupManifest を通じて、構築されたマニフェストが最終的な出力先（bbsink）へ送信されます。

前提条件と不変条件

  • マニフェストのチェックサムには、データファイルのアルゴリズム設定に関わらず、常に SHA-256 が使用されます。
  • マニフェストが不要な場合でも、NULL ポインタチェックを削減する設計のため、本構造体は初期化される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: InitializeBackupManifest で構造体をゼロクリアし、一時ファイルを作成します。 2. **データ追加**: AddFileToBackupManifest 等により、JSON 形式のメタデータが buffile に追記されます。 3. **終了処理**: SendBackupManifest が一時ファイルを読み込み、最終的なチェックサムを付与して出力先へ転送します。 4. **解放**: FreeBackupManifest により、ハッシュ計算コンテキスト等のリソースが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c のバックアップメインループから、ファイル追加やWAL情報追加の引数として参照されます。
  • **呼び出し先**: src/backend/backup/backup_manifest.c 内の各関数（InitializeBackupManifest, AddFileToBackupManifest, SendBackupManifest 等）が本構造体を操作します。

関連データ構造

  • BufFile: 一時ファイル操作用。
  • pg_cryptohash_ctx: チェックサム計算用。
  • bbsink: マニフェストの最終的な出力先インターフェース。

失敗モードと境界条件

  • マニフェストは巨大になる可能性があるため、メモリではなく一時ファイルに書き出すことでメモリ不足を回避しています。
  • 失敗時の詳細なエラーハンドリングについては、各操作関数（AddFileToBackupManifest 等）の戻り値および bbsink のエラー処理に依存します。

保守担当者向けメモ

  • 本構造体は src/backend/backup/backup_manifest.c で実装されるマニフェスト生成ロジックの核となります。
  • 変更を加える際は、basebackup.c での呼び出し順序と、bbsink へのデータ転送フローに影響がないか確認してください。
