spg_text_config

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgtextproc.c:95

目的

SP-GiST インデックスにおける text 型の構成情報を設定し、インデックスの動作特性を定義します。

シグネチャまたは形状

Datum spg_text_config(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgConfigOut 構造体へのポインタを受け取ります。
  • **出力**: なし（PG_RETURN_VOID() を返します）。spgConfigOut 構造体の各フィールドを更新することで設定を伝達します。

前提条件と不変条件

  • spgConfigOut 構造体が有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. cfg->prefixType を TEXTOID に設定し、接頭辞の型をテキスト型とします。 2. cfg->labelType を INT2OID に設定し、ノードラベルの型を 2 バイト整数とします。 3. cfg->canReturnData を true に設定し、インデックスから元のデータを復元可能にします。 4. cfg->longValuesOK を true に設定し、長い値の格納を許可します（サフィックス処理により短縮されます）。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は特定されていませんが、PostgreSQL の SP-GiST インデックス構築プロセスから呼び出されるインターフェース関数です。
  • **呼び出し先**: なし。

関連データ構造

  • spgConfigOut: SP-GiST インデックスの構成情報を保持する構造体。

失敗モードと境界条件

  • 入力引数が不正なポインタである場合、メモリ保護違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は text_ops インデックスメソッドの構成を定義する重要なエントリポイントです。設定変更時は、spg_text_choose や spg_text_picksplit などの関連処理との整合性に注意してください。
