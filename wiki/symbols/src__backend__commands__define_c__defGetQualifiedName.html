defGetQualifiedName

定義

  • 種別: function
  • 定義箇所: src/backend/commands/define.c:238

目的

DefElem ノードから、名前空間やオブジェクト名を表す List 構造体を抽出する。

シグネチャまたは形状

List *defGetQualifiedName(DefElem *def)

入力と出力

  • **入力**: DefElem *def （解析対象の定義要素）
  • **出力**: List * （名前のリスト）。成功時は TypeName->names、List そのもの、または String を単一要素リスト化したものを返す。

前提条件と不変条件

  • def および def->arg は有効なポインタである必要がある。
  • def->arg が NULL の場合、エラーが発生する。

アルゴリズムまたは振る舞い

1. def->arg が NULL の場合、ERRCODE_SYNTAX_ERROR でエラーを報告する。 2. nodeTag(def->arg) に基づき、以下のいずれかの型を判定して処理する：

  • T_TypeName: TypeName 構造体の names フィールドを返す。
  • T_List: List ポインタをそのまま返す。
  • T_String: 互換性のため、文字列を単一要素のリストとして返す。

    • 3. 上記以外の型が指定された場合、ERRCODE_SYNTAX_ERROR でエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/aggregatecmds.c: 集約関数の各種属性（transfunc, finalfunc 等）の取得に使用。
  • src/backend/commands/collationcmds.c: 照合順序の OID 取得に使用。
  • **呼び出し先**:
  • list_make1: T_String の場合に呼び出される。
  • ereport: エラー報告時に呼び出される。

関連データ構造

  • DefElem: 構文解析結果を保持するノード。
  • TypeName: 型名やオブジェクト名を保持するノード。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • def->arg が NULL の場合: ERRCODE_SYNTAX_ERROR を送出。
  • def->arg が T_TypeName, T_List, T_String 以外の型である場合: ERRCODE_SYNTAX_ERROR を送出。

保守担当者向けメモ

  • 本関数は DefElem の arg フィールドを特定の型へ安全に変換するアクセサである。
  • T_String の扱いは後方互換性のための措置であるため、将来的な仕様変更の際には注意が必要。
