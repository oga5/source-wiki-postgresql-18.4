SS_identify_outer_params

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/subselect.c:2183

目的

現在のクエリレベルから見て上位のクエリレベルで利用可能なパラメータ（Param）を特定し、root->outer_params に格納する。

シグネチャまたは形状

void SS_identify_outer_params(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（現在のクエリレベルの計画情報）。
  • **出力**: なし（副作用として root->outer_params に Bitmapset を設定する）。

前提条件と不変条件

  • root->glob->paramExecTypes が NIL の場合、パラメータは存在しないため処理を即座に終了する。
  • 親クエリが存在しない場合、outer_params は空（NULL）となる。

アルゴリズムまたは振る舞い

1. root->glob->paramExecTypes を確認し、パラメータが一つも割り当てられていない場合は早期リターンする。 2. root->parent_root を辿り、最上位の親クエリまで順次走査する。 3. 各親クエリ（proot）において以下を収集し、outer_params に追加する:

  • proot->plan_params: 下位レベルから要求されたパラメータ。
  • proot->init_plans: 各 SubPlan が出力する setParam（InitPlan の実行結果として利用可能なパラメータ）。
  • proot->wt_param_id: 再帰的クエリにおける worktable の ID（存在する場合）。

    • 4. 最終的に収集した Bitmapset を root->outer_params に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planagg.c:430
  • src/backend/optimizer/plan/planner.c:1280
  • **呼び出し先**:
  • bms_add_member（ビットマップセット操作）
  • lfirst / lfirst_int（リスト走査）

関連データ構造

  • PlannerInfo: クエリの計画情報を保持する構造体。
  • PlannerParamItem: plan_params リストに含まれるパラメータ情報。
  • SubPlan: init_plans リストに含まれるサブプラン。
  • Bitmapset: 利用可能なパラメータ ID を保持する集合。

失敗モードと境界条件

  • 親クエリが存在しない場合、ループは実行されず outer_params は NULL となる。
  • paramExecTypes が空の場合、処理はスキップされる。

保守担当者向けメモ

  • 本関数は、サブクエリが親クエリのどのパラメータに依存できるかを決定する重要なステップである。
  • proot->wt_param_id のような再帰クエリ特有のパラメータや、init_plans の出力パラメータが正しく反映されているか確認すること。
  • パラメータ ID の管理は setrefs.c と密接に関係しているため、変更時は整合性に注意が必要である。
