gbt_biteq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:41

目的

BIT 型および VARBIT 型の2つの値が等しいかどうかを判定し、GiST インデックスの比較処理をサポートします。

シグネチャまたは形状

static bool gbt_biteq(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のビット列データへのポインタ。
  • collation: 照合順序（本関数では未使用）。
  • flinfo: 関数呼び出し情報（本関数では未使用）。
  • **出力**:
  • bool: a と b が等しい場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、PostgreSQL の BIT/VARBIT 型として有効なデータ構造を指している必要があります。
  • 内部的に biteq 関数を呼び出すため、PostgreSQL の標準的なビット列比較ルールに従います。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力された void * ポインタを PostgreSQL の Datum 型に変換します。 2. DirectFunctionCall2 を介して、PostgreSQL 標準の biteq 関数を呼び出します。 3. biteq の結果（Datum）を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: biteq (PostgreSQL 標準のビット列比較関数)
  • **呼び出し元**: ソースコード内の contrib/btree_gist/btree_bit.c:115 にて、比較関数テーブルの一部として参照されています。

関連データ構造

  • Datum: PostgreSQL の汎用データ表現。
  • FmgrInfo: 関数マネージャ情報。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、DirectFunctionCall2 経由で例外が発生する可能性があります。
  • 比較対象のビット列の長さが異なる場合、標準の biteq の仕様に基づき false が返されます。

保守担当者向けメモ

  • 本関数は DirectFunctionCall2 を使用したラッパーであり、実質的な比較ロジックは src/backend/utils/adt/varbit.c の biteq に依存しています。
  • GiST インデックスの整合性チェックにおいて、等価判定が必要な場面で使用されます。
