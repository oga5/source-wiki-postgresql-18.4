BrinOpaque

定義

  • 種別: struct
  • 定義箇所: src/backend/access/brin/brin.c:202

目的

BRIN（Block Range Index）スキャン実行時の内部状態を保持するための構造体です。インデックススキャン中に必要なメタデータやアクセス用オブジェクトを管理します。

シグネチャまたは形状

typedef struct BrinOpaque
{
	BlockNumber bo_pagesPerRange;
	BrinRevmap *bo_rmAccess;
	BrinDesc   *bo_bdesc;
} BrinOpaque;

入力と出力

  • **入力**: スキャン開始時に palloc_object(BrinOpaque) を介してメモリが割り当てられ、scan->opaque に格納されます。
  • **出力**: スキャン終了まで、インデックスの範囲設定や Revmap へのアクセスポイントとして利用されます。

前提条件と不変条件

  • scan->opaque にキャストして格納されることが前提です。
  • bo_rmAccess は Revmap へのアクセスを管理し、bo_bdesc はインデックスの記述子を保持します。

アルゴリズムまたは振る舞い

  • スキャン実行時、scan->opaque から本構造体を取り出し、bo_rmAccess を通じてページ範囲の要約情報を取得します。
  • bo_pagesPerRange は、インデックスが対象とするページ範囲のサイズを保持し、スキャン時の範囲計算に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c 内のインデックススキャン関連関数（bringetbitmap 等）から参照されます。
  • **呼び出し先**: 構造体内部の bo_rmAccess（Revmap）および bo_bdesc（BrinDesc）を介して、BRIN の下位層（Revmap 管理やタプルデフォーム処理）が呼び出されます。

関連データ構造

  • BrinRevmap: ページ範囲とインデックス位置の対応管理に使用。
  • BrinDesc: インデックスの属性情報やサポート関数を保持。
  • IndexScanDesc: 本構造体が opaque フィールドとして格納される親構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時に palloc_object がエラーを発生させる可能性があります。
  • scan->opaque が正しく初期化されていない状態でキャストしてアクセスした場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体はスキャン実行時のコンテキストを保持する重要な役割を担います。
  • brin.c 内の bringetbitmap 等の関数で、scan->opaque からのキャストが正しく行われているか確認してください。
