JsonHashEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonfuncs.c:147

目的

JSONオブジェクトのフィールド名と値をハッシュテーブル内で管理するためのエントリ構造体です。

シグネチャまたは形状

typedef struct JsonHashEntry
{
    char        fname[NAMEDATALEN]; /* hash key (MUST BE FIRST) */
    char       *val;
    JsonTokenType type;
} JsonHashEntry;

入力と出力

  • **入力**: フィールド名（fname）、値のポインタ（val）、およびトークン型（type）。
  • **出力**: ハッシュテーブルの検索や挿入操作において、キーに対応する値と型情報を保持します。

前提条件と不変条件

  • fname は構造体の先頭に配置される必要があります（ハッシュキーとして機能するため）。
  • fname は NAMEDATALEN のサイズ制限を受けます。

アルゴリズムまたは振る舞い

  • ハッシュテーブル（hash_search）の要素として使用されます。
  • JSONのフィールド名をキーとして、対応する値の文字列ポインタとトークン型を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonfuncs.c 内のハッシュテーブル操作関数（hash_search を使用する箇所）から参照されます。
  • **呼び出し先**: 特定の関数呼び出し関係はソースコードから明示されていませんが、ハッシュテーブルの制御構造（ctl.entrysize）の設定時にサイズが参照されます。

関連データ構造

  • JsonTokenType: JSONのトークン型を定義する列挙型。
  • hash_search: PostgreSQLのハッシュテーブル検索関数。

失敗モードと境界条件

  • fname が NAMEDATALEN を超える場合、切り詰めやオーバーフローのリスクがあります。
  • ハッシュテーブルの初期化時に entrysize として sizeof(JsonHashEntry) が正しく指定されていない場合、メモリ破壊が発生する可能性があります。

保守担当者向けメモ

  • 本構造体はハッシュテーブルのキーとして fname を使用するため、構造体のメモリレイアウトを変更する際は fname が先頭にあることを維持してください。
  • val は動的に割り当てられたメモリを指す可能性があるため、ハッシュテーブル破棄時のメモリ管理に注意してください。
