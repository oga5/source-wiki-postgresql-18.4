ArchiverOutput

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.h:175

目的

アーカイブ処理において、現在出力中のデータ形式を識別するためのフラグを定義します。

シグネチャまたは形状

typedef enum
{
	OUTPUT_SQLCMDS = 0,
	OUTPUT_COPYDATA,
	OUTPUT_OTHERDATA,
} ArchiverOutput;

入力と出力

  • **値**:
  • OUTPUT_SQLCMDS: 一般的なSQLコマンドを出力中。
  • OUTPUT_COPYDATA: COPY データを出力中。
  • OUTPUT_OTHERDATA: INSERT コマンドとしてデータを出力中。

前提条件と不変条件

  • ArchiveHandle 構造体内の outputKind フィールドとして保持されます。
  • アーカイブの書き込み状態を追跡するために使用されます。

アルゴリズムまたは振る舞い

アーカイブ処理中に、現在どのような形式でデータが書き込まれているかを追跡し、適切な出力制御を行うために参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_archiver.h 内の ArchiveHandle 構造体（outputKind フィールド）で参照されています。
  • **呼び出し先**: 提供されたソース情報からは、具体的な関数レベルの呼び出し先は特定できません。

関連データ構造

  • ArchiveHandle: この列挙型をメンバとして保持するメイン構造体。

失敗モードと境界条件

  • 定義外の値が outputKind に代入された場合の挙動については、本ソース excerpt からは不明です。

保守担当者向けメモ

  • 本シンボルは、pg_dump が出力するデータの種類を切り替える際のステート管理に使用されます。機能追加や出力形式の拡張を行う際は、この列挙型に新しい値を追加し、各アーカイブ形式の実装（pg_backup_custom.c 等）でのハンドリングを確認してください。
