sepgsql_attribute_relabel

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/relation.c:164

目的

テーブルまたはパーティションテーブルのカラムに対する SECURITY LABEL コマンド実行時に、SELinuxポリシーに基づいた権限チェックを行う。

シグネチャまたは形状

void sepgsql_attribute_relabel(Oid relOid, AttrNumber attnum, const char *seclabel)

入力と出力

  • **入力**:
  • relOid: 対象リレーションのOID。
  • attnum: 対象カラムの属性番号。
  • seclabel: 設定しようとしている新しいセキュリティラベル文字列。
  • **出力**: なし（成功時は正常終了、権限不足や対象外オブジェクトの場合はエラーをスロー）。

前提条件と不変条件

  • 対象リレーションが通常のテーブル（RELKIND_RELATION）またはパーティションテーブル（RELKIND_PARTITIONED_TABLE）であること。それ以外の場合は ERRCODE_WRONG_OBJECT_TYPE でエラーとなる。

アルゴリズムまたは振る舞い

1. get_rel_relkind を用いてリレーション種別を検証する。 2. getObjectIdentity を使用して、対象カラムを特定するための識別名（audit_name）を取得する。 3. sepgsql_avc_check_perms を呼び出し、対象カラムに対する db_column:{setattr relabelfrom} 権限を検証する。 4. sepgsql_avc_check_perms_label を呼び出し、新しいラベルに対する db_column:{relabelto} 権限を検証する。 5. 最後に pfree で audit_name を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/label.c:512 から呼び出されていることが確認されている。
  • **呼び出し先**:
  • get_rel_relkind, getObjectIdentity, ereport, pfree (PostgreSQL内部API)
  • sepgsql_avc_check_perms, sepgsql_avc_check_perms_label (SELinux権限チェック用内部関数)

関連データ構造

  • ObjectAddress: 対象オブジェクト（クラスID、オブジェクトID、サブID）を保持するために使用。

失敗モードと境界条件

  • 対象がテーブルやパーティションテーブル以外のリレーション（ビューやシーケンス等）である場合、エラーが発生する。
  • SELinuxポリシーにより必要な権限（setattr, relabelfrom, relabelto）が許可されていない場合、権限チェック関数内でエラーが発生する。

保守担当者向けメモ

  • 本関数は SECURITY LABEL コマンドのフックとして機能する。
  • カラムに対するラベル変更は、SELinuxの db_column クラスとして扱われる。
  • sepgsql_avc_check_perms_label 呼び出し時に SEPG_DB_PROCEDURE__RELABELTO が使用されているが、これは db_column クラスに対するラベル変更の権限チェックとして機能する。
