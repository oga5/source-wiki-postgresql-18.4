initialize

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:739

目的

正規表現マッチングの開始時に、DFA（決定性有限オートマトン）の初期状態セットを準備または再利用する。

シグネチャまたは形状

static struct sset *initialize(struct vars *v, struct dfa *d, chr *start)

入力と出力

  • **入力**:
  • v: 正規表現の実行コンテキスト（struct vars）。
  • d: DFA 構造体（struct dfa）。
  • start: マッチング開始位置を示す文字ポインタ。
  • **出力**:
  • 成功時: 初期状態セット（struct sset *）。
  • 失敗時: NULL（getvacant が失敗した場合）。

前提条件と不変条件

  • d->cnfa->pre（開始状態）と d->cnfa->post（終了状態）は等しくないこと。
  • d->ssets キャッシュが初期化済みであること。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: 既存のキャッシュ（d->ssets[0]）が STARTER フラグを持ち、有効であればそれを再利用する。 2. **新規構築**: キャッシュが利用できない場合、getvacant を呼び出して新しい状態セットを確保する。

  • 状態セットのビットマップをクリアし、開始状態（d->cnfa->pre）をセットする。
  • ハッシュ値を計算し、STARTER | LOCKED | NOPROGRESS フラグを付与する。

    • 3. **状態リセット**: 全てのキャッシュエントリの lastseen を NULL にリセットする。 4. **ポインタ更新**: lastseen を start に設定し、d->lastpost および d->lastnopr を NULL に初期化して戻り値として初期状態セットを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコードの文脈からは直接の呼び出し元は特定できません。
  • **呼び出し先**:
  • getvacant: 新しい状態セットの確保。
  • BSET: 状態ビットのセット。
  • HASH: 状態セットのハッシュ計算。

関連データ構造

  • struct dfa: DFA の実行状態を管理。
  • struct sset: DFA の状態セット（キャッシュエントリ）。
  • struct vars: 正規表現の実行変数。

失敗モードと境界条件

  • getvacant が空きスロットを見つけられず NULL を返した場合、本関数も NULL を返す。
  • d->nssused が 0 の場合、必ず新規構築パスへ進む。

保守担当者向けメモ

  • 本関数は regexec.c からインクルードされる rege_dfa.c 内で定義されており、DFA 実行の起点となります。
  • lastseen の設定は、マッチングの進捗管理において重要な役割を果たします。
