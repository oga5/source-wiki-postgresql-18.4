bthandler

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtree.c:114

目的

PostgreSQL の B-tree インデックスアクセスメソッド（AM）の機能を定義する IndexAmRoutine 構造体を初期化し、システムに登録するためのハンドラ関数です。

シグネチャまたは形状

Datum bthandler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（標準的な PostgreSQL 関数呼び出し規約）
  • **出力**: Datum 型として、B-tree インデックスの全操作関数ポインタを保持する IndexAmRoutine 構造体へのポインタを返します。

前提条件と不変条件

  • makeNode(IndexAmRoutine) によりメモリが確保されること。
  • B-tree インデックスに必要なすべての関数ポインタ（ambuild, btinsert, btgettuple 等）が正しく割り当てられていること。

アルゴリズムまたは振る舞い

1. IndexAmRoutine 構造体を生成します。 2. B-tree の特性（順序保持、一意性制約、並列スキャン対応など）をフラグとして設定します。 3. B-tree の各操作に対応する関数（btinsert, btgettuple, btbulkdelete 等）を構造体のメンバに登録します。 4. 最後に、構築した amroutine を PG_RETURN_POINTER マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックス登録プロセス（詳細はソースコード外のため不明）。
  • **呼び出し先**: btbuild, btinsert, btbulkdelete, btvacuumcleanup, btbeginscan, btgettuple 等、B-tree の各操作関数。

関連データ構造

  • IndexAmRoutine: インデックスアクセスメソッドのインターフェースを定義する構造体。

失敗モードと境界条件

  • メモリ不足により makeNode が失敗した場合、PostgreSQL のエラーハンドリング機構により例外が発生します。

保守担当者向けメモ

  • 本関数は B-tree インデックスの「公開インターフェース」の定義箇所です。新しいインデックス機能を追加する場合や、既存の操作関数を変更する場合は、本関数内の関数ポインタ割り当てを更新する必要があります。
  • amparallelvacuumoptions には VACUUM_OPTION_PARALLEL_BULKDEL および VACUUM_OPTION_PARALLEL_COND_CLEANUP が設定されており、並列 VACUUM をサポートしています。
