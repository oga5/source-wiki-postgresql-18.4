ExecSeqScanInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:360

目的

並列クエリ実行時において、シーケンシャルスキャンノードのための動的共有メモリ（DSM）領域を確保・初期化し、スキャン状態をセットアップする。

シグネチャまたは形状

void ExecSeqScanInitializeDSM(SeqScanState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の SeqScanState 構造体ポインタ。
  • pcxt: 並列実行コンテキスト (ParallelContext)。
  • **出力**: なし（副作用として node 内の共有メモリ設定およびスキャン記述子が更新される）。

前提条件と不変条件

  • node->ss.ss_currentRelation が有効なリレーションを指していること。
  • node->pscan_len に必要な共有メモリサイズが事前に見積もられていること。
  • pcxt->toc が有効なテーブル・オブ・コンテンツ（TOC）を保持していること。

アルゴリズムまたは振る舞い

1. shm_toc_allocate を使用し、pcxt->toc から node->pscan_len 分の共有メモリ領域を確保する。 2. table_parallelscan_initialize を呼び出し、確保した領域に現在のスナップショット (estate->es_snapshot) を用いて並列スキャン記述子を初期化する。 3. shm_toc_insert を使用し、プランノードIDをキーとして並列スキャン記述子をTOCに登録する。 4. table_beginscan_parallel を呼び出し、初期化した並列スキャン記述子を node->ss.ss_currentScanDesc にセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:474（直接の呼び出しを確認）。
  • **呼び出し先**: shm_toc_allocate, table_parallelscan_initialize, shm_toc_insert, table_beginscan_parallel。

関連データ構造

  • SeqScanState: シーケンシャルスキャンの実行状態を保持する構造体。
  • ParallelContext: 並列実行の共有状態を管理するコンテキスト。
  • ParallelTableScanDesc: テーブルアクセスメソッドにおける並列スキャン用の共有記述子。

失敗モードと境界条件

  • 共有メモリの割り当て (shm_toc_allocate) が失敗した場合、メモリ不足エラーが発生する可能性がある。
  • node->ss.ss_currentRelation がクローズされている場合、table_parallelscan_initialize 等で予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は並列実行の初期化フェーズで実行される。
  • 関連する並列処理の再初期化には ExecSeqScanReInitializeDSM が、ワーカーごとの初期化には ExecSeqScanInitializeWorker がそれぞれ対応する。
