test_slru_page_readonly

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_slru.c:110

目的

SLRU（Simple Least Recently Used）キャッシュから指定されたページを読み取り専用モードで取得し、その内容をテキストとして返却するSQLインターフェース関数です。

シグネチャまたは形状

Datum test_slru_page_readonly(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT64(0) を介してページ番号（pageno）を受け取ります。
  • **出力**: PG_RETURN_TEXT_P を介して、ページバッファの内容を text 型として返却します。

前提条件と不変条件

  • TestSlruCtl が初期化済みである必要があります。
  • SimpleLruReadPage_ReadOnly の呼び出しにより、対象ページがバッファに読み込まれるか、既存のバッファが特定されます。
  • 処理中、対象ページに対応するバンクのロック（SimpleLruGetBankLock）が保持されている必要があります。

アルゴリズムまたは振る舞い

1. 指定された pageno に対応するバンクのロックを取得します。 2. SimpleLruReadPage_ReadOnly を呼び出し、SLRUバッファから該当ページを読み取ります。 3. 呼び出し元がロックを保持していることを Assert で検証します。 4. TestSlruCtl の共有メモリ内にある該当スロットのデータを取得します。 5. ロックを解放します。 6. 取得したデータを text 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**:
  • SimpleLruGetBankLock: ロックの取得。
  • SimpleLruReadPage_ReadOnly: SLRUバッファの読み取り。
  • LWLockHeldByMe: ロック状態の検証。
  • LWLockRelease: ロックの解放。
  • cstring_to_text: 戻り値の型変換。

関連データ構造

  • TestSlruCtl: テスト用SLRUを制御する SlruCtlData 構造体。
  • SlruShared: TestSlruCtl->shared を介してアクセスされる共有メモリ上のバッファ領域。

失敗モードと境界条件

  • pageno が無効な場合、SimpleLruReadPage_ReadOnly の内部実装に従いエラーが発生する可能性があります。
  • 共有メモリの不足やロック取得の競合が発生した場合、PostgreSQLの標準的なエラーハンドリングが適用されます。

保守担当者向けメモ

  • 本関数は読み取り専用操作であるため、ページの内容を変更しません。
  • SimpleLruReadPage_ReadOnly は、ページがバッファに存在しない場合にディスクから読み込む可能性があります。
