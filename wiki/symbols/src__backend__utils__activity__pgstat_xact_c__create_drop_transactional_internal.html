create_drop_transactional_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_xact.c:334

目的

統計情報の作成または削除操作を、現在のトランザクション状態に関連付けて記録し、トランザクション終了時に適切な処理（コミットまたはアボート）が行われるよう保留リストへ登録する。

シグネチャまたは形状

static void create_drop_transactional_internal(PgStat_Kind kind, Oid dboid, uint64 objid, bool is_create)

入力と出力

  • **入力**:
  • kind: 統計対象の種別 (PgStat_Kind)。
  • dboid: データベースのOID。
  • objid: 対象オブジェクトのID。
  • is_create: 作成操作であれば true、削除操作であれば false。
  • **出力**: なし（副作用として TopTransactionContext 上にメモリを確保し、スタックへ登録）。

前提条件と不変条件

  • TopTransactionContext が有効であること。
  • pgstat_get_xact_stack_level が現在のネストレベルに対応する PgStat_SubXactStatus を取得できること。

アルゴリズムまたは振る舞い

1. GetCurrentTransactionNestLevel() を呼び出し、現在のトランザクションネストレベルを取得する。 2. TopTransactionContext 上に PgStat_PendingDroppedStatsItem 構造体を割り当てる。 3. pgstat_get_xact_stack_level を使用して、現在のネストレベルに対応する xact_state を取得する。 4. 入力された kind, dboid, objid を PgStat_PendingDroppedStatsItem に格納する。objid は上位・下位の32ビットに分割して保持される。 5. is_create フラグをセットする。 6. dclist_push_tail を使用し、xact_state->pending_drops リストの末尾に当該アイテムを追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pgstat_create_transactional (引数 is_create = true で呼び出し)
  • pgstat_drop_transactional (引数 is_create = false で呼び出し)
  • **呼び出し先**:
  • GetCurrentTransactionNestLevel()
  • MemoryContextAlloc()
  • pgstat_get_xact_stack_level()
  • dclist_push_tail()

関連データ構造

  • PgStat_PendingDroppedStatsItem: トランザクション内で保留された統計変更を保持する構造体。
  • PgStat_SubXactStatus: トランザクションのネストレベルごとの統計状態を管理する構造体。

失敗モードと境界条件

  • メモリ不足時: MemoryContextAlloc がエラーを発生させる可能性がある。
  • objid の分割: 64ビットの objid を32ビットずつに分割して格納するため、値の整合性に注意が必要。

保守担当者向けメモ

  • 本関数は TopTransactionContext を使用してメモリを確保するため、トランザクション終了時に自動的に解放される設計となっている。
  • dclist を使用したリスト操作を行っているため、リストの順序がトランザクションの整合性に影響を与える可能性がある。
