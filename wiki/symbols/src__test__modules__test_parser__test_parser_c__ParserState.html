ParserState

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_parser/test_parser.c:24

目的

PostgreSQLのカスタムパーサーにおいて、解析対象のテキストデータと現在の走査位置を保持するための状態管理構造体です。

シグネチャまたは形状

typedef struct
{
	char	   *buffer;	/* 解析対象のテキスト */
	int			len;	/* bufferの長さ */
	int			pos;	/* 現在の解析位置 */
} ParserState;

入力と出力

  • 入力: testprs_start 関数において、解析対象の文字列と長さが本構造体のメンバに格納されます。
  • 出力: testprs_getlexeme 等の関数が本構造体を参照し、現在の pos を更新しながらトークンを抽出します。

前提条件と不変条件

  • buffer は解析対象のテキストを保持する有効なメモリ領域を指す必要があります。
  • pos は常に 0 以上 len 以下の範囲内である必要があります。

アルゴリズムまたは振る舞い

  • 解析セッションの開始時に palloc0 を用いてメモリが確保され、初期化されます。
  • testprs_getlexeme 関数内では、pos を起点として文字列を走査し、空白または単語の境界を判定します。
  • 解析が進むにつれて pos がインクリメントされ、次のトークン抽出位置が更新されます。
  • 解析終了時に testprs_end 関数によって pfree で解放されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • testprs_start: ParserState のメモリ確保と初期化に使用。
  • testprs_getlexeme: PG_GETARG_POINTER(0) を通じて引数として受け取り、解析状態の参照に使用。
  • testprs_end: PG_GETARG_POINTER(0) を通じて受け取り、メモリ解放に使用。
  • 呼び出し先: 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • LexDescr: パーサーがサポートするトークン型を定義する構造体。

失敗モードと境界条件

  • buffer が空の場合、len は 0 となり、解析ループは直ちに終了します。
  • pos が len に達した際、それ以上のトークン抽出は行われません。

保守担当者向けメモ

  • 本構造体は PostgreSQL の palloc を使用したメモリ管理を前提としています。
  • 構造体のメンバを追加・変更する場合は、testprs_start での初期化処理と整合性を取る必要があります。
