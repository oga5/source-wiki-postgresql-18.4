libpqrcv_startstreaming

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:541

目的

プライマリサーバーに対して START_REPLICATION コマンドを発行し、物理または論理レプリケーションのストリーミングセッションを開始する。

シグネチャまたは形状

bool libpqrcv_startstreaming(WalReceiverConn *conn, const WalRcvStreamOptions *options)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する WalReceiverConn 構造体。
  • options: ストリーミングの開始条件（論理/物理の別、スロット名、開始LSN、プロトコルオプション等）を格納した WalRcvStreamOptions。
  • **出力**:
  • bool: ストリーミングが COPY モードで開始された場合は true を返す。コマンドが正常終了（COPYモード非移行）した場合は false を返す。

前提条件と不変条件

  • options->logical と conn->logical が一致していること。
  • 論理レプリケーションの場合、options->slotname が指定されていること。

アルゴリズムまたは振る舞い

1. StringInfo を使用して START_REPLICATION コマンド文字列を構築する。 2. 論理レプリケーションの場合、プロトコルバージョン、ストリーミング設定、パブリケーション名などのオプションをコマンドに追加する。 3. 物理レプリケーションの場合、タイムラインIDをコマンドに追加する。 4. libpqsrv_exec を呼び出し、サーバーへコマンドを送信する。 5. サーバーからの応答を確認する:

  • PGRES_COMMAND_OK: ストリーミング開始を伴わない正常終了として false を返す。
  • PGRES_COPY_BOTH: ストリーミング開始成功として true を返す。
  • その他: プロトコル違反としてエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: WalReceiverFunctions インターフェースを介して呼び出される（直接的な呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**: libpqsrv_exec, PQserverVersion, PQerrorMessage, PQclear, stringlist_to_identifierstr, PQescapeLiteral 等の libpq 関連関数。

関連データ構造

  • WalReceiverConn: 接続情報およびストリーミング状態を保持。
  • WalRcvStreamOptions: ストリーミングのパラメータ定義。
  • StringInfoData: コマンド文字列構築用バッファ。

失敗モードと境界条件

  • メモリ不足: stringlist_to_identifierstr や PQescapeLiteral が失敗した場合、ERRCODE_OUT_OF_MEMORY でエラー終了する。
  • プロトコル違反: サーバーが PGRES_COPY_BOTH 以外の予期しない応答を返した場合、ERRCODE_PROTOCOL_VIOLATION でエラー終了する。

保守担当者向けメモ

  • 本関数は libpq の COPY モードを利用するため、PQgetCopyData 等の後続処理と密接に関連している。
  • 論理レプリケーションのオプション追加ロジックは、サーバーのバージョン（14, 15, 16以降）に応じて条件分岐しているため、プロトコル拡張時には注意が必要。
