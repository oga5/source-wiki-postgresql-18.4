make_nulltest_from_distinct

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_expr.c:3118

目的

IS DISTINCT FROM または IS NOT DISTINCT FROM 構文を、意味解析済みの NullTest ノードへ変換する。

シグネチャまたは形状

Node *make_nulltest_from_distinct(ParseState *pstate, A_Expr *distincta, Node *arg)

入力と出力

  • **入力**:
  • pstate: 現在のパーサー状態。
  • distincta: 変換元の A_Expr（AEXPR_DISTINCT または AEXPR_NOT_DISTINCT）。
  • arg: 変換対象の式ノード。
  • **出力**: NullTest ノードとして構築された Node*。

前提条件と不変条件

  • arg は transformExprRecurse を通じて再帰的に変換される。
  • 引数の型に関わらず、型強制（coerce）は行わない。
  • NullTest の argisrow は常に false に設定される（複合型であっても同様）。

アルゴリズムまたは振る舞い

1. makeNode(NullTest) で新しいノードを生成する。 2. transformExprRecurse を呼び出し、arg を意味解析済みの式に変換して nt->arg に格納する。 3. distincta->kind を判定し、AEXPR_NOT_DISTINCT であれば IS_NULL を、それ以外（AEXPR_DISTINCT）であれば IS_NOT_NULL を nt->nulltesttype に設定する。 4. nt->argisrow を false に固定する。 5. distincta->location から構文解析上の位置情報をコピーし、ノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_expr.c 内の式変換ロジック（1043行目および1045行目付近）。
  • **呼び出し先**: transformExprRecurse。

関連データ構造

  • ParseState: パーサーのコンテキスト情報。
  • A_Expr: 構文解析器が生成する生の式ノード。
  • NullTest: 意味解析済みの IS NULL / IS NOT NULL テストノード。

失敗モードと境界条件

  • transformExprRecurse が失敗した場合、そのエラーが伝播する。
  • arg が NULL の場合や、無効な式ノードである場合の挙動は transformExprRecurse の実装に依存する。

保守担当者向けメモ

  • 本関数は IS DISTINCT FROM 構文を NullTest に変換する特殊なケースを扱う。
  • argisrow = false の固定値設定が、将来的な複合型比較の拡張において制約とならないか注意すること。
