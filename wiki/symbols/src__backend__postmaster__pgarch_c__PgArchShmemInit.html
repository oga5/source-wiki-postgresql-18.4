PgArchShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pgarch.c:168

目的

WALアーカイバプロセスが使用する共有メモリ領域（PgArchData）の割り当ておよび初期化を行う。

シグネチャまたは形状

void PgArchShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（共有メモリ上のグローバル変数 PgArch を初期化）

前提条件と不変条件

  • PostgreSQLの共有メモリ初期化フェーズで呼び出される必要がある。
  • PgArchShmemSize() が返すサイズが、共有メモリ確保に十分であること。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、"Archiver Data" という名前で共有メモリ領域を確保または既存領域へのアタッチを行う。 2. found フラグにより、既に初期化済みかを確認する。 3. 初回実行時（found == false）の場合:

  • MemSet を使用して確保した領域をゼロクリアする。
  • PgArch->pgprocno を INVALID_PROC_NUMBER に設定する。
  • pg_atomic_init_u32 を使用して PgArch->force_dir_scan を 0 に初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:334（PostgreSQLの共有メモリ初期化プロセスの一部）
  • 呼び出し先: ShmemInitStruct, PgArchShmemSize, MemSet, pg_atomic_init_u32

関連データ構造

  • PgArchData: アーカイバの状態を保持する共有メモリ構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct 内部でエラーが発生しプロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数はPostmaster起動時の共有メモリセットアップの一部として実行される。
  • PgArch 構造体のフィールドを追加・変更した場合は、必ず本関数内での初期化ロジックを確認すること。
