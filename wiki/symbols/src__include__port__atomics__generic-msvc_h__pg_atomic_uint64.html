pg_atomic_uint64

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/generic-msvc.h:40

目的

MSVC環境において、64ビット整数に対するアトミック操作を安全に行うためのデータコンテナを提供します。

シグネチャまたは形状

typedef struct pg_attribute_aligned(8) pg_atomic_uint64
{
	volatile uint64 value;
} pg_atomic_uint64;

入力と出力

  • 入力: 構造体内の value フィールド（uint64 型）。
  • 出力: なし（データ構造定義のため）。

前提条件と不変条件

  • pg_attribute_aligned(8) により、メモリ上で8バイト境界に配置されることが保証されます。
  • value フィールドは volatile 修飾されており、コンパイラによる最適化で読み書きが省略されないよう制御されています。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行い、アトミックな演算（加算、比較交換等）は generic-msvc.h 内で定義される pg_atomic_*_u64_impl 関数群が、Windowsの Interlocked API を介して実行します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/access/transam/xlog.c: LSN（ログシーケンス番号）の管理やバッファ制御に使用。
  • src/backend/access/transam/xlogprefetcher.c: プリフェッチ統計情報の管理に使用。
  • 呼び出し先: 本構造体はデータ型であるため、直接的な呼び出し先はありません。

関連データ構造

  • pg_atomic_uint32: 32ビット版のアトミック操作用構造体。
  • pg_atomic_flag: アトミックなフラグ操作用構造体。

失敗モードと境界条件

  • 64ビット操作は、実行環境が64ビットビルド（_WIN64）であることを前提としています。
  • アライメントが崩れたメモリ配置で使用された場合、ハードウェアレベルでアトミック性が保証されない可能性があります。

保守担当者向けメモ

  • 本構造体は atomics.h を経由して利用されることを想定しています。
  • 構造体定義を変更する場合は、pg_attribute_aligned(8) によるアライメント制約が各プラットフォームの要件を満たしているか確認してください。
