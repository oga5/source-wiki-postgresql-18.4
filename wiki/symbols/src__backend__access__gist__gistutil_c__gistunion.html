gistunion

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistutil.c:218

目的

複数のインデックスタプルから、それらを包含する単一の「和（Union）」キーを計算し、新しいインデックスタプルとして生成する。

シグネチャまたは形状

IndexTuple gistunion(Relation r, IndexTuple *itvec, int len, GISTSTATE *giststate)

入力と出力

  • **入力**:
  • r: 対象となるインデックスのリレーション。
  • itvec: 処理対象の IndexTuple 配列。
  • len: itvec 内のタプル数。
  • giststate: GiST インデックスの操作状態を保持する構造体。
  • **出力**:
  • 複数の入力タプルを包含する新しい IndexTuple を返す。

前提条件と不変条件

  • giststate は適切に初期化されている必要がある。
  • itvec には有効な IndexTuple が len 個格納されていること。

アルゴリズムまたは振る舞い

1. gistMakeUnionItVec を呼び出し、itvec 内の全タプルの属性を統合（Union）し、結果を attr 配列および isnull 配列に格納する。 2. gistFormTuple を呼び出し、統合された属性値から新しい IndexTuple を構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuild.c:530
  • **呼び出し先**:
  • gistMakeUnionItVec (同ファイル内)
  • gistFormTuple (同ファイル内)

関連データ構造

  • IndexTuple: GiST インデックスに格納されるタプル形式。
  • GISTSTATE: インデックスの操作に必要な型情報やサポート関数を保持する構造体。
  • Relation: PostgreSQL のリレーション記述子。

失敗モードと境界条件

  • len が 0 の場合の挙動については、本ソースコード単体では定義されていない。
  • メモリ不足時には gistFormTuple 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は GiST インデックスの親ノード更新時に頻繁に使用される。
  • gistMakeUnionItVec が内部で opclass の union サポート関数を呼び出すため、インデックスの型定義に依存する。
