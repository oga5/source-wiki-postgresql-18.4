list_delete_first

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:942

目的

リストの先頭要素を削除し、更新されたリストを返します。

シグネチャまたは形状

List *list_delete_first(List *list)

入力と出力

  • **入力**: list (操作対象の List 構造体へのポインタ)
  • **出力**: 削除後の新しいリストの先頭ポインタ。リストが空の場合は NIL を返します。

前提条件と不変条件

  • 操作前に check_list_invariants(list) が呼び出され、リストの整合性が検証されます。

アルゴリズムまたは振る舞い

1. リストの整合性を確認します。 2. 入力 list が NIL の場合は、そのまま NIL を返します。 3. それ以外の場合、内部関数 list_delete_nth_cell(list, 0) を呼び出し、インデックス 0（先頭）の要素を削除してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: check_list_invariants, list_delete_nth_cell
  • **呼び出し元**:
  • src/backend/access/gist/gist.c
  • src/backend/access/gist/gistbuild.c
  • src/backend/catalog/dependency.c
  • src/backend/catalog/namespace.c
  • src/backend/commands/copyfrom.c
  • src/backend/commands/explain.c

関連データ構造

  • List: PostgreSQLの配列ベースの動的リスト構造体。

失敗モードと境界条件

  • list が NIL の場合、エラーにはならず NIL を返します。
  • 配列ベースの実装であるため、先頭要素の削除時には後続の全要素を前方にシフトする操作が発生し、$O(N)$ の計算コストがかかります。

保守担当者向けメモ

  • 本関数は list_delete_nth_cell のラッパーです。
  • リストの伸長や削除により内部配列が再配置される可能性があるため、リスト操作中に ListCell へのポインタを保持し続けることは避けてください。
