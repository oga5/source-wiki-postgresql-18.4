PLyPlanObject

定義

  • **種別**: struct
  • **定義箇所**: src/pl/plpython/plpy_planobject.h:12

目的

PostgreSQL の SPI（Server Programming Interface）で作成された実行計画を、Python から操作可能なオブジェクトとしてラップし、ライフサイクルと関連データを管理する。

シグネチャまたは形状

typedef struct PLyPlanObject
{
    PyObject_HEAD
    SPIPlanPtr    plan;
    int           nargs;
    Oid          *types;
    PLyObToDatum *args;
    MemoryContext mcxt;
} PLyPlanObject;

入力と出力

  • **入力**: Python インタプリタから生成される PyObject としてのヘッダー情報。
  • **出力**: 保持する SPIPlanPtr を通じた SQL 実行計画の実行結果、および Python 側での型チェックやメモリ管理のインターフェース。

前提条件と不変条件

  • PyObject_HEAD を含んでいるため、Python の C API 拡張のメモリ管理ルールに従う必要がある。
  • MemoryContext を保持しており、計画に関連するメモリは PostgreSQL のメモリ管理システムと同期して管理される。

アルゴリズムまたは振る舞い

1. **初期化**: PLy_plan_new によりインスタンスが生成され、各フィールドが初期化される。 2. **保持**: SPI 実行計画（plan）、引数の数（nargs）、型情報（types）、変換関数（args）を保持する。 3. **破棄**: PLy_plan_dealloc が呼び出されると、保持している SPIPlanPtr と MemoryContext が解放される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_cursorobject.c
  • src/pl/plpython/plpy_planobject.c
  • src/pl/plpython/plpy_spi.c
  • **呼び出し先**:
  • 本構造体はデータ構造の定義であり、特定の関数を呼び出すものではない。操作ロジックは plpy_planobject.c および plpy_spi.c に実装されている。

関連データ構造

  • SPIPlanPtr: PostgreSQL の SPI 実行計画ポインタ。
  • PLyObToDatum: 引数の型変換用関数ポインタ。
  • MemoryContext: PostgreSQL のメモリコンテキスト。

失敗モードと境界条件

  • PyObject_New によるインスタンス生成失敗時に NULL を返す可能性がある。
  • Python 3.8 以降とそれ以前のバージョン間で PyTypeObject の参照カウント管理に差異があるため、実装ファイル側での条件分岐が必要となる。

保守担当者向けメモ

  • 本構造体は plpy_planobject.c と密接に連携している。変更時は必ず両ファイルを確認すること。
  • メモリ管理は PostgreSQL の MemoryContext に依存しているため、リークを防ぐために PLy_plan_dealloc での適切な解放が必須である。
