mod_m

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bloomfilter.c:287

目的

ビットセットのサイズが2のべき乗であるという特性を利用し、剰余演算（val % m）をビットマスク演算に置き換えることで高速なインデックス算出を行う。

シグネチャまたは形状

static inline uint32 mod_m(uint32 val, uint64 m)

入力と出力

  • **入力**:
  • val: 剰余を求める対象の32ビット符号なし整数。
  • m: 除数。ビットセットのサイズを表す64ビット符号なし整数。
  • **出力**:
  • val を m で割った剰余（val % m）を32ビット符号なし整数として返す。

前提条件と不変条件

  • m は2のべき乗であること（((m - 1) & m) == 0 が真であること）。
  • m は PG_UINT32_MAX + 1 以下であること。

アルゴリズムまたは振る舞い

1. m が2のべき乗であることをアサーションで検証する。 2. m が PG_UINT32_MAX + 1 以下であることをアサーションで検証する。 3. val & (m - 1) を計算し、剰余演算の結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/lib/bloomfilter.c 内の以下の箇所から呼び出されている。
  • 264行目: x = mod_m(x, m);
  • 265行目: y = mod_m(y, m);
  • 271行目: x = mod_m(x + y, m);
  • 272行目: y = mod_m(y + i, m);
  • **呼び出し先**: なし（組み込みのビット演算のみ）。

関連データ構造

  • bloom_filter: 本関数は、この構造体が保持するビットセットのインデックス計算に使用される。

失敗モードと境界条件

  • m が2のべき乗でない場合、アサーション違反によりプログラムが停止する（デバッグビルド時）。
  • m が PG_UINT32_MAX + 1 を超える場合、アサーション違反によりプログラムが停止する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、コンパイル時に展開されることを意図している。
  • ビットセットサイズが2のべき乗であるという前提は、bloom_create におけるメモリ確保ロジックと密接に関連しているため、変更時は注意が必要である。
