per_MultiFuncCall

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/funcapi.c:207

目的

集合を返す関数（SRF）の実行中、複数回の呼び出しにまたがって状態を保持している FuncCallContext 構造体を取得する。

シグネチャまたは形状

FuncCallContext *per_MultiFuncCall(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（関数呼び出し情報 fcinfo を含む）。
  • **出力**: fcinfo->flinfo->fn_extra に格納されている FuncCallContext へのポインタ。

前提条件と不変条件

  • fcinfo->flinfo->fn_extra が、事前に init_MultiFuncCall 等によって FuncCallContext 構造体で初期化されている必要がある。

アルゴリズムまたは振る舞い

1. fcinfo（関数呼び出し情報）の flinfo（関数情報）から、fn_extra フィールドに格納されたポインタを読み取る。 2. 読み取ったポインタを FuncCallContext * 型としてキャストし、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SRFの実装内で、現在の実行コンテキストを取得するために呼び出される。SRF_PERCALL_SETUP() マクロを通じて利用されることが多い。
  • **呼び出し先**: なし。

関連データ構造

  • FuncCallContext: SRFの実行状態を保持する構造体。
  • FmgrInfo: 関数情報。fn_extra を保持する。

失敗モードと境界条件

  • fn_extra が初期化されていない場合、NULL または不正なポインタが返される可能性がある。

保守担当者向けメモ

  • 本関数は SRF_PERCALL_SETUP() マクロの定義内で使用されている。SRFの実装者は、直接本関数を呼ぶよりもマクロ経由での利用が推奨される。
