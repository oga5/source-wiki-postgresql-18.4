rbt_copy_data

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:126

目的

赤黒木のノードに含まれる「データ部分」のみを、あるノードから別のノードへコピーします。侵入型データ構造の設計に基づき、ノードヘッダー（RBTNode）を除いた領域を対象とします。

シグネチャまたは形状

static inline void rbt_copy_data(RBTree *rbt, RBTNode *dest, const RBTNode *src)

入力と出力

  • rbt: 操作対象の赤黒木を管理する RBTree 構造体へのポインタ。ノードサイズ情報の取得に使用されます。
  • dest: コピー先のノードへのポインタ。
  • src: コピー元のノードへのポインタ。
  • 出力: なし（dest が指すメモリ領域が更新されます）。

前提条件と不変条件

  • dest および src は、rbt で定義された node_size 以上のメモリ領域を確保済みである必要があります。
  • RBTNode は、呼び出し側が定義する構造体の先頭メンバとして配置されている必要があります。

アルゴリズムまたは振る舞い

1. dest および src のポインタを RBTNode の直後のアドレスへ進めます（dest + 1, src + 1）。 2. rbt->node_size から sizeof(RBTNode) を差し引いたバイト数を計算します。 3. 計算されたバイト数分だけ、memcpy を用いて src のデータ領域から dest のデータ領域へメモリコピーを行います。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/lib/rbtree.c:493
  • src/backend/lib/rbtree.c:671
  • 呼び出し先:
  • memcpy（標準ライブラリ関数）

関連データ構造

  • RBTree: 木の制御構造体。node_size を保持します。
  • RBTNode: 各ノードのヘッダー。このシンボルは、このヘッダーの直後に続くユーザー定義データ領域を操作対象とします。

失敗モードと境界条件

  • rbt->node_size が sizeof(RBTNode) 未満である場合、memcpy のサイズ計算でアンダーフローが発生する可能性があります。
  • dest または src が NULL の場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを重視してヘッダー直後のメモリブロックを直接操作します。
  • ノード内のデータにポインタが含まれる場合、本関数によるコピーは「浅いコピー（shallow copy）」となります。深いコピーが必要な場合は、呼び出し側で別途処理を行う必要があります。
