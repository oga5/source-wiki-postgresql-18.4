find_compatible_agg

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepagg.c:378

目的

クエリ内の既存の集約関数（Aggref）を検索し、同一の集約呼び出しの再利用、または遷移状態（transno）の共有可能性を判定します。

シグネチャまたは形状

static int find_compatible_agg(PlannerInfo *root, Aggref *newagg, List **same_input_transnos)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • newagg: 検索対象の新しい集約関数ノード。
  • same_input_transnos: 入力が同一だが集約関数が異なる場合に、共有可能な遷移状態番号を格納するリストへのポインタ。
  • **出力**:
  • 戻り値: 完全一致する既存の AggInfo のインデックス（aggno）。一致しない場合は -1。

前提条件と不変条件

  • newagg に揮発性関数（volatile function）が含まれている場合、再利用は行われません。
  • 比較対象は root->agginfos に保持されている既存の集約情報です。

アルゴリズムまたは振る舞い

1. newagg が揮発性関数を含むか確認し、含む場合は -1 を返す。 2. root->agginfos を走査し、以下の項目がすべて一致するか比較する。

  • inputcollid, aggtranstype, aggstar, aggvariadic, aggkind
  • args, aggorder, aggdistinct, aggfilter（equal() による構造比較）

    • 3. 上記が一致し、かつ関数本体（aggfnoid, aggtype, aggcollid, aggdirectargs）も一致する場合、完全一致とみなして aggno を返す。 4. 関数本体は異なるが入力が一致し、かつ agginfo->shareable が真である場合、その transno を same_input_transnos リストに追加する。 5. 走査終了後、完全一致が見つからなければ -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepagg.c:222（preprocess_aggref 内から呼び出される）。
  • **呼び出し先**: equal()（ノード比較）、contain_volatile_functions()（揮発性チェック）、lfirst_node()、linitial_node()、lappend_int()。

関連データ構造

  • PlannerInfo: プランニング中の情報を保持。
  • Aggref: 集約関数の構文木ノード。
  • AggInfo: 既存の集約情報を保持する構造体。

失敗モードと境界条件

  • 揮発性関数を含む集約は再利用されません。
  • 完全に一致する集約が見つからない場合、-1 が返されます。
  • same_input_transnos は、完全一致が見つかった場合にはクリアされます。

保守担当者向けメモ

  • 本関数はプランニングの初期段階で実行されます。
  • 遷移状態の共有判定（find_compatible_trans）は本関数の後続処理で行われるため、ここではあくまで「入力が同じか」という観点でリストを収集しています。
