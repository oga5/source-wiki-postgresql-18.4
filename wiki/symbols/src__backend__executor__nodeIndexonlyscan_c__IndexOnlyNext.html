IndexOnlyNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:60

目的

インデックスのみスキャン（Index-Only Scan）のメインループとして、インデックスから次のタプルを抽出し、可視性判定を経て結果スロットへ格納する。

シグネチャまたは形状

TupleTableSlot *IndexOnlyNext(IndexOnlyScanState *node)

入力と出力

  • **入力**: IndexOnlyScanState *node（スキャンノードの状態管理構造体）
  • **出力**: TupleTableSlot *（取得したタプルを保持するスロット、またはスキャン終了時は空のスロット）

前提条件と不変条件

  • MVCCスナップショットのみをサポートする。
  • 非MVCCスナップショットでの実行や、xs_heap_continue が設定された状態での呼び出しはエラーとなる。
  • インデックスの可視性判定において、可視性マップ（Visibility Map）のビットがクリアされている場合、ヒープへのアクセスが必要となる。

アルゴリズムまたは振る舞い

1. **初期化**: スキャン記述子（scandesc）が未初期化の場合、index_beginscan で開始し、インデックススキャン用に設定する。 2. **反復処理**: index_getnext_tid を使用してインデックスからTIDを順次取得する。 3. **可視性チェック**: VM_ALL_VISIBLE を確認し、全トランザクションから可視であればヒープアクセスをスキップする。不可視の場合は index_fetch_heap を呼び出し、ヒープから可視性を確認する。 4. **データ格納**: インデックスまたはヒープから取得したデータを TupleTableSlot に格納する。 5. **再チェック**: インデックスが損失（lossy）である場合、ExecQualAndReset を用いてインデックス条件（quals）の再評価を行う。 6. **終了判定**: 取得したタプルが条件を満たさない場合は次のエントリへ進み、全エントリを走査し終えたら終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は明示されていないが、実行エンジン内のスキャンメソッドとして登録されている（nodeIndexonlyscan.c:348）。
  • **呼び出し先**: index_beginscan, index_rescan, index_getnext_tid, VM_ALL_VISIBLE, index_fetch_heap, ExecForceStoreHeapTuple, StoreIndexTuple, ExecQualAndReset, CHECK_FOR_INTERRUPTS 等。

関連データ構造

  • IndexOnlyScanState: スキャンノードの実行状態を保持。
  • TupleTableSlot: 抽出されたタプルを保持する。
  • IndexScanDesc: インデックススキャンの制御情報。

失敗モードと境界条件

  • **エラー**: 非MVCCスナップショット使用時や、xs_heap_continue が発生した際にエラーを投げる。
  • **エラー**: データが返されない場合（xs_hitup も xs_itup も存在しない場合）にエラーを投げる。
  • **エラー**: ORDER BY 句の再チェック（xs_recheckorderby）はサポート外のためエラーを投げる。
  • **境界条件**: インデックスが損失（lossy）な場合、再チェックによりタプルがフィルタリングされる可能性がある。

保守担当者向けメモ

  • 可視性マップのチェックはロックを伴わないため、メモリバリアとインデックスのロック順序に依存した同期が行われている。
  • StoreIndexTuple は cstring 型などの変換を担うため、データ型の追加や変更時には注意が必要である。
