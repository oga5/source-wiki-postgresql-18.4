shdepChangeDep

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_shdepend.c:205

目的

pg_shdepend カタログ内の既存の共有依存関係レコードを更新、削除、または新規挿入し、オブジェクト間の依存関係を管理する。

シグネチャまたは形状

static void shdepChangeDep(Relation sdepRel, Oid classid, Oid objid, int32 objsubid, Oid refclassid, Oid refobjid, SharedDependencyType deptype)

入力と出力

  • **入力**:
  • sdepRel: pg_shdepend リレーション（オープン済み）。
  • classid, objid, objsubid: 依存元オブジェクトの識別子。
  • refclassid, refobjid: 依存先オブジェクトの識別子。
  • deptype: 依存関係の種類（SharedDependencyType）。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • shdepLockAndCheckObject により、参照先オブジェクトが処理中に削除されないことが保証される。
  • 依存関係の検索には SharedDependDependerIndexId インデックスを使用する。

アルゴリズムまたは振る舞い

1. 依存元オブジェクト（dbid, classid, objid, objsubid）をキーとして pg_shdepend をスキャンし、既存のレコードを検索する。 2. 複数のレコードが一致した場合はエラーを発生させる。 3. 参照先が IsPinnedObject（組み込みオブジェクト）である場合:

  • 既存のレコードがあれば削除する。

    • 4. 既存のレコードが存在する場合:

  • 参照先（refclassid, refobjid）を新しい値で更新する。

    • 5. 既存のレコードが存在しない場合:

  • 新規レコードを挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**: changeDependencyOnOwner, shdepReassignOwned_Owner, shdepReassignOwned_InitAcl 等から呼び出される（ソースコード内の参照より）。
  • **呼び出し先**: shdepLockAndCheckObject, systable_beginscan, systable_getnext, CatalogTupleDelete, CatalogTupleUpdate, CatalogTupleInsert 等。

関連データ構造

  • pg_shdepend: 共有依存関係を保持するシステムカタログ。
  • SharedDependencyType: 依存関係の性質を定義する列挙型。
  • ScanKeyData: インデックススキャン用の検索キー。

失敗モードと境界条件

  • 同一の依存元オブジェクトに対して複数の pg_shdepend エントリが存在する場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • oldtup は heap_copytuple でコピーされたものであるため、メモリ上で直接修正して CatalogTupleUpdate に渡すことが可能。
  • 処理終了時に heap_freetuple を呼び出し、メモリリークを防いでいる。
