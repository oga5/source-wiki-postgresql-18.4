fetch_fp_info

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/fastpath.c:118

目的

指定された OID に対応する関数情報をシステムカタログ（pg_proc）から取得し、Fastpath 実行に必要なメタデータ構造体 fp_info を構築する。

シグネチャまたは形状

static void fetch_fp_info(Oid func_id, struct fp_info *fip)

入力と出力

  • **入力**:
  • func_id: 取得対象関数の OID。
  • fip: 関数情報を格納する struct fp_info へのポインタ。
  • **出力**: なし（fip を直接更新する）。

前提条件と不変条件

  • fip は NULL であってはならない。
  • fip->funcid は、関数情報のロードが完全に成功した後にのみ、有効な OID に設定される。
  • 実行中にエラーが発生した場合、ereport(ERROR, ...) によりトランザクションが中断される可能性がある。

アルゴリズムまたは振る舞い

1. fip をゼロクリアし、funcid を InvalidOid に設定する。 2. SearchSysCache1 を使用して pg_proc から関数情報を検索する。存在しない場合は ERRCODE_UNDEFINED_FUNCTION でエラーを投げる。 3. 取得した pg_proc エントリを検証する：

  • prokind が関数（PROKIND_FUNCTION）以外、または proretset（集合を返す関数）である場合は ERRCODE_FEATURE_NOT_SUPPORTED でエラーを投げる。
  • 引数の数が FUNC_MAX_ARGS を超える場合はエラーを投げる。

    • 4. fip に名前空間、戻り値型、引数型、関数名をコピーする。 5. ReleaseSysCache でキャッシュエントリを解放する。 6. fmgr_info を呼び出し、関数実行用の FmgrInfo を fip->flinfo に設定する。 7. 最後に fip->funcid を func_id に設定し、構造体の有効化を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/fastpath.c 内の関数（HandleFunctionRequest 等）から呼び出される。
  • **呼び出し先**: SearchSysCache1, GETSTRUCT, ReleaseSysCache, fmgr_info, ereport, elog, MemSet, memcpy, strlcpy。

関連データ構造

  • struct fp_info: 関数実行に必要なメタデータ（FmgrInfo、OID、型情報等）を保持する。
  • Form_pg_proc: pg_proc システムカタログの行構造。

失敗モードと境界条件

  • **関数未存在**: 指定された OID が pg_proc に存在しない場合、ERROR となる。
  • **サポート外の関数**: 集合を返す関数や、関数以外の pg_proc エントリを指定した場合、ERROR となる。
  • **引数過多**: 引数が FUNC_MAX_ARGS を超える場合、ERROR となる。

保守担当者向けメモ

  • 本関数はトランザクション境界をまたいだキャッシュを行わないため、呼び出しごとにシステムカタログを検索する設計となっている。
