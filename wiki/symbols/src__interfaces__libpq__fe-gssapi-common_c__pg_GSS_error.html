pg_GSS_error

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-gssapi-common.c:46

目的

GSSAPI認証処理中に発生したエラー情報を解析し、人間が読める形式に変換して PGconn のエラーバッファへ格納する。

シグネチャまたは形状

void pg_GSS_error(const char *mprefix, PGconn *conn, OM_uint32 maj_stat, OM_uint32 min_stat)

入力と出力

  • mprefix: エラーメッセージの先頭に付与する接頭辞文字列。
  • conn: エラー情報を格納する対象の PGconn 構造体。
  • maj_stat: GSSAPIのメジャー状態コード。
  • min_stat: GSSAPIのマイナー状態コード。
  • 出力: なし（conn->errorMessage を直接更新する）。

前提条件と不変条件

  • conn が有効なポインタであること。
  • ENABLE_GSS マクロが定義されている環境でのみ利用可能。

アルゴリズムまたは振る舞い

1. mprefix を conn->errorMessage に追加する。 2. pg_GSS_error_int を呼び出し、maj_stat を GSS_C_GSS_CODE として解析してバッファに追加する。 3. コロン（:）をバッファに追加する。 4. pg_GSS_error_int を呼び出し、min_stat を GSS_C_MECH_CODE として解析してバッファに追加する。 5. 改行文字（\n）をバッファに追加する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-auth.c (GSSAPI継続エラー時) およびバックエンド側の src/backend/libpq/ 配下の各ファイル（auth.c, be-gssapi-common.c, be-secure-gssapi.c）。
  • 呼び出し先: pg_GSS_error_int (内部関数)、appendPQExpBuffer、appendPQExpBufferChar。

関連データ構造

  • PGconn: libpqの接続状態を保持する構造体。
  • OM_uint32: GSSAPIのステータスコード型。
  • PQExpBuffer: エラーメッセージを構築するための動的文字列バッファ。

失敗モードと境界条件

  • conn->errorMessage がメモリ不足等で拡張できない場合、appendPQExpBuffer の内部挙動に依存する。
  • maj_stat や min_stat が無効な値の場合、pg_GSS_error_int 内の gss_display_status がエラーを返す可能性がある。

保守担当者向けメモ

  • 本関数は fe-gssapi-common.c 内の pg_GSS_error_int を介して gss_display_status をループ実行するため、複数のエラー情報がスタックされている場合でもすべて回収される。
  • バックエンドとフロントエンドの両方から参照されているため、変更時は両方のコンテキストへの影響を考慮すること。
