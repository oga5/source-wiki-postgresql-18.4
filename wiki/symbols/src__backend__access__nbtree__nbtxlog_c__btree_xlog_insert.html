btree_xlog_insert

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtxlog.c:159

目的

B-tree インデックスへの挿入操作を WAL（Write-Ahead Log）からリプレイし、物理ページの状態を再構築する。

シグネチャまたは形状

static void btree_xlog_insert(bool isleaf, bool ismeta, bool posting, XLogReaderState *record)

入力と出力

  • **入力**:
  • isleaf: 挿入対象がリーフページか否か。
  • ismeta: メタページの更新を伴うか否か。
  • posting: ポスティングリストの分割を伴うか否か。
  • record: WAL レコードの状態を保持する XLogReaderState 構造体。
  • **出力**: なし（副作用としてバッファおよびメタページを更新）。

前提条件と不変条件

  • リプレイ時は並行更新が発生しないため、ページロックは簡略化されている。
  • isleaf が false（内部ページ）の場合、子レベルの不完全な分割フラグをクリアする。
  • ismeta が true の場合、最後にメタページの復元を行う。

アルゴリズムまたは振る舞い

1. **内部ページ処理**: リーフでない場合、_bt_clear_incomplete_split を呼び出し、子ページの不完全分割フラグをクリアする。 2. **バッファ読み込み**: XLogReadBufferForRedo で対象ページを取得。BLK_NEEDS_REDO の場合のみ以下を実行。 3. **挿入処理**:

  • 通常挿入 (!posting): PageAddItem を使用してタプルをページに追加。
  • ポスティングリスト分割 (posting): _bt_swap_posting を用いて既存のポスティングリストを分割し、新しいアイテムを挿入。

    • 4. **コミット**: PageSetLSN で LSN を更新し、MarkBufferDirty でバッファをダーティにする。 5. **メタページ更新**: ismeta が真であれば _bt_restore_meta を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_redo（src/backend/access/nbtree/nbtxlog.c 内の各ケースで呼び出される）。
  • **呼び出し先**:
  • _bt_clear_incomplete_split
  • XLogReadBufferForRedo
  • PageAddItem
  • _bt_swap_posting
  • PageSetLSN, MarkBufferDirty
  • _bt_restore_meta

関連データ構造

  • xl_btree_insert: WAL レコードの挿入データ構造。
  • XLogReaderState: WAL レコードの読み取り状態。
  • Buffer, Page: PostgreSQL のバッファ管理およびページ構造。

失敗モードと境界条件

  • PageAddItem が InvalidOffsetNumber を返した場合、PANIC で停止する。
  • ポスティングリスト分割時に isleaf が偽である場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数はリカバリ専用であり、通常のインデックス挿入ロジック（nbtinsert.c）とは異なり、ロックの結合を行わない。
  • ポスティングリストの分割処理は、WAL データから復元された newitem を使用して物理的なページ状態を正確に再現する必要がある。
