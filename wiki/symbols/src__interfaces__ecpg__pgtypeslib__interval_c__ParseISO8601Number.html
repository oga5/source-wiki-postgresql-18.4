ParseISO8601Number

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/interval.c:55

目的

ISO 8601 形式の文字列から数値を解析し、整数部と小数部に分離して抽出します。

シグネチャまたは形状

static int ParseISO8601Number(const char *str, char **endptr, int *ipart, double *fpart)

入力と出力

  • **入力**:
  • str: 解析対象の文字列。
  • endptr: 解析終了位置を格納するポインタ。
  • **出力**:
  • ipart: 抽出された整数部（0への切り捨て）。
  • fpart: 抽出された小数部。
  • 戻り値: 成功時は 0、エラー時は DTERR_BAD_FORMAT または DTERR_FIELD_OVERFLOW を返します。

前提条件と不変条件

  • 入力文字列の先頭文字は、数字、マイナス記号、または小数点である必要があります。
  • 抽出された数値は INT_MIN から INT_MAX の範囲内に収まる必要があります。

アルゴリズムまたは振る舞い

1. 入力文字列の妥当性を確認し、不正な場合は DTERR_BAD_FORMAT を返します。 2. strtod を使用して文字列を浮動小数点数に変換します。 3. 変換結果が範囲外（オーバーフロー）の場合は DTERR_FIELD_OVERFLOW を返します。 4. 0への切り捨て（floor を使用）を行い、整数部を ipart に格納します。 5. 元の数値から整数部を引いた値を小数部として fpart に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の DecodeISO8601Interval 等から呼び出されます。
  • **呼び出し先**: 標準ライブラリの strtod および floor を使用します。
  • 注: バックエンド側の src/backend/utils/adt/datetime.c にも同名の関数が存在しますが、本ページは ecpg ライブラリ内の実装を対象としています。

関連データ構造

  • DTERR_BAD_FORMAT および DTERR_FIELD_OVERFLOW（エラーコード定数）。

失敗モードと境界条件

  • **フォーマットエラー**: 数値として解釈できない文字列が渡された場合。
  • **オーバーフロー**: 数値が int 型の範囲を超えた場合。
  • **解析失敗**: strtod が何も読み取れなかった場合。

保守担当者向けメモ

  • 本関数は PostgreSQL バックエンドのロジックを移植したものです。
  • 浮動小数点演算を含むため、コンパイル時に -ffast-math を使用しないよう注意してください。
