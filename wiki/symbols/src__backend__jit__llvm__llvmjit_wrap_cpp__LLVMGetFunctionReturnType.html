LLVMGetFunctionReturnType

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_wrap.cpp:37

目的

LLVMのC APIでは直接提供されていない、LLVMValueRef（関数）から戻り値の型（LLVMTypeRef）を抽出する機能を提供します。

シグネチャまたは形状

LLVMTypeRef LLVMGetFunctionReturnType(LLVMValueRef r)

入力と出力

  • **入力**: LLVMValueRef r - LLVMの関数を表す不透明ポインタ。
  • **出力**: LLVMTypeRef - 関数の戻り値の型を表す不透明ポインタ。

前提条件と不変条件

  • 入力 r は、LLVMの llvm::Function オブジェクトを指している必要があります。

アルゴリズムまたは振る舞い

1. llvm::unwrap<llvm::Function>(r) を呼び出し、C APIの不透明ポインタをLLVM C++の llvm::Function オブジェクトに変換します。 2. getReturnType() メソッドを呼び出し、戻り値の型を取得します。 3. llvm::wrap() を使用して、取得した型をC API形式の LLVMTypeRef に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit.c:523
  • src/backend/jit/llvm/llvmjit.c:966
  • **呼び出し先**:
  • llvm::unwrap<llvm::Function> (LLVM C++ API)
  • llvm::Function::getReturnType (LLVM C++ API)
  • llvm::wrap (LLVM C++ API)

関連データ構造

  • LLVMValueRef: LLVMの値を表すC APIの型。
  • LLVMTypeRef: LLVMの型を表すC APIの型。
  • llvm::Function: LLVMの関数を表すC++クラス。

失敗モードと境界条件

  • 入力 r が llvm::Function にキャストできない場合、llvm::unwrap の内部でアサーション失敗や未定義動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数はLLVMのバージョン差異を吸収するラッパーです。LLVMのAPI仕様変更時には、llvm::unwrap や getReturnType の呼び出し方法を確認してください。
  • src/include/jit/llvmjit.h にプロトタイプ宣言が存在します。
