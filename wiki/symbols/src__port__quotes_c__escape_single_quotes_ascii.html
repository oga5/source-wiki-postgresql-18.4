escape_single_quotes_ascii

定義

  • 種別: function
  • 定義箇所: src/port/quotes.c:32

目的

入力文字列内のシングルクォート（'）およびバックスラッシュ（\）を二重化し、PostgreSQLの設定ファイル構文等に適したエスケープ済み文字列を生成する。

シグネチャまたは形状

char *escape_single_quotes_ascii(const char *src);

入力と出力

  • **入力**: src (ヌル終端文字列へのポインタ)
  • **出力**: エスケープ処理後の文字列を格納した動的に確保されたメモリ領域へのポインタ。メモリ確保に失敗した場合は NULL を返す。

前提条件と不変条件

  • src は有効なヌル終端文字列であること。
  • 本関数はASCIIベースの処理を前提としており、マルチバイト文字のエンコーディングは考慮しない。
  • 返り値のメモリは malloc によって確保されるため、呼び出し側で必ず free() を呼び出して解放する必要がある。

アルゴリズムまたは振る舞い

1. 入力文字列 src の長さを取得し、最大で元の2倍の長さとなるバッファを malloc で確保する。 2. src を1文字ずつ走査する。 3. SQL_STR_DOUBLE マクロ（c.h 定義）を用い、現在の文字がエスケープ対象（' または \）か判定する。 4. 対象文字であれば、結果バッファにその文字を2回書き込む。対象外であれば1回のみ書き込む。 5. 最後にヌル終端文字 \0 を付与して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc.c:4501
  • src/bin/initdb/initdb.c:408
  • src/bin/scripts/vacuumdb.c:476
  • src/fe_utils/recovery_gen.c:166
  • **呼び出し先**:
  • malloc (標準ライブラリ)
  • strlen (標準ライブラリ)
  • SQL_STR_DOUBLE (マクロ: c.h で定義)

関連データ構造

  • 特になし（標準的な char 型文字列を扱う）。

失敗モードと境界条件

  • malloc がメモリ不足により失敗した場合、NULL を返す。
  • 空文字列が入力された場合、終端文字のみを含むメモリ領域を確保して返す。

保守担当者向けメモ

  • 本関数は設定ファイル生成等の文脈で使用される。マルチバイト文字を含む文字列を扱う場合は、本関数ではなく適切なエンコーディング対応関数を使用すること。
