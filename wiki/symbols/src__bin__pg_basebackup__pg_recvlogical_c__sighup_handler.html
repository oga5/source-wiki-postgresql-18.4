sighup_handler

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_recvlogical.c:686

目的

SIGHUP シグナルを受信した際に、出力ファイルの再オープンを要求するフラグを立てるために使用されます。

シグネチャまたは形状

static void sighup_handler(SIGNAL_ARGS)

入力と出力

  • **入力**: SIGNAL_ARGS（シグナルハンドラとして標準的な引数）
  • **出力**: なし（副作用としてグローバル変数 output_reopen を更新）

前提条件と不変条件

  • output_reopen 変数がこのハンドラによって変更可能であること。
  • 本関数はシグナルハンドラとして呼び出されるため、非同期シグナル安全な操作のみを行う必要があります。

アルゴリズムまたは振る舞い

1. グローバル変数 output_reopen を true に設定します。 2. これにより、メインループ側で次回の処理タイミングにおいて出力ファイルの再オープン処理がトリガーされます。

呼び出し元と呼び出し先

  • **呼び出し元**: OS（シグナル発生時）
  • **呼び出し先**: なし
  • **登録箇所**: src/bin/pg_basebackup/pg_recvlogical.c:963 にて pqsignal(SIGHUP, sighup_handler) により登録されています。

関連データ構造

  • output_reopen: bool 型のグローバル変数。本ハンドラによって状態が制御されます。

失敗モードと境界条件

  • シグナルハンドラ内での実行であるため、複雑な処理やロックの取得は避ける必要があります。現在の実装はフラグの代入のみであり、この制約を満たしています。

保守担当者向けメモ

  • 本関数は pg_recvlogical における設定再読み込みやログローテーション等のトリガーとして機能します。
  • Windows 環境ではシグナルハンドリングの挙動が異なる場合があるため、プラットフォーム固有の制約に注意してください。
