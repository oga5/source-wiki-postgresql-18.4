Gene

定義

  • 種別: typedef
  • 定義箇所: src/include/optimizer/geqo_gene.h:30

目的

PostgreSQLの遺伝的クエリ最適化（GEQO）モジュールにおいて、遺伝的アルゴリズムの最小単位である「遺伝子」を表現するために使用されます。

シグネチャまたは形状

typedef int Gene;

入力と出力

  • **入力**: int 型の値。
  • **出力**: Relid（結合対象となるリレーションの識別子）として機能します。

前提条件と不変条件

  • int 型のエイリアスとして定義されています。
  • 結合順序を探索する際の識別子として利用されるため、実装を変更する際はコード全体への影響を考慮する必要があります。

アルゴリズムまたは振る舞い

  • Chromosome（染色体）構造体内の配列要素として保持され、遺伝的アルゴリズムの操作対象となります。
  • geqo_recombination.c や geqo_mutation.c 等の関数群により、配列内の順序入れ替えや選択操作が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/optimizer/geqo/geqo_cx.c: cx 関数
  • src/backend/optimizer/geqo/geqo_erx.c: gimme_edge, remove_gene, gimme_gene, edge_failure, gimme_edge_table, gimme_tour 各関数
  • **呼び出し先**:
  • 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • Chromosome: Gene の配列を保持し、結合順序とコストを管理します。
  • Pool: 複数の Chromosome を保持・管理します。
  • Edge: geqo_erx.c において Gene 間の関係性を定義するために使用されます。

失敗モードと境界条件

  • int 型の範囲外の値が代入された場合、リレーション識別子としての整合性が失われる可能性があります。
  • geqo_erx.c 内の edge_failure 関数のように、遺伝子操作中に特定のインデックスや値の不一致が発生するケースが想定されています。

保守担当者向けメモ

  • 本型は Relid を表現するためのものです。
  • アルゴリズムの挙動を詳細に追跡する場合は、geqo_recombination.c や geqo_selection.c などの実装ファイルと併せて参照してください。
