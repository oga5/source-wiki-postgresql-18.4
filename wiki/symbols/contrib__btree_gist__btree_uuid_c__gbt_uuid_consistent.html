gbt_uuid_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_uuid.c:132

目的

GiST インデックス検索時、インデックス内の uuidKEY（範囲）とクエリ条件が一致するかを判定する。

シグネチャまたは形状

Datum gbt_uuid_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY 構造体（インデックスエントリ）。
  • PG_GETARG_UUID_P(1): 検索対象の pg_uuid_t 値。
  • PG_GETARG_UINT16(2): 比較演算の種類を示す StrategyNumber。
  • PG_GETARG_POINTER(4): recheck フラグへのポインタ。
  • **出力**:
  • Datum (bool): 検索条件に合致する場合は true、それ以外は false。
  • *recheck: 常に false が設定される（本関数による判定は正確であるため）。

前提条件と不変条件

  • entry->key は uuidKEY 構造体へのポインタとして解釈可能であること。
  • recheck は呼び出し元から提供された有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. recheck を false に設定する。 2. entry->key から uuidKEY を取得し、GBT_NUMKEY_R 構造体に範囲（lower, upper）をマッピングする。 3. btree_utils_num.c で定義された汎用関数 gbt_num_consistent を呼び出し、インデックスの範囲とクエリ値を比較する。 4. gbt_num_consistent の戻り値をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス検索エンジン（直接の呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**: gbt_num_consistent（contrib/btree_gist/btree_utils_num.c 内）。

関連データ構造

  • uuidKEY: インデックスノードに格納される UUID の範囲（lower/upper）を保持する構造体。
  • GBT_NUMKEY_R: 汎用的な範囲比較に使用される構造体。
  • tinfo: UUID 型の比較関数や型情報を保持する静的変数。

失敗モードと境界条件

  • 本関数は常に正確な判定を行うため、recheck は常に false となる。
  • entry が無効なポインタである場合、PostgreSQL のメモリ管理機構によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の汎用ロジックに強く依存している。動作の詳細や比較ロジックの変更が必要な場合は、まず gbt_num_consistent の実装を確認すること。
  • UUID の比較はバイト列比較（memcmp 相当）に基づいている。
