debug_reconstruction

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/reconstruct.c:383

目的

再構築プロセスで使用される各ソースファイル（rfile）の読み込み状況をデバッグログに出力し、dry_run モード時にはファイルサイズが要求を満たしているかを検証する。

シグネチャまたは形状

static void debug_reconstruction(int n_source, rfile **sources, bool dry_run)

入力と出力

  • n_source (int): sources 配列の要素数。
  • sources (rfile **): 再構築に使用するソースファイルのメタデータ構造体配列。
  • dry_run (bool): 実際の読み込みを行わず、計画の検証のみを行うフラグ。
  • 出力: なし（副作用としてログ出力および検証失敗時のプロセス終了）。

前提条件と不変条件

  • sources 配列内の各 rfile は、有効なファイル記述子（fd）を保持している必要がある。
  • dry_run が真の場合、各 rfile の fd はオープン状態である必要がある。

アルゴリズムまたは振る舞い

1. sources 配列を順次走査する。 2. s == NULL または s->num_blocks_read == 0 の場合はスキップする。 3. dry_run の状態に応じて、読み込み予定のブロック数をデバッグログに出力する。 4. dry_run が真の場合、fstat を用いて各ファイルのサイズ（sb.st_size）を取得する。 5. sb.st_size が s->highest_offset_read（読み込みが必要な最大オフセット）より小さい場合、pg_fatal を呼び出してエラー終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_combinebackup/reconstruct.c 内の reconstruct_from_incremental_file 関数から呼び出される。
  • 呼び出し先: fstat（システムコール）、pg_log_debug、pg_fatal。

関連データ構造

  • rfile: ソースファイルのメタデータ、ファイル記述子、読み込みブロック数、最大オフセットを保持する構造体。

失敗モードと境界条件

  • fstat が失敗した場合: pg_fatal によりエラー終了する。
  • ファイルサイズ不足: sb.st_size < s->highest_offset_read の場合、ファイルが短すぎるとして pg_fatal によりエラー終了する。

保守担当者向けメモ

  • 本関数は再構築の実行前に計画の妥当性を確認する重要なガードレールである。
  • dry_run モードでの検証は、実際の再構築処理で発生しうる「ファイル読み込みエラー」を事前に検知することを目的としている。
