hex_enc_len

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:257

目的

バイナリデータを16進数（HEX）形式の文字列にエンコードする際に必要なバッファサイズを算出します。

シグネチャまたは形状

static uint64 hex_enc_len(const char *src, size_t srclen)

入力と出力

  • **入力**:
  • src: エンコード対象のバイナリデータ（本関数内では未使用）。
  • srclen: 入力データのバイト長。
  • **出力**:
  • 16進数表現に必要なバイト数を uint64 型で返します。

前提条件と不変条件

  • 入力データが1バイトにつき2文字の16進数表現に変換されることを前提としています。

アルゴリズムまたは振る舞い

  • 入力されたバイト長 srclen を2倍（ビットシフト演算 << 1）した値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/encode.c:600 において、pg_encoding 構造体のメンバとして登録されています。
  • **呼び出し先**:
  • 本関数は計算のみを行うため、他の関数を呼び出しません。

関連データ構造

  • pg_encoding: エンコーディング方式を定義する構造体。本関数はこの構造体の encode_len フィールドに割り当てられます。

失敗モードと境界条件

  • srclen が非常に大きい場合、戻り値が uint64 の範囲を超える可能性がありますが、本関数自体はオーバーフローチェックを行いません。呼び出し側で MaxAllocSize 等を用いた検証が必要です。

保守担当者向けメモ

  • 本関数は非常に単純な計算を行うため、将来的にエンコーディング方式が変更されない限り、ロジックの修正は不要です。
