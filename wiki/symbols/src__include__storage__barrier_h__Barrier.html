Barrier

定義

  • 種別: struct
  • 定義箇所: src/include/storage/barrier.h:25

目的

並列処理を行う複数のプロセス間で同期をとるためのバリア同期メカニズムを提供します。全参加プロセスが特定の地点に到達するまで待機させ、フェーズ単位の計算処理を安全に実行可能にします。

シグネチャまたは形状

typedef struct Barrier
{
    slock_t             mutex;
    int                 phase;
    int                 participants;
    int                 arrived;
    int                 elected;
    bool                static_party;
    ConditionVariable   condition_variable;
} Barrier;

入力と出力

本構造体はデータ保持用であり、関数呼び出しの入力として渡されます。

  • 入力: BarrierInit 等の関数へポインタとして渡され、同期状態を管理します。
  • 出力: 内部の phase や arrived カウンタが更新され、condition_variable を通じて待機中のプロセスを起床させます。

前提条件と不変条件

  • mutex（slock_t）による排他制御が、構造体内のカウンタ更新時に必須です。
  • static_party はアサーション目的でのみ使用されます。
  • 共有メモリ環境下でのプロセス間同期を前提としています。

アルゴリズムまたは振る舞い

1. **到達**: BarrierArriveAndWait 呼び出し時、mutex で排他制御を行い arrived カウンタをインクリメントします。 2. **待機**: 全参加者が到達するまで condition_variable で待機します。 3. **解放**: 全員が到達すると、phase を更新し、condition_variable を通じて待機中の全プロセスを一斉に起床させます。 4. **動的制御**: BarrierAttach により参加者数を動的に変更可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c および src/backend/executor/nodeHashjoin.c で並列実行状態の管理に使用されています。
  • **呼び出し先**: src/backend/storage/ipc/barrier.c に実装が存在し、BarrierInit, BarrierArriveAndWait, BarrierArriveAndDetach 等の関数が本構造体を操作します。

関連データ構造

  • ConditionVariable: 待機中のプロセスを効率的にスリープ・起床させるために内包されています。
  • slock_t: 構造体内のカウンタ更新時の排他制御に使用されます。

失敗モードと境界条件

  • 参加者数が不一致のまま待機するとデッドロックの可能性があります。
  • メモリバリア（CPU命令レベルの順序制御）は本構造体では提供されません（port/atomics.h を参照する必要があります）。

保守担当者向けメモ

  • 本構造体は論理的なプロセス間同期を目的としています。
  • 動的バリアとして使用する場合、BarrierAttach 後に現在の phase を確認して処理を分岐させる設計が必要です。
