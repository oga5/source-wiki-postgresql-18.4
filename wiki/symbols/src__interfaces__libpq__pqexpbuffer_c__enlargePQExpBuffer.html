enlargePQExpBuffer

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/libpq/pqexpbuffer.c:171

目的

PQExpBuffer が保持するデータバッファを、指定されたサイズ（needed）を格納できるまで動的に拡張する。

シグネチャまたは形状

int enlargePQExpBuffer(PQExpBuffer str, size_t needed);

入力と出力

  • **入力**:
  • str: 拡張対象の PQExpBuffer 構造体へのポインタ。
  • needed: 追加で必要となる最小バイト数。
  • **出力**:
  • 1: 拡張成功、または既に十分な容量がある場合。
  • 0: メモリ不足（OOM）または不正なサイズ要求により拡張に失敗した場合。

前提条件と不変条件

  • str が既に「broken」状態（メモリ確保失敗済み）である場合、関数は即座に 0 を返す。
  • needed が INT_MAX - str->len を超えるような過大な値である場合、バッファは「broken」状態に遷移し、関数は 0 を返す。
  • 成功時、str->maxlen は要求を満たす値に更新され、str->data は新しいメモリ領域を指す。

アルゴリズムまたは振る舞い

1. **状態確認**: PQExpBufferBroken(str) をチェックし、失敗済みなら終了。 2. **バリデーション**: needed が INT_MAX を超えないか検証し、超える場合は markPQExpBufferBroken を呼び出して終了。 3. **容量チェック**: 現在の str->maxlen で要求を満たせるか確認し、十分なら終了。 4. **拡張計算**: 効率のため、現在の容量の倍数（最小64バイト）で needed を満たすまでサイズを倍増させる。 5. **上限設定**: newlen が INT_MAX を超えないようクランプする。 6. **メモリ再確保**: realloc を実行。成功時は構造体のメンバを更新し 1 を返す。失敗時は markPQExpBufferBroken を呼び出し 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/fe_utils/string_utils.c (複数箇所)
  • src/interfaces/libpq/fe-auth-scram.c
  • src/interfaces/libpq/fe-connect.c
  • src/interfaces/libpq/pqexpbuffer.c (内部の文字列操作関数群)
  • **呼び出し先**:
  • markPQExpBufferBroken (エラー発生時)
  • realloc (標準ライブラリ)

関連データ構造

  • PQExpBuffer: data（バッファポインタ）、len（現在の長さ）、maxlen（最大容量）を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: realloc が NULL を返した場合、バッファは「broken」状態となり、以降の操作は無効化される。
  • **整数オーバーフロー**: needed が INT_MAX を超える要求は、バッファ破壊のトリガーとなる。

保守担当者向けメモ

  • 本関数は realloc を使用するため、呼び出し側は最終的に destroyPQExpBuffer を呼び出してメモリを解放する必要がある。
  • 「broken」状態のバッファは resetPQExpBuffer を呼び出すことで再利用可能な状態への復帰が試みられる。
