keyEntryData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/ginutil.c:434

目的

GINインデックスの構築やキー抽出プロセスにおいて、抽出された個々のインデックスキーとそのNULL状態を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	Datum		datum;
	bool		isnull;
} keyEntryData;

入力と出力

  • 本構造体は、抽出されたキー値（Datum）と、その値がNULLであるかを示すフラグ（bool）をペアとして保持します。

前提条件と不変条件

  • ginExtractEntries 等の処理において、動的に割り当てられた keyEntryData の配列として使用されます。
  • qsort_arg を用いたソート処理の対象となるため、配列として連続したメモリ領域に配置されることが前提となります。

アルゴリズムまたは振る舞い

  • 抽出されたキーの配列を保持し、qsort_arg を介してインデックス順序に従ったソートや重複排除を行うための基盤データとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/gin/ginutil.c 内の関数（ginExtractEntries 等）が、キーの配列を管理するために本構造体をインスタンス化します。
  • 呼び出し先: qsort_arg が本構造体の配列をソートするために参照します。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • ginExtractEntries: 本構造体の配列を生成・操作する主要な関数。

失敗モードと境界条件

  • メモリ不足: palloc による配列確保時にメモリが不足した場合、PostgreSQLの標準的なエラーハンドリングによりアボートされます。
  • 配列サイズ: nentries が0の場合、空の配列として扱われます。

保守担当者向けメモ

  • 本構造体は ginutil.c 内のソート処理（qsort_arg）と密接に関連しています。比較関数（cmpEntriesArg 等）を変更する際は、本構造体のフィールド定義と整合性を取る必要があります。
