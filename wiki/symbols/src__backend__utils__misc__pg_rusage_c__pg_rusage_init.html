pg_rusage_init

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/pg_rusage.c:26

目的

現在のプロセスリソース使用状況（CPU時間および経過時間）をスナップショットとして取得し、指定された構造体に記録する。

シグネチャまたは形状

void pg_rusage_init(PGRUsage *ru0)

入力と出力

  • **入力**: ru0 - リソース情報を格納するための PGRUsage 構造体へのポインタ。
  • **出力**: なし（ru0 が指すメモリ領域を更新）。

前提条件と不変条件

  • ru0 は有効な PGRUsage 構造体を指している必要がある。
  • 呼び出し元が PGRUsage 構造体のメモリを確保していること。

アルゴリズムまたは振る舞い

1. getrusage(RUSAGE_SELF, &ru0->ru) を呼び出し、現在のプロセスのリソース使用統計を取得して ru0->ru に格納する。 2. gettimeofday(&ru0->tv, NULL) を呼び出し、現在の壁時計時間を取得して ru0->tv に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/catalog/index.c
  • src/backend/commands/analyze.c
  • src/backend/commands/cluster.c
  • src/backend/commands/indexcmds.c
  • src/backend/utils/misc/pg_rusage.c (pg_rusage_show 内)
  • src/backend/utils/sort/tuplesort.c
  • **呼び出し先**:
  • getrusage (システムコール)
  • gettimeofday (システムコール)

関連データ構造

  • PGRUsage: rusage 構造体（CPU統計）と timeval 構造体（壁時計時間）を保持する構造体。

失敗モードと境界条件

  • getrusage または gettimeofday が失敗した場合の戻り値チェックは行われていない。
  • 本関数自体はスレッドセーフだが、本モジュールの pg_rusage_show が静的バッファを使用するため、マルチスレッド環境での利用には注意が必要。

保守担当者向けメモ

  • 本関数は単なるスナップショット取得用であり、差分計算は pg_rusage_show が担当する。
  • 構造体 PGRUsage の定義は src/include/utils/pg_rusage.h を参照のこと。
