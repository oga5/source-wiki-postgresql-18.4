pg_fe_sasl_mech

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/fe-auth-sasl.h:43

目的

PostgreSQL クライアントにおける SASL 認証メカニズムの抽象化レイヤーを提供し、SCRAM 等の具体的な認証方式をプラグインとして追加可能にするためのコントラクトを定義する。

シグネチャまたは形状

typedef struct pg_fe_sasl_mech
{
    void *(*init) (PGconn *conn, const char *password, const char *mech);
    SASLStatus (*exchange) (void *state, bool final, char *input, int inputlen, char **output, int *outputlen);
    bool (*channel_bound) (void *state);
    void (*free) (void *state);
} pg_fe_sasl_mech;

入力と出力

  • init: 接続情報、パスワード、メカニズム名を入力とし、メカニズム固有の状態ポインタを返す。
  • exchange: サーバーからのチャレンジ（input）と状態を受け取り、クライアントのレスポンス（output）とステータスを返す。
  • channel_bound: 状態を受け取り、チャネルバインディングの有無を真偽値で返す。
  • free: 状態ポインタを受け取り、リソースを解放する。

前提条件と不変条件

  • exchange の input は NULL 終端されているが、SASL 仕様上のバイナリデータ（埋め込み NULL を含む）を扱うため、必ず inputlen を参照すること。
  • init が失敗した場合は NULL を返すこと。
  • free は認証完了時ではなく、接続終了時に呼び出される。

アルゴリズムまたは振る舞い

1. **初期化**: init でメカニズム固有の状態を生成。 2. **交換**: exchange でサーバーのチャレンジに対するレスポンスを生成。SASL_CONTINUE（継続）、SASL_ASYNC（非同期待ち）、SASL_COMPLETE（成功）、SASL_FAILED（失敗）のいずれかを返す。 3. **検証**: channel_bound でチャネルバインディングの確立を確認。 4. **終了**: free で init が確保したリソースを解放。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-connect.c 内の認証フロー制御ロジック。
  • 呼び出し先: src/interfaces/libpq/fe-auth-scram.c（pg_scram_mech）および src/interfaces/libpq/fe-auth-oauth.c（pg_oauth_mech）が本構造体を実装している。

関連データ構造

  • SASLStatus: 認証交換の進捗状態を示す列挙型。
  • PGconn: 接続情報を保持する構造体。

失敗モードと境界条件

  • init が NULL を返した場合、認証交換は失敗とみなされる。
  • exchange で output に NULL を設定した場合、認証は中断され、SASL_FAILED を返す必要がある。
  • exchange における inputlen が -1 の場合、クライアント主導の初期レスポンス生成フェーズとして扱われる。

保守担当者向けメモ

  • 本構造体は libpq 内部 API であり、公開 API ではない。
  • 非同期認証（SASL_ASYNC）を実装する場合、conn->async_auth への設定が必須となる。
