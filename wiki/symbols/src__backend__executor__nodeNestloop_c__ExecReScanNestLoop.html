ExecReScanNestLoop

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeNestloop.c:380

目的

ネステッドループ結合ノードの状態をリセットし、再スキャンを開始するための準備を行う。

シグネチャまたは形状

void ExecReScanNestLoop(NestLoopState *node)

入力と出力

  • **入力**: NestLoopState *node（ネステッドループ結合の実行状態を保持する構造体）
  • **出力**: なし（副作用として node の内部状態を更新）

前提条件と不変条件

  • 内部プラン（innerPlan）は、外部タプルが更新されるたびに個別に再スキャンされる必要があるため、本関数内では直接 ExecReScan を呼び出してはならない。これを守らない場合、外部変数を実行時キーとして使用するインデックススキャン等で不整合が発生する。

アルゴリズムまたは振る舞い

1. 外部プラン（outerPlan）の chgParam が NULL の場合、ExecReScan(outerPlan) を呼び出し、外部プランを再スキャンする。 2. node->nl_NeedNewOuter を true に設定し、次の外部タプル取得が必要であることを示す。 3. node->nl_MatchedOuter を false に設定し、現在の外部タプルに対する一致判定をリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:242
  • **呼び出し先**: ExecReScan（外部プランに対して条件付きで実行）

関連データ構造

  • NestLoopState: 実行時の状態（nl_NeedNewOuter, nl_MatchedOuter）を保持。
  • PlanState: 外部プランの状態（chgParam を参照）。

失敗モードと境界条件

  • 外部プランの chgParam が設定されている場合、ExecReScan は自動的に行われるため、本関数内ではスキップされる。

保守担当者向けメモ

  • 内部プランの再スキャンは、ExecNestLoop のメインループ内で外部タプルが切り替わるタイミングで行われる設計である。本関数で誤って内部プランを再スキャンしないよう注意すること。
