StrategyNumber

定義

  • 種別: typedef
  • 定義箇所: src/include/access/stratnum.h:22

目的

インデックスアクセスメソッドにおいて、演算子クラス（Operator Class）がサポートする演算子の意味論を識別するための識別子を提供します。

シグネチャまたは形状

typedef uint16 StrategyNumber;

入力と出力

  • 入力: uint16 型の数値。
  • 出力: インデックススキャン時、システムカタログ（pg_amop）から取得された演算子の意味論を特定する値として利用されます。

前提条件と不変条件

  • 値 0 は InvalidStrategy として予約されており、無効なストラテジーを示します。
  • ストラテジー番号はインデックスの演算子クラス定義と密接に結びついています。

アルゴリズムまたは振る舞い

インデックススキャン実行時、システムはクエリ内の演算子に対応する StrategyNumber を参照し、インデックス構造に対する走査範囲や適合性を決定します。

関連データ構造

  • pg_amop: ストラテジー番号と演算子、演算子クラスの関連付けを保持するシステムカタログ。

失敗モードと境界条件

  • 既存のストラテジー番号の変更や順序の入れ替えは、既存インデックスの破損やクエリ結果の不正を招く可能性があります。
  • 新規追加時は既存の番号と重複しないよう注意が必要です。

保守担当者向けメモ

  • RT で始まる定数は歴史的に R-Tree に由来しますが、現在は GiST や BRIN 等の多目的インデックスで広く利用されています。
  • 新しいインデックスタイプ追加時は、必要に応じて MaxStrategyNumber を更新してください。
