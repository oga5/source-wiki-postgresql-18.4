LogicalTapeWrite

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/logtape.c:760

目的

論理テープに対してデータを順次書き込みます。バッファが満杯になった場合は自動的に新しいブロックを割り当て、物理ファイルへフラッシュします。

シグネチャまたは形状

void LogicalTapeWrite(LogicalTape *lt, const void *ptr, size_t size)

入力と出力

  • lt: 書き込み対象の LogicalTape 構造体へのポインタ。
  • ptr: 書き込むデータソースへのポインタ。
  • size: 書き込むデータのバイト数。
  • 出力: なし（副作用として lt のバッファ状態および物理ファイルが更新されます）。

前提条件と不変条件

  • lt->writing が真であること。
  • lt->offsetBlockNumber が 0 であること。
  • lt->buffer_size が BLCKSZ と一致すること。
  • 初回書き込み時に lt->buffer が未割り当てであれば、BLCKSZ サイズでメモリが割り当てられます。

アルゴリズムまたは振る舞い

1. **初期化**: 初回書き込み時、ltsGetBlock を用いて最初のブロックを割り当て、TapeBlockTrailer の prev を -1 に設定します。 2. **バッファリング**: データを lt->buffer にコピーします。 3. **ブロック溢れ時の処理**:

  • 現在のブロックが TapeBlockPayloadSize に達した場合、ltsGetBlock で次のブロックを確保します。
  • 現在のブロックの TapeBlockTrailer に next ポインタを記録し、ltsWriteBlock で物理ファイルへ書き出します。
  • 次のブロックの prev ポインタを現在のブロック番号で初期化し、書き込み位置をリセットします。

    • 4. **状態更新**: 書き込み完了後、lt->dirty を真に設定し、lt->pos および lt->nbytes を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/utils/sort/tuplesort.c
  • src/backend/utils/sort/tuplesortvariants.c
  • **呼び出し先**:
  • ltsGetBlock (ブロック割り当て)
  • ltsWriteBlock (物理ファイルへの書き出し)
  • TapeBlockGetTrailer (メタデータアクセス)
  • palloc (メモリ割り当て)
  • elog (エラーハンドリング)

関連データ構造

  • LogicalTape: テープの現在の書き込み状態（バッファ、位置、ブロック番号）を保持。
  • LogicalTapeSet: テープセット全体およびフリーリストを管理。
  • TapeBlockTrailer: ブロックの末尾に配置され、チェーンの前後関係を保持するメタデータ。

失敗モードと境界条件

  • **状態不整合**: 読み込み状態から直接書き込み状態へ移行しようとした場合、elog(ERROR) により処理が中断されます。
  • **メモリ不足**: palloc が失敗した場合、PostgreSQL の標準的なメモリ管理に従いエラーが発生します。

保守担当者向けメモ

  • 本関数は BLCKSZ 単位のブロック管理を前提としています。
  • TapeBlockPayloadSize は BLCKSZ から TapeBlockTrailer のサイズを差し引いた値です。
  • 物理ファイルへの書き込みは ltsWriteBlock を介して行われるため、I/O 負荷は LogicalTapeSet の管理下にある物理ファイルの実装に依存します。
