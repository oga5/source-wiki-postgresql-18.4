g_int_same

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_gist.c:385

目的

2つの整数配列（ArrayType）がGiSTインデックス内で同一であるかどうかを判定する。

シグネチャまたは形状

Datum g_int_same(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_ARRAYTYPE_P(0): 比較対象の配列 a
  • PG_GETARG_ARRAYTYPE_P(1): 比較対象の配列 b
  • PG_GETARG_POINTER(2): 結果を格納する bool 型ポインタ
  • **出力**:
  • result ポインタ経由で、配列が同一であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力される配列 a および b は CHECKARRVALID マクロにより有効性が検証される必要がある。

アルゴリズムまたは振る舞い

1. 両配列の要素数（ARRNELEMS）を比較し、異なる場合は直ちに false を設定して終了する。 2. 要素数が同一の場合、各配列のポインタ（ARRPTR）を取得する。 3. 各要素を順次比較し、一つでも不一致があれば false を設定してループを抜ける。 4. 全ての要素が一致した場合、true を設定する。 5. 最終的に result ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c:82 (DirectFunctionCall3経由)
  • PG_FUNCTION_INFO_V1(g_int_same) による登録
  • **呼び出し先**:
  • ARRNELEMS, CHECKARRVALID, ARRPTR, PG_GETARG_ARRAYTYPE_P, PG_GETARG_POINTER, PG_RETURN_POINTER などのPostgreSQL内部マクロ。

関連データ構造

  • ArrayType: PostgreSQLの配列型構造体。

失敗モードと境界条件

  • 配列の有効性が確認できない場合、CHECKARRVALID によりエラーが発生する可能性がある。
  • 要素数が異なる場合は、内容の比較を行わずに false を返す。

保守担当者向けメモ

  • 本関数はGiSTインデックスの same メソッドとして機能する。
  • 比較ロジックは単純なメモリ上の配列走査であるため、配列のソート順序が一致していることが前提となる（intarray の他の関数でソート済みであることが期待される）。
