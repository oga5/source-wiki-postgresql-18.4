replace_outer_agg

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:224

目的

サブクエリ内で参照される上位レベルの集約関数（Aggref）を、実行時に評価可能な PARAM_EXEC パラメータへ置換します。

シグネチャまたは形状

Param *replace_outer_agg(PlannerInfo *root, Aggref *agg)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo。
  • agg: 置換対象の Aggref ノード。
  • **出力**:
  • 置換後の Param ノード（PARAM_EXEC 型）。

前提条件と不変条件

  • agg->agglevelsup > 0 かつ agg->agglevelsup < root->query_level であること（上位レベルの集約であること）。
  • 実行時に PARAM_EXEC スロットを割り当てるため、root->glob->paramExecTypes および root->plan_params が有効であること。

アルゴリズムまたは振る舞い

1. agg->agglevelsup を辿り、対象の集約関数が定義されているクエリレベルの PlannerInfo を特定します。 2. copyObject を使用して Aggref を複製し、IncrementVarSublevelsUp を用いてレベルを調整（agglevelsup を 0 に）します。 3. 重複排除は行わず、常に新しい PARAM_EXEC ID を発行します。 4. root->glob->paramExecTypes に集約の型（aggtype）を登録します。 5. PlannerParamItem を作成し、root->plan_params に追加します。 6. 最終的に、割り当てられた paramId を持つ Param ノードを生成して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/subselect.c 内のサブクエリ処理ロジックから呼び出されます。
  • **呼び出し先**: copyObject, IncrementVarSublevelsUp, makeNode, lappend, lappend_oid 等のノード操作・リスト操作関数。

関連データ構造

  • Aggref: 集約関数の定義ノード。
  • PlannerInfo: クエリのプランニング情報を保持する構造体。
  • PlannerParamItem: plan_params リストに格納される、パラメータとノードの対応付け。
  • Param: PARAM_EXEC 実行時パラメータノード。

失敗モードと境界条件

  • agglevelsup が範囲外の場合、Assert により停止します。
  • メモリ不足時には copyObject やリスト操作関数がエラーを発生させます。

保守担当者向けメモ

  • 本関数は集約関数の重複排除を行いません。同一の集約が複数回出現しても、呼び出しごとに新しい PARAM_EXEC スロットが消費されます。
  • agg->aggtype を paramExecTypes に登録する際、型情報の整合性に注意してください。
