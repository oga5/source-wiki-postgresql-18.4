AttrDefault

定義

  • 種別: struct
  • 定義箇所: src/include/access/tupdesc.h:22

目的

リレーションの列に対するデフォルト値を保持するための構造体です。列のデフォルト式を文字列形式で管理し、必要に応じて評価可能な状態にします。

シグネチャまたは形状

typedef struct AttrDefault
{
    AttrNumber  adnum;          /* デフォルト値が適用される列番号 */
    char       *adbin;          /* 式の nodeToString 表現 */
} AttrDefault;

入力と出力

  • **入力**: adnum（属性番号）、adbin（デフォルト式を表す文字列）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • adnum は有効な属性番号である必要があります。
  • adbin は nodeToString で生成された有効な式表現である必要があります。
  • 複数の AttrDefault を含む配列を扱う場合、それらは adnum の昇順で並んでいることが想定されるケースがあります（src/backend/access/common/tupdesc.c:690 のコメント参照）。

アルゴリズムまたは振る舞い

  • 本構造体は主に TupleConstr 構造体の一部として、テーブル定義に関連付けられたデフォルト値のリストを保持します。
  • src/backend/access/common/tupdesc.c において、タプル記述子のコピー作成時や比較時に、この配列全体がメモリコピーや走査の対象となります。

呼び出し元と呼び出し先

  • **呼び出し元（参照・操作）**:
  • src/backend/access/common/tupdesc.c: タプル記述子の複製や比較処理。
  • src/backend/optimizer/util/plancat.c: プラン作成時のデフォルト値参照。
  • src/backend/utils/cache/relcache.c: リレーションキャッシュ構築時のデフォルト値読み込み。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TupleConstr: AttrDefault の配列を保持する親構造体。
  • TupleDescData: タプル記述子のメイン構造体。

失敗モードと境界条件

  • relcache.c において、期待される AttrDefault レコードが見つからない場合はエラーをスローする設計となっています。
  • メモリ割り当て時（palloc）に失敗した場合は、PostgreSQL の標準的なメモリ不足エラーが発生します。

保守担当者向けメモ

  • adbin は文字列として保持されているため、実際に使用する際には stringToNode 等でノードツリーに復元する必要があります。
  • AttrDefault 配列を操作する際は、adnum の順序が維持されているか確認してください。特に比較ロジックにおいて順序の前提が崩れると不整合の原因となります。
