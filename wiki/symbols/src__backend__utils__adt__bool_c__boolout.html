boolout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:157

目的

内部的な bool 型の値を、SQLの boolean 型の出力形式である文字列 't' または 'f' に変換します。

シグネチャまたは形状

Datum boolout(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取った bool 型のデータ（PG_GETARG_BOOL(0)）。
  • **出力**: char * 型の文字列（'t' または 'f'）を Datum として返却。

前提条件と不変条件

  • palloc(2) により、文字と終端ヌル文字を格納するためのメモリ領域が確保されます。
  • 入力値が真であれば 't'、偽であれば 'f' を出力します。

アルゴリズムまたは振る舞い

1. PG_GETARG_BOOL(0) を使用して引数からブール値を取得します。 2. palloc(2) で2バイトのメモリを割り当てます。 3. 入力値が真の場合は result[0] に 't' を、偽の場合は 'f' を代入します。 4. result[1] に終端文字 '\0' を設定します。 5. PG_RETURN_CSTRING(result) を通じて結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExprInterp.c:5133 にて DirectFunctionCall1 を介した呼び出しが確認されています。
  • **呼び出し先**: PostgreSQLのメモリ管理関数 palloc を呼び出します。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • PG_FUNCTION_ARGS: 関数呼び出し規約のためのマクロ。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 本関数は単一文字の文字列を返すため、SQL標準の true/false を返す必要がある booltext とは異なる振る舞いをします。

保守担当者向けメモ

  • 本関数は内部的な出力用であり、SQL標準の文字列出力が必要な場合は booltext が使用される点に注意してください。
