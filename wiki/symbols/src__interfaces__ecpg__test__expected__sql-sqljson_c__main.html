目的

ECPG（Embedded SQL for C）における SQL/JSON 関数の動作を検証するための回帰テストを実行するメインエントリポイントです。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム実行時にデータベース ecpg1_regression へ接続）。
  • 出力: 標準出力（printf によるテスト結果の表示）および標準エラー出力（ECPGdebug によるデバッグログ）。

前提条件と不変条件

  • 実行環境において PostgreSQL データベース ecpg1_regression が利用可能であること。
  • ECPG ランタイムライブラリが正しくリンクされていること。
  • 各 SQL 文の実行後、sqlca.sqlcode が負の値でないことが期待される（エラー時は sqlprint() が呼び出される）。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、データベースへの接続とコミットモードの設定を行う。 2. **クエリ実行**: ECPGdo を使用して、json_object、json_array、json_scalar 等の SQL/JSON 関数を含むクエリを順次実行する。 3. **データバインド**: クエリ結果をホスト変数 json（char[1024]）に格納する。 4. **検証**: 各クエリ実行後に printf を呼び出し、取得した JSON 文字列を標準出力へ書き出す。 5. **エラーハンドリング**: 各ステップで sqlca.sqlcode < 0 をチェックし、エラー発生時には sqlprint() で詳細を出力する。 6. **終了**: データベース接続を閉じる。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）。
  • 呼び出し先: ECPGconnect, ECPGsetcommit, ECPGdo, sqlprint, printf（いずれも ECPG ランタイムまたは標準ライブラリ）。

関連データ構造

  • sqlca: SQL 通信領域（SQL Communication Area）。実行結果のステータス確認に使用。
  • json: クエリ結果を保持する char 配列（ホスト変数）。
  • is_json: クエリ結果を保持する bool 配列（ホスト変数）。

失敗モードと境界条件

  • データベース接続失敗時: sqlca.sqlcode が負となり、sqlprint() が実行される。
  • SQL/JSON 関数の構文エラーや実行時エラー: ECPGdo 実行後に sqlca.sqlcode を通じて検知され、sqlprint() が実行される。
  • バッファオーバーフロー: ホスト変数 json は 1024 バイトで固定されており、これを超える結果が返る場合は境界条件となる。
