ordered_set_transition

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/orderedsetaggs.c:357

目的

順序集合集約関数において、入力値をグループごとのソート用ワークスペース（tuplesort）へ蓄積する遷移関数です。

シグネチャまたは形状

Datum ordered_set_transition(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_ARGISNULL(0): 遷移状態（OSAPerGroupState）が未初期化かを確認します。
  • PG_ARGISNULL(1): 集約対象の入力値。
  • **出力**:
  • PG_RETURN_POINTER(osastate): 更新された OSAPerGroupState 構造体へのポインタを返します。

前提条件と不変条件

  • 最初の呼び出し時（状態が NULL）には ordered_set_startup が呼び出され、OSAPerGroupState が生成される必要があります。
  • tuplesort オブジェクトは osastate->sortstate に保持されている必要があります。

アルゴリズムまたは振る舞い

1. **状態の取得/初期化**: 第1引数が NULL の場合、ordered_set_startup を呼び出して OSAPerGroupState を初期化します。それ以外の場合は既存のポインタを取得します。 2. **値の蓄積**: 第2引数（入力値）が NULL でない場合、tuplesort_putdatum を使用して osastate->sortstate に値を投入します。 3. **行数カウント**: 値が投入されるたびに osastate->number_of_rows をインクリメントします。 4. **返却**: 状態ポインタを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の集約実行エンジン（src/backend/executor/nodeAgg.c 等から間接的に呼び出される）。
  • **呼び出し先**:
  • ordered_set_startup: 状態の初期化。
  • tuplesort_putdatum: ソート用バッファへの値の書き込み。

関連データ構造

  • OSAPerGroupState: グループ単位の集約状態を保持する構造体。tuplesort オブジェクトや行数カウンタを含む。

失敗モードと境界条件

  • work_mem の制限により、tuplesort が一時ファイルを作成する可能性があります。
  • 入力値が NULL の場合は無視され、ソート対象には追加されません。

保守担当者向けメモ

  • 本関数は単一カラムの集約用です。複数カラムを扱う場合は ordered_set_transition_multi が使用されます。
  • tuplesort のリソース解放は、本関数ではなく ordered_set_shutdown（コールバック）で行われるため、メモリリークを防ぐには適切なコールバック登録が必須です。
