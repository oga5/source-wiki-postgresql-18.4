MYNULLTYPE

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-outofscope.c:125

目的

ECPG（Embedded SQL for C）プリプロセッサのテストにおいて、データベースの行データに対応するインジケータ変数として機能し、各フィールドのNULL状態を追跡するために使用されます。

シグネチャまたは形状

typedef struct mynulltype {
    int id;
    int t;
    int d1;
    int d2;
    int c;
} MYNULLTYPE;

※構造体のメンバ構成は、ECPGのSQL操作におけるバインド変数へのアクセスパターンから推測されます。

入力と出力

  • **入力**: malloc によるヒープ領域の確保、および ECPGget_var を介したSQL実行結果の書き込み対象として利用されます。
  • **出力**: SQL文の実行時に、対応するデータ構造（MYTYPE）の各フィールドがNULLであるか否かを示すインジケータ値として機能します。

前提条件と不変条件

  • ECPGset_var を使用して事前に登録された変数である必要があります。
  • ECPGget_var(1) を介して取得されるポインタは、本構造体型へのキャストが前提となっています。

アルゴリズムまたは振る舞い

1. malloc によりメモリが確保されます。 2. ECPGのSQL実行マクロ（ECPGdo）において、ECPGget_var(1) を通じて取得された本構造体の各メンバ（id, t, d1, d2, c）に対し、SQLのフェッチ結果がインジケータとして書き込まれます。 3. 構造体全体が sizeof(struct mynulltype) としてサイズ指定され、ECPGランタイムによって管理されます。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数および get_var1 関数にて参照・確保されます。
  • **呼び出し先**: ECPGランタイムライブラリの ECPGget_var 関数が、本構造体へのポインタを解決するために呼び出されます。

関連データ構造

  • MYTYPE: 本構造体と対になる、実際のデータ値を保持する構造体。

失敗モードと境界条件

  • malloc によるメモリ確保失敗時、プログラムは異常終了する可能性があります。
  • ECPGget_var が期待するインデックス（1）で変数が登録されていない場合、実行時にエラーが発生します。

保守担当者向けメモ

  • 本シンボルはECPGプリプロセッサが生成したコードの一部です。手動で変更を加える際は、プリプロセッサのテンプレートや生成ルールとの整合性に注意してください。
  • 構造体のメンバ順序は、SQL文のバインド順序と厳密に一致している必要があります。
