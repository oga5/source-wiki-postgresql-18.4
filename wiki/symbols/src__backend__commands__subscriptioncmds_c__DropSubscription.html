DropSubscription

定義

  • 種別: function
  • 定義箇所: src/backend/commands/subscriptioncmds.c:1625

目的

論理レプリケーションのサブスクリプションを削除し、関連するカタログエントリ、レプリケーションスロット、ワーカープロセス、および同期状態をクリーンアップする。

シグネチャまたは形状

void DropSubscription(DropSubscriptionStmt *stmt, bool isTopLevel)

入力と出力

  • **入力**:
  • stmt: 削除対象のサブスクリプション名を含む DropSubscriptionStmt 構造体。
  • isTopLevel: コマンドがトップレベルで実行されているかを示すフラグ。
  • **出力**: なし（副作用としてシステムカタログおよびレプリケーションリソースを更新）。

前提条件と不変条件

  • 実行ユーザーはサブスクリプションの所有者である必要がある。
  • レプリケーションスロットを伴う削除の場合、トランザクションブロック内での実行は禁止される。
  • 実行中、対象サブスクリプションに対して AccessExclusiveLock を取得し、他のプロセス（レプリケーションワーカー等）による操作を排他する。

アルゴリズムまたは振る舞い

1. pg_subscription カタログから対象を検索し、存在しない場合は missing_ok に応じてエラーまたは通知を出す。 2. 所有権チェックを行い、InvokeObjectDropHook を呼び出す。 3. AccessExclusiveLock を取得し、サブスクリプションのメタデータ（名前、接続情報、スロット名）を抽出する。 4. スロットが存在する場合、トランザクションブロック内での実行を阻止する。 5. pg_subscription からタプルを削除し、関連するレプリケーションワーカーを停止させる。 6. ランチャーのワーカー開始時刻情報を削除する。 7. 各テーブルの同期状態を走査し、関連するレプリケーションオリジンを削除する。 8. 共有依存関係を削除し、最後に RemoveSubscriptionRel を呼び出して関連するリレーション同期状態をクリーンアップする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c (SQLコマンド実行時)
  • **呼び出し先**:
  • table_open, SearchSysCache2, object_ownercheck, InvokeObjectDropHook, LockSharedObject, PreventInTransactionBlock, EventTriggerSQLDropAddObject, CatalogTupleDelete, logicalrep_workers_find, logicalrep_worker_stop, ApplyLauncherForgetWorkerStartTime, GetSubscriptionRelations, replorigin_drop_by_name, deleteSharedDependencyRecordsFor, RemoveSubscriptionRel

関連データ構造

  • DropSubscriptionStmt: SQL構文解析結果。
  • pg_subscription: サブスクリプション定義を保持するシステムカタログ。
  • SubscriptionRelState: テーブル同期状態。
  • LogicalRepWorker: レプリケーションワーカーの制御構造体。

失敗モードと境界条件

  • サブスクリプションが存在しない場合、missing_ok が偽であればエラーを発生させる。
  • 所有者でない場合、aclcheck_error を発生させる。
  • トランザクションブロック内でスロット削除を伴う操作を行おうとした場合、PreventInTransactionBlock によりエラーとなる。

保守担当者向けメモ

  • 本関数は AccessExclusiveLock を保持して処理を行うため、長時間実行される可能性がある。
  • レプリケーションスロットの削除は非トランザクション的であるため、ロールバックが発生してもスロットは削除されたままとなる点に注意すること。
