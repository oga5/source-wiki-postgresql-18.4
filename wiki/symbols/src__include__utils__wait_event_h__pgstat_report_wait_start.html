pgstat_report_wait_start

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/wait_event.h:68

目的

バックエンドプロセスが待機状態（I/O待ち、ロック待ち、IPC待ちなど）に入ったことを記録し、外部から観測可能にするためのインフラストラクチャを提供します。

シグネチャまたは形状

static inline void pgstat_report_wait_start(uint32 wait_event_info)

入力と出力

  • **入力**: wait_event_info (uint32) - 待機イベントのクラスと個別のイベントIDをエンコードした値。
  • **出力**: なし。

前提条件と不変条件

  • my_wait_event_info が指すメモリ領域は、4バイトの整数として読み書き可能である必要があります。
  • 待機イベント情報は4バイトのフィールドとして扱われ、アトミックな更新が保証されています。

アルゴリズムまたは振る舞い

1. 引数として受け取った wait_event_info を、my_wait_event_info が指すメモリ領域に書き込みます。 2. この操作は volatile 修飾子を介して行われ、常に4バイト単位でアトミックに実行されます。 3. プロセス初期化前はローカル変数を、共有メモリ接続後は共有メモリ上の MyProc 構造体内の領域を更新します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/transam/clog.c
  • src/backend/access/transam/slru.c
  • src/backend/access/transam/timeline.c
  • **呼び出し先**:
  • なし（インライン関数であり、メモリへの直接書き込みのみを実行します）。

関連データ構造

  • my_wait_event_info: 現在のプロセスの待機イベント情報を保持するメモリ領域へのポインタ。
  • utils/wait_event_types.h: 待機イベントのクラスおよび個別のイベントIDの列挙型定義。

失敗モードと境界条件

  • 共有メモリ接続前に呼び出された場合、ローカル変数 local_my_wait_event_info に対して書き込みが行われます。
  • 共有メモリ接続後に呼び出された場合、共有メモリ上の領域が更新されます。
  • 常に書き込みを行う設計であり、統計情報の追跡設定（pgstat_track_activities）のチェックは行われません。

保守担当者向けメモ

  • 本関数は非常に頻繁に呼び出されるため、オーバーヘッドを最小限にするよう設計されています。変更を加える際はパフォーマンスへの影響を考慮してください。
  • 待機状態の終了時には、対となる pgstat_report_wait_end() を呼び出して値を 0 にリセットする必要があります。
