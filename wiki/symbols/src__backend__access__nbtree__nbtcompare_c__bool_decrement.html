bool_decrement

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:83

目的

B-tree インデックスのスキップスキャンにおいて、boolean 型の値を 1 段階減少させるための補助関数です。

シグネチャまたは形状

Datum bool_decrement(Relation rel, Datum existing, bool *underflow)

入力と出力

  • **入力**:
  • rel: 対象となるリレーション（本関数内では未使用）。
  • existing: 減少対象となる boolean 型の Datum 値。
  • underflow: 減少操作が範囲外となった場合に true を格納する出力用ポインタ。
  • **出力**:
  • 減少後の Datum 値。アンダーフロー時は (Datum) 0 を返します。

前提条件と不変条件

  • underflow ポインタは有効なメモリ領域を指している必要があります。
  • boolean 型の順序において false が最小値であるとみなされます。

アルゴリズムまたは振る舞い

1. existing を bool 型に変換します。 2. 値が false の場合:

  • underflow に true を設定します。
  • (Datum) 0 を返します（戻り値は未定義）。

    • 3. 値が true の場合:

  • underflow に false を設定します。
  • true (1) から 1 を引いた結果（false / 0）を Datum 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内で、SkipSupport 構造体の decrement フィールドに本関数が登録されています。
  • **呼び出し先**: 外部関数として DatumGetBool および BoolGetDatum を使用しています。

関連データ構造

  • SkipSupport: インデックススキップスキャン用の増減ロジックを保持する構造体。

失敗モードと境界条件

  • existing が false の場合、これ以上減少できないため underflow が発生します。この際、戻り値は未定義となります。

保守担当者向けメモ

  • 本関数は SkipSupport インターフェースの一部として実装されています。新しいデータ型を追加する際は、同様のパターンに従い、アンダーフロー条件を適切に設定してください。
