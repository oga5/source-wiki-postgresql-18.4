expand_single_inheritance_child

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/inherit.c:460

目的

継承またはパーティション分割されたテーブルの親テーブルに対し、個別の物理的な子テーブル（またはパーティション）に対応する RangeTblEntry (RTE) を生成し、プランナのデータ構造に登録する。

シグネチャまたは形状

static void expand_single_inheritance_child(
    PlannerInfo *root, 
    RangeTblEntry *parentrte,
    Index parentRTindex, 
    Relation parentrel,
    PlanRowMark *top_parentrc, 
    Relation childrel,
    RangeTblEntry **childrte_p,
    Index *childRTindex_p
)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • parentrte, parentrel: 親テーブルの RTE およびリレーション記述子。
  • top_parentrc: 親が FOR UPDATE/SHARE の場合に渡される行マーク情報（NULL可）。
  • childrel: 展開対象となる子テーブルのリレーション記述子。
  • **出力**:
  • childrte_p: 生成された子テーブルの RTE へのポインタ。
  • childRTindex_p: 生成された RTE のインデックス。

前提条件と不変条件

  • parentrte->rtekind は RTE_RELATION であること。
  • root->simple_rte_array および root->append_rel_array には、新しいインデックスを格納するための十分な領域が確保されていること。
  • 呼び出し元は、親テーブルの継承階層を解決済みであること。

アルゴリズムまたは振る舞い

1. **RTE生成**: 親の RTE をコピーし、子テーブルの OID、relkind、inh フラグを更新する。セキュリティ制約 (securityQuals) は空に初期化される。 2. **登録**: 生成した RTE を parse->rtable に追加し、root->simple_rte_array に格納する。 3. **AppendRelInfo生成**: 親と子の関係を定義する AppendRelInfo を作成し、root->append_rel_list に追加する。 4. **エイリアス構築**: 子テーブルの列名情報を構築し、childrte->alias および eref に設定する（EXPLAIN 出力時の列名整合性のため）。 5. **行マーク伝播**: top_parentrc が存在する場合、子テーブル用の PlanRowMark を生成し、root->rowMarks に追加する。 6. **結果リレーション登録**: 親が更新対象（UPDATE/DELETE/MERGE）である場合、子テーブルを root->all_result_relids に追加し、非パーティションテーブルであれば root->leaf_result_relids にも追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/inherit.c 内の expand_inherited_rtentry および expand_partitioned_rtentry から呼び出される。
  • **呼び出し先**: makeNode, make_append_rel_info, copyObject, makeAlias, select_rowmark_type, bms_add_member 等のユーティリティ関数。

関連データ構造

  • RangeTblEntry: クエリ内のテーブル参照情報。
  • AppendRelInfo: 継承関係における親と子の対応付け。
  • PlanRowMark: 行ロック（FOR UPDATE等）の継承管理。
  • PlannerInfo: プランニング中のグローバル状態。

失敗モードと境界条件

  • childrel がパーティションテーブルである場合、childrte->inh が true に設定され、再帰的な展開が必要であることを示す。
  • 属性が削除されている場合（attisdropped）、エイリアス構築時に空文字列が挿入される。

保守担当者向けメモ

  • 本関数は securityQuals を意図的に空に設定する。これは親の RLS 条件を継承階層全体に一貫して適用するためである。
  • childrte->alias の設定は ruleutils.c が正しい列名を出力するために不可欠である。
  • パーティションテーブルと通常の継承テーブルで AppendRelInfo の扱いが異なる点に注意すること。
