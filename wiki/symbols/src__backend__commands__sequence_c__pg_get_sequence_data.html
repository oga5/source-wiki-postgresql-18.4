pg_get_sequence_data

定義

  • 種別: function
  • 定義箇所: src/backend/commands/sequence.c:1786

目的

指定されたシーケンスリレーションの現在のメタデータ（last_value および is_called）を取得し、レコードとして返します。

シグネチャまたは形状

Datum pg_get_sequence_data(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してシーケンスの Oid を受け取ります。
  • **出力**: last_value (int8) と is_called (bool) を含む HeapTuple を Datum として返します。

前提条件と不変条件

  • 対象リレーションが RELKIND_SEQUENCE であること。
  • 呼び出し元が対象シーケンスに対する ACL_SELECT 権限を有していること。
  • 他セッションの一時シーケンスではないこと。
  • スタンバイサーバー上の非永続（unlogged）シーケンスではないこと。
  • 上記条件を満たさない場合、関数はすべてのフィールドが NULL であるタプルを返します。

アルゴリズムまたは振る舞い

1. last_value と is_called を定義するタプルディスクリプタを初期化します。 2. try_relation_open を用いて対象リレーションを AccessShareLock で開きます。 3. 前提条件のチェックを行い、適合する場合は read_seq_tuple を呼び出して物理ページからシーケンスデータを読み取ります。 4. 読み取った値を values 配列に格納し、バッファを解放します。 5. 条件不適合の場合は isnull 配列をすべて true に設定します。 6. heap_form_tuple を使用して結果タプルを構築し、返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c において、ダンプ処理中にシーケンスの状態を取得するために参照されています。
  • **呼び出し先**: read_seq_tuple（シーケンスの物理読み取り）、try_relation_open、relation_close、heap_form_tuple 等の内部関数を呼び出します。

関連データ構造

  • Form_pg_sequence_data: シーケンスの物理的な状態を保持する構造体。
  • TupleDesc: 返却されるタプルの構造定義。

失敗モードと境界条件

  • シーケンスが存在しない、または権限がない場合、エラーを発生させずに NULL を含むタプルを返します。
  • 物理読み取り時にバッファロックの競合が発生する可能性があります。

保守担当者向けメモ

  • 本関数は pg_dump 等の外部ツールから利用されるインターフェースであるため、戻り値のスキーマ（カラム順序や型）を変更する際は注意が必要です。
  • read_seq_tuple を介してディスク上の最新状態にアクセスするため、トランザクションの分離レベルに応じた一貫性が保証されます。
