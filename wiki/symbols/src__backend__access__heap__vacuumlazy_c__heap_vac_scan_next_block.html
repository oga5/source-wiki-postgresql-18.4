heap_vac_scan_next_block

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:1571

目的

ReadStream を用いたヒープスキャンにおいて、次に処理すべきブロック番号を決定するコールバック関数です。可視性マップ（VM）を活用してスキップ可能なページを判断し、効率的なシーケンシャルスキャンを維持します。

シグネチャまたは形状

BlockNumber heap_vac_scan_next_block(ReadStream *stream,
                                     void *callback_private_data,
                                     void *per_buffer_data);

入力と出力

  • **入力**:
  • stream: 使用中の ReadStream オブジェクト。
  • callback_private_data: LVRelState 構造体へのポインタ。
  • per_buffer_data: 呼び出し元が提供するバッファごとのメタデータ格納領域。
  • **出力**:
  • 次に処理すべき BlockNumber。スキャン終了時は InvalidBlockNumber を返す。
  • per_buffer_data を通じて、対象ブロックの可視性情報（VAC_BLK_ALL_VISIBLE_ACCORDING_TO_VM 等）を呼び出し元に伝達する。

前提条件と不変条件

  • callback_private_data は有効な LVRelState を指していること。
  • vacrel->current_block は前回の処理済みブロックを保持していること（初回呼び出し時は InvalidBlockNumber が 0 にオーバーフローすることを利用）。
  • next_unskippable_block は常に次に処理すべき重要なブロックを指していること。

アルゴリズムまたは振る舞い

1. **終了判定**: 次のブロック番号がリレーションの総ページ数に達した場合、next_unskippable_vmbuffer を解放し InvalidBlockNumber を返す。 2. **スキップ判定**: 現在のブロックが next_unskippable_block を超えているか、未設定の場合、find_next_unskippable_block を呼び出して次の処理対象を特定する。 3. **範囲ジャンプ**: SKIP_PAGES_THRESHOLD を超えるスキップが可能な場合、次のスキップ不可ブロックまでジャンプする。 4. **状態分岐**:

  • **スキップ可能な範囲**: VM上で全可視と判断されたブロックを処理する場合、VAC_BLK_ALL_VISIBLE_ACCORDING_TO_VM フラグをセットして返す。
  • **スキップ不可ブロック**: 凍結や削除処理が必要なブロックを処理する場合、VM情報およびEager Scan情報をフラグとしてセットして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ReadStream を利用するスキャン処理（lazy_scan_heap 等から間接的に呼び出される）。
  • **呼び出し先**: find_next_unskippable_block を呼び出す。

関連データ構造

  • LVRelState: VACUUM実行中の状態を管理する主要構造体。
  • ReadStream: ヒープ読み込みのストリーム制御。
  • BlockNumber: PostgreSQLのブロック識別子。

失敗モードと境界条件

  • **リレーション末尾**: next_block >= vacrel->rel_pages となった場合、適切にリソースを解放して終了する。
  • **VMバッファ**: next_unskippable_vmbuffer が有効な場合、終了時に必ず ReleaseBuffer を実行する。

保守担当者向けメモ

  • 本関数は ReadStream のコールバックとして設計されており、シーケンシャルスキャンの効率を損なわないよう、小さなスキップは行わない方針をとっています。
  • per_buffer_data に書き込まれるフラグは、後続の処理（lazy_scan_prune 等）でブロックの性質を判断するために使用されます。
