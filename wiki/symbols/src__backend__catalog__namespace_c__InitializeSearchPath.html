InitializeSearchPath

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/namespace.c:4738

目的

セッション開始時またはブートストラップ処理時に、名前空間の検索パス（search_path）および関連するキャッシュ状態を初期化する。

シグネチャまたは形状

void InitializeSearchPath(void)

入力と出力

  • 入力: なし（グローバルなGUC設定およびシステムカタログ状態を参照）
  • 出力: なし（内部のグローバル変数 baseSearchPath, activeSearchPath, baseSearchPathValid 等を更新）

前提条件と不変条件

  • IsBootstrapProcessingMode() が真の場合、検索パスは強制的に pg_catalog に固定される。
  • 通常モードでは、システムカタログの変更を検知するためのコールバックが登録される。

アルゴリズムまたは振る舞い

1. **ブートストラップモードの場合**:

  • 検索パスを pg_catalog のみに設定する。
  • 関連するフラグ（baseSearchPathValid 等）を初期化し、activeSearchPath を更新する。

    • 2. **通常モードの場合**:

  • NAMESPACEOID, AUTHOID, AUTHMEMROLEMEM, DATABASEOID の各システムキャッシュに対して、無効化コールバック（InvalidationCallback）を登録する。
  • baseSearchPathValid および searchPathCacheValid を false に設定し、次回の利用時に再計算を強制する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c（InitPostgres 内）
  • 呼び出し先: MemoryContextSwitchTo, list_make1_oid, GetUserId, CacheRegisterSyscacheCallback

関連データ構造

  • baseSearchPath, activeSearchPath（スキーマOIDリスト）
  • baseSearchPathValid, searchPathCacheValid（キャッシュ有効性フラグ）

失敗モードと境界条件

  • メモリ不足時に MemoryContextSwitchTo 等でエラーが発生する可能性がある。
  • ブートストラップモードと通常モードで挙動が完全に分岐するため、呼び出しタイミングが重要となる。

保守担当者向けメモ

  • 本関数はセッション初期化の重要なステップである。検索パスの再計算ロジック（recomputeNamespacePath）と密接に関連しているため、キャッシュ無効化のトリガーを追加する際は、本関数でのコールバック登録漏れに注意すること。
