TapeShare

定義

  • **種別**: struct
  • **定義箇所**: src/include/utils/logtape.h:48

目的

並列ソート処理において、ワーカープロセスが生成したマテリアライズ済み論理テープのメタデータをリーダープロセスへ伝達するための共有構造体です。

シグネチャまたは形状

typedef struct TapeShare
{
    int64 firstblocknumber;
} TapeShare;

入力と出力

  • **入力**: LogicalTapeFreeze() を呼び出す際、対象テープの凍結状態を反映した値が格納されます。
  • **出力**: LogicalTapeImport() を通じてリーダープロセスに渡され、共有された論理テープの開始位置として利用されます。

前提条件と不変条件

  • firstblocknumber は、マテリアライズされた論理テープの最初のブロック番号を指し示す必要があります。
  • 並列ソート時、リーダープロセスはワーカーから受け取ったこの構造体の配列を使用して、各ワーカーの出力を統合します。

アルゴリズムまたは振る舞い

1. **凍結**: ワーカープロセスが LogicalTapeFreeze() を実行すると、そのテープの開始位置情報が TapeShare に書き込まれます。 2. **共有**: 生成された TapeShare は、並列処理の枠組みを通じてリーダープロセスへ渡されます。 3. **インポート**: リーダープロセスは LogicalTapeImport() を呼び出し、TapeShare を用いてワーカーのテープを読み込み可能な状態として取り込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/sort/logtape.c: LogicalTapeFreeze() 内で書き込まれ、LogicalTapeImport() 内で読み込まれます。
  • src/backend/utils/sort/tuplesort.c: 並列ソートのメタデータ管理（tapes 配列など）において使用されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • LogicalTape: 論理テープの操作ハンドル。
  • LogicalTapeSet: 複数の論理テープを管理するコンテナ。
  • SharedFileSet: 並列処理における共有一時ファイルセット。

失敗モードと境界条件

  • リーダープロセス側で作成された空のテープに対しては、書き込みができないという制約があります。
  • TapeShare の内容が不正なブロック番号である場合、LogicalTapeImport 後の読み込み操作でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は現在 firstblocknumber のみを保持していますが、将来的に共有が必要なメタデータが増えた場合はこの構造体を拡張してください。
  • tuplesort.c における FLEXIBLE_ARRAY_MEMBER を用いた配列管理と整合性を保つ必要があります。
