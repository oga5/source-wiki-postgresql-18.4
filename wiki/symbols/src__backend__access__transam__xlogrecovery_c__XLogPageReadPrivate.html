XLogPageReadPrivate

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xlogrecovery.c:196

目的

WAL読み込み処理（XLogPageRead等）において、リカバリ実行中の状態や制御情報を保持するためのプライベートデータ構造です。

シグネチャまたは形状

typedef struct XLogPageReadPrivate
{
    int         emode;
    bool        fetching_ckpt;  /* チェックポイントレコード取得中か */
    bool        randAccess;
    TimeLineID  replayTLI;
} XLogPageReadPrivate;

入力と出力

本構造体は XLogReaderState の private_data フィールドを通じて、WAL読み込み関数群に渡されます。

前提条件と不変条件

  • XLogReaderState の private_data として割り当てられ、リカバリプロセス内で利用されます。
  • palloc0 等で初期化されることが想定されています。

アルゴリズムまたは振る舞い

  • XLogPageRead などの関数内で、xlogreader->private_data を本構造体にキャストして参照します。
  • fetching_ckpt は、現在チェックポイントレコードを読み込んでいる最中であるかを示すフラグとして機能します。
  • replayTLI は、現在リプレイ（適用）対象としているタイムラインIDを保持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c: リカバリ開始時に palloc0 で割り当てられ、XLogReaderState に設定されます。
  • src/bin/pg_rewind/parsexlog.c: pg_rewind ツール内でも同様の構造体が定義・利用されています。
  • **呼び出し先**:
  • XLogPageRead などのWAL読み込み関連関数が、この構造体を通じてリカバリ状態を参照します。

関連データ構造

  • XLogReaderState: WAL読み込みのメイン状態管理構造体。本構造体はこの内部の private_data として格納されます。

失敗モードと境界条件

  • xlogreader->private_data が正しくキャストされていない場合、メモリ破壊や不正な値の参照が発生します。
  • palloc0 によるメモリ確保が失敗した場合、リカバリプロセスは異常終了します。

保守担当者向けメモ

  • 本構造体は xlogrecovery.c と pg_rewind/parsexlog.c の両方で定義されています。変更を加える際は、両方の実装に影響がないか確認してください。
  • XLogReaderState の private_data を介したアクセスであるため、型安全性の確保には注意が必要です。
