bbsink_copystream_begin_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_copy.c:164

目的

バックアップストリームにおいて、新しいアーカイブファイルの開始をクライアントに通知する。

シグネチャまたは形状

static void bbsink_copystream_begin_archive(bbsink *sink, const char *archive_name)

入力と出力

  • **入力**:
  • sink: bbsink 構造体へのポインタ。
  • archive_name: 開始するアーカイブの名前。
  • **出力**: なし（副作用としてネットワーク経由でメッセージを送信）。

前提条件と不変条件

  • sink->bbs_state が有効な bbsink_state を指していること。
  • state->tablespaces リスト内に現在の tablespace_num に対応する tablespaceinfo が存在すること。

アルゴリズムまたは振る舞い

1. sink->bbs_state から現在のテーブルスペース情報を取得する。 2. PqMsg_CopyData メッセージを開始する。 3. タイプバイト 'n'（新規アーカイブ）を送信する。 4. archive_name を送信する。 5. 現在のテーブルスペースのパス（NULLの場合は空文字列）を送信する。 6. メッセージを終了し、クライアントへ送信する。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_copystream_ops 構造体を通じて、バックアップ処理のメインループから呼び出される（直接の呼び出し元コードは提供されていない）。
  • **呼び出し先**: pq_beginmessage, pq_sendbyte, pq_sendstring, pq_endmessage（libpqプロトコル関数）。

関連データ構造

  • bbsink: バックアップシンクのインターフェース構造体。
  • bbsink_state: ストリームの状態を保持する構造体。
  • tablespaceinfo: テーブルスペースの情報を保持する構造体。

失敗モードと境界条件

  • ti->path が NULL の場合、空文字列として送信する安全策が講じられている。
  • メッセージ送信失敗時のエラーハンドリングは pq_ 系関数に依存する。

保守担当者向けメモ

  • 本関数は bbsink_copystream_ops の begin_archive フィールドに登録されている。
  • 送信されるタイプバイト 'n' は、クライアント側でのストリーム解釈の起点となる。
