RunNamespaceSearchHook

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:114

目的

名前空間（Namespace）検索操作に対する外部プラグインの介入を可能にするフック関数です。セキュリティ拡張などが、特定の名前空間へのアクセスを許可するか否かを判定するために使用されます。

シグネチャまたは形状

bool RunNamespaceSearchHook(Oid objectId, bool ereport_on_violation)

入力と出力

  • **入力**:
  • objectId (Oid): 検索対象となる名前空間のオブジェクトID。
  • ereport_on_violation (bool): 権限違反時にエラーを報告するかどうかを指定するフラグ。
  • **出力**:
  • bool: 検索操作が許可される場合は true、拒否される場合は false を返します。

前提条件と不変条件

  • グローバル変数 object_access_hook が非NULLであることが前提です（Assert により検証されます）。
  • 呼び出し元は、フックが登録されていることを確認する責任を負います。

アルゴリズムまたは振る舞い

1. ObjectAccessNamespaceSearch 構造体をスタック上に確保し、ゼロ初期化します。 2. 引数 ereport_on_violation を構造体に設定し、デフォルトの result を true に初期化します。 3. 登録されている object_access_hook 関数を呼び出し、イベントタイプ OAT_NAMESPACE_SEARCH、カタログID NamespaceRelationId、および対象の objectId を渡します。 4. フック関数による判定結果（ns_arg.result）を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/namespace.c などの名前空間検索処理を行うコアコード（具体的な呼び出し箇所はソースコードの文脈から推測されます）。
  • **呼び出し先**: グローバル変数 object_access_hook に登録された外部プラグインの関数。

関連データ構造

  • ObjectAccessNamespaceSearch: フックに渡される引数構造体。ereport_on_violation と result を保持します。
  • OAT_NAMESPACE_SEARCH: フックの種類を識別する列挙型定数。
  • NamespaceRelationId: pg_namespace カタログのOID。

失敗モードと境界条件

  • object_access_hook が NULL の状態で呼び出された場合、アサーション違反によりプロセスが停止します。
  • フック関数が result を false に書き換えた場合、呼び出し元はアクセス拒否として処理を継続します。

保守担当者向けメモ

  • 本関数はフックの実行口であり、実際の判定ロジックはロードされた共有ライブラリ内に存在します。
  • 名前ベースでの識別が必要な場合は、対となる RunNamespaceSearchHookStr の使用を検討してください。
