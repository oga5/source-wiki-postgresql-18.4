do_syncfs

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:60

目的

指定されたパス（ファイルまたはディレクトリ）を含むファイルシステム全体を同期（syncfs）し、データの永続化を試みます。

シグネチャまたは形状

static void do_syncfs(const char *path)

入力と出力

  • **入力**: path (同期対象のファイルまたはディレクトリへのパスを示す文字列)
  • **出力**: なし（戻り値は void）

前提条件と不変条件

  • path で指定されたパスは、OSの open システムコールで読み取り専用（O_RDONLY）として開ける必要があります。
  • 実行環境のOSが syncfs システムコールをサポートしていることが前提です。

アルゴリズムまたは振る舞い

1. open(path, O_RDONLY, 0) を呼び出し、対象のファイル記述子を取得します。 2. 開放に失敗した場合、pg_log_error でエラーを出力し、関数を終了します。 3. 取得したファイル記述子に対して syncfs(fd) を実行し、ファイルシステムレベルの同期を試みます。 4. syncfs が失敗した場合、pg_log_error でエラーを出力し、ファイル記述子を閉じた上で exit(EXIT_FAILURE) を呼び出し、プロセスを異常終了させます。 5. 成功した場合、ファイル記述子を close(fd) して正常に終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/fd.c 内の複数箇所
  • src/common/file_utils.c 内の sync_pgdata 等（詳細は索引を参照）
  • **呼び出し先**:
  • 標準ライブラリ: open, syncfs, close, exit
  • PostgreSQL内部: pg_log_error

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **ファイルオープン失敗**: path が存在しない、または権限不足により open が失敗した場合、エラーログを出力して復帰します。
  • **同期失敗**: syncfs がシステムエラーを返した場合、プロセスは EXIT_FAILURE で強制終了します。

保守担当者向けメモ

  • 本関数は static であり、主に src/common/file_utils.c および src/backend/storage/file/fd.c から利用されます。
  • syncfs はファイルシステム全体を同期するため、特定のファイルのみを同期したい場合は fsync_fname 等の代替手段を検討してください。
