findNewestTimeLine

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/timeline.c:263

目的

指定された開始タイムラインID（startTLI）から順に、存在するタイムライン履歴ファイルを探索し、現在利用可能な最新のタイムラインIDを特定します。

シグネチャまたは形状

TimeLineID findNewestTimeLine(TimeLineID startTLI)

入力と出力

  • **入力**: startTLI (TimeLineID) - 探索を開始する基準となるタイムラインID。
  • **出力**: newestTLI (TimeLineID) - 探索の結果、見つかった最新のタイムラインID。

前提条件と不変条件

  • タイムライン履歴ファイルは、startTLI + 1 から順に連続して存在することが想定されています。

アルゴリズムまたは振る舞い

1. newestTLI を startTLI で初期化します。 2. probeTLI を startTLI + 1 から開始し、インクリメントしながらループします。 3. existsTimeLineHistory(probeTLI) を呼び出し、該当する履歴ファイルが存在するか確認します。 4. 存在する場合は newestTLI を probeTLI に更新し、探索を続行します。 5. 存在しない場合はループを終了し、現在の newestTLI を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogrecovery.c
  • **呼び出し先**:
  • existsTimeLineHistory (同ファイル内)

関連データ構造

  • TimeLineID: タイムラインを識別するための整数型。

失敗モードと境界条件

  • 履歴ファイルが存在しない場合、入力された startTLI がそのまま返されます。
  • タイムラインIDのシーケンスに欠番がある場合、その欠番の直前で探索が停止します。

保守担当者向けメモ

  • 本関数は、タイムライン履歴ファイルが連続していることを前提としています。シーケンス内の欠番を許容すべきかについては、ソースコード内に検討の余地（XXX）が示されています。
