JsonIterateStringValuesAction

定義

  • 種別: typedef
  • 定義箇所: src/include/utils/jsonfuncs.h:35

目的

JSON または JSONB データの反復処理中に、特定の文字列要素に対してユーザー定義の処理を適用するためのコールバック関数型です。

シグネチャまたは形状

typedef void (*JsonIterateStringValuesAction) (void *state, char *elem_value, int elem_len);

入力と出力

  • **入力**:
  • void *state: 反復処理の状態を保持するポインタ。
  • char *elem_value: 抽出された文字列要素の先頭ポインタ。
  • int elem_len: 文字列要素の長さ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本コールバックは、JSON/JSONB の反復処理関数（iterate_json_values や iterate_jsonb_values 等）から呼び出されることを前提としています。
  • elem_value は有効な文字列バッファを指している必要があります。

アルゴリズムまたは振る舞い

JSON/JSONB データの走査中に、指定された条件（JsonToIndex フラグ等）に合致する文字列要素が見つかった際、その要素のポインタと長さを引数として本関数が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonfuncs.c 内の反復処理関数群から参照されています。
  • **呼び出し先**: 本シンボル自体は関数ポインタ型であり、具体的な実装は呼び出し側で提供される関数に依存します。

関連データ構造

  • JsonToIndex: 反復処理対象を指定するビットフラグ。
  • JsonSemAction: JSON 解析のセマンティックアクションを定義する構造体。

失敗モードと境界条件

  • コールバック関数内でのメモリ割り当てやエラー処理は、呼び出し元のコンテキスト（escontext 等）の規約に従う必要があります。
  • elem_len が 0 の場合や、elem_value が NULL の場合の挙動は、実装側の定義に依存します。

保守担当者向けメモ

  • 本シンボルは jsonfuncs.c で実装される反復処理のインターフェースの一部です。
  • 新しい反復処理ロジックを追加する際は、既存の JsonIterateStringValuesAction を利用する関数群（jsonfuncs.c の 5661 行目付近等）の設計を参考にしてください。
