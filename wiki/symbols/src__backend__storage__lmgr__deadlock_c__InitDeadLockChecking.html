InitDeadLockChecking

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/deadlock.c:143

目的

バックエンド起動時に、デッドロック検出および解決アルゴリズムで使用する作業用メモリ領域を確保・初期化する。

シグネチャまたは形状

void InitDeadLockChecking(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルな作業用ポインタ変数が初期化される）

前提条件と不変条件

  • TopMemoryContext が存在すること。
  • 確保されるメモリは永続的である必要があるため、TopMemoryContext に切り替えて割り当てを行う。
  • MaxBackends に基づいて必要な領域サイズが計算される。

アルゴリズムまたは振る舞い

1. TopMemoryContext へメモリコンテキストを切り替える。 2. MaxBackends を基準に、以下の作業用領域を palloc で確保する:

  • visitedProcs, deadlockDetails, topoProcs (再利用), beforeConstraints, afterConstraints
  • waitOrders, waitOrderProcs
  • curConstraints (最大制約数 maxCurConstraints を設定)
  • possibleConstraints (最大 maxPossibleConstraints を設定)

    • 3. 最後に元のメモリコンテキストへ復帰する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/lmgr/proc.c (バックエンド初期化プロセス)
  • 呼び出し先:
  • MemoryContextSwitchTo
  • palloc

関連データ構造

  • PGPROC: プロセス情報。
  • DEADLOCK_INFO: デッドロック詳細情報。
  • WAIT_ORDER: 待ち行列の順序情報。
  • EDGE: グラフ上のエッジ（制約）。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • maxCurConstraints の設定値は DeadLockCheckRecurse の再帰深さを制限する。
  • possibleConstraints のサイズは MaxBackends * 4 で固定されており、これを超える制約が発生した場合は TestConfiguration の再実行が必要となる。

保守担当者向けメモ

  • 本関数はバックエンド起動時に一度だけ実行される。
  • シグナルハンドラから呼び出される可能性があるため、メモリ確保は必ずこの初期化フェーズで完了させておく必要がある。
  • topoProcs は visitedProcs の領域を再利用しているため、同時に使用されないことを前提としている。
