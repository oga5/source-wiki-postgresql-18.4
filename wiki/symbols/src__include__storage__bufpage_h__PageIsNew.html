PageIsNew

定義

  • 種別: function
  • 定義箇所: src/include/storage/bufpage.h:234

目的

指定されたページが初期化されていない（空の）状態であるかどうかを判定します。

シグネチャまたは形状

static inline bool PageIsNew(const PageData *page)

入力と出力

  • **入力**: page (const PageData *) - 検査対象のページへのポインタ。
  • **出力**: bool - ページヘッダーの pd_upper が 0 であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力された page は有効なメモリ領域を指している必要があります。
  • ページヘッダー構造（PageHeaderData）としてキャスト可能なメモリレイアウトであることを前提としています。

アルゴリズムまたは振る舞い

1. 引数 page を PageHeaderData 型のポインタへキャストします。 2. 構造体内の pd_upper フィールドを参照します。 3. pd_upper が 0 である場合、そのページはまだ初期化されていない（新規である）と判断し true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • contrib/bloom/blinsert.c, blscan.c, blutils.c, blvacuum.c
  • contrib/pageinspect/brinfuncs.c
  • **呼び出し先**:
  • 外部関数呼び出しはありません（インライン展開される単純な構造体メンバ参照です）。

関連データ構造

  • PageData: ページの実体。
  • PageHeaderData: ページ先頭に配置されるメタデータ構造体。pd_upper フィールドを含みます。

失敗モードと境界条件

  • page が NULL の場合、あるいは PageHeaderData を保持するのに十分なサイズがないメモリ領域を指している場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本関数は、ページが初期化済みかどうかを判断する最も基本的なチェックです。
  • pd_upper はページ内の空き領域の終了位置を示すため、初期化直後の状態（0）を判定基準として利用しています。
