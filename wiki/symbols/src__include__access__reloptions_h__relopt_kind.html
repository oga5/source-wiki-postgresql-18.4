relopt_kind

定義

  • 種別: enum
  • 定義箇所: src/include/access/reloptions.h:39

目的

リレーション（テーブル、インデックス等）やテーブルスペースに対して設定可能なオプションが、どのオブジェクトタイプに適用可能かを識別するためのビットマスクとして機能します。

シグネチャまたは形状

typedef enum relopt_kind
{
    RELOPT_KIND_LOCAL = 0,
    RELOPT_KIND_HEAP = (1 << 0),
    RELOPT_KIND_TOAST = (1 << 1),
    RELOPT_KIND_BTREE = (1 << 2),
    RELOPT_KIND_HASH = (1 << 3),
    RELOPT_KIND_GIN = (1 << 4),
    RELOPT_KIND_GIST = (1 << 5),
    RELOPT_KIND_ATTRIBUTE = (1 << 6),
    RELOPT_KIND_TABLESPACE = (1 << 7),
    RELOPT_KIND_SPGIST = (1 << 8),
    RELOPT_KIND_VIEW = (1 << 9),
    RELOPT_KIND_BRIN = (1 << 10),
    RELOPT_KIND_PARTITIONED = (1 << 11),
    RELOPT_KIND_LAST_DEFAULT = RELOPT_KIND_PARTITIONED,
    RELOPT_KIND_MAX = (1 << 30)
} relopt_kind;

入力と出力

  • 入力: オプション定義の登録時や解析時に、対象となるオブジェクトの種類を指定するために使用されます。
  • 出力: add_reloption_kind() 関数により、カスタムオプション用に新しい relopt_kind 値が生成・返却されます。

前提条件と不変条件

  • ビットマスクとして定義されており、各値は 1 << n の形式をとります。
  • RELOPT_KIND_MAX は 1 << 30 であり、符号付き整数としてのオーバーフローを避けるため 1 << 31 は使用されません。
  • 新しい種類を追加する際は、RELOPT_KIND_LAST_DEFAULT を更新する必要があります。

アルゴリズムまたは振る舞い

  • オプションの適用範囲を制限するために使用されます。例えば、parseRelOptions や default_reloptions などの関数において、特定のオブジェクトタイプに対して有効なオプションのみをフィルタリングまたは検証する際の識別子として機能します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/access/common/reloptions.c: parseRelOptions, default_reloptions 等の関数で引数として利用。
  • contrib/bloom/blutils.c および src/test/modules/dummy_index_am/dummy_index_am.c: カスタムの relopt_kind を保持する静的変数として利用。
  • 呼び出し先:
  • add_reloption_kind(): 新しい種類を動的に割り当てるために呼び出されます。

関連データ構造

  • relopt_gen: オプション定義の共通ヘッダー。
  • relopt_value: 解析済みのオプション値を保持する構造体。

失敗モードと境界条件

  • RELOPT_KIND_MAX を超えるビット割り当ては行えません。
  • 誤った relopt_kind を指定してオプション解析を行うと、対象外のオプションが誤って適用される、あるいは検証がスキップされる可能性があります。

保守担当者向けメモ

  • 新しいアクセスメソッドやオブジェクトタイプを追加する際は、本列挙型に新しいビットを追加し、RELOPT_KIND_LAST_DEFAULT を更新してください。
  • カスタムオプションが必要な場合は、add_reloption_kind() を使用して動的に割り当てることが推奨されます。
