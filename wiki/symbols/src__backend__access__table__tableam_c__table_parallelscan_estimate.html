table_parallelscan_estimate

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableam.c:130

目的

並列スキャンを実行するために必要な共有メモリ領域のサイズ（バイト単位）を見積もります。

シグネチャまたは形状

Size table_parallelscan_estimate(Relation rel, Snapshot snapshot)

入力と出力

  • **入力**:
  • rel: 対象となるリレーション（Relation）。
  • snapshot: スキャンに使用するスナップショット（Snapshot）。
  • **出力**:
  • Size: 並列スキャン状態を保持するために必要な共有メモリの合計サイズ。

前提条件と不変条件

  • snapshot が MVCC スナップショットである場合、そのスナップショットを共有メモリに格納するためのサイズが見積もりに加算されます。
  • snapshot が MVCC スナップショットでない場合、SnapshotAny であることがアサーションにより強制されます。

アルゴリズムまたは振る舞い

1. 初期サイズ sz を 0 に設定します。 2. snapshot が MVCC スナップショットであれば、EstimateSnapshotSpace を用いてスナップショットの格納に必要なサイズを sz に加算します。それ以外の場合は SnapshotAny であることを確認します。 3. リレーションのアクセスメソッド（rel->rd_tableam）が提供する parallelscan_estimate 関数を呼び出し、AM 固有の並列スキャン状態サイズを sz に加算します。 4. 計算された合計サイズ sz を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/executor/nodeSeqscan.c
  • **呼び出し先**:
  • EstimateSnapshotSpace
  • rel->rd_tableam->parallelscan_estimate

関連データ構造

  • Relation: テーブルのメタデータおよびアクセスメソッドへのポインタを保持。
  • Snapshot: トランザクションの可視性判定に使用される構造体。
  • TableAmRoutine: rel->rd_tableam が指す、AM 固有の関数ポインタ群。

失敗モードと境界条件

  • snapshot が MVCC スナップショットでも SnapshotAny でもない場合、アサーション違反が発生します。
  • add_size 関数により、サイズ計算時のオーバーフローがチェックされます。

保守担当者向けメモ

  • 本関数は、並列クエリ実行時の共有メモリ割り当て計画において重要な役割を果たします。
  • AM 固有のサイズ見積もりは rel->rd_tableam->parallelscan_estimate に委譲されているため、特定のストレージエンジン（heap 等）の挙動を変更する場合は、該当する AM の実装を確認してください。
