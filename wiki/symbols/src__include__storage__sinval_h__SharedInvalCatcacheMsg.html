SharedInvalCatcacheMsg

定義

  • 種別: struct
  • 定義箇所: src/include/storage/sinval.h:61

目的

システムカタログキャッシュ（Catcache）の無効化イベントを、共有メモリを介して他のバックエンドプロセスへ通知するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct
{
	int8		id;				/* cache ID --- must be first */
	Oid			dbId;			/* database ID, or 0 if a shared relation */
	uint32		hashValue;		/* hash value of key for this catcache */
} SharedInvalCatcacheMsg;

入力と出力

  • **入力**: キャッシュ無効化のトリガーとなるカタログ変更時に、id（キャッシュ識別子）、dbId（対象データベース）、hashValue（無効化対象エントリのハッシュ値）が設定されます。
  • **出力**: 共有メモリ上の無効化キューに書き込まれ、他のバックエンドによって読み取られます。

前提条件と不変条件

  • id フィールドは構造体の先頭に配置される必要があり、メッセージの型識別子として機能します。
  • dbId が 0 の場合、共有リレーションを指します。

アルゴリズムまたは振る舞い

1. バックエンドがシステムカタログを更新すると、本構造体に情報を格納し、共有メモリ上の無効化キューへ投入します。 2. 他のバックエンドは ReceiveSharedInvalidMessages を通じて本メッセージを読み取ります。 3. 受信側は hashValue を用いて自身のローカルキャッシュ内の該当エントリを特定し、無効化（または再ロード）を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: 共有キャッシュ無効化メカニズム（SendSharedInvalidMessages 等）から利用されます。具体的な呼び出し元関数の詳細は、提供されたソース範囲内では特定できません。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SharedInvalidationMessage: 本構造体を包含する共用体。
  • src/include/storage/sinvaladt.h: 共有メモリ上の無効化キュー管理構造。

失敗モードと境界条件

  • **ハッシュ衝突**: hashValue はハッシュ値ベースであるため、稀に無関係なエントリが誤って無効化される可能性があります。これはキャッシュの再ロードによって整合性が保たれる設計となっています。

保守担当者向けメモ

  • 本構造体は SharedInvalidationMessage の一部として扱われるため、サイズ変更やフィールド追加を行う際は、共用体全体の整合性に注意してください。
