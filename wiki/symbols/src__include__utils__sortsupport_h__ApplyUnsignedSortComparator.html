ApplyUnsignedSortComparator

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/sortsupport.h:232

目的

符号なし整数型（または同等のビット表現を持つ型）の Datum 値を、NULL 処理とソート順序（昇順/降順）を考慮して比較する。

シグネチャまたは形状

int ApplyUnsignedSortComparator(Datum datum1, bool isNull1,
                                Datum datum2, bool isNull2,
                                SortSupport ssup)

入力と出力

  • **入力**:
  • datum1, datum2: 比較対象の Datum 値。
  • isNull1, isNull2: 各値が NULL かどうかを示すフラグ。
  • ssup: ソート設定を保持する SortSupport 構造体（ssup_nulls_first および ssup_reverse を参照）。
  • **出力**:
  • int: 比較結果。datum1 < datum2 なら -1、datum1 > datum2 なら 1、等価なら 0 を返す。

前提条件と不変条件

  • ssup は有効な SortSupportData 構造体を指している必要がある。
  • datum1 および datum2 は、符号なしの比較に適した形式であること。

アルゴリズムまたは振る舞い

1. **NULL 判定**:

  • 両方が NULL の場合は 0 を返す。
  • 片方のみが NULL の場合、ssup->ssup_nulls_first に基づき、NULL を先頭（-1）または末尾（1）として扱う。

    • 2. **値の比較**:

  • NULL でない場合、datum1 と datum2 を直接比較する。

    • 3. **順序の反転**:

  • ssup->ssup_reverse が真の場合、比較結果を反転させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesort.c:499 での呼び出しが確認されている。
  • **呼び出し先**: INVERT_COMPARE_RESULT マクロ（比較結果の反転に使用）。

関連データ構造

  • SortSupportData: ソートの挙動（NULL の扱い、昇順/降順）を定義する構造体。

失敗モードと境界条件

  • ssup が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • 符号付き整数を渡した場合、ビット表現によっては意図しない順序で比較される可能性がある。

保守担当者向けメモ

  • 本関数はインライン化されており、ソートのループ内での呼び出しコストを最小化するように設計されている。
  • 比較ロジックを変更する場合は、ApplySortComparator 等の他の比較関数との整合性に注意すること。
