PGTYPESinterval_to_asc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/interval.c:1061

目的

interval 構造体の内部表現を、人間が読み取り可能な文字列形式に変換する。

シグネチャまたは形状

char *PGTYPESinterval_to_asc(interval * span)

入力と出力

  • **入力**: interval *span（変換対象の時間間隔構造体へのポインタ）
  • **出力**: 変換後の文字列を保持する動的に確保されたメモリ領域へのポインタ。失敗した場合は NULL を返し、errno に PGTYPES_INTVL_BAD_INTERVAL を設定する。

前提条件と不変条件

  • span は有効な interval 構造体を指している必要がある。
  • 出力文字列は pgtypes_strdup によって確保されるため、呼び出し元で free() を使用して解放する責任がある。
  • 内部的に INTSTYLE_POSTGRES_VERBOSE 形式が強制される。

アルゴリズムまたは振る舞い

1. interval2tm を呼び出し、interval 構造体を struct tm（年・月・日・時・分・秒）と fsec_t（小数秒）に分解する。 2. 分解に失敗した場合は errno を設定し、NULL を返す。 3. EncodeInterval を呼び出し、INTSTYLE_POSTGRES_VERBOSE スタイルに従って一時バッファ buf に文字列をシリアライズする。 4. pgtypes_strdup を使用してバッファの内容をヒープにコピーし、そのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test2.c
  • src/interfaces/ecpg/test/expected/sql-array.c
  • **呼び出し先**:
  • interval2tm
  • EncodeInterval
  • pgtypes_strdup

関連データ構造

  • interval: 時間間隔を保持する構造体。
  • struct tm: 日時要素を保持する標準構造体。
  • fsec_t: 小数秒を保持する型。

失敗モードと境界条件

  • interval2tm が変換に失敗した場合、PGTYPES_INTVL_BAD_INTERVAL エラーが発生する。
  • MAXDATELEN を超える文字列長が必要な場合、バッファオーバーフローの可能性がある（buf は MAXDATELEN + 1 で定義されている）。

保守担当者向けメモ

  • 本関数は INTSTYLE_POSTGRES_VERBOSE をハードコードしているため、他の出力スタイルが必要な場合は修正が必要。
  • struct tm を使用しているため、移植元の PostgreSQL バックエンド実装との差異（struct pg_tm との混同）に注意すること。
