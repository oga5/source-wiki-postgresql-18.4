QTNClearFlags

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_util.c:433

目的

QTNode ツリー内の各ノードに設定されている特定のフラグを、再帰的にクリア（解除）します。

シグネチャまたは形状

void QTNClearFlags(QTNode *in, uint32 flags)

入力と出力

  • **入力**:
  • in: 対象となる QTNode ツリーのルートノード。
  • flags: クリア対象とするビットマスク。
  • **出力**: なし（副作用として in およびその子ノードの flags メンバが更新されます）。

前提条件と不変条件

  • in は有効な QTNode 構造体を指している必要があります。
  • 再帰処理を行うため、スタックの深さが制限を超えないことが前提です。

アルゴリズムまたは振る舞い

1. check_stack_depth() を呼び出し、スタックオーバーフローを防止します。 2. 現在のノード in の flags メンバに対して、ビット演算 &= ~flags を行い、指定されたフラグをオフにします。 3. 現在のノードが値ノード（type == QI_VAL）でない場合、すべての子ノードに対して再帰的に QTNClearFlags を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_rewrite.c:375
  • **呼び出し先**:
  • check_stack_depth()
  • QTNClearFlags（自身を再帰呼び出し）

関連データ構造

  • QTNode: 全文検索クエリを表現するツリー構造体。

失敗モードと境界条件

  • **スタックオーバーフロー**: 非常に深いツリー構造に対して実行した場合、check_stack_depth() によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は再帰的に動作するため、ツリーの構造変更を行う際はスタック消費量に注意してください。
  • flags のビット定義については tsearch/ts_utils.h を参照してください。
