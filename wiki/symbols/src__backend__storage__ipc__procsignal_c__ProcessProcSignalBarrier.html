ProcessProcSignalBarrier

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procsignal.c:499

目的

プロセスが保留中のシグナルバリアを処理し、共有メモリ上のバリア生成番号を更新して待機中のプロセスに通知する。

シグネチャまたは形状

void ProcessProcSignalBarrier(void)

入力と出力

  • **入力**: 共有メモリ上の MyProcSignalSlot（自身のスロット）および ProcSignal（グローバルヘッダー）。
  • **出力**: なし（副作用として共有メモリの状態更新と条件変数のブロードキャストを行う）。

前提条件と不変条件

  • MyProcSignalSlot が初期化されていること。
  • local_gen <= shared_gen であること（自身のバリア生成番号がグローバルな生成番号を超えない）。

アルゴリズムまたは振る舞い

1. ProcSignalBarrierPending フラグを確認し、偽であれば即座に終了する。 2. 自身の pss_barrierGeneration とグローバルの psh_barrierGeneration を比較し、一致すれば処理済みとして終了する。 3. pss_barrierCheckMask から処理すべきバリアフラグをアトミックに取得・クリアする。 4. PG_TRY ブロック内で、取得したフラグに基づき各バリア処理関数（例: ProcessBarrierSmgrRelease）を実行する。 5. 処理に失敗したバリアがある場合、ResetProcSignalBarrierBits を呼び出してフラグを再設定し、後続の試行に備える。 6. 全ての処理が完了後、自身の pss_barrierGeneration をグローバル値に更新し、pss_barrierCV をブロードキャストして待機中のプロセスを解放する。

関連データ構造

  • ProcSignalSlot: プロセスごとのシグナル状態を保持。
  • ProcSignalHeader: 全プロセスのバリア生成番号を管理。
  • ProcSignalBarrierType: バリアの種類を定義する列挙型。

失敗モードと境界条件

  • **処理失敗**: バリア処理関数が false を返した場合、ResetProcSignalBarrierBits によりフラグが復元され、次回以降の CHECK_FOR_INTERRUPTS 等で再試行される。
  • **エラー発生**: PG_CATCH ブロックにより、エラー発生時も未処理のバリアフラグが復元される。
  • **競合**: 複数のバリアが連続して発生した場合、生成番号の比較により不要な処理をスキップする。

保守担当者向けメモ

  • バリア処理関数は高コストであるため、頻繁な呼び出しは避けること。
  • pss_barrierCheckMask のクリアとバリア処理の順序は、レースコンディションを防ぐために厳密に設計されている。
