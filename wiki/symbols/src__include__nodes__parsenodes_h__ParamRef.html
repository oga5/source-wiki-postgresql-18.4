ParamRef

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/parsenodes.h:317

目的

SQL文中の明示的なパラメータ参照（例: $1, $2）をパースツリー内で表現するためのノード構造体です。

シグネチャまたは形状

typedef struct ParamRef
{
	NodeTag		type;
	int			number;			/* パラメータ番号 */
	ParseLoc	location;		/* トークン位置、不明な場合は -1 */
} ParamRef;

入力と出力

  • **入力**: SQL解析時にパーサーが生成するノード。
  • **出力**: 意味解析（transformParamRef 等）を経て、実行可能な Param ノード等へ変換されます。

前提条件と不変条件

  • number は 1 から始まるパラメータのインデックスを保持します。
  • location はソースコード内のバイトオフセットを指し、エラー報告に使用されます。

アルゴリズムまたは振る舞い

パーサーによって生成された後、解析器（parse_analyze）の過程で、transformParamRef などの関数を通じて意味的な型情報や値の解決が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/parser/parse_expr.c: transformParamRef 内で変換処理。
  • src/backend/nodes/params.c: paramlist_param_ref 等でパラメータリストとの照合に使用。
  • src/backend/executor/functions.c: sql_fn_param_ref で関数実行時のパラメータ解決に使用。
  • src/backend/parser/parse_param.c: fixed_paramref_hook および variable_paramref_hook で参照。
  • src/backend/nodes/nodeFuncs.c: location フィールドの抽出に使用。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • NodeTag: ノードの種類を識別するタグ。
  • ParseLoc: SQL文字列内の位置情報。

失敗モードと境界条件

  • location が不明な場合、値は -1 となります。
  • パラメータ番号が範囲外である場合や、未定義のパラメータが参照された場合は、後続の解析フェーズでエラーが発生します。

保守担当者向けメモ

  • 本構造体は「生のパースツリー」の一部です。解析後のクエリツリーでは Param ノードに置き換わるため、プランナーやエグゼキューターで直接扱うことはありません。
