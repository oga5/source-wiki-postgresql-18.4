toast_tuple_externalize

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/toast_helper.c:255

目的

指定された属性値を外部ストレージ（TOASTテーブル）へ移動させ、タプルのインラインサイズを削減する。

シグネチャまたは形状

void toast_tuple_externalize(ToastTupleContext *ttc, int attribute, int options)

入力と出力

  • **入力**:
  • ttc: TOAST処理のコンテキストを保持する ToastTupleContext 構造体へのポインタ。
  • attribute: 外部化対象となる属性のインデックス。
  • options: toast_save_datum に渡されるオプションフラグ。
  • **出力**: なし（副作用として ttc 内のデータおよびフラグを更新）。

前提条件と不変条件

  • ttc は toast_tuple_init によって適切に初期化されていること。
  • 指定された attribute は ttc 内で有効な範囲内であること。

アルゴリズムまたは振る舞い

1. 対象属性のフラグに TOASTCOL_IGNORE を設定し、以降の処理対象から除外する。 2. toast_save_datum を呼び出し、現在の値を外部ストレージへ保存してそのポインタ（Datum）を取得する。 3. 元の値がメモリ確保されていた場合（TOASTCOL_NEEDS_FREE フラグが立っている場合）、pfree を実行してメモリを解放する。 4. 新しい外部化された値に対して TOASTCOL_NEEDS_FREE フラグを付与する。 5. ttc->ttc_flags に TOAST_NEEDS_CHANGE および TOAST_NEEDS_FREE をセットし、タプルが変更されたことと、後続のクリーンアップが必要であることを記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heaptoast.c (複数箇所)
  • **呼び出し先**:
  • toast_save_datum (外部ストレージへの保存)
  • pfree (メモリ解放)

関連データ構造

  • ToastTupleContext: TOAST処理中のタプル状態を管理する構造体。
  • ToastAttrInfo: 各属性のTOAST状態を保持する構造体。

失敗モードと境界条件

  • toast_save_datum が失敗した場合、例外が発生し処理が中断される可能性がある。
  • 属性が既に外部化されている場合や、型定義上外部化不可能な属性に対する挙動は toast_save_datum の実装に依存する。

保守担当者向けメモ

  • 本関数は ToastTupleContext の状態を直接変更するため、呼び出し側は ttc の整合性を維持する必要がある。
  • TOASTCOL_NEEDS_FREE フラグの管理は、メモリリークを防ぐために重要である。
