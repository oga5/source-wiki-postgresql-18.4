compute_tsvector_stats

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_typanalyze.c:140

目的

tsvector 型カラムに対する ANALYZE 処理において、Lossy Counting アルゴリズムを用いて語彙（lexeme）の出現頻度を近似計算し、統計情報を生成する。

シグネチャまたは形状

static void compute_tsvector_stats(VacAttrStats *stats, AnalyzeAttrFetchFunc fetchfunc, int samplerows, double totalrows)

入力と出力

  • **入力**:
  • stats: 統計情報を格納する VacAttrStats 構造体。
  • fetchfunc: サンプル行を取得するためのコールバック関数。
  • samplerows: サンプリングする行数。
  • totalrows: テーブルの総行数。
  • **出力**:
  • stats 構造体の stanullfrac（NULL率）、stawidth（平均幅）、stadistinct（一意性）、および stats_valid フラグを更新する。

前提条件と不変条件

  • stats->attstattarget に基づいてハッシュテーブルのバケット幅が決定される。
  • CurrentMemoryContext 上でハッシュテーブルが構築される。
  • 処理対象の tsvector は detoast された後に処理され、必要に応じてメモリ解放が行われる。

アルゴリズムまたは振る舞い

1. **初期化**: num_mcelem（追跡する語彙数）と bucket_width を設定し、語彙追跡用のハッシュテーブルを作成する。 2. **サンプリング**: fetchfunc を通じて各行の tsvector を取得する。 3. **Lossy Counting**:

  • 各語彙をハッシュテーブルに挿入または更新する。
  • 新規語彙の場合は頻度を 1、誤差（delta）を現在のバケット番号から 1 引いた値で初期化する。
  • bucket_width ごとに prune_lexemes_hashtable を呼び出し、低頻度エントリを削除してメモリを維持する。

    • 4. **統計生成**: サンプリング終了後、閾値を超えた頻度の語彙を抽出し、pg_statistic 用の形式に変換する準備を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/attribute_stats.c（ANALYZE 処理フロー内）
  • src/backend/tsearch/ts_typanalyze.c（ts_typanalyze 関数による登録）
  • **呼び出し先**:
  • fetchfunc（サンプル行取得）
  • hash_create, hash_search（ハッシュテーブル操作）
  • prune_lexemes_hashtable（頻度に基づく剪定）
  • vacuum_delay_point（VACUUM 処理の遅延制御）

関連データ構造

  • VacAttrStats: 統計情報の格納先。
  • LexemeHashKey: 語彙を識別するキー。
  • TrackItem: ハッシュテーブル内の各語彙の頻度と誤差を保持するエントリ。

失敗モードと境界条件

  • samplerows が 0 の場合や、すべての値が NULL の場合は統計計算をスキップする。
  • メモリ不足時は palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は pg_statistic の MCELEM スロットを埋めるためのロジックを含む。
  • 語彙のソート順は、クエリ実行時のバイナリサーチを効率化するため「辞書順」である必要がある。
  • Lossy Counting のパラメータ（bucket_width）は、精度とメモリ消費のトレードオフを決定する重要な定数である。
