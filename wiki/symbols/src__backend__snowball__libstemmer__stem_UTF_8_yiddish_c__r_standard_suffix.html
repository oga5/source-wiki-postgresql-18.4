r_standard_suffix

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/snowball/libstemmer/stem_UTF_8_yiddish.c:744

目的

イディッシュ語のステミング処理において、単語末尾の接尾辞を照合し、条件に応じて削除または置換を行うことで語幹を抽出します。

シグネチャまたは形状

static int r_standard_suffix(struct SN_env * z);

入力と出力

  • **入力**: SN_env * z（ステミング環境構造体。処理対象の文字列や領域情報を含む）
  • **出力**: int（成功時は 1 または 0、エラー発生時は負の値を返す）

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • 内部で r_R1 関数を呼び出し、接尾辞の除去対象が単語内の特定の領域（R1）内にあることを検証します。

アルゴリズムまたは振る舞い

1. find_among_b を使用し、単語末尾から定義済みの接尾辞リスト（a_4）を逆方向に検索します。 2. 検索結果（among_var）に基づき、以下の処理を分岐します：

  • **ケース 1, 3**: r_R1 で領域を確認後、slice_del で接尾辞を削除します。
  • **ケース 2**: r_R1 で領域を確認後、slice_from_s で別の文字列に置換します。
  • **ケース 3（再帰的）**: 削除後にさらに別の接尾辞リスト（a_3）を検索し、一致した場合は置換を行います。
  • **ケース 4**: r_R1 で領域を確認後、slice_del で削除します。

    • 3. 処理中にエラーが発生した場合は、即座に負の値を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内には明示的な呼び出し元は記載されていません。
  • **呼び出し先**: find_among_b, r_R1, slice_del, slice_from_s（いずれも Snowball ランタイム関数）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • among: 接尾辞検索のためのトライ木構造（a_3, a_4）。

失敗モードと境界条件

  • find_among_b が一致する接尾辞を見つけられない場合、lab0 ラベルへジャンプし、何も変更せずに終了します。
  • r_R1 の条件を満たさない場合、接尾辞の削除・置換は行われません。
  • ランタイム関数（slice_del 等）が負の値を返した場合、そのエラーコードをそのまま呼び出し元へ伝播します。
