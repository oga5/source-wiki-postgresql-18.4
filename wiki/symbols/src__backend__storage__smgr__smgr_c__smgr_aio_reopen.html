smgr_aio_reopen

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/smgr.c:1063

目的

非同期I/O操作（AIO）の実行直前に、対象リレーションのファイル記述子（FD）を再取得し、I/O操作データ構造を更新する。

シグネチャまたは形状

static void smgr_aio_reopen(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh（再オープン対象の非同期I/Oハンドル）
  • **出力**: なし（副作用として ioh 内部の PgAioOpData が更新される）

前提条件と不変条件

  • 呼び出し元で HOLD_INTERRUPTS() が実行されており、割り込みが禁止されていること（Assert(!INTERRUPTS_CAN_BE_PROCESSED()) により検証）。

アルゴリズムまたは振る舞い

1. PgAioHandle からターゲットデータ（PgAioTargetData）と操作データ（PgAioOpData）を取得する。 2. 一時リレーションの場合は所有プロセス番号を特定し、それ以外は INVALID_PROC_NUMBER を使用する。 3. smgropen() を呼び出し、リレーションの SMgrRelation を取得する。 4. 操作の種類（PGAIO_OP_READV または PGAIO_OP_WRITEV）に応じて smgrfd() を呼び出し、ファイル記述子を取得して od->read.fd または od->write.fd に格納する。 5. 取得したオフセットが期待値と一致することを Assert で確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/smgr/smgr.c 内の非同期I/O関連のディスパッチテーブル（.reopen = smgr_aio_reopen）から参照されている。
  • **呼び出し先**: pgaio_io_get_target_data, pgaio_io_get_op_data, pgaio_io_get_owner, smgropen, pgaio_io_get_op, smgrfd。

関連データ構造

  • PgAioHandle: 非同期I/Oのハンドル。
  • SMgrRelation: ストレージマネージャのリレーション管理構造体。
  • PgAioOpData: I/O操作の詳細情報（FDやオフセットを含む）。

失敗モードと境界条件

  • PGAIO_OP_INVALID が渡された場合、pg_unreachable() により異常終了する。
  • 割り込みが許可されている状態で呼び出された場合、Assert 失敗により停止する。

保守担当者向けメモ

  • 本関数は割り込み禁止状態での実行が必須である。これは、FDが再オープンされた直後にクローズされる競合状態を防ぐためである。
  • smgrfd() を介して取得されるFDは、ストレージマネージャのキャッシュ状態に依存する。
