vacuum_is_permitted_for_relation

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuum.c:734

目的

指定されたリレーションに対して、現在のユーザーが VACUUM または ANALYZE 操作を実行する権限を持っているかを判定し、権限がない場合は警告メッセージを出力する。

シグネチャまたは形状

bool vacuum_is_permitted_for_relation(Oid relid, Form_pg_class reltuple, bits32 options)

入力と出力

  • **入力**:
  • relid: 対象リレーションの OID。
  • reltuple: 対象リレーションの pg_class 行データ。
  • options: 実行オプション（VACOPT_VACUUM および VACOPT_ANALYZE ビット）。
  • **出力**:
  • 権限がある場合は true、ない場合は false を返す。

前提条件と不変条件

  • options に VACOPT_VACUUM または VACOPT_ANALYZE のいずれかが含まれている必要がある（Assert で検証）。

アルゴリズムまたは振る舞い

1. 以下のいずれかの条件を満たす場合、権限ありとみなして true を返す。

  • 現在のロールがデータベースの所有者であり、かつリレーションが共有オブジェクトではない。
  • 現在のロールがリレーションに対する MAINTAIN 権限を持っている。

    • 2. 権限がない場合、options に応じて警告（WARNING）をログ出力する。

  • VACOPT_VACUUM が指定されている場合は vacuum 権限拒否の警告を出す。
  • VACOPT_ANALYZE が指定されている場合は analyze 権限拒否の警告を出す。

    • 3. 警告出力後、false を返す。

関連データ構造

  • Form_pg_class: リレーションのメタデータ（relisshared 等を参照）。
  • Oid: リレーションの識別子。

失敗モードと境界条件

  • 権限不足時はエラーではなく警告（WARNING）を発生させ、処理をスキップさせることで、一括処理中の継続性を維持する。

保守担当者向けメモ

  • VACUUM ANALYZE が同時に指定された場合、VACOPT_VACUUM の権限チェックが優先され、警告メッセージも VACUUM に関するものが優先的に生成される。
