BackendType

定義

  • 種別: enum
  • 定義箇所: src/include/miscadmin.h:337

目的

PostgreSQLのバックエンドプロセスおよび補助プロセスの役割を識別するために使用される列挙型です。システム内の各プロセスがどのような機能的役割（バックエンド、チェックポインター、WALライター等）を担っているかを特定します。

シグネチャまたは形状

typedef enum BackendType {
    B_INVALID = 0,
    B_BACKEND,
    B_DEAD_END_BACKEND,
    B_AUTOVAC_LAUNCHER,
    B_AUTOVAC_WORKER,
    B_BG_WORKER,
    B_WAL_SENDER,
    B_SLOTSYNC_WORKER,
    B_STANDALONE_BACKEND,
    B_ARCHIVER,
    B_BG_WRITER,
    B_CHECKPOINTER,
    B_IO_WORKER,
    B_STARTUP,
    B_WAL_RECEIVER,
    B_WAL_SUMMARIZER,
    B_WAL_WRITER,
    B_LOGGER
} BackendType;

入力と出力

  • **入力**: プロセス初期化時や管理ルーチンにおいて、この列挙値が識別子として渡されます。
  • **出力**: プロセスの役割に応じた条件分岐や、ログ出力、ビットマスク生成（btmask等）の判定基準として利用されます。

前提条件と不変条件

  • B_LOGGER は共有メモリに接続しておらず、PGPROC エントリを持ちません。
  • 補助プロセス（B_ARCHIVER から B_WAL_WRITER まで）は、特定のデータベースに紐付かず、トランザクションの実行や重量級ロックの取得ができません。
  • 補助プロセスは、IOワーカーを除き、同時に各1つのみ存在可能です。
  • 列挙型定義を変更する場合は、NUM_AUXILIARY_PROCS および関連するドキュメントの用語集を更新する必要があります。

アルゴリズムまたは振る舞い

プロセス管理の識別子として機能します。Postmasterはこれを用いて、起動するプロセスの種類を決定したり、特定のプロセスタイプに対する操作（スロット割り当てやビットマスクによるフィルタリング）を実行します。

呼び出し元と呼び出し先

  • **呼び出し元（推測される参照先）**:
  • src/backend/postmaster/launch_backend.c: PostmasterChildName, postmaster_child_launch
  • src/backend/postmaster/pmchild.c: AssignPostmasterChildSlot
  • src/backend/postmaster/postmaster.c: btmask, btmask_add_n, btmask_del, btmask_all_except_n
  • **呼び出し先**:
  • 本シンボルは列挙型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • PGPROC: バックエンドプロセスが共有メモリ上で保持する構造体（B_LOGGER を除く）。
  • BackendTypeMask: postmaster.c で定義される、BackendType をビットマスクとして扱うための型。

失敗モードと境界条件

  • B_INVALID は無効な状態を示します。
  • 補助プロセスの定義を変更した際に NUM_AUXILIARY_PROCS の更新を怠ると、システムの一貫性が損なわれる可能性があります。

保守担当者向けメモ

  • 本列挙型はプロセス管理の根幹に関わるため、新しいプロセスタイプを追加する際は、既存の補助プロセスとの排他制御や共有メモリへの接続要件を十分に検討してください。
  • postmaster.c 内のビットマスク操作関数群と密接に関連しているため、変更時はこれらの関数への影響を確認してください。
