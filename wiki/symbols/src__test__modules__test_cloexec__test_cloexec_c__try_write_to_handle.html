try_write_to_handle

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_cloexec/test_cloexec.c:219

目的

Windows環境において、指定されたOSハンドルに対してテスト用の文字列を書き込み、書き込みの成否を判定する。

シグネチャまたは形状

static bool try_write_to_handle(HANDLE h, const char *label);

入力と出力

  • **入力**:
  • h: 書き込み対象のWindowsネイティブハンドル (HANDLE)。
  • label: ログ出力に使用する識別ラベル (const char *)。
  • **出力**:
  • 書き込みが成功した場合は true、失敗した場合は false を返す。

前提条件と不変条件

  • 本関数はWindows環境でのみ動作する。
  • 引数 h は有効なWindowsハンドルである必要がある。

アルゴリズムまたは振る舞い

1. 固定のテスト文字列 "test\n" を用意する。 2. Windows API WriteFile を呼び出し、ハンドル h への書き込みを試行する。 3. WriteFile が成功し、かつ書き込まれたバイト数がテスト文字列の長さと一致する場合、成功メッセージを出力して true を返す。 4. 上記以外の場合、GetLastError() を取得し、失敗メッセージとエラーコードを出力して false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: run_child_tests (同ファイル内)
  • **呼び出し先**: WriteFile (Windows API), printf, GetLastError (Windows API)

関連データ構造

  • HANDLE: Windows APIで使用されるハンドル型。

失敗モードと境界条件

  • ハンドルが継承禁止（O_CLOEXEC 等による）の状態にある場合、WriteFile は失敗し、本関数は false を返す。
  • 書き込み権限がないハンドルや無効なハンドルが渡された場合、WriteFile は失敗する。

保守担当者向けメモ

  • 本関数は test_cloexec モジュールの検証ロジックの一部であり、子プロセスが親から継承したハンドルを正しく扱えているかを確認するために使用される。
  • ログ出力は標準出力に行われるため、テスト実行環境での確認が必要である。
