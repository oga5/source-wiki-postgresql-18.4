FuncInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.h:239

目的

データベース内の関数（Function）オブジェクトの情報を保持し、pg_dump によるダンプ処理において、その定義や属性を管理するための構造体です。

シグネチャまたは形状

typedef struct _funcInfo
{
    DumpableObject dobj;
    DumpableAcl dacl;
    const char *rolname;
    Oid         lang;
    int         nargs;
    Oid        *argtypes;
    Oid         prorettype;
    bool        postponed_def;
} FuncInfo;

入力と出力

  • **入力**: システムカタログから取得された関数定義情報。
  • **出力**: dumpFunc 等の関数を通じて、ダンプ出力先（Archive）へ書き出される SQL 定義文。

前提条件と不変条件

  • dobj フィールドにより、DumpableObject としての共通属性（ID、依存関係など）を保持します。
  • pg_dump の実行期間中、メモリ上に生存します。

アルゴリズムまたは振る舞い

  • pg_dump.c 内の getSchemaData 等でカタログから読み取られ、インスタンス化されます。
  • postponed_def が true の場合、関数の定義はダンプの後半（post-data）フェーズまで延期されます。
  • format_function_arguments 関数により、引数の型情報（argtypes）から SQL 形式の引数リスト文字列が生成されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の関数群（dumpFunc, format_function_arguments など）。
  • **呼び出し先**: pg_malloc0（メモリ確保）、dumpFunc（ダンプ実行）。

関連データ構造

  • DumpableObject: 基底構造体。
  • DumpableAcl: 関数のアクセス権限情報。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • argtypes が NULL または nargs と不整合な場合、引数生成時にエラーが発生する可能性があります。
  • メモリ不足時には pg_malloc0 によりプロセスが終了する可能性があります。

保守担当者向けメモ

  • 新しい関数属性を追加する場合は、pg_dump.c 内の読み込みロジックと dumpFunc の出力ロジックの両方を更新してください。
  • postponed_def のフラグは、依存関係の解決順序に影響を与えるため、慎重に扱う必要があります。
