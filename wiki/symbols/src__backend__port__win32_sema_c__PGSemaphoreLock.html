PGSemaphoreLock

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_sema.c:131

目的

Windows環境において、指定されたセマフォの取得（ロック）を試みます。待機中にPostgreSQLのシグナルイベントを監視し、割り込みを適切に処理することでデッドロックや応答不能を防ぎます。

シグネチャまたは形状

void PGSemaphoreLock(PGSemaphore sema)

入力と出力

  • **入力**: sema (PGSemaphore型) - 取得対象のセマフォハンドル。
  • **出力**: なし（void型）。成功した場合はセマフォを取得して復帰します。

前提条件と不変条件

  • pgwin32_signal_event が初期化されている必要があります。
  • 呼び出し元は、セマフォが利用可能になるまでこの関数内でブロックされることを許容する必要があります。

アルゴリズムまたは振る舞い

1. pgwin32_signal_event と sema を監視対象として WaitForMultipleObjectsEx を呼び出します。 2. ループ内で CHECK_FOR_INTERRUPTS() を実行し、割り込みを確認します。 3. WaitForMultipleObjectsEx の戻り値に応じて以下の処理を行います：

  • WAIT_OBJECT_0: シグナルイベントが発生。pgwin32_dispatch_queued_signals() を呼び出してシグナルを処理し、待機を継続します。
  • WAIT_OBJECT_0 + 1: セマフォを取得成功。ループを終了します。
  • WAIT_IO_COMPLETION: システムによるI/O完了ルーチン等の割り込み。待機を再開します。
  • WAIT_FAILED: GetLastError() を用いてエラーを報告し、FATAL で終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/clog.c
  • src/backend/storage/ipc/procarray.c
  • src/backend/storage/lmgr/lwlock.c
  • **呼び出し先**:
  • CHECK_FOR_INTERRUPTS()
  • WaitForMultipleObjectsEx (Win32 API)
  • pgwin32_dispatch_queued_signals()
  • ereport() / elog()

関連データ構造

  • PGSemaphore (セマフォハンドル)
  • pgwin32_signal_event (シグナル監視用イベント)

失敗モードと境界条件

  • WaitForMultipleObjectsEx が WAIT_FAILED を返した場合、GetLastError() を出力して FATAL エラーとなります。
  • 予期しない戻り値が返された場合、elog(FATAL) によりプロセスが終了します。

保守担当者向けメモ

  • 本関数は pgwin32_signal_event を優先的に監視する設計です。これは、セマフォ待機中であってもPostgreSQLのシグナル処理を確実に行うための重要な不変条件です。
  • WAIT_IO_COMPLETION は、外部DLL等による非同期プロシージャ呼び出しが発生した場合に備えて無視（待機継続）する設計となっています。
