pg_read_file_all_missing

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:332

目的

サーバー上の指定されたファイル全体をテキスト形式で読み込みます。ファイルが存在しない場合にエラーを発生させるか、NULLを返すかを制御するオプションを提供します。

シグネチャまたは形状

Datum pg_read_file_all_missing(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • filename_t (text): 読み込む対象のファイルパス。
  • missing_ok (bool): true の場合、ファイルが存在しなくてもエラーにせず NULL を返します。
  • **出力**:
  • text: ファイルの内容。ファイルが存在しないかつ missing_ok が true の場合は NULL。

前提条件と不変条件

  • 呼び出し元は PostgreSQL の権限管理システム（pg_read_server_files ロール等）による検証を通過している必要があります。
  • 指定されたパスは convert_and_check_filename により、データディレクトリやログディレクトリの範囲内であるか検証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP および PG_GETARG_BOOL を使用して引数を取得します。 2. pg_read_file_common を呼び出し、以下の引数を渡します：

  • オフセット: 0
  • 長さ: -1（ファイル全体を読み込むことを示す）
  • バイナリフラグ: true（テキストとして読み込むための設定）
  • missing_ok フラグ: 引数から受け取った値をそのまま渡します。

    • 3. pg_read_file_common の戻り値が NULL の場合は PG_RETURN_NULL() を返します。 4. それ以外の場合は、読み込んだデータを PG_RETURN_TEXT_P を通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（詳細は提供されていません）。
  • **呼び出し先**: pg_read_file_common（同ファイル内で定義）。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • **ファイル不在**: missing_ok が false の場合、ファイルが存在しないとエラーが発生します。true の場合は NULL が返されます。
  • **権限エラー**: convert_and_check_filename による検証で、許可されていないパスや権限不足が検出された場合はエラーとなります。

保守担当者向けメモ

  • 本関数は pg_read_file_all のバリエーションであり、missing_ok 引数を動的に処理するラッパーです。
  • 修正を行う際は、pg_read_file_common のインターフェース変更が本関数に与える影響を確認してください。
