tarError

定義

  • 種別: enum
  • 定義箇所: src/include/pgtar.h:19

目的

tar アーカイブ作成処理におけるエラー状態を表現するための列挙型です。

シグネチャまたは形状

enum tarError
{
    TAR_OK = 0,
    TAR_NAME_TOO_LONG,
    TAR_SYMLINK_TOO_LONG,
};

入力と出力

  • 出力: tarCreateHeader() 関数の戻り値として使用され、操作の成否または特定のエラー種別を返します。

前提条件と不変条件

  • TAR_OK は 0 と定義されており、正常終了を示します。
  • その他の値は、tar ヘッダーのフィールド長制限を超過した際のエラーを示します。

アルゴリズムまたは振る舞い

  • tarCreateHeader() がヘッダー構築時にファイル名やシンボリックリンクのパス長を検証し、制限を超えている場合に該当するエラー値を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/backup/basebackup.c において、バックアップ処理の戻り値として参照されています。
  • 呼び出し先: src/port/tar.c 内の tarCreateHeader() 実装において、エラー状態の返却値として使用されています。

関連データ構造

  • tarCreateHeader(): 本列挙型を戻り値として利用する主要関数。

失敗モードと境界条件

  • TAR_NAME_TOO_LONG: 指定されたファイル名が tar ヘッダーの格納可能サイズを超過した場合。
  • TAR_SYMLINK_TOO_LONG: 指定されたシンボリックリンクのパスが tar ヘッダーの格納可能サイズを超過した場合。

保守担当者向けメモ

  • 新たなエラー条件を追加する場合は、本列挙型を拡張し、tarCreateHeader() の実装側で適切に判定ロジックを更新してください。
