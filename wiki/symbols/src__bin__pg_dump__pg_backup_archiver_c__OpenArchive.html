OpenArchive

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:245

目的

指定されたファイルパスとアーカイブ形式に基づき、読み取り専用モードでアーカイブを初期化し、操作用のハンドルを取得します。

シグネチャまたは形状

Archive *OpenArchive(const char *FileSpec, const ArchiveFormat fmt);

入力と出力

  • **入力**:
  • FileSpec: アーカイブファイルへのパス（文字列）。
  • fmt: アーカイブの形式（ArchiveFormat 列挙型）。
  • **出力**:
  • 初期化された Archive 構造体へのポインタ。

前提条件と不変条件

  • 内部的に PG_COMPRESSION_NONE が指定され、圧縮なしの状態で初期化されます。
  • アーカイブモードは archModeRead（読み取り）に固定されます。
  • リストアワーカーのセットアップ関数として setupRestoreWorker が指定されます。

アルゴリズムまたは振る舞い

1. 圧縮仕様を PG_COMPRESSION_NONE で初期化します。 2. 内部関数 _allocAH を呼び出し、ArchiveHandle を構築します。この際、読み取りモードおよび DATA_DIR_SYNC_METHOD_FSYNC 同期メソッドが適用されます。 3. 構築された ArchiveHandle を Archive 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_restore.c:488
  • **呼び出し先**:
  • _allocAH（内部関数）

関連データ構造

  • Archive: アーカイブの抽象インターフェース。
  • ArchiveHandle: アーカイブの状態を保持するメイン構造体。
  • ArchiveFormat: アーカイブ形式を定義する列挙型。

失敗モードと境界条件

  • _allocAH 内部でのメモリ割り当て失敗や、指定された FileSpec が開けない場合、_allocAH の実装に従いエラーが発生します。

保守担当者向けメモ

  • 本関数は読み取り専用の初期化に特化しています。書き込み用アーカイブの作成には CreateArchive を使用してください。
  • _allocAH に渡される引数は固定されているため、圧縮設定や同期メソッドを変更する場合は本関数のシグネチャまたは実装の修正が必要です。
