PrepareTempTablespaces

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tablespace.c:1330

目的

GUC 変数 temp_tablespaces を解析し、現在のトランザクションで使用する一時テーブルスペースの OID リストを構築して fd.c に設定する。

シグネチャまたは形状

void PrepareTempTablespaces(void)

入力と出力

  • **入力**: グローバル変数 temp_tablespaces（文字列）、MyDatabaseTableSpace（現在のデータベースのデフォルトテーブルスペース OID）。
  • **出力**: なし（副作用として SetTempTablespaces を介して内部状態を更新）。

前提条件と不変条件

  • トランザクション状態であること（IsTransactionState() が真であること）。
  • 現在のトランザクションで既に一時テーブルスペースが設定されている場合、処理は即座に終了する。

アルゴリズムまたは振る舞い

1. TempTablespacesAreSet() を確認し、設定済みであれば終了する。 2. トランザクション外であれば安全のため終了する。 3. temp_tablespaces 文字列をカンマ区切りで解析する。 4. 各識別子について以下を検証する：

  • 空文字列または MyDatabaseTableSpace と一致する場合は InvalidOid（デフォルト）として扱う。
  • get_tablespace_oid で有効な OID を取得し、存在しない場合はスキップする。
  • object_aclcheck で ACL_CREATE 権限を確認し、権限がない場合はスキップする。

    • 5. 抽出した OID 配列を TopTransactionContext に割り当て、SetTempTablespaces を呼び出して登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablespace.c
  • src/backend/executor/nodeHash.c
  • src/backend/storage/file/buffile.c
  • src/backend/storage/file/fileset.c
  • src/backend/utils/sort/tuplesort.c
  • src/backend/utils/sort/tuplestore.c
  • **呼び出し先**:
  • TempTablespacesAreSet, IsTransactionState, pstrdup, SplitIdentifierString, SetTempTablespaces, pfree, list_free, MemoryContextAlloc, get_tablespace_oid, object_aclcheck, GetUserId

関連データ構造

  • Oid: テーブルスペース識別子。
  • TopTransactionContext: OID 配列のメモリ割り当て先。

失敗モードと境界条件

  • **構文エラー**: SplitIdentifierString が失敗した場合、SetTempTablespaces(NULL, 0) を呼び出し、解析を中断する。
  • **権限不足**: 指定されたテーブルスペースに対する ACL_CREATE 権限がない場合、その要素は無視される。
  • **存在しないテーブルスペース**: get_tablespace_oid が InvalidOid を返した場合、その要素は無視される。

保守担当者向けメモ

  • 本関数は fd.c への設定伝播を担うため、一時ファイル生成を伴う実行エンジン（ソート、ハッシュ結合等）の直前で呼び出される必要がある。
  • TopTransactionContext を使用しているため、トランザクション終了時にメモリは自動的に解放される。
