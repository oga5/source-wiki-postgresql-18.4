add_base_rels_to_query

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/initsplan.c:169

目的

クエリのジョインツリーを再帰的に走査し、ツリー内に含まれるすべてのベースリレーション（RangeTblRef）に対して build_simple_rel を呼び出し、プランナが使用する RelOptInfo 構造体を初期化します。

シグネチャまたは形状

void add_base_rels_to_query(PlannerInfo *root, Node *jtnode)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報を保持する PlannerInfo 構造体。
  • jtnode: 解析対象のジョインツリーノード（RangeTblRef, FromExpr, または JoinExpr）。
  • **出力**: なし（副作用として root 内のベースリレーション情報が構築されます）。

前提条件と不変条件

  • jtnode が NULL の場合は何も行われません。
  • jtnode は RangeTblRef、FromExpr、JoinExpr のいずれかである必要があります。それ以外のノード型が渡された場合はエラーとなります。

アルゴリズムまたは振る舞い

1. **RangeTblRef**: ノードがリレーションを参照している場合、そのインデックス（rtindex）を使用して build_simple_rel を呼び出し、リレーションの初期化を行います。 2. **FromExpr**: fromlist 内の各要素に対して再帰的に本関数を呼び出します。 3. **JoinExpr**: 左辺（larg）および右辺（rarg）のノードに対して再帰的に本関数を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planmain.c:170 (メインエントリポイントからの呼び出し)
  • src/backend/optimizer/plan/initsplan.c 内の再帰呼び出し
  • **呼び出し先**:
  • build_simple_rel
  • elog (エラー発生時)

関連データ構造

  • PlannerInfo: プランナの全体状態を管理。
  • RangeTblRef, FromExpr, JoinExpr: ジョインツリーを構成するノード型。
  • RelOptInfo: build_simple_rel によって生成・管理されるリレーション情報。

失敗モードと境界条件

  • 未知のノード型が jtnode に渡された場合、elog(ERROR, ...) により処理が中断されます。

保守担当者向けメモ

  • 本関数はジョインツリーを深さ優先で走査します。LATERAL 参照や複雑な結合条件の解析に先立ち、まずはすべてのベースリレーションを PlannerInfo に登録するために使用されます。
  • build_simple_rel は RelOptInfo を作成する役割を担っており、本関数はそのための走査ロジックに特化しています。
