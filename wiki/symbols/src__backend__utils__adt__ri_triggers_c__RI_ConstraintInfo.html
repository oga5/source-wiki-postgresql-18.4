RI_ConstraintInfo

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/ri_triggers.c:106

目的

外部キー制約のメタデータを保持し、データベースの参照整合性（RI）チェックを効率化するためのキャッシュ構造体です。

シグネチャまたは形状

typedef struct RI_ConstraintInfo {
    Oid         constraint_id;
    bool        valid;
    Oid         constraint_root_id;
    uint32      oidHashValue;
    uint32      rootHashValue;
    NameData    conname;
    Oid         pk_relid;
    Oid         fk_relid;
    char        confupdtype;
    char        confdeltype;
    int         ndelsetcols;
    int16       confdelsetcols[RI_MAX_NUMKEYS];
    char        confmatchtype;
    bool        hasperiod;
    int         nkeys;
    int16       pk_attnums[RI_MAX_NUMKEYS];
    int16       fk_attnums[RI_MAX_NUMKEYS];
    Oid         pf_eq_oprs[RI_MAX_NUMKEYS];
    Oid         pp_eq_oprs[RI_MAX_NUMKEYS];
    Oid         ff_eq_oprs[RI_MAX_NUMKEYS];
    Oid         period_contained_by_oper;
    Oid         agged_period_contained_by_oper;
    Oid         period_intersect_oper;
    dlist_node  valid_link;
} RI_ConstraintInfo;

入力と出力

本構造体は ri_LoadConstraintInfo 等の関数によって pg_constraint システムカタログから読み込まれ、メモリ上に保持されます。関数への入力として渡される際は、主に制約検証ロジックの参照用として使用されます。

前提条件と不変条件

  • constraint_id は pg_constraint の OID を指します。
  • valid フラグは、初期化が正常に完了したことを示します。
  • pk_attnums および fk_attnums 等の配列は RI_MAX_NUMKEYS を上限とします。

アルゴリズムまたは振る舞い

  • 外部キー制約の定義（ON UPDATE/DELETE アクション、マッチングタイプ、列属性）を保持します。
  • 期間（PERIOD）制約に関連する演算子情報を含みます。
  • dlist_node を用いて、有効なキャッシュエントリのリストを構成します。

呼び出し元と呼び出し先

  • **呼び出し元**: ri_FetchConstraintInfo, ri_LoadConstraintInfo, ri_PerformCheck, ri_ReportViolation 等の ri_triggers.c 内の関数群。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • RI_QueryKey: キャッシュされたSPIプランを識別するためのキー。
  • dlist_node: 有効な制約情報のリスト管理に使用。

失敗モードと境界条件

  • RI_MAX_NUMKEYS を超える列数を持つ制約には対応していません。
  • valid が false の場合、制約情報の初期化に失敗していることを示します。

保守担当者向けメモ

  • 本構造体はバックエンドの生存期間中保持されるキャッシュの一部です。
  • 構造体定義の変更は、SPIプランの生成ロジックやキャッシュの整合性に影響を与えるため注意してください。
