getcvec

定義

  • 種別: static function
  • 定義箇所: src/backend/regex/regc_cvec.c:114

目的

正規表現コンパイル中に一時的な文字セット（cvec）を取得または再確保します。コンパイルコンテキスト内で cvec を再利用することで、メモリ割り当てのオーバーヘッドを削減します。

シグネチャまたは形状

static struct cvec *getcvec(struct vars *v, int nchrs, int nranges);

入力と出力

  • **入力**:
  • v: コンパイルコンテキストを保持する vars 構造体へのポインタ。
  • nchrs: 必要な文字数分の容量。
  • nranges: 必要な範囲数分の容量。
  • **出力**:
  • 要求された容量を満たす cvec 構造体へのポインタ。メモリ確保に失敗した場合は NULL を返し、内部で ERR(REG_ESPACE) を呼び出します。

前提条件と不変条件

  • v->cv は、getcvec によって管理される一時的な cvec を保持する場所として使用されます。
  • 返された cvec は、次に getcvec が呼び出されるまでの間のみ有効です。
  • 呼び出し元は、返された cvec に対して直接 free を実行してはなりません（vars 構造体のライフサイクルに依存するため）。

アルゴリズムまたは振る舞い

1. **再利用判定**: v->cv が存在し、かつ現在の容量（chrspace, rangespace）が要求された nchrs および nranges 以上である場合、clearcvec を呼び出してカウンタをリセットし、既存の cvec を返します。 2. **再確保**: 条件を満たさない場合、既存の v->cv があれば freecvec で解放します。 3. **新規作成**: newcvec を呼び出して新しい cvec を確保し、v->cv に格納します。 4. **エラー処理**: newcvec が NULL を返した場合は、ERR(REG_ESPACE) を呼び出してエラー状態を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_locale.c 内の各関数（例: 430行目、450行目など）。
  • **呼び出し先**:
  • clearcvec: 既存の cvec を再利用する際に呼び出されます。
  • freecvec: 既存の cvec が不足している場合に解放するために呼び出されます。
  • newcvec: 新しい cvec を生成するために呼び出されます。
  • ERR: メモリ確保失敗時に呼び出されます。

関連データ構造

  • struct vars: コンパイルコンテキストを保持し、cv フィールドを通じて cvec のライフサイクルを管理します。
  • struct cvec: 文字と範囲を保持するデータ構造。regguts.h で定義されています。

失敗モードと境界条件

  • **メモリ不足**: newcvec が NULL を返した場合、ERR(REG_ESPACE) が呼び出されます。
  • **再利用の不適合**: 要求されたサイズが既存の cvec の容量を超えている場合、必ず再確保が行われます。

保守担当者向けメモ

  • 本関数は regcomp.c にインクルードされる regc_cvec.c 内で static として定義されており、コンパイル単位内での最適化対象です。
  • cvec の内部配列を直接操作できるのは、このモジュール内の関数のみであるという設計上の制約があります。
