TailMatchesArray

定義

  • 種別: function
  • 定義箇所: src/bin/psql/tab-complete.in.c:1625

目的

ユーザーが入力した直前の単語列が、指定されたパターン配列と末尾から順に一致するかを判定し、タブ補完の文脈を特定する。

シグネチャまたは形状

static bool TailMatchesArray(bool case_sensitive,
                             int previous_words_count, char **previous_words,
                             int narg, const char *const *args)

入力と出力

  • **入力**:
  • case_sensitive: 大文字小文字を区別するかどうかのフラグ。
  • previous_words_count: 入力された単語の総数。
  • previous_words: 入力された単語の配列。
  • narg: 比較対象となるパターン（args）の数。
  • args: 比較対象となるパターン文字列の配列。
  • **出力**:
  • bool: パターンがすべて一致した場合は true、それ以外は false。

前提条件と不変条件

  • previous_words_count が narg 未満の場合、比較を行わず直ちに false を返す。
  • args の各要素は、previous_words の末尾から逆順（narg - argno - 1）に比較される。

アルゴリズムまたは振る舞い

1. 入力単語数 previous_words_count がパターン数 narg より少ない場合、不一致として false を返す。 2. 0 から narg - 1 までループし、各 argno に対して以下の処理を行う。

  • args[argno] と、previous_words の末尾から argno 番目の単語を word_matches 関数で比較する。
  • word_matches は case_sensitive フラグに従って比較を行う。
  • 一致しない単語が一つでもあれば false を返す。

    • 3. すべてのパターンが一致した場合、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/tab-complete.in.c 内の複数の箇所で、補完ルールのマッチング判定に使用されている。
  • **呼び出し先**:
  • word_matches（比較ロジックの実行）

関連データ構造

  • previous_words: 入力行を単語単位に分解した文字列配列。

失敗モードと境界条件

  • previous_words_count < narg: 比較対象の単語数がパターン数に満たない場合、常に false を返す。

保守担当者向けメモ

  • 本関数は gen_tabcomplete.pl によって生成される補完ロジックの基盤となる。
  • word_matches の挙動に依存しているため、比較ルールを変更する場合は word_matches の実装を確認すること。
