AppendJumble32

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:347

目的

クエリのフィンガープリント生成プロセスにおいて、32ビットのデータをハッシュ計算用のバッファに追加します。

シグネチャまたは形状

static pg_noinline void AppendJumble32(JumbleState *jstate, const unsigned char *value)

入力と出力

  • jstate: ハッシュ計算の状態とバッファを保持する JumbleState 構造体へのポインタ。
  • value: 追加する32ビットデータへのポインタ。
  • 出力: なし（jstate 内のバッファが更新されます）。

前提条件と不変条件

  • jstate は有効な JumbleState インスタンスを指している必要があります。
  • value は少なくとも4バイトのデータ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. jstate->pending_nulls が 0 より大きい場合、FlushPendingNulls(jstate) を呼び出して保留中の NULL 情報をバッファに書き出します。 2. AppendJumbleInternal(jstate, value, 4) を呼び出し、指定された4バイトのデータをハッシュバッファに追加します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/nodes/queryjumblefuncs.c 内の各箇所（expr->item、lfirst_int、lfirst_oid、lfirst_xid 等の値を渡す処理）。
  • 呼び出し先:
  • FlushPendingNulls
  • AppendJumbleInternal

関連データ構造

  • JumbleState: クエリのハッシュ計算状態を管理する構造体。

失敗モードと境界条件

  • jstate が NULL の場合、未定義の動作を引き起こします。
  • value が指すメモリ領域が4バイト未満の場合、メモリ読み取り違反が発生する可能性があります。
  • AppendJumbleInternal 内部でバッファ溢れが発生した場合、自動的にハッシュ化とバッファの再利用が行われます。

保守担当者向けメモ

  • pg_noinline が指定されており、インライン展開を抑制することでスタック使用量を制御しています。
  • 新しいノード型を追加する際、32ビットのフィールドをハッシュに含める必要がある場合は本関数を利用してください。
