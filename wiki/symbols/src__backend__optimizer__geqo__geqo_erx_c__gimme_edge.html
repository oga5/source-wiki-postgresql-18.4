gimme_edge

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_erx.c:153

目的

遺伝的クエリ最適化（GEQO）におけるエッジ組み換え交叉（ERX）のため、2つの都市（遺伝子）間の接続関係をエッジテーブルに登録または更新する。

シグネチャまたは形状

static int gimme_edge(PlannerInfo *root, Gene gene1, Gene gene2, Edge *edge_table)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体（使用されず）。
  • gene1, gene2: 接続する2つの都市を表す Gene 型。
  • edge_table: 都市間の接続関係を保持する Edge 構造体の配列。
  • **出力**:
  • int: 新規にエッジを追加した場合は 1、既存のエッジを共有エッジとしてマークした場合は 0 を返す。

前提条件と不変条件

  • edge_table は、各都市が持つ接続先リストと、その総数（total_edges）および未使用数（unused_edges）を正しく保持している必要がある。
  • city1 をインデックスとして edge_table にアクセスするため、gene1 は有効な都市範囲内であること。

アルゴリズムまたは振る舞い

1. city1 の既存エッジリストを走査し、city2 への接続が既に存在するか確認する。 2. 既に存在する場合（共有エッジ）:

  • そのエッジの値を負数（0 - city2）に書き換え、0 を返して終了する。

    • 3. 存在しない場合:

  • city1 のエッジリストの末尾に city2 を追加する。
  • city1 の total_edges および unused_edges をインクリメントする。
  • 1 を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_erx.c 内の関数（gimme_edge_table 等）から、親ツアーの各都市ペアに対して呼び出される。
  • **呼び出し先**: 本関数は直接他の関数を呼び出さない。

関連データ構造

  • Edge: 都市間の接続関係を保持する構造体。edge_list（接続先リスト）、total_edges（総数）、unused_edges（未使用数）を含む。
  • Gene: 都市を識別する型。

失敗モードと境界条件

  • エッジリストの容量制限については本関数内ではチェックしていない（呼び出し側で十分な領域が確保されている前提）。
  • 共有エッジの判定は abs() を用いて絶対値で行われるため、既に負数化されているエッジも正しく city2 と比較される。

保守担当者向けメモ

  • 本関数は単方向の接続を処理する。双方向の接続を構築するには、呼び出し側で gimme_edge(..., city1, city2, ...) と gimme_edge(..., city2, city1, ...) の両方を呼び出す必要がある。
  • 共有エッジを負数で表現する仕様は、後続の gimme_gene における優先選択ロジックと密接に関連している。
