_variable

定義

  • 種別: struct
  • 定義箇所: src/bin/psql/variables.h:62

目的

psqlにおける変数管理のためのデータ構造。個々の変数の名前、値、およびそれに関連するフック関数を保持し、連結リスト形式で変数の集合を構成する。

シグネチャまたは形状

struct _variable
{
    char    *name;
    char    *value;
    VariableSubstituteHook substitute_hook;
    VariableAssignHook assign_hook;
    struct _variable *next;
};

入力と出力

  • 入力: name（変数名）、value（値）、substitute_hook、assign_hook（各フック関数）を保持する。
  • 出力: 連結リストの次要素へのポインタ next を介して、変数のリスト構造を形成する。

前提条件と不変条件

  • next ポインタにより、変数のリストを構成する。
  • src/bin/psql/variables.c の記述によれば、「変数空間（VariableSpace）」は、この構造体の未使用のインスタンスを先頭として表現される。

アルゴリズムまたは振る舞い

  • 変数の名前と値を保持し、値の代入や置換時にフック関数を介した処理を可能にする。
  • 連結リスト構造により、変数空間内での変数の検索、追加、削除をサポートする。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/tab-complete.in.c および src/bin/psql/variables.c 内の各関数で、この構造体へのポインタが参照されている。
  • 呼び出し先: 具体的な関数呼び出しの詳細は提供されていない。

関連データ構造

  • VariableSpace: typedef struct _variable *VariableSpace; として定義されており、変数リストのハンドルとして機能する。

失敗モードと境界条件

  • メモリ割り当ての失敗や、リストの終端（next が NULL）における操作の境界条件については、提供されたソース excerpt には記載がない。
