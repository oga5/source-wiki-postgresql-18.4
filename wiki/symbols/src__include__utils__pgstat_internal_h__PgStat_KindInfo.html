PgStat_KindInfo

定義

  • 種別: struct
  • 定義箇所: src/include/utils/pgstat_internal.h:202

目的

PostgreSQLの累積統計システムにおいて、統計種別（テーブル、関数、IO等）ごとのメタデータと振る舞いを定義する構造体です。統計の初期化、共有メモリレイアウト、シリアライズ、および更新のフラッシュ処理を制御します。

シグネチャまたは形状

typedef struct PgStat_KindInfo {
    bool        fixed_amount:1;
    bool        accessed_across_databases:1;
    bool        write_to_file:1;
    uint32      shared_size;
    uint32      snapshot_ctl_off;
    uint32      shared_ctl_off;
    uint32      shared_data_off;
    uint32      shared_data_len;
    uint32      pending_size;
    void        (*init_backend_cb) (void);
    bool        (*flush_pending_cb) (PgStat_EntryRef *sr, bool nowait);
    void        (*delete_pending_cb) (PgStat_EntryRef *sr);
    void        (*reset_timestamp_cb) (PgStatShared_Common *header, TimestampTz ts);
    void        (*to_serialized_name) (const PgStat_HashKey *key, const PgStatShared_Common *header, NameData *name);
    bool        (*from_serialized_name) (const NameData *name, PgStat_HashKey *key);
    void        (*init_shmem_cb) (void *stats);
    bool        (*flush_static_cb) (bool nowait);
    void        (*reset_all_cb) (TimestampTz ts);
    void        (*snapshot_cb) (void);
    const char *const name;
} PgStat_KindInfo;

入力と出力

  • **入力**: 統計システムが各統計種別のメタデータとして本構造体を参照します。
  • **出力**: 各種コールバック関数を通じて、統計の初期化、共有メモリへのフラッシュ、スナップショット生成、およびシリアライズ処理を実行します。

前提条件と不変条件

  • fixed_amount が真の場合、統計は固定個数として扱われ、shared_ctl_off 等のオフセット値が使用されます。
  • pending_size が 0 の場合、その統計種別は PgStat_EntryRef->pending を持ちません。
  • flush_pending_cb は、pending データを使用する可変個数統計において必須です。

アルゴリズムまたは振る舞い

1. **初期化**: バックエンド起動時に init_backend_cb が呼び出されます。 2. **更新**: バックエンドは PgStat_EntryRef を介して統計を更新し、flush_pending_cb または flush_static_cb を通じて共有メモリへマージします。 3. **管理**: fixed_amount に応じて、共有メモリ上の PgStat_ShmemControl またはハッシュテーブル内のエントリとして管理されます。 4. **永続化**: write_to_file が真の場合、ディスクへの書き出し対象となります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c 内の pgstat_get_kind_info() を経由して、統計システム全体から参照されます。
  • **呼び出し先**: 各統計種別の実装（pgstat_*.c）で定義されたコールバック関数が、本構造体のメンバとして登録されます。

関連データ構造

  • PgStatShared_Common: 統計ヘッダー情報。
  • PgStat_EntryRef: バックエンドローカルの統計参照および保留中データ保持用。
  • PgStat_ShmemControl: 統計システム全体の共有メモリ管理構造体。

失敗モードと境界条件

  • flush_pending_cb や flush_static_cb は、ロック競合が発生した場合に false を返し、フラッシュが完了しなかったことを示す場合があります。
  • pending_size が 0 の統計種別に対して pending 領域へのアクセスを試みることはできません。

保守担当者向けメモ

  • 本構造体は共有メモリレイアウトと密接に関連しているため、メンバの追加やオフセットの変更は統計システムの整合性に直結します。
  • fixed_amount 統計と可変個数統計でコールバックの要件が異なるため、実装時には注意が必要です。
