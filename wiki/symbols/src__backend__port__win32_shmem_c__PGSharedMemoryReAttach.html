PGSharedMemoryReAttach

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_shmem.c:423

目的

子プロセスにおいて、親プロセス（Postmaster）から継承した共有メモリハンドルを使用して、共有メモリセグメントを特定のアドレス空間へ再マッピングする。

シグネチャまたは形状

void PGSharedMemoryReAttach(void)

入力と出力

  • **入力**: なし（グローバル変数 UsedShmemSegID、UsedShmemSegAddr、ShmemProtectiveRegion を使用）。
  • **出力**: なし（成功時は共有メモリがマッピングされ、dsm_set_control_handle が呼び出される）。
  • **副作用**: VirtualFree によるメモリ予約の解放、MapViewOfFileEx による共有メモリのマッピング。

前提条件と不変条件

  • ShmemProtectiveRegion および UsedShmemSegAddr が NULL でないこと。
  • IsUnderPostmaster が真であること。
  • 呼び出し前に、親プロセスによって指定されたアドレス空間が予約されていること。

アルゴリズムまたは振る舞い

1. ShmemProtectiveRegion および UsedShmemSegAddr に対して VirtualFree を実行し、事前に確保されていた予約領域を解放する。 2. MapViewOfFileEx を呼び出し、UsedShmemSegID で指定された共有メモリを UsedShmemSegAddr で指定された正確なアドレスにマッピングする。 3. マッピングされたアドレスが期待値（origUsedShmemSegAddr）と一致するか検証する。 4. hdr->magic を確認し、PostgreSQL の共有メモリであることを検証する。 5. dsm_set_control_handle を呼び出し、動的共有メモリの制御ハンドルを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c
  • **呼び出し先**: VirtualFree (Win32 API), MapViewOfFileEx (Win32 API), dsm_set_control_handle (内部関数), elog (エラー報告)

関連データ構造

  • PGShmemHeader: 共有メモリのヘッダー構造体。
  • UsedShmemSegAddr: 共有メモリのベースアドレスを保持するグローバル変数。
  • UsedShmemSegID: 共有メモリのハンドルを保持するグローバル変数。
  • ShmemProtectiveRegion: アドレス空間保護用の予約領域。

失敗モードと境界条件

  • VirtualFree が失敗した場合: FATAL エラーで終了。
  • MapViewOfFileEx が失敗した場合: FATAL エラーで終了。
  • マッピングされたアドレスが期待値と異なる場合: FATAL エラーで終了。
  • マジックナンバーが一致しない場合: FATAL エラーで終了。

保守担当者向けメモ

  • 本関数は Windows 特有のアドレス空間管理（ASLR 等による競合回避）に強く依存している。
  • UsedShmemSegAddr への再代入は冗長である可能性があるが、現在の実装では明示的に行われている。
