ProcessCatchupInterrupt

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/sinval.c:173

目的

遅延しているバックエンドに対し、共有キャッシュ無効化メッセージの処理を強制的に実行させ、キャッシュの整合性を同期させる。

シグネチャまたは形状

void ProcessCatchupInterrupt(void)

入力と出力

  • 入力: なし（グローバル変数 catchupInterruptPending を参照）
  • 出力: なし（副作用としてキャッシュ無効化処理を実行）

前提条件と不変条件

  • catchupInterruptPending フラグがセットされている場合に呼び出される。
  • 処理完了後、ReceiveSharedInvalidMessages() を経由して catchupInterruptPending フラグがリセットされる必要がある。

アルゴリズムまたは振る舞い

1. catchupInterruptPending が真である間、以下のループを繰り返す。 2. 現在トランザクション内（またはトランザクションブロック内）にいる場合、AcceptInvalidationMessages() を呼び出してキャッシュを更新する。 3. トランザクション外にいる場合、StartTransactionCommand() と CommitTransactionCommand() を順次呼び出し、その過程で AcceptInvalidationMessages() を実行させる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/autovacuum.c
  • src/backend/tcop/postgres.c
  • 呼び出し先:
  • AcceptInvalidationMessages()
  • StartTransactionCommand()
  • CommitTransactionCommand()
  • IsTransactionOrTransactionBlock()

関連データ構造

  • catchupInterruptPending: キャッチアップが必要であることを示すグローバルフラグ。

失敗モードと境界条件

  • トランザクション外での実行時にエラーが発生した場合のクリーンアップ挙動については、ソースコード内で懸念が示されている（現状はトランザクションの開始・終了でラップすることで対応している）。

保守担当者向けメモ

  • 本関数はシグナルハンドラから直接呼ばれるのではなく、メインループ等の安全なタイミングで呼び出されることを想定している。
  • トランザクションのオーバーヘッドを避ける最適化の余地はあるが、エラー発生時の安全性確保のため現在の実装が維持されている。
