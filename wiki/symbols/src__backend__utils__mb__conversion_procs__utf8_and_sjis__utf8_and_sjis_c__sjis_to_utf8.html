sjis_to_utf8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_sjis/utf8_and_sjis.c:41

目的

PostgreSQLのエンコーディング変換フレームワークにおいて、Shift-JIS（SJIS）形式の文字列をUTF-8形式へ変換する。

シグネチャまたは形状

Datum sjis_to_utf8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取る。
  • src (CSTRING): 変換元文字列。
  • dest (CSTRING): 変換先バッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 変換エラー発生時の挙動制御フラグ。
  • **出力**: 変換されたバイト数を Datum 型（INT32）として返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_SJIS, PG_UTF8) により、ソースエンコーディングがSJIS、ターゲットエンコーディングがUTF-8であることを検証する。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認する。 3. LocalToUtf 関数を呼び出し、sjis_to_unicode_tree を参照して変換を実行する。 4. 変換結果のバイト数を PG_RETURN_INT32 で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの変換エンジン（直接的な呼び出し元関数の詳細はソース内に記載なし）。
  • **呼び出し先**: LocalToUtf（src/backend/utils/mb/wchar.c で定義される汎用変換関数）。

関連データ構造

  • sjis_to_unicode_tree: 変換に使用される静的なツリー構造（../../Unicode/sjis_to_utf8.map よりインクルード）。

失敗モードと境界条件

  • noError 引数が false の場合、変換不可能な文字が含まれるとエラーが発生する可能性がある（詳細は LocalToUtf の実装に依存）。

保守担当者向けメモ

  • 本関数はラッパーであり、実際の変換ロジックは LocalToUtf および sjis_to_unicode_tree に依存している。
  • 変換テーブルの修正が必要な場合は、本ファイルを直接編集せず、../../Unicode/sjis_to_utf8.map を更新すること。
