pg_checksum_type_name

定義

  • 種別: function
  • 定義箇所: src/common/checksum_helper.c:55

目的

チェックサムアルゴリズムの列挙型識別子（pg_checksum_type）を、人間が読み取り可能な文字列形式に変換します。

シグネチャまたは形状

char *pg_checksum_type_name(pg_checksum_type type);

入力と出力

  • **入力**: pg_checksum_type 型の列挙値。
  • **出力**: アルゴリズム名を表す文字列リテラル（"NONE", "CRC32C", "SHA224", "SHA256", "SHA384", "SHA512" のいずれか）。

前提条件と不変条件

  • 入力された type が定義済みの pg_checksum_type 列挙値の範囲内であること。
  • 範囲外の値が渡された場合、Assert(false) によりデバッグビルドでは停止し、リリースビルドでは "???" を返します。

アルゴリズムまたは振る舞い

1. switch 文を用いて、入力された type に対応する文字列リテラルを返します。 2. 該当するケースがない場合は、アサーション失敗を発生させた上で "???" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/backup_manifest.c
  • src/bin/pg_combinebackup/copy_file.c
  • src/bin/pg_combinebackup/reconstruct.c
  • src/bin/pg_combinebackup/write_manifest.c
  • **呼び出し先**: 本関数は標準ライブラリ関数や他の内部関数を呼び出しません。

関連データ構造

  • pg_checksum_type: チェックサムアルゴリズムを識別するための列挙型。

失敗モードと境界条件

  • 未定義の pg_checksum_type 値が渡された場合、Assert がトリガーされ、戻り値として "???" が返されます。

保守担当者向けメモ

  • 新しいチェックサムアルゴリズムを追加した際は、本関数の switch 文にも対応するケースを追加してください。
