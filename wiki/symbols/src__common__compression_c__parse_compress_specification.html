parse_compress_specification

定義

  • 種別: function
  • 定義箇所: src/common/compression.c:106

目的

圧縮アルゴリズムの設定文字列（level=N,workers=M 等）を解析し、pg_compress_specification 構造体に変換・検証する。

シグネチャまたは形状

void parse_compress_specification(pg_compress_algorithm algorithm, char *specification, pg_compress_specification *result)

入力と出力

  • **入力**:
  • algorithm: 使用する圧縮アルゴリズム（pg_compress_algorithm 型）。
  • specification: 解析対象の文字列（例: "level=5,workers=2"）。NULL の場合はデフォルト設定が適用される。
  • result: 設定結果を格納する pg_compress_specification 構造体へのポインタ。
  • **出力**: なし（result 構造体の各フィールドを更新）。エラー発生時は result->parse_error にメッセージが格納される。

前提条件と不変条件

  • result は呼び出し元で確保されていること。
  • ビルド時に各圧縮ライブラリ（Zstd, LZ4, zlib）が有効でない場合、該当アルゴリズムを指定すると parse_error が設定される。
  • parse_error が設定された場合、呼び出し側はメモリ解放（pfree）を行う責任がある。

アルゴリズムまたは振る舞い

1. **初期化**: result の各フィールドを初期化し、アルゴリズムに応じたデフォルトの圧縮レベルを設定する。 2. **特殊ケース**: specification が単なる数値文字列の場合、それを圧縮レベルとして解釈し終了する。 3. **パース処理**: カンマ区切りの keyword=value 形式をループで走査する。

  • level: 整数値として解析し result->level に格納。
  • workers: 整数値として解析し result->workers に格納。PG_COMPRESSION_OPTION_WORKERS フラグを立てる。
  • long: 真偽値として解析し result->long_distance に格納。PG_COMPRESSION_OPTION_LONG_DISTANCE フラグを立てる。

    • 4. **エラー処理**: 未知のキーワードや空のキーワードが検出された場合、parse_error を設定して処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • src/bin/pg_dump/pg_dump.c
  • **呼び出し先**:

  • expect_boolean_value
  • palloc / pfree / psprintf / pstrdup (PostgreSQLメモリ管理関数)

関連データ構造

  • pg_compress_specification: 圧縮設定を保持する構造体。
  • pg_compress_algorithm: アルゴリズムの種類を定義する列挙型。

失敗モードと境界条件

  • **ビルド依存**: サポートされていないアルゴリズムを指定した場合、エラーメッセージが生成される。
  • **解析エラー**: 不正なフォーマットや未知のオプションが指定された場合、parse_error にエラー詳細が格納される。
  • **メモリ不足**: palloc 失敗時は例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数は parse_compress_options で分割された後の詳細設定を処理する。
  • 新しい圧縮オプションを追加する場合は、キーワード解析ループに条件分岐を追加し、pg_compress_specification 構造体にフィールドを追加すること。
  • parse_error のメモリ管理（pfree）は呼び出し側に依存するため、利用箇所でのリークに注意が必要。
