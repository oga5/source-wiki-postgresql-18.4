get_encoding_name_for_icu

定義

  • 種別: function
  • 定義箇所: src/common/encnames.c:474

目的

PostgreSQL の内部エンコーディング ID を、ICU ライブラリが認識可能なエンコーディング名（文字列）に変換します。

シグネチャまたは形状

const char *get_encoding_name_for_icu(int encoding);

入力と出力

  • **入力**: encoding (int) - PostgreSQL 内部のエンコーディング ID。
  • **出力**: const char * - ICU で使用されるエンコーディング名。無効なエンコーディングの場合は NULL を返します。

前提条件と不変条件

  • 入力された encoding が PG_VALID_BE_ENCODING マクロによって定義される有効なバックエンドエンコーディング範囲内である必要があります。

アルゴリズムまたは振る舞い

1. PG_VALID_BE_ENCODING(encoding) を使用して、指定された ID が有効なバックエンドエンコーディングかを確認します。 2. 無効な場合は NULL を返して終了します。 3. 有効な場合は、内部テーブル pg_enc2icu_tbl を encoding をインデックスとして参照し、対応する文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale_icu.c 内の icu_encoding_name の割り当て処理において呼び出されています。
  • **呼び出し先**: 内部マクロ PG_VALID_BE_ENCODING を使用しています。

関連データ構造

  • pg_enc2icu_tbl: 内部エンコーディング ID から ICU エンコーディング名へのマッピングを保持するテーブル。

失敗モードと境界条件

  • encoding が有効なバックエンドエンコーディングの範囲外である場合、関数は NULL を返します。呼び出し元はこの戻り値を適切に処理する必要があります。

保守担当者向けメモ

  • 本関数はバックエンドエンコーディングのみを対象としています。
  • pg_enc2icu_tbl のサイズは、エンコーディング数と一致していることが StaticAssertDecl によって保証されています。
