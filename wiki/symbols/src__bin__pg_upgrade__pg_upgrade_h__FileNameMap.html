FileNameMap

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.h:180

目的

旧クラスタと新クラスタ間における、物理ファイルパスの対応関係を保持するための構造体です。アップグレード処理において、どの旧ファイルをどの新ファイルへ移行・リンクすべきかを管理します。

シグネチャまたは形状

typedef struct
{
	const char *old_tablespace;
	const char *new_tablespace;
	const char *old_tablespace_suffix;
	const char *new_tablespace_suffix;
	Oid			db_oid;
	RelFileNumber relfilenumber;
	char	   *nspname;
	char	   *relname;
} FileNameMap;

入力と出力

  • **入力**: gen_db_file_maps 関数により、旧クラスタの DbInfo から生成されます。
  • **出力**: transfer_relfile 等の関数に渡され、実際のファイル移行処理のパラメータとして使用されます。また、ログ出力やエラー報告のソースとして利用されます。

前提条件と不変条件

  • nspname および relname は、主にログ出力およびエラー報告の目的で使用されます。
  • relfilenumber は、ファイルマッピングの識別子として機能します。

アルゴリズムまたは振る舞い

  • pg_upgrade の移行プロセスにおいて、旧クラスタの物理ファイルと新クラスタの対応付けを定義します。
  • relfilenumber.c 内の qsort 用比較関数において、RelFileNumber に基づくソートのキーとして利用されます。
  • transfer_single_new_db 関数において、データベースごとのファイル転送処理の単位として扱われます。

呼び出し元と呼び出し先

  • **呼び出し元（生成・参照）**:
  • src/bin/pg_upgrade/info.c: gen_db_file_maps 関数で生成されます。
  • src/bin/pg_upgrade/relfilenumber.c: 移行処理のループ内で参照されます。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • DbInfo: FileNameMap を生成するための元データを提供します。
  • RelFileNumber: 構造体内でファイル識別子として使用されます。

失敗モードと境界条件

  • 構造体内の文字列ポインタ（nspname, relname）が NULL の場合、ログ出力時に適切にハンドリングする必要があります。
  • 比較関数（qsort）において、FileNameMap のポインタが正しくキャストされていない場合、未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は pg_upgrade の中核的なデータ構造であり、ファイル移行ロジックの変更時には必ず影響範囲を確認してください。
  • nspname と relname はメモリ管理の対象となる可能性があるため、割り当てと解放のライフサイクルに注意してください。
