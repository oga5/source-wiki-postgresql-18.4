show_timezone

定義

  • 種別: function
  • 定義箇所: src/backend/commands/variable.c:391

目的

現在のセッションにおけるタイムゾーン設定を、正規化された名称（canonical name）として取得し、表示用に提供する。

シグネチャまたは形状

const char *show_timezone(void)

入力と出力

  • **入力**: なし（グローバル変数 session_timezone を参照）。
  • **出力**: タイムゾーンの正規名称を表す文字列ポインタ。解決できない場合は "unknown" を返す。

前提条件と不変条件

  • session_timezone が有効なタイムゾーン設定を保持していること。
  • pg_get_timezone_name が呼び出し可能であること。

アルゴリズムまたは振る舞い

1. pg_get_timezone_name を呼び出し、現在の session_timezone に対応する正規名称を取得する。 2. 取得した名称が NULL でなければ、その文字列ポインタを返す。 3. NULL の場合は、フォールバックとして "unknown" という文字列リテラルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: GUC（Grand Unified Configuration）サブシステム（src/backend/utils/misc/guc_tables.c で show_hook として登録）。
  • **呼び出し先**: pg_get_timezone_name（タイムゾーン名の解決）。

関連データ構造

  • session_timezone: セッションのタイムゾーン状態を保持する内部変数。

失敗モードと境界条件

  • タイムゾーン名が解決できない場合、pg_get_timezone_name は NULL を返し、本関数は "unknown" を返す。

保守担当者向けメモ

  • 本関数は show_hook として機能するため、SQLの SHOW timezone コマンド実行時に呼び出される。
  • 戻り値の文字列は静的領域または pg_get_timezone_name が管理する領域を指すため、呼び出し側での解放は不要。
