PQsendQueryPrepared

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-exec.c:1649

目的

サーバー側で事前に準備されたステートメント（Prepared Statement）を、指定されたパラメータを用いて非同期に実行する。

シグネチャまたは形状

int PQsendQueryPrepared(PGconn *conn,
                        const char *stmtName,
                        int nParams,
                        const char *const *paramValues,
                        const int *paramLengths,
                        const int *paramFormats,
                        int resultFormat);

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル。
  • stmtName: 実行する準備済みステートメントの名前。
  • nParams: パラメータの数。
  • paramValues: パラメータ値の配列。
  • paramLengths: 各パラメータの長さ（バイナリ形式の場合）。
  • paramFormats: 各パラメータの形式（0: テキスト, 1: バイナリ）。
  • resultFormat: 結果の形式（0: テキスト, 1: バイナリ）。
  • **出力**:
  • 成功時は 1、失敗時は 0 を返す。

前提条件と不変条件

  • conn は有効な PGconn オブジェクトであること。
  • stmtName は NULL であってはならない。
  • nParams は 0 以上 PQ_QUERY_PARAM_MAX_LIMIT 以下であること。

アルゴリズムまたは振る舞い

1. PQsendQueryStart を呼び出し、接続状態がクエリ送信可能かを確認する。 2. 引数 stmtName および nParams の妥当性を検証する。検証に失敗した場合は libpq_append_conn_error を通じてエラーを記録し、0 を返す。 3. 検証を通過した場合、内部関数 PQsendQueryGuts を呼び出し、準備済みステートメント実行用のプロトコルメッセージを構築・送信する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/bin/pg_rewind/libpq_source.c
  • src/bin/pgbench/pgbench.c
  • src/bin/psql/common.c
  • src/test/modules/libpq_pipeline/libpq_pipeline.c
  • **呼び出し先**:
  • PQsendQueryStart
  • libpq_append_conn_error
  • PQsendQueryGuts

関連データ構造

  • PGconn: 接続状態を保持する構造体。

失敗モードと境界条件

  • stmtName が NULL の場合、エラーとして 0 を返す。
  • nParams が範囲外（負または PQ_QUERY_PARAM_MAX_LIMIT 超過）の場合、エラーとして 0 を返す。
  • 接続がビジー状態や無効な状態である場合、PQsendQueryStart が失敗し、本関数も 0 を返す。

保守担当者向けメモ

  • 本関数は PQsendQueryGuts のラッパーであり、準備済みステートメント実行に特化した引数チェックを行う。
  • 内部的なプロトコルメッセージ構築の詳細は PQsendQueryGuts に依存している。
