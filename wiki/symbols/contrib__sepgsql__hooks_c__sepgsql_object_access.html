sepgsql_object_access

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/hooks.c:88

目的

PostgreSQL のオブジェクトアクセスフックのメインディスパッチャとして機能し、DDL 操作（作成、削除、変更、切り詰め、名前空間検索）に対する SE-PostgreSQL のセキュリティ権限チェックを適切なハンドラへ振り分けます。

シグネチャまたは形状

static void sepgsql_object_access(ObjectAccessType access, Oid classId, Oid objectId, int subId, void *arg);

入力と出力

  • access: ObjectAccessType（操作の種類）
  • classId: 操作対象のシステムカタログ OID
  • objectId: 操作対象オブジェクトの OID
  • subId: 属性 ID（列操作の場合に使用）
  • arg: 操作固有のデータ構造（ObjectAccessPostCreate 等）へのポインタ

前提条件と不変条件

  • 他のセキュリティモジュールとの共存のため、next_object_access_hook が存在する場合は必ず先に実行します。
  • is_internal フラグが真の場合、システム内部的な操作（TOAST テーブル作成や一時的なクリーンアップ等）として権限チェックをスキップします。
  • OAT_NAMESPACE_SEARCH において、先行するフックが既にアクセスを拒否している場合（ns_arg->result が偽）は、処理を中断します。

アルゴリズムまたは振る舞い

1. **フック連鎖**: 登録済みの next_object_access_hook を呼び出します。 2. **ディスパッチ**: access 型に基づき分岐します。

  • OAT_POST_CREATE: is_internal を確認し、sepgsql_database_post_create 等の各オブジェクト用ハンドラを呼び出します。
  • OAT_DROP: PERFORM_DELETION_INTERNAL フラグを確認し、内部削除であればスキップ、それ以外は sepgsql_database_drop 等を呼び出します。
  • OAT_TRUNCATE: sepgsql_relation_truncate を呼び出します。
  • OAT_POST_ALTER: is_internal を確認し、sepgsql_database_setattr 等を呼び出します。
  • OAT_NAMESPACE_SEARCH: 名前空間の検索権限を検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL コアの object_access_hook（contrib/sepgsql/hooks.c:474 にて登録）。
  • **呼び出し先**: sepgsql_database_..., sepgsql_schema_..., sepgsql_relation_..., sepgsql_attribute_..., sepgsql_proc_... 等の各モジュール関数。

関連データ構造

  • ObjectAccessType (PostgreSQL コア定義)
  • ObjectAccessPostCreate, ObjectAccessDrop, ObjectAccessPostAlter, ObjectAccessNamespaceSearch (PostgreSQL コア定義)
  • sepgsql_context_info (DDL コンテキスト管理用)

失敗モードと境界条件

  • 未サポートの classId が渡された場合は、何も行わず無視します。
  • is_internal が真であるべき箇所で偽の場合、あるいはその逆の場合、Assert によりデバッグ時に停止します。

保守担当者向けメモ

  • 本関数は hooks.c 内で object_access_hook に代入されることで機能します。
  • 新しいオブジェクトタイプや操作を追加する場合は、switch 文の classId 分岐にケースを追加してください。
  • 権限チェックをバイパスする内部処理の判定ロジック（is_internal や dropflags）は、セキュリティの根幹に関わるため慎重に変更してください。
