RemoveSocketFiles

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqcomm.c:847

目的

Postmaster の終了処理において、作成済みの Unix ドメインソケットファイルをファイルシステムから削除するために使用されます。

シグネチャまたは形状

void RemoveSocketFiles(void)

入力と出力

  • 入力: なし（グローバル変数 sock_paths を参照）
  • 出力: なし（副作用としてファイルシステム上のソケットファイルを削除）

前提条件と不変条件

  • sock_paths リストには、作成されたソケットファイルのパスが格納されている必要があります。
  • 本関数はプロセス終了直前に呼び出されることを想定しており、呼び出し後に sock_paths は NIL にリセットされます。

アルゴリズムまたは振る舞い

1. グローバル変数 sock_paths に保持されているリストを走査します。 2. 各要素（ソケットファイルのパス）に対して unlink() を実行します。 3. unlink() の戻り値（エラーの有無）は無視されます。 4. 最後に sock_paths を NIL に設定し、メモリ解放を行わずに終了します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/postmaster.c 内の終了処理フローから直接呼び出されます。
  • 呼び出し先: unlink()（標準ライブラリ関数）。

関連データ構造

  • sock_paths: ソケットファイルのパス文字列を保持するリスト（List）。

失敗モードと境界条件

  • unlink() が失敗した場合（ファイルが存在しない、権限がない等）でも、エラーを報告せず処理を続行します。
  • プロセス終了処理の一部であるため、メモリリークの防止よりもソケットファイルの確実な削除が優先されます。

保守担当者向けメモ

  • 本関数は postmaster のシャットダウン時にのみ実行されることを前提としています。
  • sock_paths の管理は pqcomm.c 内の他のソケット作成関数と密接に関連しているため、パスの追加・削除ロジックを変更する際は注意してください。
