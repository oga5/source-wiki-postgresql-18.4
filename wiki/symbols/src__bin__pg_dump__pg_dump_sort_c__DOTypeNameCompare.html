DOTypeNameCompare

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump_sort.c:199

目的

DumpableObject のリストを、型優先度、名前空間、オブジェクト名、型、および特定のオブジェクト型ごとの自然キーに基づいて安定的にソートする。

シグネチャまたは形状

static int DOTypeNameCompare(const void *p1, const void *p2)

入力と出力

  • **入力**: p1, p2 (比較対象の DumpableObject へのポインタを指すポインタ)
  • **出力**: 比較結果を示す整数（負: p1 < p2, 0: p1 == p2, 正: p1 > p2）

前提条件と不変条件

  • 入力ポインタは DumpableObject* 型にキャスト可能であること。
  • dbObjectTypePriority 配列が初期化されていること。
  • オブジェクトの型に応じて、FuncInfo, OprInfo, OpclassInfo, OpfamilyInfo, CollInfo, AttrDefInfo, PolicyInfo, RuleInfo へのキャストが安全であること。

アルゴリズムまたは振る舞い

以下の優先順位で比較を行い、差分が見つかった時点で結果を返す： 1. **型優先度**: dbObjectTypePriority に基づく比較。 2. **名前空間**: 名前空間の有無（NULLは後方）および名前空間名（strcmp）による比較。 3. **オブジェクト名**: dobj.name による比較。 4. **オブジェクト型**: objType の数値比較。 5. **型固有のタイブレーク**:

  • DO_FUNC, DO_AGG: 引数の数、および各引数の型名（pgTypeNameCompare）で比較。
  • DO_OPERATOR: 演算子種別（oprkind）、左辺型、右辺型で比較。
  • DO_OPCLASS, DO_OPFAMILY: アクセスメソッド名（accessMethodNameCompare）で比較。
  • DO_COLLATION: エンコーディングで比較。
  • DO_ATTRDEF: 属性番号（adnum）で比較。
  • DO_POLICY, DO_RULE: 関連するテーブル名で比較。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていない。
  • **呼び出し先**: strcmp, pgTypeNameCompare, accessMethodNameCompare。

関連データ構造

  • DumpableObject: ソート対象の基本構造体。
  • FuncInfo, OprInfo, OpclassInfo, OpfamilyInfo, CollInfo, AttrDefInfo, PolicyInfo, RuleInfo: 各オブジェクト型固有の拡張情報。

失敗モードと境界条件

  • 名前空間が片方のみ存在する場合、NULLでない方を優先する。
  • 循環依存や複雑な依存関係がある場合、この関数によるソート順序が復元時の整合性に影響を与える可能性がある。

保守担当者向けメモ

  • ソート順序の変更は 002_pg_upgrade.pl 等のテストで差分を生む可能性があるため注意すること。
  • 新しいオブジェクト型を追加した際は、必要に応じてタイブレーク処理を拡張すること。
  • DO_TRIGGER 等、関数末尾で処理が途切れている箇所があるため、追加実装時は既存のパターンに従うこと。
