libpq_oauth_init

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq-oauth/oauth-utils.c:64

目的

libpq から動的にロードされたモジュールに対し、libpq 内部の関数ポインタを注入し、モジュールが libpq の内部機能（エラー処理やデータ構造操作など）へ安全にアクセスできるようにする。

シグネチャまたは形状

void libpq_oauth_init(pgthreadlock_t threadlock_impl,
                      libpq_gettext_func gettext_impl,
                      conn_errorMessage_func errmsg_impl,
                      conn_oauth_client_id_func clientid_impl,
                      conn_oauth_client_secret_func clientsecret_impl,
                      conn_oauth_discovery_uri_func discoveryuri_impl,
                      conn_oauth_issuer_id_func issuerid_impl,
                      conn_oauth_scope_func scope_impl,
                      conn_sasl_state_func saslstate_impl,
                      set_conn_altsock_func setaltsock_impl,
                      set_conn_oauth_token_func settoken_impl);

入力と出力

  • **入力**: libpq 内部の機能に対応する11個の関数ポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数は USE_DYNAMIC_OAUTH が定義された環境でのみ使用可能。
  • 呼び出し側は、libpq の内部構造（PGconn 等）に直接依存せず、本関数で注入されたアクセサ関数を介して操作を行う必要がある。

アルゴリズムまたは振る舞い

引数として渡された各関数ポインタを、モジュール内の静的変数（pg_g_threadlock, conn_errorMessage 等）に代入し、以降の処理で利用可能な状態にする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-oauth.c 内で dlsym を介して本関数が検索・実行される。
  • **呼び出し先**: 本関数自体は他の関数を呼び出さず、ポインタの代入のみを行う。

関連データ構造

  • PGconn（間接的に操作対象となる）
  • libpq_gettext_func などの関数ポインタ型（oauth-utils.h で定義）

失敗モードと境界条件

  • 注入される関数ポインタが NULL である場合の挙動については、本ソースコード内では明示的なチェックは行われていない。

保守担当者向けメモ

  • 本関数は libpq と libpq-oauth モジュール間の疎結合を維持するための重要な接点である。
  • 新たな内部機能へのアクセスが必要になった場合は、本関数の引数と静的変数を追加し、oauth-utils.h の型定義を更新すること。
