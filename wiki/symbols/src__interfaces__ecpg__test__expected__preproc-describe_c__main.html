目的

ECPG（Embedded SQL for C）プリプロセッサの DESCRIBE 文の動作を検証するための回帰テスト用エントリポイント。動的 SQL の準備、記述子へのメタデータ格納、およびフィールド情報の抽出プロセスを網羅的に実行する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（引数なし）
  • 出力: 正常終了時に 0 を返し、エラー発生時に exit(1) を通じて異常終了する。

前提条件と不変条件

  • データベース ecpg1_regression への接続が成功していること。
  • SQLCA（sqlca）が各 SQL 操作後のエラーチェックに使用されること。
  • ECPGdebug(1, stderr) が有効であり、実行時に標準エラー出力へ SQL トレースが出力されること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGconnect でデータベースに接続し、t1 テーブルを作成してテストデータを投入する。 2. **準備**: ECPGprepare を使用して SQL 文（SELECT id, t FROM t1）をプリコンパイルし、識別子 st_id1 を割り当てる。 3. **記述子確保**: ECPGallocate_desc を呼び出し、desc1 から desc4 までの記述子を確保する。 4. **メタデータ取得**: ECPGdescribe を使用して、準備した SQL 文のメタデータを各記述子に格納する。 5. **ヘッダー抽出**: ECPGget_desc_header を呼び出し、各記述子からフィールド数を取得する。 6. **終了処理**: 各操作後に sqlca.sqlcode < 0 を確認し、エラー時は即座に終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGtrans, ECPGallocate_desc, ECPGprepare, ECPGdescribe, ECPGget_desc_header（いずれも ecpglib の関数）

関連データ構造

  • sqlca: SQL 通信領域。エラーコードの保持に使用。
  • stmt1, stmt2: 実行対象の SQL 文を保持する文字列。
  • count1〜count4: 記述子から取得したフィールド数を格納する整数変数。

失敗モードと境界条件

  • 各 SQL 操作後に sqlca.sqlcode < 0 をチェックしており、データベース接続失敗、テーブル作成失敗、記述子割り当て失敗、準備失敗、記述失敗、ヘッダー取得失敗のいずれかにおいて exit(1) で終了する。

保守担当者向けメモ

  • 本ファイルはプリプロセッサによって自動生成されたコードの「期待値」であるため、手動編集は避けること。
  • DESCRIBE の結果として得られるフィールド名が期待通りかを確認するテスト構造となっている。
