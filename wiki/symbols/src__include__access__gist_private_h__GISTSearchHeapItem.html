GISTSearchHeapItem

定義

  • 種別: struct
  • 定義箇所: src/include/access/gist_private.h:118

目的

GiSTインデックスの検索処理において、ヒープタプルに関する情報を保持するための構造体です。インデックススキャン中に見つかったタプルの位置情報や、再チェックの要否、インデックスのみのスキャン（index-only scan）で再構成されたデータを管理します。

シグネチャまたは形状

typedef struct GISTSearchHeapItem
{
    ItemPointerData heapPtr;        /* ヒープタプルの位置 */
    bool            recheck;        /* 条件の再チェック要否 */
    bool            recheckDistances; /* 距離の再チェック要否 */
    HeapTuple       recontup;       /* インデックスから再構成されたデータ */
    OffsetNumber    offnum;         /* ページ内のオフセット（LP_DEADマーク用） */
} GISTSearchHeapItem;

入力と出力

本構造体はデータ保持用であり、関数ではありません。インデックススキャン処理の過程で、検索結果のキューや一時的なバッファとしてインスタンス化されます。

前提条件と不変条件

  • offnum は、対応するインデックスページ内でのタプル位置を指し、LP_DEAD フラグの管理に使用されます。
  • recontup は、インデックスのみのスキャンが有効な場合にのみ使用されます。

アルゴリズムまたは振る舞い

GiSTインデックスの検索アルゴリズムにおいて、インデックスページから抽出されたヒープタプルの情報を一時的に保持します。特に、検索条件の再評価が必要な場合や、距離計算の再検証が必要な場合に、この構造体のフラグ（recheck, recheckDistances）が参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/access/gist_private.h 内の他の構造体（GISTScanOpaqueData など）で利用されています。
  • 呼び出し先: 具体的な関数呼び出し関係は、提供されたソース excerpt からは特定できません。

関連データ構造

  • ItemPointerData: ヒープ上の物理的な位置を特定するために使用。
  • HeapTuple: 再構成されたタプルデータを保持するために使用。
  • GISTScanOpaqueData: スキャン実行時の状態管理構造体であり、本構造体を保持します。

失敗モードと境界条件

  • recontup が NULL の場合、インデックスのみのスキャンによるデータ再構成が行われていないことを示します。
  • offnum が無効な値である場合、ページ内のタプル操作（LP_DEAD マークなど）に失敗する可能性があります。
