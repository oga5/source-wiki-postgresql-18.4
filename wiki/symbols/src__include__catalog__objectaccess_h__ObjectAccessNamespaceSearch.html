ObjectAccessNamespaceSearch

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/objectaccess.h:108

目的

スキーマ検索時のアクセス制御を外部プラグインが判定するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    bool ereport_on_violation;
    bool result;
} ObjectAccessNamespaceSearch;

入力と出力

  • **入力**:
  • ereport_on_violation: スキーマ検索権限が拒否された際にエラーを報告すべきかを示すフラグ。
  • **出力**:
  • result: アクセス可否を示すフラグ。コアコードは初期値として true を設定し、プラグインが拒否する場合に false にリセットします。

前提条件と不変条件

  • result フィールドは、複数の拡張機能がフックを登録している場合を考慮し、すべての拡張機能がアクセスを許可する場合のみ true となるよう、プラグイン側で true を代入してはなりません（false へのリセットのみが許可されます）。

アルゴリズムまたは振る舞い

1. コアコードが ObjectAccessNamespaceSearch 構造体を初期化し、result を true に設定します。 2. 登録された外部フックが呼び出され、必要に応じて result を false に更新します。 3. 最終的な result の値に基づき、コアコードがアクセス制御を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaccess.c 内の関数（フック実行の仲介）。
  • **呼び出し先**:
  • contrib/sepgsql/hooks.c および src/test/modules/test_oat_hooks/test_oat_hooks.c でこの構造体が参照されています。

関連データ構造

  • ObjectAccessType: 監視対象イベントの種類を定義する列挙型。

失敗モードと境界条件

  • ereport_on_violation が true の場合、アクセス拒否時にプラグインまたはコアコードがエラーを発生させる必要があります。

保守担当者向けメモ

  • 本構造体は InvokeObjectAccessHook マクロを通じて利用されることを前提としています。
  • 複数の拡張機能がフックを登録している場合、result の論理積（AND）が最終的な判定結果となる点に注意してください。
