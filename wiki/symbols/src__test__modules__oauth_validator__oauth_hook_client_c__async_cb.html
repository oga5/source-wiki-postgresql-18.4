async_cb

定義

  • 種別: function
  • 定義箇所: src/test/modules/oauth_validator/oauth_hook_client.c:228

目的

libpq の非同期認証プロセスにおいて、認証データの提供または接続待機状態の制御を行うコールバック関数です。主にテスト環境下で、正常なトークン注入や意図的な接続ハングアップのシミュレーションに使用されます。

シグネチャまたは形状

static PostgresPollingStatusType async_cb(PGconn *conn, PGoauthBearerRequest *req, pgsocket *altsock)

入力と出力

  • **入力**:
  • conn: 現在のデータベース接続ハンドル。
  • req: OAuth ベアラーリクエスト構造体。
  • altsock: 非同期待機時に libpq が監視すべきソケットを格納するためのポインタ。
  • **出力**:
  • PostgresPollingStatusType: 処理状態（PGRES_POLLING_OK は完了、PGRES_POLLING_READING は待機、PGRES_POLLING_FAILED は失敗）。

前提条件と不変条件

  • hang_forever フラグが有効な場合、静的変数 sock が PGINVALID_SOCKET で初期化されている必要があります。
  • hang_forever が無効な場合、req->token に適切なトークン文字列が代入されることが期待されます。

アルゴリズムまたは振る舞い

1. **ハングアップテストモード (hang_forever が真の場合)**:

  • 初回呼び出し時に UDP ソケットを作成し、altsock に割り当てます。
  • PGRES_POLLING_READING を返し、libpq にそのソケットでの待機を強制します。これにより、接続タイムアウト処理の検証を行います。

    • 2. **通常モード**:

  • req->token にグローバルな token 変数を設定します。
  • PGRES_POLLING_OK を返し、認証プロセスを正常に終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: handle_auth_data 関数内で req->async に割り当てられ、libpq の認証フローから間接的に呼び出されます。
  • **呼び出し先**: socket() (システムコール)、WSAStartup() (Windows環境のみ)、perror()。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。
  • PGoauthBearerRequest: OAuth 認証リクエスト情報。

失敗モードと境界条件

  • WSAStartup または socket 作成に失敗した場合、PGRES_POLLING_FAILED を返します。
  • hang_forever モードでは、ソケットが読み取り可能になることはないため、接続タイムアウトが発生するまで待機状態が継続します。

保守担当者向けメモ

  • 本関数はテスト専用です。実運用コードへの組み込みは想定されていません。
  • Windows 環境におけるソケット初期化処理が含まれているため、プラットフォーム固有の挙動に注意してください。
