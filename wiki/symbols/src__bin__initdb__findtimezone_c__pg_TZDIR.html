pg_TZDIR

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/findtimezone.c:36

目的

PostgreSQLがタイムゾーン定義ファイルを読み込む際に参照すべき、タイムゾーンデータベースのルートディレクトリパスを取得します。

シグネチャまたは形状

static const char *pg_TZDIR(void)

入力と出力

  • **入力**: なし
  • **出力**: タイムゾーンディレクトリのパスを示す const char * 型の文字列。

前提条件と不変条件

  • SYSTEMTZDIR マクロが定義されている場合はその値が返され、定義されていない場合は tzdirpath 変数の値が返されます。
  • 返されるパスは、PostgreSQLがタイムゾーンファイルを探索するための基準ディレクトリとして使用されます。

アルゴリズムまたは振る舞い

1. コンパイル時に SYSTEMTZDIR が定義されているかを確認します。 2. 定義されている場合、システム標準のタイムゾーンデータベースパスを返します。 3. 定義されていない場合、PostgreSQLの共有ディレクトリ配下にあるタイムゾーンデータパス（tzdirpath）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/findtimezone.c 内の複数箇所（strlcpy の引数として使用）。
  • src/timezone/pgtz.c 内の複数箇所。
  • **呼び出し先**:
  • 外部関数呼び出しはありません。

関連データ構造

  • tzdirpath: SYSTEMTZDIR が未定義の場合に参照されるグローバル変数。

失敗モードと境界条件

  • 本関数自体は常に有効な文字列ポインタを返す設計ですが、返されたパスがファイルシステム上に存在するかどうかは呼び出し側の環境に依存します。

保守担当者向けメモ

  • 本関数は src/bin/initdb/findtimezone.c と src/timezone/pgtz.c の両方から参照されています。パスの決定ロジックを変更する場合は、両方のモジュールへの影響を考慮してください。
