JsonTableParseContext

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_jsontable.c:32

目的

JSON_TABLE 構文の解析プロセスにおいて、パーサ状態や構築中のノード、および名前解決に必要な情報を保持・共有するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct JsonTableParseContext
{
	ParseState *pstate;      /* パーサの現在の状態 */
	JsonTable  *jt;          /* 解析対象の生の JsonTable ノード */
	TableFunc  *tf;          /* 構築中の TableFunc ノード */
	List	   *pathNames;   /* パス名およびカラム名のリスト */
	int			pathNameId;  /* パス名生成用の一意なIDカウンタ */
} JsonTableParseContext;

入力と出力

  • **入力**: transformJsonTable 関数内で ParseState を受け取り初期化されます。
  • **出力**: 解析過程で tf（TableFunc）が更新され、最終的にプランナが利用可能な形式へ変換されます。

前提条件と不変条件

  • pstate は有効な ParseState を指している必要があります。
  • pathNameId は、generateJsonTablePathName を通じて一意な名前を生成するために単調増加します。

アルゴリズムまたは振る舞い

  • JSON_TABLE の解析中、このコンテキストを介してカラム名やパス名の重複チェック（CheckDuplicateColumnOrPathNames）や、名前の検索（LookupPathOrColumnName）が行われます。
  • 再帰的な NESTED 句の処理においても、このコンテキストが引き回され、一貫した名前空間とプラン構築状態が維持されます。

呼び出し元と呼び出し先

  • **呼び出し元**: transformJsonTable が初期化の起点となります。
  • **呼び出し先**: 以下の関数がこのコンテキストを引数として受け取ります。
  • CheckDuplicateColumnOrPathNames
  • LookupPathOrColumnName
  • generateJsonTablePathName
  • transformJsonTableColumns
  • transformJsonTableNestedColumns

関連データ構造

  • ParseState: パーサのグローバルな状態。
  • JsonTable: 解析前の構文木ノード。
  • TableFunc: 解析後の実行計画用ノード。

失敗モードと境界条件

  • 名前解決において重複が検出された場合、CheckDuplicateColumnOrPathNames を通じてエラーが報告されます。
  • pathNameId の管理は、複雑なネスト構造においても一意性を保証するように設計されています。

保守担当者向けメモ

  • 本構造体は src/backend/parser/parse_jsontable.c 内でのみ使用されるプライベートなコンテキストです。
  • NESTED 句の再帰処理において、コンテキスト内の pathNames リストが正しく管理されていることを確認してください。
