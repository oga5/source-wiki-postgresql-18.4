visibilitymap_get_status

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/visibilitymap.c:342

目的

指定されたヒープブロックに対応する可視性マップ（Visibility Map）のビット状態を取得します。

シグネチャまたは形状

uint8 visibilitymap_get_status(Relation rel, BlockNumber heapBlk, Buffer *vmbuf)

入力と出力

  • **入力**:
  • rel: 対象リレーション。
  • heapBlk: 可視性を確認するヒープブロック番号。
  • vmbuf: マップページを保持するためのバッファ。呼び出し間で再利用可能。
  • **出力**:
  • 戻り値: ヒープブロックに対応する可視性ビット（VISIBILITYMAP_VALID_BITS でマスクされた値）。
  • vmbuf: 読み込まれたマップページを含むバッファ。

前提条件と不変条件

  • vmbuf は InvalidBuffer または以前の呼び出しで取得した有効なバッファであること。
  • 呼び出し側は、本関数がロックなしでビットを読み取るため、並行更新によるメモリ順序付けや不整合を考慮する必要があります。

アルゴリズムまたは振る舞い

1. heapBlk からマップ上のブロック番号、バイト位置、ビットオフセットを算出します。 2. vmbuf が有効かつ対象ブロックと一致する場合、そのバッファを再利用します。一致しない場合は既存のバッファを解放します。 3. vmbuf が無効な場合、vm_readbuf を呼び出してマップページを読み込みます。読み込み失敗時は false を返します。 4. マップページの内容から該当ビットを抽出し、VISIBILITYMAP_VALID_BITS でマスクして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • contrib/pg_visibility/pg_visibility.c
  • src/backend/access/heap/vacuumlazy.c
  • **呼び出し先**:
  • vm_readbuf (マップページの読み込み)
  • BufferGetBlockNumber, ReleaseBuffer, BufferGetPage, PageGetContents (バッファ管理)

関連データ構造

  • Relation: リレーション情報。
  • Buffer: バッファプール内のページ管理構造。
  • BlockNumber: ブロック番号。

失敗モードと境界条件

  • vm_readbuf が有効なバッファを返せない場合、false を返します。
  • マップページが存在しない、または読み取り権限がない場合、処理は失敗します。

保守担当者向けメモ

  • 本関数はロックを取得せずにビットを読み取ります。パフォーマンス優先の設計であり、メモリの可視性や並行更新の影響については呼び出し側で制御してください。
  • vmbuf の再利用は、連続したブロックのチェック時に効率を向上させます。
