gbt_timele

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_time.c:74

目的

2つの TimeADT 値を比較し、第1引数が第2引数以下であるかどうか（<=）を判定します。

シグネチャまたは形状

static bool gbt_timele(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の TimeADT 値へのポインタ（void * として渡される）。
  • flinfo: 関数マネージャ情報。
  • **出力**:
  • a <= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な TimeADT 値を指している必要があります。

アルゴリズムまたは振る舞い

1. 入力された void * 型のポインタを TimeADT * 型にキャストします。 2. TimeADTGetDatumFast マクロを使用して、各値を Datum 形式に変換します。 3. PostgreSQLのコア関数 time_le を DirectFunctionCall2 経由で呼び出し、比較を実行します。 4. 比較結果を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_time.c 内の静的テーブル（tinfo 構造体の一部）から参照されています。
  • **呼び出し先**:
  • time_le (PostgreSQLコアの比較関数)

関連データ構造

  • TimeADT: PostgreSQLの内部的な時刻表現型。

失敗モードと境界条件

  • time_le 関数が内部的にエラーを発生させた場合、例外がスローされます。
  • 入力ポインタが NULL の場合の挙動は未定義です。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける time 型の比較ロジックの一部です。同様のパターンが他の型（timestamp 等）のサポート関数でも使用されています。
  • DirectFunctionCall2 を使用しているため、比較ロジックはコアの time_le 実装に依存しています。
