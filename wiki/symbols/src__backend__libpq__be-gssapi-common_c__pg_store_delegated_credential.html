pg_store_delegated_credential

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-gssapi-common.c:103

目的

クライアントから委譲されたGSSAPI認証情報をプロセス固有のメモリキャッシュに保存し、後続の外部接続（postgres_fdw等）で利用可能にする。

シグネチャまたは形状

void pg_store_delegated_credential(gss_cred_id_t cred)

入力と出力

  • **入力**: cred (GSSAPI認証情報ハンドル)
  • **出力**: なし（成功時は環境変数 KRB5CCNAME を更新し、失敗時は pg_GSS_error を通じてログ出力を行う）

前提条件と不変条件

  • PostgreSQLのバックエンドプロセスモデル（forkベース）であることを前提とする。
  • KRB5CCNAME の設定はプロセス全体に影響するため、マルチスレッド環境での動作は想定していない。

アルゴリズムまたは振る舞い

1. GSS_MEMORY_CACHE を指定して gss_store_cred_into を呼び出し、認証情報をメモリキャッシュへ書き込む。 2. 成功後、gss_release_cred を呼び出して渡された認証情報ハンドルを解放する。 3. setenv("KRB5CCNAME", GSS_MEMORY_CACHE, 1) を実行し、以降の gss_acquire_cred 呼び出しがこのキャッシュを参照するように環境変数を更新する。 4. 各GSSAPI操作でエラーが発生した場合は pg_GSS_error を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth.c:1028
  • src/backend/libpq/be-secure-gssapi.c:629
  • **呼び出し先**:
  • gss_store_cred_into
  • gss_release_cred
  • pg_GSS_error
  • setenv

関連データ構造

  • gss_cred_id_t: 認証情報ハンドル
  • GSS_MEMORY_CACHE: メモリベースのキャッシュを指定するための定数文字列

失敗モードと境界条件

  • gss_store_cred_into または gss_release_cred が GSS_S_COMPLETE 以外のステータスを返した場合、pg_GSS_error を介してエラーが報告される。

保守担当者向けメモ

  • 本関数は COMMERROR レベルでログを出力する pg_GSS_error を利用しており、クライアントへのエラー送信による無限再帰を回避する設計となっている。
