register_ENR

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/queryenvironment.c:68

目的

指定された QueryEnvironment に、一時的な名前付きリレーション（Ephemeral Named Relation: ENR）を登録します。

シグネチャまたは形状

void register_ENR(QueryEnvironment *queryEnv, EphemeralNamedRelation enr)

入力と出力

  • **入力**:
  • queryEnv: ENR を管理する環境構造体へのポインタ。
  • enr: 登録対象の EphemeralNamedRelation 構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • enr は NULL であってはならない。
  • 同一の queryEnv 内に、登録しようとしている enr と同じ名前を持つ ENR が既に存在してはならない（get_ENR によるチェックで NULL が返る必要がある）。

アルゴリズムまたは振る舞い

1. enr が NULL でないことを確認する。 2. get_ENR を呼び出し、指定された環境内に同名の ENR が存在しないことを検証する。 3. lappend を使用して、queryEnv->namedRelList の末尾に新しい enr を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/spi.c:3317
  • **呼び出し先**:
  • get_ENR (存在確認のため)
  • lappend (リスト操作のため)

関連データ構造

  • QueryEnvironment: ENR を保持するコンテナ。
  • EphemeralNamedRelation: 登録されるリレーションの実体。

失敗モードと境界条件

  • enr が NULL の場合、または同名の ENR が既に登録されている場合、Assert によりプログラムが停止する。

保守担当者向けメモ

  • 現在の実装では List を使用した線形探索を行っているため、登録数が多い場合はパフォーマンスに影響する可能性がある。
  • table_open 等のロック関連の前提条件は、本関数を呼び出す側のコンテキストに依存する。
