pgsql_thing_t

定義

  • 種別: struct
  • 定義箇所: src/bin/psql/tab-complete.in.c:1254

目的

psql のタブ補完機能において、特定の入力文脈に対応する補完候補の生成ルールを定義する構造体です。

シグネチャまたは形状

typedef struct
{
	const char *name;
	const char *query;
	const VersionedQuery *vquery;
	const SchemaQuery *squery;
	const char *const *keywords;
	const bits32 flags;
} pgsql_thing_t;

入力と出力

  • **入力**: 構造体の各フィールドに、補完対象の名称、クエリ定義（query, vquery, squery のいずれか最大1つ）、キーワードリスト、および可視性フラグを指定します。
  • **出力**: 補完エンジンがこの構造体を参照し、データベースカタログやキーワードリストから補完候補を抽出します。

前提条件と不変条件

  • query, vquery, squery の3つのクエリ定義フィールドのうち、同時に設定できるのは最大で1つのみです。

アルゴリズムまたは振る舞い

補完エンジンは、ユーザーの入力パターンに応じて pgsql_thing_t の配列を走査し、一致する定義に基づいてバックエンドへのクエリ実行やキーワードの提示を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/tab-complete.in.c 内の words_after_create 配列などで静的に定義・参照されています。
  • **呼び出し先**: 補完ロジック内で wac などのポインタ変数を通じて参照されます。

関連データ構造

  • VersionedQuery: バージョン依存のクエリ定義。
  • SchemaQuery: スキーマオブジェクト補完用のクエリ定義。

失敗モードと境界条件

  • クエリ定義がすべて NULL の場合、補完候補としてキーワードのみが考慮されます。
  • 補完クエリの実行失敗はユーザーに通知されません。

保守担当者向けメモ

  • 本構造体は tab-complete.in.c で定義されており、ビルドプロセスで変換される可能性があります。
  • 補完候補の追加や修正を行う際は、query 系のフィールドの排他制御に注意してください。
