prepareToTerminate

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_recvlogical.c:1063

目的

論理レプリケーションストリーミングを終了する際、サーバーに対して COPY プロトコルの終了を通知し、終了理由に応じたログ出力を行う。

シグネチャまたは形状

static void prepareToTerminate(PGconn *conn, XLogRecPtr endpos, StreamStopReason reason, XLogRecPtr lsn)

入力と出力

  • conn: サーバーとの接続を保持する PGconn オブジェクト。
  • endpos: ストリーミング終了位置を示す XLogRecPtr。
  • reason: 終了理由を示す StreamStopReason 列挙型。
  • lsn: 終了に関連する WAL レコードの LSN。
  • 出力: なし（void）。

前提条件と不変条件

  • reason が STREAM_STOP_NONE であることは許可されない（Assert(false) により停止）。
  • STREAM_STOP_END_OF_WAL の場合、lsn は無効値であってはならない。

アルゴリズムまたは振る舞い

1. PQputCopyEnd を呼び出し、サーバーへ COPY モードの終了を通知する。 2. PQflush を呼び出し、送信バッファをフラッシュする。 3. verbose フラグが有効な場合、reason に基づいて以下の情報をログ出力する：

  • STREAM_STOP_SIGNAL: 割り込みシグナル受信。
  • STREAM_STOP_KEEPALIVE: endpos 到達（キープアライブによる）。
  • STREAM_STOP_END_OF_WAL: endpos 到達（WAL レコードによる）。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_recvlogical.c 内のストリーミング終了処理箇所（598行目付近）。
  • 呼び出し先: libpq の PQputCopyEnd および PQflush。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • XLogRecPtr: PostgreSQL の WAL 位置を示す型。
  • StreamStopReason: ストリーミング停止理由を定義する列挙型。

失敗モードと境界条件

  • PQputCopyEnd や PQflush の戻り値は無視される。
  • reason に STREAM_STOP_NONE が渡された場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数はストリーミングの正常終了および異常終了の共通処理として機能する。
  • ログ出力は verbose モード時のみ行われるため、デバッグ時にはこのフラグの状態に注意すること。
