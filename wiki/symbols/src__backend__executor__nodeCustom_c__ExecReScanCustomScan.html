ExecReScanCustomScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCustom.c:131

目的

カスタムスキャン実行ノードの再スキャン処理をトリガーし、拡張機能側で定義された再スキャンロジックを実行する。

シグネチャまたは形状

void ExecReScanCustomScan(CustomScanState *node)

入力と出力

  • **入力**: CustomScanState *node - 再スキャン対象となるカスタムスキャン実行状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node->methods が有効であり、かつ node->methods->ReScanCustomScan 関数ポインタが NULL でないこと（Assert により検証）。

アルゴリズムまたは振る舞い

1. node->methods->ReScanCustomScan が定義されていることを確認する。 2. 拡張機能が提供する ReScanCustomScan コールバック関数を呼び出し、カスタムスキャン固有の再スキャン処理（状態のリセットやポインタの巻き戻し等）を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:238
  • **呼び出し先**: node->methods->ReScanCustomScan（拡張機能側で実装された関数）

関連データ構造

  • CustomScanState: カスタムスキャンの実行状態を保持する構造体。
  • CustomExecMethods: 拡張機能が実装すべきコールバック関数テーブル。

失敗モードと境界条件

  • node->methods->ReScanCustomScan が NULL の場合、アサーション違反により異常終了する。

保守担当者向けメモ

  • 本関数は単なるディスパッチャであり、実際の再スキャンロジックは拡張機能側の実装に依存する。
  • 拡張機能が並列クエリをサポートしている場合、再スキャン時に共有メモリ上の状態も適切に同期・リセットされる必要がある。
