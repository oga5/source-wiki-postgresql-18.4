pq_sendint32

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/libpq/pqformat.h:143

目的

32ビット整数値をネットワークバイトオーダー（ビッグエンディアン）に変換し、通信バッファ StringInfo に追加します。

シグネチャまたは形状

static inline void pq_sendint32(StringInfo buf, uint32 i)

入力と出力

  • **入力**:
  • buf: 送信データを保持する StringInfo 構造体へのポインタ。
  • i: バッファに追加する uint32 型の整数値。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • buf は有効な StringInfo オブジェクトを指している必要があります。
  • 内部で enlargeStringInfo を呼び出すため、バッファのメモリ不足時には自動的に拡張が行われます。

アルゴリズムまたは振る舞い

1. enlargeStringInfo(buf, sizeof(uint32)) を呼び出し、バッファに4バイトの空き容量を確保します。 2. pq_writeint32(buf, i) を呼び出し、整数値をネットワークバイトオーダーでバッファの現在位置に書き込み、内部カーソルを進めます。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • enlargeStringInfo
  • pq_writeint32
  • **呼び出し元**:
  • contrib/cube/cube.c
  • contrib/hstore/hstore_io.c
  • src/backend/access/common/printsimple.c
  • src/backend/access/common/printtup.c
  • src/backend/commands/async.c

関連データ構造

  • StringInfo: 通信メッセージを構築するための動的バッファ構造体。

失敗モードと境界条件

  • メモリ不足時: enlargeStringInfo 内でメモリ割り当てに失敗した場合、PostgreSQLの標準的なエラー処理（elog(ERROR, ...) 等）により処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁な呼び出しによるオーバーヘッドを最小化しています。
  • 呼び出し前にバッファ容量が確保済みであることが保証されている場合は、直接 pq_writeint32 を使用することで最適化が可能です。
