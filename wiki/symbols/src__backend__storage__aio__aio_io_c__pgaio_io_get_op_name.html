pgaio_io_get_op_name

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_io.c:174

目的

PgAioHandle に格納された操作種別（op）を、デバッグやログ出力に適した人間が読める文字列に変換します。

シグネチャまたは形状

const char *pgaio_io_get_op_name(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh（操作状態を保持するハンドルへのポインタ）
  • **出力**: 操作名を表す const char * 文字列（"invalid", "readv", "writev" のいずれか、または NULL）

前提条件と不変条件

  • ioh->op は 0 以上 PGAIO_OP_COUNT 未満である必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. ioh->op が有効な範囲内にあるかアサーションで確認します。 2. ioh->op の値を PgAioOp 型にキャストし、switch 文で評価します。 3. 対応する操作種別に応じて以下の文字列を返します。

  • PGAIO_OP_INVALID: "invalid"
  • PGAIO_OP_READV: "readv"
  • PGAIO_OP_WRITEV: "writev"

    • 4. どのケースにも該当しない場合、コンパイラの警告を抑制するために NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio_funcs.c:175
  • src/include/storage/aio_internal.h:398（マクロ経由の参照）
  • **呼び出し先**:
  • 本関数は他の関数を呼び出しません。

関連データ構造

  • PgAioHandle: I/O操作のパラメータを保持する構造体。
  • PgAioOp: 操作種別を定義する列挙型。

失敗モードと境界条件

  • ioh が NULL の場合、デリファレンスによりクラッシュします。
  • ioh->op が定義された範囲外の値を持つ場合、Assert が有効なビルドではアボートし、無効なビルドでは NULL を返します。

保守担当者向けメモ

  • 新しいI/O操作タイプを PgAioOp に追加した際は、本関数の switch 文にも必ずケースを追加してください。
