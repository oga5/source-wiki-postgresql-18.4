dshash_attach

定義

  • 種別: function
  • 定義箇所: src/backend/lib/dshash.c:269

目的

既存の共有メモリ上の動的ハッシュテーブル（dshash）に、現在のバックエンドプロセスからアクセスするためのローカルな制御構造体を初期化・割り当てます。

シグネチャまたは形状

dshash_table *dshash_attach(dsa_area *area, const dshash_parameters *params, dshash_table_handle handle, void *arg);

入力と出力

  • **入力**:
  • area: ハッシュテーブルが配置されている dsa_area。
  • params: ハッシュテーブルの動作パラメータ。
  • handle: 共有メモリ上の制御オブジェクトを指す dshash_table_handle。
  • arg: 比較関数等で使用されるユーザー定義の引数。
  • **出力**:
  • 成功時: 初期化された dshash_table 構造体へのポインタを返します。

前提条件と不変条件

  • dsa_get_address により取得した control オブジェクトの magic フィールドが DSHASH_MAGIC と一致している必要があります。
  • 返される dshash_table の buckets は初期状態では NULL であり、size_log2 は 0 に設定されます。これらは後続の ensure_valid_bucket_pointers 呼び出し時に更新されます。

アルゴリズムまたは振る舞い

1. palloc を使用して、バックエンドローカルな dshash_table 構造体をメモリ上に確保します。 2. 引数で渡された handle を使用して、共有メモリ上の制御オブジェクトのアドレスを解決します。 3. 構造体の各フィールド（area, params, arg, control）を初期化します。 4. magic 値の検証を行い、整合性を確認します。 5. バケットポインタとサイズを初期値（NULL, 0）に設定し、構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/launcher.c
  • src/backend/storage/ipc/dsm_registry.c
  • src/backend/utils/activity/pgstat_shmem.c
  • src/backend/utils/cache/typcache.c
  • **呼び出し先**:
  • dsa_get_address (DSA領域からのアドレス解決)
  • palloc (メモリ確保)

関連データ構造

  • dshash_table: バックエンドローカルな状態保持構造体。
  • dshash_table_control: 共有メモリ上に配置される制御ブロック。
  • dsa_area: 動的共有メモリ領域。

失敗モードと境界条件

  • dsa_get_address が無効なハンドルを解決できない場合、または magic 値の検証に失敗した場合はアサーションエラーが発生します。

保守担当者向けメモ

  • 本関数はハッシュテーブルへの「接続」のみを行い、バケットポインタの同期は行いません。実際のバケットへのアクセスは、後続の操作関数内で ensure_valid_bucket_pointers が呼び出されることで安全に行われます。
