pg_hmac_create

定義

  • 種別: function
  • 定義箇所: src/common/hmac_openssl.c:116

目的

HMAC 計算のためのコンテキスト（pg_hmac_ctx）を生成および初期化し、OpenSSL の HMAC_CTX を割り当てます。

シグネチャまたは形状

pg_hmac_ctx *pg_hmac_create(pg_cryptohash_type type)

入力と出力

  • **入力**: type (使用するハッシュアルゴリズムの種類)
  • **出力**: 成功時は初期化された pg_hmac_ctx へのポインタ、失敗時は NULL

前提条件と不変条件

  • OpenSSL のエラーキューをクリアしてから処理を開始します。
  • バックエンド環境（FRONTEND が未定義）では、メモリ割り当て失敗時に ereport(ERROR, ...) を発行します。
  • USE_RESOWNER_FOR_HMAC が定義されている場合、現在の ResourceOwner にコンテキストを登録します。

アルゴリズムまたは振る舞い

1. pg_hmac_ctx 構造体用のメモリを割り当て、ゼロクリアします。 2. 指定された type をコンテキストに保存し、エラー状態を初期化します。 3. ERR_clear_error() を呼び出し、OpenSSL のエラーキューをクリアします。 4. HMAC_CTX_new() を呼び出し、OpenSSL の HMAC コンテキストを生成します。 5. 生成に失敗した場合は、割り当てたメモリを解放して NULL を返します。 6. バックエンド環境であれば、ResourceOwnerRememberHMAC を使用してリソース所有者にコンテキストを紐付けます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c
  • src/common/scram-common.c
  • src/interfaces/libpq/fe-auth-scram.c
  • **呼び出し先**:
  • ALLOC / FREE (メモリ管理マクロ)
  • ERR_clear_error (OpenSSL)
  • HMAC_CTX_new (OpenSSL)
  • ResourceOwnerEnlarge / ResourceOwnerRememberHMAC (バックエンド環境のみ)

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する構造体。
  • HMAC_CTX: OpenSSL が提供する HMAC 計算用コンテキスト。

失敗モードと境界条件

  • メモリ不足: ALLOC が NULL を返した場合、または HMAC_CTX_new が NULL を返した場合は NULL を返します。
  • バックエンド環境では、メモリ不足時に ereport を通じてエラーが通知されます。

保守担当者向けメモ

  • pg_hmac_free で確実にリソースが解放されるよう設計されています。
  • バックエンド環境でのメモリリークを防ぐため、ResourceOwner への登録が必須となっています。
