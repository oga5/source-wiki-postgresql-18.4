free_stringlist

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress.c:217

目的

_stringlist 型の連結リスト構造体およびその内部の文字列データをメモリから解放し、リストのヘッドポインタを NULL にリセットする。

シグネチャまたは形状

static void free_stringlist(_stringlist **listhead)

入力と出力

  • **入力**: _stringlist **listhead （解放対象のリストの先頭ポインタへのポインタ）
  • **出力**: なし（副作用としてメモリ解放とポインタの更新を行う）

前提条件と不変条件

  • listhead が NULL であるか、または *listhead が NULL である場合、関数は何もせず即座に復帰する。
  • リストは next ポインタによって連結されている必要がある。

アルゴリズムまたは振る舞い

1. リストの終端まで再帰的に辿る（next ポインタが存在する場合、そのノードに対して自身を呼び出す）。 2. 再帰の巻き戻り過程で、各ノードの str メンバ（文字列）を free() する。 3. 各ノード自体を free() する。 4. 最後に、呼び出し元で渡された listhead を NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/regress/pg_regress.c 内の各所（resultfiles[i], expectfiles[i], tags[i], dblist の解放時）
  • **呼び出し先**:
  • free() (標準ライブラリ)
  • free_stringlist (再帰呼び出し)

関連データ構造

  • _stringlist: リストの各ノードを構成する構造体（str メンバと next ポインタを持つ）。

失敗モードと境界条件

  • listhead が NULL の場合、安全に無視される。
  • リストが空（*listhead == NULL）の場合、安全に無視される。
  • メモリ解放後のポインタを NULL に設定することで、二重解放を防止する設計となっている。

保守担当者向けメモ

  • 再帰的な解放を行っているため、リストが極端に長い場合はスタックオーバーフローのリスクを考慮する必要がある。
  • _stringlist 構造体の定義は pg_regress.h を参照すること。
