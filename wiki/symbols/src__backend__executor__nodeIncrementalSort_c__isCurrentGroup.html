isCurrentGroup

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:211

目的

入力タプルが現在のソートグループ（接頭辞キーが一致する範囲）に属しているかを判定し、グループの境界を検出する。

シグネチャまたは形状

static bool isCurrentGroup(IncrementalSortState *node, TupleTableSlot *pivot, TupleTableSlot *tuple)

入力と出力

  • **入力**:
  • node: IncrementalSortState 構造体（ソート状態およびキー情報を含む）。
  • pivot: 比較基準となる現在のグループの代表タプル。
  • tuple: 判定対象の入力タプル。
  • **出力**:
  • bool: 両タプルの接頭辞キーが一致すれば true、不一致であれば false を返す。

前提条件と不変条件

  • node->presorted_keys に有効なソートキー情報が格納されていること。
  • pivot および tuple は、node が扱うタプル形式と互換性があること。

アルゴリズムまたは振る舞い

1. nPresortedCols（ソートキーの接頭辞数）を取得する。 2. 接頭辞キーの末尾から先頭に向かって（nPresortedCols - 1 から 0 まで）順に比較を行う。これは、末尾のキーほど変化しやすく、不一致を早期に検出して関数呼び出し回数を最小化するためである。 3. 各カラムについて slot_getattr で値を抽出し、以下のルールで比較する：

  • 両方が NULL の場合は一致とみなし、次のキーへ進む。
  • 片方のみが NULL の場合は不一致として false を返す。
  • それ以外の場合は、key->fcinfo を用いて比較関数を呼び出し、結果が true（一致）でなければ false を返す。

    • 4. すべての接頭辞キーが一致した場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeIncrementalSort.c 内の ExecIncrementalSort 等の関数から、グループ継続の確認のために複数箇所で呼び出されている。
  • **呼び出し先**:
  • slot_getattr
  • FunctionCallInvoke

関連データ構造

  • IncrementalSortState: ノードの実行状態を保持する構造体。
  • PresortedKeyData: 各ソートキーの比較関数や属性番号を保持する構造体。
  • TupleTableSlot: 比較対象のタプルを保持するコンテナ。

失敗モードと境界条件

  • 比較関数が NULL を返した場合、elog(ERROR) によりエラー終了する（呼び出し側が NULL を想定していないため）。
  • nPresortedCols が 0 の場合、ループは実行されず常に true を返す。

保守担当者向けメモ

  • 本関数は比較コストを最小化するために逆順走査を行っている。キーの順序や比較ロジックを変更する際は、この最適化が維持されているか確認すること。
  • EXEC_FLAG_BACKWARD や EXEC_FLAG_MARK はサポートされていないため、本関数は前方へのストリーミング処理のみを前提としている。
