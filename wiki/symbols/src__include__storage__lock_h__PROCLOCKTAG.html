PROCLOCKTAG

定義

  • 種別: struct
  • 定義箇所: src/include/storage/lock.h:363

目的

特定のロック対象（LOCK）と、そのロックを保持または待機しているバックエンドプロセス（PGPROC）の関連付けを一意に識別するためのタグ構造体です。

シグネチャまたは形状

typedef struct PROCLOCKTAG
{
    LOCK       *myLock;
    PGPROC     *myProc;
} PROCLOCKTAG;

入力と出力

  • **入力**: LOCK オブジェクトへのポインタおよび PGPROC 構造体へのポインタ。
  • **出力**: ロックマネージャのハッシュテーブルにおいて、特定のプロセスとロックの組み合わせを検索するためのキーとして機能します。

前提条件と不変条件

  • 構造体内にパディングが含まれないことを前提としています。
  • ハッシュテーブルのキーとして使用されるため、この構造体のハッシュ値は一貫性を保つ必要があります。

アルゴリズムまたは振る舞い

  • ロックマネージャは、このタグを使用して共有メモリ上の PROCLOCK オブジェクトを検索・管理します。
  • src/backend/storage/lmgr/lock.c 内の ProcLockHashCode 関数により、このタグからハッシュコードが計算されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/lock.c 内のロック取得・解放処理（LockAcquire 等）において、ローカル変数として頻繁にインスタンス化されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • LOCK: ロック対象オブジェクト。
  • PGPROC: バックエンドプロセス情報。
  • PROCLOCK: このタグをキーとして管理される、プロセスとロックの関連付けを保持する実体構造体。

失敗モードと境界条件

  • 構造体サイズが sizeof(PROCLOCKTAG) と一致しない場合、ハッシュテーブルの検索や挿入で不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は16バイト境界やパディングの影響を受けやすいため、フィールドの追加や変更を行う際は、メモリレイアウトがハッシュ計算に与える影響を十分に考慮してください。
