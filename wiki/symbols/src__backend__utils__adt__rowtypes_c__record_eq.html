record_eq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rowtypes.c:1066

目的

2つのレコード（HeapTupleHeader）が等価であるかを判定します。

シグネチャまたは形状

Datum record_eq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの HeapTupleHeader を受け取ります。
  • **出力**: bool 型の等価判定結果を Datum として返します。

前提条件と不変条件

  • check_stack_depth() を呼び出し、再帰的なレコード比較によるスタックオーバーフローを防止します。
  • 比較対象のレコードは lookup_rowtype_tupdesc を通じて型定義（TupleDesc）が取得可能である必要があります。
  • 比較する列の型が一致しない場合、ERRCODE_DATATYPE_MISMATCH でエラーが発生します。

アルゴリズムまたは振る舞い

1. 各レコードの TupleDesc を取得し、一時的な HeapTupleData 構造体を構築します。 2. fcinfo->flinfo->fn_extra を使用して、比較に必要な型情報や関数ポインタをキャッシュします。 3. heap_deform_tuple を用いてタプルをフィールド配列に分解します。 4. 物理的な列順序を走査し、削除された列（attisdropped）をスキップしながら論理的な列ごとに比較を行います。 5. 各フィールドの型に対応する等価演算子を lookup_type_cache で取得し、再帰的に呼び出します。 6. NULL値の扱い: 両方がNULLの場合は等価とみなし、片方のみがNULLの場合は不等と判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rowtypes.c:1058 (内部的な参照)
  • src/backend/utils/adt/rowtypes.c:1285 (record_ne の実装内)
  • **呼び出し先**:
  • lookup_rowtype_tupdesc
  • heap_deform_tuple
  • lookup_type_cache
  • FunctionCallInvoke (各フィールドの等価演算子実行)

関連データ構造

  • RecordCompareData: 比較実行時の型情報や関数ポインタを保持するキャッシュ構造体。
  • TupleDesc: レコードの構造定義。
  • HeapTupleHeader: レコードの物理的なデータ表現。

失敗モードと境界条件

  • **型不一致**: 比較対象の列の型が異なる場合、エラーをスローします。
  • **演算子欠如**: フィールドの型に対して等価演算子が見つからない場合、ERRCODE_UNDEFINED_FUNCTION でエラーをスローします。
  • **NULLの扱い**: NULLを含むレコード同士の比較において、NULLは「値が存在しない」ものとして扱われ、NULL同士は等価と判定されます。

保守担当者向けメモ

  • 本関数は再帰的に呼び出される可能性があるため、スタック深さのチェックが必須です。
  • fn_extra キャッシュは、レコードの型定義が実行中に変更されないことを前提としています。
  • 削除された列（attisdropped）は比較対象から除外されるため、物理的なインデックスではなく論理的な列順序で対応付けを行うロジックを維持してください。
