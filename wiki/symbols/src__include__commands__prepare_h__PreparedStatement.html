PreparedStatement

定義

  • 種別: struct
  • 定義箇所: src/include/commands/prepare.h:28

目的

SQLレベルまたはFE/BEプロトコル経由で作成されたプリペアドステートメントのメタデータと実行計画を保持し、セッション内で一元管理するために使用されます。

シグネチャまたは形状

typedef struct {
    char        stmt_name[NAMEDATALEN];
    CachedPlanSource *plansource;
    bool        from_sql;
    TimestampTz prepare_time;
} PreparedStatement;

入力と出力

  • **入力**: 構造体生成時に stmt_name（識別子）、plansource（実行計画）、from_sql（作成元フラグ）、prepare_time（作成時刻）が設定されます。
  • **出力**: src/backend/commands/prepare.c 内の関数群（FetchPreparedStatementResultDesc 等）を通じて、保持している計画情報やメタデータが提供されます。

前提条件と不変条件

  • dynahash.c の仕様により、stmt_name フィールドが構造体の先頭に配置されている必要があります。
  • 各プリペアドステートメントはバックエンド（セッション）ローカルであり、他のバックエンドとは共有されません。

アルゴリズムまたは振る舞い

  • PREPARE コマンド実行時に StorePreparedStatement を介してハッシュテーブルに登録されます。
  • EXECUTE コマンド実行時に FetchPreparedStatement を介して取得され、plansource を用いてクエリが実行されます。
  • DEALLOCATE コマンド実行時に DropPreparedStatement を介してハッシュテーブルから削除され、メモリが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/prepare.c 内の EvaluateParams、FetchPreparedStatementResultDesc、FetchPreparedStatementTargetList 等がこの構造体を参照しています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • CachedPlanSource: 実行計画のキャッシュ本体を保持します。
  • prepared_queries: src/backend/commands/prepare.c 内で定義される、本構造体を格納するハッシュテーブル。

失敗モードと境界条件

  • stmt_name は NAMEDATALEN の長さに制限されます。
  • ハッシュテーブル操作時にエントリが見つからない場合、hash_search の結果として NULL が返される可能性があります。

保守担当者向けメモ

  • from_sql フラグは、SQL コマンドによる作成か、FE/BE プロトコルの Parse メッセージによる作成かを区別するために重要です。
  • 構造体のメモリレイアウトを変更する場合は、dynahash.c の制約（キーが先頭であること）を維持してください。
