prep_status_progress

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/util.c:155

目的

長時間実行される処理の開始時に、ユーザーに対して進捗表示のヘッダーとなるメッセージを出力し、表示を初期化します。

シグネチャまたは形状

void prep_status_progress(const char *fmt, ...)

入力と出力

  • **入力**: fmt（書式指定文字列）、および可変長引数。
  • **出力**: なし（戻り値は void）。副作用として標準出力またはログファイルへメッセージを出力します。

前提条件と不変条件

  • MAX_STRING サイズ以内のメッセージであること。
  • log_opts グローバル変数が初期化されていること。

アルゴリズムまたは振る舞い

1. 可変長引数を vsnprintf で message バッファに展開します。 2. log_opts.isatty または log_opts.verbose が真の場合、PG_REPORT 型でメッセージを出力します。それ以外の場合は PG_REPORT_NONL 型で出力します。 3. いずれの場合も、MESSAGE_WIDTH に基づいてメッセージをパディング（右詰め）し、後続の進捗項目が整列されるようにします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/dump.c:33
  • src/bin/pg_upgrade/pg_upgrade.c:577, 979
  • src/bin/pg_upgrade/relfilenumber.c:114, 117, 120, 123, 126
  • **呼び出し先**:
  • pg_log()（src/bin/pg_upgrade/util.c 内）

関連データ構造

  • LogOpts（log_opts 変数の型）: ログ出力設定を保持。
  • eLogType（列挙型）: PG_REPORT および PG_REPORT_NONL を定義。

失敗モードと境界条件

  • vsnprintf による文字列切り捨てが発生する可能性があります。
  • MESSAGE_WIDTH を超えるメッセージはパディングされず、そのまま出力されます。

保守担当者向けメモ

  • 本関数はメッセージの末尾に改行を付与するかどうかを log_opts に基づいて決定します。
  • 後続の進捗更新は pg_log(PG_STATUS, ...) を通じて行い、最後に end_progress_output() で行をクリアする一連の流れを前提としています。
