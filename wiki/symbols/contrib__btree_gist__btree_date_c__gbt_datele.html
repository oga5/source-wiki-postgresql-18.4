gbt_datele

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_date.c:54

目的

PostgreSQL の date 型に対する「以下（less than or equal to）」比較を GiST インデックスのコンテキストで実行する。

シグネチャまたは形状

static bool gbt_datele(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の DateADT 値を指すポインタ。
  • flinfo: 関数マネージャ情報（DirectFunctionCall2 で使用）。
  • **出力**:
  • a <= b であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な DateADT 型の値を指している必要がある。
  • 内部的に PostgreSQL コアの date_le 関数を呼び出すため、その実行環境および型定義に依存する。

アルゴリズムまたは振る舞い

1. a および b が指す DateADT 値を DateADTGetDatum を用いて Datum 型に変換する。 2. DirectFunctionCall2 を介して PostgreSQL コアの date_le 関数を呼び出し、比較を実行する。 3. 結果を DatumGetBool で bool 型に変換して返却する。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • date_le (PostgreSQL コア関数)
  • **呼び出し元**:
  • contrib/btree_gist/btree_date.c:110 (構造体初期化時の関数ポインタとして参照されていることが示唆される)

関連データ構造

  • DateADT: PostgreSQL の内部日付型。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、メモリ保護違反が発生する可能性がある。
  • date_le 関数が内部的にエラーを発生させた場合、本関数もそのエラーを伝播させる。

保守担当者向けメモ

  • 本関数は btree_gist における date 型の比較ロジックのラッパーである。
  • 比較ロジックの変更が必要な場合は、コアの date_le の仕様を確認すること。
