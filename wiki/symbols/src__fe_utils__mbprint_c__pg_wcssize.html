pg_wcssize

定義

  • 種別: function
  • 定義箇所: src/fe_utils/mbprint.c:210

目的

マルチバイト文字列を端末へ表示する際に必要な、表示幅、高さ、およびエスケープ処理後のバッファサイズを事前に計算します。

シグネチャまたは形状

void pg_wcssize(const unsigned char *pwcs, size_t len, int encoding,
                int *result_width, int *result_height, int *result_format_size)

入力と出力

  • **入力**:
  • pwcs: 対象の文字列（バイト列）。
  • len: 文字列の長さ。
  • encoding: 文字エンコーディングID。
  • **出力**:
  • result_width: 端末上での最大表示カラム数（ポインタ経由）。
  • result_height: 改行を含む行数（ポインタ経由）。
  • result_format_size: エスケープ処理後に必要なメモリサイズ（ポインタ経由）。

前提条件と不変条件

  • pg_wcsformat との同期: 本関数で計算したサイズは、pg_wcsformat による整形処理と完全に一致している必要があります。ロジック変更時は両方を同期させる必要があります。
  • PQmblen および PQdsplen が指定された encoding に対して正しく動作すること。

アルゴリズムまたは振る舞い

1. 文字列を走査し、PQmblen で文字境界を、PQdsplen で表示幅を特定します。 2. シングルバイト文字の場合:

  • \n: 行数を加算し、現在の行幅をリセット。
  • \r: 表示幅を2加算。
  • \t: 8カラム境界まで表示幅を拡張。
  • 制御文字（w < 0）: 表示幅を4加算。
  • その他: 表示幅をそのまま加算。

    • 3. マルチバイト文字の場合:

  • 制御文字（w < 0）: 表示幅を6加算（\uXXXX 形式）。
  • その他: PQdsplen の結果を加算。

    • 4. 最後に、終端の NUL 文字分として format_size に 1 を加算します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/print.c 内の各出力関数（テーブルヘッダーやデータ表示時）。
  • **呼び出し先**: libpq が提供する PQmblen および PQdsplen。

関連データ構造

  • src/include/fe_utils/mbprint.h: 本関数の宣言および関連する lineptr 構造体が定義されています。

失敗モードと境界条件

  • len が文字のバイト長より小さい場合、その文字の処理を中断します。
  • result_width 等のポインタが NULL の場合、その値の書き込みはスキップされます。

保守担当者向けメモ

  • 本関数は pg_wchar.h を直接インクルードしていません。これはエンコーディングIDの不一致を避けるための設計上の制約です。
  • UTF-8 以外のエンコーディングで制御文字の扱いを拡張する場合は、mbvalidate 等の関連関数との整合性に注意してください。
