ExecSerializePlan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execParallel.c:145

目的

並列クエリ実行において、リーダープロセスからワーカープロセスへプランを転送するために、実行プラン（Plan）をシリアライズ可能な形式（PlannedStmt）へ変換する。

シグネチャまたは形状

static char *ExecSerializePlan(Plan *plan, EState *estate)

入力と出力

  • **入力**:
  • plan: シリアライズ対象のプランツリー。
  • estate: 現在の実行状態（EState）。プランの再構築に必要な範囲テーブルやパーティション情報等を取得するために使用。
  • **出力**:
  • char *: nodeToString によって文字列化された PlannedStmt の表現。

前提条件と不変条件

  • 入力された plan は copyObject により複製されるため、元のプランツリーは変更されない。
  • ワーカー側で実行される際、不要なジャンクフィルタが挿入されないよう、ターゲットリスト内の全 TargetEntry の resjunk フラグが false に強制設定される。
  • 並列実行において安全でないサブプラン（subplan->parallel_safe が偽）は、シリアライズ時に NULL に置換され、ワーカーが実行しないよう保護される。

アルゴリズムまたは振る舞い

1. plan のコピーを作成する。 2. ターゲットリストを走査し、すべての resjunk を false に設定する。 3. ダミーの PlannedStmt を作成し、EState から必要なメタデータ（範囲テーブル、パーティション情報等）をコピーする。 4. estate->es_plannedstmt->subplans を走査し、並列安全なサブプランのみを抽出して PlannedStmt に格納する。安全でないものは NULL で埋める。 5. 最終的な PlannedStmt 構造体を nodeToString で文字列化して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c:643（ExecInitParallelPlan 内での呼び出しが確認されている）
  • **呼び出し先**:
  • copyObject
  • makeNode
  • pgstat_get_my_query_id / pgstat_get_my_plan_id
  • nodeToString

関連データ構造

  • Plan: 実行プランのノード。
  • EState: クエリ実行状態。
  • PlannedStmt: 実行エンジンがプランを実行するために必要な情報を保持する構造体。

失敗モードと境界条件

  • subplans のリストにおいて、並列安全でないノードは NULL に置換されるため、ワーカー側で ExecInitNode が呼び出されないよう制御されている。
  • resjunk の強制無効化は、ワーカー側でタプルを再利用する際に必要な情報を保持するためのハック的な実装である。

保守担当者向けメモ

  • 本関数はプランツリーを再帰的に走査・複製するため、新しいプランノードタイプを追加する際は copyObject が正しく動作することを確認すること。
  • resjunk の扱いについては、将来的に別の設計手法が検討される可能性がある旨がソースコード内に示唆されている。
