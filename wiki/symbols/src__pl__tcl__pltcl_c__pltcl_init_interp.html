pltcl_init_interp

定義

  • 種別: function
  • 定義箇所: src/pl/tcl/pltcl.c:493

目的

ユーザーごとの新しい Tcl インタプリタを生成し、PostgreSQL の SPI（Server Programming Interface）機能と連携するためのコマンド群をインストールして初期化する。

シグネチャまたは形状

static void pltcl_init_interp(pltcl_interp_desc *interp_desc, Oid prolang, bool pltrusted)

入力と出力

  • **入力**:
  • interp_desc: インタプリタの状態を保持する構造体。
  • prolang: 言語の OID。
  • pltrusted: 信頼された言語（pltcl）か否かを示すフラグ。
  • **出力**: なし（成功時は interp_desc を更新、失敗時は elog によりエラーをスロー）。

前提条件と不変条件

  • pltcl_hold_interp が既に存在している必要がある。
  • pltrusted が true の場合、Tcl の Tcl_Init は実行されない。

アルゴリズムまたは振る舞い

1. Tcl_CreateSlave を使用して、ユーザー ID に基づく名前のサブインタプリタを作成する。 2. Tcl_InitHashTable を呼び出し、interp_desc 内のクエリハッシュテーブルを初期化する。 3. Tcl_CreateObjCommand を使用し、SPI 操作（spi_exec, spi_prepare 等）やユーティリティ（elog, quote 等）を Tcl インタプリタに登録する。 4. PG_TRY ブロック内で call_pltcl_start_proc を呼び出し、開始プロシージャを実行する。 5. 実行中にエラーが発生した場合、PG_CATCH ブロックでインタプリタを破棄し、例外を再スローする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/tcl/pltcl.c 内のインタプリタ管理ロジックから呼び出される。
  • **呼び出し先**: Tcl_CreateSlave, Tcl_InitHashTable, Tcl_CreateObjCommand, call_pltcl_start_proc, Tcl_DeleteInterp 等の Tcl API および内部関数。

関連データ構造

  • pltcl_interp_desc: インタプリタの保持およびクエリハッシュテーブルの管理に使用。

失敗モードと境界条件

  • インタプリタの生成に失敗した場合、elog(ERROR, ...) によりバックエンドが停止する。
  • start_proc の実行に失敗した場合、インタプリタは破棄され、エラーが伝播される。

保守担当者向けメモ

  • 本関数は pltcl_hold_interp を親とするスレーブインタプリタを作成する。
  • SPI コマンドの追加は、Tcl 側から PostgreSQL の機能へアクセスするための重要なブリッジである。
  • PG_TRY/CATCH ブロックは、インタプリタの初期化失敗時にメモリリークや不正な状態を残さないために必須である。
