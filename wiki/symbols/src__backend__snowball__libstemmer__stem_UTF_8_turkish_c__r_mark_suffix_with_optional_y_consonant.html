目的

トルコ語のステミング処理において、単語末尾の接尾辞を識別し、必要に応じて「y」子音の有無を考慮したマッチングを行う。

シグネチャまたは形状

static int r_mark_suffix_with_optional_y_consonant(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowball ランタイム環境構造体。現在のカーソル位置 z->c や文字列バッファ z->p を含む）
  • 出力: int（マッチングに成功した場合は 1、失敗した場合は 0 を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • z->c（カーソル）は処理対象の単語末尾付近を指していること。

アルゴリズムまたは振る舞い

1. **「y」の確認と母音調和の検証**:

  • カーソルの直前が「y」であるかを確認する。
  • 「y」が存在する場合、その直前が母音（g_vowel）であるかを確認し、条件を満たせば成功（1）として終了する。

    • 2. **代替パス（「y」がない場合）**:

  • 「y」がない場合、あるいは「y」があっても母音条件を満たさない場合、カーソル位置を復元し、別のパターンを試行する。
  • skip_b_utf8 を用いて文字を後方にスキップし、その直前が母音であるかを確認する。

    • 3. **終了**:

  • いずれの条件も満たさない場合は 0 を返し、マッチング失敗とする。

失敗モードと境界条件

  • カーソルがバッファの先頭（z->lb）に達している場合、マッチングは失敗する。
  • skip_b_utf8 が負の値を返した場合（不正なUTF-8シーケンスや境界外へのアクセス）、関数は直ちに 0 を返す。
