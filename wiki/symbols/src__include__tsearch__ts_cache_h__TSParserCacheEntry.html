TSParserCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_cache.h:30

目的

全文検索（Tsearch）で使用するパーサーのメタデータおよび実行用関数情報をメモリ上に保持し、システムカタログ（pg_ts_parser）へのアクセスを最小化して高速化を図るためのキャッシュエントリ構造体です。

シグネチャまたは形状

typedef struct TSParserCacheEntry {
    Oid         prsId;          /* ハッシュキー（先頭必須） */
    bool        isvalid;        /* キャッシュ有効性フラグ */
    Oid         startOid;
    Oid         tokenOid;
    Oid         endOid;
    Oid         headlineOid;
    Oid         lextypeOid;
    FmgrInfo    prsstart;
    FmgrInfo    prstoken;
    FmgrInfo    prsend;
    FmgrInfo    prsheadline;
} TSParserCacheEntry;

入力と出力

  • **入力**: キャッシュ構築時にシステムカタログから読み込まれたパーサーのOIDおよび各メソッドのOID。
  • **出力**: FmgrInfo 構造体を通じて、パーサーの各メソッド（start, token, end, headline）を直接呼び出し可能な状態として提供します。

前提条件と不変条件

  • prsId は構造体の最初のメンバである必要があります（ハッシュテーブルのキーとして使用されるため）。
  • isvalid が false の場合、キャッシュは無効とみなされ、再構築が必要です。
  • FmgrInfo メンバは、対応するパーサーメソッドを高速に実行するために事前設定（セットアップ）されている必要があります。

アルゴリズムまたは振る舞い

  • キャッシュミス発生時、システムカタログから情報を読み込み、この構造体に格納します。
  • InvalidateTSCacheCallBack が呼び出されると、関連するキャッシュエントリの isvalid フラグが false に設定され、次回アクセス時に再構築が促されます。
  • FmgrInfo を保持することで、パーサーのメソッド呼び出し時にカタログ検索を介さず、直接関数ポインタ経由で実行可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: lookup_ts_parser_cache 関数（src/backend/utils/cache/ts_cache.c）により取得されます。また、src/backend/tsearch/ts_parse.c や src/backend/tsearch/wparser.c 等のパーサー処理モジュールから参照されます。
  • **呼び出し先**: FmgrInfo メンバを介して、パーサーが定義する各メソッド（start, token, end 等）が呼び出されます。

関連データ構造

  • TSAnyCacheEntry: 共通のキャッシュヘッダー構造。
  • FmgrInfo: fmgr.h で定義される関数呼び出しインターフェース。

失敗モードと境界条件

  • isvalid フラグの管理不整合により、古いキャッシュ情報が使用される可能性があります（無効化コールバックの実行順序やタイミングに依存）。
  • メモリ管理上の制約として、キャッシュエントリはバックエンドの生存期間中保持されるため、再構築時のメモリリークに注意が必要です。

保守担当者向けメモ

  • 構造体の先頭メンバ（prsId）の順序は、ハッシュテーブルの汎用的なアクセスを保証するために厳守してください。
  • パーサーメソッドの追加や変更を行う際は、FmgrInfo の初期化ロジックが ts_cache.c 内で正しく更新されているか確認してください。
