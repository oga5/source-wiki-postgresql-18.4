brin_inclusion_add_value

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_inclusion.c:137

目的

BRINインデックスの包含（Inclusion）演算子クラスにおいて、新しい値をブロック範囲の集約値（Union）に統合し、インデックスの状態を更新する。

シグネチャまたは形状

Datum brin_inclusion_add_value(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): BrinDesc *（インデックス記述子）
  • PG_GETARG_POINTER(1): BrinValues *（更新対象のブロック範囲値）
  • PG_GETARG_DATUM(2): Datum（追加する新しい値）
  • PG_GETARG_BOOL(3): bool（値がNULLかどうかのフラグ）
  • **出力**:
  • Datum（bool型として、インデックスが変更された場合は true、変更不要の場合は false を返す）

前提条件と不変条件

  • 入力値がNULLであることは許容されない（Assert(!isnull)）。
  • INCLUSION_UNMERGEABLE フラグが立っている場合、それ以上の統合は行われない。
  • bv_allnulls が真の場合、新しい値で初期化される。

アルゴリズムまたは振る舞い

1. **初期化**: bv_allnulls が真であれば、新しい値をコピーして初期値とし、フラグを更新して終了する。 2. **結合不可チェック**: INCLUSION_UNMERGEABLE が真であれば、何もせず false を返す。 3. **空要素判定**: PROCNUM_EMPTY 関数を呼び出し、値が空と判定された場合は INCLUSION_CONTAINS_EMPTY フラグを更新して true を返す。 4. **包含判定**: PROCNUM_CONTAINS 関数を呼び出し、新しい値が既存の INCLUSION_UNION に含まれている場合は false を返す。 5. **マージ可能性判定**: PROCNUM_MERGEABLE 関数を呼び出し、結合不可能と判定された場合は INCLUSION_UNMERGEABLE を真に設定し true を返す。 6. **統合**: PROCNUM_MERGE 関数を呼び出し、既存の INCLUSION_UNION を新しい値で更新する。メモリ管理が必要な場合は古い値を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は明示されていない（BRINフレームワークから動的に呼び出される）。
  • **呼び出し先**:
  • inclusion_get_procinfo（演算子情報の取得）
  • FunctionCall1Coll / FunctionCall2Coll（各演算子関数の実行）
  • datumCopy / pfree（メモリ管理）

関連データ構造

  • BrinDesc: インデックスのメタデータ。
  • BrinValues: ブロック範囲ごとの集約値を保持する構造体。
  • InclusionOpaque: インデックス列のキャッシュ情報。

失敗モードと境界条件

  • INCLUSION_UNMERGEABLE 状態に達した後は、以降の追加操作は無視される。
  • メモリ管理が必要なデータ型において、PROCNUM_MERGE の結果が新しいメモリ領域を確保する場合、古い INCLUSION_UNION のメモリは適切に解放される。

保守担当者向けメモ

  • PROCNUM_MERGE の結果が既存のポインタと異なる場合のメモリ解放ロジックに注意すること。
  • INCLUSION_UNMERGEABLE になった後も値が保持され続ける点は、将来的な最適化の余地がある。
