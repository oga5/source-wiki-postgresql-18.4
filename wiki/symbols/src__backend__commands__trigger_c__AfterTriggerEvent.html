AfterTriggerEvent

定義

  • 種別: struct (へのポインタ型)
  • 定義箇所: src/backend/commands/trigger.c:3705

目的

トランザクション内で発生した AFTER トリガーの実行イベントをキューイングおよび管理するためのデータ構造です。

シグネチャまたは形状

typedef struct AfterTriggerEventData *AfterTriggerEvent;

※ AfterTriggerEventData 構造体へのポインタとして定義されています。

入力と出力

  • 入力: AfterTriggerSaveEvent 等の関数を通じて、トリガー実行に必要なコンテキスト情報が保存されます。
  • 出力: トランザクション終了時や SET CONSTRAINTS 実行時に、キューから取り出されトリガー実行エンジンへ渡されます。

前提条件と不変条件

  • AfterTriggerEvent は、メモリチャンク内に連続して配置されることが想定されています（CHUNK_DATA_START による走査）。
  • 各イベントのサイズは固定ではなく、SizeofTriggerEvent(event) によって動的に算出されます。

アルゴリズムまたは振る舞い

  • src/backend/commands/trigger.c 内のキュー走査ロジックにおいて、eptr = (AfterTriggerEvent) (((char *) eptr) + SizeofTriggerEvent(eptr)) のように、現在のイベントサイズに基づいて次のイベントへポインタを進めることで、メモリ上のイベントリストを順次処理します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/trigger.c 内の AFTER トリガー管理関数群（AfterTriggerSaveEvent 等）。
  • 呼び出し先: 具体的な関数呼び出し関係はソース抜粋からは不明ですが、トリガー実行エンジンが本構造体を参照してトリガー関数を起動します。

関連データ構造

  • AfterTriggerEventData: 実体となる構造体。
  • AfterTriggerShared: イベントに関連する共有状態。
  • AfterTriggerEventDataNoOids, AfterTriggerEventDataOneCtid, AfterTriggerEventDataZeroCtids: イベントデータのバリエーション。

失敗モードと境界条件

  • メモリ上のイベント走査時、SizeofTriggerEvent の計算が誤っている場合、メモリ破壊や不正なメモリアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは AfterTriggerEventData へのポインタであり、実体は trigger.c 内で動的にメモリ確保・管理されています。
  • イベントの追加や削除を行う際は、SizeofTriggerEvent の整合性が保たれていることを確認してください。
