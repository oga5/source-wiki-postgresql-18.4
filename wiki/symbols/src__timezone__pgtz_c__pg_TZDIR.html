pg_TZDIR

定義

  • 種別: function
  • 定義箇所: src/timezone/pgtz.c:42

目的

PostgreSQLが利用するタイムゾーンデータベース（tzdb）のディレクトリパスを取得します。

シグネチャまたは形状

static const char *pg_TZDIR(void);

入力と出力

  • **入力**: なし
  • **出力**: タイムゾーンディレクトリの絶対パスを指す文字列ポインタを返します。

前提条件と不変条件

  • SYSTEMTZDIR が定義されている場合、その値をそのまま返します。
  • SYSTEMTZDIR が定義されていない場合、PostgreSQLのインストールディレクトリ配下の share/timezone を指すパスを構築します。
  • パス構築は初回呼び出し時に一度だけ行われ、以降はキャッシュされた静的バッファの内容を返します。

アルゴリズムまたは振る舞い

1. SYSTEMTZDIR マクロの有無を確認します。 2. 未定義の場合、get_share_path を使用して実行ファイルの場所から共有ディレクトリパスを特定します。 3. 特定したパスに /timezone を連結し、静的変数 tzdir に格納します。 4. 2回目以降の呼び出しでは、計算済みの tzdir を即座に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/findtimezone.c
  • src/timezone/pgtz.c (内部の pg_open_tzfile やディレクトリ走査処理など)
  • **呼び出し先**:
  • get_share_path (PostgreSQLの共有ディレクトリパス取得用)
  • strlcpy (パス文字列のコピー用)

関連データ構造

  • MAXPGPATH: パス文字列の最大長を規定する定数。

失敗モードと境界条件

  • MAXPGPATH を超えるパス長が発生した場合、strlcpy の仕様に従い切り詰めが発生する可能性があります。

保守担当者向けメモ

  • 本関数は static であり、src/timezone/pgtz.c 内でのみ使用されることを意図していますが、initdb 等の外部ツールからも参照されるケースがあるため、変更時は影響範囲に注意してください。
  • SYSTEMTZDIR が定義されている環境では、ビルド時の設定が優先されます。
