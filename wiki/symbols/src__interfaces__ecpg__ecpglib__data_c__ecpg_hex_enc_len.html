ecpg_hex_enc_len

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/data.c:127

目的

バイナリデータ（bytea）を 16 進数文字列形式にエンコードする際に必要なバッファサイズを算出します。

シグネチャまたは形状

unsigned ecpg_hex_enc_len(unsigned srclen)

入力と出力

  • **入力**: srclen (変換対象となるバイナリデータのバイト長)
  • **出力**: 変換後の 16 進数文字列を格納するために必要なバッファサイズ（バイト単位）

前提条件と不変条件

  • 入力値 srclen は符号なし整数です。
  • 戻り値は常に srclen の 2 倍となります。

アルゴリズムまたは振る舞い

入力されたバイト数に対してビットシフト演算（srclen << 1）を行い、2 倍の値を返します。これは、1 バイトのバイナリデータが 2 文字の 16 進数表現（例: 0xFF）に変換されることに対応しています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/data.c:529
  • src/interfaces/ecpg/ecpglib/execute.c:491
  • src/interfaces/ecpg/ecpglib/execute.c:500
  • src/interfaces/ecpg/ecpglib/execute.c:1087
  • src/interfaces/ecpg/ecpglib/execute.c:1091
  • **呼び出し先**: なし（算術演算のみ）

関連データ構造

  • なし

失敗モードと境界条件

  • 入力値が非常に大きい場合、符号なし整数のオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数は ecpg_hex_encode と対になる計算ロジックです。エンコード処理を変更する場合は、本関数の計算式も整合性を保つ必要があります。
  • 呼び出し元の一部（execute.c）では、本関数の戻り値にさらにオフセット（+ 4 + 1 など）を加算してメモリ確保を行っているため、バッファ確保時には用途に応じた追加領域の要否を確認してください。
