FuzzyAttrMatchState

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_relation.c:62

目的

SQLクエリ内の列参照解決に失敗した際、スペルミスを補完するためのヒント生成用状態を保持する構造体です。

シグネチャまたは形状

typedef struct {
    int         distance;       /* 現在または制限距離 */
    RangeTblEntry *rfirst;      /* 最も近い非完全一致のRTE、またはNULL */
    AttrNumber  first;          /* rfirst内の列インデックス */
    RangeTblEntry *rsecond;     /* 同一距離の別の非完全一致RTE */
    AttrNumber  second;         /* rsecond内の列インデックス */
    RangeTblEntry *rexact1;     /* 最初の完全一致RTE、またはNULL */
    AttrNumber  exact1;         /* rexact1内の列インデックス */
    RangeTblEntry *rexact2;     /* 2番目の完全一致RTE、またはNULL */
    AttrNumber  exact2;         /* rexact2内の列インデックス */
} FuzzyAttrMatchState;

入力と出力

  • 本構造体は、列解決エラー発生時に palloc で動的に割り当てられ、候補探索ロジックを通じて更新されます。
  • 最終的に、ユーザーへのエラーメッセージ生成時にヒントとして参照されます。

前提条件と不変条件

  • distance は、スペル距離（レーベンシュタイン距離等）の閾値管理に使用されます。
  • rfirst/rsecond は非完全一致、rexact1/rexact2 は完全一致の候補を保持します。

アルゴリズムまたは振る舞い

1. 列名解決失敗時、全 RangeTblEntry を走査します。 2. 各列名と対象文字列の距離を計算し、本構造体の各フィールドを更新します。 3. 探索終了後、保持された候補に基づき、ユーザーに対して「もしかして: ...」といった形式でエラーヒントを提示します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_relation.c 内の列解決エラーハンドリングロジック（errorMissingColumn 等）。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • RangeTblEntry: 候補となるテーブルやビューの定義。
  • AttrNumber: 列を識別するためのインデックス。

失敗モードと境界条件

  • 候補が全く見つからない場合、各ポインタフィールドは NULL となります。
  • 複数の候補が同一距離で存在する場合、rfirst/rsecond の両方が埋まる可能性があります。

保守担当者向けメモ

  • 本構造体はエラーメッセージの品質向上に寄与します。ヒント生成ロジックを修正する際は、distance の計算コストがクエリ解析のパフォーマンスに与える影響に注意してください。
