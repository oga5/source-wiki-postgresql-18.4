GetStandbyLimitTime

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/standby.c:200

目的

ホットスタンバイ環境において、リカバリプロセスがクエリの競合解決を待機する際の期限（タイムアウト時刻）を算出する。

シグネチャまたは形状

TimestampTz GetStandbyLimitTime(void)

入力と出力

  • **入力**: なし（内部的に GetXLogReceiptTime を使用し、グローバル設定値 max_standby_streaming_delay および max_standby_archive_delay を参照する）。
  • **出力**: TimestampTz 型の時刻。待機期限を返す。

前提条件と不変条件

  • 戻り値が 0 の場合、待機期限なし（無限待機）を意味する。
  • 戻り値が正の時刻である場合、その時刻が経過した時点で競合解決のための待機を終了すべきであることを示す。

アルゴリズムまたは振る舞い

1. GetXLogReceiptTime を呼び出し、最後にWALデータを受信した時刻（rtime）と、そのソースがストリーミング経由か否か（fromStream）を取得する。 2. ストリーミング経由の場合:

  • max_standby_streaming_delay が負であれば 0 を返す。
  • それ以外の場合、rtime に max_standby_streaming_delay を加算した時刻を返す。

    • 3. アーカイブ経由の場合:

  • max_standby_archive_delay が負であれば 0 を返す。
  • それ以外の場合、rtime に max_standby_archive_delay を加算した時刻を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/standby.c 内の競合解決関数（ResolveRecoveryConflictWith... 系）にて、待機期限の取得に使用される。
  • **呼び出し先**:
  • GetXLogReceiptTime
  • TimestampTzPlusMilliseconds

関連データ構造

  • TimestampTz: PostgreSQLの時刻表現型。

失敗モードと境界条件

  • 設定値が -1 の場合は無限待機として 0 を返す。
  • GetXLogReceiptTime が返す rtime が基準となるため、システム時刻の不整合やWAL受信の停止状況に依存する。

保守担当者向けメモ

  • 本関数は、スタンバイ側で実行中のクエリがプライマリのWAL適用をどれだけ遅延させてよいかを制御する重要なロジックである。
  • 待機期限の計算には、WALの受信時刻を起点とするため、ネットワーク遅延やアーカイブからの読み込み速度が直接的に影響する。
