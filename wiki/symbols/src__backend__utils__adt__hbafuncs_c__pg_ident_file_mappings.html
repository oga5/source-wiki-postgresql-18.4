pg_ident_file_mappings

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/hbafuncs.c:592

目的

pg_ident.conf ファイルの内容を解析し、SQL経由で参照可能な行形式（Tuplestore）として出力するセット返却関数（SRF）です。

シグネチャまたは形状

Datum pg_ident_file_mappings(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（関数呼び出し時の引数情報）。
  • **出力**: PG_RETURN_NULL() を返し、副作用として Tuplestore に格納された結果セットを ReturnSetInfo 経由で提供します。

前提条件と不変条件

  • InitMaterializedSRF を使用して、結果保持用の Tuplestore を初期化する必要があります。
  • クエリ実行中の pg_ident.conf ファイルの変更に対して一貫性を保つため、Materialize モードで動作します。

アルゴリズムまたは振る舞い

1. InitMaterializedSRF を呼び出し、結果を保持する Tuplestore を作成します。 2. fcinfo->resultinfo から ReturnSetInfo を取得します。 3. fill_ident_view を呼び出し、pg_ident.conf のパース結果を Tuplestore に書き込みます。 4. PG_RETURN_NULL() を返し、処理を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLビュー pg_ident_file_mappings からのクエリ実行時に呼び出されます。
  • **呼び出し先**:
  • InitMaterializedSRF (PostgreSQL内部関数)
  • fill_ident_view (同ファイル内のヘルパー関数)

関連データ構造

  • ReturnSetInfo: SRFの結果セット情報を保持する構造体。
  • Tuplestore: 構築された行データを一時的に保持するストレージ。

失敗モードと境界条件

  • ファイルの読み込みやパースに失敗した場合、fill_ident_view 内でエラーが捕捉され、ビューの error カラムに情報が格納されます。
  • Tuplestore の初期化失敗時は、PostgreSQLの標準的なエラーハンドリングに従い例外が発生します。

保守担当者向けメモ

  • 本関数は読み取り専用です。設定ファイルの再読み込みロジックは含みません。
  • fill_ident_view が Tuplestore への書き込みを完結させるため、本関数自体は非常にシンプルなラッパー構造となっています。
