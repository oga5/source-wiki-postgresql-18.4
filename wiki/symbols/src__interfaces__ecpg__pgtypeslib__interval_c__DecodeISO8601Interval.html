DecodeISO8601Interval

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/interval.c:111

目的

ISO 8601 形式の文字列を解析し、interval 構造体の内部表現（tm 構造体および小数秒 fsec_t）へ変換する。

シグネチャまたは形状

static int DecodeISO8601Interval(char *str, int *dtype, struct tm *tm, fsec_t *fsec);

入力と出力

  • **入力**:
  • str: 解析対象の ISO 8601 形式文字列。
  • **出力**:
  • dtype: DTK_DELTA が設定される。
  • tm: 解析結果が格納される struct tm。
  • fsec: 解析された小数秒。
  • **戻り値**: 成功時は 0、失敗時はエラーコード（DTERR_BAD_FORMAT 等）。

前提条件と不変条件

  • str は 'P' で始まる必要がある。
  • ClearPgTm により、解析前に tm と fsec は初期化される。
  • -ffast-math コンパイルオプションは使用不可。

アルゴリズムまたは振る舞い

1. 文字列が 'P' で始まらない場合はエラーを返す。 2. 'T' を境界として日付部と時刻部を走査する。 3. ParseISO8601Number を用いて数値を抽出し、単位（Y, M, W, D, H, M, S）に応じて tm フィールドを加算する。 4. 小数秒を含む場合は AdjustFractDays または AdjustFractSeconds を呼び出し、tm と fsec に適切に配分する。 5. ISO 8601 の代替形式（例: YYYYMMDD や HHMMSS）にも対応し、特定の桁数条件を満たす場合に解析を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/pgtypeslib/interval.c 内の関数から呼び出される（直接の呼び出し元はソースコードの文脈から特定不可）。
  • **呼び出し先**: ClearPgTm, ParseISO8601Number, AdjustFractDays, AdjustFractSeconds, ISO8601IntegerWidth。

関連データ構造

  • struct tm: 日付・時刻の構成要素を保持。
  • fsec_t: 小数秒を保持。
  • interval: ECPG で使用される時間間隔の内部表現。

失敗モードと境界条件

  • 文字列が 'P' で始まらない、または長さが 2 未満の場合は DTERR_BAD_FORMAT を返す。
  • 不正な単位やフォーマットが検出された場合、DTERR_BAD_FORMAT を返す。
  • ParseISO8601Number がエラーを返した場合、そのエラーコードを伝播する。

保守担当者向けメモ

  • 本関数は PostgreSQL バックエンドの datetime.c から移植されたロジックである。
  • struct pg_tm ではなく struct tm を使用している点に注意すること。
  • IntervalStyle は INTSTYLE_POSTGRES_VERBOSE が前提となっている。
