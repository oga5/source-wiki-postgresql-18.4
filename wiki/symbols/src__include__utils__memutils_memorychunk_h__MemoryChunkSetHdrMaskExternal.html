MemoryChunkSetHdrMaskExternal

定義

  • 種別: function
  • 定義箇所: src/include/utils/memutils_memorychunk.h:189

目的

メモリコンテキストによって管理されるメモリチャンクに対し、標準的なオフセット計算が困難な「外部管理」状態であることを示すメタデータを設定します。

シグネチャまたは形状

static inline void MemoryChunkSetHdrMaskExternal(MemoryChunk *chunk, MemoryContextMethodID methodid)

入力と出力

  • **入力**:
  • chunk: 操作対象の MemoryChunk 構造体へのポインタ。
  • methodid: チャンクを管理するメモリコンテキストの識別子 (MemoryContextMethodID)。
  • **出力**: なし（chunk->hdrmask を直接更新）。

前提条件と不変条件

  • methodid は MEMORY_CONTEXT_METHODID_MASK 以下である必要があります（Assert による検証あり）。
  • チャンクの hdrmask は、マジックナンバー、外部フラグ、および methodid を保持する形式で上書きされます。

アルゴリズムまたは振る舞い

1. methodid が有効範囲内か検証します。 2. chunk->hdrmask に対して以下のビット演算を行い、値を格納します。

  • MEMORYCHUNK_MAGIC を設定。
  • MEMORYCHUNK_EXTERNAL_BASEBIT をシフトして外部フラグを立てる。
  • methodid をビットフィールドに結合する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/aset.c
  • src/backend/utils/mmgr/bump.c
  • src/backend/utils/mmgr/generation.c
  • **呼び出し先**:
  • Assert マクロ（デバッグビルド時）。

関連データ構造

  • MemoryChunk: チャンクヘッダー構造体。
  • MemoryContextMethodID: メモリコンテキストの型を識別する列挙型。

失敗モードと境界条件

  • methodid が MEMORY_CONTEXT_METHODID_MASK を超える場合、Assert が有効な環境ではプログラムが停止します。

保守担当者向けメモ

  • 本関数は、大規模なメモリ割り当てなど、通常のブロックオフセット計算が適用できないチャンクに対して使用されます。
  • hdrmask は 64 ビットのビットフィールドとして定義されており、将来的な拡張を行う場合は既存のビット配置との競合に注意してください。
