ParseFuncOrColumn

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_func.c:89

目的

SQLクエリ内の関数呼び出し、集約関数、ウィンドウ関数、および列プロジェクション（table.column 形式）を解析し、適切な構文木ノード（FuncExpr, Aggref, WindowFunc 等）を生成する。

シグネチャまたは形状

Node *ParseFuncOrColumn(ParseState *pstate, List *funcname, List *fargs,
                        Node *last_srf, FuncCall *fn, bool proc_call, int location)

入力と出力

  • **入力**:
  • pstate: 解析状態。
  • funcname: 関数名リスト。
  • fargs: 引数リスト。
  • last_srf: 直前のセット返し関数。
  • fn: FuncCall 構造体（関数呼び出しの修飾情報）。
  • proc_call: プロシージャ呼び出しフラグ。
  • location: 構文解析上の位置。
  • **出力**: 解析結果を表す Node*。

前提条件と不変条件

  • 引数の数が FUNC_MAX_ARGS を超える場合はエラーとなる。
  • 位置指定引数は名前付き引数より前に配置する必要がある。
  • func_get_detail を呼び出し、カタログから関数定義を特定する。

アルゴリズムまたは振る舞い

1. **初期化**: FuncCall 構造体から集約やウィンドウ関数の修飾子を抽出する。 2. **検証**: 引数リストの長さを確認し、FUNC_MAX_ARGS を超えていないかチェックする。 3. **引数処理**: 引数の型を抽出し、VOID 型の Param マーカーをリストから除外する（JDBC互換性のため）。 4. **名前付き引数チェック**: 名前付き引数の重複や、位置指定引数との混在順序を検証する。 5. **列プロジェクション判定**: 単一の複合型引数を持つ場合、ParseComplexProjection を試行する。 6. **関数検索**: func_get_detail を呼び出し、カタログから関数OID、戻り値型、引数情報を取得する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/functions.c
  • src/backend/parser/analyze.c
  • src/backend/parser/parse_expr.c
  • **呼び出し先**:
  • transformWhereClause
  • ParseComplexProjection
  • setup_parser_errposition_callback
  • func_get_detail

関連データ構造

  • ParseState: 解析状態を保持。
  • FuncCall: 関数呼び出しの構文情報。
  • FuncDetailCode: 関数検索結果のステータス。
  • NamedArgExpr: 名前付き引数。

失敗モードと境界条件

  • 引数が FUNC_MAX_ARGS を超過: ERRCODE_TOO_MANY_ARGUMENTS エラー。
  • 引数名の重複: ERRCODE_SYNTAX_ERROR エラー。
  • 位置指定引数が名前付き引数の後に続く: ERRCODE_SYNTAX_ERROR エラー。

保守担当者向けメモ

  • 本関数は「関数呼び出し」と「列プロジェクション」という歴史的に混在した構文を扱うため、条件分岐が複雑である。
  • func_get_detail の結果に基づき、後続の型変換やノード生成が行われるため、この関数の戻り値の整合性が重要である。
