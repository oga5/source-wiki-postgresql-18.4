BTParallelScanDesc

定義

  • 種別: typedef (構造体 BTParallelScanDescData へのポインタ)
  • 定義箇所: src/backend/access/nbtree/nbtree.c:93

目的

B-tree インデックスの並列スキャンにおいて、複数のワーカー間で共有されるスキャン状態を指し示すための型です。

シグネチャまたは形状

typedef struct BTParallelScanDescData *BTParallelScanDesc;

入力と出力

  • 入力: 並列クエリ実行時に共有メモリから取得されたオフセット値などが、OffsetToPointer を介して本型へ変換されます。
  • 出力: 並列スキャン状態を保持する構造体へのポインタとして、スキャン制御関数へ渡されます。

前提条件と不変条件

  • 本型が指すデータは、並列スキャン開始時に btinitparallelscan によって初期化されている必要があります。
  • 並列スキャン中の状態更新には、共有メモリ上の同期機構（LWLock 等）を伴う必要があります。

アルゴリズムまたは振る舞い

  • 並列スキャン実行時、各ワーカーは本型を介して共有状態にアクセスし、現在走査中のページ位置や同期情報を取得します。
  • ページ境界を跨ぐ際やスキャン位置の更新時に、共有状態を排他的に確保・解放するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • _bt_parallel_serialize_arrays
  • _bt_parallel_restore_arrays
  • btinitparallelscan
  • nbtree.c 内の並列スキャン制御ロジック（OffsetToPointer を用いたポインタ復元箇所）
  • 呼び出し先:
  • 本型自体はポインタ型であり、直接の関数呼び出し先はありません。指し示す先の BTParallelScanDescData が保持する同期変数や状態フィールドが操作対象となります。

関連データ構造

  • BTParallelScanDescData: 本型が指す実体。並列スキャン時の共有状態を保持します。

失敗モードと境界条件

  • 共有メモリ上のオフセットが不正な場合、OffsetToPointer による変換で無効なポインタを指す可能性があります。
  • 並列スキャンが初期化されていない状態で本型を参照した場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本型は BTParallelScanDescData へのエイリアスです。並列スキャン時の同期制御（_bt_parallel_seize / _bt_parallel_release）の整合性を確認する際は、この型が指す構造体のフィールド定義を併せて参照してください。
