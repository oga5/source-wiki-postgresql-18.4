pq_sendfloat4

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:251

目的

単精度浮動小数点数（float4）をバイナリ形式に変換し、通信用バッファ（StringInfo）へ追加する。

シグネチャまたは形状

void pq_sendfloat4(StringInfo buf, float4 f)

入力と出力

  • **入力**:
  • buf: データを追加する対象の StringInfo バッファ。
  • f: 送信する float4 型の値。
  • **出力**: なし（buf の内部状態が更新される）。

前提条件と不変条件

  • buf は pq_beginmessage 等によって初期化済みであること。
  • 内部的に pq_sendint32 を呼び出すため、ネットワークバイトオーダー（ビッグエンディアン）への変換が保証される。

アルゴリズムまたは振る舞い

1. float4 型の値を、同じメモリ領域を共有する uint32 型の共用体（union）に代入する。 2. 共用体を介してビット表現を整数として取り出す。 3. 取り出した整数値を pq_sendint32 に渡し、ネットワークバイトオーダーに変換した上で buf に書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/float.c:356（直接的な呼び出しを確認）
  • **呼び出し先**:
  • pq_sendint32（src/backend/libpq/pqformat.c 内の関数）

関連データ構造

  • StringInfo: メッセージ構築用の動的バッファ構造体。
  • float4: 単精度浮動小数点数型。

失敗モードと境界条件

  • buf が指すメモリ領域が不足している場合、StringInfo の拡張処理（enlargeStringInfo）が内部で呼び出され、メモリ不足時はエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は浮動小数点数のビット表現をそのまま整数として扱うため、プラットフォーム間の浮動小数点表現（IEEE 754等）が一致していることを前提としている。
  • pq_sendfloat8 と同様の設計パターンを採用している。
