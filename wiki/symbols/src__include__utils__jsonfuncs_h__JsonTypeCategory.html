JsonTypeCategory

定義

  • 種別: enum
  • 定義箇所: src/include/utils/jsonfuncs.h:68

目的

PostgreSQL のデータ型を JSON 形式へ変換する際、変換ルールを決定するためのカテゴリ分類を提供します。

シグネチャまたは形状

typedef enum {
    JSONTYPE_NULL,
    JSONTYPE_BOOL,
    JSONTYPE_NUMERIC,
    JSONTYPE_DATE,
    JSONTYPE_TIMESTAMP,
    JSONTYPE_TIMESTAMPTZ,
    JSONTYPE_JSON,
    JSONTYPE_JSONB,
    JSONTYPE_ARRAY,
    JSONTYPE_COMPOSITE,
    JSONTYPE_CAST,
    JSONTYPE_OTHER
} JsonTypeCategory;

入力と出力

  • 入力: datum_to_json 等の変換関数において、対象データの型判定に使用されます。
  • 出力: 判定されたカテゴリに基づき、適切な出力関数（outfuncoid）が選択されます。

前提条件と不変条件

  • JSONTYPE_JSON は is_jsonb フラグが偽の場合に適用されます。
  • JSONTYPE_JSONB は is_jsonb フラグが真の場合に適用されます。
  • JSONTYPE_BOOL および JSONTYPE_NUMERIC は組み込み型のみを対象とします。

アルゴリズムまたは振る舞い

json_categorize_type 関数によって、PostgreSQL の型情報から本列挙型のいずれかに分類されます。この分類結果は、後続のデータ変換処理において、対象データがどのように JSON 表現へシリアライズされるかを制御するフラグとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execExpr.c、src/backend/executor/execExprInterp.c、src/backend/utils/adt/json.c 内の各関数から参照されています。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数を呼び出すことはありません。

関連データ構造

  • Datum: 変換対象となる PostgreSQL のデータ表現。
  • Oid: 変換関数を特定するための型識別子。

失敗モードと境界条件

  • 該当する型が定義済みカテゴリに当てはまらない場合、JSONTYPE_OTHER に分類されます。
  • 明示的なキャストが存在する場合は JSONTYPE_CAST が優先されます。

保守担当者向けメモ

  • 新しいデータ型を JSON 変換に対応させる際は、json_categorize_type のロジックを更新し、必要に応じて本列挙型に新しいカテゴリを追加してください。
  • jsonfuncs.c 内の実装と密結合しているため、変更時は変換ロジックへの影響を必ず確認してください。
