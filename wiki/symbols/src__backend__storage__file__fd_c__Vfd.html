Vfd

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/file/fd.c:199

目的

PostgreSQLのファイル管理サブシステムにおいて、OSのファイル記述子（FD）の枯渇を防ぐための仮想ファイル記述子（VFD）を保持する構造体です。LRU（Least Recently Used）キャッシュ機構の各エントリとして機能します。

シグネチャまたは形状

typedef struct vfd
{
    int            fd;                /* 現在のFD、またはVFD_CLOSED */
    unsigned short fdstate;           /* VFD状態のビットフラグ */
    ResourceOwner  resowner;          /* 自動クリーンアップ用の所有者 */
    File           nextFree;          /* フリーリスト内の次へのリンク */
    File           lruMoreRecently;   /* LRUリスト（二重連結リスト） */
    File           lruLessRecently;
    off_t          fileSize;          /* ファイルサイズ（一時ファイル以外は0） */
    char          *fileName;          /* ファイル名（未使用時はNULL） */
    int            fileFlags;         /* open(2)用フラグ */
    mode_t         fileMode;          /* open(2)用モード */
} Vfd;

入力と出力

  • **入力**: fileName（文字列）、fileFlags、fileMode 等のファイルオープンに必要な属性を保持します。
  • **出力**: fd フィールドを通じて、現在OSレベルで開かれている実際のファイル記述子を保持します。

前提条件と不変条件

  • fileName は malloc で確保されており、VFDを閉じる際に必ず free する必要があります。
  • VfdCache[0] はLRUリングのアンカー（番兵）として使用されます。
  • File 型は VfdCache 配列のインデックスとして機能します。

アルゴリズムまたは振る舞い

  • **LRU管理**: lruMoreRecently および lruLessRecently を用いて、アクセス頻度に基づいた二重連結リストを構成します。
  • **リソース追跡**: ResourceOwner を保持し、トランザクション終了時などの自動クローズをサポートします。
  • **キャッシュ溢れ対応**: プロセスが保持できるFD数を超過した場合、LRUリストの末尾にあるVFDが物理的にクローズされ、FDが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: fd.c 内の AllocateVfd（確保）、PathNameOpenFile（初期化）、LruDelete（キャッシュ解放）、および VfdCache 配列を操作する各関数から参照されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • File: VfdCache 配列のインデックスとして使用される型。
  • VfdCache: Vfd 構造体の配列（グローバルなキャッシュ領域）。
  • ResourceOwner: リソースの所有権管理に使用。

失敗モードと境界条件

  • fileName がNULLの場合、そのVFDは未使用とみなされます。
  • fd が VFD_CLOSED の場合、ファイルは現在開かれていません。
  • realloc による VfdCache の拡張時にメモリ不足が発生する可能性があります。

保守担当者向けメモ

  • fileName のメモリ管理（malloc/free）には特に注意してください。
  • VfdCache のサイズ変更（realloc）が発生すると、既存のポインタが無効になる可能性があるため、インデックス（File型）による参照を推奨します。
  • FDDEBUG マクロを有効にすると、LRUリングの操作ログが取得可能です。
