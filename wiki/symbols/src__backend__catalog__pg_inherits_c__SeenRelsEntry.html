SeenRelsEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/pg_inherits.c:37

目的

継承階層の探索中に、既に訪問したリレーションを追跡するためのハッシュテーブル用エントリ構造体。

シグネチャまたは形状

typedef struct SeenRelsEntry
{
	Oid			rel_id;			/* relation oid */
	int			list_index;		/* its position in output list(s) */
} SeenRelsEntry;

入力と出力

  • 入力: ハッシュテーブルへの挿入時に rel_id および list_index が設定される。
  • 出力: 探索アルゴリズムにおいて、重複排除および順序管理のために参照される。

前提条件と不変条件

  • rel_id はリレーションを一意に識別する OID であること。
  • ハッシュテーブルの entrysize は sizeof(SeenRelsEntry) と一致している必要がある。

アルゴリズムまたは振る舞い

find_all_inheritors 関数において、継承ツリーを幅優先探索する際、ハッシュテーブルのキーとして使用される。既に訪問済みのリレーションを記録することで、循環参照の防止や重複処理の回避を行う。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_inherits.c 内の find_all_inheritors 関数が、ハッシュテーブルの初期化およびエントリの検索・挿入を行う。
  • 呼び出し先: なし（データ構造体のため）。

関連データ構造

  • HTAB (ハッシュテーブル): 本構造体を格納するために使用される。

失敗モードと境界条件

  • メモリ不足: ハッシュテーブルの作成やエントリの挿入時にメモリが不足した場合、PostgreSQL のエラーハンドリング機構により中断される。

保守担当者向けメモ

  • 本構造体は find_all_inheritors の内部ロジックに強く依存している。ハッシュテーブルの定義やサイズ変更を行う際は、pg_inherits.c 内の関連するハッシュ制御構造体（ctl）との整合性に注意すること。
