gbt_int4_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int4.c:133

目的

GiST インデックスにおいて、int4 型のインデックスキーと検索条件を照合し、条件に一致するかどうかを判定する。

シグネチャまたは形状

Datum gbt_int4_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY 構造体（インデックスエントリ）。
  • PG_GETARG_INT32(1): 検索対象の int4 値。
  • PG_GETARG_UINT16(2): 比較戦略を示す StrategyNumber。
  • PG_GETARG_POINTER(4): recheck フラグへのポインタ。
  • **出力**:
  • Datum (bool): 条件に一致する場合は true、一致しない場合は false を返す。
  • *recheck: 常に false が書き込まれる。

前提条件と不変条件

  • int4 型の比較は完全であるため、インデックスの結果は常に正確であり、recheck は常に false となる。
  • 入力された entry->key は int32KEY 構造体としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. recheck フラグを false に設定する。 2. entry->key を int32KEY 構造体に変換し、その下限値と上限値を GBT_NUMKEY_R 構造体に格納する。 3. 共通ユーティリティ関数 gbt_num_consistent を呼び出し、int4 特有の比較ロジック（tinfo 経由）を用いて検索条件との照合を行う。 4. 照合結果をブール値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックスエンジン（直接的な呼び出し元コードはソース抜粋に含まれていない）。
  • **呼び出し先**: gbt_num_consistent（btree_utils_num.c で定義）。

関連データ構造

  • int32KEY: int4 型の範囲（lower, upper）を保持する構造体。
  • GISTENTRY: GiST インデックスのエントリ情報。
  • tinfo: gbtree_ninfo 構造体。int4 型の比較関数やサイズ情報を保持する設定テーブル。

失敗モードと境界条件

  • int4 型の範囲外の値や不正な StrategyNumber が渡された場合の挙動は、依存先の gbt_num_consistent の実装に依存する。

保守担当者向けメモ

  • 本関数は btree_utils_num.h で定義された汎用ロジックを利用している。ロジックの変更が必要な場合は、まず tinfo が参照している比較関数を確認すること。
  • recheck = false は、このインデックスが「損失のない（lossless）」検索を提供することを意味する。
