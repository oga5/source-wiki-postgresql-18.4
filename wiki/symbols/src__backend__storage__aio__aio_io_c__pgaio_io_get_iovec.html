pgaio_io_get_iovec

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_io.c:41

目的

非同期I/Oハンドル（PgAioHandle）に関連付けられたI/Oベクタ（iovec）の配列を取得します。

シグネチャまたは形状

int pgaio_io_get_iovec(PgAioHandle *ioh, struct iovec **iov)

入力と出力

  • **入力**:
  • ioh: 操作対象の PgAioHandle へのポインタ。
  • iov: iovec 配列の開始アドレスを格納するための出力用ポインタ。
  • **出力**:
  • 戻り値: int 型の定数 PG_IOV_MAX を返します。

前提条件と不変条件

  • ioh->state が PGAIO_HS_HANDED_OUT である必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. ハンドルの状態が PGAIO_HS_HANDED_OUT であることを確認します。 2. グローバルな制御構造体 pgaio_ctl が保持する iovecs 配列から、ハンドル内のオフセット ioh->iovec_off に対応する位置を特定し、iov に代入します。 3. PG_IOV_MAX を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/smgr/md.c:1011
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • PgAioHandle: I/O操作の状態とメタデータを保持する構造体。
  • pgaio_ctl: システム全体のAIO制御構造体。
  • iovec: POSIX標準のI/Oベクタ構造体。

失敗モードと境界条件

  • ioh が PGAIO_HS_HANDED_OUT 以外の状態である場合、アサーション失敗によりプロセスが異常終了します（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は、I/O操作の準備段階でバッファ情報を取得するために使用されます。
  • 戻り値の PG_IOV_MAX は固定値であり、実際の有効なベクタ数とは異なる可能性があるため、呼び出し側で適切に処理する必要があります。
