PGSemaphoreReset

定義

  • 種別: function
  • 定義箇所: src/backend/port/posix_sema.c:293

目的

POSIX セマフォの値を 0 にリセットする。POSIX 標準 API に直接的なリセット機能が存在しないため、本関数がその代替手段を提供する。

シグネチャまたは形状

void PGSemaphoreReset(PGSemaphore sema)

入力と出力

  • **入力**: sema (型: PGSemaphore) - リセット対象のセマフォへのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • sema は有効な PGSemaphore オブジェクトを指している必要がある。
  • PG_SEM_REF(sema) マクロにより、内部の sem_t 構造体が正しく参照可能であること。

アルゴリズムまたは振る舞い

1. 無限ループ内で sem_trywait を繰り返し呼び出す。 2. sem_trywait が成功する限り、セマフォの値を減算し続ける。 3. errno が EAGAIN または EDEADLK となった場合、セマフォの値が 0 に達したと判断しループを終了する。 4. EINTR が発生した場合は、シグナル割り込みとしてループを継続する。 5. その他のエラーが発生した場合は elog(FATAL, ...) を呼び出し、プロセスを異常終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/proc.c: MyProc->sem に対して呼び出し。
  • その他、sysv_sema.c および win32_sema.c にも同名の関数が存在し、プラットフォームに応じて使い分けられる。
  • **呼び出し先**:
  • sem_trywait (POSIX API)
  • elog (PostgreSQL エラーハンドリング)

関連データ構造

  • PGSemaphore: sem_t * を保持する抽象型。
  • PGSemaphoreData: セマフォの実体を含む構造体。

失敗モードと境界条件

  • **致命的エラー**: sem_trywait が EAGAIN / EDEADLK / EINTR 以外のエラーを返した場合、システム整合性を保つために FATAL エラーとして処理を停止する。

保守担当者向けメモ

  • 本関数は POSIX セマフォの仕様上の制約を回避するための実装である。
  • sem_trywait の戻り値チェックにおいて、EINTR が実際に発生しうるかについてはソースコード上のコメントで疑問が呈されているが、堅牢性のためにハンドリングが維持されている。
