ExecReScanResult

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeResult.c:248

目的

Result ノードの状態をリセットし、クエリ実行の再試行または再スキャンを可能にする。

シグネチャまたは形状

void ExecReScanResult(ResultState *node)

入力と出力

  • **入力**: ResultState *node（再スキャン対象の Result ノードの状態構造体）
  • **出力**: なし（副作用として node の内部状態を更新する）

前提条件と不変条件

  • node は有効な ResultState 構造体を指している必要がある。
  • node->resconstantqual が存在する場合、再スキャン時に定数条件の再評価が必要であることを示すフラグが設定される。

アルゴリズムまたは振る舞い

2. node->resconstantqual が NULL でない場合、node->rs_checkqual を true に設定し、定数条件の再評価を予約する。 3. 子ノード（outerPlan）が存在し、かつその子ノードのパラメータ変更（chgParam）が保留されていない場合、ExecReScan(outerPlan) を呼び出して子ノードの再スキャンを再帰的に実行する。

  • chgParam が存在する場合は、最初の ExecProcNode 呼び出し時に子ノードが自動的に再スキャンされるため、ここではスキップされる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:134（ExecReScan 経由での呼び出しが確認されている）
  • **呼び出し先**: ExecReScan（子ノードに対して再帰的に呼び出す）

関連データ構造

  • ResultState: Result ノードの実行時状態を保持する構造体。
  • PlanState: 子ノードの実行状態を保持する構造体。

失敗モードと境界条件

  • 子ノードが存在しない場合、outerPlan は NULL となり、子ノードの再スキャンは行われない。
  • chgParam が設定されている場合、この関数内では子ノードの再スキャンは実行されない。

保守担当者向けメモ

  • 本関数は Result ノード専用の再スキャン処理である。
  • Result ノード自体は Mark/Restore 操作をサポートしていないため、それらの機能は子ノードの挙動に依存する。
