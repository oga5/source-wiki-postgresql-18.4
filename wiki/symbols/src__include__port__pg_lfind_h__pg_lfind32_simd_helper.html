pg_lfind32_simd_helper

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_lfind.h:108

目的

SIMD 命令を活用し、4 つのベクトルレジスタ分（4 * sizeof(Vector32) / sizeof(uint32) 要素）のデータを一度に走査して、指定された値が含まれているかを高速に判定します。

シグネチャまたは形状

static inline bool pg_lfind32_simd_helper(const Vector32 keys, const uint32 *base)

入力と出力

  • **入力**:
  • keys: 検索対象の値を保持する Vector32 型のベクトル。
  • base: 検索対象となる uint32 配列の先頭ポインタ。
  • **出力**:
  • bool: 配列内に keys と一致する要素が少なくとも 1 つ存在する場合は true、それ以外は false を返します。

前提条件と不変条件

  • base ポインタは、少なくとも 4 * sizeof(Vector32) / sizeof(uint32) 個の uint32 要素を保持する十分なメモリ領域を指している必要があります。
  • vector32_load 等の simd.h で定義されたベクトル操作関数が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. base から 4 つの連続したベクトルブロックをロードします。 2. 各ブロックに対して vector32_eq を実行し、keys との比較結果（一致フラグ）を取得します。 3. 4 つの比較結果を vector32_or を用いて論理和で集約し、単一のベクトルにまとめます。 4. vector32_is_highbit_set を使用して、集約された結果のいずれかのビットが立っているか（＝一致が存在するか）を判定し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/pg_lfind.h 内の pg_lfind32 関数から呼び出されます。
  • **呼び出し先**: vector32_load, vector32_eq, vector32_or, vector32_is_highbit_set（いずれも port/simd.h で定義）。

関連データ構造

  • Vector32: SIMD 演算用の 32 ビット整数ベクトル型。

失敗モードと境界条件

  • **境界条件**: 配列の末尾付近で呼び出される場合、配列の範囲外アクセスが発生しないよう、呼び出し側で十分な要素数が確保されていることを保証する必要があります。
  • **失敗モード**: SIMD 命令セットがサポートされていない環境で USE_NO_SIMD が定義されている場合、本関数は利用できません。

保守担当者向けメモ

  • 本関数は pg_lfind32 のメインループおよび末尾処理の両方で使用されます。
  • USE_ASSERT_CHECKING が有効な場合、呼び出し側で逐次検索との結果比較が行われるため、変更時は整合性に注意してください。
