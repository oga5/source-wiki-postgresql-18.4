xl_brin_samepage_update

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_xlog.h:102

目的

BRINインデックスにおいて、同一ページ内での更新操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_brin_samepage_update
{
	OffsetNumber offnum;
} xl_brin_samepage_update;

入力と出力

  • **入力**: 更新対象となるインデックスページ内のオフセット番号 (offnum)。
  • **出力**: なし（構造体としてWALレコードの一部を構成する）。

前提条件と不変条件

  • offnum は、対象となるBRINインデックスページ内の有効なタプル位置を指している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、BRINインデックスのページ内更新が発生した際に、その位置情報をWALレコードとして保持します。
  • リカバリ（Redo）時には、この構造体を読み取ることで、どのオフセットのタプルを更新対象とするかを特定します。

呼び出し元と呼び出し先

  • **呼び出し元（記録側）**: src/backend/access/brin/brin_pageops.c において、インデックスのページ内更新時に本構造体がインスタンス化されます。
  • **呼び出し先（再生側）**: src/backend/access/brin/brin_xlog.c 内のRedo処理にて、XLogRecGetData を通じて本構造体が読み取られ、インデックスの復元に使用されます。
  • **その他**: src/backend/access/rmgrdesc/brindesc.c にて、WALログのデバッグ出力（説明生成）のために参照されます。

関連データ構造

  • OffsetNumber: PostgreSQLのページ内タプル位置を示す型。

失敗モードと境界条件

  • WALレコードの読み取り時にデータが破損している場合、XLogRecGetData から不正な値が取得される可能性があります。
  • 参照される offnum がページ内の有効な範囲外である場合、Redo処理中にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は非常にシンプルですが、BRINインデックスの整合性を保つための重要なWALコンポーネントです。構造を変更する場合は、brin_xlog.c のRedoロジックおよび brindesc.c の記述ロジックとの整合性を確認してください。
