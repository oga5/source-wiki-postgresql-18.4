EnumValuesCreate

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_enum.c:83

目的

列挙型（ENUM）の初期値をシステムカタログ pg_enum に挿入します。

シグネチャまたは形状

void EnumValuesCreate(Oid enumTypeOid, List *vals)

入力と出力

  • **入力**:
  • enumTypeOid: 対象となる列挙型の OID。
  • vals: 挿入する列挙ラベルの文字列リスト。
  • **出力**: なし（副作用として pg_enum カタログを更新）。

前提条件と不変条件

  • 現在のトランザクションがネストレベル 1 の場合、対象の enumTypeOid は未コミットの列挙型として追跡されます。
  • ラベルの長さは NAMEDATALEN - 1 バイト以下である必要があります。
  • 挿入される OID は偶数である必要があります。

アルゴリズムまたは振る舞い

1. **追跡登録**: トランザクションのネストレベルが 1 の場合、uncommitted_enum_types ハッシュテーブルに型 OID を登録します。 2. **OID 割り当て**: GetNewOidWithIndex を使用し、偶数の OID を各ラベルに対して生成します。 3. **ソート**: 生成した OID を oid_cmp でソートします。 4. **タプル生成**: 各ラベルに対し、enumsortorder を elemno + 1 として設定し、pg_enum のタプルを作成します。 5. **一括挿入**: CatalogTuplesMultiInsertWithInfo を使用して、バッファリングされたタプルを効率的にカタログへ書き込みます。 6. **クリーンアップ**: 使用したメモリ（OID 配列、タプルスロット）を解放し、カタログリレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/typecmds.c（CREATE TYPE AS ENUM 実行時）。
  • **呼び出し先**: init_uncommitted_enum_types, hash_search, table_open, GetNewOidWithIndex, qsort, CatalogOpenIndexes, MakeSingleTupleTableSlot, CatalogTuplesMultiInsertWithInfo 等。

関連データ構造

  • pg_enum: 列挙型定義を保持するシステムカタログ。
  • uncommitted_enum_types: トランザクション内で作成された未コミットの列挙型を追跡するハッシュテーブル。
  • TupleTableSlot: カタログ挿入用の中間データ構造。

失敗モードと境界条件

  • **重複ラベル**: リスト内に重複がある場合、一意性制約違反によりエラーとなります。
  • **ラベル長超過**: NAMEDATALEN を超えるラベルは ERRCODE_INVALID_NAME でエラーとなります。
  • **OID 枯渇**: OID カウンタが一周するほどの大量挿入は想定されていません。

保守担当者向けメモ

  • 本関数は pg_enum への初期挿入のみを扱います。ラベルの追加や変更は AddEnumLabel 等の別関数が担当します。
  • enumsortorder は float4 で管理されており、挿入順序を維持するために使用されます。
