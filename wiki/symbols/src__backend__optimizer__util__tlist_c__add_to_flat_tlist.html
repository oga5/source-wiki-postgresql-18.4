add_to_flat_tlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/tlist.c:140

目的

既存のターゲットリスト（tlist）に、指定された式のリスト（exprs）から重複を除いて追加し、新しいターゲットエントリ（TargetEntry）として構成されたリストを返します。

シグネチャまたは形状

List *add_to_flat_tlist(List *tlist, List *exprs)

入力と出力

  • **入力**:
  • tlist: 現在のターゲットリスト（List）。
  • exprs: 追加対象となる式のリスト（List）。
  • **出力**:
  • 追加後のターゲットリスト（List）。

前提条件と不変条件

  • tlist に含まれる各 TargetEntry の resno は、リストの順序に従って昇順に割り当てられます。
  • tlist_member を使用して、追加しようとする式が既に tlist 内に存在するかを判定します。

アルゴリズムまたは振る舞い

1. 現在の tlist の長さを基に、次に割り当てるべき resno（next_resno）を算出します。 2. exprs リストを反復処理します。 3. 各式について tlist_member を呼び出し、既にリスト内に存在するか確認します。 4. 存在しない場合、makeTargetEntry を使用して式をコピーし、新しい TargetEntry を作成します。この際、next_resno を割り当て、インクリメントします。 5. 新しい TargetEntry を lappend で tlist に追加します。 6. 最終的な tlist を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/deparse.c
  • contrib/postgres_fdw/postgres_fdw.c
  • **呼び出し先**:
  • list_length
  • tlist_member
  • makeTargetEntry
  • copyObject
  • lappend

関連データ構造

  • List: PostgreSQL の汎用リスト構造。
  • TargetEntry: ターゲットリストの各要素を構成するノード。
  • Expr: ターゲットリストに追加される式ノード。

失敗モードと境界条件

  • exprs が空の場合、何も追加されず元の tlist がそのまま返されます。
  • tlist が空の場合、next_resno は 1 から開始されます。
  • copyObject が失敗した場合（メモリ不足等）、PostgreSQL の標準的なエラー処理により中断されます。

保守担当者向けメモ

  • makeTargetEntry 呼び出し時の copyObject は、式ツリーの共有を防ぐために必要です。
  • tlist_member は equal() を使用して式の一致を判定するため、意味的に等価な式が重複して追加されることはありません。
