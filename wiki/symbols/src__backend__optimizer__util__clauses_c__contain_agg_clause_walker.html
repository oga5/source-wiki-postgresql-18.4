contain_agg_clause_walker

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/clauses.c:184

目的

式ツリーを走査し、現在のクエリレベルで評価される集約関数（Aggref）またはグループ化関数（GroupingFunc）が含まれているかを判定します。

シグネチャまたは形状

static bool contain_agg_clause_walker(Node *node, void *context)

入力と出力

  • **入力**:
  • node: 走査対象の式ツリーノード。
  • context: 走査コンテキスト（本関数では使用されません）。
  • **出力**:
  • bool: 集約またはグループ化関数が見つかった場合は true、それ以外は false を返します。

前提条件と不変条件

  • Aggref または GroupingFunc ノードが検出された場合、その agglevelsup（外部クエリレベルへの参照）は 0 である必要があります。
  • SubLink ノードは本関数では処理されません（Assert により禁止されています）。

アルゴリズムまたは振る舞い

1. node が NULL の場合は false を返します。 2. node が Aggref または GroupingFunc の場合、agglevelsup == 0 を検証した上で true を返し、走査を中断します。 3. 上記以外の場合、expression_tree_walker を再帰的に呼び出し、子ノードを探索します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/clauses.c 内の contain_agg_clause 関数から呼び出されます。
  • **呼び出し先**: expression_tree_walker を呼び出し、ツリーの再帰的走査を行います。

関連データ構造

  • Aggref: 集約関数ノード。
  • GroupingFunc: グループ化関数ノード。
  • Node: 汎用ノード構造体。

失敗モードと境界条件

  • SubLink ノードに遭遇するとアサーションエラーが発生します。
  • 外部クエリレベルを参照する集約（agglevelsup > 0）が含まれている場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は expression_tree_walker のコールバックとして設計されています。
  • 探索を中断させるために true を返す仕様であるため、条件判定ロジックを変更する際は戻り値の意味に注意してください。
