Zstd_close

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_zstd.c:435

目的

Zstandard 圧縮または解凍セッションを終了し、関連するストリームリソースとメモリを解放する。

シグネチャまたは形状

bool Zstd_close(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH（圧縮・解凍状態を保持するハンドル）
  • **出力**: bool（正常終了した場合は true、エラーが発生した場合は false）

前提条件と不変条件

  • CFH->private_data は ZstdCompressorState 構造体へのポインタとして初期化されている必要がある。
  • 終了処理完了後、CFH->private_data は NULL に設定される。

アルゴリズムまたは振る舞い

1. **圧縮ストリームの終了**: zstdcs->cstream が存在する場合、ZSTD_compressStream2 を ZSTD_e_end モードでループ実行し、残存するバッファをすべてフラッシュしてファイルへ書き出す。その後、ストリームと出力バッファを解放する。 2. **解凍ストリームの終了**: zstdcs->dstream が存在する場合、ストリームを解放し、入力バッファを解放する。 3. **ファイルクローズ**: 関連するファイルポインタ zstdcs->fp を閉じる。 4. **メモリ解放**: ZstdCompressorState 構造体自体を解放する。 5. **エラーハンドリング**: 処理中に ZSTD ライブラリのエラーやファイル書き込みエラーが発生した場合、zstdcs->zstderror にエラー内容を記録し、戻り値を false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていないが、src/bin/pg_dump/compress_zstd.c:569 にて CFH->close_func に代入されている。
  • **呼び出し先**: ZSTD_compressStream2, ZSTD_isError, ZSTD_getErrorName, fwrite, ZSTD_freeCStream, pg_free, ZSTD_freeDStream, fclose, strerror。

関連データ構造

  • CompressFileHandle: 圧縮処理の抽象化ハンドル。
  • ZstdCompressorState: Zstd 固有のコンテキスト（ストリーム、バッファ、ファイルポインタ）。

失敗モードと境界条件

  • ZSTD_compressStream2 がエラーを返した場合、処理を中断し false を返す。
  • fwrite による書き込みバイト数が期待値と異なる場合、ENOSPC または errno を記録し false を返す。
  • fclose が失敗した場合、エラーを記録し false を返す。

保守担当者向けメモ

  • 本関数は USE_ZSTD が定義されている環境でのみ有効な実装である。
  • unconstify マクロを使用して dstream の入力ソースを解放している点に注意すること。
