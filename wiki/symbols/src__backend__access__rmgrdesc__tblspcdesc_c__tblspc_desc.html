tblspc_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/tblspcdesc.c:20

目的

テーブルスペース操作（作成・削除）に関連するWALレコードを解析し、人間が読み取り可能な形式に変換して文字列バッファへ出力する。

シグネチャまたは形状

void tblspc_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 出力文字列を格納するための StringInfo バッファ。
  • record: 解析対象のWALレコードを保持する XLogReaderState 構造体。
  • **出力**: なし（buf を通じて文字列が書き込まれる）。

前提条件と不変条件

  • record は有効なWALレコードを指している必要がある。
  • XLogRecGetInfo で取得した情報から XLR_INFO_MASK を除外した値が、XLOG_TBLSPC_CREATE または XLOG_TBLSPC_DROP のいずれかであること。

アルゴリズムまたは振る舞い

1. XLogRecGetData を使用してWALレコードの生データを取得する。 2. XLogRecGetInfo で取得した情報から XLR_INFO_MASK をビット演算で除外し、操作タイプを特定する。 3. 操作タイプが XLOG_TBLSPC_CREATE の場合:

  • データを xl_tblspc_create_rec 構造体にキャストし、テーブルスペースIDとパスを buf に追記する。

    • 4. 操作タイプが XLOG_TBLSPC_DROP の場合:

  • データを xl_tblspc_drop_rec 構造体にキャストし、テーブルスペースIDを buf に追記する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h に定義されたリソースマネージャリストを通じて、WAL解析ツール（pg_waldump 等）から間接的に呼び出される。
  • **呼び出し先**:
  • XLogRecGetData (マクロ)
  • XLogRecGetInfo (マクロ)
  • appendStringInfo

関連データ構造

  • StringInfo: 出力用文字列バッファ。
  • XLogReaderState: WALレコードの読み取り状態。
  • xl_tblspc_create_rec: テーブルスペース作成時のWALレコード構造体。
  • xl_tblspc_drop_rec: テーブルスペース削除時のWALレコード構造体。

失敗モードと境界条件

  • info が定義された操作タイプ（CREATE / DROP）のいずれにも該当しない場合、バッファには何も書き込まれない。
  • 不正なWALレコードデータが渡された場合、構造体へのキャストにより予期せぬ値が出力される可能性がある。

保守担当者向けメモ

  • 新しいテーブルスペース操作が追加された場合は、本関数の条件分岐にケースを追加する必要がある。
  • XLR_INFO_MASK によるマスク処理は、補助フラグを除外して純粋な操作タイプを特定するために不可欠である。
