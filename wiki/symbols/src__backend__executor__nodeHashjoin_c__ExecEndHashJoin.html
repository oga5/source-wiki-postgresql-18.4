ExecEndHashJoin

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHashjoin.c:947

目的

ハッシュ結合ノードの実行終了処理を行い、関連するリソースを解放する。

シグネチャまたは形状

void ExecEndHashJoin(HashJoinState *node)

入力と出力

  • **入力**: HashJoinState *node（終了対象のハッシュ結合ノードの状態構造体）
  • **出力**: なし（void）

前提条件と不変条件

  • node は有効な HashJoinState を指している必要がある。
  • 呼び出し後、node->hj_HashTable は NULL に設定される。

アルゴリズムまたは振る舞い

1. **ハッシュテーブルの解放**: node->hj_HashTable が存在する場合、ExecHashTableDestroy を呼び出してハッシュテーブルを破棄し、ポインタを NULL に設定する。 2. **サブツリーの終了**: outerPlanState(node) および innerPlanState(node) に対して ExecEndNode を呼び出し、結合に関与する子ノードの終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:699（ExecEndNode 経由の呼び出しが想定される）
  • **呼び出し先**:
  • ExecHashTableDestroy
  • ExecEndNode（outerPlanState および innerPlanState に対して）

関連データ構造

  • HashJoinState: 結合ノードの実行状態を保持する構造体。

失敗モードと境界条件

  • node->hj_HashTable が既に NULL の場合、ハッシュテーブルの解放処理はスキップされる。

保守担当者向けメモ

  • 本関数はハッシュ結合の終了処理の標準的なエントリーポイントである。子ノードの終了処理順序（outer, inner）は ExecEndNode の規約に従う。
