cash_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/cash.c:386

目的

内部表現である Cash 型（int64）の値を、現在のロケール設定（lc_monetary）に基づいた通貨形式の文字列に変換します。

シグネチャまたは形状

Datum cash_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して Cash 型の値を1つ受け取ります。
  • **出力**: ロケールに従ってフォーマットされた通貨文字列（char *）を Datum として返します。

前提条件と不変条件

  • PGLC_localeconv() を通じて取得したロケール情報（frac_digits, mon_grouping, mon_decimal_point 等）に依存します。
  • ロケール設定が不正な場合、frac_digits は 2、mon_grouping は 3 にフォールバックされます。
  • 通貨記号が空の場合は $ が使用されます。

アルゴリズムまたは振る舞い

1. PGLC_localeconv() で現在のロケール情報を取得します。 2. 通貨記号、小数点、桁区切り文字、および正負の符号位置をロケール設定に基づいて決定します。 3. pg_abs_s64() を使用して値を絶対値化し、右から左へ向かって桁ごとに文字列バッファへ書き込みます。 4. 小数点位置（frac_digits）および桁区切り（mon_grouping）を考慮して記号を挿入します。 5. POSIX 仕様に基づき、sign_posn（符号の位置）、cs_precedes（通貨記号の前後）、sep_by_space（空白の有無）の各フラグに従って、通貨記号、符号、数値を結合して文字列を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の型出力システムから呼び出されます。
  • **呼び出し先**: PGLC_localeconv(), pg_abs_s64(), psprintf(), strlen(), memcpy() を使用します。

関連データ構造

  • Cash: int64 の別名。
  • struct lconv: ロケール情報を保持する標準構造体。

失敗モードと境界条件

  • **オーバーフロー**: 内部表現が int64 であるため、極端に大きな値は扱えません。
  • **ロケール依存**: lc_monetary の設定により出力結果が動的に変化します。
  • **メモリ**: psprintf() を使用して結果文字列を生成するため、メモリ不足時にはエラーが発生する可能性があります。

保守担当者向けメモ

  • frac_digits や mon_grouping の値が CHAR_MAX 等の影響で不正な値にならないよう、範囲チェックを行っています。
  • 通貨記号や符号の配置ロジックは POSIX の仕様に厳密に従っているため、変更時は POSIX 規格との整合性に注意してください。
