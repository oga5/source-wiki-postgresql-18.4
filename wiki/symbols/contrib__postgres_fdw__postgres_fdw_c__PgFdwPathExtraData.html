PgFdwPathExtraData

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/postgres_fdw.c:294

目的

外部テーブルに対するクエリのプランニング過程において、パス生成に必要な追加情報（ソート、制限、オフセット等）を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    PathTarget *target;
    bool        has_final_sort;
    bool        has_limit;
    double      limit_tuples;
    int64       count_est;
    int64       offset_est;
} PgFdwPathExtraData;

入力と出力

本構造体は、プランナが外部テーブルのパスを構築する際に、コスト見積もりやプラン作成関数へ渡される補助データとして機能します。

前提条件と不変条件

  • palloc0 を用いてゼロ初期化された状態で生成されることが一般的です（postgres_fdw.c:6999, 7233）。

アルゴリズムまたは振る舞い

  • プランニングの最適化フェーズにおいて、リモートサーバーへプッシュダウン可能なクエリの特性（ソートの有無や LIMIT/OFFSET の見積もり値）を格納します。
  • postgresGetForeignPaths 等の関数内で、リモート実行計画を構築するためのコンテキストとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/postgres_fdw.c 内のパス生成ロジック（6998行目および7232行目付近）でインスタンス化されます。
  • **呼び出し先**: 本構造体を受け取る関数（433行目、3115行目等）へ引数として渡されます。

関連データ構造

  • PathTarget: プランのターゲットリスト情報を保持。
  • PgFdwRelationInfo: 外部テーブルの統計情報や接続情報を保持する関連構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • count_est や offset_est の値が負数となる場合の挙動については、ソースコード上の明示的な制約は確認できません。

保守担当者向けメモ

  • 本構造体は postgres_fdw.c 内でのみ使用されるプライベートなデータ構造です。
  • プランニングのロジック変更に伴い、リモートクエリの最適化に必要なパラメータが増減した場合は、本構造体のメンバを更新してください。
