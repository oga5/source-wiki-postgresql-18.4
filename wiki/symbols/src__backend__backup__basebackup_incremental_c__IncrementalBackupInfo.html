IncrementalBackupInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup_incremental.c:74

目的

増分バックアップの準備フェーズにおいて、マニフェスト情報や変更ブロックの追跡に必要なメタデータを保持する主要な構造体です。

シグネチャまたは形状

struct IncrementalBackupInfo {
    MemoryContext mcxt;
    StringInfoData buf;
    List *manifest_wal_ranges;
    backup_file_hash *manifest_files;
    BlockRefTable *brtab;
    JsonManifestParseIncrementalState *inc_state;
};

入力と出力

  • **入力**: CreateIncrementalBackupInfo を通じて初期化され、AppendIncrementalManifestData によりマニフェストの断片データが入力されます。
  • **出力**: 最終的に BlockRefTable を介して、どのファイル・どのブロックが変更されたかという情報を提供します。

前提条件と不変条件

  • manifest_files は、過去のバックアップとの整合性チェック（サニティチェック）に使用されます。
  • brtab（ブロック参照テーブル）はメモリ上に保持されるため、大規模なデータベースではメモリ消費量が増大する可能性があります。

アルゴリズムまたは振る舞い

1. **マニフェスト解析**: inc_state を使用してJSON形式のマニフェストを増分的に解析し、buf に一時保存します。 2. **WAL範囲の抽出**: マニフェストからWAL範囲を抽出し、manifest_wal_ranges に格納します。 3. **変更ブロックの特定**: WALサマリーと照合し、brtab を構築して変更されたブロックを追跡します。 4. **整合性検証**: manifest_files を用いて、バックアップ対象ファイルの存在確認を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: 主に src/backend/backup/basebackup.c 内の SendBaseBackup 等から、増分バックアップの制御のために参照されます。
  • **呼び出し先**: src/backend/backup/basebackup_incremental.c 内の CreateIncrementalBackupInfo, AppendIncrementalManifestData, FinalizeIncrementalManifest, PrepareForIncrementalBackup 等の関数によって操作されます。

関連データ構造

  • BlockRefTable: 変更ブロックの追跡に使用。
  • backup_file_hash: マニフェスト内のファイル情報を保持。
  • JsonManifestParseIncrementalState: JSON解析の状態管理に使用。
  • StringInfoData: マニフェスト解析用の一時バッファ。

失敗モードと境界条件

  • **メモリ不足**: 大規模なデータベースで多数の変更ブロックが存在する場合、brtab がメモリを圧迫し、システムがメモリ不足に陥る可能性があります。
  • **整合性エラー**: マニフェスト内のファイル情報と実際のバックアップ状態が一致しない場合、サニティチェックにより異常が検出されます。

保守担当者向けメモ

  • 本構造体は増分バックアップの準備に特化しており、実際のデータ転送ロジックは含みません。
  • manifest_files はメモリを消費するため、将来的にオプション化や保持しない方針への変更が必要になる可能性があります。
  • brtab のメモリ効率は高い（1ブロックあたり約1ビット）ですが、極端に大規模な環境での挙動には注意してください。
