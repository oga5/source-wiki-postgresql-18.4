apw_init_shmem

定義

  • 種別: function
  • 定義箇所: contrib/pg_prewarm/autoprewarm.c:877

目的

自動プリウォーム機能で使用する共有メモリセグメントの確保および初期化を行います。

シグネチャまたは形状

static bool apw_init_shmem(void)

入力と出力

  • **入力**: なし
  • **出力**:
  • bool: 共有メモリセグメントが既に存在していた場合は true、新規作成された場合は false を返します。

前提条件と不変条件

  • GetNamedDSM を介して AutoPrewarmSharedState 構造体サイズの共有メモリが確保されます。
  • apw_init_state が初期化関数として指定されており、セグメント作成時に呼び出されます。

アルゴリズムまたは振る舞い

1. GetNamedDSMSegment を呼び出し、名前 "autoprewarm" で共有メモリセグメントを取得（または作成）します。 2. 取得したセグメント内のロック用トランチ（apw_state->lock.tranche）を LWLockRegisterTranche に登録し、PostgreSQL のロック管理システムに認識させます。 3. セグメントの存在有無を示すフラグを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c 内の複数箇所（初期化処理やワーカー起動フローなど）から呼び出されています。
  • **呼び出し先**:
  • GetNamedDSMSegment
  • LWLockRegisterTranche
  • apw_init_state（コールバックとして渡される）

関連データ構造

  • AutoPrewarmSharedState: 共有メモリ上に配置される制御構造体。
  • LWLock: 共有メモリ内のロック管理に使用。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、GetNamedDSMSegment 内部でエラーが発生し、プロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は autoprewarm 共有メモリのライフサイクル管理の起点です。トランチの登録を行っているため、共有メモリレイアウトを変更する際は apw_init_state との整合性に注意してください。
