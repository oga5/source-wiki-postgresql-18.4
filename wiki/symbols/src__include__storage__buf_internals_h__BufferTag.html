BufferTag

定義

  • 種別: struct
  • 定義箇所: src/include/storage/buf_internals.h:106

目的

共有バッファプール内のディスクページを一意に識別するためのタグ構造体です。バッファハッシュテーブルのキーとして使用されます。

シグネチャまたは形状

typedef struct buftag
{
    Oid         spcOid;         /* tablespace oid */
    Oid         dbOid;          /* database oid */
    RelFileNumber relNumber;    /* relation file number */
    ForkNumber  forkNum;        /* fork number */
    BlockNumber blockNum;       /* blknum relative to begin of reln */
} BufferTag;

入力と出力

  • **入力**: 構造体の各フィールド（spcOid, dbOid, relNumber, forkNum, blockNum）により、特定のデータベースオブジェクト内の特定のブロックを特定します。
  • **出力**: バッファハッシュテーブルのキーとして機能し、対応するバッファIDの検索、挿入、削除に使用されます。

前提条件と不変条件

  • バッファハッシュテーブルにおいて、この構造体は一意なページ識別子として機能します。
  • 比較操作（buffertag_comparator）やハッシュ計算（BufTableHashCode）の対象となります。

アルゴリズムまたは振る舞い

  • バッファマネージャは、このタグを用いて BufTableLookup を実行し、メモリ上にページが存在するかを確認します。
  • 存在しない場合は、ディスクから読み込み、BufTableInsert を用いてこのタグとバッファIDをハッシュテーブルに登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/buf_table.c および src/backend/storage/buffer/bufmgr.c 内のバッファ管理関数群。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • BufferDesc: バッファのメタデータを保持する構造体。BufferTag を内部で保持し、どのページが格納されているかを示します。

失敗モードと境界条件

  • 構造体内の各フィールドが正しく設定されていない場合、ハッシュテーブルの検索失敗や誤ったバッファの参照を招く可能性があります。
  • 構造体のサイズはハッシュテーブルのキーサイズとして使用されるため、定義変更時は注意が必要です。

保守担当者向けメモ

  • BufferTag はバッファ管理の根幹となる識別子です。フィールドの追加や変更を行う際は、buf_table.c 内のハッシュ計算ロジックや比較関数への影響を必ず確認してください。
