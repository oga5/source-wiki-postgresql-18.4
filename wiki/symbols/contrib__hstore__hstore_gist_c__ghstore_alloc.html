ghstore_alloc

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_gist.c:117

目的

GiST インデックス用の GISTTYPE 構造体をメモリ上に割り当て、初期化する。

シグネチャまたは形状

static GISTTYPE *ghstore_alloc(bool allistrue, int siglen, BITVECP sign)

入力と出力

  • **入力**:
  • allistrue (bool): 全てのビットを立てる（ALLISTRUE フラグを設定する）かどうか。
  • siglen (int): シグネチャのバイト長。
  • sign (BITVECP): 初期化に使用するビットベクトルのデータ（NULL の場合はゼロクリアされる）。
  • **出力**:
  • GISTTYPE *: 割り当ておよび初期化された GISTTYPE 構造体へのポインタ。

前提条件と不変条件

  • palloc を使用するため、PostgreSQL のメモリコンテキスト内で呼び出される必要がある。
  • siglen は CALCGTSIZE マクロが許容する範囲内であること。

アルゴリズムまたは振る舞い

1. allistrue フラグに基づき、CALCGTSIZE マクロを使用して必要なメモリサイズを計算する。 2. palloc でメモリを確保し、SET_VARSIZE で PostgreSQL の可変長型ヘッダを設定する。 3. res->flag に allistrue に応じたフラグを設定する。 4. allistrue が false の場合:

  • sign が提供されていれば memcpy でシグネチャ領域をコピーする。
  • sign が NULL であれば memset でシグネチャ領域を 0 で初期化する。

    • 5. 構築した GISTTYPE ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_gist.c 内の各関数（ghstore_compress, ghstore_union, ghstore_picksplit 等）から、インデックスノードの新規作成時に呼び出される。
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理)
  • memcpy, memset (標準 C ライブラリ)

関連データ構造

  • GISTTYPE: hstore の GiST インデックスエントリを保持する構造体。
  • BITVECP: シグネチャ（ビットベクトル）を指すポインタ型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • siglen が不適切な場合、CALCGTSIZE の計算結果や memcpy/memset の範囲外アクセスに注意が必要（ソースコード上での明示的なバリデーションは本関数内にはない）。

保守担当者向けメモ

  • 本関数は hstore GiST インデックスのメモリレイアウトを決定する重要な関数である。構造体のサイズ計算やフラグ管理を変更する場合は、hstore.h で定義されている GISTTYPE 構造体の定義と整合性を保つこと。
