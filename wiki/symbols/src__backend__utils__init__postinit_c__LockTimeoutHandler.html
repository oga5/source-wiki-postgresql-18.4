LockTimeoutHandler

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:1381

目的

ロック取得待ちがタイムアウトした際に、該当プロセスに対して SIGINT シグナルを送信し、処理を中断させるためのハンドラ関数です。

シグネチャまたは形状

static void LockTimeoutHandler(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてシグナルを送信）

前提条件と不変条件

  • MyProcPid が有効なプロセスIDを保持していること。
  • RegisterTimeout を介して LOCK_TIMEOUT イベントに登録されていること。

アルゴリズムまたは振る舞い

1. HAVE_SETSID が定義されている環境では、プロセスグループ全体に対して kill(-MyProcPid, SIGINT) を実行し、グループ内の全プロセスにシグナルを伝播させます。 2. 自身のプロセスに対して kill(MyProcPid, SIGINT) を実行し、割り込みを発生させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c 内の RegisterTimeout(LOCK_TIMEOUT, LockTimeoutHandler);（初期化時）
  • 呼び出し先: kill() システムコール

関連データ構造

  • MyProcPid: 現在のバックエンドプロセスのIDを保持するグローバル変数。

失敗モードと境界条件

  • kill システムコールが失敗した場合、シグナルは送信されません。
  • プロセスグループへのシグナル送信は、HAVE_SETSID が定義されていない環境ではスキップされます。

保守担当者向けメモ

  • 本関数は SIGINT を送信することでトランザクションのキャンセルを誘発します。シグナルハンドラ内での実行となるため、非同期シグナル安全な操作のみを行う必要があります。
