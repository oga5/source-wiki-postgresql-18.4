dsm_pin_segment

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm.c:954

目的

動的共有メモリ（DSM）セグメントを「ピン留め」し、セッション終了後もポストマスターのシャットダウンまたは明示的な dsm_unpin_segment 呼び出しまでセグメントが存続するように保証する。

シグネチャまたは形状

void dsm_pin_segment(dsm_segment *seg)

入力と出力

  • **入力**: dsm_segment *seg - ピン留め対象のセグメント構造体へのポインタ。
  • **出力**: なし（副作用として共有メモリ上の制御情報を更新）。

前提条件と不変条件

  • 対象のセグメントが既にピン留め状態（pinned が true）であってはならない。
  • DynamicSharedMemoryControlLock を排他モードで取得して実行される。

アルゴリズムまたは振る舞い

1. DynamicSharedMemoryControlLock を排他取得する。 2. 指定されたセグメントの制御スロットを確認し、既にピン留めされている場合はエラーを投げる。 3. メイン領域の DSM ハンドルではない場合、dsm_impl_pin_segment を呼び出し、OS レベルでの永続化処理を行う。 4. 共有メモリ上の制御アイテムにおいて、pinned フラグを true に設定し、参照カウント（refcnt）をインクリメントする。 5. OS 固有のハンドル情報を impl_private_pm_handle に保存する。 6. ロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/dsm_registry.c: dsm_pin_segment(seg);
  • src/backend/utils/mmgr/dsa.c: dsm_pin_segment(segment); (2箇所)
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease
  • elog (エラー発生時)
  • is_main_region_dsm_handle
  • dsm_impl_pin_segment

関連データ構造

  • dsm_segment: セグメントのローカル管理構造体。
  • dsm_control_item: 共有メモリ上の制御領域。
  • DynamicSharedMemoryControlLock: 制御領域保護用ロック。

失敗モードと境界条件

  • 既にピン留めされているセグメントに対して呼び出した場合、ERROR が発生する。

保守担当者向けメモ

  • 本関数は、セグメントの生存期間をセッションの境界を超えて延長する重要な役割を持つ。
  • dsm_impl_pin_segment は OS 依存の実装を伴うため、プラットフォーム固有の挙動に注意が必要である。
