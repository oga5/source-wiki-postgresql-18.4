pgstat_report_inj_fixed

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats_fixed.c:140

目的

インジェクションポイントに関連する固定項目（attach, detach, run, cached, loaded）の統計カウンタを共有メモリ上で更新する。

シグネチャまたは形状

void pgstat_report_inj_fixed(uint32 numattach,
                             uint32 numdetach,
                             uint32 numrun,
                             uint32 numcached,
                             uint32 numloaded)

入力と出力

  • **入力**:
  • numattach, numdetach, numrun, numcached, numloaded: 各統計項目に加算する増分値（uint32）。
  • **出力**: なし（共有メモリ上の統計値を直接更新）。

前提条件と不変条件

  • inj_fixed_loaded および inj_stats_enabled が真であること。これらが偽の場合、関数は何もせず即座に復帰する。
  • 共有メモリ上の統計データ構造 PgStatShared_InjectionPointFixed が初期化済みであること。

アルゴリズムまたは振る舞い

1. 統計機能が無効な場合は早期終了する。 2. pgstat_get_custom_shmem_data を使用して共有メモリ上の統計領域を取得する。 3. LWLock（stats_shmem->lock）を排他モードで取得し、排他制御を行う。 4. pgstat_begin_changecount_write および pgstat_end_changecount_write で囲まれた領域内で、各統計カウンタに引数の値を加算する。 5. LWLock を解放して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/injection_points/injection_points.c 内の各インジェクションポイント処理箇所（統計の増分が必要なタイミングで呼び出される）。
  • **呼び出し先**:
  • pgstat_get_custom_shmem_data
  • LWLockAcquire / LWLockRelease
  • pgstat_begin_changecount_write / pgstat_end_changecount_write

関連データ構造

  • PgStatShared_InjectionPointFixed: 統計カウンタとロック、変更カウントを保持する共有メモリ構造体。

失敗モードと境界条件

  • 統計機能が無効な場合、更新はスキップされる。
  • 共有メモリの取得に失敗した場合の挙動は、pgstat_get_custom_shmem_data の実装に依存する（本関数内では明示的なエラーハンドリングなし）。
  • 高頻度で呼び出される場合、LWLock による競合が発生する可能性がある。

保守担当者向けメモ

  • 本関数は LWLock を使用しているため、パフォーマンスへの影響を考慮すること。
  • 加算処理はアトミックではなくロックによって保護されているため、ロック取得の順序や範囲を変更する際は注意が必要である。
