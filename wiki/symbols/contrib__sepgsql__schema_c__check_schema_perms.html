check_schema_perms

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/schema.c:178

目的

PostgreSQL のスキーマ（pg_namespace）に対する操作が、SELinux ポリシーによって許可されているかを検証する内部ユーティリティ関数です。

シグネチャまたは形状

static bool check_schema_perms(Oid namespaceId, uint32 required, bool abort_on_violation)

入力と出力

  • **入力**:
  • namespaceId (Oid): 対象スキーマの識別子。
  • required (uint32): 要求される権限ビットマスク（例: SEPG_DB_SCHEMA__SETATTR）。
  • abort_on_violation (bool): 権限チェック失敗時にエラーを投げるか（true）、結果を返すか（false）のフラグ。
  • **出力**:
  • bool: 権限が許可されている場合は true、拒否された場合は false。

前提条件と不変条件

  • namespaceId は有効な pg_namespace の OID である必要があります。
  • getObjectIdentity を使用して対象スキーマの識別名を取得するため、システムカタログへのアクセスが可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. 指定された namespaceId を用いて ObjectAddress 構造体を構築します（classId には NamespaceRelationId を設定）。 2. getObjectIdentity を呼び出し、監査ログ用のスキーマ名文字列を取得します。 3. sepgsql_avc_check_perms を呼び出し、SELinux の AVC（Access Vector Cache）に対して SEPG_CLASS_DB_SCHEMA クラスの権限チェックを実行します。 4. 取得した監査名文字列を pfree で解放します。 5. sepgsql_avc_check_perms の結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/schema.c 内の各スキーマ操作フック（sepgsql_schema_setattr, sepgsql_schema_add_name, sepgsql_schema_remove_name 等）。
  • **呼び出し先**:
  • getObjectIdentity (PostgreSQL 内部関数)
  • sepgsql_avc_check_perms (contrib/sepgsql/uavc.c)
  • pfree (PostgreSQL 内部関数)

関連データ構造

  • ObjectAddress: PostgreSQL のオブジェクトを特定するための構造体。
  • NamespaceRelationId: pg_namespace カタログの OID。

失敗モードと境界条件

  • abort_on_violation が true の場合、権限チェックに失敗すると sepgsql_avc_check_perms 経由で PostgreSQL のエラー処理が呼び出され、トランザクションが中断される可能性があります。
  • getObjectIdentity が失敗した場合の挙動については、本関数内では明示的なエラーハンドリングを行っていません。

保守担当者向けメモ

  • 本関数は sepgsql_schema_... 系関数の権限チェックを共通化する目的で実装されています。
  • 監査ログにスキーマ名を含めるため getObjectIdentity を使用していますが、パフォーマンスが懸念される場合は呼び出し頻度に注意してください。
