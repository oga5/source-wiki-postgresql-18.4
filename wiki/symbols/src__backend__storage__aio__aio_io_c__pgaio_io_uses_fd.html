pgaio_io_uses_fd

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_io.c:196

目的

指定された非同期I/Oハンドル（PgAioHandle）が、特定のファイル記述子（fd）を使用してI/O操作を行っているかどうかを判定する。

シグネチャまたは形状

bool pgaio_io_uses_fd(PgAioHandle *ioh, int fd)

入力と出力

  • **入力**:
  • ioh: 判定対象の PgAioHandle 構造体へのポインタ。
  • fd: 比較対象のファイル記述子。
  • **出力**:
  • bool: 指定された fd がハンドル内の操作対象ファイルと一致する場合は true、それ以外の場合は false を返す。

前提条件と不変条件

  • ioh->state が PGAIO_HS_DEFINED 以上であること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. ハンドルの状態が定義済みであることを確認する。 2. ハンドルの操作タイプ（ioh->op）を評価する。

  • PGAIO_OP_READV の場合: ioh->op_data.read.fd と fd を比較する。
  • PGAIO_OP_WRITEV の場合: ioh->op_data.write.fd と fd を比較する。
  • PGAIO_OP_INVALID の場合: false を返す。

    • 3. 上記以外のケース、または操作タイプが一致しない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio.c:1265
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わない。

関連データ構造

  • PgAioHandle: 非同期I/O操作の状態とパラメータを保持する構造体。
  • PgAioOp: 操作の種類を定義する列挙型。

失敗モードと境界条件

  • ioh が PGAIO_HS_DEFINED 未満の状態である場合、アサーション違反によりプロセスが停止する可能性がある。
  • PGAIO_OP_INVALID 型のハンドルに対しては、常に false を返す。

保守担当者向けメモ

  • 本関数は、特定のファイル記述子に関連付けられた保留中のI/O操作を検索・管理する際に使用される。
  • ioh->op の拡張に伴い、新しい操作タイプが追加された場合は switch 文の更新が必要となる。
