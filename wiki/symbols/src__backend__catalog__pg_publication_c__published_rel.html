published_rel

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/pg_publication.c:44

目的

論理レプリケーションにおいて、特定のパブリケーションと、そのパブリケーションに含まれるリレーション（テーブル）の紐付け情報をメモリ上で保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	Oid			relid;			/* 公開されるテーブルのOID */
	Oid			pubid;			/* このテーブルを公開するパブリケーションのOID */
} published_rel;

入力と出力

本構造体は関数引数や戻り値としてリスト（List）の要素として扱われることが多く、主にカタログから取得した情報をメモリ上に展開する際に利用されます。

前提条件と不変条件

  • relid および pubid は、それぞれ pg_class および pg_publication システムカタログ内の有効な OID を参照している必要があります。

アルゴリズムまたは振る舞い

  • pg_publication.c 内の関数において、パブリケーションに関連付けられたテーブルのリストを構築する際、palloc でメモリを確保し、この構造体に値を格納してリストに追加する処理が行われます。
  • 取得したリストを走査する際、各要素を published_rel * にキャストして relid や pubid を参照します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_publication.c 内の関数（GetPublicationRelations や pg_get_publication_tables 等）が、カタログスキャン結果をこの構造体にマッピングして利用しています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • List: published_rel のインスタンスを複数保持するために使用されます。
  • pg_publication_rel: 本構造体が表現する情報の永続化先であるシステムカタログ。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • 無効な OID を保持した場合、後続のカタログ検索処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はメモリ上の作業用データ構造です。カタログ定義（pg_publication_rel）に変更を加える際は、この構造体が必要とするフィールドが不足していないか確認してください。
  • pg_publication.c 内のリスト操作において、キャストの整合性が保たれていることを確認してください。
