ExceptionalCondition

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/assert.c:29

目的

PostgreSQLのアサーション失敗時に、低レイヤーでエラー情報を記録し、プロセスを強制終了させるための関数です。elog.c などの高レベルなエラー報告システムに依存せず、システムが深刻な状態でも確実に動作するよう設計されています。

シグネチャまたは形状

void ExceptionalCondition(const char *conditionName, const char *fileName, int lineNumber)

入力と出力

  • **入力**:
  • conditionName: 失敗した条件式を表す文字列。
  • fileName: アサーションが発生したソースファイル名。
  • lineNumber: アサーションが発生した行番号。
  • **出力**: なし（プロセスが終了するため）。

前提条件と不変条件

  • postgres.h で定義される PointerIsValid マクロにより、引数の有効性が検証されます。
  • elog() などの複雑なサブシステムは使用せず、標準エラー出力（stderr）への書き出しのみを行います。

アルゴリズムまたは振る舞い

1. **エラー報告**: write_stderr を使用して、失敗した条件式、ファイル名、行番号、および現在のPIDを標準エラー出力へ書き出します。引数が無効な場合は警告を出力します。 2. **フラッシュ**: fflush(stderr) を呼び出し、メッセージの出力を確定させます。 3. **バックトレース**: HAVE_BACKTRACE_SYMBOLS が定義されている環境では、backtrace() を用いてコールスタックを取得し、stderr に出力します。 4. **デバッグ待機**: SLEEP_ON_ASSERT が定義されている場合、sleep(1000000) を実行してプロセスを停止し、デバッガのアタッチを待機します。 5. **終了**: abort() を呼び出し、コアダンプを生成してプロセスを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/c.h: Assert() マクロ経由で呼び出されます。
  • src/backend/utils/error/elog.c: pg_re_throw の異常系などで直接呼び出されるケースがあります。
  • **呼び出し先**:
  • write_stderr (PostgreSQL内部関数)
  • getpid() (システムコール)
  • backtrace() / backtrace_symbols_fd() (環境依存)
  • sleep() (環境依存)
  • abort() (標準ライブラリ)

関連データ構造

  • 特筆すべきデータ構造は使用していません。

失敗モードと境界条件

  • 引数（conditionName または fileName）が NULL または無効なポインタである場合、引数エラーを示す最小限のメッセージを出力して終了します。
  • HAVE_BACKTRACE_SYMBOLS が未定義の環境では、バックトレースは出力されません。

保守担当者向けメモ

  • 本関数は、他のサブシステムが機能不全に陥っている状況下でも動作する必要があるため、依存関係を最小限に保つ必要があります。
  • elog() への依存は厳禁です。
  • SLEEP_ON_ASSERT は開発用であり、本番環境では無効化されることが前提です。
