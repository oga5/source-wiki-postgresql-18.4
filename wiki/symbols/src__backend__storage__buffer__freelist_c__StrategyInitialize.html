StrategyInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/freelist.c:473

目的

バッファ置換戦略の共有メモリ状態を初期化し、バッファ検索用ハッシュテーブルを構築する。

シグネチャまたは形状

void StrategyInitialize(bool init)

入力と出力

  • **入力**:
  • init (bool): 呼び出し元が初期化プロセス（Postmaster等）であるかを示すフラグ。
  • **出力**: なし（副作用として共有メモリおよびグローバル変数を設定）。

前提条件と不変条件

  • BufferManagerShmemInit() が事前に実行され、フリーバッファのリストが準備されていること。
  • init が true の場合、共有メモリ構造体が未作成である必要がある。

アルゴリズムまたは振る舞い

1. InitBufTable を呼び出し、バッファ検索用のハッシュテーブルを NBuffers + NUM_BUFFER_PARTITIONS のサイズで確保する。 2. ShmemInitStruct を使用して、共有メモリ上に BufferStrategyControl 構造体を割り当てる。 3. 初回実行時（!found）の場合:

  • buffer_strategy_lock を初期化する。
  • フリーリストの範囲（firstFreeBuffer から lastFreeBuffer）を設定する。
  • クロック掃引用のポインタ nextVictimBuffer を 0 に初期化する。
  • 統計情報（completePasses, numBufferAllocs）をリセットし、bgwprocno を -1 に設定する。

    • 4. 実行後、init フラグと found（共有メモリの既存状態）の整合性を Assert で検証する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/buf_init.c:148
  • **呼び出し先**: InitBufTable, ShmemInitStruct, SpinLockInit, pg_atomic_init_u32

関連データ構造

  • BufferStrategyControl: バッファ置換戦略のグローバル状態を保持する構造体。

失敗モードと境界条件

  • ハッシュテーブルのサイズ不足は許容されないため、十分なエントリ数を確保する設計となっている。
  • init フラグと共有メモリの存在状態が矛盾する場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は共有メモリの初期化フェーズで一度だけ実行されることを想定している。
  • BufferStrategyControl の各フィールドは、後続の StrategyGetBuffer 等の関数でアトミック操作やスピンロックを用いて安全にアクセスされる。
