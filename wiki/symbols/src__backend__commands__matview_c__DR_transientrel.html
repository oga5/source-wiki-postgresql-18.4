DR_transientrel

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/matview.c:45

目的

マテリアライズド・ビュー（MV）の更新処理において、クエリ結果を一時テーブルへ書き込むための DestReceiver 派生構造体です。

シグネチャまたは形状

typedef struct
{
	DestReceiver pub;
	Oid			transientoid;
	Relation	transientrel;
	CommandId	output_cid;
	int			ti_options;
	BulkInsertState bistate;
} DR_transientrel;

入力と出力

  • **入力**: DestReceiver インターフェースを介して受け取るタプルデータ。
  • **出力**: transientrel で指定された一時テーブルへのタプル挿入。

前提条件と不変条件

  • pub フィールドは DestReceiver として初期化されている必要があります。
  • transientrel および bistate などの内部フィールドは、transientrel_startup の呼び出しによって適切に設定される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: transientrel_startup を通じて、書き込み対象のリレーションを開き、バルク挿入状態（BulkInsertState）を準備します。 2. **データ受信**: DestReceiver のインターフェースを通じてクエリ結果のタプルを受け取り、transientrel へ挿入します。 3. **終了**: transientrel_shutdown を通じて、リレーションを閉じ、バルク挿入状態を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/matview.c 内の CreateTransientRelDestReceiver 等でインスタンス化され、refresh_matview_datafill 等から操作されます。
  • **呼び出し先**: table_tuple_insert などのテーブルアクセス層の関数を呼び出し、タプルを永続化します。

関連データ構造

  • DestReceiver: 本構造体の基底となるインターフェース。
  • Relation: 書き込み先の一時テーブルを指す構造体。
  • BulkInsertState: 大量挿入時のパフォーマンス最適化のための状態管理構造体。

失敗モードと境界条件

  • 一時テーブルのオープン失敗や、権限不足による挿入失敗が発生する可能性があります。
  • transientrel_startup が呼び出される前にデータ受信を試みた場合、未定義の動作となります。

保守担当者向けメモ

  • 本構造体は matview.c 内の REFRESH MATERIALIZED VIEW 処理専用です。
  • transientrel_startup、transientrel_receive、transientrel_shutdown の各関数が、この構造体のライフサイクルを管理しています。変更時はこれら関数との整合性に注意してください。
