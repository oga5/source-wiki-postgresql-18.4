ginadjustmembers

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginvalidate.c:268

目的

GINインデックスの演算子族（opfamily）および演算子クラス（opclass）に登録される演算子やサポート関数の依存関係（ハード/ソフト）をカタログ登録前に調整します。

シグネチャまたは形状

void ginadjustmembers(Oid opfamilyoid, Oid opclassoid, List *operators, List *functions)

入力と出力

  • **入力**:
  • opfamilyoid: 対象の演算子族のOID。
  • opclassoid: 対象の演算子クラスのOID。
  • operators: 登録対象の演算子リスト（OpFamilyMember構造体のリスト）。
  • functions: 登録対象のサポート関数リスト（OpFamilyMember構造体のリスト）。
  • **出力**: なし（引数として渡されたリスト内の構造体メンバーを直接更新します）。

前提条件と不変条件

  • 演算子メンバーは常にソフト依存（ref_is_hard = false）として扱われます。
  • 必須サポート関数（extractValue, extractQuery）はハード依存（ref_is_hard = true）として扱われます。
  • オプションのサポート関数はソフト依存として扱われます。

アルゴリズムまたは振る舞い

1. **演算子の処理**: operators リストを走査し、すべての演算子に対してハード依存を無効化し、演算子族へのソフト依存を設定します。 2. **サポート関数の処理**: functions リストを走査し、各関数の番号（op->number）に基づいて依存関係を決定します。

  • 必須関数（GIN_EXTRACTVALUE_PROC, GIN_EXTRACTQUERY_PROC）: ハード依存に設定。
  • オプション関数（GIN_COMPARE_PROC, GIN_CONSISTENT_PROC 等）: ソフト依存に設定し、演算子族を対象とします。
  • 未定義の関数番号: ERROR を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginutil.c: amroutine->amadjustmembers に関数ポインタとして代入されています。
  • **呼び出し先**:
  • ereport: エラー発生時に呼び出されます。

関連データ構造

  • OpFamilyMember: 演算子や関数の依存関係情報を保持する構造体。
  • GIN_..._PROC: サポート関数を識別するための定数群。

失敗モードと境界条件

  • サポート関数番号がGINで定義されていない値である場合、ERRCODE_INVALID_OBJECT_DEFINITION エラーが発生します。

保守担当者向けメモ

  • 本関数は ALTER OPERATOR FAMILY 等の実行時に、依存関係の整合性を保つために重要です。
  • GINの演算子クラスは通常演算子族を共有しないため、依存先を演算子族に固定する実装となっています。
