JsonEncodeDateTime

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/json.c:309

目的

PostgreSQLの各種日時型（DATE, TIME, TIMETZ, TIMESTAMP, TIMESTAMPTZ）の Datum 値を、JSON形式の文字列に変換する。

シグネチャまたは形状

char *JsonEncodeDateTime(char *buf, Datum value, Oid typid, const int *tzp)

入力と出力

  • **入力**:
  • buf: 出力用バッファ（NULL の場合、内部で palloc される）。
  • value: 変換対象の Datum 値。
  • typid: 型のOID（DATEOID, TIMEOID, TIMETZOID, TIMESTAMPOID, TIMESTAMPTZOID）。
  • tzp: タイムゾーンオフセット（秒単位、TIMESTAMPTZOID で使用）。
  • **出力**:
  • 変換後の文字列を格納した char *。

前提条件と不変条件

  • buf が NULL の場合、MAXDATELEN + 1 バイトのメモリが割り当てられる。
  • すべての変換において、PostgreSQLの DateStyle 設定に関わらず、XSD形式（USE_XSD_DATES）が強制される。

アルゴリズムまたは振る舞い

1. typid に基づき分岐し、各型に応じた変換関数（date_out や timestamp2tm 等のロジック）を呼び出す。 2. 有限でない値（infinity 等）に対しては、専用の EncodeSpecial... 関数を使用する。 3. TIMESTAMPTZ の場合、tzp が指定されていればタイムゾーンシフトを適用し、指定がなければシステム標準のタイムゾーンを使用する。 4. 最終的に EncodeDateOnly、EncodeTimeOnly、または EncodeDateTime を呼び出し、buf に文字列を書き込む。 5. 未知の typid が渡された場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/json.c (複数箇所)
  • src/backend/utils/adt/jsonb.c
  • src/backend/utils/adt/jsonb_util.c
  • src/backend/utils/adt/jsonpath_exec.c
  • **呼び出し先**:
  • palloc
  • DatumGet... 系マクロ
  • j2date, date2tm, time2tm, timetz2tm, timestamp2tm
  • EncodeSpecialDate, EncodeSpecialTimestamp, EncodeDateOnly, EncodeTimeOnly, EncodeDateTime
  • ereport / elog

関連データ構造

  • Datum: PostgreSQLの汎用データ表現。
  • Oid: 型識別子。
  • pg_tm: 日時分解用構造体。

失敗モードと境界条件

  • typid がサポート対象外の場合: elog(ERROR) により処理が中断される。
  • 日時値が範囲外の場合: ERRCODE_DATETIME_VALUE_OUT_OF_RANGE エラーが発生する。
  • メモリ不足時: palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は json および jsonb 型の両方から利用されるため、変更時は両方のシリアライズ結果に影響を与える点に注意すること。
  • USE_XSD_DATES フラグがハードコードされているため、JSON出力形式の仕様変更時にはこのフラグの扱いを確認すること。
