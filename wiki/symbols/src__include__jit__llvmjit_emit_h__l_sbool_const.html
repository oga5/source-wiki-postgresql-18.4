l_sbool_const

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/jit/llvmjit_emit.h:92

目的

PostgreSQL の JIT コンパイルにおいて、C 言語の bool 値を LLVM IR 上のストレージ用ブール型定数（TypeStorageBool）に変換して生成するヘルパー関数です。

シグネチャまたは形状

static inline LLVMValueRef l_sbool_const(bool i)

入力と出力

  • **入力**: bool i（変換対象の真偽値）
  • **出力**: LLVMValueRef（LLVM IR における定数表現）

前提条件と不変条件

  • TypeStorageBool が LLVM コンテキスト内で適切に定義されている必要があります。
  • 本関数は USE_LLVM が定義されている環境でのみ有効です。

アルゴリズムまたは振る舞い

1. 引数 i を int 型にキャストします。 2. LLVM C API の LLVMConstInt を呼び出し、TypeStorageBool 型の定数値を生成します。 3. 符号なし（false）として定数値を構築し、その参照を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/llvmjit_expr.c 内の複数の箇所で、式評価の定数生成や NULL フラグのセットに使用されています。
  • **呼び出し先**: LLVMConstInt (LLVM C API)

関連データ構造

  • LLVMValueRef: LLVM IR の値を表す不透明なポインタ型。
  • TypeStorageBool: llvmjit.h 等で定義される、ストレージ用のブール型定数。

失敗モードと境界条件

  • LLVM のコンテキストが初期化されていない場合や、TypeStorageBool が未定義の状態で呼び出された場合は未定義の動作となります。

保守担当者向けメモ

  • 本関数はインライン関数であり、呼び出し側のコンテキストで展開されます。
  • LLVM C API のバージョンアップに伴い、LLVMConstInt の引数仕様に変更がないか注意してください。
  • 構造体フィールドへのアクセス等で NULL 状態を表現する際に頻繁に使用されるため、変更時は式評価ロジックへの影響を確認してください。
