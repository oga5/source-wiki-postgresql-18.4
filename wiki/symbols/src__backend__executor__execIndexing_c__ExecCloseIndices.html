ExecCloseIndices

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execIndexing.c:238

目的

テーブルに関連付けられたすべてのインデックスをクローズし、保持していたロックを解放する。

シグネチャまたは形状

void ExecCloseIndices(ResultRelInfo *resultRelInfo)

入力と出力

  • **入力**: ResultRelInfo *resultRelInfo（対象リレーションのインデックス情報および記述子を保持する構造体）
  • **出力**: なし（副作用としてリレーションのクローズとポインタの無効化を行う）

前提条件と不変条件

  • resultRelInfo は有効なポインタであること。
  • ExecOpenIndices によってインデックスがオープンされ、ロックが取得済みであること。
  • 本関数を同一の resultRelInfo に対して二度呼び出すことはできない（indexDescs[i] != NULL のアサーションにより失敗する）。

アルゴリズムまたは振る舞い

1. resultRelInfo からインデックス数、インデックス記述子配列、インデックス情報配列を取得する。 2. 各インデックスに対して以下の処理を順次実行する：

  • index_insert_cleanup を呼び出し、インデックス挿入後のクリーンアップ処理を実行する。
  • index_close を呼び出し、RowExclusiveLock を解放する。
  • インデックス記述子配列の該当要素を NULL に設定し、クローズ済みであることを示す。

    • 3. IndexInfo データ構造や配列自体のメモリ解放は行わない（これらは FreeExecutorState で処理される前提）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/indexing.c
  • src/backend/executor/execMain.c
  • src/backend/executor/execPartition.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • index_insert_cleanup
  • index_close

関連データ構造

  • ResultRelInfo: インデックス記述子 (ri_IndexRelationDescs) およびインデックス情報 (ri_IndexRelationInfo) を保持する。

失敗モードと境界条件

  • indexDescs[i] が NULL の場合、アサーション違反によりプロセスが異常終了する（二重クローズの防止）。

保守担当者向けメモ

  • 本関数はリソースのクローズのみを担う。メモリ管理は実行状態（EState）のライフサイクルに依存しているため、本関数内で IndexInfo 等を解放してはならない。
