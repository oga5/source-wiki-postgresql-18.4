pairingheap

定義

  • 種別: struct
  • 定義箇所: src/include/lib/pairingheap.h:71

目的

ペアリングヒープ（自己調整型優先度付きキュー）の管理構造体。ヒープのルートノードおよび優先度比較のためのコールバック関数を保持する。

シグネチャまたは形状

typedef struct pairingheap
{
    pairingheap_comparator ph_compare; /* 比較関数 */
    void       *ph_arg;                /* 比較関数に渡す不透明な引数 */
    pairingheap_node *ph_root;         /* ヒープの現在のルート */
} pairingheap;

入力と出力

  • 入力: pairingheap_allocate により初期化され、比較関数とオプションの引数を受け取る。
  • 出力: ph_root を通じてヒープの最小（または最大）要素へのアクセスを提供し、pairingheap_free により構造体自体を解放する。

前提条件と不変条件

  • 侵入型（Intrusive）データ構造であり、管理対象の構造体は pairingheap_node をメンバとして含む必要がある。
  • ph_compare は、ノード間の優先度を決定するために常に使用される。
  • pairingheap_free はヒープ構造体のみを解放し、ヒープ内の各ノードのメモリ管理は呼び出し側に委ねられる。

アルゴリズムまたは振る舞い

  • 挿入: pairingheap_add により、新しいノードを既存のルートとマージする。
  • 削除: pairingheap_remove_first によりルートを取り出し、残された子ノード群を再帰的にマージして新しいルートを再構築する。
  • 比較: 優先度判定は ph_compare を介して行われる。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）: src/backend/access/gist/gistget.c および src/backend/executor/nodeIndexscan.c が本ヘッダーをインクルードしている。
  • 呼び出し先: src/backend/lib/pairingheap.c 内の関数群（merge, merge_children, pairingheap_free 等）が本構造体を操作する。

関連データ構造

  • pairingheap_node: ヒープに格納される個々のノード。
  • pairingheap_comparator: 優先度比較のための関数型。
  • binaryheap: 代替となるヒープ実装。

失敗モードと境界条件

  • pairingheap_free を呼び出しても、ヒープ内に残存するノードのメモリは解放されないため、メモリリークに注意が必要。
  • 空のヒープに対する操作の挙動は、各操作関数（remove_first 等）の実装に依存する。

保守担当者向けメモ

  • PAIRINGHEAP_DEBUG を定義することで pairingheap_dump を有効化し、デバッグ時にヒープ状態を可視化できる。
  • 本構造体は侵入型であるため、ノードのライフサイクル管理は呼び出し側が責任を持つこと。
