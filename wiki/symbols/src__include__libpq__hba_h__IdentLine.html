IdentLine

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/hba.h:145

目的

pg_ident.conf ファイルの各行を表現し、OSユーザー名とPostgreSQLデータベースユーザー名のマッピングルールを保持するためのデータ構造です。

シグネチャまたは形状

typedef struct IdentLine
{
	int			linenumber;

	char	   *usermap;
	AuthToken  *system_user;
	AuthToken  *pg_user;
} IdentLine;

入力と出力

  • **入力**: pg_ident.conf ファイル内の各行の解析結果（行番号、ユーザーマップ名、システムユーザー名、DBユーザー名）。
  • **出力**: 認証プロセスにおいて、特定の接続がマッピングルールに適合するかを判定するための構造体インスタンス。

前提条件と不変条件

  • linenumber は設定ファイル内の行位置を保持します。
  • usermap はマッピング名を指定します。
  • system_user および pg_user は AuthToken 型として定義され、それぞれOS側とDB側のユーザー名（またはパターン）を格納します。

アルゴリズムまたは振る舞い

pg_ident.conf の読み込み時に、各行がこの構造体にパースされます。認証時には、この構造体のリストが走査され、提供されたOSユーザー名とDBユーザー名が system_user および pg_user と一致するかどうかが検証されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c 内の pg_ident.conf 解析処理および check_ident_usermap 関数。
  • **呼び出し先**: src/backend/utils/adt/hbafuncs.c において、設定内容の表示や検証のために参照されます。

関連データ構造

  • AuthToken: ユーザー名やパターンを保持するために使用されます。
  • List (PostgreSQL標準): IdentLine のレコード群を管理するために使用されます。

失敗モードと境界条件

  • 設定ファイルの構文エラーにより、パース時に IdentLine の生成が失敗する可能性があります。
  • usermap 名が一致しない場合や、ユーザー名が正規表現等で指定されている場合の照合失敗が境界条件となります。

保守担当者向けメモ

  • IdentLine は pg_ident.conf の1行に対応します。構造体内のフィールドを変更する場合は、hba.c 内のパースロジックおよび hbafuncs.c 内の表示ロジックを同期して更新してください。
