gbt_inetlt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_inet.c:48

目的

GiSTインデックスにおける inet 型の「より小さい（less than）」比較操作を提供します。

シグネチャまたは形状

static bool gbt_inetlt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: double 型のスカラ値へのポインタ（inet 型から変換された値）。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • a が b より小さい場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な double 型の値を指している必要があります。
  • 本関数は inet 型を double に変換した後の比較を行うため、変換後の値の順序関係が inet 型の本来の順序を保持していることを前提としています。

アルゴリズムまたは振る舞い

1. a が指す double 値を読み取ります。 2. b が指す double 値を読み取ります。 3. *a < *b の比較結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_inet.c 内の tinfo 構造体（gbtree_ninfo）の初期化時に、比較関数として参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • inetKEY: inet 型のインデックス格納用構造体。
  • gbtree_ninfo: btree_utils_num.c で定義される、数値ベースの比較ロジックを保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。
  • inet 型から double への変換精度に依存するため、非常に大きなネットワーク範囲や複雑なサブネットマスクでは、比較結果が期待通りにならない可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する汎用的な数値比較ロジックの一部として利用されます。
  • inet 型の比較ロジックを変更する場合は、convert_network_to_scalar（src/backend/utils/adt/network.c）による変換ロジックとの整合性に注意してください。
