PQdefaultSSLKeyPassHook_OpenSSL

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure.c:472

目的

OpenSSLを使用したSSL接続において、秘密鍵のパスフレーズを要求された際に呼び出されるデフォルトのフック関数です。

シグネチャまたは形状

int PQdefaultSSLKeyPassHook_OpenSSL(char *buf, int size, PGconn *conn);

入力と出力

  • buf: パスフレーズを格納するためのバッファ。
  • size: buf のサイズ。
  • conn: 接続状態を保持する PGconn 構造体へのポインタ。
  • 戻り値: 常に 0 を返します。

前提条件と不変条件

  • 本関数は、ユーザーがカスタムのパスフレーズ取得フックを登録していない場合に、デフォルトの動作として呼び出されます。
  • 戻り値が 0 であるため、このデフォルト実装はパスフレーズの提供を行いません。

アルゴリズムまたは振る舞い

1. 引数として受け取った buf、size、conn に対して何ら操作を行いません。 2. 即座に整数 0 を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-secure-openssl.c 内のパスフレーズ処理ロジックから参照されています。
  • 呼び出し先: 外部関数を呼び出しません。

関連データ構造

  • PGconn: libpqの接続状態を管理する構造体。

失敗モードと境界条件

  • 本関数自体は常に成功（0 を返却）しますが、パスフレーズを提供しないため、パスフレーズを必要とする秘密鍵を使用するSSL接続は、このフックが使用される限り失敗する可能性があります。

保守担当者向けメモ

  • 本関数はダミーのデフォルト実装です。パスフレーズの入力を必要とする場合は、PQsetSSLKeyPassHook_OpenSSL を使用して別のフック関数を登録する必要があります。
