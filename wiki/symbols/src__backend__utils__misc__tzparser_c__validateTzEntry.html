validateTzEntry

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/tzparser.c:51

目的

タイムゾーン定義ファイルから読み込まれた tzEntry 構造体の内容を検証し、PostgreSQL の内部形式への適合性を確認する。

シグネチャまたは形状

static bool validateTzEntry(tzEntry *tzentry)

入力と出力

  • **入力**: tzEntry *tzentry（検証対象のタイムゾーンエントリへのポインタ）
  • **出力**: bool（検証に成功した場合は true、失敗した場合は false）

前提条件と不変条件

  • tzentry は有効なメモリ領域を指している必要がある。
  • 検証失敗時には GUC_check_errmsg を使用してエラーメッセージをセットする必要がある（elog(ERROR) は避ける）。
  • 略称の小文字変換は、datetime.c の変換ロジックと一致している必要がある。

アルゴリズムまたは振る舞い

1. **略称の長さチェック**: tzentry->abbrev の長さが TOKMAXLEN を超えていないか確認する。 2. **オフセット範囲チェック**: tzentry->offset が ±14 時間の範囲内にあるか確認する。 3. **正規化**: tzentry->abbrev を小文字に変換する。 4. **結果の返却**: 上記のチェックをすべて通過すれば true を返し、いずれか一つでも違反があれば false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/tzparser.c 内の解析ロジック（ParseTzFile 等）から呼び出される。
  • **呼び出し先**: GUC_check_errmsg（エラー報告用）、pg_tolower（文字変換用）。

関連データ構造

  • tzEntry: タイムゾーンの略称、オフセット、ファイル名、行番号を保持する構造体。

失敗モードと境界条件

  • **略称の長さ超過**: TOKMAXLEN を超える場合、検証失敗。
  • **オフセット範囲外**: ±14 時間（14 * SECS_PER_HOUR）を超える場合、検証失敗。
  • **エラー報告**: 失敗時は tzentry->filename と tzentry->lineno を含む詳細なエラーメッセージが GUC_check_errmsg に渡される。

保守担当者向けメモ

  • 本関数は datetime.c のストレージ形式と密結合しているため、TOKMAXLEN やオフセットの制限を変更する際は datetime.c 側のロジックと整合性を取ること。
  • pg_tolower による小文字変換は破壊的であるため、呼び出し元で元の略称を保持する必要がある場合は注意が必要。
