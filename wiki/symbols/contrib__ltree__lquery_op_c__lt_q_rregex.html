lt_q_rregex

定義

  • 種別: function
  • 定義箇所: contrib/ltree/lquery_op.c:337

目的

lquery 型と ltree 型の引数を入れ替えて、正規表現マッチング演算子 lt_q_regex を呼び出すためのラッパー関数です。

シグネチャまたは形状

Datum lt_q_rregex(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（lquery 型と ltree 型の引数）
  • 出力: Datum（lt_q_regex の実行結果であるブール値）

前提条件と不変条件

  • lt_q_regex が利用可能であること。
  • 引数の順序が lt_q_regex とは逆（第2引数が ltree、第1引数が lquery）として渡されることを前提としています。

アルゴリズムまたは振る舞い

1. PG_GETARG_DATUM(1)（第2引数）と PG_GETARG_DATUM(0)（第1引数）を取得します。 2. DirectFunctionCall2 を使用して、引数を入れ替えた状態で lt_q_regex を呼び出します。 3. lt_q_regex の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の演算子インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先: lt_q_regex。

関連データ構造

  • ltree: 階層パスを表現するデータ構造。
  • lquery: 検索パターンを表現するデータ構造。

失敗モードと境界条件

  • lt_q_regex の失敗モードに依存します。本関数自体は引数の入れ替えのみを行うため、特有の境界条件はありません。

保守担当者向けメモ

  • 本関数は単なる引数順序の入れ替え用ラッパーです。ロジックの変更が必要な場合は lt_q_regex を参照してください。
