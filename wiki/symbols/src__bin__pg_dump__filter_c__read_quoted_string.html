read_quoted_string

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/filter.c:216

目的

フィルタファイル内の引用符で囲まれた文字列（複数行にまたがる可能性のある識別子など）を読み込み、正規化されたバッファに格納する。

シグネチャまたは形状

static const char *read_quoted_string(FilterStateData *fstate, const char *str, PQExpBuffer pattern);

入力と出力

  • **入力**:
  • fstate: フィルタ処理の状態を保持する FilterStateData 構造体へのポインタ。
  • str: 現在解析中の文字列ポインタ。
  • pattern: 読み取った文字列を蓄積する PQExpBuffer。
  • **出力**:
  • 戻り値: 読み込み終了後の str の現在位置を指すポインタ。

前提条件と不変条件

  • str は引用符（"）の直後を指している必要がある。
  • fstate->fp は有効なファイルポインタであること。
  • 引用符内の改行文字（\r, \n）は無視されるか、バッファに改行として適切に変換される。

アルゴリズムまたは振る舞い

1. 最初の引用符を pattern に追加し、str を進める。 2. ループ内で以下の処理を繰り返す：

  • 改行文字（\r, \n）はスキップする。
  • 文字列末尾（\0）に達した場合、pg_get_line_buf を使用して次の行を読み込む。EOF または読み込みエラー時は fstate->exit_nicely を呼び出して終了する。
  • 引用符（"）に遭遇した場合：
  • 連続する引用符（""）はエスケープされた引用符として扱い、pattern に追加する。
  • 単一の引用符であれば文字列の終了とみなし、ループを抜ける。
  • バックスラッシュ（\）に遭遇した場合：
  • \n は改行文字として、\\ はバックスラッシュとして pattern に追加する。
  • それ以外の文字はそのまま pattern に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/filter.c 内の read_pattern 関数から呼び出される。
  • **呼び出し先**: pg_get_line_buf, appendPQExpBufferChar, pg_log_error, pg_log_filter_error, fstate->exit_nicely。

関連データ構造

  • FilterStateData: フィルタ処理のコンテキスト（ファイルポインタ、行番号、バッファ等）を保持。
  • PQExpBuffer: 動的文字列バッファ。

失敗モードと境界条件

  • **EOF**: 引用符が閉じられる前にファイルが終了した場合、pg_log_filter_error を経由してエラー終了する。
  • **読み込みエラー**: ファイルからの読み込みに失敗した場合、pg_log_error を出力して終了する。

保守担当者向けメモ

  • 本関数は pg_get_line_buf を再帰的に呼び出す構造を持つため、スタックオーバーフローの懸念はないが、行番号管理（fstate->lineno）が正しく行われているか注意すること。
  • exit_nicely は外部から注入されるコールバックであるため、デバッグ時は呼び出し元の初期化ロジックを確認すること。
