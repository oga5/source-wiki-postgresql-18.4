get_password_type

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/crypt.c:89

目的

与えられたパスワード文字列（ハッシュ値または平文）のパターンを解析し、そのパスワードがどのハッシュアルゴリズム形式（MD5, SCRAM-SHA-256, または平文）で保存されているかを判定します。

シグネチャまたは形状

PasswordType get_password_type(const char *shadow_pass);

入力と出力

  • **入力**: shadow_pass (const char *) - 判定対象のパスワード文字列。
  • **出力**: PasswordType 列挙型 - 以下のいずれかを返します。
  • PASSWORD_TYPE_MD5
  • PASSWORD_TYPE_SCRAM_SHA_256
  • PASSWORD_TYPE_PLAINTEXT

前提条件と不変条件

  • 入力される shadow_pass はヌル終端文字列である必要があります。
  • MD5_PASSWD_LEN や MD5_PASSWD_CHARSET などの定数に基づき、MD5形式の厳密な構造（"md5"で始まり、特定の長さと文字セットを持つ）を前提として判定を行います。

アルゴリズムまたは振る舞い

1. **MD5判定**: 文字列が "md5" で始まり、長さが MD5_PASSWD_LEN であり、かつ接頭辞を除いた部分が MD5_PASSWD_CHARSET に含まれる文字のみで構成されているかを確認します。合致すれば PASSWORD_TYPE_MD5 を返します。 2. **SCRAM判定**: parse_scram_secret を呼び出し、SCRAM形式のシークレットとして解析可能か試行します。成功すれば PASSWORD_TYPE_SCRAM_SHA_256 を返します。 3. **デフォルト**: 上記のいずれにも該当しない場合、PASSWORD_TYPE_PLAINTEXT を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/user.c (ユーザー管理コマンド)
  • src/backend/libpq/auth-scram.c (SCRAM認証処理)
  • src/backend/libpq/auth.c (認証フロー制御)
  • src/backend/libpq/crypt.c (内部の検証・変換処理)
  • **呼び出し先**:
  • parse_scram_secret (SCRAM形式の解析用)

関連データ構造

  • PasswordType (列挙型)
  • SCRAM_MAX_KEY_LEN (バッファサイズ定義)

失敗モードと境界条件

  • 入力文字列が極端に短い場合や、期待される形式（MD5やSCRAM）の構造を欠いている場合は、最終的に PASSWORD_TYPE_PLAINTEXT として扱われます。
  • parse_scram_secret が失敗した場合は、平文としてフォールバックされます。

保守担当者向けメモ

  • 本関数は、新しいパスワードハッシュアルゴリズムが導入された際に、判定ロジックを拡張する必要があります。
  • MD5の判定ロジックは、将来的なMD5廃止に伴い変更される可能性があります。
