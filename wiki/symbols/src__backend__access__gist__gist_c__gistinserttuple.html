gistinserttuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:1259

目的

GiSTインデックスの特定のページに対して、単一のタプルを挿入するためのラッパー関数です。

シグネチャまたは形状

static bool gistinserttuple(GISTInsertState *state, GISTInsertStack *stack,
                            GISTSTATE *giststate, IndexTuple tuple, OffsetNumber oldoffnum)

入力と出力

  • **入力**:
  • state: 挿入処理の状態管理構造体。
  • stack: ツリー降下時のパス情報。
  • giststate: GiSTインデックスの操作状態。
  • tuple: 挿入対象の IndexTuple。
  • oldoffnum: 更新対象のタプル位置（新規挿入時は InvalidOffsetNumber 等）。
  • **出力**:
  • bool: 挿入が成功したか、またはページ分割が必要であったかを示すフラグ。

前提条件と不変条件

  • 呼び出し元は、対象ページに対する適切なロックを保持している必要があります。
  • 内部で gistinserttuples を呼び出すため、その関数が要求するメモリコンテキストやロック状態の不変条件に依存します。

アルゴリズムまたは振る舞い

1. 受け取った単一の IndexTuple を配列形式に変換します。 2. gistinserttuples を呼び出し、タプル数 1 を指定して挿入処理を委譲します。 3. ページ分割が発生した場合は、gistinserttuples の戻り値に従い、再帰的なツリー更新処理が継続されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c 内の複数箇所（主に gistdoinsert 等の挿入ロジックから呼び出されます）。
  • **呼び出し先**:
  • gistinserttuples（本関数は同関数の単一タプル用ラッパーです）。

関連データ構造

  • GISTInsertState: 挿入プロセスの状態を保持。
  • GISTInsertStack: ツリーの親ノードへのポインタを含むスタック構造。
  • GISTSTATE: インデックスの演算子クラス情報やメモリコンテキストを保持。
  • IndexTuple: GiSTインデックスに格納されるデータ形式。

失敗モードと境界条件

  • ページ容量不足: 挿入時にページが溢れる場合、gistinserttuples を通じてページ分割（gistSplit）がトリガーされます。
  • ロック競合: ページ分割に伴う親ノードの更新時に、並行処理によるロック待機が発生する可能性があります。

保守担当者向けメモ

  • 本関数は gistinserttuples の単純なインターフェースです。挿入ロジックの変更は、主に gistinserttuples 側で行う必要があります。
  • 呼び出し元でロックを保持したまま呼び出されることが多いため、デッドロックを避けるためのロック順序（ルートからリーフへ）が維持されているか注意してください。
