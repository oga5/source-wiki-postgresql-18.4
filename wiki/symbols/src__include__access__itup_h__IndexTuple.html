IndexTuple

定義

  • 種別: typedef
  • 定義箇所: src/include/access/itup.h:53

目的

PostgreSQLのインデックスページ内に格納される、インデックスタプル（IndexTupleData）へのポインタ型を定義します。

シグネチャまたは形状

typedef IndexTupleData *IndexTuple;

入力と出力

  • **入力**: IndexTupleData 構造体へのポインタ。
  • **出力**: インデックスタプルを指すポインタ。

前提条件と不変条件

  • IndexTuple は IndexTupleData 構造体を指す必要があります。
  • メモリレイアウト上、IndexTupleData の直後に NULL ビットマップ（存在する場合）が配置され、その後に属性値が MAXALIGN 境界で並ぶ構造を前提とします。

アルゴリズムまたは振る舞い

インデックスアクセス手法（AM）において、インデックスページ上の物理的なタプルデータにアクセスするための抽象化レイヤーとして機能します。index_getattr 等の関数に渡すことで、タプル内の特定の属性値（Datum）を抽出する際の基点となります。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/amcheck/verify_gin.c: gin_refind_parent, ginReadTupleWithoutState 等で使用。
  • contrib/amcheck/verify_nbtree.c: bt_entry_unique_check, bt_normalize_tuple 等で使用。
  • **呼び出し先**:
  • 本シンボル自体は型定義であるため、直接的な呼び出し先はありません。操作には index_getattr や index_form_tuple 等の関数が利用されます。

関連データ構造

  • IndexTupleData: 実体となる構造体。
  • IndexAttributeBitMapData: NULL 値管理用のビットマップ構造。
  • ItemPointer (t_tid): ヒープ上の対応するタプルを指すポインタ。

失敗モードと境界条件

  • 不適切なポインタ演算を行うと、MAXALIGN 境界の不整合やメモリ破壊を引き起こす可能性があります。
  • INDEX_MAX_KEYS に依存するビットマップサイズは固定長であるため、属性数がこれを超える場合は考慮が必要です。

保守担当者向けメモ

  • 本シンボルは IndexTupleData へのポインタであるため、メモリ管理は呼び出し側のコンテキストに依存します。
  • index_getattr はインライン化されており、パフォーマンスに直結するため、本型を扱う際は効率的なアクセスを維持してください。
