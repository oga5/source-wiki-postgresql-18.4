CLOGShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:786

目的

PostgreSQL の共有メモリ初期化フェーズにおいて、トランザクション状態管理（CLOG）のための共有メモリ領域を割り当ておよび初期化する。

シグネチャまたは形状

void CLOGShmemInit(void)

入力と出力

  • 入力: なし（グローバルな設定変数 transaction_buffers を参照）
  • 出力: なし（副作用として共有メモリ構造体 XactCtl を初期化）

前提条件と不変条件

  • transaction_buffers が 0 の場合、自動チューニングにより設定値を決定する。
  • 実行終了後、transaction_buffers は 0 以外であることが保証される（Assert による検証）。
  • XactCtl が指す共有メモリ領域が既に確保されている必要がある。

アルゴリズムまたは振る舞い

1. **自動チューニング**: transaction_buffers が未設定（0）の場合、CLOGShmemBuffers() を呼び出して適切なバッファ数を計算し、SetConfigOption を用いて設定を適用する。 2. **コールバック設定**: XactCtl->PagePrecedes に CLOGPagePrecedes 関数を割り当てる。 3. **SLRU 初期化**: SimpleLruInit を呼び出し、以下のパラメータで CLOG 用の SLRU キャッシュを構築する。

  • 名前: "transaction"
  • ディレクトリ名: "pg_xact"
  • ロック用トランシェ: LWTRANCHE_XACT_BUFFER, LWTRANCHE_XACT_SLRU
  • 同期ハンドラ: SYNC_HANDLER_CLOG

    • 4. **テスト実行**: SlruPagePrecedesUnitTests を呼び出し、ページ順序ロジックの整合性を検証する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c 内の共有メモリ初期化プロセスから呼び出される。
  • 呼び出し先: CLOGShmemBuffers, SetConfigOption, SimpleLruInit, SlruPagePrecedesUnitTests, CLOGPagePrecedes。

関連データ構造

  • XactCtl: CLOG の共有メモリ状態を管理する SlruCtlData 構造体。
  • transaction_buffers: CLOG バッファ数を制御する GUC 変数。

失敗モードと境界条件

  • transaction_buffers の自動設定が失敗した場合、PGC_S_OVERRIDE を用いて強制的に設定を適用するロジックが含まれる。
  • 共有メモリの確保が不十分な場合、SimpleLruInit 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は PostgreSQL の起動シーケンス（Postmaster 起動時）において重要な役割を果たす。
  • CLOGPagePrecedes は clog.c 内で定義されたページ比較ロジックであり、SLRU のページ管理において必須となる。
