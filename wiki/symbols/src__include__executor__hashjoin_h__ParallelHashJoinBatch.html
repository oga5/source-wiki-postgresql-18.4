ParallelHashJoinBatch

定義

  • 種別: struct
  • 定義箇所: src/include/executor/hashjoin.h:162

目的

並列ハッシュ結合における個別のバッチ状態を管理し、共有メモリ（DSM）上で複数プロセス間の同期とデータ共有を行うための構造体です。

シグネチャまたは形状

typedef struct ParallelHashJoinBatch
{
	dsa_pointer buckets;
	Barrier		batch_barrier;
	dsa_pointer chunks;
	size_t		size;
	size_t		estimated_size;
	size_t		ntuples;
	size_t		old_ntuples;
	bool		space_exhausted;
	bool		skip_unmatched;
} ParallelHashJoinBatch;

※構造体の末尾には可変長の SharedTuplestore オブジェクトが続きます。

入力と出力

  • **入力**: 並列実行中の各ワーカープロセスが、ハッシュ結合の構築およびプローブフェーズで本構造体を介して共有データにアクセスします。
  • **出力**: バッチ処理の進行状況（タプル数、メモリ使用量、同期状態）が共有メモリ上に更新されます。

前提条件と不変条件

  • dsa_area（Dynamic Shared Area）上に配置される必要があります。
  • batch_barrier を使用して、バッチ処理に参加する全プロセス間での同期が保証されます。

アルゴリズムまたは振る舞い

1. **同期**: batch_barrier を用いて、バッチの構築・プローブフェーズの進行を制御します。 2. **メモリ管理**: size および estimated_size により、バケットとチャンクのメモリ消費量を追跡します。 3. **再分割**: old_ntuples を保持し、メモリ不足時にバッチを再分割（repartitioning）する際の基準として使用します。 4. **終了判定**: space_exhausted が真の場合、メモリ制限に達したことを示し、skip_unmatched が真の場合は一致しないタプルのスキャンをスキップします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c 内のハッシュ結合実行ロジックから参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。アクセスマクロを介して SharedTuplestore 等のデータ構造へアクセスします。

関連データ構造

  • Barrier: プロセス間同期に使用。
  • dsa_pointer: 共有メモリ上のデータへのポインタ。
  • SharedTuplestore: 本構造体の直後に配置され、タプルの格納に使用されます。

失敗モードと境界条件

  • space_exhausted が設定されると、メモリ不足により一時ファイルへのスピルが発生する可能性があります。
  • 並列ワーカー間での同期が正しく行われない場合、batch_barrier によるデッドロックやデータ不整合のリスクがあります。

保守担当者向けメモ

  • 本構造体は可変長データを含むため、メモリレイアウトの変更には細心の注意が必要です。
  • NthParallelHashJoinBatch マクロを使用して、バッチ配列内の特定インデックスにアクセスする設計となっています。
