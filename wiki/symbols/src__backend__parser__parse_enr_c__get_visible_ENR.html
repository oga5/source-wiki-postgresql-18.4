get_visible_ENR

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_enr.c:25

目的

現在のクエリ解析環境（ParseState）において、指定された名前を持つ一時的名前付きリレーション（ENR）のメタデータを取得します。

シグネチャまたは形状

EphemeralNamedRelationMetadata get_visible_ENR(ParseState *pstate, const char *refname)

入力と出力

  • **入力**:
  • pstate: 現在のクエリ解析状態を保持する ParseState 構造体へのポインタ。
  • refname: 検索対象となるリレーション名（文字列）。
  • **出力**:
  • 指定された名前のENRが見つかった場合はそのメタデータ（EphemeralNamedRelationMetadata）、見つからない場合は NULL を返します。

前提条件と不変条件

  • pstate は有効なポインタである必要があります。
  • pstate->p_queryEnv が初期化されていることが前提となります。

アルゴリズムまたは振る舞い

1. pstate から p_queryEnv（クエリ環境）を抽出します。 2. 抽出した環境と指定された refname を引数として、外部モジュールの get_visible_ENR_metadata を呼び出します。 3. その結果をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_relation.c:2496 にて、リレーション名の解決時に呼び出されています。
  • **呼び出し先**: get_visible_ENR_metadata（src/include/utils/queryenvironment.h で定義）。

関連データ構造

  • ParseState: クエリ解析のコンテキストを保持する構造体。
  • EphemeralNamedRelationMetadata: ENRのメタデータ構造体。

失敗モードと境界条件

  • 指定された refname が現在の環境に存在しない場合、get_visible_ENR_metadata の仕様に従い NULL を返します。

保守担当者向けメモ

  • 本関数は get_visible_ENR_metadata への単なるラッパーです。ENRの検索ロジック自体を変更する場合は、src/include/utils/queryenvironment.h および関連する実装を確認してください。
