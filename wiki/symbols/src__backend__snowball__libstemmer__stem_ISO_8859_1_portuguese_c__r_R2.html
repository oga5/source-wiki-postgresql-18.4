r_R2

定義

  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_portuguese.c:634

目的

Snowball アルゴリズムにおいて、現在のカーソル位置が単語内の「R2」領域の開始位置以降にあるかどうかを判定します。

シグネチャまたは形状

static int r_R2(struct SN_env * z)

入力と出力

  • **入力**: z (Snowball 実行環境構造体 SN_env)
  • **出力**: 整数値。現在のカーソル位置 z->c が、事前に計算された R2 領域の境界インデックス z->I[0] 以上であれば 1 を、そうでなければ 0 を返します。

前提条件と不変条件

  • r_mark_regions 関数が事前に実行され、z->I[0] に正しい R2 領域の境界インデックスが格納されている必要があります。
  • z->c は現在の処理対象単語内のカーソル位置を保持しています。

アルゴリズムまたは振る舞い

1. z->I[0]（R2 領域の開始インデックス）と z->c（現在のカーソル位置）を比較します。 2. z->I[0] <= z->c が真であれば、現在のカーソル位置は R2 領域内にあると判断し、成功（1）を返します。 3. 偽であれば、失敗（0）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 本関数は、ポルトガル語ステマーの接尾辞削除ロジック内で、特定の接尾辞を削除してよいかどうかの条件判定として呼び出されます。また、他の言語（バスク語、カタラン語、オランダ語など）のステマー実装からも共通のユーティリティとして参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • struct SN_env: Snowball ランタイムの環境構造体。I[0]（領域インデックス）および c（カーソル位置）を保持します。

失敗モードと境界条件

  • r_mark_regions が未実行の場合、z->I[0] が初期値（通常は 0 または不正な値）であるため、判定結果が意図しないものになる可能性があります。
