describeAggregates

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:77

目的

データベース内の集約関数（Aggregate Functions）の情報を取得し、整形して出力する。psql のメタコマンド（\da 等）の実行エンジンとして機能する。

シグネチャまたは形状

bool describeAggregates(const char *pattern, bool verbose, bool showSystem)

入力と出力

  • **入力**:
  • pattern: 検索対象の集約関数名パターン（NULLの場合は全件）。
  • verbose: 詳細表示フラグ（本関数内では直接使用されていない）。
  • showSystem: システムオブジェクトを含めるかどうかのフラグ。
  • **出力**:
  • 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • pset（グローバル設定）が初期化されていること。
  • データベースへの接続が確立されていること。

アルゴリズムまたは振る舞い

1. PQExpBuffer を初期化し、pg_catalog.pg_proc を対象とした SQL クエリを構築する。 2. サーバーバージョン（pset.sversion）が 11 以上か否かで、集約関数の判定条件（prokind または proisagg）を切り替える。 3. showSystem が偽かつ pattern が指定されていない場合、pg_catalog および information_schema を除外する条件を追加する。 4. validateSQLNamePattern を呼び出し、ユーザー入力を安全な SQL 句に変換してバッファに追加する。 5. PSQLexec を実行し、結果セットを printQuery を用いて表形式で出力する。 6. 最後にバッファと結果セットを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:1096
  • **呼び出し先**:
  • initPQExpBuffer, printfPQExpBuffer, appendPQExpBuffer, appendPQExpBufferStr, termPQExpBuffer (libpq/pqexpbuffer)
  • validateSQLNamePattern (src/bin/psql/describe.c)
  • PSQLexec (src/bin/psql/common.c)
  • printQuery (src/bin/psql/fe_utils/print.h)
  • PQclear (libpq)

関連データ構造

  • PQExpBufferData: SQL クエリ構築用バッファ。
  • PGresult: クエリ実行結果の保持。
  • printQueryOpt: 出力オプション。

失敗モードと境界条件

  • validateSQLNamePattern が失敗した場合、バッファを解放して false を返す。
  • PSQLexec が NULL を返した場合（クエリ実行失敗）、false を返す。
  • サーバーバージョンが 11 未満の場合、古いカタログ列定義を使用する。

保守担当者向けメモ

  • 本関数は PostgreSQL 9.2 以降のサーバーを前提としている。
  • 新しい集約関数の属性を表示に追加する場合は、SQL クエリの SELECT 句と printQuery の設定を更新する必要がある。
