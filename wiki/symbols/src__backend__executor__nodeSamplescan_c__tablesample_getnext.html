tablesample_getnext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSamplescan.c:319

目的

サンプリング手法（TsmRoutine）に基づき、テーブルから統計的に抽出された次のタプルを取得して TupleTableSlot に格納する。

シグネチャまたは形状

static TupleTableSlot *tablesample_getnext(SampleScanState *scanstate)

入力と出力

  • **入力**: SampleScanState *scanstate（スキャン状態および現在のスキャン記述子を保持）
  • **出力**: 抽出されたタプルを含む TupleTableSlot。スキャンが完了した場合は NULL を返す。

前提条件と不変条件

  • scanstate は初期化済みであること。
  • scanstate->ss.ss_currentScanDesc が有効なテーブルスキャン記述子を指していること。
  • scanstate->done が true の場合、本関数は直ちに NULL を返す。

アルゴリズムまたは振る舞い

1. ExecClearTuple を呼び出し、スロットをクリアする。 2. scanstate->done が真であれば NULL を返す。 3. ループ内で以下を繰り返す：

  • scanstate->haveblock が偽の場合、table_scan_sample_next_block を呼び出し、次のブロックへ移動する。失敗した場合は scanstate->done を真にし、NULL を返す。
  • table_scan_sample_next_tuple を呼び出し、現在のブロックからタプルを取得する。
  • 取得に失敗した場合は、現在のブロックを使い切ったと判断し、scanstate->haveblock を偽にしてループを継続する。
  • 取得に成功した場合はループを抜ける。

    • 4. scanstate->donetuples をインクリメントし、タプルを格納したスロットを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SampleNext（src/backend/executor/nodeSamplescan.c 内）
  • **呼び出し先**:
  • table_scan_sample_next_block
  • table_scan_sample_next_tuple
  • ExecClearTuple

関連データ構造

  • SampleScanState: スキャンノードの状態管理構造体。
  • TupleTableSlot: タプルを保持するためのコンテナ。
  • TableScanDesc: テーブルアクセスメソッドのスキャン記述子。

失敗モードと境界条件

  • リレーション全体が走査済みの場合、table_scan_sample_next_block が失敗し、NULL を返す。
  • 現在のページ内に抽出対象のタプルが存在しない場合、table_scan_sample_next_tuple が失敗し、次のブロックへ遷移する。

保守担当者向けメモ

  • 本関数は TABLESAMPLE 句の実行におけるメインループの一部である。
  • table_scan_sample_next_block および table_scan_sample_next_tuple は access/tableam.h で定義されたテーブルアクセスメソッドのインターフェースを介して呼び出される。
