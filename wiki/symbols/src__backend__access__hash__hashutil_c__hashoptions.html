hashoptions

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashutil.c:274

目的

ハッシュインデックスの作成時や変更時に指定されるリレーションオプション（reloptions）を解析・検証し、HashOptions 構造体へ変換する。

シグネチャまたは形状

bytea *hashoptions(Datum reloptions, bool validate)

入力と出力

  • **入力**:
  • reloptions (Datum): 解析対象のオプションデータ。
  • validate (bool): オプションの妥当性を検証するかどうかを指定するフラグ。
  • **出力**:
  • bytea *: 解析結果を格納したメモリ領域。

前提条件と不変条件

  • fillfactor オプションが HashOptions 構造体内の適切なオフセットに定義されている必要がある。
  • build_reloptions 関数が提供する標準的なリレーションオプション構築ルールに従う。

アルゴリズムまたは振る舞い

1. fillfactor をキーとする relopt_parse_elt 配列を定義する。 2. build_reloptions を呼び出し、以下の引数を渡して処理を実行する。

  • RELOPT_KIND_HASH: ハッシュインデックス用のオプション種別。
  • sizeof(HashOptions): 構造体のサイズ。
  • 定義した解析用配列と、その要素数。

    • 3. 構築された bytea 形式のオプションデータを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hash.c:97: amroutine->amoptions に関数ポインタとして代入されている。
  • **呼び出し先**:
  • build_reloptions (PostgreSQL 内部の汎用リレーションオプション構築関数)。

関連データ構造

  • HashOptions: ハッシュインデックス固有のオプションを保持する構造体。
  • relopt_parse_elt: オプションの解析ルールを定義する構造体。

失敗モードと境界条件

  • validate が true の場合、不正なオプション値が渡されると build_reloptions 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は amoptions インターフェースの一部であり、ハッシュインデックスのオプション設定が変更された場合は、tab 配列の定義を更新する必要がある。
