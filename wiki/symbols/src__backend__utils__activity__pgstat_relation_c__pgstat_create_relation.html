pgstat_create_relation

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:167

目的

リレーション（テーブルやインデックス）の作成時に、統計情報追跡のためのトランザクション管理エントリを生成します。

シグネチャまたは形状

void pgstat_create_relation(Relation rel)

入力と出力

  • **入力**: Relation rel - 統計対象となるリレーションの記述子。
  • **出力**: なし。内部的に統計管理システムへエントリが登録されます。

前提条件と不変条件

  • rel は有効なリレーション記述子である必要があります。
  • リレーションが共有されているか否か（relisshared）に基づき、適切なデータベースIDが選択されます。

アルゴリズムまたは振る舞い

1. リレーションの共有フラグ（rd_rel->relisshared）を確認します。 2. 共有リレーションの場合は InvalidOid を、それ以外の場合は現在のデータベースID（MyDatabaseId）を選択します。 3. pgstat_create_transactional を呼び出し、PGSTAT_KIND_RELATION 種別で統計エントリを作成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/heap.c:404（リレーション作成処理の一部として呼び出されます）。
  • **呼び出し先**: pgstat_create_transactional（統計サブシステムの内部関数）。

関連データ構造

  • Relation: リレーションのメタデータ構造体。
  • PgStat_Kind: 統計種別（ここでは PGSTAT_KIND_RELATION を使用）。

失敗モードと境界条件

  • メモリ不足等のシステムエラーが発生した場合、pgstat_create_transactional 経由でエラーが伝播する可能性があります。
  • 共有リレーションと非共有リレーションで統計の紐付け先が正しく分岐されることが保証されています。

保守担当者向けメモ

  • 本関数はリレーションのライフサイクル管理の初期段階で呼ばれます。統計情報の永続化やトランザクション終了時の処理については、AtEOXact_PgStat_Relations 等の関連関数を参照してください。
