llvm_in_fatal_on_oom

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_error.cpp:85

目的

LLVMのメモリ不足（OOM）や致命的エラー発生時に、PostgreSQLのプロセスを安全に終了させるためのエラーハンドラが現在有効化されているかどうかを判定します。

シグネチャまたは形状

bool llvm_in_fatal_on_oom(void)

入力と出力

  • 入力: なし
  • 出力: bool 型。ハンドラが有効（ネストレベルが1以上）であれば true、それ以外は false を返します。

前提条件と不変条件

  • 静的変数 fatal_new_handler_depth が、現在のエラーハンドラ設定のネストレベルを正しく保持している必要があります。

アルゴリズムまたは振る舞い

  • 静的変数 fatal_new_handler_depth の値が 0 より大きいかどうかを評価し、その結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/jit/llvm/llvmjit.c:925 にて、エラーハンドリング状態の確認に使用されています。
  • 呼び出し先: 本関数は他の関数を呼び出しません。

関連データ構造

  • fatal_new_handler_depth: ネストレベルを追跡する静的変数。

失敗モードと境界条件

  • 本関数自体は失敗しません。
  • fatal_new_handler_depth が負の値になることは想定されていません。

保守担当者向けメモ

  • 本関数は extern "C" インターフェースの一部として提供されており、C言語側から呼び出し可能です。
  • エラーハンドラのネスト状態を判定する唯一の手段であるため、fatal_new_handler_depth を操作する llvm_enter_fatal_on_oom および llvm_leave_fatal_on_oom との整合性に注意してください。
