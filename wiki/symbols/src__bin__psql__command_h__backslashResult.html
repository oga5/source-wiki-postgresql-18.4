backslashResult

定義

  • 種別: enum
  • 定義箇所: src/bin/psql/command.h:15

目的

psql におけるバックスラッシュコマンド（メタコマンド）の実行結果を規定し、呼び出し元がその後の制御フロー（クエリ送信、プログラム終了、継続など）を決定するためのステータスを定義する。

シグネチャまたは形状

typedef enum _backslashResult
{
    PSQL_CMD_UNKNOWN = 0,
    PSQL_CMD_SEND,
    PSQL_CMD_SKIP_LINE,
    PSQL_CMD_TERMINATE,
    PSQL_CMD_NEWEDIT,
    PSQL_CMD_ERROR,
} backslashResult;

入力と出力

  • 入力: なし（列挙型の定義）
  • 出力: 各メタコマンド実行関数が戻り値としてこの型を返す。

前提条件と不変条件

  • PSQL_CMD_UNKNOWN は内部的な解析中状態として使用され、外部へ公開されるべきではない。
  • 各メタコマンド実行関数は、処理結果に応じて適切な列挙値を返す必要がある。

アルゴリズムまたは振る舞い

  • PSQL_CMD_SEND: クエリが完了したため、送信処理へ移行する。
  • PSQL_CMD_SKIP_LINE: クエリの構築を継続する。
  • PSQL_CMD_TERMINATE: プログラムを終了する。
  • PSQL_CMD_NEWEDIT: クエリバッファが変更された（例: \e による編集）。
  • PSQL_CMD_ERROR: コマンド実行中にエラーが発生した。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/command.c 内の exec_command および各 exec_command_* 関数群がこの値を返却する。
  • 呼び出し先: src/bin/psql/mainloop.c 等のコマンド実行を管理するループ処理が、この戻り値に基づいて後続の処理を分岐させる。

関連データ構造

  • PsqlScanState: コマンド解析状態を保持し、各 exec_command_* 関数に渡される。

失敗モードと境界条件

  • PSQL_CMD_ERROR は、コマンド実行が正常に完了しなかった場合に返される。
  • PSQL_CMD_UNKNOWN は解析が完了していない状態を示すため、最終的な状態として返された場合は異常系とみなされる可能性がある。

保守担当者向けメモ

  • 新しいメタコマンドを追加する際は、既存の列挙値が適切か確認し、必要に応じて psql の状態遷移ロジックを更新すること。
  • 本列挙型は psql の制御フローの根幹に関わるため、変更時は mainloop.c への影響を十分に考慮すること。
