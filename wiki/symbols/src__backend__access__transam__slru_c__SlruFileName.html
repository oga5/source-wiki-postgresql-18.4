SlruFileName

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/slru.c:90

目的

SLRU（Simple LRU）バッファ管理機構において、指定されたセグメント番号に対応するファイルパス文字列を生成する。

シグネチャまたは形状

int SlruFileName(SlruCtl ctl, char *path, int64 segno)

入力と出力

  • **入力**:
  • ctl: SLRU インスタンスの制御構造体。ディレクトリパス（Dir）および命名規則フラグ（long_segment_names）を参照する。
  • path: ファイルパスを格納するバッファ（MAXPGPATH サイズ以上が必要）。
  • segno: 対象となるセグメント番号。
  • **出力**:
  • snprintf の戻り値（書き込まれた文字数）。path バッファにファイルパスが格納される。

前提条件と不変条件

  • ctl->Dir が有効なディレクトリパスを指していること。
  • segno は負であってはならない。
  • long_segment_names が真の場合、segno は 0xFFFFFFFFFFFFFFF 以下であること。
  • long_segment_names が偽の場合、segno は 0xFFFFFF 以下であること。

アルゴリズムまたは振る舞い

1. ctl->long_segment_names の値に基づき、ファイル名のフォーマットを切り替える。 2. **真の場合**: 15桁の16進数（%015" PRIX64）でセグメント番号をフォーマットする。 3. **偽の場合**: 4桁の16進数（%04X）でセグメント番号をフォーマットする。 4. ctl->Dir をプレフィックスとして付与し、path に書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/slru.c 内の各関数（SimpleLruReadPage, SlruInternalDeleteSegment, SlruScanDirCbDeleteCutoff 等）。
  • **呼び出し先**:
  • snprintf（標準ライブラリ関数）。

関連データ構造

  • SlruCtl: SLRU の設定と状態を保持する構造体。

失敗モードと境界条件

  • segno が各モードの許容範囲を超えた場合、Assert により異常終了する可能性がある。
  • path バッファが MAXPGPATH より小さい場合、パスが切り詰められる可能性がある。

保守担当者向けメモ

  • long_segment_names が真の場合の15桁という制限は、WAL セグメントとの混同を避けるための設計上の選択である。
  • 将来的に SLRU_PAGES_PER_SEGMENT を変更する場合、この命名規則が制約となる可能性がある点に注意すること。
