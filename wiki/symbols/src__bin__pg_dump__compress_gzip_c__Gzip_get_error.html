Gzip_get_error

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:357

目的

GZIP 圧縮ファイルハンドルに関連付けられた直近のエラーメッセージを取得します。

シグネチャまたは形状

const char *Gzip_get_error(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH（GZIP 圧縮処理の状態を保持するハンドル）
  • **出力**: エラー内容を示す const char * 型の文字列。

前提条件と不変条件

  • CFH->private_data は有効な gzFile ハンドルとしてキャスト可能である必要があります。

アルゴリズムまたは振る舞い

1. CFH->private_data から gzFile オブジェクトを取得します。 2. zlib の gzerror 関数を呼び出し、エラーメッセージとエラー番号を取得します。 3. エラー番号が Z_ERRNO（システムエラー）である場合、strerror(errno) を使用して標準的なシステムエラーメッセージに差し替えます。 4. 取得したエラーメッセージを返します。

呼び出し元と呼び出し先

  • **呼び出し先**: gzerror (zlib), strerror (libc)
  • **呼び出し元**: src/bin/pg_dump/compress_gzip.c 内で CFH->get_error_func に割り当てられています。これ以外の直接的な呼び出し元は提供されたソースからは特定できません。

関連データ構造

  • CompressFileHandle: 圧縮 I/O の抽象化インターフェース。
  • gzFile: zlib が提供する GZIP ファイル操作用ハンドル。

失敗モードと境界条件

  • gzFile が無効な場合、gzerror の挙動は zlib の仕様に依存します。
  • errno が設定されていない状態で Z_ERRNO が返された場合、strerror は現在の errno に基づくメッセージを返します。

保守担当者向けメモ

  • 本関数は CompressFileHandle のインターフェースの一部として実装されています。
  • zlib のエラーコードとシステムエラーの切り分けを行っているため、エラー報告の際は errno の状態が維持されていることを前提としています。
