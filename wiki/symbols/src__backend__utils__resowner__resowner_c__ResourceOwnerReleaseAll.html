ResourceOwnerReleaseAll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/resowner/resowner.c:347

目的

指定されたリソースオーナーが保持するリソースのうち、特定のフェーズに該当するものを解放する。

シグネチャまたは形状

static void ResourceOwnerReleaseAll(ResourceOwner owner, ResourceReleasePhase phase, bool printLeakWarnings)

入力と出力

  • **入力**:
  • owner: 解放対象のリソースオーナー構造体。
  • phase: 解放対象のフェーズ（BEFORE_LOCKS, LOCKS, AFTER_LOCKS）。
  • printLeakWarnings: 解放時に警告ログを出力するかどうかのフラグ。
  • **出力**: なし（副作用としてリソースの解放を実行）。

前提条件と不変条件

  • 事前に ResourceOwnerSort が呼び出され、リソースが優先順位に従ってソートされていること。
  • owner->releasing および owner->sorted が真であること。
  • リソースは配列（arr）またはハッシュテーブル（hash）のいずれか一方にのみ格納されていること。

アルゴリズムまたは振る舞い

1. ソート済みのリソースリストを末尾から順に走査する。 2. 各リソースの release_phase が指定された phase と一致する限り、以下の処理を繰り返す。

  • printLeakWarnings が真の場合、kind->DebugPrint またはデフォルトの文字列形式で警告ログを出力する。
  • kind->ReleaseResource コールバックを呼び出し、リソースを解放する。

    • 3. 処理したリソース数を減算し、配列またはハッシュテーブルのサイズを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/resowner/resowner.c 内の ResourceOwnerRelease 等から呼び出される（行737, 799）。
  • **呼び出し先**:
  • kind->DebugPrint（定義されている場合）
  • kind->ReleaseResource（リソースごとの解放処理）

関連データ構造

  • ResourceOwnerData: リソースオーナーの内部状態を保持。
  • ResourceElem: リソースの要素を保持する構造体。
  • ResourceOwnerDesc: リソースの種類とコールバックを定義する構造体。

失敗モードと境界条件

  • リソースがソートされていない状態で呼び出された場合、Assert により停止する。
  • kind->ReleaseResource が失敗した場合の挙動は、各リソース型の実装に依存する。

保守担当者向けメモ

  • 本関数は ResourceOwnerRelease の内部処理の一部として機能する。
  • 解放順序は BEFORE_LOCKS -> LOCKS -> AFTER_LOCKS の順序で呼び出されることを前提としている。
  • 解放処理中に新しいリソースの登録は行わないこと。
