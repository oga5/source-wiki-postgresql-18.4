ExecuteSqlQueryForSingleRow

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:243

目的

データベースに対してクエリを実行し、結果として正確に1行のみが返されることを保証するラッパー関数です。

シグネチャまたは形状

PGresult *ExecuteSqlQueryForSingleRow(Archive *fout, const char *query)

入力と出力

  • **入力**:
  • fout: データベース接続を管理する Archive 構造体へのポインタ。
  • query: 実行する SQL クエリ文字列。
  • **出力**:
  • 成功時: クエリ結果を含む PGresult 構造体へのポインタ。
  • 失敗時: 戻り値なし（後述の失敗モードによりプロセスが終了する）。

前提条件と不変条件

  • fout は有効なアーカイブハンドルである必要があります。
  • 実行されるクエリは、データベースから単一の行を返すことが期待されるものである必要があります。

アルゴリズムまたは振る舞い

1. ExecuteSqlQuery を呼び出し、PGRES_TUPLES_OK を期待してクエリを実行します。 2. PQntuples を使用して、返された結果セットの行数を取得します。 3. 行数が 1 ではない場合、pg_fatal を呼び出してエラーメッセージを表示し、プロセスを終了させます。 4. 行数が 1 の場合、その PGresult ポインタを呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の複数箇所（例: pg_dump.c:1379, 1563, 3304, 5544 等）および pg_backup_db.c 内の関数から呼び出されています。
  • **呼び出し先**:
  • ExecuteSqlQuery (直接の依存)
  • PQntuples (libpq 関数)
  • pg_fatal (エラーハンドリング用)

関連データ構造

  • Archive: データベース接続およびアーカイブ状態を保持する構造体。
  • PGresult: libpq が提供するクエリ結果保持用構造体。

失敗モードと境界条件

  • **行数不一致**: クエリの結果が 0 行または 2 行以上の場合、pg_fatal によりプログラムが異常終了します。
  • **クエリ失敗**: 内部で呼び出される ExecuteSqlQuery が失敗した場合、その関数内で定義されたエラーハンドリング（die_on_query_failure 等）に従い、プロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は、結果が単一であることを厳格に要求するクエリ（設定値の取得や特定のメタデータ検索など）に利用してください。
  • 戻り値の PGresult は、呼び出し元で不要になった際に PQclear で解放する必要があります。
