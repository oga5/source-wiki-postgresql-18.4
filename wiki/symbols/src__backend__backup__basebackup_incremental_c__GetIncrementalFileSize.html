GetIncrementalFileSize

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_incremental.c:908

目的

増分バックアップファイルに必要な合計サイズ（ヘッダーおよびブロックデータ）を算出する。

シグネチャまたは形状

size_t GetIncrementalFileSize(unsigned num_blocks_required)

入力と出力

  • **入力**: num_blocks_required (増分バックアップに含まれるブロック数)
  • **出力**: size_t 型の合計バイト数

前提条件と不変条件

  • num_blocks_required は RELSEG_SIZE 以下である必要がある（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. GetIncrementalHeaderSize を呼び出し、マジックナンバーやブロック数等を含むヘッダーサイズを取得する。 2. BLCKSZ（ブロックサイズ）に num_blocks_required を乗算し、ブロックデータの合計サイズを算出する。 3. ヘッダーサイズとブロックデータサイズの合計を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:1513
  • **呼び出し先**: GetIncrementalHeaderSize

関連データ構造

  • BLCKSZ (PostgreSQLの標準ブロックサイズ定数)
  • RELSEG_SIZE (リレーションセグメントの最大ブロック数)

失敗モードと境界条件

  • num_blocks_required が RELSEG_SIZE を超える場合、アサーション違反によりプロセスが停止する。

保守担当者向けメモ

  • 本関数は増分バックアップの出力ストリームのサイズ見積もりに使用される。
  • ヘッダーサイズ計算ロジックに変更があった場合は GetIncrementalHeaderSize との整合性に注意すること。
