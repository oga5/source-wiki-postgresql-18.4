AllocSetContext

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/aset.c:152

目的

PostgreSQL における標準的なメモリコンテキスト実装である AllocSet の内部状態を保持する構造体です。小規模なメモリ割り当てを効率的に管理し、コンテキスト単位での再利用を可能にします。

シグネチャまたは形状

typedef struct AllocSetContext
{
    MemoryContextData header;
    AllocBlock        blocks;
    MemoryChunk      *freelist[ALLOCSET_NUM_FREELISTS];
    uint32            initBlockSize;
    uint32            maxBlockSize;
    uint32            nextBlockSize;
    uint32            allocChunkLimit;
    int               freeListIndex;
} AllocSetContext;

入力と出力

本構造体はメモリコンテキストの管理データそのものであり、関数への引数や戻り値として直接渡されるほか、AllocSet 型（AllocSetContext* の別名）として参照されます。

前提条件と不変条件

  • header フィールドにより MemoryContextData を継承しており、IsA(context, AllocSetContext) による型チェックが可能です。
  • blocks はこのコンテキストで確保されたメモリブロックのリストの先頭を指します。
  • freelist はサイズ別の空きチャンクを管理し、allocChunkLimit 以下の要求に対して再利用を提供します。

アルゴリズムまたは振る舞い

  • **ブロック管理**: malloc されたメモリブロックをリストとして保持し、必要に応じて nextBlockSize を用いて拡大します。
  • **チャンク再利用**: pfree されたメモリは即座に OS へ返さず、freelist に繋ぐことで次回の palloc での再利用を試みます。
  • **コンテキスト再利用**: freeListIndex が有効な場合、コンテキスト削除時にグローバルなフリーリストへ戻され、将来のコンテキスト生成時に再利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/prepare.c 等のメモリ管理インターフェース経由で参照されます。
  • **呼び出し先**: aset.c 内の AllocSetAlloc（確保）、AllocSetFree（解放）、AllocSetReset（リセット）等の関数が本構造体を操作します。

関連データ構造

  • MemoryContextData: 基底クラス。
  • AllocBlock: 管理下のメモリブロックのヘッダー。
  • MemoryChunk: ブロック内の個別の割り当て単位。
  • AllocSet: AllocSetContext へのポインタ型。

失敗モードと境界条件

  • allocChunkLimit を超える巨大な割り当て要求に対しては、フリーリストを経由せず直接ブロックを確保・解放します。
  • メモリ不足時には malloc が失敗し、PostgreSQL のエラーハンドリング機構へ制御が移ります。

保守担当者向けメモ

  • AllocSetContext のサイズは MAXALIGN で調整されており、AllocBlock のヘッダー計算時（src/backend/utils/mmgr/aset.c:245 等）にこのサイズが加味されます。
  • freelist のインデックス操作はビット演算による最適化が行われているため、変更時は pg_bitutils.h の仕様を確認してください。
