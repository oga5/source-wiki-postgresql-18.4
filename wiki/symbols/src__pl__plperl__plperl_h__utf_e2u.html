utf_e2u

定義

  • 種別: static inline function
  • 定義箇所: src/pl/plperl/plperl.h:69

目的

データベースのエンコーディングで表現された文字列を、Perl が内部的に扱う UTF-8 形式の文字列へ変換します。

シグネチャまたは形状

static inline char *utf_e2u(const char *str)

入力と出力

  • **入力**: str (データベースエンコーディングのヌル終端文字列)
  • **出力**: 変換後の UTF-8 文字列を指すポインタ。

前提条件と不変条件

  • 入力文字列 str はヌル終端である必要があります。
  • 戻り値は palloc によってメモリ確保されるため、呼び出し側で適切なメモリコンテキスト管理が必要です。

アルゴリズムまたは振る舞い

1. pg_server_to_any を呼び出し、データベースエンコーディングから PG_UTF8 への変換を試みます。 2. 変換の結果、入力文字列と同一のポインタが返された場合（変換が不要だった場合）、pstrdup を用いてメモリを複製します。 3. 常に新しいメモリ領域を確保した文字列を返すことを保証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plperl/plperl.h 内の各所、および src/pl/tcl/pltcl.c からの呼び出しが確認されています。
  • **呼び出し先**:
  • pg_server_to_any (PostgreSQL マルチバイト変換関数)
  • strlen
  • pstrdup (PostgreSQL メモリ割り当て関数)

関連データ構造

  • PG_UTF8 (エンコーディング指定定数)

失敗モードと境界条件

  • メモリ不足時には palloc / pstrdup の内部挙動によりエラーが発生します。
  • SQL_ASCII エンコーディングの場合、変換処理はスキップされますが、pstrdup によるコピーは行われます。

保守担当者向けメモ

  • 本関数はインライン関数として定義されています。
  • src/pl/tcl/pltcl.c からの呼び出しにおいて、メモリリークの可能性が指摘されているため、呼び出し側でのメモリ解放の責任範囲を明確にしてください。
