sepgsql_object_relabel

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/label.c:481

目的

SQLの SECURITY LABEL 文の実行を受け、指定されたデータベースオブジェクトに対してSELinuxセキュリティコンテキストの再割り当て（ラベル変更）を行う。

シグネチャまたは形状

void sepgsql_object_relabel(const ObjectAddress *object, const char *seclabel)

入力と出力

  • **入力**:
  • object: 再ラベル付け対象のオブジェクトを特定する ObjectAddress 構造体。
  • seclabel: 設定するセキュリティコンテキスト文字列（NULL許容）。
  • **出力**: なし（成功時はラベル変更を実行、失敗時はエラーをスロー）。

前提条件と不変条件

  • seclabel が指定されている場合、security_check_context_raw() を用いてSELinuxコンテキストとして妥当な形式である必要がある。
  • サポート対象外のオブジェクトクラスが渡された場合、ERRCODE_FEATURE_NOT_SUPPORTED エラーが発生する。

アルゴリズムまたは振る舞い

1. **形式検証**: seclabel が提供されている場合、SELinuxのコンテキスト形式として妥当か検証し、不正な場合はエラーを報告する。 2. **オブジェクト種別判定**: object->classId に基づき、以下のいずれかの関数へ処理を委譲する。

  • DatabaseRelationId: sepgsql_database_relabel
  • NamespaceRelationId: sepgsql_schema_relabel
  • RelationRelationId: object->objectSubId が 0 なら sepgsql_relation_relabel、それ以外は sepgsql_attribute_relabel
  • ProcedureRelationId: sepgsql_proc_relabel

    • 3. **例外処理**: 上記以外のオブジェクトクラスに対しては、getObjectTypeDescription を用いてエラーメッセージを生成し、エラーをスローする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/hooks.c:470
  • contrib/sepgsql/label.c:821
  • **呼び出し先**:
  • security_check_context_raw (SELinux API)
  • sepgsql_database_relabel
  • sepgsql_schema_relabel
  • sepgsql_relation_relabel
  • sepgsql_attribute_relabel
  • sepgsql_proc_relabel
  • getObjectTypeDescription

関連データ構造

  • ObjectAddress: PostgreSQLのシステムカタログ上のオブジェクトを識別する構造体。

失敗モードと境界条件

  • **無効なラベル**: security_check_context_raw が失敗した場合、ERRCODE_INVALID_NAME でエラー終了する。
  • **未サポートのオブジェクト**: sepgsql がラベル付けをサポートしていないオブジェクトクラスが渡された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。

保守担当者向けメモ

  • 本関数は SECURITY LABEL 文のディスパッチャーとして機能する。新しいオブジェクトクラスへのラベル付けをサポートする場合は、switch 文にケースを追加し、対応する sepgsql_<type>_relabel 関数を実装する必要がある。
