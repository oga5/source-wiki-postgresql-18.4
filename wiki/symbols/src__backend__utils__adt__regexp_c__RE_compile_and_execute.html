RE_compile_and_execute

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regexp.c:357

目的

正規表現のコンパイル（キャッシュ利用）と実行を単一のインターフェースで提供し、SQLレベルのパターンマッチング処理を簡略化する。

シグネチャまたは形状

bool RE_compile_and_execute(text *text_re, char *dat, int dat_len,
                            int cflags, Oid collation,
                            int nmatch, regmatch_t *pmatch)

入力と出力

  • **入力**:
  • text_re: コンパイル対象の正規表現パターン（text型）。
  • dat: 検索対象の文字列データ。
  • dat_len: dat のバイト長。
  • cflags: 正規表現コンパイルフラグ。
  • collation: 照合順序（ロケール依存の動作に使用）。
  • nmatch: 保持するサブマッチの最大数。
  • pmatch: マッチ結果を格納する regmatch_t 配列。
  • **出力**:
  • bool: マッチした場合は true、それ以外は false を返す。

前提条件と不変条件

  • nmatch < 2 の場合、内部的に REG_NOSUB フラグが強制的に追加され、サブマッチの詳細取得が抑制される。

アルゴリズムまたは振る舞い

1. **フラグ調整**: nmatch が2未満であれば、cflags に REG_NOSUB を付与する。 2. **コンパイル**: RE_compile_and_cache を呼び出し、キャッシュからコンパイル済み正規表現オブジェクト（regex_t *）を取得または生成する。 3. **実行**: RE_execute を呼び出し、対象データに対してマッチングを実行する。 4. **戻り値**: マッチング結果の真偽値をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonpath_exec.c
  • src/backend/utils/adt/regexp.c 内の各種SQL関数実装（~, !~ 等の演算子バックエンド）。
  • **呼び出し先**:
  • RE_compile_and_cache
  • RE_execute

関連データ構造

  • regex_t: POSIX正規表現エンジンが使用するコンパイル済みパターン構造体。
  • regmatch_t: マッチした部分文字列の開始・終了位置を保持する構造体。

失敗モードと境界条件

  • RE_compile_and_cache または RE_execute 内部でエラーが発生した場合、PostgreSQLの例外処理メカニズムによりトランザクションが中断される可能性がある。

保守担当者向けメモ

  • 本関数は正規表現のコンパイルと実行を連結するラッパーであるため、キャッシュ戦略や実行エンジンの詳細な挙動は RE_compile_and_cache および RE_execute に依存する。
