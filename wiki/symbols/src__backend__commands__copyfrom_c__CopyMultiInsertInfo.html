CopyMultiInsertInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/copyfrom.c:94

目的

COPY FROM コマンド実行時における、バルク挿入（Multi-Insert）のバッファリング状態を管理するための構造体です。

シグネチャまたは形状

typedef struct CopyMultiInsertInfo
{
    List       *multiInsertBuffers; /* 追跡対象の CopyMultiInsertBuffer リスト */
    int         bufferedTuples;     /* 全バッファ内の合計タプル数 */
    int         bufferedBytes;      /* 全バッファ内の合計バイト数 */
    CopyFromState cstate;           /* COPY 実行状態 */
    EState     *estate;             /* COPY 用のエグゼキュータ状態 */
    CommandId   mycid;              /* COPY 用のコマンド ID */
    int         ti_options;         /* テーブル挿入オプション */
} CopyMultiInsertInfo;

入力と出力

本構造体は関数引数として渡され、状態の更新やフラッシュ処理の対象となります。入力として ResultRelInfo 等の初期化情報を受け取り、出力としてテーブルへの一括挿入を実行します。

前提条件と不変条件

  • BEFORE トリガーや INSTEAD OF トリガーが存在する場合、バルク挿入は無効化されます。
  • メモリ割り当ては、COPY 専用のメモリコンテキスト内で行う必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: CopyMultiInsertInfoInit により状態が初期化されます。 2. **バッファリング**: CopyMultiInsertInfoSetupBuffer を通じて各パーティション等のバッファが管理されます。 3. **蓄積**: CopyMultiInsertInfoIsFull で閾値判定を行い、bufferedTuples や bufferedBytes を更新します。 4. **フラッシュ**: CopyMultiInsertInfoFlush により、蓄積されたタプルが table_multi_insert を介して物理的に挿入されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c 内の COPY 実行ループおよび関連する管理関数。
  • **呼び出し先**: table_multi_insert（テーブルアクセス層）。
  • 注: 具体的な関数呼び出しの連鎖については、ソースコードの静的解析結果に基づき、CopyMultiInsertInfoInit, CopyMultiInsertInfoFlush, CopyMultiInsertInfoIsFull 等の関数が本構造体を操作することが確認されています。

関連データ構造

  • CopyMultiInsertBuffer: 個別のバッファ管理構造体。
  • CopyFromState: COPY コマンド全体の実行状態。
  • ResultRelInfo: 挿入対象のテーブル情報。
  • EState: エグゼキュータの状態。

失敗モードと境界条件

  • **バッファ溢れ**: CopyMultiInsertInfoIsFull が真を返す場合、フラッシュ処理が必要です。
  • **トリガー制約**: トリガーが存在する場合、バルク挿入ロジックがバイパスされる境界条件があります。

保守担当者向けメモ

  • 本構造体は COPY のパフォーマンス最適化の要です。変更を加える際は、CopyMultiInsertBuffer との整合性およびメモリコンテキストのライフサイクルに注意してください。
  • 呼び出し元・呼び出し先の詳細な依存関係グラフは、現在の静的解析では部分的な参照にとどまっています。
