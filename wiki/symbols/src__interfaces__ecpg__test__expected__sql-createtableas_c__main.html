目的

ECPG（Embedded SQL for C）プリプロセッサの回帰テストとして、CREATE TABLE AS 文の実行および WITH NO DATA オプションの動作を検証するエントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しない）
  • 出力: 成功時に 0 を返し、標準出力にクエリ結果（ID = ...）を出力する。

前提条件と不変条件

  • 実行前にデータベース ecpg1_regression への接続が可能であること。
  • sqlca（SQL Communication Area）が各 SQL 文の実行後に更新され、エラーコードや警告が適切に判定されること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、ECPGconnect でデータベースに接続。ECPGsetcommit でコミットモードを on に設定。 2. **テーブル作成とデータ挿入**: cta_test テーブルを作成し、値を挿入。 3. **検証1**: CREATE TABLE AS を実行して cta_test1 を作成し、SELECT でデータを取得して id 変数に格納。結果を printf で出力。 4. **検証2**: WITH NO DATA オプション付きで cta_test2 を作成し、SELECT count(id) を実行して結果を id 変数に格納。結果を printf で出力。 5. **後始末**: 作成したテーブルを順次 DROP し、ECPGdisconnect で接続を終了。 6. **エラーハンドリング**: 各 SQL 文の実行後、sqlca.sqlcode < 0 または sqlca.sqlwarn[0] == 'W' をチェックし、該当する場合は sqlprint を呼び出す。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGsetcommit, ECPGdo, ECPGdisconnect, sqlprint, printf

関連データ構造

  • sqlca: SQL 通信領域。エラーおよび警告の監視に使用。
  • id: SQL クエリの結果を保持するホスト変数（int 型）。

失敗モードと境界条件

  • データベース接続失敗時、または SQL 文の実行エラー時に sqlprint が呼び出され、エラー情報が標準エラー出力に書き出される。
  • sqlca.sqlwarn[0] == 'W' のチェックにより、SQL 実行時の警告が捕捉される。
