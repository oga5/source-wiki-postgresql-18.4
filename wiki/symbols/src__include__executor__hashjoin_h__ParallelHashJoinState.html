ParallelHashJoinState

定義

  • 種別: struct
  • 定義箇所: src/include/executor/hashjoin.h:246

目的

並列ハッシュ結合（Parallel Hash Join）において、共有メモリ（DSM）上で複数プロセス間の同期と状態管理を行うための構造体。

シグネチャまたは形状

typedef struct ParallelHashJoinState {
    dsa_pointer batches;
    dsa_pointer old_batches;
    int         nbatch;
    int         old_nbatch;
    int         nbuckets;
    ParallelHashGrowth growth;
    dsa_pointer chunk_work_queue;
    int         nparticipants;
    size_t      space_allowed;
    size_t      total_tuples;
    LWLock      lock;
    Barrier     build_barrier;
    Barrier     grow_batches_barrier;
    Barrier     grow_buckets_barrier;
    pg_atomic_uint32 distributor;
    SharedFileSet fileset;
} ParallelHashJoinState;

入力と出力

  • **入力**: 共有メモリ領域（DSA）を介したバッチ、チャンク、およびファイルセットのポインタ。
  • **出力**: 複数プロセス間で共有されるハッシュテーブルの構築状態、バッチ分割・バケット成長の進捗、および同期用バリアの状態。

前提条件と不変条件

  • lock（LWLock）により、構造体内の主要な状態（nbatch, nbuckets, growth 等）へのアクセスが保護される必要がある。
  • 並列実行に参加する全プロセスは、build_barrier、grow_batches_barrier、grow_buckets_barrier を使用して、構築・プローブの各フェーズを同期させる必要がある。

アルゴリズムまたは振る舞い

  • **構築フェーズ**: build_barrier を用いて、全参加プロセスがハッシュテーブル構築の各段階を同期して進行する。
  • **成長制御**: メモリ制限に達した場合、grow_batches_barrier や grow_buckets_barrier を用いて、バッチの再分割やバケットの拡張を全プロセスで協調して行う。
  • **負荷分散**: distributor（アトミックカウンタ）を使用して、プロセス間のタスク割り当てを調整する。
  • **一時ファイル**: SharedFileSet を使用し、メモリ不足時にバッチをディスクへ退避・読み込みする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c 内の各関数（hashtable->parallel_state として参照）。
  • **呼び出し先**: 構造体内部の Barrier や LWLock、dsa_pointer を介したメモリ領域、および SharedFileSet が操作対象となる。具体的な関数呼び出し関係はソース抜粋からは不明。

関連データ構造

  • ParallelHashJoinBatch: バッチ管理用構造体。
  • ParallelHashGrowth: ハッシュテーブルの成長状態を示す列挙型。
  • SharedFileSet: 共有一時ファイル管理用。
  • Barrier: プロセス間同期用。
  • LWLock: 共有状態保護用。

失敗モードと境界条件

  • メモリ不足時、nbatch の増加やバッチの再分割が必要となる。
  • 共有メモリ領域（DSA）の枯渇や、SharedFileSet による一時ファイル書き出し時のディスク容量不足が発生し得る。

保守担当者向けメモ

  • 本構造体は並列実行の心臓部であり、LWLock の競合や Barrier のデッドロックに注意が必要。
  • dsa_pointer を介したメモリ操作を行っているため、ポインタの有効性とライフサイクル管理が重要。
  • 構造体内の各フィールドの更新順序と同期タイミングについては、nodeHash.c の実装ロジックを詳細に参照すること。
