PrewarmType

定義

  • 種別: enum
  • 定義箇所: contrib/pg_prewarm/pg_prewarm.c:38

目的

PostgreSQL のリレーションデータをメモリ上にロードする際の、プリウォーム手法を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	PREWARM_PREFETCH,
	PREWARM_READ,
	PREWARM_BUFFER,
} PrewarmType;

入力と出力

  • 入力: SQL 関数 pg_prewarm の引数として指定された文字列（手法名）が、内部的にこの列挙型へ変換されます。
  • 出力: プリウォーム実行ロジックにおける分岐条件として使用されます。

前提条件と不変条件

  • 列挙された 3 つの値（PREWARM_PREFETCH, PREWARM_READ, PREWARM_BUFFER）のいずれかである必要があります。

アルゴリズムまたは振る舞い

  • PREWARM_PREFETCH: OS の posix_fadvise 等を利用し、非同期に OS キャッシュへ読み込みを依頼します。
  • PREWARM_READ: smgrread を使用し、同期的に物理読み込みを行い OS キャッシュへ格納します。
  • PREWARM_BUFFER: ReadStream を使用し、共有バッファへ直接データをロードします。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_prewarm/pg_prewarm.c 内の pg_prewarm 関数にて、引数の解析結果として PrewarmType ptype 変数に代入されます。
  • 呼び出し先: 本シンボル自体はデータ型定義であり、直接的な関数呼び出しは行いません。

関連データ構造

  • pg_prewarm 関数内で、リレーションのプリウォーム処理を制御するための変数として利用されます。

失敗モードと境界条件

  • 定義外の値が指定された場合、pg_prewarm 関数内での入力検証フェーズでエラーとなります。

保守担当者向けメモ

  • 新たなプリウォーム手法を追加する場合は、この列挙型に値を追加し、pg_prewarm 関数内の実行ループ（switch 文等）を更新する必要があります。
