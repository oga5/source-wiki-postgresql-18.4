gbt_int2lt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int2.c:49

目的

GiST インデックスにおいて、2つの int2（2バイト整数）値を比較し、左辺が右辺より小さいかどうかを判定する。

シグネチャまたは形状

static bool gbt_int2lt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の int16 値へのポインタ。
  • b: 比較対象の int16 値へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • bool: *a < *b が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • a および b は、有効な int16 型の値を指している必要がある。

アルゴリズムまたは振る舞い

1. ポインタ a が指す値を int16 型として取得する。 2. ポインタ b が指す値を int16 型として取得する。 3. C 言語の標準的な比較演算子 < を用いて大小比較を行い、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int2.c:88 にて、tinfo 構造体の初期化時に参照されている。
  • **呼び出し先**: 本関数は直接的な関数呼び出しを行わない。

関連データ構造

  • int16: PostgreSQL の 2バイト整数型。

失敗モードと境界条件

  • ポインタが NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • 整数値の範囲外（オーバーフロー等）に関するチェックは本関数内では行わない。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義される汎用的な GiST 数値操作関数から呼び出されることを想定した、型固有の比較ラッパーである。
  • 同様のロジックが gbt_int2gt, gbt_int2ge, gbt_int2eq, gbt_int2le にも存在する。
