deparseDirectDeleteSql

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/deparse.c:2388

目的

リモートの PostgreSQL サーバーで直接実行可能な DELETE 文の SQL 文字列を構築します。主に postgres_fdw において、外部テーブルに対する削除操作をリモートへプッシュダウンする際に使用されます。

シグネチャまたは形状

void deparseDirectDeleteSql(StringInfo buf, PlannerInfo *root,
                            Index rtindex, Relation rel,
                            RelOptInfo *foreignrel,
                            List *remote_conds,
                            List **params_list,
                            List *returningList,
                            List **retrieved_attrs)

入力と出力

  • **入力**:
  • buf: SQL 文字列を格納する StringInfo バッファ。
  • root: プランナのグローバル情報。
  • rtindex: ターゲットリレーションの範囲テーブルインデックス。
  • rel: 削除対象のテーブルリレーション。
  • foreignrel: 外部テーブルまたは結合リレーションの情報。
  • remote_conds: リモートで実行可能な WHERE 句の条件リスト。
  • returningList: RETURNING 句のリスト。
  • **出力**:
  • buf: 構築された DELETE 文が追記されます。
  • params_list: パラメータ化された値のリストが更新されます。
  • retrieved_attrs: RETURNING 句で取得する属性のリストが設定されます。

前提条件と不変条件

  • foreignrel が RELOPT_JOINREL（結合リレーション）である場合、USING 句を用いた結合削除構文が生成されます。
  • params_list は、式構築中に発生するパラメータを保持するために適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. deparse_expr_cxt を初期化し、再帰的な式構築の準備を行います。 2. DELETE FROM [relation] をバッファに書き込みます。 3. foreignrel が結合リレーションの場合、USING 句を生成し、結合条件を remote_conds に統合します。 4. appendWhereClause を呼び出し、WHERE 句を構築します。 5. returningList が存在する場合、結合リレーションであれば deparseExplicitTargetList を、単一テーブルであれば deparseReturningList を使用して RETURNING 句を構築します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c:2592
  • **呼び出し先**:
  • deparseRelation
  • deparseFromExprForRel
  • appendWhereClause
  • deparseExplicitTargetList
  • deparseReturningList
  • planner_rt_fetch

関連データ構造

  • deparse_expr_cxt: SQL 構築中の状態を保持するコンテキスト。
  • PlannerInfo: プランナのグローバル情報。
  • RelOptInfo: リレーションの最適化情報。
  • StringInfo: 文字列構築用バッファ。

失敗モードと境界条件

  • foreignrel が結合リレーションであるか否かで、生成される SQL 構文（USING 句の有無）が分岐します。
  • additional_conds が生成された場合、処理終了後に list_free_deep でメモリ解放が行われます。

保守担当者向けメモ

  • 本関数は postgres_fdw における直接削除の SQL 生成ロジックをカプセル化しています。
  • 結合削除（DELETE ... USING）のサポートが含まれているため、結合条件の解析ロジックに変更を加える際は deparseFromExprForRel との整合性に注意してください。
