rule

定義

  • 種別: struct
  • 定義箇所: src/timezone/zic.c:57

目的

タイムゾーン定義ファイル（.zi）から読み込まれた、夏時間や標準時間の切り替えルールを保持するための構造体です。

シグネチャまたは形状

struct rule {
    const char *r_filename;      /* 定義ファイル名 */
    lineno_t    r_linenum;       /* 定義行番号 */
    const char *r_name;          /* ルール名 */
    zic_t       r_loyear;        /* 適用開始年 */
    zic_t       r_hiyear;        /* 適用終了年 */
    bool        r_lowasnum;      /* 開始年が数値か */
    bool        r_hiwasnum;      /* 終了年が数値か */
    int         r_month;         /* 月 (0..11) */
    int         r_dycode;        /* 日の指定コード */
    int         r_dayofmonth;    /* 日付 */
    int         r_wday;          /* 曜日 */
    zic_t       r_tod;           /* 真夜中からの経過時間 */
    bool        r_todisstd;      /* r_tod が標準時か */
    bool        r_todisut;       /* r_tod が UT か */
    bool        r_isdst;         /* 夏時間か */
    zic_t       r_save;          /* 標準時からのオフセット */
    const char *r_abbrvar;       /* 省略名の可変部 */
    bool        r_todo;          /* 処理対象フラグ (outzoneで使用) */
    zic_t       r_temp;          /* 一時作業用 (outzoneで使用) */
};

入力と出力

  • 入力: zic ユーティリティによる .zi ファイルの構文解析結果。
  • 出力: outzone() 関数等で参照され、最終的な TZif バイナリ生成の基礎データとして利用されます。

前提条件と不変条件

  • r_month は 0 から 11 の範囲である必要があります。
  • r_loyear および r_hiyear は適用期間を定義し、r_lowasnum/r_hiwasnum はそれらが数値として指定されたかを示します。

アルゴリズムまたは振る舞い

  • zic.c 内の infile() によって各ルールが構築されます。
  • outzone() 関数において、特定のゾーン定義と紐付けられ、指定期間内の遷移時刻計算に利用されます。
  • r_todo フラグは、計算処理の過程で当該ルールが処理済みかどうかを追跡するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: infile()（定義の読み込み時）、outzone()（バイナリ生成時）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。
  • 注記: ツールが検出した他のファイル（contrib/ 等）内の「rule」という単語は、本構造体とは無関係な用語です。

関連データ構造

  • struct zone: タイムゾーンの地理的・時間的定義を保持し、本構造体と関連付けられます。
  • lineno_t: 行番号を保持する型。
  • zic_t: タイムゾーン計算に使用される時刻型。

失敗モードと境界条件

  • メモリ不足時には memory_exhausted が発生する可能性があります。
  • 構文解析時に不正な月や日付が指定された場合、zic のエラーハンドリング機構により処理が中断されます。

保守担当者向けメモ

  • 本構造体は IANA zic ユーティリティのフォークであり、PostgreSQL のビルド環境向けに最適化されています。
  • r_temp や r_todo は outzone() 内の計算ロジックに強く依存しているため、変更時は outzone() の遷移時刻計算ロジックへの影響を確認してください。
