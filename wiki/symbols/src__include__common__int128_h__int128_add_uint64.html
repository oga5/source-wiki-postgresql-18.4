int128_add_uint64

定義

  • 種別: function
  • 定義箇所: src/include/common/int128.h:40

目的

128ビット整数型（INT128）に対して、64ビット符号なし整数（uint64）を加算する。

シグネチャまたは形状

static inline void int128_add_uint64(INT128 *i128, uint64 v)

入力と出力

  • **入力**:
  • i128: 加算対象となる128ビット整数へのポインタ（入力および出力）。
  • v: 加算する64ビット符号なし整数。
  • **出力**: なし（i128 が指すメモリ領域が直接更新される）。

前提条件と不変条件

  • i128 は有効な INT128 型のメモリ領域を指している必要がある。
  • コンパイル環境に応じて、INT128 はネイティブの int128 型、または hi/lo 構造体のいずれかとして扱われる。

アルゴリズムまたは振る舞い

  • i128 が指す値に対して、v を加算する。
  • ネイティブ型がサポートされている環境では、コンパイラの組み込み演算に委譲される。
  • ソフトウェア実装（非ネイティブ環境）では、lo 側の加算を行い、キャリーが発生した場合は hi 側をインクリメントする処理が内部的に行われる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/common/int128.h 内の複数箇所（行41, 122, 219, 227, 230）
  • src/tools/testint128.c:98
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わず、言語組み込みの加算演算子（+=）を使用する。

関連データ構造

  • INT128: 128ビット整数を保持する型。

失敗モードと境界条件

  • 128ビットの範囲を超える加算が行われた場合、オーバーフローが発生する（ラップアラウンド動作）。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結している。
  • ソフトウェア実装において、エンディアン（WORDS_BIGENDIAN）を考慮したメモリレイアウトが維持されていることを確認すること。
  • パフォーマンス上の理由から、可能な限りネイティブな int128 型が利用可能な環境での使用が推奨される。
