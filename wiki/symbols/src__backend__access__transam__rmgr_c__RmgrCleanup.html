RmgrCleanup

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/rmgr.c:73

目的

システム終了時またはリカバリ終了時に、登録されているすべてのリソースマネージャ（Resource Manager）に対してクリーンアップ処理を実行する。

シグネチャまたは形状

void RmgrCleanup(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として各リソースマネージャの rm_cleanup 関数が実行される）

前提条件と不変条件

  • RmgrTable が初期化済みであること。
  • RM_MAX_ID までの範囲でリソースマネージャが管理されていること。

アルゴリズムまたは振る舞い

1. 0 から RM_MAX_ID までの範囲でループを実行する。 2. RmgrIdExists(rmid) を使用して、対象のIDにリソースマネージャが登録されているかを確認する。登録されていない場合はスキップする。 3. 登録されているリソースマネージャの rm_cleanup 関数ポインタが NULL でない場合、その関数を呼び出す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogrecovery.c:1891
  • 呼び出し先: RmgrTable に登録された各リソースマネージャの rm_cleanup 関数（動的ディスパッチ）

関連データ構造

  • RmgrTable: リソースマネージャの定義を保持するグローバル配列。
  • RmgrData: 各リソースマネージャの情報を保持する構造体（rm_cleanup 関数ポインタを含む）。

失敗モードと境界条件

  • rm_cleanup が NULL のリソースマネージャについては、安全にスキップされる。
  • RmgrIdExists により、未定義のIDに対するアクセスは防止される。

保守担当者向けメモ

  • 本関数は rmgrlist.h で定義された全リソースマネージャの終了処理を統括する。新しいリソースマネージャを追加し、終了時にクリーンアップが必要な場合は、対応する rm_cleanup 関数を実装し RmgrData に登録すること。
