pg_prng_state

定義

  • 種別: struct
  • 定義箇所: src/include/common/pg_prng.h:19

目的

xoroshiro128** 1.0 アルゴリズムを用いた擬似乱数生成器の内部状態を保持するための構造体です。

シグネチャまたは形状

typedef struct pg_prng_state
{
	uint64		s0,
				s1;
} pg_prng_state;

入力と出力

  • **入力**: 乱数生成関数（pg_prng_uint64 等）の引数としてポインタを渡します。
  • **出力**: 内部状態（s0, s1）が更新されます。

前提条件と不変条件

  • **初期化**: ゼロ初期化は禁止されています（ゼロはアルゴリズムの固定点となるため）。必ず pg_prng_seed または pg_prng_strong_seed を使用して非ゼロ値で初期化する必要があります。

アルゴリズムまたは振る舞い

  • BlackmanとVignaによる「xoroshiro128** 1.0」アルゴリズムの状態を保持します。
  • 各生成関数が呼び出されるたびに、この構造体の s0 と s1 が更新され、新しい乱数値が生成されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な使用例）**:
  • contrib/tsm_system_rows/tsm_system_rows.c
  • contrib/tsm_system_time/tsm_system_time.c
  • src/backend/utils/adt/numeric.c
  • src/backend/utils/adt/pseudorandomfuncs.c
  • src/backend/utils/misc/sampling.c
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • pg_global_prng_state: プロセス全体で共有されるグローバルな状態変数。

失敗モードと境界条件

  • **暗号論的強度**: 本構造体を用いた生成器は暗号学的に安全ではありません。パスワードや暗号鍵の生成には使用しないでください。

保守担当者向けメモ

  • 本構造体は src/common/pg_prng.c で実装されている乱数生成関数の引数として渡されることを前提としています。
  • 状態の初期化漏れやゼロ初期化を防ぐ設計を維持してください。
