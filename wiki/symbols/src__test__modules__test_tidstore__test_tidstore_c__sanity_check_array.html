sanity_check_array

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_tidstore/test_tidstore.c:147

目的

SQL インターフェースから渡された配列引数が、TidStore テストモジュールで処理可能な形式（非 NULL かつ一次元）であることを検証する。

シグネチャまたは形状

static void sanity_check_array(ArrayType *ta)

入力と出力

  • **入力**: ArrayType *ta - 検証対象の PostgreSQL 配列型。
  • **出力**: なし。検証に失敗した場合は ereport によりエラーを発生させ、処理を中断する。

前提条件と不変条件

  • 入力される配列は ArrayType 構造体として正当である必要がある。

アルゴリズムまたは振る舞い

1. 配列内に NULL 値が含まれているかを確認し、含まれている場合は ERRCODE_NULL_VALUE_NOT_ALLOWED でエラーを報告する。 2. 配列の次元数を確認し、1 次元を超える（多次元配列である）場合は ERRCODE_DATA_EXCEPTION でエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_tidstore/test_tidstore.c 内の SQL インターフェース関数（do_set_block_offsets 等）から呼び出される。
  • **呼び出し先**:
  • array_contains_nulls (PostgreSQL 内部関数)
  • ereport (PostgreSQL エラー報告マクロ)

関連データ構造

  • ArrayType: PostgreSQL の配列型表現。

失敗モードと境界条件

  • **NULL 値**: 配列内に NULL が存在する場合、エラーとなる。
  • **多次元配列**: 2 次元以上の配列が渡された場合、エラーとなる。
  • **空配列**: 0 次元の配列は本関数のチェックを通過する（ARR_NDIM(ta) > 1 の条件に抵触しないため）。

保守担当者向けメモ

  • 本関数はテストモジュール内での入力バリデーション専用である。将来的に他のテスト関数で同様の制約が必要になった場合、共通ユーティリティへの昇格を検討すること。
