ltxtq_in

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_io.c:449

目的

SQLの ltxtquery 型に対する入力関数として、テキスト形式のクエリ文字列を内部バイナリ形式（ltxtquery 構造体）へ変換します。

シグネチャまたは形状

Datum ltxtq_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して渡される、検索クエリを表すC文字列（char *）。
  • **出力**: 変換された ltxtquery * を Datum 型として返却。解析失敗時は PG_RETURN_NULL() を実行。

前提条件と不変条件

  • 入力文字列は ltxtquery として妥当な構文である必要があります。
  • 内部処理において queryin 関数が呼び出されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) から入力文字列を取得します。 2. queryin 関数を呼び出し、文字列のトークン化および逆ポーランド記法への変換を行います。 3. queryin が NULL を返した場合は、PG_RETURN_NULL() により入力を拒否します。 4. 成功した場合は、構築された ltxtquery 構造体へのポインタを PG_RETURN_POINTER で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSQLインターフェース（型変換処理）。
  • **呼び出し先**: queryin（内部関数）。
  • 注: ソースコード上の直接的な呼び出し元として PG_FUNCTION_INFO_V1(ltxtq_in) が存在します。

関連データ構造

  • ltxtquery: 検索クエリを保持する内部バイナリ構造体。
  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に従う引数コンテキスト。

失敗モードと境界条件

  • 入力文字列の構文が不正な場合、queryin が NULL を返し、本関数は NULL を返却します。
  • メモリ不足やスタックオーバーフロー（再帰制限）が発生した場合の挙動は、内部の queryin およびその下位関数（makepol 等）の仕様に依存します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な構文解析ロジックは queryin に集約されています。
  • ltxtquery のバイナリ形式に変更を加える際は、ltxtq_recv との整合性にも注意してください。
