array_append

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:139

目的

既存の配列の末尾に新しい要素を追加し、新しい配列を生成して返します。

シグネチャまたは形状

Datum array_append(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • fcinfo (PG_FUNCTION_ARGS): SQL関数呼び出し情報。
  • 第1引数: 対象となる配列（ExpandedArrayHeader）。NULLの場合は空配列として扱われます。
  • 第2引数: 追加する要素（Datum）。
  • **出力**:
  • Datum: 要素が追加された新しい配列。

前提条件と不変条件

  • 入力配列は、空であるか、次元数が1である必要があります。
  • 多次元配列が渡された場合はエラーとなります。

アルゴリズムまたは振る舞い

1. fetch_array_arg_replace_nulls を呼び出し、第1引数の配列を展開形式で取得します。 2. 第2引数の要素を取得します（NULLの場合は (Datum) 0 を使用）。 3. 配列の次元数（ndims）を確認します。

  • ndims == 1 の場合: 現在の配列の境界（lbound）と次元数（dims）から、新しい要素を挿入するインデックスを計算します。
  • ndims == 0 の場合: インデックスを 1 とします。
  • それ以外の場合: ERRCODE_DATA_EXCEPTION を発生させます。

    • 4. fcinfo->flinfo->fn_extra からキャッシュされた型情報（ArrayMetaState）を取得します。 5. array_set_element を呼び出し、指定したインデックスに要素を挿入して新しい配列を構築します。 6. 結果を PG_RETURN_DATUM で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから直接呼び出されるほか、src/backend/optimizer/prep/prepagg.c 等のプランナ関連コードで参照されます。
  • **呼び出し先**:
  • fetch_array_arg_replace_nulls
  • pg_add_s32_overflow
  • array_set_element
  • ereport (エラー発生時)

関連データ構造

  • ExpandedArrayHeader: 展開された配列のヘッダ構造。
  • ArrayMetaState: 配列の要素型に関するメタデータ（fn_extra にキャッシュされる）。

失敗モードと境界条件

  • **エラー**:
  • 配列が多次元の場合: ERRCODE_DATA_EXCEPTION。
  • インデックス計算時に整数オーバーフローが発生した場合: ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE。

保守担当者向けメモ

  • 本関数は fn_extra を利用して型情報をキャッシュしており、同一クエリ内での繰り返し呼び出し時のカタログ検索コストを削減しています。
  • array_append_support がプランナサポート関数として定義されており、最適化の対象となっています。
