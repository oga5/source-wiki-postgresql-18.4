PQtrace

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:34

目的

PostgreSQL 接続（PGconn）に関連付けられた通信プロトコルのデバッグトレースを有効化し、指定されたファイルストリームへ出力先を設定します。

シグネチャまたは形状

void PQtrace(PGconn *conn, FILE *debug_port);

入力と出力

  • conn: トレース対象の PGconn 構造体へのポインタ。
  • debug_port: トレース出力先の FILE ストリーム。
  • 出力: なし（戻り値は void）。

前提条件と不変条件

  • conn が NULL の場合、関数は何もせず終了します。
  • debug_port が NULL の場合、既存のトレース設定を解除（PQuntrace を呼び出し）して終了します。

アルゴリズムまたは振る舞い

1. conn が NULL であれば直ちに復帰します。 2. PQuntrace(conn) を呼び出し、既存のトレース設定を初期化します。 3. debug_port が NULL であれば処理を終了します。 4. conn->Pfdebug に debug_port を割り当てます。 5. conn->traceFlags を 0 にリセットします。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/libpq_pipeline/libpq_pipeline.c 等のテストコードから呼び出されることが確認されています。
  • 呼び出し先: PQuntrace（同一ファイル内）。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。Pfdebug フィールドに出力先ストリーム、traceFlags にトレース設定を保持します。

失敗モードと境界条件

  • conn が無効なポインタである場合、未定義の動作を引き起こす可能性があります。
  • debug_port が書き込み不可能なストリームである場合、以降のトレース出力処理で失敗する可能性があります。

保守担当者向けメモ

  • 本関数は PQuntrace を内部で呼び出すことで、トレース設定の排他性を保証しています。
  • 新しいトレースフラグを追加した場合は、traceFlags の初期化処理が適切か確認してください。
