xl_hash_add_ovfl_page

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash_xlog.h:80

目的

ハッシュインデックスにおいて、オーバーフローページを追加する操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_hash_add_ovfl_page
{
    uint16      bmsize;         /* ビットマップページのサイズ */
    bool        bmpage_found;   /* ビットマップページが見つかったか否か */
} xl_hash_add_ovfl_page;

入力と出力

  • **入力**: ハッシュインデックスのオーバーフローページ追加処理時に、この構造体に現在のビットマップサイズとページ発見状態を格納し、WALレコードとして書き込みます。
  • **出力**: リカバリ時に XLogRecGetData を通じて読み出され、インデックスの再構築に使用されます。

前提条件と不変条件

  • 本構造体は、ハッシュインデックスの物理的な変更を再現するために使用されます。
  • bmpage_found が true の場合、対応するビットマップページがWALレコードのバックアップブロックとして存在している必要があります。

アルゴリズムまたは振る舞い

1. **書き込み時**: ハッシュインデックスのオーバーフローページ追加操作において、本構造体をWALレコードのペイロードとして含めます。 2. **リカバリ時**: リカバリ処理（hash_redo）において、WALレコードから本構造体をキャストして取得します。この情報に基づき、インデックスのページ状態を更新します。

呼び出し元と呼び出し先

  • **呼び出し元（書き込み側）**: src/backend/access/hash/hashovfl.c 内のオーバーフローページ追加処理。
  • **呼び出し先（読み取り側）**: src/backend/access/hash/hash_xlog.c 内の hash_redo 関数、および src/backend/access/rmgrdesc/hashdesc.c 内のログ記述関数。

関連データ構造

  • src/include/access/hash_xlog.h: 本構造体が定義されているヘッダーファイル。
  • src/backend/access/hash/hash_xlog.c: 本構造体を利用してリカバリロジックを実装するソース。

失敗モードと境界条件

  • 構造体のサイズやフィールドの整合性が取れていない場合、WALの読み込みやリカバリ処理で不正なメモリ参照が発生する可能性があります。
  • offsetof を用いた計算（src/include/access/hash_xlog.h:87）により、構造体のパディングやサイズが考慮されています。

保守担当者向けメモ

  • 本構造体はハッシュインデックスの物理的なWALログの一部です。構造体を変更する場合は、hash_redo および hash_desc の実装を必ず更新してください。
