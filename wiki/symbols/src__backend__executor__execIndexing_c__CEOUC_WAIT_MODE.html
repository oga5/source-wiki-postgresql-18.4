CEOUC_WAIT_MODE

定義

  • 種別: enum
  • 定義箇所: src/backend/executor/execIndexing.c:123

目的

排他制約（Exclusion Constraint）チェック時における、競合発生時の待機動作を制御するために使用される列挙型です。

シグネチャまたは形状

typedef enum
{
	CEOUC_WAIT,
	CEOUC_NOWAIT,
	CEOUC_LIVELOCK_PREVENTING_WAIT,
} CEOUC_WAIT_MODE;

入力と出力

  • 本シンボル自体は列挙型であり、関数引数として渡されることで、制約チェックロジックの動作モードを決定します。

前提条件と不変条件

  • 排他制約チェックは、タプル挿入後に実行されます。
  • CEOUC_LIVELOCK_PREVENTING_WAIT は、投機的挿入（INSERT ... ON CONFLICT）において、デッドロックやライブロックを回避するために使用されます。

アルゴリズムまたは振る舞い

  • CEOUC_WAIT: 競合が発生した場合、対象のロックが解放されるまで待機します。
  • CEOUC_NOWAIT: 競合が発生した場合、待機せずに直ちにエラーを発生させます。
  • CEOUC_LIVELOCK_PREVENTING_WAIT: 投機的挿入において、トランザクションID（XID）に基づく優先順位付けを行い、デッドロックを回避しつつ待機します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execIndexing.c 内の check_exclusion_or_unique_constraint 等の制約チェック関数が引数として受け取ります。
  • 呼び出し先: 本シンボル自体は列挙型であり、直接関数を呼び出すことはありません。

関連データ構造

  • ResultRelInfo: インデックス情報がキャッシュされる構造体。
  • 本列挙型は、制約チェックの制御フローにおいて waitMode パラメータとして利用されます。

失敗モードと境界条件

  • 競合検出時に CEOUC_NOWAIT が指定されている場合、競合があれば即座に失敗（エラー）となります。
  • 待機モード選択が不適切な場合、デッドロックが発生する可能性があります（特に投機的挿入時）。

保守担当者向けメモ

  • 本列挙型は check_exclusion_or_unique_constraint の動作を決定する重要なフラグです。新しい待機戦略を追加する場合は、この列挙型を拡張し、対応するロジックを execIndexing.c 内の制約チェック関数に実装してください。
