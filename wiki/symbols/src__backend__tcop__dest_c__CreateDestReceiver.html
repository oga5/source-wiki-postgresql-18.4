CreateDestReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/dest.c:112

目的

指定された CommandDest 列挙値に基づき、クエリ実行結果の出力先を管理する DestReceiver オブジェクトを生成・返却するファクトリ関数です。

シグネチャまたは形状

DestReceiver *CreateDestReceiver(CommandDest dest);

入力と出力

  • **入力**: dest (CommandDest 型) - 出力先の種類を指定する列挙値。
  • **出力**: DestReceiver * - 指定された出力先に対応するレシーバー構造体へのポインタ。

前提条件と不変条件

  • dest には定義済みの CommandDest 列挙値のいずれかを指定する必要があります。
  • 静的レシーバー（DestNone, DestRemoteSimple, DestDebug, DestSPI）が返された場合、呼び出し側はそのレシーバーを解放（rDestroy）してはなりません。
  • 動的に割り当てられたレシーバー（DestTuplestore, DestIntoRel 等）は、使用後に rDestroy メソッドを呼び出して適切に解放する必要があります。

アルゴリズムまたは振る舞い

1. dest 引数に基づき switch 文で分岐します。 2. 各ケースにおいて、以下のいずれかの処理を行います。

  • 専用の作成関数（例: printtup_create_DR）を呼び出して新しいインスタンスを生成する。
  • 静的に定義されたレシーバー（例: donothingDR）のポインタを unconstify して返却する。

    • 3. 定義されていない dest が渡された場合は pg_unreachable() を呼び出し、異常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_copy.c
  • src/backend/commands/copyto.c
  • src/backend/commands/extension.c
  • src/backend/commands/portalcmds.c
  • src/backend/executor/functions.c
  • src/backend/executor/spi.c
  • src/backend/replication/walsender.c
  • **呼び出し先**:
  • 各種 DestReceiver 生成関数（printtup_create_DR, CreateTuplestoreDestReceiver 等）。

関連データ構造

  • DestReceiver: 出力先ごとのメソッド（receiveSlot, rStartup, rShutdown, rDestroy）を保持する構造体。
  • CommandDest: 出力先を識別するための列挙型。

失敗モードと境界条件

  • 未定義の CommandDest 値が渡された場合、pg_unreachable() によりプログラムが停止します。
  • メモリ不足時には、動的生成を行うレシーバーの作成関数内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 新しい出力先を追加する場合は、CommandDest 列挙型への追加と、本関数の switch 文へのケース追加が必須です。
  • 静的レシーバーと動的レシーバーでメモリ管理の責務が異なるため、呼び出し側での rDestroy の扱いに注意してください。
