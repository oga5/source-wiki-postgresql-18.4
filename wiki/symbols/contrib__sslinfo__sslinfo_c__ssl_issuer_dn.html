ssl_issuer_dn

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:327

目的

現在のPostgreSQLセッションで使用されているSSL接続のクライアント証明書から、発行者（Issuer）の識別名（DN）を取得し、テキスト形式で返します。

シグネチャまたは形状

Datum ssl_issuer_dn(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQLのSQL関数インターフェース経由で渡される引数）。
  • **出力**: Datum 型（SQLの text 型に対応）。証明書が存在しない場合や発行者名が空の場合は NULL を返します。

前提条件と不変条件

  • グローバル変数 MyProcPort が有効であること。
  • SSL接続が確立されており、かつピア証明書が有効であること（MyProcPort->ssl_in_use および MyProcPort->peer_cert_valid が真であること）。

アルゴリズムまたは振る舞い

1. MyProcPort を参照し、SSLが使用されていない、または証明書が無効な場合は NULL を返します。 2. be_tls_get_peer_issuer_name を呼び出し、発行者名を issuer バッファ（サイズ NAMEDATALEN）に格納します。 3. バッファが空であれば NULL を返します。 4. cstring_to_text を使用してC文字列をPostgreSQLの text 型に変換し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロによる登録）。
  • **呼び出し先**: be_tls_get_peer_issuer_name（証明書情報の抽出）、cstring_to_text（データ型変換）。

関連データ構造

  • Port（MyProcPort として参照されるセッション情報構造体）

失敗モードと境界条件

  • SSL接続が確立されていない場合や、クライアント証明書が提示されていない場合は NULL を返します。
  • 発行者名が NAMEDATALEN を超える場合、be_tls_get_peer_issuer_name の実装依存の切り詰めが発生する可能性があります。

保守担当者向けメモ

  • 本関数は contrib/sslinfo/sslinfo.c 内で定義されています。
  • 内部で呼び出している be_tls_get_peer_issuer_name は、バックエンドのSSL処理層（src/backend/libpq/be-secure-openssl.c 等）に依存しています。
