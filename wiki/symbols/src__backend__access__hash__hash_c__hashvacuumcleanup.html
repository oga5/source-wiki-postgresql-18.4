hashvacuumcleanup

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:650

目的

VACUUM 処理の最終段階において、ハッシュインデックスの統計情報を更新し、インデックスのページ数を確定させる。

シグネチャまたは形状

IndexBulkDeleteResult *hashvacuumcleanup(IndexVacuumInfo *info, IndexBulkDeleteResult *stats)

入力と出力

  • **入力**:
  • info: インデックスに関する情報を含む IndexVacuumInfo 構造体。
  • stats: hashbulkdelete によって生成された統計情報（IndexBulkDeleteResult）。
  • **出力**:
  • 更新済みの IndexBulkDeleteResult 構造体へのポインタ。stats が NULL の場合は NULL を返す。

前提条件と不変条件

  • stats が NULL の場合（ANALYZE のみ実行時や hashbulkdelete が呼び出されなかった場合）、本関数は何も変更せず NULL を返す。

アルゴリズムまたは振る舞い

1. stats が NULL であるかを確認し、該当する場合は即座に終了する。 2. RelationGetNumberOfBlocks を使用して、対象インデックスの現在の総ページ数を取得する。 3. 取得したページ数を stats->num_pages に格納する。 4. 更新された stats を呼び出し元へ返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックス AM インターフェース経由で呼び出される（src/backend/access/hash/hash.c:93 にて amvacuumcleanup コールバックとして登録）。
  • **呼び出し先**: RelationGetNumberOfBlocks（マクロまたはインライン関数）。

関連データ構造

  • IndexVacuumInfo: VACUUM 実行時のコンテキスト情報。
  • IndexBulkDeleteResult: インデックスの削除結果および統計情報。

失敗モードと境界条件

  • stats が NULL の場合、処理を行わず NULL を返す。
  • RelationGetNumberOfBlocks が失敗した場合の挙動については、本関数のソースコード内では明示的なエラーハンドリングは記述されていない。

保守担当者向けメモ

  • 本関数は hashbulkdelete の後に実行されることを想定している。
  • インデックスの物理的なページ数更新のみを担う軽量な関数である。
