datumGetSize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datum.c:64

目的

指定された Datum の型情報（値渡しフラグおよび型長）に基づき、そのデータが占める物理的なバイトサイズを算出します。

シグネチャまたは形状

Size datumGetSize(Datum value, bool typByVal, int typLen)

入力と出力

  • **入力**:
  • value: サイズを測定する対象の Datum。
  • typByVal: 値渡し（pass-by-value）であれば true、参照渡し（pass-by-reference）であれば false。
  • typLen: 型の長さ。正の値は固定長、-1 は varlena（可変長）、-2 は cstring を示す。
  • **出力**:
  • Size: 計算されたバイトサイズ。

前提条件と不変条件

  • typByVal が true の場合、typLen は 0 より大きく sizeof(Datum) 以下である必要があります（アサーションにより検証）。
  • typLen が -1 または -2 の場合、value が指すポインタは有効である必要があります。無効なポインタが渡された場合はエラーを報告します。

アルゴリズムまたは振る舞い

1. **値渡しの場合**: typLen をそのままサイズとして返します。 2. **参照渡しの場合**:

  • typLen > 0: 固定長のため、typLen をそのまま返します。
  • typLen == -1: varlena 構造体として扱い、VARSIZE_ANY マクロを用いてヘッダー情報からサイズを算出します。
  • typLen == -2: cstring として扱い、strlen に 1（終端文字分）を加算して返します。
  • 上記以外: 不正な typLen としてエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/outfuncs.c
  • src/backend/utils/adt/datum.c（内部の複数箇所）
  • src/backend/utils/sort/tuplesortvariants.c
  • **呼び出し先**:
  • VARSIZE_ANY（マクロ）
  • strlen
  • ereport / elog（エラー報告）

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • varlena: 可変長データ型を保持する構造体。

失敗モードと境界条件

  • typLen が -1 または -2 で、value が指すポインタが NULL または無効な場合、ERRCODE_DATA_EXCEPTION を伴うエラーが発生します。
  • 定義されていない typLen が渡された場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は型情報（typByVal, typLen）の正確な提供に依存しています。呼び出し側で誤ったメタデータが渡されると、メモリ破壊や不正なポインタ参照を招く恐れがあるため注意してください。
