child_process_kind

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/launch_backend.c:172

目的

PostgreSQLのPostmasterが起動する各種子プロセス（バックエンド、補助プロセス等）のメタデータと実行エントリポイントを定義し、プロセス起動処理を抽象化するために使用されます。

シグネチャまたは形状

typedef struct
{
    const char *name;
    void (*main_fn) (const void *startup_data, size_t startup_data_len);
    bool shmem_attach;
} child_process_kind;

入力と出力

  • name: プロセス種別を識別する名称（文字列）。
  • main_fn: 子プロセスが実行するメイン関数への関数ポインタ。
  • shmem_attach: 子プロセス起動時に共有メモリへのアタッチが必要かを示すフラグ。

前提条件と不変条件

  • main_fn は、EXEC_BACKEND モードにおいて SubPostmasterMain から呼び出されることを想定しています。
  • shmem_attach が true の場合、子プロセスは共有メモリセグメントを正しく再アタッチできる環境にある必要があります。

アルゴリズムまたは振る舞い

本構造体は、child_process_kinds 配列の要素として保持されます。Postmasterが子プロセスを起動する際、この構造体を参照して適切なメイン関数を特定し、共有メモリの初期化要否を判断します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/launch_backend.c 内の child_process_kinds 配列定義（179行目）で参照されています。
  • 呼び出し先: 具体的な呼び出し先（main_fn に格納される関数）は、postmaster.c や各補助プロセスのメイン関数（BackendMain 等）ですが、本構造体自体はそれらを保持するコンテナとして機能します。

関連データ構造

  • child_process_kinds: 本構造体のインスタンスを格納する静的配列。
  • BackendParameters: EXEC_BACKEND モードでプロセス間で受け渡される状態情報の構造体。

失敗モードと境界条件

  • main_fn が NULL の場合、プロセス起動時に不正なメモリ参照が発生する可能性があります。
  • shmem_attach の設定が実際のプロセス要件と不一致の場合、共有メモリへのアクセス時にセグメンテーション違反や初期化エラーが発生します。

保守担当者向けメモ

  • 本構造体は EXEC_BACKEND モード（Windows等）におけるプロセス管理の要です。新しいプロセスタイプを追加する場合は、child_process_kinds 配列に新しいエントリを追加し、対応するメイン関数を実装する必要があります。
