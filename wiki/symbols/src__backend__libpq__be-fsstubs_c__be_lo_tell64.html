be_lo_tell64

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-fsstubs.c:297

目的

開いているラージオブジェクト（Large Object）の現在の読み書き位置（オフセット）を64ビット整数で取得します。

シグネチャまたは形状

Datum be_lo_tell64(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡されるラージオブジェクトディスクリプタ（int32）。
  • **出力**: 現在のオフセット位置を int64 型の Datum として返します。

前提条件と不変条件

  • 指定されたディスクリプタ fd が有効であること（0 <= fd < cookies_size かつ cookies[fd] が NULL でないこと）。
  • トランザクション内で呼び出される必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32(0) を使用して、引数からファイルディスクリプタ fd を取得します。 2. fd が範囲外であるか、対応する cookies 配列のエントリが NULL の場合、ERRCODE_UNDEFINED_OBJECT エラーを発生させます。 3. cookies[fd] に格納されたディスクリプタを引数として inv_tell を呼び出し、現在のオフセットを取得します。 4. 取得したオフセットを PG_RETURN_INT64 を介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されることを想定しています（具体的な呼び出し元のコードパスは提供されたソースからは特定できません）。
  • **呼び出し先**: inv_tell（src/backend/storage/large_object/inv_api.c で定義されるラージオブジェクトAPI）。

関連データ構造

  • cookies: 現在開いているラージオブジェクトのディスクリプタを保持する動的配列。
  • LargeObjectDesc: cookies 配列に格納されるラージオブジェクトの記述子構造体。

失敗モードと境界条件

  • **無効なディスクリプタ**: fd が負の値、cookies_size 以上、または対応するエントリが NULL の場合、ereport によりエラー終了します。

保守担当者向けメモ

  • 本関数は64ビットオフセットを扱うため、大規模なラージオブジェクトの操作に適しています。
  • inv_tell の戻り値が int64 であるため、型変換のオーバーフローを考慮する必要はありません。
