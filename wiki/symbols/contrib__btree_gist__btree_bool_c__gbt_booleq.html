gbt_booleq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bool.c:36

目的

2つの bool 型値が等しいかどうかを判定し、GiSTインデックスにおける比較演算をサポートします。

シグネチャまたは形状

static bool gbt_booleq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の bool 値へのポインタ（void * として渡される）。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: a と b が等しい場合は true、それ以外は false を返します。

前提条件と不変条件

  • a および b は、有効な bool 型の値へのポインタである必要があります。

アルゴリズムまたは振る舞い

1. 入力されたポインタ a と b を const bool * にキャストします。 2. それぞれのポインタが指す値をデリファレンスし、比較演算子 == を用いて等価性を評価します。 3. 評価結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_bool.c 内の静的配列定義（インデックス操作用の関数テーブルの一部として参照）。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行いません。

関連データ構造

  • bool: PostgreSQLの標準ブール型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、デリファレンスによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は bool 型専用の比較ロジックを実装した薄いラッパーです。
  • btree_utils_num.c 等の共通ユーティリティから呼び出されることを想定しています。
