PendingUnlinkEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/sync/sync.c:63

目的

チェックポイント完了後に物理的な削除（unlink）を行う必要があるファイルを追跡するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	FileTag		tag;			/* ハンドラおよびファイルを識別 */
	CycleCtr	cycle_ctr;		/* 要求時のチェックポイントサイクルカウンタ */
	bool		canceled;		/* キャンセル済みであれば true */
} PendingUnlinkEntry;

入力と出力

本構造体はデータ保持用であり、関数ではありません。palloc を用いてメモリ上に確保され、削除要求のリスト管理に使用されます。

前提条件と不変条件

  • tag は対象ファイルを一意に識別するために使用されます。
  • cycle_ctr は、チェックポイントの境界を跨いだ要求の処理順序を制御するために使用されます。

アルゴリズムまたは振る舞い

  • チェックポイント処理の事後処理において、pendingUnlinks リスト内の本構造体を走査し、ファイル削除を実行します。
  • 削除実行後、対応する PendingUnlinkEntry オブジェクトは pfree により解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/sync/sync.c 内の削除要求管理ロジック（SyncPostCheckpoint 等）から、リストへの追加や走査が行われます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • FileTag: ファイルを識別するためのタグ。
  • CycleCtr: チェックポイントのサイクルを管理するカウンタ。
  • pendingUnlinks: PendingUnlinkEntry を保持するリスト。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • ファイル削除時に競合が発生した場合、errno を確認して適切に処理する必要があります。

保守担当者向けメモ

  • 本構造体は SyncPostCheckpoint で走査されるため、リストのメモリ管理（pfree）が適切に行われていることを確認してください。
  • canceled フラグが true の場合、削除処理をスキップするロジックが実装されている必要があります。
