spgRedoVacuumLeaf

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgxlog.c:750

目的

SP-GiST インデックスページに対するバキューム操作（不要タプルの削除、プレースホルダー化、タプルの移動、チェーンの更新）の WAL リプレイを実行します。

シグネチャまたは形状

static void spgRedoVacuumLeaf(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（WAL レコードデータ）。レコード内には spgxlogVacuumLeaf 構造体および削除・移動・チェーン対象の OffsetNumber 配列が含まれます。
  • **出力**: なし（副作用としてインデックスページを更新）。

前提条件と不変条件

  • fillFakeState により、リプレイに必要な最小限の SpGistState が構築されていること。
  • WAL レコード内の各オフセット配列は、spgxlogVacuumLeaf ヘッダー直後に連続して配置されていること。
  • chainSrc で指定されたタプルは SPGIST_LIVE 状態であること（Assert で検証）。

アルゴリズムまたは振る舞い

1. XLogRecGetData を介して WAL レコードから制御情報と各オフセット配列を抽出します。 2. XLogReadBufferForRedo を呼び出し、対象ページをロック・取得します。 3. spgPageIndexMultiDelete を使用して、以下の順序でページを更新します。

  • toDead に指定されたタプルを SPGIST_DEAD に設定して削除。
  • toPlaceholder に指定されたタプルを SPGIST_PLACEHOLDER に設定。

    • 4. moveSrc と moveDest に基づき、ページ内のタプルを入れ替えます。 5. 入れ替え後の moveSrc 位置のタプルを SPGIST_PLACEHOLDER として削除します。 6. chainSrc に指定されたタプルの次オフセットを chainDest に更新します。 7. ページに LSN を書き込み、バッファをダーティとしてマークした後にロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: spg_redo（src/backend/access/spgist/spgxlog.c:959）
  • **呼び出し先**: fillFakeState, XLogReadBufferForRedo, spgPageIndexMultiDelete, PageGetItemId, PageGetItem, PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer

関連データ構造

  • spgxlogVacuumLeaf: WAL レコードのヘッダー構造体。
  • SpGistState: リプレイ時に使用するインデックスの状態管理構造体。
  • OffsetNumber: ページ内のタプル位置を示す識別子。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、ページ更新はスキップされます。
  • chainSrc で指定されたタプルが SPGIST_LIVE でない場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は、インデックスの物理的な整合性を維持するため、削除や移動の順序が重要です。
  • spgPageIndexMultiDelete は、指定された状態（SPGIST_DEAD や SPGIST_PLACEHOLDER）への遷移を管理します。
  • ページ内のタプル入れ替えは、ItemIdData を用いた直接的なメモリコピーで行われます。
