test_status_failed

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress.c:310

目的

テスト失敗時の状態記録と、終了時に出力する失敗テストリストへの追加、および失敗カウントの更新を行う。

シグネチャまたは形状

static void test_status_failed(const char *testname, double runtime, bool parallel)

入力と出力

  • testname (const char *): 失敗したテストの名前。
  • runtime (double): テストの実行時間（ミリ秒）。
  • parallel (bool): 並列実行されていたかどうかを示すフラグ。
  • 出力: なし（副作用としてグローバル変数およびログ出力に影響を与える）。

前提条件と不変条件

  • failed_tests が初期化されていない場合は、内部で makeStringInfo() を用いて初期化される。
  • 失敗したテスト名はカンマ区切りで failed_tests バッファに蓄積される。

アルゴリズムまたは振る舞い

1. failed_tests が未初期化であれば作成し、既に存在する場合はカンマを追記する。 2. 失敗したテスト名を failed_tests バッファに追加する。 3. グローバル変数 fail_count をインクリメントする。 4. test_status_print(false, testname, runtime, parallel) を呼び出し、失敗ステータスを標準出力へ反映させる。

関連データ構造

  • StringInfo (failed_tests): 失敗したテスト名を保持する動的文字列バッファ。

失敗モードと境界条件

  • メモリ不足: makeStringInfo や appendStringInfo が内部でメモリ確保に失敗した場合、プログラムは終了する可能性がある。

保守担当者向けメモ

  • 本関数はテスト終了時のサマリー表示を確実にするために failed_tests バッファを利用している。ログ出力の整合性を保つため、test_status_print の呼び出し順序や引数に変更を加える際は注意すること。
