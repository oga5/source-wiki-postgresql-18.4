plpgsql_ns_init

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_funcs.c:42

目的

PL/pgSQL 関数のコンパイル時における名前空間（スコープ）処理を初期化します。

シグネチャまたは形状

void plpgsql_ns_init(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 ns_top の状態を更新）

前提条件と不変条件

  • 新しい関数のコンパイルを開始する直前に呼び出される必要があります。
  • 呼び出し後、名前空間のスタックトップを示す ns_top は NULL にリセットされます。

アルゴリズムまたは振る舞い

グローバル変数 ns_top に NULL を代入し、名前空間のリンクリストを空の状態に初期化します。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/pl_comp.c 内のコンパイル処理（plpgsql_ns_init() として2箇所で確認）。
  • 呼び出し先: なし。

関連データ構造

  • ns_top: 名前空間アイテム（PLpgSQL_nsitem）のリンクリストの先頭を保持するグローバル変数。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。

保守担当者向けメモ

  • 本関数は名前空間管理の起点です。新しい関数のコンパイルごとに必ず実行される必要があるため、コンパイルフローの変更時には注意してください。
