IsTidRangeClause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/tidpath.c:150

目的

与えられた RestrictInfo が、TID（タプル識別子）に対する範囲比較演算（<, <=, >, >=）を含む句であるかを判定します。

シグネチャまたは形状

static bool IsTidRangeClause(RestrictInfo *rinfo, RelOptInfo *rel)

入力と出力

  • **入力**:
  • rinfo: 評価対象の RestrictInfo 構造体。
  • rel: 対象となるリレーションの RelOptInfo 構造体。
  • **出力**:
  • bool: 該当する範囲演算子を持つ場合は true、それ以外は false を返します。

前提条件と不変条件

  • rinfo は有効な RestrictInfo である必要があります。
  • IsBinaryTidClause が内部で呼び出され、対象がTIDに関する二項演算子であることが検証されます。

アルゴリズムまたは振る舞い

1. IsBinaryTidClause(rinfo, rel) を呼び出し、対象の句がTIDに関連する二項演算であるかを確認します。偽であれば false を返します。 2. rinfo->clause を OpExpr にキャストし、演算子 ID (opno) を取得します。 3. opno が以下のいずれかの定数と一致するかを判定します。

  • TIDLessOperator (<)
  • TIDLessEqOperator (<=)
  • TIDGreaterOperator (>)
  • TIDGreaterEqOperator (>=)

    • 4. 一致する場合は true を、それ以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/tidpath.c:410 (文脈から TidRangeQualFromRestrictInfoList 等の内部処理で使用されている可能性が高い)
  • **呼び出し先**:
  • IsBinaryTidClause

関連データ構造

  • RestrictInfo: クエリの制約条件を保持する構造体。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。
  • OpExpr: 演算子式を表すノード。

失敗モードと境界条件

  • rinfo が NULL の場合の挙動は、IsBinaryTidClause の実装に依存します。
  • opno が上記4つの演算子以外（例: 等価演算子）である場合、この関数は false を返します。

保守担当者向けメモ

  • 本関数は TidRangePath の生成可否を判断するための重要な述語です。
  • 新たなTID関連の演算子が追加された場合は、本関数の判定条件を更新する必要があります。
