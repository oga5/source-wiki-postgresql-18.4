PageInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/bufpage.c:41

目的

PostgreSQLのディスクページ（8KBブロック）を初期化し、ページヘッダーの各フィールドを空の状態に設定します。

シグネチャまたは形状

void PageInit(Page page, Size pageSize, Size specialSize)

入力と出力

  • **入力**:
  • page: 初期化対象のメモリ領域へのポインタ。
  • pageSize: ページサイズ（通常は BLCKSZ）。
  • specialSize: ページ末尾に確保する特殊領域（Opaque領域）のサイズ。
  • **出力**: なし（page が指すメモリ領域が直接書き換えられます）。

前提条件と不変条件

  • pageSize は BLCKSZ と等しいこと。
  • pageSize は specialSize と SizeOfPageHeaderData の合計よりも大きいこと。
  • specialSize は MAXALIGN によってアライメント調整されます。

アルゴリズムまたは振る舞い

1. specialSize をアライメント調整します。 2. MemSet を使用して、ページ全体を0で初期化します。 3. ページヘッダー（PageHeader）の各フィールドを設定します。

  • pd_flags: 0に設定。
  • pd_lower: ヘッダーサイズ（SizeOfPageHeaderData）に設定。
  • pd_upper: ページ末尾から specialSize を引いた位置に設定。
  • pd_special: pd_upper と同じ位置に設定。

    • 4. PageSetPageSizeAndVersion を呼び出し、ページサイズとレイアウトバージョンをヘッダーに記録します。

関連データ構造

  • PageHeaderData: ページヘッダーの構造体。
  • Page: char* 型のエイリアス。

失敗モードと境界条件

  • pageSize が小さすぎる場合、Assert によりプログラムが停止します。

保守担当者向けメモ

  • 本関数はページを物理的に初期化する低レベル操作です。呼び出し側で適切なロックやバッファ管理が行われていることを前提としています。
  • pd_prune_xid は MemSet により InvalidTransactionId (0) に初期化されます。
