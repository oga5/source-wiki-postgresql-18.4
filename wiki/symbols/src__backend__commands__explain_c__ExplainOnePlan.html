ExplainOnePlan

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:493

目的

計画されたクエリ（PlannedStmt）を実行し、その実行計画および統計情報を整形して出力する中核関数です。

シグネチャまたは形状

void ExplainOnePlan(PlannedStmt *plannedstmt, IntoClause *into, ExplainState *es,
                    const char *queryString, ParamListInfo params,
                    QueryEnvironment *queryEnv, const instr_time *planduration,
                    const BufferUsage *bufusage,
                    const MemoryContextCounters *mem_counters)

入力と出力

  • **入力**:
  • plannedstmt: 実行対象の計画済みクエリツリー。
  • into: CREATE TABLE AS のための句（存在する場合）。
  • es: 出力形式やオプションを保持する ExplainState。
  • queryString, params, queryEnv: クエリ実行に必要なコンテキスト情報。
  • planduration, bufusage, mem_counters: プランニング段階の統計情報。
  • **出力**: なし（副作用として es に出力内容が蓄積され、クライアントへ送信される）。

前提条件と不変条件

  • plannedstmt->commandType が CMD_UTILITY であってはならない（Assert で検証）。
  • ANALYZE オプションが有効な場合、実際にクエリが実行されるため、データ変更を伴うクエリには注意が必要。

アルゴリズムまたは振る舞い

1. **初期化**: ExplainState の設定に基づき、実行フラグ（instrument_option）を決定。 2. **スナップショット**: PushCopiedSnapshot により現在のスナップショットを保護し、コマンドIDを更新。 3. **実行準備**: CreateQueryDesc でクエリ記述子を作成し、ExecutorStart を呼び出す。 4. **実行（ANALYZE時のみ）**: ExecutorRun および ExecutorFinish を呼び出し、実行統計を収集。 5. **出力生成**:

  • ExplainPrintPlan でプランツリーをダンプ。
  • プランニング時のバッファ使用量やメモリ統計を表示。
  • ANALYZE 時はトリガー実行時間、JIT 統計、シリアライズ統計を順次出力。

    • 6. **終了処理**: ExecutorEnd でクエリ実行を終了し、リソースを解放。PopActiveSnapshot でスナップショットを復元。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c (内部の EXPLAIN フロー)
  • src/backend/commands/prepare.c
  • **呼び出し先**:
  • ExecutorStart, ExecutorRun, ExecutorFinish, ExecutorEnd
  • ExplainPrintPlan, ExplainPrintTriggers, ExplainPrintJITSummary, ExplainPrintSerialize
  • CreateQueryDesc, FreeQueryDesc
  • explain_per_plan_hook (プラグイン用フック)

関連データ構造

  • PlannedStmt: 実行計画の構造体。
  • ExplainState: EXPLAIN の出力状態とオプションを管理。
  • QueryDesc: クエリ実行に必要な情報を集約した構造体。
  • DestReceiver: クエリ結果の出力先を抽象化。

失敗モードと境界条件

  • into 句が存在する場合、CreateIntoRelDestReceiver が使用され、結果はテーブル作成へ送られる。
  • es->serialize が設定されている場合、専用の DestReceiver を介してシリアライズ統計が収集される。
  • ANALYZE 指定時、into->skipData が真であれば NoMovementScanDirection で実行される。

保守担当者向けメモ

  • 本関数は EXPLAIN の実行フェーズの心臓部です。プランノードの表示内容を変更する場合は ExplainPrintPlan を、実行統計の収集ロジックを変更する場合は本関数の Executor 呼び出し周辺を確認してください。
  • explain_per_plan_hook を介した外部プラグインによる拡張がサポートされています。
