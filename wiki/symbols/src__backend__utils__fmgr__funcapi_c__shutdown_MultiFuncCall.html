shutdown_MultiFuncCall

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/funcapi.c:237

目的

集合を返す関数（SRF）の実行終了時に、関連するメモリコンテキストを解放し、FmgrInfo からの参照を解除するクリーンアップ処理を行います。

シグネチャまたは形状

static void shutdown_MultiFuncCall(Datum arg)

入力と出力

  • **入力**: arg (Datum) - FmgrInfo 構造体へのポインタを保持する Datum。
  • **出力**: なし（void）。

前提条件と不変条件

  • 入力 arg は、有効な FmgrInfo 構造体を指している必要があります。
  • FmgrInfo の fn_extra フィールドには、FuncCallContext 構造体へのポインタが格納されている必要があります。

アルゴリズムまたは振る舞い

1. arg を FmgrInfo ポインタに変換し、そこから FuncCallContext を取得します。 2. FmgrInfo の fn_extra を NULL に設定し、FmgrInfo との紐付けを解除します。 3. FuncCallContext 内の multi_call_memory_ctx を MemoryContextDelete で破棄します。これにより、FuncCallContext 自体を含む、SRFの複数回呼び出しに関連するすべてのデータが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照（src/backend/utils/fmgr/funcapi.c:230 等）から、ExprContext のコールバックとして登録・実行されることが示唆されています。
  • **呼び出し先**: MemoryContextDelete を呼び出します。

関連データ構造

  • FmgrInfo: 関数呼び出し情報を保持する構造体。
  • FuncCallContext: SRFの実行状態を保持する構造体。

失敗モードと境界条件

  • flinfo->fn_extra が NULL の場合、あるいは funcctx が無効なメモリを指している場合は、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は ExprContext のコールバックとして利用されることを想定しています。
  • fn_extra を NULL にリセットすることで、二重解放や不正なポインタ参照を防止しています。
