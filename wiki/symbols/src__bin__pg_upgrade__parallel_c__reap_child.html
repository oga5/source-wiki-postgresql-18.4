reap_child

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/parallel.c:277

目的

並列実行されている子プロセスまたはスレッドの終了を監視・回収し、リソースを解放して実行中のジョブ数を更新する。

シグネチャまたは形状

bool reap_child(bool wait_for_child)

入力と出力

  • **入力**:
  • wait_for_child (bool): true の場合、いずれかの子プロセスが終了するまでブロックする。false の場合、即座に終了確認を行い、完了したものがなければ直ちに制御を戻す。
  • **出力**:
  • bool: 子プロセス/スレッドが回収された場合は true、回収対象が存在しない（または待機中にタイムアウトした）場合は false を返す。

前提条件と不変条件

  • user_opts.jobs が 1 より大きく、かつ parallel_jobs が 0 より大きい場合にのみ処理を実行する。
  • 異常終了を検知した場合は pg_fatal を呼び出し、プロセス全体を停止させる。

アルゴリズムまたは振る舞い

1. **初期チェック**: 並列実行が不要な設定、または実行中のジョブがない場合は false を返す。 2. **プラットフォーム別処理**:

  • **Unix**: waitpid を使用。wait_for_child に応じて WNOHANG フラグを制御する。終了ステータスが 0 でない場合は異常終了とみなす。
  • **Windows**: WaitForMultipleObjects を使用してスレッド終了を待機。終了したスレッドのハンドルを CloseHandle で解放する。

    • 3. **状態更新**:

  • 終了したスレッドのインデックスを特定し、配列の末尾にあるアクティブなスレッド情報をその空きスロットへ移動させる（Windows のみ）。
  • グローバル変数 parallel_jobs をデクリメントする。

    • 4. **終了**: 回収に成功した場合は true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/dump.c, src/bin/pg_upgrade/parallel.c, src/bin/pg_upgrade/pg_upgrade.c, src/bin/pg_upgrade/relfilenumber.c 内の各所から、ジョブ投入前後の完了待ちとして呼び出される。
  • **呼び出し先**:
  • Unix: waitpid, pg_fatal
  • Windows: WaitForMultipleObjects, GetExitCodeThread, CloseHandle, pg_fatal

関連データ構造

  • parallel_jobs (グローバル変数): 現在実行中のジョブ数。
  • thread_handles (Windows): スレッドハンドルの配列。
  • cur_thread_args (Windows): スレッド引数構造体の配列。

失敗モードと境界条件

  • **異常終了**: 子プロセス/スレッドが 0 以外のステータスで終了した場合、pg_fatal によりプログラムが強制終了する。
  • **タイムアウト**: wait_for_child が false の場合、待機対象がなければ即座に false を返す。
  • **Windows 制限**: WaitForMultipleObjects の仕様により、最大ジョブ数は MAXIMUM_WAIT_OBJECTS (64) に制限される。

保守担当者向けメモ

  • Windows 環境では、スレッド終了後に配列の詰め直し（スロットの再配置）を行っているため、ポインタ操作の整合性に注意すること。
  • Unix と Windows でエラーハンドリングの粒度が異なるため、修正時は両方のプラットフォームでの動作確認が必要。
