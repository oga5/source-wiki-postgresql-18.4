oauth_init

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-oauth.c:52

目的

OAuth 2.0 / OIDC 認証セッションを開始するための状態構造体（fe_oauth_state）を初期化し、SASL 認証フローの準備を行う。

シグネチャまたは形状

static void *oauth_init(PGconn *conn, const char *password, const char *sasl_mechanism);

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn オブジェクト。
  • password: 認証に使用するパスワード（またはトークン）。
  • sasl_mechanism: 使用する SASL メカニズム名。
  • **出力**:
  • 成功時: 初期化された fe_oauth_state 構造体へのポインタ。
  • 失敗時: NULL（メモリ確保失敗時）。

前提条件と不変条件

  • sasl_mechanism は NULL であってはならない。
  • sasl_mechanism は OAUTHBEARER_NAME と一致する必要がある（一致しない場合はアサーションエラーとなる）。

アルゴリズムまたは振る舞い

1. sasl_mechanism が OAUTHBEARER であることを検証する。 2. fe_oauth_state 構造体分のメモリを calloc で確保する。 3. 状態を FE_OAUTH_INIT に設定する。 4. 接続オブジェクト conn を状態構造体に保持させる。 5. 確保した状態構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: libpq の SASL 認証フレームワーク（fe-auth.c 経由で呼び出される）。
  • **呼び出し先**: calloc（標準ライブラリ）。

関連データ構造

  • fe_oauth_state: 認証セッションの状態を保持する構造体。
  • PGconn: PostgreSQL 接続オブジェクト。

失敗モードと境界条件

  • calloc がメモリ確保に失敗した場合、NULL を返す。
  • sasl_mechanism が OAUTHBEARER 以外の場合、プログラムエラーとしてアサーションにより停止する。

保守担当者向けメモ

  • 本関数で確保されたメモリは、後続の oauth_free 関数によって解放される設計となっている。
  • 認証フローの初期状態として FE_OAUTH_INIT を設定しており、この後の認証ハンドシェイクの起点となる。
