ltree_in

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_io.c:174

目的

文字列形式の ltree 表現を解析し、PostgreSQL 内部で使用されるバイナリ構造体（ltree 型）へ変換する。

シグネチャまたは形状

Datum ltree_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して渡される、解析対象の文字列（char *）。
  • **出力**: 成功時は構築された ltree 構造体へのポインタを PG_RETURN_POINTER で返す。解析失敗時は PG_RETURN_NULL() を返す。

前提条件と不変条件

  • 入力文字列は ltree 型として妥当な階層パス形式である必要がある。
  • 内部で parse_ltree を呼び出すため、メモリ割り当て（palloc）が可能なコンテキストが必要である。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) から入力文字列を取得する。 2. parse_ltree 関数を呼び出し、文字列の構文解析と構造体の構築を行う。 3. parse_ltree が NULL を返した場合は、処理を中断し PG_RETURN_NULL() を実行する。 4. 正常に構造体が構築された場合、そのポインタを PG_RETURN_POINTER で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/ltree_op.c 内の各関数（DirectFunctionCall1 を介した呼び出し）。
  • **呼び出し先**:
  • parse_ltree（文字列の解析と構造体構築を担当）。

関連データ構造

  • ltree: 階層パスを保持する可変長構造体。
  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約に従う引数コンテナ。

失敗モードと境界条件

  • 入力文字列が ltree の構文規則に違反している場合、parse_ltree が NULL を返し、本関数は NULL を返す。
  • メモリ不足時には palloc 経由で PostgreSQL のエラーハンドリング機構が作動する。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(ltree_in) により、PostgreSQL の標準的な関数インターフェースとして登録されている。
  • 構文解析ロジックの詳細は parse_ltree に依存しているため、解析ルールの変更はそちらを確認すること。
