pgstat_report_connect

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_database.c:226

目的

データベースへの新規接続イベントを統計情報として記録します。

シグネチャまたは形状

void pgstat_report_connect(Oid dboid)

入力と出力

  • **入力**: dboid (Oid) - 接続先のデータベース識別子。
  • **出力**: なし。

前提条件と不変条件

  • pgstat_should_report_connstat() が真を返す場合にのみ統計更新が実行されます。
  • 呼び出し成功後、グローバル変数 pgLastSessionReportTime が現在のセッション開始時刻（MyStartTimestamp）で更新されます。

アルゴリズムまたは振る舞い

1. pgstat_should_report_connstat() を呼び出し、接続統計の報告が有効か確認します。無効な場合は即座に終了します。 2. グローバル変数 pgLastSessionReportTime を MyStartTimestamp に設定します。 3. pgstat_prep_database_pending(dboid) を呼び出し、指定されたデータベースの統計エントリ（PgStat_StatDBEntry）をローカルに準備します。 4. 取得したエントリの sessions カウンタをインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c:4323 (直接の呼び出しを確認)
  • **呼び出し先**:
  • pgstat_should_report_connstat (内部関数)
  • pgstat_prep_database_pending (内部関数)

関連データ構造

  • PgStat_StatDBEntry: データベース単位の統計情報を保持する構造体。本関数はこの中の sessions フィールドを更新します。

失敗モードと境界条件

  • pgstat_should_report_connstat() が偽を返す場合、統計情報の更新はスキップされます。
  • pgstat_prep_database_pending() が失敗した場合の挙動については、本ソース excerpt 内では定義されていません。

保守担当者向けメモ

  • 本関数はデータベース単位の統計管理の一部です。接続統計の集計ロジックを変更する場合は、pgstat_prep_database_pending によるエントリ管理との整合性に注意してください。
