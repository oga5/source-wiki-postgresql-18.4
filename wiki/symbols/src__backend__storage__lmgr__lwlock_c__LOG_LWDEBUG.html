LOG_LWDEBUG

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lwlock.c:295

目的

軽量ロック（LWLock）の動作に関するデバッグ情報を、PostgreSQL のログシステムを通じて出力する。

シグネチャまたは形状

inline static void LOG_LWDEBUG(const char *where, LWLock *lock, const char *msg)

入力と出力

  • **入力**:
  • where: ログ出力元を示す文字列（関数名など）。
  • lock: 対象となる LWLock 構造体へのポインタ。
  • msg: 出力するデバッグメッセージ。
  • **出力**: なし（void）。Trace_lwlocks が有効な場合に限り、ereport を介してログを出力する。

前提条件と不変条件

  • グローバル変数 Trace_lwlocks が真である場合にのみログが出力される。
  • T_NAME(lock) マクロが、渡された lock ポインタから有効なトランチ名を取得できること。

アルゴリズムまたは振る舞い

1. Trace_lwlocks フラグを確認する。 2. 有効な場合、ereport(LOG, ...) を呼び出す。 3. ログ出力時には errhidestmt(true) および errhidecontext(true) を指定し、冗長なステートメントやコンテキスト情報を抑制する。 4. where、T_NAME(lock)、lock のアドレス、および msg をフォーマットして出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: LWLockRelease、LWLockAcquire、LWLockConditionalAcquire、LWLockAcquireOrWait 内の各所から呼び出される。
  • **呼び出し先**: ereport（PostgreSQL のログ出力関数）。

関連データ構造

  • LWLock: ロックの状態や待機リストを保持する構造体。

失敗モードと境界条件

  • Trace_lwlocks が無効な場合、この関数は何も行わない。
  • src/backend/storage/lmgr/lwlock.c:311 にて、マクロ定義により ((void)0) として無効化されるケースが存在する。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、デバッグ時のオーバーヘッドを最小限に抑える設計となっている。
  • ログが非常に冗長になることを防ぐため、ステートメントとコンテキストの表示を意図的に抑制している。
