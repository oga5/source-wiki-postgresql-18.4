transfer_thread_arg

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/parallel.c:36

目的

pg_upgrade において、データベースのテーブルスペースデータ転送処理を並列実行する際、各スレッドに渡す引数を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	DbInfoArr  *old_db_arr;
	DbInfoArr  *new_db_arr;
	char	   *old_pgdata;
	char	   *new_pgdata;
	char	   *old_tablespace;
} transfer_thread_arg;

入力と出力

  • **入力**: 構造体の各フィールドは、転送対象となる新旧データベースのメタデータ配列（DbInfoArr）および、新旧のデータディレクトリパス、対象テーブルスペースのパスを保持します。
  • **出力**: 本構造体自体はデータ保持用であり、直接的な戻り値はありません。

前提条件と不変条件

  • pg_malloc0 等を用いてメモリが適切に確保されている必要があります。
  • Windows 環境下での並列実行において、スレッド間で安全にデータを共有するために使用されます。

アルゴリズムまたは振る舞い

  • win32_transfer_all_new_dbs 関数に引数として渡され、スレッド内でのデータ転送処理に必要なコンテキストを提供します。
  • parallel_transfer_all_new_dbs 内で、user_opts.jobs に基づいて動的に配列が確保され、各スレッドに割り当てられます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/parallel.c 内の parallel_transfer_all_new_dbs が本構造体のインスタンスを生成・初期化します。
  • **呼び出し先**: win32_transfer_all_new_dbs 関数が本構造体を引数として受け取り、内部の情報を参照します。

関連データ構造

  • DbInfoArr: データベース情報を保持する配列構造体。
  • transfer_thread_args: 本構造体へのポインタを保持する静的配列（static transfer_thread_arg **transfer_thread_args）。

失敗モードと境界条件

  • メモリ確保失敗時（pg_malloc 関連）にプロセスが停止する可能性があります。
  • Windows 環境では MAXIMUM_WAIT_OBJECTS（64）の制限により、並列ジョブ数に上限があります。

保守担当者向けメモ

  • 本構造体は Windows 特有の並列処理（_beginthreadex）をサポートするために設計されています。
  • フィールドの追加や変更を行う際は、win32_transfer_all_new_dbs 側の参照ロジックと整合性をとる必要があります。
