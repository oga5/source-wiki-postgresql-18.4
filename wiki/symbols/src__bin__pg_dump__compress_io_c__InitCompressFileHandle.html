InitCompressFileHandle

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_io.c:193

目的

指定された圧縮仕様（pg_compress_specification）に基づき、ファイル I/O を抽象化するための CompressFileHandle 構造体を初期化し、割り当てます。

シグネチャまたは形状

CompressFileHandle *InitCompressFileHandle(const pg_compress_specification compression_spec);

入力と出力

  • **入力**: compression_spec（圧縮アルゴリズムや設定を含む構造体）
  • **出力**: 初期化済みの CompressFileHandle 構造体へのポインタ。

前提条件と不変条件

  • pg_malloc0 を使用してメモリを確保するため、メモリ不足時にはプロセスが終了する可能性があります。
  • compression_spec.algorithm に基づき、適切な初期化関数（InitCompressFileHandleNone, Gzip, LZ4, Zstd のいずれか）が呼び出されます。

アルゴリズムまたは振る舞い

1. CompressFileHandle 構造体用にメモリをゼロクリアして割り当てます。 2. compression_spec.algorithm の値を確認します。 3. アルゴリズムに応じて、以下のいずれかの初期化関数を呼び出し、ハンドルを構成します。

  • PG_COMPRESSION_NONE: InitCompressFileHandleNone
  • PG_COMPRESSION_GZIP: InitCompressFileHandleGzip
  • PG_COMPRESSION_LZ4: InitCompressFileHandleLZ4
  • PG_COMPRESSION_ZSTD: InitCompressFileHandleZstd

    • 4. 初期化されたハンドルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_io.c
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_dump/pg_backup_directory.c
  • **呼び出し先**:
  • pg_malloc0
  • InitCompressFileHandleNone
  • InitCompressFileHandleGzip
  • InitCompressFileHandleLZ4
  • InitCompressFileHandleZstd

関連データ構造

  • CompressFileHandle: ファイル操作の関数ポインタ群を保持する構造体。
  • pg_compress_specification: 圧縮アルゴリズムを指定する設定構造体。

失敗モードと境界条件

  • 未知の圧縮アルゴリズムが指定された場合、CompressFileHandle は初期化されず、各関数ポインタが NULL のまま返される可能性があります。

保守担当者向けメモ

  • 新しい圧縮アルゴリズムを追加する場合は、本関数の if-else ブロックに新しい条件分岐を追加し、対応する InitCompressFileHandle* 関数を実装してください。
