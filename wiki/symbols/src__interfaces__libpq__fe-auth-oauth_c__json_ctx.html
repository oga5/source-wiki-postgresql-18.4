json_ctx

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/fe-auth-oauth.c:168

目的

OAuth 2.0 / OIDC 認証プロセスにおけるサーバーからの JSON 応答を解析し、必要なメタデータ（status、scope、discovery_uri）を抽出するためのコンテキストを保持します。

シグネチャまたは形状

struct json_ctx {
    char       *errmsg;
    PQExpBufferData errbuf;
    int         nested;
    const char *target_field_name;
    char      **target_field;
    char       *status;
    char       *scope;
    char       *discovery_uri;
};

入力と出力

  • **入力**: JSON 解析処理中に、サーバーから受信した文字列データが target_field を通じて各メンバに格納されます。
  • **出力**: 解析エラーが発生した場合、errmsg に非 NULL 値が設定され、以降の処理が停止します。

前提条件と不変条件

  • errmsg が非 NULL の場合、解析処理は即座に停止する必要があります。
  • nested は解析の深さを表し、0 がルート階層を示します。
  • target_field は、status、scope、discovery_uri のいずれかのアドレスを指す必要があります。

アルゴリズムまたは振る舞い

1. JSON 解析の各ステップ（オブジェクト開始、フィールド開始、スカラー値取得など）において、現在のコンテキストとして本構造体が参照されます。 2. target_field_name と一致するキーが見つかった場合、対応する target_field に値がコピーされます。 3. 解析中にエラーが発生した場合は errmsg が設定され、呼び出し元に通知されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-oauth.c 内の JSON 解析コールバック関数群（oauth_json_object_start 等）および handle_oauth_sasl_error。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • PQExpBufferData: errmsg のメモリ管理に使用されるバッファ構造体。

失敗モードと境界条件

  • **解析エラー**: JSON 形式が不正、または期待するフィールドが見つからない場合、errmsg が設定され処理が中断されます。
  • **メモリ不足**: errbuf への書き込み時にメモリ不足が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は fe-auth-oauth.c 内の静的な解析フロー専用です。
  • target_field のポインタが指す先は、解析対象の JSON 構造と密接に結合しているため、フィールド追加時には status 等のメンバ定義と整合性を確認してください。
