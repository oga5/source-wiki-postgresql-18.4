pgp_s2k_read

定義

  • **種別**: 関数
  • **定義箇所**: contrib/pgcrypto/pgp-s2k.c:252

目的

PGPストリームからS2K（String-to-Key）設定情報を読み取り、PGP_S2K 構造体を初期化します。

シグネチャまたは形状

int pgp_s2k_read(PullFilter *src, PGP_S2K *s2k);

入力と出力

  • **入力**:
  • src: 読み込み元の PullFilter。
  • s2k: 読み取ったパラメータを格納する PGP_S2K 構造体へのポインタ。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は負の整数（エラーコード）を返します。

前提条件と不変条件

  • src は有効な PullFilter である必要があります。
  • s2k は書き込み可能なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. src から mode と digest_algo を順次読み取ります。 2. mode に基づき、以下の通り処理を分岐します。

  • **モード 0 (Simple)**: 追加の読み込みなし。
  • **モード 1 (Salted)**: 8バイトのソルトを s2k->salt に読み込みます。
  • **モード 3 (Iterated-Salted)**: 8バイトのソルトを読み込んだ後、反復回数（iter）を読み取ります。

    • 3. 定義されていないモードが指定された場合、PXE_PGP_BAD_S2K_MODE を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-decrypt.c:642
  • contrib/pgcrypto/pgp-pubkey.c:365
  • **呼び出し先**:
  • GETBYTE マクロ（ストリームからの1バイト読み込み）
  • pullf_read_fixed（ストリームからの固定長読み込み）

関連データ構造

  • PGP_S2K: S2Kパラメータ（モード、ハッシュアルゴリズム、ソルト、反復回数）を保持する構造体。
  • PullFilter: PGPデータストリームを抽象化する構造体。

失敗モードと境界条件

  • ストリームの終端や読み込みエラーが発生した場合、pullf_read_fixed が返すエラーコードが伝播されます。
  • 未定義のS2Kモードがストリームに含まれている場合、PXE_PGP_BAD_S2K_MODE が返されます。

保守担当者向けメモ

  • 本関数はRFC 4880のS2K仕様に基づいています。モード3の反復回数エンコードの詳細については、pgp.h の関連定義と併せて確認してください。
