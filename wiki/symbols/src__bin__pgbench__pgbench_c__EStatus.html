EStatus

定義

  • 種別: enum
  • 定義箇所: src/bin/pgbench/pgbench.c:456

目的

pgbench におけるトランザクション実行中のエラー状態を分類・識別するために使用されます。

シグネチャまたは形状

typedef enum EStatus
{
	ESTATUS_NO_ERROR = 0,
	ESTATUS_META_COMMAND_ERROR,
	ESTATUS_SERIALIZATION_ERROR,
	ESTATUS_DEADLOCK_ERROR,
	ESTATUS_OTHER_SQL_ERROR,
} EStatus;

入力と出力

  • 本シンボルは列挙型であり、関数への引数としてエラー状態を渡す際や、関数の戻り値としてエラーの種類を通知するために使用されます。

前提条件と不変条件

  • ESTATUS_NO_ERROR は値 0 を持ち、エラーが発生していない正常な状態を示します。

アルゴリズムまたは振る舞い

  • トランザクション実行中に発生したエラーを、再試行の可否やエラーの種類に基づいて分類します。
  • canRetryError(EStatus estatus) 関数において、特定の EStatus 値が再試行可能かどうかの判定基準として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/bin/pgbench/pgbench.c:631: トランザクション状態の保持に使用。
  • src/bin/pgbench/pgbench.c:1479: 関数引数として使用。
  • src/bin/pgbench/pgbench.c:3236: 関数の戻り値型として使用。
  • src/bin/pgbench/pgbench.c:3254: canRetryError の引数として使用。
  • src/bin/pgbench/pgbench.c:4590: getResultString の引数として使用。
  • **呼び出し先**:
  • 列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • CState: クライアントの状態を保持する構造体内で、現在のトランザクションのエラー状態を追跡するために利用されます。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 新しいエラータイプを追加する場合は、canRetryError 関数内の判定ロジックが適切に更新されているか確認してください。
