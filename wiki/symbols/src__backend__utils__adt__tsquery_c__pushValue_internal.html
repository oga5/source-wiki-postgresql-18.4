pushValue_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery.c:546

目的

tsquery の解析過程において、オペランド（検索語）情報を生成し、解析状態のポーランド記法リスト（polstr）の先頭に追加します。

シグネチャまたは形状

static void pushValue_internal(
    TSQueryParserState state, 
    pg_crc32 valcrc, 
    int distance, 
    int lenval, 
    int weight, 
    bool prefix
)

入力と出力

  • **入力**:
  • state: 解析状態を保持する TSQueryParserState 構造体。
  • valcrc: オペランドのハッシュ値。
  • distance: オペランドの距離情報。
  • lenval: オペランドの長さ。
  • weight: オペランドの重み。
  • prefix: プレフィックス検索かどうかを示すフラグ。
  • **出力**: なし（state->polstr を更新）。

前提条件と不変条件

  • distance が MAXSTRPOS 未満であること。
  • lenval が MAXSTRLEN 未満であること。
  • 上記条件を満たさない場合、state->escontext を介してエラーが報告されます。

アルゴリズムまたは振る舞い

1. distance および lenval の境界チェックを行い、超過している場合はエラーを返します。 2. palloc0 を使用して QueryOperand 構造体をメモリ確保し、各引数および QI_VAL 型を割り当てます。 3. lcons を使用して、作成した QueryOperand を state->polstr リストの先頭に挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery.c 内の解析ロジック（pushValue_internal への直接的な呼び出しが確認されています）。
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • lcons (リスト操作)
  • ereturn (エラーハンドリング)

関連データ構造

  • TSQueryParserState: 解析中の状態を保持する構造体。
  • QueryOperand: tsquery のオペランドを表現する構造体。

失敗モードと境界条件

  • distance >= MAXSTRPOS: ERRCODE_PROGRAM_LIMIT_EXCEEDED を発生させます。
  • lenval >= MAXSTRLEN: ERRCODE_PROGRAM_LIMIT_EXCEEDED を発生させます。

保守担当者向けメモ

  • 本関数は tsquery の内部表現であるポーランド記法リストを構築する重要なステップです。
  • state->escontext を使用しているため、呼び出し元でエラーコンテキストが適切に設定されている必要があります。
