Zstd_open_write

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_zstd.c:542

目的

Zstandard 圧縮形式で出力するためのファイルを開き、圧縮用ファイルハンドルを初期化します。

シグネチャまたは形状

static bool Zstd_open_write(const char *path, const char *mode, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • path: 出力対象のファイルパス（拡張子なし）。
  • mode: ファイルオープンモード（例: "wb"）。
  • CFH: 圧縮ファイルハンドル構造体へのポインタ。
  • **出力**:
  • 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • path は有効な文字列である必要があります。
  • CFH は初期化済みであり、open_func コールバックが適切に設定されている必要があります。

アルゴリズムまたは振る舞い

1. 入力された path に .zst 拡張子を付与したファイル名を生成します。 2. CFH->open_func を呼び出し、生成したファイル名、フラグ -1、および指定された mode を渡してファイルを開きます。 3. open_func の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_zstd.c 内で CFH->open_write_func にこの関数が代入されています。
  • **呼び出し先**:
  • CFH->open_func（CompressFileHandle が保持する関数ポインタ）。

関連データ構造

  • CompressFileHandle: 圧縮処理におけるファイル操作を抽象化するための構造体。

失敗モードと境界条件

  • path が長すぎて MAXPGPATH を超える場合、sprintf によるバッファオーバーフローのリスクがあります。
  • CFH->open_func が失敗した場合、そのエラー状態がそのまま伝播されます。

保守担当者向けメモ

  • 本関数は pg_dump の圧縮抽象化層の一部です。ファイル名に強制的に .zst を付与する仕様であるため、他の圧縮形式との一貫性に注意してください。
