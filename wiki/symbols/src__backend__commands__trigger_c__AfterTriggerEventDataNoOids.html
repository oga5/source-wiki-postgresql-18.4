AfterTriggerEventDataNoOids

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/trigger.c:3723

目的

AFTER トリガーの遅延実行イベントにおいて、OID を含まない行データ（ItemPointerData）を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct AfterTriggerEventDataNoOids
{
    TriggerFlags ate_flags;
    ItemPointerData ate_ctid1;
    ItemPointerData ate_ctid2;
} AfterTriggerEventDataNoOids;

入力と出力

  • **入力**: トリガー実行時に発生するイベントのフラグおよび対象行の物理位置（ctid）。
  • **出力**: sizeof(AfterTriggerEventDataNoOids) バイトのメモリ領域として確保され、トリガーイベントキューの一部として利用されます。

前提条件と不変条件

  • ate_ctid1 および ate_ctid2 は、対象となるタプルの物理的な位置（ItemPointer）を指し示します。
  • 本構造体は OID を持たないテーブルのトリガーイベントを想定しています。

アルゴリズムまたは振る舞い

  • AFTER トリガーのイベント発生時、トリガーエンジンはイベントの種類に応じて適切なデータ構造を選択します。本構造体は、OID を持たないテーブルに対して、最大2つの ItemPointer を格納してイベントをキューイングするために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/trigger.c 内のトリガーイベント保存処理（AfterTriggerSaveEvent 等）から、イベントデータのサイズ計算やメモリ割り当ての文脈で参照されます。
  • **呼び出し先**: 提供されたソース情報からは、特定の関数呼び出し関係は特定できません。

関連データ構造

  • TriggerFlags: トリガーの属性や状態を示すフラグ。
  • ItemPointerData: タプルの物理位置を特定するデータ構造。
  • AfterTriggerEvent: 本構造体を含む、より広範なトリガーイベント管理構造体。

失敗モードと境界条件

  • 本構造体は固定サイズであるため、格納する ctid が2つを超えるような複雑なイベントには対応できません。その場合は別のデータ構造が使用されます。

保守担当者向けメモ

  • 本構造体は AfterTriggerEventData ファミリーの一部です。OID の有無や ctid の数に応じて構造体が使い分けられているため、変更を加える際は src/backend/commands/trigger.c 内の他の AfterTriggerEventData* 構造体との整合性に注意してください。
