目的

暗号学的に安全な乱数生成機能を利用するための互換性インターフェースを提供します。

シグネチャまたは形状

void pg_strong_random_init(void);

入力と出力

  • 入力: なし
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • 本関数は、PostgreSQLの起動初期段階で呼び出される可能性があるため、バックエンド固有のインフラ（elog() や palloc() など）に依存してはなりません。

アルゴリズムまたは振る舞い

  • 現在の実装では、いかなる初期化処理も行わない「no-op（何もしない）」関数です。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/fork_process.c:117 での呼び出しが確認されています。
  • 呼び出し先: なし。

関連データ構造

  • なし。

失敗モードと境界条件

  • 処理を行わないため、失敗モードは存在しません。

保守担当者向けメモ

  • 本関数は互換性のために維持されています。将来的に特定のプラットフォームで乱数生成器の明示的な初期化が必要となった場合に備えたプレースホルダーです。
