CopyMultiInsertInfoCleanup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfrom.c:715

目的

CopyMultiInsertInfo 構造体が保持するマルチインサート用バッファのリストを解放し、各バッファに関連付けられたリソースをクリーンアップする。

シグネチャまたは形状

static inline void CopyMultiInsertInfoCleanup(CopyMultiInsertInfo *miinfo)

入力と出力

  • **入力**: miinfo (CopyMultiInsertInfo *) - クリーンアップ対象のマルチインサート情報構造体へのポインタ。
  • **出力**: なし（副作用としてメモリ解放とリソースクリーンアップを実行）。

前提条件と不変条件

  • miinfo は有効な CopyMultiInsertInfo 構造体を指している必要がある。
  • miinfo->multiInsertBuffers 内の各要素は、CopyMultiInsertBufferCleanup によって適切に処理可能な状態であること。

アルゴリズムまたは振る舞い

1. miinfo->multiInsertBuffers リスト内の各 CopyMultiInsertBuffer に対して CopyMultiInsertBufferCleanup を呼び出し、個別のバッファリソースを解放する。 2. 最後に list_free を呼び出し、バッファリスト自体を保持していたメモリ領域を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c 内の CopyMultiInsertInfoCleanup（再帰的呼び出しではなく、リスト走査の文脈）および src/backend/commands/copyfrom.c:1500。
  • **呼び出し先**: CopyMultiInsertBufferCleanup。

関連データ構造

  • CopyMultiInsertInfo: マルチインサートのバッファ管理状態を保持する構造体。
  • CopyMultiInsertBuffer: 個別のリレーションに対するタプルバッファ。

失敗モードと境界条件

  • miinfo が NULL の場合、リスト走査時にクラッシュする可能性がある（ソースコード上、ガード条件は含まれていない）。
  • リストが空の場合、foreach ループはスキップされ、list_free のみが実行される。

保守担当者向けメモ

  • 本関数は static inline で定義されており、COPY コマンドの終了処理やエラー発生時のリソース解放パスで使用される。
  • CopyMultiInsertBufferCleanup が内部でリレーションのクローズやメモリ解放を行うため、本関数を呼び出す前にバッファのフラッシュが必要な場合は、事前に CopyMultiInsertInfoFlush を呼び出す必要がある。
