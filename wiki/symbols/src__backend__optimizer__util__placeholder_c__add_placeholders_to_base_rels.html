add_placeholders_to_base_rels

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:357

目的

クエリ最適化の初期段階において、単一のリレーション（ベースリレーション）で評価可能な PlaceHolderVar (PHV) を、該当するリレーションのターゲットリスト (reltarget->exprs) に追加します。

シグネチャまたは形状

void add_placeholders_to_base_rels(PlannerInfo *root)

入力と出力

  • **入力**: root (PlannerInfoへのポインタ)。クエリ全体のプランニング情報および placeholder_list を保持します。
  • **出力**: なし（副作用として root 内のベースリレーションのターゲットリストを更新します）。

前提条件と不変条件

  • root->placeholder_list には、クエリ内で特定されたすべての PlaceHolderInfo が格納されていること。
  • 対象となる PHV は、スキャンレベルで計算されるため、外部結合による NULL 生成の影響を受けていないこと（ph_var->phnullingrels == NULL であること）。

アルゴリズムまたは振る舞い

1. root->placeholder_list を走査し、各 PlaceHolderInfo を取得します。 2. ph_eval_at（評価が必要なリレーションの集合）が単一のリレーション（varno）のみで構成されているかを確認します。 3. その PHV が ph_eval_at 以外の場所でも必要とされている（ph_needed が eval_at を含む）場合、対象リレーションを特定します。 4. find_base_rel を用いて該当する RelOptInfo を取得し、PHV をその reltarget->exprs に追加します。 5. 追加時には copyObject を使用して PHV のコピーを作成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planmain.c:246
  • **呼び出し先**:
  • find_base_rel (PostgreSQL内部関数)
  • copyObject (PostgreSQL内部関数)
  • lappend (PostgreSQL内部関数)

関連データ構造

  • PlannerInfo: クエリ全体の最適化情報を保持する構造体。
  • PlaceHolderInfo: PHV の評価レベルや依存関係を管理する構造体。
  • RelOptInfo: ベースリレーションのコストやターゲットリストを保持する構造体。

失敗モードと境界条件

  • ph_eval_at が空、または複数のリレーションにまたがる場合、この関数ではターゲットリストへの追加は行われません。
  • phnullingrels が NULL でない場合、Assert によりデバッグビルドで停止します。

保守担当者向けメモ

  • 本関数はターゲットリストの式を追加するのみであり、reltarget のコストや幅（width）の再計算は行いません。これらは後続の処理で更新される前提となっています。
