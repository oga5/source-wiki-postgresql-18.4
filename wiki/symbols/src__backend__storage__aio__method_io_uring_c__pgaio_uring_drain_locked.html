pgaio_uring_drain_locked

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/storage/aio/method_io_uring.c:525

目的

io_uring の完了キュー（CQ）をスキャンし、保留中の非同期 I/O イベントを回収して処理を実行するヘルパー関数です。

シグネチャまたは形状

static void pgaio_uring_drain_locked(PgAioUringContext *context);

入力と出力

  • **入力**: PgAioUringContext *context - io_uring インスタンスとロックを保持するコンテキスト構造体。
  • **出力**: なし（副作用として I/O 完了処理を実行）。

前提条件と不変条件

  • 呼び出し元は context->completion_lock を LW_EXCLUSIVE モードで保持している必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. **エラーハンドリング設定**: pgaio_uring_completion_error_callback をエラーコンテキストスタックに登録します。 2. **キューの確認**: io_uring_cq_ready を使用して、現在処理可能な完了イベント数を取得します。 3. **バッチ処理**:

  • io_uring_peek_batch_cqe を用いて、最大 PGAIO_MAX_LOCAL_COMPLETED_IO 個の完了キューエントリ（CQE）を一度に取得します。
  • 取得した各 CQE に対して以下の処理を行います：
  • io_uring_cqe_get_data で関連付けられた PgAioHandle を取得。
  • io_uring_cqe_seen でカーネルに対してイベントの消費を通知。
  • pgaio_io_process_completion を呼び出し、I/O 結果を反映。

    • 4. **ループ**: 全ての保留イベントが処理されるまで繰り返します。 5. **後処理**: エラーコンテキストスタックを復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/method_io_uring.c 内の pgaio_uring_wait_one 等（完了イベントの回収が必要な箇所）。
  • **呼び出し先**: pgaio_io_process_completion（I/O 完了処理の実行）、liburing API（io_uring_cq_ready, io_uring_peek_batch_cqe, io_uring_cqe_seen）。

関連データ構造

  • PgAioUringContext: io_uring リング構造体と completion_lock を保持。
  • PgAioHandle: I/O 操作のハンドル。
  • io_uring_cqe: カーネルから返される完了イベント構造体。

失敗モードと境界条件

  • ready が 0 の場合はループに入らず即座に終了します。
  • ncqes が 0 の場合、または io_uring_peek_batch_cqe が期待通りに動作しない場合でも、ループ条件により安全に終了します。
  • 処理中にエラーが発生した場合、error_context_stack に登録されたコールバックが呼び出されます。

保守担当者向けメモ

  • 本関数は completion_lock を保持した状態で実行されるため、処理時間が長くなりすぎないよう注意が必要です。
  • PGAIO_MAX_LOCAL_COMPLETED_IO によるバッチ処理は、一度に大量のイベントを処理する際の効率化を目的としています。
