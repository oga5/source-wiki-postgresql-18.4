DependencyGenerator_next

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/dependencies.c:203

目的

DependencyGenerator が保持する属性の組み合わせリストから、次の要素（属性の配列）を順次取得する。

シグネチャまたは形状

AttrNumber *DependencyGenerator_next(DependencyGenerator state)

入力と出力

  • **入力**: DependencyGenerator 型の構造体ポインタ state。
  • **出力**:
  • 成功時: 現在の属性組み合わせを指す AttrNumber 型の配列ポインタ。
  • 終了時: NULL。

前提条件と不変条件

  • state は DependencyGenerator_init によって初期化済みであること。
  • state->dependencies には、state->k 個の属性からなる組み合わせが state->ndependencies 個格納されていること。

アルゴリズムまたは振る舞い

1. state->current（現在のインデックス）が state->ndependencies（総数）に達しているか確認する。 2. 達している場合は NULL を返し、反復の終了を示す。 3. 達していない場合は、state->dependencies 内の現在の組み合わせ位置 state->k * state->current を指すポインタを返し、その後 state->current をインクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/dependencies.c 内のループ処理（例: 378行目付近）にて、生成された依存関係を順次処理するために呼び出される。
  • **呼び出し先**: 本関数は内部的なポインタ演算のみを行い、他の関数を呼び出さない。

関連データ構造

  • DependencyGenerator: 属性の組み合わせ生成状態を保持する構造体。current（現在の位置）、ndependencies（総数）、k（組み合わせのサイズ）、dependencies（データ配列）を保持する。

失敗モードと境界条件

  • すべての組み合わせを走査し終えた場合、NULL を返す。これはエラーではなく、反復の正常な終了条件である。

保守担当者向けメモ

  • 本関数は DependencyGenerator の内部状態を直接更新する。呼び出し側で state->current を操作しないよう注意すること。
  • 戻り値のポインタは state->dependencies 配列の一部を指しているため、DependencyGenerator_free が呼ばれた後は無効となる。
