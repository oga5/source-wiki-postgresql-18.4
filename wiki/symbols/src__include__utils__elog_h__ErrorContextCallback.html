ErrorContextCallback

定義

  • 種別: struct
  • 定義箇所: src/include/utils/elog.h:308

目的

エラー発生時に追加のコンテキスト情報をログ出力へ動的に付与するためのコールバック機構を提供します。

シグネチャまたは形状

typedef struct ErrorContextCallback
{
    struct ErrorContextCallback *previous;
    void        (*callback) (void *arg);
    void       *arg;
} ErrorContextCallback;

入力と出力

  • previous: スタック上の直前のコールバック構造体を指すポインタ。
  • callback: エラー発生時に実行される関数ポインタ。引数として arg を受け取ります。
  • arg: コールバック関数に渡される任意のデータへのポインタ。

前提条件と不変条件

  • 本構造体は、エラー発生時に実行されるコールバックのスタックを形成します。
  • previous メンバにより、複数のコールバックを連結したリスト構造として管理されます。

アルゴリズムまたは振る舞い

エラー報告処理（ereport 等）の過程で、登録されたコールバックがスタック順に呼び出されます。これにより、エラー発生時の詳細な状態（例：処理中のファイル名、行番号、クエリ情報など）をログに含めることが可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の各モジュールで本構造体のインスタンスが定義・利用されています。
  • contrib/file_fdw
  • contrib/postgres_fdw
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/access/transam/parallel.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/catalog/pg_proc.c
  • src/backend/commands/copyfrom.c
  • src/backend/commands/extension.c
  • src/backend/commands/foreigncmds.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/vacuumparallel.c
  • **呼び出し先**: 登録された callback 関数が実行されます。具体的な関数ポインタの割り当て先については、各利用箇所の実装に依存します。

関連データ構造

  • ErrorData: エラー発生時に収集される情報を保持する構造体。

失敗モードと境界条件

  • コールバック関数内でさらにエラーが発生した場合、再帰的なエラー処理がトリガーされます。elog.c に実装された再帰的エラー対策が適用されます。

保守担当者向けメモ

  • 本構造体はスタックとして機能するため、登録・解除の順序が重要です。
  • コールバック関数内で使用するデータ（arg）の生存期間が、エラー発生時まで保証されていることを確認してください。
