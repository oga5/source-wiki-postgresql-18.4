IndexScanDescData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/relscan.h:133

目的

インデックススキャンの実行状態を保持するための構造体です。インデックスアクセスメソッド（AM）がスキャンの進捗、パラメータ、および検索結果を管理するために使用します。

シグネチャまたは形状

typedef struct IndexScanDescData {
    /* スキャンパラメータ */
    Relation    heapRelation;
    Relation    indexRelation;
    Snapshot    xs_snapshot;
    int         numberOfKeys;
    int         numberOfOrderBys;
    ScanKey     keyData;
    ScanKey     orderByData;
    bool        xs_want_itup;
    bool        xs_temp_snap;

    /* インデックスタプルの削除制御 */
    bool        kill_prior_tuple;
    bool        ignore_killed_tuples;
    bool        xactStartedInRecovery;

    /* AM固有の状態 */
    void       *opaque;
    IndexScanInstrumentation *instrument;

    /* スキャン結果（インデックスオンリースキャン用） */
    IndexTuple  xs_itup;
    TupleDesc   xs_itupdesc;
    HeapTuple   xs_hitup;
    TupleDesc   xs_hitupdesc;
    ItemPointerData xs_heaptid;
    bool        xs_heap_continue;
    IndexFetchTableData *xs_heapfetch;

    /* 再チェックフラグ */
    bool        xs_recheck;

    /* ORDER BY 評価用 */
    Datum      *xs_orderbyvals;
    bool       *xs_orderbynulls;
    bool        xs_recheckorderby;

    /* 並列スキャン情報 */
    ParallelIndexScanDescData *parallel_scan;
} IndexScanDescData;

入力と出力

  • **入力**: スキャン対象の Relation、スナップショット、検索条件（ScanKey）、ソート条件（orderByData）。
  • **出力**: amgettuple または amgetbitmap を通じて、xs_heaptid（ヒープ上の位置）やインデックスタプル（xs_itup / xs_hitup）が返されます。

前提条件と不変条件

  • amgettuple 呼び出し時、インデックスオンリースキャンの場合は xs_itup または xs_hitup のいずれか（あるいは両方）が適切に設定される必要があります。
  • xs_recheck が真の場合、呼び出し元はスキャンキーを再評価する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: エグゼキュータが構造体を割り当て、スナップショットやキーを設定します。 2. **実行**: インデックスAMが opaque を使用して内部状態を管理し、タプルをフェッチします。 3. **更新**: kill_prior_tuple 等のフラグに基づき、インデックス内の不要タプルのマーク処理を行います。 4. **終了**: スキャン終了時に xs_temp_snap が真であればスナップショットの登録解除を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: エグゼキュータ（execnodes.h 内の IndexScanState 等が本構造体を保持）。
  • **呼び出し先**: 各インデックスアクセスメソッド（nbtree, gist, gin 等の amgettuple / amgetbitmap 実装）。
  • *注: 具体的な関数呼び出しの連鎖については、本ソース excerpt には含まれていません。*

関連データ構造

  • Relation: スキャン対象のインデックスおよびヒープリレーション。
  • ScanKeyData: 検索条件の定義。
  • IndexTuple, HeapTuple: フェッチされるタプルの形式。
  • ParallelIndexScanDescData: 並列スキャン時の共有メモリ状態。

失敗モードと境界条件

  • heapRelation が NULL の場合、インデックスオンリースキャンとして動作します。
  • xactStartedInRecovery が真の場合、インデックスタプルの削除や削除済みタプルの参照が制限されます。

保守担当者向けメモ

  • 本構造体は多くのAMで共通利用されるため、メンバの追加や変更は各AMの実装に影響を与えます。
  • 並列スキャン用の parallel_scan は共有メモリ上に配置されるため、アトミック操作やメモリレイアウトに注意してください。
