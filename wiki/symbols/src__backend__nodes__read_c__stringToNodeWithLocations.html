stringToNodeWithLocations

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/read.c:97

目的

シリアライズされた文字列からPostgreSQLの内部ノード構造（Nodeツリー）を復元します。特に、ノードのソース位置情報を含めたデシリアライズを行うためのインターフェースです。

シグネチャまたは形状

void *stringToNodeWithLocations(const char *str)

入力と出力

  • **入力**: str (シリアライズされたノード構造を表す文字列)
  • **出力**: 復元されたノード構造へのポインタ（void *）

前提条件と不変条件

  • 入力文字列 str は、outfuncs.c 等によって生成された有効なシリアライズ形式である必要があります。
  • 内部的に stringToNodeInternal を呼び出すため、同関数の再入可能性およびスレッドセーフティに関する制約（静的変数 pg_strtok_ptr の使用）に従います。

アルゴリズムまたは振る舞い

1. 引数として受け取った文字列 str をそのまま stringToNodeInternal 関数へ渡します。 2. 第2引数に true を指定することで、ソース位置情報を保持するモードでデシリアライズを実行します。 3. 復元されたノードツリーのルートポインタを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c (クエリプランやリストの復元に使用)
  • **呼び出し先**:
  • stringToNodeInternal (同ファイル内)

関連データ構造

  • Node: 復元対象となるPostgreSQLの内部ノード構造体。

失敗モードと境界条件

  • 入力文字列が不正な形式である場合、パーサー（nodeRead 等）がエラーを発生させる可能性があります。
  • メモリ不足時には、ノード生成プロセスでエラーが発生します。

保守担当者向けメモ

  • 本関数は stringToNodeInternal のラッパーです。位置情報付きのデシリアライズが必要な場合は本関数を使用してください。
  • DEBUG_NODE_TESTS_ENABLED が有効な環境下での動作を想定しています。
