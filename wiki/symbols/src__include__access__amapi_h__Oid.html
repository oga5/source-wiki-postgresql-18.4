Oid

定義

  • 種別: typedef
  • 定義箇所: src/include/access/amapi.h:105

目的

PostgreSQL システムカタログ内のオブジェクトを一意に識別するための識別子（Object Identifier）型です。

シグネチャまたは形状

  • typedef unsigned int Oid; （一般的な PostgreSQL の定義に基づく）

入力と出力

  • 入力: システムカタログの行やインデックス、演算子族などを特定するための数値。
  • 出力: データベース内の特定のオブジェクトを指し示す識別子。

前提条件と不変条件

  • データベース内で一意なオブジェクト識別子として機能します。
  • システムカタログの参照や、インデックスアクセスメソッドのメタデータ管理において、特定のオブジェクトを指すために使用されます。

アルゴリズムまたは振る舞い

  • amtranslate_strategy_function などの関数ポインタ定義において、演算子族（opfamily）を特定する引数として利用されます。
  • システムカタログ（pg_am 等）の検索や、インデックスのバリデーション処理において、対象となるリレーションやアクセスメソッドを特定するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/amcheck/verify_common.c: amcheck_lock_relation_and_check および index_checkable で使用。
  • contrib/amcheck/verify_gin.c: PG_GETARG_OID(0) を介してインデックスリレーションの OID を取得。
  • contrib/amcheck/verify_nbtree.c: 同様にインデックスリレーションの OID を取得。
  • 呼び出し先:
  • プリミティブ型であるため、特定の関数を呼び出すことはありません。

関連データ構造

  • IndexAmRoutine: インデックスアクセスメソッドのルーチン構造体。
  • OpFamilyMember: 演算子族のメンバー管理構造体。

失敗モードと境界条件

  • 無効な OID（InvalidOid）が渡された場合、システムカタログの検索失敗や、存在しないオブジェクトへのアクセスエラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは PostgreSQL 全体で多用される基本型です。amapi.h 内では主にインデックスアクセスメソッドに関連するメタデータ（演算子族やリレーションの識別）のために使用されています。
  • 変更を加える際は、システムカタログの整合性に影響を与える可能性があるため注意してください。
