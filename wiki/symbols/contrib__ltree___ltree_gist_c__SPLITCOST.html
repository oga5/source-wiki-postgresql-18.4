SPLITCOST

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/_ltree_gist.c:231

目的

GiSTインデックスの分割処理において、各エントリの分割コストを保持およびソートするために使用されるデータ構造です。

シグネチャまたは形状

typedef struct
{
	OffsetNumber pos;
	int32		cost;
} SPLITCOST;

入力と出力

  • pos (OffsetNumber): インデックス内のエントリ位置。
  • cost (int32): その位置のエントリに関連付けられたコスト値。

前提条件と不変条件

  • pos はインデックス内の有効なオフセット番号である必要があります。
  • cost は比較可能な整数値であり、分割アルゴリズムの基準として利用されます。

アルゴリズムまたは振る舞い

GiSTインデックスの分割時に、複数のエントリに対してコストを計算し、SPLITCOST の配列（costvector）に格納します。その後、qsort を用いて cost に基づくソートを行い、最適な分割位置を決定するプロセスで利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/hstore/hstore_gist.c および contrib/intarray/_int_gist.c 内の関数が、この構造体を配列として確保し、qsort に渡していることが確認されています。
  • 呼び出し先: qsort 関数が、比較関数（comparecost）を通じて本構造体の cost フィールドを参照します。

関連データ構造

  • OffsetNumber: PostgreSQLのインデックスページ内での位置を示す型。

失敗モードと境界条件

  • メモリ確保失敗: palloc を使用して SPLITCOST の配列を確保するため、メモリ不足時に失敗する可能性があります。
  • ソート時の比較: cost 値が等しい場合の挙動は、各モジュールの比較関数（comparecost）の実装に依存します。

保守担当者向けメモ

  • 本構造体は ltree モジュールだけでなく、hstore や intarray モジュールでも同様の目的で定義・利用されています。変更を加える際は、これら関連モジュールへの影響を考慮してください。
