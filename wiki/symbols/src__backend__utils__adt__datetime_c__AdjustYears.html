AdjustYears

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datetime.c:670

目的

pg_itm_in 構造体内の年（tm_year）に対して、指定された倍率（scale）を乗じた値（val）を加算し、更新する。

シグネチャまたは形状

static bool AdjustYears(int64 val, int scale, struct pg_itm_in *itm_in)

入力と出力

  • **入力**:
  • val: 加算する基準値（int64）。
  • scale: val に乗じる倍率（int）。
  • itm_in: 更新対象の pg_itm_in 構造体へのポインタ。
  • **出力**:
  • 成功した場合は true、オーバーフロー等により失敗した場合は false を返す。

前提条件と不変条件

  • val は INT_MIN から INT_MAX の範囲内である必要がある。
  • itm_in は有効な pg_itm_in 構造体を指している必要がある。

アルゴリズムまたは振る舞い

1. val が int32 の範囲外（INT_MIN 未満または INT_MAX 超）であれば false を返す。 2. val と scale の積を計算し、オーバーフローが発生しないか確認する（pg_mul_s32_overflow）。 3. 計算結果を itm_in->tm_year に加算し、オーバーフローが発生しないか確認する（pg_add_s32_overflow）。 4. 両方の計算が成功した場合のみ itm_in->tm_year を更新し、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c 内の複数の箇所（3748, 3755, 3762, 3769, 3996, 4019, 4037 行目付近）から呼び出されている。
  • **呼び出し先**:
  • pg_mul_s32_overflow
  • pg_add_s32_overflow

関連データ構造

  • struct pg_itm_in: 日付・時刻の内部表現を保持する構造体。

失敗モードと境界条件

  • val が int32 の範囲を超えている場合、処理を中断し false を返す。
  • val * scale の計算結果が int32 の範囲を超える場合、false を返す。
  • itm_in->tm_year への加算結果が int32 の範囲を超える場合、false を返す。

保守担当者向けメモ

  • 本関数は静的関数であり、datetime.c 内でのみ使用される。
  • オーバーフローチェックに pg_mul_s32_overflow および pg_add_s32_overflow を使用しているため、型安全性が確保されている。
