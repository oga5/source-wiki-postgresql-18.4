plpgsql_extra_warnings_assign_hook

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_handler.c:129

目的

PL/pgSQL の設定パラメータ（GUC）である plpgsql.extra_warnings の値が変更された際に、内部変数 plpgsql_extra_warnings を更新するための代入フックとして機能します。

シグネチャまたは形状

static void plpgsql_extra_warnings_assign_hook(const char *newvalue, void *extra)

入力と出力

  • newvalue: 設定される新しい値（文字列）。本関数内では直接使用されません。
  • extra: 設定値に対応する内部表現へのポインタ。int 型としてキャストして参照されます。
  • 出力: なし（void）。

前提条件と不変条件

  • extra 引数は、有効な int 型の値を指している必要があります。
  • 本関数は PostgreSQL の GUC システムから呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. extra 引数を int * にキャストします。 2. そのポインタが指す値をデリファレンスし、グローバル変数 plpgsql_extra_warnings に代入します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の GUC 管理システム（src/pl/plpgsql/src/pl_handler.c 内の GUC 定義構造体から参照）。
  • 呼び出し先: なし。

関連データ構造

  • plpgsql_extra_warnings: 本関数によって更新されるグローバル変数。

失敗モードと境界条件

  • extra が NULL または不正なメモリ領域を指している場合、デリファレンス時にセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は GUC の assign_hook として登録されるため、シグネチャを変更する場合は GUC 定義側の構造体定義と整合させる必要があります。
