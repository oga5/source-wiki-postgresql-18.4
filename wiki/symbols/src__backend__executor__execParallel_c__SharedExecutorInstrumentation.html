SharedExecutorInstrumentation

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/execParallel.c:98

目的

並列クエリ実行において、複数のワーカープロセスが生成した実行統計情報（Instrumentation）を共有メモリ上で集約・管理するための構造体です。

シグネチャまたは形状

struct SharedExecutorInstrumentation
{
    int         instrument_options;
    int         instrument_offset;
    int         num_workers;
    int         num_plan_nodes;
    int         plan_node_id[FLEXIBLE_ARRAY_MEMBER];
    /* array of num_plan_nodes * num_workers Instrumentation objects follows */
};

入力と出力

  • **入力**: 並列実行プランのノード数およびワーカー数に基づき、動的共有メモリ（DSM）上に確保されます。
  • **出力**: 実行終了後、各ワーカーが記録した統計情報がこの構造体を経由してリーダープロセスに集約されます。

前提条件と不変条件

  • plan_node_id は可変長配列として定義されており、構造体の末尾には num_plan_nodes * num_workers 分の Instrumentation オブジェクトが連続して配置される必要があります。
  • 構造体内の各フィールドは、並列実行の初期化フェーズで適切に設定される必要があります。

アルゴリズムまたは振る舞い

  • リーダープロセスは、並列実行の初期化時に本構造体をDSM上に割り当てます。
  • 各ワーカープロセスは、実行中に自身の担当するプランノードの統計情報を、本構造体内の対応するオフセット位置に書き込みます。
  • リーダープロセスは、ExecParallelFinish 等を通じてワーカーの終了を待機した後、本構造体から統計情報を読み取り、クエリ全体の実行結果として集約します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c 内の並列実行管理関数（ExecInitParallelPlan 等）が本構造体を操作します。
  • **呼び出し先**: 具体的な呼び出し先（関数）の詳細は提供されていませんが、統計情報の書き込みおよび読み取りを行う各実行ノードの処理と連携します。

関連データ構造

  • Instrumentation: 各プランノードの実行統計を保持する構造体。
  • ParallelExecutorInfo: 並列実行状態を管理する上位構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、並列実行の初期化が中断されます。
  • num_workers や num_plan_nodes が不整合な場合、メモリのオーバーランや不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリレイアウトの計算には offsetof マクロを使用してください（例: src/backend/executor/execParallel.c:710）。
  • 新しい並列対応ノードを追加する際は、統計情報の集約ロジックが本構造体のメモリ配置と整合しているか確認してください。
