pq_endmessage_reuse

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:313

目的

構築済みのメッセージをフロントエンドへ送信します。標準の pq_endmessage と異なり、送信後にバッファを解放（pfree）せず、再利用可能な状態で保持します。

シグネチャまたは形状

void pq_endmessage_reuse(StringInfo buf)

入力と出力

  • **入力**: StringInfo buf - 送信するメッセージデータとメタ情報（メッセージ型など）を保持するバッファ。
  • **出力**: なし（void）。

前提条件と不変条件

  • buf は pq_beginmessage または pq_beginmessage_reuse によって適切に初期化されている必要があります。
  • buf->cursor フィールドにメッセージ型（msgtype）が格納されている必要があります。

アルゴリズムまたは振る舞い

1. buf->cursor に保存されているメッセージ型、buf->data に格納されたメッセージ本体、および buf->len を引数として pq_putmessage を呼び出します。 2. pq_endmessage とは異なり、バッファのメモリ解放は行いません。これにより、同一の StringInfo 構造体を繰り返し使用してメッセージを送信することが可能です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printtup.c
  • src/backend/replication/walsender.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • pq_putmessage（内部で呼び出し）

関連データ構造

  • StringInfo: メッセージの構築および保持に使用される動的バッファ構造体。

失敗モードと境界条件

  • pq_putmessage が内部でエラーを発生させた場合、そのエラーが伝播します。
  • バッファの内容が不正な場合、プロトコルレベルの通信エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はバッファを解放しないため、呼び出し元のメモリコンテキストにおいて、不要になった時点で明示的に pfree するか、適切にライフサイクルを管理する必要があります。
  • explain_dr.c のコメントにある通り、実際の送信を伴うため、送信を抑制したい場合には使用できません。
