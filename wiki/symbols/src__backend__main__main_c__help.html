help

定義

  • 種別: function
  • 定義箇所: src/backend/main/main.c:383

目的

PostgreSQL サーバー（postgres バイナリ）のコマンドラインオプションおよび使用方法を標準出力に表示する。

シグネチャまたは形状

static void help(const char *progname)

入力と出力

  • **入力**: progname (実行ファイル名を示す文字列)
  • **出力**: なし（標準出力へのメッセージ出力のみ）

前提条件と不変条件

  • progname は有効な文字列ポインタであること。
  • printf が標準出力に対して書き込み可能であること。

アルゴリズムまたは振る舞い

1. サーバーの基本使用方法（Usage）を表示する。 2. サーバー起動時の標準オプション（-B, -c, -D 等）を列挙する。 3. 開発者向けオプション（-f, -O, -P 等）を列挙する。 4. シングルユーザーモードおよびブートストラップモード専用のオプションを列挙する。 5. 最後に、ドキュメントの参照先、バグ報告用メールアドレス、およびプロジェクトのホームページを表示する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内では、src/backend/main/main.c 内の main() 関数から呼び出されることが想定されるが、本ソース excerpt には直接の呼び出し箇所は含まれていない。
  • **呼び出し先**: 標準ライブラリの printf を使用する。
  • **注記**: ツールが検出した contrib/oid2name/oid2name.c 等の他の help 関数は、本シンボルとは別の定義である。

関連データ構造

  • PACKAGE_BUGREPORT: バグ報告先を保持するマクロ。
  • PACKAGE_NAME: プロジェクト名を保持するマクロ。
  • PACKAGE_URL: プロジェクトのホームページ URL を保持するマクロ。

失敗モードと境界条件

  • printf が失敗した場合の挙動は呼び出し元に依存する。
  • progname が NULL の場合、未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は static であり、src/backend/main/main.c 内でのみ使用される。
  • 新しい起動オプションを追加した際は、本関数内の表示リストにも必ず追記すること。
  • USE_SSL マクロによって SSL 関連オプションの表示が条件分岐される点に注意すること。
