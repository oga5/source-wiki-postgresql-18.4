ObjectAccessType

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/objectaccess.h:48

目的

PostgreSQLのSQLオブジェクトに対する操作イベントを識別し、外部プラグインや監査機能がフックを通じて監視・制御を行うための型定義です。

シグネチャまたは形状

typedef enum ObjectAccessType
{
    OAT_POST_CREATE,
    OAT_DROP,
    OAT_POST_ALTER,
    OAT_NAMESPACE_SEARCH,
    OAT_FUNCTION_EXECUTE,
    OAT_TRUNCATE,
} ObjectAccessType;

入力と出力

  • 入力: なし（列挙型定数）
  • 出力: フック関数（object_access_hook_type 等）の引数として渡され、発生したイベントの種類を通知します。

前提条件と不変条件

  • InvokeObject...Hook マクロを介して使用されることが推奨されます。
  • フック関数側で未知の ObjectAccessType が渡された場合のハンドリングが必要です（例: test_oat_hooks.c での "UNRECOGNIZED ObjectAccessType" 判定）。

アルゴリズムまたは振る舞い

コアコード（heap.c や dependency.c 等）でオブジェクト操作が発生した際、この列挙値がフック関数に渡されます。フック関数は値に応じて、権限チェックやログ記録などの固有処理を実行します。

呼び出し元と呼び出し先

  • 呼び出し元: コアコードの各操作実行箇所（InvokeObject...Hook マクロ経由）。
  • 呼び出し先: 登録された object_access_hook 関数。
  • 参照元（確認済み）:
  • contrib/sepgsql/hooks.c
  • src/test/modules/test_oat_hooks/test_oat_hooks.c

関連データ構造

  • object_access_hook_type: この型を引数に取る関数ポインタ型。
  • 各イベント固有のコンテキスト構造体（ObjectAccessPostCreate, ObjectAccessDrop 等）。

失敗モードと境界条件

  • 未知の列挙値が渡された場合、フック関数側で適切に処理されない可能性があります。
  • フック関数が NULL の場合、マクロにより呼び出しはスキップされます。

保守担当者向けメモ

  • 新しいオブジェクト操作イベントを追加する場合は、本列挙型に定数を追加し、対応する Invoke マクロおよび Run 関数を整備してください。
  • フック関数は頻繁に呼び出されるため、プラグイン実装時は処理の軽量性を維持してください。
