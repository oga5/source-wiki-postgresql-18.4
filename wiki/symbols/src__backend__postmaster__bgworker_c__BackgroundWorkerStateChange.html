BackgroundWorkerStateChange

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/bgworker.c:245

目的

共有メモリ上のバックグラウンドワーカーの状態をスキャンし、新規登録されたワーカーの初期化や、終了要求が出されたワーカーの処理（終了指示やスロット解放）をPostmaster側で行う。

シグネチャまたは形状

void BackgroundWorkerStateChange(bool allow_new_workers)

入力と出力

  • **入力**:
  • allow_new_workers (bool): true の場合、新規登録されたワーカーの起動を許可する。false の場合、新規ワーカーは即座に終了対象としてマークされる。
  • **出力**: なし（副作用として共有メモリおよびPostmaster内部の管理リストを更新する）。

前提条件と不変条件

  • max_worker_processes と共有メモリ上の BackgroundWorkerData->total_slots が一致している必要がある。不一致の場合はログを出力して即座に終了する。
  • Postmasterは共有メモリのロックを取得できないため、in_use フラグとメモリバリア（pg_read_barrier / pg_memory_barrier）を用いたロックレスプロトコルに従うこと。

アルゴリズムまたは振る舞い

1. 共有メモリ上の全スロットをループ走査する。 2. in_use フラグが立っているスロットについて、FindRegisteredWorkerBySlotNumber を用いて既に管理下にあるか確認する。 3. **既存ワーカーの場合**: terminate フラグが立っていれば SIGTERM を送信し、プロセスが未起動であれば終了報告を行う。 4. **新規ワーカーの場合**:

  • allow_new_workers が false ならば terminate を強制する。
  • terminate が設定されている場合は、bgw_notify_pid への通知（SIGUSR1）を行い、スロットを解放する。
  • それ以外の場合は、PostmasterContext にワーカー情報をコピーし、BackgroundWorkerList に追加して起動準備を整える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/postmaster.c (Postmasterのメインループ内)
  • **呼び出し先**:
  • FindRegisteredWorkerBySlotNumber
  • ReportBackgroundWorkerPID
  • PostmasterMarkPIDForWorkerNotify
  • kill (システムコール)

関連データ構造

  • BackgroundWorkerData (共有メモリ上のワーカー配列)
  • BackgroundWorkerSlot
  • RegisteredBgWorker
  • BackgroundWorkerList (Postmasterのプライベートリスト)

失敗モードと境界条件

  • **メモリ不足**: MemoryContextAllocExtended が失敗した場合、ログを出力して処理を中断する。
  • **共有メモリ破損**: max_worker_processes との不一致や、文字列がNUL終端されていない可能性を考慮し、ascii_safe_strlcpy を用いた安全なコピーを行う。
  • **PID無効**: PostmasterMarkPIDForWorkerNotify が失敗した場合、通知PIDを0にリセットし、デバッグログを出力する。

保守担当者向けメモ

  • 本関数はPostmasterプロセスからのみ呼び出されることを想定している。
  • 共有メモリの読み取り時は常にデータ破損の可能性を考慮した「パラノイア的」なチェックを維持すること。
  • BGWORKER_CLASS_PARALLEL フラグを持つワーカーの終了時は、parallel_terminate_count の更新を忘れないこと。
