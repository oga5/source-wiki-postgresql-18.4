TimeLineHistoryEntry

定義

  • 種別: struct
  • 定義箇所: src/include/access/timeline.h:25

目的

データベースの分岐（リカバリやスタンバイ昇格）に伴うタイムラインの履歴情報を保持し、特定のWAL位置がどのタイムラインに属するかを追跡するために使用されます。

シグネチャまたは形状

typedef struct
{
    TimeLineID    tli;
    XLogRecPtr    begin;
    XLogRecPtr    end;
} TimeLineHistoryEntry;

入力と出力

  • **入力**: タイムライン履歴ファイル（timeline.history）の各行からパースされたデータ。
  • **出力**: 履歴リストの要素として、リカバリプロセスやバックアップ管理において特定のWAL位置の所属タイムラインを判定するために利用されます。

前提条件と不変条件

  • begin はそのタイムラインの開始位置（inclusive）を示します。
  • end はそのタイムラインの終了位置（exclusive）を示します。
  • end が InvalidXLogRecPtr である場合、そのタイムラインが現在進行中であることを示します。

アルゴリズムまたは振る舞い

履歴ファイルは古いタイムラインから新しいタイムラインへと順に記述されます。システムは、この構造体のリストを走査することで、特定のWAL位置がどのタイムラインの区間に含まれるかを検索します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/transam/timeline.c: 履歴の読み込みおよび検索処理。
  • src/backend/access/transam/xlogrecovery.c: リカバリ中のタイムライン判定。
  • src/backend/backup/backup_manifest.c: バックアップマニフェスト生成時の参照。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TimeLineID: タイムラインを識別するID。
  • XLogRecPtr: WAL上の位置を示すポインタ。
  • List (nodes/pg_list.h): TimeLineHistoryEntry の集合を保持するために使用されます。

失敗モードと境界条件

  • end が InvalidXLogRecPtr の場合、そのタイムラインは無限（現在進行中）として扱われます。
  • 履歴ファイルが破損している場合、パース処理において不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/access/timeline.h で定義されており、src/bin/pg_rewind/timeline.c でも同様の解析ロジックで利用されます。変更時は両方の整合性に注意してください。
