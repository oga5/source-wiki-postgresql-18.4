FPlt

定義

  • 種別: function
  • 定義箇所: src/include/utils/geo_decls.h:58

目的

浮動小数点数 A が B より小さいかどうかを、許容誤差（EPSILON）を考慮して判定する。

シグネチャまたは形状

static inline bool FPlt(double A, double B)

入力と出力

  • **入力**: 比較対象となる2つの倍精度浮動小数点数 A および B。
  • **出力**: A + EPSILON < B が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 比較には EPSILON を用いた曖昧な比較が行われる。
  • NaN（非数）が含まれる場合、常に false を返す。
  • 浮動小数点演算の性質上、推移律を満たさない可能性がある。

アルゴリズムまたは振る舞い

A に EPSILON を加算した値が B より小さい場合にのみ真を返す。これにより、計算誤差の範囲内での「より小さい」という関係を判定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c
  • src/backend/access/spgist/spgkdtreeproc.c
  • src/backend/utils/adt/geo_ops.c
  • **呼び出し先**:
  • math.h（EPSILON の定義や浮動小数点演算に関連）

関連データ構造

  • double 型（浮動小数点数）

失敗モードと境界条件

  • NaN が入力された場合、比較結果は常に false となる。
  • 厳密な数値比較が必要な場面では、誤差による推移律の欠如に注意が必要である。

保守担当者向けメモ

  • 本関数は数値解析の専門家によって設計されたものではないため、厳密な計算が求められる用途での使用には注意すること。
  • src/include/utils/geo_decls.h で定義される他の比較関数（FPeq, FPne, FPle, FPgt）と一貫した設計思想に基づいている。
