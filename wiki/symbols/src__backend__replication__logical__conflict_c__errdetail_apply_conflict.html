errdetail_apply_conflict

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/conflict.c:197

目的

論理レプリケーションの競合発生時に、エラーログの詳細メッセージ（errdetail）を構築し、競合に関連する行データやメタ情報を付加する。

シグネチャまたは形状

static void errdetail_apply_conflict(
    EState *estate, ResultRelInfo *relinfo, ConflictType type,
    TupleTableSlot *searchslot, TupleTableSlot *localslot, TupleTableSlot *remoteslot,
    Oid indexoid, TransactionId localxmin, RepOriginId localorigin,
    TimestampTz localts, StringInfo err_msg
)

入力と出力

  • **入力**: 実行状態（estate）、リレーション情報（relinfo）、競合タイプ（type）、各スロット（searchslot, localslot, remoteslot）、インデックスOID、ローカル行のトランザクション情報（localxmin, localorigin, localts）、および追記先の文字列バッファ（err_msg）。
  • **出力**: なし（err_msg を直接更新する）。

前提条件と不変条件

  • CT_INSERT_EXISTS、CT_UPDATE_EXISTS、CT_MULTIPLE_UNIQUE_CONFLICTS の場合、indexoid が有効であり、かつ呼び出し元が当該インデックスに対して RowExclusiveLock を保持している必要がある。
  • 構築される err_detail は空であってはならない。

アルゴリズムまたは振る舞い

1. initStringInfo で一時的な詳細メッセージ用バッファを初期化する。 2. ConflictType に基づき、競合の種類に応じた説明文を生成する。

  • トランザクションID、タイムスタンプ、およびレプリケーションオリジン名（replorigin_by_oid を使用）を組み合わせてメッセージを構成する。

    • 3. build_tuple_value_details を呼び出し、競合に関与するキー値や行データの詳細文字列を取得する。 4. 取得した詳細文字列を一時バッファに追記する。 5. 最終的に、err_msg が空でない場合は改行を挿入し、構築した詳細情報を err_msg に結合する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/conflict.c 内の ReportApplyConflict から呼び出される。
  • **呼び出し先**:
  • initStringInfo, appendStringInfo, appendStringInfoString, appendStringInfoChar
  • get_rel_name, timestamptz_to_str, replorigin_by_oid
  • build_tuple_value_details
  • CheckRelationOidLockedByMe

関連データ構造

  • ConflictType: 競合の種類を定義する列挙型。
  • StringInfo: エラーメッセージ構築に使用される可変長文字列構造体。
  • EState, ResultRelInfo: 実行時のリレーションおよび実行状態情報。

失敗モードと境界条件

  • localorigin が無効な場合や、対応するオリジンが削除されている場合は「non-existent origin」としてメッセージを生成する。
  • localts が存在しない（0）場合は、タイムスタンプを含まない簡潔なメッセージを生成する。
  • val_desc が取得できない場合、行データの詳細付加はスキップされる。

保守担当者向けメモ

  • 本関数は StringInfo を使用してメモリを動的に確保するため、呼び出し元でのメモリ管理に注意すること。
  • 競合メッセージの多言語化（_() マクロ）が適用されているため、メッセージの変更時は翻訳への影響を考慮すること。
  • replorigin_by_oid の結果が偽の場合、オリジンが削除されたものとして処理される。
