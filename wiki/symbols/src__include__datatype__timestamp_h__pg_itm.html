pg_itm

定義

  • 種別: struct
  • 定義箇所: src/include/datatype/timestamp.h:65

目的

日時情報を分解して保持するための構造体です。主に日時演算や変換処理において、年、月、日、時、分、秒、マイクロ秒の各要素を個別に扱うために使用されます。

シグネチャまたは形状

struct pg_itm
{
	int			tm_usec;
	int			tm_sec;
	int			tm_min;
	int64		tm_hour;
	int			tm_mday;
	int			tm_mon;
	int			tm_year;
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは日時要素を表す整数値です。tm_hour は int64 型で定義されており、広範な時間計算に対応しています。

前提条件と不変条件

  • tm_hour は int64 型であり、他のフィールドよりも広い範囲を保持可能です。
  • 日時計算の過程で Interval 型との相互変換が行われることが想定されています。

アルゴリズムまたは振る舞い

日時データを各フィールドに分解して保持することで、月や日の長さが可変であるといった日時計算特有の複雑さを吸収します。interval2itm 関数などを通じて、Interval 型から本構造体への変換が行われます。

呼び出し元と呼び出し先

  • **呼び出し先**: 本構造体はデータ構造であるため、直接的な呼び出し先はありません。
  • **呼び出し元（参照元）**:
  • src/backend/utils/adt/datetime.c: EncodeInterval 等で利用。
  • src/backend/utils/adt/formatting.c: 日時フォーマット処理で利用。
  • src/backend/utils/adt/timestamp.c: interval2itm 関数等で Interval との変換に使用。

関連データ構造

  • Interval: time（マイクロ秒）, day, month の3フィールドで構成される期間データ。本構造体との相互変換が行われます。

失敗モードと境界条件

  • interval2itm を使用した変換において、pg_itm の各フィールドは十分なサイズを持つため、オーバーフローは発生しない設計となっています。

保守担当者向けメモ

  • tm_hour が int64 である理由は、計算上の広範な値を許容するためです。変更を加える際は、このフィールドの型定義が日時計算の精度と範囲に与える影響を考慮してください。
