range_constructor3

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes.c:407

目的

3つの引数（下限値、上限値、フラグ文字列）を受け取り、それらに基づいて新しい RangeType を構築します。

シグネチャまたは形状

Datum range_constructor3(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_DATUM(0): 範囲の下限値。
  • PG_GETARG_DATUM(1): 範囲の上限値。
  • PG_GETARG_TEXT_PP(2): 境界の包含状態を指定するフラグ文字列（例: [], [), (], ()）。
  • **出力**:
  • Datum: 構築された RangeType へのポインタを PG_RETURN_RANGE_P マクロで返します。

前提条件と不変条件

  • 第3引数（フラグ）は NULL であってはなりません。
  • 内部的に range_get_typcache を呼び出し、対象範囲型の型キャッシュ情報を取得します。
  • 境界値が NULL の場合、その境界は無限大（infinite）として扱われます。

アルゴリズムまたは振る舞い

1. range_get_typcache を使用して、対象の範囲型に関するメタデータを取得します。 2. range_parse_flags を呼び出し、第3引数の文字列から境界の包含フラグを解析します。 3. RangeBound 構造体（lower, upper）を初期化し、値、無限大フラグ、包含フラグを設定します。 4. make_range を呼び出し、正規化（canonicalization）を含む範囲型の構築を行います。 5. 構築された RangeType を Datum として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/typecmds.c:1782（参照）
  • **呼び出し先**:
  • range_get_typcache
  • range_parse_flags
  • make_range

関連データ構造

  • RangeType: 範囲型の内部表現。
  • RangeBound: 範囲の境界を定義する構造体。
  • TypeCacheEntry: 範囲型の型情報キャッシュ。

失敗モードと境界条件

  • 第3引数が NULL の場合、ERRCODE_DATA_EXCEPTION を伴うエラーが発生します。
  • make_range 内部で正規化関数が失敗した場合、または無効な範囲（下限が上限を超える等）が指定された場合は、make_range の仕様に従いエラーが発生します。

保守担当者向けメモ

  • 本関数は SQL レベルの範囲型コンストラクタとして機能します。
  • 境界値の比較や正規化のロジックは make_range に依存しているため、範囲型の挙動を変更する場合は make_range の実装を確認してください。
