hash_xlog_update_meta_page

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash_xlog.c:963

目的

ハッシュインデックスのメタページにおけるタプル数（hashm_ntuples）の更新をWALリプレイ時に適用する。

シグネチャまたは形状

static void hash_xlog_update_meta_page(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（適用対象のWALレコード）。
  • **出力**: なし。バッファプール上のメタページを更新し、ダーティ状態にする。

前提条件と不変条件

  • record には xl_hash_update_meta_page 構造体のデータが含まれている必要がある。
  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返した場合のみ、ページ内容の更新が行われる。

アルゴリズムまたは振る舞い

1. XLogReadBufferForRedo を呼び出し、インデックスのメタページ（ブロック0）を取得する。 2. 取得したバッファが BLK_NEEDS_REDO を返す場合：

  • ページから HashMetaPage 構造体を取得する。
  • WALレコード内の ntuples 値をメタページの hashm_ntuples フィールドに書き込む。
  • PageSetLSN を使用して、ページのLSNをWALレコードの終了位置（record->EndRecPtr）に更新する。
  • MarkBufferDirty を呼び出し、バッファをダーティとしてマークする。

    • 3. バッファが有効な場合、UnlockReleaseBuffer を呼び出してロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hash_xlog.c:1107（hash_redo 関数内のディスパッチ処理から呼び出される）。
  • **呼び出し先**: XLogReadBufferForRedo, BufferGetPage, HashPageGetMeta, PageSetLSN, MarkBufferDirty, BufferIsValid, UnlockReleaseBuffer。

関連データ構造

  • HashMetaPage: ハッシュインデックスのメタデータ構造体。
  • xl_hash_update_meta_page: WALレコードに格納される更新データ構造。
  • XLogReaderState: WALレコードを読み取るための状態管理構造体。

失敗モードと境界条件

  • XLogReadBufferForRedo が BLK_NEEDS_REDO 以外を返した場合、ページ更新はスキップされる（冪等性の確保）。
  • バッファの取得に失敗した場合、あるいは無効なバッファが渡された場合は、ロック解放処理のみが行われる。

保守担当者向けメモ

  • 本関数はWALリプレイ専用であり、通常のインデックス操作時には使用されない。
  • メタページの hashm_ntuples のみが更新対象であるため、他のメタデータフィールドには影響を与えない。
