pa_detach_all_error_mq

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/applyparallelworker.c:621

目的

並列適用ワーカーのプール全体を走査し、各ワーカーに関連付けられたエラー用共有メモリメッセージキュー（error_mq_handle）をデタッチする。

シグネチャまたは形状

void pa_detach_all_error_mq(void)

入力と出力

  • 入力: なし（グローバル変数 ParallelApplyWorkerPool を参照）
  • 出力: なし（副作用として共有メモリのデタッチを実行）

前提条件と不変条件

  • ParallelApplyWorkerPool が初期化されていること。
  • 各 ParallelApplyWorkerInfo 構造体の error_mq_handle が有効なハンドルであるか、あるいは NULL であること。

アルゴリズムまたは振る舞い

1. グローバルリスト ParallelApplyWorkerPool を反復処理する。 2. 各ワーカー情報 (ParallelApplyWorkerInfo) を取得する。 3. error_mq_handle が設定されている場合、shm_mq_detach を呼び出してキューをデタッチする。 4. デタッチ後、当該ワーカーの error_mq_handle を NULL にリセットする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/launcher.c:768
  • 呼び出し先:
  • shm_mq_detach (共有メモリメッセージキューのデタッチ処理)

関連データ構造

  • ParallelApplyWorkerPool: 並列適用ワーカーを管理するリスト。
  • ParallelApplyWorkerInfo: ワーカーごとの状態を保持する構造体。error_mq_handle をメンバとして持つ。

失敗モードと境界条件

  • ParallelApplyWorkerPool が空の場合、何も行わずに正常終了する。
  • error_mq_handle が既に NULL であるワーカーについては、デタッチ処理をスキップする。

保守担当者向けメモ

  • 本関数は、並列適用ワーカーの終了処理やクリーンアップの一環として呼び出される。
  • shm_mq_detach の呼び出しにより、共有メモリ上のキューに対する参照が解放されるため、呼び出し後の error_mq_handle へのアクセスは無効となる。
