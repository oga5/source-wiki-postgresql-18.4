pgp_init

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp.c:188

目的

PGP 処理の状態を保持する PGP_Context 構造体をメモリ上に確保し、デフォルト値で初期化する。

シグネチャまたは形状

int pgp_init(PGP_Context **ctx_p)

入力と出力

  • **入力**: ctx_p (PGP_Context **) - 初期化されたコンテキストへのポインタを格納するためのアドレス。
  • **出力**: int - 成功時は 0 を返す。

前提条件と不変条件

  • palloc0 を使用してメモリを確保するため、PostgreSQL のメモリコンテキスト内で実行される必要がある。
  • 初期化後の PGP_Context は、def_ で始まる静的なデフォルト値（暗号アルゴリズム、S2K 設定、圧縮設定など）で構成される。

アルゴリズムまたは振る舞い

1. palloc0 を呼び出し、PGP_Context 構造体分のメモリをゼロクリアして確保する。 2. 確保した構造体の各フィールド（cipher_algo, s2k_mode, compress_algo, text_mode 等）に、モジュール内の静的なデフォルト値を代入する。 3. 初期化済みのポインタを *ctx_p に格納する。 4. 0 を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pgsql.c:354 で呼び出されていることが確認できる。
  • **呼び出し先**: PostgreSQL のメモリ管理関数 palloc0 を呼び出す。

関連データ構造

  • PGP_Context: PGP 処理の全状態を保持する主要な構造体。

失敗モードと境界条件

  • palloc0 がメモリ不足によりエラーを発生させた場合、PostgreSQL のエラーハンドリング機構により処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は PGP_Context のライフサイクルの起点である。コンテキストを解放する際は、対となる pgp_free() を使用すること。
  • デフォルト値の変更が必要な場合は、pgp.c 内の def_ 変数群を確認すること。
