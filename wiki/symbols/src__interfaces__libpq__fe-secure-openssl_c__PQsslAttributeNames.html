PQsslAttributeNames

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-openssl.c:1643

目的

OpenSSL を使用した接続において、取得可能な SSL 属性名のリストを返します。

シグネチャまたは形状

const char *const *PQsslAttributeNames(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn（PostgreSQL 接続ハンドル。NULL の場合、デフォルトの SSL ライブラリ属性を返します）。
  • **出力**: const char *const *（属性名の文字列配列。終端は NULL）。

前提条件と不変条件

  • conn が NULL でない場合、conn->ssl が NULL（非暗号化接続）であれば、空の属性リストが返されます。
  • 返される配列は静的に定義されており、呼び出し側で解放してはなりません。

アルゴリズムまたは振る舞い

1. conn が NULL の場合、または conn->ssl が有効な場合、{"library", "key_bits", "cipher", "compression", "protocol", "alpn", NULL} を返します。 2. conn が存在し、かつ conn->ssl が NULL の場合、{NULL} を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure.c からの参照が確認されています。
  • **呼び出し先**: 本関数は内部で他の関数を呼び出していません。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。ssl メンバを通じて OpenSSL の接続状態を参照します。

失敗モードと境界条件

  • conn が NULL の場合でも安全に動作し、デフォルトの属性リストを返します。
  • 非暗号化接続に対しては、属性名を含まない空のリストを返すことで、属性取得が不可能であることを示します。

保守担当者向けメモ

  • 新しい SSL 属性をサポートする場合は、openssl_attrs 配列に文字列を追加してください。
  • 本関数は libpq-fe.h で公開されており、API の一部として安定しています。
