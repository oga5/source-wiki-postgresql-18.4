ParseTzFile

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/tzparser.c:276

目的

タイムゾーン定義ファイルを読み込み、行単位で解析して tzEntry 配列を構築する。再帰的なファイル読み込み（@INCLUDE）をサポートする。

シグネチャまたは形状

static int ParseTzFile(const char *filename, int depth, tzEntry **base, int *arraysize, int n);

入力と出力

  • **入力**:
  • filename: 解析対象のファイル名（英字のみ許可）。
  • depth: 現在の再帰深度。
  • base: tzEntry 配列へのポインタのポインタ。
  • arraysize: 配列の現在の容量。
  • n: 現在格納されているエントリ数。
  • **出力**:
  • 成功時: 更新されたエントリ数 n を返す。
  • 失敗時: -1 を返し、GUC_check_errmsg を通じてエラー詳細を通知する。

前提条件と不変条件

  • ファイル名は英字のみで構成される必要がある。
  • 再帰深度は 3 レベルまでに制限されている。
  • ファイルは share/timezonesets/ ディレクトリ内に存在する必要がある。
  • エラー発生時は elog(ERROR) を避け、GUC_check_errmsg を使用してエラー情報を蓄積する。

アルゴリズムまたは振る舞い

1. **検証**: ファイル名が英字か確認し、再帰深度が 3 を超えていないかチェックする。 2. **ファイルオープン**: share/timezonesets/ 配下のファイルを読み込みモードで開く。 3. **行解析**: ファイルを 1 行ずつ読み込み、以下の処理を行う。

  • 空行やコメント（#）をスキップ。
  • @INCLUDE: 指定されたファイルを再帰的に ParseTzFile で読み込む。
  • @OVERRIDE: 以降のエントリで重複を許可するフラグを立てる。
  • 通常行: splitTzLine でトークン化し、validateTzEntry で検証後、addToArray で配列に追加する。

    • 4. **終了**: ファイルをクローズし、最終的なエントリ数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/tzparser.c 内の ParseTzFile（再帰呼び出し）
  • src/backend/utils/misc/tzparser.c 内の load_tzoffsets（エントリポイント）
  • **呼び出し先**:
  • splitTzLine
  • validateTzEntry
  • addToArray
  • AllocateFile / FreeFile
  • AllocateDir / FreeDir
  • GUC_check_errmsg

関連データ構造

  • tzEntry: タイムゾーンの略称、オフセット、DST フラグを保持する構造体。

失敗モードと境界条件

  • **ファイル名不正**: 英字以外が含まれる場合、エラーを返す。
  • **再帰深度超過**: 3 レベルを超える読み込みはエラーとなる。
  • **ファイル読み込み失敗**: ファイルが存在しない、またはディレクトリが開けない場合、エラーを返す。
  • **行長超過**: 1024 バイトを超える行はエラーとなる。
  • **解析エラー**: splitTzLine や validateTzEntry が失敗した場合、処理を中断してエラーを返す。

保守担当者向けメモ

  • 本関数は GUC フックから呼ばれるため、エラー報告には GUC_check_errmsg を使用すること。
  • addToArray はソート順を維持するため、datetime.c 側の検索ロジックと密結合している点に注意。
  • セキュリティ上の理由から、ディレクトリトラバーサルを防ぐためファイル名は英字のみに制限されている。
