on_exit_nicely_callback

定義

  • 種別: typedef (関数ポインタ型)
  • 定義箇所: src/bin/pg_dump/pg_backup_utils.c:26

目的

プログラム終了時に実行されるクリーンアップ関数を定義するための型です。

シグネチャまたは形状

typedef void (*on_exit_nicely_callback) (int code, void *arg);

※ソースコード上の構造体定義における関数ポインタの型として使用されています。

入力と出力

  • 入力:
  • int code: 終了コード。
  • void *arg: コールバック登録時に渡されたユーザー定義データ。
  • 出力: なし（void）。

前提条件と不変条件

  • on_exit_nicely_list 配列の要素として保持されます。
  • 最大保持数は MAX_ON_EXIT_NICELY 個に制限されています。

アルゴリズムまたは振る舞い

  • exit_nicely が呼び出された際、登録された順序とは逆順（インデックスの大きい順）に実行されます。
  • 終了処理のインフラとして、各コンポーネントがリソース解放等のクリーンアップを行うために利用します。

呼び出し元と呼び出し先

  • 呼び出し元: exit_nicely 関数（src/bin/pg_dump/pg_backup_utils.c 内）。
  • 呼び出し先: 登録された各クリーンアップ関数（具体的な実装は各モジュールに依存）。

関連データ構造

  • on_exit_nicely_list: コールバック関数と引数を保持する構造体の配列。
  • exit_nicely: 本型を利用して終了処理を管理する関数。

失敗モードと境界条件

  • 登録数が MAX_ON_EXIT_NICELY を超えた場合の挙動については、本ソース抜粋からは判断できません。

保守担当者向けメモ

  • コールバックは登録の逆順で実行されるため、リソース解放の依存関係に注意してください。
  • Windows環境ではスレッドごとに終了処理が走る可能性があるため、コールバック内でのスレッド安全性を考慮してください。
