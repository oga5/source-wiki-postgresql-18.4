walrcv_check_conninfo_fn

定義

  • 種別: typedef
  • 定義箇所: src/include/replication/walreceiver.h:254

目的

レプリケーション接続に使用する接続文字列（conninfo）の妥当性を検証するための関数ポインタ型です。

シグネチャまたは形状

typedef void (*walrcv_check_conninfo_fn) (const char *conninfo, bool must_use_password);

入力と出力

  • conninfo (入力): 検証対象の接続文字列。
  • must_use_password (入力): パスワード認証が必須かどうかを示すフラグ。
  • 戻り値: なし（void）。検証失敗時はエラーをスローすることが想定されます。

前提条件と不変条件

  • 本シンボルは WalReceiverFunctionsType 構造体の一部として使用される関数ポインタ型です。
  • 実際の検証ロジックは、通信バックエンド（libpq 等）の実装に依存します。

アルゴリズムまたは振る舞い

  • 指定された接続文字列が、レプリケーション接続として有効な形式であるかを確認します。
  • must_use_password が true の場合、接続文字列内にパスワード認証に必要な情報が含まれているか、あるいは認証設定が適切かを検証します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/replication/walreceiver.h 内で walrcv_check_conninfo として宣言されており、レプリケーション設定の検証プロセスから呼び出されることが示唆されています。
  • 呼び出し先: 具体的な実装は src/backend/replication/libpqwalreceiver/ 等の通信モジュール内に存在します。

関連データ構造

  • WalReceiverFunctionsType: この関数ポインタを保持するインターフェース構造体。

失敗モードと境界条件

  • 接続文字列の構文エラーや、必須パラメータの欠如が発生した場合、エラーを発生させます。
  • 呼び出し元および呼び出し先の詳細なエラーハンドリング仕様については、各実装（libpq 等）のドキュメントに依存します。

保守担当者向けメモ

  • 本シンボルはインターフェース定義です。実際の動作を確認する場合は、src/backend/replication/libpqwalreceiver/ 以下の実装を参照してください。
