XLogPrefetchStats

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xlogprefetcher.c:171

目的

リカバリ処理中の先読み（Prefetching）に関する統計情報を保持し、pg_stat_recovery_prefetch ビュー等で利用可能なメトリクスを共有メモリ上で管理するために使用されます。

シグネチャまたは形状

typedef struct XLogPrefetchStats
{
    pg_atomic_uint64 reset_time;    /* 最終リセット時刻 */
    pg_atomic_uint64 prefetch;      /* 開始された先読み回数 */
    pg_atomic_uint64 hit;           /* キャッシュヒット回数 */
    pg_atomic_uint64 skip_init;     /* ゼロ初期化ブロックのスキップ回数 */
    pg_atomic_uint64 skip_new;      /* 新規/欠落ブロックのフィルタリング回数 */
    pg_atomic_uint64 skip_fpw;      /* フルページ書き込み（FPW）のスキップ回数 */
    pg_atomic_uint64 skip_rep;      /* 重複アクセスのスキップ回数 */

    /* 動的値 */
    int            wal_distance;    /* 先読み中のWALバイト数 */
    int            block_distance;  /* 先読み中のブロック参照数 */
    int            io_depth;        /* 進行中のI/O数 */
} XLogPrefetchStats;

入力と出力

  • **入力**: 各フィールドは pg_atomic_uint64 を使用しており、アトミック操作による更新が前提です。
  • **出力**: 共有メモリ上の統計情報として保持され、監視ツールや統計ビューから参照されます。

前提条件と不変条件

  • 共有メモリ上に配置されるため、XLogPrefetchShmemInit を介して初期化される必要があります。
  • 統計カウンタは pg_atomic 操作により、並行プロセス間での競合を防ぐ必要があります。

アルゴリズムまたは振る舞い

  • 先読み処理の各段階（開始、ヒット、各種スキップ条件）で対応するカウンタがインクリメントされます。
  • wal_distance、block_distance、io_depth は現在の先読み状態を反映する動的な値として更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: XLogPrefetchShmemInit で共有メモリ領域の確保と初期化が行われます。
  • **呼び出し先**: 情報は不明ですが、統計情報を更新する先読みロジック（XLogPrefetcher 関連関数）から参照・更新されます。

関連データ構造

  • XLogPrefetcher: 先読み状態を管理するメイン構造体。
  • pg_atomic_uint64: 統計情報のスレッドセーフな更新に使用される型。

失敗モードと境界条件

  • 共有メモリの初期化に失敗した場合、統計情報の追跡が不可能となります。
  • アトミック操作が適切に行われない場合、統計値に不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/access/transam/xlogprefetcher.c 内の SharedStats ポインタを通じてアクセスされます。
  • 統計項目を追加する場合は、pg_atomic_uint64 を使用して並行性を維持してください。
