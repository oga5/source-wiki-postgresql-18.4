plpython_to_jsonb

定義

  • 種別: function
  • 定義箇所: contrib/jsonb_plpython/jsonb_plpython.c:458

目的

Python オブジェクトを PostgreSQL の jsonb 型へ変換する SQL 呼び出し可能な関数。

シグネチャまたは形状

Datum plpython_to_jsonb(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される PyObject へのポインタ（PG_GETARG_POINTER(0)）。
  • **出力**: 変換された jsonb データ構造を指す Datum 型。

前提条件と不変条件

  • plpython 拡張がロードされ、初期化されていること。
  • 入力された PyObject が PLyObject_ToJsonbValue で処理可能な型（辞書、リスト、数値、文字列、真偽値、None 等）であること。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から Python オブジェクトを取得する。 2. JsonbParseState を初期化し、PLyObject_ToJsonbValue を呼び出して Python オブジェクトを再帰的に JsonbValue 構造体へ変換する。 3. 変換された JsonbValue を JsonbValueToJsonb を介して jsonb 型に変換し、その結果を PG_RETURN_POINTER で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: PLyObject_ToJsonbValue（変換処理）、JsonbValueToJsonb（最終的な jsonb 構築）。

関連データ構造

  • PyObject: Python 側のオブジェクト表現。
  • JsonbValue: JSONB の各要素を保持する中間構造体。
  • JsonbParseState: JSONB 構築時の状態管理用構造体。

失敗モードと境界条件

  • Python オブジェクトが JSONB に変換不可能な型である場合、PLyObject_ToJsonbValue 内でエラーが発生する可能性がある。
  • メモリ不足や変換中の型不整合が発生した場合、PostgreSQL のエラーハンドリング機構により処理が中断される。

保守担当者向けメモ

  • 本関数は plpython 拡張の動的ロードに依存している。
  • PLyObject_ToJsonbValue は再帰的に動作するため、深いネストを持つ Python オブジェクトの変換時にはスタック消費に注意が必要。
  • Python のメモリ管理（Py_DECREF 等）と PostgreSQL のエラーハンドリングの整合性に留意すること。
