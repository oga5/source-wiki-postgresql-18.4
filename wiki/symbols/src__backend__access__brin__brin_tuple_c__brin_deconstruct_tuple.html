brin_deconstruct_tuple

定義

  • 種別: static inline function
  • 定義箇所: src/backend/access/brin/brin_tuple.c:644

目的

ディスク上の物理的なBRINタプル（BrinTuple）から、メモリ上の表現（BrinMemTuple）へデータを展開する。具体的には、NULLビットマップの解析と、可変長データを含む属性値の抽出を行う。

シグネチャまたは形状

static inline void brin_deconstruct_tuple(BrinDesc *brdesc, char *tp, bits8 *nullbits, bool nulls, Datum *values, bool *allnulls, bool *hasnulls)

入力と出力

  • **入力**:
  • brdesc: BRINインデックスの記述子。
  • tp: ディスク上のタプルデータへのポインタ。
  • nullbits: NULL状態を示すビットマップ。
  • nulls: NULLビットマップが存在するかを示すフラグ。
  • values: 抽出した値を格納する Datum 配列（出力先）。
  • allnulls: 各属性が「全NULL」であるかを示すフラグ配列（出力先）。
  • hasnulls: 各属性が「一部NULL」であるかを示すフラグ配列（出力先）。
  • **出力**: なし（引数経由で values, allnulls, hasnulls を更新）。

前提条件と不変条件

  • nullbits 内のNULL表現は、一般的なPostgreSQLの慣習（1=非NULL）とは逆の「1=NULL」というBRIN独自の規約に従う。
  • allnulls が真である属性については、タプルデータ領域に値が存在しないため、抽出処理はスキップされる。

アルゴリズムまたは振る舞い

1. **NULLビットマップの解析**: brdesc の属性数分ループし、nullbits を参照して allnulls および hasnulls を設定する。 2. **データ抽出**:

  • brtuple_disk_tupdesc を用いてディスク上の属性定義を取得する。
  • 各属性について、allnulls が偽であれば、属性の型情報（attlen, attalignby）に基づき、tp から適切なオフセットを計算して values 配列に値をコピーする。
  • 可変長データ（attlen == -1）の場合は att_pointer_alignby を、それ以外は att_nominal_alignby を使用してアライメントを調整する。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_deform_tuple（src/backend/access/brin/brin_tuple.c:587）。
  • **呼び出し先**: brtuple_disk_tupdesc, att_isnull, att_pointer_alignby, att_nominal_alignby, fetchatt, att_addlength_pointer。

関連データ構造

  • BrinDesc: インデックスのメタデータと属性情報を保持。
  • BrinTuple: オンディスク上のタプル形式。
  • CompactAttribute: ディスク上の属性の物理レイアウトを定義。

失敗モードと境界条件

  • attlen が -1 の場合、TOASTデータや可変長データのポインタ操作が発生するため、メモリ配置が不正な場合にクラッシュする可能性がある。
  • brdesc の属性数と nullbits の整合性が取れていない場合、メモリ破壊を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は brin_deform_tuple の内部ロジックとして機能する。
  • NULLビットマップの解釈が標準的なPostgreSQLの att_isnull と逆転している点に注意すること。
  • 属性ごとに oi_nstored を参照してループするため、単一のインデックス属性が複数の物理カラムに対応している場合でも正しく動作する。
