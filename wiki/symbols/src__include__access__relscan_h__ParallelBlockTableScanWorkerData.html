ParallelBlockTableScanWorkerData

定義

  • 種別: struct
  • 定義箇所: src/include/access/relscan.h:107

目的

ブロック指向のテーブルスキャンにおいて、並列実行時の各ワーカーが処理すべきデータブロックの割り当て状態を管理するために使用されます。

シグネチャまたは形状

typedef struct ParallelBlockTableScanWorkerData
{
    uint64      phsw_nallocated;       /* 現在のスキャンにおける割り当て済みブロック数 */
    uint32      phsw_chunk_remaining;  /* 現在のチャンク内で残っているブロック数 */
    uint32      phsw_chunk_size;       /* スキャン用に一度に割り当てるブロック数 */
} ParallelBlockTableScanWorkerData;

入力と出力

  • **入力**: スキャン開始時に phsw_chunk_size が設定されます。
  • **出力**: phsw_nallocated および phsw_chunk_remaining がスキャンの進捗に応じて更新されます。

前提条件と不変条件

  • 本構造体は、並列スキャン実行時に各ワーカーが個別に保持する状態として使用されます。
  • phsw_chunk_size はスキャン実行中に変更されないことが想定されます。

アルゴリズムまたは振る舞い

  • テーブルスキャン実行時、ワーカーは phsw_chunk_size に基づいてブロックをチャンク単位で取得します。
  • phsw_chunk_remaining が枯渇した場合、共有メモリ上の並列スキャン状態から新たなチャンクを確保し、本構造体の値を更新してスキャンを継続します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c:1181 にて、palloc を用いてメモリが割り当てられます。
  • **呼び出し先**: 具体的なスキャンロジック内での参照先は、提供されたソース範囲内では明示されていません。

関連データ構造

  • ParallelBlockTableScanWorker: 本構造体へのポインタ型 (ParallelBlockTableScanWorkerData *)。
  • TableScanDescData: テーブルスキャンの基底構造体。rs_parallelworkerdata メンバを通じて本構造体を保持します。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc 呼び出し時）の挙動については、PostgreSQL の標準的なメモリ管理エラーに依存します。
  • チャンクサイズが 0 に設定された場合の挙動については、本ソース範囲内では定義されていません。

保守担当者向けメモ

  • 本構造体は heapam.c 等のテーブルアクセスメソッドの実装で利用されます。
  • 共有メモリ上の並列スキャン状態（ParallelBlockTableScanDescData）と混同しないよう注意してください。本構造体はワーカー個別の進捗管理用です。
