ToastCompressionId

定義

  • 種別: enum
  • 定義箇所: src/include/access/toast_compression.h:37

目的

TOAST（The Oversized-Attribute Storage Technique）データが格納される際の圧縮アルゴリズムを識別するために使用されます。

シグネチャまたは形状

typedef enum ToastCompressionId
{
	TOAST_PGLZ_COMPRESSION_ID = 0,
	TOAST_LZ4_COMPRESSION_ID = 1,
	TOAST_INVALID_COMPRESSION_ID = 2,
} ToastCompressionId;

入力と出力

  • 入力: varlena 構造体のヘッダー情報から抽出されたビット列。
  • 出力: 圧縮アルゴリズムを特定する列挙値（TOAST_PGLZ_COMPRESSION_ID、TOAST_LZ4_COMPRESSION_ID、または TOAST_INVALID_COMPRESSION_ID）。

前提条件と不変条件

  • varlena ヘッダー内の2ビットのみを使用して識別子を保持するため、最大4つまでのアルゴリズムに制限されます。

アルゴリズムまたは振る舞い

1. 圧縮された varlena データのヘッダーから識別子を読み取ります。 2. 読み取った ID に基づき、適切な伸長関数（pglz_decompress_datum 等）へ処理をディスパッチします。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/amcheck/verify_heapam.c、src/backend/access/common/detoast.c、src/backend/access/common/toast_compression.c、src/backend/access/common/toast_internals.c、src/backend/utils/adt/varlena.c 等で参照されています。
  • 呼び出し先: toast_get_compression_id 関数が本型を返します。

関連データ構造

  • varlena: TOAST データの格納構造体。
  • pg_attribute.attcompression: 圧縮方式を保持する文字定数。

失敗モードと境界条件

  • TOAST_INVALID_COMPRESSION_ID が返される場合、無効な圧縮方式であることを示します。
  • LZ4 圧縮を使用する場合、ビルド時に USE_LZ4 が定義されている必要があります。未定義環境での呼び出しはエラーとなります。

保守担当者向けメモ

  • 新しい圧縮アルゴリズムを追加する場合は、本列挙型へのメンバ追加と、対応する圧縮・伸長関数の実装が必要です。
