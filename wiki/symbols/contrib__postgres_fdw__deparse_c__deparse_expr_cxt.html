deparse_expr_cxt

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/deparse.c:97

目的

postgres_fdw において、ローカルの式ツリーをリモートサーバーで実行可能な SQL 文字列へ変換（デパース）する際の処理状態を保持するために使用されます。

シグネチャまたは形状

typedef struct deparse_expr_cxt
{
    PlannerInfo *root;          /* グローバルなプランナ状態 */
    RelOptInfo *foreignrel;     /* プランニング対象の外部リレーション */
    RelOptInfo *scanrel;        /* 基底となるスキャンリレーション */
    StringInfo  buf;            /* SQL 文字列を構築するための出力バッファ */
    List      **params_list;    /* リモートパラメータ（$1, $2...）として送信する式のリスト */
} deparse_expr_cxt;

入力と出力

  • 本構造体は、SQL 構築関数（deparseExpr, deparseVar, deparseFuncExpr 等）の引数として渡され、状態の共有と更新に使用されます。
  • buf には構築された SQL フラグメントが蓄積され、params_list にはリモート実行時にバインドされる式が追加されます。

前提条件と不変条件

  • foreignrel と scanrel は、結合や基底リレーションの文脈に応じて適切に設定されている必要があります。
  • buf は初期化済みの StringInfo である必要があります。

アルゴリズムまたは振る舞い

  • deparseSelectStmtForRel 等の SQL 構築処理の開始時に初期化されます。
  • 式ツリーの走査中、各ノードのデパース関数がこのコンテキストを参照し、SQL 文字列を buf に追記します。
  • Var や Param ノードに遭遇した際、必要に応じて params_list に式を追加し、リモート用のプレースホルダーを buf に書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: deparseSelectStmtForRel や、SQL 構築を行う各種 deparse 系関数から初期化・参照されます。
  • **呼び出し先**: deparseExpr, deparseVar, deparseConst, deparseParam, deparseSubscriptingRef, deparseFuncExpr, deparseOpExpr, deparseDistinctExpr, deparseSubqueryTargetList, isPlainForeignVar 等の関数がこのコンテキストを受け取ります。

関連データ構造

  • PlannerInfo: プランニング全体のグローバル状態。
  • RelOptInfo: 外部リレーションのメタデータ。
  • StringInfo: PostgreSQL の動的文字列バッファ。

失敗モードと境界条件

  • buf が未初期化の場合、SQL 構築時にセグメンテーション違反が発生する可能性があります。
  • params_list が NULL の場合、パラメータを含む式のデパース時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は postgres_fdw のデパース処理における中心的な状態保持オブジェクトです。新しいノード型のデパース処理を追加する際は、必ずこのコンテキストを介して状態を管理してください。
  • scanrel と foreignrel の関係性については、結合クエリのデパース時に特に注意が必要です。
