ltree_gist_alloc

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_gist.c:41

目的

GiST インデックスで使用される ltree_gist 構造体をメモリ上に確保し、指定されたシグネチャおよびノードデータで初期化します。

シグネチャまたは形状

ltree_gist *ltree_gist_alloc(bool isalltrue, BITVECP sign, int siglen, ltree *left, ltree *right)

入力と出力

  • **入力**:
  • isalltrue: すべてのビットがセットされているかを示すフラグ。
  • sign: シグネチャ（ビットマップ）データへのポインタ。
  • siglen: シグネチャのバイト長。
  • left, right: インデックスノードの範囲を示す ltree へのポインタ。
  • **出力**:
  • 初期化された ltree_gist 構造体へのポインタ。

前提条件と不変条件

  • siglen が 0 の場合、left は必ず存在する必要があります（Assert(left)）。
  • siglen が 0 の場合、構造体は LTG_ONENODE フラグで初期化されます。
  • left と right が同一、または right が指定されていない場合、LTG_NORIGHT フラグが設定されます。

アルゴリズムまたは振る舞い

1. LTG_HDRSIZE、シグネチャ長、およびノードサイズに基づいて必要なメモリ量を計算し、palloc で確保します。 2. SET_VARSIZE で確保したメモリのサイズを記録します。 3. siglen > 0 の場合:

  • isalltrue が真なら LTG_ALLTRUE フラグを立てます。
  • sign が提供されていれば LTG_SIGN にコピーし、なければ 0 で埋めます。
  • left を LTG_LNODE にコピーします。
  • right が存在し、かつ left と異なる場合は LTG_RNODE にコピーし、それ以外は LTG_NORIGHT フラグを立てます。

    • 4. siglen == 0 の場合:

  • LTG_ONENODE フラグを立て、left を LTG_NODE にコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/_ltree_gist.c および contrib/ltree/ltree_gist.c 内の複数の箇所から呼び出されています。
  • **呼び出し先**: palloc、memcpy、memset、VARSIZE、SET_VARSIZE、ISEQ、Assert などの PostgreSQL マクロおよび標準ライブラリ関数。

関連データ構造

  • ltree_gist: インデックスノードを保持する構造体。
  • ltree: インデックス対象のデータ型。
  • BITVECP: シグネチャを保持するためのビットベクトル型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • siglen が 0 の際に left が NULL の場合、Assert によりデバッグビルドで停止します。

保守担当者向けメモ

  • 本関数は ltree_gist のメモリレイアウトを直接操作するため、構造体のヘッダー定義（LTG_HDRSIZE 等）に変更を加える際は、本関数のオフセット計算も併せて確認してください。
