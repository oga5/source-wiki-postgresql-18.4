spgAddPendingTID

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgvacuum.c:63

目的

SP-GiST インデックスの VACUUM 処理中に、後続のクリーンアップが必要な REDIRECT 先の TID（タプル識別子）を保留リストに追加します。

シグネチャまたは形状

static void spgAddPendingTID(spgBulkDeleteState *bds, ItemPointer tid)

入力と出力

  • **入力**:
  • bds: spgBulkDeleteState 構造体へのポインタ。保留リストのヘッドを保持します。
  • tid: 追加対象の ItemPointer。
  • **出力**: なし（副作用として bds->pendingList が更新されます）。

前提条件と不変条件

  • bds は有効な spgBulkDeleteState を指している必要があります。
  • リスト内の各 spgVacPendingItem は、tid が一意であることを保証します。

アルゴリズムまたは振る舞い

1. bds->pendingList を先頭から走査し、指定された tid が既にリスト内に存在するか確認します。 2. 既に存在する場合は、何もせず終了します。 3. リストの末尾まで到達した場合、新しい spgVacPendingItem をメモリ確保（palloc）します。 4. 新しいアイテムに tid を設定し、done フラグを false に初期化してリストの末尾に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgvacuum.c 内の複数箇所（vacuumLeafPage や vacuumRedirectAndPlaceholder 等の文脈で呼び出されます）。
  • **呼び出し先**:
  • ItemPointerEquals（比較用）
  • palloc（メモリ確保用）

関連データ構造

  • spgBulkDeleteState: VACUUM 実行中の状態を保持し、pendingList の起点となります。
  • spgVacPendingItem: 保留中の TID を保持するリンクリストのノード構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • リストが空の場合、正しく先頭に追加されます。

保守担当者向けメモ

  • 本関数はリストの線形探索を行うため、保留リストが極端に長くなる場合はパフォーマンスに影響を与える可能性があります。
  • spgprocesspending 関数がこのリストを消費し、done フラグを更新またはリストを解放する役割を担います。
