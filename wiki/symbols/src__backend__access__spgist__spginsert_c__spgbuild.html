spgbuild

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spginsert.c:72

目的

SP-GiSTインデックスの新規構築を行います。ヒープスキャンを通じて全タプルを読み込み、インデックス構造を初期化・構築します。

シグネチャまたは形状

IndexBuildResult *spgbuild(Relation heap, Relation index, IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • heap: 対象のヒープリレーション。
  • index: 新規作成するインデックスリレーション。
  • indexInfo: インデックスの定義情報。
  • **出力**:
  • IndexBuildResult*: 処理されたヒープタプル数およびインデックスタプル数を含む構造体。

前提条件と不変条件

  • 対象のインデックスが空であること（空でない場合はエラー）。
  • メタページ、ルートページ、NULL用ページの初期化が成功すること。

アルゴリズムまたは振る舞い

1. **初期化**: インデックスのメタページ、ルートページ、NULL用ページを新規作成し、初期化します。 2. **構築状態の準備**: SpGistBuildState を初期化し、一時メモリコンテキストを作成します。 3. **データ挿入**: table_index_build_scan を呼び出し、ヒープ内の全タプルに対して spgistBuildCallback を実行してインデックスへ挿入します。 4. **後処理**: 一時メモリコンテキストを解放し、メタページを更新します。 5. **WAL書き込み**: WALログが必要な場合、構築した全ページを log_newpage_range で一括書き込みします。 6. **結果返却**: 処理したタプル数を格納した IndexBuildResult を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgutils.c にて ambuild インターフェースとして登録されています。
  • **呼び出し先**:
  • SpGistNewBuffer, SpGistInitMetapage, SpGistInitBuffer (インデックス構造初期化)
  • table_index_build_scan (ヒープスキャン)
  • spgistBuildCallback (各タプルの挿入)
  • SpGistUpdateMetaPage (メタデータ更新)
  • log_newpage_range (WAL書き込み)

関連データ構造

  • SpGistBuildState: 構築中の状態（タプル数、一時メモリコンテキスト等）を保持。
  • IndexBuildResult: 構築結果の統計情報。

失敗モードと境界条件

  • インデックスが既にデータを含んでいる場合、elog(ERROR) を発生させます。
  • メモリ不足時やディスク書き込みエラー時は、PostgreSQLの標準的なエラーハンドリングに従います。

保守担当者向けメモ

  • 本関数は構築中の WAL 書き込みを省略し、最後に一括書き込みを行う最適化を行っています。
  • 挿入処理に伴うメモリ消費を抑えるため、buildstate.tmpCtx を使用しています。デバッグ時はこのコンテキストのライフサイクルに注意してください。
