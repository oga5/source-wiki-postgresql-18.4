pg_prng_int64_range

定義

  • 種別: function
  • 定義箇所: src/common/pg_prng.c:191

目的

指定された範囲 [rmin, rmax] 内の int64 型の擬似乱数を生成する。

シグネチャまたは形状

int64 pg_prng_int64_range(pg_prng_state *state, int64 rmin, int64 rmax)

入力と出力

  • **入力**:
  • state: 乱数生成器の状態を保持する pg_prng_state 構造体へのポインタ。
  • rmin: 範囲の最小値（含む）。
  • rmax: 範囲の最大値（含む）。
  • **出力**:
  • rmin 以上 rmax 以下の int64 値。rmax <= rmin の場合は rmin を返す。

前提条件と不変条件

  • rmax > rmin の場合、内部で pg_prng_uint64_range を呼び出し、結果を int64 に変換する。
  • 負の数を含む範囲指定を正しく処理するため、内部で uint64 へのキャストとオフセット計算を行う。

アルゴリズムまたは振る舞い

1. rmax > rmin かどうかを判定する。 2. 真の場合、pg_prng_uint64_range(state, 0, (uint64) rmax - (uint64) rmin) を呼び出し、ベース値に加算して uval を得る。 3. uval が PG_INT64_MAX を超える場合、PG_INT64_MIN を基準としたオフセット計算を行い、int64 に変換する。それ以外は直接キャストする。 4. 偽の場合（rmax <= rmin）、rmin をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pseudorandomfuncs.c:139
  • src/backend/utils/adt/pseudorandomfuncs.c:163
  • **呼び出し先**:
  • pg_prng_uint64_range (内部で呼び出し)

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • rmax <= rmin: 乱数生成を行わず、rmin を返す。
  • uval が PG_INT64_MAX を超える場合: int64 への変換時にオーバーフローを防ぐための補正ロジックが適用される。

保守担当者向けメモ

  • 本関数は暗号学的に安全ではない。パスワードや暗号鍵の生成には使用しないこと。
  • pg_prng_uint64_range の結果を int64 に戻す際、実装定義の動作を避けるための安全な変換処理が含まれている。
