shmem_startup_hook_type

定義

  • 種別: typedef
  • 定義箇所: src/include/storage/ipc.h:22

目的

共有メモリの初期化プロセスにおいて、特定の拡張モジュールやサブシステムが独自の初期化処理を挿入するためのフック関数の型を定義します。

シグネチャまたは形状

typedef void (*shmem_startup_hook_type) (void);

入力と出力

  • 入力: なし
  • 出力: なし（void）

前提条件と不変条件

  • このフックが実行されるタイミングは、共有メモリの初期化シーケンス内です。
  • 呼び出し元がこのフックを呼び出す際、共有メモリが利用可能な状態にあることが期待されます。

アルゴリズムまたは振る舞い

  • 共有メモリ初期化時に、登録された関数ポインタが実行されます。
  • 複数のモジュールがこのフックを利用する場合、既存のフックを退避（保存）し、新しいフックを登録して、実行時に退避したフックを呼び出す「チェイン」形式の実装が一般的です。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c 内の共有メモリ初期化処理（直接の呼び出し箇所はソース抜粋外ですが、shmem_startup_hook 変数の存在から推測されます）。
  • 呼び出し先: 拡張モジュールやテストモジュールが登録した関数。
  • 参照元（直接の証拠）:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/test/modules/injection_points/injection_points.c
  • src/test/modules/test_aio/test_aio.c
  • src/test/modules/test_slru/test_slru.c

関連データ構造

  • shmem_startup_hook: src/include/storage/ipc.h で宣言されている、この型のグローバル変数。

失敗モードと境界条件

  • 呼び出し先関数内でエラーが発生した場合の挙動は、PostgreSQLの標準的なエラーハンドリング（elog 等）に依存します。
  • 複数のモジュールがフックを登録する際、適切にチェインを維持しないと、以前に登録されたフックが上書きされ実行されなくなるリスクがあります。

保守担当者向けメモ

  • 本シンボルは関数ポインタ型であるため、代入時には型安全性を確認してください。
  • 拡張モジュールがこのフックを使用する際は、必ず prev_shmem_startup_hook のような変数に既存のフックを保存し、自身の処理後にそれを呼び出す実装パターンを推奨します。
