filetime_to_time

定義

  • 種別: function
  • 定義箇所: src/port/win32stat.c:24

目的

Windows の FILETIME 構造体（100ナノ秒単位の64ビット値）を、POSIX 準拠の time_t（秒単位の64ビット値）へ変換します。

シグネチャまたは形状

static __time64_t filetime_to_time(const FILETIME *ft)

入力と出力

  • **入力**: const FILETIME *ft - 変換対象の Windows ファイル時刻。
  • **出力**: __time64_t - 1970年1月1日からの経過秒数。変換不能な場合は -1 を返します。

前提条件と不変条件

  • 入力された FILETIME が Windows のエポック（1601年1月1日）以前を表す値である場合、変換は行われず -1 が返されます。
  • EpochShift 定数（116444736000000000）は、Windows のエポックから Unix エポックまでの差分（100ナノ秒単位）として定義されています。

アルゴリズムまたは振る舞い

1. FILETIME の dwLowDateTime と dwHighDateTime を ULARGE_INTEGER に結合します。 2. 値が EpochShift 未満であれば、Unix エポック以前とみなして -1 を返します。 3. EpochShift を減算して Unix エポック基準に補正します。 4. 100ナノ秒単位の値を秒単位に変換するため、10,000,000 で除算します。 5. 結果を __time64_t として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/win32stat.c 内の以下の箇所から呼び出されています。
  • st_mtime（最終更新時刻）の計算時
  • st_atime（最終アクセス時刻）の計算時
  • st_ctime（作成時刻）の計算時
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • FILETIME: Windows API で定義されるファイル時刻構造体。
  • ULARGE_INTEGER: 64ビット値を扱うための共用体。
  • __time64_t: 64ビットの時刻型。

失敗モードと境界条件

  • 入力値が EpochShift より小さい場合、関数は -1 を返します。これは Unix エポック以前の時刻を表現できないことを示します。

保守担当者向けメモ

  • 本関数は src/port/win32stat.c 内でのみ使用される static 関数です。
  • 変換ロジックは Windows のファイルシステム時刻と POSIX 時刻のオフセットに強く依存しているため、定数 EpochShift の変更には注意が必要です。
