standby_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/standbydesc.c:78

目的

WALレコードの info フィールドに基づき、スタンバイ関連のログ種別を識別し、人間が判読可能な文字列を返します。

シグネチャまたは形状

const char *standby_identify(uint8 info);

入力と出力

  • **入力**: uint8 info - WALレコードの info ビットフィールド。
  • **出力**: const char * - 識別されたログ種別を表す文字列（"LOCK", "RUNNING_XACTS", "INVALIDATIONS" のいずれか）。該当しない場合は NULL を返します。

前提条件と不変条件

  • 入力 info に対して XLR_INFO_MASK を用いたビットマスク処理が行われ、上位ビット（フラグ領域）が除外された状態で判定されます。

アルゴリズムまたは振る舞い

1. 引数 info から XLR_INFO_MASK を除外した値に対して switch 文を実行します。 2. 以下のいずれかに合致する場合、対応する文字列を返します。

  • XLOG_STANDBY_LOCK: "LOCK"
  • XLOG_RUNNING_XACTS: "RUNNING_XACTS"
  • XLOG_INVALIDATIONS: "INVALIDATIONS"

    • 3. 上記のいずれにも該当しない場合、初期値である NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h 内の PG_RMGR マクロ定義を通じて、PostgreSQLのWALリソースマネージャ登録の一部として参照されています。
  • **呼び出し先**: 本関数内での関数呼び出しはありません。

関連データ構造

  • storage/standbydefs.h: XLOG_STANDBY_LOCK, XLOG_RUNNING_XACTS, XLOG_INVALIDATIONS などの定数が定義されています。

失敗モードと境界条件

  • info が既知のスタンバイ関連レコードタイプに一致しない場合、NULL を返します。呼び出し側はこの戻り値が NULL である可能性を考慮する必要があります。

保守担当者向けメモ

  • 新しいスタンバイ関連のWALレコードタイプを追加した場合は、必ず本関数の switch 文に新しいケースを追加してください。
  • 本関数は pg_waldump 等のツールでWALを可視化する際に使用されます。
