date_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/date.c:209

目的

外部バイナリ形式のデータを受け取り、PostgreSQLの内部表現である DateADT 型へ変換します。

シグネチャまたは形状

Datum date_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して StringInfo 形式のバッファを受け取ります。
  • **出力**: DateADT（int32）型の値を Datum として返却します。

前提条件と不変条件

  • 入力バッファは pq_getmsgint を介して sizeof(DateADT) バイトの整数として読み取られます。
  • DATE_NOT_FINITE マクロにより、無限大（Infinity）の値は妥当な値として許容されます。
  • 有限の値については IS_VALID_DATE マクロによる範囲チェックが行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) で取得した StringInfo バッファから、pq_getmsgint を用いて DateADT 値を抽出します。 2. 抽出した値が有限（!DATE_NOT_FINITE）である場合、IS_VALID_DATE を用いて有効な日付範囲内にあるか検証します。 3. 範囲外の場合は ERRCODE_DATETIME_VALUE_OUT_OF_RANGE エラーを報告します。 4. 検証を通過した値を PG_RETURN_DATEADT を介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのバイナリプロトコル受信処理から呼び出されます（具体的な関数名はソースコードの文脈からは特定できません）。
  • **呼び出し先**: pq_getmsgint（バイナリデータ読み取り）、ereport（エラー報告）。

関連データ構造

  • DateADT: int32 で表現される日付（エポックからの経過日数）。
  • StringInfo: バイナリメッセージを保持する構造体。

失敗モードと境界条件

  • **範囲外エラー**: IS_VALID_DATE の条件を満たさない日付値が入力された場合、ERROR レベルで処理が中断されます。
  • **無限大**: DATE_NOT_FINITE に該当する値は、範囲チェックをスキップして正常に処理されます。

保守担当者向けメモ

  • 本関数は date_in と同等の範囲制約を強制するように設計されています。変更を加える際は、date_in のバリデーションロジックとの整合性に注意してください。
