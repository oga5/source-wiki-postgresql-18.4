tsvectorout

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/tsvector.c:313

目的

内部バイナリ形式の TSVector 構造体を、SQL で表示可能なテキスト形式（例: 'lexeme':1,2A 'other':3B'）に変換する。

シグネチャまたは形状

Datum tsvectorout(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TSVECTOR(0) を介して取得される TSVector 型のデータ。
  • **出力**: PG_RETURN_CSTRING を介して返される、エスケープ処理済みの cstring（文字列）。

前提条件と不変条件

  • 入力された TSVector は、内部的に語彙（lexeme）がソート済みである必要がある。
  • 語彙内の '（シングルクォート）および \（バックスラッシュ）は、出力時に適切にエスケープされる。

アルゴリズムまたは振る舞い

1. **バッファサイズ計算**: 出力文字列に必要なメモリ量を、語彙の長さ、エスケープ分、および位置情報（:pos）の最大値から算出し、palloc で確保する。 2. **文字列構築**:

  • TSVector 内の WordEntry 配列を順次走査する。
  • 各語彙を ' で囲み、内部の特殊文字をエスケープする。
  • 位置情報（WordEntryPos）が存在する場合、: に続けて位置と重み（A, B, C）を付与する。
  • 複数の語彙がある場合はスペースで区切る。

    • 3. **終了処理**: 文字列の末尾にヌル終端を付与し、メモリを解放して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（SQL レベルの tsvector 型の出力関数として登録されている）。
  • **呼び出し先**: palloc（メモリ確保）、pg_mblen_range（マルチバイト文字長計算）、sprintf（位置情報の数値変換）。

関連データ構造

  • TSVector: 全文検索用データ型の内部表現。
  • WordEntry: 語彙のメタデータ（位置情報有無、長さ、オフセット）を保持する構造体。
  • WordEntryPos: 位置と重みをパックした 16 ビット整数。

失敗モードと境界条件

  • **メモリ不足**: palloc が失敗した場合、PostgreSQL のエラーハンドリングによりアボートされる。
  • **エンコーディング**: データベースのエンコーディング設定に基づき、マルチバイト文字の長さを正しく計算できない場合、不正な文字列が生成される可能性がある。

保守担当者向けメモ

  • 本関数は tsvector のテキスト表現を生成する唯一の経路である。出力形式を変更する場合は、tsvectorin（入力関数）との整合性を維持すること。
  • 位置情報の重み（Weight）の変換ロジック（3=A, 2=B, 1=C）は、tsearch/ts_type.h の定義と密接に関連しているため、変更時は注意が必要。
