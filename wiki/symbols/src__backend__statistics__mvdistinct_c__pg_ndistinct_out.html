pg_ndistinct_out

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mvdistinct.c:354

目的

pg_ndistinct 型の内部バイナリ表現を、デバッグや表示に適した人間が読める文字列形式（cstring）に変換します。

シグネチャまたは形状

Datum pg_ndistinct_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される bytea 型のデータ（pg_ndistinct のシリアライズ形式）。
  • **出力**: PG_RETURN_CSTRING を介して返される、列の組み合わせと推定NDistinct値を含む文字列（例: {"1, 2": 100, "3": 50}）。

前提条件と不変条件

  • 入力された bytea データは、statext_ndistinct_deserialize によって MVNDistinct 構造体に正しくデシリアライズ可能である必要があります。
  • 内部的に StringInfo を使用して文字列を構築するため、メモリ不足時にはエラーが発生する可能性があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_BYTEA_PP で入力データを取り出し、statext_ndistinct_deserialize を呼び出して MVNDistinct 構造体に変換します。 2. initStringInfo で文字列バッファを初期化し、開始文字 { を追加します。 3. MVNDistinct 内の各 MVNDistinctItem をループ処理します。 4. 各アイテムについて、含まれる属性番号（attnum）をカンマ区切りで列挙し、その後に対応する ndistinct 値を付与します。 5. 最後に終了文字 } を追加し、構築された文字列を cstring として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は確認されていません（通常、PostgreSQLの型システムを介して呼び出されます）。
  • **呼び出し先**:
  • statext_ndistinct_deserialize
  • initStringInfo, appendStringInfoChar, appendStringInfoString, appendStringInfo（lib/stringinfo.h）

関連データ構造

  • MVNDistinct: 多変量NDistinct統計情報を保持する構造体。
  • MVNDistinctItem: 個別の列組み合わせとそのNDistinct値を保持する構造体。
  • StringInfoData: 文字列構築用バッファ。

失敗モードと境界条件

  • 入力データが不正な形式である場合、statext_ndistinct_deserialize 内でエラーが発生する可能性があります。
  • 属性リストが空の場合や、アイテム数が0の場合の挙動は、MVNDistinct 構造体の内容に依存します。

保守担当者向けメモ

  • 本関数は pg_ndistinct 型の出力関数として機能します。
  • 統計情報の構造に変更があった場合は、この文字列フォーマットも適宜更新する必要があります。
