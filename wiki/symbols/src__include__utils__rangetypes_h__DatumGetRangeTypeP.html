DatumGetRangeTypeP

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/rangetypes.h:73

目的

Datum 型として渡された範囲型データを、メモリ上の RangeType 構造体ポインタへ変換（デトースト）します。

シグネチャまたは形状

static inline RangeType *DatumGetRangeTypeP(Datum X)

入力と出力

  • **入力**: Datum X （範囲型のデータ）
  • **出力**: RangeType * （デトーストされた範囲型構造体へのポインタ）

前提条件と不変条件

  • 入力 X は有効な範囲型の Datum である必要があります。
  • RangeType は可変長データ（varlena）であるため、本関数は必要に応じて PG_DETOAST_DATUM を介してメモリ上の実体を取得します。

アルゴリズムまたは振る舞い

1. 引数 X に対して PG_DETOAST_DATUM マクロを適用し、必要に応じて圧縮解除やメモリ配置の調整を行います。 2. 取得したポインタを RangeType * 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execIndexing.c
  • src/backend/utils/adt/multirangetypes.c
  • src/backend/utils/adt/multirangetypes_selfuncs.c
  • src/backend/utils/adt/rangetypes.c
  • src/backend/utils/adt/rangetypes_gist.c
  • **呼び出し先**:
  • PG_DETOAST_DATUM （マクロ）

関連データ構造

  • RangeType: 範囲型のメモリ表現。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • 入力 X が有効な範囲型データでない場合、PG_DETOAST_DATUM の挙動に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、範囲型を扱う際の標準的なデトースト手段です。
  • RangeType の内部フィールドを直接操作せず、必ず提供されているマクロや関数（range_deserialize 等）を経由してアクセスしてください。
