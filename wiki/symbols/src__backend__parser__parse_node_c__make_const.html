make_const

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_node.c:346

目的

構文解析器から渡された未加工の定数ノード（A_Const）を、型情報と値を持つ実行可能な Const ノードへ変換します。

シグネチャまたは形状

Const *make_const(ParseState *pstate, A_Const *aconst)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。エラー位置報告に使用。
  • aconst: 変換対象の未加工定数ノード（A_Const）。
  • **出力**:
  • 変換後の Const 構造体へのポインタ。

前提条件と不変条件

  • aconst->isnull が真の場合、UNKNOWNOID 型の NULL 定数が返されます。
  • T_String 型の定数は、後続の処理で型強制（coercion）が行われることを前提に UNKNOWNOID としてマークされます。
  • T_Float 型の入力は、int32 または int64 に収まる場合はそれぞれの整数型へ、それ以外は NUMERICOID へ変換されます。

アルゴリズムまたは振る舞い

1. aconst->isnull が真であれば、即座に NULL 定数ノードを生成して返します。 2. aconst->val のノードタグに基づき、以下の型変換を行います：

  • T_Integer: INT4OID へ変換。
  • T_Float: pg_strtoint64_safe を使用し、整数範囲内であれば INT4OID または INT8OID へ、範囲外であれば numeric_in を呼び出し NUMERICOID へ変換。
  • T_Boolean: BOOLOID へ変換。
  • T_String: UNKNOWNOID へ変換。
  • T_BitString: bit_in を呼び出し BITOID へ変換。

    • 3. 変換された値と型情報を基に makeConst を呼び出し、最終的な Const ノードを生成します。 4. aconst->location を Const ノードの location フィールドにコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_expr.c:158
  • contrib/postgres_fdw/deparse.c (参照)
  • src/backend/utils/adt/ruleutils.c (参照)
  • **呼び出し先**:
  • makeConst (ノード生成)
  • setup_parser_errposition_callback / cancel_parser_errposition_callback (エラー位置報告用)
  • DirectFunctionCall3 (numeric_in, bit_in 呼び出し)

関連データ構造

  • Const: 実行可能な定数ノード。
  • A_Const: 構文解析器が生成する未加工の定数ノード。
  • ParseState: 解析コンテキスト。

失敗モードと境界条件

  • T_Float の変換において numeric_in が失敗した場合、setup_parser_errposition_callback によりエラー位置が報告されます。
  • 未知のノードタグが aconst->val に含まれる場合、elog(ERROR, ...) が発生します。

保守担当者向けメモ

  • T_String は UNKNOWN 型として扱われますが、これは将来的な型解決を前提としています。
  • T_Float の処理において、整数への変換試行と numeric_in へのフォールバックという二段階のロジックが含まれています。
