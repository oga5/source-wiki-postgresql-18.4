pg_control_recovery

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/pg_controldata.c:162

目的

データベースの制御ファイル（global/pg_control）からリカバリ関連のステータス情報を読み取り、SQLの行型として公開します。

シグネチャまたは形状

Datum pg_control_recovery(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数として呼び出される際の引数）。
  • **出力**: Datum（リカバリ関連の5つのフィールドを含む行型データ）。

前提条件と不変条件

  • 呼び出し元はSQL関数インターフェースを介して実行される必要があります。
  • 戻り値の型は複合型（TYPEFUNC_COMPOSITE）である必要があります。
  • 制御ファイルへのアクセスには ControlFileLock（共有ロック）が必須です。

アルゴリズムまたは振る舞い

1. get_call_result_type を呼び出し、戻り値の型が複合型であることを確認します。 2. LWLockAcquire で ControlFileLock を共有モードで取得します。 3. get_controlfile を使用してディスクから制御ファイルを読み込みます。 4. LWLockRelease でロックを解放します。 5. 読み込んだデータのCRCチェックを行い、不一致の場合はエラーを発生させます。 6. ControlFileData 構造体から以下の値を抽出します：

  • minRecoveryPoint (LSN)
  • minRecoveryPointTLI (Int32)
  • backupStartPoint (LSN)
  • backupEndPoint (LSN)
  • backupEndRequired (Bool)

    • 7. heap_form_tuple を使用してこれらの値をタプルに変換し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SQL関数として実行）。
  • **呼び出し先**:
  • get_call_result_type
  • LWLockAcquire / LWLockRelease
  • get_controlfile
  • LSNGetDatum, Int32GetDatum, BoolGetDatum
  • heap_form_tuple
  • HeapTupleGetDatum

関連データ構造

  • ControlFileData (catalog/pg_control.h で定義): 制御ファイルの内容を保持する構造体。
  • TupleDesc, HeapTuple: SQL関数が返す行データを構築するためのPostgreSQL内部構造。

失敗モードと境界条件

  • 戻り値の型が複合型でない場合、elog(ERROR) で停止します。
  • 制御ファイルのCRCチェックに失敗した場合、ereport(ERROR) で停止します。

保守担当者向けメモ

  • 本関数はSQLインターフェース層であり、実際のファイル読み込みロジックは get_controlfile（src/common/controldata_utils.c）に依存しています。
  • 制御ファイルの整合性を保護するため、ロックの取得と解放の順序は厳守してください。
