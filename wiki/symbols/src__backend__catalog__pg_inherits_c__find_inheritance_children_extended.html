find_inheritance_children_extended

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_inherits.c:81

目的

指定された親テーブル（parentrelId）の直系の子テーブルの OID リストを取得します。パーティションのデタッチ処理中（inhdetachpending）の除外制御や、並行処理を考慮したロック取得を伴う拡張的な探索を行います。

シグネチャまたは形状

List *find_inheritance_children_extended(Oid parentrelId, bool omit_detached,
                                         LOCKMODE lockmode, bool *detached_exist,
                                         TransactionId *detached_xmin)

入力と出力

  • **入力**:
  • parentrelId: 親テーブルの OID。
  • omit_detached: true の場合、デタッチ処理中のパーティションを結果から除外する。
  • lockmode: 子テーブル取得時に適用するロックモード（NoLock 指定可）。
  • detached_exist: （出力用ポインタ）デタッチ処理中のパーティションが存在した場合に true が設定される。
  • detached_xmin: （出力用ポインタ）デタッチ処理中のパーティションの xmin を追跡する。
  • **出力**: 子テーブルの OID を格納した List。

前提条件と不変条件

  • has_subclass(parentrelId) が false の場合、即座に NIL を返す。
  • lockmode が NoLock でない場合、子テーブルのロック取得後に pg_class を再確認し、存在しない場合はリストから除外する。
  • 複数の子テーブルが見つかった場合、デッドロック防止のため OID 順にソートされる。

アルゴリズムまたは振る舞い

1. has_subclass で子が存在するか確認し、なければ終了。 2. pg_inherits カタログを InheritsParentIndexId でスキャンし、該当する子テーブルの OID を収集する。 3. inhdetachpending が設定されている場合、omit_detached とアクティブスナップショットの状態に基づき、除外判定を行う。 4. 収集した OID をソートし、指定された lockmode で各リレーションをロックする。 5. ロック取得後に SearchSysCacheExists1 で存在確認を行い、生存しているリレーションのみをリストに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/partitioning/partdesc.c
  • src/backend/catalog/pg_inherits.c（内部ラッパー関数）
  • **呼び出し先**:
  • has_subclass, table_open, systable_beginscan, systable_getnext, systable_endscan, table_close, qsort, LockRelationOid, UnlockRelationOid, SearchSysCacheExists1, lappend_oid 等。

関連データ構造

  • pg_inherits: 継承関係を保持するシステムカタログ。
  • List: PostgreSQL のリスト構造。
  • ScanKeyData: pg_inherits スキャン用のキー。

失敗モードと境界条件

  • デタッチ処理中のパーティションが複数存在する場合、elog(WARNING) を発行し、より新しい xmin を detached_xmin に保持する。
  • ロック待機中にテーブルが削除された場合、そのテーブルはリストから除外される。

保守担当者向けメモ

  • 本関数はパーティションのデタッチ処理という複雑な状態遷移を扱うため、omit_detached フラグの挙動を変更する際は、MVCC スナップショットとの整合性に注意すること。
  • lockmode に NoLock を使用する場合、並行的な DROP 操作によるレースコンディションが発生する可能性があるため、呼び出し側で適切な保護が必要である。
