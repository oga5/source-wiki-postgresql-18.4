pct_info

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/orderedsetaggs.c:634

目的

連続パーセンタイル計算において、補間（interpolation）が必要な行のインデックスと、その重み付けを保持するための構造体です。

シグネチャまたは形状

struct pct_info
{
    int64       first_row;      /* 最初のサンプリング対象行 */
    int64       second_row;     /* 補間が必要な場合の2番目の行 */
    double      proportion;     /* 補間比率 */
    int         idx;            /* 元の配列におけるインデックス */
};

入力と出力

  • 本構造体は percentile_cont 系の計算処理において、計算対象となる行の特定と補間係数の算出結果を格納するために使用されます。

前提条件と不変条件

  • second_row は first_row と等しいか、それより大きい値である必要があります。

アルゴリズムまたは振る舞い

  • percentile_cont_final_common 等の計算ロジック内で、指定されたパーセンタイル値 p と全行数 rowcount に基づき、first_row および second_row が算出されます。
  • 具体的には 1 + floor(p * (rowcount - 1)) および 1 + ceil(p * (rowcount - 1)) を用いて行位置が決定されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/orderedsetaggs.c 内のパーセンタイル計算関数群。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • OSAPerQueryState: 集約関数のクエリ状態を管理する構造体。
  • OSAPerGroupState: グループごとの実行状態を管理する構造体。

失敗モードと境界条件

  • rowcount が 0 または 1 の場合、計算ロジックにおいて first_row と second_row の値が境界条件に依存します。
  • p（パーセンタイル値）が 0 から 1 の範囲外である場合の挙動は、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は percentile_cont の計算ロジックに特化しています。変更を加える際は、orderedsetaggs.c 内の補間計算ロジックとの整合性を確認してください。
