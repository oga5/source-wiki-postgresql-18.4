startScanEntry

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginget.c:318

目的

GIN インデックススキャンにおいて、特定の検索キーに対応するエントリの走査を開始し、該当する TID（タプル識別子）リストまたはビットマップをメモリ上に準備する。

シグネチャまたは形状

static void startScanEntry(GinState *ginstate, GinScanEntry entry, Snapshot snapshot)

入力と出力

  • **入力**:
  • ginstate: GIN インデックスの状態管理構造体。
  • entry: 走査対象の GinScanEntry 構造体。
  • snapshot: 検索に使用するスナップショット。
  • **出力**: なし（entry 構造体の内部状態を更新する）。

前提条件と不変条件

  • entry が指すメモリ領域は有効であること。
  • ginstate は初期化済みであること。
  • 走査開始前に entry の既存のリストやビットマップは解放・初期化される。

アルゴリズムまたは振る舞い

1. **初期化**: entry の走査状態（buffer, list, matchBitmap 等）をリセットする。 2. **検索**: ginPrepareEntryScan および ginFindLeafPage を使用して、インデックスのリーフページを特定する。 3. **分岐処理**:

  • **部分一致または空クエリの場合**: collectMatchBitmap を呼び出し、該当する全 TID をビットマップに収集する。再構築が必要な場合は再帰的にリトライする。
  • **完全一致の場合**:
  • **Posting Tree の場合**: ルートページをロックして PredicateLockPage を適用後、リーフページから TID リストを読み込み、バッファをピン留めしたまま走査準備を完了する。
  • **Posting List の場合**: リーフページ上のタプルから直接 TID リストを読み込む。

    • 4. **終了処理**: 走査に使用したバッファのロックを解除し、スタックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginget.c 内の startScan 関数から呼び出される。
  • **呼び出し先**: ginPrepareEntryScan, ginFindLeafPage, collectMatchBitmap, tbm_begin_private_iterate, ginScanBeginPostingTree, GinDataLeafPageGetItems, ginReadTuple, PredicateLockPage 等。

関連データ構造

  • GinState: インデックス全体のメタデータ。
  • GinScanEntry: 個別の検索キーの走査状態を保持する構造体。
  • GinBtreeData, GinBtreeStack: B-tree 走査用の補助構造体。

失敗モードと境界条件

  • **再構築**: collectMatchBitmap がインデックスの再構築を検知した場合、restartScanEntry ラベルへジャンプし、走査を最初からやり直す。
  • **エントリ不在**: 該当するキーが見つからない場合、PredicateLockPage を適用してその場所をロックし、isFinished を true のまま終了する。

保守担当者向けメモ

  • Posting Tree を走査する際、デッドロック回避のためにエントリページのロックを一時的に解除する実装が含まれている。
  • entry->buffer は、Posting Tree の走査中にページ削除を防ぐために参照カウントが増加（IncrBufferRefCount）されている点に注意。
