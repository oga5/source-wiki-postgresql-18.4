build_client_first_message

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:349

目的

SCRAM-SHA-256 認証プロトコルにおけるクライアントの最初のメッセージ（Client-First-Message）を構築します。

シグネチャまたは形状

static char *build_client_first_message(fe_scram_state *state);

入力と出力

  • **入力**: fe_scram_state *state（認証セッションの状態を保持する構造体）。
  • **出力**: 構築されたメッセージ文字列（char *）。失敗した場合は NULL を返し、state->conn にエラー情報を設定します。

前提条件と不変条件

  • state は有効なポインタであること。
  • state->sasl_mechanism が設定されていること。
  • 内部で生成される client_nonce は、後続の認証ステップで使用されるため state に保持されます。

アルゴリズムまたは振る舞い

1. **Nonce 生成**: pg_strong_random を使用してランダムなバイト列を生成し、Base64 エンコードして state->client_nonce に格納します。 2. **GS2 ヘッダー構築**:

  • チャネルバインディングが有効（SCRAM_SHA_256_PLUS_NAME）な場合は p=tls-server-end-point を付与。
  • SSL が使用されており、かつチャネルバインディングが明示的に無効化されていない場合は y を付与。
  • それ以外は n を付与。

    • 3. **メッセージ構築**: PQExpBuffer を使用し、gs2-header,,n=,r=<nonce> の形式でメッセージを組み立てます。 4. **状態保存**: チャネルバインディング情報を含まないメッセージの断片を state->client_first_message_bare に保存します。 5. **メモリ管理**: 構築したメッセージ文字列を複製して返却し、作業用バッファを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-scram.c 内の認証フロー（scram_exchange 等）から呼び出されます。
  • **呼び出し先**: pg_strong_random, pg_b64_encode, initPQExpBuffer, appendPQExpBufferStr 等のユーティリティ関数。

関連データ構造

  • fe_scram_state: 認証セッションの状態を保持。
  • PQExpBufferData: メッセージ構築用の一時バッファ。

失敗モードと境界条件

  • pg_strong_random が失敗した場合、エラーを報告し NULL を返します。
  • メモリ割り当て（malloc, strdup）に失敗した場合、oom_error ラベルへ遷移し、エラーを報告して NULL を返します。
  • Base64 エンコードに失敗した場合、エラーを報告し NULL を返します。

保守担当者向けメモ

  • 本関数は state->client_first_message_bare を生成する責任を負います。これは後続の署名検証で必要となるため、変更時は注意してください。
  • ユーザー名はバックエンド側でスタートアップパケットの値が使用されるため、本メッセージ内では空（n=）として扱われます。
