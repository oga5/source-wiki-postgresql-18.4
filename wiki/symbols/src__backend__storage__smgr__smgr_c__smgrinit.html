smgrinit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/smgr.c:187

目的

ストレージマネージャ（SMgr）サブシステムの初期化を行い、プロセス終了時のシャットダウン処理を登録する。

シグネチャまたは形状

void smgrinit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてストレージ実装の初期化と終了ハンドラの登録を行う）

前提条件と不変条件

  • バックエンドの起動時（通常またはスタンドアロン）に呼び出される必要がある。
  • 割り込みを一時的に禁止した状態で実行される。

アルゴリズムまたは振る舞い

1. HOLD_INTERRUPTS() を呼び出し、初期化中の割り込みを禁止する。 2. smgrsw 配列を走査し、各ストレージ実装に定義されている smgr_init 関数が存在すれば順次実行する。 3. RESUME_INTERRUPTS() を呼び出し、割り込みを許可する。 4. on_proc_exit を使用して、プロセス終了時に smgrshutdown が実行されるよう登録する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c
  • 呼び出し先: smgrsw 内の各ストレージ実装の初期化関数、on_proc_exit、smgrshutdown

関連データ構造

  • smgrsw: ストレージマネージャの関数ポインタを保持するスイッチ配列。

失敗モードと境界条件

  • 割り込み禁止期間中にエラーが発生した場合の挙動は、PostgreSQL の標準的なエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数はバックエンド起動時の重要な初期化ステップである。smgrsw に新しいストレージ実装を追加した際は、その初期化関数が正しく smgr_init に登録されていることを確認すること。
