enum_ge

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:341

目的

2つの列挙型（ENUM）値の「以上（greater than or equal to）」比較を行い、真偽値を返します。

シグネチャまたは形状

Datum enum_ge(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの列挙型OID（a, b）を受け取ります。
  • **出力**: a >= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力されたOIDは、システムカタログ pg_enum に存在する有効な列挙値である必要があります。
  • 比較ロジックは enum_cmp_internal に依存しており、列挙型の順序付けルールに従います。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID を使用して、引数から2つの列挙型OIDを取得します。 2. enum_cmp_internal を呼び出し、2つの値の比較結果（整数）を取得します。 3. 比較結果が 0 以上（>= 0）であるかどうかを判定し、その結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_enum.c:42（直接の呼び出しが確認されています）
  • **呼び出し先**:
  • enum_cmp_internal（比較ロジックの実行）

関連データ構造

  • Oid: 列挙値を識別するための型。
  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に基づく引数コンテナ。

失敗モードと境界条件

  • enum_cmp_internal が内部的にエラーを発生させる場合（無効なOIDや型不整合など）、本関数もそのエラーを伝播します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の比較ロジックは enum_cmp_internal に集約されています。比較順序の変更やキャッシュ戦略の修正が必要な場合は、enum_cmp_internal を参照してください。
