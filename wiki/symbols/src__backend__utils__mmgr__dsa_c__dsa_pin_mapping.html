dsa_pin_mapping

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:634

目的

DSA（Dynamic Shared Memory Area）のメモリマッピングを現在のプロセスに固定し、リソース所有者（resowner）による自動的な解放対象から除外する。

シグネチャまたは形状

void dsa_pin_mapping(dsa_area *area)

入力と出力

  • **入力**:
  • area: 固定対象の dsa_area 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • area が有効な dsa_area 構造体を指していること。
  • area->resowner が設定されている場合、その DSA 領域は現在のリソース所有者の生存期間に紐付いている。

アルゴリズムまたは振る舞い

1. area->resowner が NULL でないかを確認する。 2. area->resowner を NULL に設定し、リソース所有者による管理を解除する。 3. area->high_segment_index までループし、各セグメントマップを確認する。 4. セグメントが割り当てられている場合（segment != NULL）、dsm_pin_mapping() を呼び出し、当該 DSM セグメントのマッピングを固定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/session.c
  • src/backend/replication/logical/launcher.c
  • src/backend/storage/ipc/dsm_registry.c
  • src/backend/utils/activity/pgstat_shmem.c
  • src/test/modules/test_tidstore/test_tidstore.c
  • **呼び出し先**:
  • dsm_pin_mapping（storage/dsm.h）

関連データ構造

  • dsa_area: DSA 領域の状態を保持する構造体。
  • dsm_segment: DSM（Dynamic Shared Memory）セグメント。

失敗モードと境界条件

  • area が NULL の場合、セグメンテーションフォールトが発生する可能性がある。
  • area->resowner が既に NULL の場合、関数は何も行わず正常終了する。

保守担当者向けメモ

  • 本関数は、DSA 領域の生存期間をリソース所有者の生存期間から切り離すために使用される。
  • dsm_pin_mapping を呼び出すことで、プロセス終了時やリソース所有者のクリーンアップ時に DSM セグメントが自動的にデタッチされるのを防ぐ。
