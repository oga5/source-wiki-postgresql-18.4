CheckDuplicateColumnOrPathNames

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_jsontable.c:170

目的

JSON_TABLE 構文において、カラム名およびパス名が重複していないかを検証し、一意性を保証する。

シグネチャまたは形状

static void CheckDuplicateColumnOrPathNames(JsonTableParseContext *cxt, List *columns)

入力と出力

  • **入力**:
  • cxt: 解析コンテキスト（JsonTableParseContext）。既存の名前リストを保持し、エラー位置情報を管理する。
  • columns: 検証対象の JsonTableColumn ノードのリスト。
  • **出力**: なし（重複がある場合は ereport(ERROR, ...) により処理を中断する）。

前提条件と不変条件

  • cxt->pathNames には、既に処理済みの名前が格納されている必要がある。
  • columns リスト内の各要素は JsonTableColumn としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. columns リストを走査する。 2. 各カラムが JTC_NESTED 型の場合:

  • pathspec->name が存在すれば LookupPathOrColumnName で重複を確認し、なければ cxt->pathNames に追加する。
  • jtc->columns に対して再帰的に本関数を呼び出し、ネストされた階層の重複もチェックする。

    • 3. JTC_NESTED 以外の場合:

  • jtc->name を LookupPathOrColumnName で重複確認し、なければ cxt->pathNames に追加する。

    • 4. 重複が検出された場合、ERRCODE_DUPLICATE_ALIAS を伴うエラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_jsontable.c 内のメイン解析処理（transformJsonTable 等）。
  • 本関数自身（NESTED 句の再帰処理）。
  • **呼び出し先**:
  • LookupPathOrColumnName: 名前の重複判定に使用。
  • ereport: エラー報告に使用。

関連データ構造

  • JsonTableParseContext: 解析状態と名前のリストを保持。
  • JsonTableColumn: JSON_TABLE のカラム定義ノード。

失敗モードと境界条件

  • **重複エラー**: 同一スコープ内または親スコープで既に定義された名前と衝突した場合、ERROR をスローする。
  • **再帰的構造**: NESTED 句が深くネストされている場合でも、再帰的にすべての階層を走査する。

保守担当者向けメモ

  • NESTED 句の処理において、pathspec->name の有無によって条件分岐している点に注意。
  • cxt->pathNames は解析が進むにつれて累積的に更新されるため、呼び出し順序が重要である。
