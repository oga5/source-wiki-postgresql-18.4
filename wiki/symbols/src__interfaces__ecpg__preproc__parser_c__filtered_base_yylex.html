filtered_base_yylex

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/parser.c:56

目的

ECPGプリプロセッサにおいて、スキャナ（base_yylex_location）から取得したトークンを先読みし、構文解析器（Bison）が曖昧さを回避できるようトークンの置換や結合を行うフィルタリング関数。

シグネチャまたは形状

int filtered_base_yylex(void)

入力と出力

  • **入力**: 内部状態変数 have_lookahead、lookahead_token 等、および base_yylex_location() を介したスキャナからのトークンストリーム。
  • **出力**: 構文解析器へ渡す int 型のトークンID。副作用として base_yylval、base_yylloc、base_yytext が更新される。

前提条件と不変条件

  • have_lookahead が真の場合、先読みされたトークン情報が優先的に使用される。
  • UESCAPE 句の処理時、UIDENT または USCONST に続くトークンが UESCAPE であれば、さらに後続の SCONST を取得して3つのトークンを1つに統合する。

アルゴリズムまたは振る舞い

1. **先読み確認**: have_lookahead が真なら保存されたトークンを返し、偽なら base_yylex_location() を呼び出す。 2. **フィルタリング判定**: FORMAT, NOT, NULLS_P, WITH, WITHOUT, UIDENT, USCONST のいずれかであれば、次トークンを先読みする。 3. **トークン置換**: 先読みしたトークンに基づき、FORMAT → FORMAT_LA 等の専用トークンへ変換する。 4. **トークン結合**: UIDENT/USCONST の直後に UESCAPE と SCONST が続く場合、これらを結合して単一のトークンとして処理し、have_lookahead を偽にリセットする。 5. **返却**: 最終的なトークンIDを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 構文解析器（gram.y に相当するパーサ）。
  • **呼び出し先**: base_yylex_location()（スキャナのラッパー）、mmerror()（エラー報告）、check_uescapechar()（Unicodeエスケープ文字検証）、make3_str()（文字列結合）。

関連データ構造

  • have_lookahead: 先読み状態を保持するフラグ。
  • lookahead_token, lookahead_yylval, lookahead_yylloc, lookahead_yytext: 先読みしたトークン情報を一時保存する変数群。
  • base_yylval, base_yylloc, base_yytext: スキャナの出力変数。

失敗モードと境界条件

  • UESCAPE の直後に SCONST が存在しない場合、mmerror により PARSE_ERROR が発生する。
  • UESCAPE のエスケープ文字列が不正（長さが3以外、または check_uescapechar が偽）な場合、mmerror によりエラーとなる。

保守担当者向けメモ

  • UESCAPE 処理は3つのトークンを消費する複雑なロジックであるため、トークンの取得順序や have_lookahead のフラグ管理に変更を加える際は注意が必要。
  • yylloc の生成や文字列の小文字化は、ECPGの歴史的互換性を維持するための仕様である。
