select_loop

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:79

目的

select() システムコールをラップし、割り込みやキャンセル要求を適切に処理しながら、ファイルディスクリプタの監視を行う。

シグネチャまたは形状

static int select_loop(int maxFd, fd_set *workerset)

入力と出力

  • **入力**:
  • maxFd: 監視対象の最大ファイルディスクリプタ番号。
  • workerset: 監視対象のファイルディスクリプタ集合（fd_set）。
  • **出力**:
  • 成功時: select() が返した準備完了ディスクリプタ数（正の整数）。
  • 失敗時: -1（キャンセル要求時、またはシステムエラー時）。

前提条件と不変条件

  • CancelRequested グローバル変数が設定されている場合、即座に -1 を返す。
  • workerset は呼び出しごとに再構築される必要がある（select() によって破壊的に更新されるため）。

アルゴリズムまたは振る舞い

1. CancelRequested を確認し、真であれば -1 を返す。 2. 無限ループ内で select() を呼び出す。

  • **Windows環境**: 1秒のタイムアウトを設定し、定期的にキャンセル要求を確認する。
  • **その他**: タイムアウトなし（NULL）でブロックする。

    • 3. select() の結果を評価する。

  • EINTR（割り込み）が発生した場合はループを継続する。
  • CancelRequested が真になった場合、または select() がエラーを返した場合は -1 を返す。
  • タイムアウト（Windowsのみ）の場合はループを継続する。
  • 準備完了ディスクリプタがある場合はループを抜けて値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/fe_utils/parallel_slot.c 内の wait_on_slots 等から呼び出される。
  • src/bin/pg_dump/parallel.c および src/bin/pg_upgrade/task.c からの参照が確認されている。
  • **呼び出し先**:
  • select() システムコール。
  • WSAGetLastError()（Windows環境のみ）。

関連データ構造

  • fd_set: I/O多重化のためのファイルディスクリプタ集合。

失敗モードと境界条件

  • CancelRequested が真の場合、処理を中断して -1 を返す。
  • select() が SOCKET_ERROR（Windows）または負の値を返した場合、エラーとして扱う。
  • EINTR は無視され、再試行される。

保守担当者向けメモ

  • Windows環境では FD_SETSIZE の制限により、1024を超えるディスクリプタを扱うとエラーになる可能性がある。
  • workerset は select() によって書き換えられるため、呼び出し元で再利用する場合は必ず再構築すること。
