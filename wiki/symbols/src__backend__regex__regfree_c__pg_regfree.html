pg_regfree

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regfree.c:48

目的

コンパイル済みの正規表現オブジェクト（regex_t）に関連付けられたメモリおよびリソースを解放するための汎用インターフェースを提供します。

シグネチャまたは形状

void pg_regfree(regex_t *re)

入力と出力

  • **入力**: regex_t *re - 解放対象の正規表現オブジェクトへのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • re が NULL の場合、関数は何もせず安全に終了します。
  • re が有効なポインタである場合、re->re_fns は struct fns 型の有効な関数テーブルを指している必要があります。

アルゴリズムまたは振る舞い

1. 引数 re が NULL かどうかを判定し、NULL であれば即座にリターンします。 2. re->re_fns を struct fns * にキャストし、その構造体内の free 関数ポインタを介して、具体的な解放処理を呼び出します。 3. この間接呼び出しにより、正規表現のコンパイル時に決定された文字サイズやエンコーディングに応じた適切な解放処理が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_regexp.c
  • src/backend/libpq/hba.c
  • src/test/modules/test_regex/test_regex.c
  • **呼び出し先**:
  • re->re_fns 内の free 関数ポインタが指す具体的な解放関数（コンパイル時に決定される）。

関連データ構造

  • regex_t: POSIX 標準の正規表現構造体。
  • struct fns: regex/regguts.h で定義される、正規表現エンジンの関数テーブル。

失敗モードと境界条件

  • re が NULL の場合はガード節により安全に処理されます。
  • re->re_fns が初期化されていない、あるいは不正なポインタである場合、間接呼び出し時に未定義の動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は単なるディスパッチャです。具体的な解放ロジックは re_fns を介して実行されるため、メモリリーク等の調査を行う際は、対象の regex_t がどの関数テーブルを使用して初期化されたかを追跡してください。
  • regcomp.c と対になるライフサイクル管理の終端です。
