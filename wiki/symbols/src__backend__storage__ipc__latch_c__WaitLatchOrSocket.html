WaitLatchOrSocket

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/latch.c:222

目的

ラッチ（Latch）のセット、ソケットのイベント、またはポストマスターの終了といった複数の待機条件を統合し、OS の待機プリミティブを用いてプロセスを効率的に待機させる。

シグネチャまたは形状

int WaitLatchOrSocket(Latch *latch, int wakeEvents, pgsocket sock, long timeout, uint32 wait_event_info);

入力と出力

  • **入力**:
  • latch: 待機対象のラッチ（NULL 可）。
  • wakeEvents: 待機するイベントのビットマスク（WL_TIMEOUT, WL_LATCH_SET, WL_POSTMASTER_DEATH, WL_EXIT_ON_PM_DEATH, WL_SOCKET_MASK 等）。
  • sock: 待機対象のソケット記述子。
  • timeout: タイムアウト時間（ミリ秒）。
  • wait_event_info: 待機イベントの統計情報用識別子。
  • **出力**:
  • 発生したイベントを示すビットマスクを int 型で返す。

前提条件と不変条件

  • wakeEvents に WL_TIMEOUT が含まれる場合、timeout は 0 以上である必要がある。
  • ポストマスター管理下のプロセスは、WL_EXIT_ON_PM_DEATH または WL_POSTMASTER_DEATH のいずれかを指定してポストマスターの終了を監視しなければならない。
  • 内部で WaitEventSet を生成・破棄するため、呼び出しごとにリソースの確保と解放が行われる。

アルゴリズムまたは振る舞い

1. CreateWaitEventSet を呼び出し、待機イベントセットを初期化する。 2. wakeEvents に基づき、ラッチ、ポストマスター終了監視、ソケットイベントを AddWaitEventToSet でセットに追加する。 3. WaitEventSetWait を呼び出し、いずれかのイベントが発生するかタイムアウトするまで待機する。 4. 発生したイベントをビットマスクとして抽出し、FreeWaitEventSet でリソースを解放して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/connection.c, src/backend/libpq/auth.c, src/backend/libpq/be-secure-gssapi.c, src/backend/libpq/be-secure-openssl.c, src/backend/replication/logical/tablesync.c 等から参照されている。
  • **呼び出し先**: CreateWaitEventSet, AddWaitEventToSet, WaitEventSetWait, FreeWaitEventSet を呼び出す。

関連データ構造

  • Latch: 待機状態を管理する構造体。
  • WaitEventSet: 複数のイベントを監視するための抽象化されたセット構造。
  • WaitEvent: 発生したイベントの詳細を保持する構造体。

失敗モードと境界条件

  • timeout が 0 の場合、即座に返る可能性がある。
  • wakeEvents で指定されたイベントが一つもセットされない場合、待機条件が不完全となる可能性がある。
  • ポストマスター終了監視が有効な環境で、ポストマスターが終了した場合はその旨がイベントとして返される。

保守担当者向けメモ

  • 本関数は内部で WaitEventSet を動的に確保するため、頻繁な呼び出しはオーバーヘッドとなる可能性がある。
  • WaitLatch と異なり、ソケット監視を統合しているため、ネットワーク I/O を伴う待機処理に適している。
