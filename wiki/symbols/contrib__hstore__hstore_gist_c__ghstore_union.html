ghstore_union

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_gist.c:309

目的

GiST インデックスの構築プロセスにおいて、複数の hstore シグネチャ（GISTTYPE）を統合し、それらを包含する単一のシグネチャを生成する。

シグネチャまたは形状

Datum ghstore_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *（統合対象のインデックスエントリ群）。
  • PG_GETARG_POINTER(1): int *（結果のサイズを格納する出力用ポインタ）。
  • **出力**:
  • 戻り値: Datum（統合された GISTTYPE 構造体へのポインタ）。
  • *size: 統合後のシグネチャのバイトサイズ。

前提条件と不変条件

  • GET_SIGLEN() により、現在のインデックス設定に基づいたシグネチャ長が取得可能であること。
  • ghstore_alloc により、指定された siglen に基づくメモリ領域が確保されること。

アルゴリズムまたは振る舞い

1. ghstore_alloc を呼び出し、初期状態（空）の GISTTYPE を作成する。 2. entryvec 内の各エントリを順次走査し、unionkey を用いてビットベクトルを統合する。 3. 統合の過程で unionkey が ALLISTRUE（全ビットがセットされた状態）を返した場合、処理を中断し、結果のフラグに ALLISTRUE をセットしてサイズを調整する。 4. 最終的な GISTTYPE のサイズを *size に書き込み、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセス（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: ghstore_alloc, unionkey, GETENTRY, GETSIGN, SET_VARSIZE, CALCGTSIZE, VARSIZE。

関連データ構造

  • GISTTYPE: インデックスに格納されるシグネチャ構造体。
  • GistEntryVector: 統合対象となる複数のインデックスエントリを保持する構造体。
  • BITVECP: シグネチャのビットベクトルを操作するための型。

失敗モードと境界条件

  • entryvec が空の場合、空のシグネチャが返される。
  • 統合中にビットベクトルが飽和（全ビットがセット）した場合、早期終了して ALLISTRUE フラグが有効化される。

保守担当者向けメモ

  • 本関数は GiST インデックスの union メソッドとして機能する。
  • ALLISTRUE フラグがセットされると、そのノードは「すべてのキーを含む」とみなされ、検索時に常にヒットするようになるため、インデックスの選択性に影響を与える。
