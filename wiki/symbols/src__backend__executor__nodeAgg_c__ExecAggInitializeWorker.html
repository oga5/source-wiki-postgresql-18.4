ExecAggInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAgg.c:4831

目的

並列クエリ実行時、各ワーカープロセスにおいて集約ノードの共有情報を初期化します。

シグネチャまたは形状

void ExecAggInitializeWorker(AggState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の AggState 構造体。
  • pwcxt: 並列ワーカーのコンテキスト情報。
  • **出力**: なし（node->shared_info への代入による副作用）。

前提条件と不変条件

  • pwcxt->toc が有効であること。
  • node->ss.ps.plan が存在し、有効な plan_node_id を保持していること。

アルゴリズムまたは振る舞い

1. pwcxt->toc（共有メモリのテーブル・オブ・コンテント）から、現在のプランノード ID に対応する共有情報を検索します。 2. 検索結果を node->shared_info に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1398
  • **呼び出し先**: shm_toc_lookup（共有メモリ管理関数）

関連データ構造

  • AggState: 集約ノードの実行状態を保持する構造体。
  • ParallelWorkerContext: 並列実行時のワーカー用コンテキスト。

失敗モードと境界条件

  • shm_toc_lookup の第3引数が true で指定されているため、対応するキーが見つからない場合はエラーが発生します。

保守担当者向けメモ

  • 本関数は並列クエリのワーカー起動時に呼び出されるため、共有メモリのレイアウト変更時には shm_toc_lookup のキー整合性に注意してください。
