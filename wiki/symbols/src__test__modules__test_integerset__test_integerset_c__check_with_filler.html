check_with_filler

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_integerset/test_integerset.c:465

目的

IntegerSet 内の特定の整数 x が、期待される集合に含まれているかを検証するテスト用関数です。単一の値 value と、連続する範囲 [filler_min, filler_max) の両方が集合に含まれている前提で、その正当性を確認します。

シグネチャまたは形状

static void check_with_filler(IntegerSet *intset, uint64 x, uint64 value, uint64 filler_min, uint64 filler_max)

入力と出力

  • **入力**:
  • intset: 検証対象の IntegerSet へのポインタ。
  • x: 存在確認を行う対象の整数。
  • value: 集合に含まれるべき単一の特定値。
  • filler_min: 集合に含まれるべき範囲の開始値（境界値を含む）。
  • filler_max: 集合に含まれるべき範囲の終了値（境界値を含まない）。
  • **出力**: なし（検証失敗時に elog(ERROR) を送出）。

前提条件と不変条件

  • intset は初期化済みであること。
  • value および [filler_min, filler_max) の範囲内の値が、事前に intset に追加されていることを前提としたテストロジックです。

アルゴリズムまたは振る舞い

1. x が value と一致するか、あるいは filler_min <= x < filler_max の範囲内にあるかを判定し、期待値 expected を算出します。 2. intset_is_member(intset, x) を呼び出し、実際の存在判定結果 actual を取得します。 3. actual と expected が一致しない場合、elog(ERROR) を発生させてテストを中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_integerset/test_integerset.c 内のテスト関数（test_single_value_and_filler 等の文脈で呼び出される）。
  • **呼び出し先**: intset_is_member（lib/integerset.h で定義）。

関連データ構造

  • IntegerSet: lib/integerset.h で定義された整数集合データ構造。

失敗モードと境界条件

  • x が期待値と異なる判定結果を返した場合、ERROR レベルのログを出力して処理を停止します。
  • filler_min や filler_max の境界値（filler_min - 1, filler_max + 1 等）において、期待通りの判定が行われるかを検証する設計となっています。

保守担当者向けメモ

  • 本関数は test_single_value_and_filler 等のテストケースから、境界値を網羅的にチェックするために呼び出されています。
  • intset_is_member の実装に変更を加えた際は、本関数がカバーする境界条件（filler_min および filler_max の前後）で回帰が発生していないか確認してください。
