ReadBuffersOperation

定義

  • 種別: struct
  • 定義箇所: src/include/storage/bufmgr.h:121

目的

非同期I/Oを利用した複数ブロックの読み込み操作を管理するための構造体です。バッファマネージャにおける読み込み要求の開始から完了までの状態を保持します。

シグネチャまたは形状

struct ReadBuffersOperation {
    /* 呼び出し元が設定するメンバ */
    Relation    rel;
    struct SMgrRelationData *smgr;
    char        persistence;
    ForkNumber  forknum;
    BufferAccessStrategy strategy;

    /* 内部状態（StartReadBuffersとWaitReadBuffers間の通信用） */
    Buffer     *buffers;
    BlockNumber blocknum;
    int         flags;
    int16       nblocks;
    int16       nblocks_done;
    PgAioWaitRef io_wref;
    PgAioReturn io_return;
};

入力と出力

  • **入力**: 呼び出し元は rel（または smgr）、persistence、forknum、strategy を設定する必要があります。
  • **出力**: buffers 配列に読み込まれたバッファが格納され、nblocks_done に完了したブロック数が反映されます。

前提条件と不変条件

  • buffers、blocknum、flags などの内部メンバは、StartReadBuffers() によって初期化されるまで変更してはなりません。
  • 読み込みが必要な場合にのみ内部状態が初期化されます。

アルゴリズムまたは振る舞い

1. **初期化**: 呼び出し元が公開メンバを設定します。 2. **開始**: StartReadBuffers() を呼び出し、非同期I/Oを開始します。 3. **待機**: WaitReadBuffers() を呼び出し、I/Oの完了を待ちます。 4. **結果処理**: ProcessReadBuffersResult() を通じて、読み込み結果の検証や後続処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/read_stream.c および src/backend/storage/buffer/bufmgr.c 内の各関数から参照されています。
  • **呼び出し先**: StartReadBuffers()、WaitReadBuffers()、CheckReadBuffersOperation()、ProcessReadBuffersResult() などのバッファマネージャ内部関数によって操作されます。

関連データ構造

  • Relation: 対象リレーション。
  • BufferAccessStrategy: バッファ置換戦略。
  • PgAioWaitRef / PgAioReturn: 非同期I/Oの制御および結果格納用構造体。

失敗モードと境界条件

  • CheckReadBuffersOperation() による健全性チェックが実装されており、操作の整合性が検証されます。
  • 読み込み対象のブロック数が nblocks を超える場合や、無効なバッファ状態での操作は失敗モードとなります。

保守担当者向けメモ

  • 本構造体は StartReadBuffers() と WaitReadBuffers() 間のプライベートな状態保持を目的としています。
  • 内部メンバ（io_wref 等）は直接変更せず、提供されている API を介して操作してください。
