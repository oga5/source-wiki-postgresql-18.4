Oid

定義

  • 種別: typedef
  • 定義箇所: src/include/nodes/nodeFuncs.h:37

目的

PostgreSQLのシステムカタログ内のオブジェクト（テーブル、関数、型など）を一意に識別するための識別子です。

シグネチャまたは形状

typedef として定義されており、内部的には通常 unsigned int（32ビット符号なし整数）として扱われます。

入力と出力

  • 入力: 該当なし（型定義のため）
  • 出力: 該当なし

前提条件と不変条件

  • システムカタログ内の各オブジェクトに対して一意であることが期待されます。
  • 0 は「無効なオブジェクト」を表す予約値として使用されます。

アルゴリズムまたは振る舞い

本シンボル自体はデータ型であり、特定のアルゴリズムを持ちません。ただし、nodeFuncs.h 内の関数（exprType など）において、式ノードの評価結果や関数識別子を表現するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck 配下の各ファイル（verify_common.c, verify_gin.c, verify_heapam.c, verify_nbtree.c）において、リレーションIDやインデックスIDを保持する変数として広範に使用されています。
  • **呼び出し先**: 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • Node: nodeFuncs.h で操作対象となるツリー構造の基本単位。
  • FuncExpr, OpExpr: Oid を保持する具体的な式ノード。

失敗モードと境界条件

  • Oid が 0 の場合、無効なオブジェクト参照として扱われます。
  • システムカタログの更新に伴い、特定の Oid が再利用される可能性があるため、永続的な参照として保持する際は注意が必要です。

保守担当者向けメモ

  • 本シンボルは PostgreSQL 全体で共通して使用される基本型です。nodeFuncs.h 以外の場所でも定義・参照されている可能性があるため、変更時は影響範囲が非常に広大であることに留意してください。
