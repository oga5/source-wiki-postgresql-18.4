currtid_byrelname

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:417

目的

指定されたリレーション名と TID (Tuple Identifier) を基に、そのタプルの最新バージョンを検索して返す。

シグネチャまたは形状

Datum currtid_byrelname(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relname (text): リレーション名。
  • tid (ItemPointer): 検索対象の物理タプル位置。
  • **出力**:
  • result (ItemPointer): 最新の TID を指すポインタ。

前提条件と不変条件

  • relname は有効なリレーション名として解決可能であること。
  • table_openrv を用いて AccessShareLock を取得するため、対象リレーションに対する読み取り権限が必要。

アルゴリズムまたは振る舞い

1. textToQualifiedNameList を用いて relname を RangeVar に変換する。 2. table_openrv を呼び出し、AccessShareLock モードでリレーションを開く。 3. currtid_internal を呼び出し、指定された TID に対応する最新のタプルバージョンを取得する。 4. table_close を呼び出し、リレーションを閉じてロックを解放する。 5. 取得した最新の TID を PG_RETURN_ITEMPOINTER で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元情報は提供されていません。
  • **呼び出し先**:
  • makeRangeVarFromNameList
  • table_openrv
  • currtid_internal
  • table_close

関連データ構造

  • ItemPointer: タプルの物理位置（ブロック番号とオフセット番号）を保持する構造体。
  • RangeVar: リレーション名を特定するための構造体。
  • Relation: リレーション記述子。

失敗モードと境界条件

  • 指定されたリレーションが存在しない場合、table_openrv がエラーを発生させる。
  • currtid_internal が内部的に失敗した場合、そのエラーが伝播する。

保守担当者向けメモ

  • 本関数は currtid_internal をラップするインターフェースであり、リレーションのオープン・クローズのライフサイクル管理を担当している。
  • VACUUM FULL 等でタプルが移動した場合、物理的な TID は無効になる可能性がある点に注意すること。
