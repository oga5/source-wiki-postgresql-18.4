CachedExpression

定義

  • 種別: struct
  • 定義箇所: src/include/utils/plancache.h:187

目的

スカラー式の計画を低コストで再利用するためにキャッシュする軽量な構造体です。

シグネチャまたは形状

typedef struct CachedExpression
{
	int			magic;			/* CACHEDEXPR_MAGIC と一致する必要あり */
	Node	   *expr;			/* 計画化された式 */
	bool		is_valid;		/* 式が現在も有効か */
	List	   *relationOids;	/* 依存するリレーションの OID リスト */
	List	   *invalItems;		/* PlanInvalItems としてのその他の依存関係 */
	MemoryContext context;		/* この構造体を保持するメモリコンテキスト */
	dlist_node	node;			/* グローバルリストへのリンク */
} CachedExpression;

入力と出力

  • 入力: GetCachedExpression を介して式から構築されます。
  • 出力: キャッシュされた式（expr）およびその有効性フラグ（is_valid）を提供します。

前提条件と不変条件

  • magic フィールドは常に CACHEDEXPR_MAGIC を保持する必要があります。
  • is_valid が false の場合、式は再計画が必要です。
  • 構造体内の relationOids 以降のフィールドは plancache.c 内部でのみ使用されるプライベートなものとして扱われます。

アルゴリズムまたは振る舞い

1. **構築**: GetCachedExpression により、式が計画化され、本構造体に格納されます。 2. **管理**: dlist を使用してグローバルなリストで管理されます。 3. **無効化**: スキーマ変更等のイベント発生時、依存関係（relationOids, invalItems）に基づき is_valid が更新されます。 4. **解放**: FreeCachedExpression を呼び出すことで、関連するすべてのストレージが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/plancache.c 内の GetCachedExpression および FreeCachedExpression が主要な操作関数です。src/backend/utils/cache/typcache.c からの参照の可能性が示唆されています。
  • **呼び出し先**: 構造体内部で Node 型の式や List 型の依存関係を保持します。

関連データ構造

  • Node: 計画化された式本体。
  • List: 依存関係（relationOids, invalItems）を保持。
  • MemoryContext: メモリ管理に使用。
  • dlist_node: グローバルリスト管理用の侵入型リストノード。

失敗モードと境界条件

  • is_valid フラグのチェックを怠ると、無効化された古い計画を誤って使用するリスクがあります。
  • メモリコンテキストの管理が不適切な場合、メモリリークが発生する可能性があります。

保守担当者向けメモ

  • relationOids 以降のフィールドは plancache.c のプライベート領域であるため、外部モジュールからの直接操作は避けてください。
  • CACHEDEXPR_MAGIC を使用した整合性チェックが実装されているため、デバッグ時にはこの値を確認してください。
