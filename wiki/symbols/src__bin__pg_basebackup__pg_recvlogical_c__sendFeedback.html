sendFeedback

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_recvlogical.c:127

目的

論理レプリケーションストリームにおいて、現在の受信位置（Write LSN）およびフラッシュ位置（Flush LSN）を PostgreSQL サーバーへ通知し、レプリケーションスロットの進捗を維持する。

シグネチャまたは形状

static bool sendFeedback(PGconn *conn, TimestampTz now, bool force, bool replyRequested)

入力と出力

  • **入力**:
  • conn: サーバーとの接続を保持する PGconn ポインタ。
  • now: 現在のタイムスタンプ。
  • force: true の場合、前回の通知から位置に変更がなくても強制的に送信する。
  • replyRequested: サーバーからの応答を要求するかどうかを示すフラグ。
  • **出力**:
  • 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • グローバル変数 output_written_lsn および output_fsync_lsn が最新の受信・フラッシュ状態を保持していること。
  • force が false の場合、前回の送信時と比較して output_written_lsn または output_fsync_lsn に変化がある場合にのみ送信を行う。

アルゴリズムまたは振る舞い

1. force が偽かつ、前回の送信時と LSN に変化がない場合は、何もせず true を返す。 2. verbose モードが有効な場合、ログに現在の LSN を出力する。 3. r 型のフィードバックメッセージを構築する（Write LSN, Flush LSN, Apply LSN, 送信時刻, 応答要求フラグ）。 4. PQputCopyData を使用してメッセージをサーバーへ送信し、PQflush でバッファをフラッシュする。 5. 送信成功後、現在の LSN を静的変数 last_written_lsn および last_fsync_lsn に保存する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_recvlogical.c 内のメインループおよび終了処理（pg_recvlogical.c:319, 1053 等）。
  • src/bin/pg_basebackup/receivelog.c 内のストリーミング処理（receivelog.c:782, 795, 1035 等）。
  • **呼び出し先**:
  • libpq の PQputCopyData, PQflush, PQerrorMessage。
  • fe_sendint64（内部的なバイト変換）。

関連データ構造

  • PGconn: libpq の接続オブジェクト。
  • XLogRecPtr: LSN（ログシーケンス番号）を保持する型。
  • output_written_lsn, output_fsync_lsn: 本関数が参照するグローバルな進捗変数。

失敗モードと境界条件

  • PQputCopyData または PQflush がエラーを返した場合、pg_log_error でエラーメッセージを出力し、false を返す。
  • force が false の場合、進捗がないと判断されるとネットワーク通信をスキップする（wal_sender_timeout による切断を避けるための定期的な呼び出しが必要）。

保守担当者向けメモ

  • 本関数は COPY プロトコルを介してフィードバックを送信する。プロトコル仕様の変更時には replybuf のサイズとオフセット計算を慎重に確認すること。
  • receivelog.c と pg_recvlogical.c の両方から参照されているが、シグネチャが異なる場合があるため、修正時は両方の呼び出し元を確認すること。
