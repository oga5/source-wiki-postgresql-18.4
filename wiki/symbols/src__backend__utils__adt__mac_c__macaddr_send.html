macaddr_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac.c:160

目的

macaddr 型のデータを、PostgreSQLのフロントエンド/バックエンド通信プロトコルで使用されるバイナリ形式（bytea）に変換します。

シグネチャまたは形状

Datum macaddr_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して macaddr 構造体へのポインタを受け取ります。
  • **出力**: bytea 型の Datum を返します。

前提条件と不変条件

  • 入力された macaddr 構造体は有効な6バイトのMACアドレスを保持している必要があります。
  • pq_begintypsend および pq_endtypsend を使用した StringInfo バッファの管理が必須です。

アルゴリズムまたは振る舞い

1. pq_begintypsend を呼び出し、バイナリ送信用のバッファを初期化します。 2. macaddr 構造体の各メンバ（a から f までの6バイト）を、pq_sendbyte を用いて順次バッファに書き込みます。 3. pq_endtypsend を呼び出してバッファを bytea 形式に変換し、その結果を PG_RETURN_BYTEA_P で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（Fmgr）から、SQLレベルのデータ送信要求に応じて呼び出されます。
  • **呼び出し先**:
  • pq_begintypsend / pq_endtypsend（libpq/pqformat.h）
  • pq_sendbyte（libpq/pqformat.h）

関連データ構造

  • macaddr: 6バイトのMACアドレスを保持する構造体。
  • StringInfoData: バイナリデータ構築用の一時バッファ。

失敗モードと境界条件

  • メモリ不足が発生した場合、pq_begintypsend 等の内部関数がエラーをスローする可能性があります。
  • 入力ポインタが NULL の場合、PG_GETARG_MACADDR_P マクロの仕様に従いエラーとなります。

保守担当者向けメモ

  • 本関数は6バイトをそのままネットワークバイトオーダー（バイト単位の順序）で送信します。エンディアン変換は不要な構造です。
  • macaddr 構造体の定義変更（メンバの追加や順序変更）があった場合は、この送信順序と整合性を取る必要があります。
