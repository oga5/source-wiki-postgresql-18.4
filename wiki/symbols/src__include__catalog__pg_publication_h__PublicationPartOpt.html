PublicationPartOpt

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/pg_publication.h:157

目的

論理レプリケーションにおいて、パーティションテーブルをパブリケーションに追加または参照する際、その階層（ルート、リーフ、またはすべて）をどのように扱うかを指定するために使用されます。

シグネチャまたは形状

typedef enum PublicationPartOpt
{
    PUBLICATION_PART_ROOT,
    PUBLICATION_PART_LEAF,
    PUBLICATION_PART_ALL,
} PublicationPartOpt;

入力と出力

  • 入力: パブリケーション関連の関数（GetPublicationRelations 等）の引数として渡されます。
  • 出力: 指定されたオプションに基づき、対象となるリレーションのリストがフィルタリングされます。

前提条件と不変条件

  • PUBLICATION_PART_ROOT: パーティション階層のルートテーブルのみを対象とします。
  • PUBLICATION_PART_LEAF: パーティション階層のリーフ（末端）パーティションのみを対象とします。
  • PUBLICATION_PART_ALL: 階層全体を対象とします。

アルゴリズムまたは振る舞い

パブリケーションの構成情報を取得する際、この列挙型を使用して、対象テーブルのパーティション構造をどの範囲まで含めるかを制御します。

呼び出し元と呼び出し先

  • 呼び出し元（確認されたもの）:
  • src/backend/catalog/pg_publication.c: GetPubPartitionOptionRelations, GetPublicationRelations, GetSchemaPublicationRelations, GetAllSchemaPublicationRelations
  • 呼び出し先:
  • 本シンボル自体は列挙型定義であり、関数を呼び出すことはありません。

関連データ構造

  • PublicationRelInfo: パブリケーションとリレーションの関連情報を保持する構造体。
  • Publication: パブリケーションのメモリ内表現。

失敗モードと境界条件

  • 未定義の列挙値を渡した場合の動作は未定義です。
  • パーティション化されていないテーブルに対して使用した場合の挙動は、各呼び出し元関数の実装に依存します。

保守担当者向けメモ

  • 本シンボルは EXPOSE_TO_CLIENT_CODE ブロックに関連しており、クライアント側でも利用可能です。
  • パーティションテーブルのレプリケーションロジックを変更する際は、この列挙型の各値がどのように解釈されているか、pg_publication.c 内の各関数を確認してください。
