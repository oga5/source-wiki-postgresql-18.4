OperatorLookup

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_operator.c:163

目的

指定された名前と左右の型に基づいて演算子を検索し、その演算子が完全に定義されているか（実体を持つか）を判定する。

シグネチャまたは形状

Oid OperatorLookup(List *operatorName, Oid leftObjectId, Oid rightObjectId, bool *defined)

入力と出力

  • **入力**:
  • operatorName: 演算子名のリスト。
  • leftObjectId: 左オペランドの型OID（単項演算子の場合は InvalidOid）。
  • rightObjectId: 右オペランドの型OID（単項演算子の場合は InvalidOid）。
  • defined: 演算子の定義状態を格納する出力用ポインタ。
  • **出力**:
  • 演算子の Oid。見つからない場合は InvalidOid。

前提条件と不変条件

  • defined ポインタは有効なメモリ領域を指している必要がある。
  • 演算子が見つからない場合、defined には false が設定される。

アルゴリズムまたは振る舞い

1. LookupOperName を呼び出し、名前と型から演算子の OID を検索する。 2. 演算子が見つからない場合、defined に false を設定し InvalidOid を返す。 3. 演算子が見つかった場合、get_opcode を使用してその演算子に関連付けられた関数（oprcode）を取得する。 4. RegProcedureIsValid を用いて関数が有効かを確認し、その結果を defined に格納する。 5. 検索された演算子の OID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_operator.c (内部参照)
  • src/backend/commands/operatorcmds.c:412
  • **呼び出し先**:
  • LookupOperName
  • get_opcode
  • OidIsValid
  • RegProcedureIsValid

関連データ構造

  • List: 演算子名を保持するリスト構造。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • 演算子が存在しない場合、InvalidOid を返し、defined は false となる。
  • 演算子が存在しても、対応する関数が定義されていない（シェル演算子である）場合、defined は false となる。

保守担当者向けメモ

  • 本関数は、演算子が「シェル（プレースホルダー）」か「実体」かを区別するために使用される。
  • get_opcode が返す値が InvalidOid（または 0）である場合、その演算子は未定義のシェルとして扱われる。
