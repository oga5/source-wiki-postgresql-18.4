entryPreparePage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:489

目的

GINエントリーツリーのページに対して、挿入または削除操作を行うための準備処理を実行します。

シグネチャまたは形状

static void entryPreparePage(GinBtree btree, Page page, OffsetNumber off, GinBtreeEntryInsertData *insertData, BlockNumber updateblkno)

入力と出力

  • **入力**:
  • btree: GIN B-treeの制御構造体。
  • page: 操作対象のページ。
  • off: 操作対象のオフセット番号。
  • insertData: 挿入データおよび削除フラグを含む構造体。
  • updateblkno: 内部ノード更新時に使用するブロック番号（InvalidBlockNumber の場合は更新なし）。
  • **出力**: なし（副作用としてページ内容を直接変更）。

前提条件と不変条件

  • insertData->entry が存在すること。
  • page がデータページ（GinPageIsData）ではないこと。

アルゴリズムまたは振る舞い

1. insertData->isDelete が真の場合、リーフページであることを確認した上で、指定されたオフセット off のタプルを PageIndexTupleDelete で削除します。 2. ページがリーフノードではなく、かつ updateblkno が有効なブロック番号である場合、指定されたオフセット off のインデックスタプルを取得し、GinSetDownlink を用いてダウンリンク（子ページへのポインタ）を updateblkno に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginentrypage.c 内の複数箇所から呼び出されています。
  • **呼び出し先**:
  • PageIndexTupleDelete
  • PageGetItem
  • PageGetItemId
  • GinSetDownlink

関連データ構造

  • GinBtree: GINインデックスのB-tree操作用構造体。
  • GinBtreeEntryInsertData: エントリー挿入時のメタデータ。
  • IndexTuple: GINエントリーツリー内のインデックスタプル。

失敗モードと境界条件

  • insertData->isDelete が真の際に page がリーフページでない場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数はエントリーツリーの構造維持を目的としており、削除操作はリーフページのみに限定されています。
  • 内部ノードのダウンリンク更新は、ページ分割や再配置に伴うツリー構造の整合性維持のために不可欠です。
