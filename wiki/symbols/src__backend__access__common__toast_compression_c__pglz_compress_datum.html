pglz_compress_datum

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/toast_compression.c:39

目的

入力された varlena データを PGLZ アルゴリズムを用いて圧縮し、圧縮後のデータを格納した新しい varlena 構造体を返します。

シグネチャまたは形状

struct varlena *pglz_compress_datum(const struct varlena *value)

入力と出力

  • **入力**: value (圧縮対象の varlena 構造体へのポインタ)
  • **出力**: 圧縮されたデータを含む varlena 構造体へのポインタ。圧縮が不可能な場合や効率が悪い場合は NULL を返します。

前提条件と不変条件

  • 入力データのサイズが PGLZ_strategy_default で定義された最小サイズ未満、または最大サイズを超過している場合、圧縮は行われず NULL が返されます。
  • 圧縮後のデータは palloc によってメモリ確保されます。

アルゴリズムまたは振る舞い

1. 入力データのサイズ（ヘッダーを除く）を取得します。 2. サイズ制限チェックを行い、範囲外であれば NULL を返します。 3. PGLZ_MAX_OUTPUT に基づいて圧縮後の最大サイズを算出し、varlena ヘッダー分を加算してメモリを確保します。 4. pglz_compress を呼び出し、データを圧縮します。 5. 圧縮結果が負（失敗）であれば、確保したメモリを解放し NULL を返します。 6. 成功した場合、SET_VARSIZE_COMPRESSED マクロを使用して圧縮データのサイズとヘッダーを適切に設定し、ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/toast_internals.c:67
  • **呼び出し先**:
  • pglz_compress (PGLZ圧縮アルゴリズム本体)
  • palloc / pfree (メモリ管理)

関連データ構造

  • struct varlena: PostgreSQLの可変長データ型。
  • PGLZ_strategy_default: 圧縮戦略を定義するグローバル構造体。

失敗モードと境界条件

  • 入力サイズが圧縮戦略の許容範囲外である場合: NULL を返します。
  • pglz_compress が圧縮失敗（戻り値が負）を報告した場合: メモリを解放し NULL を返します。

保守担当者向けメモ

  • 本関数は PGLZ アルゴリズム専用です。LZ4など他のアルゴリズムについては別の関数（lz4_compress_datum 等）が対応します。
  • 圧縮後のサイズが元のデータより大きくなる場合や、圧縮アルゴリズムが効率的でないと判断した際の挙動については、pglz_compress の実装に依存します。
