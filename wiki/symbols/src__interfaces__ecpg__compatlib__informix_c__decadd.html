decadd

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/compatlib/informix.c:150

目的

Informix 互換の decimal 型同士の加算を行い、結果を格納する。

シグネチャまたは形状

int decadd(decimal *arg1, decimal *arg2, decimal *sum)

入力と出力

  • **入力**:
  • arg1: 加算対象の decimal 型ポインタ。
  • arg2: 加算対象の decimal 型ポインタ。
  • sum: 加算結果を格納する decimal 型ポインタ。
  • **出力**:
  • 成功時は 0 を返す。
  • エラー時は Informix 互換のエラーコード（ECPG_INFORMIX_NUM_OVERFLOW, ECPG_INFORMIX_NUM_UNDERFLOW）または -1 を返す。

前提条件と不変条件

  • errno は関数呼び出しの直前に 0 にリセットされる。
  • 内部的に deccall3 を介して PGTYPESnumeric_add を利用するため、decimal 型は numeric 型として処理可能な形式である必要がある。

アルゴリズムまたは振る舞い

1. errno を 0 に初期化する。 2. deccall3 を呼び出し、arg1 と arg2 の加算を PGTYPESnumeric_add に委譲して結果を sum に格納する。 3. 処理後の errno を確認する。

  • PGTYPES_NUM_OVERFLOW の場合: ECPG_INFORMIX_NUM_OVERFLOW を返す。
  • PGTYPES_NUM_UNDERFLOW の場合: ECPG_INFORMIX_NUM_UNDERFLOW を返す。
  • その他の非ゼロ値の場合: -1 を返す。
  • 正常終了時: 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/test/expected/compat_informix-dec_test.c
  • src/interfaces/ecpg/test/expected/compat_informix-test_informix.c
  • **呼び出し先**:
  • deccall3 (内部ラッパー)
  • PGTYPESnumeric_add (PostgreSQL pgtypes ライブラリ)

関連データ構造

  • decimal: Informix 互換の数値型。
  • sqlca_t: Informix 互換の SQL 通信領域（エラーコードの伝播に関連）。

失敗モードと境界条件

  • 数値のオーバーフローまたはアンダーフローが発生した場合、それぞれ対応する Informix 互換エラーコードを返す。
  • PGTYPES ライブラリ側でその他のエラーが発生した場合は -1 を返す。

保守担当者向けメモ

  • 本関数は deccall3 を利用して decimal 型と numeric 型の変換を隠蔽している。計算ロジックの変更が必要な場合は、src/interfaces/ecpg/pgtypeslib/ 側の PGTYPESnumeric_add 実装を確認すること。
