pg_checksum_parse_type

定義

  • 種別: function
  • 定義箇所: src/common/checksum_helper.c:27

目的

チェックサムアルゴリズムを表す文字列を、内部で使用される pg_checksum_type 列挙型へ変換します。設定ファイルやコマンドライン引数からの入力を検証・正規化するために使用されます。

シグネチャまたは形状

bool pg_checksum_parse_type(char *name, pg_checksum_type *type);

入力と出力

  • **入力**:
  • name: 変換対象のアルゴリズム名（文字列）。
  • type: 変換結果を格納する pg_checksum_type 型へのポインタ。
  • **出力**:
  • 戻り値: 変換に成功した場合は true、一致するアルゴリズムが見つからない場合は false を返します。
  • type: 成功時は対応する列挙値が、失敗時は CHECKSUM_TYPE_NONE が書き込まれます。

前提条件と不変条件

  • name は大文字小文字を区別せずに比較されます（pg_strcasecmp を使用）。
  • 変換が失敗した場合、type には CHECKSUM_TYPE_NONE が設定されます。

アルゴリズムまたは振る舞い

1. name を "none", "crc32c", "sha224", "sha256", "sha384", "sha512" の各文字列と比較します。 2. 一致するものがあれば、対応する pg_checksum_type 列挙値を result_type に代入します。 3. いずれにも一致しない場合、result を false に設定します。 4. 最終的に *type へ result_type を格納し、result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/bin/pg_combinebackup/pg_combinebackup.c
  • src/common/parse_manifest.c
  • **呼び出し先**:
  • pg_strcasecmp（標準的な文字列比較関数）

関連データ構造

  • pg_checksum_type: チェックサムアルゴリズムを識別するための列挙型。

失敗モードと境界条件

  • 未知のアルゴリズム名が渡された場合、関数は false を返し、type には CHECKSUM_TYPE_NONE が設定されます。呼び出し元はこの戻り値をチェックしてエラーハンドリングを行う必要があります。

保守担当者向けメモ

  • 新しいチェックサムアルゴリズムを追加する場合は、本関数の if-else ブロックに新しい条件を追加してください。
  • 大文字小文字を区別しない比較を行っているため、将来的にアルゴリズム名が増えた場合でも既存の命名規則との互換性が維持されます。
