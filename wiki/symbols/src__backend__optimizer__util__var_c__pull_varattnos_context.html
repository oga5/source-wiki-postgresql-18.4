pull_varattnos_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/var.c:40

目的

式ツリーを走査する際、特定の varno（リレーション番号）に対応する属性番号（varattno）を収集するための状態保持用コンテキスト。

シグネチャまたは形状

typedef struct
{
	Bitmapset  *varattnos;
	Index		varno;
} pull_varattnos_context;

入力と出力

  • varattnos: 収集された属性番号を格納する Bitmapset へのポインタ。
  • varno: 走査対象とするリレーションのインデックス。

前提条件と不変条件

  • varattnos は、走査開始前に呼び出し元で初期化（または空のセットとして準備）されている必要があります。
  • varno は、対象とするリレーションを正しく識別するために設定されている必要があります。

アルゴリズムまたは振る舞い

  • pull_varattnos_walker 関数によって使用され、式ツリー内の Var ノードを探索します。
  • 現在の Var ノードの varno がコンテキスト内の varno と一致する場合、その varattno を varattnos ビットマップセットに追加します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/util/var.c 内の pull_varattnos 関数がこの構造体をインスタンス化し、走査を開始します。
  • 呼び出し先: pull_varattnos_walker 関数がこのコンテキストを引数として受け取り、再帰的にツリーを走査します。

関連データ構造

  • Bitmapset: 属性番号の集合を効率的に管理するために使用されます。
  • Var: 式ツリー内の変数ノード。

失敗モードと境界条件

  • varattnos が NULL の状態で走査が行われた場合、メモリ保護違反が発生する可能性があります。
  • varno が 0（システム属性や特殊なケース）の場合の挙動は、pull_varattnos_walker の実装ロジックに依存します。

保守担当者向けメモ

  • 本構造体は pull_varattnos の内部処理専用です。
  • varno の比較は、サブクエリのレベル（sublevels_up）を考慮した走査ロジックと密接に関連しているため、変更時は walker 側のロジックと整合性を確認してください。
