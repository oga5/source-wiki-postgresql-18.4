CheckPAMAuth

定義

  • **種別**: function
  • **定義箇所**: src/backend/libpq/auth.c:2028

目的

PAM (Pluggable Authentication Modules) を使用して、PostgreSQL 接続ユーザーの認証およびアカウント管理を行う。

シグネチャまたは形状

static int CheckPAMAuth(Port *port, const char *user, const char *password)

入力と出力

  • **入力**:
  • port: 接続情報を保持する Port 構造体。
  • user: 認証対象のユーザー名。
  • password: 認証に使用するパスワード文字列。
  • **出力**:
  • STATUS_OK: 認証成功。
  • STATUS_ERROR: 認証失敗またはシステムエラー。
  • STATUS_EOF: 認証プロセス中に EOF を検知した場合。

前提条件と不変条件

  • pam_passwd 等の静的変数を使用してパスワードやポート情報を保持する（PAM の制限回避のため）。
  • 認証終了時には必ず pam_passwd を NULL にリセットする。
  • 認証成功時には set_authn_id を呼び出し、セッションの認証済み ID を確定させる。

アルゴリズムまたは振る舞い

1. **初期化**: 静的変数にパスワードとポート情報を退避し、pam_start で PAM ハンドルを初期化する。サービス名は pg_hba.conf の設定、またはデフォルトの PGSQL_PAM_SERVICE を使用。 2. **PAM アイテム設定**: PAM_USER を設定。ローカル接続以外の場合は PAM_RHOST にクライアントのホスト情報を設定。 3. **会話設定**: PAM_CONV に pam_passw_conv を設定。 4. **認証実行**: pam_authenticate で認証を行い、続いて pam_acct_mgmt でアカウントの有効性を確認する。 5. **終了処理**: pam_end を呼び出し、成功時は set_authn_id を実行して STATUS_OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c 内の認証ディスパッチャー（ClientAuthentication 等）。
  • **呼び出し先**: pam_start, pam_set_item, pam_authenticate, pam_acct_mgmt, pam_end, pg_getnameinfo_all, set_authn_id。

関連データ構造

  • Port: 接続状態と HBA 設定（pamservice, pam_use_hostname 等）を保持。
  • pam_handle_t: PAM セッション状態を管理する構造体。

失敗モードと境界条件

  • pam_start や pam_set_item 等の PAM API が PAM_SUCCESS 以外を返した場合、ereport(LOG, ...) でエラーを記録し STATUS_ERROR を返す。
  • 認証失敗時、pam_no_password フラグが立っている場合（EOF 検知時）はログを出力せず STATUS_EOF を返す。
  • pg_getnameinfo_all が失敗した場合は STATUS_ERROR を返す。

保守担当者向けメモ

  • PAM の appdata_ptr が一部環境で正しく動作しないため、静的変数による回避策が取られている点に注意すること。
  • 認証失敗時の詳細な理由はセキュリティ上の理由からクライアントには返さない設計となっている。ログ出力には pam_strerror を活用すること。
