ExecHashJoinImpl

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHashjoin.c:220

目的

ハイブリッド・ハッシュ結合アルゴリズムの実行本体。シリアル実行および並列実行の両方において、ハッシュテーブルの構築からタプルの結合処理までをステートマシンとして駆動する。

シグネチャまたは形状

static pg_attribute_always_inline TupleTableSlot *
ExecHashJoinImpl(PlanState *pstate, bool parallel)

入力と出力

  • **入力**:
  • pstate: 実行対象の HashJoinState を含む PlanState。
  • parallel: 並列実行モードであるかを示すフラグ。
  • **出力**:
  • 結合された TupleTableSlot。処理が完了または中断した場合は NULL を返す。

前提条件と不変条件

  • pstate は HashJoinState にキャスト可能であること。
  • always_inline 指定されており、コンパイラによる最適化を前提としている。
  • 並列実行時は、バリア同期（Barrier）のフェーズ管理が正しく行われていること。

アルゴリズムまたは振る舞い

1. **初期化**: ResetExprContext を呼び出し、前回のタプル処理で確保されたメモリを解放する。 2. **ステートマシン駆動**: node->hj_JoinState に基づきループ処理を行う。

  • HJ_BUILD_HASHTABLE: ハッシュテーブルを構築する。
  • 並列実行時は、最初の参加者がテーブルを作成し、後続はアタッチする。
  • 外部リレーションが空の場合の最適化（シリアル時のみ）や、バッチ分割の準備を行う。
  • CHECK_FOR_INTERRUPTS(): ループの各反復で割り込みチェックを行い、応答性を確保する。

    • 3. **並列同期**: 並列実行時は Barrier を使用して、各フェーズ（PHJ_BUILD_HASH_OUTER 等）で全バックエンドの足並みを揃える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ExecHashJoin (シリアル実行用エントリーポイント)
  • ExecParallelHashJoin (並列実行用エントリーポイント)
  • **呼び出し先**:
  • ExecHashTableCreate: ハッシュテーブルの生成。
  • MultiExecProcNode: ハッシュテーブル構築のための内側リレーション実行。
  • ExecProcNode: 外部リレーションからのタプル取得。
  • BarrierArriveAndWait: 並列実行時の同期。
  • ExecParallelHashJoinPartitionOuter: 並列時の外部リレーションパーティション分割。

関連データ構造

  • HashJoinState: 結合ノードの状態管理。
  • HashJoinTable: ハッシュテーブルの実体。
  • ParallelHashJoinState: 並列実行時の共有状態。
  • Barrier: 並列処理の同期用構造体。

失敗モードと境界条件

  • **メモリ不足**: バッチ分割が必要な場合、nbatch を増加させディスク退避を行う。
  • **空のリレーション**: 外部リレーションが空の場合、結合をスキップして早期終了する（シリアル実行時）。
  • **並列実行の制約**: 並列版では、バリア待機中にタプルを返さないという制約がある。

保守担当者向けメモ

  • 本関数は always_inline されているため、変更時はシリアル版と並列版の両方のコードパスに影響を与える点に注意すること。
  • ステートマシンの遷移（hj_JoinState）が複雑であるため、デバッグ時はこの遷移を追跡することが推奨される。
