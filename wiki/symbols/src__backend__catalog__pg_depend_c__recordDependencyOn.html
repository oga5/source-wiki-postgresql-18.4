recordDependencyOn

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:46

目的

2つのデータベースオブジェクト間の依存関係をシステムカタログ pg_depend に永続化します。

シグネチャまたは形状

void recordDependencyOn(const ObjectAddress *depender,
                        const ObjectAddress *referenced,
                        DependencyType behavior)

入力と出力

  • **入力**:
  • depender: 依存元となるオブジェクトのアドレス。
  • referenced: 依存先（参照先）となるオブジェクトのアドレス。
  • behavior: 依存関係の性質を示す DependencyType（例: DEPENDENCY_NORMAL, DEPENDENCY_AUTO, DEPENDENCY_INTERNAL）。
  • **出力**: なし（void）。

前提条件と不変条件

  • pg_depend カタログに対する書き込み権限が必要です。
  • 依存関係の記録対象がシステムによって「固定（Pinned）」されている場合、記録は行われません（isObjectPinned による判定が内部的に関与します）。

アルゴリズムまたは振る舞い

1. 単一の依存関係を記録するためのラッパー関数として機能します。 2. 内部的に recordMultipleDependencies を呼び出し、引数として 1（単一レコード）を指定して処理を委譲します。 3. recordMultipleDependencies は pg_depend を RowExclusiveLock でロックし、カタログへの挿入を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/aclchk.c

  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/catalog/pg_attrdef.c
  • src/backend/catalog/pg_collation.c
  • **呼び出し先**:

関連データ構造

  • ObjectAddress: データベースオブジェクトを一意に識別するための構造体。
  • DependencyType: 依存関係の種類を定義する列挙型（catalog/dependency.h で定義）。
  • pg_depend: 依存関係を保持するシステムカタログテーブル。

失敗モードと境界条件

  • 依存先がシステムによって固定（Pinned）されている場合、レコードは作成されません。
  • カタログ操作に伴うロック競合や、トランザクションの制約により失敗する可能性があります。

保守担当者向けメモ

  • 本関数は単一の依存関係追加用です。複数の依存関係を一度に記録する場合は、オーバーヘッドを抑えるために recordMultipleDependencies を直接使用することを検討してください。
  • 依存関係の削除ロジック（deleteDependencyRecordsFor 等）との整合性に注意してください。
