test_thread

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/thread-thread.c:132

目的

ECPGのマルチスレッド環境におけるスレッドセーフ性を検証するため、各スレッドで個別のデータベース接続を確立し、共有テーブルへの並行挿入を実行する。

シグネチャまたは形状

void *test_thread(void *arg)

  • arg: スレッド識別子として使用される long 型の数値（intptr_t にキャストして利用）。

入力と出力

  • 入力: arg（スレッド番号）。
  • 出力: void *（常に NULL を返す）。

前提条件と不変条件

  • 呼び出し前に、共有テーブル test_thread がデータベース上に存在している必要がある。
  • グローバル変数 iterations が定義されており、各スレッドが実行する挿入回数を規定している。

アルゴリズムまたは振る舞い

1. **接続名生成**: threadnum を用いて "thread_%03ld" 形式のユニークな接続名を生成する。 2. **接続確立**: ECPGconnect を呼び出し、ecpg1_regression データベースへ接続する。接続失敗時はエラーメッセージを出力して終了する。 3. **トランザクション開始**: ECPGtrans で begin を実行する。 4. **データ挿入**: iterations 回のループ内で INSERT INTO test_thread 文を実行する。各挿入にはスレッド固有の接続名と反復回数を使用する。 5. **終了処理**: ECPGtrans で commit を実行し、ECPGdisconnect で接続を閉じる。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（pthread_create または CreateThread を介して呼び出される）。
  • 呼び出し先: ECPGconnect, ECPGtrans, ECPGdo, ECPGdisconnect（ECPGランタイムライブラリ関数）。

関連データ構造

  • sqlca: ECPGのエラー状態を保持する構造体。
  • l_connection: 接続名を格納する char 配列。

失敗モードと境界条件

  • データベース接続失敗: sqlca.sqlcode が 0 以外の場合、エラーメッセージを出力して処理を中断する。
  • SQL実行失敗: INSERT 文の実行時に sqlca.sqlcode < 0 となった場合、sqlprint() が呼び出され、エラーが報告される。

保守担当者向けメモ

  • 本関数はECPGプリプロセッサによって生成されたコードであり、直接編集する際は .pgc ソースを確認すること。
  • スレッドごとの接続分離が正しく機能しているかを検証する重要なテストケースであるため、接続名の競合が発生しないよう注意が必要。
