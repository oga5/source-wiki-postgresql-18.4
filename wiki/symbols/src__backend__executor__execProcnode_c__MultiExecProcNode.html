MultiExecProcNode

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execProcnode.c:506

目的

タプルを直接返さない特殊な実行ノード（ハッシュテーブル構築やビットマップ生成など）を処理するためのディスパッチャです。

シグネチャまたは形状

Node *MultiExecProcNode(PlanState *node)

入力と出力

  • **入力**: PlanState *node - 実行対象のプランノード状態。
  • **出力**: Node * - ノードの実行結果（例: TIDBitmap やハッシュテーブル関連の構造体）。

前提条件と不変条件

  • check_stack_depth() によりスタック深さの安全性が確認されます。
  • CHECK_FOR_INTERRUPTS() により割り込み要求の確認が行われます。
  • ノードの chgParam が非 NULL の場合、処理前に ExecReScan が呼び出されます。

アルゴリズムまたは振る舞い

1. スタック深さと割り込みのチェックを行います。 2. ノードの状態変更（chgParam）があれば ExecReScan を実行します。 3. nodeTag(node) に基づいて以下のいずれかの処理へ分岐します。

  • T_HashState: MultiExecHash を呼び出す。
  • T_BitmapIndexScanState: MultiExecBitmapIndexScan を呼び出す。
  • T_BitmapAndState: MultiExecBitmapAnd を呼び出す。
  • T_BitmapOrState: MultiExecBitmapOr を呼び出す。

    • 4. 上記以外のノードタイプが渡された場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeBitmapAnd.c
  • src/backend/executor/nodeBitmapHeapscan.c
  • src/backend/executor/nodeBitmapOr.c
  • src/backend/executor/nodeHashjoin.c
  • **呼び出し先**:
  • ExecReScan
  • MultiExecHash
  • MultiExecBitmapIndexScan
  • MultiExecBitmapAnd
  • MultiExecBitmapOr

関連データ構造

  • PlanState: 実行ノードの共通状態構造体。
  • TIDBitmap: ビットマップスキャン等で返される結果構造体。

失敗モードと境界条件

  • 未対応のノードタイプが渡された場合、elog(ERROR) により実行が停止します。
  • スタックオーバーフローが発生する深さのプランツリーが渡された場合、check_stack_depth() によりエラーとなります。

保守担当者向けメモ

  • 本関数は「マルチ実行」をサポートする特定のノードタイプのみを対象としています。新しいノードタイプでこの機能が必要な場合は、switch 文にケースを追加してください。
