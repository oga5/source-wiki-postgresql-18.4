heap_prune_record_unused

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/pruneheap.c:1296

目的

ヒープページ内の特定のタプルオフセットを「未使用（unused）」としてマークし、剪定処理の状態管理構造体（PruneState）に記録する。

シグネチャまたは形状

static void heap_prune_record_unused(PruneState *prstate, OffsetNumber offnum, bool was_normal)

入力と出力

  • prstate: 剪定処理の状態を保持する PruneState 構造体へのポインタ。
  • offnum: 未使用として記録するタプルのオフセット番号。
  • was_normal: 記録対象が通常のタプルであったかどうかを示すフラグ。
  • 出力: なし（prstate の内部状態を更新する）。

前提条件と不変条件

  • prstate->processed[offnum] が false であること（二重記録の防止）。
  • prstate->nunused が MaxHeapTuplesPerPage 未満であること。

アルゴリズムまたは振る舞い

1. 指定された offnum を processed 配列で処理済みとしてマークする。 2. prstate->nowunused 配列に offnum を追加し、nunused カウンタをインクリメントする。 3. was_normal が true の場合、削除されたタプル数（ndeleted）をインクリメントする。リダイレクトタプルの場合はカウントしない。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/pruneheap.c 内の複数箇所（heap_prune_chain 等の剪定ロジックから呼び出される）。
  • 呼び出し先: なし。

関連データ構造

  • PruneState: 剪定処理中のオフセット管理や統計情報を保持する構造体。

失敗モードと境界条件

  • nunused が MaxHeapTuplesPerPage を超える場合、アサーションにより停止する。

保守担当者向けメモ

  • 本関数は、HOTチェーンの剪定やデッドタプルの整理において、物理的なページ変更を行う前の準備段階として使用される。
  • was_normal フラグは、統計情報（ndeleted）の正確性を維持するために重要である。
