toast_decompress_datum_slice

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/detoast.c:502

目的

圧縮された varlena データの先頭から指定された長さ（slicelength）分だけを効率的に展開して取得します。

シグネチャまたは形状

static struct varlena *toast_decompress_datum_slice(struct varlena *attr, int32 slicelength)

入力と出力

  • **入力**:
  • attr: 圧縮されたデータを含む varlena 構造体へのポインタ。
  • slicelength: 展開を希望するデータの長さ（バイト単位）。
  • **出力**:
  • 展開されたデータを含む varlena 構造体へのポインタ。

前提条件と不変条件

  • 入力 attr は VARATT_IS_COMPRESSED(attr) が真であること（アサーションにより強制）。
  • 呼び出し元は、返されたメモリ領域を適切に解放する責任を負います。

アルゴリズムまたは振る舞い

1. **サイズ判定**: 指定された slicelength がデータの本来の展開後サイズ（TOAST_COMPRESS_EXTSIZE(attr)）以上である場合、toast_decompress_datum を呼び出して全データを展開します。これは、過剰なメモリ割り当てを避け、一部のライブラリ（liblz4等）の誤動作を防ぐためです。 2. **メソッド判定**: TOAST_COMPRESS_METHOD(attr) を使用して圧縮アルゴリズムを特定します。 3. **展開実行**:

  • TOAST_PGLZ_COMPRESSION_ID: pglz_decompress_datum_slice を実行。
  • TOAST_LZ4_COMPRESSION_ID: lz4_decompress_datum_slice を実行。
  • 上記以外: elog(ERROR) によりエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/detoast.c 内の関数（例: detoast_attr_slice 等からの呼び出しが確認されています）。
  • **呼び出し先**:
  • toast_decompress_datum
  • pglz_decompress_datum_slice
  • lz4_decompress_datum_slice

関連データ構造

  • varlena: PostgreSQLの可変長データ型。
  • ToastCompressionId: 圧縮メソッドを識別するID。

失敗モードと境界条件

  • **無効な圧縮ID**: 未知の圧縮メソッドIDが指定された場合、elog(ERROR) で処理を中断します。
  • **スライス長**: slicelength が展開後のサイズを超える場合、全展開に切り替えることで安全性を確保しています。

保守担当者向けメモ

  • 本関数は圧縮データの「先頭からのスライス」を前提としています。圧縮アルゴリズムの特性上、データの中間位置からの展開はサポートされていません。
  • 新しい圧縮アルゴリズムを追加する場合は、switch 文に新しいケースを追加する必要があります。
