incompatible_module_error

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/dfmgr.c:315

目的

ロードしようとした共有ライブラリの ABI（Application Binary Interface）が、現在実行中の PostgreSQL サーバーと互換性がない場合に、詳細なエラー情報を生成して報告する。

シグネチャまたは形状

static void incompatible_module_error(const char *libname, const Pg_abi_values *module_magic_data);

入力と出力

  • **入力**:
  • libname: 互換性エラーが発生したライブラリのファイル名。
  • module_magic_data: ライブラリから読み取られた Pg_abi_values 構造体（ABI 互換性情報）。
  • **出力**: なし（ereport(ERROR, ...) により例外をスローし、処理を中断する）。

前提条件と不変条件

  • サーバー側の期待値はグローバル変数 magic_data に保持されている。
  • module_magic_data は、ロードされたライブラリ内の PG_MODULE_MAGIC マクロによって定義されたデータ構造を指している必要がある。

アルゴリズムまたは振る舞い

1. **バージョンチェック**: magic_data.version と module_magic_data->version を比較。不一致の場合はバージョンエラーを報告。 2. **ABI 文字列チェック**: abi_extra フィールドを比較。不一致の場合は ABI ミスマッチを報告。 3. **詳細フィールドの比較**: 以下の各フィールドを個別に比較し、不一致がある場合は StringInfo に詳細を蓄積する。

  • funcmaxargs
  • indexmaxkeys
  • namedatalen
  • float8byval

    • 4. **エラー報告**: 蓄積された詳細情報と共に ereport(ERROR) を実行する。不一致箇所が特定できない場合は、マジックブロックの長さやパディングの差異として報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/fmgr/dfmgr.c 内のライブラリロード処理（internal_load_library 等）から呼び出される。
  • **呼び出し先**: ereport（エラー報告）、initStringInfo / appendStringInfo（文字列構築）、strcmp（文字列比較）等。

関連データ構造

  • Pg_abi_values: ライブラリの ABI 互換性を定義する構造体。
  • StringInfoData: エラー詳細メッセージを構築するための一時バッファ。

失敗モードと境界条件

  • **バージョン不一致**: サーバーとライブラリのメジャー/マイナーバージョンが異なる場合に発生。
  • **ABI ミスマッチ**: コンパイル時の設定（abi_extra）がサーバーと一致しない場合に発生。
  • **フィールド不一致**: FUNC_MAX_ARGS や NAMEDATALEN などの定数がサーバーとライブラリで異なる場合に発生。

保守担当者向けメモ

  • マジックブロックのフィールド構成が変更された場合、本関数の比較ロジックも更新する必要がある（ソース内の XXX コメント参照）。
  • 本関数は ereport(ERROR) を呼び出すため、呼び出し元はこれ以降の処理が実行されないことを前提とする必要がある。
