AlterObjectNamespace_internal

定義

  • 種別: function
  • 定義箇所: src/backend/commands/alter.c:691

目的

データベースオブジェクトのスキーマ（名前空間）を変更する汎用的な内部処理を実行します。

シグネチャまたは形状

static Oid AlterObjectNamespace_internal(Relation rel, Oid objid, Oid nspOid)

入力と出力

  • **入力**:
  • rel: 対象オブジェクトを含むカタログのリレーション。
  • objid: 変更対象オブジェクトのOID。
  • nspOid: 移動先のスキーマOID。
  • **出力**:
  • 変更前のスキーマOIDを返します。

前提条件と不変条件

  • rel は有効なシステムカタログである必要があります。
  • 呼び出し元で適切なロックが取得されていることが想定されます。
  • オブジェクトが既に移動先スキーマにある場合、何も変更せずフックのみ実行して終了します。

アルゴリズムまたは振る舞い

1. システムキャッシュから対象オブジェクトのタプルをコピー取得します。 2. CheckSetNamespace を呼び出し、スキーマ移動の妥当性を検証します。 3. 権限チェックを実行します（スーパーユーザーでない場合、オブジェクトの所有権および移動先スキーマへの CREATE 権限を確認）。 4. オブジェクト型（関数、照合順序、演算子クラス/ファミリー）に応じて、移動先での名前重複チェックを行います。 5. heap_modify_tuple を使用してタプルのスキーマ属性を更新し、CatalogTupleUpdate でカタログへ書き込みます。 6. changeDependencyFor を呼び出し、依存関係情報を新しいスキーマへ更新します。 7. InvokeObjectPostAlterHook を呼び出し、後処理（イベントトリガー等）を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/alter.c 内の ExecAlterObjectSchemaStmt 等から呼び出されます。
  • **呼び出し先**: get_object_catcache_oid, get_object_attnum_..., SearchSysCacheCopy1, CheckSetNamespace, superuser, has_privs_of_role, object_aclcheck, heap_modify_tuple, CatalogTupleUpdate, changeDependencyFor, InvokeObjectPostAlterHook 等。

関連データ構造

  • Relation: カタログリレーション。
  • HeapTuple: システムカタログのタプル。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • オブジェクトがキャッシュに見つからない場合、ERROR を発生させます。
  • 所有者情報を持たないオブジェクトのスキーマ変更を試みた場合、スーパーユーザー権限がないと ERROR となります。
  • 権限不足や名前の重複がある場合、aclcheck_error や report_namespace_conflict 等により ERROR を発生させます。
  • 依存関係の更新に失敗した場合、ERROR を発生させます。

保守担当者向けメモ

  • 本関数はカタログ属性番号を動的に取得するため、新しいオブジェクト型に対応させる際は get_object_attnum_... 系関数の実装が必須です。
  • 物理的なストレージを伴う複雑なオブジェクト（テーブル等）の移動には使用できません。
