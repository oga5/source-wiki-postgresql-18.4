SpGistSetLastUsedPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgutils.c:672

目的

SP-GiSTインデックスの挿入操作において、次に利用可能な空き領域を持つページを効率的に検索できるよう、指定されたバッファの情報をインデックスキャッシュ（SpGistCache）に記録する。

シグネチャまたは形状

void SpGistSetLastUsedPage(Relation index, Buffer buffer)

入力と出力

  • **入力**:
  • index: 対象のインデックスリレーション。
  • buffer: 情報をキャッシュに登録する対象のバッファ。
  • **出力**: なし（副作用としてキャッシュ内の lastUsedPages を更新）。

前提条件と不変条件

  • ルートページなどの固定ページ（SpGistBlockIsFixed が真）はキャッシュ対象外であり、即座にリターンする。
  • キャッシュへの登録は、ページの種類（Leaf/Inner）およびNull保持の有無に応じたフラグに基づいて行われる。

アルゴリズムまたは振る舞い

1. spgGetCache を介してインデックスのキャッシュ情報を取得する。 2. 対象バッファのページ種別（LeafかInnerか）とNull保持フラグを判定し、対応するキャッシュエントリ（lup）を特定する。 3. PageGetExactFreeSpace を用いて、当該ページの現在の空き容量を取得する。 4. 以下のいずれかの条件を満たす場合、キャッシュ内の情報を更新する：

  • キャッシュ内のブロック番号が未設定（InvalidBlockNumber）。
  • キャッシュ内のブロック番号が現在のバッファと一致する。
  • 現在のページの空き容量が、キャッシュされているページの空き容量よりも大きい。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgdoinsert.c（複数箇所）
  • src/backend/access/spgist/spgvacuum.c
  • **呼び出し先**:
  • spgGetCache
  • BufferGetPage
  • BufferGetBlockNumber
  • SpGistBlockIsFixed
  • SpGistPageIsLeaf
  • SpGistPageStoresNulls
  • GET_LUP
  • PageGetExactFreeSpace

関連データ構造

  • SpGistCache: インデックスの実行時状態を保持する構造体。
  • SpGistLastUsedPage: キャッシュ内で最後に使用されたページのブロック番号と空き容量を管理する構造体。

失敗モードと境界条件

  • 固定ページ（ルートページ等）が渡された場合、キャッシュ更新は行われず正常終了する。
  • キャッシュエントリが未設定の状態であれば、無条件で現在のバッファ情報が登録される。

保守担当者向けメモ

  • 本関数はインデックス挿入のパフォーマンスに直結する。lastUsedPages の更新ロジックを変更する場合は、ページ選択の効率性に与える影響を考慮すること。
  • GBUF_ 系フラグの定義や SpGistLastUsedPage の構造に変更を加える際は、spgutils.c 内の関連するキャッシュ管理関数との整合性に注意が必要。
