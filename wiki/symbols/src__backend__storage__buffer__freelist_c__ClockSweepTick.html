ClockSweepTick

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/freelist.c:107

目的

バッファ置換アルゴリズム（クロック掃引法）において、次に再利用候補として検討すべきバッファのインデックスを特定し、共有メモリ上の管理状態を更新する。

シグネチャまたは形状

static inline uint32 ClockSweepTick(void)

入力と出力

  • **入力**: なし（共有メモリ上の StrategyControl を参照）
  • **出力**: 次の候補となるバッファのインデックス（uint32 型）

前提条件と不変条件

  • StrategyControl が初期化されていること。
  • NBuffers（バッファ総数）が正しく定義されていること。
  • StrategySyncStart() が一貫した統計情報を取得できるよう、ラップアラウンド発生時には buffer_strategy_lock を用いて completePasses と nextVictimBuffer の整合性が保証されること。

アルゴリズムまたは振る舞い

1. pg_atomic_fetch_add_u32 を使用し、nextVictimBuffer をアトミックにインクリメントして現在の候補インデックスを取得する。 2. 取得した値が NBuffers 以上の場合、剰余演算によりインデックスをラップアラウンドさせる。 3. ラップアラウンドが発生した（インデックスが 0 に戻った）場合、buffer_strategy_lock を取得し、completePasses をインクリメントする。この際、pg_atomic_compare_exchange_u32 を用いて nextVictimBuffer の更新と同期をとる。 4. 最終的なバッファインデックスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/freelist.c 内の StrategyGetBuffer() から呼び出されることが確認されている。
  • **呼び出し先**:
  • pg_atomic_fetch_add_u32
  • pg_atomic_compare_exchange_u32
  • SpinLockAcquire / SpinLockRelease

関連データ構造

  • BufferStrategyControl: 共有メモリ上のグローバルなバッファ戦略状態を保持する構造体。nextVictimBuffer および completePasses を含む。

失敗モードと境界条件

  • **ラップアラウンド**: nextVictimBuffer が NBuffers を超える際、正しく 0 にリセットされ、completePasses が適切にカウントアップされる。
  • **競合**: 複数のプロセスが同時に呼び出した場合、アトミック操作によりインデックスがわずかに順序通りにならない可能性があるが、アルゴリズム上許容される。

保守担当者向けメモ

  • 本関数は StrategySyncStart() が一貫した統計情報を読み取れるよう、スピンロックの取得タイミングが厳密に設計されている。修正を加える際は、completePasses と nextVictimBuffer の整合性が壊れないよう注意すること。
