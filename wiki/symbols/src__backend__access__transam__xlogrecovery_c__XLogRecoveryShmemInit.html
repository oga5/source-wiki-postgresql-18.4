XLogRecoveryShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogrecovery.c:464

目的

リカバリ処理に必要な共有メモリ領域（XLogRecoveryCtl）を確保し、初期化する。

シグネチャまたは形状

void XLogRecoveryShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の XLogRecoveryCtl が初期化される）

前提条件と不変条件

  • 共有メモリの初期化フェーズで呼び出される必要がある。
  • XLogRecoveryShmemSize() が返すサイズ分の共有メモリが確保可能であること。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、"XLOG Recovery Ctl" という名前で共有メモリ領域を確保する。 2. 既に領域が確保されている場合（found が真）、初期化済みとみなして即座に終了する。 3. 初回実行の場合、XLogRecoveryCtl 構造体をゼロクリアする。 4. 構造体内の同期プリミティブを初期化する:

  • info_lck（スピンロック）の初期化。
  • recoveryWakeupLatch（共有ラッチ）の初期化。
  • recoveryNotPausedCV（条件変数）の初期化。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:290（CreateSharedMemoryAndSemaphores 内から呼び出される）
  • 呼び出し先: ShmemInitStruct, memset, SpinLockInit, InitSharedLatch, ConditionVariableInit

関連データ構造

  • XLogRecoveryCtlData: リカバリ状態を保持する共有メモリ構造体。
  • XLogRecoveryCtl: XLogRecoveryCtlData を指すグローバルポインタ。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct 経由でエラーが発生する。
  • 既に初期化済みの場合は何も行わず正常終了する。

保守担当者向けメモ

  • 本関数はスタートアッププロセスにおいて共有メモリをセットアップする重要な役割を持つ。
  • XLogRecoveryCtlData の構造に変更を加えた場合は、XLogRecoveryShmemSize() の計算ロジックも併せて更新する必要がある。
