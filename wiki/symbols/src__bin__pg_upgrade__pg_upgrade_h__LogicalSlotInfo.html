LogicalSlotInfo

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_upgrade/pg_upgrade.h:160

目的

PostgreSQL のアップグレードプロセスにおいて、論理レプリケーションスロットのメタデータと状態を保持するために使用されます。

シグネチャまたは形状

typedef struct {
    char       *slotname;    /* スロット名 */
    char       *plugin;      /* プラグイン名 */
    bool        two_phase;   /* 2PC（2相コミット）デコード可能か */
    bool        caught_up;   /* 最新の変更に追いついているか */
    bool        invalid;     /* スロットが使用不能か */
    bool        failover;    /* 物理スタンバイへの同期対象か */
} LogicalSlotInfo;

入力と出力

本構造体はデータ保持用であり、関数ではありません。info.c 内で pg_malloc を用いてメモリが割り当てられ、スロット情報の収集時に各フィールドが設定されます。

前提条件と不変条件

  • slotname および plugin は文字列へのポインタであり、適切にメモリ確保されている必要があります。
  • invalid が true の場合、そのスロットはアップグレードプロセスにおいて使用不能とみなされます。

アルゴリズムまたは振る舞い

本構造体は、アップグレード対象のクラスタから論理スロットの情報を抽出し、新クラスタへ移行するための状態管理に使用されます。check.c や pg_upgrade.c において、スロットの整合性検証や移行処理の判断基準として参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/bin/pg_upgrade/check.c:2187
  • src/bin/pg_upgrade/info.c:718, 848
  • src/bin/pg_upgrade/pg_upgrade.c:999
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • LogicalSlotInfoArr: LogicalSlotInfo の配列を保持する構造体。
  • ClusterInfo: クラスタ全体の情報を管理する構造体であり、内部で LogicalSlotInfoArr を保持します。

失敗モードと境界条件

  • メモリ不足: info.c での pg_malloc 失敗時にプロセスが終了する可能性があります。
  • スロットの無効化: invalid フラグが true の場合、アップグレードの検証フェーズで警告やエラーの対象となる可能性があります。

保守担当者向けメモ

  • 本構造体は pg_upgrade のバージョン間移行における論理スロットの互換性を保証するために重要です。
  • フィールドの追加や変更を行う際は、info.c でのデータ収集ロジックおよび check.c での検証ロジックとの整合性を確認してください。
