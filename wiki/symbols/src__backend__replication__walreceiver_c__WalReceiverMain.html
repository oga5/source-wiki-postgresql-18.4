WalReceiverMain

定義

  • **種別**: function
  • **定義箇所**: src/backend/replication/walreceiver.c:151

目的

スタンバイサーバーにおける WAL 受信プロセス（walreceiver）のメインエントリポイントとして、プライマリサーバーとの接続確立、WAL ストリームの受信、および受信データの永続化を制御します。

シグネチャまたは形状

void WalReceiverMain(const void *startup_data, size_t startup_data_len)

入力と出力

  • **入力**: startup_data（現在は使用されず NULL を想定）、startup_data_len（0 である必要がある）。
  • **出力**: なし（プロセス終了時に proc_exit を呼び出す）。

前提条件と不変条件

  • WalRcv 共有メモリ構造体が初期化済みであること。
  • プロセス起動時に WalRcv->walRcvState が WALRCV_STARTING であること。
  • is_temp_slot と slotname は同時に設定できない。

アルゴリズムまたは振る舞い

1. **初期化**: プロセスタイプを B_WAL_RECEIVER に設定し、共有メモリの WalRcv 構造体を通じて状態を WALRCV_STREAMING に更新します。 2. **環境設定**: シグナルハンドラを設定し、libpqwalreceiver を動的ロードします。 3. **接続**: walrcv_connect を呼び出し、プライマリサーバーへの接続を確立します。 4. **検証**: walrcv_identify_system を使用して、プライマリとスタンバイのシステム識別子が一致することを確認します。 5. **ストリーミングループ**: プライマリから WAL データを受信し、ローカルへの書き込みとフィードバック送信を繰り返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c（Postmaster によるプロセス起動時）。
  • **呼び出し先**: walrcv_connect, walrcv_identify_system, load_file, on_shmem_exit（WalRcvDie を登録）。

関連データ構造

  • WalRcvData（共有メモリ上の制御構造体）
  • WalRcvStreamOptions
  • wrconn（libpqwalreceiver 接続ハンドル）

失敗モードと境界条件

  • **接続失敗**: walrcv_connect が失敗した場合、ERROR を報告して終了します。
  • **システム識別子不一致**: プライマリとスタンバイの識別子が異なる場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー終了します。
  • **状態競合**: 起動時に walRcvState が WALRCV_STARTING 以外の場合、PANIC を発生させます。

保守担当者向けメモ

  • 本関数は libpq に直接依存せず、実行時に libpqwalreceiver をロードする設計です。
  • 共有メモリの WalRcv 構造体へのアクセスはスピンロックで保護されています。
  • プロセス終了処理は on_shmem_exit を介して WalRcvDie が実行されるように構成されています。
