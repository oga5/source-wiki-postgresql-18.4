PQprintTuples

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-print.c:701

目的

PGresult オブジェクトに含まれるクエリ結果を、指定された出力ストリームに対して表形式で出力するレガシーなユーティリティ関数です。

シグネチャまたは形状

void PQprintTuples(const PGresult *res, FILE *fout, int PrintAttNames, int TerseOutput, int colWidth);

入力と出力

  • res: 出力対象の PGresult オブジェクト。
  • fout: 出力先の FILE ストリーム。
  • PrintAttNames: 0以外の場合、列名（ヘッダー）を出力する。
  • TerseOutput: 0以外の場合、区切り文字（|）を省略した簡潔な形式で出力する。
  • colWidth: 各列の幅。0の場合は可変幅として扱われる。
  • 戻り値: なし。

前提条件と不変条件

  • res は有効な PGresult オブジェクトである必要があります。
  • TerseOutput が 0 の場合、内部で malloc を使用して境界線用のメモリを確保します。

アルゴリズムまたは振る舞い

1. PQnfields と PQntuples で結果セットの規模を取得します。 2. colWidth に基づき、sprintf でフォーマット文字列を生成します。 3. nFields > 0 の場合、以下の処理を行います。

  • TerseOutput が 0 の場合、列数に応じた境界線（- の繰り返し）を生成し、出力します。
  • PrintAttNames が 0 以外の場合、PQfname を用いて列名を出力します。
  • 各タプル（行）について、PQgetvalue で値を取得し、フォーマットに従って出力します。
  • TerseOutput が 0 の場合は各行の区切りに境界線を出力します。

    • 4. 最後に確保した境界線用メモリを free します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/libpq-fe.h で公開されており、外部アプリケーションから呼び出される可能性があります。
  • 呼び出し先: PQnfields, PQntuples, PQfname, PQgetvalue（いずれも libpq の API）。

関連データ構造

  • PGresult: クエリ結果を保持する構造体。

失敗モードと境界条件

  • メモリ不足: 境界線用メモリの確保に失敗した場合、stderr にエラーメッセージを出力して関数を終了します。
  • nFields が 0 の場合、何も出力せずに終了します。
  • PQgetvalue が NULL を返す場合、空文字列として扱われます。

保守担当者向けメモ

  • 本関数はレガシーなインターフェースであり、現代のアプリケーションでの使用は推奨されません。
  • TerseOutput が 0 の際の境界線生成ロジックは、列数に対して固定的な幅（14文字）を想定しているため、列数が多い場合にレイアウトが崩れる可能性があります。
