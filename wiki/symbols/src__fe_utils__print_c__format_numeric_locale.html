format_numeric_locale

定義

  • 種別: function
  • 定義箇所: src/fe_utils/print.c:313

目的

数値文字列に対して、現在のロケール設定に基づいた桁区切り文字や小数点記号を適用し、整形された文字列を生成する。

シグネチャまたは形状

char *format_numeric_locale(const char *my_str)

入力と出力

  • **入力**: my_str (整形対象の数値文字列)
  • **出力**: 整形後の新しい文字列（呼び出し元で解放が必要）。入力が数値形式でない場合は、入力文字列の複製を返す。

前提条件と不変条件

  • 入力文字列が数値（0-9, +, -, ., e, E）のみで構成されている必要がある。
  • グローバル変数 groupdigits（グループ化の桁数）、thousands_sep（桁区切り文字）、decimal_point（小数点記号）が適切に設定されていること。
  • additional_numeric_locale_len が、整形後の文字列長を正確に算出できること。

アルゴリズムまたは振る舞い

1. strspn を使用し、入力が数値形式か判定する。数値でない場合は pg_strdup で複製して即時終了する。 2. additional_numeric_locale_len を用いて必要なバッファサイズを計算し、pg_malloc でメモリを確保する。 3. 符号（+ または -）が存在する場合は、先頭にコピーする。 4. 整数部を走査し、groupdigits に基づいて thousands_sep を挿入する。 5. 小数点が存在する場合は decimal_point に置換する。 6. 残りの小数部や指数部をコピーし、ヌル終端する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/print.c:3590（セルデータの整形処理にて使用）
  • **呼び出し先**: pg_strdup, pg_malloc, integer_digits, additional_numeric_locale_len, Assert

関連データ構造

  • thousands_sep (グローバル変数)
  • decimal_point (グローバル変数)
  • groupdigits (グローバル変数)

失敗モードと境界条件

  • メモリ不足時: pg_malloc によりエラー終了する可能性がある。
  • 非数値文字列: 変換を行わず、元の文字列のコピーを返す。
  • 桁区切りが不要な数値: 変更を加えずにコピーする。

保守担当者向けメモ

  • 本関数は psql 等のフロントエンドでクエリ結果を表示する際に使用される。
  • 既にロケールが適用済みの「通貨」値などを誤って再整形しないよう、入力チェックが厳格に行われている。
  • new_len の計算において、過小評価が発生しないよう additional_numeric_locale_len との整合性を維持すること。
