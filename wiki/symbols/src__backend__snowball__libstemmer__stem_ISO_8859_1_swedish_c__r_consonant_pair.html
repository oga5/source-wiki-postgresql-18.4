r_consonant_pair

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_swedish.c:203

目的

単語の末尾にある重複子音ペアを検出し、その一部を削除することで語幹抽出を補助する。

シグネチャまたは形状

static int r_consonant_pair(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体（ステミング環境）。
  • 出力: 処理が成功した場合は 1、失敗または条件不一致の場合は 0、エラー発生時は負の値を返す。

前提条件と不変条件

  • z->I[1] に格納された領域境界よりカーソル位置 z->c が後ろにあること。
  • z->lb（下限）を一時的に z->I[1] に変更して処理を行う。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置が z->I[1] より前であれば終了する。 2. z->lb を z->I[1] に設定し、領域を制限する。 3. 文字列末尾から find_among_b を使用して a_1 配列内のパターンを検索する。 4. マッチした場合、z->ket（終了位置）を現在のカーソル位置に設定し、カーソルを1つ戻して z->bra（開始位置）とする。 5. slice_del を呼び出し、該当する子音ペアを削除する。 6. 最後に z->lb を元の値（mlimit1）に復元する。

呼び出し元と呼び出し先

  • 呼び出し元: stem_ISO_8859_1_swedish.c 内のステミング処理フローから参照される。また、デンマーク語およびノルウェー語のステミング実装（stem_ISO_8859_1_danish.c, stem_ISO_8859_1_norwegian.c 等）からも呼び出されることが確認されている。
  • 呼び出し先: find_among_b（パターンマッチング）、slice_del（文字列削除）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_1: 重複子音ペアの定義を含む among 構造体の配列。

失敗モードと境界条件

  • カーソル位置が z->I[1] より前の場合、即座に 0 を返す。
  • find_among_b がパターンに一致しない場合、0 を返す。
  • slice_del がエラー（負の値）を返した場合、その値をそのまま伝播させる。

保守担当者向けメモ

  • 本関数は自動生成コードであるため、直接編集せず、元のSnowballスクリプト（.sbl）を修正すること。
  • z->p[z->c - 1] に対するビット演算による文字種判定が行われているため、文字コードの扱いに注意が必要。
