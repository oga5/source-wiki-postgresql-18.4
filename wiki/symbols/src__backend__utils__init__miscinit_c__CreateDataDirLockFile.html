CreateDataDirLockFile

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/miscinit.c:1514

目的

PostgreSQLのデータディレクトリに対して、二重起動を防止するためのロックファイル（postmaster.pid）を作成する。

シグネチャまたは形状

void CreateDataDirLockFile(bool amPostmaster)

入力と出力

  • **入力**:
  • amPostmaster (bool): 呼び出し元がPostmasterプロセスであるかどうかを示すフラグ。
  • **出力**: なし（成功時はロックファイルを作成し、失敗時はエラーをスローする）。

前提条件と不変条件

  • DataDir グローバル変数が有効なデータディレクトリパスを保持していること。
  • 本関数は、プロセスがデータディレクトリを使用する初期化フェーズで呼び出される必要がある。

アルゴリズムまたは振る舞い

1. DIRECTORY_LOCK_FILE を指定して内部関数 CreateLockFile を呼び出す。 2. CreateLockFile に対して、amPostmaster フラグ、空の追加情報、true（データディレクトリ用であることを示すフラグ）、および DataDir を渡す。 3. CreateLockFile 内でロックファイルの作成および終了時の自動削除登録が行われる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/bootstrap/bootstrap.c
  • src/backend/postmaster/postmaster.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • CreateLockFile (同ファイル内)

関連データ構造

  • DataDir (グローバル変数): ロックファイルを作成する対象ディレクトリのパスを保持。

失敗モードと境界条件

  • 指定されたディレクトリに既にロックファイルが存在する場合、二重起動とみなされエラーが発生する。
  • ファイルシステム権限によりロックファイルを作成できない場合、プロセスは起動に失敗する。

保守担当者向けメモ

  • 本関数は CreateLockFile のラッパーであり、データディレクトリ専用のロックファイル作成に特化している。
  • プロセス終了時にロックファイルが自動的にアンリンクされる仕組みは CreateLockFile 内部の proc_exit 登録に依存している。
