LockClauseStrength

定義

  • 種別: typedef (列挙型)
  • 定義箇所: src/include/nodes/lockoptions.h:28 (参照: src/include/foreign/fdwapi.h:130)

目的

SQLクエリにおける FOR UPDATE や FOR SHARE などのロック句の強度を表現するために使用されます。

シグネチャまたは形状

列挙型として定義されており、各値はロックの強さを表します。

入力と出力

  • 入力: 該当なし（型定義のため）
  • 出力: 該当なし

前提条件と不変条件

  • ForeignScanState 等の実行ノードにおいて、ロック句がない場合は LCS_NONE が設定されることが期待されます。

アルゴリズムまたは振る舞い

  • 外部データラッパー（FDW）のインターフェースにおいて、GetForeignRowMarkType 関数等の引数として渡され、リモート側でどのような行ロックが必要かを決定するために利用されます。
  • LCS_asString 関数を通じて、デバッグやログ出力用の文字列変換が可能です。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/optimizer/plan/planner.c: クエリ計画作成時にロック強度の判定に使用。
  • src/backend/parser/analyze.c: パーサによるロック句の解析および検証に使用。
  • src/backend/rewrite/rewriteHandler.c: ルール書き換え処理でのロックポリシー適用に使用。
  • 呼び出し先:
  • 該当なし（型定義のため）

関連データ構造

  • RowMarkType: GetForeignRowMarkType_function の戻り値として、この強度に基づき決定されます。
  • ForeignScanState: 実行時にこの強度を保持するフィールドを持ちます。

失敗モードと境界条件

  • 未知の LockClauseStrength 値が渡された場合、planner.c や ruleutils.c 内の elog(ERROR, ...) によりエラーが発生します。

保守担当者向けメモ

  • 本シンボルは src/include/nodes/lockoptions.h で定義されています。FDW API 側で利用する際は、このヘッダーが適切にインクルードされていることを確認してください。
  • 新しいロック強度の種類を追加する場合は、関連する解析関数およびエラーハンドリング箇所（elog を含む箇所）の更新が必要です。
