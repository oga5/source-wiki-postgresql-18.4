gistendscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistscan.c:346

目的

GiSTインデックススキャンに関連するリソースを解放し、スキャン状態を終了させる。

シグネチャまたは形状

void gistendscan(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan - 終了対象となるインデックススキャン記述子。
  • **出力**: なし。

前提条件と不変条件

  • scan は gistbeginscan によって初期化済みであること。
  • scan->opaque には GISTScanOpaque 構造体が格納されていること。

アルゴリズムまたは振る舞い

1. scan->opaque から GISTScanOpaque 構造体を取得する。 2. 内部の giststate を引数として freeGISTstate を呼び出す。 3. freeGISTstate により、スキャン中に使用されたメモリコンテキスト（queueCxt を含む）および関連リソースが解放される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c 内の amroutine->amendscan 経由で実行エンジンから呼び出される。
  • **呼び出し先**: freeGISTstate（リソース解放用関数）。

関連データ構造

  • IndexScanDesc: PostgreSQLの標準的なインデックススキャン記述子。
  • GISTScanOpaque: GiSTスキャン固有の状態を保持する構造体。

失敗モードと境界条件

  • scan が NULL の場合、または scan->opaque が正しく初期化されていない場合は未定義の動作となる。

保守担当者向けメモ

  • freeGISTstate は gistbeginscan で確保されたすべてのリソースをクリーンアップするように設計されているため、本関数内での追加の解放処理は不要である。
