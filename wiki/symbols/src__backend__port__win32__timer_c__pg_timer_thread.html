pg_timer_thread

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/timer.c:35

目的

Windows環境において、POSIX準拠の setitimer（ITIMER_REAL）をエミュレートするためのバックグラウンドスレッド関数です。タイムアウトを監視し、期限到達時に SIGALRM を発行します。

シグネチャまたは形状

static DWORD WINAPI pg_timer_thread(LPVOID param);

入力と出力

  • 入力: param（常に NULL であることが期待される）。
  • 出力: スレッド終了コード（0）。

前提条件と不変条件

  • param は NULL であること（Assert による検証）。
  • timerCommArea を介してメインスレッドと同期を行う。
  • timerCommArea.crit_sec による排他制御下でタイマー値の更新を行う。

アルゴリズムまたは振る舞い

1. 無限ループ内で WaitForSingleObjectEx を呼び出し、イベント通知またはタイムアウトを待機する。 2. **イベント通知時**: メインスレッドからタイマー設定が更新されたと判断し、timerCommArea から新しい待機時間を計算する。値が 0 の場合はタイマーを停止（INFINITE 待機）する。 3. **タイムアウト時**: pg_queue_signal(SIGALRM) を呼び出し、シグナルをキューイングした後、待機時間を INFINITE に戻す。 4. 予期せぬエラー発生時は Assert(false) により停止する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/port/win32/timer.c 内の CreateThread 呼び出し箇所（タイマー初期化時）。
  • 呼び出し先: pg_queue_signal（src/backend/port/win32/signal.c で定義）。

関連データ構造

  • timerCommArea: メインスレッドとタイマー管理スレッド間で共有されるタイマー設定および同期用構造体。

失敗モードと境界条件

  • it_interval（インターバルタイマー）はサポートされていません。
  • WaitForSingleObjectEx を使用しているため、精度はミリ秒単位に制限されます。
  • タイムアウト計算時にミリ秒単位への切り上げ処理が行われます。

保守担当者向けメモ

  • 本スレッドは ITIMER_REAL のエミュレーション専用です。
  • timerCommArea へのアクセスは必ず CRITICAL_SECTION で保護されている必要があります。
  • デバッグ時は、イベントの取りこぼしやスレッド間のデッドロックに注意してください。
