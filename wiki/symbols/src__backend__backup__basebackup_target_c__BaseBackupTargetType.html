BaseBackupTargetType

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup_target.c:21

目的

PostgreSQLのベースバックアップにおける出力先（ターゲット）を定義し、拡張可能にするための構造体です。バックアップデータの検証および転送パイプライン（bbsink）の構築に必要なインターフェースを保持します。

シグネチャまたは形状

typedef struct BaseBackupTargetType
{
    char       *name;
    void       *(*check_detail) (char *, char *);
    bbsink     *(*get_sink) (bbsink *, void *);
} BaseBackupTargetType;

入力と出力

  • name: ターゲットを一意に識別する名称。
  • check_detail: ターゲット固有の詳細情報を検証する関数ポインタ。検証成功時に詳細データを返し、失敗時にエラーを発生させます。
  • get_sink: 最終的な bbsink パイプラインを生成する関数ポインタ。

前提条件と不変条件

  • ターゲットリストは TopMemoryContext 上で管理されます。
  • check_detail はバックアップ開始の初期段階で呼び出され、検証を行います。

アルゴリズムまたは振る舞い

1. **登録**: BaseBackupAddTarget() を通じて、組み込みターゲットおよび拡張モジュールによるカスタムターゲットがリストに登録されます。 2. **検証**: BaseBackupGetTargetHandle() がターゲット名を検索し、対応する check_detail を実行して詳細情報を検証します。 3. **構築**: BaseBackupGetSink() が呼び出されると、ターゲットの get_sink 関数が実行され、データ転送用の bbsink チェーンが構築されます。

呼び出し元と呼び出し先

  • 呼び出し元: BaseBackupAddTarget、BaseBackupGetTargetHandle、BaseBackupGetSink、および builtin_backup_targets 配列の初期化処理から参照されます。
  • 呼び出し先: ターゲットごとに実装された check_detail および get_sink 関数が実行されます。

関連データ構造

  • bbsink: バックアップデータの転送先となるシンクインターフェース。
  • builtin_backup_targets: 組み込みターゲット（server, blackhole）を保持する静的配列。

失敗モードと境界条件

  • check_detail が失敗した場合、バックアップ処理は開始されません。
  • ターゲット名がリストに見つからない場合、BaseBackupGetTargetHandle はエラーを返します。

保守担当者向けメモ

  • 新しいバックアップターゲットを追加する場合は、BaseBackupAddTarget を使用してください。
  • メモリリークを防ぐため、ターゲットの追加には TopMemoryContext を使用することが推奨されます。
