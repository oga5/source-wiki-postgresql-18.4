PLy_result_status

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_resultobject.c:226

目的

PLyResultObject が保持するクエリ実行ステータス情報を、Pythonオブジェクトとして取得する。

シグネチャまたは形状

static PyObject *PLy_result_status(PyObject *self, PyObject *args)

入力と出力

  • **入力**:
  • self: PLyResultObject インスタンス（PyObject として渡される）。
  • args: Python側からの引数（本関数では使用されない）。
  • **出力**:
  • ob->status: PLyResultObject 内部のステータスオブジェクト。参照カウントがインクリメントされた状態で返される。

前提条件と不変条件

  • self は PLyResultObject 型へのポインタとしてキャスト可能であること。
  • ob->status は有効なPythonオブジェクトを指していること。

アルゴリズムまたは振る舞い

1. 引数 self を PLyResultObject * にキャストする。 2. 内部構造体 ob が保持する status メンバの参照カウントを Py_INCREF で増加させる。 3. 増加させた status オブジェクトを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Pythonのメソッドテーブル PLyResult_spec（src/pl/plpython/plpy_resultobject.c:32）を介して、Pythonインタープリタから呼び出される。
  • **呼び出し先**: Py_INCREF マクロ（Python C API）。

関連データ構造

  • PLyResultObject: クエリ結果を保持するC構造体。status メンバを保持する。

失敗モードと境界条件

  • self が PLyResultObject 以外の型である場合、キャスト後のアクセスにより未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は METH_VARARGS 形式で定義されているが、引数 args は使用していない。
  • 戻り値の参照カウントを管理しているため、呼び出し側（Python側）は受け取ったオブジェクトの参照を適切に解放する必要がある。
