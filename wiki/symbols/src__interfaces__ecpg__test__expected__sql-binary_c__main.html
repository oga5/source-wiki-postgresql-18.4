目的

ECPG（Embedded SQL for C）におけるバイナリデータ型（bytea）およびバイナリカーソルの動作を検証するための回帰テストを実行するエントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しない）
  • 出力: 終了コード 0（成功時）、または 1（SQL エラー発生時）。標準出力へテスト結果を表示。

前提条件と不変条件

  • データベース ecpg1_regression への接続が可能であること。
  • ECPG プリプロセッサによって生成されたコードであり、実行環境には ecpglib が必要。
  • バイナリフェッチ時に割り当てられたメモリ（pointer）は、使用後に free() で解放される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、データベースへ接続。bytea_output を escape モードに設定。 2. **テーブル作成**: empl テーブルを作成し、bytea 型カラムを含むデータを挿入。 3. **カーソル操作**:

  • 通常のカーソル C を使用してデータを取得・表示。
  • BINARY カーソル B を使用してデータを取得・表示。
  • BINARY カーソル A を使用してバイナリデータをフェッチし、8進数形式で出力。

    • 4. **終了処理**: pointer のメモリ解放を行い、ECPGdisconnect で接続を終了して終了コード 0 で終了。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGset_var, ECPGdisconnect, printf, memset, free, exit

関連データ構造

  • struct TBempl: テーブル構造を模した C 構造体。
  • sqlca: SQL 通信領域（SQLCA）。各 SQL 文実行後のエラーチェックに使用。

失敗モードと境界条件

  • 各 SQL 文実行後に sqlca.sqlcode < 0 をチェックしており、エラー発生時は即座に exit(1) を呼び出して終了する。
  • pointer が NULL でない場合のみ free() を実行する安全策が講じられている。
  • プラットフォーム間のエンディアン差異を考慮し、accs（short 型）の出力は意図的にスキップされている。

保守担当者向けメモ

  • 本ファイルは binary.pgc から生成されたコードであるため、直接編集せず元の .pgc ファイルを修正すること。
  • バイナリデータのフェッチ時に動的メモリ割り当てが発生するため、メモリリークの防止に注意が必要。
