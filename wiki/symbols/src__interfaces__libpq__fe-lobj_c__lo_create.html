lo_create

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:473

目的

データベース内に新しいラージオブジェクトを作成し、そのOID（Object Identifier）を返します。

シグネチャまたは形状

Oid lo_create(PGconn *conn, Oid lobjId);

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • lobjId: 作成するラージオブジェクトのOID。0を指定するとシステムが自動的にOIDを割り当てます。
  • **出力**:
  • 成功時: 作成されたラージオブジェクトのOID。
  • 失敗時: InvalidOid。

前提条件と不変条件

  • lo_initialize(conn) が成功している必要があります。
  • サーバー側で lo_create 関数が利用可能である必要があります（PostgreSQL 8.1以降）。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、サーバー側関数のOIDキャッシュを準備します。 2. conn->lobjfuncs->fn_lo_create が存在するか確認し、存在しない場合はエラーを記録して InvalidOid を返します。 3. PQArgBlock を使用して lobjId を引数として設定します。 4. PQfn（Fastpath）を介してサーバー側の lo_create 関数を呼び出します。 5. サーバーからの応答が PGRES_COMMAND_OK であれば、結果からOIDを取得して返します。それ以外の場合は InvalidOid を返します。 6. 実行結果の PGresult は必ず解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_dump（pg_backup_archiver.c, pg_backup_null.c）および fe-lobj.c 内の他の関数から参照されています。
  • **呼び出し先**: lo_initialize（初期化）、PQfn（サーバー関数実行）、libpq_append_conn_error（エラー報告）、PQclear（メモリ解放）。

関連データ構造

  • PGconn: 接続状態およびサーバー側関数のOIDキャッシュを保持。
  • PQArgBlock: Fastpath呼び出し用の引数ブロック。
  • PGresult: サーバーからの実行結果を保持。

失敗モードと境界条件

  • 接続が初期化できない場合、またはサーバーが lo_create をサポートしていない場合は InvalidOid を返します。
  • サーバー側でエラーが発生した場合、PQresultStatus が PGRES_COMMAND_OK 以外となり、InvalidOid を返します。
  • 呼び出し元は、戻り値が InvalidOid であるかどうかを確認する必要があります。

保守担当者向けメモ

  • 本関数は8.1より前のサーバーバージョンでは利用できません。
  • 非常に大きなオブジェクトを扱う際のトランザクションアボートのリスクについて、呼び出し側で考慮が必要です。
