ExecCustomMarkPos

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCustom.c:138

目的

カスタムスキャン実行ノードにおいて、現在のスキャン位置をマーク（保存）する処理を拡張機能側の実装へ委譲します。

シグネチャまたは形状

void ExecCustomMarkPos(CustomScanState *node)

入力と出力

  • **入力**: CustomScanState *node - カスタムスキャンの実行状態を保持する構造体。
  • **出力**: なし（副作用としてスキャン位置を記録）。

前提条件と不変条件

  • node は有効な CustomScanState を指している必要があります。
  • 拡張機能側で CustomExecMethods 内の MarkPosCustomScan コールバックが定義されている必要があります。

アルゴリズムまたは振る舞い

1. node->methods->MarkPosCustomScan が存在するか確認します。 2. 存在しない場合は ERRCODE_FEATURE_NOT_SUPPORTED エラーを発生させます。 3. 存在する場合は、その関数ポインタを呼び出し、拡張機能固有のマーク処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:340
  • **呼び出し先**: node->methods->MarkPosCustomScan（拡張機能が提供する関数）

関連データ構造

  • CustomScanState: カスタムスキャンの実行状態を管理する構造体。
  • CustomExecMethods: 拡張機能が実装するコールバック関数テーブル。

失敗モードと境界条件

  • 拡張機能が MarkPosCustomScan を実装していない場合、ereport(ERROR, ...) により処理が中断されます。

保守担当者向けメモ

  • 本関数は単なるディスパッチャです。ロジックの詳細は各拡張機能の実装に依存します。
  • 拡張機能側で MarkPos をサポートしない場合、この関数を呼び出すとクエリ実行がエラーで停止するため、呼び出し側での事前チェックが必要な場合があります。
