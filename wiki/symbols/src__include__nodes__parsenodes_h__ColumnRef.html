ColumnRef

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/parsenodes.h:307

目的

SQL文中の列参照（例: table.column や column）を表現するためのパースツリーノードです。

シグネチャまたは形状

typedef struct ColumnRef
{
	NodeTag		type;
	List	   *fields;			/* フィールド名 (Stringノード) または A_Star */
	ParseLoc	location;		/* トークン位置、不明な場合は -1 */
} ColumnRef;

入力と出力

  • **入力**: パーサー（gram.y）がSQL構文を解析する際に生成されます。
  • **出力**: 意味解析器（analyze.c）がこのノードを処理し、クエリツリー内の適切な参照（Varノード等）へ変換します。

前提条件と不変条件

  • fields リストには、列名を表す String ノードが含まれるか、あるいは A_Star（*）が含まれます。
  • location はSQL文字列内のバイトオフセットを保持し、エラー報告に使用されます。

アルゴリズムまたは振る舞い

パーサーによって生成された後、解析器が fields リストを走査して参照先を特定します。A_Star が含まれる場合は、ワイルドカード展開の対象として扱われます。

呼び出し元と呼び出し先

  • **呼び出し元（生成・操作）**:
  • src/backend/commands/copy.c: COPY コマンド処理において ColumnRef を生成。
  • src/backend/parser/gram.y: SQL解析時に生成。
  • **呼び出し先（処理）**:
  • src/backend/commands/typecmds.c: replace_domain_constraint_value 等で参照。
  • src/backend/executor/functions.c: sql_fn_post_column_ref で処理。
  • src/backend/nodes/nodeFuncs.c: location フィールドの取得等に使用。
  • src/backend/catalog/namespace.c: A_Star の許容判定に使用。

関連データ構造

  • NodeTag: ノードの種類を識別するタグ。
  • List: フィールド名のリストを保持。
  • A_Star: * を表現する特殊なノード。

失敗モードと境界条件

  • location が不明な場合、値は -1 となります。
  • fields が空である、あるいは不正なノードを含む場合、解析器による意味解析フェーズでエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はパーサーの出力であるため、クエリの実行時には存在しません。実行時には Var ノード等に変換されている必要があります。
  • location フィールドはエラーメッセージのカーソル位置表示に不可欠であるため、ノード生成時には必ず適切な値を設定してください。
