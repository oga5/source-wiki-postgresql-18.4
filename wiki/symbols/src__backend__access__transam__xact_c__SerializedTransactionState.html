SerializedTransactionState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/xact.c:227

目的

並列クエリ実行時に、リーダーバックエンドのトランザクション状態をワーカーバックエンドへ伝播・同期させるためのシリアライズ用データ構造です。

シグネチャまたは形状

typedef struct SerializedTransactionState
{
	int			xactIsoLevel;
	bool		xactDeferrable;
	FullTransactionId topFullTransactionId;
	FullTransactionId currentFullTransactionId;
	CommandId	currentCommandId;
	int			nParallelCurrentXids;
	TransactionId parallelCurrentXids[FLEXIBLE_ARRAY_MEMBER];
} SerializedTransactionState;

入力と出力

  • **入力**: リーダーバックエンドの現在のトランザクション状態（分離レベル、XID、CommandId等）。
  • **出力**: 並列ワーカーが参照可能な共有メモリ上のシリアライズされた状態データ。

前提条件と不変条件

  • parallelCurrentXids は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、構造体末尾に配置されます。
  • 構造体のサイズ計算には、offsetof(SerializedTransactionState, parallelCurrentXids) を使用する必要があります。

アルゴリズムまたは振る舞い

  • リーダーバックエンドがトランザクション状態をこの構造体に書き込み、ワーカーがそれを読み取ることで、並列実行環境下でのトランザクションの一貫性を維持します。
  • 並列クエリ実行時の状態伝播において、リーダーからワーカーへの状態引き継ぎを仲介します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c 内の並列クエリ関連処理（SerializeTransactionState 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。
  • 注: 具体的な呼び出し元の関数名や、本構造体を利用するワーカー側の詳細な呼び出しフローについては、提供されたソース excerpt には記載されていません。

関連データ構造

  • FullTransactionId: 64ビットのトランザクションID。
  • TransactionId: 32ビットのトランザクションID。
  • CommandId: コマンド識別子。

失敗モードと境界条件

  • parallelCurrentXids の要素数（nParallelCurrentXids）が確保されたメモリ領域を超えないように管理する必要があります。
  • 構造体末尾の可変長配列の扱いに起因するメモリレイアウトの不整合に注意が必要です。

保守担当者向けメモ

  • 本構造体は並列クエリのトランザクション同期に直結するため、フィールドの追加や変更を行う際は、リーダーとワーカー間のシリアライズ/デシリアライズ処理（xact.c 内の関連関数）を必ず更新してください。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体全体のサイズを動的に計算する箇所（offsetof を使用している箇所）の整合性を維持してください。
