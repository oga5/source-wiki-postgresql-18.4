r_standard_suffix

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_german.c:328

目的

ドイツ語のステミング処理において、単語末尾の接尾辞を特定のルールセットに基づいて削除または置換し、語幹を抽出する。

シグネチャまたは形状

static int r_standard_suffix(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （Snowball ステミング環境へのポインタ）
  • 出力: int （成功時は 0 以外、失敗時は 0、エラー発生時は負の値を返す）

前提条件と不変条件

  • z が指す環境構造体は初期化済みであること。
  • z->I[0] (R1) および z->I[2] (R2) が r_mark_regions によって適切に設定されていること。
  • 処理対象の単語は ISO-8859-1 エンコーディングであること。

アルゴリズムまたは振る舞い

本関数は、以下の3つの主要な接尾辞グループを順次適用する。 1. **グループ1**: a_2 に定義された接尾辞を検索し、R1 領域内であれば削除または置換を行う。特定のケースでは追加の削除処理（s_11 の削除など）を伴う。 2. **グループ2**: a_3 に定義された接尾辞を検索し、R1 領域内であれば削除を行う。 3. **グループ3**: a_5 に定義された接尾辞を検索し、R2 領域内であれば削除または置換を行う。この過程で s_13 や s_14 等の条件付き削除が実行される。

各ステップは find_among_b を用いて後方からマッチングを行い、条件を満たさない場合は元の位置へ復帰する。

呼び出し元と呼び出し先

  • 呼び出し元: german_ISO_8859_1_stem （本ファイル内）
  • 呼び出し先: find_among_b, r_R1, r_R2, slice_del, slice_from_s, eq_s_b, in_grouping_b

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • among 構造体: 接尾辞マッチングのためのトライ木データ構造。
  • a_2, a_3, a_4, a_5: 接尾辞検索用の静的テーブル。

失敗モードと境界条件

  • z->c が z->lb 以下の場合、または領域（R1, R2）の条件を満たさない場合は処理をスキップする。
  • slice_del 等の内部関数が負の値を返した場合、その値をそのまま呼び出し元へ伝播させる。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。直接編集せず、元の Snowball スクリプトを修正すること。
  • z->p[z->c - 1] を用いたビット演算による文字チェックは、効率的なマッチングのための最適化である。
