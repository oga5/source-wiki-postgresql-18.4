end_progress_output

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/util.c:42

目的

進行状況を表示していた行をクリアし、後続の report_status() による出力が適切に整列されるよう、TTYまたは冗長モードにおいてパディングを適用する。

シグネチャまたは形状

void end_progress_output(void)

入力と出力

  • 入力: なし（グローバル変数 log_opts を参照）
  • 出力: なし（副作用として標準出力またはログファイルへの書き込みを行う）

前提条件と不変条件

  • log_opts 構造体が初期化されていること。
  • MESSAGE_WIDTH 定数が定義されていること。

アルゴリズムまたは振る舞い

1. log_opts.isatty が真の場合、キャリッジリターン (\r) を出力してカーソルを行頭に戻す。 2. log_opts.isatty が真、または log_opts.verbose が真の場合、MESSAGE_WIDTH 分のスペースを pg_log(PG_REPORT_NONL, ...) を介して出力し、表示領域をクリアおよび整列する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_upgrade/dump.c:76
  • src/bin/pg_upgrade/pg_upgrade.c:690, 1023
  • src/bin/pg_upgrade/relfilenumber.c:159
  • 呼び出し先:
  • pg_log() (ログ出力用インターフェース)

関連データ構造

  • LogOpts (グローバル変数 log_opts の型): ログ出力設定を保持。

失敗モードと境界条件

  • log_opts.isatty および log_opts.verbose が共に偽の場合、本関数は何も出力せず終了する。

保守担当者向けメモ

  • 本関数はメッセージの末尾に改行を含めない設計である。
  • pg_log の PG_REPORT_NONL 型を利用して、進捗表示の制御を行っている。
