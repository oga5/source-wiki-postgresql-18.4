parse_unicode_version

定義

  • 種別: function
  • 定義箇所: src/common/unicode/category_test.c:34

目的

Unicode バージョンを表す文字列（例: "15.1"）を、比較可能な整数値に変換します。

シグネチャまたは形状

static int parse_unicode_version(const char *version)

入力と出力

  • **入力**: version (const char *) - "major.minor" 形式のバージョン文字列。
  • **出力**: int - major * 100 + minor で計算された整数値。

前提条件と不変条件

  • 入力文字列は sscanf により "d.d" 形式で解析可能である必要があります。
  • minor バージョンは 100 未満である必要があります（Assert による検証）。

アルゴリズムまたは振る舞い

1. sscanf を使用して、入力文字列からメジャーバージョンとマイナーバージョンを抽出します。 2. 抽出された値が期待通り（2つの整数）であることを Assert で確認します。 3. マイナーバージョンが 100 未満であることを Assert で確認します。 4. major * 100 + minor の計算結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode/category_test.c 内の main 関数（PG_UNICODE_VERSION および U_UNICODE_VERSION の解析時）。
  • **呼び出し先**:
  • 標準ライブラリ関数: sscanf
  • PostgreSQL マクロ: Assert

関連データ構造

  • なし。

失敗モードと境界条件

  • sscanf が 2 つの整数を抽出できない場合、または minor が 100 以上の場合、アサーション失敗によりプログラムが停止します（デバッグビルド時）。

保守担当者向けメモ

  • 本関数はテストプログラム内でのバージョン比較を容易にするためのユーティリティです。
  • minor バージョンが 100 以上になるような将来の Unicode バージョン体系への変更が発生した場合は、計算ロジックの修正が必要です。
