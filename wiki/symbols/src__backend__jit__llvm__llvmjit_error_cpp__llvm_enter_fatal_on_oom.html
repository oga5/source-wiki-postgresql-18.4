llvm_enter_fatal_on_oom

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_error.cpp:54

目的

LLVMのJITコンパイル処理において、メモリ不足（OOM）や致命的エラーが発生した際に、PostgreSQLの標準的な ereport(FATAL, ...) を呼び出してプロセスを安全に終了させるためのエラーハンドラを登録します。

シグネチャまたは形状

void llvm_enter_fatal_on_oom(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルなエラーハンドラ設定を変更）

前提条件と不変条件

  • fatal_new_handler_depth 変数により、ネストされた呼び出しを追跡します。
  • 呼び出しがネストされている場合、ハンドラの再登録は行われず、深さのカウントのみが増加します。

アルゴリズムまたは振る舞い

1. fatal_new_handler_depth が 0 の場合、現在のC++の new_handler を保存し、以下のハンドラを登録します。

  • std::set_new_handler に fatal_system_new_handler を設定。
  • llvm::install_bad_alloc_error_handler に fatal_llvm_new_handler を設定。
  • llvm::install_fatal_error_handler に fatal_llvm_error_handler を設定。

    • 2. fatal_new_handler_depth をインクリメントします。

呼び出し元と呼び出し先

  • 呼び出し元（直接の証拠）:
  • src/backend/jit/llvm/llvmjit.c:271
  • src/backend/jit/llvm/llvmjit_expr.c:137
  • src/backend/jit/llvm/llvmjit_expr.c:2995
  • 呼び出し先:
  • std::set_new_handler
  • llvm::install_bad_alloc_error_handler
  • llvm::install_fatal_error_handler

関連データ構造

  • fatal_new_handler_depth: ネストレベルを管理する静的変数。
  • old_new_handler: 以前の new_handler を保持する静的変数。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • 既にハンドラが設定されている状態（depth > 0）で呼び出された場合、ハンドラの再登録はスキップされ、深さのみが更新されます。

保守担当者向けメモ

  • 本関数は extern "C" ブロック内で定義されており、C言語側からの呼び出しを想定しています。
  • エラー発生時は FATAL レベルでプロセスを終了させるため、トランザクションの整合性を維持するための防波堤として機能します。
  • 正常終了時には必ず対となる llvm_leave_fatal_on_oom() を呼び出し、ハンドラを復元する必要があります。
