transformExpressionList

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_target.c:219

目的

構文解析された式のリスト（exprlist）を走査し、各要素を意味解析済みのノードへ変換してリストとして返します。特に *（ワイルドカード）を含む式を適切に展開する役割を担います。

シグネチャまたは形状

List *transformExpressionList(ParseState *pstate, List *exprlist, ParseExprKind exprKind, bool allowDefault)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体。
  • exprlist: 変換対象となる Node のリスト。
  • exprKind: 式のコンテキストを示す ParseExprKind。
  • allowDefault: SetToDefault ノードの入力を許可するかどうか。
  • **出力**:
  • 変換後の Node を含む List。

前提条件と不変条件

  • exprlist 内の各要素は ColumnRef、A_Indirection、またはその他の式ノードであると想定されます。
  • allowDefault が true の場合、SetToDefault ノードは変換されずにそのまま保持されます。

アルゴリズムまたは振る舞い

1. exprlist を走査し、各要素 e を確認します。 2. e が ColumnRef であり、末尾が A_Star である場合、ExpandColumnRefStar を呼び出して展開し、結果リストに追加します。 3. e が A_Indirection であり、末尾が A_Star である場合、ExpandIndirectionStar を呼び出して展開し、結果リストに追加します。 4. 上記以外の場合、allowDefault が有効かつ e が SetToDefault であればそのまま保持し、それ以外は transformExpr を呼び出して意味解析を行います。 5. 処理済みのノードを result リストに追加し、最終的なリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c
  • src/backend/parser/parse_expr.c
  • src/backend/parser/parse_merge.c
  • **呼び出し先**:
  • ExpandColumnRefStar
  • ExpandIndirectionStar
  • transformExpr

関連データ構造

  • ParseState: 解析状態を管理。
  • ColumnRef, A_Indirection, SetToDefault: 構文解析ツリーのノード型。
  • List, ListCell: PostgreSQLのリスト構造。

失敗モードと境界条件

  • transformExpr が失敗した場合、内部でエラーがスローされます。
  • allowDefault が false の状態で SetToDefault が渡された場合、transformExpr がエラーを発生させます。

保守担当者向けメモ

  • 本関数は SELECT リストや INSERT/UPDATE の値リストなど、複数の式を扱う箇所で共通利用されています。
  • 新しい式ノードの型を追加した際は、transformExpr 側での対応が必要ですが、ワイルドカード展開のロジックに変更がある場合は本関数を確認してください。
