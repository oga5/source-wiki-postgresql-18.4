shm_mq_iovec

定義

  • 種別: struct
  • 定義箇所: src/include/storage/shm_mq.h:29

目的

共有メモリメッセージキュー (shm_mq) を介したデータ送信において、複数のメモリ領域を連結して単一のメッセージとして送信するための入出力ベクトル構造体です。

シグネチャまたは形状

typedef struct
{
	const char *data;
	Size		len;
} shm_mq_iovec;

入力と出力

  • data: 送信するデータが格納されたメモリ領域へのポインタ。
  • len: data が指す領域のバイト数。

前提条件と不変条件

  • data は送信処理が完了するまで有効なメモリ領域を指している必要があります。
  • len は送信するデータの正確なサイズである必要があります。

アルゴリズムまたは振る舞い

本構造体は、shm_mq_sendv 関数において、複数のデータ断片を一度の操作でキューに書き込むために使用されます。shm_mq_sendv は、提供された shm_mq_iovec 配列を順次処理し、リングバッファへデータをコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/pqmq.c: メッセージ送信時に shm_mq_iovec 配列を構築して使用しています。
  • src/backend/storage/ipc/shm_mq.c: 内部的な送信処理で shm_mq_iovec を使用しています。
  • **呼び出し先**:
  • shm_mq_sendv 関数が本構造体を引数として受け取ります。

関連データ構造

  • shm_mq: メッセージキュー本体。
  • shm_mq_handle: キューを操作するためのプライベートハンドル。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、shm_mq_sendv に渡す際に len が不適切な場合や、data が無効なメモリ領域を指している場合は、送信処理が失敗またはクラッシュする可能性があります。

保守担当者向けメモ

  • 本構造体は shm_mq_sendv のインターフェースの一部です。shm_mq_sendv の実装を変更する際は、この構造体のメモリレイアウトと整合性を保つ必要があります。
