daitch_mokotoff

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/daitch_mokotoff.c:122

目的

Daitch-Mokotoff Soundex アルゴリズムを適用し、入力文字列から音韻的な類似性に基づくコードの配列を生成する SQL 呼び出し可能な関数。

シグネチャまたは形状

Datum daitch_mokotoff(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される text 型の文字列。
  • **出力**: 音韻コードの配列（Datum 型）。入力にエンコード可能な文字が含まれない場合は NULL を返す。

前提条件と不変条件

  • 入力文字列は UTF-8 に変換される必要がある。
  • メモリ管理のため、一時的なメモリコンテキスト（tmp_ctx）内で処理が行われる。
  • 最終的な出力は PostgreSQL の配列形式（ArrayBuildState）として構築される。

アルゴリズムまたは振る舞い

1. **初期化**: 一時的なメモリコンテキストを作成し、処理のスコープを切り替える。 2. **正規化**: 入力文字列を UTF-8 に変換する。 3. **コード生成**: daitch_mokotoff_coding を呼び出し、音韻ルールに基づいてコードを生成する。 4. **結果構築**: 生成されたコードを ArrayBuildState に蓄積する。 5. **終了処理**: メモリコンテキストを元の状態に戻し、一時コンテキストを解放して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: daitch_mokotoff_coding（コアロジック）、および PostgreSQL のメモリ管理・配列操作ユーティリティ（AllocSetContextCreate, initArrayResult, makeArrayResult 等）。

関連データ構造

  • ArrayBuildState: 生成された複数の音韻コードを保持するために使用。
  • MemoryContext: 処理中のメモリ割り当てを管理するために使用。

失敗モードと境界条件

  • 入力文字列にエンコード可能な文字が含まれない場合、daitch_mokotoff_coding が失敗し、関数は NULL を返す。

保守担当者向けメモ

  • 本関数は一時的なメモリコンテキストを使用するため、メモリリークを防ぐために MemoryContextDelete が適切に呼び出されていることを確認すること。
  • アルゴリズムの詳細は daitch_mokotoff_coding および daitch_mokotoff.h に依存している。
