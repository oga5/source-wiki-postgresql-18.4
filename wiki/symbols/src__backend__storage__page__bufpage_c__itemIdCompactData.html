itemIdCompactData

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/page/bufpage.c:436

目的

ページ内のタプル再配置（デフラグメンテーション）処理において、各アイテムの現在の位置情報とサイズを一時的に保持するためのデータ構造です。

シグネチャまたは形状

typedef struct itemIdCompactData
{
	uint16		offsetindex;	/* linp array index */
	int16		itemoff;		/* page offset of item data */
	uint16		alignedlen;		/* MAXALIGN(item data len) */
} itemIdCompactData;

入力と出力

本構造体は関数ではなくデータ型です。主に compactify_tuples などの内部関数において、ページ内の各アイテムをソートまたは移動させる際のメタデータとして使用されます。

前提条件と不変条件

  • offsetindex は、ページヘッダー内のアイテムポインタ（line pointer）配列のインデックスを指します。
  • itemoff は、ページ内における実際のデータ開始位置を示します。
  • alignedlen は、MAXALIGN によってアライメント調整されたデータ長を保持します。

アルゴリズムまたは振る舞い

ページ内の断片化を解消する際、この構造体の配列（itemidbase）を作成し、各アイテムの情報を格納します。その後、この配列を基にタプルの物理的な移動順序を決定し、ページ内の空き領域を統合します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/page/bufpage.c 内の compactify_tuples 等の関数が、ページ内のタプルを整理する際にこの構造体の配列をローカル変数として確保します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • itemIdCompact: itemIdCompactData へのポインタ型（typedef itemIdCompactData *itemIdCompact;）。
  • PageHeaderData: ページヘッダー構造体。本構造体はこのヘッダーが管理するアイテムの情報を一時的にコピーするために使用されます。

失敗モードと境界条件

  • MaxHeapTuplesPerPage または MaxIndexTuplesPerPage を超える数のアイテムを処理しようとした場合、スタック上の配列サイズが不足する可能性があります。

保守担当者向けメモ

  • 本構造体は bufpage.c 内のローカルな最適化用データ構造です。ページ内のタプル移動ロジックを変更する際は、alignedlen の計算が MAXALIGN に準拠していることを維持してください。
