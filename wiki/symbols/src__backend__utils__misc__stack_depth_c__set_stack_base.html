set_stack_base

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/stack_depth.c:43

目的

スタック深さ監視のための基準点（stack_base_ptr）を設定し、スタックオーバーフロー検出機能の初期化または再設定を行う。

シグネチャまたは形状

pg_stack_base_t set_stack_base(void)

入力と出力

  • **入力**: なし
  • **出力**: pg_stack_base_t 型（設定前の stack_base_ptr の値）。

前提条件と不変条件

  • stack_base_ptr は、スタック深さの計算における起点として使用される。
  • 本関数は、スタックの基準位置を更新し、以前の値を戻り値として返すことで、呼び出し側が後で状態を復元できるように設計されている。

アルゴリズムまたは振る舞い

1. 現在の stack_base_ptr の値を old に退避する。 2. コンパイル環境に応じて以下のいずれかの方法で現在のスタック位置を取得し、stack_base_ptr を更新する。

  • HAVE__BUILTIN_FRAME_ADDRESS が定義されている場合: __builtin_frame_address(0) を使用する。
  • それ以外の場合: ローカル変数 stack_base のアドレスを使用する。

    • 3. 退避した old を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/main/main.c:120 (プロセス起動時の初期化)
  • **呼び出し先**:
  • 直接的な関数呼び出しは含まれないが、コンパイル環境により __builtin_frame_address(0) が使用される。

関連データ構造

  • stack_base_ptr: スタック深さ計算の起点となるグローバル変数。
  • pg_stack_base_t: スタック基準点を保持するための型。

失敗モードと境界条件

  • 本関数自体にエラーハンドリングは存在しない。
  • __builtin_frame_address(0) が利用できない環境では、ローカル変数のアドレスをスタックの基準として使用するが、これはコンパイラの最適化によりスタックポインタと厳密に一致しない可能性がある。

保守担当者向けメモ

  • 本関数は、PL/Java 等のバックエンド内で別スレッドを扱う特殊なケースにおいて、restore_stack_base と組み合わせてスタック基準点を一時的に変更するために使用される。
  • 通常のバックエンド処理では、プロセス起動時に一度だけ呼び出されることを想定している。
