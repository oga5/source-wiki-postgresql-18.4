StartupProcSigHupHandler

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:100

目的

起動プロセスにおいて SIGHUP シグナルを受信した際、設定ファイルの再読み込み要求をフラグとして記録し、リカバリ処理を中断・再開させるためのトリガーを引く。

シグネチャまたは形状

static void StartupProcSigHupHandler(SIGNAL_ARGS)
  • SIGNAL_ARGS: シグナルハンドラの標準引数。

入力と出力

  • 入力: シグナル発生時にカーネルから渡される引数。
  • 出力: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • 本関数はシグナルハンドラとして呼び出されるため、非同期シグナル安全な操作のみを行う必要がある。
  • got_SIGHUP フラグが外部（ProcessStartupProcInterrupts 等）から参照可能であること。

アルゴリズムまたは振る舞い

1. グローバル変数 got_SIGHUP を true に設定する。 2. WakeupRecovery() を呼び出し、リカバリ処理の待機状態を解除してシグナル処理を即座に反映させる。

呼び出し元と呼び出し先

  • 呼び出し元: OSのシグナル配送機構（pqsignal(SIGHUP, StartupProcSigHupHandler) により登録）。
  • 呼び出し先: WakeupRecovery()。

関連データ構造

  • got_SIGHUP: 設定再読み込み要求を保持するグローバルフラグ。

失敗モードと境界条件

  • シグナルハンドラ内での実行であるため、複雑な処理やロックの取得は避ける必要がある（WakeupRecovery の実装がこれに準拠していることが前提）。

保守担当者向けメモ

  • 本関数は SIGHUP 受信時のフラグ立てとリカバリプロセスの起床のみを担う。実際の再読み込み処理は ProcessStartupProcInterrupts 内で行われる。
