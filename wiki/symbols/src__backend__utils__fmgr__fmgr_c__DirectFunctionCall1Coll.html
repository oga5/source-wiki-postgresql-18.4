DirectFunctionCall1Coll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:791

目的

引数を1つ取る関数を、FmgrInfo 構造体の初期化を介さずに直接呼び出すための軽量なラッパー関数です。関数呼び出しのオーバーヘッドを最小化します。

シグネチャまたは形状

Datum DirectFunctionCall1Coll(PGFunction func, Oid collation, Datum arg1)

入力と出力

  • **入力**:
  • func: 呼び出す関数のポインタ (PGFunction)。
  • collation: 照合順序 (Oid)。
  • arg1: 関数に渡す引数 (Datum)。
  • **出力**:
  • 関数の実行結果を Datum 型で返します。

前提条件と不変条件

  • 呼び出される関数 func は、NULL を返さないことが期待されます。
  • LOCAL_FCINFO マクロを使用してスタック上に FunctionCallInfo を確保するため、呼び出し元で FmgrInfo を事前に準備する必要はありません。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFO を使用して、引数1つ用の FunctionCallInfoBaseData をスタック上に確保します。 2. InitFunctionCallInfoData を呼び出し、指定された照合順序でコンテキストを初期化します。 3. 引数 arg1 を fcinfo->args[0] に設定し、isnull を false に設定します。 4. 関数ポインタ func を実行します。 5. 実行後、fcinfo->isnull を確認し、関数が NULL を返した場合は elog(ERROR) を発生させます。 6. 正常終了時に結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c
  • src/backend/utils/adt/like.c
  • src/backend/utils/cache/catcache.c
  • src/include/fmgr.h (マクロ経由)
  • **呼び出し先**:
  • 引数として渡された PGFunction 型の関数ポインタ。

関連データ構造

  • FunctionCallInfoBaseData: 実行時の引数やコンテキストを保持する構造体。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • 関数が NULL を返した場合、elog(ERROR) によりエラーが発生します。
  • ポリモーフィック関数など、FmgrInfo の詳細な情報を必要とする関数には使用できません。

保守担当者向けメモ

  • 本関数はパフォーマンスが重要なパスで使用されます。
  • FmgrInfo を必要とする複雑な関数呼び出しには適さないため、用途に応じて fmgr_info() を使用する標準的な呼び出しフローと使い分けてください。
