bit_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:375

目的

BIT 型のデータをクライアントへ送信するためのバイナリ形式に変換します。

シグネチャまたは形状

Datum bit_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（BIT 型の Datum を含む関数呼び出し情報）。
  • **出力**: bytea 形式のバイナリデータ（bytea 型の Datum）。

前提条件と不変条件

  • 入力データは VarBit 構造体として有効である必要があります。
  • 内部的に varbit_send を呼び出すため、varbit_send が要求する不変条件（最後のバイトの未使用ビットがゼロであること等）に従います。

アルゴリズムまたは振る舞い

varbit_send 関数を呼び出すだけのラッパーです。BIT 型のバイナリ送信処理を VARBIT 型の処理と共有しています。

呼び出し元と呼び出し先

  • **呼び出し元**: 不明（PostgreSQL の関数マネージャ経由で呼び出されることが想定されます）。
  • **呼び出し先**: varbit_send

関連データ構造

  • VarBit: ビット列を保持する構造体。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • varbit_send の失敗モードに依存します。

保守担当者向けメモ

  • 本関数は varbit_send のエイリアスです。ロジックの変更が必要な場合は varbit_send を修正してください。
