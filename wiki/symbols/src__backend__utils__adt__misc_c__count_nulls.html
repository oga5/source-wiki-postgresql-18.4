count_nulls

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/misc.c:75

目的

SQL関数（pg_num_nulls および pg_num_nonnulls）の内部実装として、引数リストまたはVARIADIC配列内のNULL値をカウントします。

シグネチャまたは形状

static bool count_nulls(FunctionCallInfo fcinfo, int32 *nargs, int32 *nulls)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • nargs: 引数の総数を格納する出力用ポインタ。
  • nulls: NULLである引数の数を格納する出力用ポインタ。
  • **出力**:
  • bool: 成功した場合は true。VARIADIC引数がNULLの場合は false を返します。

前提条件と不変条件

  • fcinfo は有効な関数呼び出しコンテキストを指している必要があります。
  • VARIADIC引数として渡される値は配列型であると仮定されます。

アルゴリズムまたは振る舞い

1. get_fn_expr_variadic を使用して、VARIADIC配列か個別の引数かを判定します。 2. **VARIADIC配列の場合**:

  • 引数がNULLであれば false を返します。
  • 配列のNULLビットマップを走査し、NULL要素をカウントします。
  • nargs に配列の全要素数、nulls にNULLカウントを格納します。

    • 3. **個別引数の場合**:

  • PG_NARGS() で引数リストをループし、PG_ARGISNULL(i) で各引数を判定してカウントします。
  • nargs に引数総数、nulls にNULLカウントを格納します。

    • 4. 最後に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/misc.c:167 (pg_num_nulls 内)
  • src/backend/utils/adt/misc.c:183 (pg_num_nonnulls 内)
  • **呼び出し先**:
  • get_fn_expr_variadic, PG_NARGS, PG_ARGISNULL, get_base_element_type, get_fn_expr_argtype, PG_GETARG_ARRAYTYPE_P, ARR_NDIM, ARR_DIMS, ArrayGetNItems, ARR_NULLBITMAP

関連データ構造

  • FunctionCallInfo: SQL関数呼び出しのメタデータ。
  • ArrayType: PostgreSQLの配列型内部表現。

失敗モードと境界条件

  • VARIADIC引数としてNULLが渡された場合、関数は false を返し、呼び出し元はこれをNULL結果として処理します。
  • 配列のNULLビットマップが存在しない場合（すべての要素が非NULLとみなされる場合）、NULLカウントは0となります。

保守担当者向けメモ

  • 本関数は pg_num_nulls と pg_num_nonnulls の共通ロジックを抽出したものです。修正時は両方の関数の挙動に影響が出る点に注意してください。
  • VARIADIC引数の型チェックは Assert に依存しているため、呼び出し側のコンテキストが適切であることを前提としています。
