checkcondition_gin

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_bool.c:325

目的

GINインデックスの整合性チェックにおいて、クエリツリー内の各ノード（ITEM）が現在のコンテキストで真であるかを判定するためのコールバック関数です。

シグネチャまたは形状

static bool checkcondition_gin(void *checkval, ITEM *item, void *options);

入力と出力

  • **入力**:
  • checkval: GinChkVal 構造体へのポインタ。評価対象のクエリ状態を保持します。
  • item: 評価対象の ITEM 構造体。
  • options: 使用されていません（void *）。
  • **出力**:
  • bool: 指定された item が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • checkval は GinChkVal 型としてキャスト可能である必要があります。
  • GinChkVal 構造体内の mapped_check 配列が、クエリツリーのノード位置と整合している必要があります。

アルゴリズムまたは振る舞い

1. checkval を GinChkVal 型にキャストします。 2. item のメモリ上の位置と、GinChkVal が保持する first（クエリツリーの先頭アドレス）との差分を計算します。 3. その差分をインデックスとして mapped_check 配列を参照し、対応する真偽値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_bool.c 内の関数（execute 等の評価エンジンからコールバックとして渡される）。
  • src/backend/utils/adt/tsginidx.c において、checkcondition_gin という同名の関数が参照されています（ただし、本ファイル内の定義と同一の役割か、あるいは型シグネチャが異なる別の関数であるかは、提供された情報からは断定できません）。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • ITEM: クエリツリーを構成するノード。
  • GinChkVal: GIN インデックスのチェック時に使用される状態保持構造体。

失敗モードと境界条件

  • item が GinChkVal の管理範囲外のアドレスを指している場合、mapped_check 配列の範囲外アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数は execute 関数等からコールバックとして利用されることを想定しています。
  • src/backend/utils/adt/tsginidx.c からの参照が見受けられますが、これらが同一のシンボルを指しているか、あるいは名前の衝突であるかについては、ソースコード全体のコンテキストを確認する必要があります。
