ProcNumber

定義

  • 種別: typedef int
  • 定義箇所: src/include/storage/procnumber.h:24

目的

PostgreSQLのバックエンドプロセスおよび補助プロセスを一意に識別するための整数型。procarray におけるプロセス配列のインデックスとして機能し、共有メモリ上のプロセス状態へのアクセスや一時リレーションの所有権管理に使用されます。

シグネチャまたは形状

typedef int ProcNumber;

入力と出力

  • 入力: 整数値（プロセス配列のインデックス）。
  • 出力: プロセスを特定するための識別子。

前提条件と不変条件

  • ProcNumber は再利用されます。プロセス終了後に即座に別のバックエンドへ割り当てられる可能性があるため、永続的な識別子として保持してはなりません。
  • システムで許容される最大数は MAX_BACKENDS（MAX_BACKENDS_BITS に基づく制限）に依存します。

アルゴリズムまたは振る舞い

  • procarray.h で管理されるプロセス配列へのインデックスとして参照されます。
  • 一時リレーションの作成や検索時に、セッションの所有権を特定するためのキーとして利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/access/transam/multixact.c: PreparedXactOldestMemberMXactIdSlot 等
  • src/backend/access/transam/parallel.c: parallel_leader_proc_number
  • src/backend/access/transam/twophase.c: locking_backend
  • src/backend/access/transam/xlog.c: walwriterProc
  • src/backend/catalog/catalog.c, namespace.c, storage.c: 一時リレーション管理用変数として使用
  • 呼び出し先: 情報なし。

関連データ構造

  • PGPROC: プロセス管理の詳細構造体（src/include/storage/proc.h）。
  • MAX_BACKENDS: 同時アクティブバックエンド数の上限。

失敗モードと境界条件

  • INVALID_PROC_NUMBER: 一時リレーションに関連付けられていない場合などに使用される無効値。
  • MAX_BACKENDS を超えるインデックスは使用できません。この制限を変更する場合、バッファ参照カウントや無効化処理（inval.c）のビット幅制約をすべて見直す必要があります。

保守担当者向けメモ

  • 本型は純粋な整数型ですが、システム全体でプロセス管理の基盤となるため、変更時は procarray および関連する共有メモリ構造への影響を考慮してください。
  • 永続的な識別子ではない点に注意してください。
