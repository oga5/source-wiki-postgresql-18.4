ECPGdebug

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/misc.c:203

目的

ECPGライブラリのデバッグモードを有効化し、ログの出力先ストリームを設定します。

シグネチャまたは形状

void ECPGdebug(int n, FILE *dbgs)

入力と出力

  • **入力**:
  • n: デバッグレベル。100を超える値が指定された場合、内部回帰テストモードとして扱われます。
  • dbgs: ログ出力先の FILE ポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • debug_init_mutex および debug_mutex を使用して、マルチスレッド環境下での設定変更時の競合を防止します。
  • ログ出力設定の変更中、ecpg_log による出力は一時的にブロックされます。

アルゴリズムまたは振る舞い

1. debug_init_mutex をロックし、他の ECPGdebug 呼び出しとの競合を防ぎます。 2. debug_mutex をロックし、設定変更中のログ出力を抑止します。 3. n が 100 を超える場合、ecpg_internal_regression_mode を true に設定し、simple_debug に n - 100 を代入します。それ以外の場合は simple_debug に n を代入します。 4. debugstream に dbgs を割り当てます。 5. debug_mutex を解放します。 6. ecpg_log を呼び出し、設定されたデバッグレベルをログ出力します。 7. debug_init_mutex を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/ecpg.c（マクロ定義経由）
  • 各種テストコード（例: compat_informix-dec_test.c, compat_informix-rnull.c 等）
  • **呼び出し先**:
  • ecpg_log（デバッグ設定完了の通知用）
  • pthread_mutex_lock / pthread_mutex_unlock（排他制御用）

関連データ構造

  • simple_debug（グローバル変数: デバッグレベル）
  • debugstream（グローバル変数: 出力先ストリーム）
  • ecpg_internal_regression_mode（グローバル変数: 回帰テストモードフラグ）
  • debug_init_mutex, debug_mutex（排他制御用ミューテックス）

失敗モードと境界条件

  • dbgs に無効なファイルポインタが渡された場合の挙動は、標準ライブラリの fprintf（ecpg_log 内部で使用）の仕様に依存します。
  • 非常に頻繁な呼び出しは、ミューテックスの競合によりパフォーマンスに影響を与える可能性があります。

保守担当者向けメモ

  • 本関数はスレッドセーフです。
  • ecpg_log 内で malloc が行われるため、ログ出力が頻発する環境ではメモリ消費量に注意してください。
  • 100 を超える値による回帰テストモードの切り替えは、既存のテストスイートとの互換性を維持するために重要です。
