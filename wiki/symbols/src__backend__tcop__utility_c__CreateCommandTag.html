CreateCommandTag

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/utility.c:2361

目的

与えられた SQL パースツリー（Node *parsetree）に基づき、対応する CommandTag を決定する。

シグネチャまたは形状

CommandTag CreateCommandTag(Node *parsetree)

入力と出力

  • **入力**: Node *parsetree（解析済みの SQL 文を表すノード構造体）。
  • **出力**: CommandTag（コマンドの種類を識別する列挙型）。

前提条件と不変条件

  • 入力された parsetree は、nodeTag() によって識別可能な有効なノードである必要がある。
  • T_RawStmt が渡された場合、内部の stmt フィールドに対して再帰的に処理を行う。

アルゴリズムまたは振る舞い

1. nodeTag(parsetree) を使用してノードの種類を判定する。 2. T_RawStmt の場合は、内部のステートメントを取り出して再帰的に呼び出す。 3. T_InsertStmt, T_SelectStmt 等の標準的なクエリノードに対しては、対応する CMDTAG_* を割り当てる。 4. T_TransactionStmt 等の複雑なノードについては、内部の kind フィールド（例: TRANS_STMT_BEGIN）を評価し、詳細なタグを決定する。 5. T_ClosePortalStmt や T_FetchStmt のように、ノード内のフラグ（portalname の有無や ismove）に応じてタグを分岐させる。 6. 該当するタグが見つからない場合は CMDTAG_UNKNOWN を返す。

呼び出し元と呼び出し先

  • src/backend/commands/event_trigger.c

  • **呼び出し先**:
  • 本関数は再帰的に自身を呼び出す（T_RawStmt の場合）。

関連データ構造

  • CommandTag: コマンド種別を保持する列挙型。
  • Node: SQL パースツリーの基底構造体。
  • RawStmt, TransactionStmt, ClosePortalStmt, FetchStmt: 各種ステートメント固有の構造体。

失敗モードと境界条件

  • 未知のノードタイプが渡された場合、switch 文の default 節がないケースでは未定義の挙動となる可能性がある（一部のステートメント種別では CMDTAG_UNKNOWN を返す）。
  • parsetree が NULL の場合の挙動については、本ソース抜粋内では明示されていない。

保守担当者向けメモ

  • 新しい SQL コマンドを追加した際は、本関数の switch 文に適切なケースを追加し、正しい CommandTag を返すように更新する必要がある。
  • T_RawStmt の再帰処理により、パーサーが生成する生のステートメントと、実行時に使用されるステートメントの両方に対応している。
