r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_dutch.c:245

目的

オランダ語のステミング処理において、語幹抽出の基準となる領域（R1, R2）を計算し、SN_env 構造体の内部インデックスに格納する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （ステミング環境の状態を保持する構造体）
  • **出力**: 成功時に 1、失敗時に 0 を返す。
  • **副作用**: z->I[0], z->I[1], z->I[2] にそれぞれ計算された領域境界位置が書き込まれる。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。

アルゴリズムまたは振る舞い

1. z->I[2]（R1の開始位置）と z->I[1]（R2の開始位置）を文字列長 z->l で初期化する。 2. 単語の先頭から3文字目以降を z->I[0] に設定する。 3. 母音（g_v）の直後の子音を探し、その位置を z->I[2]（R1）とする。もし z->I[2] が z->I[0] より小さい場合は、z->I[0] を採用する。 4. さらにその後の母音の直後の子音を探し、その位置を z->I[1]（R2）とする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_ISO_8859_1_dutch.c 内のメインステミング関数から呼び出される。
  • **呼び出し先**: out_grouping および in_grouping （Snowball ランタイム関数）。

関連データ構造

  • struct SN_env: ステミングのカーソル位置や領域情報を保持する。
  • g_v: 母音を定義するビットマップ配列。

失敗モードと境界条件

  • out_grouping または in_grouping が負の値を返した場合、処理を中断し 0 を返す。
  • 文字列が短く、領域を特定できない場合は初期値（文字列長）が維持される。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接編集せず元の .sbl ファイルを修正すること。
  • z->I 配列のインデックス（0, 1, 2）は Snowball の内部仕様に依存しているため、他の関数との整合性に注意が必要。
