TSVectorBuildState

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/to_tsany.c:38

目的

JSON や JSONB などの入力データを tsvector へ変換する過程において、解析済みテキストの状態と全文検索設定（Config）を保持するために使用されます。

シグネチャまたは形状

typedef struct TSVectorBuildState
{
	ParsedText *prs;
	Oid			cfgId;
} TSVectorBuildState;

入力と出力

本構造体自体はデータ保持用であり、関数呼び出しの引数や戻り値として受け渡されます。

  • prs: 解析済みテキスト（ParsedText）へのポインタ。
  • cfgId: 全文検索設定を識別する OID。

前提条件と不変条件

  • prs は parsetext 等によって適切に初期化された ParsedText 構造体を指している必要があります。
  • cfgId は有効な全文検索設定の OID である必要があります。

アルゴリズムまたは振る舞い

JSON/JSONB の走査処理において、抽出された各要素を ParsedText に蓄積し、最終的に make_tsvector を用いて tsvector 型へ変換するためのコンテキストとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/to_tsany.c 内の jsonb_to_tsvector_worker 等の JSON 変換関数が、ローカル変数として本構造体をインスタンス化し、処理のコンテキストとして利用しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ParsedText: 解析されたトークン列を保持する構造体。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • prs が NULL の状態でアクセスした場合、セグメンテーション違反が発生します。
  • cfgId が無効な場合、後続の辞書適用やトークン正規化処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は to_tsany.c 内の JSON/JSONB 変換処理に特化しています。
  • ParsedText のメモリ管理は parsetext 側の責務であるため、本構造体を使用する際はライフサイクルに注意してください。
