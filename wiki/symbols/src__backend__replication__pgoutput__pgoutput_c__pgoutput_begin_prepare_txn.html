pgoutput_begin_prepare_txn

定義

  • **種別**: function
  • **定義箇所**: src/backend/replication/pgoutput/pgoutput.c:661

目的

論理レプリケーションにおいて、2フェーズコミット（2PC）の準備段階（PREPARE）を開始するためのトランザクション情報を出力ストリームに書き込みます。

シグネチャまたは形状

static void pgoutput_begin_prepare_txn(LogicalDecodingContext *ctx, ReorderBufferTXN *txn)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • txn: 準備対象となるトランザクション情報（ReorderBufferTXN）。
  • **出力**: なし（副作用として出力ストリームへ書き込み）。

前提条件と不変条件

  • txn が有効なトランザクション情報を含んでいること。
  • ctx が論理デコーディングの出力プラグインとして初期化されていること。

アルゴリズムまたは振る舞い

1. トランザクションのレプリケーションオリジン情報（origin_id）を確認し、送信の要否を判定します。 2. OutputPluginPrepareWrite を呼び出し、出力バッファを準備します。 3. logicalrep_write_begin_prepare を呼び出し、トランザクションの準備開始メッセージをプロトコル形式で書き込みます。 4. send_repl_origin を呼び出し、レプリケーションオリジン情報を送信します（必要な場合のみ）。 5. OutputPluginWrite を呼び出し、構築したメッセージをストリームへフラッシュします。

呼び出し元と呼び出し先

  • **呼び出し元**: _PG_output_plugin_init 内でコールバックとして登録されており、論理デコーディングのフレームワークから呼び出されます。
  • **呼び出し先**:
  • OutputPluginPrepareWrite
  • logicalrep_write_begin_prepare
  • send_repl_origin
  • OutputPluginWrite

関連データ構造

  • LogicalDecodingContext: デコーディングのセッション状態を保持。
  • ReorderBufferTXN: トランザクションのメタデータ（LSN、XID、オリジン情報など）を保持。

失敗モードと境界条件

  • レプリケーションオリジンが設定されていない場合、その情報は送信されません。
  • メモリ不足や出力ストリームの書き込みエラーが発生した場合、基盤となる OutputPlugin 関数群によってエラーが報告されます。

保守担当者向けメモ

  • 本関数は2フェーズコミットの「PREPARE」フェーズ専用です。通常のコミット処理とは異なるプロトコルメッセージが使用される点に注意してください。
