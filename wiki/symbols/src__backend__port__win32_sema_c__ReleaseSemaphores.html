ReleaseSemaphores

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_sema.c:63

目的

プロセス終了時に、確保されているすべてのWin32セマフォハンドルをクローズし、管理用メモリを解放する。

シグネチャまたは形状

static void ReleaseSemaphores(int code, Datum arg)

入力と出力

  • code: on_shmem_exit コールバックから渡される終了ステータスコード。
  • arg: on_shmem_exit コールバックから渡される補助引数。
  • 戻り値: なし。

前提条件と不変条件

  • numSems が現在確保されているセマフォの数として正しく保持されていること。
  • mySemSet が PGReserveSemaphores によって割り当てられた有効なメモリ領域であること。

アルゴリズムまたは振る舞い

1. 0 から numSems - 1 までの範囲でループを実行する。 2. mySemSet 配列に格納された各セマフォハンドルに対して CloseHandle() を呼び出し、カーネルオブジェクトを解放する。 3. 最後に free(mySemSet) を呼び出し、ハンドル配列自体を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: on_shmem_exit を介してプロセス終了時に呼び出される（src/backend/port/win32_sema.c:55）。
  • 呼び出し先: CloseHandle (Win32 API), free (標準ライブラリ)。

関連データ構造

  • mySemSet: 現在のプロセスで確保されたセマフォハンドルを保持する静的配列。
  • numSems: 確保されているセマフォの総数。

失敗モードと境界条件

  • mySemSet が初期化されていない、または numSems が不正な値である場合、メモリ破壊や不正なハンドル操作が発生する可能性がある。

保守担当者向けメモ

  • 本関数はWindows環境専用の実装である。POSIXやSystem V環境では同名の関数が存在するが、実装内容が異なるため混同しないこと。
  • on_shmem_exit に登録されるクリーンアップ処理であるため、シグナルハンドラ等からの非同期呼び出しには適さない。
