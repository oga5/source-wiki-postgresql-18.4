BTCallbackState

定義

  • 種別: struct
  • 定義箇所: contrib/amcheck/verify_nbtree.c:168

目的

B-Tree インデックスの検証プロセスにおいて、検証の実行モードやオプションを保持し、コールバック関数へ状態を伝達するために使用される構造体です。

シグネチャまたは形状

typedef struct BTCallbackState
{
	bool		parentcheck;    /* 親子関係の整合性チェックを行うか */
	bool		heapallindexed; /* ヒープとインデックスの完全同期検証を行うか */
	bool		rootdescend;    /* ルートからの降下検証を行うか */
	bool		checkunique;    /* 一意性制約の検証を行うか */
} BTCallbackState;

入力と出力

本構造体はデータ保持用であり、関数としての入力・出力は持ちません。検証処理の各フェーズにおいて、検証オプションを制御するためのフラグとして参照されます。

前提条件と不変条件

  • 各フィールドは bool 型であり、検証実行時に true または false のいずれかの状態をとります。
  • bt_index_check および bt_index_parent_check 関数内でインスタンス化され、検証ロジックの動作を決定づけます。

アルゴリズムまたは振る舞い

検証実行時に、この構造体のインスタンスが state として渡されます。各フラグの状態に応じて、以下の処理が分岐します。

  • parentcheck: 有効な場合、インデックスの親ページと子ページのリンク整合性を検証します。
  • heapallindexed: 有効な場合、インデックス内の全タプルがヒープに存在するかを Bloom フィルタを用いて検証します。
  • rootdescend: 有効な場合、ルートページからの降下による構造検証を行います。
  • checkunique: 有効な場合、インデックスの一意性制約違反をチェックします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c 内の bt_index_check および bt_index_parent_check 関数でインスタンス化されます。
  • **呼び出し先**:
  • bt_index_check_callback 関数内で、void *state 引数から BTCallbackState * へキャストされて参照されます。

関連データ構造

  • BtreeCheckState: インデックス検証全体の詳細な状態を保持する構造体。

失敗モードと境界条件

  • 本構造体自体はデータ保持用であるため、構造体単体での失敗モードはありません。
  • 適切に初期化されていない状態でコールバック関数に渡された場合、検証ロジックが意図しない動作（フラグの誤判定）を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は bt_index_check_callback への引数として使用されます。新しい検証オプションを追加する場合は、本構造体にフィールドを追加し、コールバック関数側でのキャストと参照を更新してください。
