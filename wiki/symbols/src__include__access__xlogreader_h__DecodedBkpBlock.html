DecodedBkpBlock

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogreader.h:119

目的

WALレコードに含まれる個別のブロック参照（Backup Block）の情報を保持・デコードするための構造体です。

シグネチャまたは形状

typedef struct {
    bool        in_use;
    RelFileLocator rlocator;
    ForkNumber  forknum;
    BlockNumber blkno;
    Buffer      prefetch_buffer;
    uint8       flags;
    bool        has_image;
    bool        apply_image;
    char       *bkp_image;
    uint16      hole_offset;
    uint16      hole_length;
    uint16      bimg_len;
    uint8       bimg_info;
    bool        has_data;
    char       *data;
    uint16      data_len;
    uint16      data_bufsz;
} DecodedBkpBlock;

入力と出力

  • **入力**: WALレコードの解析過程において、XLogRecordBlockHeader 等から抽出されたバイナリデータが各フィールドに格納されます。
  • **出力**: XLogReaderState を通じて、リカバリ処理や pg_waldump 等のツールがブロックのメタデータ、フルページイメージ、およびリソースマネージャ固有のデータにアクセスするために使用されます。

前提条件と不変条件

  • in_use が true の場合のみ、当該ブロック参照が有効であることを示します。
  • has_image が true の場合、bkp_image に有効なフルページイメージデータが含まれます。
  • has_data が true の場合、data にリソースマネージャ固有のデータが含まれます。

アルゴリズムまたは振る舞い

WALレコードのデコード時、XLogReaderState 内の blocks 配列（DecodedBkpBlock の集合）が順次埋められます。フルページイメージが存在する場合、そのオフセットや長さ（hole_offset, hole_length）に基づき、データの復元や検証が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogreader.c（レコードデコード処理）
  • src/backend/access/transam/xlogprefetcher.c（プリフェッチ処理）
  • contrib/pg_walinspect/pg_walinspect.c（WAL解析ツール）
  • **呼び出し先**: 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • DecodedXLogRecord: 本構造体を配列として保持し、レコード全体の情報を管理します。
  • RelFileLocator, ForkNumber, BlockNumber: ブロックの物理的な位置を特定するために使用されます。

失敗モードと境界条件

  • XLR_MAX_BLOCK_ID を超えるブロック参照はサポートされません。
  • data_bufsz を超えるデータアクセスは、バッファオーバーフローを避けるため、必ず data_len を確認する必要があります。

保守担当者向けメモ

  • 本構造体は XLogReaderState の一部として動的に確保されることが多いため、メモリレイアウトの変更には注意してください。
  • DecodedXLogRecord 内の blocks 配列として使用される際は、FLEXIBLE_ARRAY_MEMBER を用いた定義が xlogreader.h 内でなされています。
