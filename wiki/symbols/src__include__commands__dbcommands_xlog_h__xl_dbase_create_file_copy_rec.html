xl_dbase_create_file_copy_rec

定義

  • 種別: struct
  • 定義箇所: src/include/commands/dbcommands_xlog.h:29

目的

データベース作成操作（ファイルコピー方式）を WAL（Write-Ahead Logging）に記録するための構造体です。クラッシュリカバリやレプリケーション時に、物理的なデータベース作成操作を再実行（Redo）するために使用されます。

シグネチャまたは形状

typedef struct xl_dbase_create_file_copy_rec
{
    Oid            db_id;
    Oid            tablespace_id;
    Oid            src_db_id;
    Oid            src_tablespace_id;
} xl_dbase_create_file_copy_rec;

入力と出力

  • **入力**: データベース作成時に、新規データベースの識別子（db_id, tablespace_id）およびコピー元となるデータベースの識別子（src_db_id, src_tablespace_id）が本構造体に格納されます。
  • **出力**: WAL レコードとして書き込まれ、リカバリ時に dbase_redo 関数によって読み取られます。

前提条件と不変条件

  • 本構造体は、データベース作成戦略として「ファイルコピー方式」が選択された場合に使用されます。
  • 各フィールドには有効な Oid が設定されている必要があります。

アルゴリズムまたは振る舞い

1. **書き込み**: src/backend/commands/dbcommands.c がデータベース作成時に本構造体を初期化し、XLogInsert を介して WAL に書き込みます。 2. **読み込み**: リカバリプロセスにおいて、XLogReader が WAL から本レコードを読み取ります。 3. **適用**: dbase_redo 関数が本構造体の情報を参照し、指定されたコピー元から物理ファイルをコピーしてデータベースを再構築します。

呼び出し元と呼び出し先

  • **呼び出し元（書き込み）**: src/backend/commands/dbcommands.c
  • **呼び出し先（読み込み・処理）**: src/backend/access/rmgrdesc/dbasedesc.c（デバッグ出力）、src/backend/commands/dbcommands.c（dbase_redo による適用）、src/backend/access/transam/xlogprefetcher.c、src/backend/postmaster/walsummarizer.c

関連データ構造

  • XLogReaderState（WAL 読み込み用）
  • dbase_redo（本構造体を処理するリソースマネージャ関数）

失敗モードと境界条件

  • 構造体内の Oid が無効な場合、リカバリ時のファイルコピー操作が失敗する可能性があります。
  • ファイルシステム上の権限不足やディスク容量不足により、dbase_redo による再実行が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は固定長です。可変長データを含む他の WAL レコード（xl_dbase_drop_rec 等）とは扱いが異なる点に注意してください。
  • ファイルコピー方式と WAL ログ方式（xl_dbase_create_wal_log_rec）でリカバリ時の挙動が異なるため、修正時は両方の戦略への影響を確認してください。
