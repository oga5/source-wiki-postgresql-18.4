astreamer_lz4_decompressor_content

定義

  • **種別**: 関数
  • **定義箇所**: src/fe_utils/astreamer_lz4.c:311

目的

LZ4形式で圧縮されたデータストリームを伸長し、その結果をパイプライン内の後続ストリーマーへ転送する。

シグネチャまたは形状

static void astreamer_lz4_decompressor_content(
    astreamer *streamer,
    astreamer_member *member,
    const char *data,
    int len,
    astreamer_archive_context context
)

入力と出力

  • **入力**:
  • streamer: 伸長状態を保持する astreamer_lz4_frame 構造体。
  • data, len: 伸長対象の圧縮データとその長さ。
  • member, context: ストリーム処理のメタデータおよびアーカイブコンテキスト。
  • **出力**: なし（副作用として後続ストリーマーの astreamer_content を呼び出す）。

前提条件と不変条件

  • streamer は astreamer_lz4_frame 型としてキャスト可能であること。
  • mystreamer->dctx（LZ4伸長コンテキスト）が初期化済みであること。
  • 内部バッファ bbs_buffer が伸長データを保持するのに十分な容量を持つこと。

アルゴリズムまたは振る舞い

1. 入力データ data を LZ4F_decompress に渡し、内部バッファ bbs_buffer へ伸長結果を書き込む。 2. LZ4F_decompress の戻り値がエラーを示す場合は pg_fatal で終了する。 3. 伸長されたバイト数に応じて bytes_written を更新する。 4. 内部バッファが満杯になった場合、astreamer_content を呼び出して後続のストリーマーへデータを転送し、バッファをリセットする。 5. 入力データが完全に消費されるまで、上記処理をループする。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer インターフェースの .content メンバを通じて呼び出される（直接的な呼び出し元は astreamer_lz4.c 内の構造体初期化箇所で参照されている）。
  • **呼び出し先**:
  • LZ4F_decompress（LZ4ライブラリ関数）
  • astreamer_content（後続ストリーマーへのデータ転送）
  • pg_fatal（エラー発生時）

関連データ構造

  • astreamer_lz4_frame: 伸長コンテキスト (dctx) およびバッファ状態を保持する構造体。

失敗モードと境界条件

  • **伸長エラー**: LZ4F_isError が真となる場合、pg_fatal によりプロセスが終了する。
  • **バッファ境界**: bytes_written が bbs_buffer.maxlen に達した際に、バッファのフラッシュとリセットが行われる。

保守担当者向けメモ

  • 本関数は USE_LZ4 が定義されている環境でのみコンパイルされる。
  • LZ4F_decompress の read_size および out_size は、呼び出し前後で消費・生成されたバイト数を示す双方向パラメータとして機能するため、ループ内での更新ロジックに注意すること。
