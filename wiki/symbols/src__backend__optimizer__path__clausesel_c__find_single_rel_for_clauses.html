find_single_rel_for_clauses

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/clausesel.c:522

目的

与えられた条件リスト（clauses）がすべて同一の単一リレーションを参照しているかを判定し、該当する RelOptInfo を取得します。主に拡張統計（Extended Statistics）の適用可否を判断するために使用されます。

シグネチャまたは形状

RelOptInfo *find_single_rel_for_clauses(PlannerInfo *root, List *clauses)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • clauses: RestrictInfo（または AND 句）のリスト。
  • **出力**:
  • すべての条件が単一のリレーションに属する場合、その RelOptInfo を返します。
  • 条件が複数のリレーションにまたがる場合、または条件が存在しない場合は NULL を返します。

前提条件と不変条件

  • 入力リスト内の各要素は、原則として RestrictInfo である必要があります。
  • AND 句（BoolExpr）については再帰的に処理されます。
  • 変数を含まない定数のみの条件は無視されます。

アルゴリズムまたは振る舞い

1. リスト内の各条件を走査します。 2. AND 句の場合は再帰的に find_single_rel_for_clauses を呼び出し、その結果を統合します。 3. RestrictInfo の場合、clause_relids を確認します。

  • 空（定数のみ）であればスキップします。
  • 単一のリレーションIDのみを含む場合、それを記録します。
  • 複数のリレーションIDを含む場合、または以前の条件と異なるリレーションIDである場合は NULL を返します。

    • 4. すべての条件が同一リレーションを参照していることが確認できた場合、find_base_rel を用いて対応する RelOptInfo を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/clausesel.c 内の複数箇所（主に選択率計算の初期段階）。
  • **呼び出し先**:
  • find_base_rel（リレーションIDから RelOptInfo を取得）
  • find_single_rel_for_clauses（自身を再帰呼び出し）

関連データ構造

  • RelOptInfo: リレーションの統計情報やパス情報を保持する構造体。
  • RestrictInfo: WHERE 句などの条件式をラップし、メタデータ（参照リレーション等）を付与した構造体。
  • PlannerInfo: クエリ全体のプランニング状態を保持する構造体。

失敗モードと境界条件

  • **複数リレーション**: 条件リストが複数のテーブルにまたがる場合、NULL を返します。
  • **非 RestrictInfo**: RestrictInfo 以外のノードが含まれる場合（AND 句を除く）、NULL を返します。
  • **空リスト**: 条件が一つも存在しない場合、NULL を返します。

保守担当者向けメモ

  • 本関数は拡張統計の適用判断に直結します。AND 句以外の複雑な式（OR や NOT など）に対する挙動は、現在の実装では RestrictInfo ではないと判断され NULL を返す仕様です。
  • RestrictInfo 以外のノードに対する拡張統計の適用は現在サポートされていないため、この制約は意図的なものです。
