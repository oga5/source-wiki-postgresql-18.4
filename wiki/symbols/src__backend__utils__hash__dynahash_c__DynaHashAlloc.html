DynaHashAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/hash/dynahash.c:290

目的

ハッシュテーブルの内部構造体や要素を割り当てるための、動的ハッシュ専用のメモリ割り当てラッパー関数です。

シグネチャまたは形状

static void *DynaHashAlloc(Size size)

入力と出力

  • **入力**: size (Size) - 割り当てるメモリサイズ。
  • **出力**: void * - 割り当てられたメモリ領域へのポインタ。

前提条件と不変条件

  • CurrentDynaHashCxt が有効なメモリコンテキストであることを前提とします（Assert による検証）。

アルゴリズムまたは振る舞い

1. CurrentDynaHashCxt が有効か検証します。 2. MemoryContextAllocExtended を呼び出し、CurrentDynaHashCxt から指定されたサイズのメモリを確保します。 3. MCXT_ALLOC_NO_OOM フラグを指定し、メモリ不足時にエラーを発生させない動作を強制します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/hash/dynahash.c 内で、ハッシュテーブルの割り当て関数として hashp->alloc に代入されます。
  • 同ファイル内の複数の箇所で、hashp->alloc が本関数であることを検証する Assert に使用されています。
  • **呼び出し先**:
  • MemoryContextAllocExtended（PostgreSQL のメモリ管理サブシステム）。

関連データ構造

  • HTAB: ハッシュテーブルの制御構造体。alloc フィールドを通じて本関数を参照します。

失敗モードと境界条件

  • MCXT_ALLOC_NO_OOM フラグにより、メモリ割り当て失敗時には NULL が返される可能性があります。

保守担当者向けメモ

  • 本関数は static であり、主に HTAB 構造体の関数ポインタ経由で利用されます。
  • メモリコンテキストの切り替えが必要な場合は、CurrentDynaHashCxt の状態を適切に管理する必要があります。
