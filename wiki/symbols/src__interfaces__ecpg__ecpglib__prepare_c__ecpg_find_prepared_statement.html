ecpg_find_prepared_statement

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:238

目的

指定された名前を持つプリペアドステートメントを、特定のデータベース接続に関連付けられたリストから検索する。

シグネチャまたは形状

struct prepared_statement *ecpg_find_prepared_statement(
    const char *name,
    struct connection *con,
    struct prepared_statement **prev_
);

入力と出力

  • **入力**:
  • name: 検索対象のプリペアドステートメント名。
  • con: 検索対象の接続オブジェクト。
  • prev_: 検索対象の直前の要素を格納するためのポインタ（不要な場合は NULL を指定可能）。
  • **出力**:
  • 見つかった場合は prepared_statement 構造体へのポインタを返す。
  • 見つからない場合は NULL を返す。

前提条件と不変条件

  • con は有効な接続オブジェクトであること。
  • con->prep_stmts はプリペアドステートメントの連結リストの先頭を指していること。

アルゴリズムまたは振る舞い

1. con->prep_stmts から開始し、リストを順次走査する。 2. 各要素の name と引数の name を strcmp で比較する。 3. 一致した場合:

  • prev_ が NULL でなければ、その要素の直前の要素（prev）を *prev_ に代入する。
  • 見つかった要素へのポインタを返す。

    • 4. リストの末尾に到達しても一致しない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • src/interfaces/ecpg/ecpglib/prepare.c（複数箇所）
  • **呼び出し先**:
  • strcmp（標準ライブラリ）

関連データ構造

  • prepared_statement: プリペアドステートメントを保持する構造体。
  • connection: データベース接続情報を保持する構造体。

失敗モードと境界条件

  • リストが空の場合、直ちに NULL を返す。
  • 検索対象がリストの先頭にある場合、prev_ には NULL が設定される。
  • 検索対象が存在しない場合、prev_ の値は更新されない。

保守担当者向けメモ

  • 本関数はリストの線形探索を行うため、ステートメント数が多い場合には性能に影響を与える可能性がある。
  • prev_ を介したポインタ操作は、呼び出し側でリストからの削除処理を行う際に利用される。
