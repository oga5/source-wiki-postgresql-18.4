deparse_namespace

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/ruleutils.c:162

目的

クエリの逆コンパイル（deparse）処理において、特定のクエリやプランツリー内の名前空間（RangeTblEntry、CTE、サブプラン、列エイリアスなど）を管理し、SQL生成時の名前解決を支援する。

シグネチャまたは形状

typedef struct {
    List *rtable;
    List *rtable_names;
    List *rtable_columns;
    List *subplans;
    List *ctes;
    AppendRelInfo **appendrels;
    char *ret_old_alias;
    char *ret_new_alias;
    bool unique_using;
    List *using_names;
    Plan *plan;
    List *ancestors;
    Plan *outer_plan;
    Plan *inner_plan;
    List *outer_tlist;
    List *inner_tlist;
    List *index_tlist;
    char *funcname;
    int numargs;
    char **argnames;
} deparse_namespace;

入力と出力

本構造体は関数引数として渡され、逆コンパイルの過程で各フィールドが更新される。

  • **入力**: Query 構造体や Plan ツリーから抽出された情報。
  • **出力**: 逆コンパイルされた SQL 文字列を生成するための参照情報（エイリアス名や列名など）。

前提条件と不変条件

  • rtable、rtable_names、rtable_columns は互いに対応する並列リストである必要がある。
  • プランツリーを逆コンパイルする場合、plan 関連のフィールドが適切に設定されている必要がある。

アルゴリズムまたは振る舞い

  • クエリ内の各 RangeTblEntry に対して一意なエイリアスを割り当てる。
  • USING 句で使用される列名の競合を解決する。
  • 逆コンパイル中に、現在のスコープにおける変数（Var）がどのテーブルやサブプランに属するかを解決するための参照先を提供する。

呼び出し元と呼び出し先

  • **呼び出し元**: ruleutils.c 内の set_rtable_names, set_deparse_for_query, set_simple_column_names, has_dangerous_join_using, set_using_names, set_relation_column_names, set_join_column_names, colname_is_unique, make_colname_unique など。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはない。

関連データ構造

  • RangeTblEntry: rtable リストの要素。
  • deparse_columns: rtable_columns リストの要素。
  • Plan: プランツリーのノード。
  • Query: 逆コンパイル対象のクエリツリー。

失敗モードと境界条件

  • 逆コンパイル対象のクエリが複雑で名前の競合が解決できない場合、生成される SQL が不正になる可能性がある。
  • appendrels が NULL の場合、継承関係の解決は行われない。

保守担当者向けメモ

  • 本構造体は ruleutils.c 内で広範囲に使用されており、フィールドの追加や変更は逆コンパイルロジック全体に影響を与える。
  • plan 関連のフィールドは、プランツリーの逆コンパイル時のみ使用されるため、通常のクエリ逆コンパイル時には無視される。
