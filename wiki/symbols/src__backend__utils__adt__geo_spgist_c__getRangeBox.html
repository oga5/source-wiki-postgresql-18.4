getRangeBox

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_spgist.c:156

目的

PostgreSQLの幾何データ型 BOX を、SP-GiSTインデックスで利用可能な内部表現である RangeBox 構造体に変換します。

シグネチャまたは形状

static RangeBox *getRangeBox(BOX *box)

入力と出力

  • **入力**: BOX *box - 変換対象の矩形データ。
  • **出力**: RangeBox * - メモリ確保された RangeBox 構造体へのポインタ。

前提条件と不変条件

  • palloc を使用してメモリを確保するため、呼び出し元は必要に応じてメモリ管理を行う必要があります。
  • 入力 box は有効な BOX 構造体を指している必要があります。

アルゴリズムまたは振る舞い

1. palloc を用いて RangeBox サイズのメモリを確保します。 2. 入力 box の low.x と high.x を range_box->left（低域と高域）に代入します。 3. 入力 box の low.y と high.y を range_box->right（低域と高域）に代入します。 4. 構築した range_box を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/geo_spgist.c:611 (spg_box_quad_picksplit 内で prefixDatum から変換)
  • src/backend/utils/adt/geo_spgist.c:617 (spg_box_quad_picksplit 内で各クエリ矩形を変換)
  • **呼び出し先**:
  • palloc (PostgreSQLメモリ管理関数)

関連データ構造

  • BOX: utils/geo_decls.h で定義される2次元矩形構造体。
  • RangeBox: 本モジュール内で使用される、4次元空間（x_low, x_high, y_low, y_high）を表現するための構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、処理が中断されます。

保守担当者向けメモ

  • 本関数は palloc を使用するため、呼び出し元で pfree が適切に行われているか確認してください。
  • RangeBox 構造体の定義変更に伴い、メンバへの代入ロジックを更新する必要があります。
