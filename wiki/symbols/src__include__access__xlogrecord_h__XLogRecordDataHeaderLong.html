XLogRecordDataHeaderLong

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogrecord.h:221

目的

WAL（Write-Ahead Logging）レコードにおいて、データ長が短い場合に用いられる XLogRecordDataHeaderShort と対比される、より大きなデータサイズを扱うためのヘッダー構造を提供します。

シグネチャまたは形状

typedef struct XLogRecordDataHeaderLong
{
    uint8       id;             /* XLR_BLOCK_ID_DATA_LONG */
    /* followed by uint32 data_length, unaligned */
} XLogRecordDataHeaderLong;

入力と出力

  • **入力**: WALレコードのデータセクションの先頭に配置される識別子およびデータ長情報。
  • **出力**: 本構造体自体はデータ構造の定義であり、後続する uint32 型のデータ長フィールドを指し示すための基点として機能します。

前提条件と不変条件

  • id フィールドには XLR_BLOCK_ID_DATA_LONG が格納されます。
  • 本構造体の直後に uint32 型のデータ長が配置されます。
  • 構造体自体および後続のデータは整列（アライメント）されていません。メモリ上で操作する際は、整列されたバッファにコピーしてからアクセスする必要があります。

アルゴリズムまたは振る舞い

WALレコードの解析時、XLogReader はこのヘッダーを読み取り、id が XLR_BLOCK_ID_DATA_LONG であることを確認した上で、直後の uint32 値からデータペイロードの長さを特定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogreader.c:1739 において、WALレコードの解析処理中に本構造体への参照が確認されています。
  • **呼び出し先**: 本構造体はデータ構造の定義であり、関数呼び出しを行うものではありません。

関連データ構造

  • XLogRecordDataHeaderShort: データ長が短い場合に使用される代替構造体。
  • XLogRecord: WALレコード全体の固定長ヘッダー。
  • XLogRecordBlockHeader: レコードに関連付けられたブロックのメタデータ。

失敗モードと境界条件

  • 本構造体はアライメントされていないため、直接ポインタキャストしてアクセスすると、アーキテクチャによっては不正なメモリアクセスが発生する可能性があります。
  • data_length の値が実際のデータサイズと一致しない場合、WALレコードのデコード時に整合性エラーが発生します。

保守担当者向けメモ

  • 本構造体はコード内で直接インスタンス化して使用されることは稀ですが、WALの物理フォーマットを定義する重要なリファレンスです。
  • 構造体定義の変更は、WALの互換性に直接影響するため注意が必要です。
