make_notclause

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/makefuncs.c:758

目的

論理否定演算を表す BoolExpr ノードを生成し、初期化します。

シグネチャまたは形状

Expr *make_notclause(Expr *notclause)

入力と出力

  • **入力**: notclause (否定対象となる Expr へのポインタ)
  • **出力**: 構築された BoolExpr 構造体へのポインタ（Expr * 型として返却）

前提条件と不変条件

  • makeNode(BoolExpr) により、現在のメモリコンテキストにノードが割り当てられます。
  • 生成されたノードの boolop フィールドは常に NOT_EXPR に設定されます。
  • location フィールドはデフォルト値として -1（位置情報なし）に設定されます。

アルゴリズムまたは振る舞い

1. makeNode(BoolExpr) を呼び出し、メモリを確保してノードを初期化します。 2. boolop フィールドに NOT_EXPR を代入します。 3. 引数 notclause を単一要素のリストとして args フィールドに格納します。 4. location を -1 に設定します。 5. 構築したノードを Expr * にキャストして返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepqual.c 内のクエリ最適化処理から呼び出されます。
  • **呼び出し先**:
  • makeNode (マクロ)
  • list_make1

関連データ構造

  • BoolExpr: 論理演算（AND, OR, NOT）を表現するノード構造体。
  • Expr: ノードの基底型。

失敗モードと境界条件

  • メモリ不足時には makeNode 経由でエラーが発生する可能性があります。
  • 引数 notclause が NULL の場合、リストの構築時に予期せぬ動作を引き起こす可能性があるため、呼び出し側で有効なポインタを渡す必要があります。

保守担当者向けメモ

  • 本関数は BoolExpr の NOT 演算専用のコンストラクタです。
  • location フィールドは常に -1 となるため、パーサー等でソースコード上の位置情報を保持する必要がある場合は、呼び出し後に手動で更新してください。
