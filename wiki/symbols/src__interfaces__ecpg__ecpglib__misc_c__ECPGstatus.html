ECPGstatus

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/misc.c:126

目的

指定されたデータベース接続が有効かつアクティブであるかを確認します。

シグネチャまたは形状

bool ECPGstatus(int lineno, const char *connection_name)

入力と出力

  • **入力**:
  • lineno: 呼び出し元のソースコード行番号（エラー報告用）。
  • connection_name: 接続先の識別名（NULLの場合はデフォルト接続）。
  • **出力**:
  • bool: 接続が確立されていれば true、それ以外（未接続または初期化失敗）は false を返します。

前提条件と不変条件

  • ecpg_get_connection を介して接続構造体を取得できる必要があります。
  • 内部的に ecpg_init が呼び出され、ライブラリの初期化状態が保証される必要があります。

アルゴリズムまたは振る舞い

1. ecpg_get_connection を呼び出し、指定された名前の接続構造体を取得します。 2. ecpg_init を呼び出し、ライブラリの初期化と接続状態の検証を行います。失敗した場合は false を返します。 3. 取得した接続構造体の connection メンバが NULL であるかを確認します。 4. NULL の場合、ecpg_raise を使用して ECPG_NOT_CONN エラーを発生させ、false を返します。 5. 上記のいずれにも該当しない場合、接続済みと判断し true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース excerpt 内には具体的な呼び出し元は記載されていません。
  • **呼び出し先**:
  • ecpg_get_connection
  • ecpg_init
  • ecpg_raise

関連データ構造

  • struct connection: 接続状態を保持する構造体。

失敗モードと境界条件

  • **接続未確立**: con->connection が NULL の場合、ECPG_NOT_CONN エラーが発行されます。
  • **初期化失敗**: ecpg_init が失敗した場合、関数は直ちに false を返します。

保守担当者向けメモ

  • 本関数は src/interfaces/ecpg/include/ecpglib.h で公開されており、ECPGアプリケーションの接続状態チェックの標準的な手段です。
  • 接続管理のロジックは src/interfaces/ecpg/ecpglib/connect.c に依存しています。
