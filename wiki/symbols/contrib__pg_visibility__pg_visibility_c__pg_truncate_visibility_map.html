pg_truncate_visibility_map

定義

  • 種別: function
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:390

目的

指定されたリレーションの可視性マップ（Visibility Map: VM）フォークを切り詰め（truncate）、VMの状態をリセットします。

シグネチャまたは形状

Datum pg_truncate_visibility_map(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してリレーションの OID を受け取ります。
  • **出力**: PG_RETURN_VOID() により値を返しません。

前提条件と不変条件

  • 対象リレーションに対して AccessExclusiveLock を取得します。
  • VMフォークを持つリレーション種別である必要があります（check_relation_relkind による検証）。
  • 実行中、チェックポイントの開始と完了を一時的に禁止するフラグ（DELAY_CHKPT_START | DELAY_CHKPT_COMPLETE）を MyProc に設定します。

アルゴリズムまたは振る舞い

1. 対象リレーションを AccessExclusiveLock でオープンします。 2. キャッシュされたファイルサイズ情報を強制的にリセットします。 3. visibilitymap_prepare_truncate を呼び出し、切り詰め後のブロック数を計算します。 4. クリティカルセクションを開始し、必要に応じて WAL ログ（XLOG_SMGR_TRUNCATE）を書き込みます。 5. smgrtruncate を実行して物理的に VM フォークを切り詰めます。 6. クリティカルセクションを終了し、チェックポイント禁止フラグを解除します。 7. トランザクション終了を待たずに relation_close を実行し、ロックを即時解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロによる登録経由）。
  • **呼び出し先**:
  • relation_open, relation_close
  • check_relation_relkind
  • visibilitymap_prepare_truncate
  • smgrtruncate
  • XLogInsert, XLogFlush 等の WAL 関連関数

関連データ構造

  • Relation: 操作対象のリレーション情報。
  • VISIBILITYMAP_FORKNUM: 操作対象のフォーク識別子。
  • xl_smgr_truncate: WAL 記録用の構造体。

失敗モードと境界条件

  • 対象リレーションが VM フォークを持たない場合、check_relation_relkind によりエラーが発生します。
  • AccessExclusiveLock を取得できない場合、待機またはタイムアウトが発生します。
  • WAL 記録が必要な環境でログ書き込みに失敗した場合、クリティカルセクション内でパニックが発生する可能性があります。

保守担当者向けメモ

  • 本関数はトランザクション終了を待たずにロックを解放します。これは、一度に多数の VM フォークを切り詰める際に AccessExclusiveLock が蓄積するのを防ぐための設計です。
  • 非トランザクション的な smgr 無効化メッセージが即座に共有メモリへ送信されるため、ロックを早期解放しても安全であると判断されています。
