PGOutputData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/pgoutput.h:18

目的

PostgreSQL の論理レプリケーション出力プラグイン pgoutput において、デコーディングセッションごとの実行状態および設定情報を保持するために使用されます。

シグネチャまたは形状

typedef struct PGOutputData
{
	MemoryContext context;
	MemoryContext cachectx;
	MemoryContext pubctx;
	bool		in_streaming;
	uint32		protocol_version;
	List	   *publication_names;
	List	   *publications;
	bool		binary;
	char		streaming;
	bool		messages;
	bool		two_phase;
	bool		publish_no_origin;
} PGOutputData;

入力と出力

  • **入力**: pgoutput_startup 等のコールバックを通じて初期化され、クライアントから提供されたレプリケーションパラメータ（プロトコルバージョン、パブリケーション名、バイナリ転送設定など）が格納されます。
  • **出力**: 本構造体自体は出力されませんが、保持する設定値は pgoutput.c 内の各処理で参照され、論理デコーディングの結果としてクライアントへ送信されるメッセージの形式を決定します。

前提条件と不変条件

  • LogicalDecodingContext に紐付けられ、プラグインのライフサイクル全体で維持されます。
  • メモリ管理のため、用途別に context（一時的）、cachectx（キャッシュ用）、pubctx（パブリケーション用）の3つの MemoryContext を保持します。

アルゴリズムまたは振る舞い

  • 論理デコーディングのセッション開始時に palloc0 で割り当てられ、初期化されます。
  • WAL の変更処理中、in_streaming フラグにより現在のトランザクションがストリーミングモードであるかを追跡します。
  • 各種処理（get_rel_sync_entry や pgoutput_row_filter_init 等）において、プラグインの動作モードを決定するための参照元として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/pgoutput/pgoutput.c 内の関数群（pgoutput_startup, parse_output_parameters, get_rel_sync_entry 等）から参照・操作されます。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • MemoryContext: メモリ管理に使用。
  • List (nodes/pg_list.h): パブリケーション名の管理に使用。
  • LogicalDecodingContext: 本構造体が紐付けられる上位のデコーディングコンテキスト。

失敗モードと境界条件

  • メモリコンテキストの使い分けを誤るとメモリリークが発生する可能性があります。
  • protocol_version や binary モードの設定がクライアントと不整合な場合、シリアライズされたデータが正しく解釈されない可能性があります。

保守担当者向けメモ

  • pgoutput.c 内の pgoutput_startup が初期化の起点です。
  • メモリ管理（context, cachectx, pubctx）の分離は、キャッシュの生存期間とトランザクションの生存期間を適切に制御するために重要です。
