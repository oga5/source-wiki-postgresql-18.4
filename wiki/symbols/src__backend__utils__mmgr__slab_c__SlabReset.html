SlabReset

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/slab.c:430

目的

SLAB メモリコンテキスト内のすべてのメモリブロックを解放し、コンテキストを初期状態にリセットします。

シグネチャまたは形状

void SlabReset(MemoryContext context)

入力と出力

  • **入力**: context (SLAB アロケータとして初期化された MemoryContext)
  • **出力**: なし（副作用としてメモリが解放され、コンテキストの状態が更新される）

前提条件と不変条件

  • context は有効な SlabContext であること（SlabIsValid による検証）。
  • 処理終了後、context->mem_allocated が 0 であること。

アルゴリズムまたは振る舞い

1. MEMORY_CONTEXT_CHECKING が定義されている場合、SlabCheck を呼び出しメモリの整合性を検証する。 2. emptyblocks リストを走査し、保持されている空きブロックをすべて free() して context->mem_allocated を減算する。 3. blocklist 配列の全インデックスを走査し、各リスト内のブロックをすべて free() して context->mem_allocated を減算する。 4. CLOBBER_FREED_MEMORY が定義されている場合、解放前に wipe_mem を実行してメモリを上書きする。 5. curBlocklistIndex を 0 にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MemoryContextMethods の reset フィールドとして登録。
  • **呼び出し先**:
  • SlabCheck (条件付き)
  • dclist_delete_from / dlist_delete
  • wipe_mem (条件付き)
  • free

関連データ構造

  • SlabContext: メモリコンテキストのメイン構造体。
  • SlabBlock: 管理対象のメモリブロック。
  • MemoryContext: 汎用メモリコンテキストの基底構造体。

失敗モードと境界条件

  • SlabIsValid が偽を返す場合、アサーションエラーが発生する。
  • 処理終了時に context->mem_allocated が 0 でない場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は SlabRealloc をサポートしていないため、メモリの再割り当てに関する考慮は不要です。
  • SLAB_BLOCKLIST_COUNT に基づくループ処理は、コンテキスト内の全ブロックを網羅的に解放するために不可欠です。
