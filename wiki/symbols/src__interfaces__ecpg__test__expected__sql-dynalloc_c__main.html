目的

ECPG（Embedded SQL for C）における動的SQLの割り当ておよび記述子（Descriptor）の使用方法を検証する回帰テストの実行起点。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（引数なし）
  • 出力: 終了ステータス（int）

前提条件と不変条件

  • 実行環境においてデータベース ecpg1_regression への接続が可能であること。
  • ECPGランタイムライブラリが正しく初期化されていること。

アルゴリズムまたは振る舞い

1. ECPGdebug を有効化し、標準エラー出力へトレースを出力する。 2. ECPGconnect でデータベースに接続し、テスト用テーブル test を作成してデータを投入する。 3. ECPGallocate_desc を呼び出し、記述子 mydesc を確保する。 4. ECPGdo を使用して SELECT 文を実行し、結果を mydesc に格納する。 5. ECPGget_desc を繰り返し呼び出し、記述子からホスト変数（d1〜d9）およびインジケータ変数（i1〜i9）へデータを転送する。 6. 各SQL文の実行直後に sqlca.sqlcode を確認し、エラー発生時は sqlprint() で情報を出力する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGallocate_desc, ECPGget_desc, sqlprint（ECPGライブラリ関数）
  • 注記: 呼び出し元および呼び出し先の詳細な依存関係は、提供されたソースコードの範囲外です。

関連データ構造

  • sqlca: SQL通信領域（エラーコードや処理行数の保持に使用）。
  • d1〜d9, i1〜i9: ホスト変数およびインジケータ変数。

失敗モードと境界条件

  • データベース接続失敗、テーブル作成失敗、またはクエリ実行失敗時に sqlca.sqlcode < 0 となり、sqlprint() が呼び出される。
  • NULL 値を含むデータ取得時にインジケータ変数が適切に処理される。

保守担当者向けメモ

  • 本関数はECPGプリプロセッサが生成したコードであり、直接編集しないこと。
  • 実行時の挙動は ECPGdebug(1, stderr) により標準エラー出力で確認可能。
