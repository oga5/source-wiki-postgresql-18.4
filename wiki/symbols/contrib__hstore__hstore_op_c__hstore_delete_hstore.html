hstore_delete_hstore

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:370

目的

第1引数の hstore から、第2引数の hstore に含まれるキーと値のペアを削除した新しい hstore を生成する。

シグネチャまたは形状

Datum hstore_delete_hstore(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_HSTORE_P(0): 削除対象のソース HStore。
  • PG_GETARG_HSTORE_P(1): 削除条件となる HStore。
  • **出力**:
  • Datum: 削除後の結果を保持する新しい HStore へのポインタ。

前提条件と不変条件

  • 入力される2つの HStore は、内部的にキーの長さと内容でソート済みであること。
  • hstore の内部レイアウト（ヘッダー、エントリ配列、文字列領域）を操作するため、hstore.h で定義されたマクロ（ARRPTR, STRPTR, HS_COUNT 等）に依存する。

アルゴリズムまたは振る舞い

1. 出力用のメモリを確保し、ヘッダーを初期化する。 2. 第2引数が空の場合、第1引数のコピーをそのまま返す。 3. 2つの HStore をマージソートのように走査する。

  • キーが一致する場合、値も比較する。値が異なる（または一方がNULLで他方が非NULL）場合は削除対象外として出力に含める。
  • キーが一致し、値も完全に一致する場合のみ、そのペアを結果から除外する。
  • キーが一致しない場合、第1引数の要素を結果に含める。

    • 4. HS_FINALIZE を呼び出し、最終的なサイズとエントリ数を確定させて結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • palloc: メモリ確保。
  • memcpy: 入力が空の場合のコピー。
  • memcmp: キーおよび値の比較。
  • HS_COPYITEM, HS_FINALIZE: hstore 構造の構築と確定。

関連データ構造

  • HStore: 操作対象のメイン構造体。
  • HEntry: HStore 内の各キー・値ペアのメタデータ（長さ、NULLフラグ等）を保持する構造体。

失敗モードと境界条件

  • 第2引数が空の場合、第1引数の内容を維持して正常終了する。
  • メモリ不足時には palloc がエラーを発生させる。
  • キーが一致しても値が異なる場合は削除されない（完全一致のみが削除対象）。

保守担当者向けメモ

  • 本関数は hstore のソート済み特性を前提としたマージ処理である。ロジックを変更する際は、HStore の内部レイアウト（HEntry 配列と文字列領域の配置）を破壊しないよう注意すること。
  • HS_COPYITEM 等の hstore.h マクロの挙動については、同ヘッダーファイルの定義を参照すること。
