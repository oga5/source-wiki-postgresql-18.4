GetIndexInputType

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgutils.c:120

目的

指定されたインデックス列に対応する入力データの型（Oid）を特定します。特に、多相型（Polymorphic Type）が使用されている場合に、実際の基底型を解決するために使用されます。

シグネチャまたは形状

static Oid GetIndexInputType(Relation index, AttrNumber indexcol)

入力と出力

  • **入力**:
  • index: 対象となるインデックスのリレーション記述子。
  • indexcol: 調査対象のインデックス列番号（1始まり）。
  • **出力**:
  • 該当列の入力型を表す Oid。解決できない場合は InvalidOid を返す可能性があります。

前提条件と不変条件

  • index->rd_index が非NULLであること。
  • indexcol が有効な範囲（1から indnkeyatts まで）にあること。

アルゴリズムまたは振る舞い

1. インデックスの演算子クラス情報から、指定列の型 opcintype を取得します。 2. opcintype が多相型でない場合は、その値をそのまま返します。 3. 多相型の場合、以下のいずれかの方法で基底型を特定します。

  • **単純な列の場合**: indkey を参照し、get_atttype と getBaseType を用いてヒープ上の列型を取得します。
  • **式インデックスの場合**: rd_indexprs（キャッシュ済み）または RelationGetIndexExpressions から式ツリーを取得し、exprType を用いて式の戻り値の型を特定します。

    • 4. 該当する列が見つからない場合はエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgutils.c:210（直接的な呼び出しを確認）
  • **呼び出し先**:
  • RelationGetIndexExpressions
  • get_atttype
  • getBaseType
  • exprType
  • list_head, lfirst, lnext（リスト操作関数）

関連データ構造

  • Relation: インデックスのメタデータおよびキャッシュ情報へのアクセスに使用。
  • Form_pg_index: rd_index を通じてアクセスされるインデックスの定義情報。

失敗モードと境界条件

  • indexcol が範囲外の場合、Assert により停止します。
  • 式インデックスにおいて、期待される列数と式ツリーの数が一致しない場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は、SP-GiSTインデックスが多相型を扱う際の型解決の要です。式インデックスの型解決において、RelationGetIndexExpressions を呼び出すとコピーが生成されるため、キャッシュ (rd_indexprs) を優先的に利用する最適化が行われています。このキャッシュの整合性はリレーションキャッシュのライフサイクルに依存しています。
