makepol

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_io.c:213

目的

検索クエリ文字列をトークン化し、演算子の優先順位を考慮して逆ポーランド記法（後置記法）のツリー構造へ変換します。

シグネチャまたは形状

static int32 makepol(QPRS_STATE *state)

入力と出力

  • **入力**: QPRS_STATE *state（解析中の状態、バッファ、演算子スタック、エラーコンテキストを保持）。
  • **出力**: 成功時は END を返し、エラー発生時は ERR を返します。

前提条件と不変条件

  • 再帰的に呼び出されるため、スタックオーバーフローを防ぐために check_stack_depth() を実行します。
  • 演算子スタックの深さは STACKDEPTH 定数によって制限されています。

アルゴリズムまたは振る舞い

1. gettoken_query を用いてクエリ文字列からトークンを順次取得します。 2. トークンの種類に応じて以下の処理を行います：

  • VAL (オペランド): pushval_asis で出力に追加し、スタック上の & または ! 演算子を順次処理します。
  • OPR (演算子): | 以外はスタックに積みます。| は優先順位に基づき処理されます。
  • OPEN (括弧開始): 再帰的に makepol を呼び出し、サブツリーを構築します。
  • CLOSE (括弧終了): スタックに残っている演算子をすべて出力し、終了します。

    • 3. ループ終了後、スタックに残っている演算子をすべて出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/ltxtquery_io.c 内の ltxtq_in 等から呼び出されます。また、contrib/intarray/_int_bool.c や src/backend/utils/adt/tsquery.c にも同名の関数が存在しますが、これらが本関数と同一の定義を指しているかは文脈上確定できません。
  • **呼び出し先**: gettoken_query, pushval_asis, pushquery, makepol (再帰呼び出し)。

関連データ構造

  • QPRS_STATE: 解析状態を管理する構造体。
  • ITEM: 逆ポーランド記法として構築されるクエリの各要素。

失敗モードと境界条件

  • **スタックオーバーフロー**: 再帰呼び出しが深すぎる場合、check_stack_depth() によりエラーとなります。
  • **演算子スタック溢れ**: STACKDEPTH を超える演算子が積まれた場合、elog(ERROR, "stack too short") が発生します。
  • **構文エラー**: 不正なトークンや構文に対しては ERR を返し、エラーコンテキストを介して通知します。

保守担当者向けメモ

  • 本関数は再帰的なパーサであるため、変更を加える際はスタック消費量に注意してください。
  • contrib/intarray や tsquery にも同名の関数が存在するため、コードの共通化や命名の衝突について留意してください。
