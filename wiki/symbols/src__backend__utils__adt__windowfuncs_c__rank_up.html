rank_up

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/windowfuncs.c:48

目的

RANK 系ウィンドウ関数において、現在の行が直前の行と異なる順位（ランク）に遷移すべきかを判定し、パーティションごとの計算状態を更新する。

シグネチャまたは形状

static bool rank_up(WindowObject winobj)

入力と出力

  • **入力**: WindowObject winobj（ウィンドウ関数実行用のAPIコンテキスト）
  • **出力**: bool（ランクを増加させるべき場合は true、同順位の場合は false）

前提条件と不変条件

  • WinGetPartitionLocalMemory を介して rank_context が確保されていること。
  • 初回呼び出し時（context->rank == 0）は、現在の行位置（curpos）が 0 であること。
  • 2回目以降の呼び出し時は、curpos > 0 であること。

アルゴリズムまたは振る舞い

1. WinGetPartitionLocalMemory を使用して、パーティション固有の rank_context を取得する。 2. 初回呼び出し時: ランクを 1 に初期化する。 3. 2回目以降: WinRowsArePeers を呼び出し、現在の行と直前の行（curpos - 1）が ORDER BY 句に基づき同等（ピア）であるかを確認する。

  • ピアでない場合、ランクを上げる必要があると判断し up = true とする。

    • 4. WinSetMarkPosition を呼び出し、現在の行位置をマークとして設定する（直前の行へのアクセスが完了した後に実行）。 5. 判定結果 up を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/windowfuncs.c 内の各ウィンドウ関数実装（window_rank, window_dense_rank, window_percent_rank, window_cume_dist 等）。
  • **呼び出し先**:
  • WinGetCurrentPosition
  • WinGetPartitionLocalMemory
  • WinRowsArePeers
  • WinSetMarkPosition

関連データ構造

  • rank_context: パーティションごとの計算状態（ランク値など）を保持する構造体。
  • WindowObject: ウィンドウ関数の実行状態を管理する不透明なハンドル。

失敗モードと境界条件

  • パーティションの先頭行では、ピア判定を行わず常にランク 1 とみなす。
  • WinRowsArePeers の結果に基づきランクの遷移を決定するため、ORDER BY 句の定義に依存する。

保守担当者向けメモ

  • 本関数は RANK 系関数の共通ロジックである。ランクの更新タイミングは WinSetMarkPosition によるマーク位置の更新と密接に関連しているため、変更時は順序に注意すること。
  • ピア判定のロジックは WinRowsArePeers に委譲されているため、比較基準を変更する場合はそちらを確認すること。
