astreamer_tar_parser_content

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_tar.c:110

目的

未加工のバイトストリームをtarアーカイブの構造（ヘッダー、コンテンツ、トレーラー）に解析し、後続のストリーマーへ転送する。

シグネチャまたは形状

static void astreamer_tar_parser_content(
    astreamer *streamer, 
    astreamer_member *member,
    const char *data, 
    int len,
    astreamer_archive_context context
)

入力と出力

  • **入力**:
  • streamer: 解析状態を保持する astreamer_tar_parser 構造体。
  • data, len: 入力バイトストリームとその長さ。
  • member, context: 呼び出し元から渡されるメタデータ（本関数内では NULL および ASTREAMER_UNKNOWN であることが前提）。
  • **出力**: なし（副作用として後続のストリーマーへデータを転送）。

前提条件と不変条件

  • member == NULL かつ context == ASTREAMER_UNKNOWN であること。
  • astreamer_tar_parser の next_context 状態に基づき、ブロック単位の処理が保証されること。

アルゴリズムまたは振る舞い

内部の next_context に基づき状態遷移を行う： 1. **ASTREAMER_MEMBER_HEADER**: astreamer_buffer_until で TAR_BLOCK_SIZE 分のデータを蓄積し、astreamer_tar_header で解析。成功時はコンテンツまたはトレーラーへ遷移し、失敗時はアーカイブ全体を終了とみなす。 2. **ASTREAMER_MEMBER_CONTENTS**: ファイルサイズ分までデータを後続へ転送。完了後、パディングの有無に応じてトレーラーへ遷移するか、次のヘッダーへ戻る。 3. **ASTREAMER_MEMBER_TRAILER**: パディングバイトを蓄積し、後続へ転送後、次のヘッダーへ遷移。 4. **ASTREAMER_ARCHIVE_TRAILER**: アーカイブ終端後のデータをバッファリングし、finalize 時の処理へ回す。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer インターフェースの content コールバックとして登録されている（src/fe_utils/astreamer_tar.c:54）。
  • **呼び出し先**:
  • astreamer_buffer_until, astreamer_buffer_bytes (バッファリング)
  • astreamer_tar_header (ヘッダー解析)
  • astreamer_content (後続ストリーマーへの転送)
  • pg_fatal (異常終了)

関連データ構造

  • astreamer_tar_parser: 解析状態を保持する構造体。
  • astreamer_archive_context: 処理中のtarセグメントを示す列挙型。

失敗モードと境界条件

  • 不正なtarヘッダーが検出された場合、pg_fatal によりプロセスが終了する。
  • next_context が定義外の状態になった場合、pg_fatal を呼び出す。
  • アーカイブの終端（ASTREAMER_ARCHIVE_TRAILER）以降のデータは、すべてバッファリングされる。

保守担当者向けメモ

  • 本関数は astreamer パイプラインのパーサー層として機能する。
  • pax 拡張フォーマットは未サポートであり、検出時はエラーとなる。
  • astreamer_buffer_until が false を返す場合、データが不足しているため処理を中断し、次回呼び出しを待機する。
