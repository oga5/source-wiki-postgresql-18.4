checkcondition_bit

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_bool.c:254

目的

ビットベクトル形式のデータ構造において、特定のクエリ項目（ITEM）が条件を満たしているかを判定するコールバック関数です。

シグネチャまたは形状

static bool checkcondition_bit(void *checkval, ITEM *item, void *siglen)

入力と出力

  • **入力**:
  • checkval: ビットベクトルデータへのポインタ。
  • item: 評価対象のクエリ項目（ITEM 構造体）。val フィールドがハッシュ計算に使用されます。
  • siglen: ビットベクトルの長さを保持する値（void* として渡され、内部で int にキャストされます）。
  • **出力**:
  • bool: 指定されたビット位置がセットされていれば true、そうでなければ false を返します。

前提条件と不変条件

  • checkval は有効なビットベクトル領域を指している必要があります。
  • siglen はビットベクトルのサイズとして正しく解釈可能な値である必要があります。
  • item->val はハッシュ値として有効な整数である必要があります。

アルゴリズムまたは振る舞い

1. item->val と siglen を用いて HASHVAL マクロによりハッシュ値を計算します。 2. 計算されたハッシュ値をビット位置として、GETBIT マクロを用いて checkval 内の該当ビットを確認します。 3. ビットが立っている（1である）場合に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_int_bool.c:302 で参照されています。
  • **呼び出し先**: GETBIT および HASHVAL マクロを使用します。
  • **注記**: 他のモジュール（contrib/ltree や src/backend/utils/adt/tsgistidx.c）にも同名の関数が存在しますが、これらは別のコンテキストで定義されたものであり、本シンボルとの直接的な呼び出し関係は確認されていません。

関連データ構造

  • ITEM: クエリツリーのノードを表現する構造体。

失敗モードと境界条件

  • siglen が不適切な値である場合、HASHVAL の計算結果がビットベクトルの範囲外となり、メモリ破壊や不正な判定を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は intarray モジュールにおける GIST インデックス等の整合性チェックにおいて、ビットベクトルを用いた高速な包含判定を行うために使用されます。
  • void * 型で引数を受け取る汎用的なシグネチャとなっているため、変更時にはキャストの整合性に注意してください。
