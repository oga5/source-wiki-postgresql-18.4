system_samplescangetsamplesize

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/system.c:87

目的

TABLESAMPLE SYSTEM メソッドにおけるサンプリング後の推定ページ数およびタプル数を計算し、プランナに提供します。

シグネチャまたは形状

static void system_samplescangetsamplesize(
    PlannerInfo *root,
    RelOptInfo *baserel,
    List *paramexprs,
    BlockNumber *pages,
    double *tuples
)

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報。
  • baserel: 対象リレーションの統計情報。
  • paramexprs: サンプリング率を指定する式リスト。
  • **出力**:
  • pages: 推定されるサンプリング後のページ数（書き出し）。
  • tuples: 推定されるサンプリング後のタプル数（書き出し）。

前提条件と不変条件

  • paramexprs の先頭要素はサンプリング率（0〜100のパーセンテージ）として評価可能である必要があります。
  • サンプリング率が範囲外または評価不能な場合は、デフォルト値として 0.1（10%）が使用されます。
  • 出力される pages および tuples は clamp_row_est によって非負の値に調整されます。

アルゴリズムまたは振る舞い

1. estimate_expression_value を使用して、paramexprs からサンプリング率の定数値を抽出します。 2. 抽出した値が 0 から 100 の範囲内であれば、それを 100 で除算して samplefract とします。範囲外の場合は 0.1 を使用します。 3. baserel->pages に samplefract を乗算し、clamp_row_est を適用して pages を算出します。 4. 同様に baserel->tuples に samplefract を乗算し、clamp_row_est を適用して tuples を算出します。

呼び出し元と呼び出し先

  • **呼び出し元**: tsm_system_handler 内で TsmRoutine 構造体の SampleScanGetSampleSize フィールドに登録され、プランナから間接的に呼び出されます。
  • **呼び出し先**: estimate_expression_value（式評価用）、clamp_row_est（推定値の正規化用）。

関連データ構造

  • PlannerInfo
  • RelOptInfo
  • TsmRoutine

失敗モードと境界条件

  • サンプリング率が NULL または評価不能な定数の場合、デフォルトの 0.1 が適用されます。
  • samplefract が NaN の場合、デフォルトの 0.1 が適用されます。

保守担当者向けメモ

  • 本関数は TABLESAMPLE の統計推定ロジックを担います。サンプリング率の計算ロジックを変更する場合は、system.c 内の他のサンプリング関数との整合性に注意してください。
