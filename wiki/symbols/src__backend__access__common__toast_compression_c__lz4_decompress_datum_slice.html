lz4_decompress_datum_slice

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/toast_compression.c:214

目的

LZ4圧縮された varlena データから、指定された長さ（スライス）分のみを伸長して取得する。

シグネチャまたは形状

struct varlena *lz4_decompress_datum_slice(const struct varlena *value, int32 slicelength)

入力と出力

  • **入力**:
  • value: 圧縮されたデータを含む varlena 構造体へのポインタ。
  • slicelength: 伸長を要求するデータの長さ（バイト単位）。
  • **出力**: 伸長されたデータを含む、新しく割り当てられた varlena 構造体へのポインタ。

前提条件と不変条件

  • ビルド時に USE_LZ4 が定義されている必要がある。未定義の場合、本関数はエラーを発生させる。
  • LZ4ライブラリのバージョンが 1.8.3 未満の場合、部分伸長はサポートされないため、全伸長を行う lz4_decompress_datum へフォールバックする。

アルゴリズムまたは振る舞い

1. USE_LZ4 が未定義であれば NO_LZ4_SUPPORT() を呼び出しエラー終了する。 2. LZ4バージョンが 1.8.3 未満であれば、lz4_decompress_datum を呼び出して全データを伸長し、その結果を返す。 3. slicelength + VARHDRSZ 分のメモリを palloc で確保する。 4. LZ4_decompress_safe_partial を使用し、圧縮データのヘッダー以降から指定された slicelength 分のデータを伸長する。 5. 伸長結果が負の値（エラー）であれば、ERRCODE_DATA_CORRUPTED でエラーを報告する。 6. 伸長後のサイズをヘッダーに設定し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/detoast.c:530
  • **呼び出し先**:
  • LZ4_versionNumber()
  • lz4_decompress_datum() (フォールバック時)
  • palloc()
  • LZ4_decompress_safe_partial()
  • ereport()

関連データ構造

  • varlena: PostgreSQLの可変長データ型。
  • VARHDRSZ: varlena のヘッダーサイズ。
  • VARHDRSZ_COMPRESSED: 圧縮データにおけるヘッダーサイズ。

失敗モードと境界条件

  • **LZ4未サポート**: ビルド環境で USE_LZ4 がない場合、実行時にエラーとなる。
  • **データ破損**: LZ4_decompress_safe_partial が負の値を返した場合、データ破損としてエラーを投げる。
  • **バージョン依存**: 1.8.3 未満の環境では、部分伸長ではなく全伸長が実行される。

保守担当者向けメモ

  • 本関数は varlena ヘッダー内の圧縮データ領域を直接操作するため、ヘッダー構造の変更時には注意が必要である。
  • LZ4_decompress_safe_partial の戻り値が要求した slicelength と一致しない場合の挙動については、LZ4ライブラリの仕様に依存する。
