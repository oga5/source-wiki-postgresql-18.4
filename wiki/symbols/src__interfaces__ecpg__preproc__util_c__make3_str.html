make3_str

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:255

目的

3つの文字列を連結し、一時メモリ領域に新しい文字列として構築して返します。

シグネチャまたは形状

char *make3_str(const char *str1, const char *str2, const char *str3)

入力と出力

  • **入力**: 連結対象となる3つのヌル終端文字列（str1, str2, str3）。
  • **出力**: 連結された結果を格納した新しい文字列へのポインタ。

前提条件と不変条件

  • 戻り値のメモリは loc_alloc によって確保されます。
  • 呼び出し側で free を行う必要はありません。解析サイクル終了時に reclaim_local_storage() によって一括解放されることが前提です。

アルゴリズムまたは振る舞い

1. 3つの入力文字列の長さを合計し、ヌル終端文字分（+1）を加えたサイズを loc_alloc で確保します。 2. strcpy で最初の文字列をコピーします。 3. strcat を用いて、2番目および3番目の文字列を順次連結します。 4. 完成した文字列のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/parser.c:206 で確認されています。
  • **呼び出し先**: loc_alloc（メモリ確保）、strlen、strcpy、strcat（標準Cライブラリ関数）。

関連データ構造

  • loc_chunk: loc_alloc が使用する一時メモリ管理用のチャンク構造体。

失敗モードと境界条件

  • 入力文字列が NULL の場合、標準の strlen や strcpy の挙動によりセグメンテーション違反が発生する可能性があります。
  • loc_alloc がメモリ確保に失敗した場合、内部でエラーハンドリング（プロセス終了等）が行われます。

保守担当者向けメモ

  • 本関数は一時メモリを使用するため、長期間保持する文字列の生成には使用しないでください。
  • 連結対象が非常に多い場合は、メモリ効率を考慮して他の連結関数との使い分けを検討してください。
