exclude_list_item

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup.c:137

目的

ファイルシステム走査時に、バックアップ対象から除外すべきファイルやディレクトリを定義するための構造体です。

シグネチャまたは形状

struct exclude_list_item
{
	const char *name;
	bool		match_prefix;
};

入力と出力

  • name: 除外対象のファイル名またはディレクトリ名の文字列。
  • match_prefix: true の場合、name をプレフィックスとして前方一致で判定します。false の場合は完全一致で判定します。

前提条件と不変条件

  • name は const char * であり、静的に定義された文字列リテラルを指すことが想定されています。

アルゴリズムまたは振る舞い

この構造体は、バックアップ処理における除外リスト（例: excludeFiles 配列）の各要素として使用されます。走査ロジックにおいて、対象ファイル名がこの構造体の定義と一致（または前方一致）する場合、そのファイルはバックアップ対象から除外されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/backup/basebackup.c 内の excludeFiles 配列定義にて使用されています。また、src/bin/pg_checksums/pg_checksums.c および src/bin/pg_rewind/filemap.c でも同様の構造体が定義・利用されています。
  • 呼び出し先: この構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • excludeFiles: basebackup.c 内で定義される exclude_list_item 型の静的配列。

失敗モードと境界条件

  • name が NULL の場合の挙動は定義されていません。
  • match_prefix が true の際、空文字列を指定した場合の挙動は実装依存となります。

保守担当者向けメモ

  • 本構造体は basebackup.c 以外（pg_checksums や pg_rewind）でも類似の定義が存在します。変更を加える際は、これらのツール間での互換性に注意してください。
