blendscan

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blscan.c:61

目的

Bloom インデックススキャンの終了処理を行い、スキャンに関連付けられたメモリリソースを解放します。

シグネチャまたは形状

void blendscan(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan - 終了対象となるインデックススキャンの記述子。
  • **出力**: なし。

前提条件と不変条件

  • scan->opaque には BloomScanOpaque 構造体が格納されている必要があります。
  • so->sign が割り当てられている場合、それは pfree で解放可能なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. scan->opaque を BloomScanOpaque 型としてキャストし、スキャン状態を取得します。 2. so->sign が非 NULL であるかを確認します。 3. so->sign が存在する場合、pfree を呼び出してメモリを解放します。 4. so->sign を NULL に設定し、解放済みであることを明示します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blutils.c:150 にて、amroutine->amendscan に関数ポインタとして登録されています。
  • **呼び出し先**:
  • pfree (PostgreSQL メモリ管理関数)

関連データ構造

  • IndexScanDesc: PostgreSQL のインデックススキャン記述子。
  • BloomScanOpaque: Bloom インデックススキャン固有の状態を保持する構造体。sign メンバ（検索シグネチャ）を管理します。

失敗モードと境界条件

  • so->sign が既に NULL の場合、pfree は呼び出されず、安全に終了します。

保守担当者向けメモ

  • 本関数は blbeginscan で確保されたリソースのクリーンアップを担います。
  • blrescan が呼ばれた際にも同様のシグネチャ再計算が発生するため、メモリ管理の整合性に注意してください。
