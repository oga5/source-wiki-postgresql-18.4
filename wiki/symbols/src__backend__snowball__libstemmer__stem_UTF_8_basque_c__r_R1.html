r_R1

  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_basque.c:1003

目的

単語内の現在のカーソル位置が、定義済みの領域「R1」の範囲内にあるかどうかを判定します。

シグネチャまたは形状

static int r_R1(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: int（現在のカーソル位置 z->c が z->I[1] 以上であれば 1、そうでなければ 0 を返します）

前提条件と不変条件

  • z->I[1] には、r_mark_regions によって事前に計算された R1 領域の開始位置が格納されている必要があります。
  • z->c は現在の処理対象位置を示します。

アルゴリズムまたは振る舞い

1. 環境構造体 z 内の整数配列 I のインデックス 1（R1 領域の境界値）を取得します。 2. 現在のカーソル位置 z->c と比較します。 3. z->I[1] <= z->c が真であれば、現在の位置は R1 領域内（または境界上）にあるとみなして成功（1）を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_ISO_8859_1_basque.c、stem_ISO_8859_1_catalan.c、stem_ISO_8859_1_dutch.c 等のステミング処理ルーチンから参照されています。
  • 呼び出し先: 直接的な関数呼び出しはありません。

関連データ構造

  • struct SN_env: ステミング処理の環境を保持する構造体。I 配列（領域境界）および c（カーソル位置）を含みます。

失敗モードと境界条件

  • z が NULL の場合、あるいは z->I が初期化されていない場合は未定義の動作となります。
  • カーソル位置が単語の末尾を超えている場合や、領域境界が正しく設定されていない場合でも、単純な数値比較として動作します。
