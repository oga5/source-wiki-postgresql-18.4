process_sublinks_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/subselect.c:48

目的

式ツリー内の SubLink を探索・変換する際、再帰的な走査処理（process_sublinks_mutator）間で共有されるコンテキスト情報を保持するために使用されます。

シグネチャまたは形状

typedef struct process_sublinks_context
{
	PlannerInfo *root;
	bool		isTopQual;
} process_sublinks_context;

入力と出力

本構造体は関数引数として渡されるコンテキストオブジェクトであり、直接的な入力・出力値を持つ関数ではありません。

前提条件と不変条件

  • root: クエリの計画作成に必要な PlannerInfo 構造体への有効なポインタである必要があります。
  • isTopQual: 現在の走査位置がクエリのトップレベルの条件式（WHERE 句など）であるかどうかを示すフラグです。

アルゴリズムまたは振る舞い

SS_process_sublinks が式ツリーを走査する際、この構造体を介して現在のクエリの計画情報と、サブクエリ変換の制約条件（トップレベルか否か）を再帰関数へ伝播させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/subselect.c 内の SS_process_sublinks および process_sublinks_mutator でインスタンス化されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出す主体ではありません。

関連データ構造

  • PlannerInfo: クエリ全体の計画化状態を保持する構造体。
  • SubLink: 本コンテキストを用いて処理される対象のノード。

失敗モードと境界条件

  • root が NULL の状態で走査が開始された場合、計画生成処理において不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • isTopQual フラグは、サブクエリを InitPlan に変換できるか、あるいは SubPlan として扱うべきかを判断する重要なヒントとして使用されます。変更を加える際は、このフラグが再帰的に正しく引き継がれているかを確認してください。
