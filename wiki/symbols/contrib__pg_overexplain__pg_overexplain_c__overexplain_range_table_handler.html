overexplain_range_table_handler

定義

  • 種別: function
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:119

目的

EXPLAIN コマンドのオプション解析時に、range_table オプションの値を ExplainState に反映させるためのハンドラ関数です。

シグネチャまたは形状

static void overexplain_range_table_handler(ExplainState *es, DefElem *opt, ParseState *pstate)

入力と出力

  • **入力**:
  • es: 現在の EXPLAIN 実行状態を保持する ExplainState 構造体へのポインタ。
  • opt: 解析対象のオプション定義（DefElem）。
  • pstate: パーサー状態（本関数内では未使用）。
  • **出力**: なし（void）。副作用として ExplainState 内の拡張オプション状態を更新します。

前提条件と不変条件

  • es が有効な ExplainState を指している必要があります。
  • overexplain_ensure_options(es) が呼び出されることで、ExplainState に overexplain_options がアタッチされていることが保証されます。

アルゴリズムまたは振る舞い

1. overexplain_ensure_options(es) を呼び出し、現在の ExplainState に関連付けられた overexplain_options 構造体を取得（または初期化）します。 2. defGetBoolean(opt) を使用して、opt からブール値を取得します。 3. 取得した値を options->range_table に代入し、状態を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照情報に基づくと、contrib/pg_overexplain/pg_overexplain.c:76 付近でオプション登録時に参照されています。
  • **呼び出し先**:
  • overexplain_ensure_options: 状態の取得と初期化。
  • defGetBoolean: オプション値の変換。

関連データ構造

  • ExplainState: EXPLAIN コマンドの実行状態を管理する構造体。
  • overexplain_options: 本モジュールが ExplainState に付与する拡張オプション保持用構造体。
  • DefElem: EXPLAIN オプションの定義要素。

失敗モードと境界条件

  • opt が無効な値を持つ場合、defGetBoolean がエラーを発生させる可能性があります。
  • es が NULL の場合、overexplain_ensure_options 内でクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は EXPLAIN のオプション解析フェーズでのみ実行されます。
  • range_table オプションの有効/無効は、後のプラン実行計画走査時に overexplain_per_plan_hook 等から参照されます。
