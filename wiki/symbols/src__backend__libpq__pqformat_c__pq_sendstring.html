pq_sendstring

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:194

目的

NULL終端文字列を StringInfo バッファに書き込みます。その際、サーバーからクライアントへの文字エンコーディング変換を自動的に適用します。

シグネチャまたは形状

void pq_sendstring(StringInfo buf, const char *str)

入力と出力

  • **入力**:
  • buf: 書き込み先の StringInfo 構造体へのポインタ。
  • str: 書き込む対象のNULL終端文字列。
  • **出力**: なし（戻り値なし）。buf が指すバッファの内容が更新されます。

前提条件と不変条件

  • str はNULL終端されている必要があります。
  • buf は初期化済みの StringInfo である必要があります。
  • 変換後の文字列もNULL終端された状態でバッファに格納されます。

アルゴリズムまたは振る舞い

1. pg_server_to_client を呼び出し、サーバーエンコーディングからクライアントエンコーディングへの変換を試みます。 2. 変換が発生した場合（戻り値が元の str と異なる場合）:

  • 変換後の文字列の長さを取得し、appendBinaryStringInfoNT を用いてNULL終端を含めてバッファに追加します。
  • 変換によって動的に確保されたメモリ領域を pfree で解放します。

    • 3. 変換が発生しなかった場合:

  • 元の文字列をそのまま appendBinaryStringInfoNT でバッファに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printsimple.c
  • src/backend/backup/basebackup_copy.c
  • src/backend/commands/async.c
  • src/backend/replication/logical/proto.c
  • **呼び出し先**:
  • pg_server_to_client (エンコーディング変換)
  • appendBinaryStringInfoNT (バッファへの書き込み)
  • pfree (メモリ解放)

関連データ構造

  • StringInfo: メッセージ構築に使用される動的バッファ構造体。

失敗モードと境界条件

  • str がNULLの場合、strlen でセグメンテーションフォールトが発生する可能性があります。
  • メモリ不足時には appendBinaryStringInfoNT 内部でエラーが発生し、処理が中断される可能性があります。

保守担当者向けメモ

  • エンコーディング変換が不要な場合は、オーバーヘッドを避けるために pq_send_ascii_string や pq_sendbytes の使用を検討してください。
  • 変換処理で動的にメモリが確保されるため、変換が発生したケースとそうでないケースでメモリ管理のパスが異なる点に注意してください。
