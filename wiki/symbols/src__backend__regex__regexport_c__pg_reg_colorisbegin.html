pg_reg_colorisbegin

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:190

目的

指定されたカラー（文字クラス）IDが、正規表現の先頭位置（Beginning of String）を示す特殊なカラーであるかどうかを判定します。

シグネチャまたは形状

int pg_reg_colorisbegin(const regex_t *regex, int co);

入力と出力

  • **入力**:
  • regex: コンパイル済みの正規表現オブジェクトへのポインタ。
  • co: 判定対象のカラー ID。
  • **出力**:
  • 指定されたカラーが先頭位置を表す場合は true (1)、それ以外は false (0) を返します。

前提条件と不変条件

  • regex は NULL であってはならず、かつ REMAGIC マジックナンバーによって正当性が検証されている必要があります。
  • 内部構造体 struct guts が regex->re_guts に格納されていることを前提としています。

アルゴリズムまたは振る舞い

1. regex から内部の cnfa（コンパイル済みNFA）構造体を取得します。 2. 指定されたカラー ID co が、cnfa 内の先頭位置を表すカラー配列 bos[0] または bos[1] のいずれかと一致するかを比較します。 3. 一致すれば true を、一致しなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_regexp.c:1076
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • regex_t: 正規表現の公開構造体。
  • struct cnfa: 正規表現の内部的な非決定性有限オートマトン構造。
  • struct guts: 正規表現エンジンの内部状態を保持する構造体。

失敗モードと境界条件

  • regex が NULL の場合、または re_magic が不正な場合、assert によりプログラムが異常終了します。

保守担当者向けメモ

  • 本関数は、正規表現の先頭マッチング制約を外部から識別するために使用されます。
  • bos[0] と bos[1] の両方をチェックしているのは、正規表現エンジンが複数の先頭位置制約（例: 行頭や文字列先頭など）を保持する可能性があるためです。
