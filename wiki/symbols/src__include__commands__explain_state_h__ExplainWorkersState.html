ExplainWorkersState

定義

  • 種別: struct
  • 定義箇所: src/include/commands/explain_state.h:35

目的

並列クエリ実行時において、各ワーカープロセスごとの出力バッファや状態を管理するために使用されます。

シグネチャまたは形状

typedef struct ExplainWorkersState
{
	int			num_workers;	/* 使用されたワーカープロセスの数 */
	bool	   *worker_inited;	/* ワーカーごとの状態初期化フラグ */
	StringInfoData *worker_str; /* ワーカーごとの一時出力バッファ */
	int		   *worker_state_save;	/* ワーカーごとのグループ化状態保存領域 */
	StringInfo	prev_str;		/* リダイレクト中の退避用出力バッファ */
} ExplainWorkersState;

入力と出力

  • 入力: num_workers を指定して構造体を生成します。
  • 出力: 各ワーカーの統計情報や出力内容を worker_str に保持し、最終的に EXPLAIN の出力の一部として統合されます。

前提条件と不変条件

  • 並列プラン実行時にのみ使用されます。
  • worker_str は StringInfoData の配列として確保され、各ワーカーの出力が個別に管理されます。

アルゴリズムまたは振る舞い

1. ExplainCreateWorkersState により、指定されたワーカー数に基づいてメモリが割り当てられます。 2. プランツリーの走査中、各ワーカーの出力は対応する worker_str に一時的に蓄積されます。 3. prev_str は、出力先をワーカーバッファへ切り替える際に、元の出力バッファを一時的に保持するために使用されます。 4. 処理終了時に各ワーカーの情報を出力し、構造体は解放されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/explain.c 内の ExplainCreateWorkersState および EXPLAIN 実行処理。
  • 呼び出し先: StringInfo 関連のメモリ操作関数（palloc 等を含む）。

関連データ構造

  • ExplainState: 本構造体を保持するメインのコンテキスト構造体。
  • StringInfoData: 出力バッファの管理に使用されるデータ構造。

失敗モードと境界条件

  • num_workers が 0 の場合や、メモリ割り当て失敗時の挙動については、呼び出し側の実装に依存します。
  • worker_str 等のポインタ配列が正しく初期化されていない状態でアクセスするとクラッシュの原因となります。

保守担当者向けメモ

  • ExplainState 構造体内の workers_state フィールドを通じて参照されます。
  • worker_str は StringInfoData の配列であるため、メモリ管理（palloc / pfree）には注意が必要です。
