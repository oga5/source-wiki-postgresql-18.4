gbt_oid_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_oid.c:127

目的

GiSTインデックス内の Oid 型データに対して、インデックスエントリのフェッチ（取得）操作を提供します。

シグネチャまたは形状

Datum gbt_oid_fetch(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • 出力: Datum 型として、フェッチされた Oid 値を返します。

前提条件と不変条件

  • tinfo 構造体（gbtree_ninfo 型）が、Oid 型の特性を定義するために利用可能である必要があります。
  • 入力される GISTENTRY は、GiSTインデックスの有効なエントリである必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. btree_utils_num.h で定義された汎用関数 gbt_num_fetch を呼び出します。 3. その際、Oid 型固有の情報を保持する tinfo を引数として渡し、処理を委譲します。 4. gbt_num_fetch の戻り値を PG_RETURN_POINTER を介して返します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLのGiSTインデックス管理システム（直接的な呼び出し元コードはソース抜粋内には存在しません）。
  • 呼び出し先: gbt_num_fetch（btree_utils_num.h で定義された汎用関数）。
  • 登録: PG_FUNCTION_INFO_V1(gbt_oid_fetch) により、PostgreSQLの関数マネージャに登録されています。

関連データ構造

  • GISTENTRY: GiSTインデックスのエントリ情報を保持する構造体。
  • tinfo: Oid 型の比較ロジックやサイズ情報を格納する gbtree_ninfo 構造体。

失敗モードと境界条件

  • GISTENTRY が無効な場合や、tinfo が正しく初期化されていない場合に予期せぬ動作を引き起こす可能性があります。
  • 内部で呼び出される gbt_num_fetch の失敗モードに依存します。

保守担当者向けメモ

  • 本関数は Oid 型専用のラッパーであり、実質的なロジックは contrib/btree_gist/btree_utils_num.c にある gbt_num_fetch に集約されています。
  • 動作の変更が必要な場合は、まず btree_utils_num.c の汎用実装を確認してください。
