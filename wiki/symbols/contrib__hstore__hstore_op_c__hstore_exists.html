hstore_exists

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:148

目的

hstore 型のデータ内に指定されたキーが存在するかどうかを判定する。

シグネチャまたは形状

Datum hstore_exists(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_HSTORE_P(0): 検索対象の HStore 構造体。
  • PG_GETARG_TEXT_PP(1): 検索するキーを表す text 型データ。
  • **出力**:
  • bool: キーが存在する場合は true、存在しない場合は false を返す。

前提条件と不変条件

  • 入力される HStore は、内部的にキーがソートされた状態である必要がある（hstoreFindKey のバイナリサーチを利用するため）。

アルゴリズムまたは振る舞い

1. hstoreFindKey を呼び出し、HStore 内で指定されたキーをバイナリサーチで検索する。 2. 検索結果として得られたインデックス idx が 0 以上であればキーが存在すると判断し、true を返す。 3. インデックスが負であればキーは存在しないと判断し、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_op.c 内で HSTORE_POLLUTE(hstore_exists, exists) マクロにより SQL 演算子 exists として公開されている。
  • **呼び出し先**:
  • hstoreFindKey（contrib/hstore/hstore_op.c 内で定義）

関連データ構造

  • HStore: 操作対象の hstore 内部表現。
  • text: キーの入力型。

失敗モードと境界条件

  • 入力キーが NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で NULL が返される（PG_GETARG_... マクロの挙動に依存）。
  • hstoreFindKey がキーを見つけられない場合、戻り値は -1 となり、本関数は正しく false を返す。

保守担当者向けメモ

  • 本関数は hstore のキー検索における標準的な実装である。パフォーマンス上の懸念がある場合は、hstoreFindKey の検索効率や HStore 構造体のソート状態を確認すること。
