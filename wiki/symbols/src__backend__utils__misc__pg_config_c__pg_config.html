pg_config

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/pg_config.c:22

目的

PostgreSQL のビルド時設定情報（コンパイルオプションやパス設定など）を、SQL インターフェースを通じて集合を返す関数（SRF）として提供する。

シグネチャまたは形状

Datum pg_config(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL 関数呼び出しのコンテキスト情報）。
  • **出力**: Datum（常に 0 を返すが、内部的には tuplestore を介して設定情報の名前と値のペアを呼び出し元へ出力する）。

前提条件と不変条件

  • InitMaterializedSRF を呼び出し、結果を格納するためのタプルストアを初期化する必要がある。
  • グローバル変数 my_exec_path が有効であること。

アルゴリズムまたは振る舞い

1. InitMaterializedSRF を実行し、結果セットの準備を行う。 2. get_configdata を呼び出し、実行バイナリのパス (my_exec_path) に基づく設定情報の配列を取得する。 3. 取得した配列をループで走査し、各要素の name と setting を CStringGetTextDatum でテキスト型に変換する。 4. tuplestore_putvalues を使用して、変換した値をタプルストアに書き込む。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • InitMaterializedSRF
  • get_configdata
  • CStringGetTextDatum
  • tuplestore_putvalues
  • **呼び出し元**: 提供されたソース excerpt 内には、本関数を直接呼び出している具体的な呼び出し元の情報は含まれていない。

関連データ構造

  • ConfigData: 設定情報の名前と値のペアを保持する構造体。
  • ReturnSetInfo: SRF の結果セット情報を保持する構造体。
  • tuplestore: 結果を一時的に格納するためのストア。

失敗モードと境界条件

  • get_configdata が返す設定情報の数（configdata_len）に依存してループ処理が行われる。
  • メモリ消費量は設定項目の数に比例する。

保守担当者向けメモ

  • 本関数は src/common/config_info.c で定義される get_configdata のラッパーとして機能する。
  • 外部コマンド pg_config と同一の情報を返すため、ビルド環境の差異をデバッグする際に有用である。
