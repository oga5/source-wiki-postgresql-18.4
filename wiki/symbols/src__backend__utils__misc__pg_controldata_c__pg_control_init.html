pg_control_init

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/pg_controldata.c:203

目的

データベース初期化時の構成パラメータ（ブロックサイズ、セグメントサイズ、チェックサムバージョン等）を制御ファイルから読み取り、SQLの行型として公開する。

シグネチャまたは形状

Datum pg_control_init(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し時の引数）。
  • **出力**: Datum（heap_form_tuple で生成された行データ）。

前提条件と不変条件

  • 呼び出し元は、SQL関数として本関数を呼び出す必要がある。
  • 戻り値の型が複合型（TYPEFUNC_COMPOSITE）であることが必須。
  • 制御ファイル読み取り時には ControlFileLock を共有モードで取得し、整合性を保証する。

アルゴリズムまたは振る舞い

1. get_call_result_type を呼び出し、戻り値の型が複合型であることを確認する。 2. LWLockAcquire で ControlFileLock を共有モードで取得する。 3. get_controlfile を実行し、ディスクから制御ファイルを読み込む。 4. LWLockRelease でロックを解放する。 5. CRCチェックを行い、不一致の場合はエラーを投げる。 6. 読み取った ControlFileData 構造体の各フィールド（maxAlign, blcksz, relseg_size 等）を Datum 配列に格納する。 7. heap_form_tuple を使用して行データを作成し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（具体的な呼び出し元コードはソース内に明示なし）。
  • **呼び出し先**: get_call_result_type, LWLockAcquire, get_controlfile, LWLockRelease, heap_form_tuple, PG_RETURN_DATUM。

関連データ構造

  • ControlFileData: 制御ファイルの物理的な内容を保持する構造体。
  • TupleDesc, HeapTuple: SQLの行データ構築に使用される構造体。

失敗モードと境界条件

  • 戻り値の型が複合型でない場合、elog(ERROR) を発生させる。
  • 制御ファイルのCRCチェックが失敗した場合、ereport(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は ControlFileData 構造体のフィールドを直接参照しているため、構造体の定義変更時には本関数のフィールドマッピングも更新が必要。
  • 読み取り時の排他制御（ControlFileLock）は必須であり、削除しないこと。
