pg_popcount64_slow

定義

  • 種別: function
  • 定義箇所: src/port/pg_bitutils.c:378

目的

64ビット整数に含まれる「1」のビット数（ポピュレーションカウント）を計算する汎用的なフォールバック実装を提供します。

シグネチャまたは形状

static inline int pg_popcount64_slow(uint64 word)

入力と出力

  • **入力**: uint64 word（カウント対象の64ビット整数）
  • **出力**: int（word 内の1のビット数）

前提条件と不変条件

  • HAVE__BUILTIN_POPCOUNT が定義されていない環境では、pg_number_of_ones ルックアップテーブルが初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. **コンパイル時最適化**: HAVE__BUILTIN_POPCOUNT が定義されている場合、コンパイラの組み込み関数（__builtin_popcountl または __builtin_popcountll）を使用して計算します。 2. **汎用実装**: 上記が利用できない場合、8ビット単位で pg_number_of_ones テーブルを参照し、ビットシフトを繰り返して合計値を算出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/port/pg_bitutils.c 内の初期化処理およびバッファ操作関数から参照されています。
  • **呼び出し先**:
  • pg_number_of_ones（汎用実装時のみ使用）

関連データ構造

  • pg_number_of_ones: 0-255の各値に対するビット数を保持するルックアップテーブル。

失敗モードと境界条件

  • HAVE__BUILTIN_POPCOUNT が未定義かつ SIZEOF_LONG と SIZEOF_LONG_LONG が共に8でない場合、コンパイルエラーが発生します。

保守担当者向けメモ

  • 本関数はハードウェア命令（POPCNT等）が利用できない環境でのフォールバックとして機能します。
  • pg_bitutils.c 内の動的ディスパッチ機構により、実行時に本関数が選択される場合があります。
