XLogRecordBuffer

定義

  • 種別: struct
  • 定義箇所: src/include/replication/decode.h:17

目的

論理デコーディング処理において、WALレコードの読み取り位置情報と、解析対象となるレコードのコンテキストを保持するために使用されるデータ構造です。

シグネチャまたは形状

typedef struct XLogRecordBuffer
{
	XLogRecPtr	origptr;        /* レコードの開始位置 */
	XLogRecPtr	endptr;         /* レコードの終了位置 */
	XLogReaderState *record;    /* WALレコードの読み取り状態 */
} XLogRecordBuffer;

入力と出力

本構造体は関数間で情報を共有するためのコンテナとして機能します。

  • **入力**: LogicalDecodingProcessRecord 等のデコード処理の過程で初期化されます。
  • **出力**: 各リソースマネージャのデコード関数（heap_decode 等）へ参照として渡され、レコード解析のコンテキストとして利用されます。

前提条件と不変条件

  • record フィールドは、有効な XLogReaderState を指している必要があります。
  • origptr および endptr は、WALストリーム上の整合性のある位置を指している必要があります。
  • 本構造体自体はデータ保持用であり、論理デコーディングの状態管理は LogicalDecodingContext が担います。

アルゴリズムまたは振る舞い

論理デコーディングのメインループにおいて、読み込まれたWALレコードのメタデータ（位置情報）と、XLogReaderState に格納されたレコード本体をセットで管理します。これにより、各リソースマネージャのデコード関数は、レコードの物理的な位置を特定しつつ、その内容を解析することが可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c 内の LogicalDecodingProcessRecord 等が本構造体をインスタンス化し、デコード関数へ渡します。
  • **呼び出し先**: DecodeInsert, DecodeUpdate, DecodeDelete, DecodeCommit, DecodeAbort などの各リソースマネージャ用デコード関数が、引数として本構造体を受け取ります。

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態を管理する構造体。
  • LogicalDecodingContext: 論理デコーディング全体のコンテキストを保持する構造体。
  • XLogRecPtr: WAL上の位置を示すポインタ型。

失敗モードと境界条件

  • record が NULL の場合、デコード関数内で参照するとクラッシュの原因となります。
  • origptr や endptr が無効な値である場合、WALの解析処理が正しく行われません。

保守担当者向けメモ

  • 本構造体は単なるデータ保持用です。状態管理ロジックをここに追加しないでください。
  • 新しいWALレコードタイプをサポートする際は、decode.c 内のディスパッチロジックで本構造体が正しく渡されているか確認してください。
