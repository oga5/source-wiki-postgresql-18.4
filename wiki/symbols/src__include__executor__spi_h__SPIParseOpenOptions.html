SPIParseOpenOptions

定義

  • 種別: struct
  • 定義箇所: src/include/executor/spi.h:58

目的

SPI（Server Programming Interface）経由で SQL コマンドの解析やカーソルオープンを行う際に、実行パラメータやオプションをカプセル化するために使用されます。

シグネチャまたは形状

typedef struct SPIParseOpenOptions
{
    ParamListInfo params;      /* 実行時のパラメータ情報 */
    int           cursorOptions; /* カーソル制御用のビットマスク */
    bool          read_only;     /* 読み取り専用実行のフラグ */
} SPIParseOpenOptions;

入力と出力

  • **入力**: 構造体の各フィールド（params, cursorOptions, read_only）は、SPI 関連関数へ渡される設定値として機能します。
  • **出力**: 本構造体自体が関数引数として渡されることで、SPI の実行コンテキストを制御します。

前提条件と不変条件

  • params は ParamListInfo 型であり、クエリ実行に必要なパラメータ情報を含んでいる必要があります。
  • cursorOptions はカーソルの挙動を決定するビットフラグです。

アルゴリズムまたは振る舞い

本構造体は、SPI の解析および実行フェーズにおいて、呼び出し元から提供された設定を SPI 内部の実行エンジンへ伝達する役割を担います。特に read_only フラグは、実行されるクエリが書き込みを許可するかどうかを制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_exec.c:8963 で本構造体のインスタンスが生成されています。
  • **呼び出し先**: src/backend/executor/spi.c:1535 および src/include/executor/spi.h:185 で引数として参照されています。

関連データ構造

  • ParamListInfo: クエリパラメータを保持する構造体。
  • SPIExecuteOptions: 実行時のオプションを管理する別の構造体。

失敗モードと境界条件

  • 構造体の各フィールドに不正な値（特に params の不整合や無効な cursorOptions）が設定された場合、SPI 実行関数内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は SPI のインターフェースの一部であり、変更時は src/backend/executor/spi.c 側の実装との整合性に注意してください。
  • 呼び出し元・呼び出し先の詳細なフローについては、現在提供されている情報以上の解析は行っていません。
