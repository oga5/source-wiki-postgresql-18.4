alignStringInfoInt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonpath.c:483

目的

StringInfo バッファの現在の長さを、4バイト境界（INTALIGN）に合わせるためにパディング（ゼロバイトの追加）を行います。

シグネチャまたは形状

static void alignStringInfoInt(StringInfo buf)

入力と出力

  • **入力**: buf (StringInfo) - 調整対象のバッファ。
  • **出力**: なし（副作用として buf の内容と長さが更新されます）。

前提条件と不変条件

  • buf は有効な StringInfo 構造体である必要があります。
  • INTALIGN マクロが4バイトアライメントを前提としているため、本関数は4バイト境界への調整を行います。

アルゴリズムまたは振る舞い

1. INTALIGN(buf->len) と現在の buf->len の差分を計算します。 2. 差分が1、2、または3の場合、その数だけ appendStringInfoCharMacro(buf, 0) を呼び出し、バッファにゼロバイトを追加します。 3. 差分が0の場合は何もしません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonpath.c 内の関数（flattenJsonPathParseItem 等でのバイナリ構築時に使用）。
  • **呼び出し先**: appendStringInfoCharMacro（lib/stringinfo.h で定義されたマクロ）。

関連データ構造

  • StringInfo: lib/stringinfo.h で定義される、可変長文字列を扱うための構造体。

失敗モードと境界条件

  • バッファのメモリ割り当てが不足している場合、appendStringInfoCharMacro 内部でメモリ確保エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は jsonpath のバイナリ表現を構築する際に、各アイテムを4バイト境界に配置してアクセス効率を高めるために使用されます。
  • INTALIGN マクロの定義が変更された場合、本関数のロジックも影響を受ける可能性があります。
