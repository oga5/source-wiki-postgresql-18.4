replace_outer_grouping

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:270

目的

サブクエリ内で参照される上位レベルの GroupingFunc を実行時パラメータ（PARAM_EXEC）に置換し、プランナーが実行時にその値へアクセスできるようにするためのパラメータスロットを割り当てる。

シグネチャまたは形状

Param *replace_outer_grouping(PlannerInfo *root, GroupingFunc *grp)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo 構造体。
  • grp: 置換対象の GroupingFunc ノード。
  • **出力**:
  • Param: PARAM_EXEC として構成された新しいパラメータノード。

前提条件と不変条件

  • grp->agglevelsup が 0 より大きく、かつ現在のクエリレベルよりも小さいこと（上位レベルの関数であること）。
  • 実行時に root->glob->paramExecTypes に型情報が追加され、root->plan_params に PlannerParamItem が追加される。

アルゴリズムまたは振る舞い

1. grp->agglevelsup を辿り、対象の GroupingFunc が定義されているクエリレベルの PlannerInfo を特定する。 2. GroupingFunc ノードをコピーし、IncrementVarSublevelsUp を用いてレベルを調整（agglevelsup を 0 に）する。 3. 重複排除は行わず、常に新しい PARAM_EXEC ID を発行する。 4. root->glob->paramExecTypes に GroupingFunc の型（Oid）を登録する。 5. PlannerParamItem を作成し、調整済みの grp ノードを保持して root->plan_params に追加する。 6. 割り当てられた paramId を持つ Param ノードを生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/subselect.c:2001
  • **呼び出し先**:
  • copyObject (ノード複製)
  • IncrementVarSublevelsUp (レベル調整)
  • makeNode (ノード生成)
  • lappend_oid / lappend (リスト操作)
  • exprType (型取得)

関連データ構造

  • PlannerInfo: プラン作成の全体状態を保持。
  • GroupingFunc: 集約関数やグループ化関数の情報を保持するノード。
  • PlannerParamItem: 実行時パラメータとして渡される式と ID のペア。
  • Param: 実行時に値を参照するためのノード。

失敗モードと境界条件

  • agglevelsup が 0 の場合、または現在のクエリレベル以上の場合は Assert により停止する。
  • 重複排除を行わないため、同一の GroupingFunc が複数回出現すると、その都度異なる PARAM_EXEC ID が消費される。

保守担当者向けメモ

  • 本関数は GroupingFunc 専用の置換ロジックであり、他の変数置換関数（replace_outer_var 等）とは異なり、意図的に重複排除を省略している。
  • root->glob->paramExecTypes はプラン全体で共有されるため、ID の割り当て順序と型情報の整合性に注意すること。
