strupper_icu

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_icu.c:424

目的

ICUライブラリを使用して、指定されたロケールに基づき文字列を大文字に変換する。

シグネチャまたは形状

size_t strupper_icu(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: 変換結果を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換対象の文字列。
  • srclen: src の長さ（バイト単位）。
  • locale: ICUロケール情報を含む pg_locale_t 構造体。
  • **出力**:
  • 変換後の文字列長（バイト単位）を size_t で返す。

前提条件と不変条件

  • icu_to_uchar および icu_convert_case が内部で呼び出されるため、ICUの初期化およびセッションのエンコーディング変換環境が整っている必要がある。
  • 変換に使用される一時バッファは関数内で確保され、終了時に必ず解放される。

アルゴリズムまたは振る舞い

1. icu_to_uchar を呼び出し、入力文字列をデータベースエンコーディングからICU内部表現（UTF-16/UChar）へ変換する。 2. icu_convert_case を介して u_strToUpper を実行し、大文字変換を行う。 3. icu_from_uchar を呼び出し、変換結果を UChar からデータベースエンコーディングへ戻して dest に書き込む。 4. 変換に使用した一時的な UChar バッファを pfree で解放する。 5. 最終的な変換後のバイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c:1316
  • **呼び出し先**:
  • icu_to_uchar
  • icu_convert_case
  • u_strToUpper (ICU標準関数)
  • icu_from_uchar
  • pfree

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。
  • UChar: ICUで使用されるUTF-16文字型。

失敗モードと境界条件

  • ICU内部でエラーが発生した場合、icu_convert_case 等の内部処理を通じて ereport(ERROR, ...) が発行され、PostgreSQLのトランザクションメカニズムにより処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数はICUの u_strToUpper をラップする薄い層である。
  • メモリ管理は palloc/pfree に依存しているため、変換処理中のメモリ不足時にはPostgreSQLの標準的なエラーハンドリングが適用される。
