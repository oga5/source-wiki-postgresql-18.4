AcquireRewriteLocks

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteHandler.c:146

目的

クエリ内の全リレーションに対して適切なロックを確保し、スキーマ変更からリライタ、プランナ、エグゼキュータを保護します。また、JOIN内の削除済みカラムを検出し、無効化する処理を行います。

シグネチャまたは形状

void AcquireRewriteLocks(Query *parsetree, bool forExecute, bool forUpdatePushedDown)

入力と出力

  • **入力**:
  • parsetree: 対象となるクエリツリー。
  • forExecute: 実行目的か否か。false の場合は全リレーションに AccessShareLock を使用します。
  • forUpdatePushedDown: プッシュダウンされた更新ロックを反映するか否か。
  • **出力**: なし（副作用としてリレーションのロック確保および parsetree 内の RangeTblEntry や joinaliasvars の更新を行います）。

前提条件と不変条件

  • parsetree は有効なクエリツリーであること。
  • RTE_JOIN のエイリアスリスト要素は、同一クエリレベル内の先行する RTE を参照していること。

アルゴリズムまたは振る舞い

1. **RTE処理**: クエリの rtable を走査し、種類に応じて処理します。

  • RTE_RELATION: forExecute に応じたロックモード（AccessShareLock または rellockmode）でリレーションをオープンし、rte->relkind を最新の状態に更新します。
  • RTE_JOIN: joinaliasvars を走査し、参照先カラムが削除されている場合は該当箇所を NULL に置換します。
  • RTE_SUBQUERY: 再帰的に AcquireRewriteLocks を呼び出します。

    • 2. **CTE処理**: cteList 内の各クエリに対して再帰的に呼び出します。 3. **SubLink処理**: hasSubLinks が真の場合、query_tree_walker を用いてクエリ内のサブリンクを走査・処理します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_proc.c
  • src/backend/commands/matview.c
  • src/backend/executor/functions.c
  • src/backend/optimizer/util/clauses.c
  • src/backend/rewrite/rewriteHandler.c (自身を再帰呼び出し)
  • **呼び出し先**:
  • table_open, table_close
  • strip_implicit_coercions
  • get_rte_attribute_is_dropped
  • rt_fetch
  • query_tree_walker (内部関数 acquireLocksOnSubLinks を使用)

関連データ構造

  • Query: クエリツリー構造体。
  • RangeTblEntry: レンジテーブルエントリ。
  • acquireLocksOnSubLinks_context: サブリンク走査時のコンテキスト。

失敗モードと境界条件

  • varno が rt_index 以上の場合、JOIN RTE 内で予期せぬエラーが発生します。
  • RTE_SUBQUERY の再帰呼び出しにおいて、循環参照が発生する可能性については本関数単体では制御していません。

保守担当者向けメモ

  • 本関数はトランザクション終了までロックを保持します。
  • RTE_JOIN の処理において、USING カラムなどのマージされたカラムは削除されない前提で処理されています。
  • forExecute が false の場合、rellockmode を無視して AccessShareLock を強制するため、読み取り専用の解析フェーズ等で利用されます。
