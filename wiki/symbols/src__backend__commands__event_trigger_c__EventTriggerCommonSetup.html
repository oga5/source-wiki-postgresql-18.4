EventTriggerCommonSetup

定義

  • 種別: function
  • 定義箇所: src/backend/commands/event_trigger.c:637

目的

特定のイベントに関連付けられたイベントトリガーをキャッシュから検索し、現在のコマンドタグに基づいて実行対象のトリガー関数リストを生成・準備します。

シグネチャまたは形状

static List *
EventTriggerCommonSetup(Node *parsetree,
                        EventTriggerEvent event, const char *eventstr,
                        EventTriggerData *trigdata, bool unfiltered)

入力と出力

  • **入力**:
  • parsetree: 解析対象のノード（DDLコマンド等）。
  • event: 発生したイベントの種類（EventTriggerEvent）。
  • eventstr: イベント名の文字列。
  • trigdata: 実行時に使用されるコンテキスト情報（出力として書き込まれる）。
  • unfiltered: コマンドタグによるフィルタリングをスキップするかどうか。
  • **出力**:
  • 実行すべきトリガー関数のOIDリスト（List *）。対象がない場合は NIL を返します。

前提条件と不変条件

  • USE_ASSERT_CHECKING が有効な場合、EventTriggerGetTag で取得したタグが、指定されたイベントに対して許可されているものか検証されます。
  • キャッシュの無効化が発生する前に、必要なトリガー情報をメモリコンテキストへコピーする必要があります。

アルゴリズムまたは振る舞い

1. **検証**: デバッグビルド時、イベントとコマンドタグの整合性をチェックします。 2. **キャッシュ検索**: EventCacheLookup(event) を呼び出し、該当イベントのトリガーリストを取得します。 3. **フィルタリング**: unfiltered が偽の場合、現在のコマンドタグとトリガーの定義を照合し、適合するもののみを runlist に追加します。 4. **コンテキスト設定**: 実行対象が存在する場合、trigdata 構造体にイベント情報、パースツリー、コマンドタグを格納します。 5. **返却**: 実行すべき関数のOIDリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/event_trigger.c 内の各イベント処理関数（EventTriggerDDLCommandStart 等）から複数回呼び出されています。
  • **呼び出し先**:
  • EventCacheLookup
  • EventTriggerGetTag
  • filter_event_trigger
  • lappend_oid
  • command_tag_event_trigger_ok / command_tag_table_rewrite_ok (デバッグ時のみ)

関連データ構造

  • EventTriggerData: 実行時のコンテキストを保持する構造体。
  • EventTriggerCacheItem: キャッシュ内のトリガー定義を保持する構造体。
  • List: OIDリストおよびキャッシュリストの管理に使用。

失敗モードと境界条件

  • キャッシュに該当イベントが存在しない場合、NIL を返し早期終了します。
  • フィルタリングの結果、実行対象のトリガー関数が一つも存在しない場合、NIL を返します。
  • デバッグビルドにおいて、予期しないコマンドタグが渡された場合は elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数はイベントトリガー実行の準備フェーズを担います。カタログ操作を伴う処理の前にキャッシュから情報を抽出する設計となっており、キャッシュ無効化による不整合を防ぐため、処理順序の変更には注意が必要です。
