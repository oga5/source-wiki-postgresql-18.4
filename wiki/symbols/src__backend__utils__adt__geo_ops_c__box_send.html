box_send

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/geo_ops.c:500

目的

BOX 型のデータを PostgreSQL のバイナリ転送形式（bytea）に変換します。

シグネチャまたは形状

Datum box_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_BOX_P(0) を介して BOX 構造体へのポインタを受け取ります。
  • **出力**: bytea 型の Datum を返します。

前提条件と不変条件

  • 入力される BOX 構造体は有効なメモリ領域を指している必要があります。
  • pq_begintypsend および pq_endtypsend を使用した StringInfo バッファの管理が必須です。

アルゴリズムまたは振る舞い

1. pq_begintypsend でバイナリ送信用のバッファを初期化します。 2. BOX 構造体の各座標値（high.x, high.y, low.x, low.y）を pq_sendfloat8 を用いて順次バッファに書き込みます。 3. pq_endtypsend でバッファを bytea 形式に変換し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（SQL レベルのバイナリ出力要求時）。
  • **呼び出し先**: pq_begintypsend, pq_sendfloat8, pq_endtypsend（いずれも libpq/pqformat.h で定義されたバイナリフォーマット用ユーティリティ）。

関連データ構造

  • BOX: 幾何学的なボックスを表現する構造体（utils/geo_decls.h で定義）。
  • StringInfoData: バイナリメッセージ構築用のバッファ構造体。

失敗モードと境界条件

  • メモリ不足時には pq_begintypsend 等の内部関数がエラーを発生させる可能性があります。
  • 浮動小数点数の値が NaN や無限大である場合の挙動は、pq_sendfloat8 の仕様に依存します。

保守担当者向けメモ

  • 本関数は BOX 型のバイナリ表現を定義しています。データ構造のフィールド順序を変更する場合は、対応する box_recv 関数も同時に更新する必要があります。
