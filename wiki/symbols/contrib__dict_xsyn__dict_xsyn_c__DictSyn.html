DictSyn

定義

  • 種別: struct
  • 定義箇所: contrib/dict_xsyn/dict_xsyn.c:35

目的

PostgreSQLの全文検索（Tsearch2）における拡張シノニム辞書の設定および読み込まれたシノニムデータの保持。

シグネチャまたは形状

typedef struct {
    int         len;            /* シノニム配列の長さ */
    Syn        *syn;            /* シノニムデータの配列 */
    bool        matchorig;      /* 元の単語をマッチング対象とするか */
    bool        keeporig;       /* 元の単語を結果に含めるか */
    bool        matchsynonyms;  /* シノニムをマッチング対象とするか */
    bool        keepsynonyms;   /* シノニムを結果に含めるか */
} DictSyn;

入力と出力

  • 本構造体は、dxsyn_init 関数内で palloc0 を用いてメモリ確保され、辞書設定オプションを格納するために使用される。
  • dxsyn_lexize 関数において、辞書検索のコンテキストとして参照される。

前提条件と不変条件

  • syn フィールドは、read_dictionary 関数によって外部ファイルから読み込まれ、ソートされた状態で保持される必要がある。
  • 辞書ファイル内の単語は小文字で定義されていることが前提となる。

アルゴリズムまたは振る舞い

  • 辞書全体の設定（フラグ群）と、検索対象となるシノニムのリストを一元管理する。
  • dxsyn_lexize は、本構造体の syn 配列に対して bsearch を実行し、入力単語に対応するシノニムを高速に検索する。

呼び出し元と呼び出し先

  • **呼び出し元**: dxsyn_init（初期化時）、dxsyn_lexize（検索時）にて使用。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはない。
  • 注: src/backend/tsearch/dict_synonym.c にも同名の構造体定義への言及があるが、本モジュールとの直接的な依存関係については提供されたソースからは判断できない。

関連データ構造

  • Syn: 個別の単語とシノニムの対応を保持する構造体。DictSyn はこの Syn の配列を管理する。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • 辞書ファイルが空、または不正な形式である場合、len が 0 となり検索は失敗する。

保守担当者向けメモ

  • matchorig, keeporig, matchsynonyms, keepsynonyms の各フラグは、dxsyn_lexize における単語展開の挙動を決定する。変更時は dxsyn_lexize のロジックと整合性をとること。
  • 辞書データはメモリ上にロードされるため、大規模な辞書ファイルを使用する場合はメモリ消費量に注意が必要。
