pgstat_progress_end_command

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_progress.c:149

目的

実行中の長時間コマンド（VACUUM, CREATE INDEX等）の進捗追跡を終了し、共有メモリ上の進捗状態をリセットします。

シグネチャまたは形状

void pgstat_progress_end_command(void)

入力と出力

  • 入力: なし（現在のバックエンドの共有メモリ状態 MyBEEntry を暗黙的に参照）
  • 出力: なし

前提条件と不変条件

  • MyBEEntry が有効であること。
  • pgstat_track_activities が有効であること。
  • コマンドが既に終了状態（PROGRESS_COMMAND_INVALID）である場合、関数は何も行わず即座にリターンします。

アルゴリズムまたは振る舞い

1. 現在のバックエンドの進捗状態（beentry）を取得します。 2. beentry が NULL であるか、pgstat_track_activities が無効な場合は終了します。 3. st_progress_command が既に PROGRESS_COMMAND_INVALID であれば終了します。 4. PGSTAT_BEGIN_WRITE_ACTIVITY マクロを使用して共有メモリへの書き込みロックを確保します。 5. st_progress_command を PROGRESS_COMMAND_INVALID に設定し、st_progress_command_target を InvalidOid にリセットします。 6. PGSTAT_END_WRITE_ACTIVITY マクロを使用して書き込みを完了します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/access/transam/xact.c
  • src/backend/backup/basebackup_progress.c
  • src/backend/catalog/index.c
  • src/backend/commands/analyze.c
  • src/backend/commands/cluster.c
  • src/backend/commands/copyfrom.c
  • src/backend/commands/copyto.c
  • src/backend/commands/indexcmds.c
  • 呼び出し先:
  • PGSTAT_BEGIN_WRITE_ACTIVITY / PGSTAT_END_WRITE_ACTIVITY マクロ（共有メモリ操作用）

関連データ構造

  • PgBackendStatus: 共有メモリ上のバックエンド状態構造体。
  • MyBEEntry: 現在のプロセスの PgBackendStatus へのポインタ。

失敗モードと境界条件

  • MyBEEntry が NULL の場合、安全に早期リターンします。
  • pgstat_track_activities が無効な場合、処理はスキップされます。
  • 既に終了済みのコマンドに対して呼び出された場合、副作用なくリターンします。

保守担当者向けメモ

  • 本関数は進捗状態の「終了」を管理するのみであり、個別の進捗パラメータ（st_progress_param）のクリアは行いません。パラメータの初期化は pgstat_progress_start_command の責務です。
