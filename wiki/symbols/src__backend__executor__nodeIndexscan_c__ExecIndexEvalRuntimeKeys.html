ExecIndexEvalRuntimeKeys

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:600

目的

クエリ実行時に値が決定するインデックススキャンの条件（ランタイムキー）を評価し、その結果をインデックススキャン用の ScanKey に適用します。

シグネチャまたは形状

void ExecIndexEvalRuntimeKeys(ExprContext *econtext, IndexRuntimeKeyInfo *runtimeKeys, int numRuntimeKeys)

入力と出力

  • econtext: 式評価に使用するコンテキスト。
  • runtimeKeys: 評価対象となるランタイムキー情報の配列。
  • numRuntimeKeys: 配列内のキーの数。
  • 出力: なし（副作用として runtimeKeys 内の ScanKey 構造体が更新されます）。

前提条件と不変条件

  • econtext->ecxt_per_tuple_memory が現在のメモリコンテキストとして使用されます。
  • 評価結果が参照渡し（pass-by-ref）の場合、外部スキャンのタプルがスキャン期間中保持されることを前提としています。

アルゴリズムまたは振る舞い

1. econtext->ecxt_per_tuple_memory へメモリコンテキストを切り替えます。 2. 各ランタイムキーについて以下の処理を繰り返します。

  • ExecEvalExpr を呼び出し、式を評価して Datum を取得します。
  • 結果が NULL の場合、ScanKey の sk_argument に設定し、SK_ISNULL フラグを立てます。
  • 結果が NULL でない場合、key_toastable が真であれば PG_DETOAST_DATUM を適用して強制的にデトーストし、sk_argument に設定して SK_ISNULL フラグを解除します。

    • 3. 元のメモリコンテキストへ復帰します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/executor/nodeBitmapIndexscan.c:152
  • src/backend/executor/nodeIndexonlyscan.c:378
  • src/backend/executor/nodeIndexscan.c:567
  • 呼び出し先: ExecEvalExpr（式評価用）、PG_DETOAST_DATUM（データ展開用）、MemoryContextSwitchTo（メモリ管理用）。

関連データ構造

  • ExprContext: 式評価の実行環境。
  • IndexRuntimeKeyInfo: ランタイムキーの式と対応するスキャンキーを保持する構造体。
  • ScanKey: インデックススキャンで使用される検索条件。

失敗モードと境界条件

  • 評価対象の式が NULL を返す場合、適切に SK_ISNULL フラグで処理されます。
  • TOAST化された値は自動的にデトーストされ、インデックスサポート関数での重複処理を回避します。

保守担当者向けメモ

  • 本関数は ExecReScanIndexScan 等から呼び出され、実行時に動的に変化する条件をインデックススキャンに反映させる重要な役割を担います。
  • 評価結果のメモリ寿命は econtext->ecxt_per_tuple_memory に依存するため、呼び出し側のコンテキスト管理に注意してください。
