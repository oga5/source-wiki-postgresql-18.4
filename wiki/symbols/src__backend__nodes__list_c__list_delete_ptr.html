list_delete_ptr

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:871

目的

ポインタを格納するリスト（List）から、指定された値（datum）と一致する最初の要素を削除します。

シグネチャまたは形状

List *list_delete_ptr(List *list, void *datum);

入力と出力

  • **入力**:
  • list: 操作対象の List 構造体へのポインタ。
  • datum: 削除対象のポインタ値。
  • **出力**:
  • 要素が削除された後のリスト、または一致するものが見つからなかった場合は元のリストを返します。

前提条件と不変条件

  • list はポインタを保持するリスト（IsPointerList(list) が真）である必要があります。
  • 操作の前後で check_list_invariants(list) が呼び出され、リストの整合性が検証されます。

アルゴリズムまたは振る舞い

1. リスト内の各要素を foreach マクロで走査します。 2. lfirst(cell) が指定された datum と一致するか比較します。 3. 一致する要素が見つかった場合、list_delete_cell を呼び出してそのセルを削除し、結果を返します。 4. リストの末尾まで一致するものが見つからなかった場合、リストを変更せずにそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execUtils.c
  • src/backend/optimizer/plan/analyzejoins.c
  • src/backend/optimizer/util/joininfo.c
  • src/backend/replication/logical/applyparallelworker.c
  • **呼び出し先**:
  • list_delete_cell (一致時に呼び出し)
  • check_list_invariants (内部検証)

関連データ構造

  • List: リストのヘッダ構造体。
  • ListCell: リスト内の各要素を指すための構造体。

失敗モードと境界条件

  • list がポインタリストでない場合、Assert により停止する可能性があります。
  • 指定した datum がリスト内に存在しない場合、リストは変更されず、入力されたリストがそのまま返されます。
  • リストの削除操作により内部配列が再配置される可能性があるため、呼び出し元は操作後のリストポインタを再利用する必要があります。

保守担当者向けメモ

  • 本関数はポインタ比較を行うため、値の等価性ではなくアドレスの一致を確認します。
  • リストの削除操作は配列のシフトを伴うため $O(N)$ の計算量となります。頻繁な削除が発生する箇所ではデータ構造の選択を再検討してください。
