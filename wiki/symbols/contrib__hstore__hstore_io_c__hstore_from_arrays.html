hstore_from_arrays

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_io.c:602

目的

2つの text 型配列（キーと値）から hstore 構造体を生成する。

シグネチャまたは形状

Datum hstore_from_arrays(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: text 型の配列（キー）。
  • 第2引数: text 型の配列（値）。NULL許容。
  • **出力**:
  • hstore 型の Datum。第1引数が NULL の場合は NULL を返す。

前提条件と不変条件

  • 入力配列は TEXTOID 型である必要がある。
  • 配列の次元数は 0 または 1 であること。
  • 値配列が提供される場合、キー配列と値配列は同じ次元数、同じ要素数、同じ境界を持つ必要がある。
  • キーに NULL を含めることはできない。

アルゴリズムまたは振る舞い

1. キー配列を deconstruct_array_builtin で展開する。 2. 値配列が提供されている場合、同様に展開し、キー配列との整合性を検証する。 3. 各要素を Pairs 構造体に格納する。値が NULL の場合は isnull フラグを立てる。 4. hstoreCheckKeyLen および hstoreCheckValLen を用いて、各要素の長さが hstore の制限内であることを確認する。 5. hstoreUniquePairs を呼び出し、キーの重複排除とソートを行う。 6. hstorePairs を呼び出し、最終的な HStore 構造体を構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 による登録）。
  • **呼び出し先**:
  • deconstruct_array_builtin
  • hstoreCheckKeyLen / hstoreCheckValLen
  • hstoreUniquePairs
  • hstorePairs
  • ereport (エラー時)

関連データ構造

  • HStore: 生成される内部データ構造。
  • Pairs: パースおよび構築過程で使用されるキーと値のペア保持用構造体。
  • ArrayType: 入力配列の型。

失敗モードと境界条件

  • **配列の次元エラー**: 次元数が 1 を超える場合に ERRCODE_ARRAY_SUBSCRIPT_ERROR を発生させる。
  • **境界不一致**: キーと値の配列の境界が一致しない場合に ERRCODE_ARRAY_SUBSCRIPT_ERROR を発生させる。
  • **NULL キー**: キー配列に NULL が含まれる場合に ERRCODE_NULL_VALUE_NOT_ALLOWED を発生させる。
  • **メモリ制限**: ペア数が MaxAllocSize を超える場合に ERRCODE_PROGRAM_LIMIT_EXCEEDED を発生させる。

保守担当者向けメモ

  • hstoreUniquePairs は重複排除時に pfree を行う可能性があるため、メモリ管理に注意すること。
  • 値配列が NULL の場合、すべての値は NULL として扱われる。
  • hstore.h:68 にある通り、極端なケースではペア数が INT_MAX を超える可能性があるため、サイズチェックが重要である。
