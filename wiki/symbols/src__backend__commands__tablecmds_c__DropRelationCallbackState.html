DropRelationCallbackState

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/tablecmds.c:314

目的

リレーション削除処理（DROP コマンド）において、対象リレーションの検証およびロック取得を行う際のコールバック関数に渡される状態情報を保持する。

シグネチャまたは形状

struct DropRelationCallbackState {
    char        expected_relkind;
    LOCKMODE    heap_lockmode;
    Oid         heapOid;
    Oid         partParentOid;
    char        actual_relkind;
    char        actual_relpersistence;
};

入力と出力

  • **入力**: RemoveRelations 関数によって初期化され、RangeVarCallbackForDropRelation に引数として渡される。
  • **出力**: RangeVarCallbackForDropRelation 内で、actual_relkind および actual_relpersistence が更新され、呼び出し元へ結果が返される。

前提条件と不変条件

  • expected_relkind および heap_lockmode は、コールバック実行前に RemoveRelations によって設定されている必要がある。
  • heapOid および partParentOid は、処理対象のリレーションを特定するために使用される。

アルゴリズムまたは振る舞い

1. RemoveRelations が本構造体をインスタンス化し、期待されるリレーション種別とロックモードを設定する。 2. RangeVarCallbackForDropRelation が呼び出される際、この構造体へのポインタが arg として渡される。 3. コールバック関数は、対象リレーションのカタログ情報を参照し、実際の種別や永続性を構造体の actual_* フィールドに格納する。 4. 呼び出し元は、これらのフィールドを参照して、削除対象が適切であるか、あるいは追加のロックが必要かを判断する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c 内の RemoveRelations がこの構造体を生成し、コールバックへ渡す。
  • **呼び出し先**: RangeVarCallbackForDropRelation がこの構造体のフィールドを更新する。

関連データ構造

  • RangeVar: リレーションの識別に使用される。
  • LOCKMODE: ロックの強度を定義する列挙型。

失敗モードと境界条件

  • 構造体が適切に初期化されていない場合、コールバック内でのリレーション検証が誤った結果を返す可能性がある。
  • heapOid が無効な場合、カタログ検索に失敗する可能性がある。

保守担当者向けメモ

  • 本構造体は DROP コマンドの実行フローにおける状態共有用である。
  • actual_relkind や actual_relpersistence の値は、コールバック実行後にのみ有効となるため、参照タイミングに注意すること。
