RangeType

定義

  • 種別: struct
  • 定義箇所: src/include/utils/rangetypes.h:25

目的

PostgreSQLにおける範囲型（Range Types）のメモリ上のデータ構造を定義する。可変長データ（varlena）として表現され、範囲型のシリアライズ、比較、集合演算の基盤となる。

シグネチャまたは形状

typedef struct
{
    int32       vl_len_;        /* varlena header */
    Oid         rangetypid;     /* range type's own OID */
    /* 境界値（0〜2個）およびフラグバイトが続く */
} RangeType;

入力と出力

  • 本構造体は Datum としてデータベースストレージとやり取りされる。
  • 処理の際は DatumGetRangeTypeP 等でポインタを取得し、range_deserialize を経て RangeBound 構造体へ展開される。
  • 演算結果は range_serialize で本構造体に変換され、RangeTypePGetDatum で Datum に戻される。

前提条件と不変条件

  • 可変長構造体であるため、フィールドを直接操作してはならない。必ず提供されているマクロや関数経由でアクセスすること。
  • 境界値の配置は、要素型の typalign に依存する。
  • RANGE_LB_NULL および RANGE_UB_NULL フラグは将来的な拡張のために予約されており、現在は使用されない。

アルゴリズムまたは振る舞い

1. Datum から RangeType ポインタへの変換（必要に応じてデトースト）。 2. range_deserialize による内部フラグと境界値の展開。 3. 比較や集合演算の実行。 4. range_serialize による RangeType 形式への再構築。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/execIndexing.c および src/backend/utils/adt/multirangetypes.c での参照が確認されている。
  • 呼び出し先: src/include/utils/typcache.h を介して要素型のキャッシュ情報にアクセスする。

関連データ構造

  • RangeBound: 範囲の境界（下限・上限）をプログラム内で扱うための構造体。
  • RANGE_* フラグ群: 範囲の状態（空、境界の包含性、無限大など）を管理するビットフラグ。
  • MultirangeType: 範囲型の集合を扱う関連構造体。

失敗モードと境界条件

  • 範囲が空である場合や、無限大を含む場合の判定には RangeIsEmpty() や RangeIsOrContainsEmpty() マクロを使用する必要がある。
  • 構造体の末尾に配置されるフラグバイトの読み取りには、適切なアライメント計算が必須である。
