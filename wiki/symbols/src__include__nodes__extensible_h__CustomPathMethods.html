CustomPathMethods

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/extensible.h:92

目的

PostgreSQLのプランナーがカスタムパス（CustomPath）を実際の実行可能なプラン（Plan）へ変換し、また再パラメータ化を行うためのインターフェースを提供します。

シグネチャまたは形状

typedef struct CustomPathMethods
{
    const char *CustomName;

    struct Plan *(*PlanCustomPath) (PlannerInfo *root,
                                    RelOptInfo *rel,
                                    struct CustomPath *best_path,
                                    List *tlist,
                                    List *clauses,
                                    List *custom_plans);
    struct List *(*ReparameterizeCustomPathByChild) (PlannerInfo *root,
                                                     List *custom_private,
                                                     RelOptInfo *child_rel);
} CustomPathMethods;

入力と出力

  • PlanCustomPath:
  • 入力: プランナー情報、リレーション情報、最適パス、ターゲットリスト、制約句、子プランリスト。
  • 出力: 変換された Plan 構造体へのポインタ。
  • ReparameterizeCustomPathByChild:
  • 入力: プランナー情報、カスタムプライベートデータ、子リレーション情報。
  • 出力: 再パラメータ化されたプライベートデータのリスト。

前提条件と不変条件

  • CustomName は一意である必要があります。
  • 拡張機能によって実装されるメソッドは、コアシステムが期待するシグネチャに厳密に従う必要があります。

アルゴリズムまたは振る舞い

1. **プラン変換**: プランナーがカスタムパスを最終的なクエリプランに変換する際、PlanCustomPath が呼び出されます。これにより、拡張機能は独自のプランノードを生成できます。 2. **再パラメータ化**: パラメータ化されたパスが子リレーションに関連付けられている場合、ReparameterizeCustomPathByChild を通じてパスの再構築が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLのプランナー（src/include/nodes/pathnodes.h で CustomPath 構造体を通じて参照される）。
  • 呼び出し先: 拡張機能側で実装された関数（具体的な関数ポインタは登録時に指定される）。

関連データ構造

  • CustomPath: 本メソッド群を利用するパス構造体。
  • PlannerInfo, RelOptInfo, Plan: プラン生成プロセスで使用されるコア構造体。

失敗モードと境界条件

  • メソッドポインタが NULL の場合、コアシステムが呼び出しを試みるとセグメンテーション違反が発生する可能性があります。
  • CustomName が重複して登録された場合、システムは正しくメソッドを解決できない可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/nodes/pathnodes.h 内の CustomPath 構造体から参照されます。
  • 拡張機能開発者は、これらのメソッドを実装し、コアシステムに登録する必要があります。
