cfunc_hashtable_lookup

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/funccache.c:145

目的

グローバルな関数キャッシュハッシュテーブル（cfunc_hashtable）から、指定されたキーに対応するキャッシュ済み関数エントリを検索する。

シグネチャまたは形状

static CachedFunction *cfunc_hashtable_lookup(CachedFunctionHashKey *func_key)

入力と出力

  • **入力**: func_key (CachedFunctionHashKey *) - 検索対象の関数を特定するためのハッシュキー。
  • **出力**: CachedFunction * - 見つかった場合は対応する関数構造体へのポインタ、見つからない場合またはハッシュテーブルが未初期化の場合は NULL。

前提条件と不変条件

  • cfunc_hashtable が初期化されているか、あるいは NULL であることを許容する。
  • func_key は有効なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. cfunc_hashtable が NULL の場合、即座に NULL を返す。 2. hash_search を HASH_FIND モードで呼び出し、func_key に一致するエントリを検索する。 3. エントリが見つかった場合、そのエントリ内の function フィールドを返す。 4. エントリが見つからなかった場合、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/funccache.c 内の cached_function_compile から呼び出される。
  • **呼び出し先**: utils/hsearch.h で定義される hash_search を利用する。

関連データ構造

  • CachedFunction: キャッシュされる関数の基底構造体。
  • CachedFunctionHashKey: 検索に使用するキー構造体。
  • CachedFunctionHashEntry: ハッシュテーブル内のエントリ構造体。
  • cfunc_hashtable: キャッシュを保持するグローバルなハッシュテーブル。

失敗モードと境界条件

  • cfunc_hashtable が未初期化（NULL）の場合、検索は行われず安全に NULL を返す。
  • 検索キーがハッシュテーブル内に存在しない場合、hash_search は NULL を返し、本関数も NULL を返す。

保守担当者向けメモ

  • 本関数は読み取り専用の検索を行う。キャッシュの挿入や削除は行わない。
  • hash_search の戻り値である CachedFunctionHashEntry から function フィールドを抽出する際、エントリの存在チェックを確実に行っている。
