RelationCloseSmgr

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/rel.h:592

目的

リレーションに関連付けられたストレージマネージャ（SMgrRelation）のキャッシュを解放し、リレーション記述子内のポインタを無効化する。

シグネチャまたは形状

void RelationCloseSmgr(Relation relation)

入力と出力

  • **入力**: relation (Relation) - 操作対象のリレーション記述子へのポインタ。
  • **出力**: なし。副作用として relation->rd_smgr が NULL に更新される。

前提条件と不変条件

  • relation は有効な Relation 構造体を指している必要がある。
  • relation->rd_smgr が NULL の場合、本関数は何もしない（安全に呼び出し可能）。

アルゴリズムまたは振る舞い

1. relation->rd_smgr が NULL でないことを確認する。 2. smgrunpin(relation->rd_smgr) を呼び出し、ストレージマネージャのピンを解除する。 3. smgrclose(relation->rd_smgr) を呼び出し、物理的なファイルハンドルを閉じる。 4. relation->rd_smgr を NULL に設定し、キャッシュ状態をリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/storage.c
  • src/backend/utils/cache/relcache.c
  • **呼び出し先**:
  • smgrunpin
  • smgrclose

関連データ構造

  • RelationData (または Relation): リレーションのメタデータを保持する構造体。rd_smgr フィールドを介してストレージマネージャと接続する。
  • SMgrRelation: 物理ストレージ管理のための構造体。

失敗モードと境界条件

  • relation が NULL の場合、デリファレンスによりクラッシュする可能性があるため、呼び出し側で保証する必要がある。
  • rd_smgr が既に閉じられている場合、本関数はガード条件により安全に終了する。

保守担当者向けメモ

  • 本関数は RelationGetSmgr によって遅延初期化されたストレージマネージャを明示的に解放するために使用される。
  • リレーションの無効化や破棄のプロセスにおいて、物理リソースを確実に解放するために必須の関数である。
