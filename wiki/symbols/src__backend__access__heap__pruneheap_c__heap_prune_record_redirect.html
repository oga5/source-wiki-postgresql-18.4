heap_prune_record_redirect

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/pruneheap.c:1214

目的

HOT (Heap Only Tuple) チェーンの剪定処理において、特定のタプルをリダイレクト先として記録し、剪定状態（PruneState）を更新する。

シグネチャまたは形状

static void heap_prune_record_redirect(PruneState *prstate,
                                       OffsetNumber offnum, OffsetNumber rdoffnum,
                                       bool was_normal)

入力と出力

  • prstate: 剪定処理の状態を保持する PruneState 構造体へのポインタ。
  • offnum: リダイレクト元となるタプルのオフセット番号。
  • rdoffnum: リダイレクト先となるタプルのオフセット番号。
  • was_normal: リダイレクト元が通常のタプルであったかどうかを示すフラグ。
  • 出力: なし（prstate の内部状態を更新）。

前提条件と不変条件

  • offnum は prstate->processed で未処理である必要がある。
  • prstate->nredirected は MaxHeapTuplesPerPage 未満である必要がある。

アルゴリズムまたは振る舞い

1. offnum を処理済みとしてマークする。 2. prstate->redirected 配列に offnum と rdoffnum のペアを格納し、nredirected をインクリメントする。 3. was_normal が真の場合、prstate->ndeleted をインクリメントする（通常のタプルが削除対象となったため）。 4. prstate->hastup を true に設定する。 5. リダイレクト先（rdoffnum）自体は、この関数内ではマークされない。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/pruneheap.c 内の heap_prune_chain から呼び出されることが確認されている。
  • 呼び出し先: 外部関数への呼び出しは含まれない。

関連データ構造

  • PruneState: 剪定処理中のオフセットや統計情報を管理する構造体。

失敗モードと境界条件

  • prstate->nredirected が MaxHeapTuplesPerPage を超える場合、アサーション違反が発生する。
  • offnum が既に処理済みの場合、アサーション違反が発生する。

保守担当者向けメモ

  • 本関数は HOT チェーンの再構築において、リダイレクト情報を記録する役割を担う。リダイレクト先そのものは別途管理されるため、二重カウントを避ける設計となっている。
