JumbleQuery

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:136

目的

与えられたクエリツリーから正規化されたフィンガープリント（queryId）を生成し、クエリ構造の同一性を識別可能にする。

シグネチャまたは形状

JumbleState *JumbleQuery(Query *query)

入力と出力

  • **入力**: Query *query （フィンガープリント対象のクエリツリー）
  • **出力**: JumbleState * （ハッシュ計算に使用された作業用状態構造体）

前提条件と不変条件

  • IsQueryIdEnabled() が真であること（Assert により検証）。
  • 戻り値の JumbleState は呼び出し元で適切に管理される必要がある。
  • 生成された query->queryId は、ハッシュ値が 0 の場合、ユーティリティクエリであれば 2、それ以外は 1 に補正される。

アルゴリズムまたは振る舞い

1. InitJumble() を呼び出し、ハッシュ計算用の作業バッファを初期化する。 2. DoJumble() を呼び出し、クエリツリーを再帰的に走査してハッシュ値を算出する。 3. 算出された queryId を query->queryId に格納する。 4. ハッシュ値が 0 となった場合、衝突回避のために固定値（1 または 2）に置き換える。 5. 使用した JumbleState を呼び出し元へ返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/createas.c
  • src/backend/commands/explain.c
  • src/backend/commands/portalcmds.c
  • src/backend/parser/analyze.c
  • **呼び出し先**:
  • InitJumble()
  • DoJumble()
  • IsQueryIdEnabled()

関連データ構造

  • Query: フィンガープリント対象となるクエリ構造体。
  • JumbleState: ハッシュ計算の過程でバッファや状態を保持する構造体。

失敗モードと境界条件

  • IsQueryIdEnabled() が偽の場合、アサーション違反により異常終了する。
  • ハッシュ計算結果が 0 になるケースでは、ステートメントの種類に応じて 1 または 2 に強制置換される。

保守担当者向けメモ

  • 本関数はクエリの正規化におけるメインエントリーポイントである。
  • 新しいノード型を追加した際は、DoJumble 経由で _jumbleNode が適切に処理できるよう、queryjumblefuncs.switch.c 等の関連ファイルを更新すること。
