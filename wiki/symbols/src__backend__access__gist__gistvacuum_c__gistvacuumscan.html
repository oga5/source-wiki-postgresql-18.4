gistvacuumscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistvacuum.c:124

目的

GiSTインデックスの物理スキャンを行い、不要なタプルの削除、空ページの特定、および統計情報の更新を行う。

シグネチャまたは形状

static void gistvacuumscan(IndexVacuumInfo *info, IndexBulkDeleteResult *stats, IndexBulkDeleteCallback callback, void *callback_state)

入力と出力

  • **入力**:
  • info: インデックス情報およびVACUUM設定。
  • stats: 処理結果を格納する構造体。
  • callback: 不要タプルを判定するコールバック関数。
  • callback_state: コールバック関数に渡す状態データ。
  • **出力**: なし（stats を通じて統計情報を更新）。

前提条件と不変条件

  • stats の一部フィールド（num_pages, num_index_tuples 等）は、本関数内で初期化される。
  • GenerationContext を使用してメモリ管理を行うため、呼び出し終了時にコンテキストが破棄される。
  • インデックスの拡張と競合しないよう、必要に応じて LockRelationForExtension を使用する。

アルゴリズムまたは振る舞い

1. stats の統計フィールドをリセットする。 2. GistVacState を初期化し、空ページ管理用の IntegerSet を作成する。 3. ReadStream を使用し、インデックスを物理順に走査する。 4. 走査中、インデックスの拡張を考慮して RelationGetNumberOfBlocks を定期的に再確認する。 5. 各ページに対して gistvacuumpage を呼び出し、不要タプルの削除と空ページの記録を行う。 6. 全ページ走査後、回収可能なページがあれば IndexFreeSpaceMapVacuum を実行する。 7. gistvacuum_delete_empty_pages を呼び出し、記録された空リーフページをツリーから切り離す。 8. 最後にメモリコンテキストを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: gistbulkdelete（VACUUM 中の削除処理）から呼び出される。
  • **呼び出し先**:
  • gistvacuumpage: 各ページの処理。
  • gistvacuum_delete_empty_pages: 空ページのツリーからの切り離し。
  • IndexFreeSpaceMapVacuum: FSMの更新。
  • read_stream_* 系関数: インデックスの効率的な走査。

関連データ構造

  • GistVacState: スキャン中に保持する一時的な状態（空ページセット、メモリコンテキスト等）。
  • IndexVacuumInfo, IndexBulkDeleteResult: VACUUMの制御および結果格納用。
  • IntegerSet: 回収対象のページ番号を管理する集合。

失敗モードと境界条件

  • インデックスが空、または新規作成直後の場合は走査が即座に終了する。
  • 走査中にインデックスが拡張された場合、ループ条件の再評価により新しいページも処理対象となる。
  • RELATION_IS_LOCAL の場合はロックをスキップする。

保守担当者向けメモ

  • GenerationContext の初期ブロックサイズ（16KB）は、将来的にベンチマークに基づいた調整が推奨される。
  • gistvacuumpage 内の再帰的処理（goto restart）と本関数のループ構造により、インデックス構造の変化に対応している。
