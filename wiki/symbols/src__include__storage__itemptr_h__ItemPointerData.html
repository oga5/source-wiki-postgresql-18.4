ItemPointerData

定義

  • 種別: struct
  • 定義箇所: src/include/storage/itemptr.h:36

目的

PostgreSQL の物理ストレージにおいて、テーブルファイル内の特定のページ（ブロック）と、そのページ内の特定のタプル位置を指し示す「アイテムポインタ（TID: Tuple Identifier）」を保持する。

シグネチャまたは形状

typedef struct ItemPointerData
{
	BlockIdData ip_blkid;
	OffsetNumber ip_posid;
} ItemPointerData;

入力と出力

  • 本構造体は 6 バイトの固定長データとして扱われる。
  • ItemPointerGetBlockNumber や ItemPointerGetOffsetNumber などのアクセサを通じて、内部のブロック番号およびオフセット番号が抽出される。
  • Datum 型との相互変換が可能であり、システム全体での受け渡しに使用される。

前提条件と不変条件

  • 構造体全体のサイズは 6 バイトである必要がある。
  • コンパイラによるパディングを抑制するため、pg_attribute_packed 等の属性が適用されている。
  • ip_posid が MaxOffsetNumber を超える値を持つ場合、通常のタプル位置ではなく、推論的挿入トークンやパーティション間移動などの特殊状態を示す。

アルゴリズムまたは振る舞い

  • インデックスからヒープタプルへの参照や、タプルの更新履歴（t_ctid）の管理に使用される。
  • ハッシュキーとしても利用されるため、構造の変更には注意が必要である。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）の例:
  • contrib/amcheck/verify_gin.c
  • contrib/amcheck/verify_nbtree.c
  • contrib/bloom/bloom.h
  • contrib/pageinspect/brinfuncs.c
  • 呼び出し先:
  • 構造体定義自体はデータ保持を目的としており、特定の関数を呼び出すものではない。操作は src/include/storage/itemptr.h で定義されたインライン関数群によって行われる。

関連データ構造

  • BlockIdData: ブロック識別子を保持。
  • OffsetNumber: ページ内のオフセット位置を保持。
  • ItemIdData: ページ内のアイテム識別子。

失敗モードと境界条件

  • ip_posid の値が MaxOffsetNumber を超える場合、特殊な状態として解釈されるため、通常のタプル位置として扱うと誤動作の原因となる。

保守担当者向けメモ

  • 本構造体はメモリレイアウトが厳密に定義されているため、フィールドの追加や順序変更はバイナリ互換性に影響を与える可能性がある。
  • ...NoCheck 接尾辞を持つアクセサは Assert を省略するため、パフォーマンスが極めて重要なパスでのみ使用すること。
