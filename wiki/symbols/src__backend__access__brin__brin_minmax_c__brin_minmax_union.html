brin_minmax_union

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_minmax.c:207

目的

2つのBRINページ範囲（BrinValues）の要約情報を統合し、最小値と最大値を更新して col_a に反映させる。

シグネチャまたは形状

Datum brin_minmax_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): BrinDesc *（インデックス記述子）
  • PG_GETARG_POINTER(1): BrinValues *col_a（統合先範囲）
  • PG_GETARG_POINTER(2): BrinValues *col_b（統合元範囲）
  • **出力**: なし（PG_RETURN_VOID() を返す）。col_a の内容が直接更新される。

前提条件と不変条件

  • col_a->bv_attno == col_b->bv_attno であること。
  • col_a および col_b の bv_allnulls が false であること（有効な値を持つ範囲であること）。

アルゴリズムまたは振る舞い

1. col_a と col_b の属性番号が一致することを確認する。 2. minmax_get_strategy_procinfo を使用し、対象属性の比較関数（BTLessStrategyNumber および BTGreaterStrategyNumber）を取得する。 3. **最小値の統合**: col_b の最小値が col_a の最小値より小さい場合、col_a の最小値を col_b の値で更新する。 4. **最大値の統合**: col_b の最大値が col_a の最大値より大きい場合、col_a の最大値を col_b の値で更新する。 5. 値の更新時、型が attbyval でない場合は既存の値を pfree し、datumCopy を用いて新しい値をコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は明示されていません。
  • **呼び出し先**:
  • minmax_get_strategy_procinfo: 比較関数の取得。
  • FunctionCall2Coll: 比較関数の実行。
  • pfree, datumCopy: メモリ管理。

関連データ構造

  • BrinValues: ページ範囲の要約情報を保持する構造体。
  • BrinDesc: インデックスの記述子。
  • FmgrInfo: 比較関数の実行情報。

失敗モードと境界条件

  • bv_allnulls が設定されている範囲を渡すとアサーションエラーが発生する。
  • 比較関数が正しく取得できない場合、minmax_get_strategy_procinfo 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は col_a を破壊的に更新するため、呼び出し側で col_a の整合性が保たれていることを確認すること。
  • attbyval が false の場合のメモリ解放とコピー処理は、他のBRIN演算クラスの実装と共通のパターンであるため、変更時は注意が必要。
