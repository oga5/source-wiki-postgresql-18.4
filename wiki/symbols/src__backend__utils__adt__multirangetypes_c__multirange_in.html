multirange_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes.c:116

目的

テキスト形式のマルチレンジ型リテラルを解析し、内部表現である MultirangeType へ変換する。

シグネチャまたは形状

Datum multirange_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_CSTRING(0): 解析対象の文字列リテラル。
  • PG_GETARG_OID(1): マルチレンジ型のOID。
  • PG_GETARG_INT32(2): 型修飾子（typmod）。
  • **出力**:
  • Datum: 構築された MultirangeType へのポインタ。

前提条件と不変条件

  • 入力文字列は {range1, range2, ...} の形式である必要がある。
  • 内部で multirange_canonicalize を経由する前提で、レンジの集合を保持する。
  • 空のレンジ（RangeIsEmpty が真）は結果のマルチレンジから除外される。

アルゴリズムまたは振る舞い

1. get_multirange_io_data を使用して、対象の型に対応するI/Oキャッシュを取得する。 2. 文字列の先頭の { を確認し、MULTIRANGE_BEFORE_RANGE 状態から解析を開始する。 3. 有限オートマトン（parse_state）を用いて、レンジの開始・終了、クォート、エスケープシーケンスを処理する。 4. 各レンジを InputFunctionCallSafe を介して個別の RangeType に変換し、動的に確保した配列 ranges に格納する。 5. 解析完了後、MultirangeType 構造体として構築する（※ソースコード上、構築後の返却処理まで記述されている）。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのI/Oサブシステム（SQLリテラルの解析時）。
  • **呼び出し先**:
  • get_multirange_io_data: I/Oキャッシュの取得。
  • InputFunctionCallSafe: 個別のレンジ型に対する入力関数の呼び出し。
  • RangeIsEmpty: レンジが空かどうかの判定。

関連データ構造

  • MultirangeIOData: I/O処理用のキャッシュ構造体。
  • MultirangeParseState: 解析状態を管理する列挙型。
  • RangeType: 個々のレンジを保持するデータ構造。

失敗モードと境界条件

  • **不正な形式**: { で始まらない、またはレンジの開始・終了が正しくない場合に ERRCODE_INVALID_TEXT_REPRESENTATION でエラーを投げる。
  • **メモリ不足**: palloc / repalloc 失敗時はPostgreSQLのメモリ管理機構によりエラーとなる。

保守担当者向けメモ

  • 本関数は multirangetypes.c 内の他の関数と異なり、文字列パースに特化している。
  • parse_state の遷移ロジックは、クォートやエスケープを含む複雑な文字列を扱うため、変更時は状態遷移図を更新すること。
  • ranges 配列の動的拡張（range_capacity）は、要素数が増加する際のパフォーマンスに影響するため、初期値の調整は慎重に行うこと。
