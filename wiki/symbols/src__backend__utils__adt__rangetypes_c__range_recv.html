range_recv

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/rangetypes.c:178

目的

バイナリ形式の入力ストリームから範囲型（RangeType）のデータを読み込み、内部表現へ変換する。

シグネチャまたは形状

Datum range_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • PG_GETARG_POINTER(0): StringInfo 形式の入力バッファ。
  • PG_GETARG_OID(1): 対象範囲型の OID。
  • PG_GETARG_INT32(2): 型修飾子（typmod）。
  • **出力**:
  • PG_RETURN_RANGE_P(range): 構築された RangeType へのポインタを Datum として返す。

前提条件と不変条件

  • check_stack_depth() により、再帰的な型定義に対するスタックオーバーフロー保護が行われる。
  • get_range_io_data を通じて、対象型に対応するキャッシュ情報が取得されている必要がある。
  • 入力バッファのフラグは、RANGE_EMPTY, RANGE_LB_INC, RANGE_LB_INF, RANGE_UB_INC, RANGE_UB_INF のいずれかに正規化される。

アルゴリズムまたは振る舞い

1. get_range_io_data を呼び出し、型キャッシュを取得する。 2. バッファから 1 バイトのフラグを読み取り、不要なビットをマスクする。 3. 下限（LB）および上限（UB）が存在する場合、それぞれの長さを読み取り、ReceiveFunctionCall を介して要素型のバイナリデシリアライズを実行する。 4. 読み取ったフラグと境界値から RangeBound 構造体を構成する。 5. make_range を呼び出し、正規化およびシリアライズを経て RangeType を生成する。 6. pq_getmsgend でバッファの終端を確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のバイナリプロトコル経由で範囲型データを受信する際に呼び出される（直接的な呼び出し元コードは提供されたソース内には明示されていない）。
  • **呼び出し先**: get_range_io_data, pq_getmsgbyte, pq_getmsgint, pq_getmsgbytes, ReceiveFunctionCall, make_range, pq_getmsgend。

関連データ構造

  • RangeType: 範囲型の内部表現。
  • RangeBound: 境界値（値、無限大フラグ、包含フラグ）を保持する構造体。
  • RangeIOData: 型の入出力に必要なキャッシュ情報。
  • StringInfo: 入力バッファを管理する構造体。

失敗モードと境界条件

  • 入力バッファの形式が不正な場合、pq_getmsg... 関数群がエラーを発生させる。
  • 要素型のデシリアライズ中にエラーが発生した場合、ReceiveFunctionCall が例外を投げる。
  • 範囲の境界値が不正な場合、make_range 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は RangeType のバイナリ受信の入り口である。
  • 境界値のメモリ管理には pfree を使用しており、StringInfoData の一時的な割り当てを適切に解放している。
  • RANGE_xB_NULL フラグは明示的にマスクされており、後続の処理で混乱を招かないよう設計されている。
