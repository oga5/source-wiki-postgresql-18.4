gets_interactive

定義

  • 種別: function
  • 定義箇所: src/bin/psql/input.c:66

目的

対話モードにおいて、ユーザーからの入力を取得する。環境に応じて readline ライブラリまたは標準入力からの読み込みを切り替える。

シグネチャまたは形状

char *gets_interactive(const char *prompt, PQExpBuffer query_buf);

入力と出力

  • **入力**:
  • prompt: 入力時に表示するプロンプト文字列。
  • query_buf: 現在のクエリバッファ（タブ補完コールバックで使用）。
  • **出力**:
  • ユーザーが入力した文字列（malloc されたメモリ領域）。呼び出し元で解放する必要がある。

前提条件と不変条件

  • readline が有効な場合、sigint_interrupt_jmp が適切に設定されている必要がある（SIGINT 発生時に longjmp を行うため）。
  • readline 使用時は、終了後に tab_completion_query_buf が NULL にリセットされる。

アルゴリズムまたは振る舞い

1. **readline 有効時**:

  • rl_reset_screen_size() を呼び出し、端末サイズを更新する。
  • tab_completion_query_buf に query_buf をセットする。
  • sigint_interrupt_enabled を true に設定し、readline(prompt) を実行する。
  • 終了後、sigint_interrupt_enabled を false に戻し、tab_completion_query_buf を NULL にクリアして結果を返す。

    • 2. **readline 無効時**:

  • prompt を標準出力に書き出し、バッファをフラッシュする。
  • gets_fromFile(stdin) を呼び出し、標準入力から行を読み取って返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/mainloop.c（メインループでの入力取得）。
  • **呼び出し先**:
  • readline（外部ライブラリ）
  • rl_reset_screen_size（外部ライブラリ）
  • gets_fromFile（src/bin/psql/input.c 内）

関連データ構造

  • PQExpBuffer: クエリ文字列を保持するバッファ構造体。
  • tab_completion_query_buf: タブ補完時に参照されるグローバル変数。
  • sigint_interrupt_enabled: SIGINT 割り込みの有効/無効を制御するフラグ。

失敗モードと境界条件

  • readline が利用できない環境では、標準入力からの読み込みにフォールバックする。
  • SIGINT が発生した場合、sigint_interrupt_enabled が true であれば longjmp が実行され、この関数からの正常な復帰は行われない。

保守担当者向けメモ

  • readline 使用時の SIGWINCH 処理は、readline が入力を待機していない間に発生したシグナルを補足するための暫定的な対応である。
  • libedit と readline の差異については、モジュール内のマクロ定義や関連する履歴管理関数を参照すること。
