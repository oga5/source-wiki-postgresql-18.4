pg_wc_isdigit

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_pg_locale.c:298

目的

正規表現エンジンにおいて、現在のロケール設定に基づき、指定されたワイド文字（pg_wchar）が数字であるかを判定します。

シグネチャまたは形状

int pg_wc_isdigit(pg_wchar c)

入力と出力

  • **入力**: pg_wchar c（判定対象の文字コード）
  • **出力**: int（数字であれば非ゼロ、そうでなければ 0 を返す）

前提条件と不変条件

  • pg_regex_strategy が適切に初期化されている必要があります。
  • pg_wchar は符号なし型として扱われます。
  • pg_regex_locale が現在のロケール情報として有効である必要があります。

アルゴリズムまたは振る舞い

現在の pg_regex_strategy に応じて以下のいずれかの戦略で判定を行います。 1. **PG_REGEX_STRATEGY_C**: pg_char_properties 配列を参照し、ASCII範囲内の数字かを確認します。 2. **PG_REGEX_STRATEGY_BUILTIN**: PostgreSQL組み込みのUnicode判定関数 pg_u_isdigit を使用します。 3. **PG_REGEX_STRATEGY_LIBC_WIDE**: iswdigit_l を使用します（wchar_t のサイズまたは文字コード範囲に応じて分岐）。 4. **PG_REGEX_STRATEGY_LIBC_1BYTE**: isdigit_l を使用します。 5. **PG_REGEX_STRATEGY_ICU**: ICUライブラリの u_isdigit を使用します（USE_ICU 定義時）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_locale.c: pg_ctype_get_cache 内および文字クラス判定ロジックから呼び出されます。
  • **呼び出し先**:
  • 戦略に応じて pg_u_isdigit, iswdigit_l, isdigit_l, u_isdigit を呼び出します。

関連データ構造

  • pg_wchar: 判定対象の文字型。
  • PG_Locale_Strategy: 判定戦略を決定する列挙型。
  • pg_regex_locale: ロケール固有の情報を保持する構造体。

失敗モードと境界条件

  • PG_REGEX_STRATEGY_ICU が選択されているが USE_ICU が定義されていない場合、コンパイルエラーまたは未定義の挙動となる可能性があります。
  • どの戦略にも該当しない場合、関数は 0 を返します。

保守担当者向けメモ

  • 本関数は regcomp.c からインクルードされることを前提としています。
  • pg_wchar の値が各戦略の許容範囲を超えた場合の挙動は、各戦略の実装に依存します。
  • iswdigit_l や isdigit_l はロケール依存であるため、マルチスレッド環境下でのロケールの一貫性に注意してください。
