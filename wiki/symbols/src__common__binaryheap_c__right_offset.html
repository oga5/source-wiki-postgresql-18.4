right_offset

定義

  • 種別: static inline function
  • 定義箇所: src/common/binaryheap.c:95

目的

二分ヒープの配列表現において、指定されたインデックス i にあるノードの「右の子」ノードのインデックスを算出します。

シグネチャまたは形状

static inline int right_offset(int i)

入力と出力

  • **入力**: i (int型) - 対象となるノードの現在のインデックス。
  • **出力**: 2 * i + 2 (int型) - ヒープ配列内における右の子ノードのインデックス。

前提条件と不変条件

  • 入力 i は、ヒープ配列の有効なインデックス範囲内であることを想定しています。
  • 0-indexed の二分ヒープ構造に基づいています。

アルゴリズムまたは振る舞い

  • 渡されたインデックス i に対して 2 * i + 2 を計算して返します。
  • この計算は、二分ヒープの完全二分木構造における右の子ノードの位置を特定するための標準的な算術演算です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/binaryheap.c:325
  • src/backend/utils/sort/logtape.c:402
  • **呼び出し先**:
  • 特になし（算術演算のみを実行）。

関連データ構造

  • binaryheap 構造体（src/include/lib/binaryheap.h で定義）が管理するノード配列のインデックス計算に使用されます。

失敗モードと境界条件

  • 入力 i が負の値や、ヒープの最大サイズを超える値である場合、戻り値は配列の範囲外を指す可能性があります。本関数自体は境界チェックを行いません。

保守担当者向けメモ

  • 本関数は static inline で定義されており、コンパイル時にインライン展開されることが期待されています。
  • 呼び出し元で uint64 型を扱うケース（logtape.c）があるため、型変換やオーバーフローの可能性には注意が必要です。
