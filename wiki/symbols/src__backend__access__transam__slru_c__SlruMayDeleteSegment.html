SlruMayDeleteSegment

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/slru.c:1602

目的

指定された SLRU セグメントファイルが、指定されたカットオフページよりも十分に古い（削除可能である）かどうかを判定する。

シグネチャまたは形状

static bool SlruMayDeleteSegment(SlruCtl ctl, int64 segpage, int64 cutoffPage)

入力と出力

  • **入力**:
  • ctl: SLRU 制御構造体。ページ順序判定用のコールバック関数を含む。
  • segpage: 判定対象のセグメントの先頭ページ番号。
  • cutoffPage: 削除の基準となるカットオフページ番号。
  • **出力**:
  • bool: セグメント内の全ページが cutoffPage より前であると判定された場合に true を返す。

前提条件と不変条件

  • segpage は SLRU_PAGES_PER_SEGMENT の倍数であること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. セグメント内の最終ページ番号 seg_last_page を segpage + SLRU_PAGES_PER_SEGMENT - 1 として算出する。 2. ctl->PagePrecedes コールバックを使用し、セグメントの先頭ページと最終ページの両方が cutoffPage よりも前であることを確認する。 3. 両方の条件が満たされた場合のみ true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/slru.c 内のセグメント削除処理（SlruScanDirCbDeleteCutoff や SlruScanDirCbDeleteAll 等の文脈）から呼び出される。
  • 削除実行前の安全確認として Assert 内でも使用されている。
  • **呼び出し先**:
  • ctl->PagePrecedes: SlruCtl 構造体で定義された、ページ番号の前後関係を判定する関数ポインタ。

関連データ構造

  • SlruCtl: SLRU の動作を制御する構造体。PagePrecedes コールバックを保持する。

失敗モードと境界条件

  • segpage が SLRU_PAGES_PER_SEGMENT の倍数でない場合、アサーション違反によりプロセスが停止する。
  • PagePrecedes コールバックの定義が不適切な場合、誤った判定を下す可能性がある。

保守担当者向けメモ

  • 本関数はセグメント全体が削除対象として安全かを判定する保守的なロジックである。
  • PagePrecedes は各サブシステム（CLOG, MultiXact 等）で個別に定義されるため、循環するトランザクション ID の扱いに注意すること。
