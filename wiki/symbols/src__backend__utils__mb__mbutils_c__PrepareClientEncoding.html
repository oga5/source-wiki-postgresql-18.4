PrepareClientEncoding

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/mbutils.c:118

目的

指定されたクライアントエンコーディングへの切り替えに必要な変換関数を検索し、キャッシュへ登録する準備を行う。

シグネチャまたは形状

int PrepareClientEncoding(int encoding)

入力と出力

  • **入力**: encoding (対象となるクライアントエンコーディングのID)
  • **出力**: 成功時は 0、失敗時は -1 を返す。また、起動直後など処理が不要な場合は 0 を返す。

前提条件と不変条件

  • encoding は PG_VALID_FE_ENCODING マクロで検証可能であること。
  • トランザクション状態（IsTransactionState()）にある場合のみ、カタログ検索による新規キャッシュ登録が可能。
  • キャッシュ登録は TopMemoryContext に対して行われる。

アルゴリズムまたは振る舞い

1. エンコーディングの妥当性を確認し、バックエンド起動完了前であれば即座に 0 を返す。 2. サーバーエンコーディングと対象エンコーディングが同一、または一方が PG_SQL_ASCII の場合は変換不要と判断し 0 を返す。 3. トランザクション内の場合:

  • FindDefaultConversionProc を用いてサーバー・クライアント間の変換関数（Oid）を検索する。
  • 変換関数が見つかれば fmgr_info_cxt で情報を取得し、ConvProcInfo 構造体に格納して ConvProcList の先頭に追加する。

    • 4. トランザクション外の場合:

  • 既存の ConvProcList を走査し、該当するエンコーディングペアが既にキャッシュされているか確認する。キャッシュがあれば 0、なければ -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/parallel.c および src/backend/commands/variable.c 等から呼び出される。
  • **呼び出し先**: GetDatabaseEncoding, FindDefaultConversionProc, fmgr_info_cxt, MemoryContextAlloc, lcons 等。

関連データ構造

  • ConvProcInfo: 変換関数のキャッシュ情報を保持する構造体。
  • ConvProcList: ConvProcInfo を保持するリスト。

失敗モードと境界条件

  • PG_VALID_FE_ENCODING に適合しないエンコーディングが指定された場合は -1 を返す。
  • トランザクション外で、かつキャッシュに存在しないエンコーディングへの切り替えを試みた場合は -1 を返す。
  • 必要な変換関数がカタログに見つからない場合は -1 を返す。

保守担当者向けメモ

  • 本関数はキャッシュの追加のみを行い、古いエントリの削除は行わない。削除処理は SetClientEncoding が担当する。
  • トランザクション外での呼び出しはキャッシュの参照のみに制限されるため、設定変更のタイミングには注意が必要である。
