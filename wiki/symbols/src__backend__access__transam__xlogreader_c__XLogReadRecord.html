XLogReadRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogreader.c:389

目的

次の有効なWALレコードを読み取り、デコードして返す。WALの物理的な読み込みから論理的なレコード構造への変換までを抽象化するメイン関数である。

シグネチャまたは形状

XLogRecord *XLogReadRecord(XLogReaderState *state, char **errormsg)

入力と出力

  • **入力**:
  • state: XLogReaderState 構造体へのポインタ。リーダーの状態を保持する。
  • errormsg: エラー発生時にメッセージを格納するためのポインタ。
  • **出力**:
  • 成功時: 読み取られたレコードのヘッダー (XLogRecord *) を返す。
  • 失敗時: NULL を返す。

前提条件と不変条件

  • 呼び出し前に XLogReaderAllocate 等で state が適切に初期化されている必要がある。
  • 返されるポインタは、次に XLogReadRecord が呼び出されるまで有効である。
  • 内部的に state->record とデコードされたレコードが一致していることがアサーションで保証される。

アルゴリズムまたは振る舞い

1. XLogReleasePreviousRecord(state) を呼び出し、直前に返されたレコードを解放する。 2. XLogReaderHasQueuedRecordOrError を確認し、キューにレコードがない場合は XLogReadAhead を非ブロッキングモードで呼び出し、読み込みを試みる。 3. XLogNextRecord(state, errormsg) を呼び出し、デコード済みのレコードを取得する。 4. 取得できた場合、そのレコードのヘッダー部分へのポインタを返す。取得できない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_walinspect/pg_walinspect.c
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xlogprefetcher.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/access/transam/xlogreader.c (内部ループ等)
  • **呼び出し先**:
  • XLogReleasePreviousRecord
  • XLogReaderHasQueuedRecordOrError
  • XLogReadAhead
  • XLogNextRecord

関連データ構造

  • XLogReaderState: リーダーの状態を保持するメイン構造体。
  • DecodedXLogRecord: デコードされたレコード情報を保持する構造体。
  • XLogRecord: WALレコードの物理的なヘッダー構造体。

失敗モードと境界条件

  • WALの読み込みエラーやCRCチェック失敗が発生した場合、XLogNextRecord を通じて errormsg に詳細が格納され、関数は NULL を返す。
  • 読み込むべきレコードが存在しない場合、NULL を返す。

保守担当者向けメモ

  • 本関数は XLogReaderState を介してメモリ管理を行うため、フロントエンドツールで使用する際は適切なメモリコンテキストが設定されていることを確認すること。
  • XLogRecGetXXX マクロを使用してレコードの内容にアクセスする設計となっている。
