gbt_oidle

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_oid.c:43

目的

2つの Oid 型の値に対し、左辺が右辺以下であるか（less than or equal）を判定する比較関数です。

シグネチャまたは形状

static bool gbt_oidle(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の Oid 値へのポインタ。
  • b: 比較対象の Oid 値へのポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a <= *b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な Oid 型の値へのポインタである必要があります。
  • 本関数は Oid 型の値を直接キャストして比較するため、メモリ上の値が Oid として正当であることが前提です。

アルゴリズムまたは振る舞い

1. 引数 a と b をそれぞれ const Oid * にキャストします。 2. ポインタが指す値を参照し、C言語の <= 演算子を用いて大小比較を行います。 3. 比較結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_oid.c 内の静的配列（tinfo 構造体初期化時）から参照されています。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（演算子のみを使用）。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子型。
  • FmgrInfo: PostgreSQLの関数管理構造体。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、セグメンテーション違反が発生します。
  • Oid 型の範囲外の値が渡された場合の挙動は、標準的な整数比較のルールに従います。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける Oid 型の比較サポートの一部です。
  • 類似の比較関数（gbt_oidlt, gbt_oidge 等）と一貫した実装になっています。
  • flinfo 引数は現在の実装では使用されていませんが、GiSTの汎用インターフェースとの互換性のために保持されています。
