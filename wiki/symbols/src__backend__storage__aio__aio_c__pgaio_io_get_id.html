pgaio_io_get_id

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio.c:341

目的

指定された PgAioHandle が、システム全体で管理されている pgaio_ctl->io_handles 配列内のどのインデックス（ID）に位置するかを算出する。

シグネチャまたは形状

int pgaio_io_get_id(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh - 対象となるI/Oハンドルのポインタ。
  • **出力**: int - ハンドル配列内でのオフセット値（0から始まるID）。

前提条件と不変条件

  • ioh は pgaio_ctl->io_handles 配列の有効な範囲内にある必要がある。
  • 実行時に Assert によって、ioh が pgaio_ctl->io_handles の先頭アドレス以上、かつ io_handles + io_handle_count 未満であることが検証される。

アルゴリズムまたは振る舞い

1. ioh ポインタと pgaio_ctl->io_handles 配列の先頭アドレスとの差分を計算する。 2. ポインタ演算の結果として、配列内でのインデックスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio.c (デバッグログ出力時など)
  • src/backend/storage/aio/aio_funcs.c:59
  • src/backend/storage/aio/method_worker.c:195 (キューへのID格納)
  • **呼び出し先**:
  • 本関数はポインタ演算のみを行うため、他の関数を呼び出さない。

関連データ構造

  • PgAioHandle: I/O操作の状態を保持する構造体。
  • pgaio_ctl: AIOサブシステム全体の制御情報を保持するグローバル構造体。

失敗モードと境界条件

  • ioh が NULL または pgaio_ctl->io_handles 配列外のメモリを指している場合、Assert が有効なビルドではアボートする。

保守担当者向けメモ

  • 本関数はポインタの相対位置を計算するのみであり、状態遷移やメモリ確保は行わない。
  • pgaio_ctl が初期化される前に呼び出された場合の挙動は未定義である。
