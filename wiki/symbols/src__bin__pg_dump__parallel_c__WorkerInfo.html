WorkerInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/parallel.c:125

目的

pg_dump における並列処理の各ワーカープロセスに関する情報を保持し、リーダープロセスとワーカー間の連携を管理する。

シグネチャまたは形状

typedef struct
{
	ArchiveHandle *AH;			/* leader database connection */
	ParallelSlot *slot;			/* this worker's parallel slot */
} WorkerInfo;

入力と出力

  • 入力: なし（構造体定義）
  • 出力: なし（構造体定義）

前提条件と不変条件

  • AH はリーダー側のデータベース接続を指す必要がある。
  • slot は当該ワーカーに割り当てられた通信パイプやプロセス状態を管理する ParallelSlot を指す必要がある。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用であり、ロジックは含まない。parallel.c 内の並列処理フローにおいて、リーダーがワーカーの状態を追跡するために使用される。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内では、本構造体の具体的なインスタンス化箇所や参照箇所は明示されていない。
  • 呼び出し先: なし。
  • 注意: ツールが提示した src/backend/postmaster/autovacuum.c 内の WorkerInfo への言及は、本シンボル（src/bin/pg_dump/parallel.c）とは異なるコンテキスト（バックエンドの自動バキューム機能）である可能性が高く、直接的な関連性は確認できない。

関連データ構造

  • ArchiveHandle: リーダー側のアーカイブ管理構造体。
  • ParallelSlot: ワーカーの通信パイプやプロセスIDを保持する構造体。

失敗モードと境界条件

  • AH または slot が NULL の状態でアクセスされた場合、並列処理の制御ロジックがクラッシュする可能性がある。

保守担当者向けメモ

  • 本構造体は pg_dump の並列処理専用である。バックエンド側の WorkerInfo と混同しないよう注意すること。
  • プラットフォーム依存（Unix/Windows）の差異については、ParallelSlot 側で吸収される設計となっている。
