inline_cte_walker_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/subselect.c:60

目的

CTE（Common Table Expression）のインライン展開を行う際、対象となるCTEを特定し、クエリツリー内を走査するための状態を保持するコンテキスト構造体です。

シグネチャまたは形状

typedef struct inline_cte_walker_context
{
    const char *ctename;        /* ターゲットCTEの名前 */
    int         levelsup;       /* ターゲットCTEの相対的なレベル */
    Query      *ctequery;       /* 置換対象のクエリ */
} inline_cte_walker_context;

入力と出力

  • 本構造体は、inline_cte_walker 関数に渡される引数として使用されます。
  • ctename および levelsup は、走査対象のCTEを識別するための入力条件として機能します。
  • ctequery は、走査中にCTE参照が見つかった場合に置換元として利用されます。

前提条件と不変条件

  • ctename は、インライン化対象のCTE名と一致している必要があります。
  • levelsup は、現在のクエリレベルからCTE定義までの階層差を正しく反映している必要があります。

アルゴリズムまたは振る舞い

  • inline_cte_walker 関数がクエリツリーを再帰的に探索する際、この構造体を参照して、現在のノードがターゲットのCTE参照であるかを判定します。
  • 一致した場合、ctequery を用いて該当箇所をインライン展開する処理の制御に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/plan/subselect.c 内の関数（inline_cte_walker を呼び出す箇所）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • Query: PostgreSQLのクエリ表現ノード。ctequery フィールドで参照されます。
  • inline_cte_walker: 本構造体を引数として受け取り、ツリーを走査する関数。

失敗モードと境界条件

  • 構造体内の各フィールドが適切に初期化されていない場合、CTEの識別が正しく行われず、インライン展開が失敗または誤った箇所で実行される可能性があります。

保守担当者向けメモ

  • 本構造体は inline_cte_walker 関数専用の補助データ構造です。
  • CTEのネストやスコープに関連する変更を行う際は、levelsup の計算ロジックと本構造体の整合性を確認してください。
