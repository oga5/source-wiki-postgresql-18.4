itemptr_to_uint64

定義

  • 種別: static inline function
  • 定義箇所: src/backend/access/gin/ginpostinglist.c:86

目的

ItemPointer（TID）を、差分圧縮に適した単一の64ビット整数値に変換します。

シグネチャまたは形状

static inline uint64 itemptr_to_uint64(const ItemPointer iptr)

入力と出力

  • **入力**: ItemPointer 型のポインタ iptr。
  • **出力**: uint64 型の変換済み整数値。

前提条件と不変条件

  • iptr は有効な ItemPointer であること（ItemPointerIsValid が真）。
  • iptr のオフセット番号は 1 << MaxHeapTuplesPerPageBits 未満であること。

アルゴリズムまたは振る舞い

1. 入力された ItemPointer からブロック番号とオフセット番号を取得します。 2. ブロック番号を MaxHeapTuplesPerPageBits 分だけ左シフトします。 3. シフトした値に対してオフセット番号をビット論理和（OR）で結合し、64ビット整数として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginpostinglist.c:217
  • src/backend/access/gin/ginpostinglist.c:223
  • src/backend/access/gin/ginpostinglist.c:329
  • **呼び出し先**:
  • GinItemPointerGetBlockNumber (マクロ)
  • GinItemPointerGetOffsetNumber (マクロ)
  • ItemPointerIsValid (マクロ)

関連データ構造

  • ItemPointer: GINインデックスで管理されるタプルの物理位置情報。
  • MaxHeapTuplesPerPageBits: ページ内の最大タプル数を表現するために必要なビット数。

失敗モードと境界条件

  • Assert により、無効なポインタや許容範囲を超えるオフセット番号が渡された場合にプログラムが停止します。

保守担当者向けメモ

  • 本関数は ginPostingList の圧縮処理において、TIDをソート可能な整数値として扱うために不可欠です。
  • MaxHeapTuplesPerPageBits の値を変更する場合、ディスクフォーマットの互換性に影響するため注意が必要です。
