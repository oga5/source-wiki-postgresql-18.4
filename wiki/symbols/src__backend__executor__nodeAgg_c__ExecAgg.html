ExecAgg

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAgg.c:2243

目的

SQLの集約関数および GROUP BY 句の実行を担うエグゼキュータノードのメイン処理関数です。プランナが生成した Agg ノードに基づき、入力タプルをグループ化して集約状態を更新し、結果を生成します。

シグネチャまたは形状

TupleTableSlot *ExecAgg(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（AggState 構造体へのポインタとしてキャストして使用）
  • **出力**: TupleTableSlot *（集約結果のタプル、または集約完了時に NULL）

前提条件と不変条件

  • pstate は AggState 型である必要があります。
  • CHECK_FOR_INTERRUPTS() が呼び出され、クエリの中断要求を監視します。
  • node->agg_done が真の間、集約処理が継続されます。

アルゴリズムまたは振る舞い

1. node->agg_done が偽の場合、node->phase->aggstrategy に基づいて以下のいずれかの戦略を実行します。

  • AGG_HASHED: node->table_filled が偽であれば agg_fill_hash_table を呼び出しハッシュテーブルを構築し、その後 agg_retrieve_hash_table で結果を取得します。
  • AGG_MIXED: agg_retrieve_hash_table を呼び出し結果を取得します。
  • AGG_PLAIN または AGG_SORTED: agg_retrieve_direct を呼び出し、ソート済み入力に対する集約結果を取得します。

    • 2. 取得した result が NULL でない場合はそのスロットを返します。 3. すべての処理が完了した場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（aggstate->ss.ps.ExecProcNode = ExecAgg; により設定される）。
  • **呼び出し先**:
  • agg_fill_hash_table
  • agg_retrieve_hash_table
  • agg_retrieve_direct
  • castNode (マクロ)
  • CHECK_FOR_INTERRUPTS (マクロ)
  • TupIsNull (マクロ)

関連データ構造

  • AggState: 集約実行時の状態を保持する主要構造体。
  • PlanState: 実行プランのノード状態。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • agg_done が真の場合、直ちに NULL を返します。
  • result が NULL の場合、集約処理が終了したとみなされます。

保守担当者向けメモ

  • 本関数は集約戦略（ハッシュ、ソート、混合）のディスパッチャーとして機能します。
  • メモリ制限（work_mem）やディスクスピル処理の詳細は、呼び出し先の agg_fill_hash_table 等に委譲されています。
