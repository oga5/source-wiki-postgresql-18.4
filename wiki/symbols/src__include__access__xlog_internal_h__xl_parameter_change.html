xl_parameter_change

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlog_internal.h:273

目的

データベースの実行時パラメータ（設定値）が変更された際に、その変更内容をWAL（先行書き込みログ）に記録するための構造体です。

シグネチャまたは形状

typedef struct xl_parameter_change
{
	int			MaxConnections;
	int			max_worker_processes;
	int			max_wal_senders;
	int			max_prepared_xacts;
	int			max_locks_per_xact;
	int			wal_level;
	bool		wal_log_hints;
	bool		track_commit_timestamp;
} xl_parameter_change;

入力と出力

  • **入力**: データベース設定の変更時、各フィールドに現在の設定値が格納されます。
  • **出力**: WALレコードの一部としてディスクに書き込まれ、リカバリ時やレプリケーションのデコード時に読み取られます。

前提条件と不変条件

  • 本構造体は、WALレコードのデータペイロードとして使用されます。
  • 構造体の各フィールドは、PostgreSQLの主要な構成パラメータに対応しています。

アルゴリズムまたは振る舞い

1. **書き込み**: 設定変更が発生した際、xl_parameter_change 構造体に値をコピーし、WALレコードとして書き出します。 2. **読み込み**: リカバリ処理や論理デコード処理において、WALレコードから本構造体へデータを復元（memcpy等）し、現在のシステム状態を更新または検証するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/rmgrdesc/xlogdesc.c
  • src/backend/access/transam/xlog.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/replication/logical/decode.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • XLogRecData: WALレコードの構成に使用される構造体。
  • XLogReaderState: WAL読み込み時に本構造体を含むレコードを解析するために使用されます。

失敗モードと境界条件

  • **データ不整合**: WALレコードのサイズと本構造体のサイズが一致しない場合、memcpy による復元時にメモリ破壊や読み取りエラーが発生する可能性があります。
  • **互換性**: PostgreSQLのバージョン間で本構造体の定義が変更された場合、古いWALログの読み込みに失敗する可能性があります。

保守担当者向けメモ

  • 本構造体に新しいパラメータを追加する場合は、WALの互換性を考慮する必要があります。
  • src/backend/access/rmgrdesc/xlogdesc.c 内の記述子関数も併せて更新し、ログ出力時に新しいパラメータが表示されるようにしてください。
