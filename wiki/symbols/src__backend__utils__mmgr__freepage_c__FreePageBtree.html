FreePageBtree

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/freepage.c:108

目的

メモリ管理サブシステムにおいて、空き領域の範囲をページ番号順に効率的に管理・検索するための B-tree ノード構造体です。

シグネチャまたは形状

struct FreePageBtree
{
    FreePageBtreeHeader hdr;
    union
    {
        FreePageBtreeInternalKey internal_key[FPM_ITEMS_PER_INTERNAL_PAGE];
        FreePageBtreeLeafKey leaf_key[FPM_ITEMS_PER_LEAF_PAGE];
    } u;
};

入力と出力

  • 本構造体はメモリ管理対象領域内に配置され、メタデータとして機能します。
  • u 共用体により、内部ノード（internal_key）またはリーフノード（leaf_key）のいずれかとして振る舞います。

前提条件と不変条件

  • 管理対象のメモリ領域をメタデータとして再帰的に利用するため、領域が完全に枯渇した状態では本構造体の確保も不可能となります。
  • B-tree の構造は、隣接する空き領域の高速なマージを可能にするために維持されます。

アルゴリズムまたは振る舞い

  • 空き領域の検索、挿入、および削除操作の基盤となります。
  • 内部ノードは子ノードへのポインタを保持し、リーフノードは実際の空き領域の範囲情報を保持します。
  • ノードが空になった場合、FreePageBtreeRecycle を通じて再利用リストへ移動されます。

呼び出し元と呼び出し先

  • **呼び出し元**: FreePageManagerGet（確保時）、FreePageManagerPut（解放時）などの管理関数から間接的に参照されます。
  • **呼び出し先**: 以下の関数が本構造体を引数として操作を行います。
  • FreePageBtreeFindLeftSibling / FreePageBtreeFindRightSibling
  • FreePageBtreeFirstKey
  • FreePageBtreeGetRecycled
  • FreePageBtreeInsertInternal / FreePageBtreeInsertLeaf
  • FreePageBtreeRemove

関連データ構造

  • FreePageBtreeHeader: ノードのメタデータヘッダ。
  • FreePageBtreeInternalKey: 内部ノード用のキー構造。
  • FreePageBtreeLeafKey: リーフノード用のキー構造。
  • FreePageManager: 本構造体を含むメモリ管理全体のメイン構造体。

失敗モードと境界条件

  • メモリ枯渇時: メタデータ用ノードの確保に失敗する可能性があります。
  • 整合性エラー: FPM_EXTRA_ASSERTS が有効な場合、sum_free_pages による整合性チェックで不整合が検出されることがあります。

保守担当者向けメモ

  • 本構造体は src/include/utils/freepage.h にも定義が存在します。
  • 構造体のサイズは管理対象のページサイズ（4kB）に適合するように設計されています。
  • 変更を加える際は、再帰的なメモリ確保の制約に注意してください。
