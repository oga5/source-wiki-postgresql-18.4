GetForeignServerExtended

定義

  • 種別: function
  • 定義箇所: src/backend/foreign/foreign.c:123

目的

システムカタログ pg_foreign_server から外部サーバーの定義情報を取得し、メモリ上の ForeignServer 構造体に変換して返します。

シグネチャまたは形状

ForeignServer *GetForeignServerExtended(Oid serverid, bits16 flags)

入力と出力

  • **入力**:
  • serverid (Oid): 取得対象の外部サーバーの OID。
  • flags (bits16): 動作フラグ（例: FSV_MISSING_OK）。
  • **出力**:
  • 成功時: ForeignServer 構造体へのポインタ。
  • 失敗時: flags に FSV_MISSING_OK が含まれていない場合、エラーが発生。含まれている場合は NULL を返す。

前提条件と不変条件

  • serverid に対応するシステムカタログエントリが存在する必要があります（FSV_MISSING_OK 指定時を除く）。
  • 返される ForeignServer 構造体は palloc によってメモリ確保されるため、呼び出し側で適切に管理する必要があります。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して FOREIGNSERVEROID キャッシュから対象のタプルを検索します。 2. タプルが見つからない場合、flags を確認し、FSV_MISSING_OK が設定されていなければ elog(ERROR) を発行します。 3. タプルが存在する場合、GETSTRUCT で Form_pg_foreign_server を取得します。 4. palloc で ForeignServer 構造体を確保し、サーバー名、所有者、FDW ID をコピーします。 5. SysCacheGetAttr を使用して、サーバータイプ、バージョン、およびオプション（srvoptions）を抽出します。オプションは untransformRelOptions を介して List 形式に変換されます。 6. ReleaseSysCache でタプルを解放し、構築した構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/connection.c
  • src/backend/catalog/objectaddress.c
  • src/backend/foreign/foreign.c (ラッパー関数 GetForeignServer からの呼び出し)
  • **呼び出し先**:
  • SearchSysCache1, ReleaseSysCache, SysCacheGetAttr (システムキャッシュ操作)
  • palloc, pstrdup (メモリ管理)
  • untransformRelOptions (オプション変換)
  • TextDatumGetCString (データ変換)

関連データ構造

  • ForeignServer: 外部サーバーの定義を保持する構造体。
  • Form_pg_foreign_server: システムカタログ pg_foreign_server の行形式。

失敗モードと境界条件

  • 指定された serverid が存在しない場合、FSV_MISSING_OK フラグがないとエラーで終了します。
  • サーバータイプやバージョンが NULL の場合、構造体の対応するフィールドも NULL に設定されます。
  • オプションが設定されていない場合、options フィールドは NIL になります。

保守担当者向けメモ

  • 本関数は pg_foreign_server カタログのスキーマ変更に直接依存します。カタログ定義（pg_foreign_server.h）が変更された場合は、属性取得箇所を更新する必要があります。
  • untransformRelOptions を使用しているため、オプションのパースロジックは access/reloptions.h の仕様に従います。
