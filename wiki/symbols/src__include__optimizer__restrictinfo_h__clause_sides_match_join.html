clause_sides_match_join

定義

  • 種別: static inline function
  • 定義箇所: src/include/optimizer/restrictinfo.h:72

目的

結合条件（RestrictInfo）が、指定された外部リレーション集合（outerrelids）および内部リレーション集合（innerrelids）に対して適合するかを判定し、適合する場合は結合の左右の向きをメタデータとして記録します。

シグネチャまたは形状

static inline bool clause_sides_match_join(RestrictInfo *rinfo, Relids outerrelids, Relids innerrelids)

入力と出力

  • **入力**:
  • rinfo: 評価対象の RestrictInfo 構造体へのポインタ。
  • outerrelids: 外部結合側と見なされるリレーションの集合（Relids）。
  • innerrelids: 内部結合側と見なされるリレーションの集合（Relids）。
  • **出力**:
  • 適合する場合は true、適合しない場合は false を返します。

前提条件と不変条件

  • rinfo は有効な RestrictInfo 構造体を指している必要があります。
  • この関数は、rinfo->left_relids および rinfo->right_relids が、指定された outerrelids と innerrelids のいずれかの組み合わせに完全に含まれる（サブセットである）ことを検証します。

アルゴリズムまたは振る舞い

1. rinfo->left_relids が outerrelids のサブセットであり、かつ rinfo->right_relids が innerrelids のサブセットである場合：

  • rinfo->outer_is_left を true に設定し、true を返します。

    • 2. 上記に該当せず、rinfo->left_relids が innerrelids のサブセットであり、かつ rinfo->right_relids が outerrelids のサブセットである場合：

  • rinfo->outer_is_left を false に設定し、true を返します。

    • 3. いずれにも該当しない場合、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/joinpath.c
  • src/backend/optimizer/plan/analyzejoins.c
  • **呼び出し先**:
  • bms_is_subset（ビットマップセットの包含判定関数）

関連データ構造

  • RestrictInfo: 述語のメタデータを保持する構造体。
  • Relids: リレーションIDの集合を表すビットマップセット。

失敗モードと境界条件

  • どちらの条件にも適合しない場合、副作用（outer_is_left の更新）は発生せず、false が返されます。
  • rinfo が NULL の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数はインライン関数であり、頻繁に呼び出されるため、副作用として rinfo->outer_is_left を更新する点に注意してください。
  • 結合順序の最適化において、述語がどのリレーションを跨いでいるかを高速に判定するために使用されます。
