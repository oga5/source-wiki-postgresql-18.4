findeq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_rewrite.c:34

目的

tsquery の木構造（QTNode）において、指定された部分木パターン（ex）と一致するノードを検索し、一致した場合はそれを別のノード（subs）に置換します。

シグネチャまたは形状

QTNode *findeq(QTNode *node, QTNode *ex, QTNode *subs, bool *isfind)

入力と出力

  • **入力**:
  • node: 探索対象の QTNode。
  • ex: 検索対象のパターンを示す QTNode。
  • subs: 一致時に置換する QTNode（NULL の場合、一致ノードは削除される）。
  • isfind: 一致が発生したかどうかを記録するフラグへのポインタ。
  • **出力**:
  • 置換後の QTNode（または NULL）。

前提条件と不変条件

  • node および ex の子ノードは、事前に QTNSort 等で正規化（ソート）されている必要があります。
  • QTN_NOCHANGE フラグが付与されたノードは探索対象外となります。
  • AND / OR 演算子ノードの場合、ex の子ノードが node の子ノードのサブセットである場合に一致とみなされます。

アルゴリズムまたは振る舞い

1. **事前判定**: 署名（sign）およびノード型が一致しない場合、即座に node を返します。 2. **演算子ノードの処理**:

  • 演算子が一致し、子ノード数が同じ場合は QTNEq で完全一致を判定します。
  • AND / OR 演算子の場合、node の子ノードの中に ex の子ノードが含まれるかを確認し、部分一致する場合は該当ノードを削除して subs を挿入します。その後、QTNSort で再ソートします。

    • 3. **オペランドノードの処理**: valcrc および QTNEq を用いて一致判定を行います。 4. **置換**: 一致した場合、元のノードを QTNFree で解放し、subs のコピーを配置して QTN_NOCHANGE フラグを付与します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery_rewrite.c 内の探索処理（dofindsubquery 等から呼び出される）。
  • **呼び出し先**: QTNEq, QTNFree, QTNCopy, QTNSort, QTNodeCompare, palloc0, pfree。

関連データ構造

  • QTNode: tsquery を表現する木構造のノード。

失敗モードと境界条件

  • node が QTN_NOCHANGE フラグを持つ場合、置換は行われません。
  • AND / OR 以外の演算子（NOT, PHRASE）では、子ノード数の不一致は一致とみなされません。
  • subs が NULL の場合、一致したノードは単に削除されます。

保守担当者向けメモ

  • 本関数は再帰的な探索の一部として機能します。置換後のノードに対して QTN_NOCHANGE を付与することで、無限再帰や不要な再走査を防止しています。
  • AND / OR の部分一致処理において、matched 配列を用いた子ノードの走査を行っています。このロジックはソート済みであることを前提としているため、変更時は注意が必要です。
