AtPrepare_PgStat

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_xact.c:190

目的

2相コミット（2PC）の準備フェーズにおいて、トランザクションに関連付けられた統計情報の状態を保存し、準備済みトランザクションとして永続化するための準備を行う。

シグネチャまたは形状

void AtPrepare_PgStat(void)

入力と出力

  • 入力: なし（グローバルな pgStatXactStack を参照）
  • 出力: なし

前提条件と不変条件

  • pgStatXactStack が存在する場合、そのネストレベルが 1（トップレベルトランザクション）であり、かつスタックの先頭であることを前提とする。
  • 実行時、xact_state->nest_level == 1 および xact_state->prev == NULL であることがアサートされる。

アルゴリズムまたは振る舞い

1. 現在のトランザクション統計スタック pgStatXactStack を取得する。 2. スタックが空でない場合、その状態がトップレベルであることを確認する。 3. AtPrepare_PgStat_Relations(xact_state) を呼び出し、リレーションに関連する統計情報の準備処理を実行する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c:2672（トランザクション管理の準備フェーズ）
  • 呼び出し先: AtPrepare_PgStat_Relations

関連データ構造

  • pgStatXactStack: 現在のトランザクションの統計状態を保持するスタック。
  • PgStat_SubXactStatus: トランザクションの統計状態を管理する構造体。

失敗モードと境界条件

  • pgStatXactStack が NULL の場合、本関数は何も行わず正常終了する。
  • ネストレベルが 1 でない場合、アサーションエラーにより異常終了する。

保守担当者向けメモ

  • 本関数は 2PC の PREPARE 時にのみ呼び出される。通常のコミットやアボート時には AtEOXact_PgStat が使用されるため、処理の分離に注意すること。
  • AtPrepare_PgStat_Relations 以外の統計カテゴリ（データベースや関数など）の準備が必要な場合は、本関数内で追加の呼び出しを行う必要がある。
