ss_lru_item_t

定義

  • 種別: struct
  • 定義箇所: src/backend/access/common/syncscan.c:97

目的

同期スキャン（Synchronous Scan）において、各テーブルのスキャン位置情報を LRU（Least Recently Used）リスト形式で管理するためのノード構造体です。

シグネチャまたは形状

typedef struct ss_lru_item_t
{
    struct ss_lru_item_t *prev;
    struct ss_lru_item_t *next;
    ss_scan_location_t location;
} ss_lru_item_t;

入力と出力

本構造体はデータ保持用であり、関数ではないため入力・出力はありません。各フィールドは以下の役割を持ちます。

  • prev, next: 双方向リストを構成するためのポインタ。
  • location: 保持するスキャン位置情報（ss_scan_location_t 型）。

前提条件と不変条件

  • ss_scan_locations_t 構造体内の items 配列の一部としてメモリ上に確保されます。
  • リストの先頭（head）および末尾（tail）ポインタによって、常にアクセス順序が維持されます。

アルゴリズムまたは振る舞い

  • 同期スキャン実行時、特定のテーブルが参照されるたびに、該当する ss_lru_item_t がリストの先頭へ移動されます。
  • リストの容量を超えて新しいエントリが必要な場合、末尾（tail）のアイテムが再利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: ss_search 関数が本構造体のポインタを操作し、リストの順序を更新します。
  • 呼び出し先: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • ss_scan_location_t: location フィールドの型。
  • ss_scan_locations_t: 本構造体を配列として保持し、リストの head と tail を管理する親構造体。

失敗モードと境界条件

  • リストの最大要素数（SYNC_SCAN_NELEM）を超えた場合、最も古いエントリが上書きされます。
  • 共有メモリ上の固定領域に配置されるため、メモリ不足による動的な失敗は発生しません。

保守担当者向けメモ

  • ss_lru_item_t は ss_scan_locations_t 内で FLEXIBLE_ARRAY_MEMBER を用いて管理されています。リスト操作を行う際は、必ず SyncScanLock による排他制御が必要です。
