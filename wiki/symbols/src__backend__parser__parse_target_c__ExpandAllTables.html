ExpandAllTables

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_target.c:1295

目的

SQLクエリ内の SELECT * 構文を解決し、現在スコープ内にあるすべての可視テーブルの全列をターゲットリストとして展開する。

シグネチャまたは形状

static List *ExpandAllTables(ParseState *pstate, int location)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体へのポインタ。
  • location: 構文エラー発生時に報告するためのトークン位置。
  • **出力**:
  • List *: 展開された列（TargetEntry ノードのリスト）。

前提条件と不変条件

  • pstate->p_namespace に有効な名前空間項目が含まれている必要がある。
  • ターゲットリストの解析中であるため、p_lateral_only な項目は存在しないことが前提（Assert で検証）。

アルゴリズムまたは振る舞い

1. pstate->p_namespace を走査し、p_cols_visible が真である名前空間項目（ParseNamespaceItem）を特定する。 2. 各項目に対して expandNSItemAttrs を呼び出し、そのテーブルの全列を取得して target リストに連結する。 3. 走査中に一つも可視テーブルが見つからなかった場合、SELECT * が対象テーブルなしで実行されたと判断し、エラーを報告する。 4. 最終的に構築された列リストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_target.c 内の ExpandColumnRefStar（推測される文脈に基づく）。
  • **呼び出し先**:
  • expandNSItemAttrs: 各名前空間項目の属性を展開するために呼び出される。
  • ereport: エラー発生時に呼び出される。

関連データ構造

  • ParseState: 解析中のクエリ状態を保持。
  • ParseNamespaceItem: 名前空間内のテーブルやリレーション情報。
  • List: 展開された列ノードを保持するリスト。

失敗モードと境界条件

  • **エラー**: found_table が偽のまま走査が終了した場合（テーブルが指定されていない SELECT *）、ERRCODE_SYNTAX_ERROR でエラー終了する。
  • **境界条件**: 0列のテーブルが含まれる場合でも、テーブル自体が存在すればエラーにはならず、空のリストを適切に処理する。

保守担当者向けメモ

  • 本関数は SELECT * の展開ロジックの中核を担う。新しいノード型や名前空間の管理方法が変更された場合、expandNSItemAttrs への引数や p_cols_visible の判定ロジックに影響が出る可能性がある。
