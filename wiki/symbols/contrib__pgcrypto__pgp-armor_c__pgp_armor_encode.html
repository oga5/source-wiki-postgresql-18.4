pgp_armor_encode

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-armor.c:206

目的

バイナリデータを PGP ASCII アーマー形式（Base64 エンコードおよび CRC24 チェックサム付き）のテキストに変換し、指定された StringInfo バッファへ書き出します。

シグネチャまたは形状

void pgp_armor_encode(const uint8 *src, unsigned len, StringInfo dst,
                      int num_headers, char **keys, char **values);

入力と出力

  • src: エンコード対象のバイナリデータ。
  • len: src のバイト長。
  • dst: 出力先となる StringInfo 構造体。ヘッダー、エンコード済みデータ、チェックサム、フッターが順次追記されます。
  • num_headers: keys および values 配列の要素数。
  • keys, values: アーマーヘッダーとして出力するキーと値の文字列配列。
  • 出力: なし（dst を更新）。

前提条件と不変条件

  • dst は初期化済みである必要があります。
  • keys および values は num_headers 分の要素を持つ有効な文字列配列である必要があります。
  • 内部で pg_base64_encode を呼び出すため、dst に十分なメモリ領域が確保されます。

アルゴリズムまたは振る舞い

1. 入力データ src から crc24 を計算します。 2. armor_header を dst に書き込みます。 3. 指定された num_headers 分のキー・値ペアを Key: Value\n 形式で書き込み、空行を挿入します。 4. pg_base64_encode を用いてバイナリを Base64 形式に変換し、dst に追記します。 5. データ末尾に改行がない場合は改行を補完します。 6. = 記号に続けて、計算済みの CRC24 チェックサムを Base64 形式（4文字）で書き込みます。 7. 最後に armor_footer を書き込みます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pgcrypto/pgp-pgsql.c:867
  • 呼び出し先:
  • crc24
  • appendStringInfoString / appendStringInfo / appendStringInfoChar
  • pg_base64_enc_len
  • pg_base64_encode
  • enlargeStringInfo
  • elog (エラー発生時)

関連データ構造

  • StringInfo: PostgreSQL の動的文字列バッファ。

失敗モードと境界条件

  • pg_base64_encode の結果が予測サイズ（b64len）を超えた場合、FATAL エラーでプロセスが終了します。

保守担当者向けメモ

  • 本関数内の Base64 実装は独自のものであり、他のモジュールと重複しています。
  • dst バッファのメモリ管理は StringInfo の仕組みに依存しています。
  • CRC24 の計算およびエンコード形式は RFC 2440 に準拠しています。
