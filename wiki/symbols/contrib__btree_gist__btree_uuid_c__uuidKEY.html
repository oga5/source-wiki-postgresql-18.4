uuidKEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_uuid.c:12

目的

GiST インデックス内において、UUID 型の範囲（下限値および上限値）を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    pg_uuid_t    lower,
                 upper;
} uuidKEY;

入力と出力

  • 本構造体は、GiST インデックスのノードに格納されるキーとして使用されます。
  • lower および upper フィールドは、インデックス内の特定の範囲を定義するために使用されます。

前提条件と不変条件

  • リーフノードにおいては、lower と upper は同一の値（範囲の境界が一致）となります。
  • UUID の大小比較は、バイト列の比較（memcmp）に基づきます。

アルゴリズムまたは振る舞い

  • gbt_uuid_compress 関数により、入力された UUID がこの構造体に変換され、インデックスへ挿入されます。
  • gbt_uuid_consistent 関数において、クエリ条件とこの構造体が保持する範囲を比較し、検索対象となるかどうかが判定されます。
  • gbt_uuid_penalty 関数では、この構造体を使用してインデックスの拡大コストを計算します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/btree_gist/btree_uuid.c 内の各 GiST サポート関数（gbt_uuid_compress, gbt_uuid_consistent, gbt_uuid_penalty, gbt_uuid_union 等）が、この構造体をキャストして操作します。
  • 呼び出し先: 直接的な関数呼び出しはありませんが、DatumGetPointer を介してインデックスエントリから本構造体へのポインタが取得されます。

関連データ構造

  • pg_uuid_t: UUID の実体。
  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗: palloc(sizeof(uuidKEY)) を使用する箇所でメモリ不足が発生する可能性があります。
  • 不正なポインタキャスト: DatumGetPointer からのキャスト時に、データが正しく初期化されていない場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は btree_utils_num.c の汎用関数と組み合わせて使用されることを前提としています。
  • UUID のバイト列比較においてエンディアンが考慮されているため、比較ロジックを変更する際は uuid_2_double 等の変換関数との整合性に注意してください。
