TypeName

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/parsenodes.h:281

目的

SQL文中で指定されたデータ型（型名）を表現するためのノード構造体です。パーサーが構文解析時に型情報を保持し、後の意味解析フェーズで実際の型定義へ解決するために使用されます。

シグネチャまたは形状

typedef struct TypeName
{
	NodeTag		type;
	List	   *names;			/* 修飾名（Stringノードのリスト） */
	Oid			typeOid;		/* OIDで識別される型 */
	bool		setof;			/* SETOF指定の有無 */
	bool		pct_type;		/* %TYPE指定の有無 */
	List	   *typmods;		/* 型修飾子の式リスト */
	int32		typemod;		/* 事前指定された型修飾子 */
	List	   *arrayBounds;	/* 配列の境界 */
	ParseLoc	location;		/* トークン位置、不明な場合は-1 */
} TypeName;

入力と出力

  • **入力**: SQL文の構文解析時に、型名が記述された箇所から生成されます。
  • **出力**: 意味解析器（parse_analyze等）により、typeOid や typemod が解決・設定された状態で後続の処理へ渡されます。

前提条件と不変条件

  • names は String ノードのリストである必要があります。
  • location はSQL文字列内のバイトオフセットを指し、エラー報告に使用されます。

アルゴリズムまたは振る舞い

1. **生成**: パーサー（gram.y）がSQL内の型指定を読み取り、TypeName ノードを構築します。 2. **解決**: 意味解析フェーズにおいて、names に基づいてシステムカタログから対応する型を検索し、typeOid を設定します。 3. **修飾**: typmods リストから実際の typemod 値が計算・設定されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/catalog/aclchk.c
  • src/backend/catalog/objectaddress.c（get_object_address_type 等で頻繁に使用）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • NodeTag: ノードの種類を識別するタグ。
  • List: names, typmods, arrayBounds を保持するために使用される汎用リスト構造。
  • String: names リストの要素として使用されるノード。

失敗モードと境界条件

  • names が空、または解決不能な型名である場合、意味解析フェーズでエラーが発生します。
  • location が -1 の場合、エラーメッセージにおいて正確な位置を表示できません。

保守担当者向けメモ

  • 本構造体は、SQLの型指定に関するあらゆる情報を保持するため、DDLやキャスト処理の変更時に影響を受けやすい箇所です。
  • pct_type や typmods の追加・変更時は、src/backend/parser/analyze.c における型解決ロジックとの整合性に注意してください。
