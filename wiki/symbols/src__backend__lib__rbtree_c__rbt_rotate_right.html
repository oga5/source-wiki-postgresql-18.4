rbt_rotate_right

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:299

目的

赤黒木の平衡を維持するために、指定されたノード x を中心として右回転操作を行う。

シグネチャまたは形状

static void rbt_rotate_right(RBTree *rbt, RBTNode *x)

入力と出力

  • rbt: 操作対象の赤黒木を管理する RBTree 構造体へのポインタ。
  • x: 回転の中心となる RBTNode。
  • 出力: なし（副作用として木構造を更新）。

前提条件と不変条件

  • x の左の子ノード（x->left）が存在すること。
  • RBTNIL が番兵として適切に機能しており、NULLチェックの代わりに node != RBTNIL が利用可能であること。

アルゴリズムまたは振る舞い

1. x の左の子 y を取得する。 2. y の右部分木を x の左側に付け替える。 3. x の親ノードのポインタを y に付け替える。x がルートノードであった場合は、rbt->root を y に更新する。 4. x を y の右の子として配置し、親子関係を再構築する。

呼び出し元と呼び出し先

  • 呼び出し元:

  • src/backend/lib/rbtree.c:563
  • src/backend/lib/rbtree.c:583
  • src/backend/lib/rbtree.c:607
  • 呼び出し先:
  • 直接的な関数呼び出しはなし。ポインタ操作によるノードのリンク更新のみを行う。

関連データ構造

  • RBTree: 木全体の制御構造体。
  • RBTNode: 木の各ノード。
  • RBTNIL: 葉ノードを表す静的な番兵。

失敗モードと境界条件

  • x->left が RBTNIL である場合、回転操作は論理的に破綻する可能性がある（実装上は y = RBTNIL となり、その後のリンク操作で y->right 等へのアクセスが発生する）。
  • x がルートノードの場合、x->parent が NULL であるため、rbt->root の更新処理が実行される。

保守担当者向けメモ

  • 本関数は赤黒木の平衡維持アルゴリズムの一部であり、単独での呼び出しは想定されていない。必ず rbt_insert_fixup や rbt_delete_fixup 等の平衡維持ロジックの文脈で使用すること。
  • RBTNIL を介したポインタ操作が多用されているため、ノードの親子関係の更新順序を変更する際は注意が必要である。
