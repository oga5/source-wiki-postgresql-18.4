IpcSemaphoreCreate

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_sema.c:222

目的

System V セマフォセットを新規作成し、PostgreSQL 用の識別子（PGSemaMagic）で初期化する。既存のセマフォがクラッシュしたプロセスによって残されたものである場合、それを再利用または破棄して安全な状態を確保する。

シグネチャまたは形状

static IpcSemaphoreId IpcSemaphoreCreate(int numSems)

入力と出力

  • **入力**: numSems (作成するセマフォの数)
  • **出力**: 成功した場合は作成されたセマフォセットの IpcSemaphoreId を返す。

前提条件と不変条件

  • nextSemaKey がグローバルなキー生成器として機能していること。
  • 各セマフォセットの末尾（numSems 番目）は、PostgreSQL の識別用セマフォとして予約されていること。
  • 作成されたセマフォセットは、PGSemaMagic 値によって PostgreSQL が所有するものであることが保証される。

アルゴリズムまたは振る舞い

1. **キーの探索**: nextSemaKey をインクリメントしながら、InternalIpcSemaphoreCreate を用いてセマフォセットの作成を試みる。 2. **衝突時のクリーンアップ**: 作成に失敗した場合、既存のセマフォが PostgreSQL のものか確認する。

  • PGSemaMagic 値が一致し、かつ作成元の PID が存在しない（または終了している）場合、IPC_RMID でセマフォセットを削除し、再作成を試みる。

    • 3. **初期化**: セマフォセット作成後、IpcSemaphoreInitialize を呼び出し、識別用セマフォを PGSemaMagic に設定する。 4. **所有権の明示**: PGSemaphoreUnlock を呼び出し、現在のプロセスがセマフォの所有者であることをカーネルに記録させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/sysv_sema.c 内の PGSemaphoreCreate から呼び出される。
  • **呼び出し先**:
  • InternalIpcSemaphoreCreate
  • semget, semctl (System V API)
  • IpcSemaphoreGetValue, IpcSemaphoreGetLastPID, IpcSemaphoreInitialize
  • kill, getpid (システムコール)
  • PGSemaphoreUnlock

関連データ構造

  • IpcSemaphoreId: セマフォセットの識別子。
  • PGSemaphoreData: セマフォセット ID とインデックスを保持する構造体。
  • union semun: semctl システムコールで使用される共用体。

失敗モードと境界条件

  • **キー枯渇**: 1000 回の試行で作成できない場合は失敗する。
  • **カーネル制限**: SEMMSL や SEMMNI などのカーネルパラメータ制限により作成が失敗する場合がある。
  • **競合**: 他のプロセスが同時に同じキーでセマフォを作成した場合、ループして次のキーを試行する。
  • **PID 判定失敗**: IpcSemaphoreGetLastPID が失敗した場合、そのセマフォは無視して次のキーへ進む。

保守担当者向けメモ

  • 本関数は semop を使用してセマフォの所有権を確定させるため、シグナル割り込みによる中断の可能性を考慮した実装となっている。
  • PGSemaMagic は、PostgreSQL 以外のアプリケーションが作成したセマフォを誤って操作しないための重要なガードである。
