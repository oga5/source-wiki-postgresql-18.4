convert_case

定義

  • 種別: function
  • 定義箇所: src/common/unicode_case.c:212

目的

UTF-8 文字列に対して、指定されたケース変換（大文字、小文字、タイトルケース、またはケースフォールディング）を適用するコアロジックを提供します。

シグネチャまたは形状

static size_t convert_case(char *dst, size_t dstsize, const char *src, ssize_t srclen,
                           CaseKind str_casekind, bool full, WordBoundaryNext wbnext,
                           void *wbstate)

入力と出力

  • **入力**:
  • dst: 変換結果を格納するバッファ。
  • dstsize: dst バッファのサイズ。
  • src: 変換対象の UTF-8 文字列。
  • srclen: 入力文字列の長さ（負の場合はヌル終端まで処理）。
  • str_casekind: 適用する変換の種類（CaseKind）。
  • full: 特殊ケースマッピングを適用するかどうかのフラグ。
  • wbnext, wbstate: タイトルケース変換時の単語境界判定用コールバック関数とその状態。
  • **出力**: 変換後の文字列のバイト長を返します。

前提条件と不変条件

  • str_casekind が CaseTitle の場合、wbnext および wbstate が提供されている必要があります。それ以外の場合はこれらは NULL である必要があります。
  • タイトルケース変換の開始位置は常に境界とみなされます。

アルゴリズムまたは振る舞い

1. 入力文字列を UTF-8 として走査し、文字ごとに casemap 関数を呼び出して変換ルールを決定します。 2. タイトルケース変換時は、wbnext を使用して単語境界を判定し、文字ごとのケース（chr_casekind）を動的に切り替えます。 3. casemap の結果に応じて以下の処理を行います：

  • CASEMAP_SELF: 変換なし。元のバイト列をコピー。
  • CASEMAP_SIMPLE: 単一文字への変換。UTF-8 にエンコードして出力。
  • CASEMAP_SPECIAL: 複数文字への変換。special 配列内の各文字を順次 UTF-8 にエンコードして出力。

    • 4. dstsize を超える書き込みは行わず、最終的な長さを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: unicode_strlower 等の公開 API から呼び出されます。また、src/common/unicode/case_test.c でテスト目的の呼び出しが確認されています。
  • **呼び出し先**: utf8_to_unicode, unicode_utf8len, casemap, unicode_to_utf8, wbnext（タイトルケース時）。

関連データ構造

  • CaseKind: 変換の種類を定義する列挙型。
  • CaseMapResult: 変換結果の状態（CASEMAP_SELF, CASEMAP_SIMPLE, CASEMAP_SPECIAL）。
  • WordBoundaryNext: 単語境界判定用の関数ポインタ型。

失敗モードと境界条件

  • dstsize が 0 の場合、変換に必要なバッファサイズを計算するために使用可能です。
  • 入力文字列が不正な UTF-8 シーケンスを含む場合の挙動は、依存する utf8_to_unicode の実装に依存します。

保守担当者向けメモ

  • ギリシャ語の最終シグマのような文脈依存の変換は、casemap 内部で check_special_conditions を通じて処理されます。
  • タイトルケース変換の単語境界ロジックは呼び出し側に依存するため、wbnext の実装が正確であることを確認してください。
