random_relative_prime

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_time/tsm_system_time.c:334

目的

与えられた整数 n と互いに素（最大公約数が 1）であるランダムな整数 r を生成する。これは、テーブルサンプリングにおいて全ブロックを網羅的に走査するためのステップサイズを決定する際に使用される。

シグネチャまたは形状

static uint32 random_relative_prime(uint32 n, pg_prng_state *randstate)

入力と出力

  • **入力**:
  • n: 互いに素な数を選択するための上限値（uint32）。
  • randstate: 乱数生成器の状態（pg_prng_state *）。
  • **出力**:
  • n と互いに素であり、かつ 0 < r < n を満たす uint32 型の整数。

前提条件と不変条件

  • n <= 1 の場合、常に 1 を返す。
  • 戻り値 r は必ず gcd(r, n) == 1 を満たす。

アルゴリズムまたは振る舞い

1. n <= 1 の場合は直ちに 1 を返す。 2. do-while ループ内で乱数 r を生成する。

  • sampler_random_fract(randstate) * n により 0 から n-1 の範囲の値を生成する。
  • ループの各反復で CHECK_FOR_INTERRUPTS() を呼び出し、割り込み要求を監視する。

    • 3. r == 0 または gcd(r, n) > 1 である限り、乱数生成を繰り返す。 4. 条件を満たす r が得られた時点でその値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/tsm_system_time/tsm_system_time.c (行247)
  • contrib/tsm_system_rows/tsm_system_rows.c (行236)
  • **呼び出し先**:
  • gcd (最大公約数計算関数)
  • sampler_random_fract (乱数生成ユーティリティ)
  • CHECK_FOR_INTERRUPTS (マクロ)

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • n <= 1: 常に 1 を返す安全策が講じられている。
  • 無限ループ: 確率的に gcd(r, n) == 1 となる確率は約61%であり、通常は数回の反復で終了する。長時間実行される場合に備えて CHECK_FOR_INTERRUPTS() が含まれている。

保守担当者向けメモ

  • 本関数は tsm_system_time と tsm_system_rows の両方から利用されている。修正を行う際は両方のモジュールへの影響を考慮すること。
  • gcd 関数が別途定義されている前提で動作する。
