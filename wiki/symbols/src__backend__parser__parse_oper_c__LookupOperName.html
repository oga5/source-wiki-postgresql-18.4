LookupOperName

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_oper.c:98

目的

指定された名前と左右の引数型に基づき、システムカタログから演算子のOIDを厳密に検索します。型変換を伴わない完全一致のみを対象とします。

シグネチャまたは形状

Oid LookupOperName(ParseState *pstate, List *opername, Oid oprleft, Oid oprright, bool noError, int location)

入力と出力

  • **入力**:
  • pstate: パーサーの状態（エラー報告に使用）。
  • opername: 演算子名のリスト。
  • oprleft, oprright: 左および右の引数型OID（単項演算子の場合は InvalidOid を指定）。
  • noError: true の場合、検索失敗時にエラーを投げず InvalidOid を返す。
  • location: エラー発生時の構文解析位置。
  • **出力**:
  • 成功時: 該当する演算子の Oid。
  • 失敗時: noError が true なら InvalidOid、false ならエラーをスロー。

前提条件と不変条件

  • 検索は完全一致（Exact Match）を前提としており、暗黙的な型キャストは考慮されません。
  • oprright が InvalidOid の場合、後置演算子（postfix operator）として扱われますが、現在の実装ではサポートされていないためエラーとなります。

アルゴリズムまたは振る舞い

1. OpernameGetOprid を呼び出し、指定された名前と型に一致する演算子を検索します。 2. OIDが有効であれば、その値を即座に返します。 3. 検索に失敗し、かつ noError が false の場合:

  • oprright が無効であれば「後置演算子はサポートされていない」旨のエラーを報告します。
  • それ以外の場合は op_signature_string を使用して演算子のシグネチャを生成し、「演算子が存在しない」旨の ERRCODE_UNDEFINED_FUNCTION エラーを報告します。

    • 4. 検索失敗かつ noError が true の場合、InvalidOid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_aggregate.c
  • src/backend/catalog/pg_operator.c
  • src/backend/commands/opclasscmds.c
  • src/backend/parser/parse_oper.c (内部呼び出し)
  • **呼び出し先**:
  • OpernameGetOprid
  • op_signature_string
  • ereport (エラー時)

関連データ構造

  • ParseState: 構文解析のコンテキストを保持。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • **演算子未定義**: 該当する演算子が見つからない場合、noError が false ならエラーとなります。
  • **後置演算子**: oprright が InvalidOid の場合、構文エラーとして扱われます。
  • **曖昧な検索**: OpernameGetOprid の仕様に依存しますが、本関数は厳密な一致を求めるため、複数の候補が存在する場合の挙動は OpernameGetOprid の実装に準拠します。

保守担当者向けメモ

  • 本関数は「完全一致」専用です。型変換を許容する演算子解決が必要な場合は、oper() や LookupOperWithArgs などの代替関数を検討してください。
