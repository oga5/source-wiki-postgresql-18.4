ECPGget_var

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/misc.c:592

目的

ECPGの変数リスト（ivlist）から、指定された識別番号（number）に対応する変数のポインタを取得します。

シグネチャまたは形状

void *ECPGget_var(int number)

入力と出力

  • **入力**: number (int) - 検索対象の変数識別番号。
  • **出力**: void * - 見つかった場合は対応する変数のポインタ、見つからない場合は NULL。

前提条件と不変条件

  • グローバル変数または静的変数として定義された ivlist（struct var_list の連結リスト）が、検索対象として存在している必要があります。

アルゴリズムまたは振る舞い

1. ivlist を先頭から順に走査します。 2. 各要素の number フィールドが引数の number と一致するまで繰り返します。 3. 一致する要素が見つかった場合、その要素の pointer フィールドを返します。 4. リストの末尾に到達しても一致するものがない場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/test/expected/compat_informix-test_informix.c
  • src/interfaces/ecpg/test/expected/preproc-outofscope.c
  • **呼び出し先**:
  • 本関数はリスト走査のみを行うため、他の関数を呼び出しません。

関連データ構造

  • struct var_list: ivlist を構成する連結リストのノード。number（識別子）と pointer（変数へのポインタ）を保持します。

失敗モードと境界条件

  • ivlist が空の場合、または指定した number がリスト内に存在しない場合は NULL を返します。

保守担当者向けメモ

  • 本関数は ivlist の線形探索を行うため、リストの要素数が増加すると検索コストが増大します。
  • ivlist の構造や更新ロジックは本関数外で管理されているため、リストの整合性は呼び出し側の管理に依存します。
