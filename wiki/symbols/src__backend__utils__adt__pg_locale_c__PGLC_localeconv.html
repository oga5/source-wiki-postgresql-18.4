PGLC_localeconv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:531

目的

OS から取得した数値および通貨のフォーマット情報（lconv 構造体）を、現在のデータベースエンコーディングに変換して提供する。

シグネチャまたは形状

struct lconv *PGLC_localeconv(void)

入力と出力

  • **入力**: なし（内部的に locale_monetary および locale_numeric グローバル変数を使用）。
  • **出力**: struct lconv へのポインタ。

前提条件と不変条件

  • CurrentLocaleConvValid が真の場合、キャッシュされた値を即座に返す。
  • 戻り値の構造体に含まれる文字列フィールドは、データベースエンコーディングに変換済みである。
  • 変換処理中にメモリ不足が発生した場合、ereport(ERROR, ...) により中断される。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: CurrentLocaleConvValid が真であればキャッシュを返す。 2. **OS からの取得**: pg_localeconv_r を呼び出し、現在のロケール設定に基づく lconv データを取得する。 3. **メモリ確保とコピー**: 取得したデータを strdup を用いて作業用構造体にコピーする。 4. **エンコーディング変換**: pg_get_encoding_from_locale でロケールに対応するエンコーディングを特定し、db_encoding_convert を用いて各文字列フィールドをデータベースエンコーディングへ変換する。 5. **エラーハンドリング**: 変換中にエラーが発生した場合は PG_CATCH ブロックでメモリを解放し、例外を再スローする。 6. **キャッシュ更新**: 変換成功後、結果を静的変数 CurrentLocaleConv に保存し、有効フラグを立てて返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/cash.c (通貨型操作)
  • src/backend/utils/adt/formatting.c (to_char 等の書式化)
  • **呼び出し先**:
  • pg_localeconv_r (OS 依存のロケール情報取得)
  • pg_get_encoding_from_locale (ロケール名からのエンコーディング特定)
  • db_encoding_convert (エンコーディング変換)
  • free_struct_lconv / pg_localeconv_free (メモリ解放)

関連データ構造

  • struct lconv: C 標準ライブラリで定義される数値・通貨フォーマット構造体。

失敗モードと境界条件

  • **メモリ不足**: strdup 等のメモリ確保失敗時に ERRCODE_OUT_OF_MEMORY でエラー終了する。
  • **ロケール取得失敗**: pg_localeconv_r が失敗した場合、elog(ERROR) で終了する。
  • **エンコーディング特定不能**: ロケールからエンコーディングを特定できない場合、PG_SQL_ASCII として扱い、バリデーションのみを行う。

保守担当者向けメモ

  • 本関数は静的変数を使用して結果をキャッシュするため、スレッドセーフな環境下での呼び出しには注意が必要である。
  • grouping および mon_grouping フィールドはテキストデータではないため、エンコーディング変換の対象外である。
