blvalidate

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blvalidate.c:29

目的

Bloom インデックスの演算子クラス（opclass）定義が、インデックスの要件を満たしているかを検証する。CREATE OPERATOR CLASS 実行時に呼び出されるバリデータである。

シグネチャまたは形状

bool blvalidate(Oid opclassoid)

入力と出力

  • **入力**: opclassoid (検証対象の演算子クラスの OID)
  • **出力**: bool (検証に合格した場合は true、不備がある場合は false)

前提条件と不変条件

  • 対象の演算子クラスおよび演算子ファミリーがシステムカタログ（pg_opclass, pg_amop, pg_amproc）に存在すること。
  • Bloom インデックスは ORDER BY 演算子をサポートしない。
  • 必須のサポート関数（BLOOM_HASH_PROC）が定義されていること。

アルゴリズムまたは振る舞い

1. **カタログ取得**: pg_opclass から演算子クラス情報を取得し、関連する演算子とサポート関数のリストを抽出する。 2. **サポート関数の検証**:

  • 各関数の左辺型と右辺型が一致しているかを確認する。
  • BLOOM_HASH_PROC および BLOOM_OPTIONS_PROC のシグネチャが正しいかを確認する。

    • 3. **演算子の検証**:

  • 戦略番号が Bloom の許容範囲内（1 ～ BLOOM_NSTRATEGIES）であるかを確認する。
  • ORDER BY 演算子が含まれていないか（amoppurpose が AMOP_SEARCH であるか）を確認する。
  • 演算子の戻り値が boolean 型であることを確認する。

    • 4. **グループ整合性チェック**: 演算子ファミリー内のグループを解析し、必須のサポート関数が欠落していないかを検証する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/bloom/blutils.c 内で amroutine->amvalidate に割り当てられており、PostgreSQL のインデックス作成プロセスから呼び出される。
  • **呼び出し先**: SearchSysCache1, SearchSysCacheList1, check_amproc_signature, check_amop_signature, identify_opfamily_groups 等の PostgreSQL 内部ユーティリティ関数。

関連データ構造

  • Form_pg_opclass, Form_pg_amproc, Form_pg_amop: システムカタログの行構造体。
  • OpFamilyOpFuncGroup: 演算子ファミリー内の演算子と関数のグループ化情報。

失敗モードと境界条件

  • **キャッシュ検索失敗**: 指定された opclassoid が存在しない場合、elog(ERROR) で異常終了する。
  • **検証エラー**: 必須関数の欠落、不正な戦略番号、またはサポート外の ORDER BY 指定がある場合、ereport(INFO, ...) で詳細を報告し、戻り値として false を返す。
  • **オプション関数**: BLOOM_OPTIONS_PROC は省略可能であり、欠落していてもエラーにはならない。

保守担当者向けメモ

  • 本関数は CREATE OPERATOR CLASS 時にのみ動作する。
  • Bloom インデックス固有の制約（ORDER BY 禁止など）を強制する唯一の箇所であるため、仕様変更時は本関数の検証ロジックを更新すること。
