scanPostingTree

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginget.c:68

目的

GIN索引のPosting Tree構造を走査し、リーフページに含まれるすべてのTID（タプル識別子）をスキャンエントリのビットマップへ収集する。

シグネチャまたは形状

static void scanPostingTree(Relation index, GinScanEntry scanEntry, BlockNumber rootPostingTree)

入力と出力

  • **入力**:
  • index: 対象のインデックスリレーション。
  • scanEntry: 走査対象の GinScanEntry 構造体（結果格納用の matchBitmap を保持）。
  • rootPostingTree: 走査を開始するPosting Treeのルートブロック番号。
  • **出力**: なし（副作用として scanEntry->matchBitmap にTIDが追加され、predictNumberResult が更新される）。

前提条件と不変条件

  • rootPostingTree は有効なPosting Treeのルートを指している必要がある。
  • scanEntry->matchBitmap は初期化済みであること。

アルゴリズムまたは振る舞い

1. ginScanBeginPostingTree を呼び出し、Posting Treeの最左リーフページまで降下する。 2. 取得したバッファの参照カウントをインクリメントし、スタックを解放する。 3. リーフページを順次走査するループを実行する:

  • ページが GIN_DELETED フラグを持っていない場合、GinDataLeafPageGetItemsToTbm を呼び出し、ページ内のTIDを scanEntry->matchBitmap に追加する。
  • predictNumberResult に追加したTID数を加算する。
  • ページが右端（GinPageRightMost）であればループを終了する。
  • そうでなければ ginStepRight を使用して右隣のページへ移動する。

    • 4. 最後に UnlockReleaseBuffer を呼び出し、バッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginget.c 内の関数（scanGetItem 等の文脈から呼び出される）。
  • **呼び出し先**:
  • ginScanBeginPostingTree
  • IncrBufferRefCount
  • freeGinBtreeStack
  • BufferGetPage
  • GinPageGetOpaque
  • GinDataLeafPageGetItemsToTbm
  • GinPageRightMost
  • ginStepRight
  • UnlockReleaseBuffer

関連データ構造

  • Relation: インデックスリレーション。
  • GinScanEntry: スキャン状態と結果ビットマップを保持する構造体。
  • GinBtreeData, GinBtreeStack: Posting Tree走査のための内部構造。

失敗モードと境界条件

  • ページが削除済み（GIN_DELETED）の場合、そのページの内容はスキップされる。
  • Posting Treeが単一のリーフページで構成されている場合、ループは1回のみ実行される。

保守担当者向けメモ

  • Posting Treeの走査中にデッドロックを回避するため、ページ移動時にロックの解放と再取得が行われる。
  • IncrBufferRefCount は freeGinBtreeStack によるバッファの自動解放を防ぐために必須である。
