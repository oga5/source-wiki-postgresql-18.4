pg_lsn_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:62

目的

SQLの文字列形式（XX/XXXXXXXX）で表現されたログシーケンス番号（LSN）を、PostgreSQL内部の64ビット整数表現（XLogRecPtr）に変換します。

シグネチャまたは形状

Datum pg_lsn_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して取得される、LSNの文字列表現。
  • **出力**: PG_RETURN_LSN(result) を介して返される XLogRecPtr 型の Datum。

前提条件と不変条件

  • 入力文字列は pg_lsn_in_internal が許容する XX/XXXXXXXX 形式である必要があります。
  • 変換処理中にエラーが発生した場合、have_error フラグが真となり、ERRCODE_INVALID_TEXT_REPRESENTATION エラーがスローされます。

アルゴリズムまたは振る舞い

1. PG_GETARG_CSTRING(0) で入力文字列を取得します。 2. pg_lsn_in_internal を呼び出し、文字列を XLogRecPtr にデコードします。 3. 変換中にエラーが検出された場合、ereturn を使用してエラーメッセージを生成し、処理を中断します。 4. 正常終了時は、PG_RETURN_LSN マクロを使用して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c:348
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:1008
  • **呼び出し先**:
  • pg_lsn_in_internal (コアロジック)

関連データ構造

  • XLogRecPtr: LSNを保持する64ビット整数型。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • 入力文字列の形式が不正な場合、ERRCODE_INVALID_TEXT_REPRESENTATION を伴うエラーが発生します。

保守担当者向けメモ

  • 本関数はSQLの input 関数として機能します。
  • 内部的な解析ロジックは pg_lsn_in_internal に集約されているため、解析ルールの変更が必要な場合はそちらを確認してください。
