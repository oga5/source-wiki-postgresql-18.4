InvalidateOprProofCacheCallBack

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/predtest.c:2345

目的

演算子間の包含関係や矛盾を保持するキャッシュ（OprProofCacheHash）を無効化し、リセットする。

シグネチャまたは形状

static void InvalidateOprProofCacheCallBack(Datum arg, int cacheid, uint32 hashvalue)

入力と出力

  • arg: キャッシュコールバック用の汎用引数。
  • cacheid: 無効化対象のキャッシュID。
  • hashvalue: 無効化対象のハッシュ値（本関数内では未使用）。
  • 戻り値: なし。

前提条件と不変条件

  • OprProofCacheHash が初期化済みであること（Assert により検証）。
  • 本関数は、pg_amop（演算子ファミリーのメンバ）などのシステムカタログが変更された際に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. OprProofCacheHash に対してハッシュシーケンスイテレータを初期化する。 2. キャッシュ内の全エントリを走査する。 3. 各エントリ（OprProofCacheEntry）の have_implic および have_refute フラグを false に設定し、キャッシュされた推論結果を無効化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/util/predtest.c 内でコールバックとして登録されている（CacheRegisterSyscacheCallback 等による登録が示唆される）。
  • 呼び出し先: hash_seq_init および hash_seq_search（ハッシュテーブル走査用ユーティリティ）。

関連データ構造

  • OprProofCacheHash: 演算子推論結果を保持するハッシュテーブル。
  • OprProofCacheEntry: キャッシュ内の各エントリ。have_implic と have_refute を保持する。
  • HASH_SEQ_STATUS: ハッシュテーブル走査用の状態管理構造体。

失敗モードと境界条件

  • OprProofCacheHash が NULL の場合、Assert により異常終了する。

保守担当者向けメモ

  • 現在の実装はキャッシュ内の全エントリをリセットする単純な手法をとっている。特定の hashvalue に基づく選択的な無効化は行われていない。
  • pg_amop の変更時に呼び出されるため、演算子ファミリーの定義変更がプランナの推論結果に即座に反映されることを保証する。
