ScanKey

定義

  • 種別: typedef
  • 定義箇所: src/include/access/amapi.h:189

目的

インデックススキャンにおいて、検索条件や順序付けの基準を保持するためのデータ構造を指す型定義です。

シグネチャまたは形状

typedef として定義されており、インデックスアクセスメソッド（AM）の amrescan 関数ポインタの引数として使用されます。

typedef void (*amrescan_function) (IndexScanDesc scan,
                                   ScanKey keys,
                                   int nkeys,
                                   ScanKey orderbys,
                                   int norderbys);

入力と出力

  • **入力**: amrescan 関数において、検索キーの配列（keys）および順序付けキーの配列（orderbys）として渡されます。
  • **出力**: なし（void）。

前提条件と不変条件

  • amrescan を呼び出す際、nkeys および norderbys はそれぞれ keys および orderbys 配列の要素数と一致している必要があります。
  • インデックス AM の実装は、これらのキーを使用してスキャン状態を再初期化します。

アルゴリズムまたは振る舞い

インデックススキャンの再開（rescan）時に、新しい検索条件や順序付け条件を適用するために使用されます。IndexScanDesc が保持するスキャン状態に対し、指定された ScanKey 配列を適用してスキャンをリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/bloom/blscan.c の blrescan 関数などが、インデックススキャンの再開時にこの型を利用しています。
  • **呼び出し先**: src/backend/access/brin/brin.c 等で、キーのチェックやメモリ配置計算の対象として参照されています。
  • ※上記以外の呼び出し元・呼び出し先の網羅的な情報は提供されていません。

関連データ構造

  • IndexScanDesc: スキャン状態を保持する構造体。
  • amrescan_function: ScanKey を引数に取る関数ポインタ型。

失敗モードと境界条件

  • keys または orderbys が NULL の場合、nkeys または norderbys が 0 であることが期待されます。これに反する場合の動作は各 AM の実装に依存します。

保守担当者向けメモ

  • 本シンボルは src/include/access/amapi.h で定義されていますが、実際のデータ構造の定義（struct ScanKeyData 等）は src/include/access/skey.h に存在します。
  • 新しいインデックス AM を実装する際は、amrescan 内で ScanKey が正しく解釈されるよう、各 AM のスキャンロジックと整合性を取る必要があります。
