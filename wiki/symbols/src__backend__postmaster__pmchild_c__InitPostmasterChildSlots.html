InitPostmasterChildSlots

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pmchild.c:85

目的

Postmaster が管理する各子プロセスタイプに対応するスロットプールを初期化し、プロセス追跡用のメモリ構造を構築する。

シグネチャまたは形状

void InitPostmasterChildSlots(void)

入力と出力

  • 入力: なし（グローバル設定値 MaxConnections, max_wal_senders, autovacuum_worker_slots, max_worker_processes 等を参照）
  • 出力: なし（副作用としてグローバルなスロット管理構造を初期化）

前提条件と不変条件

  • pmchild_pools 配列の各要素に対して、適切な size が設定されていること。
  • 実行後、slotno が num_pmchild_slots と一致することが Assert により保証される。

アルゴリズムまたは振る舞い

1. 各プロセスタイプ（B_BACKEND, B_AUTOVAC_WORKER 等）の size を設定する。 2. 全タイプ合計のスロット数 num_pmchild_slots を算出する。 3. palloc を用いて全スロット分の PMChild 配列を確保する。 4. 各タイプごとに freelist（双方向リスト）を初期化し、確保したスロットを割り当てる。

  • 各スロットの pid を 0、bkend_type を B_INVALID に初期化する。

    • 5. 全体のアクティブプロセスリスト ActiveChildList を初期化する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/bootstrap/bootstrap.c:332
  • src/backend/postmaster/postmaster.c:952
  • src/backend/tcop/postgres.c:4121
  • 呼び出し先:
  • palloc (メモリ確保)
  • dlist_init (リスト初期化)
  • dlist_push_tail (リストへの要素追加)

関連データ構造

  • PMChild: 個々の子プロセスの状態を保持する構造体。
  • pmchild_pools: プロセスタイプごとのプール設定と freelist を保持する配列。
  • ActiveChildList: 稼働中の全子プロセスを管理する双方向リスト。

失敗モードと境界条件

  • メモリ不足時: palloc がエラーを発生させる。
  • 設定値の不整合: num_pmchild_slots が 0 の場合、空のリストが作成される。

保守担当者向けメモ

  • 本関数は Postmaster 起動時に一度だけ呼び出されることを想定している。
  • WAL Sender は B_BACKEND プールを共有する仕様であるため、設定変更時には注意が必要。
  • クラッシュリカバリ時など、特定の再起動シナリオでは呼び出されない場合がある。
