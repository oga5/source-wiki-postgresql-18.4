InstrEndParallelQuery

定義

  • 種別: function
  • 定義箇所: src/backend/executor/instrument.c:207

目的

並列クエリの実行終了時において、プロセス開始時からのバッファ使用量およびWAL生成量の差分を算出し、計測結果として出力する。

シグネチャまたは形状

void InstrEndParallelQuery(BufferUsage *bufusage, WalUsage *walusage)

入力と出力

  • **入力**:
  • bufusage: 算出されたバッファ使用量の差分を格納する BufferUsage 構造体へのポインタ。
  • walusage: 算出されたWAL生成量の差分を格納する WalUsage 構造体へのポインタ。
  • **出力**: なし（引数で渡されたポインタ経由で値を更新）。

前提条件と不変条件

  • 本関数を呼び出す前に、対応する InstrStartParallelQuery が呼び出され、グローバルな統計状態（save_pgBufferUsage, save_pgWalUsage）が保存されている必要がある。
  • pgBufferUsage および pgWalUsage は、プロセス全体で累積されるグローバル変数である。

アルゴリズムまたは振る舞い

1. bufusage をゼロクリアする。 2. BufferUsageAccumDiff を呼び出し、現在のグローバルな pgBufferUsage と保存された save_pgBufferUsage の差分を計算し、bufusage に格納する。 3. walusage をゼロクリアする。 4. WalUsageAccumDiff を呼び出し、現在のグローバルな pgWalUsage と保存された save_pgWalUsage の差分を計算し、walusage に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/executor/execParallel.c
  • **呼び出し先**:
  • BufferUsageAccumDiff
  • WalUsageAccumDiff
  • memset (標準ライブラリ)

関連データ構造

  • BufferUsage: バッファアクセス統計を保持する構造体。
  • WalUsage: WAL生成統計を保持する構造体。

失敗モードと境界条件

  • 引数に NULL が渡された場合の挙動は未定義。
  • InstrStartParallelQuery が呼び出されていない状態で実行された場合、保存された統計値が初期状態（ゼロ）であると仮定して差分が計算される。

保守担当者向けメモ

  • 本関数は並列ワーカーの統計収集において重要な役割を果たす。統計の累積値は AccumDiff を介して計算されるため、グローバル変数の更新タイミングと本関数の呼び出し順序に注意すること。
