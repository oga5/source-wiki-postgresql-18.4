PredClass

定義

  • 種別: enum
  • 定義箇所: src/backend/optimizer/util/predtest.c:50

目的

述語（Predicate）の論理構造を分類し、最適化プロセスにおける論理包含や矛盾の証明を効率化するための抽象化レイヤーを提供します。

シグネチャまたは形状

typedef enum {
    CLASS_ATOM,  /* AND/OR 以外の式 */
    CLASS_AND,   /* AND 意味論を持つ式 */
    CLASS_OR     /* OR 意味論を持つ式 */
} PredClass;

入力と出力

  • 本シンボルは列挙型であり、関数への入力や出力として使用されます。
  • predicate_classify() 関数が Node を入力として受け取り、この PredClass を返します。

前提条件と不変条件

  • predicate_classify() によって、式ツリーが CLASS_ATOM、CLASS_AND、CLASS_OR のいずれかに分類されることを前提とします。
  • 処理対象の式は、clauses.c 等で正規化されていることが期待されます。

アルゴリズムまたは振る舞い

  • predicate_classify() が式ツリーを走査し、その論理構造に基づいて適切な列挙値を割り当てます。
  • 後続の再帰関数（predicate_implied_by_recurse 等）は、この分類結果を参照して、AND/OR の論理規則に従った分解処理を行います。

呼び出し元と呼び出し先

  • 呼び出し元: predicate_classify() (定義: src/backend/optimizer/util/predtest.c:87)
  • 呼び出し元: predicate_implied_by_recurse() (定義: src/backend/optimizer/util/predtest.c:295)
  • 呼び出し元: predicate_refuted_by_recurse() (定義: src/backend/optimizer/util/predtest.c:536)
  • 呼び出し先: 情報なし（列挙型のため、直接的な関数呼び出し先はありません）。

関連データ構造

  • PredIterInfo: PredClass と組み合わせて、AND/OR 構造を抽象化するためのイテレータ情報として使用されます。

失敗モードと境界条件

  • 複雑な式ツリーに対しては再帰的な処理が行われるため、極端に深いツリー構造ではスタックオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本列挙型は predtest.c 内の論理推論ロジックの根幹です。新しい論理演算子や式タイプをサポートする際は、predicate_classify() の分類ロジックを更新する必要があります。
