MBuf

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/mbuf.c:39

目的

pgcrypto モジュールにおけるメモリバッファ管理およびデータストリーム処理のための可変長バッファ構造体です。暗号化・復号処理におけるデータの蓄積と読み取りを抽象化します。

シグネチャまたは形状

struct MBuf
{
    uint8      *data;      /* バッファの開始位置 */
    uint8      *data_end;  /* 有効データの終了位置 */
    uint8      *read_pos;  /* 現在の読み取り位置 */
    uint8      *buf_end;   /* 確保されたメモリ領域の終了位置 */
    bool        no_write;  /* 書き込み禁止フラグ */
    bool        own_data;  /* 内部データ所有フラグ */
};

入力と出力

本構造体は、mbuf_create 等で初期化され、mbuf_append によるデータ追加や mbuf_grab によるデータ取り出しの対象となります。

前提条件と不変条件

  • PostgreSQL のメモリコンテキスト内で動作します。
  • palloc を使用してメモリが確保されるため、mbuf_free による適切な解放が必要です。
  • read_pos は常に data 以上 data_end 以下の範囲にある必要があります。

アルゴリズムまたは振る舞い

  • **動的拡張**: mbuf_append 呼び出し時に必要に応じてバッファが拡張されます。
  • **データ管理**: data から data_end までの領域が有効なデータとして扱われ、read_pos が現在の読み取り進行状況を保持します。
  • **所有権**: own_data フラグにより、バッファが保持するメモリ領域の解放責任を管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: mbuf_avail, mbuf_size, mbuf_free, prepare_room, mbuf_append, mbuf_grab, mbuf_steal_data 等の mbuf.c 内の関数が本構造体を操作します。
  • **呼び出し先**: PostgreSQL のメモリ管理関数（palloc, repalloc, pfree）を内部的に利用します。

関連データ構造

  • PullFilter: 読み込み用フィルタチェーン。
  • PushFilter: 書き込み用フィルタチェーン。

失敗モードと境界条件

  • メモリ不足時には palloc 等がエラーを発生させます。
  • no_write が true の場合、書き込み操作は制限されます。
  • 読み取り位置が data_end を超える操作は、各アクセサ関数によって制御されます。

保守担当者向けメモ

  • MBuf はポリモーフィックな設計ではなく、データ保持に特化した構造体です。
  • 新しいフィルタを実装する際は、Ops 構造体を用いた PushFilter/PullFilter 側の設計に従ってください。
