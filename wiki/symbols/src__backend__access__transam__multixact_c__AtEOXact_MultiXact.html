AtEOXact_MultiXact

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/multixact.c:1957

目的

トランザクション終了時（End of Transaction）に、現在のトランザクションに関連付けられた MultiXact 関連のローカル状態をクリーンアップする。

シグネチャまたは形状

void AtEOXact_MultiXact(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてメモリ上の状態をリセットする）

前提条件と不変条件

  • 本関数はトランザクションの終了処理中に呼び出されることを前提としている。
  • MyOldestMemberMXactIdSlot() および MyOldestVisibleMXactIdSlot() が指す値の更新はアトミックであると仮定されており、ロック（MultiXactGenLock）を必要としない。

アルゴリズムまたは振る舞い

1. MyOldestMemberMXactIdSlot および MyOldestVisibleMXactIdSlot が保持する値を InvalidMultiXactId にリセットする。 2. ローカルの MultiXactId キャッシュを破棄する。

  • MXactContext を NULL に設定する（MXactContext は TopTransactionContext の子として作成されているため、明示的な削除は不要）。
  • MXactCache を dclist_init により初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c: トランザクション終了処理（CommitTransaction / AbortTransaction 等）から呼び出される。
  • src/test/modules/test_slru/test_multixact.c: テストコードから呼び出される。
  • **呼び出し先**:
  • MyOldestMemberMXactIdSlot()
  • MyOldestVisibleMXactIdSlot()
  • dclist_init()

関連データ構造

  • MXactContext: ローカルメモリコンテキスト。
  • MXactCache: MultiXactId キャッシュを管理するリスト。
  • MultiXactId: 管理対象の識別子。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。メモリコンテキストの生存期間に依存する。

保守担当者向けメモ

  • 本関数は AtPrepare_MultiXact と対になる処理や、コミット準備完了後の処理（PostPrepare_MultiXact）と関連がある。
  • MXactContext のメモリ管理は TopTransactionContext に依存しているため、コンテキストの生存期間を変更する場合は注意が必要である。
