sql_delete_callback

定義

  • 種別: function
  • 定義箇所: src/backend/executor/functions.c:1212

目的

SQL関数キャッシュエントリ（SQLFunctionHashEntry）に関連付けられたリソースを解放し、クリーンアップを行うためのコールバック関数です。

シグネチャまたは形状

static void sql_delete_callback(CachedFunction *cfunc)

入力と出力

  • **入力**: cfunc - 解放対象の CachedFunction 構造体へのポインタ（実際には SQLFunctionHashEntry として扱われる）。
  • **出力**: なし（void）。

前提条件と不変条件

  • cfunc は SQLFunctionHashEntry 型にキャスト可能であること。
  • 実行後、func->plansource_list は NIL に設定され、func->hcontext は NULL に設定される。

アルゴリズムまたは振る舞い

1. 引数 cfunc を SQLFunctionHashEntry にキャストします。 2. plansource_list 内の各 CachedPlanSource に対して DropCachedPlan を呼び出し、キャッシュされたプランを解放します。 3. plansource_list を NIL にリセットします。 4. hcontext（メモリコンテキスト）が存在する場合、MemoryContextDelete を呼び出して関連するすべての補助データを解放します。 5. hcontext を NULL に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、キャッシュ管理機構からコールバックとして登録・実行される設計です。
  • **呼び出し先**:
  • DropCachedPlan
  • MemoryContextDelete

関連データ構造

  • SQLFunctionHashEntry: 関数定義に基づくキャッシュエントリ。
  • CachedPlanSource: SQL関数内の各クエリの計画ソース。
  • MemoryContext: 関数実行に関連するメモリ領域。

失敗モードと境界条件

  • hcontext が NULL の場合、メモリ解放処理はスキップされます。
  • plansource_list が空の場合、プラン解放処理は実行されません。

保守担当者向けメモ

  • 本関数は、SQL関数のキャッシュエントリが破棄される際に呼び出されることを想定しています。
  • hcontext を削除することで、そのコンテキスト配下のメモリがすべて解放されるため、個別のデータ構造の解放漏れを防ぐ設計となっています。
