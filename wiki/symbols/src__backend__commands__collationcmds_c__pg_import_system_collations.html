pg_import_system_collations

定義

  • 種別: function
  • 定義箇所: src/backend/commands/collationcmds.c:835

目的

OS環境（libc）およびICUライブラリから利用可能な照合順序（Collation）を自動検出し、指定されたスキーマ内に一括登録する。

シグネチャまたは形状

Datum pg_import_system_collations(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) により、照合順序を登録する対象スキーマの OID を取得する。
  • **出力**: なし（Datum 型として定義されているが、主にカタログ操作の副作用を目的とする）。

前提条件と不変条件

  • 実行ユーザーはスーパーユーザーである必要がある。
  • 指定されたスキーマ OID がシステムカタログ pg_namespace に存在しなければならない。

アルゴリズムまたは振る舞い

1. **権限およびスキーマ検証**: スーパーユーザー権限とスキーマの存在を確認し、失敗時はエラーを投げる。 2. **libc 照合順序のインポート (READ_LOCALE_A_OUTPUT 定義時)**:

  • locale -a コマンドを実行し、出力されたロケール名を順次処理する。
  • create_collation_from_locale を呼び出し、カタログ登録を試みる。
  • 正規化されたエイリアス（例: en_US）を一時保存し、全ロケールの読み込み後に qsort でソートして重複を排除しつつ登録する。

    • 3. **ICU 照合順序のインポート (USE_ICU 定義時)**:

  • uloc_countAvailable および uloc_getAvailable を使用して利用可能なロケールを列挙する。
  • 非ASCII文字を含むロケール名は除外する。
  • CollationCreate を呼び出し、-x-icu サフィックスを付与した名前で登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/initdb.c からの SQL 実行による呼び出しが確認されている。
  • その他、SQL インターフェース経由でユーザーが直接呼び出すことが可能。
  • **呼び出し先**:
  • superuser(), SearchSysCacheExists1, OpenPipeStream, ClosePipeStream, create_collation_from_locale, CollationCreate, CommandCounterIncrement, get_collation_actual_version, uloc_countAvailable, uloc_getAvailable, icu_language_tag 等。

関連データ構造

  • CollAliasData: libc ロケール名とエイリアス、エンコーディング情報を保持する構造体。

失敗モードと境界条件

  • スーパーユーザーでない場合、またはスキーマが存在しない場合はエラーとなる。
  • locale -a コマンドが実行できない場合、エラーとなる。
  • ロケール名が長すぎる場合、デバッグログを出力してスキップする。
  • ICU ロケール名に非 ASCII 文字が含まれる場合、登録対象から除外される。

保守担当者向けメモ

  • プラットフォーム依存のコード（#ifdef）が多いため、OSごとのロケール列挙ロジックの変更には注意が必要。
  • locale -a の出力結果は環境に依存するため、登録される照合順序のセットは実行環境によって異なる。
