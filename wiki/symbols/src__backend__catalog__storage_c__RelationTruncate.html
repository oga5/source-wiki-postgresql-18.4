RelationTruncate

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/storage.c:288

目的

リレーション（テーブルやインデックス）の物理ファイルを指定されたブロック数まで切り詰め、関連するフリースペースマップ（FSM）および可視性マップ（VM）を更新し、整合性を維持するためのWALログを記録する。

シグネチャまたは形状

void RelationTruncate(Relation rel, BlockNumber nblocks)

入力と出力

  • **入力**:
  • rel: 切り詰め対象のリレーション記述子。
  • nblocks: 切り詰め後の目標ブロック数。
  • **出力**: なし（副作用として物理ストレージの変更とWAL記録を行う）。

前提条件と不変条件

  • 呼び出し時に MyProc->delayChkptFlags に DELAY_CHKPT_START および DELAY_CHKPT_COMPLETE が設定されていないこと。
  • 処理中、チェックポイントの進行を一時的にブロックするフラグ（DELAY_CHKPT_START | DELAY_CHKPT_COMPLETE）が MyProc に設定される。
  • 物理的な切り詰めは START_CRIT_SECTION() 内で実行され、中断不可のクリティカルセクションとして扱われる。

アルゴリズムまたは振る舞い

1. **初期化**: リレーションのキャッシュされたブロック数情報をリセットする。 2. **準備**: メインフォーク、FSM、VMの各フォークについて、現在のブロック数と切り詰め後のブロック数を算出する。 3. **前処理**: RelationPreTruncate(rel) を呼び出す。 4. **チェックポイント保護**: delayChkptFlags を設定し、切り詰め中のチェックポイントによる不整合を防ぐ。 5. **WAL記録**: リレーションがWALを必要とする場合、XLogInsert で切り詰め情報を記録し、XLogFlush でディスクへフラッシュする。 6. **物理切り詰め**: smgrtruncate を呼び出し、バッファプールから不要なページを削除し、ディスク上のファイルを物理的に切り詰める。 7. **事後処理**: クリティカルセクションを終了し、チェックポイント保護フラグを解除する。必要に応じて FreeSpaceMapVacuumRange でFSMを更新する。

関連データ構造

  • Relation: リレーションのメタデータ構造。
  • SMgrRelation: ストレージマネージャレベルのリレーション情報。
  • xl_smgr_truncate: WAL記録用の切り詰め情報構造体。

失敗モードと境界条件

  • smgrtruncate 実行中にエラーが発生した場合、クリティカルセクションによりシステムはパニックし、クラッシュリカバリへ移行する。
  • FSMやVMが存在しない場合は、それらの切り詰め処理はスキップされる。
  • WAL記録がフラッシュされる前にシステムがクラッシュした場合、リカバリ時に整合性が再構築される必要がある。

保守担当者向けメモ

  • 本関数はチェックポイントとの競合を避けるため、DELAY_CHKPT フラグを厳密に制御している。この順序を変更するとデータ破損を招く可能性がある。
  • FSMの更新はWALログに含まれないため、クラッシュ後の不整合を許容する設計となっている。
  • RelationPreTruncate の呼び出し位置や、smgrtruncate への引数順序は、ストレージマネージャの期待する仕様と密接に関連している。
