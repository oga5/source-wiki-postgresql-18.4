list_insert_nth_int

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:452

目的

整数型を保持するリストの指定された位置（インデックス）に新しい要素を挿入します。

シグネチャまたは形状

List *list_insert_nth_int(List *list, int pos, int datum)

入力と出力

  • **入力**:
  • list: 操作対象の List ポインタ。
  • pos: 挿入先のインデックス（0始まり）。
  • datum: 挿入する整数値。
  • **出力**:
  • 挿入後のリストを指す List ポインタ。リストが NIL の場合は新しく作成されたリストを返します。

前提条件と不変条件

  • list が NIL でない場合、IsIntegerList(list) が真である必要があります。
  • list が NIL の場合、pos は 0 でなければなりません。
  • 操作の終了時に check_list_invariants(list) が呼び出され、リストの内部構造（型タグや長さ）の整合性が検証されます。

アルゴリズムまたは振る舞い

1. list が NIL の場合、list_make1_int(datum) を呼び出して新しいリストを作成し、それを返します。 2. list が NIL でない場合、insert_new_cell(list, pos) を呼び出して指定位置に新しいセルを確保します。 3. 確保されたセルの値領域（lfirst_int）に datum を書き込みます。 4. check_list_invariants を実行してリストの整合性を確認し、リストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/nodes/pg_list.h で宣言されており、PostgreSQL のノード操作を行う各所から参照される可能性があります。
  • **呼び出し先**:
  • list_make1_int
  • IsIntegerList
  • insert_new_cell
  • lfirst_int
  • check_list_invariants

関連データ構造

  • List: PostgreSQL の汎用リスト構造体。
  • ListCell: リスト内の要素を保持する構造体。

失敗モードと境界条件

  • pos がリストの現在の範囲外である場合、insert_new_cell の内部実装に依存した動作（アサーション失敗やメモリ不正アクセス）が発生する可能性があります。
  • list が整数型以外のリストである場合、IsIntegerList のアサーションにより停止します。

保守担当者向けメモ

  • 本関数は配列ベースのリスト操作を行うため、挿入位置によっては memmove による要素のシフトが発生し、O(N) の計算量となります。
  • リストの拡張に伴いメモリ領域が再配置される可能性があるため、リスト内の要素を指すポインタを保持し続けることは避けてください。
