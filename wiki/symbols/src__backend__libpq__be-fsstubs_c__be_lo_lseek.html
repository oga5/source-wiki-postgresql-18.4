be_lo_lseek

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-fsstubs.c:205

目的

ラージオブジェクト（LO）の現在の読み書き位置を、指定されたオフセットと基準点（whence）に基づいて移動させる。

シグネチャまたは形状

Datum be_lo_lseek(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • fd (int32): ラージオブジェクトディスクリプタ。
  • offset (int32): 移動量。
  • whence (int32): 移動の基準点（SEEK_SET, SEEK_CUR, SEEK_END 等）。
  • **出力**:
  • Datum: 移動後の新しいオフセット位置を int32 型で返す。

前提条件と不変条件

  • fd は cookies 配列の有効な範囲内であり、かつ cookies[fd] が NULL であってはならない。
  • 戻り値は int32 の範囲内に収まる必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32 を用いて引数（fd, offset, whence）を取得する。 2. fd が有効な範囲内か、および cookies[fd] が存在するかを検証し、無効な場合は ERRCODE_UNDEFINED_OBJECT でエラーを報告する。 3. inv_seek を呼び出し、対象のラージオブジェクトのシーク操作を実行する。 4. inv_seek の結果が int32 の範囲を超えていないかを確認し、超えている場合は ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE でエラーを報告する。 5. 成功した場合、新しいオフセットを PG_RETURN_INT32 で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（詳細はソースコード上の明示的な記述なし）。
  • **呼び出し先**: inv_seek（src/backend/storage/large_object/inv_api.c で定義されるラージオブジェクトの基盤API）。

関連データ構造

  • cookies: 現在開いているラージオブジェクトのディスクリプタを保持する動的配列。

失敗モードと境界条件

  • **無効なディスクリプタ**: fd が負の値、cookies_size 以上、または cookies[fd] が NULL の場合にエラー。
  • **範囲外の戻り値**: シーク後の位置が int32 型の表現範囲を超える場合にエラー。

保守担当者向けメモ

  • 本関数は32ビットのオフセットを扱うため、64ビットオフセットが必要な場合は be_lo_lseek64 を参照すること。
  • inv_seek の結果に対するオーバーフローチェックは必須である。
