SPI_connect_ext

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/executor/spi.c:99

目的

SPI（Server Programming Interface）セッションを開始し、ネストされた呼び出しをサポートするための実行コンテキストをスタックに構築する。

シグネチャまたは形状

int SPI_connect_ext(int options);

入力と出力

  • **入力**:
  • options: 接続オプション（SPI_OPT_NONATOMIC ビットにより、非アトミックなコンテキストとして接続するかを指定）。
  • **出力**:
  • int: 成功時に SPI_OK_CONNECT を返す。

前提条件と不変条件

  • _SPI_stack が未初期化の場合は初期化され、必要に応じて動的に拡張される。
  • 接続のネスト深さがスタックの容量を超えないこと。
  • 呼び出しごとに新しいメモリコンテキスト（procCxt, execCxt）が作成され、現在のコンテキストとして設定される。

アルゴリズムまたは振る舞い

1. **スタック管理**: _SPI_stack を確認し、必要であればメモリを確保または拡張する。 2. **スタック更新**: _SPI_connected をインクリメントし、新しいスタックレベルを初期化する。 3. **状態保存**: 呼び出し元の SPI_processed、SPI_tuptable、SPI_result を現在のスタックエントリに保存する。 4. **コンテキスト作成**: options に基づき、TopTransactionContext または PortalContext を親としてメモリコンテキストを作成する。 5. **コンテキスト切り替え**: procCxt に切り替え、API グローバル変数をリセットして、呼び出し元からの状態汚染を防ぐ。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/functioncmds.c
  • src/backend/executor/spi.c
  • src/pl/plperl/plperl.c
  • src/pl/plpgsql/src/pl_handler.c
  • src/pl/plpython/plpy_main.c
  • src/pl/tcl/pltcl.c
  • **呼び出し先**:
  • MemoryContextAlloc, repalloc, AllocSetContextCreate, MemoryContextSwitchTo（メモリ管理）
  • GetCurrentSubTransactionId（トランザクション管理）
  • slist_init（リスト操作）
  • elog（エラー報告）

関連データ構造

  • _SPI_connection: SPI 接続状態を保持する内部構造体。
  • _SPI_stack: 接続状態を管理するグローバルスタック。

失敗モードと境界条件

  • **スタック破損**: _SPI_stack の整合性が取れない場合（_SPI_connected と深さの不一致など）、elog(ERROR) を発生させる。
  • **メモリ不足**: MemoryContextAlloc 等のメモリ確保失敗時には、PostgreSQL の標準的なエラー処理によりアボートされる。

保守担当者向けメモ

  • atomic コンテキストの判断基準は options に依存する。
  • procCxt と execCxt の親コンテキスト選択には、トランザクション境界を跨ぐ際の生存期間を考慮した設計上の注意が必要である。
  • SPI_processed 等のグローバル変数はスタックに退避されるため、再帰的な SPI 呼び出しは安全である。
