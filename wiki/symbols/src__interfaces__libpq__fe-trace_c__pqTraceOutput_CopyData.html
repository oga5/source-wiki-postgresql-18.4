pqTraceOutput_CopyData

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:284

目的

PostgreSQL プロトコルにおける CopyData メッセージの内容をデバッグ用トレース出力へ書き出す。

シグネチャまたは形状

static void pqTraceOutput_CopyData(FILE *f, const char *message, int *cursor, int length, bool suppress);

入力と出力

  • f: 出力先のファイルポインタ。
  • message: 解析対象のメッセージデータを含むバッファ。
  • cursor: 現在の読み取り位置を示すポインタ。処理後に更新される。
  • length: メッセージの全長。
  • suppress: 回帰テスト用モードの有効/無効フラグ。
  • 出力: f に対して CopyData というラベルと、ペイロード内容を書き出す。

前提条件と不変条件

  • cursor は有効なメモリ領域を指している必要がある。
  • length - *cursor + 1 が pqTraceOutputNchar に渡されるため、計算結果が負にならないことが期待される。

アルゴリズムまたは振る舞い

1. fprintf を使用して、出力ストリーム f に CopyData という文字列とタブ文字を出力する。 2. pqTraceOutputNchar を呼び出し、メッセージの残りの部分（length - *cursor + 1 バイト分）をトレース出力へ書き出す。 3. 処理に伴い、cursor の値が更新される。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-trace.c 内のメッセージ解析ルーチン（pqTraceOutputMessage 等のディスパッチャから呼び出される）。
  • 呼び出し先: pqTraceOutputNchar（文字列出力用ヘルパー）。

関連データ構造

  • FILE *: 標準ライブラリのファイルストリーム。
  • PGconn.Pfdebug: 本関数が書き出し先として利用するファイルポインタの格納元。

失敗モードと境界条件

  • f が NULL の場合、fprintf によりセグメンテーション違反が発生する可能性がある。
  • cursor が指す値が length を超えている場合、pqTraceOutputNchar への引数が不正になる可能性がある。

保守担当者向けメモ

  • 本関数は CopyData メッセージ専用のフォーマッタである。プロトコル仕様に変更があった場合は、出力フォーマットを適宜調整すること。
  • 回帰テストモード（suppress）の挙動については、pqTraceOutputNchar の実装に依存しているため、そちらも併せて確認すること。
