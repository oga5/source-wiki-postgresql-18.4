InitProcessLocalLatch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/miscinit.c:234

目的

現在のプロセスに対して、ローカルなラッチ（LocalLatchData）を初期化し、それを MyLatch ポインタに割り当てることで、プロセス内でのイベント待機メカニズムを有効化します。

シグネチャまたは形状

void InitProcessLocalLatch(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 MyLatch が更新され、LocalLatchData が初期化されます）

前提条件と不変条件

  • LocalLatchData が静的またはグローバルなスコープで利用可能であること。
  • InitLatch が MyLatch に対して正常に初期化処理を完了できること。

アルゴリズムまたは振る舞い

1. グローバル変数 MyLatch に、プロセスローカルなラッチ構造体 LocalLatchData のアドレスを代入します。 2. InitLatch(MyLatch) を呼び出し、当該ラッチの内部状態を初期化します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/postmaster.c:563
  • src/backend/utils/init/miscinit.c:131
  • src/backend/utils/init/miscinit.c:192
  • 呼び出し先:
  • InitLatch（storage/latch.h で定義されるラッチ初期化関数）

関連データ構造

  • MyLatch: 現在のプロセスが使用するラッチを指すグローバルポインタ。
  • LocalLatchData: プロセスローカルなラッチの実体。

失敗モードと境界条件

  • InitLatch の内部実装に依存します。本関数自体は戻り値を持たず、エラーハンドリングは呼び出し元または InitLatch 内部の挙動に委ねられます。

保守担当者向けメモ

  • 本関数は、プロセスが共有メモリ上のラッチを使用する前の初期段階、またはローカルな待機が必要な場面で使用されます。
  • SwitchToSharedLatch や SwitchBackToLocalLatch と組み合わせて使用される場合があるため、ラッチの切り替えロジックと併せて確認してください。
