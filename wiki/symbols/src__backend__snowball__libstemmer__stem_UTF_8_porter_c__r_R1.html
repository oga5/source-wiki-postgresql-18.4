r_R1

定義

  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_porter.c:232

目的

Porter Stemming アルゴリズムにおいて、語幹判定の基準となる領域「R1」の範囲内に現在のカーソル位置があるかを判定する。

シグネチャまたは形状

static int r_R1(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z（ステミング環境を保持する構造体）
  • **出力**: int 型。条件を満たす場合は 1（真）、満たさない場合は 0（偽）を返す。

前提条件と不変条件

  • z->I[1] には、事前に計算された R1 領域の開始位置が格納されている必要がある。
  • z->c は現在のカーソル位置を示す。

アルゴリズムまたは振る舞い

1. z->I[1]（R1 領域の開始インデックス）と z->c（現在のカーソル位置）を比較する。 2. z->I[1] <= z->c が成立する場合、現在のカーソル位置が R1 領域内またはその境界にあると判断し、1 を返す。 3. それ以外の場合は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 複数の言語用ステマー（stem_ISO_8859_1_basque.c, stem_ISO_8859_1_catalan.c, stem_ISO_8859_1_dutch.c 等）から参照されている。
  • **呼び出し先**: 本関数は計算のみを行い、他の関数を呼び出さない。

関連データ構造

  • struct SN_env: ステミング処理中の状態（カーソル位置、領域境界、文字列バッファ）を保持する構造体。

失敗モードと境界条件

  • z が NULL の場合、ポインタ参照によりクラッシュする可能性がある（呼び出し元でのガードが必要）。
  • z->I[1] が未初期化の場合、意図しない判定結果となる。
