close_none

定義

  • **種別**: 関数
  • **定義箇所**: src/bin/pg_dump/compress_none.c:143

目的

非圧縮モードにおけるアーカイブファイルストリームをクローズし、関連するリソースを解放します。

シグネチャまたは形状

static bool close_none(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH （非圧縮処理用のファイルハンドル構造体）
  • **出力**: bool （クローズ処理が成功した場合は true、失敗した場合は false）

前提条件と不変条件

  • CFH->private_data には、クローズ対象の FILE* が格納されている必要があります。
  • 関数実行後、CFH->private_data は NULL にリセットされます。

アルゴリズムまたは振る舞い

1. CFH->private_data から FILE* ポインタを取得します。 2. CFH->private_data を NULL に設定します。 3. FILE* が有効な場合、fclose() を呼び出してストリームを閉じます。 4. fclose() が失敗した場合、pg_log_error を使用してエラーメッセージを出力します。 5. fclose() の戻り値が 0（成功）であれば true を返し、それ以外は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_none.c 内で、CompressFileHandle 構造体の close_func メンバにこの関数ポインタが代入されています。
  • **呼び出し先**:
  • fclose (標準ライブラリ)
  • pg_log_error (エラー報告ユーティリティ)

関連データ構造

  • CompressFileHandle: 非圧縮 I/O の状態を保持する構造体。private_data メンバを介して FILE* を管理します。

失敗モードと境界条件

  • CFH->private_data が NULL の場合、クローズ処理はスキップされ、成功として true が返されます。
  • fclose() がエラーを返した場合、pg_log_error でエラーが記録され、関数は false を返します。

保守担当者向けメモ

  • 本関数は compress_io モジュールが期待するインターフェースの一部です。
  • private_data が FILE* であるという前提に依存しているため、構造体の定義変更時には注意が必要です。
