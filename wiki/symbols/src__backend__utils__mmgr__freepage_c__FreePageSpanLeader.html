FreePageSpanLeader

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/mmgr/freepage.c:68

目的

メモリ管理サブシステムにおいて、空き領域（スパン）の先頭ページに配置されるヘッダ構造体です。フリーリストのノードとして機能し、連続するページ範囲を追跡・管理するために使用されます。

シグネチャまたは形状

struct FreePageSpanLeader
{
    int            magic;    /* always FREE_PAGE_SPAN_LEADER_MAGIC */
    Size           npages;   /* number of pages in span */
    RelptrFreePageSpanLeader prev;
    RelptrFreePageSpanLeader next;
};

入力と出力

本構造体はデータ構造であり、関数ではありません。入力として Relptr（相対ポインタ）を介したメモリ領域の参照を受け、出力としてスパンのサイズや隣接するスパンへのリンク情報を提供します。

前提条件と不変条件

  • magic フィールドには常に FREE_PAGE_SPAN_LEADER_MAGIC が格納されている必要があります。
  • 共有メモリ環境で動作するため、ポインタはすべて Relptr 型で保持され、絶対アドレスへの依存を排除しています。

アルゴリズムまたは振る舞い

  • **フリーリスト管理**: prev および next フィールドを使用して、空き領域を双方向リストとして連結します。
  • **メタデータ保持**: 管理対象のメモリ領域自体をメタデータとして利用するため、スパンの先頭ページにこのヘッダが埋め込まれます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/freepage.c 内の各関数（FreePageManagerGet, FreePageManagerPut 等）が、空き領域の検索、分割、マージ、および再利用リストへの追加の際に本構造体を操作します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、特定の関数を呼び出すことはありません。

関連データ構造

  • FreePageManager: 本構造体を管理するメインの管理構造体。
  • RelptrFreePageSpanLeader: 本構造体への相対ポインタ型。
  • FreePageBtree: 空き領域の断片化を防ぐために併用される B-tree 構造。

失敗モードと境界条件

  • **メモリ枯渇**: 管理対象領域が枯渇すると、新しい FreePageSpanLeader を配置するためのページ自体が確保できなくなる再帰的な制約があります。
  • **整合性**: magic 値の不一致は、メモリ破壊や不正なポインタ操作を示唆します。

保守担当者向けメモ

  • 本構造体は src/include/utils/freepage.h にも定義が存在します。変更時は両方の定義の整合性に注意してください。
  • FPM_EXTRA_ASSERTS を有効にすると、デバッグ時に構造体の整合性が厳密にチェックされます。
