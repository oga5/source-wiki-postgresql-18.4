pg_get_wal_resource_managers

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/rmgr.c:149

目的

現在システムにロードされているすべてのWALリソースマネージャ（Resource Manager）の一覧を、SQLから参照可能な形式（テーブル）として提供します。

シグネチャまたは形状

Datum pg_get_wal_resource_managers(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（標準的なPostgreSQL関数呼び出し引数）。
  • **出力**: Datum 型。内部的には tuplestore を使用して、以下の3列からなるセットを返します。

    • 1. rmid (int4): リソースマネージャのID。 2. rm_name (text): リソースマネージャの名称。 3. is_builtin (bool): ビルトインのリソースマネージャであるかを示すフラグ。

前提条件と不変条件

  • RmgrId は uint8 型であり、最大256個まで定義可能です。
  • 本関数は InitMaterializedSRF を呼び出し、結果をマテリアライズド・セットとして返します。

アルゴリズムまたは振る舞い

1. InitMaterializedSRF を呼び出し、結果セットの初期化を行います。 2. 0 から RM_MAX_ID までの範囲でループを実行します。 3. RmgrIdExists(rmid) を用いて、該当するIDのリソースマネージャが存在するか確認します。存在しない場合はスキップします。 4. 存在する各リソースマネージャについて、以下の情報を取得します。

  • ID: rmid
  • 名称: GetRmgr(rmid).rm_name
  • ビルトイン判定: RmgrIdIsBuiltin(rmid)

    • 5. tuplestore_putvalues を使用して、取得した情報を結果セットに追加します。 6. 最後に (Datum) 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は確認されていません（SQLインターフェース経由での呼び出しを想定）。
  • **呼び出し先**:
  • InitMaterializedSRF
  • RmgrIdExists
  • GetRmgr
  • RmgrIdIsBuiltin
  • tuplestore_putvalues

関連データ構造

  • RmgrTable: 全リソースマネージャの定義を保持するグローバル配列。
  • ReturnSetInfo: 関数がセットを返すための情報構造体。

失敗モードと境界条件

  • RmgrIdExists が偽を返すIDについては、処理をスキップするため出力に含まれません。
  • RM_MAX_ID を超えるIDはループ対象外となります。

保守担当者向けメモ

  • 本関数は RmgrTable の内容を反映します。新しいリソースマネージャを追加した際、rmgrlist.h の更新が適切に行われていれば、本関数を通じて自動的に一覧に反映されます。
