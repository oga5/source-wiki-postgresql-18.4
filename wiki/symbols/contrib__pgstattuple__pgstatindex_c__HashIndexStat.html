HashIndexStat

定義

  • 種別: struct
  • 定義箇所: contrib/pgstattuple/pgstatindex.c:112

目的

PostgreSQLのHashインデックスの物理的な状態（断片化や空き容量など）を統計情報として保持するためのデータ構造です。

シグネチャまたは形状

typedef struct HashIndexStat
{
	int32		version;
	int32		space_per_page;
	BlockNumber bucket_pages;
	BlockNumber overflow_pages;
	BlockNumber bitmap_pages;
	BlockNumber unused_pages;
	int64		live_items;
	int64		dead_items;
	uint64		free_space;
} HashIndexStat;

入力と出力

本構造体は関数への出力引数として使用されます。GetHashPageStats 関数に対し、解析対象のページ情報と共にポインタとして渡され、統計値が書き込まれます。

前提条件と不変条件

  • GetHashPageStats を通じて、Hashインデックスの各ページから収集された値が格納されます。
  • BlockNumber 型のフィールドは、インデックス内の各ページタイプ（バケット、オーバーフロー、ビットマップ、未使用）の総数を保持します。

アルゴリズムまたは振る舞い

インデックス走査時に、各ページを読み込み、そのページタイプや内部のアイテム数、空き容量を本構造体の各フィールドに加算・更新することで統計情報を集計します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgstattuple/pgstatindex.c 内のインデックス走査処理（pgstatindex_impl 等の文脈）から、GetHashPageStats を呼び出す際にインスタンスが生成・利用されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • BTIndexStat, GinIndexStat: 他のインデックス形式における同様の統計保持用構造体。

失敗モードと境界条件

  • 構造体内の各フィールドは、インデックスの物理的な状態に依存します。インデックスが破損している場合、不整合な統計値が格納される可能性があります。

保守担当者向けメモ

  • 本構造体のフィールド構成を変更する場合、GetHashPageStats 内の集計ロジックおよび、統計情報を最終的に出力するSQL関数の結果セット定義（get_call_result_type 等）との整合性を確認してください。
