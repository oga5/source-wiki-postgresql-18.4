IsManifestEnabled

定義

  • 種別: function
  • 定義箇所: src/backend/backup/backup_manifest.c:33

目的

指定されたバックアップマニフェストの状態に基づき、マニフェスト生成機能が有効かどうかを判定します。

シグネチャまたは形状

static inline bool IsManifestEnabled(backup_manifest_info *manifest)

入力と出力

  • **入力**: backup_manifest_info *manifest（マニフェスト生成の状態を保持する構造体へのポインタ）
  • **出力**: bool（有効な場合は true、無効な場合は false）

前提条件と不変条件

  • 引数 manifest は NULL であってはなりません。
  • manifest->buffile が NULL でない場合、マニフェスト生成が初期化済みであるとみなされます。

アルゴリズムまたは振る舞い

manifest->buffile が NULL かどうかを確認し、その結果を返します。buffile が存在すればマニフェスト生成が有効であると判断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/backup_manifest.c:110
  • src/backend/backup/backup_manifest.c:222
  • src/backend/backup/backup_manifest.c:323
  • **呼び出し先**: 情報なし。

関連データ構造

  • backup_manifest_info: マニフェスト生成の状態を保持する構造体。

失敗モードと境界条件

  • manifest ポインタ自体が NULL の場合、デリファレンスによりクラッシュする可能性があります。

保守担当者向けメモ

  • static inline 関数として定義されており、マニフェスト生成処理の各ステップ（ファイル追加やWAL情報追加など）で、生成が有効かどうかのガード条件として頻繁に使用されます。
