TS_execute_ternary

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:1870

目的

全文検索クエリ（tsquery）を tsvector に対して評価し、三値論理（真、偽、不明）の結果を返すためのエントリポイントを提供します。

シグネチャまたは形状

TSTernaryValue TS_execute_ternary(QueryItem *curitem, void *arg, uint32 flags, TSExecuteCallback chkcond);

入力と出力

  • **入力**:
  • curitem: 評価対象の tsquery ツリーのノード。
  • arg: 評価時に使用されるコンテキスト情報（tsvector 等）。
  • flags: 実行時の動作を制御するフラグ。
  • chkcond: 語彙の照合を行うためのコールバック関数。
  • **出力**:
  • TSTernaryValue: 評価結果（真、偽、または不明）。

前提条件と不変条件

  • curitem は有効な tsquery ツリーのノードを指している必要があります。
  • 内部で再帰的な処理が行われるため、スタックオーバーフローを避けるための深さ制限が適用されます。

アルゴリズムまたは振る舞い

指定された tsquery ノードに対して TS_execute_recurse を呼び出し、その結果をそのまま返します。実質的に TS_execute_recurse へのラッパーとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsginidx.c: GIN インデックスの検索処理において、クエリの評価に使用されています。
  • **呼び出し先**:
  • TS_execute_recurse: 本関数が内部で呼び出す唯一の関数です。

関連データ構造

  • TSTernaryValue: 三値論理を表現する列挙型。
  • QueryItem: tsquery の構成要素。
  • TSExecuteCallback: 語彙照合用の関数ポインタ型。

失敗モードと境界条件

  • 評価対象のクエリツリーが極端に深い場合、スタックオーバーフローが発生する可能性があります。
  • chkcond コールバックが予期しない値を返した場合の挙動は、コールバックの実装に依存します。

保守担当者向けメモ

  • 本関数は TS_execute_recurse の単なる転送関数です。ロジックの変更が必要な場合は、TS_execute_recurse を確認してください。
  • tsginidx.c からの呼び出しが確認されており、インデックススキャン時の重要な評価パスとなっています。
