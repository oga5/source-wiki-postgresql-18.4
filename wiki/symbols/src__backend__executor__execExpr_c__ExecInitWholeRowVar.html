ExecInitWholeRowVar

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExpr.c:3165

目的

式評価において、行全体（Whole-row）を参照する Var ノードを処理するための ExprEvalStep を初期化します。

シグネチャまたは形状

static void ExecInitWholeRowVar(ExprEvalStep *scratch, Var *variable, ExprState *state)

入力と出力

  • **入力**:
  • scratch: 初期化対象の ExprEvalStep 構造体へのポインタ。
  • variable: 処理対象の Var ノード。
  • state: 現在の式評価状態を保持する ExprState。
  • **出力**: なし（scratch を介して副作用として命令ステップを構築）。

前提条件と不変条件

  • scratch->opcode は EEOP_WHOLEROW に設定されます。
  • variable->varreturningtype に応じて、state->flags に EEO_FLAG_HAS_OLD または EEO_FLAG_HAS_NEW が適切に設定されます。
  • scratch->d.wholerow.tupdesc は実行時に設定されるため、初期化時点では NULL です。

アルゴリズムまたは振る舞い

1. scratch に EEOP_WHOLEROW 命令を設定し、変数を格納します。 2. variable の型（OLD/NEW）を判定し、state->flags を更新します。 3. 親ノード（parent）が存在する場合、T_SubqueryScanState または T_CteScanState を走査します。 4. サブプランのターゲットリストに resjunk カラムが含まれている場合、ExecInitJunkFilter を用いて junkFilter を生成し、scratch に格納します。これにより、行全体の結果から不要なジャンク列を除去可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExpr.c 内の式コンパイル処理（ExecInitExprRec 等から参照）。
  • **呼び出し先**: ExecInitJunkFilter（ジャンクフィルタが必要な場合）、ExecInitExtraTupleSlot。

関連データ構造

  • ExprEvalStep: 式評価の命令単位。
  • Var: 評価対象の変数ノード。
  • ExprState: 式評価の実行状態。
  • JunkFilter: 不要な列を除去するための構造体。

失敗モードと境界条件

  • 親プランが存在しないスタンドアロン式の場合、junkFilter は NULL のままとなります（テーブル行を参照する前提のため問題なし）。
  • サブプランが存在しない場合、ジャンクフィルタの生成はスキップされます。

保守担当者向けメモ

  • 本関数は EEOP_WHOLEROW 命令の構築に特化しています。execExprInterp.c 側のインタプリタ実装と整合性を保つ必要があります。
  • junkFilter の構築は、サブクエリ等の特殊なケースでのみ発生するため、デバッグ時は親ノードの型を確認してください。
