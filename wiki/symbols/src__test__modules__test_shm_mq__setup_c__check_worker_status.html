check_worker_status

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/setup.c:304

目的

テスト実行中にバックグラウンドワーカーの生存状態を監視し、ワーカーまたはポストマスターが異常終了していないかを確認する。

シグネチャまたは形状

static bool check_worker_status(worker_state *wstate)

入力と出力

  • **入力**: wstate (起動済みワーカーのハンドルを保持する worker_state 構造体へのポインタ)
  • **出力**: bool 型。すべてのワーカーが正常に動作している場合は true、いずれかのワーカーが停止しているかポストマスターが終了している場合は false を返す。

前提条件と不変条件

  • wstate は有効なメモリ領域を指している必要がある。
  • wstate->handle には、GetBackgroundWorkerPid で参照可能な有効なワーカーハンドルが格納されている必要がある。

アルゴリズムまたは振る舞い

1. wstate->nworkers で指定された回数分、ループを実行する。 2. 各ワーカーに対して GetBackgroundWorkerPid を呼び出し、現在のステータスを取得する。 3. ステータスが BGWH_STOPPED（ワーカー停止）または BGWH_POSTMASTER_DIED（ポストマスター終了）である場合、直ちに false を返す。 4. すべてのワーカーのチェックが完了し、異常が検出されなかった場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_shm_mq/setup.c 内の関数（wait_for_workers_to_become_ready 等から参照される）。
  • **呼び出し先**: GetBackgroundWorkerPid（PostgreSQLのバックグラウンドワーカー管理API）。

関連データ構造

  • worker_state: ワーカーのハンドル配列とワーカー数を保持する構造体。
  • BgwHandleStatus: ワーカーの状態を示す列挙型。

失敗モードと境界条件

  • ワーカーが予期せず終了した場合、この関数は false を返し、呼び出し側でテストをエラー終了させるトリガーとなる。
  • wstate が NULL の場合や、内部のハンドル配列が不正な場合は未定義の動作となる。

保守担当者向けメモ

  • 本関数は、ワーカーの生存確認のみを行う。ワーカーが正常にタスクを完了したかどうか（終了コード等）の判定は行わない。
  • 呼び出し元でこの関数が false を受け取った場合、テスト環境のクリーンアップが必要となる。
