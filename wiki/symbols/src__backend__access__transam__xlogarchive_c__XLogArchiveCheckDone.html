XLogArchiveCheckDone

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:564

目的

指定されたWALファイルがアーカイブ済みであるかを確認し、削除可能かどうかを判定します。未処理の場合はアーカイブを促す通知を行います。

シグネチャまたは形状

bool XLogArchiveCheckDone(const char *xlog)

入力と出力

  • **入力**: xlog (対象となるWALファイル名)
  • **出力**: true（削除可能）、false（削除不可、またはアーカイブ処理中）

前提条件と不変条件

  • archive_mode が off の場合、常に true を返します。
  • アーカイブリカバリ中かつ archive_mode が always 以外の場合、常に true を返します。

アルゴリズムまたは振る舞い

1. アーカイブが不要な状態（archive_mode が off またはリカバリ中の非 always モード）であれば直ちに true を返します。 2. pg_wal/archive_status/ 配下の .done ファイルの存在を確認し、あれば true を返します。 3. .ready ファイルの存在を確認し、あればアーカイブ処理中とみなし false を返します。 4. レースコンディションを考慮し、再度 .done ファイルの存在を確認します。 5. 上記のいずれにも該当しない場合、XLogArchiveNotify を呼び出して .ready ファイルを再作成し、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c 内のWALファイル削除処理（xlde->d_name を引数として使用）。
  • **呼び出し先**: XLogArchivingActive, XLogArchivingAlways, GetRecoveryState, StatusFilePath, stat, XLogArchiveNotify。

関連データ構造

  • MAXPGPATH: ファイルパス格納用バッファサイズ。
  • struct stat: ファイル状態確認用構造体。

失敗モードと境界条件

  • stat システムコールが失敗した場合、ファイルが存在しないものとして処理が進行します。
  • アーカイブプロセスとの競合が発生した場合、再確認および .ready の再作成により整合性を維持します。

保守担当者向けメモ

  • 本関数は XLogArchiveIsReadyOrDone とは異なり、削除の可否を判定するロジックを含みます。
  • .ready ファイルの再作成は、アーカイブプロセスがクラッシュ等で停止した後の復旧を意図しています。
