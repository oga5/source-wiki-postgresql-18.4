fixed_paramref_hook

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_param.c:99

目的

クエリ解析中に遭遇したパラメータ参照（ParamRef）を、事前に定義された型情報を持つ Param ノードに変換する。

シグネチャまたは形状

static Node *fixed_paramref_hook(ParseState *pstate, ParamRef *pref)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体。p_ref_hook_state に FixedParamState が格納されていることを前提とする。
  • pref: 解析対象のパラメータ参照ノード。
  • **出力**:
  • 成功時: 変換された Param ノード（Node * 型）。
  • 失敗時: エラーをスローし、処理を中断する。

前提条件と不変条件

  • pstate->p_ref_hook_state は FixedParamState 型にキャスト可能であること。
  • FixedParamState 内の paramTypes 配列は、参照されるパラメータ番号に対して有効な OID を保持していること。

アルゴリズムまたは振る舞い

1. pstate->p_ref_hook_state から FixedParamState を取得する。 2. pref->number で指定されたパラメータ番号が、FixedParamState の範囲内かつ有効な型（OidIsValid）であるか検証する。無効な場合は ERRCODE_UNDEFINED_PARAMETER エラーを発生させる。 3. Param ノードを新規作成し、以下の値を設定する:

  • paramkind: PARAM_EXTERN
  • paramid: pref->number
  • paramtype: FixedParamState から取得した対応する OID
  • paramtypmod: -1
  • paramcollid: get_typcollation により取得した型照合順序
  • location: pref->location

    • 4. 作成した Param ノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_param.c 内の setup_parse_fixed_parameters 関数において、pstate->p_paramref_hook に関数ポインタとして登録される。
  • **呼び出し先**: makeNode（ノード生成）、ereport（エラー報告）、get_typcollation（照合順序取得）、parser_errposition（エラー位置特定）。

関連データ構造

  • FixedParamState: 固定パラメータの型情報を保持する構造体。
  • ParamRef: パーサーが生成するパラメータ参照ノード。
  • Param: 変換後のパラメータノード。

失敗モードと境界条件

  • パラメータ番号が 0 以下、または FixedParamState が保持する最大数を超えている場合。
  • 指定されたパラメータ番号に対応する型が未定義（InvalidOid）である場合。
  • 上記いずれの場合も ereport によりエラー終了する。

保守担当者向けメモ

  • 本関数は PARAM_EXTERN を生成する。paramtypmod が -1 に固定されている点は、実行時の型チェックを確実にするための設計である。
