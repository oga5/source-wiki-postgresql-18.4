chr

定義

  • 種別: typedef
  • 定義箇所: src/include/regex/regcustom.h:61

目的

PostgreSQL の正規表現エンジンにおいて、処理対象となる文字型を定義する抽象化レイヤーです。環境依存の文字型を pg_wchar に統一することで、移植性を確保します。

シグネチャまたは形状

typedef pg_wchar chr;

入力と出力

  • 入力: pg_wchar 型の文字コード。
  • 出力: 正規表現エンジン内部で文字を表現する型として利用されます。

前提条件と不変条件

  • CHRBITS は 32 に定義されており、chr 型は 32 ビットの幅を持つことが想定されています。
  • CHR_MAX（0x7ffffffe）は、CHR_MAX - CHR_MIN + 1 が int 型に収まり、かつ CHR_MAX + 1 が chr 型の変数に収まる必要があります。

アルゴリズムまたは振る舞い

  • 本シンボルは、正規表現エンジンが文字を扱う際の基底型として機能します。
  • regcustom.h 内で定義された iscalnum や iscdigit などのマクロを通じて、PostgreSQL のワイド文字処理関数（pg_wc_isalnum 等）と連携します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_color.c 内の pg_reg_getcolor、subcolor、および文字範囲処理ロジックなどで型として参照されています。
  • **呼び出し先**: 本シンボル自体は型定義であり、関数呼び出しは行いません。
  • 注: contrib/unaccent/generate_unaccent_rules.py にある chr() は Python の組み込み関数であり、本シンボルとは無関係です。

関連データ構造

  • uchr: chr を保持するための符号なし型。
  • CHR_MAX, CHR_MIN: 文字コードの範囲制限。
  • MAX_SIMPLE_CHR: カラーマップ処理における「単純な文字」の閾値。

失敗モードと境界条件

  • CHR_IS_IN_RANGE(c) マクロは、引数 c を複数回評価する可能性があるため、副作用のある式を渡さないよう注意が必要です。
  • chr 型の定義を変更する場合、CHRBITS や CHR_MAX との整合性を維持する必要があります。

保守担当者向けメモ

  • 本シンボルは pg_wchar のエイリアスです。文字型の定義を変更する際は、正規表現エンジンのメモリレイアウトやカラーマップの計算ロジックに影響を与えるため、十分な検証が必要です。
