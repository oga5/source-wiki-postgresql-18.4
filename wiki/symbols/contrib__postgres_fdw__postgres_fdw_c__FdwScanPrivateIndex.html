FdwScanPrivateIndex

定義

  • 種別: enum
  • 定義箇所: contrib/postgres_fdw/postgres_fdw.c:73

目的

PgFdwScanState 構造体内の fdw_private リストに格納される各要素のインデックスを定義します。これにより、外部スキャン実行時に必要なメタデータや状態情報を一貫した順序で管理します。

シグネチャまたは形状

enum FdwScanPrivateIndex {
    FdwScanPrivateSelectSql,
    FdwScanPrivateRetrievedAttrs,
    FdwScanPrivateFetchSize,
    FdwScanPrivateRelations
};

入力と出力

  • **入力**: なし（列挙型定義）
  • **出力**: fdw_private リスト内の各要素にアクセスするための整数インデックス。

前提条件と不変条件

  • fdw_private リスト内の要素は、この列挙型で定義された順序と厳密に一致している必要があります。

アルゴリズムまたは振る舞い

各列挙子は以下のデータを指し示します：

  • FdwScanPrivateSelectSql: リモートで実行する SQL 文（String ノード）。
  • FdwScanPrivateRetrievedAttrs: 取得する属性番号のリスト（Integer リスト）。
  • FdwScanPrivateFetchSize: フェッチサイズ（Integer）。
  • FdwScanPrivateRelations: 結合スキャンの場合、結合対象のテーブル名や結合型を記述する文字列。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/postgres_fdw.c 内の外部スキャン実行ロジック。
  • **呼び出し先**: 情報なし。
  • **備考**: contrib/postgres_fdw/postgres_fdw.c:1418 等で、リストの順序がこの列挙型と一致している必要がある旨が言及されています。

関連データ構造

  • PgFdwScanState: この列挙型を使用して fdw_private リストの要素を管理する構造体。

失敗モードと境界条件

  • リストの順序がこの列挙型の定義と一致しない場合、誤ったデータ型へのキャストや不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 新しいプライベートデータを追加する場合は、必ずこの列挙型の末尾に追加し、対応する fdw_private リスト構築箇所（postgresGetForeignPlan 等）を更新してください。
