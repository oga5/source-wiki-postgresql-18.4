QTNSort

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_util.c:162

目的

QTNode ツリー構造のノード順序を正規化し、クエリの等価性判定を可能にする。

シグネチャまたは形状

void QTNSort(QTNode *in)

入力と出力

  • **入力**: in - ソート対象の QTNode ツリーのルートノード。
  • **出力**: なし（ツリー構造をインプレースで書き換える）。

前提条件と不変条件

  • in は有効な QTNode ツリーであること。
  • 再帰的な処理を行うため、スタックの深さが許容範囲内であること。
  • クエリの等価性判定（QTNEq）を行う前に、本関数による正規化が完了している必要がある。

アルゴリズムまたは振る舞い

1. check_stack_depth() を呼び出し、スタックオーバーフローを防止する。 2. ノードの型が QI_OPR（演算子）でない場合は処理を終了する。 3. 子ノードに対して再帰的に QTNSort を適用する。 4. 子ノードが2つ以上存在し、かつ演算子が OP_PHRASE ではない場合、qsort と比較関数 cmpQTN を使用して子ノードの順序をソートする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_rewrite.c (複数箇所)
  • src/backend/utils/adt/tsquery_util.c (再帰呼び出し)
  • **呼び出し先**:
  • check_stack_depth()
  • QTNSort() (再帰)
  • qsort()
  • cmpQTN()

関連データ構造

  • QTNode: tsquery を表現するメモリ上のツリー構造体。

失敗モードと境界条件

  • **スタックオーバーフロー**: 深い階層を持つツリーに対して再帰的に呼び出された場合、check_stack_depth() によりエラーが発生する可能性がある。
  • **非演算子ノード**: QI_OPR 型以外のノードが渡された場合、ソート処理はスキップされる。
  • **PHRASE 演算子**: OP_PHRASE 演算子を持つノードは、順序が意味を持つためソート対象外となる。

保守担当者向けメモ

  • 本関数は QTNEq による等価性判定の前提となるため、ツリー構造を変更した後は必ず呼び出すこと。
  • 再帰関数であるため、スタックサイズに注意が必要。
