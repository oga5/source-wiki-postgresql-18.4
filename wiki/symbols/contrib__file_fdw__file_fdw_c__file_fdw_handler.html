file_fdw_handler

定義

  • 種別: function
  • 定義箇所: contrib/file_fdw/file_fdw.c:181

目的

PostgreSQL の外部データラッパー（FDW）インターフェースを初期化し、file_fdw が提供する各コールバック関数を登録した FdwRoutine 構造体を返します。

シグネチャまたは形状

Datum file_fdw_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQL の関数呼び出し規約に基づく引数）。
  • **出力**: FdwRoutine 構造体へのポインタを Datum 型として返します。

前提条件と不変条件

  • makeNode(FdwRoutine) によりメモリが割り当てられ、各関数ポインタが適切に設定されている必要があります。
  • 呼び出し元は、返された FdwRoutine を使用して外部テーブルのプランニングおよび実行フェーズを制御します。

アルゴリズムまたは振る舞い

1. makeNode(FdwRoutine) を呼び出し、FDW ルーチン構造体を生成します。 2. 以下の関数ポインタを構造体に割り当てます。

  • GetForeignRelSize: fileGetForeignRelSize
  • GetForeignPaths: fileGetForeignPaths
  • GetForeignPlan: fileGetForeignPlan
  • ExplainForeignScan: fileExplainForeignScan
  • BeginForeignScan: fileBeginForeignScan
  • IterateForeignScan: fileIterateForeignScan
  • ReScanForeignScan: fileReScanForeignScan
  • EndForeignScan: fileEndForeignScan
  • AnalyzeForeignTable: fileAnalyzeForeignTable
  • IsForeignScanParallelSafe: fileIsForeignScanParallelSafe

    • 3. PG_RETURN_POINTER マクロを使用して、構築した構造体を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のコアシステム（FDW の初期化時）。
  • **呼び出し先**: makeNode（ノード生成用マクロ）。
  • **注記**: 上記以外の具体的な呼び出し元・呼び出し先情報は、提供されたソース範囲内では確認できません。

関連データ構造

  • FdwRoutine: FDW の動作を定義するコールバック関数の集合体。

失敗モードと境界条件

  • makeNode がメモリ不足により失敗した場合、PostgreSQL のエラーハンドリング機構により例外が発生します。

保守担当者向けメモ

  • 本関数は file_fdw のエントリポイントです。新しい FDW 機能を追加または変更する場合は、この関数で対応するコールバック関数を FdwRoutine に正しく登録してください。
  • PG_FUNCTION_INFO_V1(file_fdw_handler) が同一ファイル内の 125 行目で宣言されており、動的ロードのためのメタデータを提供しています。
