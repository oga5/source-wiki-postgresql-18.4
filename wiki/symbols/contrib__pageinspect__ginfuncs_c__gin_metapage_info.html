gin_metapage_info

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/ginfuncs.c:27

目的

GINインデックスのメタページの内容を解析し、統計情報やバージョン情報を含むレコードとして出力する。

シグネチャまたは形状

Datum gin_metapage_info(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型の raw ページデータ（PG_GETARG_BYTEA_P(0)）。
  • **出力**: メタページの各フィールドを格納した HeapTuple（Datum 型）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 入力ページは GIN_META フラグを持つ有効なGINメタページである必要がある。
  • ページサイズが GinPageOpaqueData のサイズと一致しなければならない。

アルゴリズムまたは振る舞い

1. superuser() チェックを行い、権限がない場合はエラーを投げる。 2. get_page_from_raw() で raw データをメモリ上の Page 構造体に変換する。 3. PageIsNew() でページが空でないか確認し、空であれば NULL を返す。 4. PageGetSpecialSize() および GinPageGetOpaque() を使用して、ページがGINメタページであることを検証する。 5. GinPageGetMeta() を介してメタデータ構造体を取得する。 6. メタデータ内の各フィールド（head, tail, tailFreeSize, nPendingPages, nPendingHeapTuples, nTotalPages, nEntryPages, nDataPages, nEntries, ginVersion）を values 配列に格納する。 7. heap_form_tuple() を使用して結果タプルを構築し、返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • superuser()
  • get_page_from_raw()
  • PageIsNew()
  • PageGetSpecialSize()
  • GinPageGetOpaque()
  • get_call_result_type()
  • GinPageGetMeta()
  • heap_form_tuple()

関連データ構造

  • GinPageOpaqueData: ページヘッダーの不透明領域。
  • GinMetaPageData: GINメタページの主要な統計情報構造体。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE を発生させる。
  • **無効なページ**: ページサイズが不一致、または GIN_META フラグが立っていない場合、ERRCODE_INVALID_PARAMETER_VALUE を発生させる。
  • **戻り値型不一致**: 呼び出し元の期待する型が複合型でない場合、エラーを発生させる。

保守担当者向けメモ

  • 本関数は contrib/pageinspect の一部であり、デバッグ用途を想定している。
  • メタデータのフィールド構成を変更する場合は、values 配列のインデックスと tupdesc の定義を同期させる必要がある。
