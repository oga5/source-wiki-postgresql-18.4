mark_index_clustered

定義

  • 種別: function
  • 定義箇所: src/backend/commands/cluster.c:553

目的

指定されたインデックスを、リレーションのクラスタリング対象としてシステムカタログ（pg_index）上でマーク（または解除）します。

シグネチャまたは形状

void mark_index_clustered(Relation rel, Oid indexOid, bool is_internal)

入力と出力

  • rel: 対象テーブルのリレーション記述子。
  • indexOid: クラスタリング対象とするインデックスのOID。InvalidOid を指定した場合は、既存のクラスタリング設定をすべて解除します。
  • is_internal: 内部的な操作かどうかを示すフラグ（フック呼び出しに使用）。
  • 出力: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • パーティションテーブルに対しては実行できません（エラーとなります）。
  • indexOid が指定された場合、そのインデックスは有効（indisvalid）である必要があります。

アルゴリズムまたは振る舞い

1. パーティションテーブルでないことを確認します。 2. indexOid が既にクラスタリング対象としてマークされている場合は、何もせず終了します。 3. pg_index システムカタログを RowExclusiveLock で開きます。 4. 対象リレーションに属する全インデックスを走査します。 5. 各インデックスについて以下の処理を行います：

  • 既に indisclustered が設定されているインデックスは、そのビットをオフにします。
  • 現在の indexOid と一致するインデックスについては、indisclustered をオンにします。
  • CatalogTupleUpdate を用いてシステムカタログを更新し、InvokeObjectPostAlterHookArg を呼び出して変更を通知します。

    • 6. 最後に pg_index を閉じます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/cluster.c
  • src/backend/commands/tablecmds.c
  • 呼び出し先:
  • table_open, table_close
  • RelationGetIndexList
  • SearchSysCacheCopy1, heap_freetuple
  • CatalogTupleUpdate
  • InvokeObjectPostAlterHookArg

関連データ構造

  • Relation: テーブルおよびインデックスのメタデータ。
  • Form_pg_index: pg_index システムカタログの行構造。

失敗モードと境界条件

  • パーティションテーブルを渡すと ERRCODE_FEATURE_NOT_SUPPORTED でエラーになります。
  • 無効なインデックス（indisvalid が偽）をクラスタリング対象として指定するとエラーになります。
  • システムカタログのキャッシュルックアップに失敗した場合、エラーが発生します。

保守担当者向けメモ

  • 本関数は「1テーブルにつき最大1つのインデックスのみがクラスタリング対象である」という不変条件を維持するために、既存のフラグをすべてクリアしてから新しいフラグを立てる設計になっています。
  • is_internal フラグは、DDL操作のログ出力やイベントトリガーの挙動を制御するために使用されます。
