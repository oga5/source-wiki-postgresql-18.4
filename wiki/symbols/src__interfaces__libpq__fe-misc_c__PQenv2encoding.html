PQenv2encoding

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-misc.c:1284

目的

環境変数 PGCLIENTENCODING からクライアントのエンコーディング設定を取得し、内部的なエンコーディング識別子に変換する。

シグネチャまたは形状

int PQenv2encoding(void);

入力と出力

  • **入力**: なし（環境変数 PGCLIENTENCODING を直接参照）。
  • **出力**: 成功時は対応するエンコーディングの整数値。環境変数が未設定、空、または無効な値の場合は PG_SQL_ASCII を返す。

前提条件と不変条件

  • getenv を使用するため、プロセスの環境変数状態に依存する。
  • 戻り値は pg_char_to_encoding が認識可能なエンコーディング値、または PG_SQL_ASCII に限定される。

アルゴリズムまたは振る舞い

1. 環境変数 PGCLIENTENCODING を取得する。 2. 変数が存在し、かつ空文字列でない場合、pg_char_to_encoding を呼び出して文字列をエンコーディング識別子に変換する。 3. pg_char_to_encoding が負の値を返した（不明なエンコーディング）場合、デフォルトとして PG_SQL_ASCII を割り当てる。 4. 最終的なエンコーディング値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/startup.c:157 にて pset.encoding の初期化に使用されている。
  • **呼び出し先**: getenv（標準ライブラリ）、pg_char_to_encoding（内部関数）。

関連データ構造

  • なし。

失敗モードと境界条件

  • 環境変数が未設定の場合: PG_SQL_ASCII を返す。
  • 環境変数が空文字列の場合: PG_SQL_ASCII を返す。
  • 環境変数の値がサポート外のエンコーディング名の場合: PG_SQL_ASCII を返す。

保守担当者向けメモ

  • 本関数は libpq-fe.h で公開されている API である。
  • エンコーディング名の判定ロジックは pg_char_to_encoding に依存しているため、サポートするエンコーディングを追加・変更する場合はそちらを確認すること。
