RowSecurityDesc

定義

  • **種別**: struct
  • **定義箇所**: src/include/rewrite/rowsecurity.h:31

目的

特定のリレーション（テーブル）に関連付けられた行レベルセキュリティ（RLS）ポリシーの集合を保持するためのコンテナ構造体です。

シグネチャまたは形状

typedef struct RowSecurityDesc
{
    MemoryContext rscxt;        /* row security memory context */
    List         *policies;     /* list of row security policies */
} RowSecurityDesc;

入力と出力

  • **入力**: 構造体生成時に MemoryContext が割り当てられ、ポリシーリストが格納されます。
  • **出力**: リレーションキャッシュ（Relation）の一部として保持され、クエリ書き換え時にポリシー情報を提供します。

前提条件と不変条件

  • rscxt は、この構造体および関連するポリシーリストの生存期間を管理するメモリコンテキストである必要があります。
  • policies は RowSecurityPolicy 構造体のリストを保持します。

アルゴリズムまたは振る舞い

  • クエリ書き換え時、この構造体を通じてリレーションに適用すべきセキュリティポリシーが取得されます。
  • src/backend/utils/cache/relcache.c において、2つの RowSecurityDesc が等価であるかを判定する処理（equalRSDesc）が存在します。
  • リレーションキャッシュ（Relation）の rd_rsdesc フィールドを通じて、対象リレーションのポリシー情報が管理されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/commands/policy.c: ポリシー定義の操作時にインスタンス化されます。
  • src/backend/utils/cache/relcache.c: リレーションキャッシュの管理および等価性判定に使用されます。
  • src/include/utils/rel.h: Relation 構造体のメンバ rd_rsdesc として定義されています。
  • **呼び出し先**:
  • 提供されたソース情報からは、本構造体が直接呼び出す関数や、本構造体を操作する具体的な関数呼び出しの連鎖は特定されていません。

関連データ構造

  • RowSecurityPolicy: policies リスト内に格納される個別のポリシー定義。
  • Relation: rd_rsdesc メンバを通じて本構造体を保持します。
  • MemoryContext: メモリ管理に使用されます。

失敗モードと境界条件

  • rd_rsdesc が NULL の場合、そのリレーションにはRLSポリシーが設定されていないことを示します。
  • メモリコンテキストの管理が不適切な場合、ポリシーリストのメモリリークや不正な参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体はリレーションキャッシュと密接に関連しています。relcache の更新や無効化のタイミングで、rd_rsdesc の整合性が保たれていることを確認してください。
  • equalRSDesc を変更する際は、policies リスト内の各ポリシーの比較ロジックが正しく維持されているか注意が必要です。
