array_subscript_fetch_old

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arraysubs.c:399

目的

配列の要素更新（代入）操作を行う直前に、更新対象となる既存の要素値を読み取り、SubscriptingRefState に保存するために使用されます。

シグネチャまたは形状

static void array_subscript_fetch_old(ExprState *state, ExprEvalStep *op, ExprContext *econtext)

入力と出力

  • **入力**:
  • state: 式評価の状態。
  • op: 現在の式評価ステップ。op->resvalue に配列データ、op->resnull にその NULL 状態が含まれます。
  • econtext: 式の評価コンテキスト。
  • **出力**:
  • なし（副作用として op->d.sbsref.state が指す SubscriptingRefState 構造体の prevvalue および prevnull フィールドを更新します）。

前提条件と不変条件

  • op->d.sbsref.state が有効な SubscriptingRefState を指している必要があります。
  • workspace には、配列の次元情報や要素の型情報（長さ、アライメント等）が正しく設定されている必要があります。

アルゴリズムまたは振る舞い

1. op->resnull を確認し、配列全体が NULL の場合は、prevnull を true に設定して終了します。 2. 配列が NULL でない場合、workspace 内のインデックス情報（upperindex）および型情報を用いて array_get_element を呼び出します。 3. 取得した要素の値を sbsrefstate->prevvalue に、NULL 状態を sbsrefstate->prevnull にそれぞれ格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arraysubs.c 内のハンドラ設定箇所（methods->sbs_fetch_old = array_subscript_fetch_old;）で参照されています。
  • 実行エンジンから代入操作の一環として呼び出されます。
  • **呼び出し先**:
  • array_get_element（utils/array.h で定義）を呼び出し、実際の配列要素の抽出を行います。

関連データ構造

  • SubscriptingRefState: 添字操作の実行状態を保持する構造体。
  • ArraySubWorkspace: 配列の次元や要素の属性情報を保持するワークスペース。

失敗モードと境界条件

  • 配列が NULL の場合、prevvalue は (Datum) 0 に設定され、prevnull が true となります。
  • 添字が配列の範囲外である場合、array_get_element の挙動に依存します。

保守担当者向けメモ

  • 本関数は代入操作の「更新前値」を保持するための補助的な役割を担います。
  • array_subscript_fetch_old_slice と対になる設計です。
  • workspace の各フィールド（refattrlength 等）が正しく初期化されていることを前提としているため、実行計画構築時の array_exec_setup との整合性に注意してください。
