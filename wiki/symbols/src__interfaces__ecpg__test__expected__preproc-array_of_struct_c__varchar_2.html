varchar_2

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:64

目的

ECPG（Embedded SQL for C）における可変長文字列（VARCHAR）型を表現するための構造体定義です。データベースとの間で文字列データをやり取りする際のホスト変数として使用されます。

シグネチャまたは形状

struct varchar_2 {
    int len;
    char arr[50];
};

入力と出力

  • **入力**: SQL の SELECT 文等の結果セットから取得される文字列データ。
  • **出力**: データベースへの挿入や更新操作におけるホスト変数としてのデータ保持。

前提条件と不変条件

  • len メンバは、arr に格納されている有効な文字列の長さを保持する必要があります。
  • arr は固定長（50バイト）のバッファであり、これを超える文字列を格納することはできません。

アルゴリズムまたは振る舞い

ECPG プリプロセッサによって生成されるコードにおいて、この構造体は SQL の VARCHAR 型と C 言語のメモリレイアウトを橋渡しします。データベースから値を取得する際、ECPG ライブラリは len に文字列長を、arr に実際の文字データを書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPG によって生成された ECPGdo 関数呼び出しの引数として参照されます。
  • **呼び出し先**: 情報なし。

関連データ構造

  • customer / customer2 / customer3 / customer4: 本構造体を利用する可能性のあるテスト用構造体。
  • cust_ind: 構造体メンバに対応するインジケータ変数。

失敗モードと境界条件

  • arr のサイズ（50バイト）を超えるデータを格納しようとした場合、バッファオーバーフローやデータの切り捨てが発生する可能性があります。
  • ECPG の型変換ルールに従わない操作を行った場合、実行時にエラーが発生し sqlca にエラー情報が記録されます。
