ConnectDatabaseAhx

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:108

目的

ArchiveHandle を介してデータベースへの接続を確立し、初期設定（検索パスの固定、バージョン検証、通知プロセッサの設定、シグナルハンドリングの登録）を行う。

シグネチャまたは形状

void ConnectDatabaseAhx(Archive *AHX, const ConnParams *cparams, bool isReconnect);

入力と出力

  • **入力**:
  • AHX: 接続情報を保持する Archive 構造体へのポインタ。
  • cparams: 接続パラメータ（DB名、ホスト、ポート、ユーザー名等）を含む ConnParams 構造体。
  • isReconnect: 再接続かどうかを示すフラグ。
  • **出力**: なし（副作用として AHX 内の connection フィールドを更新）。

前提条件と不変条件

  • AH->connection が既に存在する場合、pg_fatal によりプロセスが終了する。
  • 再接続時（isReconnect が true）は、パスワードの入力を促さない。

アルゴリズムまたは振る舞い

1. **接続準備**: isReconnect フラグに基づきパスワード取得要否を判定し、必要に応じて simple_prompt で入力を求める。 2. **接続確立**: ConnectDatabase を呼び出し、libpq 接続を生成して AH->connection に格納する。 3. **初期化クエリ**: ALWAYS_SECURE_SEARCH_PATH_SQL を実行し、検索パスを固定する。 4. **パスワード管理**: 接続に使用されたパスワードを AH->savedPassword に保存する。 5. **検証と設定**:

  • _check_database_version でサーバーバージョンを検証。
  • PQsetNoticeProcessor で通知処理を設定。
  • set_archive_cancel_info で SIGINT によるクエリキャンセルを有効化。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c
  • src/bin/pg_dump/pg_backup_db.c
  • src/bin/pg_dump/pg_dump.c
  • **呼び出し先**:
  • ConnectDatabase (接続確立)
  • ExecuteSqlQueryForSingleRow (検索パス設定)
  • _check_database_version (バージョンチェック)
  • set_archive_cancel_info (キャンセル設定)
  • PQclear, PQconnectionUsedPassword, PQpass, PQsetNoticeProcessor (libpq 関数)

関連データ構造

  • ArchiveHandle: 接続状態やパスワードを保持する内部構造体。
  • ConnParams: 接続に必要なパラメータを保持する構造体。

失敗モードと境界条件

  • 既に接続済みの場合は致命的エラーとして終了する。
  • ConnectDatabase が失敗した場合の挙動は、呼び出し先の ConnectDatabase 実装に依存する。

保守担当者向けメモ

  • 本関数は ArchiveHandle を Archive 型からキャストして操作する。
  • パスワードの取り扱いにおいて、プロンプトで取得した値と既存の保存値を適切に管理するロジックが含まれているため、変更時は注意が必要。
