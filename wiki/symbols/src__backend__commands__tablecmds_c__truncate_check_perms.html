truncate_check_perms

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tablecmds.c:2419

目的

指定されたリレーション（テーブル）に対して、現在のユーザーが TRUNCATE 操作を実行する権限を持っているかを検証する。

シグネチャまたは形状

static void truncate_check_perms(Oid relid, Form_pg_class reltuple)

入力と出力

  • **入力**:
  • relid: 検証対象リレーションの OID。
  • reltuple: 対象リレーションの pg_class カタログエントリ（Form_pg_class 構造体）。
  • **出力**: なし（権限がない場合はエラーをスローして終了する）。

前提条件と不変条件

  • reltuple は relid に対応する有効な pg_class 行であること。
  • GetUserId() を通じて現在のセッションユーザーが特定可能であること。

アルゴリズムまたは振る舞い

1. pg_class_aclcheck を呼び出し、現在のユーザーに対して ACL_TRUNCATE 権限があるかを確認する。 2. 権限チェックの結果が ACLCHECK_OK でない場合、aclcheck_error を呼び出してエラーを発生させる。 3. エラー発生時には、get_relkind_objtype を用いてリレーションの種類に応じた適切なエラーメッセージを生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c 内の TRUNCATE 処理に関連する箇所（行 369, 1939, 2032, 19546 付近）。
  • **呼び出し先**:
  • pg_class_aclcheck
  • aclcheck_error
  • get_relkind_objtype
  • NameStr（マクロ）

関連データ構造

  • Form_pg_class: システムカタログ pg_class の行構造体。
  • AclResult: 権限チェックの結果を示す列挙型。

失敗モードと境界条件

  • ユーザーに TRUNCATE 権限がない場合、aclcheck_error によりトランザクションを中断するエラーがスローされる。
  • relid が無効な場合、内部の pg_class_aclcheck がエラーを処理する。

保守担当者向けメモ

  • 本関数は TRUNCATE コマンドの実行前チェックとして使用される。
  • 権限チェックのロジックを変更する場合は、utils/acl.h で定義されるアクセス制御の仕様と整合性を取ること。
