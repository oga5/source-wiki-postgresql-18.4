mbms_is_member

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/multibitmapset.c:125

目的

指定された二次元インデックス（listidx および bitidx）が、マルチビットマップセット（List）内に存在するかを判定する。

シグネチャまたは形状

bool mbms_is_member(int listidx, int bitidx, const List *a)

入力と出力

  • **入力**:
  • listidx: リスト内の位置を指定する整数。
  • bitidx: 指定された Bitmapset 内のビット位置を指定する整数。
  • a: 検索対象の List（Bitmapset のリスト）。
  • **出力**:
  • 指定されたインデックスが集合に含まれていれば true、含まれていない、または範囲外であれば false を返す。

前提条件と不変条件

  • listidx および bitidx は負の値であってはならない（負の場合はエラーとなる）。
  • a が NULL の場合、list_length(a) の呼び出しにより動作が依存する（list.c の仕様に準拠）。

アルゴリズムまたは振る舞い

1. listidx または bitidx が負の場合、elog(ERROR) を発生させる。 2. listidx がリスト a の長さ以上である場合、集合に含まれないと判断し false を返す。 3. list_nth_node を使用して、リストの listidx 番目の要素（Bitmapset）を取得する。 4. 取得した Bitmapset に対して bms_is_member(bitidx, bms) を呼び出し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていない。
  • **呼び出し先**:
  • list_length
  • list_nth_node
  • bms_is_member
  • elog

関連データ構造

  • List: マルチビットマップセットを構成するコンテナ。
  • Bitmapset: 各リスト要素が保持するビット集合。

失敗モードと境界条件

  • **エラー**: listidx < 0 または bitidx < 0 の場合、エラーが発生する。
  • **境界条件**: listidx がリストの範囲外（>= list_length(a)）である場合、エラーにはならず false が返される。

保守担当者向けメモ

  • 本関数はインプレース更新を行わず、読み取り専用の判定を行う。
  • リスト内に NULL 要素が含まれる可能性があるため、list_nth_node の使用には注意が必要である。
