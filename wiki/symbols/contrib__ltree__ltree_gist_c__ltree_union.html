ltree_union

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_gist.c:191

目的

GiST インデックスにおいて、複数のインデックスエントリを包含する単一の結合ノード（ltree_gist）を生成します。

シグネチャまたは形状

Datum ltree_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *entryvec（結合対象のインデックスエントリ群）。
  • PG_GETARG_POINTER(1): int *size（結果のサイズを格納するポインタ）。
  • **出力**:
  • Datum: 結合された ltree_gist 構造体へのポインタ。
  • *size: 結合結果のバイトサイズが設定されます。

前提条件と不変条件

  • LTREE_GET_SIGLEN() により、現在のシグネチャ長（siglen）が取得可能であること。
  • entryvec 内の各エントリは ltree_gist 型として解釈可能であること。

アルゴリズムまたは振る舞い

1. entryvec 内の全エントリを走査し、以下の情報を集約します。

  • **シグネチャ**: 各エントリのビットマップを論理和（OR）で結合します。いずれかのエントリが「すべて真（all true）」を示す場合、結果も「すべて真」となります。
  • **範囲**: 全エントリの中で最小の ltree 値（left）と最大の ltree 値（right）を特定します。

    • 2. 全ビットがセットされている場合は isalltrue フラグを立てます。 3. ltree_gist_alloc を呼び出し、集約したシグネチャ、範囲、フラグを用いて新しい ltree_gist 構造体を生成します。 4. 生成された構造体のサイズを *size に書き込み、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセスから呼び出されます（ソースコード内の PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • hashing(): シグネチャ生成用。
  • ltree_compare(): 範囲（left, right）の特定用。
  • ltree_gist_alloc(): 構造体のメモリ確保と初期化用。

関連データ構造

  • GistEntryVector: GiST インデックスエントリの配列。
  • ltree_gist: ltree 型の GiST インデックス用内部表現。
  • BITVECP: シグネチャを保持するビットマップ型。

失敗モードと境界条件

  • entryvec が空の場合の挙動については、本ソース抜粋からは明示されていません。
  • siglen が 0 の場合や、メモリ不足時の palloc0 の挙動は PostgreSQL の標準的なエラー処理に依存します。

保守担当者向けメモ

  • 本関数は GiST インデックスの union メソッドとして機能します。新しい演算子やインデックス戦略を追加する際は、シグネチャの結合ロジックが適切か確認してください。
  • LOOPBYTE(siglen) マクロを使用してビットマップ操作を行っているため、siglen の変更がインデックスの互換性に与える影響に注意してください。
