LogicalParallelApplyLoop

定義

  • **種別**: function
  • **定義箇所**: src/backend/replication/logical/applyparallelworker.c:733

目的

論理レプリケーションの並列適用ワーカーにおいて、リーダーワーカーから共有メモリメッセージキュー（shm_mq）経由で送信される変更データを受信し、適用処理を実行するメインループを提供します。

シグネチャまたは形状

static void LogicalParallelApplyLoop(shm_mq_handle *mqh)
  • mqh: リーダーワーカーとの通信に使用する共有メモリメッセージキューのハンドル。

入力と出力

  • **入力**: shm_mq_handle *mqh（リーダーからのメッセージソース）。
  • **出力**: なし（ループ内で apply_dispatch を呼び出し、トランザクション変更を適用）。

前提条件と不変条件

  • ApplyMessageContext が初期化され、メッセージ処理ごとにリセットされること。
  • error_context_stack に apply_error_callback がプッシュされていること。
  • リーダーから送信されるメッセージの最初のバイトは 'w' であること。

アルゴリズムまたは振る舞い

1. **初期化**: メモリコンテキスト（ApplyMessageContext）とエラーコールバックを設定します。 2. **メッセージ受信**: shm_mq_receive を使用してリーダーからのデータを待機します。

  • **成功時**: メッセージが 'w' で始まることを確認し、統計情報をスキップした後、apply_dispatch を呼び出して適用します。
  • **ブロック時**: pa_process_spooled_messages_if_required を呼び出し、ファイルにシリアライズされたメッセージがあれば処理します。その後、WaitLatch で次のメッセージを待機します。待機中にタイムアウトが発生した場合は、統計情報を報告します。
  • **切断時**: リーダーとの接続が失われたと判断し、エラーを報告します。

    • 3. **後処理**: 各メッセージ処理後に ApplyMessageContext をリセットし、ループを継続します。

呼び出し元と呼び出し先

  • **呼び出し元**: ParallelApplyWorkerMain（src/backend/replication/logical/applyparallelworker.c:986）。
  • **呼び出し先**: ProcessParallelApplyInterrupts, shm_mq_receive, pq_getmsgbyte, apply_dispatch, pa_process_spooled_messages_if_required, WaitLatch, ResetLatch, pgstat_report_stat。

関連データ構造

  • shm_mq_handle: リーダー・ワーカー間の通信キュー。
  • ApplyMessageContext: メッセージ処理ごとのメモリ管理用コンテキスト。
  • ErrorContextCallback: エラー発生時のコンテキスト保持用。

失敗モードと境界条件

  • **メッセージ形式不正**: 受信データの先頭バイトが 'w' でない場合、elog(ERROR) で停止します。
  • **キュー切断**: shm_mq_receive が SHM_MQ_DETACHED を返した場合、リーダーとの接続喪失としてエラーを報告します。
  • **メッセージ長**: 長さが 0 のメッセージを受信した場合、エラーとなります。

保守担当者向けメモ

  • 本関数は ApplyMessageContext を使用してメモリを管理しています。メッセージ処理後のメモリリークを防ぐため、ループ末尾での MemoryContextReset が必須です。
