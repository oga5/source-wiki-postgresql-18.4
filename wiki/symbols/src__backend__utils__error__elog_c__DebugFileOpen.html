DebugFileOpen

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:2115

目的

デバッグ出力用ファイルが指定されている場合に、標準エラー出力（stderr）および標準出力（stdout）をそのファイルへリダイレクトする。

シグネチャまたは形状

void DebugFileOpen(void)

入力と出力

  • **入力**: グローバル変数 OutputFileName（ファイルパス）および IsUnderPostmaster（ポストマスター配下かどうかのフラグ）。
  • **出力**: なし（副作用として stderr および stdout のリダイレクトを行う）。

前提条件と不変条件

  • OutputFileName が空文字列でない場合にのみ処理が実行される。
  • ファイルオープンやリダイレクトに失敗した場合、ereport(FATAL, ...) によりプロセスが終了する。

アルゴリズムまたは振る舞い

1. OutputFileName が設定されているか確認する。 2. 指定されたファイルを O_APPEND | O_WRONLY モードでオープンし、書き込み権限を確認する。 3. isatty() を使用して、そのファイルが端末（tty）であるかを判定する。 4. freopen() を使用して stderr を指定ファイルにリダイレクトする。 5. IsUnderPostmaster が真かつファイルが端末である場合、stdout も同様に指定ファイルへリダイレクトする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/init/postinit.c:619
  • **呼び出し先**: open, isatty, close, freopen, ereport（標準ライブラリおよびPostgreSQL内部関数）

関連データ構造

  • OutputFileName: デバッグ出力先のパスを保持するグローバル変数。
  • IsUnderPostmaster: プロセスの実行環境を示すフラグ。

失敗モードと境界条件

  • ファイルが開けない場合、または freopen によるリダイレクトに失敗した場合は FATAL エラーとなり、プロセスが停止する。
  • OutputFileName が空の場合、本関数は何の処理も行わない。

保守担当者向けメモ

  • 本関数はプロセス起動時の初期化フェーズで呼び出されることを想定している。
  • stdout のリダイレクトは IsUnderPostmaster かつ istty の場合に限定されており、これはログ出力の競合や不要なリダイレクトを避けるための設計である。
