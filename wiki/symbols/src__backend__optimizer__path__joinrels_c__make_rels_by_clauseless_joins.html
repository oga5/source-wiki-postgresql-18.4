make_rels_by_clauseless_joins

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinrels.c:313

目的

結合条件（Join Clause）が存在しないリレーションのペアに対して、結合リレーション（RelOptInfo）を構築するための探索を行います。

シグネチャまたは形状

static void make_rels_by_clauseless_joins(PlannerInfo *root, RelOptInfo *old_rel, List *other_rels)

入力と出力

  • **入力**:
  • root: クエリ全体のプランナ情報。
  • old_rel: 結合対象となる既存のリレーション。
  • other_rels: old_rel と結合を試みる候補リレーションのリスト。
  • **出力**: なし（副作用として make_join_rel を呼び出し、結合リレーションを生成する）。

前提条件と不変条件

  • other_rels 内の各要素は RelOptInfo 型である必要があります。
  • 結合対象の old_rel と other_rel の間でリレーションID（relids）が重複していないこと（bms_overlap で判定）。

アルゴリズムまたは振る舞い

1. other_rels リストを走査します。 2. 各 other_rel について、old_rel とのビットマップ（relids）が重複していないかを確認します。 3. 重複がない場合、make_join_rel(root, old_rel, other_rel) を呼び出し、結合リレーションの構築を試みます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinrels.c 内の結合探索ロジック（join_search_one_level 等）から呼び出されます。
  • **呼び出し先**: make_join_rel を呼び出します。

関連データ構造

  • PlannerInfo: プランナのグローバル状態。
  • RelOptInfo: 結合対象および生成される結合リレーションの情報を保持する構造体。

失敗モードと境界条件

  • other_rels が空の場合、何も行われません。
  • bms_overlap が真を返す場合（リレーションが既に結合済みである場合など）、そのペアに対する make_join_rel はスキップされます。

保守担当者向けメモ

  • 本関数は、結合条件がない（Clauseless）ケースを扱うため、主に直積結合（Cross Join）を生成するパスの一部として機能します。
  • 結合の正当性チェックは make_join_rel 内部で行われるため、本関数はリストの走査と重複チェックに専念しています。
