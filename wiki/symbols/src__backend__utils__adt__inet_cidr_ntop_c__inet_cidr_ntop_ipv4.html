inet_cidr_ntop_ipv4

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/inet_cidr_ntop.c:84

目的

IPv4アドレスのバイナリ表現を、CIDRプレフィックス長を含む人間が読み取り可能な文字列形式（例: 192.168.1.0/24）へ変換します。

シグネチャまたは形状

char *inet_cidr_ntop_ipv4(const u_char *src, int bits, char *dst, size_t size)

入力と出力

  • **入力**:
  • src: IPv4アドレスのネットワークバイトオーダーのバイナリデータ（u_char配列）。
  • bits: CIDRプレフィックス長（0〜32）。
  • dst: 変換結果を格納するバッファ。
  • size: dst バッファのサイズ。
  • **出力**:
  • 成功時: 変換後の文字列を格納した dst へのポインタ。
  • 失敗時: NULL を返し、errno を設定する。

前提条件と不変条件

  • bits は 0 から 32 の範囲内である必要があります。
  • dst バッファは、結果の文字列（最大18バイト程度）を格納するのに十分なサイズである必要があります。

アルゴリズムまたは振る舞い

1. **バリデーション**: bits が範囲外の場合、errno を EINVAL に設定し NULL を返します。 2. **ゼロプレフィックス処理**: bits が 0 の場合、0/0 形式の文字列を構築します。 3. **オクテット変換**: bits / 8 で指定される完全なオクテットをドット区切りで文字列化します。 4. **部分オクテット変換**: bits % 8 が存在する場合、残りのビットをマスク処理し、ドット区切りで追加します。 5. **CIDR付与**: 最後に /bits を文字列の末尾に連結します。 6. **バッファ管理**: 各ステップで size を確認し、不足する場合は EMSGSIZE を設定して NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の pg_inet_cidr_ntop から呼び出されます。
  • **呼び出し先**: SPRINTF マクロ（PostgreSQL内部の書式化関数）を使用します。

関連データ構造

  • inet 型および cidr 型（PostgreSQLのネットワークデータ型）。

失敗モードと境界条件

  • **EINVAL**: bits が 0 未満または 32 を超える場合に発生します。
  • **EMSGSIZE**: 提供された size が結果文字列を格納するには小さすぎる場合に発生します。

保守担当者向けメモ

  • 本関数はISCライセンスに基づくコードを含んでおり、PostgreSQLの他の部分とは異なるコーディングスタイル（SPRINTF マクロの使用など）が採用されています。
  • ゼロ圧縮やIPv6特有のロジックは含まれておらず、IPv4専用の単純な変換ロジックです。
  • バッファサイズチェックは厳密に行われており、変更を加える際は size の減算処理が正しく維持されているか確認してください。
