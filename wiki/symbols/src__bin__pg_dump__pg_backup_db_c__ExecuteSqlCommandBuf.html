ExecuteSqlCommandBuf

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:383

目的

ArchiveHandle の状態に応じて、SQL コマンドの実行、COPY データのストリーミング、または INSERT/COMMENT コマンドの処理を行うインターフェース関数です。

シグネチャまたは形状

int ExecuteSqlCommandBuf(Archive *AHX, const char *buf, size_t bufLen)

入力と出力

  • **入力**:
  • AHX: Archive 構造体へのポインタ（内部で ArchiveHandle にキャスト）。
  • buf: 実行または送信するデータを含むバッファ。
  • bufLen: バッファの長さ。
  • **出力**:
  • 戻り値: 処理したデータの長さ (bufLen) を返します。

前提条件と不変条件

  • AHX は有効な ArchiveHandle 構造体としてキャスト可能であること。
  • OUTPUT_COPYDATA モードの場合、AH->pgCopyIn が設定されている必要があります。
  • 一般的な SQL コマンドとして処理される場合、コマンドが呼び出しを跨いで分割されないことが想定されています。

アルゴリズムまたは振る舞い

1. AH->outputKind を確認し、以下のいずれかの処理を実行します。

  • OUTPUT_COPYDATA: PQputCopyData を使用してデータをサーバーへ送信します。失敗時は pg_fatal で終了します。
  • OUTPUT_OTHERDATA: ExecuteSimpleCommands を呼び出し、INSERT や COMMENT コマンドを処理します。
  • その他: 一般的な SQL コマンドとして扱います。buf がヌル終端されている場合はそのまま ExecuteSqlCommand を呼び出し、そうでない場合はメモリを確保してコピーし、ヌル終端を付与してから実行します。

    • 2. 処理完了後、bufLen を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_zstd.c:243
  • src/bin/pg_dump/pg_backup_archiver.c:1880
  • **呼び出し先**:
  • PQputCopyData (libpq)
  • ExecuteSimpleCommands
  • ExecuteSqlCommand
  • pg_fatal
  • pg_malloc / free

関連データ構造

  • ArchiveHandle: データベース接続や出力状態を保持する主要構造体。

失敗モードと境界条件

  • PQputCopyData がエラーを返した場合、pg_fatal によりプロセスが終了します。
  • メモリ確保が必要な状況で pg_malloc が失敗した場合、内部的にプロセスが終了する可能性があります。
  • buf がヌル終端されていない場合、一時的なメモリ確保とコピーが発生します。

保守担当者向けメモ

  • 本関数は ahwrite インターフェースとして機能します。
  • ExecuteSimpleCommands は簡易的なステートマシンで動作するため、複雑な SQL 構文の解析には限界がある点に注意してください。
