ParallelHashJoinBatchAccessor

定義

  • 種別: struct
  • 定義箇所: src/include/executor/hashjoin.h:207

目的

並列ハッシュ結合において、各バックエンドプロセスが個別に管理するバッチごとの状態を保持し、共有メモリ上のバッチ状態に対する競合を低減するためのアクセサ構造体です。

シグネチャまたは形状

typedef struct ParallelHashJoinBatchAccessor
{
    ParallelHashJoinBatch *shared;
    size_t      preallocated;
    size_t      ntuples;
    size_t      size;
    size_t      estimated_size;
    size_t      old_ntuples;
    bool        at_least_one_chunk;
    bool        outer_eof;
    bool        done;
    SharedTuplestoreAccessor *inner_tuples;
    SharedTuplestoreAccessor *outer_tuples;
} ParallelHashJoinBatchAccessor;

入力と出力

  • **入力**: 共有メモリ上の ParallelHashJoinBatch へのポインタを保持します。
  • **出力**: 各バックエンドが処理したタプル数やメモリ使用量、バッチの完了状態（done）、および内側・外側リレーションのタプルストアへのアクセス手段を提供します。

前提条件と不変条件

  • shared メンバは、並列ハッシュ結合の共有状態を指す必要があります。
  • ntuples や size などのカウンタは、プロセス間での競合を避けるために各バックエンドで部分的に管理されます。

アルゴリズムまたは振る舞い

  • **競合低減**: 各バックエンドが自身の ParallelHashJoinBatchAccessor を持つことで、共有状態への頻繁なアクセスを抑制します。
  • **状態管理**: バッチの処理状況（outer_eof や done）を追跡し、再パーティション化の前後でのタプル数（old_ntuples）を記録して成長制御を支援します。
  • **タプルアクセス**: SharedTuplestoreAccessor を介して、共有メモリ上のタプルストアに対する読み書きを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c 内のハッシュ結合実行ロジックが、hashtable->batches 配列として本構造体を初期化・参照します。
  • **呼び出し先**: SharedTuplestoreAccessor を通じてタプルストア操作を行います。

関連データ構造

  • ParallelHashJoinBatch: 共有メモリ上に配置されるバッチのメイン状態。
  • SharedTuplestoreAccessor: 共有タプルストアへのアクセス用インターフェース。
  • HashJoinTableData: 本構造体を配列として保持するハッシュ結合の全体状態構造体。

失敗モードと境界条件

  • メモリ不足時や再パーティション化発生時に、estimated_size や old_ntuples の値が適切に更新されない場合、バッチ処理の進行に影響が出る可能性があります。
  • 共有メモリの枯渇や SharedTuplestoreAccessor の初期化失敗は、本構造体を利用した処理の失敗に直結します。

保守担当者向けメモ

  • 本構造体は nodeHash.c で palloc0_array を用いて nbatch 分確保されます。バッチ数が増加する際の再割り当てやライフサイクル管理に注意してください。
  • preallocated や ntuples は、並列実行時のパフォーマンス最適化のためのローカルカウンタであるため、共有状態との同期タイミングを慎重に設計する必要があります。
