NodeTag

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/nodes.h:26

目的

PostgreSQLのクエリ処理で使用されるすべてのノード構造体を識別するための列挙型です。各ノードの先頭に配置される NodeTag フィールドを通じて、実行時にノードの型を特定し、安全なキャストや処理の分岐を可能にします。

シグネチャまたは形状

typedef enum NodeTag
{
	T_Invalid = 0,
    /* 他のノードタグは nodes/nodetags.h からインクルード */
} NodeTag;

入力と出力

  • **入力**: なし（列挙型の定義）。
  • **出力**: ノードの型を識別する整数値（列挙子）。

前提条件と不変条件

  • すべてのノード構造体は、先頭フィールドとして NodeTag を保持しなければなりません。
  • NodeTag の列挙順序はABIに直結するため、リリース済みのブランチでは変更不可です。
  • T_Invalid は 0 に固定されています。

アルゴリズムまたは振る舞い

  • makeNode マクロによってノードが生成される際、対応する NodeTag が自動的に付与されます。
  • IsA マクロや castNode マクロは、このタグを参照して実行時の型チェックを行います。
  • nodes/nodetags.h をインクルードすることで、システム全体で一貫したタグ定義を共有します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/nodes/list.c: リスト生成関数（new_list, list_make1_impl 等）でノード型を指定するために使用。
  • src/backend/executor/execAmi.c: ExecMaterializesOutput 等の実行時処理で型判定に使用。
  • src/backend/nodes/queryjumblefuncs.c: ノードツリーのシリアライズ時にタグを出力。
  • contrib/sepgsql/hooks.c: フック処理における型判定。
  • **呼び出し先**:
  • nodes/nodetags.h: 列挙子の実体定義を供給。

関連データ構造

  • Node: すべてのノード構造体の共通ヘッダー。
  • List: NodeTag を引数に取るリスト構造体。

失敗モードと境界条件

  • T_Invalid が設定されたノードを処理しようとすると、型判定ロジックでエラーまたは予期せぬ動作を引き起こす可能性があります。
  • 外部モジュール等で NodeTag の定義と実際の構造体の不整合が発生した場合、castNode によるアサーション失敗やメモリ破壊のリスクがあります。

保守担当者向けメモ

  • 新しいノード型を追加する場合は、nodes/nodetags.h を更新してください。
  • NodeTag の順序変更はABI破壊を招くため厳禁です。
  • ノード操作関数の自動生成には src/tools/gen_node_support.pl を活用してください。
