HEntry

定義

  • 種別: struct
  • 定義箇所: contrib/hstore/hstore.h:18

目的

hstore 型のデータ構造において、各キーおよび各値のメタデータ（位置情報やフラグ）を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	uint32		entry;
} HEntry;

入力と出力

  • 入力: uint32 型のビットフィールド（entry）。
  • 出力: なし（構造体としてメモリ上に配置される）。

前提条件と不変条件

  • hstore 構造体内部において、キーと値のそれぞれに対して HEntry が1つずつ割り当てられます。
  • entry フィールドは、データの終了位置やフラグ情報を保持します。実際のデータ長は、現在のエントリと前のエントリの差分から計算されます。

アルゴリズムまたは振る舞い

  • hstore のメモリレイアウトにおいて、HStore ヘッダーの直後に HEntry の配列が配置されます。
  • HSE_OFF や HSE_LEN マクロ（hstore.h で定義）を通じて、この構造体が保持する entry 値から実際のデータ領域のオフセットや長さを算出します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/hstore/hstore_compat.c（ARRPTR を介したアクセス）、contrib/hstore/hstore_gin.c など。
  • 呼び出し先: 情報なし。

関連データ構造

  • HStore: hstore 型のメイン構造体。HEntry 配列の基点となります。
  • HOldEntry: 旧バージョンの hstore 形式で使用されていた構造体。

失敗モードと境界条件

  • HEntry の操作はポインタ演算を伴うため、メモリレイアウトの整合性が崩れている場合（不正なオフセットなど）は未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • HEntry の entry フィールドは、データの終了位置を保持する設計です。長さを計算する際は、前のエントリとの差分をとる必要がある点に注意してください。
  • 本構造体は hstore の物理レイアウトに直結しているため、変更時は hstore_compat.c 等の互換性維持コードへの影響を考慮してください。
