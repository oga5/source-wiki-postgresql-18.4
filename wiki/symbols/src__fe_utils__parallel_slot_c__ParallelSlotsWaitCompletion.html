ParallelSlotsWaitCompletion

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:500

目的

ParallelSlotArray 内のすべての接続に対して処理の完了を待ち、結果を消費してスロットを解放します。

シグネチャまたは形状

bool ParallelSlotsWaitCompletion(ParallelSlotArray *sa)

入力と出力

  • **入力**: ParallelSlotArray *sa （管理対象のスロット配列へのポインタ）
  • **出力**: bool （すべての接続で結果の消費が成功した場合は true、失敗した場合は false）

前提条件と不変条件

  • sa は有効な ParallelSlotArray を指している必要があります。
  • 接続が確立されているスロットに対しては、consumeQueryResult が呼び出されます。

アルゴリズムまたは振る舞い

1. sa->numslots 分だけループし、各スロットを順次走査します。 2. 接続が存在しないスロットはスキップします。 3. 接続が存在する場合、consumeQueryResult を呼び出して結果を消費します。 4. consumeQueryResult が false を返した場合、直ちに処理を中断し false を返します。 5. 成功した場合、当該スロットの inUse フラグを false に設定し、ParallelSlotClearHandler を呼び出してハンドラをクリアします。 6. すべてのスロットの処理が正常に完了した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c:808
  • src/bin/scripts/reindexdb.c:471
  • src/bin/scripts/vacuumdb.c:746
  • src/bin/scripts/vacuumdb.c:767
  • **呼び出し先**:
  • consumeQueryResult （内部関数）
  • ParallelSlotClearHandler （マクロまたはインライン関数）

関連データ構造

  • ParallelSlotArray: 並列接続を管理する構造体。
  • ParallelSlot: 個別の接続状態を保持する構造体。

失敗モードと境界条件

  • consumeQueryResult が失敗した場合、本関数も即座に失敗として終了します。
  • 接続が NULL のスロットは無視されるため、未接続状態のスロットが混在していても安全に動作します。

保守担当者向けメモ

  • 本関数は同期的に全スロットの完了を待機します。
  • consumeQueryResult の戻り値が false の場合、エラー処理は呼び出し元に委ねられます。
  • 呼び出し先である consumeQueryResult の内部実装（PQgetResult のループ等）が、本関数の実行時間に直接影響します。
