pgxml_xpath

定義

  • 種別: function
  • 定義箇所: contrib/xml2/xpath.c:367

目的

XMLドキュメントに対してXPath式をコンパイルおよび評価し、その結果をxmlXPathObjectPtrとして取得するメインルーチンです。

シグネチャまたは形状

static xmlXPathObjectPtr pgxml_xpath(text *document, xmlChar *xpath, xpath_workspace *workspace)

入力と出力

  • **入力**:
  • document: 評価対象のXMLドキュメント（PostgreSQLのtext型）。
  • xpath: 実行するXPath式（xmlChar型）。
  • workspace: 評価に必要なコンテキストやドキュメントツリーを保持するxpath_workspace構造体。
  • **出力**:
  • 評価結果を格納したxmlXPathObjectPtr。失敗時や結果が空の場合はNULLを返す。

前提条件と不変条件

  • workspace構造体の各ポインタ（doctree, ctxt, res）は、関数内で初期化されるため、呼び出し側で事前にクリアされている必要はありません。
  • libxml2のパーサー初期化が成功している必要があります。

アルゴリズムまたは振る舞い

1. pgxml_parser_initを呼び出し、エラーハンドリングコンテキストを準備します。 2. PG_TRYブロック内で以下を実行します。

  • xmlReadMemoryによりXML文字列をDOMツリーに変換します。
  • xmlXPathNewContextでXPath評価コンテキストを作成し、ルートノードを設定します。
  • xmlXPathCtxtCompileでXPath式をコンパイルします。コンパイル失敗時はERRORをスローします。
  • xmlXPathCompiledEvalでXPathを評価し、結果をworkspace->resに格納します。
  • コンパイル済み式を解放します。

    • 3. PG_CATCHブロックでは、エラー発生時にcleanup_workspaceを呼び出してリソースを解放し、エラーを再スローします。 4. 正常終了時、結果がNULLであればcleanup_workspaceを実行し、最後にpg_xml_doneでエラーコンテキストをクローズして結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/xml2/xpath.c内のxpath_nodeset, xpath_list, xpath_string, xpath_number, xpath_boolから呼び出されています。
  • **呼び出し先**:
  • pgxml_parser_init, xmlReadMemory, xmlXPathNewContext, xmlDocGetRootElement, xmlXPathCtxtCompile, xml_ereport, xmlXPathCompiledEval, xmlXPathFreeCompExpr, cleanup_workspace, pg_xml_done。

関連データ構造

  • xpath_workspace: 評価中のドキュメントツリー、XPathコンテキスト、評価結果を保持する管理構造体。

失敗モードと境界条件

  • XMLのパース失敗やXPathの構文エラーが発生した場合、PG_CATCHブロックによりPostgreSQLのERRORとして処理されます。
  • workspace->resがNULLの場合、メモリリークを防ぐためにcleanup_workspaceが呼び出されます。

保守担当者向けメモ

  • libxml2のメモリ管理はPostgreSQLのメモリコンテキストと独立しているため、エラー発生時や関数終了時のcleanup_workspace呼び出しによるリソース解放が必須です。
  • PG_TRY/CATCHブロックは、libxml2の操作中に発生する可能性のあるエラーをPostgreSQLのトランザクション管理に統合するために使用されています。
