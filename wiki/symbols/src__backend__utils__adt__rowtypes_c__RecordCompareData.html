RecordCompareData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/rowtypes.c:59

目的

複合型（Composite Types）や匿名レコードの比較処理において、実行時の型情報および各フィールドの比較関数情報をキャッシュするために使用されます。

シグネチャまたは形状

typedef struct RecordCompareData
{
    int         ncolumns;       /* columns[] の確保済みサイズ */
    Oid         record1_type;
    int32       record1_typmod;
    Oid         record2_type;
    int32       record2_typmod;
    ColumnCompareData columns[FLEXIBLE_ARRAY_MEMBER];
} RecordCompareData;

入力と出力

  • 本構造体自体は、関数呼び出しのライフサイクル内で fcinfo->flinfo->fn_extra に格納されるキャッシュデータとして機能します。
  • 入力として、比較対象となる2つのレコードの型（Oid）と型修飾子（typmod）を保持します。
  • 出力として、各列の比較に必要な情報を columns 配列に保持します。

前提条件と不変条件

  • columns フィールドは FLEXIBLE_ARRAY_MEMBER を使用しており、構造体の末尾に可変長の配列として配置されます。
  • fn_extra を介してキャッシュされるため、型定義が変更された場合（ALTER TYPE 等）の生存期間管理に注意が必要です。

アルゴリズムまたは振る舞い

  • 比較関数（record_cmp 等）が呼び出された際、fn_extra に本構造体が存在するか確認します。
  • 存在しない場合は、対象レコードの型定義（TupleDesc）をルックアップし、各フィールドの比較関数を決定して本構造体を構築・キャッシュします。
  • 比較実行時は、キャッシュされた columns 配列を順次走査し、各フィールドの比較を再帰的に行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/rowtypes.c 内の record_cmp、record_eq、record_ne、record_lt 等の比較関数が、fcinfo->flinfo->fn_extra を通じて本構造体を参照します。
  • **呼び出し先**: 比較実行時に columns 内に保持された各フィールドの比較関数を呼び出します。

関連データ構造

  • ColumnCompareData: 各列の比較ロジックを保持する構造体。本構造体の columns 配列の要素として使用されます。
  • TupleDesc: レコードの構造定義。本構造体の構築時に参照されます。

失敗モードと境界条件

  • レコード内にレコードが含まれる場合、再帰的な処理が発生するため、スタックオーバーフローを防ぐための check_stack_depth() が併用されます。
  • 削除された列（attisdropped）は比較対象からスキップされます。

保守担当者向けメモ

  • 本構造体は offsetof(RecordCompareData, columns) を用いて動的にメモリ確保されるため、構造体のサイズ計算時には注意してください。
  • キャッシュの無効化タイミングは fn_extra の生存期間に依存します。型定義の変更が反映されない可能性があるため、キャッシュの再構築ロジックが適切に機能しているか確認してください。
