ValidateInputLSNs

定義

  • 種別: function
  • 定義箇所: contrib/pg_walinspect/pg_walinspect.c:515

目的

WAL解析関数において、ユーザーから指定された開始LSNおよび終了LSNが妥当な範囲内にあるかを検証し、必要に応じて終了LSNを現在のシステムLSNに合わせて調整します。

シグネチャまたは形状

static void ValidateInputLSNs(XLogRecPtr start_lsn, XLogRecPtr *end_lsn)

入力と出力

  • **入力**:
  • start_lsn: 解析を開始するWALのLSN。
  • end_lsn: 解析を終了するWALのLSN（ポインタ経由で受け取り、必要に応じて更新される）。
  • **出力**: なし（戻り値は void）。ただし、検証失敗時には ereport(ERROR, ...) を通じて例外を発生させます。

前提条件と不変条件

  • start_lsn は現在のシステムLSN（GetCurrentLSN()）以下である必要があります。
  • start_lsn は end_lsn 以下である必要があります。
  • end_lsn が現在のシステムLSNを超えている場合、自動的に現在のシステムLSNに切り詰められます。

アルゴリズムまたは振る舞い

1. GetCurrentLSN() を呼び出し、現在のシステムWALの末尾位置を取得します。 2. start_lsn が現在のシステムLSNより大きい場合、エラーを報告します。 3. start_lsn が end_lsn より大きい場合、エラーを報告します。 4. end_lsn が現在のシステムLSNより大きい場合、end_lsn の値を現在のシステムLSNで上書きします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_walinspect/pg_walinspect.c 内の各解析関数（pg_get_wal_records_info 等）から呼び出されています。
  • **呼び出し先**:
  • GetCurrentLSN()

関連データ構造

  • XLogRecPtr: WALの論理位置（LSN）を表す型。

失敗モードと境界条件

  • start_lsn > current_lsn: ERRCODE_INVALID_PARAMETER_VALUE でエラー終了。
  • start_lsn > end_lsn: ERRCODE_INVALID_PARAMETER_VALUE でエラー終了。
  • end_lsn > current_lsn: エラーにはならず、end_lsn が current_lsn に修正されます。

保守担当者向けメモ

  • 本関数は pg_walinspect モジュール内の複数のエントリポイントで共通して使用される検証ロジックです。
  • 修正を行う際は、本関数が ereport(ERROR, ...) を発行して処理を中断させる点に注意してください。
