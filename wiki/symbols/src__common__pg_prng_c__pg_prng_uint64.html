pg_prng_uint64

定義

  • 種別: function
  • 定義箇所: src/common/pg_prng.c:133

目的

pg_prng_state 構造体で管理される擬似乱数生成器を用いて、64ビットの符号なし整数を生成します。

シグネチャまたは形状

uint64 pg_prng_uint64(pg_prng_state *state)

入力と出力

  • **入力**: state (擬似乱数生成器の状態を保持する pg_prng_state 構造体へのポインタ)
  • **出力**: 64ビットの符号なし整数 (uint64)

前提条件と不変条件

  • state は、事前に pg_prng_seed() 等で適切に初期化されている必要があります。
  • 内部状態がすべて0である場合、生成器は0を返し続けます。この状態を回避するため、初期化時に pg_prng_seed_check() を用いた検証が推奨されます。

アルゴリズムまたは振る舞い

内部的に xoroshiro128ss() 関数を呼び出し、現在の状態を更新した上で、生成された64ビットの乱数値を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: xoroshiro128ss
  • **呼び出し元（直接的な証拠）**:
  • contrib/amcheck/verify_nbtree.c
  • src/backend/lib/bloomfilter.c
  • src/bin/pgbench/pgbench.c
  • src/test/modules/test_radixtree/test_radixtree.c

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • 本関数自体はエラーを返しません。
  • 暗号学的な強度は持たないため、パスワードや暗号鍵の生成には使用できません。

保守担当者向けメモ

  • 本関数は xoroshiro128** 1.0 アルゴリズムのラッパーです。
  • 範囲指定が必要な場合は、本関数を直接使用するのではなく、リジェクションサンプリングを実装した pg_prng_uint64_range() の利用を検討してください。
