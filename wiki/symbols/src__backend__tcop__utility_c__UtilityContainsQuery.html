UtilityContainsQuery

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/utility.c:2178

目的

ユーティリティ文の解析ツリーを再帰的に探索し、その内部に含まれる実質的な Query 構造体を取得する。

シグネチャまたは形状

Query *UtilityContainsQuery(Node *parsetree)

入力と出力

  • **入力**: Node *parsetree（解析対象のユーティリティ文ノード）
  • **出力**: 内部に含まれる Query 構造体へのポインタ。該当するものがない場合は NULL を返す。

前提条件と不変条件

  • 入力された parsetree が T_DeclareCursorStmt、T_ExplainStmt、または T_CreateTableAsStmt のいずれかである場合、その内部の query フィールドが有効な Query 構造体であることを前提とする。

アルゴリズムまたは振る舞い

1. nodeTag(parsetree) を使用してノードの種類を判定する。 2. T_DeclareCursorStmt、T_ExplainStmt、T_CreateTableAsStmt のいずれかであれば、そのノードが保持する query フィールドを取得する。 3. 取得した Query の commandType が CMD_UTILITY である場合、その utilityStmt に対して本関数を再帰的に呼び出す。 4. commandType が CMD_UTILITY でない場合、その Query をそのまま返す。 5. 上記以外のノードタイプの場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/setrefs.c:3706
  • src/backend/utils/cache/plancache.c:1905
  • src/backend/utils/cache/plancache.c:1954
  • **呼び出し先**:
  • 本関数自身（再帰呼び出し）

関連データ構造

  • Query: 実行対象のクエリを保持する構造体。
  • DeclareCursorStmt, ExplainStmt, CreateTableAsStmt: 内部に Query を保持する可能性のあるユーティリティ文ノード。

失敗モードと境界条件

  • 入力が NULL の場合、nodeTag の呼び出しによりクラッシュする可能性がある。
  • 探索対象外のノードタイプが渡された場合は NULL を返す。

保守担当者向けメモ

  • 新たに内部に Query を保持するユーティリティ文が追加された場合は、本関数の switch 文に新しい case を追加する必要がある。
