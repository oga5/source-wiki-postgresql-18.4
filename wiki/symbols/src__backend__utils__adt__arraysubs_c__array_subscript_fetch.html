array_subscript_fetch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arraysubs.c:236

目的

配列型から特定の要素を抽出し、式評価の結果として格納する。

シグネチャまたは形状

static void array_subscript_fetch(ExprState *state, ExprEvalStep *op, ExprContext *econtext)

入力と出力

  • **入力**:
  • op->d.sbsref.state: 添字操作の状態を保持する SubscriptingRefState。
  • op->resvalue: 抽出対象の配列（入力）および抽出された要素（出力）。
  • op->resnull: 抽出結果が NULL かどうかを示すフラグ。
  • **出力**: なし（op->resvalue および op->resnull を直接更新）。

前提条件と不変条件

  • op->resnull が指す値が true でないこと（呼び出し前に配列または添字が NULL でないことが保証されている必要がある）。
  • sbsrefstate->workspace が ArraySubWorkspace 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. op->d.sbsref.state から ArraySubWorkspace を取得する。 2. array_get_element を呼び出し、配列から指定された添字の要素を取得する。

  • この際、ワークスペース内の upperindex（添字）、refattrlength（属性長）、refelemlength（要素長）、refelembyval（値渡しフラグ）、refelemalign（アライメント）を使用する。

    • 3. 取得した値を op->resvalue に格納し、NULL 状態を op->resnull に反映する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/arraysubs.c:525（methods->sbs_fetch への代入を通じて実行エンジンから呼び出される）。
  • **呼び出し先**:
  • array_get_element（utils/array.h で定義される配列要素取得関数）。

関連データ構造

  • ArraySubWorkspace: 添字の境界値や要素の型情報を保持する構造体。
  • SubscriptingRefState: 実行時の添字操作の状態を管理するノード。
  • ExprEvalStep: 式評価のステップを定義する構造体。

失敗モードと境界条件

  • 配列が NULL の場合、または添字が範囲外の場合の挙動は array_get_element の実装に依存する。
  • 本関数内では Assert(!(*op->resnull)) により、入力が NULL であることを許容しない。

保守担当者向けメモ

  • 本関数は SubscriptingRef ノードの実行ステップとして登録される。
  • raw_array_subscript_handler と標準の array_subscript_handler の両方で利用される可能性があるため、変更時は両方のコンテキストへの影響を考慮すること。
