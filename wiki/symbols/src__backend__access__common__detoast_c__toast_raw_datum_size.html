toast_raw_datum_size

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/common/detoast.c:544

目的

TOASTされたデータや圧縮・間接参照・ショートヘッダー形式など、様々な形態で格納されている varlena データの「復元後の論理的なサイズ（ヘッダーを含む）」を算出します。

シグネチャまたは形状

Size toast_raw_datum_size(Datum value);

入力と出力

  • **入力**: Datum value（復元対象の varlena 構造体へのポインタを含む Datum）
  • **出力**: Size（復元後のデータ全体のバイトサイズ）

前提条件と不変条件

  • 入力 value は有効な varlena 構造体を指している必要があります。
  • 間接参照（VARATT_IS_EXTERNAL_INDIRECT）の場合、ネストされた間接参照は許可されません（Assert により検証されます）。

アルゴリズムまたは振る舞い

入力データの形式を判定し、以下のルールに従ってサイズを計算します。 1. **外部データ（On-disk）**: TOASTポインタ内の va_rawsize を返します。 2. **間接参照**: 参照先のデータに対して再帰的に本関数を呼び出します。 3. **展開済みオブジェクト**: EOH_get_flat_size を使用してサイズを取得します。 4. **圧縮データ**: 圧縮後のペイロードサイズに標準ヘッダーサイズ（VARHDRSZ）を加算します。 5. **ショートヘッダー**: ショートヘッダーのサイズからショートヘッダー分を減じ、標準ヘッダー分を加算して正規化します。 6. **通常データ**: VARSIZE マクロによりサイズを取得します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c
  • src/backend/statistics/extended_stats.c
  • src/backend/utils/adt/array_typanalyze.c
  • src/backend/utils/adt/datum.c
  • src/backend/utils/adt/rowtypes.c
  • **呼び出し先**:
  • toast_raw_datum_size（間接参照時の再帰呼び出し）
  • EOH_get_flat_size（展開済みオブジェクトの場合）

関連データ構造

  • varlena: 可変長データの基本構造体。
  • varatt_external: 外部ストレージ上のデータポインタ。
  • varatt_indirect: 間接参照用ポインタ。

失敗モードと境界条件

  • ネストされた間接参照が渡された場合、Assert が有効なビルドではアボートします。
  • 不正なポインタや破損した varlena ヘッダーが渡された場合の動作は未定義です。

保守担当者向けメモ

  • 本関数は、呼び出し元がデータのサイズを把握してメモリ確保を行うための基準値を提供します。
  • 圧縮データやショートヘッダーの計算において、ヘッダーサイズの正規化（VARHDRSZ への変換）が重要です。
