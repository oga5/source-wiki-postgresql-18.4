get_major_server_version

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/server.c:160

目的

指定されたデータベースクラスターのデータディレクトリにある PG_VERSION ファイルを読み込み、そのメジャーバージョンを数値（uint32）として抽出する。

シグネチャまたは形状

uint32 get_major_server_version(ClusterInfo *cluster);

入力と出力

  • **入力**: ClusterInfo *cluster （対象クラスターの情報を保持する構造体。pgdata メンバが参照される）
  • **出力**: uint32 （抽出されたメジャーバージョン値）

前提条件と不変条件

  • cluster->pgdata に有効なパスが格納されていること。
  • cluster->pgdata 配下に PG_VERSION ファイルが存在し、読み取り可能であること。
  • PG_VERSION ファイルの内容が X.Y 形式（例: 9.6.1 や 10.1）の文字列であること。

アルゴリズムまたは振る舞い

1. cluster->pgdata を基に PG_VERSION ファイルのパスを構築する。 2. ファイルを開き、内容を cluster->major_version_str に読み込む。 3. 文字列を v1.v2 としてパースする。 4. バージョン番号に基づき、以下の計算式で数値を算出する:

  • v1 < 10 の場合（旧形式）: v1 * 10000 + v2 * 100
  • v1 >= 10 の場合（新形式）: v1 * 10000

    • 5. ファイルを閉じて結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/exec.c:346 にて呼び出されている。
  • **呼び出し先**: fopen, fscanf, sscanf, fclose などの標準ライブラリ関数、およびエラー発生時にプログラムを終了させる pg_fatal を呼び出す。

関連データ構造

  • ClusterInfo: pg_upgrade.h で定義。pgdata（パス）および major_version_str（バージョン文字列の格納先）を含む。

失敗モードと境界条件

  • PG_VERSION ファイルが開けない場合、pg_fatal によりエラーメッセージを出力してプロセスが終了する。
  • ファイルの内容がパースできない場合（fscanf または sscanf が失敗した場合）、pg_fatal によりプロセスが終了する。

保守担当者向けメモ

  • 本関数は pg_upgrade におけるバージョン判定の基盤であり、PostgreSQL 10 を境にしたバージョン表記の変化を吸収するロジックを含んでいる。
  • cluster->major_version_str は呼び出し側の構造体メンバを更新するため、副作用として状態が変更される点に注意すること。
