inv_tell

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:436

目的

オープンされているラージオブジェクト（LargeObjectDesc）の現在の読み書きオフセット位置を取得する。

シグネチャまたは形状

int64 inv_tell(LargeObjectDesc *obj_desc)

入力と出力

  • **入力**: obj_desc (オープンされたラージオブジェクトの状態を保持する構造体へのポインタ)
  • **出力**: int64 型の現在のオフセット値

前提条件と不変条件

  • obj_desc は有効なポインタであること（Assertにより検証）。
  • 本関数は読み取りまたは書き込みの権限チェックを行わない（呼び出し元で既に保証されていることを前提とする）。

アルゴリズムまたは振る舞い

1. obj_desc が有効なポインタであることを確認する。 2. obj_desc 構造体が保持する現在の offset メンバの値をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-fsstubs.c:285
  • src/backend/libpq/be-fsstubs.c:308
  • **呼び出し先**:
  • 特になし（構造体のメンバアクセスのみ）。

関連データ構造

  • LargeObjectDesc: ラージオブジェクトのID、オフセット、フラグ、スナップショットを管理する構造体。

失敗モードと境界条件

  • obj_desc が NULL または無効なポインタである場合、アサーション違反により異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は単なるアクセサであり、副作用を持たない。
  • 権限チェックが不要である理由は、ラージオブジェクトのオープン時に既にアクセス権が検証されているためである。
