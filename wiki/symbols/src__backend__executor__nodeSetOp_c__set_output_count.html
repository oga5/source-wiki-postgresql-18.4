set_output_count

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSetOp.c:119

目的

SQL の集合演算（INTERSECT / EXCEPT）において、現在のグループの左側および右側の出現回数に基づき、出力すべき行数を決定する。

シグネチャまたは形状

static void set_output_count(SetOpState *setopstate, SetOpStatePerGroup pergroup)

入力と出力

  • **入力**:
  • setopstate: 実行状態を保持する SetOpState 構造体。
  • pergroup: 現在のグループにおける左側入力と右側の出現回数（numLeft, numRight）を保持する SetOpStatePerGroup 構造体。
  • **出力**:
  • なし（副作用として setopstate->numOutput を更新する）。

前提条件と不変条件

  • setopstate->ps.plan が SetOp ノードとして正しく初期化されていること。
  • pergroup が現在のグループの集計結果を正しく保持していること。

アルゴリズムまたは振る舞い

plannode->cmd に基づき、以下のロジックで setopstate->numOutput を算出する。

  • **SETOPCMD_INTERSECT**: 両方の出現回数が 1 以上であれば 1 を出力、それ以外は 0。
  • **SETOPCMD_INTERSECT_ALL**: 左側と右側の出現回数のうち、小さい方の値を出力。
  • **SETOPCMD_EXCEPT**: 左側の出現回数が 1 以上かつ右側が 0 であれば 1 を出力、それ以外は 0。
  • **SETOPCMD_EXCEPT_ALL**: 左側の出現回数が右側より少なければ 0、そうでなければ差分（numLeft - numRight）を出力。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeSetOp.c 内の setop_retrieve_sorted および setop_retrieve_hash_table から呼び出される。
  • **呼び出し先**:
  • elog（エラー発生時）。

関連データ構造

  • SetOpState: 実行ノードの状態管理構造体。
  • SetOpStatePerGroup: グループごとの出現回数を保持する構造体。
  • SetOp: プランノード構造体（cmd フィールドを参照）。

失敗モードと境界条件

  • plannode->cmd が定義された集合演算のいずれにも該当しない場合、elog(ERROR, ...) により処理を中断する。

保守担当者向けメモ

  • 本関数は射影を行わず、あくまで「何行出力するか」というカウントのみを決定する。
  • UNION 演算はこのノードでは処理されないため、本関数で UNION 関連のコマンドを考慮する必要はない。
