WalSndSendDataCallback

定義

  • 種別: typedef
  • 定義箇所: src/backend/replication/walsender.c:255

目的

WAL 送信プロセスにおいて、WAL データの読み出しおよびクライアントへの送信処理を実行するためのコールバック関数の型定義です。

シグネチャまたは形状

typedef void (*WalSndSendDataCallback) (void);

入力と出力

  • 入力: なし
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • 本コールバックは、WalSndLoop 内でレプリケーションストリーミングのメインループの一部として呼び出されることを前提としています。
  • 物理レプリケーションの場合は XLogSendPhysical、論理レプリケーションの場合は XLogSendLogical がこの型に適合する関数として実装されています。

アルゴリズムまたは振る舞い

  • WalSndLoop 内で呼び出され、ディスク上の WAL を読み出し、pq_putmessage_noblock 等を用いてクライアントへ送信する一連の処理を実行します。
  • 呼び出し元は、この関数が実行されるたびに WAL のストリーミングが進行することを期待します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • WalSndLoop (src/backend/replication/walsender.c:256, 2806)
  • WalSndDone (src/backend/replication/walsender.c:262, 3521)
  • 呼び出し先:
  • 提供されたソース情報からは、具体的な実装関数（XLogSendPhysical や XLogSendLogical）がこの型として渡されることが示唆されています。

関連データ構造

  • WalSnd: WAL 送信プロセスの状態を管理する構造体。
  • WalSndCtl: 共有メモリ上の制御構造体。

失敗モードと境界条件

  • ネットワークエラーや WAL 読み出しエラーが発生した場合、コールバック内で適切なエラーハンドリングまたはプロセス終了処理が行われる必要があります。

保守担当者向けメモ

  • 本シンボルは関数ポインタ型です。新しいレプリケーション方式を追加する際は、このシグネチャに適合する送信関数を実装し、WalSndLoop に渡す必要があります。
