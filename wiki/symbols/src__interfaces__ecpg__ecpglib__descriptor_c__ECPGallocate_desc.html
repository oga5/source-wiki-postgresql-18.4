ECPGallocate_desc

定義

  • src/interfaces/ecpg/ecpglib/descriptor.c:799

目的

SQL 記述子（Descriptor）を新規に割り当て、スレッドローカルな記述子リストに登録する。

シグネチャまたは形状

bool ECPGallocate_desc(int line, const char *name)

入力と出力

  • **入力**:
  • line: 呼び出し元のソースコード行番号（エラー報告用）。
  • name: 割り当てる記述子の識別名。
  • **出力**: 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • ECPGget_sqlca() が有効な sqlca_t 構造体を返す必要がある。
  • 記述子名は一意である必要がある（既存の記述子リストとの重複チェックは本関数内では明示されていないが、set_descriptors による管理下に入る）。

アルゴリズムまたは振る舞い

1. ECPGget_sqlca() で SQLCA を取得し、初期化する。 2. descriptor 構造体用のメモリを確保する。 3. get_descriptors() を呼び出し、現在のスレッドの既存記述子リストを取得して new->next に設定する。 4. 記述子名（name）を格納するためのメモリを確保し、コピーする。 5. 記述子の初期状態を設定する（count = -1, items = NULL）。 6. PQmakeEmptyPGresult(NULL, 0) を呼び出し、記述子に関連付けられた空の PGresult を作成する。 7. set_descriptors(new) を呼び出し、新しい記述子をスレッドローカルなリストの先頭に登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPG プリプロセッサが生成するコード（例: compat_informix-describe.c, preproc-describe.c 等のテストコードで確認可能）。
  • **呼び出し先**:
  • ECPGget_sqlca, ecpg_init_sqlca, ecpg_alloc, ecpg_free, ecpg_raise
  • get_descriptors, set_descriptors
  • PQmakeEmptyPGresult (libpq)

関連データ構造

  • struct descriptor: 記述子のメタデータと PGresult を保持する構造体。

失敗モードと境界条件

  • sqlca が取得できない場合、ECPG_OUT_OF_MEMORY エラーを発生させ false を返す。
  • メモリ確保（ecpg_alloc）に失敗した場合、false を返す。
  • PQmakeEmptyPGresult が失敗した場合、確保したメモリを解放し、ECPG_OUT_OF_MEMORY エラーを発生させて false を返す。

保守担当者向けメモ

  • 本関数はスレッドローカルストレージを利用して記述子を管理している。
  • PGresult の初期化に PQmakeEmptyPGresult を使用しており、libpq のリソース管理に依存している。
  • name の重複チェックは本関数内で行われていないため、呼び出し側で管理されている前提である。
