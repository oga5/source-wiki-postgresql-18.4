parse_relfilenumber

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/relfilenumber.c:331

目的

ファイル名文字列からリレーションファイル番号（RelFileNumber）を解析・抽出します。

シグネチャまたは形状

static RelFileNumber parse_relfilenumber(const char *filename)

入力と出力

  • **入力**: filename (解析対象のファイル名文字列)
  • **出力**: 成功時は RelFileNumber 型の数値、失敗時は InvalidRelFileNumber を返します。

前提条件と不変条件

  • ファイル名の先頭文字が '1' から '9' の範囲内である必要があります。
  • 数値は PG_UINT32_MAX を超えてはなりません。

アルゴリズムまたは振る舞い

1. ファイル名の先頭文字が '1'〜'9' の範囲外であれば、直ちに InvalidRelFileNumber を返します。 2. strtoul を使用して文字列を 10 進数として変換します。 3. 以下のいずれかに該当する場合、InvalidRelFileNumber を返します。

  • 変換中にエラーが発生した場合（errno が設定された場合）。
  • 文字列が空、または数値として解釈できなかった場合（filename == endp）。
  • 変換結果が 0 以下、または PG_UINT32_MAX を超える場合。

    • 4. 上記の条件をすべて満たす場合、変換された値を RelFileNumber 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/relfilenumber.c 内のディレクトリ走査処理（de->d_name を引数として渡す箇所）から呼び出されています。
  • **呼び出し先**: 標準ライブラリ関数 strtoul を呼び出します。

関連データ構造

  • RelFileNumber: 返り値の型。
  • InvalidRelFileNumber: 失敗時に返される定数。

失敗モードと境界条件

  • 先頭が '0' で始まるファイル名や、数値以外の文字で始まるファイル名は無効とみなされます。
  • 変換後の値が 0 以下、または PG_UINT32_MAX を超える場合は境界外として扱われます。

保守担当者向けメモ

  • 本関数はファイルシステム上のエントリ名からリレーション番号を安全に抽出するために使用されます。
  • strtoul のエラーチェックおよび範囲チェックが厳密に行われているため、不正なファイル名が混入してもクラッシュしない設計になっています。
