hemdist

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_gist.c:130

目的

2つの TSQuerySign（シグネチャ）間のハミング距離を計算します。この値は、GiST インデックスにおけるペナルティ計算や、インデックスページ分割時のコスト評価の基準として使用されます。

シグネチャまたは形状

static int hemdist(TSQuerySign a, TSQuerySign b)

入力と出力

  • **入力**: 比較対象となる2つの TSQuerySign 型の値 a および b。
  • **出力**: 両者のビット表現の差分（ハミング距離）を表す int 型の値。

前提条件と不変条件

  • TSQuerySign 型はビットマップとして表現されている必要があります。
  • sizebitvec 関数が、ビットマップ内のセットされたビット数を正確にカウントできることが前提です。

アルゴリズムまたは振る舞い

1. 引数 a と b に対してビット単位の排他的論理和（XOR: ^）をとり、異なるビット位置を抽出します。 2. 得られた結果に対して sizebitvec を呼び出し、セットされているビットの総数をカウントして返します。

呼び出し元と呼び出し先

  • **呼び出し先**: sizebitvec
  • **呼び出し元**:
  • 本ファイル（tsquery_gist.c）内での直接的な呼び出し元はソースコード上では明示されていませんが、GiST インデックスのペナルティ計算や分割ロジックで利用されます。
  • ツールによるヒューリスティックな調査では、contrib/hstore/hstore_gist.c や contrib/intarray/_intbig_gist.c において同名の関数が呼び出されていますが、これらは引数構成が異なる（siglen を含む）ため、本関数とは別の実装である可能性が高いです。

関連データ構造

  • TSQuerySign: tsquery を圧縮したビットマップ表現。

失敗モードと境界条件

  • 入力されるシグネチャのサイズが一致しない場合、ビット演算の結果が意図しないものになる可能性があります。
  • sizebitvec が内部で扱うビット長を超えた入力に対する挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は tsquery_gist.c 内で静的に定義されています。
  • 他の拡張モジュール（hstore や intarray）にも同名の関数が存在しますが、シグネチャが異なるため混同しないよう注意してください。
  • インデックスの検索効率に直結するヒューリスティックな計算であるため、変更時は GiST インデックスの再構築コストや検索性能への影響を評価してください。
