get_descriptors

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/descriptor.c:39

目的

現在のスレッドに紐付けられた SQL 記述子のリスト（struct descriptor）を取得します。

シグネチャまたは形状

static struct descriptor *get_descriptors(void)

入力と出力

  • 入力: なし
  • 出力: 現在のスレッドに関連付けられた descriptor 構造体の先頭ポインタを返します。

前提条件と不変条件

  • pthread_once を使用して、スレッドローカルストレージのキー（descriptor_key）が初期化済みであることを保証します。
  • スレッドごとに独立した記述子リストが管理されます。

アルゴリズムまたは振る舞い

1. pthread_once を呼び出し、descriptor_key_init を実行してスレッドキーの初期化を確実に行います。 2. pthread_getspecific を使用して、現在のスレッドに格納されている descriptor リストのポインタを取得し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/ecpglib/descriptor.c 内の記述子操作関数（リストの走査や新規追加時）。
  • 呼び出し先:
  • pthread_once
  • descriptor_key_init
  • pthread_getspecific

関連データ構造

  • struct descriptor: 記述子の情報を保持する構造体。
  • descriptor_key: スレッドローカルストレージを識別するための pthread_key_t 型のキー。

失敗モードと境界条件

  • スレッドキーが初期化されていない場合、pthread_once により初期化が試行されます。
  • 現在のスレッドに記述子が割り当てられていない場合、pthread_getspecific は NULL を返します。

保守担当者向けメモ

  • 本関数はスレッドセーフな記述子管理の基盤です。リストの走査を行う際は、戻り値が NULL になる可能性を考慮してください。
  • pthread 関連の依存関係があるため、プラットフォーム固有の挙動（Win32 互換層など）に注意してください。
