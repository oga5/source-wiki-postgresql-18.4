ecpg_init

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/misc.c:72

目的

ECPGライブラリの操作を開始する前に、スレッド固有のSQLCA（SQL Communication Area）を初期化し、指定されたデータベース接続が有効であることを検証します。

シグネチャまたは形状

bool ecpg_init(const struct connection *con, const char *connection_name, const int lineno)

入力と出力

  • **入力**:
  • con: 接続状態を保持する connection 構造体へのポインタ。
  • connection_name: 接続名（エラーメッセージ用）。
  • lineno: 呼び出し元のソースコード行番号（エラー報告用）。
  • **出力**:
  • bool: 初期化および接続確認が成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • ECPGget_sqlca() が呼び出し可能な状態であること。
  • 戻り値が false の場合、内部で ecpg_raise が呼び出され、適切なエラー状態が設定されています。

アルゴリズムまたは振る舞い

1. ECPGget_sqlca() を呼び出し、現在のスレッドに対応する sqlca_t 構造体を取得します。取得失敗時は ECPG_OUT_OF_MEMORY エラーを発生させ false を返します。 2. ecpg_init_sqlca(sqlca) を呼び出し、取得したSQLCAを初期化します。 3. 引数 con が NULL かどうかを確認します。NULL の場合は ECPG_NO_CONN エラーを発生させ false を返します。 4. 上記のチェックをすべて通過した場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/connect.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • src/interfaces/ecpg/ecpglib/misc.c
  • src/interfaces/ecpg/ecpglib/prepare.c
  • **呼び出し先**:
  • ECPGget_sqlca()
  • ecpg_init_sqlca()
  • ecpg_raise()
  • ecpg_gettext()

関連データ構造

  • struct connection: データベース接続情報を保持する構造体。
  • struct sqlca_t: SQL通信領域を定義する構造体。

失敗モードと境界条件

  • **メモリ不足**: ECPGget_sqlca() が NULL を返した場合、ECPG_OUT_OF_MEMORY エラーとなります。
  • **接続未確立**: con が NULL の場合、ECPG_NO_CONN エラーとなります。

保守担当者向けメモ

  • 本関数は、ECPGの各操作（SQL実行、準備、接続管理など）の冒頭でガード節として利用されています。
  • ecpg_init_sqlca によるSQLCAの初期化と、接続の存在確認という2つの責務を担っています。変更を加える際は、これら両方の要件に影響がないか確認してください。
