errdatatype

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/domains.c:406

目的

エラー報告時に、対象となるデータ型のスキーマ名と型名を診断情報（PG_DIAG_SCHEMA_NAME および PG_DIAG_DATATYPE_NAME）として設定します。

シグネチャまたは形状

int errdatatype(Oid datatypeOid)

入力と出力

  • **入力**: datatypeOid (対象データ型のOID)
  • **出力**: 0 (戻り値は使用されません)

前提条件と不変条件

  • 指定された datatypeOid がシステムカタログ pg_type に存在する必要があります。存在しない場合、elog(ERROR) により処理が中断されます。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して TYPEOID キャッシュから指定されたOIDの型情報を取得します。 2. GETSTRUCT で取得した Form_pg_type 構造体から、スキーマ名と型名を取り出します。 3. err_generic_string を呼び出し、エラーコンテキストにスキーマ名と型名を付与します。 4. ReleaseSysCache を呼び出してキャッシュエントリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c:4486
  • src/backend/utils/adt/domains.c:160
  • src/backend/utils/adt/domains.c:433
  • **呼び出し先**:
  • SearchSysCache1
  • HeapTupleIsValid
  • GETSTRUCT
  • err_generic_string
  • get_namespace_name
  • NameStr
  • ReleaseSysCache

関連データ構造

  • Form_pg_type: pg_type システムカタログの行構造体。

失敗モードと境界条件

  • datatypeOid に対応する型がカタログに見つからない場合、elog(ERROR) が発生し、呼び出し元へは戻りません。

保守担当者向けメモ

  • 本関数はエラー報告用であり、直接的なデータ処理は行いません。
  • err_generic_string を介してエラーメッセージのメタデータを拡張するため、主に ereport 呼び出しの直前で使用されることを想定しています。
