gbt_oidkey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_oid.c:54

目的

GiSTインデックス構築時のソート処理において、oidKEY 構造体で表現された範囲（lower, upper）を比較し、順序を決定する。

シグネチャまたは形状

int gbt_oidkey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部の t メンバが oidKEY としてキャストされる。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • 0: a と b が等しい場合。
  • 1: a が b より大きい場合。
  • -1: a が b より小さい場合。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体への有効なポインタであること。
  • oidKEY 構造体は lower および upper フィールドを持つこと。

アルゴリズムまたは振る舞い

1. a と b からそれぞれ oidKEY を抽出する。 2. lower 値を比較する。

  • ia->lower != ib->lower の場合、その大小関係を返す。

    • 3. lower 値が等しい場合、upper 値を比較する。

  • ia->upper == ib->upper ならば 0 を返す。
  • ia->upper > ib->upper ならば 1 を返し、それ以外は -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_oid.c 内のソートサポート設定箇所（94行目付近）から参照されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • oidKEY: インデックスノードに格納される Oid の範囲を保持する構造体。
  • Nsrt: ソート処理用の中間構造体。

失敗モードと境界条件

  • a または b が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • Oid の値が比較範囲を超えるような特殊なケースは、Oid 型の定義に依存する。

保守担当者向けメモ

  • 本関数は gbt_oid_sortsupport を通じてソート時の比較ロジックとして利用される。
  • 比較ロジックは lower を優先し、同値の場合に upper を評価する辞書順に近い順序付けを行う。
