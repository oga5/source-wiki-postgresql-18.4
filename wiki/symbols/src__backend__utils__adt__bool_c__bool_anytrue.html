bool_anytrue

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:397

目的

集約関数 ANY (論理和) の最終計算を行い、蓄積された状態に基づいて結果を返します。

シグネチャまたは形状

Datum bool_anytrue(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して BoolAggState 構造体へのポインタを受け取ります。
  • **出力**:
  • 状態が NULL または非 NULL 値が一度も処理されていない場合は NULL を返します。
  • 非 NULL 値の中に true が一つでも存在する場合は true を返します。
  • すべての非 NULL 値が false である場合は false を返します。

前提条件と不変条件

  • 入力引数が BoolAggState 構造体へのポインタであることを前提とします。
  • BoolAggState は bool_accum 等によって適切に初期化・更新されている必要があります。

アルゴリズムまたは振る舞い

1. 第1引数が NULL の場合は NULL を返します。 2. state->aggcount が 0（非 NULL 値が未入力）の場合、NULL を返します。 3. state->aggtrue（真と判定された回数）が 0 より大きいかを確認し、その結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL の集約関数実行エンジンから呼び出されます（具体的な呼び出し元コードの詳細は提供されていません）。
  • **呼び出し先**: PostgreSQL の関数呼び出し規約マクロ（PG_ARGISNULL, PG_GETARG_POINTER, PG_RETURN_NULL, PG_RETURN_BOOL）を使用します。

関連データ構造

  • BoolAggState: 集約計算中の状態（aggcount や aggtrue を保持）を管理する構造体。

失敗モードと境界条件

  • 入力が NULL の場合、安全に NULL を返します。
  • 非 NULL 値が一度も入力されていない場合、aggcount == 0 となり NULL を返します。

保守担当者向けメモ

  • 本関数は bool_accum と対になる最終計算関数です。状態構造体の定義に変更を加える際は、aggcount および aggtrue の整合性に注意してください。
