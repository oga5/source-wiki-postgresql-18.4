PageIsVerified

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/bufpage.c:93

目的

読み込まれたページが破損していないかを、チェックサム検証およびページヘッダーの整合性チェックを通じて判定します。

シグネチャまたは形状

bool PageIsVerified(PageData *page, BlockNumber blkno, int flags, bool *checksum_failure_p)

入力と出力

  • **入力**:
  • page: 検証対象のページデータへのポインタ。
  • blkno: ページが属するブロック番号（チェックサム計算に使用）。
  • flags: 動作制御フラグ（PIV_LOG_WARNING, PIV_LOG_LOG, PIV_IGNORE_CHECKSUM_FAILURE 等）。
  • checksum_failure_p: チェックサム失敗の有無を格納する出力用ポインタ。
  • **出力**:
  • ページが有効（または全ゼロで許容される状態）であれば true、それ以外は false を返します。

前提条件と不変条件

  • ページが PageIsNew でない場合、チェックサムが有効であれば pd_checksum と計算値が一致する必要があります。
  • ページヘッダーの整合性として、pd_lower <= pd_upper <= pd_special <= BLCKSZ が成立し、pd_special が MAXALIGN に適合している必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: checksum_failure_p が指定されていれば false に初期化します。 2. **新規ページ判定**: PageIsNew であれば検証をスキップします。 3. **チェックサム検証**: データチェックサムが有効な場合、pg_checksum_page で計算した値とヘッダーの pd_checksum を比較します。不一致なら checksum_failure を記録します。 4. **ヘッダー整合性チェック**: ヘッダーの各オフセット値が論理的に矛盾していないかを確認します。 5. **全ゼロ判定**: チェックサムやヘッダーの検証に失敗しても、ページ全体がゼロ埋めされている場合は有効とみなします。 6. **ログ出力と終了**: チェックサム失敗時は flags に応じて ereport を発行します。PIV_IGNORE_CHECKSUM_FAILURE が指定されており、かつヘッダーが健全であれば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/storage.c
  • src/backend/storage/buffer/bufmgr.c
  • **呼び出し先**:
  • pg_checksum_page (チェックサム計算)
  • pg_memory_is_all_zeros (全ゼロ判定)
  • ereport (エラー/警告ログ出力)

関連データ構造

  • PageHeaderData: ページヘッダー構造体。pd_flags, pd_lower, pd_upper, pd_special, pd_checksum を保持。

失敗モードと境界条件

  • **チェックサム不一致**: ページデータが破損している可能性を示唆します。
  • **ヘッダー不整合**: pd_lower 等のオフセット値が不正な場合、ページ構造が壊れていると判断されます。
  • **全ゼロページ**: 破損とはみなされず、有効として扱われます。

保守担当者向けメモ

  • 本関数はページをバッファプールへ受け入れるための「最低限の健全性」を保証するものであり、完全なデータ整合性を証明するものではありません。
  • flags 引数によるログ出力制御は、呼び出し側のコンテキスト（リカバリ中か通常稼働中か等）に応じて適切に設定してください。
