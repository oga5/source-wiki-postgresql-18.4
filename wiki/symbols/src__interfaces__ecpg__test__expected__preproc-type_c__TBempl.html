TBempl

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-type.c:66

目的

ECPGプリプロセッサの型定義機能の検証において、カスタム型メンバを含む構造体として利用されるテスト用データ構造。

シグネチャまたは形状

struct TBempl {
    mmInteger idnum;
    mmChar name[21];
    access accs;
};

入力と出力

本構造体は、ECPGの EXEC SQL 文におけるホスト変数として使用されます。

  • **入力**: データベースから取得した値を格納するためのバッファ。
  • **出力**: データベースへの挿入や、取得した値の検証用データとして利用。

前提条件と不変条件

  • mmInteger、mmChar、access といったカスタム型が、プリプロセッサによって適切にC言語の型へ展開されている必要があります。
  • 構造体メンバの配置は、ECPGランタイムが生成するバインド情報と一致している必要があります。

アルゴリズムまたは振る舞い

ECPGプリプロセッサによって生成されたコード内で、SQL文の実行結果をマッピングするターゲットとして機能します。ECPGdo 関数呼び出しにおいて、各メンバのアドレスと型情報（ECPGt_long, ECPGt_char, ECPGt_short 等）がランタイムライブラリに渡され、データバインディングが行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/interfaces/ecpg/test/expected/preproc-type.c:94（インスタンス宣言）
  • src/interfaces/ecpg/test/expected/preproc-type.c:155-159（ECPGdo 関数内でのメンバ参照）
  • src/interfaces/ecpg/test/expected/sql-binary.c:61（インスタンス宣言）
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • mmInteger, mmChar, access: 本構造体のメンバとして使用されるカスタム型。
  • sqlca: エラーハンドリングに使用される通信領域。

失敗モードと境界条件

  • name メンバのサイズ（21バイト）を超えるデータが挿入された場合、バッファオーバーフローや切り捨てが発生する可能性があります。
  • プリプロセッサによる型変換が正しく行われない場合、ECPGdo 実行時に型不一致エラーが発生します。

保守担当者向けメモ

  • 本構造体はプリプロセッサの出力結果であるため、直接編集しないでください。
  • テストの失敗時は、type.pgc（ソース側）の定義と、生成された本ファイルの ECPGdo 引数リストの整合性を確認してください。
