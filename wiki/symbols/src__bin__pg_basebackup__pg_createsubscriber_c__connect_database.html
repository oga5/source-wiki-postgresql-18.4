connect_database

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:535

目的

PostgreSQLデータベースへの接続を確立し、セキュリティ上の理由からセッションの search_path を安全な状態に初期化する。

シグネチャまたは形状

static PGconn *connect_database(const char *conninfo, bool exit_on_error)

入力と出力

  • **入力**:
  • conninfo: 接続文字列（libpq形式）。
  • exit_on_error: 接続または初期化に失敗した場合にプログラムを終了させるかどうかを示すフラグ。
  • **出力**:
  • 成功時: 確立された PGconn へのポインタ。
  • 失敗時: exit_on_error が false の場合は NULL。true の場合はプログラムを終了する。

前提条件と不変条件

  • 呼び出し元は、提供される conninfo が有効な接続文字列であることを保証する必要がある。
  • 接続成功後、常に ALWAYS_SECURE_SEARCH_PATH_SQL を実行して search_path をクリアする。

アルゴリズムまたは振る舞い

1. PQconnectdb を呼び出し、指定された接続文字列でデータベースへの接続を試みる。 2. 接続状態を確認し、失敗した場合はエラーログを出力して PQfinish を実行する。exit_on_error が真であれば exit(1) で終了する。 3. 接続成功後、PQexec を用いて ALWAYS_SECURE_SEARCH_PATH_SQL を実行し、search_path を安全な値に設定する。 4. SQL実行結果が PGRES_TUPLES_OK でない場合、エラーログを出力し、接続を閉じてから必要に応じて終了する。 5. 正常終了時に PGconn ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_createsubscriber.c 内の各所（行202, 599, 781, 893, 1028, 1213, 1252, 1319, 1346, 1591）。
  • **呼び出し先**: libpq 関数群（PQconnectdb, PQstatus, PQerrorMessage, PQfinish, PQexec, PQresultStatus, PQresultErrorMessage, PQclear）。

関連データ構造

  • PGconn: libpqの接続ハンドル。
  • PGresult: SQL実行結果を保持する構造体。

失敗モードと境界条件

  • 接続失敗: PQstatus が CONNECTION_OK でない場合、エラーログを出力して終了または NULL を返す。
  • SQL実行失敗: search_path のリセットに失敗した場合、エラーログを出力して接続を破棄し、終了または NULL を返す。

保守担当者向けメモ

  • 本関数は pg_createsubscriber における標準的なデータベース接続および初期化手順をカプセル化している。
  • ALWAYS_SECURE_SEARCH_PATH_SQL の定義については、インクルードされているヘッダーファイルまたは共通定義を確認すること。
