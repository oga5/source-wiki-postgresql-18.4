ColorTrgmInfo

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_regexp.c:375

目的

正規表現から抽出されたトリグラムの統計情報およびグラフ上の接続情報を保持し、インデックス検索の最適化（選択と簡略化）を行うための構造体です。

シグネチャまたは形状

typedef struct
{
	ColorTrgm	ctrgm;
	int			cnumber;
	int			count;
	float4		penalty;
	bool		expanded;
	List	   *arcs;
} ColorTrgmInfo;

入力と出力

本構造体は、正規表現の解析過程で生成される TrgmNFA 構造体の一部として管理されます。

  • **入力**: TrgmNFA 構築時に、正規表現から抽出されたトリグラム情報が各フィールドに格納されます。
  • **出力**: qsort によるソートや、グラフの簡略化処理（selectColorTrigrams 等）を経て、最終的なインデックス検索用グラフの構築に利用されます。

前提条件と不変条件

  • arcs フィールドは List 型であり、メモリ管理には PostgreSQL のメモリコンテキストが使用されます。
  • penalty 値は、トリグラムの選択・削除の優先順位を決定するために使用されます。

アルゴリズムまたは振る舞い

1. **統計保持**: 正規表現内のトリグラムの出現頻度（count）や、グラフ上での重要度を示す penalty を保持します。 2. **ソート**: colorTrgmInfoCmp 関数等を用いて、penalty や count に基づいてソートされ、インデックス検索に寄与するトリグラムが選択されます。 3. **グラフ構築**: arcs リストを通じて、トリグラム間の遷移関係を保持し、正規表現の構造をトリグラムベースのグラフへ変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_regexp.c 内のグラフ変換および最適化処理（selectColorTrigrams 等）から参照・操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ColorTrgm: 保持するトリグラムの識別子。
  • TrgmNFA: 本構造体を保持し、正規表現解析の全体を管理する親構造体。
  • List: arcs フィールドで使用される PostgreSQL のリスト構造。

失敗モードと境界条件

  • **メモリ不足**: palloc0 によるメモリ確保時に失敗する可能性があります。
  • **解析制限**: trgm_regexp.c の設計上、トリグラムの数が多すぎる場合や複雑すぎる場合は、解析が打ち切られるか、インデックス利用がスキップされます。

保守担当者向けメモ

  • 本構造体は qsort の対象となるため、比較関数 colorTrgmInfoCmp のロジックを変更する際は、本構造体のフィールド定義と整合性を取る必要があります。
  • expanded フラグは、グラフの簡略化プロセスにおいて、そのトリグラムが既に展開済みであるかを追跡するために使用されます。
