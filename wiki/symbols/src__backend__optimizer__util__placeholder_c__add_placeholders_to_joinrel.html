add_placeholders_to_joinrel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:401

目的

結合リレーション（joinrel）が形成される際、そのレベルで評価可能となった PlaceHolderVar (PHV) をターゲットリストに追加し、結合の依存関係（direct_lateral_relids）を更新する。

シグネチャまたは形状

void add_placeholders_to_joinrel(PlannerInfo *root, RelOptInfo *joinrel,
                                 RelOptInfo *outer_rel, RelOptInfo *inner_rel,
                                 SpecialJoinInfo *sjinfo)

入力と出力

  • **入力**:
  • root: クエリ全体のプランナ情報。
  • joinrel: 新たに形成される結合リレーション。
  • outer_rel, inner_rel: 結合を構成する入力リレーション。
  • sjinfo: 結合に関する特殊情報（未使用）。
  • **出力**: なし（joinrel の reltarget および direct_lateral_relids を直接更新する）。

前提条件と不変条件

  • joinrel->reltarget が初期化済みであること。
  • root->placeholder_list にクエリ内の全 PlaceHolderInfo が保持されていること。
  • PHVの phnullingrels は、この時点では NULL であることが期待される。

アルゴリズムまたは振る舞い

1. root->placeholder_list を走査し、各 PlaceHolderInfo について以下を判定する。 2. **評価可能性の判定**: ph_eval_at が joinrel->relids のサブセットである場合、そのPHVはこの結合レベルで評価可能とみなす。 3. **ターゲットリストへの追加**:

  • PHVが上位レベルで必要とされ（ph_needed が空でない）、かつ入力リレーション（outer_rel または inner_rel）のいずれでも評価できない場合に、joinrel->reltarget->exprs に追加する。
  • 追加時には cost_qual_eval_node を用いてコストを算出し、reltarget のコストと幅（width）を更新する。

    • 4. **依存関係の更新**: PHVの評価に必要なリレーション（ph_lateral）を、joinrel->direct_lateral_relids に追加する。これはPHVを実際にターゲットリストへ追加するか否かに関わらず実行される。 5. 最後に joinrel->reltarget->width を clamp_width_est で調整する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/relnode.c:788 (直接の呼び出しを確認)
  • **呼び出し先**:
  • copyObject (PHVの複製)
  • cost_qual_eval_node (コスト計算)
  • bms_is_subset, bms_nonempty_difference, bms_add_members (ビットマップ操作)
  • lappend (リスト操作)

関連データ構造

  • PlannerInfo: クエリ全体の最適化状態を保持。
  • RelOptInfo: 結合リレーションおよび入力リレーションの情報を保持。
  • PlaceHolderInfo: PHVの評価場所や依存関係を定義する構造体。
  • PlaceHolderVar: ターゲットリストに格納される式ノード。

失敗モードと境界条件

  • PHVのコスト計算は、最初に考慮された入力リレーションのペアに基づいて行われるため、他の結合パスでは二重計上される可能性がある（現状の仕様）。
  • ph_lateral の追加は、build_join_rel による後続のクリーンアップ処理に依存している。

保守担当者向けメモ

  • 本関数は、結合順序の正当性を保証するために direct_lateral_relids を更新する重要な役割を担っている。
  • PHVのコスト計算の精度向上については将来的な改善の余地があるが、現状は既存の挙動を維持している。
