BufferDesc

定義

  • 種別: struct
  • 定義箇所: src/include/storage/buf_internals.h:258

目的

共有メモリ上の各バッファのメタデータを保持し、バッファマネージャによるページキャッシュの状態管理とアクセス制御を行うための構造体です。

シグネチャまたは形状

typedef struct BufferDesc
{
    BufferTag    tag;            /* バッファ内のページID */
    int          buf_id;         /* バッファのインデックス番号 (0から開始) */
    pg_atomic_uint32 state;      /* フラグ、参照カウント、使用カウントを含む状態 */
    int          wait_backend_pgprocno; /* ピンカウント待機中のバックエンド */
    int          freeNext;       /* フリーリストのリンク */
    PgAioWaitRef io_wref;        /* AIO進行中のみ設定 */
    LWLock       content_lock;   /* バッファ内容へのアクセスを制御するロック */
} BufferDesc;

入力と出力

  • 本構造体は関数ではなくデータ構造です。
  • state フィールドは、アトミック操作を通じてバッファのフラグ（BM_DIRTY, BM_VALID 等）やカウント情報の読み書きに使用されます。

前提条件と不変条件

  • キャッシュライン境界（64バイト）を意識した設計となっており、構造体のサイズ変更には注意が必要です。
  • state フィールドの更新は、BM_LOCKED フラグの状態に依存するため、LockBufHdr / UnlockBufHdr を介した操作が推奨されます。
  • content_lock はバッファ内のデータアクセスを保護するものであり、ヘッダのメタデータ保護用ではありません。

アルゴリズムまたは振る舞い

  • バッファの状態管理において、state 変数を用いたアトミック操作によるロックフリーな更新と、BM_LOCKED フラグを用いたヘッダロックの2パターンで制御されます。
  • 非同期I/O（AIO）が進行中の場合、io_wref および BM_IO_IN_PROGRESS フラグによって管理されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/pg_buffercache/pg_buffercache_pages.c
  • contrib/pg_prewarm/autoprewarm.c
  • src/backend/storage/buffer/buf_init.c
  • src/backend/storage/buffer/bufmgr.c（PinBuffer, UnpinBuffer, WaitBufHdrUnlocked 等）
  • **呼び出し先**:
  • 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • BufferTag: ページ識別子。
  • PgAioWaitRef: 非同期I/Oの管理用。
  • LWLock: コンテンツロック用。
  • pg_atomic_uint32: 状態管理用アトミック変数。

失敗モードと境界条件

  • state フィールドの不適切な同時更新は、競合やデータ不整合を招く可能性があります。
  • 構造体のサイズがキャッシュライン境界を超過すると、パフォーマンス低下を招く恐れがあります。

保守担当者向けメモ

  • BufferDesc は共有メモリ上に配置されるため、構造体の変更は共有メモリレイアウトに影響を与えます。
  • BM_LOCKED はメタデータ保護用であり、データそのものの保護には content_lock を使用してください。
