heap_execute_freeze_tuple

定義

  • 種別: function
  • 定義箇所: src/include/access/heapam.h:440

目的

VACUUM 処理において、メモリ上のヒープタプルに対して凍結（Freeze）操作を適用し、タプルのヘッダー情報を更新する。

シグネチャまたは形状

static inline void heap_execute_freeze_tuple(HeapTupleHeader tuple, HeapTupleFreeze *frz)

入力と出力

  • **入力**:
  • tuple: 更新対象のタプルヘッダー（HeapTupleHeader）。
  • frz: 凍結に必要な情報（xmax, frzflags, t_infomask, t_infomask2）を保持する HeapTupleFreeze 構造体へのポインタ。
  • **出力**: なし（tuple を直接書き換える）。

前提条件と不変条件

  • tuple は有効なヒープタプルヘッダーを指している必要がある。
  • frz は呼び出し元によって適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

1. HeapTupleHeaderSetXmax を使用して、タプルの xmax を frz->xmax に更新する。 2. frz->frzflags に基づき、t_xvac を設定する：

  • XLH_FREEZE_XVAC がセットされている場合、FrozenTransactionId を設定する。
  • XLH_INVALID_XVAC がセットされている場合、InvalidTransactionId を設定する。

    • 3. タプルの t_infomask および t_infomask2 を frz の値で上書きする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c (複数箇所)
  • src/backend/access/heap/heapam_xlog.c
  • contrib/pg_surgery/heap_surgery.c (示唆あり)
  • **呼び出し先**:
  • HeapTupleHeaderSetXmax
  • HeapTupleHeaderSetXvac

関連データ構造

  • HeapTupleHeader: タプルの物理的なヘッダー構造。
  • HeapTupleFreeze: 凍結操作に必要なメタデータを保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性がある。
  • frzflags に矛盾するフラグ（XLH_FREEZE_XVAC と XLH_INVALID_XVAC の両方など）が設定された場合の挙動は、呼び出し元のロジックに依存する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁に呼び出されるためオーバーヘッドを最小限に抑える設計となっている。
  • heapam_xlog.c での WAL 再生時にも使用されるため、変更を加える際は WAL の互換性に注意すること。
