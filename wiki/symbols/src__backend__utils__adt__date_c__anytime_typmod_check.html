anytime_typmod_check

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/date.c:71

目的

TIME 型および TIMETZ 型の精度（typmod）の妥当性を検証し、必要に応じて正規化を行う。

シグネチャまたは形状

int32 anytime_typmod_check(bool istz, int32 typmod)

入力と出力

  • **入力**:
  • istz (bool): TIMETZ 型の場合は true、TIME 型の場合は false。
  • typmod (int32): 検証対象の精度値。
  • **出力**:
  • 検証済みの精度値（int32）。MAX_TIME_PRECISION を超える場合は、上限値に切り詰められた値が返される。

前提条件と不変条件

  • typmod が負の値である場合、エラー（ERRCODE_INVALID_PARAMETER_VALUE）が発生する。
  • typmod が MAX_TIME_PRECISION を超える場合、警告（WARNING）が発行され、値は MAX_TIME_PRECISION に制限される。

アルゴリズムまたは振る舞い

1. typmod が 0 未満であれば、ereport(ERROR) を呼び出し処理を中断する。 2. typmod が MAX_TIME_PRECISION を超える場合、ereport(WARNING) を発行し、typmod を MAX_TIME_PRECISION に書き換える。 3. 最終的な typmod 値を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_expr.c: 式の解析時に型精度を検証するために呼び出される。
  • src/backend/utils/adt/date.c: anytime_typmodin 等の型修飾子入力関数から呼び出される。
  • src/backend/utils/adt/jsonpath_exec.c: JSONパス実行時の型変換時に呼び出される。
  • **呼び出し先**:
  • ereport: エラーまたは警告の報告に使用。

関連データ構造

  • MAX_TIME_PRECISION: 許容される最大精度を定義する定数（src/include/utils/date.h 等で定義）。

失敗モードと境界条件

  • typmod < 0: ERRCODE_INVALID_PARAMETER_VALUE でエラー終了。
  • typmod > MAX_TIME_PRECISION: 警告を発行し、値を上限値に丸めて正常終了。

保守担当者向けメモ

  • 本関数は TIME 型の精度制限を強制する中心的な役割を担うため、新しい時刻関連型を追加する際や精度定義を変更する際は本関数のロジックを確認すること。
