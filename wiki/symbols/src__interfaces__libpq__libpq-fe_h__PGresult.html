PGresult

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-fe.h:214

目的

PostgreSQL データベースサーバーに対するクエリ実行結果を保持するための構造体です。

シグネチャまたは形状

typedef struct pg_result PGresult; 不透明ポインタ（opaque pointer）として定義されており、内部構造は公開されていません。

入力と出力

本シンボル自体はデータ構造の定義です。クエリ実行関数（PQexec 等）の戻り値として生成され、結果取得関数（PQgetvalue 等）の入力として使用されます。

前提条件と不変条件

  • メンバへの直接アクセスは禁止されています。必ず libpq が提供するアクセサ関数（PQntuples(), PQgetvalue() 等）を使用してください。
  • 使用後は必ず PQclear() を呼び出し、メモリを解放する必要があります。

アルゴリズムまたは振る舞い

クエリ実行結果のステータス保持、行および列データの格納、エラー情報の管理を行います。非同期クエリ実行時にも、PQgetResult() を通じてこの構造体が返されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/dblink/dblink.c 内の各関数（storeQueryResult, storeRow, dblink_res_error 等）が、クエリ結果の保持やエラーハンドリングのために本構造体を参照しています。
  • **呼び出し先**: 不透明ポインタであるため、具体的な呼び出し先（内部関数）は libpq の実装詳細に依存します。

関連データ構造

  • PGconn: データベース接続を管理する構造体。
  • ExecStatusType: クエリ実行結果のステータスを定義する列挙型。

失敗モードと境界条件

  • メモリ不足時には NULL が返される可能性があります。
  • 接続が切断された場合やクエリが失敗した場合、PQresultStatus() がエラーを示す値を返します。

保守担当者向けメモ

  • 本構造体は libpq の公開 API です。バイナリ互換性を維持するため、構造体の定義変更には細心の注意が必要です。
  • 内部実装については src/interfaces/libpq/libpq-int.h を参照してください。
