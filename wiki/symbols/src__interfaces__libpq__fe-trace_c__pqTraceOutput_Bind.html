pqTraceOutput_Bind

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:239

目的

PostgreSQL プロトコルにおける Bind メッセージの内容を解析し、デバッグ用のトレース出力ストリームへ人間が読みやすい形式で書き出す。

シグネチャまたは形状

static void pqTraceOutput_Bind(FILE *f, const char *message, int *cursor)

入力と出力

  • f: 出力先の FILE ストリーム。
  • message: 解析対象のバイナリメッセージデータ。
  • cursor: 現在の解析位置を示すポインタ（入力および更新）。
  • 出力: 戻り値なし。副作用として f に Bind メッセージの各フィールド（ポータル名、プリペアドステートメント名、パラメータフォーマット、パラメータ値、結果フォーマット）が書き出される。

前提条件と不変条件

  • cursor はメッセージの先頭から正しくインクリメントされている必要がある。
  • message は有効な Bind メッセージの構造に従っていること。

アルゴリズムまたは振る舞い

1. Bind という文字列を f に出力する。 2. ポータル名とプリペアドステートメント名を pqTraceOutputString で順次出力する。 3. パラメータフォーマットコードの数（Int16）を読み取り、その数分だけフォーマットコードを pqTraceOutputInt16 で出力する。 4. パラメータ値の数（Int16）を読み取る。 5. 各パラメータ値について、長さ（Int32）を読み取り、値が -1（NULL）でない場合は pqTraceOutputNchar で内容を出力する。 6. 結果フォーマットコードの数（Int16）を読み取り、その数分だけフォーマットコードを pqTraceOutputInt16 で出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-trace.c 内のメッセージディスパッチャ（pqTraceOutputMessage 等から間接的に呼び出される）。
  • 呼び出し先: pqTraceOutputString, pqTraceOutputInt16, pqTraceOutputInt32, pqTraceOutputNchar を使用して各フィールドをデコード・出力する。

関連データ構造

  • FILE: トレース出力先。
  • PGconn: 本関数が直接参照することはないが、呼び出し元を通じて Pfdebug が f に渡される。

失敗モードと境界条件

  • パラメータ値の長さが -1 の場合、NULL 値として処理され、データ本体の出力はスキップされる。
  • メッセージの構造が予期した形式と異なる場合、cursor の不整合により以降のトレース出力が正しく行われない可能性がある。

保守担当者向けメモ

  • 新しいプロトコルバージョンで Bind メッセージの構造が変更された場合、本関数のフィールド読み取り順序を更新する必要がある。
  • regress モードでの出力サニタイズが必要なフィールドについては、pqTraceOutputNchar 等の引数を確認すること。
