目的

PostgreSQLのJSONパーサー（再帰下降型およびインクリメンタル型）の純粋な解析速度を測定するためのベンチマーク用エントリポイントです。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • コマンドライン引数: [-i] <反復回数> <対象ファイルパス>
  • -i: 指定時はインクリメンタル型パーサーを使用。
  • **出力**:
  • 終了コード: 正常終了時は 0。解析エラー発生時は pg_fatal により異常終了。

前提条件と不変条件

  • pg_logging_init が呼び出され、ログ環境が初期化されていること。
  • 指定されたファイルが読み込み可能であること。
  • 解析結果が JSON_SUCCESS であることを不変条件として検証する。

アルゴリズムまたは振る舞い

1. **初期化**: ログ初期化後、StringInfo を準備。 2. **引数解析**: -i フラグの有無を確認し、反復回数と対象ファイルパスを取得。 3. **ファイル読み込み**: 指定ファイルを BUFSIZE (6000バイト) 単位で読み込み、StringInfo バッファへ全量格納。 4. **ベンチマーク実行**: 指定された回数分、以下の処理を繰り返す。

  • use_inc フラグに基づき、makeJsonLexContextIncremental または makeJsonLexContextCstringLen でコンテキストを生成。
  • nullSemAction を用いて解析を実行。
  • freeJsonLexContext でリソースを解放。

    • 5. **終了**: 全反復終了後、exit(0) で終了。

呼び出し元と呼び出し先

  • **呼び出し元**: OSのシェル（スタンドアロンプログラムのエントリポイント）。
  • **呼び出し先**:
  • pg_logging_init, initStringInfo, appendBinaryStringInfo
  • makeJsonLexContextIncremental, pg_parse_json_incremental
  • makeJsonLexContextCstringLen, pg_parse_json
  • freeJsonLexContext, pg_fatal
  • 注記: ツールが提示した contrib/amcheck 等の呼び出し元情報は、本シンボルとは無関係な同名の別シンボルを指しており、本関数の呼び出し元ではありません。

関連データ構造

  • JsonLexContext: JSON解析用コンテキスト。
  • StringInfoData: 入力JSONデータを保持する動的文字列バッファ。
  • nullSemAction: 解析結果に対して何もしないセマンティックアクション構造体。

失敗モードと境界条件

  • ファイルが開けない場合: pg_fatal によりエラー終了。
  • 解析結果が JSON_SUCCESS 以外の場合: pg_fatal によりエラー終了。

保守担当者向けメモ

  • 本プログラムは構文解析のオーバーヘッドのみを測定するため、意味解析は無効化されています。
  • インクリメンタルパーサーを使用する場合も、チャンク分割を行わず全データを一度に渡す仕様です。
