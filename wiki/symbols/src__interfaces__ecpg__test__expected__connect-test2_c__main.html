目的

ECPG（Embedded SQL for C）における複数データベース接続の確立、接続切り替え、および切断処理の動作を検証する回帰テスト用関数。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（void）
  • 出力: 成功時に 0 を返す。

前提条件と不変条件

  • ECPGプリプロセッサによって生成されたコードであり、実行にはECPGランタイムライブラリ（libecpg）が必要。
  • ECPGdebug が有効化されており、標準エラー出力にトレース情報が出力される。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を呼び出し、デバッグ出力を有効化する。 2. **接続確立**: ECPGconnect を使用し、"first"（ecpg2_regression）および "second"（ecpg1_regression）という名前で2つのデータベースに接続する。 3. **クエリ実行**: ECPGdo を用いて select current_database() を実行し、接続先を確認する。接続名を指定しない場合、最後に開いた接続がデフォルトとなる。 4. **接続切り替え**: ECPGsetconn を使用して、アクティブな接続を "first" に切り替える。 5. **切断**:

  • ECPGdisconnect で "CURRENT" を指定し、現在アクティブな "first" を切断する。
  • 既に切断済みの "first" に対して再度 ECPGdisconnect を試行する（エラー発生の検証）。
  • 最後に "second" を "CURRENT" として切断する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムの実行起点）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGsetconn, ECPGdisconnect（いずれもECPGライブラリ関数）

関連データ構造

  • id (char[200]): 接続名を保持するホスト変数。
  • res (char[200]): SQLクエリの結果を格納するホスト変数。

失敗モードと境界条件

  • 既に切断された接続名に対して ECPGdisconnect を呼び出した場合、エラーが発生する。
  • 接続名が正しく指定されていない場合や、データベースが存在しない場合は接続に失敗する。
