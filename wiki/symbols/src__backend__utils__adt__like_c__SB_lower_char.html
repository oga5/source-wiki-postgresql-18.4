SB_lower_char

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:93

目的

シングルバイト文字を、指定されたロケール設定に基づいて小文字に変換します。主に ILIKE 演算子における大文字小文字を区別しない比較処理で使用されます。

シグネチャまたは形状

static char SB_lower_char(unsigned char c, pg_locale_t locale)

入力と出力

  • **入力**:
  • c: 小文字化対象の unsigned char 型文字。
  • locale: ロケール情報を保持する pg_locale_t 構造体へのポインタ。
  • **出力**:
  • 変換後の小文字（char 型）。

前提条件と不変条件

  • locale ポインタが有効であること。
  • locale->ctype_is_c が真の場合、標準的な ASCII 小文字変換が行われます。

アルゴリズムまたは振る舞い

提供されたロケール情報に基づき、以下の優先順位で変換処理を分岐します。 1. **Cロケールの場合**: pg_ascii_tolower(c) を使用して変換します。 2. **デフォルトロケールの場合**: pg_tolower(c) を使用して変換します。 3. **その他のロケールの場合**: tolower_l(c, locale->info.lt) を使用して、ロケール固有のルールに基づき変換します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/like.c 内で定義される MATCH_LOWER マクロを通じて呼び出されます。
  • ILIKE 処理に関連するパターンマッチングロジックから参照されています。
  • **呼び出し先**:
  • pg_ascii_tolower
  • pg_tolower
  • tolower_l

関連データ構造

  • pg_locale_t: ロケール固有の変換情報（ctype_is_c, is_default, info.lt）を保持するために使用されます。

失敗モードと境界条件

  • 入力文字 c が有効な範囲外である場合の挙動は、各変換関数（tolower_l 等）の仕様に依存します。

保守担当者向けメモ

  • 本関数は like.c 内でマクロ MATCH_LOWER を介して利用されるため、変更時は ILIKE のマッチング精度に直接影響します。
  • ロケール処理の分岐は、パフォーマンスと正確性のバランスを考慮した設計となっています。
