pg_vsnprintf

定義

  • 種別: function
  • 定義箇所: src/port/snprintf.c:173

目的

プラットフォーム間の snprintf 実装の差異を吸収し、PostgreSQL が必要とする POSIX 準拠のフォーマット機能（%n$ による引数順序制御など）をポータブルに提供する。

シグネチャまたは形状

int pg_vsnprintf(char *str, size_t count, const char *fmt, va_list args)

入力と出力

  • **入力**:
  • str: 出力先バッファへのポインタ。
  • count: バッファの最大サイズ。
  • fmt: フォーマット文字列。
  • args: 可変長引数リスト (va_list)。
  • **出力**:
  • 成功時: 書き込まれた（または書き込まれるべきであった）文字数。
  • 失敗時: -1。

前提条件と不変条件

  • count == 0 の場合、内部的に1バイトのバッファへリダイレクトされ、呼び出し元には影響を与えない。
  • 戻り値は C99 標準に従い、バッファオーバーランが発生した場合でも「本来書き込まれるべきだったバイト数」を返す。

アルゴリズムまたは振る舞い

1. count が 0 の場合、一時的な1バイトバッファを使用して処理を継続する。 2. PrintfTarget 構造体を初期化し、バッファの範囲と状態を設定する。 3. dopr() 関数を呼び出し、フォーマット文字列の解析と引数の展開を実行する。 4. バッファの末尾にヌル終端文字 (\0) を付与する。 5. target.failed が真であれば -1 を返し、それ以外の場合は書き込み済み文字数を計算して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port.h にて vsnprintf として定義されており、PostgreSQL 内の各所から利用される。
  • **呼び出し先**: dopr()（フォーマット処理のバックエンド）。

関連データ構造

  • PrintfTarget: 出力先バッファやストリームの状態を保持する構造体。

失敗モードと境界条件

  • dopr() 内部でエラーが発生した場合、target.failed が真となり、本関数は -1 を返す。
  • count が 0 の場合、実際には書き込みが行われないが、戻り値は正常時と同様に計算される。

保守担当者向けメモ

  • 本関数は printf ファミリーの基盤であり、標準ライブラリの printf を #undef して使用しているため、内部での標準関数呼び出しには注意が必要。
  • %n$ 指定子による引数制御は PG_NL_ARGMAX (31) までサポートされている。
