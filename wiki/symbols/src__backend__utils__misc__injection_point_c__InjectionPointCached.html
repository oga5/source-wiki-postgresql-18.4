InjectionPointCached

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/injection_point.c:561

目的

ローカルキャッシュに存在するインジェクションポイントを検索し、登録されているコールバック関数を即座に実行します。

シグネチャまたは形状

void InjectionPointCached(const char *name, void *arg)

入力と出力

  • name: 実行対象のインジェクションポイント名。
  • arg: コールバック関数に渡す任意の引数。
  • 戻り値: なし。

前提条件と不変条件

  • 本機能は --enable-injection-points を指定してビルドされた環境でのみ有効です。
  • injection_point_cache_get が正常にキャッシュエントリを返した場合、そのエントリ内の callback は有効な関数ポインタである必要があります。

アルゴリズムまたは振る舞い

1. USE_INJECTION_POINTS が定義されている場合:

  • injection_point_cache_get(name) を呼び出し、ローカルキャッシュから対応する InjectionPointCacheEntry を取得します。
  • キャッシュエントリが存在する場合、そのエントリに保持されている callback を実行します。その際、name、private_data、および引数 arg を渡します。

    • 2. USE_INJECTION_POINTS が定義されていない場合:

  • elog(ERROR, ...) を発行し、ビルド時にサポートされていない旨を通知します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/utils/injection_point.h で定義されたマクロ INJECTION_POINT_CACHED(name, arg) を介して呼び出されます。
  • 呼び出し先: injection_point_cache_get（ローカルキャッシュ検索）、およびキャッシュエントリに登録されたコールバック関数。

関連データ構造

  • InjectionPointCacheEntry: キャッシュされたインジェクションポイントの定義とコールバック情報を保持する構造体。

失敗モードと境界条件

  • USE_INJECTION_POINTS が無効なビルドで呼び出された場合、エラーが発生します。
  • 指定した name に対応するキャッシュエントリが存在しない場合、何も実行されず正常に終了します。

保守担当者向けメモ

  • 本関数はロックフリーなキャッシュ読み取りを前提としています。キャッシュの整合性は InjectionPointCacheRefresh 等のキャッシュ管理ロジックに依存します。
  • デバッグ用途の機能であるため、本番環境での使用は想定されていません。
