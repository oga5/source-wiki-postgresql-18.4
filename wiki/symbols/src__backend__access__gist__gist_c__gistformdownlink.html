gistformdownlink

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gist.c:1139

目的

GiSTインデックスのページ分割時、親ノードに挿入するための新しいダウンリンク（子ページへのポインタとキー情報）を生成する。

シグネチャまたは形状

static IndexTuple gistformdownlink(Relation rel, Buffer buf, GISTSTATE *giststate, GISTInsertStack *stack, bool is_build)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • buf: ダウンリンクを作成する対象の子ページ。
  • giststate: GiSTインデックスの状態管理構造体。
  • stack: 挿入スタック（ページが空の場合のフォールバックに使用）。
  • is_build: インデックス構築中かどうかを示すフラグ。
  • **出力**:
  • 生成された IndexTuple。子ページのブロック番号が設定され、有効な状態（GistTupleSetValid）として返される。

前提条件と不変条件

  • buf は有効なGiSTページであること。
  • ページが空でない場合、ページ内の全タプルの包含範囲を計算してダウンリンクとする。
  • ページが空の場合、親ノードから既存のダウンリンク情報を取得して代用する。

アルゴリズムまたは振る舞い

1. **タプル走査**: 指定された buf 内の全タプルを走査し、gistgetadjusted を用いてそれらを包含する最小のキー（ダウンリンク）を計算する。 2. **空ページ対応**: ページが空の場合、gistFindCorrectParent を呼び出して親ノードから適切なダウンリンクをコピーする。 3. **メタデータ設定**: 生成されたダウンリンクの t_tid に buf のブロック番号を格納し、GistTupleSetValid を呼び出して有効化する。 4. **戻り値**: 完成した IndexTuple を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c 内の挿入処理（ページ分割ロジック）。
  • **呼び出し先**:
  • gistgetadjusted（キーの調整）
  • gistFindCorrectParent（親ノードの探索）
  • CopyIndexTuple（タプルの複製）
  • PageGetMaxOffsetNumber, PageGetItem 等（ページ内操作）

関連データ構造

  • IndexTuple: インデックス内のタプル形式。
  • GISTSTATE: GiSTの操作状態。
  • GISTInsertStack: ツリー降下時のスタック構造。

失敗モードと境界条件

  • **空ページ**: ページ内にタプルが存在しない場合、親ノードの情報を参照する。この際、親ノードのロック取得が必要となる。
  • **メモリ**: CopyIndexTuple によるメモリ確保が失敗した場合はエラーとなる。

保守担当者向けメモ

  • 本関数はページ分割の再帰的なプロセスにおいて、親ノードへ伝播させるキーを決定する重要な役割を担う。
  • ページが空の場合のフォールバック処理では、親ノードのロックを一時的に排他モードで取得するため、デッドロックの可能性に注意すること。
