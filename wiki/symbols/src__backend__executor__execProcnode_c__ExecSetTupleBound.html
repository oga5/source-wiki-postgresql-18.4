ExecSetTupleBound

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/executor/execProcnode.c:847

目的

プランツリーの実行ノードに対し、後続の処理で必要となるタプル数の上限（Tuple Bound）を伝播・設定する。これにより、ソート処理などのメモリ使用量を最適化する。

シグネチャまたは形状

void ExecSetTupleBound(int64 tuples_needed, PlanState *child_node)

入力と出力

  • **入力**:
  • tuples_needed: 必要なタプル数（負の値は制限なしを意味する）。
  • child_node: 設定対象となる PlanState ツリーのノード。
  • **出力**: なし（副作用として child_node 以下の状態を更新する）。

前提条件と不変条件

  • child_node は有効な PlanState であること。
  • 再帰的に呼び出されるため、スタック深さの制限に注意が必要だが、本関数自体での明示的なチェックは行われていない。

アルゴリズムまたは振る舞い

指定された child_node の型に応じて以下の処理を行う：

  • **SortState / IncrementalSortState**: bounded フラグを立て、bound に tuples_needed を設定する。tuples_needed < 0 の場合は bounded を false にリセットする。
  • **AppendState / MergeAppendState**: 子ノード（appendplans / mergeplans）に対して再帰的に本関数を呼び出す。
  • **ResultState**: 子ノード（outerPlanState）に対して再帰的に呼び出す。
  • **SubqueryScanState**: qual が存在しない場合に限り、サブプランに対して再帰的に呼び出す。
  • **GatherState / GatherMergeState**: 自身の tuples_needed を更新し、さらに子ノード（outerPlanState）に対して再帰的に呼び出す。
  • **その他**: 行を破棄または結合する可能性があるノードに到達した場合、伝播は停止する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c: ExecSetTupleBound を呼び出し、並列クエリのタプル制限を設定。
  • src/backend/executor/nodeLimit.c: Limit ノードの処理において、必要なタプル数を計算し本関数を呼び出す。
  • **呼び出し先**:
  • ExecSetTupleBound（再帰呼び出し）
  • 各ノードの PlanState 構造体への直接アクセス

関連データ構造

  • PlanState: 実行ノードの共通基底構造体。
  • SortState, IncrementalSortState, AppendState, MergeAppendState, ResultState, SubqueryScanState, GatherState, GatherMergeState: 各種実行ノードの状態構造体。

失敗モードと境界条件

  • tuples_needed < 0 の場合、制限は解除される。
  • SubqueryScanState のように qual を持つノードでは、行が破棄される可能性があるため、制限の伝播は安全のために停止される。

保守担当者向けメモ

  • 本関数は再帰的に動作する。新しいノードタイプを追加し、タプル制限の伝播をサポートする必要がある場合は、本関数の switch（または if-else）構造にケースを追加すること。
  • Sort や Gather ノード側で、設定された bound を適切に利用する実装が別途必要である。
