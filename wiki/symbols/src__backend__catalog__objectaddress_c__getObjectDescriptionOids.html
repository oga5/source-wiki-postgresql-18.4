getObjectDescriptionOids

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaddress.c:4085

目的

OID（カタログIDおよびオブジェクトID）を指定して、データベースオブジェクトの人間が読める説明文を取得します。

シグネチャまたは形状

char *getObjectDescriptionOids(Oid classid, Oid objid)

入力と出力

  • **入力**:
  • classid: オブジェクトが属するシステムカタログのOID。
  • objid: 対象オブジェクトのOID。
  • **出力**:
  • オブジェクトの説明文を含む palloc された文字列。

前提条件と不変条件

  • objectSubId は常に 0 として扱われます。
  • 内部で getObjectDescription を呼び出すため、その関数が要求するシステムカタログへのアクセス権限およびキャッシュ状態が有効である必要があります。

アルゴリズムまたは振る舞い

1. ObjectAddress 構造体を初期化し、classId に classid を、objectId に objid を、objectSubId に 0 を代入します。 2. getObjectDescription を呼び出し、その結果を返します。第2引数には false が渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c (複数の箇所でエラーメッセージ生成等に使用)
  • **呼び出し先**:
  • getObjectDescription

関連データ構造

  • ObjectAddress: オブジェクトを特定するための構造体。
  • Oid: PostgreSQLのオブジェクト識別子型。

失敗モードと境界条件

  • 指定された classid や objid がシステムカタログ上に存在しない場合、getObjectDescription の内部実装に従ってエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は getObjectDescription のラッパーです。オブジェクトの特定にサブID（列番号など）が必要な場合は、本関数ではなく getObjectDescription を直接使用してください。
