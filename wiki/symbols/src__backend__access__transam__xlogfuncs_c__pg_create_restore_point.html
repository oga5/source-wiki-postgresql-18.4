pg_create_restore_point

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogfuncs.c:232

目的

指定された名前でWAL（Write-Ahead Log）内にリストアポイントを作成し、その時点のLSN（Log Sequence Number）を返却する。

シグネチャまたは形状

Datum pg_create_restore_point(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介してリストアポイント名（text型）を受け取る。
  • **出力**: 作成されたリストアポイントのWAL位置（LSN型）を PG_RETURN_LSN マクロで返却する。

前提条件と不変条件

  • サーバーがリカバリ中（RecoveryInProgress()）ではないこと。
  • wal_level が replica または logical に設定されていること（XLogIsNeeded() が真であること）。
  • リストアポイント名が MAXFNAMELEN 未満であること。

アルゴリズムまたは振る舞い

1. RecoveryInProgress() を確認し、リカバリ中の場合はエラーを投げる。 2. XLogIsNeeded() を確認し、WALレベルが不十分な場合はエラーを投げる。 3. 入力された text 型の名称をC文字列に変換する。 4. 文字列長が MAXFNAMELEN 以上であればエラーを投げる。 5. XLogRestorePoint() を呼び出し、リストアポイントを作成する。 6. 作成されたリストアポイントのLSNを返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（直接の呼び出し元コードはソース内に明示なし）。
  • **呼び出し先**:
  • RecoveryInProgress()
  • XLogIsNeeded()
  • text_to_cstring()
  • XLogRestorePoint()

関連データ構造

  • XLogRecPtr: WAL上の位置を示すデータ型。
  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • **リカバリ中**: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE を発生させる。
  • **WALレベル不足**: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE を発生させる。
  • **名称長超過**: MAXFNAMELEN を超える場合、ERRCODE_INVALID_PARAMETER_VALUE を発生させる。

保守担当者向けメモ

  • 本関数はSQL層からのエントリポイントであり、内部のWAL管理ロジック（XLogRestorePoint）をラップしている。
  • wal_level のチェックはサーバー起動時の設定に依存する。
  • 戻り値のLSNは、後続のリカバリ操作やバックアップ管理において参照される可能性がある。
