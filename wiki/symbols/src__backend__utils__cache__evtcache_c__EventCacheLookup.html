EventCacheLookup

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/evtcache.c:62

目的

指定されたイベントに関連付けられたイベントトリガーのリストを、メモリ上のキャッシュから高速に取得する。

シグネチャまたは形状

List *EventCacheLookup(EventTriggerEvent event)

入力と出力

  • **入力**: EventTriggerEvent event（検索対象のイベント種別）
  • **出力**: List *（該当イベントで実行すべきトリガーのリスト。該当がない場合は NIL）

前提条件と不変条件

  • キャッシュの状態管理変数 EventTriggerCacheState が ETCS_VALID でない場合、内部で BuildEventTriggerCache() が呼び出され、キャッシュが再構築される。
  • 戻り値のリストは EventTriggerCacheContext 内で管理されているため、キャッシュがリセットされると無効になる可能性がある。長期間保持が必要な場合は呼び出し側でコピーを作成すること。

アルゴリズムまたは振る舞い

1. EventTriggerCacheState を確認し、キャッシュが無効であれば BuildEventTriggerCache() を実行して再構築する。 2. EventTriggerCache ハッシュテーブルに対して hash_search を行い、指定された event に対応するエントリを検索する。 3. エントリが見つかればその triggerlist を返し、見つからなければ NIL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/event_trigger.c (複数の箇所でイベントトリガーの存在確認やリスト取得に使用)
  • **呼び出し先**:
  • BuildEventTriggerCache (キャッシュ無効時の再構築)
  • hash_search (ハッシュテーブル検索)

関連データ構造

  • EventTriggerEvent: 検索キーとなるイベント種別。
  • EventTriggerCacheEntry: ハッシュテーブルに格納されるエントリ構造体。
  • EventTriggerCache: キャッシュの実体であるハッシュテーブル。

失敗モードと境界条件

  • キャッシュが空または無効な状態では、初回呼び出し時にカタログスキャンが発生するため、検索コストが増大する。
  • キャッシュの再構築中に無効化通知が届いた場合、現在の再構築処理を完了させた後に再度無効化フラグが立てられる設計となっている。

保守担当者向けメモ

  • 本関数はイベントトリガーキャッシュへの唯一の公開アクセスポイントである。
  • 戻り値のリストの生存期間はキャッシュのライフサイクルに依存するため、呼び出し側でのメモリ管理に注意すること。
