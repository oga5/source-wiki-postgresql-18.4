DllUnregisterServer

定義

  • 種別: function
  • 定義箇所: src/bin/pgevent/pgevent.c:126

目的

WindowsレジストリからPostgreSQLのイベントログソース情報を削除し、pgevent.dll の登録を解除する。

シグネチャまたは形状

STDAPI DllUnregisterServer(void)

入力と出力

  • **入力**: なし（グローバル変数 event_source を参照）。
  • **出力**: HRESULT 型のステータスコード。成功時は S_OK、失敗時は SELFREG_E_TYPELIB を返す。

前提条件と不変条件

  • レジストリの HKEY_LOCAL_MACHINE を操作するため、呼び出し元プロセスには管理者権限が必要である。
  • event_source 変数には、削除対象となる有効なイベントソース名が格納されている必要がある。

アルゴリズムまたは振る舞い

1. event_source を用いて、削除対象のレジストリキーパス（SYSTEM\CurrentControlSet\Services\EventLog\Application\%s）を key_name バッファに構築する。 2. RegDeleteKey 関数を呼び出し、指定されたレジストリキーを削除する。 3. 削除に失敗した場合、MessageBox でエラーを表示し、SELFREG_E_TYPELIB を返す。 4. 削除に成功した場合、S_OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: regsvr32 コマンド（COMエントリポイントとして）。また、ソースコード内のコメントによれば DllInstall からの呼び出しも想定されている。
  • **呼び出し先**: Windows API (_snprintf, RegDeleteKey, MessageBox)。

関連データ構造

  • event_source: 削除対象のイベントソース名を保持するグローバル変数。

失敗モードと境界条件

  • **権限不足**: 管理者権限がない場合、RegDeleteKey が失敗し、エラーメッセージボックスが表示される。
  • **キーの不在**: 指定したキーが存在しない場合、RegDeleteKey は失敗する可能性がある。
  • **バッファオーバーフロー**: key_name は400バイトで固定されており、event_source が極端に長い場合は切り詰められる可能性がある。

保守担当者向けメモ

  • 本関数は DllRegisterServer と対になる処理を行う。
  • regsvr32 を介して実行されることを前提としているため、デバッグ時には regsvr32 の挙動に注意すること。
