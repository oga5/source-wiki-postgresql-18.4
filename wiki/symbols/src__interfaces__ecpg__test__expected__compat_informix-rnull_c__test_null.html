test_null

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-rnull.c:27

目的

Informix 互換モードにおける ECPG の NULL 値判定結果を標準出力へ出力し、回帰テストの検証を補助する。

シグネチャまたは形状

static void test_null(int type, char *ptr)

入力と出力

  • **入力**:
  • type: 判定対象のデータ型（sqltypes.h で定義される定数）。
  • ptr: 判定対象の変数のメモリ領域へのポインタ。
  • **出力**: なし（標準出力へ null: <結果> を書き出す）。

前提条件と不変条件

  • Informix 互換モードでコンパイルされていること。
  • risnull 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. risnull(type, ptr) を呼び出し、指定された型の変数が NULL であるかを判定する。 2. 判定結果（通常は 0 または 1）を printf を用いて null: %d\n の形式で標準出力に書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/interfaces/ecpg/test/expected/compat_informix-rnull.c 内の複数箇所から、各データ型に対する検証として呼び出される）。
  • **呼び出し先**: risnull（Informix 互換ライブラリ関数）、printf（標準ライブラリ）。

関連データ構造

  • sqltypes.h で定義される型定数（CCHARTYPE, CSHORTTYPE, CINTTYPE 等）。

失敗モードと境界条件

  • ptr が無効なメモリ領域を指している場合、risnull の動作は未定義となる。
  • type に対応しない型が渡された場合の risnull の挙動は、ライブラリの実装に依存する。

保守担当者向けメモ

  • 本関数はテストコードの一部であり、rnull.pgc から生成される。ロジックの変更が必要な場合は、元の .pgc ファイルを修正すること。
