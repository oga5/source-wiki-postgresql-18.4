DecodeTextArrayToBitmapset

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/evtcache.c:221

目的

PostgreSQLのシステムカタログ pg_event_trigger に格納されたコマンドタグのテキスト配列を、高速な検索が可能な Bitmapset 形式へ変換します。

シグネチャまたは形状

static Bitmapset *DecodeTextArrayToBitmapset(Datum array)

入力と出力

  • **入力**: Datum array（TEXT 型の1次元配列）
  • **出力**: Bitmapset *（コマンドタグの列挙値をビットフラグとして保持するセット）

前提条件と不変条件

  • 入力配列は1次元であること。
  • 入力配列に NULL 要素が含まれていないこと。
  • 入力配列の要素型が TEXTOID であること。
  • 上記条件を満たさない場合、elog(ERROR) により処理が中断されます。

アルゴリズムまたは振る舞い

1. DatumGetArrayTypeP を用いて入力 Datum を ArrayType に変換します。 2. deconstruct_array_builtin を使用して配列を Datum の配列に分解します。 3. 各要素に対して以下の処理を繰り返します。

  • TextDatumGetCString でテキストを C 文字列に変換します。
  • GetCommandTagEnum で文字列をコマンドタグの列挙値に変換します。
  • bms_add_member を用いて、その値を Bitmapset に追加します。
  • 一時的な C 文字列を pfree で解放します。

    • 4. 最後に分解した Datum 配列を pfree で解放し、構築した Bitmapset を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/evtcache.c 内の BuildEventTriggerCache 関数（183行目）から呼び出されます。
  • **呼び出し先**: DatumGetArrayTypeP, deconstruct_array_builtin, TextDatumGetCString, GetCommandTagEnum, bms_add_member, pfree。

関連データ構造

  • Bitmapset: コマンドタグの集合を保持するために使用されます。
  • ArrayType: 入力データの構造体。

失敗モードと境界条件

  • 入力配列が多次元、NULL を含む、または TEXT 型以外の場合、エラーが発生します。
  • メモリ不足時には palloc 関連の関数を通じてエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はイベントトリガーキャッシュの構築時にのみ使用されます。
  • Bitmapset は palloc によってメモリが割り当てられるため、呼び出し側で適切なメモリコンテキスト管理が必要です。
  • GetCommandTagEnum の仕様変更がコマンドタグの解釈に直接影響するため、変更時には注意してください。
