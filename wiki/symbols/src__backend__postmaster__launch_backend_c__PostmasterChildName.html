PostmasterChildName

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/launch_backend.c:210

目的

指定された BackendType に対応する、人間が読み取り可能なプロセス名の文字列を返します。主にログ出力やエラーメッセージにおいて、プロセス種別を識別するために使用されます。

シグネチャまたは形状

const char *PostmasterChildName(BackendType child_type);

入力と出力

  • **入力**: child_type (BackendType 型) - 識別対象のプロセス種別。
  • **出力**: const char * - プロセス名を表す静的な文字列へのポインタ。

前提条件と不変条件

  • child_type は、child_process_kinds テーブルの有効なインデックス範囲内である必要があります。
  • 返される文字列は静的メモリ領域を指しており、呼び出し側で解放する必要はありません。

アルゴリズムまたは振る舞い

内部的に保持されている child_process_kinds 配列（またはテーブル）に対し、引数 child_type をインデックスとしてアクセスし、対応する name フィールドを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/launch_backend.c:191
  • src/backend/postmaster/pmchild.c:197
  • src/backend/postmaster/postmaster.c:3968
  • **呼び出し先**:
  • 本関数はデータテーブルを参照するのみであり、他の関数を呼び出しません。

関連データ構造

  • BackendType: プロセス種別を定義する列挙型。
  • child_process_kinds: 各プロセス種別のメタデータ（名前やメイン関数など）を保持するテーブル。

失敗モードと境界条件

  • child_type が定義された範囲外の値である場合、配列外参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数は src/include/postmaster/postmaster.h で宣言されており、Postmaster モジュール全体から参照可能です。
  • 新しいプロセス種別を BackendType に追加した際は、必ず child_process_kinds テーブルにも対応する名前を定義してください。
