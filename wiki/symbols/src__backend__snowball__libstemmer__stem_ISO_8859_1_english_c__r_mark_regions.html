r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_english.c:417

目的

単語内の領域（R1, R2）を特定し、ステミング処理の適用範囲を決定する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体。現在のカーソル位置や文字列データを含む）
  • 出力: int（常に 1 を返す）
  • 副作用: z->I[0] および z->I[1] に、それぞれ R2 および R1 の開始位置を格納する。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。

アルゴリズムまたは振る舞い

1. z->I[1]（R1）と z->I[0]（R2）を文字列の末尾位置で初期化する。 2. 特定の接頭辞（a_0 に含まれるもの）の有無を確認し、条件に応じて領域の開始位置を算出する。 3. out_grouping（母音以外）と in_grouping（母音）を組み合わせて、母音と子音の遷移点を探す。 4. 最初の遷移点（母音の直後）を R1 の開始位置とし、さらにその後の遷移点を R2 の開始位置として z->I に記録する。 5. 処理終了後、カーソル位置を元の値に復元する。

呼び出し元と呼び出し先

  • 呼び出し元: english_ISO_8859_1_stem（直接の言及はないが、モジュール概要の制御フローより推測される）。
  • 呼び出し先: find_among、out_grouping、in_grouping。
  • 補足: 他の言語ファイル（stem_ISO_8859_1_basque.c 等）からも同名の関数が参照されているが、本ページは stem_ISO_8859_1_english.c 内の定義に限定している。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_0: 領域判定に使用される among 構造体のテーブル。
  • g_v: 母音を定義するグループ構造体。

失敗モードと境界条件

  • out_grouping や in_grouping が条件を満たさない場合、ラベル lab0 へジャンプし、領域の更新を行わずに終了する。
  • 文字列が短く、領域を定義できない場合は初期値（末尾）が維持される。
