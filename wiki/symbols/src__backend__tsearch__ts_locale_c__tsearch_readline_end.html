tsearch_readline_end

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_locale.c:161

目的

全文検索用の設定ファイルや辞書ファイルの読み込み処理を終了し、関連するリソースの解放とエラーコンテキストの復元を行う。

シグネチャまたは形状

void tsearch_readline_end(tsearch_readline_state *stp);

入力と出力

  • **入力**: tsearch_readline_state *stp - 読み込み状態を保持する構造体へのポインタ。
  • **出力**: なし（副作用としてリソースを解放する）。

前提条件と不変条件

  • 引数 stp は有効な tsearch_readline_state を指している必要がある。
  • tsearch_readline_begin によって初期化された状態を終了させるために使用される。

アルゴリズムまたは振る舞い

1. **行バッファの解放**: stp->curline が存在する場合、それが stp->buf.data と異なるメモリ領域であれば pfree で解放し、ポインタを NULL にリセットする。 2. **内部バッファの解放**: stp->buf.data を pfree で解放する。 3. **ファイルクローズ**: FreeFile(stp->fp) を呼び出し、関連付けられたファイルポインタを閉じる。 4. **エラーコンテキストの復元**: error_context_stack を、本処理開始前の状態（stp->cb.previous）に戻す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dict_xsyn/dict_xsyn.c
  • contrib/unaccent/unaccent.c
  • src/backend/tsearch/dict_synonym.c
  • src/backend/tsearch/dict_thesaurus.c
  • src/backend/tsearch/spell.c
  • **呼び出し先**:
  • pfree (メモリ解放)
  • FreeFile (ファイルクローズ)

関連データ構造

  • tsearch_readline_state: ファイル読み込みの状態、バッファ、およびエラーコンテキスト情報を保持する構造体。

失敗モードと境界条件

  • stp が NULL の場合、あるいは stp 内のポインタが不正な場合はセグメンテーション違反を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は tsearch_readline シリーズの終了処理を担うため、読み込み処理の正常終了・異常終了を問わず、必ず呼び出されるように設計すること。
  • error_context_stack の操作を伴うため、スタックの整合性を維持することが重要である。
