datum_to_json_internal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/json.c:178

目的

任意のPostgreSQLデータ型（Datum）をJSON形式の文字列に変換し、指定された StringInfo バッファへ追記する再帰的なシリアライズ関数です。

シグネチャまたは形状

static void datum_to_json_internal(
    Datum val, 
    bool is_null, 
    StringInfo result,
    JsonTypeCategory tcategory, 
    Oid outfuncoid,
    bool key_scalar
)

入力と出力

  • **入力**:
  • val: 変換対象のDatum。
  • is_null: 値がNULLかどうかを示すフラグ。
  • result: 結果を格納する StringInfo バッファ。
  • tcategory: JsonTypeCategory による型の分類。
  • outfuncoid: 型に応じた出力関数またはキャスト関数のOID。
  • key_scalar: JSONオブジェクトのキーとして使用されるかを示すフラグ。
  • **出力**: なし（result バッファを直接更新）。

前提条件と不変条件

  • key_scalar が真の場合、is_null は偽でなければならない（アサーションにより保証）。
  • key_scalar が真の場合、tcategory が配列、複合型、JSON、またはキャスト型であってはならない（エラー送出）。
  • 再帰呼び出しを行うため、check_stack_depth() によりスタックオーバーフローを防止する。

アルゴリズムまたは振る舞い

1. **NULL処理**: is_null が真の場合、バッファに "null" を追記して終了。 2. **型カテゴリによる分岐 (switch)**:

  • JSONTYPE_ARRAY / JSONTYPE_COMPOSITE: それぞれ array_to_json_internal または composite_to_json を再帰的に呼び出す。
  • JSONTYPE_BOOL: "true" または "false" を追記。key_scalar が真の場合は引用符で囲む。
  • JSONTYPE_NUMERIC: 数値として妥当な形式であればそのまま、それ以外は引用符で囲んで追記。
  • JSONTYPE_DATE / TIMESTAMP / TIMESTAMPTZ: JsonEncodeDateTime を使用して文字列化し、引用符で囲む。
  • JSONTYPE_JSON: 既存のJSON文字列をそのまま追記。
  • JSONTYPE_CAST: 指定されたキャスト関数を実行し、結果を追記。
  • default: テキスト系型（text, varchar, bpchar）は escape_json_text で最適化処理を行い、それ以外は escape_json でエスケープ処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/json.c 内の複数の箇所（集約関数や変換関数など、計10箇所で参照を確認）。
  • **呼び出し先**: array_to_json_internal, composite_to_json, OidOutputFunctionCall, JsonEncodeDateTime, escape_json_text, escape_json 等。

関連データ構造

  • JsonTypeCategory: 型の分類を定義する列挙型。
  • StringInfo: PostgreSQLの動的文字列バッファ。

失敗モードと境界条件

  • key_scalar が真の状態で非スカラー型（配列や複合型など）が渡された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなる。
  • スタック深度が制限を超えた場合、check_stack_depth() によりエラーが発生する。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、スタック消費量に注意すること。
  • JSONTYPE_NUMERIC の判定ロジックはパフォーマンス向上のため手動で最適化されているため、変更時は数値フォーマットの仕様に留意すること。
  • テキスト型に対する escape_json_text は、汎用的な escape_json よりもメモリ割り当てを抑える設計となっている。
