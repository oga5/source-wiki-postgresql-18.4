skip

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_lex.c:981

目的

正規表現のコンパイルオプションに REG_EXPANDED が指定されている場合に、入力文字列内の空白文字およびコメント（# から行末まで）を読み飛ばす。

シグネチャまたは形状

static void skip(struct vars *v)

入力と出力

  • 入力: struct vars *v （正規表現の解析状態を保持する構造体へのポインタ）
  • 出力: なし（副作用として v->now が更新される）

前提条件と不変条件

  • v->cflags に REG_EXPANDED フラグが設定されている必要がある（assert により検証）。
  • v->now は現在の解析位置を指している。

アルゴリズムまたは振る舞い

1. 現在の解析位置 v->now を記録する。 2. ループ内で以下の処理を繰り返す：

  • iscspace を用いて、空白文字（スペース、タブ、改行など）をスキップする。
  • 文字列終端に達していない、かつ現在の文字が # である場合、行末（\n）までをコメントとしてスキップする。

    • 3. 処理の結果、解析位置が初期位置から移動していた場合、NOTE(REG_UNONPOSIX) を呼び出し、POSIX非準拠の拡張機能が使用されたことを記録する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内には明示的な呼び出し元は記載されていない。
  • 呼び出し先: iscspace（標準ライブラリ）、ATEOS（マクロ）、NEXT1（マクロ）、NOTE（マクロ）。
  • 注記: ツールが検出した contrib/amcheck 等の skip は、本関数とは無関係な別シンボルである。

関連データ構造

  • struct vars: 正規表現コンパイルのコンテキストを保持する構造体。v->now（現在の文字ポインタ）および v->cflags を参照する。

失敗モードと境界条件

  • 入力文字列の終端（ATEOS()）に達した場合、ループを終了する。
  • コメント行の終端に改行文字がない場合、文字列終端までをコメントとして扱う。

保守担当者向けメモ

  • 本関数は REG_EXPANDED モード専用である。
  • NOTE(REG_UNONPOSIX) は、POSIX標準外の構文が使用されたことを通知するために重要である。
  • iscspace の挙動はロケールに依存する可能性があるため、正規表現のコンパイル環境に注意すること。
