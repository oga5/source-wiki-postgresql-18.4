CheckCmdReplicaIdentity

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/execReplication.c:766

目的

論理レプリケーションにおいて、対象テーブルが UPDATE または DELETE 操作を実行するために必要な「レプリカアイデンティティ（Replica Identity）」要件を満たしているかを検証します。

シグネチャまたは形状

void CheckCmdReplicaIdentity(Relation rel, CmdType cmd)

入力と出力

  • **入力**:
  • rel: 検証対象のテーブルリレーション。
  • cmd: 実行しようとしているコマンド種別（CMD_UPDATE または CMD_DELETE）。
  • **出力**: なし。要件を満たさない場合は ereport(ERROR, ...) により処理を中断します。

前提条件と不変条件

  • パーティションテーブル（RELKIND_PARTITIONED_TABLE）の場合、検証はスキップされます（操作はリーフパーティションに対して行われるため）。
  • INSERT 操作に対しては検証を行いません。
  • UPDATE または DELETE 操作において、以下のいずれかが真であれば検証を通過します。
  • テーブルに有効なレプリカインデックスが設定されている。
  • テーブルのレプリカアイデンティティが REPLICA_IDENTITY_FULL に設定されている。

アルゴリズムまたは振る舞い

1. **初期チェック**: パーティションテーブルまたは INSERT 操作であれば即座に終了します。 2. **パブリケーション記述子の構築**: RelationBuildPublicationDesc を呼び出し、行フィルタやカラムリストの妥当性を確認するための記述子を生成します。 3. **制約検証**:

  • 行フィルタ、カラムリスト、および生成カラムがレプリカアイデンティティと整合しているかを確認します。
  • 整合していない場合、ERRCODE_INVALID_COLUMN_REFERENCE でエラーを報告します。

    • 4. **アイデンティティ確認**:

  • レプリカインデックスが存在するか、または REPLICA_IDENTITY_FULL であれば正常終了します。

    • 5. **最終エラー判定**:

  • 上記のいずれでもない場合、テーブルが UPDATE または DELETE をパブリッシュしているならば、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execMain.c
  • src/backend/executor/execReplication.c
  • **呼び出し先**:
  • RelationBuildPublicationDesc
  • RelationGetRelationName
  • RelationGetReplicaIndex
  • ereport (エラー報告用)

関連データ構造

  • Relation: リレーション情報。
  • CmdType: コマンド種別（CMD_UPDATE, CMD_DELETE）。
  • PublicationDesc: パブリケーションの制約情報（行フィルタの有効性、カラムリストの整合性など）。

失敗モードと境界条件

  • **無効なカラム参照**: パブリケーションの WHERE 句やカラムリストがレプリカアイデンティティでカバーされていない場合、エラーとなります。
  • **未公開の生成カラム**: レプリカアイデンティティに含まれる生成カラムがパブリッシュされていない場合、エラーとなります。
  • **アイデンティティ欠如**: レプリカアイデンティティが設定されていないテーブルで、更新や削除をパブリッシュしようとするとエラーとなります。

保守担当者向けメモ

  • 本関数は、論理レプリケーションの適用プロセスにおいて、データ整合性を保証するための重要なガードレールです。
  • XXX コメントにある通り、パブリケーションの構成によっては記述子の構築を最適化できる可能性がありますが、現在の実装では複雑性を避けるため一律で構築を行っています。
  • REPLICA_IDENTITY_FULL はすべてのカラムをアイデンティティとして扱うため、常に検証を通過します。
