RangeBound

定義

  • 種別: struct
  • 定義箇所: utils/rangetypes.h:62
  • 参照箇所: src/backend/utils/adt/multirangetypes.c:85

目的

レンジ（範囲）の境界値を表現するためのデータ構造です。

シグネチャまたは形状

具体的な構造体のメンバ構成は本ソース excerpt には含まれていません。

入力と出力

  • 入力: 範囲の境界値（下限または上限）を保持します。
  • 出力: 範囲の比較やソート、マルチレンジの正規化処理において、境界値の判定基準として利用されます。

前提条件と不変条件

  • マルチレンジ型において、RangeBound を含むレンジの集合は常に「ソート済みかつ非重複」である必要があります。
  • この不変条件を維持するため、構築時には multirange_canonicalize を経由することが求められます。

アルゴリズムまたは振る舞い

  • multirange_bsearch_comparison 型の関数ポインタ定義において、lower および upper の RangeBound ポインタが引数として使用されます。
  • マルチレンジの構築や比較処理において、レンジの境界を特定するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/multirangetypes.c 内の複数の関数（行 748, 807, 861, 862, 881, 882, 913, 1513, 1514, 1536 など）で、引数やローカル変数として参照されています。
  • 呼び出し先: 本ソース excerpt からは、特定の関数呼び出し先は特定できません。

関連データ構造

  • MultirangeType: マルチレンジのメモリ上の表現。
  • TypeCacheEntry: 境界値の比較に使用される型キャッシュ情報。

失敗モードと境界条件

  • 境界値が可変長であるため、メモリレイアウト上のオフセット計算を誤ると不正なアクセスが発生する可能性があります。
  • MultirangeGetItemsPtr などのマクロを介さない直接的なアクセスは避ける必要があります。

保守担当者向けメモ

  • 本シンボルは utils/rangetypes.h で定義されており、multirangetypes.c はそれを参照してマルチレンジの境界操作を行っています。
  • 構造体の詳細なメンバについては utils/rangetypes.h を参照してください。
