pg_bswap64

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_bswap.h:88

目的

64ビット符号なし整数のバイト順序を反転（エンディアン変換）します。

シグネチャまたは形状

static inline uint64 pg_bswap64(uint64 x)

入力と出力

  • **入力**: uint64 x （変換対象の64ビット符号なし整数）
  • **出力**: バイト順序が反転された uint64 値

前提条件と不変条件

  • 本関数は、コンパイラ組み込み関数（__builtin_bswap64 等）が利用できない環境におけるフォールバック実装です。
  • 入力値は符号なし整数として扱われます。

アルゴリズムまたは振る舞い

ビットシフト演算と論理和を用いて、以下の手順でバイト順序を入れ替えます。 1. 入力値 x の各バイトを、対応する反対側の位置へシフトします。 2. UINT64CONST マクロで定義されたビットマスクを適用し、不要なビットをクリアします。 3. 8つのシフト結果を論理和（OR）で結合して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_uuid.c 内の UUID 処理において、uu[0] および uu[1] の変換に使用されています。
  • **呼び出し先**: 本関数自体は組み込み関数を使用せず、演算のみで構成されています。
  • **注記**: pg_bswap.h 内で定義されるマクロ pg_hton64、pg_ntoh64、DatumBigEndianToNative は、環境に応じて本関数またはコンパイラ組み込み関数を呼び出します。

関連データ構造

  • uint64: PostgreSQL で定義される64ビット符号なし整数型。

失敗モードと境界条件

  • 符号付き整数を渡した場合、ビットパターンがそのまま反転されるため、予期せぬ値になる可能性があります。
  • 本関数はフォールバック用であるため、プラットフォームが組み込み関数をサポートしている場合は、コンパイル時にそちらが優先される設計となっています。

保守担当者向けメモ

  • 本関数は汎用的な実装です。パフォーマンスが重要な箇所では、コンパイラ組み込み関数が正しく選択されているか確認してください。
  • 符号付き整数を扱う場合は、呼び出し側で事前にキャストを行う必要があります。
