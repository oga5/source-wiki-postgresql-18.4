ExtensibleNode

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/extensible.h:32

目的

PostgreSQLのコアシステムに依存せず、拡張機能が独自のノード型を定義するための基底構造体を提供します。

シグネチャまたは形状

typedef struct ExtensibleNode
{
    NodeTag     type;
    const char *extnodename;
} ExtensibleNode;

入力と出力

  • **入力**: extnodename には、対応する ExtensibleNodeMethods を識別するための一意な文字列（最大64バイト）を指定します。
  • **出力**: 本構造体は、コアシステムのノード処理（コピー、比較、シリアライズ）において、拡張機能が提供するメソッドを呼び出すための識別子として機能します。

前提条件と不変条件

  • extnodename は、RegisterExtensibleNodeMethods 等を通じてコアシステムに登録された ExtensibleNodeMethods と一致している必要があります。
  • 拡張ノードのメソッド（コピー、比較、入出力）の実装は必須です。

アルゴリズムまたは振る舞い

1. コアシステムはノードの extnodename をキーとして、登録済みのメソッドテーブルを検索します。 2. 取得したメソッドポインタを通じて、nodeCopy や _outExtensibleNode などの拡張機能側で定義されたロジックが間接的に実行されます。 3. シリアライズ時には readfuncs.c 等が pg_strtok() を用いて手動パースを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/copyfuncs.c (_copyExtensibleNode), src/backend/nodes/equalfuncs.c (_equalExtensibleNode), src/backend/nodes/outfuncs.c (_outExtensibleNode), src/backend/nodes/readfuncs.c。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、具体的な処理は ExtensibleNodeMethods 内のコールバック関数へ委譲されます。

関連データ構造

  • ExtensibleNodeMethods: ノードのライフサイクルを制御するコールバック群。
  • CustomPathMethods, CustomScanMethods, CustomExecMethods: カスタムスキャン計画および実行に関連するメソッド群。

失敗モードと境界条件

  • extnodename が未登録の場合、コアシステムによる検索が失敗します。
  • nodeRead 実装時のパースエラーは、シリアライズされたノードの復元失敗を招きます。

保守担当者向けメモ

  • pg_node_attr(custom_copy_equal, custom_read_write) が指定されており、標準的なノード処理ではなく拡張機能側での実装が強制されます。
  • 拡張ノード名は EXTNODENAME_MAX_LEN（64バイト）以内で一意である必要があります。
