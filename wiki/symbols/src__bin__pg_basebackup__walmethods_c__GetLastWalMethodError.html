GetLastWalMethodError

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/walmethods.c:1382

目的

WalWriteMethod インスタンスで発生した直近のエラー内容を、人間が読み取り可能な文字列として取得します。

シグネチャまたは形状

const char *GetLastWalMethodError(WalWriteMethod *wwmethod);

入力と出力

  • **入力**: WalWriteMethod *wwmethod（WAL 書き込みメソッドのコンテキスト構造体）
  • **出力**: エラー内容を示す const char * 型の文字列。

前提条件と不変条件

  • 引数 wwmethod は有効なポインタである必要があります。
  • wwmethod 内部の lasterrstring または lasterrno が適切に設定されていることを前提とします。

アルゴリズムまたは振る舞い

1. wwmethod->lasterrstring が設定されている（NULL でない）場合、その文字列を優先して返します。 2. lasterrstring が設定されていない場合、wwmethod->lasterrno を strerror() に渡し、システムエラーメッセージを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/receivelog.c 内の複数の箇所から、WAL 受信処理中のエラーログ出力のために呼び出されています。
  • **呼び出し先**: 標準ライブラリの strerror() を呼び出します。

関連データ構造

  • WalWriteMethod: エラー状態（lasterrstring, lasterrno）を保持する構造体。

失敗モードと境界条件

  • wwmethod が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性があります。
  • lasterrno が有効なエラー番号でない場合、strerror() の実装に依存した文字列が返されます。

保守担当者向けメモ

  • 本関数は、lasterrstring が設定されている場合はそれが優先されるという仕様です。エラーハンドリングの際は、この優先順位を考慮してください。
