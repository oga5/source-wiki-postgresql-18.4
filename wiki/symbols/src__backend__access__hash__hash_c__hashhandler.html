hashhandler

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:57

目的

PostgreSQL のハッシュインデックスアクセスメソッド（AM）をシステムに登録するためのエントリポイントであり、インデックス操作に必要なコールバック関数群を定義した IndexAmRoutine 構造体を生成・返却します。

シグネチャまたは形状

Datum hashhandler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（標準的な PostgreSQL 関数呼び出し規約）
  • **出力**: Datum 型としてキャストされた IndexAmRoutine* 構造体へのポインタ。

前提条件と不変条件

  • makeNode(IndexAmRoutine) によりメモリが確保されること。
  • ハッシュインデックスの特性（amcanhash = true, amcanorder = false 等）が適切に設定されていること。

アルゴリズムまたは振る舞い

1. IndexAmRoutine 構造体を初期化します。 2. ハッシュインデックスの機能フラグ（検索能力、並列処理可否、ソート順序の有無など）を設定します。 3. インデックスのライフサイクルを管理する関数ポインタ（hashbuild, hashinsert, hashgettuple, hashbulkdelete 等）を構造体に割り当てます。 4. 設定済みの構造体を PG_RETURN_POINTER マクロで返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックス AM ロード機構（詳細は提供されたソース範囲外）。
  • **呼び出し先**: makeNode（構造体生成）、および hashbuild, hashinsert, hashbulkdelete, hashvacuumcleanup, hashcostestimate, hashoptions, hashvalidate, hashadjustmembers, hashbeginscan, hashrescan, hashgettuple, hashgetbitmap, hashendscan, hashtranslatestrategy, hashtranslatecmptype（各インデックス操作の実装関数）。

関連データ構造

  • IndexAmRoutine: インデックスアクセスメソッドのルーチンテーブル。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数はハッシュインデックスのインターフェース定義です。機能の追加や変更を行う際は、IndexAmRoutine の各フィールドが適切に設定されているか確認してください。
  • amcanorder や amcanmulticol など、ハッシュインデックスがサポートしない機能が false に設定されていることを維持してください。
