pg_GSS_load_servicename

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-gssapi-common.c:81

目的

GSSAPI認証で使用するサービスプリンシパル名を構築し、GSSAPI内部形式（gss_name_t）へ変換して PGconn 構造体に格納する。

シグネチャまたは形状

int pg_GSS_load_servicename(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn（接続情報。conn->krbsrvname および接続先ホスト名を使用）
  • **出力**: 成功時に STATUS_OK (0)、失敗時に STATUS_ERROR (-1) を返す。

前提条件と不変条件

  • conn->gtarg_nam が既に設定されている場合、本関数は何もせず STATUS_OK を返す（冪等性）。
  • 接続先ホスト名が PQhost(conn) で取得可能であること。
  • ENABLE_GSS マクロが定義されている環境でのみ有効。

アルゴリズムまたは振る舞い

1. conn->gtarg_nam が非NULLであれば即座に終了する。 2. PQhost(conn) を呼び出し、ホスト名が空でないことを確認する。空の場合はエラーを記録して終了する。 3. krbsrvname とホスト名を結合し、service@host 形式の文字列を生成する。 4. gss_import_name を呼び出し、生成した文字列を GSS_C_NT_HOSTBASED_SERVICE 形式で conn->gtarg_nam にインポートする。 5. インポートに失敗した場合は pg_GSS_error を呼び出し、エラー情報を conn に記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-auth.c
  • src/interfaces/libpq/fe-secure-gssapi.c
  • **呼び出し先**:
  • PQhost
  • libpq_append_conn_error
  • gss_import_name (GSSAPIライブラリ)
  • pg_GSS_error

関連データ構造

  • PGconn: 接続状態を保持する構造体。gtarg_nam（GSSAPIサービス名）および krbsrvname（Kerberosサービス名）を格納。
  • gss_buffer_desc: GSSAPIへ渡す文字列バッファ。

失敗モードと境界条件

  • **メモリ不足**: malloc が失敗した場合、libpq_append_conn_error を経由してエラーを報告し STATUS_ERROR を返す。
  • **ホスト名未指定**: PQhost が空文字列を返す場合、エラーを報告し STATUS_ERROR を返す。
  • **GSSAPIインポート失敗**: gss_import_name が GSS_S_COMPLETE 以外を返した場合、pg_GSS_error で詳細を記録し STATUS_ERROR を返す。

保守担当者向けメモ

  • 本関数は conn->gtarg_nam のメモリ管理を担うため、呼び出し側は conn の破棄時に適切に解放されることを前提とする。
  • gss_import_name 実行後に一時バッファは free される。
