GinStatsData

定義

  • 種別: struct
  • 定義箇所: src/include/access/gin.h:55

目的

GIN（汎用転置索引）インデックスの統計情報を保持し、プランナがコスト見積もりを行う際に利用するためのデータ構造です。

シグネチャまたは形状

typedef struct GinStatsData
{
	BlockNumber nPendingPages;
	BlockNumber nTotalPages;
	BlockNumber nEntryPages;
	BlockNumber nDataPages;
	int64		nEntries;
	int32		ginVersion;
} GinStatsData;

入力と出力

本構造体は、インデックスのメタデータとして ginGetStats や ginUpdateStats を通じて pg_class 等のシステムカタログと同期されます。

前提条件と不変条件

  • BlockNumber 型のフィールドは、インデックス内の各ページ種別のカウントを保持します。
  • nEntries はインデックス内のエントリ総数を表します。

アルゴリズムまたは振る舞い

プランナは本構造体の各フィールドを参照し、インデックスのサイズやエントリ密度を計算することで、クエリ実行時のコスト見積もりを最適化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gininsert.c（gininsert 等での統計更新）、src/backend/access/gin/ginbtree.c、src/backend/access/gin/gindatapage.c など、インデックスの構築や更新を行う関数群から参照・更新されます。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • BlockNumber: ページ番号を管理するために使用されます。
  • gin_private.h: GIN の内部的なページレイアウト定義が含まれており、本構造体の値と整合性を保つ必要があります。

失敗モードと境界条件

  • インデックスが空の場合、各ページ数は 0 となります。
  • ginVersion はインデックスの互換性を判定するために使用され、バージョン不一致が発生した場合はアクセス時にエラーとなる可能性があります。

保守担当者向けメモ

  • 本構造体は GIN の公開インターフェースの一部です。フィールドの追加や変更を行う際は、プランナのコスト見積もりロジックへの影響を考慮してください。
  • ginGetStats および ginUpdateStats の実装と密接に関連しているため、統計情報の更新フローを変更する場合はこれらの関数を確認してください。
