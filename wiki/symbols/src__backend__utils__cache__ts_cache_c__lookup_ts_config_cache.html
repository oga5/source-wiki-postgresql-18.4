lookup_ts_config_cache

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/cache/ts_cache.c:384

目的

全文検索設定（pg_ts_config）のキャッシュエントリを取得します。キャッシュが存在しない場合はシステムカタログから情報を読み込み、メモリ上にエントリを構築して返します。

シグネチャまたは形状

TSConfigCacheEntry *lookup_ts_config_cache(Oid cfgId)

入力と出力

  • **入力**: cfgId (Oid) - 取得対象の全文検索設定のOID。
  • **出力**: TSConfigCacheEntry * - キャッシュされた設定情報へのポインタ。

前提条件と不変条件

  • TSConfigCacheHash が初期化されていない場合、内部で init_ts_config_cache() が呼び出されます。
  • キャッシュエントリの isvalid フラグが false の場合、再構築が行われます。
  • 辞書マッピング情報は CacheMemoryContext 上に割り当てられます。

アルゴリズムまたは振る舞い

1. **高速キャッシュ確認**: lastUsedConfig が要求された cfgId と一致し、かつ有効であれば即座に返します。 2. **ハッシュ検索**: TSConfigCacheHash を検索し、存在すればそのエントリを返します。 3. **キャッシュミス時の構築**:

  • pg_ts_config システムカタログから設定情報を取得します。
  • pg_ts_config_map をインデックス順にスキャンし、トークンタイプごとの辞書リストを構築します。
  • 構築したデータを CacheMemoryContext に確保し、エントリを更新します。

    • 4. **更新**: 最後にアクセスしたエントリとして lastUsedConfig を更新し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_parse.c
  • src/backend/tsearch/wparser.c
  • **呼び出し先**:
  • init_ts_config_cache()
  • hash_search()
  • SearchSysCache1()
  • systable_beginscan_ordered() 等のシステムカタログスキャン関数

関連データ構造

  • TSConfigCacheEntry: キャッシュエントリの構造体。
  • ListDictionary: トークンタイプごとの辞書リストを保持する構造体。
  • CacheMemoryContext: キャッシュデータが割り当てられるメモリコンテキスト。

失敗モードと境界条件

  • **エラー**: 指定された cfgId がシステムカタログに存在しない場合、elog(ERROR) で停止します。
  • **エラー**: 設定にパーサーが定義されていない場合、エラーとなります。
  • **エラー**: maptokentype が範囲外（MAXTOKENTYPE 超過）の場合、エラーとなります。
  • **エラー**: 1つのトークンタイプに対する辞書数が MAXDICTSPERTT を超える場合、エラーとなります。

保守担当者向けメモ

  • キャッシュエントリはバックエンドの生存期間中保持されます。
  • InvalidateTSCacheCallBack によって isvalid が false に設定されると、次回の呼び出し時に再構築が強制されます。
  • map データのメモリ解放は、再構築時の既存エントリ更新処理内で行われます。
