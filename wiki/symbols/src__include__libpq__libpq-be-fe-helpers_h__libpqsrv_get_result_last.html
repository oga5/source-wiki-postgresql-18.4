libpqsrv_get_result_last

定義

  • 種別: function
  • 定義箇所: src/include/libpq/libpq-be-fe-helpers.h:289

目的

PostgreSQL バックエンド内で libpq を使用する際、クエリ実行結果のストリームを最後まで消費し、最後の PGresult を取得する。PQexec の挙動を模倣し、複数の結果が返される場合に最後の結果を返すためのヘルパー関数。

シグネチャまたは形状

PGresult *libpqsrv_get_result_last(PGconn *conn, uint32 wait_event_info)

入力と出力

  • **入力**:
  • conn: PGconn 接続オブジェクト。
  • wait_event_info: 待機イベント情報（WaitLatchOrSocket で使用）。
  • **出力**:
  • 最後に取得された PGresult へのポインタ。クエリが完了した際、またはエラー発生時に返される。

前提条件と不変条件

  • PG_TRY/PG_CATCH ブロックにより、エラー発生時には保持している PGresult を確実に解放（PQclear）する。
  • 呼び出し元は、返された PGresult を最終的に解放する責任を持つ。

アルゴリズムまたは振る舞い

1. libpqsrv_get_result を繰り返し呼び出し、結果を順次取得する。 2. 取得した結果を lastResult に保持し、新しい結果が来るたびに古い lastResult を PQclear で破棄する。 3. 以下のいずれかの条件を満たした場合、ループを終了する。

  • libpqsrv_get_result が NULL を返した（クエリ完了または失敗）。
  • PQresultStatus が PGRES_COPY_IN, PGRES_COPY_OUT, PGRES_COPY_BOTH のいずれかである。
  • PQstatus(conn) が CONNECTION_BAD である。

    • 4. エラー発生時は PG_CATCH ブロックで lastResult をクリアし、例外を再スローする。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • libpqsrv_get_result
  • PQclear
  • PQresultStatus
  • PQstatus
  • **呼び出し元**:
  • contrib/postgres_fdw/connection.c
  • src/include/libpq/libpq-be-fe-helpers.h 内の他のヘルパー関数（libpqsrv_exec 等の内部実装から参照される）。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。
  • PGresult: クエリ実行結果を保持する構造体。

失敗モードと境界条件

  • 接続が切断されている場合（CONNECTION_BAD）、ループを即座に終了する。
  • COPY モードに遷移した場合、それ以上の結果取得を停止し、現在の結果を返して終了する。
  • PG_TRY ブロック内でエラーが発生した場合、メモリリークを防ぐために lastResult をクリアする。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結している。
  • libpqsrv_get_result が内部で CHECK_FOR_INTERRUPTS() を呼び出すため、長時間実行されるクエリでも割り込み応答性が維持される。
