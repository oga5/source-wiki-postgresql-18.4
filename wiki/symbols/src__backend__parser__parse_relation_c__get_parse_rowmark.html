get_parse_rowmark

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:3564

目的

指定されたレンジテーブルインデックス（rtindex）に対応する RowMarkClause をクエリ構造体から検索して取得する。

シグネチャまたは形状

RowMarkClause *get_parse_rowmark(Query *qry, Index rtindex)

入力と出力

  • **入力**:
  • qry: 検索対象の Query 構造体へのポインタ。
  • rtindex: 検索対象のレンジテーブルインデックス（Index 型）。
  • **出力**:
  • 見つかった場合は対応する RowMarkClause 構造体へのポインタ。
  • 見つからない場合は NULL。

前提条件と不変条件

  • qry は有効な Query 構造体を指している必要がある。
  • qry->rowMarks は RowMarkClause を要素とするリストである。

アルゴリズムまたは振る舞い

1. qry->rowMarks リストを先頭から順に走査する。 2. 各要素（RowMarkClause）の rti フィールドが、引数で渡された rtindex と一致するかを確認する。 3. 一致する要素が見つかった時点でそのポインタを返し、リストの末尾まで見つからなければ NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c
  • src/backend/rewrite/rewriteHandler.c
  • **呼び出し先**:
  • 本関数はリスト走査のみを行い、他の関数を呼び出さない。

関連データ構造

  • Query: クエリ全体を表現する構造体。rowMarks リストを保持する。
  • RowMarkClause: 行ロック（FOR UPDATE等）の情報を保持する構造体。rti フィールドで対象のレンジテーブルを特定する。

失敗モードと境界条件

  • qry->rowMarks が空の場合、常に NULL を返す。
  • 指定された rtindex が rowMarks リスト内に存在しない場合、NULL を返す。

保守担当者向けメモ

  • 本関数は単純なリスト検索を行うユーティリティである。
  • src/include/parser/parsetree.h に外部宣言が存在する。
