test_single_value_and_filler

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_integerset/test_integerset.c:372

目的

特定の単一値と、指定された範囲の連続する整数（フィラー）を IntegerSet に追加し、その整合性を検証するテスト関数です。

シグネチャまたは形状

static void test_single_value_and_filler(uint64 value, uint64 filler_min, uint64 filler_max)

入力と出力

  • **入力**:
  • value: 集合に追加する単一の整数値。
  • filler_min: 連続する整数の開始値（含む）。
  • filler_max: 連続する整数の終了値（含まない）。
  • **出力**: なし（検証失敗時は elog(ERROR) を発行）。

前提条件と不変条件

  • filler_min < filler_max であること。
  • IntegerSet は内部的に Simple-8b エンコーディング等の制約を持つため、追加する値の範囲がこれに適合していること。

アルゴリズムまたは振る舞い

1. intset_create() で新しい集合を作成。 2. value が filler_min 未満であれば先に value を追加し、それ以外の場合は filler 範囲の追加後に value を追加する。 3. intset_num_entries() を呼び出し、追加した要素数と一致するか確認。 4. check_with_filler() を用いて、境界値（0, 1, filler_min 前後, value 前後, filler_max 前後, PG_UINT64_MAX）の存在確認を行う。 5. intset_begin_iterate() および intset_iterate_next() を使用し、全要素が期待通りの順序で取得できるか確認。 6. intset_memory_usage() を呼び出し、メモリ使用量が異常値（5,000〜500,000,000の範囲外）でないか確認。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_integerset/test_integerset.c 内のテスト実行ロジック（112〜116行目付近）。
  • **呼び出し先**: intset_create, intset_add_member, intset_num_entries, check_with_filler, intset_begin_iterate, intset_iterate_next, intset_memory_usage。

関連データ構造

  • IntegerSet: テスト対象の整数集合データ構造。

失敗モードと境界条件

  • intset_num_entries が期待値と一致しない場合にエラー。
  • intset_iterate_next による走査で値の不一致や予期せぬ終了が発生した場合にエラー。
  • intset_memory_usage が異常な範囲を返した場合にエラー。
  • value が filler 範囲と重複する場合や、PG_UINT64_MAX 付近の極端な値での動作が検証対象となる。

保守担当者向けメモ

  • 本関数は check_with_filler に強く依存しています。実装変更時は、check_with_filler 側の検証ロジックも併せて確認してください。
  • mem_usage のチェック範囲は経験的な値であるため、将来的な実装変更でメモリ効率が大幅に改善・悪化した場合は調整が必要です。
