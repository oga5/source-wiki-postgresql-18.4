CheckConnection

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:353

目的

データベース接続の状態を確認し、切断されている場合は再接続を試みます。

シグネチャまたは形状

bool CheckConnection(void)

入力と出力

  • **入力**: なし（グローバル変数 pset を介して現在の接続状態を参照）
  • **出力**: 接続が確立されている（または再接続に成功した）場合は true、それ以外の場合は false を返します。

前提条件と不変条件

  • pset.db が有効な接続ハンドルであるか、あるいは NULL であることを前提とします。
  • 非対話モードで接続が失われた場合、この関数は呼び出し元に戻らずにプロセスを終了します。

アルゴリズムまたは振る舞い

1. ConnectionUp() を呼び出し、現在の接続状態を確認します。 2. 接続が切断されている場合：

  • 対話モードであれば、ユーザーに再接続を試みる旨を通知し、PQreset(pset.db) および pipelineReset() を実行します。
  • 再接続後、再度 ConnectionUp() で確認します。
  • 再接続に失敗した場合は、現在の接続を pset.dead_conn に退避し、pset.db を NULL に設定して、ResetCancelConn() および UnsyncVariables() を実行します。
  • 再接続に成功した場合は、SyncVariables() を呼び出し、接続警告を再設定します。
  • 非対話モードであれば、エラーログを出力して EXIT_BADCONN で終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/common.c 内の複数箇所（行 457, 1707, 1787, 2187 等）から呼び出されています。
  • **呼び出し先**: ConnectionUp, PQreset, pipelineReset, PQfinish, ResetCancelConn, UnsyncVariables, SyncVariables, connection_warnings。

関連データ構造

  • pset (グローバル変数): 接続状態 (db, dead_conn) や対話モードフラグ (cur_cmd_interactive) を保持。

失敗モードと境界条件

  • **非対話モードでの切断**: pg_log_error を出力し、exit(EXIT_BADCONN) によりプロセスが終了します。
  • **再接続失敗**: 接続が完全に失われたと判断し、変数の同期解除と接続ハンドルのクリーンアップが行われます。

保守担当者向けメモ

  • do_connect() と同様の接続管理ロジックを維持する必要があります。
  • SyncVariables() や UnsyncVariables() の呼び出し順序は、接続状態の整合性に直結するため変更時に注意してください。
