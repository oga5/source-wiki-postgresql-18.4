pg_popcount

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_bitutils.h:362

目的

バッファ内の各バイトに含まれる「1」のビット数を合計し、全体の個数を算出する。

シグネチャまたは形状

static inline uint64 pg_popcount(const char *buf, int bytes)

入力と出力

  • **入力**:
  • buf: カウント対象のデータが格納されたバッファへのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • **出力**:
  • バッファ内の「1」のビット数の総計を uint64 型で返す。

前提条件と不変条件

  • bytes が負の値である場合の挙動は定義されていない。
  • 閾値（SIZEOF_VOID_P が 8 以上なら 8、それ未満なら 4）を境界として、処理経路が切り替わる。

アルゴリズムまたは振る舞い

1. 実行環境のポインタサイズに基づき、閾値（4 または 8）を決定する。 2. bytes が閾値未満の場合:

  • pg_number_of_ones ルックアップテーブルを使用し、ループで各バイトのビット数を加算する。

    • 3. bytes が閾値以上の場合:

  • pg_popcount_optimized を呼び出し、ハードウェア命令や SIMD を活用した高速なカウント処理へ委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_intbig_gist.c
  • contrib/ltree/_ltree_gist.c
  • contrib/pageinspect/heapfuncs.c
  • contrib/pg_trgm/trgm_gist.c
  • contrib/pg_walinspect/pg_walinspect.c
  • src/backend/access/brin/brin_bloom.c
  • src/backend/lib/bloomfilter.c
  • src/backend/utils/adt/tsgistidx.c
  • src/backend/utils/adt/varbit.c
  • src/backend/utils/adt/varlena.c
  • **呼び出し先**:
  • pg_popcount_optimized（閾値以上のデータサイズの場合）

関連データ構造

  • pg_number_of_ones[256]: 0-255 の各値に対する 1 のビット数を保持するルックアップテーブル。

失敗モードと境界条件

  • bytes が 0 の場合、ループは実行されず 0 を返す。
  • 閾値判定により、非常に小さなバッファに対してはルックアップテーブルによる逐次処理が選択され、オーバーヘッドが抑制される。

保守担当者向けメモ

  • pg_popcount_optimized は関数ポインタ経由で呼び出される場合があるため、プラットフォームごとの最適化実装を追加する際は src/port/pg_bitutils.c との整合性に注意すること。
  • intrin.h 等のプラットフォーム依存ヘッダーとの依存関係があるため、移植時はコンパイル時判定マクロの確認が必要。
