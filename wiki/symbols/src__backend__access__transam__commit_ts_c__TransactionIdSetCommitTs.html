TransactionIdSetCommitTs

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/commit_ts.c:248

目的

指定されたトランザクションID（XID）に対応するコミット時刻およびレプリケーション元ノードIDを、SLRU（Simple Least Recently Used）キャッシュ上の指定スロットへ書き込みます。

シグネチャまたは形状

static void TransactionIdSetCommitTs(TransactionId xid, TimestampTz ts, RepOriginId nodeid, int slotno)

入力と出力

  • **入力**:
  • xid: 書き込み対象のトランザクションID。
  • ts: 記録するコミット時刻（TimestampTz）。
  • nodeid: レプリケーション元ノードID（RepOriginId）。
  • slotno: 書き込み先となるSLRUバッファのスロット番号。
  • **出力**: なし（副作用として共有メモリ上のSLRUページを更新）。

前提条件と不変条件

  • xid は TransactionIdIsNormal(xid) を満たす正常なトランザクションIDである必要があります。
  • 呼び出し側は、指定された slotno に対応するSLRUページに対して適切なロックを保持している必要があります。

アルゴリズムまたは振る舞い

1. TransactionIdToCTsEntry を使用して、対象 xid がページ内のどのエントリ位置（entryno）に該当するかを算出します。 2. CommitTimestampEntry 構造体を初期化し、引数で受け取った ts と nodeid を格納します。 3. CommitTsCtl->shared->page_buffer の指定された slotno のバッファに対し、算出したオフセット位置へ memcpy を用いてデータをコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/commit_ts.c 内の SetXidCommitTsInPage 関数から呼び出されます。
  • **呼び出し先**:
  • TransactionIdToCTsEntry（マクロまたはインライン関数と推測されるが、ソース本文には定義なし）。
  • memcpy（標準ライブラリ）。

関連データ構造

  • CommitTimestampEntry: コミット時刻とノードIDを保持するデータ構造。
  • CommitTsCtl: SLRU管理用の制御構造体。本関数はこれに含まれる共有メモリ上の page_buffer を直接操作します。

失敗モードと境界条件

  • xid が正常なトランザクションIDでない場合、Assert により異常終了します。
  • slotno が有効なバッファ範囲外である場合、メモリ破壊を引き起こす可能性があります（本関数内での境界チェックは行われていません）。

保守担当者向けメモ

  • 本関数は static であり、モジュール内部でのみ使用されます。
  • CommitTsCtl を介した共有メモリへの直接書き込みを行うため、呼び出し元で適切な排他制御（SLRUロック）がなされていることを前提としています。変更を加える際は、SLRUのロックプロトコルが維持されているか確認してください。
