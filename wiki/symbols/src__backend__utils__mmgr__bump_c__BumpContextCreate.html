BumpContextCreate

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/bump.c:130

目的

バンプアロケータ方式の新しいメモリコンテキスト（BumpContext）を初期化し、最初のメモリブロックを確保する。

シグネチャまたは形状

MemoryContext BumpContextCreate(MemoryContext parent, const char *name, Size minContextSize, Size initBlockSize, Size maxBlockSize)

入力と出力

  • **入力**:
  • parent: 親メモリコンテキスト。
  • name: コンテキスト名（デバッグ用）。
  • minContextSize: 最小コンテキストサイズ。
  • initBlockSize: 初期ブロックサイズ。
  • maxBlockSize: 最大ブロックサイズ。
  • **出力**:
  • 初期化された MemoryContext へのポインタ。失敗時は ereport(ERROR) により終了する。

前提条件と不変条件

  • initBlockSize は MAXALIGN 済みかつ 1024 バイト以上であること。
  • maxBlockSize は MAXALIGN 済み、initBlockSize 以上、かつ MEMORYCHUNK_MAX_BLOCKOFFSET 以下であること。
  • minContextSize が指定される場合、MAXALIGN 済み、1024 バイト以上、かつ maxBlockSize 以下であること。

アルゴリズムまたは振る舞い

1. 引数の妥当性を検証する。 2. BumpContext 構造体と最初のブロックを保持するためのメモリを malloc で確保する。 3. dlist_init でブロックリストを初期化する。 4. KeeperBlock として最初のブロックを初期化し、リストに追加する。 5. コンテキスト固有のフィールド（initBlockSize, maxBlockSize, allocChunkLimit 等）を設定する。 6. MemoryContextCreate を呼び出し、汎用的なコンテキスト管理構造を構築する。 7. 確保したメモリサイズを記録し、コンテキストポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tidstore.c
  • src/backend/executor/nodeAgg.c
  • src/backend/utils/sort/tuplesort.c
  • **呼び出し先**:
  • malloc
  • BumpBlockInit
  • dlist_init / dlist_push_head
  • MemoryContextCreate
  • ereport

関連データ構造

  • BumpContext: バンプアロケータのメタデータ構造体。
  • BumpBlock: メモリ割り当ての単位となるブロック構造体。
  • MemoryContext: 基底となるメモリコンテキスト構造体。

失敗モードと境界条件

  • malloc が失敗した場合、MemoryContextStats を出力し、ERRCODE_OUT_OF_MEMORY でエラー終了する。
  • maxBlockSize が MEMORYCHUNK_MAX_BLOCKOFFSET を超える場合、アサーション違反となる。

保守担当者向けメモ

  • 本関数は BumpContext のヘッダーと最初のブロックを同一の malloc 領域に配置する特殊な構造をとる。
  • allocChunkLimit の計算は、チャンクヘッダーのオーバーヘッドと Bump_CHUNK_FRACTION に基づく制約を考慮している。
  • MemoryContextCreate 呼び出し以降は失敗できないため、それ以前にメモリ確保等の失敗リスクを処理している。
