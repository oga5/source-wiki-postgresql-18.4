ExecParallelHashTableAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:3288

目的

並列ハッシュ結合において、指定されたバッチ番号に対応する共有メモリ上のバケット配列を割り当て、初期化する。

シグネチャまたは形状

void ExecParallelHashTableAlloc(HashJoinTable hashtable, int batchno)

入力と出力

  • **入力**:
  • hashtable: ハッシュ結合の全状態を保持する HashJoinTable 構造体。
  • batchno: 割り当て対象のバッチ番号。
  • **出力**: なし（副作用として共有メモリ領域が確保・初期化される）。

前提条件と不変条件

  • hashtable->area（DSA: Dynamic Shared Area）が有効であること。
  • hashtable->parallel_state->nbuckets が設定済みであること。
  • 指定された batchno に対応する共有バッチ構造体が hashtable->batches 内に存在すること。

アルゴリズムまたは振る舞い

1. hashtable->batches[batchno].shared から対象のバッチ構造体を取得する。 2. hashtable->area を使用して、nbuckets 分の dsa_pointer_atomic 配列を共有メモリ上に確保する。 3. 確保した領域の先頭アドレスを batch->buckets に格納する。 4. 確保した各バケットポインタを InvalidDsaPointer で初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHash.c:612
  • src/backend/executor/nodeHashjoin.c:1315
  • **呼び出し先**:
  • dsa_allocate (共有メモリ領域の確保)
  • dsa_get_address (確保した領域のアドレス解決)
  • dsa_pointer_atomic_init (アトミックポインタの初期化)

関連データ構造

  • HashJoinTable: ハッシュ結合の制御ブロック。
  • ParallelHashJoinBatch: 並列実行時のバッチ状態を保持する構造体。
  • dsa_pointer_atomic: 共有メモリ上のアトミックなポインタ型。

失敗モードと境界条件

  • dsa_allocate がメモリ不足により失敗した場合、DSA 関連の例外が発生する可能性がある。
  • batchno が範囲外の場合、メモリ破壊やセグメンテーション違反を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は並列実行時の共有メモリ管理に依存している。DSA のライフサイクル管理と整合性を保つ必要がある。
  • nbuckets は並列実行開始時に決定されるため、本関数呼び出し時には既に確定している必要がある。
