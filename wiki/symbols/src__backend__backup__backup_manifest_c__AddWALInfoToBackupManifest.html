AddWALInfoToBackupManifest

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/backup/backup_manifest.c:212

目的

バックアップ・マニフェストに、バックアップ期間中に生成されたWAL（Write-Ahead Log）のタイムライン履歴およびLSN範囲情報をJSON形式で追加します。

シグネチャまたは形状

void AddWALInfoToBackupManifest(backup_manifest_info *manifest, XLogRecPtr startptr,
                                TimeLineID starttli, XLogRecPtr endptr,
                                TimeLineID endtli)

入力と出力

  • **入力**:
  • manifest: マニフェスト生成状態を保持する構造体。
  • startptr, starttli: バックアップ開始時のLSNとタイムラインID。
  • endptr, endtli: バックアップ終了時のLSNとタイムラインID。
  • **出力**: なし（副作用として manifest が管理する一時ファイルへJSON文字列を追記）。

前提条件と不変条件

  • IsManifestEnabled(manifest) が真であること。
  • endtli に対応するタイムライン履歴ファイルが読み取り可能であること。
  • タイムライン履歴の解析結果に、開始時の starttli が含まれていること。

アルゴリズムまたは振る舞い

1. マニフェスト内のファイルリストを終了し、"WAL-Ranges" キーを開始します。 2. readTimeLineHistory(endtli) を呼び出し、終了時点のタイムライン履歴を取得します。 3. 履歴を逆順（新しい順）に走査し、バックアップ期間に関連するタイムラインを特定します。 4. 各タイムラインについて、開始LSNと終了LSNを計算します。

  • 開始タイムラインと一致する場合は startptr を開始点とします。
  • それ以外の場合は、タイムラインの開始LSNを使用します。

    • 5. 計算した範囲情報をJSON形式でマニフェストに追記します。 6. starttli に到達した時点でループを終了し、"WAL-Ranges" リストを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:647
  • **呼び出し先**:
  • IsManifestEnabled
  • AppendStringToManifest
  • readTimeLineHistory
  • AppendToManifest
  • ereport (エラー発生時)

関連データ構造

  • backup_manifest_info: マニフェスト生成のコンテキストを保持。
  • TimeLineHistoryEntry: タイムライン履歴の各エントリ。

失敗モードと境界条件

  • **エラー**:
  • 期待される終了タイムラインと履歴上のタイムラインが一致しない場合。
  • タイムライン履歴中に開始タイムラインが見つからない場合。
  • タイムラインの開始LSNが不正（InvalidXLogRecPtr）な場合。
  • **境界条件**: バックアップが単一のタイムライン内で完結する場合や、複数のタイムラインを跨ぐ場合の両方に対応します。

保守担当者向けメモ

  • JSONの構文（カンマの制御）は first_wal_range フラグによって手動で管理されています。
  • タイムライン履歴ファイルは新しい順に並んでいるため、ループ処理の順序とLSNの更新ロジックに依存関係があります。
