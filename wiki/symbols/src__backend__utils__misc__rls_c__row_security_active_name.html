row_security_active_name

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/rls.c:152

目的

SQLインターフェースを通じて、指定されたテーブル名に対して行レベルセキュリティ（RLS）が現在有効であるかを判定します。

シグネチャまたは形状

Datum row_security_active_name(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、テーブル名を指す text 型の引数を1つ受け取ります。
  • **出力**: bool 型の Datum を返します。RLSが有効（RLS_ENABLED）であれば true を、それ以外であれば false を返します。

前提条件と不変条件

  • 入力されたテーブル名は、現在の検索パスに基づいて解決可能である必要があります。
  • テーブルのロックは取得されません（NoLock）。

アルゴリズムまたは振る舞い

1. textToQualifiedNameList および makeRangeVarFromNameList を使用して、入力されたテキストから RangeVar を構築します。 2. RangeVarGetRelid を呼び出し、テーブルのOIDを取得します。 3. check_enable_rls(tableoid, InvalidOid, true) を呼び出し、RLSの適用状態を判定します。 4. 判定結果が RLS_ENABLED と一致するかを評価し、その結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジンから直接呼び出されることを想定しています。
  • **呼び出し先**:
  • makeRangeVarFromNameList / textToQualifiedNameList
  • RangeVarGetRelid
  • check_enable_rls

関連データ構造

  • RangeVar: テーブル名を解決するために使用される構造体。
  • CheckEnableRlsResult (列挙型): check_enable_rls が返す判定結果。

失敗モードと境界条件

  • 指定されたテーブル名が存在しない場合、RangeVarGetRelid によりエラーが発生します。
  • 権限不足によりテーブルのOIDを取得できない場合、エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は check_enable_rls のラッパーであり、テーブル名からOIDへの解決を担います。
  • 実際のRLS適用ロジックは check_enable_rls に依存しているため、判定基準の変更はそちらを確認してください。
