scram_state_enum

定義

  • 種別: enum
  • 定義箇所: src/backend/libpq/auth-scram.c:126

目的

PostgreSQLのサーバー側におけるSASL SCRAM-SHA-256認証セッションの状態を管理するために使用されます。

シグネチャまたは形状

typedef enum
{
	SCRAM_AUTH_INIT,
	SCRAM_AUTH_SALT_SENT,
	SCRAM_AUTH_FINISHED,
} scram_state_enum;

入力と出力

  • 本シンボル自体は状態定数の集合であり、入力や出力は持ちません。
  • この列挙型は、認証セッションの状態遷移を追跡する変数（scram_state 構造体内のメンバなど）の型として利用されます。

前提条件と不変条件

  • 認証プロセスは SCRAM_AUTH_INIT から開始されます。
  • 認証が正常に完了または終了した状態は SCRAM_AUTH_FINISHED で表されます。

アルゴリズムまたは振る舞い

  • SCRAM_AUTH_INIT: 初期化直後の状態。
  • SCRAM_AUTH_SALT_SENT: サーバーがクライアントに対してnonceとsaltを送信した後の状態。
  • SCRAM_AUTH_FINISHED: クライアントからのProof検証が完了し、認証プロセスが終了した状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/libpq/auth-scram.c 内の認証ロジックが、セッション状態を保持する変数（scram_state 構造体）の型として参照しています。
  • 呼び出し先: 本シンボルは列挙型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • scram_state: 本列挙型をメンバとして保持し、認証セッション全体のコンテキストを管理する構造体。

失敗モードと境界条件

  • 状態遷移の順序が不正な場合（例: INIT から直接 FINISHED へ移行するなど）、認証ロジック内でエラーとして扱われる可能性があります。

保守担当者向けメモ

  • 認証プロトコルのステップを追加または変更する場合は、本列挙型に新しい状態を追加し、scram_exchange 内の遷移ロジックを更新してください。
