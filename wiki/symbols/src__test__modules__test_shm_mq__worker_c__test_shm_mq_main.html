test_shm_mq_main

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/worker.c:46

目的

PostgreSQLのバックグラウンドワーカーとして、共有メモリメッセージキューを介したデータ転送処理を実行するエントリポイント。

シグネチャまたは形状

void test_shm_mq_main(Datum main_arg)
  • main_arg: 動的共有メモリ（DSM）セグメントのID。

入力と出力

  • **入力**: main_arg（DSMセグメントID）。
  • **出力**: なし（プロセス終了により完了）。
  • **副作用**: 共有メモリへのアタッチ、メッセージキューの読み書き、メインプロセスへのLatch通知、プロセス終了。

前提条件と不変条件

  • main_arg で指定されたDSMセグメントが有効であること。
  • DSM内に PG_TEST_SHM_MQ_MAGIC で識別される shm_toc が存在すること。
  • キー 0 に test_shm_mq_header 構造体が配置されていること。

アルゴリズムまたは振る舞い

1. **初期化**: SIGTERM シグナルハンドラを設定し、BackgroundWorkerUnblockSignals() を呼び出す。 2. **DSM接続**: dsm_attach で共有メモリに接続し、shm_toc_attach で制御構造を取得する。 3. **ワーカー登録**: shm_toc からヘッダーを取得し、workers_attached をインクリメントして自身のワーカー番号を確定する。 4. **キュー接続**: attach_to_queues を呼び出し、入力用および出力用のメッセージキューを特定・接続する。 5. **準備完了通知**: workers_ready をインクリメントし、登録元プロセスのLatchをセットして待機解除を通知する。 6. **データ処理**: copy_messages を呼び出し、メッセージの送受信ループを実行する。 7. **終了**: dsm_detach を実行し、proc_exit(1) でプロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのバックグラウンドワーカー起動プロセス（src/test/modules/test_shm_mq/setup.c にて関数名が登録される）。
  • **呼び出し先**:
  • dsm_attach, shm_toc_attach, shm_toc_lookup, attach_to_queues, copy_messages, SetLatch, dsm_detach, proc_exit 等。

関連データ構造

  • dsm_segment: 動的共有メモリセグメント。
  • shm_toc: 共有メモリ内のテーブル・オブ・コンテンツ。
  • test_shm_mq_header: 共有メモリ上の制御構造（workers_attached, workers_total, workers_ready を保持）。
  • shm_mq_handle: メッセージキューのハンドル。

失敗モードと境界条件

  • DSMセグメントのマップ失敗: ERROR をスロー。
  • マジックナンバー不一致: ERROR をスロー。
  • ワーカー数が上限を超過: ERROR をスロー。
  • 登録元プロセスが既に終了している場合: proc_exit(1) で終了。

保守担当者向けメモ

  • 本関数はバックグラウンドワーカーのテンプレートとして機能する。
  • ループ内の copy_messages は CHECK_FOR_INTERRUPTS() を含んでおり、シグナルによる停止要求を処理する。
  • 共有メモリのデタッチはプロセス終了時に自動で行われるが、明示的な dsm_detach を呼び出して整合性を保っている。
