ExecParallelEstimateContext

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/execParallel.c:112

目的

並列クエリ実行の準備段階において、各プランノードが必要とする動的共有メモリ（DSM）のサイズを推定するためのコンテキスト情報を保持します。

シグネチャまたは形状

typedef struct ExecParallelEstimateContext
{
	ParallelContext *pcxt;
	int			nnodes;
} ExecParallelEstimateContext;

入力と出力

  • **入力**: ParallelContext（並列実行全体の管理構造体）へのポインタおよび、走査対象のノード数カウント。
  • **出力**: 本構造体自体は、プランツリーの再帰的な走査を通じて各ノードが必要とするメモリ量を累積的に計算するための状態保持に使用されます。

前提条件と不変条件

  • ExecParallelEstimate 関数によるプランツリー走査の過程で利用されます。
  • pcxt は有効な ParallelContext を指している必要があります。

アルゴリズムまたは振る舞い

1. リーダープロセスにおいて、ExecInitParallelPlan から呼び出される ExecParallelEstimate に本構造体のインスタンスが渡されます。 2. プランツリーを再帰的に走査し、各ノードが ExecParallelEstimate を通じて申告するメモリ量を pcxt に集約します。 3. nnodes は走査したノード数を追跡するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecInitParallelPlan（src/backend/executor/execParallel.c）が本構造体を初期化し、ExecParallelEstimate へ渡します。
  • **呼び出し先**: 本構造体は ExecParallelEstimate 関数内で各プランノードの推定ロジックに引数として渡されます。

関連データ構造

  • ParallelContext: 並列実行の全体的な共有メモリ管理を行う構造体。
  • PlanState: プランツリーの実行状態を保持する構造体。

失敗モードと境界条件

  • 供給されたソース情報からは、特定の失敗モードや境界条件に関する明示的な記述はありません。

保守担当者向けメモ

  • 新しい並列対応プランノードを追加する際は、ExecParallelEstimate 関数内で本コンテキストを使用して必要なメモリ量を適切に加算するように実装してください。
