HashScanOpaqueData

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash.h:158

目的

ハッシュインデックススキャンにおけるプライベートな状態管理を行うための構造体です。

シグネチャまたは形状

typedef struct HashScanOpaqueData
{
    uint32      hashso_sk_hash;
    Buffer      hashso_bucket_buf;
    Buffer      hashso_split_bucket_buf;
    bool        hashso_buc_populated;
    bool        hashso_buc_split;
    int        *killedItems;
    int         numKilled;
    HashScanPosData currPos;
} HashScanOpaqueData;

入力と出力

  • 入力: スキャン対象のハッシュ値（hashso_sk_hash）およびスキャン中のバッファ状態。
  • 出力: currPos を通じて、現在のスキャン位置における一致アイテムの情報を保持します。

前提条件と不変条件

  • hashso_buc_split の値は、hashso_buc_populated が true の場合にのみ参照されます。
  • killedItems が使用されない場合、値は NULL となります。

アルゴリズムまたは振る舞い

  • ハッシュインデックスの動的なバケット分割中においてもスキャンの一貫性を維持します。
  • hashso_bucket_buf はプライマリバケットに関連付けられたバッファを保持します。
  • hashso_split_bucket_buf は、分割操作中にデータが移転されるバケットのプライマリページを保持し、分割中のスキャンをサポートします。
  • currPos を使用して、ページ内のマッチングアイテムを特定し保存します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/hash/hash.c:384 にて palloc を用いてメモリ確保が行われています。
  • 呼び出し先: 具体的な関数呼び出しの詳細は提供されていません。

関連データ構造

  • HashScanPosData: 現在のスキャン位置データを保持するために使用されます。
  • Buffer: PostgreSQL のバッファ管理構造体。

失敗モードと境界条件

  • 供給されたソース情報には、特定の失敗モードや境界条件に関する記述はありません。

保守担当者向けメモ

  • 本構造体はハッシュインデックススキャンの内部状態を保持するため、スキャンロジック（hashsearch.c 等）の変更時には整合性に注意してください。
  • 分割中のバケットに対するスキャン処理は、hashso_split_bucket_buf を介した排他制御やページアクセスと密接に関係しています。
