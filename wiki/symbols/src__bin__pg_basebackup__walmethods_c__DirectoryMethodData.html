DirectoryMethodData

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/walmethods.c:73

目的

WAL（Write-Ahead Log）データをファイルシステム上のディレクトリ構造へ永続化するための、WalWriteMethod を継承したデータ構造です。

シグネチャまたは形状

typedef struct DirectoryMethodData
{
	WalWriteMethod base;
	char	   *basedir;
} DirectoryMethodData;

入力と出力

  • **入力**: CreateWalDirectoryMethod 関数内でメモリが割り当てられ、basedir に保存先ディレクトリパスが格納されます。
  • **出力**: WalWriteMethod の派生型として、ディレクトリベースの WAL 書き込み操作に使用されます。

前提条件と不変条件

  • base フィールドは WalWriteMethod 型であり、本構造体は WalWriteMethod を継承するインターフェースの一部として扱われます。
  • basedir は有効なディレクトリパスを指す文字列である必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、pg_basebackup 等において WAL を個別のファイルとしてディレクトリに保存する際のコンテキストを保持します。
  • 各種操作関数（dir_open_for_write 等）において、WalWriteMethod ポインタからキャストして取得され、保存先ディレクトリの特定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: CreateWalDirectoryMethod にてインスタンス化されます。
  • **呼び出し先**: src/bin/pg_basebackup/walmethods.c 内の dir_ で始まる各関数（dir_open_for_write など）が、本構造体を引数として受け取り、内部の basedir を参照します。

関連データ構造

  • WalWriteMethod: 基底となる抽象構造体。
  • DirectoryMethodFile: ディレクトリメソッドで使用されるファイルハンドル構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（pg_malloc0）の挙動は、呼び出し元のエラーハンドリングに依存します。
  • basedir が指すパスの権限や存在確認は、本構造体の利用側（dir_open_for_write 等）で実行されます。

保守担当者向けメモ

  • 本構造体は WalWriteMethod のサブクラスとして機能します。キャスト操作が多用されているため、構造体のメモリレイアウトを変更する際は注意してください。
  • basedir のメモリ管理（解放）は、対応する finish メソッド内で行われる必要があります。
