RelationFindReplTupleByIndex

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execReplication.c:178

目的

論理レプリケーションにおいて、指定されたインデックスを使用して対象のタプルを検索し、指定されたモードでロックを取得する。

シグネチャまたは形状

bool RelationFindReplTupleByIndex(Relation rel, Oid idxoid, LockTupleMode lockmode, TupleTableSlot *searchslot, TupleTableSlot *outslot)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • idxoid: 検索に使用するインデックスのOID。
  • lockmode: タプル取得時に適用するロックモード。
  • searchslot: 検索条件となる値を含むスロット。
  • **出力**:
  • 戻り値: タプルが見つかり、ロックに成功した場合は true、それ以外は false。
  • outslot: 検索結果のタプルが格納されるスロット。

前提条件と不変条件

  • idxoid で指定されたインデックスは、対象リレーションに対して有効であること。
  • 呼び出し元は、適切なトランザクションコンテキストを保持していること。

アルゴリズムまたは振る舞い

1. 指定されたインデックスを RowExclusiveLock でオープンする。 2. build_replindex_scan_key を用いてスキャンキーを生成し、インデックススキャンを開始する。 3. インデックススキャンによりタプルを検索する。

  • インデックスがプライマリキーまたはレプリカアイデンティティである場合を除き、tuples_equal で値の一致を確認する。
  • タプルが他トランザクションによってロックされている場合、XactLockTableWait で待機し、再試行（retry）する。

    • 4. タプルが見つかった場合、table_tuple_lock を用いて指定された lockmode でロックを試みる。

  • ロック中に競合が発生し、should_refetch_tuple が真を返す場合は、再度検索からやり直す。

    • 5. 最後にインデックススキャンを終了し、インデックスをクローズして結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • index_open, index_beginscan, index_rescan, index_getnext_slot, index_endscan, index_close
  • build_replindex_scan_key
  • tuples_equal
  • XactLockTableWait
  • table_tuple_lock
  • should_refetch_tuple

関連データ構造

  • Relation, TupleTableSlot, ScanKeyData, IndexScanDesc, SnapshotData, TM_FailureData

失敗モードと境界条件

  • インデックススキャンでタプルが見つからない場合は false を返す。
  • ロック取得時に競合が発生した場合、内部的に再試行ループに入るため、デッドロックや過度な待機が発生する可能性がある。

保守担当者向けメモ

  • 本関数はレプリケーション適用時の行特定における主要なインターフェースである。
  • retry ラベルへのジャンプによる再試行ロジックが含まれているため、状態遷移の整合性に注意すること。
  • インデックスがない場合のフォールバックとして RelationFindReplTupleSeq が存在するため、パフォーマンス要件に応じて使い分けること。
