varattrib_1b

定義

  • 種別: struct
  • 定義箇所: src/include/varatt.h:127

目的

PostgreSQLにおいて、1バイトのヘッダーを持つ可変長データ（varlena）の物理レイアウトを定義する構造体です。

シグネチャまたは形状

typedef struct
{
    uint8       va_header;
    char        va_data[FLEXIBLE_ARRAY_MEMBER];
} varattrib_1b;

入力と出力

  • **入力**: va_header（データ形式や長さを制御するフラグ）および va_data（可変長のデータ本体）。
  • **出力**: 本構造体はデータ格納形式の定義であり、直接的な関数呼び出しによる入出力は行いません。

前提条件と不変条件

  • 本構造体はアライメントが保証されません。アクセス時にはアライメントを考慮しないコードパスを通す必要があります。
  • va_header のビット配置はエンディアンに依存するため、直接操作せず src/include/varatt.h で提供されるマクロ（VARATT_IS_... 等）を介してアクセスする必要があります。

アルゴリズムまたは振る舞い

  • va_header のビットパターンにより、データの格納状態（圧縮、外部ストレージ、ショート形式など）を識別します。
  • va_data は柔軟な配列メンバーとして定義されており、ヘッダー直後にデータ本体が配置されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/varatt.h 内の各種マクロ（VARATT_IS_1B, VARATT_IS_1B_E, VARSIZE_1B 等）が、本構造体へのキャストを通じてヘッダーの読み取りや設定を行います。
  • **呼び出し先**: 本構造体自体はデータ定義であり、関数を呼び出すことはありません。

関連データ構造

  • varattrib_4b: 4バイトヘッダーを持つ可変長データ構造体。
  • varatt_external: TOASTポインタ構造体（本構造体のサブセットとして扱われる場合があります）。

失敗モードと境界条件

  • アライメントが保証されないため、構造体への直接キャストを行う際は memcpy 等で適切にコピーしてからアクセスしない場合、CPUアーキテクチャによっては境界違反や不正なメモリアクセスが発生する可能性があります。

保守担当者向けメモ

  • TOASTポインタは varattrib_1b のサブセットとして定義されており、識別用のタグバイトを含みます。
  • 1バイトヘッダーのデータは、4バイトヘッダーのデータと混在してタプル内に存在するため、判定ロジックを変更する際は varatt.h 内の全判定マクロとの整合性に注意してください。
