in_range_int8_int8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int8.c:400

目的

ウィンドウ関数等において、指定されたオフセット値に基づき、対象値（val）が基準値（base）の範囲内にあるかを判定します。

シグネチャまたは形状

Datum in_range_int8_int8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取ります。
  • val (int64): 範囲チェック対象の値。
  • base (int64): 範囲の基準値。
  • offset (int64): 基準値からのオフセット。
  • sub (bool): true の場合、オフセットを減算として扱います。
  • less (bool): true の場合、範囲の下限チェック（val <= sum）を行います。
  • **出力**: bool 型の判定結果を Datum として返します。

前提条件と不変条件

  • offset が負の値である場合、エラー（ERRCODE_INVALID_PRECEDING_OR_FOLLOWING_SIZE）を発生させます。
  • sub が true の場合、offset は符号反転され、加算処理へ回されます。

アルゴリズムまたは振る舞い

1. offset の正当性を確認し、負であればエラーを報告します。 2. sub が true ならば offset を負数に変換します。 3. base と offset の加算を行い、オーバーフローが発生するかを確認します。 4. オーバーフローが発生した場合:

  • sub が false ならば、合計値は val より確実に大きいため、less の値をそのまま返します。
  • sub が true ならば、合計値は val より確実に小さいため、!less を返します。

    • 5. オーバーフローが発生しない場合:

  • less が true ならば val <= sum を判定します。
  • less が false ならば val >= sum を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は特定されていません。
  • **呼び出し先**:
  • pg_add_s64_overflow (オーバーフローチェック)
  • ereport (エラー報告)

関連データ構造

  • int64 (64ビット整数)
  • PG_FUNCTION_ARGS (PostgreSQL 関数呼び出し規約)

失敗モードと境界条件

  • offset < 0: ERRCODE_INVALID_PRECEDING_OR_FOLLOWING_SIZE を発生させます。
  • base + offset のオーバーフロー: pg_add_s64_overflow により検知され、論理的な比較結果（less または !less）にフォールバックします。

保守担当者向けメモ

  • 本関数はウィンドウ関数のフレーム境界計算に使用されます。
  • オーバーフロー時の挙動は、算術的な加算が不可能であっても、論理的な大小関係が確定できるという前提に基づいています。
