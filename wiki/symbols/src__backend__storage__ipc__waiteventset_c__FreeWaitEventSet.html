FreeWaitEventSet

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:480

目的

WaitEventSet 構造体が保持するリソースを解放し、メモリを破棄します。

シグネチャまたは形状

void FreeWaitEventSet(WaitEventSet *set)

入力と出力

  • **入力**: set (解放対象の WaitEventSet 構造体へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • set は有効な WaitEventSet へのポインタである必要があります。
  • set->owner が設定されている場合、リソース所有権の解除が行われます。

アルゴリズムまたは振る舞い

1. **リソース所有権の解除**: set->owner が存在する場合、ResourceOwnerForgetWaitEventSet を呼び出し、所有権を解除して set->owner を NULL に設定します。 2. **プラットフォーム固有のクリーンアップ**:

  • WAIT_USE_EPOLL または WAIT_USE_KQUEUE が定義されている場合、対応するファイル記述子（epoll_fd または kqueue_fd）を close し、ReleaseExternalFD() を呼び出します。
  • WAIT_USE_WIN32 が定義されている場合、セット内の各イベントを走査し、ソケットに関連付けられたイベントオブジェクトを WSAEventSelect および WSACloseEvent で解放します。

    • 3. **メモリ解放**: pfree(set) を呼び出し、構造体自体が使用していたメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAppend.c
  • src/backend/postmaster/postmaster.c
  • src/backend/storage/ipc/latch.c
  • src/backend/storage/ipc/waiteventset.c
  • **呼び出し先**:
  • ResourceOwnerForgetWaitEventSet
  • close (OSプリミティブ)
  • ReleaseExternalFD
  • WSAEventSelect / WSACloseEvent (Windows環境)
  • pfree

関連データ構造

  • WaitEventSet: 待機対象のイベント群を管理するメイン構造体。

失敗モードと境界条件

  • set が NULL の場合、ポインタ参照によるクラッシュが発生する可能性があります（ソースコード上に明示的なガードはありません）。
  • set->owner が無効な場合、ResourceOwnerForgetWaitEventSet 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はプラットフォーム依存のクリーンアップ処理（epoll, kqueue, Win32）を内包しています。新しいプラットフォーム対応を追加する際は、この条件分岐に適切な解放処理を追加してください。
