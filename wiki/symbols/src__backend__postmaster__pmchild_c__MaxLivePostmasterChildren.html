MaxLivePostmasterChildren

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pmchild.c:69

目的

Postmaster が管理する子プロセス用スロットの総数（num_pmchild_slots）を取得します。

シグネチャまたは形状

int MaxLivePostmasterChildren(void);

入力と出力

  • **入力**: なし
  • **出力**: 現在設定されている子プロセス用スロットの最大数（int 型）

前提条件と不変条件

  • 本関数を呼び出す前に、InitPostmasterChildSlots() が実行され、num_pmchild_slots が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. num_pmchild_slots が 0 であるかを確認します。 3. 0 でない場合は、保持している num_pmchild_slots の値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/pmsignal.c: 共有メモリ上のシグナルフラグ領域のサイズ計算に使用。
  • **呼び出し先**:
  • なし（内部変数 num_pmchild_slots を参照するのみ）

関連データ構造

  • num_pmchild_slots: 本関数が値を返す静的変数。

失敗モードと境界条件

  • **失敗モード**: InitPostmasterChildSlots() による初期化が行われる前に呼び出された場合、ERROR レベルのログを出力して処理を中断します。

保守担当者向けメモ

  • 本関数は、Postmaster の子プロセス管理機構におけるスロット数の「信頼できる情報源」として機能します。
  • 共有メモリのレイアウト計算など、プロセス起動の初期段階で頻繁に参照されるため、変更時は依存関係に注意してください。
