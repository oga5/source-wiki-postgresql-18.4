test_re_execute

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_regex/test_regex.c:200

目的

PostgreSQL の正規表現エンジン（Spencer's regex）を使用して、pg_wchar 形式のデータに対して正規表現マッチングを実行し、その結果を判定する。

シグネチャまたは形状

static bool test_re_execute(regex_t *re, pg_wchar *data, int data_len,
                            int start_search, rm_detail_t *details,
                            int nmatch, regmatch_t *pmatch, int eflags)

入力と出力

  • **入力**:
  • re: コンパイル済みの正規表現オブジェクト。
  • data: 検索対象の pg_wchar 配列。
  • data_len: データの長さ。
  • start_search: 検索開始位置。
  • details: マッチングの詳細情報（rm_detail_t）。
  • nmatch: pmatch 配列の要素数。
  • pmatch: マッチング結果を格納する regmatch_t 配列。
  • eflags: 実行フラグ。
  • **出力**:
  • bool: マッチングが成功（REG_OKAY）した場合は true、失敗（REG_NOMATCH）した場合は false を返す。

前提条件と不変条件

  • pmatch 配列は nmatch 以上のサイズを持つ必要がある。
  • pg_regexec の実行前に、details および pmatch 内のすべてのマッチング位置（rm_so, rm_eo）は -1 に初期化される。

アルゴリズムまたは振る舞い

1. **初期化**: details->rm_extend および pmatch 配列の全要素の開始・終了位置を -1 に設定する。 2. **実行**: pg_regexec を呼び出し、正規表現マッチングを行う。 3. **エラーハンドリング**: pg_regexec の戻り値が REG_OKAY でも REG_NOMATCH でもない場合、pg_regerror を使用してエラーメッセージを取得し、ereport(ERROR, ...) で処理を中断する。 4. **結果返却**: マッチングが成功したか否かを判定して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_regex/test_regex.c 内のループ処理（行493付近）から呼び出される。
  • **呼び出し先**:
  • pg_regexec（PostgreSQL 正規表現エンジン）
  • pg_regerror（エラーメッセージ生成）
  • ereport（エラー報告）

関連データ構造

  • regex_t: コンパイル済み正規表現構造体。
  • pg_wchar: PostgreSQL 内部のワイド文字型。
  • rm_detail_t: マッチングの詳細情報構造体。
  • regmatch_t: マッチング位置を保持する構造体。

失敗モードと境界条件

  • pg_regexec が内部エラー（REG_OKAY および REG_NOMATCH 以外）を返した場合、PostgreSQL のエラーとして報告される。
  • nmatch が不適切な場合、pmatch 配列へのアクセスで境界外参照が発生する可能性がある（呼び出し元に依存）。

保守担当者向けメモ

  • 本関数はテストハーネスの一部であり、正規表現エンジンのデバッグを目的としている。
  • pg_regexec の戻り値が REG_OKAY か REG_NOMATCH であることを前提としており、それ以外の予期せぬエラーコードに対しては例外を投げる設計となっている。
