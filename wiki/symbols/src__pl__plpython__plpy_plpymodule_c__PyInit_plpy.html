PyInit_plpy

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:125

目的

PL/Python 拡張の Python モジュール plpy を初期化し、Python インタプリタに対してモジュール構造と例外クラスを登録する。

シグネチャまたは形状

PyMODINIT_FUNC PyInit_plpy(void)

入力と出力

  • 入力: なし
  • 出力: 初期化された plpy モジュールオブジェクト（PyObject *）、または初期化失敗時に NULL。

前提条件と不変条件

  • Python インタプリタがロードされている環境で呼び出されること。
  • PLy_module 構造体が適切に定義されていること。

アルゴリズムまたは振る舞い

1. PyModule_Create を呼び出し、PLy_module 定義に基づいて plpy モジュールオブジェクトを作成する。 2. モジュール作成に失敗した場合は直ちに NULL を返す。 3. PLy_add_exceptions(m) を呼び出し、SQLSTATE に対応する例外クラスをモジュールに追加する。 4. 初期化されたモジュールオブジェクト m を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_main.c において、PyImport_AppendInittab を介して Python の初期化プロセスから参照される。
  • 呼び出し先:
  • PyModule_Create (Python C API)
  • PLy_add_exceptions (本ファイル内)

関連データ構造

  • PLy_module: plpy モジュールのメソッドや定義を保持する PyModuleDef 構造体。

失敗モードと境界条件

  • PyModule_Create が NULL を返した場合、関数は即座に NULL を返し、モジュールの初期化は中断される。

保守担当者向けメモ

  • 本関数は Python の C API を使用しているため、参照カウントの管理が重要である。
  • PLy_add_exceptions 内での例外クラス生成処理が失敗した場合の挙動については、当該関数の実装を確認すること。
