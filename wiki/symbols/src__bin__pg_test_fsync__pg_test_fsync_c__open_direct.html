open_direct

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:262

目的

OSのキャッシュをバイパスしてファイルを開くためのラッパー関数。O_DIRECT または F_NOCACHE を使用して、ファイルシステムキャッシュの影響を受けない直接的なI/Oを試行する。

シグネチャまたは形状

static int open_direct(const char *path, int flags, mode_t mode)

入力と出力

  • **入力**:
  • path: 開くファイルのパス。
  • flags: open システムコールに渡すフラグ。
  • mode: ファイル作成時のパーミッション。
  • **出力**:
  • 成功時: ファイル記述子（int）。
  • 失敗時: -1（errno が設定される）。

前提条件と不変条件

  • O_DIRECT が定義されている環境では、flags に O_DIRECT が自動的に付与される。
  • O_DIRECT が未定義で F_NOCACHE が定義されている環境では、open 成功後に fcntl(fd, F_NOCACHE, 1) が実行される。

アルゴリズムまたは振る舞い

1. O_DIRECT が定義されていれば flags に追加する。 2. open(path, flags, mode) を呼び出す。 3. O_DIRECT が未定義かつ F_NOCACHE が定義されている場合、open が成功していれば fcntl でキャッシュ無効化を試みる。 4. fcntl が失敗した場合は、ファイルを close し、エラーを伝播させる。 5. 最終的なファイル記述子またはエラーコードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_test_fsync/pg_test_fsync.c 内の各テスト関数（test_open_sync 等）から、同期テスト用の一時ファイルを開く際に呼び出される。
  • **呼び出し先**:
  • open (システムコール)
  • fcntl (システムコール、環境依存)
  • close (システムコール、エラー発生時)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • open 自体が失敗した場合、-1 を返す。
  • F_NOCACHE を適用する環境で fcntl が失敗した場合、ファイルを閉じた上で -1 を返し、errno を維持する。
  • O_DIRECT をサポートしないファイルシステムやマウントオプション環境では、open が失敗する可能性がある。

保守担当者向けメモ

  • 本関数はプラットフォームごとのキャッシュバイパス手法を抽象化している。
  • O_DIRECT や F_NOCACHE のサポート状況はプラットフォームに依存するため、移植性を考慮する場合は pg_config.h 等の定義を確認すること。
