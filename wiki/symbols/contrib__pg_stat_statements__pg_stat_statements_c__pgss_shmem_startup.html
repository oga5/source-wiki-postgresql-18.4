pgss_shmem_startup

定義

  • 種別: function
  • 定義箇所: contrib/pg_stat_statements/pg_stat_statements.c:510

目的

共有メモリの初期化、ハッシュテーブルの作成、およびサーバー起動時の統計情報の永続ファイルからの読み込みを行う。

シグネチャまたは形状

static void pgss_shmem_startup(void)

入力と出力

  • 入力: なし（グローバル変数 pgss_max 等を参照）
  • 出力: なし（共有メモリ上の pgss および pgss_hash を初期化）

前提条件と不変条件

  • AddinShmemInitLock を介した排他制御下で実行される。
  • 共有メモリの初期化は、他のプロセスが稼働していないタイミングで行われることを前提としている。
  • pgss_save が真の場合、PGSS_DUMP_FILE から統計を読み込む。

アルゴリズムまたは振る舞い

1. **フックの連鎖**: prev_shmem_startup_hook が存在する場合は実行する。 2. **共有メモリ確保**: pgssSharedState 構造体を共有メモリ上に確保し、初回起動時のみ初期値（ロック、統計リセット時刻等）を設定する。 3. **ハッシュテーブル作成**: pgss_hash を初期化する。 4. **終了フック登録**: ポストマスター配下でない場合、終了時に pgss_shmem_shutdown が呼ばれるよう登録する。 5. **統計復元**:

  • 既に初期化済み（found が真）であれば終了する。
  • PGSS_TEXT_FILE を初期化し、PGSS_DUMP_FILE が存在すれば内容を読み込む。
  • ファイルから読み取った各エントリを entry_alloc でハッシュテーブルに再構築し、カウンタ情報を復元する。
  • 最後にグローバル統計を読み込み、一時ファイルを閉じる。

呼び出し元と呼び出し先

  • 呼び出し元: shmem_startup_hook に登録され、PostgreSQL の起動プロセスから呼び出される。
  • 呼び出し先: prev_shmem_startup_hook, ShmemInitStruct, ShmemInitHash, on_shmem_exit, AllocateFile, entry_alloc 等。

関連データ構造

  • pgssSharedState: 共有メモリ上のグローバル状態管理構造体。
  • pgssHashKey: ハッシュテーブルのキー。
  • pgssEntry: ハッシュテーブルの各エントリ。
  • pgssGlobalStats: 全体統計情報。

失敗モードと境界条件

  • AllocateFile が失敗した場合、エラー処理へ遷移する。
  • ファイルヘッダーやバージョンが一致しない場合、データ破損とみなしてエラー処理へ遷移する。
  • PG_VALID_BE_ENCODING チェックに失敗した場合、データエラーとして処理される。
  • pgss_save が偽の場合、統計の読み込みをスキップする。

保守担当者向けメモ

  • 本関数は pg_stat_statements のライフサイクルの起点となる。
  • PGSS_DUMP_FILE の読み込み処理において、buffer の動的リサイズ（repalloc）が行われるため、メモリ管理に注意すること。
  • 読み込み時に IS_STICKY なエントリはスキップされる仕様となっている。
