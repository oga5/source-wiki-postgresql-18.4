ListenServerPort

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/libpq/pqcomm.c:417

目的

Postmaster がクライアントからの接続を受け入れるために、指定されたアドレスファミリー、ホスト名、ポート番号に基づいてサーバーソケットを作成・設定する。

シグネチャまたは形状

int ListenServerPort(int family, const char *hostName, unsigned short portNumber,
                     const char *unixSocketDir,
                     pgsocket ListenSockets[], int *NumListenSockets, int MaxListen)

入力と出力

  • **入力**:
  • family: アドレスファミリー（AF_INET, AF_INET6, AF_UNIX 等）。
  • hostName: 待機するホスト名。
  • portNumber: 待機するポート番号。
  • unixSocketDir: Unix ドメインソケットのディレクトリパス。
  • ListenSockets[]: 成功したソケット記述子を格納する配列。
  • NumListenSockets: 現在のソケット数（ポインタ）。
  • MaxListen: 最大待機ソケット数。
  • **出力**: 成功時は STATUS_OK (0)、失敗時は STATUS_ERROR (-1) を返す。

前提条件と不変条件

  • NumListenSockets が MaxListen に達している場合、新しいソケットは作成されない。
  • AF_UNIX 指定時は Lock_AF_UNIX によるパスのロックが成功する必要がある。
  • WIN32 環境以外では、作成されたソケットには FD_CLOEXEC が設定される。

アルゴリズムまたは振る舞い

1. pg_getaddrinfo_all を使用して、指定されたホスト名とポートからアドレスリストを取得する。 2. リスト内の各アドレスに対し、以下の処理を繰り返す。

  • socket() システムコールでソケットを作成する。
  • WIN32 以外では fcntl で FD_CLOEXEC を設定する。
  • TCP ソケットの場合、setsockopt で SO_REUSEADDR を有効にする。
  • IPv6 ソケットの場合、IPV6_V6ONLY を設定する。

    • 3. 成功したソケット記述子を ListenSockets 配列に追加し、カウントを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c 内の各所（AF_UNSPEC や AF_UNIX を指定して呼び出し）。
  • **呼び出し先**: Lock_AF_UNIX, pg_getaddrinfo_all, pg_freeaddrinfo_all, pg_getnameinfo_all, socket, fcntl, setsockopt, closesocket, ereport 等。

関連データ構造

  • struct addrinfo: アドレス情報の解決に使用。
  • pgsocket: ソケット記述子の型。
  • SockAddr (定義は pqcomm.h): 通信アドレスの保持に使用。

失敗モードと境界条件

  • **パス長超過**: AF_UNIX でソケットパスが UNIXSOCK_PATH_BUFLEN を超える場合はエラー。
  • **名前解決失敗**: pg_getaddrinfo_all が失敗した場合はエラー。
  • **リソース枯渇**: MaxListen を超えるソケット作成要求があった場合はログ出力して中断。
  • **ソケット作成失敗**: socket() システムコールが失敗した場合はログを出力し、次のアドレスへスキップする。

保守担当者向けメモ

  • 本関数は pqcomm.c 内の低レベル通信初期化の一部であり、postmaster の起動プロセスと密接に関連している。
  • WIN32 と Unix 系でソケットオプションの挙動が異なるため、修正時はプラットフォーム固有の条件分岐に注意すること。
