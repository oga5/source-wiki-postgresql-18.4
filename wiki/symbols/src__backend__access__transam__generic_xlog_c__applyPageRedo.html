applyPageRedo

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/generic_xlog.c:452

目的

WALリカバリ時、ページに対して記録された差分（デルタ）を適用し、ページの状態を再現する。

シグネチャまたは形状

static void applyPageRedo(Page page, const char *delta, Size deltaSize)

入力と出力

  • page: 変更対象となるメモリ上のページバッファ。
  • delta: 適用する差分データが格納されたバイト列。
  • deltaSize: delta のバイトサイズ。
  • 戻り値: なし。

前提条件と不変条件

  • delta は、computeDelta 等によって生成された、offset（2バイト）と length（2バイト）および実際のデータが交互に並ぶ形式であること。
  • page は書き込み可能な状態であること。

アルゴリズムまたは振る舞い

1. delta の先頭から deltaSize に達するまで、以下のループを繰り返す。 2. 差分データから offset（変更開始位置）と length（変更長）を読み取る。 3. page + offset の位置へ、差分データから読み取った length 分のデータを memcpy でコピーする。 4. ポインタを次の差分データへ進める。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/generic_xlog.c 内の generic_redo（リカバリ時のページ復元処理）
  • src/backend/access/transam/generic_xlog.c 内の GenericXLogFinish（構築中のWALレコード適用時）
  • 呼び出し先:
  • memcpy（標準ライブラリ）

関連データ構造

  • Page: PostgreSQLのページ構造体（typedef char *Page）。

失敗モードと境界条件

  • delta の内容が不正な場合、page の範囲外へのメモリコピーが発生し、セグメンテーション違反やデータ破損を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は、汎用WALレコードの適用における低レイヤーのデータ復元を担当する。
  • 差分データのフォーマット（offset + length + data）を変更する場合は、computeDelta との整合性を維持すること。
