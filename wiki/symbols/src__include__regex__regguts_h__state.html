state

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regguts.h:330

目的

正規表現のコンパイル過程におけるNFA（非決定性有限オートマトン）の各状態を表現するデータ構造です。

シグネチャまたは形状

struct state {
    int           no;           /* 状態番号 (FREESTATE は -1) */
    char          flag;         /* 特殊状態のフラグ */
    int           nins;         /* 入力アーク数 */
    int           nouts;        /* 出力アーク数 */
    struct arc   *ins;          /* 入力アークのリスト */
    struct arc   *outs;         /* 出力アークのリスト */
    struct state *tmp;          /* 走査アルゴリズム用の一時ポインタ */
    struct state *next;         /* 全ライブ状態のリスト、または空き状態の連結用 */
    struct state *prev;         /* 全ライブ状態のリストの逆方向リンク */
};

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。NFA構築プロセスにおいて、状態の生成、アークの接続、およびリストの走査に使用されます。

前提条件と不変条件

  • no フィールドには、非負の整数または FREESTATE (-1) が格納されます。
  • ins および outs は、それぞれ struct arc の連結リストを指します。
  • next および prev フィールドは、NFA内の全ライブ状態を管理するための双方向リストを構成します。

アルゴリズムまたは振る舞い

  • **状態管理**: next フィールドは、アクティブな状態を辿るためのリストとして機能するほか、解放された状態を連結して再利用するためのリストとしても使用されます。
  • **走査**: tmp フィールドは、NFAのグラフ走査アルゴリズム（到達可能性判定や最適化など）において、訪問済みフラグや一時的な状態保持のために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regcomp.c において、NFAの構築および最適化処理中にインスタンス化・操作されます。
  • **呼び出し先**: struct arc を介してアーク情報にアクセスします。
  • **注意**: ツールが提示した contrib/amcheck 関連の参照は、本構造体とは異なる文脈（GinState等）のシンボルであり、本構造体とは無関係です。

関連データ構造

  • struct nfa: 本構造体を要素として保持する、NFA全体の管理構造体。
  • struct arc: 状態間を接続するアークの定義。

失敗モードと境界条件

  • no が FREESTATE (-1) の場合、その状態は現在使用されていない（解放済み）ことを示します。
  • メモリ確保失敗時の挙動については、regguts.h 内のメモリ管理マクロ（MALLOC 等）の定義に依存します。

保守担当者向けメモ

  • next フィールドが「ライブ状態の連結」と「空き状態の連結」という二つの役割を兼ねているため、状態のリスト操作を行う際は、現在の状態がどちらのリストに属しているかを明確に区別する必要があります。
  • tmp ポインタは再帰的なアルゴリズムやグラフ走査で頻繁に書き換えられるため、複数のアルゴリズムを並行して実行する際には競合に注意してください。
