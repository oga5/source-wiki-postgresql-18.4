gbt_var_node_truncate

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_var.c:213

目的

GiST インデックスノード内の可変長キー（lower および upper）を指定された長さで切り詰め（truncate）、インデックスのサイズを削減および最適化します。

シグネチャまたは形状

GBT_VARKEY *gbt_var_node_truncate(const GBT_VARKEY *node, int32 cpf_length, const gbtree_vinfo *tinfo)

入力と出力

  • **入力**:
  • node: 切り詰め対象の GBT_VARKEY 構造体。
  • cpf_length: 切り詰める目標の長さ（共通プレフィックス長）。
  • tinfo: 可変長型情報構造体（使用箇所なし）。
  • **出力**:
  • 切り詰められた新しい GBT_VARKEY を指すポインタ。メモリは palloc0 で確保されます。

前提条件と不変条件

  • node は gbt_var_key_readable で解釈可能な有効な GBT_VARKEY であること。
  • cpf_length は非負の整数であること。
  • 出力されるキーは、元の lower および upper の先頭 cpf_length + 1 バイトまでを保持します。

アルゴリズムまたは振る舞い

1. gbt_var_key_readable を使用して、入力ノードの lower と upper のポインタを取得します。 2. 各キーの長さを cpf_length + 1 と比較し、小さい方を採用して切り詰め後の長さを決定します。 3. 必要な合計サイズ（ヘッダーサイズとアライメントを考慮）を計算し、palloc0 でメモリを確保します。 4. lower と upper のデータを新しいメモリ領域にコピーし、それぞれの VARSIZE を適切に設定します。 5. 構築された新しい GBT_VARKEY を返します。

関連データ構造

  • GBT_VARKEY: 可変長キーの内部表現。
  • GBT_VARKEY_R: GBT_VARKEY を読み取りやすくするための構造体。
  • gbtree_vinfo: 可変長型情報の抽象化構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • cpf_length が元のデータ長より大きい場合、元の長さが維持されます。

保守担当者向けメモ

  • 本関数は INTALIGN マクロを使用して upper キーの配置を調整しています。メモリレイアウトを変更する際は、このアライメント計算が維持されているか確認してください。
  • tinfo 引数は現在関数内で使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
