pg_base64_dec_len

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-armor.c:171

目的

Base64 エンコードされたデータ長から、デコード後のバイナリデータの最大長を算出します。

シグネチャまたは形状

static unsigned pg_base64_dec_len(unsigned srclen)

入力と出力

  • **入力**: srclen (Base64 エンコードされた文字列の長さ)
  • **出力**: デコード後のバイナリデータの最大長（バイト単位）

前提条件と不変条件

  • 入力 srclen は Base64 文字列の長さを想定しています。
  • 計算結果は、Base64 の 4 文字が 3 バイトのバイナリに変換されるという特性に基づいています。

アルゴリズムまたは振る舞い

入力値に対して (srclen * 3) >> 2 を実行します。これは srclen * 3 / 4 と同等であり、Base64 のエンコード効率（4文字で3バイト）に基づいた概算値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-armor.c 内の関数（例: pgp_armor_decode 関連処理）から、デコード用バッファのサイズ確保のために呼び出されます。
  • **呼び出し先**: なし。
  • **注記**: src/backend/utils/adt/encode.c に同名の関数が存在しますが、本シンボルは contrib/pgcrypto 内で定義された静的関数であり、実装が異なります。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 入力値が 0 の場合、戻り値は 0 となります。
  • 非常に大きな srclen を指定した場合、符号なし整数のオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数は contrib/pgcrypto 独自の Base64 実装の一部です。他のモジュールにある同名の関数と混同しないよう注意してください。
  • 戻り値はあくまで「最大長」の概算であり、パディング等の影響により実際のデコード結果と一致しない場合があります。
