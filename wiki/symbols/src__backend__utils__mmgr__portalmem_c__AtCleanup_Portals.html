AtCleanup_Portals

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:857

目的

トランザクション終了後のクリーンアップ処理において、不要となったポータルをハッシュテーブルから走査し、強制的に破棄（解放）する。

シグネチャまたは形状

void AtCleanup_Portals(void)

入力と出力

  • 入力: なし（グローバルな PortalHashTable を参照）
  • 出力: なし（副作用としてポータルのメモリ解放およびハッシュテーブルからの削除を行う）

前提条件と不変条件

  • PortalHashTable が初期化されていること。
  • 実行中のポータル（PORTAL_ACTIVE）はクリーンアップ対象外とする。
  • 以前のトランザクションから保持されているポータル（createSubid が InvalidSubTransactionId）や、自動保持（autoHeld）されたポータルはスキップする。

アルゴリズムまたは振る舞い

1. PortalHashTable を hash_seq_init で走査開始する。 2. 各ポータルに対し以下の順で処理を行う：

  • PORTAL_ACTIVE 状態のポータルはスキップする。
  • createSubid が無効、または autoHeld が真の場合はスキップする。
  • portalPinned が真であれば、強制的に false に設定してピンを解除する。
  • ポータルにクリーンアップフック（cleanup）が設定されている場合、警告を出力してフックを無効化する（クリーンアップ中のユーザーコード実行を避けるため）。
  • PortalDrop(portal, false) を呼び出し、ポータルを破棄する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c（トランザクション終了処理の一部として呼び出される）。
  • 呼び出し先: hash_seq_init, hash_seq_search, PortalDrop, elog。

関連データ構造

  • PortalHashTable: 全ポータルを管理するハッシュテーブル。
  • PortalHashEnt: ハッシュテーブルの各エントリ。
  • Portal: ポータルの状態（status, createSubid, autoHeld, portalPinned, cleanup）を保持する構造体。

失敗モードと境界条件

  • 実行中のポータルが存在する場合、本関数はそれをスキップして処理を継続する。
  • クリーンアップフックが設定されているポータルについては、実行せずに警告ログを出力してスキップする。

保守担当者向けメモ

  • 本関数はトランザクション終了時の最終的なクリーンアップを担当するため、ここで呼び出される PortalDrop は、リソースリークを防ぐための最後の砦となる。
  • portalPinned の強制解除は、アボート時に中断された処理が保持していたピンを無効化するために必要である。
