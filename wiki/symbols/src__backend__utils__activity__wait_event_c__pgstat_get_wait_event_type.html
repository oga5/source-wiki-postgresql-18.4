pgstat_get_wait_event_type

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/wait_event.c:372

目的

待機イベント情報（wait_event_info）から、対応する待機クラス名（文字列）を取得する。

シグネチャまたは形状

const char *pgstat_get_wait_event_type(uint32 wait_event_info)

入力と出力

  • **入力**: wait_event_info (uint32型)。待機状態を示すビットフィールド。
  • **出力**: 待機クラス名を表す文字列リテラル。待機していない場合は NULL を返す。

前提条件と不変条件

  • wait_event_info が 0 の場合、待機中ではないとみなされ NULL が返される。
  • 既知のクラスに該当しない場合、"???" が返される。

アルゴリズムまたは振る舞い

1. wait_event_info が 0 ならば NULL を即座に返す。 2. WAIT_EVENT_CLASS_MASK を用いて wait_event_info の上位ビット（クラスID）を抽出する。 3. switch 文により classId を判定し、以下のいずれかの文字列を返す：

  • LWLock, Lock, BufferPin, Activity, Client, Extension, IPC, Timeout, IO, InjectionPoint

    • 4. 上記のいずれにも一致しない場合は "???" を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/wait_event.c (内部的なデバッグ/ログ出力用)
  • src/backend/utils/adt/pgstatfuncs.c (SQL関数 pg_stat_activity 等の表示用)
  • src/backend/utils/adt/waitfuncs.c (待機状態の確認用)
  • **呼び出し先**:
  • 外部関数呼び出しはなし。

関連データ構造

  • wait_event_info: 待機イベントを識別する整数値。
  • WAIT_EVENT_CLASS_MASK: クラスIDを抽出するためのビットマスク。

失敗モードと境界条件

  • wait_event_info に定義外のクラスIDが含まれる場合、"???" を返すことで安全に処理を継続する。

保守担当者向けメモ

  • 新しい待機クラスを追加した場合は、本関数の switch 文にもケースを追加する必要がある。
