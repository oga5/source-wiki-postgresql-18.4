JsonPathItemType

定義

  • 種別: enum
  • 定義箇所: src/include/utils/jsonpath.h:62

目的

SQL/JSON Path 式を構成する各要素（リテラル、演算子、メソッド、述語など）を識別するための列挙型です。バイナリ形式の JSON Path において、各ノードの型を特定するために使用されます。

シグネチャまたは形状

enum 型であり、jpiNull から jpiTimestampTz までの各要素を定義します。各要素はオンディスク形式のバイナリ構造と直接対応しています。

入力と出力

  • **入力**: 該当なし（型定義のため）。
  • **出力**: 評価エンジンにおいて、現在のノードがどのような操作（加算、比較、メソッド呼び出し等）を行うべきかを決定するための識別子として機能します。

前提条件と不変条件

  • **後方互換性**: 本列挙型の順序はオンディスク形式に直結しています。バイナリ形式の互換性を維持するため、新しい要素を追加する場合は必ず末尾に追加する必要があります。

アルゴリズムまたは振る舞い

  • JSON Path のバイナリ走査において、JsonPathItem 構造体の一部として保持されます。
  • 評価エンジンは、この型を参照して適切な演算子やメソッドを適用します。
  • jspOperationName 関数を通じて、デバッグやログ出力用に人間が読める文字列に変換可能です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonpath.c: 演算子の優先順位判定 (operationPriority) や名前解決 (jspOperationName) で参照。
  • src/backend/utils/adt/jsonb_gin.c: GIN インデックス処理においてアイテム型の識別に使用。
  • **呼び出し先**:
  • 本シンボルは列挙型定義であり、関数ではないため直接的な呼び出し先はありません。

関連データ構造

  • JsonPathItem: 本型をメンバとして保持し、パスの各ノードを走査するための構造体。
  • JsonPathParseItem: 解析時の中間表現で使用される型。

失敗モードと境界条件

  • 定義順序の変更は、既存のバイナリ形式の JSON Path データを破壊する可能性があります。

保守担当者向けメモ

  • 新しい JSON Path 機能を追加する際は、必ず本列挙型の末尾に新しい jpi 定数を追加してください。
  • jspOperationName 関数にも対応する文字列定義を追加することを忘れないでください。
