macaddr8_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac8.c:324

目的

2つの macaddr8 型（EUI-64 MACアドレス）の値を比較し、その大小関係を整数で返します。

シグネチャまたは形状

Datum macaddr8_cmp(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの macaddr8 型ポインタを受け取ります。
  • **出力**: int32 型の比較結果を返します（第1引数が小さい場合は負、等しい場合は0、大きい場合は正）。

前提条件と不変条件

  • 入力される macaddr8 ポインタは有効なメモリ領域を指している必要があります。
  • 内部的に macaddr8_cmp_internal を呼び出すため、同関数が要求するメモリレイアウト（8バイトのデータ構造）が維持されている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_MACADDR8_P マクロを使用して、引数から2つの macaddr8 構造体へのポインタを取得します。 2. macaddr8_cmp_internal 関数を呼び出し、実際の比較処理を委譲します。 3. 比較結果を PG_RETURN_INT32 マクロでラップして返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: macaddr8_cmp_internal
  • **呼び出し元（直接的な証拠）**:
  • contrib/btree_gin/btree_gin.c: GINインデックスのサポート関数として登録。
  • contrib/btree_gist/btree_macaddr8.c: GiSTインデックスの比較ロジック内で使用。

関連データ構造

  • macaddr8: 8バイト（EUI-64）のMACアドレスを保持するデータ構造。

失敗モードと境界条件

  • 引数が NULL の場合、PG_GETARG_MACADDR8_P の仕様に基づきエラーが発生する可能性があります。
  • 比較ロジックは macaddr8_cmp_internal に依存しており、同関数が扱うビットシフト演算（hibits, lobits マクロ）が構造体のメモリレイアウトと一致しない場合、誤った比較結果を返す可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な比較ロジックは macaddr8_cmp_internal に集約されています。変更が必要な場合は、まず macaddr8_cmp_internal の実装を確認してください。
