output_check_banner

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/check.c:570

目的

pg_upgrade の実行プロセスにおいて、一貫性チェック（Consistency Checks）の開始をユーザーに通知するためのバナーメッセージを出力します。

シグネチャまたは形状

void output_check_banner(void)

入力と出力

  • 入力: なし（グローバル変数 user_opts を参照）
  • 出力: なし（pg_log を通じて標準出力またはログファイルへメッセージを出力）

前提条件と不変条件

  • user_opts 構造体が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. user_opts.live_check の値を評価します。 2. live_check が真（ライブサーバーに対するチェック）の場合、「Performing Consistency Checks on Old Live Server」という見出しを出力します。 3. live_check が偽の場合、「Performing Consistency Checks」という見出しを出力します。 4. いずれの場合も、見出しの下に区切り線を表示します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_upgrade/pg_upgrade.c:127
  • 呼び出し先:
  • pg_log（ログ出力関数）

関連データ構造

  • user_opts: pg_upgrade の実行オプションを保持するグローバル構造体。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。ログ出力が失敗した場合は pg_log の挙動に依存します。

保守担当者向けメモ

  • 本関数は単なるUI表示用です。表示内容を変更する場合は、pg_upgrade の他のログ出力フォーマットとの整合性に注意してください。
