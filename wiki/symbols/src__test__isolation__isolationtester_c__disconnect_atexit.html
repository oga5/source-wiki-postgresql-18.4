disconnect_atexit

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolationtester.c:76

目的

プログラム終了時に、確立されているすべてのデータベース接続を安全に切断し、リソースを解放する。

シグネチャまたは形状

static void disconnect_atexit(void)

入力と出力

  • 入力: なし（グローバル変数 nconns および conns を参照）
  • 出力: なし

前提条件と不変条件

  • conns 配列が初期化されており、nconns が現在の接続数を示していること。
  • PQfinish は NULL ポインタに対して安全であるため、接続が確立されていない要素に対しても呼び出し可能。

アルゴリズムまたは振る舞い

1. 0 から nconns - 1 までの範囲でループを実行する。 2. 各インデックス i について、conns[i].conn が有効（非NULL）であるかを確認する。 3. 有効な接続が存在する場合、PQfinish(conns[i].conn) を呼び出して接続を終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/isolation/isolationtester.c:151 にて atexit(disconnect_atexit) として登録されている。
  • 呼び出し先: libpq の PQfinish 関数。

関連データ構造

  • IsoConnInfo: conns 配列の要素型。接続状態を保持する構造体。

失敗モードと境界条件

  • 接続が既に閉じられている場合や、接続が確立されていない場合（conns[i].conn が NULL の場合）、何も行わずにスキップされる。

保守担当者向けメモ

  • 本関数は atexit ハンドラとして使用されることを前提としている。
  • 他のプログラム（pg_basebackup 等）でも同名の関数が定義されているが、本ページは src/test/isolation/isolationtester.c 内の定義に限定している。
