MatchText

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_match.c:79

目的

LIKE 演算子におけるパターンマッチングのコアロジックを実行します。対象文字列とパターンを走査し、ワイルドカード（%, _）およびエスケープ文字を考慮した照合を行います。

シグネチャまたは形状

int MatchText(const char *t, int tlen, const char *p, int plen, pg_locale_t locale)

入力と出力

  • **入力**:
  • t: 対象文字列のポインタ
  • tlen: 対象文字列の長さ
  • p: パターン文字列のポインタ
  • plen: パターン文字列の長さ
  • locale: ロケール情報（照合順序の決定に使用）
  • **出力**:
  • LIKE_TRUE: マッチ成功
  • LIKE_FALSE: マッチ失敗
  • LIKE_ABORT: マッチ不可能（これ以上の走査不要）

前提条件と不変条件

  • 再帰呼び出しを行うため、スタックオーバーフローを防ぐために check_stack_depth() が呼び出されます。
  • パターンがエスケープ文字（\）で終わることは許容されず、エラーとなります。
  • マルチバイトエンコーディング環境下でも、文字境界を意識した走査が行われます。

アルゴリズムまたは振る舞い

1. **高速パス**: パターンが % 単一の場合は即座に LIKE_TRUE を返します。 2. **走査ループ**: 文字列とパターンを先頭から比較します。

  • \ (エスケープ): 次の文字をリテラルとして扱い、比較します。
  • % (ワイルドカード):
  • 後続のワイルドカードをスキップし、リテラル部分を特定します。
  • 残りのパターンが一致する位置を探索するため、再帰的に MatchText を呼び出します。
  • _ (任意1文字): 対象文字列を1文字進め、パターンを1バイト進めます。

    • 3. **非決定論的ロケール**: ロケールが非決定論的な場合、サブパターン単位で照合を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/like.c（SB_MatchText, SB_IMatchText, UTF8_MatchText 等の定義を通じて利用）。
  • **呼び出し先**: 自身を再帰的に呼び出します。また、check_stack_depth() や ereport() を呼び出します。

関連データ構造

  • pg_locale_t: ロケール設定を保持する構造体。
  • LIKE_TRUE, LIKE_FALSE, LIKE_ABORT: 戻り値として使用される定数。

失敗モードと境界条件

  • **エラー**: パターンがエスケープ文字で終了した場合、ERRCODE_INVALID_ESCAPE_SEQUENCE でエラーを報告します。
  • **スタックオーバーフロー**: 複雑なパターンや極端に長い文字列に対して再帰が深くなった場合、check_stack_depth() により保護されます。
  • **マッチ失敗**: パターンと文字列が一致しない場合、LIKE_FALSE または LIKE_ABORT が返されます。

保守担当者向けメモ

  • 本関数はテンプレートファイルの一部であり、like.c で定義されたマクロ（NextChar, GETCHAR 等）に依存しています。修正時はインクルード元の定義を確認してください。
  • 再帰関数であるため、パフォーマンスとスタック消費量に注意が必要です。特に非決定論的ロケール時の挙動は標準 SQL に準拠した複雑な処理となっています。
