SetQueryCompletion

定義

  • 種別: static inline function
  • 定義箇所: src/include/tcop/cmdtag.h:36

目的

SQLコマンドの実行結果を記録するために、QueryCompletion 構造体を指定されたコマンドタグと処理行数で初期化（設定）します。

シグネチャまたは形状

static inline void SetQueryCompletion(QueryCompletion *qc, CommandTag commandTag, uint64 nprocessed)

入力と出力

  • **入力**:
  • qc: 設定対象となる QueryCompletion 構造体へのポインタ。
  • commandTag: 実行されたSQLコマンドの種類を示す CommandTag 列挙値。
  • nprocessed: コマンドによって処理された行数（uint64）。
  • **出力**: なし（qc が指す構造体のメンバを直接更新します）。

前提条件と不変条件

  • qc は有効な QueryCompletion 構造体を指している必要があります。
  • 本関数は、コマンド実行後に実行結果を確定させるために使用されます。

アルゴリズムまたは振る舞い

1. 引数 commandTag を qc->commandTag に代入します。 2. 引数 nprocessed を qc->nprocessed に代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/createas.c
  • src/backend/commands/matview.c
  • src/backend/commands/portalcmds.c
  • src/backend/replication/walsender.c
  • src/backend/tcop/pquery.c
  • src/backend/tcop/utility.c
  • **呼び出し先**: 本関数はインライン関数であり、他の関数を呼び出しません。

関連データ構造

  • QueryCompletion: コマンドの完了状態を保持する構造体。
  • CommandTag: SQLコマンドの種類を定義する列挙型。

失敗モードと境界条件

  • qc が NULL の場合、セグメンテーション違反が発生します。
  • 処理行数 nprocessed が非常に大きい場合でも uint64 の範囲内で保持されます。

保守担当者向けメモ

  • 本関数はインライン関数として定義されているため、変更時はヘッダーファイル src/include/tcop/cmdtag.h を修正してください。
  • INSERT コマンド等の特殊なタグ処理については、BuildQueryCompletionString（cmdtag.c 内）側のロジックと整合性を取る必要があります。
