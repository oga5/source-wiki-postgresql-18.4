lookup_descriptor

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:124

目的

ECPGプリプロセッサにおいて、指定された名前と接続情報に一致する既存のSQL記述子領域（Descriptor Area）を検索し、必要に応じて接続情報を更新します。

シグネチャまたは形状

struct descriptor *lookup_descriptor(const char *name, const char *connection)

入力と出力

  • **入力**:
  • name: 検索対象の記述子名（" で始まる必要がある）。
  • connection: 記述子がバインドされている接続名（NULL許容）。
  • **出力**:
  • 成功時: 一致した struct descriptor へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • name は " で開始されている必要があります。開始されていない場合、検索を行わず直ちに NULL を返します。
  • 記述子はグローバルな descriptors リストに保持されている必要があります。

アルゴリズムまたは振る舞い

1. name の先頭文字が " でない場合、NULL を返します。 2. グローバルな descriptors リストを走査し、名前が一致する記述子を探します。 3. 名前が一致した場合:

  • 接続情報が完全に一致（または両方ともNULL）すれば、その記述子を返します。
  • 記述子側の接続情報が未設定（NULL）で、引数に connection が指定されている場合、その記述子の接続情報を引数の値で更新（mm_strdup）し、その記述子を返します。

    • 4. リスト内に一致するものがない場合、mmerror を通じて警告（ET_WARNING）を出力し、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の具体的な呼び出し箇所は、提供された抜粋からは特定できません。
  • **呼び出し先**:
  • strcmp: 文字列比較。
  • mm_strdup: 接続文字列の複製。
  • mmerror: エラー/警告報告。

関連データ構造

  • struct descriptor: 記述子の名前、接続情報、および次へのポインタを保持する構造体。
  • descriptors: 記述子を管理するグローバルなリスト。

失敗モードと境界条件

  • name が " で始まらない場合、検索は行われません。
  • 該当する記述子が見つからない場合、PARSE_ERROR として警告が出力されます。
  • 接続情報が不一致の場合、その記述子は無視され、リストの走査が継続されます。

保守担当者向けメモ

  • 本関数は preproc_extern.h で宣言されており、プリプロセッサ全体から参照可能です。
  • 接続情報の更新時に mm_strdup を使用しているため、メモリ管理に注意してください。
