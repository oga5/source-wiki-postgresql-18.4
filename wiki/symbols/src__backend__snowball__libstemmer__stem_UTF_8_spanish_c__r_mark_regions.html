r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_spanish.c:506

目的

スペイン語のステミング処理において、単語内のRV（語幹の開始位置）、R1、R2領域を特定し、SN_env 構造体の整数配列 I[0..2] に境界位置を記録する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （Snowballランタイム環境）
  • 出力: int （常に 1 を返す）
  • 副作用: z->I[0] (R2), z->I[1] (R1), z->I[2] (RV) に計算されたオフセット値を格納する。

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。
  • 領域計算は単語の末尾位置 z->l を基準に行われる。

アルゴリズムまたは振る舞い

1. **初期化**: I[0..2] を単語の末尾 z->l で初期化する。 2. **RV領域の特定**:

  • 母音の直後の子音、または単語先頭から2番目の母音の直後など、スペイン語の規則に基づきRVの開始位置を検索し I[2] に設定する。

    • 3. **R1/R2領域の特定**:

  • 母音と子音の連続パターンを走査し、R1（I[1]）およびR2（I[0]）の境界を特定する。
  • 領域の特定は out_grouping_U（母音以外）および in_grouping_U（母音）の組み合わせによって行われる。

呼び出し元と呼び出し先

  • 呼び出し元: spanish_UTF_8_stem（直接の呼び出し関係はソースコードの構造から推測されるが、本抜粋には明示されていない）。
  • 呼び出し先: in_grouping_U, out_grouping_U, skip_utf8（いずれも header.h で定義されたユーティリティ）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。I 配列（整数領域）、c（現在位置）、l（文字列長）、p（文字列データ）を含む。

失敗モードと境界条件

  • 文字列が空または母音を含まない場合、領域は初期値（単語末尾）のままとなる。
  • UTF-8の不正なシーケンスや境界外アクセスについては、skip_utf8 等のユーティリティ関数が制御する。

保守担当者向けメモ

  • 本関数は自動生成コードであるため、直接の修正は推奨されない。ロジック変更が必要な場合は、対応するSnowballソース（.sbl）を修正すること。
  • I[2]=RV, I[1]=R1, I[0]=R2 に対応する。これらは後続の接尾辞削除処理の適用可否を決定する重要なインデックスである。
