rbt_insert_fixup

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:343

目的

赤黒木へのノード挿入後に発生する可能性のある不変条件（赤ノードの連続など）の違反を修正し、木の平衡を維持する。

シグネチャまたは形状

static void rbt_insert_fixup(RBTree *rbt, RBTNode *x)

入力と出力

  • rbt: 操作対象の赤黒木構造体へのポインタ。
  • x: 挿入された赤ノード。
  • 出力: なし（副作用として rbt 内のノードの色および構造が変更される）。

前提条件と不変条件

  • x は赤ノードであること。
  • 挿入直後の状態から開始し、ループ終了時には赤黒木の不変条件（根は黒、赤ノードの子は黒、各パスの黒ノード数が等しい）が満たされる。

アルゴリズムまたは振る舞い

1. x が根ではなく、かつ x の親が赤である限りループを継続する。 2. x の親が祖父の左の子である場合と右の子である場合で対称的に処理する。 3. **叔父ノードが赤の場合**: 祖父を赤に、親と叔父を黒に塗り替え、x を祖父に移動してループを継続する。 4. **叔父ノードが黒の場合**:

  • x が親と逆側の子供である場合、回転操作により x を親の位置へ移動させる。
  • 親を黒、祖父を赤に塗り替え、祖父を軸に回転させて平衡を保つ。

    • 5. ループ終了後、根ノードの色を強制的に黒に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: rbt_insert（src/backend/lib/rbtree.c:508）
  • 呼び出し先: rbt_rotate_left, rbt_rotate_right

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTNode: 赤黒木のノード。parent、left、right、color メンバを持つ。

失敗モードと境界条件

  • 根ノードが挿入された場合、ループ条件 x != rbt->root により即座に終了し、根が黒に設定される。
  • 叔父ノードが RBTNIL（番兵）である場合も黒として扱われ、回転処理へ移行する。

保守担当者向けメモ

  • 本関数は侵入型設計に基づいているため、ノードのメモリ管理は呼び出し側に依存する。
  • RBTNIL を使用して NULL チェックを回避しているため、ノードポインタの比較時は注意すること。
