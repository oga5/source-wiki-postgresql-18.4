AFFIX

定義

  • **種別**: struct
  • **定義箇所**: src/include/tsearch/dicts/spell.h:87

目的

ISpell および Hunspell 形式の辞書における接辞（接頭辞・接尾辞）ルールを保持するための構造体です。単語の正規化プロセスにおいて、特定のフラグに基づいた文字列の置換やマッチング条件を定義します。

シグネチャまたは形状

typedef struct aff_struct
{
    const char *flag;
    uint32      type:1,
                flagflags:7,
                issimple:1,
                isregis:1,
                replen:14;
    const char *find;
    const char *repl;
    union {
        regex_t    *pregex;
        Regis       regis;
    }           reg;
} AFFIX;

入力と出力

  • **入力**: 辞書ファイルから読み込まれた接辞ルールデータ。
  • **出力**: 形態素解析時に使用される、正規化ルールおよびマッチング条件の集合。

前提条件と不変条件

  • type フィールドは FF_SUFFIX または FF_PREFIX を保持します。
  • reg 共用体は、正規表現（pregex）または Regis 形式のパターンマッチングのいずれか一方を保持します。
  • 辞書構築プロセスにおいて、AFFIX の配列はソートされることが前提となっています。

アルゴリズムまたは振る舞い

  • 辞書構築フェーズにおいて、NIImportAffixes 等を通じて読み込まれ、NISortAffixes により最適化されます。
  • 実行フェーズでは、NINormalizeWord がこの構造体を参照し、単語に対して find 文字列の検索と repl 文字列への置換、あるいは正規表現によるマッチングを行います。
  • regex_t をポインタとして保持することで、構造体自体のサイズを抑え、メモリ上の移動を容易にしています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/spell.c 内の辞書構築および検索ロジック（NIImportAffixes, NISortAffixes, NINormalizeWord 等）から参照されます。
  • **呼び出し先**: regex/regex.h（正規表現処理）および tsearch/dicts/regis.h（Regis 構造体）を利用します。

関連データ構造

  • IspellDict: 辞書全体の状態を管理する構造体であり、AFFIX の配列を保持します。
  • Regis: reg 共用体内で使用されるパターンマッチング用データ構造。

失敗モードと境界条件

  • FLEXIBLE_ARRAY_MEMBER を含む構造体と組み合わせて使用される場合があるため、サイズ計算には SPNHDRSZ などのマクロを使用する必要があります。
  • 辞書構築時にメモリ確保（palloc / repalloc）に失敗した場合、システム全体のエラーハンドリングに従います。

保守担当者向けメモ

  • AFFIX 構造体はメモリ上で移動・ソートされるため、内部に regex_t そのものを持たず、ポインタ経由で参照する設計になっています。この不変条件を維持してください。
  • type や replen 等のビットフィールドは、メモリ効率を考慮して定義されています。フィールドを追加する際は、合計ビット数が uint32 の範囲内に収まるか確認してください。
