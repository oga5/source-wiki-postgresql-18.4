LogicalRepSyncTableStart

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/tablesync.c:1317

目的

論理レプリケーションにおけるテーブル同期ワーカーの初期化処理を実行します。パブリッシャーへの接続確立、同期用スロットの作成、およびデータコピー開始のための準備を行います。

シグネチャまたは形状

char *LogicalRepSyncTableStart(XLogRecPtr *origin_startpos)

入力と出力

  • **入力**: origin_startpos (同期開始位置を格納するポインタ)
  • **出力**: 同期用スロット名（文字列）を返します。接続失敗時やエラー時は ereport(ERROR, ...) により終了します。

前提条件と不変条件

  • MyLogicalRepWorker が初期化されていること。
  • 同期対象テーブルの状態が SUBREL_STATE_INIT, SUBREL_STATE_DATASYNC, SUBREL_STATE_FINISHEDCOPY のいずれかであること。
  • パブリッシャーへの接続情報が MySubscription に保持されていること。

アルゴリズムまたは振る舞い

1. **状態確認**: GetSubscriptionRelState で現在の同期状態を取得し、共有メモリを更新。SYNCDONE 等の場合は finish_sync_worker() を呼び出し終了。 2. **接続**: パブリッシャーへ接続し、同期用スロット名を application_name として設定。 3. **状態別処理**:

  • DATASYNC: 既存の同期用スロットを削除。
  • FINISHEDCOPY: 既存のレプリケーションオリジンをセットアップし、copy_table_done へジャンプ。

    • 4. **初期化**: 新規同期の場合、状態を DATASYNC に更新し、レプリケーションオリジンを作成。 5. **トランザクション開始**: パブリッシャー側で BEGIN READ ONLY ISOLATION LEVEL REPEATABLE READ を実行。 6. **スロット作成**: walrcv_create_slot を呼び出し、キャッチアップフェーズ用の永続スロットを作成。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/tablesync.c 内の TablesyncWorkerMain から呼び出されることが示唆されています。
  • **呼び出し先**: GetSubscriptionRelState, ReplicationSlotNameForTablesync, walrcv_connect, ReplicationSlotDropAtPubNode, replorigin_by_name, walrcv_exec, walrcv_create_slot 等。

関連データ構造

  • MyLogicalRepWorker: 同期ワーカーの共有メモリ状態。
  • MySubscription: サブスクリプション情報。
  • LogRepWorkerWalRcvConn: パブリッシャーとの接続用ハンドル。

失敗モードと境界条件

  • パブリッシャーへの接続失敗時に ERRCODE_CONNECTION_FAILURE でエラー終了。
  • トランザクション開始失敗やスロット作成失敗時にエラー終了。
  • 状態が READY 等の完了済み状態である場合、ワーカーは即座に終了する。

保守担当者向けメモ

  • 本関数は同期ワーカーのライフサイクル管理の要であり、状態遷移（INIT → DATASYNC → FINISHEDCOPY）の整合性が重要です。
  • COPY 中のネットワーク切断やクラッシュ後の再開処理が含まれており、XXX コメントにある通り、スロットのクリーンアップ戦略には改善の余地があります。
