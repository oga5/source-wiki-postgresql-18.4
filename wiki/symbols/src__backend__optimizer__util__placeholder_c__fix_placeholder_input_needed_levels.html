fix_placeholder_input_needed_levels

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:301

目的

クエリ内のすべての PlaceHolderInfo に対して、その式が参照する変数を特定し、評価に必要なリレーション（ph_eval_at）のターゲットリストへそれらの変数を追加することで、必要な入力レベルを保証する。

シグネチャまたは形状

void fix_placeholder_input_needed_levels(PlannerInfo *root)

入力と出力

  • **入力**: root (PlannerInfo *) - 現在のクエリのプランナ情報。root->placeholder_list を走査対象とする。
  • **出力**: なし（副作用として root 内のターゲットリストを更新する）。

前提条件と不変条件

  • root->placeholder_list に有効な PlaceHolderInfo が格納されていること。
  • phinfo->ph_var->phexpr が有効な式ノードであること。

アルゴリズムまたは振る舞い

1. root->placeholder_list に含まれる各 PlaceHolderInfo を順次処理する。 2. pull_var_clause を使用し、プレースホルダ式 (phexpr) 内のすべての変数（集約関数、ウィンドウ関数、および他のプレースホルダを含む）を抽出する。 3. 抽出した変数のリストを add_vars_to_targetlist に渡し、phinfo->ph_eval_at で指定されたリレーションのターゲットリストにこれらの変数を追加する。 4. 一時的に作成した変数のリストを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planmain.c:221
  • **呼び出し先**:
  • pull_var_clause (PostgreSQL内部関数)
  • add_vars_to_targetlist (PostgreSQL内部関数)
  • list_free (PostgreSQL内部関数)

関連データ構造

  • PlannerInfo: クエリ全体の最適化状態を保持。
  • PlaceHolderInfo: プレースホルダの評価場所 (ph_eval_at) や式 (ph_var) を保持。
  • PlaceHolderVar: プレースホルダの式ノード。

失敗モードと境界条件

  • placeholder_list が空の場合、何も行わず正常終了する。
  • 式内に変数が存在しない場合、pull_var_clause は空リストを返し、ターゲットリストへの追加は行われない。

保守担当者向けメモ

  • 本関数は、プレースホルダが評価される場所で、その式に必要なすべての入力変数が利用可能であることを保証するために不可欠である。
  • pull_var_clause のフラグ（PVC_RECURSE_AGGREGATES 等）は、プレースホルダ式に含まれる複雑なノードを正しく捕捉するために設定されている。変更時は注意が必要。
