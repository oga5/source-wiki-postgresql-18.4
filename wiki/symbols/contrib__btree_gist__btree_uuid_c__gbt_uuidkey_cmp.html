gbt_uuidkey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_uuid.c:66

目的

GiST インデックス内における uuidKEY 構造体同士の比較を行い、ソートや順序付けを可能にするための比較関数です。

シグネチャまたは形状

int gbt_uuidkey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象となる Nsrt 構造体へのポインタ（内部で uuidKEY にキャストされます）。
  • flinfo: 関数管理情報（本関数内では使用されていません）。
  • **出力**:
  • int: a が b より小さい場合は負の値、等しい場合は 0、大きい場合は正の値を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体を指している必要があります。
  • uuidKEY 構造体内の lower および upper フィールドが有効な UUID データであることを前提とします。

アルゴリズムまたは振る舞い

1. 入力された Nsrt 構造体から uuidKEY へのポインタを取得します。 2. まず lower フィールド同士を uuid_internal_cmp で比較します。 3. lower の比較結果が 0（等しい）でない場合は、その結果を返します。 4. lower が等しい場合、upper フィールド同士を uuid_internal_cmp で比較し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の静的解析によれば、contrib/btree_gist/btree_uuid.c:90 にて参照されています。
  • **呼び出し先**: uuid_internal_cmp を呼び出しています。

関連データ構造

  • uuidKEY: インデックスノードが保持する UUID の範囲（lower/upper）を定義する構造体。
  • Nsrt: ソート処理等で使用されるラッパー構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、あるいは Nsrt 構造体として正しくキャストできないメモリ領域を指している場合は未定義の動作となります。

保守担当者向けメモ

  • 本関数は uuidKEY の範囲比較において、まず開始点（lower）を優先し、それが一致する場合のみ終了点（upper）を比較する辞書順に近いロジックを採用しています。
  • flinfo 引数は現在の実装では使用されていませんが、将来的な拡張やインターフェースの互換性のために保持されています。
