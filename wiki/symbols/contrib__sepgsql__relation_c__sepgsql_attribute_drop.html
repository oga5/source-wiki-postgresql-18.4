sepgsql_attribute_drop

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/relation.c:132

目的

カラム削除操作時におけるSELinuxの権限チェックを実行します。

シグネチャまたは形状

void sepgsql_attribute_drop(Oid relOid, AttrNumber attnum)

入力と出力

  • **入力**:
  • relOid (Oid): 対象となるリレーション（テーブル）のOID。
  • attnum (AttrNumber): 削除対象となるカラムの属性番号。
  • **出力**: なし。権限チェックに失敗した場合は例外が発生します。

前提条件と不変条件

  • 対象のリレーションが RELKIND_RELATION または RELKIND_PARTITIONED_TABLE である必要があります。それ以外のリレーション種別（ビューやシーケンス等）の場合、本関数は何も行わずに終了します。

アルゴリズムまたは振る舞い

1. get_rel_relkind を使用して対象リレーションの種別を確認し、対象外であれば即座に復帰します。 2. ObjectAddress 構造体を初期化し、classId に RelationRelationId、objectId に relOid、objectSubId に attnum を設定します。 3. getObjectIdentity を呼び出し、監査ログ用の識別名を取得します。 4. sepgsql_avc_check_perms を呼び出し、SELinuxの db_column クラスに対する SEPG_DB_COLUMN__DROP 権限を検証します。 5. 最後に pfree を使用して、取得した監査ログ用文字列のメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:179 から呼び出されます。
  • **呼び出し先**:
  • get_rel_relkind
  • getObjectIdentity
  • sepgsql_avc_check_perms
  • pfree

関連データ構造

  • ObjectAddress: 権限チェック対象のオブジェクトを特定するために使用されます。

失敗モードと境界条件

  • 対象リレーションがテーブルまたはパーティションテーブル以外の場合、権限チェックはスキップされます。
  • sepgsql_avc_check_perms 内で権限違反が検出された場合、PostgreSQLの標準的なエラー処理メカニズムにより処理が中断されます。

保守担当者向けメモ

  • 本関数はカラム削除のフックとして機能します。インデックス等の他のオブジェクト属性操作とは異なり、明示的に db_column クラスを対象としている点に注意してください。
