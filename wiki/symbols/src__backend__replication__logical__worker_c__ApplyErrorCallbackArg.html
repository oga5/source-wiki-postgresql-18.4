ApplyErrorCallbackArg

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/worker.c:213

目的

論理レプリケーションの適用処理中にエラーが発生した際、エラーコンテキスト情報（リモートノードのトランザクション情報や適用中のコマンド種別など）をエラーメッセージに付与するための引数構造体です。

シグネチャまたは形状

typedef struct ApplyErrorCallbackArg
{
    LogicalRepMsgType command;    /* 0 if invalid */
    LogicalRepRelMapEntry *rel;

    /* Remote node information */
    int         remote_attnum;    /* -1 if invalid */
    TransactionId remote_xid;
    XLogRecPtr  finish_lsn;
    char       *origin_name;
} ApplyErrorCallbackArg;

入力と出力

本構造体は、エラー発生時のコンテキスト情報を保持するコンテナとして機能します。各フィールドは、適用中の論理レプリケーションメッセージの状態を反映します。

前提条件と不変条件

  • command が 0 の場合は無効なメッセージタイプを示します。
  • remote_attnum が -1 の場合は無効な属性番号を示します。

アルゴリズムまたは振る舞い

論理レプリケーションワーカーがエラーを報告する際、この構造体を参照して、どのテーブルのどの属性、あるいはどのトランザクションで問題が発生したかを特定し、ログ出力やエラーハンドリングに利用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/worker.c 内の apply_error_callback_arg 静的変数を通じて参照されます。
  • **呼び出し先**: エラー発生時にこの構造体を参照するエラーハンドラ関数（errarg として利用）が存在します。

関連データ構造

  • LogicalRepMsgType: メッセージタイプを定義する列挙型。
  • LogicalRepRelMapEntry: 論理レプリケーションのリレーションマッピング情報。

失敗モードと境界条件

  • remote_xid や finish_lsn が初期値の状態である場合、エラーログには不完全な情報が表示される可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/replication/logical/worker.c 内の静的変数 apply_error_callback_arg を介してグローバル（ファイルスコープ）に管理されています。エラー発生時のコンテキストを正確に保つため、適用処理の各ステップでこれらのフィールドが適切に更新されていることを確認してください。
