CreateFunction

定義

  • 種別: function
  • 定義箇所: src/backend/commands/functioncmds.c:1025

目的

SQLの CREATE FUNCTION および CREATE PROCEDURE 文を処理し、関数やプロシージャの定義をシステムカタログへ登録するためのメインエントリポイントです。

シグネチャまたは形状

ObjectAddress CreateFunction(ParseState *pstate, CreateFunctionStmt *stmt)

入力と出力

  • **入力**:
  • pstate: パーサーの状態情報。
  • stmt: CREATE FUNCTION 文の構文木（CreateFunctionStmt）。
  • **出力**:
  • 作成されたオブジェクトの識別子を含む ObjectAddress を返します。

前提条件と不変条件

  • 対象のスキーマ（名前空間）に対する CREATE 権限が必要です。
  • 言語が指定されていない場合、SQL本体（sql_body）が存在しなければエラーとなります。
  • LEAKPROOF 属性を指定する場合、呼び出し元はスーパーユーザーである必要があります。
  • 信頼されていない言語を使用する場合、呼び出し元はスーパーユーザーである必要があります。

アルゴリズムまたは振る舞い

1. **名前空間の解決**: QualifiedNameGetCreationNamespace を使用して、関数を作成するスキーマを特定します。 2. **権限チェック**: スキーマに対する ACL_CREATE 権限を確認します。 3. **属性抽出**: compute_function_attributes を呼び出し、構文木から言語、揮発性、コスト等の属性を抽出します。 4. **言語検証**: pg_language システムカタログを検索し、言語の存在確認と権限（信頼された言語の場合は ACL_USAGE）を検証します。 5. **変換処理**: transformDefElem が存在する場合、型変換（Transform）のOIDを収集します。 6. **パラメータ解析**: interpret_function_parameter_list を呼び出し、引数リスト、モード、デフォルト値等を解析します。 7. **戻り値の確定**: compute_return_type を呼び出し、戻り値の型と集合（Set）を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（ProcessUtility 経由で呼び出されます）。
  • **呼び出し先**:
  • compute_function_attributes
  • interpret_function_parameter_list
  • compute_return_type
  • SearchSysCache1 / ReleaseSysCache
  • object_aclcheck / aclcheck_error
  • get_transform_oid

関連データ構造

  • CreateFunctionStmt: SQL文の構文木。
  • ParseState: 解析状態。
  • ObjectAddress: 作成されたオブジェクトの参照。
  • pg_proc, pg_language: 操作対象のシステムカタログ。

失敗モードと境界条件

  • 指定した言語が存在しない場合、エラーを報告します。
  • 権限不足（スキーマ作成権限、言語使用権限、スーパーユーザー権限）の場合、エラーを報告します。
  • OUT パラメータと RETURNS 句の型が不一致の場合、エラーを報告します。

保守担当者向けメモ

  • 本関数は CREATE PROCEDURE も処理するため、stmt->is_procedure フラグによる条件分岐が重要です。
  • pg_proc への実際の登録処理は、この関数の後半（提供されたソース範囲外）で行われます。
  • 属性解析ロジックを変更する場合は、compute_function_attributes との整合性に注意してください。
