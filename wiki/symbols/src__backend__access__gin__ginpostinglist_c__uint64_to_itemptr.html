uint64_to_itemptr

定義

  • 種別: static inline function
  • 定義箇所: src/backend/access/gin/ginpostinglist.c:101

目的

64ビット整数表現から ItemPointer（TID）構造体へ変換します。GINインデックス内で圧縮・格納されたTIDデータを復元する際に使用されます。

シグネチャまたは形状

static inline void uint64_to_itemptr(uint64 val, ItemPointer iptr)

入力と出力

  • **入力**:
  • val: 変換元の64ビット整数値。
  • iptr: 変換結果を格納する ItemPointer へのポインタ。
  • **出力**: なし（iptr が指すメモリ領域を更新）。

前提条件と不変条件

  • val は、MaxHeapTuplesPerPageBits を用いてエンコードされた形式である必要があります。
  • 変換後、iptr は ItemPointerIsValid マクロによって有効なポインタであると判定される必要があります。

アルゴリズムまたは振る舞い

1. val の下位 MaxHeapTuplesPerPageBits ビットを抽出し、GinItemPointerSetOffsetNumber を用いて iptr のオフセット番号として設定します。 2. val を MaxHeapTuplesPerPageBits 分だけ右シフトし、残りの値を GinItemPointerSetBlockNumber を用いて iptr のブロック番号として設定します。 3. 最後に Assert(ItemPointerIsValid(iptr)) を実行し、生成された ItemPointer が有効であることを検証します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginpostinglist.c:343（ginPostingListDecodeAllSegments 内での復元処理）
  • **呼び出し先**:
  • GinItemPointerSetOffsetNumber（マクロ）
  • GinItemPointerSetBlockNumber（マクロ）
  • ItemPointerIsValid（マクロ）

関連データ構造

  • ItemPointer: PostgreSQLのタプル識別子（TID）。
  • GinItemPointer: GINインデックス内部で利用されるTID操作用マクロ群。

失敗モードと境界条件

  • val が ItemPointer として表現可能な範囲を超えている場合、ブロック番号やオフセット番号が正しく復元されません。
  • MaxHeapTuplesPerPageBits（11ビット）の定数定義に依存しており、この値が変更された場合、既存の圧縮データとの互換性が失われます。

保守担当者向けメモ

  • 本関数は ginPostingListDecodeAllSegments 等の展開処理において、圧縮された整数値を物理的なTID構造へ戻すための重要なユーティリティです。
  • 性能向上のため static inline として定義されています。修正時はインライン展開の効率に留意してください。
