pgaio_worker_submission_queue_consume

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_worker.c:201

目的

共有メモリ上のI/O投入用リングバッファ（PgAioWorkerSubmissionQueue）から、次に処理すべきI/OリクエストのIDを取り出します。

シグネチャまたは形状

static int pgaio_worker_submission_queue_consume(void)

入力と出力

  • **入力**: グローバル変数 io_worker_submission_queue が指す共有メモリ上のキュー。
  • **出力**:
  • 成功時: キューから取り出したI/OリクエストのID（int）。
  • 失敗時: -1（キューが空の場合）。

前提条件と不変条件

  • 本関数は、AioWorkerSubmissionQueueLock による排他制御が呼び出し元で確保されている環境下で実行されることを前提としています。
  • キューのサイズは2のべき乗である必要があり、tail の更新にはビットマスク演算 (queue->size - 1) が使用されます。

アルゴリズムまたは振る舞い

1. io_worker_submission_queue を参照し、tail と head を比較します。 2. tail == head であればキューが空と判断し、-1 を返します。 3. キューが空でない場合、tail が指す位置の要素（I/O ID）を取得します。 4. tail をインクリメントし、キューのサイズでラップアラウンド（循環）させます。 5. 取得したI/O IDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: IoWorkerMain（src/backend/storage/aio/method_worker.c:473）において、ワーカープロセスが処理すべきジョブを取得するために呼び出されます。
  • **呼び出し先**: 外部関数への直接的な呼び出しはありません。

関連データ構造

  • PgAioWorkerSubmissionQueue: I/Oリクエストを保持するリングバッファ構造体。

失敗モードと境界条件

  • **キューが空**: tail と head が一致する場合、-1 を返して正常に終了します。これはエラーではなく、待機状態へ移行すべき合図として扱われます。

保守担当者向けメモ

  • 本関数はロックを保持しないため、呼び出し元で必ず AioWorkerSubmissionQueueLock を取得していることを確認してください。
  • キューのサイズ計算にビットマスクを使用しているため、queue->size が2のべき乗でない場合、リングバッファの整合性が破壊されます。
