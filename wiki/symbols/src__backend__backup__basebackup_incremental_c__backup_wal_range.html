backup_wal_range

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup_incremental.c:45

目的

増分バックアップにおいて、特定のタイムライン上でWAL（Write-Ahead Log）がカバーする範囲を定義するために使用されます。

シグネチャまたは形状

typedef struct
{
	TimeLineID	tli;
	XLogRecPtr	start_lsn;
	XLogRecPtr	end_lsn;
} backup_wal_range;

入力と出力

本構造体はデータ保持用であり、関数ではありません。各フィールドは以下の通りです。

  • tli (TimeLineID): WAL範囲が属するタイムラインID。
  • start_lsn (XLogRecPtr): 範囲の開始位置を示すログシーケンス番号。
  • end_lsn (XLogRecPtr): 範囲の終了位置を示すログシーケンス番号。

前提条件と不変条件

  • start_lsn は end_lsn 以前である必要があります。
  • タイムライン履歴と照合される際、有効な tli を保持していることが期待されます。

アルゴリズムまたは振る舞い

本構造体は、バックアップマニフェストから抽出されたWAL範囲を保持し、サーバー側のタイムライン履歴との照合や、必要なWALサマリーファイルの特定に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/backup/basebackup_incremental.c 内の各関数（PrepareForIncrementalBackup 等）が、マニフェスト情報の解析や処理の過程で本構造体をインスタンス化または参照します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な呼び出し先はありません。

関連データ構造

  • IncrementalBackupInfo: 本構造体のリスト（manifest_wal_ranges）を保持する主要な管理構造体。

失敗モードと境界条件

  • メモリ不足: palloc による動的確保時に失敗する可能性があります。
  • 不整合なLSN: マニフェストから読み込まれた start_lsn と end_lsn が論理的に矛盾している場合、後続のWALサマリー統合処理でエラーとなる可能性があります。

保守担当者向けメモ

  • 本構造体は basebackup_incremental.c 内で頻繁にリスト操作の対象となります。
  • タイムラインの切り替わりが発生する環境でのバックアップにおいて、tli の整合性が正しく維持されているか注意してください。
