relopt_enum_elt_def

定義

  • 種別: struct
  • 定義箇所: src/include/access/reloptions.h:117

目的

列挙型（enum）として定義されたリレーションオプションにおいて、各許容値の名称とそれに対応する内部的な整数値をマッピングするために使用されます。

シグネチャまたは形状

typedef struct relopt_enum_elt_def
{
    const char *string_val; /* オプションの文字列表現 */
    int         symbol_val; /* 対応する内部整数値 */
} relopt_enum_elt_def;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではありません。
  • relopt_enum 構造体の一部として、オプション定義の配列を構成する要素として利用されます。

前提条件と不変条件

  • string_val は、対応するオプションの有効な文字列値である必要があります。
  • symbol_val は、そのオプション型が内部的に扱う整数値と一致している必要があります。

アルゴリズムまたは振る舞い

  • リレーションオプションの解析時、ユーザーが指定した文字列を string_val と照合し、一致した場合は対応する symbol_val を設定値として採用します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/common/reloptions.c 内の各オプション定義（StdRdOptIndexCleanupValues, gistBufferingOptValues, viewCheckOptValues 等）で静的に初期化されます。
  • 呼び出し先: reloptions.c 内のオプション解析関数群が、この構造体の配列を走査して値の検証と変換を行います。

関連データ構造

  • relopt_enum: 列挙型オプションを定義するための構造体。本構造体の配列を保持します。

失敗モードと境界条件

  • ユーザー指定の文字列が members 配列内のどの string_val とも一致しない場合、オプションの解析エラーとなります。

保守担当者向けメモ

  • 新しい列挙型オプションを追加する際は、この構造体の配列を定義し、relopt_enum 構造体に渡す必要があります。
  • string_val の比較は、大文字小文字を区別するかどうかなど、解析側の実装に依存します。
