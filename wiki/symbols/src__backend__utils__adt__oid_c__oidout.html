oidout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:46

目的

Oid（Object Identifier）型の値を、人間が読み取り可能なテキスト形式（文字列）に変換します。

シグネチャまたは形状

Datum oidout(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る Oid 型の値。
  • **出力**: Datum 型として返される、Oid 値の文字列表現（char *）。

前提条件と不変条件

  • palloc(12) により、数値文字列を格納するために十分なメモリ（12バイト）が確保されます。
  • Oid は符号なし32ビット整数であるため、最大値でも10桁＋終端文字で収まることが保証されています。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID(0) を使用して、引数から Oid 値を取得します。 2. palloc を用いて、結果文字列を保持するための12バイトのメモリ領域を割り当てます。 3. snprintf を使用し、Oid 値を符号なし10進整数として文字列に変換します。 4. PG_RETURN_CSTRING を介して、変換された文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plperl/plperl.c
  • src/pl/plpython/plpy_exec.c
  • src/pl/tcl/pltcl.c
  • その他、SQLインターフェースを介した呼び出し。
  • **呼び出し先**:
  • palloc (メモリ割り当て)
  • snprintf (文字列フォーマット)

関連データ構造

  • Oid: PostgreSQLのシステムカタログ等で使用されるオブジェクト識別子（uint32）。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • Oid の値が uint32 の範囲内であれば、snprintf による変換は常に成功します。

保守担当者向けメモ

  • 本関数は Oid 型の標準的なテキスト出力ルーチンです。変更を加える際は、他の数値型出力関数との整合性に注意してください。
