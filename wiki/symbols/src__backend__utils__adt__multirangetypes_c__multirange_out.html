multirange_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes.c:298

目的

MultirangeType 構造体を、SQL のテキスト表現（例: "{[1,2],[3,4]}"）に変換する。

シグネチャまたは形状

Datum multirange_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して MultirangeType ポインタを受け取る。
  • **出力**: cstring 型のデータ（Datum）として、マルチレンジの文字列表現を返す。

前提条件と不変条件

  • 入力される MultirangeType は、正規化（ソート済みかつ非重複）されている必要がある。
  • get_multirange_io_data を通じて、対象のレンジ型に対応する出力関数が取得可能であること。

アルゴリズムまたは振る舞い

1. get_multirange_io_data を呼び出し、レンジ型の出力処理に必要なキャッシュ情報を取得する。 2. StringInfoData バッファを初期化し、開始文字 { を追加する。 3. multirange_deserialize を使用して、マルチレンジから個々の RangeType の配列を抽出する。 4. 各レンジをループで走査し、OutputFunctionCall を用いて個別のレンジを文字列に変換してバッファに追加する。 5. レンジ間にはカンマ , を挿入する。 6. 最後に終了文字 } を追加し、バッファの内容を cstring として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pseudotypes.c 内の関数（multirange_out を直接呼び出している）。
  • **呼び出し先**:
  • get_multirange_io_data
  • initStringInfo
  • appendStringInfoChar
  • multirange_deserialize
  • OutputFunctionCall
  • appendStringInfoString

関連データ構造

  • MultirangeType: マルチレンジのメモリ上の表現。
  • MultirangeIOData: I/O 処理用のキャッシュ構造体。
  • StringInfoData: 文字列構築用バッファ。
  • RangeType: 個々のレンジ型。

失敗モードと境界条件

  • メモリ不足時、StringInfo 関連関数がエラーを発生させる可能性がある。
  • multirange_deserialize や OutputFunctionCall が内部でエラーを発生させた場合、処理は中断される。

保守担当者向けメモ

  • 本関数は MultirangeType のシリアライズを担当する。レンジの境界値が複雑な場合でも、OutputFunctionCall に依存することで各レンジ型の出力ロジックを再利用している。
  • multirange_deserialize の結果として得られる ranges 配列のメモリ管理は、呼び出し元のコンテキストに依存する。
