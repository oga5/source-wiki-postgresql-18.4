XLogPageReadResult

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlogreader.h:349

目的

WAL（Write-Ahead Log）のページ読み込み操作の結果状態を表現し、読み込み処理の成否や制御フローの状態を呼び出し元に伝達するために使用されます。

シグネチャまたは形状

typedef enum XLogPageReadResult
{
    XLREAD_SUCCESS = 0,
    XLREAD_FAIL = -1,
    XLREAD_WOULDBLOCK = -2,
} XLogPageReadResult;

入力と出力

  • **出力**: WAL 読み込み処理（XLogDecodeNextRecord 等）の戻り値として返されます。

前提条件と不変条件

  • XLREAD_SUCCESS はレコードの読み込みが正常に完了したことを示します。
  • XLREAD_WOULDBLOCK は非ブロッキングモードでのみ発生し、現時点で利用可能なデータがないことを示します。

アルゴリズムまたは振る舞い

  • XLREAD_SUCCESS: 読み込み成功。
  • XLREAD_FAIL: 読み込みプロセス中にエラーが発生。
  • XLREAD_WOULDBLOCK: 非ブロッキングモードにおいて、データが未到着または準備中であることを示す制御値。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/access/transam/xlogreader.c: XLogDecodeNextRecord
  • src/backend/access/transam/xlogrecovery.c: WaitForWALToBecomeAvailable
  • **呼び出し先**:
  • 呼び出し先に関する情報は提供されていません。

関連データ構造

  • XLogReaderState: WAL 読み込みのセッション状態を保持する構造体。

失敗モードと境界条件

  • XLREAD_FAIL: 物理的な読み込みエラーや、レコードの整合性チェック失敗時に返されます。
  • XLREAD_WOULDBLOCK: 非ブロッキングモードが有効な場合にのみ発生する境界条件です。

保守担当者向けメモ

  • 本シンボルは xlogreader モジュールの状態遷移を制御する重要な列挙型です。
  • 戻り値の解釈は xlogreader.c および xlogrecovery.c のロジックに依存しているため、変更時はこれらのファイルにおける条件分岐を確認してください。
