local_traverse_files

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/local_source.c:57

目的

ローカルファイルシステム上のデータディレクトリを再帰的に走査し、各ファイルに対して指定されたコールバック関数を実行する。

シグネチャまたは形状

static void local_traverse_files(rewind_source *source, process_file_callback_t callback)

入力と出力

  • source: rewind_source 構造体へのポインタ。内部で local_source 型にキャストされ、データディレクトリのパスが参照される。
  • callback: process_file_callback_t 型の関数ポインタ。走査中に各ファイルに対して呼び出される。
  • 戻り値: なし（void）。

前提条件と不変条件

  • source 引数は local_source 構造体として初期化済みであること。
  • callback は有効な関数ポインタであること。

アルゴリズムまたは振る舞い

1. source を local_source 型にキャストし、保持されている datadir（データディレクトリパス）を取得する。 2. traverse_datadir 関数を呼び出し、取得したパスと callback を渡してディレクトリ内のファイルを再帰的に探索する。

呼び出し元と呼び出し先

  • 呼び出し元: init_local_source 関数内で、rewind_source インターフェースの関数ポインタとして登録される。
  • 呼び出し先: traverse_datadir（ディレクトリ走査の実行）。

関連データ構造

  • rewind_source: 抽象化されたソースインターフェース。
  • local_source: ローカルファイルシステム用の実装構造体。

失敗モードと境界条件

  • traverse_datadir が内部でエラーを発生させた場合の挙動は、当該関数の実装に依存する。

保守担当者向けメモ

  • 本関数は rewind_source インターフェースの具現化の一部であり、local_source 構造体へのキャストを前提としている。
  • 走査ロジック自体は traverse_datadir に委譲されているため、走査アルゴリズムの変更が必要な場合はそちらを確認すること。
