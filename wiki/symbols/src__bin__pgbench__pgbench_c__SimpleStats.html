SimpleStats

定義

  • 種別: struct
  • 定義箇所: src/bin/pgbench/pgbench.c:358

目的

ベンチマーク実行中に収集される数値データ（レイテンシやラグなど）の統計情報を保持するための構造体です。

シグネチャまたは形状

typedef struct SimpleStats
{
    int64       count;  /* 観測された値の数 */
    double      min;    /* 観測された最小値 */
    double      max;    /* 観測された最大値 */
    double      sum;    /* 値の合計 */
    double      sum2;   /* 値の二乗和 */
} SimpleStats;

入力と出力

  • 本構造体自体はデータ保持用であり、直接的な入出力関数は持ちませんが、addToSimpleStats 関数を介して値の蓄積が行われます。

前提条件と不変条件

  • initSimpleStats 関数によってゼロ初期化されることが想定されています。
  • sum2 フィールドは、分散や標準偏差の計算に利用されることを前提とした二乗和を保持します。

アルゴリズムまたは振る舞い

  • 統計情報の蓄積（addToSimpleStats）: 新しい値が追加されるたびに、count をインクリメントし、min/max を更新し、sum および sum2 を加算します。
  • 統計情報の統合（mergeSimpleStats）: 複数の SimpleStats インスタンスを合算し、全体的な統計を算出します。

呼び出し元と呼び出し先

  • 呼び出し元: pgbench.c 内の各所で、トランザクションのレイテンシやラグの計測に使用されています。
  • 呼び出し先: 以下の関数が本構造体を操作します。
  • initSimpleStats: 初期化
  • addToSimpleStats: 値の追加
  • mergeSimpleStats: 統計の統合
  • printSimpleStats: 結果の出力

関連データ構造

  • StatsData: pgbench におけるより広範な統計情報を管理する構造体。
  • TState: スレッドごとの状態を保持し、その中で SimpleStats が利用されます。

失敗モードと境界条件

  • count が 0 の状態で統計計算（平均や分散）を行うと、ゼロ除算が発生する可能性があるため、利用側で count > 0 のチェックが必要です。

保守担当者向けメモ

  • 本構造体は pgbench.c 内で定義されていますが、pgbench.h との整合性に注意してください。
  • sum2 の計算において、値が非常に大きい場合に浮動小数点数の精度制限による誤差が生じる可能性がある点に留意してください。
