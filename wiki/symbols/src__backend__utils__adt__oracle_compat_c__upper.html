upper

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:79

目的

入力された text 型文字列をすべて大文字に変換し、新しい text 型として返します。Oracle データベースとの互換性を目的とした関数です。

シグネチャまたは形状

Datum upper(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る text 型のデータ。
  • **出力**: 入力文字列の全文字を大文字に変換した新しい text 型の Datum。

前提条件と不変条件

  • 入力文字列は PostgreSQL の text 型として有効である必要があります。
  • 内部で PG_GET_COLLATION() を使用しているため、現在のデータベースの照合順序（Collation）設定に依存します。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP(0) を使用して、入力された text 型データを取得します。 2. str_toupper を呼び出し、入力文字列のデータポインタ、ヘッダーを除いたサイズ、および現在の照合順序を渡して大文字変換を実行します。 3. str_toupper が返した char * 文字列を cstring_to_text で PostgreSQL の text 型に変換します。 4. 変換に使用した一時的な char * バッファを pfree で解放します。 5. PG_RETURN_TEXT_P を使用して、変換後の text 型データを返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • str_toupper (utils/formatting.h): 大文字変換のコアロジック。
  • cstring_to_text: C 文字列から text 型への変換。
  • pfree: メモリ解放。
  • **呼び出し元**:
  • ソースコード内の直接的な呼び出し元は、提供された情報からは特定できません。ツールによる検索結果に表示される upper は、本関数とは異なる文脈（amcheck や btree_gist 等の構造体メンバや変数名）を指しており、本関数への参照ではありません。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • メモリ不足: palloc（cstring_to_text 内部で使用）が失敗した場合、PostgreSQL のエラーハンドリングによりアボートされます。
  • エンコーディング: データベースのエンコーディング設定に依存し、マルチバイト文字が含まれる場合は str_toupper の実装に従って適切に処理されます。

保守担当者向けメモ

  • 本関数は utils/formatting.h の str_toupper をラップする薄いラッパーです。変換ロジックの変更が必要な場合は、formatting.c 側の実装を確認してください。
  • ツールによる呼び出し元検索結果には、本関数と名前が重複する別のシンボルが含まれています。混同しないよう注意してください。
