psprintf

定義

  • 種別: function
  • 定義箇所: src/common/psprintf.c:42

目的

可変長引数を受け取り、フォーマット済みの文字列を格納したメモリ領域を動的に確保して返します。呼び出し側はバッファサイズを事前に指定する必要がありません。

シグネチャまたは形状

char *psprintf(const char *fmt, ...);

入力と出力

  • **入力**: fmt（フォーマット文字列）、および可変長引数。
  • **出力**: フォーマットされた文字列を格納した char * 型のメモリ領域。

前提条件と不変条件

  • 呼び出し側は、返されたメモリ領域を pfree（バックエンド）または free（フロントエンド）で解放する責任があります。
  • pvsnprintf が内部で使用されるため、呼び出し側で errno が保持されている必要があります（%m 指定子によるエラーメッセージ出力のため）。
  • バックエンドでは palloc を使用するため、適切なメモリコンテキストで呼び出す必要があります。

アルゴリズムまたは振る舞い

1. 初期バッファサイズを 128 バイトと仮定します。 2. palloc を使用してバッファを確保します。 3. pvsnprintf を呼び出し、フォーマットを試みます。 4. pvsnprintf が成功（戻り値がバッファサイズ未満）した場合は、そのバッファを返します。 5. バッファが不足している場合、pfree で現在のバッファを解放し、pvsnprintf が要求したサイズで再試行します。 6. エラー発生時は、環境に応じてプロセスを終了またはエラー報告を行います。

呼び出し元と呼び出し先

  • **呼び出し先**: palloc、pvsnprintf、pfree、va_start、va_end。
  • **呼び出し元**: contrib/amcheck/verify_heapam.c 内の各箇所（ヒープ整合性チェック時のエラーメッセージ生成など）。

関連データ構造

  • va_list: 可変長引数を処理するために使用。

失敗モードと境界条件

  • メモリ不足時: palloc の仕様に従い、バックエンドではエラー報告、フロントエンドでは exit を伴う終了が発生します。
  • ライブラリ利用時の注意: 本関数はエラー時にプロセスを終了させる可能性があるため、libpq 等のライブラリ内での使用には注意が必要です。

保守担当者向けメモ

  • StringInfo 型と比較して、頻繁な文字列構築には StringInfo の方が効率的です。
  • フロントエンド環境では malloc にマッピングされ、エラー時に終了する挙動をとります。
