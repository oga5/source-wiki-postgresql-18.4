WalCompression

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlog.h:80

目的

WAL（先行書き込みログ）レコードの圧縮アルゴリズムを選択するための列挙型定義です。

シグネチャまたは形状

typedef enum WalCompression
{
	WAL_COMPRESSION_NONE = 0,
	WAL_COMPRESSION_PGLZ,
	WAL_COMPRESSION_LZ4,
	WAL_COMPRESSION_ZSTD,
} WalCompression;

入力と出力

  • 入力: なし（列挙定数）
  • 出力: なし（列挙定数）

前提条件と不変条件

  • WAL_COMPRESSION_NONE は 0 に初期化されます。
  • 各列挙値は、WAL 挿入時の圧縮処理におけるアルゴリズムの識別子として使用されます。

アルゴリズムまたは振る舞い

WAL レコードの生成時に、システム設定に基づいて圧縮アルゴリズムを切り替えるための識別子として機能します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • src/backend/access/transam/xloginsert.c: wal_compression 変数の値をキャストして圧縮アルゴリズムを判定する switch 文で使用されています。
  • 呼び出し先:
  • 情報なし。

関連データ構造

  • WalLevel: WAL の詳細度を決定する列挙型（関連するシステム設定）。

失敗モードと境界条件

  • 未定義の整数値がキャストされた場合の挙動については、ソースコード上の明示的なガード条件は本抜粋からは確認できません。

保守担当者向けメモ

  • 本列挙型に新しい圧縮アルゴリズムを追加する場合は、xloginsert.c 内の switch 文を更新し、対応する圧縮処理の実装を追加する必要があります。
