IsReservedOriginName

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/origin.c:208

目的

指定されたレプリケーション起点の名前が、システム予約語（none または any）と一致するかを判定します。

シグネチャまたは形状

static bool IsReservedOriginName(const char *name)

入力と出力

  • **入力**: name (const char *) - チェック対象の起点名。
  • **出力**: bool - 名前が予約済みであれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力 name はヌル終端文字列である必要があります。
  • 比較はケースインセンシティブ（大文字小文字を区別しない）で行われます。

アルゴリズムまたは振る舞い

1. pg_strcasecmp を使用して、入力された name を LOGICALREP_ORIGIN_NONE と比較します。 2. 同様に、LOGICALREP_ORIGIN_ANY と比較します。 3. いずれかと一致する場合に true を返し、両方と一致しない場合に false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/origin.c:1305 において、名前の妥当性検証の一部として呼び出されています。
  • **呼び出し先**: pg_strcasecmp（標準的な文字列比較関数）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • name が NULL の場合、pg_strcasecmp の仕様に依存した動作（通常はセグメンテーション違反）となります。

保守担当者向けメモ

  • 本関数は static であり、origin.c 内でのみ使用されます。
  • 予約語の定義が変更された場合は、LOGICALREP_ORIGIN_NONE および LOGICALREP_ORIGIN_ANY のマクロ定義を確認してください。
