gbt_bytealt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bytea.c:55

目的

bytea型に対する「より小さい（less than）」比較演算を、GiSTインデックスの内部ロジックから呼び出し可能な形式で提供します。

シグネチャまたは形状

static bool gbt_bytealt(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のbyteaデータへのポインタ。
  • collation: 照合順序（本関数では未使用）。
  • flinfo: 関数呼び出し情報（本関数では未使用）。
  • **出力**: a < b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は有効なbyteaデータ構造を指している必要があります。
  • PostgreSQLの標準関数 bytealt が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. a と b を Datum 型に変換します。 2. PostgreSQLのコア関数 bytealt を DirectFunctionCall2 を介して呼び出します。 3. 呼び出し結果（Datum）を bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: bytealt（PostgreSQLコアの比較関数）。
  • **呼び出し元**: contrib/btree_gist/btree_bytea.c 内の静的配列（gbtree_vinfo 構造体の初期化用）から参照されています。具体的な実行時の呼び出し元については、提供されたソース情報からは特定できません。

関連データ構造

  • gbtree_vinfo: btree_bytea.c 内で定義される、bytea型のGiSTインデックス用サポート情報を保持する構造体。本関数はこの構造体を通じて汎用ユーティリティ関数から利用されます。

失敗モードと境界条件

  • bytealt 関数が内部的にエラーを発生させた場合、例外がスローされます。
  • a または b が NULL の場合の挙動は、基底となる bytealt 関数の仕様に依存します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、ロジックの大部分は bytealt に委譲されています。
  • collation や flinfo が渡されていますが、現在の実装では使用されていません。将来的な拡張で必要になる可能性があります。
