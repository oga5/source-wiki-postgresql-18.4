r_prelude

定義

目的

ポルトガル語のステミング処理の前処理として、特定の文字（ã や õ など）を内部表現に変換し、後続の処理に適した形式へ正規化する。

シグネチャまたは形状

static int r_prelude(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （Snowball ランタイム環境構造体）
  • 出力: 成功時に 1 を返す。内部処理で slice_from_s が失敗した場合は、そのエラーコード（負の値）を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • a_0 配列に定義されたルールに基づき、入力文字列を走査する。

アルゴリズムまたは振る舞い

1. 文字列全体をループで走査する。 2. find_among(z, a_0, 3) を使用して、現在のカーソル位置にある文字が変換対象（ã または õ に該当するコード）か判定する。 3. 対象が見つかった場合、slice_from_s を用いて対応する内部表現に置換する。 4. 対象外の文字であれば、カーソルを1つ進める。 5. 文字列の末尾に達するまでこれを繰り返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_ISO_8859_1_portuguese.c 内のメイン処理から呼び出される。
  • 呼び出し先: find_among（ユーティリティ関数）、slice_from_s（Snowball ランタイム関数）。

関連データ構造

  • struct SN_env: ステミングの状態を保持する構造体。
  • a_0: 変換ルールを定義した among 構造体の配列。

失敗モードと境界条件

  • slice_from_s が失敗した場合、その戻り値をそのまま伝播させる。
  • 文字列の末尾（z->l）に達した際のループ終了条件が適切に制御されている。
