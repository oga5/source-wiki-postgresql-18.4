LogicalDecodingContext

定義

  • 種別: struct
  • 定義箇所: src/include/replication/logical.h:33

目的

論理デコーディングセッション全体の状態を保持・管理する中核的なデータ構造です。WALの読み込みから、出力プラグインを通じた変更データのストリーミング、レプリケーションスロットの進捗管理までを一元的に制御します。

シグネチャまたは形状

typedef struct LogicalDecodingContext {
    MemoryContext context;
    ReplicationSlot *slot;
    XLogReaderState *reader;
    struct ReorderBuffer *reorder;
    struct SnapBuild *snapshot_builder;
    bool fast_forward;
    OutputPluginCallbacks callbacks;
    OutputPluginOptions options;
    List *output_plugin_options;
    LogicalOutputPluginWriterPrepareWrite prepare_write;
    LogicalOutputPluginWriterWrite write;
    LogicalOutputPluginWriterUpdateProgress update_progress;
    StringInfo out;
    void *output_plugin_private;
    void *output_writer_private;
    bool streaming;
    bool twophase;
    bool twophase_opt_given;
    bool accept_writes;
    bool prepared_write;
    XLogRecPtr write_location;
    TransactionId write_xid;
    bool end_xact;
    bool processing_required;
} LogicalDecodingContext;

入力と出力

  • **入力**: XLogReaderState を介したWALデータ、および出力プラグイン経由で提供される設定やコールバック。
  • **出力**: StringInfo out バッファを介して変換された変更データが、write コールバックを通じて外部（walsender 等）へストリーミングされます。

前提条件と不変条件

  • fast_forward モードが有効な場合、出力プラグインはロードされず、関連するプロパティ（callbacks 等）は使用されません。
  • twophase フラグはプラグインが二相コミット用コールバックを提供しているかを示し、twophase_opt_given はユーザーが明示的にオプションを指定したかを示します。

アルゴリズムまたは振る舞い

1. **初期化**: CreateDecodingContext によりコンテキストが生成され、WALリーダーやプラグインが紐付けられます。 2. **デコーディング**: ReorderBuffer がトランザクション単位で変更を蓄積し、コミット時にプラグインのコールバックを呼び出します。 3. **書き出し**: prepare_write / write コールバックを通じて、プラグインが変換したデータをストリーミングします。 4. **進捗管理**: update_progress を通じてレプリケーションスロットの xmin や restart_lsn を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/test_decoding/test_decoding.c 内の各デコーディング関数（pg_decode_startup, pg_decode_commit_txn 等）が、この構造体を引数として受け取り操作します。
  • **呼び出し先**: prepare_write, write, update_progress などの関数ポインタを通じて、外部の書き出し処理やプラグイン固有のロジックを呼び出します。

関連データ構造

  • ReplicationSlot: 関連付けられたレプリケーションスロット。
  • XLogReaderState: WAL読み込み用リーダー。
  • ReorderBuffer: トランザクション変更のバッファリング用。
  • SnapBuild: スナップショット構築用。
  • OutputPluginCallbacks: プラグインが実装するコールバック関数群。

失敗モードと境界条件

  • fast_forward モード時は、プラグイン関連のコールバックやオプションが無視されるため、これらに依存する処理は実行されません。
  • accept_writes や prepared_write の状態管理が不整合な場合、データの書き出し処理が正しく行われない可能性があります。

保守担当者向けメモ

  • 本構造体は論理レプリケーションの「制御センター」です。変更を加える際は、ReorderBuffer や SnapBuild との整合性に注意してください。
  • fast_forward モードの挙動については、プラグインがロードされないという制約を前提として設計されています。
