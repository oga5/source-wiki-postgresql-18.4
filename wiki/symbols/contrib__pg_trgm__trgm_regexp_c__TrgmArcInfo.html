TrgmArcInfo

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_regexp.c:360

目的

正規表現から変換されたグラフ構造において、状態遷移（アーク）の接続情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct
{
	TrgmState  *source;
	TrgmState  *target;
} TrgmArcInfo;

入力と出力

  • 入力: TrgmState 型のポインタ source（遷移元）および target（遷移先）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • source および target は、グラフ構築プロセスにおいて有効な TrgmState を指している必要があります。

アルゴリズムまたは振る舞い

  • グラフの構築および走査過程において、ある状態から別の状態への接続関係を定義します。
  • palloc を用いてメモリ確保され、リスト構造（List）の要素として管理されることが一般的です。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_trgm/trgm_regexp.c 内のグラフ構築および走査ロジック（1485, 1607, 1684, 1721 行付近）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TrgmState: グラフのノード（状態）を表す構造体。
  • TrgmArc: グラフ内のアークを表現する関連構造体。

失敗モードと境界条件

  • メモリ不足時には palloc によりエラーが発生する可能性があります。
  • グラフ構築時に無効な TrgmState を参照した場合、後続のグラフ走査で不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は TrgmState 間の接続を単純に保持する役割を持ちます。グラフの最適化やパッキングの過程で、この情報がどのように TrgmPackedGraph へ変換されるかを追跡する際は、trgm_regexp.c 内のリスト操作ロジックを参照してください。
