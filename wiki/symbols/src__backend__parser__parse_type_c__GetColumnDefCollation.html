GetColumnDefCollation

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_type.c:539

目的

列定義（ColumnDef）において、適用すべき照合順序（Collation）の OID を決定する。

シグネチャまたは形状

Oid GetColumnDefCollation(ParseState *pstate, const ColumnDef *coldef, Oid typeOid);

入力と出力

  • **入力**:
  • pstate: パーサーの状態（エラー報告用）。
  • coldef: 照合順序の指定を含む列定義構造体。
  • typeOid: 対象となる型の OID。
  • **出力**: 決定された照合順序の Oid を返す。

前提条件と不変条件

  • typeOid に基づく型が照合順序をサポートしない場合、COLLATE 句が指定されていればエラーを発生させる。

アルゴリズムまたは振る舞い

1. get_typcollation(typeOid) を呼び出し、型のデフォルト照合順序を取得する。 2. coldef->collClause が存在する場合、LookupCollation を呼び出して指定された照合順序を検索・解決する。 3. coldef->collClause がなく、coldef->collOid が有効な場合、その値を採用する。 4. 上記いずれでもない場合、型のデフォルト照合順序を使用する。 5. 決定された照合順序が有効であり、かつ対象の型が照合順序をサポートしていない（typcollation が無効）場合、ERRCODE_DATATYPE_MISMATCH でエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c (テーブル定義や変更処理)
  • src/backend/parser/parse_relation.c (リレーション解析)
  • **呼び出し先**:
  • get_typcollation
  • LookupCollation
  • ereport / format_type_be / parser_errposition

関連データ構造

  • ColumnDef: 列定義情報を保持する構造体。
  • ParseState: 構文解析中の状態を保持する構造体。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • 指定された照合順序がシステム内に存在しない場合、LookupCollation がエラーを発生させる。
  • 照合順序をサポートしない型に対して明示的に COLLATE 句が指定された場合、ERRCODE_DATATYPE_MISMATCH で処理を中断する。

保守担当者向けメモ

  • 本関数は ColumnDef 内の collClause（構文解析直後の未解決状態）と collOid（解決済み状態）の両方を扱うため、呼び出し元がどの段階にあるかに注意すること。
  • 戻り値の Oid が InvalidOid である場合、その列は照合順序を持たないことを意味する。
