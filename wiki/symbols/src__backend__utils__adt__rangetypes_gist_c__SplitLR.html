SplitLR

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/rangetypes_gist.c:62

目的

GiSTインデックスのページ分割処理において、特定の範囲やグループを左側のページ（SPLIT_LEFT）と右側のページ（SPLIT_RIGHT）のどちらに配置するかを識別するために使用されます。

シグネチャまたは形状

typedef enum {
    SPLIT_LEFT = 0,
    SPLIT_RIGHT,
} SplitLR;

入力と出力

  • 本シンボルは列挙型であり、関数への引数や配列の要素として使用されます。
  • SPLIT_LEFT は 0 に初期化されます。

前提条件と不変条件

  • 特になし。

アルゴリズムまたは振る舞い

  • インデックスの分割アルゴリズム（range_gist_picksplit 等）において、各要素の所属先を決定するフラグとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/utils/adt/rangetypes_gist.c:169: 関数プロトタイプ引数として使用。
  • src/backend/utils/adt/rangetypes_gist.c:706: classes_groups 配列の型として使用。
  • src/backend/utils/adt/rangetypes_gist.c:1189: 関数引数として使用。
  • **呼び出し先**:
  • 情報なし。

関連データ構造

  • range_gist_picksplit: 本列挙型を利用してインデックスの分割を実行する主要関数。

失敗モードと境界条件

  • 特になし。

保守担当者向けメモ

  • SPLIT_LEFT が 0 に設定されているのは、初期化を容易にするための設計上の工夫です。
  • classes_groups 配列のインデックス操作と組み合わせて使用されるため、この列挙値の順序を変更する場合は関連するロジックの整合性に注意してください。
