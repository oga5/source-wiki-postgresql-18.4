convert_tuples_by_name

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tupconvert.c:101

目的

入力タプル記述子（indesc）と出力タプル記述子（outdesc）を名前ベースで比較し、物理構造が異なる場合にタプル変換用のマップを生成する。

シグネチャまたは形状

TupleConversionMap *convert_tuples_by_name(TupleDesc indesc, TupleDesc outdesc)

入力と出力

  • **入力**:
  • indesc: 変換元のタプル記述子。
  • outdesc: 変換先のタプル記述子。
  • **出力**:
  • 変換が必要な場合は TupleConversionMap 構造体へのポインタ。
  • 変換が不要（物理的に同一）な場合は NULL。

前提条件と不変条件

  • indesc と outdesc は有効な TupleDesc であること。
  • 内部で呼び出される build_attrmap_by_name_if_req により、属性の互換性が検証される。

アルゴリズムまたは振る舞い

1. build_attrmap_by_name_if_req(indesc, outdesc, false) を呼び出し、属性の対応関係を構築する。 2. 属性マップが不要（NULL）であれば、変換処理をスキップして NULL を返す。 3. 変換が必要な場合、convert_tuples_by_name_attrmap を呼び出し、属性マップを保持した TupleConversionMap を生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c
  • src/backend/executor/execExprInterp.c
  • src/backend/executor/execUtils.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • build_attrmap_by_name_if_req (属性マップの生成)
  • convert_tuples_by_name_attrmap (変換マップの構築)

関連データ構造

  • TupleDesc: タプルの記述子。
  • TupleConversionMap: 変換に必要な属性マッピング情報とワークスペースを保持する構造体。
  • AttrMap: 属性番号の対応関係を保持する構造体。

失敗モードと境界条件

  • 入力と出力の記述子間で属性名や型に互換性がない場合、build_attrmap_by_name_if_req がエラーを発生させる可能性がある。
  • 物理的に同一の構造であると判断された場合は NULL を返すため、呼び出し側で NULL チェックを行う必要がある。

保守担当者向けメモ

  • 本関数はファクトリとして機能し、実際の変換ロジックは execute_attr_map_tuple や execute_attr_map_slot に委譲される。
  • TupleConversionMap 内の TupleDesc は参照のみであり、本関数でメモリ解放は行わない。呼び出し側の生存期間管理に注意すること。
