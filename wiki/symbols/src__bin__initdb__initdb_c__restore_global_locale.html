restore_global_locale

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/initdb.c:388

目的

以前に保存されたロケール設定を復元し、保存時に割り当てられたメモリを解放する。

シグネチャまたは形状

static void restore_global_locale(int category, save_locale_t save)

入力と出力

  • category: 復元対象のロケールカテゴリ（例: LC_TIME）。
  • save: save_locale_t 型のロケール文字列（save_global_locale 等で取得したもの）。
  • 出力: なし（成功時）。失敗時は pg_fatal によりプロセスを終了する。

前提条件と不変条件

  • 引数 save は、過去に save_global_locale 等によって動的に確保された有効な文字列ポインタである必要がある。
  • 復元処理の完了後、save が指すメモリ領域は free() によって解放される。

アルゴリズムまたは振る舞い

1. プラットフォームに応じてロケールを復元する。

  • WIN32 の場合: _wsetlocale を使用。
  • その他: setlocale を使用。

    • 2. 復元に失敗した場合、pg_fatal を呼び出してエラーメッセージを出力し、プログラムを終了する。 3. 最後に free(save) を実行し、ロケール文字列を保持していたメモリを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/initdb/initdb.c:2167 (LC_TIME の復元)
  • src/bin/initdb/initdb.c:2228 (汎用的なカテゴリの復元)
  • 呼び出し先:
  • _wsetlocale (WIN32環境)
  • setlocale (非WIN32環境)
  • pg_fatal
  • free

関連データ構造

  • save_locale_t: ロケール情報を保持するための型。

失敗モードと境界条件

  • ロケールの復元に失敗した場合、pg_fatal によりプログラムが異常終了する。
  • save ポインタが NULL である場合や、既に解放済みのメモリを渡した場合の挙動は未定義（標準的な free の制約に依存）。

保守担当者向けメモ

  • 本関数は save_global_locale と対で使用されることを前提としている。
  • WIN32 とそれ以外のプラットフォームでロケール設定関数が異なるため、修正時は両方のパスを考慮すること。
