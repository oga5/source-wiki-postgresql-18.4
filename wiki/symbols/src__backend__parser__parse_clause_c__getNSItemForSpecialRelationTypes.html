getNSItemForSpecialRelationTypes

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:1010

目的

RangeVar で指定されたリレーションが、通常のテーブルではなく、CTE（共通テーブル式）または ENR（Ephemeral Named Relation）である場合に、それに対応する ParseNamespaceItem を取得・生成します。

シグネチャまたは形状

static ParseNamespaceItem *getNSItemForSpecialRelationTypes(ParseState *pstate, RangeVar *rv)

入力と出力

  • **入力**:
  • pstate: 現在の構文解析状態を保持する ParseState 構造体。
  • rv: 解決対象の RangeVar 構造体。
  • **出力**:
  • 該当する CTE または ENR が見つかった場合は、それに対応する ParseNamespaceItem へのポインタ。
  • いずれにも該当しない、または修飾名（スキーマ指定あり）が含まれる場合は NULL。

前提条件と不変条件

  • rv->schemaname が設定されている場合、CTE や ENR としては扱われず、即座に NULL を返します。

アルゴリズムまたは振る舞い

1. rv->schemaname が存在するか確認し、あれば NULL を返す。 2. scanNameSpaceForCTE を呼び出し、rv->relname に一致する CTE を検索する。

  • 見つかった場合、addRangeTableEntryForCTE を呼び出して ParseNamespaceItem を生成し、返す。

    • 3. CTE が見つからない場合、scanNameSpaceForENR を呼び出し、rv->relname に一致する ENR を検索する。

  • 見つかった場合、addRangeTableEntryForENR を呼び出して ParseNamespaceItem を生成し、返す。

    • 4. いずれにも該当しない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_clause.c 内の transformTableEntry（推定）。
  • **呼び出し先**:
  • scanNameSpaceForCTE
  • addRangeTableEntryForCTE
  • scanNameSpaceForENR
  • addRangeTableEntryForENR

関連データ構造

  • ParseState: 構文解析のコンテキスト。
  • RangeVar: テーブルやビューなどのリレーション参照。
  • ParseNamespaceItem: 名前空間内の項目を表現する構造体。
  • CommonTableExpr: CTE の定義情報。

失敗モードと境界条件

  • rv->schemaname が指定されている場合、CTE/ENR の検索対象外となり NULL を返します。
  • CTE および ENR のいずれにも一致しない名前が渡された場合、NULL を返します。

保守担当者向けメモ

  • 本関数は static であり、parse_clause.c 内でのみ使用されます。
  • CTE と ENR の優先順位は、CTE が先（scanNameSpaceForCTE）となっています。
