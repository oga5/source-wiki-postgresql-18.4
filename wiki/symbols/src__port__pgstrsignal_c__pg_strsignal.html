pg_strsignal

定義

  • 種別: function
  • 定義箇所: src/port/pgstrsignal.c:38

目的

Unix シグナル番号に対応する説明文字列を取得するための移植性レイヤーを提供します。プラットフォーム間の差異を吸収し、呼び出し元に対して常に有効な文字列ポインタを返すことを保証します。

シグネチャまたは形状

const char *pg_strsignal(int signum);

入力と出力

  • **入力**: int signum（対象のシグナル番号）
  • **出力**: const char *（シグナル番号に対応する説明文字列）

前提条件と不変条件

  • 戻り値の文字列は、後続の strsignal 呼び出しによって無効化される可能性があるため、呼び出し直後に使用するか、必要に応じてコピーする必要があります。
  • 常に有効な文字列ポインタを返すことが保証されています。

アルゴリズムまたは振る舞い

1. コンパイル時に HAVE_STRSIGNAL マクロが定義されている場合、標準ライブラリの strsignal(signum) を呼び出します。

  • strsignal が NULL を返した場合は、固定文字列 "unrecognized signal" を返します。

    • 2. HAVE_STRSIGNAL が未定義の場合、固定文字列 "(signal names not available on this platform)" を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: strsignal(3)（プラットフォーム依存）
  • **呼び出し元**: 以下の各所から参照されていることが確認されています。
  • src/backend/archive/shell_archive.c
  • src/backend/postmaster/postmaster.c
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/common/wait_error.c
  • src/test/regress/pg_regress.c

関連データ構造

  • 特になし。

失敗モードと境界条件

  • strsignal が NULL を返す環境や、HAVE_STRSIGNAL が定義されていない環境においても、エラーを発生させず、代替の固定文字列を返すことで安全に動作します。

保守担当者向けメモ

  • 過去に存在した sys_siglist[] を利用する実装は、現代のプラットフォームでは不要と判断され削除済みです。
  • コーディング規約として、本関数が返す文字列だけでなく、数値のシグナル番号も併せてログ出力することが推奨されています。
