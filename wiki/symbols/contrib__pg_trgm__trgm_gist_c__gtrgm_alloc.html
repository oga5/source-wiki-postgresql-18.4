gtrgm_alloc

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_gist.c:76

目的

GiST インデックスで使用される TRGM 構造体をメモリ上に割り当て、初期化する。

シグネチャまたは形状

static TRGM *gtrgm_alloc(bool isalltrue, int siglen, BITVECP sign)

入力と出力

  • **入力**:
  • isalltrue (bool): 全てのトリグラムが含まれる状態（ALLISTRUE）を示すフラグ。
  • siglen (int): シグネチャのバイト長。
  • sign (BITVECP): 初期化に使用するビットベクトル（NULL の場合はゼロクリアされる）。
  • **出力**:
  • 初期化済みの TRGM 型ポインタ。

前提条件と不変条件

  • palloc を使用するため、PostgreSQL のメモリコンテキスト内で呼び出される必要がある。
  • CALCGTSIZE マクロにより、フラグとシグネチャ長に基づいた適切なメモリサイズが算出される。

アルゴリズムまたは振る舞い

1. isalltrue フラグに基づき、flag（SIGNKEY および必要に応じて ALLISTRUE）を決定する。 2. CALCGTSIZE で必要なメモリサイズを計算し、palloc でメモリを確保する。 3. SET_VARSIZE で確保したメモリのサイズ情報を設定する。 4. isalltrue が false の場合:

  • sign が提供されていれば memcpy でシグネチャをコピーする。
  • sign が NULL であれば memset でシグネチャを 0 で埋める。

    • 5. 初期化済みの TRGM ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_gist.c 内の複数箇所（インデックス構築、分割、圧縮処理時）。
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理関数)
  • SET_VARSIZE (PostgreSQL マクロ)
  • GETSIGN (PostgreSQL マクロ)
  • memcpy, memset (標準 C ライブラリ)

関連データ構造

  • TRGM: GiST インデックスのキーを保持する構造体。
  • BITVECP: シグネチャ（ビットベクトル）を指すポインタ型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • siglen が不正な値の場合、CALCGTSIZE の計算結果や memcpy/memset の動作に影響を与える可能性がある。

保守担当者向けメモ

  • 本関数は pg_trgm の GiST インデックスにおけるメモリ確保の共通パスである。
  • isalltrue が true の場合、シグネチャのコピーはスキップされるため、呼び出し側で sign 引数に NULL を渡すことが一般的である。
