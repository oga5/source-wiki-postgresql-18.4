colordesc

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regguts.h:184

目的

正規表現エンジンにおいて、文字クラスを抽象化した「色（color）」の属性および状態を管理するために使用されます。

シグネチャまたは形状

struct colordesc {
    int         nschrs;     /* この色の単純文字数 */
    int         nuchrs;     /* この色の拡張マップエントリ数 */
    color       sub;        /* 開いているサブカラー、またはフリーチェーンポインタ */
    struct arc *arcs;       /* この色の全アークのチェーン */
    chr         firstchr;   /* この色に最初に割り当てられた単純文字 */
    int         flags;      /* フラグビットマスク */
};

入力と出力

本構造体は関数ではなくデータ構造であるため、入力と出力の概念は適用されません。ただし、colormap 構造体の一部としてメモリ確保や参照が行われます。

前提条件と不変条件

  • sub フィールドがサブカラーを持たない場合、値は NOSUB (COLORLESS) となります。
  • flags フィールドには以下のビットが定義されています：
  • FREECOL (01): 現在未使用。
  • PSEUDO (02): 擬似色（実在する文字を持たない）。
  • COLMARK (04): 一時的なマーカー。

アルゴリズムまたは振る舞い

正規表現のコンパイルおよび実行フェーズにおいて、文字コードを直接扱う代わりに、この構造体で定義された「色」を介して状態遷移や文字クラスの判定を行います。arcs ポインタを通じて、その色に関連付けられたすべての遷移アークを追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_color.c 内の関数群が、colormap の一部として本構造体を動的に確保（MALLOC/REALLOC）および参照しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出し先はありません。

関連データ構造

  • colormap: 本構造体の配列を保持する親構造体。
  • arc: arcs フィールドを通じて参照される遷移アークのリスト。
  • color: sub フィールドや色識別子として使用される型。

失敗モードと境界条件

  • メモリ確保時（MALLOC/REALLOC）に失敗する可能性があります。
  • nschrs や nuchrs のカウントが不整合を起こすと、文字クラスの判定ロジックに影響を及ぼします。

保守担当者向けメモ

  • sub フィールドは文脈によって「サブカラー」または「フリーリストのポインタ」という異なる役割を持つため、操作時には現在の状態（FREECOL フラグの有無など）を十分に確認してください。
  • regc_color.c でのメモリ管理（MALLOC）と密接に関連しているため、構造体のサイズ変更時には注意が必要です。
