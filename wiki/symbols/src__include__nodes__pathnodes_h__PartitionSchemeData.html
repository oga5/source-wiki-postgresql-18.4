PartitionSchemeData

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/pathnodes.h:612

目的

パーティション分割されたテーブルのスキーマ情報（戦略、属性、演算子族、型情報など）を保持し、プランナがパーティションに関連する最適化を行うためのメタデータを提供します。

シグネチャまたは形状

typedef struct PartitionSchemeData
{
    char        strategy;
    int16       partnatts;
    Oid        *partopfamily;
    Oid        *partopcintype;
    Oid        *partcollation;
    int16      *parttyplen;
    bool       *parttypbyval;
    struct FmgrInfo *partsupfunc;
} PartitionSchemeData;

入力と出力

  • 本構造体はメモリ上に割り当てられ、パーティションキーの属性数（partnatts）に基づいた配列（partopfamily, parttyplen 等）を保持します。
  • 関数呼び出しの入力として渡されることで、パーティションの比較や型変換に必要な情報を供給します。

前提条件と不変条件

  • partnatts はパーティションキーの属性数を示し、各配列メンバはこの数に応じたサイズを持つ必要があります。
  • partsupfunc はパーティション比較関数に関するキャッシュ情報を含みます。

アルゴリズムまたは振る舞い

  • パーティションキーのデータ型に関する情報（長さ、値渡し可否）をキャッシュし、実行時のオーバーヘッドを削減します。
  • パーティションの演算子族や照合順序（Collation）を保持することで、クエリプランナがパーティションプルーニングや結合の最適化を行う際の判断基準となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/util/plancat.c にて palloc0 を用いてインスタンス化されることが確認されています。
  • 呼び出し先: 構造体内部の FmgrInfo を通じて、パーティション比較関数が呼び出されることが示唆されています。

関連データ構造

  • PartitionScheme: PartitionSchemeData へのポインタ型として定義されています。
  • FmgrInfo: 関数マネージャ情報。パーティション比較関数の実行に使用されます。

失敗モードと境界条件

  • メモリ不足時に palloc0 が失敗する可能性があります。
  • partnatts が 0 の場合、各配列ポインタが指す領域の整合性に注意が必要です。

保守担当者向けメモ

  • 本構造体はプランナの最適化フェーズで使用されるため、変更を加える際は src/backend/optimizer/ 以下の関連ロジックへの影響を考慮してください。
  • parttyplen や parttypbyval などのキャッシュ情報は、パーティションキーの型情報が変更された場合に適切に更新される必要があります。
