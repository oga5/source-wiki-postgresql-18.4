pg_pwrite_zeros

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:708

目的

指定されたファイル記述子（fd）の指定オフセット位置から、指定されたサイズ（size）分だけゼロバイトを書き込む。

シグネチャまたは形状

ssize_t pg_pwrite_zeros(int fd, size_t size, off_t offset);

入力と出力

  • **入力**:
  • fd: 書き込み対象のファイル記述子。
  • size: 書き込むゼロバイトの総サイズ（バイト単位）。
  • offset: 書き込みを開始するファイル内のオフセット位置。
  • **出力**:
  • 成功時: 書き込まれた総バイト数（size と一致）。
  • 失敗時: pg_pwritev_with_retry が返す負のエラーコード。

前提条件と不変条件

  • total_written は最終的に size と等しくなることが Assert で保証されている。
  • 書き込みには静的に確保されたゼロ埋めされたブロック（zbuffer）が再利用される。

アルゴリズムまたは振る舞い

1. ゼロで初期化された静的バッファ zbuffer を用意する。 2. remaining_size が 0 になるまで以下のループを繰り返す。

  • PG_IOV_MAX を上限として、iovec 配列に zbuffer へのポインタと書き込みサイズを詰める。
  • 各ブロックのサイズは BLCKSZ を超えないように調整する。
  • pg_pwritev_with_retry を呼び出し、ベクトルI/Oを実行する。
  • 書き込み結果に応じて offset と total_written を更新する。

    • 3. すべての書き込みが完了した時点で、書き込み総数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c
  • src/backend/storage/file/fd.c
  • src/backend/storage/smgr/md.c
  • src/bin/pg_basebackup/walmethods.c
  • **呼び出し先**:
  • pg_pwritev_with_retry (直接のI/O実行)

関連データ構造

  • PGIOAlignedBlock: ゼロ埋め用バッファの型定義。
  • struct iovec: ベクトルI/Oに使用される構造体。

失敗モードと境界条件

  • pg_pwritev_with_retry が負の値を返した場合、そのエラーコードをそのまま呼び出し元へ伝播させる。
  • size が 0 の場合、ループは実行されず 0 が返される。

保守担当者向けメモ

  • 本関数は FRONTEND 定義時にコンパイルされるユーティリティの一部である。
  • BLCKSZ 単位での書き込みを行うため、ファイルシステム上のアライメントを意識した実装となっている。
