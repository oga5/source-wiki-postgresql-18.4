DecodeNumberField

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/dt_common.c:1088

目的

文字列として表現された日付または時刻の数値フィールドを解析し、struct tm 構造体および関連するマスク情報へ変換します。

シグネチャまたは形状

static int DecodeNumberField(int len, char *str, int fmask, int *tmask, struct tm *tm, fsec_t *fsec, bool *is2digits)

入力と出力

  • **入力**:
  • len: 文字列の長さ。
  • str: 解析対象の文字列。
  • fmask: 現在までに解析済みのフィールドを示すビットマスク。
  • **出力**:
  • tmask: 更新されたフィールドマスク。
  • tm: 解析結果が格納される struct tm。
  • fsec: 小数点以下の秒数（存在する場合）。
  • is2digits: 2桁年が使用された場合に true が設定されるフラグ。
  • **戻り値**: 成功時は DTK_DATE または DTK_TIME、失敗時は -1。

前提条件と不変条件

  • str はヌル終端文字列である必要があります。
  • 小数点が含まれる場合、小数点以下の数値は最大6桁まで処理されます（7桁目以降は切り捨てられます）。

アルゴリズムまたは振る舞い

1. **小数点処理**: str に . が含まれる場合、小数点以下の数値を fsec に変換し、str から小数点以降を除去して len を更新します。

  • 8桁: yyyymmdd
  • 6桁: yymmdd
  • 5桁: yyddd
  • 6桁: hhmmss
  • 4桁: hhmm（秒は0に設定）

    • 4. いずれのパターンにも適合しない場合、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ツールによる解析では src/backend/utils/adt/datetime.c 内の複数箇所で呼び出されていることが示唆されていますが、これらは PostgreSQL バックエンド側のコードであり、本ファイル（ECPGライブラリ）との直接的な呼び出し関係については、提供されたソース excerpt 内に明示的な記述はありません。
  • **呼び出し先**: strchr, strlen, strtoint, atoi などの標準ライブラリ関数を呼び出します。

関連データ構造

  • struct tm: 日付・時刻の各要素を保持する標準構造体。
  • fsec_t: 小数点以下の秒を保持する型。
  • DTK_DATE, DTK_TIME, DTK_DATE_M, DTK_TIME_M: 日付・時刻の型およびマスク定数（dt.h で定義）。

失敗モードと境界条件

  • 指定された長さ（len）が期待されるパターン（5, 6, 8桁など）と一致しない場合、解析は失敗し -1 を返します。
  • 小数点以下の処理において、7桁目以降は切り捨てられます（バックエンドの丸め処理とは異なる挙動をとる旨の注釈あり）。

保守担当者向けメモ

  • 本関数は dt_common.c 内の低レベルユーティリティであり、ECPG の日付解析ロジックの根幹を成します。
  • 修正を行う際は、dt.h で定義されているマスク定数との整合性に注意してください。
