tuplestore_set_eflags

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplestore.c:370

目的

Tuplestore に対して、実行フラグ（eflags）を設定します。これにより、後続の読み取り操作における巻き戻し（rewind）や逆方向スキャン（backward scan）の可否を制御します。

シグネチャまたは形状

void tuplestore_set_eflags(Tuplestorestate *state, int eflags)

入力と出力

  • state: 操作対象の Tuplestorestate 構造体へのポインタ。
  • eflags: 設定する実行フラグのビットマスク。
  • 戻り値: なし。

前提条件と不変条件

  • Tuplestore が初期状態（TSS_INMEM）であり、かつタプルが一つも格納されていない（memtupcount == 0）状態で呼び出す必要があります。これに違反すると elog(ERROR) が発生します。

アルゴリズムまたは振る舞い

1. 状態チェックを行い、条件を満たさない場合はエラーを投げます。 2. 読み取りポインタのインデックス 0 (readptrs[0]) に指定された eflags を適用します。 3. 既存のすべての読み取りポインタ（readptrs[1...n]）のフラグを統合し、state->eflags を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeCtescan.c
  • src/backend/executor/nodeMaterial.c
  • src/backend/executor/nodeWindowAgg.c
  • **呼び出し先**:
  • elog (エラー発生時)

関連データ構造

  • Tuplestorestate: Tuplestore の状態を保持する構造体。
  • TSReadPointer: 読み取り位置とフラグを管理する構造体。

失敗モードと境界条件

  • タプルが既に格納されている状態で呼び出した場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は、タプル格納が開始される前の初期化フェーズでのみ有効です。
  • eflags の設定は、Tuplestore がメモリからディスクへ退避する際の挙動や、読み取りポインタの制約に影響を与えます。
