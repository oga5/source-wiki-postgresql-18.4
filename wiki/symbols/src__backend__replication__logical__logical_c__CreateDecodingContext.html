CreateDecodingContext

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logical.c:499

目的

論理デコーディングを開始するための LogicalDecodingContext を生成・初期化します。

シグネチャまたは形状

LogicalDecodingContext *CreateDecodingContext(
    XLogRecPtr start_lsn,
    List *output_plugin_options,
    bool fast_forward,
    XLogReaderRoutine *xl_routine,
    LogicalOutputPluginWriterPrepareWrite prepare_write,
    LogicalOutputPluginWriterWrite do_write,
    LogicalOutputPluginWriterUpdateProgress update_progress
)

入力と出力

  • **入力**:
  • start_lsn: デコーディング開始位置（InvalidXLogRecPtr の場合はスロットの confirmed_flush を使用）。
  • output_plugin_options: プラグインに渡すオプションリスト。
  • fast_forward: 変更を生成せずWALをスキップするモードの有効化フラグ。
  • xl_routine: WAL読み取り用のルーチン。
  • prepare_write, do_write, update_progress: 出力プラグイン用のコールバック関数。
  • **出力**: 初期化済みの LogicalDecodingContext 構造体へのポインタ。

前提条件と不変条件

  • MyReplicationSlot が取得済みであること。
  • 物理レプリケーションスロットではないこと。
  • スロットが現在接続中のデータベースと一致していること（fast_forward 時を除く）。
  • リカバリ中の同期スロットではないこと。
  • スロットが有効（RS_INVAL_NONE）であり、restart_lsn が設定されていること。

アルゴリズムまたは振る舞い

1. **スロット検証**: 現在のスロットが論理デコーディングに適しているか確認し、エラーがあれば報告します。 2. **LSN調整**: start_lsn が confirmed_flush より古い場合、confirmed_flush まで進めて警告を出力します。 3. **コンテキスト生成**: StartupDecodingContext を呼び出し、基本構造を構築します。 4. **プラグイン初期化**: startup_cb_wrapper を介して出力プラグインの初期化コールバックを実行します。 5. **2フェーズコミット設定**: スロットの two_phase 状態を更新し、必要に応じて永続化します。 6. **ログ出力**: デコーディング開始の旨をログに記録し、コンテキストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/logical.c (内部の複数箇所)
  • src/backend/replication/logical/logicalfuncs.c
  • src/backend/replication/walsender.c
  • **呼び出し先**:
  • StartupDecodingContext
  • startup_cb_wrapper
  • ReplicationSlotSave
  • SnapBuildSetTwoPhaseAt

関連データ構造

  • LogicalDecodingContext: デコーディングの状態を保持する主要構造体。
  • ReplicationSlot: 現在のレプリケーションスロット情報。

失敗モードと境界条件

  • スロットが未取得の場合、エラー（ERROR）が発生します。
  • 物理スロットやデータベース不一致、同期中のスロットを使用しようとした場合、エラーが発生します。
  • start_lsn が confirmed_flush より小さい場合、ログに警告を出力し、開始位置を confirmed_flush に強制変更します。

保守担当者向けメモ

  • 本関数は論理デコーディングのライフサイクルにおける初期化の要です。
  • 2フェーズコミットの有効化ロジックは、スロットの永続化を伴うため、変更時には ReplicationSlotSave の呼び出し順序に注意してください。
