tliInHistory

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/timeline.c:525

目的

指定されたタイムラインID（tli）が、提供されたタイムライン履歴エントリのリスト（expectedTLEs）内に存在するかどうかを判定します。

シグネチャまたは形状

bool tliInHistory(TimeLineID tli, List *expectedTLEs)

入力と出力

  • **入力**:
  • tli: 検索対象の TimeLineID。
  • expectedTLEs: TimeLineHistoryEntry 構造体へのポインタを保持する List。
  • **出力**:
  • bool: リスト内に tli と一致するエントリが見つかった場合は true、それ以外の場合は false を返します。

前提条件と不変条件

  • expectedTLEs は、TimeLineHistoryEntry 構造体へのポインタを要素として持つ有効なリストである必要があります。
  • リスト内の各要素は TimeLineHistoryEntry 型にキャスト可能であると想定されています。

アルゴリズムまたは振る舞い

1. expectedTLEs リストを先頭から順に走査します。 2. 各要素（TimeLineHistoryEntry）の tli フィールドを、引数で渡された tli と比較します。 3. 一致するエントリが見つかった時点で直ちに true を返します。 4. リストの末尾まで走査しても一致するものがない場合、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c: リカバリプロセスにおいて、新しいタイムラインの妥当性を検証する際に呼び出されます。
  • **呼び出し先**:
  • 本関数はリストの走査のみを行い、他の関数を呼び出しません。

関連データ構造

  • TimeLineID: タイムラインを識別する整数型。
  • TimeLineHistoryEntry: タイムラインの履歴情報を保持する構造体（src/include/access/timeline.h で定義）。
  • List: PostgreSQLの汎用リスト構造。

失敗モードと境界条件

  • expectedTLEs が空リスト（NIL）の場合、常に false を返します。
  • tli が無効な値であっても、リスト内に存在しなければ正しく false を返します。

保守担当者向けメモ

  • 本関数はリストの線形探索を行うため、expectedTLEs の要素数が増加した場合、呼び出し頻度によっては性能に影響を与える可能性があります。
  • src/include/access/timeline.h にて外部公開されています。
