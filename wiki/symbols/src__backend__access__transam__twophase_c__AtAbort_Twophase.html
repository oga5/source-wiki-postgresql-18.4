AtAbort_Twophase

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/twophase.c:303

目的

トランザクションのアボート時に、現在ロックされている2フェーズコミット（2PC）関連のグローバル・トランザクション・エントリ（GXACT）の状態を整理し、共有メモリ上の整合性を維持する。

シグネチャまたは形状

void AtAbort_Twophase(void)

入力と出力

  • 入力: なし（グローバル変数 MyLockedGxact を参照）
  • 出力: なし（副作用として共有メモリ上の TwoPhaseState を更新）

前提条件と不変条件

  • TwoPhaseStateLock を排他モードで取得して操作を行う。
  • MyLockedGxact が NULL の場合は何もしない。
  • 準備中または終了処理中のトランザクションが、WAL書き込みの前後でどのような状態にあるかに応じて、エントリを削除するかロックを解除する。

アルゴリズムまたは振る舞い

1. MyLockedGxact が NULL であれば即座に終了する。 2. TwoPhaseStateLock を排他モードで取得する。 3. MyLockedGxact->valid が偽（準備完了前または終了処理の特定段階）であれば、RemoveGXact を呼び出して共有メモリからエントリを削除する。 4. MyLockedGxact->valid が真であれば、locking_backend を INVALID_PROC_NUMBER に設定し、ロックを解放する。 5. TwoPhaseStateLock を解放する。 6. MyLockedGxact を NULL にリセットする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/twophase.c (内部のクリーンアップ処理)
  • src/backend/access/transam/xact.c:2924 (トランザクションアボート処理の一部)
  • 呼び出し先:
  • RemoveGXact (条件による)
  • LWLockAcquire / LWLockRelease

関連データ構造

  • GlobalTransactionData (エントリの本体)
  • TwoPhaseState (共有メモリ上の管理構造体)

失敗モードと境界条件

  • MyLockedGxact が指すエントリが既に無効化されている場合、RemoveGXact による削除が行われる。
  • WAL書き込み後のアボートにおいて、メモリ上の状態とディスク上の状態が不整合になる可能性があるが、本関数はメモリ上のエントリ管理に専念する。

保守担当者向けメモ

  • 本関数はトランザクションのアボート時に呼び出されるため、デッドロックを避けるため TwoPhaseStateLock の取得順序に注意すること。
  • MyLockedGxact は現在のバックエンドが保持している準備済みトランザクションへのポインタであり、アボート時に必ずクリアする必要がある。
