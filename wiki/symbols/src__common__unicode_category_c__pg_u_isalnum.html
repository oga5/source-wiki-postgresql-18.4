pg_u_isalnum

定義

  • 種別: function
  • 定義箇所: src/common/unicode_category.c:225

目的

指定された Unicode コードポイントが英数字（アルファベットまたは数字）であるかどうかを判定します。

シグネチャまたは形状

bool pg_u_isalnum(pg_wchar code, bool posix)

入力と出力

  • **入力**:
  • code: 判定対象の Unicode コードポイント（pg_wchar 型）。
  • posix: true の場合、POSIX 互換の判定ルールを適用します。
  • **出力**:
  • 指定されたコードポイントが英数字であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力されるコードポイントは UTF-8 エンコーディングを前提としています。
  • posix フラグは、標準的な Unicode 動作と POSIX 互換動作の差異を吸収するために使用されます。

アルゴリズムまたは振る舞い

1. pg_u_isalpha(code) を呼び出し、文字がアルファベットであるか判定します。 2. pg_u_isdigit(code, posix) を呼び出し、文字が数字であるか判定します。 3. 上記いずれかが true であれば true を返し、両方とも false であれば false を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • pg_u_isalpha
  • pg_u_isdigit
  • **呼び出し元**:
  • src/backend/regex/regc_pg_locale.c
  • src/backend/utils/adt/pg_locale_builtin.c
  • src/common/unicode/case_test.c
  • src/common/unicode/category_test.c

関連データ構造

  • pg_wchar: Unicode コードポイントを保持する型。

失敗モードと境界条件

  • 0x80 未満の ASCII 文字については、内部の unicode_opt_ascii テーブルを参照し定数時間で判定されます。
  • それ以外のコードポイントについては、unicode_category_table.h に定義されたテーブルに対する二分探索が行われます。

保守担当者向けメモ

  • 本関数は Unicode 規格の安定性ポリシーに基づき、カテゴリ値が固定されていることを前提としています。
  • 判定ロジックの変更が必要な場合は、src/common/unicode_category_table.h の生成スクリプトおよび関連する pg_u_is* 系関数の実装を確認してください。
