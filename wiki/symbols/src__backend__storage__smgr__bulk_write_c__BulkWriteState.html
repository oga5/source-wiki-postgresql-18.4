BulkWriteState

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/smgr/bulk_write.c:61

目的

リレーションの新規作成や初期ロード時において、バッファマネージャを介さず直接ストレージへ書き込むための状態管理を行う構造体です。WAL ログのバッチ処理や書き込みキューの管理を担います。

シグネチャまたは形状

struct BulkWriteState
{
    SMgrRelation smgr;
    ForkNumber   forknum;
    bool         use_wal;
    int          npending;
    PendingWrite pending_writes[MAX_PENDING_WRITES];
    BlockNumber  relsize;
    XLogRecPtr   start_RedoRecPtr;
    MemoryContext memcxt;
};

入力と出力

  • **入力**: smgr_bulk_start_rel または smgr_bulk_start_smgr によって初期化され、対象リレーション情報や WAL 使用フラグが設定されます。
  • **出力**: smgr_bulk_finish を通じて、保留中の書き込みが物理ストレージへフラッシュされ、必要に応じて fsync が実行されます。

前提条件と不変条件

  • 対象リレーションに対して、他のバックエンドがアクセスしない（排他制御が保証されている）環境で使用される必要があります。
  • smgr_bulk_write に渡されたバッファの所有権は本構造体へ移譲されるため、呼び出し側で pfree してはなりません。

アルゴリズムまたは振る舞い

1. **キューイング**: smgr_bulk_write を介してページを pending_writes に追加します。 2. **バッチ処理**: npending が MAX_PENDING_WRITES に達すると、smgr_bulk_flush が自動的に呼び出され、WAL ログの生成と物理書き込みが行われます。 3. **整合性維持**: 操作開始時の start_RedoRecPtr を保持し、終了時にチェックポイントとの競合を考慮した同期処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: gistbuild.c, rewriteheap.c, nbtree.c, nbtsort.c, spginsert.c, catalog/storage.c など、インデックス構築やテーブル再構築を行うモジュールから利用されます。
  • **呼び出し先**: smgr_bulk_flush（内部関数）を呼び出します。

関連データ構造

  • PendingWrite: pending_writes 配列の各要素として、書き込み待ちのページ情報を保持します。
  • SMgrRelation: 書き込み対象のリレーションハンドル。

失敗モードと境界条件

  • MAX_PENDING_WRITES を超える書き込み要求は、自動フラッシュによって処理されます。
  • チェックポイント発生時に smgr_bulk_finish が実行される場合、MyProc->delayChkptFlags を用いてチェックポイントを一時的に抑制し、書き込み完了後の fsync 漏れを防ぐ必要があります。

保守担当者向けメモ

  • 本構造体は共有バッファプールをバイパスするため、キャッシュ整合性の管理は呼び出し側の責務となります。
  • memcxt は、バルク書き込み操作に関連するメモリ割り当てのライフサイクル管理に使用されます。
