pg_atomic_test_set_flag

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics.h:182

目的

テスト・アンド・セット（TAS）操作を実行し、フラグをアトミックに設定して、以前の状態を取得するための簡易的なロック取得機構を提供します。

シグネチャまたは形状

static inline bool pg_atomic_test_set_flag(volatile pg_atomic_flag *ptr)

入力と出力

  • **入力**: ptr - 操作対象となる pg_atomic_flag 型へのポインタ。volatile 修飾子が指定されています。
  • **出力**: bool 型。フラグが以前に設定されていなかった（ロックを取得できた）場合は true を、既に設定されていた場合は false を返します。

前提条件と不変条件

  • ptr は適切にアライメントされている必要があります。
  • 本関数はアトミックな操作を保証するため、対象のメモリ領域は他のスレッドからも正しく同期される必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った ptr を、プラットフォーム固有の実装関数である pg_atomic_test_set_flag_impl(ptr) に渡します。 2. pg_atomic_test_set_flag_impl の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/autovacuum.c (ワーカー情報のバランス調整時)
  • src/test/regress/regress.c (回帰テストにおけるアトミック操作の検証)
  • **呼び出し先**:
  • pg_atomic_test_set_flag_impl (プラットフォームおよびコンパイラ固有の実装へ委譲)

関連データ構造

  • pg_atomic_flag: アトミック操作の対象となるフラグ型。

失敗モードと境界条件

  • 本関数自体はエラーを返しませんが、メモリのアライメントが不適切な場合、ハードウェアレベルで未定義の動作やパフォーマンス低下を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、実際のロジックは _impl 接尾辞を持つ関数に依存しています。プラットフォーム固有の実装（arch-*.h 等）を調査する際は、その実装先を確認してください。
