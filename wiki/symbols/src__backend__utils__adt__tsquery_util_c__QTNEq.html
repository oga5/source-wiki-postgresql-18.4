QTNEq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_util.c:182

目的

2つの QTNode ツリー構造が論理的に等価であるかを判定します。

シグネチャまたは形状

bool QTNEq(QTNode *a, QTNode *b)

入力と出力

  • **入力**: 比較対象となる2つの QTNode 構造体へのポインタ a および b。
  • **出力**: 両者が等価であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力される QTNode ツリーは、比較前に QTNSort によって正規化（ソート）されている必要があります。ソートされていない場合、構造が同一でも等価と判定されない可能性があります。

アルゴリズムまたは振る舞い

1. 各ノードの sign（シグネチャ）ビットマスクを比較します。 2. a->sign & b->sign の結果が双方の sign と一致しない場合、即座に false を返します（高速な不一致判定）。 3. ビットマスクが一致する場合、QTNodeCompare を呼び出し、ツリー構造の再帰的な比較結果が 0（一致）であるかを確認して返します。

  • **呼び出し先**:

関連データ構造

  • QTNode: tsquery を表現するメモリ上のツリー構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動については、提供されたソースからは不明です。
  • QTNodeCompare が再帰的に呼び出されるため、非常に深いツリー構造を比較する場合、スタックオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本関数は QTNSort による正規化を前提としています。比較ロジックを変更する際は、QTNodeCompare の実装と整合性が取れているか確認してください。
  • sign による高速判定は、ツリーの比較コストを削減するための最適化です。QTNode の構造変更時には sign の計算ロジックが維持されているか注意してください。
