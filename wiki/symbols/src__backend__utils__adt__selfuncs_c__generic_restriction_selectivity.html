generic_restriction_selectivity

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/selfuncs.c:921

目的

演算子と定数を用いた制限句（WHERE 句）に対し、統計情報（MCV およびヒストグラム）を使用して選択率（0.0〜1.0）を推定する。

シグネチャまたは形状

double generic_restriction_selectivity(PlannerInfo *root, Oid oproid, Oid collation,
                                       List *args, int varRelid,
                                       double default_selectivity)

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報。
  • oproid: 評価対象の演算子の OID。
  • collation: 照合順序。
  • args: 演算子の引数リスト。
  • varRelid: 変数が属するリレーションの ID。
  • default_selectivity: 統計情報が利用できない場合に使用するデフォルト値。
  • **出力**:
  • 推定された選択率（double 型）。

前提条件と不変条件

  • args が「変数 OP 定数」または「定数 OP 変数」の形式である必要がある。
  • 計算結果は CLAMP_PROBABILITY マクロにより 0.0 から 1.0 の範囲に収められる。
  • 最終的な推定値が 0.0001 未満の場合は 0.0001 に、0.9999 を超える場合は 0.9999 に調整される。

アルゴリズムまたは振る舞い

1. **変数抽出**: get_restriction_variable を呼び出し、式から変数と定数を分離する。失敗した場合は default_selectivity を返す。 2. **NULL 判定**: 定数が NULL の場合、演算子が厳密（strict）であると仮定し 0.0 を返す。 3. **統計計算**:

  • mcv_selectivity を使用して最頻値（MCV）の選択率を算出。
  • histogram_selectivity を使用してヒストグラムに基づく選択率を算出。
  • ヒストグラムサイズに応じて、デフォルト値とヒストグラムの値を重み付け合成する。

    • 4. **統合**: NULL 割合（stanullfrac）を考慮し、MCV とヒストグラムの推定値を統合する。 5. **クリーンアップ**: ReleaseVariableStats を呼び出し、統計情報を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/ltree_op.c:655
  • src/backend/utils/adt/selfuncs.c:3302
  • **呼び出し先**:
  • get_restriction_variable
  • fmgr_info / get_opcode
  • mcv_selectivity
  • histogram_selectivity
  • ReleaseVariableStats

関連データ構造

  • VariableStatData: 統計情報取得時に使用されるメタデータ構造体。
  • PlannerInfo: プランナのグローバル情報。
  • pg_statistic: 統計情報の元となるシステムカタログ。

失敗モードと境界条件

  • 式が定数比較でない場合、default_selectivity を返す。
  • 統計情報が欠落している場合、nullfrac は 0.0 として扱われる。
  • ヒストグラムサイズが小さい場合、デフォルト値への依存度が高まる。

保守担当者向けメモ

  • 本関数は汎用的な選択率推定器であり、特定のデータ型に依存しない演算子に対して使用される。
  • histogram_selectivity の戻り値が負の場合は統計情報が不十分と判断し、デフォルト値へフォールバックする。
