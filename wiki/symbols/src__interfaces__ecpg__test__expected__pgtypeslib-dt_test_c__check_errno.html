check_errno

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c:483

目的

グローバル変数 errno の値を解析し、pgtypes 固有のエラーコードまたは標準ライブラリのエラー内容を人間が読める形式で標準出力（stdout）へ出力する補助関数です。

シグネチャまたは形状

static void check_errno(void)

入力と出力

  • **入力**: グローバル変数 errno を参照。
  • **出力**: なし（printf による標準出力への書き込みのみ）。

前提条件と不変条件

  • errno.h がインクルードされており、errno が有効な値を持っていること。
  • pgtypes_error.h で定義されたエラーコード（例: PGTYPES_TS_BAD_TIMESTAMP）が環境で利用可能であること。

アルゴリズムまたは振る舞い

1. errno の値を switch 文で評価する。 2. errno == 0 の場合、(no errno set) - と出力する。 3. errno == PGTYPES_TS_BAD_TIMESTAMP の場合、(errno == PGTYPES_TS_BAD_TIMESTAMP) - と出力する。 4. 上記以外の場合、以下の順で出力する。

  • (unknown errno (%d))\n （数値を出力）
  • (libc: (%s)) （strerror(errno) を用いてシステムエラー文字列を出力）

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/compat_informix-dec_test.c 内の複数の箇所から呼び出されていることが確認されています。
  • **呼び出し先**: 標準ライブラリ関数 printf および strerror を呼び出します。

関連データ構造

  • errno（標準Cライブラリ）
  • PGTYPES_TS_BAD_TIMESTAMP（pgtypes_error.h で定義されたエラー定数）

失敗モードと境界条件

  • errno が定義済みのケース（0 または PGTYPES_TS_BAD_TIMESTAMP）以外の場合、default 節により未知のエラーとして処理され、strerror を通じてシステム側のエラーメッセージが表示されます。

保守担当者向けメモ

  • 本関数はテストプログラム用の補助関数であり、実運用コードではなく回帰テストの出力検証を目的としています。
  • 新たな pgtypes 固有のエラーコードを追加した際は、本関数の switch 文にもケースを追加することを検討してください。
