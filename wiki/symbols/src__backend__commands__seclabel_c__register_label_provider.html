register_label_provider

定義

  • 種別: function
  • 定義箇所: src/backend/commands/seclabel.c:569

目的

外部のセキュリティプロバイダー（拡張機能）が、データベースオブジェクトのセキュリティラベルを検証するためのフック関数を登録する。

シグネチャまたは形状

void register_label_provider(const char *provider_name, check_object_relabel_type hook)

入力と出力

  • provider_name: 登録するプロバイダーの名前（文字列）。
  • hook: ラベルの妥当性を検証する関数ポインタ（check_object_relabel_type）。
  • 出力: なし（void）。

前提条件と不変条件

  • TopMemoryContext 上でメモリ確保が行われるため、プロバイダー情報は永続的に保持される。
  • 登録されたプロバイダーは label_provider_list に追加される。

アルゴリズムまたは振る舞い

1. MemoryContextSwitchTo(TopMemoryContext) を呼び出し、メモリコンテキストをトップレベルへ切り替える。 2. LabelProvider 構造体用のメモリを確保する。 3. provider_name を複製し、hook を構造体に格納する。 4. label_provider_list に新しいプロバイダーを追加する。 5. 元のメモリコンテキストへ復帰する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/sepgsql/hooks.c
  • src/test/modules/dummy_seclabel/dummy_seclabel.c
  • 呼び出し先:
  • palloc
  • pstrdup
  • lappend
  • MemoryContextSwitchTo

関連データ構造

  • LabelProvider: プロバイダー名とフック関数を保持する構造体。
  • label_provider_list: 登録済みプロバイダーを管理する静的リスト。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • 呼び出し元で提供される provider_name が NULL の場合の挙動は、本関数内では明示的にガードされていない。

保守担当者向けメモ

  • 本関数は拡張機能の初期化フェーズで呼び出されることを想定している。
  • 登録されたフックは、SECURITY LABEL コマンド実行時にラベルの妥当性チェックを行うために使用される。
