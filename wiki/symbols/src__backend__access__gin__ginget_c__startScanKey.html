startScanKey

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginget.c:506

目的

GINインデックススキャンにおいて、検索キーに含まれるエントリを「必須（Required）」と「追加（Additional）」の2つの集合に分類し、走査効率を最適化する。

シグネチャまたは形状

static void startScanKey(GinState *ginstate, GinScanOpaque so, GinScanKey key)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの状態管理構造体。
  • so: GINスキャンの不透明な状態記述子。
  • key: 処理対象の検索キー（GinScanKey）。
  • **出力**: なし（key 構造体内の requiredEntries、additionalEntries、および関連するカウント変数が更新される）。

前提条件と不変条件

  • key は初期化済みであること。
  • so->keyCtx および so->tempCtx がメモリ管理のために使用可能であること。
  • key->excludeOnly が真の場合、必須エントリは存在しないものとして扱われる。

アルゴリズムまたは振る舞い

1. **初期化**: key->curItem を最小値にリセットし、フラグ類を初期化する。 2. **分類ロジック**:

  • excludeOnly の場合: 全てのエントリを additionalEntries に割り当てる。
  • nentries > 1 の場合:
  • エントリを出現頻度（predictNumberResult）に基づいてソートする。
  • 頻度の低い順に必須セットへ割り当て、triConsistentFn が GIN_FALSE を返すまで繰り返すことで、最小限の必須エントリ集合を特定する。
  • 残りのエントリを additionalEntries に割り当てる。
  • nentries == 1 の場合: 単一のエントリを必須セットに割り当てる。

    • 3. **メモリ管理**: so->keyCtx に結果を割り当て、一時的なメモリ領域（so->tempCtx）をリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginget.c 内の startScan 関数から呼び出される。
  • **呼び出し先**:
  • qsort_arg（エントリのソート）
  • key->triConsistentFn（一致判定ロジック）
  • palloc / MemoryContextReset（メモリ管理）
  • CHECK_FOR_INTERRUPTS（割り込みチェック）

関連データ構造

  • GinScanKey: 検索キーの状態を保持する構造体。
  • GinScanOpaque: スキャン全体の不透明な状態を保持する構造体。
  • GinScanEntry: 個々のインデックスエントリへのポインタ。

失敗モードと境界条件

  • nentries が 0 または 1 の場合の分岐処理が含まれる。
  • triConsistentFn の実行中に割り込みが発生した場合、CHECK_FOR_INTERRUPTS() により中断される可能性がある。
  • メモリ不足時には palloc がエラーをスローする。

保守担当者向けメモ

  • 本関数はクエリ性能に直結する最適化を行うため、triConsistentFn の挙動変更時には本関数の分類ロジックへの影響を考慮すること。
  • entryIndexes のソート順序は entryIndexByFrequencyCmp に依存している。
