get_partition_natts

定義

  • src/include/utils/partcache.h:64

目的

パーティションキー構造体から、定義されているパーティションキー列の総数を取得します。

シグネチャまたは形状

static inline int get_partition_natts(PartitionKey key)

入力と出力

  • **入力**: PartitionKey key （パーティションキーのメタデータ構造体へのポインタ）
  • **出力**: int （パーティションキーに含まれる列の数）

前提条件と不変条件

  • 引数 key は有効な PartitionKey 構造体を指している必要があります。
  • パーティションキーはリレーションが開かれている間は不変であるため、本関数は常に一貫した値を返します。

アルゴリズムまたは振る舞い

key が指す PartitionKeyData 構造体の partnatts フィールドを直接参照して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/partition.c:267
  • src/backend/executor/execPartition.c:1626
  • src/backend/parser/parse_utilcmd.c:4293
  • **呼び出し先**: なし（インライン関数として定義）

関連データ構造

  • PartitionKey: PartitionKeyData 構造体へのポインタ。パーティションキーの全情報を保持します。

失敗モードと境界条件

  • key が NULL の場合、セグメンテーション違反が発生します。呼び出し側で key が有効であることを保証する必要があります。

保守担当者向けメモ

  • 本関数は PartitionKeyData 構造体への直接アクセスをカプセル化するインライン関数です。構造体の内部レイアウトが変更された場合、この関数を通じて抽象化が維持されます。
