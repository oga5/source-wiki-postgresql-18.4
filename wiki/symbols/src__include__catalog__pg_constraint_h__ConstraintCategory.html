ConstraintCategory

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/pg_constraint.h:213

目的

制約が適用される対象（リレーションまたはドメイン）を識別し、区別するために使用されます。

シグネチャまたは形状

typedef enum ConstraintCategory
{
	CONSTRAINT_RELATION,
	CONSTRAINT_DOMAIN,
	CONSTRAINT_ASSERTION,
} ConstraintCategory;

入力と出力

  • 入力: 制約のカテゴリを判定する関数（例: ConstraintNameIsUsed）の引数として利用されます。
  • 出力: 列挙型定数（CONSTRAINT_RELATION, CONSTRAINT_DOMAIN, CONSTRAINT_ASSERTION）を返します。

前提条件と不変条件

  • CONSTRAINT_ASSERTION は将来の拡張用として定義されており、現時点での具体的な振る舞いは定義されていません。

アルゴリズムまたは振る舞い

  • 制約の管理において、対象オブジェクトがテーブル等のリレーションであるか、ドメインであるかを分類します。
  • ConstraintNameIsUsed 関数において、制約名が特定のカテゴリ内で既に使用されているかを検証する際の識別子として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_constraint.c:412 の ConstraintNameIsUsed 関数にて引数として参照されています。
  • 呼び出し先: 本シンボル自体は列挙型定義であり、関数を呼び出すことはありません。

関連データ構造

  • pg_constraint: 本列挙型が分類対象とする制約情報を保持するシステムカタログ。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • CONSTRAINT_ASSERTION は将来の拡張用であるため、現在の実装では使用されていません。
  • 制約のカテゴリ追加や変更を行う際は、pg_constraint カタログの整合性に影響を与えるため、関連するカタログ操作関数（CreateConstraintEntry 等）のロジックを確認してください。
