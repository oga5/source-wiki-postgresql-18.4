generate_series_numeric_fctx

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/numeric.c:328

目的

generate_series 関数において、NUMERIC 型の値を連続生成する際のイテレータ状態を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
    NumericVar  current;
    NumericVar  stop;
    NumericVar  step;
} generate_series_numeric_fctx;

入力と出力

本構造体は関数呼び出しのコンテキストとして使用されます。

  • 入力: current（現在の値）、stop（終了値）、step（増分値）を保持します。
  • 出力: 呼び出し元でこれらの値を更新・参照することで、連続する数値列を生成します。

前提条件と不変条件

  • 各フィールドは NumericVar 型であり、算術演算に使用されるメモリ上の作業用構造体です。
  • palloc を使用してメモリが割り当てられるため、使用後は適切なメモリ管理が必要です。

アルゴリズムまたは振る舞い

  • generate_series の実行中、current に step を加算し、stop に達するまで値を生成し続けるための状態管理を行います。
  • 内部的には NumericVar を使用しているため、高精度な数値計算が維持されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/numeric.c 内の generate_series 関連関数（1711行目、1779行目付近）で、コンテキストの初期化および割り当てに使用されています。
  • 呼び出し先: 具体的な関数呼び出しの詳細はソース抜粋からは不明です。

関連データ構造

  • NumericVar: 本構造体の各フィールドの型であり、NUMERIC 型の算術演算用作業構造体です。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生する可能性があります。
  • 算術的境界: step が 0 の場合や、current と stop の関係性により無限ループが発生する可能性があるため、呼び出し側でのバリデーションが必要です。

保守担当者向けメモ

  • NumericVar を保持しているため、メモリリークを防ぐために free_var() を適切に呼び出す必要があります。
  • 構造体の各フィールドは NumericVar であるため、numeric.c 内の算術演算関数（add_var 等）との親和性が高い設計になっています。
