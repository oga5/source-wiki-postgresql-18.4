tdr_attach_shmem

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_dsm_registry/test_dsm_registry.c:38

目的

共有メモリセグメントへのアタッチおよび、共有メモリ上のデータ構造に対する軽量ロック（LWLock）のトランシェ登録を行う。

シグネチャまたは形状

static void tdr_attach_shmem(void)

入力と出力

  • 入力: なし
  • 出力: なし（内部静的変数 tdr_state を更新）

前提条件と不変条件

  • GetNamedDSMSegment が正常に共有メモリを確保または取得できること。
  • tdr_state が指し示すメモリ領域に TestDSMRegistryStruct が配置されていること。

アルゴリズムまたは振る舞い

1. GetNamedDSMSegment を呼び出し、"test_dsm_registry" という名前の共有メモリセグメントを取得する。セグメントが未初期化の場合は tdr_init_shmem を実行して初期化する。 2. 取得した共有メモリ領域のポインタを静的変数 tdr_state に格納する。 3. tdr_state->lck.tranche を使用して、LWLockRegisterTranche によりロックのトランシェを登録する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • set_val_in_shmem (src/test/modules/test_dsm_registry/test_dsm_registry.c:54)
  • get_val_in_shmem (src/test/modules/test_dsm_registry/test_dsm_registry.c:69)
  • 呼び出し先:
  • GetNamedDSMSegment
  • tdr_init_shmem
  • LWLockRegisterTranche

関連データ構造

  • TestDSMRegistryStruct: 共有メモリ上に配置される構造体。

失敗モードと境界条件

  • GetNamedDSMSegment が失敗した場合、PostgreSQLの標準的なエラー処理によりプロセスが中断される可能性がある。

保守担当者向けメモ

  • 本関数はSQL関数から呼び出されるたびに実行される。GetNamedDSMSegment は内部でキャッシュや既存セグメントの検索を行うため、効率的に動作する。
  • ロックのトランシェ登録はデバッグや統計情報取得において重要であるため、本関数で確実に行う必要がある。
