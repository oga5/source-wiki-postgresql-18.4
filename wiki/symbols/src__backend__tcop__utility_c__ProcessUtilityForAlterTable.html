ProcessUtilityForAlterTable

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/utility.c:1958

目的

ALTER TABLE 実行中に発生するサブコマンドを処理し、イベントトリガーの整合性を維持しながらユーティリティコマンドを実行する。

シグネチャまたは形状

void ProcessUtilityForAlterTable(Node *stmt, AlterTableUtilityContext *context)

入力と出力

  • **入力**:
  • stmt: 実行対象のユーティリティ文（Node*）。
  • context: AlterTableUtilityContext 構造体。現在のクエリ文字列、パラメータ、親の PlannedStmt、対象リレーションの OID を保持。
  • **出力**: なし（副作用としてコマンドを実行）。

前提条件と不変条件

  • 呼び出し元は ALTER TABLE の実行コンテキスト内であること。
  • context が有効な AlterTableUtilityContext を指していること。

アルゴリズムまたは振る舞い

1. **イベントトリガーの終了**: EventTriggerAlterTableEnd() を呼び出し、現在の複雑なコマンドセットを終了させる。 2. **ラッパーの作成**: makeNode(PlannedStmt) を使用して、サブコマンド実行用の PlannedStmt を作成する。commandType を CMD_UTILITY に設定し、canSetTag を false にする。 3. **コマンド実行**: ProcessUtility を呼び出し、サブコマンドを実行する。この際、PROCESS_UTILITY_SUBCOMMAND フラグを渡す。 4. **イベントトリガーの再開**: EventTriggerAlterTableStart() および EventTriggerAlterTableRelid() を呼び出し、イベントトリガーの状態を復元する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/tablecmds.c 内の ALTER TABLE 処理フロー（tablecmds.c:5743, 6113）。
  • **呼び出し先**: EventTriggerAlterTableEnd, makeNode, ProcessUtility, EventTriggerAlterTableStart, EventTriggerAlterTableRelid。

関連データ構造

  • AlterTableUtilityContext: ALTER TABLE の実行状態を管理する構造体。
  • PlannedStmt: ユーティリティコマンドをラップするために使用されるノード。

失敗モードと境界条件

  • ProcessUtility の内部で発生するエラー（構文エラー、権限不足、制約違反など）は、そのまま上位へ伝播する。
  • context が不正な場合、セグメンテーションフォールトが発生する可能性がある。

保守担当者向けメモ

  • 本関数はイベントトリガーの順序を保証するために不可欠である。サブコマンドを追加・変更する際は、イベントトリガーのライフサイクル（Start/End）が正しく維持されているか確認すること。
