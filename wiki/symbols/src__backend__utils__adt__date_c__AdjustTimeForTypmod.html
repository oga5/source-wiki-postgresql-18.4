AdjustTimeForTypmod

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/date.c:1709

目的

TIME 型の内部表現（マイクロ秒単位の int64）に対し、指定された精度（typmod）に基づいて値を丸め、精度を調整する。

シグネチャまたは形状

void AdjustTimeForTypmod(TimeADT *time, int32 typmod)

入力と出力

  • **入力**:
  • time: 調整対象となる TimeADT 型の値へのポインタ。
  • typmod: 保持すべき精度（0〜6の範囲）。
  • **出力**:
  • なし（ポインタ経由で time の値を直接更新する）。

前提条件と不変条件

  • typmod が 0 以上 MAX_TIME_PRECISION（通常 6）以下であること。
  • time が指す値はマイクロ秒単位の int64 であること。

アルゴリズムまたは振る舞い

1. typmod に応じたスケーリング係数（TimeScales）とオフセット値（TimeOffsets）を選択する。 2. time が正または 0 の場合、time + offset を scale で除算し、再度 scale を乗算することで切り捨て/四捨五入を行う。 3. time が負の場合、符号を反転させて同様の計算を行い、最後に符号を戻す。 4. typmod が範囲外の場合は何もしない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/date.c 内の各種時刻関連関数（time_in, time_recv, timetz_in 等）および src/backend/utils/adt/formatting.c の書式化関数から呼び出される。
  • **呼び出し先**: 内部的な定数配列を参照するのみで、他の関数は呼び出さない。

関連データ構造

  • TimeADT: int64 型のエイリアス。マイクロ秒単位の時刻を保持する。
  • MAX_TIME_PRECISION: 時刻精度の最大値（6）。

失敗モードと境界条件

  • typmod が負または MAX_TIME_PRECISION を超える場合、関数は何も行わず値を変更しない。
  • 負の時刻値に対しては、絶対値ベースで丸め処理が行われる。

保守担当者向けメモ

  • TimeScales および TimeOffsets は静的定数として定義されており、精度変更のロジックはこれらに依存している。
  • typmod の妥当性チェックは呼び出し側で行うことが推奨される（anytime_typmod_check 等を参照）。
