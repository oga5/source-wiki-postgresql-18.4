eval_const_expressions_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/clauses.c:61

目的

式ツリーの定数評価および簡略化処理（eval_const_expressions）において、再帰的な走査中に状態を保持・伝播するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    ParamListInfo boundParams;
    PlannerInfo *root;
    List       *active_fns;
    Node       *case_val;
    bool        estimate;
} eval_const_expressions_context;

入力と出力

本構造体は関数引数として渡され、以下の情報を保持します。

  • boundParams: 実行時にバインドされるパラメータ情報。
  • root: プランナのグローバル情報（PlannerInfo）。
  • active_fns: 再帰的なインライン化を防ぐための現在評価中の関数リスト。
  • case_val: CASE 式の評価用値。
  • estimate: 評価が推定目的であるかを示すフラグ。

前提条件と不変条件

  • eval_const_expressions 関連の関数群において、走査の開始時に初期化され、スタック上で管理されます。
  • 循環参照を防ぐため、active_fns を用いて再帰的な関数呼び出しを追跡する必要があります。

アルゴリズムまたは振る舞い

  • expression_tree_mutator を用いた式ツリーの変換処理において、各ノードの評価に必要な環境情報を提供します。
  • ece_function_is_safe などの補助関数に渡され、関数の安全性を判定するための判断材料として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/clauses.c 内の eval_const_expressions および関連する変換関数からインスタンス化されます。
  • **呼び出し先**: ece_function_is_safe 等の判定関数へ引数として渡されます。

関連データ構造

  • PlannerInfo: プランナの全体的なコンテキスト。
  • ParamListInfo: パラメータのバインド状態。
  • Node: 式ツリーの構成要素。

失敗モードと境界条件

  • 循環的な関数呼び出しが発生した場合、active_fns によるチェックが不十分だとスタックオーバーフローを引き起こす可能性があります。
  • estimate フラグの誤設定により、本来定数化すべき式がそのまま残る、あるいは誤って定数化されるリスクがあります。

保守担当者向けメモ

  • 本構造体は eval_const_expressions のロジックと密結合しています。フィールドを追加・変更する場合は、clauses.c 内のすべての _walker / _mutator 関数への影響を確認してください。
  • active_fns の管理は、SQL関数のインライン化における無限再帰を防止する重要な機構です。
