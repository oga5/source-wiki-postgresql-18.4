pg_read_binary_file_all

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:380

目的

サーバー上の指定されたファイルをバイナリデータ（bytea）としてすべて読み込み、SQLインターフェースへ返却します。

シグネチャまたは形状

Datum pg_read_binary_file_all(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介してファイルパス（text型）を受け取ります。
  • **出力**: ファイルの内容全体を bytea 型として返します。ファイルが存在しない場合は NULL を返します。

前提条件と不変条件

  • 呼び出し元は、PostgreSQLの権限管理システム（pg_read_server_files ロール等）による検証を通過している必要があります。
  • 指定されたパスは、convert_and_check_filename によってデータディレクトリまたはログディレクトリ内に制限されている必要があります。

アルゴリズムまたは振る舞い

1. 引数からファイルパスを取得します。 2. pg_read_binary_file_common を呼び出し、以下の引数を渡してファイル読み込みを実行します。

  • オフセット: 0
  • 長さ: -1（全読み込みを指定）
  • missing_ok: true（ファイル不在をエラーにしない）
  • include_size: false

    • 3. 読み込み結果が NULL の場合、PG_RETURN_NULL() で終了します。 4. 読み込み成功時、結果を PG_RETURN_BYTEA_P マクロで bytea 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（src/include/catalog/pg_proc.h に定義されたSQL関数経由）。
  • **呼び出し先**: pg_read_binary_file_common。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • text: ファイルパスを保持する可変長文字列型。
  • bytea: ファイル内容を格納するバイナリ型。

失敗モードと境界条件

  • ファイルが存在しない場合: missing_ok=true によりエラーにならず NULL を返します。
  • 権限不足またはパスが不正な場合: pg_read_binary_file_common 内部の convert_and_check_filename によりエラーが発生します。

保守担当者向けメモ

  • 本関数は pg_read_binary_file_all_missing 等と同様のラッパー関数です。機能変更を行う際は、他の関連ラッパー関数との整合性に注意してください。
  • ファイルシステムへの直接アクセスを伴うため、セキュリティ上の制約（pg_read_server_files）が正しく適用されていることを確認してください。
