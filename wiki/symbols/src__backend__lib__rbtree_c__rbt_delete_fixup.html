rbt_delete_fixup

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:520

目的

赤黒木のノード削除後に発生する平衡の崩れ（黒ノードの削除による黒の高さの不整合）を、回転と色変更によって修正する。

シグネチャまたは形状

static void rbt_delete_fixup(RBTree *rbt, RBTNode *x)

入力と出力

  • rbt: 操作対象の赤黒木構造体へのポインタ。
  • x: 削除されたノードの元の子ノード（黒ノード）。
  • 戻り値: なし。

前提条件と不変条件

  • 引数 x は常に黒ノードであること。
  • 削除操作により黒の高さが減少した箇所から修正を開始する。
  • ループ終了時、x の色は黒に設定される。

アルゴリズムまたは振る舞い

1. x がルートではなく、かつ黒ノードである間、以下の処理を繰り返す。 2. x が親の左子である場合と右子である場合で対称的な処理を行う。 3. 兄弟ノード（w）が赤の場合、回転と色変更を行い、w を黒にする。 4. 兄弟ノードの双方が黒の場合、w を赤に変更し、x を親ノードへ移動させて上に登る。 5. それ以外の場合、適切な回転と色変更を行い、平衡を回復させてループを終了する。 6. 最後に x を黒に設定し、不変条件を確定させる。

呼び出し元と呼び出し先

  • 呼び出し元: rbt_delete (src/backend/lib/rbtree.c:678)
  • 呼び出し先: rbt_rotate_left, rbt_rotate_right

関連データ構造

  • RBTree: 木の管理構造体。
  • RBTNode: ノード構造体。color メンバ（RBTBLACK, RBTRED）および parent, left, right ポインタを使用。

失敗モードと境界条件

  • x がルートに到達した場合、黒の高さの減少が許容されるためループが終了する。
  • 兄弟ノードが RBTNIL（番兵）である場合も、色の判定により適切に処理される。

保守担当者向けメモ

  • 本関数は Thomas Niemann 氏のアルゴリズムに基づいている。
  • 左右対称のケースが実装されているため、修正時は両方のパスに一貫性を持たせること。
