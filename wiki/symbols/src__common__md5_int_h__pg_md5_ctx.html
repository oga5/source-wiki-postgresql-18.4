pg_md5_ctx

定義

  • 種別: struct
  • 定義箇所: src/common/md5_int.h:54

目的

MD5 ハッシュ計算における進行状況と内部状態を保持するためのコンテキスト構造体です。PostgreSQL 内部のフォールバック実装で使用されます。

シグネチャまたは形状

typedef struct {
    union {
        uint32 md5_state32[4];
        uint8  md5_state8[16];
    } md5_st;
    union {
        uint64 md5_count64;
        uint8  md5_count8[8];
    } md5_count;
    unsigned int md5_i;
    uint8        md5_buf[MD5_BUFLEN];
} pg_md5_ctx;

入力と出力

  • **入力**: pg_md5_init で初期化され、pg_md5_update を通じてデータが蓄積されます。
  • **出力**: pg_md5_final を呼び出すことで、最終的なハッシュ値が指定されたバッファに書き出されます。

前提条件と不変条件

  • md5_buf は MD5_BUFLEN（64バイト）のサイズを持ち、データがこのサイズに達するごとに内部状態が更新されます。
  • md5_st および md5_count は、共用体を通じて 32/64 ビット整数とバイト配列の両面からアクセス可能です。

アルゴリズムまたは振る舞い

1. **初期化**: pg_md5_init により状態変数がリセットされます。 2. **更新**: pg_md5_update がデータを md5_buf に蓄積し、ブロック単位でハッシュ計算を進行させます。 3. **終了**: pg_md5_final がパディング処理を行い、最終的なハッシュ値を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash.c および src/common/md5.c 内の各関数（md5_calc, md5_pad, md5_result 等）がこの構造体を操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • MD5_BUFLEN: ブロックサイズ（64バイト）を定義する定数。
  • マクロ: md5_sta ～ md5_std（状態変数アクセス用）、md5_n / md5_n8（カウント用）。

失敗モードと境界条件

  • 外部ライブラリ（OpenSSL等）が利用可能な環境では、本構造体ではなく上位の抽象化インターフェース（cryptohash.h）の使用が推奨されます。直接操作する場合、メモリレイアウトやエンディアンの整合性に注意が必要です。

保守担当者向けメモ

  • 本構造体は内部実装用です。外部モジュールからの直接利用は避けてください。
  • 共用体によるアクセスは、エンディアンを意識した設計となっています。変更時は md5_common.c 等の関連実装との整合性を確認してください。
