AutoPrewarmReadStreamData

定義

  • 種別: struct
  • 定義箇所: contrib/pg_prewarm/autoprewarm.c:83

目的

read_stream API を使用して共有バッファへデータをプリウォームする際、読み込み対象のブロック情報と現在の進捗状態を保持するために使用されます。

シグネチャまたは形状

typedef struct AutoPrewarmReadStreamData
{
	BlockInfoRecord *block_info;
	int			pos;
	Oid			tablespace;
	RelFileNumber filenumber;
	ForkNumber	forknum;
	BlockNumber nblocks;
} AutoPrewarmReadStreamData;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。各フィールドはプリウォーム対象のブロック群（block_info）と、現在の読み込み位置（pos）、および対象リレーションの識別子（tablespace, filenumber, forknum, nblocks）を格納します。

前提条件と不変条件

  • pos は block_info 配列のインデックスを指します。
  • read_stream API による先読み（read-ahead）が発生するため、pos はメインループの処理位置よりも先行する可能性があります。

アルゴリズムまたは振る舞い

本構造体は、apw_read_stream_next_block 等のコールバック関数において、次に読み込むべきブロックを特定するためのコンテキストとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: autoprewarm_database_main 内でインスタンス化され、read_stream API へ渡されます。
  • **呼び出し先**: apw_read_stream_next_block 等のコールバック関数が、本構造体を callback_private_data として受け取り、内部の pos を参照・更新します。

関連データ構造

  • BlockInfoRecord: プリウォーム対象ブロックのメタデータ配列。
  • read_stream: PostgreSQL の効率的なブロック読み込み API。

失敗モードと境界条件

  • block_info が適切に初期化されていない場合、または pos が配列の範囲外を指す場合に不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • pos の更新は read_stream の動作と密接に関連しています。この構造体を変更する場合は、read_stream API のコールバック仕様との整合性に注意してください。
