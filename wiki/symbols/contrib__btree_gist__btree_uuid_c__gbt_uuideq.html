gbt_uuideq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_uuid.c:48

目的

2つの UUID 型の値が等しいかどうかを判定し、GiST インデックスにおける等価性比較を提供します。

シグネチャまたは形状

static bool gbt_uuideq(const void *a, const void *b, FmgrInfo *flinfo);

入力と出力

  • **入力**:
  • a, b: 比較対象となる pg_uuid_t 型へのポインタ（void * として渡される）。
  • flinfo: 関数管理情報（本関数内では使用されません）。
  • **出力**:
  • bool: a と b が等しい場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な pg_uuid_t 構造体を指している必要があります。
  • uuid_internal_cmp が提供する比較ロジックに基づき、バイト列としての完全一致を判定します。

アルゴリズムまたは振る舞い

1. uuid_internal_cmp を呼び出し、2つの UUID を比較します。 2. 比較結果が 0（等しい）であるかどうかを評価し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_uuid.c 内の静的データ構造（演算子クラス定義）から参照されています。
  • **呼び出し先**: uuid_internal_cmp を呼び出します。

関連データ構造

  • pg_uuid_t: PostgreSQL の UUID 型を保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の動作は定義されていません（セグメンテーション違反の可能性があります）。
  • uuid_internal_cmp の仕様に依存し、UUID のバイト表現が異なる場合は等価とみなされません。

保守担当者向けメモ

  • 本関数は btree_gist の UUID 演算子クラスにおける等価性判定のバックエンドです。
  • flinfo 引数は現在の実装では使用されていませんが、GiST のサポート関数インターフェースとの互換性のために保持されています。
