pqTraceOutput_NotificationResponse

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:230

目的

PostgreSQL プロトコルにおける NotificationResponse メッセージの内容をデバッグ用トレース出力として人間が読みやすい形式に変換する。

シグネチャまたは形状

void pqTraceOutput_NotificationResponse(FILE *f, const char *message, int *cursor, bool regress)

入力と出力

  • f: 出力先のファイルストリーム。
  • message: 解析対象のメッセージバッファ。
  • cursor: メッセージ内の現在の読み取り位置（オフセット）。関数実行後に更新される。
  • regress: 回帰テストモードフラグ。true の場合、可変情報が抑制される。
  • 出力: なし（f への書き込みのみ）。

前提条件と不変条件

  • message は有効な NotificationResponse メッセージのペイロードを指している必要がある。
  • cursor はメッセージの先頭を正しく指している必要がある。

アルゴリズムまたは振る舞い

1. 文字列 "NotificationResponse" を出力ストリームに書き込む。 2. pqTraceOutputInt32 を呼び出し、メッセージ内の整数フィールド（プロセスID等）をデコードして出力する。 3. pqTraceOutputString を2回呼び出し、メッセージに含まれる2つの文字列フィールド（通知名とペイロード）を順次出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-trace.c 内のメッセージディスパッチャ（pqTraceOutputMessage 等から呼び出される）。
  • 呼び出し先: pqTraceOutputInt32, pqTraceOutputString。

関連データ構造

  • FILE: 標準ライブラリのファイルストリーム。

失敗モードと境界条件

  • message や cursor が不正な場合、メモリ読み取りエラーや誤った解析結果を出力する可能性がある。
  • f が無効な場合、書き込み時にエラーが発生する。

保守担当者向けメモ

  • 本関数はプロトコル v3 の NotificationResponse メッセージ構造に依存している。プロトコル仕様に変更があった場合は、フィールドの順序や型を確認すること。
  • 回帰テストモード（regress）は、テスト結果の安定性を保つために重要である。新しいフィールドを追加する際は、テストへの影響を考慮すること。
