目的

pg_resetwal ユーティリティのメインエントリーポイントであり、コマンドライン引数の解析、データディレクトリの検証、および WAL（Write-Ahead Log）と制御ファイルの再構築プロセスを制御します。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。データディレクトリ指定 (-D) や各種パラメータのオーバーライドオプション（-x, -o, -m 等）を受け付けます。
  • **出力**:
  • 正常終了時は 0 を返します。
  • 引数エラーや処理失敗時は 1 を返し、エラーメッセージを標準エラー出力に出力します。

前提条件と不変条件

  • 実行ユーザーが root ではないこと（ユーティリティ内部でチェックされます）。
  • 実行中の PostgreSQL サーバー（postmaster.pid）が存在しないこと。
  • 破損したデータディレクトリに対して実行される場合、-f (force) オプションが指定されない限り、推測値での更新は行われません。

アルゴリズムまたは振る舞い

1. **初期化**: ログ設定およびロケール設定を初期化します。 2. **引数解析**: getopt_long を使用し、データディレクトリパスや、トランザクションID、OID、マルチトランザクションID等の手動設定値を解析します。 3. **検証**: 指定されたディレクトリの妥当性を確認し、postmaster.pid の有無によりサーバーが停止していることを保証します。 4. **制御ファイル処理**: read_controlfile() で既存の pg_control を読み込みます。読み込み失敗時は GuessControlValues() で値を推測します。 5. **更新処理**: ユーザー指定のオプションを反映し、RewriteControlFile() で制御ファイルを更新します。 6. **WAL 再構築**: KillExistingXLOG() 等で古い WAL 関連ファイルを削除し、WriteEmptyXLOG() で新しい WAL セグメントを生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（コマンドライン実行）。
  • **呼び出し先**:
  • getopt_long: 引数解析。
  • read_controlfile, GuessControlValues, RewriteControlFile: 制御ファイルの管理。
  • KillExistingXLOG, WriteEmptyXLOG: WAL の削除および再構築。
  • pg_log_error, pg_fatal: エラー報告。
  • **注記**: ツールによる contrib/amcheck 等からの参照は、本シンボルとは無関係な「main」という名称の偶然の一致であり、本関数への呼び出しではありません。

関連データ構造

  • ControlFileData (制御ファイル構造体)
  • struct option (コマンドライン引数定義)

失敗モードと境界条件

  • 引数に不正な値（範囲外のトランザクションIDやOIDなど）が指定された場合、エラーメッセージを表示して終了します。
  • 必須のデータディレクトリが指定されていない、またはアクセス権限がない場合に失敗します。
  • 実行中のサーバーが存在する場合、安全のため処理を中断します。

保守担当者向けメモ

  • 本関数はユーティリティの全フローを統括するため、変更時は pg_control の整合性チェックや WAL 再構築の順序に注意してください。
  • getopt_long のケース文で各オプションのバリデーションを行っています。新しいオプションを追加する際は、既存の strtoul 等による型変換とエラーハンドリングのパターンに従ってください。
