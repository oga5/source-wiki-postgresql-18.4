KillExistingWALSummaries

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:1077

目的

データディレクトリ内の pg_wal/summaries ディレクトリから、WAL サマリーファイルをすべて削除し、クリーンな状態にする。

シグネチャまたは形状

static void KillExistingWALSummaries(void)

入力と出力

  • 入力: なし（XLOGDIR 定数に基づき、固定のディレクトリパスを参照する）
  • 出力: なし（成功時はディレクトリ内の対象ファイルを削除。失敗時は pg_fatal によりプロセスを終了する）

前提条件と不変条件

  • XLOGDIR が指すディレクトリ内に summaries サブディレクトリが存在すること。
  • プロセスが当該ディレクトリおよびファイルに対する削除権限を持っていること。

アルゴリズムまたは振る舞い

1. pg_wal/summaries ディレクトリを opendir で開く。失敗時は pg_fatal を発行。 2. ディレクトリ内のエントリを readdir で順次走査する。 3. ファイル名が「40桁の16進数 + .summary」という形式に合致する場合、そのファイルを unlink で削除する。削除失敗時は pg_fatal を発行。 4. 走査完了後、closedir でディレクトリを閉じる。閉鎖失敗時は pg_fatal を発行。

呼び出し元と呼び出し先

  • 呼び出し元: main (src/bin/pg_resetwal/pg_resetwal.c:524)
  • 呼び出し先: opendir, readdir, strspn, strcmp, snprintf, unlink, closedir, pg_fatal

関連データ構造

  • DIR, struct dirent: ディレクトリ走査に使用される標準ライブラリ構造体。

失敗モードと境界条件

  • ディレクトリが開けない場合、または走査中にエラーが発生した場合は pg_fatal により異常終了する。
  • ファイルの削除（unlink）に失敗した場合も pg_fatal により異常終了する。
  • 形式に合致しないファイルは無視され、削除されない。

保守担当者向けメモ

  • 本関数は pg_resetwal の破壊的更新フェーズの一部として実行される。
  • WALSUMMARY_NHEXCHARS（40桁）は、WAL サマリーファイルの命名規則に依存しているため、将来的に命名規則が変更された場合は修正が必要。
