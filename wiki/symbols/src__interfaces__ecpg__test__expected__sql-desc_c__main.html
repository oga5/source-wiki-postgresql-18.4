目的

ECPG（Embedded SQL for C）における SQL 記述子（Descriptor）領域の操作および動的 SQL 実行の挙動を検証するための回帰テスト用エントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし
  • 出力: 終了コード（通常は 0）

前提条件と不変条件

  • 実行環境においてデータベース ecpg1_regression への接続が可能であること。
  • プリプロセッサによって生成されたコードであるため、SQL 文の実行直後には必ず sqlca.sqlcode を確認し、エラー発生時に sqlprint() を呼び出す不変条件が各ステップに適用されています。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、ECPGallocate_desc で記述子領域（"indesc" および "outdesc"）を確保します。 2. **設定**: ECPGset_desc を使用して、記述子内のデータフィールドに C 言語の変数や定数をマッピングします。 3. **接続と準備**: ECPGconnect でデータベースに接続し、ECPGprepare で複数の SQL 文（INSERT, SELECT）をプリペアドステートメントとして登録します。 4. **実行**: ECPGdo を介して、記述子に格納されたパラメータを動的 SQL に渡して実行します。 5. **更新**: 記述子の内容を ECPGset_desc で再設定し、異なるパラメータで再度 SQL を実行します。 6. **終了**: ECPGdeallocate を呼び出し、リソースを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（テスト実行プロセス）
  • 呼び出し先: ECPGdebug, ECPGallocate_desc, ECPGset_desc, ECPGconnect, ECPGdo, ECPGprepare, ECPGdeallocate, sqlprint

関連データ構造

  • sqlca: SQL 通信領域（エラーチェックに使用）。
  • stmt1, stmt2, stmt3: 実行される SQL 文を保持する文字列。
  • val1, val2, val2i, val2null: 記述子にバインドされる C 言語変数。

失敗モードと境界条件

  • データベース接続失敗や SQL 文の実行エラーが発生した場合、sqlca.sqlcode < 0 の条件により sqlprint() が実行され、標準エラー出力にエラー詳細が出力されます。
  • 記述子のインデックスは 1 から始まる仕様となっており、これに違反する設定はエラーの原因となります。
