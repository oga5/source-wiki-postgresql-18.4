StatsArgInfo

定義

  • 種別: struct
  • 定義箇所: src/include/statistics/stat_utils.h:21

目的

統計情報操作関数に渡される引数のメタデータ（名前と型）を保持し、バリデーションロジックを汎用化するための構造体です。

シグネチャまたは形状

struct StatsArgInfo
{
	const char *argname;
	Oid			argtype;
};

入力と出力

  • **入力**: 構造体の各フィールド（argname: 引数名、argtype: 引数のデータ型を示す Oid）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • argname は引数の名称を指す文字列である必要があります。
  • argtype は PostgreSQL のシステムカタログにおける有効な型 Oid である必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行います。この構造体を利用する関数（stat_utils.c 内のバリデーション関数など）は、FunctionCallInfo から取得した引数と本構造体のメタデータを照合し、型の一致や NULL チェック、配列の次元数検証を行います。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/statistics/attribute_stats.c: attarginfo, cleararginfo の定義で使用。
  • src/backend/statistics/relation_stats.c: relarginfo の定義で使用。
  • src/backend/statistics/stat_utils.c: 各種バリデーション関数および get_arg_by_name で引数として使用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • FunctionCallInfo: fmgr.h で定義される、関数呼び出し時の引数情報。本構造体と組み合わせてバリデーションに使用されます。

失敗モードと境界条件

  • 構造体自体に失敗モードはありません。本構造体を利用するバリデーション関数側で、引数の型不一致や NULL 違反が発生した場合に ereport によるエラーが発生します。

保守担当者向けメモ

  • 本構造体は統計情報操作関数の引数検証を共通化するためのものです。新しい統計操作関数を追加する際は、この構造体の配列を定義し、stat_utils.c の検証関数に渡すことで、定型的なバリデーションを簡潔に実装できます。
