compute_array_stats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_typanalyze.c:215

目的

配列型カラムに対する統計情報（pg_statistic の MCELEM および DECHIST）を収集する。Lossy Counting アルゴリズムを用いて、メモリ効率を維持しつつ配列内の頻出要素を特定する。

シグネチャまたは形状

static void compute_array_stats(VacAttrStats *stats, AnalyzeAttrFetchFunc fetchfunc, int samplerows, double totalrows);

入力と出力

  • **入力**:
  • stats: 統計情報を格納する VacAttrStats 構造体。
  • fetchfunc: サンプリングされた行を取得するための関数ポインタ。
  • samplerows: 解析対象のサンプル行数。
  • totalrows: テーブルの総行数。
  • **出力**: なし（stats 構造体を通じて pg_statistic 用のデータを更新する）。

前提条件と不変条件

  • stats->extra_data に ArrayAnalyzeExtraData が設定されていること。
  • ARRAY_WIDTH_THRESHOLD を超えるサイズの配列は解析対象から除外される。
  • CurrentMemoryContext 上でハッシュテーブルが構築され、処理終了時に自動的に解放される。

アルゴリズムまたは振る舞い

1. **標準統計の収集**: std_compute_stats を呼び出し、スカラー型の統計を先に生成する。 2. **ハッシュテーブル構築**: 要素追跡用の elements_tab と、一意な要素数カウント用の count_tab を初期化する。 3. **サンプリングループ**:

  • 各行の配列をデコードし、要素を走査する。
  • Lossy Counting アルゴリズムに基づき、要素の出現頻度をハッシュテーブルで追跡する。
  • 配列ごとの一意な要素数をカウントし、分布を記録する。

    • 4. **メモリ管理**: 要素型が参照渡しの場合は datumCopy を使用し、ハッシュテーブル内にデータを保持する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_typanalyze.c 内の array_typanalyze（stats->compute_stats に代入される）。
  • **呼び出し先**:
  • std_compute_stats（標準統計生成）
  • hash_create, hash_search（ハッシュテーブル操作）
  • deconstruct_array（配列の分解）
  • datumCopy（要素のコピー）
  • vacuum_delay_point（VACUUM 処理の遅延制御）

関連データ構造

  • ArrayAnalyzeExtraData: 型情報や比較・ハッシュ関数を保持。
  • TrackItem: elements_tab で使用される要素追跡用構造体。
  • DECountItem: count_tab で使用される要素数カウント用構造体。

失敗モードと境界条件

  • **NULL 値**: 配列全体が NULL の場合はスキップされる。
  • **サイズ制限**: ARRAY_WIDTH_THRESHOLD を超える配列は解析から除外される。
  • **メモリ**: ハッシュテーブルはローカルメモリを使用するため、システム全体のリソース制限に依存する。

保守担当者向けメモ

  • 本関数は Manku & Motwani の Lossy Counting アルゴリズムを実装している。精度は bucket_width に依存する。
  • DEBUG3 レベルでハッシュテーブルのサイズ等のログが出力されるため、統計精度に問題がある場合は確認すること。
  • std_compute_stats の呼び出し前後で extra_data を切り替える必要がある点に注意。
