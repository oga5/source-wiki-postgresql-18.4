get_tables_to_cluster_partitioned

定義

  • 種別: function
  • 定義箇所: src/backend/commands/cluster.c:1696

目的

パーティション化されたテーブルに対し、クラスタリング対象となるリーフパーティションのリストを収集します。

シグネチャまたは形状

List *get_tables_to_cluster_partitioned(MemoryContext cluster_context, Oid indexOid)

入力と出力

  • **入力**:
  • cluster_context: 結果リストを割り当てるためのメモリコンテキスト。
  • indexOid: クラスタリング対象のインデックスのOID。
  • **出力**:
  • List *: RelToCluster 構造体のリスト。各要素は対象となるテーブルOIDとインデックスOIDを保持します。

前提条件と不変条件

  • indexOid はパーティション化されたテーブルに関連するインデックスである必要があります。
  • リストの生成には cluster_context が使用され、メモリ管理の整合性が保たれます。

アルゴリズムまたは振る舞い

1. find_all_inheritors を使用し、指定されたインデックスに関連するすべての継承関係（パーティション）を NoLock モードで取得します。 2. 取得したリストを反復処理します。

  • RELKIND_INDEX ではない要素はスキップします。
  • cluster_is_permitted_for_relation を呼び出し、現在のユーザーが当該パーティションに対してクラスタリング権限を持っているか検証します。権限がない場合はスキップします。

    • 3. 権限がある場合、cluster_context に切り替えて RelToCluster 構造体を割り当て、テーブルOIDとインデックスOIDを格納してリストに追加します。 4. 最終的に収集されたリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/cluster.c 内の get_tables_to_cluster から呼び出されます。
  • **呼び出し先**:
  • find_all_inheritors
  • IndexGetRelation
  • get_rel_relkind
  • cluster_is_permitted_for_relation
  • GetUserId
  • MemoryContextSwitchTo
  • palloc
  • lappend

関連データ構造

  • RelToCluster: テーブルOIDとインデックスOIDを保持する構造体。
  • List: PostgreSQLの標準的なリスト構造。

失敗モードと境界条件

  • ユーザーが特定のパーティションに対する権限を持たない場合、そのパーティションはリストから除外されます（エラーにはなりません）。
  • indexOid が無効な場合や、継承関係が見つからない場合の挙動については、find_all_inheritors の仕様に依存します。

保守担当者向けメモ

  • 本関数は NoLock でパーティションを検索するため、後続の処理で適切にロックを取得する必要があります。
  • 権限チェックは各リーフパーティションに対して個別に実行されるため、親テーブルへの権限とリーフへの権限が異なる場合に注意が必要です。
