network_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network.c:269

目的

inet または cidr 型の内部表現を、PostgreSQL のフロントエンド/バックエンド通信用のバイナリ形式（bytea）に変換します。

シグネチャまたは形状

bytea *network_send(inet *addr, bool is_cidr)

入力と出力

  • **入力**:
  • addr: 変換対象の inet 構造体へのポインタ。
  • is_cidr: cidr 型として扱う場合は true、inet 型として扱う場合は false。
  • **出力**:
  • ネットワークバイトオーダーでシリアライズされた bytea 形式のデータ。

前提条件と不変条件

  • addr は有効な inet 構造体を指している必要があります。
  • pq_begintypsend および pq_endtypsend を使用するため、本関数は PostgreSQL の通信プロトコル層のコンテキスト内で呼び出される必要があります。

アルゴリズムまたは振る舞い

1. pq_begintypsend で送信バッファを初期化します。 2. IP ファミリー（IPv4/IPv6）、ネットマスク長、is_cidr フラグ、およびアドレスのバイトサイズを順次バッファに書き込みます。 3. アドレス本体のバイト列をバッファに追加します。 4. pq_endtypsend を呼び出し、構築された StringInfo を bytea 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • inet_send（inet 型のバイナリ送信関数）
  • cidr_send（cidr 型のバイナリ送信関数）
  • **呼び出し先**:
  • pq_begintypsend, pq_sendbyte, pq_endtypsend（libpq/pqformat.h）
  • ip_family, ip_bits, ip_addrsize, ip_addr（utils/inet.h のマクロ）

関連データ構造

  • inet: ネットワークアドレスを保持する内部構造体。
  • StringInfoData: 通信用のバッファ構造体。
  • bytea: PostgreSQL の可変長バイナリ型。

失敗モードと境界条件

  • メモリ不足時には pq_begintypsend 等の内部関数がエラーを発生させます。
  • ip_addrsize が返すサイズと実際の addrptr の内容が不整合な場合、不正なバイナリデータが生成される可能性があります。

保守担当者向けメモ

  • 本関数は inet と cidr の両方で共通のシリアライズロジックを提供しています。
  • プロトコル形式を変更する場合は、inet_recv 側のデシリアライズロジックと整合性を取る必要があります。
