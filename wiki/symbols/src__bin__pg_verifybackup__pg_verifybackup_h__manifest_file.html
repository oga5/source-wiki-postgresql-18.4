manifest_file

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.h:28

目的

バックアップマニフェストに含まれる個々のファイルに関するメタデータと、検証プロセスにおける状態を保持するために使用されます。

シグネチャまたは形状

typedef struct manifest_file
{
	uint32		status;			/* hash status */
	const char *pathname;
	uint64		size;
	pg_checksum_type checksum_type;
	int			checksum_length;
	uint8	   *checksum_payload;
	bool		matched;
	bool		bad;
} manifest_file;

入力と出力

  • 本構造体は、マニフェスト解析時に生成されるデータモデルの一部です。
  • 検証プロセスにおいて、ファイルパスをキーとして検索され、matched や bad フラグが更新されます。

前提条件と不変条件

  • pathname はマニフェスト内で一意である必要があります。
  • checksum_payload は checksum_length に基づくバイト配列を指します。

アルゴリズムまたは振る舞い

  • lib/simplehash.h を用いたハッシュテーブルの要素として利用されます。
  • 検証中、ファイルシステム上の実ファイルと照合された際に matched が true に設定されます。
  • チェックサムの不一致やファイルサイズの不整合が検出された場合、bad が true に設定されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_verifybackup/pg_verifybackup.c がマニフェスト解析時に本構造体をインスタンス化し、ハッシュテーブルに格納します。
  • 呼び出し先: src/bin/pg_verifybackup/astreamer_verify.c がアーカイブストリーム処理中に本構造体を参照し、検証状態を更新します。
  • 注意: src/bin/pg_basebackup/pg_basebackup.c や src/bin/pg_combinebackup/load_manifest.c にも同名のシンボルが存在しますが、これらが本構造体と同一の定義を参照しているか、あるいは単なる名前の衝突であるかは、提供されたソース情報からは断定できません。

関連データ構造

  • manifest_data: マニフェスト全体の情報を保持し、manifest_file のハッシュテーブルを管理します。
  • verifier_context: 検証実行時のコンテキストを保持し、本構造体へのアクセスを仲介します。

失敗モードと境界条件

  • checksum_payload が NULL の場合、チェックサム検証はスキップされます。
  • matched が false のまま検証が終了した場合、バックアップ内にマニフェストに記載のないファイルが存在するか、あるいはマニフェストに記載されたファイルが欠落していることを示します。

保守担当者向けメモ

  • simplehash.h のテンプレート生成マクロ（SH_ プレフィックス）と密接に関連しています。構造体のフィールドを変更する場合は、ハッシュテーブルの定義側も確認してください。
  • status フィールドの具体的な値の定義については、simplehash.h の実装および pg_verifybackup.c での利用箇所を参照してください。
