scan_profile

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/args.c:197

目的

プロファイルファイル（.indent.pro）から設定オプションを読み込み、トークン化して set_option 関数へ渡すことで、プログラムの設定を反映させる。

シグネチャまたは形状

static void scan_profile(FILE *f)

入力と出力

  • 入力: FILE *f（読み込み対象のプロファイルファイルポインタ）
  • 出力: なし（副作用としてグローバルな設定変数を更新する）

前提条件と不変条件

  • f は読み込み可能な有効なファイルストリームであること。
  • プロファイル内のオプションは空白文字で区切られていること。
  • /* ... */ 形式のコメントは解析対象から除外される。

アルゴリズムまたは振る舞い

1. ファイルから文字を順次読み込み、buf に格納する。 2. /* を検出するとコメントモードに入り、*/ が出現するまで入力をスキップする。 3. 空白文字（isspace）を検出すると、現在のトークンを終了し set_option(buf) を呼び出す。 4. verbose が真の場合、解析したオプションを標準出力に表示する。 5. ファイルの終端（EOF）に達するまでこの処理を繰り返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/tools/pg_bsd_indent/args.c 内の set_profile 関数から呼び出される。
  • 呼び出し先: set_option 関数を呼び出し、個別のオプションを適用する。

関連データ構造

  • buf[BUFSIZ]: トークンを一時保持するためのバッファ。
  • verbose: 解析状況を出力するかを制御するグローバル変数。

失敗モードと境界条件

  • 不明なオプションが渡された場合、set_option 内部で errx が呼び出され、プログラムが終了する可能性がある。
  • ファイルが空の場合、何も処理を行わずに終了する。
  • コメントが閉じられていない場合や、バッファサイズを超えるトークンが存在する場合の挙動は、標準ライブラリの getc およびバッファ操作に依存する。
