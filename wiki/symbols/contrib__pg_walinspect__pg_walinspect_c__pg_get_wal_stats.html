pg_get_wal_stats

定義

  • 種別: function
  • 定義箇所: contrib/pg_walinspect/pg_walinspect.c:803

目的

指定されたWAL（Write-Ahead Log）の範囲内における統計情報を集計し、SQLインターフェースを通じて提供します。

シグネチャまたは形状

Datum pg_get_wal_stats(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • start_lsn (LSN): 解析を開始するWALのLSN。
  • end_lsn (LSN): 解析を終了するWALのLSN。
  • stats_per_record (bool): レコード単位で統計を集計するかどうかを指定するフラグ。
  • **出力**:
  • 戻り値は void ですが、内部で統計情報を集計し、結果をテーブル形式で返します。

前提条件と不変条件

  • ValidateInputLSNs により、入力された start_lsn および end_lsn が有効な範囲内にあることが検証されます。
  • 内部で XLogReaderState を使用するため、WALの読み取りが可能な環境である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_LSN および PG_GETARG_BOOL を使用して引数を取得します。 2. ValidateInputLSNs を呼び出し、指定されたLSN範囲の妥当性を確認します。 3. GetWalStats を呼び出し、指定された範囲のWALレコードを走査して統計情報を集計します。 4. 最後に PG_RETURN_VOID() を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • PG_FUNCTION_INFO_V1(pg_get_wal_stats) マクロにより、PostgreSQLの関数として登録されています。
  • **呼び出し先**:
  • ValidateInputLSNs: 入力LSNの検証。
  • GetWalStats: 統計情報の集計処理。

関連データ構造

  • XLogRecPtr: LSN（Log Sequence Number）を保持する型。
  • PG_FUNCTION_ARGS: PostgreSQLの関数引数構造体。

失敗モードと境界条件

  • ValidateInputLSNs が無効なLSN範囲を検出した場合、エラーが発生します。
  • WALの読み取り中にエラーが発生した場合、PostgreSQLの標準的なエラー報告メカニズムに従い処理が中断されます。

保守担当者向けメモ

  • 本関数は pg_walinspect 拡張の統計機能の入り口です。
  • 統計情報の集計ロジックを変更する場合は、GetWalStats の実装を確認してください。
  • _till_end_of_wal を含む関連関数との整合性に注意してください。
