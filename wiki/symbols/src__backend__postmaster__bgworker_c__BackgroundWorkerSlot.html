BackgroundWorkerSlot

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/bgworker.c:74

目的

共有メモリ上でバックグラウンドワーカーの状態を保持し、Postmasterとバックエンドプロセス間でのワーカー管理（起動、監視、終了）を仲介するために使用されます。

シグネチャまたは形状

typedef struct BackgroundWorkerSlot
{
    bool        in_use;
    bool        terminate;
    pid_t       pid;            /* InvalidPid = 未起動; 0 = 停止済み */
    uint64      generation;     /* スロット再利用時にインクリメント */
    BackgroundWorker worker;
} BackgroundWorkerSlot;

入力と出力

  • **入力**: BackgroundWorker 構造体（ワーカーの構成情報）。
  • **出力**: プロセスID (pid)、終了フラグ (terminate)、およびワーカーの実行状態。

前提条件と不変条件

  • Postmasterは共有メモリのロックを取得できないため、in_use フラグとメモリバリアを用いたロックレスなプロトコルで同期されます。
  • generation はスロットが再利用されるたびに更新され、古いスロット情報との混同を防ぎます。

アルゴリズムまたは振る舞い

1. **登録**: バックエンドが RegisterDynamicBackgroundWorker を通じてスロットを確保し、in_use を true に設定します。 2. **監視**: Postmasterは BackgroundWorkerStateChange を介してスロットをスキャンし、in_use かつ pid が InvalidPid であるワーカーを起動します。 3. **終了**: terminate フラグが設定されると、Postmasterは対象プロセスに終了シグナルを送信します。 4. **解放**: ワーカー終了後、ReportBackgroundWorkerExit によりスロットの状態が更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/bgworker.c 内のワーカー管理関数群（RegisterDynamicBackgroundWorker, BackgroundWorkerStateChange, ReportBackgroundWorkerExit 等）。
  • **呼び出し先**: 直接的な関数呼び出しはありません。データ構造として BackgroundWorkerData 配列の一部として参照されます。

関連データ構造

  • BackgroundWorkerArray: 共有メモリ上の BackgroundWorkerSlot の配列。
  • BackgroundWorker: ワーカーの定義情報を含む構造体。

失敗モードと境界条件

  • **データ破損**: Postmasterが共有メモリを読み取る際、データが破損している可能性があるため、常にパラノイア的なチェックが求められます。
  • **スロット枯渇**: 登録可能なワーカー数は共有メモリ上のスロット数に制限されます。

保守担当者向けメモ

  • 本構造体へのアクセスは、ロックレスプロトコルに従い、必ず適切なメモリバリア（pg_read_barrier / pg_write_barrier）を伴う必要があります。
  • pid の値が InvalidPid（未起動）か 0（停止済み）かによって、Postmasterの再起動ロジックが分岐するため、状態遷移の正確な管理が重要です。
