ExecInitSeqScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:206

目的

シーケンシャルスキャン（全件走査）を実行するための SeqScanState ノードを構築・初期化し、実行時の条件（フィルタやプロジェクションの有無）に応じて最適な実行関数（ExecProcNode）を割り当てる。

シグネチャまたは形状

SeqScanState *ExecInitSeqScan(SeqScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の SeqScan プランノード。
  • estate: 実行状態を管理する EState。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの SeqScanState 構造体へのポインタ。

前提条件と不変条件

  • outerPlan(node) および innerPlan(node) は NULL であること（シーケンシャルスキャンは子プランを持たない）。
  • EvalPlanQual がアクティブな場合を除き、qual（フィルタ条件）と ps_ProjInfo（プロジェクション情報）の有無に基づいて実行関数が決定される。

アルゴリズムまたは振る舞い

1. SeqScanState 構造体を生成し、プランノードと実行状態を関連付ける。 2. ExecAssignExprContext で式評価コンテキストを初期化する。 3. ExecOpenScanRelation で対象リレーションを開き、ExecInitScanTupleSlot でタプルスロットを構築する。 4. ExecInitResultTypeTL および ExecAssignScanProjectionInfo でプロジェクション情報を設定する。 5. ExecInitQual でフィルタ条件を初期化する。 6. 以下の条件分岐により、scanstate->ss.ps.ExecProcNode に最適な関数を割り当てる：

  • es_epq_active が有効な場合: ExecSeqScanEPQ
  • qual が NULL かつ ps_ProjInfo が NULL: ExecSeqScan
  • qual が NULL かつ ps_ProjInfo が存在: ExecSeqScanWithProject
  • qual が存在し ps_ProjInfo が NULL: ExecSeqScanWithQual
  • qual と ps_ProjInfo が共に存在: ExecSeqScanWithQualProject

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c（ExecInitSeqScan を呼び出すことが確認されている）。
  • **呼び出し先**: ExecInitScanTupleSlot, ExecInitQual, ExecAssignScanProjectionInfo 等の実行基盤関数。

関連データ構造

  • SeqScanState: スキャンノードの実行状態を保持する構造体。
  • SeqScan: プランノード。
  • EState: クエリ実行全体のグローバルな状態。

失敗モードと境界条件

  • リレーションのオープン失敗やメモリ不足時には、PostgreSQLの標準的なエラーハンドリングにより例外が発生する。

保守担当者向けメモ

  • 本関数は、実行時のオーバーヘッドを最小化するために、条件の有無で実行関数を動的に切り替える最適化を行っている。新しい実行パスを追加する場合は、この分岐ロジックを更新する必要がある。
