escape_quotes_bki

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/initdb.c:421

目的

文字列を postgres.bki（ブートストラップ用カタログ定義ファイル）の構文に適した形式へ変換する。具体的には、文字列をエスケープした上で、全体をシングルクォートで囲む処理を行う。

シグネチャまたは形状

char *escape_quotes_bki(const char *src);

入力と出力

  • **入力**: src (変換対象の文字列)
  • **出力**: 変換後の文字列を格納した動的に割り当てられたメモリ領域。呼び出し元で free() する必要がある。

前提条件と不変条件

  • src はヌル終端文字列であること。
  • 内部で escape_quotes() を呼び出しているため、その関数がメモリを割り当て可能であること。
  • pg_malloc を使用しているため、メモリ不足時にはプログラムが終了する可能性がある。

アルゴリズムまたは振る舞い

1. escape_quotes(src) を呼び出し、エスケープ処理済みの文字列 data を取得する。 2. data の長さにシングルクォート2文字分とヌル終端文字分を加えたサイズでメモリを確保する。 3. 確保したメモリの先頭に ' を書き込む。 4. data の内容をコピーする。 5. 最後に ' を書き込み、ヌル終端する。 6. 中間生成物である data を free() し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/initdb/initdb.c 内の複数箇所（ユーザー名、ロケール設定、ICUルール等の文字列を BKI ファイルへ書き出す際に使用）。
  • **呼び出し先**:
  • escape_quotes()
  • pg_malloc()
  • free()

関連データ構造

  • 特になし。標準的な C 文字列（char *）を扱う。

失敗モードと境界条件

  • src が NULL の場合、escape_quotes() の実装に依存する（本ソース excerpt には escape_quotes の定義が含まれていないため、詳細は不明）。
  • メモリ割り当て失敗時は pg_malloc によりプロセスが異常終了する。

保守担当者向けメモ

  • 本関数は postgres.bki への書き出し専用である。通常の SQL 文字列リテラル生成には使用しないこと。
  • escape_quotes() の戻り値を free() する必要があるため、メモリリークに注意すること。
