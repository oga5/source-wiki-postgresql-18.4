check_relation_privileges

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/dml.c:141

目的

指定されたリレーション（テーブル、シーケンス、ビュー）およびそのカラムに対するSELinuxセキュリティ権限を検証します。

シグネチャまたは形状

static bool check_relation_privileges(Oid relOid,
                                      Bitmapset *selected,
                                      Bitmapset *inserted,
                                      Bitmapset *updated,
                                      uint32 required,
                                      bool abort_on_violation)

入力と出力

  • **入力**:
  • relOid: 対象リレーションのOID。
  • selected, inserted, updated: 各操作対象となるカラムのビットマップ。
  • required: 要求されるテーブルレベルの権限ビットマスク。
  • abort_on_violation: 権限違反時にエラーを発生させるかを示すフラグ。
  • **出力**:
  • 権限チェックが成功した場合は true、失敗した場合は false（またはエラー発生）。

前提条件と不変条件

  • sepgsql_getenforce() > 0 の場合、システムカタログへのDML操作およびTOASTリレーションへのアクセスは、ポリシー設定に関わらず強制的に拒否されます。
  • カラムレベルの権限チェックは、RELKIND_RELATION または RELKIND_PARTITIONED_TABLE の場合にのみ実行されます。

アルゴリズムまたは振る舞い

1. **ハードコードされたポリシーの適用**: システムカタログへの更新操作や、TOASTテーブルへのアクセスを検出し、違反時はエラーを投げます。 2. **リレーションレベルの検証**: relkind に応じて適切なSELinuxクラス（DB_TABLE, DB_SEQUENCE, DB_VIEW）を選択し、sepgsql_avc_check_perms を呼び出します。 3. **カラムレベルの検証**:

  • fixup_whole_row_references を用いて行全体参照を全カラム参照へ展開します。
  • 選択・挿入・更新の各ビットマップを統合し、対象となる全カラムを走査します。
  • 各カラムに対して必要な権限を計算し、sepgsql_avc_check_perms で検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/dml.c 内の関数（sepgsql_dml_privileges 等からの呼び出しが確認されています）。
  • **呼び出し先**:
  • sepgsql_getenforce()
  • getObjectIdentity(), getObjectDescription()
  • sepgsql_avc_check_perms()
  • fixup_whole_row_references()
  • bms_union(), bms_next_member(), bms_is_member()

関連データ構造

  • Bitmapset: カラム操作の追跡に使用。
  • ObjectAddress: 権限チェック対象の識別に使用。

失敗モードと境界条件

  • **エラー**: システムカタログへのDML操作やTOASTテーブルへのアクセスが試みられた場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了します。
  • **境界条件**: relkind が RELKIND_RELATION 等以外の場合、カラムレベルのチェックはスキップされます。

保守担当者向けメモ

  • 属性番号の計算には FirstLowInvalidHeapAttributeNumber を用いたオフセット調整が含まれています。
  • 継承テーブルの処理において、親と子で属性番号が異なる可能性があるため、呼び出し元での適切な補正が必要です。
