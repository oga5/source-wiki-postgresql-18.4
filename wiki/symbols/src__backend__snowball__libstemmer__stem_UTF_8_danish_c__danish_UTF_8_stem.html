danish_UTF_8_stem

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_danish.c:280

目的

デンマーク語の単語に対し、UTF-8エンコーディングを前提としたステミング（語幹抽出）処理を実行する。

シグネチャまたは形状

int danish_UTF_8_stem(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境を保持する構造体。現在の単語データやカーソル位置を含む）
  • 出力: int（処理成功時に 1 を返し、エラー発生時に負の値を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 処理開始時に r_mark_regions が呼び出され、単語内の処理領域（R1, R2相当）が定義される。
  • 処理中、z->c（カーソル位置）および z->lb（境界）が適切に更新され、単語の範囲内で操作が行われる。

アルゴリズムまたは振る舞い

1. r_mark_regions を実行し、単語内の処理境界を特定する。 2. z->lb を現在のカーソル位置に設定し、カーソルを単語の末尾（z->l）へ移動させる。 3. 以下の順序でルーチンを適用し、各ステップでカーソル位置を復元しながら処理を進める：

  • r_main_suffix: 主要な接尾辞の削除。
  • r_consonant_pair: 子音ペアの処理。
  • r_other_suffix: その他の接尾辞の削除と置換。
  • r_undouble: 重複子音の解消。

    • 4. 最後にカーソルを z->lb に戻し、成功を示す 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定不可。
  • 呼び出し先: r_mark_regions, r_main_suffix, r_consonant_pair, r_other_suffix, r_undouble。

関連データ構造

  • struct SN_env: ステミング処理の状態（カーソル c、境界 lb、文字列長 l 等）を保持する。

失敗モードと境界条件

  • 各内部ルーチン（r_ で始まる関数）が負の値を返した場合、その時点で処理を中断し、エラーコードを呼び出し元へ伝播させる。

保守担当者向けメモ

  • 本関数は自動生成コードであるため、直接編集せず、元のSnowballソース（.sblファイル）を修正すること。
  • デバッグ時は z->c（カーソル位置）と z->p（文字列バッファ）の状態を追跡することが推奨される。
