BumpContext

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/mmgr/bump.c:66

目的

PostgreSQL のメモリ管理サブシステムにおける「バンプアロケータ」のメタデータを保持する構造体です。短命かつ大量のメモリ割り当てを高速に行うために使用されます。

シグネチャまたは形状

typedef struct BumpContext
{
    MemoryContextData header;
    uint32      initBlockSize;
    uint32      maxBlockSize;
    uint32      nextBlockSize;
    uint32      allocChunkLimit;
    dlist_head  blocks;
} BumpContext;

入力と出力

本構造体はメモリコンテキストの管理状態を保持するデータ構造であり、関数ではありません。BumpContextCreate 等の初期化関数を通じて生成され、BumpAlloc などのメモリ割り当て関数の引数として渡されます。

前提条件と不変条件

  • MemoryContextData を継承しており、標準的なメモリコンテキストのインターフェースに従います。
  • blocks フィールドは、現在使用中のブロックを先頭とする双方向リスト（dlist_head）を管理します。
  • このコンテキストで確保されたメモリに対して pfree を呼び出すことは禁止されており、コンテキスト全体のリセットまたは削除による一括解放のみが許可されます。

アルゴリズムまたは振る舞い

  • **バンプ割り当て**: チャンクヘッダーを排除し、連続したメモリブロック内のポインタをずらすことで高速な割り当てを実現します。
  • **ブロック管理**: 割り当て要求に応じて動的にメモリブロックを確保し、blocks リストに連結します。
  • **制限**: allocChunkLimit を超える巨大な割り当て要求は、専用のブロック確保ロジックへ委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**: BumpContextCreate（初期化時）、BumpAlloc（割り当て時）、BumpReset（解放時）などで参照されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • MemoryContextData: 基底となるメモリコンテキスト構造体。
  • BumpBlock: コンテキストが管理する個別のメモリブロック単位。
  • dlist_head: lib/ilist.h で定義される、ブロックを管理するための双方向リストヘッダー。

失敗モードと境界条件

  • **メモリ不足**: 新規ブロック確保時に malloc が失敗した場合、システムはエラーを発生させます。
  • **不正な操作**: 本コンテキストで確保したメモリに対して pfree を実行すると、システムは ERROR を送出します。

保守担当者向けメモ

  • MEMORY_CONTEXT_CHECKING が有効な場合、デバッグのためにチャンクヘッダーが付与されます。リリースビルドではこれが無効化され、オーバーヘッドが最小化されます。
  • 構造体内の blocks リストの先頭は、常に現在割り当てに使用されているブロックを指すように維持されます。
