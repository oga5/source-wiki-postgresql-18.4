ExecEndAgg

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAgg.c:4399

目的

集約実行ノード（AggState）の終了処理を行い、関連するリソースの解放と統計情報の集計を行う。

シグネチャまたは形状

void ExecEndAgg(AggState *node)

入力と出力

  • **入力**: AggState *node（終了対象の集約ノード状態）
  • **出力**: なし（副作用としてリソース解放と統計情報の更新を行う）

前提条件と不変条件

  • node は有効な AggState 構造体を指している必要がある。
  • 並列クエリ実行中の場合、node->shared_info が適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

1. **並列統計の保存**: 並列ワーカーとして実行されている場合、ハッシュ集約の統計情報（バッチ数、ディスク使用量、ピークメモリ使用量）を共有メモリへコピーする。 2. **ソートリソースの解放**: sort_in および sort_out が存在する場合、tuplesort_end を呼び出して終了させる。 3. **ハッシュ集約のクリーンアップ**: hashagg_reset_spill_state を呼び出し、スピル状態をリセットする。その後、hash_metacxt および hash_tablecxt メモリコンテキストを削除する。 4. **グループごとのソート状態解放**: 各集約状態（pertrans）およびグループセットごとに、保持されている tuplesort 状態を解放する。 5. **式コンテキストの再スキャン**: 集約コンテキストおよびハッシュコンテキストに対して ReScanExprContext を呼び出し、シャットダウンコールバックを実行する。 6. **子ノードの終了**: outerPlanState(node) を取得し、ExecEndNode を呼び出して下位プランノードの終了処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c から呼び出されることが確認されている。
  • **呼び出し先**: tuplesort_end, hashagg_reset_spill_state, MemoryContextDelete, ReScanExprContext, ExecEndNode 等を呼び出す。

関連データ構造

  • AggState: 集約ノードの実行状態を保持する主要構造体。
  • AggStatePerTrans: 集約の遷移状態を管理する構造体。
  • AggregateInstrumentation: 並列実行時の統計情報を保持する構造体。

失敗モードと境界条件

  • node が NULL の場合、あるいは node 内の各ポインタが初期化されていない場合、セグメンテーションフォールトが発生する可能性がある。
  • 並列ワーカー数と shared_info の不整合がある場合、Assert により停止する可能性がある。

保守担当者向けメモ

  • 本関数は ExecInitAgg で確保されたリソースの解放を網羅する必要がある。新しいリソース（メモリコンテキストやソート状態）を追加した場合は、必ず本関数での解放処理を確認すること。
  • 並列クエリの統計情報は ParallelWorkerNumber に依存しているため、並列実行環境での変更時は注意が必要である。
