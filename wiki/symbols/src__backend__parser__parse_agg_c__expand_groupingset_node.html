expand_groupingset_node

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_agg.c:1802

目的

GROUPING SETS, CUBE, ROLLUP 構文で指定された抽象的なグループ化ノードを、個別のグループ化リストの集合（フラットな List）へと展開・正規化する。

シグネチャまたは形状

List *expand_groupingset_node(GroupingSet *gs)

入力と出力

  • **入力**: GroupingSet *gs - 展開対象のグループ化セットノード。
  • **出力**: List * - 展開された各グループ化セット（List の List）を保持するリスト。

前提条件と不変条件

  • 入力される GroupingSet の各要素（gs->content 内のノード）は、再帰的に GROUPING_SET_SIMPLE である必要がある（Assert による検証あり）。
  • GROUPING_SET_CUBE の場合、要素数は 31 未満である必要がある。

アルゴリズムまたは振る舞い

gs->kind に基づき以下の処理を行う：

  • GROUPING_SET_EMPTY: 空のリストを返す。
  • GROUPING_SET_SIMPLE: 内容をそのままリストとして返す。
  • GROUPING_SET_ROLLUP: 指定されたリストの接頭辞を順次生成し、最後に空リストを追加する。
  • GROUPING_SET_CUBE: ビットマスクを用いて、指定された要素の全組み合わせ（べき集合）を生成する。
  • GROUPING_SET_SETS: 再帰的に各要素を呼び出し、結果を結合する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_agg.c 内の関数（再帰呼び出しを含む）。
  • **呼び出し先**: 自身の再帰呼び出し、および list_make1, list_concat, lappend などのリスト操作関数。

関連データ構造

  • GroupingSet: 構文解析ツリー上のグループ化ノード。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • GROUPING_SET_CUBE において要素数が 31 以上の場合、Assert により停止する。
  • 非常に多くの組み合わせが生成される可能性があるが、パーサー側で制限が設けられている。

保守担当者向けメモ

  • 本関数は GROUPING SETS の展開ロジックの中核です。sublevels_up 等のコンテキスト管理は行わず、純粋に構文的な展開を行います。
  • ROLLUP や CUBE の展開ロジックを変更する際は、生成されるリストの順序が後続の処理（GROUP BY の正規化）に影響を与えないか確認してください。
