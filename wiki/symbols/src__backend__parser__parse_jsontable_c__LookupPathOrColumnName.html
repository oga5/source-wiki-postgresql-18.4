LookupPathOrColumnName

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_jsontable.c:213

目的

JsonTableParseContext 内の pathNames リストを走査し、指定された名前が既に存在するかどうかを判定する。

シグネチャまたは形状

static bool LookupPathOrColumnName(JsonTableParseContext *cxt, char *name)

入力と出力

  • **入力**:
  • cxt: 解析コンテキスト (JsonTableParseContext*)。pathNames リストを保持する。
  • name: 検索対象の文字列 (char*)。
  • **出力**:
  • bool: 名前がリスト内に存在すれば true、存在しなければ false を返す。

前提条件と不変条件

  • cxt は有効なポインタであること。
  • cxt->pathNames は List 構造体であり、各要素は文字列 (char*) であること。

アルゴリズムまたは振る舞い

1. cxt->pathNames リストを foreach マクロで反復処理する。 2. 各要素に対して strcmp を実行し、引数 name と一致するか比較する。 3. 一致する要素が見つかった時点で直ちに true を返す。 4. リストの末尾まで一致が見つからなかった場合、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_jsontable.c:184 (カラム定義のパス名チェック)
  • src/backend/parser/parse_jsontable.c:198 (カラム名チェック)
  • **呼び出し先**:
  • strcmp (標準Cライブラリ)

関連データ構造

  • JsonTableParseContext: pathNames リストを保持する解析用コンテキスト構造体。

失敗モードと境界条件

  • cxt->pathNames が空リストの場合、常に false を返す。
  • name が NULL の場合の挙動は strcmp の仕様に依存する（本関数内での明示的なチェックはない）。

保守担当者向けメモ

  • 本関数は名前の重複チェックに使用される。リストの線形探索を行うため、カラム数が多い場合には計算量に注意が必要だが、通常 JSON_TABLE のカラム定義数であれば許容範囲内である。
