xl_btree_delete

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtxlog.h:239

目的

B-treeインデックスページからのタプル削除操作をWAL（Write-Ahead Log）に記録するための構造体です。

シグネチャまたは形状

typedef struct xl_btree_delete
{
    TransactionId snapshotConflictHorizon;
    uint16      ndeleted;
    uint16      nupdated;
    bool        isCatalogRel;
} xl_btree_delete;

入力と出力

  • **入力**: 削除対象のオフセット番号、更新対象のオフセット番号、および更新タプルのメタデータ（xl_btree_update 形式）が、本構造体に続くブロック0のペイロードとして格納されます。
  • **出力**: WALレコードの一部としてディスクへ書き出され、リカバリ時にインデックスページの再構築に使用されます。

前提条件と不変条件

  • 本構造体は、xl_btree_vacuum と類似した構造を持ちますが、削除対象のオフセット数と更新対象のオフセット数を個別に管理する点で異なります。
  • isCatalogRel は、スタンバイ側での論理デコーディング中にリカバリ競合を適切に処理するために使用されます。

アルゴリズムまたは振る舞い

1. インデックス操作時に、削除および更新されたタプルの情報を本構造体に設定します。 2. 続いて、削除対象および更新対象のオフセット番号、更新タプルのメタデータをWALペイロードとして記録します。 3. クラッシュリカバリ時、btree_redo がこのレコードを読み込み、対応するインデックスページに対して削除および更新操作を再現します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtpage.c 等のB-tree操作ルーチン。
  • **呼び出し先**: src/backend/access/nbtree/nbtxlog.c（REDO処理）、src/backend/access/rmgrdesc/nbtdesc.c（ログ解析・デバッグ出力）。

関連データ構造

  • xl_btree_update: 更新タプルのメタデータとしてペイロード内で使用されます。
  • xl_btree_vacuum: 類似した目的を持つWALレコード構造体。

失敗モードと境界条件

  • ページ内のオフセット番号の整合性が取れていない場合、REDO処理時にインデックスの破損を招く可能性があります。
  • SizeOfBtreeDelete マクロにより、構造体のサイズが定義されています。

保守担当者向けメモ

  • xl_btree_vacuum との差異を明確に理解すること。本構造体は削除と更新の両方を扱うため、ペイロードの解析時には ndeleted と nupdated の値に基づいたオフセットの読み取り順序に注意してください。
