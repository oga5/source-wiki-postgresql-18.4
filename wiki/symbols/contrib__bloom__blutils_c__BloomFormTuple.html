BloomFormTuple

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blutils.c:297

目的

インデックス挿入時に、ヒープタプルに対応する BloomTuple を生成し、各属性値に基づいて Bloom フィルタの署名ビット列を構築する。

シグネチャまたは形状

BloomTuple *BloomFormTuple(BloomState *state, ItemPointer iptr, Datum *values, bool *isnull)

入力と出力

  • **入力**:
  • state: インデックスの実行時状態（BloomState）。
  • iptr: ヒープタプルを指す ItemPointer。
  • values: インデックス対象となる属性値の配列（Datum）。
  • isnull: 各属性が NULL かどうかを示すブール値配列。
  • **出力**:
  • 構築された BloomTuple 構造体へのポインタ。メモリは palloc0 により確保される。

前提条件と不変条件

  • state->sizeOfBloomTuple に基づいてメモリが確保される。
  • isnull 配列が state->nColumns 分の要素を持つこと。
  • signValue 関数が内部で呼び出され、res->sign にビットが反映される。

アルゴリズムまたは振る舞い

1. state->sizeOfBloomTuple サイズのメモリをゼロ初期化して確保する。 2. 引数 iptr を res->heapPtr にコピーする。 3. state->nColumns 分のループを実行する。

  • isnull[i] が真の場合はスキップする。
  • 偽の場合は signValue を呼び出し、values[i] をハッシュ化して res->sign にビットをセットする。

    • 4. 完成した BloomTuple を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blinsert.c:84
  • contrib/bloom/blinsert.c:201
  • **呼び出し先**:
  • signValue (同ファイル内)

関連データ構造

  • BloomTuple: インデックス内に格納されるタプル構造。
  • BloomState: インデックスの構成情報（列数や署名サイズなど）を保持。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • isnull 配列の範囲外アクセスについては、呼び出し側の整合性に依存する。

保守担当者向けメモ

  • 本関数は blinsert.c からのインデックス構築プロセスにおいて中心的な役割を果たす。
  • signValue によるビット操作は BloomSignatureWord 単位で行われるため、署名長やビット計算の変更時には bloom.h の定義と整合性を確認すること。
