build_client_final_message

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:454

目的

SCRAM-SHA-256 認証プロトコルにおける「クライアント最終メッセージ（client-final-message）」を構築します。これにはチャネルバインディング情報、nonce、および HMAC によるクライアント証明が含まれます。

シグネチャまたは形状

static char *build_client_final_message(fe_scram_state *state);

入力と出力

  • **入力**: fe_scram_state *state（認証状態を保持する構造体）
  • **出力**: 構築されたメッセージ文字列（char *）。失敗した場合は NULL を返し、state->conn->errorMessage にエラー詳細が設定されます。

前提条件と不変条件

  • state が有効であり、state->conn が初期化されていること。
  • state->sasl_mechanism が設定されていること。
  • チャネルバインディングを使用する場合、SSL が有効（USE_SSL）であること。
  • 構築されたメッセージの一部（証明を除く部分）は state->client_final_message_without_proof に保持されます。

アルゴリズムまたは振る舞い

1. PQExpBuffer を初期化し、チャネルバインディング属性 c= を構築します。

  • SCRAM-SHA-256-PLUS の場合、SSL 証明書のハッシュを取得し、p=tls-server-end-point 形式でエンコードします。
  • それ以外の場合、y,, または n,, をベース64エンコードして付与します。

    • 2. nonce (r=) をメッセージに追加します。 3. 構築した文字列を state->client_final_message_without_proof に保存します。 4. calculate_client_proof を呼び出し、HMAC を計算します。 5. 計算された証明を p= 属性としてメッセージの末尾に付与します。 6. 最終的な文字列を strdup して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-scram.c 内のステートマシン（scram_exchange）から呼び出されます。
  • **呼び出し先**:
  • pgtls_get_peer_certificate_hash (SSL 有効時)
  • calculate_client_proof
  • libpq_append_conn_error

関連データ構造

  • fe_scram_state: 認証セッションの状態管理。
  • PQExpBufferData: メッセージ構築用バッファ。

失敗モードと境界条件

  • **メモリ不足**: malloc や strdup が失敗した場合、oom_error ラベルへ遷移し NULL を返します。
  • **SSL 関連エラー**: チャネルバインディングが必要な環境で SSL が無効、または証明書ハッシュの取得に失敗した場合に NULL を返します。
  • **証明計算失敗**: calculate_client_proof が失敗した場合、エラーを記録して NULL を返します。

保守担当者向けメモ

  • state->client_final_message_without_proof は、後続のサーバー署名検証のために保持される必要があります。
  • チャネルバインディングのフラグ（p, y, n）は、build_client_first_message との一貫性が必須です。
  • input 文字列の破壊的変更は行いませんが、PQExpBuffer のメモリ管理に注意してください。
