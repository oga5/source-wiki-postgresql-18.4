PGTYPESchar_free

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/common.c:144

目的

Windows環境において、DLL境界を跨いだメモリ解放を安全に行うためのインターフェースを提供します。ライブラリとアプリケーションが異なるランタイムを使用している場合に発生するメモリ解放エラーを回避します。

シグネチャまたは形状

void PGTYPESchar_free(char *ptr)

入力と出力

  • 入力: char *ptr （解放対象のメモリ領域へのポインタ）
  • 出力: なし（void）

前提条件と不変条件

  • pgtypeslib ライブラリによって確保されたメモリ領域を解放するために使用する必要があります。
  • free() のラッパーであるため、ptr は malloc 等で確保された有効なポインタ、または NULL である必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取ったポインタ ptr に対して、標準ライブラリの free(ptr) を実行します。

呼び出し元と呼び出し先

  • 呼び出し先: free (標準ライブラリ)
  • 呼び出し元: src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c 等のテストコードから呼び出されています。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • ptr が NULL の場合、標準の free の仕様に従い、何も行われません。
  • 無効なポインタを渡した場合の挙動は、標準の free に依存します。

保守担当者向けメモ

  • 本関数は pgtypeslib のメモリ管理の整合性を保つために不可欠です。メモリ解放が必要な際は、直接 free を呼ばず、必ずこの関数を経由してください。
