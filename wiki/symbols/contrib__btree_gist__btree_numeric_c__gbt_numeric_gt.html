gbt_numeric_gt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:28

目的

PostgreSQL の numeric 型に対する「より大きい（greater than）」比較演算を GiST インデックスのコンテキストで提供します。

シグネチャまたは形状

static bool gbt_numeric_gt(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の numeric 型データへのポインタ（void *）。
  • collation: 照合順序（本関数では使用されません）。
  • flinfo: 関数呼び出し情報（本関数では使用されません）。
  • **出力**:
  • bool: a > b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な numeric 型のデータ構造を指している必要があります。
  • 内部的に DirectFunctionCall2 を使用するため、PostgreSQL の関数呼び出し規約に従う必要があります。

アルゴリズムまたは振る舞い

1. a と b を PointerGetDatum を用いて Datum 型に変換します。 2. PostgreSQL コアの numeric_gt 関数を DirectFunctionCall2 経由で呼び出し、比較を実行します。 3. 比較結果を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_numeric.c 内の静的データ構造（tinfo 構造体の初期化時など）から参照されています。
  • **呼び出し先**: PostgreSQL コアの numeric_gt 関数。

関連データ構造

  • tinfo: gbtree_vinfo 構造体。本関数を含む比較関数群を保持し、btree_gist の汎用関数へ提供します。

失敗モードと境界条件

  • numeric_gt は numeric 型の比較ルールに従います。NaN（Not a Number）の比較については、コアの numeric_gt 実装に依存します。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける numeric 型用のアダプター関数です。ロジックの変更が必要な場合は、コアの numeric_gt の挙動を確認してください。
