IsTidEqualClause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/tidpath.c:130

目的

与えられた RestrictInfo が、TID（タプル識別子）に対する等価比較演算（ctid = ...）であるかどうかを判定する。

シグネチャまたは形状

static bool IsTidEqualClause(RestrictInfo *rinfo, RelOptInfo *rel)

入力と出力

  • **入力**:
  • rinfo: 評価対象の RestrictInfo 構造体。
  • rel: 対象となるリレーションの RelOptInfo 構造体。
  • **出力**:
  • bool: 条件がTIDに対する等価比較であれば true、そうでなければ false を返す。

前提条件と不変条件

  • rinfo は有効な RestrictInfo 構造体である必要がある。
  • IsBinaryTidClause が内部で呼び出され、対象の式がTIDに関連する二項演算子であることを前提とする。

アルゴリズムまたは振る舞い

1. IsBinaryTidClause(rinfo, rel) を呼び出し、対象がTIDに関する二項演算子でない場合は直ちに false を返す。 2. rinfo->clause を OpExpr にキャストし、その演算子（opno）が TIDEqualOperator と一致するかを確認する。 3. 一致すれば true を、一致しなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/tidpath.c 内の複数箇所（行254、行452など）で、TIDスキャンパス生成の条件判定に使用されている。
  • **呼び出し先**:
  • IsBinaryTidClause（直接の依存）

関連データ構造

  • RestrictInfo: クエリ内の制約条件を保持する構造体。
  • OpExpr: 演算子式を表すノード。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。
  • TIDEqualOperator: TIDの等価比較に使用される演算子ID。

失敗モードと境界条件

  • rinfo->clause が OpExpr ではない場合、キャストにより予期せぬ動作を引き起こす可能性がある（IsBinaryTidClause の判定に依存）。
  • rel が NULL の場合の挙動については、ソースコード上では明示的なガードは確認できない。

保守担当者向けメモ

  • 本関数は tidpath.c におけるTIDスキャンパス生成のフィルタリングロジックの基盤である。
  • 演算子の判定には TIDEqualOperator を使用しているため、TIDの比較演算子が変更・拡張された場合は本関数のロジックも再検討が必要である。
