GinScanKey

定義

  • 種別: typedef (ポインタ型)
  • 定義箇所: src/include/access/gin_private.h:265
  • 実体: struct GinScanKeyData へのポインタ

目的

GIN インデックススキャンにおけるクエリ条件の管理および評価を行うための抽象的な参照を提供します。

シグネチャまたは形状

typedef struct GinScanKeyData *GinScanKey;

入力と出力

  • 入力: GIN インデックススキャン実行時のクエリ条件。
  • 出力: consistentFn 等の論理関数を通じて、インデックスエントリがクエリ条件を満たすかどうかの判定結果を導出するために使用されます。

前提条件と不変条件

  • GinScanKeyData 構造体を指す有効なポインタである必要があります。
  • スキャン実行中、GinScanOpaque 構造体内の keys 配列の一部として管理されることが一般的です。

アルゴリズムまたは振る舞い

  • GIN インデックスの検索処理において、クエリ条件の評価（Consistent 関数）の引数として渡されます。
  • ginget.c 内の各処理において、スキャン対象のキーを特定し、インデックスエントリとの照合を行うためのインターフェースとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginget.c: スキャンループ内でのキーの取得および startScanKey への引き渡し。
  • src/backend/access/gin/ginlogic.c: trueConsistentFn, trueTriConsistentFn, directBoolConsistentFn, directTriConsistentFn などの論理関数。
  • **呼び出し先**:
  • 本シンボル自体はポインタ型であるため、直接の呼び出し先はありません。指し示す先の GinScanKeyData が保持する関数ポインタ（consistentFn 等）が実行時に呼び出されます。

関連データ構造

  • GinScanKeyData: 本型が指す実体。
  • GinScanOpaque: スキャン状態を保持し、GinScanKey の配列を管理する構造体。
  • GinState: インデックス操作のコンテキスト。

失敗モードと境界条件

  • NULL ポインタの参照: スキャンが初期化されていない状態でアクセスした場合に発生します。
  • 範囲外アクセス: GinScanOpaque が管理する keys 配列の境界を超えてインデックス操作を行った場合に発生します。
