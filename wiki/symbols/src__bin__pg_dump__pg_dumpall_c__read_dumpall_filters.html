read_dumpall_filters

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dumpall.c:1839

目的

指定されたファイルからデータベース識別子のパターンを読み込み、ダンプ対象のフィルタリング設定を構築する。

シグネチャまたは形状

static void read_dumpall_filters(const char *filename, SimpleStringList *pattern)

入力と出力

  • **入力**:
  • filename: フィルタ定義が記述されたファイルパス。
  • pattern: 読み込んだパターンを格納する SimpleStringList 構造体へのポインタ。
  • **出力**: なし（pattern を直接更新する）。

前提条件と不変条件

  • filter_init を介してフィルタ状態が初期化されること。
  • フィルタファイル内のコマンドタイプが FILTER_COMMAND_TYPE_INCLUDE である場合、エラーとして処理を終了する。

アルゴリズムまたは振る舞い

1. filter_init を呼び出し、指定ファイルからフィルタ状態を初期化する。 2. filter_read_item を使用して、ファイル内の各項目をループ処理する。 3. 項目が FILTER_COMMAND_TYPE_INCLUDE の場合はエラーを出力し、exit_nicely(1) で終了する。 4. 項目が FILTER_OBJECT_TYPE_DATABASE の場合、その名前を pattern リストに追加する。 5. その他のオブジェクトタイプ（関数、インデックス、テーブル等）が指定された場合は、サポート外としてエラーを出力し終了する。 6. 最後に filter_free を呼び出し、リソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dumpall.c 内の main 関数（optarg を引数として呼び出される）。
  • **呼び出し先**: filter_init, filter_read_item, pg_log_filter_error, exit_nicely, filter_object_type_name, simple_string_list_append, free, filter_free。

関連データ構造

  • FilterStateData: フィルタ読み込みの状態を保持する構造体。
  • SimpleStringList: データベース名のリストを保持する構造体。
  • FilterCommandType, FilterObjectType: フィルタのコマンドおよびオブジェクト種別を定義する列挙型。

失敗モードと境界条件

  • 指定されたファイルが読み込めない場合や、フィルタ構文が不正な場合は filter_read_item の挙動に依存する。
  • FILTER_COMMAND_TYPE_INCLUDE が指定された場合、またはサポート外のオブジェクトタイプが指定された場合は、エラーメッセージを表示してプロセスが終了する。

保守担当者向けメモ

  • 本関数は pg_dumpall におけるデータベース除外フィルタの読み込みに特化している。
  • 将来的に他のオブジェクトタイプをフィルタリング対象に追加する場合は、switch 文の case 節を拡張する必要がある。
