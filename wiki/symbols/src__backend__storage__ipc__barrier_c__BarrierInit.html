BarrierInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/barrier.c:99

目的

並列処理を行うプロセス間で同期をとるための Barrier 構造体を初期化し、同期状態を初期値にリセットします。

シグネチャまたは形状

void BarrierInit(Barrier *barrier, int participants)

入力と出力

  • **入力**:
  • barrier: 初期化対象となる Barrier 構造体へのポインタ。
  • participants: バリアに参加するプロセス数。0を指定すると動的バリアモードとして動作します。
  • **出力**: なし（barrier が指すメモリ領域を更新）。

前提条件と不変条件

  • barrier ポインタは有効なメモリ領域を指している必要があります。
  • 初期化後、barrier->arrived は 0、barrier->phase は 0、barrier->elected は 0 に設定されます。
  • participants > 0 の場合、static_party フラグが真（静的バリア）となります。

アルゴリズムまたは振る舞い

1. SpinLockInit を呼び出し、バリア内部の mutex を初期化します。 2. 指定された participants を構造体に格納します。 3. 到着数 (arrived)、フェーズ (phase)、選出フラグ (elected) を 0 にリセットします。 4. participants が 0 より大きいかどうかに基づき、static_party を設定します。 5. ConditionVariableInit を呼び出し、待機に使用する条件変数を初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHash.c
  • src/backend/executor/nodeHashjoin.c
  • **呼び出し先**:
  • SpinLockInit
  • ConditionVariableInit

関連データ構造

  • Barrier: 同期状態（スピンロック、参加者数、到達数、フェーズ、条件変数）を保持する構造体。

失敗モードと境界条件

  • participants に 0 を指定した場合、動的バリアとして動作し、実行中の参加・離脱が可能なモードへ移行します。

保守担当者向けメモ

  • 本関数はバリアの再初期化にも使用可能ですが、その際は他のプロセスが当該バリアを使用していないことを保証する必要があります。
  • static_party の値は、バリアの動作モード（静的か動的か）を決定する重要なフラグです。
