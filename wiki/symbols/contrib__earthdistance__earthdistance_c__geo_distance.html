geo_distance

定義

  • 種別: function
  • 定義箇所: contrib/earthdistance/earthdistance.c:99

目的

SQLから呼び出し可能なPostgreSQL関数として、地球上の2点間の大圏距離をマイル単位で算出するインターフェースを提供します。

シグネチャまたは形状

Datum geo_distance(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINT_P マクロを介して取得される2つの Point 構造体（pt1, pt2）。各 Point の x は経度、y は緯度として扱われます。
  • **出力**: float8 型の距離値。PG_RETURN_FLOAT8 マクロを介して返却されます。

前提条件と不変条件

  • 入力される Point 型の座標は度（degrees）単位である必要があります。
  • 地球を完全な球体と仮定して計算されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINT_P を使用して、引数から2つの Point 構造体を取得します。 2. geo_distance_internal 関数を呼び出し、球面三角法を用いた距離計算を実行します。 3. 計算結果を PG_RETURN_FLOAT8 を介してSQL層へ返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: geo_distance_internal
  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1(geo_distance) により登録）。具体的なSQLクエリ以外の呼び出し元情報は提供されていません。

関連データ構造

  • Point: utils/geo_decls.h で定義される、座標（x, y）を保持する構造体。

失敗モードと境界条件

  • 入力された Point が無効な場合、PostgreSQLの関数呼び出し規約に従いエラーが発生する可能性があります。
  • 距離計算の精度は math.h の浮動小数点演算に依存します。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1 マクロによってPostgreSQLの関数として公開されています。
  • 距離の単位は「マイル」です。キロメートル単位への変更が必要な場合は、計算ロジックの修正が必要です。
  • 内部計算は geo_distance_internal に委譲されているため、アルゴリズムの変更はそちらを確認してください。
