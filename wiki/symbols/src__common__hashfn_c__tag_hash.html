tag_hash

定義

  • 種別: function
  • 定義箇所: src/common/hashfn.c:676

目的

固定サイズのタグ値（データ構造のキーなど）に対して、ハッシュテーブルで使用するためのハッシュ値を生成します。

シグネチャまたは形状

uint32 tag_hash(const void *key, Size keysize)

入力と出力

  • **入力**:
  • key: ハッシュ化対象のデータへのポインタ。
  • keysize: 対象データのサイズ（バイト単位）。
  • **出力**:
  • uint32: 計算されたハッシュ値。

前提条件と不変条件

  • hash_bytes を内部で呼び出すため、同関数の制約（アライメントやメモリ境界の考慮）に従います。
  • elog(ERROR) を投げない設計となっており、クリティカルなパスでも安全に使用可能です。

アルゴリズムまたは振る舞い

1. 受け取った key ポインタを unsigned char * にキャストします。 2. hash_bytes 関数を呼び出し、バイト列からハッシュ値を算出します。 3. 計算結果をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: hash_bytes
  • **呼び出し元（直接的な証拠）**:
  • src/backend/utils/hash/dynahash.c: ハッシュテーブルのキー計算関数として設定されます。
  • src/backend/lib/dshash.c: ハッシュ計算の転送先として使用されます。
  • contrib/pg_stat_statements/pg_stat_statements.c: HASH_BLOBS を使用したハッシュ生成に利用されます。

関連データ構造

  • dynahash: PostgreSQL のハッシュテーブル実装。本関数はキー計算の標準的な選択肢の一つです。

失敗モードと境界条件

  • hash_bytes に依存するため、入力データのメモリ配置やサイズが不正な場合、ハッシュ計算結果が期待通りにならない可能性があります。
  • 非常に小さなキーサイズに対しても動作しますが、より特化した関数（uint32_hash 等）と比較してオーバーヘッドが生じる場合があります。

保守担当者向けメモ

  • 本関数は hash_bytes のラッパーです。ハッシュアルゴリズム自体の変更が必要な場合は hash_bytes を参照してください。
  • dynahash 等で利用される標準的なハッシュ関数であるため、変更時はハッシュの分布特性（アバランチ効果）に十分注意してください。
