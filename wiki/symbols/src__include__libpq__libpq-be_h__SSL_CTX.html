SSL_CTX

定義

  • 種別: typedef（OpenSSL ライブラリの構造体へのポインタ型）
  • 定義箇所: src/include/libpq/libpq-be.h:335

目的

PostgreSQL のバックエンドおよびフロントエンドにおける SSL/TLS 通信のセッション設定、証明書、および暗号化パラメータを保持するコンテキスト構造体です。

シグネチャまたは形状

OpenSSL の ssl.h で定義される SSL_CTX 構造体へのポインタとして使用されます。

入力と出力

  • 入力: openssl_tls_init_hook_typ などの関数において、初期化対象のコンテキストとして渡されます。
  • 出力: initialize_dh や initialize_ecdh などの関数により、暗号化アルゴリズムの設定が適用されます。

前提条件と不変条件

  • USE_OPENSSL が定義されている環境で有効です。
  • SSL/TLS 通信の初期化プロセスにおいて、有効な SSL_CTX オブジェクトが生成されている必要があります。

アルゴリズムまたは振る舞い

  • SSL/TLS 接続の初期化、DH/ECDH パラメータの設定、および TLS 初期化フックの実行に使用されます。
  • 接続ごとに個別のコンテキストを持つ場合と、共有される場合があります。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/libpq/be-secure-openssl.c: default_openssl_tls_init, initialize_dh, initialize_ecdh 等。
  • src/interfaces/libpq/fe-secure-openssl.c: クライアント側の SSL コンテキスト管理。
  • src/test/modules/ssl_passphrase_callback/ssl_passphrase_func.c: テスト用コールバック関数。
  • 呼び出し先: OpenSSL ライブラリの関数群（ソースコード上での直接的な呼び出し先詳細は本ヘッダーファイルには含まれません）。

関連データ構造

  • openssl_tls_init_hook_typ: SSL_CTX を引数に取る関数ポインタ型。

失敗モードと境界条件

  • SSL_context が NULL の状態で初期化関数が呼び出された場合、適切なエラーハンドリングが必要です。
  • OpenSSL ライブラリ側の制約により、証明書や鍵ファイルの読み込み失敗時にコンテキストの構築が失敗する可能性があります。

保守担当者向けメモ

  • 本シンボルは OpenSSL の外部ライブラリ定義に依存しています。
  • src/backend/libpq/be-secure-openssl.c 内の SSL_context 静的変数との関連性に注意してください。
  • 構造体の内部メンバへの直接アクセスは避け、OpenSSL の API を介して操作してください。
