add_bool_reloption

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/reloptions.c:859

目的

システム全体で利用可能な新しいブール型リレーションオプションを定義し、登録します。

シグネチャまたは形状

void add_bool_reloption(bits32 kinds, const char *name, const char *desc,
                        bool default_val, LOCKMODE lockmode)

入力と出力

  • kinds: オプションが適用されるリレーションの種類（ビットマスク）。
  • name: オプション名。
  • desc: オプションの説明文。
  • default_val: オプションのデフォルト値。
  • lockmode: オプション変更時に要求されるロックレベル。
  • 戻り値: なし。

前提条件と不変条件

  • init_bool_reloption が正常にメモリを確保し、初期化できる必要があります。
  • 登録されるオプション名は、既存のオプションと重複しないことが期待されます。

アルゴリズムまたは振る舞い

1. init_bool_reloption を呼び出し、指定された引数に基づいて relopt_bool 構造体を生成・初期化します。 2. 生成された構造体を relopt_gen 型にキャストし、add_reloption を呼び出してグローバルなオプションリストへ登録します。

呼び出し元と呼び出し先

  • 呼び出し先: init_bool_reloption, add_reloption
  • 呼び出し元:
  • src/test/modules/dummy_index_am/dummy_index_am.c での利用が確認されています。
  • その他、PostgreSQL コア内のリレーションオプション初期化処理から呼び出されます。

関連データ構造

  • relopt_bool: ブール型オプションのメタデータを保持する構造体。
  • relopt_gen: 全オプションの基底構造体。
  • LOCKMODE: オプション変更時の排他制御レベル。

失敗モードと境界条件

  • メモリ不足時には init_bool_reloption 内部でエラーが発生し、処理が中断される可能性があります。

保守担当者向けメモ

  • 新しいオプションを追加する際は、適切な kinds を指定し、既存のオプション名と競合しないように注意してください。
  • lockmode の選択は、そのオプションが実行計画やデータ整合性に与える影響を考慮して決定する必要があります。
