plpgsql_compile_inline

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:738

目的

DO ステートメントなどで実行される匿名コードブロックのソーステキストを解析し、実行可能な PLpgSQL_function 構造体（実行ツリー）を構築する。

シグネチャまたは形状

PLpgSQL_function *plpgsql_compile_inline(char *proc_source)

入力と出力

  • **入力**: proc_source (コンパイル対象の PL/pgSQL ソースコード文字列)
  • **出力**: 構築された PLpgSQL_function 構造体へのポインタ。

前提条件と不変条件

  • コンパイル中のメモリは専用の func_cxt で管理され、成功時にのみ永続化される。
  • 匿名ブロックは VOID 型を返す関数として扱われる。
  • コンパイル処理は再入不可である。

アルゴリズムまたは振る舞い

1. **初期化**: スキャナを初期化し、エラー報告用のコンテキストスタックを設定する。 2. **メモリ管理**: AllocSetContextCreate により、コンパイル専用のメモリコンテキストを作成・切り替えする。 3. **構造体設定**: PLpgSQL_function を割り当て、匿名ブロック用のデフォルト値（fn_rettype = VOIDOID 等）で初期化する。 4. **名前空間とデータ**: plpgsql_ns_init で名前空間を初期化し、found 変数を生成する。 5. **解析**: plpgsql_yyparse を呼び出し、ソーステキストからステートメントツリーを構築する。 6. **完了処理**: VOID 型の戻り値に対するダミーの RETURN を追加し、データムの確定、例外ブロックのマーク付けを行う。 7. **後始末**: エラーコンテキストを復元し、メモリコンテキストを元の状態に戻して構築済みの関数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_handler.c (匿名ブロック実行時)
  • **呼び出し先**: plpgsql_scanner_init, plpgsql_ns_init, plpgsql_yyparse, plpgsql_build_variable, add_dummy_return, plpgsql_finish_datums 等の内部関数。

関連データ構造

  • PLpgSQL_function: コンパイル結果を保持するメイン構造体。
  • PLpgSQL_variable: found 変数などの保持に使用。
  • MemoryContext: コンパイル中のメモリ割り当て管理に使用。

失敗モードと境界条件

  • plpgsql_yyparse が 0 以外の値を返した場合、elog(ERROR) によりコンパイルエラーとして処理を中断する。
  • コンパイル失敗時は、専用メモリコンテキストの破棄によりメモリリークが抑制される。

保守担当者向けメモ

  • 本関数は DO ブロック専用であり、通常の関数コンパイルとは一部の初期化ロジックが異なる。
  • コンパイル中のメモリは plpgsql_compile_tmp_cxt を介して管理されているため、メモリ管理の変更時には注意が必要である。
