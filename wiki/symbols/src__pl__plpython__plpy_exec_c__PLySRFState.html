PLySRFState

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_exec.c:25

目的

PostgreSQL の集合を返す関数（SRF）として実行される PL/Python 関数のイテレータ状態を保持し、実行コンテキストを管理します。

シグネチャまたは形状

typedef struct PLySRFState
{
    PyObject   *iter;           /* Python iterator producing results */
    PLySavedArgs *savedargs;    /* function argument values */
    MemoryContextCallback callback; /* for releasing refcounts when done */
} PLySRFState;

入力と出力

  • **入力**: PLy_exec_function 内で初期化され、funcctx->user_fctx を通じて保持されます。
  • **出力**: 実行終了時、MemoryContextCallback を介して関連する Python オブジェクトの参照カウントが解放されます。

前提条件と不変条件

  • SRF の実行中、iter は有効な Python イテレータを指している必要があります。
  • メモリコンテキストの削除時に callback が実行され、リソースのクリーンアップが保証される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: PLy_exec_function 内でメモリが割り当てられ、user_fctx に格納されます。 2. **反復処理**: 複数回の呼び出しを通じて iter から結果を順次取得します。 3. **終了処理**: メモリコンテキストの破棄に伴い、登録されたコールバック関数が呼び出され、保持している Python オブジェクトの参照カウントを適切にデクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_exec.c 内の PLy_exec_function が本構造体を生成・管理します。
  • **呼び出し先**: メモリコンテキスト削除時に、クリーンアップ用のコールバック関数（src/pl/plpython/plpy_exec.c:674 付近）が本構造体を引数として受け取ります。

関連データ構造

  • PyObject: Python 側のイテレータオブジェクト。
  • PLySavedArgs: 関数呼び出し時の引数状態を保持する構造体。
  • MemoryContextCallback: PostgreSQL のメモリコンテキスト破棄時に実行されるコールバック。

失敗モードと境界条件

  • iter が NULL の場合、あるいは Python 側のイテレータ生成に失敗した場合の挙動は、PLy_exec_function のエラーハンドリングに依存します。
  • メモリコンテキストの管理が不適切な場合、Python オブジェクトの参照カウントリークが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は PLy_exec_function における SRF のライフサイクル管理の要です。
  • PG_TRY/PG_CATCH ブロック内でのメモリ管理と、MemoryContextRegisterResetCallback によるクリーンアップの整合性を維持してください。
