pg_blocking_pids

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:465

目的

指定されたプロセスID（PID）が現在待機しているロックを保持している、あるいは待機順序において先行している他のプロセスIDのリストを返します。

シグネチャまたは形状

Datum pg_blocking_pids(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT32(0) により、対象となるPID（blocked_pid）を取得。
  • **出力**: int32 型の配列（ArrayType）。ブロックの原因となっているプロセスのPID群を格納。

前提条件と不変条件

  • GetBlockerStatusData(blocked_pid) を呼び出し、ロックマネージャから現在のロック状態のスナップショットを取得していること。
  • 戻り値の配列サイズは、スナップショット内のロック数（lockData->nlocks）を超えないこと。
  • 自身（blocked_pid）および同一ロックグループ内のプロセスは、ブロック対象から除外されること。

アルゴリズムまたは振る舞い

1. GetBlockerStatusData でロック状態を取得し、結果格納用の配列をメモリ確保します。 2. 対象PIDが保持するロックおよび待機中のロック情報を特定します。 3. ロックマネージャの競合テーブル（conflictTab）を参照し、以下のいずれかの場合に「ブロックしている」と判定します。

  • **ハードブロック**: 他のプロセスが既に保持しているロックと競合する場合。
  • **ソフトブロック**: 待機キューにおいて、他のプロセスが対象PIDよりも先行している場合。

    • 4. 判定されたPIDを配列に追加し、最後に construct_array_builtin を用いてSQL配列型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/waitfuncs.c:76（DirectFunctionCall1 を介した呼び出し）
  • **呼び出し先**:
  • GetBlockerStatusData（ロック状態の取得）
  • GetLockTagsMethodTable（ロックタグのメソッドテーブル取得）
  • construct_array_builtin（配列の構築）
  • palloc（メモリ確保）

関連データ構造

  • BlockedProcsData: ロックマネージャから取得したプロセスおよびロック状態の全体構造。
  • LockInstanceData: 個別のロックインスタンス情報。
  • BlockedProcData: 特定のプロセスに関連するロック情報。

失敗モードと境界条件

  • 対象PIDがどのロックも待機していない場合、空の配列を返します。
  • 内部的なロック状態の不整合が発生した場合、Assert により異常終了する可能性があります。

保守担当者向けメモ

  • 本関数はロックマネージャの状態をスナップショットとして取得するため、呼び出し時点での一時的な状態を反映します。
  • lockData->nlocks を超える配列アクセスが発生しないよう、narrayelems の管理には注意してください。
