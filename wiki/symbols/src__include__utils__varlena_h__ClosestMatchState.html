ClosestMatchState

定義

  • 種別: struct
  • 定義箇所: src/include/utils/varlena.h:41

目的

近似マッチング（スペルチェック等）の計算過程において、候補リストの中からソース文字列に最も近い文字列を特定するための状態を保持する。

シグネチャまたは形状

typedef struct ClosestMatchState
{
	const char *source;
	int			min_d;
	int			max_d;
	const char *match;
} ClosestMatchState;

入力と出力

  • **入力**: initClosestMatch を介してソース文字列と許容される最大編集距離（max_d）が設定される。その後、updateClosestMatch を通じて複数の候補文字列が逐次入力される。
  • **出力**: getClosestMatch を呼び出すことで、入力された候補の中で最小の編集距離を持つ文字列（match）が返される。

前提条件と不変条件

  • initClosestMatch による初期化が必須である。
  • min_d は、現在までに発見された最小の編集距離を保持する。
  • max_d は、マッチングを考慮する上限の編集距離を定義する。

アルゴリズムまたは振る舞い

1. **初期化**: initClosestMatch でソース文字列と許容距離をセットする。 2. **更新**: updateClosestMatch が候補文字列を受け取るたびに、ソース文字列との編集距離を計算する。計算された距離が現在の min_d より小さく、かつ max_d 以内であれば、min_d を更新し、その候補を match として保持する。 3. **取得**: getClosestMatch は、保持されている match を返す。

  • **呼び出し先**:

関連データ構造

  • varlena 構造体（PostgreSQLの可変長データ型）に関連するユーティリティ群。

失敗モードと境界条件

  • 候補文字列が一つも max_d 以内に収まらない場合、getClosestMatch は NULL を返す可能性がある（実装依存）。
  • 編集距離計算のコストは候補数に比例するため、大量の候補を渡す場合はパフォーマンスに影響する。
