sepgsql_getenforce

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/selinux.c:650

目的

SELinux の強制モード（Enforcing mode）が現在有効であるかどうかを判定します。

シグネチャまたは形状

bool sepgsql_getenforce(void)

入力と出力

  • **入力**: なし。
  • **出力**: bool 型。SELinux が強制モードであれば true、それ以外（無効、または Permissive モード等）であれば false を返します。

前提条件と不変条件

  • sepgsql_mode 変数が SEPGSQL_MODE_DEFAULT である場合にのみ、カーネル側の状態を確認します。
  • selinux_status_getenforce() は libselinux の関数であり、カーネルの現在の強制状態を返します。

アルゴリズムまたは振る舞い

1. 現在の動作モード sepgsql_mode が SEPGSQL_MODE_DEFAULT であるかを確認します。 2. SEPGSQL_MODE_DEFAULT かつ selinux_status_getenforce() が 0 より大きい（Enforcing モード）場合、true を返します。 3. 上記以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/dml.c:161
  • contrib/sepgsql/hooks.c:365
  • contrib/sepgsql/uavc.c:383
  • contrib/sepgsql/uavc.c:402
  • **呼び出し先**:
  • selinux_status_getenforce() (libselinux)

関連データ構造

  • sepgsql_mode: モジュールの動作モードを保持するグローバル変数。

失敗モードと境界条件

  • selinux_status_getenforce() がエラーを返す場合や、SELinux がシステム上で無効化されている場合の挙動は、libselinux の実装に依存します。本関数は、強制モードが明示的に有効でない限り false を返す安全な設計となっています。

保守担当者向けメモ

  • 本関数は sepgsql_mode の値に強く依存しています。モード設定の変更が正しく反映されるか注意してください。
  • uavc.c 内でキャッシュの有効性を判断する際にも使用されており、セキュリティ判定のパフォーマンスに直結する関数です。
