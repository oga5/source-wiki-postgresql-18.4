r_has_min_length

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_greek.c:2325

目的

ステミング処理において、対象となる文字列が最小限の長さ（3文字以上）を満たしているかを判定する。

シグネチャまたは形状

static int r_has_min_length(struct SN_env * z)

入力と出力

  • **入力**: SN_env 構造体へのポインタ z。内部の p メンバ（文字列データ）を参照する。
  • **出力**: int 型。条件を満たす場合は 1（真）、満たさない場合は 0（偽）を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • z->p には UTF-8 エンコードされた文字列が格納されていること。

アルゴリズムまたは振る舞い

1. z->p が指す文字列に対して len_utf8 関数を呼び出し、文字数をカウントする。 2. カウントされた文字数が 3 以上であるかを比較する。 3. 比較結果を整数値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_greek.c および src/backend/snowball/libstemmer/stem_UTF_8_tamil.c 内の各ステミングステップ関数から参照されている。
  • **呼び出し先**: len_utf8（Snowball ランタイムの補助関数）。

関連データ構造

  • struct SN_env: ステミング処理の状態管理を行う構造体。

失敗モードと境界条件

  • z が NULL の場合、または z->p が不正な UTF-8 文字列である場合の挙動は、len_utf8 の実装に依存する。
  • 文字数が 3 未満の場合は常に 0 を返す。
