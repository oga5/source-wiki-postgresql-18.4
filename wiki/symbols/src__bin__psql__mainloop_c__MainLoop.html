MainLoop

定義

  • 種別: function
  • 定義箇所: src/bin/psql/mainloop.c:32

目的

psql における対話的および非対話的な入力処理の中核を担い、指定されたストリームから SQL 文やバックスラッシュコマンドを読み取り、解析および実行するライフサイクルを管理します。

シグネチャまたは形状

int MainLoop(FILE *source)

入力と出力

  • **入力**: FILE *source（読み取り対象の入力ストリーム）
  • **出力**: int（終了ステータス。成功時は EXIT_SUCCESS、失敗時は EXIT_FAILURE または EXIT_USER）

前提条件と不変条件

  • pset 構造体が初期化されている必要があります。
  • MainLoop は再入可能であり、\i コマンド実行時には新しいコンテキストで呼び出されます。
  • 呼び出し時に pset.cur_cmd_source 等のグローバルな状態が保存され、終了時に復元されます。

アルゴリズムまたは振る舞い

1. **初期化**: 入力ソースを pset に設定し、字句解析器（psql_scan）および条件分岐スタック（cond_stack）を生成します。 2. **ループ処理**:

  • sigsetjmp を用いて Ctrl-C 割り込み時の復帰ポイントを設定します。
  • gets_interactive または gets_fromFile を介して入力を取得します。
  • 入力が NULL（EOF）の場合、対話モードであれば ignoreeof 設定に従い終了するか警告を表示します。
  • 入力行の先頭に UTF-8 BOM がある場合は除去します。
  • 取得した行を解析し、SQL 文であれば SendQuery を、バックスラッシュコマンドであれば HandleSlashCmds を呼び出して実行します。

    • 3. **終了処理**: EOF 到達時や終了コマンド時に、確保したバッファを解放し、保存していた元のコンテキストを復元して制御を戻します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/startup.c (メイン起動時)
  • src/bin/psql/command.c (\i 等のコマンド実行時)
  • **呼び出し先**:
  • psql_scan_create, conditional_stack_create
  • gets_interactive, gets_fromFile
  • SendQuery (直接の呼び出しは command.c 等を介する)
  • HandleSlashCmds (直接の呼び出しは command.c 等を介する)

関連データ構造

  • PsqlScanState: 字句解析の状態保持。
  • ConditionalStack: \if ブロックのネスト管理。
  • PQExpBuffer: クエリ蓄積用バッファ（query_buf, previous_buf, history_buf）。
  • pset: グローバルな設定構造体。

失敗モードと境界条件

  • **メモリ不足**: PQExpBuffer の生成失敗時に pg_fatal を呼び出します。
  • **Ctrl-C 割り込み**: sigsetjmp により捕捉され、解析状態とバッファをリセットします。対話モードでない場合は EXIT_USER で終了します。
  • **EOF**: count_eof が pset.ignoreeof に達するまで終了を抑制します。
  • **ファイル読み込みエラー**: ferror が真の場合、EXIT_FAILURE を返します。

保守担当者向けメモ

  • sigint_interrupt_jmp はループの各イテレーションで再設定されるため、割り込みハンドラとの整合性に注意してください。
  • query_buf と previous_buf はポインタ交換により効率的に管理されており、\e コマンド等で直前のクエリを参照する際に利用されます。
  • pset.stmt_lineno はループ内でリセットされますが、pset.lineno は入力行ごとにインクリメントされます。
