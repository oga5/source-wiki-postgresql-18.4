compress_flush

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-compress.c:138

目的

圧縮ストリームの終了処理を行い、zlib の内部バッファに残存するデータをすべてフラッシュして後続のフィルタへ書き出す。

シグネチャまたは形状

static int compress_flush(PushFilter *next, void *priv);

入力と出力

  • **入力**:
  • next: 書き出し先となる PushFilter オブジェクト。
  • priv: 圧縮状態を保持する ZipStat 構造体へのポインタ。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は負の pgcrypto エラーコード（PXE_PGP_COMPRESSION_ERROR または pushf_write の戻り値）を返す。

前提条件と不変条件

  • priv は ZipStat 構造体として初期化済みであること。
  • st->stream は deflateInit 等で適切に設定されていること。

アルゴリズムまたは振る舞い

1. st->stream の入力側をクリアし、deflate() を Z_FINISH モードで呼び出す。 2. zlib が出力したデータを st->buf から取得し、pushf_write() を介して next フィルタへ書き出す。 3. zlib が Z_STREAM_END を返すまで、上記の手順をループで繰り返す。 4. 処理中に zlib が Z_STREAM_END 以外のエラーを返した場合は、PXE_PGP_COMPRESSION_ERROR を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、pgp-compress.c 内のフィルタ管理構造体から参照される設計です。
  • **呼び出し先**:
  • deflate() (zlib ライブラリ)
  • pushf_write() (pgcrypto フィルタインターフェース)

関連データ構造

  • ZipStat: 圧縮状態（z_stream、バッファ、バッファ長）を保持する内部構造体。
  • PushFilter: フィルタパイプラインの抽象化構造体。

失敗モードと境界条件

  • deflate() が Z_STREAM_END および Z_OK 以外を返した場合、圧縮エラーとして終了する。
  • pushf_write() が負の値を返した場合、そのエラーコードをそのまま伝播させる。

保守担当者向けメモ

  • 本関数は zlib の Z_FINISH を使用してストリームの完全な終了を保証します。
  • zlib がインストールされていない環境では、この関数は PXE_PGP_UNSUPPORTED_COMPR を返すスタブとして機能します。
