LargeObjectCreate

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/catalog/pg_largeobject.c:35

目的

新規ラージオブジェクトのメタデータを pg_largeobject_metadata システムカタログに登録し、デフォルトのアクセス権限（ACL）を設定します。

シグネチャまたは形状

Oid LargeObjectCreate(Oid loid)

入力と出力

  • **入力**: loid (Oid) - 指定する場合、そのOIDで作成を試みる。InvalidOid の場合はシステムが新規OIDを割り当てる。
  • **出力**: 作成されたラージオブジェクトの Oid を返す。

前提条件と不変条件

  • 呼び出し元のトランザクション境界内で実行される。
  • pg_largeobject_metadata テーブルに対する RowExclusiveLock を取得して操作する。

アルゴリズムまたは振る舞い

1. pg_largeobject_metadata テーブルをオープンする。 2. loid が有効であればそれを使用し、無効であれば GetNewOidWithIndex を用いて一意なOIDを生成する。 3. 現在のユーザーIDを取得し、get_user_default_acl を呼び出してデフォルトのACLを取得する。 4. メタデータ（OID、所有者、ACL）を構築し、CatalogTupleInsert でテーブルに挿入する。 5. recordDependencyOnNewAcl を呼び出し、ACLに関連するロールへの依存関係を記録する。 6. テーブルをクローズし、生成したOIDを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/large_object/inv_api.c:180
  • **呼び出し先**:
  • table_open, table_close
  • GetNewOidWithIndex
  • GetUserId
  • get_user_default_acl
  • heap_form_tuple, heap_freetuple
  • CatalogTupleInsert
  • recordDependencyOnNewAcl

関連データ構造

  • pg_largeobject_metadata: メタデータが格納されるシステムカタログ。
  • Acl: ラージオブジェクトの権限情報。

失敗モードと境界条件

  • loid が既に存在する場合、インデックス制約によりエラーが発生する可能性がある。
  • メタデータテーブルへのアクセス権限がない場合、またはトランザクションが異常終了した場合は処理が中断される。

保守担当者向けメモ

  • 本関数はシステムキャッシュを使用せず、カタログへ直接アクセスするため、頻繁な呼び出しはカタログI/O負荷に影響する可能性がある。
  • 削除処理（LargeObjectDrop）との整合性を保つため、メタデータの挿入順序と依存関係の記録順序には注意が必要である。
