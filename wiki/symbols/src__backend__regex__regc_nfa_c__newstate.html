newstate

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:136

目的

正規表現コンパイル中のNFA（非決定性有限オートマトン）において、新しい状態（struct state）を割り当て、NFAの管理リストに登録する。

シグネチャまたは形状

struct state *newstate(struct nfa *nfa)

入力と出力

  • **入力**: nfa (NFA構造体へのポインタ)
  • **出力**: 新規作成された struct state へのポインタ。エラー発生時（メモリ不足やコンパイル空間制限超過）は NULL を返す。

前提条件と不変条件

  • nfa->v->re を通じた割り込みチェックが実行される。
  • nfa->nstates は常に 0 以上であること。
  • 状態の割り当ては、メモリ効率化のため statebatch を介したバッチ割り当て、または freestates リストからの再利用が行われる。

アルゴリズムまたは振る舞い

1. **割り込み確認**: INTERRUPT マクロにより、コンパイルの中断要求を確認する。 2. **メモリ確保**:

  • nfa->freestates に空きがあれば再利用する。
  • なければ nfa->lastsb（現在のバッチ）の空きを確認し、あれば使用する。
  • バッチも使い切っている場合、新しい statebatch を MALLOC で確保する。この際、REG_MAX_COMPILE_SPACE を超えていないかチェックする。

    • 3. **初期化**:

  • 状態番号（no）を割り当て、フラグをクリアする。
  • 遷移リスト（ins, outs）を空に初期化する。

    • 4. **リスト登録**: NFAの状態リストの末尾に新しい状態を追加し、nfa->slast を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_nfa.c 内の複数の箇所（newnfa、newfstate、およびグラフ構築・最適化処理）。
  • **呼び出し先**: MALLOC（メモリ確保）、NERR（エラー報告）。
  • *注: src/backend/commands/matview.c に同名の識別子が存在するが、本関数とは無関係である。*

関連データ構造

  • struct nfa: NFA全体を管理する構造体。
  • struct state: NFAの各状態を表現する構造体。
  • struct statebatch: メモリ効率化のための状態の塊。

失敗モードと境界条件

  • **メモリ不足**: MALLOC が失敗した場合、REG_ESPACE を記録し NULL を返す。
  • **コンパイル空間制限**: nfa->v->spaceused が REG_MAX_COMPILE_SPACE を超える場合、REG_ETOOBIG を記録し NULL を返す。

保守担当者向けメモ

  • 本関数は statebatch を用いてメモリを管理しているため、個別の free は行わない。NFA全体の解放は freenfa で一括して行う設計である。
  • 状態の追加はコンパイルの頻繁な操作であるため、割り込みチェックが挿入されている。
