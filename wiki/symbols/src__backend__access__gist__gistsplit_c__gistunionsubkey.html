gistunionsubkey

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistsplit.c:79

目的

GiSTインデックスのページ分割処理において、分割された左右の各ページに対して、全列のユニオンキー（境界値）を再計算し、インデックスの整合性を維持する。

シグネチャまたは形状

static void gistunionsubkey(GISTSTATE *giststate, IndexTuple *itvec, GistSplitVector *spl)

入力と出力

  • **入力**:
  • giststate: GiSTインデックスの操作状態を保持する構造体。
  • itvec: インデックス内のタプル配列。
  • spl: 分割結果を保持する GistSplitVector 構造体。左右のページ情報および属性情報を含む。
  • **出力**: なし（副作用として spl 内の属性情報が更新される）。

前提条件と不変条件

  • spl には、gistUserPicksplit 等によって生成された有効な分割結果が格納されている必要がある。
  • gistunionsubkeyvec を呼び出すことで、各ページに含まれるタプルのユニオンが計算され、spl->spl_lattr / spl->spl_rattr 等に反映される。

アルゴリズムまたは振る舞い

1. GistSplitUnion 構造体 gsvp を初期化し、spl->spl_dontcare を設定する。 2. 左側のページ（spl->splitVector.spl_left）について、gsvp の各フィールドを更新し、gistunionsubkeyvec を呼び出してユニオンキーを計算する。 3. 右側のページ（spl->splitVector.spl_right）について、同様に gsvp の各フィールドを更新し、gistunionsubkeyvec を呼び出してユニオンキーを計算する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistsplit.c 内の複数箇所（行543, 689, 777など）。
  • **呼び出し先**: gistunionsubkeyvec（同ファイル内）。

関連データ構造

  • GistSplitVector: 分割結果を保持する構造体。
  • GistSplitUnion: ユニオン計算用の一時的な作業用構造体。
  • GISTSTATE: インデックス操作のコンテキスト。

失敗モードと境界条件

  • spl が指すデータが不整合な場合、gistunionsubkeyvec 内でエラーが発生する可能性がある。
  • 呼び出し元で提供される itvec が spl 内のインデックスと一致しない場合、誤ったユニオンキーが計算される。

保守担当者向けメモ

  • 本関数は、分割後のページが保持するキーの範囲を再確定させる重要なステップである。
  • gistunionsubkeyvec が内部で呼び出されるため、ユニオン計算ロジックの変更はそちらを参照すること。
