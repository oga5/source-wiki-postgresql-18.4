AtEOXact_ApplyLauncher

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/launcher.c:1095

目的

トランザクション終了時（End of Transaction）に、論理レプリケーションランチャーの起動要求フラグをリセットし、必要に応じてランチャープロセスを起床させる。

シグネチャまたは形状

void AtEOXact_ApplyLauncher(bool isCommit)

入力と出力

  • isCommit (bool): トランザクションがコミットされた場合は true、アボートされた場合は false を指定する。
  • 出力: なし（副作用としてランチャーの起床およびフラグのリセットを行う）。

前提条件と不変条件

  • 本関数はトランザクション終了処理の一部として呼び出される。
  • on_commit_launcher_wakeup という静的フラグの状態に基づき動作する。
  • 実行後、必ず on_commit_launcher_wakeup は false にリセットされる。

アルゴリズムまたは振る舞い

1. isCommit が true の場合:

  • on_commit_launcher_wakeup が true であれば、ApplyLauncherWakeup() を呼び出してランチャープロセスを起床させる。

    • 2. isCommit が false の場合:

  • 何もせず、フラグのリセットのみを行う。

    • 3. 最後に on_commit_launcher_wakeup を false に設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xact.c: トランザクションのコミットおよびアボート処理の各所で呼び出される。
  • 呼び出し先:
  • ApplyLauncherWakeup() (条件付き)

関連データ構造

  • on_commit_launcher_wakeup (静的変数): トランザクションコミット時にランチャーを起床させるべきかを示すフラグ。

失敗モードと境界条件

  • 本関数自体はエラーを発生させない。
  • ApplyLauncherWakeup() が内部でロック取得等を行うため、その過程でエラーが発生する可能性はある。

保守担当者向けメモ

  • 本関数はトランザクションの成否に関わらず、終了時に必ずフラグをクリアする設計となっている。
  • on_commit_launcher_wakeup フラグの管理箇所と併せて確認すること。
