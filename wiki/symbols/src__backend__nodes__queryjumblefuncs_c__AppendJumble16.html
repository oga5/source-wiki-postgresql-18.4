AppendJumble16

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/queryjumblefuncs.c:333

目的

クエリのフィンガープリント生成プロセスにおいて、16ビット（2バイト）のデータをハッシュ計算用のバッファへ追加します。

シグネチャまたは形状

static pg_noinline void AppendJumble16(JumbleState *jstate, const unsigned char *value)

入力と出力

  • jstate: ハッシュ計算の状態とバッファを保持する JumbleState 構造体へのポインタ。
  • value: 追加する16ビットデータが格納されたメモリ領域へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • jstate は有効な JumbleState を指している必要があります。
  • value は少なくとも2バイトのデータを含んでいる必要があります。

アルゴリズムまたは振る舞い

1. jstate->pending_nulls が 0 より大きい場合、FlushPendingNulls を呼び出して保留中の NULL 情報をバッファに書き出します。 2. AppendJumbleInternal を呼び出し、value から 2 バイトをハッシュバッファに追加します。 3. pg_noinline 指定により、インライン展開を抑制しスタック消費を制御しています。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/nodes/queryjumblefuncs.c 内の各ノード走査処理（例: 541行目付近）。
  • 呼び出し先: FlushPendingNulls、AppendJumbleInternal。

関連データ構造

  • JumbleState: クエリのハッシュ計算状態を管理する構造体。

失敗モードと境界条件

  • バッファが溢れた場合、AppendJumbleInternal 内部のロジックにより、既存のバッファ内容がハッシュ化されバッファが再利用されます。

保守担当者向けメモ

  • 新しいノード型を追加する際、16ビット長のフィールドをハッシュに含める場合は本関数を使用してください。
  • pg_noinline が指定されているため、頻繁に呼び出される箇所でのパフォーマンス特性に留意してください。
