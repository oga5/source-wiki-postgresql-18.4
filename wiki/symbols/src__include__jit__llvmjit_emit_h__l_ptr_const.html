l_ptr_const

定義

  • 種別: static inline function
  • 定義箇所: src/include/jit/llvmjit_emit.h:27

目的

任意のメモリポインタを LLVM IR 上の定数ポインタ値（LLVMValueRef）に変換するヘルパー関数です。

シグネチャまたは形状

LLVMValueRef l_ptr_const(void *ptr, LLVMTypeRef type)

入力と出力

  • **入力**:
  • ptr: 変換対象のメモリ上のポインタ。
  • type: 生成する LLVM ポインタ型の型定義（LLVMTypeRef）。
  • **出力**:
  • 指定された型を持つ、ポインタ値としての LLVMValueRef。

前提条件と不変条件

  • TypeSizeT が LLVM コンテキスト内で定義されている必要があります。
  • ptr は実行時に有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. ptr を uintptr_t にキャストし、LLVMConstInt を用いて TypeSizeT 型の整数定数 c を生成します。 2. LLVMConstIntToPtr を呼び出し、整数定数 c を指定された type（ポインタ型）へ変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit.c: v_fn_addr の生成に使用。
  • src/backend/jit/llvm/llvmjit_expr.c: 式評価ステップにおけるポインタや関数ポインタの定数化に多用（v_resvaluep, v_resnullp, v_func, v_params 等）。
  • **呼び出し先**:
  • LLVMConstInt (LLVM C API)
  • LLVMConstIntToPtr (LLVM C API)

関連データ構造

  • LLVMValueRef: LLVM IR の値を表す型。
  • LLVMTypeRef: LLVM の型定義。
  • TypeSizeT: PostgreSQL の JIT コンテキストにおけるサイズ型定数。

失敗モードと境界条件

  • ptr が NULL の場合も、有効なポインタ定数として変換されます。
  • type にポインタ型以外を指定した場合、LLVM の定数生成ルールに従いエラーとなる可能性があります。

保守担当者向けメモ

  • 本関数は USE_LLVM が定義されている環境でのみ利用可能です。
  • LLVM C API の変更に伴い、定数生成ロジックが変更される可能性があるため、LLVM バージョンアップ時には挙動を確認してください。
