DatumGetJsonPathP

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/jsonpath.h:34

目的

Datum 型として渡された JSON Path データを、メモリ上で操作可能な JsonPath 構造体へのポインタとして取得します。

シグネチャまたは形状

static inline JsonPath *DatumGetJsonPathP(Datum d)

入力と出力

  • **入力**: Datum d (JSON Path を保持する Datum)
  • **出力**: JsonPath * (デトースト処理済みの JsonPath 構造体へのポインタ)

前提条件と不変条件

  • 入力 d は、有効な JsonPath 型の Datum である必要があります。
  • PG_DETOAST_DATUM マクロを使用しているため、入力が圧縮またはアウトオブライン（TOAST）化されている場合は、メモリ上に展開された形式に変換されます。

アルゴリズムまたは振る舞い

1. 引数 d に対して PG_DETOAST_DATUM を実行し、必要に応じてデータを展開します。 2. 展開された結果を JsonPath * 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c
  • src/backend/optimizer/util/clauses.c
  • src/backend/utils/adt/jsonpath_exec.c
  • src/include/utils/jsonpath.h (マクロ PG_GETARG_JSONPATH_P 内)
  • **呼び出し先**:
  • PG_DETOAST_DATUM (PostgreSQL のコアマクロ)

関連データ構造

  • JsonPath: 本関数が指し示す対象となる、バイナリ形式の JSON Path 構造体。

失敗モードと境界条件

  • 入力が NULL の場合、PG_DETOAST_DATUM の仕様に従い、呼び出し元で適切な処理が行われない限りエラーとなる可能性があります。

保守担当者向けメモ

  • 本関数はインライン化されており、パフォーマンスを重視して設計されています。
  • JsonPath 構造体のバイナリ形式に変更を加える際は、本関数が依存する PG_DETOAST_DATUM の挙動と、JsonPath のメモリレイアウトに影響がないか確認してください。
