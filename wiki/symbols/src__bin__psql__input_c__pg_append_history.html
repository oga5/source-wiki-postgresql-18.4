pg_append_history

定義

  • 種別: function
  • 定義箇所: src/bin/psql/input.c:112

目的

ユーザーが入力した文字列を、履歴管理用のバッファ（PQExpBuffer）に蓄積する。

シグネチャまたは形状

void pg_append_history(const char *s, PQExpBuffer history_buf);

入力と出力

  • **入力**:
  • s: 追加対象の文字列。
  • history_buf: 履歴を蓄積する PQExpBuffer 構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • USE_READLINE が定義されている環境でのみ機能する。
  • グローバル変数 useHistory が真である必要がある。
  • s が NULL の場合、何も行われない。

アルゴリズムまたは振る舞い

1. useHistory が有効かつ s が非ヌルであることを確認する。 2. appendPQExpBufferStr を使用して、文字列 s を history_buf に追加する。 3. 文字列 s が空であるか、末尾が改行文字（\n）でない場合、appendPQExpBufferChar を使用して改行文字を history_buf に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/mainloop.c (複数箇所で呼び出しを確認)
  • **呼び出し先**:
  • appendPQExpBufferStr
  • appendPQExpBufferChar
  • strlen

関連データ構造

  • PQExpBuffer: PostgreSQL の動的文字列バッファ構造体。

失敗モードと境界条件

  • s が空文字列の場合、改行のみがバッファに追加される。
  • USE_READLINE が定義されていない環境では、この関数は何も実行しない（空の関数として振る舞う）。

保守担当者向けメモ

  • 本関数は履歴バッファへの一時的な蓄積を行うものであり、readline の履歴リストへのコミットは別途 pg_send_history が担当する。
  • readline ライブラリの有無による条件分岐が含まれているため、ビルド環境の差異に注意すること。
