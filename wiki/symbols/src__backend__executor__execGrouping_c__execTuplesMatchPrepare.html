execTuplesMatchPrepare

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execGrouping.c:57

目的

タプルのグループ化や重複排除処理において、指定された属性に基づきタプル間の等価性を判定するための式評価状態（ExprState）を準備します。

シグネチャまたは形状

ExprState *execTuplesMatchPrepare(TupleDesc desc,
                                  int numCols,
                                  const AttrNumber *keyColIdx,
                                  const Oid *eqOperators,
                                  const Oid *collations,
                                  PlanState *parent)

入力と出力

  • **入力**:
  • desc: 対象タプルの記述子。
  • numCols: 比較対象の列数。
  • keyColIdx: 比較対象の属性番号配列。
  • eqOperators: 各列の比較に使用する等価演算子の OID 配列。
  • collations: 各列の照合順序（Collation）OID 配列。
  • parent: 実行ノードの親状態（PlanState）。
  • **出力**:
  • 構築された ExprState オブジェクト。numCols が 0 の場合は NULL を返します。

前提条件と不変条件

  • numCols が 0 の場合、比較は不要とみなされ直ちに NULL が返されます。
  • 内部で palloc を使用して eqFunctions 配列を確保します。

アルゴリズムまたは振る舞い

1. numCols が 0 であれば NULL を返す。 2. numCols 分のメモリを確保し、get_opcode を用いて各 eqOperators から対応する等価関数（演算子関数）の OID を取得する。 3. ExecBuildGroupingEqual を呼び出し、提供された情報に基づいてタプル比較用の ExprState を構築する。 4. 構築した ExprState を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nodeAgg.c (ハッシュ集約)
  • nodeGroup.c (グループ化)
  • nodeLimit.c (制限処理)
  • nodeUnique.c (重複排除)
  • nodeWindowAgg.c (ウィンドウ関数)
  • src/backend/utils/adt/orderedsetaggs.c (順序付き集合集約)
  • **呼び出し先**:
  • get_opcode (システムカタログから演算子関数を取得)
  • ExecBuildGroupingEqual (式評価状態の構築)

関連データ構造

  • ExprState: 式評価の実行状態を保持する構造体。
  • TupleDesc: タプルの構造定義。
  • PlanState: 実行計画ノードの状態。

失敗モードと境界条件

  • numCols が 0 の場合、比較関数は生成されず NULL が返される。
  • メモリ不足時には palloc によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は ExecBuildGroupingEqual のラッパーとして機能しており、主に演算子 OID から関数 OID への変換を担います。
  • 呼び出し側で ExprState のライフサイクルを管理する必要があります。
