vacuum_get_cutoffs

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuum.c:1115

目的

VACUUM処理において、タプルの凍結（freeze）および削除の基準となるトランザクションID（XID）とマルチトランザクションID（MXID）のカットオフ値を計算し、対象リレーションに対して「積極的（aggressive）VACUUM」が必要かどうかを判定する。

シグネチャまたは形状

bool vacuum_get_cutoffs(Relation rel, const VacuumParams *params, struct VacuumCutoffs *cutoffs)

入力と出力

  • **入力**:
  • rel: 対象リレーションの Relation 構造体。
  • params: 凍結年齢設定を含む VacuumParams 構造体。
  • **出力**:
  • cutoffs: 計算された OldestXmin, OldestMxact, FreezeLimit, MultiXactCutoff 等を格納する VacuumCutoffs 構造体。
  • 戻り値: 積極的VACUUMが必要な場合は true、それ以外は false。

前提条件と不変条件

  • cutoffs->OldestXmin は TransactionIdIsNormal であること。
  • cutoffs->OldestMxact は MultiXactIdIsValid であること。
  • FreezeLimit は常に OldestXmin 以下であること。
  • MultiXactCutoff は常に OldestMxact 以下であること。

アルゴリズムまたは振る舞い

1. **初期値取得**: GetOldestNonRemovableTransactionId および GetOldestMultiXactId を用いて、システム全体で保持すべき最古のXID/MXIDを取得する。 2. **安全性の検証**: 現在のXID/MXIDと autovacuum_freeze_max_age 等を比較し、ラップアラウンドの危険性が高い場合は警告を発する。 3. **カットオフ計算**:

  • freeze_min_age および multixact_freeze_min_age を、指定値とシステム上限の半分（autovacuum_freeze_max_age / 2）の小さい方で決定する。
  • FreezeLimit および MultiXactCutoff を現在値から上記年齢を引いて算出する。

    • 4. **積極的VACUUM判定**:

  • freeze_table_age および multixact_freeze_table_age を用いて、テーブルの凍結年齢が閾値を超えているか確認する。
  • relfrozenxid または relminmxid が閾値より古い場合、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c:777
  • src/backend/commands/cluster.c:920
  • **呼び出し先**:
  • GetOldestNonRemovableTransactionId, GetOldestMultiXactId, ReadNextTransactionId, ReadNextMultiXactId, MultiXactMemberFreezeThreshold 等。

関連データ構造

  • VacuumParams: VACUUM実行時のパラメータ。
  • VacuumCutoffs: 計算結果を保持する構造体。
  • Relation: 対象テーブルのメタデータ。

失敗モードと境界条件

  • safeOldestXmin 等が FirstNormalTransactionId を下回る場合、最小値にクランプされる。
  • 凍結年齢設定が負の値の場合、GUC設定値が使用される。
  • 積極的VACUUM判定において、閾値計算時に autovacuum_freeze_max_age * 0.95 を上限とする制限が適用される。

保守担当者向けメモ

  • 本関数はトランザクションのラップアラウンド防止のための重要な判定ロジックを含む。変更時は autovacuum の挙動に直結するため注意すること。
  • relfrozenxid の更新タイミングと本関数の計算結果の整合性に留意すること。
