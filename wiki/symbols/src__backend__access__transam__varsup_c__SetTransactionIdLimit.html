SetTransactionIdLimit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/varsup.c:371

目的

データベースの凍結状態（oldest_datfrozenxid）に基づき、トランザクションID（XID）の枯渇（Wraparound）を防ぐための各種閾値を計算・更新し、必要に応じて警告や自動バキュームの起動信号を送信する。

シグネチャまたは形状

void SetTransactionIdLimit(TransactionId oldest_datfrozenxid, Oid oldest_datoid)

入力と出力

  • **入力**:
  • oldest_datfrozenxid: データベース内で最も古い凍結済みXID。
  • oldest_datoid: 上記XIDを持つデータベースのOID。
  • **出力**: なし（共有メモリ上の TransamVariables を更新）。

前提条件と不変条件

  • oldest_datfrozenxid は TransactionIdIsNormal を満たす必要がある。
  • autovacuum_freeze_max_age はGUCにより妥当な範囲に制限されており、xidVacLimit が xidWarnLimit より前に来るよう保証されている。

アルゴリズムまたは振る舞い

1. **閾値の算出**:

  • xidWrapLimit: oldest_datfrozenxid からXID空間の半分先。
  • xidStopLimit: xidWrapLimit の300万前（対話モードでのXID割り当て拒否閾値）。
  • xidWarnLimit: xidWrapLimit の4000万前（警告開始閾値）。
  • xidVacLimit: oldest_datfrozenxid + autovacuum_freeze_max_age（自動バキューム強制閾値）。

    • 2. **状態更新**: XidGenLock を排他取得し、TransamVariables 構造体の各フィールドを更新する。 3. **信号送信**: 現在のXIDが xidVacLimit を超えており、かつPostmaster配下でリカバリ中でない場合、PMSIGNAL_START_AUTOVAC_LAUNCHER を送信する。 4. **警告出力**: 現在のXIDが xidWarnLimit を超えている場合、警告ログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c (チェックポイント処理時)
  • src/backend/commands/vacuum.c (VACUUM実行時)
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (XidGenLock)
  • ereport (ログ出力)
  • SendPostmasterSignal (自動バキューム起動)
  • get_database_name (警告メッセージ用)

関連データ構造

  • TransamVariables: 共有メモリ上の状態管理構造体。
  • XidGenLock: ID生成および閾値更新を保護する軽量ロック。

失敗モードと境界条件

  • **リカバリ中**: InRecovery の場合、自動バキューム信号の送信や警告出力の一部が抑制される。
  • **データベース名取得失敗**: get_database_name がNULLを返した場合、OIDのみを含む警告メッセージにフォールバックする。
  • **XIDのラップアラウンド**: 計算結果が FirstNormalTransactionId を下回る場合、補正が行われる。

保守担当者向けメモ

  • autovacuum_freeze_max_age は PGC_POSTMASTER パラメータであるため、実行中の動的な変更には対応していない。
  • XidGenLock の保持時間を最小限にするため、ログ出力や信号送信はロック解放後に行われている。
  • 警告メッセージは、DBAが手動でVACUUMや古いレプリケーションスロットの削除を行うよう促す内容となっている。
