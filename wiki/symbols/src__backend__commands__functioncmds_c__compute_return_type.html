compute_return_type

定義

  • 種別: function
  • 定義箇所: src/backend/commands/functioncmds.c:87

目的

関数定義における戻り値の型（prorettype）と、集合を返すかどうか（returnsSet）を解決・検証する。

シグネチャまたは形状

static void compute_return_type(TypeName *returnType, Oid languageOid, Oid *prorettype_p, bool *returnsSet_p)

入力と出力

  • **入力**:
  • returnType: 戻り値の型名を表す TypeName 構造体。
  • languageOid: 関数言語の OID。
  • **出力**:
  • prorettype_p: 解決された型の OID を格納するポインタ。
  • returnsSet_p: SETOF 指定の有無を格納するポインタ。

前提条件と不変条件

  • returnType が既存の型を参照している場合、その型に対する ACL_USAGE 権限が必要。
  • シェル型（未定義型）の作成は、C 言語または内部言語（INTERNAL）で定義される関数にのみ許可される。

アルゴリズムまたは振る舞い

1. LookupTypeName を使用して型を検索する。 2. 型が存在する場合:

  • 型がシェル型（typisdefined が偽）であれば、SQL 言語の場合はエラー、それ以外は通知を発行する。
  • 型の OID を取得し、システムキャッシュを解放する。

    • 3. 型が存在しない場合:

  • 言語が C または内部言語以外であればエラーとする。
  • typmods が指定されている場合はエラーとする。
  • TypeShellMake を呼び出し、シェル型を新規作成する。

    • 4. 最後に、解決された型に対する ACL_USAGE 権限をチェックし、結果をポインタ経由で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/functioncmds.c 内の関数定義処理（CreateFunction 等）から呼び出される。
  • **呼び出し先**: LookupTypeName, TypeNameToString, GETSTRUCT, ReleaseSysCache, typeTypeId, QualifiedNameGetCreationNamespace, object_aclcheck, TypeShellMake, aclcheck_error, aclcheck_error_type 等。

関連データ構造

  • TypeName: SQL パーサーが生成する型名構造体。
  • pg_type: 型情報を保持するシステムカタログ。

失敗モードと境界条件

  • **権限エラー**: 型に対する USAGE 権限がない場合、または名前空間に対する CREATE 権限がない場合にエラーを投げる。
  • **型定義エラー**: SQL 言語でシェル型を指定した場合、または非 C/内部言語で存在しない型を指定した場合にエラーを投げる。
  • **構文エラー**: シェル型作成時に typmod が指定されている場合にエラーを投げる。

保守担当者向けメモ

  • 本関数は pg_proc の prorettype を決定する重要なステップである。
  • シェル型の作成ロジックは、型定義のタイポによるカタログ汚染を防ぐための制約を含んでいるため、変更時は注意が必要。
