process_data_type_check

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/check.c:389

目的

データベース内のデータ型使用状況チェックの結果（PGresult）を処理し、互換性違反がある場合にログ出力およびレポートファイルへの書き出しを行う。

シグネチャまたは形状

static void process_data_type_check(DbInfo *dbinfo, PGresult *res, void *arg)

入力と出力

  • **入力**:
  • dbinfo: 現在処理中のデータベース情報。
  • res: SQLクエリの実行結果（nspname, relname, attname を含む）。
  • arg: data_type_check_state 構造体へのポインタ。
  • **出力**: なし（副作用としてログ出力およびファイル書き込みを行う）。

前提条件と不変条件

  • AssertVariableIsOfType(&process_data_type_check, UpgradeTaskProcessCB) により、UpgradeTaskProcessCB 型のコールバックとして適合することが保証されている。
  • arg は struct data_type_check_state * にキャスト可能であること。

アルゴリズムまたは振る舞い

1. クエリ結果のタプル数（ntups）が0であれば即座に終了する。 2. state->check->report_filename に基づき出力先パスを決定する。 3. 初回の失敗検知時:

  • state->report が未初期化であれば createPQExpBuffer() で確保する。
  • ログに失敗メッセージを出力し、レポートバッファに詳細情報を追加する。

    • 4. state->result を true に設定する。 5. 指定されたレポートファイル（output_path）を追記モードで開き、データベース名と問題のあるカラム名（スキーマ名.リレーション名.属性名）を書き出して閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c 内の upgrade_task_add_step を介して呼び出される。
  • **呼び出し先**: PQntuples, PQfnumber, PQgetvalue (libpq), createPQExpBuffer, appendPQExpBuffer (fe_utils), fopen_priv, pg_log, pg_fatal。

関連データ構造

  • DbInfo: データベースのメタデータ。
  • data_type_check_state: チェックの進行状況と結果を保持する構造体。
  • PGresult: libpq のクエリ結果オブジェクト。

失敗モードと境界条件

  • fopen_priv が失敗した場合、pg_fatal を呼び出しプロセスを異常終了させる。
  • クエリ結果に期待されるカラム（nspname, relname, attname）が存在しない場合、PQfnumber は -1 を返し、その後の PQgetvalue で未定義動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は UpgradeTask インフラの一部として動作する。
  • レポートファイルへの書き込みは追記モード（"a"）で行われるため、複数データベースで同じチェックに失敗した場合、一つのファイルに集約される。
