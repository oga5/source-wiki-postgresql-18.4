pg_sub_s64_overflow

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:261

目的

64ビット符号付き整数（int64）の減算において、結果が型の表現範囲を超えるオーバーフローが発生するかを判定し、安全に計算を行う。

シグネチャまたは形状

static inline bool pg_sub_s64_overflow(int64 a, int64 b, int64 *result)

入力と出力

  • **入力**:
  • a: 被減数（int64）
  • b: 減数（int64）
  • result: 計算結果を格納するポインタ（int64 *）
  • **出力**:
  • 戻り値: オーバーフローが発生した場合は true、それ以外は false を返す。
  • *result: 演算成功時は a - b の値。オーバーフロー時はダミー値 0x5EED が書き込まれる。

前提条件と不変条件

  • int64 型の範囲内での演算を前提とする。
  • コンパイラが __builtin_sub_overflow をサポートしている場合はそれが優先される。
  • HAVE_INT128 が定義されている環境では、128ビット整数を用いて範囲チェックを行う。

アルゴリズムまたは振る舞い

1. **コンパイラ組み込み関数**: HAVE__BUILTIN_OP_OVERFLOW が定義されていれば、組み込み関数で判定・計算する。 2. **128ビット拡張**: HAVE_INT128 が定義されている場合、int128 にキャストして減算し、結果が PG_INT64_MAX を超えるか PG_INT64_MIN 未満であればオーバーフローとみなす。 3. **手動チェック**: 上記以外の場合、符号の組み合わせと境界値（PG_INT64_MIN 等）に基づき、減算結果が範囲外となる条件を個別に判定する。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • 外部関数への依存なし（インライン展開される）。

関連データ構造

  • int64（PostgreSQLの標準的な64ビット符号付き整数型）

失敗モードと境界条件

  • a - b が int64 の範囲（-9,223,372,036,854,775,808 ～ 9,223,372,036,854,775,807）を超える場合に true を返す。
  • 特に b が PG_INT64_MIN の場合など、符号反転や減算時にオーバーフローが発生しやすいケースを適切に処理する。
