directBoolConsistentFn

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginlogic.c:64

目的

GINインデックスの検索において、演算子クラスが提供する二値論理の整合性関数を呼び出し、インデックスエントリが検索条件に合致するかを判定するラッパー関数です。

シグネチャまたは形状

static bool directBoolConsistentFn(GinScanKey key)

入力と出力

  • **入力**: GinScanKey key（スキャン状態と検索条件を保持する構造体）
  • **出力**: bool（インデックスエントリが検索条件に合致するかどうか）

前提条件と不変条件

  • key->consistentFmgrInfo が有効な関数情報として初期化されている必要があります。
  • 呼び出し前に key->recheckCurItem が初期化されます。

アルゴリズムまたは振る舞い

1. key->recheckCurItem を true に設定します（整合性関数側で設定されない場合の安全策）。 2. FunctionCall8Coll を使用し、key->consistentFmgrInfo が指す関数を呼び出します。 3. 呼び出しには、entryRes、strategy、query、nuserentries、extra_data、recheckCurItem、queryValues、queryCategories の8つの引数が渡されます。 4. 呼び出し結果を DatumGetBool で変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginlogic.c 内の各所（ginInitConsistentFunction による関数ポインタ割り当てを含む）。
  • **呼び出し先**:
  • FunctionCall8Coll（PostgreSQLの関数呼び出しインターフェース）。

関連データ構造

  • GinScanKey: 整合性チェックに必要な状態を保持する構造体。

失敗モードと境界条件

  • recheckCurItem がデフォルトで true に設定されるため、整合性関数が明示的に値を設定しない場合、常に再チェックが必要なものとして扱われます。

保守担当者向けメモ

  • 本関数は、opclassが二値論理の整合性関数を直接提供している場合に使用されます。
  • FunctionCall8Coll を介した動的な関数呼び出しを行うため、パフォーマンスは呼び出される整合性関数の実装に依存します。
