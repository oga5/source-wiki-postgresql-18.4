dsa_pin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:974

目的

指定された dsa_area を「ピン留め」し、共有メモリ領域が意図せず解放されることを防ぐための参照カウント管理を行う。

シグネチャまたは形状

void dsa_pin(dsa_area *area)

入力と出力

  • **入力**: area (操作対象の dsa_area 構造体へのポインタ)
  • **出力**: なし（成功時）。エラー発生時は elog(ERROR) によりプロセスが中断される。

前提条件と不変条件

  • DSA_AREA_LOCK(area) を介した排他制御が行われる。
  • 呼び出し時点で、対象の dsa_area が既にピン留め状態（area->control->pinned が true）であってはならない。

アルゴリズムまたは振る舞い

1. DSA_AREA_LOCK(area) を排他モードで取得する。 2. area->control->pinned を確認し、既に true であればロックを解放してエラーを投げる。 3. area->control->pinned を true に設定する。 4. area->control->refcnt をインクリメントする。 5. DSA_AREA_LOCK(area) を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/launcher.c:1018
  • src/backend/storage/ipc/dsm_registry.c:106
  • src/backend/utils/activity/pgstat_shmem.c:186
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (共有メモリの排他制御)
  • elog (エラー報告)

関連データ構造

  • dsa_area: DSA 領域を管理するバックエンドごとの状態構造体。
  • dsa_area_control: 共有メモリの先頭に配置される領域全体の制御構造体。

失敗モードと境界条件

  • **エラー**: 既にピン留めされている領域に対して本関数を呼び出した場合、elog(ERROR) が発生する。

保守担当者向けメモ

  • 本関数は dsa_unpin と対になる操作である。
  • 領域のライフサイクル管理において、共有メモリ領域が不要になる前に適切にピン留めが解除されることを保証する必要がある。
