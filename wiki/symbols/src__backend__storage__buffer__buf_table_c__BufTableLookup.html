BufTableLookup

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/buf_table.c:89

目的

共有バッファプール内のハッシュテーブルから、指定された BufferTag に対応するバッファIDを検索します。

シグネチャまたは形状

int BufTableLookup(BufferTag *tagPtr, uint32 hashcode)

入力と出力

  • **入力**:
  • tagPtr: 検索対象のページを識別する BufferTag へのポインタ。
  • hashcode: tagPtr から事前に計算されたハッシュ値。
  • **出力**:
  • 成功時: 対応するバッファID（int 型）。
  • 失敗時: -1。

前提条件と不変条件

  • 本関数はロックを保持しません。呼び出し側は、対応する BufMappingLock パーティションを適切にロックしている必要があります。
  • SharedBufHash が初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. hash_search_with_hash_value を呼び出し、共有ハッシュテーブル SharedBufHash に対して HASH_FIND アクションを実行します。 2. 検索結果が NULL（見つからない）の場合、-1 を返します。 3. 検索結果が存在する場合、そのエントリ（BufferLookupEnt）から id フィールドを取得して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c (複数の箇所でバッファ検索時に利用)
  • **呼び出し先**:
  • hash_search_with_hash_value (PostgreSQL 共有ハッシュテーブル API)

関連データ構造

  • BufferTag: ページを識別するキー構造体。
  • BufferLookupEnt: ハッシュテーブルに格納されるエントリ構造体。
  • SharedBufHash: バッファマッピングを保持するグローバルな共有ハッシュテーブル。

失敗モードと境界条件

  • 指定されたタグがハッシュテーブル内に存在しない場合、-1 を返します。これはエラーではなく、キャッシュミスとして扱われます。

保守担当者向けメモ

  • 本関数はスレッドセーフではありません。呼び出し側が BufMappingLock を管理する設計を維持してください。
  • ハッシュ値の計算は呼び出し側で行うため、本関数内での計算コストは発生しません。
