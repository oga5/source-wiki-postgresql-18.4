longest

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:41

目的

正規表現の最長一致（Longest Match）を求めるためのメインループを実行します。決定性有限オートマトン（DFA）を用いて、入力文字列をスキャンし、マッチング条件を満たす最も長い位置を特定します。

シグネチャまたは形状

static chr *longest(struct vars *v, struct dfa *d, chr *start, chr *stop, int *hitstopp)

入力と出力

  • **入力**:
  • v: 正規表現の実行状態を保持する vars 構造体。
  • d: DFA の実行状態を保持する dfa 構造体。
  • start: マッチングを開始する文字列位置。
  • stop: マッチングを終了すべき上限位置。
  • hitstopp: stop に到達したかを記録するフラグ用ポインタ。
  • **出力**:
  • マッチした文字列の末尾位置を示す chr *。マッチしない場合は NULL を返します。

前提条件と不変条件

  • d->cnfa が有効なコンパイル済み NFA であること。
  • initialize() が正常に開始状態を生成できること。
  • d->backno >= 0 の場合、v->pmatch に有効なバックリファレンス情報が存在すること。

アルゴリズムまたは振る舞い

1. **バックリファレンス処理**: d->backno が設定されている場合、dfa_backref を呼び出し、既知の文字列とのマッチングを優先的に行います。 2. **高速パス**: MATCHALL フラグが有効な場合、NFA の制約に基づき、計算を省略して即座に結果を返します。 3. **初期化**: initialize() で DFA の開始状態セットを準備します。 4. **スキャンループ**: 入力文字列を 1 文字ずつ進め、GETCOLOR で文字を色（クラス）に変換し、css->outs[co] を通じて状態遷移を行います。キャッシュミス時は miss() を呼び出して遷移を計算します。 5. **終了処理**: 文字列終端で POSTSTATE フラグを持つ状態に到達したかを確認します。 6. **結果確定**: 全てのキャッシュエントリ（sset）を走査し、POSTSTATE を持つ状態の中で最も後ろの lastseen 位置を特定して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコードの文脈からは直接の呼び出し元は特定できません（regexec.c からの呼び出しが想定されます）。
  • **呼び出し先**: initialize, miss, dfa_backref, GETCOLOR マクロ。

関連データ構造

  • struct dfa: DFA の実行状態を保持。
  • struct sset: DFA の状態セット（キャッシュエントリ）。
  • struct vars: 正規表現の実行コンテキスト。

失敗モードと境界条件

  • initialize() または miss() が NULL を返した場合、エラーとして NULL を返します。
  • ISERR() が真の場合、エラーとして NULL を返します。
  • マッチングが一度も成功しなかった場合、NULL を返します。

保守担当者向けメモ

  • 本関数は regexec.c にインクルードされる rege_dfa.c 内で定義されており、単体でのコンパイルは想定されていません。
  • REG_DEBUG が定義されている場合、デバッグトレース用のコードパスが有効になります。
  • lacon() が再帰的に DFA を呼び出す可能性があるため、複雑な正規表現におけるスタック消費量に注意してください。
