run_crypt_sha

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px-crypt.c:70

目的

SHA-2ベースのパスワードハッシュ化アルゴリズム（SHA-256/512）を pgcrypto の抽象化レイヤーに適合させるためのラッパー関数です。

シグネチャまたは形状

static char *run_crypt_sha(const char *psw, const char *salt, char *buf, unsigned len);

入力と出力

  • **入力**:
  • psw: ハッシュ化対象のパスワード文字列。
  • salt: ハッシュ化に使用するソルト文字列。
  • buf: 結果を格納するための出力バッファ。
  • len: buf のサイズ。
  • **出力**:
  • ハッシュ化された文字列へのポインタ（buf を指す）。

前提条件と不変条件

  • buf は、計算結果を格納するのに十分なサイズである必要があります。
  • 内部で呼び出される px_crypt_shacrypt が正常に動作する環境である必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取ったパスワード、ソルト、バッファ、およびバッファサイズをそのまま px_crypt_shacrypt 関数へ転送します。 2. px_crypt_shacrypt の戻り値（ハッシュ化された文字列）をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/px-crypt.c 内の静的テーブル px_crypt_list から参照されています。
  • **呼び出し先**: px_crypt_shacrypt（crypt-sha.c で定義されていると推測される関数）。

関連データ構造

  • px_crypt_list: ソルトのプレフィックス（$5$ および $6$）と本関数を紐付ける静的テーブル。

失敗モードと境界条件

  • px_crypt_shacrypt が失敗した場合の挙動は、当該関数の実装に依存します。本関数自体はエラーハンドリングを行わず、結果を透過的に返します。

保守担当者向けメモ

  • 本関数は px_crypt_list において、SHA-256（$5$）および SHA-512（$6$）のハッシュアルゴリズムとして登録されています。
  • 実際の暗号計算ロジックは px_crypt_shacrypt に委譲されているため、アルゴリズムの修正が必要な場合は crypt-sha.c を確認してください。
