start_postmaster

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/server.c:199

目的

pg_upgrade において、指定されたクラスターの Postmaster プロセスを pg_ctl を介して起動し、接続確認を行う。

シグネチャまたは形状

bool start_postmaster(ClusterInfo *cluster, bool report_and_exit_on_error);

入力と出力

  • **入力**:
  • cluster: 起動対象のクラスター情報（ClusterInfo 構造体）。
  • report_and_exit_on_error: 起動失敗時にエラーログを出力して終了するかどうかを制御するフラグ。
  • **出力**:
  • 成功時に true、失敗時に false を返す。ただし、report_and_exit_on_error が true の場合、失敗時は内部で pg_fatal を呼び出しプロセスが終了する。

前提条件と不変条件

  • 初回呼び出し時に stop_postmaster_atexit が atexit フックとして登録される。
  • 新クラスター（new_cluster）の起動時には、パフォーマンス向上のため synchronous_commit=off、fsync=off、full_page_writes=off が強制的に付与される。
  • 非 Windows 環境では、TCP/IP 接続を無効化し、Unix ソケット権限を 0700 に制限する設定が自動的に追加される。

アルゴリズムまたは振る舞い

1. atexit フックの登録（初回のみ）。 2. クラスターのバージョンや種類に応じた pg_ctl 起動コマンド文字列を構築。 3. exec_prog を使用して pg_ctl start を実行。 4. 起動成功時、os_info.running_cluster に対象クラスターを記録。 5. get_db_conn を呼び出し、template1 データベースへの接続を試行してサーバーの稼働を確認。 6. 接続失敗または pg_ctl の実行失敗を検知した場合、エラーハンドリング（ログ出力または終了）を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c
  • src/bin/pg_upgrade/pg_upgrade.c
  • ※src/bin/pg_ctl/pg_ctl.c に同名の関数が存在するが、本ページは pg_upgrade 内の定義を対象とする。
  • **呼び出し先**:
  • atexit, initPQExpBuffer, appendPQExpBufferStr, termPQExpBuffer, snprintf, exec_prog, get_db_conn, PQstatus, PQerrorMessage, PQfinish, pg_log, pg_fatal

関連データ構造

  • ClusterInfo: 起動対象のクラスター構成情報。
  • os_info: 実行中のクラスター状態を保持するグローバル構造体。

失敗モードと境界条件

  • pg_ctl による起動失敗時、または起動後の接続確認（template1 への接続）失敗時にエラーとなる。
  • report_and_exit_on_error が false の場合、起動失敗時に false を返して呼び出し元へ制御を戻す。
  • サーバーが既に起動している場合、pg_ctl の戻り値と接続確認の結果に基づき適切にエラー処理が行われる。

保守担当者向けメモ

  • 新クラスター起動時に付与される fsync=off 等のオプションは、クラッシュ時にクラスターの再作成が必要となる前提に基づいている。
  • pg_ctl -w の失敗と接続確認の失敗は個別に扱われ、接続確認が成功すれば pg_ctl の終了コードに関わらず処理が継続される場合がある。
