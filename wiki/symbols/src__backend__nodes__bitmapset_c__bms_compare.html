bms_compare

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/bitmapset.c:182

目的

2つの Bitmapset を比較し、qsort 等のソート関数で使用可能な順序関係（-1, 0, 1）を返します。

シグネチャまたは形状

int bms_compare(const Bitmapset *a, const Bitmapset *b)

入力と出力

  • **入力**: 比較対象となる2つの Bitmapset へのポインタ a および b。
  • **出力**:
  • a < b の場合: -1
  • a > b の場合: +1
  • a == b の場合: 0

前提条件と不変条件

  • 入力される Bitmapset は bms_is_valid_set によって検証可能である必要があります。
  • 空集合は NULL ポインタとして表現されます。

アルゴリズムまたは振る舞い

1. **NULLチェック**: どちらか一方が NULL の場合、NULL を小さいものとして順序を決定します。両方 NULL なら 0 を返します。 2. **ワード数比較**: nwords が異なる場合、ワード数が多い方を大きいと見なします。 3. **ビット配列比較**: ワード数が同じ場合、末尾のワード（高位ビット側）から順に do-while ループで比較します。異なるワードが見つかった時点で、その値の大小関係を返します。 4. 全てのワードが一致すれば 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/pathnode.c（パスの relids 比較によるタイブレーク処理）
  • **呼び出し先**:
  • bms_is_valid_set（アサーション内）

関連データ構造

  • Bitmapset: nwords（ワード数）と words（ビット配列）を保持する構造体。

失敗モードと境界条件

  • 入力が NULL の場合、適切に処理されます。
  • nwords が異なる場合、ビットの内容に関わらずワード数のみで判定されます。

保守担当者向けメモ

  • 本関数は qsort スタイルの比較関数として設計されています。
  • 比較ロジックは高位のワードから順に行われるため、集合の包含関係ではなく、辞書順に近い比較が行われる点に注意してください。
