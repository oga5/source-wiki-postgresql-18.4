pg_check_fipsmode

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:489

目的

システムが FIPS モードで動作しているかどうかを確認し、その状態を SQL インターフェースへ返却する。

シグネチャまたは形状

Datum pg_check_fipsmode(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（PostgreSQL の関数呼び出し規約に従う引数）
  • 出力: Datum 型としてラップされた bool 値（FIPS モードが有効な場合は true、無効な場合は false）

前提条件と不変条件

  • PostgreSQL の関数呼び出し規約（PG_FUNCTION_ARGS）に従って呼び出される必要がある。
  • 内部で呼び出される CheckFIPSMode() が、現在の実行環境の FIPS 状態を正しく判定できること。

アルゴリズムまたは振る舞い

1. CheckFIPSMode() を呼び出し、現在の FIPS モードの状態を取得する。 2. 取得した結果を PG_RETURN_BOOL マクロを使用して PostgreSQL の Datum 型に変換し、呼び出し元へ返却する。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の SQL エンジン（SQL 関数として定義されているため）。
  • 呼び出し先: CheckFIPSMode()（外部関数）。
  • 補足: ソースコード上では PG_FUNCTION_INFO_V1(pg_check_fipsmode) により関数情報が登録されている。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数引数構造体。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • CheckFIPSMode() が実行環境でサポートされていない、あるいは判定不能な場合の挙動については、本関数のソースコード内には明示的なエラーハンドリングは存在しない。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の判定ロジックは CheckFIPSMode() に依存している。
  • FIPS モードの判定ロジックを変更または調査する場合は、CheckFIPSMode() の実装を確認すること。
