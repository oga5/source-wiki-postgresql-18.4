remoteConn

定義

  • 種別: struct
  • 定義箇所: contrib/dblink/dblink.c:73

目的

PostgreSQL からリモートデータベースへの接続状態および、その接続に関連付けられたカーソル管理情報を保持するために使用されます。

シグネチャまたは形状

typedef struct remoteConn
{
	PGconn	   *conn;			/* リモート接続の保持 */
	int			openCursorCount;	/* オープン中のカーソル数 */
	bool		newXactForCursor;	/* カーソル用にトランザクションを開始したか */
} remoteConn;

入力と出力

本構造体はデータ保持用であり、関数ではありません。conn フィールドは libpq の PGconn オブジェクトを指し、リモートデータベースとの通信インターフェースとして機能します。

前提条件と不変条件

  • conn は有効な PGconn オブジェクトである必要があります。
  • openCursorCount は、現在この接続上で開かれているカーソルの数を正確に反映している必要があります。

アルゴリズムまたは振る舞い

本構造体は、dblink モジュール内での接続管理の基盤となります。remoteConnHash ハッシュテーブルを通じて名前付き接続として管理されるほか、単一の接続を保持する変数としても使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: getConnectionByName、createNewConnection、および dblink の各関数（dblink_connect, dblink_exec 等）がこの構造体を生成または参照します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • remoteConnHashEnt: remoteConn をハッシュテーブル内で管理するためのエントリ構造体。
  • PGconn: libpq が提供するリモート接続ハンドル。

失敗モードと境界条件

  • conn が NULL の場合、リモート接続が確立されていないか、切断済みであることを示します。
  • openCursorCount の不整合は、カーソルのリークやトランザクション管理の誤りを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は TopMemoryContext に割り当てられるケースがあるため、メモリ管理には注意が必要です。
  • newXactForCursor フラグは、カーソル操作に伴うトランザクションのライフサイクルを制御するために重要です。
