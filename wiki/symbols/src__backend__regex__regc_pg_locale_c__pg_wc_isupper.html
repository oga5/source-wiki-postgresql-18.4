pg_wc_isupper

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_pg_locale.c:388

目的

指定された pg_wchar 型の文字が、現在のロケール設定において大文字（upper case）であるかを判定します。

シグネチャまたは形状

int pg_wc_isupper(pg_wchar c)

入力と出力

  • **入力**: pg_wchar c（判定対象の文字コード）
  • **出力**: int（大文字であれば非ゼロ、そうでなければ 0）

前提条件と不変条件

  • pg_regex_strategy が適切に設定されている必要があります。
  • pg_wchar は符号なし型として扱われます。
  • PG_REGEX_STRATEGY_ICU を選択する場合、USE_ICU が定義されている必要があります。

アルゴリズムまたは振る舞い

現在のロケール戦略（pg_regex_strategy）に基づき、以下のいずれかの方法で判定を行います。 1. **PG_REGEX_STRATEGY_C**: pg_char_properties テーブルを参照し、ASCII範囲内（127以下）で判定します。 2. **PG_REGEX_STRATEGY_BUILTIN**: pg_u_isupper(c) を呼び出し、組み込みのUnicodeテーブルで判定します。 3. **PG_REGEX_STRATEGY_LIBC_WIDE**: wchar_t のサイズに応じて iswupper_l を使用します。 4. **PG_REGEX_STRATEGY_LIBC_1BYTE**: isupper_l を使用します。 5. **PG_REGEX_STRATEGY_ICU**: u_isupper(c) を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_locale.c:654 (pg_ctype_get_cache 経由)
  • src/backend/regex/regc_locale.c:702
  • **呼び出し先**:
  • pg_u_isupper (戦略が BUILTIN の場合)
  • iswupper_l / isupper_l (戦略が LIBC 系の場合)
  • u_isupper (戦略が ICU の場合)

関連データ構造

  • pg_wchar: 判定対象の文字型。
  • PG_Locale_Strategy: 判定戦略を決定する列挙型。
  • pg_regex_strategy: 現在有効な判定戦略を保持するグローバル変数。
  • pg_regex_locale: ロケール情報を保持する構造体。

失敗モードと境界条件

  • PG_REGEX_STRATEGY_ICU が選択されているが USE_ICU が未定義の場合、コンパイルエラーまたは意図しない動作の可能性があります。
  • pg_regex_strategy が予期しない値の場合、関数は 0 を返します。
  • LIBC 戦略において、文字コードが UCHAR_MAX を超える場合の挙動は戦略の分岐に依存します。

保守担当者向けメモ

  • 本関数は regcomp.c からインクルードされることを前提としています。
  • キャッシュ機構（pg_ctype_get_cache）と密接に関連しているため、判定ロジックを変更する際はキャッシュの整合性に注意してください。
