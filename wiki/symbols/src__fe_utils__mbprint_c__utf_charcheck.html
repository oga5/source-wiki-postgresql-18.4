utf_charcheck

定義

  • 種別: static function
  • 定義箇所: src/fe_utils/mbprint.c:81

目的

UTF-8 文字列の先頭バイトを解析し、その文字が有効なマルチバイトシーケンスであるかを検証するとともに、文字のバイト長を特定します。

シグネチャまたは形状

static int utf_charcheck(const unsigned char *c)

入力と出力

  • **入力**: c - 解析対象の文字列の先頭バイトを指すポインタ。
  • **出力**:
  • 1〜4: 有効な UTF-8 文字のバイト数。
  • -1: 不正なバイトシーケンス、または非推奨のコードポイント（サロゲートペアや非文字など）である場合。

前提条件と不変条件

  • 入力ポインタ c は、少なくとも 4 バイト以上の読み取り可能なメモリ領域を指している必要があります（4 バイト文字の検証を行うため）。
  • UTF-8 エンコーディングを前提としています。

アルゴリズムまたは振る舞い

1. **1バイト文字**: 0x80 未満であれば 1 を返す。 2. **2バイト文字**: 0xc0〜0xdf の範囲で、後続バイトが 0x80〜0xbf であり、かつ冗長なエンコーディング（オーバーロング）でない場合に 2 を返す。 3. **3バイト文字**: 0xe0〜0xef の範囲で、後続バイトの整合性を確認。さらに、以下の不正なコードポイントを除外する：

  • 0xfffe, 0xffff
  • 0xfdd0〜0xfedf の範囲
  • サロゲートペア領域 (0xd800〜0xdfff)

    • 条件を満たせば 3 を返す。 4. **4バイト文字**: 0xf0〜0xf7 の範囲で、後続バイトの整合性とコードポイント範囲（U+00〜U+10FFFF）を確認。また、0xzzzzfffe および 0xzzzzffff を除外し、条件を満たせば 4 を返す。 5. 上記以外はすべて -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/mbprint.c 内の関数（mb_utf_validate 等の文脈で利用されていることが示唆されています）。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • 特になし（プリミティブなバイト操作のみ）。

失敗モードと境界条件

  • **不正なシーケンス**: 不正なバイトパターンや、UTF-8 として定義されていない範囲のコードポイントが入力された場合、-1 を返して拒否します。
  • **バッファオーバーラン**: 入力ポインタが文字列の終端付近にある場合、4 バイト目までアクセスするため、呼び出し側で十分なバッファ長を確保する必要があります。

保守担当者向けメモ

  • 本関数は UTF-8 特化です。他のエンコーディングへの対応が必要な場合は、このロジックを流用せず、別途実装を追加してください。
  • 0xfffe などの特殊なコードポイントの除外ロジックは、Unicode 規格の更新に伴い変更が必要になる可能性があります。
