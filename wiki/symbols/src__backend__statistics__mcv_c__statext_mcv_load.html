statext_mcv_load

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mcv.c:557

目的

指定された拡張統計オブジェクト（pg_statistic_ext）に対応する多変量MCVリストをシステムカタログから読み込み、メモリ上の MCVList 構造体として復元する。

シグネチャまたは形状

MCVList *statext_mcv_load(Oid mvoid, bool inh)

入力と出力

  • **入力**:
  • mvoid (Oid): 読み込む統計オブジェクトのOID。
  • inh (bool): 継承された統計情報を含めるかどうかを示すフラグ。
  • **出力**:
  • 復元された MCVList 構造体へのポインタ。

前提条件と不変条件

  • STATEXTDATASTXOID システムキャッシュを使用して統計データにアクセスする。
  • 該当する統計オブジェクトが存在し、かつMCVデータが構築済みである必要がある。

アルゴリズムまたは振る舞い

1. SearchSysCache2 を使用して、指定されたOIDと継承フラグに対応する pg_statistic_ext_data のタプルを取得する。 2. タプルが見つからない場合はエラーを投げる。 3. SysCacheGetAttr を介して stxdmcv 属性（MCVデータ）を取得する。 4. 属性が空（isnull）の場合は、MCVデータが未構築であるとしてエラーを投げる。 5. statext_mcv_deserialize を呼び出し、取得した bytea データを MCVList 構造体に変換する。 6. ReleaseSysCache でキャッシュタプルを解放し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/extended_stats.c:1855
  • src/backend/statistics/mcv.c:2063
  • **呼び出し先**:
  • SearchSysCache2
  • SysCacheGetAttr
  • statext_mcv_deserialize
  • ReleaseSysCache

関連データ構造

  • MCVList: 復元された多変量MCVリストを保持する構造体。
  • pg_statistic_ext_data: MCVデータが格納されているシステムカタログ。

失敗モードと境界条件

  • 指定したOIDの統計オブジェクトが存在しない場合、elog(ERROR) により処理が中断される。
  • 統計オブジェクトは存在するが、MCVデータがまだ構築されていない場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は statext_mcv_deserialize に依存しており、シリアライズ形式の変更時には両方の整合性を確認すること。
  • キャッシュの解放漏れを防ぐため、ReleaseSysCache は必ず実行されるパスを通る必要がある。
