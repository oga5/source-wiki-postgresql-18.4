gbt_datege

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_date.c:37

目的

date 型の2つの値（a と b）を比較し、a >= b であるかを判定します。

シグネチャまたは形状

static bool gbt_datege(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の DateADT 型へのポインタ。
  • b: 比較対象の DateADT 型へのポインタ。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されません）。
  • **出力**:
  • bool: a >= b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な DateADT 型の値を指している必要があります。
  • PostgreSQL のコア関数 date_ge が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. a と b のポインタを DateADT 型として参照し、DateADTGetDatum を用いて PostgreSQL の Datum 形式に変換します。 2. DirectFunctionCall2 を介して PostgreSQL コアの date_ge 関数を呼び出します。 3. date_ge の戻り値を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • date_ge (PostgreSQL コア関数)
  • **呼び出し元**:
  • contrib/btree_gist/btree_date.c 内の静的配列（インデックス操作用テーブル）から参照されています。

関連データ構造

  • DateADT: PostgreSQL の日付型を表す内部データ構造。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、メモリ保護違反が発生する可能性があります。
  • date_ge 関数が内部でエラーを発生させた場合、PostgreSQL のエラー処理メカニズムにより処理が中断されます。

保守担当者向けメモ

  • 本関数は btree_gist における date 型の比較ロジックのラッパーです。同様の比較関数（gbt_dategt, gbt_dateeq 等）と一貫した実装パターンに従っています。
