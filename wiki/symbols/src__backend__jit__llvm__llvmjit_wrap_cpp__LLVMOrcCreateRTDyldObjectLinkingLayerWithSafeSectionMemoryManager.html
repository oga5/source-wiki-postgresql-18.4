LLVMOrcCreateRTDyldObjectLinkingLayerWithSafeSectionMemoryManager

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_wrap.cpp:53

目的

JIT実行時にメモリ保護を考慮したカスタムメモリマネージャ（llvm::backport::SectionMemoryManager）を組み込んだ、llvm::orc::RTDyldObjectLinkingLayer オブジェクトを生成します。

シグネチャまたは形状

LLVMOrcObjectLayerRef LLVMOrcCreateRTDyldObjectLinkingLayerWithSafeSectionMemoryManager(LLVMOrcExecutionSessionRef ES)

入力と出力

  • **入力**: LLVMOrcExecutionSessionRef ES (LLVM ORC実行セッションへの参照)
  • **出力**: LLVMOrcObjectLayerRef (生成されたオブジェクトリンク層への不透明ポインタ)

前提条件と不変条件

  • ES は有効な LLVMOrcExecutionSessionRef である必要があります。
  • llvm::backport::SectionMemoryManager が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. ES を llvm::unwrap でC++の ExecutionSession オブジェクトに変換します。 2. llvm::orc::RTDyldObjectLinkingLayer をインスタンス化します。 3. コンストラクタのメモリマネージャ生成用ラムダ式内で、llvm::backport::SectionMemoryManager を生成します。

  • LLVM_VERSION_MAJOR >= 21 の場合、MemoryBuffer を引数に取るラムダを使用します。
  • それ以外の場合、引数なしのラムダを使用します。

    • 4. 生成されたC++オブジェクトを wrap 関数で LLVMOrcObjectLayerRef に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/llvmjit.c:1182
  • **呼び出し先**:
  • llvm::unwrap (C APIからC++オブジェクトへの変換)
  • llvm::orc::RTDyldObjectLinkingLayer (コンストラクタ)
  • llvm::backport::SectionMemoryManager (コンストラクタ)
  • wrap (C++オブジェクトからC APIへの変換)

関連データ構造

  • LLVMOrcExecutionSessionRef
  • LLVMOrcObjectLayerRef
  • llvm::orc::RTDyldObjectLinkingLayer
  • llvm::backport::SectionMemoryManager

失敗モードと境界条件

  • ES が無効な場合、unwrap 呼び出し時に未定義動作が発生する可能性があります。
  • メモリ不足時には std::make_unique が例外をスローする可能性があります。

保守担当者向けメモ

  • 本関数はLLVMのバージョン差異を吸収する役割を担っています。LLVMのメジャーバージョンアップに伴い、RTDyldObjectLinkingLayer のコンストラクタ引数やメモリマネージャの生成ロジックに変更が必要になる場合があります。
  • llvm::backport::SectionMemoryManager の詳細は src/backend/jit/llvm/SectionMemoryManager.cpp を参照してください。
