wait_for_workers_to_become_ready

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/setup.c:256

目的

バックグラウンドワーカーが共有メモリ上のヘッダーを更新し、すべてのワーカーが準備完了状態になるまで待機する同期処理を行います。

シグネチャまたは形状

static void wait_for_workers_to_become_ready(worker_state *wstate, volatile test_shm_mq_header *hdr)

入力と出力

  • wstate (入力): 起動したワーカーの状態を保持する worker_state 構造体へのポインタ。
  • hdr (入力): 共有メモリ上の test_shm_mq_header 構造体。ワーカーの準備状況を監視するために使用。
  • 戻り値: なし。成功時は正常終了。失敗時は ereport(ERROR) により処理を中断。

前提条件と不変条件

  • hdr->mutex を使用して hdr->workers_ready を保護する必要があります。
  • check_worker_status がワーカーの生存確認のために呼び出されます。

アルゴリズムまたは振る舞い

1. 無限ループ内で以下の処理を繰り返します。

  • hdr->mutex を取得し、hdr->workers_ready が wstate->nworkers 以上であるか確認します。条件を満たせばループを抜けます。
  • check_worker_status(wstate) を呼び出し、ワーカーまたはポストマスターが異常終了していないか確認します。異常があればループを抜けます。
  • WaitEventExtensionNew でカスタム待機イベント TestShmMqBgWorkerStartup を取得（初回のみ）。
  • WaitLatch を使用して、シグナルが送られるまで待機します。
  • ResetLatch でラッチをリセットし、CHECK_FOR_INTERRUPTS() で割り込みを確認します。

    • 2. ループ終了後、準備が完了していない場合は ERRCODE_INSUFFICIENT_RESOURCES エラーを発生させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/test_shm_mq/setup.c 内の test_shm_mq_setup 関数から呼び出されます。
  • 呼び出し先: SpinLockAcquire/Release、check_worker_status、WaitEventExtensionNew、WaitLatch、ResetLatch、CHECK_FOR_INTERRUPTS、ereport。

関連データ構造

  • worker_state: ワーカーの管理状態。
  • test_shm_mq_header: 共有メモリ上の制御ヘッダー。

失敗モードと境界条件

  • ワーカーが起動前にクラッシュした場合、check_worker_status がそれを検知し、ereport(ERROR) によりテスト環境の構築が失敗します。
  • ポストマスターが終了した場合、WaitLatch の WL_EXIT_ON_PM_DEATH フラグにより待機が終了します。

保守担当者向けメモ

  • 待機イベント TestShmMqBgWorkerStartup はグローバル変数 we_bgworker_startup にキャッシュされます。
  • WaitLatch の待機条件に WL_EXIT_ON_PM_DEATH が含まれており、ポストマスターの生存に依存した設計となっています。
