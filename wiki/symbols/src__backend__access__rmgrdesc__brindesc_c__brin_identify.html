brin_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/brindesc.c:73

目的

BRIN（Block Range Index）インデックスに関連するWALレコードの info ビットマスクを解析し、対応する操作名を文字列として返します。

シグネチャまたは形状

const char *brin_identify(uint8 info);

入力と出力

  • **入力**: uint8 info - WALレコードの info フィールド。
  • **出力**: const char * - 操作名を表す文字列（例: "INSERT", "UPDATE"）。該当するものがない場合は NULL を返します。

前提条件と不変条件

  • 入力された info は、XLR_INFO_MASK を除外した状態でBRINの操作タイプと一致する必要があります。

アルゴリズムまたは振る舞い

1. 入力された info に対して ~XLR_INFO_MASK を適用し、フラグを除去して純粋な操作タイプを抽出します。 2. switch 文を用いて、抽出された値に基づき以下のいずれかの文字列を返します。

  • XLOG_BRIN_CREATE_INDEX -> "CREATE_INDEX"
  • XLOG_BRIN_INSERT -> "INSERT"
  • XLOG_BRIN_INSERT | XLOG_BRIN_INIT_PAGE -> "INSERT+INIT"
  • XLOG_BRIN_UPDATE -> "UPDATE"
  • XLOG_BRIN_UPDATE | XLOG_BRIN_INIT_PAGE -> "UPDATE+INIT"
  • XLOG_BRIN_SAMEPAGE_UPDATE -> "SAMEPAGE_UPDATE"
  • XLOG_BRIN_REVMAP_EXTEND -> "REVMAP_EXTEND"
  • XLOG_BRIN_DESUMMARIZE -> "DESUMMARIZE"

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されている RmgrData 構造体の rm_identify フィールドを通じて、WAL解析ツール（pg_waldump 等）から間接的に呼び出されます。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • access/brin_xlog.h: BRINインデックスのWALレコードに関連する定数定義を提供します。

失敗モードと境界条件

  • 定義されていない info ビットマスクが渡された場合、id は初期値の NULL のまま返されます。

保守担当者向けメモ

  • 新しいBRIN操作をWALに追加した場合は、必ず本関数の switch 文に新しいケースを追加してください。
  • 本関数はログ出力用のヘルパーであり、インデックスのロジックそのものには影響しません。
