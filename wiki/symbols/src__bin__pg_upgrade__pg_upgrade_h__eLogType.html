eLogType

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.h:271

目的

pg_upgrade におけるログ出力の重要度およびフォーマット制御を分類するために使用されます。

シグネチャまたは形状

typedef enum
{
	PG_VERBOSE,
	PG_STATUS,
	PG_REPORT_NONL,
	PG_REPORT,
	PG_WARNING,
	PG_FATAL,
} eLogType;

入力と出力

  • 入力: ログ出力関数（pg_log, report_status 等）の引数として渡されます。
  • 出力: 該当しません（列挙型定義のため）。

前提条件と不変条件

  • PG_STATUS および PG_REPORT_NONL は、出力時に改行コードが付加されないという不変条件を持ちます。

アルゴリズムまたは振る舞い

ログ出力関数において、渡された eLogType の値に基づき、出力先や改行の有無、プログラムの終了判定（PG_FATAL の場合）などの挙動が分岐します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/bin/pg_upgrade/pg_upgrade.h: report_status, pg_log の宣言
  • src/bin/pg_upgrade/util.c: pg_log_v, report_status, pg_log の実装
  • 呼び出し先: 該当しません（列挙型定義のため）。

関連データ構造

  • pg_log 関数および report_status 関数で使用される引数型として密接に関連しています。

失敗モードと境界条件

  • PG_FATAL が指定された場合、ログ出力後にプログラムが異常終了する可能性があります。

保守担当者向けメモ

  • 新しいログレベルを追加する場合は、util.c 内のログ出力関数群（pg_log_v 等）のスイッチ文を更新する必要があります。
