pgss_shmem_shutdown

定義

  • 種別: function
  • 定義箇所: contrib/pg_stat_statements/pg_stat_statements.c:736

目的

PostgreSQL サーバーの停止時に、共有メモリ上のクエリ統計データおよびグローバル統計を永続化ファイルへダンプする。

シグネチャまたは形状

static void pgss_shmem_shutdown(int code, Datum arg)

入力と出力

  • **入力**:
  • code: 終了コード（0以外の場合はクラッシュとみなされ、ダンプをスキップする）。
  • arg: on_shmem_exit から渡される補助データ（本関数では未使用）。
  • **出力**: なし（副作用としてファイルシステムへ統計情報を書き出す）。

前提条件と不変条件

  • 共有メモリ領域（pgss および pgss_hash）が初期化されていること。
  • pgss_save が真であること（設定により保存が有効であること）。
  • 正常終了時のみ実行されること。

アルゴリズムまたは振る舞い

1. **ガード条件**: 異常終了時、共有メモリ未初期化時、または保存無効時には即座に終了する。 2. **ファイル作成**: pg_stat_statements.stat.tmp を作成し、ヘッダー情報とエントリ数を書き込む。 3. **データシリアライズ**:

  • 共有メモリ上のクエリテキストファイルを読み込む。
  • ハッシュテーブルを走査し、各 pgssEntry と対応するクエリ文字列をファイルに書き出す。

    • 4. **統計保存**: pgssGlobalStats を書き出す。 5. **アトミックな更新**: durable_rename を使用して .tmp ファイルを正式な統計ファイルへ置換する。 6. **クリーンアップ**: 一時的なクエリテキストファイルを削除する。 7. **エラー処理**: 書き込み失敗時はエラーログを出力し、一時ファイルを削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: on_shmem_exit に登録され、サーバー終了時に呼び出される。
  • **呼び出し先**: AllocateFile, fwrite, hash_get_num_entries, qtext_load_file, hash_seq_init, hash_seq_search, qtext_fetch, pfree, FreeFile, durable_rename, unlink, ereport。

関連データ構造

  • pgssEntry: ハッシュテーブル内の各クエリ統計エントリ。
  • pgssGlobalStats: 拡張機能全体のグローバル統計。
  • pgss_hash: 統計を保持する共有メモリ上のハッシュテーブル。

失敗モードと境界条件

  • ファイルシステムへの書き込み権限不足やディスクフルにより fwrite が失敗した場合、エラーログを出力し、一時ファイルおよびテキストファイルを削除して終了する。
  • クラッシュ時にはダンプを行わず、データの一貫性を保護する。

保守担当者向けメモ

  • 本関数は on_shmem_exit を介して登録されるため、シグネチャは on_shmem_exit の要件に従っている。
  • クエリテキストの永続化において、メモリ上のオフセットとファイル上の位置を同期させる必要があるため、qtext_fetch による読み出しロジックの変更には注意が必要。
