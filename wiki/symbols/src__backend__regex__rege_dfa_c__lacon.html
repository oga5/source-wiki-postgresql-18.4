lacon

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:924

目的

正規表現エンジンにおける先読み（Lookahead）および後読み（Lookbehind）制約の成否を判定する。

シグネチャまたは形状

static int lacon(struct vars *v, struct cnfa *pcnfa, chr *cp, color co);

入力と出力

  • **入力**:
  • v: 現在の正規表現実行状態を保持する構造体。
  • pcnfa: 親となる CNFA（コンパイル済みNFA）。
  • cp: 現在の入力文字列ポインタ。
  • co: 評価対象の制約を示す色（color）。
  • **出力**:
  • int: 制約が満たされた場合は 1、満たされない場合は 0 を返す。

前提条件と不変条件

  • 再帰的に呼び出されるため、スタックオーバーフローを防ぐための STACK_TOO_DEEP チェックが必須。
  • co は pcnfa->ncolors を超える値であり、有効な lacons インデックスに対応している必要がある。

アルゴリズムまたは振る舞い

1. **スタックチェック**: 再帰深度が限界を超えていないか確認し、超過時は REG_ETOOBIG エラーを報告して 0 を返す。 2. **制約の特定**: co からインデックスを算出し、対応する subre 構造体と DFA を取得する。 3. **制約評価**:

  • **先読み (Ahead)**: shortest() を呼び出し、マッチの有無を判定する。
  • **後読み (Behind)**: matchuntil() を呼び出し、キャッシュを活用して効率的に判定する。

    • 4. **結果の反転**: ポジティブ制約でない（ネガティブ制約である）場合、判定結果を反転させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/rege_dfa.c 内の miss() 関数から呼び出される。
  • **呼び出し先**: getladfa(), shortest(), matchuntil()。

関連データ構造

  • struct vars: 実行コンテキスト。
  • struct cnfa: 制約の親となる NFA。
  • struct subre: 制約の詳細情報を保持。
  • struct dfa: 制約評価用の DFA 構造体。

失敗モードと境界条件

  • getladfa() が NULL を返した場合、制約評価は失敗（0）とみなされる。
  • スタック深度制限を超えた場合、エラーとして処理される。
  • 後読み制約において、プローブ位置が後退するようなケースでは再計算が発生する。

保守担当者向けメモ

  • 本関数は再帰的であるため、複雑な正規表現ではスタック消費に注意が必要。
  • matchuntil() を使用することで、後読み制約の評価における O(N^2) の計算量を抑制している。
  • FDEBUG マクロによるデバッグ出力が組み込まれている。
