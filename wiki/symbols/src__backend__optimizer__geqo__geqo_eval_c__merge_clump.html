merge_clump

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_eval.c:237

目的

GEQO（遺伝的クエリ最適化）において、結合対象のリレーション集合（Clump）を既存のリストへマージし、必要に応じて再帰的に結合を試みることで、クエリの結合順序を構築する。

シグネチャまたは形状

List *merge_clump(PlannerInfo *root, List *clumps, Clump *new_clump, int num_gene, bool force)

入力と出力

  • **入力**:
  • root: プランナのグローバル状態。
  • clumps: 現在管理されている Clump のリスト。
  • new_clump: マージ対象の新しい Clump。
  • num_gene: 遺伝子の総数。
  • force: ヒューリスティック判定をスキップして強制的に結合を試みるフラグ。
  • **出力**: マージ後の Clump リストを返す。

前提条件と不変条件

  • new_clump は clumps リストに含まれていないこと。
  • 結合が成功した場合、old_clump は更新され、new_clump は pfree される。
  • 結合が成功した際、root->join_rel_list に該当する joinrel が存在しないことが期待される。

アルゴリズムまたは振る舞い

1. clumps リストを走査し、force が真、または desirable_join が真となる old_clump を探す。 2. 結合可能な old_clump が見つかった場合：

  • make_join_rel で新しい joinrel を構築する。
  • パーティション単位の結合パス生成、Gatherパスの検討、および set_cheapest によるコスト計算を行う。
  • old_clump を更新し、new_clump を解放してリストから削除する。
  • 拡大された old_clump を用いて、自身を再帰的に呼び出し、さらなるマージを試みる。

    • 3. 結合可能な相手が見つからない場合：

  • new_clump をサイズ順（降順）に従って clumps リストの適切な位置に挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_eval.c 内の gimme_tree から呼び出される。
  • **呼び出し先**:
  • desirable_join
  • make_join_rel
  • generate_partitionwise_join_paths
  • generate_useful_gather_paths
  • set_cheapest
  • pfree
  • list_insert_nth 等のリスト操作関数

関連データ構造

  • Clump: 結合済みのリレーション集合を管理する構造体。
  • PlannerInfo: クエリ最適化の全体状態を保持する構造体。
  • RelOptInfo: 結合後のリレーション情報を保持する構造体。

失敗モードと境界条件

  • make_join_rel が NULL を返した場合（結合順序が無効な場合）、その old_clump との結合はスキップされ、リストの探索が継続される。
  • clumps が空の場合、または new_clump のサイズが 1 の場合は、リストの末尾に即座に追加される。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、スタックオーバーフローのリスクは低いが、リスト操作の効率に注意が必要。
  • root->join_rel_list を一時的に変更する副作用があるため、呼び出し元での適切なクリーンアップが前提となっている。
