PQcancelCreate

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:67

目的

既存のデータベース接続からキャンセル要求に必要な認証情報と接続先情報を抽出し、キャンセル専用の接続オブジェクト（PGcancelConn）を生成する。

シグネチャまたは形状

PGcancelConn *PQcancelCreate(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn（有効なデータベース接続オブジェクト）
  • **出力**: PGcancelConn *（キャンセル要求専用の接続オブジェクト。失敗時はエラー情報を含むオブジェクトを返し、メモリ不足時は CONNECTION_BAD 状態のオブジェクトを返す）

前提条件と不変条件

  • 引数 conn はオープン状態であること（sock が PGINVALID_SOCKET でないこと）。
  • conn にキャンセルキー（be_cancel_key）が受信済みであること。
  • 戻り値のオブジェクトは、呼び出し元が PQcancelFinish を用いて解放する責任がある。

アルゴリズムまたは振る舞い

1. pqMakeEmptyPGconn で空の接続オブジェクトを生成する。 2. 入力 conn の有効性とキャンセルキーの存在を確認する。 3. pqCopyPGconn を用いて、元の接続から基本情報をコピーする。 4. pqConnectOptions2 で派生オプションを計算する。 5. 元の接続から be_pid、be_cancel_key、pversion をコピーする。 6. キャンセル要求専用として、接続先ホストとアドレス情報を元の接続から単一要素として複製する（pqReleaseConnHosts で既存のホスト情報を解放後、手動で再構築）。 7. 接続状態を CONNECTION_ALLOCATED に設定して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/connect_utils.c、src/include/libpq/libpq-be-fe-helpers.h、src/test/isolation/isolationtester.c、src/test/modules/libpq_pipeline/libpq_pipeline.c など。
  • **呼び出し先**: pqMakeEmptyPGconn, libpq_append_conn_error, pqCopyPGconn, pqConnectOptions2, pqReleaseConnHosts, malloc, memcpy, calloc, strdup。

関連データ構造

  • PGconn: 元となるデータベース接続。
  • PGcancelConn: キャンセル要求専用の接続構造体。
  • pg_conn_host: 接続先ホスト情報を保持する構造体。

失敗モードと境界条件

  • conn が NULL または未接続の場合: エラーメッセージを設定したオブジェクトを返す。
  • キャンセルキーが未受信の場合: エラーメッセージを設定したオブジェクトを返す。
  • メモリ不足（malloc/calloc/strdup 失敗）: CONNECTION_BAD 状態のオブジェクトを返し、エラーメッセージに "out of memory" を設定する。

保守担当者向けメモ

  • 本関数は非同期キャンセルAPIの一部であり、スレッドセーフな設計となっている。
  • ホスト情報の複製において strdup を多用しているため、メモリリークを防ぐため PQcancelFinish による適切なクリーンアップが必須である。
