ExecAsyncForeignScanRequest

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeForeignscan.c:455

目的

外部データソースに対する非同期スキャンリクエストを、対応するFDW（Foreign Data Wrapper）のルーチンへ転送する。

シグネチャまたは形状

void ExecAsyncForeignScanRequest(AsyncRequest *areq)

入力と出力

  • **入力**: AsyncRequest *areq - 非同期リクエスト情報を保持する構造体。areq->requestee には対象の ForeignScanState が格納されている必要がある。
  • **出力**: なし（void）。

前提条件と不変条件

  • areq->requestee から取得した ForeignScanState 内の fdwroutine が有効であること。
  • fdwroutine->ForeignAsyncRequest 関数ポインタが NULL でないこと（Assert により検証）。

アルゴリズムまたは振る舞い

1. areq->requestee を ForeignScanState 型にキャストし、実行ノードの状態を取得する。 2. 該当ノードに関連付けられた FdwRoutine を取得する。 3. fdwroutine->ForeignAsyncRequest を呼び出し、リクエストを外部モジュールへ委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAsync.c:38 からの呼び出しが確認されている。
  • **呼び出し先**: fdwroutine->ForeignAsyncRequest（FDW実装依存の関数）。

関連データ構造

  • AsyncRequest: 非同期リクエストを管理する構造体。
  • ForeignScanState: 外部テーブルスキャンの実行状態を保持する構造体。
  • FdwRoutine: FDWが提供する関数ポインタ群を保持する構造体。

失敗モードと境界条件

  • fdwroutine->ForeignAsyncRequest が NULL の場合、アサーション違反によりプロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の非同期処理ロジックは各FDWの実装に依存する。
  • 呼び出し元情報の詳細は src/backend/executor/execAsync.c を参照のこと。
