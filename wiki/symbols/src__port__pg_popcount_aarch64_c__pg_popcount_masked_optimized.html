pg_popcount_masked_optimized

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_aarch64.c:286

目的

AArch64環境において、指定されたマスクでフィルタリングされたバイト列のビットカウント（popcount）を実行するための最適化されたインターフェースを提供します。

シグネチャまたは形状

uint64 pg_popcount_masked_optimized(const char *buf, int bytes, bits8 mask)

入力と出力

  • buf: カウント対象のバイト列へのポインタ。
  • bytes: バイト列の長さ。
  • mask: 各バイトに対して適用するビットマスク。
  • 戻り値: マスクを適用した後のビットの総数（uint64）。

前提条件と不変条件

  • POPCNT_AARCH64 が定義されている環境で有効です。
  • 本関数は、実行時に選択された最適化実装（現在は pg_popcount_masked_neon）を呼び出すラッパーとして機能します。

アルゴリズムまたは振る舞い

1. 呼び出されると、内部的に pg_popcount_masked_neon を実行します。 2. pg_popcount_masked_neon は、Neon命令セットを活用してバッファ単位の高速なビットカウントを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/pg_bitutils.h 内のインライン関数や、src/port/pg_bitutils.c 等のモジュールから参照されます。
  • **呼び出し先**: pg_popcount_masked_neon を直接呼び出します。

関連データ構造

  • bits8: マスクに使用される8ビット符号なし整数型。

失敗モードと境界条件

  • バッファサイズが0の場合の挙動は、内部の pg_popcount_masked_neon の実装に依存します。
  • CPUがNeon命令セットをサポートしていない環境での動作については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 本関数は関数ポインタとして管理されており、初期化時に pg_popcount_masked_choose によって適切な実装へ切り替えられる設計となっています。
  • SVEサポートが有効な環境では、将来的に pg_popcount_masked_sve へ動的に切り替わる可能性があります。
