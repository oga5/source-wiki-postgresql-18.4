InitRecoveryTransactionEnvironment

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/standby.c:94

目的

ホットスタンバイモードにおいて、リカバリプロセスがプライマリのロック状態を模倣し、クエリ実行バックエンドとの競合を管理するための環境を初期化します。

シグネチャまたは形状

void InitRecoveryTransactionEnvironment(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数および共有メモリ状態を更新）

前提条件と不変条件

  • RecoveryLockHash が NULL であること（二重初期化の防止）。
  • 呼び出し側はリカバリプロセス（Startup process）であること。

アルゴリズムまたは振る舞い

1. **ハッシュテーブル作成**: トランザクションごとのロックを追跡するための RecoveryLockHash および RecoveryLockXidHash を作成します。 2. **共有無効化管理の初期化**: SharedInvalBackendInit(true) を呼び出し、自身をメッセージ受信不要な sendOnly プロセスとして登録します。 3. **仮想トランザクションIDの割り当て**:

  • GetNextLocalTransactionId() を使用してローカルトランザクションIDを取得し、自身の MyProc->vxid を設定します。
  • VirtualXactLockTableInsert() を呼び出し、リカバリプロセスをロックマネージャに登録します。

    • 4. **状態更新**: グローバル変数 standbyState を STANDBY_INITIALIZED に設定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c
  • 呼び出し先: hash_create, SharedInvalBackendInit, GetNextLocalTransactionId, VirtualXactLockTableInsert

関連データ構造

  • RecoveryLockHash: ロック追跡用ハッシュテーブル
  • RecoveryLockXidHash: トランザクションID追跡用ハッシュテーブル
  • MyProc: 現在のプロセスのプロセス制御ブロック

失敗モードと境界条件

  • RecoveryLockHash が既に初期化されている場合、Assert により停止します。
  • 共有メモリの確保に失敗した場合、システムは起動を継続できません。

保守担当者向けメモ

  • 本関数はスタンバイのスタートアッププロセスが単一の仮想トランザクションとして振る舞うための基盤です。
  • XactLockTableInsert() は呼び出されません。これは、スタンバイ側のクエリは AccessShareLock を保持するため、リカバリプロセスが書き込みや行ロックを行う際にもブロックされないという設計に基づいています。
