be_gssapi_read

定義

  • **種別**: function
  • **定義箇所**: src/backend/libpq/be-secure-gssapi.c:269

目的

GSSAPIで暗号化された通信路からデータを読み取り、復号してアプリケーション層へ提供する。

シグネチャまたは形状

ssize_t be_gssapi_read(Port *port, void *ptr, size_t len)

入力と出力

  • **入力**:
  • port: 接続状態を保持する Port 構造体。
  • ptr: 読み取ったデータを格納するバッファ。
  • len: 要求するデータ長。
  • **出力**:
  • 成功時: 実際に読み取ったバイト数（ssize_t）。
  • 失敗時: -1 を返し、errno を設定する。

前提条件と不変条件

  • GSSAPI認証が完了し、port->gss->ctx が有効であること。
  • 内部バッファ PqGSSRecvBuffer および PqGSSResultBuffer を使用して状態を管理する。
  • 読み取りは固定サイズのパケット（ヘッダー＋暗号化ペイロード）単位で行われる。

アルゴリズムまたは振る舞い

1. **バッファ確認**: PqGSSResultBuffer に未読データがあれば、それを優先的に ptr へコピーして返す。 2. **パケット受信**: バッファが空の場合、secure_raw_read を用いて4バイトの長さヘッダーを読み取る。 3. **検証**: パケット長が PQ_GSS_MAX_PACKET_SIZE を超えていないか確認する。 4. **データ読み込み**: パケット全体が揃うまで secure_raw_read を繰り返す。 5. **復号**: gss_unwrap を呼び出し、暗号化データを復号して PqGSSResultBuffer に格納する。 6. **機密性確認**: 復号されたデータが機密性保護（confidentiality）を伴っているか確認する。 7. **返却**: 復号されたデータを要求されたサイズ分だけ ptr にコピーして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure.c 内のセキュア通信抽象化レイヤーから呼び出される。
  • **呼び出し先**: secure_raw_read（低レベルI/O）、gss_unwrap（GSSAPI復号）、gss_release_buffer（メモリ解放）。

関連データ構造

  • Port: 接続情報を保持。
  • PqGSSRecvBuffer / PqGSSResultBuffer: 送受信および復号結果を保持する静的バッファ。
  • PQ_GSS_MAX_PACKET_SIZE: 最大パケットサイズ制限。

失敗モードと境界条件

  • **パケット長超過**: クライアントから許容サイズを超えるパケットが送られた場合、COMMERROR をログし ECONNRESET を返す。
  • **復号失敗**: gss_unwrap が GSS_S_COMPLETE 以外を返した場合、ECONNRESET を返す。
  • **機密性欠如**: 暗号化されていないメッセージを受信した場合、COMMERROR をログし ECONNRESET を返す。
  • **非ブロッキングI/O**: データが不足している場合は EWOULDBLOCK を設定して -1 を返す。

保守担当者向けメモ

  • 本関数は静的変数を用いて接続状態を保持するため、単一のGSSAPI接続を前提としている。
  • エラー発生時は elog(COMMERROR) を使用すること（FATAL は無限再帰のリスクがあるため）。
  • パケットサイズはプロトコル仕様の一部であるため、変更時はクライアントとの互換性に注意が必要。
