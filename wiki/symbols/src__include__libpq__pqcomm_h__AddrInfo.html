AddrInfo

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/pqcomm.h:36

目的

通信アドレス情報と、そのアドレスファミリを保持するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	int			family;
	SockAddr	addr;
} AddrInfo;

入力と出力

  • **入力**: 構造体初期化時に family（アドレスファミリ）および addr（SockAddr 型の通信アドレス）を受け取ります。
  • **出力**: 接続先アドレスのリスト管理や、接続試行時のアドレス情報保持に使用されます。

前提条件と不変条件

  • family はソケット通信におけるアドレスファミリ（例: AF_INET, AF_INET6, AF_UNIX）と整合している必要があります。
  • addr フィールドは SockAddr 型であり、libpq における汎用的なソケットアドレス定義に依存します。

アルゴリズムまたは振る舞い

本構造体は、接続先候補となる複数のアドレスを配列として保持するために使用されます。接続処理において、この構造体の配列を走査することで、利用可能なアドレスに対して順次接続を試みる振る舞いをサポートします。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/interfaces/libpq/fe-cancel.c: キャンセル接続用のアドレス配列確保に使用。
  • src/interfaces/libpq/fe-connect.c: 接続先アドレスの管理および走査に使用。
  • src/interfaces/libpq/libpq-int.h: 接続状態を保持する内部構造体の一部として定義。
  • **呼び出し先**:
  • 本構造体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • SockAddr: AddrInfo 内でアドレスの実体を保持するために使用される構造体。

失敗モードと境界条件

  • メモリ確保失敗: calloc を用いて配列を確保する箇所（fe-cancel.c や fe-connect.c）において、メモリ不足時に失敗する可能性があります。
  • 不正なアドレス情報: family と addr の内容が不整合な場合、後続のソケット接続処理でエラーが発生します。

保守担当者向けメモ

  • 本構造体は libpq のクライアント側（src/interfaces/libpq/）で主に利用されています。
  • 接続先アドレスのリストを扱う際は、naddr（アドレス数）と AddrInfo の配列が正しく同期されていることを確認してください。
