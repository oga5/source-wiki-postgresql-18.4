debackslash_path

定義

  • 種別: function
  • 定義箇所: src/port/path.c:163

目的

パス文字列内のバックスラッシュ（\）をスラッシュ（/）に置換し、プラットフォーム間で一貫したパス形式へ変換します。特に Shift-JIS のようなマルチバイト文字にバックスラッシュのバイト値（0x5C）が含まれるエンコーディングを考慮した安全な置換を行います。

シグネチャまたは形状

static void debackslash_path(char *path, int encoding)

入力と出力

  • path (char *): 変換対象のパス文字列。関数内で直接書き換えられます。
  • encoding (int): パス文字列のエンコーディング（例: PG_SJIS）。
  • 出力: なし（path が直接更新されます）。

前提条件と不変条件

  • path はヌル終端文字列である必要があります。
  • Shift-JIS エンコーディングの場合、pg_sjis_mblen を用いてマルチバイト文字の境界を正しく認識し、文字の一部である 0x5C を誤って置換しないことが保証されます。

アルゴリズムまたは振る舞い

1. encoding が PG_SJIS の場合:

  • 文字列を先頭から走査し、pg_sjis_mblen を使用してマルチバイト文字の長さを加算しながら進みます。
  • 現在の文字が \（0x5C）であれば / に置換します。

    • 2. その他のエンコーディングの場合:

  • 文字列を 1 バイトずつ走査し、\ が出現するたびに / に置換します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/path.c 内の複数箇所で利用されています。
  • 呼び出し先: pg_sjis_mblen（encoding == PG_SJIS の場合）。

関連データ構造

  • PG_SJIS: エンコーディングを指定する定数。
  • pg_sjis_mblen: Shift-JIS の文字長を判定する関数。

失敗モードと境界条件

  • path が NULL の場合、未定義の動作となります。
  • Shift-JIS 以外のマルチバイトエンコーディングで 0x5C を含むものがある場合、本関数はそれを誤って置換する可能性があります（現状は Shift-JIS のみを特別扱いしています）。

保守担当者向けメモ

  • 本関数は wchar.c の汎用的なエンコーディング処理を呼び出すのが実用的でないため、個別のケースとして実装されています。他のマルチバイトエンコーディングで 0x5C 問題が発生する場合は、本関数の条件分岐に追加が必要です。
