pg_comp_crc32c_choose

定義

  • 種別: function
  • 定義箇所: src/port/pg_crc32c_armv8_choose.c:115

目的

ARMv8 CPU における CRC-32C 計算の最適化実装をランタイムで判定し、グローバルな関数ポインタを適切な実装へ動的に切り替える。

シグネチャまたは形状

static pg_crc32c pg_comp_crc32c_choose(pg_crc32c crc, const void *data, size_t len);

入力と出力

  • 入力:
  • crc: 初期 CRC 値。
  • data: 計算対象のデータバッファへのポインタ。
  • len: データバッファの長さ。
  • 出力:
  • 計算された CRC-32C 値を返します。

前提条件と不変条件

  • 本関数は、グローバル関数ポインタ pg_comp_crc32c が初期化された直後の初回呼び出し時に実行されることを前提としています。
  • 実行環境が ARMv8 アーキテクチャであることを想定しています。

アルゴリズムまたは振る舞い

1. pg_crc32c_armv8_available() を呼び出し、CPU が CRC 拡張命令をサポートしているかを確認します。 2. サポートされている場合は pg_comp_crc32c を pg_comp_crc32c_armv8 に、そうでない場合は pg_comp_crc32c_sb8（ソフトウェア実装）に再割り当てします。 3. 再割り当てされた関数を呼び出し、その結果を返します。 4. 以降の呼び出しでは pg_comp_crc32c が直接実装関数を指すため、本関数はスキップされます。

呼び出し元と呼び出し先

  • 呼び出し元: CRC-32C 計算を要求する任意のコード（pg_comp_crc32c ポインタ経由）。
  • 呼び出し先:
  • pg_crc32c_armv8_available()（判定用）
  • pg_comp_crc32c_armv8 または pg_comp_crc32c_sb8（切り替え先の実装）

関連データ構造

  • pg_comp_crc32c: CRC-32C 計算を行うグローバルな関数ポインタ。

失敗モードと境界条件

  • OS 固有の機能判定（getauxval や sysctl）が失敗した場合の挙動については、本関数のソースコード内には明示的なエラーハンドリングの記述はありません。

保守担当者向けメモ

  • 本関数は「Function Pointer Swapping」パターンを採用しています。デバッグ時には、初回呼び出し後に pg_comp_crc32c がどの関数を指しているかを確認してください。
  • pg_comp_crc32c_armv8 と pg_comp_crc32c_sb8 の両方が利用可能であることを前提としたロジックです。
