px_hmac

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/px.h:129

目的

HMAC（Hash-based Message Authentication Code）操作を抽象化するためのインターフェース構造体です。具体的なハッシュアルゴリズムの実装詳細を隠蔽し、ポリモーフィックな呼び出しを可能にします。

シグネチャまたは形状

struct px_hmac {
    unsigned (*result_size) (PX_HMAC *h);
    unsigned (*block_size) (PX_HMAC *h);
    void     (*reset) (PX_HMAC *h);
    void     (*update) (PX_HMAC *h, const uint8 *data, unsigned dlen);
    void     (*finish) (PX_HMAC *h, uint8 *dst);
    void     (*free) (PX_HMAC *h);
    void     (*init) (PX_HMAC *h, const uint8 *key, unsigned klen);

    PX_MD    *md;
    struct {
        uint8 *ipad;
        uint8 *opad;
    } p;
};

入力と出力

  • **入力**: 各関数ポインタは PX_HMAC 構造体へのポインタを第一引数として受け取ります。init は鍵データとその長さを、update は入力データとその長さを、finish は結果格納先のバッファを受け取ります。
  • **出力**: result_size および block_size は unsigned 型のサイズを返します。その他の関数は void を返します。

前提条件と不変条件

  • md メンバは、HMACの基盤となるハッシュアルゴリズムを保持する PX_MD 構造体へのポインタである必要があります。
  • p メンバ（ipad, opad）は内部的なプライベート領域として使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: init 関数により、指定された鍵を用いてHMACコンテキストを初期化します。 2. **更新**: update 関数を繰り返し呼び出すことで、メッセージデータを処理します。 3. **終了**: finish 関数により、計算されたMAC値を dst バッファに書き込みます。 4. **リセット**: reset 関数により、現在の状態を初期化し、再利用可能な状態にします。 5. **解放**: free 関数により、構造体および内部リソースを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • 呼び出し先: PX_MD 構造体（md メンバ）を介してハッシュアルゴリズムの機能を利用します。

関連データ構造

  • PX_HMAC: struct px_hmac の typedef 名（contrib/pgcrypto/px.h:101）。
  • PX_MD: HMACの基盤となるメッセージダイジェスト構造体。

失敗モードと境界条件

  • 関数ポインタが適切に初期化されていない場合、呼び出し時に未定義の動作が発生する可能性があります。
  • 内部的なメモリ割り当て（ipad, opad）の失敗については、本構造体定義からは直接的なエラーハンドリングの詳細は不明です。

保守担当者向けメモ

  • 本構造体は関数ポインタによるポリモーフィズムを採用しています。新しいハッシュアルゴリズムを追加する際は、各関数ポインタに対応する実装を提供してください。
  • p メンバはプライベート領域として定義されているため、外部からの直接操作は避けてください。
