get_controlfile

定義

  • 種別: function
  • 定義箇所: src/common/controldata_utils.c:51

目的

データベースのデータディレクトリから制御ファイル（global/pg_control）を読み込み、整合性を検証した上で構造体として取得します。

シグネチャまたは形状

ControlFileData *get_controlfile(const char *DataDir, bool *crc_ok_p);

入力と出力

  • **入力**:
  • DataDir: データベースのデータディレクトリへのパス。
  • crc_ok_p: CRCチェックの結果を格納するためのブール値へのポインタ。
  • **出力**:
  • 読み込んだ制御データを保持する ControlFileData 構造体へのポインタ。

前提条件と不変条件

  • 指定された DataDir 内に global/pg_control が存在する必要があります。
  • 呼び出し元は、返された ControlFileData 構造体のメモリ解放を行う責任があります。

アルゴリズムまたは振る舞い

1. DataDir と XLOG_CONTROL_FILE を結合し、制御ファイルのフルパスを生成します。 2. 生成したパスを引数として get_controlfile_by_exact_path を呼び出し、実際の読み込みとCRC検証を実行します。 3. フロントエンド実行時には、並行書き込みによる読み込み不整合を回避するため、CRC不一致時に最大10回のリトライを行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/pg_controldata.c
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_checksums/pg_checksums.c
  • src/bin/pg_controldata/pg_controldata.c
  • src/bin/pg_ctl/pg_ctl.c
  • **呼び出し先**:
  • get_controlfile_by_exact_path (同一ファイル内)

関連データ構造

  • ControlFileData: 制御ファイルの内容を保持する構造体（catalog/pg_control.h で定義）。

失敗モードと境界条件

  • ファイルが存在しない、または読み取り権限がない場合、エラーが発生します。
  • CRCチェックが失敗した場合、crc_ok_p に false が設定されます。
  • フロントエンドとバックエンドでエラー時の挙動が異なり、バックエンドでは ereport(PANIC, ...)、フロントエンドでは pg_fatal が使用されます。

保守担当者向けメモ

  • 本関数はパスの構築のみを行い、実際のファイル操作は get_controlfile_by_exact_path に委譲しています。
  • FRONTEND マクロの定義有無によりエラーハンドリングの挙動が切り替わるため、修正時は両方の環境への影響を考慮してください。
