pg_gen_salt_rounds

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:189

目的

指定されたアルゴリズム名と反復回数（rounds）に基づき、暗号化用のソルト文字列を生成します。

シグネチャまたは形状

Datum pg_gen_salt_rounds(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • arg0 (text): ソルト生成に使用するアルゴリズム名。
  • rounds (int32): アルゴリズムに適用する反復回数。
  • **出力**:
  • Datum: 生成されたソルト文字列（text 型）。

前提条件と不変条件

  • buf は PX_MAX_SALT_LEN + 1 のサイズで確保されており、アルゴリズム名がこのバッファに収まる必要があります。
  • px_gen_salt が負の値を返した場合、エラーとして処理されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP でアルゴリズム名、PG_GETARG_INT32 で反復回数を取得します。 2. text_to_cstring_buffer を使用して、アルゴリズム名をローカルバッファ buf にコピーします。 3. px_gen_salt(buf, buf, rounds) を呼び出し、ソルトを生成します。 4. 生成結果が負の値（エラー）であれば、px_strerror を用いてエラーメッセージを構築し、ereport(ERROR, ...) で中断します。 5. 正常終了時、PG_FREE_IF_COPY で入力引数のメモリを解放し、cstring_to_text_with_len で結果を text 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • px_gen_salt (内部ライブラリ)
  • px_strerror (エラーメッセージ取得用)
  • PostgreSQL のメモリ管理および型変換関数 (PG_GETARG_*, PG_FREE_IF_COPY, cstring_to_text_with_len)

関連データ構造

  • PX_MAX_SALT_LEN: ソルトバッファの最大長を定義する定数。

失敗モードと境界条件

  • px_gen_salt が失敗した場合（アルゴリズムが不明、または反復回数が不適切な場合など）、ERRCODE_INVALID_PARAMETER_VALUE を伴う SQL エラーが発生します。

保守担当者向けメモ

  • 本関数は SQL レイヤーと px レイヤーの仲介役です。アルゴリズム固有のソルト生成ロジックについては px_gen_salt の実装（px.c 等）を参照してください。
