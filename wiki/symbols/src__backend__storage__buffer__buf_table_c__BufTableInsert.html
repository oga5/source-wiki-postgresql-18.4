BufTableInsert

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/buf_table.c:117

目的

共有バッファハッシュテーブル（SharedBufHash）に、指定された BufferTag と buffer ID のペアを登録します。

シグネチャまたは形状

int BufTableInsert(BufferTag *tagPtr, uint32 hashcode, int buf_id)

入力と出力

  • **入力**:
  • tagPtr: 登録対象のページを識別する BufferTag へのポインタ。
  • hashcode: tagPtr から事前に計算されたハッシュ値。
  • buf_id: 割り当てるバッファID（0以上であること）。
  • **出力**:
  • 既にテーブル内にタグが存在した場合は、既存の buf_id を返します。
  • 新規登録に成功した場合は -1 を返します。

前提条件と不変条件

  • buf_id は 0 以上であること（-1 はテーブル未登録を示す予約値）。
  • tagPtr->blockNum は P_NEW ではないこと（無効なタグの挿入禁止）。
  • 本関数はロックを保持しないため、呼び出し側が事前に適切な BufMappingLock を取得していること。

アルゴリズムまたは振る舞い

1. hash_search_with_hash_value を使用して、共有ハッシュテーブルから tagPtr を検索（HASH_ENTER モード）。 2. 既にエントリが存在する場合（found が真）、そのエントリの id を返して終了。 3. エントリが存在しない場合、新規作成されたエントリの id フィールドに buf_id を設定し、-1 を返して終了。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c: バッファの読み込みや置換処理において呼び出されます。
  • **呼び出し先**:
  • hash_search_with_hash_value: 共有ハッシュテーブルの操作を行う内部関数。

関連データ構造

  • BufferLookupEnt: ハッシュテーブル内のエントリ構造体。
  • SharedBufHash: バッファマッピングを保持する共有ハッシュテーブル。
  • BufferTag: ページを一意に識別するキー。

失敗モードと境界条件

  • buf_id が負の値の場合、Assert により異常終了します。
  • tagPtr が無効な場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数はアトミックな操作を保証するため、呼び出し側での BufMappingLock の管理が必須です。
  • 戻り値が -1 かどうかで、挿入が新規に行われたか、既存のバッファが再利用されたかを判別します。
