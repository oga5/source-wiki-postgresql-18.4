ChooseRelationName

定義

  • 種別: function
  • 定義箇所: src/backend/commands/indexcmds.c:2607

目的

指定された名前の構成要素から一意なリレーション名（またはインデックス名）を生成します。既存の同名リレーションとの衝突を回避するため、必要に応じてサフィックス（数値）を付加します。

シグネチャまたは形状

char *ChooseRelationName(const char *name1, const char *name2,
                         const char *label, Oid namespaceid,
                         bool isconstraint)

入力と出力

  • **入力**:
  • name1, name2: 名前生成のベースとなる文字列。
  • label: リレーションの役割を示すラベル。
  • namespaceid: 検索対象のスキーマ（名前空間）のOID。
  • isconstraint: 制約名としての一意性チェックも行うかどうかのフラグ。
  • **出力**:
  • 衝突しないことが確認された、新しく割り当てられたリレーション名の文字列（呼び出し元で pfree が必要）。

前提条件と不変条件

  • pg_class システムカタログに対する AccessShareLock を保持した状態で呼び出される必要があります。
  • InitDirtySnapshot を使用して、コミットされていない変更を含む可能性のあるスナップショットで検索を行います。

アルゴリズムまたは振る舞い

1. makeObjectName を使用して、name1, name2, label から候補名を生成します。 2. pg_class をスキャンし、指定された namespaceid 内で同名の既存リレーションが存在するか確認します。 3. 衝突がある場合、label にカウンタ（pass）を付加した文字列を生成し、衝突がなくなるまで繰り返します。 4. isconstraint が true の場合、ConstraintNameExists を呼び出し、制約名としての重複もチェックします。 5. 最終的に衝突のない名前を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/indexcmds.c 内のインデックス作成処理（DefineIndex 等）や、src/backend/parser/parse_utilcmd.c などで利用されています。
  • **呼び出し先**: makeObjectName, table_open, systable_beginscan, systable_getnext, systable_endscan, ConstraintNameExists, table_close 等。

関連データ構造

  • SnapshotData: 検索に使用するスナップショット。
  • Relation: pg_class リレーションのハンドル。
  • ScanKeyData: pg_class 検索用のキー。

失敗モードと境界条件

  • 名前が長すぎる場合、makeObjectName の仕様に依存します。
  • 非常に多くの同名リレーションが存在する場合、ループ回数が増加しパフォーマンスに影響を与える可能性があります。

保守担当者向けメモ

  • 本関数は pg_class を直接スキャンするため、ロックの競合やデッドロックに注意してください。
  • isconstraint フラグによる制約名チェックは、インデックス名と制約名が同一名前空間で衝突することを防ぐための重要な安全策です。
