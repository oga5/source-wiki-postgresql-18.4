GetSchemaPublicationRelations

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_publication.c:988

目的

指定されたスキーマ内に存在するすべてのリレーション（テーブル）を列挙し、パブリケーション対象として適切なものをリストとして取得する。

シグネチャまたは形状

List *GetSchemaPublicationRelations(Oid schemaid, PublicationPartOpt pub_partopt)

入力と出力

  • **入力**:
  • schemaid (Oid): 対象とするスキーマのOID。
  • pub_partopt (PublicationPartOpt): パーティションテーブルを扱う際のオプション。
  • **出力**:
  • List *: スキーマ内のパブリッシュ可能なリレーションOIDのリスト。

前提条件と不変条件

  • schemaid は有効なOIDである必要がある（Assert(OidIsValid(schemaid))）。
  • pg_class システムカタログに対する AccessShareLock を保持してスキャンを行う。

アルゴリズムまたは振る舞い

1. pg_class カタログを relnamespace インデックスを用いてスキャンし、指定された schemaid に属する全リレーションを取得する。 2. 各リレーションに対し is_publishable_class を呼び出し、パブリケーション対象として妥当か判定する。 3. リレーションの種類（relkind）に応じて以下の処理を行う:

  • 通常のテーブル (RELKIND_RELATION): リストに追加する。
  • パーティションテーブル (RELKIND_PARTITIONED_TABLE): GetPubPartitionOptionRelations を呼び出し、pub_partopt に基づいて解決されたパーティション群をリストに結合する。

    • 4. スキャン終了後、ロックを解放して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c 内の複数箇所（パブリケーションのスキーマ紐付け処理など）。
  • src/backend/commands/publicationcmds.c:1644
  • **呼び出し先**:
  • table_open, table_beginscan_catalog, heap_getnext, table_endscan, table_close
  • is_publishable_class
  • get_rel_relkind
  • GetPubPartitionOptionRelations
  • lappend_oid, list_concat_unique_oid

関連データ構造

  • List: OIDのリストを保持する。
  • ScanKeyData: pg_class スキャン用の検索キー。
  • Form_pg_class: pg_class タプルの構造体。

失敗モードと境界条件

  • 指定された schemaid が存在しない場合、空のリストが返される。
  • is_publishable_class により除外されたリレーション（システムテーブルや一時テーブルなど）は結果に含まれない。
  • パーティションテーブルの構成要素が異なるスキーマに存在する場合でも、GetPubPartitionOptionRelations を通じて適切に収集される。

保守担当者向けメモ

  • 本関数は pg_class を直接スキャンするため、カタログの変更が頻繁な環境ではパフォーマンスに注意が必要。
  • パーティションの解決ロジックは GetPubPartitionOptionRelations に依存しているため、パーティション階層の挙動変更時は当該関数と併せて確認すること。
