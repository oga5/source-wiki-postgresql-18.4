brin_start_evacuating_page

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/brin/brin_pageops.c:523

目的

BRINインデックスページを再利用または削除する準備として、当該ページへの新規タプル追加を禁止するためのフラグ設定を行います。

シグネチャまたは形状

bool brin_start_evacuating_page(Relation idxRel, Buffer buf)

入力と出力

  • **入力**:
  • idxRel: 対象のインデックスリレーション。
  • buf: 退避対象のページを含むバッファ。
  • **出力**:
  • bool: ページ内に有効なタプルが存在し、退避フラグが設定された場合は true、ページが空または新規作成直後の場合は false を返します。

前提条件と不変条件

  • buf は呼び出し元によって適切にロックされている必要があります。
  • ページが新規（PageIsNew）である場合、処理は行われず false を返します。

アルゴリズムまたは振る舞い

1. バッファからページを取得し、新規ページであれば終了します。 2. ページ内の全オフセットを走査し、使用中のタプル（ItemIdIsUsed）を検索します。 3. 有効なタプルが一つでも見つかった場合:

  • ページヘッダーのフラグに BRIN_EVACUATE_PAGE を付与します。
  • MarkBufferDirtyHint(buf, true) を呼び出し、変更を記録します。
  • true を返して終了します。

    • 4. 有効なタプルが一つも存在しない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_revmap.c:589
  • **呼び出し先**:
  • BufferGetPage
  • PageIsNew
  • PageGetMaxOffsetNumber
  • PageGetItemId
  • ItemIdIsUsed
  • BrinPageFlags (マクロ)
  • MarkBufferDirtyHint

関連データ構造

  • Buffer: PostgreSQLのバッファ管理構造体。
  • Page: インデックスページのメモリレイアウト。
  • BRIN_EVACUATE_PAGE: ページヘッダーのフラグビット。

失敗モードと境界条件

  • ページが空の場合、フラグは設定されず false が返されます。
  • このフラグ設定自体はWALログに記録されません（偶発的な記録を除く）。

保守担当者向けメモ

  • 本関数が設定する BRIN_EVACUATE_PAGE フラグは、br_page_get_freespace が当該ページを空き領域として認識しないように制御するために使用されます。
  • ページ退避の完全なプロセスについては、関連する brin_evacuate_page を参照してください。
