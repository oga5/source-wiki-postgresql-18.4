InitPlanCache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/plancache.c:145

目的

バックエンドの初期化プロセスにおいて、クエリ実行計画のキャッシュ無効化（Invalidation）に必要なコールバック関数をシステムへ登録する。

シグネチャまたは形状

void InitPlanCache(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてキャッシュ無効化コールバックが登録される）

前提条件と不変条件

  • InitPostgres の実行中に呼び出される必要がある。
  • 本関数が実行されることで、以降のシステムカタログ（リレーション、プロシージャ、型、演算子等）の変更イベントが plancache.c 内の適切なハンドラに通知されるようになる。

アルゴリズムまたは振る舞い

以下の各キャッシュ無効化イベントに対して、対応するコールバック関数を登録する： 1. **リレーション**: CacheRegisterRelcacheCallback を使用し PlanCacheRelCallback を登録。 2. **システムカタログ**: CacheRegisterSyscacheCallback を使用し、以下のOIDに対して PlanCacheObjectCallback または PlanCacheSysCallback を登録。

  • PROCOID (プロシージャ)
  • TYPEOID (型)
  • NAMESPACEOID (名前空間)
  • OPEROID (演算子)
  • AMOPOPID (アクセス手法演算子)
  • FOREIGNSERVEROID (外部サーバー)
  • FOREIGNDATAWRAPPEROID (外部データラッパー)

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c (InitPostgres内)
  • 呼び出し先: CacheRegisterRelcacheCallback, CacheRegisterSyscacheCallback

関連データ構造

  • PlanCacheRelCallback (コールバック関数)
  • PlanCacheObjectCallback (コールバック関数)
  • PlanCacheSysCallback (コールバック関数)

失敗モードと境界条件

  • 本関数自体はメモリ不足等の致命的なエラーを除き、正常に完了する。
  • 登録されるコールバック関数は、キャッシュ無効化イベント発生時に plancache.c 内の CachedPlanSource の有効性フラグを適切に更新する責務を負う。

保守担当者向けメモ

  • 新たなシステムカタログの変更が計画キャッシュに影響を与えるようになった場合、本関数に CacheRegisterSyscacheCallback を追加する必要がある。
