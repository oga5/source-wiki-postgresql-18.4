spgConfigIn

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist.h:36

目的

SP-GiSTインデックスの演算子クラス（opclass）における設定関数（config サポート関数）の入力パラメータを保持する構造体です。

シグネチャまたは形状

typedef struct spgConfigIn
{
    Oid    attType;    /* インデックス対象のデータ型 */
} spgConfigIn;

入力と出力

  • **入力**: インデックス作成時や検証時に、演算子クラスの config 関数へ渡される引数です。
  • **出力**: 本構造体自体は入力専用のコンテナであり、設定結果は対応する spgConfigOut 構造体に格納されます。

前提条件と不変条件

  • attType は、インデックスが構築されるカラムのデータ型を示す有効な Oid である必要があります。

アルゴリズムまたは振る舞い

SP-GiSTの演算子クラスが提供する config 関数は、この構造体を通じてインデックス対象のデータ型を受け取り、その型に応じたインデックスの構成情報（リーフノードの型やプレフィックスの有無など）を spgConfigOut に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgutils.c
  • src/backend/access/spgist/spgvalidate.c
  • 各種演算子クラスの実装（spgkdtreeproc.c, spgquadtreeproc.c, spgtextproc.c, network_spgist.c, rangetypes_spgist.c, spgist_name_ops.c 等）において、PG_GETARG_POINTER(0) を介して取得されるケースが確認されています。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • spgConfigOut: config 関数の出力側構造体。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • attType が無効な場合、演算子クラスの config 関数内で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体はSP-GiSTの演算子クラス定義の根幹に関わるため、変更時は spgConfigOut との整合性に注意してください。
  • 各演算子クラスの config 関数がこの構造体をどのように利用しているかについては、各 proc.c ファイルの実装を参照してください。
