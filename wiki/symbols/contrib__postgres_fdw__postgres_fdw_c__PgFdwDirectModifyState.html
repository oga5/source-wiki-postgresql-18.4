PgFdwDirectModifyState

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/postgres_fdw.c:220

目的

外部テーブルに対する直接的な更新（UPDATE/DELETE）操作において、リモートサーバーへのクエリ実行状態、パラメータ管理、および結果セットの保持を行うための状態管理構造体です。

シグネチャまたは形状

typedef struct PgFdwDirectModifyState {
    Relation    rel;
    AttInMetadata *attinmeta;
    char       *query;
    bool        has_returning;
    List       *retrieved_attrs;
    bool        set_processed;
    PGconn     *conn;
    PgFdwConnState *conn_state;
    int         numParams;
    FmgrInfo   *param_flinfo;
    List       *param_exprs;
    const char **param_values;
    PGresult   *result;
    int         num_tuples;
    int         next_tuple;
    MemoryContextCallback result_cb;
    Relation    resultRel;
    AttrNumber *attnoMap;
    AttrNumber  ctidAttno;
    AttrNumber  oidAttno;
    bool        hasSystemCols;
    MemoryContext temp_cxt;
} PgFdwDirectModifyState;

入力と出力

  • **入力**: fdw_private データから抽出されたクエリ文字列、パラメータ式、および対象テーブルのメタデータ。
  • **出力**: リモート実行結果のタプルセット（PGresult 経由）、および RETURNING 句がある場合は変換後のタプルスロット。

前提条件と不変条件

  • temp_cxt はタプルごとの一時データ処理に使用される。
  • result_cb は PGresult の解放を保証するために設定される。
  • conn はリモートサーバーへの有効な接続を指している必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: palloc0 によりメモリを確保し、fdw_state として保持。 2. **クエリ実行**: query 文字列と param_values を使用してリモートサーバーへコマンドを送信。 3. **結果処理**: PGresult を取得し、num_tuples と next_tuple を用いて結果セットを反復処理する。 4. **RETURNING 処理**: has_returning が真の場合、apply_returning_filter を通じてリモートから返された値をローカルのタプル形式へ変換する。

呼び出し元と呼び出し先

  • **呼び出し元**: postgres_fdw.c 内の実行関数（postgresExecForeignDelete 等）が node->fdw_state からこの構造体を取得して操作する。
  • **呼び出し先**: init_returning_filter および apply_returning_filter が本構造体を引数として受け取り、RETURNING 句の処理を行う。

関連データ構造

  • PgFdwConnState: リモート接続に関する追加状態。
  • PGconn, PGresult: libpq の接続および結果オブジェクト。
  • AttInMetadata: データ型変換メタデータ。

失敗モードと境界条件

  • リモート接続が切断された場合、conn を介したクエリ実行が失敗する。
  • RETURNING 句の属性マッピング（attnoMap）が不整合な場合、タプル変換時にエラーが発生する可能性がある。

保守担当者向けメモ

  • 本構造体は postgres_fdw の直接更新パス専用である。
  • result_cb によるメモリ解放のライフサイクル管理が適切に行われているか確認すること。
  • システム列（ctid, oid）の扱いについては hasSystemCols フラグに依存しているため、変更時は注意が必要。
