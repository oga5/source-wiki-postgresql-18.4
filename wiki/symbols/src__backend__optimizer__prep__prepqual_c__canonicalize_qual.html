canonicalize_qual

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepqual.c:292

目的

クエリの最適化前段階において、論理式（Qualification）を最適化器が扱いやすい形式へ整理・簡略化するメインエントリポイントです。

シグネチャまたは形状

Expr *canonicalize_qual(Expr *qual, bool is_check)

入力と出力

  • **入力**:
  • qual: 最適化対象の式ツリー。
  • is_check: CHECK 制約の場合は true、WHERE 句の場合は false を指定。NULL 値の扱い（FALSE 扱いか TRUE 扱いか）を制御します。
  • **出力**:
  • 最適化・簡略化された Expr 構造体。入力が NULL の場合は NULL を返します。

前提条件と不変条件

  • 入力 qual は、暗黙的な AND 形式（List ノード）であってはなりません（Assert により検証）。
  • eval_const_expressions による定数畳み込みが完了した後の式に対して実行されることを想定しています。

アルゴリズムまたは振る舞い

1. **空チェック**: qual が NULL の場合は即座に NULL を返します。 2. **冗長性の排除**: find_duplicate_ors を呼び出し、トップレベルの AND/OR 構造内において以下の処理を行います。

  • ネストされた OR-of-AND 構造内の冗長なサブ句を抽出（分配法則の逆を適用）。
  • 不要な NULL 定数を削除。

    • 3. **結果の返却**: 整理された新しい式ツリーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/partition.c
  • src/backend/commands/copy.c
  • src/backend/commands/tablecmds.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/plan/subselect.c
  • src/backend/optimizer/util/plancat.c
  • **呼び出し先**:
  • find_duplicate_ors (直接の内部処理)

関連データ構造

  • Expr: PostgreSQLの式ノードの基底型。

失敗モードと境界条件

  • List ノードが渡された場合、Assert によりプログラムが停止します。
  • is_check フラグの誤設定は、NULL 値の評価結果を誤らせる可能性があるため注意が必要です。

保守担当者向けメモ

  • 本関数は「正規形」への変換ではなく、最適化器が効率的に走査できる構造への整理を目的としています。
  • pull_ands や pull_ors は本関数から呼び出される find_duplicate_ors の内部で必要に応じて使用されます。
