verifybackup_per_wal_range_cb

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:578

目的

backup_manifest ファイルの解析中に検出された WAL 範囲情報を、manifest_data 構造体内のリストへ追加・保持するために使用されるコールバック関数です。

シグネチャまたは形状

static void verifybackup_per_wal_range_cb(JsonManifestParseContext *context, TimeLineID tli, XLogRecPtr start_lsn, XLogRecPtr end_lsn)

入力と出力

  • **入力**:
  • context: マニフェスト解析のコンテキスト（manifest_data を保持）。
  • tli: タイムライン ID。
  • start_lsn: WAL 範囲の開始 LSN。
  • end_lsn: WAL 範囲の終了 LSN。
  • **出力**: なし（context->private_data が指す manifest_data 構造体のリストを更新）。

前提条件と不変条件

  • context->private_data は manifest_data 型の有効なポインタである必要があります。
  • manifest_data 構造体内の first_wal_range および last_wal_range は、リストの末尾への追加を正しく維持するように更新されます。

アルゴリズムまたは振る舞い

1. palloc を使用して新しい manifest_wal_range 構造体をメモリ上に確保します。 2. 引数として渡された tli、start_lsn、end_lsn を構造体に設定します。 3. リストの双方向リンク（prev および next）を適切に設定します。 4. manifest_data 内のリスト末尾（last_wal_range）を更新し、必要に応じてリストの先頭（first_wal_range）を初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_verifybackup/pg_verifybackup.c 内で、マニフェスト解析コンテキストのコールバックとして設定されています（context.per_wal_range_cb = verifybackup_per_wal_range_cb;）。
  • **呼び出し先**: palloc（メモリ確保）。

関連データ構造

  • manifest_data: マニフェスト全体の情報を保持する構造体。
  • manifest_wal_range: WAL 範囲情報を保持する連結リストのノード。
  • JsonManifestParseContext: マニフェスト解析のコンテキスト構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、処理が中断される可能性があります。
  • リストが空の状態（first_wal_range == NULL）から最初の要素を追加するケースを正しく処理します。

保守担当者向けメモ

  • 本関数は parse_manifest_file が実行する JSON 解析プロセスの一部として呼び出されます。
  • リストの末尾への追加処理は、manifest_data の状態を直接変更するため、解析ロジックとの整合性に注意してください。
