inet

定義

  • 種別: struct
  • 定義箇所: src/include/utils/inet.h:52

目的

PostgreSQLにおいて、ネットワークアドレス（inet型）を可変長データ（varlena）として格納するための内部表現を提供します。

シグネチャまたは形状

typedef struct
{
	char		vl_len_[4];
	inet_struct inet_data;
} inet;

入力と出力

  • 本構造体は varlena 形式のデータとして扱われます。
  • vl_len_ フィールドは可変長データのヘッダーサイズを保持します。
  • inet_data フィールドは inet_struct 型のネットワークアドレス情報を保持します。

前提条件と不変条件

  • vl_len_ フィールドを直接操作してはなりません。サイズ設定には SET_INET_VARSIZE マクロを使用する必要があります。
  • inet 型の属性（ファミリー、ビット数、アドレス）へのアクセスには、提供されている抽象化マクロ（ip_family, ip_bits, ip_addr 等）を使用してください。
  • メモリ確保時は palloc0 等を用いてゼロ初期化を行い、VARDATA_ANY がヘッダーを正しく認識できるようにする必要があります。

アルゴリズムまたは振る舞い

  • 1バイトヘッダーの varlena 形式を優先しますが、既存コードが期待する4バイトヘッダー形式にも対応可能な設計となっています。
  • PG_GETARG_INET_PP マクロを介して SQL 層の Datum からポインタへ変換され、処理後に PG_RETURN_INET_P で Datum へ戻されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/brin/brin_minmax_multi.c において、PG_GETARG_INET_PP を介して inet 値を取得する例が確認されています。
  • 呼び出し先: 本構造体は inet_struct を内包しており、ネットワーク操作関数（src/backend/utils/adt/network.c 等）によって inet_data フィールドが操作されます。

関連データ構造

  • inet_struct: ネットワークアドレスのコア構造体。
  • varlena: PostgreSQL の可変長データ型基底。

失敗モードと境界条件

  • family フィールドが正しく設定されていない状態で ip_addrsize() や SET_INET_VARSIZE() を呼び出すと、不正なメモリ操作やサイズ計算エラーを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は fmgr.h に依存しています。
  • vl_len_ フィールドの直接操作は厳禁です。
  • inet 型の操作には、必ず src/include/utils/inet.h で定義されたマクロインターフェースを介してください。
