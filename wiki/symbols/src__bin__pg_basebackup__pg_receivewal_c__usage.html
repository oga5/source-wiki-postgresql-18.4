usage

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_receivewal.c:74

目的

pg_receivewal コマンドの利用方法（ヘルプメッセージ）を標準出力に表示し、プログラムを終了させるための関数です。

シグネチャまたは形状

static void usage(void);

入力と出力

  • 入力: なし（グローバル変数 progname を参照）
  • 出力: なし（標準出力へのメッセージ出力のみ）

前提条件と不変条件

  • progname が適切に初期化されている必要があります。
  • PACKAGE_BUGREPORT、PACKAGE_NAME、PACKAGE_URL がビルド時に定義されている必要があります。

アルゴリズムまたは振る舞い

1. プログラム名を含む概要を表示します。 2. 利用可能なコマンドラインオプション（ディレクトリ指定、終了LSN、接続設定、スロット管理、圧縮設定など）を分類して表示します。 3. バグ報告先およびプロジェクトのホームページURLを表示します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内には直接の呼び出し元は記載されていません。
  • 呼び出し先: printf を使用して標準出力へ書き込みます。
  • 注意: ツールが検出した他のファイル（contrib/ 等）の「usage」という単語を含む参照は、本関数とは無関係な用語としての言及であり、本関数の呼び出し元ではありません。

関連データ構造

  • なし

失敗モードと境界条件

  • 標準出力への書き込みが失敗した場合の挙動は printf の仕様に依存します。

保守担当者向けメモ

  • 新しいコマンドラインオプションを追加した際は、本関数にも対応する説明文を追加してください。
  • standby_message_timeout のデフォルト値が変更された場合は、表示されるデフォルト値の計算式も適宜更新してください。
