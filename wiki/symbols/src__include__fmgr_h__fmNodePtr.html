fmNodePtr

定義

  • 種別: typedef (struct Node へのポインタ)
  • 定義箇所: src/include/fmgr.h:22

目的

fmgr.h 内で nodes.h をインクルードすることなく、抽象的なノード構造体へのポインタを扱うための型エイリアスです。

シグネチャまたは形状

typedef struct Node *fmNodePtr;

入力と出力

  • 入力: Node 構造体へのポインタ。
  • 出力: Node 構造体へのポインタ。

前提条件と不変条件

  • fmgr.h において Node 構造体の完全な定義は提供されません。
  • 本型を使用するコードは、必要に応じて適切なヘッダーで Node の定義を参照する必要があります。

アルゴリズムまたは振る舞い

  • Node 構造体へのポインタを保持する単なる型定義であり、それ自体に実行時のアルゴリズムは存在しません。
  • 主に FmgrInfo や FunctionCallInfo 構造体内で、式ツリーやコンテキスト情報を保持するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/fmgr.h 内の FmgrInfo 構造体（fn_expr メンバ）および FunctionCallInfoBaseData 構造体（context, resultinfo メンバ）。
  • 呼び出し先: src/backend/utils/fmgr/fmgr.c における関数呼び出しの引数や、get_call_expr_argtype 等のユーティリティ関数。

関連データ構造

  • Node: PostgreSQL の構文解析ツリーや実行計画ツリーの基底構造体。
  • FmgrInfo: 関数呼び出しのメタデータを保持する構造体。
  • FunctionCallInfo: 関数呼び出し時のコンテキストを保持する構造体。

失敗モードと境界条件

  • NULL ポインタを保持する可能性があります（例: fn_expr が式を持たない場合）。
  • 型定義であるため、本シンボル自体に失敗モードはありませんが、参照先が NULL である場合にデリファレンスを行うとクラッシュします。

保守担当者向けメモ

  • fmgr.h の依存関係を最小化するために導入されています。Node 構造体の定義が必要な場合は、適切なソースファイルで nodes/nodes.h をインクルードしてください。
