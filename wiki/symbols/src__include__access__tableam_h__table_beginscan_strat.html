table_beginscan_strat

定義

  • 種別: function
  • 定義箇所: src/include/access/tableam.h:898

目的

テーブルスキャンを開始するための拡張APIを提供します。標準の table_beginscan と比較して、スキャン戦略（allow_strat）や同期スキャン（allow_sync）の有効・無効を明示的に制御可能です。

シグネチャまたは形状

static inline TableScanDesc table_beginscan_strat(
    Relation rel, 
    Snapshot snapshot,
    int nkeys, 
    struct ScanKeyData *key,
    bool allow_strat, 
    bool allow_sync
)

入力と出力

  • **入力**:
  • rel: スキャン対象のリレーション。
  • snapshot: MVCCスナップショット。
  • nkeys: スキャンキーの数。
  • key: スキャンキーの配列。
  • allow_strat: スキャン戦略（SO_ALLOW_STRAT）を許可するかどうか。
  • allow_sync: 同期スキャン（SO_ALLOW_SYNC）を許可するかどうか。
  • **出力**:
  • TableScanDesc: 初期化されたスキャン記述子。

前提条件と不変条件

  • rel->rd_tableam が有効な TableAmRoutine を指している必要があります。
  • 呼び出し側は、対象のリレーションに対して適切なロックを保持している必要があります。

アルゴリズムまたは振る舞い

1. デフォルトフラグとして SO_TYPE_SEQSCAN および SO_ALLOW_PAGEMODE を設定します。 2. allow_strat が真の場合、SO_ALLOW_STRAT フラグを追加します。 3. allow_sync が真の場合、SO_ALLOW_SYNC フラグを追加します。 4. 設定したフラグを引数として、rel->rd_tableam->scan_begin を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • contrib/pgstattuple/pgstattuple.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/index/genam.c
  • src/backend/catalog/index.c
  • **呼び出し先**:
  • rel->rd_tableam->scan_begin（テーブルアクセス手法固有の実装）

関連データ構造

  • TableScanDesc: スキャン状態を保持する構造体。
  • Relation: リレーションキャッシュ構造体。
  • Snapshot: MVCCスナップショット。
  • ScanKeyData: スキャン条件を定義する構造体。

失敗モードと境界条件

  • rel->rd_tableam が未定義または scan_begin が実装されていない場合、セグメンテーション違反が発生する可能性があります。
  • 物理ストレージ層でのエラー（ページ読み込み失敗等）は、scan_begin の実装を通じて伝播されます。

保守担当者向けメモ

  • 本関数はインライン関数であり、TableAmRoutine の scan_begin をラップする役割を持ちます。
  • 新しいテーブルアクセス手法を追加する際は、scan_begin 実装において本関数が渡すフラグ（SO_ALLOW_STRAT, SO_ALLOW_SYNC 等）を適切に解釈する必要があります。
