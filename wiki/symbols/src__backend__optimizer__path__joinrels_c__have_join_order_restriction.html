have_join_order_restriction

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinrels.c:1065

目的

2つのリレーション（rel1, rel2）を結合すべきか、あるいは結合順序の制約により強制的に結合する必要があるかを判定する。主に、結合条件が存在しない場合でも、外部結合や LATERAL 参照、PlaceHolderVar の評価要件を満たすために結合を許可すべきかを判断するために使用される。

シグネチャまたは形状

bool have_join_order_restriction(PlannerInfo *root, RelOptInfo *rel1, RelOptInfo *rel2)

入力と出力

  • **入力**:
  • root: クエリ全体のプランナ情報。
  • rel1, rel2: 結合対象となるリレーション情報。
  • **出力**:
  • bool: 結合を強制すべき（制約がある）場合は true、それ以外は false。

前提条件と不変条件

  • rel1 または rel2 が互いに direct_lateral_relids を持つ場合、常に true を返す。
  • PlaceHolderVar の評価に必要なリレーションが両方のリレーションに含まれる場合、常に true を返す。
  • SpecialJoinInfo（外部結合の制約）に基づき、結合順序が論理的に必須であると判断された場合に true を返す。

アルゴリズムまたは振る舞い

1. **LATERAL 参照チェック**: いずれかのリレーションが他方への direct_lateral_relids を持つ場合、即座に true を返す。 2. **PlaceHolderVar チェック**: root->placeholder_list を走査し、両方のリレーションが特定の PlaceHolderVar の評価対象（ph_eval_at）のサブセットである場合、true を返す。 3. **外部結合制約チェック**: root->join_info_list を走査し、SpecialJoinInfo の min_lefthand / min_righthand との包含関係またはオーバーラップを確認する。 4. **ヒューリスティックによる抑制**: 上記の条件で true と判定された場合でも、rel1 または rel2 のいずれかが has_legal_joinclause を満たす（他のリレーションと結合条件を持つ）場合は、結合を強制せず false に戻す。これにより、結合条件のない「Bushy Join」を可能な限り遅延させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_eval.c
  • src/backend/optimizer/path/joinrels.c
  • **呼び出し先**:
  • has_legal_joinclause

関連データ構造

  • PlannerInfo: クエリ全体のプランニング状態。
  • RelOptInfo: 結合対象のリレーション情報。
  • PlaceHolderInfo: PlaceHolderVar の評価制約。
  • SpecialJoinInfo: 外部結合の順序制約。

失敗モードと境界条件

  • FULL JOIN は本関数では無視され、他のメカニズムで処理される。
  • 結合条件が存在する場合、制約による強制結合は抑制される。

保守担当者向けメモ

  • 本関数は、結合順序の制約が厳しい場合にプラン作成を成功させるための「強制的な結合」を管理する。
  • has_legal_joinclause による抑制は、不要な結合の組み合わせを探索するコストを削減するためのヒューリスティックである。
