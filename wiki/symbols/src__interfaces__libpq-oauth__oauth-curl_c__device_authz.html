device_authz

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq-oauth/oauth-curl.c:136

目的

OAuth 2.0 Device Authorization Grant フローにおいて、認可サーバーから取得したデバイス認証情報を保持するデータ構造です。

シグネチャまたは形状

struct device_authz
{
    char       *device_code;
    char       *user_code;
    char       *verification_uri;
    char       *verification_uri_complete;
    char       *expires_in_str;
    char       *interval_str;
    int         expires_in;
    int         interval;
};

入力と出力

本構造体は、認可サーバーからの JSON レスポンスをパースした結果を格納するために使用されます。文字列型のフィールドは JSON から抽出された生の値を保持し、int 型のフィールドはそれらを数値に変換した値を保持します。

前提条件と不変条件

  • 文字列型の各フィールドは、メモリ確保されている場合、free_device_authz 関数によって解放される必要があります。
  • expires_in および interval は、対応する _str フィールドからパースされた整数値である必要があります。

アルゴリズムまたは振る舞い

認可サーバーから返されたデバイス認証情報を構造化して保持し、認証フローの各ステップで必要な URI やコードを参照可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_device_authz 関数（src/interfaces/libpq-oauth/oauth-curl.c:1025）が本構造体へのポインタを受け取り、値を設定します。
  • **呼び出し先**: free_device_authz 関数（src/interfaces/libpq-oauth/oauth-curl.c:151）が本構造体のリソース解放を行います。

関連データ構造

  • async_ctx: 認証フロー全体のコンテキストを保持する構造体。
  • OAuthStep: 認証プロセスの現在地を示す列挙型。

失敗モードと境界条件

  • JSON パース時に必須フィールドが欠落している場合、構造体内のポインタが NULL になる可能性があります。
  • メモリ割り当て失敗時の挙動については、本構造体定義からは直接確認できません。

保守担当者向けメモ

  • 本構造体の各文字列フィールドは動的に確保されるため、使用終了時には必ず free_device_authz を呼び出し、メモリリークを防いでください。
  • _str フィールドと数値フィールドの同期は、パース処理を行う関数側で保証する必要があります。
