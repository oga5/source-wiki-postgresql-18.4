avl_dbase

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/autovacuum.c:171

目的

Autovacuumランチャープロセスにおいて、バキューム対象となるデータベースのスケジュール管理および優先順位付けを行うための情報を保持する。

シグネチャまたは形状

typedef struct avl_dbase
{
    Oid         adl_datid;      /* ハッシュキー -- 構造体の先頭である必要がある */
    TimestampTz adl_next_worker;
    int         adl_score;
    dlist_node  adl_node;
} avl_dbase;

入力と出力

  • **入力**: データベースのOID (adl_datid)、次回の処理予定時刻 (adl_next_worker)、および優先度スコア (adl_score)。
  • **出力**: dlist_node を介したリスト構造への組み込み。

前提条件と不変条件

  • adl_datid は構造体の先頭メンバーである必要がある（ハッシュキーとして利用されるため）。
  • dlist_node を使用して DatabaseList に管理される。

アルゴリズムまたは振る舞い

  • ランチャープロセス内で、バキュームが必要なデータベースのリストを管理するために使用される。
  • adl_score に基づいて処理の優先順位が決定される。
  • adl_next_worker により、次回のバキューム実行タイミングが制御される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/autovacuum.c 内のランチャー処理において、DatabaseList の操作（dlist_tail_element 等）やハッシュテーブルの管理（hctl.entrysize の設定）を通じて参照・操作される。
  • **呼び出し先**: 直接的な関数呼び出し関係は本構造体には存在しない。

関連データ構造

  • dlist_node: リスト管理に使用。
  • DatabaseList: 本構造体を要素として保持するリスト。

失敗モードと境界条件

  • 共有メモリ上のハッシュテーブルとして使用される場合、entrysize が sizeof(avl_dbase) と一致している必要がある。
  • リスト操作時に dlist_node の整合性が保たれていない場合、ランチャーのループ処理で不正なメモリ参照が発生する可能性がある。

保守担当者向けメモ

  • 本構造体は autovacuum.c 内のランチャーのロジックに密接に結びついている。
  • 構造体のメンバー順序を変更する場合、ハッシュキーとしての adl_datid の配置制約に注意すること。
