spg_range_quad_picksplit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_spgist.c:199

目的

SP-GiSTインデックス構築時、ノード分割（picksplit）において範囲型の集合を重心（Centroid）に基づいて4つの象限に分割し、インデックスの階層構造を形成する。

シグネチャまたは形状

Datum spg_range_quad_picksplit(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgPickSplitIn（分割対象のタプル群）および spgPickSplitOut（分割結果の格納先）を受け取る。
  • **出力**: PG_RETURN_VOID()。out 構造体にノード数、プレフィックス（重心）、各タプルの所属ノード番号が書き込まれる。

前提条件と不変条件

  • 入力されるデータは範囲型（RangeType）であること。
  • 空の範囲（Empty Range）は、ルートノードであれば第5象限として、それ以外は通常の4象限のいずれかに分類される。
  • 重心は、全非空範囲の下限および上限のメディアン値から算出される。

アルゴリズムまたは振る舞い

1. **初期化**: 入力タプルから範囲をデシリアライズし、空範囲を除外して非空範囲の数をカウントする。 2. **空範囲のみの場合**: 全てが空範囲であれば、重心なしの2ノード構成とし、全タプルをノード0に割り当てる。 3. **重心の決定**: 非空範囲の下限・上限をそれぞれソートし、中央値（メディアン）を組み合わせて「重心」範囲を生成する。これをインデックスのプレフィックスとして設定する。 4. **象限への割り当て**: getQuadrant 関数を用い、各範囲を重心に対する相対位置（1〜4象限）に基づいてノードへ振り分ける。 5. **ノード数設定**: ルートノードの場合は空範囲用の第5象限を含め5ノード、それ以外は4ノードを確保する。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックス構築プロセス（詳細はソースコード内に明示的な呼び出し元情報の記載なし）。
  • **呼び出し先**: range_get_typcache, range_deserialize, qsort_arg, bound_cmp, range_serialize, getQuadrant。

関連データ構造

  • spgPickSplitIn, spgPickSplitOut: SP-GiSTの分割インターフェース構造体。
  • RangeBound: 範囲の境界値を保持する構造体。
  • TypeCacheEntry: 範囲型の型情報を保持するキャッシュ。

失敗モードと境界条件

  • **全範囲が空**: 非空範囲が0個の場合、重心を計算せず特殊なノード割り当てを行う。
  • **ルートノードか否か**: in->level == 0 の判定により、空範囲を許容するノード数が変化する（5または4）。

保守担当者向けメモ

  • 本関数は範囲を2次元平面上の点として扱う四分木ロジックの中核である。
  • getQuadrant の象限定義と、本関数でのノードインデックス（quadrant - 1）の整合性に注意すること。
  • 検索時の枝刈りロジック（inner_consistent）と分割ロジックは対になっているため、変更時は両方の整合性を確認すること。
