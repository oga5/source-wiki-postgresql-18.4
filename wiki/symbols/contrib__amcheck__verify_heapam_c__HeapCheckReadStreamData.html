HeapCheckReadStreamData

定義

  • 種別: struct
  • 定義箇所: contrib/amcheck/verify_heapam.c:165

目的

ヒープリレーションの検証時におけるページ走査状態を保持するための構造体です。read_stream API を使用した効率的なページ読み込みと、特定のページスキップオプション（SkipPages）の管理を目的とします。

シグネチャまたは形状

typedef struct HeapCheckReadStreamData
{
	BlockRangeReadStreamPrivate range;
	SkipPages	skip_option;
	Relation	rel;
	Buffer	   *vmbuffer;
} HeapCheckReadStreamData;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • **入力**: 初期化時に Relation や SkipPages オプションが設定されます。
  • **出力**: read_stream_next_buffer() 呼び出し時に、range メンバを通じて現在のブロック番号が追跡・更新されます。

前提条件と不変条件

  • range メンバは、SKIP_PAGES_NONE オプション使用時に block_range_read_stream_cb() のプライベートデータとして利用されます。
  • vmbuffer は、可視性マップ（Visibility Map）の検証やスキップ判定に関連して使用されます。

アルゴリズムまたは振る舞い

  • read_stream_next_buffer() の呼び出しを跨いで、現在のブロック位置を range に保持します。
  • skip_option に基づき、不要なページ（all-visible/frozen）をスキップするロジックの制御状態を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c 内の verify_heapam 関数でインスタンス化（stream_skip_data）されます。
  • **呼び出し先**: read_stream_next_buffer() などの read_stream API が本構造体を内部データとして参照します。また、heapcheck_read_stream_next_unskippable 等の関数内で callback_private_data としてキャストされ利用されます。

関連データ構造

  • BlockRangeReadStreamPrivate: ページ走査の範囲管理に使用。
  • SkipPages: ページスキップの挙動を定義する列挙型。
  • Relation: 検証対象のテーブル情報。
  • Buffer: 可視性マップバッファ。

失敗モードと境界条件

  • 構造体内の rel が無効な場合や、vmbuffer が適切に初期化されていない状態で read_stream API を呼び出した場合の動作は未定義です。

保守担当者向けメモ

  • 本構造体は verify_heapam におけるページ走査のコンテキストをカプセル化しています。修正を行う際は、read_stream API の仕様変更が range メンバの扱いに影響を与えないか確認してください。
