get_rel_all_updated_cols

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/util/inherit.c:655

目的

UPDATE クエリにおいて、指定されたリレーション（テーブル）で更新される列の集合を特定します。これには、直接更新される列に加えて、それらに依存する生成列（generated columns）も含まれます。

シグネチャまたは形状

Bitmapset *get_rel_all_updated_cols(PlannerInfo *root, RelOptInfo *rel)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 対象となるリレーション情報。
  • **出力**:
  • 更新対象となる全列（直接更新列 + 依存する生成列）を示す Bitmapset。

前提条件と不変条件

  • root->parse->commandType が CMD_UPDATE であること。
  • rel が単純なリレーション（IS_SIMPLE_REL(rel)）であること。

アルゴリズムまたは振る舞い

1. クエリの resultRelation から対象の RangeTblEntry (RTE) を取得し、そのパーミッション情報から直接更新される列の集合を取得します。 2. 対象リレーションが resultRelation と異なる場合（継承やパーティションの子テーブルの場合）、translate_col_privs_multilevel を呼び出し、親テーブルの列権限情報を子テーブルの属性番号に変換します。 3. get_dependent_generated_columns を呼び出し、更新対象列に依存する生成列を特定します。 4. 直接更新される列と依存する生成列の和集合（bms_union）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c:1836
  • **呼び出し先**:
  • planner_rt_fetch
  • getRTEPermissionInfo
  • find_base_rel
  • translate_col_privs_multilevel
  • get_dependent_generated_columns
  • bms_union

関連データ構造

  • PlannerInfo: プランニング中のクエリ情報。
  • RelOptInfo: リレーションの最適化情報。
  • Bitmapset: 列番号の集合を保持するビットマップ。
  • RTEPermissionInfo: RTE に関連付けられた権限情報。

失敗モードと境界条件

  • IS_SIMPLE_REL(rel) が偽の場合、アサーションエラーが発生します。
  • commandType が CMD_UPDATE でない場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は継承階層やパーティションテーブルを扱う際、親の更新列情報を子テーブルのコンテキストへ正しくマッピングするために重要です。
  • 生成列の依存関係は get_dependent_generated_columns に依存しているため、生成列の定義変更時には当該関数の挙動を確認してください。
