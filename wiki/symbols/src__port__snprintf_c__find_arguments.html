find_arguments

定義

  • 種別: function
  • 定義箇所: src/port/snprintf.c:763

目的

フォーマット文字列内に %n$ 形式の引数指定子が含まれる場合、引数の型と順序を事前に解析・収集する。

シグネチャまたは形状

static bool find_arguments(const char *format, va_list args, PrintfArgValue *argvalues);

入力と出力

  • **入力**:
  • format: 解析対象のフォーマット文字列。
  • args: 可変長引数リスト (va_list)。
  • argvalues: 解析結果を格納する PrintfArgValue 型の配列。
  • **出力**:
  • 成功時は true を返し、argvalues に引数情報が格納される。
  • 失敗時（不正な指定子や引数順序の不整合など）は false を返す。

前提条件と不変条件

  • PG_NL_ARGMAX (31) までの引数順序制御をサポートする。
  • フォーマット文字列内のすべての引数参照は、%n$ 形式か、そうでないかのいずれかに統一されている必要がある。
  • %n$ 指定子が含まれる場合にのみ呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. フォーマット文字列を走査し、% で始まる変換指定子を特定する。 2. 各指定子を解析し、%n$ 形式の引数位置 (fmtpos) を抽出する。 3. * (動的幅指定) や型修飾子 (l, ll, z 等) を考慮し、argtypes 配列に引数の型を記録する。 4. 既に記録された型と異なる型が指定された場合、または非 %n$ 指定子が混在している場合は false を返す。 5. 成功した場合、解析結果に基づいて argvalues を構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/snprintf.c 内の dopr() 関数から呼び出される（src/port/snprintf.c:527）。
  • **呼び出し先**: 内部で Max() マクロや strchr() を使用する。

関連データ構造

  • PrintfArgValue: 引数の値を保持する構造体。
  • PrintfArgType: 引数の型を識別する列挙型。
  • PG_NL_ARGMAX: サポートされる最大引数インデックス（31）。

失敗モードと境界条件

  • %n$ 指定子と非 %n$ 指定子が混在している場合、false を返す。
  • accum（位置指定）が 0 以下または PG_NL_ARGMAX を超える場合、false を返す。
  • 過去に記録された型と矛盾する型が指定された場合、false を返す。
  • * 指定子に対して $ による位置指定が欠落している場合、false を返す。

保守担当者向けメモ

  • 本関数は dopr() と同様のフォーマット文字列走査を行う必要があるが、詳細な解析は不要である。
  • Windows 環境下では I32, I64 などのプラットフォーム固有のサイズ修飾子を処理するロジックが含まれている。
  • va_list の操作については、標準ライブラリの printf との再帰的な呼び出しを避けるため、src/port/snprintf.c 内の定義に従うこと。
