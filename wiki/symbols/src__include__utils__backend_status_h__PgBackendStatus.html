PgBackendStatus

定義

  • **種別**: struct
  • **定義箇所**: src/include/utils/backend_status.h:98

目的

各バックエンドプロセスの現在の活動状況（実行中のクエリ、状態、接続情報など）を共有メモリ上で管理するためのデータ構造です。

シグネチャまたは形状

typedef struct PgBackendStatus {
    int         st_changecount;
    int         st_procpid;
    BackendType st_backendType;
    TimestampTz st_proc_start_timestamp;
    TimestampTz st_xact_start_timestamp;
    TimestampTz st_activity_start_timestamp;
    TimestampTz st_state_start_timestamp;
    Oid         st_databaseid;
    Oid         st_userid;
    SockAddr    st_clientaddr;
    char       *st_clienthostname;
    bool        st_ssl;
    PgBackendSSLStatus *st_sslstatus;
    bool        st_gss;
    PgBackendGSSStatus *st_gssstatus;
    BackendState st_state;
    char       *st_appname;
    char       *st_activity_raw;
    ProgressCommandType st_progress_command;
    Oid         st_progress_command_target;
    int64       st_progress_param[PGSTAT_NUM_PROGRESS_PARAM];
    int64       st_query_id;
    int64       st_plan_id;
} PgBackendStatus;

入力と出力

  • **入力**: 各バックエンドプロセスが自身の状態を更新するために使用します。
  • **出力**: 統計情報収集システムや監視ツールが、共有メモリ上のこの構造体を読み取ることで、プロセスの活動状況を取得します。

前提条件と不変条件

  • st_procpid > 0 の場合のみエントリは有効とみなされます（0 は未使用）。
  • st_clienthostname、st_appname、st_activity_raw は必ず NULL 終端されている必要があります。
  • 読み書きには「チェンジカウンタ」プロトコルが必須であり、直接 st_changecount を操作せず、専用のマクロ（PGSTAT_BEGIN_WRITE_ACTIVITY 等）を使用する必要があります。

アルゴリズムまたは振る舞い

ロックのオーバーヘッドを避けるため、以下のプロトコルを採用しています。 1. **書き込み**: 更新前に st_changecount をインクリメントし、更新後に再度インクリメントします。 2. **読み取り**: st_changecount を読み取り、データをコピーした後、再度 st_changecount を確認します。値が一致し、かつ偶数であればコピーは有効です。不一致または奇数の場合は再試行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/backend_status.c および src/backend/utils/activity/backend_progress.c 内の各関数が、共有メモリ上のエントリ（MyBEEntry）を操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • BackendState: バックエンドのライフサイクル状態。
  • PgBackendSSLStatus, PgBackendGSSStatus: 接続セキュリティ情報。
  • SockAddr: クライアント接続情報。

失敗モードと境界条件

  • 書き込みマクロ内でエラーが発生すると PANIC に至る可能性があるため、クリティカルセクション内のコードは極力短くする必要があります。

保守担当者向けメモ

  • 本構造体は累積統計ではなく「現在の状態」を追跡するものです。
  • メモリバリアが必須であるため、カウンタ操作には必ず提供されたマクロを使用してください。
