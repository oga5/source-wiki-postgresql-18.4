table_open

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/table.c:39

目的

指定された OID に対応するリレーションをオープンし、それがテーブルであることを検証した上で Relation 構造体を返します。

シグネチャまたは形状

Relation table_open(Oid relationId, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relationId (Oid): オープン対象のリレーション識別子。
  • lockmode (LOCKMODE): リレーション取得時に適用するロックモード。
  • **出力**:
  • Relation: 検証済みのリレーション記述子。

前提条件と不変条件

  • 指定された relationId がシステムカタログ上に存在する必要があります。
  • 戻り値の Relation は、インデックスや複合型ではなく、テーブル（relkind がテーブル関連）であることを保証します。

アルゴリズムまたは振る舞い

1. relation_open(relationId, lockmode) を呼び出し、リレーションをオープンして Relation 構造体を取得します。 2. validate_relation_kind(r) を呼び出し、取得したリレーションがテーブルとして適切か検証します。 3. 検証を通過した場合、その Relation ポインタを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • relation_open (低レベルリレーション取得)
  • validate_relation_kind (型検証)
  • **呼び出し元**:
  • contrib/amcheck/verify_common.c
  • contrib/amcheck/verify_heapam.c
  • contrib/dblink/dblink.c
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/deparse.c
  • contrib/postgres_fdw/postgres_fdw.c
  • contrib/sepgsql/database.c
  • contrib/sepgsql/label.c

関連データ構造

  • Relation: リレーションのメタデータおよび状態を保持する構造体。
  • Oid: リレーションを一意に識別するオブジェクト識別子。

失敗モードと境界条件

  • relation_open がリレーションを見つけられない場合、エラーが発生します。
  • validate_relation_kind が、対象がテーブルではない（インデックスや複合型など）と判断した場合、ereport(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は relation_open のラッパーであり、テーブルアクセス専用のガードレールとして機能します。
  • 実際のデータアクセスロジックは含まれておらず、テーブルアクセスの入り口としての整合性維持に特化しています。
