namerecv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:81

目的

PostgreSQLのバイナリ転送プロトコル形式で送られてきたデータを、name 型の内部表現に変換します。

シグネチャまたは形状

Datum namerecv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して StringInfo 型のバッファを受け取ります。
  • **出力**: Name 型の Datum を返します。

前提条件と不変条件

  • 入力データは pq_getmsgtext を介して取得されます。
  • name 型は固定長であり、NAMEDATALEN バイトのメモリ領域を必要とします。
  • 戻り値のメモリ領域は palloc0 によって確保され、ゼロクリアされます。

アルゴリズムまたは振る舞い

1. pq_getmsgtext を使用して、入力バッファから文字列データとバイト数を抽出します。 2. 抽出されたバイト数（nbytes）が NAMEDATALEN 以上である場合、ERRCODE_NAME_TOO_LONG エラーを発生させます。 3. palloc0(NAMEDATALEN) を呼び出し、name 型用のメモリを確保します。 4. 抽出した文字列を確保したメモリ領域にコピーします。 5. 一時的に使用した文字列バッファを pfree で解放します。 6. PG_RETURN_NAME マクロを使用して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 不明（PostgreSQLの型システムにおけるバイナリ入力関数として登録されていると推測されます）。
  • **呼び出し先**: pq_getmsgtext, ereport, palloc0, memcpy, pfree。

関連データ構造

  • NameData: name 型の内部表現。
  • StringInfo: バイナリプロトコル上のデータ保持構造体。
  • NAMEDATALEN: name 型の最大長を定義する定数。

失敗モードと境界条件

  • **エラー**: 入力された識別子の長さが NAMEDATALEN 以上の場合、ereport によりエラーがスローされます。

保守担当者向けメモ

  • name 型の長さ制限は NAMEDATALEN に依存しています。この定数を変更する場合は、本関数のチェックロジックと整合性が取れていることを確認してください。
  • palloc0 を使用しているため、コピーされなかった残りの領域は確実にゼロで埋められます。
