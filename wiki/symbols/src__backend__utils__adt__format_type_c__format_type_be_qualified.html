format_type_be_qualified

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/format_type.c:352

目的

指定されたデータ型OIDに対応する、スキーマ修飾された標準的なSQL形式の型名文字列を生成します。

シグネチャまたは形状

char *format_type_be_qualified(Oid type_oid)

入力と出力

  • **入力**: type_oid (Oid) - 変換対象のデータ型OID。
  • **出力**: char * - 型名を表す文字列（pallocによりメモリ確保済み）。

前提条件と不変条件

  • type_oid はシステムカタログ pg_type に存在する有効なOIDである必要があります。
  • 戻り値の文字列は呼び出し元で pfree する必要があります。

アルゴリズムまたは振る舞い

1. format_type_extended 関数を呼び出します。 2. その際、第2引数（typmod）に -1 を指定し、第3引数（flags）に FORMAT_TYPE_FORCE_QUALIFY を指定します。 3. これにより、型修飾子を無視し、常にスキーマ修飾（schema.type）された形式で型名を取得します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c
  • src/backend/utils/adt/regproc.c
  • **呼び出し先**:
  • format_type_extended

関連データ構造

  • Oid: PostgreSQLのシステムカタログにおけるオブジェクト識別子。

失敗モードと境界条件

  • type_oid が無効な場合、format_type_extended の内部実装に従い、エラーが発生するか、あるいは ??? のような代替文字列が返される可能性があります。

保守担当者向けメモ

  • 本関数は format_type_extended の特定のユースケース（強制的なスキーマ修飾）を簡略化するためのラッパーです。
  • 型名のフォーマットロジックに変更を加える場合は、format_type_extended を確認してください。
