slot_compile_deform

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_deform.c:33

目的

タプル（行データ）を物理的なメモリレイアウトから TupleTableSlot 形式へ変換（デフォーム）するための専用 LLVM IR 関数を生成する。

シグネチャまたは形状

LLVMValueRef slot_compile_deform(LLVMJitContext *context, TupleDesc desc, const TupleTableSlotOps *ops, int natts);

入力と出力

  • **入力**:
  • context: JIT コンパイルのコンテキスト。
  • desc: 対象タプルの記述子（TupleDesc）。
  • ops: スロットの操作セット（TupleTableSlotOps）。
  • natts: カラム数。
  • **出力**: 生成された LLVM 関数を表す LLVMValueRef。対象外のスロット型の場合は NULL を返す。

前提条件と不変条件

  • ops が TTSOpsVirtual の場合、デフォームは不要なため早期リターンする。
  • ops が TTSOpsHeapTuple, TTSOpsBufferHeapTuple, TTSOpsMinimalTuple のいずれかである必要がある。
  • 最終カラム（0 始まり）は存在することが保証されている。

アルゴリズムまたは振る舞い

1. **初期化**: スロット型を確認し、対応外であれば NULL を返す。 2. **静的解析**: TupleDesc を走査し、NOT NULL 制約を持つカラムを特定して最適化戦略を決定する。 3. **関数生成**: LLVM モジュール内に deform 関数を定義し、必要な基本ブロック（entry, find_startblock 等）を構築する。 4. **コード生成**:

  • TupleTableSlot の各フィールド（tts_values, tts_ISNULL 等）へのポインタを取得。
  • スロット型に応じて、タプルヘッダやオフセットへのアクセスコードを生成。
  • カラムごとのデフォーム処理（NULL チェック、アライメント調整、データコピー）を行うためのブロックを順次構築する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit_expr.c:408
  • **呼び出し先**:
  • LLVM C API（LLVMAddFunction, LLVMBuildAlloca, LLVMGetModuleContext 等）
  • llvm_mutable_module, llvm_expand_funcname, llvm_copy_attributes 等の JIT ユーティリティ

関連データ構造

  • TupleTableSlot, TupleDesc
  • LLVMJitContext, LLVMModuleRef, LLVMBuilderRef
  • HeapTupleTableSlot, MinimalTupleTableSlot

失敗モードと境界条件

  • 未対応の TupleTableSlotOps が渡された場合、NULL を返して終了する。
  • 仮想タプル（TTSOpsVirtual）はデフォーム処理をスキップする。

保守担当者向けメモ

  • 本関数はコンパイル時にカラムオフセットを確定させることで最適化を図る。attguaranteedalign が true の場合、定数オフセットによるロード命令が生成される。
  • 生成されたコードは llvm_mutable_module に追加される。デバッグ時は jit_debugging_support を有効にして LLVM IR をダンプすること。
