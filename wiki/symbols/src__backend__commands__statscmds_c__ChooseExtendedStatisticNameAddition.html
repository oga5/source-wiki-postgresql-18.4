ChooseExtendedStatisticNameAddition

定義

  • 種別: function
  • 定義箇所: src/backend/commands/statscmds.c:889

目的

拡張統計情報オブジェクトの名前が明示的に指定されていない場合に、対象となる列名や式に基づいて自動生成するためのサフィックス文字列を生成する。

シグネチャまたは形状

char *ChooseExtendedStatisticNameAddition(List *exprs)

入力と出力

  • **入力**: exprs (統計対象となる StatsElem のリスト)
  • **出力**: 生成された名前の断片を含む palloc された文字列。

前提条件と不変条件

  • exprs リスト内の各要素は StatsElem 型である必要がある（それ以外はスキップされる）。
  • 生成される文字列の長さは NAMEDATALEN に制限される。

アルゴリズムまたは振る舞い

1. 空のバッファ buf を初期化する。 2. exprs リストを走査し、各 StatsElem から名前を取得する。 3. 名前が NULL の場合は "expr" を使用する。 4. 複数の要素がある場合はアンダースコア _ で連結する。 5. strlcpy を用いてバッファにコピーし、NAMEDATALEN を超える場合はその時点で打ち切る。 6. 最終的な文字列を pstrdup して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/statscmds.c 内の統計情報作成処理から呼び出される（直接的な呼び出し元関数の詳細はソース抜粋からは不明）。
  • **呼び出し先**: lfirst、IsA、strlcpy、strlen、pstrdup などの標準的なマクロやユーティリティ関数。

関連データ構造

  • StatsElem: 統計対象の列や式を保持する構造体。
  • List: PostgreSQL のリスト構造。

失敗モードと境界条件

  • exprs が空の場合、空文字列が返される。
  • StatsElem 以外の型がリストに含まれる場合、その要素は無視される。

保守担当者向けメモ

  • 本関数は名前の自動生成のみを担当する。名前の衝突回避については、呼び出し側で CommandCounterIncrement 等を用いて適切に制御する必要がある。
  • 式（expression）に対しては一律で "expr" という名前を割り当てる仕様である。より詳細な命名が必要な場合は、ユーザーによる明示的な指定を推奨する。
