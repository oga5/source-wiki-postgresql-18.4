test_rb_tree

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_rbtree/test_rbtree.c:502

目的

PostgreSQL の赤黒木ライブラリ (lib/rbtree.h) の正当性を検証するためのメインエントリポイントです。SQL から呼び出され、指定されたサイズのデータセットを用いて一連の木操作テストを実行します。

シグネチャまたは形状

Datum test_rb_tree(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT32(0) により size（テスト対象のノード数）を受け取ります。
  • **出力**: 成功時は PG_RETURN_VOID() を返します。検証に失敗した場合は elog(ERROR, ...) により例外を発生させます。

前提条件と不変条件

  • size は 0 より大きく、かつ MaxAllocSize / sizeof(int) 以下である必要があります。これに違反する場合、関数はエラーを投げます。

アルゴリズムまたは振る舞い

1. 引数 size の妥当性を検証します。 2. 以下の順序でテスト関数を呼び出し、赤黒木の各操作を検証します。

  • testleftright(size): 順方向走査
  • testrightleft(size): 逆方向走査
  • testfind(size): 検索操作
  • testfindltgt(size): 範囲検索（less/great）
  • testleftmost(size): 最左ノードの取得
  • testdelete(size, ...): 削除操作

    • 3. 全てのテストが完了すると正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: testleftright, testrightleft, testfind, testfindltgt, testleftmost, testdelete（いずれも同ファイル内のテスト関数）。

関連データ構造

  • IntRBTreeNode: テストで使用される整数キーを持つ赤黒木ノード。

失敗モードと境界条件

  • size <= 0 または size がメモリ制限を超える場合、elog(ERROR) で停止します。
  • 各テスト関数内で赤黒木の不変条件が崩れていることが検出された場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は PostgreSQL の拡張機能として SQL から実行されることを前提としています。
  • テストの網羅性を高めるために新しい検証ロジックを追加する場合は、本関数の呼び出しシーケンスに新しいテスト関数を追加してください。
