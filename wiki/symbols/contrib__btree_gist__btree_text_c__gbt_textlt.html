gbt_textlt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_text.c:63

目的

PostgreSQLのGiSTインデックスにおいて、text型データ同士の「より小さい（less than）」比較を行うためのサポート関数です。

シグネチャまたは形状

static bool gbt_textlt(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の text 型データへのポインタ。
  • collation: 照合順序（Collation）のOID。
  • flinfo: 関数呼び出し情報（使用されません）。
  • **出力**:
  • bool: a < b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な text 型のデータ構造を指している必要があります。
  • 内部で text_lt 関数を呼び出すため、PostgreSQLの標準的なテキスト比較ルールに従います。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力された void * ポインタを Datum 型に変換します。 2. DirectFunctionCall2Coll を介して、PostgreSQLのコア関数である text_lt を呼び出します。 3. text_lt の結果（bool）を DatumGetBool で取得し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: text_lt（PostgreSQLコアのテキスト比較関数）。
  • **呼び出し元**:
  • contrib/btree_gist/btree_text.c 内の gbtree_vinfo 構造体（tinfo）の初期化時に参照されています。
  • 直接的な関数呼び出しの詳細は、提供されたソース範囲内では確認できません。

関連データ構造

  • gbtree_vinfo: text 型のインデックス操作に必要な比較関数群を保持する構造体。

失敗モードと境界条件

  • text_lt が内部でエラーを発生させた場合（照合順序の不整合やメモリ不足など）、そのエラーがそのまま伝播します。

保守担当者向けメモ

  • 本関数は text 型専用の比較ロジックを抽象化するためのラッパーです。
  • bpchar 型用の同様の関数（gbt_bpcharlt）とは比較ルールが異なるため、混同しないよう注意してください。
