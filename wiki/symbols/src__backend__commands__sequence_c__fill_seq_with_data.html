fill_seq_with_data

定義

  • 種別: function
  • 定義箇所: src/backend/commands/sequence.c:337

目的

シーケンスリレーションの物理ページを、指定されたタプルデータで初期化または更新します。

シグネチャまたは形状

static void fill_seq_with_data(Relation rel, HeapTuple tuple)

入力と出力

  • **入力**:
  • rel: 更新対象のシーケンスリレーション（Relation）。
  • tuple: シーケンスの新しい状態を保持するヒープタプル（HeapTuple）。
  • **出力**: なし（副作用としてストレージを更新）。

前提条件と不変条件

  • rel は有効なシーケンスリレーションである必要があります。
  • tuple はシーケンスの定義データを含んでいる必要があります。

アルゴリズムまたは振る舞い

1. MAIN_FORKNUM に対して fill_seq_fork_with_data を呼び出し、メインデータを書き込みます。 2. リレーションの永続性が RELPERSISTENCE_UNLOGGED である場合、以下の処理を追加で行います。

  • INIT_FORKNUM を作成し、WALログに記録します。
  • INIT_FORKNUM に対して fill_seq_fork_with_data を呼び出します。
  • バッファをフラッシュし、ストレージマネージャを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/sequence.c 内のシーケンス作成・更新処理（DefineSequence 等から間接的に参照）。
  • **呼び出し先**:
  • fill_seq_fork_with_data
  • smgropen, smgrcreate, log_smgrcreate, FlushRelationBuffers, smgrclose（RELPERSISTENCE_UNLOGGED の場合）

関連データ構造

  • Relation: シーケンスのメタデータおよびストレージ情報を保持。
  • HeapTuple: シーケンスの現在値や設定値を含むデータ構造。
  • SMgrRelation: ストレージ管理用ハンドル。

失敗モードと境界条件

  • RELPERSISTENCE_UNLOGGED の場合、ストレージ操作（smgrcreate 等）の失敗によりトランザクションが中断される可能性があります。
  • 物理的な書き込みエラーが発生した場合、PostgreSQLの標準的なエラーハンドリングにより例外がスローされます。

保守担当者向けメモ

  • 本関数はシーケンスの物理的な初期化を担うため、WALログの整合性に直結します。
  • UNLOGGED シーケンスの初期化時に INIT_FORKNUM を扱うロジックが含まれているため、将来的なストレージ層の変更時にはこの分岐の挙動を確認してください。
