apw_init_state

定義

  • 種別: function
  • 定義箇所: contrib/pg_prewarm/autoprewarm.c:862

目的

共有メモリ上に配置された AutoPrewarmSharedState 構造体の初期化を行います。

シグネチャまたは形状

static void apw_init_state(void *ptr)

入力と出力

  • **入力**: ptr (共有メモリ領域を指すポインタ。AutoPrewarmSharedState 型としてキャストされる)
  • **出力**: なし（引数で渡されたメモリ領域を直接更新）

前提条件と不変条件

  • 引数 ptr は、AutoPrewarmSharedState 構造体を格納するのに十分なサイズの有効な共有メモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. 引数 ptr を AutoPrewarmSharedState 型のポインタにキャストします。 2. state->lock に対して LWLockInitialize を実行し、新しいトランシェ ID を割り当てて初期化します。 3. state->bgworker_pid を InvalidPid に設定します。 4. state->pid_using_dumpfile を InvalidPid に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照情報に基づくと、apw_init_shmem 等の共有メモリ初期化処理から呼び出されることが示唆されますが、本 excerpt 内では明示的な呼び出し元は特定されていません。
  • **呼び出し先**: LWLockInitialize (PostgreSQL 内部関数)

関連データ構造

  • AutoPrewarmSharedState: 本関数が初期化対象とする共有メモリ上の制御構造体。

失敗モードと境界条件

  • LWLockNewTrancheId() が失敗した場合、PostgreSQL の標準的なエラー処理メカニズムによりプロセスが終了する可能性があります。

保守担当者向けメモ

  • 本関数は共有メモリの初期化フェーズでのみ使用されることを想定しています。
  • LWLock の初期化が含まれるため、共有メモリの確保が完了した直後に実行される必要があります。
