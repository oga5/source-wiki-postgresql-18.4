revmap_get_blkno

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_revmap.c:441

目的

指定されたヒープブロック番号に対応する、BRINインデックス内のリバースレンジマップ（revmap）の物理ブロック番号を算出します。

シグネチャまたは形状

static BlockNumber revmap_get_blkno(BrinRevmap *revmap, BlockNumber heapBlk)

入力と出力

  • **入力**:
  • revmap: BrinRevmap 構造体へのポインタ。現在のrevmapの状態（ページ範囲設定など）を保持。
  • heapBlk: 対象となるヒープのブロック番号。
  • **出力**:
  • 成功時: 対応するrevmapの物理ブロック番号。
  • 失敗時: InvalidBlockNumber（対象ブロックがまだrevmapに割り当てられていない場合）。

前提条件と不変条件

  • revmap ポインタは有効であること。
  • インデックスのメタページは、revmap->rm_lastRevmapPage に最新のrevmap末尾ページ番号を保持していること。
  • revmapの物理ブロック番号は、メタページ（ブロック0）をスキップするため、算出値に1を加算した値となる。

アルゴリズムまたは振る舞い

1. HEAPBLK_TO_REVMAP_BLK マクロを使用して、ヒープブロック番号から相対的なrevmapブロック番号を計算し、メタページ分（+1）を加算します。 2. 算出された targetblk が、revmap->rm_lastRevmapPage（キャッシュされた末尾ページ番号）以下であるかを確認します。 3. targetblk が範囲内であればその値を返し、範囲外であれば InvalidBlockNumber を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_revmap.c 内の複数箇所（brinGetTupleForHeapBlock 等の内部処理から参照）。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありませんが、HEAPBLK_TO_REVMAP_BLK マクロを利用します。

関連データ構造

  • BrinRevmap: revmapのアクセス状態を保持する構造体。rm_pagesPerRange や rm_lastRevmapPage を本関数で使用します。

失敗モードと境界条件

  • 指定されたヒープブロックに対応するrevmapページがまだ物理的に拡張されていない場合、InvalidBlockNumber を返します。これはエラーではなく、呼び出し元がページ拡張（revmap_extend_and_get_blkno 等）を行うための正常な判定結果として扱われます。

保守担当者向けメモ

  • 本関数は静的関数であり、brin_revmap.c 内でのみ使用されます。
  • 算出ロジックは HEAPBLK_TO_REVMAP_BLK マクロに依存しているため、BRINのページレイアウト構造に変更を加える際は、当該マクロの定義と整合性を確認してください。
