inet_gist_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_gist.c:114

目的

ネットワーク型（INET / CIDR）のGiSTインデックスにおいて、検索クエリとインデックスノードの整合性を判定し、インデックスの探索を継続すべきか（枝刈りを行うか）を決定します。

シグネチャまたは形状

Datum inet_gist_consistent(PG_FUNCTION_ARGS)
  • 引数: GISTENTRY（インデックスエントリ）、inet（検索クエリ）、StrategyNumber（演算子戦略）、bool*（再チェックフラグ）

入力と出力

  • 入力: インデックスエントリ、検索対象のネットワーク値、比較演算子。
  • 出力: bool（一致の可能性がある場合は true、枝刈り可能な場合は false）。
  • 副作用: recheck フラグを常に false に設定します（本関数が提供する演算子はすべて正確であるため）。

前提条件と不変条件

  • GistInetKey 構造体を使用してインデックスキーを解釈します。
  • 内部ノードでは、子ノードのビット長が gk_ip_minbits(key) 以上であることを前提に探索を行います。
  • リーフノードでは、commonbits がアドレス長と等しいことを前提に比較を行います。

アルゴリズムまたは振る舞い

1. **ファミリーチェック**: インデックスキーとクエリのアドレスファミリーを比較します。不一致の場合、戦略に応じて探索の可否を判定します。キーが複数のファミリーを保持している場合（内部ノード）、常に一致とみなします。 2. **ビット長チェック**: minbits を用いて、サブネット/スーパーネット演算子の探索範囲を絞り込みます。 3. **共通ビット比較**: bitncmp を使用し、共通プレフィックス長に基づいてアドレス部分を比較します。 4. **リーフノードの最終判定**: リーフノード到達時は、ネットマスク長（minbits）の比較を含めた詳細な判定を行い、演算子ごとの整合性を確定させます。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLのGiSTインデックススキャンエンジン（直接の呼び出し元情報はソースコード上では明示されていません）。
  • 呼び出し先: bitncmp（ビット比較）、gk_ip_* マクロ群（キー属性取得）、ip_* マクロ群（クエリ属性取得）。

関連データ構造

  • GistInetKey: GiSTインデックス内でネットワーク情報を保持する構造体。
  • GISTENTRY: GiSTインデックスの各ノードエントリ。

失敗モードと境界条件

  • アドレスファミリーが混在する内部ノードでは、常に true を返して探索を継続します。
  • 比較対象のビット長が minbits や commonbits を超える場合、Min 関数による制限を適用して安全に比較を行います。
  • 演算子戦略が INETSTRAT_NE（不等号）の場合、リーフノード以外では常に一致の可能性があると判断します。

保守担当者向けメモ

  • 本関数は network_cmp_internal()（network.c）のロジックと整合するように設計されています。
  • 修正を行う際は、GistInetKey の構造が INET 型の内部表現と異なる点に注意してください。
  • 演算子戦略の追加時は、Check 2（ビット長）および Check 3（共通ビット）の分岐条件を網羅的に確認してください。
