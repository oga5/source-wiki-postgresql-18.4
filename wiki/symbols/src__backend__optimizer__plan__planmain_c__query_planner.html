query_planner

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/planmain.c:53

目的

クエリの結合ツリーを解析し、最適な結合順序とアクセスパスを決定して、最終的な RelOptInfo を構築する。

シグネチャまたは形状

RelOptInfo *query_planner(PlannerInfo *root, query_pathkeys_callback qp_callback, void *qp_extra);

入力と出力

  • **入力**:
  • root: プランニング対象のクエリ情報 (PlannerInfo)。
  • qp_callback: 等価クラス統合完了後に呼び出されるコールバック関数。
  • qp_extra: コールバックに渡す補助データ。
  • **出力**:
  • 結合ツリー全体を表す RelOptInfo 構造体。

前提条件と不変条件

  • root->append_rel_list は呼び出し前に設定済みであること。
  • PlannerInfo 内の結合関連リスト（join_rel_list 等）は、本関数内で初期化（クリア）される。
  • FROM 句が単一の RTE_RESULT である場合、最適化をバイパスして即座にパスを生成する。

アルゴリズムまたは振る舞い

1. **初期化**: PlannerInfo の結合関連リストをリセットし、ベースリレーション配列を構築する。 2. **単純ケースの処理**: FROM 句が単一の RTE_RESULT の場合、GroupResultPath を作成して終了する。 3. **ベースリレーション構築**: add_base_rels_to_query を呼び出し、クエリ内のリレーションを特定する。 4. **前処理**: remove_useless_groupby_columns 等で不要な列を削除し、build_base_rel_tlists 等でターゲットリストや等価クラス（EquivalenceClasses）を構築する。 5. **最適化**:

  • reconsider_outer_join_clauses で外部結合句を再検討する。
  • generate_base_implied_equalities で等価クラスに基づく制約を追加する。
  • qp_callback を実行し、パスキーを計算する。
  • fix_placeholder_input_needed_levels でプレースホルダーの依存関係を解決する。
  • remove_useless_joins で不要な結合を削除する。

    • 6. **終了**: 最終的な結合パスを決定する（以降の処理はソース抜粋外）。

関連データ構造

  • PlannerInfo: プランニングの全状態を保持。
  • RelOptInfo: 結合関係やパス情報を保持する主要構造体。
  • Query: 解析済みのクエリツリー。

失敗モードと境界条件

  • jointree->fromlist が空の場合、Assert により停止する。
  • 外部結合の最適化や等価クラスの統合は、ベースリレーションの構築後でなければ実行できない制約がある。

保守担当者向けメモ

  • 本関数は「結合」の最適化に特化しており、グループ化やソートなどのトップレベル処理は含まない。
  • qp_callback は、等価クラスの統合が完了するまでパスキーを計算できないという制約を回避するための重要な仕組みである。
  • パーティション展開などの「他のリレーション」の処理は、結合最適化に必要な情報が揃った後に行われる。
