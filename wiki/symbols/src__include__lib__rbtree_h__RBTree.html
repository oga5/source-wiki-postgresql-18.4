RBTree

定義

  • 種別: struct
  • 定義箇所: src/include/lib/rbtree.h:32

目的

PostgreSQL における赤黒木（Red-Black Tree）の制御構造体です。バランスの取れた二分探索木を管理し、挿入・削除・検索において $O(\lg n)$ の計算量を保証します。

シグネチャまたは形状

typedef struct RBTree RBTree;

※構造体の内部フィールドは非公開であり、ライブラリ内部で管理されます。

入力と出力

  • 入力: rbt_create() を通じて比較関数、結合関数、メモリ確保関数などのコールバックを登録します。
  • 出力: RBTree インスタンスへのポインタを返します。

前提条件と不変条件

  • **侵入型設計**: ユーザー定義構造体の先頭に RBTNode を配置する必要があります。
  • **内部操作の禁止**: 呼び出し側は RBTNode の内部フィールド（color, left, right, parent）を直接操作してはなりません。
  • **メモリ管理**: ノードの確保・解放は rbt_create で指定した関数に依存します。

アルゴリズムまたは振る舞い

  • 挿入・削除時に赤黒木の平衡維持アルゴリズム（回転や再着色）を自動的に実行します。
  • 既存キーとの重複時には、登録された combiner コールバックを呼び出し、既存ノードの更新を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/lib/rbtree.c 内の各関数（rbt_create, rbt_find, rbt_find_great, rbt_find_less 等）がこの構造体を操作します。
  • 呼び出し先: rbt_create で指定されたユーザー定義のコールバック関数を呼び出します。

関連データ構造

  • RBTNode: 木の各要素として埋め込まれるノード。
  • RBTreeIterator: 木の走査状態を保持する構造体。

失敗モードと境界条件

  • 比較関数やメモリ確保関数が提供されない場合、または不適切な実装である場合の挙動については、本ソース excerpt には記載がありません。

保守担当者向けメモ

  • 本構造体は src/backend/lib/rbtree.c で実装されています。
  • 構造体内部のフィールド定義は公開されていないため、ライブラリの API を介した操作のみを許可してください。
