verify_gist_page

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/gistfuncs.c:40

目的

入力された bytea 形式の生データが、有効な GiST インデックスページであるかを検証し、内部処理用の Page 構造体へ変換する。

シグネチャまたは形状

static Page verify_gist_page(bytea *raw_page)

入力と出力

  • **入力**: bytea *raw_page（SQLから渡される生のページデータ）
  • **出力**: Page（検証済みのページポインタ）

前提条件と不変条件

  • 入力ページが PageIsNew である場合、検証をスキップしてそのまま返却する。
  • 有効な GiST ページであるためには、特殊領域（Special Space）のサイズが MAXALIGN(sizeof(GISTPageOpaqueData)) と一致する必要がある。
  • ページ内の gist_page_id が GIST_PAGE_ID と一致する必要がある。

アルゴリズムまたは振る舞い

1. get_page_from_raw を呼び出し、bytea を Page 型に変換する。 2. ページが新規作成直後（PageIsNew）であれば、そのまま返却する。 3. PageGetSpecialSize を使用し、特殊領域サイズが GiST の期待値と一致するか確認する。不一致の場合は ERROR をスローする。 4. GistPageGetOpaque を介してページメタデータを取得し、gist_page_id が GIST_PAGE_ID であるかを確認する。不一致の場合は ERROR をスローする。 5. 検証を通過した Page を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/gistfuncs.c 内の以下の関数から呼び出される（直接的な証拠に基づく）:
  • gist_page_opaque_info
  • gist_page_items
  • gist_page_items_bytea
  • **呼び出し先**:
  • get_page_from_raw
  • PageIsNew
  • PageGetSpecialSize
  • GistPageGetOpaque

関連データ構造

  • Page: PostgreSQL の標準的なページ構造体。
  • GISTPageOpaqueData: GiST インデックスページ固有のメタデータ構造体。

失敗モードと境界条件

  • **ERRCODE_INVALID_PARAMETER_VALUE**:
  • 特殊領域のサイズが GiST の期待値と異なる場合。
  • gist_page_id が GIST_PAGE_ID と一致しない場合。
  • ページが破損している場合や、GiST 以外のインデックスページが渡された場合にエラーを発生させる。

保守担当者向けメモ

  • 本関数は contrib/pageinspect 内の GiST 関連関数における共通の検証ゲートウェイである。
  • 呼び出し元情報の詳細は、ソースコード内の各 SQL 関数実装箇所を参照すること。
