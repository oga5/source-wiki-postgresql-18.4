makeStringInfoExt

定義

  • 種別: function
  • 定義箇所: src/common/stringinfo.c:84

目的

指定された初期サイズで StringInfoData 構造体を動的に確保し、初期化された StringInfo オブジェクトを生成する。

シグネチャまたは形状

StringInfo makeStringInfoExt(int initsize)

入力と出力

  • **入力**: initsize (int型) - バッファの初期確保サイズ。
  • **出力**: StringInfo (構造体へのポインタ) - 初期化済みのバッファ管理オブジェクト。

前提条件と不変条件

  • 内部でメモリ確保を行うため、呼び出し元はメモリ不足が発生しない環境を想定している。
  • 返される StringInfo オブジェクトは、len が 0、data がヌル終端された空文字列を指す状態に初期化される。

アルゴリズムまたは振る舞い

1. 引数 initsize を受け取る。 2. 内部関数 makeStringInfoInternal(initsize) を呼び出す。 3. makeStringInfoInternal が確保した StringInfoData 構造体へのポインタをそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/crypt-sha.c (複数箇所)
  • **呼び出し先**:
  • makeStringInfoInternal (ソースコード上の直接の呼び出し先)

関連データ構造

  • StringInfoData: バッファの実体（data, len, maxlen, cursor を保持）。
  • StringInfo: StringInfoData へのポインタ型。

失敗モードと境界条件

  • initsize に負の値や極端に小さい値を指定した場合の挙動は、内部の makeStringInfoInternal の実装に依存する。
  • メモリ確保に失敗した場合、PostgreSQL のメモリ管理規約に従い、通常はエラーがスローされる。

保守担当者向けメモ

  • 本関数は makeStringInfoInternal の単なるラッパーである。
  • src/include/lib/stringinfo.h にて宣言されており、外部モジュールからも利用可能である。
