md5_crypt_verify

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/crypt.c:201

目的

MD5認証方式において、クライアントから提供されたチャレンジ応答と、システムカタログに保存されたパスワードハッシュを照合する。

シグネチャまたは形状

int md5_crypt_verify(const char *role, const char *shadow_pass,
                     const char *client_pass,
                     const uint8 *md5_salt, int md5_salt_len,
                     const char **logdetail);

入力と出力

  • **入力**:
  • role: 認証対象のロール名。
  • shadow_pass: システムカタログから取得した保存済みパスワード文字列。
  • client_pass: クライアントから送信されたMD5ハッシュ応答。
  • md5_salt: MD5チャレンジに使用したソルト。
  • md5_salt_len: ソルトの長さ。
  • **出力**:
  • 戻り値: 認証成功時に STATUS_OK、失敗時に STATUS_ERROR を返す。
  • logdetail: 認証失敗時の詳細なエラーメッセージ（サーバーログ出力用）。

前提条件と不変条件

  • md5_salt_len は 0 より大きい必要がある（Assert による検証）。
  • 保存された shadow_pass は PASSWORD_TYPE_MD5 形式である必要がある。

アルゴリズムまたは振る舞い

1. get_password_type を呼び出し、shadow_pass がMD5形式であることを確認する。不一致の場合はエラーを返す。 2. pg_md5_encrypt を使用し、保存されたパスワードと提供されたソルトから期待されるハッシュ値を計算する。 3. timingsafe_bcmp を使用して、計算されたハッシュと client_pass を比較する。タイミング攻撃を防ぐため、比較は定数時間で行われる。 4. 一致すれば STATUS_OK、不一致であればエラー詳細を設定して STATUS_ERROR を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c:904
  • **呼び出し先**:
  • get_password_type
  • pg_md5_encrypt
  • timingsafe_bcmp
  • psprintf

関連データ構造

  • PasswordType (列挙型): パスワード形式の判定に使用。

失敗モードと境界条件

  • **形式エラー**: shadow_pass がMD5形式でない場合、認証は失敗し、ログ詳細が設定される。
  • **ハッシュ計算失敗**: pg_md5_encrypt が失敗した場合、エラーを返す。
  • **認証失敗**: パスワードが一致しない場合、STATUS_ERROR を返し、ログ詳細に不一致の旨が記録される。

保守担当者向けメモ

  • 本関数はMD5認証専用である。MD5は非推奨であるため、将来的な廃止の可能性がある。
  • セキュリティ上の理由から、認証失敗の詳細はクライアントには送信されず、logdetail を通じてサーバーログにのみ出力される設計となっている。
