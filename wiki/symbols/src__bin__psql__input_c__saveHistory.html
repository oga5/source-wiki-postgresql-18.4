saveHistory

定義

  • 種別: function
  • 定義箇所: src/bin/psql/input.c:412

目的

メモリ上のコマンド履歴をファイルへ永続化する。

シグネチャまたは形状

static bool saveHistory(char *fname, int max_lines)

入力と出力

  • **入力**:
  • fname: 履歴を保存するファイルパス。
  • max_lines: 保存する履歴の最大行数（負の値は制限なし）。
  • **出力**:
  • 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • fname が /dev/null の場合は書き込みを行わず false を返す。
  • 履歴保存前に encode_history() が呼び出され、履歴内の改行がエスケープされる必要がある。

アルゴリズムまたは振る舞い

1. fname が /dev/null でないことを確認する。 2. encode_history() を実行し、履歴内の改行を変換する。 3. 環境が HAVE_HISTORY_TRUNCATE_FILE および HAVE_APPEND_HISTORY をサポートしている場合:

  • history_truncate_file で履歴ファイルを指定行数に切り詰める。
  • append_history を使用してメモリ上の履歴をファイル末尾に追加する。

    • 4. 上記サポートがない場合:

  • stifle_history で履歴サイズを制限し、write_history でファイル全体を上書きする。

    • 5. 書き込み失敗時は pg_log_error でエラーを記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/input.c 内の終了処理（finishInput 等）から呼び出される。
  • **呼び出し先**:
  • encode_history()
  • history_truncate_file() (条件付き)
  • append_history() (条件付き)
  • stifle_history() (条件付き)
  • write_history() (条件付き)
  • pg_log_error()

関連データ構造

  • history_lines_added: 履歴に追加された行数を追跡するグローバル変数（readline 関連）。

失敗モードと境界条件

  • fname が /dev/null の場合は書き込みをスキップする（macOS 等での chmod エラー回避のため）。
  • append_history はファイルが存在しない場合に失敗する可能性があるため、事前に open でファイル作成を試みる。
  • 同時実行セッション間での競合が発生する可能性がある。

保守担当者向けメモ

  • libedit と readline の両方に対応するため、履歴の切り詰めと追加のロジックが条件分岐されている。
  • 履歴の永続化において、既存ファイルを破壊しないよう可能な限り append_history を優先する設計となっている。
