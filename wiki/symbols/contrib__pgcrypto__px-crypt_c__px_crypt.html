px_crypt

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px-crypt.c:101

目的

パスワードとソルトを受け取り、ソルトのプレフィックスに基づいて適切なハッシュアルゴリズムを選択・実行して、ハッシュ化された文字列を返すディスパッチャー関数です。

シグネチャまたは形状

char *px_crypt(const char *psw, const char *salt, char *buf, unsigned len)

入力と出力

  • **入力**:
  • psw: ハッシュ化対象のパスワード文字列。
  • salt: アルゴリズム識別子を含むソルト文字列。
  • buf: 結果を格納するバッファ。
  • len: buf のサイズ。
  • **出力**: ハッシュ化された文字列を格納した buf へのポインタ。アルゴリズムが見つからない場合は NULL を返します。

前提条件と不変条件

  • CheckBuiltinCryptoMode() を呼び出し、モジュールが安全なモードで動作していることを確認します。
  • px_crypt_list テーブルの走査により、salt の先頭文字列と一致するアルゴリズムを特定します。

アルゴリズムまたは振る舞い

1. CheckBuiltinCryptoMode() を実行し、セキュリティガードを確認します。 2. px_crypt_list を順次走査し、salt の先頭部分が各アルゴリズムの id と一致するかを strncmp で判定します。 3. 一致するアルゴリズムが見つかった場合、そのアルゴリズムの crypt 関数を呼び出し、結果を返します。 4. 一致するアルゴリズムがない、または crypt 関数が定義されていない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgcrypto.c:228 からの呼び出しが確認されています。
  • **呼び出し先**: px_crypt_list に登録された各アルゴリズムの crypt 関数（run_crypt_* 系）。

関連データ構造

  • px_crypt_algo: アルゴリズムの識別子（id）、長さ（id_len）、および実行関数（crypt）を保持する構造体。
  • px_crypt_list: 利用可能なアルゴリズムを定義した静的テーブル。

失敗モードと境界条件

  • salt に対応するアルゴリズムが px_crypt_list 内に存在しない場合、NULL を返します。
  • px_crypt_list の走査中に id_len が 0 のエントリに到達した場合、検索を終了します。

保守担当者向けメモ

  • px_crypt_list の順序は重要です。プレフィックスが短いもの（空文字など）が後に配置されるように管理してください。
  • 実際の暗号計算ロジックは本関数には含まれず、crypt-*.c 各ファイルに委譲されています。
