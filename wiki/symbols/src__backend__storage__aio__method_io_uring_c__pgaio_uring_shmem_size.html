pgaio_uring_shmem_size

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_io_uring.c:261

目的

io_uring を使用した非同期 I/O サブシステムが共有メモリ上で必要とする合計サイズを算出します。

シグネチャまたは形状

size_t pgaio_uring_shmem_size(void)

入力と出力

  • 入力: なし
  • 出力: io_uring コンテキストおよびリングバッファに必要な共有メモリの合計サイズ（バイト単位）。

前提条件と不変条件

  • 本関数を呼び出す前に、カーネルおよび liburing の機能サポート状況を判定するため pgaio_uring_check_capabilities() が実行される必要があります。

アルゴリズムまたは振る舞い

1. pgaio_uring_check_capabilities() を呼び出し、実行環境の機能判定を行います。 2. pgaio_uring_context_shmem_size() を呼び出し、コンテキスト構造体に必要なサイズを取得します。 3. pgaio_uring_ring_shmem_size() を呼び出し、リングバッファに必要なサイズを取得します。 4. add_size() を用いて両者の合計を計算し、その結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/aio/method_io_uring.c 内の pgaio_uring_ops 構造体（shmem_size フィールド）。
  • src/backend/storage/aio/method_io_uring.c 内の共有メモリ初期化処理（ShmemInitStruct 呼び出し時）。
  • 呼び出し先:
  • pgaio_uring_check_capabilities()
  • pgaio_uring_context_shmem_size()
  • pgaio_uring_ring_shmem_size()
  • add_size()

関連データ構造

  • PgAioUringContext: 本関数が算出するメモリサイズに関連する主要なコンテキスト構造体。

失敗モードと境界条件

  • add_size() を使用しているため、サイズ計算時にオーバーフローが発生した場合は PostgreSQL のエラーハンドリング機構により適切に処理されます。

保守担当者向けメモ

  • 本関数は io_uring 関連の共有メモリ確保の起点となります。サイズ計算ロジックを変更する場合は、コンテキストまたはリングバッファの構造体定義と整合性が取れているか確認してください。
