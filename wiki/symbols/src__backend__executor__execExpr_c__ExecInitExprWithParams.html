ExecInitExprWithParams

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExpr.c:179

目的

プランツリー外で評価されるスタンドアロンの式ツリーを、実行可能な ExprState 構造体へ変換（コンパイル）します。

シグネチャまたは形状

ExprState *ExecInitExprWithParams(Expr *node, ParamListInfo ext_params)

入力と出力

  • **入力**:
  • node: コンパイル対象の式ツリー（Expr）。
  • ext_params: 式内で参照可能な外部パラメータリスト（ParamListInfo）。
  • **出力**:
  • 実行準備が完了した ExprState へのポインタ。node が NULL の場合は NULL を返します。

前提条件と不変条件

  • node が NULL でない場合、有効な ExprState が生成されます。
  • 生成された ExprState は、ExecReadyExpr を経て実行可能な状態に遷移済みです。

アルゴリズムまたは振る舞い

1. **初期化**: node が NULL なら即座に NULL を返す。そうでなければ ExprState ノードを確保し、ext_params を設定する。 2. **セットアップ**: ExecCreateExprSetupSteps を呼び出し、タプルデフォーム等の必要な準備ステップを挿入する。 3. **コンパイル**: ExecInitExprRec を再帰的に呼び出し、式ツリーを ExprEvalStep の命令配列へ変換する。 4. **終了処理**: 命令配列の末尾に EEOP_DONE_RETURN を追加し、評価終了を明示する。 5. **準備**: ExecReadyExpr を呼び出し、JIT コンパイルの試行やインタプリタ用データの最終調整を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execPartition.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • ExecCreateExprSetupSteps
  • ExecInitExprRec
  • ExprEvalPushStep
  • ExecReadyExpr

関連データ構造

  • ExprState: 式評価の実行状態を保持するメイン構造体。
  • ExprEvalStep: 式評価の各ステップ（命令）を定義する構造体。
  • ParamListInfo: 外部パラメータの参照情報。

失敗モードと境界条件

  • node が NULL の場合、安全に NULL を返します。
  • 式ツリーが非常に深い場合、ExecInitExprRec 内でスタックオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数はスタンドアロン式のコンパイル用です。プランツリー内の式については ExecInitExpr 等の利用を検討してください。
  • ExprEvalStep の命令順序は execExprInterp.c のインタプリタと密接に結合しているため、変更時は整合性に注意してください。
