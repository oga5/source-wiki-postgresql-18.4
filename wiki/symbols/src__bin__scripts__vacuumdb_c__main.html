目的

vacuumdb コマンドラインユーティリティのメインエントリポイントであり、ユーザー入力の解析、実行オプションの初期化、およびデータベースメンテナンス処理の制御フローを開始します。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。
  • getopt_long を介して、接続情報（ホスト、ポート、ユーザー等）や vacuumingOptions 構造体に格納される各種実行フラグ（--analyze, --full, --jobs 等）を受け取ります。
  • **出力**:
  • 正常終了時に 0、エラー発生時に 1 以上の終了コードを返します。

前提条件と不変条件

  • vacopts 構造体は、デフォルト値（parallel_workers = -1, do_truncate = true 等）で初期化されます。
  • ログ出力機能（pg_logging_init）およびロケール設定が初期化されている必要があります。
  • getopt_long による引数解析において、数値指定が必要なオプション（-j, -P, --min-xid-age 等）は option_parse_int によって範囲チェックが行われます。

アルゴリズムまたは振る舞い

1. **初期化**: vacuumingOptions 構造体をゼロクリアし、デフォルト値を設定します。 2. **環境設定**: プログラム名、ロケール、およびヘルプ/バージョンオプションの処理を行います。 3. **引数解析**: getopt_long を使用してコマンドライン引数をループ処理し、以下の操作を行います。

  • 接続パラメータの保持。
  • vacuumingOptions へのフラグ設定。
  • objfilter（処理対象フィルタ）の更新。
  • simple_string_list_append を用いた対象オブジェクト（テーブルやスキーマ）のリスト構築。

    • 4. **バリデーション**: 解析終了後、指定されたオプションの整合性を確認します（ソース抜粋以降の処理）。 5. **実行**: フィルタリングされた対象に基づき、データベース接続を確立し、vacuum_one_database または vacuum_all_databases を呼び出して SQL コマンドを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（プログラムのエントリポイント）。
  • **呼び出し先**:
  • getopt_long: 引数解析。
  • option_parse_int: 数値引数の検証。
  • simple_string_list_append: 対象オブジェクトのリスト管理。
  • pg_logging_init, get_progname, set_pglocale_pgservice: 初期化ユーティリティ。
  • vacuum_one_database / vacuum_all_databases: (ソース抜粋以降) 実際の処理実行。

関連データ構造

  • vacuumingOptions: VACUUM/ANALYZE の動作を制御するフラグ群を保持。
  • SimpleStringList: 処理対象のテーブルやスキーマ名のリスト。
  • ConnParams: データベース接続設定。
  • enum trivalue: パスワードプロンプトの制御状態。

失敗モードと境界条件

  • **引数エラー**: option_parse_int が範囲外の値を検出した場合、exit(1) を呼び出して終了します。
  • **メモリ不足**: pg_strdup 等のメモリ確保失敗時に終了する可能性があります。
  • **オプションの競合**: check_objfilter()（ソース抜粋以降で呼び出し）により、矛盾するオプション指定が検出された場合にエラーとなります。

保守担当者向けメモ

  • 本関数は vacuumdb の制御フローの起点です。新しいコマンドラインオプションを追加する場合は、long_options 配列への追加と switch 文でのケース処理が必要です。
  • ツールによる呼び出し元調査では、contrib/amcheck 等のコードが「main」という単語を別の文脈（テーブルのメインフォーク等）で使用していることが確認されました。これらは本関数とは無関係です。
