disable_old_cluster

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/controldata.c:754

目的

アップグレード後の旧クラスタの pg_control ファイルをリネームし、誤って旧サーバが起動されることを防止する。

シグネチャまたは形状

void disable_old_cluster(transferMode transfer_mode)

入力と出力

  • 入力: transferMode transfer_mode（転送モード。TRANSFER_MODE_LINK または TRANSFER_MODE_SWAP を想定）
  • 出力: なし（副作用としてファイルシステム上のリネームを実行）

前提条件と不変条件

  • old_cluster.pgdata が有効なパスとして設定されていること。
  • XLOG_CONTROL_FILE が旧クラスタのデータディレクトリ内に存在すること。

アルゴリズムまたは振る舞い

1. old_cluster.pgdata 内の pg_control ファイルパスを構築する。 2. 対象ファイルを pg_control.old にリネームする。リネームに失敗した場合は pg_fatal で処理を中断する。 3. 転送モード（transfer_mode）に応じて、旧クラスタの起動が安全ではない旨の警告メッセージを標準出力へログ記録する。

  • TRANSFER_MODE_LINK の場合: ファイルの復元方法と、リンクモードにおける起動の危険性を通知する。
  • TRANSFER_MODE_SWAP の場合: 起動が安全ではない旨を通知する。
  • 上記以外の場合: pg_fatal でエラーを報告する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/pg_upgrade.c:183
  • 呼び出し先: prep_status, snprintf, pg_mv_file, pg_fatal, check_ok, pg_log

関連データ構造

  • transferMode (列挙型)
  • old_cluster (グローバル変数、ClusterInfo 型)

失敗モードと境界条件

  • pg_mv_file が失敗した場合、pg_fatal によりプログラムが終了する。
  • transfer_mode が定義されたモード以外の場合、pg_fatal によりプログラムが終了する。

保守担当者向けメモ

  • 本関数は pg_upgrade の最終段階で実行される。
  • pg_control のリネームは、旧クラスタの意図しない再起動を防ぐための重要な安全策である。
  • 転送モードごとのメッセージ内容は、将来的な転送方式の追加や変更に応じて更新が必要となる可能性がある。
