int_custom_invalid_stats

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_types/test_custom_types.c:81

目的

PostgreSQLのANALYZE処理において、統計情報が有効ではない状態を意図的にシミュレートするために使用されるコールバック関数です。

シグネチャまたは形状

static void int_custom_invalid_stats(VacAttrStats *stats, AnalyzeAttrFetchFunc fetchfunc, int samplerows, double totalrows);

入力と出力

  • **入力**:
  • stats: 統計情報を格納するVacAttrStats構造体へのポインタ。
  • fetchfunc: 属性値を取得するための関数ポインタ。
  • samplerows: サンプリングされた行数。
  • totalrows: テーブルの総行数。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数はVacAttrStats構造体のcompute_statsフィールドにセットされることを前提としています。
  • 実行後、stats->stats_validは必ずfalseに設定されます。

アルゴリズムまたは振る舞い

1. 引数として受け取ったVacAttrStats構造体のstats_validメンバをfalseに設定します。 2. 統計情報の計算処理は行わず、即座に制御を戻します。これにより、後続の統計収集フェーズで当該属性の統計情報が無効として扱われるようになります。

呼び出し元と呼び出し先

  • **呼び出し元**: int_custom_typanalyze_invalid関数内で、stats->compute_statsに本関数が代入されることで参照されます。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • VacAttrStats: commands/vacuum.hで定義される、ANALYZE処理中の統計情報を保持する構造体。

失敗モードと境界条件

  • 本関数自体はエラーを発生させませんが、この関数が設定された属性に対してANALYZEを実行すると、統計情報が「無効」と見なされるため、オプティマイザが当該属性の統計を利用できなくなります。

保守担当者向けメモ

  • 本関数はテスト専用です。実運用環境のコードで利用してはなりません。
  • int_custom_typanalyze_invalid関数からのみ参照される静的関数です。
