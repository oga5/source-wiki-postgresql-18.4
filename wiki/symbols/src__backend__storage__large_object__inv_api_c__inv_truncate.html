inv_truncate

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:739

目的

ラージオブジェクト（LO）を指定されたサイズに切り詰め（truncate）、不要となったデータページを削除し、必要に応じて切り詰め境界のページを調整する。

シグネチャまたは形状

void inv_truncate(LargeObjectDesc *obj_desc, int64 len)

入力と出力

  • obj_desc: 操作対象のラージオブジェクトを記述する LargeObjectDesc 構造体へのポインタ。
  • len: 切り詰め後の目標サイズ（バイト単位）。
  • 戻り値: なし。

前提条件と不変条件

  • obj_desc は有効なポインタであること。
  • obj_desc に書き込み権限（IFS_WRLOCK）が付与されていること。
  • len は 0 以上かつ MAX_LARGE_OBJECT_SIZE 以下であること。
  • 内部で open_lo_relation() を呼び出し、pg_largeobject カタログへのアクセスを確立する。

アルゴリズムまたは振る舞い

1. **権限とパラメータの検証**: 書き込み権限を確認し、len の範囲をチェックする。 2. **スキャン開始**: pg_largeobject テーブルに対し、対象LOIDかつ切り詰め位置以降のページを順序立ててスキャンする。 3. **境界ページの処理**:

  • 切り詰め位置を含むページが存在する場合、そのデータを読み込み、切り詰め位置までを保持して更新（またはゼロ埋め）し、カタログを更新する。
  • 存在しない場合（穴がある場合）、切り詰め位置に新しいページを作成する。

    • 4. **不要ページの削除**: 切り詰め位置より後のページをすべてカタログから削除する。 5. **可視化**: CommandCounterIncrement() を呼び出し、トランザクション内の後続操作から変更を可視化する。

呼び出し元と呼び出し先

関連データ構造

  • LargeObjectDesc: オープンされたLOの状態を保持。
  • pg_largeobject: LOのデータを格納するシステムカタログ。
  • CatalogIndexState: カタログインデックスの更新状態を管理。

失敗モードと境界条件

  • **権限エラー**: 書き込み権限がない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了する。
  • **パラメータエラー**: len が負または上限を超える場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了する。
  • **データ不整合**: pg_largeobject 内に NULL フィールドが存在する場合、elog でエラー終了する。

保守担当者向けメモ

  • 本関数は pg_largeobject を直接操作するため、トランザクションの整合性に注意が必要である。
  • LOBLKSIZE 単位でのページ管理を行っており、切り詰め位置がページ境界でない場合は、既存データの切り詰めまたはゼロ埋めが行われる。
  • CommandCounterIncrement() の呼び出しにより、本関数の実行後に同一トランザクション内でLOを読み取ると、切り詰め後の状態が正しく反映される。
