ExecEndTidScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidscan.c:480

目的

TidScan ノードの実行を終了し、関連するリソースを解放します。

シグネチャまたは形状

void ExecEndTidScan(TidScanState *node)

入力と出力

  • **入力**: TidScanState *node - 終了対象となる TidScan ノードの状態構造体。
  • **出力**: なし。

前提条件と不変条件

  • node は有効な TidScanState 構造体を指している必要があります。
  • node->ss.ss_currentScanDesc が初期化されている場合、そのスキャン記述子はクローズされます。

アルゴリズムまたは振る舞い

1. node->ss.ss_currentScanDesc が非 NULL であるかを確認します。 2. 非 NULL の場合、table_endscan(node->ss.ss_currentScanDesc) を呼び出し、テーブルアクセスメソッド層のスキャンを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:656 にて呼び出されていることが確認されています。
  • **呼び出し先**: table_endscan（テーブルアクセスメソッドの終了関数）。

関連データ構造

  • TidScanState: TidScan ノードの実行状態を保持する構造体。ss.ss_currentScanDesc を介してスキャン記述子を管理します。

失敗モードと境界条件

  • node->ss.ss_currentScanDesc が NULL の場合、関数は何も行わず正常に終了します。

保守担当者向けメモ

  • 本関数は TidScan ノードのクリーンアップを担当します。メモリ解放等の追加リソース管理が必要な場合は、この関数内で適切に処理してください。
