SyncReplicationSlots

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/slotsync.c:1857

目的

プライマリサーバーから論理レプリケーションスロットの情報を取得し、スタンバイサーバー上のローカルスロットと同期させる。

シグネチャまたは形状

void SyncReplicationSlots(WalReceiverConn *wrconn)

入力と出力

  • **入力**: wrconn (プライマリサーバーへの接続を確立した WalReceiverConn 構造体へのポインタ)
  • **出力**: なし（副作用としてローカルスロットの作成・更新・削除を行う）

前提条件と不変条件

  • wrconn が有効な接続状態であること。
  • 同期処理中、SlotSyncCtx のスピンロックにより、他の同期プロセスとの競合が回避されること。
  • 処理の開始と終了時に同期フラグの管理（check_and_set_sync_info / reset_syncing_flag）が行われること。

アルゴリズムまたは振る舞い

1. PG_ENSURE_ERROR_CLEANUP を設定し、エラー発生時に slotsync_failure_callback が実行されるよう保証する。 2. check_and_set_sync_info を呼び出し、現在のプロセスを同期中としてマークする。 3. validate_remote_info を呼び出し、プライマリ側の情報を検証する。 4. synchronize_slots を呼び出し、プライマリからスロット情報を取得してローカルへ反映する。 5. ReplicationSlotCleanup(true) を呼び出し、同期された一時的なスロットのクリーンアップを行う。 6. reset_syncing_flag を呼び出し、同期フラグをリセットする。 7. PG_END_ENSURE_ERROR_CLEANUP によりクリーンアップを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/slotfuncs.c:933 (pg_sync_replication_slots SQL 関数経由)
  • **呼び出し先**:
  • check_and_set_sync_info
  • validate_remote_info
  • synchronize_slots
  • ReplicationSlotCleanup
  • reset_syncing_flag
  • slotsync_failure_callback (エラー発生時)

関連データ構造

  • WalReceiverConn: プライマリとの通信用接続ハンドル。
  • SlotSyncCtx: 同期プロセスの状態を保持する共有メモリ構造体。

失敗モードと境界条件

  • 接続が切断された場合や、プライマリとの通信エラーが発生した場合は slotsync_failure_callback が呼び出される。
  • スタンバイ側の LSN がプライマリの要求に追いついていない場合、データ損失を防ぐための警告や同期の制限が発生する可能性がある。

保守担当者向けメモ

  • 本関数は、SQL 関数 pg_sync_replication_slots() およびバックグラウンドワーカーの両方から利用される同期のメインエントリーポイントである。
  • 同期処理の整合性は RS_TEMPORARY から RS_PERSISTENT への状態遷移によって保証されているため、変更を加える際はスロットの永続化ロジックに注意すること。
