start_lo_xact

定義

  • 種別: function
  • 定義箇所: src/bin/psql/large_obj.c:55

目的

ラージオブジェクト操作（エクスポート、インポート、削除）を開始する前に、現在のデータベース接続のトランザクション状態を確認し、必要に応じてトランザクションを開始します。

シグネチャまたは形状

static bool start_lo_xact(const char *operation, bool *own_transaction)

入力と出力

  • **入力**:
  • operation: 操作名を示す文字列（エラーログに使用）。
  • own_transaction: 本関数がトランザクションを開始したかどうかを格納するフラグへのポインタ。
  • **出力**:
  • 成功時は true、失敗時は false を返します。
  • own_transaction には、本関数が BEGIN を発行した場合は true、既存のトランザクションを利用する場合は false が設定されます。

前提条件と不変条件

  • pset.db が有効なデータベース接続を保持している必要があります。
  • ラージオブジェクト操作はトランザクションブロック内で行われる必要があります。

アルゴリズムまたは振る舞い

1. pset.db が接続されているか確認し、未接続ならエラーを出力して false を返します。 2. PQtransactionStatus で現在のトランザクション状態を取得します。 3. 状態に応じた分岐処理を行います：

  • PQTRANS_IDLE: トランザクション外のため、PSQLexec("BEGIN") を実行してトランザクションを開始し、own_transaction を true に設定します。
  • PQTRANS_INTRANS: 既存のトランザクションを利用するため、何もせず true を返します。
  • PQTRANS_INERROR: トランザクションが異常終了しているため、エラーを出力して false を返します。
  • その他: 不明な状態としてエラーを出力し false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • do_lo_export (src/bin/psql/large_obj.c:147)
  • do_lo_import (src/bin/psql/large_obj.c:183)
  • do_lo_unlink (src/bin/psql/large_obj.c:245)
  • **呼び出し先**:
  • PQtransactionStatus (libpq)
  • PSQLexec
  • PQclear
  • pg_log_error

関連データ構造

  • pset: 接続情報や出力設定を保持するグローバル構造体。
  • PGTransactionStatusType: libpq が定義するトランザクション状態の列挙型。

失敗モードと境界条件

  • データベース未接続時: false を返し、エラーログを出力。
  • トランザクションが異常終了状態 (PQTRANS_INERROR): false を返し、エラーログを出力。
  • BEGIN コマンドの実行失敗: PSQLexec が失敗した場合、その結果を伝播して false を返す。

保守担当者向けメモ

  • 本関数は libpq のトランザクション状態管理に依存しています。autocommit 設定の有無に関わらず、ラージオブジェクト操作の整合性を保つために明示的なトランザクション制御を行っています。
