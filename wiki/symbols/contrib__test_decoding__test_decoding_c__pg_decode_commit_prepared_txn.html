pg_decode_commit_prepared_txn

定義

  • 種別: function
  • 定義箇所: contrib/test_decoding/test_decoding.c:400

目的

論理デコーディングにおいて、準備済みトランザクション（Prepared Transaction）のコミットイベントをテキスト形式で出力するコールバック関数です。

シグネチャまたは形状

static void pg_decode_commit_prepared_txn(LogicalDecodingContext *ctx, ReorderBufferTXN *txn, XLogRecPtr commit_lsn)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • txn: コミットされる準備済みトランザクションの情報。
  • commit_lsn: コミット時のWALログシーケンス番号。
  • **出力**: なし（ctx->out にテキストを書き込み、OutputPluginWrite を通じて出力）。

前提条件と不変条件

  • ctx->output_plugin_private に TestDecodingData が格納されている必要があります。
  • txn->gid が有効な文字列として存在している必要があります。

アルゴリズムまたは振る舞い

1. OutputPluginPrepareWrite を呼び出し、出力バッファを準備します。 2. COMMIT PREPARED 文字列と、引用符で囲まれたグローバル識別子（txn->gid）をバッファに追加します。 3. data->include_xids が真の場合、トランザクションID（txn->xid）を追記します。 4. data->include_timestamp が真の場合、コミット時刻（txn->xact_time.commit_time）を追記します。 5. OutputPluginWrite を呼び出し、バッファの内容を送信します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの論理デコーディングエンジン（_PG_output_plugin_init で commit_prepared_cb に登録されることで間接的に呼び出されます）。
  • **呼び出し先**:
  • OutputPluginPrepareWrite
  • appendStringInfo
  • quote_literal_cstr
  • timestamptz_to_str
  • OutputPluginWrite

関連データ構造

  • LogicalDecodingContext: デコーディングの実行状態を保持。
  • ReorderBufferTXN: トランザクションの詳細情報（GID、XID、コミット時刻など）を保持。
  • TestDecodingData: プラグインの設定（出力オプション）を保持。

失敗モードと境界条件

  • txn->gid がNULLまたは不正な場合、quote_literal_cstr の挙動に依存します。
  • ctx->out のメモリ不足が発生した場合、appendStringInfo 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は test_decoding プラグインの出力形式を定義するものであり、実運用環境での利用は想定されていません。
  • include_xids や include_timestamp のフラグは、pg_decode_startup で設定された TestDecodingData に依存します。
