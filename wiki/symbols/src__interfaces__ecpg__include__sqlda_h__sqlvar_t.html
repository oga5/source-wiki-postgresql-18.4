sqlvar_t

定義

  • 種別: struct (typedef)
  • 定義箇所: src/interfaces/ecpg/include/sqlda.h:7

目的

ECPG（Embedded SQL for C）において、動的 SQL の結果セットや入力パラメータに含まれる各列（変数）の情報を保持するための汎用的な構造体型を提供します。

シグネチャまたは形状

typedef struct sqlvar_compat sqlvar_t; （※コンパイル時の設定により、sqlvar_compat または sqlvar_struct のいずれかへエイリアスされます）

入力と出力

  • 入力: 動的 SQL の実行時に、列のデータ型、長さ、値へのポインタ、名前などのメタデータおよび実データを格納します。
  • 出力: ECPG ライブラリが SQLDA を介してアプリケーションへデータを渡す際、またはアプリケーションがバインド変数としてデータを渡す際に使用されます。

前提条件と不変条件

  • sqlda_t 構造体の一部として使用されることが前提です。
  • メモリ確保時には sizeof(sqlvar_t) を使用して適切なサイズを割り当てる必要があります。

アルゴリズムまたは振る舞い

  • プリプロセッサ定義（_ECPG_INFORMIX_H の有無）に基づき、Informix 互換モードまたは PostgreSQL ネイティブモードのいずれかの構造体定義に解決されます。
  • 実行時には、malloc によるメモリ確保や memset による初期化が行われる対象となります。

呼び出し元と呼び出し先

  • 呼び出し元（利用例）: src/interfaces/ecpg/test/expected/compat_informix-sqlda.c 等のテストコードにおいて、malloc を用いた動的確保や、sqlda->sqlvar[i] を介したデータアクセスが行われています。
  • 呼び出し先: 本シンボル自体は型定義であるため、直接的な関数呼び出しは発生しません。

関連データ構造

  • sqlda_t: sqlvar_t を配列として保持する親構造体。
  • sqlda-compat.h: Informix 互換モード時の構造体定義元。
  • sqlda-native.h: PostgreSQL ネイティブモード時の構造体定義元。

失敗モードと境界条件

  • malloc によるメモリ確保失敗時に、sqlvar_t へのポインタが NULL になる可能性があります。
  • 構造体のサイズはコンパイル時のモードに依存するため、異なるモードでコンパイルされたオブジェクト間でのバイナリ互換性は保証されません。

保守担当者向けメモ

  • 本シンボルは sqlda.h 内の条件分岐によって実体が切り替わるラッパーです。詳細な構造体メンバを確認する場合は、ビルド設定に応じて sqlda-compat.h または sqlda-native.h を参照してください。
  • 実際のデータ変換ロジックについては src/interfaces/ecpg/ecpglib/sqlda.c を確認してください。
