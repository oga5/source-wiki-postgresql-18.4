euc_kr_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_kr_and_mic/euc_kr_and_mic.c:43

目的

PostgreSQLの文字セット変換フレームワークにおいて、EUC_KR（韓国語）エンコーディングから MULE_INTERNAL（MIC）エンコーディングへの変換を実行する。

シグネチャまたは形状

Datum euc_kr_to_mic(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取る。
  • src (CSTRING): 変換元の EUC_KR 文字列。
  • dest (CSTRING): 変換結果を格納するバッファ。
  • len (INT32): 入力文字列の長さ。
  • noError (BOOL): 変換エラー時に例外を投げるか否かのフラグ。
  • **出力**: Datum 型として、変換されたバイト数を返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_EUC_KR, PG_MULE_INTERNAL) マクロにより、エンコーディングIDの整合性が検証される。
  • noError が true の場合、変換エラーが発生しても例外を発生させず、処理を中断して変換済みのバイト数を返す。

アルゴリズムまたは振る舞い

1. fmgr インターフェースを介して引数を取得する。 2. エンコーディングIDの検証を行う。 3. 内部関数 euc_kr2mic を呼び出し、実際の変換処理（ASCII文字のコピーおよび LC_KS5601 プレフィックスを用いた2バイト文字の変換）を実行する。 4. 変換されたバイト数を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(euc_kr_to_mic) マクロによる登録を通じて、PostgreSQLの関数マネージャから呼び出される。
  • **呼び出し先**: euc_kr2mic（実際のバイト変換ロジック）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数引数管理用構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • 入力エンコーディングまたは出力エンコーディングが期待するものと一致しない場合、CHECK_ENCODING_CONVERSION_ARGS によりエラーが発生する可能性がある。
  • noError フラグの設定により、不正な文字が含まれる場合の挙動が制御される。

保守担当者向けメモ

  • 本関数は fmgr インターフェースに準拠したラッパーである。実際の変換ロジックは euc_kr2mic に依存しているため、変換アルゴリズムの変更が必要な場合はそちらを参照すること。
