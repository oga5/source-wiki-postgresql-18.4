pgstat_report_recovery_conflict

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_database.c:80

目的

ホットスタンバイ環境において発生したリカバリ競合（Recovery Conflict）の回数を、データベース単位の統計情報として記録する。

シグネチャまたは形状

void pgstat_report_recovery_conflict(int reason)

入力と出力

  • **入力**: reason (int) - 発生したリカバリ競合の種類を示す識別子（storage/procsignal.h で定義された PROCSIG_RECOVERY_CONFLICT_* 値）。
  • **出力**: なし（副作用として、現在のデータベースに対応する統計構造体のカウンタをインクリメントする）。

前提条件と不変条件

  • IsUnderPostmaster が真であること（Postmaster配下で動作していること）。
  • pgstat_track_counts が有効であること。無効な場合は何も行わずに終了する。

アルゴリズムまたは振る舞い

1. pgstat_prep_database_pending(MyDatabaseId) を呼び出し、現在のデータベースの統計エントリを取得または作成する。 2. reason に基づき、以下のいずれかのカウンタをインクリメントする：

  • conflict_tablespace
  • conflict_lock
  • conflict_snapshot
  • conflict_bufferpin
  • conflict_logicalslot
  • conflict_startup_deadlock

    • 3. PROCSIG_RECOVERY_CONFLICT_DATABASE が指定された場合は、データベース削除に関する競合であるため、統計には記録せず無視する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/postgres.c 内のリカバリ競合処理箇所（3221行目、3240行目）。
  • **呼び出し先**: pgstat_prep_database_pending（統計エントリの準備）。

関連データ構造

  • PgStat_StatDBEntry: データベース単位の統計情報を保持する構造体。
  • PROCSIG_RECOVERY_CONFLICT_*: 競合理由を定義する列挙型定数。

失敗モードと境界条件

  • pgstat_track_counts が無効な場合、統計は更新されない。
  • PROCSIG_RECOVERY_CONFLICT_DATABASE は意図的に記録されない。
  • 渡された reason が上記スイッチ文のいずれにも該当しない場合、何も更新されない。

保守担当者向けメモ

  • 本関数はホットスタンバイ時の統計収集に特化している。
  • 統計情報の更新はローカルな PgStat_StatDBEntry に対して行われるため、共有メモリへの反映は後続のフラッシュ処理（pgstat_database_flush_cb 等）に依存する。
