pg_read_file_off_len

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:284

目的

サーバー上の指定されたファイルから、オフセットと長さを指定してテキストデータを読み込み、SQLインターフェースへ返却します。

シグネチャまたは形状

Datum pg_read_file_off_len(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): 読み込むファイルパス（text型）。
  • PG_GETARG_INT64(1): 読み込み開始位置のオフセット（int64）。
  • PG_GETARG_INT64(2): 読み込むバイト数（int64）。
  • **出力**:
  • 読み込んだファイル内容を格納した text 型の Datum。ファイルが読み込めない場合は NULL を返します。

前提条件と不変条件

  • 呼び出し元は、PostgreSQLの権限管理システム（pg_read_server_filesロール等）によるアクセス権限チェックを通過している必要があります。
  • 指定されたファイルパスは、convert_and_check_filename を通じてデータディレクトリまたはログディレクトリ内に制限されている必要があります。

アルゴリズムまたは振る舞い

1. 引数からファイルパス、オフセット、読み込み長を取得します。 2. pg_read_file_common を呼び出し、以下の引数でファイル読み込みを実行します。

  • missing_ok = false: ファイルが存在しない場合はエラーを発生させます。
  • include_binary = false: テキストとして読み込みます。

    • 3. pg_read_file_common が NULL を返した場合は、SQLの NULL を返却します。 4. 正常に読み込めた場合、結果を text 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_read_file_common
  • **呼び出し元**: SQLインターフェース（pg_proc.h経由で定義されたSQL関数）から呼び出されます。具体的なSQLレベルの呼び出し元情報はソースコード上には明示されていません。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • ファイルが存在しない場合、pg_read_file_common の仕様によりエラーが発生します。
  • 権限がない場合、convert_and_check_filename 内でエラーが発生します。
  • オフセットや読み込み長がファイルサイズを超える場合、または不正な値である場合の挙動は pg_read_file_common の実装に依存します。

保守担当者向けメモ

  • 本関数は pg_read_file_off_len_missing 等のバリエーション関数と対になっています。機能変更時は、関連するラッパー関数群と一貫性を保つようにしてください。
  • pg_read_file_common の引数フラグを変更する際は、本関数の要件（missing_ok=false）が意図通りか確認してください。
