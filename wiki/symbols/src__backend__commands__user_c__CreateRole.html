CreateRole

定義

  • 種別: function
  • 定義箇所: src/backend/commands/user.c:131

目的

PostgreSQLにおいて、CREATE ROLE、CREATE USER、または CREATE GROUP コマンドを実行し、新しいロールをシステムに登録します。

シグネチャまたは形状

Oid CreateRole(ParseState *pstate, CreateRoleStmt *stmt)

入力と出力

  • **入力**:
  • pstate: パーサーの状態。
  • stmt: 実行対象の CreateRoleStmt 構造体（ロール名、パスワード、権限属性、メンバーシップ情報などを含む）。
  • **出力**:
  • 新しく作成されたロールの Oid を返します。

前提条件と不変条件

  • stmt->stmt_type に応じて、canlogin などのデフォルト値が決定されます。
  • SYSID オプションは指定できません（指定された場合は通知を発行）。
  • connectionlimit が指定された場合、値が -1 未満であればエラーとなります。
  • オプションの重複指定は errorConflictingDefElem により禁止されています。

アルゴリズムまたは振る舞い

1. **初期化**: ロール属性（issuper, inherit, createrole 等）のデフォルト値を設定します。 2. **オプション解析**: stmt->options リストを走査し、各 DefElem からロールの属性値を抽出します。 3. **バリデーション**: 抽出された値の妥当性を確認し、不正な値や重複がある場合はエラーを発生させます。 4. **カタログ更新**: 抽出された属性に基づき、pg_authid カタログへの挿入準備を行います（ソース抜粋以降の処理）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:907（パーサーからのディスパッチ）。
  • **呼び出し先**: ソース抜粋内では、GetUserId(), errorConflictingDefElem(), strVal(), boolVal(), intVal(), ereport(), elog() 等のユーティリティ関数が使用されています。

関連データ構造

  • CreateRoleStmt: SQL文の解析結果を保持する構造体。
  • DefElem: オプション定義を保持する構造体。
  • GrantRoleOptions: ロール付与オプションを管理する構造体。
  • pg_authid: ロール情報を格納するシステムカタログ。

失敗モードと境界条件

  • **エラー**:
  • 未知のオプションが指定された場合。
  • オプションが重複して指定された場合。
  • connectionlimit に -1 未満の値が指定された場合。
  • **境界条件**: ROLESTMT_USER 型の場合、デフォルトで canlogin が true に設定されます。

保守担当者向けメモ

  • 本関数は pg_authid カタログを直接操作する基盤ロジックです。
  • check_password_hook が存在する場合、パスワード検証がフックされる可能性があります。
  • メンバーシップの追加（addroleto 等）は、この関数の後半（ソース抜粋外）で処理されます。
