SessionBackupState

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlog.h:291

目的

現在のセッションにおけるバックアップ状態を管理・追跡するために使用される列挙型です。

シグネチャまたは形状

typedef enum SessionBackupState
{
    SESSION_BACKUP_NONE,    /* バックアップ未実行 */
    SESSION_BACKUP_RUNNING, /* バックアップ実行中 */
} SessionBackupState;

入力と出力

  • 入力: なし（状態定義）
  • 出力: SESSION_BACKUP_NONE または SESSION_BACKUP_RUNNING のいずれかの値。

前提条件と不変条件

  • システムは、バックアップの開始および終了のセッションレベルの状態を保持します。
  • SESSION_BACKUP_NONE がデフォルトの状態です。

アルゴリズムまたは振る舞い

  • get_backup_status() 関数を通じて、現在のバックアップ状態を取得可能です。
  • バックアップ処理の開始時に SESSION_BACKUP_RUNNING へ遷移し、終了時に SESSION_BACKUP_NONE へ戻ることで、セッション単位でのバックアップ実行状況を制御します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/access/transam/xlog.c: 静的変数 sessionBackupState の型として使用。
  • src/backend/access/transam/xlogfuncs.c: get_backup_status() の戻り値として使用。
  • src/backend/backup/basebackup.c: get_backup_status() の戻り値として使用。
  • 呼び出し先:
  • 外部関数 get_backup_status(void) が本型を返します。

関連データ構造

  • src/include/access/xlog.h 内のバックアップ関連 API と密接に関連しています。

失敗モードと境界条件

  • 状態遷移の整合性（バックアップが開始されていない状態で終了処理が行われる等）については、本型定義自体には含まれず、実装側のロジックに依存します。

保守担当者向けメモ

  • 本型はセッションレベルの状態管理に使用されるため、バックアップ関連のロジックを変更する際は、この列挙値の遷移が正しく行われているかを確認してください。
