pqTraceOutput_DataRow

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-trace.c:292

目的

PostgreSQL プロトコルにおける DataRow メッセージの内容を解析し、デバッグ用のトレース出力ストリームへ人間が読みやすい形式で書き出す。

シグネチャまたは形状

static void pqTraceOutput_DataRow(FILE *f, const char *message, int *cursor)

入力と出力

  • f: 出力先の FILE ポインタ。
  • message: 解析対象のメッセージデータを含むバッファ。
  • cursor: 現在の解析位置を示すポインタ。関数内でメッセージの読み取りに伴い更新される。
  • 出力: なし（副作用として f にテキストを出力）。

前提条件と不変条件

  • cursor は、メッセージ内の DataRow 型バイトの直後を指している必要がある。
  • message は有効なプロトコルメッセージのバイト列であること。

アルゴリズムまたは振る舞い

1. ストリーム f に "DataRow" というラベルを出力する。 2. pqTraceOutputInt16 を呼び出し、メッセージに含まれるフィールド数（nfields）を取得する。 3. nfields 回ループし、各フィールドを処理する:

  • pqTraceOutputInt32 を呼び出し、フィールド長（len）を取得する。
  • len が -1 の場合は NULL 値としてスキップする。
  • それ以外の場合は pqTraceOutputNchar を呼び出し、指定された長さのデータを文字列として出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-trace.c 内のメッセージディスパッチャ（pqTraceOutputMessage 等から呼び出される）。
  • 呼び出し先: pqTraceOutputInt16, pqTraceOutputInt32, pqTraceOutputNchar。

関連データ構造

  • FILE: 標準ライブラリのストリーム構造体。
  • PGconn: 本関数が直接参照するわけではないが、呼び出し元を通じて Pfdebug（トレース出力先）が提供される。

失敗モードと境界条件

  • len が -1 の場合、当該フィールドは NULL として扱われ、データ出力は行われない。
  • メッセージの構造が不正な場合、cursor の進み方が不整合を起こす可能性がある。

保守担当者向けメモ

  • 本関数はプロトコル v3 の DataRow メッセージ専用である。
  • フィールドの解析ロジックを変更する場合は、pqTraceOutputInt32 および pqTraceOutputNchar の仕様と整合性を取ること。
