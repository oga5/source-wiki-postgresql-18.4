MultiExecPrivateHash

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:137

目的

ハッシュ結合の構築フェーズにおいて、入力リレーションからタプルを読み込み、メモリ上のハッシュテーブルへ挿入する。

シグネチャまたは形状

static void MultiExecPrivateHash(HashState *node)

入力と出力

  • **入力**: HashState *node（ハッシュ結合の実行状態を保持するノード構造体）。
  • **出力**: なし（副作用として node->hashtable が更新される）。

前提条件と不変条件

  • node->hashtable が初期化済みであること。
  • node の子プランノード（outerPlanState）からタプルが取得可能であること。

アルゴリズムまたは振る舞い

1. 子ノードから ExecProcNode を用いてタプルを順次取得する。 2. 各タプルに対してハッシュ式を評価し、ハッシュ値を算出する。 3. スキュー最適化の対象である場合、ExecHashSkewTableInsert を用いて専用のバケットに挿入する。 4. 通常のタプルは ExecHashTableInsert を用いてハッシュテーブルに挿入する。 5. 全タプルの処理後、現在のバケット数が最適値と異なる場合は ExecHashIncreaseNumBuckets を呼び出して拡張する。 6. 最終的に hashtable->spaceUsed を更新し、統計情報を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は src/backend/executor/nodeHash.c 内の関数と推測されるが、具体的な上位関数名は提供されていない。
  • **呼び出し先**:
  • outerPlanState(node)
  • ExecProcNode(outerNode)
  • ExecEvalExprSwitchContext
  • ExecHashGetSkewBucket
  • ExecHashSkewTableInsert
  • ExecHashTableInsert
  • ExecHashIncreaseNumBuckets

関連データ構造

  • HashState: 実行ノードの状態を保持。
  • HashJoinTable: ハッシュテーブルの制御ブロック。
  • TupleTableSlot: 処理対象のタプルを保持。

失敗モードと境界条件

  • **メモリ不足**: ExecHashTableInsert 内部でメモリ制限に達した場合、バッチ処理（ディスクへのスピル）へ移行する可能性がある。
  • **スキュー**: 頻出値（MCV）が存在する場合、スキューバケットへの挿入処理が行われる。
  • **バケット拡張**: NTUP_PER_BUCKET を超過した場合、バケット数の再計算と拡張が発生する。

保守担当者向けメモ

  • 本関数は並列処理を行わない「Private」な構築パスである。並列実行時の挙動については MultiExecParallelHash を参照すること。
  • hashtable->spaceUsed の計算には HashJoinTuple のサイズが使用されており、メモリ使用量の監視において重要な役割を果たす。
