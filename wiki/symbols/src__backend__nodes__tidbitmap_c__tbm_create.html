tbm_create

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/tidbitmap.c:254

目的

TID（タプル識別子）の集合を管理するための TIDBitmap 構造体を初期化し、空の状態で生成する。

シグネチャまたは形状

TIDBitmap *tbm_create(Size maxbytes, dsa_area *dsa)

入力と出力

  • **入力**:
  • maxbytes: ビットマップが使用可能な最大メモリサイズ（バイト単位）。
  • dsa: 動的共有メモリ領域へのポインタ（並列クエリ用）。ローカル使用時は NULL を指定。
  • **出力**:
  • 初期化済みの TIDBitmap 構造体へのポインタ。

前提条件と不変条件

  • makeNode(TIDBitmap) によりメモリが確保され、全フィールドがゼロクリアされる。
  • status は TBM_EMPTY に設定される。
  • maxentries は maxbytes に基づいて算出される。
  • DSA 関連のポインタ（dsapagetable 等）は InvalidDsaPointer で初期化される。

アルゴリズムまたは振る舞い

1. makeNode を呼び出し、TIDBitmap 構造体を割り当てる。 2. 現在のメモリコンテキストを mcxt に保存する。 3. tbm_calculate_entries を用いて、指定された maxbytes から許容される最大エントリ数を算出する。 4. 共有メモリ領域（dsa）および各ポインタを初期化し、空のビットマップとして構築する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginget.c
  • src/backend/executor/nodeBitmapIndexscan.c
  • src/backend/executor/nodeBitmapOr.c
  • **呼び出し先**:
  • makeNode（マクロ）
  • tbm_calculate_entries

関連データ構造

  • TIDBitmap: 本関数が生成するメイン構造体。
  • dsa_area: 並列クエリ用の動的共有メモリ領域。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性がある。
  • maxbytes が極端に小さい場合、tbm_calculate_entries の計算結果に依存する。

保守担当者向けメモ

  • 本関数はビットマップの初期状態を定義する。並列クエリ対応のため、dsa 引数が NULL か否かで動作が分岐する設計となっている。
