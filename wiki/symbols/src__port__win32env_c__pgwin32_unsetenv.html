pgwin32_unsetenv

定義

  • 種別: function
  • 定義箇所: src/port/win32env.c:149

目的

Windows 環境において、POSIX 互換の unsetenv セマンティクスを提供します。プロセス環境およびロードされているすべての C ランタイムライブラリ（CRT）間で環境変数の削除を同期させます。

シグネチャまたは形状

int pgwin32_unsetenv(const char *name);

入力と出力

  • **入力**: name (削除対象の環境変数名)
  • **出力**: 成功時は 0、失敗時は -1 を返します。失敗時には errno が設定されます。

前提条件と不変条件

  • name は NULL であってはならず、空文字列でもなく、かつ = 文字を含んではなりません。これらに違反した場合、errno に EINVAL が設定されます。
  • 内部でメモリ確保を行うため、メモリ不足時には失敗します。

アルゴリズムまたは振る舞い

1. 引数 name の妥当性を検証します。 2. name に = を付加した文字列を保持するためのバッファを動的に確保します。 3. 確保したバッファに name= という形式で文字列を書き込みます。 4. pgwin32_putenv を呼び出し、OS レベルおよび全 CRT モジュールに対して環境変数の削除（空値の設定）を伝播させます。 5. 確保したバッファを解放し、pgwin32_putenv の結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて unsetenv マクロとして定義されており、POSIX 互換を求めるコードから呼び出されます。
  • **呼び出し先**: pgwin32_putenv (同期処理のため)。

関連データ構造

  • なし。

失敗モードと境界条件

  • name が NULL、空、または = を含む場合: EINVAL を返し失敗します。
  • malloc が失敗した場合: -1 を返します。
  • pgwin32_putenv が失敗した場合: その戻り値をそのまま返します。

保守担当者向けメモ

  • 本関数は pgwin32_putenv に依存しています。CRT 同期ロジックに変更が必要な場合は pgwin32_putenv を参照してください。
  • modulenames に含まれない新しい CRT が導入された場合は、モジュール全体の同期リストを更新する必要があります。
