PLy_cursor_iternext

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_cursorobject.c:335

目的

Python のイテレータプロトコル（__next__）を実装し、PostgreSQL カーソルから次の 1 行を取得して Python オブジェクトとして返します。

シグネチャまたは形状

static PyObject *PLy_cursor_iternext(PyObject *self)

入力と出力

  • **入力**: self（PLyCursorObject 型の Python オブジェクト）
  • **出力**: 成功時は取得した行を表す PyObject、反復終了時は NULL（かつ StopIteration を設定）、エラー時は NULL（かつ Python 例外を設定）。

前提条件と不変条件

  • self は PLyCursorObject にキャスト可能であること。
  • カーソルがクローズされていないこと。
  • カーソルに関連付けられたポータルが有効であること（サブトランザクションのアボートにより無効化されていないこと）。

アルゴリズムまたは振る舞い

1. カーソルの状態を確認し、クローズ済みまたはポータルが無効な場合は例外を発生させる。 2. SPI サブトランザクションを開始する。 3. PG_TRY ブロック内で SPI_cursor_fetch を呼び出し、カーソルから 1 行を取得する。 4. 取得結果が 0 行の場合、StopIteration を設定して終了する。 5. 取得結果がある場合、PLy_input_setup_tuple で型情報を設定し、PLy_input_from_tuple で Python オブジェクトへ変換する。 6. 取得したタプルテーブルを解放し、サブトランザクションをコミットする。 7. PG_CATCH ブロックでエラーを捕捉した場合はサブトランザクションをアボートし、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Python インタプリタのイテレータプロトコル（Py_tp_iternext スロット経由）。
  • **呼び出し先**: GetPortalByName, PLy_spi_subtransaction_begin, SPI_cursor_fetch, PLy_input_setup_tuple, PLy_input_from_tuple, SPI_freetuptable, PLy_spi_subtransaction_commit, PLy_spi_subtransaction_abort。

関連データ構造

  • PLyCursorObject: カーソル状態（ポータル名等）を保持。
  • Portal: PostgreSQL のカーソル実体。
  • SPI_tuptable: フェッチされた行データ。

失敗モードと境界条件

  • **ValueError**: カーソルが既に閉じている、またはサブトランザクションのアボートによりポータルが無効な場合に発生。
  • **StopIteration**: カーソルの終端に達した際に発生。
  • **SPI エラー**: SPI_cursor_fetch 等の内部操作が失敗した場合、サブトランザクションがアボートされ、エラーが伝播する。

保守担当者向けメモ

  • 本関数は Python の C API を使用しており、参照カウントの管理に注意が必要です。
  • SPI_tuptable の解放漏れを防ぐため、SPI_freetuptable は必ず呼び出される必要があります。
  • サブトランザクションの管理は PLy_spi_subtransaction_* 関数群に依存しています。
