create_logical_replication_slots

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.c:976

目的

アップグレード後の新しいクラスタにおいて、旧クラスタから引き継いだ論理レプリケーションスロットを再作成する。

シグネチャまたは形状

static void create_logical_replication_slots(void)

入力と出力

  • **入力**: グローバル変数 old_cluster（旧クラスタのデータベース情報およびスロット情報）および new_cluster（接続先クラスタ情報）。
  • **出力**: なし（副作用として新クラスタへスロットを作成）。

前提条件と不変条件

  • old_cluster が初期化され、各データベースの slot_arr にスロット情報が格納されていること。
  • 新クラスタが起動しており、connectToServer を通じて接続可能であること。

アルゴリズムまたは振る舞い

1. 進捗状況を表示し、旧クラスタの全データベースをループ処理する。 2. スロットを持たないデータベースはスキップする。 3. データベースごとに新クラスタへ接続し、pg_catalog.pg_create_logical_replication_slot を呼び出すクエリを構築する。 4. スロット名、プラグイン名、two_phase（2フェーズコミット）、failover（フェイルオーバー）の各属性をクエリに含める。 5. executeQueryOrDie を使用してクエリを実行し、スロットを作成する。 6. 全データベースの処理完了後、進捗表示を終了し、成功を確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/pg_upgrade.c 内の main 関数（行212）。
  • **呼び出し先**:
  • prep_status_progress, end_progress_output, check_ok (進捗管理)
  • connectToServer (新クラスタへの接続)
  • createPQExpBuffer, appendPQExpBufferStr, appendStringLiteralConn, appendPQExpBuffer, resetPQExpBuffer, destroyPQExpBuffer (クエリ構築)
  • executeQueryOrDie (クエリ実行)
  • PQclear, PQfinish (libpq リソース解放)

関連データ構造

  • old_cluster: アップグレード元クラスタ情報を保持するグローバル構造体。
  • DbInfo: データベース情報を保持する構造体。
  • LogicalSlotInfo: 論理レプリケーションスロットの属性（名前、プラグイン、2フェーズコミット可否、フェイルオーバー可否）を保持する構造体。
  • LogicalSlotInfoArr: スロット情報の配列を保持する構造体。

失敗モードと境界条件

  • スロットが存在しないデータベースでは、ループ内で continue により処理をスキップする。
  • executeQueryOrDie が失敗した場合、内部でエラーが発生し処理が中断される可能性がある。
  • 新クラスタへの接続に失敗した場合、connectToServer がエラーを発生させる。

保守担当者向けメモ

  • 本関数は pg_upgrade の最終段階で実行される。
  • スロットの属性（two_phase, failover）は LogicalSlotInfo から取得されるため、旧クラスタからの情報収集ロジックに変更があった場合は本関数のクエリ構築部も確認すること。
