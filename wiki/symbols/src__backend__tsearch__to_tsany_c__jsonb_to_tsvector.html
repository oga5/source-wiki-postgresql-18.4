jsonb_to_tsvector

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/to_tsany.c:343

目的

JSONB データを全文検索用の tsvector 型へ変換する SQL インターフェース関数です。

シグネチャまたは形状

Datum jsonb_to_tsvector(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_JSONB_P(0): 変換対象の Jsonb データ。
  • PG_GETARG_JSONB_P(1): インデックス作成オプションを指定する Jsonb フラグ。
  • **出力**:
  • PG_RETURN_TSVECTOR(result): 変換された TSVector 型のデータ。

前提条件と不変条件

  • 現在の全文検索設定（getTSCurrentConfig(true)）が有効である必要があります。
  • 入力された Jsonb ポインタは、処理終了後に PG_FREE_IF_COPY によって適切にメモリ解放されます。

アルゴリズムまたは振る舞い

1. parse_jsonb_index_flags を呼び出し、第2引数の JSONB からインデックス作成用のフラグ（uint32）を抽出します。 2. getTSCurrentConfig(true) を使用して、現在のセッション設定から cfgId を取得します。 3. jsonb_to_tsvector_worker を呼び出し、cfgId、入力データ、フラグを渡して tsvector を生成します。 4. 入力に使用したメモリを解放し、生成された result を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（直接的な C 言語レベルの呼び出し元情報は提供されていません）。
  • **呼び出し先**:
  • parse_jsonb_index_flags
  • getTSCurrentConfig
  • jsonb_to_tsvector_worker

関連データ構造

  • Jsonb: 入力データ構造。
  • TSVector: 出力データ構造。

失敗モードと境界条件

  • getTSCurrentConfig が設定を取得できない場合、例外が発生する可能性があります。
  • jsonb_to_tsvector_worker 内部での解析エラーが発生した場合は、PostgreSQL のエラーハンドリング機構により処理が中断されます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の変換ロジックは jsonb_to_tsvector_worker に集約されています。
  • JSONB の走査ロジックを変更する場合は jsonb_to_tsvector_worker を参照してください。
