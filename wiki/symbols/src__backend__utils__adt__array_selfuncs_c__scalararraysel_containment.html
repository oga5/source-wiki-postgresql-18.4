scalararraysel_containment

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_selfuncs.c:80

目的

ANY または ALL 演算子を用いた配列の包含関係（@> や <@）に関するクエリの選択率（Selectivity）を推定します。

シグネチャまたは形状

Selectivity scalararraysel_containment(PlannerInfo *root,
                                       Node *leftop, Node *rightop,
                                       Oid elemtype, bool isEquality, bool useOr,
                                       int varRelid);

入力と出力

  • **入力**:
  • root: プランナ情報。
  • leftop: 定数配列（Const ノードである必要がある）。
  • rightop: 変数（カラム）。
  • elemtype: 配列要素の型 OID。
  • isEquality: 等価演算子かどうか。
  • useOr: ANY（useOr=true）か ALL（useOr=false）か。
  • varRelid: 変数のリレーションID。
  • **出力**: 0.0 から 1.0 の範囲の選択率（Selectivity）。失敗時は -1.0 を返却。

前提条件と不変条件

  • rightop は変数である必要があります。
  • leftop は Const ノードである必要があります。
  • leftop が NULL の場合、選択率は 0.0 となります。
  • 統計情報が存在しない場合や、型情報の取得に失敗した場合は -1.0 を返却し、プランナのデフォルト推定に委ねます。

アルゴリズムまたは振る舞い

1. **検証**: rightop が変数、leftop が定数であることを確認します。 2. **統計取得**: pg_statistic から最頻要素（MCELEM）および要素数ヒストグラム（DECHIST）を取得します。 3. **推定計算**:

  • ANY (useOr=true): mcelem_array_contain_overlap_selec を呼び出し、@> 演算子として推定します。
  • ALL (useOr=false): mcelem_array_contained_selec を呼び出し、<@ 演算子として推定します。

    • 4. **補正**:

  • 統計情報がある場合、stanullfrac（NULL率）を用いて結果を補正します。
  • 演算子が <> の場合、最終結果を 1.0 - selec で反転させます。

    • 5. **クランプ**: 最終的な値が 0.0〜1.0 の範囲内に収まるよう CLAMP_PROBABILITY マクロで調整します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/selfuncs.c（scalararraysel 等から参照される可能性あり）
  • **呼び出し先**:
  • examine_variable, lookup_type_cache, get_attstatsslot, mcelem_array_contain_overlap_selec, mcelem_array_contained_selec 等。

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • VariableStatData: 変数の統計情報保持用構造体。
  • TypeCacheEntry: 型情報のキャッシュ。
  • AttStatsSlot: pg_statistic から抽出された統計スロット。

失敗モードと境界条件

  • rightop が変数でない場合や、leftop が定数でない場合は -1.0 を返します。
  • 要素型の比較関数が見つからない場合、-1.0 を返します。
  • 統計情報が欠落している場合、統計なしの計算ロジックへフォールバックします。

保守担当者向けメモ

  • 本関数は ANY/ALL 演算子の選択率推定のエントリポイントです。
  • 統計情報の有無によって計算パスが分岐するため、修正時は mcelem_array_contain_overlap_selec および mcelem_array_contained_selec との整合性に注意してください。
