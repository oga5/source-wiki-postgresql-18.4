gbt_datekey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_date.c:72

目的

GiST インデックス構築時における dateKEY 構造体同士の比較を行い、ソート順序を決定する。

シグネチャまたは形状

static int gbt_datekey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ（内部で dateKEY にキャストされる）。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されていない）。
  • **出力**:
  • int: 比較結果。a < b なら負、a == b なら 0、a > b なら正の値を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体として有効なメモリ領域を指している必要がある。
  • dateKEY 構造体内の lower および upper フィールドは、有効な DateADT 値を保持している必要がある。

アルゴリズムまたは振る舞い

1. a と b の lower フィールドを抽出し、PostgreSQL コアの date_cmp 関数を用いて比較する。 2. lower の比較結果が 0（等しい）でない場合、その結果を直ちに返す。 3. lower が等しい場合、upper フィールドを抽出し、同様に date_cmp を用いて比較した結果を返す。 4. 比較には DirectFunctionCall2 を使用し、PostgreSQL の型システムを介して実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_date.c 内のソートサポート関連の初期化箇所（112行目付近）で参照されている。
  • **呼び出し先**:
  • date_cmp (PostgreSQL コア関数)

関連データ構造

  • dateKEY: lower と upper を保持する範囲構造体。
  • Nsrt: ソート処理のために dateKEY をラップする構造体。

失敗モードと境界条件

  • dateKEY 内の lower や upper が不正な日付データである場合、date_cmp の挙動に依存する。
  • 入力ポインタが NULL の場合の挙動は定義されていない。

保守担当者向けメモ

  • 本関数は date 型のソート順序を定義するものであり、インデックスの構築効率に直接影響する。
  • date_cmp への依存は PostgreSQL コアの仕様変更に影響を受ける可能性がある。
