PLy_output_setup_record

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_typeio.c:260

目的

RECORD型のSQL出力変換に必要なメタデータを初期化し、変換構造体（PLyObToDatum）を準備する。

シグネチャまたは形状

void PLy_output_setup_record(PLyObToDatum *arg, TupleDesc desc, PLyProcedure *proc)

入力と出力

  • **入力**:
  • arg: 変換状態を保持する PLyObToDatum 構造体へのポインタ。
  • desc: 対象となるレコードの TupleDesc。
  • proc: 現在のPL/Pythonプロシージャ情報。
  • **出力**: なし（arg 構造体の内部状態を更新）。

前提条件と不変条件

  • arg->typoid および desc->tdtypeid は共に RECORDOID である必要がある（Assertによる検証）。

アルゴリズムまたは振る舞い

1. BlessTupleDesc(desc) を呼び出し、レコード型をシステムカタログで認識可能な状態にする。 2. arg->typmod を desc->tdtypmod で更新する。 3. arg->u.tuple.recdesc（キャッシュされたタプル記述子）が現在の typmod と一致しない場合、これを NULL にリセットし、後続の PLyObject_ToComposite での再ルックアップを強制する。 4. PLy_output_setup_tuple を呼び出し、タプル変換に必要な派生データを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_exec.c:246
  • **呼び出し先**: BlessTupleDesc, PLy_output_setup_tuple

関連データ構造

  • PLyObToDatum: 変換ロジックと型情報を保持する構造体。
  • TupleDesc: タプルの構造を記述するPostgreSQLのデータ構造。
  • PLyProcedure: PL/Python関数の実行状態を管理する構造体。

失敗モードと境界条件

  • RECORDOID 以外の型が渡された場合、Assert が有効なビルドではプログラムが停止する。
  • BlessTupleDesc の失敗はPostgreSQLの標準的なエラーハンドリング（elog(ERROR)）に依存する。

保守担当者向けメモ

  • 本関数は、実行時に型が決定するRECORD型に対して、動的に TupleDesc を解決・キャッシュするためのパスを構築する。
  • arg->u.tuple.recdesc のリセット処理は、型定義の変更を正しく追従するために不可欠である。
