ts_parserstate

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/tsquery.c:40

目的

tsquery の解析プロセスにおいて、現在の構文解析状態を追跡するために使用されます。

シグネチャまたは形状

typedef enum
{
	WAITOPERAND = 1,
	WAITOPERATOR = 2,
	WAITFIRSTOPERAND = 3,
} ts_parserstate;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • tsquery の解析ロジックにおいて、トークンが期待される順序（オペランドまたは演算子）を管理するための状態遷移に使用されます。

アルゴリズムまたは振る舞い

  • WAITOPERAND: 演算子の後など、次にオペランドが来ることを期待している状態。
  • WAITOPERATOR: オペランドの後など、次に演算子が来ることを期待している状態。
  • WAITFIRSTOPERAND: 解析の開始直後など、最初のオペランドを待機している状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/tsquery.c 内の解析ロジック（TSQueryParserStateData 構造体のメンバとして使用）。
  • 呼び出し先: 情報なし。

関連データ構造

  • TSQueryParserStateData: この列挙型を state メンバとして保持し、解析状態を管理します。

失敗モードと境界条件

  • 状態遷移の不整合（例：WAITOPERAND の状態で演算子が入力される等）が発生した場合、パーサーは構文エラーを報告します。

保守担当者向けメモ

  • 本列挙型は tsquery の構文解析のステートマシンを定義しています。解析ロジックを変更する際は、この状態遷移が正しく維持されているか確認してください。
