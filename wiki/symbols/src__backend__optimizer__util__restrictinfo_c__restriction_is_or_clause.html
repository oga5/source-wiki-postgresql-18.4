restriction_is_or_clause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:406

目的

与えられた RestrictInfo ノードが OR 句として構成されているかどうかを判定します。

シグネチャまたは形状

bool restriction_is_or_clause(RestrictInfo *restrictinfo)

入力と出力

  • **入力**: RestrictInfo *restrictinfo - 検査対象の制約情報ノード。
  • **出力**: bool - restrictinfo->orclause が NULL でない場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • restrictinfo 引数は有効な RestrictInfo 構造体へのポインタである必要があります。

アルゴリズムまたは振る舞い

1. restrictinfo 構造体の orclause フィールドを確認します。 2. orclause が NULL でなければ true を返し、NULL であれば false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/path/tidpath.c
  • src/backend/optimizer/plan/analyzejoins.c
  • src/backend/optimizer/plan/initsplan.c
  • src/backend/optimizer/util/orclauses.c
  • **呼び出し先**:
  • 本関数は他の関数を呼び出しません。

関連データ構造

  • RestrictInfo: クエリプランナが使用する制約条件のラッパー構造体。orclause フィールドを保持します。

失敗モードと境界条件

  • restrictinfo が NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は RestrictInfo 構造体の orclause フィールドの存在チェックを行う単純なアクセサです。OR 句の解析ロジックに変更があった場合でも、この判定基準は維持される想定です。
