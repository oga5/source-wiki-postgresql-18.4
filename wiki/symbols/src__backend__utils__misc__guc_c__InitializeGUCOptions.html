InitializeGUCOptions

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/misc/guc.c:1529

目的

PostgreSQLのGUC（Grand Unified Configuration）システムを初期化し、全設定パラメータをデフォルト値でセットアップする。

シグネチャまたは形状

void InitializeGUCOptions(void)

入力と出力

  • **入力**: なし
  • **出力**: なし（副作用としてグローバルな設定状態を初期化）

前提条件と不変条件

  • pg_timezone_initialize() が呼び出され、タイムゾーン処理が有効化されていること。
  • build_guc_variables() により guc_hashtab が構築済みであること。
  • 各GUC変数の初期値とデフォルト値の整合性が check_GUC_init() によって保証されていること。

アルゴリズムまたは振る舞い

1. pg_timezone_initialize() を実行し、タイムゾーン処理を準備する。 2. build_guc_variables() を呼び出し、GUCメモリコンテキストの作成とハッシュテーブルの構築を行う。 3. guc_hashtab を走査し、各パラメータに対して InitializeOneGUCOption() を実行してデフォルト値を適用する。 4. reporting_enabled を false に設定する。 5. トランザクション関連のパラメータ（transaction_isolation, transaction_read_only, transaction_deferrable）を PGC_POSTMASTER レベルで強制的に初期設定する。 6. InitializeGUCOptionsFromEnvironment() を呼び出し、環境変数由来のデフォルト値を適用する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/bootstrap/bootstrap.c
  • src/backend/postmaster/launch_backend.c
  • src/backend/postmaster/postmaster.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • pg_timezone_initialize()
  • build_guc_variables()
  • hash_seq_init() / hash_seq_search()
  • check_GUC_init()
  • InitializeOneGUCOption()
  • SetConfigOption()
  • InitializeGUCOptionsFromEnvironment()

関連データ構造

  • GUCHashEntry: ハッシュテーブル内の各パラメータエントリ。
  • guc_hashtab: 全パラメータを保持するハッシュテーブル。
  • GUCMemoryContext: GUC関連データが格納されるメモリコンテキスト。

失敗モードと境界条件

  • check_GUC_init() が失敗した場合、Assert により異常終了する。
  • 環境変数からの設定読み込み時に不正な値が指定された場合、そのパラメータの初期化に影響を与える可能性がある。

保守担当者向けメモ

  • 本関数はバックエンド起動時の極めて早い段階で実行される。
  • transaction_* パラメータの強制設定は、非対話的なソースからの上書きを防ぐための歴史的な制約に基づく。
  • reporting_enabled の制御は、初期化中のログ出力等を抑制するために重要である。
