ParseCommitRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xactdesc.c:34

目的

トランザクションコミットのWALレコード（xl_xact_commit）を解析し、可変長データを含む詳細情報を構造体（xl_xact_parsed_commit）へ展開する。

シグネチャまたは形状

void ParseCommitRecord(uint8 info, xl_xact_commit *xlrec, xl_xact_parsed_commit *parsed)

入力と出力

  • **入力**:
  • info: WALレコードのフラグ（XLOG_XACT_HAS_INFO 等）。
  • xlrec: 解析対象のコミットレコードへのポインタ。
  • **出力**:
  • parsed: 解析結果が格納される構造体。

前提条件と不変条件

  • parsed は呼び出し元で確保されたメモリ領域を指している必要がある。
  • xlrec は有効なコミットレコードの先頭を指していること。
  • 関数内で memset により parsed は初期化される。

アルゴリズムまたは振る舞い

1. xlrec の直後からデータポインタ data を開始する。 2. info フラグに基づき、以下のオプション情報を順次パースする:

  • XLOG_XACT_HAS_INFO: 拡張情報（xinfo）の抽出。
  • XACT_XINFO_HAS_DBINFO: データベースおよびテーブルスペースIDの抽出。
  • XACT_XINFO_HAS_SUBXACTS: サブトランザクションID配列の抽出。
  • XACT_XINFO_HAS_RELFILELOCATORS: リレーションファイルロケータの抽出。
  • XACT_XINFO_HAS_DROPPED_STATS: 削除された統計情報の抽出。
  • XACT_XINFO_HAS_INVALS: 無効化メッセージの抽出。
  • XACT_XINFO_HAS_TWOPHASE: 2フェーズコミット関連情報（GID含む）の抽出。
  • XACT_XINFO_HAS_ORIGIN: レプリケーションオリジン情報の抽出。

    • 3. 可変長データは data ポインタをオフセット分進めることで順次読み取る。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/rmgrdesc/xactdesc.c
  • src/backend/access/transam/xact.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/replication/logical/decode.c
  • **呼び出し先**:
  • memcpy（アライメント保証がないデータ用）

関連データ構造

  • xl_xact_commit: 入力となるWALレコード構造体。
  • xl_xact_parsed_commit: 解析結果を保持する構造体。
  • xl_xact_xinfo, xl_xact_dbinfo, xl_xact_subxacts 等: WAL内の各セクションに対応する構造体。

失敗モードと境界条件

  • **アライメント**: XACT_XINFO_HAS_ORIGIN 以降、データのアライメントは保証されないため、memcpy を使用してスタック上の変数へコピーする。
  • **可変長**: nsubxacts や nrels などの個数に基づきポインタを計算するため、WALレコードが破損している場合は誤ったメモリ位置を参照する可能性がある。

保守担当者向けメモ

  • 本関数はバックエンドとフロントエンド（pg_waldump）で共有されるため、依存関係の追加には注意が必要。
  • data ポインタのオフセット計算は、xl_xact_... 構造体の定義と厳密に一致させる必要がある。
