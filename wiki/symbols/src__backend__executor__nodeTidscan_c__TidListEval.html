TidListEval

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidscan.c:133

目的

クエリ内の ctid 関連式（OpExpr, ScalarArrayOpExpr, CurrentOfExpr）を評価し、スキャン対象となる物理的な ItemPointer（TID）のリストを構築する。

シグネチャまたは形状

static void TidListEval(TidScanState *tidstate)

入力と出力

  • **入力**: TidScanState *tidstate（スキャン状態および式リストを保持する構造体）
  • **出力**: なし（副作用として tidstate 内の tss_TidList, tss_NumTids, tss_TidPtr を更新する）

前提条件と不変条件

  • tidstate->tss_tidexprs に有効な式リストが格納されていること。
  • table_beginscan_tid が未実行の場合、本関数内で実行され ss_currentScanDesc が初期化される。
  • CurrentOfExpr が含まれる場合、他の式との OR 条件は存在しないこと（アサーションにより保証）。

アルゴリズムまたは振る舞い

1. **スキャン開始**: ss_currentScanDesc が未初期化であれば、table_beginscan_tid を呼び出し遅延初期化を行う。 2. **TID 抽出**: tss_tidexprs を走査し、各式の型に応じて以下を実行する。

  • **単一値**: ExecEvalExprSwitchContext で評価し、table_tuple_tid_valid で有効性を確認してリストに追加。
  • **配列**: deconstruct_array_builtin で展開し、各要素の有効性を確認してリストに追加。
  • **CurrentOf**: execCurrentOf を呼び出し、カーソルに関連付けられた TID を取得。

    • 3. **動的拡張**: リストの容量が不足した場合は repalloc で領域を拡張する。 4. **後処理**:

  • numTids > 1 の場合、qsort と qunique を用いて TID をソートし、重複を除去する（OR 演算の正当性と効率的な I/O を保証）。
  • 結果を tidstate に格納し、スキャンポインタ tss_TidPtr を -1 に初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeTidscan.c 内の TidNext（初回実行時）。
  • **呼び出し先**: table_beginscan_tid, ExecEvalExprSwitchContext, table_tuple_tid_valid, deconstruct_array_builtin, execCurrentOf, qsort, qunique。

関連データ構造

  • TidScanState: スキャン状態を保持するメイン構造体。
  • TidExpr: 評価対象の式を保持する構造体。
  • ItemPointerData: 物理的なタプル位置を示すデータ構造。

失敗モードと境界条件

  • **無効な TID**: table_tuple_tid_valid が偽を返す TID は無視される。
  • **NULL 値**: 式の評価結果が NULL の場合はスキップされる。
  • **空の結果**: 抽出された TID が存在しない場合、tss_NumTids は 0 となる。

保守担当者向けメモ

  • 本関数は初回実行時にのみ呼び出される遅延評価の仕組みをとっている。
  • TID リストのソートは、物理的なブロック順序でのアクセスを促し、I/O 効率を向上させるために重要である。
  • CurrentOfExpr は他の式と混在できない仕様であるため、変更を加える際は注意が必要。
