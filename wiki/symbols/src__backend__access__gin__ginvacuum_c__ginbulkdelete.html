ginbulkdelete

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginvacuum.c:563

目的

GINインデックスのバキューム処理におけるメインエントリポイントとして、インデックス内の不要なタプル（TID）の削除およびポストティングツリーの整理を行う。

シグネチャまたは形状

IndexBulkDeleteResult *ginbulkdelete(
    IndexVacuumInfo *info, 
    IndexBulkDeleteResult *stats,
    IndexBulkDeleteCallback callback, 
    void *callback_state
)

入力と出力

  • **入力**:
  • info: バキューム対象のインデックス情報。
  • stats: 統計情報（初回呼び出し時は NULL）。
  • callback: 不要なTIDを判定するコールバック関数。
  • callback_state: コールバック関数に渡す状態データ。
  • **出力**: 更新された IndexBulkDeleteResult 構造体へのポインタ。

前提条件と不変条件

  • 初回呼び出し時（stats == NULL）には、保留中の挿入を処理するために ginInsertCleanup が実行される。
  • 処理中、一時的なメモリコンテキスト（gvs.tmpCxt）が作成され、終了時に解放される。
  • エントリツリーの走査において、ルートからリーフまで辿るロック順序が維持される。

アルゴリズムまたは振る舞い

1. **初期化**: GinVacuumState を構築し、必要に応じて保留中の挿入をクリーンアップする。 2. **リーフ探索**: GINインデックスのルートから開始し、エントリツリーの左端リーフページまで降下する。 3. **エントリツリー走査**:

  • 各リーフページに対して ginVacuumEntryPage を呼び出し、不要なTIDを削除する。
  • 変更があった場合は xlogVacuumPage でWALログを記録し、ページを更新する。
  • ページ内のポストティングツリーへの参照を収集する。

    • 4. **ポストティングツリーのバキューム**: 収集した各ポストティングツリーに対して ginVacuumPostingTree を実行し、リーフのバキュームと空ページの削除を行う。 5. **反復**: 右リンク（rightlink）を辿り、インデックスの全リーフページに対して上記処理を繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginutil.c: ambulkdelete インターフェース経由で呼び出される。
  • **呼び出し先**:
  • ginInsertCleanup: 保留中の挿入処理。
  • ginVacuumEntryPage: エントリページのバキューム。
  • ginVacuumPostingTree: ポストティングツリーのバキューム。
  • xlogVacuumPage: WALログの記録。
  • vacuum_delay_point: バキュームの遅延制御。

関連データ構造

  • GinVacuumState: バキューム処理中の状態を保持する構造体。
  • IndexBulkDeleteResult: バキューム結果の統計情報。
  • IndexVacuumInfo: バキューム対象のインデックス情報。

失敗モードと境界条件

  • stats が NULL の場合、新規に統計構造体を割り当てる。
  • ポストティングツリーの削除時、親から子へのロック順序が厳密に管理されており、デッドロック回避のための実装が求められる。
  • 右端ページに到達した時点で走査を終了する。

保守担当者向けメモ

  • ginVacuumEntryPage および ginVacuumPostingTree の呼び出し前後で vacuum_delay_point が呼ばれており、長時間実行時の負荷を調整している。
  • エントリツリーのリーフページ更新には PageRestoreTempPage を使用しており、変更が細かいためフルページイメージのWAL記録が必須となっている。
