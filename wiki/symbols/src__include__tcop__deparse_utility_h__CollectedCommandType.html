CollectedCommandType

定義

  • 種別: enum
  • 定義箇所: src/include/tcop/deparse_utility.h:24

目的

PostgreSQLのユーティリティコマンド（DDL等）の実行過程を追跡・記録する際、コマンドの種類を識別するために使用されます。イベントトリガーや監査ログ、レプリケーション等の目的で、コマンドの構造を保持する際の型判別子として機能します。

シグネチャまたは形状

typedef enum CollectedCommandType
{
	SCT_Simple,
	SCT_AlterTable,
	SCT_Grant,
	SCT_AlterOpFamily,
	SCT_AlterDefaultPrivileges,
	SCT_CreateOpClass,
	SCT_AlterTSConfig,
} CollectedCommandType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: CollectedCommand 構造体内の type フィールドとして保持されます。

前提条件と不変条件

  • CollectedCommand 構造体は共用体（union）を使用しているため、本列挙型の値を確認してから適切な共用体メンバにアクセスする必要があります。

アルゴリズムまたは振る舞い

ユーティリティコマンドの実行時に生成される CollectedCommand インスタンスにおいて、実行されたコマンドの種類を特定するために設定されます。これにより、後続の処理でコマンドの内容を再利用可能な形式（Deparse可能な形式）で管理することが可能になります。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/tcop/deparse_utility.h 内の CollectedCommand 構造体定義において、メンバ変数 type の型として参照されています。
  • 呼び出し先: 本シンボル自体は列挙型であり、直接的な呼び出し先はありません。

関連データ構造

  • CollectedCommand: 本列挙型を保持するメインの構造体。
  • CollectedATSubcmd: ALTER TABLE 等の階層的なコマンド管理に使用される構造体。

失敗モードと境界条件

  • 列挙型に含まれないコマンド種別が指定された場合、CollectedCommand の処理ロジックにおいて未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しい種類のユーティリティコマンドを追跡対象に追加する場合は、本列挙型に新しいメンバを追加し、対応する CollectedCommand の共用体定義を更新してください。
