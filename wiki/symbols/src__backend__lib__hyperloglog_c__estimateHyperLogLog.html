estimateHyperLogLog

定義

  • 種別: function
  • 定義箇所: src/backend/lib/hyperloglog.c:185

目的

HyperLogLogアルゴリズムを用いて、蓄積されたハッシュ値のレジスタ状態から集合の濃度（ユニークな要素数）を近似推定する。

シグネチャまたは形状

double estimateHyperLogLog(hyperLogLogState *cState)

入力と出力

  • **入力**: hyperLogLogState *cState（推定に必要なレジスタ配列と補正係数 alphaMM を保持する構造体へのポインタ）。
  • **出力**: double 型の推定されたカーディナリティ。

前提条件と不変条件

  • cState->hashesArr には、addHyperLogLog によって適切に更新されたレジスタ値が格納されていること。
  • cState->alphaMM は、初期化時に精度に基づいて計算された補正係数であること。

アルゴリズムまたは振る舞い

1. **生推定値の算出**: 全レジスタ値の調和平均を求め、alphaMM を用いて基本となるカーディナリティ（result）を算出する。 2. **小規模範囲の補正**: result がレジスタ数の 2.5 倍以下の場合、ゼロレジスタの数を用いて線形カウント法による補正を行う。 3. **大規模範囲の補正**: result が $2^{32}/30$ を超える場合、ハッシュ衝突の影響を考慮した対数補正を行う。 4. **戻り値**: 補正後の推定値を返す。

関連データ構造

  • hyperLogLogState: 推定状態を保持する構造体。

失敗モードと境界条件

  • zero_count が 0 の場合、小規模範囲の補正はスキップされる。
  • bwidth（ビット幅）が 4 未満または 16 を超える設定での動作は保証されない（モジュール概要による）。
