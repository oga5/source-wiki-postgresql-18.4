WalRcvWaitForStartPosition

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiver.c:644

目的

ストリーミングレプリケーションの待機状態へ移行し、スタートアッププロセスからの再開指示（WALRCV_RESTARTING）または終了指示（WALRCV_STOPPING）を待機する。

シグネチャまたは形状

static void WalRcvWaitForStartPosition(XLogRecPtr *startpoint, TimeLineID *startpointTLI)

入力と出力

  • **入力**:
  • startpoint: 再開位置（LSN）を格納するポインタ。
  • startpointTLI: 再開するタイムラインIDを格納するポインタ。
  • **出力**: なし（引数ポインタ経由で再開情報を書き戻す）。

前提条件と不変条件

  • 呼び出し時点で walrcv->walRcvState が WALRCV_STREAMING であること。
  • 待機ループ中、walrcv->walRcvState は WALRCV_RESTARTING、WALRCV_WAITING、または WALRCV_STOPPING のいずれかであること。

アルゴリズムまたは振る舞い

1. 共有メモリの walrcv->mutex を取得し、状態を WALRCV_WAITING に変更して receiveStart 情報をリセットする。 2. プロセス表示を "idle" に更新し、WakeupRecovery() を呼び出してスタートアッププロセスに待機状態への移行を通知する。 3. ループ内で WaitLatch を使用して待機する。 4. walrcv->walRcvState が WALRCV_RESTARTING に変化した場合、共有メモリから startpoint と startpointTLI を取得し、状態を WALRCV_STREAMING に戻してループを抜ける。 5. WALRCV_STOPPING の場合はプロセスを終了する。 6. 終了直前にプロセス表示を再開位置情報を含む文字列に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: WalReceiverMain（src/backend/replication/walreceiver.c:636 付近）。
  • **呼び出し先**: WakeupRecovery, ResetLatch, CHECK_FOR_INTERRUPTS, SpinLockAcquire/Release, WaitLatch, set_ps_display, proc_exit/exit。

関連データ構造

  • WalRcvData（共有メモリ上の構造体）
  • XLogRecPtr
  • TimeLineID
  • MyLatch

失敗モードと境界条件

  • 呼び出し時に状態が WALRCV_STREAMING 以外の場合、WALRCV_STOPPING であれば正常終了し、それ以外は FATAL エラーとなる。
  • 待機中にポストマスターが死亡した場合、WL_EXIT_ON_PM_DEATH により終了する。

保守担当者向けメモ

  • 本関数はスタートアッププロセスとの同期点である。共有メモリの walRcvState を介した状態遷移が厳密に管理されているため、変更時は walreceiverfuncs.c 側の状態遷移ロジックとの整合性に注意すること。
