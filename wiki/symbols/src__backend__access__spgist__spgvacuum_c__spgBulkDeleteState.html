spgBulkDeleteState

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/spgist/spgvacuum.c:41

目的

SP-GiST インデックスの VACUUM 処理（バルク削除）実行中に必要な状態、統計情報、および保留中のタスクを保持するための構造体です。

シグネチャまたは形状

typedef struct spgBulkDeleteState
{
    IndexVacuumInfo *info;
    IndexBulkDeleteResult *stats;
    IndexBulkDeleteCallback callback;
    void       *callback_state;
    SpGistState spgstate;
    spgVacPendingItem *pendingList;
    TransactionId myXmin;
    BlockNumber lastFilledBlock;
} spgBulkDeleteState;

入力と出力

本構造体は関数間で受け渡される状態保持オブジェクトとして機能します。

  • **入力**: spgvacuumscan 等の関数において、VACUUM 実行に必要なパラメータ（info, stats, callback 等）が初期化時に設定されます。
  • **出力**: 処理の過程で stats（統計情報）が更新され、pendingList（再訪が必要な TID リスト）が操作されます。

前提条件と不変条件

  • spgstate は SP-GiST 操作を実行するために初期化されている必要があります。
  • pendingList は、同時実行中の挿入によって発生した REDIRECT 先を追跡するために使用されます。

アルゴリズムまたは振る舞い

  • VACUUM 実行中、インデックスの走査（spgvacuumscan）およびページ処理（spgvacuumpage）を通じて、この構造体を介して状態が共有されます。
  • vacuumLeafPage や vacuumRedirectAndPlaceholder などの関数が、この構造体を参照してリーフページの整理やリダイレクトタプルのクリーンアップを行います。
  • spgprocesspending により、保留リストに登録された TID に対して後続のクリーンアップ処理が実行されます。

呼び出し元と呼び出し先

  • **主な呼び出し元**: spgbulkdelete（メインエントリポイント）、spgvacuumscan。
  • **主な呼び出し先**: spgAddPendingTID、spgClearPendingList、vacuumLeafPage、vacuumLeafRoot、spgvacuumpage、spgprocesspending。

関連データ構造

  • SpGistState: SP-GiST 操作用の状態構造体。
  • spgVacPendingItem: pendingList を構成する保留中の TID アイテム。
  • IndexVacuumInfo, IndexBulkDeleteResult: VACUUM 処理の標準的なパラメータおよび結果格納用構造体。

失敗モードと境界条件

  • 同時実行中の挿入操作との競合が発生した場合、myXmin を使用して新しく追加されたリダイレクトを検出します。
  • インデックスの切り詰め（Truncation）は現在無効化されているため、lastFilledBlock の扱いに注意が必要です。

保守担当者向けメモ

  • 本構造体は spgvacuum.c 内の各処理フェーズを繋ぐ重要なコンテキストです。
  • pendingList の管理は、インデックススキャン中の整合性を保つために不可欠です。変更を加える際は、同時実行中の挿入操作への影響を考慮してください。
