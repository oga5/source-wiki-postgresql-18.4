equal

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/equalfuncs.c:222

目的

PostgreSQLのノードツリー構造が論理的に等価であるかを判定します。クエリ最適化やプランナにおいて、式やプランノードの同一性を検証するために使用されます。

シグネチャまたは形状

bool equal(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つのノードポインタ a および b。
  • **出力**: 両者が等価であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a と b は Node 構造体（またはその派生型）を指している必要があります。
  • 再帰的な比較を行うため、スタックオーバーフローを防ぐための check_stack_depth() が呼び出されます。
  • ノード内の location フィールドは比較対象から除外されます。

アルゴリズムまたは振る舞い

1. **ポインタ比較**: a == b の場合は即座に true を返します。 2. **NULLチェック**: 片方のみが NULL の場合は false を返します。 3. **型チェック**: nodeTag(a) と nodeTag(b) を比較し、型が異なれば false を返します。 4. **スタック保護**: check_stack_depth() を実行し、再帰の深さを確認します。 5. **ディスパッチ**: nodeTag に基づき、以下のいずれかの処理を行います。

  • equalfuncs.switch.c に含まれる自動生成コードによる比較。
  • T_List, T_IntList, T_OidList, T_XidList の場合は _equalList を呼び出し。

    • 6. **エラーハンドリング**: 未知のノード型が渡された場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し先**: _equalList、および equalfuncs.switch.c 内の各ノード型専用比較関数。
  • **呼び出し元**: 提供されたソース情報には、本関数を直接呼び出している特定の関数名は含まれていません。なお、contrib/amcheck/verify_nbtree.c 等で「equal」という単語を含むログ出力やコメントが見られますが、これらが本関数を直接呼び出しているという証拠はありません。

関連データ構造

  • NodeTag: ノードの型を識別するための列挙型。
  • List, IntList, OidList, XidList: _equalList で処理されるリスト構造。

失敗モードと境界条件

  • **型不一致**: 異なるノード型が渡された場合は false を返します。
  • **未知の型**: 定義されていないノード型が渡された場合、elog(ERROR) により処理が中断されます。
  • **NULLポインタ**: 片方のみが NULL の場合は false を返します。

保守担当者向けメモ

  • 本関数はノードツリーの再帰的な比較の起点となります。新しいノード型を追加した際は、equalfuncs.switch.c への反映が必要です。
  • location フィールドの比較除外は、クエリ内の位置情報に依存せず論理的な同一性を判定するための仕様です。
