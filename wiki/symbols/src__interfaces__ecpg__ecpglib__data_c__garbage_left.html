garbage_left

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/data.c:45

目的

パース後の文字列（ホスト変数への変換対象データ）に、予期しない残余データが含まれていないかを検証する。

シグネチャまたは形状

static bool garbage_left(enum ARRAY_TYPE isarray, char **scan_length, enum COMPAT_MODE compat);

入力と出力

  • **入力**:
  • isarray: 配列型か否かを示すフラグ。
  • scan_length: パース後の現在位置を指すポインタのポインタ。
  • compat: 互換性モード（Informix 等の挙動を制御）。
  • **出力**:
  • bool: 不正な残余データが存在する場合は true、それ以外は false を返す。

前提条件と不変条件

  • scan_length は有効な文字列ポインタを指している必要がある。
  • INFORMIX_MODE が有効な場合、数値の小数点以下の切り捨てを許容する特殊な処理が行われる。

アルゴリズムまたは振る舞い

1. **非配列型の場合**:

  • INFORMIX_MODE が有効かつ現在位置が . で始まる場合、後続の数字をスキップする（切り捨ての許容）。
  • その後、現在位置がスペースまたは終端文字 (\0) でなければ、残余データありと判断し true を返す。

    • 2. **配列型の場合**:

  • array_delimiter および array_boundary を使用して、現在位置が配列の区切り文字や境界文字ではないことを確認する。
  • いずれにも該当しない文字が残っている場合は true を返す。

    • 3. **正常終了**:

  • 上記の条件に該当しない場合、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/data.c 内の各データ変換処理（ecpg_get_data 等）から、変換後の文字列検証のために呼び出される。
  • **呼び出し先**: array_delimiter, array_boundary（いずれも data.c 内）。

関連データ構造

  • enum ARRAY_TYPE: 配列の形式を定義。
  • enum COMPAT_MODE: 互換性モードを定義。

失敗モードと境界条件

  • scan_length が指す文字列が空、または予期せぬ文字で終わっている場合に true を返す。
  • INFORMIX_MODE において、小数点以下の数値が続く場合はスキップされるが、それ以外の不正な文字が混入している場合は正しく検出される。

保守担当者向けメモ

  • 本関数は ecpg_get_data におけるデータ変換の最終的な整合性チェックを担う。
  • 配列処理のロジックは array_delimiter および array_boundary の実装に依存しているため、配列のパース仕様を変更する際はこれらとの整合性に注意すること。
