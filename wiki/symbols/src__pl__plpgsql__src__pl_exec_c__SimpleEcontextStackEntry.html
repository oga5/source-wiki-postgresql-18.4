SimpleEcontextStackEntry

定義

  • **種別**: struct
  • **定義箇所**: src/pl/plpgsql/src/pl_exec.c:84

目的

PL/pgSQL の実行エンジンにおいて、式評価コンテキスト（ExprContext）をサブトランザクション単位でスタック管理するためのエントリ構造体です。

シグネチャまたは形状

typedef struct SimpleEcontextStackEntry
{
    ExprContext *stack_econtext;    /* スタックされた econtext */
    SubTransactionId xact_subxid;   /* 現在のサブトランザクション ID */
    struct SimpleEcontextStackEntry *next; /* スタックの次エントリ */
} SimpleEcontextStackEntry;

入力と出力

  • **入力**: 構造体生成時に ExprContext および SubTransactionId が割り当てられます。
  • **出力**: なし（データ構造として状態を保持）。

前提条件と不変条件

  • next ポインタは、スタックのより上位（以前）のエントリを指します。
  • xact_subxid は、対応する stack_econtext が有効であるべきサブトランザクションを識別します。

アルゴリズムまたは振る舞い

  • 本構造体は、simple_econtext_stack という静的変数によって管理される連結リストの一部として機能します。
  • サブトランザクションの開始や終了に伴い、このスタックを操作することで、実行コンテキストのスコープを制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_exec.c 内の関数（simple_econtext_stack を操作する箇所）。
  • **呼び出し先**: 情報なし。

関連データ構造

  • ExprContext: 式評価に使用されるメモリコンテキストおよびリソース管理構造体。
  • SubTransactionId: PostgreSQL のサブトランザクション識別子。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc 等の失敗）に構造体の生成が失敗する可能性があります。
  • スタックが空の状態で next を参照しようとすると NULL ポインタ参照が発生するため、操作前に必ず NULL チェックが必要です。

保守担当者向けメモ

  • simple_econtext_stack はグローバルな静的変数として定義されています。マルチスレッド環境や再帰的な実行における整合性に注意してください。
  • この構造体は pl_exec.c 内の特定の実行ロジックに密結合しているため、変更時は関連するスタック操作関数（simple_econtext_stack を操作する箇所）の整合性を確認してください。
