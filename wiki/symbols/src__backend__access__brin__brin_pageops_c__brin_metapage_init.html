brin_metapage_init

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_pageops.c:485

目的

BRINインデックスのメタページを初期化し、インデックスの基本設定（バージョン、範囲あたりのページ数）を書き込む。

シグネチャまたは形状

void brin_metapage_init(Page page, BlockNumber pagesPerRange, uint16 version)

入力と出力

  • page: 初期化対象となるページバッファ。
  • pagesPerRange: インデックスが管理する範囲あたりのブロック数。
  • version: BRINインデックスのバージョン番号。
  • 出力: なし（pageの内容が直接更新される）。

前提条件と不変条件

  • pageは有効なバッファページであること。
  • brin_page_initを介してBRIN_PAGETYPE_METAとして初期化される。
  • pd_lowerは、メタデータ構造体の直後に設定されなければならない（WAL圧縮時のデータ消失を防ぐため）。

アルゴリズムまたは振る舞い

1. brin_page_initを呼び出し、ページをメタページタイプとして初期化する。 2. PageGetContentsを使用してページ内のメタデータ領域を取得する。 3. BrinMetaPageData構造体に以下の値を設定する:

  • brinMagic: BRIN_META_MAGIC
  • brinVersion: 指定されたversion
  • pagesPerRange: 指定されたpagesPerRange
  • lastRevmapPage: 0（初期状態として設定）

    • 4. pd_lowerをBrinMetaPageDataのサイズ分だけ進め、メタデータ領域を保護する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/brin/brin.c: インデックス構築時および再構築時に呼び出される。
  • src/backend/access/brin/brin_xlog.c: WALリプレイ時に呼び出される。
  • 呼び出し先:
  • brin_page_init（同ファイル内）

関連データ構造

  • BrinMetaPageData: メタページに格納される主要な構造体。
  • PageHeader: ページレイアウト管理に使用。

失敗モードと境界条件

  • lastRevmapPageを0に初期化する点は、メタページ自体がRevmapの0番目ブロックとして扱われることによる特殊な設計である。

保守担当者向けメモ

  • pd_lowerの設定は、WALログによるページ圧縮時にメタデータが切り詰められるのを防ぐための必須処理である。変更時は注意すること。
