ComboCidEntryData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/time/combocid.c:64

目的

Combo CID（コマンドID）のハッシュテーブル管理において、キーとなる cmin/cmax のペアと、それに対応する単一の combocid を保持するためのエントリ構造体です。

シグネチャまたは形状

typedef struct
{
	ComboCidKeyData key;
	CommandId	combocid;
} ComboCidEntryData;

入力と出力

  • 入力: ComboCidKeyData 型の key（検索キー）および CommandId 型の combocid。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • ComboCidKeyData に構造体のパディングが存在しないことを前提としてハッシュ操作が行われます。
  • 本構造体は TopTransactionContext 上に確保されるハッシュテーブルのエントリとして使用されます。

アルゴリズムまたは振る舞い

  • comboHash ハッシュテーブルにおいて、cmin/cmax のペアから combocid を高速に検索・格納するために使用されます。
  • src/backend/utils/time/combocid.c:227 にて、ハッシュテーブルの entrysize として本構造体のサイズが指定されています。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/time/combocid.c 内のハッシュテーブル初期化処理。
  • 呼び出し先: 直接的な関数呼び出しはありません（データ構造定義のため）。

関連データ構造

  • ComboCidKeyData: 本構造体の key フィールドの型。
  • ComboCidEntry: 本構造体へのポインタ型定義 (typedef ComboCidEntryData *ComboCidEntry;)。

失敗モードと境界条件

  • メモリ不足: TopTransactionContext への確保に失敗した場合、PostgreSQL の標準的なメモリ確保エラーが発生します。
  • 構造体パディング: ComboCidKeyData のレイアウト変更はハッシュ計算の整合性に影響を与えるため注意が必要です。

保守担当者向けメモ

  • 本構造体はトランザクション終了時に AtEOXact_ComboCid を通じてメモリコンテキストごと破棄されるため、個別の解放処理は不要です。
  • 構造体定義を変更する場合は、ハッシュテーブルのキーとしての整合性が保たれているか確認してください。
