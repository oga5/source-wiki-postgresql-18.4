sepgsql_database_setattr

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/database.c:159

目的

データベースオブジェクトの属性変更操作に対し、SELinuxの db_database:{setattr} 権限チェックを適用する。

シグネチャまたは形状

void sepgsql_database_setattr(Oid databaseId)

入力と出力

  • **入力**: databaseId (Oid) - 属性変更対象となるデータベースのOID。
  • **出力**: なし。権限チェックに失敗した場合は例外が発生する。

前提条件と不変条件

  • databaseId は有効なデータベースを指している必要がある。
  • 本関数は、PostgreSQLのデータベース属性変更フックから呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. ObjectAddress 構造体を初期化し、DatabaseRelationId と対象の databaseId を設定する。 2. getObjectIdentity を呼び出し、監査ログ用のオブジェクト識別名を取得する。 3. sepgsql_avc_check_perms を呼び出し、現在のセキュリティコンテキストが対象データベースに対して SEPG_DB_DATABASE__SETATTR 権限を持っているか検証する。 4. 権限チェックの失敗時は例外が発生し、成功時は pfree で監査用文字列のメモリを解放して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:216
  • **呼び出し先**:
  • getObjectIdentity (PostgreSQLコア関数)
  • sepgsql_avc_check_perms (contrib/sepgsql/uavc.c)
  • pfree (PostgreSQLメモリ管理関数)

関連データ構造

  • ObjectAddress: データベースオブジェクトを特定するための構造体。
  • DatabaseRelationId: pg_database カタログを参照するための定数。

失敗モードと境界条件

  • getObjectIdentity が失敗した場合、または sepgsql_avc_check_perms が権限拒否を判定した場合、PostgreSQLの標準的なエラー処理メカニズムにより処理が中断される。

保守担当者向けメモ

  • 本関数は sepgsql_avc_check_perms のラッパーとして機能する。
  • 権限チェックの監査ログには audit_name が使用されるため、オブジェクトの識別が困難な状況では getObjectIdentity の挙動に依存する。
