plperl_query_entry

定義

  • 種別: struct
  • 定義箇所: src/pl/plperl/plperl.c:201

目的

PL/Perl において、プリペアドステートメント（SPI クエリ）をハッシュテーブル内で管理するためのエントリ構造体です。

シグネチャまたは形状

typedef struct plperl_query_entry
{
	char		query_name[NAMEDATALEN];
	plperl_query_desc *query_data;
} plperl_query_entry;

入力と出力

  • **入力**: query_name（クエリ識別子）および query_data（クエリの詳細情報へのポインタ）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • query_name は NAMEDATALEN バイト以内の文字列である必要があります。
  • 本構造体は、query_hash（HTAB）の要素として使用されることを前提としています。

アルゴリズムまたは振る舞い

  • plperl.c 内の hash_ctl.entrysize 設定において、ハッシュテーブルの各エントリサイズとして本構造体が指定されています。
  • 実行時に query_hash から特定のクエリを検索・取得する際のキーとなるデータ構造として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plperl/plperl.c 内の複数の箇所（行 3573, 3726, 3849, 3965）で、ハッシュテーブルからエントリを取得する際の型として参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • plperl_query_desc: query_data フィールドが指し示す、クエリの実行状態やメタデータを保持する構造体。
  • query_hash: 本構造体を格納するハッシュテーブル（HTAB）。

失敗モードと境界条件

  • ハッシュテーブルの初期化やエントリの挿入失敗時には、PostgreSQL のメモリ管理機構によるエラーが発生する可能性があります。
  • query_name が NAMEDATALEN を超える場合、切り詰めやバッファオーバーフローのリスクがあるため、適切な文字列操作が必要です。

保守担当者向けメモ

  • 本構造体は plperl_query_desc と密接に関連しています。query_data のライフサイクル管理（メモリ解放や参照カウント）については、plperl_query_desc 側の実装を確認してください。
  • ハッシュテーブルのサイズやエントリの追加・削除ロジックは plperl.c 内の query_hash 操作関数群に依存しています。
