ExecInitFunctionResultSet

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execSRF.c:443

目的

集合を返す関数（SRF）の実行状態を管理する SetExprState 構造体を初期化する。

シグネチャまたは形状

SetExprState *ExecInitFunctionResultSet(Expr *expr, ExprContext *econtext, PlanState *parent)

入力と出力

  • **入力**:
  • expr: 評価対象の式（FuncExpr または OpExpr）。
  • econtext: 式評価用のコンテキスト。
  • parent: 親プランノードの状態。
  • **出力**:
  • 初期化された SetExprState 構造体へのポインタ。

前提条件と不変条件

  • 入力 expr は FuncExpr または OpExpr である必要がある。
  • 選択された関数は集合を返す（fn_retset が真である）必要がある。

アルゴリズムまたは振る舞い

1. SetExprState ノードを割り当て、funcReturnsSet を true に設定する。 2. expr の型に応じて引数リストを初期化する。

  • FuncExpr の場合: func->args を評価用に初期化し、init_sexpr を呼び出す。
  • OpExpr の場合: op->opfuncid を使用して init_sexpr を呼び出す。

    • 3. init_sexpr を通じて関数メタデータをキャッシュし、実行準備を行う。 4. 最後に state->func.fn_retset が真であることをアサートし、状態を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeProjectSet.c（直接の呼び出しを確認）
  • **呼び出し先**:
  • ExecInitExprList
  • init_sexpr
  • IsA (マクロ)
  • nodeTag (マクロ)
  • elog

関連データ構造

  • SetExprState: SRF の実行状態を保持する主要構造体。
  • ExprContext: 式評価の実行環境。
  • FuncExpr / OpExpr: 評価対象の式ノード。

失敗モードと境界条件

  • expr が FuncExpr でも OpExpr でもない場合、elog(ERROR) によりエラー終了する。
  • 関数が集合を返さない場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • init_sexpr への引数として econtext->ecxt_per_query_memory が渡されており、関数情報はクエリ期間中保持される。
  • FuncExpr と OpExpr の両方を扱うため、新しい式タイプを追加する際は init_sexpr の呼び出し条件を拡張する必要がある。
