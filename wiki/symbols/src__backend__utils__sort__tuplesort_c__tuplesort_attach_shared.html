tuplesort_attach_shared

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesort.c:2960

目的

並列ソート処理において、ワーカープロセスが共有タプルソート状態（Sharedsort）および関連するファイルセットにアタッチするために使用されます。

シグネチャまたは形状

void tuplesort_attach_shared(Sharedsort *shared, dsm_segment *seg)

入力と出力

  • shared (入力): アタッチ対象となる共有ソート状態構造体へのポインタ。
  • seg (入力): 共有メモリセグメント（dsm_segment）へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • 本関数は、並列ソートを実行するすべてのワーカープロセスによって呼び出される必要があります。
  • shared および seg は有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った shared 構造体内の fileset メンバに対し、指定された共有メモリセグメント seg を用いて SharedFileSetAttach を実行します。 2. これにより、プロセス間で共有される一時ファイル群へのアクセス権が確立されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • 呼び出し先:
  • SharedFileSetAttach (内部関数)

関連データ構造

  • Sharedsort: 並列ソートの共有状態を保持する構造体。
  • dsm_segment: 動的共有メモリセグメントを管理する構造体。
  • SharedFileSet: 共有一時ファイルを管理する構造体。

失敗モードと境界条件

  • seg が無効な場合や、共有メモリへのアタッチに失敗した場合は、SharedFileSetAttach 内部でエラーが発生し、プロセスが中断される可能性があります。

保守担当者向けメモ

  • 本関数は並列ソートのセットアップにおいて不可欠なステップです。ワーカープロセスが正しく共有リソースにアクセスできるよう、呼び出し順序（初期化後、ソート実行前）を維持してください。
