visibilitymap_set

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/visibilitymap.c:247

目的

指定されたヒープブロックに対応する可視性マップ（VM）のビットをセットし、必要に応じて WAL 記録を行う。

シグネチャまたは形状

uint8 visibilitymap_set(Relation rel, BlockNumber heapBlk, Buffer heapBuf,
                        XLogRecPtr recptr, Buffer vmBuf, TransactionId cutoff_xid,
                        uint8 flags)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • heapBlk: 対象のヒープブロック番号。
  • heapBuf: ヒープページのバッファ（存在する場合）。
  • recptr: WAL 記録用の LSN（リカバリ時以外は無効値）。
  • vmBuf: 対象の可視性マップページのバッファ。
  • cutoff_xid: 可視性判定に使用するトランザクション ID。
  • flags: セットするビットフラグ（VISIBILITYMAP_VALID_BITS の範囲内）。
  • **出力**:
  • uint8: 更新前のビット状態。

前提条件と不変条件

  • vmBuf は対象のヒープブロックに対応するマップページであること。
  • heapBuf が提供される場合、そのブロック番号は heapBlk と一致すること。
  • flags に VISIBILITYMAP_ALL_FROZEN を指定する場合、同時に VISIBILITYMAP_ALL_VISIBLE も指定しなければならない。
  • リカバリ中または recptr が無効であること。

アルゴリズムまたは振る舞い

1. vmBuf に対して排他ロックを取得する。 2. 現在のビット状態を確認し、指定された flags と異なる場合のみ更新処理を行う。 3. 更新が必要な場合、クリティカルセクションを開始し、ビットをセットしてバッファをダーティにする。 4. リレーションが WAL を必要とする場合、log_heap_visible を呼び出して WAL 記録を行う。 5. データチェックサムが有効な場合、ヒープページの LSN を更新する。 6. クリティカルセクションを終了し、ロックを解放して更新前の状態を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_xlog.c
  • src/backend/access/heap/vacuumlazy.c
  • **呼び出し先**:
  • log_heap_visible（WAL 記録用）
  • LockBuffer, MarkBufferDirty, PageGetContents, BufferGetPage 等のバッファ管理関数。

関連データ構造

  • Relation: リレーション記述子。
  • Buffer: バッファプール内のページ。
  • Page: 可視性マップのページ内容。

失敗モードと境界条件

  • heapBuf または vmBuf のブロック番号が不一致の場合、elog(ERROR) を発生させる。
  • 不正なフラグが渡された場合、アサーション違反となる。

保守担当者向けメモ

  • 本関数はビットの「セット」のみを担当する。ビットの「クリア」は visibilitymap_clear が担当する。
  • WAL 記録は log_heap_visible を介して行われ、ヒープページの LSN 更新は XLogHintBitIsNeeded の判定に従う。
