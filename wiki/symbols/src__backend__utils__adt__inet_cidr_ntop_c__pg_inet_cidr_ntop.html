pg_inet_cidr_ntop

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/inet_cidr_ntop.c:55

目的

ネットワークアドレス（バイナリ形式）を、CIDRプレフィックス長を含む人間が読み取り可能な文字列形式へ変換します。

シグネチャまたは形状

char *pg_inet_cidr_ntop(int af, const void *src, int bits, char *dst, size_t size);

入力と出力

  • **入力**:
  • af: アドレスファミリ（PGSQL_AF_INET または PGSQL_AF_INET6）。
  • src: 変換対象のネットワークアドレスを保持するバッファへのポインタ。
  • bits: CIDRプレフィックス長。
  • dst: 変換結果を格納する出力バッファ。
  • size: dst バッファのサイズ。
  • **出力**:
  • 成功時: dst へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • src は指定されたアドレスファミリに対応する適切なバイナリ形式である必要があります。
  • dst バッファは、結果の文字列（CIDR表記を含む）を格納するのに十分なサイズである必要があります。

アルゴリズムまたは振る舞い

1. af 引数に基づき、IPv4 または IPv6 用の内部関数へ処理をディスパッチします。

  • PGSQL_AF_INET の場合: inet_cidr_ntop_ipv4 を呼び出します。
  • PGSQL_AF_INET6 の場合: inet_cidr_ntop_ipv6 を呼び出します。

    • 2. 内部関数は、指定された bits に基づきホスト部をゼロクリアした上で文字列化を行います。 3. 変換後の文字列を dst にコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network.c:1211
  • **呼び出し先**:
  • inet_cidr_ntop_ipv4 (IPv4の場合)
  • inet_cidr_ntop_ipv6 (IPv6の場合)

関連データ構造

  • inet 型および cidr 型（PostgreSQLのネットワーク関連データ型）。

失敗モードと境界条件

  • af がサポートされていない場合: errno に EAFNOSUPPORT を設定し、NULL を返します。
  • バッファサイズ不足: 内部関数にて EMSGSIZE が設定され、NULL が返されます。

保守担当者向けメモ

  • 本関数はディスパッチャとして機能します。具体的な変換ロジックの修正は、各アドレスファミリに対応する内部関数（inet_cidr_ntop_ipv4 または inet_cidr_ntop_ipv6）を確認してください。
  • ISCライセンスに基づくコードが含まれており、PostgreSQLの他モジュールとは異なるコーディングスタイルが混在しています。
