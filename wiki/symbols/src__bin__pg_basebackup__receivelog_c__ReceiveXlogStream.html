ReceiveXlogStream

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:451

目的

PostgreSQLサーバーからストリーミングレプリケーションプロトコルを使用してWAL（Write-Ahead Log）データを受信し、ローカルへ書き出すためのメインエントリーポイントとして機能します。

シグネチャまたは形状

bool ReceiveXlogStream(PGconn *conn, StreamCtl *stream);

入力と出力

  • **入力**:
  • conn: サーバーへの有効な PGconn 接続。
  • stream: ストリーミングの開始位置、タイムライン、レプリケーションスロット名、同期設定などを保持する StreamCtl 構造体。
  • **出力**:
  • bool: ストリーミングが正常に完了した場合は true、エラーが発生した場合は false を返します。

前提条件と不変条件

  • conn は既に確立されている必要があります。
  • stream->startpos は有効なLSNである必要があります。
  • サーバーのバージョンがストリーミングに対応している必要があります。
  • stream->sysidentifier が指定されている場合、サーバーのシステム識別子と一致する必要があります。

アルゴリズムまたは振る舞い

1. **初期化と検証**: サーバーバージョンを確認し、レプリケーションスロットや同期設定に基づいてフラッシュ位置の報告要否を決定します。システム識別子とタイムラインの整合性を検証します。 2. **タイムライン履歴の取得**: 必要なタイムライン履歴ファイルがローカルにない場合、TIMELINE_HISTORY コマンドを発行して取得し、ディスクに書き込みます。 3. **ストリーミングループ**:

  • stream_stop コールバックを呼び出し、終了条件を満たしているか確認します。
  • START_REPLICATION コマンドを発行し、COPY モードを開始します。
  • HandleCopyStream を呼び出し、WALデータの受信ループを実行します。

    • 4. **タイムライン切り替え**: HandleCopyStream が正常終了（PGRES_TUPLES_OK）した場合、サーバーから次のタイムライン情報を読み取り、新しいタイムラインでストリーミングを再開します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • **呼び出し先**:
  • CheckServerVersionForStreaming
  • RunIdentifySystem
  • existsTimeLineHistoryFile / writeTimeLineHistoryFile
  • PQexec / PQclear
  • HandleCopyStream
  • ReadEndOfStreamingResult

関連データ構造

  • StreamCtl: ストリーミングの制御パラメータを保持する構造体。
  • PGconn: libpq の接続オブジェクト。
  • XLogRecPtr: WALの論理シーケンス番号（LSN）。

失敗モードと境界条件

  • サーバーバージョンが不適合な場合、またはシステム識別子が不一致の場合は false を返します。
  • TIMELINE_HISTORY コマンドや START_REPLICATION コマンドが失敗した場合、エラーログを出力して false を返します。
  • サーバーから報告された新しいタイムラインが現在のタイムラインより古い場合、エラーとして終了します。

保守担当者向けメモ

  • stream->stream_stop コールバックはストリーミングの継続可否を決定する重要なフックです。
  • HandleCopyStream が返す res が NULL の場合は致命的なエラーとして扱われます。
  • タイムライン切り替え時の startpos の更新は、サーバー側のプロトコル仕様に依存するため注意が必要です。
