brin_form_placeholder_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_tuple.c:387

目的

BRINインデックスにおいて、特定のブロック範囲が空であることを示す「プレースホルダー」タプルを生成します。

シグネチャまたは形状

BrinTuple *brin_form_placeholder_tuple(BrinDesc *brdesc, BlockNumber blkno, Size *size)

入力と出力

  • **入力**:
  • brdesc: インデックスの記述子。
  • blkno: プレースホルダーを割り当てるブロック番号。
  • size: 生成されたタプルのサイズを格納するための出力ポインタ。
  • **出力**:
  • ゼロ初期化され、必要なフラグとNULLビットマップが設定された BrinTuple 構造体へのポインタ。

前提条件と不変条件

  • brdesc は有効なインデックス記述子であること。
  • 生成されるタプルは、BRIN_NULLS_MASK、BRIN_PLACEHOLDER_MASK、および BRIN_EMPTY_RANGE_MASK フラグが設定された状態となります。
  • 全ての属性に対して allnulls ビットがセットされます。

アルゴリズムまたは振る舞い

1. SizeOfBrinTuple に属性数に応じたNULLビットマップ領域を加算し、MAXALIGN でアライメント調整して必要なメモリサイズを算出します。 2. palloc0 を使用してメモリを確保し、bt_blkno と bt_info（フラグ群）を初期化します。 3. NULLビットマップ領域を走査し、全ての属性に対してNULLビット（1）をセットします。 4. 最終的なサイズを size 引数に書き込み、タプルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c:1774
  • **呼び出し先**:
  • palloc0 (PostgreSQLメモリ管理関数)

関連データ構造

  • BrinTuple: オンディスク上のBRINタプル構造体。
  • BrinDesc: インデックスの属性情報を含む記述子。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • 属性数が0の場合の挙動は brdesc->bd_tupdesc->natts に依存します。

保守担当者向けメモ

  • 本関数は BRIN_NULLS_MASK を使用するため、ビット操作のロジック（NULLを 1 とする仕様）を変更する際は、brin_tuple.c 内の他の関数との整合性に注意してください。
  • src/backend/access/brin/brin_tuple.c:96 にある通り、タプル構造の変更時には本関数の実装も確認する必要があります。
