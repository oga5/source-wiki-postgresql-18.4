NODE

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:22

目的

tsquery（全文検索クエリ）をツリー構造として表現し、最適化や不要ノードの除去といった再帰的な処理を可能にするための内部データ構造です。

シグネチャまたは形状

typedef struct NODE
{
	struct NODE *left;
	struct NODE *right;
	QueryItem  *valnode;
} NODE;

入力と出力

  • **入力**: maketree() 関数により、フラットな QueryItem 配列からツリー構造へ変換されます。
  • **出力**: plaintree() 関数により、処理後のツリー構造から再びフラットな QueryItem 配列へ変換されます。

前提条件と不変条件

  • ツリーの各ノードは、QueryItem を保持するか、あるいは演算子として左右の子ノードを保持します。
  • 再帰的な処理を行うため、スタックオーバーフローを防ぐ目的で check_stack_depth() を併用することが推奨されます。

アルゴリズムまたは振る舞い

  • tsquery のクエリツリーを走査し、clean_NOT_intree や clean_stopword_intree といった関数を通じて、特定のノード（NOT 演算子やストップワード）の除去や、フレーズ演算子の距離再計算を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: tsquery_cleanup.c 内の maketree, plaintree, clean_NOT_intree, clean_stopword_intree などがこの構造体を操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • **注意**: contrib/intarray/_int_bool.c や contrib/ltree/ltxtquery_io.c にも同名の NODE 構造体が定義されていますが、これらは別個の定義であり、本シンボルとの直接的な関連性は確認されていません。

関連データ構造

  • QueryItem: tsquery の各要素（演算子や検索語）を保持する構造体。
  • TSQuery: tsquery 型の全体を表現するオブジェクト。

失敗モードと境界条件

  • ツリーが非常に深い場合、再帰処理によるスタックオーバーフローのリスクがあります。
  • ストップワード除去時に、フレーズ演算子の距離（ladd, radd）の伝播ロジックが複雑であり、不適切な削除を行うとクエリの整合性が失われる可能性があります。

保守担当者向けメモ

  • 本構造体は tsquery_cleanup.c 内の再帰処理専用です。他のモジュール（contrib 配列など）にある同名構造体と混同しないよう注意してください。
  • ストップワード除去時の距離調整ロジックは、この構造体のポインタ操作に強く依存しています。変更時は clean_stopword_intree の再帰的な距離伝播ロジックを十分に検証してください。
