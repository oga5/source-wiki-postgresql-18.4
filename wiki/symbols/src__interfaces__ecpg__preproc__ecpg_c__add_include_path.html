add_include_path

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/ecpg.c:67

目的

EXEC SQL INCLUDE 文で検索されるディレクトリのリスト（include_paths）に、新しいパスを追加する。

シグネチャまたは形状

static void add_include_path(char *path)

入力と出力

  • **入力**: path (追加するディレクトリパスを示す文字列へのポインタ)
  • **出力**: なし（グローバル変数 include_paths を更新する副作用を持つ）

前提条件と不変条件

  • include_paths は struct _include_path 型の連結リストの先頭を指すグローバル変数である。
  • mm_alloc を使用してメモリを確保するため、メモリ不足時は呼び出し元で例外的な挙動となる可能性がある。

アルゴリズムまたは振る舞い

1. mm_alloc を用いて struct _include_path 型の新しいノードを確保する。 2. 新しいノードの path メンバに引数の path を設定し、next を NULL に初期化する。 3. include_paths が空（NULL）であれば、新しいノードをリストの先頭に設定する。 4. リストが空でない場合、リストの末尾まで走査し、最後のノードの next に新しいノードを連結する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/ecpg.c 内の main 関数にて、コマンドライン引数（-I オプション等）やデフォルトのインクルードパスを処理する際に呼び出される。
  • **呼び出し先**: mm_alloc（メモリ確保用ユーティリティ）。

関連データ構造

  • struct _include_path: インクルードパスのリストを保持する構造体。path（文字列）と next（次ノードへのポインタ）を持つ。

失敗モードと境界条件

  • リストが空の状態から初めてパスを追加する場合、正しく先頭ノードとして設定される。
  • 既に複数のパスが存在する場合、末尾に追加されるため、検索順序は追加された順序（FIFO）となる。

保守担当者向けメモ

  • 本関数はリストの末尾を探索するため、追加するパスの数が多い場合、計算量はリストの長さに比例する。
  • include_paths はグローバル変数であるため、複数ファイル処理時のリセット処理が適切に行われているか注意すること。
