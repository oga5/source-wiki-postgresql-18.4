目的

ネパール語のステミング処理において、特定のカテゴリ（カテゴリ2）に分類される接尾辞を単語の末尾から識別し、削除する。

シグネチャまたは形状

static int r_remove_category_2(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z（Snowballランタイム環境へのポインタ）
  • **出力**: int（処理が成功した場合は 1、失敗またはマッチしなかった場合は 0、エラー発生時は負の値を返す）

前提条件と不変条件

  • z->c（現在のカーソル位置）が単語の末尾付近にあること。
  • find_among_b を用いた後方検索が可能な状態であること。
  • 処理対象の文字がUTF-8エンコーディングに従っていること。

アルゴリズムまたは振る舞い

1. **初期化**: z->ket を現在のカーソル位置 z->c に設定する。 2. **事前チェック**: 文字列の長さと、末尾文字のビットパターンに基づき、削除対象の可能性があるか判定する。 3. **検索**: a_2 テーブルを用いて find_among_b を実行し、接尾辞を特定する。 4. **マッチングと削除**:

  • among_var == 1: s_2, s_3, s_4, s_5 のいずれかの接尾辞と一致するか順次確認し、一致した場合は slice_del で削除する。
  • among_var == 2: s_6 と一致するか確認し、一致した場合は slice_del で削除する。

    • 5. **終了**: 削除が成功した場合は 1 を返し、それ以外は 0 またはエラーコードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_nepali.c 内のメイン処理ループから呼び出される。
  • **呼び出し先**: find_among_b, eq_s_b, slice_del（いずれもSnowballランタイム関数）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_2: カテゴリ2の接尾辞検索用 among 構造体テーブル。
  • s_2 ～ s_6: 比較対象となる接尾辞文字列。

失敗モードと境界条件

  • find_among_b がマッチしない場合、0 を返して終了する。
  • slice_del が内部エラー（負の値）を返した場合、その値をそのまま呼び出し元へ伝播させる。
  • 文字列が短すぎて接尾辞を構成できない場合、事前チェックにより早期終了する。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによって自動生成されたコードである。直接編集せず、元のSnowballスクリプト（.sbl）を修正すること。
