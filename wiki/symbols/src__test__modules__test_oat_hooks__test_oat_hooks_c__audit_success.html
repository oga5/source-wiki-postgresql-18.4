audit_success

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_oat_hooks/test_oat_hooks.c:263

目的

PostgreSQLのオブジェクトアクセスフック等の処理が正常に完了したことを示す監査メッセージを出力します。

シグネチャまたは形状

static void audit_success(const char *hook, char *action, char *objName)

入力と出力

  • **入力**:
  • hook: 実行されたフックの種類を示す文字列。
  • action: 実行された操作内容を示す文字列。
  • objName: 操作対象のオブジェクト名。
  • **出力**: なし（void）。内部で emit_audit_message を呼び出し、ログを出力します。

前提条件と不変条件

  • 本関数は、対象となる操作がエラーなく完了した直後に呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. 引数として受け取った hook、action、objName を保持します。 2. emit_audit_message 関数を呼び出し、第一引数に "finished" を指定して、操作が完了した旨の監査ログを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • REGRESS_object_access_hook_str (行318)
  • REGRESS_object_access_hook (行341)
  • REGRESS_exec_check_perms (行367)
  • REGRESS_utility_command (行412)
  • **呼び出し先**:
  • emit_audit_message

関連データ構造

  • なし。

失敗モードと境界条件

  • emit_audit_message の内部実装に依存します。並列ワーカープロセス内での呼び出しについては、emit_audit_message 側で重複出力防止の制御が行われています。

保守担当者向けメモ

  • 本関数はテストモジュール専用です。監査ログの形式を変更する場合は、emit_audit_message の仕様と整合性をとる必要があります。
