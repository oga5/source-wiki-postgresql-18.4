verify_plain_backup_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:668

目的

プレーン形式のバックアップディレクトリ内の個別のファイルまたはディレクトリを検証し、マニフェスト情報との整合性を確認する。

シグネチャまたは形状

static void verify_plain_backup_file(verifier_context *context, char *relpath, char *fullpath);

入力と出力

  • **入力**:
  • context: 検証の状態（マニフェストデータ、無視リスト等）を保持する verifier_context 構造体へのポインタ。
  • relpath: バックアップルートからの相対パス。
  • fullpath: ファイルシステム上の絶対パス。
  • **出力**: なし（副作用としてエラー報告やコンテキストの更新を行う）。

前提条件と不変条件

  • context->manifest が初期化済みであること。
  • fullpath が指すパスがファイルシステム上に存在するか、あるいはアクセス可能であること。

アルゴリズムまたは振る舞い

1. **ファイル状態の確認**: stat を実行し、失敗した場合はエラーを報告して relpath を無視リストに追加する。 2. **ディレクトリの処理**: ディレクトリの場合は verify_plain_backup_directory を再帰的に呼び出す。 3. **ファイル形式の検証**: 通常のファイル（S_ISREG）であることを確認し、それ以外の場合はエラーを報告する。 4. **マニフェスト照合**: manifest_files_lookup を使用し、マニフェストに存在しない場合はエラーを報告する。 5. **整合性チェック**:

  • マニフェスト上のファイルサイズとディスク上のサイズを比較し、不一致があればエラーを報告して m->bad フラグを立てる。
  • マニフェストバージョンが 1 より大きく、かつ pg_control ファイルである場合、verify_control_file を実行する。

    • 6. **統計更新**: 進捗表示が有効でチェックサム検証がスキップされていない場合、total_size を加算する。 7. **注記**: チェックサム自体の計算・検証はこの関数内では行わず、ファイル構造とサイズの検証を優先する。

関連データ構造

  • verifier_context: 検証全体のコンテキスト。
  • manifest_file: マニフェスト内のファイル情報。
  • struct stat: ファイルシステム上の属性情報。

失敗モードと境界条件

  • **stat 失敗**: ファイルまたはディレクトリにアクセスできない場合、エラーを報告し、そのパスを無視リストに追加して処理を中断する。
  • **ファイル種別不一致**: 通常のファイルやディレクトリではない場合（シンボリックリンクやデバイスファイル等）、エラーを報告する。
  • **マニフェスト不一致**: マニフェストに存在しないファイル、またはサイズが一致しないファイルはエラーとして記録される。

保守担当者向けメモ

  • 本関数はチェックサム検証の「前段階」の検証を担当する。チェックサム計算は負荷が高いため、構造的な不整合を先に排除する設計となっている。
  • XLOG_CONTROL_FILE の検証は、マニフェストバージョン 1 ではスキップされる仕様である。
