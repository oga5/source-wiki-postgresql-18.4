like_regex_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_support.c:155

目的

プランナからの要求（選択率推定またはインデックス条件生成）を振り分け、パターンマッチング演算子に対する最適化支援を行うメインエントリポイントです。

シグネチャまたは形状

static Node *like_regex_support(Node *rawreq, Pattern_Type ptype)

入力と出力

  • **入力**:
  • rawreq: プランナからのサポート要求ノード（SupportRequestSelectivity または SupportRequestIndexCondition）。
  • ptype: パターンマッチングの種類を示す Pattern_Type 列挙値。
  • **出力**:
  • 処理結果を含む Node 構造体。要求が満たせない場合は NULL を返します。

前提条件と不変条件

  • rawreq は SupportRequestSelectivity または SupportRequestIndexCondition のいずれかである必要があります。
  • インデックス条件生成時、インデックスキーが左辺にある場合のみ処理が行われます。

アルゴリズムまたは振る舞い

1. **選択率推定 (SupportRequestSelectivity)**:

  • 結合（Join）の場合はデフォルト値（DEFAULT_MATCH_SEL）を返します。
  • 単一テーブルの制約の場合は patternsel_common を呼び出し、統計情報に基づいた選択率を計算して req->selectivity に格納します。

    • 2. **インデックス条件生成 (SupportRequestIndexCondition)**:

  • OpExpr または FuncExpr 形式の句を解析します。
  • match_pattern_prefix を呼び出し、パターンから固定接頭辞を抽出してインデックススキャン可能な条件式を構築します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • textlike_support
  • texticlike_support
  • textregexeq_support
  • texticregexeq_support
  • text_starts_with_support
  • **呼び出し先**:
  • patternsel_common (選択率計算)
  • match_pattern_prefix (インデックス条件生成)

関連データ構造

  • Pattern_Type: パターンマッチングの種類を定義する列挙型。
  • SupportRequestSelectivity: 選択率推定要求用ノード。
  • SupportRequestIndexCondition: インデックス条件生成要求用ノード。

失敗モードと境界条件

  • req->indexarg != 0 の場合（インデックスキーが左辺にない場合）、インデックス条件生成は行われず NULL を返します。
  • 結合クエリの選択率推定は現在サポートされておらず、デフォルト値が返されます。

保守担当者向けメモ

  • 本関数はプランナのフックとして機能します。新しいパターンマッチング手法を追加する場合は、Pattern_Type を拡張し、本関数内の分岐を更新する必要があります。
  • match_pattern_prefix への引数として opfamily や indexcollation を渡しており、インデックスの照合順序に依存した最適化が行われます。
