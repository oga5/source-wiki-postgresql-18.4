utf8_to_shift_jis_2004

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_sjis2004/utf8_and_sjis2004.c:62

目的

UTF-8 エンコーディングの文字列を SHIFT_JIS_2004 エンコーディングへ変換する PostgreSQL の変換プロシージャです。

シグネチャまたは形状

Datum utf8_to_shift_jis_2004(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取ります。
  • src (第3引数): 変換元 UTF-8 文字列バッファ。
  • dest (第4引数): 変換先 SHIFT_JIS_2004 文字列バッファ。
  • len (第5引数): 変換対象のバイト長。
  • noError (第6引数): 変換不能文字発生時のエラー抑制フラグ。
  • **出力**: 変換されたバイト数を Datum 型（int32）として返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_UTF8, PG_SHIFT_JIS_2004) により、呼び出し時のエンコーディングIDが UTF-8 および SHIFT_JIS_2004 であることが検証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. UtfToLocal 関数を呼び出し、以下のマッピングデータを用いて変換を実行します。

  • shift_jis_2004_from_unicode_tree
  • ULmapSHIFT_JIS_2004_combined

    • 3. noError フラグが真の場合、変換不能な文字が含まれていてもエラーを発生させずに処理を継続します。 4. 変換されたバイト数を PG_RETURN_INT32 で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の文字コード変換サブシステム（mb）から動的に呼び出されます。ソースコード内では PG_FUNCTION_INFO_V1(utf8_to_shift_jis_2004) によりエントリポイントとして登録されています。
  • **呼び出し先**: UtfToLocal（mb/pg_wchar.h で定義）を呼び出します。

関連データ構造

  • shift_jis_2004_from_unicode_tree: 変換用ツリー構造。
  • ULmapSHIFT_JIS_2004_combined: 変換用マップテーブル。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断されます。
  • noError が偽の場合、変換不能な文字が見つかると UtfToLocal 内部でエラーが発生します。

保守担当者向けメモ

  • 本関数は utf8_to_shift_jis_2004.map に依存しています。マッピングの修正が必要な場合は、src/backend/utils/Unicode/ 以下の生成スクリプトを確認してください。
  • 実際の変換ロジックは UtfToLocal に委譲されているため、変換エンジンの挙動変更は mb/pg_wchar.h 関連のコードを参照してください。
