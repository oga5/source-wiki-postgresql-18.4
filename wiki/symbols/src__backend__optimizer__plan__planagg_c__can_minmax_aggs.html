can_minmax_aggs

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/planagg.c:236

目的

クエリ内の集約関数が MIN または MAX であり、かつインデックススキャンによる最適化が可能かどうかを検証し、対象となる集約関数のメタデータを収集する。

シグネチャまたは形状

static bool can_minmax_aggs(PlannerInfo *root, List **context)

入力と出力

  • **入力**:
  • root: 現在のクエリの PlannerInfo 構造体。
  • context: 最適化対象の MinMaxAggInfo 構造体を格納するためのリストへのポインタ。
  • **出力**:
  • 戻り値: すべての集約関数が最適化可能と判断された場合は true、一つでも条件を満たさないものがあれば false。
  • context: 最適化対象として抽出された MinMaxAggInfo のリストが追加される。

前提条件と不変条件

  • root->agginfos に集約関数の情報が格納されていること。
  • 対象となる Aggref の agglevelsup が 0 であること（サブクエリ内の集約ではないこと）。

アルゴリズムまたは振る舞い

1. root->agginfos リストを走査し、各集約関数に対して以下の条件をチェックする。

  • 引数が単一であること。
  • ORDER BY 句が含まれていないこと。
  • FILTER 句が含まれていないこと。
  • fetch_agg_sort_op により有効なソート演算子（MIN/MAX に対応するもの）が取得できること。
  • 引数式に可変関数（contain_mutable_functions）が含まれていないこと。
  • 引数の型がロウタイプ（type_is_rowtype）ではないこと。

    • 2. すべての条件を満たす場合、MinMaxAggInfo 構造体を生成し、context リストに追加する。 3. いずれかの条件で不適合と判断された場合、直ちに false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: preprocess_minmax_aggregates (src/backend/optimizer/plan/planagg.c:144)
  • **呼び出し先**:
  • fetch_agg_sort_op
  • contain_mutable_functions
  • type_is_rowtype
  • exprType
  • makeNode

関連データ構造

  • AggInfo: root->agginfos の要素。
  • MinMaxAggInfo: 最適化対象の集約関数情報を保持する構造体。
  • PlannerInfo: クエリのプランニング情報を保持する構造体。

失敗モードと境界条件

  • ORDER BY 句や FILTER 句が存在する場合、最適化を断念し false を返す。
  • 引数が可変関数である場合、インデックスによる効率的な検索が保証できないため false を返す。
  • ロウタイプ（複合型）の引数は、IS NOT NULL などのセマンティクスが複雑になるため対象外とする。

保守担当者向けメモ

  • 本関数は MIN/MAX 最適化のフィルタリングを担当する。将来的に FILTER 句付きの集約をサポートする場合、この関数内のチェックロジックを拡張する必要がある。
  • MinMaxAggInfo の subroot や path フィールドは本関数では初期化されず、後続のパス生成フェーズで設定される。
