CRSSnapshotAction

定義

  • 種別: enum
  • 定義箇所: src/include/replication/walsender.h:20

目的

レプリケーションスロット作成時における、スナップショットの取り扱い方針を定義する列挙型です。

シグネチャまたは形状

typedef enum
{
	CRS_EXPORT_SNAPSHOT,
	CRS_NOEXPORT_SNAPSHOT,
	CRS_USE_SNAPSHOT,
} CRSSnapshotAction;

入力と出力

本シンボル自体は列挙型であり、関数ではありません。レプリケーション関連の関数（walsender.c や libpqwalreceiver.c 内の関数）において、引数やローカル変数としてスナップショットの挙動を制御するために使用されます。

前提条件と不変条件

  • CRS_EXPORT_SNAPSHOT、CRS_NOEXPORT_SNAPSHOT、CRS_USE_SNAPSHOT のいずれかの値をとります。
  • レプリケーションプロトコルにおけるスナップショットのライフサイクル管理と密接に関連しています。

アルゴリズムまたは振る舞い

レプリケーションスロット作成プロセスにおいて、以下のいずれかの動作を指示するために使用されます。

  • CRS_EXPORT_SNAPSHOT: スナップショットをエクスポートする。
  • CRS_NOEXPORT_SNAPSHOT: スナップショットをエクスポートしない。
  • CRS_USE_SNAPSHOT: 既存のスナップショットを使用する。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c
  • src/backend/replication/walsender.c
  • src/include/replication/walreceiver.h
  • **呼び出し先**:
  • 本シンボルは列挙型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • XLogRecPtr（access/xlogdefs.h で定義）: スナップショットに関連するLSN情報の保持に使用される場合があります。

失敗モードと境界条件

  • 不適切な列挙値が渡された場合の動作については、本シンボル定義からは判断できません。各利用箇所でのバリデーションに依存します。

保守担当者向けメモ

  • 本列挙型は、WAL SenderとWAL Receiver間の通信プロトコルにおけるスナップショット処理の分岐点となります。変更を加える際は、libpqwalreceiver.c および walsender.c 側のロジックとの整合性に注意してください。
