uniq

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_op.c:249

目的

整数配列から重複する要素を削除し、一意な要素のみを持つ配列を返します。

シグネチャまたは形状

Datum uniq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される ArrayType 型の整数配列。
  • **出力**: 重複が排除された新しい ArrayType 型の配列（Datum）。

前提条件と不変条件

  • 入力配列は CHECKARRVALID マクロにより整合性が検証されている必要があります。
  • 内部処理において、入力配列のコピーが作成されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_ARRAYTYPE_P_COPY(0) を使用して入力配列のコピーを取得します。 2. CHECKARRVALID マクロで配列の妥当性を確認します。 3. 配列の要素数が2未満の場合は、そのまま配列を返します。 4. _int_unique 関数を呼び出し、重複排除処理を実行します。 5. 処理結果の配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されます（contrib/intarray/_int_op.c:172 にて PG_FUNCTION_INFO_V1 が定義されています）。
  • **呼び出し先**: _int_unique（contrib/intarray/_int_tool.c 内の関数と推測されます）。

関連データ構造

  • ArrayType: PostgreSQLの配列データ型。

失敗モードと境界条件

  • 要素数が0または1の配列が渡された場合、重複排除を行わずにそのまま返却されます。
  • CHECKARRVALID による検証に失敗した場合、エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は入力配列をコピーして操作するため、元のデータは変更されません。
  • 効率的な重複排除のためには、事前に配列がソートされていることが期待されます。
