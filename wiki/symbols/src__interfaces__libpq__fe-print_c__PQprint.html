PQprint

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-print.c:68

目的

PGresult オブジェクトに含まれるクエリ結果を、PQprintOpt で指定されたオプションに基づき、表形式やHTMLなどの人間が読みやすい形式で出力ストリームへ書き出します。

シグネチャまたは形状

void PQprint(FILE *fout, const PGresult *res, const PQprintOpt *po);

入力と出力

  • **入力**:
  • fout: 出力先のファイルストリーム（NULLの場合は stdout を使用）。
  • res: 出力対象の PGresult オブジェクト。
  • po: 出力形式を制御する PQprintOpt 構造体へのポインタ。
  • **出力**: なし（戻り値は void）。指定されたストリームへ整形済みテキストを出力します。

前提条件と不変条件

  • res に含まれるフィールド数が 0 より大きい場合にのみ処理を実行します。
  • po->fieldSep の長さが strlen で正しく計算可能である必要があります。
  • ページャ機能を使用する場合、fout が stdout であり、かつ標準入力と標準出力が端末（isatty）である必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: PQnfields で列数を取得し、フィールド名や最大幅を格納するためのメモリを確保します。 2. **メタデータ計算**: 各フィールドの名称と最大幅を計算し、出力に必要な行全体の長さを算出します。 3. **ページャ判定**: 端末サイズ（TIOCGWINSZ）を確認し、出力が画面サイズを超える場合は getenv("PAGER") を使用してページャを起動します。 4. **ヘッダー出力**: po->header が有効な場合、表のヘッダー行や枠線を出力します。 5. **データ出力**: po->expanded（展開表示）や po->html3（HTML形式）の指定に従い、各行のデータを整形して出力します。 6. **終了処理**: 確保したメモリを解放し、ページャを使用した場合は pclose でストリームを閉じます。

呼び出し元と呼び出し先

  • src/test/isolation/isolationtester.c:1121
  • **呼び出し先**:

  • pq_block_sigpipe (内部ヘルパー)

関連データ構造

  • PGresult: 出力対象となるクエリ結果セット。
  • PQprintOpt: 出力形式（展開表示、ヘッダー、HTML、区切り文字等）を定義する構造体。
  • winsize: 端末の行数・列数を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: calloc が失敗した場合、エラーメッセージを stderr に出力して終了します。
  • **不正な区切り文字**: po->fieldSep が異常に長い場合、エラーを出力して終了します。
  • **ページャ起動失敗**: popen が失敗した場合、通常の stdout への出力にフォールバックします。
  • **フィールド数 0**: フィールドが存在しない結果セットの場合、何も出力せずに終了します。

保守担当者向けメモ

  • 本関数はレガシーなインターフェースであり、現代の psql では使用されていません。将来的に削除される可能性があるため、新規開発での利用は推奨されません。
  • ページャの起動処理にはプラットフォーム依存のシグナル制御（SIGPIPE マスク）が含まれており、変更時には注意が必要です。
  • do_field 等の内部補助関数と連携して動作するため、修正時は関連する補助関数との整合性を確認してください。
