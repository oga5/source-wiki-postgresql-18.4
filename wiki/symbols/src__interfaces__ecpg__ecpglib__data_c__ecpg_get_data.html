ecpg_get_data

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/ecpg/ecpglib/data.c:205

目的

PostgreSQL のクエリ結果セット（PGresult）から特定のフィールド値を取得し、ECPG アプリケーションのホスト変数およびインジケータ変数へ変換・転送する。

シグネチャまたは形状

bool ecpg_get_data(const PGresult *results, int act_tuple, int act_field, int lineno,
                   enum ECPGttype type, enum ECPGttype ind_type,
                   char *var, char *ind, long varcharsize, long offset,
                   long ind_offset, enum ARRAY_TYPE isarray, enum COMPAT_MODE compat, bool force_indicator);

入力と出力

  • **入力**:
  • results: PGresult オブジェクト。
  • act_tuple, act_field: 取得対象の行および列インデックス。
  • type, ind_type: ホスト変数およびインジケータ変数の ECPG 型。
  • var, ind: ホスト変数およびインジケータ変数のメモリ領域へのポインタ。
  • varcharsize, offset, ind_offset: メモリ配置とサイズ制御用パラメータ。
  • isarray, compat, force_indicator: 処理モード指定。
  • **出力**: 成功時に true、失敗時に false を返す。

前提条件と不変条件

  • ECPGget_sqlca() が有効な sqlca_t 構造体を返すこと。
  • PQgetvalue で取得した値が NULL の場合、インジケータ変数に -1 が設定される。
  • isarray が ECPG_ARRAY_ARRAY の場合、入力値は {...} 形式である必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: PQgetvalue で値を取得し、バイナリかテキストかを確認。 2. **インジケータ処理**: PQgetisnull を判定し、NULL であればインジケータ変数に -1 を書き込み、処理を終了する。 3. **配列検証**: 配列型の場合、開始文字 { を確認し、必要に応じてポインタを進める。 4. **データ転送**:

  • **バイナリモード**: memcpy を使用して直接コピー。切り捨てが発生した場合はインジケータ変数に実際のサイズを書き込み、sqlca に警告を設定。
  • **テキストモード**: 指定された type に応じて適切な変換関数（strtol, strtod 等）を呼び出す（ソース抜粋では ECPGt_short までの実装を確認）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/execute.c
  • src/interfaces/ecpg/ecpglib/sqlda.c
  • **呼び出し先**:
  • ECPGget_sqlca, PQgetvalue, PQfformat, PQgetlength, PQgetisnull
  • ecpg_raise, ecpg_log, ECPGset_noind_null, ecpg_type_name

関連データ構造

  • PGresult: libpq の結果セット構造体。
  • sqlca_t: SQL 通信領域。
  • ECPGttype: ECPG で定義された型列挙。

失敗モードと境界条件

  • sqlca が取得できない場合、ECPG_OUT_OF_MEMORY を発生させ false を返す。
  • 値が見つからない場合、ECPG_NOT_FOUND を発生させ false を返す。
  • インジケータ変数の型がサポート外の場合、ECPG_UNSUPPORTED を発生させる。
  • 配列形式ではない値に対して配列処理を試みた場合、ECPG_DATA_NOT_ARRAY を発生させる。
  • force_indicator が有効でインジケータ変数がない場合に NULL が返された場合、ECPG_MISSING_INDICATOR を発生させる。

保守担当者向けメモ

  • INFORMIX_MODE 等の互換性モードによる挙動差異が switch 文や条件分岐に含まれるため、修正時は注意が必要。
  • 配列処理のロジックは複雑であり、pval ポインタの操作が正確に行われているか確認すること。
  • バイナリモードでの切り捨て警告は sqlca->sqlwarn を更新する。
