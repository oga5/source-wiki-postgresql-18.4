create_or_open_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:935

目的

ダンプ出力先として指定されたディレクトリの存在確認、作成、および権限設定を行い、ディレクトリの状態がダンプ処理に適しているかを検証する。

シグネチャまたは形状

void create_or_open_dir(const char *dirname)

入力と出力

  • **入力**: dirname (作成または開く対象のディレクトリパス)
  • **出力**: なし（成功時は正常終了、失敗時は pg_fatal によりプロセスを終了）

前提条件と不変条件

  • pg_dir_create_mode で定義された権限設定が適用可能であること。
  • ディレクトリが既に存在する場合、そのディレクトリは空でなければならない（空でない場合はエラーとなる）。

アルゴリズムまたは振る舞い

1. pg_check_dir(dirname) を呼び出し、ディレクトリの状態を判定する。 2. 判定結果に基づき以下の処理を行う:

  • -1: opendir 失敗（ENOENT 以外）。pg_fatal で異常終了。
  • 0: ディレクトリが存在しない。mkdir で作成し、pg_dir_create_mode を適用。失敗時は pg_fatal。
  • 1: ディレクトリが存在し、かつ空である。chmod で pg_dir_create_mode を適用。失敗時は pg_fatal。
  • default: ディレクトリが存在するが空ではない。pg_fatal で異常終了。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_directory.c:161 での呼び出しが確認されている。
  • **呼び出し先**: pg_check_dir（ディレクトリ状態確認）、mkdir（ディレクトリ作成）、chmod（権限変更）、pg_fatal（エラー報告）。

関連データ構造

  • pg_dir_create_mode: ディレクトリ作成時に使用される権限定数。

失敗モードと境界条件

  • 指定されたパスがディレクトリでない、またはアクセス権限がない場合は pg_fatal により終了する。
  • 既存のディレクトリが空でない場合、ダンプファイルの破損を防ぐためにエラーとして処理される。

保守担当者向けメモ

  • 本関数は pg_check_dir の結果に強く依存している。ディレクトリの検証ロジックを変更する場合は pg_check_dir の仕様と整合性を取ること。
  • pg_fatal を使用しているため、呼び出し側でエラーをキャッチして復旧することはできない。
