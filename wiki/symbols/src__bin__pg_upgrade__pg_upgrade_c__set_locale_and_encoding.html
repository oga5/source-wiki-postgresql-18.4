set_locale_and_encoding

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.c:440

目的

新クラスタの template0 データベースに対し、旧クラスタから取得したロケールおよびエンコーディング設定を適用し、メタデータを同期させる。

シグネチャまたは形状

static void set_locale_and_encoding(void)

入力と出力

  • **入力**: グローバル変数 old_cluster（old_cluster.template0 を介してロケール情報を参照）および new_cluster。
  • **出力**: なし（副作用として新クラスタの pg_catalog.pg_database を更新）。

前提条件と不変条件

  • old_cluster が初期化済みであること。
  • new_cluster への接続が可能であること。
  • new_cluster のメジャーバージョンに応じて、更新対象のシステムカタログ列（datlocale、daticulocale 等）が適切に選択されること。

アルゴリズムまたは振る舞い

1. connectToServer を使用して新クラスタの template1 に接続する。 2. old_cluster.template0 から取得したロケール情報（db_collate, db_ctype, db_locale）を、新クラスタの接続先に対して PQescapeLiteral でエスケープする。 3. 新クラスタのメジャーバージョンを確認し、以下のいずれかの SQL を実行して pg_database の template0 行を更新する：

  • **v17以降**: encoding, datlocprovider, datcollate, datctype, datlocale を更新。
  • **v15-16**: encoding, datlocprovider, datcollate, datctype, daticulocale を更新。
  • **v14以前**: encoding, datcollate, datctype を更新。

    • 4. エスケープされたリテラルメモリを解放し、接続を閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_upgrade/pg_upgrade.c:150）。
  • **呼び出し先**: connectToServer, PQescapeLiteral, executeQueryOrDie, PQclear, PQfreemem, PQfinish, prep_status, check_ok。

関連データ構造

  • DbLocaleInfo: ロケール情報を保持する構造体（old_cluster.template0 が参照）。
  • ClusterInfo: 新旧クラスタの情報を保持する構造体。

失敗モードと境界条件

  • connectToServer が失敗した場合、処理は中断される（executeQueryOrDie の内部挙動に依存）。
  • メジャーバージョン判定が不適切な場合、存在しないシステムカタログ列へのアクセスにより SQL エラーが発生する可能性がある。
  • locale->db_locale が NULL の場合、SQL 上で "NULL" 文字列として処理される。

保守担当者向けメモ

  • 本関数は PostgreSQL のメジャーバージョンアップグレードにおける「新クラスタの初期化」フェーズの一部である。
  • バージョンごとの SQL 分岐は、システムカタログのスキーマ変更に追従する必要がある。
  • PQescapeLiteral の使用により、ロケール名に含まれる特殊文字による SQL インジェクションを防止している。
