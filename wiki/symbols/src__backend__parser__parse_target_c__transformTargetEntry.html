transformTargetEntry

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_target.c:74

目的

構文解析されたターゲットリストの要素（ResTarget相当）を、プランナーが処理可能な TargetEntry ノードへ変換する。

シグネチャまたは形状

TargetEntry *transformTargetEntry(ParseState *pstate, Node *node, Node *expr, ParseExprKind exprKind, char *colname, bool resjunk)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態。
  • node: 変換対象の構文解析ノード。
  • expr: 既に変換済みの式（未変換の場合は NULL）。
  • exprKind: 式のコンテキスト（ParseExprKind）。
  • colname: 列名（指定がない場合は NULL）。
  • resjunk: ターゲットリストから除外すべき「ジャンク」列かどうかを示すフラグ。
  • **出力**: 変換済みの TargetEntry 構造体へのポインタ。

前提条件と不変条件

  • pstate->p_next_resno がターゲットリストの連番管理に使用される。
  • expr が NULL の場合、transformExpr を用いて式が変換される。ただし、UPDATE ソースかつ SetToDefault ノードの場合は変換をスキップする。

アルゴリズムまたは振る舞い

1. **式の解決**: expr が NULL の場合、transformExpr を呼び出して式を変換する。 2. **列名の推論**: colname が NULL かつ resjunk が false の場合、FigureColname(node) を呼び出して式から列名を自動生成する。 3. **ノード生成**: makeTargetEntry を呼び出し、pstate->p_next_resno をインクリメントしながら TargetEntry を構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_clause.c:2213
  • src/backend/parser/parse_target.c:181
  • **呼び出し先**:
  • transformExpr (式変換)
  • FigureColname (列名推論)
  • makeTargetEntry (ノード構築)

関連データ構造

  • TargetEntry: 変換後のターゲットリスト要素。
  • ParseState: 解析状態を保持する構造体。
  • SetToDefault: UPDATE 文などで使用される特殊なノード。

失敗モードと境界条件

  • expr が NULL でかつ transformExpr が失敗した場合、例外が発生する。
  • resjunk が true の場合、列名の推論はスキップされる。

保守担当者向けメモ

  • 本関数は p_next_resno をインクリメントするため、呼び出し順序がターゲットリストの順序に直結する。
  • SetToDefault の扱いは UPDATE 文の整合性に依存するため、変更時は parse_expr.c の型チェックロジックと整合させること。
