pg_SSPI_startup

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth.c:363

目的

SSPI（Security Support Provider Interface）認証プロセスの初期化を行い、認証用クレデンシャルの取得とターゲットプリンシパル名の設定を行う。

シグネチャまたは形状

static int pg_SSPI_startup(PGconn *conn, int use_negotiate, int payloadlen)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体へのポインタ。
  • use_negotiate: ネゴシエーションモードを使用するかどうかを示すフラグ（真なら "negotiate"、偽なら "kerberos" を使用）。
  • payloadlen: 認証ペイロードの長さ。
  • **出力**:
  • 成功時は pg_SSPI_continue の戻り値を返し、失敗時は STATUS_ERROR を返す。

前提条件と不変条件

  • conn->sspictx が既に存在する場合、二重の認証要求としてエラーを返す。
  • 接続先のホスト名（conn->connhost[...].host）が空でないこと。

アルゴリズムまたは振る舞い

1. 既に SSPI コンテキストが存在しないか確認する。 2. conn->sspicred 用のメモリを確保する。 3. AcquireCredentialsHandle を呼び出し、指定されたモード（negotiate または kerberos）でクレデンシャルを取得する。失敗時は pg_SSPI_error を呼び出し、メモリを解放して終了する。 4. ホスト名が有効であることを確認し、conn->krbsrvname とホスト名からターゲットプリンシパル名（conn->sspitarget）を生成する。 5. conn->usesspi を 1 に設定し、SSPI モードであることを記録する。 6. pg_SSPI_continue を呼び出し、認証の継続処理へ移行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth.c 内の認証フロー（pg_fe_sendauth 等の文脈）。
  • **呼び出し先**:
  • AcquireCredentialsHandle (Windows API)
  • pg_SSPI_error
  • pg_SSPI_continue
  • libpq_append_conn_error

関連データ構造

  • PGconn: 接続情報を保持する構造体。sspicred（クレデンシャルハンドル）、sspitarget（ターゲット名）、usesspi（フラグ）を使用。
  • CredHandle: SSPI クレデンシャルを保持する構造体。

失敗モードと境界条件

  • メモリ不足（malloc 失敗）: STATUS_ERROR を返す。
  • クレデンシャル取得失敗: AcquireCredentialsHandle が SEC_E_OK 以外を返した場合、エラーを記録して STATUS_ERROR を返す。
  • ホスト名未指定: STATUS_ERROR を返す。
  • 重複する認証要求: STATUS_ERROR を返す。

保守担当者向けメモ

  • 本関数は Windows 環境での SSPI 認証に特化しています。
  • conn->sspitarget のメモリ確保は krbsrvname とホスト名の長さに依存するため、バッファオーバーランに注意してください。
  • 認証プロセスは非同期的に進行する可能性があるため、pg_SSPI_continue への引き継ぎが正しく行われるよう設計されています。
