SetMessageEncoding

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/mbutils.c:1296

目的

現在のメッセージエンコーディング設定（MessageEncoding）を、指定されたエンコーディングIDに基づいて更新します。

シグネチャまたは形状

void SetMessageEncoding(int encoding)

入力と出力

  • **入力**: encoding (int型) - 設定するエンコーディングのID。
  • **出力**: なし（void型）。

前提条件と不変条件

  • encoding は PG_VALID_ENCODING マクロによって有効なエンコーディングであることが保証されていなければなりません。
  • 実行後、MessageEncoding が指し示す構造体の encoding フィールドは、入力された encoding と一致する必要があります。

アルゴリズムまたは振る舞い

1. PG_VALID_ENCODING(encoding) を用いて、入力値が有効なエンコーディング範囲内にあるかアサーションで検証します。 2. pg_enc2name_tbl テーブルから指定されたIDに対応するエントリを検索し、グローバル変数 MessageEncoding を更新します。 3. 更新後の MessageEncoding->encoding が入力値と一致することを確認するアサーションを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c:244
  • src/backend/utils/adt/pg_locale.c:246
  • **呼び出し先**:
  • 本関数内での明示的な関数呼び出しはありません（マクロおよびテーブル参照のみ）。

関連データ構造

  • pg_enc2name_tbl: エンコーディングIDと名称を保持する静的テーブル。
  • MessageEncoding: 現在のメッセージエンコーディング情報を保持するグローバル変数。

失敗モードと境界条件

  • PG_VALID_ENCODING チェックに失敗した場合、アサーションによりプロセスが停止します。
  • elog() が使用できない初期化段階でも呼び出される可能性があるため、エラー報告にはアサーションが使用されています。

保守担当者向けメモ

  • 本関数は非常に早い段階で呼び出される可能性があるため、複雑なロジックやメモリ割り当てを含めないようにしてください。
  • MessageEncoding の更新は、エラーメッセージ等の出力エンコーディングに直結するため、整合性に注意してください。
