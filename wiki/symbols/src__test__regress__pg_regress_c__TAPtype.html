TAPtype

定義

  • 種別: enum
  • 定義箇所: src/test/regress/pg_regress.c:84

目的

PostgreSQL の回帰テスト実行時における、TAP (Test Anything Protocol) 形式の出力種別を識別するために使用されます。

シグネチャまたは形状

typedef enum TAPtype
{
	DIAG = 0,
	BAIL,
	NOTE,
	NOTE_DETAIL,
	NOTE_END,
	TEST_STATUS,
	PLAN,
	NONE,
} TAPtype;

入力と出力

  • 入力: なし（列挙型定数）
  • 出力: なし

前提条件と不変条件

  • emit_tap_output および emit_tap_output_v 関数において、出力フォーマットを決定するための識別子として利用されます。

アルゴリズムまたは振る舞い

TAP プロトコルの各要素（診断メッセージ、テスト結果、計画など）を論理的に分類します。各列挙値は、テストドライバが生成する出力の性質を規定します。

呼び出し元と呼び出し先

  • 呼び出し元: emit_tap_output および emit_tap_output_v の引数として使用されます。
  • 呼び出し先: 情報なし。

関連データ構造

  • emit_tap_output (関数)
  • emit_tap_output_v (関数)

失敗モードと境界条件

  • 列挙型定義外の値が渡された場合の動作については、ソースコード内に明示的なガード条件は確認できません。

保守担当者向けメモ

  • TAP 仕様の変更に伴い新しい出力タイプが必要になった場合は、この列挙型に値を追加し、emit_tap_output 側の処理を更新してください。
