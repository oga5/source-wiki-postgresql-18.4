InitBufferTag

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/buf_internals.h:156

目的

共有バッファ内のページを一意に識別するための BufferTag 構造体を初期化します。

シグネチャまたは形状

static inline void InitBufferTag(BufferTag *tag, const RelFileLocator *rlocator, ForkNumber forkNum, BlockNumber blockNum)

入力と出力

  • **入力**:
  • tag: 初期化対象の BufferTag 構造体へのポインタ。
  • rlocator: リレーションを特定するための RelFileLocator 構造体。
  • forkNum: 対象のフォーク番号（ForkNumber）。
  • blockNum: 対象のブロック番号（BlockNumber）。
  • **出力**: なし（tag が指すメモリ領域を直接更新）。

前提条件と不変条件

  • tag は有効な BufferTag 構造体を指している必要があります。
  • rlocator は有効なリレーション識別子を含んでいる必要があります。

アルゴリズムまたは振る舞い

1. rlocator から spcOid および dbOid を tag にコピーします。 2. BufTagSetRelForkDetails を呼び出し、リレーション番号とフォーク番号を tag に設定します。 3. blockNum を tag->blockNum に代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c (複数箇所)
  • src/backend/storage/buffer/localbuf.c (複数箇所)
  • **呼び出し先**:
  • BufTagSetRelForkDetails

関連データ構造

  • BufferTag: バッファ内のページを識別するタグ構造体。
  • RelFileLocator: リレーションの物理的な場所を特定する構造体。

失敗モードと境界条件

  • ポインタ tag または rlocator が NULL の場合、未定義の動作となります。

保守担当者向けメモ

  • BufferTag 構造体にパディングバイトが含まれる場合、本関数による初期化がそれらを適切に処理しているか確認してください。
  • BufTagSetRelForkDetails の実装変更が本関数の動作に影響を与える可能性があります。
