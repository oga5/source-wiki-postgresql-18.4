LockWaitPolicy

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/lockoptions.h:36

目的

行レベルのロック取得時における競合発生時の挙動（待機、スキップ、エラー）を制御するためのポリシーを定義する。

シグネチャまたは形状

typedef enum LockWaitPolicy
{
    LockWaitBlock,
    LockWaitSkip,
    LockWaitError,
} LockWaitPolicy;

入力と出力

  • 本シンボルは列挙型であり、関数への引数として渡されることでロック処理の振る舞いを決定する。
  • 出力値は LockWaitBlock（デフォルト）、LockWaitSkip（SKIP LOCKED）、LockWaitError（NOWAIT）のいずれかをとる。

前提条件と不変条件

  • LockWaitBlock がデフォルトの挙動として扱われる。
  • 各列挙値は、SQLレベルのロック指定（NOWAIT や SKIP LOCKED）と一対一で対応する。

アルゴリズムまたは振る舞い

  • LockWaitBlock: ロックが利用可能になるまで待機する。
  • LockWaitSkip: ロックが取得できない行をスキップする。
  • LockWaitError: ロックが取得できない場合にエラーを発生させる。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/parser/analyze.c
  • src/backend/rewrite/rewriteHandler.c
  • src/include/access/heapam.h
  • src/include/access/tableam.h
  • src/include/nodes/execnodes.h
  • 呼び出し先: 本シンボル自体は列挙型定義であり、関数呼び出しを行わない。

関連データ構造

  • LockClauseStrength: ロックの強度を定義する関連列挙型。
  • RowMarkClause (src/include/nodes/parsenodes.h): 本ポリシーを保持する構造体。
  • PlanRowMark (src/include/nodes/plannodes.h): 実行計画におけるロック指定保持に使用。

失敗モードと境界条件

  • LockWaitError が指定された状態でロック競合が発生した場合、呼び出し側の処理フローにおいてエラーが送出される。

保守担当者向けメモ

  • 新しい待機ポリシーを追加する場合は、既存の heapam や tableam の各関数におけるスイッチ文や条件分岐が網羅されているか確認すること。
