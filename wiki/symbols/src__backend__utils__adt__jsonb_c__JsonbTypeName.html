JsonbTypeName

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb.c:179

目的

JsonbValue 構造体が保持するデータ型を、SQL レベルで識別可能な文字列（型名）として返します。

シグネチャまたは形状

const char *JsonbTypeName(JsonbValue *val)

入力と出力

  • **入力**: JsonbValue *val - 型を判定する対象の JSONB 値。
  • **出力**: const char * - 型名を表す文字列リテラル（例: "object", "number", "date" など）。

前提条件と不変条件

  • val は有効な JsonbValue ポインタである必要があります。
  • val->type が jbvDatetime の場合、val->val.datetime.typid に対応する OID が定義されている必要があります。

アルゴリズムまたは振る舞い

1. val->type を評価し、対応する型名を返します。 2. jbvBinary の場合は JsonbContainerTypeName を呼び出し、その結果を委譲します。 3. jbvDatetime の場合は、内部の typid を参照し、DATEOID や TIMESTAMPOID 等の PostgreSQL 型 OID に基づいて具体的な日時型名を返します。 4. 認識できない型や日時型の場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonb.c:164
  • src/backend/utils/adt/jsonpath_exec.c:1093
  • **呼び出し先**:
  • JsonbContainerTypeName (型が jbvBinary の場合)
  • elog (エラー発生時)

関連データ構造

  • JsonbValue: JSONB の各ノードを表現する共用体。
  • JsonbContainerTypeName: jbvBinary 型の内部構造を解析するために使用される関数。

失敗モードと境界条件

  • val->type が未知の値である場合、elog(ERROR) により処理が中断されます。
  • val->type が jbvDatetime であり、かつ typid がサポートされていない OID である場合、elog(ERROR) が発生します。

保守担当者向けメモ

  • 新しい JSONB 型や日時型が追加された場合は、本関数の switch 文を更新する必要があります。
  • jbvBinary の判定は JsonbContainerTypeName に依存しているため、バイナリ表現の変更時は両方の関数を確認してください。
