get_template0_info

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/info.c:338

目的

指定された PostgreSQL クラスタから template0 データベースのロケールおよびエンコーディング情報を取得し、ClusterInfo 構造体に格納する。

シグネチャまたは形状

static void get_template0_info(ClusterInfo *cluster);

入力と出力

  • **入力**: ClusterInfo *cluster（対象クラスタ情報へのポインタ）
  • **出力**: なし（副作用として cluster->template0 に DbLocaleInfo 構造体が割り当てられ、値が設定される）

前提条件と不変条件

  • cluster 引数は有効な ClusterInfo 構造体を指していること。
  • connectToServer 関数により template1 への接続が確立可能であること。
  • ターゲットクラスタ内に datname='template0' を持つレコードが必ず1件存在すること。

アルゴリズムまたは振る舞い

1. connectToServer を使用して template1 データベースに接続する。 2. クラスタのメジャーバージョンに基づき、pg_catalog.pg_database から template0 の情報を取得するクエリを分岐させる。

  • バージョン 1700 以上: datlocale を直接取得。
  • バージョン 1500 以上: daticulocale を datlocale として取得。
  • それ以外: datlocale を NULL として取得。

    • 3. クエリ結果が1件であることを確認し、取得した値を DbLocaleInfo 構造体にコピーする。 4. cluster->template0 に構造体を格納する。 5. データベース接続を閉じ、クエリ結果を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/info.c 内の関数（get_db_rel_and_slot_infos 等から呼び出される）。
  • **呼び出し先**: connectToServer, executeQueryOrDie, PQntuples, pg_malloc, PQfnumber, atoi, PQgetvalue, pg_strdup, PQgetisnull, PQclear, PQfinish。

関連データ構造

  • ClusterInfo: クラスタ全体の情報を保持する構造体。
  • DbLocaleInfo: データベースのロケール設定（エンコーディング、照合順序、文字分類等）を保持する構造体。

失敗モードと境界条件

  • PQntuples(dbres) != 1: template0 が見つからない場合、pg_fatal を呼び出しプロセスを終了する。
  • メモリ不足: pg_malloc が失敗した場合、内部的にエラー終了する可能性がある。

保守担当者向けメモ

  • PostgreSQL のバージョンアップに伴い、pg_database のカラム構成が変更される可能性があるため、クエリの条件分岐（GET_MAJOR_VERSION）のメンテナンスが必要。
  • pg_strdup で割り当てられたメモリは、呼び出し元またはクラスタのクリーンアップ処理で適切に解放される必要がある。
