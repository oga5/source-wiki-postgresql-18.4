SQLFunctionHashEntry

定義

  • **種別**: struct
  • **定義箇所**: src/backend/executor/functions.c:112

目的

SQL言語で記述された関数の実行に必要な情報を保持する、長寿命なキャッシュ構造体です。複数の関数実行間で共有され、関数の解析結果や計画を再利用可能にします。

シグネチャまたは形状

typedef struct SQLFunctionHashEntry {
    CachedFunction cfunc;
    char *fname;
    char *src;
    SQLFunctionParseInfoPtr pinfo;
    int16 *argtyplen;
    Oid rettype;
    int16 typlen;
    bool typbyval;
    bool returnsSet;
    bool returnsTuple;
    bool readonly_func;
    char prokind;
    TupleDesc rettupdesc;
    List *source_list;
    int num_queries;
    bool raw_source;
    List *plansource_list;
    MemoryContext pcontext;
    MemoryContext hcontext;
} SQLFunctionHashEntry;

入力と出力

本構造体は関数実行のキャッシュエントリとして機能します。

  • **入力**: pg_proc から読み込まれた関数定義（ソースリスト、型情報など）。
  • **出力**: 実行エンジンが利用する CachedPlanSource のリスト（plansource_list）および結果の型情報。

前提条件と不変条件

  • pcontext は source_list を保持し、hcontext はそれ以外の構造体メンバを保持するメモリコンテキストとして機能します。
  • cfunc フィールドは funccache.c によって管理されます。

アルゴリズムまたは振る舞い

  • 関数実行時に init_sql_fcache を通じて初期化され、fmgr_sql の実行ループ内で参照されます。
  • prepare_next_query を呼び出すことで、関数内の各文を CachedPlanSource に変換し、実行可能な状態にします。
  • raw_source が真の場合、source_list には解析前の RawStmt が含まれており、必要に応じて計画化が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: fmgr_sql（実行エントリポイント）、init_sql_fcache、prepare_next_query など。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しを行いません。
  • **注記**: 呼び出し元・呼び出し先の詳細なコールグラフは、提供されたソース抜粋からは網羅的に特定できません。

関連データ構造

  • CachedFunction: funccache.c で管理される基底構造。
  • SQLFunctionParseInfoPtr: パーサーフック用のデータ。
  • CachedPlanSource: plansource_list に格納されるクエリ計画のソース。
  • TupleDesc: 関数の戻り値のタプル記述子。

失敗モードと境界条件

  • メモリコンテキストの管理不備により、長寿命キャッシュとしてのリークが発生する可能性があります。
  • source_list の解析中にエラーが発生した場合、sql_exec_error_callback を介したエラーコンテキストの報告が行われます。

保守担当者向けメモ

  • 本構造体はセッション寿命を持つ可能性があるため、リソース解放時には RemoveSQLFunctionCache 等の適切なクリーンアップ関数を介して参照カウントを管理してください。
  • lazyEval フラグとの関連性に注意し、セットを返す関数の実行状態が正しく維持されているか確認してください。
