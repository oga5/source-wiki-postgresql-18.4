expanded_record_get_tupdesc

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/expandedrecord.h:217

目的

展開されたレコード（ExpandedRecordHeader）に関連付けられたタプル記述子（TupleDesc）を取得します。

シグネチャまたは形状

TupleDesc expanded_record_get_tupdesc(ExpandedRecordHeader *erh)

入力と出力

  • **入力**: ExpandedRecordHeader *erh（対象の展開されたレコードへのポインタ）
  • **出力**: TupleDesc（レコードの型定義情報）

前提条件と不変条件

  • erh は有効な ExpandedRecordHeader を指している必要があります。
  • er_tupdesc がキャッシュされていない場合、expanded_record_fetch_tupdesc を呼び出して取得を試みます。

アルゴリズムまたは振る舞い

1. erh->er_tupdesc が NULL でないかを確認します。 2. キャッシュが存在する場合は、即座にその TupleDesc を返します。 3. キャッシュが存在しない場合は、expanded_record_fetch_tupdesc(erh) を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c
  • src/backend/utils/adt/expandedrecord.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • expanded_record_fetch_tupdesc（キャッシュミス時）

関連データ構造

  • ExpandedRecordHeader: 展開されたレコードのメイン構造体。
  • TupleDesc: タプルの構造を記述するデータ構造。

失敗モードと境界条件

  • erh が NULL の場合、未定義の動作となります。
  • expanded_record_fetch_tupdesc が内部的に失敗した場合、その関数に依存するエラーが発生します。

保守担当者向けメモ

  • 本関数はインライン化されており、頻繁に呼び出されるパスでのパフォーマンスを最適化しています。
  • expanded_record_fetch_tupdesc は本関数のアウトライン部分として機能します。
