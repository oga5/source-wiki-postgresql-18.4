subxids_array_status

定義

  • 種別: enum
  • 定義箇所: src/include/storage/standby.h:78

目的

ホットスタンバイ環境において、サブトランザクションID（SubXID）の配列が保持する情報の完全性や、その状態を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
    SUBXIDS_IN_ARRAY,
    SUBXIDS_MISSING,
    SUBXIDS_IN_SUBTRANS,
} subxids_array_status;

入力と出力

  • 本シンボルは列挙型であり、関数への入力や戻り値として、現在のサブトランザクション情報の管理状態を伝達するために利用されます。

前提条件と不変条件

  • スタンバイサーバーがプライマリから受け取ったトランザクション状態（RunningTransactionsData）を解釈する際の判断基準となります。

アルゴリズムまたは振る舞い

  • SUBXIDS_IN_ARRAY: 配列内に実行中のすべてのサブトランザクションIDが含まれている状態。
  • SUBXIDS_MISSING: スナップショットのオーバーフロー等により、サブトランザクションIDが欠落している状態。
  • SUBXIDS_IN_SUBTRANS: 配列にはサブトランザクションIDが含まれていないが、pg_subtrans が完全に最新の状態であり、そこから情報を取得可能な状態。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース抜粋には含まれていません。

関連データ構造

  • RunningTransactionsData: 本列挙型をメンバとして保持し、スタンバイ起動時のトランザクション状態を管理します。

失敗モードと境界条件

  • SUBXIDS_MISSING は、スナップショットのオーバーフローが発生した際の境界条件を示します。

保守担当者向けメモ

  • 本シンボルは src/include/storage/standby.h 内の RunningTransactionsData 構造体と密接に関連しています。
  • 競合解決やリカバリ処理のロジックを変更する際は、各列挙値が示す「情報の所在」の定義が正しく反映されているか確認してください。
