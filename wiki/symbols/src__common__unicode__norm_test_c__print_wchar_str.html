print_wchar_str

定義

  • 種別: function
  • 定義箇所: src/common/unicode/norm_test.c:22

目的

pg_wchar 型の文字列を、人間が読みやすい U+XXXX 形式の 16 進数表記文字列に変換して出力するためのデバッグ用補助関数です。

シグネチャまたは形状

static char *print_wchar_str(const pg_wchar *s)

入力と出力

  • **入力**: const pg_wchar *s（変換対象の pg_wchar 文字列へのポインタ）
  • **出力**: char *（U+XXXX 形式でフォーマットされた静的バッファへのポインタ）

前提条件と不変条件

  • 本関数は静的バッファを再利用するため、スレッドセーフではありません。
  • テストプログラム内でのシングルスレッド実行を前提としています。

アルゴリズムまたは振る舞い

1. 最大 50 文字（BUF_DIGITS）まで処理をループします。 2. 各 pg_wchar 文字を sprintf を用いて U+XXXX 形式の文字列に変換し、静的バッファ buf に書き込みます。 3. 文字列の終端に達するか、50 文字に達した時点でループを終了します。 4. バッファの末尾にヌル終端文字を付与し、バッファの先頭アドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/unicode/norm_test.c 内の main 関数。テスト失敗時に期待値と実際の値を表示するために呼び出されます。
  • **呼び出し先**: 標準ライブラリの sprintf。

関連データ構造

  • pg_wchar: PostgreSQL で使用されるワイド文字型。

失敗モードと境界条件

  • 入力文字列が 50 文字を超える場合、51 文字目以降は切り捨てられます。
  • 静的バッファを使用しているため、同一スコープ内で本関数を複数回呼び出した場合、以前の戻り値の内容は上書きされます。

保守担当者向けメモ

  • 本関数はテストツール専用です。本番コードへの流用は避けてください。
  • バッファサイズ（BUF_DIGITS）を変更する場合は、静的バッファ buf のサイズ計算式も併せて調整する必要があります。
