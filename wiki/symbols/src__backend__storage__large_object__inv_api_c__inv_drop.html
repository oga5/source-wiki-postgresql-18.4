inv_drop

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:310

目的

指定されたOIDを持つラージオブジェクト（Large Object）をシステムから削除します。これには、関連するコメントや依存関係の削除が含まれます。

シグネチャまたは形状

int inv_drop(Oid lobjId)

入力と出力

  • **入力**: lobjId (削除対象のラージオブジェクトのOID)
  • **出力**: 成功時に 1 を返します（歴史的な理由による仕様）。

前提条件と不変条件

  • 呼び出し元は、対象のラージオブジェクトに対する適切な権限を持っている必要があります。
  • performDeletion を介して削除処理が行われるため、依存関係の整合性が維持されます。

アルゴリズムまたは振る舞い

1. ObjectAddress 構造体を初期化し、対象のラージオブジェクト（LargeObjectRelationId）を指定します。 2. performDeletion を呼び出し、DROP_CASCADE モードでオブジェクトおよび関連する依存関係を削除します。 3. CommandCounterIncrement() を呼び出し、削除操作を現在のトランザクション内の後続の操作から可視化します。 4. 成功値として 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-fsstubs.c（SQLインターフェース経由の呼び出し）
  • **呼び出し先**: performDeletion（カタログ操作）、CommandCounterIncrement（トランザクション管理）

関連データ構造

  • ObjectAddress: 削除対象を特定するために使用される構造体。
  • LargeObjectRelationId: pg_largeobject カタログを指す識別子。

失敗モードと境界条件

  • 存在しないOIDを指定した場合、performDeletion 内部でエラーが発生する可能性があります。
  • 権限不足の場合、performDeletion が例外をスローします。

保守担当者向けメモ

  • 本関数は歴史的な経緯により、成功時に常に 1 を返す仕様となっています。
  • CommandCounterIncrement() の呼び出しは、削除直後の操作で当該オブジェクトが「存在しない」と正しく認識されるために不可欠です。
