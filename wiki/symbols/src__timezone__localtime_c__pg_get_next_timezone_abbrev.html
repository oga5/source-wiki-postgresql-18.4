pg_get_next_timezone_abbrev

定義

  • 種別: function
  • 定義箇所: src/timezone/localtime.c:1935

目的

指定されたタイムゾーン構造体に含まれるタイムゾーン略称（abbreviation）を順次取得する。

シグネチャまたは形状

const char *pg_get_next_timezone_abbrev(int *indx, const pg_tz *tz);

入力と出力

  • **入力**:
  • indx: 略称の読み取り開始位置を示すインデックスへのポインタ。
  • tz: タイムゾーン情報を保持する pg_tz 構造体へのポインタ。
  • **出力**:
  • 戻り値: 現在のインデックスが指す略称文字列へのポインタ。範囲外の場合は NULL。
  • indx: 次の略称の開始位置へ更新される。

前提条件と不変条件

  • tz は有効な pg_tz 構造体を指している必要がある。
  • indx は、呼び出し前に初期化（通常は 0）されている必要がある。
  • tz->state.chars には、複数のヌル終端文字列が連続して格納されている必要がある。

アルゴリズムまたは振る舞い

1. tz->state 内の略称格納領域（chars）と現在のインデックス *indx を参照する。 2. *indx が 0 未満、または sp->charcnt（略称領域の総サイズ）以上であれば NULL を返す。 3. 現在の *indx 位置にある文字列へのポインタを result に保持する。 4. chars 配列内で現在の文字列の終端（\0）まで *indx を進め、さらに次の文字へ進めることで、次の略称の開始位置を計算して *indx を更新する。 5. result を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c:5170
  • **呼び出し先**: 本関数は直接的な関数呼び出しを行わず、pg_tz 構造体のメンバを参照する。

関連データ構造

  • pg_tz: タイムゾーン情報を保持する構造体。内部に state を持つ。
  • state: タイムゾーンの遷移ルールや略称情報を保持する構造体。

失敗モードと境界条件

  • *indx が sp->charcnt に達している場合、NULL を返し、それ以上の略称取得は行われない。
  • *indx が負の値である場合、NULL を返す。

保守担当者向けメモ

  • 本関数は state->chars 内のメモリレイアウト（ヌル終端文字列の連続）に強く依存している。tzloadbody 等でこの構造が変更される場合は注意が必要である。
