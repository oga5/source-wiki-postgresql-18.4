randomize_mem

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/memdebug.c:74

目的

メモリ確保領域を疑似乱数で埋めることで、初期化漏れによる依存関係を顕在化させ、メモリ管理のバグを診断するためのデバッグ支援機能を提供します。

シグネチャまたは形状

void randomize_mem(char *ptr, size_t size)

入力と出力

  • ptr: 疑似乱数で埋める対象メモリ領域の先頭ポインタ。
  • size: 対象領域のサイズ（バイト単位）。
  • 出力: なし（副作用としてメモリ内容を書き換える）。

前提条件と不変条件

  • RANDOMIZE_ALLOCATED_MEMORY が有効なビルド環境でのみ使用されます。
  • 内部的に static int save_ctr を保持しており、呼び出し間で疑似乱数の状態を維持します。

アルゴリズムまたは振る舞い

1. Valgrind に対して対象領域を UNDEFINED（アクセス可能だが未定義）として通知します。 2. save_ctr から開始する 1 から 251 までの循環する疑似乱数列を生成し、対象領域の各バイトに書き込みます。 3. 書き込み完了後、再度 Valgrind に対して対象領域を UNDEFINED としてマークします。 4. 次回の呼び出しのために、現在のカウンタ値を save_ctr に保存します。

呼び出し元と呼び出し先

  • 呼び出し元: aset.c、bump.c、generation.c 内の各アロケータが、メモリ確保時や再確保時に呼び出します。
  • 呼び出し先: Valgrind クライアントリクエストマクロ（VALGRIND_MAKE_MEM_UNDEFINED）。

関連データ構造

  • 特になし（静的変数 save_ctr を使用）。

失敗モードと境界条件

  • size が 0 の場合、ループは実行されず、Valgrind への通知のみが行われます。
  • 疑似乱数のカウンタは 251 を超えると 1 にリセットされます。

保守担当者向けメモ

  • 本機能は開発用であり、本番環境では無効化されることを前提としています。
  • Valgrind 連携は、メモリのアクセス権限を動的に制御することで、メモリ管理のバグを検出する仕組みです。変更を加える際は memdebug.h の定義と整合性を確認してください。
