SimpleStringListCell

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/simple_list.h:32

目的

PostgreSQL のフロントエンドツールにおいて、文字列を格納するための単方向連結リストの各要素（セル）を定義します。

シグネチャまたは形状

typedef struct SimpleStringListCell
{
    struct SimpleStringListCell *next;
    bool        touched;
    char        val[FLEXIBLE_ARRAY_MEMBER];
} SimpleStringListCell;

入力と出力

  • next: リスト内の次のセルへのポインタ。末尾の場合は NULL。
  • touched: 検索操作によって要素が参照されたかを示すフラグ。
  • val: 格納されるヌル終端文字列。FLEXIBLE_ARRAY_MEMBER を使用してインラインで保持されます。

前提条件と不変条件

  • val に格納される文字列はヌル終端である必要があります。
  • メモリは pg_malloc 等で確保されるため、リストの破棄時には対応する destroy 関数を呼び出し、メモリリークを防ぐ必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、SimpleStringList の構成要素として機能します。
  • simple_string_list_member 関数による検索時、一致した要素の touched フラグが true に更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のソースファイルでリストの走査や参照が行われています。
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_dump/pg_dump.c
  • src/bin/pg_dump/pg_dumpall.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • src/bin/scripts/clusterdb.c
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SimpleStringList: 本セルをリスト構造として管理する親構造体。

失敗モードと境界条件

  • リストの末尾において next が NULL であることを正しく処理する必要があります。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズは格納する文字列の長さに依存します。

保守担当者向けメモ

  • バックエンドの List 構造体とは異なり、非常に軽量な実装です。複雑な操作が必要な場合は別のデータ構造を検討してください。
  • touched フラグは、リスト内の全要素が処理されたかを確認する用途に最適化されています。
