RunObjectPostCreateHook

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:31

目的

データベースオブジェクトの作成完了時に、登録された外部フック（プラグイン等）へ通知を行うためのエントリーポイントを提供します。

シグネチャまたは形状

void RunObjectPostCreateHook(Oid classId, Oid objectId, int subId, bool is_internal)

入力と出力

  • **入力**:
  • classId (Oid): 作成されたオブジェクトが属するカタログのOID。
  • objectId (Oid): 作成されたオブジェクトのOID。
  • subId (int): サブオブジェクトID（属性番号など）。
  • is_internal (bool): 内部的な作成処理かどうかを示すフラグ。
  • **出力**: なし（void）。

前提条件と不変条件

  • グローバル変数 object_access_hook が非NULLである必要があります（関数内で Assert により検証）。

アルゴリズムまたは振る舞い

1. ObjectAccessPostCreate 構造体 pc_arg をスタック上に確保し、memset でゼロ初期化します。 2. 引数 is_internal を pc_arg.is_internal に代入します。 3. 登録されているフック関数 object_access_hook を呼び出し、イベントタイプ OAT_POST_CREATE、オブジェクト識別情報、および pc_arg へのポインタを渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は提供されていませんが、src/backend/catalog/objectaccess.h で宣言されており、コアコードの各所から参照されることが想定されています。
  • **呼び出し先**: グローバル関数ポインタ object_access_hook が指す関数（外部プラグイン等）。

関連データ構造

  • ObjectAccessPostCreate: フック関数に渡される引数構造体。
  • OAT_POST_CREATE: ObjectAccessType 列挙型の一部であり、作成後のフックであることを示す定数。

失敗モードと境界条件

  • object_access_hook が NULL の状態で呼び出された場合、アサーション違反によりプロセスが停止します。

保守担当者向けメモ

  • 本関数はフックの呼び出し口です。実際の処理はロードされた共有ライブラリ内の関数で行われます。
  • is_internal フラグは、監査ログやセキュリティ拡張がオブジェクトの作成元を判断するために使用されます。
