secure_loaded_verify_locations

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure.c:98

目的

SSL証明書の検証場所（CA証明書等）が正常に読み込まれているかどうかを判定する。

シグネチャまたは形状

bool secure_loaded_verify_locations(void);

入力と出力

  • 入力: なし
  • 出力: bool 型。検証場所が読み込まれていれば true、それ以外は false を返す。

前提条件と不変条件

  • USE_SSL が定義されている環境では、内部変数 ssl_loaded_verify_locations の状態に依存する。
  • USE_SSL が定義されていない環境では、常に false を返す。

アルゴリズムまたは振る舞い

1. USE_SSL マクロの定義有無を確認する。 2. 定義されている場合、SSL検証場所の読み込み状態を示す ssl_loaded_verify_locations の値を返す。 3. 定義されていない場合、定数 false を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/libpq/auth.c:402
  • 呼び出し先:
  • 特になし（直接的な関数呼び出しは行わない）

関連データ構造

  • ssl_loaded_verify_locations (グローバル変数、USE_SSL 定義時のみ)

失敗モードと境界条件

  • USE_SSL が無効なビルドでは、検証場所の有無に関わらず常に false を返す。

保守担当者向けメモ

  • 本関数は libpq/libpq.h で公開されているインターフェースである。
  • SSL関連の検証ロジックを変更する際は、ssl_loaded_verify_locations が適切に更新されていることを確認すること。
