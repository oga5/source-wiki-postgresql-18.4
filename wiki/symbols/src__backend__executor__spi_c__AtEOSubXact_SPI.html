AtEOSubXact_SPI

定義

  • 種別: function
  • 定義箇所: src/backend/executor/spi.c:481

目的

サブトランザクションの終了（コミットまたはアボート）時に、SPI（Server Programming Interface）の実行コンテキストとリソースをクリーンアップし、スタックの整合性を維持する。

シグネチャまたは形状

void AtEOSubXact_SPI(bool isCommit, SubTransactionId mySubid)

入力と出力

  • isCommit (bool): true の場合はサブトランザクションのコミット時、false の場合はアボート時の処理を行う。
  • mySubid (SubTransactionId): 終了対象となるサブトランザクションの ID。
  • 出力: なし（副作用として SPI スタックおよびメモリコンテキストを操作する）。

前提条件と不変条件

  • _SPI_stack が有効な SPI 接続状態を保持していること。
  • isCommit が true の場合、SPI スタックが空でないままサブトランザクションが終了すると警告（WARNING）が発行される。

アルゴリズムまたは振る舞い

1. **スタックのクリーンアップ**: _SPI_connected から順に、現在のサブトランザクション ID (mySubid) に一致する SPI 接続をスタックから探索する。 2. **リソース解放**: 一致した接続に対し、execCxt および procCxt メモリコンテキストを削除し、グローバル変数（SPI_processed 等）をスタック上の保存値で復元する。 3. **アボート時の追加処理**: isCommit が false の場合、現在の SPI コンテキストにおいて、指定された mySubid 以降に作成された実行状態（execCxt のリセット）およびタプルテーブル（SPITupleTable）を破棄し、メモリリークを防止する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c（サブトランザクションのコミットおよびアボート処理の過程で呼び出される）。
  • 呼び出し先: MemoryContextDelete, MemoryContextReset, ereport 等のメモリ管理およびエラー報告関数。

関連データ構造

  • _SPI_stack: SPI 接続のネストを管理する内部スタック。
  • _SPI_connection: 各接続のコンテキスト情報を保持する構造体。
  • SPITupleTable: SPI 経由で返されるタプルセットを管理する構造体。

失敗モードと境界条件

  • サブトランザクション終了時に SPI スタックが正しくクリーンアップされていない場合、WARNING が発生する。
  • アボート時に _SPI_current が存在する場合、関連するメモリコンテキストやタプルテーブルが強制的に解放される。

保守担当者向けメモ

  • 本関数は SPI_finish() と異なり、既に解放されている可能性のあるメモリコンテキストへの切り替えを行わずにスタックを操作する設計となっている。
  • タプルテーブルの解放において、SPI_freetuptable() を使用すると O(N^2) の計算量になるため、手動でリストを走査して削除する実装が採用されている。
