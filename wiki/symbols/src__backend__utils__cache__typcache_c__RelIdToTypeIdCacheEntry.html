RelIdToTypeIdCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/typcache.c:89

目的

リレーション（テーブル等）の OID と、それに対応する複合型の OID との間のマッピングをキャッシュし、システムカタログへの繰り返しアクセスを回避するために使用されます。

シグネチャまたは形状

typedef struct RelIdToTypeIdCacheEntry
{
    Oid         relid;          /* リレーションのOID */
    Oid         composite_typid; /* リレーションの複合型のOID */
} RelIdToTypeIdCacheEntry;

入力と出力

本構造体はハッシュテーブルの要素として機能します。

  • 入力: relid をキーとして検索に使用。
  • 出力: 対応する composite_typid を取得可能。

前提条件と不変条件

  • RelIdToTypeIdCacheHash ハッシュテーブル内で管理されることを前提としています。
  • relid は一意のキーとして機能します。

アルゴリズムまたは振る舞い

  • hash_search を用いて RelIdToTypeIdCacheHash を検索し、キャッシュヒットした場合は既存の composite_typid を返します。
  • キャッシュミス時には、システムカタログから情報を取得し、本構造体を介してハッシュテーブルに格納します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/typcache.c 内の関数（hash_search を呼び出す箇所）から参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • RelIdToTypeIdCacheHash: 本構造体を格納するハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時: hash_search が失敗する可能性があります。
  • キャッシュ無効化: カタログ更新時に RelIdToTypeIdCacheHash が適切にクリアされない場合、古い OID マッピングを参照するリスクがあります。

保守担当者向けメモ

  • 本構造体は typcache.c 内の RelIdToTypeIdCacheHash のエントリサイズ定義（sizeof(RelIdToTypeIdCacheEntry)）に使用されています。
  • 構造体定義を変更する場合は、ハッシュテーブルの初期化および検索ロジックとの整合性に注意してください。
