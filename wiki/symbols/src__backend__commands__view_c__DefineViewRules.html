DefineViewRules

定義

  • 種別: function
  • 定義箇所: src/backend/commands/view.c:331

目的

ビューの定義クエリをルールシステムに登録し、ON SELECT ルールとして永続化する。

シグネチャまたは形状

static void DefineViewRules(Oid viewOid, Query *viewParse, bool replace)

入力と出力

  • viewOid: 対象ビューの OID。
  • viewParse: ビュー定義の解析済み Query 構造体。
  • replace: CREATE OR REPLACE VIEW の場合は true、それ以外は false。
  • 出力: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • viewParse は既に解析済み（parse_analyze_fixedparams 適用済み）であること。

アルゴリズムまたは振る舞い

1. DefineQueryRewrite を呼び出し、ViewSelectRuleName をルール名として ON SELECT ルールを登録する。 2. replace フラグを DefineQueryRewrite に渡し、既存ルールの置換または新規作成を制御する。 3. 現在のところ ON INSERT 等の自動ルール生成は行わない。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/view.c 内のビュー作成処理（DefineView 等）。
  • 呼び出し先: DefineQueryRewrite（ルールシステムへの登録）。

関連データ構造

  • Query: ビューの定義クエリを保持する構造体。
  • Oid: ビューを識別するオブジェクト ID。

失敗モードと境界条件

  • DefineQueryRewrite 内部でのカタログ操作に伴うエラー（権限不足やロック競合など）が発生する可能性がある。

保守担当者向けメモ

  • 本関数は ON SELECT ルールの登録に特化している。将来的に ON INSERT などの自動ルール生成を実装する際は、この関数を拡張する。
