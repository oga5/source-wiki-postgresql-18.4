pg_get_wal_records_info

定義

  • 種別: function
  • 定義箇所: contrib/pg_walinspect/pg_walinspect.c:591

目的

指定されたLSN範囲内のWALレコード情報を取得し、SQLインターフェースを通じてテーブル形式で出力するためのSRF（Set Returning Function）のエントリポイントです。

シグネチャまたは形状

Datum pg_get_wal_records_info(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • start_lsn (第1引数): 読み取りを開始するWALのLSN。
  • end_lsn (第2引数): 読み取りを終了するWALのLSN。
  • **出力**: PG_RETURN_VOID() を返しますが、内部で GetWALRecordsInfo を呼び出し、SRFとして結果セットを構築・返却します。

前提条件と不変条件

  • 入力された start_lsn および end_lsn は、ValidateInputLSNs によって有効性が検証される必要があります。
  • 呼び出し元はPostgreSQLの関数呼び出し規約に従う必要があります。

アルゴリズムまたは振る舞い

1. 引数から start_lsn と end_lsn を取得します。 2. ValidateInputLSNs を呼び出し、入力されたLSNの妥当性を確認・調整します。 3. GetWALRecordsInfo を呼び出し、指定範囲のWALレコードを走査・解析して結果を生成します。 4. 処理完了後、PG_RETURN_VOID() を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • PG_FUNCTION_INFO_V1(pg_get_wal_records_info) マクロによる登録（contrib/pg_walinspect/pg_walinspect.c:39）。
  • **呼び出し先**:
  • ValidateInputLSNs
  • GetWALRecordsInfo

関連データ構造

  • XLogRecPtr (LSNの型)
  • FunctionCallInfo (PG_FUNCTION_ARGS 経由)

失敗モードと境界条件

  • ValidateInputLSNs による検証で無効なLSNが指定された場合、エラーが発生する可能性があります。
  • WALの読み取り中にエラーが発生した場合、xlogreader のメカニズムにより例外がスローされます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のロジックは GetWALRecordsInfo に依存しています。
  • SRFとしての実装であるため、メモリ管理（MemoryContext の切り替え等）は GetWALRecordsInfo 内部で行われる設計となっています。
