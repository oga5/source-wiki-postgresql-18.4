gnuish_strerror_r

定義

  • 種別: function
  • 定義箇所: src/port/strerror.c:84

目的

プラットフォーム間で異なる strerror_r() の API 仕様（POSIX版とGNU版）および strerror() の有無を抽象化し、一貫したエラーメッセージ取得手段を提供します。

シグネチャまたは形状

static char *gnuish_strerror_r(int errnum, char *buf, size_t buflen);

入力と出力

  • **入力**:
  • errnum: エラー番号（errno）。
  • buf: エラーメッセージを格納する呼び出し元提供のバッファ。
  • buflen: buf のサイズ。
  • **出力**:
  • 成功時: エラーメッセージが格納された buf へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • buf は少なくとも buflen バイトの書き込み可能な領域である必要があります。
  • HAVE_STRERROR_R が定義されていない環境では、標準の strerror() を使用するため、スレッドセーフではない可能性があります。

アルゴリズムまたは振る舞い

1. **POSIX API (HAVE_STRERROR_R かつ STRERROR_R_INT)**: strerror_r を呼び出し、戻り値が 0 であれば buf を返します。失敗時は NULL を返します。 2. **GNU API (HAVE_STRERROR_R かつ !STRERROR_R_INT)**: strerror_r の戻り値をそのまま返します。 3. **フォールバック (!HAVE_STRERROR_R)**: strerror(errnum) を呼び出し、結果が NULL でなければ strlcpy を用いて buf にコピーし、buf を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/strerror.c 内の pg_strerror_r から呼び出されます。
  • **呼び出し先**: プラットフォーム依存の strerror_r または strerror を呼び出します。

関連データ構造

  • c.h: PG_STRERROR_R_BUFLEN などの定数定義が含まれます。

失敗モードと境界条件

  • strerror_r がエラーを返した場合、または strerror が NULL を返した場合は NULL を返します。
  • strlcpy を使用しているため、buflen を超えるメッセージは切り詰められます。

保守担当者向けメモ

  • 本関数は pg_strerror_r の内部ヘルパーとして機能します。
  • GNU 版 strerror_r は必ずしも buf を使用するとは限らないため、呼び出し元は戻り値のポインタを信頼する必要があります。
