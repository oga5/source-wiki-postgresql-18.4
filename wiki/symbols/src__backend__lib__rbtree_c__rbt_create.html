rbt_create

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:101

目的

空の赤黒木（RBTree）を初期化し、メモリ上に生成します。

シグネチャまたは形状

RBTree *rbt_create(Size node_size,
                   rbt_comparator comparator,
                   rbt_combiner combiner,
                   rbt_allocfunc allocfunc,
                   rbt_freefunc freefunc,
                   void *arg);

入力と出力

  • **入力**:
  • node_size: 各ノードが占めるメモリサイズ。
  • comparator: キー比較用コールバック関数。
  • combiner: キー重複時の結合用コールバック関数。
  • allocfunc: ノードメモリ確保用コールバック関数。
  • freefunc: ノードメモリ解放用コールバック関数。
  • arg: コールバック関数に渡される任意のコンテキストデータ。
  • **出力**: 初期化された RBTree 構造体へのポインタを返します。

前提条件と不変条件

  • node_size は sizeof(RBTNode) より大きい必要があります（Assert により検証）。
  • 戻り値の RBTree は、ルートノードが RBTNIL（番兵）を指す空の状態として初期化されます。

アルゴリズムまたは振る舞い

1. palloc を使用して RBTree 構造体分のメモリを確保します。 2. 渡された各コールバック関数および node_size、arg を構造体の各メンバに代入します。 3. 木のルートを RBTNIL に設定し、初期化済みの構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbulk.c:115
  • src/test/modules/test_rbtree/test_rbtree.c:82
  • **呼び出し先**:
  • palloc（PostgreSQLのメモリ管理関数）

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTNode: 各ノードのヘッダー。
  • RBTNIL: 葉ノードを表す静的な番兵。

失敗モードと境界条件

  • node_size が sizeof(RBTNode) 以下の場合、アサーション違反により異常終了します。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は侵入型設計の基点となります。freefunc はノード自体のメモリ解放のみを想定しており、ノード内のサブデータに対する再帰的な解放は行わない点に注意してください。
