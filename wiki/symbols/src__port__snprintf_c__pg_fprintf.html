pg_fprintf

定義

  • 種別: function
  • 定義箇所: src/port/snprintf.c:263

目的

PostgreSQL 独自の printf ファミリー実装の一部として、指定されたファイルストリームへフォーマット済み文字列を出力する。OS 標準ライブラリに依存せず、ポータブルかつ POSIX 準拠のフォーマット機能を提供する。

シグネチャまたは形状

int pg_fprintf(FILE *stream, const char *fmt, ...);

入力と出力

  • **入力**:
  • stream: 出力先の FILE ポインタ。
  • fmt: フォーマット文字列。
  • ...: フォーマット指定子に対応する可変長引数。
  • **出力**:
  • 戻り値: 書き込まれたバイト数（エラー発生時は負の値）。

前提条件と不変条件

  • va_list を使用するため、呼び出し側は標準的な可変長引数の規約に従う必要がある。
  • src/port/snprintf.c 内で標準の printf 系関数は #undef されているため、本関数は内部で PostgreSQL 独自の pg_vfprintf を呼び出すことで再帰や標準ライブラリとの競合を回避している。

アルゴリズムまたは振る舞い

1. va_start マクロを使用して可変長引数リスト args を初期化する。 2. pg_vfprintf(stream, fmt, args) を呼び出し、実際のフォーマット処理とストリームへの書き込みを実行する。 3. va_end マクロを使用して引数リストを終了する。 4. pg_vfprintf が返した書き込みバイト数を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/port.h にて fprintf マクロとして定義されており、PostgreSQL 全体で標準の fprintf の代わりに使用される。
  • src/pl/plperl/plperl_system.h にも同様の定義が存在する。
  • **呼び出し先**:
  • pg_vfprintf (直接の内部呼び出し)

関連データ構造

  • FILE: 標準 C ライブラリのストリーム型。
  • va_list: 可変長引数を扱うための型。

失敗モードと境界条件

  • pg_vfprintf の実装に依存する。バッファオーバーランやフォーマットエラーが発生した場合、標準の snprintf 準拠の戻り値（書き込まれるべきだったバイト数など）を返す。

保守担当者向けメモ

  • 本関数は fprintf のラッパーである。動作の修正や機能追加が必要な場合は、バックエンドである dopr() 関数または pg_vfprintf を参照すること。
  • Windows 環境など、プラットフォーム固有のフォーマット差異を吸収するロジックは dopr() 内に集約されている。
