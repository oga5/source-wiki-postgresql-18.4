adjust_child_relids_multilevel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:611

目的

階層的な継承関係（パーティションテーブル等）において、親リレーションのIDセットを、指定された子リレーションのIDセットへ多段階に変換する。

シグネチャまたは形状

Relids adjust_child_relids_multilevel(PlannerInfo *root, Relids relids,
                                      RelOptInfo *childrel,
                                      RelOptInfo *parentrel)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • relids: 変換対象のリレーションIDセット（Bitmapset）。
  • childrel: 変換先となる子リレーションの RelOptInfo。
  • parentrel: 変換元となる親リレーションの RelOptInfo。
  • **出力**:
  • 変換後の Relids。親リレーションが含まれない場合は元のセットがそのまま返される。

前提条件と不変条件

  • childrel は parentrel の直接または間接的な子孫である必要がある。
  • childrel->parent が存在しない場合、childrel が parentrel と一致しない限りエラーとなる。

アルゴリズムまたは振る舞い

1. **早期リターン**: relids が parentrel->relids と重複しない場合、変換の必要がないため入力をそのまま返す。 2. **再帰的走査**: childrel->parent が parentrel と一致しない場合、childrel->parent を新たな親として再帰的に呼び出し、階層を遡る。 3. **変換実行**: 階層の末端（または対象レベル）に到達後、find_appinfos_by_relids を使用して現在の childrel に対応する AppendRelInfo 配列を取得する。 4. **ID置換**: adjust_child_relids を呼び出し、親のIDを子のIDへ置換する。 5. **メモリ解放**: 取得した appinfos 配列を pfree で解放し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/appendinfo.c (再帰呼び出し)
  • src/backend/optimizer/util/pathnode.c:4644
  • **呼び出し先**:
  • bms_overlap
  • adjust_child_relids_multilevel (再帰)
  • find_appinfos_by_relids
  • adjust_child_relids
  • pfree
  • elog (エラー時)

関連データ構造

  • Relids (Bitmapset)
  • RelOptInfo
  • AppendRelInfo
  • PlannerInfo

失敗モードと境界条件

  • childrel が parentrel の子孫関係にない場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は多段階の継承関係を扱うため、再帰的な構造となっている。
  • find_appinfos_by_relids が返す appinfos 配列は呼び出し側で解放する必要があるため、pfree を忘れないこと。
