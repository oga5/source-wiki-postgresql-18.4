ExecInitNestLoop

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeNestloop.c:261

目的

ネステッドループ結合を実行するための実行エンジンノード（NestLoopState）を初期化し、子プランの準備および結合に必要なメタデータを設定する。

シグネチャまたは形状

NestLoopState *ExecInitNestLoop(NestLoop *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の NestLoop プランノード。
  • estate: 実行状態を保持する EState。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの NestLoopState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと（Assert による検証）。
  • 結合タイプが JOIN_INNER, JOIN_SEMI, JOIN_LEFT, JOIN_ANTI のいずれかであること。

アルゴリズムまたは振る舞い

1. NestLoopState 構造体を割り当て、ExecNestLoop を実行関数として登録する。 2. 式評価コンテキスト（ExprContext）を作成する。 3. 子プラン（外部および内部）を ExecInitNode で初期化する。この際、外部から内部へパラメータが渡される場合は内部プランの EXEC_FLAG_REWIND を無効化する。 4. 結果タプルスロットと投影情報を初期化する。 5. 結合条件（joinqual）およびフィルタ条件（qual）を初期化する。 6. inner_unique または JOIN_SEMI の場合に single_match フラグを立てる。 7. LEFT JOIN または ANTI JOIN の場合、一致しない場合に備えて NULL タプル用スロットを準備する。 8. 外部タプルの再取得フラグ（nl_NeedNewOuter）を true に設定し、マッチング状態（nl_MatchedOuter）を false に初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode から呼び出される。
  • **呼び出し先**:
  • ExecInitNode (子プランの初期化)
  • ExecInitQual (条件式の初期化)
  • ExecInitResultTupleSlotTL, ExecAssignProjectionInfo (結果処理の初期化)
  • ExecInitNullTupleSlot (外部結合時のNULL補完用)

関連データ構造

  • NestLoopState: 実行時の状態を保持する構造体。
  • NestLoop: プランノード。
  • EState: 実行状態。

失敗モードと境界条件

  • 未知の結合タイプが指定された場合、elog(ERROR) により処理を中断する。
  • eflags にサポート外のフラグが含まれる場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 内部プランは外部タプルの変更ごとに再スキャンされるため、パラメータ依存がある場合は効率的なインデックススキャンが選択されるようプランナが設計されている必要がある。
  • nl_NeedNewOuter フラグは、メインループでのタプル取得制御の要となる。
