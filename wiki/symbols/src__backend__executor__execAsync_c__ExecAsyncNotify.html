ExecAsyncNotify

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAsync.c:87

目的

非同期実行イベント発生時に、対応するノードの処理を起動し、実行統計の計測および結果の応答処理を行うための通知ハンドラ。

シグネチャまたは形状

void ExecAsyncNotify(AsyncRequest *areq)

入力と出力

  • **入力**: AsyncRequest *areq - 非同期リクエストの状態を保持する構造体。
  • **出力**: なし（void）。副作用として、ノードの実行統計の更新および親ノードへの応答処理が行われる。

前提条件と不変条件

  • areq および areq->requestee が有効であること。
  • areq->requestee が非同期実行をサポートするノード型（現時点では T_ForeignScanState のみ）であること。

アルゴリズムまたは振る舞い

1. **計測開始**: areq->requestee->instrument が存在する場合、InstrStartNode を呼び出し実行時間を計測開始する。 2. **ディスパッチ**: nodeTag(areq->requestee) に基づき、ノード固有の通知関数（例: ExecAsyncForeignScanNotify）を呼び出す。 3. **応答処理**: ExecAsyncResponse(areq) を呼び出し、結果を待機していた親ノードへ制御を戻す。 4. **計測終了**: areq->requestee->instrument が存在する場合、InstrStopNode を呼び出し、タプルが空でない場合は 1.0、空の場合は 0.0 を加算して計測を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAppend.c 等の非同期実行を管理するノード。
  • **呼び出し先**:
  • ExecAsyncForeignScanNotify (ノード型が T_ForeignScanState の場合)
  • ExecAsyncResponse
  • InstrStartNode / InstrStopNode (計測用)
  • **注記**: 呼び出し元・呼び出し先の網羅的な関係性は、静的解析の範囲内では限定的です。

関連データ構造

  • AsyncRequest: 非同期リクエストのメタデータと結果を保持する構造体。
  • ForeignScanState: 非同期実行をサポートする具体的なノード型。

失敗モードと境界条件

  • **未対応ノード**: nodeTag が T_ForeignScanState 以外の場合、elog(ERROR) によりエラーが発生する。
  • **計測対象外**: instrument フィールドが NULL の場合、計測処理はスキップされる。

保守担当者向けメモ

  • 新しい非同期対応ノードを追加する場合は、本関数の switch 文に新しい nodeTag を追加し、対応する ExecAsync...Notify 関数を実装する必要があります。
  • 実行統計の計測（InstrStartNode/InstrStopNode）は、非同期実行のコストを正確に把握するために必須です。
