LSEG

定義

  • 種別: struct
  • 定義箇所: src/include/utils/geo_decls.h:106

目的

PostgreSQLにおける線分（Line Segment）を表現するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    Point       p[2];
} LSEG;

入力と出力

  • 入力: 2つの Point 型の座標値。
  • 出力: なし（データ構造体）。

前提条件と不変条件

  • p[0] と p[1] は線分の両端点を表します。
  • 幾何演算において、浮動小数点数の比較には FPeq 等の許容誤差を考慮した関数が使用されます。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータの保持のみを行います。
  • 幾何学的な操作（交差判定、距離計算、包含判定など）は、src/backend/utils/adt/geo_ops.c 内の関数群によって実行されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/utils/adt/geo_ops.c 内の各種幾何演算関数（statlseg_construct, lseg_sl, lseg_invsl, lseg_interpt_line, lseg_interpt_lseg, lseg_contain_point, lseg_closept_point, lseg_closept_line, lseg_closept_lseg, box_contain_lseg, box_interpt_lseg, box_closept_lseg）。
  • 呼び出し先:
  • Point 型（p[0], p[1] の要素として使用）。

関連データ構造

  • Point: 線分を構成する端点として使用されます。
  • LINE, BOX: 幾何演算の対象として LSEG と組み合わせて使用されます。

失敗モードと境界条件

  • 浮動小数点演算の特性上、NaNが含まれる場合の挙動は未定義または FALSE となる可能性があります。
  • 数値解析の専門的な実装ではないため、推移律を満たさない比較結果が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/utils/geo_decls.h で定義されており、math.h および fmgr.h に依存しています。
  • 演算ロジックの詳細は src/backend/utils/adt/geo_ops.c を参照してください。
