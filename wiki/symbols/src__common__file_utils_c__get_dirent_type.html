get_dirent_type

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:546

目的

指定されたパスまたはディレクトリ構造体（dirent）に基づき、ファイルの種類（通常ファイル、ディレクトリ、シンボリックリンク等）を判定します。

シグネチャまたは形状

PGFileType get_dirent_type(const char *path, const struct dirent *de, bool look_through_symlinks, int elevel);

入力と出力

  • **入力**:
  • path: 対象ファイルのパス文字列。
  • de: dirent 構造体へのポインタ（d_type フィールドを利用）。
  • look_through_symlinks: シンボリックリンクを追跡するかどうかを指定するフラグ。
  • elevel: エラー発生時のログレベル。
  • **出力**:
  • PGFileType 型の列挙値（PGFILETYPE_REG, PGFILETYPE_DIR, PGFILETYPE_LNK, PGFILETYPE_UNKNOWN, PGFILETYPE_ERROR のいずれか）。

前提条件と不変条件

  • dirent.h が提供する d_type フィールドが利用可能な環境では、まずそれを使用して高速に判定を行います。
  • d_type が利用できない、または判定不能な場合は、stat または lstat システムコールにフォールバックします。

アルゴリズムまたは振る舞い

1. **高速判定**: DT_REG, DT_DIR, DT_LNK マクロが定義されている環境では、de->d_type を直接確認し、一致すれば結果を返します。 2. **フォールバック**: 高速判定で結果が得られない場合、look_through_symlinks の値に応じて stat または lstat を実行します。 3. **ファイル属性確認**: stat 構造体の st_mode を S_ISREG, S_ISDIR, S_ISLNK マクロで検証し、ファイルタイプを特定します。 4. **エラー処理**: システムコールが失敗した場合、elevel に応じてログ出力またはエラー報告を行い、PGFILETYPE_ERROR を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/replication/logical/snapbuild.c
  • src/backend/replication/slot.c
  • src/backend/storage/file/copydir.c
  • src/backend/storage/file/fd.c
  • src/backend/utils/misc/conffiles.c
  • **呼び出し先**:
  • stat / lstat (システムコール)
  • pg_log_generic (FRONTEND定義時)
  • ereport (バックエンド時)

関連データ構造

  • PGFileType: ファイルタイプを表現する列挙型。
  • struct dirent: ディレクトリ内のエントリ情報。
  • struct stat: ファイルのメタデータ構造体。

失敗モードと境界条件

  • stat/lstat が失敗した場合（ファイルが存在しない、権限がない等）、PGFILETYPE_ERROR を返し、指定されたログレベルでエラーを記録します。
  • シンボリックリンクの追跡設定（look_through_symlinks）が、実際のファイルシステム上の挙動と矛盾する場合、意図しないファイルタイプが返される可能性があります。

保守担当者向けメモ

  • 本関数は FRONTEND マクロの有無により、エラー報告の仕組み（pg_log_generic vs ereport）が切り替わります。
  • d_type の利用はプラットフォーム依存であるため、移植性を維持するために stat によるフォールバック処理は必須です。
