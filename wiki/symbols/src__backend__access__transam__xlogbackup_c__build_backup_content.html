build_backup_content

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogbackup.c:28

目的

BackupState 構造体に保持されたバックアップ情報を、backup_label またはバックアップ履歴ファイルとして使用可能なテキスト形式に変換して返却します。

シグネチャまたは形状

char *build_backup_content(BackupState *state, bool ishistoryfile)

入力と出力

  • **入力**:
  • state: バックアップの開始・終了位置、タイムライン、チェックポイント位置等を保持する BackupState 構造体へのポインタ。
  • ishistoryfile: true の場合、バックアップ終了情報を含む履歴ファイル形式で生成し、false の場合は backup_label 形式で生成します。
  • **出力**:
  • 生成されたテキスト内容を格納した palloc 済みの文字列（呼び出し元で解放が必要）。

前提条件と不変条件

  • state は NULL であってはなりません。
  • XLogRecPtrIsInvalid(state->istartpoint) と (state->istarttli == 0) は論理的に一致している必要があります。
  • タイムゾーンにはセッション設定ではなく log_timezone が使用されます。

アルゴリズムまたは振る舞い

1. makeStringInfo() で動的バッファを確保します。 2. state->starttime を log_timezone に基づき文字列化します。 3. XLByteToSeg および XLogFileName を使用して、開始LSNからWALファイル名を算出します。 4. 共通項目（開始WAL位置、チェックポイント位置、バックアップ手法、開始時刻、ラベル、開始タイムライン）をバッファに追記します。 5. ishistoryfile が true の場合、終了WAL位置、終了時刻、終了タイムラインを追記します。 6. 増分バックアップの情報（istartpoint, istarttli）が存在する場合、それらを追記します。 7. StringInfo のデータポインタを抽出し、構造体自体を pfree して文字列を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c:9333
  • src/backend/access/transam/xlogfuncs.c:151
  • src/backend/backup/basebackup.c:336
  • **呼び出し先**:
  • makeStringInfo, appendStringInfo, appendStringInfoString (文字列構築)
  • pg_strftime, pg_localtime (時刻フォーマット)
  • XLByteToSeg, XLogFileName (WAL情報変換)
  • pfree (メモリ解放)

関連データ構造

  • BackupState (src/include/access/xlogbackup.h で定義): バックアップのメタデータを保持する構造体。
  • StringInfo: 文字列構築用の一時バッファ構造体。

失敗モードと境界条件

  • メモリ不足時には palloc 経由でエラーが発生します。
  • state が NULL の場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は純粋なフォーマット生成器であり、ファイルI/Oは行いません。
  • 増分バックアップの起点情報が正しく出力されるよう、istartpoint と istarttli の整合性チェックが組み込まれています。
