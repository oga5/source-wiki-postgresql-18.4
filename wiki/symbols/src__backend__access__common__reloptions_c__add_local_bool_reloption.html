add_local_bool_reloption

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/reloptions.c:875

目的

ローカルなリレーションオプションのリストに対し、真偽値（bool）型のオプション定義を追加する。

シグネチャまたは形状

void add_local_bool_reloption(local_relopts *relopts, const char *name, const char *desc, bool default_val, int offset)

入力と出力

  • relopts: オプションを追加する対象の local_relopts 構造体へのポインタ。
  • name: オプションの名称。
  • desc: オプションの説明文。
  • default_val: オプションのデフォルト値。
  • offset: 構造体内での当該オプションの格納オフセット。
  • 戻り値: なし（void）。

前提条件と不変条件

  • relopts は初期化済みである必要がある。
  • RELOPT_KIND_LOCAL 種別のオプションとして登録される。

アルゴリズムまたは振る舞い

1. init_bool_reloption を呼び出し、RELOPT_KIND_LOCAL 種別で真偽値オプションのメタデータ構造体（relopt_bool）を生成する。 2. 生成されたオプションを add_local_reloption に渡し、指定された relopts リストおよび offset に登録する。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内の直接的な呼び出し元は提供された情報からは特定できない。
  • 呼び出し先: init_bool_reloption および add_local_reloption を呼び出す。

関連データ構造

  • local_relopts: ローカルオプションを管理する構造体。
  • relopt_bool: 真偽値オプションの定義を保持する構造体。
  • relopt_gen: relopt_bool の基底構造体。

失敗モードと境界条件

  • メモリ不足時には init_bool_reloption 内部でエラーが発生する可能性がある。
  • 呼び出し元が NULL ポインタを渡した場合の挙動は未定義。

保守担当者向けメモ

  • 本関数は local_relopts を拡張するためのラッパーである。新しい真偽値オプションを特定のアクセスメソッド等に追加する際に使用する。
