pgstat_checkpointer_reset_all_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_checkpointer.c:96

目的

チェックポイント統計のリセット処理を実行します。共有メモリ上の統計情報を現在の値でオフセットとして保存し、リセット時刻を更新することで、リセット以降の累積値計算を可能にします。

シグネチャまたは形状

void pgstat_checkpointer_reset_all_cb(TimestampTz ts)

入力と出力

  • **入力**: ts (TimestampTz) - リセット時刻として記録するタイムスタンプ。
  • **出力**: なし。共有メモリ上の PgStatShared_Checkpointer 構造体の内容を直接更新します。

前提条件と不変条件

  • 共有メモリ上のチェックポイント統計構造体 (pgStatLocal.shmem->checkpointer) が初期化済みであること。
  • 統計更新時の整合性を保つため、内部で LWLock を使用します。

アルゴリズムまたは振る舞い

1. 共有メモリ上のチェックポイント統計構造体へのポインタを取得します。 2. stats_shmem->lock を排他モード (LW_EXCLUSIVE) で取得します。 3. pgstat_copy_changecounted_stats を呼び出し、現在の統計値 (stats) をリセットオフセット (reset_offset) にコピーします。この際、changecount を用いてアトミックなコピーを保証します。 4. stats_shmem->stats.stat_reset_timestamp に引数 ts を代入し、リセット時刻を更新します。 5. stats_shmem->lock を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:428 において、統計リセット用コールバックとして登録されています。
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease
  • pgstat_copy_changecounted_stats

関連データ構造

  • PgStatShared_Checkpointer: チェックポイント統計を共有メモリ上で保持する構造体。
  • pgStatLocal: プロセスローカルな統計管理構造体。

失敗モードと境界条件

  • ロック取得時に他のプロセスとの競合が発生する可能性がありますが、LWLock により排他制御されます。
  • 共有メモリが未初期化の場合、ポインタ参照によりクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は pgstat のリセットプロトコルに従っています。統計の累積値をリセットする際は、単にゼロクリアするのではなく、現在の値を reset_offset に退避させることで、リセット後の差分計算を正しく行う設計となっています。
