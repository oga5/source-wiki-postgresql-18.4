storeInfo

定義

  • 種別: struct
  • 定義箇所: contrib/dblink/dblink.c:80

目的

リモートデータベースから取得したクエリ結果を、PostgreSQL の Tuplestore へマテリアライズする過程で、必要な状態と一時的なメモリリソースを保持するために使用されます。

シグネチャまたは形状

typedef struct storeInfo
{
	FunctionCallInfo fcinfo;
	Tuplestorestate *tuplestore;
	AttInMetadata *attinmeta;
	MemoryContext tmpcontext;
	char	  **cstrs;
	PGresult   *last_res;
	PGresult   *cur_res;
} storeInfo;

入力と出力

本構造体は関数間で受け渡される状態保持用オブジェクトです。

  • 入力: FunctionCallInfo や Tuplestorestate 等の初期化済みポインタ。
  • 出力: 処理中の PGresult や Tuplestore への参照、およびメモリリークを防ぐための tmpcontext。

前提条件と不変条件

  • tmpcontext は、例外発生時のメモリリークを防ぐための作業用メモリコンテキストとして使用されます。
  • last_res および cur_res は、クエリ実行中の結果セットを追跡するために使用されます。

アルゴリズムまたは振る舞い

リモートクエリの結果を Tuplestore に格納する際、storeQueryResult および storeRow 関数において、この構造体を介して状態が管理されます。特に、volatile 修飾子を付与して使用されることが多く、エラー発生時の PG_TRY/PG_FINALLY ブロック内での整合性維持に寄与します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/dblink/dblink.c 内のクエリ実行処理（例: dblink_record 等の関数内での初期化）。
  • 呼び出し先: storeQueryResult および storeRow 関数が、この構造体を引数として受け取り、結果の格納処理を行います。

関連データ構造

  • Tuplestorestate: 結果セットを保持する PostgreSQL の標準的な構造体。
  • PGresult: libpq から返されるリモートクエリの結果。
  • AttInMetadata: タプルの属性変換用メタデータ。

失敗モードと境界条件

  • tmpcontext の管理が不適切な場合、大規模な結果セットの処理時にメモリリークが発生する可能性があります。
  • PGresult の解放漏れを防ぐため、last_res と cur_res のポインタ管理には注意が必要です。

保守担当者向けメモ

  • 本構造体は volatile として宣言して使用されることが前提となっています。関数内で storeInfo を操作する際は、例外安全性を確保するため、必ず PG_TRY ブロック内での適切な初期化と解放を維持してください。
