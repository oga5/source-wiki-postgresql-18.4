gbt_inet_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_inet.c:116

目的

GiST インデックス検索において、inet 型のインデックスエントリが指定されたクエリ条件を満たすかどうかを判定します。

シグネチャまたは形状

Datum gbt_inet_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY 構造体（インデックスエントリ）。
  • PG_GETARG_DATUM(1): 検索対象のクエリ値。
  • PG_GETARG_UINT16(2): 比較戦略（StrategyNumber）。
  • PG_GETARG_POINTER(4): recheck フラグへのポインタ。
  • **出力**:
  • PG_RETURN_BOOL: 条件を満たす場合は true、満たさない場合は false。
  • *recheck: 常に true が設定されます。

前提条件と不変条件

  • inet 型の値を double 型へ変換する convert_network_to_scalar が成功する必要があります。
  • インデックスエントリのキーは inetKEY 構造体として解釈されます。
  • 本関数による判定結果は常に不正確（inexact）であると見なされ、エンジン側での再チェックが必須です。

アルゴリズムまたは振る舞い

1. convert_network_to_scalar を呼び出し、クエリ値を double 型のスカラ値に変換します。 2. recheck ポインタに true を書き込みます。 3. inetKEY から lower と upper の境界値を取り出し、GBT_NUMKEY_R 構造体に格納します。 4. gbt_num_consistent を呼び出し、変換後のスカラ値とインデックスキーを比較して判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス検索エンジン（直接の呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**:
  • convert_network_to_scalar
  • gbt_num_consistent

関連データ構造

  • inetKEY: インデックス内に格納される境界値構造体。
  • GISTENTRY: GiST インデックスエントリ。
  • GBT_NUMKEY_R: 数値比較用の範囲構造体。
  • tinfo: inet 型の比較ロジックを定義する gbtree_ninfo 構造体。

失敗モードと境界条件

  • convert_network_to_scalar が失敗した場合、Assert(!failure) によりデバッグビルドではアボートします。
  • inet 型の変換精度に依存するため、非常に大きなネットワーク範囲や複雑なサブネットマスクでは近似的な判定となります。

保守担当者向けメモ

  • 本関数は inet 型を double に変換して処理するため、数値変換の精度限界が検索結果に影響を与える可能性があります。
  • *recheck = true の設定は、GiST インデックスの特性上、インデックスのみでは完全な結果を保証できないために行われています。
