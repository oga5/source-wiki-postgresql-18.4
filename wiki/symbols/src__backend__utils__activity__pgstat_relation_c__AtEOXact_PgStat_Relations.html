AtEOXact_PgStat_Relations

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:550

目的

トランザクション終了時（コミットまたはアボート）に、トランザクション内で蓄積されたリレーション統計情報（PgStat_TableXactStatus）を、リレーションの永続的な統計状態（PgStat_TableStatus）へ転送・統合します。

シグネチャまたは形状

void AtEOXact_PgStat_Relations(PgStat_SubXactStatus *xact_state, bool isCommit)

入力と出力

  • xact_state: 処理対象のサブトランザクション状態を保持する構造体へのポインタ。
  • isCommit: トランザクションがコミットされる場合は true、アボートされる場合は false を指定します。
  • 出力: なし（副作用として tabstat->counts および tabstat->trans を更新）。

前提条件と不変条件

  • xact_state->first から辿れる各 PgStat_TableXactStatus 要素について、nest_level == 1 かつ upper == NULL であること。
  • tabstat->trans == trans が成立していること。

アルゴリズムまたは振る舞い

1. xact_state に紐付くすべての PgStat_TableXactStatus を走査します。 2. アボート時（isCommit == false）には、restore_truncdrop_counters を呼び出し、TRUNCATE/DROP 実行前の統計値を復元します。 3. 挿入・更新・削除の試行回数を、親の tabstat->counts に加算します。 4. コミット時（isCommit == true）:

  • TRUNCATE 実行フラグを反映し、実行された場合はライブ/デッドタプル数をリセットします。
  • 挿入・削除数に基づき delta_live_tuples を更新します。
  • 更新・削除数に基づき delta_dead_tuples を更新します。
  • 変更イベント数（changed_tuples）を更新します。

    • 5. アボート時（isCommit == false）:

  • 挿入・更新されたタプルをデッドタプルとして計上します。

    • 6. 最後に tabstat->trans を NULL に設定し、統計の紐付けを解除します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/activity/pgstat_xact.c 内のトランザクション終了処理。
  • 呼び出し先: restore_truncdrop_counters（アボート時のみ）。

関連データ構造

  • PgStat_SubXactStatus: トランザクション内の統計状態リストを保持。
  • PgStat_TableXactStatus: トランザクションローカルな統計カウンタ。
  • PgStat_TableStatus: リレーションごとの永続的な統計カウンタ。

失敗モードと境界条件

  • isCommit が false の場合、変更イベント（changed_tuples）は加算されません。
  • trans->truncdrop が設定されている場合、コミット時にはライブ/デッドタプルのデルタ計算がリセットされます。

保守担当者向けメモ

  • 本関数はトランザクションのネストレベルが 1 の状態でのみ実行されることを前提としています。
  • tabstat->trans のポインタ管理は、統計情報の二重反映を防ぐために重要です。デバッグ時は trans 階層の整合性に注意してください。
