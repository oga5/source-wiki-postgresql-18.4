XLogFileInit

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/transam/xlog.c:3375

目的

指定された WAL セグメント番号とタイムライン ID に基づき、新しい WAL ファイルを初期化または既存のファイルを開いてファイル記述子を返します。

シグネチャまたは形状

int XLogFileInit(XLogSegNo logsegno, TimeLineID logtli);

入力と出力

  • **入力**:
  • logsegno: 初期化対象の WAL セグメント番号。
  • logtli: 対象のタイムライン ID。
  • **出力**:
  • 成功時: 開かれたファイルのファイル記述子（int）。
  • 失敗時: ereport によりエラーを発生させ、プロセスを終了します。

前提条件と不変条件

  • logtli は 0 であってはならない（Assert(logtli != 0)）。
  • wal_sync_method に基づく同期ビットがファイルオープン時に適用されます。

アルゴリズムまたは振る舞い

1. XLogFileInitInternal を呼び出し、内部的な初期化処理（ファイルの作成やパスの生成）を試みます。 2. XLogFileInitInternal が有効なファイル記述子（fd >= 0）を返した場合は、その値をそのまま返します。 3. 内部初期化でファイルが開かれなかった場合、BasicOpenFile を使用して対象のパスを O_RDWR モードで開き直します。 4. ファイルのオープンに失敗した場合は、ereport(ERROR, ...) を発行して処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c (複数箇所: 2375, 5177, 5301行目)
  • src/backend/replication/walreceiver.c:910
  • **呼び出し先**:
  • XLogFileInitInternal
  • BasicOpenFile
  • ereport

関連データ構造

  • XLogSegNo (WAL セグメント番号)
  • TimeLineID (タイムライン識別子)
  • wal_sync_method (グローバル設定)

失敗モードと境界条件

  • 指定されたパスのファイルが開けない場合、ereport によりエラーが報告されます。
  • logtli が 0 の場合、アサーション違反によりデバッグビルドでは停止します。

保守担当者向けメモ

  • 本関数は XLogFileInitInternal のラッパーとして機能しています。ファイル作成ロジックの変更が必要な場合は、まず XLogFileInitInternal を確認してください。
  • wal_sync_method に依存したファイルオープンフラグを使用しているため、プラットフォーム固有の I/O 挙動に注意が必要です。
