SlabBlock

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/mmgr/slab.c:146

目的

SLAB アロケータにおいて、メモリブロック単位の管理を行うための構造体です。固定サイズのチャンクを保持し、空き領域の追跡とブロックのリスト管理を担います。

シグネチャまたは形状

typedef struct SlabBlock
{
    SlabContext *slab;          /* 所有するコンテキスト */
    int32       nfree;          /* 空きチャンク数（free + unused） */
    int32       nunused;        /* 未使用チャンク数 */
    MemoryChunk *freehead;      /* 解放済みチャンクのリスト先頭 */
    MemoryChunk *unused;        /* 未使用領域の先頭ポインタ */
    dlist_node  node;           /* ブロックリスト管理用双方向リストノード */
} SlabBlock;

入力と出力

本構造体は関数ではなくデータ構造ですが、SlabAlloc 等の関数において、SlabContext から取得されるか、malloc を介して新規確保されます。

前提条件と不変条件

  • nfree は freehead リスト内のチャンク数と nunused の合計値と一致する必要があります。
  • node は SlabContext が保持する blocklist または emptyblocks リストのいずれかに所属している必要があります。

アルゴリズムまたは振る舞い

  • **チャンク管理**: freehead（解放済みチャンクの再利用リスト）と unused（ブロック内の未割り当て領域）の2系統で空き領域を管理します。
  • **リスト移動**: チャンクの割り当てや解放により nfree が変化するたびに、ブロックは空き数に応じた適切な blocklist へ移動されます。
  • **ブロック解放**: nfree がブロック内の全チャンク数と等しくなった場合、ブロックは emptyblocks リストへ移動されるか、メモリが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: SlabAlloc、SlabFree、SlabGetNextFreeChunk、SlabAllocSetupNewChunk 等の slab.c 内の関数から参照・操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SlabContext: 本ブロックを所有する親コンテキスト。
  • MemoryChunk: ブロック内部で管理される個別のメモリ領域。
  • dlist_node: lib/ilist.h で定義される双方向リスト用ノード。

失敗モードと境界条件

  • malloc によるブロック確保失敗時はメモリ不足エラーとなります。
  • SlabBlock 自体のヘッダーサイズは Slab_BLOCKHDRSZ マクロで定義されており、ブロックの先頭に配置されます。

保守担当者向けメモ

  • SlabBlock のサイズは MAXALIGN されている必要があります。
  • SlabRealloc はサポートされていないため、本構造体を通じたサイズ変更ロジックの実装は不要です。
  • MEMORY_CONTEXT_CHECKING が有効な場合、SlabCheck によって本構造体の整合性が検証されます。
