GinFormInteriorTuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:200

目的

GINエントリーツリーの内部ノードで使用するインデックスタプルを生成します。既存のタプルを複製し、指定されたブロック番号（ダウンリンク）を格納した新しいタプルを作成します。

シグネチャまたは形状

IndexTuple GinFormInteriorTuple(IndexTuple itup, Page page, BlockNumber childblk)

入力と出力

  • **入力**:
  • itup: 複製元となる既存のインデックスタプル。
  • page: itup が格納されているページ。
  • childblk: 新しいタプルに設定するダウンリンク先のブロック番号。
  • **出力**:
  • 複製およびダウンリンク設定済みの新しい IndexTuple（メモリは palloc で確保）。

前提条件と不変条件

  • itup は有効なインデックスタプルであること。
  • 内部ノードへの昇格時、リーフページかつポスティングリストを含むタプルの場合は、ポスティングリスト部分を除去してサイズを調整する必要があります。

アルゴリズムまたは振る舞い

1. **タプルの複製**:

  • ページがリーフであり、かつタプルがポスティングツリーでない場合、ポスティングリスト部分を除去したサイズでメモリを確保し、ヘッダー情報を修正してコピーします。
  • それ以外の場合は、タプル全体をそのままコピーします。

    • 2. **ダウンリンクの設定**:

  • GinSetDownlink を使用し、コピーしたタプルに対して childblk をダウンリンクとして書き込みます。

    • 3. **返却**:

  • 構築された新しい IndexTuple を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginentrypage.c 内のページ分割処理（entrySplitPage 等）において、親ノードへ挿入するタプルを作成する際に呼び出されます。
  • **呼び出し先**:
  • palloc (メモリ確保)
  • memcpy (メモリコピー)
  • GinPageIsLeaf, GinIsPostingTree, GinGetPostingOffset, GinSetDownlink, IndexTupleSize (GIN/PostgreSQLマクロ)

関連データ構造

  • IndexTuple: PostgreSQLのインデックスエントリ構造。
  • Page: GINインデックスのページ構造。
  • BlockNumber: ブロック識別子。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • リーフページか否かの判定により、タプルのコピー範囲が動的に変化します。

保守担当者向けメモ

  • 本関数は、リーフノードのタプルを内部ノードへ昇格させる際の「ポスティングリストの切り離し」を担う重要な箇所です。
  • GinSetDownlink によるダウンリンク更新は、タプルの物理的な構造に依存するため、将来的なタプルフォーマット変更時には注意が必要です。
