dataFindChildPtr

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gindatapage.c:318

目的

GINポスティングツリーの内部ノードにおいて、指定されたブロック番号（blkno）を指す子ポインタのオフセット番号を検索します。

シグネチャまたは形状

static OffsetNumber dataFindChildPtr(GinBtree btree, Page page, BlockNumber blkno, OffsetNumber storedOff)

入力と出力

  • **入力**:
  • btree: GIN B-treeの制御構造体。
  • page: 検索対象のページ。
  • blkno: 検索対象のブロック番号。
  • storedOff: 前回の検索結果など、ヒントとして使用されるオフセット番号。
  • **出力**:
  • 見つかった場合は該当する OffsetNumber を返し、見つからない場合は InvalidOffsetNumber を返します。

前提条件と不変条件

  • page はリーフページであってはならない（!GinPageIsLeaf(page)）。
  • page はデータページでなければならない（GinPageIsData(page)）。

アルゴリズムまたは振る舞い

1. **ヒントの検証**: storedOff が有効な範囲内であれば、まずその位置の PostingItem を確認し、ブロック番号が一致するかチェックします。 2. **前方検索**: ヒント位置で一致しない場合、削除が発生していないと仮定して、storedOff 以降のオフセットを順次走査します。 3. **全件検索**: 上記で見つからない場合、あるいはヒントが無効な場合は、ページの先頭（FirstOffsetNumber）から maxoff まで全件走査を行います。 4. **戻り値**: 一致するブロック番号を持つ PostingItem が見つかればそのオフセットを返し、最後まで見つからなければ InvalidOffsetNumber を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c:1893 にて、btree->findChildPtr 関数ポインタに代入されています。
  • **呼び出し先**:
  • GinPageGetOpaque(page)
  • GinPageIsLeaf(page)
  • GinPageIsData(page)
  • GinDataPageGetPostingItem(page, i)
  • PostingItemGetBlockNumber(pitem)

関連データ構造

  • GinBtree: GIN B-treeの操作状態を保持する構造体。
  • PostingItem: ポスティングツリーの内部ノードに格納される、子ページへのポインタとキー情報を含む構造体。

失敗モードと境界条件

  • storedOff がページ内の有効なオフセット範囲外である場合、ヒント検索をスキップして全件検索へ移行します。
  • ページ内に該当する blkno が存在しない場合、InvalidOffsetNumber を返します。

保守担当者向けメモ

  • 本関数は GinBtree 構造体のメソッドとして利用されることを想定しています。
  • 削除操作が行われた後のページでは、ヒント情報（storedOff）が一致しない可能性が高いため、全件検索へのフォールバックが重要な役割を果たします。
