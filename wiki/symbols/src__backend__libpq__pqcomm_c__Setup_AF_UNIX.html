Setup_AF_UNIX

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqcomm.c:719

目的

Unixドメインソケットファイルの所有権（グループ）およびアクセス権限を設定する。

シグネチャまたは形状

static int Setup_AF_UNIX(const char *sock_path)

入力と出力

  • **入力**: sock_path (Unixドメインソケットのファイルパス)
  • **出力**: 成功時は STATUS_OK、失敗時は STATUS_ERROR を返す。

前提条件と不変条件

  • sock_path が抽象ソケット（先頭が @）である場合、ファイルシステム上の権限設定はスキップされる。
  • Unix_socket_group および Unix_socket_permissions がグローバル設定として利用可能であること。
  • listen() を呼び出す前に実行される必要がある（接続の受け入れ開始前に権限を確定させるため）。

アルゴリズムまたは振る舞い

1. **抽象ソケットの判定**: パスが @ で始まる場合は即座に STATUS_OK を返す。 2. **グループ設定**: Unix_socket_group が指定されている場合、数値IDまたはグループ名から gid を解決し、chown を実行する。

  • Windows環境ではサポート外のため警告を発する。

    • 3. **権限設定**: Unix_socket_permissions を用いて chmod を実行する。 4. **エラーハンドリング**: getgrnam によるグループ解決失敗、chown または chmod の実行失敗時には LOG レベルでエラーを報告し、STATUS_ERROR を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/pqcomm.c 内のソケット初期化処理（ListenServerPort 等から参照される）。
  • **呼び出し先**: getgrnam, chown, chmod, ereport 等の標準ライブラリおよびPostgreSQL内部関数。

関連データ構造

  • Unix_socket_group (グローバル変数)
  • Unix_socket_permissions (グローバル変数)
  • struct group (グループ情報取得用)

失敗モードと境界条件

  • **グループ解決失敗**: 指定されたグループ名が存在しない場合、ログを出力して終了する。
  • **ファイルアクセス権限エラー**: chown または chmod がシステム制限等で失敗した場合、ログを出力して終了する。
  • **プラットフォーム依存**: Windows環境では unix_socket_group の設定は無視され、警告が記録される。

保守担当者向けメモ

  • 本関数はソケットのリスン開始前に呼び出すことが必須である。
  • 抽象ソケット（Linux特有の機能）に対する権限設定は行われない仕様となっている。
