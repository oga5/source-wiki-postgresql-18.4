InitAuxiliaryProcess

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/proc.c:614

目的

補助プロセス（Auxiliary Process）が共有メモリ上で自身の状態を管理するための PGPROC 構造体を確保し、初期化を行う。

シグネチャまたは形状

void InitAuxiliaryProcess(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として MyProc グローバル変数および共有メモリ上の PGPROC エントリを初期化する）

前提条件と不変条件

  • ProcGlobal および AuxiliaryProcs が初期化済みであること。
  • 呼び出し時に MyProc が NULL であること（既に初期化済みでないこと）。
  • ProcStructLock を用いて AuxiliaryProcs の排他制御を行う。

アルゴリズムまたは振る舞い

1. ProcStructLock を取得し、AuxiliaryProcs 配列から pid == 0 である空きエントリを線形探索する。 2. 空きエントリが見つかった場合、その pid に現在のプロセスID (MyProcPid) を設定し、MyProc に割り当てる。 3. MyProc の各フィールド（links, waitStatus, xid, lwWaiting 等）を初期状態にリセットする。 4. OwnLatch および SwitchToSharedLatch を呼び出し、プロセスラッチを共有メモリ上のものへ切り替える。 5. pgstat_set_wait_event_storage を呼び出し、待機イベントの報告先を共有メモリに設定する。 6. PGSemaphoreReset でセマフォを初期化し、on_shmem_exit に終了処理関数 AuxiliaryProcKill を登録する。 7. InitLWLockAccess を呼び出し、軽量ロック利用のためのローカル状態を初期化する。 8. EXEC_BACKEND 環境下では、AttachSharedMemoryStructs を呼び出し共有メモリ構造体へのポインタを再アタッチする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/auxprocess.c:65
  • 呼び出し先: SpinLockAcquire, set_spins_per_delay, SpinLockRelease, GetNumberFromPGProc, dlist_node_init, pg_atomic_write_u64, OwnLatch, SwitchToSharedLatch, pgstat_set_wait_event_storage, PGSemaphoreReset, on_shmem_exit, InitLWLockAccess, AttachSharedMemoryStructs（EXEC_BACKEND時）

関連データ構造

  • PGPROC: プロセス状態を保持する共有メモリ上の構造体。
  • AuxiliaryProcs: 補助プロセス用 PGPROC の配列。
  • ProcGlobal: プロセス管理のグローバルヘッダー。
  • ProcStructLock: AuxiliaryProcs 配列を保護するスピンロック。

失敗モードと境界条件

  • ProcGlobal または AuxiliaryProcs が NULL の場合: PANIC で停止。
  • MyProc が既に設定されている場合: ERROR で停止。
  • 空き AuxiliaryProcs エントリが存在しない場合: FATAL で停止。

保守担当者向けメモ

  • AuxiliaryProcs の探索は線形探索であるため、補助プロセスの数が増加した場合は注意が必要。
  • EXEC_BACKEND 定義時は、共有メモリの再アタッチが必須となるため、この関数の実行順序が重要。
  • PGPROC のフィールド初期化は、InitProcGlobal で行われない項目を網羅する必要がある。
