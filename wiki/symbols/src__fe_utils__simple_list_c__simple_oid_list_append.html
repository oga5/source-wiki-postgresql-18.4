simple_oid_list_append

定義

  • 種別: function
  • 定義箇所: src/fe_utils/simple_list.c:25

目的

SimpleOidList 構造体で管理される単方向連結リストの末尾に、新しい OID 要素を追加します。

シグネチャまたは形状

void simple_oid_list_append(SimpleOidList *list, Oid val)

入力と出力

  • **入力**:
  • list: 操作対象の SimpleOidList 構造体へのポインタ。
  • val: リストに追加する Oid 値。
  • **出力**: なし（副作用としてリストを更新）。

前提条件と不変条件

  • list は有効な SimpleOidList 構造体を指している必要があります。
  • リストが空の場合、list->head と list->tail は NULL であることが期待されます。

アルゴリズムまたは振る舞い

1. pg_malloc を使用して新しい SimpleOidListCell をメモリ上に確保します。 2. 新しいセルの val フィールドに指定された val を格納し、next フィールドを NULL に初期化します。 3. リストが既に要素を持つ場合（list->tail が非 NULL）、現在の末尾セルの next を新しいセルに接続します。 4. リストが空の場合、list->head を新しいセルに設定します。 5. 最後に、list->tail が新しいセルを指すように更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c (複数箇所)
  • src/bin/scripts/reindexdb.c
  • **呼び出し先**:
  • pg_malloc (メモリ確保用)

関連データ構造

  • SimpleOidList: リストの head と tail を保持する構造体。
  • SimpleOidListCell: リストの各要素を構成するセル（val と next ポインタを保持）。

失敗モードと境界条件

  • pg_malloc がメモリ不足により失敗した場合、プログラムは終了します。
  • リストが空の状態から追加する場合、head と tail の両方が正しく新しいセルを指すように処理されます。

保守担当者向けメモ

  • 本関数は単方向リストの末尾への追加のみをサポートします。挿入や削除といった複雑な操作は提供されていません。
  • pg_malloc を使用しているため、リストの破棄には対応する simple_oid_list_destroy を使用してメモリリークを防ぐ必要があります。
