box_penalty

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistproc.c:96

目的

GiSTインデックスにおいて、既存の矩形（original）に新しい矩形（new）を追加する際のコスト（面積の増加量）を算出します。

シグネチャまたは形状

static float8 box_penalty(const BOX *original, const BOX *new)

入力と出力

  • **入力**:
  • original: 既存の矩形領域を示す BOX 構造体へのポインタ。
  • new: 追加対象の矩形領域を示す BOX 構造体へのポインタ。
  • **出力**:
  • 矩形を拡張した際の面積増加分を float8 型で返します。

前提条件と不変条件

  • original および new は有効な BOX 構造体を指している必要があります。
  • 内部で rt_box_union および size_box を使用しており、これらが幾何学的に正しい値を返すことが前提となります。

アルゴリズムまたは振る舞い

1. rt_box_union を呼び出し、original と new を包含する最小境界矩形（MBR）を unionbox として計算します。 2. size_box を用いて unionbox の面積を算出します。 3. 同様に original の面積を算出します。 4. unionbox の面積から original の面積を減算し、その差分をペナルティ値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c 内の複数箇所（インデックス挿入時のコスト評価や、分割時のエントリ分配ロジックで使用）。
  • **呼び出し先**:
  • rt_box_union
  • size_box
  • float8_mi

関連データ構造

  • BOX: 2次元幾何データ型。

失敗モードと境界条件

  • 入力された矩形が不正な座標値を持つ場合、size_box の計算結果に依存します。
  • 面積の差分が非常に大きい場合、浮動小数点演算の精度制限を受ける可能性があります。

保守担当者向けメモ

  • 本関数はGiSTのペナルティ計算ロジックの基礎であり、インデックスの木のバランスに直接影響します。
  • 幾何学的な面積計算ロジックを変更する場合は、size_box の実装と整合性が取れているか確認してください。
