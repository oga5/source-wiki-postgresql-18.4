InfoItem

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_config/pg_config.c:36

目的

コマンドライン引数（--bindir 等）と、内部的な設定項目名（BINDIR 等）を紐付けるためのマッピング構造体です。

シグネチャまたは形状

typedef struct
{
	const char *switchname;
	const char *configname;
} InfoItem;

入力と出力

  • 入力: switchname（コマンドライン引数名）、configname（設定項目名）の各文字列。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • switchname および configname は、pg_config がサポートする設定項目と一対一で対応している必要があります。
  • info_items[] テーブル内で定義される各要素は、help() 関数内の出力内容と同期している必要があります。

アルゴリズムまたは振る舞い

  • pg_config 実行時、ユーザーから指定された引数が info_items 配列内の switchname と照合されます。
  • 一致した場合、対応する configname を用いて show_item() 関数が設定情報を検索・出力します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_config/pg_config.c 内の info_items[] 配列定義（src/bin/pg_config/pg_config.c:42）。
  • 呼び出し先: 情報なし。

関連データ構造

  • info_items[]: InfoItem 型の静的配列。サポートする全オプションの定義テーブル。
  • ConfigData: common/config_info.h で定義される、実際のビルド設定情報を持つ構造体。

失敗モードと境界条件

  • 引数が info_items に存在しない場合、プログラムはエラーメッセージを表示して終了します。

保守担当者向けメモ

  • 新しいオプションを追加する際は、info_items テーブルへの追加と help() 関数内の出力内容の更新を両方行う必要があります。
