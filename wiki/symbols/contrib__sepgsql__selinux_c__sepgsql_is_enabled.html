sepgsql_is_enabled

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/selinux.c:615

目的

SE-PostgreSQL 拡張機能が現在有効な状態であるかどうかを判定する。

シグネチャまたは形状

bool sepgsql_is_enabled(void)

入力と出力

  • **入力**: なし
  • **出力**: bool 型。SE-PostgreSQL が有効であれば true、無効であれば false を返す。

前提条件と不変条件

  • グローバル変数 sepgsql_mode が現在の動作モードを保持している必要がある。
  • SEPGSQL_MODE_DISABLED は、SE-PostgreSQL が無効化されている状態を示す定数として定義されている。

アルゴリズムまたは振る舞い

現在の動作モード sepgsql_mode を参照し、その値が SEPGSQL_MODE_DISABLED と等しくない場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/label.c: 541行目、584行目、622行目、868行目にて、ラベル操作の実行可否判定に使用。
  • **呼び出し先**:
  • 外部関数呼び出しなし（グローバル変数 sepgsql_mode の参照のみ）。

関連データ構造

  • sepgsql_mode: 動作モード（Disabled, Default, Permissive, Internal）を保持する変数。

失敗モードと境界条件

  • 本関数自体は単純な比較演算であるため、失敗モードは存在しない。
  • sepgsql_mode が初期化されていない場合や不正な値である場合の挙動は、本関数の定義範囲外である。

保守担当者向けメモ

  • 本関数は、SE-PostgreSQL の機能が有効かを確認するための標準的なインターフェースである。機能の有効/無効を判定する際は、直接 sepgsql_mode を参照せず、本関数を使用すること。
