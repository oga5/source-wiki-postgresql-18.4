xl_heap_prune

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam_xlog.h:285

目的

VACUUM処理に伴うヒープページの整理（Pruning）およびタプルの凍結（Freezing）操作をWAL（Write-Ahead Logging）に記録するためのメイン構造体です。

シグネチャまたは形状

typedef struct xl_heap_prune
{
    uint8       reason;
    uint8       flags;
    /* 続くデータは可変長 */
} xl_heap_prune;

入力と出力

  • **入力**: reason（整理の理由）、flags（XLHP_*系フラグ）。
  • **出力**: WALレコードの一部としてシリアライズされ、リカバリ時にヒープページの再構築に使用されます。

前提条件と不変条件

  • flags に XLHP_HAS_CONFLICT_HORIZON が設定されている場合、構造体の直後にアラインメントなしで TransactionId（conflict horizon XID）が続きます。
  • 構造体サイズは SizeOfHeapPrune マクロ（offsetof(xl_heap_prune, flags) + sizeof(uint8)）で計算可能です。

アルゴリズムまたは振る舞い

1. **記録**: heapam.c または pruneheap.c において、整理対象のページ情報とともに本構造体がWALレコードの先頭に配置されます。 2. **再生**: リカバリ時、heap_redo 等の関数が本構造体を読み込み、flags を確認して後続の可変長データ（凍結計画やオフセット配列など）をデシリアライズします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/pruneheap.c および src/backend/access/heap/heapam_xlog.c でレコード構築時に使用されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出しません。リカバリ時には heap_redo 等のREDO関数が本構造体を解釈します。

関連データ構造

  • xlhp_freeze_plan: 凍結計画を保持するサブレコード。
  • xlhp_prune_items: 整理対象アイテムの配列。
  • xlhp_freeze_plans: 複数の凍結計画を管理する構造体。

失敗モードと境界条件

  • 可変長データを含むため、デシリアライズ時に flags の判定を誤るとメモリ破壊や不正なポインタ参照を引き起こす可能性があります。
  • XLHP_HAS_CONFLICT_HORIZON の有無によるデータ配置のズレに注意が必要です。

保守担当者向けメモ

  • 本構造体は単体で完結せず、後に続く可変長データ（凍結計画やオフセット配列）とセットで扱われます。解析やデバッグの際は heap_xlog_deserialize_prune_and_freeze の実装を参照してください。
  • XLOG_HEAP2_ 系操作の一部として扱われることが多いため、関連するオペコードの定義と併せて確認してください。
