StatsData

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pgbench/pgbench.c:378

目的

pgbench におけるトランザクション実行結果の統計情報を保持するための構造体です。成功、失敗、スキップされたトランザクションの数や、再試行回数、レイテンシ、ラグの集計に使用されます。

シグネチャまたは形状

typedef struct StatsData
{
    pg_time_usec_t start_time;
    int64          cnt;
    int64          skipped;
    int64          retries;
    int64          retried;
    int64          serialization_failures;
    int64          deadlock_failures;
    SimpleStats    latency;
    SimpleStats    lag;
} StatsData;

入力と出力

  • **入力**: 各種統計関数（initStats, accumStats 等）を通じて、トランザクションの実行結果やタイミング情報が更新されます。
  • **出力**: getFailures 関数等で集計値が読み取られ、最終的なベンチマーク結果の出力に使用されます。

前提条件と不変条件

  • 全トランザクション数 = skipped + cnt + failed（ここで failed は serialization_failures + deadlock_failures）。
  • cnt（成功数）は、直接成功したものと、再試行を経て成功したものの合計です。
  • retries は、成功した再試行と失敗した再試行の両方を含む、すべての再試行回数の合計です。

アルゴリズムまたは振る舞い

  • トランザクションのライフサイクル（開始、成功、失敗、再試行、スキップ）を追跡します。
  • --rate や --latency-limit オプションが指定された場合、実行が遅延したトランザクションを skipped としてカウントします。
  • 直列化エラーやデッドロックが発生した場合、再試行回数および失敗の内訳を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: advanceConnectionState（状態遷移）、initStats（初期化）、accumStats（集計）、getFailures（失敗数取得）などから参照・操作されます。
  • **呼び出し先**: SimpleStats 構造体のメソッド（latency, lag の更新用）が内部的に利用されます。

関連データ構造

  • SimpleStats: レイテンシやラグの統計計算に使用される構造体。
  • pg_time_usec_t: 時間計測に使用される型。

失敗モードと境界条件

  • serialization_failures および deadlock_failures は、再試行が成功しなかった場合にのみインクリメントされます。
  • cnt にはスキップされたトランザクションは含まれません。

保守担当者向けメモ

  • 本構造体は pgbench.c 内の統計集計の基盤です。フィールドの追加や変更を行う際は、accumStats 関数内の集計ロジックとの整合性に注意してください。
  • SimpleStats が保持する統計情報の精度や範囲については、SimpleStats 側の定義を確認してください。
