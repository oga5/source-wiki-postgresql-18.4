ParallelWorkerInfo

定義

  • 種別: struct
  • 定義箇所: src/include/access/parallel.h:25

目的

並列クエリ実行における個々のバックグラウンドワーカーの管理状態を保持する構造体です。

シグネチャまたは形状

typedef struct ParallelWorkerInfo
{
	BackgroundWorkerHandle *bgwhandle;
	shm_mq_handle *error_mqh;
} ParallelWorkerInfo;

入力と出力

  • **入力**: ParallelContext 内でワーカーの数に応じて動的に割り当てられる配列要素として利用されます。
  • **出力**: 保持するハンドルを通じて、ワーカープロセスの制御およびエラー情報の取得に使用されます。

前提条件と不変条件

  • ParallelContext のライフサイクル内で管理される必要があります。
  • bgwhandle は BackgroundWorker の起動状態を追跡するために使用されます。

アルゴリズムまたは振る舞い

  • リーダープロセスが LaunchParallelWorkers を実行する際、ワーカー数分の ParallelWorkerInfo 領域が確保されます。
  • bgwhandle を介してワーカーの起動・終了監視を行い、error_mqh を介してワーカーから送信されるエラーメッセージをリーダープロセスが受信します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/parallel.c において、ParallelContext の初期化時に palloc0 を通じてメモリが割り当てられます。
  • **呼び出し先**: 具体的な関数呼び出しの詳細は提供されていません。

関連データ構造

  • BackgroundWorkerHandle: ワーカープロセスのライフサイクル管理に使用。
  • shm_mq_handle: ワーカーとリーダー間のエラー報告用メッセージキュー。
  • ParallelContext: 本構造体を保持し、並列実行セッション全体を管理する親構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc0）の挙動については、本構造体単体では定義されていません。
  • ワーカーが異常終了した場合、error_mqh を通じたエラー情報の取得が失敗または不完全になる可能性があります。

保守担当者向けメモ

  • 本構造体は src/bin/pg_dump/parallel.c で定義される並列処理とは無関係であり、バックエンドの並列クエリ実行専用です。
  • ParallelContext のクリーンアップ時に、本構造体が保持するハンドルが適切に解放されることを確認してください。
