OprCacheKey

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_oper.c:51

目的

演算子検索の高速化を目的とした、演算子キャッシュ（OprCacheHash）のルックアップキーとして機能する構造体です。

シグネチャまたは形状

typedef struct OprCacheKey
{
	char		oprname[NAMEDATALEN];
	Oid			left_arg;		/* 左辺の型OID、前置演算子の場合は0 */
	Oid			right_arg;		/* 右辺の型OID */
	Oid			search_path[MAX_CACHED_PATH_LEN];
} OprCacheKey;

入力と出力

  • **入力**: 演算子名、左右の引数型OID、および現在の検索パス。
  • **出力**: ハッシュテーブルのキーとして使用され、対応する OprCacheEntry（演算子のOID）を特定するために利用されます。

前提条件と不変条件

  • left_arg が 0 の場合、その演算子は前置演算子として扱われます。
  • search_path は検索のコンテキストを保持し、キャッシュの一貫性を保証します。

アルゴリズムまたは振る舞い

1. make_oper_cache_key を通じて、現在の ParseState から演算子名、引数型、検索パスを抽出し、構造体を初期化します。 2. find_oper_cache_entry がこのキーを使用してハッシュテーブルを検索し、キャッシュヒット時は対応する演算子OIDを返します。 3. キャッシュミス時は、解決後に make_oper_cache_entry を呼び出し、このキーと解決結果をハッシュテーブルに登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_oper.c 内の演算子解決ロジック（make_oper_cache_key, find_oper_cache_entry, make_oper_cache_entry など）。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • OprCacheEntry: キャッシュの値を保持する構造体。
  • OprCacheHash: OprCacheKey をキーとして使用するハッシュテーブル。

失敗モードと境界条件

  • NAMEDATALEN を超える演算子名は切り詰められる可能性があります。
  • search_path の長さが MAX_CACHED_PATH_LEN を超える場合、キャッシュの正確性に影響を与える可能性があります。

保守担当者向けメモ

  • pg_operator や pg_cast の変更時に InvalidateOprCacheCallBack が呼ばれ、キャッシュがフラッシュされるため、このキーを用いた検索結果は常に最新のカタログ状態と同期されます。
