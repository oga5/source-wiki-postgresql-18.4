growable_trgm_array

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_op.c:78

目的

文字列から抽出されたトリグラム（3文字の連続）を格納し、必要に応じて動的に拡張可能な配列を管理するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    TRGM       *datum;          /* トリグラム配列 */
    int         length;         /* 配列内のトリグラム数 */
    int         allocated;      /* datum に割り当てられた容量（トリグラム数） */
} growable_trgm_array;

入力と出力

本構造体は、トリグラム生成処理におけるバッファとして機能します。

  • **入力**: init_trgm_array による初期化、および enlarge_trgm_array による容量拡張。
  • **出力**: make_trigrams や generate_trgm_only 等の関数を通じて、抽出されたトリグラムデータが datum に格納されます。

前提条件と不変条件

  • datum は TRGM 型の配列を指すポインタである必要があります。
  • length は常に allocated 以下である必要があります。
  • メモリ割り当て時には MaxAllocSize を超えないよう注意が必要です。

アルゴリズムまたは振る舞い

  • init_trgm_array を呼び出して初期メモリを確保し、enlarge_trgm_array を呼び出すことで、配列の容量が不足した際に動的にメモリを再割り当てします。
  • 文字列処理関数（make_trigrams 等）は、この構造体を介してトリグラムの抽出・格納を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: init_trgm_array, enlarge_trgm_array, make_trigrams, generate_trgm_only 等の関数内でインスタンスが生成・操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TRGM: トリグラムを表現する型。
  • TrgmBound: generate_trgm_only 等で使用される境界情報。

失敗モードと境界条件

  • メモリ不足時に enlarge_trgm_array が失敗する可能性があります。
  • MaxAllocSize を超えるサイズの割り当て要求が発生した場合、システムエラーとなります。

保守担当者向けメモ

  • CMPTRGM 関数ポインタを介した比較ロジックと併用されることが多いため、トリグラムのソートや重複排除を行う際は注意してください。
  • init_trgm_array と enlarge_trgm_array を適切に使用し、メモリリークやバッファオーバーフローを防ぐ設計となっています。
