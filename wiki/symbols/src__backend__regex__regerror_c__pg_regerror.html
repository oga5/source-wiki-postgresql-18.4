pg_regerror

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regerror.c:59

目的

正規表現エンジンのエラーコードを、人間が読み取り可能なメッセージやシンボル名に変換、またはその逆変換を行うためのインターフェースを提供します。

シグネチャまたは形状

size_t pg_regerror(int errcode, const regex_t *preg, char *errbuf, size_t errbuf_size);

入力と出力

  • **入力**:
  • errcode: エラーコード、または変換モードを指定する定数 (REG_ATOI, REG_ITOA)。
  • preg: 関連する regex_t 構造体（現在は未使用）。
  • errbuf: 結果を格納するバッファ。
  • errbuf_size: errbuf のサイズ。0 を指定可能。
  • **出力**:
  • 戻り値: メッセージを格納するために必要なバッファサイズ（終端文字 \0 を含む）。

前提条件と不変条件

  • rerrs 配列は code が負の値（番兵）になるまで線形探索されます。
  • errbuf_size が 0 の場合、バッファへの書き込みは行われませんが、必要なサイズは計算されて返されます。
  • errbuf_size が不足している場合、文字列は切り詰められ、終端文字が保証されます。

アルゴリズムまたは振る舞い

1. errcode に基づき switch 文で処理を分岐します。

  • REG_ATOI: errbuf 内の文字列（名前）を対応する数値コードに変換し、convbuf に格納します。
  • REG_ITOA: errbuf 内の数値文字列を対応する名前（REG_...）に変換します。該当がない場合は REG_ + 数値の形式で生成します。
  • 通常の errcode: rerrs 配列から対応する説明文 (explain) を検索します。該当がない場合は unk テンプレートを使用して未知のエラーメッセージを生成します。

    • 2. 取得したメッセージの長さを計算します。 3. errbuf_size が 0 より大きい場合、バッファの容量に応じてメッセージをコピーまたは切り詰めて書き込みます。 4. 最終的に必要なバッファサイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_regexp.c
  • src/backend/libpq/hba.c
  • src/backend/tsearch/spell.c
  • src/backend/utils/adt/regexp.c
  • src/backend/utils/adt/varlena.c

  • **呼び出し先**:
  • 標準ライブラリ関数: strcmp, sprintf, atoi, strlen, strcpy, memcpy。

関連データ構造

  • rerr: エラーコード、名前、説明文を保持する構造体。
  • rerrs: rerr 構造体の静的配列（regerrs.h で定義）。

失敗モードと境界条件

  • rerrs 内に該当するコードや名前が見つからない場合、未知のエラーとして処理されます。
  • errbuf_size が 1 の場合、終端文字のみが書き込まれます。
  • atoi の失敗に対する明示的なエラーハンドリングは行われません。

保守担当者向けメモ

  • POSIX の regerror に準拠しつつ、PostgreSQL 固有の拡張を含みます。
  • rerrs 配列の末尾には {-1, "", "oops"} という番兵が存在するため、探索ループの終了条件として利用されています。
