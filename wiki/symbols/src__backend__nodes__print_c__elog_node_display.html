elog_node_display

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/print.c:71

目的

PostgreSQL の内部ノード構造をデバッグログ（ereport）に出力する。

シグネチャまたは形状

void elog_node_display(int lev, const char *title, const void *obj, bool pretty)

入力と出力

  • lev: ログレベル（例: LOG）。
  • title: ログ出力時に付与される見出し文字列。
  • obj: 表示対象のノード構造体へのポインタ。
  • pretty: true の場合、整形（インデント）を適用する。
  • 出力: 戻り値なし。副作用としてログシステムへ文字列を出力する。

前提条件と不変条件

  • obj は nodeToStringWithLocations が処理可能な有効なノード構造体であること。
  • 内部で pfree を使用するため、渡された obj 自体は解放されないが、生成された一時的な文字列バッファは関数内で適切に解放される。

アルゴリズムまたは振る舞い

1. nodeToStringWithLocations(obj) を呼び出し、ノードをシリアライズされた文字列 s に変換する。 2. pretty 引数に応じて pretty_format_node_dump(s) または format_node_dump(s) を呼び出し、整形済み文字列 f を生成する。 3. pfree(s) でシリアライズ文字列を解放する。 4. ereport を使用し、title を見出しとして、f を詳細情報としてログに出力する。 5. pfree(f) で整形済み文字列を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c: 解析ツリーやプランツリーのデバッグ出力に使用。
  • **呼び出し先**:
  • nodeToStringWithLocations
  • pretty_format_node_dump
  • format_node_dump
  • pfree
  • ereport

関連データ構造

  • Node (汎用ノード構造体)
  • StringInfo (整形処理の内部で使用)

失敗モードと境界条件

  • obj が NULL の場合、nodeToStringWithLocations の実装に依存する（通常はクラッシュまたはエラー）。
  • 非常に巨大なノード構造の場合、nodeToStringWithLocations による文字列化および整形処理でメモリ消費量が増大する可能性がある。

保守担当者向けメモ

  • 本関数はデバッグ専用であり、本番環境のパスで呼び出すことは推奨されない。
  • pretty フラグによる整形はヒューリスティックな解析に基づくため、複雑なノード構造では表示が崩れる場合がある。
