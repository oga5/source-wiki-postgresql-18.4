sortDumpableObjects

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump_sort.c:557

目的

データベースオブジェクトの依存関係に基づき、復元時に整合性が保たれるようトポロジカルソートを実行し、ダンプ順序を決定する。

シグネチャまたは形状

void sortDumpableObjects(DumpableObject **objs, int numObjs, DumpId preBoundaryId, DumpId postBoundaryId)

入力と出力

  • **入力**:
  • objs: ソート対象の DumpableObject ポインタ配列。
  • numObjs: 配列内のオブジェクト数。
  • preBoundaryId, postBoundaryId: 依存関係の境界条件を定義する識別子。
  • **出力**: なし（objs 配列の内容がソート後の順序に書き換えられる）。

前提条件と不変条件

  • numObjs が 0 以下の場合は何も行わない。
  • 内部的に preDataBoundId および postDataBoundId という静的変数に境界 ID を保持する。

アルゴリズムまたは振る舞い

1. 引数で受け取った境界 ID を静的変数に保存する。 2. ソート用の一時作業領域 ordering を確保する。 3. TopoSort を呼び出し、依存関係に基づく順序付けを試みる。 4. TopoSort が循環依存を検出して失敗した場合、findDependencyLoops を呼び出して循環を特定・修復し、再度ソートを試みる。 5. 最終的な順序を objs 配列にコピーし、作業領域を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c (複数箇所)
  • **呼び出し先**:
  • TopoSort
  • findDependencyLoops
  • pg_malloc / free (メモリ管理)
  • memcpy (結果の反映)

関連データ構造

  • DumpableObject: ダンプ対象のオブジェクト情報を保持する構造体。
  • DumpId: オブジェクトを識別するための型。

失敗モードと境界条件

  • numObjs が 0 の場合、処理をスキップして即座に復帰する。
  • 循環依存が解決できない場合、findDependencyLoops の内部ロジックに依存する。

保守担当者向けメモ

  • 境界 ID を静的変数に保存する設計は、補助関数への引数追加を避けるための暫定的な実装である。
  • TopoSort の結果が不安定な場合、DOTypeNameCompare 等の比較ロジックが適切か確認すること。
