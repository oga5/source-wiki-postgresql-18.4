PQsendQueryContinue

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-exec.c:1438

目的

非公開のクエリ送信インターフェースとして、バックエンドへのクエリ送信処理を継続または実行するために使用されます。

シグネチャまたは形状

int PQsendQueryContinue(PGconn *conn, const char *query);

入力と出力

  • conn: 接続状態を保持する PGconn 構造体へのポインタ。
  • query: 送信する SQL クエリ文字列。
  • 戻り値: 処理の成否を示す整数値（成功時に 1、失敗時に 0）。

前提条件と不変条件

  • conn が有効な PGconn オブジェクトを指している必要があります。
  • 内部的に PQsendQueryInternal を呼び出すため、同関数の制約に従います。

アルゴリズムまたは振る舞い

内部関数 PQsendQueryInternal を呼び出し、第3引数に false を渡すことでクエリ送信処理を実行します。この関数は非公開（非エクスポート）のバージョンとして定義されています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-connect.c 内の複数箇所から呼び出されています。
  • **呼び出し先**: PQsendQueryInternal を呼び出します。

関連データ構造

  • PGconn: libpq の接続状態を管理する主要な構造体。

失敗モードと境界条件

  • PQsendQueryInternal の振る舞いに依存します。接続が確立されていない場合や、引数が不正な場合には 0 を返します。

保守担当者向けメモ

  • 本関数は libpq-int.h で宣言されており、ライブラリ内部での利用を想定しています。
  • PQsendQuery と同様の機能を提供しますが、内部的な制御フローの一部として使用されます。
