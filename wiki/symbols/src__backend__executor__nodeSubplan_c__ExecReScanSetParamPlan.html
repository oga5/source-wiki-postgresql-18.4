ExecReScanSetParamPlan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:1282

目的

InitPlan（クエリ実行中にパラメータ変更に応じて再評価されるサブプラン）の再スキャン準備を行う。具体的には、依存するパラメータの再計算が必要であることを示すフラグを立て、親プランノードに再スキャンの必要性を通知する。

シグネチャまたは形状

void ExecReScanSetParamPlan(SubPlanState *node, PlanState *parent)

入力と出力

  • **入力**:
  • node: 再スキャン対象の SubPlanState。
  • parent: このサブプランを保持する親の PlanState。
  • **出力**: なし（副作用として estate 内のパラメータデータおよび parent->chgParam を更新する）。

前提条件と不変条件

  • subplan->parParam が NIL であること（相関サブクエリは InitPlan としてサポートされない）。
  • subplan->setParam が空でないこと。
  • planstate->plan->extParam が空でないこと。
  • 上記条件を満たさない場合、エラーが発生する。

アルゴリズムまたは振る舞い

1. サブプランが InitPlan として妥当か検証する。 2. 実際の再スキャン処理は行わず、必要に応じて ExecSetParamPlan 内で実行されるよう待機させる。 3. subplan->setParam リストを走査し、各パラメータ ID に対応する ParamExecData を取得する。 4. CTE_SUBLINK 型でない場合、prm->execPlan に現在の node を設定し、再計算が必要であることをマークする。 5. 親ノードの chgParam ビットマップに該当パラメータ ID を追加し、依存するプランノードへ再スキャンの必要性を伝播させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c（ExecReScan 等の実行エンジン経由で呼び出される）。
  • **呼び出し先**:
  • bms_add_member（ビットマップ操作）
  • elog（エラー報告）
  • lfirst_int（リスト操作）
  • foreach（リスト走査）

関連データ構造

  • SubPlanState: サブプランの実行状態を保持する構造体。
  • PlanState: 親プランの実行状態。
  • ParamExecData: パラメータの実行時値を保持する構造体。
  • EState: クエリ実行全体の実行状態。

失敗モードと境界条件

  • 相関サブクエリ（parParam != NIL）が渡された場合、エラーとなる。
  • setParam リストが空の場合、エラーとなる。
  • extParam が空の場合、エラーとなる。

保守担当者向けメモ

  • 本関数は再スキャンそのものではなく、再スキャンをトリガーするための「フラグ立て」を行う関数である。
  • CTE サブプランはパラメータ再計算による実行を行わないため、execPlan の設定はスキップされるが、chgParam の更新は行われる点に注意すること。
