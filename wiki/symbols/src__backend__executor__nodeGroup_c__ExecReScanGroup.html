ExecReScanGroup

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGroup.c:234

目的

Group ノードの実行状態をリセットし、再スキャン可能な状態に戻します。

シグネチャまたは形状

void ExecReScanGroup(GroupState *node)

入力と出力

  • **入力**: GroupState *node （グループ化処理の実行状態を保持する構造体）
  • **出力**: なし（副作用として node の状態を更新）

前提条件と不変条件

  • 入力ノードは Group ノードの実行状態である必要があります。
  • 子プランがソート済みであることを前提として動作します。

アルゴリズムまたは振る舞い

1. node->grp_done フラグを false に設定し、グループ化処理が完了していない状態に戻します。 2. ExecClearTuple を呼び出し、現在保持しているスキャン用タプル（ss.ss_ScanTupleSlot）をクリアします。 3. 子プランの chgParam が NULL の場合、子プランに対して ExecReScan を実行し、子ノードの再スキャンをトリガーします。chgParam が NULL でない場合は、子ノードが最初の ExecProcNode 呼び出し時に自動的に再スキャンされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:270
  • **呼び出し先**:
  • ExecClearTuple
  • ExecReScan （条件付き）

関連データ構造

  • GroupState: グループ化処理の実行状態を保持。
  • PlanState: 子プランの実行状態。

失敗モードと境界条件

  • node が NULL の場合の挙動については、本ソースコード内では明示されていません。

保守担当者向けメモ

  • 本関数は Group ノードの再実行を制御する重要な役割を担います。grp_done フラグのリセットとスロットのクリアが必須の手順です。
  • 子プランの再スキャン制御は chgParam に依存するため、このロジックを変更する際はプラン実行エンジンの再スキャン仕様に注意してください。
