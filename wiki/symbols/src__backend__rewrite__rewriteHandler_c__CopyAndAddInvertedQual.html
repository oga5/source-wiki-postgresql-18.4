CopyAndAddInvertedQual

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteHandler.c:2354

目的

クエリ書き換えルールにおける条件式（qual）を複製し、指定されたリレーションのコンテキストに合わせて変数を置換した上で、対象のクエリツリーに付与する。

シグネチャまたは形状

static Query *CopyAndAddInvertedQual(Query *parsetree, Node *rule_qual, int rt_index, CmdType event)

入力と出力

  • **入力**:
  • parsetree: 書き換え対象のクエリツリー。
  • rule_qual: 付与する条件式のノード（リレーションキャッシュ由来）。
  • rt_index: 置換対象となるリレーションの範囲テーブルインデックス。
  • event: コマンド種別（CMD_INSERT, CMD_UPDATE 等）。
  • **出力**:
  • 条件式が付与された parsetree へのポインタ。

前提条件と不変条件

  • rule_qual はリレーションキャッシュ内に存在するため、直接変更してはならない（copyObject で複製が必要）。
  • event が CMD_INSERT または CMD_UPDATE の場合、生成列（generated columns）の解決が必要となる。

アルゴリズムまたは振る舞い

1. rule_qual を copyObject で複製し、サブクエリ内のロック取得とJOINの整合性調整を行う。 2. PRS2_OLD_VARNO を rt_index に置換する。 3. event が INSERT または UPDATE の場合:

  • 対象リレーションから生成列定義を取得する。
  • 生成列の式をターゲットリストに基づいて書き換え、new_qual の置換用リストに統合する。

    • 4. ReplaceVarsFromTargetList を使用して、new_qual 内の PRS2_NEW_VARNO をターゲットリストおよび生成列の式で置換する。 5. AddInvertedQual を呼び出し、加工済みの new_qual をクエリツリーに結合する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteHandler.c 内のルール処理ロジック（qual_product の生成時など）。
  • **呼び出し先**: copyObject, acquireLocksOnSubLinks, ChangeVarNodes, relation_open, get_generated_columns, ReplaceVarsFromTargetList, AddInvertedQual。

関連データ構造

  • Query: PostgreSQLのクエリツリー構造体。
  • acquireLocksOnSubLinks_context: サブクエリ内のロック管理用コンテキスト。
  • RangeTblEntry: 範囲テーブルエントリ。

失敗モードと境界条件

  • rule_qual がNULLの場合の挙動は本関数内では明示されていない。
  • rt_fetch が無効な rt_index を受け取った場合、システムエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は rewriteRuleAction と類似の処理（ロック取得や変数置換）を行っているため、将来的に共通化の検討が可能である。
  • 生成列の処理において、REPLACEVARS_CHANGE_VARNO と REPLACEVARS_SUBSTITUTE_NULL をコマンド種別に応じて使い分けている点に注意すること。
