InstrAggNode

定義

  • 種別: function
  • 定義箇所: src/backend/executor/instrument.c:168

目的

実行計画ノードの計測データ（Instrumentation）を別のノードの計測データへ集約・累積します。主に並列クエリ実行時に、ワーカープロセスの統計をリーダープロセスへ統合するために使用されます。

シグネチャまたは形状

void InstrAggNode(Instrumentation *dst, Instrumentation *add)

入力と出力

  • dst: 集約先の Instrumentation 構造体（入力および出力）。
  • add: 集約元の Instrumentation 構造体（入力）。

前提条件と不変条件

  • dst および add は有効な Instrumentation 構造体を指している必要があります。
  • dst が保持する need_bufusage および need_walusage フラグの状態に応じて、バッファおよび WAL の統計が加算されます。

アルゴリズムまたは振る舞い

1. **実行状態の統合**: add が実行中であれば dst も実行中としてマークし、firsttuple（最初のタプル取得時刻）を早い方の値で更新します。 2. **カウンタの累積**: 経過時間（counter）、タプル数（tuplecount, ntuples, ntuples2）、ループ回数（nloops）、フィルタリング統計（nfiltered1, nfiltered2）、および時間統計（startup, total）を add から dst へ加算します。 3. **リソース使用量の集約**: dst->need_bufusage が真であれば BufferUsageAdd を、dst->need_walusage が真であれば WalUsageAdd を呼び出し、リソース使用量の差分を累積します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c 内の並列クエリ処理ロジックから呼び出されます。
  • **呼び出し先**: BufferUsageAdd および WalUsageAdd を呼び出します。

関連データ構造

  • Instrumentation: 計測データを保持する主要構造体。
  • BufferUsage: バッファアクセス統計。
  • WalUsage: WAL 生成量統計。

失敗モードと境界条件

  • dst または add が NULL の場合、セグメンテーション違反が発生する可能性があります。
  • firsttuple の更新ロジックは、両方のノードが実行中である場合にのみ、より早い時刻を選択するよう制御されています。

保守担当者向けメモ

  • 本関数は Instrumentation 構造体の各フィールドを直接加算するため、構造体に新しい計測項目を追加した場合は、本関数にも加算処理を追加する必要があります。
  • BufferUsage や WalUsage の加算はフラグ制御されているため、計測対象のノードで適切にフラグが設定されていることを確認してください。
