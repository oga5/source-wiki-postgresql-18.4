unicode_is_normalized_quickcheck

定義

  • 種別: function
  • 定義箇所: src/common/unicode_norm.c:616

目的

入力文字列が指定された Unicode 正規化形式（NFC, NFKC, NFD, NFKD）に適合しているかを高速に判定する。

シグネチャまたは形状

UnicodeNormalizationQC unicode_is_normalized_quickcheck(UnicodeNormalizationForm form, const pg_wchar *input)

入力と出力

  • **入力**:
  • form: 判定対象の正規化形式（UnicodeNormalizationForm 型）。
  • input: 判定対象のヌル終端文字列（pg_wchar * 型）。
  • **出力**:
  • UNICODE_NORM_QC_YES: 文字列は既に正規化済みである。
  • UNICODE_NORM_QC_NO: 文字列は正規化されていない。
  • UNICODE_NORM_QC_MAYBE: 正規化されている可能性がある（詳細な正規化処理が必要）。

前提条件と不変条件

  • UNICODE_NFD および UNICODE_NFKD が指定された場合、常に UNICODE_NORM_QC_MAYBE を返す。
  • 文字列の走査中、結合クラス（Canonical Class）が降順（lastCanonicalClass > canonicalClass かつ canonicalClass != 0）になった場合、正規順序違反として直ちに UNICODE_NORM_QC_NO を返す。

アルゴリズムまたは振る舞い

1. form が NFD または NFKD の場合、即座に MAYBE を返す。 2. 文字列を先頭から順に走査する。 3. 各文字に対して get_canonical_class を呼び出し、正規順序の不変条件を検証する。 4. qc_is_allowed を呼び出し、各文字が指定された形式において正規化済みかを確認する。

  • NO が返された場合、即座に NO を返す。
  • MAYBE が返された場合、結果を MAYBE に更新し、走査を継続する。

    • 5. 全ての文字の検証が完了した後、最終的な判定結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/varlena.c:6700 での利用が確認されている。
  • **呼び出し先**:
  • get_canonical_class
  • qc_is_allowed

関連データ構造

  • UnicodeNormalizationForm: 正規化形式を定義する列挙型。
  • UnicodeNormalizationQC: 判定結果を保持する列挙型。
  • pg_wchar: PostgreSQL で使用されるワイド文字型。

失敗モードと境界条件

  • 入力文字列が空の場合、ループは実行されず初期値の YES が返される。
  • NFD/NFKD 形式のチェックは実装されておらず、常に MAYBE を返すことで、後続の低速な正規化処理へフォールバックさせる設計となっている。

保守担当者向けメモ

  • 本関数はバックエンドでの高速化を主目的としており、フロントエンドでの利用についてはソースコード内の FRONTEND マクロ等の条件分岐に注意すること。
  • qc_is_allowed の戻り値が MAYBE の場合、文字列全体の結果も MAYBE になる点に留意すること。
