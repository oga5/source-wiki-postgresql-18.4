get_all_vacuum_rels

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuum.c:1053

目的

データベース内の全リレーションを走査し、VACUUM 操作の対象となり得るテーブル（通常のテーブル、マテリアライズドビュー、パーティションテーブル）のリストを作成する。

シグネチャまたは形状

List *get_all_vacuum_rels(MemoryContext vac_context, int options)

入力と出力

  • **入力**:
  • vac_context: リレーションリストを割り当てるためのメモリコンテキスト。
  • options: VACUUM の実行オプション（権限チェックに使用）。
  • **出力**:
  • List *: VacuumRelation 構造体のリスト。

前提条件と不変条件

  • pg_class システムカタログに対する AccessShareLock を保持した状態で実行される。
  • 呼び出し元は、返されたリストのメモリ管理を vac_context を通じて制御する。

アルゴリズムまたは振る舞い

1. pg_class を開く。 2. table_beginscan_catalog を使用して全リレーションをスキャンする。 3. 各タプルに対し以下を実行する:

  • relkind が RELKIND_RELATION、RELKIND_MATVIEW、または RELKIND_PARTITIONED_TABLE であるか確認し、該当しない場合はスキップする。
  • vacuum_is_permitted_for_relation を呼び出し、現在のユーザーが対象リレーションを処理する権限を持つか確認する。
  • 権限がある場合、vac_context に切り替えて makeVacuumRelation を呼び出し、リストに追加する。

    • 4. スキャンを終了し、pg_class を閉じてリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/vacuum.c 内のメイン処理フロー（vacuum() 関数など）から呼び出される。
  • **呼び出し先**:
  • table_open, table_beginscan_catalog, heap_getnext, table_endscan, table_close (アクセス層)
  • vacuum_is_permitted_for_relation (権限チェック)
  • makeVacuumRelation (データ構造構築)
  • MemoryContextSwitchTo (メモリ管理)

関連データ構造

  • VacuumRelation: 本関数が生成するリストの要素。
  • Form_pg_class: pg_class のタプル構造。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • 権限不足のリレーションはリストから除外される。
  • pg_class のスキャン中にエラーが発生した場合、トランザクションの異常終了に伴いリソースが解放される。

保守担当者向けメモ

  • 本関数は VACUUM コマンドが対象を指定せずに実行された場合に、全テーブルを列挙するために使用される。
  • パーティションテーブルが含まれるため、後続の処理でそれらをどう扱うか（再帰的に処理するか、親のみを対象とするか）は呼び出し側のロジックに依存する。
