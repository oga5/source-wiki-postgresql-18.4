LookupFuncName

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_func.c:2143

目的

関数名と引数の型情報に基づき、システムカタログから対応する関数のOIDを検索する。

シグネチャまたは形状

Oid LookupFuncName(List *funcname, int nargs, const Oid *argtypes, bool missing_ok)

入力と出力

  • **入力**:
  • funcname: 関数名のリスト（修飾名を含む）。
  • nargs: 引数の数（-1は任意数を意味する）。
  • argtypes: 引数の型OIDの配列。
  • missing_ok: 関数が見つからない場合にエラーを発生させるか（false）、InvalidOid を返すか（true）を指定するフラグ。
  • **出力**:
  • 成功時: 特定された関数の Oid。
  • 失敗時: missing_ok が true の場合は InvalidOid。それ以外はエラーにより処理中断。

前提条件と不変条件

  • LookupFuncNameInternal を呼び出し、その結果に基づいてエラーハンドリングを行う。
  • FUNCLOOKUP_AMBIGUOUS が発生した場合は、missing_ok の値に関わらず常にエラーを投げる。

アルゴリズムまたは振る舞い

1. LookupFuncNameInternal を呼び出し、関数検索を実行する。 2. 検索結果が有効なOIDであれば、そのOIDを返す。 3. 検索結果が無効な場合、FuncLookupError の値に応じて分岐する。

  • FUNCLOOKUP_NOSUCHFUNC: missing_ok が true なら InvalidOid を返す。false なら引数の有無に応じて適切なエラーメッセージを生成し ereport(ERROR) を発行する。
  • FUNCLOOKUP_AMBIGUOUS: 関数名が曖昧であるため、常に ereport(ERROR) を発行する。

呼び出し元と呼び出し先

  • **呼び出し先**: LookupFuncNameInternal
  • **呼び出し元**:
  • src/backend/commands/amcmds.c
  • src/backend/commands/conversioncmds.c
  • src/backend/commands/event_trigger.c
  • src/backend/commands/foreigncmds.c
  • src/backend/commands/functioncmds.c

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • List: 関数名を保持するリスト構造。
  • FuncLookupError: 検索結果のステータスを示す列挙型。

失敗モードと境界条件

  • **関数未定義**: FUNCLOOKUP_NOSUCHFUNC が発生。missing_ok が false の場合、エラーとして処理される。
  • **曖昧な関数名**: FUNCLOOKUP_AMBIGUOUS が発生。引数リストの指定を促すエラーが発行される。

保守担当者向けメモ

  • 本関数は、主にDDLコマンドやシステムカタログ操作において、特定の関数を名前で解決するために使用される。
  • 構文解析器のメインフロー（ParseFuncOrColumn）とは異なり、直接的なカタログ検索インターフェースとして機能する。
