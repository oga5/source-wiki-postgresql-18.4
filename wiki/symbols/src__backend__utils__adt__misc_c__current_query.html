current_query

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/misc.c:211

目的

現在実行中のクエリ文字列をテキスト形式で取得します。

シグネチャまたは形状

Datum current_query(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数）。
  • **出力**: 現在のクエリ文字列（text型）。クエリ文字列が利用できない場合は NULL を返します。

前提条件と不変条件

  • グローバル変数 debug_query_string が有効なクエリ文字列を保持している必要があります。

アルゴリズムまたは振る舞い

1. グローバル変数 debug_query_string が非NULLか確認します。 2. 非NULLの場合、cstring_to_text を使用してC文字列をPostgreSQLの text 型に変換し、PG_RETURN_TEXT_P で返却します。 3. NULLの場合、PG_RETURN_NULL() を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/dblink/dblink.c 内の関数から、組み込み関数 current_query() のエイリアスとして呼び出されています。
  • **呼び出し先**: cstring_to_text（文字列変換用ユーティリティ）。

関連データ構造

  • Datum: PostgreSQLの汎用データ表現。
  • debug_query_string: 現在のバックエンドで実行中のクエリを保持するグローバル変数。

失敗モードと境界条件

  • debug_query_string が設定されていない場合、関数は安全に NULL を返します。

保守担当者向けメモ

  • 本関数は debug_query_string に依存しています。より詳細なクエリ情報が必要な場合でも、現時点ではこの変数以外の簡潔なアクセス手段は提供されていません。
