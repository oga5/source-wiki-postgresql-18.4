sortouts_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:728

目的

NFA（非決定性有限オートマトン）の遷移（arc）のリストをソートするための比較関数を提供します。

シグネチャまたは形状

static int sortouts_cmp(const void *a, const void *b)

入力と出力

  • **入力**: a, b は struct arc * へのポインタを指す void * 型のポインタ。
  • **出力**: a が b より小さい場合は負の整数、大きい場合は正の整数、等しい場合は 0 を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な struct arc * を指している必要があります。
  • 比較対象の struct arc 構造体には to（遷移先状態）、co（色）、type（遷移タイプ）の各フィールドが存在する必要があります。

アルゴリズムまたは振る舞い

以下の優先順位でフィールドを比較し、順序を決定します。 1. 遷移先状態の番号 (to->no) 2. 遷移の色 (co) 3. 遷移のタイプ (type)

この順序は、NFA の遷移を正規化または整理する際に、状態の接続関係を確定させるために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_nfa.c:709 (qsort の比較関数として使用)
  • src/backend/regex/regc_nfa.c:1120
  • src/backend/regex/regc_nfa.c:1217
  • **呼び出し先**:
  • 直接的な関数呼び出しは行いません。

関連データ構造

  • struct arc: NFA の遷移を表現する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合や、指し示す先の struct arc が初期化されていない場合は未定義の動作となります。
  • 比較フィールドがすべて等しい場合、0 を返し、ソート順序は維持されます。

保守担当者向けメモ

  • 本関数は qsort と組み合わせて使用されるため、比較ロジックを変更する場合は、NFA の最適化パイプライン全体における遷移の順序付け要件に影響がないか確認してください。
