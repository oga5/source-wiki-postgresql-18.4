gin_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/gindesc.c:179

目的

GIN（Generalized Inverted Index）インデックスに関連するWALレコードの info フィールドを解析し、対応する操作の種類を識別するための文字列を返します。

シグネチャまたは形状

const char *gin_identify(uint8 info);

入力と出力

  • **入力**: uint8 info - WALレコードの info ビットフィールド。
  • **出力**: const char * - 操作の種類を示す文字列（例: "INSERT", "SPLIT"）。該当する操作がない場合は NULL を返します。

前提条件と不変条件

  • 入力された info ビットから XLR_INFO_MASK を除外した値に基づいて判定を行います。
  • GINインデックスのWALレコード構造（access/ginxlog.h で定義）と整合している必要があります。

アルゴリズムまたは振る舞い

1. info から XLR_INFO_MASK をマスクして、GIN固有の操作タイプを抽出します。 2. switch 文を用いて、以下のいずれかの定数と照合します。

  • XLOG_GIN_CREATE_PTREE
  • XLOG_GIN_INSERT
  • XLOG_GIN_SPLIT
  • XLOG_GIN_VACUUM_PAGE
  • XLOG_GIN_VACUUM_DATA_LEAF_PAGE
  • XLOG_GIN_DELETE_PAGE
  • XLOG_GIN_UPDATE_META_PAGE
  • XLOG_GIN_INSERT_LISTPAGE
  • XLOG_GIN_DELETE_LISTPAGE

    • 3. 一致した操作名に対応する文字列リテラルを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/access/rmgrlist.h において、GINのリソースマネージャ定義（PG_RMGR）の一部として登録されています。
  • **呼び出し先**:
  • 提供されたソース情報には、本関数が内部で呼び出している関数はありません。

関連データ構造

  • access/ginxlog.h: GINのWALレコードタイプ定義。

失敗モードと境界条件

  • info が定義済みの XLOG_GIN_* 定数のいずれにも該当しない場合、NULL を返します。

保守担当者向けメモ

  • 本関数はWALレコードの識別のみを行う読み取り専用のルーチンです。
  • 新しいGIN WAL操作を追加した場合は、本関数の switch 文にもケースを追加する必要があります。
