build_tlist_to_deparse

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/deparse.c:1173

目的

リモートサーバーへ送信するクエリのターゲットリスト（tlist）を構築します。リモートリレーションから取得すべき列、およびローカル条件の評価に必要な列を特定します。

シグネチャまたは形状

List *build_tlist_to_deparse(RelOptInfo *foreignrel)

入力と出力

  • **入力**: RelOptInfo *foreignrel（対象となる外部リレーションの情報）
  • **出力**: List *（リモートクエリで要求される式を含むターゲットリスト）

前提条件と不変条件

  • foreignrel->fdw_private に PgFdwRelationInfo 構造体が格納されている必要があります。
  • foreignrel がアッパーリレーション（IS_UPPER_REL）である場合、既に構築済みの fpinfo->grouped_tlist が返されます。

アルゴリズムまたは振る舞い

1. **アッパーリレーションの判定**: IS_UPPER_REL(foreignrel) が真であれば、事前に計算済みの grouped_tlist を即座に返します。 2. **ベースリレーションのターゲットリスト構築**:

  • foreignrel->reltarget->exprs に含まれる変数を抽出し、フラットなターゲットリストに追加します。
  • fpinfo->local_conds を走査し、各 RestrictInfo 内の句から必要な変数を抽出してターゲットリストに追加します。

    • 3. **結果の返却**: 収集したすべての式を含むリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c:1347
  • contrib/postgres_fdw/postgres_fdw.c:3161
  • **呼び出し先**:
  • add_to_flat_tlist
  • pull_var_clause
  • lfirst_node

関連データ構造

  • RelOptInfo: プランニング中のリレーション情報。
  • PgFdwRelationInfo: postgres_fdw 固有のリレーション情報。
  • RestrictInfo: クエリの制約条件。

失敗モードと境界条件

  • foreignrel が NULL の場合や、fdw_private が正しく初期化されていない場合は異常終了する可能性があります。
  • 抽出対象の式に PVC_RECURSE_PLACEHOLDERS が適用されるため、プレースホルダーノードが含まれる場合は再帰的に処理されます。

保守担当者向けメモ

  • 本関数は postgres_fdw における列のプッシュダウン判断の基礎となります。ターゲットリストの構成に変更を加える際は、local_conds の評価に必要な列が漏れなく含まれているか確認してください。
