iovec

定義

  • 種別: struct
  • 定義箇所: src/include/port/pg_iovec.h:25

目的

複数のメモリバッファを一度の I/O 操作（ベクトル I/O）で扱うための構造体です。OS 固有の差異を吸収し、PostgreSQL がサポートする全環境で一貫したインターフェースを提供します。

シグネチャまたは形状

struct iovec
{
    void       *iov_base; /* バッファの開始アドレス */
    size_t      iov_len;  /* バッファのサイズ */
};

入力と出力

  • 入力: iov_base（データ領域へのポインタ）、iov_len（データ長）。
  • 出力: なし（データ構造体）。

前提条件と不変条件

  • PG_IOV_MAX（Min(IOV_MAX, 128)）が、スタック上に確保する際の安全な上限値として定義されています。これを超える配列を扱う場合はヒープメモリの使用が推奨されます。

アルゴリズムまたは振る舞い

  • ベクトル I/O 操作において、複数の iovec 構造体を配列として並べることで、散布（scatter）または集約（gather）I/O を実現します。
  • preadv/pwritev が利用可能な環境では直接呼び出され、利用不可な環境では pread/pwrite を用いたループ処理によってエミュレーションされます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/storage/aio/aio_funcs.c: iov_byte_length 関数内でのサイズ計算や、iov_copy 配列の定義に使用。
  • src/backend/storage/aio/aio_init.c: pgaio_ctl->iovecs のメモリ確保および初期化に使用。
  • src/backend/storage/aio/aio_io.c: pgaio_io_get_iovec 関数での取得や、pgaio_ctl->iovecs へのアクセスに使用。
  • 呼び出し先: 情報なし。

関連データ構造

  • PG_IOV_MAX: iovec 配列の安全な上限値。
  • PgAioHandle: 非同期 I/O において iovec を関連付けるハンドル構造体。

失敗モードと境界条件

  • Windows 環境では、pg_preadv/pg_pwritev を使用するとファイルポインタが移動する副作用があるため注意が必要です。
  • スタックオーバーフローを避けるため、PG_IOV_MAX を超える要素数をスタック上に確保してはなりません。

保守担当者向けメモ

  • 本構造体は sys/uio.h が存在しない環境や不完全な環境を補完するために定義されています。
  • IWYU（Include What You Use）の観点から、このヘッダーをインクルードすることで必要な型定義が確実に利用可能になります。
