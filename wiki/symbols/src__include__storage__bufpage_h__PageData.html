PageData

定義

  • 種別: typedef
  • 定義箇所: src/include/storage/bufpage.h:81

目的

PostgreSQLにおけるディスク上の物理的なページ（ブロック）の内容を表現するための型定義です。

シグネチャまたは形状

typedef char PageData;

入力と出力

  • **入力**: 本型は char の別名であり、ページ全体を指すメモリ上のバイト配列として扱われます。
  • **出力**: ページ操作関数（PageGetFreeSpace 等）において、ページの内容を読み取るためのポインタとして渡されます。

前提条件と不変条件

  • PageData は、PageHeaderData を先頭に持つ「スロット付きページ（Slotted Page）」構造に従うメモリ領域を指すことが前提です。
  • ページ内のデータは、ラインポインタ（先頭側）とタプル本体（末尾側）が互いに向かって配置されるレイアウトを維持する必要があります。

アルゴリズムまたは振る舞い

  • 本型自体はデータ型定義ですが、この型へのポインタはページ内の空き領域計算、整合性検証、一時的なページコピー作成などの操作において、ページレイアウトの基点として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/transam/xloginsert.c: WALログ挿入時のバックアップブロック処理で使用。
  • src/backend/storage/page/bufpage.c: ページ検証、空き領域計算、一時ページ作成関数で使用。
  • **呼び出し先**:
  • 本型はプリミティブな型定義であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • PageHeaderData: ページ先頭に配置されるメタデータ構造体。
  • ItemIdData: ページ内のタプル位置を特定するラインポインタ。
  • Page: PageData * として定義される、ページを指すポインタ型。

失敗モードと境界条件

  • PageData を指すポインタが NULL である場合や、割り当てられたバッファサイズがページサイズ（BLCKSZ）に満たない場合、ページ操作関数内でメモリ破壊や不正アクセスが発生する可能性があります。

保守担当者向けメモ

  • PageData は単なる char の別名であるため、型安全性を確保するために Page 型（char *）を介した操作が推奨されます。
  • xloginsert.c 等で const PageData * として扱われている箇所は、ページ内容の読み取り専用アクセスであることを示しています。
