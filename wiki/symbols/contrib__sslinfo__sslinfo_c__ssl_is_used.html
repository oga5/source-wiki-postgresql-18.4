ssl_is_used

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:45

目的

現在のPostgreSQLセッションにおいて、SSL接続が使用されているかどうかを判定し、その結果をSQL層へ返します。

シグネチャまたは形状

Datum ssl_is_used(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQLの関数呼び出し規約に基づく引数）。
  • **出力**: bool 型（Datumとして返却）。SSLが使用されている場合は true、それ以外は false。

前提条件と不変条件

  • グローバル変数 MyProcPort が初期化されている必要があります。
  • 本関数は MyProcPort 構造体内の ssl_in_use フラグを参照するため、バックエンドプロセス内での実行を前提とします。

アルゴリズムまたは振る舞い

1. グローバル変数 MyProcPort にアクセスします。 2. MyProcPort->ssl_in_use の値を読み取ります。 3. PG_RETURN_BOOL マクロを使用して、その値をPostgreSQLのブール型として呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で本関数が呼び出されます。ソースコード内では PG_FUNCTION_INFO_V1(ssl_is_used) マクロによる登録が確認されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。MyProcPort 構造体のメンバを参照するのみです。

関連データ構造

  • Port 構造体（MyProcPort として参照されるバックエンドのポート情報）。

失敗モードと境界条件

  • MyProcPort が NULL の環境（バックエンドプロセス外など）で呼び出された場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は単純なフラグの読み取りを行うラッパーです。SSL状態の判定ロジック自体は libpq 側の接続確立処理に依存しています。
