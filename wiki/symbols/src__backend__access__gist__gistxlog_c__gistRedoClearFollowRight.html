gistRedoClearFollowRight

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistxlog.c:39

目的

GiSTインデックスのページ分割処理において、特定のインデックスページから「Follow-Right」フラグを解除し、NSN（Node Sequence Number）を更新することで、WALリプレイ時の整合性を維持する。

シグネチャまたは形状

static void gistRedoClearFollowRight(XLogReaderState *record, uint8 block_id)

入力と出力

  • record: リプレイ対象のWALレコードを保持する XLogReaderState へのポインタ。
  • block_id: 対象となるバッファブロックの識別子。
  • 出力: なし（副作用としてバッファの内容を更新）。

前提条件と不変条件

  • XLogReadBufferForRedo が対象ブロックを正常に読み込めること。
  • ページがフルページイメージから復元された場合であっても、NSNの更新が必要であるため、本関数による処理が必須となる。

アルゴリズムまたは振る舞い

1. XLogReadBufferForRedo を呼び出し、対象ブロックのバッファを取得する。 2. アクションが BLK_NEEDS_REDO または BLK_RESTORED の場合、以下の処理を行う：

  • GistPageSetNSN を使用して、ページのNSNをWALレコードの終了LSNに更新する。
  • GistClearFollowRight を呼び出し、ページ内のFollow-Rightフラグをクリアする。
  • PageSetLSN を使用して、ページのLSNを更新する。
  • MarkBufferDirty を呼び出し、バッファをダーティ状態にする。

    • 3. バッファが有効な場合、UnlockReleaseBuffer を呼び出してロックを解放する。

関連データ構造

  • XLogReaderState: WALレコードの読み取り状態を管理する構造体。
  • Buffer: PostgreSQLのバッファ管理用ハンドル。
  • Page: インデックスページのメモリ表現。

失敗モードと境界条件

  • XLogReadBufferForRedo がバッファを返さない場合（BLK_NOTFOUND 等）、ページ更新はスキップされる。
  • バッファが有効でない場合はロック解放処理を行わない。

保守担当者向けメモ

  • 本関数はGiSTのページ分割リプレイにおいて、右隣のページへのポインタを無効化する重要な役割を担う。
  • NSNはフルページイメージには含まれないため、イメージ復元後も必ず本関数を通過させてNSNを最新化する必要がある点に注意すること。
