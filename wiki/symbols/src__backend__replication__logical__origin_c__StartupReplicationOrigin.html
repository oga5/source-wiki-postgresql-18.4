StartupReplicationOrigin

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/origin.c:721

目的

チェックポイントファイルから論理レプリケーションの進捗状態を読み込み、共有メモリ上の replication_states 配列へ復元します。

シグネチャまたは形状

void StartupReplicationOrigin(void)

入力と出力

  • **入力**: 外部ファイル PG_REPLORIGIN_CHECKPOINT_FILENAME。
  • **出力**: なし（副作用として共有メモリ replication_states を更新）。

前提条件と不変条件

  • max_active_replication_origins が 0 の場合、本関数は即座に終了します。
  • 共有メモリの初期化後に一度だけ呼び出されることを想定しています（USE_ASSERT_CHECKING 有効時は already_started フラグで保護）。
  • 呼び出し時に他のプロセスによる共有メモリへのアクセスがないことが前提です。

アルゴリズムまたは振る舞い

1. チェックポイントファイルを開き、マジックナンバー（REPLICATION_STATE_MAGIC）を検証します。 2. ファイルの末尾に到達するまで ReplicationStateOnDisk 構造体を順次読み込みます。 3. 読み込んだ各状態を共有メモリの replication_states 配列にコピーします。 4. 読み込み中に CRC32C を計算し、ファイルの末尾に記録されたチェックサムと比較してデータの整合性を検証します。 5. 整合性エラーやファイル読み込みエラーが発生した場合は PANIC で停止します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c:5671（データベース起動時のリカバリプロセス）。
  • **呼び出し先**: OpenTransientFile, read, CloseTransientFile などのファイル操作関数、および COMP_CRC32C 等のチェックサム計算マクロ。

関連データ構造

  • ReplicationStateOnDisk: ディスク上の永続化フォーマット。
  • replication_states: 共有メモリ上の進捗管理配列。

失敗モードと境界条件

  • **ファイル不在**: ENOENT の場合は正常終了（初期状態とみなす）。
  • **マジックナンバー不一致**: PANIC を発生させます。
  • **データ破損**: チェックサム不一致時に PANIC を発生させます。
  • **容量超過**: 復元する状態数が max_active_replication_origins を超える場合、ERRCODE_CONFIGURATION_LIMIT_EXCEEDED で PANIC します。

保守担当者向けメモ

  • 本関数は起動時の単一スレッド実行を前提としており、ロックを取得せずに共有メモリを操作しています。
  • max_active_replication_origins の設定値が前回の実行時と異なる場合、復元処理に影響が出る可能性があるため注意してください。
