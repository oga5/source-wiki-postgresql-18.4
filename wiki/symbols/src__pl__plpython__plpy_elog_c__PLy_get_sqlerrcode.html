PLy_get_sqlerrcode

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:360

目的

Python 例外オブジェクトから sqlstate 属性を抽出し、PostgreSQL のエラーコード（int）へ変換して格納する。

シグネチャまたは形状

static void PLy_get_sqlerrcode(PyObject *exc, int *sqlerrcode)

入力と出力

  • **入力**:
  • exc: 属性 sqlstate を持つ可能性のある Python 例外オブジェクト。
  • sqlerrcode: 変換後の SQLSTATE を格納する整数型ポインタ。
  • **出力**: なし（sqlerrcode が指すメモリ領域を更新する）。

前提条件と不変条件

  • exc は有効な Python オブジェクトであること。
  • sqlerrcode は書き込み可能な有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. PyObject_GetAttrString を使用して exc から sqlstate 属性を取得する。属性が存在しない場合は即座に終了する。 2. PLyUnicode_AsString を使用して sqlstate を文字列バッファに変換する。 3. 文字列が長さ 5 であり、かつ英数字のみで構成されているか検証する。 4. 条件を満たす場合、MAKE_SQLSTATE マクロを用いて 5 文字の文字列を PostgreSQL 内部の整数表現に変換し、*sqlerrcode に代入する。 5. 最後に Py_DECREF を呼び出し、取得した sqlstate オブジェクトの参照カウントを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_elog.c 内の関数（412行目、434行目付近）。
  • **呼び出し先**: PyObject_GetAttrString, PLyUnicode_AsString, strlen, strspn, MAKE_SQLSTATE, Py_DECREF。

関連データ構造

  • PyObject: Python の汎用オブジェクト型。
  • sqlstate: PostgreSQL のエラーコードを表現する 5 文字の文字列。

失敗モードと境界条件

  • sqlstate 属性が存在しない場合、何も行わず正常終了する。
  • sqlstate の長さが 5 ではない、または英数字以外が含まれる場合、sqlerrcode は更新されない。

保守担当者向けメモ

  • 本関数は Python の例外オブジェクトを PostgreSQL のエラーシステムに適合させるためのユーティリティである。
  • MAKE_SQLSTATE マクロの定義に依存しているため、PostgreSQL のエラーコード体系に変更があった場合は注意が必要である。
