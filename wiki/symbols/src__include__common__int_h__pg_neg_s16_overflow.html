pg_neg_s16_overflow

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:120

目的

int16 型の値を符号反転（否定）する際、オーバーフローが発生するかを判定し、安全に演算を行う。

シグネチャまたは形状

static inline bool pg_neg_s16_overflow(int16 a, int16 *result)

入力と出力

  • **入力**:
  • a: 符号反転対象の int16 値。
  • result: 演算結果を格納するポインタ。
  • **出力**:
  • 戻り値: オーバーフローが発生した場合は true、それ以外は false。
  • *result: 演算成功時は -a の値。オーバーフロー時はダミー値 0x5EED が格納される。

前提条件と不変条件

  • int16 型の範囲において、PG_INT16_MIN を反転させるとオーバーフローが発生する。

アルゴリズムまたは振る舞い

1. コンパイラが HAVE__BUILTIN_OP_OVERFLOW をサポートしている場合、__builtin_sub_overflow(0, a, result) を使用して演算と判定を行う。 2. サポートしていない場合:

  • a が PG_INT16_MIN と等しいか判定する。
  • 等しい場合、*result に 0x5EED を代入し、true を返す。
  • それ以外の場合、*result に -a を代入し、false を返す。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定できません。
  • 呼び出し先: コンパイラ組み込み関数（__builtin_sub_overflow）または標準的な算術演算。

関連データ構造

  • int16: 16ビット符号付き整数型。

失敗モードと境界条件

  • **境界条件**: a が PG_INT16_MIN（-32768）の場合、符号反転後の値（+32768）は int16 の範囲（-32768〜32767）を超えるため、オーバーフローとして扱われる。
