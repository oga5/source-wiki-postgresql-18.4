SH_COMPUTE_SIZE

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/lib/simplehash.h:310

目的

ハッシュテーブルのバケット数を、2のべき乗かつ指定されたサイズ以上となる最小の値に正規化し、メモリ割り当ての妥当性を検証する。

シグネチャまたは形状

static inline uint64 SH_COMPUTE_SIZE(uint64 newsize)

入力と出力

  • **入力**: newsize (uint64) - 要求される最小バケット数。
  • **出力**: uint64 - 調整後のバケット数。

前提条件と不変条件

  • newsize は 2 以上として扱われる（Max(newsize, 2) による補正）。
  • 戻り値は常に 2 のべき乗である。
  • 戻り値は SH_MAX_SIZE 以下でなければならない（Assert による検証）。

アルゴリズムまたは振る舞い

1. newsize と 2 の大きい方を採用する。 2. pg_nextpower2_64 を呼び出し、値を 2 のべき乗に切り上げる。 3. SH_MAX_SIZE を超えていないかアサーションで確認する。 4. SH_ELEMENT_TYPE のサイズと計算されたバケット数を乗算し、SIZE_MAX / 2 を超える場合は sh_error を呼び出してエラーを報告する。 5. 妥当なサイズを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/lib/simplehash.h 内のハッシュテーブル初期化およびリサイズ処理（SH_GROW 等に関連する箇所）。
  • **呼び出し先**:
  • pg_nextpower2_64（port/pg_bitutils.h）
  • sh_error（エラー報告用）

関連データ構造

  • SH_ELEMENT_TYPE: ハッシュテーブルに格納される要素型。本関数内でのメモリサイズ計算に使用される。

失敗モードと境界条件

  • **エラー**: (sizeof(SH_ELEMENT_TYPE) * size) >= SIZE_MAX / 2 となる場合、メモリ割り当てが不可能と判断され sh_error が発生する。
  • **境界**: newsize が 0 や 1 の場合でも、最小値 2 が保証される。

保守担当者向けメモ

  • 本関数は simplehash.h のマクロテンプレートによって生成されるコードの一部である。
  • SH_ELEMENT_TYPE はコンパイル時に決定されるため、この関数もその型に依存したコードとして展開される。
  • SIZE_MAX / 2 を閾値としているのは、メモリ割り当て時のオーバーフローを安全に防ぐためである。
