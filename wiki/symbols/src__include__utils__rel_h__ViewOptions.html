ViewOptions

定義

  • 種別: struct
  • 定義箇所: src/include/utils/rel.h:424

目的

ビュー（View）に関連するオプション設定を保持するためのデータ構造です。リレーションのメタデータとして、セキュリティや更新チェックに関する動作を規定します。

シグネチャまたは形状

typedef struct ViewOptions
{
    int32        vl_len_;        /* varlena header */
    bool         security_barrier;
    bool         security_invoker;
    ViewOptCheckOption check_option;
} ViewOptions;

入力と出力

  • 本構造体は、リレーションの rd_options フィールドに bytea として格納されます。
  • 読み出し時には ViewOptions 型へのキャストが行われ、各フィールド（security_barrier, security_invoker, check_option）が参照されます。

前提条件と不変条件

  • vl_len_ フィールドは可変長データ（varlena）のヘッダーであるため、直接操作してはなりません。
  • rd_options が NULL の場合、各フィールドへのアクセスは安全に行われる必要があります（マクロ定義では false やデフォルト値を返すようガードされています）。

アルゴリズムまたは振る舞い

  • ビューのセキュリティおよび更新チェックの挙動を制御します。
  • src/include/utils/rel.h 内で定義されたマクロを通じて、RelationData から各オプション値を抽出するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/backend/access/common/reloptions.c: オプションのオフセットおよびサイズの定義に使用。
  • src/include/utils/rel.h: rd_options から各フラグを取得するマクロ内で使用。
  • **呼び出し先**:
  • 情報なし。

関連データ構造

  • RelationData: 本構造体を rd_options として保持する親構造体。
  • ViewOptCheckOption: check_option フィールドの型。

失敗モードと境界条件

  • rd_options が ViewOptions 構造体として正しく初期化されていない場合、キャスト後のフィールド参照は未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は bytea としてシリアライズされるため、フィールドの追加や変更を行う際は reloptions.c 側の定義と整合性を取る必要があります。
  • 直接アクセスせず、提供されているアクセスマクロ経由での利用を推奨します。
