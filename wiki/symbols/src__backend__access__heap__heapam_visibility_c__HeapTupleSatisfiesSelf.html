HeapTupleSatisfiesSelf

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/heap/heapam_visibility.c:169

目的

現在のトランザクション自身によって作成されたタプルが、現在のトランザクションから可視であるかを判定します。

シグネチャまたは形状

bool HeapTupleSatisfiesSelf(HeapTuple htup, Snapshot snapshot, Buffer buffer)

入力と出力

  • **入力**:
  • htup: 判定対象のタプル。
  • snapshot: 使用されるスナップショット（本関数内では主に型チェックや整合性確認に使用）。
  • buffer: タプルが格納されているバッファ。
  • **出力**:
  • bool: タプルが可視であれば true、不可視であれば false を返します。

前提条件と不変条件

  • htup->t_self が有効であること。
  • htup->t_tableOid が InvalidOid ではないこと。
  • 呼び出し元は、対象のバッファに対して少なくとも共有ロックを保持している必要があります。

アルゴリズムまたは振る舞い

1. **Xminの確認**: タプルの挿入トランザクション（Xmin）がコミット済みかを確認します。

  • 未コミットかつ無効な場合、不可視（false）。
  • 9.0以前のバイナリアップグレード用フラグ（HEAP_MOVED_OFF, HEAP_MOVED_IN）を処理します。
  • 現在のトランザクションが挿入したタプルである場合、Xmaxの状態（削除やロック）を評価して可視性を決定します。
  • 挿入トランザクションが進行中の場合、不可視（false）。
  • 挿入トランザクションがコミット済みの場合、ヒントビットを更新し、Xmaxの判定へ進みます。

    • 2. **Xmaxの確認**: タプルの削除またはロック状態（Xmax）を評価します。

  • Xmaxが無効またはアボート済みの場合、可視（true）。
  • Xmaxがコミット済みの場合、ロックのみであれば可視（true）、更新済みであれば不可視（false）。
  • マルチトランザクション（HEAP_XMAX_IS_MULTI）の場合、更新トランザクションの状態に応じて判定します。
  • 判定過程でトランザクション状態が確定した場合、SetHintBits を呼び出してバッファを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_visibility.c 内の他の可視性判定ロジックから呼び出されます。
  • **呼び出し先**: SetHintBits, TransactionIdIsCurrentTransactionId, TransactionIdIsInProgress, TransactionIdDidCommit 等のトランザクション管理関数。

関連データ構造

  • HeapTuple: タプル本体およびヘッダー情報。
  • Snapshot: 可視性判定の基準となるスナップショット構造体。
  • Buffer: タプルが保持されるバッファ。

失敗モードと境界条件

  • 挿入トランザクションがアボートまたはクラッシュしている場合、HEAP_XMIN_INVALID を設定し false を返します。
  • 削除トランザクションがアボートまたはクラッシュしている場合、HEAP_XMAX_INVALID を設定し true を返します。
  • 9.0以前の移行用フラグが設定されている場合、TransactionId の状態に応じて特殊な判定を行います。

保守担当者向けメモ

  • 本関数は、判定の過程で SetHintBits を呼び出し、バッファをダーティにする副作用があります。
  • TransactionIdIsInProgress を TransactionIdDidCommit より先にチェックする順序は、レースコンディションを防ぐために厳格に守る必要があります。
