getTypeOutputInfo

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:3084

目的

指定されたデータ型の出力関数（OID）および、その型が可変長（varlena）であるかどうかの情報をシステムカタログキャッシュから取得する。

シグネチャまたは形状

void getTypeOutputInfo(Oid type, Oid *typOutput, bool *typIsVarlena)

入力と出力

  • **入力**:
  • type (Oid): 情報を取得する対象のデータ型OID。
  • typOutput (Oid *): 出力関数OIDを格納するポインタ。
  • typIsVarlena (bool *): 可変長フラグを格納するポインタ。
  • **出力**: なし（引数ポインタ経由で値を返却）。

前提条件と不変条件

  • type に対応する pg_type カタログエントリが存在すること。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して TYPEOID キャッシュから対象型の HeapTuple を検索する。 2. キャッシュミス時は elog(ERROR) を発生させる。 3. GETSTRUCT マクロで Form_pg_type 構造体へキャストし、以下の検証を行う：

  • pt->typisdefined が偽であればエラー。
  • pt->typoutput が無効なOIDであればエラー。

    • 4. 抽出した pt->typoutput を *typOutput に代入する。 5. pt->typbyval が偽（値渡し不可）かつ pt->typlen が -1（可変長）である場合、*typIsVarlena を真に設定する。 6. ReleaseSysCache を呼び出し、キャッシュの参照カウントを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_io.c
  • contrib/pageinspect/brinfuncs.c, gistfuncs.c
  • contrib/pg_trgm/trgm_op.c
  • contrib/postgres_fdw/deparse.c, postgres_fdw.c
  • contrib/test_decoding/test_decoding.c
  • src/backend/access/brin/brin_minmax_multi.c
  • **呼び出し先**:
  • SearchSysCache1
  • ReleaseSysCache
  • elog / ereport

関連データ構造

  • Form_pg_type: pg_type システムカタログの行構造体。

失敗モードと境界条件

  • **キャッシュ検索失敗**: elog(ERROR) によりトランザクションがアボートされる。
  • **シェル型**: typisdefined が偽の場合、ERRCODE_UNDEFINED_OBJECT でエラー報告。
  • **出力関数未定義**: typoutput が無効な場合、ERRCODE_UNDEFINED_FUNCTION でエラー報告。

保守担当者向けメモ

  • 本関数は pg_type カタログの特定のフィールドを抽出するラッパーである。
  • 戻り値の型情報（typIsVarlena）は、PostgreSQLの内部的なデータ表現（byval と typlen）に基づいている。
