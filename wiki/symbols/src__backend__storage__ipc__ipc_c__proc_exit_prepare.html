proc_exit_prepare

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipc.c:165

目的

プロセス終了シーケンスの準備および実行を行う内部関数。終了処理中フラグのセット、割り込みの無効化、共有メモリのクリーンアップ、および登録済み終了コールバックの実行を順次行う。

シグネチャまたは形状

static void proc_exit_prepare(int code)

入力と出力

  • **入力**: code (終了ステータスコード)
  • **出力**: なし（副作用としてプロセスの終了処理を実行）

前提条件と不変条件

  • 本関数が呼び出されると proc_exit_inprogress が true に設定され、以降の ereport() はメインループへ戻らず本関数へ制御を戻す。
  • 割り込みフラグ（InterruptPending, ProcDiePending, QueryCancelPending）は false にリセットされる。
  • error_context_stack および debug_query_string は NULL にリセットされる。

アルゴリズムまたは振る舞い

1. proc_exit_inprogress を true に設定し、終了処理の不可逆性を確定させる。 2. 割り込みフラグを無効化し、InterruptHoldoffCount を 1 に設定して割り込みを抑制する。 3. エラーコンテキストスタックとデバッグ用クエリ文字列をクリアする。 4. shmem_exit(code) を呼び出し、共有メモリ関連のクリーンアップを実行する。 5. 登録された終了コールバック（on_proc_exit_list）を LIFO（後入れ先出し）順で実行する。

  • コールバック実行中にエラーが発生した場合、再帰的な無限ループを避けるため、実行前にリストから削除される設計となっている。

    • 6. 最後に on_proc_exit_index を 0 にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/ipc.c 内の proc_exit() から呼び出される。
  • **呼び出し先**:
  • shmem_exit()
  • on_proc_exit_list に登録された各コールバック関数

関連データ構造

  • proc_exit_inprogress (終了処理中フラグ)
  • on_proc_exit_list (終了コールバックの配列)
  • on_proc_exit_index (コールバック配列のインデックス)
  • InterruptPending, ProcDiePending, QueryCancelPending (割り込み制御フラグ)

失敗モードと境界条件

  • コールバック内で ereport(ERROR) または FATAL が発生した場合、そのコールバックは再実行されず、終了処理が継続される。
  • 終了処理中に elog(FATAL) 等が発生しても、proc_exit_inprogress により終了処理が優先される。

保守担当者向けメモ

  • 本関数はプロセス終了のクリティカルパスである。コールバックの追加や変更を行う際は、LIFO順序とエラー発生時の再帰防止設計を維持すること。
  • InterruptHoldoffCount を 1 に設定しているため、本関数内での割り込み処理は抑制される。
