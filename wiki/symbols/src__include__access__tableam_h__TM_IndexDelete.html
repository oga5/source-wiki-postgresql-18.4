TM_IndexDelete

定義

  • 種別: struct
  • 定義箇所: src/include/access/tableam.h:209

目的

インデックスから削除対象となるタプルの物理的な位置（TID）と、関連するインデックス状態の管理情報を保持するために使用されます。

シグネチャまたは形状

typedef struct TM_IndexDelete
{
	ItemPointerData tid;		/* table TID from index tuple */
	int16		id;				/* Offset into TM_IndexStatus array */
} TM_IndexDelete;

入力と出力

  • **入力**: インデックスから取得されたタプルの識別子（tid）および、TM_IndexStatus 配列へのインデックス（id）。
  • **出力**: なし（データ構造体）。

前提条件と不変条件

  • sizeof(TM_IndexDelete) は 8 バイト以下であることが期待されています（src/backend/access/heap/heapam.c にて静的アサーションによる検証あり）。

アルゴリズムまたは振る舞い

本構造体は、テーブルのタプル削除処理に伴うインデックス更新のバッチ処理やソート処理において、削除対象のタプルとそれに対応するインデックスの状態を紐付けるために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c 内のインデックス削除処理およびプリフェッチ処理に関連する関数群。
  • **呼び出し先**: 情報なし。

関連データ構造

  • ItemPointerData: タプルの物理位置を特定するための構造体。
  • TM_IndexStatus: 本構造体の id フィールドが指し示すインデックス状態管理用配列の要素型。

失敗モードと境界条件

  • id フィールドが TM_IndexStatus 配列の範囲外を指す場合、後続の処理でメモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は heapam.c におけるインデックス削除の最適化（ソートやグループ化）において頻繁に利用されます。構造体のサイズ変更を行う際は、heapam.c 内の StaticAssertDecl による制約に注意してください。
