gbt_numeric_penalty

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:146

目的

GiST インデックスにおいて、新しいキーを挿入する際に既存のインデックス範囲がどれだけ拡大するかを計算し、ペナルティ値として算出する。

シグネチャまたは形状

Datum gbt_numeric_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): 既存のインデックスエントリ (GISTENTRY *o)
  • PG_GETARG_POINTER(1): 新しく挿入するキー (GISTENTRY *n)
  • PG_GETARG_POINTER(2): 計算結果を格納する float *result
  • **出力**: PG_RETURN_POINTER(result) を通じて、拡大率を示す float 値を返す。

前提条件と不変条件

  • numeric 型の比較および算術演算（numeric_sub, numeric_div, numeric_gt 等）が利用可能であること。
  • tinfo（gbtree_vinfo 構造体）が numeric 型の比較関数群を保持していること。

アルゴリズムまたは振る舞い

1. **範囲の統合**: gbt_var_bin_union を使用し、既存の範囲と新しいキーを統合した新しい範囲を計算する。 2. **差分計算**: 統合後の範囲の幅と、元の範囲の幅をそれぞれ numeric_sub で算出する。 3. **拡大率の算出**:

  • 統合後の幅が元の幅より大きい場合、その比率（ds / us）をペナルティとして算出する。
  • NaN が含まれる場合は、特殊な重み付けを行う。

    • 4. **スケーリング**: 算出されたペナルティ値に、インデックスの属性数に基づく係数（FLT_MAX / (natts + 1)）を乗じて正規化する。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの挿入処理（PostgreSQL コアの GiST 実装から呼び出される）。
  • **呼び出し先**:
  • gbt_var_key_readable, gbt_var_key_copy, gbt_var_bin_union（btree_utils_var.c）
  • numeric_sub, numeric_div, numeric_gt, numeric_is_nan, numeric_float8_no_overflow（src/backend/utils/adt/numeric.c）

関連データ構造

  • GISTENTRY: GiST インデックスエントリを保持する構造体。
  • GBT_VARKEY: 可変長データ型を GiST インデックスで扱うためのキー構造体。
  • tinfo: numeric 型の比較ロジックを定義する gbtree_vinfo 構造体。

失敗モードと境界条件

  • **NaN の扱い**: us（統合後の幅）が NaN の場合、os（元の幅）も NaN であればペナルティは 0.0 とし、そうでなければ 1.0 を返す。
  • **オーバーフロー**: numeric_float8_no_overflow を使用して、float への変換時のオーバーフローを回避している。
  • **範囲の拡大なし**: 拡大がない場合、ペナルティは 0.0 となる。

保守担当者向けメモ

  • 本関数は btree_utils_var.c で定義された汎用関数を numeric 型向けに適合させるアダプターの役割を果たす。
  • numeric 型の演算はコストが高いため、パフォーマンスが重要なパスであることを考慮すること。
  • 呼び出し元情報の詳細は、PostgreSQL の GiST インデックス構築ロジック（src/backend/access/gist/）を参照のこと。
