pg_conv_map

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_win/utf8_and_win.c:62

目的

PostgreSQLのWindows系エンコーディングとUTF-8間の相互変換において、特定のエンコーディングIDと、変換に使用するRadix Tree（変換テーブル）を紐付けるためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	pg_enc		encoding;
	const pg_mb_radix_tree *map1;	/* to UTF8 map name */
	const pg_mb_radix_tree *map2;	/* from UTF8 map name */
} pg_conv_map;

入力と出力

  • encoding: 変換対象のエンコーディングID（pg_enc型）。
  • map1: 指定エンコーディングからUTF-8へ変換するためのRadix Treeへのポインタ。
  • map2: UTF-8から指定エンコーディングへ変換するためのRadix Treeへのポインタ。

前提条件と不変条件

  • map1 および map2 は、src/backend/utils/mb/Unicode/ 配下の各 .map ファイルで定義された有効なRadix Tree構造を指している必要があります。
  • 本構造体は、変換プロシージャ内で静的配列 maps の要素として定義されることを前提としています。

アルゴリズムまたは振る舞い

変換処理のディスパッチ時に、maps 配列を走査して指定されたエンコーディングIDと一致する pg_conv_map を特定します。特定された構造体の map1 または map2 を用いて、汎用関数 LocalToUtf または UtfToLocal が実際のバイト変換を実行します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/mb/conversion_procs/utf8_and_win/utf8_and_win.c 内の変換関数（win_to_utf8, utf8_to_win 等）が、変換テーブルを特定するために本構造体を参照します。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • pg_mb_radix_tree: 変換テーブルの実体。
  • maps: pg_conv_map 型の静的配列。サポートされる全Windowsエンコーディングの定義を保持します。

失敗モードと境界条件

  • 指定されたエンコーディングIDが maps 配列内に存在しない場合、変換処理は失敗しエラーを報告します。
  • 構造体内のポインタがNULLの場合、変換関数側で適切なエラーハンドリングが必要です（ソースコード上の明示的なNULLチェックの有無については、本構造体定義のみからは判断できません）。

保守担当者向けメモ

  • 新しいWindowsエンコーディングを追加する場合は、対応する .map ファイルを準備し、maps 配列に新しい pg_conv_map エントリを追加してください。
  • 本構造体は utf8_and_iso8859 など他の変換プロシージャでも同様の名称で定義されていますが、それぞれ独立した定義です。
