InitializeWalConsistencyChecking

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlog.c:4821

目的

共有ライブラリのロード完了後に、遅延されていた wal_consistency_checking GUC（設定パラメータ）の適用を確定させるために使用されます。

シグネチャまたは形状

void InitializeWalConsistencyChecking(void)

入力と出力

  • 入力: なし（グローバル変数 check_wal_consistency_checking_deferred および wal_consistency_checking_string を参照）
  • 出力: なし（副作用として GUC 設定を更新）

前提条件と不変条件

  • process_shared_preload_libraries_done が真であること（アサーションにより強制）。
  • 実行後、check_wal_consistency_checking_deferred は必ず偽となる。

アルゴリズムまたは振る舞い

1. check_wal_consistency_checking_deferred が真の場合のみ処理を実行する。 2. find_option を使用して wal_consistency_checking 設定項目を検索する。 3. check_wal_consistency_checking_deferred を false に設定する。 4. set_config_option_ext を呼び出し、保持されていた設定値（wal_consistency_checking_string）を適用する。 5. 処理終了後、check_wal_consistency_checking_deferred が偽であることをアサーションで確認する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/postmaster.c:975
  • src/backend/tcop/postgres.c:4142
  • src/backend/access/transam/xlog.c 内の初期化プロセス
  • 呼び出し先:
  • find_option
  • set_config_option_ext

関連データ構造

  • config_generic: GUC 設定のメタデータ構造体。

失敗モードと境界条件

  • find_option が wal_consistency_checking を見つけられない場合、ERROR が発生し処理が中断される。
  • set_config_option_ext への引数に不正な値が含まれる場合、ERROR が発生する。

保守担当者向けメモ

  • 本関数は、カスタムリソースマネージャがロードされた後に実行される必要があるため、呼び出しタイミングが重要です。
  • check_wal_consistency_checking_deferred フラグは、設定の適用を適切なタイミングまで遅延させるためのガードとして機能しています。
