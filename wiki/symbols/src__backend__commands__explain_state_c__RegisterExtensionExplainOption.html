RegisterExtensionExplainOption

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_state.c:317

目的

拡張機能（Extension）が独自の EXPLAIN オプションを登録するためのハンドラを管理する。

シグネチャまたは形状

void RegisterExtensionExplainOption(const char *option_name, ExplainOptionHandler handler)

入力と出力

  • option_name: 登録するオプション名（文字列）。
  • handler: オプション処理用の ExplainOptionHandler 型の関数ポインタ。
  • 出力: なし（副作用として内部配列 ExplainExtensionOptionArray を更新）。

前提条件と不変条件

  • ExplainExtensionOptionArray が初期化されていない場合は TopMemoryContext 上に確保される。
  • 同名のオプションが既に存在する場合、ハンドラが上書きされる。
  • 配列が満杯の場合は repalloc により容量が拡張される。

アルゴリズムまたは振る舞い

1. ExplainExtensionOptionArray を走査し、指定された option_name と一致する既存エントリを探す。見つかればハンドラを更新して終了する。 2. 配列が未割り当てであれば、初期サイズ 16 でメモリを確保する。 3. 配列が満杯であれば、pg_nextpower2_32 を用いて容量を拡張する。 4. 新しいエントリを配列の末尾に追加し、ExplainExtensionOptionsAssigned をインクリメントする。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_overexplain/pg_overexplain.c 等の拡張機能モジュール。
  • 呼び出し先: MemoryContextAlloc, repalloc, pg_nextpower2_32（メモリ管理および配列拡張用）。

関連データ構造

  • ExplainExtensionOption: オプション名とハンドラを保持する構造体。
  • ExplainExtensionOptionArray: 登録されたオプションを保持するグローバルな配列。

失敗モードと境界条件

  • メモリ不足時: MemoryContextAlloc または repalloc がエラーを発生させる。
  • 境界条件: 配列のサイズが ExplainExtensionOptionsAllocated に達した際、自動的に再割り当てが行われる。

保守担当者向けメモ

  • 本関数で登録されたオプションは、EXPLAIN コマンド実行時に ParseExplainOptionList から参照される。
  • ExplainExtensionOptionsAssigned は、登録されたオプションの総数を追跡するグローバル変数である。
