pg_log_standby_snapshot

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogfuncs.c:201

目的

現在のプライマリサーバーにおいて、スタンバイサーバーが利用可能なスナップショット情報をWALに強制的に書き込み、そのWAL位置（LSN）を返却します。

シグネチャまたは形状

Datum pg_log_standby_snapshot(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQLインターフェース経由の引数）
  • **出力**: Datum 型のLSN（最後に挿入されたWALレコードの位置）

前提条件と不変条件

  • サーバーがリカバリ中（RecoveryInProgress()）ではないこと。
  • wal_level 設定が replica 以上であること（XLogStandbyInfoActive() が真であること）。
  • 上記条件を満たさない場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. RecoveryInProgress() をチェックし、リカバリ中であればエラーを報告します。 2. XLogStandbyInfoActive() をチェックし、wal_level が不十分であればエラーを報告します。 3. LogStandbySnapshot() を呼び出し、スナップショット情報をWALに書き込みます。 4. 書き込まれたWALレコードのLSNを PG_RETURN_LSN マクロを介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQLインターフェース経由で呼び出されます。
  • src/bin/pg_basebackup/pg_createsubscriber.c:829 からの呼び出しが確認されています。
  • **呼び出し先**:
  • LogStandbySnapshot()（内部的なWAL書き込み処理）

関連データ構造

  • XLogRecPtr（WAL位置を保持するために使用）

失敗モードと境界条件

  • **リカバリ中の実行**: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーを発生させます。
  • **wal_level不足**: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーを発生させます。

保守担当者向けメモ

  • 本関数はSQLインターフェースとして公開されており、主にスタンバイサーバーの同期やバックアップ関連のツールから利用されます。
  • LogStandbySnapshot() の実装変更が本関数の戻り値や動作に直接影響するため、変更時は注意してください。
