create_index_paths

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/indxpath.c:239

目的

リレーションに対して利用可能なすべてのインデックススキャンおよびビットマップスキャンパスを生成し、プランナのパスリストに追加する。

シグネチャまたは形状

void create_index_paths(PlannerInfo *root, RelOptInfo *rel)

入力と出力

  • **入力**:
  • root: クエリ全体のプランナ情報。
  • rel: パスを生成する対象のリレーション情報。
  • **出力**: なし（副作用として rel->pathlist 等にパスを追加する）。

前提条件と不変条件

  • rel->indexlist が空の場合、関数は直ちに終了する。
  • 各インデックスのキー列数（nkeycolumns）は INDEX_MAX_KEYS 以下である必要がある。
  • 部分インデックス（Partial Index）の場合、indpred がクエリ条件と適合（predOK）している必要がある。

アルゴリズムまたは振る舞い

1. **インデックス走査**: リレーションの全インデックスを順次処理する。 2. **制約条件のマッチング**: 各インデックスに対し、制限句（match_restriction_clauses_to_index）および結合句（match_join_clauses_to_index、match_eclass_clauses_to_index）を適合させる。 3. **パス生成**:

  • 制限句から非パラメータ化インデックスパスを生成し、add_path で登録。
  • 結合句からパラメータ化インデックスパスを生成（consider_index_join_clauses）。
  • OR 条件から BitmapOrPath を生成し、ビットマップパスリストに蓄積。

    • 4. **ビットマップ統合**:

  • 蓄積されたビットマップパスに対し、choose_bitmap_and でコストに基づき最適な組み合わせを選択。
  • create_bitmap_heap_path を呼び出し、最終的なビットマップヒープパスを生成して登録。
  • パラメータ化された結合パスについては、パラメータの組み合わせごとに個別に最適化を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/allpaths.c:798
  • **呼び出し先**:
  • match_restriction_clauses_to_index, match_join_clauses_to_index, match_eclass_clauses_to_index
  • get_index_paths, consider_index_join_clauses
  • generate_bitmap_or_paths, choose_bitmap_and, create_bitmap_heap_path, add_path

関連データ構造

  • PlannerInfo: クエリ全体のプランニング情報。
  • RelOptInfo: 対象リレーションのパスやインデックス情報。
  • IndexClauseSet: インデックス列と適合した条件を保持する構造体。

失敗モードと境界条件

  • インデックスが存在しない場合、処理をスキップする。
  • 部分インデックスがクエリ条件を満たさない場合、そのインデックスは無視される。
  • ビットマップパスが生成できない場合、ビットマップ関連のパス構築処理はスキップされる。

保守担当者向けメモ

  • 本関数は indxpath.c のメインエントリポイントであり、インデックスパス生成の全体フローを制御する。
  • ビットマップパスの最適化（choose_bitmap_and）はヒューリスティックなコスト見積もりに依存しているため、プランナの挙動を調整する際は注意が必要。
  • パラメータ化されたパスの生成ロジックは、結合条件の複雑さに応じて計算量が増大する可能性がある。
