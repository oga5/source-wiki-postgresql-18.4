spgxlogAddLeaf

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgxlog.h:46

目的

SP-GiST インデックスにおいて、新しいリーフタプルを追加する操作を記録するための WAL（Write-Ahead Log）レコード構造体です。リカバリ時にページの状態を復元するために使用されます。

シグネチャまたは形状

typedef struct spgxlogAddLeaf
{
    bool        newPage;
    bool        storesNulls;
    OffsetNumber offnumLeaf;
    OffsetNumber offnumHeadLeaf;
    OffsetNumber offnumParent;
    uint16      nodeI;
    /* 新しいリーフタプルが続く（アライメントなし） */
} spgxlogAddLeaf;

入力と出力

  • **入力**: インデックス操作時に各フィールドへ値を設定し、XLogInsert を介して WAL へ書き出されます。
  • **出力**: リカバリ時、spg_redo 関数がこの構造体を読み取り、対象ページ上のアイテムを更新します。

前提条件と不変条件

  • 構造体直後に続くリーフタプルデータはアライメントされていません。アクセス時には memcpy 等を用いて安全にコピーする必要があります。
  • offnumLeaf や offnumParent などのオフセット値は、対象ページ内での有効な位置を指している必要があります。

アルゴリズムまたは振る舞い

1. **記録**: インデックスへのリーフ追加時、本構造体にメタデータを格納し、その直後に可変長のタプルデータを配置して WAL レコードを作成します。 2. **再生**: リカバリ時、spg_redo がレコードを読み込みます。newPage が真であればページを初期化し、offnumLeaf 等の情報を基にページ内のタプル配置を再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgdoinsert.c: インデックス挿入処理中に本構造体を初期化。
  • **呼び出し先**:
  • src/backend/access/rmgrdesc/spgdesc.c: ログ出力（spg_desc）時に参照。
  • src/backend/access/spgist/spgxlog.c: リカバリ処理（spg_redo）時に構造体ポインタとしてキャストされ、データが読み取られる。

関連データ構造

  • spgxlogState: リカバリ時に必要なインデックス状態を保持する構造体。
  • src/include/access/spgist_private.h: SP-GiST 内部のタプル形式定義。

失敗モードと境界条件

  • 可変長データを含むため、ポインタ演算時に sizeof(spgxlogAddLeaf) を基準としたオフセット計算を誤るとメモリ破壊や不正なデータ読み取りが発生します。
  • ページ内のオフセットが不正な場合、リカバリ処理が失敗します。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER 的な扱いを受けており、構造体サイズを超えたメモリ領域にタプルデータが存在します。ポインタ操作時は必ず sizeof(spgxlogAddLeaf) を使用してください。
  • リカバリロジックを追う際は、spgxlog.c 内でのポインタキャストとデータ読み取り順序に注意してください。
