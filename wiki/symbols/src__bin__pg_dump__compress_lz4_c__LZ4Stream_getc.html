LZ4Stream_getc

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:624

目的

LZ4 圧縮されたファイルストリームから 1 バイトを読み込み、呼び出し元に返します。

シグネチャまたは形状

static int LZ4Stream_getc(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH（圧縮ファイルハンドルへのポインタ）。
  • **出力**: 読み込まれた 1 バイトのデータ（unsigned char を int に昇格したもの）。
  • **副作用**: 内部状態の更新および、読み込み失敗時の pg_fatal によるプロセス終了。

前提条件と不変条件

  • CFH->private_data は有効な LZ4State 構造体を指している必要があります。
  • LZ4Stream_read_internal を介してデータが供給されるため、ストリームの初期化が完了していることが前提です。

アルゴリズムまたは振る舞い

1. CFH->private_data から LZ4State を取得します。 2. LZ4Stream_read_internal を呼び出し、1 バイトの読み込みを試みます。 3. 読み込みが 0 バイト以下の場合（エラーまたは EOF）:

  • LZ4Stream_eof を確認し、EOF であれば「end of file」エラーで終了します。
  • EOF でない場合は、LZ4Stream_get_error で取得したエラーメッセージと共に「could not read from input file」として終了します。

    • 4. 正常に読み込めた場合、そのバイト値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_lz4.c 内の初期化処理にて、CFH->getc_func に関数ポインタとして代入されています。
  • **呼び出し先**:
  • LZ4Stream_read_internal
  • LZ4Stream_eof
  • LZ4Stream_get_error
  • pg_fatal

関連データ構造

  • CompressFileHandle: 圧縮ストリームの抽象化ハンドル。
  • LZ4State: LZ4 圧縮・解凍のコンテキストおよびバッファ状態を保持する構造体。

失敗モードと境界条件

  • **EOF**: 読み込み中に終端に達した場合、pg_fatal によりプログラムが終了します。
  • **読み込みエラー**: LZ4Stream_read_internal がエラーを返した場合、pg_fatal によりエラー詳細を出力して終了します。

保守担当者向けメモ

  • 本関数は CompressFileHandle インターフェースの一部として実装されています。
  • 読み込み失敗時のエラーハンドリングは pg_fatal に依存しており、呼び出し元でのエラー制御は行われません。
