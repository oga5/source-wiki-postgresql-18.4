ScanKeyData

定義

  • 種別: struct
  • 定義箇所: src/include/access/skey.h:64

目的

テーブルやインデックスのスキャン実行時に、検索条件（属性番号、演算子、比較値など）を抽象化してアクセスメソッド（AM）に伝達するためのデータ構造です。

シグネチャまたは形状

typedef struct ScanKeyData
{
	int			sk_flags;		/* フラグ */
	AttrNumber	sk_attno;		/* 列番号 */
	StrategyNumber sk_strategy; /* 演算子戦略番号 */
	Oid			sk_subtype;		/* 戦略サブタイプ */
	Oid			sk_collation;	/* 照合順序 */
	FmgrInfo	sk_func;		/* 比較関数情報 */
	Datum		sk_argument;	/* 比較対象の値 */
} ScanKeyData;

入力と出力

  • **入力**: ScanKeyInit 等の初期化関数を通じて、属性番号、演算子情報、比較値が設定されます。
  • **出力**: スキャン実行時、AM は sk_func を介して比較関数を呼び出し、条件判定結果を得ます。

前提条件と不変条件

  • sk_flags の下位 16 ビットはシステム全体で予約されており、上位 16 ビットは各インデックス AM が独自に使用可能です。
  • 行比較（SK_ROW_HEADER）を使用する場合、B-tree インデックスでのみサポートされており、SK_ROW_MEMBER および SK_ROW_END の適切な設定が必要です。

アルゴリズムまたは振る舞い

1. **初期化**: ScanKeyInit や ScanKeyEntryInitialize により構造体が構築されます。 2. **スキャン実行**: 構築された ScanKeyData の配列が AM に渡されます。 3. **評価**: AM は sk_flags を確認し、sk_func（FmgrInfo）を使用して sk_argument と対象データを比較し、条件を満たすか判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c、contrib/bloom/blscan.c、contrib/dblink/dblink.c、contrib/sepgsql/ 配下の各モジュールなどで、スキャン条件の保持用としてインスタンス化されています。
  • **呼び出し先**: sk_func に格納された関数ポインタを介して、比較関数が動的に呼び出されます。

関連データ構造

  • ScanKey: ScanKeyData へのポインタ型。
  • FmgrInfo: 関数マネージャのインターフェース。
  • AttrNumber: 属性番号。
  • StrategyNumber: 演算子戦略番号。

失敗モードと境界条件

  • メモリ管理: ScanKey は通常、実行時に動的に確保されるため、メモリコンテキストのライフサイクル管理が不適切だとメモリリークや不正アクセスが発生します。
  • フラグ設定: sk_flags のビット設定が誤っている場合、AM が条件を正しく解釈できず、スキャン結果が不正になる可能性があります。

保守担当者向けメモ

  • 本構造体はスキャン処理の核となるため、変更時は src/backend/access/common/scankey.c の初期化ロジックとの整合性を確認してください。
  • 行比較などの複雑な条件を扱う際は、関連するフラグの定義（SK_ROW_HEADER 等）を skey.h で再確認してください。
