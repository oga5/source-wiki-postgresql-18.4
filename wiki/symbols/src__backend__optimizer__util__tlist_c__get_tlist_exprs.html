get_tlist_exprs

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/tlist.c:171

目的

ターゲットリスト（List *tlist）から、各 TargetEntry が保持する式（expr）のみを抽出したリストを生成します。

シグネチャまたは形状

List *get_tlist_exprs(List *tlist, bool includeJunk)

入力と出力

  • **入力**:
  • tlist: TargetEntry ノードを含むリスト。
  • includeJunk: true の場合、ジャンク列（resjunk が設定された項目）の式も結果に含めます。false の場合は除外します。
  • **出力**:
  • 抽出された式（Node *）のリスト。

前提条件と不変条件

  • 入力 tlist は TargetEntry ノードのリストである必要があります。
  • 戻り値のリストは、呼び出し元で適切にメモリ管理（解放）される必要があります。

アルゴリズムまたは振る舞い

1. 空のリスト result を初期化します。 2. tlist の各要素を走査します。 3. 各要素（TargetEntry）について、resjunk フラグを確認します。 4. includeJunk が false かつ tle->resjunk が true の場合は、その要素をスキップします。 5. それ以外の場合、tle->expr を result リストの末尾に追加します。 6. 走査終了後、構築した result リストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepunion.c:665
  • **呼び出し先**:
  • lfirst（マクロ）
  • lappend（マクロ）

関連データ構造

  • TargetEntry: ターゲットリストの各要素を構成する構造体。resjunk フラグと expr フィールドを持ちます。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • tlist が NIL の場合、空のリストを返します。
  • tlist 内のすべての要素がジャンク列であり、かつ includeJunk が false の場合、空のリストを返します。

保守担当者向けメモ

  • 本関数は単純なリスト変換ユーティリティであり、副作用はありません。
  • includeJunk フラグの制御により、プランナが内部的に使用する補助的な列（ジャンク列）をフィルタリングする用途で多用されます。
