sockaddr_un

定義

  • 種別: struct
  • 定義箇所: src/include/port/win32/sys/un.h:11

目的

Windows環境において、Unixドメインソケット（AF_UNIX）を利用するための互換性を提供します。POSIX準拠のコードをWindows上でコンパイルする際に、標準的なUnix系OSの <sys/un.h> インターフェースをエミュレートします。

シグネチャまたは形状

struct sockaddr_un
{
    unsigned short sun_family;
    char           sun_path[108];
};

入力と出力

本シンボルはデータ構造の定義であり、関数ではありません。

  • sun_family: アドレスファミリーを保持する符号なし短整数。
  • sun_path: ソケットのパス名を格納する108バイトの文字配列。

前提条件と不変条件

  • sun_path のサイズは108バイトに固定されています。
  • Windows SDKの <afunix.h> が利用できない環境でのフォールバックとして機能します。

アルゴリズムまたは振る舞い

本構造体はデータ保持のみを行い、実行時のロジックは含みません。コンパイル時にこのヘッダーをインクルードすることで、Windows環境下でのUnixドメインソケット関連APIの利用を可能にします。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/common/ip.c: getnameinfo_unix 関数の引数や、calloc によるメモリ確保、ai_addrlen の計算に使用。
  • src/include/libpq/pqcomm.h: UNIXSOCK_PATH_BUFLEN マクロの定義において、sun_path のサイズ取得に使用。
  • 呼び出し先:
  • 情報なし（本シンボルはデータ構造定義のため、直接的な呼び出し先は存在しません）。

関連データ構造

  • src/include/port/win32/sys/socket.h: ソケット関連の定義を補完する関連ヘッダー。

失敗モードと境界条件

  • sun_path のサイズ制限（108バイト）を超えるパス名は格納できません。移植先の環境でこのサイズが適切か確認が必要です。

保守担当者向けメモ

  • 本ファイルは、Windowsのツールチェーンが <afunix.h> を提供していない環境をサポートするために存在します。
  • 将来的にツールチェーンが整備された場合、この定義は不要になる可能性があります。
