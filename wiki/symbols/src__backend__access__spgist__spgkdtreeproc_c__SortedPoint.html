SortedPoint

定義

  • 種別: struct
  • 定義箇所: src/backend/access/spgist/spgkdtreeproc.c:78

目的

SP-GiSTインデックスのk-d木構築時、ポイントデータのソートおよび中央値の特定を行うための補助構造体です。

シグネチャまたは形状

typedef struct SortedPoint
{
    Point      *p;  /* ポイントデータへのポインタ */
    int         i;  /* 元のインデックス位置 */
} SortedPoint;

入力と出力

  • 本構造体は、spg_kd_picksplit 関数内でポイントの配列をソートする際に使用されます。
  • 入力: Point 型データと、その元の配列内インデックス。
  • 出力: ソートされた SortedPoint の配列。

前提条件と不変条件

  • spg_kd_picksplit において、データの中央値を用いて空間を二分割するために使用されます。
  • ソート処理において、x_cmp および y_cmp 関数によって SortedPoint の比較が行われます。

アルゴリズムまたは振る舞い

1. spg_kd_picksplit 内で、入力されたポイント群を SortedPoint 配列に変換します。 2. x 座標または y 座標に基づいて qsort を実行し、データをソートします。 3. ソートされた結果から中央値を特定し、空間分割の境界を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgkdtreeproc.c 内の spg_kd_picksplit 関数。
  • **呼び出し先**: x_cmp および y_cmp 関数（qsort の比較関数として使用）。

関連データ構造

  • Point: utils/geo_decls.h で定義される幾何データ型。
  • spg_kd_picksplit: 本構造体を利用してk-d木の分割ロジックを実行する関数。

失敗モードと境界条件

  • 構造体自体に特有の失敗モードはありませんが、メモリ割り当てが失敗した場合や、入力されたポイント配列が空である場合に spg_kd_picksplit の処理が影響を受けます。

保守担当者向けメモ

  • 本構造体は spgkdtreeproc.c 内でのみ使用されるローカルな補助構造体です。
  • ソートロジックを変更する際は、x_cmp および y_cmp との整合性に注意してください。
