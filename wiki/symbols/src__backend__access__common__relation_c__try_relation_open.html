try_relation_open

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/relation.c:87

目的

OIDを指定してリレーションを開くための汎用インターフェースです。リレーションが存在しない場合にエラーを発生させるのではなく、NULL を返すことで呼び出し元が安全に存在確認を行えるようにします。

シグネチャまたは形状

Relation try_relation_open(Oid relationId, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relationId: 開く対象リレーションのOID。
  • lockmode: 取得するロックモード（NoLock を含む）。
  • **出力**:
  • 成功時: 開かれたリレーションの記述子（Relation）。
  • 失敗時: リレーションが存在しない場合は NULL。

前提条件と不変条件

  • lockmode は NoLock 以上かつ MAX_LOCKMODES 未満である必要があります（Assert で検証）。
  • NoLock を指定した場合、呼び出し元が既に適切なロックを保持している必要があります。

アルゴリズムまたは振る舞い

1. **ロック取得**: lockmode が NoLock でない場合、LockRelationOid を呼び出し指定されたロックを取得します。 2. **存在確認**: SearchSysCacheExists1 を使用してシステムカタログ上でリレーションの存在を確認します。存在しない場合は、取得したロックを解放（NoLock でない場合）し、NULL を返します。 3. **リレーション取得**: RelationIdGetRelation を呼び出し、リレーション記述子をメモリ上にロードします。 4. **検証**: 取得した記述子が有効でない場合はエラー（elog(ERROR)）を発生させます。 5. **状態更新**:

  • 一時リレーション（RelationUsesLocalBuffers）である場合、MyXactFlags に XACT_FLAGS_ACCESSEDTEMPNAMESPACE を設定します。
  • pgstat_init_relation を呼び出し、統計情報収集の準備を行います。

    • 6. **戻り値**: 取得した Relation 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_prewarm/autoprewarm.c
  • src/backend/access/index/indexam.c
  • src/backend/access/table/table.c
  • src/backend/commands/sequence.c
  • src/backend/commands/tablecmds.c
  • src/backend/commands/vacuum.c
  • src/backend/rewrite/rewriteHandler.c
  • **呼び出し先**:
  • LockRelationOid / UnlockRelationOid
  • SearchSysCacheExists1
  • RelationIdGetRelation
  • RelationIsValid
  • RelationUsesLocalBuffers
  • pgstat_init_relation
  • CheckRelationLockedByMe (Assert内)

関連データ構造

  • Relation: リレーション記述子。
  • Oid: リレーションを一意に識別するオブジェクトID。
  • LOCKMODE: ロックの強度を指定する列挙型。

失敗モードと境界条件

  • **リレーション不在**: システムカタログに存在しないOIDが渡された場合、NULL を返します。
  • **ロック競合**: LockRelationOid 呼び出し時に他のセッションとのロック競合が発生した場合、ロックが取得できるまで待機します。
  • **無効な記述子**: RelationIdGetRelation が有効な記述子を返せなかった場合、elog(ERROR) によりトランザクションが中断されます。

保守担当者向けメモ

  • 本関数は relation_open と異なり、存在しないリレーションを許容する設計です。
  • NoLock を指定して呼び出す際は、呼び出し元が既に適切なロックを保持していることを保証する CheckRelationLockedByMe のアサーションが機能します。
  • 一時リレーションのアクセスフラグ更新は、トランザクション管理と密接に関連しているため、変更時には注意が必要です。
