strlcat

定義

  • 種別: function
  • 定義箇所: src/port/strlcat.c:32

目的

バッファオーバーフローを防止しつつ、宛先バッファへ文字列を安全に連結する。標準ライブラリが strlcat を提供しない環境における移植性レイヤーとして機能する。

シグネチャまたは形状

size_t strlcat(char *dst, const char *src, size_t siz);

入力と出力

  • **入力**:
  • dst: 連結先のヌル終端文字列バッファ。
  • src: 連結するソース文字列。
  • siz: dst バッファ全体のサイズ（バイト単位）。
  • **出力**:
  • 連結後の文字列の合計長（dst の初期長と src の長さの和）。

前提条件と不変条件

  • siz はバッファの「残りの空き容量」ではなく「バッファ全体のサイズ」である必要がある。
  • 連結後の文字列は、バッファサイズが 0 でない限り、必ずヌル終端される。

アルゴリズムまたは振る舞い

1. dst の終端を探索し、現在の文字列長 dlen を算出する。 2. 残りのバッファサイズ n を計算する。 3. n が 0 の場合、dlen と src の長さを加算して即座に返す。 4. n が 1 より大きい間、src の文字を dst の末尾にコピーし、n を減算する。 5. 最後に *d = '\0' を実行し、ヌル終端を強制する。 6. 連結後の文字列長（dlen + strlen(src)）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/init/miscinit.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_dump/pg_backup_directory.c
  • src/bin/psql/prompt.c
  • src/common/config_info.c
  • src/interfaces/libpq/fe-secure.c
  • **呼び出し先**:
  • strlen（標準ライブラリ関数）

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **切り詰め**: 戻り値が siz 以上の場合、データが切り詰められたことを示す。
  • **バッファサイズ 0**: siz が 0 の場合、dst には何も書き込まれず、strlen(dst) + strlen(src) が返される。

保守担当者向けメモ

  • OpenBSD の strlcat 仕様に準拠している。
  • strncat とは引数の意味が異なるため、呼び出し時に sizeof を渡す際は注意が必要である。
