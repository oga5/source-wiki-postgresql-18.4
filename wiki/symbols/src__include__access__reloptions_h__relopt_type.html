relopt_type

定義

  • 種別: enum
  • 定義箇所: src/include/access/reloptions.h:29

目的

リレーションやテーブルスペースのオプション設定における、データ型の分類を定義する。

シグネチャまたは形状

typedef enum relopt_type
{
    RELOPT_TYPE_BOOL,
    RELOPT_TYPE_INT,
    RELOPT_TYPE_REAL,
    RELOPT_TYPE_ENUM,
    RELOPT_TYPE_STRING,
} relopt_type;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • 各オプション定義構造体（relopt_gen を継承するもの）は、この列挙型のいずれかの値を保持する必要がある。

アルゴリズムまたは振る舞い

  • オプションの解析および検証プロセスにおいて、値の型を識別するためのタグとして使用される。
  • build_reloptions 等の関数において、入力された Datum を適切な C 構造体へ変換する際の型判定基準となる。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/access/reloptions.h 内の構造体定義（relopt_gen 等）で型フィールドとして参照されている。
  • 呼び出し先: なし（列挙型のため）。

関連データ構造

  • relopt_gen: オプション定義の基底構造体。この構造体内で relopt_type が使用される。
  • relopt_value: 解析済みのオプション値を保持する構造体。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 新しいオプション型を追加する場合は、この列挙型に値を追加し、対応する解析・検証ロジックを src/backend/access/common/reloptions.c に実装する必要がある。
