pg_atomic_uint64

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/generic-gcc.h:104

目的

64ビット整数に対する原子操作（Atomic Operations）を安全に実行するためのデータ構造を提供します。

シグネチャまたは形状

typedef struct pg_atomic_uint64
{
	volatile uint64 value pg_attribute_aligned(8);
} pg_atomic_uint64;

入力と出力

  • **入力**: 構造体内部の value メンバは uint64 型であり、原子操作関数の対象となります。
  • **出力**: なし（データ構造定義のため）。

前提条件と不変条件

  • pg_attribute_aligned(8) により、メモリ上で8バイト境界に配置されることが保証されます。
  • volatile 修飾子により、コンパイラによる最適化で値の読み書きが省略されることを防ぎます。
  • PG_DISABLE_64_BIT_ATOMICS が定義されている環境では、本構造体を用いた64ビット原子操作は無効化されます。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用であり、具体的な原子操作（加算、比較交換など）は、generic-gcc.h 内で定義される static inline 関数群によって __atomic または __sync 組み込み関数を介して実行されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/transam/xlog.c: insertingAt, unloggedLSN, logInsertResult 等の変数定義およびメモリ確保に使用。
  • src/backend/access/transam/xlogprefetcher.c: reset_time, prefetch, hit 等の統計管理に使用。
  • **呼び出し先**: 構造体定義のため、直接的な呼び出し先はありません。

関連データ構造

  • pg_atomic_uint32: 32ビット版の原子操作用構造体。
  • pg_atomic_flag: フラグ操作用の原子構造体。

失敗モードと境界条件

  • 64ビット原子操作をサポートしないプラットフォームやコンパイラ設定下では、本構造体を利用した操作はコンパイルエラーまたはリンクエラーとなる可能性があります。
  • メモリ配置が8バイト境界に整列されていない場合、ハードウェアによっては原子性が保証されない可能性があります。

保守担当者向けメモ

  • 本構造体は atomics.h を経由して使用されることを前提としています。直接インクルードしないでください。
  • 構造体のメンバ value に直接アクセスすることは推奨されません。必ず pg_atomic_read_u64() 等の提供されるAPIを介して操作してください。
