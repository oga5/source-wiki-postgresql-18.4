FigureColnameInternal

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_target.c:1751

目的

SQLクエリ内の式ノードから、デフォルトの列名（resname）を推論する内部再帰関数です。

シグネチャまたは形状

static int FigureColnameInternal(Node *node, char **name);

入力と出力

  • **入力**:
  • node: 解析対象の式ノード（Node*）。
  • name: 推論された列名を格納するためのポインタのポインタ（char**）。
  • **出力**:
  • 戻り値: 推論の「強度（strength）」を示す整数値。値が大きいほど、より確実な名前であることを示します。

前提条件と不変条件

  • node が NULL の場合、強度 0 を返し、name は更新されません。
  • 再帰的に呼び出される際、下位ノードから得られた強度に基づき、より適切な名前が優先的に選択されます。

アルゴリズムまたは振る舞い

  • ノードの型（nodeTag）に応じて名前を決定します。
  • ColumnRef や A_Indirection では、パスの末尾にある文字列を名前として抽出します。
  • FuncCall, CoalesceExpr, MinMaxExpr 等の関数呼び出し系ノードは、関数名をそのまま列名として使用します。
  • TypeCast や CollateClause 等のラッパーノードは、内部の引数ノードに対して再帰的に処理を行います。
  • SubLink の場合、サブクエリのターゲットリストから名前を抽出するか、EXISTS や ARRAY などの特定のサブリンク型に応じた固定名を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の FigureColname（推論のメインエントリ）および FigureColnameInternal 自身（再帰呼び出し）。
  • **呼び出し先**: FigureColnameInternal 自身（再帰）。

関連データ構造

  • Node, ColumnRef, A_Indirection, FuncCall, TypeCast, SubLink, CaseExpr 等の各種ASTノード。

失敗モードと境界条件

  • 名前を特定できないノード（例: 特定の A_Expr や SubLink の一部の型）に遭遇した場合、強度は 0 となり、name は更新されません。
  • TypeCast 等で強度が 1 以下の場合は、より上位のノードで名前が上書きされる可能性があります。

保守担当者向けメモ

  • 新しい式ノードをパーサーに追加した際は、本関数の switch 文に該当ノードの処理を追加し、適切な列名推論ロジックを定義してください。
  • SubLink の変換は遅延評価される可能性があるため、ノードの状態確認には注意が必要です。
