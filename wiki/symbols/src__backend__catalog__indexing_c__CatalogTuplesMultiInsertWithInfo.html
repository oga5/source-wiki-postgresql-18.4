CatalogTuplesMultiInsertWithInfo

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/indexing.c:272

目的

システムカタログに対して複数のタプルを一度に挿入し、関連するインデックスを効率的に更新する。

シグネチャまたは形状

void CatalogTuplesMultiInsertWithInfo(Relation heapRel, TupleTableSlot **slot, int ntuples, CatalogIndexState indstate)

入力と出力

  • **入力**:
  • heapRel: 挿入対象のカタログリレーション。
  • slot: 挿入するタプルを保持する TupleTableSlot の配列。
  • ntuples: 挿入するタプルの数。
  • indstate: インデックス更新に必要な状態情報（CatalogIndexState）。
  • **出力**: なし（void）。

前提条件と不変条件

  • ntuples が 0 以下の場合は何も行わずに終了する。
  • カタログのインデックス更新は、ヒープ挿入とは異なり、個別のループ処理によって実行される。

アルゴリズムまたは振る舞い

1. heap_multi_insert を呼び出し、指定されたすべてのタプルをヒープに一括挿入する。 2. 挿入された各タプルに対してループ処理を行う。

  • ExecFetchSlotHeapTuple を使用してスロットからヒープタプルを取得する。
  • タプルの t_tableOid をスロットの tts_tableOid で更新する。
  • CatalogIndexInsert を呼び出し、indstate を用いてインデックスを更新する。
  • 取得したタプルが一時的なものである場合は heap_freetuple で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/heap.c、src/backend/catalog/pg_depend.c、src/backend/catalog/pg_enum.c、src/backend/catalog/pg_shdepend.c、src/backend/commands/tsearchcmds.c など、複数のカタログ操作箇所から参照されている。
  • **呼び出し先**: heap_multi_insert、ExecFetchSlotHeapTuple、CatalogIndexInsert、heap_freetuple。

関連データ構造

  • Relation: カタログリレーションの記述子。
  • TupleTableSlot: タプルデータを保持する実行時構造体。
  • CatalogIndexState: インデックス更新のための不透明な状態保持構造体。

失敗モードと境界条件

  • ntuples <= 0 の場合は早期リターンする。
  • システムカタログの制約（NOT NULL 等）に違反する場合、CatalogIndexInsert 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は heap_multi_insert を利用してヒープ挿入を最適化しているが、インデックス更新には一括処理の仕組みがないため、個別のループ処理が必要である。
  • カタログ操作におけるインデックス更新のオーバーヘッドを抑えるため、CatalogOpenIndexes で取得した indstate を再利用する設計となっている。
