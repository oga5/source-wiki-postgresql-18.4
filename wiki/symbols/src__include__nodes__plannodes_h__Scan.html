Scan

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/plannodes.h:470

目的

クエリ実行計画におけるスキャン操作の基底となる抽象構造体です。テーブルやインデックスなどのリレーションからタプルを読み取るための共通情報を保持します。

シグネチャまたは形状

typedef struct Scan
{
	pg_node_attr(abstract)

	Plan		plan;
	Index		scanrelid;
} Scan;

入力と出力

  • **入力**: Plan 構造体（基底クラスとして継承）。
  • **出力**: scanrelid（レンジテーブルへのインデックス）。

前提条件と不変条件

  • pg_node_attr(abstract) が指定されている通り、本構造体は抽象的な定義であり、直接インスタンス化されることはありません。
  • 全てのプランノードと同様に、Plan 構造体を最初のメンバとして保持しており、Plan* への安全なキャストが可能です。

アルゴリズムまたは振る舞い

  • 本構造体は、具体的なスキャンノード（SeqScan など）が継承する共通のメタデータを提供します。
  • scanrelid は、クエリのレンジテーブル（Range Table）内のどのリレーションを対象とするかを識別するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（src/backend/optimizer/）が実行計画生成時に本構造体を継承したノードを構築します。
  • **呼び出し先**: エグゼキュータ（src/backend/executor/）が実行時に本構造体のメンバを参照します。
  • **直接の証拠**: contrib/pg_overexplain/pg_overexplain.c において、Scan 構造体へのキャストおよび scanrelid メンバへのアクセスが確認されています。

関連データ構造

  • Plan: 全てのプランノードの基底。
  • Index: レンジテーブルのインデックスを保持する型。

失敗モードと境界条件

  • 抽象構造体であるため、本構造体単体での操作は想定されていません。
  • scanrelid がレンジテーブルの有効な範囲外を指している場合、エグゼキュータによる参照時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は継承を前提としているため、メンバを追加する際は、派生する全ての具体的なスキャンノードに影響が及ぶことに留意してください。
  • scanrelid はレンジテーブルのインデックスであるため、クエリの解析段階で構築されたレンジテーブルとの整合性が必須です。
