FloatExceptionHandler

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/postgres.c:3073

目的

浮動小数点演算例外（SIGFPE）が発生した際に、PostgreSQL のエラー報告メカニズムを通じて例外を捕捉し、適切なエラーメッセージを記録して処理を停止させるために使用されます。

シグネチャまたは形状

void FloatExceptionHandler(SIGNAL_ARGS)
  • SIGNAL_ARGS: シグナルハンドラとして呼び出されるための標準的な引数マクロ。

入力と出力

  • 入力: シグナルハンドラとして OS から渡される引数（SIGNAL_ARGS）。
  • 出力: なし（関数は復帰せず、ereport によりエラーを発生させて終了します）。

前提条件と不変条件

  • 本関数は SIGFPE シグナルハンドラとして登録されることを前提としています。
  • ereport(ERROR, ...) を呼び出すため、呼び出し元は PG_TRY ブロックやエラーリカバリのコンテキスト内にある必要があります。

アルゴリズムまたは振る舞い

1. ereport を呼び出し、ERRCODE_FLOATING_POINT_EXCEPTION を指定してエラーを報告します。 2. エラーメッセージとして「floating-point exception」を、詳細情報として「範囲外の結果やゼロ除算などの無効な演算」が発生した旨を記録します。 3. 本関数は復帰しないため、errno の保存は行われません。

呼び出し元と呼び出し先

  • 呼び出し元: OS のシグナルハンドラとして呼び出されます。以下の箇所で pqsignal(SIGFPE, FloatExceptionHandler) として登録されていることが確認されています。
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/bgworker.c
  • src/backend/replication/logical/slotsync.c
  • src/backend/tcop/postgres.c
  • src/pl/plperl/plperl.c
  • 呼び出し先: ereport マクロ（PostgreSQL のエラー報告システム）。

関連データ構造

  • SIGNAL_ARGS: シグナルハンドラの引数定義に使用されるマクロ。

失敗モードと境界条件

  • 本関数自体がエラーハンドラであるため、この関数内でさらなる例外が発生した場合の挙動は、PostgreSQL のエラー処理フレームワークの依存します。

保守担当者向けメモ

  • pg_noreturn 属性が src/include/tcop/tcopprot.h で宣言されています。
  • 浮動小数点演算の例外を捕捉する唯一のハンドラであるため、変更時は PostgreSQL のエラー報告システムとの整合性に注意してください。
