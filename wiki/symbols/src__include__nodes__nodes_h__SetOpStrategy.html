SetOpStrategy

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/nodes.h:410

目的

集合演算（UNION, INTERSECT, EXCEPT）を実行する際のアルゴリズム戦略を識別するために使用されます。

シグネチャまたは形状

typedef enum SetOpStrategy
{
    SETOP_SORTED,
    SETOP_HASHED,
} SetOpStrategy;

入力と出力

  • 入力: 集合演算ノード（SetOp）の生成時やプランニング時に、戦略を選択するパラメータとして使用されます。
  • 出力: 選択された戦略に基づき、実行エンジンがソート処理またはハッシュテーブル構築のいずれかを選択します。

前提条件と不変条件

  • SETOP_SORTED: 入力データが集合演算に必要な順序でソートされている必要があります。
  • SETOP_HASHED: 内部ハッシュテーブルを使用して集合演算を実行します。

アルゴリズムまたは振る舞い

  • SETOP_SORTED: ソート済み入力を前提とした逐次的な集合演算を行います。
  • SETOP_HASHED: メモリ上にハッシュテーブルを構築し、集合演算を効率化します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/optimizer/plan/createplan.c: make_setop 関数にて使用。
  • src/backend/optimizer/util/pathnode.c: プランパスの構築過程で使用。
  • 呼び出し先: 実行エンジン側のノード処理ロジック（詳細は本シンボル定義からは直接特定不可）。

関連データ構造

  • SetOp (プランノード): src/include/nodes/plannodes.h
  • SetOpPath (パスノード): src/include/nodes/pathnodes.h
  • SetOpCmd: 集合演算の種類（UNION等）を定義する列挙型。

失敗モードと境界条件

  • 適切な戦略が選択されない場合、実行時に不正なデータ処理やパフォーマンス低下が発生する可能性があります。
  • 実行エンジンがサポートしていない組み合わせ（例: ソートされていないデータに対する SETOP_SORTED の適用）は、プランニング段階で排除される必要があります。

保守担当者向けメモ

  • 本列挙型は SetOp ノードの動作を決定する重要なインターフェースです。新しい戦略を追加する場合は、copyfuncs.c や equalfuncs.c 等のノードサポート関数への反映が必要です。
