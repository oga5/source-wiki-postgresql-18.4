gist_indexsortbuild_levelstate_flush

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuild.c:492

目的

ソート済み構築（Sorted Build）において、蓄積されたインデックスページ群をフラッシュし、GiSTインデックスの物理ページとして書き出し、親レベルへのダウンリンクを生成する。

シグネチャまたは形状

static void gist_indexsortbuild_levelstate_flush(GISTBuildState *state, GistSortedBuildLevelState *levelstate)

入力と出力

  • **入力:**
  • state: インデックス構築全体の状態を保持する GISTBuildState 構造体。
  • levelstate: フラッシュ対象のレベル状態を保持する GistSortedBuildLevelState 構造体。
  • **出力:** なし（副作用としてインデックスページがディスクへ書き出され、親レベルの状態が更新される）。

前提条件と不変条件

  • levelstate には構築済みのページが保持されていること。
  • state->bulkstate が初期化されており、バルク書き込みが可能であること。
  • 呼び出しごとに levelstate->current_page がリセットされる。

アルゴリズムまたは振る舞い

1. **タプル抽出:** levelstate 内の全ページからインデックスタプルを抽出・結合する。 2. **ページ分割:**

  • 複数のページが存在する場合、gistSplit を適用してページレイアウトを決定する。
  • 単一ページの場合は、そのタプル群から単一のレイアウトを作成する。

    • 3. **ページ構築と書き出し:**

  • 各分割レイアウトに対し、新しいページを初期化し、タプルを挿入する。
  • rightlink を設定し、ページを smgr_bulk_write で物理書き出しする。

    • 4. **親への伝播:**

  • 書き出したページのダウンリンク（union_tuple）を親レベルに追加する。
  • 親が存在しない場合は、新規にルートページを作成する。

呼び出し元と呼び出し先

  • **呼び出し元:** gist_indexsortbuild 内の各所（インデックス構築の進行に伴い呼び出される）。
  • **呼び出し先:** gistextractpage, gistjoinvector, gistSplit, gistunion, gistfillitupvec, smgr_bulk_get_buf, gistinitpage, PageAddItem, smgr_bulk_write, gist_indexsortbuild_levelstate_add。

関連データ構造

  • GISTBuildState: 構築全体のコンテキスト。
  • GistSortedBuildLevelState: 特定レベルの構築状態（ページバッファ、親への参照など）。
  • SplitPageLayout: ページ分割の結果を保持する構造体。

失敗モードと境界条件

  • PageAddItem が失敗した場合、elog(ERROR) を発生させる。
  • 割り込みが発生した場合、CHECK_FOR_INTERRUPTS() により処理を中断する。

保守担当者向けメモ

  • 本関数はGiSTのボトムアップ構築における重要なステップであり、ページ分割と親へのダウンリンク伝播を再帰的に処理する。
  • rightlink の設定はデバッグ目的であり、GiSTのインデックス構築中には並行スキャンが発生しないため、論理的な順序は厳密に問われない。
