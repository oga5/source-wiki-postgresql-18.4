markPQExpBufferBroken

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.c:49

目的

PQExpBuffer を「broken（破損）」状態に遷移させ、メモリ不足等のエラー発生時に以降のバッファ操作を無効化する。

シグネチャまたは形状

static void markPQExpBufferBroken(PQExpBuffer str);

入力と出力

  • **入力**: str (PQExpBuffer) - 破損状態へ移行させる対象のバッファ構造体。
  • **出力**: なし（副作用として str の内部状態を更新）。

前提条件と不変条件

  • str は有効な PQExpBuffer 構造体を指している必要がある。
  • 実行後、str->data は静的な oom_buffer を指し、str->len および str->maxlen は 0 に設定される。

アルゴリズムまたは振る舞い

1. str->data が oom_buffer と等しくない場合、現在の str->data を free() で解放する。 2. str->data を静的な oom_buffer_ptr（oom_buffer）で上書きする。 3. str->len および str->maxlen を 0 にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/pqexpbuffer.c 内のバッファ拡張や初期化処理（行 45, 187, 223, 316, 337 など）。
  • **呼び出し先**: free()（標準ライブラリ）。

関連データ構造

  • PQExpBufferData: 操作対象となるバッファ構造体。
  • oom_buffer: メモリ不足時に使用される静的な空文字列領域。

失敗モードと境界条件

  • str->data が既に oom_buffer を指している場合、二重解放を避けるため free() はスキップされる。

保守担当者向けメモ

  • oom_buffer は const として定義されており、破損状態のバッファに対して書き込みを試みるとコンパイル時または実行時の保護が働くよう設計されている。unconstify マクロを使用してこの制約を一時的に解除している。
