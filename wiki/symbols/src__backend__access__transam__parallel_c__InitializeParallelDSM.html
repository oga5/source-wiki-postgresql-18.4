InitializeParallelDSM

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/parallel.c:210

目的

並列クエリ実行のために、リーダープロセスからワーカープロセスへ共有すべきトランザクション状態やセッション情報を動的共有メモリ（DSM）にシリアライズし、初期化する。

シグネチャまたは形状

void InitializeParallelDSM(ParallelContext *pcxt)

入力と出力

  • **入力**: ParallelContext *pcxt（並列実行コンテキスト。ワーカー数やメモリ見積もり情報を含む）。
  • **出力**: なし（副作用として pcxt 内の seg（DSMセグメント）および toc（テーブル・オブ・コンテンツ）が初期化され、共有メモリ領域に状態が書き込まれる）。

前提条件と不変条件

  • 呼び出し時に INTERRUPTS_CAN_BE_PROCESSED() が真であること（偽の場合はワーカー数が0に強制される）。
  • pcxt->estimator が適切に見積もられていること。
  • 共有メモリの確保に失敗した場合、並列実行を断念し、pcxt->nworkers を0に設定してバックエンドプライベートメモリへフォールバックする。

アルゴリズムまたは振る舞い

1. **見積もり**: shm_toc_estimate_chunk を使用し、固定状態および可変状態（GUC、スナップショット、ライブラリ状態等）に必要なメモリ量を算出する。 2. **DSM確保**: dsm_create により共有メモリセグメントを確保する。確保できない場合はプライベートメモリを使用する。 3. **固定状態の書き込み**: FixedParallelState 構造体に現在のユーザーID、トランザクション開始時刻、シリアライザブルトランザクションハンドル等を格納し、shm_toc_insert で登録する。 4. **可変状態のシリアライズ**: nworkers > 0 の場合、各コンポーネント（GUC、スナップショット、ComboCID等）のシリアライズ関数を呼び出し、共有メモリ領域へ書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/executor/execParallel.c
  • **呼び出し先**:
  • EstimateLibraryStateSpace, SerializeLibraryState
  • EstimateGUCStateSpace, SerializeGUCState
  • EstimateSnapshotSpace, EstimateTransactionStateSpace 等の各種シリアライズ関連関数
  • dsm_create, shm_toc_create, shm_toc_allocate, shm_toc_insert

関連データ構造

  • ParallelContext: 並列実行の全体管理構造体。
  • FixedParallelState: ワーカーがリーダーから継承する固定サイズの共有状態。
  • shm_toc: 共有メモリ内のデータ配置を管理するテーブル・オブ・コンテンツ。

失敗モードと境界条件

  • **DSM不足**: dsm_create が失敗した場合、pcxt->nworkers を0にリセットし、プライベートメモリで動作を継続する。
  • **割り込み不可状態**: INTERRUPTS_CAN_BE_PROCESSED() が偽の場合、ワーカー起動を回避するため nworkers を0にする。
  • **セッションDSM欠如**: セッションDSMセグメントが作成できない場合、タプル交換が不可能なため nworkers を0にする。

保守担当者向けメモ

  • 本関数内で shm_toc_estimate_chunk を追加した際は、対応する shm_toc_estimate_keys のカウントを更新する必要がある。
  • 新しい並列化対象の機能を追加する場合は、本関数でのシリアライズ処理と、ワーカー側（ParallelWorkerMain 等）での復元処理の両方を実装すること。
