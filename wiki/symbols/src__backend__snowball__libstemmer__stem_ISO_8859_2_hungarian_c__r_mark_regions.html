r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_2_hungarian.c:479

目的

ハンガリー語のステミング処理において、単語内の処理対象領域（R1）の境界位置を決定し、SN_env 構造体の整数配列 I[0] に格納する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （ステミング環境。単語データおよび現在のカーソル位置 c を保持）
  • **出力**: int （成功時に 1 を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • g_v（母音グループ）および a_0（特定の接尾辞パターン）が定義されていること。

アルゴリズムまたは振る舞い

1. 初期状態として z->I[0] に単語の末尾位置（z->l）を代入する。 2. 母音の出現パターンに基づき、以下のいずれかのロジックで R1 の開始位置を特定する：

  • **パターン1**: 母音の後に非母音が続く場合、その直後の位置を R1 とする。特定の接尾辞（a_0）が続く場合は、そのマッチング結果を優先する。
  • **パターン2**: 母音の後に母音が続く場合、その後の位置を R1 とする。

    • 3. 特定された位置を z->I[0] に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: hungarian_ISO_8859_2_stem（メインのステミング関数）から呼び出される。
  • **呼び出し先**: in_grouping, out_grouping, find_among（Snowball ランタイム関数）。

関連データ構造

  • struct SN_env: ステミング環境を管理する構造体。
  • a_0: among 構造体の配列（領域判定に使用される接尾辞パターン）。
  • g_v: 母音を定義するビットマップ。

失敗モードと境界条件

  • 単語が空、または母音が含まれない場合、領域判定ロジックがスキップされ、初期値（単語末尾）が維持される。
  • in_grouping や out_grouping が失敗した場合、カーソル位置を復元して終了する。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接編集せず、対応する .sbl ファイルを修正すること。
  • z->I[0] は後続のステミングルール（r_case 等）で領域制限として参照されるため、本関数の判定ロジックはステミング精度に直結する。
