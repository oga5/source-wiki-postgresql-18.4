parse_lsn

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/backup_label.c:240

目的

文字列バッファからWAL位置（LSN）を解析し、XLogRecPtr 形式に変換する。

シグネチャまたは形状

static bool parse_lsn(char *s, char *e, XLogRecPtr *lsn, char **c)

入力と出力

  • **入力**:
  • s: 解析対象文字列の開始ポインタ。
  • e: 解析対象文字列の終了位置を示すポインタ。
  • lsn: 解析結果を格納する XLogRecPtr へのポインタ。
  • c: 解析終了後の次の文字位置を格納するポインタのポインタ。
  • **出力**:
  • 成功時に true を返し、lsn と c を更新する。失敗時は false を返す。

前提条件と不変条件

  • e は s が指す文字列内の有効な位置を指している必要がある。
  • 関数実行中、*e の位置を一時的に \0 で置換するため、呼び出し元はバッファが書き換え可能であることを保証する必要がある。

アルゴリズムまたは振る舞い

1. *e の値を一時保存し、その位置を \0 で終端する。 2. sscanf を使用して "%X/%X%n" 形式で16進数の上位・下位ビットを読み取る。 3. 一時保存した値を *e に書き戻し、バッファを復元する。 4. 解析に成功した場合、上位・下位ビットを結合して XLogRecPtr を構築し、c に解析終了位置（s + nchars）を代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/backup_label.c 内の parse_backup_label 関数（行65および84）。
  • **呼び出し先**: 標準ライブラリの sscanf。

関連データ構造

  • XLogRecPtr: WAL位置を保持する型。

失敗モードと境界条件

  • sscanf が2つの値を正しく読み取れなかった場合、false を返す。
  • 入力文字列が期待される XX/XX 形式でない場合、解析は失敗する。

保守担当者向けメモ

  • 本関数はバッファの破壊的変更を伴うため、呼び出し元のバッファが読み取り専用でないことを確認すること。
  • sscanf の %n 指定子を使用して、解析した文字数を正確に追跡している。
