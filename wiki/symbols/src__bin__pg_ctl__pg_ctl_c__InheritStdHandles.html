InheritStdHandles

定義

  • 種別: function
  • 定義箇所: src/bin/pg_ctl/pg_ctl.c:1743

目的

Windows環境において、現在のプロセスの標準入出力ハンドル（stdin, stdout, stderr）を、新しく生成するプロセスに継承させるための設定を行う。

シグネチャまたは形状

static void InheritStdHandles(STARTUPINFO *si)

入力と出力

  • **入力**: STARTUPINFO *si（Windows APIのプロセス起動情報構造体へのポインタ）。
  • **出力**: なし（引数 si のメンバを直接更新する）。

前提条件と不変条件

  • Windows環境での実行を前提とする。
  • si は有効な STARTUPINFO 構造体を指している必要がある。

アルゴリズムまたは振る舞い

1. si->dwFlags に STARTF_USESTDHANDLES フラグを付与し、標準ハンドルの使用を有効化する。 2. GetStdHandle を用いて現在のプロセスの標準入力、標準出力、標準エラーのハンドルを取得する。 3. 各ハンドルが NULL（取得失敗）の場合は、INVALID_HANDLE_VALUE を代入して無効なハンドルとして設定する。 4. 取得したハンドルを si->hStdInput、si->hStdOutput、si->hStdError にそれぞれ格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_ctl/pg_ctl.c 内のプロセス生成処理（CreateRestrictedProcess 等の周辺ロジック）から呼び出される。
  • **呼び出し先**: Windows API の GetStdHandle を呼び出す。

関連データ構造

  • STARTUPINFO (Windows API 構造体)

失敗モードと境界条件

  • GetStdHandle が NULL を返した場合、関数は安全のために INVALID_HANDLE_VALUE を設定する。これにより、子プロセス側でハンドルが無効であることを適切に扱えるようにしている。

保守担当者向けメモ

  • 本関数はWindows特有のプロセス管理ロジックの一部である。Unix系OSでは使用されない。
  • STARTUPINFO 構造体のフラグ操作を行っているため、他のプロセス生成オプションと競合しないよう注意すること。
