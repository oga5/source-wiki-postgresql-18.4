PQcancelPoll

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:225

目的

非同期キャンセル処理において、サーバーからの接続切断（EOF）を待機し、キャンセル要求の完了を確認する。

シグネチャまたは形状

PostgresPollingStatusType PQcancelPoll(PGcancelConn *cancelConn)

入力と出力

  • **入力**: PGcancelConn *cancelConn（キャンセル専用の接続オブジェクト）
  • **出力**: PostgresPollingStatusType 型のステータス
  • PGRES_POLLING_READING: サーバーからの応答（切断）を待機中。
  • PGRES_POLLING_OK: キャンセル要求が完了。
  • PGRES_POLLING_FAILED: エラー発生。

前提条件と不変条件

  • cancelConn は有効なオブジェクトであること。
  • 接続状態が CONNECTION_AWAITING_RESPONSE である場合に、サーバーによる接続切断を待機するロジックが実行される。

アルゴリズムまたは振る舞い

1. 接続状態が CONNECTION_AWAITING_RESPONSE でない場合、PQconnectPoll に処理を委譲する。 2. CONNECTION_AWAITING_RESPONSE の場合、pqReadData を呼び出し、サーバーからのデータ受信を試みる。 3. 受信データが 0 バイト（EOF）であれば、キャンセル要求が受理されたとみなし、接続状態を CONNECTION_OK に更新して PGRES_POLLING_OK を返す。 4. 受信データが 0 より大きい場合、予期せぬ応答と判断し、エラーを記録して PGRES_POLLING_FAILED を返す。 5. 非 Windows 環境では、errno が 0 でないエラーが発生した場合に PGRES_POLLING_FAILED を返す。

関連データ構造

  • PGcancelConn: キャンセル要求の状態を保持する構造体。
  • PGconn: PGcancelConn 内に含まれる接続情報。

失敗モードと境界条件

  • **Windows環境**: TCPの切断挙動の差異により、EOF検知時に ECONNRESET 等が発生する場合があるが、本関数はこれを無視して成功とみなす設計となっている。
  • **予期せぬデータ**: サーバーからデータが送信された場合、プロトコル違反として失敗を返す。
  • **通信エラー**: 読み込み時にエラーが発生した場合、接続状態を CONNECTION_BAD に遷移させる。

保守担当者向けメモ

  • 本関数は非同期APIの一部であり、シグナル安全性が求められる PQcancel とは異なる設計方針（非同期ポーリング）に基づいている。
  • WindowsにおけるEOF検知の特殊性については、ソース内のコメントに記載されたメーリングリストの議論を参照すること。
