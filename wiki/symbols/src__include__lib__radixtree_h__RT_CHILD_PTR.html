RT_CHILD_PTR

定義

  • 種別: typedef union
  • 定義箇所: src/include/lib/radixtree.h:417

目的

適応型基数木（ART）において、子ノードへのポインタを抽象化して保持するための共用体です。メモリ管理方式（ローカルメモリまたは共有メモリ）に応じて、適切なポインタ表現を切り替える役割を持ちます。

シグネチャまたは形状

typedef union RT_CHILD_PTR {
    RT_PTR_ALLOC alloc;
    RT_NODE     *local;
} RT_CHILD_PTR;

入力と出力

  • **入力**: RT_PTR_ALLOC 型（共有メモリ用）または RT_NODE * 型（ローカルメモリ用）の値を保持します。
  • **出力**: 基数木のノード探索や操作において、次階層のノードを指し示すポインタとして利用されます。

前提条件と不変条件

  • 本シンボルは radixtree.h のテンプレート展開により生成される構造の一部です。
  • 共有メモリモード（RT_SHMEM 定義時）とローカルメモリモードのいずれか一方の文脈で利用されることを想定しています。

アルゴリズムまたは振る舞い

  • alloc メンバは、共有メモリ領域における dsa_pointer 等をラップした RT_PTR_ALLOC を格納します。
  • local メンバは、プロセスローカルなメモリ上の RT_NODE 構造体への直接ポインタを格納します。
  • 基数木の階層を辿る際、この共用体を通じて適切なメモリ空間上のノードへアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元**: RT_RADIX_TREE の操作関数（RT_SET, RT_FIND, RT_DELETE 等）およびイテレータ関数から参照されます。
  • **呼び出し先**: 保持するポインタを介して RT_NODE 構造体へアクセスします。
  • 詳細は src/include/lib/radixtree.h 内の RT_COPY_COMMON や RT_FREE_NODE 等の関数で直接利用されています。

関連データ構造

  • RT_NODE: 基数木のノード構造体。
  • RT_PTR_ALLOC: 共有メモリ管理用のポインタ型。
  • RT_RADIX_TREE: 本シンボルを保持・管理するメイン構造体。

失敗モードと境界条件

  • 無効なポインタや初期化されていない RT_CHILD_PTR を参照した場合の動作は未定義です。
  • 共有メモリモードにおいて、dsa 領域が枯渇している場合、alloc メンバへの代入時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルはマクロ展開の一部であるため、デバッグ時にはプリプロセス後のコードを確認してください。
  • 共有メモリとローカルメモリの切り替えロジックは、RT_PTR_ALLOC の定義に依存しているため、メモリ管理方式を変更する際は注意が必要です。
