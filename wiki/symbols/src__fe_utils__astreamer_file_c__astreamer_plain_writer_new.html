astreamer_plain_writer_new

定義

  • src/fe_utils/astreamer_file.c:80

目的

ファイルシステムへの書き込みを行う astreamer インスタンスを生成し、初期化する。

シグネチャまたは形状

astreamer *astreamer_plain_writer_new(char *pathname, FILE *file);

入力と出力

  • **入力**:
  • pathname: 書き込み対象のファイルパス。
  • file: 既存の FILE ポインタ（指定がない場合は NULL）。
  • **出力**:
  • 初期化された astreamer 構造体へのポインタ。

前提条件と不変条件

  • astreamer_plain_writer 構造体は palloc0 によりゼロ初期化される。
  • base.bbs_ops には astreamer_plain_writer_ops が静的に割り当てられる。

アルゴリズムまたは振る舞い

1. astreamer_plain_writer コンテキストをメモリ確保する。 2. pathname を複製し、コンテキストに保持する。 3. file が NULL の場合、pathname をバイナリ書き込みモード（"wb"）で開き、should_close_file フラグを true に設定する。 4. ファイルオープンに失敗した場合は pg_fatal を呼び出し、プロセスを終了する。 5. 初期化済みの astreamer 基底構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c (複数箇所)
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • pstrdup (文字列複製)
  • fopen (ファイルオープン)
  • pg_fatal (エラーハンドリング)

関連データ構造

  • astreamer: 基底となるストリーマー構造体。
  • astreamer_plain_writer: 本関数で生成される、ファイル書き込み用コンテキスト。

失敗モードと境界条件

  • file が NULL で、かつ指定された pathname のファイル作成に失敗した場合、pg_fatal によりプログラムが異常終了する。

保守担当者向けメモ

  • 本関数は astreamer_plain_writer のライフサイクル管理の起点となる。
  • file 引数が渡された場合、呼び出し元がそのファイルのクローズ責任を負う（should_close_file が false となるため）。
