freenfa

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:106

目的

NFA（非決定性有限オートマトン）構造体およびその内部でバッチ割り当てされた状態（state）と遷移（arc）のメモリを一括解放する。

シグネチャまたは形状

static void freenfa(struct nfa *nfa)

入力と出力

  • 入力: struct nfa *nfa （解放対象のNFA構造体へのポインタ）
  • 出力: なし

前提条件と不変条件

  • nfa ポインタは有効である必要がある。
  • 内部の lastsb（状態バッチ）および lastab（遷移バッチ）は、適切にリンクされたリストであるか、NULLである必要がある。

アルゴリズムまたは振る舞い

1. nfa->lastsb から開始し、リンクされたすべての statebatch を走査して解放する。各バッチのサイズ分を nfa->v->spaceused から減算する。 2. nfa->lastab から開始し、リンクされたすべての arcbatch を走査して解放する。各バッチのサイズ分を nfa->v->spaceused から減算する。 3. nfa->nstates を -1 に設定し、最後に nfa 構造体自体を FREE() マクロで解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/regex/regc_nfa.c 内の初期化失敗時やエラー処理パス。
  • src/backend/regex/regcomp.c 内の regcomp 処理終了時やエラー発生時（v->nfa の解放など）。
  • 呼び出し先:
  • FREE() マクロ（メモリ解放用）。

関連データ構造

  • struct nfa: NFAのルート構造体。
  • struct statebatch: 状態を保持するバッチ構造体。
  • struct arcbatch: 遷移を保持するバッチ構造体。

失敗モードと境界条件

  • nfa が NULL の場合、FREE(nfa) の挙動は FREE マクロの実装に依存する。
  • nfa->lastsb や nfa->lastab が NULL の場合、ループはスキップされ、メモリ解放は行われない。

保守担当者向けメモ

  • 本関数は個別の状態や遷移を個別に解放せず、バッチ単位で一括解放する設計である。
  • nfa->v->spaceused の更新を行っているため、メモリ使用量の追跡と整合性を保つ必要がある。
