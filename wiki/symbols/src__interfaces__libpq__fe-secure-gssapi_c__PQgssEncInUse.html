PQgssEncInUse

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-gssapi.c:770

目的

指定されたデータベース接続において、GSSAPI による暗号化が現在有効であるかどうかを判定する。

シグネチャまたは形状

int PQgssEncInUse(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn （PostgreSQL 接続オブジェクトへのポインタ）
  • **出力**: int 型。暗号化が有効な場合は非ゼロ（真）、無効または接続が無効な場合は 0（偽）を返す。

前提条件と不変条件

  • conn が NULL の場合、または conn->gctx（GSSAPI セキュリティコンテキスト）が初期化されていない場合は、常に 0 を返す。
  • 戻り値は conn->gssenc の状態に依存する。

アルゴリズムまたは振る舞い

1. 引数 conn が NULL であるか、conn->gctx が NULL であるかを確認する。いずれかであれば 0 を返す。 2. 上記条件を満たさない場合、conn->gssenc の値をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c:4530
  • src/interfaces/libpq/fe-secure.c:489
  • **呼び出し先**:
  • 本関数は内部で他の関数を呼び出さない。

関連データ構造

  • PGconn: libpq-int.h で定義される接続状態を保持する構造体。gctx および gssenc メンバを参照する。

失敗モードと境界条件

  • conn が NULL の場合、セグメンテーションフォールトを発生させずに 0 を返す安全な実装となっている。
  • 接続が確立されていても GSSAPI が使用されていない場合、正しく 0 を返す。

保守担当者向けメモ

  • 本関数は libpq-fe.h で公開されている API である。
  • conn->gssenc の値は、pqsecure_open_gss による認証ネゴシエーションの成功時に設定される。
