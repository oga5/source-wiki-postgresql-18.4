StrategyShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/freelist.c:452

目的

PostgreSQLのバッファ管理サブシステムが共有メモリ上に必要とする総バイト数を算出する。

シグネチャまたは形状

Size StrategyShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: 共有メモリ領域の合計サイズ（Size 型）

前提条件と不変条件

  • NBuffers および NUM_BUFFER_PARTITIONS が定義されている環境で呼び出される必要がある。
  • 算出されるサイズは、StrategyInitialize で行われる共有メモリ割り当てと整合していなければならない。

アルゴリズムまたは振る舞い

1. BufTableShmemSize を呼び出し、バッファルックアップ用ハッシュテーブルに必要なサイズを算出する。 2. BufferStrategyControl 構造体のサイズを MAXALIGN でアライメント調整して加算する。 3. 合計値を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/buffer/buf_init.c (共有メモリ初期化プロセスの一部として呼び出される)
  • 呼び出し先: BufTableShmemSize (ハッシュテーブルサイズ算出用)

関連データ構造

  • BufferStrategyControl: 共有メモリ上の置換戦略制御ブロック。

失敗モードと境界条件

  • add_size 関数を使用しているため、計算結果が Size 型の最大値を超える場合はエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数で算出するサイズは、StrategyInitialize で実際に確保されるメモリレイアウトと厳密に一致させる必要がある。構造体に変更を加えた場合は、必ず本関数の計算式を更新すること。
