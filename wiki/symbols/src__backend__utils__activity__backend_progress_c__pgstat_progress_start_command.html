pgstat_progress_start_command

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_progress.c:26

目的

長時間実行コマンド（VACUUM, CREATE INDEX等）の開始時に、共有メモリ上の進捗状態を初期化し、追跡を開始するために使用されます。

シグネチャまたは形状

void pgstat_progress_start_command(ProgressCommandType cmdtype, Oid relid)

入力と出力

  • cmdtype (ProgressCommandType): 開始するコマンドの種類。
  • relid (Oid): コマンドの対象となるリレーションのOID（対象がない場合は InvalidOid）。
  • 出力: なし。共有メモリ上の PgBackendStatus を直接更新します。

前提条件と不変条件

  • MyBEEntry が有効であること。
  • pgstat_track_activities が有効であること。
  • 実行中、共有メモリへの書き込みは PGSTAT_BEGIN_WRITE_ACTIVITY および PGSTAT_END_WRITE_ACTIVITY マクロによって保護されます。

アルゴリズムまたは振る舞い

1. MyBEEntry が NULL または pgstat_track_activities が無効な場合は、何もせず即座にリターンします。 2. PGSTAT_BEGIN_WRITE_ACTIVITY を呼び出し、共有メモリへの書き込み権限を確保します。 3. st_progress_command に cmdtype を設定します。 4. st_progress_command_target に relid を設定します。 5. st_progress_param 配列を MemSet を用いて 0 で初期化します。 6. PGSTAT_END_WRITE_ACTIVITY を呼び出し、書き込みを完了します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/backup/basebackup_progress.c
  • src/backend/catalog/index.c
  • src/backend/commands/analyze.c
  • src/backend/commands/cluster.c
  • src/backend/commands/copyfrom.c
  • src/backend/commands/copyto.c
  • src/backend/commands/indexcmds.c
  • 呼び出し先:
  • MemSet (標準ライブラリ)
  • PGSTAT_BEGIN_WRITE_ACTIVITY / PGSTAT_END_WRITE_ACTIVITY (マクロ)

関連データ構造

  • PgBackendStatus: 共有メモリ上に存在するバックエンドの状態構造体。
  • ProgressCommandType: 進捗を追跡するコマンドの種類を定義する列挙型。

失敗モードと境界条件

  • pgstat_track_activities が無効な場合、進捗追跡は行われません。
  • MyBEEntry が初期化されていない場合、関数は安全に終了します。

保守担当者向けメモ

  • 本関数は進捗パラメータの初期化のみを担当します。各パラメータの具体的な意味論は、各コマンドの実装側に依存します。
  • 複数のパラメータを更新する必要がある場合は、本関数で初期化した後、pgstat_progress_update_multi_param を使用してアトミックに更新することが推奨されます。
