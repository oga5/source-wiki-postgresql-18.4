float8_mul

定義

  • 種別: function
  • 定義箇所: src/include/utils/float.h:207

目的

2つの float8 型の値を乗算し、PostgreSQLの要件に基づいたエラーチェック（オーバーフローおよびアンダーフロー）を適用した結果を返す。

シグネチャまたは形状

static inline float8 float8_mul(const float8 val1, const float8 val2)

入力と出力

  • **入力**: val1 (float8), val2 (float8)
  • **出力**: 乗算結果の float8 値。

前提条件と不変条件

  • 本関数はインライン関数として実装されている。
  • 演算結果が無限大（inf）になる場合、入力値が元々無限大でない限り、float_overflow_error() を呼び出して処理を中断する。
  • 演算結果がゼロ（0.0）になる場合、入力値のいずれもゼロでない限り、float_underflow_error() を呼び出して処理を中断する。

アルゴリズムまたは振る舞い

1. val1 と val2 の積を計算する。 2. isinf() を用いて結果が無限大か判定し、入力値が有限であればオーバーフローエラーを報告する。 3. 結果が 0.0 か判定し、入力値が非ゼロであればアンダーフローエラーを報告する。 4. 正常な場合は計算結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c
  • src/backend/utils/adt/cash.c
  • src/backend/utils/adt/float.c
  • src/backend/utils/adt/geo_ops.c
  • **呼び出し先**:
  • float_overflow_error() (エラー報告用)
  • float_underflow_error() (エラー報告用)
  • isinf() (標準ライブラリ <math.h>)

関連データ構造

  • float8: PostgreSQLにおける倍精度浮動小数点型。

失敗モードと境界条件

  • **オーバーフロー**: 演算結果が inf になり、かつ入力が有限値である場合に発生。
  • **アンダーフロー**: 演算結果が 0.0 になり、かつ入力が非ゼロである場合に発生。

保守担当者向けメモ

  • 本関数はパフォーマンスを重視してインライン化されている。
  • エラー報告関数（float_overflow_error 等）は pg_noinline 属性を持ち、コードの肥大化を防ぐ設計となっている。
  • NaNの扱いや比較については、本ヘッダー内の他の関数群と一貫したルールが適用される。
