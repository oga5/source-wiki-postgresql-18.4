json_parse_manifest_incremental_shutdown

定義

  • 種別: function
  • 定義箇所: src/common/parse_manifest.c:168

目的

インクリメンタルなバックアップマニフェスト解析で使用されたリソースを解放し、解析状態を破棄します。

シグネチャまたは形状

void json_parse_manifest_incremental_shutdown(JsonManifestParseIncrementalState *incstate)

入力と出力

  • **入力**: incstate - 解析状態を保持する JsonManifestParseIncrementalState 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • incstate は json_parse_manifest_incremental_init によって正常に初期化されている必要があります。
  • incstate->manifest_ctx は本関数呼び出し時点で既に解放済みである必要があります。

アルゴリズムまたは振る舞い

1. incstate->sem.semstate を pfree で解放します。 2. incstate->lex（JSON字句解析コンテキスト）を freeJsonLexContext で解放します。 3. 最後に incstate 自体を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c:243
  • src/bin/pg_combinebackup/load_manifest.c:211
  • src/bin/pg_verifybackup/pg_verifybackup.c:491
  • **呼び出し先**:
  • pfree (メモリ解放)
  • freeJsonLexContext (JSON字句解析器のクリーンアップ)

関連データ構造

  • JsonManifestParseIncrementalState: 解析のライフサイクル全体を管理する構造体。

失敗モードと境界条件

  • incstate が NULL の場合、pfree 等の内部関数でセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数はフロントエンド（postgres_fe.h）環境での使用を想定しています。
  • manifest_ctx の解放は呼び出し元または別のフェーズで完了している前提のため、二重解放を避けるよう注意してください。
