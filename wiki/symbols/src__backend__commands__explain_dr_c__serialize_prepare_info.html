serialize_prepare_info

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_dr.c:51

目的

EXPLAIN (SERIALIZE) 実行時に、各属性のシリアライズ（出力変換）に必要な関数情報をキャッシュし、SerializeDestReceiver に設定する。

シグネチャまたは形状

static void serialize_prepare_info(SerializeDestReceiver *receiver, TupleDesc typeinfo, int nattrs)

入力と出力

  • **入力**:
  • receiver: 計測状態を保持する SerializeDestReceiver 構造体へのポインタ。
  • typeinfo: タプルの属性定義情報 (TupleDesc)。
  • nattrs: 属性数。
  • **出力**: なし（receiver 構造体内部の finfos および attrinfo を更新）。

前提条件と不変条件

  • receiver->format が 0（テキスト形式）または 1（バイナリ形式）であること。
  • nattrs > 0 の場合、receiver->finfos に FmgrInfo の配列が割り当てられる。

アルゴリズムまたは振る舞い

1. 既存の receiver->finfos が存在する場合は解放する。 2. nattrs が 0 以下の場合は早期リターンする。 3. nattrs 分の FmgrInfo 配列をメモリ確保する。 4. 各属性について、receiver->format に応じて以下の処理を行う：

  • **format 0 (Text)**: getTypeOutputInfo を使用して出力関数を特定し、fmgr_info でキャッシュする。
  • **format 1 (Binary)**: getTypeBinaryOutputInfo を使用してバイナリ送信関数を特定し、fmgr_info でキャッシュする。
  • 上記以外の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain_dr.c 内の処理フローから呼び出される。
  • **呼び出し先**:
  • pfree, palloc0 (メモリ管理)
  • TupleDescAttr (属性取得)
  • getTypeOutputInfo, getTypeBinaryOutputInfo (型情報取得)
  • fmgr_info (関数情報キャッシュ)
  • ereport (エラー処理)

関連データ構造

  • SerializeDestReceiver: 本関数が情報を格納する主要な構造体。
  • FmgrInfo: 関数呼び出しに必要な情報を保持する構造体。
  • TupleDesc: タプルのメタデータ。

失敗モードと境界条件

  • receiver->format が定義外の値（0, 1 以外）である場合、エラーが発生する。
  • nattrs が 0 以下の場合はキャッシュの初期化を行わず終了する。

保守担当者向けメモ

  • 本関数は EXPLAIN (SERIALIZE) の初期化フェーズで呼び出されます。
  • fmgr_info を使用して関数ポインタをキャッシュすることで、タプル処理ループ内での検索コストを削減しています。
