astreamer_verify

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_verifybackup/astreamer_verify.c:21

目的

pg_verifybackup において、tarアーカイブのストリーム処理中に検証状態を保持・管理するための構造体です。

シグネチャまたは形状

typedef struct astreamer_verify
{
    astreamer    base;
    verifier_context *context;
    char        *archive_name;
    Oid         tblspc_oid;
    manifest_file *mfile;
    bool        verify_checksum;
    bool        verify_control_data;
    pg_checksum_context *checksum_ctx;
    uint64      checksum_bytes;
    ControlFileData control_file;
    uint64      control_file_bytes;
} astreamer_verify;

入力と出力

本構造体は astreamer インターフェースの一部として機能します。入力としてアーカイブの各メンバーデータを受け取り、内部状態（チェックサム計算コンテキストや制御ファイルバッファ）を更新します。出力は直接的な戻り値ではなく、検証結果の蓄積および比較処理を通じて行われます。

前提条件と不変条件

  • base フィールド（astreamer）は、ストリーマーの基底として初期化されている必要があります。
  • 初期化後に変更されないフィールド（context, archive_name, tblspc_oid）と、メンバーごとにリセットされるフィールド（mfile, checksum_ctx 等）が明確に分離されています。

アルゴリズムまたは振る舞い

  • **状態保持**: アーカイブの各メンバー処理において、検証対象かどうかのフラグ（verify_checksum 等）と、進行状況（checksum_bytes 等）を保持します。
  • **制御ファイル処理**: ControlFileData 構造体を保持し、制御ファイルのストリームデータが到着するたびにバッファリングと検証を行います。
  • **ライフサイクル**: astreamer_verify_content_new で生成され、各メンバーのヘッダー・コンテンツ・トレーラー処理を経て、最終的に解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_verifybackup.c 等のパイプライン構築コードから、astreamer_verify_content_new を通じてインスタンス化されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、member_verify_header、member_compute_checksum、member_verify_checksum などの関数が本構造体を引数として受け取り、その状態を操作します。

関連データ構造

  • astreamer: 基底構造体。
  • verifier_context: 検証全体のコンテキスト。
  • manifest_file: マニフェスト内のファイル情報。
  • pg_checksum_context: チェックサム計算用コンテキスト。
  • ControlFileData: 制御ファイルの内容を保持する構造体。

失敗モードと境界条件

  • マニフェストに存在しないファイルや、チェックサムが不一致の場合、検証処理が失敗します。
  • 制御ファイルのサイズが ControlFileData の期待値と一致しない場合、検証エラーとなります。

保守担当者向けメモ

  • 本構造体は astreamer_ops を通じて操作されます。フィールドの追加や変更を行う際は、member_reset_info で適切にリセット処理が行われるか確認してください。
  • 制御ファイルの検証は、マニフェストのバージョンが1より大きい場合にのみ有効化される設計です。
