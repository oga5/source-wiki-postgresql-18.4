ArchiveMode

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlog.h:63

目的

PostgreSQL の WAL（先行書き込みログ）アーカイブ処理の有効化状態を定義する。

シグネチャまたは形状

typedef enum ArchiveMode
{
    ARCHIVE_MODE_OFF = 0,
    ARCHIVE_MODE_ON,
    ARCHIVE_MODE_ALWAYS,
} ArchiveMode;

入力と出力

  • ARCHIVE_MODE_OFF: アーカイブを無効化する。
  • ARCHIVE_MODE_ON: 通常稼働時にアーカイブを有効化する。
  • ARCHIVE_MODE_ALWAYS: リカバリ中を含め、常にアーカイブを有効化する。

前提条件と不変条件

  • wal_level 設定と組み合わせて、WAL アーカイブの動作を制御するガードレールとして機能する。

アルゴリズムまたは振る舞い

  • ARCHIVE_MODE_OFF: アーカイブ処理は実行されない。
  • ARCHIVE_MODE_ON: サーバーが通常稼働している間、WAL セグメントのアーカイブが実行される。
  • ARCHIVE_MODE_ALWAYS: サーバーがリカバリ状態にある場合でも、WAL セグメントのアーカイブが継続される。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_dump.c において、アーカイブ形式の解析（parseArchiveFormat）や設定保持に使用されている。
  • 呼び出し先: 提供されたソース情報からは、特定の関数呼び出し関係は特定できない。

関連データ構造

  • WalLevel: WAL の詳細度を決定する列挙型であり、本モードと連携して動作する。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 本シンボルは src/include/access/xlog.h で定義されており、WAL サブシステムの重要な設定状態を示す。
