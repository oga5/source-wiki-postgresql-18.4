pg_logical_slot_get_changes

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logicalfuncs.c:330

目的

論理レプリケーションスロットからWALの変更内容を抽出し、SQLインターフェースを通じて結果セットとして返却します。

シグネチャまたは形状

Datum pg_logical_slot_get_changes(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し時の引数群）。
  • **出力**: Datum 型（抽出された変更内容を含む Tuplestore の結果セット）。

前提条件と不変条件

  • 本関数はSQLインターフェースとして呼び出されることを前提としています。
  • 内部的に pg_logical_slot_get_changes_guts を呼び出し、変更の取得（get）とテキスト形式での出力を指定して実行します。

アルゴリズムまたは振る舞い

1. pg_logical_slot_get_changes_guts を呼び出します。 2. 第2引数に true を渡し、読み取り位置を更新（確認）する動作を指定します。 3. 第3引数に false を渡し、バイナリ形式ではなくテキスト形式での出力を指定します。 4. 取得した結果を呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（SQLコマンド経由での実行）。
  • **呼び出し先**: pg_logical_slot_get_changes_guts。

関連データ構造

  • Tuplestore: 抽出された変更内容を一時的に保持する構造体。
  • DecodingOutputState: 出力プラグインから受け取ったデータを変換・保持するための状態構造体。

失敗モードと境界条件

  • 内部関数 pg_logical_slot_get_changes_guts のエラー処理（PG_TRY/PG_CATCH ブロック等）に依存します。
  • 指定されたスロットが存在しない場合や、論理デコーディングのコンテキスト作成に失敗した場合はエラーが発生します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実体は pg_logical_slot_get_changes_guts です。ロジックの変更はそちらで行ってください。
  • CurrentResourceOwner の管理など、トランザクションに関連する副作用については logicalfuncs.c の概要を参照してください。
