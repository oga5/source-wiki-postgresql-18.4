spgInnerConsistentIn

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist.h:132

目的

SP-GiSTインデックスの内部ノードを走査する際、検索条件（ScanKey）に基づいて次に探索すべき子ノードを特定するための入力情報を保持する構造体です。

シグネチャまたは形状

typedef struct spgInnerConsistentIn
{
    ScanKey     scankeys;
    ScanKey     orderbys;
    int         nkeys;
    int         norderbys;
    Datum       reconstructedValue;
    void       *traversalValue;
    MemoryContext traversalMemoryContext;
    int         level;
    bool        returnData;
    bool        allTheSame;
    bool        hasPrefix;
    Datum       prefixDatum;
    int         nNodes;
    Datum      *nodeLabels;
} spgInnerConsistentIn;

入力と出力

  • **入力**: 検索条件（scankeys, orderbys）、現在のノード情報（allTheSame, prefixDatum, nodeLabels 等）、および走査状態（level, traversalValue）。
  • **出力**: 本構造体自体は入力専用です。結果は対応する spgInnerConsistentOut 構造体に格納されます。

前提条件と不変条件

  • scankeys および orderbys は、それぞれ nkeys および norderbys の長さを持つ配列であること。
  • traversalMemoryContext は、新しい traversalValue を割り当てるための有効なメモリコンテキストであること。
  • nodeLabels は nNodes 個の要素を持つ配列、またはノードにラベルがない場合は NULL であること。

アルゴリズムまたは振る舞い

SP-GiSTの検索処理において、インデックスの内部ノードに到達した際に呼び出されます。演算子クラスは、この構造体に含まれる情報を使用して、現在のノードの各子ノードが検索条件を満たす可能性があるかを判定し、探索を継続すべきノードを spgInnerConsistentOut に出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgscan.c（spgInitInnerConsistentIn を介した初期化）
  • 各種演算子クラスの実装（spgkdtreeproc.c, spgquadtreeproc.c, spgtextproc.c, geo_spgist.c, network_spgist.c, rangetypes_spgist.c, spgist_name_ops.c）
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • spgInnerConsistentOut: 検索結果を格納する対となる構造体。
  • ScanKey: 検索演算子と比較値を保持する標準的なPostgreSQL構造体。
  • Datum: インデックスに格納された値やノードラベルを表す汎用型。

失敗モードと境界条件

  • nNodes が 0 の場合、子ノードが存在しないため、それ以上の走査は行われません。
  • nodeLabels が NULL の場合、ラベルに基づかない分岐処理が必要です。
  • traversalValue が NULL の場合、演算子クラスは初期状態として処理を行う必要があります。

保守担当者向けメモ

  • traversalValue は検索中にノード間で状態を引き継ぐために使用されます。複雑な検索アルゴリズムを実装する際は、このポインタを適切に管理してください。
  • allTheSame フラグが立っている場合、すべてのノードが同じ値を持つとみなされるため、通常の分岐ロジックとは異なる処理が必要になる場合があります。
