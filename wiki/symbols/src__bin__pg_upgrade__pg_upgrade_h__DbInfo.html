DbInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.h:196

目的

PostgreSQLのアップグレード対象となる個々のデータベースに関するメタデータおよび、そのデータベース内に含まれるリレーションや論理スロットの情報を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	Oid			db_oid;
	char	   *db_name;
	char		db_tablespace[MAXPGPATH];
	RelInfoArr	rel_arr;
	LogicalSlotInfoArr slot_arr;
} DbInfo;

入力と出力

本構造体はデータ保持用であり、関数ではありません。info.c 等のロジックによって、旧クラスタおよび新クラスタから収集された情報が各フィールドに格納されます。

前提条件と不変条件

  • db_tablespace は MAXPGPATH バイトの固定長配列であり、データベースのデフォルトテーブルスペースパスを保持します。
  • rel_arr および slot_arr は、それぞれ当該データベース内のリレーションおよび論理スロットの情報を管理する配列構造体です。

アルゴリズムまたは振る舞い

本構造体自体は状態保持に特化しています。アップグレードプロセスにおいて、pg_upgrade はこの構造体を参照して、データベースごとの互換性チェック（check.c）や、ライブラリのロード確認（function.c）を実行します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/bin/pg_upgrade/check.c: 各種チェック関数（process_data_type_check 等）の引数として渡されます。
  • src/bin/pg_upgrade/dump.c: old_cluster.dbarr.dbs から個別の DbInfo が取得されます。
  • src/bin/pg_upgrade/function.c: process_loadable_libraries で参照されます。
  • src/bin/pg_upgrade/info.c: 比較ロジック等で参照されます。
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • RelInfoArr: データベース内のリレーション情報を保持する配列。
  • LogicalSlotInfoArr: データベース内の論理レプリケーションスロット情報を保持する配列。
  • ClusterInfo: DbInfo の配列（DbInfoArr）を保持する上位構造体。

失敗モードと境界条件

  • db_name は動的に割り当てられた文字列であるため、メモリ管理に依存します。
  • db_tablespace は MAXPGPATH を超えるパス長を持つ場合、切り詰めやバッファオーバーフローのリスクを伴う可能性があります（PostgreSQLの標準的なパス長制限に準拠）。

保守担当者向けメモ

  • DbInfo は info.c で構築されます。データベースの構成要素に変更を加える際は、RelInfoArr や LogicalSlotInfoArr の整合性が保たれているかを確認してください。
  • check.c 内の多くの関数が DbInfo* を引数に取るため、この構造体のフィールド追加・削除は広範囲な影響を及ぼします。
