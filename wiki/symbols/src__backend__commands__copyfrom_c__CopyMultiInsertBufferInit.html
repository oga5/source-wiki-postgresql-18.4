CopyMultiInsertBufferInit

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfrom.c:362

目的

COPY FROM コマンドにおけるバルク挿入用のバッファ領域を初期化し、管理構造体を生成する。

シグネチャまたは形状

CopyMultiInsertBuffer *CopyMultiInsertBufferInit(ResultRelInfo *rri)

入力と出力

  • **入力**: ResultRelInfo *rri（挿入対象テーブルの情報を保持する構造体）
  • **出力**: 初期化された CopyMultiInsertBuffer 構造体へのポインタ。

前提条件と不変条件

  • rri は有効な ResultRelInfo を指している必要がある。
  • 戻り値のバッファは、MAX_BUFFERED_TUPLES 分の TupleTableSlot を保持できるメモリ領域を確保する。

アルゴリズムまたは振る舞い

1. CopyMultiInsertBuffer 構造体分のメモリを palloc で確保する。 2. バッファ内のスロット配列（slots）を memset でゼロクリアする。 3. resultRelInfo フィールドに引数の rri を設定する。 4. rri->ri_FdwRoutine が NULL（外部データラッパーを使用しない）の場合、GetBulkInsertState() を呼び出して bistate を初期化する。それ以外の場合は bistate を NULL とする。 5. 使用済みスロット数 nused を 0 に初期化して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c 内の CopyMultiInsertInfoSetupBuffer から呼び出される。
  • **呼び出し先**: GetBulkInsertState()（条件付き）。

関連データ構造

  • CopyMultiInsertBuffer: バルク挿入の状態を保持する構造体。
  • ResultRelInfo: ターゲットテーブルのメタデータ。
  • BulkInsertState: バルク挿入の最適化状態を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • rri が NULL の場合、関数内でデリファレンスが発生しクラッシュする可能性がある（呼び出し元でのガードが必要）。

保守担当者向けメモ

  • 本関数は COPY 実行時のメモリコンテキスト（copycontext）内で呼び出されることを前提としている。
  • MAX_BUFFERED_TUPLES は copyfrom_internal.h で定義されている定数であり、バッファサイズを変更する場合はこの定数を確認すること。
