PostPrepare_Twophase

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/twophase.c:343

目的

準備済みトランザクション（Prepared Transaction）の処理完了後、現在のバックエンドと当該トランザクションの関連付けを解除し、共有メモリ上の管理状態を更新する。

シグネチャまたは形状

void PostPrepare_Twophase(void)

入力と出力

  • 入力: なし（グローバル変数 MyLockedGxact を参照）
  • 出力: なし（副作用として共有メモリ状態を更新）

前提条件と不変条件

  • MyLockedGxact が有効な準備済みトランザクションを指している必要がある。
  • TwoPhaseStateLock を介して共有メモリ上の GlobalTransactionData を排他的に操作する。

アルゴリズムまたは振る舞い

1. TwoPhaseStateLock を排他モードで取得する。 2. 現在のバックエンドが保持していた MyLockedGxact->locking_backend を INVALID_PROC_NUMBER に設定し、準備済みトランザクションとバックエンドの紐付けを解除する。 3. TwoPhaseStateLock を解放する。 4. ローカルポインタ MyLockedGxact を NULL にリセットし、現在のバックエンドが準備済みトランザクションを保持していない状態にする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xact.c（PREPARE TRANSACTION 実行の最終段階で呼び出される）
  • 呼び出し先:
  • LWLockAcquire / LWLockRelease（共有メモリ保護のため）

関連データ構造

  • GlobalTransactionData: 準備済みトランザクションの情報を保持する共有メモリ上の構造体。
  • TwoPhaseStateLock: 準備済みトランザクションの状態を保護するロック。

失敗モードと境界条件

  • TwoPhaseStateLock の取得失敗や、MyLockedGxact が NULL の状態で呼び出された場合の動作については、本ソースコード内では明示的なエラーハンドリングは記述されていない。

保守担当者向けメモ

  • 本関数は、トランザクションが準備状態へ移行した直後に、バックエンドがそのトランザクションの所有権を放棄するために使用される。
  • MyLockedGxact は twophase.c 内で管理されるバックエンドローカルなポインタであり、この関数を呼び出すことでバックエンドは準備済みトランザクションから完全に切り離される。
