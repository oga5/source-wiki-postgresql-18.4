pg_GSS_have_cred_cache

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-gssapi-common.c:60

目的

GSSAPI認証において、利用可能な資格情報（Credential Cache）が存在するかを確認し、取得する。

シグネチャまたは形状

bool pg_GSS_have_cred_cache(gss_cred_id_t *cred_out)

入力と出力

  • **入力**: cred_out (資格情報を格納するための gss_cred_id_t へのポインタ)
  • **出力**:
  • 戻り値: 資格情報の取得に成功した場合は true、失敗した場合は false。
  • cred_out: 成功時は取得した資格情報、失敗時は NULL が設定される。

前提条件と不変条件

  • ENABLE_GSS マクロが定義されている環境でのみ有効。
  • gss_acquire_cred を使用して資格情報を取得する。

アルゴリズムまたは振る舞い

1. gss_acquire_cred を呼び出し、GSS_C_INITIATE モードで資格情報の取得を試みる。 2. 取得結果（major ステータス）が GSS_S_COMPLETE であれば、取得した資格情報を cred_out に格納し true を返す。 3. 取得に失敗した場合は、cred_out に NULL を設定し false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-auth.c
  • src/interfaces/libpq/fe-connect.c
  • src/interfaces/libpq/fe-secure-gssapi.c
  • **呼び出し先**:
  • GSSAPIライブラリ関数: gss_acquire_cred

関連データ構造

  • gss_cred_id_t: GSSAPIの資格情報を表す不透明な型。

失敗モードと境界条件

  • gss_acquire_cred が GSS_S_COMPLETE 以外のステータスを返した場合、資格情報の取得失敗として処理される。

保守担当者向けメモ

  • 本関数は libpq の認証プロセスにおいて、GSSAPI認証の可否を判断するゲートキーパーとして機能する。
  • 呼び出し側で cred_out に渡されるポインタが有効であることを確認すること。
