rule

定義

  • 種別: struct
  • 定義箇所: src/timezone/localtime.c:72

目的

タイムゾーンの遷移ルールを定義するための構造体です。主に POSIX 形式のタイムゾーン文字列の解析や、夏時間（DST）の切り替わりタイミングを計算する際に使用されます。

シグネチャまたは形状

struct rule
{
    enum r_type r_type;     /* ルールのタイプ */
    int         r_day;      /* ルールの日付 */
    int         r_week;     /* ルールの週番号 */
    int         r_mon;      /* ルールの月番号 */
    int32       r_time;     /* ルールの遷移時刻 */
};

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。入力データは tzparse 等の解析関数によって本構造体の各フィールドに格納され、その後 state 構造体の構築や時刻変換計算のパラメータとして利用されます。

前提条件と不変条件

  • r_type は r_type 列挙型で定義された有効な値である必要があります。
  • 各フィールドは、IANA タイムゾーンデータベースまたは POSIX TZ 文字列の仕様に準拠した範囲の値である必要があります。

アルゴリズムまたは振る舞い

本構造体は、特定の期間における時刻の遷移条件（例：毎年 3 月の第 2 日曜日の 02:00 に切り替わる等）を表現します。localtime.c 内のロジックは、この構造体を参照して、指定された時刻がどのタイムゾーンオフセットに該当するかを判定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/timezone/localtime.c 内の tzparse 関数が、POSIX TZ 文字列を解析する際に本構造体をインスタンス化します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。
  • 注記: ツールが検出した contrib/ 配下の rule という単語を含むコードは、本構造体とは無関係な変数名やコメントであり、本シンボルの呼び出し元ではありません。

関連データ構造

  • r_type: ルールの種類を識別する列挙型。
  • state: タイムゾーン情報を保持するメインの構造体。本構造体で定義されたルールが state に組み込まれます。

失敗モードと境界条件

  • 不正な POSIX TZ 文字列が入力された場合、tzparse による本構造体への値の割り当てが失敗、あるいは不整合な値が設定される可能性があります。

保守担当者向けメモ

  • 本構造体は zic.c（タイムゾーンコンパイラ）と共通の定義を使用しています。変更を加える際は、src/timezone/zic.c への影響を考慮してください。
  • r_time は int32 型であり、秒単位のオフセットを保持します。極端な値が設定された場合のオーバーフローには注意が必要です。
