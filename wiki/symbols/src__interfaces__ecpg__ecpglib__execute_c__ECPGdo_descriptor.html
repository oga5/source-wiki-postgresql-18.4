ECPGdo_descriptor

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:2289

目的

SQL 記述子（Descriptor）を使用して SQL 文を実行するためのラッパー関数です。

シグネチャまたは形状

bool ECPGdo_descriptor(int line, const char *connection, const char *descriptor, const char *query)

入力と出力

  • **入力**:
  • line: 呼び出し元のソースコード行番号。
  • connection: 接続名（NULL の場合はデフォルト接続）。
  • descriptor: 使用する SQL 記述子の名前。
  • query: 実行する SQL 文。
  • **出力**:
  • 実行が成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • 内部的に ECPGdo を呼び出し、互換モードとして ECPG_COMPAT_PGSQL を強制します。
  • 記述子を使用する実行形式であるため、入力パラメータとして ECPGt_descriptor を指定して ECPGdo へ転送します。

アルゴリズムまたは振る舞い

1. 引数として受け取った line、connection、descriptor、query を、ECPGdo 関数の引数へマッピングします。 2. ECPGt_EOIT（入力終了）および ECPGt_EORT（結果終了）のマーカーを付与し、記述子を指定した形式で ECPGdo を呼び出します。 3. ECPGdo の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/include/ecpglib.h に宣言があり、ECPG プリプロセッサによって生成されたコードから呼び出されることが想定されます。
  • **呼び出し先**: ECPGdo（同ファイル内）。

関連データ構造

  • descriptor: SQL 記述子を管理する構造体（src/interfaces/ecpg/ecpglib/descriptor.c に関連ロジックが存在）。

失敗モードと境界条件

  • ECPGdo の実行結果に依存します。接続が確立されていない場合や、SQL 文の構文エラー、記述子の名前が不正な場合などに false を返します。

保守担当者向けメモ

  • 本関数は ECPGdo への単なるインターフェース変換層です。ロジックの変更が必要な場合は、ECPGdo の実装を確認してください。
  • 記述子に関連する詳細な処理は src/interfaces/ecpg/ecpglib/descriptor.c で管理されています。
