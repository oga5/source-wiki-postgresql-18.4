TParserCopyClose

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/wparser_def.c:396

目的

TParserCopyInit によって複製されたパーサーインスタンスおよびその内部状態スタックをメモリから解放し、破棄する。

シグネチャまたは形状

static void TParserCopyClose(TParser *prs)

入力と出力

  • **入力**: prs (TParser *) - 解放対象のパーサー構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • prs は有効なメモリ領域を指している必要がある。
  • prs->state が指すスタック構造は、pfree によって安全に解放可能であること。

アルゴリズムまたは振る舞い

1. prs->state が指すスタックをループで辿る。 2. 各スタック要素（TParserPosition）を pfree で解放し、prs->state を次の要素へ更新する。 3. WPARSER_TRACE が定義されている場合、デバッグ出力を実行する。 4. 最後に prs 自体を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/wparser_def.c 内の関数（tmpprs を引数として呼び出し）。
  • **呼び出し先**:
  • pfree (PostgreSQL メモリ管理関数)

関連データ構造

  • TParser: 解析状態を保持するメイン構造体。
  • TParserPosition: 解析状態のスタック要素。

失敗モードと境界条件

  • prs が NULL の場合、pfree の仕様に依存する（通常はセグメンテーション違反を引き起こす可能性がある）。
  • スタックが空の場合、ループは実行されず prs の解放のみが行われる。

保守担当者向けメモ

  • 本関数は複製されたパーサー専用のクリーンアップ処理である。通常のパーサー終了処理である TParserClose と混同しないこと。
  • WPARSER_TRACE マクロによるデバッグ出力は標準エラー出力 (stderr) に行われる。
