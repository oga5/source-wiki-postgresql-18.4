pg_malloc_extended

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:64

目的

PostgreSQLのフロントエンドプログラムにおいて、指定されたフラグに基づき安全なメモリ確保を行うための汎用インターフェースを提供します。

シグネチャまたは形状

void *pg_malloc_extended(size_t size, int flags);

入力と出力

  • **入力**:
  • size: 確保するメモリサイズ（バイト単位）。
  • flags: メモリ確保の挙動を制御するビットフラグ（MCXT_ALLOC_ZERO や MCXT_ALLOC_NO_OOM など）。
  • **出力**:
  • 確保されたメモリ領域へのポインタを返します。メモリ確保に失敗し、かつ MCXT_ALLOC_NO_OOM フラグが指定されていない場合は、エラーメッセージを出力してプロセスを終了（exit(EXIT_FAILURE)）します。

前提条件と不変条件

  • SIZE_MAX / 2 を超える確保要求は、物理メモリの有無に関わらずエラーとして扱われます。
  • size が 0 の場合、移植性の観点から malloc(1) として扱われます。

アルゴリズムまたは振る舞い

1. 内部関数 pg_malloc_internal(size, flags) を呼び出します。 2. pg_malloc_internal 内で、標準の malloc を実行し、必要に応じてメモリのゼロクリアやエラーハンドリングを行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_ctl/pg_ctl.c
  • src/bin/pg_dump/compress_zstd.c
  • src/bin/psql/large_obj.c
  • src/common/logging.c
  • **呼び出し先**:
  • pg_malloc_internal（直接の呼び出し先）

関連データ構造

  • MCXT_ALLOC_ZERO: メモリ確保後にゼロクリアを行うためのフラグ。
  • MCXT_ALLOC_NO_OOM: メモリ不足時にプロセスを終了させず、呼び出し側に制御を戻すためのフラグ。

失敗モードと境界条件

  • メモリ確保失敗時: MCXT_ALLOC_NO_OOM が指定されていない場合、stderr にエラーを出力してプロセスを終了します。
  • サイズ制限: SIZE_MAX / 2 を超えるサイズ要求は、バックエンドとの整合性維持のため拒否されます。

保守担当者向けメモ

  • 本関数は FRONTEND マクロが定義されている環境でのみ有効です。
  • pg_malloc_internal のラッパーとして機能しており、将来的なメモリ管理ポリシーの変更は pg_malloc_internal 側で吸収される設計となっています。
