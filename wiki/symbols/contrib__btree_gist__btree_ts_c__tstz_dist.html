tstz_dist

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_ts.c:170

目的

2つの timestamptz 型の値の間の距離を Interval 型として計算する。

シグネチャまたは形状

Datum tstz_dist(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS を介して2つの TimestampTz 型の値を受け取る。
  • 出力: Interval 型のポインタを Datum として返す。

前提条件と不変条件

  • 入力値のいずれかが有限でない（TIMESTAMP_NOT_FINITE）場合、距離は最大値として扱われる。
  • 正常な計算結果は abs_interval によって絶対値に変換される。

アルゴリズムまたは振る舞い

1. 入力された2つの TimestampTz 値（a, b）を確認する。 2. いずれかが有限でない場合、day, month, time をそれぞれ最大値（INT_MAX, INT_MAX, PG_INT64_MAX）に設定した Interval を割り当てて返す。 3. 両方とも有限の場合、timestamp_mi 関数を呼び出して差分を計算する。 4. 得られた Interval に対して abs_interval を適用し、その絶対値を返す。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内に直接的な呼び出し元は明示されていない。
  • 呼び出し先:
  • timestamp_mi (PostgreSQL内部関数)
  • abs_interval (PostgreSQL内部関数)
  • palloc (メモリ割り当て)

関連データ構造

  • TimestampTz: タイムスタンプ（タイムゾーン付き）型。
  • Interval: 時間間隔を表す構造体。

失敗モードと境界条件

  • 入力値が無限大（infinity または -infinity）の場合、計算は行われず、定義された最大値の Interval が返される。
  • メモリ不足時には palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は timestamptz 型の距離計算を担う。timestamp 型の距離計算については ts_dist を参照すること。
  • timestamp_mi の結果を abs_interval でラップする設計となっており、負の距離は発生しない。
