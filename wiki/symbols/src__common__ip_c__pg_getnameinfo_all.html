pg_getnameinfo_all

定義

  • 種別: function
  • 定義箇所: src/common/ip.c:113

目的

ソケットアドレスからホスト名およびサービス名を解決し、文字列として取得する。標準的な getnameinfo API をラップし、Unix ドメインソケットを透過的にサポートする。

シグネチャまたは形状

int pg_getnameinfo_all(const struct sockaddr_storage *addr, int salen,
                       char *node, int nodelen,
                       char *service, int servicelen,
                       int flags);

入力と出力

  • **入力**:
  • addr: 解決対象のソケットアドレス構造体。
  • salen: addr のサイズ。
  • node, nodelen: ホスト名を格納するバッファとそのサイズ。
  • service, servicelen: サービス名を格納するバッファとそのサイズ。
  • flags: getnameinfo に渡すフラグ。
  • **出力**:
  • 成功時は 0、失敗時は非ゼロの整数を返す。

前提条件と不変条件

  • addr が AF_UNIX ファミリーである場合、内部で getnameinfo_unix が呼び出される。
  • それ以外の場合は標準の getnameinfo が呼び出される。
  • 解決に失敗した場合、呼び出し側の安全性を確保するため、node および service バッファには "???" が書き込まれる。

アルゴリズムまたは振る舞い

1. addr->ss_family を確認し、AF_UNIX であれば Unix ソケット専用の解決処理へ委譲する。 2. それ以外の場合は標準の getnameinfo を実行する。 3. 解決処理の結果（rc）が 0 以外（エラー）の場合、提供されたバッファが有効であれば、それぞれに "???" をコピーする。 4. 最終的な解決結果（成功時は 0、失敗時はエラーコード）を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth.c
  • src/backend/libpq/hba.c
  • src/backend/libpq/pqcomm.c
  • src/backend/tcop/backend_startup.c
  • src/backend/utils/adt/hbafuncs.c
  • **呼び出し先**:
  • getnameinfo_unix (内部関数)
  • getnameinfo (標準ライブラリ)
  • strlcpy (エラー時のフォールバック用)

関連データ構造

  • struct sockaddr_storage: 汎用ソケットアドレス構造体。
  • struct sockaddr_un: Unix ドメインソケット用アドレス構造体。

失敗モードと境界条件

  • 名前解決が失敗した場合、関数はエラーコードを返すと同時に、バッファを "???" で埋めることで呼び出し側のクラッシュや未定義動作を防止する。
  • node や service が NULL の場合、それらへの書き込みはスキップされる。

保守担当者向けメモ

  • 本関数は AF_UNIX とそれ以外のプロトコルを抽象化する役割を持つ。
  • 解決失敗時の "???" 補完は、ログ出力や認証処理において呼び出し側がエラーチェックを省略しても安全に動作するように設計されている。
