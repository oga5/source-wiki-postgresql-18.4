InitLockManagerAccess

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lock.c:503

目的

バックエンドプロセスごとのローカルメモリ上に、ロックの保持数やリソース所有者情報を管理するためのハッシュテーブル（LOCALLOCK 構造体用）を初期化する。

シグネチャまたは形状

void InitLockManagerAccess(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 LockMethodLocalHash にハッシュテーブルへのポインタを格納する）

前提条件と不変条件

  • 本関数は、ロックマネージャのローカル管理用ハッシュテーブルが未作成の状態で呼び出される必要がある。
  • 作成されるハッシュテーブルは共有メモリではなく、プロセスローカルなメモリ領域に配置される。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を初期化し、キーサイズを LOCALLOCKTAG、エントリサイズを LOCALLOCK に設定する。 2. hash_create を呼び出し、"LOCALLOCK hash" という名称でハッシュテーブルを作成する。 3. 初期サイズを 16 エントリとし、HASH_ELEM および HASH_BLOBS フラグを指定してハッシュテーブルを構成する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/lmgr/lock.c:440
  • src/backend/utils/init/postinit.c:659
  • 呼び出し先:
  • hash_create (PostgreSQL のハッシュテーブルライブラリ関数)

関連データ構造

  • LOCALLOCK: ローカルで管理されるロック情報を保持する構造体。
  • LOCALLOCKTAG: ロックを識別するためのキー構造体。
  • LockMethodLocalHash: 作成されたハッシュテーブルを指すグローバル変数。

失敗モードと境界条件

  • hash_create がメモリ不足等で失敗した場合、PostgreSQL の標準的なエラーハンドリングによりプロセスがアボートする可能性がある。

保守担当者向けメモ

  • 本関数は共有メモリ上のロックテーブル（LockMethodLockHash 等）とは異なり、各バックエンドのローカルメモリのみを対象とする。
  • LOCALLOCK 構造体は、トランザクション終了時のロック解放処理（LockReleaseAll）などで参照される重要なデータ構造である。
