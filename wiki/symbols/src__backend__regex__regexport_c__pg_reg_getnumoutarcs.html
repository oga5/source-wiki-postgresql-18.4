pg_reg_getnumoutarcs

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:133

目的

コンパイル済み正規表現の指定された状態から、LACON（先読み・後読み制約）を透過的に処理した上で、実際に遷移可能なアーク（エッジ）の総数を取得する。

シグネチャまたは形状

int pg_reg_getnumoutarcs(const regex_t *regex, int st)

入力と出力

  • **入力**:
  • regex: コンパイル済みの正規表現オブジェクトへのポインタ。
  • st: 遷移元となる状態のインデックス。
  • **出力**:
  • 指定された状態から遷移可能なアークの数（整数）。

前提条件と不変条件

  • regex は NULL であってはならず、かつ regex->re_magic が REMAGIC と一致している必要がある（assert による検証）。
  • 内部的に struct guts へキャストして search（cnfa）構造体を参照する。

アルゴリズムまたは振る舞い

1. regex 構造体から内部の cnfa（非決定性有限オートマトン）を取得する。 2. 指定された状態 st が有効な範囲内（0 以上かつ cnfa->nstates 未満）にあるか確認する。範囲外の場合は 0 を返す。 3. traverse_lacons 関数を呼び出し、LACON を再帰的に解決しながら、文字を消費する実質的な遷移先をカウントする。 4. カウントされたアーク数を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_regexp.c 内の複数箇所（trgmNFA の解析処理）。
  • **呼び出し先**:
  • traverse_lacons（同ファイル内の内部ヘルパー関数）。

関連データ構造

  • regex_t: 正規表現のコンパイル結果を保持する構造体。
  • struct cnfa: 正規表現の NFA 構造。
  • struct guts: regex_t の内部表現を保持する構造体。

失敗モードと境界条件

  • st が負の値、または cnfa->nstates 以上の場合は、エラーを発生させず 0 を返す。
  • traverse_lacons は再帰関数であるため、極端に深い LACON 構造を持つ正規表現に対しては、スタックオーバーフローを防ぐための保護機構が内部で動作する。

保守担当者向けメモ

  • 本関数は LACON を「常に満足されるもの」としてスキップする設計である。
  • pg_reg_getoutarcs() と対で使用されることを想定している。
  • 呼び出し元で pg_reg_getnumoutarcs() の結果に基づいてバッファを確保し、その後 pg_reg_getoutarcs() を呼び出すフローが一般的である。
