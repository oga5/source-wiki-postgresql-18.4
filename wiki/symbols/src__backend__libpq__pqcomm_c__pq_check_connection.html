pq_check_connection

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqcomm.c:2056

目的

現在のクライアント接続が維持されているか（ソケットがクローズされていないか）を、待機イベントセットを使用して確認します。

シグネチャまたは形状

bool pq_check_connection(void)

入力と出力

  • **入力**: なし（内部的な FeBeWaitSet および MyLatch を参照）
  • **出力**: bool 型。接続が有効な場合は true、ソケットがクローズされている場合は false を返します。

前提条件と不変条件

  • FeBeWaitSet が初期化済みであること。
  • ModifyWaitEvent により、ソケットの監視対象に WL_SOCKET_CLOSED が含まれる状態になります。

アルゴリズムまたは振る舞い

1. ModifyWaitEvent を呼び出し、FeBeWaitSet のソケット監視対象を WL_SOCKET_CLOSED に設定します。 2. WaitEventSetWait を使用して、イベントの発生を待機します。 3. 発生したイベントをループで走査します：

  • WL_SOCKET_CLOSED が検出された場合、直ちに false を返します。
  • WL_LATCH_SET が検出された場合、ResetLatch(MyLatch) を実行してラッチをリセットし、再度 WaitEventSetWait を呼び出します（再試行）。

    • 4. 上記のいずれにも該当しない場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/postgres.c:3371
  • **呼び出し先**:
  • ModifyWaitEvent
  • WaitEventSetWait
  • ResetLatch

関連データ構造

  • WaitEvent: 待機イベントを保持する構造体。
  • FeBeWaitSet: フロントエンド/バックエンド間の通信イベントを管理する待機セット。
  • MyLatch: 現在のプロセスのラッチ。

失敗モードと境界条件

  • ソケットが既に閉じている場合、WL_SOCKET_CLOSED イベントにより false が返されます。
  • ラッチがセットされている場合、ループにより再試行が行われます。

保守担当者向けメモ

  • 本関数はソケットのイベントフィルタを一時的に変更しますが、他の FeBeWaitSet サイトも同様の挙動を前提としているため、復元処理は行われません。
  • ラッチの再試行ロジックは、CHECK_FOR_INTERRUPTS() を跨いでラッチが生存することを期待すべきではないという設計に基づいています。
