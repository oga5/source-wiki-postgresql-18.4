eof_none

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_none.c:162

目的

非圧縮モードにおけるファイルストリームの終端（EOF）状態を判定し、CompressFileHandle インターフェースを通じて呼び出し元に通知します。

シグネチャまたは形状

static bool eof_none(CompressFileHandle *CFH);

入力と出力

  • **入力**: CompressFileHandle *CFH（非圧縮処理用のファイルハンドル構造体）。
  • **出力**: bool 型。ストリームが EOF に達している場合は true、それ以外は false を返します。

前提条件と不変条件

  • CFH->private_data には有効な FILE * が格納されている必要があります。
  • 本関数は CompressFileHandle の関数ポインタ経由で呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. CFH->private_data を FILE * 型にキャストします。 2. 標準ライブラリの feof() を呼び出し、対象ストリームの EOF インジケータを確認します。 3. feof() の戻り値が 0 でない（EOF に達している）場合に true を返し、そうでない場合に false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_none.c 内の InitCompressFileHandleNone 関数において、CFH->eof_func に本関数が代入されます。
  • **呼び出し先**: 標準 C ライブラリの feof()。

関連データ構造

  • CompressFileHandle: 非圧縮 I/O 操作を抽象化するための構造体。private_data メンバを介して FILE * を保持します。

失敗モードと境界条件

  • CFH または CFH->private_data が NULL の場合、未定義の動作（セグメンテーション違反など）が発生する可能性があります。

保守担当者向けメモ

  • 本関数は compress_none.c 内の他の関数と同様、非常に単純なラッパーです。
  • feof() の仕様に基づき、ファイル操作でエラーが発生しただけでは EOF と判定されない点に注意してください。
