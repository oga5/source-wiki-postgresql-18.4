contain_placeholder_references_to

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:492

目的

指定された式（clause）の中に、特定のリレーション（relid）を参照する PlaceHolderVar が含まれているかどうかを判定します。

シグネチャまたは形状

bool contain_placeholder_references_to(PlannerInfo *root, Node *clause, int relid)

入力と出力

  • **入力**:
  • root: クエリ全体のプランナ情報。
  • clause: 検索対象の式ツリー。
  • relid: 検索対象のリレーションID。
  • **出力**:
  • bool: 指定されたリレーションを参照する PlaceHolderVar が見つかった場合は true、それ以外は false を返します。

前提条件と不変条件

  • root->glob->lastPHId が 0 の場合、クエリ内に PlaceHolderVar が存在しないため、即座に false を返します。

アルゴリズムまたは振る舞い

1. クエリ内に PlaceHolderVar が存在するかを確認し、存在しない場合は即座に終了します。 2. 検索用のコンテキスト構造体 contain_placeholder_references_context を初期化し、対象の relid と初期のサブレベル深度（0）を設定します。 3. contain_placeholder_references_walker を呼び出し、式ツリーを再帰的に走査して該当する PlaceHolderVar を探索します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/initsplan.c:1911
  • **呼び出し先**:
  • contain_placeholder_references_walker

関連データ構造

  • PlannerInfo: クエリ全体の最適化情報を保持する構造体。
  • contain_placeholder_references_context: 走査中に使用される状態保持用の構造体。
  • PlaceHolderVar: 検索対象となる特殊なノード。

失敗モードと境界条件

  • lastPHId が 0 の場合、探索を行わずに false を返す最適化が行われます。
  • 式ツリーが空（NULL）の場合の挙動については、contain_placeholder_references_walker の実装に依存します。

保守担当者向けメモ

  • 本関数は PlaceHolderVar の依存関係をチェックするためのエントリーポイントです。
  • 走査ロジックの詳細は contain_placeholder_references_walker に委譲されています。
