remove_rel_from_eclass

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:750

目的

結合削除に伴い、EquivalenceClass (EC) 内の不要なリレーション参照を削除または置換し、データ構造の整合性を維持します。

シグネチャまたは形状

static void remove_rel_from_eclass(EquivalenceClass *ec, SpecialJoinInfo *sjinfo, int relid, int subst)

入力と出力

  • ec: 更新対象の EquivalenceClass 構造体。
  • sjinfo: 関連する SpecialJoinInfo（存在しない場合は NULL）。
  • relid: 削除対象のリレーション ID。
  • subst: relid を置換する新しいリレーション ID（置換不要な場合は 0）。

前提条件と不変条件

  • ec->ec_childmembers が NULL であること（子メンバーが存在する EC の処理には未対応）。

アルゴリズムまたは振る舞い

1. **EC 全体のリレーション集合更新**: ec->ec_relids から relid（および sjinfo があれば ojrelid）を削除または置換します。 2. **メンバー式の更新**: ec->ec_members を走査し、削除対象リレーションを含むメンバーを更新します。更新後にリレーション集合が空になったメンバーはリストから削除されます。 3. **ソース句の更新**: ec->ec_sources 内の RestrictInfo を更新します。sjinfo がない場合は ChangeVarNodesExtended を使用し、ある場合は remove_rel_from_restrictinfo を呼び出します。 4. **派生句の破棄**: 既存の派生句は不要となるため、ec_clear_derived_clauses を呼び出してすべて削除します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/analyzejoins.c 内の結合削除処理（例: remove_useless_joins 関連のロジック）。
  • **呼び出し先**: adjust_relid_set, bms_is_member, foreach_delete_current, ChangeVarNodesExtended, remove_rel_from_restrictinfo, ec_clear_derived_clauses。

関連データ構造

  • EquivalenceClass: 等価クラス。
  • EquivalenceMember: 等価クラスのメンバー。
  • SpecialJoinInfo: 結合の特性情報。
  • RestrictInfo: 制約情報。

失敗モードと境界条件

  • ec->ec_childmembers が非 NULL の場合、Assert により停止します。
  • sjinfo が NULL の場合、ojrelid に関する処理はスキップされます。

保守担当者向けメモ

  • 本関数は EquivalenceClass の内部状態を直接書き換えるため、呼び出し側でプランナの整合性が保たれていることを確認してください。
  • ChangeVarNodesExtended を使用した参照置換は、クエリ内の varno を書き換える複雑な処理の一部です。変更を加える際は、プランナのデータ構造が破壊されないよう注意が必要です。
