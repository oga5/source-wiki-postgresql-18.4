Const

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/primnodes.h:323

目的

クエリ実行時に使用される定数値を保持するためのノード構造体です。

シグネチャまたは形状

typedef struct Const {
    Expr        xpr;
    Oid         consttype;
    int32       consttypmod;
    Oid         constcollid;
    int         constlen;
    Datum       constvalue;
    bool        constisnull;
    bool        constbyval;
    ParseLoc    location;
} Const;

入力と出力

  • **入力**: クエリ解析時に定数として認識された値が、各フィールド（constvalue 等）に格納されます。
  • **出力**: 実行エンジン（ExecEvalExpr 等）がこのノードを評価する際、constvalue に格納された値（またはポインタ）を返します。

前提条件と不変条件

  • constisnull が true の場合、constvalue の内容は未定義です。
  • constbyval が true の場合、値は Datum に直接格納されます。false の場合、Datum は値へのポインタを保持します。
  • location はクエリ内の位置を示し、不明な場合は -1 です。

アルゴリズムまたは振る舞い

  • クエリのハッシュ化（Jumble）において、consttypmod, constcollid, constlen, constvalue, constisnull, constbyval は無視されます。
  • location はクエリのハッシュ化において、パラメータとしてマークするために追跡されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_selfuncs.c（定数の型や値の検証に使用）
  • contrib/postgres_fdw/deparse.c（SQL生成時の定数デパースに使用）
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Expr: 基底構造体。
  • Datum: 定数値そのものを保持する型。
  • Oid: 型識別子。

失敗モードと境界条件

  • constisnull が true の状態で constvalue を参照すると、不定な値を取得する可能性があります。
  • location が -1 の場合、クエリ内の正確な位置情報は失われています。

保守担当者向けメモ

  • pg_node_attr マクロが付与されたフィールドは、クエリのハッシュ化や等価性判定において特別な扱いを受けるため、変更時は注意してください。
  • constvalue の解釈は constbyval の値に依存するため、アクセス時には必ずこのフラグを確認してください。
