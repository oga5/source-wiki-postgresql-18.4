PGLZ_Strategy

定義

  • 種別: struct
  • 定義箇所: src/include/common/pg_lzcompress.h:57

目的

PostgreSQL の LZ 圧縮アルゴリズムにおける圧縮処理の挙動（閾値や探索範囲など）を制御するためのパラメータ群を保持する構造体です。

シグネチャまたは形状

typedef struct PGLZ_Strategy
{
	int32		min_input_size;
	int32		max_input_size;
	int32		min_comp_rate;
	int32		first_success_by;
	int32		match_size_good;
	int32		match_size_drop;
} PGLZ_Strategy;

入力と出力

本構造体は pglz_compress() 関数へ引数として渡され、圧縮アルゴリズムの判断基準として利用されます。

前提条件と不変条件

  • pglz_compress を呼び出す際、この構造体を通じて圧縮の可否や効率が制御されます。
  • TOAST 等の用途では、標準的な設定として PGLZ_strategy_default が提供されています。

アルゴリズムまたは振る舞い

pglz_compress は、この構造体の各フィールドを参照して以下の判断を行います。

  • 圧縮の開始可否（入力サイズに基づく判定）。
  • 圧縮効率の評価（min_comp_rate 等に基づく判定）。
  • マッチング探索の打ち切り条件（match_size_good や match_size_drop 等に基づく判定）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/pg_lzcompress.c 内の pglz_compress 関数が、圧縮戦略を決定するために本構造体を参照します。
  • **定義の参照**: PGLZ_strategy_default および PGLZ_strategy_always が、それぞれ strategy_default_data および strategy_always_data を指す定数として定義されています。

関連データ構造

  • PGLZ_strategy_default: 一般的な用途向けの標準戦略。
  • PGLZ_strategy_always: 常に圧縮を試みるための戦略。

失敗モードと境界条件

  • 構造体内の各パラメータが不適切な値に設定された場合、圧縮効率の低下や、圧縮処理そのもののスキップが発生する可能性があります。
  • 圧縮アルゴリズム自体は、オフセット最大 4095 バイト、長さ最大 273 バイトまでのコピーをサポートしています。

保守担当者向けメモ

  • 特定のデータ特性に合わせて圧縮戦略をカスタマイズすることが可能です。
  • 圧縮処理の実装詳細は src/common/pg_lzcompress.c を参照してください。
