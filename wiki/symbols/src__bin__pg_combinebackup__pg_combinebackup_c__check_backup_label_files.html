check_backup_label_files

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:510

目的

複数のバックアップディレクトリに含まれる backup_label ファイルを読み込み、バックアップチェーン（増分バックアップの連続性）が整合しているかを検証する。

シグネチャまたは形状

static StringInfo check_backup_label_files(int n_backups, char **backup_dirs)

入力と出力

  • **入力**:
  • n_backups: 処理対象のバックアップディレクトリ数。
  • backup_dirs: 各バックアップディレクトリのパスを格納した文字列配列。
  • **出力**:
  • 最後に読み込んだ（最新の）バックアップの backup_label 内容を保持する StringInfo 構造体。

前提条件と不変条件

  • backup_dirs に指定された各ディレクトリには backup_label ファイルが存在すること。
  • バックアップチェーンは、最初のバックアップがフルバックアップであり、以降が増分バックアップである必要がある。
  • 各バックアップの開始タイムライン（start_tli）と開始LSN（start_lsn）は、直前のバックアップの終了情報と一致しなければならない。

アルゴリズムまたは振る舞い

1. 指定されたバックアップディレクトリを、最後（最新）から最初（最古）に向かって逆順に走査する。 2. 各ディレクトリの backup_label を読み込み、parse_backup_label で内容を解析する。 3. 以下の整合性チェックを実行する:

  • 最初のバックアップ以外でフルバックアップが検出された場合、または最初のバックアップが増分バックアップである場合にエラーとする。
  • タイムラインおよびLSNが、直前のバックアップの期待値と一致しない場合にエラーとする。

    • 4. 走査中に得られた previous_tli と previous_lsn を次のループの期待値として更新する。 5. 最後に読み込んだ backup_label の内容を保持し、それ以外の一時的なバッファは解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/pg_combinebackup.c 内の main 関数（281行目）。
  • **呼び出し先**:
  • makeStringInfo, resetStringInfo, destroyStringInfo (lib/stringinfo.h)
  • pg_log_debug, pg_fatal (common/logging.h)
  • slurp_file (common/file_utils.h)
  • parse_backup_label (backup_label.c)

関連データ構造

  • StringInfo: backup_label の内容を保持する可変長文字列バッファ。
  • TimeLineID, XLogRecPtr: PostgreSQLのタイムラインおよびログシーケンス番号。

失敗モードと境界条件

  • ファイルのオープン失敗や読み込み失敗時に pg_fatal を呼び出しプログラムを終了する。
  • バックアップチェーンの順序や内容に不整合がある場合、pg_fatal によりエラー終了する。
  • n_backups が0以下の場合の挙動については、本ソースコード上では明示的なガード句は確認できない。

保守担当者向けメモ

  • 本関数は最新のバックアップから順に遡るため、チェーンの整合性検証には「直前のバックアップの終了情報」を保持する変数が不可欠である。
