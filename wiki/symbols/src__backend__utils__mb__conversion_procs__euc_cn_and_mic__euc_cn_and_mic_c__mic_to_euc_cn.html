mic_to_euc_cn

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_cn_and_mic/euc_cn_and_mic.c:59

目的

PostgreSQLの内部文字コード（MULE_INTERNAL）からEUC_CN（中国語）への変換を行うための関数マネージャ（fmgr）インターフェースです。

シグネチャまたは形状

Datum mic_to_euc_cn(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • src (CSTRING): 変換元のMULE_INTERNAL文字列。
  • dest (CSTRING): 変換結果を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): エラー発生時の挙動フラグ（trueの場合は例外を投げず中断）。
  • **出力**: PG_RETURN_INT32 により、変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_MULE_INTERNAL, PG_EUC_CN) により、エンコーディングの整合性が検証されます。
  • 呼び出し元はPostgreSQLの関数マネージャ規約に従う必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの妥当性を確認します。 3. 内部関数 mic2euc_cn を呼び出し、MULE_INTERNAL形式からEUC_CN形式へのバイト変換を実行します。 4. 変換されたバイト数を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）経由で呼び出されます。ソースコード内では PG_FUNCTION_INFO_V1(mic_to_euc_cn) による登録が確認できます。
  • **呼び出し先**: mic2euc_cn を直接呼び出します。

関連データ構造

  • PG_FUNCTION_ARGS: 関数マネージャの引数コンテナ。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • noError が false の場合、変換不可能な文字や不正なシーケンスに対してエラーが報告されます。
  • noError が true の場合、エラー発生時に処理を中断し、それまでに変換できたバイト数を返します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の変換ロジックは mic2euc_cn に依存しています。
  • MULE_INTERNALの識別子（LC_GB2312_80等）の解釈については mic2euc_cn の実装を参照してください。
