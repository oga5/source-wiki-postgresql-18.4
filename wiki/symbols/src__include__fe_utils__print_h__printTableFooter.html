printTableFooter

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/print.h:153

目的

クエリ結果のテーブル出力において、末尾に表示されるフッター情報を保持するための連結リスト構造を提供します。

シグネチャまたは形状

typedef struct printTableFooter
{
	char	   *data;
	struct printTableFooter *next;
} printTableFooter;

入力と出力

  • data: フッターとして表示する文字列（char *）。
  • next: リスト内の次のフッター要素へのポインタ。リストの終端は NULL となります。

前提条件と不変条件

  • next ポインタが NULL でない場合、それは有効な printTableFooter 構造体を指している必要があります。
  • data フィールドは、出力処理時に適切に参照される文字列を保持します。

アルゴリズムまたは振る舞い

  • 本構造体は連結リストとして機能し、複数のフッター行を順次保持します。
  • src/fe_utils/print.c 内の関数（footers_with_default 等）によって、テーブル出力時にフッターのリストが走査され、各要素の data が順次出力されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/print.c 内の複数の関数（printTable 等）が、printTableContent 構造体からフッターリストを取得し、走査を行っています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • printTableContent: テーブル全体の情報を保持する構造体であり、この中に printTableFooter のリストが含まれます。

失敗モードと境界条件

  • メモリ不足により data への文字列割り当てや構造体の確保が失敗する可能性があります。
  • リストの走査中に next ポインタが不正なアドレスを指している場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • printTableCleanup 関数において、このリストの各要素および data が適切に解放されることを確認してください。
  • src/fe_utils/print.c:53 にある default_footer_cell のように、静的に初期化されたフッター要素がリストの起点やデフォルト値として使用されるケースがあります。
