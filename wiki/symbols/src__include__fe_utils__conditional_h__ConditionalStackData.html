ConditionalStackData

定義

  • **種別**: struct
  • **定義箇所**: src/include/fe_utils/conditional.h:66

目的

クライアントアプリケーション（psql 等）における、ネストされた \if ブロックの制御状態を管理するためのスタック構造体です。

シグネチャまたは形状

typedef struct ConditionalStackData
{
	IfStackElem *head;
} ConditionalStackData;

入力と出力

  • **入力**: IfStackElem 型の連結リストの先頭ポインタを保持します。
  • **出力**: スタックのトップを参照することで、現在の条件分岐ブロックが実行対象か否かの判定に使用されます。

前提条件と不変条件

  • head はスタックのトップ（最新の \if ブロック）を指します。
  • スタックが空の場合、head は NULL となります。

アルゴリズムまたは振る舞い

  • \if コマンド実行時に新しい要素をスタックにプッシュし、\endif でポップすることで、ネストされた条件分岐の階層を追跡します。
  • 非アクティブなブロックに入った際、この構造体を通じて現在の解析状態（クエリ長や括弧の深さ）を保持し、不要なテキストの破棄を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/conditional.c 内の関数（pg_malloc を用いたインスタンス生成など）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、具体的な操作は ConditionalStack 型を介して提供される関数群が行います。

関連データ構造

  • ConditionalStack: ConditionalStackData へのポインタ型。
  • IfStackElem: スタックの各要素を構成する構造体。
  • ifState: 条件分岐の状態（IFSTATE_TRUE, IFSTATE_FALSE 等）を定義する列挙型。

失敗モードと境界条件

  • メモリ割り当て失敗時、pg_malloc が呼び出し元でエラーを発生させる可能性があります。
  • スタックが空の状態でポップ操作を行った場合の挙動については、本ソース excerpt には記載されていません。

保守担当者向けメモ

  • 本構造体は ConditionalStack 型として抽象化して利用されることが一般的です。
  • 非アクティブなブロックの制御は、このスタックのトップの状態に依存するため、スタック操作の整合性が解析器の正確性に直結します。
