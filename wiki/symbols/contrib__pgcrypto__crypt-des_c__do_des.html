do_des

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/crypt-des.c:482

目的

DES（Data Encryption Standard）アルゴリズムのコアとなるラウンド処理を実行し、データの暗号化または復号を行う。

シグネチャまたは形状

int do_des(uint32 l_in, uint32 r_in, uint32 *l_out, uint32 *r_out, int count)

入力と出力

  • **入力**:
  • l_in, r_in: 擬似ビッグエンディアン形式の入力データ（各32ビット）。
  • count: 処理回数。正の値は暗号化、負の値は復号（絶対値がラウンド数）を示す。
  • **出力**:
  • l_out, r_out: 処理結果を格納するポインタ。
  • 戻り値: 成功時に 0 を返す。count が 0 の場合は 1 を返す。

前提条件と不変条件

  • l_in, r_in, l_out, r_out は擬似ビッグエンディアン形式として扱われる。
  • 実行前に des_init によるマスクテーブルの初期化および des_setkey によるサブキーの生成が完了している必要がある。
  • count が負の場合、内部で符号を反転させて処理を行う。

アルゴリズムまたは振る舞い

1. **初期化**: count の符号に基づき、暗号化用または復号用のサブキーセットを選択する。 2. **初期置換 (IP)**: 入力データに対し ip_maskl および ip_maskr を用いて初期置換を行う。 3. **ラウンド処理**: count 回繰り返す。各ループ内で以下を実行する：

  • CHECK_FOR_INTERRUPTS() を呼び出し、割り込みをチェックする。
  • 16ラウンドの Feistel ネットワークを実行する。
  • 各ラウンドで、Rを48ビットに拡張し、ソルトおよびサブキーと XOR をとる。
  • S-box および P-box を用いて変換を行い、LとRを更新する。

    • 4. **最終置換 (FP)**: 処理後のデータに対し fp_maskl および fp_maskr を用いて最終置換を行い、結果を l_out, r_out に書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/crypt-des.c 内の関数（px_crypt_des 等から間接的に呼び出される）。
  • **呼び出し先**:
  • CHECK_FOR_INTERRUPTS()（PostgreSQL の割り込みチェックマクロ）。

関連データ構造

  • en_keysl, en_keysr: 暗号化用サブキー。
  • de_keysl, de_keysr: 復号用サブキー。
  • ip_maskl, ip_maskr: 初期置換用マスクテーブル。
  • fp_maskl, fp_maskr: 最終置換用マスクテーブル。
  • psbox, m_sbox: S-box および P-box 変換テーブル。

失敗モードと境界条件

  • count が 0 の場合、処理を行わず 1 を返す。
  • 長時間の計算中にクエリがキャンセルされた場合、CHECK_FOR_INTERRUPTS() により処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は DES アルゴリズムの低レイヤー実装であり、CPU アライメントに依存しない設計となっている。
  • CHECK_FOR_INTERRUPTS() の配置により、長時間実行されるハッシュ計算の応答性を確保している。
  • DES は現代の基準では脆弱であるため、互換性維持以外の目的での使用は避けること。
