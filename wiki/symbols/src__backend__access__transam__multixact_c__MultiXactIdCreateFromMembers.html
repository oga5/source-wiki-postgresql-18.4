MultiXactIdCreateFromMembers

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/access/transam/multixact.c:859

目的

複数のトランザクションIDとロック状態（MultiXactMember）を束ね、単一の MultiXactId を生成または取得する。

シグネチャまたは形状

MultiXactId MultiXactIdCreateFromMembers(int nmembers, MultiXactMember *members)

入力と出力

  • **入力**:
  • nmembers: メンバー数。
  • members: MultiXactMember 構造体の配列。
  • **出力**:
  • 成功時: 生成または再利用された MultiXactId。
  • 失敗時: エラー（elog(ERROR)）が発生し、処理が中断される。

前提条件と不変条件

  • メンバーリスト内に ISUPDATE_from_mxstatus が真となるメンバーは最大で1つでなければならない。
  • START_CRIT_SECTION() により、本関数実行中はクリティカルセクションとして保護される。

アルゴリズムまたは振る舞い

1. **キャッシュ検索**: mXactCacheGetBySet を呼び出し、同一メンバーセットを持つ MultiXactId が既に存在すればそれを返す。 2. **バリデーション**: メンバーリスト内に更新（Update）属性を持つメンバーが複数存在しないか確認する。 3. **ID 予約**: GetNewMultiXactId を呼び出し、新しい MultiXactId とオフセットを予約する。 4. **WAL 記録**: XLogBeginInsert から XLogInsert を通じ、XLOG_MULTIXACT_CREATE_ID レコードを生成する。 5. **永続化**: RecordNewMultiXact を呼び出し、SLRU（offsets および members）へ情報を書き込む。 6. **キャッシュ更新**: mXactCachePut を呼び出し、生成した ID をローカルキャッシュに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/transam/multixact.c（内部の複数箇所）
  • **呼び出し先**:
  • mXactCacheGetBySet, GetNewMultiXactId, XLogBeginInsert, XLogRegisterData, XLogInsert, RecordNewMultiXact, mXactCachePut, INJECTION_POINT_LOAD, INJECTION_POINT_CACHED

関連データ構造

  • MultiXactId: 32ビットの識別子。
  • MultiXactMember: トランザクションIDとロック状態を保持する構造体。
  • xl_multixact_create: WAL 記録用の構造体。

失敗モードと境界条件

  • **エラー**: メンバーリスト内に複数の更新メンバーが含まれる場合、elog(ERROR) により中断される。
  • **境界条件**: MultiXactId の周回防止のため、GetNewMultiXactId 内で適切なチェックが行われる。

保守担当者向けメモ

  • 本関数は MultiXactIdCreate や MultiXactIdExpand と異なり、MultiXactIdSetOldestMember を呼び出さない。これは VACUUM 等の処理でバックエンドがメンバーに含まれない場合があるためである。
  • WAL レコードの構造にはパディングが含まれており、将来的な最適化の余地がある。
