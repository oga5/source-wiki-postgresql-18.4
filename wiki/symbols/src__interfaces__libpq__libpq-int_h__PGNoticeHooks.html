PGNoticeHooks

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-int.h:147

目的

PostgreSQL サーバーから送信される通知メッセージ（Notice）を処理するためのコールバック関数と、その引数を保持する構造体です。

シグネチャまたは形状

typedef struct {
    PQnoticeReceiver noticeRec;   /* 通知メッセージ受信用の関数ポインタ */
    void            *noticeRecArg; /* noticeRec に渡すユーザー定義引数 */
    PQnoticeProcessor noticeProc;  /* 通知メッセージ処理用の関数ポインタ */
    void            *noticeProcArg; /* noticeProc に渡すユーザー定義引数 */
} PGNoticeHooks;

入力と出力

  • **入力**: PQnoticeReceiver および PQnoticeProcessor 型の関数ポインタ、およびそれらに付随する void * 型のユーザーデータ。
  • **出力**: 本構造体自体はデータ保持用であり、直接的な戻り値はありません。

前提条件と不変条件

  • noticeRec および noticeProc には、有効な関数ポインタまたは NULL が格納されます。
  • noticeRecArg および noticeProcArg は、対応するコールバック関数が呼び出される際に、その第一引数（または指定された引数）として渡されます。

アルゴリズムまたは振る舞い

  • 本構造体は、libpq 内部で通知メッセージが発生した際に、登録されたコールバックを呼び出すためのフックとして機能します。
  • pqInternalNotice 関数が本構造体を参照し、通知メッセージの処理を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-exec.c 内の pqInternalNotice が、通知処理の実行時に本構造体を参照します。
  • **保持箇所**: PGconn 構造体内部で noticeHooks フィールドとして保持されています（libpq-int.h:183, 452）。

関連データ構造

  • PGconn: 接続状態を管理するメイン構造体であり、本構造体をメンバとして保持します。

失敗モードと境界条件

  • コールバック関数ポインタが NULL の場合、通知メッセージの処理はスキップされます。
  • ユーザー定義の引数（*Arg）が不正なメモリ領域を指している場合、コールバック実行時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は libpq 内部のプライベートな定義です。アプリケーション開発者は libpq-fe.h で提供される公開 API を通じて通知ハンドラを設定してください。
  • pqInternalNotice のシグネチャ extern void pqInternalNotice(const PGNoticeHooks *hooks, const char *fmt,...) に適合するように管理してください。
