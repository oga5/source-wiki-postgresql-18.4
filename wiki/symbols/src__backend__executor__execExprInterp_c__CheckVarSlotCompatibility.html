CheckVarSlotCompatibility

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExprInterp.c:2385

目的

実行時のタプルスロット（TupleTableSlot）が、クエリ計画作成時の想定と一致しているかを確認する。主に、計画作成後にテーブル定義が変更（列の削除や型変更）された場合に、実行エンジンが不正なデータにアクセスするのを防ぐための防衛的チェックを行う。

シグネチャまたは形状

void CheckVarSlotCompatibility(TupleTableSlot *slot, int attnum, Oid vartype)

入力と出力

  • **入力**:
  • slot: 検査対象の TupleTableSlot。
  • attnum: 検査する属性番号（1始まり）。
  • vartype: クエリが期待する属性の型（Oid）。
  • **出力**: なし（戻り値なし）。
  • **副作用**: 互換性がない場合、ERROR を発生させ実行を中断する。

前提条件と不変条件

  • attnum > 0 の場合のみチェックが実行される（システム属性は型変更がないため対象外）。
  • 実行時に slot の tts_tupleDescriptor が有効である必要がある。

アルゴリズムまたは振る舞い

1. attnum がスロットの列数を超えていないか確認する（超えていれば ERROR）。 2. TupleDescAttr を使用して、対象属性の定義情報を取得する。 3. 属性が仮想生成列（ATT_GENERATED_VIRTUAL）である場合、予期せぬ参照として ERROR を発生させる。 4. 属性が削除済み（attisdropped）である場合、ERRCODE_UNDEFINED_COLUMN で ERROR を発生させる。 5. 属性の型（atttypid）と期待される vartype を比較し、不一致であれば ERRCODE_DATATYPE_MISMATCH で ERROR を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExpr.c:405（計画開始時）、および src/backend/executor/execExprInterp.c 内の各スロット操作箇所（innerslot, outerslot, scanslot, oldslot, newslot に対するチェック）。
  • **呼び出し先**: format_type_be（エラーメッセージ生成用）、TupleDescAttr（マクロ）。

関連データ構造

  • TupleTableSlot: 実行時のタプル保持構造体。
  • TupleDesc: タプルの構造定義。
  • Form_pg_attribute: 属性のメタデータ。

失敗モードと境界条件

  • **列数超過**: attnum > slot_tupdesc->natts の場合、内部エラーとして ERROR。
  • **仮想列参照**: 仮想生成列に直接アクセスしようとした場合、ERROR。
  • **列削除**: 属性が削除されている場合、ERRCODE_UNDEFINED_COLUMN で終了。
  • **型不一致**: 期待する型と実際の型が異なる場合、ERRCODE_DATATYPE_MISMATCH で終了。
  • **typmod**: 現在の設計では typmod のチェックは行われない（実装上の制約による）。

保守担当者向けメモ

  • 本関数は実行時のホットパスで呼び出される可能性があるため、チェックは最小限に留められている。
  • attnum > 0 の条件により、システム属性（ctid 等）のチェックはスキップされる。
  • エラーメッセージには format_type_be を使用しており、型名がユーザーフレンドリーに表示されるよう配慮されている。
