CFuncHashTabEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:45

目的

動的にロードされた外部C言語関数のキャッシュエントリを保持するためのデータ構造です。関数検索のオーバーヘッドを削減するために使用されます。

シグネチャまたは形状

typedef struct {
    Oid                fn_oid;      /* ハッシュキー */
    TransactionId      fn_xmin;     /* キャッシュの有効性確認用 */
    ItemPointerData    fn_tid;      /* キャッシュの有効性確認用 */
    PGFunction         user_fn;     /* 関数ポインタ */
    const Pg_finfo_record *inforec; /* 情報レコードへのポインタ */
} CFuncHashTabEntry;

入力と出力

  • 本構造体はハッシュテーブルの要素として管理されます。
  • fn_oid がハッシュキーとして機能し、構造体の先頭に配置される必要があります。

前提条件と不変条件

  • fn_oid はハッシュテーブルの検索キーとして使用されるため、構造体の先頭メンバである必要があります。
  • fn_xmin および fn_tid は、キャッシュされた関数定義が最新であるかを検証するために使用されます。

アルゴリズムまたは振る舞い

  • fmgr サブシステムにおいて、外部C関数のロード時にこの構造体が生成され、ハッシュテーブルに格納されます。
  • 以降の関数呼び出しでは、ハッシュテーブルを検索し、一致する CFuncHashTabEntry があれば、ロード済みの user_fn を直接利用します。

呼び出し元と呼び出し先

  • 呼び出し元: lookup_C_func および fmgr.c 内の関数検索ロジック。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しは user_fn を介して行われます。

関連データ構造

  • Pg_finfo_record: C言語関数が提供するメタデータ構造体。
  • CFuncHash: 本構造体を格納するハッシュテーブル。

失敗モードと境界条件

  • キャッシュの有効性確認（fn_xmin や fn_tid の比較）に失敗した場合、キャッシュは無効とみなされ、再ロードが必要となります。

保守担当者向けメモ

  • fn_oid が構造体の先頭にあるという制約は、ハッシュテーブルの検索ロジックに依存しているため、メンバの順序を変更しないでください。
  • ハッシュテーブルの初期化時に hash_ctl.entrysize として sizeof(CFuncHashTabEntry) が指定されています。
