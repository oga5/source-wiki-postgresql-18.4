pq_endmessage

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:295

目的

StringInfo バッファに構築されたメッセージをネットワークへ送信し、バッファのリソースを解放する。

シグネチャまたは形状

void pq_endmessage(StringInfo buf)

入力と出力

  • **入力**: buf (StringInfo) - メッセージデータが格納されたバッファ。cursor フィールドにメッセージタイプが保持されている必要がある。
  • **出力**: なし（void）。

前提条件と不変条件

  • buf は pq_beginmessage によって初期化され、有効なデータが書き込まれていること。
  • buf->cursor には送信すべきメッセージタイプが格納されていること。
  • 呼び出し後、buf->data は解放され、NULL に設定される。

アルゴリズムまたは振る舞い

1. buf->cursor（メッセージタイプ）、buf->data（ペイロード）、buf->len（データ長）を引数として pq_putmessage を呼び出し、ネットワークへデータを送出する。 2. pq_putmessage の実行結果（エラー等）については、下位層の pqcomm.c ですでに処理済みであるため、本関数内では明示的なエラーチェックを行わない。 3. pfree(buf->data) を実行し、バッファのメモリを解放する。 4. buf->data を NULL にリセットし、二重解放や不正なアクセスを防止する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printsimple.c
  • src/backend/access/transam/parallel.c
  • src/backend/backup/basebackup_copy.c
  • src/backend/commands/async.c
  • src/backend/commands/copyfromparse.c
  • src/backend/commands/copyto.c
  • src/backend/libpq/auth.c
  • **呼び出し先**:
  • pq_putmessage（ネットワーク送信）
  • pfree（メモリ解放）

関連データ構造

  • StringInfo: メッセージ構築に使用される動的バッファ構造体。

失敗モードと境界条件

  • pq_putmessage が失敗した場合、下位層でエラーが報告されるため、本関数はそのまま処理を継続してメモリ解放を行う。
  • buf が NULL の場合や、buf->data がすでに解放されている状態で呼び出した場合の挙動は未定義（セグメンテーション違反の可能性がある）。

保守担当者向けメモ

  • 本関数はバッファを解放するため、送信後にバッファを再利用する場合は pq_endmessage_reuse を検討すること。
  • メッセージ構築のライフサイクル管理において、pq_beginmessage と対になる終了処理である。
