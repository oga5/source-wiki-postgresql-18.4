disconnect_atexit

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_recvlogical.c:179

目的

プログラム終了時に、確立されているデータベース接続を安全に切断するために使用されるクリーンアップ関数です。

シグネチャまたは形状

static void disconnect_atexit(void)

入力と出力

  • 入力: なし
  • 出力: なし

前提条件と不変条件

  • グローバル変数 conn が有効な PGconn オブジェクトを指しているか、あるいは NULL である必要があります。

アルゴリズムまたは振る舞い

1. グローバル変数 conn が NULL でないことを確認します。 2. PQfinish(conn) を呼び出し、サーバーとの接続を終了し、関連するリソースを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: atexit() を介してプログラム終了時に呼び出されます。src/bin/pg_basebackup/pg_recvlogical.c:954 で登録されています。
  • 呼び出し先: PQfinish (libpq)

関連データ構造

  • PGconn: libpq の接続ハンドル。

失敗モードと境界条件

  • conn が NULL の場合、関数は何もしません。
  • 接続が既に切断されている場合、PQfinish の挙動は libpq の仕様に従います。

保守担当者向けメモ

  • 本関数は atexit ハンドラとして設計されています。他のツール（pg_basebackup, pg_receivewal, pg_rewind 等）でも同名の関数が定義されていますが、それぞれが自身のスコープ内の conn 変数を参照している点に注意してください。
