syslogger_fdget

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/syslogger.c:800

目的

FILE ポインタから、プラットフォームに応じたファイル記述子（または OS ハンドル）を取得する。

シグネチャまたは形状

int syslogger_fdget(FILE *file)

入力と出力

  • **入力**: FILE *file （対象のファイルストリーム）
  • **出力**:
  • 非 Windows 環境: fileno(file) の結果（ファイル記述子）。file が NULL の場合は -1。
  • Windows 環境: _get_osfhandle(_fileno(file)) の結果（OS ハンドル）。file が NULL の場合は 0。

前提条件と不変条件

  • file が有効な FILE ストリームを指しているか、または NULL であること。
  • Windows 環境では、標準的な C ライブラリのファイル記述子から OS ハンドルへの変換が成功することを前提としている。

アルゴリズムまたは振る舞い

1. file が NULL かどうかを判定する。 2. NULL の場合、プラットフォームに応じてエラー値（-1 または 0）を返す。 3. NULL でない場合：

  • 非 Windows 環境では fileno() を使用してファイル記述子を返す。
  • Windows 環境では _fileno() でファイル記述子を取得し、さらに _get_osfhandle() を介して OS ハンドルに変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/syslogger.c 内の以下の箇所から呼び出されている。
  • startup_data.syslogFile = syslogger_fdget(syslogFile); (行698)
  • startup_data.csvlogFile = syslogger_fdget(csvlogFile); (行699)
  • startup_data.jsonlogFile = syslogger_fdget(jsonlogFile); (行700)
  • **呼び出し先**:
  • 非 Windows: fileno()
  • Windows: _fileno(), _get_osfhandle()

関連データ構造

  • FILE (標準ライブラリのファイルストリーム)

失敗モードと境界条件

  • file 引数に NULL を渡した場合、プラットフォーム固有のデフォルト値（-1 または 0）が返される。
  • Windows 環境において、_fileno() や _get_osfhandle() が無効なハンドルを返す場合の挙動については、本関数の実装内では明示的なエラーハンドリングを行っていない。

保守担当者向けメモ

  • 本関数はプラットフォーム間のファイルハンドル管理の差異を吸収するために使用されている。Windows 環境での移植性を維持するため、WIN32 マクロによる条件分岐が必須となっている。
