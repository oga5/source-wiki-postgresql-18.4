select_common_collation

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_collate.c:207

目的

式リスト（List *exprs）を走査し、それら全体で共通して適用可能な照合順序（Collation）を特定します。

シグネチャまたは形状

Oid select_common_collation(ParseState *pstate, List *exprs, bool none_ok)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体。
  • exprs: 照合順序を評価する式のリスト。
  • none_ok: 照合順序が特定できない（COLLATE_NONE）場合に InvalidOid を返すことを許可するかどうかのフラグ。
  • **出力**:
  • 共通の照合順序を示す Oid。競合が発生した場合はエラーをスローします。

前提条件と不変条件

  • assign_collations_context を用いて再帰的な木走査を行い、照合順序の状態をマージします。
  • 照合順序が COLLATE_NONE の場合、戻り値は InvalidOid となります。

アルゴリズムまたは振る舞い

1. assign_collations_context を初期化し、assign_collations_walker を呼び出して式リストを走査します。 2. 走査の結果、照合順序の競合（COLLATE_CONFLICT）が検出された場合：

  • none_ok が true であれば InvalidOid を返します。
  • none_ok が false であれば、ERRCODE_COLLATION_MISMATCH エラーをスローします。

    • 3. 競合がない場合、特定された照合順序の Oid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c
  • src/backend/parser/parse_collate.c（内部再帰）
  • src/backend/parser/parse_cte.c
  • **呼び出し先**:
  • assign_collations_walker（走査用）
  • ereport（エラー報告用）
  • get_collation_name（エラーメッセージ生成用）
  • parser_errposition（エラー位置特定用）

関連データ構造

  • assign_collations_context: 走査中の照合順序状態（collation, strength, location 等）を保持する構造体。
  • ParseState: 解析中のクエリ状態を管理する構造体。

失敗モードと境界条件

  • **照合順序の競合**: 異なる照合順序を持つ式が混在し、かつ none_ok が false の場合、ereport によりエラーが発生します。
  • **照合順序なし**: どの式も照合順序を持たない場合、InvalidOid が返されます。

保守担当者向けメモ

  • 本関数は式リストの照合順序を解決するユーティリティであり、assign_collations_walker の結果に依存しています。
  • 競合時のエラーメッセージには get_collation_name を使用して照合順序名を表示するため、カタログ検索が正常に行える環境である必要があります。
