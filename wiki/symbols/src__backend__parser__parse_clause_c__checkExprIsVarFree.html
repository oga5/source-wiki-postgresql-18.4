checkExprIsVarFree

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:1924

目的

特定のSQL構文（LIMITなど）の引数として渡された式が、現在のクエリレベルの変数（Varノード）を含んでいないことを検証します。

シグネチャまたは形状

static void checkExprIsVarFree(ParseState *pstate, Node *n, const char *constructName)

入力と出力

  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。エラー位置の特定に使用。
  • n: 検証対象の式を表す Node。
  • constructName: エラーメッセージに使用されるSQL構文名（例: "LIMIT"）。
  • 戻り値: なし。検証に失敗した場合は ereport(ERROR, ...) により処理を中断します。

前提条件と不変条件

  • n は contain_vars_of_level 関数で走査可能な有効なノードである必要があります。
  • 変数が含まれている場合、ERRCODE_INVALID_COLUMN_REFERENCE エラーが発生します。

アルゴリズムまたは振る舞い

1. contain_vars_of_level(n, 0) を呼び出し、現在のクエリレベル（レベル0）の変数が式 n に含まれているかを確認します。 2. 変数が検出された場合、ereport を発行してエラーを報告します。 3. エラーメッセージには constructName が埋め込まれ、locate_var_of_level(n, 0) によって特定された変数の位置がエラー箇所として示されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_clause.c 内の複数箇所（LIMIT 句の解析など）。
  • 呼び出し先: contain_vars_of_level および locate_var_of_level（いずれも nodes/nodeFuncs.h 等で定義）。

関連データ構造

  • ParseState: 解析中のクエリ状態を管理。
  • Node: SQL式の抽象構文木ノード。

失敗モードと境界条件

  • 式が変数を含む場合、常にエラーとなります。
  • locate_var_of_level が変数の位置を特定できない場合、エラー位置の報告が不完全になる可能性があります。

保守担当者向けメモ

  • 本関数は static であり、parse_clause.c 内の特定の句の制約チェックに限定して使用されます。
  • 他の句で同様の制約が必要になった場合、本関数の再利用を検討してください。
