gist_page_opaque_info

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/gistfuncs.c:70

目的

GiSTインデックスページのヘッダー情報（LSN、NSN、右リンク、フラグ）を抽出し、人間が読み取り可能な形式で提供する。

シグネチャまたは形状

Datum gist_page_opaque_info(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: bytea 型の生ページデータ（PG_GETARG_BYTEA_P(0)）。
  • **出力**: 以下のフィールドを持つ複合型（Datum）:
  • lsn (LSN)
  • nsn (LSN)
  • rightlink (int8)
  • flags (text[])

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 入力データは verify_gist_page によってGiSTページとして検証される必要がある。
  • ページが新規（PageIsNew）の場合、結果は NULL となる。

アルゴリズムまたは振る舞い

1. **権限チェック**: superuser() を確認し、権限不足時はエラーを投げる。 2. **検証**: verify_gist_page を呼び出し、ページが有効なGiSTページか確認する。 3. **フラグ解析**: ページ内の GISTPageOpaqueData からフラグビットを取得し、leaf, deleted, tuples_deleted, follow_right, has_garbage の各状態を文字列配列に変換する。認識できないビットは16進数文字列として追加する。 4. **データ構築**: PageGetLSN、GistPageGetNSN、rightlink、およびフラグ配列を heap_form_tuple でタプル化して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 マクロによる登録）。
  • **呼び出し先**: verify_gist_page（検証用）、get_call_result_type（戻り値型定義）、heap_form_tuple（結果構築）。

関連データ構造

  • GISTPageOpaqueData: GiSTページ固有のヘッダー情報（access/gist.h で定義）。
  • Page: PostgreSQLの標準的なページ構造。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザーでない場合に ERRCODE_INSUFFICIENT_PRIVILEGE を発生させる。
  • **型エラー**: 戻り値の型が複合型でない場合にエラーを発生させる。
  • **ページ検証エラー**: verify_gist_page がGiSTページとして不適切と判断した場合、エラーを発生させる。

保守担当者向けメモ

  • 本関数は pageinspect モジュールの一部であり、デバッグ用途に特化している。
  • フラグの定義が変更された場合は、本関数のビットマスク処理も更新する必要がある。
