jsonpath_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonpath.c:114

目的

クライアントから送信されたバイナリ形式の jsonpath データをデシリアライズし、内部的な Datum 表現へ変換します。

シグネチャまたは形状

Datum jsonpath_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_POINTER(0) を介して StringInfo 形式のバイナリデータを受け取ります。
  • **出力**: jsonPathFromCstring の戻り値である Datum（jsonpath 型の内部表現）を返します。

前提条件と不変条件

  • 入力バッファの先頭1バイトにバージョン番号が含まれている必要があります。
  • バージョン番号が JSONPATH_VERSION と一致している必要があります。

アルゴリズムまたは振る舞い

1. pq_getmsgint を用いてバッファからバージョン番号を読み取ります。 2. バージョンが JSONPATH_VERSION と一致するか検証します。一致しない場合は elog(ERROR) で処理を中断します。 3. 一致する場合、pq_getmsgtext を用いてバッファ内の文字列データを抽出します。 4. 抽出した文字列と長さを jsonPathFromCstring に渡し、バイナリ形式の jsonpath オブジェクトを構築して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのデータ型受信インターフェース（外部からのバイナリ入力時）。
  • **呼び出し先**:
  • pq_getmsgint / pq_getmsgtext（libpqプロトコル処理）
  • jsonPathFromCstring（文字列から内部形式への変換）
  • 補足: 提供されたソース情報には、具体的な呼び出し元の関数名は記載されていません。

関連データ構造

  • StringInfo: PostgreSQLの可変長バッファ構造体。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • **バージョン不一致**: JSONPATH_VERSION 以外のバージョン番号が渡された場合、エラーが発生します。
  • **バッファ読み取り失敗**: pq_getmsgtext 等の内部関数がバッファの枯渇や不正な形式を検知した場合、エラーが発生します。

保守担当者向けメモ

  • 本関数はバイナリプロトコル経由の入力のみを扱います。SQL文字列からの変換は jsonpath_in が担当します。
  • バージョン管理は JSONPATH_VERSION マクロに依存しているため、バイナリ形式の変更時にはこの定数の整合性に注意してください。
