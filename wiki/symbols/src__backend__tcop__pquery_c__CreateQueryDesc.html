CreateQueryDesc

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:67

目的

クエリ実行に必要な情報を保持する QueryDesc 構造体を割り当て、初期化する。

シグネチャまたは形状

QueryDesc *CreateQueryDesc(PlannedStmt *plannedstmt,
                           const char *sourceText,
                           Snapshot snapshot,
                           Snapshot crosscheck_snapshot,
                           DestReceiver *dest,
                           ParamListInfo params,
                           QueryEnvironment *queryEnv,
                           int instrument_options)

入力と出力

  • **入力**:
  • plannedstmt: 実行対象の計画済みステートメント。
  • sourceText: クエリのソース文字列。
  • snapshot, crosscheck_snapshot: 実行に使用するスナップショット。
  • dest: 結果の出力先（DestReceiver）。
  • params: クエリに渡されるパラメータ。
  • queryEnv: クエリ環境。
  • instrument_options: 計測オプション。
  • **出力**: 初期化済みの QueryDesc 構造体へのポインタ。

前提条件と不変条件

  • snapshot および crosscheck_snapshot は RegisterSnapshot によって参照カウントが増加される。
  • tupDesc, estate, planstate, totaltime フィールドは NULL に初期化される。
  • already_executed フラグは false に設定される。

アルゴリズムまたは振る舞い

1. palloc を使用して QueryDesc 構造体分のメモリを確保する。 2. 引数として渡された計画、ソーステキスト、出力先、パラメータ等を構造体の各フィールドにコピーする。 3. RegisterSnapshot を呼び出し、スナップショットを登録する。 4. 実行状態を示すポインタ類を NULL に、実行済みフラグを false に設定する。 5. 初期化済みの QueryDesc を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyto.c
  • src/backend/commands/createas.c
  • src/backend/commands/explain.c
  • src/backend/commands/extension.c
  • src/backend/executor/execParallel.c
  • src/backend/executor/functions.c
  • src/backend/executor/spi.c
  • **呼び出し先**:
  • palloc (メモリ確保)
  • RegisterSnapshot (スナップショット登録)

関連データ構造

  • QueryDesc: クエリ実行の全状態を保持する主要構造体。
  • PlannedStmt: 実行計画。
  • DestReceiver: 結果の配送先インターフェース。
  • Snapshot: トランザクションスナップショット。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる可能性がある。
  • RegisterSnapshot が失敗した場合の挙動は、当該関数の仕様に依存する。

保守担当者向けメモ

  • 本関数で作成された QueryDesc は、後に ExecutorStart 等の実行エンジン関数によってフィールドが埋められる設計となっている。
  • already_executed フラグは、実行の多重防止や状態管理に使用される。
