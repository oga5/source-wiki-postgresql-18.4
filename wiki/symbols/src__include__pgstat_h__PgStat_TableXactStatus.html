PgStat_TableXactStatus

定義

  • 種別: struct
  • 定義箇所: src/include/pgstat.h:187

目的

現在のトランザクションまたはサブトランザクション内における、リレーション（テーブル）ごとの統計情報の変更を追跡・保持する。

シグネチャまたは形状

typedef struct PgStat_TableXactStatus
{
    PgStat_Counter tuples_inserted;
    PgStat_Counter tuples_updated;
    PgStat_Counter tuples_deleted;
    bool           truncdropped;
    PgStat_Counter inserted_pre_truncdrop;
    PgStat_Counter updated_pre_truncdrop;
    PgStat_Counter deleted_pre_truncdrop;
    int            nest_level;
    struct PgStat_TableXactStatus *upper;
    PgStat_TableStatus *parent;
    struct PgStat_TableXactStatus *next;
} PgStat_TableXactStatus;

入力と出力

  • **入力**: トランザクション実行中のタプル操作（挿入、更新、削除）およびリレーションの切り詰め（truncate）や削除（drop）操作。
  • **出力**: parent 構造体（PgStat_TableStatus）への統計情報の伝播、およびサブトランザクション階層間での統計の保持。

前提条件と不変条件

  • nest_level は現在のサブトランザクションの入れ子レベルを示す。
  • upper ポインタは、より上位のトランザクションレベルの構造体を指す。
  • truncdropped が真の場合、*_pre_truncdrop フィールドに切り詰め/削除前のカウンタ値が保持される。

アルゴリズムまたは振る舞い

1. **統計蓄積**: トランザクション内のタプル操作を各カウンタに加算する。 2. **切り詰め/削除対応**: truncdropped が発生すると、その時点のカウンタ値を *_pre_truncdrop に退避させる。 3. **階層管理**: upper および next ポインタを使用して、トランザクション階層および同一レベル内のリレーション統計をリンクリストとして管理する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_relation.c 内の統計更新処理（save_truncdrop_counters, restore_truncdrop_counters 等）。
  • **呼び出し先**: 構造体内部で PgStat_TableStatus を参照。

関連データ構造

  • PgStat_TableStatus: この構造体が属するリレーションごとのメイン統計ステータス。
  • PgStat_Counter: 統計値保持に使用される int64 型。

失敗モードと境界条件

  • メモリ割り当て失敗: pgstat_relation.c での動的確保時に発生する可能性がある。
  • 階層の不整合: nest_level や upper ポインタの管理が正しく行われない場合、統計の二重計上や欠落が発生する。

保守担当者向けメモ

  • 本構造体はトランザクションの入れ子構造に深く依存している。変更を加える際は、pgstat_relation.c におけるサブトランザクションの開始・終了・ロールバック時の処理との整合性を確認すること。
  • truncdropped 関連のフィールドは、truncate/drop 操作後の統計の正確性を保つために重要である。
