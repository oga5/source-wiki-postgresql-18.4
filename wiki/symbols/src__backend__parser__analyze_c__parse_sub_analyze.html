parse_sub_analyze

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:221

目的

サブクエリやCTE（共通テーブル式）など、メインクエリの一部として含まれる解析ツリーを個別にセマンティック解析し、Query ノードへ変換する。

シグネチャまたは形状

Query *parse_sub_analyze(Node *parseTree, ParseState *parentParseState,
                         CommonTableExpr *parentCTE,
                         bool locked_from_parent,
                         bool resolve_unknowns)

入力と出力

  • **入力**:
  • parseTree: 解析対象のノード（Node*）。
  • parentParseState: 親クエリの解析状態（ParseState*）。
  • parentCTE: 親クエリのCTE情報（CommonTableExpr*）。
  • locked_from_parent: 親からロックを引き継ぐかどうかのフラグ。
  • resolve_unknowns: 未知の型を解決するかどうかのフラグ。
  • **出力**:
  • 変換後の Query 構造体へのポインタ。

前提条件と不変条件

  • parentParseState を基に新しい ParseState が作成される。
  • 解析終了後、内部的に作成された ParseState は free_parsestate により解放される。

アルゴリズムまたは振る舞い

1. make_parsestate(parentParseState) を呼び出し、親のコンテキストを継承した新しい ParseState を初期化する。 2. 引数で渡された parentCTE、locked_from_parent、resolve_unknowns を pstate に設定する。 3. transformStmt(pstate, parseTree) を呼び出し、解析ツリーをクエリツリーへ変換する。 4. free_parsestate(pstate) で一時的な解析状態を破棄する。 5. 生成された Query を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c (サブクエリ解析時)
  • src/backend/parser/parse_clause.c
  • src/backend/parser/parse_cte.c
  • src/backend/parser/parse_expr.c
  • **呼び出し先**:
  • make_parsestate
  • transformStmt
  • free_parsestate

関連データ構造

  • Query: 解析結果として返されるクエリツリー構造。
  • ParseState: 解析中の状態を保持する構造体。
  • CommonTableExpr: CTEの定義情報。

失敗モードと境界条件

  • transformStmt が解析エラーを検出した場合、PostgreSQL のエラーハンドリング機構により例外が発生し、本関数は正常にリターンしない可能性がある。

保守担当者向けメモ

  • 本関数は再帰的な解析を行うため、スタックオーバーフローを避ける必要がある。
  • ParseState のライフサイクル管理（作成と解放）が関数内で完結していることを確認すること。
