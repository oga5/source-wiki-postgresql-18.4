ProcSignalBarrierType

定義

  • 種別: enum
  • 定義箇所: src/include/storage/procsignal.h:56

目的

PostgreSQLの全バックエンドプロセスに対して、特定のグローバルな状態変更を通知し、同期を行うためのバリア同期タイプを定義します。

シグネチャまたは形状

typedef enum
{
	PROCSIGNAL_BARRIER_SMGRRELEASE, /* ask smgr to close files */
} ProcSignalBarrierType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: EmitProcSignalBarrier 関数の引数として使用され、バリア同期の種別を指定します。

前提条件と不変条件

  • 現在定義されている値は PROCSIGNAL_BARRIER_SMGRRELEASE のみです。
  • この列挙型は、共有メモリ上のフラグを介したプロセス間通信の制御に使用されます。

アルゴリズムまたは振る舞い

  • EmitProcSignalBarrier(ProcSignalBarrierType type) を呼び出すことで、指定されたタイプのバリアを全プロセスに対して発行します。
  • 受信側プロセスは、シグナルハンドラ経由でこのタイプを識別し、対応する処理（例：SMGRRELEASE の場合はストレージマネージャによるファイルクローズ）を実行します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/backend/storage/ipc/procsignal.c: EmitProcSignalBarrier 関数内で引数として利用。
  • src/backend/storage/ipc/procsignal.c: シグナルハンドラ内でフラグから変換される型として利用。
  • 呼び出し先:
  • 情報なし。

関連データ構造

  • ProcSignalReason: シグナル発生理由を定義する別の列挙型。
  • EmitProcSignalBarrier(): 本型を引数に取る同期関数。

失敗モードと境界条件

  • SIGUSR1 シグナルが多重化されているため、短時間に同一のバリアタイプが複数回発行された場合、受信側で一度しか観測されない可能性があります。

保守担当者向けメモ

  • 新しいバリアタイプを追加する場合は、この列挙型に値を追加し、受信側のシグナルハンドラで適切な処理を実装する必要があります。
