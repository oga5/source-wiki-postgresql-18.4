visibilitymap_clear

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/visibilitymap.c:139

目的

ヒープページの状態変更に伴い、可視性マップ（Visibility Map）上の対応するビットをクリアし、必要に応じてバッファをダーティ状態にマークする。

シグネチャまたは形状

bool visibilitymap_clear(Relation rel, BlockNumber heapBlk, Buffer vmbuf, uint8 flags)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • heapBlk: クリア対象のヒープブロック番号。
  • vmbuf: 可視性マップのバッファ（visibilitymap_pin 等で取得済みであること）。
  • flags: クリアするビットフラグ（VISIBILITYMAP_ALL_VISIBLE, VISIBILITYMAP_ALL_FROZEN 等）。
  • **出力**:
  • bool: 実際にビットがクリアされた場合に true、既にクリアされていた場合は false を返す。

前提条件と不変条件

  • vmbuf は有効であり、かつ heapBlk に対応するマップページを保持している必要がある。
  • flags は VISIBILITYMAP_VALID_BITS の範囲内であること。
  • all_visible ビットをクリアする際、all_frozen ビットがセットされたままになるような操作は禁止されている（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. heapBlk からマップ上のバイト位置とオフセットを算出する。 2. vmbuf に対して排他ロック（BUFFER_LOCK_EXCLUSIVE）を取得する。 3. マップページの内容を確認し、対象ビットがセットされているか判定する。 4. ビットがセットされている場合、ビットをクリアし、MarkBufferDirty(vmbuf) を呼び出してバッファをダーティにする。 5. 排他ロックを解除し、ビットの変更有無を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c（ヒープ更新処理）
  • src/backend/access/heap/heapam_xlog.c（WAL リプレイ処理）
  • contrib/pg_surgery/heap_surgery.c
  • **呼び出し先**:
  • LockBuffer, PageGetContents, BufferGetPage, BufferGetBlockNumber, MarkBufferDirty, RelationGetRelationName（いずれも内部ユーティリティ）。

関連データ構造

  • Buffer: バッファプール内のページ管理構造。
  • Relation: リレーション記述子。
  • VISIBILITYMAP_VALID_BITS: 可視性マップで定義される有効なビットフラグ群。

失敗モードと境界条件

  • vmbuf が無効、または heapBlk と対応しないバッファが渡された場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数はヒープ側の変更と同一のクリティカルセクション内で呼び出されるため、個別の WAL 記録は不要である。
  • ビットのクリアは「保守的」な操作であり、ビットが立っていない状態は「可視性が不明（または偽）」であることを示す。
