RangeVarAdjustRelationPersistence

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/namespace.c:845

目的

リレーションの永続性（relpersistence）と、作成先スキーマ（nspid）の整合性を検証および調整します。

シグネチャまたは形状

void RangeVarAdjustRelationPersistence(RangeVar *newRelation, Oid nspid)

入力と出力

  • newRelation: 調整対象のリレーション定義（RangeVar 構造体）。relpersistence フィールドが入力として参照され、必要に応じて更新されます。
  • nspid: リレーションを作成する対象スキーマの OID。
  • 出力: なし（副作用として newRelation->relpersistence の変更、またはエラー発生）。

前提条件と不変条件

  • nspid が一時名前空間であるかどうかの判定は isTempOrTempToastNamespace および isAnyTempNamespace に依存します。
  • 一時リレーションは一時スキーマ以外には作成できません。
  • 他セッションの一時スキーマに対してリレーションを作成することは禁止されています。

アルゴリズムまたは振る舞い

1. newRelation->relpersistence の値に基づき分岐します。

  • RELPERSISTENCE_TEMP: nspid が一時スキーマでない場合、エラーを発生させます。
  • RELPERSISTENCE_PERMANENT: nspid が一時スキーマであれば、relpersistence を RELPERSISTENCE_TEMP に変更します。それ以外の一時スキーマであればエラーを発生させます。
  • その他: nspid が一時スキーマである場合、エラーを発生させます。

呼び出し元と呼び出し先

  • 呼び出し先:

関連データ構造

  • RangeVar: リレーションの識別情報と永続性フラグを保持する構造体。

失敗モードと境界条件

  • ERRCODE_INVALID_TABLE_DEFINITION:
  • 他セッションの一時スキーマにリレーションを作成しようとした場合。
  • 一時リレーションを非一時スキーマに作成しようとした場合。
  • 一時スキーマに一時リレーション以外を作成しようとした場合。

保守担当者向けメモ

  • 本関数はリレーション作成時のスキーマと永続性の不整合を防止するガードレールとして機能します。ロジックを変更する際は、一時テーブルの可視性ルールに影響がないか確認してください。
