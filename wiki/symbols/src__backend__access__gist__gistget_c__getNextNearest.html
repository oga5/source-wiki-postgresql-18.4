getNextNearest

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistget.c:559

目的

GiSTインデックスの順序付きスキャン（ORDER BY）において、優先度付きキューから最小距離のアイテムを取り出し、ヒープタプルに到達するまでインデックスページを再帰的に走査して、次に返すべきタプルを特定する。

シグネチャまたは形状

static bool getNextNearest(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan（現在のスキャン状態を保持する記述子）
  • **出力**: bool（次に返すべきタプルが見つかった場合は true、それ以上タプルが存在しない場合は false）

前提条件と不変条件

  • scan->opaque が GISTScanOpaque 型としてキャスト可能であること。
  • スキャン中に xs_hitup が保持されている場合、本関数内でメモリ解放（pfree）が行われる。
  • pairingheap を用いた検索アイテムの管理が初期化されていること。

アルゴリズムまたは振る舞い

1. 前回の呼び出しで返された scan->xs_hitup が存在すれば解放する。 2. do-while ループ内で getNextGISTSearchItem を呼び出し、キューから最小距離のアイテムを取得する。 3. アイテムがヒープタプルの場合:

  • scan->xs_heaptid に対象のTIDを格納する。
  • index_store_float8_orderby_distances を呼び出し、距離情報をスキャン記述子に記録する。
  • インデックスのみのスキャン（Index-Only Scan）が要求されている場合、scan->xs_hitup にタプルをセットする。
  • res = true とし、ループを終了する。

    • 4. アイテムがインデックスページの場合:

  • CHECK_FOR_INTERRUPTS() を実行し、割り込みを確認する。
  • gistScanPage を呼び出し、当該ページ内のアイテムをキューに追加する。

    • 5. アイテムが取得できなくなるまで繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: gistgettuple（ソースコード内の直接的な呼び出しを確認）
  • **呼び出し先**:
  • getNextGISTSearchItem: 次の検索アイテムの取得
  • gistScanPage: インデックスページの走査とキューへの追加
  • index_store_float8_orderby_distances: 距離情報の保存
  • pfree: メモリ解放
  • CHECK_FOR_INTERRUPTS: 割り込みチェック

関連データ構造

  • IndexScanDesc: スキャン全体の制御構造体。
  • GISTScanOpaque: GiST特有のスキャン状態（キューやメモリコンテキストを含む）。
  • GISTSearchItem: 検索対象（ヒープタプルまたはインデックスページ）を保持する構造体。

失敗モードと境界条件

  • キューが空の場合、getNextGISTSearchItem が NULL を返し、ループを抜けて false を返す。
  • メモリ不足時は pfree や内部処理でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は ORDER BY 句を伴うスキャンにおいて、GiSTの木構造を距離順に探索する中核ロジックである。
  • gistScanPage への引き渡し時に item->distances を渡しており、距離計算の整合性が重要となる。
  • xs_hitup のメモリ管理は gistgettuple との連携に依存しているため、変更時はライフサイクルに注意すること。
