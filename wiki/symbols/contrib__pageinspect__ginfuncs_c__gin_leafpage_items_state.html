gin_leafpage_items_state

定義

  • 種別: struct
  • 定義箇所: contrib/pageinspect/ginfuncs.c:171

目的

GIN インデックスのリーフページに含まれるポストリスト（Posting List）を走査する際、関数呼び出し間で状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct gin_leafpage_items_state
{
    TupleDesc       tupd;
    GinPostingList *seg;
    GinPostingList *lastseg;
} gin_leafpage_items_state;

入力と出力

本構造体はデータ保持用であり、直接的な入力・出力関数ではありません。gin_leafpage_items 関数内でメモリ確保および状態管理に使用されます。

前提条件と不変条件

  • gin_leafpage_items 関数において、SRF（Set Returning Function）の初回呼び出し時に palloc を用いてメモリが割り当てられる必要があります。

アルゴリズムまたは振る舞い

  • tupd: 返却するタプルの記述子を保持します。
  • seg: 現在走査中のポストリストセグメントを指します。
  • lastseg: 走査対象のポストリストの終端を指します。
  • gin_leafpage_items 関数は、この構造体を使用して圧縮されたポストリストを順次デコードし、アイテムポインタを列挙します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pageinspect/ginfuncs.c 内の gin_leafpage_items 関数が、この構造体をインスタンス化して状態管理に使用します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • GinPostingList: GIN インデックスのポストリスト圧縮形式を定義する構造体。
  • TupleDesc: PostgreSQL のタプル記述子。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • 構造体内のポインタが不正なメモリ領域を指している場合、ポストリスト走査時にセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は gin_leafpage_items の内部的な状態管理専用です。他の関数から参照されることは想定されていません。
  • ポストリストの走査ロジックを変更する場合は、seg および lastseg の更新手順を gin_leafpage_items 内で適切に同期させる必要があります。
