CheckCopyStreamStop

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:1216

目的

ストリーミングレプリケーションの停止条件を評価し、条件を満たしている場合にWALファイルのクローズとサーバーへの終了通知（COPYプロトコルの終了）を実行します。

シグネチャまたは形状

static bool CheckCopyStreamStop(PGconn *conn, StreamCtl *stream, XLogRecPtr blockpos)

入力と出力

  • **入力**:
  • conn: サーバーとの接続を管理する PGconn オブジェクト。
  • stream: ストリーミング設定および状態を保持する StreamCtl 構造体。
  • blockpos: 現在処理中のWAL位置（XLogRecPtr）。
  • **出力**:
  • bool: 正常終了または停止条件未達の場合は true、WALファイルのクローズや終了パケットの送信に失敗した場合は false を返します。

前提条件と不変条件

  • stream->stream_stop コールバック関数が定義されている必要があります。

アルゴリズムまたは振る舞い

1. still_sending が真であり、かつ stream->stream_stop(blockpos, stream->timeline, false) が真を返す場合、停止処理を開始します。 2. close_walfile を呼び出し、現在開いているWALファイルをクローズします。失敗した場合は false を返します。 3. PQputCopyEnd を使用してサーバーに COPY モードの終了を通知し、PQflush で送信を確定させます。失敗した場合はエラーログを出力し false を返します。 4. 正常に終了処理が完了した場合、still_sending を false に更新します。 5. 停止条件を満たさない場合、または既に停止処理が完了している場合は true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/receivelog.c 内のストリーミング受信ループ（762行目、841行目付近）。
  • **呼び出し先**:
  • stream->stream_stop (コールバック)
  • close_walfile
  • PQputCopyEnd (libpq)
  • PQflush (libpq)
  • pg_log_error

関連データ構造

  • StreamCtl: ストリーミングの動作を制御する構造体。
  • XLogRecPtr: WAL上の位置を示すデータ型。
  • PGconn: libpqの接続ハンドル。

失敗モードと境界条件

  • close_walfile が失敗した場合、関数は直ちに false を返します。
  • PQputCopyEnd または PQflush が失敗した場合、エラーメッセージがログ出力され、関数は false を返します。

保守担当者向けメモ

  • 本関数は still_sending という静的変数に依存しています。マルチスレッド環境や、複数のストリームを同時に扱うような拡張を行う場合は、この状態管理の局所性に注意してください。
  • stream_stop コールバックの戻り値がストリーミング終了のトリガーとなるため、このコールバックの実装が意図した停止条件（LSN到達やタイムライン切り替えなど）を正しく反映しているか確認してください。
