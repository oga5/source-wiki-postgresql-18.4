injection_points_stats_numcalls

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/injection_stats.c:187

目的

指定されたインジェクションポイント名の実行回数（numcalls）を統計情報から取得し、SQLインターフェースを通じて返却します。

シグネチャまたは形状

Datum injection_points_stats_numcalls(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介してインジェクションポイント名（text型）を受け取ります。
  • **出力**: 該当するインジェクションポイントが存在する場合はその実行回数（int64）、存在しない場合は NULL を返します。

前提条件と不変条件

  • PostgreSQLの統計サブシステムが初期化されている必要があります。
  • 入力された名前は text_to_cstring によってC文字列に変換されます。

アルゴリズムまたは振る舞い

1. 入力された text 型の引数をC文字列に変換します。 2. pgstat_fetch_stat_injentry(name) を呼び出し、共有メモリ上の統計エントリを取得します。 3. エントリが見つからない場合（NULL）、PG_RETURN_NULL() で終了します。 4. エントリが見つかった場合、その構造体内の numcalls フィールドを PG_RETURN_INT64() で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されることを想定しています（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • **呼び出し先**:
  • text_to_cstring (PostgreSQL内部関数)
  • pgstat_fetch_stat_injentry (統計サブシステム)

関連データ構造

  • PgStat_StatInjEntry: 統計情報を保持する構造体。本関数はこの構造体の numcalls フィールドを参照します。

失敗モードと境界条件

  • 指定した名前のインジェクションポイントが統計情報に存在しない場合、エラーにはならず NULL を返します。
  • 引数が不正な場合、PG_GETARG_TEXT_PP 等の内部処理によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はSQLから呼び出される統計取得用関数です。統計情報の構造に変更があった場合は、PgStat_StatInjEntry の定義と併せて確認してください。
  • pgstat_fetch_stat_injentry は共有メモリ上の統計データを読み取るため、高頻度な呼び出しは統計サブシステムの負荷に影響を与える可能性があります。
