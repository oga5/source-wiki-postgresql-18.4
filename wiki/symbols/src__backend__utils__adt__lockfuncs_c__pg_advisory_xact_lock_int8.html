pg_advisory_xact_lock_int8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:642

目的

ユーザー指定の64ビット整数値をキーとして、現在のトランザクションスコープで排他的なアドバイザリーロックを取得します。

シグネチャまたは形状

Datum pg_advisory_xact_lock_int8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して int64 型のキーを受け取ります。
  • **出力**: void（成功時はロックが保持され、失敗時または待機中はブロックされます）。

前提条件と不変条件

  • ロックのスコープは現在のトランザクションに限定されます。
  • ロックキーは MyDatabaseId と組み合わされて管理されるため、データベースを跨いだロックは行えません。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT64(0) を使用して、引数から64ビットのロックキーを取得します。 2. SET_LOCKTAG_INT64 マクロを用いて、取得したキーを LOCKTAG 構造体に変換します。 3. LockAcquire 関数を呼び出し、ExclusiveLock（排他ロック）モードでロックを要求します。

  • 第3引数（notHeld）は false、第4引数（wait）は false が指定されており、ロックが取得できるまで待機します。

    • 4. PG_RETURN_VOID() を返し、関数を終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されます（具体的な呼び出し元コードの詳細は提供されていません）。
  • **呼び出し先**:
  • LockAcquire（src/backend/storage/lmgr/lock.c 相当のロックマネージャ機能）

関連データ構造

  • LOCKTAG: ロック対象を識別するための構造体。
  • ExclusiveLock: ロックモードの定数。

失敗モードと境界条件

  • ロックの取得に失敗した場合や、デッドロックが検出された場合は、PostgreSQLの標準的なエラー処理メカニズムにより例外が発生します。
  • データベースIDが異なる環境間でのロック共有は不可能です。

保守担当者向けメモ

  • 本関数はトランザクション終了時に自動的に解放されるロックを管理します。
  • LockAcquire の第4引数が false であるため、ロックが即座に取得できない場合は待機状態となります。
