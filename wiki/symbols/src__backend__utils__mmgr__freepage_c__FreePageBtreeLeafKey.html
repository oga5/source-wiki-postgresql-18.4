FreePageBtreeLeafKey

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/freepage.c:93

目的

メモリ管理における空き領域（スパン）の範囲を、B-tree のリーフノード内で表現するためのデータ構造です。

シグネチャまたは形状

typedef struct FreePageBtreeLeafKey
{
	Size		first_page;		/* スパンの開始ページ番号 */
	Size		npages;			/* スパン内のページ数 */
} FreePageBtreeLeafKey;

入力と出力

本構造体はデータ保持を目的としており、関数としての入出力は持ちません。メモリ操作（コピーやサイズ計算）の対象として利用されます。

前提条件と不変条件

  • first_page は管理対象領域内での相対的なページ番号を示します。
  • npages は 1 以上の値であり、連続する空き領域のサイズを表します。

アルゴリズムまたは振る舞い

  • B-tree のリーフノードにおいて、空き領域の管理単位として配列形式（leaf_key[FPM_ITEMS_PER_LEAF_PAGE]）で保持されます。
  • メモリの確保（Get）や解放（Put）に伴い、B-tree のノード内でキーの挿入、削除、および配列のシフト操作が行われます。
  • sizeof(FreePageBtreeLeafKey) を用いて、ノード内のキー配列に対するメモリコピーやサイズ計算が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/freepage.c 内の B-tree 操作関数群（ノードの分割、キーの挿入・削除処理など）から参照されています。
  • **呼び出し先**: 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • FreePageBtree: B-tree 全体を管理する構造体。
  • FreePageBtreeInternalKey: B-tree の内部ノードで使用されるキー構造。
  • FreePageSpanLeader: 空き領域の先頭に配置されるヘッダ。

失敗モードと境界条件

  • ノード内のキー配列が満杯の場合、B-tree の分割処理が必要です。
  • npages が 0 になるような不正な状態は、マージ処理の不整合を示唆します。

保守担当者向けメモ

  • 本構造体は freepage.c 内の B-tree 実装に密結合しています。サイズ変更を行う場合は、FPM_ITEMS_PER_LEAF_PAGE の定義およびノードのメモリレイアウトへの影響を考慮してください。
  • sizeof を用いたメモリ操作が多用されているため、構造体のメンバ追加時には memcpy や memmove の引数に注意が必要です。
