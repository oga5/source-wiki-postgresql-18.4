SystemAttributeByName

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:247

目的

指定された名前を持つシステム属性（ctid, xmin 等）のメタデータ（FormData_pg_attribute）を検索し、取得する。

シグネチャまたは形状

const FormData_pg_attribute *SystemAttributeByName(const char *attname)

入力と出力

  • **入力**: attname (検索対象の属性名を示す文字列)
  • **出力**: 見つかった場合は対応する FormData_pg_attribute 構造体へのポインタ。見つからない場合は NULL を返す。

前提条件と不変条件

  • システム属性の定義を保持する静的配列 SysAtt が存在すること。
  • attname は有効な C 文字列であること。

アルゴリズムまたは振る舞い

1. 静的配列 SysAtt を先頭から順に走査する。 2. 各要素の属性名（att->attname）と引数 attname を strcmp で比較する。 3. 一致する属性が見つかった時点でそのポインタを返し、配列の末尾まで見つからなければ NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/executor/spi.c
  • src/backend/parser/parse_relation.c
  • src/backend/parser/parse_utilcmd.c
  • src/backend/utils/adt/expandedrecord.c
  • **呼び出し先**:
  • strcmp (標準ライブラリ)
  • NameStr (マクロ)

関連データ構造

  • FormData_pg_attribute: システムカタログ pg_attribute の行構造体。
  • SysAtt: heap.c 内で定義されているシステム属性の静的配列（ハードコーディングされたメタデータ）。

失敗モードと境界条件

  • 指定された名前がシステム属性として存在しない場合、NULL を返す。
  • 大文字・小文字は区別される（strcmp による比較のため）。

保守担当者向けメモ

  • 本関数は heap.c 内でハードコーディングされた SysAtt 配列に依存している。システム属性の追加や変更を行う際は、この配列の定義を更新する必要がある。
  • 頻繁に呼び出される可能性があるため、検索効率は SysAtt の要素数に依存する。
