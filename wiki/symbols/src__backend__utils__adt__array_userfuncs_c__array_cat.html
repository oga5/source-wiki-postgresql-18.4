array_cat

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:316

目的

2つの配列を連結し、新しい配列を生成する。

シグネチャまたは形状

Datum array_cat(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの ArrayType を受け取る。
  • **出力**: 連結された新しい ArrayType を Datum として返す。

前提条件と不変条件

  • 両方の配列の要素型（element_type）が一致している必要がある。
  • 連結可能な次元の組み合わせは以下のいずれかである必要がある。

    • 1. 一方が空配列（次元数0）である。 2. 両方の次元数が等しい（ndims1 == ndims2）。 3. 次元数が1つ異なる（ndims1 == ndims2 - 1 または ndims1 == ndims2 + 1）。

アルゴリズムまたは振る舞い

1. **NULLチェック**: 片方がNULLの場合、もう一方をそのまま返す。両方NULLならNULLを返す。 2. **型チェック**: ARR_ELEMTYPE を比較し、不一致の場合はエラーを投げる。 3. **空配列の処理**: 一方が空配列の場合、もう一方を結果として返す。 4. **次元チェック**: 次元数の組み合わせが前提条件を満たさない場合、エラーを投げる。 5. **次元の構築**:

  • ndims1 == ndims2 の場合: 最初の次元を合計し、残りの次元が一致することを確認する。
  • ndims1 == ndims2 - 1 の場合: 第2引数を外側の配列とし、第1引数をその先頭に挿入する構造を構築する。
  • ndims1 == ndims2 + 1 の場合: 第1引数を外側の配列とし、第2引数を末尾に追加する構造を構築する（※ソース excerpt はここで終了）。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される。src/bin/pg_upgrade/check.c からの参照が確認されている。
  • **呼び出し先**: palloc によるメモリ確保、ARR_* マクロによる配列メタデータへのアクセス、ereport によるエラー報告を行う。

関連データ構造

  • ArrayType: PostgreSQLの配列型内部表現。
  • PG_FUNCTION_ARGS: 関数呼び出し規約のための引数コンテナ。

失敗モードと境界条件

  • **型不一致**: ERRCODE_DATATYPE_MISMATCH を発生させる。
  • **次元不整合**: ERRCODE_ARRAY_SUBSCRIPT_ERROR を発生させる。
  • **メモリ不足**: palloc 失敗時にシステム標準の例外が発生する。

保守担当者向けメモ

  • 本関数は多次元配列の連結をサポートしているが、次元の組み合わせルールが厳格であるため、変更時は次元計算ロジックの整合性に注意すること。
  • fn_extra を利用したキャッシュ戦略の適用可能性については、本関数単体では明示されていない。
