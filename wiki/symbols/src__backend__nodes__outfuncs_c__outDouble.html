outDouble

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/outfuncs.c:216

目的

double 型の値を、デバッグやシリアライズに適した最短の 10 進数文字列形式に変換し、StringInfo バッファへ書き込みます。

シグネチャまたは形状

static void outDouble(StringInfo str, double d)

入力と出力

  • **入力**:
  • str: 文字列構築用の StringInfo バッファへのポインタ。
  • d: シリアライズ対象の double 型の値。
  • **出力**: なし（str が指すバッファの内容が更新されます）。

前提条件と不変条件

  • str は初期化済みの有効な StringInfo オブジェクトである必要があります。
  • DOUBLE_SHORTEST_DECIMAL_LEN で定義されたバッファサイズが、変換後の文字列を格納するのに十分である必要があります。

アルゴリズムまたは振る舞い

1. double_to_shortest_decimal_buf を呼び出し、double 値 d を最短の 10 進数表現に変換してローカルバッファ buf に格納します。 2. appendStringInfoString を使用して、変換された文字列を str バッファの末尾に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/outfuncs.c 内のノード出力処理（WRITE_FLOAT_FIELD マクロ等を通じて呼び出されることが想定されます）。
  • **呼び出し先**:
  • double_to_shortest_decimal_buf
  • appendStringInfoString

関連データ構造

  • StringInfo: PostgreSQL の動的文字列構築用データ構造。

失敗モードと境界条件

  • double 値が NaN や無限大の場合の挙動は、double_to_shortest_decimal_buf の実装に依存します。
  • StringInfo バッファのメモリ不足が発生した場合、appendStringInfoString 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は static であり、outfuncs.c 内でのみ使用されます。
  • 数値の精度を維持しつつ最短の表現を得るために common/shortest_dec.h の関数を利用しています。変更時はこの依存関係に注意してください。
