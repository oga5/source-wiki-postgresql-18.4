plperl_proc_ptr

定義

  • 種別: struct
  • 定義箇所: src/pl/plperl/plperl.c:163

目的

PL/Perl の関数キャッシュ用ハッシュテーブルにおけるエントリ構造体です。関数検索キーと、対応する関数記述子へのポインタを保持するために使用されます。

シグネチャまたは形状

typedef struct plperl_proc_ptr
{
    plperl_proc_key proc_key;    /* Hash key (must be first!) */
    plperl_proc_desc *proc_ptr;
} plperl_proc_ptr;

入力と出力

  • **入力**: ハッシュテーブルのキーとなる plperl_proc_key 構造体。
  • **出力**: コンパイル済み関数情報へのポインタ plperl_proc_ptr *。

前提条件と不変条件

  • proc_key フィールドは構造体の先頭に配置される必要があります（ハッシュテーブルのキーとして機能するため）。
  • ハッシュテーブルの entrysize は sizeof(plperl_proc_ptr) に設定されます。

アルゴリズムまたは振る舞い

  • 本構造体は、plperl_proc_hash ハッシュテーブルの要素として機能します。
  • 関数実行時や検証時に、この構造体を介して plperl_proc_desc（コンパイルされた関数のメタデータ）にアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/pl/plperl/plperl.c: ハッシュテーブルの初期化時（hash_ctl.entrysize の設定）および関数検証関数 validate_plperl_function 内で参照されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数呼び出しを行うものではありません。

関連データ構造

  • plperl_proc_key: ハッシュテーブルの検索キー。
  • plperl_proc_desc: コンパイルされた関数の詳細情報（Perl サブルーチン参照など）を保持する構造体。

失敗モードと境界条件

  • ハッシュテーブルの管理において、複数のエントリが存在する場合でも有効なエントリは一つであるという制約があります（ソースコード内のコメントによる）。

保守担当者向けメモ

  • 本構造体は plperl_proc_hash のエントリサイズ定義に直接依存しているため、メンバの追加や変更を行う際はハッシュテーブルの初期化ロジックとの整合性に注意してください。
