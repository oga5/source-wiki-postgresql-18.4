btree_xlog_split

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/nbtree/nbtxlog.c:250

目的

B-tree インデックスのページ分割（Split）操作を WAL リプレイ時に再現し、物理的なページ状態を復元する。

シグネチャまたは形状

static void btree_xlog_split(bool newitemonleft, XLogReaderState *record)

入力と出力

  • **入力**:
  • newitemonleft: 新しいアイテムが左側のページに挿入されたかどうかを示すフラグ。
  • record: リプレイ対象の WAL レコード。
  • **出力**: なし（副作用としてバッファの状態を更新）。

前提条件と不変条件

  • record は xl_btree_split 構造体を含む有効な WAL レコードであること。
  • リプレイ時は並行更新が発生しないため、ページごとのロックで整合性が保たれる。
  • 物理的なタプル順序を維持することで、WAL 一貫性チェックが可能となる。

アルゴリズムまたは振る舞い

1. **メタデータ取得**: WAL レコードから分割対象のページ番号（origpagenumber）、右側の兄弟ページ番号（rightpagenumber）、および分割後の兄弟ページ番号（spagenumber）を取得する。 2. **不完全な分割フラグの解除**: 非リーフページの場合、_bt_clear_incomplete_split を呼び出し、下位レベルの不完全な分割フラグを解除する。 3. **右側（新規）ページの構築**:

  • XLogInitBufferForRedo で新しいページを初期化。
  • _bt_pageinit でページヘッダーを初期化し、兄弟関係（btpo_prev, btpo_next）を設定。
  • _bt_restore_page で WAL データからタプルを復元し、LSN を更新してダーティとしてマークする。

    • 4. **左側（元）ページの再構築**:

  • XLogReadBufferForRedo で元のページを読み込む。
  • 一時的なページ（leftpage）を作成し、ハイキーおよび元のタプルを順序通りに再配置する。
  • newitemonleft が真の場合、新しいアイテムを適切な位置に挿入する。
  • PageRestoreTempPage で一時ページの内容を元のページに反映し、BTP_INCOMPLETE_SPLIT フラグを設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_redo（nbtxlog.c 内のディスパッチャから呼び出される）。
  • **呼び出し先**:
  • _bt_clear_incomplete_split
  • XLogInitBufferForRedo / XLogReadBufferForRedo
  • _bt_pageinit
  • _bt_restore_page
  • _bt_swap_posting
  • PageGetTempPageCopySpecial / PageRestoreTempPage
  • PageAddItem

関連データ構造

  • xl_btree_split: WAL レコードの構造体。
  • BTPageOpaque: B-tree ページヘッダーのメタデータ。
  • Buffer: バッファ管理用構造体。

失敗モードと境界条件

  • PageAddItem が失敗した場合、elog(ERROR) によりリカバリ処理が中断される。
  • datalen が想定と一致しない場合、アサーションエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は物理的なページ再構築に特化している。
  • _bt_restore_page と同様のロジックで物理順序を維持しているため、変更時は WAL 一貫性チェックへの影響を考慮すること。
  • newitemonleft や postingoff の処理は、インデックス挿入時の複雑なケースを再現するために重要である。
