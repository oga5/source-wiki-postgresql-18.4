SetDatabaseHasLoginEventTriggers

定義

  • 種別: function
  • 定義箇所: src/backend/commands/event_trigger.c:389

目的

現在のデータベースにおいて、ログインイベントトリガーが存在することをシステムカタログ pg_database に記録（dathasloginevt フラグを true に設定）します。

シグネチャまたは形状

void SetDatabaseHasLoginEventTriggers(void)

入力と出力

  • 入力: なし（内部的に MyDatabaseId を使用）
  • 出力: なし（副作用として pg_database カタログを更新）

前提条件と不変条件

  • pg_database テーブルに対する RowExclusiveLock が取得可能であること。
  • MyDatabaseId が有効なデータベースを指していること。

アルゴリズムまたは振る舞い

1. pg_database を RowExclusiveLock でオープンします。 2. 他のバックエンドとの競合を防ぐため、LockSharedObject を使用して AccessExclusiveLock モードでカスタムロックを取得します。 3. DATABASEOID システムキャッシュから現在のデータベースのタプルを検索・取得します。 4. dathasloginevt フラグが既に true でない場合のみ、以下の処理を行います。

  • フラグを true に更新します。
  • CatalogTupleUpdate を呼び出してカタログを更新します。
  • CommandCounterIncrement を実行して変更を可視化します。

    • 5. ロックの解除、テーブルのクローズ、およびメモリの解放を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/event_trigger.c 内の複数箇所（詳細はソースコード上の静的解析による推測であり、明示的なコールグラフは提供されていません）。
  • 呼び出し先: table_open, LockSharedObject, SearchSysCacheLockedCopy1, CatalogTupleUpdate, CommandCounterIncrement, UnlockTuple, table_close, heap_freetuple 等のPostgreSQL内部API。

関連データ構造

  • pg_database (システムカタログ)
  • Form_pg_database (カタログ構造体)
  • HeapTuple

失敗モードと境界条件

  • SearchSysCacheLockedCopy1 が有効なタプルを返さない場合、elog(ERROR) により処理が中断されます。
  • データベースロックの取得に失敗した場合、トランザクションの競合が発生する可能性があります。

保守担当者向けメモ

  • 本関数は pg_database.dathasloginevt フラグを更新する唯一の手段として機能します。
  • EventTriggerOnLogin() が同フラグをリセットする処理と競合しないよう、LockSharedObject による排他制御が行われています。このロックはデータベース全体をブロックするものではなく、フラグ更新の整合性を保つためのものです。
