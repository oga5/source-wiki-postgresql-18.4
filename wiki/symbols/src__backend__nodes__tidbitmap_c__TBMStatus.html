TBMStatus

定義

  • 種別: enum
  • 定義箇所: src/backend/nodes/tidbitmap.c:120

目的

TIDBitmap 構造体の現在の内部状態を識別するために使用されます。ビットマップが保持するデータの形式（ハッシュテーブルの有無や単一ページ管理など）を定義します。

シグネチャまたは形状

typedef enum {
    TBM_EMPTY,      /* ハッシュテーブルなし、nentries == 0 */
    TBM_ONE_PAGE,   /* entry1 に単一のエントリを保持 */
    TBM_HASH,       /* ページテーブルが有効、entry1 は未使用 */
} TBMStatus;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: TIDBitmap 構造体内の status フィールドとして保持される。

前提条件と不変条件

  • TBM_EMPTY: nentries が 0 である必要がある。
  • TBM_ONE_PAGE: entry1 フィールドに有効な単一エントリが格納されている必要がある。
  • TBM_HASH: ページテーブルが初期化済みであり、entry1 は無視される。

アルゴリズムまたは振る舞い

ビットマップのデータ蓄積状況に応じて、以下の遷移が発生します。 1. 初期状態は TBM_EMPTY。 2. 最初のページが追加されると TBM_ONE_PAGE へ移行。 3. 複数のページが追加されるか、メモリ制限によりハッシュテーブルが必要になると TBM_HASH へ移行。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/nodes/tidbitmap.c 内の TIDBitmap 構造体（status フィールドとして参照）。
  • 呼び出し先: 情報なし。

関連データ構造

  • TIDBitmap: この列挙型を状態管理に使用するメイン構造体。

失敗モードと境界条件

  • 状態遷移の不整合（例: TBM_EMPTY なのに nentries > 0）が発生した場合、ビットマップのデータ整合性が失われる。

保守担当者向けメモ

  • 新しい状態を追加する場合は、TIDBitmap 構造体内の関連フィールド（entry1 やページテーブル）の排他制御およびメモリ管理ロジックを更新する必要があります。
