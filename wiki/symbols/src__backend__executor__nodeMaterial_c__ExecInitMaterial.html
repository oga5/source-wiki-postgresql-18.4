ExecInitMaterial

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMaterial.c:163

目的

Material ノードの実行状態（MaterialState）を初期化し、子ノードの実行準備を行う。

シグネチャまたは形状

MaterialState *ExecInitMaterial(Material *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Material プランノード。
  • estate: 実行状態を保持する EState 構造体。
  • eflags: 実行フラグ（EXEC_FLAG_REWIND, EXEC_FLAG_BACKWARD, EXEC_FLAG_MARK 等）。
  • **出力**:
  • 初期化済みの MaterialState 構造体へのポインタ。

前提条件と不変条件

  • Material ノードは ExecProcNode フィールドに ExecMaterial 関数を割り当てる必要がある。
  • Material ノード自体は射影（Projection）を行わないため、ps_ProjInfo は NULL に設定される。
  • 子ノードの初期化時には、REWIND, BACKWARD, MARK/RESTORE 関連のフラグが除去される。

アルゴリズムまたは振る舞い

1. MaterialState 構造体を割り当て、プランノードと実行状態をリンクする。 2. eflags に基づき、Tuplestore のサポート要否を判定する。BACKWARD フラグがある場合は REWIND フラグも付与する。 3. eof_underlying を false に、tuplestorestate を NULL に初期化する。 4. 子ノードに対して ExecInitNode を呼び出し、実行準備を行う。 5. ExecInitResultTupleSlotTL を用いて結果タプルスロットを初期化する。 6. ExecCreateScanSlotFromOuterPlan を用いて、子ノードからのタプルを受け取るためのスキャンスロットを作成する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内のノード初期化処理から呼び出される。
  • **呼び出し先**: ExecInitNode（子ノードの初期化）、ExecInitResultTupleSlotTL、ExecCreateScanSlotFromOuterPlan。

関連データ構造

  • MaterialState: 本ノードの状態を保持する構造体。
  • Material: プランノード。
  • EState: 実行状態。
  • Tuplestore: データを一時保存するための構造体（tuplestorestate 経由）。

失敗モードと境界条件

  • eflags の設定により、Tuplestore を作成しない最適化が行われる場合がある。
  • 子ノードの初期化に失敗した場合は、ExecInitNode がエラーを発生させる。

保守担当者向けメモ

  • 本関数は ExprContext を初期化しない（ExecQual や ExecProject を呼び出さないため）。
  • Tuplestore のフラグ解釈は一般的な実行フラグと微妙に異なるため、フラグ操作時は注意が必要。
  • Material ノードはマテリアライズされたリレーションからのみタプルを返す。
