bloom_add_element

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bloomfilter.c:134

目的

指定された要素をブルームフィルタに追加します。要素のハッシュ値を計算し、対応するビットセット上の位置をセットすることで、集合への包含を記録します。

シグネチャまたは形状

void bloom_add_element(bloom_filter *filter, unsigned char *elem, size_t len)

入力と出力

  • filter: 操作対象の bloom_filter 構造体へのポインタ。
  • elem: 追加する要素のバイト列。
  • len: elem の長さ（バイト単位）。
  • 戻り値: なし。

前提条件と不変条件

  • filter は bloom_create によって適切に初期化されている必要があります。
  • ビットセットのサイズは 2 のべき乗であるという前提で、ビット操作が行われます。

アルゴリズムまたは振る舞い

1. k_hashes を呼び出し、要素 elem から filter->k_hash_funcs 個のハッシュ値を生成して hashes 配列に格納します。 2. 各ハッシュ値について、以下のビット操作を実行します。

  • hashes[i] >> 3: バイト単位のインデックスを算出します。
  • 1 << (hashes[i] & 7): バイト内のビットオフセットを算出し、ビットマスクを作成します。
  • filter->bitset[...] |= ...: ビット単位の OR 操作を行い、該当ビットを 1 にセットします。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/amcheck/verify_nbtree.c
  • src/backend/utils/adt/acl.c
  • src/test/modules/test_bloomfilter/test_bloomfilter.c
  • 呼び出し先:
  • k_hashes (同ファイル内)

関連データ構造

  • bloom_filter: フィルタの状態（ビットセット、ハッシュ関数数など）を保持する構造体。

失敗モードと境界条件

  • ビットセットのサイズ制限（最大 512MB）を超過するような不正な filter 状態が渡された場合、メモリ破壊の可能性があります。
  • hashes 配列のサイズは MAX_HASH_FUNCS（10）に固定されているため、フィルタ設定がこれを超える場合は考慮されていません。

保守担当者向けメモ

  • 本関数はビットセットの更新のみを行い、包含判定は bloom_lacks_element が担当します。
  • 拡張ダブルハッシュ法により、ビットセットサイズが 2 のべき乗であっても衝突を抑える設計となっています。
