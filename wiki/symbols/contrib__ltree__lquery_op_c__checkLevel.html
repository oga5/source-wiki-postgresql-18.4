checkLevel

定義

  • 種別: function
  • 定義箇所: contrib/ltree/lquery_op.c:164

目的

lquery の特定のレベル（lquery_level）が、ltree の特定のレベル（ltree_level）と一致するかを判定します。

シグネチャまたは形状

static bool checkLevel(lquery_level *curq, ltree_level *curt)

入力と出力

  • **入力**:
  • curq: 評価対象の lquery レベル構造体。
  • curt: 比較対象の ltree レベル構造体。
  • **出力**:
  • bool: 一致した場合は true、一致しない場合は false を返します。LQL_NOT フラグが設定されている場合は、論理が反転します。

前提条件と不変条件

  • curq->numvar == 0 の場合、ワイルドカード * として扱われ、常に一致（true）と判定されます。
  • curvar->flag に基づき、プレフィックスマッチング（LVAR_ANYEND）や大文字小文字を区別しない比較（LVAR_INCASE）が適用されます。

アルゴリズムまたは振る舞い

1. LQL_NOT フラグを確認し、期待される成功の論理値を初期化します。 2. curq->numvar が 0 の場合、即座に成功を返します。 3. curq 内の各バリアント（lquery_variant）を順次走査します。 4. LVAR_SUBLEXEME フラグの有無に応じて、以下のいずれかを呼び出します。

  • 有効な場合: compare_subnode を使用して部分文字列の一致を確認。
  • 無効な場合: ltree_label_match を使用してラベル全体の一致を確認。

    • 5. いずれかのバリアントが一致した場合、初期化した成功値を返します。 6. すべてのバリアントを走査しても一致しない場合、成功値の否定を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/lquery_op.c 内の checkCond 関数（直接の呼び出しを確認）。
  • **呼び出し先**:
  • compare_subnode
  • ltree_label_match

関連データ構造

  • lquery_level: クエリの階層レベルを保持する構造体。
  • ltree_level: ツリーの階層レベルを保持する構造体。
  • lquery_variant: クエリ内の個別の比較条件を保持する構造体。

失敗モードと境界条件

  • curq->numvar が 0 の場合、ラベルの内容に関わらず常に成功します。
  • LQL_NOT フラグが設定されている場合、一致条件が反転するため、通常のラベルマッチング結果とは逆のブール値が返されます。

保守担当者向けメモ

  • 本関数は checkCond から再帰的に呼び出される可能性があるため、スタック消費量に注意してください。
  • LVAR_SUBLEXEME と ltree_label_match の分岐は、クエリの定義に基づいています。比較ロジックを変更する場合は、これら両方の関数が整合性を保っていることを確認してください。
