GetLatestLSN

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/walsummarizer.c:803

目的

WAL Summarizer プロセスが要約対象とすべき最新の WAL 位置（LSN）とタイムライン ID を取得する。

シグネチャまたは形状

XLogRecPtr GetLatestLSN(TimeLineID *tli)

入力と出力

  • **入力**: tli (TimeLineID へのポインタ。取得した最新のタイムライン ID が格納される)
  • **出力**: XLogRecPtr (最新の WAL 位置)

前提条件と不変条件

  • データベースの状態（リカバリ中か否か）に応じて、適切な LSN 取得元を選択する。
  • リカバリ中であっても、挿入用タイムラインが設定されている場合はその位置を優先する。

アルゴリズムまたは振る舞い

1. **通常稼働時**: RecoveryInProgress() が偽の場合、GetFlushRecPtr() を呼び出し、ディスクにフラッシュされた最新の LSN を返す。 2. **リカバリ中**:

  • GetWALInsertionTimeLineIfSet() が有効な値を返す場合、そのタイムラインと GetXLogReplayRecPtr() の値を返す。
  • それ以外の場合、WAL レシーバーのフラッシュ位置 (GetWalRcvFlushRecPtr) とリプレイ位置 (GetXLogReplayRecPtr) を比較し、より進んでいる方の LSN とタイムラインを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/walsummarizer.c 内の複数箇所（メインループおよび要約処理の制御フロー）。
  • **呼び出し先**:
  • RecoveryInProgress()
  • GetFlushRecPtr()
  • GetWALInsertionTimeLineIfSet()
  • GetXLogReplayRecPtr()
  • GetWalRcvFlushRecPtr()

関連データ構造

  • XLogRecPtr (WAL 位置を示す型)
  • TimeLineID (タイムライン識別子)

失敗モードと境界条件

  • WAL レシーバーが動作していない場合、GetWalRcvFlushRecPtr は利用できない可能性があるが、関数内でリプレイ位置との比較により安全に処理される。
  • 挿入用タイムラインが未設定（0）の場合は、リカバリ中のロジックに従いフラッシュ位置またはリプレイ位置が選択される。

保守担当者向けメモ

  • 本関数は WAL Summarizer が要約の進捗を判断するための基点となる。
  • リカバリ終了直後の過渡期における挙動（RecoveryInProgress が真だがリプレイが停止している状態）を考慮した設計となっている。
