ExplainOneQuery

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:292

目的

EXPLAIN コマンドの実行フローを制御し、指定されたクエリに対する実行計画の生成および出力処理を管理する。

シグネチャまたは形状

static void ExplainOneQuery(Query *query, int cursorOptions, IntoClause *into, ExplainState *es, ParseState *pstate, ParamListInfo params)

入力と出力

  • **入力**:
  • query: 解析済みのクエリツリー。
  • cursorOptions: カーソルオプション。
  • into: SELECT INTO 等の出力先指定。
  • es: EXPLAIN の状態管理構造体。
  • pstate: パーサの状態。
  • params: クエリパラメータ。
  • **出力**: なし（void）。副作用として es を通じて実行計画が整形・出力される。

前提条件と不変条件

  • query->commandType が CMD_UTILITY の場合、ExplainOneUtility に処理を委譲する。
  • ExplainOneQuery_hook が定義されている場合、標準の処理ではなくフック関数が優先的に呼び出される。

アルゴリズムまたは振る舞い

1. クエリタイプがユーティリティ文（CMD_UTILITY）であるかを確認し、該当する場合は ExplainOneUtility を呼び出して終了する。 2. ExplainOneQuery_hook が設定されているかを確認する。

  • 設定されている場合: フック関数を呼び出し、クエリの実行計画生成を委譲する。
  • 設定されていない場合: standard_ExplainOneQuery を呼び出し、標準的な実行計画生成処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c 内の複数箇所（ExplainQuery 等から呼び出される）。
  • **呼び出し先**:
  • ExplainOneUtility
  • ExplainOneQuery_hook (存在する場合)
  • standard_ExplainOneQuery

関連データ構造

  • Query: 解析済みクエリツリー。
  • ExplainState: EXPLAIN コマンドの出力形式やオプションを保持する状態構造体。
  • ParseState: パーサのコンテキスト情報。

失敗モードと境界条件

  • CMD_UTILITY 以外のクエリについては、standard_ExplainOneQuery またはフック関数の実装に依存する。
  • フック関数が設定されている場合、その実装が失敗した際の挙動はフック側に依存する。

保守担当者向けメモ

  • 本関数は EXPLAIN の実行フローにおける分岐点である。特定のクエリタイプに対する挙動をカスタマイズしたい場合は、フックの利用または standard_ExplainOneQuery の修正を検討すること。
  • ANALYZE オプションが指定されている場合、standard_ExplainOneQuery 内で実際のクエリ実行が行われるため、副作用に注意が必要である。
