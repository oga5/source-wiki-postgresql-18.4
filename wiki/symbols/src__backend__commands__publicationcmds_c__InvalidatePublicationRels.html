InvalidatePublicationRels

定義

  • 種別: function
  • 定義箇所: src/backend/commands/publicationcmds.c:1177

目的

パブリケーションに関連付けられたリレーション（テーブル）のキャッシュを無効化し、論理レプリケーションの定義変更をシステム全体に反映させる。

シグネチャまたは形状

void InvalidatePublicationRels(List *relids)

入力と出力

  • **入力**: relids - 無効化対象となるリレーションのOIDリスト（List *）。
  • **出力**: なし（void）。

前提条件と不変条件

  • relids は有効なOIDのリストであること。
  • 本関数は、パブリケーションの定義変更（追加・削除・変更）が発生したトランザクション内で呼び出される必要がある。

アルゴリズムまたは振る舞い

1. relids のリスト長を MAX_RELCACHE_INVAL_MSGS と比較する。 2. リスト長が閾値未満の場合、リスト内の各リレーションOIDに対して CacheInvalidateRelcacheByRelid を個別に呼び出し、キャッシュを無効化する。 3. リスト長が閾値以上の場合、個別のメッセージ送信を避け、CacheInvalidateRelcacheAll を呼び出してリレーションキャッシュ全体をリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c
  • src/backend/commands/publicationcmds.c
  • **呼び出し先**:
  • CacheInvalidateRelcacheByRelid
  • CacheInvalidateRelcacheAll

関連データ構造

  • List（PostgreSQLのリスト構造）
  • Oid（リレーション識別子）

失敗モードと境界条件

  • relids が空リストの場合、何も行わずに正常終了する。
  • リスト長が MAX_RELCACHE_INVAL_MSGS を超える場合、全キャッシュリセットという高コストな操作に切り替わる。

保守担当者向けメモ

  • 本関数は論理レプリケーションの整合性に直結する。キャッシュ無効化のロジックは、パフォーマンスと正確性のトレードオフ（個別無効化 vs 全体リセット）に基づいて設計されている。
  • MAX_RELCACHE_INVAL_MSGS の値は、システム全体のキャッシュ効率に影響を与えるため、変更時は慎重に評価すること。
