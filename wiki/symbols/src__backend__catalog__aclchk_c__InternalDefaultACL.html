InternalDefaultACL

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/aclchk.c:91

目的

ALTER DEFAULT PRIVILEGES 文の実行時や、デフォルト権限の操作において、対象となる権限設定情報を内部的に保持・伝播するために使用される構造体です。

シグネチャまたは形状

typedef struct
{
	Oid			roleid;			/* 所有ロール */
	Oid			nspid;			/* 名前空間（指定なしの場合は InvalidOid） */
	bool		is_grant;		/* GRANTの場合はtrue、REVOKEの場合はfalse */
	ObjectType	objtype;		/* 対象オブジェクト型 */
	bool		all_privs;		/* 全権限フラグ */
	AclMode		privileges;		/* 権限ビットマスク */
	List	   *grantees;		/* 被付与者リスト */
	bool		grant_option;	/* GRANT OPTIONの有無 */
	DropBehavior behavior;		/* 削除時の挙動 */
} InternalDefaultACL;

入力と出力

本構造体は関数間の引数として渡され、主に SetDefaultACL や SetDefaultACLsInSchemas への入力として使用されます。

前提条件と不変条件

  • nspid が InvalidOid の場合、特定のスキーマに限定されないデフォルト権限設定を意味します。
  • grantees は List 型であり、権限の対象となるロールのリストを保持します。

アルゴリズムまたは振る舞い

SQL 解析によって生成された権限変更要求を、システムカタログ（pg_default_acl）の更新処理に適した形式へ集約します。この構造体を通じて、特定のロールおよびスキーマに対する権限の付与・剥奪ロジックが統一的に処理されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c 内の ALTER DEFAULT PRIVILEGES を処理する関数群（例: ExecAlterDefaultPrivilegesStmt 関連の処理フロー）。
  • **呼び出し先**:
  • SetDefaultACL(InternalDefaultACL *iacls)
  • SetDefaultACLsInSchemas(InternalDefaultACL *iacls, List *nspnames)

関連データ構造

  • ObjectType: 対象オブジェクトの種類を定義する列挙型。
  • AclMode: 権限ビットマスク。
  • DropBehavior: 依存関係削除時の挙動（CASCADE / RESTRICT）。

失敗モードと境界条件

  • roleid や nspid が無効な場合、カタログ更新時にエラーが発生する可能性があります。
  • grantees リストが空の場合、権限操作の対象が存在しないため、処理がスキップされるかエラーとなる可能性があります。

保守担当者向けメモ

  • 本構造体は InternalGrant 構造体とフィールドの一部を共有しています。変更を加える際は、両者の整合性に注意してください。
  • pg_default_acl カタログのスキーマ変更が発生した場合は、本構造体のフィールド定義も追従する必要があります。
