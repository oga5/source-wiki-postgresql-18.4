pg_strcasecmp

定義

  • 種別: function
  • 定義箇所: src/port/pgstrcasecmp.c:35

目的

プラットフォームやロケールに依存しない、一貫した大文字・小文字を区別しない文字列比較を提供します。

シグネチャまたは形状

int pg_strcasecmp(const char *s1, const char *s2)

入力と出力

  • **入力**: 比較対象となる2つのヌル終端文字列 s1, s2。
  • **出力**:
  • 両文字列が等しい（大文字・小文字を無視して）場合: 0
  • 最初の不一致箇所において、変換後の s1 の文字コードが s2 より小さい場合: 負の整数
  • 最初の不一致箇所において、変換後の s1 の文字コードが s2 より大きい場合: 正の整数

前提条件と不変条件

  • 入力文字列はヌル終端されている必要があります。
  • 7ビットASCII範囲（'A'-'Z'）は常に小文字に変換されます。
  • ASCII範囲外の文字（IS_HIGHBIT_SET が真）については、現在のロケール設定に従い isupper および tolower が適用されます。

アルゴリズムまたは振る舞い

1. 文字列を1文字ずつ走査します。 2. 各文字について、以下の順序で変換を試みます。

  • 'A' から 'Z' の範囲内であれば、固定の計算式で小文字に変換します。
  • 上記以外で IS_HIGHBIT_SET が真かつ isupper が真であれば、tolower を使用してロケール依存の変換を行います。

    • 3. 変換後の文字を比較し、不一致があればその差分を返します。 4. ヌル終端に達するまで比較を継続し、最後まで一致すれば 0 を返します。

関連データ構造

  • c.h で定義される IS_HIGHBIT_SET マクロ。

失敗モードと境界条件

  • マルチバイト文字セットに対しては不完全な動作となります。
  • ヌル終端に達した際の終了判定は正確に行われます。

保守担当者向けメモ

  • 本関数はトルコ語ロケール等の特殊なケースでの不整合を避けるため、ASCII範囲外の文字に対してのみロケール関数を呼び出す設計です。
  • 厳密なASCII比較が必要な場合は、pg_ascii_ 接頭辞を持つ関数への切り替えを検討してください。
