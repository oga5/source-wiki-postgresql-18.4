setRuleCheckAsUser_walker

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteDefine.c:636

目的

ルール定義内のクエリツリーを再帰的に走査し、権限チェック対象となるユーザーID（Oid）を各 Query ノードに伝播させるためのウォーカー関数です。

シグネチャまたは形状

static bool setRuleCheckAsUser_walker(Node *node, Oid *context)

入力と出力

  • **入力**:
  • node: 走査対象のノード（Node *）。
  • context: 権限チェックに使用するユーザーID（Oid *）。
  • **出力**:
  • 走査を継続する場合は false、中断する場合は true を返します。

前提条件と不変条件

  • node が NULL の場合は直ちに false を返します。
  • context は有効なユーザーIDを指している必要があります。

アルゴリズムまたは振る舞い

1. ノードが Query 型である場合、setRuleCheckAsUser_Query を呼び出して当該クエリの権限設定を更新し、false を返して走査を継続します。 2. それ以外のノード型である場合、expression_tree_walker を使用して子ノードに対して再帰的に setRuleCheckAsUser_walker を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/rewrite/rewriteDefine.c 内の setRuleCheckAsUser および query_tree_walker から呼び出されます。
  • **呼び出し先**:
  • setRuleCheckAsUser_Query
  • expression_tree_walker

関連データ構造

  • Node: PostgreSQLの汎用ノード構造体。
  • Query: ルール内のクエリツリー構造体。
  • Oid: ユーザー識別子。

失敗モードと境界条件

  • node が NULL の場合は安全に処理を終了します。
  • expression_tree_walker の再帰的な走査により、ノードツリーの深さに応じたスタック消費が発生します。

保守担当者向けメモ

  • 本関数は expression_tree_walker を介した再帰的なツリー走査を行うため、Query ノード以外の構造変更が必要な場合は setRuleCheckAsUser_Query 側のロジックを確認してください。
