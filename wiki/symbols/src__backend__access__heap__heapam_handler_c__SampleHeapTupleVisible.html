SampleHeapTupleVisible

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_handler.c:2410

目的

テーブルスキャンにおいて、特定のタプルが現在のスナップショットに対して可視であるかを判定します。

シグネチャまたは形状

static bool SampleHeapTupleVisible(TableScanDesc scan, Buffer buffer, HeapTuple tuple, OffsetNumber tupoffset)

入力と出力

  • **入力**:
  • scan: テーブルスキャン記述子。
  • buffer: タプルを含むバッファ。
  • tuple: 判定対象のヒープタプル。
  • tupoffset: タプルのオフセット番号。
  • **出力**:
  • bool: 可視であれば true、そうでなければ false を返します。

前提条件と不変条件

  • scan は HeapScanDesc にキャスト可能である必要があります。
  • SO_ALLOW_PAGEMODE フラグが有効な場合、hscan->rs_vistuples 配列はソート済みであると仮定されます。

アルゴリズムまたは振る舞い

1. scan->rs_flags に SO_ALLOW_PAGEMODE が含まれている場合:

  • heap_prepare_pagescan() によって事前に計算された可視タプルリスト（rs_vistuples）を対象とします。
  • 二分探索を用いて、指定された tupoffset が可視リストに含まれているかを確認します。

    • 2. 上記フラグが含まれていない場合:

  • HeapTupleSatisfiesVisibility を呼び出し、スナップショットに基づいて個別に可視性を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c:2318
  • **呼び出し先**:
  • HeapTupleSatisfiesVisibility (可視性判定の実行)

関連データ構造

  • TableScanDesc / HeapScanDesc: スキャン状態を保持する構造体。
  • HeapTuple: ヒープタプルのデータ構造。
  • Buffer: バッファ管理用構造体。

失敗モードと境界条件

  • SO_ALLOW_PAGEMODE が有効な場合、リスト内の探索は二分探索に依存するため、リストがソートされていない場合は誤った結果を返す可能性があります。

保守担当者向けメモ

  • 本関数は heapam_handler.c 内の最適化されたスキャンパスの一部です。可視性判定ロジックそのものは HeapTupleSatisfiesVisibility に依存しているため、判定基準の変更が必要な場合はそちらを確認してください。
