OpClassCacheEnt

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/relcache.c:261

目的

演算子クラス（Operator Class）に関する情報をメモリ上にキャッシュし、システムカタログ（pg_am 等）への繰り返しアクセスを回避するためのデータ構造です。

シグネチャまたは形状

typedef struct opclasscacheent
{
    Oid            opclassoid;      /* 検索キー: 演算子クラスのOID */
    bool           valid;           /* 読み込み完了後にtrueに設定 */
    StrategyNumber numSupport;      /* サポートプロシージャの最大数 (pg_amより) */
    Oid            opcfamily;       /* 演算子クラスのファミリーOID */
    Oid            opcintype;       /* 演算子クラスの宣言された入力型OID */
    RegProcedure  *supportProcs;    /* サポートプロシージャのOID配列 */
} OpClassCacheEnt;

入力と出力

  • **入力**: 演算子クラスの OID をキーとしてハッシュテーブルから検索されます。
  • **出力**: 演算子クラスに関連するファミリー情報、入力型、およびサポートプロシージャのリストを提供します。

前提条件と不変条件

  • valid フィールドが true である場合、構造体内の各フィールドは有効な値で埋められていることが保証されます。
  • supportProcs は、numSupport に基づく適切なサイズの配列を指す必要があります。

アルゴリズムまたは振る舞い

  • LookupOpclassInfo 関数によって、キャッシュ内のエントリが検索または構築されます。
  • hash_search を用いて OpClassCache ハッシュテーブルからエントリを取得します。
  • キャッシュミスが発生した場合、システムカタログから情報を読み取り、エントリを初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**: LookupOpclassInfo 関数がこの構造体を操作する主要な呼び出し元です。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • OpClassCache: この構造体を要素として保持するハッシュテーブル。
  • pg_am, pg_opclass: このキャッシュの元となるシステムカタログ。

失敗モードと境界条件

  • キャッシュ検索時にメモリ不足が発生した場合、hash_search が失敗する可能性があります。
  • 存在しない OID を指定した場合、キャッシュエントリは構築されません。

保守担当者向けメモ

  • supportProcs 配列のメモリ管理は relcache.c 内のキャッシュ構築ロジックに依存しています。エントリの再構築や破棄を行う際は、メモリリークが発生しないよう注意してください。
