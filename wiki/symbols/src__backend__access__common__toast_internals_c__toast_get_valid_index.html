toast_get_valid_index

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/toast_internals.c:527

目的

指定されたTOASTテーブルのOIDから、そのテーブルに関連付けられた有効なインデックスのOIDを取得します。

シグネチャまたは形状

Oid toast_get_valid_index(Oid toastoid, LOCKMODE lock)

入力と出力

  • **入力**:
  • toastoid (Oid): 対象となるTOASTテーブルのOID。
  • lock (LOCKMODE): テーブルを開く際に使用するロックモード。
  • **出力**:
  • Oid: 特定された有効なインデックスのOID。

前提条件と不変条件

  • 指定された toastoid が有効なTOASTテーブルを指していること。
  • 呼び出し元は、適切なトランザクションコンテキストおよびスナップショットを保持していること。

アルゴリズムまたは振る舞い

1. table_open を使用して、指定された toastoid のリレーションを開きます。 2. toast_open_indexes を呼び出し、当該TOASTテーブルに関連する全インデックスをオープンし、その中から有効なインデックスのインデックス番号を取得します。 3. 取得したインデックス番号を用いて、toastidxs 配列から該当するインデックスのOIDを抽出します。 4. toast_close_indexes および table_close を呼び出し、オープンしたリレーションとインデックスをクローズします。 5. 抽出したインデックスのOIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/cluster.c (複数箇所)
  • **呼び出し先**:
  • table_open
  • toast_open_indexes
  • RelationGetRelid
  • toast_close_indexes
  • table_close

関連データ構造

  • Relation: TOASTテーブルおよびインデックスのリレーション情報を保持する構造体。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • 指定された toastoid が存在しない場合、table_open によりエラーが発生します。
  • TOASTテーブルに有効なインデックスが存在しない場合、toast_open_indexes の挙動に依存します。

保守担当者向けメモ

  • 本関数は、TOASTテーブルのインデックスを一時的に開いてOIDを特定するユーティリティです。インデックスのオープン・クローズを完結させるため、呼び出し元でインデックスを保持し続ける必要がない場合に適しています。
