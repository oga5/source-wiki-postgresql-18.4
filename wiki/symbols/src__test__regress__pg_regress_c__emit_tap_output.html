emit_tap_output

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress.c:330

目的

テスト結果や診断情報を TAP (Test Anything Protocol) 形式で出力するためのラッパー関数です。可変長引数を受け取り、内部で emit_tap_output_v を呼び出すことで出力処理を委譲します。

シグネチャまたは形状

static void emit_tap_output(TAPtype type, const char *fmt, ...);

入力と出力

  • **入力**:
  • type: TAPtype 列挙型。出力する TAP メッセージの種類を指定します。
  • fmt: 出力フォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（戻り値は void）。標準出力へ TAP 準拠の文字列を書き出します。

前提条件と不変条件

  • fmt 引数は printf 形式のフォーマット文字列である必要があります。
  • va_list を使用するため、呼び出し元は適切な引数を提供する必要があります。

アルゴリズムまたは振る舞い

1. va_start マクロを使用して可変長引数リスト argp を初期化します。 2. emit_tap_output_v(type, fmt, argp) を呼び出し、実際の出力処理を実行します。 3. va_end マクロを使用して argp をクリーンアップします。

呼び出し元と呼び出し先

  • **呼び出し先**: emit_tap_output_v
  • **呼び出し元**:
  • マクロ経由での呼び出し: plan(), note(), note_detail(), diag(), note_end()
  • 直接の呼び出し: src/test/regress/pg_regress.c:290（テストステータス出力時）

関連データ構造

  • TAPtype: 出力メッセージの分類を定義する列挙型。

失敗モードと境界条件

  • 可変長引数の型と fmt の指定が一致しない場合、未定義の動作を引き起こす可能性があります。
  • 標準出力への書き込みが失敗した場合の挙動は、呼び出し先の emit_tap_output_v に依存します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、ロジックの変更は emit_tap_output_v に対して行う必要があります。
  • pg_attribute_printf(2, 3) が宣言されているため、コンパイラによるフォーマット文字列の型チェックが有効です。修正時は引数の整合性に注意してください。
