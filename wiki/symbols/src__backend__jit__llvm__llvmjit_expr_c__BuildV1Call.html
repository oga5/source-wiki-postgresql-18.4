BuildV1Call

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_expr.c:3007

目的

PostgreSQL の V1 形式関数呼び出しを LLVM IR に変換し、実行可能なコードを生成するヘルパー関数。

シグネチャまたは形状

static LLVMValueRef BuildV1Call(LLVMJitContext *context, LLVMBuilderRef b, LLVMModuleRef mod, FunctionCallInfo fcinfo, LLVMValueRef *v_fcinfo_isnull)

入力と出力

  • **入力**:
  • context: JIT コンテキスト。
  • b: LLVM ビルダー。
  • mod: LLVM モジュール。
  • fcinfo: 呼び出す関数の情報（FunctionCallInfo）。
  • v_fcinfo_isnull: NULL 判定結果を格納するためのポインタ（オプション）。
  • **出力**:
  • 呼び出し結果を表す LLVMValueRef。

前提条件と不変条件

  • fcinfo は有効な FunctionCallInfoData 構造体を指している必要がある。
  • LLVM_VERSION_MAJOR < 22 の環境では、create_LifetimeEnd を用いたメモリ生存期間の注釈が生成される。

アルゴリズムまたは振る舞い

1. llvm_function_reference を呼び出し、対象関数の参照を取得する。 2. FunctionCallInfoData の isnull フィールドを 0 に初期化する。 3. l_call を使用して、対象関数を呼び出す LLVM IR 命令を生成する。 4. v_fcinfo_isnull が指定されている場合、呼び出し後の isnull 状態をロードして格納する。 5. LLVM バージョンが 22 未満の場合、create_LifetimeEnd を呼び出し、引数および isnull 配列に対する生存期間終了の注釈を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/llvmjit_expr.c 内の各箇所（行 744, 1587, 1682, 1844, 2206, 2479, 2802）。
  • **呼び出し先**: llvm_function_reference, l_ptr_const, l_struct_gep, LLVMBuildStore, l_call, l_load, create_LifetimeEnd 等。

関連データ構造

  • FunctionCallInfoData: 関数呼び出し情報を保持する構造体。
  • LLVMJitContext: JIT コンパイルの管理コンテキスト。

失敗モードと境界条件

  • fcinfo が無効な場合、LLVM IR 生成時に未定義の動作を引き起こす可能性がある。
  • LLVM バージョン依存の条件分岐により、古いバージョンでは create_LifetimeEnd が必須となる。

保守担当者向けメモ

  • l_ で始まる関数は llvmjit_emit.h で定義されたラッパーである。
  • 新しい LLVM バージョンへの移行時には、#if LLVM_VERSION_MAJOR < 22 ブロックの必要性を再評価すること。
