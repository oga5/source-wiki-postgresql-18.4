JsonbInState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonb.c:28

目的

JSONB データの構築プロセスにおいて、パース中の状態を保持および管理するために使用される構造体です。

シグネチャまたは形状

typedef struct JsonbInState
{
	JsonbParseState *parseState;
	JsonbValue *res;
	bool		unique_keys;
	Node	   *escontext;
} JsonbInState;

入力と出力

  • **入力**: JsonbParseState（パース状態の追跡用）、JsonbValue（結果格納用）、unique_keys（キーの重複チェックフラグ）、escontext（エラー報告用ノード）。
  • **出力**: 構築された JsonbValue を含む状態オブジェクト。

前提条件と不変条件

  • parseState は、jsonb_util.c で提供される pushJsonbValue 等の関数による構造構築の基盤として機能します。
  • unique_keys が true の場合、構築過程でキーの重複が検証されることが期待されます。

アルゴリズムまたは振る舞い

  • jsonb_from_cstring 等の関数において、JSON テキストの解析結果を JsonbValue 形式へ積み上げるためのコンテキストとして機能します。
  • composite_to_jsonb や datum_to_jsonb_internal などの再帰的な変換関数に渡され、変換中のデータ構造を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: jsonb_in、composite_to_jsonb、array_to_jsonb_internal、datum_to_jsonb_internal、add_jsonb など、jsonb.c 内の変換ロジックから参照されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、JsonbParseState を介して jsonb_util.c の関数群と連携します。

関連データ構造

  • JsonbParseState: パースの進捗を管理する構造体。
  • JsonbValue: JSONB の各ノード（スカラ、オブジェクト、配列）を表現する構造体。

失敗モードと境界条件

  • escontext が提供されている場合、パースエラーや構築エラーは escontext を通じて報告されます。
  • メモリ不足時には、PostgreSQL のメモリ管理機構によりエラーがスローされます。

保守担当者向けメモ

  • 本構造体は jsonb_util.c で定義される低レベルなバイナリ操作関数と、jsonb.c の SQL インターフェースを繋ぐ重要な仲介役です。
  • 構造体のフィールド追加や変更を行う際は、datum_to_jsonb_internal などの再帰的な変換関数への影響を考慮してください。
