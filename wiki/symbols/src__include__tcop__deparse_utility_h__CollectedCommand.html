CollectedCommand

定義

  • 種別: struct
  • 定義箇所: src/include/tcop/deparse_utility.h:44

目的

実行されたユーティリティコマンド（DDL等）の構造を保持し、イベントトリガーや監査ログ、レプリケーション等の後続処理で再利用可能な形式（Deparse可能な形式）で管理するために使用されます。

シグネチャまたは形状

typedef struct CollectedCommand
{
    CollectedCommandType type;
    bool        in_extension;
    Node       *parsetree;
    union { ... } d;
    struct CollectedCommand *parent;
} CollectedCommand;

入力と出力

  • **入力**: ユーティリティコマンド実行時に、コマンドの種類に応じた情報が d 共用体および関連フィールドに格納されます。
  • **出力**: 構造体自体がコマンドの追跡情報として保持され、必要に応じて parent ポインタを介してネストされたコマンド構造を提供します。

前提条件と不変条件

  • d 共用体内のメンバにアクセスする際は、必ず type フィールドを確認し、コマンドの種類に応じた適切な共用体メンバを選択する必要があります。
  • コマンドがネストしている場合、parent ポインタが上位の CollectedCommand を指します。

アルゴリズムまたは振る舞い

1. ユーティリティコマンド実行時にインスタンスが生成されます。 2. コマンドの種類（CollectedCommandType）に応じて、d 共用体内の適切な構造体（simple, alterTable, grant, opfam, createopc, atscfg, defprivs のいずれか）が埋められます。 3. 拡張機能の一部として実行された場合は in_extension フラグが設定されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/event_trigger.c 内の各関数（palloc を用いてメモリ確保が行われていることが確認されています）。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接的な関数呼び出し先はありません。

関連データ構造

  • CollectedCommandType: コマンドの種類を識別する列挙型。
  • CollectedATSubcmd: ALTER TABLE のサブコマンドを管理する構造体。
  • ObjectAddress: データベースオブジェクトを特定する構造体。
  • InternalGrant: GRANT/REVOKE 情報を保持する構造体。
  • Node: パースツリーの基底ノード。

失敗モードと境界条件

  • 共用体を使用しているため、type と不一致のメンバにアクセスした場合の動作は未定義です。
  • メモリ確保失敗時の挙動については、標準的な palloc の動作に依存します。

保守担当者向けメモ

  • 本構造体は純粋なデータ定義です。実装ロジックは含まれていません。
  • in_extension フラグは、コマンドが拡張機能の一部であるかを判定する重要なフラグです。
  • 複雑なコマンド（ALTER TABLE 等）は階層構造を持つため、parent ポインタによるトラバースを考慮した設計が必要です。
