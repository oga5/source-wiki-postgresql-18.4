shimTriConsistentFn

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginlogic.c:147

目的

二値論理のみをサポートする演算子クラス（opclass）の整合性関数から、GINスキャンに必要な三値論理（GinTernaryValue）の結果を導出するアダプター関数。

シグネチャまたは形状

GinTernaryValue shimTriConsistentFn(GinScanKey key)

入力と出力

  • **入力**: GinScanKey 型のポインタ。key->entryRes に現在のエントリの合致状態（GIN_TRUE, GIN_FALSE, GIN_MAYBE）が格納されている。
  • **出力**: GinTernaryValue 型。GIN_TRUE, GIN_FALSE, GIN_MAYBE のいずれかを返す。

前提条件と不変条件

  • key->entryRes 内の GIN_MAYBE の数が MAX_MAYBE_ENTRIES（4）以下であること。
  • 実行終了時に、一時的に変更した key->entryRes の状態を元の GIN_MAYBE に復元すること。

アルゴリズムまたは振る舞い

1. key->entryRes 内の GIN_MAYBE を検索し、そのインデックスを記録する。数が MAX_MAYBE_ENTRIES を超える場合は即座に GIN_MAYBE を返す。 2. MAYBE が存在しない場合は directBoolConsistentFn を呼び出し、その結果を返す。 3. MAYBE が存在する場合、それらを GIN_FALSE と GIN_TRUE の全組み合わせ（最大 $2^4$ 通り）で順次入れ替え、directBoolConsistentFn を繰り返し実行する。 4. 試行した結果、論理値が変化した場合は GIN_MAYBE と判定する。 5. 最終的な結果が GIN_TRUE であり、かつ recheck フラグが立っている場合は GIN_MAYBE に変換して返す。 6. 最後に key->entryRes を元の状態に復元する。

呼び出し元と呼び出し先

  • **呼び出し元**: ginInitConsistentFunction により GinScanKey の triConsistentFn フィールドに割り当てられる（src/backend/access/gin/ginlogic.c:248）。
  • **呼び出し先**: directBoolConsistentFn を繰り返し呼び出す。

関連データ構造

  • GinScanKey: 整合性チェックの状態を保持する構造体。
  • GinTernaryValue: 三値論理（TRUE/FALSE/MAYBE）を表現する列挙型。

失敗モードと境界条件

  • MAYBE 入力が MAX_MAYBE_ENTRIES を超える場合、計算コストを避けるため即座に GIN_MAYBE を返す。
  • recheck フラグが TRUE と共に返された場合、呼び出し元に対して GIN_MAYBE として振る舞う。

保守担当者向けメモ

  • 本関数は MAYBE 入力に対して指数関数的な計算量を要するため、パフォーマンス上のボトルネックとなる可能性がある。
  • key->entryRes を一時的に書き換えるため、将来的な並列スキャン実装時には排他制御の検討が必要。
