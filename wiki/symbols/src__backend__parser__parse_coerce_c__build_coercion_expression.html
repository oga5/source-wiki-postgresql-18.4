build_coercion_expression

定義

  • src/backend/parser/parse_coerce.c:836

目的

指定された変換パス（CoercionPathType）に基づき、式ノードをターゲット型へ変換するための適切なノード（FuncExpr, ArrayCoerceExpr, CoerceViaIO）を構築する。

シグネチャまたは形状

Node *build_coercion_expression(Node *node,
                                CoercionPathType pathtype,
                                Oid funcId,
                                Oid targetTypeId, int32 targetTypMod,
                                CoercionContext ccontext, CoercionForm cformat,
                                int location)

入力と出力

  • **入力**: 変換対象の node、変換手法 pathtype、変換関数 funcId（必要な場合）、ターゲット型情報（targetTypeId, targetTypMod）、変換コンテキストおよびフォーマット、構文位置。
  • **出力**: 変換後の Node*。変換に失敗した場合はエラーを投げる。

前提条件と不変条件

  • pathtype が COERCION_PATH_FUNC の場合、funcId は有効な OID である必要がある。
  • pathtype が COERCION_PATH_COERCEVIAIO の場合、funcId は無効である必要がある。
  • 変換関数を使用する場合、その関数は PROKIND_FUNCTION であり、引数数は1〜3であること。

アルゴリズムまたは振る舞い

1. **関数検証**: funcId が有効な場合、システムカタログ pg_proc を参照し、関数が型変換として適切か（引数数や型）を検証する。 2. **ノード生成**: pathtype に応じて以下のいずれかのノードを生成する：

  • COERCION_PATH_FUNC: FuncExpr を生成。必要に応じてターゲットの typmod や明示的変換フラグを引数として追加する。
  • COERCION_PATH_ARRAYCOERCE: ArrayCoerceExpr を生成。配列要素ごとに coerce_to_target_type を再帰的に呼び出し、要素の変換ノードを構築する。
  • COERCION_PATH_COERCEVIAIO: CoerceViaIO を生成。

    • 3. **メタデータ付与**: 生成したノードに location や resulttype などの情報を設定して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_coerce.c 内の coerce_type 等から呼び出される。
  • **呼び出し先**: coerce_to_target_type（配列要素の変換時）、makeFuncExpr, makeConst, makeNode 等のノード生成関数。

関連データ構造

  • FuncExpr, ArrayCoerceExpr, CoerceViaIO: 変換結果として返されるノード型。
  • pg_proc: 変換関数のメタデータ取得に使用。
  • CaseTestExpr: 配列要素の変換時に一時的に使用される。

失敗モードと境界条件

  • funcId が無効な場合や、キャッシュルックアップに失敗した場合は elog(ERROR) を発生させる。
  • サポートされていない pathtype が渡された場合、elog(ERROR) を発生させる。
  • 配列要素の変換に失敗した場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は元の式ツリーを破壊せず、新しいノードでラップして返す設計である。
  • 配列変換時の CaseTestExpr は、配列要素の型変換を表現するための便宜的な利用であるため、このノードが他のコンテキストで再利用されないよう注意が必要である。
