GistHstoreOptions

定義

  • 種別: struct
  • 定義箇所: contrib/hstore/hstore_gist.c:15

目的

hstore 型の GiST インデックスにおける設定オプションを保持するための構造体です。特にインデックスのシグネチャ長（siglen）を管理します。

シグネチャまたは形状

typedef struct
{
	int32		vl_len_;		/* varlena header */
	int			siglen;			/* signature length in bytes */
} GistHstoreOptions;

入力と出力

  • 入力: インデックス作成時や設定変更時に reloptions として提供されます。
  • 出力: siglen フィールドを通じて、インデックスのシグネチャ生成および検索アルゴリズムにシグネチャ長を提供します。

前提条件と不変条件

  • vl_len_ は varlena ヘッダーとして機能するため、直接操作してはなりません。
  • siglen はインデックスのシグネチャサイズを決定し、インデックスの衝突率とサイズに影響を与えます。

アルゴリズムまたは振る舞い

  • インデックスの初期化時に init_local_reloptions を介してメモリが割り当てられます。
  • インデックスの演算子クラスオプションとして取得され、シグネチャの計算ロジックにおいて動的なサイズ指定に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/hstore/hstore_gist.c 内のインデックスオプション初期化処理および演算子クラスオプション取得処理。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すものではありません。

関連データ構造

  • GISTTYPE: インデックスに格納されるシグネチャを保持する構造体。
  • reloptions: PostgreSQL のインデックスオプション管理機構。

失敗モードと境界条件

  • siglen の値が不適切な場合、インデックスの構築や検索時のメモリ割り当てに影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体は varlena 型として定義されているため、メモリレイアウトの変更には注意が必要です。
  • siglen はデフォルトで 16 バイトですが、ユーザー設定により変更可能です。
