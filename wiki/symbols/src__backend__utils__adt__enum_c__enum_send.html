enum_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:220

目的

列挙型（ENUM）の値をバイナリ形式に変換し、クライアントへ送信するためのデータ変換関数です。

シグネチャまたは形状

Datum enum_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して列挙型の内部表現（OID）を受け取ります。
  • **出力**: bytea 型として、列挙型のラベル文字列をバイナリ形式で返します。

前提条件と不変条件

  • 入力されたOIDに対応するエントリがシステムカタログ pg_enum に存在する必要があります。
  • 存在しないOIDが渡された場合、ERRCODE_INVALID_BINARY_REPRESENTATION エラーが発生します。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用し、ENUMOID キャッシュから指定されたOIDに対応する pg_enum のタプルを取得します。 2. タプルが有効であることを確認し、GETSTRUCT を介して Form_pg_enum 構造体へキャストします。 3. pq_begintypsend でバイナリ送信用のバッファを初期化します。 4. pq_sendtext を使用し、列挙型のラベル（en->enumlabel）をバッファに書き込みます。 5. ReleaseSysCache でキャッシュタプルを解放します。 6. pq_endtypsend で構築されたバッファを bytea 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（SQLレベルの列挙型出力処理）。
  • **呼び出し先**:
  • SearchSysCache1 / ReleaseSysCache（システムカタログキャッシュ操作）
  • pq_begintypsend / pq_sendtext / pq_endtypsend（libpqプロトコル変換）
  • GETSTRUCT（タプル構造体アクセス）

関連データ構造

  • Form_pg_enum: pg_enum カタログの行構造体。
  • StringInfoData: バイナリ送信用の動的バッファ。
  • HeapTuple: システムカタログから取得したタプル。

失敗モードと境界条件

  • 指定されたOIDがシステムカタログに存在しない場合、エラーを報告して処理を中断します。

保守担当者向けメモ

  • 本関数は列挙型のバイナリ出力のみを担います。列挙型の整合性チェックについては、必要に応じて check_safe_enum_use 等のガード関数が呼び出し元で考慮されていることを前提としています。
