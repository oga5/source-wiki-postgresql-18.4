conditional_stack_pop

定義

  • 種別: function
  • 定義箇所: src/fe_utils/conditional.c:68

目的

ConditionalStack のスタックトップにある要素を削除し、メモリを解放します。

シグネチャまたは形状

bool conditional_stack_pop(ConditionalStack cstack)

入力と出力

  • **入力**: cstack (操作対象の ConditionalStack 構造体へのポインタ)
  • **出力**: bool 型。スタックから要素が正常にポップされた場合は true、スタックが空でポップできなかった場合は false を返します。

前提条件と不変条件

  • cstack は有効な ConditionalStack 構造体を指している必要があります。
  • スタックの内部構造（head ポインタ）は、conditional_stack_push によって適切に管理されている必要があります。

アルゴリズムまたは振る舞い

1. スタックの現在の先頭要素 (cstack->head) をローカル変数 p に保持します。 2. p が NULL（スタックが空）であれば、何もせず false を返します。 3. スタックの先頭ポインタを次の要素 (cstack->head->next) に更新します。 4. 保持していた要素 p を free() で解放します。 5. true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pgbench/pgbench.c
  • src/bin/psql/command.c
  • src/bin/psql/mainloop.c
  • src/fe_utils/conditional.c (内部の conditional_stack_reset 等から)
  • **呼び出し先**:
  • 標準ライブラリの free()

関連データ構造

  • ConditionalStack: スタック全体を管理する構造体。
  • IfStackElem: スタックの各要素を構成する構造体。

失敗モードと境界条件

  • **スタックが空の場合**: p が NULL となり、関数は false を返して安全に終了します。

保守担当者向けメモ

  • 本関数は要素のメモリ解放のみを行います。スタック全体の破棄が必要な場合は conditional_stack_destroy を使用してください。
  • cstack 自体が NULL である場合の挙動については、ソースコード上での明示的なガードは含まれていません。
