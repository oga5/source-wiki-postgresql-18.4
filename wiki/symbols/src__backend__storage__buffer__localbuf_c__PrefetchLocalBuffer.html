PrefetchLocalBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/localbuf.c:70

目的

一時テーブルのブロックに対するプリフェッチ（先読み）要求を処理します。データがローカルバッファ内に存在しない場合、ストレージマネージャを介して非同期的な読み込みを開始します。

シグネチャまたは形状

PrefetchBufferResult PrefetchLocalBuffer(SMgrRelation smgr, ForkNumber forkNum, BlockNumber blockNum)

入力と出力

  • **入力**:
  • smgr: 対象リレーションのストレージマネージャハンドル。
  • forkNum: 対象のフォーク番号。
  • blockNum: プリフェッチ対象のブロック番号。
  • **出力**:
  • PrefetchBufferResult: プリフェッチの結果を格納する構造体。recent_buffer（バッファが存在する場合のID）と initiated_io（I/Oが開始されたかどうかのフラグ）を含む。

前提条件と不変条件

  • 初回呼び出し時に LocalBufHash が未初期化であれば、InitLocalBuffers() を呼び出して初期化する必要があります。
  • 本関数はプロセスローカルなバッファのみを対象とします。

アルゴリズムまたは振る舞い

1. InitBufferTag を使用して、指定されたブロックを識別するためのタグを生成します。 2. LocalBufHash を検索し、対象ブロックが既にローカルバッファに存在するか確認します。 3. 存在する場合、そのバッファIDを result.recent_buffer に設定して終了します。 4. 存在しない場合、USE_PREFETCH が定義されており、かつ IO_DIRECT_DATA フラグが設定されていない条件で smgrprefetch を呼び出し、I/O を開始します。 5. I/O が開始された場合、result.initiated_io を true に設定して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c:665
  • **呼び出し先**:
  • InitLocalBuffers() (初期化が必要な場合)
  • hash_search() (ハッシュテーブル検索)
  • smgrprefetch() (ストレージ層へのプリフェッチ要求)

関連データ構造

  • LocalBufferLookupEnt: ハッシュテーブルの検索結果として使用される構造体。
  • PrefetchBufferResult: 戻り値の型。
  • LocalBufHash: ローカルバッファの検索に使用されるハッシュテーブル。

失敗モードと境界条件

  • LocalBufHash が初期化されていない場合、内部で InitLocalBuffers() が自動的に実行されます。
  • USE_PREFETCH が無効な環境や、IO_DIRECT_DATA が有効な場合はプリフェッチがスキップされます。

保守担当者向けメモ

  • 本関数は一時テーブル専用のバッファマネージャの一部です。共有バッファマネージャの PrefetchBuffer とは異なり、ロックの取得は不要です。
  • smgrprefetch の戻り値が true の場合のみ initiated_io が true になる点に注意してください。
