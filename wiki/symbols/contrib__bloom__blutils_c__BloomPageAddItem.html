BloomPageAddItem

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blutils.c:322

目的

Bloom インデックスページ内に新しい Bloom タプルを格納します。

シグネチャまたは形状

bool BloomPageAddItem(BloomState *state, Page page, BloomTuple *tuple)

入力と出力

  • **入力**:
  • state: インデックスの実行時状態を保持する BloomState 構造体。
  • page: タプルを追加する対象のページ。
  • tuple: 追加対象の BloomTuple。
  • **出力**:
  • bool: タプルがページに正常に格納された場合は true、空き容量不足で格納できなかった場合は false を返します。

前提条件と不変条件

  • page は初期化済みである必要があり、PageIsNew または BloomPageIsDeleted であってはなりません。
  • 処理終了時、ページヘッダーの pd_lower が pd_upper を超えていないこと（領域のオーバーランがないこと）が保証されます。

アルゴリズムまたは振る舞い

1. BloomPageGetFreeSpace を使用し、ページ内に state->sizeOfBloomTuple 分の空きがあるか確認します。不足している場合は false を返します。 2. BloomPageGetOpaque でページ内のメタデータ（opaque）を取得します。 3. opaque->maxoff を基準に、ページ内の末尾位置を特定し、memcpy で tuple をコピーします。 4. opaque->maxoff をインクリメントし、ページヘッダーの pd_lower を更新して新しい空き領域の境界を再設定します。 5. 成功時に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/bloom/blinsert.c 内の複数の箇所（インデックス構築および挿入処理）から呼び出されます。
  • **呼び出し先**: ページ操作用のマクロ（BloomPageGetFreeSpace, BloomPageGetOpaque, BloomPageGetTuple）および標準の memcpy を使用します。

関連データ構造

  • BloomState: インデックスの構成情報（タプルサイズ等）を保持。
  • BloomTuple: インデックスに格納されるデータ単位。
  • BloomPageOpaque: ページ内のオフセット管理情報。

失敗モードと境界条件

  • ページ内にタプルを格納するための十分な空き領域がない場合、関数は false を返し、ページの状態は変更されません。

保守担当者向けメモ

  • 本関数はページ内の物理的なレイアウト操作を直接行います。pd_lower の更新ロジックはページヘッダーの構造に依存しているため、ページレイアウトを変更する際は注意してください。
