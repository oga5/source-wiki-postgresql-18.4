xact_desc_prepare

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xactdesc.c:396

目的

トランザクションのプリペア（PREPARE TRANSACTION）に関連するWALレコードを解析し、人間が読める形式の文字列として StringInfo バッファへ出力する。

シグネチャまたは形状

static void xact_desc_prepare(StringInfo buf, uint8 info, xl_xact_prepare *xlrec, RepOriginId origin_id)

入力と出力

  • buf: 出力先の StringInfo バッファ。
  • info: WALレコードのフラグ情報。
  • xlrec: プリペアレコードのデータ構造体。
  • origin_id: レプリケーションオリジンID。
  • 出力: なし（buf に文字列が追記される）。

前提条件と不変条件

  • xlrec は有効な xl_xact_prepare 構造体を指している必要がある。
  • ParsePrepareRecord が内部で呼び出され、xlrec から xl_xact_parsed_prepare 構造体へデータが展開される。

アルゴリズムまたは振る舞い

1. ParsePrepareRecord を呼び出し、WALレコードから解析済み構造体 parsed を生成する。 2. グローバル・トランザクションID (gid) とタイムスタンプを buf に書き込む。 3. xact_desc_relations を使用して、コミット時およびアボート時に影響を受けるリレーション情報を出力する。 4. xact_desc_stats を使用して、コミットおよびアボートに関連する統計情報を出力する。 5. xact_desc_subxacts を使用して、サブトランザクション情報を出力する。 6. standby_desc_invalidations を使用して、スタンバイ用の無効化メッセージを出力する。 7. origin_id が InvalidRepOriginId でない場合、レプリケーションオリジンのノードID、LSN、タイムスタンプを追記する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/rmgrdesc/xactdesc.c 内の xact_desc 関数（WALレコードの種別判定に基づき呼び出される）。
  • 呼び出し先: ParsePrepareRecord, appendStringInfo, appendStringInfoString, timestamptz_to_str, xact_desc_relations, xact_desc_stats, xact_desc_subxacts, standby_desc_invalidations。

関連データ構造

  • xl_xact_prepare: WAL上のプリペアレコード構造体。
  • xl_xact_parsed_prepare: 解析後のプリペアレコード構造体。
  • StringInfo: 文字列構築用バッファ。

失敗モードと境界条件

  • origin_id が InvalidRepOriginId の場合、レプリケーションオリジンに関する情報は出力されない。
  • 解析対象のレコード内容が不正な場合、ParsePrepareRecord の挙動に依存する。

保守担当者向けメモ

  • 本関数はバックエンドとフロントエンド（pg_waldump）で共有されるコードであるため、依存関係の追加には注意が必要。
  • WALレコードのパースはオフセット計算を伴うため、構造体の変更時には ParsePrepareRecord との整合性を確認すること。
