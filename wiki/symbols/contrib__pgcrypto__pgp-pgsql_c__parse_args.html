parse_args

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pgsql.c:305

目的

ユーザーから提供された設定文字列（例: cipher-algo=aes,compress-algo=1）を解析し、その内容を PGP_Context に反映させるためのラッパー関数です。

シグネチャまたは形状

static int parse_args(PGP_Context *ctx, uint8 *args, int arg_len, struct debug_expect *ex);

入力と出力

  • **入力**:
  • ctx: 設定を適用する PGP_Context 構造体へのポインタ。
  • args: 解析対象のオプション文字列（バイト配列）。
  • arg_len: args の長さ。
  • ex: デバッグ用の期待値構造体。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は負の整数（PXE_ARGUMENT_ERROR 等）を返します。

前提条件と不変条件

  • args は downcase_convert によって小文字に変換された後、内部で処理されます。
  • 解析された各キーと値のペアは、set_arg を通じて ctx に適用されます。
  • 処理終了時に、内部で確保した一時メモリは pfree により解放されます。

アルゴリズムまたは振る舞い

1. downcase_convert を呼び出し、入力文字列を小文字に正規化したコピーを作成します。 2. getword を使用して、key=value 形式のペアを順次抽出します。 3. 各ペアに対して以下の検証を行います：

  • = および , 区切り文字の存在確認。
  • キーまたは値が空でないことの確認。

    • 4. set_arg を呼び出し、抽出したキーと値を ctx に設定します。 5. 解析ループ終了後、一時的に確保した文字列バッファを pfree で解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pgsql.c 内の関数（init_work 等）から呼び出されます。
  • **呼び出し先**:
  • downcase_convert
  • getword
  • set_arg
  • pfree

関連データ構造

  • PGP_Context: PGP処理の状態を保持する構造体。
  • debug_expect: テスト目的で設定を検証するための構造体。

失敗モードと境界条件

  • 文字列のフォーマットが不正（= や , の欠落）な場合、PXE_ARGUMENT_ERROR を返します。
  • キーまたは値が空の場合、解析を中断しエラーを返します。
  • set_arg が負の値を返した場合、解析を中断しそのエラーコードを伝播させます。

保守担当者向けメモ

  • 本関数は pgcrypto のSQLインターフェースにおける引数パースの要です。新しい設定オプションを追加する場合は、set_arg 側の修正と併せて確認してください。
  • downcase_convert でメモリを動的に確保しているため、エラー発生時を含め確実に pfree が実行されるフローであることを維持してください。
