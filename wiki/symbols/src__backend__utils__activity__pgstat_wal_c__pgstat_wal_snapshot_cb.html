pgstat_wal_snapshot_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_wal.c:167

目的

共有メモリ上に保持されているWAL統計情報を、プロセスローカルなスナップショット領域へコピーする。

シグネチャまたは形状

void pgstat_wal_snapshot_cb(void)

入力と出力

  • 入力: なし（共有メモリ上の pgStatLocal.shmem->wal を参照）
  • 出力: なし（pgStatLocal.snapshot.wal を更新）

前提条件と不変条件

  • pgStatLocal.shmem が初期化されていること。
  • 共有メモリ上のWAL統計構造体 PgStatShared_Wal が有効であること。

アルゴリズムまたは振る舞い

1. 共有メモリ上のWAL統計構造体へのポインタを取得する。 2. LWLockAcquire を用いて、共有メモリ上の統計データに対して LW_SHARED モードでロックを取得する。 3. memcpy を使用し、共有メモリ上の統計データ（stats_shmem->stats）をローカルスナップショット（pgStatLocal.snapshot.wal）へコピーする。 4. LWLockRelease を呼び出し、ロックを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/activity/pgstat.c において、統計スナップショット取得用コールバックとして登録されている。
  • 呼び出し先: LWLockAcquire, memcpy, LWLockRelease（いずれもPostgreSQL内部のユーティリティ関数）。

関連データ構造

  • PgStatShared_Wal: 共有メモリ上のWAL統計管理構造体。
  • pgStatLocal: 統計サブシステムのプロセスローカルな状態を保持するグローバル変数。

失敗モードと境界条件

  • 共有メモリのロック取得時に競合が発生した場合、ロックが取得できるまで待機する。

保守担当者向けメモ

  • 本関数は pgstat モジュールのコールバックインターフェースの一部である。
  • 共有メモリへのアクセスには必ずロックが必要であり、本実装では LW_SHARED を使用して読み取りを行っている。
