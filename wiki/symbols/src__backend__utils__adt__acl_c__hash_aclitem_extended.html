hash_aclitem_extended

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/acl.c:782

目的

AclItem 構造体の内容に基づき、ハッシュ値を計算します。シード値を用いた拡張ハッシュ計算をサポートします。

シグネチャまたは形状

Datum hash_aclitem_extended(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_ACLITEM_P(0): ハッシュ対象の AclItem 構造体へのポインタ。
  • PG_GETARG_INT64(1): ハッシュ計算に使用するシード値。
  • **出力**:
  • Datum: 計算されたハッシュ値（uint64 または uint32）。

前提条件と不変条件

  • 入力される AclItem は有効なポインタである必要があります。
  • AclItem 構造体の ai_privs、ai_grantee、ai_grantor フィールドがハッシュ計算の基盤となります。

アルゴリズムまたは振る舞い

1. AclItem の各フィールド（ai_privs, ai_grantee, ai_grantor）の値を加算し、uint32 型の sum を算出します。 2. シード値が 0 の場合、sum をそのまま Datum として返します。 3. シード値が 0 以外の場合、hash_uint32_extended(sum, seed) を呼び出し、シードを考慮したハッシュ値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**: hash_uint32_extended（common/hashfn.h で定義されていると推測される関数）を呼び出します。

関連データ構造

  • AclItem: 権限付与の最小単位を保持する構造体。ai_privs（権限ビット）、ai_grantee（被付与者）、ai_grantor（付与者）で構成されます。

失敗モードと境界条件

  • シード値が 0 の場合、単純な加算結果が返されるため、ハッシュの衝突が発生しやすくなる可能性があります。

保守担当者向けメモ

  • 本関数は AclItem をハッシュテーブルのキーとして使用する際に利用されます。
  • hash_uint32_extended の実装変更が本関数の出力に直接影響を与えるため、ハッシュアルゴリズムの変更時には注意が必要です。
