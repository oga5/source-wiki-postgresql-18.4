InitPostgres

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/postinit.c:711

目的

PostgreSQL バックエンドプロセスの初期化を統括し、プロセスを特定のデータベースおよびユーザーコンテキストに紐付け、実行可能な状態へ移行させる。

シグネチャまたは形状

void InitPostgres(const char *in_dbname, Oid dboid,
                  const char *username, Oid useroid,
                  bits32 flags,
                  char *out_dbname)

入力と出力

  • **入力**:
  • in_dbname: 接続先データベース名。
  • dboid: 接続先データベースの OID。
  • username: 接続ユーザー名。
  • useroid: 接続ユーザーの OID。
  • flags: 初期化フラグ。
  • out_dbname: データベース名を格納するバッファ。
  • **出力**: なし（副作用としてプロセスの状態を初期化）。

前提条件と不変条件

  • カタログキャッシュが構築される前にシステムカタログへアクセスしてはならない。
  • InitPostgres は非常に多くの依存関係を持つため、呼び出し順序が厳格に守られる必要がある。
  • 認証やロック取得のハングに備え、ProcArray への登録および PgBackendStatus の初期化が先行して行われる。

アルゴリズムまたは振る舞い

1. **プロセス登録**: InitProcessPhase2 で ProcArray に登録し、他バックエンドから可視化する。 2. **統計・シグナル初期化**: pgstat_beinit および ProcSignalInit を実行。 3. **タイムアウト設定**: ブートストラップ時を除き、デッドロックや各種タイムアウトハンドラを登録。 4. **XLOG 起動**: Postmaster 配下でない場合、StartupXLOG を実行し終了時のクリーンアップを登録。 5. **キャッシュ構築**: RelationCache, CatalogCache, InitPlanCache を初期化し、ポータルマネージャを有効化。 6. **終了処理登録**: before_shmem_exit に ShutdownPostgres を登録。 7. **トランザクション開始**: ブートストラップ時を除き、初期化トランザクションを開始。 8. **認証・ユーザー設定**: 実行モード（ブートストラップ、バックグラウンドワーカー等）に応じて認証およびセッションユーザーIDの設定を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/bootstrap/bootstrap.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/bgworker.c
  • **呼び出し先**:
  • InitProcessPhase2, pgstat_beinit, SharedInvalBackendInit, ProcSignalInit, RegisterTimeout, StartupXLOG, RelationCacheInitialize, InitCatalogCache, InitPlanCache, EnablePortalManager, before_shmem_exit, StartTransactionCommand, InitializeSessionUserIdStandalone 等。

関連データ構造

  • PgBackendStatus
  • ProcArray
  • RelationCache
  • CatalogCache

失敗モードと境界条件

  • カタログキャッシュ構築前のシステムカタログアクセスはクラッシュの原因となる。
  • EXEC_BACKEND 定義環境（Windows等）では、認証情報のロード方法が異なるため注意が必要。
  • 初期化トランザクション中に失敗した場合、AbortTransaction が適切にクリーンアップを行う必要がある。

保守担当者向けメモ

  • 本関数はバックエンド起動のメインエントリーポイントであり、副作用が極めて多い。
  • タイムアウトハンドラ（StatementTimeoutHandler 等）はバックエンドの安定性に不可欠である。
  • AmAutoVacuumLauncherProcess の場合は、pgstat_bestart_final を実行して早期リターンする。
