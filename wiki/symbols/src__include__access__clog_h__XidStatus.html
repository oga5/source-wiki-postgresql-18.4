XidStatus

定義

  • 種別: typedef
  • 定義箇所: src/include/access/clog.h:25

目的

トランザクションのコミットまたはアボート状態を表現するための型定義です。

シグネチャまたは形状

typedef int XidStatus;

入力と出力

  • 入力: 整数値（トランザクションの状態をビット表現で保持）。
  • 出力: なし（型定義のため）。

前提条件と不変条件

  • 1トランザクションあたり2ビットを使用し、1バイトで4トランザクション分の状態を保持します。
  • 状態値として IN_PROGRESS、COMMITTED、ABORTED、SUB_COMMITTED のいずれかをとることが想定されています。

アルゴリズムまたは振る舞い

  • TransactionIdGetStatus() がディスク上のページからビットを抽出する際の戻り値として使用されます。
  • TransactionIdSetTreeStatus() や TransactionGroupUpdateXidStatus() 等の関数において、トランザクションの状態更新を指示するための引数として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/clog.c 内の複数の関数（TransactionIdSetStatusBit, TransactionGroupUpdateXidStatus 等）から引数として参照されています。
  • 呼び出し先: 型定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • TransactionId: トランザクションを一意に識別するID。
  • SLRU (Simple LRU): CLOGが状態を永続化するために利用するバッファ管理アルゴリズム。

失敗モードと境界条件

  • 該当するトランザクションIDが範囲外である場合や、CLOGページが破損している場合、状態取得時に不整合が発生する可能性があります。

保守担当者向けメモ

  • 本型は int として定義されていますが、実質的には列挙型としての役割を果たしています。状態の追加や変更を行う際は、clog.c 内のビット操作ロジックと整合性を取る必要があります。
