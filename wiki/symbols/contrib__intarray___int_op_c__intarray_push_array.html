intarray_push_array

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_op.c:336

目的

2つの整数配列を結合し、新しい配列として返すSQL呼び出し可能関数です。

シグネチャまたは形状

Datum intarray_push_array(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_ARRAYTYPE_P(0): 第1引数（ArrayType*）
  • PG_GETARG_ARRAYTYPE_P(1): 第2引数（ArrayType*）
  • **出力**:
  • PG_RETURN_POINTER(result): 結合された結果の ArrayType* を Datum 型として返却。

前提条件と不変条件

  • 入力として2つの有効な ArrayType が必要です。
  • 内部で intarray_concat_arrays を呼び出し、結果を生成します。

アルゴリズムまたは振る舞い

1. PG_GETARG_ARRAYTYPE_P を使用して、SQLから渡された2つの配列ポインタを取得します。 2. intarray_concat_arrays を呼び出し、配列の結合処理を実行します。 3. PG_FREE_IF_COPY を使用して、入力配列がコピーである場合にメモリを解放します。 4. 結合後の新しい配列を PG_RETURN_POINTER で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（PG_FUNCTION_INFO_V1(intarray_push_array) により登録）。
  • **呼び出し先**: intarray_concat_arrays（配列結合の内部ロジック）。

関連データ構造

  • ArrayType: PostgreSQLの配列データ型。

失敗モードと境界条件

  • 入力配列が不正な場合、intarray_concat_arrays 内部でエラーが発生する可能性があります。
  • メモリ不足時にはPostgreSQLのメモリ管理機構によりエラーがスローされます。

保守担当者向けメモ

  • 本関数は intarray_concat_arrays のラッパーとして機能しています。
  • メモリ管理のため、PG_FREE_IF_COPY を使用して入力配列の解放を確実に行っています。
