ExecEndTableFuncScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTableFuncscan.c:219

目的

TableFuncScanState ノードに関連付けられたリソース、特に tuplestore を解放し、クリーンアップを行う。

シグネチャまたは形状

void ExecEndTableFuncScan(TableFuncScanState *node)

入力と出力

  • **入力**: TableFuncScanState *node （実行中のテーブル関数スキャンノードの状態構造体）
  • **出力**: なし（副作用としてメモリリソースの解放を行う）

前提条件と不変条件

  • node は有効な TableFuncScanState 構造体を指している必要がある。
  • 実行終了時に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. node->tupstore が NULL でないかを確認する。 2. NULL でない場合、tuplestore_end(node->tupstore) を呼び出して tuplestore に割り当てられたリソースを解放する。 3. node->tupstore を NULL に設定し、二重解放を防ぐ。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:672 から呼び出されることが確認されている。
  • **呼び出し先**: tuplestore_end （tuplestore リソースの解放用）。

関連データ構造

  • TableFuncScanState: テーブル関数スキャンの実行状態を保持する構造体。tupstore フィールドを介して本関数と連携する。

失敗モードと境界条件

  • node->tupstore が既に NULL である場合、関数は何も行わず安全に終了する。

保守担当者向けメモ

  • 本関数はマテリアライズ型スキャンの終了処理の一部である。tuplestore 以外のリソース（opaque ポインタが指す解析エンジン固有の領域など）の解放については、本関数内では明示的に扱われていないため、必要に応じて確認すること。
