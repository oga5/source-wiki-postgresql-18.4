gtsquery_penalty

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_gist.c:138

目的

GiST インデックスにおいて、既存のインデックスエントリに新しい値を挿入する際のコスト（ペナルティ）を計算する。

シグネチャまたは形状

Datum gtsquery_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): 既存のインデックスエントリ（GISTENTRY）へのポインタ。
  • PG_GETARG_POINTER(1): 新しく挿入する値を含むエントリ（GISTENTRY）へのポインタ。
  • PG_GETARG_POINTER(2): ペナルティ値を格納するための float 型ポインタ。
  • **出力**:
  • PG_RETURN_POINTER(penalty): 計算されたペナルティ値が格納されたポインタを返す。

前提条件と不変条件

  • 入力される GISTENTRY のキーは TSQuerySign 型として解釈可能であること。
  • hemdist 関数が、2つの TSQuerySign 間のハミング距離を正しく算出できること。

アルゴリズムまたは振る舞い

1. 第1引数および第2引数の GISTENTRY から TSQuerySign 型のキーを抽出する。 2. hemdist 関数を呼び出し、2つのシグネチャ間のハミング距離を計算する。 3. 計算結果を第3引数の penalty ポインタに代入する。 4. 最終的にそのポインタを戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの挿入処理（PostgreSQL の GiST インデックスマネージャ）。
  • **呼び出し先**: hemdist（シグネチャ間の距離計算）。
  • 注: ソースコードの文脈上、具体的な呼び出し元の関数名は明示されていません。

関連データ構造

  • TSQuerySign: tsquery を圧縮したビットマップ表現。
  • GISTENTRY: GiST インデックスの各ノードにおけるエントリ構造体。

失敗モードと境界条件

  • 入力ポインタが NULL である場合、または GISTENTRY のキーが不正な形式である場合、セグメンテーションフォールトが発生する可能性がある。

保守担当者向けメモ

  • 本関数は GiST インデックスの木構造のバランスを最適化するためのヒューリスティックとして機能します。
  • ペナルティ値は hemdist に完全に依存しているため、検索効率を改善したい場合は hemdist のロジックを検討してください。
