HashAggSpill

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeAgg.c:332

目的

ハッシュ集約処理において、メモリ制限（work_mem）を超過した際に、集約状態をディスクへ退避（スピル）するための管理構造体です。

シグネチャまたは形状

typedef struct HashAggSpill
{
	int			npartitions;	/* パーティション数 */
	LogicalTape **partitions;	/* スピル用パーティションテープ */
	int64	   *ntuples;		/* 各パーティションのタプル数 */
	uint32		mask;			/* ハッシュ値からパーティションを特定するマスク */
	int			shift;			/* マスク後のシフト量 */
	hyperLogLogState *hll_card; /* 内容のカーディナリティ推定用 */
} HashAggSpill;

入力と出力

  • **入力**: ハッシュ集約実行中にメモリ不足が発生した際、AggState 内の hash_spills 配列として初期化・保持されます。
  • **出力**: LogicalTape を介してディスク上のテンポラリファイルへタプルを書き出し、後続の再読み込みを可能にします。

前提条件と不変条件

  • AggState の num_hashes に基づいてメモリが割り当てられます。
  • ハッシュ値からパーティションを決定するための mask と shift は、スピル開始時に一貫して計算される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: hashagg_spill_init により、パーティション数やテープセットが設定されます。 2. **退避**: hashagg_spill_tuple を通じて、入力タプルをハッシュ値に基づいて適切なパーティション（LogicalTape）へ書き出します。 3. **カーディナリティ推定**: hyperLogLogState を使用し、各パーティションのデータ量を推定して効率的な分割を支援します。 4. **終了**: hashagg_spill_finish により、すべてのパーティションの書き込みを完了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAgg.c 内のハッシュ集約実行ロジック（ExecAgg 等）。
  • **呼び出し先**: LogicalTape API（logtape.h）および hyperLogLog API（hyperloglog.h）。

関連データ構造

  • AggState: 集約ノードの実行状態を保持する親構造体。
  • LogicalTape: ディスクへの書き出しに使用されるテープ構造体。
  • hyperLogLogState: カーディナリティ推定用構造体。

失敗モードと境界条件

  • **メモリ不足**: work_mem を超えた場合に本構造体が使用されます。
  • **ディスク容量**: LogicalTape への書き込み時にディスク容量が不足した場合、エラーが発生する可能性があります。
  • **パーティション数**: npartitions が適切に設定されていない場合、ハッシュ衝突やパーティションの偏りが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は nodeAgg.c 内のハッシュ集約専用です。
  • hashagg_spill_init, hashagg_spill_tuple, hashagg_spill_finish の各関数がこの構造体を操作する主要なインターフェースです。
  • 並列クエリ実行時には、ワーカー間での統計情報共有の整合性に注意してください。
