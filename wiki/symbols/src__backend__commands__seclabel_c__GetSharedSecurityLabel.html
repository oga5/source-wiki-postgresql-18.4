GetSharedSecurityLabel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/seclabel.c:223

目的

共有データベースオブジェクトに対して、指定されたセキュリティプロバイダーに関連付けられたセキュリティラベルを取得する。

シグネチャまたは形状

char *GetSharedSecurityLabel(const ObjectAddress *object, const char *provider)

入力と出力

  • **入力**:
  • object: ラベルを取得する対象オブジェクトの ObjectAddress。
  • provider: ラベルを管理するセキュリティプロバイダー名（文字列）。
  • **出力**:
  • 成功時: 該当するセキュリティラベル文字列（palloc されたメモリ領域）。
  • 該当なし: NULL。

前提条件と不変条件

  • pg_shseclabel システムカタログに対する読み取り権限が必要。
  • AccessShareLock を使用してカタログをスキャンする。

アルゴリズムまたは振る舞い

1. pg_shseclabel システムカタログに対し、対象オブジェクトの OID、クラス OID、およびプロバイダー名をキーとしてスキャン条件（ScanKey）を初期化する。 2. table_open を用いて pg_shseclabel を AccessShareLock モードで開く。 3. systable_beginscan を実行し、インデックス SharedSecLabelObjectIndexId を使用して該当タプルを検索する。 4. systable_getnext でタプルを取得し、存在する場合は heap_getattr で label 列の値を抽出する。 5. 抽出した値を TextDatumGetCString で C 文字列に変換して返す。 6. スキャン終了後、systable_endscan および table_close を呼び出し、リソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は確認されていない。
  • **呼び出し先**: ScanKeyInit, table_open, systable_beginscan, systable_getnext, heap_getattr, RelationGetDescr, TextDatumGetCString, systable_endscan, table_close。

関連データ構造

  • ObjectAddress: 操作対象オブジェクトを特定するための構造体。
  • pg_shseclabel: 共有オブジェクトのセキュリティラベルを保持するシステムカタログ。

失敗モードと境界条件

  • 指定されたオブジェクトまたはプロバイダーに一致するラベルが存在しない場合、NULL を返す。
  • pg_shseclabel が開けない場合や、システムカタログの整合性に問題がある場合はエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は共有オブジェクト専用である。通常オブジェクトについては GetSecurityLabel を参照すること。
  • ラベルの取得には AccessShareLock を使用しており、同時実行性を考慮した設計となっている。
