gzip_pfree

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_gzip.c:298

目的

zlib ライブラリがメモリ解放を行う際に使用するカスタムメモリ解放関数として機能します。PostgreSQL のメモリ管理システム（pfree）を zlib に適合させるために提供されています。

シグネチャまたは形状

static void gzip_pfree(void *opaque, void *address);

入力と出力

  • opaque: zlib のコンテキストポインタ（本関数内では使用されません）。
  • address: 解放対象となるメモリ領域のポインタ。
  • 出力: なし。

前提条件と不変条件

  • address は、PostgreSQL の palloc 等で割り当てられた有効なメモリ領域である必要があります。
  • zlib の z_stream 構造体の zfree フィールドに本関数が設定されていることが前提です。

アルゴリズムまたは振る舞い

受け取った address を PostgreSQL の pfree() 関数に渡し、メモリを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: zlib ライブラリ内部（z_stream の zfree を介して呼び出されます）。
  • 呼び出し先: pfree（PostgreSQL のメモリ解放関数）。
  • 補足: src/backend/backup/basebackup_gzip.c:123 にて、z_stream 構造体の zfree フィールドに本関数が代入されています。

関連データ構造

  • z_stream（zlib.h）: 圧縮状態を保持する構造体。本関数はこの構造体のメモリ管理インターフェースとして利用されます。

失敗モードと境界条件

  • address が NULL の場合、pfree の仕様に従い何も行われません。
  • 無効なポインタを渡した場合の挙動は pfree に依存します。

保守担当者向けメモ

  • 本関数は zlib のメモリ管理を PostgreSQL のメモリコンテキストに統合するためのラッパーです。
  • 同名の関数が src/fe_utils/astreamer_gzip.c にも存在しますが、これらは別々のコンパイル単位で定義されています。
