ExecCreateTableAs

定義

  • 種別: function
  • 定義箇所: src/backend/commands/createas.c:222

目的

CREATE TABLE AS (CTAS) および CREATE MATERIALIZED VIEW コマンドの実行を制御し、クエリ結果を新規テーブルへ永続化する。

シグネチャまたは形状

ObjectAddress ExecCreateTableAs(ParseState *pstate, CreateTableAsStmt *stmt,
                                 ParamListInfo params, QueryEnvironment *queryEnv,
                                 QueryCompletion *qc)

入力と出力

  • **入力**:
  • pstate: パーサ状態。
  • stmt: CTAS 文の構文木。
  • params: 実行時パラメータ。
  • queryEnv: クエリ環境。
  • qc: 完了状態を記録する構造体。
  • **出力**: 作成されたリレーションの ObjectAddress を返す。対象が存在する場合は InvalidObjectAddress を返す。

前提条件と不変条件

  • CreateTableAsRelExists により、対象リレーションが既に存在しないことが確認されている必要がある。
  • query->commandType は CMD_SELECT であるか、CMD_UTILITY (かつ ExecuteStmt) である必要がある。
  • マテリアライズドビューの場合、into->skipData は内部的に true に強制される。

アルゴリズムまたは振る舞い

1. **初期化**: CreateIntoRelDestReceiver を用いて、結果をテーブルに書き込むための DestReceiver を作成する。 2. **分岐処理**:

  • EXECUTE 文の場合: ExecuteQuery を呼び出し、結果を返して終了する。
  • WITH NO DATA 指定時: create_ctas_nodata を呼び出し、プランナやエグゼキュータを介さずにテーブル定義のみを作成する。マテリアライズドビューの場合は RefreshMatViewByOid を実行する。
  • 通常の SELECT 時:
  • QueryRewrite でクエリを書き換える。
  • pg_plan_query でプランを作成する。
  • CreateQueryDesc で実行記述子を作成し、ExecutorStart / ExecutorRun / ExecutorFinish を経由してデータをテーブルへ挿入する。

    • 3. **終了処理**: スナップショットの解放および ExecutorEnd によるクリーンアップを行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c (直接の証拠あり)。
  • **呼び出し先**: CreateTableAsRelExists, CreateIntoRelDestReceiver, ExecuteQuery, create_ctas_nodata, RefreshMatViewByOid, QueryRewrite, pg_plan_query, ExecutorStart, ExecutorRun, ExecutorFinish, ExecutorEnd 等。

関連データ構造

  • IntoClause: CTAS のターゲット設定。
  • DestReceiver / DR_intorel: クエリ結果をテーブルへ流し込むための受信器。
  • QueryDesc: クエリ実行の制御構造。

失敗モードと境界条件

  • 対象リレーションが既に存在する場合、InvalidObjectAddress を返し処理を中断する。
  • 書き換え後のクエリが SELECT 以外（または複数）になった場合、elog(ERROR) を発生させる。
  • WITH NO DATA 指定時はプランナをバイパスするため、依存関係の不整合を回避する。

保守担当者向けメモ

  • CTAS は歴史的に SELECT の亜種として実装されているため、QueryCompletion への行数報告などのレガシーな名残が含まれている。
  • intorel_startup で行レベルセキュリティ (RLS) のエラーチェックが行われるため、拡張時は注意が必要。
