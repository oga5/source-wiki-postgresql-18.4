pq_sendint16

定義

  • 種別: static inline function
  • 定義箇所: src/include/libpq/pqformat.h:135

目的

16ビット整数（uint16）をネットワークバイトオーダーに変換し、通信バッファ StringInfo に追加する。

シグネチャまたは形状

static inline void pq_sendint16(StringInfo buf, uint16 i)

入力と出力

  • **入力**:
  • buf: データを書き込む対象の StringInfo 構造体へのポインタ。
  • i: 書き込む対象の 16ビット符号なし整数値。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • buf は初期化済みであること。
  • 書き込み後にバッファの容量が不足する場合、enlargeStringInfo によって自動的に拡張される。

アルゴリズムまたは振る舞い

1. enlargeStringInfo(buf, sizeof(uint16)) を呼び出し、バッファに 2バイト分の空き容量を確保する。 2. pq_writeint16(buf, i) を呼び出し、ネットワークバイトオーダー（ビッグエンディアン）で値をバッファに書き込み、バッファのカーソル位置を進める。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printsimple.c
  • src/backend/access/common/printtup.c
  • src/backend/backup/basebackup_copy.c
  • src/backend/commands/copyfromparse.c
  • src/backend/commands/copyto.c
  • **呼び出し先**:
  • enlargeStringInfo
  • pq_writeint16

関連データ構造

  • StringInfo: 通信データのバッファを管理する構造体（lib/stringinfo.h で定義）。

失敗モードと境界条件

  • メモリ不足時: enlargeStringInfo 内部でメモリ確保に失敗した場合、システムはエラーをスローする可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、頻繁な呼び出しによるオーバーヘッドを抑える設計となっている。
  • バッファの動的拡張が不要な場合は、より低レイヤーの pq_writeint16 を直接使用することが推奨される。
