AtPrepare_PgStat_Relations

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:689

目的

トランザクションの準備（PREPARE TRANSACTION）時に、リレーションの統計情報を2フェーズコミット（2PC）用の永続化レコードとして登録します。

シグネチャまたは形状

void AtPrepare_PgStat_Relations(PgStat_SubXactStatus *xact_state)

入力と出力

  • **入力**: xact_state (現在のトランザクションにおけるリレーション統計状態を保持する PgStat_SubXactStatus へのポインタ)
  • **出力**: なし。副作用として RegisterTwoPhaseRecord を通じて統計データが2PC状態管理へ登録されます。

前提条件と不変条件

  • xact_state 内の各 PgStat_TableXactStatus について、nest_level == 1 かつ upper == NULL であること（トップレベルトランザクションの統計のみが対象）。
  • tabstat->trans == trans であること。

アルゴリズムまたは振る舞い

1. xact_state が保持するリレーション統計のリストを走査します。 2. 各リレーションの挿入・更新・削除回数、および TRUNCATE 関連のフラグを TwoPhasePgStatRecord 構造体にコピーします。 3. リレーションの識別子（id）と共有状態（shared）をレコードに含めます。 4. RegisterTwoPhaseRecord を呼び出し、TWOPHASE_RM_PGSTAT_ID を用いて統計情報を永続化ストレージへ書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_xact.c 内のトランザクション準備処理。
  • **呼び出し先**: RegisterTwoPhaseRecord（2PCレコード登録用API）。

関連データ構造

  • PgStat_SubXactStatus: トランザクション内のリレーション統計状態を管理する構造体。
  • PgStat_TableXactStatus: 個別のリレーションに対するトランザクションレベルの統計。
  • TwoPhasePgStatRecord: 2PCで永続化される統計情報のデータ構造。

失敗モードと境界条件

  • Assert 文により、トップレベル以外のトランザクション状態が渡された場合は異常終了します。
  • 2PCレコードの登録に失敗した場合、トランザクションの準備処理自体が失敗します。

保守担当者向けメモ

  • 本関数はトップレベルトランザクションの統計のみを対象としています。サブトランザクションの統計は、コミット時に親へマージされるため、ここでは考慮されません。
  • TRUNCATE 実行時の統計リセットに関連するカウンタ（_pre_truncdrop）も永続化対象に含まれている点に注意してください。
