VariableStatData

定義

  • 種別: struct
  • 定義箇所: src/include/utils/selfuncs.h:87

目的

クエリプランナが選択率（Selectivity）を推定する際、対象となる列や式に関する統計情報（ヒストグラムやMCVなど）を保持・管理するために使用されます。

シグネチャまたは形状

typedef struct VariableStatData
{
    Node       *var;            /* 対象のVarまたは式ツリー */
    RelOptInfo *rel;            /* リレーション情報、特定不可の場合はNULL */
    HeapTuple   statsTuple;     /* pg_statisticタプル、存在しない場合はNULL */
    void        (*freefunc) (HeapTuple tuple); /* statsTuple解放用関数 */
    Oid         vartype;        /* 式の公開型 */
    Oid         atttype;        /* 実際の型（リラベル剥離後） */
    int32       atttypmod;      /* 実際のtypmod（リラベル剥離後） */
    bool        isunique;       /* 一意インデックス、DISTINCT、GROUP-BYとの一致 */
    bool        acl_ok;         /* テーブル/列の全行に対するSELECT権限の有無 */
} VariableStatData;

入力と出力

  • **入力**: プランナが examine_variable() を呼び出すことで、対象の列や式に基づき本構造体が初期化されます。
  • **出力**: 選択率推定関数（scalararraysel 等）が本構造体内の統計情報を参照し、推定結果を算出します。

前提条件と不変条件

  • statsTuple が NULL でない場合、呼び出し元は処理終了後に必ず freefunc を使用してメモリを解放する必要があります。
  • acl_ok は、ユーザーが対象データに対する適切な権限を持っているかを示します。

アルゴリズムまたは振る舞い

1. **初期化**: examine_variable() により、統計情報が収集され構造体に格納されます。 2. **推定**: 演算子ごとの選択率算出関数が、構造体内の統計情報と定数値を比較して確率を算出します。 3. **解放**: 統計情報の利用終了後、ReleaseVariableStats() マクロを介して statsTuple を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_int_selfuncs.c、src/backend/tsearch/ts_selfuncs.c、src/backend/utils/adt/array_selfuncs.c、src/backend/utils/adt/like_support.c、src/backend/utils/adt/multirangetypes_selfuncs.c 等の各選択率推定ロジック。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、freefunc フィールドに関数ポインタを保持します。

関連データ構造

  • RelOptInfo: リレーションのプランニング情報を保持。
  • HeapTuple: pg_statistic から取得した統計タプル。
  • Node: 対象の式ツリー。

失敗モードと境界条件

  • statsTuple が NULL の場合、統計情報が利用できないことを意味し、デフォルトの選択率（DEFAULT_EQ_SEL 等）が使用されます。
  • メモリ解放を怠るとメモリリークが発生します。

保守担当者向けメモ

  • 本構造体を使用する際は、必ず ReleaseVariableStats() を使用して statsTuple の解放を確実に行ってください。
  • 統計情報が不足している場合、インデックススキャンが優先的に選択されるよう、デフォルト値は意図的に小さく設定されています。
