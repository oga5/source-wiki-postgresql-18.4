pg_cancel

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:40

目的

PostgreSQLサーバーに対して実行中のクエリをキャンセルするために必要な接続情報およびリクエストパケットを保持する構造体です。

シグネチャまたは形状

struct pg_cancel
{
    SockAddr    raddr;
    int         be_pid;
    int         pgtcp_user_timeout;
    int         keepalives;
    int         keepalives_idle;
    int         keepalives_interval;
    int         keepalives_count;
    int32       cancel_pkt_len;
    char        cancel_req[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。libpq の内部関数によって、既存の PGconn から抽出された情報が格納されます。

前提条件と不変条件

  • cancel_pkt_len はネットワークバイトオーダーで保持される必要があります。
  • cancel_req は CancelRequestPacket を格納するための可変長配列として機能します。

アルゴリズムまたは振る舞い

本構造体は、キャンセル要求を送信するための「バックアップデータ」として機能します。接続状態とは独立して、サーバーのPIDやキャンセルキー、およびTCP接続設定（keepalive等）を保持することで、メインの接続がブロックされている場合でもキャンセル要求を送信可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: libpq-fe.h にて typedef struct pg_cancel PGcancel; として公開されており、PQgetCancel 等の関数を通じてインスタンス化されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • PGcancel (本構造体の別名)
  • SockAddr (リモートアドレス保持用)
  • CancelRequestPacket (送信されるパケットの形式)

失敗モードと境界条件

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のメモリ確保は動的に行われる必要があります。
  • ネットワークバイトオーダーへの変換が正しく行われない場合、サーバー側でキャンセル要求が正しく解釈されません。

保守担当者向けメモ

  • 本構造体は libpq の内部的なキャンセル処理の基盤です。フィールドを追加する際は、シグナル安全性（malloc 等を避ける設計）を維持する必要があるか検討してください。
  • Windows環境におけるソケットのEOF検知に関連するフィールドが必要な場合は、プラットフォーム固有の定義に注意してください。
