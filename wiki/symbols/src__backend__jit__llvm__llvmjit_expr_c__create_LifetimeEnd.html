create_LifetimeEnd

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_expr.c:3090

目的

LLVM の組み込み関数 llvm.lifetime.end.p0 を表す LLVMValueRef を取得または生成します。この関数は、メモリ領域の有効期間が終了したことを LLVM の最適化パスに通知するために使用されます。

シグネチャまたは形状

LLVMValueRef create_LifetimeEnd(LLVMModuleRef mod);

入力と出力

  • **入力**: mod (LLVM モジュールへの参照)
  • **出力**: llvm.lifetime.end.p0 に対応する LLVMValueRef

前提条件と不変条件

  • 指定されたモジュール mod が有効であること。
  • llvm.lifetime.end.p0 が LLVM の組み込み関数として認識されていること。

アルゴリズムまたは振る舞い

1. LLVMGetNamedFunction を使用して、モジュール内に既に llvm.lifetime.end.p0 が存在するか確認し、存在すればそれを返します。 2. 存在しない場合、以下のシグネチャで関数を新規作成します。

  • 引数: i64 (サイズ), i8* (ポインタ)
  • 戻り値: void

    • 3. 作成した関数に LLVMCCallConv 呼び出し規約を適用します。 4. Assert により、作成された関数が LLVM 組み込み関数として正しく識別されることを検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/jit/llvm/llvmjit_expr.c 内の関数（例: 3040 行目付近）から呼び出されています。
  • **呼び出し先**: LLVM C API (LLVMGetNamedFunction, LLVMGetModuleContext, LLVMInt64TypeInContext, LLVMFunctionType, LLVMAddFunction, LLVMSetFunctionCallConv, LLVMGetIntrinsicID) を呼び出します。

関連データ構造

  • LLVMModuleRef: LLVM IR を保持するコンテナ。
  • LLVMValueRef: 生成された関数オブジェクト。

失敗モードと境界条件

  • モジュールコンテキストが不正な場合、LLVM API 呼び出しによりプロセスが異常終了する可能性があります。
  • LLVMGetIntrinsicID が 0 を返した場合、Assert によりアサーション失敗が発生します。

保守担当者向けメモ

  • 本関数は llvm.lifetime.end イントリンシックを生成するユーティリティです。LLVM のバージョンアップに伴い、イントリンシックの仕様や名称が変更された場合はここを修正する必要があります。
  • l_ptr マクロ（llvmjit_emit.h 由来）を使用してポインタ型を構築しています。
