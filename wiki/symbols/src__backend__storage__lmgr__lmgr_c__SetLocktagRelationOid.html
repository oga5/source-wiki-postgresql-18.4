SetLocktagRelationOid

定義

  • 種別: static inline function
  • 定義箇所: src/backend/storage/lmgr/lmgr.c:87

目的

リレーションに対するロック操作を行うための LOCKTAG 構造体を、指定された OID を用いて初期化する。

シグネチャまたは形状

static inline void SetLocktagRelationOid(LOCKTAG *tag, Oid relid)

入力と出力

  • **入力**:
  • tag: 初期化対象となる LOCKTAG 構造体へのポインタ。
  • relid: ロック対象のリレーション OID。
  • **出力**: なし（tag が指すメモリ領域が更新される）。

前提条件と不変条件

  • relid が共有リレーション（IsSharedRelation(relid) が真）である場合、データベース ID は InvalidOid としてタグに設定される。
  • 共有リレーションでない場合、現在のセッションのデータベース ID（MyDatabaseId）がタグに設定される。

アルゴリズムまたは振る舞い

1. IsSharedRelation(relid) を用いて、対象リレーションが共有されているか判定する。 2. 判定結果に基づき、データベース ID を InvalidOid または MyDatabaseId に決定する。 3. SET_LOCKTAG_RELATION マクロを使用して、決定したデータベース ID と relid を tag に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/lmgr.c 内の LockRelationOid, ConditionalLockRelationOid, UnlockRelationOid 等の関数から呼び出されている。
  • **呼び出し先**:
  • IsSharedRelation (マクロ/関数)
  • SET_LOCKTAG_RELATION (マクロ)

関連データ構造

  • LOCKTAG: ロック対象を一意に識別するための構造体。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。引数の tag が NULL ポインタである場合、未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は lmgr.c 内で頻繁に利用されるユーティリティである。リレーションロックのタグ構築ロジックを一元管理しているため、タグの構成に変更を加える際は本関数を確認すること。
