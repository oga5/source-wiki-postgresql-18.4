copy_special_str

定義

  • 種別: static inline function
  • 定義箇所: src/common/ryu_common.h:94

目的

浮動小数点数の特殊値（NaN、Infinity、ゼロ）を文字列バッファへ書き込み、書き込んだバイト数を返します。

シグネチャまたは形状

static inline int copy_special_str(char *const result, const bool sign, const bool exponent, const bool mantissa)

入力と出力

  • **入力**:
  • result: 書き込み先の文字バッファ。
  • sign: 負の値であれば true。
  • exponent: 指数部が最大値（Infinityの判定条件）であれば true。
  • mantissa: 仮数部が非ゼロ（NaNの判定条件）であれば true。
  • **出力**:
  • int: バッファに書き込まれた文字数。

前提条件と不変条件

  • result は、少なくとも 9 バイト以上の書き込み可能な領域を指している必要があります（"Infinity" の 8 バイト + 符号用 1 バイト）。

アルゴリズムまたは振る舞い

1. mantissa が真の場合、"NaN" を書き込み 3 を返します。 2. mantissa が偽の場合:

  • sign が真であれば、result[0] に '-' を書き込みます。
  • exponent が真であれば、result + sign の位置に "Infinity" を書き込み、sign + 8 を返します。
  • exponent が偽（ゼロ）であれば、result[sign] に '0' を書き込み、sign + 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/d2s.c:1031
  • src/common/f2s.c:758
  • **呼び出し先**:
  • memcpy（標準ライブラリ関数）

関連データ構造

  • 特になし。

失敗モードと境界条件

  • result バッファが不足している場合、メモリ破壊が発生します。
  • mantissa が真の場合、sign や exponent の値に関わらず常に "NaN" が優先されます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーインクルード時に展開されます。
  • 浮動小数点変換の最終段階で、数値以外の特殊ケースを処理するために使用されます。
