RVROption

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/namespace.h:70

目的

RangeVarGetRelidExtended 関数において、リレーションの検索やロック取得時の挙動を制御するためのビットフラグを提供します。

シグネチャまたは形状

typedef enum RVROption
{
    RVR_MISSING_OK = 1 << 0,
    RVR_NOWAIT = 1 << 1,
    RVR_SKIP_LOCKED = 1 << 2,
} RVROption;

入力と出力

  • 本シンボルは列挙型であり、関数呼び出し時の引数としてビットマスク形式で渡されます。
  • 各フラグは、リレーションが存在しない場合やロックが取得できない場合の関数の戻り値およびエラー発生の有無を決定します。

前提条件と不変条件

  • 各値はビット演算（|）による組み合わせが可能です。
  • RVR_NOWAIT と RVR_SKIP_LOCKED の同時指定時の優先順位や排他性については、実装側の RangeVarGetRelidExtended のロジックに依存します。

アルゴリズムまたは振る舞い

  • RVR_MISSING_OK: リレーションが存在しない場合にエラーを発生させず、適切な値（通常は InvalidOid）を返します。
  • RVR_NOWAIT: リレーションのロックが即座に取得できない場合にエラーを発生させます。
  • RVR_SKIP_LOCKED: リレーションのロックが即座に取得できない場合に、そのリレーションをスキップします。

呼び出し元と呼び出し先

  • 呼び出し元: パーサーやコマンド実行部が RangeVarGetRelidExtended を呼び出す際に使用します。
  • 呼び出し先: 本シンボル自体はデータ定義であり、呼び出し先はありません。

関連データ構造

  • RangeVar: リレーション名を保持する構造体。
  • RangeVarGetRelidExtended: 本フラグを利用してリレーションの OID を解決する関数。

失敗モードと境界条件

  • フラグが指定されない場合、リレーションが存在しないとエラーが発生し、ロックが取得できない場合は待機（ブロック）が発生する可能性があります。
  • RVR_MISSING_OK を指定しない場合、対象リレーションがカタログ内に存在しないとエラーが発生します。

保守担当者向けメモ

  • RangeVarGetRelid は本フラグを使用しない簡易版であるため、より詳細な制御が必要な場合は RangeVarGetRelidExtended を使用してください。
  • 本フラグの追加や変更を行う際は、src/backend/catalog/namespace.c 内の RangeVarGetRelidExtended 実装への影響を確認してください。
