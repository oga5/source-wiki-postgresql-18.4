pg_snapshot

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/xid8funcs.c:54

目的

64ビットトランザクションID（FullTransactionId）を用いたトランザクションスナップショットを保持し、可視性判定や外部表現（シリアライズ）を可能にするための構造体です。

シグネチャまたは形状

typedef struct {
    int32       __varsz;
    uint32      nxip;
    FullTransactionId xmin;
    FullTransactionId xmax;
    FullTransactionId xip[FLEXIBLE_ARRAY_MEMBER];
} pg_snapshot;

入力と出力

  • 本構造体は可変長データとして扱われます。
  • nxip は xip 配列内の要素数を示します。
  • xip 配列には、xmin <= xip[i] < xmax の範囲にある進行中のトランザクションIDが格納されます。

前提条件と不変条件

  • __varsz は内部的なヘッダーであり、直接操作してはなりません。
  • xip 配列は FLEXIBLE_ARRAY_MEMBER を使用しており、メモリ確保時には nxip に基づく適切なサイズ計算が必要です。
  • xip 配列内のトランザクションIDは、可視性判定のためにソートされていることが期待されます（sort_snapshot 関数による）。

アルゴリズムまたは振る舞い

  • is_visible_fxid 関数において、対象の xid8 がスナップショットの範囲内にあるかを確認し、範囲内の場合は xip 配列を探索して進行中かどうかを判定するために使用されます。
  • sort_snapshot 関数により、格納された進行中トランザクションIDのソートと重複排除が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: sort_snapshot, is_visible_fxid, parse_snapshot 等の関数が本構造体を引数として受け取ります。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数呼び出しの対象ではありません。

関連データ構造

  • FullTransactionId: 64ビットのトランザクションID。
  • StringInfo: pg_snapshot の作成やシリアライズ処理において補助的に使用されます。

失敗モードと境界条件

  • nxip の値が大きすぎる場合、メモリ割り当て制限（MaxAllocSize）を超える可能性があります。
  • xip 配列のアクセス時には、nxip による境界チェックが必須です。

保守担当者向けメモ

  • 本構造体はトップレベルのトランザクションIDのみを対象としています。サブトランザクションIDは含まれません。
  • xip 配列のサイズ計算には offsetof(pg_snapshot, xip) を使用してください。
  • 従来の32ビットXIDベースのスナップショットから64ビット版への移行を目的としています。
