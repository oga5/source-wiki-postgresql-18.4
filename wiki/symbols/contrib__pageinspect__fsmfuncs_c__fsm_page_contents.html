fsm_page_contents

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/fsmfuncs.c:34

目的

PostgreSQLの空き領域マップ（FSM）ページの内部構造を可視化し、デバッグや低レベルなデータ解析のためにテキスト形式で出力する。

シグネチャまたは形状

Datum fsm_page_contents(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る bytea 型のrawページデータ。
  • **出力**: text 型。FSMの各ノード値および fp_next_slot を含むテキストデータ。ページが空の場合は NULL を返す。

前提条件と不変条件

  • 実行者はスーパーユーザーである必要がある。
  • 入力されたデータは有効なPostgreSQLページ形式であること。

アルゴリズムまたは振る舞い

1. **権限確認**: superuser() を呼び出し、権限がない場合はエラーを発生させる。 2. **ページ変換**: get_page_from_raw() を使用して bytea を Page 構造に変換する。 3. **空判定**: PageIsNew() が真であれば NULL を返して終了する。 4. **構造体解釈**: PageGetContents() を介してページ内容を FSMPage 構造体としてキャストする。 5. **データ抽出**: NodesPerPage 分のループを実行し、fp_nodes 配列内の非ゼロ値を StringInfo バッファに追記する。 6. **メタデータ追加**: fp_next_slot の値をバッファに追記する。 7. **返却**: 構築された文字列を text 型に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • superuser()
  • get_page_from_raw()
  • PageIsNew()
  • PageGetContents()
  • initStringInfo()
  • appendStringInfo()
  • cstring_to_text_with_len()

関連データ構造

  • FSMPage: FSMページの物理レイアウトを定義する構造体。
  • StringInfoData: 文字列構築用バッファ。
  • NodesPerPage: ページ内のノード数。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーとなる。
  • **空ページ**: PageIsNew の場合、NULL を返す。
  • **ノード値**: fp_nodes[i] が0のノードは出力から除外される。

保守担当者向けメモ

  • FSMの内部構造はPostgreSQLのバージョン間で変更される可能性があるため、本関数はデバッグ用途に限定すること。
  • get_page_from_raw は contrib/pageinspect/rawpage.c で定義されている。
