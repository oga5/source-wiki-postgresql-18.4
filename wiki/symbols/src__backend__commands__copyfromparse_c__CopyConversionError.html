CopyConversionError

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfromparse.c:532

目的

COPY FROM コマンド実行中に発生したエンコーディング変換エラーを報告し、適切なエラーメッセージを生成する。

シグネチャまたは形状

void CopyConversionError(CopyFromState cstate)

入力と出力

  • **入力**: CopyFromState cstate（COPY 操作の現在の状態を保持する構造体）
  • **出力**: なし（エラー発生時は elog(ERROR, ...) により処理を中断する）

前提条件と不変条件

  • cstate->raw_buf_len > 0 であること。
  • cstate->input_reached_error が真であること。

アルゴリズムまたは振る舞い

1. cstate->need_transcoding が偽の場合：

  • エンコーディング変換が不要なケース。report_invalid_encoding を呼び出し、無効な文字を含むバッファ範囲を特定してエラーを報告する。

    • 2. cstate->need_transcoding が真の場合：

  • エンコーディング変換が必要なケース。pg_do_encoding_conversion_buf を noError=false で再実行し、変換ルーチン側で詳細なエラーをスローさせる。
  • 変換ルーチンがエラーをスローしなかった場合、elog(ERROR, ...) で異常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfromparse.c 内の解析処理（直接の呼び出し箇所として line 533 および line 681 が確認されている）。
  • **呼び出し先**: report_invalid_encoding、pg_do_encoding_conversion_buf、elog。

関連データ構造

  • CopyFromState: COPY FROM の実行状態を管理する構造体（commands/copyfrom_internal.h で定義）。

失敗モードと境界条件

  • 変換ルーチンがエラーを報告せずに終了した場合、elog(ERROR) により強制的にエラー状態へ遷移する。
  • raw_buf 内の不正な文字位置が raw_buf_index または input_buf_len によって正しく特定できない場合、誤ったエラー報告が行われる可能性がある。

保守担当者向けメモ

  • 本関数は COPY 解析パイプラインにおけるエラーハンドリングの最終防衛線である。
  • need_transcoding の有無でエラー報告の経路が異なるため、修正時は両方のパスを確認すること。
