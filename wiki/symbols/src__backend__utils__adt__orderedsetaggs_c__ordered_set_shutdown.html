ordered_set_shutdown

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/orderedsetaggs.c:338

目的

順序集合集約関数の実行終了時に、割り当てられたリソース（ソート用の一時ファイルやメモリ）を解放し、状態をクリーンアップする。

シグネチャまたは形状

static void ordered_set_shutdown(Datum arg)

入力と出力

  • **入力 (arg)**: OSAPerGroupState 構造体へのポインタを保持する Datum 型の値。
  • **出力**: なし（void）。

前提条件と不変条件

  • arg は有効な OSAPerGroupState 構造体を指している必要がある。
  • 本関数は集約終了時のコールバックとして登録されることを想定している。

アルゴリズムまたは振る舞い

1. arg を OSAPerGroupState 型にキャストする。 2. osastate->sortstate が存在する場合、tuplesort_end を呼び出してソート処理に関連するメモリおよび一時ファイルを解放し、ポインタを NULL に設定する。 3. osastate->qstate->tupslot が存在する場合、ExecClearTuple を呼び出してタプルスロットをクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAgg.c（集約関数の実行制御プロセスからコールバックとして呼び出される）。
  • **呼び出し先**:
  • tuplesort_end（src/include/utils/tuplesort.h）
  • ExecClearTuple（src/include/executor/executor.h）

関連データ構造

  • OSAPerGroupState: グループ単位の実行状態を保持する構造体。
  • tuplesort: ソート処理を行うためのエンジン。

失敗モードと境界条件

  • osastate->sortstate が NULL の場合、tuplesort_end は呼び出されない（安全にスキップされる）。
  • osastate->qstate->tupslot が NULL の場合、ExecClearTuple は呼び出されない。

保守担当者向けメモ

  • 本関数は AggRegisterCallback を介して登録されるクリーンアップ処理の要である。メモリリークや一時ファイルの残留を防ぐため、OSAPerGroupState のライフサイクル終了時には必ず本関数が実行されることを保証する必要がある。
