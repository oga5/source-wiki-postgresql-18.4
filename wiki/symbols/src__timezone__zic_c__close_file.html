close_file

定義

  • 種別: function
  • 定義箇所: src/timezone/zic.c:526

目的

ファイルストリームを安全にクローズし、I/Oエラーまたはクローズ失敗が発生した際にエラーメッセージを出力してプログラムを終了させる。

シグネチャまたは形状

static void close_file(FILE *stream, char const *dir, char const *name)

入力と出力

  • stream: クローズ対象の FILE ポインタ。
  • dir: エラーメッセージに含めるディレクトリ名（NULL可）。
  • name: エラーメッセージに含めるファイル名（NULL可）。
  • 出力: なし（成功時）。エラー時は stderr にメッセージを出力し、exit(EXIT_FAILURE) を実行する。

前提条件と不変条件

  • stream は有効な FILE ポインタであること。
  • dir および name は、エラーメッセージの構築に使用される。これらが NULL の場合、メッセージには含まれない。

アルゴリズムまたは振る舞い

1. ferror(stream) を呼び出し、ストリームにエラーフラグが立っているか確認する。 2. エラーがない場合、fclose(stream) を実行し、戻り値が 0 以外（失敗）であれば strerror(errno) でエラー文字列を取得する。 3. 上記のいずれかでエラーが検出された場合、progname、dir、name、およびエラー内容を整形して stderr に出力する。 4. エラー発生時は exit(EXIT_FAILURE) を呼び出し、プロセスを異常終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/timezone/zic.c 内の複数箇所（main 関数やファイル処理を行う各関数から、ファイル操作の終了時に呼び出される）。
  • 呼び出し先: ferror, fclose, strerror, fprintf, exit（標準ライブラリ関数）。

関連データ構造

  • FILE: 標準入出力ライブラリのストリーム構造体。

失敗モードと境界条件

  • ferror が真を返す場合、または fclose が失敗した場合にエラー処理がトリガーされる。
  • dir や name が NULL の場合、エラーメッセージの該当部分は空文字列として処理される。

保守担当者向けメモ

  • 本関数は zic ユーティリティにおける標準的なファイルクローズ処理をカプセル化している。
  • エラーメッセージの出力形式は progname に依存するため、呼び出し側で progname が適切に設定されていることを前提としている。
