SetShellResultVariables

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:515

目的

シェルコマンドの実行結果（終了ステータス）を psql の内部変数として設定し、後続の処理で参照可能にする。

シグネチャまたは形状

void SetShellResultVariables(int wait_result)

入力と出力

  • **入力**: wait_result (int型): pclose 等から返されるシェルコマンドの終了ステータス。
  • **出力**: なし（副作用として pset.vars を更新）。

前提条件と不変条件

  • pset.vars が初期化されていること。
  • wait_result_to_exit_code 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. wait_result が 0 の場合は SHELL_ERROR 変数に "false" を、それ以外の場合は "true" を設定する。 2. wait_result_to_exit_code を用いて wait_result をシステム依存の終了コードに変換する。 3. 変換後の終了コードを文字列に変換し、SHELL_EXIT_CODE 変数に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c (複数箇所)
  • src/bin/psql/common.c (複数箇所)
  • src/bin/psql/copy.c
  • **呼び出し先**:
  • SetVariable (psql変数管理)
  • wait_result_to_exit_code (終了ステータス変換)

関連データ構造

  • pset.vars: psql のグローバル設定および変数を保持する構造体。

失敗モードと境界条件

  • SetVariable が失敗した場合の挙動については、本関数のソースコード内では明示的にハンドリングされていない。

保守担当者向けメモ

  • 本関数は psql 内でシェルコマンドを実行した後の状態管理を行うため、pclose の戻り値を扱う箇所で一貫して使用すること。
