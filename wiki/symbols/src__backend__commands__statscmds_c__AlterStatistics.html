AlterStatistics

定義

  • 種別: function
  • 定義箇所: src/backend/commands/statscmds.c:637

目的

既存の拡張統計情報オブジェクトの属性（主に統計ターゲット stxstattarget）を更新する。

シグネチャまたは形状

ObjectAddress AlterStatistics(AlterStatsStmt *stmt)

入力と出力

  • **入力**: AlterStatsStmt *stmt（ALTER STATISTICS 文の解析ツリー）
  • **出力**: ObjectAddress（更新された統計オブジェクトの識別子）

前提条件と不変条件

  • 呼び出し元は、対象の統計オブジェクトに対する所有権を持っている必要がある。
  • 統計ターゲット（stxstattarget）を指定する場合、値は 0 以上 MAX_STATISTICS_TARGET 以下である必要がある。これを超える場合は警告とともに上限値に切り詰められ、負の値はエラーとなる。

アルゴリズムまたは振る舞い

1. stmt から統計ターゲットの値を検証し、設定値を決定する。 2. get_statistics_object_oid を使用して対象統計オブジェクトの OID を取得する。存在しない場合、IF EXISTS 指定があれば NOTICE を出力して終了する。 3. pg_statistic_ext システムカタログを RowExclusiveLock で開く。 4. SearchSysCache1 で対象のタプルを検索し、所有権を確認する。 5. heap_modify_tuple を用いて、stxstattarget カラムを新しい値（またはデフォルト値）で更新する。 6. CatalogTupleUpdate でカタログを更新し、InvokeObjectPostAlterHook を呼び出す。 7. 最後にリソース（タプル、リレーション）を解放し、更新されたオブジェクトの ObjectAddress を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:1909
  • **呼び出し先**: get_statistics_object_oid, table_open, SearchSysCache1, object_ownercheck, heap_modify_tuple, CatalogTupleUpdate, InvokeObjectPostAlterHook, table_close 等。

関連データ構造

  • AlterStatsStmt: ALTER STATISTICS 文の構文解析結果を保持するノード。
  • pg_statistic_ext: 拡張統計情報を格納するシステムカタログ。

失敗モードと境界条件

  • 統計オブジェクトが存在しない場合（IF EXISTS がない場合）、エラーが発生する。
  • 統計ターゲットが負の値の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなる。
  • 呼び出し元が統計オブジェクトの所有者でない場合、aclcheck_error が発生する。
  • キャッシュルックアップに失敗した場合、elog(ERROR) で中断する。

保守担当者向けメモ

  • 現在、本関数は統計ターゲットの更新のみをサポートしている。他のフィールドの更新が必要な場合は、依存関係の再計算ロジックを追加する必要がある。
  • 統計ターゲットのデフォルト値は -1 であり、これは stxstattarget カラムを NULL に設定することで表現される。
