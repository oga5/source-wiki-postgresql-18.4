setup_dynamic_shared_memory

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/setup.c:90

目的

テスト用の動的共有メモリ（DSM）セグメントを確保し、shm_toc（テーブル・オブ・コンテンツ）を用いてヘッダー領域と複数のメッセージキューを配置・初期化する。

シグネチャまたは形状

static void setup_dynamic_shared_memory(int64 queue_size, int nworkers,
                                        dsm_segment **segp, test_shm_mq_header **hdrp,
                                        shm_mq **outp, shm_mq **inp)

入力と出力

  • **入力**:
  • queue_size: 各メッセージキューのサイズ（バイト単位）。
  • nworkers: 起動するワーカー数。
  • **出力**:
  • segp: 確保された dsm_segment へのポインタ。
  • hdrp: 初期化された test_shm_mq_header へのポインタ。
  • outp: 送信側として設定された shm_mq へのポインタ。
  • inp: 受信側として設定された shm_mq へのポインタ。

前提条件と不変条件

  • queue_size は shm_mq_minimum_size 以上であること。
  • queue_size は Size 型の範囲内に収まること。
  • 共有メモリのレイアウトは、ヘッダー用に1キー、メッセージキュー用に nworkers + 1 キーを使用する。

アルゴリズムまたは振る舞い

1. **サイズ見積もり**: shm_toc_estimator を使用し、ヘッダーと nworkers + 1 個のキューに必要なメモリ量を計算する。 2. **セグメント確保**: dsm_create で計算されたサイズのセグメントを確保し、shm_toc_create で TOC を初期化する。 3. **ヘッダー構築**: test_shm_mq_header を割り当て、スピンロックの初期化とワーカー管理用カウンタの初期化を行い、TOC に登録する。 4. **キュー構築**: nworkers + 1 個のメッセージキューを生成し、TOC に登録する。

  • インデックス 0 のキューを送信側（outp）として設定。
  • インデックス nworkers のキューを受信側（inp）として設定。

    • 5. **結果返却**: 確保したセグメントとヘッダーのポインタを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_shm_mq/setup.c 内の関数（test_shm_mq_setup からの呼び出しが確認されている）。
  • **呼び出し先**: shm_toc_initialize_estimator, shm_toc_estimate_chunk, shm_toc_estimate_keys, shm_toc_estimate, dsm_create, shm_toc_create, shm_toc_allocate, SpinLockInit, shm_toc_insert, shm_mq_create, shm_mq_set_sender, shm_mq_set_receiver。

関連データ構造

  • dsm_segment: 動的共有メモリセグメント。
  • shm_toc: 共有メモリ内のデータ配置を管理するテーブル。
  • test_shm_mq_header: テストモジュール固有の共有メモリヘッダー。
  • shm_mq: 共有メモリメッセージキュー。

失敗モードと境界条件

  • queue_size が shm_mq_minimum_size 未満の場合、または Size 型の範囲を超える場合、ereport(ERROR, ...) により処理が中断される。
  • メモリ確保や TOC 構築に失敗した場合、PostgreSQL のメモリ管理機構によりエラーがスローされる。

保守担当者向けメモ

  • shm_toc のパディングによるサイズ変動を考慮し、各チャンクを個別に shm_toc_estimate_chunk で見積もっている点に注意すること。
  • 送信側と受信側のキュー設定は、ループ内のインデックス判定（i == 0 および i == nworkers）に依存している。
