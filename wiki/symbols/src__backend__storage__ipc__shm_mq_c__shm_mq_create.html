shm_mq_create

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_mq.c:176

目的

共有メモリ領域上に、プロセス間通信のためのメッセージキュー構造体（shm_mq）を初期化する。

シグネチャまたは形状

shm_mq *shm_mq_create(void *address, Size size)

入力と出力

  • **入力**:
  • address: キューを配置する共有メモリ上の開始アドレス。
  • size: 割り当てられたメモリ領域の合計サイズ。
  • **出力**:
  • 初期化された shm_mq 構造体へのポインタ。

前提条件と不変条件

  • size は MAXALIGN 境界に切り捨てられる。
  • size は、キューヘッダー（shm_mq）とリングバッファの最小限のデータ領域を保持できる十分な大きさである必要がある（Assert(size > data_offset)）。

アルゴリズムまたは振る舞い

1. address を shm_mq 構造体としてキャストする。 2. MAXALIGN を用いて、ヘッダー直後のリングバッファ開始位置（data_offset）を算出する。 3. size を MAXALIGN_DOWN で調整し、余分なバイトを切り捨てる。 4. mq_mutex（スピンロック）を初期化する。 5. 送信側・受信側のポインタを NULL に設定する。 6. 読み込み済みバイト数（mq_bytes_read）と書き込み済みバイト数（mq_bytes_written）を 0 に初期化する。 7. リングバッファの有効サイズ（mq_ring_size）を計算し、mq_detached フラグを false に設定する。 8. 構造体内のオフセット情報を記録し、初期化済み構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c
  • src/backend/executor/execParallel.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/test/modules/test_shm_mq/setup.c
  • **呼び出し先**:
  • SpinLockInit
  • pg_atomic_init_u64

関連データ構造

  • shm_mq: 共有メモリ上に配置されるキューのメイン構造体。

失敗モードと境界条件

  • size が小さすぎてヘッダー領域を確保できない場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数はキューの初期化のみを行い、送信側・受信側のプロセス登録は別途 shm_mq_set_sender / shm_mq_set_receiver を呼び出す必要がある。
