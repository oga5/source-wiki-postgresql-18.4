CheckSASLAuth

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-sasl.c:43

目的

SASL認証プロトコルのハンドシェイクを管理し、クライアントとのメッセージ交換ループを通じて認証を完遂する。

シグネチャまたは形状

int CheckSASLAuth(const pg_be_sasl_mech *mech, Port *port, char *shadow_pass, const char **logdetail);

入力と出力

  • **入力**:
  • mech: 使用するSASLメカニズムの関数ポインタ群。
  • port: クライアント接続情報。
  • shadow_pass: 認証に使用するパスワード情報（メカニズムにより使用可否が異なる）。
  • **出力**:
  • 戻り値: STATUS_OK（成功）、STATUS_ERROR（失敗）、または STATUS_EOF（接続切断）。
  • logdetail: 認証失敗時の詳細メッセージ（出力引数）。

前提条件と不変条件

  • mech->get_mechanisms がサポートするメカニズムを正しく提示できること。
  • ユーザーが存在しない場合やパスワードが不正な場合でも、認証プロセスを最後まで継続し、ユーザー列挙攻撃を防ぐ「doomed」状態を維持すること。
  • input バッファの末尾には必ず \0 が存在すること。

アルゴリズムまたは振る舞い

1. **初期化**: mech->get_mechanisms を呼び出し、サポートするメカニズムをクライアントに通知する。 2. **ループ**: PqMsg_SASLResponse メッセージを待機し、受信する。 3. **初回応答**: クライアントが選択したメカニズムを解析し、mech->init でセッション状態（opaq）を初期化する。 4. **交換**: mech->exchange を呼び出し、入力データから応答を生成する。 5. **送信**: 生成された応答を AUTH_REQ_SASL_CONT（継続）または AUTH_REQ_SASL_FIN（完了）としてクライアントに送信する。 6. **終了**: result が PG_SASL_EXCHANGE_SUCCESS になるまで繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth.c（CheckSASLAuth(&pg_be_scram_mech, ...) 等）。
  • **呼び出し先**: mech 構造体内の関数ポインタ（get_mechanisms, init, exchange）、および sendAuthRequest, pq_getmessage 等の通信関数。

関連データ構造

  • pg_be_sasl_mech: SASLメカニズムの振る舞いを定義する構造体。
  • Port: クライアント接続状態を保持する構造体。
  • StringInfoData: メッセージ送受信用のバッファ。

失敗モードと境界条件

  • PqMsg_SASLResponse 以外のメッセージを受信した場合はプロトコル違反としてエラー終了する。
  • mech->exchange が PG_SASL_EXCHANGE_FAILURE を返した際に output が生成されている場合はエラーとする。

保守担当者向けメモ

  • 本関数はSASLの制御フローのみを担い、具体的な暗号化ロジックは mech 経由で外部（auth-scram.c 等）に委譲される。
  • 認証失敗時も処理を継続する設計であるため、result の判定と logdetail の設定がセキュリティ上重要である。
