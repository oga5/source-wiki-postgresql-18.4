when

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/preproc/type.h:88

目的

ECPG（Embedded SQL for C）プリプロセッサにおいて、SQL実行時のエラーや警告発生時（WHENEVER文）の動作を定義・保持するために使用されます。

シグネチャまたは形状

struct when
{
	enum WHEN_TYPE code;
	char	   *command;
	char	   *str;
};

入力と出力

  • **入力**: WHENEVER文の解析時に、アクションの種類（code）および実行されるコマンドや文字列（command, str）が設定されます。
  • **出力**: プリプロセッサが生成するCコード内で、エラーハンドリングのロジックを構築するために参照されます。

前提条件と不変条件

  • code フィールドは WHEN_TYPE 列挙型に基づき、特定のアクション（SQL_WHENEVER_DO_CONTINUE 等）を識別します。
  • command および str は、動的に確保された文字列を指すポインタである可能性があります。

アルゴリズムまたは振る舞い

本構造体は、ECPGプリプロセッサがソースコードをスキャンする際に、WHENEVERディレクティブの状態を保持するデータコンテナとして機能します。解析されたアクションの内容を保持し、後続のSQL文生成時に適切なエラー処理コードを挿入するためのメタデータを提供します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、本構造体を直接操作する具体的な関数呼び出し元は記載されていません。
  • **呼び出し先**: 本構造体はデータ保持用であり、関数を呼び出すものではありません。
  • **注意**: ツールが提示した contrib/amcheck 等の参照は、PostgreSQLの別のモジュールにおける一般的な単語としての "when" であり、本シンボル（ECPGプリプロセッサの構造体）とは無関係です。

関連データ構造

  • WHEN_TYPE: code フィールドに使用される列挙型。

失敗モードと境界条件

  • command や str が NULL の場合、特定の code（例: GOTO や CALL）では不正な参照が発生する可能性があるため、プリプロセッサ側での適切なバリデーションが必要です。

保守担当者向けメモ

  • 本構造体は src/interfaces/ecpg/preproc/type.c で実装されるメモリ管理関数（ECPGfree_* 等）によって適切に解放される必要があります。
  • 構造体の再帰的な参照は含まれていませんが、プリプロセッサの状態管理の一部として、解析のコンテキストに応じて適切に更新・破棄される点に注意してください。
