PartClauseMatchStatus

定義

  • 種別: enum
  • 定義箇所: src/backend/partitioning/partprune.c:78

目的

パーティションプルーニング処理において、クエリの WHERE 句（RestrictInfo）がパーティションキーとどのように適合するか、あるいは適合しないかを分類するために使用される状態列挙型です。

シグネチャまたは形状

typedef enum PartClauseMatchStatus
{
	PARTCLAUSE_NOMATCH,           /* 適合せず */
	PARTCLAUSE_MATCH_CLAUSE,      /* 句として適合 */
	PARTCLAUSE_MATCH_NULLNESS,    /* NULL判定として適合 */
	PARTCLAUSE_MATCH_STEPS,       /* ステップとして適合 */
	PARTCLAUSE_MATCH_CONTRADICT,  /* 矛盾（常に偽） */
	PARTCLAUSE_UNSUPPORTED,       /* サポート対象外 */
} PartClauseMatchStatus;

入力と出力

本シンボルは列挙型であり、関数がクエリ条件を解析した結果を返す際の戻り値として使用されます。

前提条件と不変条件

特になし。

アルゴリズムまたは振る舞い

クエリの条件式をパーティションキーと比較する際、その適合状態を識別します。

  • PARTCLAUSE_MATCH_CLAUSE: 通常の比較演算子による適合。
  • PARTCLAUSE_MATCH_NULLNESS: IS NULL や IS NOT NULL による適合。
  • PARTCLAUSE_MATCH_CONTRADICT: 条件が論理的に不可能（例: x = 1 AND x = 2）であることを示し、プルーニングの最適化に寄与します。

呼び出し元と呼び出し先

  • **呼び出し元**: match_clause_to_partition_key、match_boolean_partition_clause、および partprune.c 内の解析関数がこの値を返します。
  • **呼び出し先**: 本シンボル自体は列挙型であり、呼び出し先はありません。

関連データ構造

  • GeneratePruningStepsContext: ステップ生成中の状態管理に使用されます。

失敗モードと境界条件

  • PARTCLAUSE_NOMATCH: 条件がパーティションキーと関連付けられない場合。
  • PARTCLAUSE_UNSUPPORTED: パーティションプルーニングで扱えない複雑な式や演算子が含まれる場合。

保守担当者向けメモ

  • 新しいプルーニングロジックを追加する際は、この列挙型に新しい状態が必要か検討してください。
  • match_clause_to_partition_key 等の解析関数が、この値をどのように解釈して後続の PartitionPruneStep 生成に繋げているかを確認してください。
