ginVacuumEntryPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginvacuum.c:454

目的

GINインデックスのリーフページ（エントリページ）を走査し、不要なTID（ItemPointer）の削除とポストティングリストの再圧縮を行う。また、ポストティングツリーのルートを収集し、後続のバキューム処理へ引き渡す。

シグネチャまたは形状

static Page ginVacuumEntryPage(GinVacuumState *gvs, Buffer buffer, BlockNumber *roots, uint32 *nroot)

入力と出力

  • **入力**:
  • gvs: バキュームの状態を保持する GinVacuumState 構造体。
  • buffer: 処理対象のページバッファ。
  • roots: ポストティングツリーのルートブロック番号を格納する配列。
  • nroot: 収集したルートの数を格納するポインタ。
  • **出力**:
  • ページに変更があった場合は、変更を適用した一時的な Page を返す。変更がない場合は NULL を返す。

前提条件と不変条件

  • buffer は有効なGINエントリページであること。
  • roots 配列は、ページ内のポストティングツリーの数以上の容量を持つこと。
  • ポストティングツリーのバキュームは、デッドロック回避のため本関数内では行わず、ルートの収集のみを行う。

アルゴリズムまたは振る舞い

1. ページ内の全インデックスエントリを走査する。 2. エントリがポストティングツリーへのポインタを持つ場合、そのルートブロック番号を roots に追加する。 3. エントリがポストティングリストを持つ場合：

  • リストをデコードし、ginVacuumItemPointers を呼び出して削除対象のTIDを除去する。
  • TIDが削除された場合、ginCompressPostingList でリストを再圧縮し、GinFormTuple で新しいタプルを作成する。
  • ページに変更を加える必要がある場合、PageGetTempPageCopy で一時ページを作成し、そこに新しいタプルを挿入する。

    • 4. 変更があった場合は一時ページを、なければ NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginvacuum.c 内の ginbulkdelete から呼び出される。
  • **呼び出し先**:
  • ginVacuumItemPointers (TIDの削除)
  • ginPostingListDecode / ginCompressPostingList (ポストティングリストの操作)
  • GinFormTuple (タプルの再構築)
  • PageGetTempPageCopy / PageAddItem / PageIndexTupleDelete (ページ操作)

関連データ構造

  • GinVacuumState: バキューム処理のコンテキスト。
  • IndexTuple: GINインデックスのタプル構造。
  • GinPostingList: 圧縮されたTIDリスト。

失敗モードと境界条件

  • PageAddItem が失敗した場合、elog(ERROR) を発生させる。
  • ページ内にポストティングツリーが一つも存在しない場合、nroot は 0 となる。
  • 削除対象のTIDが存在しない場合、ページは変更されず NULL が返される。

保守担当者向けメモ

  • 本関数はページ単位の処理を行う。ポストティングツリーのバキュームは別関数（ginVacuumPostingTree）に委譲されるため、ロック順序の整合性に注意すること。
  • tmppage を使用したインプレース更新のロジックは、最初の変更が発生した時点で一時ページを作成する効率的な設計となっている。
