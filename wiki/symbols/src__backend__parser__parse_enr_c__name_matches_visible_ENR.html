name_matches_visible_ENR

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_enr.c:19

目的

現在のクエリ環境において、指定された名前（refname）を持つ一時的名前付きリレーション（ENR）が可視であるかどうかを判定する。

シグネチャまたは形状

bool name_matches_visible_ENR(ParseState *pstate, const char *refname)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体へのポインタ。
  • refname: 検索対象のリレーション名（文字列）。
  • **出力**:
  • bool: 指定された名前のENRが可視であれば true、そうでなければ false を返す。

前提条件と不変条件

  • pstate は有効な ParseState 構造体を指している必要がある。
  • pstate->p_queryEnv を介してENRの検索が行われるため、クエリ環境が適切に初期化されている必要がある。

アルゴリズムまたは振る舞い

1. pstate から p_queryEnv を取得する。 2. get_visible_ENR_metadata 関数を呼び出し、p_queryEnv 内で refname に一致するメタデータを検索する。 3. 検索結果が NULL でない場合（メタデータが存在する場合）は true を、NULL の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_relation.c:345
  • **呼び出し先**:
  • get_visible_ENR_metadata（src/include/utils/queryenvironment.h で定義）

関連データ構造

  • ParseState: クエリ解析中の状態を保持する構造体。
  • QueryEnvironment: pstate->p_queryEnv として保持され、ENRの検索対象となる環境。

失敗モードと境界条件

  • pstate が NULL の場合、あるいは p_queryEnv が初期化されていない場合は、呼び出し元の環境に依存するが、セグメンテーション違反が発生する可能性がある。
  • refname が NULL の場合の挙動は get_visible_ENR_metadata の実装に依存する。

保守担当者向けメモ

  • 本関数は get_visible_ENR_metadata への薄いラッパーである。ENRの検索ロジックそのものに変更が必要な場合は、src/include/utils/queryenvironment.h および関連する実装を確認すること。
