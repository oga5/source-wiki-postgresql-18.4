SerializeLibraryState

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/dfmgr.c:718

目的

ロード済みの動的ライブラリのファイル名リストを、並列クエリ等のプロセス間通信のために指定されたメモリ領域へシリアライズ（直列化）します。

シグネチャまたは形状

void SerializeLibraryState(Size maxsize, char *start_address)

入力と出力

  • maxsize (入力): シリアライズに使用可能なバッファの最大サイズ。
  • start_address (入力/出力): ライブラリ名リストを書き込む開始アドレス。リストの末尾にはヌル終端文字が追加されます。

前提条件と不変条件

  • file_list が有効な DynamicFileList の連結リストを指していること。
  • 書き込み先の start_address が maxsize 分のメモリを確保していること。
  • 各ライブラリ名のコピーにおいて、strlcpy によるコピー長とヌル終端文字の合計が maxsize を超えないこと（Assert により検証）。

アルゴリズムまたは振る舞い

1. グローバル変数 file_list を先頭から順に走査します。 2. 各ライブラリの filename を start_address にコピーします。 3. コピーした文字列の長さ（ヌル終端含む）分だけ start_address を進め、maxsize を減算します。 4. リストの末尾に到達したら、start_address の先頭にヌル終端文字を書き込み、リストの終了を示します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/parallel.c:383
  • 呼び出し先: strlcpy (標準ライブラリ)

関連データ構造

  • DynamicFileList: ロード済みライブラリを管理する連結リスト。

失敗モードと境界条件

  • maxsize が不足している場合、Assert(len < maxsize) によりデバッグビルド環境でアボートします。
  • file_list が空の場合、即座に start_address[0] に \0 が書き込まれ、処理を終了します。

保守担当者向けメモ

  • 本関数は RestoreLibraryState と対になる処理です。シリアライズ形式を変更する場合は、必ず復元側のロジックも更新してください。
  • Assert は実行時の安全性を保証するものではないため、本番環境でのバッファオーバーフロー防止には呼び出し元での十分なサイズ見積もり（EstimateLibraryStateSpace 等の利用）が必須です。
