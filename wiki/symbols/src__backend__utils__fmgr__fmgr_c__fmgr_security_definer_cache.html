fmgr_security_definer_cache

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:611

目的

SECURITY DEFINER 属性を持つ関数の実行に必要なメタデータ、ユーザーID、および設定パラメータ（GUC）をキャッシュし、実行時のオーバーヘッドを削減するために使用されます。

シグネチャまたは形状

struct fmgr_security_definer_cache
{
    FmgrInfo    flinfo;         /* 対象関数のルックアップ情報 */
    Oid         userid;         /* 設定するユーザーID、または InvalidOid */
    List       *configNames;    /* 設定するGUC名リスト */
    List       *configHandles;  /* 設定するGUCハンドルリスト */
    List       *configValues;   /* 設定するGUC値リスト */
    Datum       arg;            /* プラグインモジュール用のパススルー引数 */
};

入力と出力

本構造体はデータ保持用であり、関数呼び出しの引数や戻り値として直接やり取りされるものではありません。実行時に fmgr_security_definer などの関数内で、対象関数の実行コンテキストを保持するために充填されます。

前提条件と不変条件

  • flinfo は、対象となる関数の有効な FmgrInfo を保持している必要があります。
  • configNames, configHandles, configValues は、それぞれ対応するGUC設定を保持するか、設定がない場合は NIL である必要があります。

アルゴリズムまたは振る舞い

SECURITY DEFINER 関数が呼び出される際、この構造体を用いて以下の処理が行われます。 1. flinfo を使用して対象関数の実行準備を行う。 2. userid が指定されている場合、実行時にユーザーIDを切り替える。 3. configNames 等のリストに基づき、関数の実行に必要なGUC設定を一時的に適用する。 4. 実行終了後、元のユーザーIDおよびGUC設定に復元する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/fmgr/fmgr.c 内の fmgr_security_definer 関数において、ローカル変数 fcache として参照されています。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • FmgrInfo: 関数呼び出しのメタデータを保持する構造体。
  • List: PostgreSQLのリスト構造（GUC設定の管理に使用）。

失敗モードと境界条件

  • userid が InvalidOid の場合、ユーザーIDの切り替えは行われません。
  • GUC設定リストが NIL の場合、GUCの切り替え処理はスキップされます。
  • メモリ不足等のシステムエラーが発生した場合、PG_TRY/CATCH ブロックによるクリーンアップが想定されています。

保守担当者向けメモ

  • 本構造体は fmgr_security_definer の実行コンテキストを保持する重要なキャッシュです。構造体のフィールドを追加・変更する場合は、fmgr_security_definer 内の初期化およびクリーンアップロジックとの整合性を確認してください。
