Bitmapset

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/bitmapset.h:49

目的

非負整数の集合をメモリ効率良く表現し、クエリ最適化やプランナにおける集合演算（和、積、差、包含判定）を高速に行うためのデータ構造です。

シグネチャまたは形状

typedef struct Bitmapset
{
    NodeTag     type;
    int         nwords;
    bitmapword  words[FLEXIBLE_ARRAY_MEMBER];
} Bitmapset;

入力と出力

  • **入力**: 集合を操作する関数（bms_add_member 等）は、既存の Bitmapset ポインタを受け取ります。
  • **出力**: 集合の変更を伴う操作は、新しい Bitmapset ポインタを返します。メモリ再配置の可能性があるため、呼び出し元は戻り値を元の変数に再代入する必要があります。

前提条件と不変条件

  • **空集合の表現**: 空集合は NULL ポインタとして表現されます。
  • **メモリ管理**: 集合を操作する関数は、入力が NULL の場合を適切に処理し、結果が空になる場合は NULL を返します。
  • **構造**: nwords は words 配列内の有効なワード数を保持します。

アルゴリズムまたは振る舞い

  • **可変長配列**: FLEXIBLE_ARRAY_MEMBER を使用し、保持する整数範囲に応じた動的なサイズ管理を行います。
  • **最適化**: CPUアーキテクチャに応じたビット操作（bmw_popcount 等）を利用して高速な集合演算を実現します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/file_fdw, contrib/pg_overexplain, contrib/postgres_fdw, contrib/sepgsql 等の各モジュールで、属性や列の集合管理に使用されています。
  • **呼び出し先**: src/port/pg_bitutils.h（ビット操作関数）および src/backend/nodes/bitmapset.c（演算の実装）。

関連データ構造

  • bitmapword: 集合を構成するビット単位の型（32bit または 64bit）。
  • BMS_Comparison / BMS_Membership: 集合の比較やメンバーシップ状態を示す列挙型。
  • multibitmapset.h: より大規模な集合を扱うための拡張モジュール。

失敗モードと境界条件

  • **NULL ポインタ**: 空集合を NULL として扱う規約に違反すると、セグメンテーション違反や誤った演算結果を招きます。bms_is_empty マクロによるチェックが必須です。
  • **再代入の欠落**: 集合変更関数の戻り値を無視すると、解放済みのメモリ領域を参照するリスクがあります。

保守担当者向けメモ

  • REALLOCATE_BITMAPSETS を有効にしてビルドすることで、メモリ解放後の不正アクセスをデバッグ可能です。
  • 集合を変更する関数はインプレース更新を保証しないため、戻り値の再代入を徹底してください。
