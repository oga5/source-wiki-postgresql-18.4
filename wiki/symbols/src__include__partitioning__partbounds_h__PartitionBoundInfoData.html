PartitionBoundInfoData

定義

  • 種別: struct
  • 定義箇所: src/include/partitioning/partbounds.h:79

目的

パーティション分割テーブルにおける境界情報（範囲、リスト、ハッシュ）を保持し、特定のデータがどのパーティションに属するかを判定するための構造体です。

シグネチャまたは形状

typedef struct PartitionBoundInfoData
{
    PartitionStrategy strategy;
    int            ndatums;
    Datum        **datums;
    PartitionRangeDatumKind **kind;
    Bitmapset     *interleaved_parts;
    int            nindexes;
    int           *indexes;
    int            null_index;
    int            default_index;
} PartitionBoundInfoData;

入力と出力

  • **入力**: パーティション定義（PartitionBoundSpec）から partition_bounds_create を通じて構築されます。
  • **出力**: 検索関数（partition_list_bsearch 等）により、該当するパーティションのインデックス（int）が返されます。

前提条件と不変条件

  • ndatums は datums 配列の長さを示します。
  • nindexes は indexes 配列の長さを示します。
  • kind は範囲パーティションでのみ使用され、ハッシュおよびリストパーティションでは NULL です。
  • interleaved_parts はリストパーティションでのみ使用されます。
  • null_index および default_index は、該当するパーティションが存在しない場合 -1 となります。
  • 戦略によって ndatums と nindexes の関係が異なります（例: Range 戦略では nindexes = ndatums + 1）。

アルゴリズムまたは振る舞い

  • **検索**: datums 配列に対して二分探索を行い、クエリ対象のパーティションを特定します。
  • **結合**: partition_bounds_merge により、結合対象の境界情報を統合して新しい境界情報を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partbounds.c 内の各関数（partition_bounds_create, partition_bounds_merge 等）および src/include/nodes/pathnodes.h で参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、関連する検索ロジック（partprune.h 等）で利用されます。

関連データ構造

  • PartitionStrategy: パーティション戦略（Hash, List, Range）。
  • PartitionBoundSpec: カタログ上の境界定義。
  • PartitionDesc: パーティション記述子。

失敗モードと境界条件

  • null_index や default_index が -1 の場合、該当する特殊パーティションは存在しません。
  • 境界値の不整合は partition_bounds_create 実行時の検証で検出されます。

保守担当者向けメモ

  • indexes 配列の解釈は戦略ごとに異なるため、定義直前のコメントを必ず参照してください。
  • プランナとエグゼキュータの両方から参照されるため、構造体の変更は広範囲に影響を及ぼします。
