sanitize_line

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:51

目的

入力文字列に含まれる改行コード（\n）および復帰コード（\r）をスペースに置換し、ダンプ出力の整合性を維持する。

シグネチャまたは形状

char *sanitize_line(const char *str, bool want_hyphen)

入力と出力

  • **入力**:
  • str: サニタイズ対象の文字列（const char *）。
  • want_hyphen: str が NULL の場合に返す代替文字列を制御するフラグ（bool）。
  • **出力**:
  • サニタイズ済みの新しい文字列（char *）。呼び出し元でメモリ解放が必要。

前提条件と不変条件

  • str が NULL の場合、want_hyphen が真なら "-" を、偽なら空文字列 "" を複製して返す。
  • 戻り値は pg_strdup によってヒープ領域に確保される。

アルゴリズムまたは振る舞い

1. str が NULL の場合、指定された代替文字列を複製して即座に返す。 2. str が非 NULL の場合、pg_strdup で文字列を複製する。 3. 複製した文字列を走査し、各文字が \n または \r である場合、その位置をスペース（' '）で上書きする。 4. 処理後の文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_archiver.c (複数箇所: AH->archdbname, te->tag, te->namespace, te->owner, te->tablespace 等の処理)
  • src/bin/pg_dump/pg_dump.c (copyFrom の処理)
  • src/bin/pg_dump/pg_dumpall.c (username の処理)
  • **呼び出し先**:
  • pg_strdup (PostgreSQL フロントエンド用メモリ確保関数)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • メモリ不足時には pg_strdup 経由でプロセスが終了する可能性がある。
  • 入力文字列が空の場合、空文字列を返す。

保守担当者向けメモ

  • 本関数は SQL インジェクション防止の防壁として機能するため、ダンプ出力に名前や識別子を含める際は必ず経由すること。
  • pg_strdup を使用しているため、呼び出し元で必ず free() を行うこと。
