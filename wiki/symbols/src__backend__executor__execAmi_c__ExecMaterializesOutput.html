ExecMaterializesOutput

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAmi.c:635

目的

指定されたプランノードタイプが、その出力をメモリや一時ストレージにマテリアライズ（保持）するかどうかを判定します。主に再スキャンコストの見積もりに使用されます。

シグネチャまたは形状

bool ExecMaterializesOutput(NodeTag plantype)

入力と出力

  • **入力**: NodeTag plantype（判定対象のプランノードタイプ）
  • **出力**: bool（マテリアライズを行う場合は true、それ以外は false）

前提条件と不変条件

  • 入力される plantype は、PostgreSQL のプランノードタグである必要があります。

アルゴリズムまたは振る舞い

以下のノードタイプが渡された場合に true を返し、それ以外は false を返します。

  • T_Material
  • T_FunctionScan
  • T_TableFuncScan
  • T_CteScan
  • T_NamedTuplestoreScan
  • T_WorkTableScan
  • T_Sort

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/costsize.c
  • src/backend/optimizer/path/joinpath.c
  • src/backend/optimizer/plan/subselect.c
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（switch 文による判定のみ）。

関連データ構造

  • NodeTag (プランノードの識別子)

失敗モードと境界条件

  • 未知の NodeTag が渡された場合、デフォルトで false を返します。

保守担当者向けメモ

  • 新しいプランノードタイプが追加され、それが結果をマテリアライズする性質を持つ場合は、本関数の switch 文にケースを追加する必要があります。
