rendezvousHashEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/fmgr/dfmgr.c:35

目的

動的ライブラリ間でポインタ情報を共有するための「Rendezvous」機構において、ハッシュテーブルの各エントリを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    char        varName[NAMEDATALEN]; /* ハッシュキー（先頭である必要がある） */
    void       *varValue;             /* 共有されるポインタ値 */
} rendezvousHashEntry;

入力と出力

  • 入力: varName（キー）、varValue（値）
  • 出力: なし（データ構造体）

前提条件と不変条件

  • varName は構造体の先頭メンバである必要があります（ハッシュテーブルのキーとして機能するため）。
  • varName の長さは NAMEDATALEN に制限されます。

アルゴリズムまたは振る舞い

本構造体は hash_search を用いたハッシュテーブル操作の単位として機能します。dfmgr.c 内の find_rendezvous_variable 関数において、指定された名前の変数を検索または作成する際の格納先として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/fmgr/dfmgr.c 内の find_rendezvous_variable 関数が、ハッシュテーブルのサイズ定義およびエントリの型キャストに使用しています。
  • 呼び出し先: 本構造体自体はデータ構造であり、直接関数を呼び出すことはありません。

関連データ構造

  • rendezvousHash: 本構造体を要素として格納するハッシュテーブル。
  • HTAB (PostgreSQL のハッシュテーブル管理構造体): ctl.entrysize を通じて本構造体のサイズが登録されます。

失敗モードと境界条件

  • varName が NAMEDATALEN を超える場合、切り詰めやバッファオーバーフローのリスクが生じる可能性があります（PostgreSQL の標準的な文字列処理規則に従います）。
  • ハッシュテーブルが初期化されていない状態で本構造体を使用しようとするとエラーになります。

保守担当者向けメモ

  • 本構造体は find_rendezvous_variable によるライブラリ間変数共有の基盤です。構造を変更する場合は、ハッシュテーブルの初期化ロジック（ctl.entrysize の設定箇所）との整合性に注意してください。
