AtEOSubXact_LargeObject

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-fsstubs.c:652

目的

サブトランザクションの終了（コミットまたはアボート）時に、そのサブトランザクション内で開かれたラージオブジェクト（LO）のリソース管理を行う。

シグネチャまたは形状

void AtEOSubXact_LargeObject(bool isCommit, SubTransactionId mySubid, SubTransactionId parentSubid)

入力と出力

  • isCommit (bool): true の場合はコミット、false の場合はアボートとして処理する。
  • mySubid (SubTransactionId): 終了対象のサブトランザクションID。
  • parentSubid (SubTransactionId): 親トランザクションまたは親サブトランザクションのID。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • fscxt（メモリコンテキスト）が初期化されていない場合、本関数は何もせず即座に復帰する。
  • cookies 配列に格納された LargeObjectDesc の subid フィールドが、サブトランザクションの所有権を追跡するために使用される。

アルゴリズムまたは振る舞い

1. fscxt が NULL であれば終了する。 2. cookies 配列を走査し、現在のサブトランザクション (mySubid) に属するすべてのラージオブジェクトディスクリプタを特定する。 3. isCommit が true の場合、対象LOの所有権を親サブトランザクション (parentSubid) に移譲する。 4. isCommit が false の場合、closeLOfd(i) を呼び出して対象LOをクローズし、リソースを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xact.c:5151 (コミット時)
  • src/backend/access/transam/xact.c:5322 (アボート時)
  • 呼び出し先:
  • closeLOfd (アボート時に呼び出される)

関連データ構造

  • cookies: 開いているラージオブジェクトディスクリプタを保持する動的配列。
  • LargeObjectDesc: ラージオブジェクトの状態を保持する構造体。
  • fscxt: ラージオブジェクト操作用のメモリコンテキスト。

失敗モードと境界条件

  • fscxt が未初期化の状態では、LO操作が存在しないものとして安全に無視される。
  • cookies 配列内のエントリが NULL である場合、スキップされる。

保守担当者向けメモ

  • 本関数はサブトランザクションの階層構造におけるリソースの引き継ぎとクリーンアップを担う。
  • トランザクション全体の終了処理については AtEOXact_LargeObject が担当する。
