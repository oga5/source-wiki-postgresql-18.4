plpgsql_token_length

定義

  • src/pl/plpgsql/src/pl_scanner.c:326

目的

現在解析中のトークンの長さを取得します。

シグネチャまたは形状

int plpgsql_token_length(yyscan_t yyscanner);

入力と出力

  • **入力**: yyscan_t yyscanner (スキャナの状態を保持する不透明なポインタ)
  • **出力**: int 型のトークン長

前提条件と不変条件

  • yyscanner が有効なスキャナ状態を指している必要があります。
  • 内部的に yyextra（plpgsql_yy_extra_type）が初期化されていることが前提です。

アルゴリズムまたは振る舞い

  • yyscanner からアクセス可能な yyextra 構造体内の plpgsql_yyleng フィールドの値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/plpgsql.h:1315 に外部宣言が存在します。具体的な呼び出し箇所は提供されたソース内にはありません。
  • **呼び出し先**: なし（直接的なフィールドアクセスのみ）。

関連データ構造

  • yyscan_t: スキャナのコンテキスト。
  • plpgsql_yy_extra_type: plpgsql_yyleng を保持するスキャナ拡張データ構造。

失敗モードと境界条件

  • yyscanner が NULL の場合、yyextra へのアクセスによりセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本関数は plpgsql_yyleng のゲッターとして機能します。スキャナの内部状態に依存しているため、スキャナのライフサイクル外で呼び出さないよう注意してください。
