GetForeignColumnOptions

定義

  • 種別: function
  • 定義箇所: src/backend/foreign/foreign.c:292

目的

指定されたリレーション（テーブル）および属性（カラム）に関連付けられた外部データラッパー（FDW）固有のオプション情報を取得します。

シグネチャまたは形状

List *GetForeignColumnOptions(Oid relid, AttrNumber attnum)

入力と出力

  • **入力**:
  • relid (Oid): 対象リレーションのOID。
  • attnum (AttrNumber): 対象属性の番号。
  • **出力**:
  • List *: 属性に設定されたオプションのリスト（DefElem のリスト）。設定がない場合は NIL を返します。

前提条件と不変条件

  • relid および attnum がシステムカタログ pg_attribute に存在する必要があります。存在しない場合、elog(ERROR) により処理が中断されます。

アルゴリズムまたは振る舞い

1. SearchSysCache2 を使用し、ATTNUM キャッシュから対象属性のタプルを検索します。 2. タプルが有効であることを確認し、SysCacheGetAttr を介して attfdwoptions 属性の値を取得します。 3. 値が NULL の場合は NIL を返します。 4. 値が存在する場合は untransformRelOptions を呼び出し、内部表現（text[]）を List 形式に変換して返します。 5. 最後に ReleaseSysCache を呼び出し、キャッシュの参照カウントを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/deparse.c
  • **呼び出し先**:
  • SearchSysCache2
  • SysCacheGetAttr
  • untransformRelOptions
  • ReleaseSysCache

関連データ構造

  • List: オプションのリスト構造。
  • DefElem: オプションの各要素を保持する構造体。

失敗モードと境界条件

  • 指定された relid と attnum の組み合わせがシステムカタログに見つからない場合、エラーが発生します。
  • 属性にオプションが設定されていない場合、正常に NIL を返します。

保守担当者向けメモ

  • 本関数は pg_attribute システムカタログの attfdwoptions 列を直接参照します。カタログスキーマの変更時には注意が必要です。
  • 返される List は palloc されたメモリ領域にあるため、呼び出し側で適切に管理（必要に応じてメモリコンテキストの解放）を行う必要があります。
