regex_arc_t

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regexport.h:38

目的

正規表現エンジンが構築したNFA（非決定性有限オートマトン）における、状態間の遷移（アーク）を表現するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	int			co;				/* label (character-set color) of arc */
	int			to;				/* next state number */
} regex_arc_t;

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • co: 遷移のラベルとなる「カラー（文字クラス）」を保持します。
  • to: 遷移先の状態番号を保持します。

前提条件と不変条件

  • co は正規表現エンジン内部で管理される文字クラス（カラー）に対応します。
  • to はNFA内の有効な状態番号を指す必要があります。

アルゴリズムまたは振る舞い

本構造体自体にロジックは含まれません。pg_reg_getoutarcs() などの関数を通じて、NFAの遷移情報を外部へ提供する際のコンテナとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な使用例）**:
  • contrib/pg_trgm/trgm_regexp.c: 正規表現の解析処理において、遷移先アークの配列を確保・参照するために使用。
  • src/backend/regex/regexport.c: pg_reg_getoutarcs 等の実装において、NFAの遷移情報を書き出す際の型として使用。
  • **呼び出し先**:
  • 本構造体はデータ構造であるため、特定の関数を呼び出すことはありません。

関連データ構造

  • regex_t: 正規表現のコンパイル結果を保持する構造体。
  • cnfa: regguts.h で定義される圧縮NFA構造。regex_arc_t はこの内部情報を外部へ公開するために使用されます。

失敗モードと境界条件

  • 本構造体自体に失敗モードはありません。
  • 利用側（pg_reg_getoutarcs 等）において、呼び出し元が確保したバッファサイズが不足している場合、メモリ破壊やデータ欠損が発生する可能性があります。

保守担当者向けメモ

  • 本構造体はNFAの構造を外部公開するためのインターフェースの一部です。変更を加える際は、src/backend/regex/regexport.c の実装および contrib/pg_trgm などの依存コードへの影響を確認してください。
  • 「カラー」の概念は regguts.h で定義されており、本構造体の co フィールドを解釈する際は、pg_reg_getcharacters() を通じて文字コードとの対応を確認する必要があります。
