XLogRecordDataHeaderShort

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogrecord.h:213

目的

WAL（Write-Ahead Logging）レコードの「メインデータ」部分を格納するための、短縮形式のヘッダー構造体です。

シグネチャまたは形状

typedef struct XLogRecordDataHeaderShort
{
    uint8       id;             /* XLR_BLOCK_ID_DATA_SHORT */
    uint8       data_length;    /* ペイロードのバイト数 */
} XLogRecordDataHeaderShort;

入力と出力

  • **入力**: WALレコード構築時に、リソースマネージャが提供するメインデータの長さが本構造体の data_length に格納されます。
  • **出力**: WALストリームの解析時、XLogReader が本構造体を読み取ることで、直後に続くペイロードのサイズを特定します。

前提条件と不変条件

  • id フィールドは常に XLR_BLOCK_ID_DATA_SHORT である必要があります。
  • 本構造体は、WALレコード内のメインデータ部分の先頭に配置されます。

アルゴリズムまたは振る舞い

WALレコードの構築フローにおいて、メインデータが短い場合にこのヘッダーが使用されます。data_length フィールドにより、後続するペイロードのバイト数を直接指定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c において、レコード構築時に本構造体への書き込みが行われます。
  • **呼び出し先**: src/backend/access/transam/xlogreader.c において、WALレコードの解析時に本構造体が参照されます。

関連データ構造

  • XLogRecordDataHeaderLong: メインデータがより長い場合に使用される代替構造体。
  • XLogRecord: WALレコード全体の固定長ヘッダー。

失敗モードと境界条件

  • data_length は uint8 型であるため、このヘッダー形式で表現可能なペイロードの最大サイズは255バイトに制限されます。これを超えるデータには XLogRecordDataHeaderLong が必要です。

保守担当者向けメモ

  • 本構造体はコード内で直接使用されることは稀ですが、WALの物理フォーマットを定義する重要なリファレンスです。
  • メモリ上で操作する際は、アライメントに注意してください。WALレコードのヘッダー以降は必ずしも MAXALIGN に整列されていません。
