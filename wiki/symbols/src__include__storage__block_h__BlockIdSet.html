BlockIdSet

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/block.h:80

目的

BlockNumber（32ビット）を、オンディスク構造体に適した BlockIdData 形式へ分解・格納するために使用されます。

シグネチャまたは形状

static inline void BlockIdSet(BlockIdData *blockId, BlockNumber blockNumber)

入力と出力

  • **入力**:
  • blockId: 書き込み対象となる BlockIdData 構造体へのポインタ。
  • blockNumber: 格納する 32ビットのブロック番号。
  • **出力**: なし（blockId が指すメモリ領域を直接更新します）。

前提条件と不変条件

  • blockId は有効な BlockIdData 構造体を指している必要があります。
  • BlockIdData は bi_hi（上位16ビット）と bi_lo（下位16ビット）の2つの uint16 フィールドで構成されます。

アルゴリズムまたは振る舞い

1. blockNumber を右に16ビットシフトし、上位16ビットを blockId->bi_hi に代入します。 2. blockNumber と 0xffff のビットごとの論理積（AND）をとり、下位16ビットを blockId->bi_lo に代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginbtree.c
  • src/include/access/ginblock.h
  • src/include/storage/itemptr.h
  • **呼び出し先**:
  • 特になし（標準的なビット演算のみを実行します）。

関連データ構造

  • BlockIdData: 本関数が操作対象とする構造体。
  • BlockNumber: 入力値として使用される 32ビット符号なし整数型。
  • ItemPointerData: BlockIdData を内部に保持する構造体であり、本関数の主要な利用先の一つです。

失敗モードと境界条件

  • blockNumber が InvalidBlockNumber（0xFFFFFFFF）の場合、bi_hi と bi_lo は共に 0xFFFF となります。
  • ポインタ blockId が NULL の場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は、構造体のアライメントを最適化し、ディスク上のパディングを削減するために設計されています。
  • BlockIdGetBlockNumber と対になっており、データの読み書きを行う際は必ずこのペアで変換を行ってください。
