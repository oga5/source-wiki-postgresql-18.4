tlist_member_match_var

定義

  • src/backend/optimizer/util/tlist.c:110

目的

ターゲットリスト（targetlist）の中から、指定された Var ノードと一致する TargetEntry を検索します。標準的な equal() 関数よりも緩和された条件で比較を行うためのユーティリティです。

シグネチャまたは形状

static TargetEntry *tlist_member_match_var(Var *var, List *targetlist)

入力と出力

  • **入力**:
  • var: 検索対象の Var 構造体へのポインタ。
  • targetlist: 検索対象となる TargetEntry のリスト。
  • **出力**:
  • 一致する TargetEntry へのポインタ。見つからない場合は NULL を返します。

前提条件と不変条件

  • リスト内の各要素は TargetEntry 構造体である必要があります。
  • TargetEntry の expr フィールドが Var ノードである場合にのみ比較が成立します。

アルゴリズムまたは振る舞い

1. targetlist を順次走査します。 2. 各 TargetEntry の expr が Var ノードであるかを確認し、そうでない場合はスキップします。 3. 以下の4つの属性がすべて一致する場合に、その TargetEntry を返します。

  • varno (リレーション番号)
  • varattno (属性番号)
  • varlevelsup (外側のクエリレベルへの参照数)
  • vartype (変数のデータ型)

    • 4. リストの最後まで一致が見つからなければ NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/tlist.c 内の関数から呼び出されています（例: 810行目付近）。
  • **呼び出し先**:
  • lfirst() マクロを使用してリストの要素にアクセスします。
  • IsA() マクロを使用してノード型の判定を行います。

関連データ構造

  • Var: 検索対象の変数表現。
  • TargetEntry: ターゲットリストの各要素を構成する構造体。
  • List: TargetEntry を保持する PostgreSQL のリスト構造。

失敗モードと境界条件

  • targetlist が空の場合、直ちに NULL を返します。
  • TargetEntry の expr が Var ではない場合、その要素は無視されます。
  • 属性の一部のみが一致しても、4つの条件すべてを満たさない限り一致とは見なされません。

保守担当者向けメモ

  • 本関数は equal() による厳密な比較ではなく、特定の最適化コンテキストで必要とされる「緩和された比較」を提供します。
  • 比較条件（varno, varattno, varlevelsup, vartype）を変更する場合は、プランナの変数解決ロジックへの影響を考慮してください。
