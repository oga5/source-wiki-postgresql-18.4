WalReceiverConn

定義

  • **種別**: struct
  • **定義箇所**: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:45

目的

PostgreSQLのWALレシーバープロセスにおいて、プライマリサーバーとの接続状態およびデータ転送用バッファを保持・管理するための内部構造体です。

シグネチャまたは形状

struct WalReceiverConn
{
    PGconn     *streamConn; /* プライマリへの現在の接続 */
    bool        logical;    /* 論理レプリケーションか物理レプリケーションかの識別 */
    char       *recvBuf;    /* 受信レコード用バッファ */
};

入力と出力

本構造体自体はデータ保持用であり、関数呼び出しの引数や戻り値として、レプリケーション接続のコンテキストを伝播するために使用されます。

前提条件と不変条件

  • streamConn は libpq の PGconn オブジェクトを指す必要があります。
  • logical フラグは、接続の性質（物理または論理）を正しく反映している必要があります。

アルゴリズムまたは振る舞い

  • libpqrcv_connect によって初期化され、プライマリサーバーとの接続情報を格納します。
  • libpqrcv_receive 等の関数において、streamConn を介した libpq の COPY モードによるデータ受信や、recvBuf を用いたデータの一時保持に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/subscriptioncmds.c 内の各関数（fetch_table_list, check_publications_origin, ReplicationSlotDropAtPubNode 等）が、レプリケーション接続を管理する引数として本構造体を利用しています。
  • **呼び出し先**: libpqrcv_connect や libpqrcv_get_conninfo 等の libpqwalreceiver.c 内の関数が、本構造体を操作対象として受け取ります。

関連データ構造

  • PGconn: libpq ライブラリが提供する接続ハンドル。
  • WalReceiverFunctions: replication/walreceiver.h で定義されるインターフェース構造体。

失敗モードと境界条件

  • streamConn が NULL の場合、接続が確立されていない状態を示します。
  • recvBuf のメモリ管理は、本構造体を利用する関数群の責務となります。

保守担当者向けメモ

  • 本構造体は src/backend/replication/libpqwalreceiver/libpqwalreceiver.c と replication/walreceiver.h の両方で定義されています。変更時は両方の整合性に注意してください。
  • libpq の COPY モードを利用する際は、バッファのオーバーフローやメモリリークが発生しないよう、recvBuf のライフサイクル管理を徹底してください。
