transform_MERGE_to_join

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepjointree.c:182

目的

MERGE 文をプランナが処理可能な JoinExpr 形式のジョインツリーへ変換します。ターゲットリレーションとソースリレーションを結合し、MERGE アクション（MATCHED/NOT MATCHED）に応じた結合型（INNER/LEFT/RIGHT/FULL）を決定します。

シグネチャまたは形状

void transform_MERGE_to_join(Query *parse)

入力と出力

  • **入力**: Query *parse（CMD_MERGE 型のクエリ構造体）
  • **出力**: なし（副作用として parse->jointree を書き換え、parse->rtable に結合用 RTE を追加）

前提条件と不変条件

  • parse->commandType が CMD_MERGE であること。
  • parse->jointree->fromlist にはソースリレーションが1つだけ含まれていること。
  • ターゲットリレーションは parse->mergeTargetRelation で指定されていること。

アルゴリズムまたは振る舞い

1. **結合型の決定**: mergeActionList を走査し、MERGE_WHEN_NOT_MATCHED_BY_SOURCE および BY_TARGET の有無に基づき、JOIN_INNER, JOIN_LEFT, JOIN_RIGHT, JOIN_FULL のいずれかを選択する。 2. **RTE の作成**: RTE_JOIN 型の新しい RangeTblEntry を作成し、rtable に追加する。 3. **ジョインツリーの構築**: ターゲット（FromExpr）とソースを結合する JoinExpr を作成し、parse->jointree->fromlist をこの単一の JoinExpr に置き換える。 4. **NULL 伝播の処理**: 外部結合（LEFT/RIGHT/FULL）が発生する場合、add_nulling_relids を使用して、結合条件、アクション、RETURNING リスト内の変数を結合によって NULL になり得るものとしてマークする。 5. **クリーンアップ**: 不要となった parse->jointree->quals をクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c
  • **呼び出し先**: add_nulling_relids（src/backend/optimizer/prep/prepjointree.c 内の補助関数）

関連データ構造

  • Query: 変換対象のクエリ構造。
  • JoinExpr: 構築される結合ノード。
  • RangeTblEntry: 結合用に生成される RTE。
  • MergeAction: アクションの判定に使用。

失敗モードと境界条件

  • ソースリレーションが1つでない場合、Assert により停止する。
  • 未知のソースノードタグが検出された場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は MERGE 文のプランニングにおける初期段階の変換を担当します。
  • add_nulling_relids による NULL 許容属性の付与は、ExecMergeMatched 等の実行時処理で MATCHED/NOT MATCHED を区別するために不可欠です。
  • 結合条件のコピーが作成される箇所があるため、メモリ管理とノードの共有関係に注意してください。
