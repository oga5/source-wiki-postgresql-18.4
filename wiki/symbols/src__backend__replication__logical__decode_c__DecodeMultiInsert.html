DecodeMultiInsert

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/decode.c:1111

目的

WALの XLOG_HEAP_MULTI_INSERT レコードを解析し、論理デコーディング用の変更（REORDER_BUFFER_CHANGE_INSERT）として再構築して reorderbuffer にキューイングする。

シグネチャまたは形状

static void DecodeMultiInsert(LogicalDecodingContext *ctx, XLogRecordBuffer *buf)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • buf: 解析対象のWALレコードおよび関連するLSN情報を含むバッファ。
  • **出力**: なし（副作用として reorderbuffer に変更が追加される）。

前提条件と不変条件

  • xlrec->flags に XLH_INSERT_CONTAINS_NEW_TUPLE が含まれている必要がある。
  • 対象のデータベースOIDが ctx->slot->data.database と一致すること。
  • FilterByOrigin によるフィルタリングを通過すること。
  • tupledata が NULL でないこと（アサーションにより保証）。

アルゴリズムまたは振る舞い

1. WALレコードから xl_heap_multi_insert 構造体を取得する。 2. データベースOIDおよびレプリケーションオリジンに基づき、処理対象外であれば早期リターンする。 3. レコード内の各タプルをループ処理する：

  • ReorderBufferAllocChange を呼び出し、変更オブジェクトを確保する。
  • タプルのヘッダー情報（t_infomask 等）をWALデータから復元する。
  • ItemPointerSetInvalid を用いてタプルの物理位置を無効化する。
  • 最後のタプルかつ XLH_INSERT_LAST_IN_MULTI フラグが立っている場合、TOAST再構築状態をクリアするよう設定する。
  • ReorderBufferQueueChange を呼び出し、変更をキューに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c 内のレコード処理ロジック（LogicalDecodingProcessRecord からのディスパッチ経由）。
  • **呼び出し先**:
  • XLogRecGetData, XLogRecGetBlockTag, XLogRecGetBlockData, XLogRecGetOrigin, XLogRecGetXid (WAL解析用マクロ)
  • FilterByOrigin
  • ReorderBufferAllocChange, ReorderBufferAllocTupleBuf, ReorderBufferQueueChange

関連データ構造

  • LogicalDecodingContext: デコーディングの全体状態。
  • XLogRecordBuffer: WALレコードのラッパー。
  • xl_heap_multi_insert: WAL上のマルチインサートレコード構造。
  • ReorderBufferChange: 再構築された論理変更データ。

失敗モードと境界条件

  • **フィルタリング**: データベースOID不一致やオリジンフィルタにより、レコードが無視される場合がある。
  • **データ欠損**: XLH_INSERT_CONTAINS_NEW_TUPLE フラグがない場合、処理を行わずに終了する。
  • **アサーション**: tupledata が NULL の場合、デバッグビルドではアサーション失敗により停止する。

保守担当者向けメモ

  • 本関数は heap_multi_insert によって生成されたWALレコードを処理する。
  • TOASTの再構築状態管理は、マルチインサートの最後の行であるかどうかに依存するため、フラグの扱いに注意すること。
  • tuple->t_tableOid はトランザクション再構築後に決定されるため、ここでは InvalidOid に設定される。
