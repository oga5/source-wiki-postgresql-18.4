pg_sockaddr_cidr_mask

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/ifaddr.c:104

目的

CIDR表記のビット数（文字列）から、指定されたアドレスファミリー（IPv4/IPv6）に対応するネットワークマスクを生成し、sockaddr_storage 構造体に格納します。

シグネチャまたは形状

int pg_sockaddr_cidr_mask(struct sockaddr_storage *mask, char *numbits, int family);

入力と出力

  • **入力**:
  • mask: マスクを書き込むための sockaddr_storage 構造体へのポインタ。
  • numbits: CIDRビット数を示す文字列（例: "24"）。NULL の場合はフルマスク（IPv4なら32、IPv6なら128）とみなされます。
  • family: アドレスファミリー（AF_INET または AF_INET6）。
  • **出力**:
  • 成功時は 0 を返し、mask に結果が格納されます。
  • 失敗時は -1 を返します。

前提条件と不変条件

  • family は AF_INET または AF_INET6 である必要があります。
  • numbits が指定されている場合、有効な数値文字列である必要があります。
  • bits の値は、AF_INET の場合は 0〜32、AF_INET6 の場合は 0〜128 の範囲内である必要があります。

アルゴリズムまたは振る舞い

1. numbits が NULL の場合、ファミリーに応じてデフォルトのビット数（32または128）を設定します。 2. numbits が文字列の場合、strtol で数値に変換します。変換失敗時はエラーを返します。 3. family に基づき処理を分岐します：

  • **AF_INET**: sockaddr_in を使用。ビット数から32ビットのマスク値を計算し、pg_hton32 でネットワークバイトオーダーに変換して格納します。
  • **AF_INET6**: sockaddr_in6 を使用。16バイトの配列をループで処理し、ビット数に応じて 0xff を埋めることでマスクを構築します。

    • 4. 最後に mask->ss_family に family を設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c: 接続許可設定の解析時に呼び出されます。
  • src/backend/libpq/ifaddr.c: インターフェース走査時のデフォルトマスク生成やテストコード内で呼び出されます。
  • **呼び出し先**:
  • strtol (標準ライブラリ)
  • memset, memcpy (標準ライブラリ)
  • pg_hton32 (PostgreSQL内部ユーティリティ)

関連データ構造

  • struct sockaddr_storage: 汎用的なソケットアドレス構造体。
  • struct sockaddr_in: IPv4用ソケットアドレス構造体。
  • struct sockaddr_in6: IPv6用ソケットアドレス構造体。

失敗モードと境界条件

  • numbits が空文字列、または数値以外の文字を含む場合は -1 を返します。
  • bits が指定されたファミリーの許容範囲（IPv4: 0-32, IPv6: 0-128）を超えている場合は -1 を返します。
  • サポートされていない family が渡された場合は -1 を返します。

保守担当者向けメモ

  • 本関数はネットワークバイトオーダーを意識したビット演算を行っています。特に AF_INET における x << 32 のような非ポータブルな操作を避けるためのガード（if (bits > 0)）が含まれています。
  • IPv6のマスク生成は1バイト単位のループで行われており、移植性が考慮されています。
