ExecInitSubqueryScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubqueryscan.c:96

目的

SubqueryScan ノードを初期化し、サブクエリ実行のための状態構造体（SubqueryScanState）を作成・設定する。

シグネチャまたは形状

SubqueryScanState *ExecInitSubqueryScan(SubqueryScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の SubqueryScan プランノード。
  • estate: 実行状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの SubqueryScanState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_MARK が含まれていないこと（Assert で検証）。
  • SubqueryScan ノードは outerPlan および innerPlan を持たないこと（Assert で検証）。

アルゴリズムまたは振る舞い

1. SubqueryScanState ノードを割り当て、ExecSubqueryScan を実行関数として登録する。 2. ExecAssignExprContext を呼び出し、式評価用のコンテキストを作成する。 3. ExecInitNode を呼び出し、サブクエリ（node->subplan）を再帰的に初期化する。 4. ExecInitScanTupleSlot を使用して、サブクエリの出力結果と同期するスキャンタプルスロットを初期化する。 5. スキャンおよび結果の操作関数（scanops / resultops）をサブクエリの出力から取得し、状態構造体に設定する。 6. ExecInitResultTypeTL および ExecAssignScanProjectionInfo を呼び出し、プロジェクションと結果型をセットアップする。 7. ExecInitQual を呼び出し、スキャンノードの qual（WHERE 句）を初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c の ExecInitNode から呼び出されることが確認されている。
  • **呼び出し先**: ExecInitNode（サブプラン用）、ExecInitScanTupleSlot、ExecGetResultType、ExecGetResultSlotOps、ExecInitResultTypeTL、ExecAssignScanProjectionInfo、ExecInitQual 等の実行エンジン関数。

関連データ構造

  • SubqueryScan: 入力プランノード。
  • SubqueryScanState: 実行時の状態を保持する構造体。
  • EState: クエリ実行全体のグローバルな状態。

失敗モードと境界条件

  • eflags に EXEC_FLAG_MARK が指定されている場合、アサーション違反により停止する。
  • 子プラン（subplan）の初期化が失敗した場合、ExecInitNode の挙動に従いエラーが発生する。

保守担当者向けメモ

  • 本関数は SubqueryScan ノードのセットアップに特化しており、SubPlan（式の一部として評価されるもの）とは異なるコードパスである。
  • scanops および resultops の設定において、子プランのスロットを直接参照することでオーバーヘッドを削減している。
