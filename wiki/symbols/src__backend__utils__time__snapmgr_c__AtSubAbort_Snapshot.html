AtSubAbort_Snapshot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/snapmgr.c:979

目的

サブトランザクションの異常終了時に、そのサブトランザクションレベルで作成されたアクティブなスナップショットをスタックから除去し、リソースを解放する。

シグネチャまたは形状

void AtSubAbort_Snapshot(int level)

入力と出力

  • **入力**: level (int) - 破棄対象となるサブトランザクションのネストレベル。
  • **出力**: なし（副作用としてグローバルなスナップショットスタックとXmin状態を更新）。

前提条件と不変条件

  • ActiveSnapshot スタックの各要素において、as_snap->active_count >= 1 であること。
  • 呼び出し後、指定された level 以上のサブトランザクションに関連付けられた ActiveSnapshot 要素はスタックから削除される。

アルゴリズムまたは振る舞い

1. ActiveSnapshot スタックを走査し、as_level が指定された level 以上の要素を順次処理する。 2. 各要素について、対応するスナップショットの active_count をデクリメントする。 3. active_count が 0 かつ regd_count も 0 の場合、FreeSnapshot を呼び出してスナップショットを解放する。 4. スタック要素自体を pfree で解放し、スタックポインタを更新する。 5. 最後に SnapshotResetXmin() を呼び出し、保持されているスナップショットに基づきグローバルな xmin を再計算する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c:5363 (サブトランザクションのアボート処理時)
  • **呼び出し先**: FreeSnapshot, SnapshotResetXmin, pfree

関連データ構造

  • ActiveSnapshot (スタックのヘッド)
  • ActiveSnapshotElt (スタック要素)
  • SnapshotData (スナップショット本体)

失敗モードと境界条件

  • スタックが空の場合、ループは実行されず何も行われない。
  • active_count が 0 未満になるような不正な状態が発生した場合、Assert により停止する可能性がある。

保守担当者向けメモ

  • 本関数はサブトランザクションのロールバック時に呼び出されるため、メモリリークを防ぐためにスタック要素の解放とスナップショットの参照カウント管理を正確に行う必要がある。
  • SnapshotResetXmin() の呼び出しは、VACUUMの進行を妨げないために不可欠である。
