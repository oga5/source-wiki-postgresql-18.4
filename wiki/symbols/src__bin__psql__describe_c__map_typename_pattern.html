map_typename_pattern

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:743

目的

ユーザーが入力した型名のエイリアス（int や varchar[] など）を、PostgreSQL のシステムカタログで定義されている正規の型名に変換し、クエリの整合性を保つ。

シグネチャまたは形状

static const char *map_typename_pattern(const char *pattern);

入力と出力

  • **入力**: pattern (型名を表す文字列)
  • **出力**: マッピングテーブルに一致した場合は正規化された型名（const char *）、一致しない場合または入力が NULL の場合は入力された pattern そのものを返す。

前提条件と不変条件

  • pattern が NULL の場合、関数は NULL を返す。
  • マッピングテーブルは NULL で終端されている。
  • 比較は pg_strcasecmp を使用するため、大文字小文字を区別しない。

アルゴリズムまたは振る舞い

1. 内部の静的配列 typename_map を順次走査する。 2. 入力された pattern と配列内のキー（偶数インデックス）を大文字小文字を区別せずに比較する。 3. 一致した場合、対応する値（奇数インデックス）を返す。 4. 配列の末尾まで一致しなかった場合、入力された pattern をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/describe.c 内の各所（describe.c:585, 707, 904 など）。主に \d 系コマンドでユーザー入力を SQL クエリへ組み込む直前に呼び出される。
  • **呼び出し先**: pg_strcasecmp（標準ライブラリまたは PostgreSQL 互換関数）。

関連データ構造

  • typename_map: 型名のエイリアスと正規名のペアを保持する静的配列。

失敗モードと境界条件

  • 入力が NULL の場合は安全に NULL を返す。
  • マッピングテーブルに存在しない型名が渡された場合、エラーにはならず、入力値をそのまま返す（呼び出し側で後続の SQL 実行時に処理される）。

保守担当者向けメモ

  • 新しい型名のエイリアスを追加する必要がある場合は、typename_map 配列にペアを追加する。
  • gram.y で許容されるが pg_type や format_type() と異なる名称がある場合は、この関数で正規化を維持すること。
