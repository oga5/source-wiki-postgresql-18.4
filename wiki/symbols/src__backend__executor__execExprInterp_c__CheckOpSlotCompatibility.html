CheckOpSlotCompatibility

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExprInterp.c:2440

目的

式評価ステップ（ExprEvalStep）が参照するタプルスロット（TupleTableSlot）の型が、期待される型と互換性があるかを検証する。

シグネチャまたは形状

static void CheckOpSlotCompatibility(ExprEvalStep *op, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • op: 検証対象の式評価ステップ。op->d.fetch.kind に期待されるスロット型情報が含まれる。
  • slot: 実際に使用されるタプルスロット。
  • **出力**: なし（検証失敗時はアサーションエラーが発生する）。

前提条件と不変条件

  • USE_ASSERT_CHECKING が定義されている環境でのみ動作する。
  • op->d.fetch.fixed が真である場合のみ検証を行う。

アルゴリズムまたは振る舞い

1. op->d.fetch.fixed が偽であれば即座に終了する。 2. 以下のいずれかの条件に該当する場合は互換性ありとみなし、検証をパスする。

  • slot が TTSOpsBufferHeapTuple で、期待型が TTSOpsHeapTuple である（またはその逆）。
  • slot の型が TTSOpsVirtual である（変形不要なため）。

    • 3. 上記以外の場合、op->d.fetch.kind と slot->tts_ops が一致していることを Assert で確認する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExprInterp.c 内の式評価ループおよび初期化処理（ExecInterpExpr 等のコンテキスト内）。
  • **呼び出し先**: なし。

関連データ構造

  • ExprEvalStep: 式評価の各ステップを保持する構造体。
  • TupleTableSlot: タプルを保持するスロット構造体。
  • TupleTableSlotOps: スロットの型定義（TTSOpsBufferHeapTuple, TTSOpsHeapTuple, TTSOpsVirtual 等）。

失敗モードと境界条件

  • USE_ASSERT_CHECKING が無効なビルドでは、本関数は何も行わない。
  • 互換性のないスロット型が渡された場合、アサーション違反によりプロセスが停止する。

保守担当者向けメモ

  • 現在、TTSOpsBufferHeapTuple と TTSOpsHeapTuple は相互運用可能として扱われているが、将来的に厳密なチェックが必要になる可能性がある。
  • TTSOpsVirtual は変形（deforming）を必要としないため、常に互換性があるとみなしている。
