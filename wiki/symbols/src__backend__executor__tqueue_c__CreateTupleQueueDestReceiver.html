CreateTupleQueueDestReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tqueue.c:118

目的

並列クエリ実行において、タプルを共有メモリキューへ送信するための DestReceiver インターフェースを初期化・作成する。

シグネチャまたは形状

DestReceiver *CreateTupleQueueDestReceiver(shm_mq_handle *handle)

入力と出力

  • **入力**: shm_mq_handle *handle（タプル送信先となる共有メモリキューのハンドル）
  • **出力**: DestReceiver *（初期化された送信側レシーバーのポインタ）

前提条件と不変条件

  • palloc0 により TQueueDestReceiver 構造体分のメモリがゼロクリアされた状態で割り当てられる。
  • 返される DestReceiver の mydest フィールドは DestTupleQueue に固定される。

アルゴリズムまたは振る舞い

1. TQueueDestReceiver 構造体用のメモリを確保する。 2. DestReceiver の関数ポインタを以下の通り設定する:

  • receiveSlot = tqueueReceiveSlot
  • rStartup = tqueueStartupReceiver
  • rShutdown = tqueueShutdownReceiver
  • rDestroy = tqueueDestroyReceiver

    • 3. mydest を DestTupleQueue に設定する。 4. 引数で受け取った handle を構造体内部の queue フィールドに保持する。 5. 初期化済みの構造体を DestReceiver 型として返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c:1254
  • src/backend/tcop/dest.c:154
  • **呼び出し先**:
  • palloc0（メモリ確保）
  • tqueueReceiveSlot, tqueueStartupReceiver, tqueueShutdownReceiver, tqueueDestroyReceiver（関数ポインタの割り当て）

関連データ構造

  • TQueueDestReceiver: 本関数が作成する送信側状態保持構造体。
  • DestReceiver: PostgreSQL のタプル送信先汎用インターフェース。
  • shm_mq_handle: 共有メモリキューの制御ハンドル。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • handle が NULL の場合でも関数は実行されるが、その後の receiveSlot 呼び出し時に共有メモリ操作で障害が発生する可能性がある。

保守担当者向けメモ

  • 本関数は DestReceiver の各メソッドを tqueue.c 内の静的関数にバインドする役割を持つ。インターフェースの変更が必要な場合は、対応する関数ポインタの更新を忘れないこと。
