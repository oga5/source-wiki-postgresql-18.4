Port

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/libpq-be.h:128

目的

PostgreSQL バックエンドにおいて、クライアント接続の全状態を保持・管理するための中心的なデータ構造です。認証プロセス、通信プロトコル、ネットワーク属性、およびセキュリティ（SSL/TLS, GSSAPI）の状態を追跡します。

シグネチャまたは形状

typedef struct Port {
    pgsocket sock;
    bool noblock;
    ProtocolVersion proto;
    SockAddr laddr;
    SockAddr raddr;
    char *remote_host;
    char *remote_hostname;
    int remote_hostname_resolv;
    int remote_hostname_errcode;
    char *remote_port;
    char local_host[64];
    char *database_name;
    char *user_name;
    char *cmdline_options;
    List *guc_options;
    char *application_name;
    HbaLine *hba;
    int default_keepalives_idle, default_keepalives_interval, default_keepalives_count, default_tcp_user_timeout;
    int keepalives_idle, keepalives_interval, keepalives_count, tcp_user_timeout;
    uint8 scram_ClientKey[SCRAM_MAX_KEY_LEN];
    uint8 scram_ServerKey[SCRAM_MAX_KEY_LEN];
    bool has_scram_keys;
    pg_gssinfo *gss; // または void *gss
    bool ssl_in_use;
    char *peer_cn;
    char *peer_dn;
    bool peer_cert_valid;
    bool alpn_used;
    bool last_read_was_eof;
    SSL *ssl; // または void *ssl
    X509 *peer; // または void *peer
    char *raw_buf;
    ssize_t raw_buf_consumed, raw_buf_remaining;
} Port;

入力と出力

  • **入力**: 接続確立時に Postmaster によって初期化され、スタートアップパケットの内容やソケット情報が格納されます。
  • **出力**: 認証ロジックや通信レイヤー（SSL/GSSAPI）が本構造体を参照し、接続状態に応じた処理を行います。

前提条件と不変条件

  • すべてのメンバおよび配下のデータは TopMemoryContext で割り当てられる必要があります。
  • GSSAPI や SSL が無効なビルドであっても、構造体のオフセットを維持するためにプレースホルダー（void *）が使用されます。

アルゴリズムまたは振る舞い

  • **認証**: hba フィールドや SCRAM 関連フィールドを使用して認証サイクルを管理します。
  • **SSL ハンドシェイク**: raw_buf を使用して、SSL レイヤーへ移行する前に読み込みすぎたデータを一時保持し、再読み込みを可能にするハック的なバッファリングを行います。
  • **通信**: ssl_in_use や gss のフラグに基づき、適切な通信関数（be_tls_* や be_gssapi_*）へ処理を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c（初期化）、src/backend/libpq/auth.c（認証）、および contrib/ 配下の認証関連モジュール（auth_delay.c, sepgsql/label.c, auth-oauth.c, auth-sasl.c, auth-scram.c など）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しの起点ではありません。

関連データ構造

  • SockAddr: ソケットアドレス情報。
  • HbaLine: ホストベース認証設定。
  • pg_gssinfo: GSSAPI 認証状態。
  • List: GUC オプション保持用。

失敗モードと境界条件

  • raw_buf の管理は、SSL 接続確立時の「読み込みすぎ」データに対する一時的な回避策であり、誤った操作は通信の不整合を招く可能性があります。
  • default_* 系の値が -1 の場合、getsockopt によるデフォルト値の取得に失敗したことを示します。

保守担当者向けメモ

  • 拡張機能開発者は、内部構造を直接操作する際に ABI 互換性に注意してください。
  • application_name は接続認証時のログ用であり、認証後の変更には GUC を使用してください。
