sset

定義

  • 種別: struct
  • 定義箇所: src/backend/regex/regexec.c:45

目的

正規表現エンジンにおける「遅延評価DFA（決定性有限オートマトン）」の状態セットを保持・管理するために使用されます。

シグネチャまたは形状

struct sset {
    unsigned   *states;         /* ビットベクトルへのポインタ */
    unsigned    hash;           /* ビットベクトルのハッシュ値 */
    int         flags;          /* 状態フラグ (STARTER, POSTSTATE, LOCKED, NOPROGRESS) */
    struct arcp ins;            /* ここを指す入力アークのチェーン */
    chr        *lastseen;       /* この状態に到達した際の最後の文字位置 */
    struct sset **outs;         /* カラー（文字クラス）でインデックスされた出力アークベクトル */
    struct arcp *inchain;       /* 出力アーク用のチェーンポインタベクトル */
};

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではないため直接的な入出力はありません。
  • rege_dfa.c 内のロジックにより、マッチング実行時に動的に初期化・参照されます。

前提条件と不変条件

  • hash フィールドは、states が指すビットベクトルの内容と整合している必要があります。
  • flags には以下のいずれか、または組み合わせが設定されます：
  • STARTER: 初期状態セット。
  • POSTSTATE: ゴール状態を含む。
  • LOCKED: キャッシュから削除されないようロックされている。
  • NOPROGRESS: 進捗のない状態セット。

アルゴリズムまたは振る舞い

  • **ハッシュ照合**: HASH マクロによりビットベクトルのハッシュを計算し、HIT マクロにより既存の sset が特定のハッシュおよびビットベクトルと一致するかを判定します。
  • **状態遷移**: outs ベクトルを介して、入力文字のカラーに応じた次の sset への遷移を管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: 主に src/backend/regex/rege_dfa.c 内の関数群が、DFA の構築および状態遷移の過程で本構造体を生成・操作します。
  • **呼び出し先**: 本構造体はデータ構造であるため、特定の関数を呼び出すことはありませんが、rege_dfa.c 内のロジックによってフィールドが読み書きされます。

関連データ構造

  • struct dfa: DFA 全体の状態管理を行う構造体。
  • struct arcp: アーク（遷移）のチェーンを管理する構造体。
  • struct smalldfa: 小規模な正規表現用の最適化された DFA 構造体。

失敗モードと境界条件

  • メモリ不足: rege_dfa.c において MALLOC_ARRAY を使用して確保されるため、メモリ枯渇時に失敗する可能性があります。
  • キャッシュ溢れ: LOCKED フラグが付与されていない状態セットは、DFA のキャッシュ制限に達した際に再利用または破棄される可能性があります。

保守担当者向けメモ

  • 本構造体は「遅延評価」の要です。rege_dfa.c での動的な状態生成ロジックを変更する際は、flags の整合性と outs ベクトルのメモリ管理に注意してください。
  • HIT マクロは memcmp を使用してビットベクトルを比較するため、states が指すメモリ領域のサイズ（nw）が正しく管理されていることが不可欠です。
