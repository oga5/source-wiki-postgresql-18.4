ExplainPropertyFloat

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_format.c:249

目的

浮動小数点数値を EXPLAIN 出力形式（TEXT, JSON, XML, YAML）に合わせて整形し、指定されたラベルと単位を付与して出力する。

シグネチャまたは形状

void ExplainPropertyFloat(const char *qlabel, const char *unit, double value, int ndigits, ExplainState *es)

入力と出力

  • qlabel (入力): プロパティのラベル名。
  • unit (入力): 数値の単位（例: "ms"）。NULL 指定可能。
  • value (入力): 出力対象の浮動小数点数値。
  • ndigits (入力): 小数点以下の表示桁数。
  • es (入力/出力): ExplainState 構造体。出力バッファおよびフォーマット状態を保持。
  • 出力: 戻り値なし。副作用として es が保持するバッファに整形済みの文字列が書き込まれる。

前提条件と不変条件

  • es は初期化済みであること。
  • ndigits は psprintf の精度指定として有効な値であること。

アルゴリズムまたは振る舞い

1. psprintf を使用し、ndigits で指定された精度で value を文字列 buf に変換する。 2. ExplainProperty を呼び出し、整形済み文字列 buf を渡して出力処理を委譲する。 3. pfree(buf) を呼び出し、一時的に確保したメモリを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/explain.c（実行計画の統計情報出力時）、contrib/pg_overexplain/pg_overexplain.c。
  • 呼び出し先: psprintf (メモリ確保とフォーマット), ExplainProperty (出力層への引き渡し), pfree (メモリ解放)。

関連データ構造

  • ExplainState: 出力形式や現在のネスト状態を管理する構造体。

失敗モードと境界条件

  • メモリ不足: psprintf がメモリ確保に失敗した場合、PostgreSQL のエラーハンドリング機構によりアボートされる可能性がある。
  • ndigits が負の値の場合の psprintf の挙動は、標準ライブラリの仕様に依存する。

保守担当者向けメモ

  • 本関数は ExplainProperty のラッパーであり、数値の文字列変換を担う。
  • フォーマットごとのエスケープや構造化（JSONのキー/値など）は ExplainProperty 内部で処理されるため、本関数で直接フォーマットを意識する必要はない。
