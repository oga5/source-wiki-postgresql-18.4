KeySuffix

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/formatting.c:122

目的

TO_CHAR や TO_DATE 等のフォーマット文字列解析において、キーワードに付随する接尾辞（suffix）を定義・識別するために使用されます。

シグネチャまたは形状

typedef struct {
    const char *name;    /* 接尾辞文字列 */
    int         len;     /* 接尾辞の長さ */
    int         id;      /* node->suffix に格納される識別子 */
    int         type;    /* 接頭辞(prefix)か接尾辞(postfix)かの区分 */
} KeySuffix;

入力と出力

本構造体自体はデータ定義であり、関数ではありません。解析処理において、入力された文字列と KeySuffix の配列を照合し、一致する構造体を返却する形式で利用されます。

前提条件と不変条件

  • name はヌル終端文字列である必要があります。
  • len は name の長さを正しく反映している必要があります。

アルゴリズムまたは振る舞い

フォーマット文字列の解析時、suff_search() 関数などがこの構造体の配列（DCH_suff 等）を走査し、指定された文字列が定義済みの接尾辞と一致するかを判定します。一致した場合、対応する id がノード情報として保持されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/formatting.c 内の suff_search() 関数などが、フォーマット解析の過程で本構造体を参照します。
  • **呼び出し先**: 本構造体はデータ定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • FormatNode: フォーマット文字列の解析結果を保持するツリー構造体。
  • DCH_suff: KeySuffix 型の静的配列。フォーマット解析時に参照される具体的な接尾辞定義群。

失敗モードと境界条件

  • 該当する接尾辞が見つからない場合、検索関数は NULL を返します。
  • len が実際の文字列長と不一致の場合、解析ロジックが誤ったトークンを識別する可能性があります。

保守担当者向けメモ

  • 新しいフォーマット接尾辞を追加する場合は、DCH_suff 配列を更新し、適切な id を割り当ててください。
  • type フィールドは、解析対象のフォーマットが接頭辞として扱われるか接尾辞として扱われるかを制御するため、追加時には既存の定義と整合性を取る必要があります。
