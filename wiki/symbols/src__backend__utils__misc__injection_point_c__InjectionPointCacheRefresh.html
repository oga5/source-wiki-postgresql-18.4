InjectionPointCacheRefresh

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/injection_point.c:421

目的

共有メモリ上のインジェクションポイント情報を検索し、バックエンドローカルなキャッシュを最新の状態に更新または同期する。

シグネチャまたは形状

static InjectionPointCacheEntry *InjectionPointCacheRefresh(const char *name);

入力と出力

  • **入力**: name (検索対象のインジェクションポイント名)
  • **出力**: 成功時は InjectionPointCacheEntry へのポインタを返し、該当なしまたは削除時は NULL を返す。

前提条件と不変条件

  • 本機能は --enable-injection-points ビルド時のみ有効。
  • 共有メモリ上の ActiveInjectionPoints が初期化されていること。
  • 世代管理カウンタ (generation) を用いたロックフリーな読み取りを前提とする。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: 指定された name がローカルキャッシュに存在する場合、共有メモリ上の対応するエントリの世代カウンタと比較する。一致すればキャッシュをそのまま返す。 2. **共有メモリ走査**: キャッシュミスまたは世代不一致の場合、共有メモリ上の全スロットを走査する。 3. **整合性検証**:

  • 世代カウンタが偶数のスロットは空とみなしてスキップする。
  • 読み取り前後に pg_read_barrier() を配置し、メモリの整合性を確保する。
  • 読み取り中に世代カウンタが変化した場合は、当該エントリを破棄して走査を継続する。

    • 4. **キャッシュ更新**: 一致したエントリをローカルメモリにコピーし、injection_point_cache_load を介してキャッシュに登録して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • InjectionPointRun (実行時)
  • InjectionPointCached (キャッシュ確認時)
  • IsInjectionPointAttached (存在確認時)
  • **呼び出し先**:
  • injection_point_cache_get
  • injection_point_cache_remove
  • injection_point_cache_load
  • hash_destroy

関連データ構造

  • InjectionPointEntry: 共有メモリ上のエントリ定義。
  • InjectionPointCacheEntry: バックエンドローカルなキャッシュエントリ。
  • ActiveInjectionPoints: 共有メモリ上の管理構造体。

失敗モードと境界条件

  • **共有メモリ空**: max_inuse が 0 の場合、キャッシュを破棄して NULL を返す。
  • **競合**: 走査中にインジェクションポイントが動的に追加・削除された場合、古い値や NULL を返す可能性があるが、これは設計上許容される。
  • **不一致**: 世代カウンタの不一致が発生した場合は、そのエントリを無視して走査を継続する。

保守担当者向けメモ

  • 本関数はロックフリーな読み取りを行うため、pg_read_barrier() の配置と世代カウンタの比較ロジックには細心の注意を払うこと。
  • 共有メモリ上のエントリは動的に変更される可能性があるため、ローカルコピー (local_copy) を作成した後に再検証を行う手順は必須である。
