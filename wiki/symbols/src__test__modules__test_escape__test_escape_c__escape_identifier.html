escape_identifier

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_escape/test_escape.c:273

目的

PQescapeIdentifier を呼び出し、その結果を PQExpBuffer に格納またはエラー報告を行うためのラッパー関数です。テストスイート内で識別子エスケープ処理を統一的に扱うために使用されます。

シグネチャまたは形状

static bool escape_identifier(PGconn *conn, PQExpBuffer target, const char *unescaped, size_t unescaped_len, PQExpBuffer escape_err);

入力と出力

  • **入力**:
  • conn: PostgreSQL接続ハンドル。
  • target: エスケープ後の文字列を格納するバッファ。
  • unescaped: エスケープ対象の文字列。
  • unescaped_len: 入力文字列の長さ。
  • escape_err: エラー発生時にエラーメッセージを格納するバッファ。
  • **出力**:
  • 成功時に true を返し、target にエスケープ済み文字列を追記します。
  • 失敗時に false を返し、escape_err にエラーメッセージを格納します。

前提条件と不変条件

  • conn は有効な接続状態である必要があります。
  • PQescapeIdentifier がメモリを割り当てるため、成功時は PQfreemem による解放が必須です。

アルゴリズムまたは振る舞い

1. PQescapeIdentifier を呼び出し、識別子をエスケープします。 2. 結果が NULL の場合（失敗）:

  • PQerrorMessage を取得し、escape_err に追記します。
  • 末尾の改行文字を除去する調整を行います。
  • false を返します。

    • 3. 結果が有効な場合（成功）:

  • target バッファにエスケープ済み文字列を追記します。
  • PQfreemem で割り当てられたメモリを解放します。
  • true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_escape/test_escape.c:408 （pe_test_escape_func 構造体の初期化において参照されています）
  • **呼び出し先**:
  • PQescapeIdentifier (libpq)
  • appendPQExpBufferStr
  • PQerrorMessage
  • PQfreemem

関連データ構造

  • PGconn: libpqの接続オブジェクト。
  • PQExpBuffer: 文字列操作用バッファ。
  • pe_test_escape_func: 本関数が登録されているテスト用構造体。

失敗モードと境界条件

  • PQescapeIdentifier がメモリ不足や不正な入力により NULL を返した場合、エラーメッセージの取得とバッファ調整が行われます。
  • 入力文字列の長さが正しく指定されていない場合、PQescapeIdentifier の仕様に従い動作します。

保守担当者向けメモ

  • 本関数はテスト用ラッパーであり、PQescapeIdentifier の戻り値のライフサイクル管理（PQfreemem）を確実に行う役割を担っています。
  • エラーメッセージの末尾改行除去処理は、テスト出力のフォーマットを整えるためのものです。
