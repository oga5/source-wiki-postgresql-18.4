xl_smgr_truncate

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/storage_xlog.h:46

目的

リレーションのファイル切り詰め（truncate）操作をWAL（Write-Ahead Logging）に記録するための構造体です。リカバリ時に物理ストレージの状態を整合させるために使用されます。

シグネチャまたは形状

typedef struct xl_smgr_truncate
{
    BlockNumber blkno;      /* 切り詰め後のブロック数（または切り詰め位置） */
    RelFileLocator rlocator; /* 対象リレーションの物理位置情報 */
    int         flags;      /* 切り詰め対象フォークを示すフラグ */
} xl_smgr_truncate;

入力と出力

本構造体はWALレコードのデータ部として書き込まれ、リカバリプロセスにおいて smgr_redo 関数によって読み取られます。

前提条件と不変条件

  • flags フィールドは、メインデータ、可視性マップ（VM）、空き領域マップ（FSM）のどのフォークを対象とするかを制御します。
  • リカバリ時には、本構造体に含まれる rlocator と blkno を用いて、物理ファイルに対して適切な切り詰め操作を再実行する必要があります。

アルゴリズムまたは振る舞い

1. ストレージ層で切り詰めが発生すると、本構造体に情報を格納しWALへ書き出します。 2. リカバリ時、XLogReader を介してレコードを読み込みます。 3. smgr_redo がレコードを解釈し、指定されたリレーションの物理ファイルを blkno まで切り詰めます。

呼び出し元と呼び出し先

  • **呼び出し元（記録側）**: src/backend/catalog/storage.c 等のストレージ操作関数。
  • **呼び出し先（解釈側）**: src/backend/access/rmgrdesc/smgrdesc.c（デバッグ出力）、src/backend/catalog/storage.c（smgr_redo による再実行）、src/backend/access/xlogprefetcher.c、src/backend/postmaster/walsummarizer.c。
  • ※上記はソースコード内の参照箇所に基づく推論であり、直接的な関数呼び出し関係の全容ではありません。

関連データ構造

  • RelFileLocator: リレーションを特定するための識別子。
  • BlockNumber: ストレージ内のブロック位置。
  • XLOG_SMGR_TRUNCATE: 本構造体に関連付けられたWALレコード識別子。

失敗モードと境界条件

  • 不正な blkno や rlocator が指定された場合、リカバリ時に物理ファイルの一貫性が損なわれる可能性があります。
  • flags の解釈に誤りがあると、意図しないフォーク（VMやFSMなど）が切り詰められるリスクがあります。

保守担当者向けメモ

  • 本構造体はファイル「削除」には関与しません（削除は xact.c 側で管理）。
  • flags の定義（SMGR_TRUNCATE_ALL 等）は storage_xlog.h 内の関連定数を参照してください。
  • リカバリ時の物理的な一貫性維持において、本構造体の各フィールドの正確な解釈が極めて重要です。
