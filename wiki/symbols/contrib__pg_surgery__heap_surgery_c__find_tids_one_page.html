find_tids_one_page

定義

  • 種別: function
  • 定義箇所: contrib/pg_surgery/heap_surgery.c:397

目的

ソート済みのTID（タプル識別子）配列から、同一のヒープページ（ブロック）に属するTIDの範囲を特定し、そのブロック番号を返します。

シグネチャまたは形状

static BlockNumber find_tids_one_page(ItemPointer tids, int ntids, OffsetNumber *next_start_ptr)

入力と出力

  • **入力**:
  • tids: ソート済みの ItemPointer 配列。
  • ntids: 配列内のTID総数。
  • next_start_ptr: 処理を開始する配列のインデックスを指すポインタ。
  • **出力**:
  • 戻り値: 同一ブロックに属するTID群の BlockNumber。
  • next_start_ptr: 次の処理開始位置（現在のブロックに含まれない最初のTIDのインデックス）が更新されます。

前提条件と不変条件

  • 入力される tids 配列は、ブロック番号順にソートされている必要があります。
  • next_start_ptr が指す値は、配列の有効なインデックス範囲内である必要があります。

アルゴリズムまたは振る舞い

1. next_start_ptr で指定された位置から配列を走査します。 2. 最初のTIDのブロック番号を基準（prev_blkno）として保持します。 3. 後続のTIDを順次確認し、ブロック番号が prev_blkno と一致しなくなるまで走査を続けます。 4. 異なるブロック番号に到達した時点、または配列の末尾に到達した時点で走査を終了します。 5. 最後に next_start_ptr を走査終了位置に更新し、特定した prev_blkno を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_surgery/heap_surgery.c 内の heap_force_common（163行目付近）から呼び出されます。
  • **呼び出し先**: ItemPointerGetBlockNumberNoCheck を使用してTIDからブロック番号を抽出します。

関連データ構造

  • ItemPointer: タプルを特定するためのTID構造体。
  • BlockNumber: ヒープ内のページ番号を識別する型。

失敗モードと境界条件

  • ntids が 0 の場合、または next_start_ptr が ntids を超えている場合、ループは実行されず InvalidBlockNumber が返されます。
  • すべてのTIDが同一ブロックに属する場合、next_start_ptr は ntids に更新されます。

保守担当者向けメモ

  • 本関数は heap_force_common におけるページ単位のバッチ処理を支えるユーティリティです。TID配列がソートされていない場合、期待通りのグループ化が行われないため、呼び出し側でソートが保証されていることを確認してください。
