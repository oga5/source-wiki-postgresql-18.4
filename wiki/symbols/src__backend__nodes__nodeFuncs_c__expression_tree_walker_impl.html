expression_tree_walker_impl

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/nodeFuncs.c:2088

目的

ノードツリーを再帰的に走査し、各ノードに対して指定されたコールバック関数（walker）を適用する汎用的な走査基盤を提供します。

シグネチャまたは形状

bool expression_tree_walker_impl(Node *node, tree_walker_callback walker, void *context);

入力と出力

  • **入力**:
  • node: 走査を開始するノード（Node *）。
  • walker: 各ノードで呼び出されるコールバック関数。
  • context: コールバックに渡される任意のデータポインタ。
  • **出力**:

前提条件と不変条件

  • walker は、現在のノードに対して既に呼び出し済みであることを前提としています。本関数は、そのノードが保持するサブノード（子ノード）に対して再帰的に適用されます。
  • スタックオーバーフローを防止するため、内部で check_stack_depth() を呼び出します。

アルゴリズムまたは振る舞い

1. node が NULL の場合は false を返します。 2. check_stack_depth() を実行し、再帰の深さを検証します。 3. nodeTag(node) に基づき、ノードの種類に応じたサブノードの走査を行います。

  • T_Var 等のプリミティブなノードは何もせず終了します。
  • Aggref や FuncExpr 等の複雑なノードは、保持するリスト（args 等）に対して LIST_WALK マクロを用いて再帰的に走査します。
  • SubLink 等の特定のノードでは、サブクエリの Query ノードに対しても再帰的に走査を行います。

    • 4. 走査中に walker が true を返した場合は、その時点で処理を終了し true を伝播させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/nodes/nodeFuncs.h で定義されるインターフェースを通じて呼び出されます。
  • **呼び出し先**: 引数として渡された walker 関数を呼び出します。また、リスト構造を処理するために自身を再帰的に呼び出します。

関連データ構造

  • Node: 走査対象となるツリーの基本構造体。
  • ListCell: リスト構造の走査に使用されます。
  • 各種ノード構造体（Aggref, FuncExpr, SubLink 等）: nodeTag に応じてキャストされ、サブノードへのアクセスに使用されます。

失敗モードと境界条件

  • **スタックオーバーフロー**: 非常に複雑な式ツリーを走査する場合、check_stack_depth() によりエラーが発生する可能性があります。
  • **NULL ポインタ**: node が NULL の場合は安全に false を返します。

保守担当者向けメモ

  • 新しいノード型を PostgreSQL に追加した際は、本関数の switch 文に適切なケースを追加し、そのノードが持つサブノードを走査対象に含める必要があります。
  • LIST_WALK マクロはリストノード自体を走査対象とせず、リスト内の要素に対してのみ walker を適用する設計となっています。
