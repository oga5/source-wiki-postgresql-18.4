spgConfigOut

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist.h:41

目的

SP-GiSTインデックスの演算子クラス（opclass）が、インデックスの構成情報（データ型や機能のサポート状況）をアクセスメソッドに通知するための構造体です。

シグネチャまたは形状

typedef struct spgConfigOut
{
    Oid         prefixType;     /* 内部タプル接頭辞のデータ型 */
    Oid         labelType;      /* 内部タプルノードラベルのデータ型 */
    Oid         leafType;       /* リーフタプル値のデータ型 */
    bool        canReturnData;  /* 元データの再構築が可能か */
    bool        longValuesOK;   /* 1ページを超える値を扱えるか */
} spgConfigOut;

入力と出力

本構造体は、演算子クラスの config サポート関数によって値が設定され、SP-GiSTアクセスメソッドへ出力されます。

前提条件と不変条件

  • 各 Oid フィールドには、有効なデータ型のOIDが格納されている必要があります。
  • canReturnData および longValuesOK は、実装する演算子クラスの特性に応じて適切にブール値を設定する必要があります。

アルゴリズムまたは振る舞い

SP-GiSTインデックスの作成時や検証時に、アクセスメソッドが演算子クラスの config 関数を呼び出し、この構造体を通じてインデックスの挙動を決定します。

呼び出し元と呼び出し先

  • **呼び出し元（設定側）**:
  • src/backend/access/spgist/spgkdtreeproc.c
  • src/backend/access/spgist/spgquadtreeproc.c
  • src/backend/access/spgist/spgtextproc.c
  • src/backend/utils/adt/geo_spgist.c
  • src/backend/utils/adt/network_spgist.c
  • src/backend/utils/adt/rangetypes_spgist.c
  • src/test/modules/spgist_name_ops/spgist_name_ops.c
  • **呼び出し元（検証・利用側）**:
  • src/backend/access/spgist/spgvalidate.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出し先はありません。

関連データ構造

  • spgConfigIn: 対応する入力用構造体。
  • spgist_private.h 内の構造体（config フィールドとして保持）。

失敗モードと境界条件

  • 演算子クラスが config 関数内で本構造体の各フィールドを適切に初期化しなかった場合、インデックスの動作が未定義となります。

保守担当者向けメモ

  • 本構造体はSP-GiSTの演算子クラス開発におけるインターフェースの根幹です。フィールドの追加や変更を行う際は、既存のすべての演算子クラス実装（geo_spgist.c 等）の更新が必要です。
