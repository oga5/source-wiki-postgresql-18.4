binary_upgrade_set_next_array_pg_type_oid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_upgrade_support.c:63

目的

pg_upgrade 実行時、次に作成される配列型の OID をシステムカタログへ強制的に割り当てるための予約値を設定する。

シグネチャまたは形状

Datum binary_upgrade_set_next_array_pg_type_oid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) により、予約対象となる OID を受け取る。
  • **出力**: なし（void）。

前提条件と不変条件

  • CHECK_IS_BINARY_UPGRADE マクロにより、サーバーが binary_upgrade モードで起動されていることが必須条件。モード外で呼び出された場合はエラーとなる。

アルゴリズムまたは振る舞い

1. 引数から OID を取得する。 2. CHECK_IS_BINARY_UPGRADE を実行し、アップグレードモードであることを検証する。 3. グローバル変数 binary_upgrade_next_array_pg_type_oid に取得した OID を代入する。 4. PG_RETURN_VOID() により正常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c において、アップグレード時のダンプ出力の一部として SQL 経由で呼び出される。
  • **呼び出し先**: 本関数内から直接呼び出される関数はない。

関連データ構造

  • binary_upgrade_next_array_pg_type_oid: catalog/binary_upgrade.h で定義されたグローバル変数。本関数によって値が保持される。

失敗モードと境界条件

  • binary_upgrade モード以外で実行された場合、CHECK_IS_BINARY_UPGRADE により処理が中断される。

保守担当者向けメモ

  • 本関数は pg_upgrade のプロセスにおいて、カタログの OID 整合性を維持するために不可欠な管理用インターフェースである。
  • 類似の binary_upgrade_set_next_* 系関数群と同一の定型的な実装パターンに従っている。
