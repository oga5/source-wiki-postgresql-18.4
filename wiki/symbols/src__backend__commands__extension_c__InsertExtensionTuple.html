InsertExtensionTuple

定義

  • 種別: function
  • 定義箇所: src/backend/commands/extension.c:2191

目的

拡張機能のメタデータを pg_extension カタログに登録し、所有者、スキーマ、および依存する他拡張機能との依存関係を記録する。

シグネチャまたは形状

ObjectAddress InsertExtensionTuple(const char *extName, Oid extOwner,
                                   Oid schemaOid, bool relocatable, const char *extVersion,
                                   Datum extConfig, Datum extCondition,
                                   List *requiredExtensions)

入力と出力

  • **入力**: 拡張機能名、所有者Oid、スキーマOid、再配置可否フラグ、バージョン文字列、設定情報（extConfig）、条件情報（extCondition）、依存拡張機能のOidリスト。
  • **出力**: 作成された拡張機能の ObjectAddress を返す。

前提条件と不変条件

  • pg_extension リレーションに対する RowExclusiveLock が保持されていること。
  • 拡張機能のメタデータは pg_extension カタログのスキーマ定義に従って構築される。

アルゴリズムまたは振る舞い

1. pg_extension リレーションを開き、新しいOidを生成する。 2. 入力パラメータを基に pg_extension 用のタプルを構築し、CatalogTupleInsert で挿入する。 3. recordDependencyOnOwner を呼び出し、所有者との依存関係を記録する。 4. requiredExtensions リストを走査し、スキーマおよび各依存拡張機能との依存関係を ObjectAddresses 構造体に収集する。 5. record_object_address_dependencies を使用して、収集した依存関係を pg_depend に一括登録する。 6. InvokeObjectPostCreateHook を呼び出し、作成後のフック処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/extension.c 内の拡張機能作成ロジック、および src/backend/utils/adt/pg_upgrade_support.c からの呼び出しが確認されている。
  • **呼び出し先**: table_open, GetNewOidWithIndex, heap_form_tuple, CatalogTupleInsert, recordDependencyOnOwner, record_object_address_dependencies, InvokeObjectPostCreateHook 等のカタログ操作関数群。

関連データ構造

  • pg_extension (カタログ)
  • ObjectAddress
  • ObjectAddresses

失敗モードと境界条件

  • extConfig または extCondition が NULL の場合、対応するタプル属性は nulls 配列により欠損値として扱われる。
  • 依存関係の登録時に重複するオブジェクトアドレスは自動的に排除される。

保守担当者向けメモ

  • 本関数は拡張機能のインストールプロセスの最終段階でカタログを確定させるために使用される。
  • 依存関係の管理には catalog/dependency.h の機能を利用しているため、依存関係の定義変更時には関連するカタログ操作を確認すること。
