PGpipelineStatus

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-fe.h:185

目的

PostgreSQL 接続におけるパイプラインモードの現在の状態を表現するために使用されます。

シグネチャまたは形状

typedef enum
{
	PQ_PIPELINE_OFF,
	PQ_PIPELINE_ON,
	PQ_PIPELINE_ABORTED
} PGpipelineStatus;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • libpq の接続状態の一部として管理されます。
  • パイプラインモードが有効な場合、クエリは非同期に送信され、結果は順次処理されます。

アルゴリズムまたは振る舞い

  • PQ_PIPELINE_OFF: パイプラインモードが無効な状態。
  • PQ_PIPELINE_ON: パイプラインモードが有効な状態。
  • PQ_PIPELINE_ABORTED: パイプライン処理中にエラーが発生し、同期が必要な状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/prompt.c:193 にて PQpipelineStatus(pset.db) の戻り値として参照されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • PGconn: 接続状態を保持する構造体。内部メンバ pipelineStatus (src/interfaces/libpq/libpq-int.h:467) として保持されます。
  • PQpipelineStatus(): この列挙型を返す関数が src/interfaces/libpq/libpq-fe.h:417 で宣言されています。

失敗モードと境界条件

  • パイプラインモードでエラーが発生すると PQ_PIPELINE_ABORTED に遷移します。この状態では、エラーを解消してパイプラインを終了させるまで、新しいクエリの送信は制限されます。

保守担当者向けメモ

  • 本シンボルは libpq のパイプライン処理の制御フローを決定する重要な状態変数です。
  • PGconn 構造体の内部実装 (libpq-int.h) に依存しているため、直接メンバを操作せず、必ず PQpipelineStatus() 関数を介して状態を取得してください。
