BufTagGetRelFileLocator

定義

  • 種別: function
  • 定義箇所: src/include/storage/buf_internals.h:135

目的

BufferTag からリレーションを識別するための RelFileLocator 構造体を抽出します。

シグネチャまたは形状

static inline RelFileLocator BufTagGetRelFileLocator(const BufferTag *tag)

入力と出力

  • **入力**: const BufferTag *tag （対象のバッファタグへのポインタ）
  • **出力**: RelFileLocator （抽出されたリレーション識別子）

前提条件と不変条件

  • 入力される tag は有効な BufferTag 構造体を指している必要があります。
  • RelFileLocator の relNumber フィールドは、内部の BufTagGetRelNumber 関数を介して tag から導出されます。

アルゴリズムまたは振る舞い

1. 引数 tag から spcOid および dbOid をコピーします。 2. BufTagGetRelNumber(tag) を呼び出し、リレーション番号を取得して rlocator.relNumber に代入します。 3. 構築された RelFileLocator 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 主に src/backend/storage/buffer/bufmgr.c 内の各関数（relpathbackend, smgropen, RelFileLocatorSkippingWAL, relpathperm など）から、バッファタグに基づいたファイルパスの特定やリレーションのオープンに使用されています。
  • **呼び出し先**: BufTagGetRelNumber を内部で呼び出します。

関連データ構造

  • BufferTag: バッファを特定するための識別子。
  • RelFileLocator: リレーションを物理的に特定するための識別子（Tablespace OID, Database OID, RelFileNumber）。

失敗モードと境界条件

  • 入力ポインタ tag が NULL の場合、未定義の動作となります。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • RelFileLocator の構造に変更を加える際は、本関数が正しくフィールドをマッピングしているか確認してください。
