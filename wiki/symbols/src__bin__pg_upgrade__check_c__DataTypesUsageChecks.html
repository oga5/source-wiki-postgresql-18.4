DataTypesUsageChecks

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/check.c:41

目的

pg_upgrade において、旧クラスタ内のデータ型使用状況を検証するためのメタデータ構造体です。バージョンアップに伴う破壊的変更や非推奨となったデータ型の検出ルールを定義します。

シグネチャまたは形状

typedef struct {
    const char *status;
    const char *report_filename;
    const char *base_query;
    const char *report_text;
    int         threshold_version;
    DataTypesUsageVersionCheck version_hook;
} DataTypesUsageChecks;

入力と出力

本構造体自体はデータ定義であり、入力や出力は持ちません。この構造体を用いた検証処理は、base_query を実行して得られた結果に基づき、エラー報告用のファイル（report_filename）へ report_text を出力します。

前提条件と不変条件

  • base_query は、検証対象のデータ型の OID を抽出する有効な SQL クエリである必要があります。
  • version_hook は、特定のチェックが現在の環境に適用可能かどうかを判定する関数ポインタとして機能します。

アルゴリズムまたは振る舞い

data_types_usage_checks 配列として定義された本構造体のインスタンス群を走査し、各チェックを実行します。threshold_version および version_hook を用いて、アップグレード対象のバージョンに対してチェックが必要か否かを動的に判断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c 内の check_for_data_types_usage 関数および関連するループ処理から参照されています。
  • **呼び出し先**: version_hook フィールドを通じて、動的に判定関数が呼び出されます。

関連データ構造

  • data_types_usage_checks[]: 本構造体のインスタンスを保持する静的配列。
  • DataTypesUsageVersionCheck: version_hook で使用される関数ポインタ型。

失敗モードと境界条件

  • base_query が空の結果を返した場合、そのデータ型に関するチェックはスキップされます。
  • version_hook が false を返した場合、そのチェックは適用されません。
  • 実行時にクエリが失敗した場合の挙動は、呼び出し側の check_for_data_types_usage の実装に依存します。

保守担当者向けメモ

  • 新しいデータ型の互換性チェックを追加する場合は、data_types_usage_checks 配列に新しいインスタンスを追加してください。
  • threshold_version は、チェックが適用される最新のバージョンを示します。
  • 複雑な条件判定が必要な場合は、version_hook に適切な判定関数を実装してください。
