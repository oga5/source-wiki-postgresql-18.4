ExecInitHashJoin

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/nodeHashjoin.c:715

目的

ハッシュ結合実行ノード（HashJoinState）の初期化を行い、結合に必要なハッシュテーブル構築用式、結合条件、およびタプルスロットを準備する。

シグネチャまたは形状

HashJoinState *ExecInitHashJoin(HashJoin *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 実行対象の HashJoin プランノード。
  • estate: 実行状態（EState）。
  • eflags: 実行フラグ（EXEC_FLAG_BACKWARD や EXEC_FLAG_MARK はサポート外）。
  • **出力**:
  • 初期化済みの HashJoinState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと。
  • 結合タイプが JOIN_INNER, JOIN_SEMI, JOIN_RIGHT_SEMI, JOIN_LEFT, JOIN_ANTI, JOIN_RIGHT, JOIN_RIGHT_ANTI, JOIN_FULL のいずれかであること。

アルゴリズムまたは振る舞い

1. **状態構造体の作成**: HashJoinState を割り当て、プランノードと実行状態を紐付ける。 2. **子ノードの初期化**: outerPlan（外側）および innerPlan（内側/Hashノード）を ExecInitNode で再帰的に初期化する。 3. **スロットと射影の準備**: 結果タプルスロットおよび外側リレーション用のタプルスロットを初期化する。 4. **結合タイプの判定**: inner_unique または JOIN_SEMI の場合は single_match を設定する。 5. **外部結合用スロット**: JOIN_LEFT, JOIN_ANTI, JOIN_RIGHT, JOIN_FULL の各ケースに応じて、必要な NULL タプルスロットを初期化する。 6. **ハッシュ式の構築**:

  • get_op_hash_functions を用いて、結合キーに対応するハッシュ関数を取得する。
  • ExecBuildHash32Expr を呼び出し、外側および内側のハッシュ値生成用 ExprState を構築する。この際、結合タイプに基づき NULL 入力の扱いを決定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c:308
  • **呼び出し先**:
  • ExecInitNode (子ノード初期化)
  • ExecAssignExprContext, ExecInitResultTupleSlotTL, ExecAssignProjectionInfo
  • ExecInitExtraTupleSlot, ExecInitNullTupleSlot
  • ExecBuildHash32Expr
  • get_op_hash_functions

関連データ構造

  • HashJoinState: 本関数が初期化するメインの実行状態構造体。
  • HashJoin: プランノード。
  • HashState: 内側プラン（Hash ノード）の状態。

失敗モードと境界条件

  • **エラー**:
  • 未知の結合タイプが指定された場合、elog(ERROR) を発行する。
  • 結合演算子に対応するハッシュ関数が見つからない場合、elog(ERROR) を発行する。
  • **制約**: Hash ノードは ExecProcNode() を介してタプルを返さないため、hj_HashTupleSlot は HashState の結果スロットを直接参照する。

保守担当者向けメモ

  • 本関数はシリアル実行用であり、並列クエリが起動された場合は ExecHashJoinInitializeDSM や ExecHashJoinInitializeWorker で並列版に置き換えられる可能性がある。
  • ExecBuildHash32Expr の呼び出しは、ExecHashTableCreate より前に行う必要があるため、この順序を変更しないこと。
