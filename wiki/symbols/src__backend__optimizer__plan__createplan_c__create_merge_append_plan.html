create_merge_append_plan

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/plan/createplan.c:1436

目的

MergeAppendPath を実行可能な MergeAppend プランノードへ変換します。複数の子プランをマージし、指定されたソート順序に従って結果を統合する計画を構築します。

シグネチャまたは形状

Plan *create_merge_append_plan(PlannerInfo *root, MergeAppendPath *best_path, int flags)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • best_path: 変換対象の MergeAppendPath。
  • flags: ターゲットリスト生成時の制約フラグ（CP_EXACT_TLIST 等）。
  • **出力**:
  • 変換された Plan ノード（MergeAppend または射影が必要な場合は ProjectionPath を含む Plan）。

前提条件と不変条件

  • best_path->path.param_info は NULL であること（パラメータ化された MergeAppend は未サポート）。
  • すべての子プランは、MergeAppend が期待する同一のターゲットリストを返す必要がある。
  • ソートキー情報（列番号、演算子、照合順序、NULL順序）は、親ノードとすべての子プラン間で一致しなければならない。

アルゴリズムまたは振る舞い

1. MergeAppend ノードを初期化し、ターゲットリストを構築する。 2. prepare_sort_from_pathkeys を使用して、親ノードに必要なソート列情報を計算する。 3. 各子パスに対して再帰的に create_plan_recurse を呼び出し、子プランを生成する。 4. 子プランが要求された順序を満たさない場合、Sort ノードを挿入する。 5. enable_partition_pruning が有効な場合、baserestrictinfo からパーティションプルーニング情報を抽出する。 6. ターゲットリストがソート列追加により変更され、かつ flags で厳密なリストが要求されている場合、inject_projection_plan を介して射影ノードを挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**: create_plan_recurse（src/backend/optimizer/plan/createplan.c:427 付近）。
  • **呼び出し先**:
  • build_path_tlist, makeNode, copy_generic_path_info
  • prepare_sort_from_pathkeys, create_plan_recurse
  • pathkeys_contained_in, make_sort, label_sort_with_costsize
  • extract_actual_clauses, make_partition_pruneinfo
  • inject_projection_plan

関連データ構造

  • MergeAppendPath: 入力となるパス構造体。
  • MergeAppend: 生成されるプランノード。
  • PlannerInfo: プランナのコンテキスト。

失敗モードと境界条件

  • 子プランのターゲットリストが MergeAppend の期待と一致しない場合、elog(ERROR) を発生させる。
  • 射影が必要な場合、inject_projection_plan によりプランツリーが拡張される。

保守担当者向けメモ

  • MergeAppend ノード自体の作成関数（make_merge_append のようなもの）は分離されておらず、本関数内で直接構築される。これはソート情報の整合性チェックを容易にするためである。
  • tlist_was_changed フラグは、ソートのために追加された列を最終出力から取り除く必要があるかを判断するために使用される。
