SPLITCOST

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsgistidx.c:579

目的

GiST インデックスのページ分割（picksplit）処理において、各エントリの分割コストと位置情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct
{
	OffsetNumber pos;
	int32		cost;
} SPLITCOST;

入力と出力

  • pos (OffsetNumber): インデックスページ内のエントリ位置。
  • cost (int32): 分割アルゴリズムによって算出された、そのエントリを移動させる際のコスト値。

前提条件と不変条件

  • 本構造体は、GiST インデックスの分割最適化ロジックにおいて、コストに基づくソートを行うために使用されます。
  • cost 値が小さいほど、そのエントリを特定のページに配置することが望ましい（あるいは分割に適している）ことを示します。

アルゴリズムまたは振る舞い

  • picksplit 処理中に、インデックス内の各エントリに対して SPLITCOST が割り当てられます。
  • 算出されたコストに基づき、qsort 等を用いてエントリを並べ替え、最適な分割境界を決定するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_gist.c および contrib/intarray/_int_gist.c において、picksplit 実装の一部として本構造体が使用されています。
  • **呼び出し先**: qsort などの標準ライブラリ関数が、本構造体の cost フィールドを比較するために呼び出されます。

関連データ構造

  • OffsetNumber: PostgreSQL のインデックスページ内でのエントリ位置を示す型。

失敗モードと境界条件

  • maxoff（エントリ数）が 0 の場合、本構造体の配列確保やソート処理は行われません。
  • cost の計算ロジック（各モジュール固有）がオーバーフローや不適切な値を生成した場合、分割の最適化が正しく機能しない可能性があります。

保守担当者向けメモ

  • 本構造体は tsgistidx.c で定義されていますが、hstore や intarray といった外部モジュールでも共通の設計思想で利用されています。変更を加える際は、これら関連モジュールへの影響を確認してください。
