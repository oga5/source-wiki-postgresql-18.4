allocateDataSection

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/SectionMemoryManager.cpp:145

目的

JITコンパイルされたデータセクション（読み取り専用または読み書き可能）に対して、適切なメモリ領域を確保する。

シグネチャまたは形状

uint8_t *allocateDataSection(uintptr_t Size,
                             unsigned Alignment,
                             unsigned SectionID,
                             StringRef SectionName,
                             bool IsReadOnly)

入力と出力

  • **入力**:
  • Size: 確保するメモリサイズ。
  • Alignment: メモリのアライメント要件。
  • SectionID: セクション識別子。
  • SectionName: セクション名。
  • IsReadOnly: 読み取り専用セクションか否かを示すフラグ。
  • **出力**: 確保されたメモリ領域の先頭アドレス（uint8_t *）。

前提条件と不変条件

  • IsReadOnly が true の場合は ROData として、false の場合は RWData としてメモリが割り当てられる。

アルゴリズムまたは振る舞い

1. IsReadOnly フラグを評価する。 2. true の場合、SectionMemoryManager::AllocationPurpose::ROData を指定して allocateSection を呼び出す。 3. false の場合、SectionMemoryManager::AllocationPurpose::RWData を指定して allocateSection を呼び出す。 4. 確保されたメモリ領域のポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/jit/SectionMemoryManager.h:154 に宣言が存在する（具体的な呼び出し箇所はソース抜粋からは不明）。
  • **呼び出し先**: allocateSection を呼び出す。

関連データ構造

  • SectionMemoryManager::AllocationPurpose: メモリ確保の目的を定義する列挙型。

失敗モードと境界条件

  • allocateSection がメモリ確保に失敗した場合の挙動は、当該メソッドの仕様に依存する。

保守担当者向けメモ

  • 本メソッドはLLVMのメモリ管理インターフェースをラップする薄いラッパーである。
  • 確保のロジックを変更する場合は、allocateSection 側の実装を確認すること。
