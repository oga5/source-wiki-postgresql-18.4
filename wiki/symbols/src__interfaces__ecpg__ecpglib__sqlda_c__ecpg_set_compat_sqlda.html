ecpg_set_compat_sqlda

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/sqlda.c:254

目的

構築済みの Informix 互換モード用 SQLDA 構造体に対し、PGresult から取得した特定の行データを格納し、型変換およびメモリ配置を行う。

シグネチャまたは形状

void ecpg_set_compat_sqlda(int lineno, struct sqlda_compat **_sqlda, const PGresult *res, int row, enum COMPAT_MODE compat)

入力と出力

  • **入力**:
  • lineno: エラーログ出力用行番号。
  • _sqlda: 設定対象の sqlda_compat 構造体へのポインタ。
  • res: データソースとなる PGresult オブジェクト。
  • row: 抽出対象の行インデックス。
  • compat: 互換モード指定。
  • **出力**: なし（_sqlda が指す構造体内部の sqldata およびメタデータが更新される）。

前提条件と不変条件

  • row が 0 未満の場合は何も行わない。
  • SQLDA 構造体は、メタデータとデータ領域が単一の連続したメモリブロックとして確保されている必要がある。
  • 各フィールドのデータ配置は、ecpg_sqlda_align_add_size を用いたアライメント計算に従う。

アルゴリズムまたは振る舞い

1. sqlda_compat_empty_size を用いて、データ格納開始位置のオフセットを算出する。 2. 各列（sqld）に対してループ処理を行う：

  • 列の型（sqltype）に基づき、必要なメモリサイズとアライメントを計算し、sqldata ポインタを割り当てる。
  • numeric 型の場合、PGTYPESnumeric_from_asc を介して変換し、バッファを構造体内のメモリ領域にコピーする。
  • その他の型は ecpg_get_data を呼び出し、PGresult から適切な型へ変換して格納する。
  • sqlind（インジケータ）を value_is_null または value_is_not_null に設定する。

    • 3. 処理の過程で offset を更新し、次のフィールドの配置位置を決定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/execute.c
  • **呼び出し先**:
  • sqlda_compat_empty_size
  • ecpg_sqlda_align_add_size
  • ecpg_get_data
  • ECPGset_noind_null
  • PGTYPESnumeric_from_asc / PGTYPESnumeric_free
  • PQgetisnull / PQgetvalue

関連データ構造

  • struct sqlda_compat: Informix 互換モードの SQLDA 構造体。
  • PGresult: libpq のクエリ結果構造体。

失敗モードと境界条件

  • row が無効な場合、関数は早期リターンする。
  • numeric 型の変換に失敗した場合、ECPGset_noind_null を呼び出し、該当フィールドを NULL として扱う。
  • データ長が 32768 を超える場合、sqlilongdata にポインタが設定される。

保守担当者向けメモ

  • 本関数は Informix 互換モード専用である。ネイティブモードの変更時は ecpg_set_native_sqlda を参照すること。
  • メモリレイアウトはメタデータとデータが混在しているため、sqldata ポインタの計算には必ず ecpg_sqlda_align_add_size を使用し、アライメント規則を遵守すること。
