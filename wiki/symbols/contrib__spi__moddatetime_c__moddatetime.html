moddatetime

定義

  • 種別: function
  • 定義箇所: contrib/spi/moddatetime.c:32

目的

レコードの UPDATE 時に、指定されたカラムの値を現在のタイムスタンプで自動更新する PostgreSQL トリガー関数です。

シグネチャまたは形状

Datum moddatetime(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (トリガー実行コンテキスト TriggerData を含む)
  • **出力**: Datum (更新後の HeapTuple を指すポインタ)

前提条件と不変条件

  • BEFORE UPDATE 行レベルトリガーとして呼び出される必要があります。
  • トリガー引数として、更新対象のカラム名を正確に1つ受け取る必要があります。
  • 対象カラムのデータ型は TIMESTAMP または TIMESTAMPTZ である必要があります。

アルゴリズムまたは振る舞い

1. **検証**: CALLED_AS_TRIGGER 等のマクロを用い、関数が適切なトリガーコンテキスト（行レベル、BEFORE、UPDATE）で起動されたかを確認します。 2. **引数取得**: トリガー定義から更新対象のカラム名を取得します。 3. **型確認**: SPI_fnumber および SPI_gettypeid を使用し、対象カラムの存在と型（TIMESTAMP または TIMESTAMPTZ）を検証します。 4. **時刻生成**: DirectFunctionCall3 を介して timestamp_in または timestamptz_in を呼び出し、現在時刻（"now"）を生成します。 5. **タプル更新**: heap_modify_tuple_by_cols を呼び出し、対象カラムを新しいタイムスタンプ値で置換した新しいタプルを構築して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のトリガーマネージャー（直接的な呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: SPI_getrelname, SPI_fnumber, SPI_gettypeid, DirectFunctionCall3, heap_modify_tuple_by_cols, pfree 等の SPI および内部ユーティリティ関数。

関連データ構造

  • TriggerData: トリガー実行時のコンテキスト情報。
  • Trigger: トリガー定義情報。
  • HeapTuple: 更新対象および更新後のタプル。
  • TupleDesc: リレーションのタプル記述子。

失敗モードと境界条件

  • **エラー**: 以下の条件で elog(ERROR) または ereport(ERROR) を発生させます。
  • トリガーとして呼び出されていない場合。
  • 行レベルトリガーではない場合。
  • BEFORE トリガーではない場合。
  • INSERT または DELETE イベントで呼び出された場合。
  • 引数の数が1ではない場合。
  • 指定されたカラムが存在しない場合。
  • 指定されたカラムが TIMESTAMP または TIMESTAMPTZ 型ではない場合。

保守担当者向けメモ

  • 本関数はトランザクション開始時刻ではなく、関数実行時の時刻を記録します。
  • heap_modify_tuple_by_cols を使用してタプルを再構築するため、メモリ管理に注意してください。
  • 呼び出し元と呼び出し先の詳細なコールグラフは、現在のソースコードの静的解析範囲を超えています。
