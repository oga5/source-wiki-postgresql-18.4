SetArchiveOptions

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:274

目的

アーカイブハンドル（Archive）に対して、ダンプおよび復元に関するオプション設定を保持させる。

シグネチャまたは形状

void SetArchiveOptions(Archive *AH, DumpOptions *dopt, RestoreOptions *ropt)

入力と出力

  • **入力**:
  • AH: アーカイブの状態を管理する Archive 構造体へのポインタ。
  • dopt: ダンプオプション（DumpOptions）へのポインタ。
  • ropt: 復元オプション（RestoreOptions）へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • dopt が NULL であり、かつ ropt が非 NULL である場合、dumpOptionsFromRestoreOptions(ropt) を介してダンプオプションが自動的に合成される。

アルゴリズムまたは振る舞い

1. dopt が NULL かつ ropt が有効な場合、dumpOptionsFromRestoreOptions を呼び出してダンプオプションを生成する。 2. 引数として受け取った（または合成された）dopt および ropt を、Archive 構造体のメンバ AH->dopt および AH->ropt にそれぞれ代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_tar.c
  • src/bin/pg_dump/pg_dump.c
  • src/bin/pg_dump/pg_restore.c
  • **呼び出し先**:
  • dumpOptionsFromRestoreOptions（条件付きで呼び出し）

関連データ構造

  • Archive: アーカイブのライフサイクルと状態を保持するメイン構造体。
  • DumpOptions: ダンプ処理の設定を保持する構造体。
  • RestoreOptions: 復元処理の設定を保持する構造体。

失敗モードと境界条件

  • AH が NULL の場合、ポインタ参照によるクラッシュが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、アーカイブ処理の初期化フェーズにおいて、設定情報を永続化するために使用される。
  • dumpOptionsFromRestoreOptions の呼び出しは、ダンプオプションが明示的に提供されない場合のフォールバックとして機能する。
