OutputPluginOptions

定義

  • 種別: struct
  • 定義箇所: src/include/replication/output_plugin.h:26

目的

論理デコーディングにおいて、出力プラグインの動作モードおよび設定を制御するための構造体です。

シグネチャまたは形状

typedef struct OutputPluginOptions
{
	OutputPluginOutputType output_type;
	bool		receive_rewrites;
} OutputPluginOptions;

入力と出力

本構造体は、プラグインの初期化時（startup_cb）にコアエンジンからプラグインへ渡されます。

  • output_type: 出力形式（OutputPluginOutputType 型）。
  • receive_rewrites: 書き換え（rewrite）イベントを受け取るかどうかを示すフラグ。

前提条件と不変条件

  • プラグインの startup_cb コールバック関数内で、プラグイン側がこれらの値を適切に設定することで、以降のデコーディング処理の挙動が決定されます。

アルゴリズムまたは振る舞い

論理デコーディングの開始時、コアエンジンはプラグインの startup_cb を呼び出し、この構造体へのポインタを渡します。プラグインはこの構造体を介して、自身の出力要件をエンジンに伝達します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/logical.c (startup_cb_wrapper)
  • **呼び出し先**:
  • contrib/test_decoding/test_decoding.c (pg_decode_startup)
  • src/backend/replication/pgoutput/pgoutput.c (pgoutput_startup)

    • ※上記は、本構造体を引数として受け取る関数として確認されています。

関連データ構造

  • OutputPluginOutputType: output_type フィールドで使用される列挙型。
  • LogicalDecodingContext: 本構造体が渡される際のコンテキスト構造体。

失敗モードと境界条件

  • 構造体内の各フィールドに不正な値が設定された場合の挙動は、各出力プラグインの実装に依存します。

保守担当者向けメモ

  • 本構造体は論理デコーディングのインターフェースの一部であり、プラグインの互換性に直結します。フィールドの追加や変更を行う際は、既存の全プラグイン実装への影響を考慮してください。
