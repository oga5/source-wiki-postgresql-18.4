bbsink_server_manifest_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_server.c:252

目的

バックアップマニフェストのデータをローカルファイルシステム上のファイルへ書き込み、後続のシンクへ転送する。

シグネチャまたは形状

static void bbsink_server_manifest_contents(bbsink *sink, size_t len)

入力と出力

  • sink: bbsink 構造体へのポインタ（実際には bbsink_server 型としてキャストされる）。
  • len: 書き込むデータのバイト数。
  • 出力: なし（成功時）。失敗時は ereport(ERROR, ...) により処理を中断する。

前提条件と不変条件

  • sink は bbsink_server 型のインスタンスであること。
  • mysink->file が有効なファイル記述子としてオープンされていること。
  • mysink->base.bbs_buffer に書き込むべきデータが格納されていること。

アルゴリズムまたは振る舞い

1. bbsink_server 構造体からファイル記述子とバッファを取得する。 2. FileWrite を呼び出し、バッファの内容を現在のファイル位置 (mysink->filepos) に書き込む。 3. 書き込みバイト数が len と一致しない場合、エラー（ディスクフルまたはファイルアクセスエラー）を報告する。 4. 成功した場合、mysink->filepos を書き込んだバイト数分進める。 5. bbsink_forward_manifest_contents を呼び出し、データを後続のシンクへ転送する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink_server_ops 構造体を通じて、バックアップ処理のメインフローから間接的に呼び出される。
  • 呼び出し先: FileWrite（ストレージ層）、bbsink_forward_manifest_contents（シンクのパイプライン処理）。

関連データ構造

  • bbsink_server: ローカルファイル保存用の状態を保持する構造体。
  • bbsink: 基底となるシンクインターフェース。

失敗モードと境界条件

  • FileWrite が負の値を返した場合: ファイルアクセスエラーとして ERROR を発生させる。
  • FileWrite が len 未満の値を返した場合: ディスクフルと見なし ERRCODE_DISK_FULL で ERROR を発生させる。

保守担当者向けメモ

  • 本関数は bbsink インターフェースの manifest_contents メソッドとして登録されている。
  • 書き込み失敗時のエラーメッセージには FilePathName を使用しており、対象ファイル名が特定可能である。
