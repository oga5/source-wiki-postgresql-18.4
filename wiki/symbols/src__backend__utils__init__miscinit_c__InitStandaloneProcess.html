InitStandaloneProcess

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/miscinit.c:174

目的

PostgreSQLのスタンドアロンモード（Postmasterを介さない単独起動）において、プロセス環境の初期化を行う。

シグネチャまたは形状

void InitStandaloneProcess(const char *argv0)

入力と出力

  • 入力: argv0 (実行ファイル名)
  • 出力: なし（副作用としてグローバル変数およびプロセス環境を初期化）

前提条件と不変条件

  • IsPostmasterEnvironment が偽であること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. プロセスタイプを B_STANDALONE_BACKEND に設定。 2. Windows環境の場合、pgwin32_signal_initialize() を実行。 3. InitProcessGlobals() を呼び出し、プロセスグローバル変数を初期化。 4. 待機イベントサポート、プロセスローカルラッチ、ラッチ待機セットを初期化。 5. pqinitmask() でシグナルマスクを初期化し、sigprocmask でブロックシグナルを適用。 6. 実行パス (my_exec_path) が未設定の場合、find_my_exec を用いて特定。 7. ライブラリパス (pkglib_path) が未設定の場合、get_pkglib_path を用いて特定。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/bootstrap/bootstrap.c
  • src/backend/tcop/postgres.c
  • 呼び出し先:
  • pgwin32_signal_initialize (Windowsのみ)
  • InitProcessGlobals
  • InitializeWaitEventSupport
  • InitProcessLocalLatch
  • InitializeLatchWaitSet
  • pqinitmask
  • sigprocmask
  • find_my_exec
  • get_pkglib_path
  • elog (FATAL時)

関連データ構造

  • MyBackendType (グローバル変数)
  • my_exec_path (グローバル変数)
  • pkglib_path (グローバル変数)

失敗モードと境界条件

  • 実行ファイルのパス特定に失敗した場合、elog(FATAL, ...) を呼び出しプロセスを終了させる。

保守担当者向けメモ

  • InitPostmasterChild との整合性を保つため、シグナルマスクの初期化順序には注意が必要である。
  • Windows環境特有の処理が含まれているため、プラットフォーム固有の挙動変更時には WIN32 マクロの条件分岐を確認すること。
