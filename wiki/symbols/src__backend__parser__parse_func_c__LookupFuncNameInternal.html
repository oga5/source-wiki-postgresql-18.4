LookupFuncNameInternal

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_func.c:2048

目的

指定された名前と引数型に基づき、システムカタログから単一の関数またはプロシージャのOIDを検索する。

シグネチャまたは形状

Oid LookupFuncNameInternal(ObjectType objtype, List *funcname, int nargs, const Oid *argtypes, bool include_out_arguments, bool missing_ok, FuncLookupError *lookupError)

入力と出力

  • **入力**:
  • objtype: 検索対象のオブジェクト種別（OBJECT_FUNCTION, OBJECT_AGGREGATE, OBJECT_PROCEDURE, OBJECT_ROUTINE）。
  • funcname: 関数名のリスト。
  • nargs: 引数の数（-1は引数数不問を意味する）。
  • argtypes: 引数型の配列（nargs > 0 の場合必須）。
  • include_out_arguments: 出力引数を含めるかどうかのフラグ。
  • missing_ok: 見つからない場合にエラーとしないかどうかのフラグ。
  • lookupError: 失敗時のエラーコードを格納するポインタ。
  • **出力**:
  • 成功時: 該当するオブジェクトの Oid。
  • 失敗時: InvalidOid。

前提条件と不変条件

  • argtypes が NULL の場合、nargs は 0 でなければならない。
  • lookupError は必ず初期化される。

アルゴリズムまたは振る舞い

1. FuncnameGetCandidates を呼び出し、名前と引数数に一致する候補リストを取得する。 2. 候補リストを走査し、以下の条件でフィルタリングを行う：

  • nargs >= 0 の場合、argtypes が候補の引数型と一致するかを memcmp で確認する。
  • objtype に応じて、get_func_prokind を用いてプロシージャか否かを判定し、対象外のオブジェクトを除外する。

    • 3. 候補リスト内に InvalidOid が含まれる場合（重複報告）、FUNCLOOKUP_AMBIGUOUS を設定し終了する。 4. 複数の有効な候補が見つかった場合、FUNCLOOKUP_AMBIGUOUS を設定し InvalidOid を返す。 5. 唯一の候補が見つかった場合、その Oid を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_func.c 内の複数の箇所（LookupFuncName, LookupFuncWithArgs 等の内部実装として利用）。
  • **呼び出し先**: FuncnameGetCandidates, get_func_prokind。

関連データ構造

  • ObjectType: オブジェクト種別を定義する列挙型。
  • FuncCandidateList: 検索候補を保持するリスト構造。
  • FuncLookupError: 検索失敗時の詳細を示す列挙型。

失敗モードと境界条件

  • 候補が見つからない場合: lookupError に FUNCLOOKUP_NOSUCHFUNC を設定し InvalidOid を返す。
  • 候補が重複または曖昧な場合: lookupError に FUNCLOOKUP_AMBIGUOUS を設定し InvalidOid を返す。
  • objtype が不正な場合: Assert(false) により停止する。

保守担当者向けメモ

  • 本関数は LookupFuncName 等のラッパー関数から呼び出される内部的な検索ロジックである。
  • nargs が -1 の場合の挙動や、argtypes の比較ロジックは、PostgreSQLの型解決の厳密さに直結するため、変更時は注意が必要である。
