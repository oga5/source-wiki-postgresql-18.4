lo_read

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:244

目的

データベース内のラージオブジェクトから指定されたバイト数を読み込み、クライアント側のバッファに格納する。

シグネチャまたは形状

int lo_read(PGconn *conn, int fd, char *buf, size_t len);

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル。
  • fd: ラージオブジェクトのファイルディスクリプタ。
  • buf: 読み込み先バッファ。
  • len: 読み込むバイト数。
  • **出力**:
  • 成功時: 実際に読み込まれたバイト数（int）。
  • 失敗時: -1。

前提条件と不変条件

  • lo_initialize(conn) が成功している必要がある。
  • len は INT_MAX を超えてはならない（超える場合はエラーとなる）。
  • ラージオブジェクト操作は現在のトランザクション内で実行される。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、サーバー側関数のOIDを解決する。 2. len が INT_MAX を超えていないか検証し、超えていればエラーを記録して終了する。 3. fd と len を PQArgBlock に設定する。 4. PQnfn を介してサーバー側のラージオブジェクト読み込み関数を呼び出す。 5. サーバーからの応答が PGRES_COMMAND_OK であれば、読み込みバイト数を返し、そうでなければ -1 を返す。 6. 実行結果の PGresult は必ず解放される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-lobj.c 内の lo_import 等のストリーミング処理や、外部アプリケーション（pg_dump 等）。
  • **呼び出し先**: lo_initialize（初期化）、PQnfn（サーバー側関数実行）、libpq_append_conn_error（エラー記録）、PQclear（結果解放）。

関連データ構造

  • PGconn: 接続状態とラージオブジェクト関数のOIDキャッシュを保持。
  • PQArgBlock: サーバーへ送信する引数構造体。
  • PGresult: サーバーからの実行結果。

失敗モードと境界条件

  • len が INT_MAX を超える場合、エラーを返し -1 を戻す。
  • サーバー側でエラーが発生した場合、PQresultStatus により検知され -1 を戻す。
  • 失敗時はトランザクションがアボート状態になる可能性がある。

保守担当者向けメモ

  • 引数 len は size_t だが、内部的に int32 へキャストされるため、非常に大きなデータ転送には制限がある。
  • サーバー側のラージオブジェクト関数 fn_lo_read のOIDは PGconn 構造体にキャッシュされる。
