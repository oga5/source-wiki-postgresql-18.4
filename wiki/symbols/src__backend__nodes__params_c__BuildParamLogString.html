BuildParamLogString

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/params.c:334

目的

ParamListInfo 構造体に保持されたパラメータ値を、ログ出力に適したテキスト形式（$1 = 'val1', $2 = 'val2', ...）に変換します。

シグネチャまたは形状

char *BuildParamLogString(ParamListInfo params, char **knownTextValues, int maxlen)

入力と出力

  • **入力**:
  • params: 変換対象のパラメータリスト。
  • knownTextValues: 各パラメータの文字列表現が既知の場合に渡す配列（NULL可）。
  • maxlen: 各パラメータ値の切り詰め上限。
  • **出力**:
  • 変換された文字列を格納した palloc 済みのバッファ。失敗時や条件不適合時は NULL を返します。

前提条件と不変条件

  • params->paramFetch フックが設定されている場合、本関数は処理を行わず NULL を返します。
  • トランザクションがアボート状態（IsAbortedTransactionBlockState()）である場合、処理を行わず NULL を返します。
  • 戻り値の文字列は、ParamListInfo 構造体内部の paramValuesStr フィールドとは独立して生成されます。

アルゴリズムまたは振る舞い

1. initStringInfo で出力用バッファを初期化します。 2. 変換処理中のメモリリークを防ぐため、一時的なメモリコンテキスト（BuildParamLogString）を作成して切り替えます。 3. params->numParams 分だけループし、各パラメータを順次処理します。

  • パラメータが NULL または型が無効な場合は "NULL" を出力します。
  • knownTextValues が提供されている場合はそれを使用し、そうでない場合は OidOutputFunctionCall を用いて Datum を文字列に変換します。
  • 各値は appendStringInfoStringQuoted によりクォートおよび maxlen に基づく切り詰めが行われます。

    • 4. 一時コンテキストを削除し、生成した文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • initStringInfo, appendStringInfo, appendStringInfoString, appendStringInfoStringQuoted
  • AllocSetContextCreate, MemoryContextSwitchTo, MemoryContextDelete
  • getTypeOutputInfo, OidOutputFunctionCall
  • IsAbortedTransactionBlockState

関連データ構造

  • ParamListInfo: パラメータの配列を保持する主要構造体。
  • ParamExternData: 個々のパラメータの型情報や値（Datum）を保持する構造体。
  • StringInfoData: 文字列構築用のバッファ構造体。

失敗モードと境界条件

  • paramFetch フックが使用されている場合、またはトランザクションがアボート状態の場合は NULL を返します。
  • maxlen を超える値は切り詰められます。
  • メモリ不足時は palloc 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はログ出力用であり、ParamListInfo のシリアライズには使用しないでください。
  • paramValuesStr フィールドは本関数の出力結果をキャッシュする目的で使用されることがありますが、本関数自体は常に再計算を行います。
