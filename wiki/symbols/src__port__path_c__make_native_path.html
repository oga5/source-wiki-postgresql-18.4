make_native_path

定義

  • 種別: function
  • 定義箇所: src/port/path.c:235

目的

パス文字列内のディレクトリ区切り文字を、実行環境のネイティブ形式（Windows環境では \）に変換します。

シグネチャまたは形状

void make_native_path(char *filename)

入力と出力

  • **入力**: filename (変更対象のパス文字列へのポインタ)
  • **出力**: なし（引数の文字列を直接書き換える）

前提条件と不変条件

  • filename はヌル終端文字列である必要があります。
  • Windows環境以外（WIN32 が未定義）では、本関数は何も行いません。

アルゴリズムまたは振る舞い

1. WIN32 が定義されている場合、文字列の先頭から末尾までを走査します。 2. 文字列中の / を見つけるたびに、それを \ に置換します。 3. WIN32 が定義されていない場合、処理は行われません。

関連データ構造

  • なし

失敗モードと境界条件

  • filename が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性があります。
  • 文字列内にマルチバイト文字が含まれる場合、特定のエンコーディング（Shift-JIS等）において \（0x5C）が文字の一部として誤認されるリスクについては、本関数単体では考慮されていません。

保守担当者向けメモ

  • 本関数はインプレースで文字列を書き換えます。呼び出し側は、元の文字列が書き換え可能であることを保証してください。
  • Windows 以外のプラットフォームでは実質的に何もしないため、クロスプラットフォームコードでパスをネイティブ形式に統一する際の安全なラッパーとして機能します。
