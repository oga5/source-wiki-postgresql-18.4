Form_pg_transform

定義

  • 種別: typedef
  • 定義箇所: src/include/catalog/pg_transform.h:43

目的

システムカタログ pg_transform の行データ構造である FormData_pg_transform を指すポインタ型を提供し、変換ルール情報のアクセスを抽象化します。

シグネチャまたは形状

typedef FormData_pg_transform *Form_pg_transform;

入力と出力

  • **入力**: GETSTRUCT() マクロ等で取得されたタプルポインタ。
  • **出力**: FormData_pg_transform 構造体へのポインタ。

前提条件と不変条件

  • GETSTRUCT() を介して取得された有効なタプルデータに対してキャストして使用する必要があります。
  • 参照先の FormData_pg_transform は、pg_transform カタログのスキーマ定義と一致している必要があります。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、実行時のロジックは持ちません。主に GETSTRUCT(tup) の戻り値をキャストし、trffromsql や trftosql といった変換関数 OID へのアクセスを容易にするために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/catalog/objectaddress.c: オブジェクトアドレスの解決に使用。
  • src/backend/commands/functioncmds.c: 関数コマンド処理内で使用。
  • src/backend/utils/cache/lsyscache.c: システムキャッシュから変換関数 ID を取得する際に使用。
  • **呼び出し先**:
  • FormData_pg_transform 構造体（直接の参照先）。

関連データ構造

  • FormData_pg_transform: カタログの物理的な行構造。
  • pg_transform: 本構造体が表現するシステムカタログ。

失敗モードと境界条件

  • 無効なタプルポインタに対してキャストを行った場合、メモリ破壊や不正なメモリ参照を引き起こす可能性があります。

保守担当者向けメモ

  • 本型は pg_transform カタログのスキーマ変更に伴い、FormData_pg_transform が更新された際に自動的に追従します。
  • 変換ルール（trffromsql, trftosql）の OID 参照を行う際は、本型を介してアクセスすることが一般的です。
