getProcedureTypeDescription

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaddress.c:4786

目的

指定されたプロシージャ（関数、プロシージャ、または集約）のOIDに基づき、そのオブジェクトの種類を説明する文字列（"aggregate", "procedure", "function" のいずれか）を StringInfo バッファに追加します。

シグネチャまたは形状

static void getProcedureTypeDescription(StringInfo buffer, Oid procid, bool missing_ok)

入力と出力

  • buffer (入出力): 説明文が追記される StringInfo。
  • procid (入力): 対象となるプロシージャのOID。
  • missing_ok (入力): true の場合、OIDが見つからなくてもエラーにせず "routine" を出力して終了します。

前提条件と不変条件

  • procid に対応する pg_proc システムカタログのタプルを PROCOID システムキャッシュから検索します。
  • 処理終了時に ReleaseSysCache を呼び出し、取得したタプルの参照カウントを解放します。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して pg_proc からタプルを取得します。 2. タプルが存在しない場合:

  • missing_ok が false ならばエラーを発生させます。
  • missing_ok が true ならば "routine" をバッファに追加して終了します。

    • 3. タプルが存在する場合、prokind フィールドを確認します:

  • PROKIND_AGGREGATE の場合: "aggregate" を追加。
  • PROKIND_PROCEDURE の場合: "procedure" を追加。
  • それ以外（関数またはウィンドウ関数）の場合: "function" を追加。

    • 4. ReleaseSysCache でタプルを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/objectaddress.c 内のオブジェクト説明生成ロジックから参照されています。
  • 呼び出し先: SearchSysCache1, HeapTupleIsValid, elog, appendStringInfoString, GETSTRUCT, ReleaseSysCache。

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • Form_pg_proc: pg_proc カタログの行構造体。

失敗モードと境界条件

  • missing_ok が false の状態で、存在しない procid が渡された場合、elog(ERROR, ...) によりトランザクションが中断されます。

保守担当者向けメモ

  • 新しいプロシージャの種類が pg_proc に追加された場合、本関数の prokind 分岐条件を更新する必要があります。
