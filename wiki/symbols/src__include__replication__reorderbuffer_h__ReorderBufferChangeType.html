ReorderBufferChangeType

定義

  • 種別: enum
  • 定義箇所: src/include/replication/reorderbuffer.h:50

目的

論理デコーディングにおいて、WALレコードから抽出された変更内容の種類を識別するために使用されます。

シグネチャまたは形状

typedef enum ReorderBufferChangeType
{
    REORDER_BUFFER_CHANGE_INSERT,
    REORDER_BUFFER_CHANGE_UPDATE,
    REORDER_BUFFER_CHANGE_DELETE,
    REORDER_BUFFER_CHANGE_MESSAGE,
    REORDER_BUFFER_CHANGE_INVALIDATION,
    REORDER_BUFFER_CHANGE_INTERNAL_SNAPSHOT,
    REORDER_BUFFER_CHANGE_INTERNAL_COMMAND_ID,
    REORDER_BUFFER_CHANGE_INTERNAL_TUPLECID,
    REORDER_BUFFER_CHANGE_INTERNAL_SPEC_INSERT,
    REORDER_BUFFER_CHANGE_INTERNAL_SPEC_CONFIRM,
    REORDER_BUFFER_CHANGE_INTERNAL_SPEC_ABORT,
    REORDER_BUFFER_CHANGE_TRUNCATE,
} ReorderBufferChangeType;

入力と出力

  • **入力**: WALからデコードされた各変更イベント。
  • **出力**: ReorderBufferChange 構造体の action フィールドとして保持され、後続の処理（出力プラグインへの通知等）の分岐条件として利用されます。

前提条件と不変条件

  • ReorderBufferChange 構造体の一部としてのみ存在し、単独で状態を持つものではありません。
  • 各列挙値は、論理デコーディングエンジンがWALレコードを解析する際に割り当てられます。

アルゴリズムまたは振る舞い

  • 論理デコーディングエンジンは、WALレコードを読み取る際、その内容に応じて本列挙型のいずれかを ReorderBufferChange に設定します。
  • 出力プラグインは、この型を参照して、挿入・更新・削除・メッセージなどの適切な処理を選択します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/pgoutput/pgoutput.c において、変更内容の判定や処理分岐のために参照されています。
  • **呼び出し先**: 本シンボル自体は列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • ReorderBufferChange: 本型を保持するメインの構造体。
  • ReorderBufferTXN: 変更が蓄積されるトランザクション管理構造体。

失敗モードと境界条件

  • 本列挙型に含まれない未知の変更タイプが渡された場合、処理側で未定義の挙動となる可能性があります。

保守担当者向けメモ

  • pgoutput.c のコードコメントにある通り、出力プラグインのプロトコル変換時に本列挙型への依存を避けるためのマッピングが必要になる場合があります。
  • 新しいWALレコードタイプをサポートする際は、本列挙型への追加が必要です。
