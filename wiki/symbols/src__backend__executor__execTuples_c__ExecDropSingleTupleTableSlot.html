ExecDropSingleTupleTableSlot

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execTuples.c:1442

目的

単一の TupleTableSlot を解放し、関連付けられたリソース（タプル、記述子、メモリ領域）を破棄する。

シグネチャまたは形状

void ExecDropSingleTupleTableSlot(TupleTableSlot *slot)

入力と出力

  • **入力**: slot (TupleTableSlot *) - 解放対象のスロット。
  • **出力**: なし。

前提条件と不変条件

  • slot は TupleTableSlot 型の有効なポインタであること（Assert による検証）。
  • ExecResetTupleTable と同様の処理順序に従うこと。

アルゴリズムまたは振る舞い

1. ExecClearTuple(slot) を呼び出し、スロット内の現在のタプルを解放する。 2. slot->tts_ops->release(slot) を呼び出し、スロット固有の解放処理を実行する。 3. slot->tts_tupleDescriptor が存在する場合、ReleaseTupleDesc を呼び出して記述子の参照カウントを減らす。 4. スロットが固定サイズ（TTS_FIXED）でない場合、tts_values および tts_isnull 配列を pfree で解放する。 5. 最後にスロット構造体自体を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/index/genam.c
  • src/backend/access/table/tableam.c
  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/catalog/indexing.c
  • src/backend/catalog/pg_depend.c
  • src/backend/catalog/pg_enum.c
  • **呼び出し先**:
  • ExecClearTuple
  • slot->tts_ops->release
  • ReleaseTupleDesc
  • pfree

関連データ構造

  • TupleTableSlot: タプルを保持するコンテナ。
  • TupleTableSlotOps: スロットの操作を定義する関数テーブル。
  • TupleDesc: タプルのメタデータ。

失敗モードと境界条件

  • slot が NULL の場合、あるいは TupleTableSlot 型でない場合、Assert により異常終了する可能性がある。
  • tts_tupleDescriptor が NULL の場合は解放処理をスキップする。

保守担当者向けメモ

  • 本関数は単一スロット専用である。複数のスロットを管理するテーブル全体を解放する場合は ExecResetTupleTable を使用すること。
  • TTS_FIXED フラグの確認により、動的に割り当てられた属性配列のみを安全に解放する設計となっている。
