_tocEntry

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_dump/pg_backup_archiver.h:348

目的

ダンプおよび復元対象となる各データベースオブジェクトのメタデータ、依存関係、および処理状態を管理するための構造体です。アーカイブ内の各エントリを表現し、TOC（Table of Contents）リストのノードとして機能します。

シグネチャまたは形状

struct _tocEntry
{
    struct _tocEntry *prev;
    struct _tocEntry *next;
    CatalogId    catalogId;
    DumpId       dumpId;
    teSection    section;
    bool         hadDumper;
    char        *tag;
    char        *namespace;
    char        *tablespace;
    char        *tableam;
    char         relkind;
    char        *owner;
    char        *desc;
    char        *defn;
    char        *dropStmt;
    char        *copyStmt;
    DumpId      *dependencies;
    int          nDeps;
    DataDumperPtr dataDumper;
    const void  *dataDumperArg;
    void        *formatData;
    DefnDumperPtr defnDumper;
    const void  *defnDumperArg;
    size_t       defnLen;
    pgoff_t      dataLength;
    int          reqs;
    bool         created;
    struct _tocEntry *pending_prev;
    struct _tocEntry *pending_next;
    int          depCount;
    DumpId      *revDeps;
    int          nRevDeps;
    DumpId      *lockDeps;
    int          nLockDeps;
};

入力と出力

  • **入力**: オブジェクトの識別子（catalogId, dumpId）、定義文、所有者情報、依存関係リストなど。
  • **出力**: アーカイブ処理時に参照されるメタデータおよび、並列復元時の依存関係解決のための状態情報。

前提条件と不変条件

  • prev および next ポインタにより、TOC エントリは循環リストを形成します。
  • pending_prev および pending_next は、並列復元時の保留中アイテムリストを管理するために使用されます。
  • tableam や relkind は、tag が TABLE 型である場合にのみ有効です。

アルゴリズムまたは振る舞い

  • **TOC 管理**: アーカイブ内の全オブジェクトをリスト構造で保持し、ダンプや復元の順序を制御します。
  • **データダンプ/復元**: dataDumper および defnDumper 関数ポインタを介して、オブジェクトのデータや定義を動的に処理します。
  • **並列復元**: depCount（未復元の依存関係数）や revDeps（逆依存関係）を用いて、復元可能なオブジェクトを判定し、並列ジョブとしてスケジュールします。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_dump および pg_restore のメイン処理において、アーカイブの構築や読み込み時にインスタンス化されます。
  • **呼び出し先**: dataDumper および defnDumper に割り当てられた関数ポインタを通じて、各アーカイブ形式（pg_backup_custom.c 等）固有の処理が呼び出されます。

関連データ構造

  • TocEntry (typedef: struct _tocEntry)
  • ArchiveHandle: アーカイブ全体の状態を保持し、TOC リストのヘッダーを管理します。
  • DumpId, CatalogId: オブジェクトを一意に識別するための型。

失敗モードと境界条件

  • dependencies や revDeps のメモリ割り当て失敗。
  • 依存関係の循環参照が発生した場合、depCount がゼロにならず復元が停止する可能性があります。
  • dataLength が 0 の場合、データが存在しないか不明であることを示します。

保守担当者向けメモ

  • 本構造体は pg_backup_archiver.h 内で定義されており、pg_dump の中核的なデータ構造です。
  • 並列復元ロジック（parallel.c）と密接に関連しているため、depCount や revDeps の更新ロジックを変更する際は、並列ジョブのスケジューリングへの影響を考慮してください。
  • formatData は各アーカイブ形式（Custom, Tar, Directory）が独自に拡張データを保持するための領域です。
