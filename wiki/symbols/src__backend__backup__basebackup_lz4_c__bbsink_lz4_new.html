bbsink_lz4_new

定義

  • src/backend/backup/basebackup_lz4.c:61

目的

LZ4圧縮を行う bbsink オブジェクトを生成し、バックアップパイプラインに組み込むためのコンストラクタ。

シグネチャまたは形状

bbsink *bbsink_lz4_new(bbsink *next, pg_compress_specification *compress)

入力と出力

  • **入力**:
  • next: 圧縮後のデータを受け取る後続の bbsink オブジェクト。
  • compress: 圧縮設定（圧縮レベルを含む）を保持する pg_compress_specification 構造体。
  • **出力**:
  • 初期化された bbsink オブジェクトへのポインタ。

前提条件と不変条件

  • next は NULL であってはならない。
  • compress->level は 0 から 12 の範囲内であること。
  • ビルド時に USE_LZ4 が定義されている必要がある。

アルゴリズムまたは振る舞い

1. USE_LZ4 が未定義の場合、エラーを報告する。 2. bbsink_lz4 構造体用にメモリを確保する。 3. bbsink_lz4_ops を用いてシンクの操作テーブルを初期化する。 4. next シンクを bbsink_lz4 オブジェクトの bbs_next に設定し、パイプラインを接続する。 5. 指定された圧縮レベルを保持し、bbsink 基底構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:1046
  • **呼び出し先**: palloc0（PostgreSQLメモリ管理関数）

関連データ構造

  • bbsink_lz4: 圧縮コンテキストやバッファを保持する構造体。
  • bbsink: パイプラインの基底構造体。
  • bbsink_ops: シンクの動作を定義する関数ポインタテーブル。

失敗モードと境界条件

  • USE_LZ4 が定義されていない環境で呼び出された場合、ERRCODE_FEATURE_NOT_SUPPORTED エラーが発生する。
  • next が NULL の場合、Assert により異常終了する。
  • 圧縮レベルが 0〜12 の範囲外の場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は USE_LZ4 マクロに依存しているため、ビルド環境の構成を確認すること。
  • 圧縮レベルの検証は Assert に依存しているため、実行時の入力値の正当性は呼び出し元で保証される必要がある。
