compare_val_int4

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_selfuncs.c:337

目的

int32 型のキーと Datum 型（int32 を保持）の値を比較し、qsort や bsearch 等の標準ライブラリ関数で使用可能な比較結果を返します。

シグネチャまたは形状

static int compare_val_int4(const void *a, const void *b)

入力と出力

  • **入力**:
  • a: 比較対象の int32 値へのポインタ。
  • b: 比較対象の Datum 型値へのポインタ。
  • **出力**:
  • key < value の場合: -1
  • key > value の場合: 1
  • key == value の場合: 0

前提条件と不変条件

  • a は int32 型の値を指している必要があります。
  • b は Datum 型の値を指しており、その Datum は int32 を保持している必要があります。

アルゴリズムまたは振る舞い

1. a が指すメモリを int32 として読み込み、key に格納します。 2. b が指す Datum を DatumGetInt32 マクロで int32 に変換し、value に格納します。 3. key と value を比較し、標準的な比較結果（負、正、ゼロ）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_int_selfuncs.c 内の関数（bsearch 等の比較関数として渡される）。
  • **呼び出し先**: DatumGetInt32 マクロ。

関連データ構造

  • int32
  • Datum

失敗モードと境界条件

  • b が指す Datum が int32 以外の型を保持している場合、誤った比較結果を返します。
  • ポインタ a または b が NULL の場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本関数は qsort や bsearch のコールバックとして設計されています。型変換のロジックを変更する場合は、Datum の内部表現との整合性に注意してください。
