execute_attr_map_cols

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tupconvert.c:251

目的

入力タプルの列集合（Bitmapset）を、属性マッピング情報（AttrMap）に基づいて出力タプルの列集合へ変換する。

シグネチャまたは形状

Bitmapset *execute_attr_map_cols(AttrMap *attrMap, Bitmapset *in_cols)

入力と出力

  • **入力**:
  • attrMap: 属性の対応関係を保持する AttrMap 構造体。
  • in_cols: 入力タプルにおける列番号を示す Bitmapset。
  • **出力**:
  • 変換後の列番号を含む新しい Bitmapset。in_cols が NULL の場合は NULL を返す。

前提条件と不変条件

  • in_cols が NULL の場合、即座に NULL を返す（高速パス）。
  • システム列（負の属性番号）はマッピングされず、そのまま保持される。
  • 削除された列（attrMap で 0 が指定されている列）は出力に含まれない。

アルゴリズムまたは振る舞い

1. out_cols を初期化する。 2. FirstLowInvalidHeapAttributeNumber から attrMap->maplen までループし、各出力列番号（out_attnum）を走査する。 3. out_attnum が負（システム列）であれば、そのまま in_attnum とする。 4. out_attnum が正（ユーザー列）であれば、attrMap を参照して対応する in_attnum を取得する。in_attnum が 0 の場合はスキップする。 5. in_cols 内に in_attnum が存在するかを確認し、存在すれば out_cols に out_attnum を追加する。 6. 最終的な out_cols を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execUtils.c: execUtils.c 内の insertedCols および updatedCols の変換処理で呼び出される。
  • **呼び出し先**:
  • bms_is_member / bms_add_member（Bitmapset 操作用関数）

関連データ構造

  • AttrMap: 属性番号の対応関係を保持する構造体。
  • Bitmapset: 列番号の集合を表現するビットマップ。

失敗モードと境界条件

  • in_cols が NULL の場合は安全に NULL を返す。
  • 属性番号 0 は無効な列としてスキップされる。

保守担当者向けメモ

  • 本関数はシステム列を考慮したビットマップ変換を行うため、通常のタプル変換とは異なるパスを通る点に注意すること。
  • Bitmapset の操作は FirstLowInvalidHeapAttributeNumber を基準としたオフセット計算で行われるため、ビットマップ操作時のインデックス計算に注意が必要。
