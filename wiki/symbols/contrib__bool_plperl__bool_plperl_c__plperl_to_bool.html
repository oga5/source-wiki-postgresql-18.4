plperl_to_bool

定義

  • 種別: function
  • 定義箇所: contrib/bool_plperl/bool_plperl.c:26

目的

Perl のスカラ値（SV）を PostgreSQL の boolean 型へ変換するアダプター関数です。

シグネチャまたは形状

Datum plperl_to_bool(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される、Perl のスカラ値へのポインタ（SV *）。
  • **出力**: Datum 型として返される PostgreSQL の bool 値。

前提条件と不変条件

  • Perl インタプリタのコンテキスト（dTHX）が正しく初期化されている必要があります。
  • 入力値は PG_GETARG_POINTER(0) を介して取得されるため、有効な SV* であることが前提です。

アルゴリズムまたは振る舞い

1. dTHX マクロにより、現在のスレッドの Perl インタプリタコンテキストを取得します。 2. PG_GETARG_POINTER(0) を使用して、引数から Perl のスカラ値（SV*）を取り出します。 3. Perl の SvTRUE マクロを適用し、Perl の真偽値判定ルール（undef、空文字列、数値の 0 は偽、それ以外は真）に従って評価します。 4. 評価結果を PG_RETURN_BOOL を通じて PostgreSQL の bool 型として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • dTHX (Perl マクロ)
  • PG_GETARG_POINTER (PostgreSQL マクロ)
  • SvTRUE (Perl マクロ)
  • PG_RETURN_BOOL (PostgreSQL マクロ)

関連データ構造

  • SV (Perl のスカラ値構造体)
  • Datum (PostgreSQL の汎用データ型)

失敗モードと境界条件

  • SvTRUE の判定ルールに依存するため、Perl 側で undef や空文字列が渡された場合は false として扱われます。
  • 呼び出し元から不正なポインタが渡された場合の挙動は、Perl インタプリタの動作に依存します。

保守担当者向けメモ

  • 本関数はマルチスレッド環境下での動作を想定しているため、dTHX マクロの呼び出しを省略しないでください。
  • Perl の真偽値判定ルールが PostgreSQL の bool 型の期待値と一致していることを確認してください。
