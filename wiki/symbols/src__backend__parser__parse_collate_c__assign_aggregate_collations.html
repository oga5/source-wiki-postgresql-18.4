assign_aggregate_collations

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_collate.c:880

目的

集約関数ノード（Aggref）に含まれる引数に対して、照合順序（Collation）の割り当て処理を行う。

シグネチャまたは形状

static void assign_aggregate_collations(Aggref *aggref, assign_collations_context *loccontext)

入力と出力

  • **入力**:
  • aggref: 処理対象の Aggref 構造体へのポインタ。
  • loccontext: 照合順序の走査状態を保持する assign_collations_context 構造体へのポインタ。
  • **出力**: なし（副作用として aggref 内のノードに照合順序を適用する）。

前提条件と不変条件

  • aggref->aggdirectargs が NIL であること（通常の集約関数であることを前提とする）。

アルゴリズムまたは振る舞い

1. aggref->aggdirectargs が空であることをアサートで確認する。 2. aggref->args リストを走査する。 3. 各要素（TargetEntry）に対し、resjunk フラグを確認する。

  • resjunk が真の場合: assign_expr_collations を呼び出して照合順序を割り当てる。
  • resjunk が偽の場合: assign_collations_walker を再帰的に呼び出し、照合順序を解決・伝播させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_collate.c 内の assign_collations_walker（推測される文脈上の呼び出し元）。
  • **呼び出し先**:
  • assign_expr_collations
  • assign_collations_walker

関連データ構造

  • Aggref: 集約関数を表すノード構造体。
  • assign_collations_context: 照合順序解決のコンテキスト。
  • TargetEntry: 集約関数の引数リストを構成する要素。

失敗モードと境界条件

  • aggref->aggdirectargs が NIL でない場合、アサート失敗により異常終了する。

保守担当者向けメモ

  • 本関数は通常の集約関数を対象としています。順序付き集合集約（Ordered-set aggregates）や仮説的集約（Hypothetical-set aggregates）については、別の専用関数（assign_ordered_set_collations 等）が用意されているため、混同しないよう注意してください。
