libpq_append_conn_error

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq-oauth/oauth-utils.c:94

目的

libpq の接続エラーバッファに対し、翻訳済みのエラーメッセージを安全に追加します。libpq 内部のデータ構造に直接依存せず、注入されたアクセサを介してエラー情報を記録します。

シグネチャまたは形状

void libpq_append_conn_error(PGconn *conn, const char *fmt, ...);

入力と出力

  • **入力**:
  • conn: 操作対象の PGconn 構造体へのポインタ。
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（戻り値なし）。副作用として conn 内部のエラーバッファが更新されます。

前提条件と不変条件

  • fmt で渡される文字列の末尾に改行文字（\n）が含まれていないこと（Assert により検証）。
  • libpq_oauth_init が呼び出され、必要な関数ポインタ（conn_errorMessage 等）が初期化されていること。

アルゴリズムまたは振る舞い

1. 現在の errno を保存します。 2. conn_errorMessage(conn) を呼び出し、対象の PQExpBuffer を取得します。 3. バッファが既に破損（PQExpBufferBroken）している場合は何もせず終了します。 4. libpq_gettext を用いてフォーマット文字列を翻訳し、appendPQExpBufferVA を使用してバッファへ書き込みます。 5. バッファの拡張が必要な場合は、成功するまでループして再試行します。 6. 最後に appendPQExpBufferChar を呼び出し、メッセージの末尾に改行文字を付与します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq-oauth/oauth-curl.c
  • src/interfaces/libpq/fe-auth-oauth.c
  • **呼び出し先**:
  • conn_errorMessage (注入された関数ポインタ)
  • libpq_gettext
  • appendPQExpBufferVA
  • appendPQExpBufferChar
  • PQExpBufferBroken

関連データ構造

  • PGconn: PostgreSQL 接続状態を保持する構造体。
  • PQExpBuffer: 動的に拡張可能な文字列バッファ。

失敗モードと境界条件

  • PQExpBufferBroken が真の場合、エラーメッセージの追加はスキップされます。
  • メモリ不足により PQExpBuffer の拡張に失敗した場合、appendPQExpBufferVA は内部的にバッファを破損状態に設定し、ループを終了させます。

保守担当者向けメモ

  • 本関数は libpq の内部ヘッダー（libpq-int.h）への依存を避けるため、関数ポインタ経由で libpq の機能を利用しています。
  • fmt 引数には pg_attribute_printf(2, 3) が指定されており、コンパイル時にフォーマット文字列の整合性がチェックされます。
