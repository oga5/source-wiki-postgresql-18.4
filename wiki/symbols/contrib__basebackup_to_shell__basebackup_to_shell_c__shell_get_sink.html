shell_get_sink

定義

  • 種別: function
  • 定義箇所: contrib/basebackup_to_shell/basebackup_to_shell.c:126

目的

バックアップ開始時に bbsink インスタンスを初期化し、外部シェルコマンドの実行環境をセットアップします。

シグネチャまたは形状

static bbsink *shell_get_sink(bbsink *next_sink, void *detail_arg)

入力と出力

  • **入力**:
  • next_sink: 次のバックアップシンクへのポインタ。
  • detail_arg: ユーザー指定のターゲット詳細文字列。
  • **出力**:
  • 初期化された bbsink 構造体へのポインタ。

前提条件と不変条件

  • basebackup_to_shell.shell_command GUCパラメータが設定されていること。
  • 外部コマンド文字列に %d が含まれる場合、detail_arg が必須。含まれない場合は detail_arg が存在してはならない。
  • detail_arg が指定されている場合、英数字のみで構成されていること。

アルゴリズムまたは振る舞い

1. bbsink_shell 構造体を割り当て、bbsink_shell_ops を設定。 2. 現在の shell_command を複製して保持（バックアップ中の変更を防止）。 3. コマンド文字列が空でないことを確認。 4. コマンド文字列内の %d プレースホルダーの有無を走査し、detail_arg の有無と整合性を検証。 5. detail_arg が存在する場合、シェルインジェクションを防ぐため、英数字以外の文字が含まれていないかバリデーションを実行。 6. 成功時に bbsink インスタンスを返却。

呼び出し元と呼び出し先

  • **呼び出し元**: BaseBackupAddTarget への登録を通じて、バックアップターゲットフレームワークから呼び出されます。
  • **呼び出し先**: palloc0, pstrdup, ereport などのPostgreSQLメモリ管理およびエラー報告関数。

関連データ構造

  • bbsink: バックアップシンクの基底構造体。
  • bbsink_shell: 外部コマンド実行状態を保持する拡張構造体。
  • bbsink_shell_ops: bbsink インターフェースの関数ポインタテーブル。

失敗モードと境界条件

  • **エラー**:
  • shell_command が空の場合。
  • %d の有無と detail_arg の指定が不一致の場合。
  • detail_arg に英数字以外の文字が含まれる場合。
  • メモリ割り当て失敗時（palloc0 等による）。

保守担当者向けメモ

  • 本関数は popen に渡す文字列を準備するため、セキュリティ上の制約が非常に厳格です。バリデーションロジックを変更する際は、シェルインジェクションのリスクを再評価してください。
