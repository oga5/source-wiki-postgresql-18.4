SnapBuildSnapDecRefcount

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/snapbuild.c:327

目的

論理デコーディングで使用されるヒストリカルスナップショットの参照カウントを減らし、カウントがゼロに達した際にスナップショットを解放する。

シグネチャまたは形状

void SnapBuildSnapDecRefcount(Snapshot snap)

入力と出力

  • **入力**: Snapshot snap（操作対象のスナップショット構造体）
  • **出力**: なし（副作用として参照カウントの減算およびメモリ解放を行う）

前提条件と不変条件

  • snap->snapshot_type が SNAPSHOT_HISTORIC_MVCC であること。
  • snap->curcid が FirstCommandId であること。
  • snap->suboverflowed および snap->takenDuringRecovery が偽であること。
  • snap->regd_count が 0 であること。
  • snap->active_count が 0 より大きいこと。
  • snap->copied が偽であること（コピーされたスナップショットの解放は禁止）。

アルゴリズムまたは振る舞い

1. 前提条件の各項目を Assert で検証する。 2. snap->copied が真であればエラーを投げる。 3. snap->active_count を 1 減算する。 4. 減算後の active_count が 0 になった場合、SnapBuildFreeSnapshot(snap) を呼び出してメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/reorderbuffer.c (複数箇所)
  • src/backend/replication/logical/snapbuild.c (複数箇所)
  • **呼び出し先**:
  • SnapBuildFreeSnapshot (条件付きで呼び出し)
  • elog (エラー発生時)

関連データ構造

  • Snapshot: PostgreSQLのMVCCスナップショット構造体。
  • SnapBuild: 論理デコーディングにおけるスナップショット構築状態を管理する構造体。

失敗モードと境界条件

  • snap->copied が真の場合、elog(ERROR, ...) により処理が中断される。
  • active_count が 0 の状態で呼び出された場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は SNAPSHOT_HISTORIC_MVCC 型のスナップショット専用です。汎用的なスナップショット解放関数ではない点に注意してください。
  • regd_count（登録済みカウント）が 0 であることを前提としているため、登録済みのスナップショットに対しては使用できません。
