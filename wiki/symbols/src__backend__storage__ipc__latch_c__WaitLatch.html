WaitLatch

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/latch.c:171

目的

プロセスが特定のイベント（ラッチのセット、タイムアウト、ポストマスターの終了など）を待機してスリープし、イベント発生時に起床するための同期インターフェースを提供します。

シグネチャまたは形状

int WaitLatch(Latch *latch, int wakeEvents, long timeout, uint32 wait_event_info);

入力と出力

  • **入力**:
  • latch: 待機対象のラッチ構造体へのポインタ。
  • wakeEvents: 待機するイベントの種類（ビットマスク）。
  • timeout: タイムアウト時間（ミリ秒）。
  • wait_event_info: 待機イベントの識別子（統計情報用）。
  • **出力**:
  • 発生したイベントを示すビットマスク、またはタイムアウト発生時に WL_TIMEOUT を返します。

前提条件と不変条件

  • ポストマスター管理下のプロセスは、wakeEvents に WL_EXIT_ON_PM_DEATH または WL_POSTMASTER_DEATH を含める必要があります（Assert による検証）。
  • 呼び出し前に ResetLatch を呼び出し、ラッチの状態をクリアしておくことが推奨されます。

アルゴリズムまたは振る舞い

1. wakeEvents に基づき、LatchWaitSet を更新して待機対象を構成します。 2. IsUnderPostmaster が真の場合、ポストマスターの終了イベントを監視対象に追加します。 3. WaitEventSetWait を呼び出し、OS 固有の待機プリミティブを通じてイベントを待ちます。 4. 待機が解除された後、発生したイベントのビットマスクを返します。タイムアウトした場合は WL_TIMEOUT を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: autoprewarm.c, vacuumlazy.c, parallel.c, xlog.c, xlogfuncs.c, xlogrecovery.c, basebackup_throttle.c, vacuum.c など多数。
  • **呼び出し先**: ModifyWaitEvent, WaitEventSetWait（いずれも src/backend/storage/ipc/waiteventset.c 内の関数）。

関連データ構造

  • Latch: 待機状態を保持する構造体。
  • WaitEvent: 発生したイベント情報を格納する構造体。
  • LatchWaitSet: WaitLatch が内部で使用する待機イベントセット。

失敗モードと境界条件

  • wakeEvents で WL_LATCH_SET が指定されていない場合、latch 引数は無視され NULL として扱われます。
  • タイムアウト値が指定されていない場合、WaitEventSetWait には -1 が渡され、無期限待機となります。

保守担当者向けメモ

  • WaitLatch は LatchWaitSet という静的な待機セットを内部的に利用しています。
  • シグナルハンドラから SetLatch が呼ばれることを想定しているため、呼び出し側で errno の保存・復元が必要になる場合があります。
  • 競合を防ぐため、ループ内での ResetLatch の呼び出し位置には細心の注意を払ってください。
