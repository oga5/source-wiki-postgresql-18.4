ECPGdump_a_simple

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/type.c:387

目的

ECPGプリプロセッサにおいて、単一の変数（または定数）の型情報をCソースコードとして出力し、ECPGライブラリが実行時に解釈可能な形式（型記述子）を生成する。

シグネチャまたは形状

static void ECPGdump_a_simple(
    FILE *o, 
    const char *name, 
    enum ECPGttype type,
    char *varcharsize,
    char *arrsize,
    const char *size,
    const char *prefix,
    int counter
)

入力と出力

  • **入力**:
  • o: 出力先のファイルストリーム。
  • name: 変数名。
  • type: ECPGttype 列挙型の型タグ。
  • varcharsize, arrsize: サイズ情報（文字列）。
  • size: 構造体の一部である場合のサイズ（NULL可）。
  • prefix: 変数名の接頭辞。
  • counter: 自動生成された構造体識別用のカウンタ。
  • **出力**:
  • 指定されたファイルストリーム o に対して、ECPGライブラリ用の型記述子（ECPGt_... で始まる定数やサイズ情報）を書き出す。

前提条件と不変条件

  • ECPGdump_a_simple 内の fprintf フォーマットは、ECPGライブラリ側の実行時読み取り形式と厳密に一致している必要がある。
  • mm_alloc を使用してメモリを動的に確保するため、関数終了時に free を呼び出しメモリを解放する。
  • arrsize が負の値かつ size が NULL の場合、内部的に arrsize を "1" として扱う。

アルゴリズムまたは振る舞い

1. type が ECPGt_NO_INDICATOR, ECPGt_descriptor, ECPGt_sqlda の場合は、それぞれに対応する固定的な型記述子を出力する。 2. それ以外の場合、型に応じて以下の処理を行う：

  • varchar や bytea 等の特殊型、または基本型（char, numeric, date 等）に応じて、変数への参照方法（ポインタ演算の有無）とサイズ計算式（offset）を決定する。
  • ECPGt_const の場合は、定数文字列として出力する。

    • 3. 決定した型タグ、変数名、サイズ、オフセットを fprintf でファイルストリームへ書き出す。 4. 確保した一時メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/type.c 内の ECPGdump_a_struct 等から呼び出される。
  • **呼び出し先**:
  • get_type(type): 型タグに対応する文字列を取得。
  • ecpg_type_name(type): 型名を取得。
  • mm_alloc, free: メモリ管理。
  • fprintf, sprintf, strlen, strcmp, atoi: 標準ライブラリ関数。

関連データ構造

  • ECPGttype: 変数の型を識別する列挙型。
  • ECPGtype: プリプロセッサ内で変数の型情報を保持する構造体。

失敗モードと境界条件

  • arrsize や varcharsize が "0" の場合、文字列比較により配列やポインタの扱いを分岐させるロジックが含まれる。
  • size が NULL または空文字列の場合、size 引数ではなく計算された offset が出力される。

保守担当者向けメモ

  • fprintf の出力形式はECPGライブラリの実行時処理と直結しているため、変更時はライブラリ側の解析ロジックとの整合性を確認すること。
  • mm_alloc によるメモリ確保を行っているため、将来的なロジック追加時にはメモリリークに注意が必要。
