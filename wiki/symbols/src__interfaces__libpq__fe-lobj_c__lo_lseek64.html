lo_lseek64

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:384

目的

データベース内のラージオブジェクトに対して、64ビットのオフセット指定によるシーク操作を行う。

シグネチャまたは形状

int64_t lo_lseek64(PGconn *conn, int fd, int64_t offset, int whence);

入力と出力

  • **入力**:
  • conn: データベース接続ハンドル。
  • fd: ラージオブジェクトのファイルディスクリプタ。
  • offset: シーク先のオフセット値（64ビット）。
  • whence: シークの基準点（SEEK_SET, SEEK_CUR, SEEK_END 等）。
  • **出力**:
  • 成功時: シーク後の現在位置（64ビット整数）。
  • 失敗時: -1。

前提条件と不変条件

  • lo_initialize(conn) が成功している必要がある。
  • サーバー側で lo_lseek64 関数がサポートされていること（conn->lobjfuncs->fn_lo_lseek64 が有効であること）。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、サーバー側関数のOID解決を行う。 2. サーバー側関数 lo_lseek64 のOIDが未解決の場合はエラーを返し終了する。 3. 引数（fd、ネットワークバイトオーダーに変換した offset、whence）を PQArgBlock 配列に格納する。 4. PQnfn を介してサーバー側の lo_lseek64 を実行する。 5. サーバーからの応答が正常（PGRES_COMMAND_OK）かつ結果長が8バイトである場合、結果をホストバイトオーダーに変換して返す。 6. それ以外の場合はエラーとして -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/examples/testlo64.c 等のクライアントアプリケーション。
  • **呼び出し先**: lo_initialize（内部初期化）、PQnfn（Fastpath関数実行）、lo_hton64/lo_ntoh64（バイトオーダー変換）。

関連データ構造

  • PGconn: 接続状態およびラージオブジェクト関数OIDのキャッシュを保持。
  • PQArgBlock: サーバーへ送信する引数構造体。

失敗モードと境界条件

  • 接続が確立されていない、または初期化に失敗した場合は -1 を返す。
  • サーバー側で lo_lseek64 が未定義の場合、エラーメッセージをセットして -1 を返す。
  • サーバー側での実行エラーや通信エラーが発生した場合は -1 を返す。

保守担当者向けメモ

  • 本関数は64ビット対応のラージオブジェクト操作用である。古いサーバーバージョンでは存在しない可能性があるため、実行時にOIDの有無を確認するガードが必須である。
  • lo_read/lo_write と異なり、オフセットに int64_t を使用するため、大規模なラージオブジェクトの操作に適している。
