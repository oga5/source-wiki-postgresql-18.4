ExecEndForeignScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeForeignscan.c:296

目的

外部テーブルスキャンノード（ForeignScanState）に関連付けられたリソースを解放し、FDW（Foreign Data Wrapper）固有の終了処理を実行する。

シグネチャまたは形状

void ExecEndForeignScan(ForeignScanState *node)

入力と出力

  • **入力**: ForeignScanState *node - 終了対象となる外部スキャンノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は初期化済みであること。
  • node->fdwroutine が有効な関数ポインタを保持していること。

アルゴリズムまたは振る舞い

1. **FDW終了処理**:

  • plan->operation が CMD_SELECT 以外（直接更新操作）の場合:
  • estate->es_epq_active が NULL（EPQ実行中でない）であれば、fdwroutine->EndDirectModify を呼び出す。
  • CMD_SELECT の場合:
  • fdwroutine->EndForeignScan を呼び出し、FDW側のスキャンリソースを解放する。

    • 2. **子ノードの終了**:

  • outerPlanState(node) が存在する場合、ExecEndNode を呼び出して再帰的に子ノードを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c:680
  • **呼び出し先**:
  • node->fdwroutine->EndDirectModify または node->fdwroutine->EndForeignScan
  • ExecEndNode（子ノードが存在する場合）

関連データ構造

  • ForeignScanState: 外部スキャンノードの実行時状態。
  • ForeignScan: 外部スキャンのプランノード。
  • FdwRoutine: FDWが提供する関数ポインタ群。
  • EState: 実行時のグローバル状態。

失敗モードと境界条件

  • fdwroutine 内の各関数ポインタが NULL の場合、セグメンテーション違反が発生する可能性がある。
  • outerPlanState が存在しない場合、子ノードの終了処理はスキップされる。

保守担当者向けメモ

  • 本関数は ForeignScan ノードのライフサイクル管理の最終段階を担う。
  • EvalPlanQual（EPQ）実行中は EndDirectModify が呼び出されない仕様である点に注意すること。
