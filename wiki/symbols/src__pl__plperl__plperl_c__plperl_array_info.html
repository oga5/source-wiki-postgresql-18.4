plperl_array_info

定義

  • 種別: struct
  • 定義箇所: src/pl/plperl/plperl.c:210

目的

PostgreSQL の配列型を Perl の配列参照（AV）へ変換する過程において、配列の次元情報、要素データ、および変換に必要な関数情報を保持するための構造体です。

シグネチャまたは形状

typedef struct plperl_array_info
{
    int         ndims;              /* 配列の次元数 */
    bool        elem_is_rowtype;    /* 要素が複合型（rowtype）の場合 true */
    Datum      *elements;           /* 配列要素の Datum 配列 */
    bool       *nulls;              /* 各要素の NULL 状態を示すフラグ配列 */
    int        *nelems;             /* 各次元の要素数 */
    FmgrInfo    proc;               /* 要素変換用関数情報 */
    FmgrInfo    transform_proc;     /* 変換用プロシージャ情報 */
} plperl_array_info;

入力と出力

  • 本構造体は palloc0 を介してメモリ確保され、plperl_ref_from_pg_array 等の関数内で初期化されます。
  • 変換処理の過程で、split_array や make_array_ref に引数として渡され、最終的に Perl のデータ構造（SV*）を生成するための参照元として利用されます。

前提条件と不変条件

  • ndims は配列の次元数を表します。
  • elements、nulls、nelems は、配列の構造に応じて適切に割り当てられている必要があります。
  • proc および transform_proc は、PostgreSQL の関数マネージャ（fmgr）が要素変換を実行するために有効な状態である必要があります。

アルゴリズムまたは振る舞い

  • PostgreSQL の多次元配列を再帰的に走査し、Perl の配列参照（AV）へ変換する際のコンテキストとして機能します。
  • split_array 関数がこの構造体を参照し、再帰的にネストされた配列構造を構築します。
  • make_array_ref 関数がこの構造体を参照し、特定の次元範囲の要素を Perl の配列として組み立てます。

呼び出し元と呼び出し先

  • **呼び出し元**: plperl_ref_from_pg_array（推測される主な生成元）
  • **呼び出し先**:
  • split_array(plperl_array_info *info, ...)
  • make_array_ref(plperl_array_info *info, ...)

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • FmgrInfo: PostgreSQL の関数呼び出し情報を保持する構造体。
  • SV (Perl): Perl のスカラー値（配列参照を含む）。

失敗モードと境界条件

  • メモリ不足時: palloc0 によるメモリ確保失敗時に PostgreSQL のエラーハンドリングが起動します。
  • 配列の次元数や要素数が不正な場合、split_array 等の再帰処理において境界条件の逸脱が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は plperl.c 内の配列変換ロジックに密結合しています。配列変換の仕様変更や新しいデータ型のサポートを追加する際は、FmgrInfo の初期化手順と split_array の再帰ロジックを確認してください。
