fixup_inherited_columns

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/dml.c:92

目的

親テーブルの属性番号（カラム）を、継承関係にある子テーブルの属性番号へマッピングし、権限チェック用のビットマップを変換する。

シグネチャまたは形状

Bitmapset *fixup_inherited_columns(Oid parentId, Oid childId, Bitmapset *columns)

入力と出力

  • **入力**:
  • parentId: 親テーブルの OID。
  • childId: 子テーブルの OID。
  • columns: 親テーブルにおけるカラム権限を示すビットマップ。
  • **出力**:
  • 子テーブルの属性番号に対応する新しい Bitmapset。

前提条件と不変条件

  • ビットマップ内の各ビットは、FirstLowInvalidHeapAttributeNumber をオフセットとして加算した値であること。
  • parentId と childId が同一の場合、入力された columns がそのまま返される。

アルゴリズムまたは振る舞い

1. parentId と childId が一致する場合は入力をそのまま返す。 2. columns ビットマップを走査し、各ビットに対応する属性番号を特定する。 3. 行全体参照（InvalidAttrNumber）の場合は、変換を行わずそのまま結果に追加する。 4. それ以外の場合、get_attname で親テーブルのカラム名を取得し、get_attnum で子テーブルにおける同名の属性番号を検索する。 5. 検索結果を新しいビットマップにセットし、メモリを解放して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/dml.c 内の sepgsql_dml_privileges 関数（selectedCols, insertedCols, updatedCols の変換時に呼び出し）。
  • **呼び出し先**:
  • bms_next_member, bms_add_member (PostgreSQL ビットマップ操作関数)
  • get_attname, get_attnum (システムカタログ検索関数)
  • elog (エラー報告)
  • pfree (メモリ解放)

関連データ構造

  • Bitmapset: カラム権限を管理するためのビットマップ構造。
  • Oid: テーブル識別子。
  • AttrNumber: 属性番号。

失敗モードと境界条件

  • 子テーブルに親テーブルと同名のカラムが存在しない場合、get_attnum が InvalidAttrNumber を返し、elog(ERROR, ...) により処理が中断される。

保守担当者向けメモ

  • 継承階層が深い場合、カタログ検索（get_attname/get_attnum）が繰り返されるため、パフォーマンスへの影響に注意すること。
  • ビットマップ操作時は FirstLowInvalidHeapAttributeNumber によるオフセット計算が必須である。
