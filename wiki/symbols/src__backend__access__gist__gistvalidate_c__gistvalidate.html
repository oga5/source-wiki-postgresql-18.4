gistvalidate

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistvalidate.c:31

目的

GiSTインデックスの演算子クラス（opclass）および演算子族（opfamily）の定義が、PostgreSQLのシステムカタログ上で正当であるかを検証する。

シグネチャまたは形状

bool gistvalidate(Oid opclassoid)

入力と出力

  • **入力**: opclassoid (検証対象の演算子クラスのOID)
  • **出力**: bool (検証に合格した場合は true、不整合がある場合は false)

前提条件と不変条件

  • 指定された opclassoid がシステムカタログ pg_opclass に存在すること。
  • GiSTサポート関数は、左辺型と右辺型が一致している必要がある。
  • ORDER BY 演算子を使用する場合、対応する GIST_DISTANCE_PROC が定義されている必要がある。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して演算子クラスのメタデータを取得する。 2. 演算子族に属するすべてのサポート関数（proclist）を走査し、以下の検証を行う:

  • 左辺型と右辺型の不一致をチェック。
  • switch 文により、サポート番号（GIST_CONSISTENT_PROC 等）に応じた関数シグネチャが check_amproc_signature で期待通りか確認。

    • 3. 演算子族に属するすべての演算子（oprlist）を走査し、以下の検証を行う:

  • 戦略番号が1以上であるか確認。
  • ORDER BY 演算子の場合、距離計算関数（GIST_DISTANCE_PROC）の存在と、結果型がソート可能かを確認。

    • 4. 検証中に不整合が見つかった場合は ereport(INFO, ...) で詳細を報告し、戻り値を false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gist.c 内で amroutine->amvalidate に割り当てられている。
  • **呼び出し先**: SearchSysCache1, SearchSysCacheList1, check_amproc_signature, check_amoptsproc_signature, get_opfamily_name, get_opfamily_proc, get_op_rettype, opfamily_can_sort_type 等。

関連データ構造

  • Form_pg_opclass: 演算子クラスの定義情報。
  • Form_pg_amproc: サポート関数の定義情報。
  • Form_pg_amop: 演算子の定義情報。
  • CatCList: システムカタログのキャッシュリスト。

失敗モードと境界条件

  • opclassoid が無効な場合、elog(ERROR) により処理を中断する。
  • サポート番号が未知の場合、エラーを報告し検証を継続する。
  • 必須のサポート関数が欠落している場合や、シグネチャが不適切な場合は false を返す。

保守担当者向けメモ

  • 新しいGiSTサポート関数を追加する際は、本関数の switch 文に新しいケースを追加すること。
  • check_amproc_signature の引数は、各サポート関数が期待する引数の数と型に厳密に従う必要がある。
