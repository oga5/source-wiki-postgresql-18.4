pg_control_system

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/pg_controldata.c:31

目的

PostgreSQLの制御ファイル（global/pg_control）からシステムの基本情報を読み取り、SQLの行型として公開する。

シグネチャまたは形状

Datum pg_control_system(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出しの引数）
  • **出力**: Datum（pg_control_version, catalog_version_no, system_identifier, time を含む行型）

前提条件と不変条件

  • 呼び出し元は、戻り値の型として複合型（TYPEFUNC_COMPOSITE）を期待している必要がある。
  • 制御ファイルの読み取りには ControlFileLock（共有ロック）の取得が必須である。
  • 制御ファイルのCRCチェックが成功しなければならない。

アルゴリズムまたは振る舞い

1. get_call_result_type を呼び出し、戻り値の型が複合型であることを確認する。 2. LWLockAcquire で ControlFileLock を共有モードで取得する。 3. get_controlfile を使用してディスクから制御データを読み込む。 4. LWLockRelease でロックを解放する。 5. CRCチェックを行い、不一致の場合はエラーを投げる。 6. 読み取った ControlFileData 構造体から各フィールドを抽出し、heap_form_tuple を用いて行データを作成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQLインターフェース経由での呼び出し。
  • src/bin/pg_basebackup/pg_createsubscriber.c:601（SELECT 文による参照）。
  • **呼び出し先**:
  • get_call_result_type
  • LWLockAcquire / LWLockRelease
  • get_controlfile
  • heap_form_tuple
  • time_t_to_timestamptz

関連データ構造

  • ControlFileData: 制御ファイルの内容を保持する構造体（catalog/pg_control.h で定義）。
  • HeapTuple: SQL層へ返す行データ形式。

失敗モードと境界条件

  • 戻り値の型が複合型でない場合、elog(ERROR) を発生させる。
  • 制御ファイルのCRCチェックが失敗した場合、ereport(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数はSQL関数としてのインターフェース層であり、物理的な読み込み処理は get_controlfile（src/common/controldata_utils.c）に依存している。
  • 制御ファイルへのアクセスは排他制御が必須であるため、ロック取得・解放の順序を変更してはならない。
