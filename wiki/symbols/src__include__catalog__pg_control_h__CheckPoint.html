CheckPoint

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/pg_control.h:35

目的

データベースのチェックポイント処理において、クラスタの最新の状態を保持し、リカバリ開始点や整合性チェックの基準となるメタデータを格納する。

シグネチャまたは形状

typedef struct CheckPoint
{
    XLogRecPtr    redo;
    TimeLineID    ThisTimeLineID;
    TimeLineID    PrevTimeLineID;
    bool          fullPageWrites;
    int           wal_level;
    FullTransactionId nextXid;
    Oid           nextOid;
    MultiXactId   nextMulti;
    MultiXactOffset nextMultiOffset;
    TransactionId oldestXid;
    Oid           oldestXidDB;
    MultiXactId   oldestMulti;
    Oid           oldestMultiDB;
    pg_time_t     time;
    TransactionId oldestCommitTsXid;
    TransactionId newestCommitTsXid;
    TransactionId oldestActiveXid;
} CheckPoint;

入力と出力

  • **入力**: チェックポイント実行時に、現在のシステム状態（トランザクションID、WAL位置、タイムライン等）が各フィールドに書き込まれる。
  • **出力**: pg_control ファイルの一部としてディスクに永続化される。リカバリ時には、この構造体が読み込まれ、システムの状態復元に使用される。

前提条件と不変条件

  • ControlFileData 構造体の一部として管理され、全体サイズは PG_CONTROL_MAX_SAFE_SIZE（512バイト）を超えてはならない。
  • oldestActiveXid は、オンラインチェックポイントかつ wal_level が replica の場合にのみ計算され、それ以外は InvalidTransactionId となる。

アルゴリズムまたは振る舞い

  • チェックポイント処理（CreateCheckPoint 等）において、現在の REDO 開始点やトランザクションの進行状況を記録する。
  • リカバリプロセスでは、この構造体を参照して WAL の適用範囲を決定し、データベースの整合性を確保する。

呼び出し元と呼び出し先

  • **呼び出し元（参照・更新）**:
  • src/backend/access/transam/xlog.c: チェックポイントの作成、読み込み、および RecoveryRestartPoint での参照。
  • src/backend/access/transam/xlogrecovery.c: リカバリ時の状態復元。
  • src/backend/access/rmgrdesc/xlogdesc.c: チェックポイント情報の記述・ダンプ。
  • **呼び出し先**: 直接的な関数呼び出しは行わないデータ構造体である。

関連データ構造

  • ControlFileData: 本構造体を含む、制御ファイル全体のメイン構造体。
  • XLogRecPtr: WAL の位置を示す型。
  • FullTransactionId, TransactionId: トランザクション管理用型。
  • TimeLineID: タイムライン識別子。

失敗モードと境界条件

  • **破損**: pg_control ファイルの CRC チェックにより、本構造体を含むデータの破損が検知される。
  • **互換性**: 構造体のフィールド変更時には PG_CONTROL_VERSION の更新が必須であり、不一致時は読み込み時にエラーとなる。

保守担当者向けメモ

  • 構造体のフィールド順序は、バイナリ互換性とアライメントを考慮して設計されている。変更を加える際は注意が必要。
  • oldestActiveXid の計算ロジックは、ホットスタンバイモードの初期化に依存しているため、変更時はリカバリプロセスへの影響を精査すること。
