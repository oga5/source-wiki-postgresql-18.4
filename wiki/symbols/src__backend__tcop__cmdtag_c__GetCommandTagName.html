GetCommandTagName

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/cmdtag.c:46

目的

指定された CommandTag 列挙値に対応する、人間が読み取り可能なSQLコマンド名の文字列を取得します。

シグネチャまたは形状

const char *GetCommandTagName(CommandTag commandTag)

入力と出力

  • **入力**: CommandTag 型の列挙値。
  • **出力**: 対応するコマンド名を表す const char * 型の文字列。

前提条件と不変条件

  • commandTag は tag_behavior 配列の有効なインデックスである必要があります。
  • tag_behavior は cmdtaglist.h を通じて静的に定義されており、実行中に変更されることはありません。

アルゴリズムまたは振る舞い

1. グローバルな tag_behavior 配列に対し、引数 commandTag をインデックスとしてアクセスします。 2. 該当する CommandTagBehavior 構造体の name フィールドを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/event_trigger.c (エラーログ出力)
  • src/backend/commands/functioncmds.c (コマンド名取得)
  • src/backend/executor/spi.c (SPI実行結果のコマンド名取得)
  • src/backend/tcop/utility.c (コマンド実行制限チェック)
  • src/include/tcop/utility.h (インライン関数経由)
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、配列アクセスのみを行います。

関連データ構造

  • CommandTag: コマンドの種類を識別する列挙型。
  • CommandTagBehavior: コマンドタグの属性（名前を含む）を保持する構造体。
  • tag_behavior: 全コマンドタグの属性を保持する静的配列。

失敗モードと境界条件

  • commandTag が配列の範囲外である場合、未定義のメモリ領域にアクセスする可能性があります。

保守担当者向けメモ

  • 新しいコマンドを追加する場合は cmdtaglist.h を編集してください。本関数は自動的に新しいエントリを反映します。
  • 返される文字列は静的メモリ領域を指しているため、呼び出し側で解放する必要はありません。
