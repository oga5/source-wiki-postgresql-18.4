transformExpr

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_expr.c:118

目的

生の式（raw expression）を、意味的に確定した式ツリー（expression tree）へ変換するための最上位エントリーポイントです。

シグネチャまたは形状

Node *transformExpr(ParseState *pstate, Node *expr, ParseExprKind exprKind)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。
  • expr: 変換対象の生の式ノード。
  • exprKind: 式が使用されるコンテキスト（ParseExprKind）。
  • **出力**:
  • 意味解析および型チェックが完了した Node 構造体へのポインタ。

前提条件と不変条件

  • exprKind は EXPR_KIND_NONE であってはならない（Assert による検証）。
  • 変換処理中、pstate->p_expr_kind は一時的に指定された exprKind に更新され、終了後に元の値へ復元される。

アルゴリズムまたは振る舞い

1. 現在の pstate->p_expr_kind を退避する。 2. pstate->p_expr_kind を引数 exprKind で上書きする。 3. transformExprRecurse を呼び出し、ノードの種類に応じた再帰的な変換を実行する。 4. 退避していた元の pstate->p_expr_kind を復元する。 5. 変換結果のノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_subs.c
  • src/backend/catalog/heap.c
  • src/backend/commands/copy.c
  • src/backend/commands/functioncmds.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/prepare.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • transformExprRecurse（直接の再帰的変換処理）

関連データ構造

  • ParseState: 解析のコンテキストを保持する構造体。
  • ParseExprKind: 式が使用される場所（WHERE句、列のデフォルト値など）を識別する列挙型。
  • Node: PostgreSQLの構文解析ツリーを構成する基本ノード。

失敗モードと境界条件

  • transformExprRecurse 内で型解決や名前解決に失敗した場合、エラーが発生して処理が中断される。
  • 特定のコンテキスト（exprKind）において許可されない構文（サブクエリや集約関数など）が含まれている場合、変換処理中に拒否される。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の変換ロジックは transformExprRecurse に依存しています。
  • ParseExprKind の切り替えは、特定のコンテキストでのみ有効な構文制約を強制するために不可欠です。変更を加える際は、pstate の状態管理が正しく行われていることを確認してください。
