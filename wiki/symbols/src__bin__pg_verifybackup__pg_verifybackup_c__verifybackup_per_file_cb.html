verifybackup_per_file_cb

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:549

目的

バックアップマニフェストの解析中に、各ファイルのエントリをメモリ内のハッシュテーブルへ登録し、初期化を行うためのコールバック関数です。

シグネチャまたは形状

static void verifybackup_per_file_cb(JsonManifestParseContext *context,
                                     const char *pathname, uint64 size,
                                     pg_checksum_type checksum_type,
                                     int checksum_length, uint8 *checksum_payload)

入力と出力

  • **入力**:
  • context: マニフェスト解析コンテキスト。private_data を介して manifest_data 構造体へアクセスします。
  • pathname, size, checksum_type, checksum_length, checksum_payload: マニフェストから抽出された個別のファイル情報。
  • **出力**: なし（副作用としてハッシュテーブルを更新）。

前提条件と不変条件

  • context->private_data が有効な manifest_data 構造体を指している必要があります。
  • マニフェスト内に同一のパス名が複数存在する場合、致命的エラーとして処理を中断します。

アルゴリズムまたは振る舞い

1. context->private_data からハッシュテーブルを取得します。 2. manifest_files_insert を呼び出し、指定された pathname でハッシュテーブルにエントリを作成します。 3. 同一パス名が既に存在する場合は report_fatal_error を呼び出します。 4. 新規エントリに対し、サイズ、チェックサムの種類、長さ、ペイロードを格納します。 5. 検証状態を示すフラグ matched および bad を false に初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_manifest_file 内で設定されたコールバックとして、マニフェスト解析処理中に呼び出されます。
  • **呼び出し先**: manifest_files_insert（ハッシュテーブル操作）、report_fatal_error（エラー報告）。

関連データ構造

  • JsonManifestParseContext: マニフェスト解析のコンテキスト。
  • manifest_data: バックアップ全体の情報を保持する構造体。
  • manifest_file: 個別のファイル情報を保持するハッシュテーブルエントリ。

失敗モードと境界条件

  • **重複パス**: マニフェスト内に同一のファイルパスが複数定義されている場合、report_fatal_error によりプログラムが終了します。

保守担当者向けメモ

  • 本関数は parse_manifest_file における per_file_cb コールバックとして登録されます。
  • checksum_payload は呼び出し元から提供されたメモリ領域を指すため、マニフェスト解析のライフサイクルに依存します。
