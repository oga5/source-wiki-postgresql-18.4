FilterCommandType

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_dump/filter.h:37

目的

フィルタファイル内で指定される操作の種類（包含または除外）を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	FILTER_COMMAND_TYPE_NONE,
	FILTER_COMMAND_TYPE_INCLUDE,
	FILTER_COMMAND_TYPE_EXCLUDE,
} FilterCommandType;

入力と出力

  • 入力: フィルタファイルの解析過程において、特定の行から抽出されたコマンド文字列を本列挙型へ変換します。
  • 出力: フィルタ処理の判定ロジックにおいて、対象オブジェクトを処理対象とするか否かの判断基準として利用されます。

前提条件と不変条件

  • FILTER_COMMAND_TYPE_NONE は初期状態または未定義状態を示します。
  • フィルタファイル解析時には、INCLUDE または EXCLUDE のいずれかが明示的に指定されることが期待されます。

アルゴリズムまたは振る舞い

  • フィルタファイルから読み取ったコマンドに基づき、filter_read_item() 等の関数を通じて本列挙型の値が設定されます。
  • 呼び出し側は、この値を確認することで、後続の処理でオブジェクトをリストに含めるべきか、あるいは除外すべきかを決定します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/bin/pg_dump/filter.c:394
  • src/bin/pg_dump/pg_dump.c:20330
  • src/bin/pg_dump/pg_dumpall.c:1844
  • src/bin/pg_dump/pg_restore.c:623
  • 呼び出し先: 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • FilterObjectType: フィルタ対象となるオブジェクトの種類を定義する列挙型。
  • FilterStateData: フィルタ解析の状態を保持する構造体。

失敗モードと境界条件

  • フィルタファイル内に定義外のコマンドが記述されている場合、解析処理においてエラーが発生する可能性があります。

保守担当者向けメモ

  • 新しいフィルタコマンドを追加する場合は、本列挙型を拡張し、filter.c 内の解析ロジックを更新してください。
  • pg_dump および pg_restore の両方で利用されるため、変更時は両アプリケーションへの影響を考慮してください。
