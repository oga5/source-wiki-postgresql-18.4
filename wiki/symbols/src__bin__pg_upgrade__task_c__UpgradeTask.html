UpgradeTask

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_upgrade/task.c:82

目的

pg_upgrade において、各データベースに対して実行すべき一連のクエリステップを保持し、並列処理フレームワークで管理するためのコンテナ構造体です。

シグネチャまたは形状

struct UpgradeTask
{
	UpgradeTaskStep *steps;
	int			num_steps;
	PQExpBuffer queries;
};

入力と出力

  • **入力**: upgrade_task_add_step() を介して、実行するクエリと結果処理用のコールバックが登録されます。
  • **出力**: 登録されたステップ群は upgrade_task_run() によって各データベース接続上で順次実行されます。

前提条件と不変条件

  • 最初のステップとして ALWAYS_SECURE_SEARCH_PATH_SQL が自動的に追加される仕様です。
  • 複数のステップが登録された場合、それらは同一のデータベース接続上で順次実行されます。

アルゴリズムまたは振る舞い

  • UpgradeTask は、pg_upgrade の各フェーズ（チェックやデータ移行など）で発生する「全データベースに対する同一処理」を抽象化します。
  • PQExpBuffer を用いてクエリを保持し、libpq の非同期 API を利用した並列実行ループ内で利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c 内の各チェック関数（check_for_data_types_usage, check_for_isn_and_int8_passing_mismatch, check_for_user_defined_postfix_ops, check_for_incompatible_polymorphics, check_for_tables_with_oids など）から upgrade_task_create() を経由して生成・利用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。処理ロジックは upgrade_task_run() 等のフレームワーク関数によって参照されます。

関連データ構造

  • UpgradeTaskStep: タスク内の個別のステップを定義します。
  • PQExpBuffer: クエリ文字列を保持するために使用されます。

失敗モードと境界条件

  • 接続失敗やクエリ実行エラーが発生した場合、process_slot() 内のステートマシンによって管理されます。
  • 構造体内の num_steps が 0 の場合の挙動については、ソースコードの文脈からは明示されていません。

保守担当者向けメモ

  • 本構造体は task.c の並列実行フレームワークの中核です。ステップの追加や実行順序を変更する際は、upgrade_task_add_step() の実装と整合性を取る必要があります。
  • 実行時の並列数は user_opts.jobs に依存します。
