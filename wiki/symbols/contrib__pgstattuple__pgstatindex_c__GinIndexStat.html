GinIndexStat

定義

  • 種別: struct
  • 定義箇所: contrib/pgstattuple/pgstatindex.c:99

目的

GIN（Generalized Inverted Index）インデックスの統計情報を保持し、インデックスの物理的な状態を可視化するためのデータ構造です。

シグネチャまたは形状

typedef struct GinIndexStat
{
	int32		version;
	BlockNumber pending_pages;
	int64		pending_tuples;
} GinIndexStat;

入力と出力

本構造体は関数内部で統計情報を蓄積するために使用されます。

  • version: GINインデックスのバージョン情報。
  • pending_pages: ペンディングリスト内のページ数。
  • pending_tuples: ペンディングリスト内のタプル総数。

前提条件と不変条件

  • pgstatginindex_internal 関数等の統計収集処理において、対象インデックスのメタページおよび物理構造を走査する際に使用されます。

アルゴリズムまたは振る舞い

GINインデックスのメタページから情報を抽出し、本構造体の各フィールドに値を格納することで、インデックスの断片化やメンテナンスの必要性を判断するためのデータを提供します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgstattuple/pgstatindex.c:510 にて GinIndexStat stats; としてインスタンス化されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • BTIndexStat（B-tree用統計構造体）
  • HashIndexStat（Hashインデックス用統計構造体）

失敗モードと境界条件

  • インデックスがGIN形式でない場合や、アクセス権限がない場合には統計情報の取得に失敗する可能性があります。
  • 巨大なインデックスに対して実行した場合、I/O負荷が高くなる可能性があります。

保守担当者向けメモ

  • 本構造体は pgstattuple 拡張モジュールの一部であり、インデックスの物理構造の変更に合わせてフィールドの追加や変更が必要になる場合があります。
