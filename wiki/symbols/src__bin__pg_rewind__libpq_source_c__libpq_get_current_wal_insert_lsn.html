libpq_get_current_wal_insert_lsn

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/libpq_source.c:207

目的

リモートの PostgreSQL サーバーから現在の WAL 挿入位置（LSN）を取得する。

シグネチャまたは形状

static XLogRecPtr libpq_get_current_wal_insert_lsn(rewind_source *source)

入力と出力

  • **入力**: rewind_source *source （libpq_source 構造体へのポインタとしてキャストして使用）
  • **出力**: XLogRecPtr （取得した 64 ビットの LSN 値）

前提条件と不変条件

  • 引数 source は libpq_source 型のインスタンスである必要がある。
  • source->conn が有効な PGconn 接続を保持していること。

アルゴリズムまたは振る舞い

1. source から PGconn を取得する。 2. run_simple_query を介してリモートサーバーで SELECT pg_current_wal_insert_lsn() を実行する。 3. 取得した文字列結果を sscanf で %X/%X 形式（16進数）として解析し、上位 32 ビットと下位 32 ビットに分離する。 4. 分離した値を uint64 に合成して XLogRecPtr として返す。 5. クエリ結果のメモリを pg_free で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は確認されていない。ただし、src/bin/pg_rewind/libpq_source.c:94 にて rewind_source インターフェースの関数ポインタとして代入されている。
  • **呼び出し先**:
  • run_simple_query （クエリ実行用）
  • sscanf （文字列解析用）
  • pg_fatal （エラーハンドリング用）
  • pg_free （メモリ解放用）

関連データ構造

  • rewind_source: インターフェース定義。
  • libpq_source: rewind_source をラップする構造体。

失敗モードと境界条件

  • sscanf が 2 つの値を正しく解析できなかった場合、pg_fatal を呼び出しプログラムを終了させる。

保守担当者向けメモ

  • 本関数は libpq_source の初期化時に get_current_wal_insert_lsn フィールドへ登録される。
  • 接続が切断されている場合やクエリが失敗した場合の挙動は run_simple_query の実装に依存する。
