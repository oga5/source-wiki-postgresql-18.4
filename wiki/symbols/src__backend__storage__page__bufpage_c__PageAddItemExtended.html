PageAddItemExtended

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/bufpage.c:192

目的

PostgreSQLのページ内に新しいアイテム（タプル）を挿入します。挿入位置の自動検索や指定、および必要に応じたページ内のアイテムID配列の再配置（シャッフル）を行います。

シグネチャまたは形状

OffsetNumber PageAddItemExtended(Page page, Item item, Size size, OffsetNumber offsetNumber, int flags)

入力と出力

  • **入力**:
  • page: 操作対象のページポインタ。
  • item: 挿入するデータへのポインタ。
  • size: アイテムのサイズ。
  • offsetNumber: 挿入先のオフセット番号（InvalidOffsetNumber を指定すると自動検索）。
  • flags: 動作制御フラグ（PAI_OVERWRITE, PAI_IS_HEAP 等）。
  • **出力**:
  • 成功時: 挿入された位置の OffsetNumber。
  • 失敗時: InvalidOffsetNumber。

前提条件と不変条件

  • ページヘッダーのポインタ（pd_lower, pd_upper, pd_special）が整合していること。破損している場合は PANIC を発生させます。
  • offsetNumber が指定された場合、PAI_OVERWRITE フラグがない限り、既存のアイテムを上書きしません。
  • 挿入後の pd_lower が pd_upper を超えないこと。

アルゴリズムまたは振る舞い

1. **検証**: ページポインタの整合性をチェックします。 2. **位置決定**: offsetNumber が指定されていればそれを使用し、未指定であれば PageHasFreeLinePointers ヒントに基づき空きスロットを走査します。 3. **制約チェック**: 指定されたオフセットが範囲外でないか、ヒープページの場合は最大タプル数を超えていないかを確認します。 4. **領域計算**: 挿入に必要な領域を計算し、ページ内に収まるか判定します。 5. **再配置**: needshuffle が真の場合、既存のアイテムID配列を memmove でずらして空きを作ります。 6. **挿入**: ItemIdSetNormal でアイテムIDを更新し、memcpy でデータをページ末尾（pd_upper 側）にコピーします。 7. **ヘッダー更新**: pd_lower および pd_upper を更新して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/page/bufpage.c 内の各所（詳細はソースコードの参照箇所を参照）。
  • **呼び出し先**: PageGetMaxOffsetNumber, PageGetItemId, PageHasFreeLinePointers, PageClearHasFreeLinePointers, ItemIdSetNormal, memcpy, memmove 等。

関連データ構造

  • PageHeaderData: ページヘッダー構造体。
  • ItemIdData: アイテムID配列の要素。

失敗モードと境界条件

  • **ページ破損**: pd_lower 等の不整合により PANIC が発生します。
  • **領域不足**: ページ内にアイテムを格納する十分な空きがない場合、InvalidOffsetNumber を返します。
  • **上書き禁止**: PAI_OVERWRITE なしで既存のアイテム位置を指定した場合、または PAI_OVERWRITE ありで既にアイテムが存在する場合に失敗します。
  • **制約違反**: ヒープページで MaxHeapTuplesPerPage を超える場合や、オフセットが大きすぎる場合に失敗します。

保守担当者向けメモ

  • 本関数内では ereport(ERROR) を使用してはなりません。トランザクションの中断を避けるため、エラー時は戻り値による制御を行います。
  • PAI_IS_HEAP フラグはヒープページ特有の制約を適用するために使用されます。
  • VALGRIND_CHECK_MEM_IS_DEFINED が含まれており、メモリの初期化状態を検証しています。
