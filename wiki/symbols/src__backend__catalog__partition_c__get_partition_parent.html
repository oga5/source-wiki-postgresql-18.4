get_partition_parent

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/partition.c:52

目的

指定されたリレーション（パーティション）の直接の親リレーションのOIDを取得します。

シグネチャまたは形状

Oid get_partition_parent(Oid relid, bool even_if_detached)

入力と出力

  • **入力**:
  • relid: 親を検索する対象リレーションのOID。
  • even_if_detached: 分離（detach）処理中のパーティションであっても親OIDを返すかどうかのフラグ。
  • **出力**:
  • 成功時: 親リレーションの Oid。
  • 失敗時: 該当する親が見つからない、または分離状態の制約によりエラーが発生します。

前提条件と不変条件

  • 対象のリレーションがパーティションであることを前提としています。
  • InheritsRelationId（pg_inherits）に対する AccessShareLock を保持して操作を行います。

アルゴリズムまたは振る舞い

1. pg_inherits カタログを AccessShareLock で開きます。 2. get_partition_parent_worker を呼び出し、relid に対応する親OIDと分離状態（detach_pending）を取得します。 3. 親OIDが有効でない場合、エラーを発生させます。 4. detach_pending が真であり、かつ even_if_detached が偽である場合、分離中のリレーションとしてエラーを発生させます。 5. pg_inherits を閉じて、取得した親OIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/catalog/partition.c
  • src/backend/commands/tablecmds.c
  • src/backend/catalog/trigger.c
  • src/backend/utils/cache/partcache.c
  • **呼び出し先**:
  • table_open / table_close
  • get_partition_parent_worker
  • elog

関連データ構造

  • InheritsRelationId: pg_inherits システムカタログのOID。

失敗モードと境界条件

  • **エラー**:
  • 親リレーションが見つからない場合（elog(ERROR, ...)）。
  • detach_pending が真で、かつ even_if_detached が偽の場合（分離中のパーティションに対するアクセス制限）。

保守担当者向けメモ

  • 本関数は pg_inherits を直接スキャンするため、呼び出し側で対象がパーティションであることを確認してから使用することが推奨されます。
  • 分離処理中のパーティションを扱う必要がある場合は、even_if_detached を true に設定してください。
