geqo_selection

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_selection.c:53

目的

遺伝的クエリ最適化（GEQO）において、次世代の交叉（Crossover）処理に使用する2つの親個体（momma および daddy）を個体プールから選択します。

シグネチャまたは形状

void geqo_selection(PlannerInfo *root, Chromosome *momma, Chromosome *daddy, Pool *pool, double bias)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • pool: 選択対象となる個体プール（Pool 構造体）。
  • bias: 選択圧を制御する倍精度浮動小数点数。
  • **出力**:
  • momma, daddy: 選択された染色体データがコピーされる出力先。

前提条件と不変条件

  • pool は適応度順にソートされている必要があります。
  • pool->size が 1 より大きい場合、momma と daddy には異なる個体が選出されます。

アルゴリズムまたは振る舞い

1. linear_rand 関数を2回呼び出し、bias に基づく線形確率分布からプール内のインデックスを生成します。 2. pool->size > 1 の場合、first と second が同一にならないよう、重複が発生した際は second の再選出を繰り返します。 3. 確定したインデックスに基づき、geqo_copy を使用して pool->data から momma および daddy へ染色体情報をコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_main.c:186
  • **呼び出し先**:
  • linear_rand (内部関数)
  • geqo_copy (外部関数)

関連データ構造

  • Chromosome: 染色体データを保持する構造体。
  • Pool: 個体群を管理する構造体。

失敗モードと境界条件

  • pool->size が 1 の場合、重複排除のループはスキップされ、同一の個体が momma と daddy に選出されます。
  • linear_rand の計算において、bias が 1.0 に近い場合に計算が不安定になる可能性があります。

保守担当者向けメモ

  • 本関数は Whitley の「Genitor」アルゴリズムに基づいた線形選択スキームを実装しています。
  • 選択ロジックの変更が必要な場合は、linear_rand の確率分布計算を確認してください。
