SetTuplestoreDestReceiverParams

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tstoreReceiver.c:265

目的

DestReceiver オブジェクト（TStoreState）に対して、タプル格納先である Tuplestore や変換オプションなどの実行時パラメータを注入・初期化する。

シグネチャまたは形状

void SetTuplestoreDestReceiverParams(DestReceiver *self,
                                     Tuplestorestate *tStore,
                                     MemoryContext tContext,
                                     bool detoast,
                                     TupleDesc target_tupdesc,
                                     const char *map_failure_msg)

入力と出力

  • self: 初期化対象の DestReceiver インターフェース。
  • tStore: 格納先の Tuplestorestate。
  • tContext: 書き込みに使用する MemoryContext。
  • detoast: 外部 TOAST データを強制展開するかどうかのフラグ。
  • target_tupdesc: 変換が必要な場合のターゲット TupleDesc。
  • map_failure_msg: 変換失敗時に使用するエラーメッセージ。
  • 出力: なし（副作用として self が指す TStoreState 構造体のフィールドを更新）。

前提条件と不変条件

  • detoast と target_tupdesc を同時に true/非NULL にすることはできない（Assert により保護）。
  • self は DestTuplestore 型の DestReceiver である必要がある（Assert により保護）。

アルゴリズムまたは振る舞い

1. 引数 self を TStoreState 型にキャストする。 2. 前提条件の整合性を Assert で検証する。 3. 引数で渡された各パラメータを TStoreState 構造体の対応するフィールドに代入する。

関連データ構造

  • TStoreState: DestReceiver を継承したプライベート構造体。
  • Tuplestorestate: タプルの格納先となるデータ構造。
  • DestReceiver: 実行エンジンの出力先抽象インターフェース。

失敗モードと境界条件

  • detoast と target_tupdesc の両方が指定された場合、アサーション失敗によりプロセスが異常終了する（デバッグビルド時）。
  • self が DestTuplestore 以外の型である場合、アサーション失敗によりプロセスが異常終了する（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は CreateTuplestoreDestReceiver で生成された後の初期化フェーズで使用される。
  • 現在の実装では、detoast 処理と target_tupdesc による変換処理は排他的であるため、将来的に両方の機能を同時に必要とする要件が発生した場合は、この制約と tstoreStartupReceiver のロジックを見直す必要がある。
