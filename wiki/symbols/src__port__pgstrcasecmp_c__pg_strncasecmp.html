pg_strncasecmp

定義

  • 種別: function
  • 定義箇所: src/port/pgstrcasecmp.c:68

目的

プラットフォームやロケールに依存せず、最大 n バイトまで大文字・小文字を区別せずに文字列を比較する。

シグネチャまたは形状

int pg_strncasecmp(const char *s1, const char *s2, size_t n)

入力と出力

  • **入力**:
  • s1, s2: 比較対象のヌル終端文字列へのポインタ。
  • n: 比較する最大バイト数。
  • **出力**:
  • 比較結果として、s1 と s2 の差分（int 型）を返す。一致する場合は 0 を返す。

前提条件と不変条件

  • s1 および s2 は有効な文字列ポインタであること。
  • ASCII範囲外の文字に対しては、現在のロケール設定（isupper, tolower）が適用される。

アルゴリズムまたは振る舞い

1. n バイトに達するまで、またはヌル終端文字に達するまで、s1 と s2 を1文字ずつ走査する。 2. 各文字について以下の変換を行う：

  • 'A'-'Z' の範囲であれば、小文字に変換する。
  • IS_HIGHBIT_SET が真かつ isupper が真であれば、現在のロケールに従い tolower で変換する。

    • 3. 変換後の文字を比較し、不一致があればその差分を返す。 4. 比較が終了した時点で 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/variable.c
  • src/backend/libpq/auth-oauth.c
  • src/backend/tsearch/wparser_def.c
  • src/backend/utils/adt/bool.c
  • **呼び出し先**:
  • 標準Cライブラリ関数: isupper, tolower
  • マクロ: IS_HIGHBIT_SET

関連データ構造

  • なし（標準的な char 型および size_t 型を使用）。

失敗モードと境界条件

  • n が 0 の場合、ループは実行されず 0 を返す。
  • マルチバイト文字セットに対する完全なサポートは提供していない。

保守担当者向けメモ

  • 本関数はトルコ語ロケール等の特殊なケースを考慮し、ASCII範囲外の文字に対してのみロケール依存関数を呼び出す設計となっている。
  • ロケールに左右されない厳密なASCII比較が必要な場合は、pg_ascii_ 接頭辞を持つ関数への切り替えを検討すること。
  • src/backend/parser/scansup.c の downcase_truncate_identifier() とロジックの整合性を維持する必要がある。
