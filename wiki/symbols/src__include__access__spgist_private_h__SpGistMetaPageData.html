SpGistMetaPageData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/spgist_private.h:116

目的

SP-GiSTインデックスのメタページ（ブロック番号0）に格納される永続的な管理情報を保持します。

シグネチャまたは形状

typedef struct SpGistMetaPageData
{
    uint32      magicNumber;    /* 識別用クロスチェック値 */
    SpGistLUPCache lastUsedPages; /* 最後に使用されたページ情報の共有ストレージ */
} SpGistMetaPageData;

入力と出力

  • **入力**: インデックスのメタページ読み込み時に、バッファの内容からキャストして取得されます。
  • **出力**: インデックスの挿入処理やメタデータ更新時に、SpGistLUPCache を通じて最新のページ利用状況が参照・更新されます。

前提条件と不変条件

  • インデックスのメタページ（SPGIST_METAPAGE_BLKNO）の先頭に配置される必要があります。
  • magicNumber は、インデックスの整合性を検証するための識別子として機能します。

アルゴリズムまたは振る舞い

  • SpGistLUPCache を介して、各バックエンドが最後に使用したページをキャッシュし、メタページへのアクセス競合を低減します。
  • spgutils.c 内の関数により、メタページの初期化（memset）や、ページ内容の取得（SpGistPageGetMeta）が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgutils.c 内の複数の関数（メタページの初期化、メタデータ取得、ページ内オフセット計算など）。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、SpGistLUPCache 構造体を通じてキャッシュ管理ロジックと連携します。

関連データ構造

  • SpGistLUPCache: メタページ内に保持される、ページ利用状況のキャッシュ構造体。

失敗モードと境界条件

  • magicNumber が期待値と一致しない場合、インデックスの破損または誤ったページへのアクセスと見なされます。
  • メタページが正しく初期化されていない状態でアクセスした場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は物理レイアウトに直結しているため、メンバの追加や変更を行う際は、インデックスのオンディスクフォーマットとの互換性に注意してください。
  • SpGistLUPCache の更新ロジックは、同時実行制御における競合のボトルネックになり得るため、変更時にはパフォーマンスへの影響を考慮してください。
