umulh

定義

  • 種別: function
  • 定義箇所: src/common/d2s_intrinsics.h:119

目的

2つの 64ビット符号なし整数の積のうち、上位 64ビット分を抽出して返します。

シグネチャまたは形状

static inline uint64 umulh(const uint64 a, const uint64 b)

入力と出力

  • **入力**: a (uint64), b (uint64)
  • **出力**: a * b の結果の上位 64ビット (uint64)

前提条件と不変条件

  • umul128 関数が利用可能であること。
  • 本関数は static inline で定義されており、コンパイラによる最適化（下位ビット計算の除去）を前提としています。

アルゴリズムまたは振る舞い

1. umul128 関数を呼び出し、2つの 64ビット整数の積を 128ビットとして算出します。 2. その際、上位 64ビットの結果を hi に格納します。 3. hi を戻り値として返します。 4. コンパイラは、下位 64ビットの計算結果が使用されないことを検知し、関連する命令を最適化により削除します。

呼び出し元と呼び出し先

  • **呼び出し先**: umul128
  • **呼び出し元**:
  • src/common/d2s_intrinsics.h 内の定数除算処理（div5, div10, div100, div1e8 等）にて、マジックナンバーを用いた乗算処理の一部として呼び出されています。

関連データ構造

  • uint64 (64ビット符号なし整数)

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。
  • 64ビット環境と 32ビット環境で umul128 の実装が切り替わりますが、本関数はそれを抽象化して一貫した結果を返します。

保守担当者向けメモ

  • 本関数は umul128 に依存しています。umul128 の実装が変更された場合、本関数のパフォーマンスや挙動に影響を与える可能性があります。
  • 32ビット環境における定数除算の最適化において重要な役割を果たすため、変更時はマジックナンバーの数学的妥当性を再検証してください。
