QTNBinary

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_util.c:249

目的

QTNode ツリー構造において、3つ以上の子ノードを持つ演算子ノードを、2つずつ子ノードを持つバイナリツリー構造へ再構成します。

シグネチャまたは形状

void QTNBinary(QTNode *in)

入力と出力

  • **入力**: in (QTNode *) - 変換対象のツリーノード。
  • **出力**: なし（引数 in が指すツリー構造を直接書き換えます）。

前提条件と不変条件

  • in は有効な QTNode 構造体である必要があります。
  • 再帰的に処理を行うため、スタックオーバーフローを防ぐための十分なスタックサイズが必要です。

アルゴリズムまたは振る舞い

1. check_stack_depth() を呼び出し、スタックの深さを確認します。 2. ノードの型が QI_OPR（演算子）でない場合は即座に終了します。 3. すべての子ノードに対して再帰的に QTNBinary を呼び出します。 4. 子ノードの数（nchild）が2より大きい間、以下の処理を繰り返してバイナリ化します。

  • 新しい QTNode（nn）を割り当て、現在のノードの最初の2つの子を nn の子として移動させます。
  • nn の演算子情報を現在のノードからコピーし、QTN_NEEDFREE フラグを付与します。
  • 現在のノードの最初の子を nn に置き換え、残りの子を再配置して nchild を減らします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_rewrite.c:393
  • src/backend/utils/adt/tsquery_rewrite.c:453
  • src/backend/utils/adt/tsquery_util.c:261（再帰呼び出し）
  • **呼び出し先**:
  • check_stack_depth()
  • palloc0()（メモリ確保）

関連データ構造

  • QTNode: tsquery を表現するメモリ上のツリー構造体。
  • QueryItem: tsquery の要素（演算子やオペランド）を保持する構造体。

失敗モードと境界条件

  • **スタックオーバーフロー**: 深い入れ子構造を持つクエリに対して再帰呼び出しがスタック制限を超えた場合、check_stack_depth() によりエラーが発生します。
  • **メモリ不足**: palloc0 によるメモリ確保が失敗した場合、PostgreSQL の標準的なエラー処理によりアボートします。

保守担当者向けメモ

  • 本関数は QTNode ツリーを破壊的に変更します。呼び出し後にツリーの構造が期待通りであることを確認してください。
  • QTN_NEEDFREE フラグが設定されたノードは、後続の QTNFree 呼び出し時に適切に解放される必要があります。
