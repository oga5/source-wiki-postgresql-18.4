lookahead

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/io.c:274

目的

入力ストリームから文字を先読みし、必要に応じてバッファを動的に拡張しながら、解析対象の文字を返却する。

シグネチャまたは形状

int lookahead(void)

入力と出力

  • **入力**: グローバルな入力ストリーム input、および先読み用バッファの状態（lookahead_bp_save, lookahead_ptr, lookahead_end 等）。
  • **出力**: 読み取った文字（unsigned char にキャストされた int 型）。EOF に達した場合は EOF を返す。

前提条件と不変条件

  • lookahead_bp_save が設定されている場合、まずその内容が優先的に消費される。
  • lookahead_ptr が lookahead_end に達した際、input から新しい文字が読み込まれる。
  • 読み込まれた文字が \0 の場合はスキップされる。
  • バッファが不足した場合は malloc または realloc によりサイズが倍増される。

アルゴリズムまたは振る舞い

1. lookahead_bp_save に未処理のデータがある場合、それを順次返却する。 2. lookahead_ptr が lookahead_end に達している場合、getc(input) を呼び出して新しい文字を取得する。

  • EOF ならそのまま終了。
  • \0 なら無視して次の文字を取得。

    • 3. バッファ容量が不足している場合、malloc/realloc で領域を確保し、ポインタを再計算する。確保に失敗した場合は errx で異常終了する。 4. 取得した文字を lookahead_end に格納し、ポインタを進める。 5. lookahead_ptr が指す文字を返却し、ポインタをインクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述なし。
  • **呼び出し先**: getc (標準ライブラリ), malloc, realloc, errx。
  • *注記*: ツールによる検索結果に含まれる他の lookahead 関連の識別子は、本関数とは異なるコンテキスト（PostgreSQL バックエンドのパーサー等）のものであり、本関数との直接的な関係はない。

関連データ構造

  • lookahead_buf: 先読み用バッファの先頭ポインタ。
  • lookahead_ptr: 現在の読み取り位置。
  • lookahead_end: バッファ内の有効データの末尾。
  • lookahead_bp_save: 保存された先読みバッファのポインタ。

失敗モードと境界条件

  • **メモリ不足**: realloc が失敗した場合、errx(1, "too much lookahead required") によりプログラムが終了する。
  • **EOF**: 入力ストリームが終了した場合は EOF を返す。
  • **NULL 文字**: 入力に含まれる \0 は無視され、次の有効な文字が読み込まれるまでループする。

保守担当者向けメモ

  • 本関数は indent_globs.h で定義されたグローバルなバッファポインタ群に強く依存している。
  • バッファの再配置（realloc）時にポインタのオフセットを再計算するロジックが含まれているため、バッファ管理変数の追加・変更時には注意が必要である。
