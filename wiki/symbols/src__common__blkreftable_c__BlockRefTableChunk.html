BlockRefTableChunk

定義

  • 種別: typedef (uint16 *)
  • 定義箇所: src/common/blkreftable.c:82

目的

リレーションフォーク内の変更されたブロック情報を保持するためのデータチャンクを指すポインタ型です。

シグネチャまたは形状

typedef uint16 *BlockRefTableChunk;

入力と出力

  • 入力: BlockRefTableEntry 構造体内の chunk_data 配列から取得されます。
  • 出力: 特定のチャンク範囲内における変更済みブロックのオフセット情報（uint16 配列）を指します。

前提条件と不変条件

  • BlockRefTableEntry の chunk_data 配列の一部として管理されます。
  • 変更密度に応じて、オフセット配列またはビットマップとして解釈されます。
  • BLOCKS_PER_CHUNK (64KB) 単位で管理されるデータ構造の一部です。

アルゴリズムまたは振る舞い

  • BlockRefTableEntry の初期化時に palloc0_array を介してメモリが割り当てられます。
  • BlockRefTableMarkBlockModified 実行時に、この型を介して特定のチャンク内の変更状態が更新されます。
  • 一度ビットマップ形式へ変換されると、元のオフセット配列形式には戻りません。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/blkreftable.c 内の複数の関数（BlockRefTableMarkBlockModified 等）が、entry->chunk_data を通じて本型を参照・操作しています。
  • 呼び出し先: 本型自体はポインタ型であり、直接的な関数呼び出しは行いません。

関連データ構造

  • BlockRefTableEntry: 本型を保持する親構造体。
  • BlockRefTable: 変更ブロックを管理するメインのハッシュテーブル。

失敗モードと境界条件

  • メモリ割り当て失敗時: palloc0_array が失敗した場合、PostgreSQL のメモリ管理規約に従いエラーが発生します。
  • 範囲外アクセス: chunkno が max_chunks を超える場合、未定義の動作となります。

保守担当者向けメモ

  • 本型は uint16 の配列を指すため、ポインタ演算を行う際は BLOCKS_PER_CHUNK の境界を意識してください。
  • シリアライズ時には、この型が指すデータが tablespace -> database -> relfilenumber -> forknumber の順でソートされたエントリの一部として処理される点に注意してください。
