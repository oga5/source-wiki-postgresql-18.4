r_case_ending

定義

  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_estonian.c:1020

目的

エストニア語のステミング処理において、単語末尾の格変化接尾辞を特定し、条件に応じて削除することで語幹を抽出します。

シグネチャまたは形状

static int r_case_ending(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowball ランタイム環境構造体）
  • 出力: 成功時に 1、失敗時に 0 または負の値を返します。

前提条件と不変条件

  • z->c が z->I[0]（処理領域の境界）以上であること。
  • 処理対象の単語末尾が、a_5 配列に定義された接尾辞のいずれかに一致すること。

アルゴリズムまたは振る舞い

1. **境界チェック**: 現在位置 z->c が領域境界 z->I[0] より前であれば終了します。 2. **接尾辞検索**: find_among_b を使用し、a_5 配列に基づいて後方から接尾辞を検索します。 3. **条件分岐**:

  • case 1: g_RV グルーピングに含まれるか、または r_LONGV（長母音チェック）が成功した場合に削除対象とします。
  • case 2: 4バイトの UTF-8 文字列をスキップするテストを行い、条件を満たせば削除対象とします。

    • 4. **削除**: slice_del を呼び出し、特定された接尾辞を単語から削除します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_estonian.c:1229（および stem_ISO_8859_1_finnish.c, stem_UTF_8_finnish.c からの参照あり）
  • 呼び出し先: find_among_b, in_grouping_b_U, r_LONGV, skip_b_utf8, slice_del

関連データ構造

  • SN_env: Snowball ランタイム環境。
  • a_5: 格変化接尾辞を格納した among 構造体の配列。
  • g_RV: 処理領域（RV）を定義する文字グループ。

失敗モードと境界条件

  • 領域境界 z->I[0] に達した場合、または接尾辞が見つからない場合は 0 を返します。
  • slice_del 実行時にエラーが発生した場合は負の値を返します。
