MultiXactStateData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/multixact.c:242

目的

PostgreSQL における共有行ロック（MultiXact）の生成、管理、および周回防止（Anti-Wraparound）のための共有メモリ状態を保持する構造体です。

シグネチャまたは形状

typedef struct MultiXactStateData {
    MultiXactId nextMXact;
    MultiXactOffset nextOffset;
    bool finishedStartup;
    MultiXactId oldestMultiXactId;
    Oid oldestMultiXactDB;
    MultiXactOffset oldestOffset;
    bool oldestOffsetKnown;
    MultiXactId multiVacLimit;
    MultiXactId multiWarnLimit;
    MultiXactId multiStopLimit;
    MultiXactId multiWrapLimit;
    MultiXactOffset offsetStopLimit;
    MultiXactId perBackendXactIds[FLEXIBLE_ARRAY_MEMBER];
} MultiXactStateData;

入力と出力

  • **入力**: 構造体末尾の perBackendXactIds 配列は、各バックエンドのトランザクション状態（OldestMemberMXactId および OldestVisibleMXactId）を格納します。
  • **出力**: nextMXact および nextOffset は、次に割り当てられる MultiXactId とオフセットを追跡します。

前提条件と不変条件

  • perBackendXactIds は、全バックエンド（MaxBackends）および準備済みトランザクション（max_prepared_xacts）の情報を保持する可変長領域として機能します。
  • 各バックエンドは、自身の OldestVisibleMXactId より古い MultiXactId の SLRU データにアクセスしてはなりません（同時実行される VACUUM による切り捨てを防ぐため）。

アルゴリズムまたは振る舞い

  • **周回防止**: multiVacLimit 等の閾値に基づき、VACUUM が古い MultiXactId を削除・凍結する際の判断基準を提供します。
  • **メンバー管理**: 各バックエンドは共有ロック取得時に OldestMemberMXactId を更新し、トランザクション終了時にクリアします。
  • **可視性計算**: OldestVisibleMXactId は、全バックエンドの OldestMemberMXactId の最小値として算出され、VACUUM が安全に削除可能な範囲を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: MultiXactShmemInit 等の初期化処理や、MultiXactIdCreate 関連の生成処理から参照されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • MultiXactId: MultiXact を識別する 32 ビット ID。
  • MultiXactOffset: SLRU 内のオフセット。
  • MultiXactMember: トランザクション ID とロック状態のペア。

失敗モードと境界条件

  • **周回**: multiWrapLimit に達すると、システムは新しい MultiXactId の割り当てを停止し、データ破損を防ぐためにシャットダウンまたは警告を発する必要があります。
  • **メモリ不足**: perBackendXactIds のサイズは MaxBackends に依存するため、設定値の変更時には共有メモリの再計算が必要です。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリ確保時には offsetof(MultiXactStateData, perBackendXactIds) を用いてサイズを算出してください。
  • perBackendXactIds 配列のインデックス構造（通常バックエンドと準備済みトランザクションの分離）に注意してください。
