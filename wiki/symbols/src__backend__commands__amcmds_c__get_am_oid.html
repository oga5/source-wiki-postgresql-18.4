get_am_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/amcmds.c:182

目的

アクセスメソッド名から、対応するシステムカタログ pg_am 上の OID を取得する。

シグネチャまたは形状

Oid get_am_oid(const char *amname, bool missing_ok)

入力と出力

  • **入力**:
  • amname: 検索対象のアクセスメソッド名。
  • missing_ok: true の場合、対象が見つからない場合にエラーを発生させず InvalidOid を返す。false の場合はエラーを投げる。
  • **出力**:
  • 見つかったアクセスメソッドの Oid。見つからないかつ missing_ok が true の場合は InvalidOid。

前提条件と不変条件

  • pg_am システムカタログがアクセス可能であること。
  • 内部的に get_am_type_oid を呼び出すため、その前提条件（システムキャッシュの利用等）に依存する。

アルゴリズムまたは振る舞い

1. 引数として受け取った amname と、型指定なし（'\0'）を引数に渡し、内部関数 get_am_type_oid を呼び出す。 2. get_am_type_oid の戻り値をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c:1260
  • **呼び出し先**:
  • get_am_type_oid (同一ファイル内)

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。
  • pg_am: アクセスメソッドを管理するシステムカタログ。

失敗モードと境界条件

  • missing_ok が false の状態で存在しないアクセスメソッド名を指定した場合、get_am_type_oid 経由でエラーが発生する。
  • amname が NULL の場合の挙動は get_am_type_oid の実装に依存する。

保守担当者向けメモ

  • 本関数は get_am_type_oid の単なるラッパーである。アクセスメソッドの型（INDEX/TABLE）を問わず検索を行う必要がある場合に利用すること。
  • 特定の型に限定した検索が必要な場合は、get_index_am_oid や get_table_am_oid の利用を検討すること。
