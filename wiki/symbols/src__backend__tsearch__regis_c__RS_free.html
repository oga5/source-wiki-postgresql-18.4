RS_free

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/regis.c:160

目的

RS_compile によって生成された Regis 構造体およびその連結リスト（RegisNode）が保持する動的メモリを解放し、メモリリークを防止する。

シグネチャまたは形状

void RS_free(Regis *r)

入力と出力

  • **入力**: Regis *r - 解放対象の Regis 構造体へのポインタ。
  • **出力**: なし（副作用としてメモリを解放し、構造体内のポインタを更新する）。

前提条件と不変条件

  • 引数 r は有効な Regis 構造体を指している必要がある。
  • r->node が指す連結リストは、palloc 系関数で確保されたメモリで構成されている必要がある。

アルゴリズムまたは振る舞い

1. r->node から開始し、連結リストを順次走査する。 2. 各ノードに対して pfree を呼び出し、メモリを解放する。 3. リストの走査完了後、r->node を NULL に設定し、構造体が空であることを示す。

呼び出し元と呼び出し先

  • **呼び出し先**: pfree（PostgreSQL のメモリ解放関数）。
  • **呼び出し元**: ソースコード内の明示的な呼び出し元情報は提供されていませんが、regis.c のモジュール概要より、ISpell 辞書等の利用側で不要になった際に呼び出されることが想定されています。

関連データ構造

  • Regis: 正規表現のコンパイル結果を保持する構造体。
  • RegisNode: Regis 構造体が保持する連結リストの各ノード。

失敗モードと境界条件

  • r が NULL の場合、あるいは r->node が既に NULL の場合、関数は安全に終了する（while ループが実行されないため）。

保守担当者向けメモ

  • 本関数は RS_compile で確保されたメモリを解放するための対となる関数です。Regis 構造体を使用するコンポーネントは、利用終了時に必ず本関数を呼び出す必要があります。
