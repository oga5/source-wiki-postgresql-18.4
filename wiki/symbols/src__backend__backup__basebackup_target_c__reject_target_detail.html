reject_target_detail

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_target.c:212

目的

バックアップターゲットが詳細設定（target_detail）を受け付けない場合に、ユーザーによる誤った指定を拒否し、エラーを報告する。

シグネチャまたは形状

static void *reject_target_detail(char *target, char *target_detail)

入力と出力

  • **入力**:
  • target: ターゲット名を示す文字列。
  • target_detail: 指定された詳細設定文字列（または NULL）。
  • **出力**:
  • 正常時は NULL を返す。

前提条件と不変条件

  • target_detail が NULL でない場合、常にエラーが発生する。

アルゴリズムまたは振る舞い

1. target_detail が NULL かどうかを判定する。 2. NULL でない場合、ereport(ERROR, ...) を呼び出し、ERRCODE_SYNTAX_ERROR を伴うエラーメッセージを出力して処理を中断する。 3. NULL の場合は、何もせず NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_target.c 内の builtin_backup_targets 配列の初期化において、blackhole ターゲットの検証関数として参照されている。
  • **呼び出し先**:
  • ereport マクロ（PostgreSQLのエラー報告機構）。

関連データ構造

  • BaseBackupTargetType: 本関数はこの構造体の check_detail フィールドに設定されることを想定している。

失敗モードと境界条件

  • target_detail に値が渡された場合、バックアップ処理はエラー終了する。

保守担当者向けメモ

  • 本関数は、詳細設定をサポートしないターゲットに対して一律で適用可能な汎用的な拒否関数である。
  • 拡張モジュールで詳細設定をサポートしないターゲットを定義する場合にも再利用可能である。
