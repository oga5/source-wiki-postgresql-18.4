TParserInit

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/wparser_def.c:288

目的

全文検索パーサーの解析状態を保持する TParser 構造体を初期化し、入力文字列を解析可能な形式（ワイド文字またはマルチバイト文字列）へ変換する。

シグネチャまたは形状

static TParser *TParserInit(char *str, int len)

入力と出力

  • **入力**:
  • str: 解析対象の文字列。
  • len: 文字列の長さ。
  • **出力**: 初期化済みの TParser 構造体へのポインタ。

前提条件と不変条件

  • データベースのエンコーディング最大長（charmaxlen）に基づき、ワイド文字変換の要否を決定する。
  • charmaxlen > 1 の場合、ロケール設定に応じて pg_wchar または wchar_t 形式へ変換を行う。
  • 解析状態の初期値として TPS_Base が設定される。

アルゴリズムまたは振る舞い

1. palloc0 を使用して TParser 構造体をメモリ確保し、入力文字列と長さを格納する。 2. pg_database_encoding_max_length() を取得し、マルチバイト文字を含む可能性があるか判定する。 3. マルチバイト環境（charmaxlen > 1）の場合:

  • database_ctype_is_c が真であれば pg_mb2wchar_with_len で pg_wchar へ変換。
  • それ以外の場合は char2wchar を使用して wchar_t へ変換。

    • 4. newTParserPosition(NULL) を呼び出し、初期状態 TPS_Base を持つスタックを生成する。 5. WPARSER_TRACE が定義されている場合、解析開始文字列を標準エラー出力へログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/wparser_def.c:1899 (SQL関数インターフェース経由)
  • **呼び出し先**:
  • palloc0
  • pg_database_encoding_max_length
  • pg_mb2wchar_with_len (Cロケール時)
  • char2wchar (非Cロケール時)
  • newTParserPosition

関連データ構造

  • TParser: 解析状態を保持するメイン構造体。
  • TParserPosition: 解析の現在地を示すスタック構造。
  • pg_wchar, wchar_t: 文字変換用型。

失敗モードと境界条件

  • メモリ不足時、palloc 関連関数によりエラーが発生する可能性がある。
  • database_ctype_is_c が真の場合、char2wchar ではなく pg_mb2wchar_with_len が使用される。

保守担当者向けメモ

  • WPARSER_TRACE マクロによるデバッグ出力は、開発環境での動作確認に有用である。
