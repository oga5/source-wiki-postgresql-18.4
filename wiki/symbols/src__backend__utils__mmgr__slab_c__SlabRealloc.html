SlabRealloc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/slab.c:833

目的

SLAB アロケータにおいて、既存のメモリチャンクのサイズ変更を試みるためのインターフェースを提供します。ただし、SLAB アロケータは固定サイズ管理を前提としているため、実質的には再割り当てを拒否する役割を持ちます。

シグネチャまたは形状

void *SlabRealloc(void *pointer, Size size, int flags)

入力と出力

  • **入力**:
  • pointer: 再割り当て対象のメモリ領域へのポインタ。
  • size: 要求する新しいサイズ。
  • flags: メモリ割り当てフラグ。
  • **出力**:
  • 要求サイズが現在のチャンクサイズと一致する場合、元の pointer を返します。
  • それ以外の場合、エラーを発生させます。

前提条件と不変条件

  • pointer は、有効な SlabContext によって管理されているチャンクを指している必要があります。
  • SlabBlockIsValid(block) が真であること（ブロックヘッダーが有効なコンテキストを参照していること）が検証されます。

アルゴリズムまたは振る舞い

1. pointer から MemoryChunk ヘッダーを取得し、Valgrind 用にアクセスを許可します。 2. チャンクが属する SlabBlock を特定し、ブロックの有効性を検証します。検証に失敗した場合は ERROR を送出します。 3. 要求された size が、コンテキストの chunkSize と一致するか確認します。

  • 一致する場合、再割り当ての必要がないため、元の pointer をそのまま返します。

    • 4. サイズが一致しない場合、SLAB アロケータは再割り当てをサポートしていないため、elog(ERROR) を呼び出し処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c:79: MemoryContextMethods 構造体の realloc フィールドに登録されています。
  • **呼び出し先**:
  • PointerGetMemoryChunk (マクロ)
  • MemoryChunkGetBlock (マクロ)
  • SlabBlockIsValid (マクロ)
  • elog (PostgreSQL エラー報告関数)

関連データ構造

  • SlabContext: SLAB アロケータの管理構造体。
  • SlabBlock: チャンクを保持するブロック構造体。
  • MemoryChunk: メモリチャンクのヘッダー情報。

失敗モードと境界条件

  • **エラー**: SlabBlockIsValid が偽の場合、または size が現在の chunkSize と異なる場合、elog(ERROR) により処理が停止します。
  • **境界条件**: size == slab->chunkSize の場合は成功として扱われます。

保守担当者向けメモ

  • 本関数は MemoryContextMethods インターフェースを満たすために存在しますが、SLAB の性質上、実質的な再割り当て機能は実装されていません。
  • VALGRIND_MAKE_MEM_DEFINED および VALGRIND_MAKE_MEM_NOACCESS を使用して、チャンクヘッダーへのアクセスを制御しています。
