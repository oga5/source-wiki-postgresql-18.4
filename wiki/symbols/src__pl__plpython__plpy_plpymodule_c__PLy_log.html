PLy_log

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_plpymodule.c:283

目的

Python スクリプトから呼び出された際に、PostgreSQL の LOG レベルでメッセージを出力するためのインターフェースを提供します。

シグネチャまたは形状

static PyObject *PLy_log(PyObject *self, PyObject *args, PyObject *kw);

入力と出力

  • **入力**:
  • self: Python モジュールオブジェクト。
  • args: Python から渡された可変長引数（タプル）。
  • kw: Python から渡されたキーワード引数（辞書）。
  • **出力**:
  • PLy_output 関数の戻り値（通常は None を表す PyObject）。

前提条件と不変条件

  • 本関数は plpy モジュールのメソッドとして登録されている必要があります。
  • 内部で呼び出される PLy_output が PostgreSQL のエラー処理機構（ereport）を正しく扱える状態である必要があります。

アルゴリズムまたは振る舞い

1. Python からの引数 args および kw を受け取ります。 2. PostgreSQL のログレベル定数 LOG を指定して PLy_output を呼び出します。 3. PLy_output が実行した ereport(LOG, ...) の結果をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Python インタプリタ（plpy.log() メソッド経由）。
  • **呼び出し先**: PLy_output（src/pl/plpython/plpy_elog.c で定義）。

関連データ構造

  • PyObject: Python C API の汎用オブジェクト型。
  • LOG: PostgreSQL のログレベルを示すマクロ定数。

失敗モードと境界条件

  • 引数の解析に失敗した場合、PLy_output 内部で Python 例外が発生し、PostgreSQL のエラーとして伝播します。
  • PLy_output が ereport を通じて PostgreSQL のエラー処理をトリガーした場合、制御は Python 側に戻らず、PostgreSQL のエラーハンドラへ移行します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のログ出力ロジックは PLy_output に依存しています。ログ出力の挙動を変更する場合は PLy_output を参照してください。
  • PLy_methods テーブルにおいて METH_VARARGS | METH_KEYWORDS フラグで登録されており、Python 側からの柔軟な引数指定に対応しています。
