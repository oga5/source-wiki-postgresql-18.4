validator_startup

定義

  • 種別: function
  • 定義箇所: src/test/modules/oauth_validator/validator.c:92

目的

OAuth検証モジュールのライフサイクル開始時に、状態の整合性チェックと初期化を行う。

シグネチャまたは形状

static void validator_startup(ValidatorModuleState *state)

入力と出力

  • **入力**: ValidatorModuleState *state（検証モジュールの状態を保持する構造体へのポインタ）
  • **出力**: なし（副作用として state を更新）

前提条件と不変条件

  • state->sversion が現在の PostgreSQL バージョン（PG_VERSION_NUM）と一致している必要がある。

アルゴリズムまたは振る舞い

1. state->sversion を確認し、現在の PostgreSQL バージョンと一致しない場合は elog(ERROR) を発生させる。 2. state->private_data に定数 PRIVATE_COOKIE を格納し、状態の整合性を維持するためのマーカーとする。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL サーバーの OAuth 検証フック機構（validator_callbacks 構造体の startup_cb フィールドを介して間接的に呼び出される）。
  • **呼び出し先**: elog（エラー発生時）。

関連データ構造

  • ValidatorModuleState: 検証モジュールの状態を管理する構造体。
  • PRIVATE_COOKIE: 状態の整合性確認に使用される定数。

失敗モードと境界条件

  • state->sversion が PG_VERSION_NUM と異なる場合、エラーが発生し処理が中断される。これはテスト用モジュールとしての制約であり、実運用モジュールでは推奨されない。

保守担当者向けメモ

  • 本関数はテスト目的で実装されており、バージョンチェックの厳格な強制はアップグレード互換性を損なう可能性があるため、実運用コードへの流用は避けること。
