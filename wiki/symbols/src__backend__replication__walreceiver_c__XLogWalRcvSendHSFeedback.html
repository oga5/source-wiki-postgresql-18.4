XLogWalRcvSendHSFeedback

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiver.c:1160

目的

スタンバイサーバーの現在のトランザクション状態（xmin および catalog_xmin）をプライマリサーバーへ通知し、プライマリ側で必要な WAL やデータが削除されないように保護する（Hot Standby フィードバック）。

シグネチャまたは形状

static void XLogWalRcvSendHSFeedback(bool immed)

入力と出力

  • **入力**:
  • immed (bool): true の場合、インターバル設定を無視して即座に送信を試みる。
  • **出力**: なし（副作用としてプライマリへメッセージを送信）。

前提条件と不変条件

  • HotStandbyActive() が true であること。
  • wal_receiver_status_interval が正の値であるか、または hot_standby_feedback が有効であること。
  • 内部静的変数 primary_has_standby_xmin により、前回の送信状態を追跡する。

アルゴリズムまたは振る舞い

1. **ガードチェック**: 設定が無効かつ前回の送信で xmin を保持していない場合は早期リターンする。 2. **間隔制御**: immed が false の場合、前回の送信から wal_receiver_status_interval が経過していなければリターンする。 3. **状態取得**: HotStandbyActive() を確認後、GetReplicationHorizons() を呼び出して現在の xmin と catalog_xmin を取得する。 4. **エポック計算**: ReadNextFullTransactionId() を用いて現在のエポックを算出し、必要に応じて xmin との整合性を調整する。 5. **送信**: メッセージ種別 'h' を付与し、タイムスタンプ、xmin、xmin_epoch、catalog_xmin、catalog_xmin_epoch を walrcv_send 経由でプライマリへ送信する。 6. **状態更新**: 送信した xmin が有効であれば primary_has_standby_xmin を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/walreceiver.c 内のストリーミングループおよびエラーハンドリング箇所（408, 441, 586, 1026 行目）。
  • **呼び出し先**:
  • GetCurrentTimestamp, WalRcvComputeNextWakeup, HotStandbyActive, GetReplicationHorizons, ReadNextFullTransactionId, XidFromFullTransactionId, EpochFromFullTransactionId, resetStringInfo, pq_sendbyte, pq_sendint64, pq_sendint32, walrcv_send。

関連データ構造

  • reply_message (StringInfo): 送信メッセージ構築用バッファ。
  • wrconn: プライマリとの接続ハンドル。
  • wakeup[]: 次回送信タイミングを管理する配列。

失敗モードと境界条件

  • HotStandbyActive() が false の場合、フィードバックは送信されない。
  • wal_receiver_status_interval が 0 以下の場合は、フィードバック送信が抑制される（ただし primary_has_standby_xmin が true の場合は例外的に送信される可能性がある）。
  • xmin が無効な場合、プライマリへは InvalidTransactionId が通知される。

保守担当者向けメモ

  • 本関数は libpqwalreceiver を介してプライマリと通信するため、ネットワーク切断時は walrcv_send がエラーを投げる可能性がある。
  • primary_has_standby_xmin は静的変数であるため、プロセス再起動時にリセットされる。
