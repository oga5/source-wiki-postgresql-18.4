InsertRule

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteDefine.c:51

目的

システムカタログ pg_rewrite に対して、書き換えルール（Rewrite Rule）の挿入または更新を行い、関連する依存関係を構築する。

シグネチャまたは形状

static Oid InsertRule(const char *rulname, int evtype, Oid eventrel_oid, bool evinstead, Node *event_qual, List *action, bool replace);

入力と出力

  • **入力**:
  • rulname: ルール名。
  • evtype: イベントタイプ（CMD_SELECT 等）。
  • eventrel_oid: ルールが適用されるリレーションの OID。
  • evinstead: INSTEAD ルールかどうか。
  • event_qual: ルールの条件式（Node ツリー）。
  • action: ルールのアクション（List）。
  • replace: 既存ルールを置換するかどうかのフラグ。
  • **出力**:
  • 成功時: 挿入または更新されたルールの Oid を返す。
  • 失敗時: ルールが既に存在し replace が false の場合、エラーを発生させる。

前提条件と不変条件

  • 対象リレーションに対して RowExclusiveLock を保持していること。
  • event_qual および action は nodeToString でシリアライズ可能な形式であること。

アルゴリズムまたは振る舞い

1. ルール定義を pg_rewrite のスキーマに適合する Datum 配列へ変換する。 2. RULERELNAME システムキャッシュを使用して、対象リレーションに同名のルールが存在するか確認する。 3. **更新の場合**: heap_modify_tuple を用いて既存タプルを更新し、既存の依存関係を削除する。 4. **新規挿入の場合**: GetNewOidWithIndex で新しい OID を取得し、heap_form_tuple でタプルを生成して挿入する。 5. ルールと対象リレーション間の依存関係（DEPENDENCY_INTERNAL または DEPENDENCY_AUTO）を構築する。 6. ルールのアクションおよび条件式に含まれるオブジェクトに対する依存関係（DEPENDENCY_NORMAL）を記録する。 7. InvokeObjectPostCreateHook を呼び出し、作成後フックを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteDefine.c 内の DefineQueryRewrite（推論される直接の呼び出し元）。
  • **呼び出し先**: table_open, SearchSysCache2, CatalogTupleUpdate, CatalogTupleInsert, recordDependencyOn, recordDependencyOnExpr, InvokeObjectPostCreateHook 等。

関連データ構造

  • pg_rewrite: ルール定義を保持するシステムカタログ。
  • Form_pg_rewrite: pg_rewrite タプルの構造体。
  • ObjectAddress: 依存関係管理に使用されるオブジェクト識別子。

失敗モードと境界条件

  • replace が false の状態で同名のルールが既に存在する場合、ERRCODE_DUPLICATE_OBJECT でエラーとなる。
  • pg_rewrite へのアクセスやカタログ更新時にエラーが発生した場合、トランザクションのロールバックが必要となる。

保守担当者向けメモ

  • ON SELECT ルールは DEPENDENCY_INTERNAL として扱われ、ビューの削除時にルールも削除されるよう制御されている。
  • ルール定義の変更後は、呼び出し元で必ず CacheInvalidateRelcache を呼び出し、リレーションキャッシュを更新する必要がある。
