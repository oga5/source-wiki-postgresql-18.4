gbt_intvlt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_interval.c:55

目的

2つの interval 型の値が「より小さい（less than）」という関係にあるかを判定し、GiST インデックスの比較処理をサポートします。

シグネチャまたは形状

bool gbt_intvlt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象となる interval 型へのポインタ。
  • flinfo: 関数呼び出し情報（PostgreSQL の関数マネージャ用）。
  • **出力**:
  • bool: a < b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な interval 型のデータである必要があります。
  • 内部的に interval_lt 関数を利用するため、PostgreSQL の標準的な interval 型の比較規則に従います。

アルゴリズムまたは振る舞い

1. IntervalPGetDatum マクロを使用して、入力されたポインタを Datum 型に変換します。 2. PostgreSQL のコア関数である interval_lt を DirectFunctionCall2 を介して呼び出します。 3. interval_lt の結果（bool）を DatumGetBool で取得し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: interval_lt（PostgreSQL コア関数）
  • **呼び出し元**:
  • contrib/btree_gist/btree_interval.c 内の tinfo 構造体（gbtree_ninfo）の初期化時に参照されています。

関連データ構造

  • interval: PostgreSQL の時間間隔データ型。
  • FmgrInfo: 関数呼び出しのメタデータ構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、基盤となる interval_lt の挙動に依存します。
  • interval 型の比較ロジック（オーバーフローや不正な形式）に起因するエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.h で定義される gbtree_ninfo 構造体で使用される比較関数の一つです。
  • ロジックの変更が必要な場合は、interval_lt の仕様変更を確認してください。
