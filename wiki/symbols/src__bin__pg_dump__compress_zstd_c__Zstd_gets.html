Zstd_gets

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_zstd.c:404

目的

Zstd 圧縮されたデータストリームから、改行文字または EOF に達するまで 1 バイトずつ読み込み、文字列としてバッファに格納します。主に Large Object (LO) リストの読み込みに使用されます。

シグネチャまたは形状

char *Zstd_gets(char *buf, int len, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • buf: 読み込んだデータを格納する文字配列。
  • len: buf の最大サイズ。
  • CFH: 圧縮ファイルハンドル (CompressFileHandle)。
  • **出力**:
  • 成功時: buf へのポインタを返します。
  • 失敗または EOF 到達時: i > 0 であれば buf を返し、それ以外（読み込みバイト数が 0）であれば NULL を返します。

前提条件と不変条件

  • len > 0 である必要があります（Assert により検証）。
  • CFH は有効な CompressFileHandle である必要があります。

アルゴリズムまたは振る舞い

1. len - 1 バイトを上限として、ループ内で Zstd_read_internal を呼び出し、1 バイトずつデータを読み込みます。 2. 読み込み結果が 1 バイト未満（EOF 等）になった場合、ループを終了します。 3. 改行文字 (\n) が読み込まれた場合、その文字をバッファに含めた上でループを終了します。 4. 最後にバッファの末尾にヌル終端文字 (\0) を付与します。 5. 1 バイト以上読み込めていれば buf を、そうでなければ NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_zstd.c 内で CFH->gets_func に代入されています。直接的な呼び出し元はソースコードの断片からは特定できません。
  • **呼び出し先**: Zstd_read_internal を呼び出します。

関連データ構造

  • CompressFileHandle: 圧縮処理の抽象化に使用されるハンドル構造体。

失敗モードと境界条件

  • len が 1 以下の場合、Assert によりプログラムが停止する可能性があります。
  • 読み込み中にエラーが発生した場合、または最初の 1 バイト目で EOF に達した場合は NULL が返されます。

保守担当者向けメモ

  • 本関数は Zstd_read_internal を介してバッファリングされた I/O を行います。
  • pg_dump のアーカイブ形式における LO リスト読み込み専用のユーティリティとして設計されています。
