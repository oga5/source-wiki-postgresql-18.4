JsonPathMutableContext

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonpath.c:1263

目的

JSONパス式の評価における可変性（mutability）判定および、評価コンテキストの保持を行うための構造体です。

シグネチャまたは形状

struct JsonPathMutableContext
{
    List       *varnames;                /* 変数名のリスト */
    List       *varexprs;                /* 変数式のリスト */
    enum JsonPathDatatypeStatus current; /* @ アイテムのステータス */
    bool        lax;                     /* laxモードかstrictモードか */
    bool        mutable;                 /* 最終的な可変性ステータス */
};

入力と出力

  • 本構造体は、主に jspIsMutableWalker 関数において、JSONパス式の走査状態を保持する引数として使用されます。
  • mutable フィールドは、走査の結果としてパス式が可変であるかどうかの判定結果を格納します。

前提条件と不変条件

  • varnames および varexprs は PostgreSQL の List 構造体として管理されます。
  • current フィールドは JsonPathDatatypeStatus 列挙型に基づき、現在の評価対象の型状態を追跡します。

アルゴリズムまたは振る舞い

  • jspIsMutableWalker 関数内で、パス式内の各ノードを再帰的に走査する際に、このコンテキストを通じて現在の評価モード（lax）や可変性のフラグ（mutable）を伝播させます。
  • パス式内に可変な要素が含まれている場合、mutable フラグが更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonpath.c 内の jspIsMutable 関数などが、この構造体を初期化して jspIsMutableWalker へ渡します。
  • **呼び出し先**: jspIsMutableWalker 関数が本構造体を引数として受け取り、走査中にその内容を参照・更新します。

関連データ構造

  • List: PostgreSQL のリスト構造。
  • JsonPathDatatypeStatus: current フィールドで使用される列挙型。
  • JsonPathItem: パス式の各ノードを表現する構造体（jspIsMutableWalker の走査対象）。

失敗モードと境界条件

  • 構造体自体のメモリ割り当て失敗については、PostgreSQL の標準的なメモリ管理（palloc 等）の失敗モードに従います。
  • 複雑なパス式を走査する場合、再帰深度によるスタックオーバーフローのリスクがあるため、check_stack_depth() による保護が必要です。

保守担当者向けメモ

  • 本構造体は jspIsMutable 関連のロジックに特化しています。パス式の評価ロジックを変更する際は、mutable フラグの伝播が正しく行われているか確認してください。
  • varnames と varexprs のリストが同期して管理されていることを前提としています。
