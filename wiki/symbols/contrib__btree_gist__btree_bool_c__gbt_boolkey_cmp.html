gbt_boolkey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bool.c:52

目的

GiSTインデックスにおける boolKEY 構造体同士の比較を行い、ソート順序を決定する。

シグネチャまたは形状

int gbt_boolkey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部で boolKEY にキャストされる。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • 0: a と b が等しい場合。
  • 1: a が b より大きい場合。
  • -1: a が b より小さい場合。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体を指している必要がある。
  • boolKEY 構造体は lower および upper フィールドを持つ。

アルゴリズムまたは振る舞い

1. 入力された Nsrt 構造体から boolKEY を抽出する。 2. まず lower 値を比較する。ia->lower と ib->lower が異なる場合、その大小関係を返す。 3. lower 値が等しい場合、upper 値を比較する。 4. upper 値も等しければ 0 を返し、そうでなければ upper の大小関係を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_bool.c 内のソートサポート設定箇所（gbt_bool_sortsupport 等での参照が示唆される）。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • boolKEY: bool 型の範囲を表現する構造体。
  • Nsrt: ソート処理時に使用される中間構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動は定義されていない。
  • bool 型は実質的に 0/1 の数値として比較されるため、範囲外の値が入力された場合の動作は保証されない。

保守担当者向けメモ

  • 本関数は bool 型を数値として扱う btree_utils_num のロジックに準拠している。
  • boolKEY のリーフノードにおける lower == upper という不変条件を前提とした比較ロジックとなっている。
