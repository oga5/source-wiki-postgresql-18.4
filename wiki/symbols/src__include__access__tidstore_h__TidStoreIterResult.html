TidStoreIterResult

定義

  • 種別: struct
  • 定義箇所: src/include/access/tidstore.h:27

目的

TidStore 内に格納された TID（ItemPointerData）を走査する際、現在処理中のブロック番号と、そのブロックに関連する内部的なページ状態を保持するために使用されます。

シグネチャまたは形状

typedef struct TidStoreIterResult
{
    BlockNumber blkno;
    void       *internal_page;
} TidStoreIterResult;

入力と出力

  • 本構造体は TidStoreIterateNext 関数の戻り値として生成されます。
  • 保持する internal_page は不透明なポインタであり、直接の操作は推奨されません。

前提条件と不変条件

  • 本構造体は TidStoreGetBlockOffsets 関数への入力として使用されることが前提です。
  • 走査中の TidStore が有効である間のみ、本構造体および internal_page が指すメモリ領域は有効です。

アルゴリズムまたは振る舞い

1. TidStoreIterateNext が呼び出されると、ストア内の次のブロックを指す TidStoreIterResult が返されます。 2. 呼び出し側は、この結果を TidStoreGetBlockOffsets に渡すことで、該当ブロック内の具体的なオフセット配列を取得します。 3. 内部的には、効率化のために少数のオフセットがヘッダー内に直接保持される最適化が行われています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tidstore.c（TidStoreIterateNext の戻り値として）
  • src/backend/access/heap/vacuumlazy.c
  • src/test/modules/test_tidstore/test_tidstore.c
  • **呼び出し先**:
  • TidStoreGetBlockOffsets（本構造体を引数として受け取る）

関連データ構造

  • TidStore: TID を保持するメインのデータ構造。
  • TidStoreIter: 走査状態を保持するイテレータ。
  • BlockNumber: storage/itemptr.h で定義されるブロック識別子。

失敗モードと境界条件

  • 走査が終了した場合、TidStoreIterateNext は NULL を返します。
  • internal_page が指すメモリは、TidStore の実装（ローカルメモリまたは dsa_area）に依存するため、ストアの生存期間を超えて保持してはなりません。

保守担当者向けメモ

  • 本構造体は不透明なデータ構造として扱う設計です。データ抽出には必ず TidStoreGetBlockOffsets を経由してください。
  • 共有メモリモードで動作している場合、internal_page は dsa_area 内のポインタを指す可能性があるため、メモリの有効範囲に注意してください。
