revmap_extend_and_get_blkno

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_revmap.c:499

目的

指定されたヒープブロックに対応するリバースレンジマップ（revmap）のブロック番号を特定し、必要に応じてインデックスを物理的に拡張する。

シグネチャまたは形状

static BlockNumber revmap_extend_and_get_blkno(BrinRevmap *revmap, BlockNumber heapBlk)

入力と出力

  • **入力**:
  • revmap: BrinRevmap 構造体へのポインタ。現在のrevmap状態を保持。
  • heapBlk: 対象となるヒープブロック番号。
  • **出力**:
  • 対象のヒープブロックを管理するrevmapの物理ブロック番号（BlockNumber）。

前提条件と不変条件

  • revmap->rm_pagesPerRange が適切に初期化されていること。
  • メタページはブロック番号0に配置されているため、revmapのブロック番号は HEAPBLK_TO_REVMAP_BLK の計算結果に1を加算した値となる。

アルゴリズムまたは振る舞い

1. HEAPBLK_TO_REVMAP_BLK マクロを使用して、ヒープブロックに対応する論理的なrevmapブロック番号を計算し、メタページ分（+1）をオフセットとして加算する。 2. 計算された targetblk が、現在のrevmapの末尾ページ番号（revmap->rm_lastRevmapPage）を超えているかを確認する。 3. 超えている場合、CHECK_FOR_INTERRUPTS() を呼び出しつつ、revmap_physical_extend(revmap) を繰り返し実行してインデックスを物理的に拡張する。 4. 最終的に確定した targetblk を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: brinSetHeapBlockItemptr（src/backend/access/brin/brin_revmap.c:116）から呼び出される。
  • **呼び出し先**: revmap_physical_extend を呼び出す。

関連データ構造

  • BrinRevmap: revmapのキャッシュ状態およびメタページ情報を保持する構造体。

失敗モードと境界条件

  • インデックスの拡張が必要な場合、revmap_physical_extend 内で発生するエラー（ディスク容量不足やロック競合など）が伝播する可能性がある。
  • 割り込みが発生した場合、CHECK_FOR_INTERRUPTS() により処理が中断される。

保守担当者向けメモ

  • 本関数は、revmapがヒープの成長に追従するための動的拡張ロジックの中核を担う。
  • targetblk の計算においてメタページをスキップするために +1 している点に注意すること。
