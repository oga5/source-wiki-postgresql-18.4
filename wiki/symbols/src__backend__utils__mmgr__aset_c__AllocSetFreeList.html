AllocSetFreeList

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/aset.c:250

目的

AllocSetContext（メモリコンテキスト）自体を再利用するためのフリーリストを管理する構造体です。コンテキストの生成・破棄コストを削減するために使用されます。

シグネチャまたは形状

typedef struct AllocSetFreeList
{
	int			num_free;		/* 現在のリスト長 */
	AllocSetContext *first_free;	/* リストの先頭ポインタ */
} AllocSetFreeList;

入力と出力

  • 入力: AllocSetContext 型のポインタをリストの要素として保持します。
  • 出力: リスト内のコンテキスト数（num_free）およびリストの先頭要素（first_free）を提供します。

前提条件と不変条件

  • num_free はリスト内に保持されている AllocSetContext の数と一致する必要があります。
  • first_free が NULL の場合、num_free は 0 である必要があります。

アルゴリズムまたは振る舞い

  • コンテキスト削除時に、メモリを完全に解放する代わりにこのリストへ繋ぐことで再利用を可能にします。
  • 新規コンテキスト作成要求時に、このリストから既存のコンテキストを優先的に取り出すことで、メモリ確保のオーバーヘッドを抑制します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/mmgr/aset.c 内のコンテキスト生成および削除処理（AllocSetAlloc や AllocSetDelete 等の関連処理）から参照されます。
  • 呼び出し先: グローバル変数 context_freelists（src/backend/utils/mmgr/aset.c:257）の要素として使用されます。

関連データ構造

  • AllocSetContext: このリストが管理する主要なデータ構造です。
  • context_freelists: AllocSetFreeList 型の静的配列であり、サイズ別にコンテキストを管理します。

失敗モードと境界条件

  • リストが空の場合、first_free は NULL となり、新規のメモリ割り当てが必要となります。
  • 境界条件に関する明示的なエラーハンドリングはソース内に記述されていません。

保守担当者向けメモ

  • 本構造体は AllocSet のパフォーマンス最適化の要です。リスト操作時のポインタ整合性に注意してください。
  • context_freelists 配列のインデックスがどのように決定されるかについては、AllocSetFreeIndex の実装を参照してください。
