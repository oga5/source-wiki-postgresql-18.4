heap_copytuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/heaptuple.c:777

目的

既存の HeapTuple 構造体とそのデータ本体を、現在のメモリコンテキストに物理的に複製する。

シグネチャまたは形状

HeapTuple heap_copytuple(HeapTuple tuple)

入力と出力

  • **入力**: tuple (コピー対象の HeapTuple へのポインタ)
  • **出力**: 複製された HeapTuple 構造体へのポインタ。入力が無効な場合は NULL を返す。

前提条件と不変条件

  • 入力 tuple が HeapTupleIsValid マクロで有効と判定されること。
  • tuple->t_data が NULL でないこと。
  • 戻り値の HeapTuple 構造体と t_data は、単一の palloc ブロックとして連続したメモリ領域に確保される。

アルゴリズムまたは振る舞い

1. 入力 tuple の有効性を確認し、無効であれば NULL を返す。 2. HEAPTUPLESIZE（構造体自体のサイズ）と tuple->t_len（データ本体のサイズ）の合計分を palloc で確保し、newTuple とする。 3. newTuple の各フィールド（t_len, t_self, t_tableOid）を入力 tuple からコピーする。 4. newTuple->t_data を、確保したメモリ領域内の HEAPTUPLESIZE オフセット位置に設定する。 5. memcpy を使用して、入力 tuple->t_data から newTuple->t_data へデータをコピーする。 6. 複製された newTuple を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/index/genam.c
  • src/backend/catalog/heap.c
  • src/backend/catalog/index.c
  • src/backend/catalog/objectaddress.c
  • src/backend/catalog/pg_constraint.c
  • **呼び出し先**:
  • palloc (メモリ確保)
  • memcpy (メモリコピー)
  • HeapTupleIsValid (マクロによる検証)

関連データ構造

  • HeapTuple: タプルのヘッダー情報およびデータ本体へのポインタを保持する構造体。
  • HeapTupleHeader: タプルの物理的なデータレイアウトを定義する構造体。

失敗モードと境界条件

  • tuple が NULL または t_data が NULL の場合、関数は NULL を返す。
  • メモリ不足時には palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は HeapTuple 構造体とデータ本体を単一のメモリブロックとして確保する。これは、個別にメモリ確保を行う他のコピー手法とは異なるメモリレイアウト特性を持つ。
