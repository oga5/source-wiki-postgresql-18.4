record_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rowtypes.c:328

目的

複合型（Composite Types）または匿名レコード（RECORD）を、SQLのテキスト表現（例: (val1,val2,...)）に変換する出力ルーチンです。

シグネチャまたは形状

Datum record_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して HeapTupleHeader 型のレコードデータを受け取ります。
  • **出力**: cstring 型のテキスト表現を返します。

前提条件と不変条件

  • check_stack_depth() により、再帰的なレコード構造に対するスタックオーバーフロー保護が行われます。
  • 実行時に lookup_rowtype_tupdesc を使用してレコードの型定義（TupleDesc）を取得します。
  • fn_extra を使用して、レコードの構造および各フィールドのI/O関数情報をキャッシュし、連続する呼び出しでの再ルックアップを回避します。

アルゴリズムまたは振る舞い

1. レコードの型情報（OID/Typmod）から TupleDesc を取得し、タプルを分解します。 2. RecordIOData キャッシュを確認し、必要に応じてフィールドごとの出力関数をルックアップします。 3. 各フィールドを順次処理し、以下のルールで文字列化します。

  • 削除された列（attisdropped）はスキップします。
  • NULL値は空として扱います。
  • 特殊文字（", \, (, ), ,）や空白が含まれる場合、値をダブルクォートで囲み、内部の " や \ をエスケープします。

    • 4. フィールド間をカンマで区切り、全体を括弧 () で囲んで返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: 直接的な呼び出し元はソースコード内に明示されていませんが、PostgreSQLの型システムを通じてSQLレベルの出力処理として呼び出されます。contrib/hstore や jsonfuncs.c 等のコードで、本関数のロジックが参照・模倣されていることが示唆されています。
  • **呼び出し先**: lookup_rowtype_tupdesc, heap_deform_tuple, getTypeOutputInfo, OutputFunctionCall, ReleaseTupleDesc 等のユーティリティ関数を呼び出します。

関連データ構造

  • RecordIOData: フィールドごとのI/O関数情報や型情報を保持するキャッシュ構造体。
  • ColumnIOData: 個々の列の出力関数情報を保持する構造体。
  • TupleDesc: レコードの列定義情報。

失敗モードと境界条件

  • レコード内に別のレコードが含まれる場合、再帰的な処理が行われます。
  • 削除された列（attisdropped）は出力から除外されます。
  • 空文字列や特殊文字を含む値は、適切にクォートおよびエスケープされます。

保守担当者向けメモ

  • 本関数は fn_extra を介して型情報をキャッシュするため、型定義が変更された場合（ALTER TYPE 等）のキャッシュの整合性に注意が必要です。
  • hstore や jsonfuncs 等の他モジュールが本関数のロジックを模倣しているため、出力フォーマットを変更する際はこれらの関連コードへの影響を考慮してください。
