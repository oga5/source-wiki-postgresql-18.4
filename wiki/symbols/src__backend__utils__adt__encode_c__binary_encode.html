binary_encode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:47

目的

バイナリデータ（bytea）を指定されたエンコーディング方式（HEX, Base64, Escape等）でテキスト形式に変換し、SQL関数 encode() のバックエンド実装として機能する。

シグネチャまたは形状

Datum binary_encode(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BYTEA_PP(0): 変換対象のバイナリデータ。
  • PG_GETARG_DATUM(1): エンコーディング方式を指定する文字列（例: "hex", "base64"）。
  • **出力**:
  • PG_RETURN_TEXT_P(result): 変換後のテキストデータ。

前提条件と不変条件

  • 指定されたエンコーディング名が pg_find_encoding で解決可能であること。
  • 変換後のデータサイズが MaxAllocSize - VARHDRSZ を超えないこと。
  • enc->encode による実際の書き込みサイズが、enc->encode_len による見積もりサイズ以下であること。

アルゴリズムまたは振る舞い

1. TextDatumGetCString でエンコーディング名を取得し、pg_find_encoding で対応する関数テーブル pg_encoding を検索する。 2. enc->encode_len を呼び出し、変換後のバッファサイズを見積もる。 3. サイズが制限を超えていないか検証し、palloc でメモリを確保する。 4. enc->encode を呼び出し、データを変換してバッファに書き込む。 5. 書き込み後のサイズが見積もりを超えていないか確認し、SET_VARSIZE で結果のサイズを確定させて返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL関数 encode() からの呼び出し（直接的な呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**:
  • pg_find_encoding: エンコーディング方式の検索。
  • enc->encode_len: サイズ見積もり関数。
  • enc->encode: 変換実行関数。

関連データ構造

  • bytea: 入力バイナリデータ型。
  • text: 出力テキストデータ型。
  • pg_encoding: エンコーディング方式を定義する関数ポインタテーブル。

失敗モードと境界条件

  • **未対応のエンコーディング**: pg_find_encoding が NULL を返した場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告。
  • **メモリ制限超過**: 変換後のサイズが MaxAllocSize を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーを報告。
  • **バッファオーバーフロー**: 実際の書き込みサイズが見積もりを超えた場合、FATAL エラーでプロセスを終了。

保守担当者向けメモ

  • 新しいエンコーディングを追加する場合は、enclist 配列に定義を追加し、encode_len および encode 関数を実装すること。
  • 32bit環境でのメモリ安全性を確保するため、サイズ計算には uint64 を使用し、MaxAllocSize との比較を厳密に行っている点に注意。
