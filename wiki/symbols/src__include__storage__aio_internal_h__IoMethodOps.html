IoMethodOps

定義

  • 種別: struct
  • 定義箇所: src/include/storage/aio_internal.h:260

目的

PostgreSQL の非同期 I/O (AIO) サブシステムにおいて、異なる I/O 実装（同期、ワーカー、io_uring 等）をプラグイン可能にするための関数ポインタテーブルおよびプロパティ定義を提供します。

シグネチャまたは形状

typedef struct IoMethodOps
{
    bool        wait_on_fd_before_close;
    size_t      (*shmem_size) (void);
    void        (*shmem_init) (bool first_time);
    void        (*init_backend) (void);
    bool        (*needs_synchronous_execution) (PgAioHandle *ioh);
    int         (*submit) (uint16 num_staged_ios, PgAioHandle **staged_ios);
    void        (*wait_one) (PgAioHandle *ioh, uint64 ref_generation);
} IoMethodOps;

入力と出力

  • shmem_size: 共有メモリの追加サイズを size_t で返します。
  • submit: 投入した I/O 数を int で返します。
  • wait_one: 戻り値はありません。PgAioHandle の状態を完了状態へ遷移させます。

前提条件と不変条件

  • submit はクリティカルセクション内で呼び出されます。
  • submit 呼び出し時、num_staged_ios は PGAIO_SUBMIT_BATCH_SIZE（32）以下である必要があります。
  • wait_one は、ハンドルが既に完了状態にある場合や再利用されている場合にはブロックしてはなりません。

アルゴリズムまたは振る舞い

  • **初期化**: shmem_size, shmem_init, init_backend により、各 I/O メソッド固有のメモリ管理やバックエンドごとのセットアップを行います。
  • **実行**: needs_synchronous_execution が true を返した場合、submit は呼び出されません。それ以外の場合、submit は I/O を発行し、状態を PGAIO_HS_SUBMITTED 以上へ進めます。
  • **待機**: wait_one は I/O 完了を待ちます。完了後、状態は PGAIO_HS_COMPLETED_IO、PGAIO_HS_COMPLETED_SHARED、または PGAIO_HS_COMPLETED_LOCAL のいずれかである必要があります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/aio.c 内の AIO サブシステム制御ロジックから参照されます。
  • **呼び出し先**: src/backend/storage/aio/method_io_uring.c、method_sync.c、method_worker.c で各メソッドの実装が定義されています。

関連データ構造

  • PgAioHandle: I/O 操作の追跡に使用される主要な構造体。
  • PGAIO_SUBMIT_BATCH_SIZE: 一括投入可能な最大 I/O 数。

失敗モードと境界条件

  • wait_one が提供されない場合、I/O メソッドは pgaio_io_process_completion() を自律的に呼び出し、完了を通知する責任を負います。

保守担当者向けメモ

  • 本構造体は extern PGDLLIMPORT として宣言されたグローバルなメソッドテーブル（pgaio_sync_ops 等）を通じて利用されます。
  • 新しい I/O メソッドを追加する際は、各関数ポインタが定義する状態遷移の不変条件を厳守してください。
