TupleConstr

定義

  • 種別: struct
  • 定義箇所: src/include/access/tupdesc.h:38

目的

タプル記述子（TupleDesc）に関連付けられた、列のデフォルト値、チェック制約、および欠損値（missing values）などの制約情報を保持するデータ構造です。

シグネチャまたは形状

typedef struct TupleConstr
{
	AttrDefault *defval;		/* デフォルト値の配列 */
	ConstrCheck *check;			/* チェック制約の配列 */
	struct AttrMissing *missing;	/* 欠損値、存在しない場合は NULL */
	uint16		num_defval;		/* デフォルト値の数 */
	uint16		num_check;		/* チェック制約の数 */
	bool		has_not_null;	/* NOT NULL 制約の有無 */
	bool		has_generated_stored;	/* 格納型生成列の有無 */
	bool		has_generated_virtual;	/* 仮想型生成列の有無 */
} TupleConstr;

入力と出力

  • 本構造体は TupleDesc 構造体の一部として保持されます。
  • TupleDesc の生成時や変更時に、関連する制約情報が本構造体に格納されます。

前提条件と不変条件

  • num_defval および num_check は、それぞれ defval および check 配列の要素数と一致している必要があります。
  • missing フィールドは、欠損値がない場合は NULL である必要があります。

アルゴリズムまたは振る舞い

  • タプル記述子に関連する制約のメタデータを一元管理します。
  • 実行エンジンや DDL コマンドが、タプルの挿入や更新時に制約チェックを行う際、この構造体を参照してデフォルト値の適用や NOT NULL 制約の検証を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tupdesc.c (コピーおよび参照)
  • src/backend/catalog/heap.c
  • src/backend/commands/tablecmds.c
  • src/backend/executor/execMain.c
  • src/backend/optimizer/util/plancat.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出し先は提供されていません。

関連データ構造

  • TupleDescData / TupleDesc: 本構造体を保持する親構造体。
  • AttrDefault: デフォルト値定義。
  • ConstrCheck: チェック制約定義。
  • AttrMissing: 属性の欠損値情報。

失敗モードと境界条件

  • メモリ不足時に palloc0 等による割り当て失敗が発生する可能性があります。
  • num_defval や num_check が実際の配列サイズと不整合を起こすと、アクセス時にメモリ破壊やセグメンテーション違反を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体を拡張する場合、TupleDesc のコピー処理（tupdesc.c 内）で適切に複製されるよう注意してください。
  • has_not_null などのフラグは、制約チェックの最適化に使用されるため、カタログ定義との整合性を維持してください。
