gintuple_get_attrnum

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginutil.c:230

目的

GINインデックスのタプルから、そのタプルが属する属性番号（列番号）を取得する。

シグネチャまたは形状

OffsetNumber gintuple_get_attrnum(GinState *ginstate, IndexTuple tuple)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの実行時状態を保持する構造体。
  • tuple: 属性番号を抽出対象とする IndexTuple。
  • **出力**:
  • 属性番号を表す OffsetNumber。

前提条件と不変条件

  • ginstate が有効であること。
  • tuple が有効な IndexTuple であること。
  • ginstate->oneCol が偽（複数列インデックス）の場合、タプルの最初の属性は int16 型であり、有効な属性番号を保持していること。

アルゴリズムまたは振る舞い

1. ginstate->oneCol を確認する。

  • 真の場合: インデックスが単一列用であるため、属性番号は常に FirstOffsetNumber を返す。
  • 偽の場合: index_getattr を使用してタプルの最初の属性値を取得する。

    • 2. 取得した属性値（Datum）を uint16 に変換し、属性番号として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • src/backend/access/gin/ginentrypage.c
  • src/backend/access/gin/ginfast.c
  • src/backend/access/gin/ginget.c
  • **呼び出し先**:
  • index_getattr (PostgreSQLコア関数)

関連データ構造

  • GinState: GINインデックスの演算クラスや属性情報を保持する構造体。
  • IndexTuple: GINインデックスに格納されるタプルの形式。

失敗モードと境界条件

  • index_getattr で取得した属性値が NULL の場合、Assert により異常終了する。
  • 取得した属性番号が FirstOffsetNumber 未満、または ginstate->origTupdesc->natts を超える場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は、GINインデックスが単一列か複数列かによって属性番号の格納場所が異なることを抽象化している。
  • 複数列インデックスの場合、タプルの最初の属性が常に属性番号として使用されるため、タプル構造の変更時には注意が必要である。
