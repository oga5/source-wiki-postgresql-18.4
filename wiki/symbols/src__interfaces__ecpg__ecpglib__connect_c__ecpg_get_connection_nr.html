ecpg_get_connection_nr

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/connect.c:35

目的

指定された名前、または現在のスレッドに関連付けられたデータベース接続構造体（struct connection）を取得します。

シグネチャまたは形状

static struct connection *ecpg_get_connection_nr(const char *connection_name);

入力と出力

  • **入力**: connection_name (接続名を表す文字列、または NULL / "CURRENT")
  • **出力**: 見つかった場合は struct connection へのポインタ、見つからない場合は NULL を返します。

前提条件と不変条件

  • connection_name が NULL または "CURRENT" の場合、内部で ecpg_pthreads_init() が呼び出され、スレッド固有データ（TSD）の初期化が保証されます。
  • グローバルな接続リスト（all_connections）へのアクセスは、呼び出し元で適切に同期されている必要があります。

アルゴリズムまたは振る舞い

1. **カレント接続の検索**: connection_name が NULL または "CURRENT" の場合：

  • pthread_getspecific を使用して、スレッド固有データから接続を取得します。
  • スレッド固有の接続が存在しない場合、グローバル変数 actual_connection をフォールバックとして返します。

    • 2. **名前指定による検索**: connection_name が指定されている場合：

  • all_connections リストを走査し、name メンバが一致する接続構造体を検索します。
  • 一致するノードが見つかった時点で走査を終了し、そのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/connect.c 内の接続管理関数（ECPGget_PGconn 等）から参照されています。
  • **呼び出し先**:
  • ecpg_pthreads_init() (スレッド固有データ初期化)
  • pthread_getspecific() (POSIXスレッドAPI)

関連データ構造

  • struct connection: 接続情報を保持する構造体。
  • all_connections: 全接続を管理する連結リストのヘッド。
  • actual_connection: グローバルなデフォルト接続。
  • actual_connection_key: スレッド固有データを識別するためのキー。

失敗モードと境界条件

  • 指定された名前の接続が存在しない場合、NULL を返します。
  • connection_name が "CURRENT" で、かつスレッド固有接続もグローバル接続も設定されていない場合、NULL を返します。
  • all_connections リストが空の場合、名前指定による検索は失敗し NULL を返します。

保守担当者向けメモ

  • 本関数は static であり、connect.c 内でのみ使用されます。
  • スレッド固有データ（TSD）の検索に失敗した際、グローバル接続へフォールバックする挙動は、マルチスレッド環境下での競合リスクを伴うため、呼び出し側での排他制御の有無に注意してください。
