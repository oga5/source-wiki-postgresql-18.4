ExprState

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/execnodes.h:86

目的

クエリ実行エンジンにおいて、式ツリーの評価状態および実行命令列を保持するための構造体です。静的な式ツリーを、実行時に評価可能な形式へ変換した状態を管理します。

シグネチャまたは形状

typedef struct ExprState {
    NodeTag type;
    uint8 flags;
    bool resnull;
    Datum resvalue;
    TupleTableSlot *resultslot;
    struct ExprEvalStep *steps;
    ExprStateEvalFunc evalfunc;
    Expr *expr;
    void *evalfunc_private;
    int steps_len;
    int steps_alloc;
    struct PlanState *parent;
    ParamListInfo ext_params;
    Datum *innermost_caseval;
    bool *innermost_casenull;
    Datum *innermost_domainval;
    bool *innermost_domainnull;
    ErrorSaveContext *escontext;
} ExprState;

入力と出力

  • **入力**: ExecInitExpr 等の初期化プロセスを通じて生成されます。評価時には ExprContext を介してタプルやパラメータを受け取ります。
  • **出力**: resvalue（評価結果の Datum）および resnull（結果が NULL かどうかを示すフラグ）に評価結果が格納されます。タプル投影を行う場合は resultslot に結果が保持されます。

前提条件と不変条件

  • copyObject や equal といったノードサポート関数はサポートしていません。
  • steps フィールドは式評価の命令列を保持し、JITコンパイルやバイトコード実行の基盤となります。
  • parent フィールドは、この式が属する PlanState ノードを指します。

アルゴリズムまたは振る舞い

1. **初期化**: ExecInitExpr により、式ツリーから steps（評価ステップ）が構築されます。 2. **評価**: evalfunc が呼び出され、steps に格納された命令列を順次実行することで式を評価します。 3. **エラーハンドリング**: escontext が設定されている場合、評価中のソフトエラーを捕捉可能です。設定されていない場合はエラーがスローされます。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecEvalExprSwitchContext（catalog/index.c）、hstore_subscript_fetch（contrib/hstore/hstore_subs.c）など、式評価を必要とする実行エンジン内の各ノードから参照されます。
  • **呼び出し先**: evalfunc を通じて具体的な評価ロジックが実行されます。

関連データ構造

  • ExprEvalStep: 式評価の個々の命令。
  • ExprContext: 式評価の実行コンテキスト。
  • PlanState: この式状態の親となる実行ノード。
  • TupleTableSlot: 投影結果を保持するスロット。
  • ErrorSaveContext: エラー発生時の制御用構造体。

失敗モードと境界条件

  • escontext が NULL の場合、評価中にエラーが発生すると例外がスローされます。
  • steps_len および steps_alloc は初期化フェーズでのみ使用され、実行時には変更されません。

保守担当者向けメモ

  • 本構造体は実行時専用であり、永続化やコピーを目的としていません。
  • XXX コメントにある通り、steps_len 等のコンパイル用フィールドは初期化後に不要となる可能性があります。
  • evalfunc は式の複雑さに応じて動的に切り替わる可能性があるため、デバッグ時は expr フィールド（元の式ツリー）を参照してください。
