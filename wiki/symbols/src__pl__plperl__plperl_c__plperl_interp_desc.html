plperl_interp_desc

定義

  • 種別: struct
  • 定義箇所: src/pl/plperl/plperl.c:85

目的

PL/Perl における Perl インタプリタの実行コンテキストを管理し、PostgreSQL のユーザーIDとインタプリタインスタンス、およびそのインタプリタに紐付くクエリキャッシュを関連付けるために使用されます。

シグネチャまたは形状

typedef struct plperl_interp_desc
{
    Oid            user_id;        /* ハッシュキー（先頭である必要がある） */
    PerlInterpreter *interp;       /* Perl インタプリタ本体 */
    HTAB          *query_hash;     /* plperl_query_entry 構造体のハッシュテーブル */
} plperl_interp_desc;

入力と出力

  • **入力**: 本構造体は、plperl_interp_hash と呼ばれる PostgreSQL のハッシュテーブルに格納されるエントリとして機能します。
  • **出力**: 実行中の関数がどのインタプリタを使用すべきかを特定するためのコンテキスト情報を提供します。

前提条件と不変条件

  • user_id フィールドはハッシュキーとして機能するため、構造体の先頭に配置される必要があります。
  • 信頼された（trusted）PL/Perl 関数を実行する場合、この構造体は SQL の実行ユーザーIDごとに個別に作成され、キャッシュされます。
  • 信頼されない（untrusted）PL/PerlU 関数の場合、ユーザーID 0 (InvalidOid) を持つ単一のインスタンスがシステム全体で共有されます。

アルゴリズムまたは振る舞い

  • 本構造体は、plperl_init_interp 等によってインタプリタが初期化される際に生成されます。
  • activate_interpreter 関数を介して、現在アクティブなインタプリタ（plperl_active_interp）として切り替えられます。
  • 保持している query_hash は、そのインタプリタ内でコンパイルされたプリペアドステートメントの管理に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • _PG_init（初期化時）
  • select_perl_context（インタプリタ選択時）
  • activate_interpreter（コンテキスト切り替え時）
  • その他、plperl.c 内のインタプリタ管理ロジック全般。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。ただし、内部の interp ポインタが Perl API の実行に使用されます。

関連データ構造

  • plperl_query_entry: query_hash 内に格納されるクエリ記述子。
  • plperl_active_interp: 現在アクティブな plperl_interp_desc を指す静的ポインタ。

失敗モードと境界条件

  • **マルチインタプリタ非対応環境**: Perl が MULTIPLICITY なしでビルドされている場合、複数のインタプリタを割り当てることができず、本構造体の複数生成時にエラーが発生します。
  • **メモリ不足**: ハッシュテーブル query_hash の作成やインタプリタの割り当て時にメモリ不足が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は plperl_interp_hash に格納されるため、構造体のサイズ変更やフィールド順序の変更には注意が必要です（特に user_id が先頭であるという制約）。
  • 参照カウント管理は明示的に行われていないため、インタプリタの破棄やキャッシュからの削除タイミングについては plperl.c のライフサイクル管理ロジックを十分に確認してください。
