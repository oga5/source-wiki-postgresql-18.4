WalRcvExecResult

定義

  • 種別: struct
  • 定義箇所: src/include/replication/walreceiver.h:218

目的

WALレシーバーを介して実行されたコマンドやクエリの結果を保持し、呼び出し元へ伝達するための構造体です。

シグネチャまたは形状

typedef struct WalRcvExecResult
{
	WalRcvExecStatus status;
	int			sqlstate;
	char	   *err;
	Tuplestorestate *tuplestore;
	TupleDesc	tupledesc;
} WalRcvExecResult;

入力と出力

  • **入力**: 実行結果のステータス、エラー情報、および取得されたタプルデータ。
  • **出力**: 呼び出し元（サブスクリプション管理やスロット同期プロセスなど）が参照する実行結果のコンテナ。

前提条件と不変条件

  • tuplestore が設定されている場合、対応する tupledesc が有効である必要があります。
  • 構造体のメモリは、呼び出し元または実装側で適切に割り当て（palloc0等）および解放される必要があります。

アルゴリズムまたは振る舞い

1. コマンド実行後、WalRcvExecStatus に基づいて成功または失敗のステータスを格納します。 2. エラーが発生した場合は err にメッセージを、sqlstate にエラーコードを格納します。 3. クエリ結果としてタプルが返される場合、tuplestore にデータを格納し、その構造を tupledesc で記述します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/slotsync.c
  • src/backend/replication/logical/tablesync.c
  • **呼び出し先**:
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c 内の libpqrcv_exec および libpqrcv_processTuples が本構造体の生成およびデータ設定を行います。

関連データ構造

  • WalRcvExecStatus: 実行結果のステータスを定義する列挙型。
  • Tuplestorestate: 取得したタプルを保持するストレージ。
  • TupleDesc: タプルの構造定義。

失敗モードと境界条件

  • メモリ不足時、構造体や内部フィールドの割り当てに失敗する可能性があります。
  • 通信エラーやクエリ実行エラーが発生した場合、status に失敗を示す値が設定され、err に詳細が格納されます。

保守担当者向けメモ

  • 本構造体は libpqwalreceiver などの通信バックエンド実装と、それを利用する上位ロジック間のインターフェースです。
  • tuplestore のライフサイクル管理については、呼び出し元での適切なクリーンアップが求められます。
