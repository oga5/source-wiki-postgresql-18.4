pg_ctype_cache

定義

  • 種別: struct
  • 定義箇所: src/backend/regex/regc_pg_locale.c:634

目的

正規表現コンパイル時における、ロケール依存の文字クラス判定結果を保持するためのキャッシュ構造体です。同一の判定関数とロケール設定に対する再計算を回避し、パフォーマンスを向上させます。

シグネチャまたは形状

typedef struct pg_ctype_cache
{
	pg_wc_probefunc probefunc;	/* 判定関数 (例: pg_wc_isalpha) */
	pg_locale_t locale;			/* 適用対象のロケール */
	struct cvec cv;				/* キャッシュされた文字セット */
	struct pg_ctype_cache *next;	/* 次のキャッシュエントリへのポインタ */
} pg_ctype_cache;

入力と出力

  • **入力**: probefunc（判定ロジック）、locale（対象ロケール）、cv（判定結果の文字セット）。
  • **出力**: next ポインタを介したキャッシュリストの連結。

前提条件と不変条件

  • pg_wchar 型を対象とした文字判定結果を格納します。
  • キャッシュされた cvec 構造体は、一度生成されると解放されない設計となっています。
  • メモリ割り当てには palloc ではなく malloc が使用されます。

アルゴリズムまたは振る舞い

1. 正規表現コンパイル中に文字クラス判定が必要となった際、pg_ctype_get_cache が呼び出されます。 2. 既存のキャッシュリスト（pg_ctype_cache_list）を走査し、一致する probefunc と locale を持つエントリを探します。 3. 見つからない場合、新しい pg_ctype_cache を malloc で確保し、リストの先頭に追加します。 4. store_match 関数等を通じて、判定結果が cv メンバに格納されます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_ctype_get_cache（キャッシュの検索・生成）、store_match（キャッシュへのデータ書き込み）。
  • **呼び出し先**: 構造体内部で pg_wc_probefunc 型の関数ポインタを保持し、間接的に判定ロジックを呼び出します。

関連データ構造

  • cvec: 文字クラスの集合を保持する構造体。
  • pg_locale_t: ロケール情報を保持する型。
  • pg_wc_probefunc: pg_wchar を引数にとる文字判定関数の型。

失敗モードと境界条件

  • malloc によるメモリ確保が失敗した場合、プロセスは終了する可能性があります（正規表現エンジンが安全にエラーをハンドリングするため）。
  • キャッシュは永続的に保持されるため、多数の異なるロケールや判定関数を組み合わせた正規表現を大量にコンパイルすると、メモリ使用量が増大します。

保守担当者向けメモ

  • 本構造体は regcomp.c からインクルードされる regc_pg_locale.c 内で定義されています。
  • メモリリークを避ける設計方針により、キャッシュの解放処理は実装されていません。
  • pg_wchar は符号なし型として扱われる前提で設計されています。
