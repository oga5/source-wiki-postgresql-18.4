finishInput

定義

  • 種別: function
  • 定義箇所: src/bin/psql/input.c:539

目的

psql 終了時に、メモリ上に保持されているコマンド履歴をファイルへ永続化し、関連するメモリリソースを解放する。

シグネチャまたは形状

static void finishInput(void)

入力と出力

  • 入力: なし（グローバル変数 psql_history および pset.histsize を参照）
  • 出力: なし（副作用として履歴ファイルの更新とメモリ解放を行う）

前提条件と不変条件

  • USE_READLINE が定義されている環境でのみ動作する。
  • useHistory が真であり、かつ psql_history が有効なポインタである場合にのみ履歴の保存と解放が行われる。

アルゴリズムまたは振る舞い

1. USE_READLINE が有効な場合、useHistory フラグと psql_history の存在を確認する。 2. saveHistory 関数を呼び出し、現在の履歴を psql_history（ファイルパス）へ書き出す。この際、pset.histsize を履歴サイズの上限として使用する。 3. psql_history が指すメモリ領域を free で解放する。 4. psql_history ポインタを NULL に設定し、二重解放を防止する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/input.c:399 において atexit(finishInput) として登録されており、プログラム終了時に自動的に呼び出される。
  • 呼び出し先: saveHistory（履歴の書き出し）、free（標準ライブラリ）。

関連データ構造

  • psql_history: 履歴ファイルパスを保持する文字列変数。
  • pset: グローバル設定構造体。pset.histsize を通じて履歴の保存件数を制御する。

失敗モードと境界条件

  • psql_history が NULL の場合、何も行わずに終了する。
  • saveHistory が失敗した場合の戻り値は無視される。

保守担当者向けメモ

  • 本関数は atexit で登録されるため、プログラム終了処理の一部として実行される。
  • USE_READLINE が定義されていない環境では、本関数は実質的に何もしない（空の関数となる）。
