sepgsql_fmgr_hook

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/label.c:310

目的

信頼されたプロシージャ（trusted procedure）の実行時に、クライアントのセキュリティラベルを動的に切り替え、実行終了後に元のラベルへ復元するためのフック関数です。

シグネチャまたは形状

static void sepgsql_fmgr_hook(FmgrHookEventType event, FmgrInfo *flinfo, Datum *private);

入力と出力

  • event: 実行イベント（FHET_START, FHET_END, FHET_ABORT）。
  • flinfo: 実行対象関数の情報。
  • private: フック間で状態を保持するためのポインタ。内部で stack 構造体（old_label, new_label, next_private）を保持します。

前提条件と不変条件

  • FHET_START 時に new_label が存在する場合、db_procedure:entrypoint 権限および process:transition 権限のチェックが必須です。
  • FHET_END または FHET_ABORT 発生時、client_label_func は必ず元のラベル状態に復元される必要があります。

アルゴリズムまたは振る舞い

1. **FHET_START**:

  • private が未初期化の場合、flinfo->fn_oid に基づき sepgsql_avc_trusted_proc で新しいラベルを取得し、権限チェックを実行します。
  • 現在の client_label_func を stack->old_label に退避し、client_label_func を new_label に更新します。
  • 登録されている next_fmgr_hook があれば呼び出します。

    • 2. **FHET_END / FHET_ABORT**:

  • 登録されている next_fmgr_hook があれば呼び出します。
  • stack->old_label を用いて client_label_func を元の値に戻します。

    • 3. **その他**:

  • 未定義のイベントに対しては elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/label.c 内で fmgr_hook 変数に代入されており、PostgreSQL の関数実行フック機構から呼び出されます。
  • **呼び出し先**:
  • sepgsql_avc_trusted_proc
  • sepgsql_avc_check_perms
  • sepgsql_avc_check_perms_label
  • next_fmgr_hook（存在する場合）

関連データ構造

  • FmgrHookEventType: フックイベントの種類。
  • FmgrInfo: 関数実行情報。
  • stack (内部構造体): ラベルの退避・復元およびフックチェーン管理用。

失敗モードと境界条件

  • sepgsql_avc_check_perms 等の権限チェックで拒否された場合、例外が発生し処理が中断されます。
  • private が指すメモリは flinfo->fn_mcxt 上で確保されます。
  • 未定義の event が渡された場合はエラーとなります。

保守担当者向けメモ

  • 本関数は client_label_func を直接書き換えるため、再入可能性やスタックの整合性に注意してください。
  • next_fmgr_hook を介したフックチェーンの順序が正しく維持されているか確認が必要です。
