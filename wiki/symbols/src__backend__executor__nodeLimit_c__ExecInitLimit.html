ExecInitLimit

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeLimit.c:446

目的

Limit ノードの初期化を行い、実行に必要な状態構造体（LimitState）の構築、式評価の準備、および下位ノードの初期化を実行する。

シグネチャまたは形状

LimitState *ExecInitLimit(Limit *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Limit プランノード。
  • estate: クエリ実行状態（EState）。
  • eflags: 実行フラグ。
  • **出力**:
  • 初期化済みの LimitState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_MARK フラグが含まれていないこと（Assert により検証）。
  • Limit ノードは ExecQual や ExecProject を直接呼び出さないが、limitOffset および limitCount 式を評価するための ExprContext を保持する必要がある。

アルゴリズムまたは振る舞い

1. LimitState 構造体を割り当て、ExecLimit を実行関数として設定する。 2. LIMIT_INITIAL 状態に初期化する。 3. ExecAssignExprContext を呼び出し、式評価用コンテキストを割り当てる。 4. ExecInitNode を呼び出し、下位ノード（outerPlan）を初期化する。 5. ExecInitExpr を使用して limitOffset および limitCount 式をコンパイルする。 6. ExecInitResultTypeTL を呼び出し、結果タプルの型情報を初期化する。 7. LIMIT_OPTION_WITH_TIES が指定されている場合、last_slot 用のタプルスロットを確保し、execTuplesMatchPrepare を呼び出して等価性判定関数を準備する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:381（ExecInitNode 経由での呼び出しが示唆される）。
  • **呼び出し先**:
  • makeNode
  • ExecAssignExprContext
  • ExecInitNode
  • ExecInitExpr
  • ExecInitResultTypeTL
  • ExecGetResultSlotOps
  • ExecInitExtraTupleSlot (条件付き)
  • execTuplesMatchPrepare (条件付き)

関連データ構造

  • LimitState: 本関数が構築する主要な状態構造体。
  • Limit: プランノード。
  • EState: 実行状態。
  • ExprContext: 式評価用コンテキスト。

失敗モードと境界条件

  • eflags に EXEC_FLAG_MARK が含まれている場合、Assert により異常終了する。
  • LIMIT_OPTION_WITH_TIES 指定時は、下位ノードからタプル型情報を取得し、比較用のスロットと関数を初期化する。

保守担当者向けメモ

  • 本ノードは射影（Projection）を行わないため、ps_ProjInfo は明示的に NULL に設定される。
  • WITH TIES 処理は last_slot を介して境界値を保持するため、メモリ管理に注意が必要である。
