toast_flatten_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heaptoast.c:349

目的

外部ストレージ（TOAST）に退避されている属性をメモリ上に展開し、すべての属性がインライン化された新しいタプルを生成する。

シグネチャまたは形状

HeapTuple toast_flatten_tuple(HeapTuple tup, TupleDesc tupleDesc)

入力と出力

  • **入力**:
  • tup: 処理対象のヒープタプル。
  • tupleDesc: タプルの属性定義情報。
  • **出力**:
  • 外部データがインライン化された新しい HeapTuple。

前提条件と不変条件

  • tupleDesc->natts が MaxTupleAttributeNumber 以下であること。
  • 処理対象のタプルが heap_deform_tuple によって分解可能であること。
  • 外部データ（VARATT_IS_EXTERNAL）は detoast_external_attr によって展開可能であること。

アルゴリズムまたは振る舞い

1. heap_deform_tuple を使用して、入力タプルを属性値の配列に分解する。 2. 各属性を走査し、外部ストレージにある可変長属性（attlen == -1 かつ VARATT_IS_EXTERNAL）を特定する。 3. 該当する属性を detoast_external_attr で展開し、値をメモリ上に配置する。 4. heap_form_tuple を使用して、展開後の値から新しいタプルを再構築する。 5. 元のタプルの識別情報（t_self, t_tableOid）およびメタデータ（t_choice, t_ctid, トランザクション情報）を新しいタプルにコピーする。 6. 展開処理で一時的に確保したメモリを pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/utils/adt/expandedrecord.c
  • src/backend/utils/cache/catcache.c
  • **呼び出し先**:
  • heap_deform_tuple
  • detoast_external_attr
  • heap_form_tuple
  • pfree

関連データ構造

  • HeapTuple: 処理対象および生成されるタプル構造。
  • TupleDesc: 属性定義情報。
  • varlena: 可変長データ構造。

失敗モードと境界条件

  • 属性数が MaxTupleAttributeNumber を超える場合、アサーションエラーが発生する。
  • メモリ不足時には detoast_external_attr や heap_form_tuple 内でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は元のタプルを破壊せず、新しいタプルを生成して返すため、呼び出し側で返り値のメモリ管理（解放）を行う必要がある。
  • トランザクション情報（infomask 等）のコピーを行っているため、システムキャッシュ等で利用されるタプルの整合性が維持される。
