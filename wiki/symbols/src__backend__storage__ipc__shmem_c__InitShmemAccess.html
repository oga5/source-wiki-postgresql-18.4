InitShmemAccess

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:101

目的

プロセス起動時に共有メモリ領域へのアクセスに必要なグローバルポインタを初期化し、共有メモリの境界を確立します。

シグネチャまたは形状

void InitShmemAccess(PGShmemHeader *seghdr)

入力と出力

  • **入力**: seghdr (共有メモリセグメントのヘッダー構造体へのポインタ)
  • **出力**: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • seghdr は有効な共有メモリセグメントヘッダーを指している必要があります。
  • 本関数は、共有メモリを利用する各プロセスが初期化フェーズで呼び出す必要があります。

アルゴリズムまたは振る舞い

1. 引数 seghdr をグローバル変数 ShmemSegHdr に代入します。 2. ShmemBase を seghdr の開始アドレスに設定します。 3. ShmemEnd を ShmemBase に seghdr->totalsize を加算したアドレスに設定し、共有メモリ領域の終端を定義します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/launch_backend.c:708
  • src/backend/storage/ipc/ipci.c:225
  • **呼び出し先**:
  • 外部関数呼び出しなし（ポインタ代入および算術演算のみ）。

関連データ構造

  • PGShmemHeader: 共有メモリセグメントのヘッダー構造体。
  • ShmemSegHdr, ShmemBase, ShmemEnd: 共有メモリ管理に使用されるグローバル変数。

失敗モードと境界条件

  • seghdr が NULL の場合、後続の共有メモリ操作でセグメンテーション違反が発生する可能性があります。本関数自体にはチェックロジックは含まれていません。

保守担当者向けメモ

  • 本関数は共有メモリ管理の基盤となるポインタを設定するのみであり、メモリの割り当てやロックの初期化は行いません。
  • Windows等の EXEC_BACKEND 環境では、各プロセスが個別に共有メモリへアタッチした後に本関数を呼び出すことで、共有メモリ上のデータ構造へのアクセスが可能になります。
