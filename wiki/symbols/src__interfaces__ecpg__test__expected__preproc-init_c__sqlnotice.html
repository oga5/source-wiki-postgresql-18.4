sqlnotice

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-init.c:117

目的

ECPG（Embedded SQL for C）の回帰テストにおいて、SQLエラー発生時に通知メッセージを標準出力へ表示するハンドラ関数。

シグネチャまたは形状

static void sqlnotice(const char *notice, short trans)

入力と出力

  • notice (const char*): 表示する通知メッセージ。NULL の場合は "-empty-" として扱われる。
  • trans (short): トランザクションに関連する数値（テスト用）。
  • 戻り値: なし（void）。
  • 副作用: printf を使用して標準出力に "in sqlnotice (%s, %d)\n" という形式で文字列を出力する。

前提条件と不変条件

  • notice が NULL の場合、関数内部で安全なデフォルト値 "-empty-" に置換される。

アルゴリズムまたは振る舞い

1. notice 引数が NULL かどうかを判定し、NULL であれば "-empty-" を代入する。 2. printf を呼び出し、通知メッセージとトランザクション値をフォーマットして出力する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/test/expected/preproc-init.c:251（EXEC SQL WHENEVER 文の展開結果としてコメント内で言及）
  • src/interfaces/ecpg/test/expected/preproc-init.c:257（sqlca.sqlcode < 0 の条件分岐内）
  • 呼び出し先:
  • 標準ライブラリ関数 printf。

関連データ構造

  • sqlca_t: 本関数が呼び出される条件判定に使用される sqlca.sqlcode を保持する構造体。

失敗モードと境界条件

  • notice に NULL が渡された場合、プログラムのクラッシュを回避し、デフォルト文字列を表示する。

保守担当者向けメモ

  • 本関数は ECPG プリプロセッサの回帰テスト用コードの一部であり、実運用環境のライブラリコードではない。
  • WHENEVER 文のテストにおいて、エラーハンドリングの動作確認用として使用されている。
