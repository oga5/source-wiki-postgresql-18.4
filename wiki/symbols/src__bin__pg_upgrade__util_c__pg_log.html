pg_log

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/util.c:258

目的

pg_upgrade におけるログ出力のメインインターフェースです。ユーザーへの進捗表示と内部ログ記録を一貫した形式で提供します。

シグネチャまたは形状

void pg_log(eLogType type, const char *fmt, ...);

入力と出力

  • **入力**:
  • type: eLogType 列挙型。メッセージの重要度や種類を指定します。
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（void）。副作用として標準出力およびログファイルへの書き込みを行います。

前提条件と不変条件

  • メッセージの末尾に改行を含めないことが前提です（内部で pg_log_v が改行を付与します）。
  • グローバル変数 log_opts が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化します。 2. 内部関数 pg_log_v を呼び出し、type に応じた出力先（TTYまたはログファイル）への整形出力を行います。 3. va_end を使用して引数リストを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c 内の各箇所（PG_REPORT, PG_WARNING 等の型を指定して呼び出し）。
  • **呼び出し先**: pg_log_v（同一ファイル内）。

関連データ構造

  • eLogType: ログの種類を定義する列挙型（pg_upgrade.h で定義）。
  • LogOpts: ログ出力設定を保持する構造体（pg_upgrade.h で定義）。

失敗モードと境界条件

  • 可変長引数の不整合によるフォーマットエラーは、標準的な printf 系関数と同様の挙動となります。
  • ログファイルへの書き込み権限がない場合やディスクフル等の環境要因による失敗は、pg_log_v の実装に依存します。

保守担当者向けメモ

  • 本関数はログ出力のラッパーであり、実際の出力ロジックは pg_log_v に集約されています。
  • ログのフォーマットを変更する場合は pg_log_v を参照してください。
