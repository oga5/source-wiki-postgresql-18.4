VacuumCutoffs

定義

  • 種別: struct
  • 定義箇所: src/include/commands/vacuum.h:253

目的

VACUUM 処理の開始時に一度だけ計算される、トランザクションおよびマルチトランザクションの凍結境界（Cutoff）値を保持するための構造体です。データ破損を防ぐための不変条件として機能します。

シグネチャまたは形状

struct VacuumCutoffs
{
    TransactionId relfrozenxid;
    MultiXactId   relminmxid;
    TransactionId OldestXmin;
    MultiXactId   OldestMxact;
    TransactionId FreezeLimit;
    MultiXactId   MultiXactCutoff;
};

入力と出力

本構造体は関数への引数として渡されることが多く、主に VACUUM のメインロジックからヒープアクセスメソッド（heapam.c）やヒープ剪定処理（pruneheap.c）へ、凍結判断の基準値を提供します。

前提条件と不変条件

  • VACUUM 開始時に計算される不変の境界条件です。
  • OldestXmin および OldestMxact は、vac_update_relstats() を呼び出す際の frozenxid および minmulti 引数として渡せる最新の値です。
  • テーブル内に未凍結の XID/MXID が残らない場合、これらの値がそのまま統計情報として反映されます。

アルゴリズムまたは振る舞い

  • OldestXmin: 削除済みタプルが「DEAD」とみなされる閾値。
  • OldestMxact: 実行中のトランザクションから可視ではないとみなされるマルチトランザクションの閾値。
  • FreezeLimit: ページスキャンおよびクリーンアップロック時に、すべての XID が凍結または削除される閾値。
  • MultiXactCutoff: ページスキャンおよびクリーンアップロック時に、すべての MultiXactId が Xmax から削除される閾値。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/vacuum.c（vacuum() 関数等）で初期化されます。
  • **呼び出し先（参照箇所）**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/pruneheap.c
  • src/backend/access/heap/vacuumlazy.c
  • src/backend/commands/cluster.c
  • src/backend/commands/vacuum.c（vacuum_xid_failsafe_check 等）

関連データ構造

  • VacuumParams: VACUUM の動作オプションを規定する構造体。
  • ParallelVacuumState: 並列処理時の状態管理構造体。

失敗モードと境界条件

  • 不適切な境界値の設定はデータ破損を招くため、計算ロジックの正確性が極めて重要です。
  • vacuum_xid_failsafe_check 等で、これらの値に基づいた安全性の検証が行われます。

保守担当者向けメモ

  • 本構造体は VACUUM の仕様を定義する重要な契約です。
  • 凍結ロジックの変更や新しいデータ型の追加時には、OldestXmin 等の定義が各アクセスメソッドで正しく解釈されているか確認してください。
  • 実装の詳細については src/backend/commands/vacuum.c および vacuumlazy.c を参照してください。
