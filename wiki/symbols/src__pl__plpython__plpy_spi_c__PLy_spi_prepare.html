PLy_spi_prepare

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:35

目的

Python から呼び出された SQL クエリを解析・コンパイルし、実行可能な PLyPlanObject を生成する。

シグネチャまたは形状

PyObject *PLy_spi_prepare(PyObject *self, PyObject *args)

入力と出力

  • **入力**:
  • self: Python モジュールまたはオブジェクトインスタンス。
  • args: Python タプル。第1引数にクエリ文字列（s）、第2引数（オプション）に引数型のリスト（O）を受け取る。
  • **出力**: 成功時は生成された PLyPlanObject（PyObject*）、失敗時は NULL を返す。

前提条件と不変条件

  • list 引数が指定された場合、それは Python のシーケンス型でなければならない。
  • PLyPlanObject のメモリコンテキスト（mcxt）は TopMemoryContext を親として作成される。
  • SPI 操作はサブトランザクション内で実行される必要がある。

アルゴリズムまたは振る舞い

1. 引数を解析し、クエリ文字列と型リストを取得する。 2. PLy_plan_new() で新しいプランオブジェクトを確保し、専用のメモリコンテキストを作成する。 3. PLy_spi_subtransaction_begin() を呼び出し、SPI 操作のためのサブトランザクションを開始する。 4. PG_TRY() ブロック内で以下を実行する:

  • 型リストの各要素を parseTypeString で解決し、型情報を plan->types に格納する。
  • PLy_output_setup_func を用いて、各引数の型変換関数をセットアップする。
  • SPI_prepare() を呼び出し、クエリをプリペアドステートメントに変換する。
  • SPI_keepplan() でプランを保持し、PLy_spi_subtransaction_commit() でサブトランザクションをコミットする。

    • 5. PG_CATCH() ブロックでは、エラー発生時に PLy_spi_subtransaction_abort() を呼び出し、プランオブジェクトを解放して NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_plpymodule.c 内のメソッド定義テーブルから plpy.prepare として呼び出される。
  • **呼び出し先**: PLy_plan_new, PLy_spi_subtransaction_begin/commit/abort, parseTypeString, SPI_prepare, SPI_keepplan, PLy_output_setup_func 等。

関連データ構造

  • PLyPlanObject: プリペアドステートメントと関連する型情報、メモリコンテキストを保持する構造体。
  • PLyExecutionContext: 現在の実行コンテキスト。

失敗モードと境界条件

  • 引数が不正な型の場合、TypeError を発生させる。
  • 型名が文字列でない場合、ereport(ERROR, ...) により PostgreSQL エラーを発生させる。
  • SPI_prepare が失敗した場合、elog(ERROR, ...) によりエラーを報告する。
  • メモリ不足やサブトランザクションの失敗時は PG_CATCH を経由して Python 例外として処理される。

保守担当者向けメモ

  • PG_TRY ブロック内でのメモリ管理（MemoryContextSwitchTo）が重要である。optr 等の Python オブジェクトはエラー時に適切に Py_XDECREF されるよう注意が必要。
  • SPI_prepare の結果は TopMemoryContext に保持されるため、プランの寿命管理には注意を払うこと。
