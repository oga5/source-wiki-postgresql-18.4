InvalidateOprCacheCallBack

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_oper.c:1035

目的

システムカタログ（pg_operator や pg_cast）の変更に伴い、演算子キャッシュ（OprCacheHash）を全フラッシュし、整合性を維持する。

シグネチャまたは形状

static void InvalidateOprCacheCallBack(Datum arg, int cacheid, uint32 hashvalue)

入力と出力

  • arg: コールバック引数（未使用）。
  • cacheid: 無効化対象のキャッシュID（未使用）。
  • hashvalue: 無効化対象のハッシュ値（未使用）。
  • 戻り値: なし。

前提条件と不変条件

  • OprCacheHash が初期化済みであること（Assert(OprCacheHash != NULL) により保証）。

アルゴリズムまたは振る舞い

1. hash_seq_init を使用して OprCacheHash のシーケンシャルスキャンを開始する。 2. hash_seq_search を用いてハッシュテーブル内の全エントリを反復処理する。 3. 各エントリに対して hash_search を HASH_REMOVE モードで呼び出し、キャッシュから削除する。 4. 削除処理中にハッシュテーブルの破損が検出された場合、elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_oper.c 内で、キャッシュの無効化イベントに関連する箇所から参照されている（直接的な呼び出しフローの詳細はソース抜粋からは不明）。
  • 呼び出し先: hash_seq_init, hash_seq_search, hash_search（いずれもハッシュテーブル操作用ユーティリティ）。

関連データ構造

  • OprCacheHash: 演算子検索結果を保持するハッシュテーブル。
  • OprCacheEntry: キャッシュ内の各エントリ構造体。

失敗モードと境界条件

  • hash_search が NULL を返した場合、ハッシュテーブルの破損とみなして ERROR を送出する。

保守担当者向けメモ

  • 現在の実装では、特定の変更に対する部分的な無効化ではなく、キャッシュ全体をフラッシュする戦略をとっている。
  • pg_operator や pg_cast の変更が演算子の適合性に影響を与えるため、このコールバックは必須である。
