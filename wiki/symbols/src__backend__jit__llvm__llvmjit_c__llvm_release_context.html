llvm_release_context

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit.c:251

目的

JIT コンテキストに関連付けられた LLVM モジュールおよび LLVMJitHandle リソースを解放し、メモリリークを防ぐために使用されます。

シグネチャまたは形状

static void llvm_release_context(JitContext *context)

入力と出力

  • **入力**: JitContext *context（LLVMJitContext へキャストして使用）
  • **出力**: なし（副作用としてリソースを解放）

前提条件と不変条件

  • llvm_jit_context_in_use_count がグローバルな追跡用としてデクリメントされます。
  • proc_exit_inprogress が真の場合、LLVM への再入によるクラッシュを避けるため、クリーンアップ処理をスキップして即座に終了します。

アルゴリズムまたは振る舞い

1. llvm_jit_context_in_use_count をデクリメントします。 2. プロセス終了処理中であれば即座にリターンします。 3. llvm_enter_fatal_on_oom() を呼び出し、メモリ不足時のエラーハンドリングを有効化します。 4. llvm_jit_context->module が存在する場合、LLVMDisposeModule で破棄します。 5. llvm_jit_context->handles リストを走査し、各 LLVMJitHandle に対して以下を実行します。

  • LLVMOrcResourceTrackerRemove および LLVMOrcReleaseResourceTracker によるリソーストラッカーの解放。
  • LLVMOrcSymbolStringPoolClearDeadEntries によるシンボル文字列プールのクリーンアップ。
  • pfree によるハンドルメモリの解放。

    • 6. リスト自体を解放し、llvm_leave_fatal_on_oom() を呼び出します。 7. resowner が設定されている場合、ResourceOwnerForgetJIT を呼び出してリソースオーナーから登録を解除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit.c 内で JitContext のコールバック構造体 (cb->release_context) に登録されています。
  • **呼び出し先**:
  • LLVMDisposeModule
  • LLVMOrcResourceTrackerRemove
  • LLVMOrcReleaseResourceTracker
  • LLVMOrcLLJITGetExecutionSession
  • LLVMOrcExecutionSessionGetSymbolStringPool
  • LLVMOrcSymbolStringPoolClearDeadEntries
  • pfree
  • list_free
  • ResourceOwnerForgetJIT
  • llvm_enter_fatal_on_oom / llvm_leave_fatal_on_oom

関連データ構造

  • LLVMJitContext: JIT コンテキストのメイン構造体。
  • LLVMJitHandle: JIT されたコードのハンドル情報。
  • ResourceOwner: トランザクション単位のリソース管理用構造体。

失敗モードと境界条件

  • proc_exit_inprogress が真の場合、安全のためクリーンアップをスキップします。
  • メモリ不足発生時は llvm_enter_fatal_on_oom により PostgreSQL のエラー処理系へ制御が移ります。

保守担当者向けメモ

  • 本関数は JitContext の破棄フローの一部です。リソースオーナーとの整合性を保つため、ResourceOwnerForgetJIT の呼び出し順序や条件に変更を加える際は注意してください。
  • シンボル文字列プールのクリアはメモリリーク防止のために行われていますが、パフォーマンスへの影響を考慮しつつ実装されています。
