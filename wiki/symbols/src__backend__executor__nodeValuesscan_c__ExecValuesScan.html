ExecValuesScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeValuesscan.c:195

目的

SQLの VALUES 句によって生成された定数リストをスキャンし、実行エンジンに対して仮想タプルを提供するためのインターフェース関数です。

シグネチャまたは形状

TupleTableSlot *ExecValuesScan(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（ValuesScanState にキャスト可能な実行ノードの状態）。
  • **出力**: TupleTableSlot *（評価された行データを含むスロット、またはスキャン終了時の NULL）。

前提条件と不変条件

  • pstate は ValuesScanState 構造体として初期化済みであること。
  • ExecInitValuesScan によって式リストの準備とメモリコンテキストの割り当てが完了していること。

アルゴリズムまたは振る舞い

1. pstate を ValuesScanState 型にキャストします。 2. ExecScan 関数を呼び出し、スキャン処理を委譲します。 3. ExecScan 内で、以下の関数ポインタを使用してスキャンを実行します。

  • ValuesNext: 現在のインデックスに基づき、式を評価して仮想タプルを生成します。
  • ValuesRecheck: スキャン中の再チェック処理を行います。

    • 4. 最終的に ExecScan が返した TupleTableSlot を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン内のノード実行ループ（ExecProcNode 経由で呼び出される）。
  • **呼び出し先**:
  • ExecScan (共通基盤)
  • ValuesNext (スキャン実行の実体)
  • ValuesRecheck (再チェック用)

関連データ構造

  • ValuesScanState: VALUES スキャンノードの状態を保持する構造体。
  • PlanState: 実行ノードの共通状態構造体。
  • TupleTableSlot: 仮想タプルを格納するコンテナ。

失敗モードと境界条件

  • 全ての行を処理し終えた場合、ValuesNext を通じて NULL が返されます。
  • VALUES 句の式評価中にエラーが発生した場合、PostgreSQLの標準的なエラーハンドリングメカニズムにより中断されます。

保守担当者向けメモ

  • 本関数は ExecScan を呼び出すラッパーであり、具体的なスキャンロジックは ValuesNext に依存しています。
  • JITコンパイルは、VALUES 句の式が通常一度しか評価されないという特性に基づき、意図的に無効化されています。
  • 物理的なタプルではなくメモリ上の式を評価する「仮想タプル」を扱うため、メモリ管理（expandeddatum 等）のライフサイクルに注意してください。
