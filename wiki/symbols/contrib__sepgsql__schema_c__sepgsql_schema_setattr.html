sepgsql_schema_setattr

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/schema.c:201

目的

指定されたスキーマ（pg_namespace）に対する属性変更操作（SETATTR）が、SELinux ポリシーによって許可されているかを検証します。

シグネチャまたは形状

void sepgsql_schema_setattr(Oid namespaceId)

入力と出力

  • **入力**:
  • namespaceId (Oid): 権限チェック対象となるスキーマのオブジェクト識別子。
  • **出力**: なし。権限チェックに失敗した場合はエラーを発生させます。

前提条件と不変条件

  • namespaceId は有効な pg_namespace カタログの OID である必要があります。
  • 内部で呼び出される check_schema_perms により、権限違反時にはトランザクションを中断するエラーが投げられます。

アルゴリズムまたは振る舞い

1. check_schema_perms 関数を呼び出します。 2. 第1引数に namespaceId を渡します。 3. 第2引数に SEPG_DB_SCHEMA__SETATTR を指定し、スキーマの属性変更権限を要求します。 4. 第3引数に true を指定し、権限がない場合にエラーを発生させる（abort_on_violation）よう設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:221 から呼び出されます。
  • **呼び出し先**: check_schema_perms（内部ユーティリティ関数）。

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子型。
  • SEPG_DB_SCHEMA__SETATTR: SE-PostgreSQL におけるスキーマ属性変更権限を示す定数。

失敗モードと境界条件

  • 指定された namespaceId が存在しない場合や、SELinux ポリシー上で SETATTR 権限が拒否されている場合、check_schema_perms がエラーを発生させ、処理が停止します。

保守担当者向けメモ

  • 本関数は sepgsql_hooks.c を介して PostgreSQL のフックポイントから呼び出される設計です。
  • 権限チェックのロジックを変更する場合は、check_schema_perms の実装および contrib/sepgsql/uavc.c の AVC キャッシュ管理を確認してください。
