spg_box_quad_choose

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_spgist.c:416

目的

SP-GiSTインデックスにおいて、新しい矩形データ（BOX）をどのノード（象限）に配置するかを決定する。

シグネチャまたは形状

Datum spg_box_quad_choose(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgChooseIn（現在のノード情報と挿入対象データ）および spgChooseOut（決定結果の格納先）を受け取る。
  • **出力**: void。spgChooseOut 構造体の result.matchNode フィールドを更新することで、挿入先のノード番号（nodeN）およびデータ（restDatum）を返す。

前提条件と不変条件

  • in->prefixDatum は現在のノードの重心（centroid）として解釈可能であること。
  • in->leafDatum は挿入対象の BOX データであること。
  • allTheSame フラグが偽の場合、getQuadrant 関数により対象データが16象限のいずれかに正しく分類されること。

アルゴリズムまたは振る舞い

1. in->prefixDatum から重心を、in->leafDatum から挿入対象の矩形を取得する。 2. out->resultType を spgMatchNode に設定し、restDatum に挿入対象の矩形をセットする。 3. in->allTheSame が偽であれば、getQuadrant を呼び出し、重心に対する挿入対象矩形の相対位置（象限番号）を計算して out->result.matchNode.nodeN に格納する。 4. allTheSame が真の場合は、コアシステム側でノード選択が行われるため、本関数では nodeN の設定をスキップする。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックス構築または挿入プロセス（PostgreSQLコアのインデックスアクセス層）。
  • **呼び出し先**: getQuadrant（象限判定用）。

関連データ構造

  • spgChooseIn, spgChooseOut: SP-GiSTの挿入インターフェース用構造体。
  • BOX: 幾何データ型（矩形）。

失敗モードと境界条件

  • in->prefixDatum や in->leafDatum が不正なデータ形式である場合、DatumGetBoxP マクロ内でエラーが発生する可能性がある。
  • allTheSame が真の場合、nodeN は本関数では設定されない。

保守担当者向けメモ

  • 本関数は4次元四分木の実装の一部であり、getQuadrant と密接に連携している。
  • 象限の分割ロジックを変更する場合は、getQuadrant と spg_box_quad_picksplit の整合性を確認すること。
