errcode_apply_conflict

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/conflict.c:167

目的

論理レプリケーション中に発生した競合の種類（ConflictType）に基づき、PostgreSQLの標準エラーコード（errcode）をマッピングして返します。

シグネチャまたは形状

static int errcode_apply_conflict(ConflictType type)

入力と出力

  • **入力**: ConflictType type（競合の種類を示す列挙型）
  • **出力**: int（errcode() 関数が返すエラーコード値）

前提条件と不変条件

  • 入力される type は ConflictType 列挙型の有効な値である必要があります。
  • 定義されたいずれのケースにも該当しない場合、Assert(false) によりプログラムは停止します。

アルゴリズムまたは振る舞い

type に応じて以下のエラーコードを返します：

  • **ERRCODE_UNIQUE_VIOLATION**: CT_INSERT_EXISTS, CT_UPDATE_EXISTS, CT_MULTIPLE_UNIQUE_CONFLICTS の場合。
  • **ERRCODE_T_R_SERIALIZATION_FAILURE**: CT_UPDATE_ORIGIN_DIFFERS, CT_UPDATE_MISSING, CT_DELETE_ORIGIN_DIFFERS, CT_DELETE_MISSING の場合。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の ReportApplyConflict 関数（125行目）から呼び出されます。
  • **呼び出し先**: errcode() 関数を呼び出します。

関連データ構造

  • ConflictType: src/include/replication/conflict.h で定義されている競合タイプ列挙型。

失敗モードと境界条件

  • 定義されていない ConflictType が渡された場合、アサーション違反が発生します。

保守担当者向けメモ

  • 本関数はエラーコードの変換のみを担当します。競合の解決ロジックやログ出力の詳細は含まれません。
  • 新しい競合タイプを ConflictType に追加した場合は、本関数の switch 文にも必ずケースを追加してください。
