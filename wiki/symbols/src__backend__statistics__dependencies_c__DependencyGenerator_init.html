DependencyGenerator_init

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/dependencies.c:172

目的

関数従属性の計算に必要な属性の組み合わせを生成するための DependencyGenerator 状態を初期化し、全バリエーションを事前生成する。

シグネチャまたは形状

DependencyGenerator DependencyGenerator_init(int n, int k)

入力と出力

  • **入力**:
  • n: 属性の総数。
  • k: 組み合わせる属性の数。
  • **出力**:
  • 初期化済みの DependencyGenerator 構造体へのポインタ。

前提条件と不変条件

  • (n >= k) && (k > 0) であること（Assert により検証）。
  • メモリ割り当てが成功すること。

アルゴリズムまたは振る舞い

1. DependencyGeneratorData 構造体用にメモリを確保し、k 個の AttrNumber 配列を割り当てる。 2. 内部状態（ndependencies, current, k, n）を初期化する。 3. generate_dependencies(state) を呼び出し、属性の全組み合わせを事前生成する。 4. 初期化された状態ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/dependencies.c 内の statext_dependencies_build 関数から呼び出される。
  • **呼び出し先**: generate_dependencies 関数を呼び出す。

関連データ構造

  • DependencyGenerator: 属性の組み合わせ生成状態を保持する構造体（DependencyGeneratorData へのポインタ）。
  • AttrNumber: 属性番号を保持する配列。

失敗モードと境界条件

  • n < k または k <= 0 の場合、Assert によりプログラムが停止する。
  • メモリ不足時には palloc によりエラーが発生する。

保守担当者向けメモ

  • 本関数は DependencyGenerator_free と対で使用されることを想定している。
  • generate_dependencies による事前生成は、属性数に対して指数関数的な計算コストがかかるため、大規模なデータセットでの実行時には注意が必要。
