plpgsql_CastExprHashEntry

定義

  • **種別**: struct
  • **定義箇所**: src/pl/plpgsql/src/pl_exec.c:161

目的

PL/pgSQL 実行エンジンにおいて、型キャスト式をキャッシュするためのハッシュテーブルのエントリとして機能します。コンパイル済みのキャスト式を保持し、実行時の再計算コストを削減します。

シグネチャまたは形状

typedef struct {
    plpgsql_CastHashKey key;        /* ハッシュキー（先頭に配置必須） */
    Expr           *cast_expr;     /* キャスト式。no-opキャストの場合はNULL */
    CachedExpression *cast_cexpr;  /* 上記を保持するキャッシュ済み式 */
} plpgsql_CastExprHashEntry;

入力と出力

  • **入力**: ハッシュテーブル検索時に plpgsql_CastHashKey がキーとして使用されます。
  • **出力**: 検索成功時に、対応する cast_expr および cast_cexpr が取得されます。

前提条件と不変条件

  • key フィールドは構造体の先頭に配置されなければなりません。
  • cast_expr が NULL の場合、それは no-op（何もしない）キャストであることを示します。

アルゴリズムまたは振る舞い

  • cast_expr_hash ハッシュテーブル内で、特定の型変換要求に対するコンパイル済み式を管理します。
  • 実行時にキャストが必要な際、このエントリを介してキャッシュされた式を再利用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpgsql/src/pl_exec.c 内のハッシュテーブル操作箇所（hash_search を使用する関数など）。
  • src/pl/plpgsql/src/pl_exec.c:4042 にて、ハッシュテーブル作成時のエントリサイズとして参照されています。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出し先はありません。

関連データ構造

  • plpgsql_CastHashKey: ハッシュテーブルの検索キー。
  • Expr: PostgreSQL の式ノード。
  • CachedExpression: キャッシュされた式を管理する構造体。
  • plpgsql_CastHashEntry: 関連するキャストハッシュエントリ。

失敗モードと境界条件

  • ハッシュテーブルの初期化や検索失敗時の挙動については、呼び出し側の hash_search の実装に依存します。
  • メモリ不足時にはハッシュテーブルの拡張が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は cast_expr_hash テーブルの要素です。変更を加える際は、plpgsql_CastHashKey との整合性およびハッシュテーブルの初期化ロジック（ctl.entrysize の設定）に注意してください。
