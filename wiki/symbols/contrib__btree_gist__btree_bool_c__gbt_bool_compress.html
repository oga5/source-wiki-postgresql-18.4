gbt_bool_compress

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bool.c:88

目的

GiSTインデックスに格納する前に、boolean型の値をインデックス用の圧縮形式（boolKEY）に変換します。

シグネチャまたは形状

Datum gbt_bool_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（GISTENTRY構造体へのポインタを第1引数として含む）。
  • **出力**: Datum（圧縮されたboolKEY構造体へのポインタ）。

前提条件と不変条件

  • tinfo（gbtree_ninfo構造体）が、bool型の比較ロジックやサイズ情報を保持している必要があります。
  • 入力されたGISTENTRYは、GiSTインデックスの圧縮処理に適した状態である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. 共通ユーティリティ関数 gbt_num_compress を呼び出し、tinfo を渡して bool 型の値を boolKEY 形式に変換します。 3. 変換結果を PG_RETURN_POINTER を介して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックス構築プロセス（直接的な呼び出し元コードの詳細は提供されていません）。
  • **呼び出し先**: gbt_num_compress（contrib/btree_gist/btree_utils_num.c で定義された共通ユーティリティ関数）。

関連データ構造

  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。
  • boolKEY: bool型の範囲を表現するために使用される構造体。
  • tinfo: gbtree_ninfo 構造体。bool型のメタデータを提供します。

失敗モードと境界条件

  • 入力ポインタが不正な場合、PostgreSQLの標準的なメモリ保護機能によりエラーが発生する可能性があります。
  • 共通ユーティリティ gbt_num_compress の仕様に依存します。

保守担当者向けメモ

  • 本関数は btree_utils_num.h の数値系ユーティリティを流用する薄いラッパーです。bool型は内部的に0/1の数値として扱われます。
  • 複雑なロジックは共通ユーティリティ側に委譲されているため、変更が必要な場合は btree_utils_num.c の影響範囲を確認してください。
