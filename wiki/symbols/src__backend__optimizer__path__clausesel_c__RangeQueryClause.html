RangeQueryClause

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/path/clausesel.c:31

目的

クエリプランナが範囲条件（例: x > 10 AND x < 20）の選択率を正確に見積もるため、同一変数に対する境界条件を収集・保持する内部データ構造です。

シグネチャまたは形状

typedef struct RangeQueryClause
{
    struct RangeQueryClause *next;    /* 連結リストの次要素 */
    Node       *var;                  /* 共通の変数 */
    bool        have_lobound;         /* 下限条件の有無 */
    bool        have_hibound;         /* 上限条件の有無 */
    Selectivity lobound;              /* 下限条件の選択率 */
    Selectivity hibound;              /* 上限条件の選択率 */
} RangeQueryClause;

入力と出力

  • 入力: addRangeClause 関数により、個別の Node（条件式）がリストに追加されます。
  • 出力: 最終的に lobound および hibound フィールドに計算済みの選択率が格納され、範囲全体の選択率補正に使用されます。

前提条件と不変条件

  • var は同一の変数（列）を参照している必要があります。
  • have_lobound または have_hibound が true の場合、対応する lobound または hibound に有効な選択率が保持されます。

アルゴリズムまたは振る舞い

1. addRangeClause を通じて、クエリ内の条件式を解析し、変数ごとに RangeQueryClause を生成または更新します。 2. 範囲条件の重なりを考慮するため、単純な積ではなく、保持された境界値を用いて hiselectivity + loselectivity - 1.0 といった補正計算を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/path/clausesel.c 内の選択率計算ロジック。
  • 呼び出し先: palloc（メモリ確保用）。
  • 注記: 具体的な呼び出し元関数名はソース抜粋からは特定できません。

関連データ構造

  • Node: 条件式を表現する汎用ノード。
  • Selectivity: 選択率（float 型の別名）を保持する型。

失敗モードと境界条件

  • 境界条件: 単一の変数に対して上限・下限の両方が指定された場合、範囲の重なりを考慮した補正が必要です。
  • 失敗モード: 範囲外の条件や、解析不能な式が渡された場合の挙動については、本構造体単体では定義されていません。

保守担当者向けメモ

  • 本構造体は clausesel.c 内でのみ使用されるプライベートなデータ構造です。
  • 拡張統計（Extended Statistics）が適用される場合、この構造体による見積もりと併用される可能性があるため、変更時は clauselist_selectivity 周りのロジックとの整合性に注意してください。
