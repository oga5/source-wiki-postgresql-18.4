check_relation_block_range

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/btreefuncs.c:202

目的

指定されたブロック番号が、対象リレーションの有効な範囲内にあるかを検証する。

シグネチャまたは形状

static void check_relation_block_range(Relation rel, int64 blkno)

入力と出力

  • **入力**:
  • rel: 検証対象のリレーション（Relation）。
  • blkno: 検証するブロック番号（int64）。
  • **出力**: なし。検証に失敗した場合はエラーをスローする。

前提条件と不変条件

  • blkno は MaxBlockNumber 以下である必要がある。
  • blkno は 0 以上である必要がある。
  • rel はオープンされている必要がある。

アルゴリズムまたは振る舞い

1. blkno が負の値、または MaxBlockNumber を超えている場合にエラーを報告する。 2. RelationGetNumberOfBlocks(rel) を呼び出し、リレーションの現在の総ブロック数と比較する。 3. blkno が総ブロック数以上である場合にエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/btreefuncs.c 内の各関数（bt_page_stats 等の内部処理において、ブロックアクセス前に呼び出される）。
  • **呼び出し先**:
  • RelationGetNumberOfBlocks (PostgreSQLコアAPI)
  • ereport (PostgreSQLコアAPI)

関連データ構造

  • Relation: リレーションのメタデータおよび物理構造を保持する構造体。

失敗モードと境界条件

  • blkno < 0: ERRCODE_INVALID_PARAMETER_VALUE でエラー。
  • blkno > MaxBlockNumber: ERRCODE_INVALID_PARAMETER_VALUE でエラー。
  • blkno >= RelationGetNumberOfBlocks(rel): ERRCODE_INVALID_PARAMETER_VALUE でエラー。

保守担当者向けメモ

  • 本関数は contrib/pageinspect 内のB-tree検査関数におけるガード節として機能する。
  • int64 型の入力を BlockNumber 型へキャストして比較するため、型の範囲外チェックが重要である。
