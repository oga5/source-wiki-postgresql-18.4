BufFileReadCommon

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/buffile.c:592

目的

BufFile 構造体で管理されるバッファ付きファイルから、指定されたバイト数を読み込み、メモリ領域へコピーする共通処理を提供します。

シグネチャまたは形状

static size_t BufFileReadCommon(BufFile *file, void *ptr, size_t size, bool exact, bool eofOK)

入力と出力

  • **入力**:
  • file: 読み込み対象の BufFile 構造体へのポインタ。
  • ptr: 読み込んだデータを格納するバッファへのポインタ。
  • size: 読み込むバイト数。
  • exact: true の場合、要求されたバイト数が読み込めない場合にエラーを発生させる。
  • eofOK: exact が true の際、読み込みバイト数が 0 であれば EOF として許容する。
  • **出力**:
  • 実際に読み込まれたバイト数を size_t 型で返す。

前提条件と不変条件

  • 処理開始時に BufFileFlush(file) が呼び出され、書き込みバッファがフラッシュされます。
  • file->pos は現在のバッファ内の読み込み位置、file->nbytes はバッファ内の有効データ長を指します。
  • BufFileLoadBuffer が呼び出される際、file->pos は 0 にリセットされます。

アルゴリズムまたは振る舞い

1. BufFileFlush を実行し、保留中の書き込みを完了させます。 2. 要求された size に達するまでループし、以下の処理を繰り返します。

  • バッファが空（file->pos >= file->nbytes）の場合、BufFileLoadBuffer を呼び出し、物理ファイルからバッファへデータをロードします。データがロードできなければループを終了します。
  • バッファ内の残存データと要求サイズのうち小さい方を memcpy で ptr にコピーします。
  • file->pos、ptr ポインタ、残りの size、累積読み込み量 nread を更新します。

    • 3. exact が true で、かつ読み込み量が要求と一致しない場合（かつ eofOK の例外条件に該当しない場合）、ereport でエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/file/buffile.c 内の読み込み関数群（具体的な関数名はソース内の参照から推測可能ですが、明示的な呼び出し元リストは提供されていません）。
  • **呼び出し先**:
  • BufFileFlush
  • BufFileLoadBuffer
  • memcpy
  • ereport

関連データ構造

  • BufFile: バッファリング状態、物理ファイル記述子、現在のオフセットを保持する構造体。

失敗モードと境界条件

  • **EOF**: BufFileLoadBuffer がデータをロードできない場合、読み込みを中断し、その時点までのバイト数を返します。
  • **読み込み不足**: exact が true の場合、要求されたバイト数に満たない読み込みはエラー（ERROR）となります。ただし、eofOK が true で読み込みが 0 バイトの場合は例外として扱われます。

保守担当者向けメモ

  • 本関数は BufFile の読み込み処理の基盤であり、exact フラグによる厳密な読み込み制御を担っています。
  • 物理ファイルセグメントの切り替えは BufFileLoadBuffer 側に依存しています。
