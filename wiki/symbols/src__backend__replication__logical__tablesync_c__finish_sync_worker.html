finish_sync_worker

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/tablesync.c:142

目的

論理レプリケーションにおけるテーブル同期ワーカーの終了処理を実行し、リーダーである適用ワーカーへ完了を通知する。

シグネチャまたは形状

pg_noreturn static void finish_sync_worker(void)

入力と出力

  • 入力: なし（グローバルな MySubscription および MyLogicalRepWorker を参照）
  • 出力: なし（プロセスを終了させるため戻り値なし）

前提条件と不変条件

  • 本関数は pg_noreturn であり、呼び出し後に制御は戻らない。
  • 呼び出し時点で、同期ワーカーが実行すべきデータ同期タスクが完了している必要がある。

アルゴリズムまたは振る舞い

1. **トランザクションのコミット**: 現在のトランザクション状態であれば CommitTransactionCommand() を呼び出し、統計情報を報告する。 2. **ログのフラッシュ**: XLogFlush を実行し、書き込みを確定させる。 3. **終了ログの記録**: 新たなトランザクションを開始し、同期完了のメッセージをログに出力した後、コミットする。 4. **リーダーへの通知**: logicalrep_worker_wakeup を呼び出し、サブスクリプション ID を指定してリーダーの適用ワーカーを起床させる。 5. **プロセス終了**: proc_exit(0) を呼び出し、ワーカープロセスを正常終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/tablesync.c 内の複数箇所（同期完了判定ロジックなどから参照）。
  • 呼び出し先: CommitTransactionCommand, pgstat_report_stat, XLogFlush, GetXLogWriteRecPtr, StartTransactionCommand, ereport, logicalrep_worker_wakeup, proc_exit。

関連データ構造

  • MySubscription: 現在のサブスクリプション情報。
  • MyLogicalRepWorker: 現在の論理レプリケーションワーカーの状態。

失敗モードと境界条件

  • トランザクションの状態によっては、コミット処理がスキップされる場合がある。
  • ログ出力や通知処理でエラーが発生した場合、プロセスは異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は同期ワーカーのライフサイクルの最終段階を担う。状態遷移の整合性を保つため、呼び出し前に必ず同期状態が SYNCDONE 等の完了状態に到達していることを確認すること。
