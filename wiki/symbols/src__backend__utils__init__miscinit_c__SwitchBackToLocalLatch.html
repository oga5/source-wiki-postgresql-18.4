SwitchBackToLocalLatch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/miscinit.c:241

目的

現在プロセスが使用しているラッチ（MyLatch）を、共有メモリ上のプロセスラッチからローカルなラッチ（LocalLatchData）へ切り替えます。

シグネチャまたは形状

void SwitchBackToLocalLatch(void)

入力と出力

  • 入力: なし（グローバル変数 MyLatch、MyProc、LocalLatchData、FeBeWaitSet を参照）
  • 出力: なし（副作用として MyLatch の更新および FeBeWaitSet の修正、SetLatch の実行を行う）

前提条件と不変条件

  • MyLatch が &LocalLatchData ではないこと。
  • MyProc が NULL ではなく、かつ MyLatch が &MyProc->procLatch を指していること。

アルゴリズムまたは振る舞い

1. MyLatch を &LocalLatchData に更新します。 2. FeBeWaitSet が存在する場合、ModifyWaitEvent を呼び出し、待機イベントの対象ラッチを新しい MyLatch に更新します。 3. SetLatch(MyLatch) を呼び出し、切り替え後のローカルラッチをセット状態にします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/proc.c 内の2箇所から呼び出されています。
  • 呼び出し先: ModifyWaitEvent および SetLatch を呼び出します。

関連データ構造

  • LocalLatchData: プロセスローカルなラッチ構造体。
  • MyProc: 現在のバックエンドプロセスのプロセス制御ブロック。
  • FeBeWaitSet: フロントエンド/バックエンド通信用の待機セット。

失敗モードと境界条件

  • Assert 文により、呼び出し時に MyLatch が既にローカルラッチを指している場合や、MyProc が初期化されていない場合は異常終了します。

保守担当者向けメモ

  • 本関数は、共有メモリ上のラッチからローカルラッチへの復帰を安全に行うためのものです。FeBeWaitSet を使用している環境では、必ず待機イベントの更新が伴う点に注意してください。
