win866_to_koi8r

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:485

目的

WIN866 エンコーディングから KOI8-R エンコーディングへ文字列を変換する PostgreSQL の変換プロシージャです。

シグネチャまたは形状

Datum win866_to_koi8r(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得します。
  • src: 変換元文字列（PG_GETARG_CSTRING(2)）
  • dest: 変換先バッファ（PG_GETARG_CSTRING(3)）
  • len: 文字列長（PG_GETARG_INT32(4)）
  • noError: エラーハンドリングフラグ（PG_GETARG_BOOL(5)）
  • **出力**: 変換されたバイト数を Datum 型（int32）として返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_WIN866, PG_KOI8R) マクロにより、ソースエンコーディングが WIN866、宛先エンコーディングが KOI8-R であることが検証されます。

アルゴリズムまたは振る舞い

1. 引数から変換に必要なバッファ、長さ、フラグを取得します。 2. エンコーディングの整合性を検証します。 3. local2local 関数を呼び出し、win8662koi マッピングテーブルを使用して WIN866 から KOI8-R への変換を実行します。 4. 変換されたバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: local2local（変換実行用ユーティリティ）。

関連データ構造

  • win8662koi: WIN866 から KOI8-R への変換用マッピングテーブル。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断されます。
  • noError フラグが false の場合、変換不可能な文字が含まれるとエラーが発生する可能性があります（local2local の仕様に依存）。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_ARGS を引数に取るため、SQL インターフェース経由での利用を前提としています。直接呼び出すことは避けてください。
  • 変換ロジックを変更する場合は、win8662koi テーブルの整合性を確認してください。
