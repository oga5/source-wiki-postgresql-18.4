test_bloomfilter

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_bloomfilter/test_bloomfilter.c:112

目的

PostgreSQLのブルームフィルタ実装の精度を検証するためのSQL呼び出し可能なエントリポイント。指定されたパラメータに基づき、ブルームフィルタの偽陽性率を測定するテストを反復実行する。

シグネチャまたは形状

Datum test_bloomfilter(PG_FUNCTION_ARGS)

入力と出力

  • **入力 (SQL引数)**:
  • power (int32): ブルームフィルタのビット配列サイズ（2^power）。
  • nelements (int64): フィルタに投入する要素数。
  • seed (int32): 乱数生成器のシード値。
  • tests (int32): テストの反復回数。
  • **出力**:
  • void (PG_RETURN_VOID)

前提条件と不変条件

  • power は 23 から 32 の範囲内であること。
  • tests は 0 より大きいこと。
  • nelements は 0 以上であること。
  • 上記条件を満たさない場合、elog(ERROR) により処理が中断される。

アルゴリズムまたは振る舞い

1. 引数の妥当性を検証する。 2. 指定された tests 回数分、ループを実行する。 3. ループ内で create_and_test_bloom(power, nelements, seed) を呼び出し、個別のテストケースを実行する。 4. 処理完了後、値を返さずに終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: create_and_test_bloom（同一ファイル内の内部関数）。

関連データ構造

  • bloomfilter (lib/bloomfilter.h で定義される構造体): create_and_test_bloom 内で操作される対象。

失敗モードと境界条件

  • 引数が範囲外の場合、エラーをスローして停止する。
  • 偽陽性率が閾値（0.01）を超えた場合、create_and_test_bloom 内で ereport を介して警告が出力される。

保守担当者向けメモ

  • 本関数はテスト実行の制御のみを行い、実際のフィルタ操作は create_and_test_bloom に委譲されている。
  • power 引数の制限（23〜32）は、メモリ消費量とテストの実行時間を考慮した設計上の制約である。
