ExplainPrintSettings

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:688

目的

EXPLAIN コマンドの出力において、現在のセッション設定（GUC）情報を取得し、指定されたフォーマット（TEXT, JSON, XML等）で整形して出力する。

シグネチャまたは形状

static void ExplainPrintSettings(ExplainState *es)

入力と出力

  • **入力**: ExplainState *es（出力状態を保持する構造体）。
  • **出力**: なし（es が保持するバッファに設定情報を書き込む）。

前提条件と不変条件

  • es->settings が真である場合にのみ処理を実行する。偽の場合は即座に復帰する。
  • get_explain_guc_options を介して取得した設定項目が対象となる。

アルゴリズムまたは振る舞い

1. es->settings を確認し、要求されていない場合は終了する。 2. get_explain_guc_options を呼び出し、対象となる設定項目の一覧を取得する。 3. フォーマットが EXPLAIN_FORMAT_TEXT 以外の場合:

  • "Settings" グループを開く。
  • 各設定項目について GetConfigOptionByName で値を取得し、ExplainPropertyText で出力する。
  • "Settings" グループを閉じる。

    • 4. フォーマットが EXPLAIN_FORMAT_TEXT の場合:

  • 設定項目が存在しない場合は何も出力しない。
  • 各設定項目を key = 'value' の形式でカンマ区切りの文字列に連結する。
  • 最終的な文字列を ExplainPropertyText で出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c 内の EXPLAIN 実行フロー（詳細はソースコード上の直接的な参照箇所のみ確認済み）。
  • **呼び出し先**: get_explain_guc_options, ExplainOpenGroup, ExplainCloseGroup, GetConfigOptionByName, ExplainPropertyText, initStringInfo, appendStringInfoString, appendStringInfo。

関連データ構造

  • ExplainState: 出力形式や設定フラグを保持する。
  • config_generic: GUC設定のメタデータ構造体。

失敗モードと境界条件

  • 設定項目がない場合、TEXTモードでは何も出力されない。
  • GetConfigOptionByName が値を返さない（NULL）場合、TEXTモードでは NULL という文字列として出力される。

保守担当者向けメモ

  • 本関数は EXPLAIN の SETTINGS オプションに対応している。
  • 出力形式の追加や変更を行う際は、es->format による条件分岐が適切に行われているか確認すること。
  • get_explain_guc_options が返す設定リストの定義については、関連するGUC管理モジュールを参照すること。
