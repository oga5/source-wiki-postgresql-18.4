z_alloc

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-compress.c:57

目的

zlib ライブラリが PostgreSQL のメモリ管理機構（palloc）を使用してメモリを確保できるようにするためのラッパー関数です。

シグネチャまたは形状

static void *z_alloc(void *priv, unsigned n_items, unsigned item_len);

入力と出力

  • **入力**:
  • priv: zlib の仕様に基づく不透明なポインタ（本実装では使用されません）。
  • n_items: 確保する要素数。
  • item_len: 各要素のサイズ（バイト単位）。
  • **出力**:
  • 確保されたメモリ領域へのポインタ。

前提条件と不変条件

  • palloc が呼び出されるため、PostgreSQL のメモリコンテキストが有効である必要があります。
  • n_items * item_len の計算結果がメモリ確保可能なサイズであることを前提としています。

アルゴリズムまたは振る舞い

1. n_items と item_len を乗算し、必要な合計バイト数を算出します。 2. PostgreSQL の palloc() を呼び出し、メモリを確保して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-compress.c 内の compress_init および decompress_init において、z_stream 構造体の zalloc フィールドに本関数が代入されます。
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理関数)

関連データ構造

  • z_stream (zlib 構造体): zalloc フィールドを通じて本関数を参照します。

失敗モードと境界条件

  • palloc がメモリ不足を検知した場合、PostgreSQL のエラー処理機構により例外がスローされます。

保守担当者向けメモ

  • 本関数は zlib が PostgreSQL のメモリ管理外でメモリを確保することを防ぎ、メモリリークを防止する設計となっています。
  • 対となる解放関数として z_free が存在します。
