var_eq_non_const

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/selfuncs.c:473

目的

定数以外の値（変数や式など）に対する等価演算子（=）の選択率を推定します。

シグネチャまたは形状

double var_eq_non_const(VariableStatData *vardata, Oid oproid, Oid collation,
                        Node *other, bool varonleft, bool negate)

入力と出力

  • **入力**:
  • vardata: 変数の統計情報（VariableStatData）。
  • oproid: 演算子の OID。
  • collation: 照合順序。
  • other: 比較対象のノード。
  • varonleft: 変数が左辺にあるかを示すフラグ。
  • negate: 不等価演算子（<>）として計算するかを示すフラグ。
  • **出力**:
  • double: 0.0 から 1.0 の範囲の選択率。

前提条件と不変条件

  • vardata が有効な統計情報を持っている場合、nullfrac（NULL 割合）が考慮されます。
  • negate が真の場合、計算された選択率は 1.0 - selec - nullfrac に変換されます。
  • 最終的な結果は CLAMP_PROBABILITY マクロにより 0.0〜1.0 の範囲に収められます。

アルゴリズムまたは振る舞い

1. **一意性チェック**: vardata->isunique が真であれば、選択率を 1.0 / テーブルのタプル数 と推定します。 2. **統計情報に基づく推定**: pg_statistic が利用可能な場合、非 NULL 割合（1.0 - nullfrac）を個別の値の数（ndistinct）で除算します。 3. **MCV 補正**: 最頻値（MCV）の頻度情報が存在する場合、推定値が MCV の頻度を超えないように制限します。 4. **統計情報欠落時のフォールバック**: 統計情報がない場合、get_variable_numdistinct から得られる推定値を用いて 1.0 / ndistinct を算出します。 5. **不等価演算子への対応**: negate が指定されている場合、算出した等価選択率を反転させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/selfuncs.c 内の等価演算子選択率推定ロジック（例: 289行目付近）。
  • **呼び出し先**: get_variable_numdistinct, get_attstatsslot, free_attstatsslot 等の統計情報アクセス関数。

関連データ構造

  • VariableStatData: 変数の統計情報を保持する構造体。
  • Form_pg_statistic: システムカタログ pg_statistic の行構造。
  • AttStatsSlot: 統計情報スロット。

失敗モードと境界条件

  • vardata->rel->tuples が 0 の場合や、ndistinct が 1 以下の場合は、除算による不適切な値の生成を避けるロジックが働きます。
  • 統計情報が全く存在しない場合、デフォルトの推定値に依存するため精度が低下します。

保守担当者向けメモ

  • 本関数は eqsel の一部として、定数以外の比較対象に対して呼び出されます。
  • ndistinct の推定精度が選択率全体の精度に直結するため、統計情報の収集状況に強く依存します。
