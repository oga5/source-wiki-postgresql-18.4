hash_get_num_entries

定義

  • 種別: function
  • 定義箇所: src/backend/utils/hash/dynahash.c:1343

目的

ハッシュテーブル HTAB に現在格納されている全要素数を取得します。

シグネチャまたは形状

long hash_get_num_entries(HTAB *hashp)

入力と出力

  • **入力**: hashp (操作対象のハッシュテーブルへのポインタ)
  • **出力**: テーブル内の総要素数を示す long 型の整数値。

前提条件と不変条件

  • 本関数は、呼び出し元がテーブルの全パーティションに対してロックを保持している状況での使用を想定しています。
  • 内部でミューテックスの取得を行わないため、ロックなしで呼び出した場合、取得される値は不整合である可能性があります。

アルゴリズムまたは振る舞い

1. ハッシュテーブルのヘッダー情報 (hctl) にある最初のフリーリスト (freeList[0]) のエントリ数 nentries を初期合計値とします。 2. テーブルがパーティション分割されている場合 (IS_PARTITIONED マクロが真)、残りの全フリーリスト (freeList[1] から NUM_FREELISTS - 1 まで) の nentries を順次加算します。 3. 最終的な合計値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • contrib/pg_trgm/trgm_regexp.c
  • contrib/tablefunc/tablefunc.c
  • src/backend/access/transam/xlogutils.c
  • src/backend/catalog/pg_enum.c
  • src/backend/catalog/storage.c
  • src/backend/storage/file/reinit.c
  • **呼び出し先**:
  • ソースコード上での直接的な関数呼び出しは確認されません。

関連データ構造

  • HTAB: ハッシュテーブルの制御構造体。
  • HASHHDR: 共有メモリ上のハッシュテーブルヘッダー。
  • FreeListData: パーティションごとの管理構造体。

失敗モードと境界条件

  • hashp が無効なポインタである場合、セグメンテーション違反が発生します。
  • パーティション分割されていないテーブルの場合、freeList[0] の値のみが返されます。

保守担当者向けメモ

  • 本関数はロックを取得しないため、並行アクセスが発生する環境では、呼び出し元が適切に同期制御を行っていることを確認してください。
