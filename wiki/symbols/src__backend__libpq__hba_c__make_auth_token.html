make_auth_token

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/hba.c:258

目的

認証設定ファイル（pg_hba.conf 等）の解析中に使用されるトークン構造体（AuthToken）を動的に生成し、文字列データを格納する。

シグネチャまたは形状

static AuthToken *make_auth_token(const char *token, bool quoted)

入力と出力

  • **入力**:
  • token: 格納する文字列へのポインタ。
  • quoted: トークンが引用符で囲まれていたかどうかを示すフラグ。
  • **出力**:
  • 確保されたメモリ領域を指す AuthToken 構造体へのポインタ。

前提条件と不変条件

  • palloc0 を使用して、AuthToken 構造体と文字列データ領域を単一のメモリブロックとして確保する。
  • authtoken->string は構造体直後のメモリ領域を指す。
  • authtoken->regex は初期値として NULL に設定される。

アルゴリズムまたは振る舞い

1. 入力文字列 token の長さを取得する。 2. AuthToken 構造体のサイズと文字列長（+ 終端文字分）を合わせたメモリを palloc0 で確保する。 3. 構造体の各フィールド（quoted, regex）を初期化する。 4. 入力文字列を確保したメモリ領域の末尾にコピーする。 5. 確保した AuthToken 構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c 内のトークン化処理および設定解析処理（copy_auth_token、tokenize_auth_file、next_field_expand 等）。
  • **呼び出し先**:
  • palloc0（メモリ確保）
  • strlen / memcpy（文字列操作）

関連データ構造

  • AuthToken: hba.c で使用されるトークン保持用構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • token が NULL の場合、strlen で未定義の動作を引き起こす可能性がある（ソースコード上、呼び出し元で適切に制御されていることが前提）。

保守担当者向けメモ

  • 本関数は AuthToken 構造体と文字列を同一のメモリブロックに配置することで、メモリ管理の効率化を図っている。
  • regex フィールドは本関数では初期化のみ行われ、実際の正規表現コンパイルは regcomp_auth_token で別途行われる。
