get_auto_allocs

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/memory.c:87

目的

スレッドローカルストレージから、現在のスレッドに関連付けられた自動メモリ追跡リスト（auto_mem 構造体）の先頭ポインタを取得します。

シグネチャまたは形状

static struct auto_mem *get_auto_allocs(void)

入力と出力

  • 入力: なし
  • 出力: 現在のスレッドのメモリ追跡リストの先頭を指す struct auto_mem * 型のポインタ。リストが未初期化または空の場合は NULL を返します。

前提条件と不変条件

  • pthread_once を使用して、スレッドローカルキー auto_mem_key が初期化済みであることを保証します。
  • 呼び出し元は、返されたポインタがスレッド固有のコンテキストに依存していることを前提とします。

アルゴリズムまたは振る舞い

1. pthread_once を呼び出し、auto_mem_key_init 関数によってスレッドローカルキーの初期化が一度だけ行われるようにします。 2. pthread_getspecific を使用して、現在のスレッドに割り当てられている auto_mem 構造体のリストを取得し、そのポインタを返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/ecpglib/memory.c 内のメモリ管理関数（ecpg_auto_alloc、ECPGfree_auto_mem、ecpg_clear_auto_mem など）から参照されています。
  • 呼び出し先:
  • pthread_once
  • auto_mem_key_init
  • pthread_getspecific

関連データ構造

  • struct auto_mem: 自動解放対象のメモリブロックを保持するリスト構造体。
  • auto_mem_key: スレッドローカルストレージを識別するためのキー。

失敗モードと境界条件

  • スレッドローカルキーが初期化されていない場合、pthread_once が初期化を試みます。
  • スレッドに対してメモリリストがまだ割り当てられていない場合、pthread_getspecific は NULL を返します。

保守担当者向けメモ

  • 本関数は static であり、memory.c 内のプライベートなヘルパー関数です。
  • スレッドセーフなメモリ管理の根幹であるため、キーの初期化ロジックを変更する際は注意が必要です。
