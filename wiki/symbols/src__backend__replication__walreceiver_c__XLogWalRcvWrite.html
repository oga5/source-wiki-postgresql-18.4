XLogWalRcvWrite

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiver.c:889

目的

プライマリサーバーから受信した WAL データをローカルの WAL セグメントファイルに書き込み、共有メモリ上の書き込み進捗状況を更新します。

シグネチャまたは形状

static void XLogWalRcvWrite(char *buf, Size nbytes, XLogRecPtr recptr, TimeLineID tli)

入力と出力

  • **入力**:
  • buf: 受信した WAL データを含むバッファ。
  • nbytes: 書き込むデータのバイト数。
  • recptr: 書き込み開始位置を示す WAL ポインタ。
  • tli: タイムライン ID。
  • **出力**: なし（副作用としてファイル書き込みおよび共有メモリ更新を実行）。

前提条件と不変条件

  • tli != 0 であること。
  • 書き込み対象のセグメントファイルが適切に管理されていること（必要に応じて XLogFileInit が呼び出される）。

アルゴリズムまたは振る舞い

1. nbytes が 0 になるまでループし、WAL セグメント境界を跨ぐ書き込みを処理します。 2. 現在のセグメントが完了している場合、XLogWalRcvClose を呼び出してファイルを閉じます。 3. ファイルが開かれていない場合、XLogFileInit を使用して新しいセグメントファイルを作成します。 4. pg_pwrite を使用して、現在のセグメント内のオフセット位置へデータを書き込みます。この際、pgstat_report_wait_start を用いて I/O 待機イベントを記録します。 5. 書き込み失敗時は PANIC レベルでエラーを報告します（ディスクフル等）。 6. 書き込み完了後、LogstreamResult.Write を更新し、共有メモリ上の WalRcv->writtenUpto を最新の書き込み位置に同期させます。 7. セグメントが完全に書き込まれた場合、アーカイブ通知のために再度 XLogWalRcvClose を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walreceiver.c 内のストリーミングループ（850行目付近）。
  • **呼び出し先**:
  • XLogWalRcvClose
  • XLogFileInit
  • pg_pwrite
  • pgstat_report_wait_start / pgstat_report_wait_end
  • pgstat_count_io_op_time
  • pg_atomic_write_u64

関連データ構造

  • LogstreamResult: 書き込み進捗を保持するグローバル変数。
  • WalRcv: 共有メモリ上のレシーバー状態構造体。
  • recvFile, recvSegNo: 現在書き込み中のファイルハンドルとセグメント番号。

失敗モードと境界条件

  • **ディスクフル**: pg_pwrite が失敗し、errno が設定されていない場合は ENOSPC とみなして PANIC を発生させます。
  • **セグメント境界**: XLByteInSeg を使用してセグメント境界を判定し、必要に応じてファイルを切り替えます。

保守担当者向けメモ

  • 本関数は pg_pwrite を使用して直接ファイルシステムへ書き込みます。fsync による永続化は本関数ではなく、別途 XLogWalRcvFlush が担当します。
  • pg_stat_io 統計情報収集のため、I/O タイミングの計測が含まれています。
