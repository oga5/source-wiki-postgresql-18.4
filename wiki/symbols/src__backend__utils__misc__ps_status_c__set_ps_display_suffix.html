set_ps_display_suffix

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/ps_status.c:370

目的

現在のプロセス表示名（ps コマンド等で確認可能）の末尾に、指定された文字列（サフィックス）を追加または更新します。

シグネチャまたは形状

void set_ps_display_suffix(const char *suffix)

入力と出力

  • **入力**: suffix (追加する文字列へのポインタ)
  • **出力**: なし（副作用として内部バッファおよびOS側のプロセス名表示を更新）

前提条件と不変条件

  • PS_USE_NONE が定義されていない環境でのみ動作します。
  • update_ps_display_precheck() が真を返す場合にのみ処理が実行されます。
  • 処理終了後、strlen(ps_buffer) と ps_buffer_cur_len は一致します。

アルゴリズムまたは振る舞い

1. update_ps_display_precheck() を呼び出し、更新の可否を確認します。 2. 既にサフィックスが付与されている場合は、ps_buffer_nosuffix_len を基準にバッファを切り詰め、元の状態に戻してから新しいサフィックスを適用します。 3. バッファ容量に余裕がある場合は、スペースを挟んでサフィックスを追記します。 4. バッファ容量が不足している場合、可能な範囲で切り詰めて追記し、終端文字を補完します。 5. 最後に flush_ps_display() を呼び出し、OSに対してプロセス名の変更を通知します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/syncrep.c
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/ipc/standby.c
  • src/backend/storage/lmgr/lock.c
  • **呼び出し先**:
  • update_ps_display_precheck()
  • flush_ps_display()

関連データ構造

  • ps_buffer: プロセス名を保持するグローバルバッファ。
  • ps_buffer_cur_len: 現在のバッファ使用長。
  • ps_buffer_nosuffix_len: サフィックス付与前のバッファ長。
  • ps_buffer_size: バッファの最大容量。

失敗モードと境界条件

  • update_ps_display_precheck() が偽を返す場合、何も行わずに終了します。
  • バッファが既に満杯に近い場合、サフィックスの一部のみが書き込まれるか、あるいは書き込みが行われません。

保守担当者向けメモ

  • 本関数は ps_buffer を直接操作するため、マルチスレッド環境での安全性については ps_status.c 全体の排他制御設計に依存します。
  • set_ps_display_remove_suffix と対になる機能です。修正時は整合性に注意してください。
