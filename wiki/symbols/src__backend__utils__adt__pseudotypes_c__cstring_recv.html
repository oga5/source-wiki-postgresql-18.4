cstring_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pseudotypes.c:122

目的

バイナリ形式のデータストリームから cstring 型の値を読み取り、PostgreSQL 内部表現（Datum）に変換します。

シグネチャまたは形状

Datum cstring_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される StringInfo 型のバッファ（PG_GETARG_POINTER(0)）。
  • **出力**: 読み取った文字列を含む cstring 型の Datum。

前提条件と不変条件

  • 入力バッファは libpq のバイナリプロトコル形式に従っている必要があります。
  • pq_getmsgtext が内部でメモリ確保を行うため、呼び出し元で適切なメモリ管理が行われている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から StringInfo 型のバッファを取得します。 2. pq_getmsgtext を呼び出し、バッファの現在位置から残りのバイト数分を文字列として抽出します。 3. 抽出された文字列を PG_RETURN_CSTRING マクロを介して Datum として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: 不明（PostgreSQL の関数マネージャ経由で呼び出されることを想定）。
  • **呼び出し先**: pq_getmsgtext（libpq/pqformat.h で定義）。

関連データ構造

  • StringInfo: バイナリデータの送受信に使用される可変長バッファ構造体。
  • Datum: PostgreSQL の汎用的なデータ表現型。

失敗モードと境界条件

  • バッファの読み取り位置が不正な場合や、メモリ不足が発生した場合は pq_getmsgtext 内部でエラーが発生し、処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は cstring 型のバイナリ受信を担います。cstring は擬似型として扱われますが、本関数によりバイナリプロトコル経由でのデータ受け入れが可能です。
  • エラーハンドリングは libpq の関数に依存しています。
