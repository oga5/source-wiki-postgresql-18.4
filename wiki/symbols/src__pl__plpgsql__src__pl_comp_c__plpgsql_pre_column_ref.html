plpgsql_pre_column_ref

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_comp.c:997

目的

SQL 式の解析中に ColumnRef（列参照）が出現した際、それが PL/pgSQL 変数であるかどうかを判定・解決するためのパーサーコールバックとして機能します。

シグネチャまたは形状

static Node *plpgsql_pre_column_ref(ParseState *pstate, ColumnRef *cref)

入力と出力

  • **入力**:
  • pstate: PostgreSQL のパーサー状態。p_ref_hook_state に PLpgSQL_expr 構造体が格納されていることを前提とします。
  • cref: 解析対象の列参照ノード。
  • **出力**:
  • 解決された Node（通常は Param ノード）、または解決できない場合は NULL を返します。

前提条件と不変条件

  • pstate->p_ref_hook_state が PLpgSQL_expr 型にキャスト可能である必要があります。
  • 呼び出し元は、この関数を pstate->p_pre_columnref_hook に設定することでフックとして利用します。

アルゴリズムまたは振る舞い

1. pstate から PLpgSQL_expr を取得します。 2. 式の解決オプション (resolve_option) が PLPGSQL_RESOLVE_VARIABLE であるかを確認します。 3. 変数解決が有効な場合、resolve_column_ref を呼び出して識別子を PL/pgSQL 変数として解決し、その結果を返します。 4. 変数解決が不要な設定であれば、NULL を返してパーサーの標準的な列解決処理へ委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のパーサー（p_pre_columnref_hook を介して間接的に呼び出されます）。
  • **呼び出し先**: resolve_column_ref（内部関数）。

関連データ構造

  • ParseState: パーサーの状態管理。
  • ColumnRef: SQL パーサーが生成する列参照ノード。
  • PLpgSQL_expr: PL/pgSQL の式を保持する構造体。

失敗モードと境界条件

  • p_ref_hook_state が正しく設定されていない場合、不正なメモリ参照が発生する可能性があります。
  • resolve_column_ref が解決に失敗した場合の挙動は、その関数の実装に依存します。

保守担当者向けメモ

  • 本関数は plpgsql_parser_setup でフックとして登録されます。
  • resolve_option の判定ロジックを変更する際は、PL/pgSQL の名前解決ルール（変数優先か列優先か）との整合性に注意してください。
