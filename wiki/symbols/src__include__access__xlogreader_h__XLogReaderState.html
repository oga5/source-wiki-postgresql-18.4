XLogReaderState

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogreader.h:59

目的

PostgreSQL の WAL（Write-Ahead Log）を読み取り、解析するためのセッション状態を保持するメイン構造体です。WAL の物理的な読み込みから、バイナリデータから論理的なレコード構造へのデコードまでの一連の状態を管理します。

シグネチャまたは形状

typedef struct XLogReaderState XLogReaderState;

（詳細は src/include/access/xlogreader.h に定義）

入力と出力

  • **入力**: XLogReaderAllocate を介して初期化され、XLogReaderRoutine で定義されたコールバック関数を通じて WAL データを受け取ります。
  • **出力**: 解析済みのレコードは DecodedXLogRecord 構造体として保持され、アクセサ関数やマクロを通じて提供されます。

前提条件と不変条件

  • 本構造体は、提供されている関数やマクロ（XLogRecGet* 等）を介して操作することが推奨されます。
  • バックエンドおよびフロントエンド（pg_waldump 等）の両方で利用可能な設計となっており、バックエンド専用の ereport 等は使用できません。

アルゴリズムまたは振る舞い

1. **初期化**: XLogReaderAllocate により状態が作成され、コールバックが登録されます。 2. **読み込み**: XLogReadRecord が呼び出されると、登録された page_read コールバックが実行され、WAL ページが内部バッファにロードされます。 3. **デコード**: ロードされたバイナリデータが DecodedXLogRecord へ変換され、キューイングされます。 4. **アクセス**: 保持されたレコードに対し、マクロを用いて XID やブロック参照などのフィールドを抽出します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_walinspect/pg_walinspect.c 内の各関数（InitXLogReaderState, ReadNextXLogRecord, GetWALRecordInfo, GetWALBlockInfo など）が本構造体を参照しています。
  • **呼び出し先**: 内部で XLogReaderRoutine に登録されたコールバック関数を呼び出します。

関連データ構造

  • XLogReaderRoutine: 読み込みに必要なコールバック関数群。
  • DecodedXLogRecord: 解析済みの WAL レコード。
  • XLogRecPtr: WAL 内の論理位置を示すポインタ。

失敗モードと境界条件

  • 読み込みエラー発生時は、XLogPageReadResult を通じて通知されます。
  • XLogReaderHasQueuedRecordOrError を使用して、キュー内のレコードまたはエラー状態を確認可能です。

保守担当者向けメモ

  • 本構造体のフィールドを直接操作することは避け、提供されている API を使用してください。
  • 先読み（read-ahead）を有効にする場合は XLogReaderSetDecodeBuffer を使用して循環バッファを確保してください。
