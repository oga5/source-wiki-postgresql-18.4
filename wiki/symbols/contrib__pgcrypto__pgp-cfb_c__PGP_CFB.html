PGP_CFB

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/pgp-cfb.c:39

目的

OpenPGP 仕様に基づく CFB（Cipher Feedback）モードの暗号化および復号処理において、暗号化コンテキストと状態を保持するために使用されます。

シグネチャまたは形状

struct PGP_CFB
{
    PX_Cipher  *ciph;           /* 使用するブロック暗号 */
    int         block_size;     /* ブロックサイズ */
    int         pos;            /* 現在のブロック内位置 */
    int         block_no;       /* 現在のブロック番号 */
    int         resync;         /* 再同期モードフラグ */
    uint8       fr[PGP_MAX_BLOCK];  /* フィードバックレジスタ */
    uint8       fre[PGP_MAX_BLOCK]; /* 暗号化されたフィードバックレジスタ */
    uint8       encbuf[PGP_MAX_BLOCK]; /* 内部バッファ */
};

入力と出力

本構造体は、pgp_cfb_create で初期化され、pgp_cfb_encrypt および pgp_cfb_decrypt の引数として渡されます。各処理関数は、この構造体内の状態（fr, fre, pos 等）を更新することで、ストリームデータの連続的な処理を実現します。

前提条件と不変条件

  • ciph は PX_Cipher インターフェースを介して初期化されている必要があります。
  • resync フラグが有効な場合、特定のブロック境界で特殊な再同期ロジックが適用されます。
  • fr、fre、encbuf のサイズは PGP_MAX_BLOCK に固定されています。

アルゴリズムまたは振る舞い

  • **状態管理**: ブロック暗号の出力と入力データの XOR 演算結果をフィードバックレジスタ（fr）に保持し、次ブロックの入力として利用します。
  • **再同期処理**: resync モード時には、通常の CFB とは異なるレジスタ更新ロジック（mix_encrypt_resync / mix_decrypt_resync）が適用されます。
  • **処理フロー**: cfb_process を介して、ブロック単位の暗号化/復号とフィードバック更新が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: pgp-encrypt.c、pgp-decrypt.c などの上位層が、暗号化/復号のライフサイクル管理のために本構造体を利用します。
  • **呼び出し先**: PX_Cipher インターフェースを介して、実際のブロック暗号処理（ciph）を呼び出します。また、mix_encrypt_normal、mix_decrypt_normal、mix_encrypt_resync、mix_decrypt_resync といった内部関数が本構造体を操作します。

関連データ構造

  • PX_Cipher: 暗号化アルゴリズムの実装を抽象化する構造体。
  • PGP_MAX_BLOCK: バッファサイズを規定する定数。

失敗モードと境界条件

  • resync フラグの不適切な設定は、復号時のデータ破損を招きます。
  • pos や block_no の管理が不整合を起こすと、ストリームの同期が失われます。

保守担当者向けメモ

  • mix_encrypt_resync および mix_decrypt_resync は古い PGP 仕様への対応であり、複雑なロジックが含まれています。修正時は resync フラグの影響範囲を十分に確認してください。
  • デバッグ時は ctx->resync の値が期待通りかを確認することが重要です。
