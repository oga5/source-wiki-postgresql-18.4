FinishWalRecovery

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogrecovery.c:1476

目的

リカバリ終了時のクリーンアップ処理を行い、データベースを書き込み可能な通常状態へ移行するための情報を収集・生成する。

シグネチャまたは形状

EndOfWalRecoveryInfo *FinishWalRecovery(void)

入力と出力

  • **入力**: なし（グローバルなリカバリ状態および共有メモリ上の XLogRecoveryCtl を参照）。
  • **出力**: EndOfWalRecoveryInfo 構造体へのポインタ。リカバリ終了後のWAL書き込み開始位置、最後のWALページ、リカバリ停止理由、およびタイムライン情報を含む。

前提条件と不変条件

  • 本関数呼び出し時点で、WALの適用（Redo）は完了している必要がある。
  • WalRcvStreaming() が偽であることを前提とする（呼び出し前に XLogShutdownWalRcv() が実行されていること）。

アルゴリズムまたは振る舞い

1. **サービス停止**: XLogShutdownWalRcv() および ShutDownSlotSync() を呼び出し、WAL受信およびスロット同期を停止する。 2. **モード切替**: StandbyMode を false に設定する。 3. **終端特定**: XLogRecoveryCtl から最後にリプレイしたレコード位置を取得し、ReadRecord を用いてWALの物理的な終端（endOfLog）を確定する。 4. **アーカイブリカバリ終了**: ArchiveRecoveryRequested が真の場合、InArchiveRecovery を false にし、開いているWALファイルがあれば閉じる。 5. **部分ブロック処理**: endOfLog がブロック境界で終わっていない場合、その部分的なページ内容を result->lastPage にコピーし、呼び出し元がWAL書き込みを再開できるようにする。 6. **メタデータ収集**: リカバリ停止理由、タイムラインID、および中断されたレコード情報を result に格納して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c
  • **呼び出し先**: XLogShutdownWalRcv, ShutDownSlotSync, XLogPrefetcherBeginRead, ReadRecord, getRecoveryStopReason, palloc, memcpy, close

関連データ構造

  • EndOfWalRecoveryInfo: リカバリ終了情報を保持する構造体。
  • XLogRecoveryCtl: リカバリの進捗状態を管理する共有メモリ構造体。
  • xlogreader: WAL読み込み状態を保持する内部状態。

失敗モードと境界条件

  • endOfLog がブロック境界にある場合、lastPage は NULL となる。
  • readFile が開いている場合、Windows環境でのファイル操作競合を避けるため明示的にクローズする。
  • リカバリ中にタイムライン切り替えが発生した場合、result->endOfLogTLI は現在のセグメントのタイムラインを正しく反映する。

保守担当者向けメモ

  • 本関数はスタートアッププロセスがリカバリから通常運用へ移行する際の境界点である。
  • lastPage のコピー処理は、WALの末尾に新しいレコードを追記するための重要な初期化ステップであるため、変更時は xlogreader のバッファ状態との整合性に注意すること。
