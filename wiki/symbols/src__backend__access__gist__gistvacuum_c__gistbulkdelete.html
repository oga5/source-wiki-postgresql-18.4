gistbulkdelete

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistvacuum.c:58

目的

GiST インデックスに対する VACUUM 処理のメインエントリポイントとして、インデックス内の不要なタプルを削除し、統計情報を更新する。

シグネチャまたは形状

IndexBulkDeleteResult *gistbulkdelete(
    IndexVacuumInfo *info, 
    IndexBulkDeleteResult *stats,
    IndexBulkDeleteCallback callback, 
    void *callback_state
)

入力と出力

  • **入力**:
  • info: VACUUM 処理に関する情報。
  • stats: 統計情報を保持する構造体（初回呼び出し時は NULL）。
  • callback: 不要なタプルを判定するためのコールバック関数。
  • callback_state: コールバック関数に渡す状態データ。
  • **出力**:
  • 更新された IndexBulkDeleteResult 構造体へのポインタ。

前提条件と不変条件

  • stats が NULL の場合、内部で palloc0 を使用して新しい IndexBulkDeleteResult が割り当てられる。
  • gistvacuumscan を呼び出し、インデックス全体の物理スキャンと削除処理を実行する。

アルゴリズムまたは振る舞い

1. stats が NULL の場合、メモリを確保して初期化する。 2. gistvacuumscan を呼び出し、インデックスの物理スキャンを開始する。 3. スキャン処理内で各ページを走査し、callback を用いて不要なタプルを特定・削除する。 4. 処理完了後、更新された stats を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c: GiST インデックスの ambulkdelete メソッドとして登録されている。
  • **呼び出し先**:
  • gistvacuumscan: インデックス走査のメインループを実行する。
  • palloc0: メモリ確保に使用。

関連データ構造

  • IndexVacuumInfo: VACUUM 操作のメタデータ。
  • IndexBulkDeleteResult: 削除処理の結果（タプル数やページ数など）を格納する。
  • IndexBulkDeleteCallback: 不要タプル判定用の関数ポインタ。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる可能性がある。
  • インデックスの物理スキャン中に発生するページ分割や並行更新については、gistvacuumscan 内部のロック管理および再帰的再スキャンによって制御される。

保守担当者向けメモ

  • 本関数は GiST インデックスのメンテナンスにおける主要なエントリポイントである。
  • 実際の削除ロジックは gistvacuumscan およびその下位関数（gistvacuumpage 等）に委譲されているため、詳細な削除挙動の調査時はそちらを参照すること。
