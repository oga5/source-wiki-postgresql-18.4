pg_decode_rollback_prepared_txn

定義

  • 種別: function
  • 定義箇所: contrib/test_decoding/test_decoding.c:422

目的

論理デコーディングにおいて、準備済みトランザクション（Prepared Transaction）のロールバックイベントを検出し、その情報をテキスト形式で出力プラグインの出力バッファへ書き込む。

シグネチャまたは形状

static void pg_decode_rollback_prepared_txn(
    LogicalDecodingContext *ctx,
    ReorderBufferTXN *txn,
    XLogRecPtr prepare_end_lsn,
    TimestampTz prepare_time
)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • txn: ロールバック対象のトランザクション情報。
  • prepare_end_lsn: 準備完了時のWAL終了位置（本関数内では未使用）。
  • prepare_time: 準備完了時刻（本関数内では未使用）。
  • **出力**: なし（副作用として ctx->out に文字列が書き込まれる）。

前提条件と不変条件

  • ctx->output_plugin_private に TestDecodingData 構造体が格納されていること。
  • txn->gid が有効な文字列として存在すること。

アルゴリズムまたは振る舞い

1. OutputPluginPrepareWrite を呼び出し、出力バッファを準備する。 2. ROLLBACK PREPARED <gid> という形式でトランザクションID（GID）をバッファに書き込む。 3. data->include_xids が真の場合、txn->xid を追記する。 4. data->include_timestamp が真の場合、txn->xact_time.commit_time を文字列に変換して追記する。 5. OutputPluginWrite を呼び出し、バッファの内容を確定・出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの論理デコーディングエンジン（ReorderBuffer 経由）。ソースコード内では _PG_output_plugin_init 内でコールバックとして登録されている。
  • **呼び出し先**:
  • OutputPluginPrepareWrite
  • appendStringInfo
  • quote_literal_cstr
  • timestamptz_to_str
  • OutputPluginWrite

関連データ構造

  • LogicalDecodingContext: デコーディングの実行状態を保持。
  • ReorderBufferTXN: トランザクションのメタデータ（GID、XID、時刻など）を保持。
  • TestDecodingData: プラグインの設定（出力オプション）を保持。

失敗モードと境界条件

  • txn->gid が NULL の場合、quote_literal_cstr の呼び出しによりクラッシュする可能性がある。
  • メモリ不足時には appendStringInfo 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は _PG_output_plugin_init で rollback_prepared_cb として登録されるコールバック関数である。
  • prepare_end_lsn および prepare_time 引数は現在の実装では使用されていないが、将来的な拡張のためにシグネチャに含まれている。
