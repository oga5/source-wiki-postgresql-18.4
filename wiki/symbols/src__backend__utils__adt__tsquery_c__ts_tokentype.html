ts_tokentype

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/adt/tsquery.c:50

目的

tsquery の解析プロセスにおいて、入力文字列から抽出されたトークンの種類を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
    PT_END = 0,   /* 解析終了 */
    PT_ERR = 1,   /* 解析エラー */
    PT_VAL = 2,   /* 値（オペランド） */
    PT_OPR = 3,   /* 演算子 */
    PT_OPEN = 4,  /* 開き括弧 */
    PT_CLOSE = 5, /* 閉じ括弧 */
} ts_tokentype;

入力と出力

  • 本シンボル自体は列挙型定義であり、関数ではありません。
  • トークナイザー関数（ts_tokenizer 型）の戻り値として使用され、解析状態を後続の処理へ伝達します。

前提条件と不変条件

  • 各列挙値は、tsquery の構文解析におけるトークンの論理的な分類を網羅しています。
  • PT_END は解析の正常終了を、PT_ERR は解析不能な入力を示します。

アルゴリズムまたは振る舞い

  • parse_tsquery 等の解析関数において、現在のトークンが演算子かオペランドかを判定し、スタック操作やツリー構築の分岐条件として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/backend/utils/adt/tsquery.c: トークナイザー関数（gettoken_query_standard 等）の戻り値型として使用。
  • src/include/tsearch/ts_type.h: QueryItemType のコメント内で、演算子やオペランドの種別として言及されています。
  • **呼び出し先**:
  • なし（列挙型定義のため）。

関連データ構造

  • TSQueryParserStateData: 解析状態を保持する構造体。
  • ts_tokenizer: ts_tokentype を返す関数ポインタ型。

失敗モードと境界条件

  • PT_ERR が返された場合、解析処理は中断され、エラーハンドリング（escontext を介した報告など）へ移行します。

保守担当者向けメモ

  • 新しい演算子やトークン種別を追加する場合は、この列挙型に値を追加し、各トークナイザー関数および makepol 等の解析ロジックを更新する必要があります。
