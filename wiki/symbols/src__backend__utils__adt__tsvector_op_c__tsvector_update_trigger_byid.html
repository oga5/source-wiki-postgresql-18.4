tsvector_update_trigger_byid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:2730

目的

テーブルの更新時に、特定のカラムの内容に基づいて tsvector 型の値を自動生成または更新するためのトリガー関数インターフェースを提供します。

シグネチャまたは形状

Datum tsvector_update_trigger_byid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（トリガー実行コンテキスト情報）。
  • **出力**: Datum 型（更新後の tsvector 値、またはトリガーの戻り値としての HeapTuple）。

前提条件と不変条件

  • 本関数は PostgreSQL のトリガー関数として呼び出されることを前提としています。
  • 内部的に tsvector_update_trigger を呼び出す際、第二引数に false を渡すことで、カラム指定の挙動を制御しています。

アルゴリズムまたは振る舞い

1. fcinfo（関数呼び出し情報）を引数として受け取ります。 2. tsvector_update_trigger(fcinfo, false) を呼び出し、実際のトリガー処理を委譲します。 3. 委譲先の戻り値をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のトリガーマネージャ（SQL 文の CREATE TRIGGER により登録されたトリガー経由）。
  • **呼び出し先**: tsvector_update_trigger（同ファイル内）。

関連データ構造

  • Datum: PostgreSQL の汎用データ表現。
  • PG_FUNCTION_ARGS: 関数呼び出し時の引数コンテキスト。

失敗モードと境界条件

  • トリガーが適用されるテーブルのスキーマ定義と、本関数に渡される引数（カラム名等）が一致しない場合、実行時にエラーが発生する可能性があります。
  • 詳細は tsvector_update_trigger の実装に依存します。

保守担当者向けメモ

  • 本関数は tsvector_update_trigger への単なるラッパーです。ロジックの変更が必要な場合は、呼び出し先の tsvector_update_trigger を確認してください。
  • tsvector_update_trigger_bycolumn と対になる関数であり、トリガー定義時の引数指定方法によって使い分けられます。
