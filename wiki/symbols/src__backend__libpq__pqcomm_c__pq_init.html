pq_init

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqcomm.c:173

目的

バックエンド起動時にクライアントとの通信用ソケットを初期化し、通信に必要なバッファ、イベント待機セット、および接続状態を構築する。

シグネチャまたは形状

Port *pq_init(ClientSocket *client_sock)

入力と出力

  • **入力**: client_sock (確立済みのクライアントソケット情報)
  • **出力**: 初期化された Port 構造体へのポインタ。失敗時は ereport(FATAL, ...) によりプロセスが終了する。

前提条件と不変条件

  • client_sock は有効なソケットを保持していること。
  • TopMemoryContext が初期化済みであること。
  • 呼び出し後、通信状態を示すグローバル変数（PqSendBuffer 等）が初期化される。

アルゴリズムまたは振る舞い

1. **構造体初期化**: Port 構造体を割り当て、ソケット情報とリモートアドレスをコピーする。 2. **ローカルアドレス取得**: getsockname でローカル側の接続情報を取得する。 3. **ソケット設定**: TCP接続の場合、TCP_NODELAY および SO_KEEPALIVE を有効化し、Keepalive関連パラメータを適用する（Windows環境では SO_SNDBUF の最適化も行う）。 4. **バッファ確保**: PqSendBuffer を TopMemoryContext 上に確保し、送受信ポインタをリセットする。 5. **終了処理登録**: on_proc_exit を使用して、プロセス終了時に socket_close が呼ばれるよう登録する。 6. **ノンブロッキング設定**: 非Windows環境では pg_set_noblock でソケットをノンブロッキングモードにし、FD_CLOEXEC を設定する。 7. **イベントセット構築**: FeBeWaitSet を作成し、ソケット書き込み、ラッチ、Postmasterの死活監視イベントを登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/backend_startup.c (バックエンド起動時)
  • **呼び出し先**: palloc0, getsockname, setsockopt, getsockopt, MemoryContextAlloc, on_proc_exit, pg_set_noblock, fcntl, CreateWaitEventSet, AddWaitEventToSet

関連データ構造

  • Port: 通信セッションの状態を保持する構造体。
  • ClientSocket: 初期接続情報。
  • FeBeWaitSet: 通信イベント待機用の WaitEventSet。

失敗モードと境界条件

  • getsockname や setsockopt 等のシステムコールが失敗した場合、FATAL エラーとしてプロセスを終了する。
  • TCP接続以外（AF_UNIX等）では、TCP固有のオプション設定はスキップされる。

保守担当者向けメモ

  • 本関数はバックエンドの通信基盤の起点である。
  • FeBeWaitSet のイベント順序は FeBeWaitSetSocketPos 等の定数と一致している必要があるため、イベント追加順序を変更する際は注意すること。
  • Windows環境特有のバッファ最適化ロジックが含まれている。
