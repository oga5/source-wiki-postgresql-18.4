test_empty

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_integerset/test_integerset.c:483

目的

IntegerSet が空の状態における基本的な動作（メンバ判定およびイテレーション）が正しく機能することを検証する。

シグネチャまたは形状

static void test_empty(void)

入力と出力

  • 入力: なし
  • 出力: なし（検証失敗時に elog(ERROR) を発行）

前提条件と不変条件

  • intset_create() によって生成された直後の IntegerSet インスタンスに対して実行される。
  • 空の集合に対して intset_is_member を呼び出した場合、いかなる値に対しても false を返す必要がある。
  • 空の集合に対してイテレータを初期化し intset_iterate_next を呼び出した場合、値は取得されず false を返す必要がある。

アルゴリズムまたは振る舞い

1. intset_create() で空の IntegerSet を生成する。 2. intset_is_member を使用し、0、1、および PG_UINT64_MAX に対して存在確認を行い、すべて false であることを検証する。 3. intset_begin_iterate でイテレーションを開始し、intset_iterate_next が値を返さない（false を返す）ことを検証する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/test_integerset/test_integerset.c 内の test_integerset 関数から呼び出される。
  • 呼び出し先: intset_create, intset_is_member, intset_begin_iterate, intset_iterate_next（すべて lib/integerset.h で定義された API）。
  • 注意: src/test/modules/test_radixtree/test_radixtree.c にも同名の関数が存在するが、本ページは test_integerset.c 内の定義を対象としている。

関連データ構造

  • IntegerSet: テスト対象のデータ構造。

失敗モードと境界条件

  • intset_is_member が空集合に対して true を返した場合、エラーが発生する。
  • intset_iterate_next が空集合に対して値を取得した場合、エラーが発生する。

保守担当者向けメモ

  • 本関数は IntegerSet の初期状態における挙動を保証する最小限のテストである。実装変更により intset_create の初期化ロジックが変わった場合、本テストが最初の検知ポイントとなる。
