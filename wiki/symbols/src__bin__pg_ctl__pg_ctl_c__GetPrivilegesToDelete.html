GetPrivilegesToDelete

定義

  • 種別: function
  • 定義箇所: src/bin/pg_ctl/pg_ctl.c:1907

目的

指定されたアクセストークンから、特定の特権（SE_LOCK_MEMORY_NAME および SE_CHANGE_NOTIFY_NAME）を除外した特権リストを生成して返します。

シグネチャまたは形状

static PTOKEN_PRIVILEGES GetPrivilegesToDelete(HANDLE hToken)

入力と出力

  • **入力**: hToken (Windows のアクセストークンハンドル)
  • **出力**: 成功時は除外処理済みの PTOKEN_PRIVILEGES 構造体へのポインタ。失敗時は NULL を返します。

前提条件と不変条件

  • Windows 環境での実行を前提としています。
  • pg_malloc_extended を使用してメモリを確保するため、呼び出し元は返されたポインタに対して適切な解放処理を行う必要があります。

アルゴリズムまたは振る舞い

1. LookupPrivilegeValue を使用して、対象となる特権（メモリロック、変更通知）の LUID を取得します。 2. GetTokenInformation を使用して、トークン内の現在の特権リストのサイズを特定します。 3. pg_malloc_extended で必要なメモリを確保し、再度 GetTokenInformation で特権リストの内容をコピーします。 4. 特権リストをループで走査し、対象の LUID と一致する特権をリストから削除（後続の要素を前方にシフトしてカウントを減算）します。 5. 編集済みの特権リストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_ctl/pg_ctl.c:1818
  • **呼び出し先**: LookupPrivilegeValue, GetTokenInformation, pg_malloc_extended, write_stderr, free

関連データ構造

  • PTOKEN_PRIVILEGES (Windows API 構造体)
  • LUID (Windows API 構造体)

失敗モードと境界条件

  • LookupPrivilegeValue が失敗した場合、エラーメッセージを出力し NULL を返します。
  • GetTokenInformation が失敗した場合、エラーメッセージを出力し NULL を返します（メモリ確保失敗時は free を実行）。
  • メモリ不足時は pg_malloc_extended が失敗し、エラーメッセージを出力して NULL を返します。

保守担当者向けメモ

  • 本関数は Windows 特有のプロセス権限管理の一部です。特権の追加・削除が必要な場合は、LookupPrivilegeValue の対象定数を確認してください。
  • 戻り値のメモリ管理（free の要否）については、呼び出し元のライフサイクルと整合しているか注意してください。
