initStringInfoFromString

定義

  • 種別: static inline function
  • 定義箇所: src/include/lib/stringinfo.h:174

目的

既存のメモリバッファを StringInfo 構造体に割り当て、初期化を行う。バッファの動的拡張を前提としない、または既存の文字列をラップして利用する場合に使用される。

シグネチャまたは形状

static inline void initStringInfoFromString(StringInfo str, char *data, int len)

入力と出力

  • **入力**:
  • str: 初期化対象の StringInfo 構造体へのポインタ。
  • data: 文字列データが格納されたバッファへのポインタ。
  • len: バッファ内の文字列の長さ（strlen(data) に相当）。
  • **出力**: なし（str が指す構造体の各フィールドが更新される）。

前提条件と不変条件

  • data[len] がヌル終端文字 ('\0') であること（Assert により検証される）。
  • data は少なくとも len + 1 バイトの領域を確保している必要がある。

アルゴリズムまたは振る舞い

1. data[len] が '\0' であることを検証する。 2. str->data に data ポインタを代入する。 3. str->len を len に設定する。 4. str->maxlen を len + 1 に設定する。 5. str->cursor を 0 に初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/proto.c:909 での呼び出しが確認されている。
  • **呼び出し先**:
  • 本関数は Assert マクロを除き、他の関数を呼び出さない。

関連データ構造

  • StringInfoData: 本関数が初期化する対象の構造体。
  • StringInfo: StringInfoData へのポインタ型。

失敗モードと境界条件

  • data[len] がヌル終端でない場合、アサーションエラーが発生する。
  • maxlen が len + 1 に固定されるため、この関数で初期化した StringInfo に対して append 系関数を呼び出すと、バッファ容量不足によりメモリ再割り当てが発生する可能性がある。

保守担当者向けメモ

  • 本関数は既存のバッファをラップする目的で使用される。メモリ管理（pfree 等）の所有権は呼び出し元にある点に注意すること。
  • maxlen が len + 1 に設定されるため、読み取り専用として扱う場合は initReadOnlyStringInfo の使用を検討すること。
