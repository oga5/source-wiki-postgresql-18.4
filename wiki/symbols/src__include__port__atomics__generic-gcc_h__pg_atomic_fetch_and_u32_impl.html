pg_atomic_fetch_and_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic-gcc.h:224

目的

pg_atomic_uint32 型の変数に対して、指定された値とのビット単位の論理積（AND）を原子的に実行し、操作前の元の値を返します。

シグネチャまたは形状

static inline uint32 pg_atomic_fetch_and_u32_impl(volatile pg_atomic_uint32 *ptr, uint32 and_)

入力と出力

  • **入力**:
  • ptr: 操作対象となる pg_atomic_uint32 構造体へのポインタ（volatile 修飾あり）。
  • and_: 論理積をとる対象の uint32 値。
  • **出力**:
  • 操作が実行される直前の、ptr が指す値（uint32）。

前提条件と不変条件

  • 本関数は atomics.h を経由して呼び出されることを前提としています。
  • ptr は有効な pg_atomic_uint32 構造体を指している必要があります。

アルゴリズムまたは振る舞い

GCCの組み込み関数 __sync_fetch_and_and を呼び出し、ptr->value に対して and_ とのビット単位の論理積を原子的に適用します。操作はインライン展開されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/port/atomics.h:399
  • src/include/port/atomics/generic.h:195
  • **呼び出し先**:
  • __sync_fetch_and_and（コンパイラ組み込み関数）

関連データ構造

  • pg_atomic_uint32: 原子操作対象の値を保持する構造体。

失敗モードと境界条件

  • ptr が NULL の場合や、適切にアライメントされていないメモリを指している場合の動作は未定義です。
  • 本実装は __sync 系組み込み関数に依存しており、ターゲットアーキテクチャがこの組み込み関数をサポートしている必要があります。

保守担当者向けメモ

  • 本関数は generic-gcc.h 内で定義されており、プラットフォーム固有の最適化が適用されない場合のフォールバックとして機能します。
  • __atomic API が利用可能な環境であっても、本実装は __sync API を使用しています。必要に応じて __atomic_fetch_and への移行を検討してください。
