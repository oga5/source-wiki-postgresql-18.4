swap_catalog_files

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/relfilenumber.c:362

目的

pg_upgrade の --swap モードにおいて、新旧のデータベースディレクトリ間でカタログファイルを物理的に入れ替える。

シグネチャまたは形状

static void swap_catalog_files(FileNameMap *maps, int size, const char *old_catalog_dir, const char *new_db_dir, const char *moved_db_dir)

入力と出力

  • maps: 移行対象のファイルマッピング配列。
  • size: maps 配列の要素数。
  • old_catalog_dir: 旧カタログファイルを退避させるディレクトリパス。
  • new_db_dir: 現在のデータベースディレクトリパス。
  • moved_db_dir: 新しいカタログファイルが格納されているディレクトリパス。
  • 出力: なし（成功時）。失敗時は pg_fatal によりプロセスを終了する。

前提条件と不変条件

  • maps 内の relfilenumber に該当するファイルは入れ替え対象外としてスキップされる。
  • new_db_dir および moved_db_dir は有効なディレクトリとしてオープン可能であること。

アルゴリズムまたは振る舞い

1. **旧カタログの退避**: new_db_dir 内の全ファイルを走査し、maps に含まれない通常ファイルを old_catalog_dir へ rename する。 2. **新カタログの配置**: moved_db_dir 内の全ファイルを走査し、maps に含まれない通常ファイルを new_db_dir へ rename する。 3. **同期処理**: user_opts.do_sync が有効な場合、移動した各ファイルを sync_queue_push に登録する。 4. **永続化**: 最後に new_db_dir 自体とその親ディレクトリに対して fsync_fname を実行し、変更をディスクへフラッシュする。

呼び出し元と呼び出し先

  • 呼び出し元: do_swap (同一ファイル内)
  • 呼び出し先: opendir, readdir, closedir, parse_relfilenumber, RelFileNumberIsValid, bsearch, FileNameMapCmp, rename, sync_queue_push, fsync_fname, fsync_parent_path, pg_fatal

関連データ構造

  • FileNameMap: ファイルの対応関係を保持する構造体。
  • RelFileNumber: リレーションファイル番号。

失敗モードと境界条件

  • ディレクトリのオープン失敗、ファイルのリネーム失敗、またはディレクトリの同期（fsync）失敗時に pg_fatal を呼び出し、処理を中断する。
  • readdir 実行中にエラーが発生した場合も同様に終了する。

保守担当者向けメモ

  • 本関数は pg_restore 時に fsync=off で作成されたカタログファイルを対象とするため、明示的な同期処理が含まれている。
  • ユーザーデータファイルは既に永続化されていると仮定し、カタログファイルのみを同期対象としている点に注意すること。
