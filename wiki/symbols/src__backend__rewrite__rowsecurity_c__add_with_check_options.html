add_with_check_options

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rowsecurity.c:795

目的

行レベルセキュリティ（RLS）において、データ挿入・更新時に適用される WITH CHECK 制約をクエリツリーに注入する。

シグネチャまたは形状

static void add_with_check_options(
    Relation rel,
    int rt_index,
    WCOKind kind,
    List *permissive_policies,
    List *restrictive_policies,
    List **withCheckOptions,
    bool *hasSubLinks,
    bool force_using
)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • rt_index: リレーションの範囲テーブルインデックス。
  • kind: WCOKind（INSERT, UPDATE等）。
  • permissive_policies, restrictive_policies: 適用対象のポリシーリスト。
  • force_using: WITH CHECK 句の代わりに USING 句の条件を使用するかを指定するフラグ。
  • **出力**:
  • withCheckOptions: 生成された WithCheckOption ノードが追加されるリスト。
  • hasSubLinks: サブリンクの有無を示すフラグ（更新される）。

前提条件と不変条件

  • permissive_policies が空の場合、デフォルト拒否（常に偽となる WithCheckOption）が生成される。
  • permissive_policies が存在する場合、それらは OR で結合され、単一の WithCheckOption として追加される。
  • restrictive_policies はそれぞれ個別の WithCheckOption として追加され、AND 条件として機能する。

アルゴリズムまたは振る舞い

1. QUAL_FOR_WCO マクロを用い、ポリシーから適用すべき条件式（qual）を抽出する。 2. permissive_policies を走査し、条件式を lappend で収集する。 3. permissive_quals が存在する場合:

  • 複数の条件がある場合は OR 式にまとめ、単一の WithCheckOption を作成して withCheckOptions に追加する。
  • restrictive_policies を走査し、各ポリシーに対して個別の WithCheckOption を作成して追加する（ポリシー名を含めるため）。

    • 4. permissive_quals が存在しない場合:

  • 常に偽（false）を返す WithCheckOption を作成し、デフォルト拒否を適用する。

    • 5. ChangeVarNodes を呼び出し、条件式内の変数ノードを現在のリレーションの rt_index に適合させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rowsecurity.c 内の各所（get_row_security_policies 等から、INSERT/UPDATE/MERGE処理時に呼び出される）。
  • **呼び出し先**: lappend, linitial, makeNode, makeBoolExpr, makeConst, ChangeVarNodes, list_append_unique, pstrdup, RelationGetRelationName 等のPostgreSQL内部ユーティリティ関数。

関連データ構造

  • RowSecurityPolicy: ポリシー定義を保持する構造体。
  • WithCheckOption: クエリツリーに注入される制約ノード。
  • WCOKind: 制約の種類（INSERT, UPDATE等）。

失敗モードと境界条件

  • ポリシーが一つも定義されていない場合、デフォルトで「すべての書き込みを拒否する」制約が生成される。
  • qual が NULL のポリシーは無視される。

保守担当者向けメモ

  • 本関数は add_security_quals と類似したロジックを持つが、書き込み制限（WithCheckOption）に特化している。
  • restrictive_policies はエラーレポート時にポリシー名を表示するため、個別に WithCheckOption を作成している点に注意すること。
