ReservePrivateRefCountEntry

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/bufmgr.c:258

目的

バックエンドごとのピン留め回数を追跡するための PrivateRefCountEntry を確保します。配列に空きがない場合は、既存のエントリをハッシュテーブルへ退避させることで、新しいエントリのためのスロットを強制的に確保します。

シグネチャまたは形状

static void ReservePrivateRefCountEntry(void)

入力と出力

  • 入力: なし（グローバルな ReservedRefCountEntry および PrivateRefCountArray を参照）
  • 出力: なし（副作用として ReservedRefCountEntry が有効なエントリを指すように更新される）

前提条件と不変条件

  • ReservedRefCountEntry が既に設定されている場合、本関数は何も行わずに即座に復帰します。
  • 配列が満杯でハッシュテーブルへの退避が発生する場合、退避対象の buffer は InvalidBuffer ではないことが保証されます。

アルゴリズムまたは振る舞い

1. **既存確認**: ReservedRefCountEntry が既に確保済みであれば終了します。 2. **配列検索**: PrivateRefCountArray を走査し、buffer が InvalidBuffer である空きスロットを探します。見つかればそれを ReservedRefCountEntry に割り当てて終了します。 3. **ハッシュテーブルへの退避**: 配列が満杯の場合、PrivateRefCountClock を用いて循環的に退避対象（victim）を選択します。

  • 選択したスロットの情報を PrivateRefCountHash に登録します。
  • 登録後、当該配列スロットを InvalidBuffer にリセットし、ReservedRefCountEntry として再利用します。
  • PrivateRefCountOverflowed カウンタをインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/bufmgr.c 内の複数の箇所（例: 行番号 397, 693, 2015, 2356, 2724, 3926 など）から呼び出されています。
  • **呼び出し先**: hash_search（ハッシュテーブル操作用）を呼び出します。

関連データ構造

  • PrivateRefCountEntry: ピン留め回数を保持する構造体。
  • PrivateRefCountArray: 固定サイズの配列。
  • PrivateRefCountHash: 配列から溢れたエントリを保持するハッシュテーブル。

失敗モードと境界条件

  • **配列満杯時**: 全てのスロットが使用中の場合、クロックアルゴリズムにより強制的にハッシュテーブルへ退避が行われます。
  • **アサーション**: 退避対象が InvalidBuffer である場合や、ハッシュテーブルへの挿入時に既にエントリが存在する場合は Assert により停止します。

保守担当者向けメモ

  • 本関数は、リソース所有者（ResourceOwner）の管理下でバッファのピン留めを行う前に呼び出される必要があります。
  • PrivateRefCountClock による退避は、配列の利用効率を維持するための仕組みです。ハッシュテーブルへの退避が頻発するとパフォーマンスに影響を与える可能性があるため、REFCOUNT_ARRAY_ENTRIES のサイズ設定には注意してください。
