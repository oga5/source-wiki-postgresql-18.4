InitCatalogCache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/syscache.c:109

目的

システムカタログの高速ルックアップを実現するため、バックエンド起動時に各システムキャッシュ（CatCache）を初期化し、関連するOIDリストを構築する。

シグネチャまたは形状

void InitCatalogCache(void)

入力と出力

  • 入力: なし（グローバルな cacheinfo 配列および SysCacheSize を参照）
  • 出力: なし（副作用としてグローバル変数 SysCache[] 配列、CacheInitialized フラグ、およびOIDリストが更新される）

前提条件と不変条件

  • CacheInitialized が false であること。
  • cacheinfo 配列内の各要素が有効なリレーションOIDおよびインデックスOIDを保持していること。
  • キャッシュ対象のリレーションが RelationInvalidatesSnapshotsOnly を返さないこと。

アルゴリズムまたは振る舞い

1. CacheInitialized が偽であることを確認する。 2. cacheinfo 配列を走査し、各定義に基づいて InitCatCache を呼び出し、SysCache[] 配列に格納する。 3. 初期化中にリレーションOIDおよびインデックスOIDを収集し、SysCacheRelationOid および SysCacheSupportingRelOid 配列に蓄積する。 4. 収集したOID配列を qsort でソートし、qunique で重複排除を行う（バイナリサーチを可能にするため）。 5. 最後に CacheInitialized を true に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c
  • 呼び出し先: InitCatCache（catcache.c 内の関数）、qsort、qunique

関連データ構造

  • SysCache[]: 各システムキャッシュを保持する CatCache 構造体の配列。
  • cacheinfo: キャッシュのメタデータを保持する構造体配列。
  • SysCacheRelationOid / SysCacheSupportingRelOid: キャッシュ対象のリレーションOIDを保持する配列。

失敗モードと境界条件

  • InitCatCache が有効なポインタを返さない場合、elog(ERROR) により初期化を中断する。
  • OID配列のサイズが定義された上限（lengthof）を超えた場合、Assert により停止する。

保守担当者向けメモ

  • 新しいキャッシュを追加する場合は syscache_info.h を編集すること。
  • OIDリストのソートと重複排除は、キャッシュの検索効率と整合性に直結するため変更時は注意が必要である。
