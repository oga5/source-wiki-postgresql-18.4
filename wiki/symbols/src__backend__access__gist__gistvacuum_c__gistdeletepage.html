gistdeletepage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistvacuum.c:630

目的

GiSTインデックスにおいて、空になったリーフページをツリーから切り離し、親ページから対応するダウンリンクを削除する。

シグネチャまたは形状

static bool gistdeletepage(IndexVacuumInfo *info, IndexBulkDeleteResult *stats,
                           Buffer parentBuffer, OffsetNumber downlink,
                           Buffer leafBuffer)

入力と出力

  • **入力**:
  • info: インデックスのVACUUM情報。
  • stats: 削除統計情報を保持する構造体。
  • parentBuffer: 親ページのバッファ。
  • downlink: 親ページ内での対象リーフページへのオフセット番号。
  • leafBuffer: 削除対象のリーフページバッファ。
  • **出力**:
  • bool: 削除に成功した場合は true、並行処理による競合や条件不一致で削除を見送った場合は false を返す。

前提条件と不変条件

  • 対象の leafBuffer はリーフページであること。
  • ページが空（PageGetMaxOffsetNumber が InvalidOffsetNumber）であること。
  • 並行したページ分割（GistFollowRight）が発生していないこと。
  • 親ページが削除済みやリーフページではないこと。

アルゴリズムまたは振る舞い

1. **検証**: ページがリーフであるか、空であるか、並行分割中ではないかをチェックする。 2. **ダウンリンク確認**: 親ページのダウンリンクが対象リーフを指しているかを確認する。不一致の場合は削除を中止する。 3. **削除処理**:

  • 現在のトランザクションIDを取得し、ページを削除状態（GistPageSetDeleted）にマークする。
  • 親ページから PageIndexTupleDelete を用いてダウンリンクを削除する。
  • WALが必要な場合は gistXLogPageDelete を呼び出し、LSNを更新する。

    • 4. **統計更新**: stats の削除ページ数をインクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistvacuum.c 内の gistvacuum_delete_empty_pages から呼び出される。
  • **呼び出し先**: BufferGetPage, GistPageIsLeaf, GistFollowRight, PageGetMaxOffsetNumber, PageIsNew, GistPageIsDeleted, PageGetItemId, PageGetItem, ItemPointerGetBlockNumber, ReadNextFullTransactionId, START_CRIT_SECTION, MarkBufferDirty, GistPageSetDeleted, PageIndexTupleDelete, RelationNeedsWAL, gistXLogPageDelete, gistGetFakeLSN, PageSetLSN, END_CRIT_SECTION。

関連データ構造

  • IndexVacuumInfo: VACUUM実行時のメタデータ。
  • IndexBulkDeleteResult: 削除結果を記録する統計構造体。
  • Buffer: PostgreSQLのバッファ管理構造体。

失敗モードと境界条件

  • **並行処理**: 挿入や分割が進行中の場合、安全のために削除をスキップする。
  • **ダウンリンク不一致**: 親ページのダウンリンクが移動している場合、再スキャンは行わず false を返して終了する。
  • **アサーション**: リーフページが非リーフページに変化している場合など、予期せぬ状態では Assert(false) が発生する。

保守担当者向けメモ

  • 削除されたページは即座に再利用されず、トランザクションIDによる保護が行われる。これは進行中のスキャンが古いダウンリンクを参照している可能性があるためである。
  • 削除処理はクリティカルセクション内で行われる。
