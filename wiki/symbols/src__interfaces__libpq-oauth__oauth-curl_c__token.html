token

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq-oauth/oauth-curl.c:191

目的

OAuth 2.0 認可サーバーから返されるトークンレスポンス、またはエラーレスポンスを保持するためのデータ構造です。

シグネチャまたは形状

struct token
{
    char *access_token;
    char *token_type;
    struct token_error err;
};

入力と出力

本構造体は、認可サーバーからの JSON レスポンスをパースした結果を格納します。

  • access_token および token_type: 成功時のレスポンスから抽出された文字列。
  • err: 失敗時のレスポンスから抽出されたエラー情報。

前提条件と不変条件

  • access_token および token_type は、成功時に割り当てられます。
  • err フィールドは、OAuth プロトコル上のエラー発生時に token_error 構造体の定義に従って設定されます。

アルゴリズムまたは振る舞い

oauth_parse 関数等を通じて、認可サーバーからの JSON レスポンスを解析し、その結果を本構造体にマッピングします。成功時はアクセストークン情報を、失敗時はエラー詳細を保持し、後続の認証フロー判断に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: oauth_parse などの JSON 解析処理を行う関数。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • 注記: ツールが提示した contrib/ 配下の token への言及は、本構造体とは無関係な文字列解析等の別シンボルを指すものであり、本構造体の呼び出し元ではありません。

関連データ構造

  • token_error: エラーレスポンスの詳細を保持するためにネストされています。
  • async_ctx: 認証フローのコンテキスト内で、取得したトークン情報を管理するために使用されます。

失敗モードと境界条件

  • JSON パース失敗時や、必須フィールド（access_token 等）が欠落している場合、構造体内の各フィールドは未初期化または空の状態となります。
  • メモリ割り当て失敗時は、各文字列ポインタが NULL になる可能性があります。

保守担当者向けメモ

  • 本構造体は libpq-oauth の認証フロー専用です。contrib/ ディレクトリ等で見られる他の token という名称のシンボルと混同しないよう注意してください。
  • access_token 等の文字列ポインタは、不要になった際に free_token 関数を用いて適切に解放する必要があります。
