check_file_excluded

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/filemap.c:409

目的

指定されたファイルパスが、pg_rewind の同期対象から除外されるべき一時ファイル、ディレクトリ、または特定のファイル名パターンに該当するかを判定する。

シグネチャまたは形状

static bool check_file_excluded(const char *path, bool is_source)

入力と出力

  • **入力**:
  • path: 検査対象のファイルパス（文字列）。
  • is_source: 呼び出し元がソースディレクトリかターゲットディレクトリかを示すフラグ。ログ出力の文脈で使用される。
  • **出力**:
  • bool: 除外対象であれば true、それ以外は false を返す。

前提条件と不変条件

  • excludeFiles および excludeDirContents という外部の除外リストが定義されている必要がある。
  • path は有効な文字列ポインタであること。

アルゴリズムまたは振る舞い

1. **一時ファイルの判定**: パスに PG_TEMP_FILE_PREFIX または PG_TEMP_FILES_DIR が含まれる場合、即座に true を返す。 2. **個別ファイルの判定**: excludeFiles リストを走査し、パスの末尾（ファイル名部分）がリスト内の名前と一致するかを確認する。一致した場合、ログを出力して true を返す。 3. **ディレクトリの判定**: excludeDirContents リストを走査し、パスが除外対象ディレクトリの配下にあるかを確認する。一致した場合、ログを出力して true を返す。 4. 上記のいずれにも該当しない場合、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/filemap.c 内のファイル処理ロジック（例: process_source_file 等）から参照されている。
  • **呼び出し先**: strstr, strlen, last_dir_separator, strncmp, snprintf, pg_log_debug などの標準ライブラリおよび内部ユーティリティ関数。

関連データ構造

  • excludeFiles: 除外対象のファイル名とマッチング条件を保持する配列。
  • excludeDirContents: 除外対象のディレクトリパスを保持する文字列配列。

失敗モードと境界条件

  • path がディレクトリの区切り文字を含まない場合、ファイル名全体を対象として excludeFiles と比較する。
  • excludeDirContents の判定では、パスの先頭一致（strstr の結果がパスの先頭と一致）を条件としているため、サブディレクトリの包含関係が正しく判定される。

保守担当者向けメモ

  • 本関数で使用される除外リスト（excludeFiles, excludeDirContents）は basebackup.c と同期させる必要がある。修正時は整合性に注意すること。
  • pg_log_debug によるログ出力は、is_source フラグに基づいてソースかターゲットかを区別する。
