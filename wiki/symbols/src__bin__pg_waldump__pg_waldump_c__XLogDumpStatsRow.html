XLogDumpStatsRow

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/pg_waldump.c:584

目的

WALの統計情報を表形式で標準出力（stdout）に整形して表示する。

シグネチャまたは形状

static void XLogDumpStatsRow(const char *name,
                             uint64 n, uint64 total_count,
                             uint64 rec_len, uint64 total_rec_len,
                             uint64 fpi_len, uint64 total_fpi_len,
                             uint64 tot_len, uint64 total_len)

入力と出力

  • **入力**:
  • name: 行のラベル（リソースマネージャ名など）。
  • n, rec_len, fpi_len, tot_len: 各項目の現在の値。
  • total_count, total_rec_len, total_fpi_len, total_len: 各項目の合計値（パーセンテージ計算の分母）。
  • **出力**: なし（printf による標準出力への書き出しのみ）。

前提条件と不変条件

  • 各合計値（total_count 等）が0の場合、対応するパーセンテージ計算は0として扱われる。
  • name は文字列として適切に終端されている必要がある。

アルゴリズムまたは振る舞い

1. 各項目について、現在の値が合計値に占める割合（%）を計算する。 2. 合計値が0の場合は、除算を避けるため割合を0に設定する。 3. ラベル名、各値、および計算した割合を、固定幅のフォーマットで printf を用いて出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_waldump/pg_waldump.c 内の統計表示処理（XLogDumpDisplayStats 等から呼び出される）。
  • **呼び出し先**:
  • 標準ライブラリの printf。

関連データ構造

  • 特になし（引数として渡されるプリミティブ型のみを使用）。

失敗モードと境界条件

  • 合計値が0の場合でも、ゼロ除算を回避するロジックが組み込まれているため、実行時エラーは発生しない。
  • 出力先のストリームが閉じられている場合、printf の振る舞いに依存する。

保守担当者向けメモ

  • 出力フォーマットを変更する場合は、printf の書式指定子と、対応する PRIu64 マクロの整合性に注意すること。
  • 本関数は static であり、pg_waldump.c 内でのみ使用される。
