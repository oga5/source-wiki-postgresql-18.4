ExecReScanTidScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidscan.c:457

目的

TidScan ノードの再スキャン（Rescan）処理を実行し、パラメータ変更等に伴うスキャン状態の初期化を行う。

シグネチャまたは形状

void ExecReScanTidScan(TidScanState *node)

入力と出力

  • **入力**: TidScanState *node（再スキャン対象の TidScan ノード状態）
  • **出力**: なし（副作用として node 内部の状態がリセットされる）

前提条件と不変条件

  • node は有効な TidScanState 構造体へのポインタであること。
  • tss_TidList が割り当てられている場合、メモリリークを防ぐため pfree が実行される。

アルゴリズムまたは振る舞い

1. **TID リストの解放**: node->tss_TidList が存在する場合、pfree でメモリを解放し、ポインタを NULL に設定する。 2. **状態変数のリセット**: tss_NumTids を 0 に、tss_TidPtr を -1 に初期化する。 3. **テーブルスキャンのリセット**: node->ss.ss_currentScanDesc が存在する場合、table_rescan を呼び出し、テーブルアクセスメソッド側のスキャン状態をリセットする。 4. **共通スキャン状態の更新**: ExecScanReScan を呼び出し、ScanState レベルの共通的な再スキャン処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:198（ExecReScan 経由での呼び出しが確認されている）。
  • **呼び出し先**:
  • pfree（メモリ解放）
  • table_rescan（テーブルアクセスメソッドの再スキャン）
  • ExecScanReScan（共通スキャン状態の再スキャン）

関連データ構造

  • TidScanState: TidScan ノードの実行時状態を保持する構造体。
  • ScanState: TidScanState が継承する共通スキャン状態。

失敗モードと境界条件

  • node->tss_TidList が NULL の場合、解放処理はスキップされるため安全である。
  • node->ss.ss_currentScanDesc が NULL の場合、table_rescan は呼び出されない。

保守担当者向けメモ

  • 本関数は TidScan ノードの実行時パラメータが変更された際に呼び出される。
  • TidListEval は初回実行時に遅延評価されるため、本関数でリストをクリアすることで、次回の TidNext 呼び出し時に再評価が強制される仕組みとなっている。
