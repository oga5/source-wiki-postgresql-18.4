analyzeCTE

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_cte.c:242

目的

WITH 句内の個別の共通テーブル式（CTE）を解析し、CommonTableExpr 構造体内の未解析のクエリを Query ノードへ変換・検証します。

シグネチャまたは形状

static void analyzeCTE(ParseState *pstate, CommonTableExpr *cte)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体。
  • cte: 解析対象の CommonTableExpr 構造体。
  • **出力**: なし（副作用として cte->ctequery が Query ノードに書き換えられ、cycle_clause 等のメタデータが更新されます）。

前提条件と不変条件

  • cte->ctequery は解析前であること（Assert(!IsA(cte->ctequery, Query))）。
  • 再帰的CTEの場合、出力列の型、照合順序、型修飾子が事前に決定されていること。

アルゴリズムまたは振る舞い

1. **CYCLE句の処理**: cycle_clause が存在する場合、マーク値とデフォルト値を解析し、共通の型・照合順序へ強制変換（coerce）します。また、等価演算子および否定演算子（neop）を特定します。 2. **クエリ解析**: parse_sub_analyze を呼び出し、CTE内のクエリを Query ノードへ変換します。 3. **妥当性チェック**:

  • 変換結果が Query ノードであることを確認します。
  • データ変更を伴うCTE（INSERT/UPDATE/DELETE）がトップレベル以外にある場合はエラーとします。
  • query->canSetTag を false に設定します。

    • 4. **型・照合順序の検証**:

  • 非再帰的CTEの場合、analyzeCTETargetList を呼び出します。
  • 再帰的CTEの場合、非再帰項で決定された型・照合順序と、実際のクエリ結果を比較し、不一致があればエラーを報告します。

    • 5. **SEARCH/CYCLE句の検証**: 句が存在する場合、CTEが再帰的であることを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_cte.c 内の transformWithClause（直接的な呼び出しが確認されています）。
  • **呼び出し先**: transformExpr, select_common_type, coerce_to_common_type, lookup_type_cache, parse_sub_analyze, analyzeCTETargetList 等。

関連データ構造

  • CommonTableExpr: 解析対象のCTE定義。
  • ParseState: 解析状態。
  • CTECycleClause, CTESearchClause: CTEのオプション句。
  • Query: 解析後のクエリノード。

失敗モードと境界条件

  • **型不一致**: 再帰的CTEにおいて、非再帰項と再帰項の列型や照合順序が一致しない場合に ERRCODE_DATATYPE_MISMATCH または ERRCODE_COLLATION_MISMATCH を発生させます。
  • **演算子欠落**: CYCLE 句で使用する型の等価演算子が見つからない場合にエラーとなります。
  • **配置制限**: データ変更を伴うCTEがサブクエリ内にある場合、ERRCODE_FEATURE_NOT_SUPPORTED を発生させます。
  • **構文エラー**: 再帰的でないCTEに対して SEARCH または CYCLE 句が指定されている場合にエラーとなります。

保守担当者向けメモ

  • 再帰的CTEの型検証ロジックは、非再帰項と再帰項の不一致を許容しない厳格な設計です。修正が必要な場合は cte->ctecoltypes 等のメタデータ管理との整合性に注意してください。
  • cycle_clause の解析は parse_sub_analyze の前に行われるため、CTEクエリ内でサイクルマーク列を参照する際の型解決に影響を与えます。
