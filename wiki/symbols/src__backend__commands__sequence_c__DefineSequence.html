DefineSequence

定義

  • 種別: function
  • 定義箇所: src/backend/commands/sequence.c:120

目的

SQLの CREATE SEQUENCE コマンドを実行し、シーケンスオブジェクトの作成、初期化、およびシステムカタログへの登録を行う。

シグネチャまたは形状

ObjectAddress DefineSequence(ParseState *pstate, CreateSeqStmt *seq)

入力と出力

  • **入力**:
  • pstate: パーサー状態。
  • seq: CreateSeqStmt 構造体（シーケンスのオプションや名前情報を含む）。
  • **出力**:
  • 作成されたシーケンスの ObjectAddress を返す。IF NOT EXISTS 指定時に既存のシーケンスが存在する場合は InvalidObjectAddress を返す。

前提条件と不変条件

  • シーケンスは内部的にヒープテーブルとして作成される。
  • IF NOT EXISTS が指定されている場合、既存オブジェクトの所有権や存在チェックが行われる。
  • シーケンスのデータ（last_value, log_cnt, is_called）は初期化時に特定のデフォルト値（is_called は false など）で設定される。

アルゴリズムまたは振る舞い

1. **存在確認**: if_not_exists が真の場合、対象のシーケンスが既に存在するか確認し、存在すれば処理をスキップする。 2. **パラメータ解析**: init_params を呼び出し、シーケンスのオプションを検証・設定する。 3. **リレーション作成**: DefineRelation を呼び出し、シーケンス用のヒープテーブルを作成する。 4. **データ初期化**: fill_seq_with_data を使用して、シーケンスの初期状態を物理ページに書き込む。 5. **所有権設定**: owned_by が指定されている場合、process_owned_by を呼び出して関連付けを行う。 6. **カタログ登録**: pg_sequence システムカタログにシーケンスの定義情報を挿入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c (SQLコマンド実行時)
  • **呼び出し先**:
  • DefineRelation: シーケンス用テーブルの作成。
  • fill_seq_with_data: シーケンスデータの初期化。
  • CatalogTupleInsert: pg_sequence への登録。
  • process_owned_by: OWNED BY 句の処理。
  • init_params: オプションの解析。

関連データ構造

  • CreateSeqStmt: SQL文の解析結果を保持。
  • FormData_pg_sequence: pg_sequence カタログの行データ。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • 既に同名のオブジェクトが存在し、IF NOT EXISTS が指定されていない場合は DefineRelation がエラーを発生させる。
  • 権限不足や名前空間の競合が発生した場合、PostgreSQLの標準的なエラー処理により中断される。

保守担当者向けメモ

  • 本関数はシーケンスの物理的な作成とカタログ登録の双方を担う。
  • シーケンスの物理データは fill_seq_with_data で管理されるため、データ構造の変更時は同関数との整合性に注意すること。
