descriptor_item_name

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:226

目的

SQL記述子領域（Descriptor Area）の操作項目を表す列挙型 ECPGdtype を、対応するSQL標準の文字列名に変換します。

シグネチャまたは形状

static const char *descriptor_item_name(enum ECPGdtype itemcode);

入力と出力

  • **入力**: itemcode (enum ECPGdtype) - 変換対象の記述子項目コード。
  • **出力**: const char * - 項目名を表す文字列リテラル。一致する項目がない場合は NULL を返します。

前提条件と不変条件

  • 入力される itemcode は ECPGdtype 列挙型で定義された値である必要があります。
  • 返される文字列は静的メモリ領域を指しており、呼び出し側で解放する必要はありません。

アルゴリズムまたは振る舞い

switch 文を使用して itemcode を評価し、対応する文字列（例: ECPGd_count に対して "COUNT"）を返します。デフォルトケースでは NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c 内の記述子操作に関連する処理（results->value を引数として渡す箇所など）。
  • **呼び出し先**:
  • 本関数は外部関数を呼び出しません。

関連データ構造

  • enum ECPGdtype: 記述子操作の項目を定義する列挙型。

失敗モードと境界条件

  • 定義されていない itemcode が渡された場合、NULL を返します。呼び出し側でこの戻り値に対するチェックが必要です。

保守担当者向けメモ

  • 新しい記述子項目が ECPGdtype に追加された場合は、本関数の switch 文にもケースを追加してください。
