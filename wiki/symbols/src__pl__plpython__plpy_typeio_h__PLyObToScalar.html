PLyObToScalar

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_typeio.h:93

目的

PythonオブジェクトからPostgreSQLのスカラ型（Datum）へ変換する際に必要なメタデータと関数情報を保持します。

シグネチャまたは形状

typedef struct PLyObToScalar
{
    FmgrInfo    typfunc;        /* lookup info for type's input function */
    Oid         typioparam;     /* argument to pass to it */
} PLyObToScalar;

入力と出力

  • **入力**: PythonオブジェクトをPostgreSQLの Datum に変換するための型情報。
  • **出力**: typfunc（入力関数情報）および typioparam（入力関数に渡すパラメータ）を格納した構造体。

前提条件と不変条件

  • typfunc は、PostgreSQLの関数マネージャ（fmgr.h）によって初期化された有効な FmgrInfo である必要があります。
  • typioparam は、対象となるデータ型の入力関数が期待する型情報（OID）と一致している必要があります。

アルゴリズムまたは振る舞い

本構造体は、PythonからPostgreSQLへの変換処理（PLy_output_convert）において、特定のスカラ型を変換するための設定情報を保持します。実行時には、この構造体内の typfunc を使用して、Pythonオブジェクトから変換された値をPostgreSQLの内部表現である Datum へと変換します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_typeio.h:141 において、PLyObToDatum 構造体の一部（scalar メンバ）として参照されています。
  • **呼び出し先**: 具体的な関数呼び出しの証拠は提供されていません。

関連データ構造

  • PLyObToDatum: 本構造体を含む、PythonからPostgreSQLへの変換状態を管理する上位構造体。
  • FmgrInfo: PostgreSQLの関数呼び出し情報を保持する構造体。

失敗モードと境界条件

  • typfunc が適切に初期化されていない場合、変換実行時に fmgr 経由でエラーが発生する可能性があります。
  • typioparam が不正なOIDである場合、型変換関数が予期しない動作をする可能性があります。

保守担当者向けメモ

  • 本構造体は plpy_typeio.c での利用を前提としています。
  • 変換処理は再帰的に行われる可能性があるため、メモリコンテキストの管理には注意してください。
