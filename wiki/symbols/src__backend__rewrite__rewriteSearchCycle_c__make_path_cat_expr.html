make_path_cat_expr

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteSearchCycle.c:179

目的

再帰的CTEの SEARCH および CYCLE 句において、再帰ステップごとにパス情報を記録するための配列連結式（ARRAY_CAT）を生成する。

シグネチャまたは形状

static Expr *make_path_cat_expr(RowExpr *rowexpr, AttrNumber path_varattno)

入力と出力

  • **入力**:
  • rowexpr: 現在の行データを表す RowExpr。
  • path_varattno: 既存のパス配列を保持する列の属性番号。
  • **出力**:
  • Expr *: ARRAY_CAT 関数を呼び出し、既存のパス配列に新しい行要素を追加する FuncExpr。

前提条件と不変条件

  • RECORDARRAYOID 型および RECORDOID 型が使用可能であること。
  • path_varattno は、再帰クエリのコンテキストにおいて有効な列を参照していること。

アルゴリズムまたは振る舞い

1. 入力された rowexpr を単一要素として持つ ArrayExpr を構築する。 2. F_ARRAY_CAT 関数を呼び出す FuncExpr を生成する。 3. FuncExpr の引数として、既存のパス配列（path_varattno で指定された Var ノード）と、手順1で作成した ArrayExpr を渡す。 4. 構築された FuncExpr を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/rewrite/rewriteSearchCycle.c:535
  • src/backend/rewrite/rewriteSearchCycle.c:581
  • **呼び出し先**:
  • makeNode (マクロ)
  • list_make1, list_make2
  • makeFuncExpr
  • makeVar

関連データ構造

  • RowExpr: パスに追加される行データ。
  • ArrayExpr: 連結対象の配列構造。
  • FuncExpr: 生成される ARRAY_CAT 呼び出し。

失敗モードと境界条件

  • メモリ不足によりノード生成が失敗した場合、PostgreSQL の標準的なエラーハンドリングによりアボートされる。
  • path_varattno が無効な場合、後続の解析フェーズでエラーとなる可能性がある。

保守担当者向けメモ

  • 本関数は RECORDARRAYOID を前提としているため、将来的にパスの型定義が変更された場合は、型IDの整合性に注意すること。
  • makeVar の引数 varno が 1 に固定されている点は、再帰的CTEの構造に依存しているため、クエリツリーの構成変更時には再確認が必要。
