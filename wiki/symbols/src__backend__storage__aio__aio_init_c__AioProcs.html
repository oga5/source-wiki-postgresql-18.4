AioProcs

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_init.c:37

目的

AIO サブシステムが管理対象とするプロセス数の総数を算出します。

シグネチャまたは形状

uint32 AioProcs(void)

入力と出力

  • 入力: なし
  • 出力: MaxBackends と NUM_AUXILIARY_PROCS の合計値を uint32 型で返します。

前提条件と不変条件

  • MaxBackends および NUM_AUXILIARY_PROCS がシステム内で定義されている必要があります。
  • AIO ワーカープロセスに対しても、ProcNumber が割り当てられる可能性があることを考慮した計算結果となります。

アルゴリズムまたは振る舞い

MaxBackends（最大バックエンド数）に NUM_AUXILIARY_PROCS（補助プロセス数）を加算した値を返します。これは、共有メモリ上の AIO 関連構造体（PgAioBackend 等）の割り当て範囲を決定するための基準値として使用されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/aio/aio_init.c 内の各所（共有メモリサイズの計算、io_handle_count や iovec_count の設定、ループ処理の終了条件、プロセス番号の検証に使用）。
  • 呼び出し先:
  • 情報なし。

関連データ構造

  • PgAioBackend: 本関数が返す値を基に、共有メモリ上に配列として確保されます。

失敗モードと境界条件

  • 戻り値は uint32 型であるため、システム設定値が極端に大きい場合はオーバーフローの可能性がありますが、PostgreSQL のプロセス制限範囲内では通常発生しません。

保守担当者向けメモ

  • 本関数は AIO サブシステムにおける「プロセス数」の定義そのものです。共有メモリレイアウトを変更する際は、この計算式がシステム全体のメモリ消費量に直結することを留意してください。
