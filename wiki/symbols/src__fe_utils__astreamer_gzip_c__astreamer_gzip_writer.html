astreamer_gzip_writer

定義

  • 種別: struct
  • 定義箇所: src/fe_utils/astreamer_gzip.c:39

目的

astreamer インターフェースを介して、入力データを gzip 形式で圧縮し、ファイルへ書き出すための構造体です。

シグネチャまたは形状

typedef struct astreamer_gzip_writer
{
    astreamer    base;
    char        *pathname;
    gzFile       gzfile;
} astreamer_gzip_writer;

入力と出力

  • **入力**: astreamer インターフェースの content メソッドを通じて渡されるデータストリーム。
  • **出力**: gzfile を介したファイルシステムへの gzip 圧縮済みデータ書き出し。

前提条件と不変条件

  • HAVE_LIBZ が定義されている環境でのみ利用可能です。
  • gzfile は astreamer_gzip_writer_new によって初期化されている必要があります。
  • 本構造体はファイル書き込み専用であり、圧縮結果を別の astreamer へ転送することはできません。

アルゴリズムまたは振る舞い

  • astreamer の基底構造体 base を継承しています。
  • content メソッド呼び出し時、内部の gzwrite を使用してデータを圧縮・出力します。
  • finalize メソッド呼び出し時、gzclose を実行してリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_gzip_writer_new（インスタンス生成）、astreamer_gzip_writer_content、astreamer_gzip_writer_finalize、astreamer_gzip_writer_free 等の関数から参照されます。
  • **呼び出し先**: zlib ライブラリの gzwrite および gzclose を呼び出します。

関連データ構造

  • astreamer: 本構造体が継承する基底インターフェース。
  • gzFile: zlib が提供する圧縮ファイルハンドル。

失敗モードと境界条件

  • zlib の初期化失敗や書き込みエラーが発生した場合、get_gz_error を通じてエラー情報が取得されます。
  • gzclose は内部で close() を実行するため、ファイルハンドル管理には注意が必要です（astreamer_gzip_writer_new では dup() を使用して元のハンドルを保護しています）。

保守担当者向けメモ

  • 本構造体は astreamer_gzip_decompressor とは非対称な設計となっており、書き込み専用である点に留意してください。
  • astreamer インターフェースの拡張を行う際は、base メンバの整合性を維持してください。
