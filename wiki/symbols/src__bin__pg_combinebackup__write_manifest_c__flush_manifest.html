flush_manifest

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/write_manifest.c:243

目的

メモリ上のバッファに蓄積されたマニフェストデータをディスクへ書き出し、必要に応じてマニフェスト全体のチェックサム計算を更新する。

シグネチャまたは形状

static void flush_manifest(manifest_writer *mwriter)

入力と出力

  • **入力**: manifest_writer *mwriter（マニフェスト生成の状態を保持する構造体）
  • **出力**: なし（副作用としてディスク書き込みおよびバッファのクリアを行う）

前提条件と不変条件

  • mwriter->buf に書き込むべきデータが存在する場合、そのデータは mwriter->buf.data に格納されていること。
  • mwriter->fd が -1 の場合、ファイルは新規作成される。
  • mwriter->still_checksumming が真の場合、書き出されたデータは mwriter->manifest_ctx に反映される。

アルゴリズムまたは振る舞い

1. **ファイルオープン**: mwriter->fd が -1 の場合、mwriter->pathname を O_WRONLY | O_CREAT | O_EXCL モードで開き、ファイル記述子を保持する。失敗時は pg_fatal を呼び出す。 2. **ディスク書き込み**: バッファ mwriter->buf にデータが存在する場合、その内容をファイルへ書き出す。書き込みバイト数がバッファ長と一致しない場合は pg_fatal を呼び出す。 3. **チェックサム更新**: mwriter->still_checksumming が有効な場合、書き出したデータを pg_checksum_update に渡し、manifest_ctx を更新する。失敗時は pg_fatal を呼び出す。 4. **バッファリセット**: resetStringInfo を呼び出し、バッファを空にする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/write_manifest.c 内の各関数（バッファが一定サイズを超えた際や終了処理時に呼び出される）。
  • **呼び出し先**: open, write, pg_fatal, pg_checksum_update, resetStringInfo。

関連データ構造

  • manifest_writer: マニフェスト生成の状態（ファイル記述子、パス、バッファ、チェックサムコンテキスト）を管理する構造体。

失敗モードと境界条件

  • **ファイルオープン失敗**: 指定パスへの書き込み権限がない、または既にファイルが存在する場合に pg_fatal で終了する。
  • **書き込み失敗**: ディスク容量不足や I/O エラーにより書き込みが完了しない場合に pg_fatal で終了する。
  • **チェックサム計算失敗**: pg_checksum_update がエラーを返した場合に pg_fatal で終了する。

保守担当者向けメモ

  • 本関数は StringInfo バッファをフラッシュする役割を持つ。チェックサム計算の整合性を保つため、必ず書き出しとセットで manifest_ctx の更新が行われる設計となっている。
  • pg_fatal を使用しているため、呼び出し元は本関数が成功することを前提として処理を継続する。
