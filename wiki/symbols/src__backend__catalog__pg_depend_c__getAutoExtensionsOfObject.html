getAutoExtensionsOfObject

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:779

目的

指定されたオブジェクト（classId および objectId）が依存している拡張機能（Extension）のうち、DEPENDENCY_AUTO_EXTENSION 型のものを検索し、その OID のリストを返します。

シグネチャまたは形状

List *getAutoExtensionsOfObject(Oid classId, Oid objectId)

入力と出力

  • **入力**:
  • classId (Oid): 対象オブジェクトのカタログ ID。
  • objectId (Oid): 対象オブジェクトの OID。
  • **出力**:
  • List *: 対象オブジェクトが自動的に依存している拡張機能の OID リスト。該当がない場合は NIL。

前提条件と不変条件

  • pg_depend システムカタログに対する AccessShareLock を取得して読み取りを行います。
  • DependDependerIndexId インデックスを使用して検索を行います。

アルゴリズムまたは振る舞い

1. pg_depend テーブルを AccessShareLock で開きます。 2. classId と objectId をキーとして、DependDependerIndexId インデックスを用いたスキャンを開始します。 3. スキャン結果の各レコードに対し、以下を検証します。

  • refclassid が ExtensionRelationId であること。
  • deptype が DEPENDENCY_AUTO_EXTENSION であること。

    • 4. 条件を満たすレコードの refobjid（拡張機能の OID）をリストに追加します。 5. スキャンを終了し、テーブルをクローズしてリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c:515
  • **呼び出し先**:
  • table_open, ScanKeyInit, systable_beginscan, systable_getnext, lappend_oid, systable_endscan, table_close（すべて PostgreSQL 内部 API）

関連データ構造

  • pg_depend: 依存関係を保持するシステムカタログ。
  • List: PostgreSQL のリスト構造。
  • DEPENDENCY_AUTO_EXTENSION: 拡張機能による自動依存関係を示す列挙値。

失敗モードと境界条件

  • 対象オブジェクトが存在しない場合や、拡張機能に依存していない場合は空のリスト（NIL）を返します。
  • データベースアクセスエラーが発生した場合、table_open 等の内部関数により例外がスローされる可能性があります。

保守担当者向けメモ

  • 本関数は DEPENDENCY_AUTO_EXTENSION 型の依存関係のみを抽出します。他の依存関係（DEPENDENCY_NORMAL 等）は無視されます。
  • pg_depend のインデックス構造に依存しているため、カタログ定義の変更時には注意が必要です。
