TimestampDifferenceMicroseconds

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/timestamp.h:89

目的

2つのタイムスタンプ（TimestampTz）間の差分をマイクロ秒単位の uint64 値として算出する。

シグネチャまたは形状

static inline uint64 TimestampDifferenceMicroseconds(TimestampTz start_time, TimestampTz stop_time)

入力と出力

  • **入力**:
  • start_time: 開始時刻（TimestampTz 型）
  • stop_time: 終了時刻（TimestampTz 型）
  • **出力**:
  • uint64: 2つの時刻の差分（マイクロ秒）。start_time が stop_time 以降の場合は 0 を返す。

前提条件と不変条件

  • TimestampTz は内部的に int64 として表現される。
  • 本関数は、負の差分（start_time > stop_time）を許容せず、境界値として 0 に正規化する。

アルゴリズムまたは振る舞い

1. start_time が stop_time 以上であるか判定する。 2. 真であれば 0 を返す。 3. 偽であれば、stop_time から start_time を減算し、その結果を uint64 にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/postgres.c: 接続タイミング（ソケット作成、フォーク開始、認証開始）の計測に使用されている。
  • **呼び出し先**:
  • 特になし（組み込みの整数演算のみ）。

関連データ構造

  • TimestampTz: PostgreSQLのタイムスタンプ型（内部表現は int64）。

失敗モードと境界条件

  • start_time > stop_time: 負の値を返さず、0 を返す。
  • オーバーフロー: TimestampTz の値域を超えるような極端な差分については考慮されていない（通常の時刻計算の範囲内を想定）。

保守担当者向けメモ

  • 本関数はヘッダーファイル内で static inline として定義されており、呼び出し側でインライン展開されることでオーバーヘッドを最小化する設計となっている。
  • 算術演算は整数演算として行われるため、精度は TimestampTz の定義に依存する。
