PLy_plan_cursor

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_planobject.c:116

目的

PLyPlan オブジェクトからカーソルを作成するためのPythonメソッドインターフェースを提供します。

シグネチャまたは形状

static PyObject *PLy_plan_cursor(PyObject *self, PyObject *args);

入力と出力

  • **入力**:
  • self: PLyPlan オブジェクト（実行計画）。
  • args: Pythonの引数タプル。オプションとして、計画実行時に使用する引数リスト（planargs）を受け取ります。
  • **出力**:
  • 成功時: カーソルオブジェクト（PyObject*）。
  • 失敗時: NULL。

前提条件と不変条件

  • PyArg_ParseTuple を使用して、引数がオプションのオブジェクト（|O）として正しく解析可能である必要があります。

アルゴリズムまたは振る舞い

1. PyArg_ParseTuple を呼び出し、引数から planargs を抽出します。 2. 抽出した self（計画）と planargs を引数として PLy_cursor_plan を呼び出し、処理を委譲します。 3. PLy_cursor_plan の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: Pythonインタープリタ（PLyPlan オブジェクトのメソッド cursor として登録）。
  • **呼び出し先**: PLy_cursor_plan（src/pl/plpython/plpy_cursorobject.c 内の関数と推測されますが、本ソース excerpt には定義が含まれていません）。

関連データ構造

  • PLyPlanObject: 実行計画を保持する構造体。
  • PyObject: Pythonの汎用オブジェクト型。

失敗モードと境界条件

  • PyArg_ParseTuple が引数の解析に失敗した場合、NULL を返します。
  • PLy_cursor_plan が内部でエラーを発生させた場合、その結果が伝播されます。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のカーソル生成ロジックは PLy_cursor_plan に依存しています。
  • PLyPlan オブジェクトのメソッドテーブル（src/pl/plpython/plpy_planobject.c:23）に cursor として登録されています。
