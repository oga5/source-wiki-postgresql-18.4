ECPGis_noind_null

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/misc.c:360

目的

ECPGにおいて、インジケータ変数を使用せずにNULL値を表現するために定義された特殊値（最小値や特定のビットパターン）を判定する。

シグネチャまたは形状

bool ECPGis_noind_null(enum ECPGttype type, const void *ptr)

入力と出力

  • **入力**:
  • type: 判定対象のECPG型（enum ECPGttype）。
  • ptr: 判定対象のデータへのポインタ。
  • **出力**:
  • bool: 指定されたデータがNULLと見なされる場合は true、それ以外は false を返す。

前提条件と不変条件

  • ptr は type に対応する有効なメモリ領域を指している必要がある。
  • 浮動小数点数、interval、timestamp 型については、内部で _check 関数による検証が行われる。

アルゴリズムまたは振る舞い

type に応じて以下のルールでNULL判定を行う：

  • **整数型**: SHRT_MIN, INT_MIN, LONG_MIN, LONG_LONG_MIN と一致する場合にNULLと判定。
  • **文字・文字列型**: 先頭バイトが \0 の場合にNULLと判定。
  • **VARCHAR/BYTEA**: ECPGgeneric_varchar の配列先頭が 0x00、または ECPGgeneric_bytea の長さが 0 の場合にNULLと判定。
  • **DECIMAL/NUMERIC**: sign フィールドが NUMERIC_NULL と一致する場合にNULLと判定。
  • **その他**: _check 関数を呼び出す型（float, double, interval, timestamp）を除き、該当しない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c:1051
  • src/interfaces/ecpg/ecpglib/execute.c:551
  • **呼び出し先**:
  • _check（内部関数）

関連データ構造

  • enum ECPGttype: ECPGの型定義。
  • ECPGgeneric_varchar, ECPGgeneric_bytea: 可変長データ構造。
  • decimal, numeric: 数値型構造体。

失敗モードと境界条件

  • type が定義されたケースに含まれない場合、デフォルトで false を返す。
  • ptr が NULL ポインタである場合の挙動は、本ソースコード内では明示的にガードされていない。

保守担当者向けメモ

  • 新しい型をECPGに追加した際は、本関数の switch 文に該当するNULL判定ロジックを追加する必要がある。
  • _check 関数の実装詳細については、本ファイル内の定義を参照すること。
