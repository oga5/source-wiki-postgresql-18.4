EnableStandbyMode

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogrecovery.c:484

目的

PostgreSQLをスタンバイモードへ移行させ、リカバリ進行状況のログ出力を抑制する。

シグネチャまたは形状

static void EnableStandbyMode(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • 本関数は、リカバリプロセスの初期化または実行中に呼び出されることを想定している。
  • スタンバイモードが有効化されると、昇格（プロモーション）が行われない限り、システムはリカバリ状態を維持する。

アルゴリズムまたは振る舞い

1. グローバル変数 StandbyMode を true に設定する。 2. disable_startup_progress_timeout() を呼び出し、スタートアップ進行状況のタイムアウト監視を無効化する。これにより、スタンバイモード中における不要な startup_progress_timeout_handler() の呼び出しを回避する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogrecovery.c 内の複数箇所（初期化およびリカバリ制御フロー内）。
  • 呼び出し先: disable_startup_progress_timeout()。

関連データ構造

  • StandbyMode (グローバル変数): スタンバイモードの有効/無効を保持するフラグ。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。

保守担当者向けメモ

  • 本関数は、スタンバイモード特有のログ出力抑制とタイマー管理を行うための補助的な関数である。
  • スタンバイモードの切り替えロジックを変更する場合は、StandbyMode フラグを参照している他のリカバリ関連関数への影響を考慮すること。
