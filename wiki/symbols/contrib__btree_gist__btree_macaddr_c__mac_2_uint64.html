mac_2_uint64

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr.c:94

目的

macaddr 型の値を uint64 型の数値に変換します。主に GiST インデックスのペナルティ計算において、範囲の広さを数値的に算出するために使用されます。

シグネチャまたは形状

static uint64 mac_2_uint64(macaddr *m)

入力と出力

  • **入力**: macaddr *m （変換対象のMACアドレスへのポインタ）
  • **出力**: uint64 （6バイトのMACアドレスをビッグエンディアン形式で格納した数値）

前提条件と不変条件

  • 入力される macaddr は6バイトのデータ構造であることを前提としています。
  • uint64 型の範囲内に収まることが保証されています。

アルゴリズムまたは振る舞い

1. macaddr ポインタを unsigned char 配列として扱います。 2. 6バイトの各オクテットをループで走査します。 3. 各バイトを適切なビット位置（上位バイトから順に 40, 32, 24, 16, 8, 0 ビット）へシフトし、uint64 型の変数 res に加算します。 4. 最終的な 64 ビットの数値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_macaddr.c 内の gbt_macad_penalty 関数（直接の呼び出しが確認されています）。
  • **呼び出し先**:
  • 特筆すべき外部関数呼び出しはありません。

関連データ構造

  • macaddr: PostgreSQLの標準的なMACアドレス型。

失敗モードと境界条件

  • 特段の失敗モードはありません。入力ポインタが NULL でない限り、常に計算結果を返します。

保守担当者向けメモ

  • macaddr は6バイトであるため、uint64 への変換は情報の欠落なく行われます。
  • ビットシフトの計算式 (5 - i) * 8 は、ネットワークバイトオーダー（ビッグエンディアン）での数値化を意図しています。
