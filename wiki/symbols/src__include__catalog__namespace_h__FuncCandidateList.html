FuncCandidateList

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/namespace.h:29

目的

関数や演算子のオーバーロード解決時に、候補となる関数群をリスト形式で保持するための構造体です。

シグネチャまたは形状

typedef struct _FuncCandidateList
{
    struct _FuncCandidateList *next;
    int         pathpos;        /* 名前空間検索の内部用 */
    Oid         oid;            /* 関数または演算子のOID */
    int         nominalnargs;   /* pronargs または proallargtypes の長さ */
    int         nargs;          /* 返された引数の型数 */
    int         nvargs;         /* 可変長引数配列となる引数の数 */
    int         ndargs;         /* デフォルト引数の数 */
    int        *argnumbers;     /* 名前付き引数呼び出し時の位置インデックス */
    Oid         args[FLEXIBLE_ARRAY_MEMBER]; /* 引数の型 */
} *FuncCandidateList;

入力と出力

  • 本構造体は、関数解決プロセスにおいて候補リストのノードとして機能します。
  • args フィールドは FLEXIBLE_ARRAY_MEMBER を使用しており、可変長の引数型配列を保持します。

前提条件と不変条件

  • リストの各ノードは next ポインタにより連結されます。
  • args は構造体の末尾に配置され、動的に確保されたメモリ領域を指すか、構造体自体に続く領域として扱われます。

アルゴリズムまたは振る舞い

  • 関数や演算子のオーバーロード解決において、検索パス上の候補を順次追加し、最終的に最適な候補を特定するために使用されます。
  • pathpos は名前空間検索の内部的な位置管理に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/namespace.c 内の関数群（例: 1191行目、1239行目、1384行目など）が本構造体をインスタンス化し、リスト操作を行っています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出し先はありません。

関連データ構造

  • Oid: オブジェクト識別子。
  • nodes/primnodes.h: 関連するノード定義が含まれます。

失敗モードと境界条件

  • メモリ不足時にリストの拡張が失敗する可能性があります。
  • FLEXIBLE_ARRAY_MEMBER の取り扱いにおいて、メモリ確保サイズが不適切である場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • namespace.c 内で頻繁にリスト操作が行われるため、リストの構築および解放処理（メモリ管理）が適切に行われているか注意してください。
  • args 配列のサイズは nargs に依存するため、構造体生成時のメモリ確保ロジックを必ず確認してください。
