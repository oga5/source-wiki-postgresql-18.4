ShmemInitStruct

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/storage/ipc/shmem.c:386

目的

共有メモリ上に名前付きデータ構造を確保または取得します。指定された名前の構造体が既に存在する場合はそのアドレスを返し、存在しない場合は新規にメモリを割り当てて初期化します。

シグネチャまたは形状

void *ShmemInitStruct(const char *name, Size size, bool *foundPtr);

入力と出力

  • **入力**:
  • name: 構造体を識別する一意な名前。
  • size: 確保するメモリサイズ（バイト単位）。
  • foundPtr: 構造体が既に存在したか否かを格納するブール値へのポインタ。
  • **出力**:
  • 確保または取得された共有メモリ領域へのポインタ。

前提条件と不変条件

  • ShmemIndexLock による排他制御下で実行されます。
  • 戻り値のアドレスは ShmemAddrIsValid を満たし、かつキャッシュライン境界（CACHELINEALIGN）にアライメントされていることが保証されます。
  • 既存の構造体を取得する場合、要求された size が既存の result->size と一致する必要があります。

アルゴリズムまたは振る舞い

1. ShmemIndexLock を取得します。 2. ShmemIndex が未初期化の場合（ブートストラップ時）:

  • 名前が "ShmemIndex" であることを確認し、Postmaster配下であれば既存のインデックスを返し、そうでなければ ShmemAlloc で領域を確保してインデックスを初期化します。

    • 3. ShmemIndex が初期化済みの場合:

  • hash_search を用いて指定された name を検索します。
  • 存在する場合: size の整合性を確認し、既存の location を返します。
  • 存在しない場合: ShmemAllocRaw でメモリを確保し、ShmemIndex に登録して初期化します。

    • 4. ShmemIndexLock を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_stat_statements.c, syncscan.c, nbtutils.c, commit_ts.c, multixact.c, slru.c, twophase.c, varsup.c, xlog.c, xlogprefetcher.c, xlogrecovery.c 等（詳細はソースコードの参照箇所に依存）。
  • **呼び出し先**: LWLockAcquire, LWLockRelease, hash_search, ShmemAlloc, ShmemAllocRaw, ereport。

関連データ構造

  • ShmemIndex: 共有メモリ上の全データ構造を管理するハッシュテーブル。
  • ShmemIndexEnt: ShmemIndex 内の各エントリを保持する構造体。
  • ShmemIndexLock: 本関数が使用する排他ロック。

失敗モードと境界条件

  • **メモリ不足**: ShmemAllocRaw が失敗した場合、ereport(ERROR, ...) を発行し、確保しようとしたインデックスエントリを削除します。
  • **サイズ不一致**: 既存の構造体と要求サイズが異なる場合、ereport(ERROR, ...) を発行します。
  • **インデックス作成失敗**: ShmemIndex へのエントリ作成に失敗した場合、エラーを報告します。

保守担当者向けメモ

  • 本関数は共有メモリの初期化フェーズにおいて中心的な役割を果たします。
  • ShmemIndex 自体の初期化と、それ以外の構造体の初期化で処理パスが分岐するため、ブートストラップ時の挙動変更には注意が必要です。
  • 共有メモリは一度割り当てると解放できないため、動的なメモリ管理が必要な場合は、確保した領域内で独自のフリーリスト等を実装してください。
