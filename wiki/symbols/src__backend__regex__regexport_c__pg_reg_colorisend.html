pg_reg_colorisend

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:207

目的

指定されたカラー ID が、正規表現の終了状態（EOS: End Of String）を表すカラーであるかどうかを判定する。

シグネチャまたは形状

int pg_reg_colorisend(const regex_t *regex, int co)

入力と出力

  • **入力**:
  • regex: コンパイル済みの正規表現オブジェクトへのポインタ。
  • co: 判定対象のカラー ID。
  • **出力**:
  • true (1): 指定されたカラーが終了状態を表す場合。
  • false (0): それ以外の場合。

前提条件と不変条件

  • regex は NULL であってはならず、regex->re_magic が REMAGIC と一致している必要がある（assert による検証）。

アルゴリズムまたは振る舞い

1. regex オブジェクトから内部構造体 guts を取得し、その中の search（cnfa 構造体）を参照する。 2. 引数 co が cnfa->eos[0] または cnfa->eos[1] のいずれかと一致するかを比較する。 3. 一致する場合は true を、一致しない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_regexp.c:1091
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない。

関連データ構造

  • regex_t: 正規表現の公開構造体。
  • guts: regex_t の内部実装詳細を保持する構造体。
  • cnfa: 正規表現の NFA 構造を保持する内部構造体。

失敗モードと境界条件

  • regex が無効な場合、assert によりプログラムが異常終了する可能性がある。
  • co が有効なカラー ID でない場合でも、単に false を返す。

保守担当者向けメモ

  • 本関数は cnfa 内の eos 配列を参照して終了状態を判定する。eos[0] と eos[1] の両方をチェックしているのは、正規表現エンジンが複数の終了条件を保持し得るためである。
