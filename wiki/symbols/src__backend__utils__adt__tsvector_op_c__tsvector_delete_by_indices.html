tsvector_delete_by_indices

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:463

目的

指定されたインデックスリストに基づいて、TSVector から特定の語彙（lexeme）を削除し、新しい TSVector を生成する。

シグネチャまたは形状

static TSVector tsvector_delete_by_indices(TSVector tsv, int *indices_to_delete, int indices_count)

入力と出力

  • **入力**:
  • tsv: 操作対象の TSVector。
  • indices_to_delete: 削除対象のインデックスを格納した配列。
  • indices_count: indices_to_delete 配列の要素数。
  • **出力**: 削除後の要素を含む新しい TSVector。

前提条件と不変条件

  • indices_to_delete に含まれるインデックスは [0, tsv->size) の範囲内であること。
  • indices_to_delete は関数内でソートおよび重複排除される。
  • 戻り値の TSVector は palloc0 で確保され、SET_VARSIZE で適切なサイズが設定される。

アルゴリズムまたは振る舞い

1. indices_to_delete を qsort でソートし、qunique で重複を排除する。 2. 削除後のサイズを考慮して TSVector 用のメモリを確保する。 3. 元の tsv を走査し、削除対象インデックスに含まれない語彙のみを新しい TSVector にコピーする。 4. 語彙のコピー時、関連する位置情報（WordEntryPos）と重みも保持される。 5. 最後に SET_VARSIZE を用いて、コピーされたデータ量に基づき正確なサイズを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:

  • **呼び出し先**:

関連データ構造

  • TSVector: 全文検索用ベクトル型。
  • WordEntry: TSVector 内の語彙エントリ。
  • WordEntryPos: 語彙の位置情報。

失敗モードと境界条件

  • indices_to_delete に範囲外のインデックスが含まれる場合、Assert(k == indices_count) によりアサーションエラーが発生する。
  • indices_count が 0 の場合、元の TSVector をコピーしたものが返される。

保守担当者向けメモ

  • 本関数は TSVector の内部構造（ARRPTR, STRPTR）に直接依存しているため、構造変更時には注意が必要。
  • curoff を用いたオフセット計算と SHORTALIGN によるアライメント処理が正しく行われていることを確認すること。
