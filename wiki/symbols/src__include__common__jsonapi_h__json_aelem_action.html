json_aelem_action

定義

  • 種別: typedef
  • 定義箇所: src/include/common/jsonapi.h:129

目的

JSON 配列の要素処理（開始または終了）時に呼び出されるコールバック関数の型定義です。

シグネチャまたは形状

typedef JsonParseErrorType (*json_aelem_action) (void *state, bool isnull);

入力と出力

  • **入力**:
  • void *state: ユーザー定義の解析状態データへのポインタ。
  • bool isnull: 要素が JSON の null 値であるかどうかを示すフラグ。
  • **出力**:
  • JsonParseErrorType: 解析の成功または失敗を示す列挙値。

前提条件と不変条件

  • 本関数は JsonSemAction 構造体を通じてパーサーに登録され、配列の要素処理イベントが発生した際に呼び出されます。
  • 戻り値が JSON_SEM_ACTION_FAILED の場合、エラーの詳細はコールバック関数内で報告済みであると見なされます。

アルゴリズムまたは振る舞い

1. パーサーが JSON 配列の要素を認識した際、登録された json_aelem_action 型の関数が実行されます。 2. ユーザーは state を介して解析結果を蓄積し、isnull に基づいて適切な処理（値のスキップや null の記録など）を行います。 3. 処理結果として JsonParseErrorType を返し、パーサーの制御フローに解析の継続可否を伝達します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/jsonapi.c 内の JSON 解析ロジック（array_element_start および array_element_end イベント発生時）。
  • **呼び出し先**: ユーザーが定義したコールバック関数（JsonSemAction に設定された関数ポインタ）。

関連データ構造

  • JsonSemAction: 本関数をメンバとして保持し、解析時のアクションテーブルを構成します。
  • JsonParseErrorType: 戻り値として使用されるエラー分類。

失敗モードと境界条件

  • コールバック内でエラーが発生した場合、JSON_SEM_ACTION_FAILED を返すことで解析を停止させる必要があります。
  • state ポインタが適切に初期化されていない場合、未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は JsonSemAction の array_element_start および array_element_end フィールドで使用されます。
  • 解析の増分処理（pg_parse_json_incremental）においても同様のインターフェースで呼び出されるため、状態の整合性に注意してください。
