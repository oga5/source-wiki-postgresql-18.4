make_and_qual

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/makefuncs.c:779

目的

2つのクエリ条件（Node）を論理積（AND）で結合します。一方または両方が NULL の場合に、冗長な BoolExpr ノードを作成せずに適切な結果を返すためのユーティリティ関数です。

シグネチャまたは形状

Node *make_and_qual(Node *qual1, Node *qual2)

入力と出力

  • **入力**:
  • qual1: 結合対象の条件式ノード（NULL 許容）。
  • qual2: 結合対象の条件式ノード（NULL 許容）。
  • **出力**:
  • 結合された条件式を表す Node。両方が NULL でない場合は BoolExpr ノードが返されます。

前提条件と不変条件

  • 入力ノードは NULL であるか、有効なクエリ条件式ノードである必要があります。
  • 戻り値として返されるノードは、呼び出し側のメモリコンテキスト（CurrentMemoryContext）に割り当てられます。

アルゴリズムまたは振る舞い

1. qual1 が NULL の場合、qual2 をそのまま返します。 2. qual2 が NULL の場合、qual1 をそのまま返します。 3. 両方が NULL でない場合、list_make2(qual1, qual2) を用いてリストを作成し、make_andclause を呼び出して論理積ノードを構築して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/prep/prepjointree.c
  • src/backend/rewrite/rewriteManip.c
  • **呼び出し先**:
  • make_andclause (同ファイル内)
  • list_make2 (リスト操作用ユーティリティ)

関連データ構造

  • Node: PostgreSQLのクエリツリーを構成する基本構造体。
  • BoolExpr: 論理演算（AND/OR/NOT）を表すノード構造体。

失敗モードと境界条件

  • 両方の引数が NULL の場合、結果として NULL が返されます。
  • メモリ不足時には make_andclause 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は、クエリ最適化や書き換えの過程で条件式を動的に追加する際に、不要な AND ノードの生成を抑制するために使用されます。
  • 戻り値の型は Node * ですが、実体は BoolExpr であることが多いため、後続の処理で型を確認する必要がある場合は注意してください。
