xl_hash_split_complete

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/hash_xlog.h:117

目的

ハッシュインデックスのバケット分割処理が完了したことを記録するWAL（Write-Ahead Logging）レコードのデータ構造です。リカバリ時に、分割対象となった旧バケットと新バケットのフラグ状態を復元するために使用されます。

シグネチャまたは形状

typedef struct xl_hash_split_complete
{
    uint16      old_bucket_flag;
    uint16      new_bucket_flag;
} xl_hash_split_complete;

入力と出力

  • **入力**: WALレコードのペイロードとして書き込まれる2つの uint16 型フラグ。
  • **出力**: リカバリ処理において、対象ページのバケットフラグを更新するためのメタデータとして利用されます。

前提条件と不変条件

  • 本構造体は、ハッシュインデックスの分割操作の最終段階でWALに書き込まれます。
  • old_bucket_flag および new_bucket_flag は、ハッシュインデックスのページヘッダーにおけるフラグ状態と整合している必要があります。

アルゴリズムまたは振る舞い

1. **書き込み**: _hash_split_complete 等の処理において、分割後のバケット状態を本構造体に格納し、XLogInsert を介してWALへ記録します。 2. **リカバリ**: hash_redo 関数が本レコードを読み取った際、XLogRecGetData を通じて本構造体を取得します。その後、対応するバケットページのフラグを、構造体内の値（old_bucket_flag, new_bucket_flag）で上書き更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hash_xlog.c: リカバリ時のレコード解析 (hash_redo)。
  • src/backend/access/hash/hashpage.c: 分割処理実行時のレコード生成。
  • src/backend/access/rmgrdesc/hashdesc.c: WALログのデバッグ出力生成。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • src/include/access/hash.h: ハッシュインデックスのページレイアウトおよびメタページ定義。

失敗モードと境界条件

  • WALレコードの読み込み時にデータが破損している場合、XLogRecGetData が不正なポインタを返す可能性があります。
  • 構造体のサイズは sizeof(uint16) * 2 であり、これに満たないレコード長の場合はリカバリ処理が失敗します。

保守担当者向けメモ

  • 本構造体は hash_xlog.h 内で定義されており、offsetof を使用したサイズ計算 (src/include/access/hash_xlog.h:124) が行われています。構造体を変更する場合は、このオフセット計算式との整合性に注意してください。
