heap_attisnull

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/heaptuple.c:455

目的

指定されたタプル内の特定の属性がNULLであるか、あるいは存在しない（欠落している）かを判定する。

シグネチャまたは形状

bool heap_attisnull(HeapTuple tup, int attnum, TupleDesc tupleDesc)

入力と出力

  • **入力**:
  • tup: 検査対象の HeapTuple。
  • attnum: 検査する属性番号（1始まり）。
  • tupleDesc: タプルの記述子（NULL許容）。
  • **出力**:
  • bool: 属性がNULLまたは欠落している場合に true、それ以外の場合に false を返す。

前提条件と不変条件

  • tupleDesc が提供される場合、attnum は tupleDesc->natts 以下でなければならない（Assertによる検証）。
  • システム属性（attnum <= 0）については、定義済みの特定の属性番号のみが許容される。

アルゴリズムまたは振る舞い

1. **属性の存在確認**: attnum がタプルヘッダーに記録された属性数を超える場合、tupleDesc を参照して欠落値（missing value）が定義されているかを確認する。定義されていれば false を、そうでなければ true を返す。 2. **NULL判定**: attnum > 0 の場合、タプル全体にNULLが含まれないフラグ（HeapTupleNoNulls）を確認し、含まれる場合は att_isnull マクロを用いてNULLビットマップを走査して判定する。 3. **システム属性の処理**: attnum <= 0 の場合、有効なシステム属性番号（TableOidAttributeNumber 等）であれば false を返し、それ以外は elog(ERROR) でエラーを発生させる。

関連データ構造

  • HeapTuple: 操作対象のタプル構造体。
  • TupleDesc: タプルのメタデータ（属性定義）。

失敗モードと境界条件

  • attnum がシステム属性として定義されていない負の値や0の場合、elog(ERROR) により処理が中断される。
  • tupleDesc がNULLの場合、欠落値の判定は行われず、タプルヘッダー内の情報のみに基づいて判定される。

保守担当者向けメモ

  • 本関数は、列追加操作等でタプル内に物理的に存在しない属性を扱う際、TupleDesc の atthasmissing フラグを適切に参照する。
  • システム属性は常にNULLではないとみなされるため、switch 文で明示的に許可されたもの以外はエラーとなる。
