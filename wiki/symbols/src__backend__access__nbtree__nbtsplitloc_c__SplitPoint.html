SplitPoint

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/nbtree/nbtsplitloc.c:28

目的

B-tree インデックスのページ分割時における、候補となる分割位置の情報を保持するために使用されます。分割後の左右ページの状態（空き容量やタプルの配置）を評価し、最適な分割点を選択するためのコンテキストとして機能します。

シグネチャまたは形状

typedef struct {
    int16       curdelta;        /* 左右ページの空き容量の差分 */
    int16       leftfree;        /* 分割後の左ページ空き容量 */
    int16       rightfree;       /* 分割後の右ページ空き容量 */
    OffsetNumber firstrightoff;  /* 右ページに移動する最初のタプル位置 */
    bool        newitemonleft;   /* 新規挿入アイテムが左ページに配置されるか否か */
} SplitPoint;

入力と出力

本構造体は _bt_findsplitloc によって生成・評価されます。

  • **入力**: ページ分割の計算過程で、各候補位置に対して curdelta、leftfree、rightfree、firstrightoff、newitemonleft が設定されます。
  • **出力**: 評価された SplitPoint の配列が FindSplitData に格納され、最終的に最適な分割点が選択されます。

前提条件と不変条件

  • leftfree および rightfree は、分割後のページにおける空き容量を正確に反映している必要があります。
  • firstrightoff は、元のページ内の有効なオフセット番号を指す必要があります。

アルゴリズムまたは振る舞い

1. **候補列挙**: _bt_recsplitloc を通じて、ページ内の全合法的な分割点が SplitPoint として生成されます。 2. **均衡化**: _bt_deltasortsplits により、curdelta（左右の空き容量の均衡度）に基づいて候補がソートされます。 3. **戦略適用**: _bt_strategy および _bt_bestsplitloc が、この構造体の情報を参照し、インデックスの特性（重複の多さやサフィックス切り詰め効率）を考慮して最適な分割点を選択します。

呼び出し元と呼び出し先

  • **呼び出し元**: _bt_findsplitloc が本構造体の配列を確保し、各候補を初期化します。また、_bt_strategy、_bt_split_penalty、_bt_bestsplitloc 等の内部関数が引数として受け取り、評価を行います。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • FindSplitData: 分割計算全体を管理するコンテキストであり、SplitPoint の配列を保持します。
  • FindSplitStrat: 分割戦略を定義する列挙型で、SplitPoint の評価基準を決定するために併用されます。

失敗モードと境界条件

  • **境界条件**: ページ内のタプル数が少ない場合や、すべてのタプルが右ページに移動するような極端な分割点も候補として計算されます。
  • **失敗モード**: 構造体自体に起因する失敗モードは明示されていませんが、メモリ不足時に palloc が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は、仮想的なページ状態をシミュレートするために使用されます。内部ページとリーフページで「ペナルティ」の計算ロジックが異なるため、本構造体のフィールドを参照する際は、現在のページタイプに応じた評価関数が使用されていることを確認してください。
