ListenAction

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/async.c:339

目的

LISTEN または UNLISTEN コマンドに関連するアクションを保持し、バックエンドが購読するチャネル名を管理するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	ListenActionKind action;
	char		channel[FLEXIBLE_ARRAY_MEMBER];
} ListenAction;

入力と出力

  • **入力**: ListenActionKind 型のアクション種別と、チャネル名を表す文字列。
  • **出力**: 構造体インスタンス。channel フィールドは可変長配列として定義されており、末尾にヌル終端文字列を保持します。

前提条件と不変条件

  • channel はヌル終端文字列である必要があります。
  • 構造体のメモリ割り当て時には、offsetof(ListenAction, channel) にチャネル名の長さを加算したサイズが必要です。

アルゴリズムまたは振る舞い

  • 本構造体は、トランザクション内で実行された LISTEN/UNLISTEN コマンドをリストとして蓄積するために使用されます。
  • palloc を用いて動的にメモリ確保され、チャネル名がコピーされます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/async.c 内のコマンド処理関数（LISTEN/UNLISTEN 実行時）。
  • **呼び出し先**: 蓄積された ListenAction のリストは、トランザクションのコミット処理やアクションの適用プロセスで参照されます。
  • ※具体的な関数呼び出しの連鎖については、提供されたソース情報からは詳細なフローを特定できません。

関連データ構造

  • ListenActionKind: アクションの種類（LISTEN か UNLISTEN か）を定義する列挙型。
  • List: ListenAction 構造体を要素として保持するリスト。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保が失敗する可能性があります。
  • チャネル名の長さ: FLEXIBLE_ARRAY_MEMBER を使用しているため、チャネル名の長さに応じた適切なメモリ確保が不可欠です。

保守担当者向けメモ

  • channel フィールドが可変長であるため、構造体のサイズ計算には sizeof(ListenAction) だけでなく offsetof を使用する慣習を維持してください。
  • 本構造体はトランザクションローカルなリストとして管理されるため、トランザクション終了時のメモリ解放漏れに注意してください。
