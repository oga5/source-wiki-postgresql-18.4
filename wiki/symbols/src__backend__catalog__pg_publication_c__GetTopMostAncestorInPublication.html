GetTopMostAncestorInPublication

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_publication.c:353

目的

指定されたリレーションの祖先リストの中から、特定のパブリケーションに含まれる「最も上位（topmost）」の祖先リレーションを特定する。

シグネチャまたは形状

Oid GetTopMostAncestorInPublication(Oid puboid, List *ancestors, int *ancestor_level)

入力と出力

  • **入力**:
  • puboid: 検索対象のパブリケーションOID。
  • ancestors: 祖先リレーションのOIDリスト（List *）。
  • ancestor_level: 見つかった祖先の階層レベルを格納するポインタ（NULL可）。
  • **出力**:
  • パブリケーションに含まれる最も上位の祖先リレーションのOID。該当がない場合は InvalidOid を返す。

前提条件と不変条件

  • ancestors リストは、子から親へと順に並んでいることを前提としている。
  • 祖先がパブリケーションに直接含まれているか、その祖先が属するスキーマがパブリケーションに含まれている場合に「含まれている」と判定される。

アルゴリズムまたは振る舞い

1. ancestors リストを先頭から順に走査する。 2. 各祖先リレーションについて以下を確認する：

  • GetRelationPublications を呼び出し、そのリレーションが puboid に含まれるか判定する。
  • 含まれない場合、get_rel_namespace で取得したスキーマが GetSchemaPublications を通じて puboid に含まれるか判定する。

    • 3. 条件を満たした場合、そのリレーションを topmost_relid として保持し、ancestor_level が指定されていれば現在の階層レベルを記録する。 4. リストの最後まで走査を継続し、最終的に見つかった最も上位の祖先を返す。 5. 走査中に使用したメモリ（apubids, aschemaPubids）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/publicationcmds.c
  • src/backend/replication/pgoutput/pgoutput.c
  • **呼び出し先**:
  • GetRelationPublications
  • GetSchemaPublications
  • get_rel_namespace
  • list_member_oid, lfirst_oid, list_free

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • List: PostgreSQLのリスト構造。

失敗モードと境界条件

  • 祖先リストが空の場合、InvalidOid を返す。
  • どの祖先も指定されたパブリケーションに含まれない場合、InvalidOid を返す。
  • ancestor_level が NULL の場合、レベルの記録はスキップされる。

保守担当者向けメモ

  • 本関数はパーティション階層の解決において重要である。
  • GetRelationPublications および GetSchemaPublications の呼び出しによるカタログ検索が発生するため、頻繁な呼び出しはパフォーマンスに影響を与える可能性がある。
