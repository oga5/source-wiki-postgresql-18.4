gseg_picksplit_item

定義

  • 種別: struct
  • 定義箇所: contrib/seg/seg.c:39

目的

GiST インデックス構築時の picksplit 処理において、区間データ（SEG）をその中心値に基づいてソートし、効率的にノードを分割するために使用される一時的なデータ構造です。

シグネチャまたは形状

typedef struct
{
	float		center;
	OffsetNumber index;
	SEG		   *data;
} gseg_picksplit_item;

入力と出力

本構造体はデータ保持用であり、関数ではないため入出力はありません。ただし、gseg_picksplit 関数内で palloc を用いて配列として確保され、qsort の対象となります。

前提条件と不変条件

  • center メンバには、対応する data（SEG 構造体）の中心値が格納されている必要があります。
  • index メンバは、インデックス内の位置を識別するために使用されます。

アルゴリズムまたは振る舞い

本構造体は、gseg_picksplit において以下の手順で利用されます。 1. インデックス対象の各 SEG データから中心値を計算し、本構造体の配列に格納する。 2. gseg_picksplit_item_cmp 関数を比較関数として qsort を実行し、中心値順にソートする。 3. ソートされた順序に基づいて、ツリーの分割位置を決定する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/seg/seg.c 内の gseg_picksplit 関数が、インデックス構築時に本構造体の配列を動的に割り当てて使用します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。比較処理において gseg_picksplit_item_cmp が本構造体を参照します。

関連データ構造

  • SEG: 実際の区間データを保持する構造体。
  • OffsetNumber: PostgreSQL のインデックスにおけるオフセット識別子。

失敗モードと境界条件

  • メモリ不足: palloc による配列確保時にメモリが不足した場合、PostgreSQL のエラーハンドリングにより処理が中断されます。
  • ソート対象の欠如: maxoff が 0 の場合、配列確保やソート処理は実行されません。

保守担当者向けメモ

  • 本構造体は gseg_picksplit の内部的なソート処理専用です。
  • center の計算ロジックやソート順序を変更する場合は、gseg_picksplit_item_cmp との整合性に注意してください。
