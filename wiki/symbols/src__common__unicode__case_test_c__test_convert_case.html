test_convert_case

定義

  • 種別: function
  • 定義箇所: src/common/unicode/case_test.c:330

目的

Unicode 大文字・小文字変換（Case Mapping）機能の正確性を検証するため、定義済みの文字列セットを用いて変換結果を期待値と比較するテストを実行する。

シグネチャまたは形状

static void test_convert_case(void)

入力と出力

  • 入力: なし（関数内部でハードコードされた文字列セットを使用）
  • 出力: なし（検証失敗時は exit(1) で異常終了、成功時は標準出力に成功メッセージを表示）

前提条件と不変条件

  • 変換関数（tfunc_lower, tfunc_upper, tfunc_title, tfunc_fold）が正しく実装されていること。
  • USE_ICU が定義されている環境では、ICU ライブラリが利用可能であること。

アルゴリズムまたは振る舞い

1. **基本変換テスト**: test_convert を呼び出し、大文字・小文字変換、タイトルケース変換、ケースフォールディングの各関数に対し、特定の入力文字列と期待される出力文字列を検証する。 2. **特殊ケースの検証**: 以下の特殊な変換ルールをテストする。

  • バイト長が変化する変換（例: Ⱥ → ⱥ）
  • 1対多の変換（例: ß → SS）
  • ギリシャ語の最終シグマ（ς）の文脈依存変換
  • アルファベットと数字の境界判定

    • 3. **ICU 比較テスト**: USE_ICU が定義されている場合、icu_test_full を呼び出し、PostgreSQL の変換結果と ICU の結果を網羅的に比較する。 4. **完了通知**: 全てのテストが成功した場合、case_test: convert_case: success を出力する。

呼び出し元と呼び出し先

  • 呼び出し元: ソースコード内の記述から、main 関数等のテスト実行フローから呼び出されることが示唆されるが、具体的な呼び出し元関数名は提供されていない。
  • 呼び出し先:
  • test_convert
  • icu_test_full（USE_ICU 定義時）
  • tfunc_lower, tfunc_upper, tfunc_title, tfunc_fold

関連データ構造

  • TestFunc（test_convert の引数として使用）

失敗モードと境界条件

  • **変換不一致**: 期待値と実際の変換結果が異なる場合、test_convert 内部で異常終了する。
  • **ICU バージョン差異**: PostgreSQL 内蔵の Unicode データと ICU のバージョンが異なる場合、一部のコードポイントで不一致が発生する可能性がある。

保守担当者向けメモ

  • 本関数は src/common/unicode/case_test.c 内の主要なテストスイートの一部である。
  • 新しい Unicode バージョンへの対応や変換ロジックの変更時には、本関数内のテストケースに新しい文字列を追加することが推奨される。
