ginInitBA

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbulk.c:108

目的

GINインデックス構築時に使用する BuildAccumulator 構造体を初期化し、キーとTIDの集約に必要な赤黒木（Red-Black Tree）を準備する。

シグネチャまたは形状

void ginInitBA(BuildAccumulator *accum)

入力と出力

  • **入力**: accum (BuildAccumulator *) - 初期化対象の構造体へのポインタ。
  • **出力**: なし（accum の各フィールドが初期化される）。

前提条件と不変条件

  • accum が指すメモリ領域は有効であること。
  • accum->ginstate は本関数内では意図的に設定されない。

アルゴリズムまたは振る舞い

1. accum->allocatedMemory を 0 にリセットする。 2. accum->entryallocator を NULL に、accum->eas_used を 0 に設定する。 3. rbt_create を呼び出し、以下の設定で赤黒木を生成して accum->tree に格納する。

  • ノードサイズ: sizeof(GinEntryAccumulator)
  • 比較関数: cmpEntryAccumulator
  • 結合関数: ginCombineData
  • 割り当て関数: ginAllocEntryAccumulator
  • 解放関数: NULL（不要）
  • コンテキスト: accum 自身を渡す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginfast.c (ginInitBA(&accum);)
  • src/backend/access/gin/gininsert.c (ginInitBA(&buildstate->accum); 等)
  • **呼び出し先**:
  • rbt_create (赤黒木の初期化)

関連データ構造

  • BuildAccumulator: 構築状態を保持するメイン構造体。
  • GinEntryAccumulator: 赤黒木の各ノードとして使用される構造体。

失敗モードと境界条件

  • rbt_create 内部でメモリ不足が発生した場合、PostgreSQLの標準的なエラー処理（palloc 等による ereport(ERROR)）が呼び出される可能性がある。

保守担当者向けメモ

  • 本関数は BuildAccumulator の初期化のみを行い、ginstate の設定は呼び出し元に委ねられている点に注意すること。
  • 赤黒木の構築には src/include/lib/rbt.h の実装が利用されている。
