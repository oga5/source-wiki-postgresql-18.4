get_expr_result_type

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/funcapi.c:298

目的

与えられた式（Node *expr）の戻り値の型（Oid）およびタプル記述子（TupleDesc）を解決し、その型クラス（TypeFuncClass）を返します。

シグネチャまたは形状

TypeFuncClass get_expr_result_type(Node *expr, Oid *resultTypeId, TupleDesc *resultTupleDesc);

入力と出力

  • **入力**:
  • expr: 評価対象の式ノード。
  • resultTypeId: 戻り値の型OIDを格納するポインタ（不要な場合はNULL可）。
  • resultTupleDesc: 戻り値のタプル記述子を格納するポインタ（不要な場合はNULL可）。
  • **出力**:
  • 戻り値: TypeFuncClass（TYPEFUNC_COMPOSITE, TYPEFUNC_RECORD 等）。

前提条件と不変条件

  • expr が FuncExpr または OpExpr の場合、内部的に internal_get_result_type を呼び出し、関数または演算子の定義に基づいた解決を行います。
  • RowExpr の場合、その引数リストと列名リストから動的に TupleDesc を構築します。
  • Const (RECORD型) の場合、HeapTupleHeader から型情報を抽出します。

アルゴリズムまたは振る舞い

1. **FuncExpr/OpExpr**: 関数または演算子の定義をカタログから参照し、型を解決します。 2. **RowExpr**: CreateTemplateTupleDesc を使用して新しいタプル記述子を作成し、各列の型・照合順序を TupleDescInitEntry 等で設定します。 3. **Const**: RECORDOID を持つ定数の場合、ヘッダーから型と型修飾子を読み取り、必要に応じて lookup_rowtype_tupdesc_copy で記述子を取得します。 4. **その他**: exprType を用いて汎用的な型解決を行い、複合型であれば lookup_rowtype_tupdesc_copy を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execSRF.c
  • src/backend/executor/nodeFunctionscan.c
  • src/backend/optimizer/prep/prepjointree.c
  • src/backend/optimizer/util/clauses.c
  • src/backend/parser/parse_relation.c
  • **呼び出し先**:
  • internal_get_result_type
  • get_opcode
  • CreateTemplateTupleDesc
  • BlessTupleDesc
  • lookup_rowtype_tupdesc_copy
  • exprType, exprTypmod, exprCollation

関連データ構造

  • TypeFuncClass: 戻り値の型クラスを示す列挙型。
  • TupleDesc: 複合型の構造定義。
  • FuncExpr, OpExpr, RowExpr, Const: PostgreSQLの式ノード構造体。

失敗モードと境界条件

  • RowExpr の引数リストと列名リストの長さが一致しない場合、Assert により停止します。
  • Const 型のレコードで型解決が不可能な場合、resultTupleDesc に NULL が設定され TYPEFUNC_RECORD が返されます。
  • 汎用的な式で複合型ではない場合、resultTupleDesc は NULL になります。

保守担当者向けメモ

  • 本関数は get_call_result_type と比較して、より広範な式ノード（RowExpr や Const を含む）を扱います。
  • 複合型の解決にはキャッシュ（lookup_rowtype_tupdesc_copy）が利用されるため、呼び出し頻度が高い場合は注意が必要です。
