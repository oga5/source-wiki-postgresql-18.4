int4_dist

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int4.c:93

目的

2つの int4 型の値の間の距離（絶対差分）を計算し、SQL から呼び出し可能な形式で返します。

シグネチャまたは形状

Datum int4_dist(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの int32 値を受け取ります。
  • **出力**: 2値の差の絶対値を int32 型の Datum として返します。

前提条件と不変条件

  • 入力値は int32 型である必要があります。
  • 差分計算時にオーバーフローが発生しないこと、および結果が PG_INT32_MIN にならないことが前提です。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32 を使用して2つの引数 a と b を取得します。 2. pg_sub_s32_overflow を用いて a - b を計算し、オーバーフローをチェックします。 3. 差分 r が PG_INT32_MIN であるか、またはオーバーフローが発生した場合は ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE エラーをスローします。 4. abs(r) により差分の絶対値を求め、PG_RETURN_INT32 を介して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから直接呼び出されることを想定しています。ソースコード内では PG_FUNCTION_INFO_V1(int4_dist) マクロによる登録が確認されています。
  • **呼び出し先**: PostgreSQL の内部関数 pg_sub_s32_overflow、ereport、abs を呼び出します。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数引数受け渡し用マクロ。

失敗モードと境界条件

  • **オーバーフロー**: 減算結果が int32 の範囲を超える場合、エラーとなります。
  • **最小値**: 差分が PG_INT32_MIN となる場合（絶対値が int32 の正の範囲に収まらないため）、エラーとなります。

保守担当者向けメモ

  • 本関数は GiST インデックスの内部処理用ではなく、ユーザーが SQL から距離計算を行うために提供されています。
  • 比較ロジックの変更が必要な場合は、btree_utils_num.c で定義されている共通ユーティリティとの整合性に注意してください。
