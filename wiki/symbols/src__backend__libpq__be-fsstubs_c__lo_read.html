lo_read

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-fsstubs.c:153

目的

開いているラージオブジェクト（Large Object）から指定されたバイト数を読み込みます。

シグネチャまたは形状

int lo_read(int fd, char *buf, int len)

入力と出力

  • **入力**:
  • fd: ラージオブジェクト記述子（cookies 配列のインデックス）。
  • buf: 読み込み先バッファ。
  • len: 読み込むバイト数。
  • **出力**:
  • 実際に読み込まれたバイト数を返します。

前提条件と不変条件

  • fd は有効な範囲内（0 以上 cookies_size 未満）であり、かつ cookies[fd] が NULL であってはなりません。
  • 対象のラージオブジェクトは、読み込み権限（IFS_RDLOCK フラグ）を保持してオープンされている必要があります。

アルゴリズムまたは振る舞い

1. fd が有効な記述子であるか検証し、無効な場合は ERRCODE_UNDEFINED_OBJECT でエラーを報告します。 2. cookies[fd] から LargeObjectDesc を取得します。 3. lobj->flags を確認し、IFS_RDLOCK が設定されていない場合は ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラーを報告します。 4. inv_read(lobj, buf, len) を呼び出し、物理的な読み込みを実行してその結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-fsstubs.c 内のSQLインターフェース関数（lo_read 内部のロジックを含む）。
  • src/bin/pg_dump/pg_dump.c（lo_read を呼び出す直接の証拠あり）。
  • src/interfaces/libpq/fe-lobj.c（クライアント側の lo_read と混同しないよう注意が必要）。
  • **呼び出し先**:
  • inv_read（src/backend/storage/large_object/inv_api.c で定義される低レベルAPI）。

関連データ構造

  • LargeObjectDesc: ラージオブジェクトの状態を保持する構造体。
  • cookies: 開いているラージオブジェクト記述子を管理するグローバルな動的配列。

失敗モードと境界条件

  • fd が範囲外または未初期化の場合: ERRCODE_UNDEFINED_OBJECT を発生させます。
  • 読み込みモードで開かれていない場合: ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE を発生させます。
  • inv_read 内部でのエラー: inv_read がエラーをスローした場合、そのエラーがそのまま伝播します。

保守担当者向けメモ

  • 本関数はサーバーバックエンド側の実装です。クライアントライブラリ（libpq）側の同名関数と混同しないよう注意してください。
  • inv_read への委譲前に状態チェックを行うことで、権限不足と記述子の無効状態を区別してエラー報告する設計になっています。
