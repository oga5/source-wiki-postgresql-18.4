desirable_join

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_eval.c:324

目的

GEQO（遺伝的クエリ最適化）において、指定された2つのリレーション（outer_rel と inner_rel）を結合することがヒューリスティック的に適切かどうかを判定する。

シグネチャまたは形状

static bool desirable_join(PlannerInfo *root, RelOptInfo *outer_rel, RelOptInfo *inner_rel)

入力と出力

  • **入力**:
  • root: 現在のクエリのプランナ情報。
  • outer_rel, inner_rel: 結合対象となるリレーション情報。
  • **出力**:
  • bool: 結合が推奨される場合は true、それ以外は false。

前提条件と不変条件

  • root、outer_rel、inner_rel は有効なポインタである必要がある。

アルゴリズムまたは振る舞い

以下のいずれかの条件を満たす場合に true を返し、結合を推奨する。 1. have_relevant_joinclause: 両リレーション間に適用可能な結合句が存在する。 2. have_join_order_restriction: 結合順序の制約により、これらのリレーションを結合する必要がある。 上記条件のいずれも満たさない場合は false を返し、結合を後回しにする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_eval.c 内の結合順序構築ロジックから参照されている（例: merge_clump 等の文脈）。
  • **呼び出し先**:
  • have_relevant_joinclause
  • have_join_order_restriction

関連データ構造

  • PlannerInfo: クエリ全体のプランナ状態を保持する構造体。
  • RelOptInfo: 結合対象となるリレーションの最適化情報を保持する構造体。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。入力が NULL の場合の挙動は、呼び出し側の責任において制御される。

保守担当者向けメモ

  • 本関数は GEQO によるプラン探索のヒューリスティックを制御する重要な判定ロジックである。結合句や制約の定義が変更された場合、本関数の判定基準も影響を受ける可能性がある。
