ResOwnerReleaseCryptoHash

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash_openssl.c:382

目的

PostgreSQLのバックエンド環境において、ResourceOwner によって管理されている暗号学的ハッシュコンテキストを安全に解放する。

シグネチャまたは形状

static void ResOwnerReleaseCryptoHash(Datum res)

入力と出力

  • **入力**: res (Datum) - 解放対象の pg_cryptohash_ctx 構造体へのポインタを保持する Datum 型の値。
  • **出力**: なし。

前提条件と不変条件

  • res は有効な pg_cryptohash_ctx 構造体を指している必要がある。
  • 本関数は ResourceOwner のクリーンアップ処理の一部として呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. Datum から pg_cryptohash_ctx 構造体へのポインタを取得する。 2. コンテキスト内の resowner フィールドを NULL に設定し、リソース所有者との関連付けを解除する。 3. pg_cryptohash_free(ctx) を呼び出し、OpenSSLの EVP_MD_CTX を破棄し、関連するメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash_openssl.c 内の ResourceOwner 関連の登録処理（ResourceOwner のコールバックテーブルに登録されている）。
  • **呼び出し先**: pg_cryptohash_free。

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態を保持する構造体。
  • ResourceOwner: PostgreSQLバックエンドにおけるリソース管理機構。

失敗モードと境界条件

  • res が NULL または不正なポインタである場合、未定義の動作を引き起こす可能性がある。
  • pg_cryptohash_free 内部でのエラー発生時の挙動については、当該関数の仕様に依存する。

保守担当者向けメモ

  • 本関数は ResourceOwner のコールバックとして登録されるため、直接呼び出すのではなく、ResourceOwner のメカニズムを通じて実行される。
  • コンテキスト解放時には機密情報保護のため explicit_bzero が内部的に使用される（pg_cryptohash_free 経由）。
