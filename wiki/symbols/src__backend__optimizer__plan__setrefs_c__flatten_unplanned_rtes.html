flatten_unplanned_rtes

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:484

目的

サブクエリ内のレンジテーブルエントリ（RTE）を走査し、それらをグローバルなレンジテーブルリストへ統合するための準備を行う。

シグネチャまたは形状

static void flatten_unplanned_rtes(PlannerGlobal *glob, RangeTblEntry *rte)

入力と出力

  • **入力**:
  • glob: プラン全体のグローバル情報（PlannerGlobal）。
  • rte: 処理対象のサブクエリを含むレンジテーブルエントリ。
  • **出力**: なし（副作用として glob 内のレンジテーブルリストを更新）。

前提条件と不変条件

  • rte は RTE_SUBQUERY 型のレンジテーブルエントリである必要がある。
  • rte->subquery が有効なクエリツリーを保持していること。

アルゴリズムまたは振る舞い

1. flatten_rtes_walker_context を初期化し、glob と対象のサブクエリを格納する。 2. query_tree_walker を呼び出し、サブクエリ内のすべての RTE を再帰的に走査する。 3. 走査中、flatten_rtes_walker をコールバックとして使用し、QTW_EXAMINE_RTES_BEFORE フラグにより各 RTE を順次処理する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/setrefs.c 内の add_rtes_to_flat_rtable から呼び出されることが確認されている。
  • **呼び出し先**: query_tree_walker を呼び出し、その中で flatten_rtes_walker を実行する。

関連データ構造

  • PlannerGlobal: プラン全体の情報を保持する構造体。
  • RangeTblEntry: クエリ内のリレーションやサブクエリを定義する構造体。
  • flatten_rtes_walker_context: 走査時に状態を保持するためのコンテキスト構造体。

失敗モードと境界条件

  • サブクエリが空の場合や、RTE を含まない場合は query_tree_walker が何も行わず正常終了する。
  • 呼び出し元で提供される glob が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は setrefs.c におけるレンジテーブルフラット化処理の一部である。
  • query_tree_walker を介した再帰的な走査を行うため、サブクエリが深くネストしている場合のスタック消費に注意すること。
