prepare_new_cluster

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.c:519

目的

アップグレード後の新クラスタに対して統計情報の収集（Analyze）および全行の凍結（Freeze）を行い、データベースの整合性とパフォーマンスを確保する。

シグネチャまたは形状

static void prepare_new_cluster(void);

入力と出力

  • 入力: なし（グローバル変数 new_cluster および log_opts を参照）
  • 出力: なし（副作用として外部コマンド vacuumdb を実行）

前提条件と不変条件

  • new_cluster が初期化され、接続可能な状態であること。
  • スキーマの復元（ロード）が完了していること（凍結処理が復元された frozenxids を上書きしないようにするため）。

アルゴリズムまたは振る舞い

1. **統計情報の更新**: vacuumdb --all --analyze を実行し、新クラスタ内の全行の統計情報を更新する。これにより、後の自動バキュームによる統計更新を抑制する。 2. **行の凍結**: vacuumdb --all --freeze を実行し、全行を凍結する。これにより pg_statistic を含む全テーブルが凍結状態となる。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/pg_upgrade.c 内の main 関数（152行目）。
  • 呼び出し先: exec_prog（外部コマンド vacuumdb の実行）、prep_status、check_ok。

関連データ構造

  • new_cluster: アップグレード先のクラスタ情報を保持するグローバル構造体。
  • log_opts: ログ出力設定を保持するグローバル構造体。

失敗モードと境界条件

  • vacuumdb コマンドの実行に失敗した場合、exec_prog を通じてエラーが報告され、処理が中断される可能性がある。
  • template0 データベースは本関数内では凍結されない（別途 datfrozenxid 等の調整が行われる）。

保守担当者向けメモ

  • 本関数はスキーマロード後に実行される必要がある。
  • vacuumdb の実行オプションは new_cluster.bindir に依存するため、新クラスタのバイナリパスが正しく設定されていることを前提としている。
