ECPGmake_struct_member

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/type.c:52

目的

構造体メンバーのリストに新しいメンバーを追加し、ECPGプリプロセッサが扱う型定義構造体の一部として管理する。

シグネチャまたは形状

void ECPGmake_struct_member(const char *name, struct ECPGtype *type, struct ECPGstruct_member **start)

入力と出力

  • **入力**:
  • name: メンバー名（文字列）。
  • type: メンバーの型情報を保持する ECPGtype 構造体へのポインタ。
  • start: メンバーリストの先頭を指すポインタへのポインタ。
  • **出力**: なし（start が指すリストを直接更新する）。

前提条件と不変条件

  • mm_alloc を使用してメモリを確保するため、呼び出し元はメモリ不足時にこの関数が終了することに依存する。
  • start はリストの先頭、または空のリストを指している必要がある。

アルゴリズムまたは振る舞い

1. mm_alloc を用いて新しい ECPGstruct_member 構造体を割り当てる。 2. name を mm_strdup で複製し、type を設定して新しいノードを初期化する。 3. start が指すリストの末尾まで走査する。 4. リストが空であれば *start に新しいノードを代入し、空でなければ末尾の next ポインタに新しいノードを連結する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/type.c 内のコード（ECPGmake_struct_type 等から呼び出されることが示唆されている）。
  • **呼び出し先**: mm_alloc, mm_strdup（メモリ管理用ラッパー）。

関連データ構造

  • ECPGstruct_member: 構造体のメンバーを保持する連結リスト構造。
  • ECPGtype: 変数の型情報を保持する構造体。

失敗モードと境界条件

  • start が指すリストが空の場合、新しいノードがリストの先頭となる。
  • リストが既に存在する場合、末尾に追加される。

保守担当者向けメモ

  • 本関数は ECPGstruct_member リストの末尾を線形探索するため、非常に多くのメンバーを持つ構造体に対しては計算コストが増大する可能性がある。
  • メモリ管理は mm_alloc に依存しているため、この関数で作成されたメンバーは対応する ECPGfree_struct_member 等で適切に解放される必要がある。
