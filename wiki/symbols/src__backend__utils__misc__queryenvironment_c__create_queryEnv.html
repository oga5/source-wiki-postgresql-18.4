create_queryEnv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/queryenvironment.c:38

目的

クエリ実行時やプランニング時に使用される、一時的な名前付きリレーション（ENR）を管理するための QueryEnvironment 構造体を生成・初期化します。

シグネチャまたは形状

QueryEnvironment *create_queryEnv(void);

入力と出力

  • **入力**: なし
  • **出力**: 初期化された QueryEnvironment 構造体へのポインタ。

前提条件と不変条件

  • palloc0 を使用してメモリを確保するため、PostgreSQL のメモリコンテキストが有効である必要があります。
  • 返される構造体はゼロ初期化されており、内部の ENR リストは空の状態です。

アルゴリズムまたは振る舞い

1. palloc0 を呼び出し、QueryEnvironment 構造体のサイズ分のメモリをゼロクリアして確保します。 2. 確保したメモリ領域へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/spi.c:3315 (_SPI_current->queryEnv の初期化に使用)
  • **呼び出し先**:
  • palloc0 (PostgreSQL メモリ管理関数)

関連データ構造

  • QueryEnvironment: ENR を保持するためのコンテナ構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させ、処理が中断されます。

保守担当者向けメモ

  • 現在の実装は単純なメモリ確保のみですが、将来的に QueryEnvironment の内部構造（リストからハッシュテーブルへの変更など）が変更された場合でも、この関数が初期化の責務を負うことで呼び出し側への影響を最小限に抑える設計となっています。
