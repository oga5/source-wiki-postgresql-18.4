CustomExecMethods

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/extensible.h:124

目的

PostgreSQLのエグゼキュータがカスタムスキャンを実行する際に呼び出すコールバック群を定義し、拡張機能が独自の実行ロジックをコアシステムに組み込むためのインターフェースを提供します。

シグネチャまたは形状

typedef struct CustomExecMethods {
    const char *CustomName;
    void (*BeginCustomScan) (CustomScanState *node, EState *estate, int eflags);
    TupleTableSlot *(*ExecCustomScan) (CustomScanState *node);
    void (*EndCustomScan) (CustomScanState *node);
    void (*ReScanCustomScan) (CustomScanState *node);
    void (*MarkPosCustomScan) (CustomScanState *node);
    void (*RestrPosCustomScan) (CustomScanState *node);
    Size (*EstimateDSMCustomScan) (CustomScanState *node, ParallelContext *pcxt);
    void (*InitializeDSMCustomScan) (CustomScanState *node, ParallelContext *pcxt, void *coordinate);
    void (*ReInitializeDSMCustomScan) (CustomScanState *node, ParallelContext *pcxt, void *coordinate);
    void (*InitializeWorkerCustomScan) (CustomScanState *node, shm_toc *toc, void *coordinate);
    void (*ShutdownCustomScan) (CustomScanState *node);
    void (*ExplainCustomScan) (CustomScanState *node, List *ancestors, ExplainState *es);
} CustomExecMethods;

入力と出力

  • **入力**: 各メソッドは CustomScanState *node を第一引数として受け取り、実行状態にアクセスします。
  • **出力**: ExecCustomScan は処理結果として TupleTableSlot * を返し、EstimateDSMCustomScan は必要な共有メモリサイズ（Size）を返します。その他は主に副作用（状態更新や出力）を目的とします。

前提条件と不変条件

  • CustomName は一意である必要があります。
  • BeginCustomScan, ExecCustomScan, EndCustomScan, ReScanCustomScan は必須の実装です。
  • マーク/リストア機能や並列実行機能を使用する場合、対応するオプションメソッドの実装が必要です。

アルゴリズムまたは振る舞い

コアシステムは CustomScanState 内の methods ポインタを通じて、実行フェーズに応じたコールバックを間接的に呼び出します。 1. **初期化**: BeginCustomScan で実行状態を準備。 2. **実行**: ExecCustomScan でタプルを順次取得。 3. **終了**: EndCustomScan でリソースを解放。 4. **拡張機能**: 必要に応じて EXPLAIN 出力や並列実行のための共有メモリ管理（DSM）ロジックを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeCustom.c 内の各実行関数が、CustomScanState に保持された methods を参照して呼び出します。
  • **呼び出し先**: 拡張機能側で実装された具体的な関数ポインタが呼び出されます。

関連データ構造

  • CustomScanState: 実行時の状態を保持する構造体。
  • EState, TupleTableSlot, ParallelContext, ExplainState: PostgreSQLコアの実行および出力用データ構造。

失敗モードと境界条件

  • 必須メソッドが NULL の場合、実行時にクラッシュする可能性があります。
  • ExplainCustomScan が実装されていない場合、EXPLAIN 出力時にカスタムスキャン固有の情報が表示されません。

保守担当者向けメモ

  • 本構造体は src/include/nodes/execnodes.h の CustomScanState 構造体から参照されます。
  • 並列実行やマーク/リストア機能のサポート状況に応じて、メソッドの呼び出し可否を適切に判断してください。
