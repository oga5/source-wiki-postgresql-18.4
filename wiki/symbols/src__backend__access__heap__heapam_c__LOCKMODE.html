LOCKMODE

定義

  • 種別: struct
  • 定義箇所: src/backend/access/heap/heapam.c:125

目的

タプルロックの強度（LockTupleMode）と、それに対応する heavyweight lock および MultiXactStatus をマッピングするための静的データ構造の一部として定義されています。

シグネチャまたは形状

static const struct
{
	LOCKMODE	hwlock;
	int			lockstatus;
	int			updstatus;
}

入力と出力

本シンボルは構造体の定義であり、関数ではないため入力と出力はありません。この構造体は、タプルロック操作時にロック強度を解決するための参照テーブルとして利用されます。

前提条件と不変条件

  • heapam.c 内で静的に定義されており、実行時に変更されることは想定されていません。
  • 関連する tupleLockExtraInfo テーブルの要素として使用されることを前提としています。

アルゴリズムまたは振る舞い

タプルに対するロック取得時、要求されたロックモードに基づいて、対応する heavyweight lock の種類（hwlock）や、複数トランザクション状態（lockstatus, updstatus）をルックアップするために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: heapam.c 内のタプルロック関連関数（heap_acquire_tuplock 等）から参照されます。
  • **呼び出し先**: 本シンボル自体はデータ構造の定義であり、関数を呼び出すことはありません。

関連データ構造

  • tupleLockExtraInfo: 本構造体が格納される静的テーブル。
  • LockTupleMode: ロック強度を定義する列挙型。
  • MultiXactStatus: 複数トランザクションのロック状態を管理する型。

失敗モードと境界条件

本シンボル自体に失敗モードはありません。ただし、この構造体を参照するロジックにおいて、無効なロックモードが指定された場合に不整合が生じる可能性があります。

保守担当者向けメモ

  • heapam.c 内の tupleLockExtraInfo 配列の定義とセットで管理してください。
  • ロックモードの追加や変更を行う際は、このマッピングテーブルの整合性を確認する必要があります。
