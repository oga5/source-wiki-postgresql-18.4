CreateDirAndVersionFile

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dbcommands.c:456

目的

データベースの物理ディレクトリを作成し、そのディレクトリ内に PostgreSQL のメジャーバージョンを記録した PG_VERSION ファイルを生成する。

シグネチャまたは形状

static void CreateDirAndVersionFile(char *dbpath, Oid dbid, Oid tsid, bool isRedo)

入力と出力

  • dbpath: 作成対象のデータベースディレクトリパス。
  • dbid: データベースの OID。
  • tsid: テーブルスペースの OID。
  • isRedo: WAL リプレイ中か否かを示すフラグ。
  • 出力: なし（成功時はディレクトリとファイルが作成される）。

前提条件と不変条件

  • PG_MAJORVERSION がシステム全体で一意であること。
  • isRedo が false の場合、操作はクリティカルセクション内で WAL ログに記録される。

アルゴリズムまたは振る舞い

1. MakePGDirectory を呼び出し、dbpath にディレクトリを作成する。 2. PG_VERSION ファイルを dbpath 内に作成し、PG_MAJORVERSION を書き込む。 3. 書き込み後、pg_fsync および fsync_fname を実行してディスクへの永続化を保証する。 4. isRedo が false の場合、XLogInsert を使用して XLOG_DBASE_CREATE_WAL_LOG レコードを WAL に書き込む。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/dbcommands.c 内のデータベース作成処理および WAL リプレイ処理から呼び出される。
  • 呼び出し先: MakePGDirectory, OpenTransientFile, write, pg_fsync, fsync_fname, CloseTransientFile, XLogBeginInsert, XLogRegisterData, XLogInsert 等。

関連データ構造

  • xl_dbase_create_wal_log_rec: WAL 記録に使用される構造体。

失敗モードと境界条件

  • ディレクトリ作成失敗時: isRedo が false か、errno が EEXIST でない場合にエラーを報告する。
  • ファイル作成失敗時: OpenTransientFile が失敗した場合にエラーを報告する。
  • 書き込み失敗時: ディスクフル等の理由で書き込みバイト数が一致しない場合、エラーを報告する。

保守担当者向けメモ

  • 本関数は非トランザクション的なファイルシステム操作を含むため、エラー発生時のクリーンアップには注意が必要である。
  • isRedo モード時は、既存の PG_VERSION ファイルを上書きする挙動が含まれている。
