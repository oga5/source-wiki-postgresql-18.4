nameeq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:147

目的

2つの name 型の値が等しいかどうかを判定し、ブール値を返します。

シグネチャまたは形状

Datum nameeq(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの Name 型引数を受け取ります。
  • **出力**: 比較結果が等しければ true、そうでなければ false を Datum 型として返します。

前提条件と不変条件

  • 入力される Name 型は、NAMEDATALEN バイトの固定長文字列として扱われます。
  • 比較には現在の照合順序（Collation）が使用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_NAME マクロを使用して、第1引数および第2引数を取得します。 2. namecmp 関数を呼び出し、2つの引数と現在の照合順序を渡して比較を実行します。 3. namecmp の戻り値が 0（等しい）であるかどうかを判定します。 4. 判定結果を PG_RETURN_BOOL マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し先**: namecmp
  • **呼び出し元**: ソースコード内の直接的な呼び出し元情報は提供されていません。

関連データ構造

  • Name: システムカタログ等で使用される固定長文字列型。
  • Datum: PostgreSQL の汎用的なデータ表現型。

失敗モードと境界条件

  • namecmp の振る舞いに依存します。namecmp は NAMEDATALEN を超える部分は無視して比較を行います。

保守担当者向けメモ

  • 本関数は name 型の等価比較演算子として機能します。
  • 照合順序を考慮した比較を行うため、ロケール設定によって比較結果が影響を受ける可能性があります。
