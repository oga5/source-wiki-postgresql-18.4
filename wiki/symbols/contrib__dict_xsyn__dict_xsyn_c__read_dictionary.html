read_dictionary

定義

  • 種別: function
  • 定義箇所: contrib/dict_xsyn/dict_xsyn.c:77

目的

外部のシノニム定義ファイルを読み込み、DictSyn 構造体に格納されたシノニム配列を構築・ソートする。

シグネチャまたは形状

static void read_dictionary(DictSyn *d, const char *filename)

入力と出力

  • **入力**:
  • d: 辞書データを保持する DictSyn 構造体へのポインタ。
  • filename: 読み込むシノニム定義ファイルのパス。
  • **出力**: なし（d が指す構造体の syn 配列および len フィールドを更新）。

前提条件と不変条件

  • filename で指定されたファイルは、PostgreSQLの全文検索用設定ファイル形式（スペース区切り、# はコメント）に従う必要がある。
  • 辞書ファイル内の各行は、str_tolower により小文字化されるため、小文字で定義されていることが前提となる。
  • 処理終了後、d->syn は compare_syn 関数に基づきソートされた状態となる。

アルゴリズムまたは振る舞い

1. get_tsearch_config_filename を用いて設定ファイルのフルパスを取得し、tsearch_readline_begin で読み込みを開始する。 2. ファイルの各行を読み込み、str_tolower で小文字化する。 3. find_word を使用して行内の単語を順次抽出する。 4. DictSyn 構造体の syn 配列が満杯であれば repalloc で拡張する。 5. 設定フラグ（matchorig, matchsynonyms）に基づき、抽出した単語をキーおよび値として syn 配列に格納する。 6. 全行の読み込み終了後、qsort を用いて d->syn をソートする。

呼び出し元と呼び出し先

  • **呼び出し元**: dxsyn_init（contrib/dict_xsyn/dict_xsyn.c:196）
  • **呼び出し先**:
  • get_tsearch_config_filename
  • tsearch_readline_begin, tsearch_readline, tsearch_readline_end
  • str_tolower
  • find_word
  • qsort
  • palloc, repalloc, pfree, pnstrdup, pstrdup

関連データ構造

  • DictSyn: 辞書全体の設定とシノニム配列を保持。
  • Syn: 個別の単語とシノニムのペアを保持。

失敗モードと境界条件

  • ファイルが開けない場合、ereport(ERROR, ...) により処理を中断する。
  • 空行はスキップされる。
  • d->len が 0 の場合、初期サイズとして 16 が割り当てられる。

保守担当者向けメモ

  • 本関数は str_tolower を使用しているため、ロケール依存の動作に注意すること。
  • d->syn のメモリ管理は PostgreSQL のメモリコンテキストに依存している。
