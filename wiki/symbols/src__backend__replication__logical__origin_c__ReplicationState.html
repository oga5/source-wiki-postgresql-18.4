ReplicationState

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/origin.c:109

目的

論理レプリケーションにおける各リモートノードの進捗状況を共有メモリ上で管理し、クラッシュリカバリ後の再開位置を特定するために使用されます。

シグネチャまたは形状

typedef struct ReplicationState
{
    RepOriginId roident;
    XLogRecPtr  remote_lsn;
    XLogRecPtr  local_lsn;
    int         acquired_by;
    ConditionVariable origin_cv;
    LWLock      lock;
} ReplicationState;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。各フィールドは共有メモリ上で更新され、チェックポイント時にディスクへ永続化されます。

前提条件と不変条件

  • remote_lsn および local_lsn の読み書きには、必ず lock フィールド（LWLock）を取得する必要があります。
  • acquired_by が変更された際は、origin_cv（ConditionVariable）を介して通知が行われます。

アルゴリズムまたは振る舞い

  • **進捗追跡**: リモートノードからのコミット位置（remote_lsn）と、それに対応するローカルのコミットレコード位置（local_lsn）を保持します。
  • **永続化支援**: チェックポイント実行時、local_lsn を参照して XLogFlush() を行うことで、コミットレコードがディスク上に安全に書き込まれたことを保証します。
  • **排他制御**: acquired_by フィールドにより、どのバックエンドがこのスロットを占有しているかを管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/origin.c 内の各関数（replorigin_create, replorigin_advance, CheckPointReplicationOrigin 等）がこの構造体のインスタンスを操作します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ReplicationStateOnDisk: チェックポイント時にディスクへ書き出される際の形式。
  • ReplicationStateCtl: 共有メモリ上の状態全体を制御する構造体。
  • RepOriginId: 起点を識別するための型。

失敗モードと境界条件

  • **最大ノード数**: 内部ID（RepOriginId）が2バイトに制限されているため、最大ノード数は65kに制限されます。
  • **メモリ制約**: 共有メモリ上の確保サイズは max_active_replication_origins GUCパラメータに依存します。
  • **アトミック性**: LSNの読み書きはアトミックではない可能性があるため、必ず lock を介した排他制御が必須です。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、フィールドの追加や変更を行う際は、ReplicationOriginShmemSize および関連する初期化処理との整合性に注意してください。
  • lock は remote_lsn と local_lsn の保護を目的としています。これ以外のフィールドへのアクセスが必要な場合は、適切なロック戦略を再検討してください。
