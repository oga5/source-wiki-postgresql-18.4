BaseBackupTargetHandle

定義

  • 種別: struct（不透明な構造体）
  • 定義箇所: src/include/backup/basebackup_target.h:18

目的

選択されたバックアップターゲットの情報を保持し、バックアップデータの書き込み先を抽象化するためのハンドルとして機能します。

シグネチャまたは形状

typedef struct BaseBackupTargetHandle BaseBackupTargetHandle;

入力と出力

  • **入力**: BaseBackupGetTargetHandle() 関数により、ターゲット名と詳細情報から生成されます。
  • **出力**: BaseBackupGetSink() 関数に渡すことで、バックアップデータを処理する bbsink を生成するために使用されます。

前提条件と不変条件

  • 本構造体は不透明（opaque）であり、外部からは直接メンバにアクセスせず、提供される API を通じて操作する必要があります。
  • BaseBackupGetTargetHandle() によって検証済みのハンドルとして取得されることが前提となります。

アルゴリズムまたは振る舞い

バックアップのパイプラインにおいて、ターゲット固有の処理（check_detail や get_sink コールバック）をカプセル化します。バックアップ開始前にターゲットの妥当性を検証し、開始後は bbsink チェーンの構築を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c: バックアップ処理のメインフローで使用。
  • src/backend/backup/basebackup_target.c: ハンドルの生成および bbsink の取得処理内で使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。関連する操作は BaseBackupGetSink() 等の API を介して行われます。

関連データ構造

  • bbsink: バックアップデータのパイプライン処理を行う構造体。本ハンドルを使用して生成されます。

失敗モードと境界条件

  • BaseBackupGetTargetHandle() が失敗した場合、有効なハンドルは返されません。
  • ターゲットの検証（check_detail）に失敗した場合、バックアップ処理は開始されません。

保守担当者向けメモ

  • 本構造体は拡張機能が新しいバックアップターゲットを定義する際の基盤となります。
  • 構造体の内部定義を変更する場合は、src/backend/backup/basebackup_target.c でのメモリ割り当て（palloc）および各コールバック関数との整合性に注意してください。
