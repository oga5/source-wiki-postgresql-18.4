SyncStandbySlotsConfigData

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/slot.c:92

目的

同期スタンバイとして設定されたレプリケーションスロットの名称リストを保持するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    int nslotnames;
    char slot_names[FLEXIBLE_ARRAY_MEMBER];
} SyncStandbySlotsConfigData;

入力と出力

  • **入力**: nslotnames（スロット名の数）、および slot_names（連続したNULL終端文字列の配列）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • slot_names は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、nslotnames 個のNULL終端文字列が連続して格納される必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、GUC（設定パラメータ）として指定された同期スタンバイのリストをメモリ上に保持するために使用されます。
  • offsetof(SyncStandbySlotsConfigData, slot_names) を用いて必要なメモリサイズを計算し、動的に割り当てられます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/slot.c 内のGUC設定処理（synchronized_standby_slots 関連）にて、設定値の変換およびメモリ割り当てが行われます。
  • **呼び出し先**: 特定の関数呼び出しの詳細はソース抜粋からは不明です。

関連データ構造

  • ReplicationSlot: レプリケーションスロットの共有メモリ上の状態を管理する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（guc_malloc 経由）にエラーが発生する可能性があります。
  • nslotnames と実際の slot_names 内の文字列数が一致しない場合、データ破損や読み取りエラーの原因となります。

保守担当者向けメモ

  • slot_names は柔軟な配列メンバであるため、構造体自体のサイズ計算には必ず offsetof を使用してください。
  • 本構造体は synchronized_standby_slots_config グローバル変数を通じて参照されます。
