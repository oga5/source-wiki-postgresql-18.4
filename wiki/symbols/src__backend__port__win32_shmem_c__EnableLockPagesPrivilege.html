EnableLockPagesPrivilege

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_shmem.c:136

目的

Windows 環境において、現在のプロセスに対して「メモリ内のページをロック (SeLockMemoryPrivilege)」権限を有効化します。これは Huge Pages を使用する際に必須となる特権の取得を目的としています。

シグネチャまたは形状

static bool EnableLockPagesPrivilege(int elevel);

入力と出力

  • **入力**: elevel (int) - 権限取得失敗時に報告するエラーレベル（FATAL や DEBUG1 など）。
  • **出力**: bool - 成功した場合は TRUE、失敗した場合は FALSE を返します。

前提条件と不変条件

  • 実行中の Windows ユーザーアカウントに、OS レベルで「メモリ内のページをロック」権限が割り当てられている必要があります。
  • 呼び出し元は、エラー発生時のログ出力レベルを制御する elevel を適切に指定する必要があります。

アルゴリズムまたは振る舞い

1. OpenProcessToken を呼び出し、現在のプロセスのトークンハンドルを取得します。 2. LookupPrivilegeValue を使用して、SE_LOCK_MEMORY_NAME に対応する LUID を取得します。 3. AdjustTokenPrivileges を呼び出し、取得した LUID に対して SE_PRIVILEGE_ENABLED 属性を設定します。 4. GetLastError を確認し、ERROR_NOT_ALL_ASSIGNED が返された場合は、ユーザーに権限が不足している旨を ereport で通知します。 5. 処理の終了時に、取得したトークンハンドルを CloseHandle で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/win32_shmem.c 内の共有メモリ初期化処理（huge_pages 設定が有効な場合）。
  • **呼び出し先**: Windows API (OpenProcessToken, LookupPrivilegeValue, AdjustTokenPrivileges, CloseHandle, GetLastError) および PostgreSQL のエラー報告関数 (ereport)。

関連データ構造

  • HANDLE: Windows プロセストークンハンドル。
  • TOKEN_PRIVILEGES: 特権情報を保持する Windows 構造体。
  • LUID: ローカル一意識別子。

失敗モードと境界条件

  • **権限不足**: OS 側でユーザーに権限が付与されていない場合、ERROR_NOT_ALL_ASSIGNED が発生し、ereport を通じてユーザーへのヒントが表示されます。
  • **システムコール失敗**: OpenProcessToken や AdjustTokenPrivileges 自体が失敗した場合、エラーコードと共に詳細が報告されます。
  • **リソースリーク**: 失敗時に CloseHandle を呼び出し、トークンハンドルを適切にクローズする設計となっています。

保守担当者向けメモ

  • 本関数は Windows 特有の権限管理を扱うため、Huge Pages の利用可否に直結します。
  • _("Lock pages in memory") という文字列は Windows のローカライズと一致させる必要があるため、変更時は注意してください。
