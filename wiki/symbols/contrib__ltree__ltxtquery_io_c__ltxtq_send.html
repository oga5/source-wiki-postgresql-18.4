ltxtq_send

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_io.c:643

目的

ltxtquery 型のデータを、ネットワーク転送に適したバイナリ形式（bytea）へシリアライズします。

シグネチャまたは形状

Datum ltxtq_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して ltxtquery 型のポインタを受け取ります。
  • **出力**: bytea 型のシリアライズされたデータ（Datum）を返します。

前提条件と不変条件

  • 入力された ltxtquery の size が 0 であってはなりません（0 の場合は構文エラーとして処理されます）。
  • 内部的にバージョン番号 1 を使用してシリアライズを行います。

アルゴリズムまたは振る舞い

1. 入力された ltxtquery のサイズを検証し、空であればエラーを報告します。 2. INFIX 構造体を初期化し、infix 関数を呼び出してクエリを中置記法の文字列形式に変換します。 3. pq_begintypsend を使用してバイナリ転送用のバッファを開始します。 4. バッファにバージョン番号（1）を書き込みます。 5. 変換済みの文字列をバッファに書き込みます。 6. pfree で一時バッファを解放し、pq_endtypsend で生成された bytea を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の SQL インターフェースから呼び出されます（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: infix（文字列変換用）、pq_begintypsend / pq_sendint8 / pq_sendtext / pq_endtypsend（libpq プロトコル処理用）。

関連データ構造

  • ltxtquery: 入力されるクエリ構造体。
  • INFIX: 文字列変換時に使用される作業用構造体。
  • StringInfoData: ネットワーク転送用バッファ。

失敗モードと境界条件

  • query->size == 0: 空のクエリが渡された場合、ERRCODE_SYNTAX_ERROR でエラー終了します。

保守担当者向けメモ

  • 本関数は ltxtquery のバイナリ表現をバージョン 1 として固定しています。将来的なフォーマット変更時には、このバージョン番号の管理に注意してください。
  • infix 関数による文字列変換は再帰的に行われるため、スタック消費量に注意が必要です。
