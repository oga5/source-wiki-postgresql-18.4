statext_mcv_serialize

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mcv.c:620

目的

多変量MCV（Most Common Values）リストを、pg_statistic_ext_data カタログへの格納に適したバイナリ形式（bytea）へ変換（シリアライズ）する。

シグネチャまたは形状

bytea *statext_mcv_serialize(MCVList *mcvlist, VacAttrStats **stats)

入力と出力

  • **入力**:
  • mcvlist: シリアライズ対象の MCVList 構造体。
  • stats: 各属性の統計情報を含む VacAttrStats の配列。
  • **出力**:
  • シリアライズされたデータを含む bytea 型のポインタ。

前提条件と不変条件

  • mcvlist 内の各次元のデータは、対応する属性の型情報に基づいてソートおよび重複排除される。
  • 重複排除後の各次元の個別値（distinct values）の数は PG_UINT16_MAX を超えてはならない。
  • varlena 型や cstring 型のデータは、シリアライズ前にデトースト（detoast）される。

アルゴリズムまたは振る舞い

1. 各次元（属性）ごとに、MCVList 内の値を収集し、qsort_interruptible を用いてソートする。 2. ソートされた配列から重複する値を排除し、バイナリ検索可能な状態を維持する。 3. 各属性の型情報（typlen, typbyval）に基づき、シリアライズに必要なサイズを計算する。

  • 固定長データ型はそのまま格納する。
  • varlena 型および cstring 型は、長さを付与した形式に変換する。

    • 4. 最終的に、インデックス化された値の配列とメタデータを含む bytea 形式へパックする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/extended_stats.c:811
  • **呼び出し先**:
  • lookup_type_cache
  • PrepareSortSupportFromOrderingOp
  • qsort_interruptible
  • compare_scalars_simple
  • compare_datums_simple
  • PG_DETOAST_DATUM
  • VARSIZE_ANY_EXHDR
  • DatumGetCString

関連データ構造

  • MCVList: 多変量統計情報を保持するメイン構造体。
  • VacAttrStats: 属性ごとの統計情報。
  • DimensionInfo: 各次元の型情報やシリアライズサイズを管理する内部構造体。
  • SortSupportData: ソート処理をサポートするためのデータ構造。

失敗モードと境界条件

  • 重複排除後の値の数が PG_UINT16_MAX を超える場合、アサーションエラーが発生する。
  • counts[dim] == 0 の場合（NULL値のみの場合）、その次元の処理はスキップされる。

保守担当者向けメモ

  • varlena 型の処理において、アライメントを考慮せずパックする仕様であるため、デシリアライズ時にはコピーを介したアクセスが必要となる点に注意してください。
