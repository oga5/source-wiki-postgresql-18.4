VarStringSortSupport

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/varlena.c:88

目的

可変長データ型（text, bpchar, bytea, name 等）のソート処理において、比較操作を最適化するためのキャッシュおよび状態保持用構造体です。

シグネチャまたは形状

typedef struct {
    char       *buf1;
    char       *buf2;
    int         buflen1;
    int         buflen2;
    int         last_len1;
    int         last_len2;
    int         last_returned;
    bool        cache_blob;
    bool        collate_c;
    Oid         typid;
    hyperLogLogState abbr_card;
    hyperLogLogState full_card;
    double      prop_card;
    pg_locale_t locale;
} VarStringSortSupport;

入力と出力

本構造体は関数引数として直接渡されるものではなく、ソートサポートのコンテキスト（ssup_extra）として保持されます。

  • **入力**: ソート対象のデータ型情報（typid）、ロケール情報（locale）、および比較用バッファ。
  • **出力**: 比較結果のキャッシュ（last_returned）およびカーディナリティ統計情報。

前提条件と不変条件

  • ソート実行前に palloc を用いてメモリを確保し、初期化する必要があります。
  • typid には text, bpchar, bytea, name のいずれかの OID が格納されます。
  • collate_c が真の場合、ロケール依存の比較をスキップする最適化パスが選択されます。

アルゴリズムまたは振る舞い

  • **比較キャッシュ**: 直前の比較結果（last_returned）を保持し、同一データの再比較を回避します。
  • **省略キー（Abbreviated Key）生成**: abbr_card および full_card を用いて、ソートの高速化に必要な省略キーの有効性を評価します。
  • **バッファ管理**: buf1, buf2 を使用して、比較に必要な文字列や strxfrm() 変換後のバイナリデータを一時的に保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/varlena.c 内のソートサポート関数群（varstr_sortsupport 等）から、ssup->ssup_extra を介して参照・更新されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • hyperLogLogState: カーディナリティ推定に使用される構造体。
  • pg_locale_t: ロケール依存の比較処理に使用される型。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • typid がサポート対象外の型である場合、比較ロジックが予期せぬ動作をする可能性があります。

保守担当者向けメモ

  • 本構造体は varstr_sortsupport のライフサイクルと同期して管理されます。メモリコンテキストの切り替え時には注意が必要です。
  • cache_blob フラグは、buf2 が単なる文字列ではなく strxfrm() 等による変換済みバイナリであるかどうかを識別するために重要です。
