test_aio_shmem_startup

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:67

目的

PostgreSQL の共有メモリ初期化プロセスにおいて、AIO テスト用モジュールの共有状態（InjIoErrorState）を確保し、エラー注入ポイント（Injection Points）を登録・ロードする。

シグネチャまたは形状

static void test_aio_shmem_startup(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリの初期化と注入ポイントの登録を行う）

前提条件と不変条件

  • AddinShmemInitLock による排他制御下で実行される。
  • prev_shmem_startup_hook が存在する場合は、本関数の実行前に呼び出される。

アルゴリズムまたは振る舞い

1. 既存の prev_shmem_startup_hook があれば実行する。 2. AddinShmemInitLock を取得し、共有メモリ領域 injection_points を確保またはアタッチする。 3. 初回実行時（!found）:

  • InjIoErrorState のフラグを初期化する。
  • USE_INJECTION_POINTS が定義されている場合、aio-process-completion-before-shared および aio-worker-after-reopen をアタッチし、ロードする。

    • 4. 2回目以降の実行時:

  • 注入ポイントをロードし、ログを出力する。

    • 5. AddinShmemInitLock を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: shmem_startup_hook に本関数が代入されている（src/test/modules/test_aio/test_aio.c:130）。
  • 呼び出し先: prev_shmem_startup_hook（存在する場合）、LWLockAcquire, ShmemInitStruct, InjectionPointAttach, InjectionPointLoad, LWLockRelease, elog。

関連データ構造

  • InjIoErrorState: エラー注入の状態を保持する共有メモリ構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、PostgreSQL の標準的なメモリ確保エラーが発生する可能性がある。
  • InjectionPoint 関連の関数は、ビルド時に USE_INJECTION_POINTS が定義されていない場合は動作しない。

保守担当者向けメモ

  • 本関数は shmem_startup_hook に登録されるフック関数である。
  • 注入ポイントのロードは、クリティカルセクション内での呼び出しを可能にするために行われる。
