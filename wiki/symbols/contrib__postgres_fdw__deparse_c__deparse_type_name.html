deparse_type_name

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/deparse.c:1154

目的

リモートの PostgreSQL サーバーで実行可能な SQL 文を構築する際、指定されたデータ型（type_oid）に対応する型名を文字列として取得する。

シグネチャまたは形状

char *deparse_type_name(Oid type_oid, int32 typemod)

入力と出力

  • **入力**:
  • type_oid (Oid): 変換対象のデータ型の OID。
  • typemod (int32): データ型の型修飾子（精度や長さなど）。
  • **出力**:
  • char *: リモート SQL で使用可能な型名の文字列。

前提条件と不変条件

  • is_builtin(type_oid) が偽（組み込み型ではない）の場合、返される型名にはスキーマ修飾が強制される。
  • format_type_extended を利用して文字列を生成するため、呼び出し元は返された文字列のメモリ管理（解放など）に責任を持つ必要がある。

アルゴリズムまたは振る舞い

1. FORMAT_TYPE_TYPEMOD_GIVEN フラグを初期設定する。 2. is_builtin(type_oid) を呼び出し、対象型が組み込み型か判定する。 3. 組み込み型でない場合、FORMAT_TYPE_FORCE_QUALIFY フラグを追加し、スキーマ修飾を強制する。 4. format_type_extended を呼び出し、型名文字列を生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/deparse.c 内の複数の箇所（定数ノードの処理、型キャスト、配列型、パラメータの型名生成など）。
  • **呼び出し先**:
  • is_builtin (PostgreSQL コア)
  • format_type_extended (PostgreSQL コア)

関連データ構造

  • Oid (PostgreSQL システム型)
  • int32 (PostgreSQL システム型)

失敗モードと境界条件

  • type_oid が無効な場合、format_type_extended の挙動に依存する。
  • is_builtin の判定ロジックにより、ユーザー定義型が正しくスキーマ修飾されない場合、リモート側での名前解決に失敗する可能性がある（shippable.c:143 に関連する懸念事項の言及あり）。

保守担当者向けメモ

  • 本関数は postgres_fdw における SQL 構築の基盤であり、型名の正確な出力が求められる。
  • is_builtin の判定基準に変更があった場合、本関数のスキーマ修飾ロジックに影響を与える可能性がある。
