window_cume_dist

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/windowfuncs.c:329

目的

SQLのウィンドウ関数 CUME_DIST（累積分布）を計算します。パーティション内の全行数に対する、現在の行以下の値を持つ行の割合を算出します。

シグネチャまたは形状

Datum window_cume_dist(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して WindowObject を受け取ります。
  • **出力**: float8 型の累積分布値（context->rank / totalrows）を Datum として返却します。

前提条件と不変条件

  • totalrows > 0 であること（Assert により保証）。
  • パーティション固有の状態を保持するため、WinGetPartitionLocalMemory を使用して rank_context を管理します。

アルゴリズムまたは振る舞い

1. WinGetPartitionRowCount でパーティション内の総行数を取得します。 2. rank_up を呼び出し、現在の行が前の行と異なる順位（ピアではない）かどうかを判定します。 3. 新しい順位に到達した場合（または最初の行の場合）、現在の行位置から順に WinRowsArePeers を用いてピア（同順位）である行を走査し、context->rank を更新します。 4. 最終的に、現在の累積順位を総行数で除算した値を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: ウィンドウ関数実行エンジン（src/backend/executor/nodeWindowAgg.c）から呼び出されます。
  • **呼び出し先**:
  • rank_up(): ピア判定の共通ユーティリティ。
  • WinGetPartitionLocalMemory(): パーティションごとの状態保持。
  • WinGetCurrentPosition(): 現在の行位置の取得。
  • WinRowsArePeers(): 行同士の同等性判定。
  • WinGetPartitionRowCount(): パーティション内の総行数取得。

関連データ構造

  • rank_context: パーティションごとの計算状態（現在のランク情報）を保持する構造体。

失敗モードと境界条件

  • パーティションが空の場合、Assert により停止します。
  • ピア判定は ORDER BY 句の定義に依存します。

保守担当者向けメモ

  • 本関数は window_cume_dist_support（ソースコード内での言及あり）と対になっており、オプティマイザによるフレーム最適化の対象となります。
  • ピア判定のロジックは rank_up に依存しているため、順位付けの仕様変更時は rank_up の実装を確認してください。
