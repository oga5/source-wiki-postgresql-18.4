SetExplainExtensionState

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_state.c:277

目的

ExplainState 構造体内に、拡張機能が独自の実行状態を保持するための不透明ポインタ（opaque）を格納する。

シグネチャまたは形状

void SetExplainExtensionState(ExplainState *es, int extension_id, void *opaque)

入力と出力

  • es: 操作対象の ExplainState 構造体へのポインタ。
  • extension_id: GetExplainExtensionId によって割り当てられた拡張機能固有のインデックス。
  • opaque: 拡張機能が保持したい状態データへのポインタ。
  • 出力: なし（副作用として es->extension_state 配列を更新）。

前提条件と不変条件

  • extension_id は 0 以上であること（Assert により検証）。
  • es は有効な ExplainState 構造体を指していること。

アルゴリズムまたは振る舞い

1. es->extension_state が未割り当ての場合、pg_nextpower2_32 を用いて必要なサイズを計算し、palloc0 で配列を確保する。 2. extension_id が現在の配列サイズ（extension_state_allocated）を超過している場合、repalloc0 を用いて配列を拡張し、新しいサイズを記録する。 3. 指定された extension_id の位置に opaque ポインタを格納する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_overexplain/pg_overexplain.c:99 での利用が確認されている。
  • 呼び出し先: palloc0, repalloc0, pg_nextpower2_32（いずれも PostgreSQL 内部のメモリ管理およびユーティリティ関数）。

関連データ構造

  • ExplainState: EXPLAIN コマンドの実行状態を保持する構造体。extension_state 配列およびそのサイズ管理用メンバを保持する。

失敗モードと境界条件

  • メモリ不足時: palloc0 または repalloc0 がエラーを発生させる。
  • extension_id が負の場合: Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • extension_state 配列は動的に拡張されるため、拡張機能側でポインタを保持し続ける場合は、repalloc0 による再配置の可能性を考慮する必要がある。
  • ID はバックエンドの生存期間中のみ有効である。
