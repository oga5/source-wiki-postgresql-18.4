Datum

定義

  • 種別: struct
  • 定義箇所: contrib/sepgsql/label.c:314

目的

contrib/sepgsql/label.c 内において、セキュリティラベルの遷移履歴を保持するスタック構造の各要素を定義します。

シグネチャまたは形状

struct {
    char *old_label;
    char *new_label;
    Datum next_private;
} *stack;

入力と出力

  • **入力**: old_label（変更前のラベル文字列）、new_label（変更後のラベル文字列）、next_private（スタックの次要素への参照）。
  • **出力**: なし（構造体定義）。

前提条件と不変条件

  • 本構造体は contrib/sepgsql/label.c 内のスタック操作において使用されます。
  • next_private フィールドは、PostgreSQL コアの Datum 型として定義されており、スタックの連結リスト構造を維持するために使用されます。

アルゴリズムまたは振る舞い

  • 本シンボルは、信頼されたプロシージャ実行時などのセキュリティコンテキスト遷移において、元のラベルを退避し、遷移後のラベルを記録するためのデータコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは、本構造体を利用する具体的な関数呼び出しの直接的な証拠は確認できません。
  • **呼び出し先**: なし（データ構造定義のため）。
  • 注: ツールによる Datum の参照結果（amcheck 等）は PostgreSQL コアの typedef Datum を指すものであり、本ファイルで定義された struct とは別物です。

関連データ構造

  • pending_label: トランザクションに関連するラベル変更を管理するリスト。
  • sepgsql_fmgr_hook: 本構造体を利用してラベルのスタック退避・復元を行う可能性のあるフック関数。

失敗モードと境界条件

  • メモリ管理: old_label および new_label の文字列ポインタは、適切なメモリコンテキスト（TopMemoryContext 等）で管理されない場合、メモリリークや不正な参照を引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは postgres.h で定義される typedef Datum と名前が重複していますが、contrib/sepgsql/label.c 内ではローカルな struct として定義されています。コード修正時には、コアの Datum 型と混同しないよう注意してください。
