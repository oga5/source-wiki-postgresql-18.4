gbt_int4key_cmp

定義

  • contrib/btree_gist/btree_int4.c:53

目的

GiST インデックスにおける int4 型の範囲キー（int32KEY）同士を比較し、ソート順序を決定する。

シグネチャまたは形状

static int gbt_int4key_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部の t メンバが int32KEY としてキャストされる。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • 0: 両方のキーが等しい場合。
  • 1: a が b より大きい場合。
  • -1: a が b より小さい場合。

前提条件と不変条件

  • 入力 a および b は Nsrt 構造体へのポインタであり、その t メンバは int32KEY 構造体として解釈可能であること。
  • int32KEY は lower および upper メンバを持つ。

アルゴリズムまたは振る舞い

1. a と b の lower 値を比較する。

  • ia->lower != ib->lower の場合、その大小関係を返す。

    • 2. lower が等しい場合、upper 値を比較する。

  • ia->upper == ib->upper ならば 0 を返す。
  • ia->upper > ib->upper ならば 1 を返し、そうでなければ -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int4.c:87（tinfo 構造体の初期化時に参照されている）。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • int32KEY: int4 型の範囲を保持する構造体（lower, upper）。
  • Nsrt: ソート対象のデータを保持するラッパー構造体。

失敗モードと境界条件

  • a または b が NULL の場合、または Nsrt 構造体として正しくキャストできない場合は未定義の動作となる。
  • 整数値の比較であるため、オーバーフロー等の境界条件は int4 型の仕様に準拠する。

保守担当者向けメモ

  • 本関数は int4 型のソート順序を定義するものであり、インデックス構築時のソートサポートに使用される。
  • flinfo 引数は現在の実装では使用されていないが、シグネチャの一部として維持されている。
