pg_sha1_ctx

定義

  • 種別: struct
  • 定義箇所: src/common/sha1_int.h:56

目的

SHA-1ハッシュ計算における計算状態（コンテキスト）を保持するための内部構造体です。

シグネチャまたは形状

typedef struct
{
	union { uint8 b8[20]; uint32 b32[5]; } h;
	union { uint8 b8[8]; uint64 b64[1]; } c;
	union { uint8 b8[64]; uint32 b32[16]; } m;
	uint8 count;
} pg_sha1_ctx;

入力と出力

  • h: ハッシュ値（中間状態および最終結果）を保持します。
  • c: データ長を保持します。
  • m: メッセージブロックを一時的に蓄積するバッファです。
  • count: 現在のブロック内オフセットを管理します。

前提条件と不変条件

  • 本構造体は src/common/sha1.c で実装されるSHA-1計算のライフサイクル（初期化・更新・終了）を通じて使用されます。
  • 構造体内の各 union は、バイト単位およびワード単位での効率的なアクセスを目的としています。

アルゴリズムまたは振る舞い

1. pg_sha1_init により、コンテキストがゼロクリアされ初期化されます。 2. pg_sha1_update を通じて入力データが m バッファに蓄積され、ハッシュ値が逐次更新されます。 3. pg_sha1_final にてパディング処理が行われ、最終的なハッシュ値が指定されたメモリ領域へ書き出されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/common/cryptohash.c: 暗号ハッシュAPIのラッパーとして使用。
  • src/common/sha1.c: SHA-1アルゴリズムの各ステップ（sha1_step, sha1_pad, sha1_result 等）で引数として使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • src/common/sha1.h: 公開インターフェース定義。
  • src/common/cryptohash.c: 本構造体を利用する上位の抽象化レイヤー。

失敗モードと境界条件

  • 外部モジュールからの直接参照は非推奨であり、cryptohash.h 経由のインターフェース利用が推奨されます。
  • 構造体内部の union へのアクセスにおいて、アライメントや境界外アクセスの不整合が発生しないよう、実装側（sha1.c）での厳密な管理が必要です。

保守担当者向けメモ

  • FIPS 180-1に基づく古典的な実装です。
  • 内部用ヘッダーであるため、変更時は src/common/sha1.c との整合性に注意してください。
