SharedInvalSnapshotMsg

定義

  • 種別: struct
  • 定義箇所: src/include/storage/sinval.h:107

目的

共有キャッシュ無効化メカニズムにおいて、スナップショットに関連する無効化イベントを伝達するためのメッセージ構造体です。

シグネチャまたは形状

typedef struct
{
    int8    id;         /* メッセージ型識別子（先頭必須） */
    Oid     dbId;       /* データベースID（共有リレーションの場合は0） */
    Oid     relId;      /* リレーションID */
} SharedInvalSnapshotMsg;

入力と出力

  • 入力: 共有メモリ上の無効化キューを介して、他のバックエンドから受信されます。
  • 出力: 受信したバックエンドにおいて、指定された dbId および relId に関連するスナップショットやキャッシュ状態の無効化処理をトリガーします。

前提条件と不変条件

  • id フィールドは構造体の先頭に配置されなければなりません。
  • dbId が 0 の場合、そのリレーションは共有リレーションとして扱われます。

アルゴリズムまたは振る舞い

1. 送信側バックエンドがスナップショットに関連する変更を行った際、本構造体に値を設定し共有キューへ投入します。 2. 受信側バックエンドは ReceiveSharedInvalidMessages を通じて本メッセージを読み取ります。 3. メッセージの id に基づき、対応するキャッシュ無効化関数が呼び出され、ローカルキャッシュの整合性が維持されます。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数名は、提供されたソース抜粋からは特定できません。
  • src/include/storage/sinval.h 内で SharedInvalSnapshotMsg sn; として参照されています。

関連データ構造

  • SharedInvalidationMessage: 本構造体を含む共用体。
  • SharedInvalCatcacheMsg 等: 他の無効化メッセージ型。

失敗モードと境界条件

  • 共有メモリ上の無効化キューが溢れた場合、PROCSIG_CATCHUP_INTERRUPT による強制的な同期処理が発生する可能性があります。
  • ハッシュ衝突により、無関係なエントリが誤って無効化される可能性がありますが、これはキャッシュの再ロードによって補完されます。

保守担当者向けメモ

  • 本構造体は SharedInvalidationMessage 共用体の一部として使用されるため、サイズやフィールドの配置変更には注意が必要です。
  • トランザクションのコミット時に送信されるメッセージであるため、処理の遅延がトランザクションの整合性に影響を与えないよう設計されています。
