ExecSetSlotDescriptor

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execTuples.c:1477

目的

TupleTableSlot に新しいタプル記述子（TupleDesc）を割り当て、スロットが保持する属性値配列（tts_values および tts_isnull）を新しい記述子の属性数に合わせて再構築する。

シグネチャまたは形状

void ExecSetSlotDescriptor(TupleTableSlot *slot, TupleDesc tupdesc)

入力と出力

  • **入力**:
  • slot: 変更対象の TupleTableSlot。
  • tupdesc: 新しく設定する TupleDesc。
  • **出力**: なし（副作用としてスロット内部の状態を更新）。

前提条件と不変条件

  • スロットが固定（TTS_FIXED）状態であってはならない（Assert による検証）。
  • スロットの属性値配列は、スロット自身のメモリコンテキスト（slot->tts_mcxt）で管理される必要がある。

アルゴリズムまたは振る舞い

1. ExecClearTuple(slot) を呼び出し、スロットを空にする。 2. 既存の tts_tupleDescriptor があれば ReleaseTupleDesc で解放する。 3. 既存の tts_values および tts_isnull 配列を pfree で解放する。 4. 新しい tupdesc をスロットにセットし、PinTupleDesc で参照カウントを増やす。 5. 新しい tupdesc->natts に基づき、スロットのメモリコンテキストから tts_values（Datum 配列）と tts_isnull（bool 配列）を再割り当てする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execJunk.c
  • src/backend/executor/execUtils.c
  • **呼び出し先**:
  • ExecClearTuple
  • ReleaseTupleDesc
  • pfree
  • PinTupleDesc
  • MemoryContextAlloc

関連データ構造

  • TupleTableSlot: タプルを保持するコンテナ。
  • TupleDesc: タプルの構造を定義するメタデータ。

失敗モードと境界条件

  • TTS_FIXED なスロットに対して呼び出した場合、アサーション違反が発生する。
  • メモリ不足時には MemoryContextAlloc がエラーをスローする。

保守担当者向けメモ

  • 本関数はスロットの記述子を動的に変更する際に使用される。
  • 既存の属性値配列を再利用せず、常に新しいサイズで割り当て直す実装となっている。
