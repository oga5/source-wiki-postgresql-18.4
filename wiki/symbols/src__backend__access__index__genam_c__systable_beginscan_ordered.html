systable_beginscan_ordered

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:649

目的

システムカタログに対する順序付きインデックススキャンを開始するための記述子（SysScanDesc）を初期化する。

シグネチャまたは形状

SysScanDesc systable_beginscan_ordered(Relation heapRelation,
                                       Relation indexRelation,
                                       Snapshot snapshot,
                                       int nkeys,
                                       ScanKey key)

入力と出力

  • **入力**:
  • heapRelation: スキャン対象のヒープリレーション。
  • indexRelation: スキャンに使用するインデックスリレーション。
  • snapshot: スキャンに使用するスナップショット（NULLの場合はカタログスナップショットを新規登録）。
  • nkeys: スキャンキーの数。
  • key: インデックススキャンに使用するキー配列。
  • **出力**:
  • 初期化済みの SysScanDesc 構造体へのポインタ。

前提条件と不変条件

  • 指定された indexRelation がリインデックス処理中である場合、エラーが発生する。
  • IgnoreSystemIndexes が有効な場合、警告が出力される。
  • key に含まれる属性番号は、インデックス内の列番号に変換される必要がある。変換できない場合はエラーとなる。

アルゴリズムまたは振る舞い

1. SysScanDescData 構造体をメモリ確保し、ヒープおよびインデックスリレーションを保持する。 2. table_slot_create を用いてタプルスロットを作成する。 3. スナップショットが提供されない場合、GetCatalogSnapshot で取得し、RegisterSnapshot で登録する。 4. 入力された key をインデックスの列順序に合わせて変換し、idxkey を作成する。 5. index_beginscan を呼び出し、インデックススキャン記述子を生成する。 6. index_rescan を呼び出し、変換済みの idxkey を適用する。 7. CheckXidAlive が有効な場合、グローバル変数 bsysscan を true に設定する。 8. 最後に idxkey を解放し、生成した SysScanDesc を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • src/backend/access/common/toast_internals.c
  • src/backend/access/heap/heaptoast.c
  • src/backend/commands/event_trigger.c
  • src/backend/storage/large_object/inv_api.c
  • src/backend/utils/adt/enum.c
  • **呼び出し先**:
  • index_beginscan, index_rescan, table_slot_create, RegisterSnapshot, GetCatalogSnapshot 等（ソースコード内の直接的な呼び出しに基づく）。

関連データ構造

  • SysScanDesc: システムカタログスキャンを管理する構造体。
  • ScanKey: スキャン条件を保持するデータ構造。
  • Relation: リレーション記述子。

失敗モードと境界条件

  • インデックスがリインデックス処理中の場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。
  • スキャンキーの属性がインデックスに含まれていない場合、elog(ERROR) で終了する。

保守担当者向けメモ

  • 本関数は順序付きスキャン専用であり、systable_beginscan とは異なりヒープスキャンへのフォールバック機能を持たない。
  • bsysscan フラグの操作はトランザクション管理と密接に関連しているため、変更時は xact.c の関連コメントを参照すること。
