get_am_type_string

定義

  • 種別: function
  • 定義箇所: src/backend/commands/amcmds.c:211

目的

アクセスメソッド（AM）の型を示す内部文字コードを、人間が判読可能な文字列（"INDEX" または "TABLE"）に変換する。

シグネチャまたは形状

const char *get_am_type_string(char amtype);

入力と出力

  • **入力**: amtype (char) - アクセスメソッドの型を表す文字コード（AMTYPE_INDEX または AMTYPE_TABLE）。
  • **出力**: const char * - 対応する文字列リテラル。

前提条件と不変条件

  • 入力 amtype は AMTYPE_INDEX または AMTYPE_TABLE のいずれかである必要がある。

アルゴリズムまたは振る舞い

1. switch 文を用いて amtype を評価する。 2. AMTYPE_INDEX の場合は "INDEX" を返す。 3. AMTYPE_TABLE の場合は "TABLE" を返す。 4. 上記以外の値が渡された場合、elog(ERROR, ...) を呼び出し、エラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/amcmds.c 内のコードから参照されている（行145など）。
  • **呼び出し先**: elog（エラー報告用マクロ）。

関連データ構造

  • AMTYPE_INDEX, AMTYPE_TABLE: アクセスメソッドの型を識別するための定数（catalog/pg_am.h 等で定義）。

失敗モードと境界条件

  • 定義されていない amtype が渡された場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は主にエラーメッセージの生成やデバッグ出力に使用される。
  • 新しいアクセスメソッドタイプがシステムに追加された場合は、本関数の switch 文に新しいケースを追加する必要がある。
