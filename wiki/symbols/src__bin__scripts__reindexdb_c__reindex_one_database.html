reindex_one_database

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/reindexdb.c:278

目的

指定されたデータベースに対して、対象オブジェクト（データベース、テーブル、インデックス、スキーマ）の再構築（REINDEX）処理を実行する。

シグネチャまたは形状

static void reindex_one_database(ConnParams *cparams, ReindexType type,
                                 SimpleStringList *user_list,
                                 const char *progname, bool echo,
                                 bool verbose, bool concurrently, int concurrentCons,
                                 const char *tablespace)

入力と出力

  • **入力**: 接続パラメータ、再構築タイプ、対象リスト、実行オプション（並列数、CONCURRENTLY 指定、テーブルスペース等）。
  • **出力**: なし（成功時）。エラー発生時は pg_fatal によりプロセスを終了する。

前提条件と不変条件

  • concurrently が true の場合、サーバーバージョンは 12 以上であること。
  • tablespace が指定されている場合、サーバーバージョンは 14 以上であること。
  • 並列実行（concurrentCons > 1）時、REINDEX SYSTEM はサポートされない。

アルゴリズムまたは振る舞い

1. connectDatabase を呼び出し、対象データベースへ接続する。 2. 並列実行モードでない場合、対象リストを構築する。 3. 並列実行モードの場合、get_parallel_tables_list または get_parallel_tabidx_list を用いて対象オブジェクトのリストを生成する。 4. ParallelSlotsSetup を使用して接続スロットを初期化し、既存の接続を移行する。 5. リスト内の各項目に対し、gen_reindex_command で SQL を生成し、ParallelSlot を介して非同期に実行する。 6. インデックス単位の並列再構築では、同一テーブルに属するインデックスをグループ化し、一つの SQL クエリとして発行する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/scripts/reindexdb.c 内の複数箇所から呼び出し）。
  • **呼び出し先**: connectDatabase, get_parallel_tables_list, get_parallel_tabidx_list, ParallelSlotsSetup, ParallelSlotsAdoptConn, ParallelSlotsGetIdle, ParallelSlotSetHandler, gen_reindex_command 等。

関連データ構造

  • ConnParams: 接続設定。
  • ReindexType: 再構築対象の種別（列挙型）。
  • SimpleStringList: 処理対象オブジェクトのリスト。
  • ParallelSlotArray: 並列実行のための接続管理構造体。

失敗モードと境界条件

  • サーバーバージョンがオプション要件を満たさない場合、pg_fatal で終了する。
  • 並列実行時に処理対象リストが空の場合、接続を閉じて早期リターンする。
  • CancelRequested が検知された場合、処理を中断する。
  • 並列スロットが確保できない場合、エラーとして終了する。

保守担当者向けメモ

  • REINDEX_INDEX の並列処理では、同一テーブルのインデックスが連続するようにソートされている前提で SQL が生成される。
  • gen_reindex_command はサーバーバージョンに応じてオプションの配置を制御するため、新しいオプション追加時はこの関数と gen_reindex_command の両方の確認が必要。
