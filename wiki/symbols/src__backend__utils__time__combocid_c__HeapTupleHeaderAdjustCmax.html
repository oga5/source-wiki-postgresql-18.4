HeapTupleHeaderAdjustCmax

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/combocid.c:152

目的

タプル削除時、同一トランザクション内で挿入されたタプルに対して、限られたヘッダー領域を効率的に利用するための「Combo CID」を生成・適用する。

シグネチャまたは形状

void HeapTupleHeaderAdjustCmax(const HeapTupleHeaderData *tup, CommandId *cmax, bool *iscombo)

入力と出力

  • **入力**:
  • tup: 対象タプルのヘッダーデータ。
  • cmax: 削除時のコマンドID（ポインタ経由で更新される）。
  • **出力**:
  • cmax: Combo CIDが必要な場合、変換後のIDで上書きされる。
  • iscombo: Combo CIDが適用された場合は true、それ以外は false が設定される。

前提条件と不変条件

  • HeapTupleHeaderXminCommitted(tup) が偽であり、かつ TransactionIdIsCurrentTransactionId が真である場合（＝自トランザクション内で挿入されたタプルを削除する場合）にのみ、Combo CIDが生成される。

アルゴリズムまたは振る舞い

1. タプルの xmin がコミット済みか確認し、未コミットかつ自トランザクション内の挿入であるか判定する。 2. 条件を満たす場合、HeapTupleHeaderGetCmin(tup) で取得した cmin と現在の cmax を GetComboCommandId に渡し、Combo CIDを取得する。 3. 取得したCombo CIDを cmax に代入し、iscombo を true に設定する。 4. 条件を満たさない場合は iscombo を false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c (複数箇所でタプル削除処理時に呼び出し)
  • **呼び出し先**:
  • HeapTupleHeaderXminCommitted
  • TransactionIdIsCurrentTransactionId
  • HeapTupleHeaderGetRawXmin
  • HeapTupleHeaderGetCmin
  • GetComboCommandId

関連データ構造

  • HeapTupleHeaderData: 操作対象のタプルヘッダー。
  • CommandId: コマンドID。

失敗モードと境界条件

  • 供給されたソース情報には、特定の失敗モードや境界条件に関する明示的な記述はありません。

保守担当者向けメモ

  • 本関数は SetCmax を実行する前に呼び出す必要があります。
  • GetComboCommandId を介して comboHash および comboCids 配列を操作するため、メモリ管理は TopTransactionContext に依存しています。
