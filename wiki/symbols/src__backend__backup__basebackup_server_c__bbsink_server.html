bbsink_server

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup_server.c:21

目的

PostgreSQL のベースバックアップをサーバー上のローカルファイルシステムへ直接保存するための状態を保持する構造体です。bbsink インターフェースを実装する具象クラスとして機能します。

シグネチャまたは形状

typedef struct bbsink_server
{
    bbsink      base;
    char       *pathname;
    File        file;
    off_t       filepos;
} bbsink_server;

入力と出力

  • **入力**: バックアップストリームのデータ（bbsink インターフェース経由で受信）。
  • **出力**: 指定されたディレクトリへのファイル書き込みおよびマニフェストファイルの永続化。

前提条件と不変条件

  • pathname にはバックアップを保存するディレクトリの絶対パスが格納されます。
  • file は現在オープン中のファイル記述子を保持し、何も開いていない場合は 0 となります。
  • filepos は現在のファイル書き込み位置を追跡します。

アルゴリズムまたは振る舞い

  • **アーカイブ処理**: begin_archive でファイルを作成し、archive_contents でデータを書き込み、end_archive で FileSync を実行してクローズします。
  • **マニフェスト処理**: begin_manifest で一時ファイル（.tmp）を作成し、manifest_contents で書き込み、end_manifest でクローズ後に durable_rename を用いて確定させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup_server.c 内の各メソッド（bbsink_server_new 等）がこの構造体をインスタンス化またはキャストして使用します。
  • **呼び出し先**: storage/fd.h で定義されたファイル操作関数（FileWrite, FileSync 等）を呼び出します。

関連データ構造

  • bbsink: 基底インターフェース構造体。
  • File: storage/fd.h で定義されるファイル記述子型。

失敗モードと境界条件

  • fsync 失敗時は ERROR を発生させます。
  • マニフェスト保存時に durable_rename を使用しており、ファイルが存在すればバックアップが正常に完了したとみなす設計です。

保守担当者向けメモ

  • 本構造体はサーバー上のローカルディレクトリにバックアップを出力する場合にのみ使用されます。
  • durable_rename によるアトミックな保存を行っているため、マニフェストファイルの存在確認がバックアップ完了の判定基準となります。
