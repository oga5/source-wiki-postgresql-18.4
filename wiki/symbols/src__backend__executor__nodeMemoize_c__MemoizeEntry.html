MemoizeEntry

定義

  • **種別**: struct
  • **定義箇所**: src/backend/executor/nodeMemoize.c:115

目的

パラメータ化されたプランノードの実行結果をキャッシュするためのハッシュテーブルエントリを保持する。

シグネチャまたは形状

typedef struct MemoizeEntry
{
    MemoizeKey *key;            /* ハッシュテーブル検索用キー */
    MemoizeTuple *tuplehead;    /* キャッシュされたタプルの先頭、またはNULL */
    uint32      hash;           /* キャッシュされたハッシュ値 */
    char        status;         /* ハッシュステータス */
    bool        complete;       /* 外側のプランを完了まで読み込んだか */
} MemoizeEntry;

入力と出力

  • **入力**: MemoizeKey を介したハッシュ検索キー。
  • **出力**: tuplehead を通じてキャッシュされたタプル群へのアクセスを提供。

前提条件と不変条件

  • tuplehead が NULL の場合、そのキーに対してキャッシュされたタプルは存在しない。
  • complete フラグは、サブプランの全タプルが取得済みであることを示す。

アルゴリズムまたは振る舞い

  • simplehash.h を用いたハッシュテーブルの要素として機能する。
  • キャッシュの充填（MEMO_FILLING_CACHE）時にタプルが格納され、検索（MEMO_CACHE_LOOKUP）時に利用される。
  • メモリ制限超過時には、LRUリストに基づきエントリ単位で破棄される。

呼び出し元と呼び出し先

  • **呼び出し元**: nodeMemoize.c 内のキャッシュ管理関数（entry_purge_tuples, remove_cache_entry 等）およびハッシュテーブル操作関数。
  • **呼び出し先**: MemoizeKey（キー定義）、MemoizeTuple（タプル格納）。

関連データ構造

  • MemoizeKey: ハッシュテーブルの検索キー。
  • MemoizeTuple: キャッシュされるタプルのリスト構造。
  • MemoizeState: 本エントリを管理するノード実行状態。

失敗モードと境界条件

  • メモリ不足時: キャッシュエントリが削除されるか、バイパスモードへ移行する。
  • singlerow 最適化: 最初のタプル取得後に complete が真となる場合がある。

保守担当者向けメモ

  • simplehash.h の再配置（リサイズ）が発生するため、ポインタの扱いに注意すること。
  • EMPTY_ENTRY_MEMORY_BYTES マクロにより、エントリ自体のメモリ消費量が計算される。
