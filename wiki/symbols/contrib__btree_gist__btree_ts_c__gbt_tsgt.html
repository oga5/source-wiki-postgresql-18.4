gbt_tsgt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_ts.c:45

目的

2つの Timestamp 値を比較し、左辺が右辺より大きいかどうかを判定する。GiSTインデックスにおける比較演算子として使用される。

シグネチャまたは形状

static bool gbt_tsgt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の Timestamp へのポインタ。
  • b: 比較対象の Timestamp へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では直接使用されない）。
  • **出力**:
  • bool: *a > *b であれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な Timestamp 型の値を指している必要がある。

アルゴリズムまたは振る舞い

1. 入力された void * ポインタを Timestamp * にキャストする。 2. PostgreSQLのコア関数 timestamp_gt を DirectFunctionCall2 を介して呼び出す。 3. TimestampGetDatumFast マクロを使用して、各タイムスタンプを Datum 形式に変換して比較を行う。 4. 比較結果を DatumGetBool で論理値に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_ts.c 内の静的構造体（tinfo 等）で参照されている。
  • **呼び出し先**: timestamp_gt（PostgreSQLコアのタイムスタンプ比較関数）。

関連データ構造

  • Timestamp: PostgreSQLのタイムスタンプ型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性がある。
  • タイムスタンプ値が範囲外（オーバーフロー等）の場合、timestamp_gt の挙動に依存する。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する共通ロジックから呼び出されることを想定したラッパー関数である。
  • timestamptz 型の比較においても、内部的にGMTへ正規化された Timestamp が渡される前提で動作する。
