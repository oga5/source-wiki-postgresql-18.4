TBlockState

定義

  • 種別: enum
  • 定義箇所: src/backend/access/transam/xact.c:157

目的

SQLレベルのトランザクションブロックの状態を管理し、トランザクションのライフサイクル（開始、コミット、アボート、サブトランザクション）を追跡するために使用されます。

シグネチャまたは形状

typedef enum TBlockState {
    TBLOCK_DEFAULT,
    TBLOCK_STARTED,
    TBLOCK_BEGIN,
    TBLOCK_INPROGRESS,
    TBLOCK_IMPLICIT_INPROGRESS,
    TBLOCK_PARALLEL_INPROGRESS,
    TBLOCK_END,
    TBLOCK_ABORT,
    TBLOCK_ABORT_END,
    TBLOCK_ABORT_PENDING,
    TBLOCK_PREPARE,
    TBLOCK_SUBBEGIN,
    TBLOCK_SUBINPROGRESS,
    TBLOCK_SUBRELEASE,
    TBLOCK_SUBCOMMIT,
    TBLOCK_SUBABORT,
    TBLOCK_SUBABORT_END,
    TBLOCK_SUBABORT_PENDING,
    TBLOCK_SUBRESTART,
    TBLOCK_SUBABORT_RESTART
} TBlockState;

入力と出力

  • 入力: トランザクション制御コマンド（BEGIN, COMMIT, ROLLBACK, SAVEPOINT等）および内部的な状態遷移イベント。
  • 出力: 現在のトランザクション状態を示す列挙値。

前提条件と不変条件

  • TransactionStateData 構造体の blockState メンバとして保持されます。
  • トランザクションの階層構造（メイントランザクションとサブトランザクション）に応じて、適切な状態遷移が行われる必要があります。

アルゴリズムまたは振る舞い

  • トランザクションの進行状況を「非トランザクションブロック」「トランザクションブロック」「サブトランザクション」の3つのカテゴリで管理します。
  • StartTransactionCommand や CommitTransactionCommand などの制御関数によって、現在の状態が遷移します。
  • BlockStateAsString() 関数により、デバッグやログ出力用に状態名を文字列へ変換可能です。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c 内のトランザクション制御ロジック。
  • 呼び出し先: 直接的な関数呼び出しではなく、状態遷移の判定基準として参照されます。
  • 補足: 具体的な関数呼び出しのフローについては、提供されたソース excerpt には詳細な記述がありません。

関連データ構造

  • TransactionStateData: TBlockState を保持する主要な構造体。
  • CurrentTransactionState: 現在のトランザクション状態を指すポインタ。

失敗モードと境界条件

  • 不正な状態遷移（例: トランザクション外でのCOMMITなど）が発生した場合、PostgreSQLのトランザクション管理ロジックによってエラーがスローされます。
  • 並列クエリ実行時の TBLOCK_PARALLEL_INPROGRESS 状態など、特殊な実行環境下での状態管理に依存します。

保守担当者向けメモ

  • 本列挙型はトランザクションの挙動を理解する上で最も重要な指標の一つです。
  • 状態遷移のロジックは xact.c 内に集中しているため、変更を加える際はトランザクションの整合性に細心の注意を払ってください。
  • 呼び出し元・呼び出し先の詳細な依存関係については、ソースコード全体の解析が必要です。
