CachedPlanSource

定義

  • **種別**: struct
  • **定義箇所**: src/include/utils/plancache.h:105

目的

SQLクエリの解析から実行計画の生成に至るライフサイクルを管理し、再利用可能な計画（キャッシュ）を保持するための構造体です。

シグネチャまたは形状

typedef struct CachedPlanSource {
    int magic;
    struct RawStmt *raw_parse_tree;
    struct Query *analyzed_parse_tree;
    const char *query_string;
    CommandTag commandTag;
    Oid *param_types;
    int num_params;
    ParserSetupHook parserSetup;
    void *parserSetupArg;
    PostRewriteHook postRewrite;
    void *postRewriteArg;
    int cursor_options;
    bool fixed_result;
    TupleDesc resultDesc;
    MemoryContext context;
    List *query_list;
    List *relationOids;
    List *invalItems;
    struct SearchPathMatcher *search_path;
    MemoryContext query_context;
    Oid rewriteRoleId;
    bool rewriteRowSecurity;
    bool dependsOnRLS;
    struct CachedPlan *gplan;
    bool is_oneshot;
    bool is_complete;
    bool is_saved;
    bool is_valid;
    int generation;
    dlist_node node;
    double generic_cost;
    double total_custom_cost;
    int64 num_custom_plans;
    int64 num_generic_plans;
} CachedPlanSource;

入力と出力

  • **入力**: クエリ文字列、解析木、パラメータ型情報、パーサー設定フックなど。
  • **出力**: 実行可能な CachedPlan（gplan フィールド経由）、およびクエリの実行結果の型情報（resultDesc）。

前提条件と不変条件

  • magic フィールドは常に CACHEDPLANSOURCE_MAGIC と一致する必要があります。
  • is_saved が真の場合、この構造体はグローバルリストのメンバーとして管理されます。
  • is_valid が偽の場合、クエリリストは無効であり、再計画（Replan）が必要です。
  • is_oneshot が真の場合、キャッシュ無効化の対象外となります。

アルゴリズムまたは振る舞い

1. **初期化**: CreateCachedPlan 等で生成され、クエリの解析・書き換え結果を保持します。 2. **計画選択**: generic_cost や過去のカスタム計画のコスト統計に基づき、汎用計画（Generic Plan）とカスタム計画のどちらを使用するかを判断します。 3. **無効化**: スキーマ変更等のイベント発生時、依存関係（relationOids, invalItems）に基づき is_valid が false に設定されます。 4. **再計画**: GetCachedPlan() 呼び出し時に is_valid を確認し、無効であれば再解析・再計画を実行して generation を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/prepare.c（プリペアドステートメントの作成・管理）、src/backend/executor/functions.c（関数実行時の計画管理）からの参照が確認されています。
  • **呼び出し先**: 構造体内部で CachedPlan を保持し、ParserSetupHook や PostRewriteHook を介して外部ロジックを呼び出します。

関連データ構造

  • CachedPlan: 実行可能な計画本体。
  • RawStmt, Query: 解析木。
  • TupleDesc: 結果のタプル記述子。
  • dlist_node: グローバルリスト管理用の侵入型リストノード。

失敗モードと境界条件

  • is_valid が false の状態で計画を取得しようとすると、再計画処理がトリガーされます。
  • is_oneshot クエリはキャッシュ無効化イベントを監視しないため、メモリコンテキストの寿命管理に注意が必要です。
  • 参照カウント管理を誤るとメモリリークが発生します。

保守担当者向けメモ

  • 構造体の整合性チェックには CACHEDPLANSOURCE_MAGIC を使用してください。
  • is_saved 状態の CachedPlanSource は、src/backend/utils/cache/plancache.c のグローバルリストを通じて共有キャッシュ無効化イベントの影響を受けます。
