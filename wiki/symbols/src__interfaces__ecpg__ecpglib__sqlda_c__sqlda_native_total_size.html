sqlda_native_total_size

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/sqlda.c:185

目的

ECPGネイティブモードにおいて、PGresult から構築される SQLDA 構造体全体（メタデータおよびデータ領域）に必要なメモリサイズを算出します。

シグネチャまたは形状

static long sqlda_native_total_size(const PGresult *res, int row, enum COMPAT_MODE compat)

入力と出力

  • **入力**:
  • res: 参照する PGresult オブジェクト。
  • row: 対象とする行番号（負の場合はメタデータ領域のみを計算）。
  • compat: 互換モード指定。
  • **出力**:
  • long: 必要なメモリ合計サイズ（バイト単位）。

前提条件と不変条件

  • res は有効な PGresult ポインタである必要があります。
  • 戻り値は、sqlda_native_empty_size で算出されるベースサイズに、sqlda_common_total_size で算出されるデータ領域サイズを加算した値となります。

アルゴリズムまたは振る舞い

1. sqlda_native_empty_size(res) を呼び出し、構造体のヘッダーおよびフィールド定義に必要な初期サイズを取得します。 2. row が負の場合、初期サイズをそのまま返します。 3. row が 0 以上の場合、sqlda_common_total_size を呼び出し、指定された行のデータ格納に必要なサイズを初期サイズに加算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/sqlda.c:418（ecpg_build_native_sqlda 内でのメモリ確保サイズ算出に使用）。
  • **呼び出し先**:
  • sqlda_native_empty_size
  • sqlda_common_total_size

関連データ構造

  • PGresult: PostgreSQL のクエリ結果セット。
  • SQLDA: ECPG で使用される SQL Descriptor Area。
  • COMPAT_MODE: 互換性モードを定義する列挙型。

失敗モードと境界条件

  • row が負の値の場合、データ領域を含まないメタデータのみのサイズを返す仕様となっています。
  • PGresult が NULL の場合の挙動については、本関数内では明示的なチェックを行っていません。

保守担当者向けメモ

  • 本関数はネイティブモード専用です。Informix 互換モードについては sqlda_compat_total_size を参照してください。
  • メモリレイアウトの整合性を維持するため、サイズ計算ロジックを変更する際は sqlda_common_total_size および ecpg_sqlda_align_add_size との整合性を確認してください。
