cube_in

定義

  • 種別: function
  • 定義箇所: contrib/cube/cube.c:120

目的

PostgreSQL の cube 型（多次元ボックス）の入力関数であり、外部表現（文字列）から内部表現（NDBOX 構造体）への変換を行います。

シグネチャまたは形状

Datum cube_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して受け取る文字列（char *）。
  • **出力**: NDBOX 構造体へのポインタを PG_RETURN_NDBOX_P マクロでラップして返します。

前提条件と不変条件

  • 入力文字列は cube 型として有効な形式である必要があります。
  • cube_scanner_init および cube_yyparse を使用するため、スキャナとパーサーの初期化・終了処理が正しく行われる必要があります。

アルゴリズムまたは振る舞い

1. cube_scanner_init を呼び出し、入力文字列からスキャナを初期化します。 2. cube_yyparse を呼び出し、構文解析を行って NDBOX 構造体を生成します。 3. cube_scanner_finish を呼び出し、スキャナのリソースを解放します（解析失敗時も実行されます）。 4. 生成された NDBOX 構造体を Datum 型として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の型システム（SQL レベルでの型変換時に呼び出されます）。
  • **呼び出し先**:
  • cube_scanner_init
  • cube_yyparse
  • cube_scanner_finish
  • 注: 呼び出し元および呼び出し先の詳細なコードパスは、提供されたソース excerpt の範囲外については明示されていません。

関連データ構造

  • NDBOX: 多次元ボックスを保持する主要なデータ構造。
  • yyscan_t: 字句解析器の状態を保持する構造体。

失敗モードと境界条件

  • 入力文字列が不正な形式である場合、cube_yyparse 内でエラーが発生し、処理が中断される可能性があります。
  • メモリ不足等のシステムエラーが発生した場合、PostgreSQL のエラーハンドリング機構により処理が停止します。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(cube_in) を通じて PostgreSQL に登録されています。
  • 構文解析ロジックは cube_yyparse に依存しているため、文法の変更が必要な場合はパーサー定義を確認してください。
