BTScanKeyPreproc

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtpreprocesskeys.c:25

目的

B-tree インデックススキャンにおける検索キー（ScanKey）の正規化および変換処理中に、個々のキーの状態を保持するための内部構造体です。

シグネチャまたは形状

typedef struct BTScanKeyPreproc
{
	ScanKey		inkey;      /* 処理対象の検索キー */
	int			inkeyi;     /* 検索キーのインデックス */
	int			arrayidx;   /* 配列キー（ArrayKey）のインデックス */
} BTScanKeyPreproc;

入力と出力

本構造体は関数内部での状態保持に使用されます。入力として ScanKey を受け取り、正規化プロセスを経て、最終的にインデックススキャンに適した形式へ変換されます。

前提条件と不変条件

  • nbtpreprocesskeys.c 内のキー前処理ロジックにおいて、インデックスの属性ごとに最大戦略数（BTMaxStrategyNumber）分確保される配列の一部として利用されます。

アルゴリズムまたは振る舞い

  • _bt_preprocess_keys において、スキャンキーの正規化、冗長なキーの排除、およびスキップスキャン用キーの生成を行う際の作業用コンテキストとして機能します。
  • インデックスのソート順や演算子の特性に基づき、inkey が指す条件を調整する際の参照点となります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtpreprocesskeys.c 内の _bt_preprocess_keys 関数が、ローカル変数 xform[BTMaxStrategyNumber] として本構造体の配列を定義・利用しています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ScanKey: 検索条件を保持する標準的なデータ構造。
  • BTMaxStrategyNumber: 本構造体を配列として確保する際のサイズ定義。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、本構造体を利用する処理において、矛盾する検索条件（例: x = 1 AND x > 2）が検出された場合、スキャンが早期終了される可能性があります。

保守担当者向けメモ

  • 本構造体は nbtpreprocesskeys.c 内の局所的な処理に特化しています。
  • 配列キー（SK_SEARCHARRAY）を扱う際、arrayidx が適切に管理されているかを確認してください。
  • スキャンキーの正規化ロジックを変更する際は、xform 配列内の各 BTScanKeyPreproc 要素がどのように更新されるかを追跡する必要があります。
