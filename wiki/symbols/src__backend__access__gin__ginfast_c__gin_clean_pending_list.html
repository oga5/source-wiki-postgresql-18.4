gin_clean_pending_list

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginfast.c:1030

目的

ユーザーが明示的にGINインデックスのペンディングリストをクリーンアップし、蓄積されたデータをメインのインデックス構造へ転送するために提供されるSQL呼び出し可能な関数です。

シグネチャまたは形状

Datum gin_clean_pending_list(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を通じて対象インデックスの Oid を受け取ります。
  • **出力**: 削除されたペンディングページの総数を int64 型で返します。

前提条件と不変条件

  • データベースのリカバリ中ではないこと。
  • 指定されたオブジェクトがGINインデックスであること。
  • 呼び出し元が対象インデックスの所有者であること。
  • 対象インデックスが有効（indisvalid）であること。
  • 他セッションの一時インデックスではないこと。

アルゴリズムまたは振る舞い

1. 指定されたインデックスを RowExclusiveLock でオープンします。 2. 前提条件のチェックを行い、違反がある場合はエラーを発生させます。 3. インデックスが有効な場合、initGinState で状態を初期化し、ginInsertCleanup を呼び出してペンディングリストのデータをメインインデックスへ転送・統合します。 4. インデックスが有効でない場合は DEBUG1 レベルのメッセージを出力し、処理をスキップします。 5. インデックスをクローズし、削除されたページ数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由でユーザーから呼び出されます。
  • **呼び出し先**: index_open, index_close, initGinState, ginInsertCleanup などを呼び出します。

関連データ構造

  • Relation: インデックスのメタデータ管理に使用。
  • IndexBulkDeleteResult: 削除されたページ数などの統計情報を保持。
  • GinState: GINインデックスの操作状態を管理。

失敗モードと境界条件

  • **エラー**: リカバリ中の実行、対象がGINインデックスでない場合、所有者権限がない場合、他セッションの一時インデックスへのアクセスはエラーとなります。
  • **境界条件**: indisvalid が偽のインデックスに対しては、エラーではなく DEBUG1 メッセージを出力して処理を終了します。

保守担当者向けメモ

  • 本関数は ginInsertCleanup のラッパーとして機能します。
  • ginInsertCleanup の引数（force 等）はすべて true で渡されており、強制的なクリーンアップを意図しています。
  • 統計情報の stats.pages_deleted が戻り値として使用されます。
