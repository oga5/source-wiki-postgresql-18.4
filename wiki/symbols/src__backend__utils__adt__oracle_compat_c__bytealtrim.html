bytealtrim

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:664

目的

bytea 型のデータから、指定されたバイトセットに含まれる文字を先頭（左側）から除去する。

シグネチャまたは形状

Datum bytealtrim(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BYTEA_PP(0): 対象となる bytea 型データ。
  • PG_GETARG_BYTEA_PP(1): 除去対象のバイトを含む bytea 型データ。
  • **出力**:
  • PG_RETURN_BYTEA_P(ret): 除去処理後の新しい bytea 型データ。

前提条件と不変条件

  • 入力引数は PG_FUNCTION_ARGS マクロを通じて渡される PostgreSQL の Datum 形式であること。
  • 内部で dobyteatrim を呼び出すため、そのメモリ管理およびデータ構造の要件に従う。

アルゴリズムまたは振る舞い

1. PG_GETARG_BYTEA_PP を使用して、入力文字列と除去対象セットを取得する。 2. dobyteatrim 関数を呼び出し、左側トリミング（left=true, right=false）を実行する。 3. 処理結果を bytea 型として呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PostgreSQL の関数マネージャ経由）。
  • **呼び出し先**: dobyteatrim（src/backend/utils/adt/oracle_compat.c 内で定義）。

関連データ構造

  • bytea: PostgreSQL の可変長バイナリデータ型。

失敗モードと境界条件

  • 入力データが NULL の場合、PG_GETARG_BYTEA_PP の挙動に従いエラーまたは NULL を返す。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は dobyteatrim のラッパーである。トリミングのロジックを変更する場合は dobyteatrim を参照すること。
  • bytea 型の操作であるため、文字エンコーディングの考慮は不要（バイト単位の比較が行われる）。
