pg_atomic_uint64

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/arch-x86.h:72

目的

64ビットプラットフォームにおいて、アトミック操作が可能な64ビット符号なし整数値を保持するためのデータ構造を提供します。

シグネチャまたは形状

typedef struct pg_atomic_uint64
{
    volatile uint64 value;
} pg_atomic_uint64;

入力と出力

  • 入力: 構造体内の value フィールドに対して、アトミック操作関数（pg_atomic_compare_exchange_u64_impl 等）が直接アクセスします。
  • 出力: なし（構造体定義のため）。

前提条件と不変条件

  • 64ビットプラットフォーム上で使用されることを前提としています。
  • 64ビットプラットフォーム上では、アライメントが保証されます。

アルゴリズムまたは振る舞い

  • volatile 修飾子により、コンパイラによる最適化（レジスタへのキャッシュ等）を抑制し、メモリ上の値を直接読み書きすることを強制します。
  • 本構造体は、x86アーキテクチャにおけるインラインアセンブラを用いたアトミック命令（lockプレフィックス付き命令など）の対象として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/access/transam/xlog.c: insertingAt, unloggedLSN, logInsertResult, logWriteResult, logFlushResult, xlblocks 等の変数定義で使用。
  • src/backend/access/transam/xlogprefetcher.c: reset_time, prefetch, hit 等の変数定義で使用。
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • pg_atomic_uint32: 32ビット版のアトミック操作用構造体。
  • pg_atomic_flag: フラグ操作用構造体。

失敗モードと境界条件

  • 32ビット環境では、本構造体を用いた64ビットアトミック操作のインラインアセンブラ実装が提供されないため、使用できません。

保守担当者向けメモ

  • 本構造体は __x86_64__ 環境での利用を想定しています。
  • xlog.c 等で動的にメモリ確保される場合があるため、アライメント要件が変更される場合は注意が必要です。
