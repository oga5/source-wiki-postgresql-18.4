AdjustFractSeconds

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/interval.c:22

目的

浮動小数点数で表現された時間間隔の端数（frac）を、整数秒とマイクロ秒単位の端数（fsec_t）に分解し、指定された tm 構造体に加算する。

シグネチャまたは形状

static void AdjustFractSeconds(double frac, struct tm *tm, fsec_t *fsec, int scale)

入力と出力

  • frac (double): 調整対象となる時間間隔の端数。
  • tm (struct tm *): 秒単位の加算先となる構造体。
  • fsec (fsec_t *): マイクロ秒単位の端数の加算先。
  • scale (int): frac を秒単位に変換するための倍率（例: SECS_PER_DAY）。
  • 戻り値: なし（void）。

前提条件と不変条件

  • frac が 0 の場合、関数は何も行わず即座に復帰する。
  • fsec はマイクロ秒単位の精度で加算される。

アルゴリズムまたは振る舞い

1. frac に scale を乗算し、秒単位の値を算出する。 2. 算出された値を整数 sec にキャストし、tm->tm_sec に加算する。 3. frac から整数部分 sec を減算し、残りの端数を求める。 4. 残りの端数に 1,000,000 を乗算し、rint 関数で丸めた値を *fsec に加算する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/pgtypeslib/interval.c 内の複数の箇所から、SECS_PER_DAY、SECS_PER_HOUR、SECS_PER_MINUTE、または 1 を scale として呼び出されている。
  • 呼び出し先: rint (標準数学ライブラリ関数)。

関連データ構造

  • struct tm: C 標準の日時構造体。
  • fsec_t: interval.c 内で使用されるマイクロ秒単位の端数型。

失敗モードと境界条件

  • frac が 0 の場合は早期リターンする。
  • 浮動小数点演算の精度に依存するため、-ffast-math コンパイルオプションの使用は禁止されている。

保守担当者向けメモ

  • 本関数は PostgreSQL バックエンドのロジックを移植したものである。
  • struct pg_tm ではなく struct tm を使用している点に注意すること。
