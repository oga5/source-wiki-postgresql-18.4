brin_doupdate

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_pageops.c:52

目的

BRINインデックス内の既存タプルを新しい値で更新します。同一ページ内での更新を優先し、領域不足の場合は別ページへの移動を試みます。

シグネチャまたは形状

bool brin_doupdate(Relation idxrel, BlockNumber pagesPerRange,
                   BrinRevmap *revmap, BlockNumber heapBlk,
                   Buffer oldbuf, OffsetNumber oldoff,
                   const BrinTuple *origtup, Size origsz,
                   const BrinTuple *newtup, Size newsz,
                   bool samepage)

入力と出力

  • **入力**:
  • idxrel: 対象のインデックスリレーション。
  • oldbuf, oldoff: 更新対象タプルのバッファとオフセット。
  • origtup, origsz: 更新前のタプルデータとそのサイズ。
  • newtup, newsz: 更新後のタプルデータとそのサイズ。
  • samepage: 同一ページ内での更新を強制するかどうか。
  • **出力**:
  • bool: 更新が成功した場合は true、失敗（競合や領域不足）した場合は false を返します。

前提条件と不変条件

  • newsz は MAXALIGN でアラインメントされている必要があります。
  • newsz が BrinMaxItemSize を超える場合、エラーが発生します。
  • ページ操作はクリティカルセクション内で行われ、WAL記録と整合性が保証されます。

アルゴリズムまたは振る舞い

1. **事前チェック**: newsz のサイズ制限を確認し、Revmapを拡張します。 2. **バッファ確保**: samepage が false の場合、brin_getinsertbuffer を用いて挿入先バッファを確保します。 3. **整合性検証**:

  • ページが正規のBRINページであるか、オフセットが有効かを確認します。
  • brin_tuples_equal を用いて、対象タプルが更新前と一致するか（並行更新による不整合がないか）を確認します。

    • 4. **更新実行**:

  • 同一ページ内に十分な空きがある場合、PageIndexTupleOverwrite で上書きし、WALを記録します。

    • 5. **後処理**: ロックの解放および、必要に応じて確保したバッファの初期化・解放を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c (複数箇所)
  • src/backend/access/brin/brin_pageops.c (内部再帰的呼び出しの可能性あり)
  • **呼び出し先**:
  • brinRevmapExtend, brin_getinsertbuffer, brin_initialize_empty_new_buffer, brin_can_do_samepage_update, brin_tuples_equal 等。

関連データ構造

  • BrinTuple: インデックスに格納されるタプル構造。
  • BrinRevmap: ヒープブロックとインデックスの対応表。
  • Buffer, Page: PostgreSQLのバッファおよびページ管理構造。

失敗モードと境界条件

  • **サイズ超過**: newsz > BrinMaxItemSize の場合、エラー終了します。
  • **並行更新**: ページの状態変更やタプルの不一致が検出された場合、false を返して更新を中断します。
  • **領域不足**: samepage が true でかつページ内に空きがない場合、false を返します。

保守担当者向けメモ

  • ページ退避（BRIN_EVACUATE_PAGE）フラグが立っている場合、同一ページ内更新はスキップされます。
  • 失敗時にバッファを拡張していた場合は、brin_initialize_empty_new_buffer を呼び出し、FSMを適切にクリーンアップする必要があります。
