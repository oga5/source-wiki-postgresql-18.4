emit_tap_output_v

定義

  • 種別: function
  • 定義箇所: src/test/regress/pg_regress.c:340

目的

TAP (Test Anything Protocol) 形式のメッセージを標準出力または標準エラー出力へ書き出し、同時にログファイル（logfile）が存在する場合はそこへも記録する。

シグネチャまたは形状

static void emit_tap_output_v(TAPtype type, const char *fmt, va_list argp);

入力と出力

  • type (TAPtype): 出力メッセージの種類（DIAG, BAIL, NOTE, NOTE_DETAIL など）。
  • fmt: vfprintf に渡される書式指定文字列。
  • argp: 可変長引数リスト。
  • 出力: 指定されたストリーム（stdout または stderr）およびログファイルへの書き込み。

前提条件と不変条件

  • errno は、fprintf 等の副作用から保護されるため、呼び出し時の値が vfprintf 実行前に復元される。
  • NOTE_DETAIL 型の出力時、in_note フラグを使用して、複数行にわたる注釈の先頭にのみ # を付与する制御が行われる。
  • BAIL 型の場合、TAP プロトコルに従い Bail out! という文字列が別途出力される。

アルゴリズムまたは振る舞い

1. errno を保存する。 2. type に応じて出力先 fp を決定する（DIAG または BAIL なら stderr、それ以外は stdout）。 3. NOTE_END の場合は改行を出力して終了する。 4. NOTE, DIAG, BAIL、または in_note が偽の NOTE_DETAIL の場合、行頭に # を付与する。 5. vfprintf を用いて書式付き文字列を fp および logfile に出力する。 6. NOTE_DETAIL の場合は in_note を真に設定する。 7. BAIL の場合は Bail out! を出力する。 8. NOTE_DETAIL 以外の場合は改行を出力し、fflush(NULL) で全ストリームをフラッシュする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/test/regress/pg_regress.c 内の emit_tap_output (行336)
  • src/test/regress/pg_regress.c 内の bail_out (行260)
  • 呼び出し先:
  • vfprintf
  • fprintf
  • fflush
  • va_copy, va_end

関連データ構造

  • TAPtype: 出力タイプを定義する列挙型。
  • in_note: NOTE_DETAIL の状態を保持する静的変数。
  • logfile: ログ出力先を指すグローバルなファイルポインタ。

失敗モードと境界条件

  • logfile が NULL の場合、ログファイルへの書き込みはスキップされる。
  • type が NOTE_DETAIL の場合、改行の出力が抑制される。

保守担当者向けメモ

  • 本関数は va_list を受け取るため、呼び出し側で va_start を行う必要がある。
  • fmt に %m を含める場合、errno が正しく復元されるため、直前の fprintf 等の影響を受けない設計となっている。
