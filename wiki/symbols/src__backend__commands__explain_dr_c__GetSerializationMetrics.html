GetSerializationMetrics

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain_dr.c:298

目的

EXPLAIN (SERIALIZE) 実行時に収集されたシリアライズ処理の計測メトリクスを取得する。

シグネチャまたは形状

SerializeMetrics GetSerializationMetrics(DestReceiver *dest);

入力と出力

  • **入力**: DestReceiver *dest - 対象のレシーバー構造体。
  • **出力**: SerializeMetrics - 収集された計測データ（時間、バイト数等）。

前提条件と不変条件

  • dest が DestExplainSerialize 型である場合、有効なメトリクスが返される。
  • それ以外の型である場合、ゼロ初期化された SerializeMetrics が返される。

アルゴリズムまたは振る舞い

1. 引数 dest の mydest フィールドを確認する。 2. DestExplainSerialize と一致する場合、dest を SerializeDestReceiver 型にキャストし、保持されている metrics を返す。 3. 一致しない場合、SerializeMetrics 構造体を memset でゼロクリアし、INSTR_TIME_SET_ZERO で経過時間をリセットした上で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c:590 にて呼び出されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • SerializeMetrics: 計測結果を保持する構造体。
  • DestReceiver: PostgreSQL のデータ出力先を抽象化した構造体。
  • SerializeDestReceiver: DestReceiver を継承し、計測状態を保持する構造体。

失敗モードと境界条件

  • dest が NULL の場合、ポインタ参照によるクラッシュが発生する可能性がある（ソースコード上にガードなし）。
  • DestExplainSerialize 以外のレシーバーが渡された場合、エラーにはならず、安全にゼロ値が返される。

保守担当者向けメモ

  • 本関数は EXPLAIN (SERIALIZE) 以外の実行パスでも安全に呼び出せるよう設計されている。
  • SerializeMetrics 構造体の定義変更時には、本関数のゼロ初期化ロジックも併せて確認すること。
