ExecFindJunkAttributeInTlist

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execJunk.c:221

目的

ターゲットリスト（targetlist）から、指定された名前を持つ「ジャンク属性（resjunk が真である属性）」を検索し、その属性番号を返します。

シグネチャまたは形状

AttrNumber ExecFindJunkAttributeInTlist(List *targetlist, const char *attrName)

入力と出力

  • **入力**:
  • targetlist: 検索対象の TargetEntry ノードのリスト。
  • attrName: 検索するジャンク属性の名前（文字列）。
  • **出力**:
  • 成功時: 見つかったジャンク属性の属性番号（AttrNumber）。
  • 失敗時: InvalidAttrNumber。

前提条件と不変条件

  • targetlist 内の各要素は TargetEntry 構造体である必要があります。
  • ジャンク属性は TargetEntry の resjunk フラグが真であることで識別されます。

アルゴリズムまたは振る舞い

1. targetlist を先頭から順に走査します。 2. 各要素（TargetEntry）について、以下の条件をすべて満たすか確認します。

  • resjunk フラグが真であること。
  • resname が存在すること。
  • resname が引数 attrName と一致すること。

    • 3. 条件に一致する要素が見つかった場合、その resno を直ちに返します。 4. リストの最後まで一致するものが見つからなかった場合、InvalidAttrNumber を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/executor/execJunk.c
  • src/backend/executor/execMain.c
  • src/backend/executor/nodeModifyTable.c
  • **呼び出し先**:
  • lfirst (マクロ)
  • strcmp (標準ライブラリ)

関連データ構造

  • TargetEntry: ターゲットリストの各要素を構成する構造体。resjunk フラグ、resname、resno を保持します。
  • List: PostgreSQL のリスト構造。

失敗モードと境界条件

  • リストが空の場合、または指定された名前のジャンク属性が存在しない場合は InvalidAttrNumber を返します。
  • resname が NULL の TargetEntry がリスト内に存在する場合、その要素はスキップされます。

保守担当者向けメモ

  • 本関数は、プランナーによって付与された resjunk フラグに依存しています。
  • 属性番号は 1-based です。
  • 現在の実装は線形探索であるため、ターゲットリストが非常に長い場合はパフォーマンスに影響する可能性があります。
