set_ps_display_remove_suffix

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/ps_status.c:422

目的

現在表示されているプロセス名のサフィックス（補足情報）を削除し、プロセス名を直前の状態に戻す。

シグネチャまたは形状

void set_ps_display_remove_suffix(void)

入力と出力

  • 入力: なし（内部状態 ps_buffer_nosuffix_len 等を参照）
  • 出力: なし（副作用としてプロセス名表示バッファを更新）

前提条件と不変条件

  • PS_USE_NONE が定義されていない環境でのみ動作する。
  • ps_buffer_nosuffix_len が 0 より大きい場合（サフィックスが付与されている状態）にのみ削除処理が実行される。
  • 処理完了後、ps_buffer_cur_len は ps_buffer の文字列長と一致する。

アルゴリズムまたは振る舞い

1. update_ps_display_precheck() を呼び出し、プロセス名更新が許可されているか確認する。許可されていない場合は即座に終了する。 2. ps_buffer_nosuffix_len が 0 の場合は、サフィックスが存在しないと判断して終了する。 3. ps_buffer の ps_buffer_nosuffix_len 位置にヌル文字を挿入し、文字列を切り詰める。 4. ps_buffer_cur_len を ps_buffer_nosuffix_len に更新し、ps_buffer_nosuffix_len を 0 にリセットする。 5. flush_ps_display() を呼び出し、OSに対して更新されたプロセス名を反映させる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/syncrep.c:362
  • src/backend/storage/buffer/bufmgr.c:5732
  • src/backend/storage/ipc/standby.c:454
  • src/backend/storage/lmgr/lock.c:1978

関連データ構造

  • ps_buffer: プロセス名を保持するグローバルバッファ。
  • ps_buffer_cur_len: 現在のプロセス名の長さ。
  • ps_buffer_nosuffix_len: サフィックス追加前のプロセス名の長さ。

失敗モードと境界条件

  • update_ps_display_precheck が偽を返す場合、何も変更せずに終了する。
  • サフィックスが付与されていない状態（ps_buffer_nosuffix_len == 0）で呼び出された場合、何も変更せずに終了する。

保守担当者向けメモ

  • 本関数は ps_buffer を直接操作するため、マルチスレッド環境や非同期シグナルハンドラからの呼び出しには注意が必要である（PostgreSQLのバックエンドは通常シングルスレッドで動作する前提）。
  • PS_USE_NONE が定義されているプラットフォームでは、本関数は空の操作となる。
