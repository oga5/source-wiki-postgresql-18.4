pretty_format_node_dump

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/print.c:150

目的

nodeToString 等で生成されたノードのシリアライズ文字列を、インデントと改行を挿入して人間が読みやすい形式に整形する。

シグネチャまたは形状

char *pretty_format_node_dump(const char *dump)

入力と出力

  • **入力**: dump (ノードのシリアライズ文字列)
  • **出力**: 整形済みの文字列を保持する char *（呼び出し元で解放が必要）

前提条件と不変条件

  • 入力 dump はヌル終端文字列であること。
  • 内部で StringInfo を使用してメモリを動的に確保するため、呼び出し元は返却された文字列を適切に管理する必要がある。

アルゴリズムまたは振る舞い

1. StringInfoData を初期化し、論理インデントレベルと物理インデント距離を 0 に設定する。 2. 入力文字列を走査し、LINELEN (78文字) を上限として行を構築する。 3. 文字列内の特定の記号（} 等）をヒューリスティックに検出し、インデントレベルを調整する。 4. 記号の出現に応じて、現在の行をバッファにフラッシュし、必要に応じて改行とインデントを挿入する。 5. 入力文字列の終端まで処理を繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/print.c 内の関数（pprint 等の内部処理から参照）。
  • **呼び出し先**: initStringInfo, appendStringInfo（lib/stringinfo.h）。

関連データ構造

  • StringInfoData: 整形後の文字列を構築するためのバッファ構造体。

失敗モードと境界条件

  • LINELEN を超える長い文字列は、適切な位置で改行される。
  • 記号の解析はヒューリスティックであるため、複雑なノード構造ではインデントが完璧に一致しない場合がある。

保守担当者向けメモ

  • 本関数はデバッグ用途に限定されている。
  • インデントの挙動（INDENTSTOP, MAXINDENT）を変更する場合は、src/backend/nodes/print.c 内の定数定義を確認すること。
