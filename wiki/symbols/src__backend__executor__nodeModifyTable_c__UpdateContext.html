UpdateContext

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeModifyTable.c:122

目的

UPDATE 操作の実行中に必要な状態情報を保持し、パーティション間更新の追跡やインデックス更新の制御、および同時実行制御（EvalPlanQual）のためのロックモード管理を行うためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct UpdateContext
{
    bool        crossPartUpdate;    /* パーティション間更新か否か */
    TU_UpdateIndexes updateIndexes; /* 必要なインデックス更新の定義 */
    LockTupleMode lockmode;         /* EvalPlanQual 実行前のロックモード */
} UpdateContext;

入力と出力

本構造体は関数引数として渡され、ExecUpdate 関連の処理フローにおいて状態の受け渡しに使用されます。戻り値としての直接的な出力はありません。

前提条件と不変条件

  • lockmode は、EvalPlanQual（EPQ）を実行する前に、最新のタプルバージョンに対して取得すべきロックモードを保持します。
  • crossPartUpdate は、更新操作がパーティションを跨ぐ場合に真となります。

アルゴリズムまたは振る舞い

ExecUpdate 実行時に初期化され、更新処理の各フェーズ（プロローグ、本体、エピローグ）を通じて、更新対象のインデックス制御やパーティションルーティングのステータスを追跡するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeModifyTable.c 内の ExecUpdate および関連するエピローグ処理関数（ExecUpdateEpilogue など）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • TU_UpdateIndexes: 更新が必要なインデックスの集合を定義する型。
  • LockTupleMode: タプルロックのモードを定義する列挙型。
  • ModifyTableContext: ModifyTable ノード全体の実行状態を保持する上位コンテキスト。

失敗モードと境界条件

  • 構造体の初期化時に {0} でゼロクリアされることが想定されています。不適切な初期化状態で使用された場合、lockmode が不正な値となり、同時実行制御においてデッドロックや不整合を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は ExecUpdate の実行フローに密接に依存しています。更新ロジックに変更を加える際は、crossPartUpdate の判定条件や lockmode の選定基準が適切に維持されているか確認してください。
