AttachSession

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/session.c:154

目的

リーダープロセスから提供された動的共有メモリ（DSM）ハンドルを使用して、ワーカープロセスをセッション共有メモリ領域に接続し、並列クエリ実行に必要な共有状態を有効化する。

シグネチャまたは形状

void AttachSession(dsm_handle handle)

入力と出力

  • **入力**: handle (dsm_handle) - 接続対象となるDSMセグメントの識別子。
  • **出力**: なし（副作用として CurrentSession グローバル変数が更新される）。

前提条件と不変条件

  • TopMemoryContext が有効であること。
  • 指定された handle が有効なDSMセグメントを指していること。
  • CurrentSession が初期化済みであること。
  • 接続されたDSMセグメントおよびDSA領域は、バックエンドの終了または DetachSession() が呼び出されるまで保持（pin）される。

アルゴリズムまたは振る舞い

1. メモリコンテキストを TopMemoryContext に切り替える。 2. dsm_attach(handle) を呼び出し、DSMセグメントに接続する。失敗した場合はエラーを投げる。 3. shm_toc_attach を使用して、セグメント内のテーブル・オブ・コンテンツ（TOC）を読み込む。 4. TOCから SESSION_KEY_DSA を検索し、dsa_attach_in_place で動的共有メモリ領域（DSA）をアタッチする。 5. 取得したセグメントとDSA領域を CurrentSession に格納する。 6. TOCから SESSION_KEY_RECORD_TYPMOD_REGISTRY を検索し、共有レコード型修飾子レジストリをアタッチする。 7. dsm_pin_mapping および dsa_pin_mapping を実行し、バックエンド終了までマッピングを維持する。 8. 元のメモリコンテキストに復帰する。

関連データ構造

  • dsm_segment: DSMセグメント管理構造体。
  • shm_toc: 共有メモリ内のデータ配置を管理するテーブル。
  • dsa_area: 動的共有メモリ領域管理構造体。
  • CurrentSession: セッション情報を保持するグローバル変数。

失敗モードと境界条件

  • dsm_attach が NULL を返した場合、elog(ERROR) により処理が中断される。
  • shm_toc_lookup で指定されたキー（SESSION_KEY_DSA または SESSION_KEY_RECORD_TYPMOD_REGISTRY）が見つからない場合、内部的な整合性エラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は並列クエリのワーカープロセスにおいて、リーダープロセスと同一の共有メモリ空間を共有するために不可欠である。
  • SESSION_MAGIC やキー定義に変更を加える際は、リーダーとワーカー間でバイナリ互換性が保たれるよう注意すること。
