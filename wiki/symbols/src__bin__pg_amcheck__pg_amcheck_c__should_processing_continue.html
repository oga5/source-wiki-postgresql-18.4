should_processing_continue

定義

  • 種別: function
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:963

目的

データベース操作の結果（PGresult）を評価し、後続の処理を継続すべきか否かを判定する。

シグネチャまたは形状

static bool should_processing_continue(PGresult *res)

入力と出力

  • **入力**: PGresult *res（libpq から返されたクエリ実行結果）
  • **出力**: bool（true の場合処理を継続、false の場合処理を中断）

前提条件と不変条件

  • 入力される res は PQresultStatus を介して検証可能な状態であること。
  • PGRES_FATAL_ERROR の場合、PQ_DIAG_SEVERITY_NONLOCALIZED フィールドからエラーの深刻度を特定できること。

アルゴリズムまたは振る舞い

1. PQresultStatus(res) を評価する。 2. PGRES_COMMAND_OK, PGRES_TUPLES_OK, PGRES_NONFATAL_ERROR の場合は true を返す。 3. PGRES_FATAL_ERROR の場合:

  • 深刻度が取得できない、または "FATAL" か "PANIC" である場合は false を返す。
  • それ以外は true を返す。

    • 4. 上記以外のステータス（PGRES_BAD_RESPONSE, PGRES_COPY_OUT 等）は、予期しない状態として false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c:1103
  • src/bin/pg_amcheck/pg_amcheck.c:1172
  • **呼び出し先**:
  • PQresultStatus (libpq)
  • PQresultErrorField (libpq)
  • strcmp (標準ライブラリ)

関連データ構造

  • PGresult (libpq)

失敗モードと境界条件

  • PQresultErrorField が NULL を返す場合（libpq の接続喪失など）、安全側に倒して false を返す。
  • PGRES_FATAL_ERROR であっても、深刻度が "FATAL" や "PANIC" でない場合は処理を継続する。

保守担当者向けメモ

  • 本関数は pg_amcheck におけるエラーハンドリングのゲートキーパーです。新しい libpq のステータスコードが導入された場合、本関数の switch 文に適切に追加する必要があります。
