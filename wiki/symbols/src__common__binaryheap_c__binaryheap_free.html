binaryheap_free

定義

  • 種別: function
  • 定義箇所: src/common/binaryheap.c:74

目的

binaryheap_allocate によって動的に確保されたバイナリヒープ構造体のメモリを解放します。

シグネチャまたは形状

void binaryheap_free(binaryheap *heap)

入力と出力

  • **入力**: binaryheap *heap - 解放対象となるヒープ構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • 引数 heap は binaryheap_allocate によって正当に確保されたメモリ領域を指している必要があります。
  • 解放後の heap ポインタへのアクセスは未定義動作となります。

アルゴリズムまたは振る舞い

内部的に pfree(heap) を呼び出し、ヒープ構造体全体をメモリから解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/reorderbuffer.c:1525
  • src/backend/storage/buffer/bufmgr.c:3597
  • src/bin/pg_dump/pg_backup_archiver.c:4537
  • src/bin/pg_dump/pg_dump_sort.c:735
  • **呼び出し先**:
  • pfree (PostgreSQL メモリ管理関数)

関連データ構造

  • binaryheap: src/include/lib/binaryheap.h で定義されるヒープ管理構造体。

失敗モードと境界条件

  • heap が NULL の場合、pfree の実装に依存します（通常、PostgreSQL の pfree は NULL ポインタを許容しません）。

保守担当者向けメモ

  • 本関数は binaryheap_allocate と対になるライフサイクル管理関数です。ヒープの使用終了時には必ず呼び出してください。
