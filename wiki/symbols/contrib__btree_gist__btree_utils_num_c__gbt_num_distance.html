gbt_num_distance

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_num.c:315

目的

GiST インデックスにおける最近傍探索（KNN）をサポートするため、クエリ値とインデックス内の数値範囲（GBT_NUMKEY_R）との距離を算出します。

シグネチャまたは形状

float8 gbt_num_distance(const GBT_NUMKEY_R *key, const void *query, bool is_leaf, const gbtree_ninfo *tinfo, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • key: 距離計算対象となるインデックス内の範囲（下限値・上限値）。
  • query: 検索対象のクエリ値。
  • is_leaf: リーフノードかどうかを示すフラグ。
  • tinfo: データ型固有の比較関数や距離関数を保持する設定構造体。
  • flinfo: 関数呼び出し情報。
  • **出力**:
  • float8: クエリ値から範囲までの距離。範囲内にクエリ値が含まれる場合は 0.0 を返します。

前提条件と不変条件

  • tinfo->f_dist が NULL でないこと。NULL の場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. tinfo->f_dist が定義されているか確認し、未定義であればエラーをスローします。 2. クエリ値が範囲の下限（key->lower）より小さい場合、f_dist を用いて下限との距離を計算します。 3. クエリ値が範囲の上限（key->upper）より大きい場合、f_dist を用いて上限との距離を計算します。 4. クエリ値が範囲内にある場合、距離は 0.0 となります。

呼び出し元と呼び出し先

  • **呼び出し元**: btree_cash.c, btree_date.c, btree_float4.c, btree_float8.c, btree_int2.c, btree_int4.c, btree_int8.c, btree_interval.c, btree_oid.c, btree_time.c, btree_ts.c の各ファイルから呼び出されています。
  • **呼び出し先**: tinfo 構造体に含まれる f_dist, f_le, f_ge 関数を呼び出します。

関連データ構造

  • GBT_NUMKEY_R: 数値範囲を保持する構造体。
  • gbtree_ninfo: データ型固有の演算関数を保持する構造体。

失敗モードと境界条件

  • tinfo->f_dist が NULL の場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は数値型全般で共通利用されるため、新しいデータ型を追加する際は gbtree_ninfo に適切な距離計算関数を実装して渡す必要があります。
