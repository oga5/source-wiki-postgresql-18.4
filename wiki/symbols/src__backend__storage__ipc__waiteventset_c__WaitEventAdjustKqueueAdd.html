WaitEventAdjustKqueueAdd

定義

  • 種別: static inline function
  • 定義箇所: src/backend/storage/ipc/waiteventset.c:837

目的

BSD系OSの kqueue インターフェースで使用される kevent 構造体を、PostgreSQL の WaitEvent 情報に基づいて初期化・設定する。

シグネチャまたは形状

static inline void
WaitEventAdjustKqueueAdd(struct kevent *k_ev, int filter, int action,
                         WaitEvent *event)

入力と出力

  • **入力**:
  • k_ev: 設定対象の kevent 構造体へのポインタ。
  • filter: kqueue のフィルタタイプ（EVFILT_READ, EVFILT_WRITE 等）。
  • action: kqueue のフラグ（EV_ADD, EV_DELETE 等）。
  • event: 監視対象の WaitEvent 構造体へのポインタ。
  • **出力**: なし（k_ev を直接書き換える）。

前提条件と不変条件

  • event が有効な WaitEvent 構造体を指していること。
  • k_ev がメモリ上に確保された有効な kevent 構造体であること。

アルゴリズムまたは振る舞い

1. event->fd を k_ev->ident に代入する。 2. 引数 filter と action をそれぞれ k_ev->filter および k_ev->flags に代入する。 3. k_ev->fflags および k_ev->data を 0 に初期化する。 4. AccessWaitEvent(k_ev) マクロを使用して、k_ev のユーザーデータ領域に event ポインタを格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/waiteventset.c 内の関数群（EVFILT_READ や EVFILT_WRITE を用いて k_ev を設定する箇所）。
  • **呼び出し先**: AccessWaitEvent マクロ。

関連データ構造

  • struct kevent: BSD kqueue システムコールのイベント構造体。
  • WaitEvent: PostgreSQL の待機イベント管理用構造体。

失敗モードと境界条件

  • 本関数自体はメモリ操作のみを行うため、引数が NULL でない限り失敗しない。
  • k_ev が指すメモリ領域が不足している場合、呼び出し元でバッファオーバーランが発生する可能性がある。

保守担当者向けメモ

  • 本関数は kqueue 実装専用のヘルパーである。epoll 等の他プラットフォーム用実装と混同しないこと。
  • AccessWaitEvent マクロの定義が kevent 構造体のどのフィールドをユーザーデータとして利用しているかを確認すること。
