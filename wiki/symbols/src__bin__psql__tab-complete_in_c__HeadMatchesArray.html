HeadMatchesArray

定義

  • 種別: function
  • 定義箇所: src/bin/psql/tab-complete.in.c:1675

目的

入力された単語列の末尾から指定された引数リスト（パターン）と照合し、一致するかどうかを判定する。

シグネチャまたは形状

static bool HeadMatchesArray(bool case_sensitive,
                             int previous_words_count, char **previous_words,
                             int narg, const char *const *args)

入力と出力

  • **入力**:
  • case_sensitive: 大文字小文字を区別するかどうかのフラグ。
  • previous_words_count: 入力済み単語の総数。
  • previous_words: 入力済み単語の配列。
  • narg: 照合するパターン数。
  • args: 照合対象となるパターン文字列の配列。
  • **出力**:
  • bool: パターンがすべて一致した場合は true、それ以外は false。

前提条件と不変条件

  • previous_words_count が narg 未満の場合、常に false を返す。

アルゴリズムまたは振る舞い

1. 入力単語数がパターン数に満たない場合は即座に false を返す。 2. パターン配列 args の各要素に対し、入力単語配列 previous_words の末尾から逆順に（previous_words_count - argno - 1）比較を行う。 3. word_matches 関数を用いて各単語の照合を行い、一つでも不一致があれば false を返す。 4. すべてのパターンが一致した場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/tab-complete.in.c 内の各箇所（行1676, 1716, 1758, 1766, 2013, 2019）。
  • **呼び出し先**: word_matches（内部で呼び出される）。

関連データ構造

  • previous_words: char ** 型の単語リスト。
  • args: const char *const * 型のパターンリスト。

失敗モードと境界条件

  • previous_words_count < narg: 照合に必要な単語数が不足している場合は失敗（false）とみなす。
  • word_matches の判定結果に依存し、大文字小文字の区別設定が一致判定に影響する。

保守担当者向けメモ

  • 本関数は gen_tabcomplete.pl によって生成されるコードの一部として利用される可能性がある。
