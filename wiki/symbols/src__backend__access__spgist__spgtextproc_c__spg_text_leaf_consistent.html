spg_text_leaf_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgtextproc.c:573

目的

SP-GiST 索引のリーフノードにおいて、再構築された文字列とスキャンキーを比較し、検索条件を満たすかどうかを判定する。

シグネチャまたは形状

Datum spg_text_leaf_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgLeafConsistentIn 構造体を受け取る。これには現在のレベル、リーフタプルのデータ、再構築された接頭辞、およびスキャンキーが含まれる。
  • **出力**: PG_RETURN_BOOL(res) により、検索条件に一致した場合は true、そうでない場合は false を返す。また、out->leafValue に完全な文字列を格納し、out->recheck を false に設定する。

前提条件と不変条件

  • in->reconstructedValue が存在する場合、そのサイズは level と一致していること。
  • 比較対象の文字列は、level（接頭辞）と leafValue（リーフデータ）を連結することで復元可能であること。

アルゴリズムまたは振る舞い

1. **文字列の復元**: level と leafValue を連結し、完全な文字列 fullValue を生成する。 2. **条件判定**: すべてのスキャンキー（in->nkeys）に対して以下の処理を繰り返す。

  • RTPrefixStrategyNumber の場合: text_starts_with を使用して接頭辞一致を確認する。
  • ロケール依存の比較が必要な場合: varstr_cmp を使用して照合順序に基づいた比較を行う。
  • それ以外の場合: memcmp を使用してバイト単位の比較を行う。

    • 3. **戦略の適用**: 比較結果（r）に基づき、B-Tree 演算子（<, <=, =, >=, >）に従って論理値を判定する。 4. **早期終了**: いずれかの条件で res が false になった場合、残りの条件判定をスキップする。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiST のインデックススキャン処理（具体的な呼び出し元関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: text_starts_with, varstr_cmp, memcmp, palloc 等の PostgreSQL 内部関数。

関連データ構造

  • spgLeafConsistentIn: 検索入力パラメータを保持。
  • spgLeafConsistentOut: 検索結果および復元された値を保持。

失敗モードと境界条件

  • 未知の strategy が渡された場合、elog(ERROR) によりエラーを発生させる。
  • level が 0 かつ leafValue が空の場合の処理分岐を含む。
  • 照合順序が有効な場合、pg_verifymbstr によりエンコーディングの整合性が検証される。

保守担当者向けメモ

  • 本関数は「完全一致」を前提としており、out->recheck は常に false に設定される。
  • SPG_IS_COLLATION_AWARE_STRATEGY マクロにより、C ロケールとそれ以外で比較ロジックが分岐するため、ロケール変更時の挙動に注意すること。
