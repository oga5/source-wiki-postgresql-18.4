ReceiveCopyBegin

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfromparse.c:169

目的

COPY FROM コマンドの実行開始時に、フロントエンド（クライアント）に対してデータ転送の準備が整ったことを通知し、通信プロトコルの初期化を行う。

シグネチャまたは形状

void ReceiveCopyBegin(CopyFromState cstate)

入力と出力

  • **入力**: CopyFromState cstate（COPY 実行状態を保持する構造体）
  • **出力**: なし（副作用としてフロントエンドへのメッセージ送信および cstate の更新を行う）

前提条件と不変条件

  • cstate が有効な CopyFromState 構造体を指していること。
  • cstate->attnumlist が適切に設定されていること。
  • フロントエンドとの通信チャネルが確立されていること。

アルゴリズムまたは振る舞い

1. PqMsg_CopyInResponse メッセージを構築する。 2. cstate->opts.binary の値に基づき、全体フォーマット（テキストかバイナリか）を決定し送信する。 3. カラム数（natts）を送信し、続いて各カラムのフォーマット情報を送信する。 4. メッセージを送信（pq_endmessage）する。 5. cstate->copy_src を COPY_FRONTEND に設定する。 6. cstate->fe_msgbuf を初期化する。 7. pq_flush() を呼び出し、フロントエンドがデータ送信を開始できる状態であることを保証する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c:1848
  • **呼び出し先**: pq_beginmessage, pq_sendbyte, pq_sendint16, pq_endmessage, makeStringInfo, pq_flush（すべて libpq 関連関数）

関連データ構造

  • CopyFromState: COPY FROM の実行状態を管理する構造体（commands/copyfrom_internal.h で定義）。
  • StringInfoData: メッセージ構築用バッファ。

失敗モードと境界条件

  • メッセージ送信時にネットワークエラーが発生した場合、libpq 側のエラー処理により中断される可能性がある。
  • cstate が NULL の場合、セグメンテーション違反が発生する。

保守担当者向けメモ

  • 本関数は COPY プロトコルのハンドシェイクの要である。プロトコル仕様（特にカラムフォーマットの送信順序）を変更する場合は、フロントエンド側の実装との整合性に注意すること。
  • pq_flush() は必須であり、これを除去するとクライアント側でデッドロックが発生する可能性がある。
