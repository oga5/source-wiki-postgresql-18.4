shell_send_data

定義

  • 種別: function
  • 定義箇所: contrib/basebackup_to_shell/basebackup_to_shell.c:279

目的

バックアップストリームのバッファデータを、実行中の外部シェルコマンドのパイプへ書き込む。

シグネチャまたは形状

static void shell_send_data(bbsink_shell *sink, size_t len)

入力と出力

  • **入力**:
  • sink: 外部コマンドの状態を保持する bbsink_shell 構造体へのポインタ。
  • len: 書き込むデータのバイト数。
  • **出力**: なし（成功時）。エラー発生時は ereport(ERROR, ...) によりバックアップ処理を中断する。

前提条件と不変条件

  • sink->current_command が NULL でないこと。
  • sink->pipe が有効なパイプストリームであること。
  • sink->base.bbs_buffer に書き込むべきデータが格納されていること。

アルゴリズムまたは振る舞い

1. sink->base.bbs_buffer から len バイトのデータを sink->pipe へ fwrite を用いて書き込む。 2. 書き込みに失敗した場合、または ferror が発生した場合はエラーを報告する。 3. errno が EPIPE（パイプの切断）である場合、shell_finish_command を呼び出して終了処理を試みた後、エラーを投げる。 4. それ以外の書き込みエラー時は、errcode_for_file_access() を用いて詳細なエラーメッセージと共にバックアップ処理を停止する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • bbsink_shell_archive_contents (contrib/basebackup_to_shell/basebackup_to_shell.c:326)
  • bbsink_shell_manifest_contents (contrib/basebackup_to_shell/basebackup_to_shell.c:362)
  • **呼び出し先**:
  • shell_finish_command (エラー発生時)

関連データ構造

  • bbsink_shell: 外部コマンド実行状態（パイプ、現在のコマンド文字列）を保持する構造体。

失敗モードと境界条件

  • **EPIPE**: 外部コマンドが予期せず終了した場合、shell_finish_command を経由してエラーを報告する。
  • **書き込み失敗**: fwrite が指定バイト数を書き込めない、または ferror がセットされた場合、バックアップ全体がエラーとして扱われる。

保守担当者向けメモ

  • 本関数は bbsink フレームワークのデータ転送フェーズで呼び出される。
  • bbs_buffer の内容は bbsink フレームワークによって管理されているため、本関数内でのバッファ操作は読み取りのみに限定すること。
