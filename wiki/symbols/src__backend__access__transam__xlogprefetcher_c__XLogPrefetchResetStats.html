XLogPrefetchResetStats

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogprefetcher.c:302

目的

リカバリ時の先読み統計情報（pg_stat_recovery_prefetch 用）をリセットし、各カウンタをゼロに初期化するとともに、リセット時刻を現在時刻で更新します。

シグネチャまたは形状

void XLogPrefetchResetStats(void)

入力と出力

  • 入力: なし
  • 出力: なし（共有メモリ上の統計構造体 SharedStats を直接更新）

前提条件と不変条件

  • SharedStats が初期化済みであること。
  • アトミック操作（pg_atomic_write_u64）により、統計情報の更新はスレッドセーフに行われます。

アルゴリズムまたは振る舞い

1. GetCurrentTimestamp() を呼び出し、SharedStats->reset_time を現在時刻で更新します。 2. 以下の統計カウンタをすべて 0 にリセットします。

  • prefetch: 発行された先読みリクエスト数
  • hit: 先読みヒット数
  • skip_init: 初期化スキップ数
  • skip_new: 新規ブロックスキップ数
  • skip_fpw: フルページ書き込みスキップ数
  • skip_rep: リプレイ済みスキップ数

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/adt/pgstatfuncs.c: 統計情報リセット関数から呼び出されます。
  • 呼び出し先:
  • GetCurrentTimestamp() (utils/timestamp.h)
  • pg_atomic_write_u64 (port/atomics.h)

関連データ構造

  • XLogPrefetchStats: 統計情報を保持する共有メモリ上の構造体。

失敗モードと境界条件

  • 共有メモリが初期化されていない状態で呼び出された場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は pg_stat_recovery_prefetch ビューの統計をクリアするために使用されます。
  • カウンタの追加や変更を行った場合は、本関数内のリセット対象にも忘れずに追加してください。
