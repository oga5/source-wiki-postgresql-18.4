bbsink_state

定義

  • 種別: struct
  • 定義箇所: src/include/backup/basebackup_sink.h:66

目的

ベースバックアップ処理全体を通じて、進捗状況やメタデータをバックアップシンク（bbsink）チェーン内の全コンポーネント間で共有するために使用される状態管理構造体です。

シグネチャまたは形状

typedef struct bbsink_state
{
	List	   *tablespaces;
	int			tablespace_num;
	uint64		bytes_done;
	uint64		bytes_total;
	bool		bytes_total_is_valid;
	XLogRecPtr	startptr;
	TimeLineID	starttli;
} bbsink_state;

入力と出力

  • **入力**: バックアップの開始時に初期化され、バックアップの進行に伴い bytes_done や tablespace_num が更新されます。
  • **出力**: バックアップの進捗監視や、最終的なバックアップ情報の記録に使用されます。

前提条件と不変条件

  • bbsink_begin_backup を呼び出す前に、呼び出し元が bbsink_state を初期化しておく必要があります。
  • シンクチェーン内の各シンクは、この共有オブジェクトを通じてバックアップの全体像を参照します。

アルゴリズムまたは振る舞い

  • バックアップの進捗（転送済みバイト数や現在のテーブルスペース）を追跡します。
  • basebackup_progress.c 等のモジュールにより、現在のテーブルスペースの更新や転送バイト数の加算が行われます。
  • 複数のシンクがこの構造体を介してバックアップのメタデータ（開始位置、タイムライン等）を参照します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/backup/basebackup.c
  • src/backend/backup/basebackup_copy.c
  • src/backend/backup/basebackup_progress.c
  • **呼び出し先**: 構造体であるため、直接的な関数呼び出し先はありません。各モジュールがこの構造体のフィールドを直接読み書きします。

関連データ構造

  • bbsink: この状態オブジェクトを保持し、チェーンを構成する基本構造体。
  • List: テーブルスペース情報を保持するために使用されます。

失敗モードと境界条件

  • bytes_total_is_valid が false の場合、bytes_total の値は信頼できません。
  • 構造体の初期化が不完全な状態で bbsink チェーンに渡された場合、バックアップ処理中に不正な参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は複数のシンク間で共有されるため、フィールドの更新タイミングとスレッドセーフティ（該当する場合）には注意してください。
  • basebackup_progress.c での進捗更新処理が、この構造体のフィールドと密接に結合しています。変更時は影響範囲を確認してください。
