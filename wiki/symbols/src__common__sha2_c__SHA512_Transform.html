SHA512_Transform

定義

  • 種別: function
  • 定義箇所: src/common/sha2.c:643

目的

SHA-512 アルゴリズムにおけるメッセージブロック単位の圧縮処理を実行し、ハッシュコンテキストの中間状態を更新します。

シグネチャまたは形状

static void SHA512_Transform(pg_sha512_ctx *context, const uint8 *data)

入力と出力

  • context: 更新対象の pg_sha512_ctx 構造体へのポインタ。現在のハッシュ状態（state）を保持し、計算結果で更新されます。
  • data: 処理対象となる 128 バイトのメッセージブロック。
  • 出力: なし（context->state を直接更新）。

前提条件と不変条件

  • context は初期化済みであること。
  • data は 128 バイトのデータブロックを指していること。
  • 内部計算において uint64 型のレジスタを使用し、SHA-512 の標準的なラウンド関数（ROUND512_0_TO_15 および ROUND512）を用いて状態を遷移させます。

アルゴリズムまたは振る舞い

1. context->state から 8 つの 64 ビットレジスタ（a〜h）を初期化します。 2. 最初の 16 ラウンド（j < 16）を実行し、入力データを処理します。 3. 残りのラウンド（j < 80）を実行し、ハッシュ変換を完了させます。 4. 計算されたレジスタ値を context->state に加算して中間状態を更新します。 5. セキュリティ上の理由から、計算に使用したローカル変数をゼロクリアします。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/sha2.c 内の pg_sha512_update、SHA512_Last 等の関数から、バッファがブロック長に達した際や最終化処理時に呼び出されます。
  • 呼び出し先: ROUND512_0_TO_15 および ROUND512 マクロを実行します。

関連データ構造

  • pg_sha512_ctx: SHA-512 の計算状態（state）とバッファを保持する構造体。sha2_int.h で定義されています。

失敗モードと境界条件

  • 入力データが 128 バイト未満の場合の挙動については、本関数単体では制御されません（呼び出し元でパディング処理が行われる前提です）。
  • メモリ破壊や不正なポインタ参照に対する保護は呼び出し元に依存します。

保守担当者向けメモ

  • 本関数は Aaron D. Gifford 氏の実装に基づいています。
  • パフォーマンス最適化のため、コンパイル時に SHA2_UNROLL_TRANSFORM を定義することでループのアンロールが可能です。
  • エンディアン変換が必要な環境では、REVERSE64 マクロ等の適切な処理が呼び出し元またはコンテキスト内で適用されていることを確認してください。
