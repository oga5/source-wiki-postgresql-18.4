HeapTupleHeaderGetRawXmin

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/access/htup_details.h:322

目的

ヒープタプルヘッダーから、タプルを挿入したトランザクションID（Xmin）を未加工の状態で取得する。

シグネチャまたは形状

TransactionId HeapTupleHeaderGetRawXmin(const HeapTupleHeaderData *tup)

入力と出力

  • **入力**: tup - 読み取り対象の HeapTupleHeaderData 構造体へのポインタ。
  • **出力**: TransactionId 型のトランザクションID。

前提条件と不変条件

  • tup は有効な HeapTupleHeaderData 構造体を指している必要がある。
  • 本関数は、タプルヘッダー内の t_choice.t_heap.t_xmin フィールドに直接アクセスするため、構造体のメモリレイアウトが整合していることが前提となる。

アルゴリズムまたは振る舞い

タプルヘッダー構造体内の共用体 t_choice を経由して、t_heap 構造体内の t_xmin フィールドの値を直接返却する。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • contrib/pageinspect/heapfuncs.c
  • src/backend/access/common/heaptuple.c
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/heapam_visibility.c
  • **呼び出し先**:
  • 呼び出し先に関する情報は提供されていない。

関連データ構造

  • HeapTupleHeaderData: タプルのヘッダー構造。
  • TransactionId: トランザクションIDを保持する型。

失敗モードと境界条件

  • tup が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • 本関数は値を直接読み取るのみであり、トランザクションIDの有効性チェックや可視性判定は行わない。

保守担当者向けメモ

  • 本関数はインライン化されており、パフォーマンスが重視される可視性判定処理（heapam_visibility.c 等）で頻繁に使用される。
  • 構造体のレイアウト変更時には、t_choice.t_heap へのパスが維持されているか確認すること。
