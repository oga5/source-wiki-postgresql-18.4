pgp_armor_headers_state

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/pgp-pgsql.c:906

目的

PGP Armor形式の出力時に付与するカスタムヘッダー（キーと値のペア）を保持するための状態管理構造体です。

シグネチャまたは形状

typedef struct {
    int      nheaders;
    char   **keys;
    char   **values;
} pgp_armor_headers_state;

入力と出力

  • **入力**: nheaders（ヘッダー数）、keys（ヘッダー名の配列）、values（ヘッダー値の配列）。
  • **出力**: なし（構造体としてのデータ保持）。

前提条件と不変条件

  • keys と values は、nheaders の数に対応する文字列配列として管理されることが想定されています。
  • PostgreSQLのメモリコンテキスト内で palloc を使用して確保される必要があります。

アルゴリズムまたは振る舞い

  • PGP Armor変換処理において、ユーザーが指定した追加ヘッダー情報を一時的に保持し、変換処理中に参照可能な状態を提供します。
  • funcctx->user_fctx を介して、PostgreSQLの関数呼び出しコンテキスト間で状態を維持するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pgsql.c 内の関数。
  • palloc を用いてインスタンスが生成されます。
  • funcctx->user_fctx に格納され、後続の処理で参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • funcctx (PostgreSQLの関数実行コンテキスト)

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • nheaders が負の値や不適切な値である場合、配列アクセス時に境界外参照が発生するリスクがあります（ソースコード上のバリデーションに依存）。

保守担当者向けメモ

  • 本構造体は pgp-pgsql.c 内の特定の関数コンテキスト内でのみ使用されています。
  • メモリ管理はPostgreSQLのメモリコンテキストに依存しているため、手動での pfree を行う際は、関連するすべてのポインタが適切に解放されるよう注意してください。
