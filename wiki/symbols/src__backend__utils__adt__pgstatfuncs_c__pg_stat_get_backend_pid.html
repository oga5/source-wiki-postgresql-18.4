pg_stat_get_backend_pid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pgstatfuncs.c:696

目的

指定されたプロセス番号に対応するバックエンドプロセスのPID（プロセスID）を取得し、SQLインターフェースへ返却します。

シグネチャまたは形状

Datum pg_stat_get_backend_pid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT32(0) を介して、対象となるバックエンドのプロセス番号（int32）を受け取ります。
  • **出力**: 該当するバックエンドが存在する場合はそのPID（int32）を返します。存在しない場合は NULL を返します。

前提条件と不変条件

  • 統計情報システムが初期化され、アクセス可能である必要があります。
  • 渡されるプロセス番号は、現在アクティブなバックエンドのリスト内でのインデックスとして解釈されます。

アルゴリズムまたは振る舞い

1. 引数として受け取った procNumber を使用して、pgstat_get_beentry_by_proc_number を呼び出します。 2. 戻り値が NULL の場合、対象のバックエンドが見つからなかったと判断し、PG_RETURN_NULL() を実行します。 3. 戻り値が有効な PgBackendStatus 構造体へのポインタである場合、その構造体内の st_procpid フィールドを抽出し、PG_RETURN_INT32 を介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジンから直接呼び出されることを想定しています。
  • **呼び出し先**:
  • pgstat_get_beentry_by_proc_number: バックエンド状態エントリの取得に使用。
  • PG_GETARG_INT32, PG_RETURN_NULL, PG_RETURN_INT32: PostgreSQLの関数呼び出し規約マクロ。

関連データ構造

  • PgBackendStatus: バックエンドの稼働状況を保持する構造体。st_procpid フィールドにPIDが格納されています。

失敗モードと境界条件

  • 指定された procNumber に対応するバックエンドが存在しない場合、関数はエラーを発生させず NULL を返します。

保守担当者向けメモ

  • 本関数は pgstat システムの内部構造に依存しています。PgBackendStatus 構造体の定義や pgstat_get_beentry_by_proc_number の実装が変更された場合、本関数の動作も影響を受ける可能性があります。
