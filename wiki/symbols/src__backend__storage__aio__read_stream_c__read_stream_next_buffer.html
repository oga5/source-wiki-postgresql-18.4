read_stream_next_buffer

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/read_stream.c:789

目的

ReadStream から次のバッファを取得します。必要に応じて I/O 待機や先読み処理をトリガーし、効率的なデータ読み込みを実現します。

シグネチャまたは形状

Buffer read_stream_next_buffer(ReadStream *stream, void **per_buffer_data)

入力と出力

  • **入力**:
  • stream: 操作対象の ReadStream 構造体へのポインタ。
  • per_buffer_data: バッファに関連付けられたデータを格納するためのポインタ（不要な場合は NULL）。
  • **出力**:
  • 取得した Buffer。ストリームが終了している場合は InvalidBuffer を返します。

前提条件と不変条件

  • stream は read_stream_begin_relation 等で適切に初期化されている必要があります。
  • fast_path が有効な場合、pinned_buffers == 1 かつ distance == 1 であることが前提となります。
  • per_buffer_data が指定されている場合、呼び出し元は返されたバッファの有効期間中のみそのデータにアクセス可能です。

アルゴリズムまたは振る舞い

1. **Fast Path**: キャッシュヒット率が高い場合、キュー管理をスキップして直接バッファを返却します。 2. **初期化チェック**: バッファがピン留めされていない場合、read_stream_look_ahead を呼び出して先読みを開始します。 3. **バッファ取得**: キューの先頭からバッファを取得し、必要に応じて get_per_buffer_data で付随データを抽出します。 4. **I/O 待機**: 取得したバッファに対して I/O が進行中の場合、WaitReadBuffers を呼び出して完了を待ちます。完了後、先読み距離（distance）を指数関数的に増加させます。 5. **クリーンアップ**: 使用済みバッファのキューエントリを InvalidBuffer でリセットし、メモリ保護のために古いデータを無効化します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c, contrib/pg_prewarm/autoprewarm.c, contrib/pg_prewarm/pg_prewarm.c, contrib/pg_visibility/pg_visibility.c, src/backend/access/gist/gistvacuum.c, src/backend/access/heap/heapam.c, src/backend/access/heap/heapam_handler.c など。
  • **呼び出し先**: read_stream_get_block, StartReadBuffer, read_stream_look_ahead, get_per_buffer_data, WaitReadBuffers。

関連データ構造

  • ReadStream: ストリームの状態を保持するメイン構造体。
  • Buffer: PostgreSQL のバッファ識別子。

失敗モードと境界条件

  • **ストリーム終了**: 次のブロックが存在しない場合、InvalidBuffer を返します。
  • **I/O 待機**: I/O が完了していないバッファに到達した場合、同期的に待機が発生します。
  • **Fast Path 無効化**: Fast Path の前提条件が崩れた場合、自動的に通常のキュー管理パスへフォールバックします。

保守担当者向けメモ

  • fast_path は I/O が発生しないケースに最適化されています。このパスを修正する際は、Assert 文で定義された不変条件を厳守してください。
  • per_buffer_data のメモリ管理は、CLOBBER_FREED_MEMORY や USE_VALGRIND が定義されている場合にデバッグ支援機能が有効になります。
