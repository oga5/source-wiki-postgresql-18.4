usage

定義

  • 種別: function
  • 定義箇所: src/test/modules/oauth_validator/oauth_hook_client.c:32

目的

oauth_hook_client プログラムのコマンドライン引数および使用方法を標準出力に表示する。

シグネチャまたは形状

static void usage(char *argv[])

入力と出力

  • **入力**: argv（プログラム名を含むコマンドライン引数の配列）。
  • **出力**: なし（標準出力へのメッセージ出力のみ）。

前提条件と不変条件

  • argv[0] にプログラム名が格納されていること。

アルゴリズムまたは振る舞い

1. プログラム名（argv[0]）を含む使用方法の概要を表示する。 2. サポートされている以下のフラグの一覧を表示する。

  • --help / -h
  • --expected-scope
  • --expected-uri
  • --misbehave (指定可能なモード: no-hook, fail-async, no-token, no-socket)
  • --no-hook
  • --hang-forever
  • --token
  • --stress-async

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内で明示的な呼び出し箇所は提供されていません。
  • **呼び出し先**: 標準ライブラリの printf を呼び出します。

関連データ構造

  • なし。

失敗モードと境界条件

  • argv[0] が NULL または不正なポインタである場合、printf 実行時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • ツールによる find_callers の結果には、他のソースファイルにおける「usage」という単語の言及が含まれていますが、本関数 usage の呼び出し元を示す直接的な証拠はありません。
  • 新しいコマンドライン引数を追加した際は、本関数にも対応する説明文を追加してください。
