pq_beginmessage_reuse

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:108

目的

既存の StringInfo バッファを再利用して、新しいフロントエンド・バックエンド間通信メッセージの構築を開始する。

シグネチャまたは形状

void pq_beginmessage_reuse(StringInfo buf, char msgtype)

入力と出力

  • buf: 再利用する StringInfo 構造体へのポインタ。
  • msgtype: メッセージタイプを示す文字コード。
  • 戻り値: なし。

前提条件と不変条件

  • buf は有効な StringInfo 構造体であること。
  • pq_sendXXX 系関数が buf->cursor を変更しないことが前提となっている。

アルゴリズムまたは振る舞い

1. resetStringInfo(buf) を呼び出し、バッファの内容をクリアして再利用可能な状態にする。 2. msgtype を buf->cursor フィールドに格納する。これは、後続の pq_endmessage_reuse 等でメッセージタイプを識別するために利用される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printtup.c
  • src/backend/commands/explain_dr.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • resetStringInfo (内部で呼び出し)

関連データ構造

  • StringInfo: メッセージ構築に使用される動的バッファ構造体。

失敗モードと境界条件

  • buf が NULL の場合、プログラムは異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は pq_beginmessage と異なり、メモリの再割り当てを避けて既存のバッファを再利用する。
  • メッセージタイプを cursor フィールドに一時退避させる実装であるため、pq_sendXXX 系関数がこのフィールドを上書きしないよう注意すること。
