JsonUniqueCheckState

定義

  • 種別: typedef (ポインタ型)
  • 定義箇所: src/backend/utils/adt/json.c:41
  • 実体: struct HTAB *

目的

JSONオブジェクトの解析時において、同一階層内に重複するキーが存在しないかを高速に検証するためのハッシュテーブルを保持する。

シグネチャまたは形状

typedef struct HTAB *JsonUniqueCheckState;

入力と出力

  • 入力: JSON解析プロセスにおいて、キーの重複チェックが必要なコンテキストで初期化される。
  • 出力: キーの重複が検出された場合、エラーを発生させるための状態管理に使用される。

前提条件と不変条件

  • HTAB（ハッシュテーブル）へのポインタとして機能する。
  • JSONオブジェクトの階層ごとに、キー名とオブジェクトIDを管理する。

アルゴリズムまたは振る舞い

  • json_unique_check_init 関数により初期化される。
  • json_unique_check_key 関数を通じて、現在のオブジェクトIDとキー名がハッシュテーブルに登録・照合される。
  • 既に同一キーが存在する場合は、重複エラーを通知する仕組みの基盤となる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/adt/json.c 内の解析ロジック（json_unique_check_init および json_unique_check_key の呼び出し元）。
  • 呼び出し先:
  • HTAB 関連のハッシュテーブル操作関数（PostgreSQLのハッシュテーブルAPI）。

関連データ構造

  • JsonUniqueHashEntry: ハッシュテーブルに格納されるエントリ構造体。キー文字列、キー長、オブジェクトIDを保持する。

失敗モードと境界条件

  • メモリ不足: ハッシュテーブルの作成時にメモリが不足した場合、PostgreSQLのメモリ管理機構によりエラーが発生する。
  • 重複検出: json_unique_check_key が同一キーを検出した際の挙動は、呼び出し側の実装に依存する。

保守担当者向けメモ

  • 本シンボルは HTAB へのエイリアスであるため、ハッシュテーブルのメモリ管理（MemoryContext）のライフサイクルに注意すること。
  • キーの比較は文字列ベースで行われるため、エンコーディングや正規化の要件に変更がある場合は JsonUniqueHashEntry の扱いを確認すること。
