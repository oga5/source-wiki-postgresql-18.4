DllMain

定義

  • 種別: function
  • 定義箇所: src/bin/pgevent/pgevent.c:151

目的

Windows DLLのオプションのエントリポイントとして機能し、DLLがプロセスにロードされた際にモジュールハンドルを保持する。

シグネチャまたは形状

BOOL WINAPI DllMain(HANDLE hModule, DWORD ul_reason_for_call, LPVOID lpReserved)

入力と出力

  • **入力**:
  • hModule: DLLのモジュールハンドル。
  • ul_reason_for_call: DLLが呼び出された理由を示すフラグ。
  • lpReserved: システム予約領域。
  • **出力**:
  • 常に TRUE を返す。

前提条件と不変条件

  • Windows環境下で実行されるDLLの標準的なエントリポイントであること。

アルゴリズムまたは振る舞い

1. ul_reason_for_call が DLL_PROCESS_ATTACH（プロセスへのロード時）であるかを確認する。 2. 一致する場合、引数として渡された hModule をグローバル変数 g_module に格納する。 3. 処理の成否に関わらず TRUE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Windows OS（DLLロード時）。
  • **呼び出し先**: 本関数内での関数呼び出しは存在しない。

関連データ構造

  • g_module: DLLのモジュールハンドルを保持するグローバル変数。

失敗モードと境界条件

  • 本関数は常に TRUE を返すため、明示的な失敗モードは定義されていない。

保守担当者向けメモ

  • g_module は、他の関数（DllRegisterServer 等）がDLLのパスを取得するために使用される重要な変数である。
  • 本関数は pgevent.c 内でプロトタイプ宣言（152行目付近）およびコメントによる説明（148行目付近）がなされている。
