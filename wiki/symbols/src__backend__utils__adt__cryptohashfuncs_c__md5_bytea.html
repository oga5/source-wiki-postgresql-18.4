md5_bytea

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/cryptohashfuncs.c:58

目的

bytea 型の入力データから MD5 ハッシュを計算し、その結果を 16 進数文字列（text 型）として返します。

シグネチャまたは形状

Datum md5_bytea(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る bytea 型のデータ。
  • **出力**: 計算された MD5 ハッシュ値の 16 進数表現を含む text 型の Datum。

前提条件と不変条件

  • 入力データは PG_GETARG_BYTEA_PP を使用して取得され、TOAST 展開（detoast）が考慮されます。
  • ハッシュ計算には common/md5.h で定義された pg_md5_hash が使用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_BYTEA_PP(0) を用いて入力データを取得します。 2. VARSIZE_ANY_EXHDR(in) により、ヘッダーを除いたデータ長を算出します。 3. pg_md5_hash を呼び出し、入力データから MD5 ハッシュを計算します。結果は hexsum バッファに格納されます。 4. ハッシュ計算が失敗した場合（false が返された場合）、ereport を通じて ERRCODE_INTERNAL_ERROR を発生させます。 5. 成功した場合、cstring_to_text を用いて 16 進数文字列を text 型に変換し、PG_RETURN_TEXT_P で返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（SQL クエリから md5() 関数として呼び出される）。
  • **呼び出し先**: pg_md5_hash（common/md5.h）、ereport、cstring_to_text。

関連データ構造

  • bytea: PostgreSQL の可変長バイナリ型。
  • text: PostgreSQL の文字列型。
  • MD5_HASH_LEN: MD5 ハッシュの長さを定義する定数。

失敗モードと境界条件

  • pg_md5_hash が内部エラーを返した場合、PostgreSQL のエラーメカニズムによりトランザクションが中断されます。
  • 入力データが NULL の場合、PG_GETARG_BYTEA_PP の仕様に従い処理されます。

保守担当者向けメモ

  • 本関数は歴史的な経緯により、バイナリ入力に対して 16 進数文字列を返します。SHA-2 ファミリーの関数がバイナリ（bytea）を返すのと挙動が異なるため、利用時には注意が必要です。
  • MD5 は暗号学的に脆弱であるため、セキュリティ要件が厳しい用途での利用は推奨されません。
