spgAllocSearchItem

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgscan.c:113

目的

SP-GiST 検索処理において、走査対象となるノードやタプルを保持するための SpGistSearchItem 構造体をメモリ上に割り当て、初期化します。

シグネチャまたは形状

static SpGistSearchItem *spgAllocSearchItem(SpGistScanOpaque so, bool isnull, double *distances)

入力と出力

  • **入力**:
  • so: スキャン状態を保持する SpGistScanOpaque。
  • isnull: 項目が NULL かどうかを示すフラグ。
  • distances: KNN 検索用の距離配列（isnull が false の場合に使用）。
  • **出力**:
  • 初期化済みの SpGistSearchItem へのポインタ。

前提条件と不変条件

  • so->numberOfNonNullOrderBys が有効な値であること。
  • isnull が false の場合、distances には少なくとも so->numberOfNonNullOrderBys 個の要素が含まれている必要があります。

アルゴリズムまたは振る舞い

1. SizeOfSpGistSearchItem マクロを使用して、必要なメモリサイズを計算します。NULL 項目の場合は距離配列分を省略し、それ以外は numberOfNonNullOrderBys 分の領域を確保します。 2. palloc を用いてメモリを割り当てます。 3. item->isNull に isnull を設定します。 4. isnull が false かつ numberOfNonNullOrderBys > 0 の場合、distances 配列の内容を item->distances にコピーします。 5. 構築した item を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgscan.c 内の複数箇所（spgAddStartItem および走査ループ内）。
  • **呼び出し先**:
  • palloc（メモリ割り当て）
  • memcpy（距離配列のコピー）

関連データ構造

  • SpGistSearchItem: 検索キューに格納される項目を定義する構造体。
  • SpGistScanOpaque: スキャン中の状態（メモリコンテキストや検索キー等）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc によりエラーが発生します。
  • numberOfNonNullOrderBys が 0 の場合、距離配列のコピーはスキップされます。

保守担当者向けメモ

  • 本関数は SpGistSearchItem のメモリレイアウトに依存しています。構造体の定義が変更された場合は SizeOfSpGistSearchItem マクロと併せて確認してください。
