ExecGather

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGather.c:136

目的

並列クエリ実行において、バックグラウンドワーカーが生成したタプルを収集し、単一のストリームとして上位ノードへ提供する。

シグネチャまたは形状

TupleTableSlot *ExecGather(PlanState *pstate)

入力と出力

  • **入力**: pstate (Gatherノードの実行状態を保持する PlanState)
  • **出力**: 収集された TupleTableSlot。タプルが存在しない場合は NULL を返す。

前提条件と不変条件

  • 初回実行時に node->initialized が false であること。
  • ParallelContext の初期化およびワーカーの起動は、初回実行時にのみ行われる。
  • 実行ごとに ResetExprContext が呼び出され、タプルごとのメモリが解放される。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時、ParallelContext を作成し、LaunchParallelWorkers を通じてワーカーを起動する。ワーカーが起動した場合は TupleQueueReader を作成し、リーダープロセスが読み取る準備を行う。 2. **ローカル実行判定**: ワーカーが利用できない場合、または single_copy が無効かつ parallel_leader_participation が有効な場合、リーダープロセス自身がプランを実行するよう need_to_scan_locally を設定する。 3. **タプル取得**: gather_getnext を呼び出し、ワーカーからのキュー読み取りまたはローカル実行によりタプルを取得する。 4. **射影**: ps_ProjInfo が存在する場合、ExecProject を適用して結果を整形する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecProcNode (実行エンジンにより gatherstate->ps.ExecProcNode として登録される)。
  • **呼び出し先**:
  • ExecInitParallelPlan, ExecParallelReinitialize, LaunchParallelWorkers, ExecParallelCreateReaders (初期化時)
  • gather_getnext (タプル取得)
  • ExecProject (射影)

関連データ構造

  • GatherState: ノードの実行状態を保持。
  • ParallelContext: 並列実行のコンテキスト。
  • TupleQueueReader: ワーカーからのタプル読み取り用。
  • PlanState: 実行プランの共通状態。

失敗モードと境界条件

  • **ワーカー起動失敗**: ワーカーが1つも起動しなかった場合、nreaders は 0 となり、リーダープロセスのみで実行される。
  • **タプル枯渇**: gather_getnext が NULL を返した場合、本関数も NULL を返す。
  • **割り込み**: CHECK_FOR_INTERRUPTS() により、実行中に割り込みが確認された場合は処理を中断する。

保守担当者向けメモ

  • 本関数は初回実行時に重いリソース（共有メモリ等）を確保するため、初期化の遅延評価を行っている。
  • node->reader 配列はワーカーの起動数に基づいて動的に確保される。
  • parallel_leader_participation 設定により、リーダープロセスがワーカーと並行してタプル生成に参加するかどうかが決定される点に注意すること。
