RemoveWalSummaryIfOlderThan

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummary.c:229

目的

指定されたWAL要約ファイルの最終更新日時（st_mtime）が、指定された期限（cutoff_time）よりも古い場合に、当該ファイルをファイルシステムから削除します。

シグネチャまたは形状

void RemoveWalSummaryIfOlderThan(WalSummaryFile *ws, time_t cutoff_time)

入力と出力

  • **入力**:
  • ws: 削除対象の要約ファイルを特定するための WalSummaryFile 構造体へのポインタ。
  • cutoff_time: 削除の基準となる時刻（time_t）。
  • **出力**: なし（void）。

前提条件と不変条件

  • ws が指す構造体には、有効な tli、start_lsn、end_lsn が格納されている必要があります。
  • XLOGDIR "/summaries/" 配下に、命名規則に従った要約ファイルが存在することが期待されます。

アルゴリズムまたは振る舞い

1. ws の情報から、対応する要約ファイルのパスを snprintf で生成します。 2. lstat を使用してファイルのメタデータを取得します。

  • ファイルが存在しない場合（ENOENT）、処理を正常終了します。
  • その他のエラーが発生した場合は ereport(ERROR, ...) で処理を中断します。

    • 3. 取得したファイルの st_mtime と cutoff_time を比較します。

  • st_mtime >= cutoff_time の場合、ファイルは新しいため何もせず終了します。

    • 4. unlink を実行してファイルを削除します。

  • 削除に失敗した場合は ereport(ERROR, ...) で処理を中断します。

    • 5. 成功時は DEBUG2 レベルで削除した旨をログ出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/walsummarizer.c:1729
  • **呼び出し先**:
  • lstat, unlink (システムコール)
  • ereport (PostgreSQLエラー報告機構)

関連データ構造

  • WalSummaryFile: 要約ファイルの識別情報（TLIおよびLSN範囲）を保持する構造体。

失敗モードと境界条件

  • **ファイルアクセスエラー**: lstat または unlink が失敗した場合、ereport(ERROR) によりトランザクションまたはプロセスが停止します。
  • **ファイル不在**: 削除対象のファイルが既に存在しない場合は、エラーとせず静かに終了します。

保守担当者向けメモ

  • 本関数は XLOGDIR を基準としたパス構築を行うため、WAL要約ファイルの配置場所が変更された場合はパス生成ロジックの更新が必要です。
  • DEBUG2 レベルのログは、要約ファイルのクリーンアップ状況を追跡する際に有用です。
