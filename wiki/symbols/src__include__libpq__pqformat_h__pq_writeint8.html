pq_writeint8

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/libpq/pqformat.h:45

目的

通信バッファ（StringInfoData）の末尾に、1バイトの整数値（uint8）を直接書き込みます。

シグネチャまたは形状

static inline void pq_writeint8(StringInfoData *pg_restrict buf, uint8 i)

入力と出力

  • **入力**:
  • buf: 書き込み対象の StringInfoData 構造体へのポインタ。
  • i: 書き込む uint8 型の整数値。
  • **出力**: なし（戻り値なし）。バッファの内容が更新されます。

前提条件と不変条件

  • buf が指すバッファには、sizeof(uint8) 分の空き容量が確保されている必要があります。
  • 実行時に Assert(buf->len + (int) sizeof(uint8) <= buf->maxlen); により、バッファオーバーフローが発生しないことが保証されます。

アルゴリズムまたは振る舞い

1. 入力値 i をローカル変数 ni にコピーします。 2. バッファの現在の書き込み位置（buf->data + buf->len）に ni を memcpy でコピーします。 3. バッファの長さ（buf->len）を sizeof(uint8) 分だけインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/libpq/pqformat.h:131 での利用が確認されています。
  • **呼び出し先**: memcpy（標準ライブラリ関数）を呼び出します。

関連データ構造

  • StringInfoData: 通信データを保持するバッファ構造体（lib/stringinfo.h で定義）。

失敗モードと境界条件

  • バッファの容量（maxlen）が不足している場合、アサーションが有効なビルドではプログラムが停止します。
  • 本関数は動的なバッファ拡張を行わないため、呼び出し側で事前に十分な容量を確保しておく必要があります。

保守担当者向けメモ

  • static inline 関数であるため、ヘッダーファイル内で定義されています。
  • バッファの自動拡張が必要な場合は、代わりに pq_sendint8 を使用してください。
  • pg_restrict キーワードにより、コンパイラによる最適化が促進されています。
