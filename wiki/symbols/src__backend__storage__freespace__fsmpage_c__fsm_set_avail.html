fsm_set_avail

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/fsmpage.c:62

目的

FSM（空き領域マップ）ページ内の指定されたスロットの空き容量を更新し、その変更をツリー構造の上位ノードへ伝播させて整合性を維持します。

シグネチャまたは形状

bool fsm_set_avail(Page page, int slot, uint8 value)

入力と出力

  • **入力**:
  • page: 操作対象のFSMページ。
  • slot: 更新対象のスロット番号。
  • value: 設定する空き容量（uint8）。
  • **出力**:
  • bool: 値が実際に変更され、ツリーの更新が行われた場合は true、変更が不要であった場合は false を返します。

前提条件と不変条件

  • slot は LeafNodesPerPage 未満である必要があります（Assert による検証）。
  • 呼び出し元は、対象の page に対して適切なバッファロックを保持している必要があります。
  • ツリーのルートノード（fp_nodes[0]）は、常にページ内の最大空き容量を保持している必要があります。

アルゴリズムまたは振る舞い

1. 指定された slot に対応するリーフノードの値を更新します。 2. 現在値と新しい値が等しく、かつ新しい値がルートノードの値以下であれば、更新不要と判断し false を返して終了します。 3. リーフノードからルート方向へ向かって、parentof() を用いて親ノードを順次辿ります。 4. 各親ノードにおいて、左右の子ノードの最大値を計算し、親ノードの値を更新します。値に変更がない場合は伝播を停止します。 5. 処理終了後、設定した value がルートノードの値よりも大きい場合、ページが破損していると判断し fsm_rebuild_page(page) を呼び出して再構築を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/freespace/freespace.c（複数箇所）
  • **呼び出し先**:
  • fsm_rebuild_page（破損検知時）
  • parentof, leftchild（マクロ）
  • Max（マクロ）
  • PageGetContents（マクロ）

関連データ構造

  • FSMPage: ページ内のノード配列（fp_nodes）を保持する構造体。
  • NonLeafNodesPerPage, NodesPerPage: ツリー構造のノード配置を定義する定数。

失敗モードと境界条件

  • **破損検知**: 更新後の値がルートノードの保持する最大値を超えた場合、ツリーの整合性が失われていると見なし、fsm_rebuild_page による強制再構築が行われます。
  • **変更不要**: 設定値が既存値と同一であり、かつルートノードの最大値を超えない場合は、何も変更せずに false を返します。

保守担当者向けメモ

  • 本関数はバッファロックを前提としています。ロックの競合を避けるため、探索のヒント（fp_next_slot）と組み合わせて使用されることが一般的です。
  • fsm_rebuild_page はコストが高い操作であるため、頻繁な呼び出しはパフォーマンスに影響を与えます。
