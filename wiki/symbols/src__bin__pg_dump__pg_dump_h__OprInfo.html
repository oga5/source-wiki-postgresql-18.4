OprInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.h:258

目的

データベース内の演算子（Operator）情報を保持し、pg_dump によるダンプ処理において演算子オブジェクトを管理・ソート・出力するための構造体です。

シグネチャまたは形状

typedef struct _oprInfo
{
    DumpableObject dobj;
    const char *rolname;
    char        oprkind;
    Oid         oprleft;
    Oid         oprright;
    Oid         oprcode;
} OprInfo;

入力と出力

  • **入力**: システムカタログから取得された演算子の属性情報。
  • **出力**: dumpOpr 関数を介して、アーカイブへ演算子定義を出力するために使用されます。

前提条件と不変条件

  • dobj フィールドにより DumpableObject としての共通属性（ID、依存関係など）を継承しています。
  • pg_dump の実行期間中、メモリ上に生存し続けます。

アルゴリズムまたは振る舞い

  • pg_dump.c 内で pg_malloc を用いて配列として確保され、システムカタログから読み取った演算子情報が格納されます。
  • pg_dump_sort.c において、依存関係に基づいたダンプ順序決定の対象となります。
  • dumpOpr 関数により、特定の演算子定義をアーカイブへ書き出す処理に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c: 演算子情報の確保および dumpOpr の呼び出し。
  • src/bin/pg_dump/pg_dump_sort.c: ダンプ順序ソート時の型キャスト。
  • src/bin/pg_dump/common.c: DumpableObject からのキャスト。
  • **呼び出し先**:
  • findOprByOid(Oid oid): OID をキーに OprInfo を検索する関数が定義されています。

関連データ構造

  • DumpableObject: 本構造体の基底となる構造体。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • メモリ確保失敗時（pg_malloc 経由）の挙動は pg_dump 全体のエラーハンドリングに依存します。
  • 演算子情報の取得時にカタログデータが不整合である場合、ダンプ出力が不正になる可能性があります。

保守担当者向けメモ

  • DumpableObjectType に新しい型を追加する際は、本構造体を利用するソートロジック（pg_dump_sort.c）との整合性に注意してください。
  • OprInfo の各フィールドはシステムカタログの定義と密接に関連しているため、カタログスキーマの変更時には本構造体の定義を見直す必要があります。
