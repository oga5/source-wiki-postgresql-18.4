find_word

定義

  • 種別: function
  • 定義箇所: contrib/dict_xsyn/dict_xsyn.c:50

目的

文字列から次の単語を切り出し、その開始位置と終了位置を特定します。

シグネチャまたは形状

static char *find_word(char *in, char **end)

入力と出力

  • **入力**:
  • in: 解析対象の文字列へのポインタ。
  • end: 単語の終了位置（次の空白文字または終端）を格納するためのポインタのポインタ。
  • **出力**:
  • 戻り値: 切り出された単語の先頭ポインタ。単語が見つからない場合は NULL。
  • end: 単語の直後の位置を指すポインタ。

前提条件と不変条件

  • in はヌル終端文字列である必要があります。
  • pg_mblen_cstr を使用しているため、マルチバイト文字を考慮した走査が行われます。

アルゴリズムまたは振る舞い

1. in を進め、先頭の空白文字をスキップします。 2. 文字列の終端に達するか、コメント文字 # が現れた場合は NULL を返します。 3. 空白文字以外の連続する領域を単語として特定します。 4. 特定した単語の開始位置を戻り値とし、単語の直後の位置を *end に書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dict_xsyn/dict_xsyn.c 内の read_dictionary 関数（105行目、241行目付近）。
  • **呼び出し先**:
  • pg_mblen_cstr（PostgreSQLのマルチバイト文字長取得関数）。
  • **注意**: contrib/pg_trgm/trgm_op.c に同名の関数が存在しますが、シグネチャが異なるため、本シンボルとは別の実装です。

関連データ構造

  • なし。

失敗モードと境界条件

  • 文字列が空白のみ、または # で始まる場合は NULL を返します。

保守担当者向けメモ

  • 本関数は contrib/dict_xsyn 辞書ファイルのパース処理において、行内のトークン分割に使用されています。
  • isspace に渡す際は (unsigned char) へのキャストを行っており、ロケール依存の動作に対する安全性が確保されています。
