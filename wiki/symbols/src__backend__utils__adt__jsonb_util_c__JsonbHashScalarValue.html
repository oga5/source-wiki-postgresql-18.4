JsonbHashScalarValue

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_util.c:1328

目的

JsonbValue 構造体に含まれるスカラー値（null, 文字列, 数値, 真偽値）に対してハッシュ値を計算し、既存のハッシュ値と組み合わせて更新する。

シグネチャまたは形状

void JsonbHashScalarValue(const JsonbValue *scalarVal, uint32 *hash)

入力と出力

  • scalarVal: ハッシュ計算対象となる JsonbValue へのポインタ。
  • hash: 更新対象のハッシュ値へのポインタ。計算結果がこのアドレスに書き込まれる。

前提条件と不変条件

  • scalarVal はスカラー型（jbvNull, jbvString, jbvNumeric, jbvBool）である必要がある。
  • 数値型（jbvNumeric）のハッシュ計算には hash_numeric 関数が使用されるため、等価な数値は必ず等価なハッシュコードを生成する。

アルゴリズムまたは振る舞い

1. scalarVal->type に基づき、以下の通り一時的なハッシュ値 tmp を算出する。

  • jbvNull: 0x01
  • jbvString: hash_any を使用して文字列内容から算出。
  • jbvNumeric: hash_numeric を使用して算出。
  • jbvBool: true なら 0x02、false なら 0x04。

    • 2. 既存の *hash を左に 1 ビット回転（pg_rotate_left32）させる。 3. 回転後の値と tmp の XOR を取り、結果を *hash に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonb_gin.c (GIN インデックス構築時)
  • src/backend/utils/adt/jsonb_op.c (JSONB 演算時)
  • **呼び出し先**:
  • hash_any (文字列ハッシュ用)
  • hash_numeric (数値ハッシュ用)
  • pg_rotate_left32 (ビット回転用)

関連データ構造

  • JsonbValue: JSONB のノード表現。
  • uint32: ハッシュ値の型。

失敗モードと境界条件

  • scalarVal->type が上記以外の型（コンテナ型など）である場合、elog(ERROR) を発生させ処理を中断する。

保守担当者向けメモ

  • 本関数は JsonbHashScalarValueExtended と類似の目的を持つ。
  • GIN インデックスのハッシュ計算において、キーや要素の順序に応じたハッシュの蓄積に使用されるため、計算ロジックの変更はインデックスの互換性に影響を与える可能性がある。
