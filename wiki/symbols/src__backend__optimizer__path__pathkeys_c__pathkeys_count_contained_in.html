pathkeys_count_contained_in

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/pathkeys.c:557

目的

2つのパスキーリスト（keys1 と keys2）を先頭から比較し、共通するパスキーの数（プレフィックス長）を算出します。また、keys1 が keys2 の先頭部分と完全に一致するかどうかを判定します。

シグネチャまたは形状

bool pathkeys_count_contained_in(List *keys1, List *keys2, int *n_common)

入力と出力

  • **入力**:
  • keys1: 比較対象となるパスキーのリスト。
  • keys2: 比較対象となるパスキーのリスト。
  • n_common: 共通するパスキーの数を格納するための出力用ポインタ。
  • **出力**:
  • bool: keys1 の全要素が keys2 の先頭部分と一致する場合に true を返します。

前提条件と不変条件

  • keys1 および keys2 は、正規化された PathKey のリストである必要があります。
  • 正規化されているため、パスキーの比較はポインタ比較（pathkey1 != pathkey2）で行われます。

アルゴリズムまたは振る舞い

1. **最適化チェック**:

  • keys1 と keys2 が同一ポインタの場合、リスト長を n_common に設定し true を返します。
  • keys1 が空（NIL）の場合、n_common を 0 に設定し true を返します。
  • keys2 が空（NIL）の場合、n_common を 0 に設定し false を返します。

    • 2. **逐次比較**:

  • forboth マクロを用いて両リストを先頭から順に走査します。
  • 各要素のポインタが一致しない場合、その時点までのカウントを n_common に格納し、false を返します。

    • 3. **終了判定**:

  • ループ終了後、keys1 が完全に走査し終えた（key1 == NULL）場合に true を返し、そうでない場合は false を返します。

関連データ構造

  • PathKey: 順序を表現する構造体。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • keys1 が keys2 より長い場合、共通部分のカウント後に false を返します。
  • keys1 が keys2 の一部と一致しない場合、一致した箇所までのカウントを返し false を返します。

保守担当者向けメモ

  • 本関数はプランナのパフォーマンスに影響するため、リストの比較を最小限にする最適化が施されています。変更を加える際は、このループ効率を維持してください。
