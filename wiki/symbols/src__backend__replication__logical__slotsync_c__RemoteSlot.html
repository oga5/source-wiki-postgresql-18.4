RemoteSlot

定義

  • **種別**: struct
  • **定義箇所**: src/backend/replication/logical/slotsync.c:144

目的

プライマリサーバーから取得した論理レプリケーションスロットのメタデータを、スタンバイサーバー上で一時的に保持および管理するために使用されます。

シグネチャまたは形状

typedef struct RemoteSlot
{
    char       *name;
    char       *plugin;
    char       *database;
    bool        two_phase;
    bool        failover;
    XLogRecPtr  restart_lsn;
    XLogRecPtr  confirmed_lsn;
    XLogRecPtr  two_phase_at;
    TransactionId catalog_xmin;
    ReplicationSlotInvalidationCause invalidated;
} RemoteSlot;

入力と出力

本構造体は、プライマリサーバーの pg_replication_slots ビューから取得された情報を格納するコンテナとして機能します。出力として、ローカルのレプリケーションスロットの状態更新や永続化処理の引数として利用されます。

前提条件と不変条件

  • invalidated フィールドが RS_INVAL_NONE である場合、そのスロットは有効とみなされます。
  • 同期プロセスにおいて、この構造体はプライマリから取得した最新状態を反映しており、ローカルの update_local_synced_slot 等を通じて整合性が検証されます。

アルゴリズムまたは振る舞い

1. プライマリから取得したスロット情報が RemoteSlot インスタンスとしてメモリ上に割り当てられます。 2. synchronize_one_slot 関数がこの構造体を受け取り、ローカルのレプリケーションスロットとの比較・同期を行います。 3. update_local_synced_slot または update_and_persist_local_synced_slot を介して、ローカル側のスロットメタデータが更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: synchronize_slots（ループ内での生成）、synchronize_one_slot（引数として受け取り）、update_local_synced_slot（引数として受け取り）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ReplicationSlotInvalidationCause: スロットの無効化理由を定義する列挙型。
  • XLogRecPtr: LSN（Log Sequence Number）を保持する型。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc0 使用時）の例外。
  • プライマリから取得した情報が不完全な場合、invalidated フィールドにより無効なスロットとして処理される可能性があります。

保守担当者向けメモ

  • 本構造体は slotsync.c 内でのみ使用される一時的なデータ構造です。
  • スロットの属性（two_phase や failover 等）が追加された場合は、本構造体にもフィールドを追加し、同期ロジックを更新する必要があります。
