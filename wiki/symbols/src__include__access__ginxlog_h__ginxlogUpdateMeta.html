ginxlogUpdateMeta

定義

  • 種別: struct
  • 定義箇所: src/include/access/ginxlog.h:168

目的

GINインデックスのメタページ更新操作をWAL（Write-Ahead Log）に記録するためのデータ構造です。リカバリ時にメタページの情報を再構築し、インデックスの整合性を維持するために使用されます。

シグネチャまたは形状

typedef struct ginxlogUpdateMeta
{
    RelFileLocator locator;
    GinMetaPageData metadata;
    BlockNumber prevTail;
    BlockNumber newRightlink;
    int32       ntuples;
    /* array of inserted tuples follows */
} ginxlogUpdateMeta;

入力と出力

  • **入力**: 構造体自体がWALレコードのデータ部として書き込まれます。ntuples が0より大きい場合は、メタページの末尾（tail）が更新されたことを示し、それ以外の場合は新しいサブリストが挿入されたことを示します。
  • **出力**: 構造体の直後に、挿入されたタプルの配列が可変長データとして続きます。

前提条件と不変条件

  • 構造体内の各フィールドは、WAL再生時に memcpy で直接読み取られるため、メモリレイアウトの変更はバイナリ互換性に影響します。
  • ntuples の値に応じて、後続する可変長データの解釈が異なります。

アルゴリズムまたは振る舞い

1. **記録**: GINインデックスの更新処理において、本構造体にメタデータやブロック情報を格納し、XLogRegisterData を通じてWALレコードとして登録します。 2. **再生**: XLogRecGetData を介してレコードから本構造体を復元し、metadata や ntuples の情報に基づき、対象のGINメタページを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginfast.c
  • src/backend/access/gin/ginutil.c
  • **呼び出し先**:
  • 提供されたソース情報からは、特定の関数呼び出し関係は特定できません。

関連データ構造

  • RelFileLocator: 対象リレーションの識別子。
  • GinMetaPageData: GINインデックスのメタページ情報。
  • BlockNumber: ページ番号（prevTail, newRightlink）。

失敗モードと境界条件

  • ntuples の値と後続するタプルデータの不整合が発生した場合、リカバリ処理が正しく行われません。
  • WALレコードのサイズが適切に計算されていない場合、XLogRegisterData による登録時にバッファオーバーフローやデータ破損のリスクがあります。

保守担当者向けメモ

  • 本構造体は可変長データを含むため、フィールドの追加や変更を行う際は、アライメントとパディングに細心の注意を払ってください。
  • ginxlogUpdateMeta を使用する際は、src/backend/access/gin/ginxlog.c のREDO処理ロジックと整合性が取れていることを確認してください。
