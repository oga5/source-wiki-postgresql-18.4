testcustomrmgrs_redo

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_custom_rmgrs/test_custom_rmgrs.c:81

目的

WALリカバリ（Redo）処理において、カスタムWALレコードの操作コードを検証し、必要に応じてリカバリ処理を実行するためのコールバック関数です。

シグネチャまたは形状

void testcustomrmgrs_redo(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record - 解析対象のWALレコード状態へのポインタ。
  • **出力**: なし（void）。ただし、不正な操作コードが検出された場合は PANIC レベルでエラーを発生させます。

前提条件と不変条件

  • 本関数は、PostgreSQLのWALリカバリプロセスから呼び出されることを前提としています。
  • XLogRecGetInfo(record) で取得される情報から、XLR_INFO_MASK を除外した値が XLOG_TEST_CUSTOM_RMGRS_MESSAGE と一致する必要があります。

アルゴリズムまたは振る舞い

1. XLogRecGetInfo を使用して、レコードから操作コード（info）を抽出します。 2. XLR_INFO_MASK ビットをマスクして純粋な操作コードを取得します。 3. 取得したコードが XLOG_TEST_CUSTOM_RMGRS_MESSAGE と一致するか確認します。 4. 一致しない場合、elog(PANIC, ...) を呼び出し、リカバリを停止させます。 5. 本モジュールにおいては、一致した場合の具体的なデータ復元処理（No-op）は実装されていません。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのWALリカバリサブシステム（RmgrData 構造体の .rm_redo フィールド経由で間接的に呼び出されます）。
  • **呼び出し先**: XLogRecGetInfo（マクロ）、elog（PostgreSQLエラー報告関数）。

関連データ構造

  • XLogReaderState: WALレコードの読み込み状態を保持する構造体。
  • RmgrData: カスタムリソースマネージャのコールバックを定義する構造体。

失敗モードと境界条件

  • **失敗モード**: XLOG_TEST_CUSTOM_RMGRS_MESSAGE 以外の操作コードが渡された場合、PANIC エラーが発生し、データベースのリカバリが強制終了されます。

保守担当者向けメモ

  • 本関数はテスト用テンプレートとして実装されており、現在は検証のみを行うNo-opです。実運用環境でカスタムRmgrを拡張する場合は、この関数内に具体的なデータ復元ロジックを実装する必要があります。
