PQsetResultAttrs

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-exec.c:249

目的

PGresult オブジェクトに対して列属性（メタデータ）を設定します。

シグネチャまたは形状

int PQsetResultAttrs(PGresult *res, int numAttributes, PGresAttDesc *attDescs);

入力と出力

  • **入力**:
  • res: 対象となる PGresult オブジェクト。
  • numAttributes: 設定する列の数。
  • attDescs: 列の記述情報を含む PGresAttDesc 配列。
  • **出力**: 成功した場合は 1（true）、失敗した場合は 0（false）を返します。

前提条件と不変条件

  • res が NULL または OOM_result である場合、関数は直ちに失敗します。
  • res->numAttributes が既に 0 より大きい場合、属性の上書きは許可されず、関数は失敗します。
  • numAttributes が 0 以下、または attDescs が NULL の場合、何もしない（no-op）として成功を返します。

アルゴリズムまたは振る舞い

1. PQresultAlloc を使用して、numAttributes 分の PGresAttDesc を格納するメモリを確保します。 2. 入力された attDescs を確保したメモリ領域にコピーします。 3. 各列の属性名（name）を pqResultStrdup を用いて PGresult のメモリ管理下に複製します。名前が NULL の場合は res->null_field を割り当てます。 4. 全ての列の format フィールドを走査し、一つでも 0（テキスト形式）が含まれていれば res->binary を 0 に設定します。それ以外の場合はデフォルトで 1（バイナリ形式）とみなします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-exec.c 内で、PGresult を複製する際などに利用されます。
  • **呼び出し先**: PQresultAlloc（メモリ確保）、pqResultStrdup（文字列複製）。

関連データ構造

  • PGresult: クエリ結果を保持する構造体。
  • PGresAttDesc: 列のメタデータ（名前、フォーマット等）を保持する構造体。

失敗モードと境界条件

  • メモリ不足により PQresultAlloc または pqResultStrdup が失敗した場合、関数は 0 を返します。
  • 既に属性が設定されている PGresult に対して呼び出した場合、変更は行われず 0 を返します。

保守担当者向けメモ

  • 本関数は PGresult の初期化フェーズで使用されることを想定しています。
  • res->binary フラグの決定ロジックは、全ての列がバイナリ形式である場合にのみ 1 となる仕様です。
