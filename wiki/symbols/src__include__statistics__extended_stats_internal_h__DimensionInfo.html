DimensionInfo

定義

  • 種別: struct
  • 定義箇所: src/include/statistics/extended_stats_internal.h:34

目的

拡張統計情報（特にMCVリスト）の構築およびシリアライズ処理において、各次元（列）のデータ特性とメモリレイアウト情報を保持するために使用されます。

シグネチャまたは形状

typedef struct DimensionInfo
{
	int			nvalues;		/* 重複排除された値の数 */
	int			nbytes;			/* シリアライズ後のバイト数 */
	int			nbytes_aligned; /* アライメント調整後のデシリアライズデータサイズ */
	int			typlen;			/* pg_type.typlen */
	bool		typbyval;		/* pg_type.typbyval */
} DimensionInfo;

入力と出力

本構造体は、統計情報の構築プロセスにおいて、列のメタデータ（typlen, typbyval）および計算された統計量（nvalues, nbytes 等）を保持するコンテナとして機能します。

前提条件と不変条件

  • typlen および typbyval は、対象となるデータ型の pg_type カタログ情報と一致している必要があります。
  • nbytes_aligned は、デシリアライズ時のメモリ配置を考慮したサイズである必要があります。

アルゴリズムまたは振る舞い

統計情報のシリアライズおよびデシリアライズ時に、次元ごとのメタデータとして配列形式でメモリ上に配置されます。src/backend/statistics/mcv.c において、統計データのバイナリ表現の先頭部分に DimensionInfo の配列が格納され、それに続いて実際のデータ本体が配置される構造をとります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/mcv.c 内の統計構築およびシリアライズ/デシリアライズ処理（mcv_clauselist_selectivity 等の関連処理）から、配列として動的に割り当てられ、参照・コピーされます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • MCVList: 本構造体を用いて各次元の情報を管理する統計オブジェクト。
  • pg_type: typlen や typbyval の情報の参照元。

失敗モードと境界条件

  • ndims（次元数）が 0 の場合、本構造体の配列割り当ては行われません。
  • メモリ割り当て時に ndims * sizeof(DimensionInfo) の計算においてオーバーフローが発生しないことが前提となります。

保守担当者向けメモ

  • 本構造体は src/backend/statistics/mcv.c におけるバイナリ形式のヘッダーの一部として機能しています。構造体のメンバを変更した場合は、シリアライズおよびデシリアライズを行う memcpy 処理やポインタ演算の整合性を必ず確認してください。
