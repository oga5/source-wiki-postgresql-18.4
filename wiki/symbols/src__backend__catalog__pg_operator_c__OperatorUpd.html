OperatorUpd

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_operator.c:683

目的

システムカタログ pg_operator において、特定の演算子に対する交換子（Commutator）および否定子（Negator）のリンク情報を更新または削除する。

シグネチャまたは形状

void OperatorUpd(Oid baseId, Oid commId, Oid negId, bool isDelete)

入力と出力

  • **入力**:
  • baseId: 更新対象の演算子（ベース）のOID。
  • commId: 設定または削除対象の交換子演算子のOID。
  • negId: 設定または削除対象の否定子演算子のOID。
  • isDelete: true の場合、リンクを解除（InvalidOid に設定）する。false の場合、リンクを baseId に設定する。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • pg_operator テーブルに対する RowExclusiveLock が内部で取得される。
  • isDelete が false の場合、対象の交換子または否定子の oprcom/oprnegate フィールドが既に別の演算子を指しているとエラー（ERRCODE_INVALID_FUNCTION_DEFINITION）が発生する。
  • 自身の交換子/否定子として設定する場合、CommandCounterIncrement() が呼び出され、更新の可視性が保証される。

アルゴリズムまたは振る舞い

1. isDelete が false の場合、CommandCounterIncrement() を実行。 2. pg_operator テーブルをオープンする。 3. commId が有効な場合、SearchSysCacheCopy1 で交換子のタプルを取得し、isDelete に応じて oprcom フィールドを更新（またはクリア）して CatalogTupleUpdate を実行する。 4. 同様に negId が有効な場合、否定子のタプルを取得し、oprnegate フィールドを更新（またはクリア）して CatalogTupleUpdate を実行する。 5. 更新が発生した場合は CommandCounterIncrement() を呼び出し、変更を可視化する。 6. 最後にテーブルをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_operator.c (演算子作成処理内)
  • src/backend/commands/operatorcmds.c (演算子削除および作成処理内)
  • **呼び出し先**:
  • CommandCounterIncrement
  • table_open / table_close
  • SearchSysCacheCopy1
  • CatalogTupleUpdate
  • get_opname (エラーメッセージ生成時)

関連データ構造

  • pg_operator (システムカタログ)
  • Form_pg_operator (カタログタプルの構造体)

失敗モードと境界条件

  • **カタログ破損の可能性**: 既に別の演算子とリンクされているフィールドを上書きしようとするとエラーが発生する。
  • **無効なOID**: commId や negId が InvalidOid の場合、そのリンク更新処理はスキップされる。

保守担当者向けメモ

  • 本関数は pg_operator の整合性を維持するためのガードレールとして機能する。
  • 相互参照（交換子や否定子）のリンク更新時に CommandCounterIncrement を適切に挟むことで、自己参照時の可視性問題を回避している。
