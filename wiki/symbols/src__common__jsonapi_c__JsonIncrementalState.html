JsonIncrementalState

定義

  • 種別: struct
  • 定義箇所: src/common/jsonapi.c:162

目的

JSONのインクリメンタル（非再帰的）構文解析において、チャンクを跨いだ解析状態を保持するために使用されます。

シグネチャまたは形状

struct JsonIncrementalState
{
	bool		started;
	bool		is_last_chunk;
	bool		partial_completed;
	jsonapi_StrValType partial_token;
};

入力と出力

  • **入力**: pg_parse_json_incremental を通じて渡されるJSONチャンクデータ。
  • **出力**: 解析状態の更新（started フラグの遷移や partial_token へのトークン断片の保持）。

前提条件と不変条件

  • JsonLexContext 内の inc_state メンバとして保持されます。
  • チャンク単位の処理において、トークンがチャンク境界で分断された場合、partial_token にその断片が保持されます。

アルゴリズムまたは振る舞い

  • started: 解析が開始されたかどうかを追跡します。
  • is_last_chunk: 現在処理中のチャンクが最終チャンクであるかを示します。
  • partial_completed: チャンクを跨いでトークンが完結したかどうかを管理します。
  • partial_token: チャンク境界で分断されたトークンの値を保持し、次回のチャンク処理時に結合して解析を継続します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/jsonapi.c 内の pg_parse_json_incremental がこの構造体を割り当て、状態を更新します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • JsonLexContext: 本構造体を保持する親コンテキスト。
  • JsonParserStack: インクリメンタル解析時に使用される予測スタック。

失敗モードと境界条件

  • メモリ不足時: src/common/jsonapi.c:295 に定義された failed_inc_oom が参照される可能性があります。
  • チャンクの不整合: 最終チャンクでないにもかかわらず、トークンが不正な形式で終了した場合、解析エラーが発生します。

保守担当者向けメモ

  • 本構造体は src/include/common/jsonapi.h:64 にも定義が存在します。
  • インクリメンタル解析のロジックを変更する際は、partial_token のメモリ管理と、チャンク境界でのトークン再構築の整合性に注意してください。
