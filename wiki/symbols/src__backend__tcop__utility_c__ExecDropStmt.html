ExecDropStmt

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/utility.c:1994

目的

DROP 文（DropStmt）の実行を制御し、削除対象のオブジェクト型に応じて適切な削除処理関数へディスパッチする。

シグネチャまたは形状

static void ExecDropStmt(DropStmt *stmt, bool isTopLevel)

入力と出力

  • **入力**:
  • stmt: 実行対象の DropStmt 構造体へのポインタ。
  • isTopLevel: コマンドがトップレベルで実行されているかを示すフラグ。
  • **出力**: なし（void）。

前提条件と不変条件

  • stmt->removeType に基づいて分岐する。
  • OBJECT_INDEX かつ stmt->concurrent が真の場合、トランザクションブロック内での実行を禁止する制約チェックが行われる。

アルゴリズムまたは振る舞い

1. stmt->removeType を評価する。 2. OBJECT_INDEX で concurrent が指定されている場合、PreventInTransactionBlock を呼び出し、トランザクション内での実行を阻止する。 3. 以下のオブジェクト型（OBJECT_TABLE, OBJECT_SEQUENCE, OBJECT_VIEW, OBJECT_MATVIEW, OBJECT_FOREIGN_TABLE）の場合は RemoveRelations(stmt) を呼び出す。 4. 上記以外のオブジェクト型については、デフォルトとして RemoveObjects(stmt) を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c 内の standard_ProcessUtility 等から呼び出される（ソースコード上の直接的な参照箇所として line 986 および line 1774 が確認されている）。
  • **呼び出し先**:
  • PreventInTransactionBlock
  • RemoveRelations
  • RemoveObjects

関連データ構造

  • DropStmt: 削除対象のオブジェクト情報やオプションを保持するパーサー生成構造体。

失敗モードと境界条件

  • DROP INDEX CONCURRENTLY がトランザクションブロック内で呼び出された場合、PreventInTransactionBlock によりエラーが発生する。

保守担当者向けメモ

  • 本関数は utility.c 内のディスパッチロジックの一部である。新しいオブジェクト型を DROP 対象に追加する場合、switch 文のケース追加を検討すること。
