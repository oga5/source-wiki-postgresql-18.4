startup_hacks

定義

  • 種別: function
  • 定義箇所: src/backend/main/main.c:279

目的

PostgreSQL サーバー起動時における、プラットフォーム固有（主に Windows）の実行環境の初期化およびハック処理を行う。

シグネチャまたは形状

static void startup_hacks(const char *progname)

入力と出力

  • **入力**: progname (実行ファイル名)
  • **出力**: なし（副作用としてグローバルな環境設定を変更する）

前提条件と不変条件

  • WIN32 環境下でのみ実行される。
  • プロセス起動の極めて初期段階で呼び出される必要がある。

アルゴリズムまたは振る舞い

Windows 環境において以下の処理を実行する： 1. **ストリーム設定**: stdout および stderr を非バッファリング (_IONBF) に設定。 2. **Winsock 初期化**: WSAStartup を呼び出し、ネットワーク機能を準備する。失敗時はエラーを出力して終了する。 3. **例外処理設定**: _set_abort_behavior を使用し、abort() 発生時にクラッシュダンプを生成するように設定（MinGW 以外）。 4. **エラーモード制御**: SetErrorMode および _set_error_mode を使用し、ポップアップダイアログを抑制してエラーを stderr に出力するよう設定。 5. **デバッグ出力設定**: _CrtSetReportMode および _CrtSetReportFile を使用し、デバッグビルド時のアサーションやエラーをポップアップではなく stderr へリダイレクトする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/main/main.c 内の main() 関数。
  • **呼び出し先**: Windows API (WSAStartup, SetErrorMode) および C ランタイム関数 (setvbuf, _set_abort_behavior, _set_error_mode, _CrtSetReportMode 等)。

関連データ構造

  • WSADATA (Winsock 初期化用構造体)

失敗モードと境界条件

  • WSAStartup が失敗した場合、write_stderr を経由してエラーメッセージを出力し、exit(1) でプロセスを終了する。
  • MinGW 環境では _set_abort_behavior が呼び出されない。

保守担当者向けメモ

  • 本関数は Windows 特有の挙動を制御するゲートウェイである。Windows 以外のプラットフォームでは実質的に空の関数となる。
  • デバッグ時のポップアップ抑制設定は、CI や自動テスト環境でのプロセス停止を防ぐために重要である。
