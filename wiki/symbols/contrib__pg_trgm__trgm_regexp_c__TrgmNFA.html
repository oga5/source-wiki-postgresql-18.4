TrgmNFA

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_regexp.c:402

目的

正規表現からトリグラムベースの検索グラフを構築する過程で、解析状態、グラフ構造、および最適化情報を保持するための作業用構造体です。

シグネチャまたは形状

typedef struct {
    regex_t    *regex;
    TrgmColorInfo *colorInfo;
    int         ncolors;
    HTAB       *states;
    TrgmState  *initState;
    int         nstates;
    List       *queue;
    List       *keysQueue;
    int         arcsCount;
    bool        overflowed;
    ColorTrgmInfo *colorTrgms;
    int         colorTrgmsCount;
    int         totalTrgmCount;
} TrgmNFA;

入力と出力

  • **入力**: regex_t（コンパイル済みの正規表現）を保持し、変換プロセスを通じてグラフ構造へ展開します。
  • **出力**: 最終的に TrgmPackedGraph への変換元となるグラフ情報およびトリグラムの統計情報を提供します。

前提条件と不変条件

  • 正規表現の解析からパッキングまでの各ステージ（Stage 1: 色付け、Stage 2: グラフ展開、Stage 3: トリグラム選択）で段階的にデータが埋められます。
  • overflowed フラグが true の場合、複雑すぎる正規表現として解析が打ち切られたことを示します。

アルゴリズムまたは振る舞い

1. **解析**: regex から colorInfo を抽出し、正規表現を色（文字クラス）の集合に分解します。 2. **グラフ構築**: states ハッシュテーブルを用いて NFA をトリグラムをラベルとするグラフへ展開します。 3. **最適化**: colorTrgms を用いて、検索効率を最大化するために不要なトリグラムの削除やマージを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: createTrgmNFA()（直接の言及はないが、本構造体を利用する主要な公開API）
  • **呼び出し先**: 以下の関数が本構造体を引数として操作します。
  • getColorInfo()
  • transformGraph()
  • processState()
  • addKey() / addKeyToQueue()
  • addArcs() / addArc()
  • getState()
  • selectColorTrigrams()
  • expandColorTrigrams()
  • packGraph()

関連データ構造

  • TrgmState: グラフのノード。
  • TrgmStateKey: 状態遷移のキー。
  • ColorTrgmInfo: グラフ内で使用されるトリグラムの統計情報。
  • TrgmPackedGraph: 本構造体から生成される最終的なコンパクトなグラフ表現。

失敗モードと境界条件

  • overflowed: グラフのノード数やトリグラム数が上限（MAX_EXPANDED_STATES, MAX_TRGM_COUNT）を超えた場合、インデックス検索を諦めるためのフラグが立ちます。

保守担当者向けメモ

  • 本構造体はメモリコンテキストに依存するため、生成および破棄の際は適切なメモリ管理が必要です。
  • 変換過程で HTAB や List を使用しているため、複雑な正規表現ではメモリ消費量が増大する可能性があります。
