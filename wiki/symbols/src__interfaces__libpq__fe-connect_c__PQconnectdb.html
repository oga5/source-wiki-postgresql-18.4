PQconnectdb

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/libpq/fe-connect.c:819

目的

PostgreSQL データベースへの同期的な接続を確立し、接続状態を保持する PGconn オブジェクトを返します。

シグネチャまたは形状

PGconn *PQconnectdb(const char *conninfo)

入力と出力

  • **入力**: conninfo (接続文字列。URI 形式または key=value 形式)
  • **出力**: 接続状態を管理する PGconn 構造体へのポインタ。接続失敗時も PGconn オブジェクトが返されるため、呼び出し元は PQstatus で接続の成否を確認する必要があります。

前提条件と不変条件

  • conninfo は有効な接続文字列である必要があります。
  • 戻り値の PGconn が NULL でない場合、呼び出し元は最終的に PQfinish を呼び出してメモリを解放する責任があります。

アルゴリズムまたは振る舞い

1. PQconnectStart(conninfo) を呼び出し、接続プロセスを開始して PGconn オブジェクトを初期化します。 2. 取得した conn が有効であり、かつ接続状態が CONNECTION_BAD でない場合、pqConnectDBComplete(conn) を呼び出します。 3. pqConnectDBComplete は、接続が完了（または失敗）するまでブロッキング待機し、認証を含むネゴシエーションを完結させます。 4. 最終的な PGconn オブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_rewind/pg_rewind.c
  • src/bin/pg_upgrade/server.c
  • **呼び出し先**:
  • PQconnectStart
  • pqConnectDBComplete

関連データ構造

  • PGconn: 接続のステータス、パラメータ、ソケット情報などを保持する主要な構造体。

失敗モードと境界条件

  • 接続文字列のパース失敗やメモリ不足により PQconnectStart が NULL を返す場合があります。
  • ネットワークエラーや認証失敗が発生した場合、PGconn のステータスは CONNECTION_BAD となります。この場合でも PQconnectdb はオブジェクトを返すため、呼び出し側での PQstatus チェックが必須です。

保守担当者向けメモ

  • 本関数は同期的な接続処理のラッパーです。非同期接続が必要な場合は PQconnectStart と PQconnectPoll を直接使用してください。
  • 内部で呼び出される pqConnectDBComplete はブロッキング待機を行うため、GUI アプリケーション等でメインスレッドから呼び出す際は注意が必要です。
