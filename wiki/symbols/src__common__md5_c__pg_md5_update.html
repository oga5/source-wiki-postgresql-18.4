pg_md5_update

定義

  • src/common/md5.c:399

目的

MD5 ハッシュ計算において、入力データストリームをコンテキストに蓄積し、64バイトのブロック単位でハッシュ計算を進行させる。

シグネチャまたは形状

void pg_md5_update(pg_md5_ctx *ctx, const uint8 *data, size_t len)

入力と出力

  • ctx: MD5 計算状態を保持する pg_md5_ctx 構造体へのポインタ。
  • data: ハッシュ対象の入力データバイト列。
  • len: 入力データの長さ。
  • 出力: なし（ctx 内部の状態が更新される）。

前提条件と不変条件

  • ctx は pg_md5_init によって初期化済みであること。
  • ctx->md5_n は処理された全ビット数を保持し、入力 len に基づいて更新される。

アルゴリズムまたは振る舞い

1. 入力長 len をビット単位に換算し、ctx->md5_n に加算する。 2. 内部バッファ ctx->md5_buf の空き容量（gap）を確認する。 3. 入力データがバッファを埋めるのに十分な場合:

  • バッファの残りを埋めて md5_calc を呼び出し、ブロックを処理する。
  • 残りの入力データを 64 バイト単位で順次 md5_calc に渡す。
  • 端数分をバッファにコピーし、ctx->md5_i を更新する。

    • 4. 入力データがバッファを埋めるのに足りない場合:

  • データをバッファの末尾に追加し、ctx->md5_i を更新する。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/cryptohash.c:144
  • 呼び出し先: md5_calc（内部関数）

関連データ構造

  • pg_md5_ctx: ハッシュ計算の状態（バッファ、ビット数、インデックス）を保持する構造体。

失敗モードと境界条件

  • len が 0 の場合、状態は変化しない。
  • 入力データが 64 バイトの倍数である場合、バッファは空の状態（ctx->md5_i == 0）で終了する。

保守担当者向けメモ

  • 本関数は RFC 1321 に準拠した低レベル実装であり、パフォーマンスよりも移植性を優先している。
  • 現代のセキュリティ要件が求められる用途には、SHA-256 等のより強力なアルゴリズムの使用を検討すること。
