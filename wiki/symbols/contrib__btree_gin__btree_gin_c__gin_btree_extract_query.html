gin_btree_extract_query

定義

  • 種別: function
  • 定義箇所: contrib/btree_gin/btree_gin.c:55

目的

GINインデックスにおいて、B-treeの比較演算子（<, <=, =, >=, >）を用いた検索条件を解析し、インデックススキャンを開始するためのキーおよびメタデータを生成する。

シグネチャまたは形状

static Datum gin_btree_extract_query(
    FunctionCallInfo fcinfo,
    bool is_varlena,
    Datum (*leftmostvalue) (void),
    Datum (*typecmp) (FunctionCallInfo)
)

入力と出力

  • **入力**:
  • fcinfo: GINサポート関数の呼び出し情報。
  • is_varlena: データ型が可変長かを示すフラグ。
  • leftmostvalue: 型ごとの最小値を返す関数ポインタ。
  • typecmp: 型ごとの比較関数ポインタ。
  • **出力**:
  • 戻り値: 検索キーの配列（Datum *）。
  • nentries: 抽出されたキーの数（1に設定）。
  • partialmatch: 部分一致フラグ（bool *）。
  • extra_data: QueryInfo 構造体へのポインタ。

前提条件と不変条件

  • nentries は常に 1 に設定される。
  • strategy が BTLessStrategyNumber 等の既知のB-tree戦略番号でない場合、エラーが発生する。
  • 可変長データ型の場合、datum はデトースト（detoast）される。

アルゴリズムまたは振る舞い

1. 引数から strategy（検索戦略）と datum（検索値）を取得する。 2. QueryInfo 構造体を割り当て、戦略、検索値、比較関数を格納し、extra_data に設定する。 3. strategy に基づき以下の処理を行う:

  • BTLess / BTLessEqual: leftmostvalue() を呼び出し、インデックスの開始点を最小値に設定。partialmatch を true にする。
  • BTGreater / BTGreaterEqual: partialmatch を true にし、検索値を datum に設定。
  • BTEqual: 検索値を datum に設定。

    • 4. 抽出したキー（entries）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gin/btree_gin.c 内の各データ型用ラッパー関数（例: gin_extract_query_[type]）。
  • **呼び出し先**: leftmostvalue 関数ポインタ経由で各型の最小値取得関数を呼び出す。

関連データ構造

  • QueryInfo: 検索戦略や比較ロジックを保持する構造体。
  • StrategyNumber: access/stratnum.h で定義される比較戦略定数。

失敗モードと境界条件

  • 未知の StrategyNumber が渡された場合、elog(ERROR) により処理が中断される。
  • メモリ割り当て（palloc）失敗時はPostgreSQLの標準的なエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数は GIN_SUPPORT マクロを通じて各データ型に適用される汎用ロジックである。
  • 新しいデータ型を追加する際は、適切な leftmostvalue 関数を提供することが必須である。
  • partialmatch はGINエンジンがインデックススキャンを継続するか判断するために使用される。
