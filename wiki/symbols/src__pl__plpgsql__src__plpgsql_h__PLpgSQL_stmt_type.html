PLpgSQL_stmt_type

定義

  • 種別: enum
  • 定義箇所: src/pl/plpgsql/src/plpgsql.h:103

目的

PL/pgSQL 言語における各文（Statement）の種類を識別するための列挙型です。抽象構文木（AST）のノードにおいて、実行エンジンが処理すべき文のタイプを特定するために使用されます。

シグネチャまたは形状

typedef enum PLpgSQL_stmt_type {
    PLPGSQL_STMT_BLOCK,
    PLPGSQL_STMT_ASSIGN,
    PLPGSQL_STMT_IF,
    PLPGSQL_STMT_CASE,
    PLPGSQL_STMT_LOOP,
    PLPGSQL_STMT_WHILE,
    PLPGSQL_STMT_FORI,
    PLPGSQL_STMT_FORS,
    PLPGSQL_STMT_FORC,
    PLPGSQL_STMT_FOREACH_A,
    PLPGSQL_STMT_EXIT,
    PLPGSQL_STMT_RETURN,
    PLPGSQL_STMT_RETURN_NEXT,
    PLPGSQL_STMT_RETURN_QUERY,
    PLPGSQL_STMT_RAISE,
    PLPGSQL_STMT_ASSERT,
    PLPGSQL_STMT_EXECSQL,
    PLPGSQL_STMT_DYNEXECUTE,
    PLPGSQL_STMT_DYNFORS,
    PLPGSQL_STMT_GETDIAG,
    PLPGSQL_STMT_OPEN,
    PLPGSQL_STMT_FETCH,
    PLPGSQL_STMT_CLOSE,
    PLPGSQL_STMT_PERFORM,
    PLPGSQL_STMT_CALL,
    PLPGSQL_STMT_COMMIT,
    PLPGSQL_STMT_ROLLBACK
} PLpgSQL_stmt_type;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: PLpgSQL_stmt 構造体内の cmd_type メンバとして保持されます。

前提条件と不変条件

  • PLpgSQL_stmt 構造体（およびその派生構造体）の先頭メンバとして配置され、実行時にノードの型を判別するために使用されます。
  • 列挙値は、PL/pgSQL のコンパイル時（pl_comp.c）に決定され、実行時（pl_exec.c）まで不変です。

アルゴリズムまたは振る舞い

実行エンジン（pl_exec.c）は、この列挙値に基づいて switch 文やディスパッチテーブルを構成し、各文に対応する実行ロジックへ制御を分岐させます。

呼び出し元と呼び出し先

  • 呼び出し元: pl_comp.c（構文解析時にノードへ値を設定）、pl_exec.c（実行時にノードの型を判定）。
  • 呼び出し先: なし（本シンボルは型定義であり、関数ではない）。

関連データ構造

  • PLpgSQL_stmt: 全ての文の基底構造体。cmd_type メンバが本列挙型を参照します。

失敗モードと境界条件

  • 該当なし（定義体のため）。ただし、未定義の列挙値が cmd_type に代入された場合、実行エンジン側で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しい文の種類（例: 新しい SQL コマンドのサポート）を追加する場合は、本列挙型に値を追加し、pl_exec.c の実行ループ（exec_stmt 等）で対応する処理を実装する必要があります。
