toast_compress_header

定義

  • 種別: struct
  • 定義箇所: src/include/access/toast_internals.h:23

目的

圧縮されたTOASTデータの先頭に付与されるメタデータ構造体です。元のデータサイズと使用された圧縮アルゴリズムの情報を保持します。

シグネチャまたは形状

typedef struct toast_compress_header
{
    int32       vl_len_;    /* varlena header */
    uint32      tcinfo;     /* 圧縮手法(2bit)と外部サイズ(30bit) */
} toast_compress_header;

入力と出力

本構造体はデータ構造であり、関数ではありません。tcinfo フィールドを通じて、圧縮手法の識別子および元のデータサイズをエンコード・デコードするために使用されます。

前提条件と不変条件

  • vl_len_ は varlena 構造体の一部であるため、直接操作してはなりません。
  • tcinfo の上位2ビットは圧縮手法、下位30ビットは外部サイズを保持します。

アルゴリズムまたは振る舞い

TOAST圧縮データが読み込まれる際、このヘッダーを参照して圧縮手法を特定し、データを展開（detoast）します。また、圧縮時には TOAST_COMPRESS_SET_SIZE_AND_COMPRESS_METHOD マクロを用いて、サイズと手法を tcinfo に書き込みます。

呼び出し元と呼び出し先

  • 呼び出し元: toast_compress_datum（圧縮時）、およびTOASTデータの展開処理を行う関数群。
  • 呼び出し先: 提供されたソース情報からは、特定の関数呼び出し関係は特定できません。

関連データ構造

  • varlena: PostgreSQLの可変長データ型ヘッダー。
  • toast_compress_datum: 本構造体を利用して圧縮データを生成する関数。

失敗モードと境界条件

  • tcinfo のビット操作において、定義されたマスク（VARLENA_EXTSIZE_MASK）を超えた値を書き込むとデータ破損の原因となります。

保守担当者向けメモ

  • 本構造体は内部APIです。直接のフィールド操作は避け、提供されているマクロ（src/include/access/toast_internals.h 内で定義）を介してアクセスしてください。
