WalSndWakeupProcessRequests

定義

  • 種別: static inline function
  • 定義箇所: src/include/replication/walsender.h:64

目的

保留されているWAL Senderの起動要求を処理し、必要に応じて実際のウェイクアップ処理を実行する。

シグネチャまたは形状

static inline void WalSndWakeupProcessRequests(bool physical, bool logical)

入力と出力

  • **入力**:
  • physical (bool): 物理レプリケーションのウェイクアップを許可するかどうか。
  • logical (bool): 論理レプリケーションのウェイクアップを許可するかどうか。
  • **出力**: なし。

前提条件と不変条件

  • グローバル変数 wake_wal_senders が true の場合にのみ、ウェイクアップ処理が試行される。
  • max_wal_senders > 0 であることが、WalSndWakeup を呼び出すための条件である。

アルゴリズムまたは振る舞い

1. グローバルフラグ wake_wal_senders を確認する。 2. フラグが true であれば、即座に false にリセットする。 3. max_wal_senders が 0 より大きい場合、WalSndWakeup(physical, logical) を呼び出し、待機中のWAL Senderプロセスを起床させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c (2箇所): XLogFlush 等の処理に関連して呼び出される。
  • **呼び出し先**:
  • WalSndWakeup (関数): 実際のプロセス通知処理を実行する。

関連データ構造

  • wake_wal_senders (グローバル変数): ウェイクアップ要求の有無を示すフラグ。
  • max_wal_senders (グローバル変数): 設定可能なWAL Senderの最大数。

失敗モードと境界条件

  • max_wal_senders が 0 の場合、要求フラグはリセットされるが、WalSndWakeup は呼び出されない。
  • wake_wal_senders が false の場合、本関数は何も行わず即座に復帰する。

保守担当者向けメモ

  • 本関数はロック競合を回避するため、WalSndWakeupRequest マクロでフラグを立てた後、適切なタイミングで呼び出される設計となっている。
  • src/backend/replication/walsender.c:3234 のコメントによれば、XLogFlush() 実行後に呼び出されることが想定されている。
