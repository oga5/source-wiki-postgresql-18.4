role_auth_extra

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/variable.c:807

目的

SESSION AUTHORIZATION および ROLE 設定の変更時、GUC（Grand Unified Configuration）サブシステム間で受け渡される「追加状態（extra state）」を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	Oid			roleid;
	bool		is_superuser;
} role_auth_extra;

入力と出力

  • **入力**: GUCの check_hook 実行時に guc_malloc を介して割り当てられ、検証済みのロールIDとスーパーユーザー権限フラグが格納されます。
  • **出力**: GUCの assign_hook に渡され、セッション状態の更新に使用されます。

前提条件と不変条件

  • check_hook 内で guc_malloc を使用してメモリを確保する必要があります。
  • 確保されたメモリはGUCサブシステムによって管理されます。

アルゴリズムまたは振る舞い

1. SET コマンド実行時、check_hook が値を検証し、成功時に role_auth_extra 構造体を生成して extra 引数に格納します。 2. assign_hook が呼び出されると、extra 引数から本構造体をキャストして取り出し、保持されている roleid と is_superuser を用いてセッションの認証状態を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/variable.c 内の check_session_authorization、assign_session_authorization、check_role、assign_role 等のGUCフック関数。
  • **呼び出し先**: 特定の関数呼び出しは確認されていませんが、GUCサブシステムのメモリ管理関数（guc_malloc）が本構造体の確保に使用されます。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • メモリ確保失敗時、guc_malloc がエラーを発生させる可能性があります。
  • extra 引数が期待される型（role_auth_extra *）と異なる場合、キャスト後のアクセスで未定義の動作が発生します。

保守担当者向けメモ

  • 本構造体はGUCサブシステムを介して受け渡されるため、メモリリークを防ぐ設計になっています。check_hook 内で割り当てたメモリの解放はGUCフレームワークに依存します。
