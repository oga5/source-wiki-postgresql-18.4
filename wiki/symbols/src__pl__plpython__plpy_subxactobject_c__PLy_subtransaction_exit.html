PLy_subtransaction_exit

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_subxactobject.c:146

目的

Pythonのコンテキストマネージャ（with文）の終了処理として、PostgreSQLの内部サブトランザクションをコミットまたはロールバックし、関連するリソースを解放する。

シグネチャまたは形状

static PyObject *PLy_subtransaction_exit(PyObject *self, PyObject *args)
  • self: PLySubtransactionObject インスタンス。
  • args: Pythonの __exit__ メソッドに渡される (type, value, traceback) タプル。

入力と出力

  • 入力: args タプル（例外情報を含む）。
  • 出力: 成功時は Py_None を返し、失敗時は NULL を返してPython例外をセットする。

前提条件と不変条件

  • self は PLySubtransactionObject 型であること。
  • subxact->started が真であること（未開始のサブトランザクションは終了不可）。
  • subxact->exited が偽であること（二重終了は不可）。
  • explicit_subtransactions リストが空でないこと。

アルゴリズムまたは振る舞い

1. PyArg_ParseTuple で引数を解析する。 2. 状態チェック（開始済みか、終了済みでないか、リストが空でないか）を行い、違反時は ValueError を送出する。 3. subxact->exited を真に設定する。 4. type 引数が None であれば ReleaseCurrentSubTransaction() を呼び出し、それ以外（例外発生時）は RollbackAndReleaseCurrentSubTransaction() を呼び出す。 5. explicit_subtransactions リストから先頭の PLySubtransactionData を取り出し、リストを更新する。 6. 保存されていた oldcontext にメモリコンテキストを切り替え、oldowner にリソースオーナーを復元する。 7. PLySubtransactionData をメモリから解放する。

呼び出し元と呼び出し先

  • 呼び出し元: Pythonの with 文終了時に、__exit__ メソッド経由で呼び出される。
  • 呼び出し先:
  • RollbackAndReleaseCurrentSubTransaction / ReleaseCurrentSubTransaction
  • MemoryContextSwitchTo
  • list_delete_first / linitial
  • pfree

関連データ構造

  • PLySubtransactionObject: Pythonオブジェクトとしてのサブトランザクション状態を保持。
  • PLySubtransactionData: トランザクション開始時のメモリコンテキストとリソースオーナーを保持する構造体。
  • explicit_subtransactions: ネストされたサブトランザクションを管理するグローバルなリスト。

失敗モードと境界条件

  • started が偽の場合: ValueError を送出。
  • exited が既に真の場合: ValueError を送出。
  • explicit_subtransactions が空の場合: ValueError を送出。

保守担当者向けメモ

  • 本関数はPythonのコンテキストマネージャプロトコルに依存している。
  • explicit_subtransactions リストの操作は、PostgreSQLのトランザクションスタックと同期している必要がある。
  • メモリコンテキストの復元順序は、サブトランザクションのネスト構造を維持するために重要である。
