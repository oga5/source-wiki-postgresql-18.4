tuplestore_get_stats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplestore.c:1546

目的

Tuplestorestate が使用したストレージの最大量およびその種別（メモリまたはディスク）を取得します。

シグネチャまたは形状

void tuplestore_get_stats(Tuplestorestate *state, char **max_storage_type, int64 *max_space)

入力と出力

  • state: 統計情報を取得する対象の Tuplestorestate 構造体へのポインタ。
  • max_storage_type: 出力パラメータ。ストレージがディスクに及んだ場合は "Disk"、メモリ内に収まった場合は "Memory" という文字列リテラルへのポインタが格納されます。
  • max_space: 出力パラメータ。state->maxSpace に記録された最大使用量が格納されます。

前提条件と不変条件

  • state は有効な Tuplestorestate 構造体を指している必要があります。
  • 内部的に tuplestore_updatemax(state) を呼び出し、現在の状態を反映させてから統計情報を抽出します。

アルゴリズムまたは振る舞い

1. tuplestore_updatemax(state) を呼び出し、現在のメモリ/ディスク使用量を最新の状態に更新します。 2. state->usedDisk フラグを確認し、max_storage_type に適切な文字列を設定します。 3. state->maxSpace の値を max_space に代入して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c (EXPLAIN コマンド実行時に統計情報を表示するために複数箇所から呼び出されています)
  • **呼び出し先**:
  • tuplestore_updatemax (同ファイル内)

関連データ構造

  • Tuplestorestate: タプル格納の状態を保持する構造体。usedDisk および maxSpace メンバが本関数で参照されます。

失敗モードと境界条件

  • state が NULL の場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数は EXPLAIN による実行計画の統計表示を主目的としています。
  • max_storage_type に返される文字列は静的リテラルであるため、呼び出し側で解放する必要はありません。
