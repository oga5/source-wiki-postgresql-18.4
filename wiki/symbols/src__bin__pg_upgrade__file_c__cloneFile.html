cloneFile

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/file.c:38

目的

OSがサポートするreflink（ファイルクローン）機能を利用し、データディレクトリ間でファイルを高速に複製します。

シグネチャまたは形状

void cloneFile(const char *src, const char *dst, const char *schemaName, const char *relName)

入力と出力

  • **入力**:
  • src: コピー元ファイルパス
  • dst: コピー先ファイルパス
  • schemaName: エラーログ出力用のスキーマ名
  • relName: エラーログ出力用のリレーション名
  • **出力**: なし（成功時はファイルを複製、失敗時は pg_fatal によりプロセスを終了）

前提条件と不変条件

  • コンパイル環境において HAVE_COPYFILE (macOS等) または FICLONE (Linux) が定義されている必要があります。
  • dst ファイルは存在しない状態である必要があります（O_EXCL フラグによる排他制御）。

アルゴリズムまたは振る舞い

1. **プラットフォーム別の分岐**:

  • HAVE_COPYFILE かつ COPYFILE_CLONE_FORCE が定義されている場合: copyfile() API を呼び出します。
  • __linux__ かつ FICLONE が定義されている場合:
  • src を読み取り専用、dst を書き込み専用（新規作成）で開きます。
  • ioctl に FICLONE リクエストを送り、ファイルシステムレベルでのクローンを実行します。
  • 成功後、ファイル記述子を閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/relfilenumber.c:625
  • **呼び出し先**:
  • open, ioctl, unlink, close (OSシステムコール)
  • pg_fatal (ユーティリティ関数)

関連データ構造

  • pg_file_create_mode: ファイル作成時のパーミッション定義（common/file_perm.h 由来）

失敗モードと境界条件

  • ファイルのオープン失敗（権限不足、パスの誤り等）
  • ioctl によるクローン操作の拒否（ファイルシステムがreflinkをサポートしていない場合など）
  • dst ファイルが既に存在する場合（O_EXCL により失敗）

保守担当者向けメモ

  • 本関数は #ifdef によるプラットフォーム依存の実装です。新しいOSやファイルシステムへの対応が必要な場合は、適切なシステムコールを追加してください。
  • エラー発生時に pg_fatal を使用しているため、呼び出し側でエラーをキャッチして継続する設計にはなっていません。
