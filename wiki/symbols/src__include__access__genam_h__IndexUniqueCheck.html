IndexUniqueCheck

定義

  • 種別: enum
  • 定義箇所: src/include/access/genam.h:138

目的

インデックスへのタプル挿入時における一意性制約のチェックモードを定義し、制御するために使用されます。

シグネチャまたは形状

typedef enum IndexUniqueCheck
{
    UNIQUE_CHECK_NO,        /* 一意性チェックを行わない */
    UNIQUE_CHECK_YES,       /* 挿入時に一意性を強制する */
    UNIQUE_CHECK_PARTIAL,   /* 一意性をテストするが、エラーは発生させない */
    UNIQUE_CHECK_EXISTING,  /* 既存のタプルが一意であるかを確認する */
} IndexUniqueCheck;

入力と出力

  • 本シンボルは列挙型であり、関数への引数として渡されることで、インデックス挿入処理の挙動を切り替えます。

前提条件と不変条件

  • 挿入処理を行う各インデックスアクセス手法（AM）は、この列挙値に基づいて一意性チェックの要否およびエラーハンドリングを決定する必要があります。

アルゴリズムまたは振る舞い

  • UNIQUE_CHECK_NO: チェックをスキップします。
  • UNIQUE_CHECK_YES: 重複が検出された場合にエラーを発生させます。
  • UNIQUE_CHECK_PARTIAL: 重複の有無を検証しますが、エラーは送出しません。
  • UNIQUE_CHECK_EXISTING: 既に存在するタプルに対して一意性の検証を行います。

  • **呼び出し先**: 本シンボルは列挙型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • IndexInfo: インデックスのメタデータ構造体（本列挙値と組み合わせて利用されることが多い）。

失敗モードと境界条件

  • UNIQUE_CHECK_YES が指定された場合、重複タプルが存在すると挿入処理はエラーで中断されます。

保守担当者向けメモ

  • 新しいインデックスアクセス手法を実装する際は、checkUnique 引数として本列挙値を受け取り、各モードに応じた適切な処理を実装してください。
