xl_heap_lock

定義

  • 種別: struct
  • 定義箇所: src/include/access/heapam_xlog.h:396

目的

ヒープテーブル内のタプルに対するロック操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_heap_lock
{
    TransactionId xmax;         /* might be a MultiXactId */
    OffsetNumber offnum;        /* locked tuple's offset on page */
    uint8        infobits_set;  /* infomask and infomask2 bits to set */
    uint8        flags;         /* XLH_LOCK_* flag bits */
} xl_heap_lock;

入力と出力

  • **入力**: heapam.c におけるロック操作時に、この構造体に各フィールドが設定され、WALレコードとして書き込まれます。
  • **出力**: リカバリ（REDO）処理時に XLogRecGetData を通じて読み出され、対象ページのタプル状態を更新するために使用されます。

前提条件と不変条件

  • xmax は通常の TransactionId または MultiXactId を保持します。
  • offnum は対象ページ内の有効なタプルオフセットを指す必要があります。
  • SizeOfHeapLock マクロにより、構造体のサイズ（flags フィールドまで）が定義されています。

アルゴリズムまたは振る舞い

1. **記録**: ヒープ上のタプルをロックする際、heapam.c が本構造体を初期化し、WALレコードの一部として挿入します。 2. **再生**: リカバリ時、heap_redo（または関連するREDO関数）がレコードから本構造体を復元します。 3. **適用**: 復元された infobits_set を用いて、対象ページのタプルヘッダーのフラグを更新し、ロック状態を再現します。

呼び出し元と呼び出し先

  • **呼び出し元（記録側）**: src/backend/access/heap/heapam.c 内のロック関連処理（heap_lock_tuple 等）。
  • **呼び出し先（再生側）**: src/backend/access/heap/heapam_xlog.c および src/backend/access/rmgrdesc/heapdesc.c が本構造体を読み取ります。

関連データ構造

  • TransactionId: トランザクション識別子。
  • OffsetNumber: ページ内のタプル位置。
  • XLH_LOCK_*: flags フィールドで使用されるフラグビット群。

失敗モードと境界条件

  • 構造体サイズが SizeOfHeapLock と一致しない場合、WALレコードのデコード時に不整合が生じる可能性があります。
  • offnum がページ内の無効な位置を指している場合、REDO適用時にページ破損を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は xl_heap_lock_updated とフラグ定義を共有しています。変更を加える際は src/include/access/heapam_xlog.h:392 付近の定義を確認してください。
  • infobits_set はタプルの infomask および infomask2 を直接操作するため、ビット定義の変更には細心の注意が必要です。
