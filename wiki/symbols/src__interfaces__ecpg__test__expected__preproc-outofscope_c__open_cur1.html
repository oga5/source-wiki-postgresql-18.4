open_cur1

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-outofscope.c:208

目的

ECPGプリプロセッサの回帰テストにおいて、mycur という名称のカーソルを宣言し、データベースからデータを取得するための準備を行う。

シグネチャまたは形状

static void open_cur1(void)

入力と出力

  • 入力: なし（内部的に ECPGget_var を介して外部スコープの変数へアクセスする）
  • 出力: なし（副作用としてカーソルをオープンする）

前提条件と不変条件

  • ECPGget_var を通じて、MYTYPE および MYNULLTYPE 構造体へのポインタが正しく登録されている必要がある。
  • データベース接続が確立されていること。

アルゴリズムまたは振る舞い

1. ECPGdo マクロを使用して SQL 文 declare mycur cursor for select * from a1 を実行する。 2. 構造体 MYTYPE（データ保持用）および MYNULLTYPE（NULLインジケータ用）の各フィールドをバインド変数として登録する。 3. sqlca.sqlcode を確認し、エラー（負の値）が発生した場合は exit(1) を呼び出してプロセスを終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/preproc-outofscope.c 内の main 関数（行209および行326）。
  • 呼び出し先: ECPGランタイムライブラリの ECPGdo および ECPGget_var。

関連データ構造

  • MYTYPE: データベースの行データを保持する構造体。
  • MYNULLTYPE: 各フィールドのNULL状態を追跡するインジケータ構造体。
  • sqlca: SQL通信領域（エラーコード確認に使用）。

失敗モードと境界条件

  • sqlca.sqlcode < 0: SQL実行エラーが発生した場合、即座にプログラムを終了する。
  • ECPGget_var が適切なインデックスで初期化されていない場合、ランタイムエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はプリプロセッサによって生成されたコードである。SQL文の変更やバインド変数の追加を行う場合は、元の .pgc ファイルを修正する必要がある。
  • ECPGget_var の引数（0や1）は、変数の登録順序に依存しているため、変更時には注意が必要である。
