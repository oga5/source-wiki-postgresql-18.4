get_policies_for_relation

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rowsecurity.c:540

目的

指定されたリレーションとコマンド種別に基づき、適用すべき行レベルセキュリティ（RLS）ポリシーを収集し、Permissive（許可）と Restrictive（制限）のリストに分類する。

シグネチャまたは形状

static void get_policies_for_relation(Relation relation, CmdType cmd, Oid user_id,
                                      List **permissive_policies,
                                      List **restrictive_policies)

入力と出力

  • **入力**:
  • relation: 対象のリレーション（rd_rsdesc を保持）。
  • cmd: 実行対象のコマンド種別（CMD_SELECT, CMD_INSERT 等）。
  • user_id: ポリシー適用判定に使用する現在のロール ID。
  • **出力**:
  • permissive_policies: 適用可能な Permissive ポリシーのリスト（ポインタ経由で返却）。
  • restrictive_policies: 適用可能な Restrictive ポリシーのリスト（ポインタ経由で返却）。

前提条件と不変条件

  • relation->rd_rsdesc が有効であること。
  • Restrictive ポリシーは、生成される WithCheckOptions（WCO）の順序を確定させるため、名前順にソートされる。
  • 拡張モジュールによるフック（row_security_policy_hook_restrictive / permissive）が定義されている場合、組み込みポリシーの後にそれらが追加される。

アルゴリズムまたは振る舞い

1. permissive_policies と restrictive_policies を空のリストで初期化する。 2. リレーションに定義されたポリシーを走査し、コマンド種別（* は全コマンド対象）とロールが一致するものを抽出する。 3. 抽出した Restrictive ポリシーを sort_policies_by_name でソートする。 4. 拡張モジュール用のフックが存在する場合、フックから取得したポリシーを同様にフィルタリングし、Restrictive ポリシーについてはソートした上でリストに追加する。 5. MERGE コマンドについては個別のポリシーを持たず、他のコマンドのポリシーから派生する前提で処理される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rowsecurity.c 内の各リライト処理関数（get_row_security_policies 等）。
  • **呼び出し先**:
  • check_role_for_policy（ロール判定）
  • sort_policies_by_name（ポリシーのソート）
  • row_security_policy_hook_restrictive / row_security_policy_hook_permissive（拡張モジュールフック）

関連データ構造

  • Relation: リレーションキャッシュ構造体。
  • RowSecurityPolicy: ポリシー定義を保持する構造体。
  • List: ポリシーを格納するリスト構造。

失敗モードと境界条件

  • 未知の cmd 型が渡された場合、elog(ERROR) により処理を中断する。
  • MERGE コマンドが渡された場合、明示的なポリシー検索は行われない。

保守担当者向けメモ

  • 組み込みの Restrictive ポリシーは、フックによって追加されたポリシーよりも先にチェックされる順序が保証されている。
  • MERGE コマンドの挙動は、他のコマンドのポリシー定義に依存するため、ポリシー追加時には注意が必要。
