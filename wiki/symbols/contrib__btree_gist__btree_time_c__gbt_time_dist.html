gbt_time_dist

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_time.c:110

目的

2つの TimeADT 値の間の時間的距離を計算し、float8 型の秒数として返します。

シグネチャまたは形状

static float8 gbt_time_dist(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の TimeADT 値へのポインタ。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されず、DirectFunctionCall2 に渡されます）。
  • **出力**:
  • 2つの時刻間の差分を秒単位に変換した float8 値。

前提条件と不変条件

  • 入力ポインタ a および b は TimeADT 型としてキャスト可能である必要があります。
  • 内部で time_mi_time 関数を呼び出すため、PostgreSQLの時刻演算環境が整っている必要があります。

アルゴリズムまたは振る舞い

1. 入力された a と b を TimeADT ポインタにキャストします。 2. DirectFunctionCall2 を介して time_mi_time を呼び出し、2つの時刻の差分を Interval 型として取得します。 3. INTERVAL_TO_SEC マクロを使用して、取得した Interval を秒単位の数値に変換します。 4. fabs を適用し、差分の絶対値を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • time_mi_time: 時刻の減算を行うPostgreSQL内部関数。
  • INTERVAL_TO_SEC: Interval を秒に変換するマクロ。
  • fabs: 絶対値を計算する標準ライブラリ関数。
  • **呼び出し元**:
  • 提供されたソースコード内では、具体的な呼び出し元は明示されていません。

関連データ構造

  • TimeADT: PostgreSQLの時刻型（内部表現）。
  • Interval: 時刻の差分を表す構造体。

失敗モードと境界条件

  • time_mi_time がエラーを発生させた場合、本関数も例外をスローします。
  • 時刻の差が非常に大きい場合や、Interval 変換時にオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_time.c 内の他のGiSTサポート関数と同様に、TimeADT を数値的に扱うための補助的な役割を担います。
  • timetz 型の距離計算には別の関数が使用される可能性があるため、本関数が time 型専用である点に注意してください。
