RangeVarGetAndCheckCreationNamespace

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/namespace.c:738

目的

オブジェクト作成先となる名前空間（スキーマ）を特定し、そのスキーマに対する CREATE 権限の確認および、既存オブジェクトの存在確認とロック取得を安全に行う。

シグネチャまたは形状

Oid RangeVarGetAndCheckCreationNamespace(RangeVar *relation, LOCKMODE lockmode, Oid *existing_relation_id)

入力と出力

  • **入力**:
  • relation: 対象オブジェクトの RangeVar 構造体。
  • lockmode: 既存オブジェクトに対して取得するロックモード（NoLock 指定可）。
  • existing_relation_id: 既存オブジェクトの OID を格納するポインタ（不要な場合は NULL）。
  • **出力**:
  • 戻り値: 作成先スキーマの OID。
  • existing_relation_id 引数経由: 既存オブジェクトの OID（存在しない場合は InvalidOid）。

前提条件と不変条件

  • BootstrapProcessingMode の場合は権限チェックやロックをスキップする。
  • 処理中に並行して DDL が実行された場合、SharedInvalidMessageCounter を用いて整合性を再確認する。

アルゴリズムまたは振る舞い

1. RangeVarGetCreationNamespace を呼び出し、作成先スキーマを特定する。 2. existing_relation_id が指定されている場合、対象スキーマ内で同名オブジェクトを検索する。 3. ループ内で以下を繰り返す（並行 DDL 対策）:

  • 名前空間に対する ACL_CREATE 権限をチェックする。
  • 状態に変更があれば、古いロックを解放し、新しい対象に対してロックを取得する。
  • SharedInvalidMessageCounter を監視し、カウンターが変化していれば再試行する。

    • 4. 最後に RangeVarAdjustRelationPersistence を呼び出し、永続性を調整する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/sequence.c, tablecmds.c, typecmds.c, view.c, src/backend/parser/parse_utilcmd.c 等。
  • **呼び出し先**: RangeVarGetCreationNamespace, get_relname_relid, object_aclcheck, LockDatabaseObject, LockRelationOid, RangeVarAdjustRelationPersistence 等。

関連データ構造

  • RangeVar: オブジェクトの識別情報（カタログ名、スキーマ名、リレーション名）。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • **エラー**:
  • クロスデータベース参照が指定された場合（ERRCODE_FEATURE_NOT_SUPPORTED）。
  • 対象スキーマに対する CREATE 権限がない場合。
  • 既存オブジェクトに対して所有者権限がない場合（ACLCHECK_NOT_OWNER）。
  • **境界条件**:
  • existing_relation_id が NULL の場合、既存オブジェクトの検索は行われない。
  • lockmode が NoLock の場合、リレーションのロックは取得されない。

保守担当者向けメモ

  • 本関数は RangeVarGetRelidExtended と同様の並行 DDL 対策ロジックを採用している。修正時は SharedInvalidMessageCounter の監視ループを維持すること。
