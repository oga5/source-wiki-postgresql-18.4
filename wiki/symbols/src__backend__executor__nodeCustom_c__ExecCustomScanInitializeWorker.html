ExecCustomScanInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeCustom.c:204

目的

並列クエリ実行時、各ワーカープロセスにおいてカスタムスキャンノードの初期化処理を拡張機能側へ委譲する。

シグネチャまたは形状

void ExecCustomScanInitializeWorker(CustomScanState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • node: 初期化対象の CustomScanState 構造体。
  • pwcxt: 並列ワーカーのコンテキスト情報。
  • 出力なし（戻り値 void）。

前提条件と不変条件

  • node->methods が有効な CustomExecMethods を指していること。
  • 拡張機能が InitializeWorkerCustomScan コールバックを実装している場合のみ、その関数が呼び出される。

アルゴリズムまたは振る舞い

1. node->methods からカスタム実行メソッドテーブルを取得する。 2. InitializeWorkerCustomScan が定義されているか確認する。 3. 定義されている場合、node->ss.ps.plan->plan_node_id をキーとして、pwcxt->toc（共有メモリのテーブル・オブ・コンテンツ）から共有データ領域（coordinate）を検索する。 4. 取得した coordinate を引数に渡し、拡張機能側の InitializeWorkerCustomScan を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1370
  • **呼び出し先**: 拡張機能が提供する CustomExecMethods 内の InitializeWorkerCustomScan 関数。

関連データ構造

  • CustomScanState: カスタムスキャンの実行状態を保持する構造体。
  • CustomExecMethods: 拡張機能が実装するコールバック関数群。
  • ParallelWorkerContext: 並列ワーカーの共有メモリ管理用コンテキスト。

失敗モードと境界条件

  • methods->InitializeWorkerCustomScan が定義されていない場合、本関数は何もしない。
  • shm_toc_lookup で指定された plan_node_id が見つからない場合、shm_toc_lookup の第3引数が false であるためエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、拡張機能が並列クエリをサポートするために必要なDSM（動的共有メモリ）の同期ポイントの一つである。
  • 拡張機能側で InitializeWorkerCustomScan を実装する際は、shm_toc_lookup で取得したポインタが、Estimate および InitializeDSM フェーズで確保・初期化したメモリ領域と一致していることを確認すること。
