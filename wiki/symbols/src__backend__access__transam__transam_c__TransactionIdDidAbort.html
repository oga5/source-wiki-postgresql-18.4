TransactionIdDidAbort

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:187

目的

指定されたトランザクションID（XID）がアボート（中止）された状態であるかを判定する。

シグネチャまたは形状

bool TransactionIdDidAbort(TransactionId transactionId)

入力と出力

  • **入力**: transactionId (判定対象のトランザクションID)
  • **出力**: bool (対象がアボート済みであれば true、そうでなければ false)

前提条件と不変条件

  • TransactionLogFetch を介して CLOG から状態を取得する。
  • サブトランザクションの場合、再帰的に親トランザクションの状態を辿る。
  • TransactionXmin より古いトランザクションが SUB_COMMITTED 状態である場合、親トランザクションの追跡が不可能なため、安全側に倒して true（アボート済み）を返す。

アルゴリズムまたは振る舞い

1. TransactionLogFetch(transactionId) を呼び出し、現在のステータスを取得する。 2. ステータスが TRANSACTION_STATUS_ABORTED であれば true を返す。 3. ステータスが TRANSACTION_STATUS_SUB_COMMITTED の場合:

  • transactionId が TransactionXmin より古い場合は true を返す。
  • SubTransGetParent で親XIDを取得する。
  • 親XIDが無効な場合は警告を出力し true を返す。
  • 親XIDに対して再帰的に TransactionIdDidAbort を呼び出し、その結果を返す。

    • 4. 上記以外（COMMITTED 等）の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c (複数箇所で可視性判定に使用)
  • src/backend/access/heap/heapam_visibility.c (注: 直接使用できないケースについての言及あり)
  • **呼び出し先**:
  • TransactionLogFetch
  • TransactionIdPrecedes
  • SubTransGetParent
  • TransactionIdIsValid
  • TransactionIdDidAbort (再帰呼び出し)

関連データ構造

  • XidStatus (トランザクション状態の列挙型)
  • TransactionId (XID型)

失敗モードと境界条件

  • **サブトランザクションの親不在**: SubTransGetParent が無効なXIDを返した場合、警告ログを出力し、アボート済みとみなす。
  • **XIDのラップアラウンド**: TransactionIdPrecedes を使用して比較を行っており、ラップアラウンドを考慮した論理的な前後関係判定が行われる。
  • **再帰の深さ**: サブトランザクションの階層が深い場合、再帰呼び出しによるスタック消費が発生する可能性がある。

保守担当者向けメモ

  • 本関数は TransactionIdDidCommit と対になるロジックを持つ。
  • SUB_COMMITTED 状態の解決には pg_subtrans を参照するため、TransactionXmin による境界チェックが重要である。
