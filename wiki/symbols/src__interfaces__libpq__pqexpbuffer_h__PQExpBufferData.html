PQExpBufferData

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.h:44

目的

動的に拡張可能な文字列バッファを管理するためのデータ構造です。メモリ管理を抽象化し、libpq 内での文字列操作やバイナリデータの構築を安全に行うために使用されます。

シグネチャまたは形状

typedef struct PQExpBufferData
{
	char	   *data;
	size_t		len;
	size_t		maxlen;
} PQExpBufferData;

入力と出力

  • data: 確保された文字列バッファへのポインタ。
  • len: 現在バッファに格納されているデータの長さ。
  • maxlen: 確保済みのバッファ容量。

前提条件と不変条件

  • malloc を使用してメモリを確保するため、使用後は必ず destroyPQExpBuffer や termPQExpBuffer を呼び出して解放する必要があります。
  • メモリ確保に失敗した場合、バッファは「broken」状態（maxlen = 0）に遷移します。

アルゴリズムまたは振る舞い

  • createPQExpBuffer または initPQExpBuffer により初期化されます。
  • appendPQExpBuffer 等の関数を通じてデータが追記されます。
  • 必要に応じて enlargePQExpBuffer が内部的に呼び出され、malloc によるバッファの再確保が行われます。
  • 「broken」状態のバッファに対する操作は安全に無視されるか、エラーとして扱われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c、src/bin/initdb/initdb.c、src/bin/pg_amcheck/pg_amcheck.c 等の各所でインスタンスが宣言されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、操作は pqexpbuffer.c 内の関数群（appendPQExpBuffer 等）によって行われます。

関連データ構造

  • PQExpBuffer: PQExpBufferData へのポインタ型。

失敗モードと境界条件

  • メモリ不足（OOM）発生時: maxlen が 0 に設定され、バッファが「broken」状態となります。この状態は PQExpBufferBroken(str) マクロで判定可能です。

保守担当者向けメモ

  • 本構造体はバックエンドの StringInfo に相当しますが、palloc ではなく標準の malloc を使用する設計です。
  • 文字列構築のループ内では、必ず PQExpBufferBroken による状態チェックを行うことが推奨されます。
