fmt_tz

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/formatting.c:460

目的

do_to_timestamp 関数において、タイムゾーン関連情報の出力および保持を行うために使用されます。

シグネチャまたは形状

struct fmt_tz
{
    bool    has_tz;      /* TZ/TZH/TZM フィールドが存在したか */
    int     gmtoffset;   /* GMT オフセット（秒単位） */
};

入力と出力

本構造体はデータ保持用であり、関数ではありません。

  • **入力**: do_to_timestamp 等の処理過程で、タイムゾーン指定子（TZ, TZH, TZM）の解析結果が各メンバに格納されます。
  • **出力**: 解析されたタイムゾーン情報が、後続の時刻変換処理へ渡されます。

前提条件と不変条件

  • has_tz が true の場合、gmtoffset には有効なオフセット値が格納されていることが期待されます。

アルゴリズムまたは振る舞い

do_to_timestamp 内部で、フォーマット文字列の解析中にタイムゾーン関連の指定子が見つかると、この構造体のインスタンスが更新されます。具体的には、has_tz フラグが立てられ、対応するオフセット値が gmtoffset に設定されます。

  • **呼び出し先**: 本構造体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • pg_tm: 日時情報の保持に使用される構造体。
  • fsec_t: 小数秒の保持に使用される型。

失敗モードと境界条件

  • タイムゾーン指定子が含まれない場合、has_tz は false となり、gmtoffset は初期化状態（または未定義）となります。

保守担当者向けメモ

  • 本構造体は do_to_timestamp のロジックに密結合しています。タイムゾーン関連の新しいフォーマット指定子を追加する場合は、本構造体のメンバがその情報を保持するのに十分か確認してください。
