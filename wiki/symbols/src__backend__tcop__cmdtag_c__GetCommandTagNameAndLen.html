GetCommandTagNameAndLen

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/cmdtag.c:52

目的

指定された CommandTag 列挙値に対応するコマンド名（文字列）とその長さを取得します。

シグネチャまたは形状

const char *GetCommandTagNameAndLen(CommandTag commandTag, Size *len);

入力と出力

  • **入力**:
  • commandTag: 取得対象のコマンドを示す CommandTag 列挙値。
  • len: コマンド名の長さを格納するための Size 型ポインタ。
  • **出力**:
  • 戻り値: コマンド名を表す const char * 型の文字列。
  • len 引数経由: コマンド名の長さ（バイト数）。

前提条件と不変条件

  • commandTag は有効な CommandTag 列挙値である必要があります。
  • len は有効なメモリ領域を指している必要があります。
  • 内部テーブル tag_behavior が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. tag_behavior 配列から commandTag をインデックスとして該当するエントリを参照します。 2. エントリ内の namelen を len が指す領域に書き込みます。 3. エントリ内の name（文字列ポインタ）を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/cmdtag.c:126
  • src/backend/tcop/postgres.c:1120
  • src/backend/tcop/postgres.c:2196
  • **呼び出し先**:
  • 外部関数呼び出しなし（tag_behavior 配列への直接アクセスのみ）。

関連データ構造

  • CommandTag: コマンドを識別する列挙型。
  • CommandTagBehavior: 各コマンドの属性を保持する構造体。
  • tag_behavior: 全コマンドの属性を保持する静的配列。

失敗モードと境界条件

  • commandTag が配列の範囲外である場合、未定義の動作（メモリ不正アクセス）が発生します。

保守担当者向けメモ

  • 本関数は tag_behavior 配列の構造に強く依存しています。cmdtaglist.h を通じて新しいコマンドタグを追加する際は、tag_behavior の整合性が自動的に保たれる仕組みになっています。
