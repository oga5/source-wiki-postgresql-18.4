GetAttrDefaultOid

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_attrdef.c:277

目的

指定されたリレーション（テーブル）および属性番号（列番号）に対応する、pg_attrdef カタログ内のデフォルト値エントリの OID を取得する。

シグネチャまたは形状

Oid GetAttrDefaultOid(Oid relid, AttrNumber attnum)

入力と出力

  • **入力**:
  • relid (Oid): 対象テーブルの OID。
  • attnum (AttrNumber): 対象列の属性番号。
  • **出力**:
  • Oid: 見つかった場合はデフォルト値の OID、存在しない場合は InvalidOid。

前提条件と不変条件

  • pg_attrdef カタログに対する AccessShareLock を取得して読み取りを行う。
  • AttrDefaultIndexId インデックスを使用して検索を行う。

アルゴリズムまたは振る舞い

1. table_open を用いて pg_attrdef カタログを AccessShareLock モードで開く。 2. ScanKeyInit を使用し、adrelid（テーブルOID）と adnum（属性番号）をキーとして検索条件を構築する。 3. systable_beginscan を介してインデックススキャンを開始する。 4. systable_getnext でタプルを取得し、有効なタプルであれば GETSTRUCT を通じて Form_pg_attrdef 構造体から oid フィールドを抽出する。 5. systable_endscan でスキャンを終了し、table_close でカタログを閉じて結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c:1580
  • src/backend/commands/tablecmds.c:8697, 8878, 15006
  • **呼び出し先**:
  • table_open, table_close
  • ScanKeyInit
  • systable_beginscan, systable_getnext, systable_endscan
  • GETSTRUCT

関連データ構造

  • pg_attrdef: デフォルト値を保持するシステムカタログ。
  • Form_pg_attrdef: pg_attrdef カタログのタプル構造体。

失敗モードと境界条件

  • 指定された relid と attnum に一致するデフォルト値が存在しない場合、InvalidOid を返す。
  • 内部的なカタログ検索エラーが発生した場合、PostgreSQL のエラーハンドリング機構によりアボートされる可能性がある。

保守担当者向けメモ

  • 本関数は pg_attrdef のインデックススキャンに依存しているため、カタログ定義やインデックス定義に変更を加える際は注意が必要である。
  • 戻り値が InvalidOid であることは、その列にデフォルト値が設定されていないことを意味する。
