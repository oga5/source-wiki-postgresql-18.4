libpq_ngettext

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-misc.c:1359

目的

国際化対応されたメッセージの複数形を、指定された数 n に基づいて取得します。

シグネチャまたは形状

char *libpq_ngettext(const char *msgid, const char *msgid_plural, unsigned long n);

入力と出力

  • **入力**:
  • msgid: 単数形のメッセージ文字列。
  • msgid_plural: 複数形のメッセージ文字列。
  • n: 翻訳の選択に使用する数値。
  • **出力**:
  • ロケール設定に基づき選択された翻訳済み文字列（char *）。

前提条件と不変条件

  • 呼び出し前に libpq_binddomain() が実行され、メッセージドメインが適切にバインドされている必要があります。

アルゴリズムまたは振る舞い

1. libpq_binddomain() を呼び出し、libpq 用のメッセージドメインを確実に設定します。 2. dngettext を使用して、PG_TEXTDOMAIN("libpq") ドメインから n に応じた適切な翻訳文字列を検索し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-secure-common.c:286
  • **呼び出し先**:
  • libpq_binddomain (同ファイル内)
  • dngettext (システムライブラリ)

関連データ構造

  • なし（標準的な文字列ポインタと数値を使用）。

失敗モードと境界条件

  • dngettext の仕様に依存します。翻訳が見つからない場合、入力された msgid または msgid_plural がそのまま返される可能性があります。

保守担当者向けメモ

  • 本関数は libpq-int.h にて pg_attribute_format_arg 属性が付与されており、コンパイル時にフォーマット文字列の整合性チェックが行われます。
  • 翻訳ドメインの変更や追加を行う際は、libpq_binddomain の実装と整合性を保ってください。
