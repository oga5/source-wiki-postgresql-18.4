EstimateParamExecSpace

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execParallel.c:318

目的

並列クエリ実行において、リーダープロセスからワーカープロセスへ転送される実行時パラメータ（PARAM_EXEC）を格納するために必要な共有メモリ領域のサイズを算出します。

シグネチャまたは形状

Size EstimateParamExecSpace(EState *estate, Bitmapset *params)

入力と出力

  • **入力**:
  • estate: 実行状態を保持する EState 構造体。
  • params: 転送対象のパラメータIDを含む Bitmapset。
  • **出力**:
  • Size: パラメータのIDおよび各値をシリアライズするために必要な合計バイト数。

前提条件と不変条件

  • estate->es_param_exec_vals が初期化されており、指定された params に対応するインデックスが有効であること。
  • estate->es_plannedstmt->paramExecTypes に各パラメータの型情報が格納されていること。

アルゴリズムまたは振る舞い

1. 初期サイズとして sizeof(int) を確保します。 2. params に含まれる各パラメータIDを bms_next_member で順次走査します。 3. 各パラメータについて以下を実行します：

  • パラメータIDを格納するための sizeof(int) を加算します。
  • estate->es_plannedstmt->paramExecTypes から該当パラメータの型OIDを取得します。
  • 型OIDが有効な場合は get_typlenbyval を使用し、無効な場合はデフォルト（sizeof(Datum)、値渡し）として、datumEstimateSpace を呼び出して値のシリアライズに必要なサイズを加算します。

    • 4. 最終的な合計サイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c 内の並列実行初期化プロセスから呼び出されます。
  • **呼び出し先**: bms_next_member, list_nth_oid, add_size, get_typlenbyval, datumEstimateSpace。

関連データ構造

  • EState: クエリ実行状態。
  • ParamExecData: 実行時パラメータの値とNULLフラグを保持する構造体。
  • Bitmapset: 転送対象パラメータの集合。

失敗モードと境界条件

  • params が空の場合、初期値 sizeof(int) を返します。
  • 型OIDが不明な場合、datumEstimateSpace は値渡し（typByVal = true）かつ sizeof(Datum) のサイズとして計算します。

保守担当者向けメモ

  • 新しいデータ型をパラメータとして扱う場合、datumEstimateSpace が適切にサイズを算出できるか確認してください。
  • add_size を使用しているため、計算過程でのオーバーフローは適切に処理されます。
