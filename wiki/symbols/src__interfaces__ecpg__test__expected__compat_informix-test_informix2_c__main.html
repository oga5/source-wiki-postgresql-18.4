目的

ECPG（Embedded SQL for C）における Informix 互換機能の回帰テストを実行し、データベース操作およびタイムスタンプ型の計算処理が正しく行われるかを検証する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（コマンドライン引数は受け取らない）。
  • 出力: 正常終了時に 0 を返し、標準出力に "All OK!" を出力する。エラー発生時は exit(1) により異常終了する。

前提条件と不変条件

  • データベース ecpg1_regression が接続可能であること。
  • sqlca 構造体が SQL 実行後の状態を保持していること。
  • タイムスタンプ計算において PGTYPES ライブラリが利用可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGconnect でデータベースに接続し、DateStyle を設定する。 2. **テーブル操作**: history テーブルを作成し、初期データを挿入する。 3. **データ処理**:

  • SELECT max(timestamp) で最新のタイムスタンプを取得する。
  • PGTYPESinterval_from_asc を用いてインターバルを生成し、PGTYPEStimestamp_add_interval でタイムスタンプを加算する。
  • 計算結果を新しいレコードとして INSERT する。

    • 4. **終了処理**: トランザクションをコミットし、テーブルを削除して接続を切断する。 5. **エラーハンドリング**: 各 SQL 操作後に sql_check を呼び出し、sqlca.sqlcode が負の場合は即座に終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）。
  • 呼び出し先: ECPGconnect, ECPGdo, ECPGtrans, ECPGdisconnect, sql_check, PGTYPESinterval_from_asc, PGTYPEStimestamp_add_interval, printf, strcpy, free, exit。

関連データ構造

  • sqlca: SQL 通信領域（グローバル構造体）。
  • timestamp: タイムスタンプ型。
  • interval: インターバル型。

失敗モードと境界条件

  • データベース接続失敗時、または SQL 操作（CREATE, INSERT, SELECT, DROP）でエラーが発生した場合は sqlca.sqlcode < 0 を検知して終了する。
  • sql_check 関数は、特定の SQL ステートメント（select max など）において、期待される戻り値（例: データなしを示す 100）を許容する。

保守担当者向けメモ

  • 本関数は自動生成されたコードであるため、直接編集しないこと。
  • デバッグが必要な場合は ECPGdebug(1, stderr) を有効にすることで、プリプロセッサが生成した SQL 実行の詳細ログを確認できる。
