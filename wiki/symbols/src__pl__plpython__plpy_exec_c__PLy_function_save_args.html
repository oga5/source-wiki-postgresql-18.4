PLy_function_save_args

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_exec.c:495

目的

PL/Python 関数の実行時、再帰呼び出しや状態保持のために、現在の Python グローバル辞書内の引数状態を退避させる。

シグネチャまたは形状

static PLySavedArgs *PLy_function_save_args(PLyProcedure *proc)

入力と出力

  • **入力**: proc (PLyProcedure *) - 現在実行中のプロシージャ定義。
  • **出力**: PLySavedArgs * - 退避された引数情報を格納した構造体。

前提条件と不変条件

  • proc->mcxt（プロシージャのメモリコンテキスト）が有効であること。
  • proc->globals に必要なキー（"args" 等）が存在すること。
  • 戻り値の PLySavedArgs は proc->mcxt 上に割り当てられる。

アルゴリズムまたは振る舞い

1. proc->mcxt 上に PLySavedArgs 構造体分のメモリを確保する。 2. proc->globals から "args" リストを取得し、参照カウントをインクリメントして保存する。 3. proc->is_trigger が真の場合、"TD"（トリガーデータ）を取得し、参照カウントをインクリメントして保存する。 4. proc->argnames が存在する場合、定義されているすべての名前付き引数を proc->globals から取得し、それぞれ参照カウントをインクリメントして namedargs 配列に保存する。 5. 構築した PLySavedArgs 構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_exec.c:179 (SRF 状態の初期化時)
  • src/pl/plpython/plpy_exec.c:619 (再帰呼び出し等のコンテキスト保存時)
  • **呼び出し先**:
  • MemoryContextAllocZero
  • PyDict_GetItemString
  • Py_XINCREF

関連データ構造

  • PLyProcedure: プロシージャの定義と実行環境を保持する構造体。
  • PLySavedArgs: 退避された引数（"args" リスト、"TD"、名前付き引数）を保持する構造体。

失敗モードと境界条件

  • proc->globals 内に期待されるキーが存在しない場合、PyDict_GetItemString は NULL を返し、Py_XINCREF は NULL に対して安全に動作する（Py_XINCREF の仕様による）。
  • メモリ不足時は MemoryContextAllocZero がエラーを発生させる。

保守担当者向けメモ

  • 本関数でインクリメントした参照カウントは、対応する PLy_function_restore_args または PLy_function_drop_args 等で適切に Py_DECREF される必要がある。メモリリークを防ぐため、呼び出し側のライフサイクル管理を厳密に行うこと。
