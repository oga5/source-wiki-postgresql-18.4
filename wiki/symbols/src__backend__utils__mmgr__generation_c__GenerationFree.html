GenerationFree

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/mmgr/generation.c:688

目的

世代別メモリコンテキスト内の指定されたメモリチャンクを解放し、チャンクが属するブロックの空き状況に応じて、ブロックの再利用またはメモリの返却（free()）を行う。

シグネチャまたは形状

void GenerationFree(void *pointer)

入力と出力

  • **入力**: pointer (解放対象のメモリ領域を指すポインタ)
  • **出力**: なし

前提条件と不変条件

  • pointer は有効なメモリチャンクを指している必要がある。
  • MEMORY_CONTEXT_CHECKING が有効な場合、二重解放（double pfree）はエラーとして扱われる。
  • 解放後のブロックにおいて、block->nfree <= block->nchunks が常に成立すること。

アルゴリズムまたは振る舞い

1. **チャンク特定**: pointer からチャンクヘッダーを取得し、所属する GenerationBlock を特定する。 2. **検証**: デバッグビルド時、チャンクの有効性や境界外書き込みの有無をチェックする。 3. **解放処理**: チャンクを解放済みとしてマークし、ブロック内の空きチャンク数（nfree）をインクリメントする。 4. **ブロック管理**:

  • ブロック内にまだ使用中のチャンクがある場合、処理を終了する。
  • ブロックが空になった場合、以下のいずれかの処理を行う：
  • **KeeperBlock または現在のブロック**: GenerationBlockMarkEmpty を呼び出し、再利用可能な状態にする。
  • **freeblock が未設定**: そのブロックを freeblock として保持し、将来の割り当てに備える。
  • **上記以外**: GenerationBlockFree を呼び出し、メモリをシステムへ解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MCTX_GENERATION_ID の free_p 関数ポインタとして登録。
  • その他、メモリコンテキスト経由で pfree を呼び出す各所。
  • **呼び出し先**:
  • PointerGetMemoryChunk, ExternalChunkGetBlock, MemoryChunkGetBlock
  • GenerationBlockIsValid, GenerationBlockMarkEmpty, GenerationBlockFree
  • wipe_mem (定義時のみ)

関連データ構造

  • GenerationContext: メモリコンテキストの管理構造体。
  • GenerationBlock: メモリブロックの管理単位。
  • MemoryChunk: チャンクヘッダー。

失敗モードと境界条件

  • **二重解放**: chunk->requested_size が InvalidAllocSize の場合、elog(ERROR) を発生させる。
  • **境界外書き込み**: sentinel_ok チェックに失敗した場合、elog(WARNING) を発生させる。
  • **不正なブロック**: GenerationBlockIsValid が偽の場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • FIFO ワークロードにおいて、freeblock を保持することで頻繁な malloc/free を回避する設計となっている。
  • KeeperBlock はコンテキスト自体の寿命と同期するため、この関数で解放されることはない。
  • VALGRIND_MAKE_MEM_DEFINED を使用して、チャンクヘッダーへのアクセスを制御している。
