do_array_start

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_json_parser/test_json_parser_incremental.c:295

目的

JSON配列の開始イベントを処理し、テスト出力として [ を表示するとともに、配列内の要素管理状態を初期化する。

シグネチャまたは形状

static JsonParseErrorType do_array_start(void *state)

入力と出力

  • **入力**: void *state （DoState 構造体へのポインタとしてキャストして使用）
  • **出力**: JSON_SUCCESS を返す。

前提条件と不変条件

  • 引数 state は DoState 型の有効なポインタである必要がある。
  • 本関数は JsonSemAction コールバックテーブルの一部として呼び出される。

アルゴリズムまたは振る舞い

1. 引数 state を DoState 型にキャストする。 2. 標準出力に [\n を書き出す。 3. _state->elem_is_first を true に設定し、配列の最初の要素であることを示すフラグを初期化する。 4. JSON_SUCCESS を返して処理を完了する。

呼び出し元と呼び出し先

  • **呼び出し元**: インクリメンタルJSONパーサーのセマンティックアクション実行時に、JsonSemAction 構造体を通じて呼び出される（ソースコード内の JsonSemAction 初期化箇所で参照されている）。
  • **呼び出し先**: 標準ライブラリの printf を呼び出す。

関連データ構造

  • DoState: パース中の状態を保持する構造体。本関数はこの構造体の elem_is_first メンバを更新する。

失敗モードと境界条件

  • 本関数自体はエラーを返さない（常に JSON_SUCCESS を返す）。
  • state が NULL の場合、キャスト後のアクセスで未定義の動作が発生する可能性がある。

保守担当者向けメモ

  • 本関数はテストモジュールの一部であり、パース結果の再構成（エコーバック）を目的としている。
  • elem_is_first フラグは、後続の do_array_element_start 等でカンマの挿入制御に使用される。
