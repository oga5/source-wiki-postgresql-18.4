sepgsql_avc_unlabeled

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:169

目的

SELinux の初期セキュリティラベルである "unlabeled" コンテキスト文字列を取得し、キャッシュして返します。

シグネチャまたは形状

char *sepgsql_avc_unlabeled(void)

入力と出力

  • 入力: なし
  • 出力: char * 型の "unlabeled" セキュリティコンテキスト文字列。

前提条件と不変条件

  • 静的変数 avc_unlabeled が初期化されていない場合、security_get_initial_context_raw を呼び出してカーネルから取得します。
  • 取得した文字列は avc_mem_cxt メモリコンテキスト内に複製（MemoryContextStrdup）され、以降の呼び出しではキャッシュされた値が返されます。

アルゴリズムまたは振る舞い

1. avc_unlabeled が NULL か確認します。 2. NULL の場合、security_get_initial_context_raw("unlabeled", ...) を実行します。 3. 取得に失敗した場合は ereport(ERROR, ...) で処理を中断します。 4. 成功した場合、PG_TRY ブロック内で avc_mem_cxt に文字列をコピーし、PG_FINALLY ブロックで一時的なバッファを freecon で解放します。 5. キャッシュされた avc_unlabeled を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/sepgsql/uavc.c 内の各所（キャッシュ検索や権限チェックのデフォルト値として使用）。
  • 呼び出し先:
  • security_get_initial_context_raw (SELinux API)
  • MemoryContextStrdup (PostgreSQL メモリ管理)
  • freecon (SELinux API)

関連データ構造

  • avc_mem_cxt: キャッシュ文字列を保持するメモリコンテキスト。
  • avc_unlabeled: 取得した文字列を保持する静的変数。

失敗モードと境界条件

  • security_get_initial_context_raw が負の値を返した場合、PostgreSQL のエラーとして報告され、トランザクションがアボートされます。

保守担当者向けメモ

  • 本関数は初回呼び出し時にのみカーネルへ問い合わせを行う遅延初期化パターンを採用しています。
  • avc_mem_cxt がリセットされるタイミング（sepgsql_avc_reset 等）で avc_unlabeled も適切に管理されているか注意してください。
