EstimateComboCIDStateSpace

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/combocid.c:296

目的

パラレルクエリ実行時などに、現在のトランザクションで使用されている Combo CID の状態をシリアライズするために必要な共有メモリ上のバイト数を算出します。

シグネチャまたは形状

Size EstimateComboCIDStateSpace(void)

入力と出力

  • **入力**: なし
  • **出力**: Size 型のバイト数。Combo CID の個数（usedComboCids）と、各キーデータ（ComboCidKeyData）のサイズに基づいた合計値を返します。

前提条件と不変条件

  • usedComboCids が現在のトランザクション内で正しく管理されている必要があります。
  • add_size および mul_size を使用してオーバーフローを防止した計算が行われます。

アルゴリズムまたは振る舞い

1. Combo CID の個数を格納するための sizeof(int) 分の領域を確保します。 2. usedComboCids 個の ComboCidKeyData 構造体を格納するために必要な領域を計算し、加算します。 3. 計算された合計サイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/parallel.c:274 にて、パラレルワーカーへの状態移行準備のために呼び出されています。
  • **呼び出し先**: add_size および mul_size（PostgreSQL のメモリ計算ユーティリティ）を呼び出します。

関連データ構造

  • ComboCidKeyData: Combo CID のキー情報を保持する構造体。
  • usedComboCids: 現在のトランザクションで使用中の Combo CID 数を保持する内部変数。

失敗モードと境界条件

  • usedComboCids が 0 の場合、sizeof(int) 分のサイズのみが返されます。
  • メモリ計算関数（add_size, mul_size）により、サイズ計算時のオーバーフローが適切に処理されます。

保守担当者向けメモ

  • 本関数はシリアライズに必要なサイズを算出するのみであり、実際のデータコピーは別の関数で行われます。
  • ComboCidKeyData の構造に変更を加える際は、本関数でのサイズ計算ロジックが整合しているか確認してください。
