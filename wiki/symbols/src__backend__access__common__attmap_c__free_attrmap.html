free_attrmap

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/attmap.c:55

目的

AttrMap 構造体およびその内部の属性番号配列（attnums）が占有しているメモリ領域を解放します。

シグネチャまたは形状

void free_attrmap(AttrMap *map)

入力と出力

  • **入力**: map (AttrMap *) - 解放対象の AttrMap 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 map は NULL であってはなりません（pfree が内部で呼び出されるため）。
  • map->attnums は、make_attrmap 等によって事前にメモリが割り当てられている必要があります。

アルゴリズムまたは振る舞い

1. map->attnums が指すメモリ領域を pfree で解放します。 2. map 構造体自体を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/attmap.c (変換不要と判定された際のクリーンアップ)
  • src/backend/access/common/tupconvert.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/tablecmds.c
  • src/backend/replication/logical/relation.c
  • **呼び出し先**: pfree (PostgreSQLのメモリ管理関数)。

関連データ構造

  • AttrMap: 属性番号の対応関係を保持する構造体。本関数はこの構造体のライフサイクルを終了させます。

失敗モードと境界条件

  • map が NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は AttrMap の所有権を完全に放棄する際に使用してください。
  • pfree を使用しているため、PostgreSQLのメモリコンテキスト内で適切に管理されている必要があります。
