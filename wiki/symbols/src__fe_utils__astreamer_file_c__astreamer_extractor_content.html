astreamer_extractor_content

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_file.c:205

目的

アーカイブストリームからメンバー情報を解析し、ファイルシステム上にディレクトリ構造の再現、ファイルの作成、またはシンボリックリンクの作成を行う。

シグネチャまたは形状

static void astreamer_extractor_content(astreamer *streamer, astreamer_member *member, const char *data, int len, astreamer_archive_context context)

入力と出力

  • streamer: 抽出コンテキスト（astreamer_extractor 型へキャストして使用）。
  • member: 処理対象のアーカイブメンバー情報。
  • data: 書き込み対象のデータバッファ。
  • len: data の長さ。
  • context: 処理状態（ASTREAMER_MEMBER_HEADER, CONTENTS, TRAILER, ARCHIVE_TRAILER）。
  • 出力: なし（副作用としてファイルシステム操作を実行）。

前提条件と不変条件

  • member は ASTREAMER_ARCHIVE_TRAILER 以外のコンテキストでは非 NULL であること。
  • context は ASTREAMER_UNKNOWN 以外であること。
  • ASTREAMER_MEMBER_HEADER 処理時、mystreamer->file は NULL であること。

アルゴリズムまたは振る舞い

context に応じて以下の状態遷移を行う： 1. **ASTREAMER_MEMBER_HEADER**:

  • パス安全性を検証し、ベースパスと結合。
  • メンバー種別（通常ファイル、ディレクトリ、シンボリックリンク）に応じて create_file_for_extract、extract_directory、extract_link を呼び出す。

    • 2. **ASTREAMER_MEMBER_CONTENTS**:

  • 開いているファイルハンドルに対して fwrite を実行。書き込み失敗時は pg_fatal を発行。

    • 3. **ASTREAMER_MEMBER_TRAILER**:

  • ファイルハンドルを fclose し、コンテキスト内のポインタを NULL にリセット。

    • 4. **ASTREAMER_ARCHIVE_TRAILER**:

  • 何も行わない。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_extractor 構造体の .content フィールドを介して呼び出される（直接の呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**:
  • path_is_safe_for_extraction
  • create_file_for_extract
  • extract_directory
  • extract_link
  • pg_fatal
  • mystreamer->link_map（設定されている場合）
  • mystreamer->report_output_file（設定されている場合）

関連データ構造

  • astreamer_extractor: 抽出処理のコンテキストを保持する構造体。
  • astreamer_member: アーカイブメンバーのメタデータ（パス、モード、種別等）。

失敗モードと境界条件

  • **パスの安全性**: path_is_safe_for_extraction が偽を返す場合、pg_fatal で異常終了。
  • **書き込みエラー**: fwrite が失敗した場合、errno に基づき pg_fatal で異常終了。
  • **クローズエラー**: fclose が失敗した場合、pg_fatal で異常終了。
  • **不正な状態**: 定義外の context が渡された場合、pg_fatal で異常終了。

保守担当者向けメモ

  • 本関数は astreamer フレームワークの終端処理を担う。ファイルシステムへの書き込み権限やパスの検証ロジックは、fe_utils 内の関連関数に依存している。
  • link_map や report_output_file は関数ポインタであり、インスタンス生成時に注入される。
