clauselist_selectivity_or

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/clausesel.c:358

目的

OR で結合された条件リスト全体の選択率（Selectivity）を計算します。各条件の選択率を統合し、条件間の重複を考慮した推定値を提供します。

シグネチャまたは形状

static Selectivity clauselist_selectivity_or(
    PlannerInfo *root,
    List *clauses,
    int varRelid,
    JoinType jointype,
    SpecialJoinInfo *sjinfo,
    bool use_extended_stats
)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • clauses: OR で結合される条件のリスト。
  • varRelid: 対象リレーションの ID。
  • jointype: 結合の種類。
  • sjinfo: 特殊な結合情報。
  • use_extended_stats: 拡張統計を使用するかどうかのフラグ。
  • **出力**:
  • Selectivity (double): 計算された選択率（0.0 から 1.0 の範囲）。

前提条件と不変条件

  • use_extended_stats が有効かつ対象リレーションに拡張統計が存在する場合、優先的に適用されます。
  • 拡張統計で推定された条件は estimatedclauses ビットマップで追跡され、重複計算が回避されます。

アルゴリズムまたは振る舞い

1. **拡張統計の適用**: find_single_rel_for_clauses を呼び出し、単一リレーションに対する拡張統計が利用可能であれば statext_clauselist_selectivity を実行して初期選択率 s1 を算出します。 2. **個別条件の推定**: 拡張統計でカバーされなかった条件に対し、clause_selectivity_ext を再帰的に呼び出し、個別の選択率 s2 を取得します。 3. **統合計算**: OR 条件の性質に基づき、s1 = s1 + s2 - s1 * s2 の式を用いて、選択されたタプル集合の重複を考慮した累積選択率を算出します。 4. **戻り値**: 最終的な s1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/clausesel.c 内の他の関数から呼び出されています（詳細はソースコードの行番号 824 等を参照）。
  • **呼び出し先**:
  • find_single_rel_for_clauses
  • statext_clauselist_selectivity
  • clause_selectivity_ext

関連データ構造

  • PlannerInfo: プランナのコンテキスト情報。
  • Selectivity: 選択率を保持する型（double）。
  • Bitmapset: 拡張統計で推定済みの条件を管理するために使用。

失敗モードと境界条件

  • 拡張統計が適用できない場合、各条件は独立しているものとして計算されます。
  • OR 条件の重複を考慮する式（s1 + s2 - s1 * s2）は、条件が独立していると仮定した保守的な推定値です。

保守担当者向けメモ

  • 本関数は OR 条件の選択率を計算する際に、統計情報と個別の条件推定を組み合わせる重要な役割を担います。
