estimate_size

定義

  • 種別: function
  • 定義箇所: contrib/file_fdw/file_fdw.c:1062

目的

外部テーブルの物理サイズ（ページ数）および行数（タプル数）を推定し、クエリプランナがコスト計算に使用する統計情報を FileFdwPlanState および RelOptInfo に設定する。

シグネチャまたは形状

static void estimate_size(PlannerInfo *root, RelOptInfo *baserel, FileFdwPlanState *fdw_private)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • baserel: 外部テーブルのプランニング情報。
  • fdw_private: ファイル固有のプランニング状態（ファイルパス、プログラム実行フラグ等）。
  • **出力**:
  • fdw_private->pages: 推定ページ数。
  • fdw_private->ntuples: 推定タプル数。
  • baserel->rows: フィルタ適用後の推定行数。

前提条件と不変条件

  • fdw_private は初期化済みであること。
  • baserel->reltarget が有効であること。
  • stat システムコールが利用可能であること。

アルゴリズムまたは振る舞い

1. **ファイルサイズの取得**: stat を使用してファイルサイズを取得する。プログラム入力の場合、または stat が失敗した場合は 10 * BLCKSZ をデフォルト値とする。 2. **ページ数の算出**: ファイルサイズを BLCKSZ で除算し、最低 1 ページを保証する。 3. **タプル数の推定**:

  • pg_class に統計情報（ANALYZE 結果）が存在する場合、既存の密度（タプル/ページ）を現在のページ数に適用する。
  • 統計情報がない場合、リレーションの幅（tuple_width）からファイルサイズを割ることで概算する。

    • 4. **フィルタ適用後の行数推定**: clauselist_selectivity を使用して、baserestrictinfo（WHERE 句の条件）による選択率を乗算し、最終的な行数を算出する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/file_fdw/file_fdw.c 内の fileGetForeignRelSize から呼び出される。
  • **呼び出し先**: stat (システムコール), clamp_row_est, clauselist_selectivity。

関連データ構造

  • FileFdwPlanState: 外部テーブルのプランニング状態を保持する構造体。
  • RelOptInfo: プランナがリレーションの統計情報を保持する構造体。
  • PlannerInfo: プランナのグローバルコンテキスト。

失敗モードと境界条件

  • ファイルが存在しない、またはアクセス権限がない場合、デフォルトサイズ（10 * BLCKSZ）にフォールバックする。
  • pages が 0 と算出された場合、最低値として 1 が強制される。
  • baserel->tuples が負の場合、統計情報不足とみなして計算式を切り替える。

保守担当者向けメモ

  • 本関数は ANALYZE による統計情報があるかどうかに依存する。精度向上のためには、ユーザーに対して対象テーブルの ANALYZE 実行を推奨する仕様となっている。
  • tuple_width の計算は内部表現とテキスト表現の差異を考慮していないため、あくまで概算である点に注意すること。
