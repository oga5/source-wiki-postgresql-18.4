PrefetchBufferResult

定義

  • 種別: struct
  • 定義箇所: src/include/storage/bufmgr.h:59

目的

バッファのプリフェッチ操作の結果を保持し、呼び出し元に対して対象バッファが即座に利用可能か、あるいは非同期I/Oが開始されたかを通知するために使用されます。

シグネチャまたは形状

typedef struct PrefetchBufferResult
{
	Buffer		recent_buffer;	/* 有効な場合、ヒット（再チェックが必要） */
	bool		initiated_io;	/* 真の場合、ミスが発生し非同期I/Oを開始 */
} PrefetchBufferResult;

入力と出力

  • **入力**: プリフェッチ関数（PrefetchBuffer, PrefetchSharedBuffer, PrefetchLocalBuffer）の戻り値として生成されます。
  • **出力**: 呼び出し元は recent_buffer が有効（InvalidBuffer 以外）かどうか、および initiated_io が真かどうかを判定して後続の処理を決定します。

前提条件と不変条件

  • recent_buffer が有効な場合、それはバッファヒットを意味しますが、呼び出し元は内容の整合性を確認するために再チェックを行う必要があります。
  • initiated_io が真の場合、指定されたブロックに対する非同期I/Oが開始されています。

アルゴリズムまたは振る舞い

1. プリフェッチ関数が呼び出されると、対象ブロックがバッファ内に存在するか確認します。 2. 存在する場合、recent_buffer にそのバッファIDを格納し、initiated_io を false に設定します。 3. 存在しない場合、非同期I/Oを開始し、initiated_io を true に設定します。

関連データ構造

  • Buffer: バッファを識別する整数型。
  • SMgrRelationData: 物理ストレージ管理のための構造体。

失敗モードと境界条件

  • recent_buffer が InvalidBuffer である場合、ヒットしなかったことを示します。
  • initiated_io が false かつ recent_buffer が InvalidBuffer の場合、プリフェッチが何らかの理由で実行されなかった（あるいは不要であった）状態を示します。

保守担当者向けメモ

  • 本構造体は非同期I/Oの制御フローにおいて重要な役割を果たします。recent_buffer を使用する際は、必ず再チェックのロジックが伴っていることを確認してください。
