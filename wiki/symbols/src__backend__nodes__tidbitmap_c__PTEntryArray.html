PTEntryArray

定義

  • 種別: struct
  • 定義箇所: src/backend/nodes/tidbitmap.c:103

目的

共有メモリ（DSA）上で PagetableEntry の配列を保持し、複数のイテレータ間で共有されるビットマップのページテーブルを管理するために使用されます。

シグネチャまたは形状

typedef struct PTEntryArray
{
    pg_atomic_uint32 refcount;    /* イテレータの参照カウント */
    PagetableEntry ptentry[FLEXIBLE_ARRAY_MEMBER];
} PTEntryArray;

入力と出力

  • **入力**: dsa_allocate を介してメモリ領域を確保する際、sizeof(PTEntryArray) + size の形式でサイズが指定されます。
  • **出力**: 共有メモリ上に配置された PagetableEntry の連続領域へのアクセスを提供します。

前提条件と不変条件

  • refcount はアトミック操作によって管理される必要があります。
  • ptentry は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、構造体自体のサイズに加えて動的に確保された領域を指します。

アルゴリズムまたは振る舞い

  • 並列クエリ実行時、共有メモリ上のページテーブルを複数のバックエンドが参照できるようにします。
  • dsa_allocate を用いて動的に領域が確保され、イテレータの参照カウントを保持することで、不要になった際の解放を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/tidbitmap.c 内の関数（tbm_add_tuples やイテレータ初期化処理など）が、共有メモリ上のページテーブル領域を確保・参照する際に使用します。
  • **呼び出し先**: 具体的な関数呼び出し関係はソース抜粋からは特定できません。

関連データ構造

  • PagetableEntry: 配列の各要素として格納される、ページ単位の情報を管理する構造体。
  • TIDBitmap: 本構造体を dsapagetable として保持するメインのビットマップ管理構造体。

失敗モードと境界条件

  • dsa_allocate によるメモリ確保失敗時に、システムはエラーを発生させる可能性があります。
  • 可変長配列のサイズ計算が不適切な場合、メモリ破壊や境界外アクセスのリスクがあります。

保守担当者向けメモ

  • 本構造体は並列クエリにおける共有メモリ管理に直結しているため、refcount の操作には細心の注意を払ってください。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には sizeof 演算子と動的確保サイズを正しく組み合わせる必要があります。
