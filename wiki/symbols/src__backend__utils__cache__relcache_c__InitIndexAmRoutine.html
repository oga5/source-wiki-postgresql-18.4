InitIndexAmRoutine

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relcache.c:1420

目的

リレーション記述子（Relation）に関連付けられたインデックスアクセスメソッドのルーチン情報（IndexAmRoutine）を初期化し、リレーションのキャッシュコンテキスト内に保持する。

シグネチャまたは形状

static void InitIndexAmRoutine(Relation relation)

入力と出力

  • **入力**: relation (対象となる Relation 構造体へのポインタ)
  • **出力**: なし（副作用として relation->rd_indam が更新される）

前提条件と不変条件

  • relation->rd_amhandler が有効なインデックスアクセスメソッドハンドラを指していること。
  • relation->rd_indexcxt がメモリコンテキストとして有効であること。

アルゴリズムまたは振る舞い

1. GetIndexAmRoutine を呼び出し、relation->rd_amhandler に基づく IndexAmRoutine 構造体を取得する。この際、メモリリークを防ぐため、一時的なメモリコンテキストで実行される。 2. relation->rd_indexcxt 内に IndexAmRoutine サイズのメモリを確保する。 3. 取得したルーチン情報を確保したメモリ領域にコピーする。 4. relation->rd_indam にコピー先のアドレスを格納する。 5. 一時的に確保した tmp 領域を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/relcache.c 内の複数箇所（行 1511, 6348 等）。
  • **呼び出し先**: GetIndexAmRoutine（インデックスアクセスメソッドの取得）、MemoryContextAlloc（メモリ確保）、pfree（メモリ解放）。

関連データ構造

  • Relation: リレーション記述子。
  • IndexAmRoutine: インデックスアクセスメソッドの関数ポインタ群を保持する構造体。

失敗モードと境界条件

  • GetIndexAmRoutine が失敗した場合、またはメモリ確保に失敗した場合は、例外が発生し処理が中断される可能性がある。

保守担当者向けメモ

  • 本関数は rd_indam の初期化を行うため、リレーションの再構築時や初期化時に呼び出される。
  • rd_indexcxt はリレーションのライフサイクルに依存するため、メモリ管理の整合性に注意すること。
