forkname_to_number

定義

  • 種別: function
  • 定義箇所: src/common/relpath.c:49

目的

文字列形式のフォーク名（"main", "fsm", "vm", "init"）を、内部的な ForkNumber 列挙型へ変換します。

シグネチャまたは形状

ForkNumber forkname_to_number(const char *forkName)

入力と出力

  • **入力**: forkName (文字列へのポインタ)
  • **出力**: 一致する ForkNumber。一致しない場合は InvalidForkNumber を返します。

前提条件と不変条件

  • forkNames 配列が ForkNumber の各値と対応する文字列を保持している必要があります。
  • MAX_FORKNUM までの範囲で走査が行われます。

アルゴリズムまたは振る舞い

1. 0 から MAX_FORKNUM までループし、forkNames 配列内の文字列と引数 forkName を strcmp で比較します。 2. 一致した時点で、そのインデックス（forkNum）を即座に返します。 3. ループ終了まで一致が見つからない場合:

  • バックエンド環境（FRONTEND が未定義）であれば、ereport(ERROR, ...) を呼び出し、有効なフォーク名を示すヒントと共にエラーを報告します。
  • フロントエンド環境であれば、エラー報告を行わず InvalidForkNumber を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/rawpage.c
  • contrib/pg_prewarm/pg_prewarm.c
  • src/backend/utils/adt/dbsize.c
  • src/bin/pg_waldump/pg_waldump.c
  • **呼び出し先**:
  • strcmp (標準ライブラリ)
  • ereport (バックエンド環境のみ)

関連データ構造

  • ForkNumber: フォーク番号を定義する列挙型。
  • forkNames: フォーク番号と文字列の対応を保持するグローバル配列。

失敗モードと境界条件

  • 引数に未知の文字列が渡された場合、バックエンドではエラー終了し、フロントエンドでは InvalidForkNumber が返されます。
  • forkNames 配列の範囲外アクセスは発生しない設計ですが、MAX_FORKNUM の定義と配列の整合性が保たれている必要があります。

保守担当者向けメモ

  • 新しいフォークを追加する場合は、ForkNumber 列挙型、forkNames 配列、および MAX_FORKNUM の更新が必須です。
  • フロントエンドとバックエンドでエラーハンドリングの挙動が異なる点に注意してください。
