ExecEndBitmapOr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapOr.c:195

目的

BitmapOr ノードの終了処理を行い、保持しているすべてのサブプランのリソースを解放する。

シグネチャまたは形状

void ExecEndBitmapOr(BitmapOrState *node)

入力と出力

  • **入力**: BitmapOrState *node - 終了対象となる BitmapOr ノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は有効な BitmapOrState 構造体を指している必要がある。
  • node->bitmapplans に格納された各サブプランの状態は、初期化済みであること。

アルゴリズムまたは振る舞い

1. node からサブプランの配列 (bitmapplans) とサブプラン数 (nplans) を取得する。 2. 0 から nplans - 1 までループし、各サブプランに対して ExecEndNode を呼び出して終了処理を実行する。 3. 各サブプランのポインタが NULL でないことを確認してから ExecEndNode を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:617 から呼び出されることが確認されている。
  • **呼び出し先**: ExecEndNode を呼び出す。

関連データ構造

  • BitmapOrState: BitmapOr ノードの実行状態を保持する構造体。bitmapplans（サブプランの配列）と nplans（サブプラン数）を含む。
  • PlanState: 各サブプランの実行状態を保持する構造体。

失敗モードと境界条件

  • bitmapplans 内の要素が NULL の場合、そのインデックスに対する ExecEndNode の呼び出しはスキップされる。

保守担当者向けメモ

  • 本関数は BitmapOr ノードのクリーンアップを担当する。新しいサブプランの追加やリソース管理の変更を行う際は、ExecInitBitmapOr での初期化と整合性が取れていることを確認すること。
  • ExecProcNode は本ノードではサポートされていないため、実行時の終了処理は本関数に依存している。
