CallbackParam

定義

  • 種別: typedef（PMINIDUMP_CALLBACK_INFORMATION 型の別名）
  • 定義箇所: src/backend/port/win32/crashdump.c:71

目的

Windows API の MiniDumpWriteDump 関数を呼び出す際、ミニダンプ生成プロセス中にコールバックを受け取るための情報構造体へのポインタを定義するために使用されます。

シグネチャまたは形状

MINIDUMPWRITEDUMP 関数ポインタ型の引数の一部として定義されています。

  • 型: CONST PMINIDUMP_CALLBACK_INFORMATION

入力と出力

  • 入力: MiniDumpWriteDump 呼び出し時に、ダンプ生成のカスタマイズや進捗報告を行うためのコールバックルーチンとコンテキスト情報を提供します。
  • 出力: なし（本シンボル自体は型定義です）。

前提条件と不変条件

  • dbghelp.h がインクルードされている必要があります。
  • Windows API の MINIDUMP_CALLBACK_INFORMATION 構造体定義に準拠している必要があります。

アルゴリズムまたは振る舞い

MiniDumpWriteDump 実行時に、ダンプファイルへの書き込み処理の挙動を制御するためのコールバック関数を登録する役割を担います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/port/win32/crashdump.c 内の MINIDUMPWRITEDUMP 関数ポインタ定義において使用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • MINIDUMPWRITEDUMP: この型が引数として使用される関数ポインタ型。
  • dbghelp.h: 本シンボルが依存する Windows デバッグヘルパーライブラリのヘッダー。

失敗モードと境界条件

  • MiniDumpWriteDump に渡されるポインタが NULL の場合、コールバックは実行されません。
  • dbghelp.dll が正しくロードされていない場合、この型を使用した関数呼び出しは失敗します。

保守担当者向けメモ

  • 本シンボルは dbghelp.h で定義された標準的な Windows API 型のエイリアスです。PostgreSQL 固有のロジックではなく、Windows のミニダンプ生成メカニズムに依存しています。
