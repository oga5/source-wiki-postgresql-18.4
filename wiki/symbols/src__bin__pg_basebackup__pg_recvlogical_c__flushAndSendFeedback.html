flushAndSendFeedback

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_recvlogical.c:1046

目的

受信した論理デコーディングデータをディスクへ永続化（フラッシュ）し、その最新のフラッシュ位置をサーバーへ通知する。

シグネチャまたは形状

static bool flushAndSendFeedback(PGconn *conn, TimestampTz *now)

入力と出力

  • **入力**:
  • conn: サーバーとの接続を保持する PGconn オブジェクト。
  • now: 現在時刻を保持する TimestampTz へのポインタ。
  • **出力**:
  • 成功時に true、失敗時に false を返す。
  • now は関数内で feGetCurrentTimestamp() により更新される。

前提条件と不変条件

  • OutputFsync が正常に完了し、データがディスクに書き込まれている必要がある。
  • sendFeedback を呼び出す前に、now が最新のタイムスタンプで更新されている必要がある。

アルゴリズムまたは振る舞い

1. OutputFsync(*now) を呼び出し、受信データをディスクへフラッシュする。失敗した場合は直ちに false を返す。 2. feGetCurrentTimestamp() を呼び出し、now を最新時刻に更新する。 3. sendFeedback(conn, *now, true, false) を呼び出し、更新されたフラッシュ位置をサーバーへ通知する。 4. sendFeedback の結果を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_recvlogical.c 内のメインループ処理（498行目、543行目、588行目）。
  • **呼び出し先**:
  • OutputFsync
  • feGetCurrentTimestamp
  • sendFeedback

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • TimestampTz: PostgreSQL のタイムスタンプ型。

失敗モードと境界条件

  • OutputFsync がディスク書き込みエラー等で失敗した場合、この関数も false を返す。
  • sendFeedback がサーバーへの通信エラー等で失敗した場合、この関数も false を返す。

保守担当者向けメモ

  • 本関数は、データの永続化とサーバーへの進捗報告をアトミックに行うためのラッパーである。
  • fsync の間隔は pg_recvlogical の設定に依存するため、呼び出し元のループ制御と整合性を保つこと。
