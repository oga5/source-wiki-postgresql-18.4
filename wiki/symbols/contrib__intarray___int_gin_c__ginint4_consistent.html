ginint4_consistent

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_gin.c:109

目的

GIN インデックスにおいて、インデックスの検索結果がクエリ条件に適合するかを判定し、再チェック（recheck）が必要かどうかを決定します。

シグネチャまたは形状

Datum ginint4_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • check[] (bool*): インデックスキーの存在フラグ配列。
  • strategy (StrategyNumber): 演算子戦略番号。
  • nkeys (int32): キーの数。
  • query (QUERYTYPE*): BooleanSearchStrategy の場合のみ使用。
  • **出力**:
  • recheck (bool*): 検索結果の再チェックが必要かを示すフラグ。
  • 戻り値 (Datum): 適合判定結果（bool）。

前提条件と不変条件

  • check 配列は nkeys のサイズを持つ必要があります。
  • BooleanSearchStrategy 以外の戦略では、recheck フラグは各戦略の性質（損失の有無）に応じて静的に決定されます。

アルゴリズムまたは振る舞い

strategy に基づき以下の判定を行います。 1. **RTOverlapStrategyNumber**: recheck = false, res = true。 2. **RTContainedByStrategyNumber / RTOldContainedByStrategyNumber**: recheck = true, res = true。 3. **RTSameStrategyNumber**: recheck = true。全キーが check に含まれる場合のみ res = true。 4. **RTContainsStrategyNumber / RTOldContainsStrategyNumber**: recheck = false。全キーが check に含まれる場合のみ res = true。 5. **BooleanSearchStrategy**: recheck = false。gin_bool_consistent を呼び出し、複雑なブール条件を評価。 6. **その他**: elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: GIN インデックスのフレームワーク（直接の呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: gin_bool_consistent（BooleanSearchStrategy の場合）。

関連データ構造

  • QUERYTYPE: ブール検索時に使用されるクエリ構造体。
  • StrategyNumber: 演算子戦略を識別する列挙型。

失敗モードと境界条件

  • 未知の strategy が渡された場合、elog(ERROR) により処理が中断されます。
  • nkeys が 0 の場合、ループ処理はスキップされ、戦略に応じた初期値が返されます。

保守担当者向けメモ

  • recheck フラグの扱いはインデックスの精度に直結するため、戦略を追加・変更する際は慎重に判断してください。
  • ブール検索のロジックは _int_bool.c に依存しているため、複雑な条件のデバッグ時はそちらを参照してください。
