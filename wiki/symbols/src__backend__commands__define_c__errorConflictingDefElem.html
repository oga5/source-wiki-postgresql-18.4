errorConflictingDefElem

定義

  • 種別: function
  • 定義箇所: src/backend/commands/define.c:370

目的

SQLコマンドの解析中に、重複または競合するオプション（DefElem）が検出された際、標準化された構文エラーを報告する。

シグネチャまたは形状

void errorConflictingDefElem(DefElem *defel, ParseState *pstate)

入力と出力

  • **入力**:
  • defel: 競合の原因となった DefElem 構造体へのポインタ。
  • pstate: 現在の解析状態を保持する ParseState 構造体へのポインタ。
  • **出力**: なし（エラー発生時に ereport を介して処理を中断する）。

前提条件と不変条件

  • defel および pstate は有効なポインタであること。
  • 本関数が呼び出されると、ERRCODE_SYNTAX_ERROR を伴うエラーが発行され、呼び出し元の制御フローは中断される。

アルゴリズムまたは振る舞い

1. ereport を呼び出し、エラーレベル ERROR を設定する。 2. エラーコードとして ERRCODE_SYNTAX_ERROR を指定する。 3. エラーメッセージとして "conflicting or redundant options" を出力する。 4. parser_errposition を使用し、defel->location に基づいて構文解析上のエラー位置を特定・表示する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/aclchk.c
  • src/backend/commands/collationcmds.c
  • src/backend/commands/copy.c
  • **呼び出し先**:
  • ereport（PostgreSQL のエラー報告マクロ）
  • parser_errposition（エラー位置特定用関数）

関連データ構造

  • DefElem: オプション定義を保持するノード構造体。
  • ParseState: SQL解析中の状態を管理する構造体。

失敗モードと境界条件

  • 本関数自体がエラーを報告する手段であるため、関数内部での失敗モードは定義されない。
  • defel が NULL の場合、呼び出し元でセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は、複数のオプションを処理する際に、既に設定済みの値に対して重複した指定があったことを示すための共通ユーティリティである。
  • 新しいコマンドでオプションの重複チェックを実装する際は、独自のエラーメッセージを生成せず、本関数を利用して一貫性を保つこと。
