test_icu

定義

  • 種別: function
  • 定義箇所: src/common/unicode/case_test.c:169

目的

Unicode の全コードポイントを走査し、PostgreSQL の大文字・小文字変換ロジックと ICU ライブラリの変換結果を比較検証する。

シグネチャまたは形状

static void test_icu(void)

入力と出力

  • 入力: なし（内部で 0 から 0x10ffff までの全コードポイントを走査）
  • 出力: なし（標準出力へテスト結果の統計を出力し、不一致がある場合は呼び出し元で exit(1) を引き起こす可能性がある）

前提条件と不変条件

  • USE_ICU が定義されている環境で実行されることを想定している。
  • ICU の Unicode バージョンと PostgreSQL 内蔵の Unicode データバージョンが一致していることが望ましい。

アルゴリズムまたは振る舞い

1. 0 から 0x10ffff までの全コードポイントをループで走査する。 2. unicode_category(code) を使用し、割り当てられていないコードポイントを除外する。 3. ICU の u_charType を呼び出し、ICU 側でも割り当て済みか確認する。バージョン不一致により ICU 側で未割り当ての場合はスキップし、カウントする。 4. icu_test_simple(code) を呼び出し、単一文字の変換結果を比較する。 5. unicode_to_utf8 で文字列化し、icu_test_full(code_str) を呼び出して文字列全体の変換結果を比較する。 6. 最後に、スキップ数と成功したコードポイント数を標準出力に報告する。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/unicode/case_test.c 内の main 関数（推測）。
  • 呼び出し先:
  • unicode_category
  • u_charType (ICU)
  • icu_test_simple
  • unicode_to_utf8
  • icu_test_full

関連データ構造

  • pg_wchar (コードポイント表現)
  • pg_unicode_category (Unicode カテゴリ分類)

失敗モードと境界条件

  • **バージョン不一致**: ICU と PostgreSQL の Unicode バージョンが異なる場合、一部のコードポイントで不一致が発生し、テストが失敗する可能性がある。
  • **未割り当てコードポイント**: PG_U_UNASSIGNED の判定により、テスト対象外のコードポイントは適切にスキップされる。

保守担当者向けメモ

  • 本関数は全コードポイントを網羅するため、実行には一定の時間を要する。
  • icu_test_simple や icu_test_full 内で不一致が検出された場合、それらの関数が異常終了をトリガーする設計となっているため、本関数自体は終了までループを継続する。
