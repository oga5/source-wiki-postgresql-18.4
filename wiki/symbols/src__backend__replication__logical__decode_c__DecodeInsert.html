DecodeInsert

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/decode.c:893

目的

WALのINSERTレコードを解析し、論理デコーディング用の変更情報として再構築して reorderbuffer にキューイングする。

シグネチャまたは形状

static void DecodeInsert(LogicalDecodingContext *ctx, XLogRecordBuffer *buf)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。
  • buf: 解析対象のWALレコードおよびLSN情報を含むバッファ。
  • **出力**: なし（副作用として reorderbuffer への変更追加を行う）。

前提条件と不変条件

  • buf->record は有効な XLogReaderState であること。
  • 処理対象のWALレコードがINSERT種別であることを前提とする。
  • xlrec->flags に XLH_INSERT_CONTAINS_NEW_TUPLE が含まれていない場合、処理は即座に終了する。

アルゴリズムまたは振る舞い

1. WALレコードから xl_heap_insert 構造体を取得する。 2. 新しいタプルを含まないレコード（XLH_INSERT_CONTAINS_NEW_TUPLE フラグなし）は無視する。 3. レコードのデータベースOIDが現在のスロットのデータベースと一致するか確認し、不一致なら終了する。 4. FilterByOrigin を呼び出し、レプリケーションオリジンによるフィルタリングを行う。 5. ReorderBufferAllocChange で変更オブジェクトを確保し、推測的挿入（Speculative Insert）か否かに応じてアクション種別を設定する。 6. XLogRecGetBlockData からタプルデータを抽出し、DecodeXLogTuple でデコードして change->data.tp.newtuple に格納する。 7. ReorderBufferQueueChange を呼び出し、トランザクションIDとLSNを紐付けてバッファにキューイングする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/decode.c 内のレコードディスパッチャ（LogicalDecodingProcessRecord 等から間接的に呼び出される）。
  • **呼び出し先**:
  • XLogRecGetData, XLogRecGetBlockTag, XLogRecGetOrigin, XLogRecGetBlockData, XLogRecGetXid
  • FilterByOrigin
  • ReorderBufferAllocChange, ReorderBufferAllocTupleBuf, ReorderBufferQueueChange
  • DecodeXLogTuple

関連データ構造

  • LogicalDecodingContext: デコーディング状態を保持。
  • XLogRecordBuffer: WALレコードのラッパー。
  • xl_heap_insert: ヒープ挿入のWALレコード構造体。
  • ReorderBufferChange: 論理変更を表現する構造体。
  • RelFileLocator: ターゲットリレーションの識別子。

失敗モードと境界条件

  • **データベース不一致**: ターゲットのDB OIDがスロットのDBと異なる場合、何もせず終了する。
  • **フィルタリング**: FilterByOrigin が真を返す場合、処理をスキップする。
  • **TOAST**: XLH_INSERT_ON_TOAST_RELATION フラグが立っている場合、キューイング時にその旨が伝達される。

保守担当者向けメモ

  • 本関数はヒープ挿入の物理的なWAL構造を論理的な変更に変換する責務を持つ。
  • DecodeXLogTuple によるタプル復元後のメモリ管理は reorderbuffer 側に委ねられる。
  • XLH_INSERT_IS_SPECULATIVE フラグの扱いは、PostgreSQLの推測的挿入（主に INSERT ... ON CONFLICT で使用）の論理レプリケーションにおける挙動を決定する。
