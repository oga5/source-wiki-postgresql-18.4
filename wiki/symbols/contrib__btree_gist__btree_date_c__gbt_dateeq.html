gbt_dateeq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_date.c:45

目的

2つの DateADT 型の値が等しいかどうかを判定し、GiST インデックスの比較処理をサポートします。

シグネチャまたは形状

static bool gbt_dateeq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の DateADT 型値へのポインタ（void * として渡される）。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されず、DirectFunctionCall2 に渡される）。
  • **出力**:
  • bool: a と b が等しい場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • a および b は有効な DateADT 型の値を指している必要があります。
  • PostgreSQL のコア関数 date_eq が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. 入力された void * 型のポインタ a および b を DateADT * にキャストし、値を参照します。 2. DateADTGetDatum マクロを使用して、各値を Datum 型に変換します。 3. DirectFunctionCall2 を介して PostgreSQL コアの date_eq 関数を呼び出し、比較を実行します。 4. 結果を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_date.c 内の静的テーブル（gbt_dateeq が登録されている箇所）から参照されています。
  • **呼び出し先**: PostgreSQL コアの date_eq 関数。

関連データ構造

  • DateADT: PostgreSQL の日付型を表現するデータ型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • date_eq 関数が内部でエラーを発生させた場合、その例外が伝播します。

保守担当者向けメモ

  • 本関数は btree_gist における date 型の比較ロジックのラッパーです。ロジックの変更が必要な場合は、コアの date_eq の仕様を確認してください。
