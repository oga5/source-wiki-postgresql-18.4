expect_boolean_value

定義

  • 種別: function
  • 定義箇所: src/common/compression.c:310

目的

圧縮オプションの文字列値を解析し、論理値（真偽値）へ変換します。

シグネチャまたは形状

static bool expect_boolean_value(char *keyword, char *value, pg_compress_specification *result);

入力と出力

  • **入力**:
  • keyword: オプション名（エラーメッセージに使用）。
  • value: 解析対象の文字列。
  • result: 解析結果を格納する pg_compress_specification 構造体へのポインタ。
  • **出力**:
  • bool: 解析結果が真であれば true、偽であれば false を返します。

前提条件と不変条件

  • value が NULL の場合、関数は即座に true を返します。
  • value が認識可能な真偽値文字列（後述）のいずれにも該当しない場合、result->parse_error にエラーメッセージが設定され、関数は false を返します。

アルゴリズムまたは振る舞い

1. value が NULL ならば true を返す。 2. value を大文字小文字を区別せずに比較する (pg_strcasecmp)。 3. 以下のいずれかに一致すれば true を返す:

  • "yes", "on", "1"

    • 4. 以下のいずれかに一致すれば false を返す:

  • "no", "off", "0"

    • 5. 上記以外の場合、psprintf を使用してエラーメッセージを生成し、result->parse_error に格納した上で false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/compression.c 内の圧縮設定解析処理（例: parse_compress_specification 内のループ処理）。
  • **呼び出し先**: pg_strcasecmp, psprintf（PostgreSQL 共通ユーティリティ）。

関連データ構造

  • pg_compress_specification: 圧縮設定を保持する構造体。本関数はこの構造体の parse_error フィールドを更新する可能性があります。

失敗モードと境界条件

  • **失敗**: 不正な文字列が渡された場合、result->parse_error が割り当てられます。呼び出し側はこのフィールドを確認し、必要に応じてメモリを解放する責任があります。
  • **境界**: value が NULL の場合は成功とみなされます。

保守担当者向けメモ

  • 本関数は pg_compress_specification の parse_error を更新するため、呼び出し側でエラー発生時のメモリ管理（pfree）が適切に行われているか確認してください。
  • 新たな真偽値表現を追加する場合は、pg_strcasecmp の比較リストを拡張してください。
