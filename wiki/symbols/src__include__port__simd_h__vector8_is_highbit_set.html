vector8_is_highbit_set

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/simd.h:270

目的

Vector8 型のベクトルに含まれる各要素のうち、少なくとも1つの要素で最上位ビット（MSB）がセットされているかどうかを判定します。

シグネチャまたは形状

static inline bool vector8_is_highbit_set(const Vector8 v)

入力と出力

  • **入力**: Vector8 v（プラットフォームに応じたベクトル型）
  • **出力**: bool（いずれかの要素の最上位ビットが 1 であれば true、すべて 0 であれば false）

前提条件と不変条件

  • Vector8 型は、コンパイル時の環境（SSE2, NEON, またはフォールバック）に応じて適切な SIMD 型または整数型にマッピングされている必要があります。

アルゴリズムまたは振る舞い

コンパイル時の機能フラグに応じて以下のいずれかの実装が選択されます。

  • **SSE2 (USE_SSE2)**: _mm_movemask_epi8 を使用し、各バイトの最上位ビットを抽出したマスク値が 0 でないかを判定します。
  • **NEON (USE_NEON)**: vmaxvq_u8 を使用し、ベクトル内の最大値が 0x7F を超える（すなわち最上位ビットがセットされた要素が存在する）かを判定します。
  • **フォールバック**: vector8_broadcast(0x80) で生成したマスクとのビット論理積（AND）をとり、結果が非ゼロであるかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/port/simd.h 内の内部ロジック
  • src/include/utils/ascii.h:72
  • **呼び出し先**:
  • vector8_broadcast（フォールバック時）
  • 各プラットフォーム固有の組み込み関数（_mm_movemask_epi8, vmaxvq_u8）

関連データ構造

  • Vector8: SIMD 演算用の抽象化されたベクトル型。

失敗モードと境界条件

  • SIMD 命令セットが利用できない環境では、標準的な整数演算によるフォールバックが行われるため、ハードウェア固有の例外は発生しません。
  • 入力ベクトルが空である場合の挙動は、Vector8 の定義に依存します。

保守担当者向けメモ

  • 本関数は vector32_is_highbit_set と類似のロジックを持ちますが、対象とする要素サイズが異なります。
  • USE_ASSERT_CHECKING が有効な環境では、SIMD 演算の正当性が逐次処理によって検証される可能性があります。
