is_log_level_output

定義

  • 種別: function
  • 定義箇所: src/backend/utils/error/elog.c:201

目的

指定されたエラーレベル（elevel）が、現在のログ出力設定レベル（log_min_level）に基づいてログ出力対象となるかを判定します。

シグネチャまたは形状

static inline bool is_log_level_output(int elevel, int log_min_level)

入力と出力

  • **入力**:
  • elevel: 判定対象のエラーレベル。
  • log_min_level: ログ出力の閾値となるレベル。
  • **出力**:
  • bool: 出力対象であれば true、そうでなければ false を返します。

前提条件と不変条件

  • elevel および log_min_level は、PostgreSQL のエラーレベル定義に従う整数値であることを前提とします。

アルゴリズムまたは振る舞い

以下のロジックで判定を行います。 1. elevel が LOG または LOG_SERVER_ONLY の場合: log_min_level が LOG または ERROR 以下であれば true。 2. elevel が WARNING_CLIENT_ONLY の場合: 常に false（ログには出力されません）。 3. log_min_level が LOG の場合: elevel が FATAL 以上であれば true。 4. 上記以外の場合: elevel >= log_min_level であれば true。

失敗モードと境界条件

  • WARNING_CLIENT_ONLY は log_min_level の設定に関わらず常に除外されます。
  • LOG レベルの扱いは、通常の深刻度比較とは異なる特殊な条件分岐が適用されます。
