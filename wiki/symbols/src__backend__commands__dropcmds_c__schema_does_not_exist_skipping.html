schema_does_not_exist_skipping

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dropcmds.c:173

目的

SQLの DROP 文において IF EXISTS が指定された際、指定されたスキーマが存在しない場合に、後続の処理をスキップするための警告メッセージと対象スキーマ名を準備する。

シグネチャまたは形状

static bool schema_does_not_exist_skipping(List *object, const char **msg, char **name)

入力と出力

  • **入力**:
  • object: 削除対象オブジェクトの識別子リスト（RangeVar への変換に使用）。
  • msg: 警告メッセージを格納するためのポインタ。
  • name: スキーマ名を格納するためのポインタ。
  • **出力**:
  • 戻り値: スキーマが存在しない場合は true、存在する場合は false。
  • msg および name: スキーマが存在しない場合、それぞれ適切なメッセージとスキーマ名が設定される。

前提条件と不変条件

  • object は makeRangeVarFromNameList で解析可能な形式であること。
  • LookupNamespaceNoError を使用してスキーマの存在を確認する。

アルゴリズムまたは振る舞い

1. object リストから RangeVar 構造体を生成する。 2. rel->schemaname が指定されており、かつ LookupNamespaceNoError が無効な OID を返す場合、以下の処理を行う。

  • msg に "schema \"%s\" does not exist, skipping" を設定する。
  • name に rel->schemaname を設定する。
  • true を返す。

    • 3. 上記以外の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dropcmds.c 内の各削除コマンド処理関数（RemoveObjects 等）から、IF EXISTS 処理の一環として呼び出される。
  • **呼び出し先**: makeRangeVarFromNameList, LookupNamespaceNoError。

関連データ構造

  • RangeVar: オブジェクトのスキーマおよび名前を保持する構造体。

失敗モードと境界条件

  • schemaname が NULL の場合（スキーマ指定がない場合）、この関数はスキーマ不在によるスキップ判定を行わず false を返す。

保守担当者向けメモ

  • 本関数は IF EXISTS 構文のユーザー向けエラーメッセージの統一を目的としている。メッセージを変更する場合は、他の does_not_exist_skipping 系関数との整合性に注意すること。
