ReplicationSlotNameForTablesync

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/tablesync.c:1301

目的

論理レプリケーションのテーブル同期プロセスで使用される、一意なレプリケーションスロット名を生成します。

シグネチャまたは形状

void ReplicationSlotNameForTablesync(Oid suboid, Oid relid, char *syncslotname, Size szslot)

入力と出力

  • **入力**:
  • suboid (Oid): サブスクリプションの識別子。
  • relid (Oid): 同期対象テーブルの識別子。
  • szslot (Size): syncslotname バッファのサイズ。
  • **出力**:
  • syncslotname (char *): 生成されたスロット名が格納されるバッファ。

前提条件と不変条件

  • syncslotname は、生成される文字列（pg_<suboid>_sync_<relid>_<system_identifier>）を格納するのに十分なサイズである必要があります。
  • システム識別子（GetSystemIdentifier()）に依存するため、同一データベースクラスタ内での一意性が保証されます。

アルゴリズムまたは振る舞い

1. snprintf を使用して、サブスクリプションOID、テーブルOID、およびシステム識別子を組み合わせた文字列を生成します。 2. 形式は "pg_%u_sync_%u_" UINT64_FORMAT です。 3. 生成された文字列を syncslotname バッファに書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c (複数箇所)
  • src/backend/replication/logical/tablesync.c (複数箇所)
  • **呼び出し先**:
  • GetSystemIdentifier() (システム識別子取得用)
  • snprintf (標準ライブラリ)

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • szslot が生成される文字列の長さに満たない場合、snprintf の仕様に従い文字列が切り詰められます。

保守担当者向けメモ

  • 本関数で生成されるスロット名は、論理レプリケーションのテーブル同期ワーカーがパブリッシャー側で作成する一時的なスロットを識別するために使用されます。
  • スロット名の形式を変更する場合は、関連するすべての呼び出し箇所およびスロット名を解析するロジックとの整合性に注意してください。
