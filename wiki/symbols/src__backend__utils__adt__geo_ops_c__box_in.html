box_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_ops.c:421

目的

文字列形式の入力からPostgreSQLの内部表現であるBOX構造体を生成します。

シグネチャまたは形状

Datum box_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して取得される、ボックスを表す文字列。
  • **出力**: BOX 構造体へのポインタを Datum 型として返却。解析失敗時は NULL を返却。

前提条件と不変条件

  • 入力文字列は path_decode 関数によって解析可能な形式である必要があります。
  • BOX 構造体はメモリ確保（palloc）後に座標の正規化が行われます。
  • 座標の正規化により、low（最小値）と high（最大値）の各軸の値が常に low <= high となることが保証されます。

アルゴリズムまたは振る舞い

1. path_decode を呼び出し、入力文字列から座標値を抽出して box->high に格納します。 2. 抽出された座標値に対し、float8_lt を用いて大小比較を行います。 3. high 座標が low 座標より小さい場合、値を入れ替えて正規化します。 4. 正規化された BOX 構造体を PG_RETURN_BOX_P マクロで返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLレベルでの型変換処理から呼び出されます（具体的な呼び出し元関数名はソース内に明示されていません）。
  • **呼び出し先**: path_decode（文字列解析用）、palloc（メモリ確保）、float8_lt（比較演算）。

関連データ構造

  • BOX: 幾何学的なボックスを表現する構造体（high と low の Point を保持）。
  • Point: x, y 座標を持つ構造体。

失敗モードと境界条件

  • path_decode が解析に失敗した場合、PG_RETURN_NULL() により処理が中断されます。
  • 浮動小数点数の比較には float8_lt を使用しており、厳密な等価性ではなく誤差を考慮した比較が行われます。

保守担当者向けメモ

  • 本関数は box_out と対になる入出力関数の一部です。
  • 座標の入れ替えロジックは、入力順序に依存せず常に正しいバウンディングボックスを保持するために不可欠です。
  • path_decode の第3引数 2 は、ボックスが2点（対角線上の2点）で定義されることを示しています。
