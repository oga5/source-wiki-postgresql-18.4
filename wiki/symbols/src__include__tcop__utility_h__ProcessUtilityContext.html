ProcessUtilityContext

定義

  • 種別: enum
  • 定義箇所: src/include/tcop/utility.h:20

目的

PostgreSQLにおけるユーティリティコマンド（CREATE, ALTER, DROP 等）の実行コンテキストを識別し、実行環境の階層や性質を区別するために使用されます。

シグネチャまたは形状

typedef enum
{
    PROCESS_UTILITY_TOPLEVEL,         /* トップレベルの対話型コマンド */
    PROCESS_UTILITY_QUERY,            /* 完全なクエリ（トップレベルではない） */
    PROCESS_UTILITY_QUERY_NONATOMIC,  /* 完全なクエリ（非アトミック実行コンテキスト） */
    PROCESS_UTILITY_SUBCOMMAND,       /* クエリの一部（サブコマンド） */
} ProcessUtilityContext;

入力と出力

  • 入力: ユーティリティコマンド実行関数（ProcessUtility 等）の引数として渡されます。
  • 出力: 実行関数内での条件分岐や、フック関数（ProcessUtility_hook）における実行制御の判断材料として利用されます。

前提条件と不変条件

  • ユーティリティコマンドが実行される際、必ずいずれか一つのコンテキスト値が割り当てられます。
  • PROCESS_UTILITY_SUBCOMMAND は、主に ALTER TABLE 等の再帰的な処理において使用されます。

アルゴリズムまたは振る舞い

  • 実行コンテキストに応じて、トランザクションの境界やエラーハンドリングの挙動が制御されます。
  • ProcessUtility はこの値に基づき、標準的な処理を行うか、あるいはフックを介した拡張機能の介入を許可するかを決定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tcop/utility.c の ProcessUtility 関数、src/backend/executor/spi.c、および contrib/pg_stat_statements や contrib/sepgsql などの拡張機能モジュールから参照されています。
  • 呼び出し先: 本シンボル自体は列挙型であるため、直接の呼び出し先はありません。この値を受け取る関数群が、このコンテキスト情報を使用して後続のコマンド実行処理を分岐させます。

関連データ構造

  • ProcessUtility_hook: ユーティリティ実行を監視・制御するフック関数ポインタ。
  • AlterTableUtilityContext: ALTER TABLE 等のサブコマンド実行時に引き継がれる関連コンテキスト。

失敗モードと境界条件

  • 不適切なコンテキスト値が渡された場合、コマンドの実行制御（特にトランザクションの整合性やリカバリ中の実行可否判定）が誤る可能性があります。

保守担当者向けメモ

  • 新しいユーティリティコマンドを追加する際は、そのコマンドがどのコンテキストで実行されるのが適切か（特に NONATOMIC かどうか）を検討してください。
  • 拡張機能がこのコンテキストを監視しているため、列挙値の順序変更や削除は互換性に影響を与える可能性があります。
