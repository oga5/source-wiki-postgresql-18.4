PreCommit_Portals

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:676

目的

トランザクション終了時（コミット時）に、全ポータルの状態を評価し、保持すべきカーソル（WITH HOLD）の永続化と、不要なポータルの破棄を行う。

シグネチャまたは形状

bool PreCommit_Portals(bool isPrepare)

入力と出力

  • **入力**:
  • isPrepare (bool): PREPARE TRANSACTION の実行中であるかを示すフラグ。
  • **出力**:
  • bool: ポータルの状態変更（破棄または永続化）が発生した場合に true を返す。

前提条件と不変条件

  • PortalHashTable が初期化されていること。
  • 実行中のポータル（PORTAL_ACTIVE）が存在する場合、そのリソース所有者（resowner）およびスナップショットの参照を解除し、ダングリングポインタを防ぐ。
  • portalPinned かつ autoHeld ではないポータルが存在する場合、エラー（elog(ERROR)）を発生させる。

アルゴリズムまたは振る舞い

1. PortalHashTable を走査する。 2. PORTAL_ACTIVE 状態のポータルについては、スナップショットとリソース所有者の参照をクリアする。 3. CURSOR_OPT_HOLD 指定があり、かつ現在のサブトランザクションで作成された PORTAL_READY 状態のポータルに対し、isPrepare が true の場合はエラーを投げる。それ以外の場合は HoldPortal() を呼び出して永続化する。 4. 保持対象外のポータル（createSubid が有効なもの）は PortalDrop() で破棄する。 5. ポータルの破棄や永続化によってハッシュテーブルの構造が変化する可能性があるため、操作のたびにハッシュシーケンスを再初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c (トランザクションコミット処理内)
  • **呼び出し先**:
  • HoldPortal()
  • PortalDrop()
  • UnregisterSnapshotFromOwner()

関連データ構造

  • PortalHashTable: 全ポータルを管理するハッシュテーブル。
  • PortalHashEnt: ハッシュテーブルの各エントリ。
  • Portal: ポータルの状態を保持する構造体。

失敗モードと境界条件

  • **エラー**: portalPinned かつ autoHeld でないポータルが残存している場合、コミットを中断する。
  • **エラー**: PREPARE TRANSACTION 中に WITH HOLD カーソルが存在する場合、サポート外としてエラーを投げる。
  • **再走査**: ポータルの破棄処理中にユーザー定義コードが実行され、ハッシュテーブルが変更される可能性があるため、常にシーケンスを再初期化する設計となっている。

保守担当者向けメモ

  • 本関数はトランザクションのコミット処理の根幹に関わるため、ポータルのライフサイクル管理（特に WITH HOLD カーソルの永続化ロジック）を変更する際は、PortalDrop との整合性に注意すること。
  • PORTAL_ACTIVE 状態のポータルに対する処理は、VACUUM 等のマルチトランザクションコマンドを考慮した安全策である。
