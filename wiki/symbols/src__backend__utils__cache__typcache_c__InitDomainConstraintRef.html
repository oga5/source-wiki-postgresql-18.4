InitDomainConstraintRef

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/typcache.c:1401

目的

ドメイン型の制約情報を評価するための DomainConstraintRef 構造体を初期化する。

シグネチャまたは形状

void InitDomainConstraintRef(Oid type_id, DomainConstraintRef *ref, MemoryContext refctx, bool need_exprstate)

入力と出力

  • type_id: 対象となるドメイン型の OID。
  • ref: 初期化対象の DomainConstraintRef 構造体へのポインタ。
  • refctx: 制約情報の保持に使用するメモリコンテキスト。
  • need_exprstate: 式の状態（ExprState）を構築する必要があるかを示すフラグ。
  • 出力: ref 構造体の各フィールドが設定され、refctx にリセットコールバックが登録される。

前提条件と不変条件

  • lookup_type_cache が返す TypeCacheEntry は、バックエンドの生存期間中永続することを前提とする。
  • refctx は、制約情報の生存期間を管理するために使用される。

アルゴリズムまたは振る舞い

1. lookup_type_cache を呼び出し、指定された type_id の型キャッシュエントリを取得する。 2. ref 構造体の基本フィールド（tcache, need_exprstate, refctx）を初期化する。 3. refctx に対して dccref_deletion_callback をリセットコールバックとして登録する。 4. 型キャッシュ内にドメイン制約情報（domainData）が存在する場合:

  • dccRefCount をインクリメントする。
  • need_exprstate が真であれば、prep_domain_constraints を呼び出して制約の式状態を構築する。
  • 偽であれば、既存の制約リストをそのまま参照する。

    • 5. 制約が存在しない場合は、ref->constraints を NIL に設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/executor/execExpr.c:3564
  • src/backend/utils/adt/domains.c:114
  • 呼び出し先:
  • lookup_type_cache
  • MemoryContextRegisterResetCallback
  • prep_domain_constraints
  • dccref_deletion_callback (コールバックとして登録)

関連データ構造

  • DomainConstraintRef: 本関数で初期化される構造体。
  • TypeCacheEntry: 型情報を保持するキャッシュエントリ。
  • DomainConstraintCache: ドメイン制約のキャッシュ構造体。

失敗モードと境界条件

  • 指定された type_id がドメイン型でない場合、または制約を持たない場合は ref->constraints が NIL となる。
  • メモリ不足時には MemoryContextRegisterResetCallback 等でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は DomainConstraintRef のライフサイクル管理を refctx のリセットコールバックに依存しているため、refctx の管理には注意が必要である。
  • dccRefCount のインクリメントは、キャッシュエントリの整合性を維持するために重要である。
