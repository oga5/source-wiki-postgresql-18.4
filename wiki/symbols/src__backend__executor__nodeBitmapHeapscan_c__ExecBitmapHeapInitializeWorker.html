ExecBitmapHeapInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapHeapscan.c:557

目的

並列ビットマップヒープスキャンを実行するワーカープロセスにおいて、共有メモリ上の並列実行状態（ParallelBitmapHeapState）および統計情報領域を初期化する。

シグネチャまたは形状

void ExecBitmapHeapInitializeWorker(BitmapHeapScanState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の BitmapHeapScanState 構造体。
  • pwcxt: 並列ワーカーのコンテキスト情報（共有メモリの TOC を含む）。
  • **出力**: なし（node 構造体の pstate および sinstrument フィールドが更新される）。

前提条件と不変条件

  • node->ss.ps.state->es_query_dsa が NULL でないこと（DSA が割り当てられていること）。
  • pwcxt->toc を通じて、現在のプランノード ID に対応する共有メモリ領域が参照可能であること。

アルゴリズムまたは振る舞い

1. shm_toc_lookup を使用し、プランノード ID をキーとして共有メモリ上の並列状態領域を検索する。 2. 取得したポインタを node->pstate に格納する。 3. ポインタを ParallelBitmapHeapState のサイズ分（MAXALIGN 調整済み）進める。 4. node->ss.ps.instrument が有効な場合、進めたポインタを node->sinstrument（共有統計情報）として割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1375（並列実行の初期化プロセスから呼び出される）。
  • **呼び出し先**: shm_toc_lookup（共有メモリの検索）。

関連データ構造

  • BitmapHeapScanState: スキャンノードの実行状態を保持。
  • ParallelBitmapHeapState: 並列実行時の共有状態。
  • SharedBitmapHeapInstrumentation: 並列実行時の統計情報共有用構造体。
  • ParallelWorkerContext: 並列ワーカーの共有メモリコンテキスト。

失敗モードと境界条件

  • shm_toc_lookup が指定された ID を見つけられない場合、エラーが発生する可能性がある（false 指定のため）。
  • 共有メモリのレイアウトが ParallelBitmapHeapState のサイズと一致しない場合、ポインタ演算により不正なメモリ領域を参照するリスクがある。

保守担当者向けメモ

  • 本関数は並列ワーカーの起動時に一度だけ実行される。
  • sinstrument の割り当ては instrument オプションが有効な場合にのみ行われるため、統計情報の有無による条件分岐に注意すること。
