PageGetItemIdCareful

定義

  • **種別**: 関数
  • **定義箇所**: contrib/amcheck/verify_nbtree.c:3493

目的

B-Tree インデックスページ内のアイテム識別子（ItemId）を安全に取得し、その内容が物理的な破損を示していないかを検証する。インデックス破損時に検証プロセス自体がクラッシュするのを防ぐための防御的実装である。

シグネチャまたは形状

static ItemId PageGetItemIdCareful(BtreeCheckState *state, BlockNumber block, Page page, OffsetNumber offset);

入力と出力

  • **入力**:
  • state: 検証状態を保持する BtreeCheckState 構造体。
  • block: 対象のブロック番号。
  • page: 対象のページへのポインタ。
  • offset: 取得対象のアイテムオフセット。
  • **出力**:
  • 正常時は対象の ItemId を返す。検証に失敗した場合はエラー（ERROR）を発生させ、処理を中断する。

前提条件と不変条件

  • nbtree の仕様上、ItemId は LP_REDIRECT であってはならず、かつ LP_UNUSED であってはならない。
  • nbtree 内のすべてのアイテムはストレージを持つ必要があり、長さが 0 であってはならない。

アルゴリズムまたは振る舞い

1. PageGetItemId を呼び出し、指定されたオフセットの ItemId を取得する。 2. 取得した ItemId が指す領域が、ページ内のタプル格納領域を超えていないかを確認する（BLCKSZ および BTPageOpaqueData のサイズに基づく境界チェック）。 3. ItemId のフラグおよび長さを検査し、LP_REDIRECT、LP_UNUSED、または長さ 0 の不正な状態でないかを確認する。 4. 上記のいずれかの条件に違反する場合、ERRCODE_INDEX_CORRUPTED を伴うエラーを報告する。 5. 検証を通過した場合、取得した ItemId を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_nbtree.c 内の各検証関数（bt_target_page_check 等）から、ページ内のアイテムを走査する際に呼び出される。
  • **呼び出し先**: PageGetItemId（PostgreSQL 内部マクロ）、ItemIdGetOffset、ItemIdGetLength、ItemIdGetFlags、RelationGetRelationName 等。
  • **注記**: contrib/amcheck/verify_gin.c にも同名の関数が存在するが、引数構成が異なるため、本ページで解説しているのは verify_nbtree.c で定義されたものに限定される。

関連データ構造

  • BtreeCheckState: 検証中のインデックス情報やリレーション情報を保持する。
  • ItemId: ページ内のアイテム位置を指す構造体。
  • BTPageOpaqueData: B-Tree ページの末尾に配置されるメタデータ。

失敗モードと境界条件

  • **破損検出**: インデックスページが物理的に破損しており、ラインポインタがページ末尾を超過している場合、または nbtree で使用されないフラグが設定されている場合にエラーを投げる。
  • **境界条件**: ページ内の有効なオフセット範囲外のアクセスについては、呼び出し元で適切に制御されている必要がある。

保守担当者向けメモ

  • 本関数は readonly モードでの検証において、破損したインデックスを安全に読み取るための重要な防御層である。
  • nbtree の構造変更に伴い、ItemId の有効性チェック条件が変更される可能性があるため、修正時は access/nbtree.h の定義と整合性を取ること。
