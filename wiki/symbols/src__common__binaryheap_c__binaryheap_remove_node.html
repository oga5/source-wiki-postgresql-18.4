binaryheap_remove_node

定義

  • 種別: function
  • 定義箇所: src/common/binaryheap.c:224

目的

バイナリヒープ内の指定された位置にあるノードを削除し、ヒーププロパティを維持した状態で構造を再構成します。

シグネチャまたは形状

void binaryheap_remove_node(binaryheap *heap, int n)

入力と出力

  • heap: 操作対象の binaryheap 構造体へのポインタ。
  • n: 削除対象ノードのインデックス（0 から heap->bh_size - 1 の範囲）。
  • 出力: なし（副作用としてヒープ構造が更新されます）。

前提条件と不変条件

  • ヒープが空であってはならない。
  • heap->bh_has_heap_property が真であること。
  • n は有効なインデックス範囲内（0 <= n < heap->bh_size）であること。

アルゴリズムまたは振る舞い

1. ヒープの末尾要素と、削除対象ノード n を比較します。 2. ヒープの末尾要素を削除し、その値を削除対象ノード n の位置に移動させます。 3. 移動させた要素と、元の削除対象ノードの比較結果に基づき、ヒーププロパティを復元します。

  • 末尾要素が削除対象より大きい場合: sift_up(heap, n) を実行。
  • 末尾要素が削除対象より小さい場合: sift_down(heap, n) を実行。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_backup_archiver.c:4750 での呼び出しが確認されています。
  • 呼び出し先: sift_up および sift_down を呼び出します。

関連データ構造

  • binaryheap: src/include/lib/binaryheap.h で定義されるヒープ構造体。

失敗モードと境界条件

  • Assert により、ヒープが空の場合やインデックスが範囲外の場合はプログラムが停止します。

保守担当者向けメモ

  • 本関数は、ヒープ内の任意のノードを削除する際に使用されます。ルート要素のみを削除する場合は binaryheap_remove_first の使用を検討してください。
  • 比較関数 bh_compare の結果に基づいて sift_up か sift_down のいずれか一方のみが実行されます。
