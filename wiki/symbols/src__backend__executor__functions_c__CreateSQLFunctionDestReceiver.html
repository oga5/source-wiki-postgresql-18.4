CreateSQLFunctionDestReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/executor/functions.c:2616

目的

SQL関数実行時の結果収集を行うための DestReceiver オブジェクトを生成し、初期化する。

シグネチャまたは形状

DestReceiver *CreateSQLFunctionDestReceiver(void)

入力と出力

  • **入力**: なし
  • **出力**: DestReceiver 型のポインタ。内部的には DR_sqlfunction 構造体として割り当てられる。

前提条件と不変条件

  • 呼び出し元で palloc0 によるメモリ割り当てが成功している必要がある。
  • 返される DestReceiver の mydest フィールドは DestSQLFunction に設定される。

アルゴリズムまたは振る舞い

1. DR_sqlfunction 構造体分のメモリをゼロクリアして割り当てる。 2. DestReceiver の関数ポインタ（receiveSlot, rStartup, rShutdown, rDestroy）を、SQL関数専用のハンドラ（sqlfunction_receive 等）に割り当てる。 3. mydest を DestSQLFunction に設定する。 4. プライベートフィールドは、後続の postquel_start 呼び出し時に設定されるため、この時点では未設定のまま返される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/dest.c:148 および src/backend/executor/functions.c:2614 からの参照が確認されている。
  • **呼び出し先**: 内部で palloc0 を使用する。各関数ポインタは sqlfunction_receive, sqlfunction_startup, sqlfunction_shutdown, sqlfunction_destroy を指す。

関連データ構造

  • DestReceiver: 汎用的な結果受信インターフェース。
  • DR_sqlfunction: SQL関数実行用に拡張された DestReceiver の具象構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。

保守担当者向けメモ

  • 本関数は DestReceiver のライフサイクル管理の一部であり、rDestroy に割り当てられた sqlfunction_destroy が適切にリソースを解放することを確認すること。
  • プライベートフィールドの初期化が postquel_start に依存しているため、実行順序の変更には注意が必要。
