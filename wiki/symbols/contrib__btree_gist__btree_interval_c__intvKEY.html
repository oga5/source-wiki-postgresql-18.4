intvKEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_interval.c:12

目的

PostgreSQL の interval 型を GiST インデックス内で表現するための構造体です。インデックスノードにおいて、interval 値の範囲（下限および上限）を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
    Interval    lower,
                upper;
} intvKEY;

入力と出力

  • **入力**: interval 型の値を gbt_intv_compress 関数等で変換する際に、この構造体のインスタンスが生成・格納されます。
  • **出力**: インデックス検索時、DatumGetPointer を介して intvKEY へのポインタとして取得され、範囲判定に使用されます。

前提条件と不変条件

  • lower は範囲の下限、upper は範囲の上限を表します。
  • interval 型のサイズは 16 バイトに固定されており、本構造体もその制約に基づいたメモリレイアウトを持ちます。

アルゴリズムまたは振る舞い

GiST インデックスのノード内で、特定の interval 値がこの構造体で定義された範囲内に含まれるか（consistent）、あるいはインデックスの分割や結合（union, picksplit）を行う際の境界値として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_interval.c 内の各 GiST サポート関数（gbt_intv_compress, gbt_intv_consistent 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • Interval: PostgreSQL の標準的な時間間隔型。
  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗: palloc を使用している箇所でメモリ不足が発生した場合、PostgreSQL のエラーハンドリングによりアボートします。
  • 範囲の逆転: lower が upper を超えるような不正な値が格納された場合、インデックスの検索結果が正しく得られない可能性があります。

保守担当者向けメモ

  • 本構造体は btree_gist モジュールにおける数値系 GiST ユーティリティ（btree_utils_num.h）と連携して動作します。
  • 構造体のサイズ変更は、インデックスの物理的な格納形式に直接影響するため注意が必要です。
