pthread_mutex_lock

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/pthread-win32.c:41

目的

Windows環境において、libpq が内部的に使用するミューテックスのロック処理を提供します。POSIXスレッドライブラリが存在しない環境で、CRITICAL_SECTION を用いた排他制御を実現します。

シグネチャまたは形状

int pthread_mutex_lock(pthread_mutex_t *mp);

入力と出力

  • **入力**: mp (初期化済みまたは未初期化の pthread_mutex_t 構造体へのポインタ)
  • **出力**: 成功時は 0 を返します。

前提条件と不変条件

  • mp は有効な pthread_mutex_t 構造体を指している必要があります。
  • 初回呼び出し時に、mp->initstate を用いた遅延初期化が実行されます。

アルゴリズムまたは振る舞い

1. **遅延初期化**: mp->initstate が 1（初期化済み）でない場合、初期化処理を開始します。

  • InterlockedExchange を使用して initstate を 2（初期化中）に設定し、他スレッドとの競合を回避します。
  • 他スレッドが初期化中の場合（istate == 2）、Sleep(0) を呼び出して待機します。

    • 2. **ロック取得**: EnterCriticalSection を呼び出し、OSレベルの排他制御に入ります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/ 配下の各ファイル（connect.c, misc.c 等）および src/interfaces/libpq/ 配下の fe-auth-oauth.c, fe-connect.c などから参照されています。
  • **呼び出し先**: Windows API の InterlockedExchange, Sleep, InitializeCriticalSection, EnterCriticalSection を呼び出します。

関連データ構造

  • pthread_mutex_t: src/port/pthread-win32.h で定義される、Windows環境用のミューテックス構造体。

失敗モードと境界条件

  • 本実装は EnterCriticalSection を使用しているため、同一スレッドによる再帰的なロックは CRITICAL_SECTION の仕様に従います。
  • InterlockedExchange によるスピンロック的な待機は、初期化の競合を避けるためのものであり、長時間の待機を想定した設計ではありません。

保守担当者向けメモ

  • 本関数は libpq 内部の同期ニーズに特化した実装です。汎用的な pthread ライブラリとしての完全性は備えていません。
  • initstate の管理にはアトミック操作が使用されており、マルチスレッド環境での安全性が確保されています。
