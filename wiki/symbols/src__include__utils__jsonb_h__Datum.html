Datum

定義

  • **種別**: typedef（PostgreSQLのコア型）
  • **定義箇所**: src/include/utils/jsonb.h:286（構造体メンバとしての利用例）

目的

PostgreSQLにおいて、任意のデータ型を汎用的に保持するための型です。jsonb モジュール内では、JSONの値をメモリ上で表現する際の基底として使用されます。

シグネチャまたは形状

Datum は、src/include/utils/jsonb.h 内の datetime 構造体において、以下のようにメンバとして定義されています。

struct {
    Datum value;
    Oid   typid;
    int32 typmod;
    int   tz;
} datetime;

入力と出力

  • **入力**: SQL層から渡される任意のデータ値（ポインタまたはスカラ値）。
  • **出力**: Datum 型として保持された値。

前提条件と不変条件

  • Datum は、値がポインタ（pass-by-reference）であるか、直接値（pass-by-value）であるかによって解釈が異なります。
  • jsonb 構造体内で使用される場合、適切な typid および typmod と組み合わせて管理される必要があります。

アルゴリズムまたは振る舞い

Datum 自体は単なる型定義であり、具体的な振る舞いは、この型を扱う関数（PG_GETARG_* や PG_RETURN_* マクロなど）に依存します。jsonb コンテキストでは、JSONの各要素を汎用的に保持・転送するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、Datum を直接呼び出す特定の関数は記載されていません。
  • **呼び出し先**: 提供されたソース情報には、Datum が直接呼び出す関数は記載されていません。
  • **注記**: contrib/amcheck 等の外部モジュールで Datum 型が参照されていることがツールにより示唆されていますが、これらは jsonb.h で定義された Datum そのものへの依存か、PostgreSQLコアの fmgr.h で定義された Datum への依存か、文脈上区別できません。

関連データ構造

  • JsonbValue: メモリ上でJSON要素を扱うための構造体。Datum を保持する場合があります。
  • Oid: Datum の型を識別するために併用されます。

失敗モードと境界条件

  • **型不整合**: Datum に格納された値と、typid が示す型が一致しない場合、後続の処理でメモリ破壊や不正なデータアクセスが発生する可能性があります。
  • **アライメント**: Datum をポインタとして扱う場合、プラットフォーム固有のアライメント要件を満たす必要があります。

保守担当者向けメモ

  • 本シンボルはPostgreSQLの基盤型であり、jsonb.h はその利用側の一例です。
  • Datum の定義自体は fmgr.h に由来します。jsonb.h 内での利用は、JSONデータとPostgreSQLの型システムを橋渡しする役割を担っています。
