ExpandedRecordGetRODatum

定義

  • 種別: function
  • 定義箇所: src/include/utils/expandedrecord.h:148

目的

展開されたレコード（ExpandedRecordHeader）から、読み取り専用（Read-Only）の Datum 表現を取得します。

シグネチャまたは形状

static inline Datum ExpandedRecordGetRODatum(const ExpandedRecordHeader *erh)

入力と出力

  • **入力**: erh - 展開されたレコードヘッダーへのポインタ。
  • **出力**: 展開されたレコードの読み取り専用 Datum 値。

前提条件と不変条件

  • erh は有効な ExpandedRecordHeader を指している必要があります。
  • 内部で EOHPGetRODatum を呼び出すため、ExpandedRecordHeader の先頭メンバである hdr（ExpandedObjectHeader）が適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った ExpandedRecordHeader のメンバ hdr を参照します。 2. EOHPGetRODatum 関数を呼び出し、展開されたオブジェクトの読み取り専用 Datum を取得して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/expandedrecord.c:1359
  • src/backend/utils/adt/expandedrecord.c:1561
  • src/backend/utils/adt/expandedrecord.c:1624

    • (いずれも domain_check 関数内での呼び出し)

  • **呼び出し先**:
  • EOHPGetRODatum（utils/expandeddatum.h で定義される基底関数）

関連データ構造

  • ExpandedRecordHeader: 展開されたレコードのメイン構造体。
  • ExpandedObjectHeader: ExpandedRecordHeader の先頭に埋め込まれた基底構造体。

失敗モードと境界条件

  • erh が NULL の場合の挙動は定義されていません。
  • 呼び出し先である EOHPGetRODatum の制約に依存します。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスを考慮して EOHPGetRODatum への単純なラッパーとして実装されています。
  • 変更を加える際は、ExpandedObjectHeader との整合性を維持してください。
