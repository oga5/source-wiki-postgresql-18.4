sqlvar_compat

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/include/sqlda-compat.h:8

目的

ECPG（Embedded SQL for C）において、Informix 互換の SQLDA（SQL Descriptor Area）を構成する個々の変数情報を保持するための構造体です。動的 SQL 実行時のホスト変数とデータベース間のデータ受け渡しを仲介します。

シグネチャまたは形状

struct sqlvar_compat {
    short sqltype;
    int sqllen;
    char *sqldata;
    short *sqlind;
    char *sqlname;
    char *sqlformat;
    short sqlitype;
    short sqlilen;
    char *sqlidata;
    int sqlxid;
    char *sqltypename;
    short sqltypelen;
    short sqlownerlen;
    short sqlsourcetype;
    char *sqlownername;
    int sqlsourceid;
    char *sqlilongdata;
    int sqlflags;
    void *sqlreserved;
};

入力と出力

  • **入力**: ECPG ランタイムライブラリが SQL 文を実行する際、アプリケーションによって各フィールド（型、データポインタ、インジケータ等）が設定されます。
  • **出力**: データベースからのクエリ結果が sqldata や sqlilongdata に格納され、アプリケーション側で参照されます。

前提条件と不変条件

  • Informix の仕様に準拠したフィールド配置を維持する必要があります。
  • sqlilen および sqlidata は 32KB 以下のデータ用であり、それ以上のデータには sqlilongdata を使用します。
  • sqlflags および sqlreserved は内部利用または将来の拡張用であり、アプリケーションによる直接操作は推奨されません。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを目的としており、ロジックは含みません。ECPG ランタイムがこの構造体を走査し、SQL 文のバインド変数や結果セットの格納先として利用します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/sqlda.c 内の関数が、この構造体を動的に割り当てたり、ポインタ演算によってアクセスしたりしています。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • struct sqlda_compat: 本構造体を配列として保持する SQLDA のメイン構造体。
  • sqlvar_t: src/interfaces/ecpg/include/sqlda.h 等で定義される struct sqlvar_compat の別名（typedef）。

失敗モードと境界条件

  • 32KB を超えるデータを取り扱う場合、従来の sqllen や sqlilen では不足するため、sqlilongdata を使用する必要があります。
  • 構造体のメモリレイアウトは Informix 互換であるため、異なる環境間でのバイナリ互換性には注意が必要です。

保守担当者向けメモ

  • sqlilongdata は 32KB 制限を回避するための拡張フィールドです。
  • sqlflags や sqlreserved の変更は、Informix 互換性に影響を与える可能性があるため慎重に行ってください。
