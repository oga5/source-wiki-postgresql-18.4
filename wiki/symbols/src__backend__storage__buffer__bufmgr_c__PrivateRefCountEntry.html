PrivateRefCountEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/buffer/bufmgr.c:93

目的

各バックエンドプロセスが保持するバッファのピン留め（pin）回数を追跡するために使用されます。

シグネチャまたは形状

typedef struct PrivateRefCountEntry
{
	Buffer		buffer;    /* バッファID */
	int32		refcount;  /* 現在のピン留め回数 */
} PrivateRefCountEntry;

入力と出力

  • 本構造体は、特定のバッファに対するプロセスローカルな参照カウントを保持します。
  • 入力: Buffer 型のバッファ識別子。
  • 出力: int32 型の参照カウント値。

前提条件と不変条件

  • 各バックエンドは、自身がピン留めしたバッファの回数をこの構造体を通じて管理します。
  • 参照カウントが 0 になった場合、当該エントリは解放される対象となります。

アルゴリズムまたは振る舞い

  • バックエンドがバッファをピン留めする際、対応する PrivateRefCountEntry が検索または生成され、refcount がインクリメントされます。
  • バッファのピンを解放する際、refcount がデクリメントされ、0 に達した場合はエントリが破棄されます。

呼び出し元と呼び出し先

  • 呼び出し元: bufmgr.c 内のバッファ管理関数群（NewPrivateRefCountEntry, GetPrivateRefCountEntry 等）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出し先はありません。

関連データ構造

  • Buffer: バッファを識別するための型。
  • PrivateRefCountArray: バックエンドが保持するエントリの配列。
  • ReservedRefCountEntry: 予約済みのエントリを指すポインタ。

失敗モードと境界条件

  • エントリの検索や生成に失敗した場合の挙動については、ソースコードの文脈からは明示されていません。
  • 配列の容量制限（REFCOUNT_ARRAY_ENTRIES）を超過した場合の挙動については、本ソース抜粋からは不明です。

保守担当者向けメモ

  • 本構造体はプロセスローカルな管理用であり、共有メモリ上の BufferDesc とは独立してピン留め状態を追跡します。
  • デバッグ時には CheckForBufferLeaks() 等を用いて、本構造体のカウントと実際のピン留め状態に乖離がないか確認してください。
