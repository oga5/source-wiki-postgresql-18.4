ExecInitMergeTupleSlots

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeModifyTable.c:3964

目的

MERGE 操作において、ターゲットリレーションに対するタプル処理用スロット（旧タプル用および新タプル用）を初期化し、ResultRelInfo に割り当てる。

シグネチャまたは形状

void ExecInitMergeTupleSlots(ModifyTableState *mtstate, ResultRelInfo *resultRelInfo)

入力と出力

  • **入力**:
  • mtstate: ModifyTableState 構造体へのポインタ。実行状態と EState を保持。
  • resultRelInfo: ResultRelInfo 構造体へのポインタ。初期化対象のリレーション情報。
  • **出力**: なし（副作用として resultRelInfo のメンバを更新）。

前提条件と不変条件

  • resultRelInfo->ri_projectNewInfoValid が false であること（アサーションにより検証）。
  • resultRelInfo->ri_RelationDesc が有効なリレーション記述子として設定されていること。

アルゴリズムまたは振る舞い

1. mtstate から実行状態 (EState) を取得する。 2. table_slot_create を呼び出し、ri_RelationDesc に基づく ri_oldTupleSlot を作成する。 3. 同様に ri_newTupleSlot を作成する。 4. resultRelInfo->ri_projectNewInfoValid を true に設定し、初期化完了を示す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeModifyTable.c:3726
  • src/backend/executor/execPartition.c:900
  • **呼び出し先**:
  • table_slot_create（テーブルアクセスメソッド層の関数）

関連データ構造

  • ModifyTableState: 実行ノードの状態を保持。
  • ResultRelInfo: ターゲットリレーションごとの実行時情報を保持。
  • EState: クエリ実行全体のグローバルな状態。

失敗モードと境界条件

  • メモリ不足時: table_slot_create 内部でメモリ割り当てエラーが発生する可能性がある。
  • ri_RelationDesc が未設定の場合: table_slot_create 呼び出し時にクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は MERGE 操作の準備フェーズで呼び出される。
  • ri_projectNewInfoValid フラグは、スロットの二重初期化を防ぐためのガードとして機能している。
