pglz_decompress

定義

  • 種別: function
  • 定義箇所: src/common/pg_lzcompress.c:691

目的

LZ77 アルゴリズムで圧縮されたデータを伸長し、元のデータを復元する。

シグネチャまたは形状

int32 pglz_decompress(const char *source, int32 slen, char *dest, int32 rawsize, bool check_complete)

入力と出力

  • **入力**:
  • source: 圧縮済みデータへのポインタ。
  • slen: 圧縮済みデータのバイト長。
  • dest: 伸長先バッファへのポインタ。
  • rawsize: 伸長後の期待されるデータサイズ。
  • check_complete: 伸長完了後にデータ長の一致を確認するかどうかのフラグ。
  • **出力**:
  • 成功時: 伸長されたデータのバイト数（int32）。
  • 失敗時: -1。

前提条件と不変条件

  • dest バッファは少なくとも rawsize バイトの領域を確保していること。
  • 圧縮データが破損している場合、または off がバッファ境界外を指す場合は -1 を返す。

アルゴリズムまたは振る舞い

1. 制御バイトを読み取り、各ビット（8ビット分）に基づいてリテラルコピーまたは一致タグ（match tag）の処理を行う。 2. **リテラル**: 制御ビットが 0 の場合、入力から 1 バイトをそのまま出力へコピーする。 3. **一致タグ**: 制御ビットが 1 の場合、オフセットと長さを抽出し、出力バッファ内の過去のデータからコピーを行う。

  • 長さが 18 の場合は拡張タグバイトを読み取り、長さを加算する。
  • コピー元とコピー先が重なる場合（off < len）、memcpy の未定義動作を避けるため、off を倍加させながら段階的にコピーを行う。

    • 4. check_complete が真の場合、処理終了時に source と dest の消費量が期待値と一致するか検証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/toast_compression.c
  • src/backend/access/transam/xlogreader.c
  • src/test/regress/regress.c
  • **呼び出し先**:
  • 標準ライブラリ関数: memcpy
  • マクロ: Min, unlikely

関連データ構造

  • PGLZ_Strategy（圧縮戦略定義、本関数は伸長時に直接参照しないが、圧縮形式の前提となる）

失敗モードと境界条件

  • 入力データが不完全（タグ読み取り時に srcend を超える）な場合、-1 を返す。
  • オフセットが 0 またはバッファの開始位置より前を指す場合、データ破損とみなして -1 を返す。
  • check_complete が真で、処理後のポインタ位置が期待値と異なる場合、-1 を返す。

保守担当者向けメモ

  • 伸長時の memcpy は、コピー元とコピー先が重なるケース（off < len）を考慮した再帰的なコピー処理を含んでいる。
  • 圧縮形式は最大 4095 バイトのオフセットと 273 バイトの長さをサポートする LZ77 の変種である。
