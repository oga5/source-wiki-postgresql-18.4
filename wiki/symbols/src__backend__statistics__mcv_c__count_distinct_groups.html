count_distinct_groups

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mcv.c:378

目的

ソート済みの SortItem 配列において、重複を除いたグループの総数をカウントします。

シグネチャまたは形状

static int count_distinct_groups(int numrows, SortItem *items, MultiSortSupport mss)

入力と出力

  • **入力**:
  • numrows: 配列内の要素数。
  • items: 比較対象となる SortItem の配列。
  • mss: 比較ロジックを保持する MultiSortSupport 構造体。
  • **出力**:
  • int: 配列内に存在する一意なグループの数。

前提条件と不変条件

  • items 配列は、mss に基づいて昇順にソートされている必要があります。
  • numrows が 0 より大きい場合、少なくとも 1 つのグループが存在するとみなされます。

アルゴリズムまたは振る舞い

1. 初期値 ndistinct = 1 を設定します。 2. 配列の 2 番目の要素から順に、直前の要素と比較します。 3. 各ステップで multi_sort_compare を呼び出し、配列がソートされていることを Assert で検証します。 4. 直前の要素と現在の要素が異なる場合（!= 0）、ndistinct をインクリメントします。 5. 全要素の走査終了後、最終的な ndistinct を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/mcv.c 内の関数から呼び出されています（例: build_distinct_groups 等の文脈）。
  • **呼び出し先**: multi_sort_compare を呼び出します。

関連データ構造

  • SortItem: 比較対象となるデータ項目。
  • MultiSortSupport: 複数の列や式を比較するためのサポート構造体。

失敗モードと境界条件

  • numrows が 0 の場合、ループは実行されず初期値の 1 が返されます（空の入力に対する挙動は呼び出し側の設計に依存します）。
  • 配列がソートされていない場合、Assert が有効なビルドではプログラムが停止します。

保守担当者向けメモ

  • 本関数はソート済みであることを前提としているため、呼び出し前に必ずソート処理が完了していることを確認してください。
  • multi_sort_compare の結果が 0 未満になるケースは、ソート順序の不整合を意味するため Assert で保護されています。
