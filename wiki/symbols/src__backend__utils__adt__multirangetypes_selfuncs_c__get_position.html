get_position

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/multirangetypes_selfuncs.c:793

目的

ヒストグラムの2つの境界値（hist1, hist2）の間における、対象値（value）の相対的な位置（0.0から1.0の範囲）を算出します。

シグネチャまたは形状

static float8 get_position(TypeCacheEntry *typcache, const RangeBound *value, const RangeBound *hist1, const RangeBound *hist2)

入力と出力

  • **入力**:
  • typcache: 型キャッシュエントリ（rng_subdiff_finfo を使用）。
  • value: 位置を特定したい対象の境界値。
  • hist1, hist2: ビンの下限および上限境界。
  • **出力**:
  • float8: 0.0から1.0の間の相対位置。

前提条件と不変条件

  • typcache に有効な rng_subdiff_finfo が存在する場合、それを用いて線形補間を行います。
  • hist1 と hist2 が有限の場合、value も有限であることが期待されます。
  • 戻り値は常に [0.0, 1.0] の範囲にクランプされます。

アルゴリズムまたは振る舞い

1. **有限値の場合**: subdiff 関数を使用して hist1 から hist2 までの幅を計算し、hist1 から value までの距離をその幅で除算します。 2. **無限値の処理**:

  • 両方の境界が有限で value が無限の場合は 0.5 を返します。
  • hist1 が無限で hist2 が有限の場合、value が無限なら 0.0、そうでなければ 1.0 を返します。
  • hist1 が有限で hist2 が無限の場合、value が無限なら 1.0、そうでなければ 0.0 を返します。
  • 両方が無限の場合は 0.5 を返します。

    • 3. **エラー回避**: subdiff が NaN を返す場合や、計算結果が不正な場合は 0.5 を返してフォールバックします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/multirangetypes_selfuncs.c 内の複数箇所（統計計算ロジック）および src/backend/utils/adt/rangetypes_selfuncs.c から呼び出されます。
  • **呼び出し先**: typcache 内の rng_subdiff_finfo（関数ポインタ経由）。

関連データ構造

  • RangeBound: 範囲の境界値を保持する構造体。
  • TypeCacheEntry: 型固有の比較関数やサブ差分関数を保持するキャッシュ構造体。

失敗モードと境界条件

  • subdiff 関数が未定義の場合、常に 0.5 を返します。
  • bin_width が 0 または NaN の場合、計算を中断し 0.5 を返します。
  • value が範囲外にある場合、Max(position, 0.0) および Min(position, 1.0) により境界値にクランプされます。

保守担当者向けメモ

  • 本関数は rangetypes_selfuncs.c と共通のロジックを持っています。修正時は両方のファイルへの影響を考慮してください。
  • subdiff 関数が正しく定義されていない型では精度が低下するため、型定義側の実装に依存します。
