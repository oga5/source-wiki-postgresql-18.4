XLogRecordHasFPW

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/pg_waldump.c:468

目的

指定されたWALレコード内に、フルページイメージ（FPW: Full Page Write）が含まれているかどうかを判定する。

シグネチャまたは形状

static bool XLogRecordHasFPW(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record （解析対象のWALレコード状態）
  • **出力**: bool （レコード内に少なくとも1つのブロックイメージが含まれていれば true、そうでなければ false）

前提条件と不変条件

  • record は有効な XLogReaderState 構造体を指している必要がある。
  • XLogRecMaxBlockId を使用してレコード内のブロックID範囲を走査するため、レコードが正しくデコード済みであること。

アルゴリズムまたは振る舞い

1. レコード内の最大ブロックID（XLogRecMaxBlockId(record)）までループを実行する。 2. 各ブロックIDについて、XLogRecHasBlockRef(record, block_id) を呼び出し、参照が存在するか確認する。 3. 参照が存在する場合、XLogRecHasBlockImage(record, block_id) を呼び出し、そのブロックにフルページイメージが含まれているかを確認する。 4. イメージが見つかった時点で直ちに true を返し、ループを終了する。 5. 全ブロックを確認してもイメージが見つからない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_waldump/pg_waldump.c 内のメインループ（config.filter_by_fpw が有効な場合にレコードのフィルタリング条件として使用）。
  • **呼び出し先**:
  • XLogRecMaxBlockId
  • XLogRecHasBlockRef
  • XLogRecHasBlockImage

関連データ構造

  • XLogReaderState: WALレコードの読み込み状態を保持する構造体。

失敗モードと境界条件

  • record が NULL の場合、あるいは初期化されていない場合は未定義の動作となる。
  • レコード内にブロック参照が全く含まれない場合、ループは即座に終了し false を返す。

保守担当者向けメモ

  • 本関数は pg_waldump のフィルタリング機能の一部である。XLogReader インターフェースの変更に伴い、ブロック参照やイメージの判定ロジックに変更が生じた場合は本関数の更新が必要となる。
