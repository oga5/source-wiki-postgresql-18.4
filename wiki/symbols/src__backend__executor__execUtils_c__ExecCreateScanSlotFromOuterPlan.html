ExecCreateScanSlotFromOuterPlan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execUtils.c:703

目的

プランノードの実行状態（ScanState）に対し、その下位プラン（outer plan）の出力形式に基づいたタプルスロットを初期化します。

シグネチャまたは形状

void ExecCreateScanSlotFromOuterPlan(EState *estate, ScanState *scanstate, const TupleTableSlotOps *tts_ops)

入力と出力

  • estate: クエリ実行全体の状態を保持する EState 構造体へのポインタ。
  • scanstate: 初期化対象となる ScanState 構造体へのポインタ。
  • tts_ops: スロットの操作方法を定義する TupleTableSlotOps 構造体へのポインタ。
  • 出力: なし（scanstate 内のタプルスロットが初期化されます）。

前提条件と不変条件

  • scanstate が有効な outerPlanState を保持している必要があります。
  • ExecGetResultType を通じて、下位プランから有効な TupleDesc が取得できる必要があります。

アルゴリズムまたは振る舞い

1. outerPlanState(scanstate) を呼び出し、対象の ScanState に紐付く下位プランノードを取得します。 2. ExecGetResultType を使用して、下位プランの出力タプル記述子（TupleDesc）を取得します。 3. 取得した TupleDesc と指定された tts_ops を用いて、ExecInitScanTupleSlot を呼び出し、スロットを生成・設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nodeAgg.c
  • nodeGroup.c
  • nodeIncrementalSort.c
  • nodeMaterial.c
  • nodeMemoize.c
  • nodeSort.c
  • nodeWindowAgg.c
  • **呼び出し先**:
  • outerPlanState (マクロ)
  • ExecGetResultType
  • ExecInitScanTupleSlot

関連データ構造

  • EState: クエリ実行状態。
  • ScanState: スキャン操作の状態。
  • TupleTableSlotOps: タプルスロットの操作関数テーブル。
  • TupleDesc: タプルの構造定義。

失敗モードと境界条件

  • 下位プランが存在しない場合や、ExecGetResultType が NULL を返すような不正なプラン構造の場合、後続の ExecInitScanTupleSlot でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は、複数のプランノードで共通して行われる「下位プランの出力を受け取るためのスロット作成」を抽象化したものです。
  • tts_ops に渡す値（TTSOpsMinimalTuple や TTSOpsVirtual など）は、各ノードの特性に応じて適切に選択されています。
