replace_nestloop_param_placeholdervar

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/paramassign.c:462

目的

ネストループ結合において、上位レベルの PlaceHolderVar を実行時に参照可能な PARAM_EXEC パラメータへ置換し、必要に応じて PlannerInfo の curOuterParams リストへ登録する。

シグネチャまたは形状

Param *replace_nestloop_param_placeholdervar(PlannerInfo *root, PlaceHolderVar *phv)

入力と出力

  • **入力**:
  • root: 現在のプラン作成状態を保持する PlannerInfo 構造体。
  • phv: 置換対象の PlaceHolderVar。
  • **出力**:
  • Param: 置換後の PARAM_EXEC ノード。

前提条件と不変条件

  • root->curOuterParams は、現在のネストループ結合で利用可能なパラメータのリストとして維持されている必要がある。
  • phv の式（phexpr）から型、型修飾子（typmod）、照合順序（collation）が取得可能であること。

アルゴリズムまたは振る舞い

1. root->curOuterParams を走査し、指定された phv と等価な NestLoopParam が既に存在するか確認する。 2. 存在する場合:

  • その NestLoopParam が保持する paramno を使用して新しい Param ノードを作成し、即座に返す。

    • 3. 存在しない場合:

  • generate_new_exec_param を呼び出し、新規の PARAM_EXEC ID を割り当てる。
  • 新しい NestLoopParam ノードを作成し、phv をコピーして格納する。
  • root->curOuterParams に新しい NestLoopParam を追加する。
  • 作成した Param ノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c:5095
  • **呼び出し先**:
  • equal (マクロ)
  • makeNode (マクロ)
  • exprType, exprTypmod, exprCollation (型情報取得)
  • generate_new_exec_param
  • copyObject
  • lappend

関連データ構造

  • PlannerInfo: プラン作成の全体状態を管理。
  • PlaceHolderVar: プレースホルダ変数。
  • NestLoopParam: ネストループ結合で受け渡されるパラメータの定義。
  • Param: 実行時に使用されるパラメータノード。

失敗モードと境界条件

  • phv が NULL の場合の挙動は定義されていない。
  • root->curOuterParams が空の場合、常に新規の PARAM_EXEC ID が割り当てられる。

保守担当者向けメモ

  • 本関数は replace_nestloop_param_var と対になる処理を行う。
  • root->curOuterParams への追加はプラン作成の文脈に依存するため、プランノード生成の順序に注意すること。
