mdc_init

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-decrypt.c:323

目的

PGP 復号パイプラインにおける MDC（Modification Detection Code）検証用のコンテキストを初期化します。

シグネチャまたは形状

static int mdc_init(void **priv_p, void *arg, PullFilter *src)

入力と出力

  • **入力**:
  • priv_p: 初期化されたコンテキストを格納するためのポインタ。
  • arg: PGP_Context 構造体へのポインタ。
  • src: 読み取り元となる PullFilter（使用されません）。
  • **出力**:
  • 成功時は 0 を返し、*priv_p に PGP_Context が設定されます。
  • 失敗時はエラーコードを返します。

前提条件と不変条件

  • arg は有効な PGP_Context を指している必要があります。
  • pgp_load_digest が内部で呼び出され、SHA-1 ダイジェストコンテキストが ctx->mdc_ctx に割り当てられる必要があります。

アルゴリズムまたは振る舞い

1. arg から PGP_Context を取得します。 2. priv_p に PGP_Context を代入します。 3. pgp_load_digest を呼び出し、PGP_DIGEST_SHA1 を使用して MDC 用のダイジェストコンテキストを初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-decrypt.c 内のフィルタ初期化処理から参照されています。
  • **呼び出し先**: pgp_load_digest（ダイジェストコンテキストの初期化）。
  • 注: contrib/pgcrypto/pgp-encrypt.c にも同名のシンボルが存在しますが、本ページは pgp-decrypt.c の実装を対象としています。

関連データ構造

  • PGP_Context: 復号状態を保持する構造体。
  • PullFilter: フィルタチェーンを構成する構造体。

失敗モードと境界条件

  • pgp_load_digest が失敗した場合、その戻り値がそのまま返されます。

保守担当者向けメモ

  • 本関数は PullFilter の初期化関数として設計されています。
  • pgp-encrypt.c にも同名の関数が存在するため、コードの追跡時にはファイルパスに注意してください。
