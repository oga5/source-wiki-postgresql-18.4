pad_eme_pkcs1_v15

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubenc.c:39

目的

公開鍵暗号化の準備として、入力データをPKCS#1 v1.5形式に従ってパディングする。

シグネチャまたは形状

int pad_eme_pkcs1_v15(uint8 *data, int data_len, int res_len, uint8 **res_p)

入力と出力

  • data: パディング対象のデータ（セッション鍵とチェックサム等）。
  • data_len: 入力データの長さ。
  • res_len: パディング後の総バイト数（公開鍵のサイズに依存）。
  • res_p: パディング済みデータが格納されたバッファへのポインタを返す出力引数。
  • 戻り値: 成功時は 0、失敗時は PXE_BUG または PXE_NO_RANDOM を返す。

前提条件と不変条件

  • パディング領域（res_len - 2 - data_len）は8バイト以上である必要がある。
  • パディング領域にはゼロバイトを含めてはならない。
  • 成功時、*res_p には palloc で確保された res_len バイトのメモリが割り当てられる。

アルゴリズムまたは振る舞い

1. パディング長を計算し、8バイト未満の場合は PXE_BUG を返す。 2. res_len バイトのバッファを確保し、先頭バイトを 0x02（PKCS#1 v1.5のブロックタイプ）に設定する。 3. pg_strong_random を使用して、パディング領域を非ゼロの乱数で埋める。 4. パディング領域内にゼロバイトが含まれた場合、再度 pg_strong_random を呼び出して非ゼロ値で上書きする。 5. パディング領域の直後に 0x00 を配置し、その後に元の data をコピーする。 6. 成功時に 0 を返し、呼び出し元へバッファの所有権を渡す。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-pubenc.c 内の処理（pgp_write_pubenc_sesskey からのフローで利用）。
  • 呼び出し先: palloc, pfree, pg_strong_random, px_memset, memcpy。

関連データ構造

  • uint8: 符号なし8ビット整数。

失敗モードと境界条件

  • pad_len < 8: パディング領域が不足している場合に PXE_BUG を返す。
  • 乱数生成失敗: pg_strong_random が失敗した場合、メモリを解放し PXE_NO_RANDOM を返す。

保守担当者向けメモ

  • 本関数で確保されたメモリは、呼び出し元で適切に解放される必要がある。
  • セキュリティ上の理由から、パディング処理中にゼロバイトを排除する厳密な制御を行っている。
  • 処理中にエラーが発生した場合、機密情報を含む可能性のあるバッファは px_memset でゼロクリアしてから解放される。
