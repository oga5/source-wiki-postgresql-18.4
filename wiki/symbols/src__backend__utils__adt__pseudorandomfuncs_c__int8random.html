int8random

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pseudorandomfuncs.c:149

目的

指定された範囲内（rmin 以上 rmax 以下）で一様分布に従う 64 ビット整数（int8）の疑似乱数を生成し、SQL インターフェースへ返却します。

シグネチャまたは形状

Datum int8random(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_INT64(0): 乱数範囲の下限値 (rmin)
  • PG_GETARG_INT64(1): 乱数範囲の上限値 (rmax)
  • **出力**:
  • Datum: 生成された 64 ビット整数値

前提条件と不変条件

  • rmin <= rmax である必要があります。この条件を満たさない場合、エラーが発生します。
  • 内部で使用する prng_state は、初回呼び出し時に initialize_prng() によって初期化されます。

アルゴリズムまたは振る舞い

1. 引数 rmin と rmax を取得します。 2. rmin > rmax の場合、ERRCODE_INVALID_PARAMETER_VALUE を伴うエラーを報告します。 3. initialize_prng() を呼び出し、乱数生成器の状態を初期化します。 4. pg_prng_int64_range(&prng_state, rmin, rmax) を呼び出し、指定範囲内の乱数を取得します。 5. 取得した値を PG_RETURN_INT64 マクロを通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（具体的な SQL クエリからの呼び出し）。
  • **呼び出し先**:
  • initialize_prng()
  • pg_prng_int64_range()

関連データ構造

  • prng_state: プロセス単位で保持される乱数生成器の状態構造体。

失敗モードと境界条件

  • **エラー**: rmin が rmax より大きい場合、ereport によりエラーが発生し、処理が中断されます。
  • **境界条件**: rmin と rmax が等しい場合、常にその値が返されます。

保守担当者向けメモ

  • 本関数は pg_prng ライブラリに依存しています。乱数生成アルゴリズムの変更や改善が必要な場合は、common/pg_prng.h および関連する pg_prng 実装を確認してください。
  • prng_state はバックエンドプロセスごとに静的変数として保持されるため、セッション間での乱数系列の共有は行われません。
