to_regproc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regproc.c:117

目的

関数名（文字列）を対応するシステムカタログ上の OID に変換します。エラー発生時に例外をスローせず、安全に NULL を返すためのラッパー関数です。

シグネチャまたは形状

Datum to_regproc(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介して渡される関数名（text 型）。
  • **出力**: 成功した場合は対応する関数 OID（Datum）、失敗した場合は NULL。

前提条件と不変条件

  • 入力文字列は text_to_cstring によって C 文字列に変換されます。
  • ErrorSaveContext を使用してエラーを捕捉するため、呼び出し元でトランザクションがアボートされることを防ぎます。

アルゴリズムまたは振る舞い

1. 入力された text 型の引数を C 文字列 pro_name に変換します。 2. ErrorSaveContext を初期化します。 3. DirectInputFunctionCallSafe を呼び出し、regprocin 関数を使用して pro_name から OID への解決を試みます。 4. 解決に成功した場合はその Datum を返し、失敗した場合は PG_RETURN_NULL() を実行して NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に明示的な呼び出し元の記述はありません。
  • **呼び出し先**:
  • text_to_cstring
  • PG_GETARG_TEXT_PP
  • DirectInputFunctionCallSafe
  • regprocin

関連データ構造

  • ErrorSaveContext: エラー情報を保持し、例外を抑制するための構造体。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • 指定された関数名がシステムカタログ（pg_proc）に見つからない場合、DirectInputFunctionCallSafe が失敗し、本関数は NULL を返します。
  • 入力文字列が不正な形式である場合も同様に NULL を返します。

保守担当者向けメモ

  • 本関数は regprocin の安全な実行版です。ロジックの変更が必要な場合は、regprocin の名前解決ルール（parseNameAndArgTypes 等）が影響を受ける点に注意してください。
