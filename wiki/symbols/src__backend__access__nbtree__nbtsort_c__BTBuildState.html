BTBuildState

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtsort.c:204

目的

B-Treeインデックスのバルク構築プロセスにおける状態を管理する構造体です。インデックス構築中のタプル処理、一意性制約の確認、および並列構築時のリーダープロセスの制御情報を保持します。

シグネチャまたは形状

typedef struct BTBuildState
{
    bool        isunique;
    bool        nulls_not_distinct;
    bool        havedead;
    Relation    heap;
    BTSpool    *spool;
    BTSpool    *spool2;
    double      indtuples;
    BTLeader   *btleader;
} BTBuildState;

入力と出力

  • **入力**: インデックス構築開始時に btbuild 関数によって初期化されます。
  • **出力**: 構築完了後、indtuples に処理されたタプル総数が格納されます。

前提条件と不変条件

  • spool2 は一意インデックス（isunique が真）の場合のみ使用され、デッドタプルを分離して一意性チェックを回避するために利用されます。
  • 並列構築時、btleader はリーダープロセスのみが保持します。ワーカープロセスは個別の spool を持ちますが、BTBuildState 全体はリーダーのみが管理します。

アルゴリズムまたは振る舞い

1. **初期化**: btbuild にて構築対象のヒープリレーションと一意性フラグを設定。 2. **タプル収集**: _bt_spools_heapscan を通じてヒープをスキャンし、タプルを spool（または一意インデックスの場合は spool2）に蓄積します。 3. **並列処理**: 並列構築時は _bt_begin_parallel により btleader が初期化され、ワーカーとの同期が行われます。 4. **統計**: 処理されたタプル数は indtuples に累積されます。

呼び出し元と呼び出し先

  • **呼び出し元**: btbuild（インデックス構築のメインエントリーポイント）および並列構築関連のヘルパー関数（_bt_begin_parallel, _bt_parallel_heapscan 等）。
  • **呼び出し先**: BTSpool および BTLeader 構造体を介して、ソートおよび並列同期処理へ連携します。

関連データ構造

  • BTSpool: ソート中のタプルを保持する構造体。
  • BTLeader: 並列インデックス構築時のリーダープロセス用制御構造体。
  • Relation: 構築対象のヒープリレーション。

失敗モードと境界条件

  • 一意インデックス構築時に重複タプルが検出された場合、tuplesort 経由でエラーが発生する可能性があります。
  • 並列構築時、リーダーとワーカー間のメモリ共有や同期に失敗した場合、構築プロセスが中断される可能性があります。

保守担当者向けメモ

  • BTBuildState はリーダープロセスにのみ存在します。ワーカープロセスが自身の状態を保持する仕組みと混同しないよう注意してください。
  • spool2 の使用条件（一意インデックスかつデッドタプル存在時）は、インデックスの整合性維持に直結するため、変更時は _bt_spools_heapscan のロジックを併せて確認してください。
