ginPostingListDecodeAllSegmentsToTbm

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginpostinglist.c:357

目的

圧縮されたGINポスティングリストの全セグメントを復元し、その結果を TIDBitmap に追加します。

シグネチャまたは形状

int ginPostingListDecodeAllSegmentsToTbm(GinPostingList *ptr, int len, TIDBitmap *tbm)

入力と出力

  • **入力**:
  • ptr: 圧縮されたポスティングリストの先頭ポインタ。
  • len: ポスティングリストのバイト長。
  • tbm: 復元されたTIDを追加する対象の TIDBitmap 構造体。
  • **出力**:
  • 戻り値: 復元されたTIDの総数（int）。

前提条件と不変条件

  • ptr は有効な GinPostingList を指している必要があります。
  • tbm は初期化済みの有効な TIDBitmap である必要があります。
  • 内部でメモリ確保を行うため、呼び出し元はメモリコンテキストが適切であることを保証する必要があります。

アルゴリズムまたは振る舞い

1. ginPostingListDecodeAllSegments を呼び出し、圧縮されたセグメント群を走査して全TIDをメモリ上に復元します。 2. 復元されたTID配列を tbm_add_tuples を用いて TIDBitmap に追加します。 3. ginPostingListDecodeAllSegments が確保した一時的なTID配列を pfree で解放します。 4. 復元したTIDの個数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/gindatapage.c:192
  • **呼び出し先**:
  • ginPostingListDecodeAllSegments (内部関数)
  • tbm_add_tuples (外部関数)
  • pfree (外部関数)

関連データ構造

  • GinPostingList: 圧縮されたTIDリストを保持する構造体。
  • TIDBitmap: 復元されたTIDを格納するビットマップ構造体。

失敗モードと境界条件

  • メモリ不足時: ginPostingListDecodeAllSegments 内でのメモリ確保に失敗した場合、例外が発生します。
  • 入力データが不正な場合: ginPostingListDecodeAllSegments の挙動に依存し、不正なデータ構造に対しては未定義の動作やエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は ginPostingListDecodeAllSegments の結果を直接ビットマップへ流し込むためのラッパーです。
  • pfree によるメモリ解放が確実に行われるよう、例外発生時のクリーンアップには注意してください。
