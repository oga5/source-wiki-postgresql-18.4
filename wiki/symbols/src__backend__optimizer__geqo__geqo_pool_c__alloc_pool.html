alloc_pool

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_pool.c:41

目的

遺伝的アルゴリズム（GEQO）で使用される個体群管理用データ構造 Pool をメモリ上に確保し、初期化します。

シグネチャまたは形状

Pool *alloc_pool(PlannerInfo *root, int pool_size, int string_length)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体へのポインタ（未使用）。
  • pool_size: プールに格納する個体数。
  • string_length: 各染色体が保持する遺伝子（Gene）の長さ。
  • **出力**:
  • 確保および初期化された Pool 構造体へのポインタ。

前提条件と不変条件

  • pool_size および string_length は正の整数である必要があります。
  • PostgreSQLのメモリ管理関数 palloc が使用可能であること。

アルゴリズムまたは振る舞い

1. Pool 構造体自体のメモリを確保し、size および string_length を設定します。 2. 指定された pool_size 分の Chromosome 構造体の配列を確保し、Pool->data に割り当てます。 3. 各 Chromosome に対して、string_length + 1 分の Gene 配列を個別に確保します。 4. 確保した Pool 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_main.c:116
  • **呼び出し先**:
  • palloc (PostgreSQLメモリ管理関数)

関連データ構造

  • Pool: 個体群全体を管理する構造体。
  • Chromosome: 個体情報を保持する構造体。
  • Gene: 染色体を構成する要素。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、処理が中断されます。

保守担当者向けメモ

  • 本関数は Pool のメモリ確保のみを担当します。個体の解放には free_pool を使用してください。
  • PlannerInfo *root 引数は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
