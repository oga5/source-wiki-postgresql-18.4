btree_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/nbtdesc.c:23

目的

B-treeインデックスに関連するWAL（Write-Ahead Log）レコードの内容を解析し、人間が読めるテキスト形式に変換してバッファへ出力する。

シグネチャまたは形状

void btree_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 解析結果を格納する StringInfo バッファ。
  • record: 解析対象のWALレコード状態を保持する XLogReaderState へのポインタ。
  • **出力**: なし（buf を通じて文字列が書き出される）。

前提条件と不変条件

  • record は有効なWALレコードを指している必要がある。
  • XLogRecGetInfo で取得したレコードの info フィールドから XLR_INFO_MASK を除外した値が、B-treeの既知の操作タイプと一致する必要がある。

アルゴリズムまたは振る舞い

1. XLogRecGetData を呼び出し、WALレコードのペイロードを取得する。 2. XLogRecGetInfo から操作タイプ（info）を特定する。 3. switch 文により操作タイプに応じた構造体（xl_btree_insert, xl_btree_split 等）へキャストする。 4. キャストした構造体の各フィールドを appendStringInfo を用いて buf に追記する。 5. XLOG_BTREE_VACUUM または XLOG_BTREE_DELETE の場合、XLogRecHasBlockData でブロックデータを確認し、存在すれば delvacuum_desc を呼び出して詳細なオフセット情報を出力する。 6. XLOG_BTREE_META_CLEANUP の場合、XLogRecGetBlockData を介してメタデータ構造体を取得し出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で RM_BTREE_ID に対応する記述子として登録されており、pg_waldump 等のWAL解析ツールから間接的に呼び出される。
  • **呼び出し先**:
  • XLogRecGetData, XLogRecGetInfo, XLogRecHasBlockData, XLogRecGetBlockData: WALレコード操作用API。
  • appendStringInfo: 文字列構築用API。
  • delvacuum_desc: 削除・更新オフセット表示用の補助関数。
  • EpochFromFullTransactionId, XidFromFullTransactionId: トランザクションID変換用マクロ。

関連データ構造

  • StringInfo: 出力バッファ。
  • XLogReaderState: WALレコードの読み取り状態。
  • xl_btree_* 各種構造体: src/include/access/nbtxlog.h で定義されるB-tree WALレコードの物理レイアウト。

失敗モードと境界条件

  • 不明な info 型が渡された場合、何も出力せずに終了する。
  • XLOG_BTREE_VACUUM や XLOG_BTREE_DELETE においてブロックデータが存在しない場合、詳細なオフセット情報は出力されない。
  • delvacuum_desc 内でのポインタ演算は、nbtxlog.h の構造体定義と物理レイアウトが一致していることに依存する。

保守担当者向けメモ

  • 本関数はWALの「表示」のみを目的としており、インデックスの論理操作には関与しない。
  • delvacuum_desc を呼び出す際は、XLogRecGetBlockData の戻り値が正しくキャストされているか確認すること。
  • XLR_INFO_MASK を除外して判定する手法は他の記述子ルーチンと共通の慣習である。
