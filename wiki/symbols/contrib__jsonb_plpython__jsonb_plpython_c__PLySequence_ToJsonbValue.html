PLySequence_ToJsonbValue

定義

  • 種別: function
  • 定義箇所: contrib/jsonb_plpython/jsonb_plpython.c:321

目的

Python のシーケンス型オブジェクトを PostgreSQL の jsonb 配列構造へ変換します。

シグネチャまたは形状

static JsonbValue *PLySequence_ToJsonbValue(PyObject *obj, JsonbParseState **jsonb_state)

入力と出力

  • **入力**:
  • obj: 変換対象の Python シーケンスオブジェクト。
  • jsonb_state: JSONB 構築状態を保持する JsonbParseState へのポインタ。
  • **出力**:
  • 変換後の JsonbValue 構造体へのポインタ（配列の終了を示す WJB_END_ARRAY）。

前提条件と不変条件

  • obj は PySequence_Size が呼び出し可能なシーケンス型である必要があります。
  • jsonb_state は有効な JsonbParseState を指している必要があります。
  • 変換処理中に発生した Python 側の例外は、PostgreSQL のエラーハンドリング機構（PG_TRY/PG_CATCH）によって捕捉され、メモリリークを防ぐために Py_XDECREF が実行されます。

アルゴリズムまたは振る舞い

1. PySequence_Size でシーケンスの要素数を取得します。 2. pushJsonbValue を呼び出し、WJB_BEGIN_ARRAY を用いて JSONB 配列の開始を記録します。 3. ループ内で各要素を PySequence_GetItem で取得し、PLyObject_ToJsonbValue を再帰的に呼び出して JSONB 要素へ変換します。 4. 各要素の処理後、Py_XDECREF で参照カウントを減らします。 5. エラー発生時は PG_CATCH ブロックで現在の要素の参照カウントを解放し、例外を再送出（PG_RE_THROW）します。 6. 最後に WJB_END_ARRAY を pushJsonbValue で記録し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PLyObject_ToJsonbValue（ソースコード内の参照より推測）。
  • **呼び出し先**:
  • pushJsonbValue
  • PySequence_Size
  • PySequence_GetItem
  • PLyObject_ToJsonbValue
  • Py_XDECREF

関連データ構造

  • JsonbValue: JSONB の各要素を表現する構造体。
  • JsonbParseState: JSONB 構築中の状態を管理する構造体。
  • PyObject: Python のオブジェクト型。

失敗モードと境界条件

  • PySequence_GetItem が失敗した場合、PG_CATCH ブロックにより例外処理が行われ、処理が中断されます。
  • 空のシーケンスが渡された場合、WJB_BEGIN_ARRAY と WJB_END_ARRAY が連続して記録され、空の JSONB 配列が生成されます。

保守担当者向けメモ

  • 本関数は再帰的な変換器の一部です。PLyObject_ToJsonbValue との相互再帰的な関係にあるため、変更時はスタックオーバーフローや循環参照の可能性に注意してください。
  • volatile 修飾された value 変数は、PG_TRY ブロック内でのエラー発生時に安全に参照カウントを解放するために使用されています。
