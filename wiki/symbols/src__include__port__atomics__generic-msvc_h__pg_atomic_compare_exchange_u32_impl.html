pg_atomic_compare_exchange_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic-msvc.h:47

目的

MSVC環境において、32ビット符号なし整数に対するアトミックな比較交換（Compare-and-Swap）操作を提供します。

シグネチャまたは形状

static inline bool pg_atomic_compare_exchange_u32_impl(volatile pg_atomic_uint32 *ptr, uint32 *expected, uint32 newval)

入力と出力

  • ptr: 操作対象となる pg_atomic_uint32 構造体へのポインタ。
  • expected: 期待値が格納された変数へのポインタ。操作後、実際のメモリ値で更新されます。
  • newval: 比較が成功した場合に書き込まれる新しい値。
  • 戻り値: 比較が成功（*expected とメモリ値が一致）した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • ptr は有効な pg_atomic_uint32 構造体を指している必要があります。
  • 本関数は InterlockedCompareExchange を使用するため、メモリバリアが内包されます。

アルゴリズムまたは振る舞い

1. InterlockedCompareExchange を呼び出し、ptr->value の現在の値と *expected を比較します。 2. 一致する場合、ptr->value を newval で更新します。 3. 一致しない場合、メモリ値は変更されません。 4. 最後に、*expected に操作前の ptr->value の値を代入し、比較結果を返します。

呼び出し元と呼び出し先

  • 呼び出し先: Windows API の InterlockedCompareExchange。
  • 呼び出し元: src/include/port/atomics.h および src/include/port/atomics/generic.h 内の汎用アトミック関数から呼び出されます。

関連データ構造

  • pg_atomic_uint32: アトミック操作対象となる値を保持する構造体。

失敗モードと境界条件

  • *expected が現在のメモリ値と一致しない場合、操作は失敗し、*expected は現在のメモリ値で更新されます。

保守担当者向けメモ

  • 本関数は atomics.h を経由して使用されることを前提としています。
  • MSVC の Interlocked API はメモリバリアを内包しているため、追加のバリア処理は不要です。
