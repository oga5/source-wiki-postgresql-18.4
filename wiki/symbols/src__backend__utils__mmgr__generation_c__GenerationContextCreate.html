GenerationContextCreate

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/generation.c:159

目的

世代別（Generational）メモリコンテキストを新規作成し、初期ブロック（KeeperBlock）を確保してコンテキスト構造体を初期化する。

シグネチャまたは形状

MemoryContext GenerationContextCreate(MemoryContext parent,
                                      const char *name,
                                      Size minContextSize,
                                      Size initBlockSize,
                                      Size maxBlockSize)

入力と出力

  • **入力**:
  • parent: 親メモリコンテキスト。
  • name: コンテキスト名（デバッグ用）。
  • minContextSize: 最小初期ブロックサイズ。
  • initBlockSize: 標準的なブロックサイズ。
  • maxBlockSize: 最大ブロックサイズ。
  • **出力**:
  • 初期化された MemoryContext へのポインタ。

前提条件と不変条件

  • initBlockSize は MAXALIGN であり、かつ 1024 バイト以上であること。
  • maxBlockSize は MAXALIGN であり、initBlockSize 以上であること。
  • maxBlockSize は MEMORYCHUNK_MAX_BLOCKOFFSET 以下であること。
  • minContextSize が指定される場合、1024 バイト以上かつ maxBlockSize 以下であること。

アルゴリズムまたは振る舞い

1. パラメータの妥当性を検証する。 2. コンテキストヘッダーと最初のブロックを保持するのに十分なメモリを malloc で確保する。 3. 確保したメモリの先頭を GenerationContext として扱い、dlist を初期化する。 4. KeeperBlock を設定し、GenerationBlockInit を呼び出して初期ブロックを構成する。 5. 初期ブロックをブロックリストの先頭に追加し、現在の割り当てブロックとして設定する。 6. allocChunkLimit を計算し、チャンクサイズの制限を決定する。 7. MemoryContextCreate を呼び出し、型固有のメタデータを登録して完了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistvacuum.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/utils/sort/tuplestore.c
  • **呼び出し先**:
  • malloc
  • GenerationBlockInit
  • MemoryContextCreate
  • MemoryContextStats (エラー時)
  • ereport (エラー時)

関連データ構造

  • GenerationContext: コンテキストのメイン構造体。
  • GenerationBlock: メモリ管理の単位。
  • MemoryContext: 共通のコンテキストインターフェース。

失敗モードと境界条件

  • malloc が失敗した場合、ereport(ERROR, ...) を発行し、メモリ不足エラーとして処理を中断する。
  • パラメータが前提条件を満たさない場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は KeeperBlock をコンテキストヘッダーと同一のメモリ領域に確保するため、コンテキストの寿命とブロックの寿命が同期する。
  • allocChunkLimit の計算ロジックは、ブロック内に一定割合以上のチャンクを収めるためのヒューリスティックである。
