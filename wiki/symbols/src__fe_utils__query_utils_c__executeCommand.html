executeCommand

定義

  • 種別: function
  • 定義箇所: src/fe_utils/query_utils.c:46

目的

PostgreSQLのフロントエンドツールにおいて、結果を返さないSQLコマンドを簡便に実行し、エラー発生時にはログを出力してプログラムを終了させるためのユーティリティ関数です。

シグネチャまたは形状

void executeCommand(PGconn *conn, const char *query, bool echo);

入力と出力

  • **入力**:
  • conn: libpq のデータベース接続オブジェクト (PGconn *)。
  • query: 実行するSQLコマンド文字列 (const char *)。
  • echo: true の場合、実行前にクエリを標準出力へエコーするフラグ (bool)。
  • **出力**: なし（成功時は正常終了、失敗時はプログラムを終了）。

前提条件と不変条件

  • conn は有効な PGconn オブジェクトである必要があります。
  • 実行されるクエリは、結果セットを返さない（PGRES_COMMAND_OK を期待する）コマンドである必要があります。

アルゴリズムまたは振る舞い

1. echo が true の場合、query を標準出力に表示します。 2. PQexec を呼び出し、データベースに対してクエリを実行します。 3. 実行結果が NULL であるか、ステータスが PGRES_COMMAND_OK でない場合：

  • pg_log_error を使用してエラーメッセージを出力します。
  • pg_log_error_detail を使用して実行したクエリ内容を出力します。
  • PQfinish(conn) を呼び出して接続を閉じます。
  • exit(1) を呼び出し、プログラムを異常終了させます。

    • 4. 成功した場合は PQclear(res) を呼び出し、結果オブジェクトを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_dump/pg_dumpall.c
  • src/bin/scripts/common.c
  • src/bin/scripts/vacuumdb.c
  • src/fe_utils/parallel_slot.c
  • **呼び出し先**:
  • libpq ライブラリの PQexec, PQresultStatus, PQerrorMessage, PQclear, PQfinish。
  • common/logging.h の pg_log_error, pg_log_error_detail。

関連データ構造

  • PGconn: libpq の接続管理構造体。
  • PGresult: libpq のクエリ実行結果構造体。

失敗モードと境界条件

  • PQexec が失敗した場合、またはクエリの実行結果が PGRES_COMMAND_OK 以外の場合、本関数は exit(1) によりプロセスを終了させます。
  • 接続が切断されている場合や、SQL構文エラーが発生した場合は、エラーログを出力して終了します。

保守担当者向けメモ

  • 本関数はライブラリ的な柔軟性よりも、CLIツールにおける「失敗即終了」という一貫した挙動を優先しています。
  • 複雑なトランザクション制御や、エラーを呼び出し元でハンドリングする必要がある場合は、本関数を使用せず直接 libpq を使用してください。
