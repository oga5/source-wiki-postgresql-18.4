DeadLockCheckRecurse

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/deadlock.c:311

目的

ロック待ちグラフの探索を通じて、デッドロックを解消可能なロック待ち行列の順序（有効な構成）を再帰的に探索します。

シグネチャまたは形状

static bool DeadLockCheckRecurse(PGPROC *proc)

入力と出力

  • **入力**: proc (対象となるプロセス構造体へのポインタ)
  • **出力**: bool 型。デッドロックを解消できる有効な構成が見つかった場合は false を返し、解決不能な場合は true を返します。

前提条件と不変条件

  • 本関数は、ロックマネージャが管理するロック待ちグラフの探索中に呼び出されます。
  • nCurConstraints および maxCurConstraints を用いて、現在の制約状態を追跡します。
  • possibleConstraints 配列に十分な空きがある場合、エッジリストをキャッシュして再帰処理を行います。

アルゴリズムまたは振る舞い

1. TestConfiguration(proc) を呼び出し、現在の構成における「ソフトエッジ（解消可能な競合）」の数を取得します。 2. 戻り値が -1 の場合はハードデッドロック（解決不能）と判定し true を返します。 3. 戻り値が 0 の場合は有効な構成が見つかったとみなし false を返します。 4. nCurConstraints が上限に達している場合は探索を打ち切り true を返します。 5. 利用可能なソフトエッジを一つずつ curConstraints に追加し、自身を再帰呼び出しします。 6. 再帰呼び出しが false を返せば、有効な解決策が見つかったものとして伝播させます。 7. すべての試行で解決策が見つからなかった場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: DeadLockCheck および DeadLockCheckRecurse 自身（再帰呼び出し）。
  • **呼び出し先**: TestConfiguration。

関連データ構造

  • PGPROC: ロック待ち状態にあるプロセス。
  • curConstraints: 現在の探索パスで適用されている制約のリスト。
  • possibleConstraints: 探索可能なソフトエッジの候補リスト。

失敗モードと境界条件

  • **ハードデッドロック**: TestConfiguration が -1 を返した場合、解決不能として即座に終了します。
  • **制約上限**: nCurConstraints が maxCurConstraints を超える場合、探索を停止します。
  • **メモリ不足**: possibleConstraints の容量が不足する場合、エッジリストのキャッシュを行わず、都度 TestConfiguration を再実行して整合性を検証します。この際、結果が不一致であれば FATAL エラーを発生させます。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、スタック消費量に注意が必要です。
  • savedList が false の場合、再帰の各ステップで TestConfiguration が再実行されるため、計算コストが増大する可能性があります。
