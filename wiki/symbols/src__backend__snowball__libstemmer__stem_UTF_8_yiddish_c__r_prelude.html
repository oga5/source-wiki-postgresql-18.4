r_prelude

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_yiddish.c:502

目的

イディッシュ語のステミング処理における前処理として、入力文字列の正規化を行う。

シグネチャまたは形状

static int r_prelude(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体（処理対象の文字列およびステミング環境を保持）。
  • 出力: 成功時に 1、失敗（メモリ不足や処理エラー）時に負の整数を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • find_among および slice_from_s 等の Snowball ランタイム関数が正常に動作する環境であること。

アルゴリズムまたは振る舞い

1. **文字置換**: find_among を使用して定義済みのパターン（a_0）を検索し、一致した場合は対応する文字列に置換する。一致しない場合は skip_utf8 で次の文字へ進む。 2. **ニクダー（母音記号）の削除**: in_grouping_U を用いて g_niked グループに含まれる文字（Unicode範囲 1456-1474）を特定し、該当する場合は slice_del で削除する。 3. 上記の処理を文字列全体に対して繰り返し適用する。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内には明示的な呼び出し元は記載されていない。
  • 呼び出し先: find_among, eq_s, slice_from_s, skip_utf8, in_grouping_U, slice_del（いずれも Snowball ランタイム関数）。

関連データ構造

  • struct SN_env: ステミング環境を管理する構造体。
  • a_0: 文字置換ルールを定義した among 構造体の配列。
  • g_niked: 削除対象の文字グループを定義した構造体。

失敗モードと境界条件

  • slice_from_s や slice_del が負の値を返した場合、その値をそのまま呼び出し元へ伝播させる。
  • 文字列の終端に達した際の skip_utf8 の挙動によりループが終了する。

  • g_niked の範囲（1456-1474）はイディッシュ語のニクダーに対応している。
