ltxtquery

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltree.h:153

目的

ポーランド記法を用いた複雑な全文検索クエリを格納するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    int32       vl_len_;        /* varlena header */
    int32       size;           /* クエリ内のアイテム数 */
    char        data[FLEXIBLE_ARRAY_MEMBER]; /* 可変長データ領域 */
} ltxtquery;

入力と出力

  • 入力: PostgreSQL の Datum から PG_DETOAST_DATUM 等を介して変換されます。
  • 出力: 検索演算子や GiST インデックスの照合ロジックにおいて、クエリの評価対象として利用されます。

前提条件と不変条件

  • vl_len_ フィールドは PostgreSQL の可変長データ（varlena）ヘッダーであり、直接操作してはなりません。
  • data 領域には ITEM 構造体の配列が格納されます。
  • 構造体のサイズ計算には sizeof を直接使用せず、LTREE_HDRSIZE 等の専用マクロを使用する必要があります。

アルゴリズムまたは振る舞い

  • ポーランド記法（前置記法）で記述されたクエリを保持し、ltree 型のデータに対するマッチング処理をサポートします。
  • GETOPERAND マクロ等を用いて、構造体内の ITEM 配列の直後に続くオペランド文字列へアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/ltree/_ltree_gist.c: gist_qtxt 関数内でインデックス検索に使用。
  • contrib/ltree/_ltree_op.c: PG_GETARG_LTXTQUERY_P を介して SQL 関数から取得。
  • contrib/ltree/ltxtquery_io.c: 入出力処理時に使用。
  • 呼び出し先:
  • PostgreSQL のメモリ管理およびデータ型変換ユーティリティ（PG_DETOAST_DATUM 等）。

関連データ構造

  • ITEM: ltxtquery 内の各要素を構成する構造体。
  • ltree: 検索対象となる階層パス構造体。

失敗モードと境界条件

  • クエリサイズが過大な場合、ltxtquery_io.c にてエラーが発生する可能性があります。
  • 不正なメモリレイアウトやアライメント違反は、FLEXIBLE_ARRAY_MEMBER の扱いを誤った場合に発生します。

保守担当者向けメモ

  • 本構造体はディスクフォーマットとの互換性を維持する必要があるため、フィールドの追加や変更には注意が必要です。
  • ltxtquery を扱う際は、必ず DatumGetLtxtqueryP などのマクロ経由でアクセスし、TOAST されたデータの展開を確実に行ってください。
