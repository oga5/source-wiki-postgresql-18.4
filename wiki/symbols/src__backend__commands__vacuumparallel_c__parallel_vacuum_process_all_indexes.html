parallel_vacuum_process_all_indexes

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuumparallel.c:610

目的

VACUUM 処理におけるインデックスのバルク削除またはクリーンアップフェーズにおいて、並列実行環境を構築し、ワーカープロセスを起動してインデックス処理を分散実行する。

シグネチャまたは形状

static void parallel_vacuum_process_all_indexes(ParallelVacuumState *pvs, int num_index_scans, bool vacuum)

入力と出力

  • pvs: 並列 VACUUM の状態を保持する ParallelVacuumState 構造体へのポインタ。
  • num_index_scans: 現在のインデックススキャン回数。
  • vacuum: true の場合はバルク削除フェーズ、false の場合はクリーンアップフェーズとして動作する。
  • 出力: なし（副作用としてインデックスの処理および統計情報の更新を行う）。

前提条件と不変条件

  • 本関数はリーダープロセスからのみ呼び出されること（!IsParallelWorker() をアサート）。
  • 各インデックスのステータスは、処理開始時に PARALLEL_INDVAC_STATUS_INITIAL であること。
  • 処理終了時には、すべてのインデックスが PARALLEL_INDVAC_STATUS_COMPLETED に遷移していること。

アルゴリズムまたは振る舞い

1. **初期化**: vacuum フラグに基づき、実行するフェーズ（バルク削除またはクリーンアップ）を決定し、必要なワーカー数を計算する。 2. **状態設定**: 全インデックスのステータスを更新し、各インデックスが並列処理可能か（parallel_vacuum_index_is_parallel_safe）を判定してマークする。 3. **並列実行準備**: 並列インデックス処理カウンタをリセットし、必要に応じて ReinitializeParallelDSM および ReinitializeParallelWorkers を呼び出してワーカーを起動する。 4. **コスト管理**: 共有コストベースの遅延制御（VacuumSharedCostBalance）を有効化する。 5. **インデックス処理**:

  • リーダーのみが処理可能なインデックスを parallel_vacuum_process_unsafe_indexes で処理。
  • 並列安全なインデックスを parallel_vacuum_process_safe_indexes で処理（リーダーも参加）。

    • 6. **同期**: ワーカーの終了を待ち、統計情報（バッファ/WAL使用量）を集約する。 7. **終了処理**: インデックスのステータスを初期状態に戻し、共有コストバランスを無効化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/vacuumparallel.c 内の parallel_vacuum_bulkdel_all_indexes および parallel_vacuum_cleanup_all_indexes。
  • 呼び出し先: parallel_vacuum_index_is_parallel_safe, ReinitializeParallelDSM, ReinitializeParallelWorkers, LaunchParallelWorkers, parallel_vacuum_process_unsafe_indexes, parallel_vacuum_process_safe_indexes, WaitForParallelWorkersToFinish, InstrAccumParallelQuery。

関連データ構造

  • ParallelVacuumState: 並列 VACUUM の全体状態。
  • PVIndStats: 各インデックスの処理状態および並列実行可否。
  • PVShared: 共有メモリ上の制御情報（インデックスカウンタ、コストバランス等）。

失敗モードと境界条件

  • インデックス処理が完了していない場合、elog(ERROR) を発生させる。
  • ワーカー内でエラーが発生した場合、parallel_vacuum_error_callback が呼び出される（本関数自体はエラーハンドリングの枠組みの中で実行される）。
  • ワーカー数が 0 の場合、リーダーのみで全インデックスを処理する。

保守担当者向けメモ

  • 本関数は vacuum フェーズと cleanup フェーズの両方で再利用されるため、フェーズごとのワーカー数計算ロジックに注意すること。
  • 共有コストバランスの有効化・無効化のタイミングは、ワーカーの起動および終了と密接に関連しているため、変更時は同期処理を確認すること。
