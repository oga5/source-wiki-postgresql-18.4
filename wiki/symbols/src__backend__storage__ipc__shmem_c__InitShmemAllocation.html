InitShmemAllocation

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:114

目的

共有メモリ領域の割り当て管理機能を初期化し、Postmaster起動時に共有メモリのベースアドレスとセグメントヘッダーをセットアップします。

シグネチャまたは形状

void InitShmemAllocation(void)

入力と出力

  • 入力: なし（グローバル変数 ShmemSegHdr を参照）
  • 出力: なし（副作用として共有メモリ管理用のスピンロックおよびヘッダー情報を初期化）

前提条件と不変条件

  • ShmemSegHdr が NULL でないこと。
  • 本関数は共有メモリの初期化シーケンスの初期段階で呼び出されること。

アルゴリズムまたは振る舞い

1. ShmemAllocUnlocked を使用して、共有メモリ管理用のスピンロック（ShmemLock）を確保し、SpinLockInit で初期化します。 2. 以降の ShmemAlloc による割り当てがキャッシュライン境界で開始されるよう、現在の共有メモリの空き領域オフセット（freeoffset）を CACHELINEALIGN で調整します。 3. ShmemIndex およびヘッダー内のインデックスポインタを NULL に設定します（LWLocksの初期化が完了するまでインデックスは使用不可であるため）。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:237
  • 呼び出し先: ShmemAllocUnlocked（内部関数）, SpinLockInit（マクロ）

関連データ構造

  • PGShmemHeader: 共有メモリセグメントのヘッダー構造体。
  • ShmemLock: 共有メモリ割り当てを保護するスピンロック。
  • ShmemIndex: 共有メモリ上のデータ構造を管理するハッシュテーブル（本関数実行時点では未初期化）。

失敗モードと境界条件

  • ShmemSegHdr が NULL の場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は ShmemAlloc が利用可能になる前の準備を行うため、割り当てには必ず ShmemAllocUnlocked を使用する必要があります。
  • 共有メモリのインデックス（ShmemIndex）の初期化は、LWLocksの準備が整った後の別フェーズで行われます。
