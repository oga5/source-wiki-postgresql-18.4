Size

定義

  • 種別: struct
  • 定義箇所: src/include/replication/output_plugin.h:85

目的

論理デコーディングにおいて、メッセージやデータバッファのサイズを表現するために使用される型です。

シグネチャまたは形状

struct Size;

※ソースコード上では typedef 等の別名として利用されることが一般的ですが、本ヘッダーでは struct として定義されています。

入力と出力

  • **入力**: 主に LogicalDecodeMessageCB などのコールバック関数において、メッセージのバイト長を示す引数として渡されます。
  • **出力**: 該当しません。

前提条件と不変条件

  • 値は非負の整数である必要があります。
  • メモリバッファのサイズを示す場合、対象となるバッファの実際のバイト数と一致している必要があります。

アルゴリズムまたは振る舞い

本シンボル自体はデータサイズを保持するための型定義であり、特定のアルゴリズムは持ちません。論理デコーディングの文脈では、プラグインが受信したメッセージの長さを判定し、メモリコピーやデータ処理の範囲を決定するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: LogicalDecodeMessageCB の定義において、引数として参照されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • LogicalDecodingContext: 論理デコーディングのコンテキスト構造体。
  • ReorderBufferTXN: トランザクション情報を保持する構造体。
  • LogicalDecodeMessageCB: 本型を利用するコールバック関数の型定義。

失敗モードと境界条件

  • **境界条件**: システムが許容する最大メモリサイズを超える値が指定された場合、後続のメモリ確保やデータ処理でエラーが発生する可能性があります。
  • **失敗モード**: 負の値が渡された場合、または実際のデータ長と不一致がある場合、バッファオーバーフローや不正なメモリ参照を引き起こすリスクがあります。

保守担当者向けメモ

  • 本シンボルは src/include/replication/output_plugin.h と replication/reorderbuffer.h の両方で定義されています。環境やコンパイル設定によっては、標準ライブラリの size_t と混同しないよう注意してください。
  • ツールによる検索結果には、contrib モジュール内の同名シンボルが含まれていますが、これらは本シンボルとは直接関係のない別個の定義である可能性が高いです。
