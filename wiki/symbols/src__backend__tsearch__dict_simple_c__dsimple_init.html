dsimple_init

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_simple.c:30

目的

PostgreSQL の「Simple」全文検索辞書の設定を初期化し、ユーザー指定のパラメータ（ストップワードリストおよび受理フラグ）を解析して DictSimple 構造体に格納する。

シグネチャまたは形状

Datum dsimple_init(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して List *dictoptions を受け取る。これは CREATE TEXT SEARCH DICTIONARY で指定されたパラメータのリスト。
  • **出力**: DictSimple * を Datum 型として返す。

前提条件と不変条件

  • DictSimple 構造体は palloc0 によりゼロ初期化される。
  • accept パラメータのデフォルト値は true である。
  • stopwords パラメータが指定された場合、readstoplist 関数によりメモリ上にロードされる。

アルゴリズムまたは振る舞い

1. DictSimple 構造体をメモリ確保し、accept を true に初期化する。 2. dictoptions リストを走査し、各 DefElem を検証する。

  • stopwords: readstoplist を呼び出し、ストップワードを読み込む。重複指定はエラー。
  • accept: defGetBoolean を使用してフラグを更新する。重複指定はエラー。
  • その他: 未知のパラメータとしてエラーを報告する。

    • 3. 初期化済みの DictSimple 構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の辞書管理サブシステム（詳細はソースコード内で明示的な呼び出し元は特定されていない）。
  • **呼び出し先**: readstoplist（ストップワード読み込み）、defGetString / defGetBoolean（パラメータ解析）、ereport（エラー報告）。

関連データ構造

  • DictSimple: 辞書の設定を保持する構造体。
  • DefElem: 辞書オプションの定義要素。

失敗モードと境界条件

  • stopwords または accept パラメータが複数回指定された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了する。
  • 未定義のパラメータが指定された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了する。

保守担当者向けメモ

  • 本関数は palloc0 を使用しているため、メモリリークの懸念はないが、辞書設定のライフサイクルは PostgreSQL のメモリコンテキストに依存する。
  • str_tolower を readstoplist に渡しているため、ストップワードの照合は小文字化された状態で行われる前提となっている。
