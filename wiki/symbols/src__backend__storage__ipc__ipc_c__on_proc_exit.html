on_proc_exit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipc.c:315

目的

プロセス終了時に実行されるコールバック関数を登録し、システムが安全に終了処理を行えるように管理する。

シグネチャまたは形状

void on_proc_exit(pg_on_exit_callback function, Datum arg)

入力と出力

  • function: 終了時に呼び出される関数ポインタ。
  • arg: コールバック関数に渡される Datum 型の引数。
  • 出力: なし。

前提条件と不変条件

  • 登録可能なコールバック数は MAX_ON_EXITS によって制限されている。
  • コールバックは登録された順序とは逆の順序（LIFO）で実行される。

アルゴリズムまたは振る舞い

1. 現在の登録数が MAX_ON_EXITS に達していないか確認し、超過している場合は FATAL エラーを発生させる。 2. 指定された function と arg を on_proc_exit_list の現在のインデックス位置に格納する。 3. on_proc_exit_index をインクリメントする。 4. atexit_callback_setup フラグを確認し、未設定であれば atexit(atexit_callback) を呼び出してプロセス終了時のバックストップ処理を登録し、フラグを true に更新する。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • contrib/sepgsql/uavc.c
  • src/backend/jit/llvm/llvmjit.c
  • src/backend/libpq/pqcomm.c
  • src/backend/postmaster/postmaster.c
  • 呼び出し先:
  • atexit (標準ライブラリ関数)
  • atexit_callback (内部関数)

関連データ構造

  • on_proc_exit_list: コールバック関数と引数を保持する配列。
  • on_proc_exit_index: 次に登録する配列のインデックス。
  • atexit_callback_setup: atexit が既に登録済みかを示すブール値。

失敗モードと境界条件

  • on_proc_exit_index >= MAX_ON_EXITS: 登録上限を超えた場合、FATAL エラーとなりプロセスが終了する。

保守担当者向けメモ

  • 本関数は proc_exit() の実行シーケンスの一部として機能する。
  • コールバック内で ereport(ERROR) を発生させた場合、そのコールバックは再実行されず、終了処理が継続される設計となっている。
  • on_exit_reset() を呼び出すことで、フォーク後のプロセスにおいて不要なコールバック登録をリセットする必要がある点に注意すること。
