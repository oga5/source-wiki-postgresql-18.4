GISTTYPE

定義

  • 種別: struct
  • 定義箇所: contrib/hstore/hstore_gist.c:48

目的

hstore 型の GiST インデックスにおいて、ハッシュベースのシグネチャ（ビットベクトル）を保持するための可変長データ構造です。

シグネチャまたは形状

typedef struct
{
	int32		vl_len_;		/* varlena header */
	int32		flag;
	char		data[FLEXIBLE_ARRAY_MEMBER];
} GISTTYPE;

入力と出力

  • 本構造体は、ghstore_compress 等の関数を通じて hstore データから生成されます。
  • data メンバには、キーと値を CRC-32 でハッシュ化したビットベクトルが格納されます。

前提条件と不変条件

  • vl_len_ は PostgreSQL の可変長データ（varlena）ヘッダーであり、直接操作してはなりません。
  • flag メンバは、インデックスの状態（例: ALLISTRUE フラグ）を保持するために使用されます。

アルゴリズムまたは振る舞い

  • hstore の各キー・値を CRC-32 でハッシュ化し、data 領域のビットベクトルに SETBIT することでインデックス用のシグネチャを構築します。
  • 検索時には、この構造体内のビットベクトルを参照して包含関係や存在確認のフィルタリングを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: ghstore_compress, ghstore_alloc, ghstore_consistent, hemdist, unionkey 等の関数が本構造体を生成または参照します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出す側ではありません。

関連データ構造

  • BITVECP: GISTTYPE 内のビットベクトルを操作するためのポインタ型。
  • GistHstoreOptions: インデックスのオプション（siglen）を管理する構造体。

失敗モードと境界条件

  • 本構造体を用いた検索は「シグネチャによるフィルタリング」であるため、インデックスでヒットした場合でも、最終的な一致確認にはヒープ上の実データに対する recheck が必須となります。
  • ハッシュ衝突が発生する可能性があるため、シグネチャの一致はあくまで「可能性のあるもの」の絞り込みとして機能します。

保守担当者向けメモ

  • siglen オプションによりシグネチャ長が可変であるため、メモリ確保時には ghstore_alloc を経由して適切なサイズを割り当ててください。
  • ISALLTRUE マクロのように、flag メンバを直接参照してインデックスの状態を判定するコードが存在します。
