pgaio_io_update_state

定義

  • 種別: static inline function
  • 定義箇所: src/backend/storage/aio/aio.c:385

目的

非同期I/Oハンドル（PgAioHandle）の状態遷移を管理し、メモリバリアを適用して状態変更の可視性を保証します。

シグネチャまたは形状

static inline void pgaio_io_update_state(PgAioHandle *ioh, PgAioHandleState new_state)

入力と出力

  • ioh: 更新対象の PgAioHandle 構造体へのポインタ。
  • new_state: 設定する新しい状態（PgAioHandleState 型）。
  • 出力: なし（副作用として ioh->state を更新）。

前提条件と不変条件

  • 呼び出し元で HOLD_INTERRUPTS() が実行されていること（Assert(!INTERRUPTS_CAN_BE_PROCESSED()) により検証）。
  • 割り込み処理中にI/O完了待ちが発生し、状態の不整合が生じることを防ぐため、割り込みが禁止されている必要があります。

アルゴリズムまたは振る舞い

1. 現在の割り込み状態を検証し、許可されていればアサーションエラーを発生させます。 2. pgaio_debug_io を介して、デバッグレベル DEBUG5 で状態遷移を記録します。 3. pg_write_barrier() を実行し、状態更新に伴うメモリ上の変更が、新しい状態値の可視化よりも先に完了することを保証します。 4. ioh->state に new_state を代入します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/aio/aio.c 内の各状態遷移箇所（PGAIO_HS_HANDED_OUT, PGAIO_HS_DEFINED, PGAIO_HS_STAGED, PGAIO_HS_SUBMITTED, PGAIO_HS_COMPLETED_IO, PGAIO_HS_COMPLETED_SHARED, PGAIO_HS_COMPLETED_LOCAL, PGAIO_HS_IDLE への遷移時）。
  • 呼び出し先: pgaio_io_state_get_name（デバッグログ出力用）。

関連データ構造

  • PgAioHandle: I/O操作の状態を保持する主要な構造体。
  • PgAioHandleState: I/Oハンドルの状態を定義する列挙型。

失敗モードと境界条件

  • 割り込みが許可された状態で呼び出された場合、アサーションが有効なビルドではプロセスが停止します。

保守担当者向けメモ

  • AIOサブシステムにおけるすべての状態更新は、この関数を経由することが強制されています（src/include/storage/aio_internal.h に記載）。
  • 状態遷移の整合性を維持するため、直接 ioh->state を書き換えることは避けてください。
