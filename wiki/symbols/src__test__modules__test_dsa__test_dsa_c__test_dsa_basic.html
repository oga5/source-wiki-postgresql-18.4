test_dsa_basic

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_dsa/test_dsa.c:26

目的

PostgreSQLのDSA（Dynamic Shared Memory Area）機能の基本的なCRUD操作（割り当て、書き込み、読み取り、解放）を検証するテスト関数です。

シグネチャまたは形状

Datum test_dsa_basic(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（SQL関数として呼び出し可能）
  • 出力: PG_RETURN_VOID()（戻り値なし）

前提条件と不変条件

  • LWLockNewTrancheId() によって新しいロックトランチが確保される必要があります。
  • dsa_create() で生成された dsa_area が有効である必要があります。

アルゴリズムまたは振る舞い

1. LWLockNewTrancheId() でトランチIDを取得し、LWLockRegisterTranche() で登録します。 2. dsa_create() でDSA領域を生成します。 3. 100個のメモリブロックを dsa_allocate() で確保し、各ブロックに文字列（"foobar0"〜"foobar99"）を書き込みます。 4. 書き込んだ全ブロックの内容を読み出し、期待値と一致するか検証します。一致しない場合は elog(ERROR) を発生させます。 5. dsa_free() で全ブロックを解放し、最後に dsa_detach() でDSA領域を破棄します。

呼び出し元と呼び出し先

  • 呼び出し元: SQLインターフェース経由で呼び出されることを想定（PG_FUNCTION_INFO_V1 マクロによる登録あり）。
  • 呼び出し先: LWLockNewTrancheId, LWLockRegisterTranche, dsa_create, dsa_allocate, dsa_get_address, dsa_free, dsa_detach 等のDSA/LWLock API。

関連データ構造

  • dsa_area: DSA領域を管理する構造体。
  • dsa_pointer: DSA領域内のメモリ位置を指すポインタ型。

失敗モードと境界条件

  • 検証ループ内で期待値と異なるデータが読み出された場合、elog(ERROR) により処理が中断されます。
  • LWLockNewTrancheId() で確保されたトランチは明示的に解放されないため、テスト実行プロセス終了までメモリリークが発生します。

保守担当者向けメモ

  • 本関数はDSAの基本的なライフサイクルを網羅する回帰テストとして機能します。
  • 内部実装の変更に伴うメモリ管理の整合性チェックに利用してください。
