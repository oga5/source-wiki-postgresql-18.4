ExecEndSort

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSort.c:300

目的

ソート処理ノード（SortState）の終了処理を行い、ソートに関連するリソースを解放する。

シグネチャまたは形状

void ExecEndSort(SortState *node)

入力と出力

  • **入力**: SortState *node（終了対象のソートノード状態）
  • **出力**: なし（void）

前提条件と不変条件

  • node は有効な SortState 構造体へのポインタであること。
  • 実行終了後、node->tuplesortstate は NULL に設定される。

アルゴリズムまたは振る舞い

1. デバッグ用ログ（SO1_printf）を出力する。 2. node->tuplesortstate が NULL でない場合、tuplesort_end を呼び出してソート用のメモリおよび一時ファイルリソースを解放し、ポインタを NULL にリセットする。 3. outerPlanState(node) を引数として ExecEndNode を呼び出し、下位ノードの終了処理を実行する。 4. 終了を示すデバッグ用ログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:710（ExecEndNode 経由での呼び出しが確認されている）。
  • **呼び出し先**:
  • tuplesort_end（src/backend/utils/sort/tuplesort.c）
  • ExecEndNode（src/backend/executor/execProcnode.c）

関連データ構造

  • SortState: ソートノードの実行状態を保持する構造体。
  • Tuplesortstate: tuplesort.c が管理するソートアルゴリズムの内部状態。

失敗モードと境界条件

  • node->tuplesortstate が既に NULL の場合、tuplesort_end は呼び出されず、安全にスキップされる。

保守担当者向けメモ

  • 本関数は ExecEndNode の一部として呼び出されるため、ノードのクリーンアップ順序（下位ノードの解放など）を維持する必要がある。
  • SO1_printf はデバッグビルド時のみ有効なマクロである点に留意すること。
