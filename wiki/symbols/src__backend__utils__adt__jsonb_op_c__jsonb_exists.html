jsonb_exists

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_op.c:20

目的

jsonb 型のデータにおいて、指定されたキーまたは文字列要素がトップレベルに存在するかを判定します。SQL演算子 ? の実装として機能します。

シグネチャまたは形状

Datum jsonb_exists(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_JSONB_P(0): 検索対象の Jsonb オブジェクト。
  • PG_GETARG_TEXT_PP(1): 検索するキーまたは文字列要素。
  • **出力**:
  • bool: 存在する場合は true、存在しない場合は false を返します。

前提条件と不変条件

  • 検索対象は Jsonb のトップレベルのみです。再帰的な探索は行われません。
  • 文字列型のキー（オブジェクト）または文字列型の要素（配列）のみがマッチ対象です。
  • 非文字列のスカラー要素（数値、真偽値、nullなど）は、たとえ配列内に存在していてもマッチしません。

アルゴリズムまたは振る舞い

1. 入力された text 型のキーを JsonbValue（型: jbvString）に変換します。 2. findJsonbValueFromContainer 関数を呼び出し、Jsonb コンテナのルートから指定された値を探します。 3. 検索対象として JB_FOBJECT（オブジェクト）および JB_FARRAY（配列）のフラグを指定します。 4. 検索結果が NULL でなければ true を、そうでなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）を介してSQLから呼び出されます。ソースコード内での直接的な呼び出し元は確認されていません。
  • **呼び出し先**:
  • findJsonbValueFromContainer（src/backend/utils/adt/jsonb_util.c 相当の機能）

関連データ構造

  • Jsonb: 検索対象となる jsonb 型の内部構造体。
  • JsonbValue: 検索キーを保持するための一時的な構造体。

失敗モードと境界条件

  • 入力が空の jsonb である場合、常に false を返します。
  • 検索対象がオブジェクトまたは配列でない場合、マッチしません。
  • 検索対象が非文字列スカラーである場合、明示的にマッチ対象外となります。

保守担当者向けメモ

  • 本関数は再帰的な探索を行わない「トップレベル専用」の存在確認関数です。深い階層の探索が必要な場合は別の実装を検討してください。
  • findJsonbValueFromContainer の引数フラグにより、オブジェクトのキーと配列の要素の両方を検索対象としています。
