PgAioTargetInfo

定義

  • 種別: struct
  • 定義箇所: src/include/storage/aio.h:158

目的

非同期 I/O (AIO) 操作のターゲット（ストレージマネージャ等）に関するメタデータおよび操作インターフェースを定義し、ログ出力やプロセス間での I/O 再開を支援する。

シグネチャまたは形状

struct PgAioTargetInfo
{
    void        (*reopen) (PgAioHandle *ioh);
    char       *(*describe_identity) (const PgAioTargetData *sd);
    const char *name;
};

入力と出力

  • reopen: PgAioHandle へのポインタを入力とし、戻り値はない。
  • describe_identity: PgAioTargetData へのポインタを入力とし、ターゲットを識別する文字列（char *）を返す。
  • name: ターゲット名を保持する定数文字列。

前提条件と不変条件

  • reopen 関数は、ワーカープロセス等で I/O 用のファイル記述子を再オープンする必要がある場合に呼び出される。
  • name および各関数ポインタは、ターゲットのライフサイクルを通じて有効である必要がある。

アルゴリズムまたは振る舞い

  • ターゲット固有の I/O ロジックを抽象化する。
  • reopen を通じて、プロセス境界を越えた I/O ハンドルの再構築をサポートする。
  • describe_identity を通じて、ログやビューで使用するためのターゲット詳細情報を提供する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/aio/aio_target.c および src/backend/storage/smgr/smgr.c にて、各ターゲットの実装が本構造体を初期化している。
  • 呼び出し先: 具体的な関数ポインタの実装に依存する（ソース抜粋からは詳細な呼び出し先は特定不可）。

関連データ構造

  • PgAioHandle: reopen 関数の引数として使用される。
  • PgAioTargetData: describe_identity 関数の引数として使用される。

失敗モードと境界条件

  • reopen が失敗した場合の挙動については、本ソース抜粋には記載がない。
  • 各関数ポインタが NULL である場合の動作については、本ソース抜粋には記載がない。

保守担当者向けメモ

  • 本構造体は src/include/storage/aio_types.h:24 にも定義が存在する。
  • aio_smgr_target_info が src/include/storage/smgr.h で外部公開されており、ストレージマネージャ層との統合点となっている。
