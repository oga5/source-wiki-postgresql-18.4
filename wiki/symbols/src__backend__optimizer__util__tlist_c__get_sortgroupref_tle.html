get_sortgroupref_tle

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/tlist.c:353

目的

ターゲットリスト（targetList）の中から、指定されたソートまたはグループ化の参照インデックス（sortref）と一致する TargetEntry を検索して返します。

シグネチャまたは形状

TargetEntry *get_sortgroupref_tle(Index sortref, List *targetList)

入力と出力

  • **入力**:
  • sortref (Index): 検索対象の ressortgroupref 値。
  • targetList (List *): 検索対象となる TargetEntry のリスト。
  • **出力**:
  • 成功時: 一致する TargetEntry 構造体へのポインタ。
  • 失敗時: 戻り値なし（elog(ERROR) により処理が中断される）。

前提条件と不変条件

  • targetList 内の各要素は TargetEntry 型である必要があります。
  • 検索対象の sortref は、リスト内のいずれかの TargetEntry の ressortgroupref フィールドに存在していることが期待されます。

アルゴリズムまたは振る舞い

1. targetList を先頭から順に走査します。 2. 各 TargetEntry の ressortgroupref フィールドが、引数 sortref と一致するかを比較します。 3. 一致するものが見つかった場合、その TargetEntry を即座に返します。 4. リストの末尾まで走査しても一致するものが見つからない場合、elog(ERROR) を発行してエラー終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/deparse.c
  • src/backend/optimizer/path/pathkeys.c
  • src/backend/optimizer/plan/createplan.c
  • src/backend/parser/parse_clause.c
  • src/backend/utils/adt/ruleutils.c
  • src/backend/optimizer/util/tlist.c（自身を再帰的に呼び出す箇所あり）
  • **呼び出し先**:
  • lfirst (マクロ): リスト要素の取得。
  • elog: エラー報告。

関連データ構造

  • TargetEntry: ターゲットリストの各要素を構成する構造体。ressortgroupref フィールドを保持します。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • **失敗モード**: 指定された sortref が targetList 内に存在しない場合、elog(ERROR) が発生し、クエリ処理が停止します。
  • **境界条件**: 空のリストが渡された場合、必ずエラーとなります。

保守担当者向けメモ

  • 本関数はプランナがソートやグループ化のキーをターゲットリストと照合する際に頻繁に使用されます。
  • ressortgroupref が 0 の場合（ソートやグループ化に関与しない列）は、通常この関数で検索対象とすべきではありません。
