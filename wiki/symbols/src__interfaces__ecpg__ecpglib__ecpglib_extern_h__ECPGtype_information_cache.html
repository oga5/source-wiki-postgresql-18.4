ECPGtype_information_cache

定義

  • **種別**: struct
  • **定義箇所**: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:52

目的

データベースから取得した型情報（OIDおよび配列属性）をキャッシュし、ECPGライブラリ内での型解決を効率化するためのリンクリスト構造体です。

シグネチャまたは形状

struct ECPGtype_information_cache {
    struct ECPGtype_information_cache *next;
    int oid;
    enum ARRAY_TYPE isarray;
};

入力と出力

  • **入力**: 構造体生成時に oid（型識別子）と isarray（配列型かどうか）が設定されます。
  • **出力**: リンクリストの next ポインタを介して、後続のキャッシュエントリへアクセス可能です。

前提条件と不変条件

  • ecpg_alloc を使用してメモリが確保される必要があります。
  • リストの末尾は NULL で終端される必要があります。

アルゴリズムまたは振る舞い

  • ecpg_type_infocache_push 関数により、新しい型情報がリストの先頭または末尾に追加されます。
  • 実行時に型情報を検索する際、このキャッシュを走査することで、データベースへの再問い合わせを回避します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/execute.c 内の ecpg_type_infocache_push がこの構造体のインスタンスを生成します。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ARRAY_TYPE: isarray フィールドで使用される列挙型。
  • connection: 接続ごとのキャッシュヘッド（cache_head）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: ecpg_alloc が失敗した場合、プログラムは終了またはエラーハンドリングへ移行します。
  • 検索失敗時: キャッシュ内に該当する oid が存在しない場合、リストの走査が終了し、新規登録が必要となります。

保守担当者向けメモ

  • 本構造体は ecpglib 内部の型解決キャッシュとして機能します。
  • ecpg_alloc を使用しているため、メモリ解放の際は対応する解放関数が適切に呼ばれているか確認してください。
  • 構造体の追加・削除ロジックは execute.c に依存しています。
