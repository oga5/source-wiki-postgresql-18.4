ExecInitJunkFilterConversion

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execJunk.c:136

目的

ターゲットリストからジャンク属性を除外したタプルを生成するための JunkFilter 構造体を初期化し、元のタプル属性と出力タプル属性間のマッピング（cleanMap）を構築します。

シグネチャまたは形状

JunkFilter *ExecInitJunkFilterConversion(List *targetList, TupleDesc cleanTupType, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • targetList: ターゲットエントリのリスト。
  • cleanTupType: 出力用タプルの記述子。
  • slot: フィルタリング結果を格納するタプルスロット（NULLの場合は新規作成）。
  • **出力**:
  • 初期化済みの JunkFilter 構造体へのポインタ。

前提条件と不変条件

  • cleanMap は cleanTupType の属性数（natts）に基づき割り当てられます。
  • cleanMap の各要素は、出力タプルの属性に対応する元のタプルの属性番号を保持します。
  • 削除された属性（attisdropped）に対応する cleanMap のエントリは 0 となります。

アルゴリズムまたは振る舞い

1. **スロット設定**: slot が提供されていればその記述子を cleanTupType に設定し、なければ MakeSingleTupleTableSlot で新規作成します。 2. **マッピング計算**: cleanTupType の各属性に対し、targetList を走査します。

  • resjunk フラグが立っていない TargetEntry を見つけるまでリストを辿り、その属性番号を cleanMap に記録します。

    • 3. **構造体構築**: JunkFilter ノードを割り当て、ターゲットリスト、記述子、マッピング、結果スロットを格納して返します。

関連データ構造

  • JunkFilter: フィルタリングに必要な情報を保持する構造体。
  • TargetEntry: ターゲットリストの各要素。resjunk フラグによりジャンク属性か否かを判定します。
  • TupleDesc: タプルの構造定義。

失敗モードと境界条件

  • cleanTupType->natts が 0 の場合、cleanMap は NULL となり、フィルタリング処理は行われません。
  • メモリ不足時には palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本関数は、targetList 内の非ジャンク属性を順次マッピングする設計です。targetList の順序と cleanTupType の属性順序が一致していることが前提となります。
  • cleanMap の値 0 は、出力タプルにおいて NULL が生成されるべき箇所を指します。
