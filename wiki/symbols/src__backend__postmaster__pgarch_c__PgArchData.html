PgArchData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/postmaster/pgarch.c:85

目的

WALアーカイバプロセスの状態を共有メモリ上で管理するためのデータ構造です。

シグネチャまたは形状

typedef struct PgArchData
{
    int             pgprocno;               /* アーカイバプロセスのプロセス番号 */
    pg_atomic_uint32 force_dir_scan;        /* ディレクトリ再スキャンを強制するフラグ */
} PgArchData;

入力と出力

  • **入力**: force_dir_scan フィールドを介して、外部からディレクトリのスキャンをトリガーするシグナルを受け取ります。
  • **出力**: アーカイバプロセスが自身の状態（プロセス番号等）を共有メモリに公開します。

前提条件と不変条件

  • 共有メモリ上に配置され、Postmasterおよびアーカイバプロセス間で共有されます。
  • force_dir_scan はアトミック操作によって更新される必要があります。

アルゴリズムまたは振る舞い

  • pgarch_readyXlog() 内で force_dir_scan が参照され、値が設定されている場合にディレクトリの強制スキャンが行われます。
  • アーカイバプロセスの起動時に、自身のプロセス番号（pgprocno）が本構造体に格納されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/pgarch.c 内の共有メモリ初期化処理およびアーカイバメインループから参照されます。
  • **呼び出し先**:
  • 特定の関数呼び出し関係はソース抜粋からは不明です。

関連データ構造

  • PgArch (静的ポインタ変数): PgArchData 型の共有メモリ領域を指すポインタ。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、アーカイバプロセスは正常に初期化されません。
  • アトミック操作の競合については、pg_atomic_uint32 の使用により保護されています。

保守担当者向けメモ

  • 本構造体は src/backend/postmaster/pgarch.c 内の共有メモリ管理（PgArchShmemInit 等）と密接に関連しています。
  • 構造体へのフィールド追加時は、共有メモリサイズ計算（PgArchShmemSize）の更新を忘れないでください。
