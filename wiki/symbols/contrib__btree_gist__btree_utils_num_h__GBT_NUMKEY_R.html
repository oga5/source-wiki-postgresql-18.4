GBT_NUMKEY_R

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_utils_num.h:16

目的

数値型 GiST 索引において、範囲検索や比較操作を容易にするため、キーの最小値（下限）と最大値（上限）を保持する構造体です。

シグネチャまたは形状

typedef struct
{
	const GBT_NUMKEY *lower,
			   *upper;
} GBT_NUMKEY_R;

入力と出力

  • **入力**: lower および upper メンバは、GBT_NUMKEY 型へのポインタとして初期化されます。
  • **出力**: 本構造体自体が、GiST 索引の各メソッド（consistent 等）において範囲情報を伝達するコンテナとして機能します。

前提条件と不変条件

  • lower および upper は GBT_NUMKEY を指す必要があります。
  • 範囲の整合性（lower が upper 以下であること）は、本構造体を利用する各関数側のロジックに依存します。

アルゴリズムまたは振る舞い

GiST 索引の数値型サポートにおいて、複数のキーから最小包含範囲を算出する際や、クエリ条件との一致判定を行う際に、範囲の境界を一時的に保持するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のソースファイルでローカル変数として宣言・利用されています。
  • contrib/btree_gist/btree_bool.c
  • contrib/btree_gist/btree_cash.c
  • contrib/btree_gist/btree_date.c
  • contrib/btree_gist/btree_enum.c
  • contrib/btree_gist/btree_float4.c
  • contrib/btree_gist/btree_float8.c
  • contrib/btree_gist/btree_inet.c
  • contrib/btree_gist/btree_int2.c
  • **呼び出し先**: 構造体メンバへのアクセスのみであり、本構造体自体が関数を呼び出すことはありません。

関連データ構造

  • GBT_NUMKEY: 数値型キーの内部表現。本構造体はこれら2つを束ねる役割を持ちます。

失敗モードと境界条件

  • ポインタが NULL の場合、参照時にセグメンテーション違反が発生する可能性があります。
  • 範囲の逆転（lower > upper）が発生した場合の挙動は、本構造体を利用する各メソッドの実装に依存します。

保守担当者向けメモ

  • 本構造体は btree_gist モジュール内の多くの数値型サポートファイルで共通利用されています。変更を加える際は、依存するすべての btree_*.c ファイルへの影響を考慮してください。
