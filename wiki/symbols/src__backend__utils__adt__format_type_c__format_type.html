format_type

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/format_type.c:59

目的

データ型OIDおよび型修飾子（typmod）を、人間が読みやすい標準的なSQL形式の文字列に変換します。

シグネチャまたは形状

Datum format_type(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: Oid (データ型のOID)
  • 第2引数: int32 (型修飾子、省略時は -1)
  • **出力**:
  • text 型の文字列（SQL形式の型名）
  • 第1引数がNULLの場合は NULL を返却

前提条件と不変条件

  • 本関数は strict ではないため、呼び出し側で引数のNULLチェックを行っています。
  • 内部で format_type_extended を呼び出し、変換処理を委譲します。

アルゴリズムまたは振る舞い

1. 第1引数（OID）がNULLであれば即座にNULLを返却します。 2. 第2引数（typmod）がNULLであれば -1 を設定し、それ以外の場合は FORMAT_TYPE_TYPEMOD_GIVEN フラグを立てて値を設定します。 3. format_type_extended を呼び出し、型名と修飾子を解決します。 4. 取得した文字列を cstring_to_text で text 型に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: format_type_extended
  • **呼び出し元**: ソースコード内のツール検索結果には、postgres_fdw やパーサー関連のコードで format_type という名前の構造体メンバへの言及がありますが、本関数（format_type 関数）を直接呼び出している具体的な箇所は、提供されたソース excerpt の範囲内では特定できません。

関連データ構造

  • Oid: PostgreSQLのシステムカタログにおけるオブジェクト識別子。
  • Datum: PostgreSQLの汎用データ表現。
  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • 第1引数がNULLの場合、安全にNULLを返却します。
  • typmod が指定されていない場合、デフォルト値 -1 として処理されます。

保守担当者向けメモ

  • 本関数は format_type_extended のラッパーです。ロジックの変更が必要な場合は、まず format_type_extended の仕様を確認してください。
  • typmod の有無による挙動の違いは、SQLパーサーの解釈を維持するための重要な仕様です。
