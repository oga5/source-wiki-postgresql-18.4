aclitem_match

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/acl.c:713

目的

2つの AclItem 構造体が、同一の被付与者（grantee）および付与者（grantor）を持つかどうかを判定します。

シグネチャまたは形状

static bool aclitem_match(const AclItem *a1, const AclItem *a2)

入力と出力

  • **入力**: 比較対象となる2つの AclItem 構造体へのポインタ (a1, a2)。
  • **出力**: 両者の ai_grantee および ai_grantor フィールドが一致する場合は true、それ以外の場合は false を返します。

前提条件と不変条件

  • a1 および a2 は有効な AclItem 構造体を指している必要があります。
  • 権限ビット（ai_privs）の内容は比較対象に含まれません。

アルゴリズムまたは振る舞い

1. a1 の ai_grantee と a2 の ai_grantee を比較します。 2. a1 の ai_grantor と a2 の ai_grantor を比較します。 3. 両方の条件が一致する場合にのみ true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/acl.c:1026 (ACL 更新処理内)
  • src/backend/utils/adt/acl.c:1188 (ACL 比較処理内)
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、構造体フィールドの比較のみを行います。

関連データ構造

  • AclItem: 権限付与の最小単位を保持する構造体。ai_grantee（被付与者）、ai_grantor（付与者）、ai_privs（権限ビット）で構成されます。

失敗モードと境界条件

  • ポインタが NULL の場合の動作は定義されていません。
  • 権限ビットが異なる場合でも、被付与者と付与者が一致すれば true を返します。

保守担当者向けメモ

  • 本関数は AclItem のアイデンティティ（誰から誰への権限か）を判定する目的で使用されます。権限内容そのものの比較が必要な場合は、別途 aclitem_eq 等の利用を検討してください。
