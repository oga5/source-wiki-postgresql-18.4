CreateStandaloneExprContext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execUtils.c:356

目的

式評価に必要な ExprContext 構造体を、特定の実行状態（EState）に依存せずに独立して生成・初期化します。

シグネチャまたは形状

ExprContext *CreateStandaloneExprContext(void)

入力と出力

  • 入力: なし
  • 出力: 初期化済みの ExprContext 構造体へのポインタ。

前提条件と不変条件

  • 呼び出し時の CurrentMemoryContext が、ExprContext ノード自体の割り当て先となります。
  • ecxt_per_tuple_memory は、CurrentMemoryContext を親として AllocSetContextCreate により作成されます。
  • 初期化後の ExprContext は、ecxt_estate が NULL であるため、EState を必要とする式評価には使用できません。

アルゴリズムまたは振る舞い

1. makeNode(ExprContext) を使用してメモリを確保します。 2. 各タプルポインタ（ecxt_scantuple, ecxt_innertuple, ecxt_outertuple）を NULL に初期化します。 3. ecxt_per_query_memory を現在のメモリコンテキストに設定します。 4. 式評価用の作業メモリとして ecxt_per_tuple_memory を作成します。 5. パラメータリスト、集約値、ケース式・ドメイン式の値を初期状態（NULL またはデフォルト値）に設定します。 6. コールバックリスト ecxt_callbacks を NULL に初期化します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/executor/execGrouping.c:267
  • src/backend/utils/adt/domains.c:172
  • src/backend/utils/adt/orderedsetaggs.c:1325
  • 呼び出し先:
  • makeNode (マクロ)
  • AllocSetContextCreate

関連データ構造

  • ExprContext: 本関数が生成する主要な構造体。
  • MemoryContext: ecxt_per_tuple_memory の管理に使用。

失敗モードと境界条件

  • メモリ不足時には AllocSetContextCreate または makeNode がエラーを発生させます。
  • EState が NULL であるため、EState を参照する式評価関数を呼び出すとクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は、クエリ実行のメインフロー外で式評価を行う必要があるコンポーネント（ハッシュテーブルのグループ化やドメインチェックなど）で利用されます。
  • ecxt_per_tuple_memory は AllocSetContextCreate で作成されるため、不要になった際は適切にメモリコンテキストを解放する必要があります。
