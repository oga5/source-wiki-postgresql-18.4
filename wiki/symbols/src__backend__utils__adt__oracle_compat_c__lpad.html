lpad

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:162

目的

入力文字列の左側に指定されたパディング文字列を繰り返し付与し、指定された文字数に調整する。

シグネチャまたは形状

Datum lpad(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • string1 (text): 対象文字列。
  • len (int32): 目標とする文字数。
  • string2 (text): パディングに使用する文字列。
  • **出力**:
  • ret (text): パディングまたは切り詰め後の文字列。

前提条件と不変条件

  • len が負の値の場合、0 として扱われる。
  • string1 の文字数が len を超える場合、左側から len 文字に切り詰められる。
  • string2 が空の場合、パディングは行われない。
  • マルチバイト文字を考慮し、文字単位での処理を行う。

アルゴリズムまたは振る舞い

1. string1 の文字数をカウントし、len と比較して切り詰めが必要か判定する。 2. 出力に必要なメモリ量を計算し、オーバーフローチェックを行った上で palloc を実行する。 3. len と string1 の文字数の差分（m）だけ、string2 を先頭から順にコピーする。string2 の末尾に達した場合は先頭に戻る（ラップアラウンド）。 4. 続いて string1 の内容をコピーする。 5. 最終的なバイトサイズを確定し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出される（ソースコード内の直接的な呼び出し関係は、提供された抜粋からは確認できません）。
  • **呼び出し先**: pg_mbstrlen_with_len, pg_mul_s32_overflow, pg_add_s32_overflow, AllocSizeIsValid, palloc, pg_mblen_range, memcpy, pg_mblen_unbounded, SET_VARSIZE 等の PostgreSQL 内部関数およびマクロ。

関連データ構造

  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • **メモリ不足**: palloc が失敗した場合、PostgreSQL のエラー処理機構により中断される。
  • **サイズ超過**: 計算された出力サイズが許容範囲を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーを報告する。
  • **空文字列**: string2 が空の場合、パディング処理はスキップされる。

保守担当者向けメモ

  • 本関数はマルチバイト文字を正しく扱うため、pg_mblen_range および pg_mblen_unbounded を使用している。新しい文字列操作関数を追加する際は、この文字境界判定パターンを維持すること。
  • string2 のラップアラウンド処理は、ptr2 が ptr2end に達した際に ptr2start へリセットすることで実現されている。
