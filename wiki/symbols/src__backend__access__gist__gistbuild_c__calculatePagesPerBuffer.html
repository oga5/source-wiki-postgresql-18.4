calculatePagesPerBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuild.c:788

目的

GiSTインデックスのバッファリング構築において、特定のレベルステップ数に基づき、各バッファに割り当てるべきページ数を算出します。

シグネチャまたは形状

static int calculatePagesPerBuffer(GISTBuildState *buildstate, int levelStep)

入力と出力

  • **入力**:
  • buildstate: 現在のインデックス構築状態を保持する GISTBuildState 構造体へのポインタ。
  • levelStep: バッファリングの階層ステップ数。
  • **出力**:
  • 算出されたバッファあたりのページ数（int 型）。

前提条件と不変条件

  • buildstate が有効であり、indtuples（挿入済みタプル数）が0でないこと。
  • BLCKSZ（ページサイズ）が、ヘッダーおよび空き領域設定よりも十分に大きいこと。

アルゴリズムまたは振る舞い

1. インデックスページ内でタプル格納に使用可能な領域（pageFreeSpace）を算出します。 2. buildstate に蓄積された統計情報から、平均タプルサイズ（itupAvgSize）を求めます。 3. 1ページあたりの平均タプル数（avgIndexTuplesPerPage）を算出します。 4. 以下の式を用いてバッファあたりのページ数を計算します： pagesPerBuffer = 2 * (avgIndexTuplesPerPage ^ levelStep) 5. 計算結果を rint() で丸め、整数値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuild.c 内のバッファリング構築ロジック（行767および行877）から呼び出されます。
  • **呼び出し先**:
  • pow（数学関数）
  • rint（数学関数）

関連データ構造

  • GISTBuildState: 構築中のインデックス統計情報（indtuplesSize, indtuples, freespace）を保持するために使用されます。

失敗モードと境界条件

  • buildstate->indtuples が0の場合、除算によるゼロ除算が発生する可能性があります。
  • levelStep が極端に大きい場合、pow 関数の結果が double 型の範囲を超える可能性があります。

保守担当者向けメモ

  • 本関数はGiSTのバッファリング構築におけるメモリ効率を左右するヒューリスティックな計算を行っています。計算式の変更は、インデックス構築時のI/O効率に直接影響します。
