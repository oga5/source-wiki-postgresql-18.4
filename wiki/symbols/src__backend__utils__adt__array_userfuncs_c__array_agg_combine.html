array_agg_combine

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:601

目的

array_agg 集約関数において、並列クエリ実行時などに分割された複数の集約状態（ArrayBuildState）を一つに統合するために使用されます。

シグネチャまたは形状

Datum array_agg_combine(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: 2つの ArrayBuildState ポインタ（PG_GETARG_POINTER 経由）。
  • **出力**: 統合された ArrayBuildState へのポインタ。いずれの入力も NULL の場合は NULL を返すことがあります。

前提条件と不変条件

  • AggCheckCallContext により、集約コンテキスト内で呼び出されている必要があります。
  • 統合される2つの状態の要素型（element_type）は一致している必要があります（Assert で検証）。

アルゴリズムまたは振る舞い

1. **コンテキスト確認**: 集約コンテキストを取得し、非集約コンテキストからの呼び出しをエラーとします。 2. **状態の統合**:

  • state2 が NULL の場合: state1 をそのまま返します。
  • state1 が NULL の場合: state2 の内容を現在の集約コンテキストにコピーして新しい ArrayBuildState を作成し、返します。
  • 両方存在する場合: state1 の容量が不足していれば repalloc で拡張し、state2 の要素を state1 の末尾に追加します。

    • 3. **メモリ管理**: datumCopy を使用して要素をコピーし、必要に応じて MemoryContextSwitchTo でメモリコンテキストを切り替えます。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の集約実行エンジン（nodeAgg.c 等）から呼び出されることが想定されますが、ソースコード上での直接的な呼び出し元情報は提供されていません。
  • **呼び出し先**: initArrayResultWithSize, datumCopy, pg_nextpower2_32, repalloc, MemoryContextSwitchTo 等のメモリ管理および配列操作ユーティリティ。

関連データ構造

  • ArrayBuildState: 配列構築中の状態を保持する構造体。
  • MemoryContext: メモリ割り当ての管理コンテキスト。

失敗モードと境界条件

  • 集約コンテキスト外で呼び出された場合、elog(ERROR) により停止します。
  • state1 と state2 の要素型が異なる場合、アサーション違反が発生します。
  • メモリ不足時には repalloc 等がエラーを発生させます。

保守担当者向けメモ

  • 本関数は array_agg の並列処理を支える重要な関数です。state1 の拡張時に pg_nextpower2_32 を使用して再割り当て回数を抑える最適化が行われています。
  • state2 の要素をコピーする際、dnulls 配列のコピーには memcpy が使用されています。
