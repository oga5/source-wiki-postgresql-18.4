make_parsestate

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_node.c:38

目的

クエリ解析のコンテキストを保持する ParseState 構造体を生成および初期化します。

シグネチャまたは形状

ParseState *make_parsestate(ParseState *parentParseState);

入力と出力

  • **入力**: parentParseState (親となる ParseState へのポインタ、または NULL)
  • **出力**: 初期化済みの ParseState 構造体へのポインタ

前提条件と不変条件

  • palloc0 を使用してメモリを確保するため、戻り値の構造体はゼロクリアされています。
  • p_next_resno は 1 に初期化されます。
  • p_resolve_unknowns は true に設定されます。

アルゴリズムまたは振る舞い

1. palloc0 により ParseState 用のメモリを確保します。 2. 引数 parentParseState を pstate->parentParseState に格納します。 3. p_next_resno および p_resolve_unknowns をデフォルト値で初期化します。 4. parentParseState が提供されている場合、以下の情報を親からコピーします。

  • p_sourcetext
  • 各種フック関数 (p_pre_columnref_hook, p_post_columnref_hook, p_paramref_hook, p_coerce_param_hook)
  • p_ref_hook_state
  • p_queryEnv

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/commands/functioncmds.c
  • src/backend/commands/policy.c
  • src/backend/commands/prepare.c
  • src/backend/commands/publicationcmds.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**: palloc0 (PostgreSQL メモリ管理ルーチン)

関連データ構造

  • ParseState: 解析中のクエリ状態を保持する主要な構造体。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • parentParseState が NULL の場合、フックや環境情報はコピーされず、デフォルトの初期化のみが行われます。

保守担当者向けメモ

  • ParseState は再帰的なクエリ解析をサポートしており、parentParseState を介して階層構造を形成します。
  • 新しいフックや状態変数を ParseState に追加した場合は、本関数での初期化およびコピー処理を更新する必要があります。
