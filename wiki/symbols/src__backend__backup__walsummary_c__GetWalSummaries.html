GetWalSummaries

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummary.c:42

目的

pg_wal/summaries ディレクトリを走査し、指定されたタイムラインおよびLSN範囲の条件に合致するWAL要約ファイルのリストを生成します。

シグネチャまたは形状

List *GetWalSummaries(TimeLineID tli, XLogRecPtr start_lsn, XLogRecPtr end_lsn)

入力と出力

  • **入力**:
  • tli: フィルタリング対象のタイムラインID（0の場合は全タイムラインを対象とする）。
  • start_lsn: 範囲の開始LSN（InvalidXLogRecPtr の場合は制限なし）。
  • end_lsn: 範囲の終了LSN（InvalidXLogRecPtr の場合は制限なし）。
  • **出力**:
  • 条件に合致する WalSummaryFile 構造体のリスト（List *）。

前提条件と不変条件

  • pg_wal/summaries ディレクトリが存在し、読み取り可能であること。
  • ファイル名は TLI + StartLSN + EndLSN を16進数で連結した形式であること。
  • start_lsn または end_lsn が指定されている場合、要約ファイルの範囲と重複するものが抽出されます。

アルゴリズムまたは振る舞い

1. AllocateDir を使用して XLOGDIR "/summaries" を開きます。 2. ReadDir でディレクトリ内の全ファイルを走査します。 3. IsWalSummaryFilename でファイル名の形式を検証し、適合しない場合はスキップします。 4. ファイル名から tli、start_lsn、end_lsn をデコードします。 5. 以下のフィルタ条件を適用します：

  • tli が指定されている場合、一致しないものは除外。
  • start_lsn が指定されている場合、start_lsn >= file_end_lsn なら除外。
  • end_lsn が指定されている場合、end_lsn <= file_start_lsn なら除外。

    • 6. 条件を通過したファイルを palloc で確保した WalSummaryFile 構造体に格納し、リストに追加します。 7. FreeDir でディレクトリを閉じ、リストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c
  • src/backend/backup/walsummaryfuncs.c
  • src/backend/postmaster/walsummarizer.c
  • **呼び出し先**:
  • AllocateDir, ReadDir, FreeDir (ディレクトリ操作)
  • IsWalSummaryFilename (ファイル名検証)
  • sscanf (ファイル名デコード)
  • palloc, lappend (メモリおよびリスト操作)

関連データ構造

  • WalSummaryFile: ファイルのタイムラインとLSN範囲を保持する構造体。

失敗モードと境界条件

  • ディレクトリが開けない場合や読み取りエラーが発生した場合は、PostgreSQLの標準的なエラー処理（ereport等）が呼び出される可能性があります。
  • tli が 0 の場合はタイムラインのフィルタリングを行いません。
  • start_lsn や end_lsn が InvalidXLogRecPtr の場合は、その境界に関するフィルタリングをスキップします。

保守担当者向けメモ

  • ファイル名のデコードには sscanf を使用しており、ファイル名の形式変更には注意が必要です。
  • 戻り値のリストは palloc されているため、呼び出し側で適切にメモリを解放する必要があります。
