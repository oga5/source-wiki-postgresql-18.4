SimpleXLogPageRead

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/parsexlog.c:274

目的

XLogReader が WAL ページを読み込む際に使用するコールバック関数であり、指定された WAL セグメントファイルからページデータを取得してバッファへ読み込む。

シグネチャまたは形状

static int SimpleXLogPageRead(XLogReaderState *xlogreader, XLogRecPtr targetPagePtr, int reqLen, XLogRecPtr targetRecPtr, char *readBuf)

入力と出力

  • **入力**:
  • xlogreader: WAL リーダーの状態構造体。
  • targetPagePtr: 読み込み対象の WAL ページ位置。
  • reqLen: 要求された長さ。
  • targetRecPtr: 読み込み対象のレコード位置。
  • readBuf: 読み込んだデータを格納するバッファ。
  • **出力**:
  • 成功時: XLOG_BLCKSZ（読み込んだバイト数）。
  • 失敗時: -1。

前提条件と不変条件

  • xlogreader->private_data に XLogPageReadPrivate 構造体が格納されていること。
  • WalSegSz がグローバル変数として定義されていること。
  • 読み込み成功後、xlogreader->seg.ws_tli が現在のタイムライン ID に更新されること。

アルゴリズムまたは振る舞い

1. targetPagePtr から対象のセグメント番号を計算する。 2. 現在開いているファイルが対象セグメントと異なる場合、既存のファイルディスクリプタを閉じる。 3. ファイルが未オープンであれば、targetHistory を参照して適切なタイムラインの WAL ファイルを特定し、開く。 4. ファイルが開けない場合、restoreCommand が設定されていれば RestoreArchivedFile を呼び出してアーカイブから取得を試みる。 5. lseek で対象オフセットへ移動し、read で XLOG_BLCKSZ 分のデータを readBuf に読み込む。 6. 読み込み完了後、リーダーのタイムライン情報を更新して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: XLogReader の内部処理（XL_ROUTINE 構造体を通じてコールバックとして登録）。
  • **呼び出し先**: RestoreArchivedFile（アーカイブからの復元が必要な場合）。

関連データ構造

  • XLogReaderState: WAL リーダーの状態管理。
  • XLogPageReadPrivate: タイムライン情報や restore_command を保持するプライベートデータ。
  • targetHistory: タイムラインの切り替わりを管理する配列。

失敗モードと境界条件

  • 指定された WAL ファイルが存在せず、かつ restore_command も提供されていない場合はエラーを返して終了する。
  • lseek または read が失敗した場合はエラーを記録し -1 を返す。
  • 読み込みサイズが XLOG_BLCKSZ に満たない場合はエラーとして扱う。

保守担当者向けメモ

  • 本関数は pg_rewind における WAL 走査の基盤であり、タイムライン切り替えロジックを含む。
  • 複数のタイムラインにまたがる WAL を扱う際、private->tliIndex の更新ロジックが正しく機能していることを確認すること。
  • xlogreadfd はグローバル変数として管理されているため、並列実行には適さない点に注意。
