pgstat_copy_relation_stats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_relation.c:56

目的

あるリレーション（src）の統計情報を別のリレーション（dst）へコピーし、共有メモリ上の統計エントリを更新する。

シグネチャまたは形状

void pgstat_copy_relation_stats(Relation dst, Relation src)

入力と出力

  • **入力**:
  • dst: 統計情報のコピー先となるリレーション。
  • src: 統計情報のコピー元となるリレーション。
  • **出力**: なし（副作用として共有メモリ上の統計エントリが更新される）。

前提条件と不変条件

  • src および dst は有効な Relation オブジェクトであること。
  • pgstat_fetch_stat_tabentry_ext が src の統計情報を取得できること。

アルゴリズムまたは振る舞い

1. pgstat_fetch_stat_tabentry_ext を呼び出し、src リレーションの統計情報（PgStat_StatTabEntry）を取得する。取得できない場合は即座に終了する。 2. pgstat_get_entry_ref_locked を呼び出し、dst リレーションに対応する共有メモリ上の統計エントリへの参照をロック付きで取得する。

  • 共有リレーションか否かに応じてデータベースIDを判定する。

    • 3. 取得した dst の統計エントリ（PgStatShared_Relation）の stats フィールドに、src から取得した統計情報をコピーする。 4. pgstat_unlock_entry を呼び出し、ロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c:1798
  • **呼び出し先**:
  • pgstat_fetch_stat_tabentry_ext
  • pgstat_get_entry_ref_locked
  • pgstat_unlock_entry

関連データ構造

  • Relation: リレーションキャッシュ構造体。
  • PgStat_StatTabEntry: リレーションの統計情報を保持する構造体。
  • PgStatShared_Relation: 共有メモリ上に配置されるリレーション統計エントリ。
  • PgStat_EntryRef: 統計エントリへの参照を管理する構造体。

失敗モードと境界条件

  • src の統計情報が存在しない場合、関数は何もせず正常終了する。
  • pgstat_get_entry_ref_locked が失敗した場合の挙動は、当該関数の実装に依存する（通常はエラーを投げる）。

保守担当者向けメモ

  • 本関数は統計情報の直接的な上書きを行うため、呼び出し元で dst の状態が整合していることを確認すること。
  • 共有メモリのロックを保持するため、呼び出し後の pgstat_unlock_entry が確実に実行される設計となっている。
