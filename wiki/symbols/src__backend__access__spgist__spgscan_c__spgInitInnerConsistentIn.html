spgInitInnerConsistentIn

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgscan.c:605

目的

SP-GiST インデックスの内部ノードに対する整合性チェック（inner_consistent）を実行するために必要な入力構造体 spgInnerConsistentIn を初期化します。

シグネチャまたは形状

static void spgInitInnerConsistentIn(spgInnerConsistentIn *in, SpGistScanOpaque so, SpGistSearchItem *item, SpGistInnerTuple innerTuple)

入力と出力

  • **入力**:
  • in: 初期化対象の spgInnerConsistentIn 構造体。
  • so: 現在のスキャン状態を保持する SpGistScanOpaque。
  • item: 走査中の検索アイテム（SpGistSearchItem）。
  • innerTuple: 処理対象の内部タプル（SpGistInnerTuple）。
  • **出力**: なし（in が指す構造体の各フィールドが設定されます）。

前提条件と不変条件

  • item->isLeaf が false であること（アサーションにより検証）。
  • so および innerTuple は有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. so からスキャンキー、Order-by キー、キー数、メモリコンテキスト等の情報を in にコピーします。 2. item から再構築された値、トラバーサル値、レベル情報を in に設定します。 3. innerTuple から allTheSame フラグ、プレフィックスの有無、プレフィックスデータ、ノード数、およびノードラベル（spgExtractNodeLabels を使用）を抽出し、in に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgscan.c 内の spgInnerTest 関数（行681）から呼び出されます。
  • **呼び出し先**: spgExtractNodeLabels を呼び出します。

関連データ構造

  • spgInnerConsistentIn: 演算子クラスの inner_consistent メソッドに渡される入力構造体。
  • SpGistScanOpaque: スキャン状態を保持する構造体。
  • SpGistSearchItem: 検索キュー内のアイテム。
  • SpGistInnerTuple: SP-GiST の内部ノードタプル。

失敗モードと境界条件

  • item->isLeaf が真の場合、アサーションエラーが発生します。
  • innerTuple->prefixSize が 0 以下の場合、hasPrefix は偽となり、prefixDatum は設定されません。

保守担当者向けメモ

  • 本関数は inner_consistent メソッドへ渡すパラメータの準備に特化しています。演算子クラス側で必要な情報が不足している場合は、本関数にフィールドを追加してください。
  • SGITDATUM マクロを使用してプレフィックスデータを抽出しているため、SpGistInnerTuple のメモリレイアウト変更時には注意が必要です。
