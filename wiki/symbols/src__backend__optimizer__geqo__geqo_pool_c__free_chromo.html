free_chromo

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_pool.c:175

目的

遺伝的アルゴリズムで使用される染色体（Chromosome）構造体およびその内部データが保持するメモリ領域を解放します。

シグネチャまたは形状

void free_chromo(PlannerInfo *root, Chromosome *chromo)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体へのポインタ（本関数内では使用されません）。
  • chromo: 解放対象の Chromosome 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • chromo は有効なメモリ領域を指している必要があります。
  • chromo->string は、palloc 等で確保された有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. chromo->string が指すメモリ領域を pfree で解放します。 2. chromo 構造体自体を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_main.c 内の各所（momma, daddy, kid といった染色体変数の破棄時に呼び出されます）。
  • **呼び出し先**: PostgreSQL のメモリ管理関数 pfree。

関連データ構造

  • Chromosome: 遺伝的アルゴリズムにおける個体情報を保持する構造体。
  • PlannerInfo: クエリ最適化の情報を保持する構造体。

失敗モードと境界条件

  • chromo が NULL の場合、pfree の仕様に従いセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は Chromosome のメモリ管理を担うため、染色体生成関数（alloc_chromo 等）と対で管理してください。
  • root 引数は現在の実装では使用されていませんが、将来的なメモリコンテキストの拡張に備えてシグネチャに含まれています。
