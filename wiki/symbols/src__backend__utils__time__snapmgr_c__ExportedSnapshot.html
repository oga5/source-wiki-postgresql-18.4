ExportedSnapshot

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/time/snapmgr.c:205

目的

トランザクション間で共有（エクスポート）されるスナップショットのメタデータと実体を保持するために使用されます。

シグネチャまたは形状

typedef struct ExportedSnapshot
{
	char	   *snapfile;
	Snapshot	snapshot;
} ExportedSnapshot;

入力と出力

  • snapfile: エクスポートされたスナップショットが保存されているファイル名（パス）。
  • snapshot: 共有対象となる Snapshot 構造体へのポインタ。

前提条件と不変条件

  • 本構造体は、現在のトランザクションにおいてエクスポートされたスナップショットのリストを管理するために利用されます。

アルゴリズムまたは振る舞い

  • pg_export_snapshot 等の処理において、エクスポートされたスナップショットを追跡するリストの要素としてインスタンス化されます。
  • palloc を用いてメモリが割り当てられ、スナップショットの永続化と管理に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/time/snapmgr.c 内の関数（pg_export_snapshot 等の関連処理）から、リストの走査や新規作成時に参照されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Snapshot: snapshot フィールドが指す実体。
  • List: 本構造体を要素として保持するリスト構造（lfirst(lc) 等でアクセスされる）。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ割り当て失敗時にエラーが発生する可能性があります。
  • ファイル操作: snapfile に関連するファイルシステム上の制約や権限エラーが、エクスポート処理全体に影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/utils/time/snapmgr.c 内で管理されるエクスポート済みスナップショットリストの一部です。
  • トランザクション終了時にこれらのファイルが適切にクリーンアップされるよう、AtEOXact_Snapshot 等のライフサイクル管理関数との整合性に注意してください。
