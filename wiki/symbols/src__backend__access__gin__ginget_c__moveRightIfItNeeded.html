moveRightIfItNeeded

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginget.c:42

目的

GINインデックスの走査中、現在のページで処理すべきオフセットを使い果たした場合に、必要に応じて右隣のページへ移動し、スキャン状態を更新する。

シグネチャまたは形状

static bool moveRightIfItNeeded(GinBtreeData *btree, GinBtreeStack *stack, Snapshot snapshot)

入力と出力

  • **入力**:
  • btree: GIN B-treeの制御構造体。
  • stack: 現在の走査位置（バッファ、ブロック番号、オフセット）を保持するスタック。
  • snapshot: 述語ロックに使用するスナップショット。
  • **出力**:
  • bool: 右隣のページへ正常に移動した、または移動の必要がなかった場合は true。右端のページに到達しており、これ以上移動できない場合は false。

前提条件と不変条件

  • stack->buffer は有効なバッファを指している必要がある。
  • stack->off が現在のページ内の最大オフセット番号を超えている場合、右隣への移動が試行される。

アルゴリズムまたは振る舞い

1. 現在のページ（stack->buffer）の最大オフセット番号を確認する。 2. stack->off が最大値を超えている場合：

  • 現在のページが右端（GinPageRightMost）であれば、移動不可として false を返す。
  • 右端でない場合、ginStepRight を呼び出して右隣のページへ移動する。
  • stack 内の buffer、blkno、off（FirstOffsetNumber にリセット）を更新する。
  • 新しいページに対して PredicateLockPage を実行し、述語ロックを確保する。

    • 3. 移動が完了した、または移動の必要がなかった場合は true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginget.c 内の走査ロジック（gingetbitmap 等の関連関数）から複数回呼び出されている。
  • **呼び出し先**:
  • BufferGetPage
  • PageGetMaxOffsetNumber
  • GinPageRightMost
  • ginStepRight
  • BufferGetBlockNumber
  • PredicateLockPage

関連データ構造

  • GinBtreeData: GIN B-treeの操作に必要な情報を含む構造体。
  • GinBtreeStack: 走査中のページ位置とバッファを管理するスタック構造。

失敗モードと境界条件

  • **右端ページ**: 現在のページがインデックスの右端である場合、false を返して走査の終了を通知する。
  • **ロック**: ページ移動時に ginStepRight を介して共有ロック（GIN_SHARE）を取得する。

保守担当者向けメモ

  • 本関数は、GINのB-tree走査においてページを跨ぐ際の標準的な制御フローである。
  • 述語ロックの整合性を保つため、ページ移動時には必ず PredicateLockPage が呼び出される点に注意すること。
