check_in_colors_match

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:3468

目的

指定された状態 s に流入する遷移（inarcs）において、色 co1 を持つ遷移元と色 co2 を持つ遷移元が、同一の集合を構成しているか（一致しているか）を検証します。

シグネチャまたは形状

static bool check_in_colors_match(struct state *s, color co1, color co2)

入力と出力

  • **入力**:
  • s: 検証対象の struct state へのポインタ。
  • co1, co2: 比較対象となる color 型の値。
  • **出力**:
  • bool: 両方の色が同一の遷移元集合を指している場合に true、不一致がある場合に false を返します。

前提条件と不変条件

  • s->ins を介して状態 s に流入するすべての遷移を走査します。
  • 処理中、一時的に a->from->tmp フィールドをマーカーとして使用します。
  • 処理終了時には、使用した tmp フィールドをすべて NULL にリセットして不変条件を維持します。

アルゴリズムまたは振る舞い

1. **マーク付け**: s に流入する遷移のうち、色が co1 である遷移元の tmp フィールドに自身（from）を代入します。 2. **検証（co2）**: 色が co2 である遷移を走査します。

  • tmp が設定されていれば、対応する co1 遷移が存在するため tmp をクリアします。
  • tmp が設定されていなければ、co2 に対応する co1 遷移がないため false を返します。

    • 3. **残留確認（co1）**: 再度 co1 の遷移を走査し、tmp がクリアされていない（co2 と対にならなかった）ものがあれば false を返します。 4. **終了**: すべての条件を満たせば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_nfa.c:3163-3164 (NFA の後処理における検証)
  • src/backend/regex/regc_nfa.c:3461 (コメント内での言及)
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。struct state および struct arc のメンバアクセスを行います。

関連データ構造

  • struct state: NFA の状態を保持。ins（流入遷移リスト）および tmp（一時的な作業用ポインタ）を使用。
  • struct arc: 状態間の遷移を保持。co（色）、from（遷移元）、inchain（流入リストの連結）を使用。

失敗モードと境界条件

  • s->ins が NULL の場合、ループは実行されず true を返します。
  • co1 と co2 が同一の場合、すべての co1 遷移が tmp にマークされ、その後の検証で全てクリアされるため true となります。
  • co1 または co2 の遷移が片方にしか存在しない場合、false を返します。

保守担当者向けメモ

  • check_out_colors_match とアルゴリズムが同一です。修正時は両方の関数を確認してください。
  • a->from->tmp を使用するため、この関数が再帰的または並行して呼び出される文脈では競合が発生する可能性があります。現在の実装は単一スレッドでの逐次処理を前提としています。
