ExecInitIndexScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:908

目的

IndexScan ノードの実行状態（IndexScanState）を初期化し、インデックススキャンを実行するための準備を行う。

シグネチャまたは形状

IndexScanState *ExecInitIndexScan(IndexScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の IndexScan プランノード。
  • estate: クエリ実行全体の実行状態（EState）。
  • eflags: 実行フラグ（EXEC_FLAG_EXPLAIN_ONLY 等）。
  • **出力**:
  • 初期化済みの IndexScanState 構造体へのポインタ。

前提条件と不変条件

  • node が有効な IndexScan プランノードであること。
  • eflags に EXEC_FLAG_EXPLAIN_ONLY が含まれる場合、インデックスのオープンやスキャンキーの構築はスキップされる。
  • ORDER BY 句が存在する場合、pairingheap を用いた再ソート用のデータ構造が初期化される。

アルゴリズムまたは振る舞い

1. IndexScanState 構造体を生成し、ExecProcNode に ExecIndexScan を割り当てる。 2. 式評価用のコンテキスト（ExprContext）を割り当てる。 3. ExecOpenScanRelation を呼び出し、対象のテーブルリレーションを開く。 4. スキャンタプルスロットを初期化し、プロジェクション情報を設定する。 5. qual（フィルタ条件）、indexqualorig、indexorderbyorig を初期化する。 6. EXEC_FLAG_EXPLAIN_ONLY でない場合、index_open でインデックスリレーションを開く。 7. ExecIndexBuildScanKeys を使用して、インデックスの ScanKey および OrderByKeys を構築する。 8. ORDER BY 句がある場合、SortSupport を準備し、再ソート用の pairingheap を割り当てる。 9. ランタイムキーが存在する場合、それ専用の iss_RuntimeContext を作成する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:220（ExecInitNode からの呼び出しが確認されている）。
  • **呼び出し先**:
  • ExecAssignExprContext, ExecOpenScanRelation, ExecInitScanTupleSlot, ExecInitResultTypeTL, ExecAssignScanProjectionInfo, ExecInitQual, ExecInitExprList, index_open, ExecIndexBuildScanKeys, PrepareSortSupportFromOrderingOp, pairingheap_allocate 等。

関連データ構造

  • IndexScanState: 本関数が生成・初期化する主要な状態構造体。
  • IndexScan: プランツリー上のノード定義。
  • SortSupportData: ORDER BY 句の評価に使用されるソートサポート情報。
  • pairingheap: ORDER BY 句の再ソートに使用される優先度付きキュー。

失敗モードと境界条件

  • インデックスが存在しない場合やロック取得に失敗した場合はエラーとなる（index_open の挙動に依存）。
  • ORDER BY 句の数と indexorderbyops / indexorderbyorig のリスト長が一致しない場合、Assert により停止する。

保守担当者向けメモ

  • indexqualorig は一部の特殊なケースでのみ使用されるが、現在は常に初期化されている。将来的な最適化の余地がある。
  • ランタイムキーの評価には専用の ExprContext が使用されるため、再スキャン時のコンテキストリセットに注意が必要。
