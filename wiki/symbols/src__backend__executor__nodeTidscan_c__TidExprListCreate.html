TidExprListCreate

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidscan.c:69

目的

TidScan ノードの実行に必要な TID 抽出式をコンパイルし、TidScanState に格納する。

シグネチャまたは形状

static void TidExprListCreate(TidScanState *tidstate)

入力と出力

  • **入力**: tidstate (TidScanState *) - スキャンノードの状態構造体。プラン内の tidquals を参照する。
  • **出力**: なし（tidstate->tss_tidexprs および tidstate->tss_isCurrentOf を更新する副作用を持つ）。

前提条件と不変条件

  • tidstate->ss.ps.plan が TidScan 型としてキャスト可能であること。
  • CurrentOfExpr が含まれる場合、他の式と OR 結合されていないこと（list_length が 1 であること）。

アルゴリズムまたは振る舞い

1. tidstate->tss_tidexprs を NIL に初期化する。 2. node->tidquals リスト内の各式を走査する。 3. 式の種類に応じて以下の処理を行う：

  • **OpExpr**: ctid と比較されている引数を特定し、ExecInitExpr で式状態を初期化する。
  • **ScalarArrayOpExpr**: ctid との配列比較を特定し、ExecInitExpr で式状態を初期化する。isarray フラグを true に設定する。
  • **CurrentOfExpr**: cexpr を保持し、tss_isCurrentOf を true に設定する。

    • 4. 処理した式を tss_tidexprs リストに追加する。 5. 最後に CurrentOfExpr に関する不変条件を Assert で検証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeTidscan.c 内の ExecInitTidScan（直接の呼び出し）。
  • src/backend/executor/nodeTidrangescan.c 内の TidRangeScan 初期化処理（直接の呼び出し）。
  • **呼び出し先**:
  • ExecInitExpr: 式のコンパイルに使用。
  • palloc0: メモリ確保に使用。
  • elog: エラー報告に使用。

関連データ構造

  • TidScanState: スキャン実行時の状態保持。
  • TidScan: プランノード。
  • TidExpr: 抽出された式とメタデータを保持する構造体。
  • ExprState: ExecInitExpr が返す実行可能な式状態。

失敗モードと境界条件

  • ctid を含む式を特定できない場合、elog(ERROR) を発生させる。
  • CurrentOfExpr が他の式と混在している場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は TidScan と TidRangeScan の両方から利用されている。
  • CurrentOfExpr は動的な TID 解決を行うため、通常の ctid 比較とは異なるコードパスを辿る点に注意すること。
