EndOfWalRecoveryInfo

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogrecovery.h:91

目的

WALリカバリ処理が終了した時点での状態（最後に適用されたレコード位置、リカバリ終了理由、リカバリ設定ファイルの検出状況など）を保持し、リカバリ完了後のシステム状態を確定させるために使用されます。

シグネチャまたは形状

typedef struct {
    XLogRecPtr  lastRec;
    TimeLineID  lastRecTLI;
    XLogRecPtr  endOfLog;
    TimeLineID  endOfLogTLI;
    XLogRecPtr  lastPageBeginPtr;
    char       *lastPage;
    XLogRecPtr  abortedRecPtr;
    XLogRecPtr  missingContrecPtr;
    char       *recoveryStopReason;
    bool        standby_signal_file_found;
    bool        recovery_signal_file_found;
} EndOfWalRecoveryInfo;

入力と出力

  • 本構造体は FinishWalRecovery() 関数によって生成・返却されます。
  • 呼び出し元は、この構造体を通じてリカバリ終了時の詳細なメタデータを取得します。

前提条件と不変条件

  • lastPage は endOfLog を含む最後のページ内容のコピーを保持します（endOfLog がページ境界にある場合は NULL）。
  • endOfLogTLI は、最後に適用されたレコードを含むXLOGセグメントのタイムラインIDを示します。

アルゴリズムまたは振る舞い

1. リカバリプロセス終了時に、最後に適用したWALレコードの開始位置（lastRec）と終了位置（endOfLog）を記録します。 2. タイムラインの切り替えが発生した場合、lastRecTLI と endOfLogTLI を適切に区別して保持します。 3. リカバリ終了時に破損したレコードが存在する場合、その開始位置を abortedRecPtr に、欠落した継続レコードの位置を missingContrecPtr に格納します。 4. リカバリ終了の理由を recoveryStopReason に文字列として保持し、スタンバイまたはリカバリ信号ファイルの有無をフラグとして記録します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c 等のリカバリ制御フロー。
  • 呼び出し先: FinishWalRecovery() 関数が本構造体をインスタンス化し、情報を設定します。

関連データ構造

  • XLogRecPtr: WAL上の位置を示すポインタ型。
  • TimeLineID: タイムライン識別子。
  • FinishWalRecovery(): 本構造体を生成する関数。

失敗モードと境界条件

  • endOfLog がページ境界と一致する場合、lastPage は NULL になります。
  • 正常なリカバリ終了時以外に、破損したWALレコードや欠落した継続レコードが検出された場合、abortedRecPtr や missingContrecPtr に非ゼロの値が設定されます。

保守担当者向けメモ

  • 本構造体は src/backend/access/transam/xlogrecovery.c 内でメモリ確保（palloc）が行われます。
  • リカバリ終了時の詳細な診断情報が含まれているため、リカバリ失敗時のデバッグにおいて重要な役割を果たします。
