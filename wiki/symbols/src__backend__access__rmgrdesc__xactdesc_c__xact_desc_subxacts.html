xact_desc_subxacts

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xactdesc.c:298

目的

トランザクションのWALレコードに含まれるサブトランザクションIDのリストを、人間が読める形式の文字列としてバッファに追加します。

シグネチャまたは形状

static void xact_desc_subxacts(StringInfo buf, int nsubxacts, TransactionId *subxacts)

入力と出力

  • buf (入出力): 文字列を追記する StringInfo バッファ。
  • nsubxacts (入力): サブトランザクションの数。
  • subxacts (入力): サブトランザクションIDの配列。

前提条件と不変条件

  • nsubxacts が 0 より大きい場合のみ、バッファに文字列が追加されます。
  • subxacts は少なくとも nsubxacts 個の要素を持つ有効な配列である必要があります。

アルゴリズムまたは振る舞い

1. nsubxacts > 0 の場合、バッファに ; subxacts: という文字列を追記します。 2. 配列 subxacts をループで走査し、各トランザクションIDをスペース区切りでバッファに追記します。 3. nsubxacts <= 0 の場合は何も行いません。

呼び出し元と呼び出し先

  • src/backend/access/rmgrdesc/xactdesc.c:383

関連データ構造

  • StringInfo: 文字列構築用の可変長バッファ構造体。
  • TransactionId: トランザクションを一意に識別する型。

失敗モードと境界条件

  • nsubxacts が 0 の場合、出力には何も影響を与えません。
  • buf がメモリ不足状態にある場合、appendStringInfo 内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は pg_waldump 等のログ出力用であり、WALレコードのパース結果を整形する目的で使用されます。
  • 呼び出し元で parsed.nsubxacts および parsed.subxacts を渡す構成となっており、パース済み構造体との整合性に注意してください。
