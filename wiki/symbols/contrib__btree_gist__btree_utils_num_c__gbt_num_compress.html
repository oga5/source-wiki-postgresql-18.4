gbt_num_compress

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_num.c:13

目的

GiSTインデックスのリーフノードに格納される数値を、インデックス管理用の範囲データ（下限値と上限値のペア）に変換します。

シグネチャまたは形状

GISTENTRY *gbt_num_compress(GISTENTRY *entry, const gbtree_ninfo *tinfo)

入力と出力

  • **入力**:
  • entry: 圧縮対象のデータを含む GISTENTRY 構造体。
  • tinfo: データ型固有のサイズや型情報を保持する gbtree_ninfo 構造体。
  • **出力**:
  • 圧縮後のデータを含む新しい GISTENTRY を返します。リーフノードでない場合は、入力された entry をそのまま返します。

前提条件と不変条件

  • tinfo->indexsize は 2 * tinfo->size 以上である必要があります（Assert で検証）。
  • entry->leafkey が真の場合のみ、数値の圧縮処理が行われます。

アルゴリズムまたは振る舞い

1. entry->leafkey が真の場合、tinfo->t（型情報）に基づき switch 文で入力値を抽出します。 2. 抽出した値を GBT_NUMKEY 構造体（palloc0 で確保）の先頭と、tinfo->size 分オフセットした位置にそれぞれコピーします。これにより、単一の値を [値, 値] という範囲形式に変換します。 3. gistentryinit を使用して、変換後のデータを保持する新しい GISTENTRY を初期化し、返却します。 4. entry->leafkey が偽の場合は、入力された entry をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の各データ型用 GiST 実装ファイルから呼び出されます。
  • btree_bool.c, btree_cash.c, btree_date.c, btree_enum.c, btree_float4.c, btree_float8.c, btree_int2.c, btree_int4.c, btree_int8.c, btree_macaddr.c, btree_macaddr8.c, btree_oid.c
  • **呼び出し先**: PostgreSQL のメモリ管理関数（palloc0, palloc）、データ型変換マクロ（DatumGet...）、および gistentryinit マクロ。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • gbtree_ninfo: 数値型のサイズや型定義を保持する設定構造体。
  • GBT_NUMKEY: インデックス内に格納される範囲データの基底構造。

失敗モードと境界条件

  • tinfo->indexsize が 2 * tinfo->size 未満の場合、Assert により異常終了します。
  • サポートされていない型が tinfo->t に指定された場合、default ケースにより DatumGetPointer が実行されます。

保守担当者向けメモ

  • 新しい数値型をサポートする場合は、switch 文に新しい case を追加する必要があります。
  • GBT_NUMKEY は [下限, 上限] の形式を前提としているため、圧縮後のデータ構造を変更する際は注意してください。
