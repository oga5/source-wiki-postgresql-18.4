BlockIdGetBlockNumber

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/block.h:102

目的

ディスク上の構造体（BlockIdData）に分割保持されているブロック識別子を、メモリ上で操作可能な単一の BlockNumber 型へ復元する。

シグネチャまたは形状

static inline BlockNumber BlockIdGetBlockNumber(const BlockIdData *blockId)

入力と出力

  • **入力**: const BlockIdData *blockId - 復元対象のブロック識別子を指すポインタ。
  • **出力**: BlockNumber - 復元された32ビットのブロック番号。

前提条件と不変条件

  • blockId は有効な BlockIdData 構造体を指している必要がある。
  • BlockIdData 内の bi_hi（上位16ビット）と bi_lo（下位16ビット）は、ビッグエンディアンまたはリトルエンディアンの環境に関わらず、論理的なブロック番号を構成するように格納されているものとする。

アルゴリズムまたは振る舞い

1. blockId->bi_hi を BlockNumber 型にキャストし、16ビット左シフトする。 2. blockId->bi_lo を BlockNumber 型にキャストする。 3. 上記2つの値をビット単位の論理和（OR）で結合し、32ビットの BlockNumber を生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • src/backend/access/gin/ginxlog.c
  • src/backend/access/rmgrdesc/gindesc.c
  • src/include/access/ginblock.h
  • src/include/storage/itemptr.h
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わず、ビット演算のみを実行する。

関連データ構造

  • BlockIdData: BlockNumber を2つの uint16（bi_hi, bi_lo）に分割して保持する構造体。
  • BlockNumber: 32ビット符号なし整数。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作を引き起こす可能性がある。
  • BlockNumber の範囲（0 ～ MaxBlockNumber）を超える値が BlockIdData に格納されていた場合、そのままのビットパターンが返される。

保守担当者向けメモ

  • 本関数は BlockIdSet と対になる変換関数である。
  • ディスク上のパディングを最小化するために BlockIdData を使用しているため、メモリ上の BlockNumber との変換ロジックを変更する場合は、ディスクフォーマットとの整合性に注意すること。
