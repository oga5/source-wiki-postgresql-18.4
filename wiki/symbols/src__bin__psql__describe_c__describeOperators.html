describeOperators

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:793

目的

データベース内の演算子（Operator）情報を取得し、整形して出力する。psql のメタコマンド（\do 等）の実行エンジンとして機能する。

シグネチャまたは形状

bool describeOperators(const char *oper_pattern, char **arg_patterns, int num_arg_patterns, bool verbose, bool showSystem)

入力と出力

  • **入力**:
  • oper_pattern: 演算子名のパターン（NULL可）。
  • arg_patterns: 引数の型パターン配列。
  • num_arg_patterns: 引数の数（最大2まで）。
  • verbose: 詳細表示フラグ。
  • showSystem: システムオブジェクトを含めるかどうかのフラグ。
  • **出力**: 成功時に true、失敗時に false を返す。結果は pset.queryFout に出力される。

前提条件と不変条件

  • PostgreSQL 9.2 以降のサーバーとの接続を想定している。
  • num_arg_patterns が 2 を超える場合は無視され、2 として扱われる。
  • 演算子の説明（Description）取得において、pg_operator のコメントがない場合は pg_proc のコメントをフォールバックとして使用する。

アルゴリズムまたは振る舞い

1. PQExpBuffer を初期化し、pg_catalog.pg_operator を主軸とした SQL クエリを構築する。 2. verbose モードに応じて、関数名や leakproof 属性の列を追加する。 3. 引数の数に応じて pg_type および pg_namespace との LEFT JOIN を動的に追加する。 4. validateSQLNamePattern を使用して、演算子名および型名のパターンを SQL の WHERE 句に安全に変換・結合する。 5. PSQLexec を介してクエリを実行し、printQuery で結果を整形して出力する。 6. 使用したバッファと結果セットを解放し、終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:1332（メタコマンド処理の一部として呼び出される）。
  • **呼び出し先**: initPQExpBuffer, printfPQExpBuffer, appendPQExpBuffer, validateSQLNamePattern, map_typename_pattern, PSQLexec, printQuery, termPQExpBuffer, PQclear 等。

関連データ構造

  • PQExpBufferData: SQL クエリ構築用バッファ。
  • PGresult: クエリ実行結果の保持。
  • printQueryOpt: 出力オプション設定。

失敗モードと境界条件

  • PSQLexec が失敗した場合（接続断やクエリエラー）、false を返して終了する。
  • validateSQLNamePattern によるパターン検証に失敗した場合、error_return ラベルへジャンプし、バッファを解放して false を返す。
  • num_arg_patterns が 1 の場合、左引数が存在しない（oprleft = 0）演算子を対象とする。

保守担当者向けメモ

  • pg_operator の oprkind が 'l'（左単項）や 'r'（右単項）の場合の型表示制御が含まれている。
  • ポストフィックス演算子のサポートは PostgreSQL 14 以降ではデッドコードだが、古いサーバーとの互換性維持のため保持されている。
  • translate_columns 配列により、出力結果の特定の列に対して翻訳が適用される。
