metastring

定義

  • **種別**: struct
  • **定義箇所**: contrib/fuzzystrmatch/dmetaphone.c:218

目的

Double Metaphone アルゴリズムにおいて、発音コード生成過程の文字列を保持・操作するための動的バッファ構造体です。

シグネチャまたは形状

typedef struct
{
    char       *str;                    /* 文字列バッファ */
    int         length;                 /* 現在の文字列長 */
    int         bufsize;                /* 割り当て済みのバッファサイズ */
    int         free_string_on_destroy; /* 破棄時にメモリを解放するかどうかのフラグ */
} metastring;

入力と出力

  • 本構造体自体はデータ保持用であり、関数への引数として渡される際は、主に metastring *s の形式でポインタとして受け渡されます。

前提条件と不変条件

  • str は bufsize 分のメモリ領域を指している必要があります。
  • length は常に bufsize 以下である必要があります。
  • MetaphAdd 等の操作関数により、必要に応じて bufsize が動的に拡張されることが想定されています。

アルゴリズムまたは振る舞い

  • Double Metaphone アルゴリズムの実行中、プライマリおよびオルタネートのコードを構築するために使用されます。
  • MetaphAdd 関数を介して文字が追加され、バッファが不足した場合は自動的に拡張されます。
  • DestroyMetaString 関数によって、メモリ管理ポリシーに従い適切にクリーンアップされます。

呼び出し元と呼び出し先

  • **呼び出し元**: DoubleMetaphone 関数内で初期化され、各補助関数へ渡されます。
  • **呼び出し先**: 以下の関数が本構造体を操作します。
  • DestroyMetaString
  • IncreaseBuffer
  • MakeUpper
  • IsVowel
  • SlavoGermanic
  • GetAt / SetAt
  • StringAt
  • MetaphAdd

関連データ構造

  • DoubleMetaphone 関数内で使用されるプライマリおよびセカンダリの文字列保持用構造体として機能します。

失敗モードと境界条件

  • メモリ不足時: IncreaseBuffer 内でメモリ割り当てに失敗した場合、PostgreSQL のメモリ管理機構によりエラーが発生します。
  • バッファ境界: GetAt や SetAt 等のアクセサ関数において、範囲外アクセスが発生しないよう呼び出し側で制御する必要があります。

保守担当者向けメモ

  • 本構造体は PostgreSQL のメモリコンテキスト（palloc 等）を利用して管理されています。
  • free_string_on_destroy フラグは、メモリの所有権が構造体にあるか外部にあるかを制御するために使用されます。
