table_slot_create

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableam.c:91

目的

指定されたリレーションの属性定義に基づき、適切なコールバックを備えた TupleTableSlot を作成します。

シグネチャまたは形状

TupleTableSlot *table_slot_create(Relation relation, List **reglist)

入力と出力

  • **入力**:
  • relation: スロット作成の基となるリレーション。
  • reglist: 作成したスロットを追跡するためのリストへのポインタ（NULL指定可能）。
  • **出力**:
  • 作成された TupleTableSlot へのポインタ。

前提条件と不変条件

  • relation は有効なリレーション記述子である必要があります。
  • table_slot_callbacks(relation) がそのリレーションに適した TupleTableSlotOps を返すことが前提となります。

アルゴリズムまたは振る舞い

1. table_slot_callbacks(relation) を呼び出し、リレーションの種類に応じたスロット操作関数（tts_cb）を取得します。 2. RelationGetDescr(relation) で取得したタプル記述子と、取得した tts_cb を用いて MakeSingleTupleTableSlot を実行し、スロットを生成します。 3. reglist が NULL でない場合、生成したスロットを lappend を用いてリストに追加します。 4. 生成したスロットを返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • table_slot_callbacks
  • MakeSingleTupleTableSlot
  • RelationGetDescr
  • lappend
  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/index/genam.c
  • src/backend/catalog/index.c
  • src/backend/commands/analyze.c
  • src/backend/commands/constraint.c
  • src/backend/commands/copyfrom.c
  • src/backend/access/table/tableam.c (内部再帰的呼び出し)

関連データ構造

  • TupleTableSlot: タプルを保持するための構造体。
  • TupleTableSlotOps: スロット操作のための関数ポインタ群。
  • Relation: テーブルのメタデータ構造。

失敗モードと境界条件

  • メモリ不足時には MakeSingleTupleTableSlot または lappend 内でエラーが発生し、処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は table_slot_callbacks を介して動的にスロットの挙動を決定するため、新しいアクセスメソッドを追加する際は table_slot_callbacks の実装を確認してください。
  • reglist を使用する場合、呼び出し側でリストのメモリ管理を行う必要があります。
