PQresultInstanceData

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/libpq-events.c:164

目的

PGresult オブジェクトに関連付けられた、特定のイベントプロシージャに対応するインスタンスデータを取得します。

シグネチャまたは形状

void *PQresultInstanceData(const PGresult *result, PGEventProc proc);

入力と出力

  • **入力**:
  • result: データを取得する対象の PGresult 構造体へのポインタ。
  • proc: データを識別するためのイベントプロシージャ関数ポインタ。
  • **出力**:
  • 成功時: 該当するイベントプロシージャに関連付けられたデータポインタ（void *）。
  • 失敗時: NULL。

前提条件と不変条件

  • result および proc が NULL でないこと。
  • イベントプロシージャは、事前に PQregisterEventProc 等を通じて登録されている必要があります。

アルゴリズムまたは振る舞い

1. result または proc が NULL の場合、直ちに NULL を返します。 2. result が保持するイベント配列 (result->events) を走査します。 3. 配列内の各エントリの proc が、引数で渡された proc と一致するか確認します。 4. 一致するエントリが見つかった場合、そのエントリに格納されている data ポインタを返します。 5. 配列の最後まで一致するものが見つからなかった場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内では、直接的な呼び出し元は特定されていません。
  • **呼び出し先**: 本関数は内部で他の関数を呼び出しません。

関連データ構造

  • PGresult: クエリ結果を保持する構造体。内部にイベント管理用の配列 events を持ちます。
  • PGEventProc: イベント処理関数の型定義。

失敗モードと境界条件

  • result が NULL の場合、安全に NULL を返します。
  • proc が NULL の場合、安全に NULL を返します。
  • 登録されていない proc を指定した場合、NULL を返します。

保守担当者向けメモ

  • 本関数は PGresult に紐づくデータへの読み取り専用アクセスを提供します。データの更新には PQresultSetInstanceData を使用してください。
  • イベント識別には関数ポインタのアドレスが使用されるため、同一の関数ポインタを複数回登録することはできません。
