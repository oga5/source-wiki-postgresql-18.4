X509_NAME_field_to_text

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:196

目的

X.509証明書の識別名（DN）から、指定されたフィールド名に対応する値を抽出し、PostgreSQLの text 型（Datum）に変換して返す。

シグネチャまたは形状

static Datum X509_NAME_field_to_text(X509_NAME *name, text *fieldName)

入力と出力

  • **入力**:
  • name: 検索対象の X509_NAME 構造体へのポインタ。
  • fieldName: 検索したいフィールド名を表すPostgreSQLの text 型。
  • **出力**:
  • 成功時: 抽出された値を変換した Datum。
  • フィールドが見つからない場合: (Datum) 0。

前提条件と不変条件

  • fieldName はOpenSSLが認識可能な有効なNID（Name ID）に変換できる文字列であること。
  • name は有効な X509_NAME 構造体であること。

アルゴリズムまたは振る舞い

1. text_to_cstring を使用して fieldName をC文字列に変換する。 2. OBJ_txt2nid を呼び出し、フィールド名をOpenSSLのNIDに変換する。変換失敗時は ereport(ERROR) でトランザクションを中断する。 3. X509_NAME_get_index_by_NID を使用して、指定されたNIDに対応するエントリのインデックスを検索する。 4. インデックスが負（見つからない）の場合は (Datum) 0 を返す。 5. インデックスが正の場合、X509_NAME_get_entry と X509_NAME_ENTRY_get_data を介してASN.1データを取得する。 6. ASN1_STRING_to_text を呼び出し、ASN.1データをPostgreSQLの text 型へ変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ssl_client_dn_field (contrib/sslinfo/sslinfo.c:246)
  • ssl_issuer_field (contrib/sslinfo/sslinfo.c:281)
  • **呼び出し先**:
  • text_to_cstring
  • OBJ_txt2nid
  • X509_NAME_get_index_by_NID
  • X509_NAME_get_entry
  • X509_NAME_ENTRY_get_data
  • ASN1_STRING_to_text

関連データ構造

  • X509_NAME: OpenSSLの証明書識別名構造体。
  • ASN1_STRING: OpenSSLのASN.1文字列データ構造。

失敗モードと境界条件

  • **エラー**: 指定された fieldName がOpenSSLで定義されていない場合、ERRCODE_INVALID_PARAMETER_VALUE を伴うエラーを報告する。
  • **境界条件**: 指定されたNIDが証明書内に存在しない場合は、エラーにはならず (Datum) 0 を返す。

保守担当者向けメモ

  • string_fieldname は pfree で解放されるが、ASN1_STRING_to_text の戻り値のメモリ管理は呼び出し先の規約に従う必要がある。
  • OpenSSLのNID検索は OBJ_txt2nid に依存しているため、将来的なOpenSSLの仕様変更やNIDの追加に影響を受ける可能性がある。
