remove_nulling_relids_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteManip.c:51

目的

クエリツリーの走査中に、特定の関係（リレーション）IDをNULL許容属性のビットマップセットから除去するためのコンテキスト情報を保持します。

シグネチャまたは形状

typedef struct
{
	const Bitmapset *removable_relids;
	const Bitmapset *except_relids;
	int			sublevels_up;
} remove_nulling_relids_context;

入力と出力

  • removable_relids: 除去対象とするリレーションIDのビットマップセット。
  • except_relids: 除去から除外するリレーションIDのビットマップセット。
  • sublevels_up: 現在のネストレベル（サブクエリの深さ）。

前提条件と不変条件

  • sublevels_up は、ツリー走査の深さに応じて適切にインクリメントおよびデクリメントされる必要があります。
  • removable_relids および except_relids は、走査中に変更されない定数として扱われます。

アルゴリズムまたは振る舞い

本構造体は、expression_tree_walker や query_tree_walker を用いた走査において、現在の走査状態を保持するために使用されます。走査関数は、このコンテキストを参照しながら、クエリ内のノードに含まれるNULL許容リレーションIDを特定し、指定された条件に基づいてビットマップから除去を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/rewrite/rewriteManip.c 内の関数（1343行目、1356行目付近で参照を確認）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、具体的な関数呼び出しの詳細はソース抜粋からは不明です。

関連データ構造

  • Bitmapset: リレーションIDの集合を表現するために使用されます。
  • add_nulling_relids_context: 対照的な操作を行うための関連構造体。

失敗モードと境界条件

  • sublevels_up の管理が不適切な場合、誤ったネストレベルのノードに対して操作が行われる可能性があります。
  • removable_relids に含まれるIDが、except_relids にも含まれている場合の優先順位については、この構造体単体では規定されていません。

保守担当者向けメモ

  • 本構造体は rewriteManip.c 内の走査ロジックと密接に関連しています。変更を加える際は、sublevels_up の管理ルール（サブクエリ突入時のインクリメント等）が維持されているか確認してください。
