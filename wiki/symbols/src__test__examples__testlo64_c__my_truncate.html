my_truncate

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlo64.c:151

目的

PostgreSQL のラージオブジェクトを指定されたサイズに切り詰める（truncate）ためのラッパー関数です。

シグネチャまたは形状

static void my_truncate(PGconn *conn, Oid lobjId, int64_t len)

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • lobjId: 操作対象のラージオブジェクトの Oid。
  • len: 切り詰め後のサイズ（int64_t）。
  • **出力**: なし（void）。

前提条件と不変条件

  • conn は有効なデータベース接続である必要があります。
  • lo_open が成功し、読み書き権限（INV_READ | INV_WRITE）でファイル記述子が取得できる必要があります。

アルゴリズムまたは振る舞い

1. lo_open を使用して、指定された lobjId を読み書きモードで開きます。 2. 開いたファイル記述子が負の値（エラー）の場合、標準エラー出力にメッセージを出力します。 3. lo_truncate64 を呼び出し、ラージオブジェクトを指定された len に切り詰めます。 4. lo_truncate64 が失敗した場合は、PQerrorMessage を用いてエラー内容を標準エラー出力に出力します。 5. 最後に lo_close を呼び出し、ファイル記述子を閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/examples/testlo64.c:291 にて呼び出されています。
  • **呼び出し先**:
  • lo_open
  • lo_truncate64
  • lo_close
  • fprintf
  • PQerrorMessage

関連データ構造

  • PGconn: PostgreSQL 接続オブジェクト。
  • Oid: ラージオブジェクト識別子。

失敗モードと境界条件

  • lo_open が失敗した場合、エラーメッセージが表示されますが、関数はそのまま処理を継続しようとします。
  • lo_truncate64 が失敗した場合、エラーメッセージが表示されます。
  • 4GB を超えるオフセットを扱う設計となっており、64 ビット API を使用しています。

保守担当者向けメモ

  • 本関数は static であり、testlo64.c 内でのみ使用されます。
  • エラー発生時にプログラムを終了させる処理は含まれていないため、呼び出し側で必要に応じて状態を確認してください。
