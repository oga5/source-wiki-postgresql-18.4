hstoreValidNewFormat

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_compat.c:118

目的

与えられた HStore データが新形式（hstore-new）として妥当であるかを検証し、その適合度を判定します。

シグネチャまたは形状

static int hstoreValidNewFormat(HStore *hs)

入力と出力

  • **入力**: HStore *hs （検証対象の hstore データ構造へのポインタ）
  • **出力**: int 型のステータスコード
  • 2: 新形式として完全に妥当。
  • 1: 形式は新形式の構造に従っているが、データサイズに不一致がある。
  • 0: 新形式として無効。

前提条件と不変条件

  • HStore 構造体のエントリサイズが旧形式と新形式で同一である必要があります。
  • entries 配列内の各エントリの終了位置（HSE_ENDPOS）は、単調増加である必要があります。
  • キーの長さは単調増加であり、かつキー自体が NULL であってはなりません。

アルゴリズムまたは振る舞い

1. HS_FLAG_NEWVERSION フラグが立っている場合、即座に 2 を返します。 2. 要素数が 0 の場合、2 を返します。 3. 最初の要素が HSE_ISFIRST フラグを満たさない場合、0 を返します。 4. 計算されたデータサイズ（vsize）が実際のデータサイズ（VARSIZE(hs)）を超える場合、0 を返します。 5. 全エントリを走査し、以下の条件を検証します：

  • 各エントリの終了位置が前のエントリより小さくないこと。
  • キーの長さが前のキーより小さくないこと。
  • キーが NULL でないこと。

    • 6. 全ての検証を通過し、かつ vsize が VARSIZE(hs) と一致すれば 2 を、一致しなければ 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_compat.c 内の関数（hstoreUpgrade 等）から呼び出されます。
  • **呼び出し先**:
  • HS_COUNT, ARRPTR, HSE_ENDPOS, CALCDATASIZE, VARSIZE, HSE_ISFIRST, HSTORE_KEYLEN, HSE_ISNULL などの hstore.h で定義されたマクロを使用します。

関連データ構造

  • HStore: 検証対象のメイン構造体。
  • HEntry: hstore 内の各要素を管理する構造体。

失敗モードと境界条件

  • count == 0 の場合は妥当とみなされます。
  • エントリの順序やキーの長さが単調増加でない場合、またはキーが NULL の場合は 0（無効）を返します。
  • 実際のデータサイズと計算上のサイズが一致しない場合は 1 を返します。

保守担当者向けメモ

  • 本関数は hstore のアップグレードプロセスにおける検証ゲートウェイです。
  • 非常に稀なケースとして、旧形式と新形式の両方として解釈可能なデータが存在する場合、この関数の判定ロジックが重要となります。
  • HSE_ISFIRST や HSTORE_KEYLEN などのマクロの挙動は hstore.h に依存しているため、構造体定義の変更時には注意が必要です。
