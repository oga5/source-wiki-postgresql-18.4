dlist_iter

定義

  • 種別: struct
  • 定義箇所: src/include/lib/ilist.h:177

目的

二重連結リスト（dlist）を走査するためのイテレータ構造体です。リストの現在のノードと走査終了地点を保持し、リストの順次アクセスをサポートします。

シグネチャまたは形状

typedef struct dlist_iter
{
    dlist_node *cur;    /* 現在の要素 */
    dlist_node *end;    /* 走査終了地点のノード */
} dlist_iter;

入力と出力

  • **入力**: dlist_iter はスタック上に確保され、dlist_foreach などのマクロを通じてリストの head から初期化されます。
  • **出力**: 走査の過程で cur ポインタが更新され、リスト内の各 dlist_node を順次参照します。

前提条件と不変条件

  • cur はリスト内の有効なノード、または走査終了を示すポインタを指している必要があります。
  • 走査中にリスト構造が変更される場合、この構造体ではなく dlist_mutable_iter を使用する必要があります。

アルゴリズムまたは振る舞い

  • dlist_foreach マクロと組み合わせて使用されます。
  • 初期化時に cur に開始ノード、end に走査を停止する境界ノードが設定されます。
  • 走査の各ステップで cur が次のノードへ更新され、cur == end となった時点で走査が終了します。

呼び出し元と呼び出し先

  • **呼び出し元（確認された参照先）**:
  • src/backend/access/gin/gindatapage.c
  • src/backend/access/transam/multixact.c
  • src/backend/access/transam/parallel.c
  • src/backend/postmaster/autovacuum.c
  • **呼び出し先**: 構造体定義のため、直接的な関数呼び出しは行いません。

関連データ構造

  • dlist_node: リストのリンク要素。
  • dlist_head: リストの起点。
  • dlist_mutable_iter: 走査中の安全な削除を可能にする代替イテレータ。

失敗モードと境界条件

  • 空のリストに対して走査を開始した場合、即座に終了条件が満たされる必要があります。
  • 走査中にリストのノードが外部から削除された場合、cur が無効なメモリを指す可能性があるため、注意が必要です。

保守担当者向けメモ

  • 本構造体は走査専用です。走査中にノードの削除や挿入を行う場合は、dlist_mutable_iter を使用するようにコードをレビューしてください。
  • ILIST_DEBUG を有効にすると、リスト操作時の整合性チェックが強化されます。
