ResetReindexProcessing

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/index.c:4169

目的

現在実行中の再構築（REINDEX）対象であるヒープおよびインデックスの追跡状態をリセットします。

シグネチャまたは形状

static void ResetReindexProcessing(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • reindexingNestLevel は、サブトランザクション終了まで保持されるため、本関数ではリセットされません。

アルゴリズムまたは振る舞い

1. グローバル変数 currentlyReindexedHeap を InvalidOid に設定します。 2. グローバル変数 currentlyReindexedIndex を InvalidOid に設定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/index.c 内の複数箇所から呼び出されています。
  • 呼び出し先: なし（直接的な関数呼び出しは含まれていません）。

関連データ構造

  • currentlyReindexedHeap (Oid)
  • currentlyReindexedIndex (Oid)
  • reindexingNestLevel (関連する状態変数)

失敗モードと境界条件

  • 本関数自体はエラーを発生させませんが、呼び出しタイミングがトランザクションの整合性に依存します。

保守担当者向けメモ

  • 本関数は再構築処理の終了時に状態をクリーンアップするために使用されます。
  • reindexingNestLevel がリセットされない点に注意してください。これはトランザクションのネスト構造を維持するためです。
