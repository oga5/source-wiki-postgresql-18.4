lastcold

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:584

目的

正規表現のマッチング処理において、入力文字列中で「進捗がなかった（NOPROGRESS）」状態が最後に観測された位置を特定します。

シグネチャまたは形状

chr *lastcold(struct vars *v, struct dfa *d)

入力と出力

  • **入力**:
  • v: 現在の正規表現実行コンテキスト（struct vars）。
  • d: DFA 実行状態を保持する構造体（struct dfa）。
  • **出力**:
  • chr *: 進捗がなかった最後の位置を示すポインタ。該当がない場合は v->start または d->lastnopr を返します。

前提条件と不変条件

  • d->ssets が有効な DFA 状態セットの配列を指していること。
  • d->nssused がキャッシュ内の有効な状態セット数を示していること。

アルゴリズムまたは振る舞い

1. 初期値として、DFA 構造体が保持する lastnopr を取得します。これが未設定（NULL）の場合は、実行コンテキストの開始位置 v->start を初期値とします。 2. DFA の全状態セット（d->ssets）を走査します。 3. 各状態セットにおいて NOPROGRESS フラグが立っている場合、その状態が最後に観測された位置（ss->lastseen）を確認します。 4. ss->lastseen が現在の nopr よりも後ろの位置であれば、nopr を更新します。 5. 最終的に特定された最も後ろの位置を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/rege_dfa.c:334
  • **呼び出し先**:
  • 直接的な関数呼び出しは含まれていません。

関連データ構造

  • struct dfa: DFA の実行状態とキャッシュを管理する構造体。
  • struct sset: DFA の状態セット（キャッシュエントリ）。NOPROGRESS フラグと lastseen メンバを保持します。
  • struct vars: 正規表現実行の全体的な変数を保持する構造体。

失敗モードと境界条件

  • d->lastnopr が NULL かつ v->start が有効な場合、開始位置が基準となります。
  • NOPROGRESS フラグを持つ状態セットが存在しない場合、初期値がそのまま返されます。

保守担当者向けメモ

  • 本関数は、マッチングの進捗が停滞した箇所を特定することで、バックトラッキングや再試行の最適化に寄与します。
  • sset の走査は d->nssused に依存するため、キャッシュの整合性が保たれていることが前提となります。
