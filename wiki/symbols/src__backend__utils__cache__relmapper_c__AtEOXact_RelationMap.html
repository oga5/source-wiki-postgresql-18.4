AtEOXact_RelationMap

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relmapper.c:540

目的

トランザクション終了時（コミットまたはアボート）に、リレーションマッピングの更新状態を確定または破棄する。

シグネチャまたは形状

void AtEOXact_RelationMap(bool isCommit, bool isParallelWorker)

入力と出力

  • isCommit: トランザクションがコミットされる場合は true、アボートされる場合は false。
  • isParallelWorker: 並列ワーカーの終了処理である場合は true。
  • 出力: なし（副作用としてマップファイルへの書き込みまたはメモリ上の更新状態のリセットを行う）。

前提条件と不変条件

  • コミット時、pending_shared_updates および pending_local_updates にマッピングが存在してはならない（num_mappings == 0 であること）。
  • 並列ワーカーの場合、pending 状態の更新は存在してはならない。

アルゴリズムまたは振る舞い

1. **コミット時（かつ並列ワーカーでない場合）**:

  • active_shared_updates および active_local_updates に保持されている更新内容を perform_relmap_update を呼び出して物理マップファイルへ書き出す。
  • 書き出し後、各更新構造体の num_mappings を 0 にリセットする。

    • 2. **アボート時または並列ワーカーの場合**:

  • active_ および pending_ の各更新構造体の num_mappings を 0 にリセットし、すべての更新予約を破棄する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c（トランザクション終了処理の一部として呼び出される）。
  • 呼び出し先: perform_relmap_update（コミット時のファイル書き出し用）。

関連データ構造

  • active_shared_updates, active_local_updates: トランザクション内で有効化された更新内容を保持する構造体。
  • pending_shared_updates, pending_local_updates: 保留中の更新内容を保持する構造体。

失敗モードと境界条件

  • コミット時に pending 更新が存在する場合、アサーション違反により停止する。
  • マップファイルの書き込みに失敗した場合、perform_relmap_update 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はトランザクションのコミット/アボートの境界で呼び出されるため、マッピングの整合性を維持する重要な役割を担う。
  • 並列ワーカーでの実行時は、状態の更新を行わず破棄のみを行う設計となっている。
