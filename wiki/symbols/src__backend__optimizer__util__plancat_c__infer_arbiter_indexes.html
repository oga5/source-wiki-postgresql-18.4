infer_arbiter_indexes

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/plancat.c:712

目的

ON CONFLICT 句で指定された競合解決条件（arbiter）に合致するユニークインデックスの OID リストを特定します。

シグネチャまたは形状

List *infer_arbiter_indexes(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（クエリのプランニング情報。root->parse->onConflict を参照）
  • **出力**: List *（合致したインデックスの OID リスト。該当なしの場合は NIL）

前提条件と不変条件

  • 対象リレーションは既にロック済みであること（本関数内ではロックを取得しない）。
  • ON CONFLICT 句にインデックス推論仕様または名前付き制約が含まれていること。
  • 全行（whole-row）のユニークインデックス推論はサポート外であり、エラーを発生させる。

アルゴリズムまたは振る舞い

1. **初期チェック**: onConflict 構造体から推論要素（arbiterElems）と制約（constraint）を確認し、指定がない場合は NIL を返す。 2. **準備**: 対象リレーションを開き、推論対象の属性を Bitmapset 形式に正規化する。 3. **制約指定時の処理**: constraint が指定されている場合、get_constraint_index で対応するインデックスを特定する。 4. **インデックス走査**: リレーションの全インデックスを反復し、以下を判定する：

  • インデックスが有効（indisvalid）であること。
  • 名前付き制約の場合は、その OID と一致するかを確認する。
  • 通常の推論の場合は、ユニークインデックスであり、かつ除外制約（indisexclusion）ではないこと。
  • インデックスの属性構成が推論条件と一致すること。

    • 5. **式インデックスの対応**: インデックスに式が含まれる場合、ChangeVarNodes を用いて現在のクエリの varno に適合させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/createplan.c:7242
  • **呼び出し先**: table_open, rt_fetch, get_constraint_index, RelationGetIndexList, index_open, RelationGetIndexExpressions, ChangeVarNodes, index_close, table_close

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • OnConflictExpr: ON CONFLICT 句の解析結果。
  • InferenceElem: 推論対象の式または属性。
  • Form_pg_index: インデックスのカタログ情報。

失敗モードと境界条件

  • **エラー**:
  • 全行（whole-row）の推論指定時。
  • 指定された制約に対応するインデックスが見つからない場合。
  • ON CONFLICT DO UPDATE と除外制約（exclusion constraint）の組み合わせ。
  • **境界条件**:
  • ON CONFLICT 指定がない場合、即座に NIL を返す。
  • インデックスが indisvalid でない場合はスキップされる。

保守担当者向けメモ

  • 本関数は ON CONFLICT 句の解決に特化しており、インデックスのロックモードは実行時に使用されるものと同一のものを取得する。
  • ChangeVarNodes による式変換は、インデックスの述語や式を現在のクエリコンテキストで評価するために不可欠である。
