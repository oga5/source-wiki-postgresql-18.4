cancel_before_shmem_exit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipc.c:400

目的

プロセス終了処理の before_shmem_exit リストに登録された直近のコールバックを削除します。

シグネチャまたは形状

void cancel_before_shmem_exit(pg_on_exit_callback function, Datum arg)

入力と出力

  • function: 削除対象のコールバック関数ポインタ。
  • arg: 削除対象のコールバックに渡される引数。
  • 出力: なし（成功時）。失敗時はエラーを発生させます。

前提条件と不変条件

  • before_shmem_exit_list の末尾（before_shmem_exit_index - 1）に、指定された function と arg が存在している必要があります。

アルゴリズムまたは振る舞い

1. before_shmem_exit_index が 0 より大きいことを確認します。 2. リストの末尾にあるコールバックが、引数で指定された function および arg と一致するか比較します。 3. 一致する場合、before_shmem_exit_index をデクリメントしてリストから削除します。 4. 一致しない場合、elog(ERROR) を呼び出し、処理を中断します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/storage/ipc.h で定義されたマクロ（PG_ENSURE_ERROR_CLEANUP 等）から呼び出されることが想定されています。
  • 呼び出し先: elog（エラー発生時）。

関連データ構造

  • before_shmem_exit_list: 終了処理用コールバックを保持する静的配列。
  • before_shmem_exit_index: リストの現在の要素数を指すインデックス。

失敗モードと境界条件

  • 指定されたコールバックがリストの末尾にない場合、elog(ERROR) によりプロセスが異常終了します。
  • リストが空（before_shmem_exit_index == 0）の場合、同様にエラーとなります。

保守担当者向けメモ

  • 本関数は LIFO（後入れ先出し）順序で管理されるコールバックリストの末尾のみを操作する設計です。
  • 呼び出し元が意図したコールバックを正しく指定していることを前提としています。
