BipartiteMatch

定義

  • 種別: function
  • 定義箇所: src/backend/lib/bipartite_match.c:38

目的

Hopcroft-Karp アルゴリズムを用いて、二部グラフにおける最大マッチングを計算します。

シグネチャまたは形状

BipartiteMatchState *BipartiteMatch(int u_size, int v_size, short **adjacency)

入力と出力

  • **入力**:
  • u_size, v_size: 二部グラフの各集合のノード数。
  • adjacency: 隣接リスト。adjacency[u] の先頭要素が隣接ノード数 k、以降がノード番号である必要があります。
  • **出力**:
  • 計算結果と状態を保持する BipartiteMatchState 構造体へのポインタ。

前提条件と不変条件

  • ノード数 u_size, v_size は 0 以上かつ SHRT_MAX 未満である必要があります。
  • adjacency リストの所有権は呼び出し元にあり、本関数はこれを解放しません。

アルゴリズムまたは振る舞い

1. BipartiteMatchState 用のメモリを確保し、マッチング状態を初期化します。 2. hk_breadth_search を呼び出し、増大路の距離を計算します。 3. hk_breadth_search が成功する限り、未マッチのノードに対して hk_depth_search を再帰的に呼び出し、増大路を特定してマッチングを更新します。 4. 探索の各反復で CHECK_FOR_INTERRUPTS() を実行し、安全な実行を保証します。 5. 増大路が見つからなくなった時点で終了し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c:3299
  • **呼び出し先**:
  • hk_breadth_search
  • hk_depth_search
  • palloc / palloc0
  • elog (エラー時)
  • CHECK_FOR_INTERRUPTS

関連データ構造

  • BipartiteMatchState: 計算状態、マッチングペア (pair_uv, pair_vu)、距離情報、探索用キューを保持します。

失敗モードと境界条件

  • u_size または v_size が負の値、あるいは SHRT_MAX 以上の場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数が確保したメモリは BipartiteMatchFree で解放する必要があります。
  • distance 配列の初期値として SHRT_MAX を「無限大」として使用しているため、ノード数の上限制約は厳守してください。
