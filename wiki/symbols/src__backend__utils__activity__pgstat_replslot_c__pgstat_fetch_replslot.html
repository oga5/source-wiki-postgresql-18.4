pgstat_fetch_replslot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_replslot.c:169

目的

指定された名前のレプリケーションスロットに対応する統計情報エントリを取得します。

シグネチャまたは形状

PgStat_StatReplSlotEntry *pgstat_fetch_replslot(NameData slotname)

入力と出力

  • **入力**: slotname (取得対象のレプリケーションスロット名)
  • **出力**: 該当するスロットの統計情報構造体 PgStat_StatReplSlotEntry へのポインタ。スロットが見つからない場合は NULL を返します。

前提条件と不変条件

  • ReplicationSlotControlLock を共有モードで取得して操作を行います。
  • スロットのインデックス解決には get_replslot_index を使用します。

アルゴリズムまたは振る舞い

1. ReplicationSlotControlLock を共有ロックとして取得します。 2. get_replslot_index を呼び出し、スロット名から対応するインデックス idx を特定します。 3. idx が有効（-1 以外）であれば、pgstat_fetch_entry を呼び出し、PGSTAT_KIND_REPLSLOT 種別の統計エントリを取得します。 4. ReplicationSlotControlLock を解放します。 5. 取得したエントリへのポインタ、または見つからなかった場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c:2149
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease (ストレージ管理)
  • get_replslot_index (インデックス解決)
  • pgstat_fetch_entry (統計エントリ取得)

関連データ構造

  • PgStat_StatReplSlotEntry: レプリケーションスロットの統計情報を保持する構造体。
  • NameData: スロット名を格納する型。

失敗モードと境界条件

  • 指定された名前のスロットが存在しない場合、get_replslot_index が -1 を返し、関数は NULL を返します。

保守担当者向けメモ

  • 本関数はスロット名から統計情報を検索するインターフェースです。スロットのインデックスは再起動時に変更される可能性があるため、永続化された名前をキーとして使用する設計になっています。
