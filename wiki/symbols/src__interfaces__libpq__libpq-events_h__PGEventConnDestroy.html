PGEventConnDestroy

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-events.h:47

目的

PGconn オブジェクトが破棄される際に、登録されたイベントハンドラへ通知を行うためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	PGconn	   *conn;
} PGEventConnDestroy;

入力と出力

  • **入力**: conn は破棄対象となる PGconn オブジェクトへのポインタです。
  • **出力**: なし（本構造体はイベントハンドラへの引数として渡されます）。

前提条件と不変条件

  • 本構造体は PGEVT_CONNDESTROY イベントが発生した際にハンドラへ渡されます。
  • ハンドラは、このイベントを受け取った際に、当該接続に関連付けていたカスタムデータを解放する責任を負います。

アルゴリズムまたは振る舞い

1. libpq 内部で接続の破棄処理が開始されます。 2. 登録済みの各イベントハンドラに対し、PGEVT_CONNDESTROY イベントが発行されます。 3. ハンドラは PGEventConnDestroy 構造体を通じて対象の PGconn を受け取り、関連付けられたリソースのクリーンアップを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-connect.c 内で本構造体がインスタンス化されていることが確認されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。
  • PGEventId: イベントの種類を識別する列挙型（PGEVT_CONNDESTROY を含む）。
  • PGEventProc: イベントハンドラの関数ポインタ型。

失敗モードと境界条件

  • イベントハンドラ内でリソースの解放を怠った場合、メモリリークが発生します。
  • 接続が既に無効化されている場合でも、ハンドラはクリーンアップ処理を適切に完了させる必要があります。

保守担当者向けメモ

  • 本構造体は libpq のイベント API の一部です。イベントハンドラを実装する際は、PGEVT_CONNDESTROY イベントを適切に処理し、メモリ管理を確実に行ってください。
