init_tsvector_parser

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_parser.c:56

目的

全文検索（tsvector および tsquery）の入力文字列を解析するための状態管理オブジェクト（TSVectorParseState）を生成・初期化します。

シグネチャまたは形状

TSVectorParseState init_tsvector_parser(char *input, int flags, Node *escontext);

入力と出力

  • **入力**:
  • input: 解析対象の文字列。
  • flags: パーサーの挙動を制御するビットフラグ（P_TSV_OPR_IS_DELIM, P_TSV_IS_TSQUERY, P_TSV_IS_WEB）。
  • escontext: エラーハンドリング用のコンテキスト（Node*）。
  • **出力**:
  • 初期化済みの TSVectorParseState 構造体へのポインタ。

前提条件と不変条件

  • palloc を使用してメモリを確保するため、PostgreSQL のメモリコンテキストが有効である必要があります。
  • input 文字列は解析終了まで有効である必要があります。

アルゴリズムまたは振る舞い

1. TSVectorParseStateData 構造体分のメモリを割り当てます。 2. 入力文字列ポインタ、解析用バッファ（初期サイズ32バイト）、およびデータベースの最大文字長を状態に格納します。 3. flags に基づき、演算子の扱い（oprisdelim）、対象型（is_tsquery）、Web 検索モード（is_web）をフラグとして設定します。 4. エラー処理用の escontext を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery.c:859
  • src/backend/utils/adt/tsvector.c:202
  • **呼び出し先**:
  • palloc (メモリ確保)
  • pg_database_encoding_max_length (エンコーディング情報の取得)

関連データ構造

  • TSVectorParseStateData: 本関数が初期化する解析状態保持構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • flags の指定が不適切な場合、後続のトークン解析（gettoken_tsvector）で意図しない構文解釈が行われる可能性があります。

保守担当者向けメモ

  • 本関数は tsvector と tsquery の両方で共有されています。新しいフラグを追加する場合は、ts_utils.h の定義と整合性を取ってください。
  • prsbuf は入力文字列を直接参照するため、呼び出し元で入力文字列を解放しないよう注意してください。
