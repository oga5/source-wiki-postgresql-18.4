outBitmapset

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/outfuncs.c:330

目的

Bitmapset 構造体の内容を、デバッグやシリアライズに適したテキスト形式（(b <整数値> ...)）に変換し、指定された StringInfo バッファへ書き出します。

シグネチャまたは形状

void outBitmapset(StringInfo str, const Bitmapset *bms)

入力と出力

  • **入力**:
  • str: 書き込み先の StringInfo バッファ。
  • bms: シリアライズ対象の Bitmapset 構造体へのポインタ。
  • **出力**: なし（str が指すバッファの内容が更新されます）。

前提条件と不変条件

  • str は初期化済みの有効な StringInfo である必要があります。
  • bms が NULL の場合、bms_next_member の挙動に依存します（通常、bms は有効なポインタであることが想定されます）。

アルゴリズムまたは振る舞い

1. バッファ str に開始文字 '(' と識別子 'b' を追加します。 2. bms_next_member を用いて、Bitmapset 内のすべてのセット済みビット位置を順次走査します。 3. 各ビット位置（整数値）を " %d" の形式でバッファに追記します。 4. 最後に終了文字 ')' を追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/outfuncs.c 内の各所（ノードのシリアライズ処理中）。
  • bmsToString 関数などから参照されています。
  • **呼び出し先**:
  • appendStringInfoChar
  • appendStringInfo
  • bms_next_member

関連データ構造

  • Bitmapset: ビット集合を保持する構造体。
  • StringInfo: 文字列構築用のバッファ構造体。

失敗モードと境界条件

  • bms が空の集合である場合、(b ) という形式で出力されます。
  • メモリ不足が発生した場合、appendStringInfo 経由でエラーが報告される可能性があります。

保守担当者向けメモ

  • 本関数は readfuncs.c 側の対応する読み込み関数と形式を一致させる必要があります。出力形式を変更する場合は、復元側のロジックも更新してください。
