SignTSVector

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsgistidx.c:63

目的

PostgreSQL の tsvector 型に対する GiST インデックスにおいて、インデックスキーを格納するための可変長構造体です。

シグネチャまたは形状

typedef struct
{
    int32       vl_len_;        /* varlena header */
    int32       flag;           /* ARRKEY, SIGNKEY, または ALLISTRUE */
    char        data[FLEXIBLE_ARRAY_MEMBER];
} SignTSVector;

入力と出力

  • **入力**: tsvector の内容を圧縮・変換したデータ。
  • **出力**: GiST インデックスのリーフまたは内部ノードに格納されるバイナリ形式のキー。

前提条件と不変条件

  • vl_len_ は varlena ヘッダーであり、直接操作してはなりません。
  • flag フィールドにより、data 部がハッシュ値の配列（ARRKEY）か、ビットベクトル（SIGNKEY）か、あるいは全真（ALLISTRUE）かを識別します。
  • data は FLEXIBLE_ARRAY_MEMBER を使用しており、flag に応じた可変長サイズを持ちます。

アルゴリズムまたは振る舞い

  • tsvector の各単語をハッシュ化し、そのサイズやインデックスの構成に応じて ARRKEY（配列）または SIGNKEY（ビットベクトル）として保持します。
  • gtsvector_compress 等の関数により、インデックス構築時にこの構造体へ変換されます。
  • 検索時には ISARRKEY, ISSIGNKEY, ISALLTRUE マクロを用いて flag を判定し、適切な照合ロジックへ分岐します。

呼び出し元と呼び出し先

  • **呼び出し元**: gtsvector_compress, gtsvector_alloc などの tsgistidx.c 内の関数がこの構造体を生成・操作します。
  • **呼び出し先**: checkcondition_arr や checkcondition_bit が、この構造体の data を参照して検索条件の適合性を判定します。

関連データ構造

  • GistTsVectorOptions: インデックス作成時のオプション。
  • BITVECP: SIGNKEY 時に使用されるビットベクトルへのポインタ型。

失敗モードと境界条件

  • siglen（シグネチャ長）の設定値により、SIGNKEY のサイズが決定されます。
  • インデックスは「不正確（inexact）」であるため、検索結果には必ずヒープ上の実データに対する再チェックが必要です。

保守担当者向けメモ

  • 本構造体は varlena 型として扱われるため、メモリ割り当てには palloc を使用し、必要に応じて repalloc でサイズを調整してください。
  • flag のビット定義（ARRKEY, SIGNKEY, ALLISTRUE）を変更または追加する場合は、関連するすべてのマクロおよび checkcondition_* 関数を更新する必要があります。
