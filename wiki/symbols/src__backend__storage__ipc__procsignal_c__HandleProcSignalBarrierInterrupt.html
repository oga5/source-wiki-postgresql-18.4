HandleProcSignalBarrierInterrupt

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procsignal.c:483

目的

プロセスがグローバルバリア割り込みを受信したことを記録し、後続の処理で割り込みが処理されるようにフラグを設定します。

シグネチャまたは形状

static void HandleProcSignalBarrierInterrupt(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • 本関数は SIGUSR1 シグナルハンドラ（procsignal_sigusr1_handler）のコンテキストから呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. InterruptPending を true に設定し、割り込み処理が必要であることをシステムに通知します。 2. ProcSignalBarrierPending を true に設定し、バリア処理が保留中であることを記録します。 3. ラッチ（latch）の設定は、本関数を呼び出す procsignal_sigusr1_handler 側で行われます。

呼び出し元と呼び出し先

  • 呼び出し元: procsignal_sigusr1_handler（src/backend/storage/ipc/procsignal.c 内）
  • 呼び出し先: なし

関連データ構造

  • InterruptPending（グローバル変数）
  • ProcSignalBarrierPending（グローバル変数）

失敗モードと境界条件

  • 本関数自体は単純なフラグ設定のみを行うため、失敗モードはありません。
  • 割り込みが連続して発生した場合、フラグは単に true に再設定されます。

保守担当者向けメモ

  • 本関数は割り込みハンドラ内から呼ばれるため、非同期シグナル安全（async-signal-safe）である必要があります。フラグの更新のみに留め、複雑な処理は行わないでください。
