sqlvar_struct

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/include/sqlda-native.h:24

目的

ECPG（Embedded SQL for C）において、動的SQL実行時の個々のカラム属性（型、データ長、値へのポインタ、NULLインジケータ）を保持するために使用されます。

シグネチャまたは形状

struct sqlvar_struct
{
    short       sqltype;    /* データ型 */
    short       sqllen;     /* データ長 */
    char       *sqldata;    /* データ値へのポインタ */
    short      *sqlind;     /* NULLインジケータへのポインタ */
    struct sqlname sqlname; /* カラム名等の識別子 */
};

入力と出力

  • **入力**: ECPGランタイムライブラリにより、クエリ結果やバインドパラメータのメタデータが各フィールドにマッピングされます。
  • **出力**: アプリケーションは本構造体を介して、データベースから取得した値や、送信するパラメータ値にアクセスします。

前提条件と不変条件

  • sqlvar[1] として sqlda_struct 内で配列の先頭として定義されますが、実際には sqld（カラム数）に基づいたメモリ確保が必要です。
  • sqlname は NAMEDATALEN（64バイト）の制限に従う必要があります。

アルゴリズムまたは振る舞い

  • 本構造体はデータ保持に特化しており、直接的な実行ロジックは持ちません。
  • メモリ確保時には、sizeof(struct sqlda_struct) + (sqld - 1) * sizeof(struct sqlvar_struct) の計算式を用いて、可変長部分を含めたサイズが算出されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/sqlda.c 内の関数が、SQLDAのメモリ確保や初期化時に本構造体のサイズを参照しています。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • struct sqlda_struct: 本構造体を配列として保持するメインの記述子領域。
  • struct sqlname: カラム名やテーブル名を格納する構造体。
  • sqlvar_t: sqlda.h 等で定義される sqlvar_struct の別名（typedef）。

失敗モードと境界条件

  • sqld の値と実際のメモリ確保サイズが不一致の場合、バッファオーバーフローのリスクがあります。
  • NAMEDATALEN を超える識別子は切り詰められる可能性があります。

保守担当者向けメモ

  • 本構造体はECPGネイティブ形式です。Informix互換の sqlda-compat.h と混同しないよう注意してください。
  • sqlvar[1] はC言語の慣用句であり、構造体末尾の可変長領域を指します。メモリ操作時は必ず sqld を基準としたサイズ計算を行ってください。
