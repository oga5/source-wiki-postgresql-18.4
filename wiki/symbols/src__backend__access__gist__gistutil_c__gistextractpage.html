gistextractpage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistutil.c:94

目的

GiSTインデックスページからすべてのインデックスタプルを抽出し、配列として返します。

シグネチャまたは形状

IndexTuple *gistextractpage(Page page, int *len /* out */ )

入力と出力

  • **入力**:
  • page: 抽出対象となるGiSTインデックスページ。
  • **出力**:
  • len: 抽出されたタプルの総数が格納されるポインタ。
  • 戻り値: 抽出された IndexTuple の配列（palloc によりメモリ確保）。

前提条件と不変条件

  • page は有効なGiSTインデックスページである必要があります。
  • 戻り値の配列は呼び出し元で解放（pfree）する必要があります。

アルゴリズムまたは振る舞い

1. PageGetMaxOffsetNumber を使用して、ページ内の最大オフセット番号を取得し、出力引数 len に設定します。 2. 抽出するタプル数分（maxoff）の IndexTuple 配列をメモリ上に確保します。 3. FirstOffsetNumber から maxoff までループし、PageGetItem を用いて各オフセットのタプルを配列にコピーします。 4. 構築した配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c:304
  • src/backend/access/gist/gistbuild.c:510
  • src/backend/access/gist/gistbuild.c:517
  • **呼び出し先**:
  • PageGetMaxOffsetNumber
  • palloc
  • PageGetItem
  • PageGetItemId
  • OffsetNumberNext

関連データ構造

  • IndexTuple: GiSTインデックス内に格納されるタプルの型。
  • Page: PostgreSQLのインデックスページ構造。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗し、エラーが発生する可能性があります。
  • 空のページ: maxoff が 0 の場合、len に 0 が設定され、空の配列が返されます。

保守担当者向けメモ

  • 本関数はページ内の全タプルをメモリにコピーするため、非常に大きなページやタプルを扱う際にはメモリ消費量に注意してください。
  • 抽出されたタプルはページ内の物理的な順序に従って格納されます。
