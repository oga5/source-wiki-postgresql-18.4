r_standard_suffix

定義

  • **種別**: function
  • **定義箇所**: src/backend/snowball/libstemmer/stem_UTF_8_italian.c:752

目的

イタリア語のステミング処理において、単語末尾の標準的な接尾辞を識別し、ルールに基づいて削除または置換を行う。

シグネチャまたは形状

static int r_standard_suffix(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（Snowball ステミング環境構造体）
  • **出力**: int（成功時は 1、失敗時は 0 または負のステータスコード）

前提条件と不変条件

  • z->c が単語の末尾を指していること。
  • find_among_b を使用して、定義済みの接尾辞配列 a_6 から後方検索を行う。
  • 多くのケースで、接尾辞の削除や置換には r_R2（R2領域内であること）の条件が必須となる。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置を z->ket に設定し、a_6 配列を用いて後方検索を実行する。 2. マッチした接尾辞の種類（among_var）に応じて以下の処理を行う：

  • **ケース 1, 3, 4, 5, 8, 9**: r_R2 を確認後、接尾辞を削除または指定文字列に置換する。一部のケースでは、削除後に特定の文字列（s_11, s_16, s_17 等）の有無を再確認し、追加の削除を行う。
  • **ケース 2**: r_R2 を確認して削除後、さらに s_11 が存在すれば r_R2 を条件に削除する。
  • **ケース 6**: r_RV を確認後、接尾辞を削除する。
  • **ケース 7**: r_R1 を確認後、接尾辞を削除する。その後、特定の文字条件を満たす場合に a_4 を用いた追加の検索と削除を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: italian_UTF_8_stem（直接の呼び出し関係はソースコードの制御フローに基づく）。
  • **呼び出し先**: find_among_b, r_R1, r_R2, r_RV, slice_del, slice_from_s, eq_s_b。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_6: 接尾辞検索用の among 構造体配列。

失敗モードと境界条件

  • find_among_b がマッチしない場合、0 を返して終了する。
  • r_R1, r_R2, r_RV などの領域チェックで条件を満たさない場合、処理を中断して 0 または負の値を返す。
  • slice_del や slice_from_s が失敗した場合、負の値を返す。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接編集せず .sbl ソースを修正すること。
  • among_var の各ケースにおける r_R2 などの領域制約は、イタリア語のステミング精度に直結するため、変更時は注意が必要。
