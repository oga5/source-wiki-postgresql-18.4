flagInhAttrs

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/common.c:477

目的

テーブルの継承関係に基づき、属性（列）のデフォルト値、NOT NULL制約、および生成列（GENERATED）の情報を解析し、子テーブルの TableInfo 構造体に反映させる。

シグネチャまたは形状

static void flagInhAttrs(Archive *fout, DumpOptions *dopt, TableInfo *tblinfo, int numTables)

入力と出力

  • **入力**:
  • fout: アーカイブハンドル（バージョン判定に使用）。
  • dopt: ダンプオプション。
  • tblinfo: ダンプ対象テーブルの配列。
  • numTables: テーブル数。
  • **出力**:
  • tblinfo 内の各 TableInfo 構造体（notnull_islocal フラグや attrdefs 配列）を直接更新する。

前提条件と不変条件

  • tblinfo はOID順にソートされていることが期待される。
  • RELKIND_SEQUENCE, RELKIND_VIEW, RELKIND_MATVIEW は処理対象外。
  • dobj.dump が偽のテーブルはスキップされる。

アルゴリズムまたは振る舞い

1. 全テーブルを走査し、親を持つテーブルに対して各列を検証する。 2. 親テーブルの属性を検索し、以下の情報を収集する:

  • **NOT NULL制約**: バージョン18未満では親に存在すれば継承とみなす。18以上では全ての親で NOT VALID な場合に限り、子で明示的に出力する。
  • **DEFAULT値**: 親にデフォルト値が存在し、かつ "NULL" でない場合に継承対象とする。子に定義がない場合は AttrDefInfo を新規作成し、必要に応じて依存関係を追加する。
  • **GENERATED列**: 親と一致する式を持つ場合はダンプ対象から除外する（DUMP_COMPONENT_NONE）。

    • 3. 属性が削除済み（attisdropped）の場合は処理をスキップする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/common.c 内のメイン処理フロー（getSchemaData 等の過程）。
  • **呼び出し先**: strInArray, pg_malloc_object, AssignDumpId, shouldPrintColumn, addObjectDependency 等。

関連データ構造

  • TableInfo: テーブルのメタデータと属性情報を保持。
  • AttrDefInfo: 列のデフォルト値定義。
  • Archive: 接続先データベースのバージョン情報を含む。

失敗モードと境界条件

  • 親テーブルがダンプ対象外の場合、その属性情報は考慮されない。
  • バージョン18を境界として、NOT NULL制約の判定ロジックが大きく異なる。
  • pg_malloc_object 等のメモリ確保失敗時はプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は、ループ内での状態変更が後続の反復に影響を与えないよう設計されている。デフォルト値の生成ロジックを変更する際は、adef_expr が "NULL" かどうかの判定条件を維持すること。
  • fout->remoteVersion による条件分岐は、PostgreSQLのバージョンアップに伴う継承仕様の変更を吸収するためのものである。
