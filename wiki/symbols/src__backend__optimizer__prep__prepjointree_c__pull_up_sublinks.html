pull_up_sublinks

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepjointree.c:467

目的

クエリのジョインツリー内に存在する ANY や EXISTS などのサブリンク（SubLink）を、プランナが最適化可能なジョイン構造へ変換・引き上げを行う。

シグネチャまたは形状

void pull_up_sublinks(PlannerInfo *root)

入力と出力

  • **入力**: root (PlannerInfo *) - 現在のクエリのプランニング情報。
  • **出力**: なし（root->parse->jointree を直接更新する）。

前提条件と不変条件

  • root->parse->jointree は有効なジョインツリー構造であること。
  • 処理後、root->parse->jointree は必ず FromExpr 型として保持される必要がある。

アルゴリズムまたは振る舞い

1. pull_up_sublinks_jointree_recurse を呼び出し、ジョインツリー全体を再帰的に走査してサブリンクの引き上げ処理を実行する。 2. 再帰の結果得られたノード（jtnode）を確認する。 3. jtnode が FromExpr であれば、それをそのまま root->parse->jointree に設定する。 4. jtnode が FromExpr 以外（RangeTblRef や JoinExpr など）である場合、makeFromExpr を使用してダミーの FromExpr でラップし、root->parse->jointree に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c:757
  • src/backend/optimizer/prep/prepjointree.c:1366
  • **呼び出し先**:
  • pull_up_sublinks_jointree_recurse (再帰的なジョインツリー走査)
  • makeFromExpr (ジョインツリーの正規化用)

関連データ構造

  • PlannerInfo: クエリのプランニング状態を保持する構造体。
  • FromExpr: ジョインツリーのノード型。
  • SubLink: 引き上げ対象となるサブクエリ構造。

失敗モードと境界条件

  • 再帰処理中にスタックオーバーフローが発生する可能性がある（check_stack_depth() による保護が前提）。
  • ジョインツリーが空の場合や、単一の RangeTblRef の場合でも、makeFromExpr によるラップにより FromExpr 構造が維持される。

保守担当者向けメモ

  • 本関数はクエリ最適化の初期段階で実行されるため、呼び出し順序が厳密に管理されています。
  • pull_up_sublinks_jointree_recurse が返すノード型に応じて FromExpr を補完するロジックは、プランナが期待するツリー構造を維持するために不可欠です。
