varbit_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:586

目的

VarBit 構造体（BIT および BIT VARYING 型の内部表現）を、SQL での文字列表現（'0' および '1' の並び）に変換します。

シグネチャまたは形状

Datum varbit_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して VarBit 型のポインタを受け取ります。
  • **出力**: cstring（ヌル終端文字列）として、ビット列の文字列表現を返します。

前提条件と不変条件

  • VARBIT_CORRECTLY_PADDED(s): 入力された VarBit の最後のバイトにおいて、未使用ビットがゼロクリアされている必要があります。この不変条件が満たされていない場合、変換結果が正しくなりません。

アルゴリズムまたは振る舞い

1. VARBITLEN(s) でビット長を取得し、結果格納用のメモリを palloc で確保します。 2. VARBITS(s) でデータ領域の先頭ポインタを取得します。 3. バイト単位のループで、各バイト内のビットを上位ビットから順に走査し、'0' または '1' の文字に変換して結果バッファに書き込みます。 4. 最後に残った端数ビット（8ビット未満の残り）がある場合、同様に走査して変換します。 5. 最後にヌル終端文字を付与し、文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/varbit.c 内の他の関数から呼び出される可能性があります（ソースコード内の直接的な参照として varbit.c:284 が確認されています）。
  • **呼び出し先**: palloc（メモリ確保）、VARBITLEN、VARBITS、IS_HIGHBIT_SET、PG_RETURN_CSTRING などのマクロや関数を使用します。

関連データ構造

  • VarBit: ビット列を保持する構造体。vl_len_（ヘッダ）、bit_len（ビット長）、bit_dat（データ）で構成されます。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • パディングが正しくないデータが渡された場合、論理的に誤った文字列が出力されます。

保守担当者向けメモ

  • 本関数は bit_dat の最上位ビット（MSB）から順に処理します。ビット操作のロジックを変更する際は、VARBIT_PAD マクロによるパディングの不変条件が維持されていることを前提としてください。
