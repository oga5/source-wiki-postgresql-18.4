DistinctColInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:44

目的

サブクエリの出力列が、上位レベルの結合条件等において一意性を判定するために必要な情報を保持するデータ構造です。

シグネチャまたは形状

typedef struct DistinctColInfo
{
    int         colno;          /* サブクエリ出力列の resno */
    Oid         opid;           /* 上位レベルの等価演算子 */
    Oid         collid;         /* opid の入力照合順序 */
} DistinctColInfo;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。各フィールドは、サブクエリの一意性検証プロセスにおいて、特定の列に対する比較条件を定義するために使用されます。

前提条件と不変条件

  • colno はサブクエリの出力列を指す有効な resno である必要があります。
  • opid は、対象列の比較に使用される適切な等価演算子である必要があります。

アルゴリズムまたは振る舞い

本構造体は、List 形式で管理され、一意性検証の際に各列の比較ルールを定義するために参照されます。distinct_col_search 関数等を通じて、特定の列番号に対応する比較情報が検索されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/analyzejoins.c 内の関数群（distinct_col_search など）が、一意性検証の過程で本構造体を生成・検索します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • List: DistinctColInfo のエントリを保持するために使用されます。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保が失敗する可能性があります。
  • 無効な opid や collid: 比較処理において予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は、サブクエリの一意性制約を最適化するプロセスにおいて重要な役割を果たします。
  • distinct_cols リストを操作する際は、foreach_ptr マクロ等を用いて適切にイテレーションを行ってください。
  • 構造体のフィールド追加や変更を行う場合は、関連する検索関数 distinct_col_search への影響を確認してください。
