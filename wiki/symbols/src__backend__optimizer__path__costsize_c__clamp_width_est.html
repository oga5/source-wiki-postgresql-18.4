clamp_width_est

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/costsize.c:241

目的

タプル幅の推定値がシステム上の許容範囲を超えないよう制限（クランプ）し、安全な int32 型の値として返します。

シグネチャまたは形状

int32 clamp_width_est(int64 tuple_width)

入力と出力

  • **入力**: tuple_width (int64) - 推定されたタプル幅。
  • **出力**: int32 - 調整後のタプル幅。

前提条件と不変条件

  • 入力値 tuple_width は負であってはなりません（Assert(tuple_width >= 0) により検証されます）。

アルゴリズムまたは振る舞い

1. 入力値が MaxAllocSize を超える場合、MaxAllocSize を返します。これは、それ以上のサイズのタプルは作成不可能であるという前提に基づいています。 2. 入力値が負でないことをアサートで確認します。 3. 上記の条件に該当しない場合、入力値を int32 にキャストしてそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/costsize.c (複数箇所)
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/util/placeholder.c
  • src/backend/optimizer/util/plancat.c
  • src/backend/optimizer/util/relnode.c
  • **呼び出し先**:
  • 本関数は計算ロジックを完結しており、他の関数を呼び出しません。

関連データ構造

  • int64, int32: 数値の型。
  • MaxAllocSize: メモリ割り当ての最大制限値。

失敗モードと境界条件

  • **負の値**: Assert によりデバッグビルド時に捕捉されます。
  • **過大な値**: MaxAllocSize を超える値は、強制的に MaxAllocSize に切り詰められます。

保守担当者向けメモ

  • clamp_row_est とは異なり、負の値をマスクせずアサートでチェックする仕様です。変更を加える際は、このエラーハンドリング方針の一貫性に注意してください。
