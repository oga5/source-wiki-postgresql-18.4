SetSecurityLabel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/seclabel.c:403

目的

データベースオブジェクトに対するセキュリティラベルの永続化（追加、更新、または削除）を行います。

シグネチャまたは形状

void SetSecurityLabel(const ObjectAddress *object, const char *provider, const char *label)

入力と出力

  • **入力**:
  • object: 対象オブジェクトの識別情報（ObjectAddress）。
  • provider: ラベルプロバイダー名。
  • label: 設定するラベル文字列（NULL の場合はラベルの削除を意味する）。
  • **出力**: なし（副作用としてシステムカタログを更新）。

前提条件と不変条件

  • 共有オブジェクト（IsSharedRelation が真）の場合は、内部で SetSharedSecurityLabel に処理を委譲します。
  • 対象オブジェクトには RowExclusiveLock が取得されます。

アルゴリズムまたは振る舞い

1. 共有オブジェクトであれば SetSharedSecurityLabel を呼び出して終了します。 2. pg_seclabel カタログを RowExclusiveLock で開きます。 3. 指定された object と provider をキーとして、既存のラベルタプルをインデックス（SecLabelObjectIndexId）から検索します。 4. 既存のタプルが見つかった場合:

  • label が NULL ならば CatalogTupleDelete で削除します。
  • label が存在すれば heap_modify_tuple で内容を更新し、CatalogTupleUpdate を実行します。

    • 5. 既存のタプルが見つからず、かつ label が NULL でない場合、heap_form_tuple で新規タプルを作成し CatalogTupleInsert を実行します。 6. 最後に table_close を呼び出し、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/seclabel.c（ExecSecLabelStmt 等）
  • contrib/sepgsql/ 配下の各ファイル（database.c, label.c, proc.c, relation.c, schema.c）
  • **呼び出し先**:
  • SetSharedSecurityLabel
  • table_open / table_close
  • systable_beginscan / systable_getnext / systable_endscan
  • CatalogTupleInsert / CatalogTupleUpdate / CatalogTupleDelete

関連データ構造

  • ObjectAddress: 操作対象のオブジェクトを特定する構造体。
  • pg_seclabel: セキュリティラベルを保持するシステムカタログ。

失敗モードと境界条件

  • label が NULL で、かつ対象のラベルが存在しない場合、何も行われません。
  • 共有カタログと通常カタログの切り分けは IsSharedRelation に依存します。

保守担当者向けメモ

  • 本関数はラベルの「保存」のみを担当します。ラベルの妥当性検証は、本関数を呼び出す前の段階でプロバイダーのフック関数を通じて行われる必要があります。
  • pg_seclabel のスキーマ変更やインデックス定義の変更時には、ScanKey の設定および values 配列のインデックス指定を同期させる必要があります。
