acldefault_sql

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/acl.c:920

目的

SQL インターフェースを通じて、指定されたオブジェクトタイプと所有者に対するデフォルトのアクセス制御リスト（ACL）を取得します。

シグネチャまたは形状

Datum acldefault_sql(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_CHAR(0): オブジェクトタイプを表す文字（例: 'r' はテーブル）。
  • PG_GETARG_OID(1): オブジェクト所有者の OID。
  • **出力**:
  • Datum: acldefault 関数が生成したデフォルトの Acl 配列を PG_RETURN_ACL_P マクロで返却。

前提条件と不変条件

  • 入力された文字が定義済みのオブジェクトタイプ（'c', 'r', 's', 'd', 'f', 'l', 'L', 'n', 'p', 't', 'F', 'S', 'T'）のいずれかである必要があります。

アルゴリズムまたは振る舞い

1. 引数として渡された文字を switch 文で評価し、対応する ObjectType 列挙型に変換します。 2. 未定義の文字が渡された場合は elog(ERROR) を発生させます。 3. 変換された ObjectType と所有者 OID を引数として acldefault を呼び出します。 4. acldefault の戻り値を PG_RETURN_ACL_P を介して呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（直接的な呼び出し元情報はソースコード上では未特定）。
  • **呼び出し先**: acldefault（ACL 配列を生成する内部関数）。

関連データ構造

  • ObjectType: オブジェクトの種類を識別する列挙型。
  • Acl: アクセス制御リストを保持する配列型。

失敗モードと境界条件

  • 未知のオブジェクトタイプ文字が渡された場合、elog(ERROR) によりトランザクションが中断されます。

保守担当者向けメモ

  • 本関数は SQL レベルのラッパーであり、ロジックの本体は acldefault に依存しています。
  • 新しいオブジェクトタイプをサポートする場合は、switch 文にケースを追加する必要があります。
