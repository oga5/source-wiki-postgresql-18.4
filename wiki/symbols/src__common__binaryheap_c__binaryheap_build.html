binaryheap_build

定義

  • 種別: function
  • 定義箇所: src/common/binaryheap.c:137

目的

順序付けされていない要素群が格納されたバイナリヒープに対し、ヒーププロパティ（最大ヒープまたは最小ヒープの条件）を適用し、有効なヒープ構造へと再構築する。

シグネチャまたは形状

void binaryheap_build(binaryheap *heap)

入力と出力

  • **入力**: heap - 構築対象の binaryheap 構造体へのポインタ。
  • **出力**: なし（副作用として heap 構造体の内部状態を更新）。

前提条件と不変条件

  • heap は binaryheap_allocate 等で適切に初期化されていること。
  • ヒープ内の要素は binaryheap_add_unordered 等で既に配列に格納されていること。
  • 実行完了後、heap->bh_has_heap_property が true に設定される。

アルゴリズムまたは振る舞い

1. ヒープの最後の要素の親ノード（parent_offset(heap->bh_size - 1)）から開始し、インデックス 0 に向かって逆順に sift_down を呼び出す。 2. 各ノードに対して sift_down を適用することで、部分木を順次ヒープ化する。 3. 全ノードの処理完了後、bh_has_heap_property フラグを true に更新する。 4. 計算量は O(n) である。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeGatherMerge.c
  • src/backend/executor/nodeMergeAppend.c
  • src/backend/postmaster/pgarch.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/storage/buffer/bufmgr.c
  • src/bin/pg_dump/pg_dump_sort.c
  • **呼び出し先**: sift_down（内部関数）、parent_offset（インライン関数）。

関連データ構造

  • binaryheap: ヒープのノード配列、サイズ、比較関数を保持する構造体。

失敗モードと境界条件

  • heap->bh_size が 0 の場合、ループは実行されず、フラグのみが true に更新される。
  • heap が NULL の場合の挙動は定義されていない（セグメンテーション違反の可能性がある）。

保守担当者向けメモ

  • 本関数は binaryheap_add_unordered で要素を詰め込んだ後に呼び出すことで、個別に binaryheap_add を繰り返すよりも効率的にヒープを構築できる。
  • 比較関数（binaryheap_comparator）の定義が、最大ヒープか最小ヒープかを決定する。
