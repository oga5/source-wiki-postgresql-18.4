findoprnd

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_io.c:311

目的

ltxtquery 構造体内の ITEM 配列を走査し、各演算子ノードが参照すべき左オペランドへのオフセット（left フィールド）を計算・設定します。

シグネチャまたは形状

static bool findoprnd(QPRS_STATE *state, ITEM *ptr, int32 *pos);

入力と出力

  • **入力**:
  • state: 解析状態を保持する QPRS_STATE 構造体へのポインタ。
  • ptr: ITEM 型の配列（クエリの逆ポーランド記法表現）。
  • pos: 現在処理中の ITEM インデックスを指すポインタ。
  • **出力**:
  • 成功時は true を返し、ptr 内の各ノードの left フィールドが更新されます。
  • 失敗時は false を返し、エラーコンテキストにエラー情報が設定されます。

前提条件と不変条件

  • ptr は有効な ITEM 配列であること。
  • pos は配列の先頭を指して呼び出されること。
  • 再帰呼び出しを行うため、スタック深さの制限（check_stack_depth()）を遵守すること。

アルゴリズムまたは振る舞い

1. 現在の pos を記録し、pos をインクリメントして次の要素へ進みます。 2. 現在のノード型に応じて処理を分岐します：

  • **VAL / VALTRUE**: オペランドを持たないため、left を 0 に設定します。
  • **単項演算子 (!)**: left を 1 に設定し、再帰的に右オペランドを走査します。
  • **二項演算子**:
  • 右オペランドを再帰的に走査します。
  • 現在の pos と開始位置の差分（delta）を計算し、左オペランドへのオフセットとして left に格納します。
  • 左オペランドを再帰的に走査します。

    • 3. 処理中に delta が PG_INT16_MAX を超えた場合、ltxtquery が大きすぎるとしてエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/ltxtquery_io.c 内のクエリ構築処理。
  • contrib/intarray/_int_bool.c 内の論理演算処理。
  • **呼び出し先**:
  • findoprnd（再帰呼び出し）。
  • check_stack_depth()（スタックオーバーフロー防止）。
  • ereturn（エラー発生時）。

関連データ構造

  • ITEM: ltxtquery を構成する演算子またはオペランドの要素。
  • QPRS_STATE: 解析中の状態を保持する構造体。

失敗モードと境界条件

  • **スタックオーバーフロー**: 再帰が深すぎる場合に発生。
  • **プログラム制限超過**: delta が 16 ビット符号付き整数の最大値を超える場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED を発生させます。
  • **不正な構造**: Assert(delta > 0) により、オフセット計算が正しくない場合に異常終了します。

保守担当者向けメモ

  • 本関数は再帰的に動作するため、クエリの複雑さに応じてスタックを消費します。
  • left フィールドは int16 型であるため、クエリの構造が非常に大きい場合に制限に達する可能性があります。
