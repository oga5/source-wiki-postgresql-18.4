executeQuery

定義

  • 種別: function
  • 定義箇所: src/fe_utils/query_utils.c:21

目的

PostgreSQLフロントエンドツールにおいて、SQLクエリを実行し、その結果を検証して返すためのユーティリティ関数です。エラー発生時にはログを出力し、プログラムを終了させることで、呼び出し側のエラーハンドリングを簡略化します。

シグネチャまたは形状

PGresult *executeQuery(PGconn *conn, const char *query, bool echo);

入力と出力

  • **入力**:
  • conn: データベース接続を保持する PGconn オブジェクト。
  • query: 実行するSQL文字列。
  • echo: 真の場合、実行前にクエリを標準出力へ表示するフラグ。
  • **出力**:
  • 成功時: クエリ結果を保持する PGresult ポインタ。
  • 失敗時: 戻り値は返さず、プロセスが終了します。

前提条件と不変条件

  • conn は有効なデータベース接続である必要があります。
  • 実行されるクエリは、PGRES_TUPLES_OK ステータスを返すことが期待されるクエリである必要があります。

アルゴリズムまたは振る舞い

1. echo が真であれば、query を標準出力に書き出します。 2. libpq の PQexec を呼び出し、クエリを実行します。 3. 結果が空であるか、ステータスが PGRES_TUPLES_OK でない場合：

  • pg_log_error を使用してエラーメッセージを出力します。
  • pg_log_error_detail を使用して実行したクエリ内容を出力します。
  • PQfinish(conn) を呼び出して接続を閉じます。
  • exit(1) を呼び出し、プログラムを異常終了させます。

    • 4. 正常終了時は、取得した PGresult ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_dump/connectdb.c
  • src/bin/pg_dump/pg_dumpall.c
  • **呼び出し先**:
  • libpq: PQexec, PQerrorMessage, PQfinish
  • common/logging.h: pg_log_error, pg_log_error_detail
  • 標準ライブラリ: printf, exit

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGresult: libpq のクエリ結果ハンドル。

失敗モードと境界条件

  • クエリ実行が失敗した場合（PQexec が NULL を返す、またはステータスが PGRES_TUPLES_OK 以外の場合）、即座にプログラムが終了します。
  • 戻り値の PGresult は呼び出し元で適切に解放（PQclear）される必要があります。

保守担当者向けメモ

  • 本関数は「失敗即終了」を前提としたCLIツール向けの設計です。トランザクション制御やエラーの回復が必要な複雑な処理には適していません。
  • PGRES_TUPLES_OK 以外のステータス（例: PGRES_COMMAND_OK）を期待するクエリを実行する場合は、本関数ではなく executeCommand 等の利用を検討してください。
