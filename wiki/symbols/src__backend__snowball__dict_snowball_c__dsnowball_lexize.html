dsnowball_lexize

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/dict_snowball.c:274

目的

PostgreSQLの全文検索（Tsearch）フレームワークにおいて、Snowballアルゴリズムを用いて入力トークンの語幹抽出（ステミング）を行う。

シグネチャまたは形状

Datum dsnowball_lexize(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): DictSnowball 構造体へのポインタ。
  • PG_GETARG_POINTER(1): 入力文字列（char *）。
  • PG_GETARG_INT32(2): 入力文字列の長さ。
  • **出力**:
  • PG_RETURN_POINTER(TSLexeme *): 語幹抽出結果、またはストップワード判定時の空ポインタを含む TSLexeme 配列。

前提条件と不変条件

  • 入力文字列は str_tolower により小文字化される。
  • 1000バイトを超える文字列はステミング処理をスキップし、小文字化した元の文字列をそのまま返す。
  • DictSnowball 内の dictCtx を使用してメモリコンテキストを切り替える必要がある。

アルゴリズムまたは振る舞い

1. 入力文字列を小文字化する。 2. 文字列長が1000バイトを超える場合、ステミングを行わず結果として返す。 3. ストップワードリスト（searchstoplist）に含まれる場合、メモリを解放し空の辞書結果を返す。 4. 必要に応じて pg_server_to_any でUTF-8へ変換する。 5. SN_set_current でSnowball環境に文字列をセットし、d->stem 関数を実行する。 6. ステミング結果を元のエンコーディングに戻し（pg_any_to_server）、TSLexeme として格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのTsearch辞書インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: str_tolower, searchstoplist, pg_server_to_any, SN_set_current, d->stem, pg_any_to_server。

関連データ構造

  • DictSnowball: 辞書設定、Snowball環境（SN_env）、ストップワードリストを保持する構造体。
  • TSLexeme: 語幹抽出結果を保持するPostgreSQLの全文検索用構造体。

失敗モードと境界条件

  • **1000バイト制限**: ステマーの無限再帰や誤動作を防ぐため、長大な文字列はステミングを回避する。
  • **エンコーディング変換**: サーバーエンコーディングとステマーのエンコーディングが異なる場合、動的な再エンコード処理が発生する。
  • **メモリ管理**: dictCtx を使用したコンテキスト切り替えに失敗した場合、メモリリークや不正アクセスの可能性がある。

保守担当者向けメモ

  • d->stem の実行前後で MemoryContextSwitchTo を使用している点に注意。
  • トルコ語ステマーなど、特定の言語で無限再帰の報告があるため、1000バイト制限は厳守すること。
