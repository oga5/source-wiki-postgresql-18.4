IpcSemaphoreKey

定義

  • 種別: typedef
  • 定義箇所: src/backend/port/sysv_sema.c:45

目的

System V セマフォ API の semget(2) システムコールに渡すためのセマフォキーを保持する型です。

シグネチャまたは形状

typedef key_t IpcSemaphoreKey;

入力と出力

  • **入力**: key_t 型（通常は long 相当）の値。
  • **出力**: semget システムコールにおけるセマフォセットの識別子として使用されます。

前提条件と不変条件

  • sys/ipc.h で定義される key_t 型に依存します。
  • PostgreSQL が作成するセマフォセットを識別するため、一意なキー値が割り当てられる必要があります。

アルゴリズムまたは振る舞い

  • nextSemaKey 変数によって、次に試行すべきキー値が管理されます。
  • InternalIpcSemaphoreCreate 関数において、このキーを用いてカーネルからセマフォセットの取得または作成が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: InternalIpcSemaphoreCreate 関数の引数として使用されます。
  • **呼び出し先**: semget(2) システムコール（InternalIpcSemaphoreCreate 内で呼び出されます）。

関連データ構造

  • IpcSemaphoreId: semget が返すセマフォセット ID。
  • nextSemaKey: 次に使用されるキーを保持する静的変数。

失敗モードと境界条件

  • キーの衝突が発生した場合、InternalIpcSemaphoreCreate 内で既存のセマフォセットの所有者確認やクリーンアップ処理が試行されます。
  • カーネルの制限（SEMMNI 等）により、新しいキーを用いたセマフォセットの作成が失敗する可能性があります。

保守担当者向けメモ

  • 本型は sys/ipc.h に依存しているため、プラットフォーム固有の key_t の定義に注意してください。
  • nextSemaKey の更新ロジックは sysv_sema.c 内のセマフォ割り当てフローに密接に関連しています。
