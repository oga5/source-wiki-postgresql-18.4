AlterSubscriptionOwner_internal

定義

  • 種別: function
  • 定義箇所: src/backend/commands/subscriptioncmds.c:1969

目的

論理レプリケーションサブスクリプションの所有者を変更し、システムカタログの更新および依存関係の再構築を行う。

シグネチャまたは形状

static void AlterSubscriptionOwner_internal(Relation rel, HeapTuple tup, Oid newOwnerId)

入力と出力

  • rel: pg_subscription システムカタログの開かれたリレーション。
  • tup: 変更対象のサブスクリプションを表す pg_subscription のタプル。
  • newOwnerId: 新しい所有者のロール OID。
  • 出力: なし（副作用としてカタログおよび依存関係を更新）。

前提条件と不変条件

  • 呼び出し元は、対象のサブスクリプションに対する適切なロックを保持している必要がある。
  • newOwnerId は、現在のユーザーがロールとして設定可能な権限を持つ必要がある。
  • 現在のユーザーは、データベースに対する CREATE 権限を持つ必要がある。

アルゴリズムまたは振る舞い

1. **所有権チェック**: 現在のユーザーがサブスクリプションの所有者であることを確認する。 2. **特権検証**: password_required=false のサブスクリプションの場合、スーパーユーザー以外による変更を禁止する。 3. **ロール設定権限の確認**: check_can_set_role により、新しい所有者への変更が可能か検証する。 4. **データベース権限の確認**: 現在のデータベースに対する CREATE 権限を検証する。 5. **カタログ更新**: pg_subscription の subowner フィールドを更新し、CatalogTupleUpdate を実行する。 6. **依存関係更新**: changeDependencyOnOwner を呼び出し、所有者に関する依存関係を更新する。 7. **フック実行**: InvokeObjectPostAlterHook を呼び出し、変更を通知する。 8. **ワーカー通知**: ApplyLauncherWakeupAtCommit および LogicalRepWorkersWakeupAtCommit を呼び出し、関連するバックグラウンドプロセスに変更を即時反映させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/subscriptioncmds.c 内の AlterSubscriptionOwner および AlterSubscriptionOwner_oid から呼び出される。
  • 呼び出し先: object_ownercheck, superuser, check_can_set_role, object_aclcheck, CatalogTupleUpdate, changeDependencyOnOwner, InvokeObjectPostAlterHook, ApplyLauncherWakeupAtCommit, LogicalRepWorkersWakeupAtCommit。

関連データ構造

  • Form_pg_subscription: pg_subscription カタログの構造体。
  • Relation: システムカタログへのアクセスに使用。
  • HeapTuple: カタログタプル。

失敗モードと境界条件

  • **権限エラー**: 所有者でない場合、または password_required=false で非スーパーユーザーが操作しようとした場合に ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了する。
  • **ACL エラー**: データベースに対する CREATE 権限がない場合、aclcheck_error によりエラー終了する。
  • **同一所有者**: newOwnerId が現在の所有者と同一の場合、何もせず即座に復帰する。

保守担当者向けメモ

  • 本関数は AlterSubscriptionOwner の内部実装であり、直接呼び出されることは想定されていない。
  • トランザクション終了時にワーカーをウェイクアップさせるため、トランザクションの整合性に依存している。
