close_lo_relation

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:96

目的

トランザクション終了時に、ラージオブジェクト（LO）操作のためにオープンされた pg_largeobject カタログテーブルおよびそのインデックスのリレーション参照をクローズし、リソースを解放します。

シグネチャまたは形状

void close_lo_relation(bool isCommit)

入力と出力

  • **入力**:
  • isCommit (bool): トランザクションがコミットされる場合は true、アボートされる場合は false を指定します。
  • **出力**: なし。

前提条件と不変条件

  • lo_heap_r および lo_index_r は、このモジュール内で管理されるグローバルなリレーション参照です。
  • isCommit が true の場合のみ、明示的なクローズ処理（index_close, table_close）が実行されます。
  • アボート時（isCommit が false）は、リソースオーナーによるクリーンアップ処理に委ねられます。

アルゴリズムまたは振る舞い

1. lo_heap_r または lo_index_r が非 NULL であるかを確認します。 2. isCommit が true の場合:

  • 現在の CurrentResourceOwner を一時的に TopTransactionResourceOwner に切り替えます。
  • lo_index_r が存在すれば index_close(lo_index_r, NoLock) を呼び出します。
  • lo_heap_r が存在すれば table_close(lo_heap_r, NoLock) を呼び出します。
  • CurrentResourceOwner を元の値に戻します。

    • 3. 最後に lo_heap_r および lo_index_r を NULL にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-fsstubs.c:640
  • **呼び出し先**:
  • index_close
  • table_close

関連データ構造

  • lo_heap_r: pg_largeobject テーブルへのキャッシュされたリレーション参照。
  • lo_index_r: pg_largeobject のインデックスへのキャッシュされたリレーション参照。

失敗モードと境界条件

  • lo_heap_r や lo_index_r が既に NULL の場合、関数は何も行わずに終了します。
  • isCommit が false の場合、明示的なクローズは行われません。

保守担当者向けメモ

  • 本関数はトランザクション終了時のクリーンアップを目的としています。リレーションのクローズ時に NoLock を指定しているため、ロックの解放は呼び出し側またはリソース管理機構に依存します。
