PGChecksummablePage

定義

  • 種別: typedef (共用体)
  • 定義箇所: src/include/storage/checksum_impl.h:111

目的

PostgreSQLのデータページを32列の行列として扱い、SIMDベクトル化による高速なチェックサム計算を促進するための共用体構造です。

シグネチャまたは形状

typedef union
{
    PageHeaderData phdr;
    uint32         data[BLCKSZ / (sizeof(uint32) * N_SUMS)][N_SUMS];
} PGChecksummablePage;

入力と出力

  • 本シンボル自体はデータ構造であり、関数ではありません。
  • pg_checksum_block 関数において、ページデータを本構造体にキャストして入力として受け取ります。

前提条件と不変条件

  • sizeof(PGChecksummablePage) == BLCKSZ であることが不変条件としてアサートされています。
  • N_SUMS（32）および checksumBaseOffsets の値は、計算結果の互換性に直結するため変更不可です。

アルゴリズムまたは振る舞い

  • ページデータを PageHeaderData としてアクセスする機能と、uint32 の2次元配列としてアクセスする機能を共用体によって提供します。
  • 32個の並列ハッシュ計算を行うために、データを [BLCKSZ / (4 * 32)][32] の行列形式で解釈します。

呼び出し元と呼び出し先

  • 呼び出し元: pg_checksum_block 関数が本構造体へのポインタを引数として受け取ります。
  • 呼び出し先: 本シンボルはデータ構造であるため、直接の呼び出し先はありません。

関連データ構造

  • PageHeaderData: ページヘッダーの定義（storage/bufpage.h 依存）。
  • N_SUMS: 並列計算数（32）。

失敗モードと境界条件

  • 外部プログラムで利用する場合、Assert() マクロが定義されていないとコンパイルエラーになる可能性があります。
  • BLCKSZ が sizeof(uint32) * N_SUMS で割り切れない場合、構造体の定義が意図通りに機能しない可能性があります。

保守担当者向けメモ

  • コンパイラによる自動ベクトル化（-ftree-vectorize 等）を強く意識した設計です。手動でループを修正する際は、SIMD命令の効率を損なわないよう注意してください。
  • 外部ツールから利用する際は、依存関係を最小限に抑えるため、必要に応じて Assert を空定義に置き換えるなどの調整を行ってください。
