gistMakeUnionKey

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistutil.c:232

目的

2つのGiSTエントリ（またはNULL値）から、それらを包含する「和（Union）」キーを計算し、インデックスの親ノードに格納するための値を生成します。

シグネチャまたは形状

void gistMakeUnionKey(GISTSTATE *giststate, int attno,
                      GISTENTRY *entry1, bool isnull1,
                      GISTENTRY *entry2, bool isnull2,
                      Datum *dst, bool *dstisnull)

入力と出力

  • **入力**:
  • giststate: GiSTインデックスの操作状態を保持する構造体。
  • attno: 対象となる属性番号。
  • entry1, entry2: 結合対象のGiSTエントリ。
  • isnull1, isnull2: 各エントリがNULLかどうかを示すフラグ。
  • **出力**:
  • dst: 計算されたUnionキーのDatum値が格納されるポインタ。
  • dstisnull: 結果がNULLかどうかを示すフラグ。

前提条件と不変条件

  • giststate は有効なインデックス状態を指している必要があります。
  • attno に対応する unionFn が giststate 内に定義されている必要があります。
  • 内部的に GistEntryVector を使用して2つのエントリを保持するため、スタック上に一時的な領域を確保します。

アルゴリズムまたは振る舞い

1. 2つのエントリを保持する GistEntryVector を構築します。 2. 両方がNULLの場合、結果をNULLとして返します。 3. いずれか一方がNULLの場合、非NULL側のエントリを2つ並べたベクトルを作成します。 4. 両方が非NULLの場合、それぞれのエントリをベクトルに格納します。 5. giststate->unionFn[attno] を呼び出し、ベクトルからUnionキーを計算して dst に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistsplit.c: ページ分割処理において、新しいUnionキーを生成する際に呼び出されます。
  • src/backend/access/gist/gistutil.c: インデックスの調整処理等で呼び出されます。
  • **呼び出し先**:
  • giststate->unionFn[attno]: インデックスのopclassで定義されたUnion関数。

関連データ構造

  • GISTSTATE: GiSTインデックスの操作に必要なサポート関数や照合順序を保持する構造体。
  • GISTENTRY: GiSTインデックスのキーとメタデータを保持する構造体。
  • GistEntryVector: 複数の GISTENTRY を保持するための可変長構造体。

失敗モードと境界条件

  • isnull1 と isnull2 が共に真の場合、結果はNULLとして正しく処理されます。
  • unionFn が提供されていない場合、または実行中にエラーが発生した場合は、PostgreSQLの関数呼び出し機構により例外がスローされます。

保守担当者向けメモ

  • 本関数は GistEntryVector をスタック上の union 構造体で一時的に確保しています。この領域は2つの GISTENTRY を格納するのに十分なサイズとして設計されています。
  • dstsize 変数はコンパイラの警告を抑制するために初期化されていますが、現在の実装では unionFn の戻り値として直接使用されていません。
