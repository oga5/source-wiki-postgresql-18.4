InitDiscoverCompressFileHandle

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_io.c:239

目的

ファイルパスと拡張子に基づいて圧縮アルゴリズムを自動検出し、対応する CompressFileHandle を構築および初期化します。

シグネチャまたは形状

CompressFileHandle *InitDiscoverCompressFileHandle(const char *path, const char *mode);

入力と出力

  • **入力**:
  • path: 対象ファイルのパス。
  • mode: ファイルオープンモード（PG_BINARY_R であることが前提）。
  • **出力**:
  • 成功時: 初期化済みの CompressFileHandle 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • mode 引数は PG_BINARY_R と一致する必要があります（Assert による検証）。
  • 内部で pg_strdup を使用するため、呼び出し元は返り値が NULL でない場合に適切な解放を行う必要があります。

アルゴリズムまたは振る舞い

1. compression_spec を PG_COMPRESSION_NONE で初期化します。 2. ファイル名のサフィックス（.gz, .lz4, .zst）を hasSuffix で確認し、対応するアルゴリズムを決定します。 3. サフィックスが一致しない場合、stat でファイルの存在を確認し、存在しない場合は check_compressed_file を用いて圧縮ファイルとしての存在を順次試行します。 4. 決定したアルゴリズムで InitCompressFileHandle を呼び出し、ハンドルを生成します。 5. 生成されたハンドルの open_func を実行し、ファイルを開きます。 6. 開放に失敗した場合は free_keep_errno でメモリを解放し、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_backup_directory.c 内の複数箇所（tocFH や CFH の初期化時）。
  • **呼び出し先**: hasSuffix, stat, check_compressed_file, InitCompressFileHandle, free_keep_errno。

関連データ構造

  • CompressFileHandle: ファイル I/O を抽象化する関数ポインタ群を保持。
  • pg_compress_specification: 圧縮アルゴリズムの指定に使用。

失敗モードと境界条件

  • 指定されたパスが存在しない、または圧縮形式が特定できない場合は NULL を返します。
  • open_func によるファイルオープンが失敗した場合、内部でメモリを解放し NULL を返します。

保守担当者向けメモ

  • 新しい圧縮アルゴリズムを追加する場合は、サフィックス判定ロジックおよび check_compressed_file の呼び出し順序を更新する必要があります。
  • errno を保持したままメモリ解放を行う free_keep_errno を使用しているため、エラーハンドリングの整合性に注意してください。
