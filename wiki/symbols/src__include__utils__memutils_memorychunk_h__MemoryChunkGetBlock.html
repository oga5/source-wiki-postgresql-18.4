MemoryChunkGetBlock

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/memutils_memorychunk.h:234

目的

メモリチャンクのヘッダー情報から、そのチャンクが属する親ブロックの先頭アドレスを逆算して取得する。

シグネチャまたは形状

static inline void *MemoryChunkGetBlock(MemoryChunk *chunk)

入力と出力

  • **入力**: MemoryChunk *chunk - 調査対象のメモリチャンクヘッダーへのポインタ。
  • **出力**: void * - チャンクを含む親ブロックの開始アドレス。

前提条件と不変条件

  • 対象のチャンクが「外部管理モード（External）」ではないこと（Assert により検証される）。
  • チャンクヘッダーの hdrmask に格納されたオフセット値が、有効なメモリ配置（MAXALIGN 制約）に従っていること。

アルゴリズムまたは振る舞い

1. HdrMaskIsExternal マクロを用いて、チャンクが外部管理されていないことを確認する。 2. HdrMaskBlockOffset マクロを用いて、chunk->hdrmask からブロックへのオフセット量を抽出する。 3. 現在の chunk ポインタから抽出したオフセット量を減算し、ブロックの開始アドレスを計算して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/alignedalloc.c
  • src/backend/utils/mmgr/aset.c
  • src/backend/utils/mmgr/bump.c
  • src/backend/utils/mmgr/generation.c
  • **呼び出し先**:
  • HdrMaskIsExternal (マクロ)
  • HdrMaskBlockOffset (マクロ)

関連データ構造

  • MemoryChunk: チャンクヘッダー構造体。
  • hdrmask: チャンクのメタデータを保持する 64 ビットフィールド。

失敗モードと境界条件

  • **アサーション失敗**: 外部管理チャンクに対して本関数を呼び出した場合、デバッグビルドでは Assert により停止する。
  • **不正なポインタ**: chunk 自体が NULL または不正なアドレスを指している場合、計算結果は未定義の動作を引き起こす。

保守担当者向けメモ

  • 本関数は MemoryChunk の内部レイアウトに強く依存している。ヘッダー構造や hdrmask のエンコード方式を変更する場合は、オフセット計算ロジックとの整合性に注意すること。
  • 外部管理チャンク（MemoryChunkIsExternal が真となるもの）には適用できないため、呼び出し側で事前に判定を行う必要がある。
