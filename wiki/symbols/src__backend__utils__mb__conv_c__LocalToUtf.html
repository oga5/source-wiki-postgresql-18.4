LocalToUtf

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conv.c:716

目的

指定されたローカルエンコーディングの文字列をUTF-8へ変換します。

シグネチャまたは形状

int LocalToUtf(const unsigned char *iso, int len,
               unsigned char *utf,
               const pg_mb_radix_tree *map,
               const pg_local_to_utf_combined *cmap, int cmapsize,
               utf_local_conversion_func conv_func,
               int encoding,
               bool noError);

入力と出力

  • **入力**:
  • iso: 変換元文字列のポインタ。
  • len: 変換元文字列のバイト長。
  • map: ラディックスツリー（変換テーブル）。
  • cmap: 結合文字マップ。
  • conv_func: アルゴリズムによる変換関数。
  • encoding: ローカルエンコーディングID。
  • noError: 変換失敗時にエラーを投げず中断するかのフラグ。
  • **出力**: 実際に処理されたバイト数（iso から消費された長さ）。utf バッファには変換後のUTF-8文字列が格納されます。

前提条件と不変条件

  • encoding は PG_VALID_ENCODING を満たす必要があります。
  • 変換後の utf バッファは、十分な領域を確保している必要があります。
  • 変換成功時は utf の末尾にヌル終端文字が書き込まれます。

アルゴリズムまたは振る舞い

1. 入力文字列をバイト単位で走査し、ASCII文字はそのままコピーします。 2. pg_encoding_verifymbchar で文字境界を特定し、マルチバイト文字を抽出します。 3. 以下の順序で変換を試行します：

  • map を用いたラディックスツリー検索。
  • cmap を用いた結合文字の bsearch 検索。
  • conv_func を用いたアルゴリズム変換。

    • 4. 変換成功時は store_coded_char で結果を書き込みます。 5. 変換失敗時、noError が真であれば処理を中断し、偽であればエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mb/conversion_procs/ 配下の各エンコーディング変換プロシージャ（例: utf8_and_big5.c, utf8_and_euc_jp.c 等）。
  • **呼び出し先**: pg_encoding_verifymbchar, pg_mb_radix_conv, bsearch (比較関数 compare4), store_coded_char, report_untranslatable_char, report_invalid_encoding。

関連データ構造

  • pg_mb_radix_tree: 高速検索用ラディックスツリー。
  • pg_local_to_utf_combined: 結合文字変換用構造体。

失敗モードと境界条件

  • 不正なエンコーディングIDが渡された場合、ereport(ERROR) を発生させます。
  • サポート外の文字長（5バイト以上）が検出された場合、elog(ERROR) を発生させます。
  • 変換不可の文字に遭遇し、noError が偽の場合、report_untranslatable_char を呼び出します。

保守担当者向けメモ

  • 本関数はパフォーマンスに敏感なため、変換ロジックの追加・変更時はラディックスツリーの構造を考慮してください。
  • noError フラグ使用時は、戻り値が正しく消費バイト数を反映しているか確認が必要です。
