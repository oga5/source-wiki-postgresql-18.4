PLy_procedure_compile

定義

  • src/pl/plpython/plpy_procedure.c:348

目的

PL/Python関数またはコードブロックのソースコードをPythonインタープリタ上で実行可能な形式にコンパイルし、実行環境（globals および statics）を初期化する。

シグネチャまたは形状

void PLy_procedure_compile(PLyProcedure *proc, const char *src)

入力と出力

  • **入力**:
  • proc: コンパイル対象の関数情報を保持する PLyProcedure 構造体へのポインタ。
  • src: コンパイル対象のPythonソースコード文字列。
  • **出力**: なし（成功時は proc 構造体の各フィールドにコンパイル済みオブジェクトが格納される）。

前提条件と不変条件

  • proc->mcxt が有効なメモリコンテキストであること。
  • PLy_interp_globals が初期化済みであること。
  • proc->pyname が関数名として設定されていること。

アルゴリズムまたは振る舞い

1. **環境構築**: PLy_interp_globals をコピーして proc->globals を作成し、SD（静的データ）辞書を proc->statics として追加する。 2. **ソース変換**: PLy_procedure_munge_source を呼び出し、ソースコードをPythonの def 文形式に整形する。 3. **関数定義のロード**: 整形後のソースを Py_CompileString でコンパイルし、PyEval_EvalCode で実行してPythonインタープリタに関数をロードする。 4. **呼び出しコードの生成**: 関数名に基づいた呼び出し式（name()）を生成し、Py_CompileString を用いて実行可能なコードオブジェクトとして proc->code に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_main.c:317
  • src/pl/plpython/plpy_procedure.c:329
  • **呼び出し先**:
  • PLy_procedure_munge_source
  • PyDict_Copy, PyDict_New, PyDict_SetItemString
  • Py_CompileString, PyEval_EvalCode, Py_DECREF
  • MemoryContextStrdup, pfree, PLy_elog

関連データ構造

  • PLyProcedure: 関数定義、メモリコンテキスト、コンパイル済みコード、グローバル/静的辞書を保持する主要構造体。

失敗モードと境界条件

  • proc->statics の作成失敗時、またはコンパイル失敗時には PLy_elog(ERROR, ...) を呼び出し、トランザクションを中断する。
  • 関数名がバッファ NAMEDATALEN + 256 を超える場合、elog(ERROR, ...) によりエラーとなる。

保守担当者向けメモ

  • 本関数は proc->mcxt を使用してソース文字列を複製するため、メモリ管理のライフサイクルに注意すること。
  • PLy_procedure_munge_source によるインデント調整は単純な置換であるため、複雑なネストを持つコードのデバッグ時には生成されたソースを確認すること。
