ParallelApplyWorkerInfo

定義

  • **種別**: struct
  • **定義箇所**: src/include/replication/worker_internal.h:188

目的

論理レプリケーションにおけるリーダーワーカーと並列適用ワーカー間の通信、および並列ワーカーのライフサイクル管理を行うための内部構造体です。

シグネチャまたは形状

typedef struct ParallelApplyWorkerInfo
{
    shm_mq_handle *mq_handle;        /* リーダーからワーカーへの変更データ送信キュー */
    shm_mq_handle *error_mq_handle;  /* ワーカーからリーダーへのエラー通知キュー */
    dsm_segment   *dsm_seg;          /* 動的共有メモリセグメント */
    bool           serialize_changes; /* タイムアウト時のファイルシリアライズ要否 */
    bool           in_use;           /* ワーカーの使用状態（true: 使用中, false: 再利用可能） */
    ParallelApplyWorkerShared *shared; /* 共有同期構造体へのポインタ */
} ParallelApplyWorkerInfo;

入力と出力

  • **入力**: リーダーワーカーが mq_handle を介して変更データを送信し、並列ワーカーがこれを受信します。
  • **出力**: 並列ワーカーが error_mq_handle を介してエラーメッセージをリーダーへ送信します。

前提条件と不変条件

  • in_use が false の場合、そのワーカーは再利用可能な状態であることを示します。
  • serialize_changes は、共有メモリ経由のデータ送信がタイムアウトした場合に、残りの変更をファイルへシリアライズする必要があることを示します。

アルゴリズムまたは振る舞い

  • リーダーワーカーは本構造体を介して並列ワーカーを確保し、shm_mq を用いて変更データをストリーミングします。
  • 負荷やタイムアウト状況に応じて、リーダーは serialize_changes フラグを制御し、必要に応じて FileSet を用いた永続化へ移行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/applyparallelworker.c 内の関数群（pa_setup_dsm 等）が本構造体を生成・管理しています。
  • **呼び出し先**: 共有メモリキュー (shm_mq) および共有同期構造体 (ParallelApplyWorkerShared) を操作します。

関連データ構造

  • shm_mq_handle: 通信キューのハンドル。
  • dsm_segment: 共有メモリセグメント。
  • ParallelApplyWorkerShared: リーダーと並列ワーカー間で共有される同期用構造体。

失敗モードと境界条件

  • 共有メモリ経由のデータ送信がタイムアウトした場合、serialize_changes を用いてファイルベースの処理へ切り替える必要があります。
  • エラー発生時は error_mq_handle を通じてリーダーへ通知されます。

保守担当者向けメモ

  • 本構造体は src/backend/replication/logical/applyparallelworker.c で集中的に利用されています。
  • ParallelApplyWorkerShared との整合性を維持するため、変更時はリーダーと並列ワーカー間の同期ロジックに十分注意してください。
