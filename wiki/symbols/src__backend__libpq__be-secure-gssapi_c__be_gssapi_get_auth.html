be_gssapi_get_auth

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-gssapi.c:743

目的

指定された接続（Port）において、GSSAPI認証が完了しているかどうかを判定します。

シグネチャまたは形状

bool be_gssapi_get_auth(Port *port)

入力と出力

  • **入力**: Port *port（接続情報を保持する構造体へのポインタ）
  • **出力**: bool（GSSAPI認証済みであれば true、それ以外は false）

前提条件と不変条件

  • port が NULL の場合、または port->gss が初期化されていない場合は false を返します。

アルゴリズムまたは振る舞い

1. 引数 port が有効かつ port->gss が存在するかを確認します。 2. 条件を満たさない場合は直ちに false を返します。 3. 条件を満たす場合、port->gss->auth の値を評価して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/backend_status.c: バックエンドのステータス情報更新時に使用。
  • src/backend/utils/init/postinit.c: 初期化プロセスにおける認証状態のログ出力に使用。
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行いません。

関連データ構造

  • Port: libpq-be.h で定義される接続状態を保持する構造体。gss メンバを通じてGSSAPI関連の状態にアクセスします。

失敗モードと境界条件

  • port が NULL の場合、セグメンテーションフォールトを回避し安全に false を返します。
  • port->gss が未初期化の場合も同様に false を返します。

保守担当者向けメモ

  • 本関数は port->gss->auth の値を直接参照するゲッターです。GSSAPI認証状態の定義が変更された場合は、この構造体メンバの整合性に注意してください。
