ecpg_type_infocache_push

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:147

目的

PostgreSQL の型情報（OID および配列型フラグ）をキャッシュリストの先頭に挿入し、型情報の検索効率を向上させるために使用されます。

シグネチャまたは形状

bool ecpg_type_infocache_push(struct ECPGtype_information_cache **cache, int oid, enum ARRAY_TYPE isarray, int lineno)

入力と出力

  • **入力**:
  • cache: キャッシュリストのヘッドポインタへのポインタ。
  • oid: キャッシュ対象の PostgreSQL 型 OID。
  • isarray: ARRAY_TYPE 列挙型による配列状態の指定。
  • lineno: エラー発生時に使用されるソースコード行番号。
  • **出力**:
  • 成功時は true、メモリ確保に失敗した場合は false を返します。

前提条件と不変条件

  • cache が指すポインタは有効である必要があります。
  • ecpg_alloc によるメモリ確保が成功することが前提です。
  • リストはスタック形式（先頭挿入）で管理されます。

アルゴリズムまたは振る舞い

1. ecpg_alloc を呼び出し、ECPGtype_information_cache 構造体分のメモリを確保します。 2. 確保に失敗した場合は false を返します。 3. 成功した場合、引数 oid および isarray を新しいエントリに設定します。 4. 新しいエントリの next ポインタを現在のリストの先頭（*cache）に向けます。 5. リストのヘッドポインタ *cache を新しいエントリで更新し、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/execute.c 内の各箇所から、stmt->connection->cache_head を引数として呼び出されています（例: BOOLOID, BYTEAOID, INT4OID 等の型登録時）。
  • **呼び出し先**: ecpg_alloc（メモリ確保用）。

関連データ構造

  • ECPGtype_information_cache: 型情報（OID, 配列フラグ）と次要素へのポインタを保持する構造体。
  • ARRAY_TYPE: 配列の状態を示す列挙型。

失敗モードと境界条件

  • ecpg_alloc が NULL を返した場合、関数は即座に false を返し、リストの更新は行われません。

保守担当者向けメモ

  • 本関数はリストの先頭に要素を追加する単純なプッシュ操作です。リストの解放処理については、対応する free 関数が別途存在するか確認してください。
  • 呼び出し元で stmt->connection->cache_head を渡しているため、接続ごとのキャッシュ管理に依存しています。
