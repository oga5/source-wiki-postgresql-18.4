get_non_null_list_datum_count

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partbounds.c:435

目的

リストパーティションの定義において、NULL値を除外したデータ値（datum）の総数をカウントします。

シグネチャまたは形状

static int get_non_null_list_datum_count(PartitionBoundSpec **boundspecs, int nparts)

入力と出力

  • **入力**:
  • boundspecs: PartitionBoundSpec 構造体の配列。各要素は個別のパーティションの境界定義を保持します。
  • nparts: パーティションの総数。
  • **出力**:
  • int: 全パーティションに含まれる非NULLのリストデータ値の合計数。

前提条件と不変条件

  • boundspecs 配列は nparts 個の有効な要素を指している必要があります。
  • 各 boundspecs[i]->listdatums は Const ノードのリストであると想定されています。

アルゴリズムまたは振る舞い

1. count を 0 に初期化します。 2. nparts 回ループし、各パーティションの listdatums リストを走査します。 3. 各リスト要素（Const ノード）の constisnull フラグを確認します。 4. constisnull が偽（非NULL）である場合のみ、count をインクリメントします。 5. 全ての走査終了後、最終的な count を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/partitioning/partbounds.c 内の create_list_bounds 関数（482行目付近）から呼び出されます。
  • **呼び出し先**: lfirst_node マクロを使用してリスト内のノードにアクセスします。

関連データ構造

  • PartitionBoundSpec: パーティションの境界定義を保持する構造体。
  • Const: PostgreSQLの定数ノード。constisnull フィールドによりNULL判定を行います。

失敗モードと境界条件

  • nparts が 0 の場合、ループは実行されず 0 を返します。
  • listdatums が空のリストである場合、そのパーティションについてはカウントされません。

保守担当者向けメモ

  • 本関数は create_list_bounds において、必要なメモリ領域を事前に確保するためのサイズ計算に使用されています。
  • Const ノード以外の型がリストに含まれる可能性については、現在の実装では考慮されていません。
