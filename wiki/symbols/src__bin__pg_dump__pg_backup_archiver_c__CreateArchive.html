CreateArchive

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:229

目的

ダンプまたは復元処理のためのアーカイブハンドル（Archive）を初期化し、生成します。

シグネチャまたは形状

Archive *CreateArchive(const char *FileSpec, const ArchiveFormat fmt,
                       const pg_compress_specification compression_spec,
                       bool dosync, ArchiveMode mode,
                       SetupWorkerPtrType setupDumpWorker,
                       DataDirSyncMethod sync_method);

入力と出力

  • **入力**:
  • FileSpec: アーカイブのファイルパスまたは識別子。
  • fmt: アーカイブ形式（ArchiveFormat）。
  • compression_spec: 圧縮設定。
  • dosync: 同期書き込みを行うかどうかのフラグ。
  • mode: アーカイブの操作モード（ArchiveMode）。
  • setupDumpWorker: 並列ダンプ用ワーカーのセットアップ関数ポインタ。
  • sync_method: データディレクトリの同期手法。
  • **出力**:
  • 初期化された Archive 構造体へのポインタ。

前提条件と不変条件

  • 内部で _allocAH を呼び出し、メモリ確保と構造体の初期化を行います。
  • 戻り値は ArchiveHandle 型から Archive 型へキャストされます。

アルゴリズムまたは振る舞い

1. 引数として受け取った各種設定を _allocAH 関数に渡します。 2. _allocAH が生成した ArchiveHandle 構造体を受け取ります。 3. 取得したハンドルを Archive 型にキャストして呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c:975（fout の初期化に使用）。
  • **呼び出し先**: _allocAH（同ファイル内の内部関数）。

関連データ構造

  • Archive: アーカイブの抽象インターフェース。
  • ArchiveHandle: アーカイブの内部状態を保持するメイン構造体。
  • ArchiveFormat: アーカイブ形式を定義する列挙型。

失敗モードと境界条件

  • _allocAH 内でのメモリ確保失敗や、無効な引数による初期化失敗が発生する可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な初期化ロジックは _allocAH に依存しています。
  • アーカイブ形式ごとの具体的な初期化処理を確認する場合は、_allocAH 内で呼び出される各フォーマット（custom, tar, directory）の初期化関数を参照してください。
