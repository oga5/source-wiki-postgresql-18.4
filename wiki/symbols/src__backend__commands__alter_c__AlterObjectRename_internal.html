AlterObjectRename_internal

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/alter.c:164

目的

PostgreSQLのシステムカタログに登録された汎用オブジェクトの名前変更処理を実行する内部関数です。カタログの属性情報を動的に取得し、権限チェック、重複チェック、およびカタログタプルの更新を行います。

シグネチャまたは形状

static void AlterObjectRename_internal(Relation rel, Oid objectId, const char *new_name)

入力と出力

  • **入力**:
  • rel: 対象オブジェクトが格納されているシステムカタログの Relation。
  • objectId: 変更対象オブジェクトの Oid。
  • new_name: 新しいオブジェクト名。
  • **出力**: なし（成功時はカタログを更新し、失敗時はエラーをスロー）。

前提条件と不変条件

  • rel は有効なシステムカタログの Relation であること。
  • objectId に対応するタプルが rel 内に存在すること。
  • 呼び出し元で適切なロックが取得されていること。
  • オブジェクトが所有者を持つ場合、実行ユーザーは所有者であるか、スーパーユーザーである必要がある。
  • 名前空間（スキーマ）に属するオブジェクトの場合、実行ユーザーは対象名前空間に対する CREATE 権限が必要。

アルゴリズムまたは振る舞い

1. **タプル取得**: SearchSysCache1 を使用して対象オブジェクトのタプルを取得。 2. **権限検証**: スーパーユーザーでない場合、所有権チェックおよび名前空間に対する CREATE 権限チェックを実行。Subscription の場合は追加のデータベース権限チェックとパスワード要件の検証を行う。 3. **重複チェック**: オブジェクト型に応じて、名前空間内での名前の衝突をチェック（IsThere... 系関数や SearchSysCacheExists を使用）。 4. **タプル更新**: heap_modify_tuple を用いて名前属性を更新し、CatalogTupleUpdate でカタログへ書き込む。 5. **フック実行**: InvokeObjectPostAlterHook を呼び出し、イベントトリガーやキャッシュ無効化等の後処理を行う。 6. **後処理**: Publication 等、特定のオブジェクト型に対して必要な追加処理を実行。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/alter.c 内の RENAME ステートメント処理フローから呼び出される。
  • **呼び出し先**: SearchSysCache1, heap_getattr, superuser, has_privs_of_role, object_aclcheck, IsThere... 系関数, heap_modify_tuple, CatalogTupleUpdate, InvokeObjectPostAlterHook 等。

関連データ構造

  • Relation: システムカタログの構造体。
  • HeapTuple: カタログタプル。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • **キャッシュルックアップ失敗**: objectId が存在しない場合、elog(ERROR) を発生。
  • **権限不足**: 所有権がない、または名前空間への CREATE 権限がない場合、aclcheck_error を発生。
  • **名前の衝突**: 同一名前空間内に同名のオブジェクトが存在する場合、report_name_conflict または report_namespace_conflict を経由してエラーを発生。
  • **サブスクリプション制約**: password_required=false のサブスクリプションを非スーパーユーザーが変更しようとした場合、エラーを発生。

保守担当者向けメモ

  • 本関数は get_object_attnum_... 系関数を使用して属性番号を動的に解決するため、新しいオブジェクト型をサポートする場合は、それらの関数が対象カタログに対して正しく実装されている必要があります。
  • Subscription や Procedure 等、型固有のチェックロジックがハードコードされているため、複雑なオブジェクトを追加する際は条件分岐の肥大化に注意してください。
