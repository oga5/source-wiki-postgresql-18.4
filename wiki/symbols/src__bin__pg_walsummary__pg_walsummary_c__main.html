目的

pg_walsummary コマンドラインユーティリティのエントリーポイントとして、指定された WAL サマリファイルを解析し、含まれるブロック参照情報を人間が読める形式で出力する。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。オプションとして --individual (-i) および --quiet (-q) を受け付け、引数の末尾に解析対象のファイルパスを指定する。
  • **出力**:
  • 成功時: 0 を返して終了。
  • 失敗時: 1 を返して終了（引数エラーやファイルオープン失敗時）。

前提条件と不変条件

  • argv[0] はプログラム名として適切に初期化されている必要がある。
  • 引数として少なくとも1つのファイルパスが指定されている必要がある。
  • 指定されたファイルは BlockRefTableReader が読み取り可能なバイナリ形式であること。

アルゴリズムまたは振る舞い

1. **初期化**: ロギング、ロケール、ヘルプオプションの初期化を行う。 2. **引数解析**: getopt_long を使用してオプションを解析し、ws_options 構造体に状態を格納する。 3. **ファイル処理**: 指定された各ファイルに対して以下のループを実行する。

  • ファイルを O_RDONLY モードでオープンする。
  • CreateBlockRefTableReader を呼び出し、walsummary_read_callback を用いてリーダーを初期化する。
  • BlockRefTableReaderNextRelation を用いてリレーション単位で反復処理を行い、dump_one_relation を呼び出して各リレーションのブロック情報を出力する。
  • DestroyBlockRefTableReader でリーダーを解放し、ファイルをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（エントリーポイントのため）。
  • **呼び出し先**:
  • pg_logging_init, get_progname, set_pglocale_pgservice, handle_help_version_opts
  • getopt_long
  • CreateBlockRefTableReader, BlockRefTableReaderNextRelation, DestroyBlockRefTableReader
  • dump_one_relation
  • open, close
  • pg_log_error, pg_fatal, exit
  • 注記: ツールが提示した contrib/amcheck 関連の呼び出し元情報は、本シンボルとは無関係なコードベース上の言及であり、本関数の呼び出し元ではない。

関連データ構造

  • ws_options: 実行時オプションを保持。
  • ws_file_info: ファイル記述子とファイル名を保持。
  • BlockRefTableReader: サマリファイル読み込み用のリーダー構造体。
  • RelFileLocator: リレーションの識別情報。

失敗モードと境界条件

  • 引数なしで実行された場合、エラーメッセージを表示して終了する。
  • ファイルのオープンに失敗した場合、pg_fatal によりエラーを出力して終了する。
  • 不正なオプションが指定された場合、使用方法を表示して終了する。

保守担当者向けメモ

  • 本関数は BlockRefTableReader のクライアント実装として機能する。リーダーのインターフェース変更時には、CreateBlockRefTableReader の引数やループ処理の整合性に注意すること。
  • ws_file_info はコールバック関数間で共有されるため、構造体のライフサイクル管理に留意すること。
