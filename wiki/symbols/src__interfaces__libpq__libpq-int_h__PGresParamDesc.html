PGresParamDesc

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-int.h:107

目的

クエリ実行時に使用されるパラメータの型情報を保持するための内部構造体です。

シグネチャまたは形状

typedef struct pgresParamDesc
{
    Oid typid; /* type id */
} PGresParamDesc;

入力と出力

  • **入力**: typid（PostgreSQL のデータ型を識別する OID）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • PGresult 構造体の一部として動的に割り当てられます。
  • fe-protocol3.c において、パラメータ数（nparams）に基づいたメモリ領域が確保されます。

アルゴリズムまたは振る舞い

  • fe-protocol3.c 内で、クエリ実行結果に関連付けられたパラメータ記述の配列として初期化されます。
  • pqResultAlloc を介してメモリが確保され、MemSet によってゼロ初期化されることが確認されています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c（パラメータ記述の割り当ておよび初期化）。
  • **呼び出し先**: 情報なし。

関連データ構造

  • PGresult: 本構造体をメンバ（paramDescs）として保持するクエリ結果構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時: pqResultAlloc が失敗した場合、呼び出し元である fe-protocol3.c のロジックに依存します。

保守担当者向けメモ

  • 本構造体は libpq 内部専用です。公開 API には含まれていません。
  • PGresult のライフサイクル管理（pqResultAlloc）と密接に関連しているため、メモリ管理の変更時には注意が必要です。
