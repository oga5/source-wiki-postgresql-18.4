pgstat_drop_replslot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_replslot.c:155

目的

レプリケーションスロットの削除に伴い、対応する統計エントリを統計サブシステムから削除します。

シグネチャまたは形状

void pgstat_drop_replslot(ReplicationSlot *slot)

入力と出力

  • **入力**: slot (削除対象の ReplicationSlot 構造体へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • 呼び出し元は ReplicationSlotAllocationLock を排他モード（LW_EXCLUSIVE）で保持している必要があります。

アルゴリズムまたは振る舞い

1. ReplicationSlotAllocationLock が排他ロックされていることを Assert で確認します。 2. pgstat_drop_entry を呼び出し、PGSTAT_KIND_REPLSLOT 種別かつ対象スロットのインデックス（ReplicationSlotIndex(slot)）に対応する統計エントリの削除を試みます。 3. pgstat_drop_entry が偽を返した場合（エントリが即座に削除できなかった場合）、pgstat_request_entry_refs_gc() を呼び出し、統計エントリ参照のガベージコレクションを要求します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/slot.c:1069
  • **呼び出し先**:
  • pgstat_drop_entry (統計サブシステム内部関数)
  • pgstat_request_entry_refs_gc (統計サブシステム内部関数)
  • ReplicationSlotIndex (マクロ)

関連データ構造

  • ReplicationSlot: レプリケーションスロットの管理構造体。
  • PGSTAT_KIND_REPLSLOT: 統計種別を示す定数。

失敗モードと境界条件

  • ロックが保持されていない場合、Assert によりアボートします。
  • 統計エントリが存在しない場合、pgstat_drop_entry は偽を返し、ガベージコレクション要求が発行されます。

保守担当者向けメモ

  • 本関数はレプリケーションスロットのライフサイクル管理と統計サブシステムの同期を担います。
  • スロットのインデックスは再起動時に変化する可能性があるため、統計の識別には ReplicationSlotIndex を使用しています。
