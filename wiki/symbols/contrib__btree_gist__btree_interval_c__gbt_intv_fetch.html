gbt_intv_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_interval.c:176

目的

GiST インデックス内の interval 型データに対して、インデックスエントリのフェッチ（取得）処理を提供します。

シグネチャまたは形状

Datum gbt_intv_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、gbt_num_fetch が返すフェッチ結果を返します。

前提条件と不変条件

  • tinfo（gbtree_ninfo 構造体）が contrib/btree_gist/btree_interval.c 内で適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. gbt_num_fetch 関数を呼び出し、entry と tinfo を渡して処理を委譲します。 3. gbt_num_fetch の戻り値を PG_RETURN_POINTER マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックスフレームワークから呼び出されます（直接的な呼び出し元コードの詳細はソース内に明示されていません）。
  • **呼び出し先**: gbt_num_fetch（btree_utils_num.h で定義されたユーティリティ関数）。

関連データ構造

  • GISTENTRY: GiST インデックスエントリを保持する構造体。
  • tinfo: interval 型の比較や操作を定義した gbtree_ninfo 構造体。

失敗モードと境界条件

  • gbt_num_fetch の内部実装に依存します。入力された entry が無効な場合や、メモリ不足等のシステムエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.h で提供される汎用関数 gbt_num_fetch のラッパーです。ロジックの変更が必要な場合は、まず gbt_num_fetch の仕様を確認してください。
  • PG_FUNCTION_INFO_V1(gbt_intv_fetch) により、PostgreSQL の関数マネージャに登録されています。
