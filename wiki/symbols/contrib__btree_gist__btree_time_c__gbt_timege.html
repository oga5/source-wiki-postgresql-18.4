gbt_timege

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_time.c:52

目的

PostgreSQLのGiSTインデックスにおいて、2つのTimeADT値の「以上（greater than or equal to）」比較を行うためのサポート関数です。

シグネチャまたは形状

static bool gbt_timege(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象のTimeADT値へのポインタ。
  • b: 比較対象のTimeADT値へのポインタ。
  • flinfo: 関数マネージャ情報（本関数内では直接使用されず、DirectFunctionCall2へ渡される）。
  • **出力**:
  • bool: *a >= *b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な TimeADT 型のデータ領域を指している必要があります。
  • 内部的に time_ge 関数を呼び出すため、PostgreSQLの型システムにおける time 型の比較規則に従います。

アルゴリズムまたは振る舞い

1. 入力された void * 型のポインタを TimeADT * 型にキャストします。 2. TimeADTGetDatumFast マクロを使用して、各値を Datum 型に変換します。 3. DirectFunctionCall2 を介して PostgreSQL のコア関数 time_ge を呼び出し、比較結果を取得します。 4. 取得した結果を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_time.c 内のテーブル定義（gbtree_ninfo 構造体の一部）から参照されています。
  • **呼び出し先**:
  • time_ge (PostgreSQLコア関数)
  • TimeADTGetDatumFast (マクロ)
  • DatumGetBool (マクロ)

関連データ構造

  • TimeADT: PostgreSQLの内部的な時間表現型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーションフォールトが発生する可能性があります。
  • time_ge 関数が内部でエラーを発生させた場合、PostgreSQLのエラーハンドリング機構により処理が中断されます。

保守担当者向けメモ

  • 本関数は btree_utils_num.c が提供する数値処理ユーティリティと連携するためのラッパーです。
  • 比較ロジックを変更する場合は、コアの time_ge 関数の仕様を確認してください。
