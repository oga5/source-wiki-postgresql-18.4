pg_decode_shutdown

定義

  • 種別: function
  • 定義箇所: contrib/test_decoding/test_decoding.c:279

目的

論理デコーディングプラグインの終了処理として、プラグインが確保したメモリリソースを解放する。

シグネチャまたは形状

static void pg_decode_shutdown(LogicalDecodingContext *ctx)

入力と出力

  • **入力**: LogicalDecodingContext *ctx - 論理デコーディングのコンテキスト構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • ctx->output_plugin_private に TestDecodingData 構造体が格納されていること。
  • data->context が有効なメモリコンテキストとして初期化されていること。

アルゴリズムまたは振る舞い

1. ctx->output_plugin_private からプラグイン固有のデータ構造 TestDecodingData を取得する。 2. TestDecodingData 内の context メンバに対して MemoryContextDelete を実行し、プラグインが使用していたメモリコンテキストを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの論理デコーディングエンジン（_PG_output_plugin_init で shutdown_cb として登録される）。
  • **呼び出し先**: MemoryContextDelete（PostgreSQLメモリ管理API）。

関連データ構造

  • LogicalDecodingContext: 論理デコーディングの実行状態を保持する構造体。
  • TestDecodingData: プラグインの設定やメモリコンテキストを保持する構造体。

失敗モードと境界条件

  • ctx が NULL の場合、あるいは ctx->output_plugin_private が初期化されていない場合は異常終了する可能性がある。

保守担当者向けメモ

  • 本関数はプラグインの終了時に一度だけ呼び出されることを想定している。
  • メモリコンテキストの削除により、TestDecodingData 自体も解放されるため、本関数の実行後に data ポインタへアクセスしてはならない。
