heap_insert

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/heap/heapam.c:2079

目的

リレーション（テーブル）に対して単一のタプルを挿入します。

シグネチャまたは形状

void heap_insert(Relation relation, HeapTuple tup, CommandId cid, int options, BulkInsertState bistate)

入力と出力

  • **入力**:
  • relation: 挿入先のリレーション。
  • tup: 挿入するタプル（HeapTuple）。
  • cid: 現在のコマンドID。
  • options: 挿入オプション（HEAP_INSERT_SPECULATIVE 等）。
  • bistate: 一括挿入用状態（BulkInsertState）。
  • **出力**: なし（副作用としてリレーションにタプルが追加されます）。

前提条件と不変条件

  • 呼び出し前に AssertHasSnapshotForToast(relation) が満たされている必要があります。
  • タプルの属性数がリレーションの定義を超えていないことを前提とします。
  • クリティカルセクション内では ereport(ERROR) を発生させてはなりません。

アルゴリズムまたは振る舞い

1. **準備**: heap_prepare_insert を呼び出し、タプルのヘッダー設定および必要に応じたTOAST処理を行います。 2. **バッファ確保**: RelationGetBufferForTuple を使用し、挿入可能な空き領域を持つページバッファを確保します。 3. **競合チェック**: CheckForSerializableConflictIn により、シリアライザブル隔離レベルでの競合をチェックします。 4. **物理書き込み**: クリティカルセクションを開始し、RelationPutHeapTuple でページにタプルを書き込みます。 5. **可視性マップ更新**: ページが「全可視（All-Visible）」状態であれば、フラグをクリアし可視性マップを更新します。 6. **WAL記録**: リレーションがWALを必要とする場合、XLogInsert を介して挿入操作を記録します。 7. **終了処理**: バッファのロック解除、キャッシュ無効化（CacheInvalidateHeapTuple）、統計情報の更新（pgstat_count_heap_insert）を行い、必要に応じて一時タプルのメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/toast_internals.c
  • src/backend/access/heap/heapam.c（内部のラッパー関数等）
  • **呼び出し先**:
  • heap_prepare_insert, RelationGetBufferForTuple, CheckForSerializableConflictIn, RelationPutHeapTuple, PageClearAllVisible, visibilitymap_clear, MarkBufferDirty, XLogInsert, CacheInvalidateHeapTuple, pgstat_count_heap_insert, heap_freetuple 等。

関連データ構造

  • Relation: ターゲットとなるリレーション情報。
  • HeapTuple: 挿入対象のタプルデータ。
  • Buffer: 物理ページを保持するバッファ。
  • BulkInsertState: 一括挿入時の状態管理。

失敗モードと境界条件

  • 挿入時にページ容量を超える場合、TOAST処理が実行されます。
  • 投機的挿入（HEAP_INSERT_SPECULATIVE）の場合、WALレコードにその旨が記録されます。
  • ページが初期化直後の場合、XLOG_HEAP_INIT_PAGE フラグが使用されます。

保守担当者向けメモ

  • 本関数はクリティカルセクションを伴うため、メモリ割り当てやエラー発生の可能性がある処理は START_CRIT_SECTION() 前に完了させる必要があります。
  • 投機的挿入の取り消しやロールバックが発生した場合でも、統計情報（pgstat）はカウントされる仕様です。
