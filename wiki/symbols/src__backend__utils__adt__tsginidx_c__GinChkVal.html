GinChkVal

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsginidx.c:172

目的

GINインデックスの検索時、tsqueryの論理条件とインデックス内のキー状態を照合するためのコンテキスト情報を保持する構造体です。

シグネチャまたは形状

typedef struct
{
	QueryItem  *first_item;
	GinTernaryValue *check;
	int		   *map_item_operand;
} GinChkVal;

入力と出力

本構造体は関数間で受け渡されるコンテキストとして機能します。

  • first_item: 評価対象となるtsqueryのアイテム配列の先頭ポインタ。
  • check: 各クエリ条件の真偽状態（GinTernaryValue）を格納する配列。
  • map_item_operand: クエリのアイテムとインデックスのオペランドを対応付けるためのマッピング情報。

前提条件と不変条件

  • check 配列は、対応するクエリのアイテム数分以上の要素を持つ必要があります。
  • first_item は有効な QueryItem 配列を指している必要があります。

アルゴリズムまたは振る舞い

GINインデックスの整合性チェック（consistent関数）において、TS_execute_ternary等の評価関数へ渡すための補助データとして使用されます。インデックスから得られたキーの存在有無（check）と、クエリの論理構造（first_item）を組み合わせて、最終的な検索結果の合致判定を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsginidx.c 内の gin_tsquery_consistent および gin_tsquery_triconsistent 等からインスタンス化されます。また、contrib/intarray/_int_bool.c でも同様の目的で使用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、これを引数として受け取る checkcondition_gin 等のコールバック関数へ渡されます。

関連データ構造

  • QueryItem: tsquery の論理構造を構成する要素。
  • GinTernaryValue: GINインデックスにおけるキーの存在状態（TRUE, FALSE, MAYBE）。

失敗モードと境界条件

  • check や first_item が NULL の状態で評価関数に渡された場合、セグメンテーションフォールトが発生する可能性があります。
  • クエリの複雑さに対して map_item_operand の整合性が取れていない場合、誤った判定結果を返す可能性があります。

保守担当者向けメモ

  • 本構造体は tsginidx.c だけでなく contrib/intarray でも利用されているため、変更を加える際は両方のモジュールへの影響を確認してください。
  • checkcondition_gin 関数内でこの構造体がどのように参照されているかを追跡することで、インデックスの評価ロジックを理解できます。
