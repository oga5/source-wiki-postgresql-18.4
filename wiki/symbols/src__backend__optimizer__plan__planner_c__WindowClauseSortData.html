WindowClauseSortData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/optimizer/plan/planner.c:115

目的

ウィンドウ関数処理において、複数の WindowClause をソートまたは比較するための補助データ構造です。ウィンドウ定義と、それに関連する一意な順序付け/パーティショニング句を保持します。

シグネチャまたは形状

typedef struct
{
	WindowClause *wc;
	List	   *uniqueOrder;
} WindowClauseSortData;

入力と出力

  • **入力**: WindowClause 構造体へのポインタおよび、ウィンドウごとの一意な順序付け/パーティショニング句のリスト。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • wc は有効な WindowClause を指している必要があります。
  • uniqueOrder はウィンドウの順序付けまたはパーティショニングに関連するリストを保持します。

アルゴリズムまたは振る舞い

  • 本構造体は、ウィンドウ句のソート順序を決定するための比較処理（common_prefix_cmp）において、比較対象の要素として使用されます。
  • planner.c 内で palloc を用いて配列として確保され、qsort を介してウィンドウ句の順序を最適化するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c 内のウィンドウ処理ロジック（6238 行目付近でメモリ確保、6292 行目で qsort 呼び出し）。
  • **呼び出し先**: 比較関数 common_prefix_cmp が本構造体を受け取ります（6351 行目以降の定義）。

関連データ構造

  • WindowClause: ウィンドウ関数の定義を保持するノード。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • wc が NULL の場合、比較関数内で未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は planner.c 内のローカルなソート処理に特化しています。ウィンドウ関数の最適化ロジックを変更する際は、common_prefix_cmp との整合性に注意してください。
