ArrayBuildStateArr

定義

  • 種別: struct
  • 定義箇所: src/include/utils/array.h:205

目的

PostgreSQLにおいて、配列を逐次的に構築する際の作業用状態を保持するための構造体です。

シグネチャまたは形状

typedef struct ArrayBuildStateArr
{
	MemoryContext mcontext;
	char	   *data;
	bits8	   *nullbitmap;
	int			abytes;
	int			nbytes;
	int			aitems;
	int			nitems;
	int			ndims;
	int			dims[MAXDIM];
	int			lbs[MAXDIM];
	Oid			array_type;
	Oid			element_type;
	bool		private_cxt;
} ArrayBuildStateArr;

入力と出力

  • **入力**: 配列構築時に必要なメモリコンテキスト、対象となる配列型および要素型のOID。
  • **出力**: 構築された配列データ（data）、NULLビットマップ、次元情報、および現在の構築進捗状況。

前提条件と不変条件

  • data は構築中の配列要素を格納するバッファを指します。
  • nullbitmap はNULL要素が存在する場合に使用され、存在しない場合は NULL となります。
  • dims および lbs は MAXDIM（最大次元数）までの次元情報と下限値を保持します。
  • mcontext は構築に使用されるメモリコンテキストを指します。

アルゴリズムまたは振る舞い

  • 配列構築の過程で、要素が追加されるたびに nbytes（使用済みバイト数）や nitems（要素数）が更新されます。
  • abytes および aitems は、メモリ再割り当てを最小限にするための確保済み容量を管理します。
  • private_cxt が真の場合、この構造体は専用のメモリコンテキストを管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_userfuncs.c 内の複数の関数（例: 938行目、961行目、988行目など）で、PG_GETARG_POINTER を介して取得・参照されています。
  • **呼び出し先**: 構造体内部の関数ポインタや特定の関数呼び出しに関する直接的な情報は、提供されたソース excerpt には含まれていません。

関連データ構造

  • ArrayBuildState: 配列構築のための別の作業用構造体。
  • MemoryContext: メモリ管理のためのコンテキスト。
  • Oid: 配列型および要素型を識別するための型ID。

失敗モードと境界条件

  • MAXDIM を超える次元数の指定は境界条件となります。
  • メモリ割り当て失敗時は MemoryContext の仕様に従いエラーが発生します。
  • dataoffset が 0 の場合、NULLビットマップは生成されません。

保守担当者向けメモ

  • 本構造体は array_userfuncs.c においてシリアライズやデータ操作の対象として頻繁に使用されています。
  • 構造体の各フィールド（特に dims や lbs）の更新ロジックは、配列の次元操作と密接に関連しているため、変更時は arrayfuncs.c の構築ロジックと整合性を確認してください。
