compute_range_stats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_typanalyze.c:124

目的

範囲型（Range Types）およびマルチ範囲型（Multirange Types）カラムに対する統計情報を収集し、プランナが選択率を適切に推定できるようにする。

シグネチャまたは形状

static void compute_range_stats(VacAttrStats *stats, AnalyzeAttrFetchFunc fetchfunc, int samplerows, double totalrows);

入力と出力

  • **入力**:
  • stats: 統計情報を格納する VacAttrStats 構造体。
  • fetchfunc: サンプル行を取得するための関数ポインタ。
  • samplerows: サンプリング対象の行数。
  • totalrows: テーブルの総行数。
  • **出力**: なし（stats 構造体の各フィールドを更新する）。

前提条件と不変条件

  • stats->extra_data には TypeCacheEntry が格納されている必要がある。
  • 範囲型またはマルチ範囲型として正しく初期化されていること。
  • non_null_cnt > 0 の場合にのみ、統計情報（stanullfrac, stawidth, stadistinct 等）が有効化される。

アルゴリズムまたは振る舞い

1. **初期化**: typcache から型情報を取得し、メモリを確保する。 2. **サンプリング**: fetchfunc を使用して各行を走査し、範囲型をデシリアライズする。

  • マルチ範囲型の場合は、全体を包含する最小の範囲として扱う。
  • empty な範囲とそれ以外を分類し、境界値と長さを抽出する。

    • 3. **統計計算**:

  • NULL率、平均幅、空範囲の割合を算出する。
  • non_empty_cnt >= 2 の場合、境界値（下限・上限）をソートし、ヒストグラム用のビンを生成する。
  • 整数オーバーフローを避けるため、ヒストグラムのインデックス計算には加算ロジックを用いる。

    • 4. **格納**: 計算結果を VacAttrStats 構造体に書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • range_typanalyze (統計計算関数として登録)
  • multirange_typanalyze (統計計算関数として登録)
  • **呼び出し先**:
  • vacuum_delay_point
  • fetchfunc
  • multirange_get_bounds / range_deserialize
  • FunctionCall2Coll (サブ差分計算用)
  • qsort_interruptible (ソート用)

関連データ構造

  • VacAttrStats: ANALYZE処理の統計情報を保持する構造体。
  • TypeCacheEntry: 型のキャッシュ情報。
  • RangeBound: 範囲の境界値を保持する構造体。

失敗モードと境界条件

  • **NULL値**: NULL値は個別にカウントされ、統計計算から除外される。
  • **空範囲**: empty な範囲は長さの計算から除外されるが、empty_cnt として記録される。
  • **無限大**: 無限大を含む範囲の長さは get_float8_infinity() を使用して無限大として扱われる。
  • **整数オーバーフロー**: ヒストグラムのビン計算において、attstattarget が大きい場合に発生する可能性があるため、インデックス計算に工夫がなされている。

保守担当者向けメモ

  • 範囲型のヒストグラムは、下限と上限のペアを保持する特殊な形式である。
  • subdiff 関数が定義されていない型の場合は、長さのデフォルト値として 1.0 が使用される。
  • ソート処理には qsort_interruptible を使用し、長時間実行時の割り込みに対応している。
