check_log_timezone

定義

  • 種別: function
  • 定義箇所: src/backend/commands/variable.c:417

目的

GUC（Grand Unified Configuration）パラメータ log_timezone の設定値が有効なタイムゾーン名であるかを検証し、適用フェーズへ渡すためのデータを準備します。

シグネチャまたは形状

bool check_log_timezone(char **newval, void **extra, GucSource source);

入力と出力

  • **入力**:
  • newval: 設定しようとしているタイムゾーン名の文字列ポインタ。
  • extra: 適用フック（assign_log_timezone）へ受け渡すためのデータ格納領域。
  • source: 設定のソース（GUCフレームワークより提供）。
  • **出力**:
  • 成功時は true を返し、extra に解決された pg_tz 構造体へのポインタを格納します。
  • 失敗時は false を返します。

前提条件と不変条件

  • newval は有効なタイムゾーン名として解決可能である必要があります。
  • 閏秒を含むタイムゾーンはサポートされません。

アルゴリズムまたは振る舞い

1. pg_tzset(*newval) を呼び出し、入力文字列からタイムゾーン構造体を取得します。解決できない場合は false を返します。 2. pg_tz_acceptable(new_tz) を呼び出し、閏秒の使用有無を確認します。閏秒を使用するタイムゾーンの場合はエラーメッセージを記録し、false を返します。 3. 検証成功時、guc_malloc を使用して pg_tz * を格納するためのメモリを確保し、extra に設定して true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GUCサブシステム（src/backend/utils/misc/guc_tables.c にてフックとして登録）。
  • **呼び出し先**:
  • pg_tzset（タイムゾーン解決）
  • pg_tz_acceptable（閏秒チェック）
  • GUC_check_errmsg / GUC_check_errdetail（エラー報告）
  • guc_malloc（メモリ確保）

関連データ構造

  • pg_tz: タイムゾーン情報を保持する構造体。
  • extra: assign_log_timezone へ渡される中間データ。

失敗モードと境界条件

  • タイムゾーン名が不正な場合（pg_tzset が NULL を返す）。
  • タイムゾーンが閏秒を使用している場合（pg_tz_acceptable が偽を返す）。
  • メモリ確保（guc_malloc）に失敗した場合。

保守担当者向けメモ

  • 本関数は log_timezone の検証専用です。同様のロジックを timezone パラメータで再利用する場合は、共通化の検討が必要です。
  • extra に割り当てられたメモリはGUCサブシステムによって管理されるため、呼び出し側での解放は不要です。
