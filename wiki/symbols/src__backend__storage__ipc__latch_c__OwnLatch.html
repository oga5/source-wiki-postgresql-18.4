OwnLatch

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/latch.c:125

目的

共有ラッチ（Latch）を現在のプロセスに関連付け、所有権を割り当てるために使用されます。

シグネチャまたは形状

void OwnLatch(Latch *latch)

入力と出力

  • **入力**: latch - 所有権を設定する対象の Latch 構造体へのポインタ。
  • **出力**: なし。成功時は対象ラッチの owner_pid が現在のプロセス ID (MyProcPid) に更新されます。

前提条件と不変条件

  • 対象のラッチは共有ラッチ（latch->is_shared が真）である必要があります。
  • 本関数はロックを伴わないため、呼び出し側で排他制御を行う必要があります。
  • ラッチが既に他のプロセスによって所有されている場合（owner_pid != 0）、パニックが発生します。

アルゴリズムまたは振る舞い

1. latch->is_shared が真であることを Assert で検証します。 2. 対象ラッチの現在の所有者 PID を確認します。 3. 所有者が既に存在する場合（0 以外）、elog(PANIC) を呼び出しプロセスを終了させます。 4. 所有者が存在しない場合、latch->owner_pid に現在のプロセス ID (MyProcPid) を代入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c:555
  • src/backend/storage/lmgr/proc.c:536
  • src/backend/storage/lmgr/proc.c:706
  • **呼び出し先**:
  • elog (エラー発生時)

関連データ構造

  • Latch: ラッチの状態を管理する構造体。owner_pid および is_shared フラグを保持します。

失敗モードと境界条件

  • **PANIC**: 対象ラッチが既に他のプロセスによって所有されている場合、elog(PANIC) によりプロセスが異常終了します。
  • **アサーション失敗**: is_shared が偽であるラッチに対して呼び出した場合、デバッグビルドではアサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は共有メモリ上のラッチを扱う際に使用されます。所有権の移譲や初期化のタイミングには十分注意してください。
  • OwnLatch はロックを伴わないため、呼び出し側で適切な同期が取られていることを確認してください。
