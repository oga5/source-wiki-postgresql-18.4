percentile_cont_final_common

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/orderedsetaggs.c:525

目的

連続パーセンタイル（percentile_cont）集約関数の最終計算ロジックを提供し、ソートされたデータセットから指定されたパーセンタイル値に対応する値を算出します。

シグネチャまたは形状

static Datum percentile_cont_final_common(FunctionCallInfo fcinfo, Oid expect_type, LerpFunc lerpfunc)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • expect_type: 期待されるソートカラムの型（Oid）。
  • lerpfunc: 2値間の線形補間を行う関数ポインタ。
  • **出力**:
  • 計算されたパーセンタイル値（Datum）。入力がNULLまたはデータが存在しない場合はNULLを返します。

前提条件と不変条件

  • AggCheckCallContext により、集約コンテキスト内での呼び出しであることが保証されている必要があります。
  • 第2引数（パーセンタイル値）は 0 から 1 の範囲内である必要があります。
  • osastate->qstate->sortColType は expect_type と一致する必要があります。

アルゴリズムまたは振る舞い

1. **検証**: パーセンタイル引数の範囲チェックを行い、範囲外であればエラーを報告します。 2. **状態確認**: osastate->number_of_rows が 0 の場合は NULL を返します。 3. **ソート**: tuplesort を使用し、未ソートであれば tuplesort_performsort を実行、ソート済みであれば tuplesort_rescan を行います。 4. **位置特定**: 指定されたパーセンタイルに基づき、補間に必要な first_row と second_row のインデックスを算出します。 5. **データ抽出**: tuplesort_skiptuples と tuplesort_getdatum を用いて対象行の値を読み取ります。 6. **補間**:

  • first_row と second_row が同一であればその値を返します。
  • 異なる場合は、lerpfunc を使用して線形補間を行い、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/orderedsetaggs.c 内の percentile_cont_float8_final（float8_lerp を指定）
  • src/backend/utils/adt/orderedsetaggs.c 内の percentile_cont_interval_final（interval_lerp を指定）
  • **呼び出し先**:
  • tuplesort_performsort, tuplesort_rescan, tuplesort_skiptuples, tuplesort_getdatum（tuplesort インターフェース）
  • lerpfunc（引数として渡された補間関数）

関連データ構造

  • OSAPerGroupState: 集約グループの実行状態を保持し、tuplesort オブジェクトや行数情報を含みます。
  • FunctionCallInfo: PostgreSQL の関数呼び出し規約に基づく引数コンテナ。

失敗モードと境界条件

  • **エラー**: パーセンタイル値が 0 未満、1 超、または NaN の場合に ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE を発生させます。
  • **NULL処理**: 入力値が NULL の場合や、ソート対象の行が存在しない場合は NULL を返します。
  • **データ欠損**: tuplesort から期待される行が取得できない場合、elog(ERROR, ...) を発生させます。

保守担当者向けメモ

  • 本関数は percentile_cont のコアロジックであり、型に依存しない補間処理を lerpfunc で抽象化しています。
  • tuplesort のリソース管理は呼び出し元の集約ライフサイクルに依存するため、本関数内でのクリーンアップは行われません。
