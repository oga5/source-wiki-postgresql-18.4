CopyToGetRoutine

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyto.c:176

目的

COPY TO 操作の出力形式（CSV、バイナリ、またはテキスト）に基づき、適切な CopyToRoutine インターフェースを選択して返します。

シグネチャまたは形状

static const CopyToRoutine *CopyToGetRoutine(const CopyFormatOptions *opts)

入力と出力

  • **入力**: CopyFormatOptions *opts（COPY コマンドのオプション設定を保持する構造体へのポインタ）
  • **出力**: CopyToRoutine 構造体への定数ポインタ。選択された出力形式に応じた関数ポインタ群を提供します。

前提条件と不変条件

  • opts は有効な CopyFormatOptions 構造体を指している必要があります。
  • 戻り値の CopyToRoutine は、COPY TO の実行フェーズで出力処理を抽象化するために使用されます。

アルゴリズムまたは振る舞い

1. opts->csv_mode が真であれば、CopyToRoutineCSV を返します。 2. opts->binary が真であれば、CopyToRoutineBinary を返します。 3. 上記のいずれでもない場合、デフォルトとして CopyToRoutineText を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyto.c:706（DoCopyTo 関数内にて、cstate->routine の初期化に使用）。
  • **呼び出し先**: 選択された CopyToRoutine 構造体（CopyToRoutineCSV, CopyToRoutineBinary, CopyToRoutineText）を参照します。

関連データ構造

  • CopyFormatOptions: COPY コマンドのオプションを保持。
  • CopyToRoutine: 出力形式ごとの関数ポインタを定義するインターフェース構造体。

失敗モードと境界条件

  • opts が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • 複数のフラグ（例: csv_mode と binary）が同時に真である場合の優先順位は、この関数の条件分岐順序（CSV > Binary > Text）に依存します。

保守担当者向けメモ

  • 新しい出力形式を追加する場合は、CopyToRoutine 構造体に新しい実装を定義し、この関数に条件分岐を追加してください。
