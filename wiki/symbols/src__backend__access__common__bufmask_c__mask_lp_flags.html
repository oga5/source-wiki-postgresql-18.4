mask_lp_flags

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/bufmask.c:94

目的

WAL 整合性テストにおいて、ページ内の全ラインポインタ（ItemId）のフラグを LP_UNUSED に強制設定し、比較時の差異を排除する。

シグネチャまたは形状

void mask_lp_flags(Page page)

入力と出力

  • **入力**: Page page（操作対象のページポインタ）
  • **出力**: なし（引数 page をインプレースで書き換える）

前提条件と不変条件

  • page は有効な PostgreSQL ページ構造を指している必要がある。
  • ページ内のラインポインタ配列に対して反復処理を行うため、ページヘッダーが破損していないことが前提となる。

アルゴリズムまたは振る舞い

1. PageGetMaxOffsetNumber(page) を呼び出し、ページ内の最大オフセット番号を取得する。 2. FirstOffsetNumber から最大オフセットまで順次走査する。 3. 各オフセットに対応する ItemId を取得する。 4. ItemIdIsUsed(itemId) が真（使用中）である場合、その lp_flags を LP_UNUSED に上書きする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistxlog.c
  • src/backend/access/hash/hash_xlog.c
  • src/backend/access/nbtree/nbtxlog.c
  • **呼び出し先**:
  • PageGetMaxOffsetNumber
  • PageGetItemId
  • ItemIdIsUsed
  • OffsetNumberNext

関連データ構造

  • Page: 操作対象のページ構造体。
  • ItemId: ページ内のラインポインタ構造体。

失敗モードと境界条件

  • ページが空の場合、ループは実行されず何も変更されない。
  • ページヘッダーが正しく初期化されていない場合、PageGetMaxOffsetNumber の結果が不正となり、予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はテスト目的の正規化ツールであり、実稼働時のページ更新には使用しないこと。
  • LP_UNUSED への変更は、WAL リプレイ時の比較において、ラインポインタのステータス差異を無視させるために行われる。
