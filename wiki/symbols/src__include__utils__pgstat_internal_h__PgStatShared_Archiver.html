PgStatShared_Archiver

定義

  • 種別: struct
  • 定義箇所: src/include/utils/pgstat_internal.h:361

目的

アーカイバ統計情報を共有メモリ上で管理するための構造体です。統計データの更新とリセット操作を、共有メモリの競合を避けつつ一貫性を持って行うための基盤を提供します。

シグネチャまたは形状

typedef struct PgStatShared_Archiver
{
    LWLock        lock;
    uint32        changecount;
    PgStat_ArchiverStats stats;
    PgStat_ArchiverStats reset_offset;
} PgStatShared_Archiver;

入力と出力

  • **入力**: 統計情報の更新（stats への書き込み）およびリセット操作。
  • **出力**: 共有メモリ上の統計データ（stats）およびリセット基準値（reset_offset）。

前提条件と不変条件

  • lock は reset_offset および stats 内の stat_reset_timestamp を保護します。
  • 統計の読み取り時には changecount プロトコルを使用し、書き込みとの競合を検知して一貫性を保証する必要があります。

アルゴリズムまたは振る舞い

  • **統計更新**: バックエンドは共有メモリ上の stats フィールドを直接更新します。
  • **リセットプロトコル**: 統計のリセット時には lock を取得し、現在の統計値を reset_offset に保存することで、リセット後の差分計算を可能にします。
  • **競合制御**: changecount を用いることで、ロックを最小限に抑えつつ、読み取り側が書き込み中のデータ不整合を検知して再試行できる設計となっています。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/utils/activity/pgstat.c: 共有メモリ上のオフセット計算に使用。
  • src/backend/utils/activity/pgstat_archiver.c: アーカイバ統計の読み書きおよびリセット処理の起点として使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • PgStat_ArchiverStats: アーカイバ統計の本体データ。
  • LWLock: 共有メモリ上の排他制御用ロック。
  • PgStat_ShmemControl: 本構造体を含む統計システム全体のルート構造体。

失敗モードと境界条件

  • 共有メモリへのアクセス時に changecount が不一致となった場合、読み取り側は再試行を行う必要があります。
  • lock の取得に失敗した場合、リセット操作はブロックされます。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、ポインタではなくオフセットや dsa_pointer を介したアクセスが前提となります。
  • リセットプロトコルの詳細は src/backend/utils/activity/pgstat_archiver.c 内のコメントを参照してください。
