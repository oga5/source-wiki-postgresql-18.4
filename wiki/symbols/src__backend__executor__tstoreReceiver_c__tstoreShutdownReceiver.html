tstoreShutdownReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tstoreReceiver.c:205

目的

Tuplestore へのタプル格納処理において、使用済みの一時的なワークスペース、変換マップ、およびスロットリソースを解放する。

シグネチャまたは形状

static void tstoreShutdownReceiver(DestReceiver *self)

入力と出力

  • **入力**: DestReceiver *self（TStoreState 構造体としてキャストして使用される）。
  • **出力**: なし。副作用として TStoreState 内のポインタが解放され、NULL に設定される。

前提条件と不変条件

  • self は TStoreState 型の有効なポインタであること。
  • 本関数呼び出し後、TStoreState 内の outvalues、tofree、tupmap、mapslot はすべて NULL となる。

アルゴリズムまたは振る舞い

1. self を TStoreState *myState にキャストする。 2. myState->outvalues が存在する場合、pfree で解放し NULL に設定する。 3. myState->tofree が存在する場合、pfree で解放し NULL に設定する。 4. myState->tupmap が存在する場合、free_conversion_map を呼び出して解放し NULL に設定する。 5. myState->mapslot が存在する場合、ExecDropSingleTupleTableSlot を呼び出して解放し NULL に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（DestReceiver インターフェースの rShutdown フィールド経由で呼び出される）。
  • **呼び出し先**: pfree、free_conversion_map、ExecDropSingleTupleTableSlot。
  • 注: 呼び出し元の具体的な関数名はソースコードの断片からは特定できません。

関連データ構造

  • TStoreState: 本関数が操作するプライベート構造体。
  • DestReceiver: 本関数が実装するインターフェース。

失敗モードと境界条件

  • myState の各メンバが既に NULL である場合、各ポインタに対する解放処理はスキップされるため、安全に実行可能である。

保守担当者向けメモ

  • 本関数は DestReceiver の終了処理の一部であり、tstoreDestroyReceiver とは役割が異なる（こちらはリソースの解放のみを行い、構造体自体の破棄は行わない）。
  • 新しい変換ロジックやワークスペース用メモリを追加した場合は、本関数で適切に解放処理を追加すること。
