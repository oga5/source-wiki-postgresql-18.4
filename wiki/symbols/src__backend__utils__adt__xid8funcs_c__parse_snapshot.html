parse_snapshot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid8funcs.c:264

目的

文字列形式（xmin:xmax:xip1,xip2,...）で表現されたスナップショット情報を解析し、pg_snapshot 構造体へ変換する。

シグネチャまたは形状

static pg_snapshot *parse_snapshot(const char *str, Node *escontext)

入力と出力

  • **入力**:
  • str: 解析対象の文字列。xmin と xmax をコロンで区切り、その後にカンマ区切りのトランザクションIDリストが続く形式。
  • escontext: エラー報告用コンテキスト（Node）。
  • **出力**:
  • 成功時: 解析済みの pg_snapshot * を返す。
  • 失敗時: escontext を介してエラーを報告し、NULL を返す。

前提条件と不変条件

  • xmin および xmax は有効な FullTransactionId であること。
  • xmin は xmax より先行していること（FullTransactionIdPrecedes(xmax, xmin) が偽であること）。
  • 入力文字列内のトランザクションIDリストは、昇順かつ重複がない状態で提供される必要がある。

アルゴリズムまたは振る舞い

1. 文字列から xmin と xmax を抽出し、妥当性を検証する。 2. buf_init(xmin, xmax) を呼び出し、スナップショット構築用のバッファを初期化する。 3. 文字列の残りの部分を走査し、各トランザクションIDを順次読み取る。 4. 読み取ったIDが xmin 未満、または xmax 以上である場合、あるいは順序が不正な場合はエラーとする。 5. 重複を除外しながら buf_add_txid を用いてバッファにIDを追加する。 6. 最後に buf_finalize を呼び出し、完成した pg_snapshot を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/xid8funcs.c 内の pg_snapshot 型の入力関数（pg_snapshot_in 等）から呼び出される。
  • **呼び出し先**:
  • buf_init, buf_add_txid, buf_finalize: スナップショット構築用ヘルパー。
  • FullTransactionIdFromU64, FullTransactionIdIsValid, FullTransactionIdPrecedes, FullTransactionIdFollowsOrEquals, FullTransactionIdEquals: FullTransactionId 操作用マクロ/関数。
  • ereturn: エラー処理用マクロ。

関連データ構造

  • pg_snapshot: 解析結果として生成されるスナップショット構造体。
  • FullTransactionId: 64ビットトランザクションID。
  • StringInfo: 構築過程で使用されるバッファ構造体。

失敗モードと境界条件

  • 入力文字列のフォーマットが不正（コロンやカンマの欠落、数値以外の文字など）な場合、bad_format ラベルへ遷移しエラーを報告する。
  • xmin や xmax が無効な場合、または xmin > xmax の場合はエラーとなる。
  • 入力されたトランザクションIDリストが昇順でない場合、エラーとなる。

保守担当者向けメモ

  • 本関数は pg_snapshot 型の入力処理を担う。将来的にスナップショットの内部表現が変更された場合、buf_ 系のヘルパー関数と併せて修正が必要となる。
  • escontext を使用しているため、呼び出し元でのエラーハンドリング方針（ereport による中断か、escontext へのエラー格納か）に注意すること。
