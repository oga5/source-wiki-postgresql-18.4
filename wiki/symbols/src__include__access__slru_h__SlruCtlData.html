SlruCtlData

定義

  • 種別: struct
  • 定義箇所: src/include/access/slru.h:127

目的

SLRU（Simple LRU）バッファ管理機構の各インスタンスを制御するための非共有データ構造です。共有メモリ上の状態（SlruShared）と連携し、特定のメタデータ（CLOG, SUBTRANS等）のディスクI/Oおよびバッファ管理を統括します。

シグネチャまたは形状

typedef struct SlruCtlData
{
    SlruShared    shared;
    uint16        nbanks;
    bool          long_segment_names;
    SyncRequestHandler sync_handler;
    bool          (*PagePrecedes) (int64, int64);
    char          Dir[64];
} SlruCtlData;

入力と出力

  • **入力**: SimpleLruInit 等の初期化関数を通じて設定値が与えられます。PagePrecedes コールバックはページ番号の比較ロジックを提供します。
  • **出力**: 構造体自体は制御情報として保持され、Dir（ディレクトリ名）や sync_handler（同期ハンドラ）等の設定に基づき、SLRU の動作を決定します。

前提条件と不変条件

  • Dir は SimpleLruInit で設定された後、変更されません。
  • PagePrecedes コールバックは、SimpleLruTruncate を使用する SLRU においてはモジュロ演算に基づいた比較を行う必要があります。
  • sync_handler が SYNC_HANDLER_NONE の場合、fsync は無効化されます。

アルゴリズムまたは振る舞い

  • **ページ置換**: shared メンバを介して LRU アルゴリズムに基づいたページ置換を管理します。
  • **セグメント管理**: long_segment_names フラグにより、ファイル名の命名規則（短縮形式か長形式か）を切り替えます。
  • **切り捨て**: PagePrecedes コールバックを用いて、どのページが「より古い」かを判定し、ログの切り捨てや LRU 置換のヒントとして利用します。

呼び出し元と呼び出し先

  • **呼び出し元（静的参照）**:
  • clog.c (XactCtlData)
  • commit_ts.c (CommitTsCtlData)
  • multixact.c (MultiXactOffsetCtlData, MultiXactMemberCtlData)
  • subtrans.c (SubTransCtlData)
  • async.c (NotifyCtlData)
  • predicate.c (SerialSlruSlruCtlData)
  • test_slru.c (TestSlruCtlData)
  • **呼び出し先**: 構造体内部で SyncRequestHandler や PagePrecedes コールバックを保持しており、これらが実行時に呼び出されます。

関連データ構造

  • SlruShared: 共有メモリ上に配置される SLRU の状態構造体。
  • SlruCtl: SlruCtlData へのポインタ型。

失敗モードと境界条件

  • PagePrecedes の実装が不適切な場合、SimpleLruTruncate によるログ切り捨てが正しく機能せず、データ整合性に影響を与える可能性があります。
  • Dir のサイズ（64バイト）を超えるパス名は指定できません。

保守担当者向けメモ

  • 本構造体は非共有メモリ上に配置されるため、各バックエンドプロセスで個別に保持されます。
  • PagePrecedes の挙動は、!PagePrecedes(a,b) && !PagePrecedes(b,a) が必ずしも a == b を意味しない点に注意してください。
