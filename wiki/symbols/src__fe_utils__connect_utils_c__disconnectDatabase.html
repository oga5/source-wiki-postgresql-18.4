disconnectDatabase

定義

  • 種別: function
  • 定義箇所: src/fe_utils/connect_utils.c:157

目的

PostgreSQLデータベースへの接続を安全に終了させ、リソースを解放します。アクティブなトランザクションが存在する場合は、接続を閉じる前にキャンセルを試みます。

シグネチャまたは形状

void disconnectDatabase(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn - 終了対象のデータベース接続ハンドル。
  • **出力**: なし（void）。

前提条件と不変条件

  • conn は NULL であってはなりません（Assert により検証されます）。

アルゴリズムまたは振る舞い

1. PQtransactionStatus を呼び出し、接続の状態を確認します。 2. トランザクションがアクティブ（PQTRANS_ACTIVE）である場合：

  • PQcancelCreate を使用してキャンセルリクエストを作成します。
  • PQcancelBlocking を呼び出してキャンセル処理を実行します。
  • PQcancelFinish を呼び出してキャンセル用リソースを解放します。

    • 3. 最後に PQfinish(conn) を呼び出し、データベース接続を閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/fe_utils/parallel_slot.c
  • **呼び出し先**:
  • libpq ライブラリ関数: PQtransactionStatus, PQcancelCreate, PQcancelBlocking, PQcancelFinish, PQfinish

関連データ構造

  • PGconn: libpq が管理するデータベース接続オブジェクト。

失敗モードと境界条件

  • conn が NULL の場合、アサーション違反によりプログラムが異常終了します。
  • 接続が既に閉じられている場合や無効なポインタが渡された場合の挙動は、libpq の仕様に依存します。

保守担当者向けメモ

  • 本関数は libpq の接続終了処理をラップしたものです。接続終了時のクリーンアップを一貫させるため、フロントエンドツールで接続を閉じる際は直接 PQfinish を呼ばず、本関数を使用してください。
