pthread_barrier_init

定義

  • 種別: function
  • 定義箇所: src/port/pthread_barrier_wait.c:18

目的

ネイティブの POSIX スレッドバリアをサポートしていないプラットフォーム向けに、バリア同期オブジェクトを初期化する。

シグネチャまたは形状

int pthread_barrier_init(pthread_barrier_t *barrier, const void *attr, int count);

入力と出力

  • **入力**:
  • barrier: 初期化対象となる pthread_barrier_t 構造体へのポインタ。
  • attr: バリア属性（本実装では使用されない）。
  • count: バリアを解除するために必要なスレッド数。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時はエラーコード（整数）を返す。

前提条件と不変条件

  • barrier が指すメモリ領域は有効であること。
  • count は正の整数であること。
  • 内部状態として sense は false に、arrived は 0 に初期化される。

アルゴリズムまたは振る舞い

1. barrier->sense を false に設定する。 2. barrier->count に指定された count を格納する。 3. barrier->arrived を 0 にリセットする。 4. pthread_cond_init を呼び出し、条件変数を初期化する。失敗した場合はエラーを返す。 5. pthread_mutex_init を呼び出し、ミューテックスを初期化する。

  • ミューテックスの初期化に失敗した場合は、先に初期化した条件変数を pthread_cond_destroy で破棄してからエラーを返す。

    • 6. 正常終了時に 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pgbench/pgbench.c:154 からの呼び出しが確認されている。
  • **呼び出し先**: pthread_cond_init、pthread_mutex_init、および失敗時のクリーンアップとして pthread_cond_destroy を呼び出す。

関連データ構造

  • pthread_barrier_t: バリアの状態（到達数、目標数、条件変数、ミューテックス、センスフラグ）を保持する構造体。

失敗モードと境界条件

  • pthread_cond_init または pthread_mutex_init が失敗した場合、そのエラーコードがそのまま返される。
  • ミューテックス初期化失敗時には、条件変数の破棄が確実に行われる。

保守担当者向けメモ

  • 本関数は「センス反転（Sense Reversal）」アルゴリズムの一部として機能する。
  • pthread_barrier_t の定義は port/pg_pthread.h に依存している。
