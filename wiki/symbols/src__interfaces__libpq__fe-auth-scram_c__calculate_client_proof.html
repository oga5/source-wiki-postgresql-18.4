calculate_client_proof

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:765

目的

SCRAM-SHA-256 認証プロセスにおいて、クライアントの証明（ClientProof）を計算し、サーバーへ送信するための値を生成します。

シグネチャまたは形状

static bool calculate_client_proof(fe_scram_state *state,
                                   const char *client_final_message_without_proof,
                                   uint8 *result, const char **errstr);

入力と出力

  • **入力**:
  • state: 認証状態を保持する fe_scram_state 構造体。
  • client_final_message_without_proof: 証明部分を除いたクライアントの最終メッセージ文字列。
  • **出力**:
  • result: 計算されたクライアント証明（ClientKey と ClientSignature の XOR 結果）が格納されるバッファ。
  • errstr: 失敗時にエラーメッセージが格納されるポインタ。
  • **戻り値**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • state 内の hash_type、password、salt、iterations が適切に初期化されている必要があります。
  • state->conn->scram_client_key_binary が存在しない場合、scram_SaltedPassword および scram_ClientKey を用いて鍵が計算されます。

アルゴリズムまたは振る舞い

1. HMAC コンテキストを作成します。 2. ClientKey を取得または計算します。 3. ClientKey から StoredKey を計算します。 4. StoredKey を鍵として、client-first-message-bare、server-first-message、および client-final-message-without-proof を連結したデータに対して HMAC を実行し、ClientSignature を生成します。 5. ClientKey と ClientSignature の各バイトを XOR し、その結果を result に格納します。 6. 使用した HMAC コンテキストを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth-scram.c 内の認証フロー（scram_exchange 等）から呼び出されます。
  • **呼び出し先**: pg_hmac_create, scram_SaltedPassword, scram_ClientKey, scram_H, pg_hmac_init, pg_hmac_update, pg_hmac_final, pg_hmac_free 等の HMAC および SCRAM 共通計算関数。

関連データ構造

  • fe_scram_state: 認証セッションの状態を保持する構造体。
  • pg_hmac_ctx: HMAC 計算用コンテキスト。

失敗モードと境界条件

  • メモリ不足（OOM）により pg_hmac_create が失敗した場合、false を返します。
  • scram_SaltedPassword や scram_ClientKey 等の計算関数がエラーを返した場合、処理を中断し false を返します。
  • HMAC の初期化や更新、最終化の過程でエラーが発生した場合、エラーメッセージをセットして false を返します。

保守担当者向けメモ

  • 本関数は ClientKey を ClientSignature で XOR するという SCRAM プロトコルの仕様に基づいた計算を行っています。
  • state->conn->scram_client_key_binary が存在する場合は再利用されるため、キャッシュの整合性に注意してください。
