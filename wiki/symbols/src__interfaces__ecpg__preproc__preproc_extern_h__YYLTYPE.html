YYLTYPE

定義

  • 種別: typedef
  • 定義箇所: src/interfaces/ecpg/preproc/preproc_extern.h:20

目的

ECPGプリプロセッサのパーサーにおいて、ソースコード内の位置情報を追跡するために使用されます。

シグネチャまたは形状

typedef const char *YYLTYPE;

定数文字列へのポインタとして定義されています。

入力と出力

  • **入力**: パーサーの字句解析プロセスにおいて、現在のトークンの位置情報（ファイル名や行番号等に相当する文字列）を保持します。
  • **出力**: パーサーがエラー報告や構文解析の文脈で使用する位置情報として参照されます。

前提条件と不変条件

  • const char * 型であるため、指し示す先の文字列は変更不可として扱われます。
  • パーサーの実行中、現在の解析位置を指す有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

パーサーがトークンを読み進める際、この型を使用して現在の解析位置を更新・保持します。字句解析器（lexer）と構文解析器（parser）の間で、トークンの位置情報を共有するためのインターフェースとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/parser.c 内の lookahead_yylloc や cur_yylloc 等の変数定義で利用されています。
  • **呼び出し先**: 本シンボル自体は型定義であり、関数ではないため「呼び出し先」という概念は適用されません。
  • **注記**: 他のソースファイル（src/backend/parser/ 等）でも同名の型が定義されていますが、本ページは src/interfaces/ecpg/preproc/preproc_extern.h で定義された YYLTYPE に限定しています。

関連データ構造

  • YYSTYPE: パーサーがトークンの値を保持するために使用する型（本ファイルに関連するパーサー定義で併用されます）。

失敗モードと境界条件

  • ポインタ型であるため、初期化されていない場合や無効なメモリ領域を指している場合にデリファレンスを行うと未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本定義は const char * ですが、他の PostgreSQL コンポーネント（バックエンドのパーサー等）では int など別の型として定義されている場合があります。ECPG プリプロセッサ内での変更を行う際は、この型定義がパーサーの期待する位置情報表現と一致していることを確認してください。
