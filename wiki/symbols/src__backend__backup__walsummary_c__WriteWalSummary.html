WriteWalSummary

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummary.c:293

目的

WalSummaryIO 構造体で管理されるファイルに対して、指定されたデータを書き込み、内部のファイルオフセットを更新します。

シグネチャまたは形状

int WriteWalSummary(void *wal_summary_io, void *data, int length)

入力と出力

  • **入力**:
  • wal_summary_io: WalSummaryIO 構造体へのポインタ（void * として受け取り内部でキャスト）。
  • data: 書き込むデータバッファ。
  • length: 書き込むバイト数。
  • **出力**:
  • 戻り値: 実際に書き込まれたバイト数（int）。

前提条件と不変条件

  • wal_summary_io は有効な WalSummaryIO 構造体を指している必要があります。
  • io->file は書き込み可能なファイル記述子として初期化されている必要があります。

アルゴリズムまたは振る舞い

1. FileWrite を呼び出し、io->file に対して data を length 分だけ書き込みます。この際、WAIT_EVENT_WAL_SUMMARY_WRITE を待機イベントとして使用します。 2. 書き込み結果を検証します。

  • 書き込み結果が負の場合、ファイルアクセスエラーとして ereport(ERROR, ...) を発行します。
  • 書き込まれたバイト数が要求した length と一致しない場合、ディスク容量不足の可能性を示唆するエラーとして ereport(ERROR, ...) を発行します。

    • 3. 書き込み成功時、io->filepos を書き込んだバイト数分だけ進めます。 4. 最終的に書き込んだバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/walsummarizer.c 内の WriteBlockRefTable から呼び出されます。
  • **呼び出し先**: FileWrite（PostgreSQLのファイルI/O抽象化レイヤー）。

関連データ構造

  • WalSummaryIO: ファイル記述子 (file) と現在の書き込み位置 (filepos) を保持する構造体。

失敗モードと境界条件

  • **ファイル書き込み失敗**: FileWrite が負の値を返した場合、ファイルシステムエラーとして処理を中断します。
  • **部分書き込み**: 要求したバイト数と実際の書き込みバイト数が異なる場合、ディスクフル等のエラーとして処理を中断します。

保守担当者向けメモ

  • 本関数は WalSummaryIO 構造体の filepos を自動的に更新するため、呼び出し側でオフセット管理を行う必要はありません。
  • エラー発生時は ereport を使用してPostgreSQLの標準的なエラーハンドリング機構に委譲されます。
