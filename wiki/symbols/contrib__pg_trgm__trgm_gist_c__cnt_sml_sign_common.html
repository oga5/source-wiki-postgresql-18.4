cnt_sml_sign_common

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_gist.c:178

目的

クエリに含まれるトリグラムの集合が、GiST インデックスのシグネチャ（ビットベクトル）内にいくつ存在するかをカウントします。

シグネチャまたは形状

int32 cnt_sml_sign_common(TRGM *qtrg, BITVECP sign, int siglen)

入力と出力

  • **入力**:
  • qtrg: 検索対象のトリグラム集合（TRGM 構造体）。
  • sign: インデックスノードのシグネチャ（BITVECP）。
  • siglen: シグネチャの長さ（バイト単位）。
  • **出力**:
  • int32: シグネチャ内に含まれていたトリグラムの総数。

前提条件と不変条件

  • qtrg は有効なトリグラム配列を指している必要があります。
  • sign は siglen バイト以上のメモリ領域を確保している必要があります。

アルゴリズムまたは振る舞い

1. qtrg 内の各トリグラムを順次走査します。 2. 各トリグラムに対して CPTRGM マクロで値を抽出し、HASHVAL マクロでシグネチャ上のビット位置を計算します。 3. GETBIT マクロを使用して、計算された位置のビットが立っているか（1であるか）を確認します。 4. ビットが立っている場合、カウントをインクリメントします。 5. 全トリグラムの走査終了後、合計値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_gist.c:334
  • contrib/pg_trgm/trgm_gist.c:518
  • **呼び出し先**:
  • ARRNELEM, GETARR, CPTRGM, GETBIT, HASHVAL（いずれも trgm.h 等で定義されたマクロ）。

関連データ構造

  • TRGM: トリグラムを保持する可変長構造体。
  • BITVECP: ビットベクトルへのポインタ型。

失敗モードと境界条件

  • qtrg が空の場合、ループは実行されず 0 を返します。
  • siglen が不適切な値の場合、HASHVAL によるインデックス計算でメモリ境界外アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数は GiST インデックスの検索および距離計算における類似度判定の基礎となります。
  • HASHVAL マクロのハッシュ計算ロジックに変更を加える場合は、インデックスの再構築が必要になる可能性があるため注意してください。
