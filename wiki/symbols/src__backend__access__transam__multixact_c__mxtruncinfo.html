mxtruncinfo

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/multixact.c:3187

目的

MultiXact の SLRU（Simple Least Recently Used）領域における、切り詰め（Truncation）可能な境界情報を保持するための構造体です。

シグネチャまたは形状

typedef struct mxtruncinfo
{
	int64		earliestExistingPage;
} mxtruncinfo;

入力と出力

  • 入力: int64 型の earliestExistingPage（現存する最も古いページ番号）。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • earliestExistingPage は、MultiXact の SLRU 管理において有効なページ番号を指す必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、TruncateMultiXact 処理において、どのページまでを削除可能か、あるいはどのページが保持されるべきかを判定するための情報を格納するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/multixact.c 内の TruncateMultiXact 関数において、ローカル変数として宣言およびポインタ経由でのキャストに使用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • MultiXactOffsetCtl / MultiXactMemberCtl: SLRU 管理用構造体。本構造体はこれらの SLRU 領域の切り詰め処理と密接に関連します。

失敗モードと境界条件

  • earliestExistingPage に不正なページ番号が設定された場合、SLRU の整合性が損なわれ、MultiXact データの読み取りエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は TruncateMultiXact 処理専用の補助的なデータ保持用です。
  • src/backend/access/transam/multixact.c 内の TruncateMultiXact 関数における使用箇所（3199行目、3281行目）を参照してください。
