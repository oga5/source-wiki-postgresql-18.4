RecoveryTargetTimeLineGoal

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlogrecovery.h:36

目的

リカバリ処理において、どのタイムラインをターゲットとしてWALの再生を行うかを決定するための目標設定を定義します。

シグネチャまたは形状

typedef enum
{
	RECOVERY_TARGET_TIMELINE_CONTROLFILE,
	RECOVERY_TARGET_TIMELINE_LATEST,
	RECOVERY_TARGET_TIMELINE_NUMERIC,
} RecoveryTargetTimeLineGoal;

入力と出力

  • 本シンボルは列挙型であり、リカバリ設定の決定に使用されます。
  • RECOVERY_TARGET_TIMELINE_CONTROLFILE: 制御ファイルに基づいたタイムラインを使用。
  • RECOVERY_TARGET_TIMELINE_LATEST: 利用可能な最新のタイムラインを使用。
  • RECOVERY_TARGET_TIMELINE_NUMERIC: 指定された数値のタイムラインを使用。

前提条件と不変条件

  • リカバリプロセスにおいて、この列挙値は recoveryTargetTimeLineGoal 変数を通じてリカバリの挙動を制御します。

アルゴリズムまたは振る舞い

  • リカバリ開始時に、この設定値に基づいてWALの読み込み対象となるタイムラインが決定されます。
  • src/backend/access/transam/xlogrecovery.c 内で、GUCパラメータの更新やリカバリ状態の初期化時にこの値が参照・設定されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogrecovery.c において、リカバリ設定の初期化およびGUCパラメータの処理に関連して参照されています。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数を呼び出すものではありません。

関連データ構造

  • recoveryTargetTimeLineGoal: 本列挙型を保持するグローバル変数（src/include/access/xlogrecovery.h:69 で宣言）。

失敗モードと境界条件

  • 適切なタイムラインが指定されていない場合や、リカバリターゲットの設定と矛盾するタイムラインが選択された場合の挙動については、本ソース excerpt からは判断できません。

保守担当者向けメモ

  • 本シンボルはリカバリのタイムライン選択ロジックの根幹です。変更時は xlogrecovery.c 内のタイムライン切り替え処理への影響を考慮してください。
