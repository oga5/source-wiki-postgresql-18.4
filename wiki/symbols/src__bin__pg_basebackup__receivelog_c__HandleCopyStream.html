HandleCopyStream

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:743

目的

PostgreSQLのストリーミングレプリケーションにおける COPY プロトコルを通じたWALデータ受信ループを管理し、サーバーとの同期およびデータ書き込み処理を実行する。

シグネチャまたは形状

static PGresult *HandleCopyStream(PGconn *conn, StreamCtl *stream, XLogRecPtr *stoppos)

入力と出力

  • **入力**:
  • conn: サーバーへの接続用 PGconn ポインタ。
  • stream: ストリーミング設定を保持する StreamCtl 構造体。
  • stoppos: ストリーミング終了位置を格納するポインタ。
  • **出力**:
  • 成功時: 終了時の PGresult ポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • stream->startpos が有効なWAL開始位置であること。
  • walfile がオープンされている場合、その書き込みメソッドが有効であること。
  • still_sending フラグがループ開始時に true に設定される。

アルゴリズムまたは振る舞い

1. **ループ処理**: CopyStreamReceive を呼び出し、サーバーからのメッセージを待機・受信する。 2. **同期書き込み**: stream->synchronous が有効な場合、未フラッシュのWALデータがあれば walmethod->ops->sync を実行し、サーバーへフィードバックを送信する。 3. **生存確認**: standby_message_timeout に基づき、定期的にサーバーへフィードバックを送信する。 4. **メッセージ処理**:

  • 'k' (Keepalive): ProcessKeepaliveMsg を呼び出し、サーバーの生存確認に応答する。
  • 'w' (XLogData): ProcessXLogDataMsg を呼び出し、WALデータをローカルへ書き込む。

    • 5. **終了判定**: CheckCopyStreamStop によりストリーミング継続可否を判定し、終了条件を満たした場合は HandleEndOfCopyStream を経由して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ReceiveXlogStream (同ファイル内)
  • **呼び出し先**:
  • CheckCopyStreamStop, feGetCurrentTimestamp, feTimestampDifferenceExceeds, sendFeedback, CalculateCopyStreamSleeptime, PQfreemem, CopyStreamReceive, HandleEndOfCopyStream, ProcessKeepaliveMsg, ProcessXLogDataMsg, pg_fatal, pg_log_error

関連データ構造

  • StreamCtl: ストリーミングの動作パラメータを保持。
  • PGconn: libpqの接続ハンドル。
  • WalWriteMethod: WAL書き込みの抽象化インターフェース。

失敗モードと境界条件

  • CopyStreamReceive が -1 を返した場合、エラーとして終了する。
  • 未知のメッセージヘッダーを受信した場合、エラーログを出力して終了する。
  • fsync 失敗時、pg_fatal によりプロセスが終了する。
  • CheckCopyStreamStop が false を返した場合、ループを抜けて終了する。

保守担当者向けメモ

  • walmethod->ops->sync はファイルシステムへの物理的な書き込みを伴うため、パフォーマンスに影響を与える。
  • copybuf のメモリ管理は PQfreemem を介して行われており、ループ内でのリークに注意すること。
  • still_sending はグローバル変数として参照されている可能性があるため、並行実行時の競合に注意が必要。
