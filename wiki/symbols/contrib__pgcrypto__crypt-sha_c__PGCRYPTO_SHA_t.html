PGCRYPTO_SHA_t

定義

  • 種別: enum
  • 定義箇所: contrib/pgcrypto/crypt-sha.c:55

目的

sha256crypt および sha512crypt アルゴリズムを識別するための列挙型です。

シグネチャまたは形状

typedef enum
{
    PGCRYPTO_SHA256CRYPT = 0,
    PGCRYPTO_SHA512CRYPT = 1,
    PGCRYPTO_SHA_UNKOWN
} PGCRYPTO_SHA_t;

入力と出力

  • 入力: なし（列挙定数）
  • 出力: なし（型定義）

前提条件と不変条件

  • PGCRYPTO_SHA256CRYPT は 0、PGCRYPTO_SHA512CRYPT は 1 に固定されています。
  • PGCRYPTO_SHA_UNKOWN は、アルゴリズムが特定できない場合や初期化状態を示すために使用されます。

アルゴリズムまたは振る舞い

本シンボル自体はデータ型であり、ロジックは持ちません。crypt-sha.c 内のハッシュ生成処理において、ソルト文字列から解析されたアルゴリズム種別を保持するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/crypt-sha.c 内の関数において、アルゴリズム種別を格納する変数型として参照されます（例: PGCRYPTO_SHA_t type = PGCRYPTO_SHA_UNKOWN;）。
  • 呼び出し先: なし。

関連データ構造

  • px_crypt_shacrypt: 本列挙型を利用してハッシュアルゴリズムを切り替えるメイン関数。

失敗モードと境界条件

  • PGCRYPTO_SHA_UNKOWN が設定されたまま処理が進行した場合、アルゴリズムの不一致やエラーとして扱われる可能性があります。

保守担当者向けメモ

  • 新しい SHA ベースのハッシュアルゴリズムを追加する場合は、本列挙型に定数を追加し、px_crypt_shacrypt 内の解析ロジックを更新してください。
