pg_perm_setlocale

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:197

目的

指定されたロケールカテゴリをシステム環境に適用し、環境変数（LC_COLLATE, LC_CTYPE 等）を更新して永続化する。

シグネチャまたは形状

char *pg_perm_setlocale(int category, const char *locale);

入力と出力

  • **入力**:
  • category: 設定対象のロケールカテゴリ（LC_COLLATE, LC_CTYPE 等）。
  • locale: 設定するロケール名文字列。
  • **出力**:
  • 成功時: 設定されたロケール名文字列へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • LC_CTYPE を設定する場合、内部的にメッセージエンコーディングが更新される。
  • Windows 環境では LC_MESSAGES の設定に IsoLocaleName を介した変換が適用される。
  • 成功した場合、指定されたカテゴリに対応する環境変数が setenv により更新される。

アルゴリズムまたは振る舞い

1. **ロケール適用**: setlocale を呼び出し、OS レベルでロケールを適用する（Windows の LC_MESSAGES は例外処理あり）。 2. **エラー判定**: setlocale が NULL を返した場合は即座に終了する。 3. **LC_CTYPE 特殊処理**:

  • 戻り値を静的バッファにコピーし、以降の呼び出しによる上書きを防止する。
  • ENABLE_NLS の有無に応じて、SetMessageEncoding を呼び出し、メッセージエンコーディングをデータベースエンコーディングまたはテキストドメインのコードセットに同期させる。

    • 4. **環境変数更新**: 指定された category に対応する環境変数名（例: LC_COLLATE）を特定し、setenv を用いて値を永続化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/main/main.c
  • src/backend/utils/adt/pg_locale.c
  • src/backend/utils/init/postinit.c
  • **呼び出し先**:
  • setlocale (libc)
  • SetMessageEncoding
  • pg_bind_textdomain_codeset
  • textdomain
  • GetDatabaseEncoding
  • IsoLocaleName (Windows のみ)
  • setenv
  • elog

関連データ構造

  • LOCALE_NAME_BUFLEN: LC_CTYPE の値を保持する静的バッファのサイズ定義。

失敗モードと境界条件

  • setlocale が NULL を返した場合、関数は即座に NULL を返す。
  • 未知の category が渡された場合、elog(FATAL) によりプロセスが異常終了する。
  • setenv が失敗した場合、関数は NULL を返す。

保守担当者向けメモ

  • 本関数は、PostgreSQL の起動時やデータベース接続時にロケールを「固定」するために使用される。
  • Windows における LC_MESSAGES の扱いはプラットフォーム固有の制約があるため、修正時は WIN32 マクロの条件分岐に注意すること。
  • LC_CTYPE 処理時の静的バッファは、スレッドセーフではない点に留意が必要（PostgreSQL のバックエンドプロセスモデル前提）。
