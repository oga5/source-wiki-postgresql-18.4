ginInitConsistentFunction

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginlogic.c:226

目的

GINインデックスのスキャン開始時に、指定された GinScanKey に対して適切な整合性チェック関数（Consistent Check）を割り当て、セットアップを行う。

シグネチャまたは形状

void ginInitConsistentFunction(GinState *ginstate, GinScanKey key)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの状態を保持する GinState 構造体。
  • key: 初期化対象の GinScanKey 構造体。
  • **出力**: なし（key 構造体内の関数ポインタおよび関連フィールドを直接更新する）。

前提条件と不変条件

  • key->attnum が有効な属性番号を指していること。
  • ginstate が初期化済みであり、対応する演算子クラスの関数OIDが格納されていること。

アルゴリズムまたは振る舞い

1. **EVERYTHINGモードの判定**: key->searchMode が GIN_SEARCH_MODE_EVERYTHING の場合、常に合致を返すダミー関数（trueConsistentFn, trueTriConsistentFn）を割り当てる。 2. **通常モードのセットアップ**:

  • ginstate から整合性関数および照合順序（collation）情報を key にコピーする。
  • **二値論理関数の決定**: ginstate に関数OIDが存在すれば directBoolConsistentFn を、存在しなければ shimBoolConsistentFn を割り当てる。
  • **三値論理関数の決定**: ginstate に関数OIDが存在すれば directTriConsistentFn を、存在しなければ shimTriConsistentFn を割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginscan.c:201
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、関数ポインタの代入のみを行う。

関連データ構造

  • GinState: インデックスのメタデータや関数OIDを保持。
  • GinScanKey: スキャン中の検索条件や、割り当てられた整合性関数ポインタを保持。

失敗モードと境界条件

  • attnum が範囲外の場合、ginstate の配列アクセスでメモリ違反が発生する可能性がある。
  • 演算子クラスが整合性関数を定義していない場合、シム（Shim）関数が割り当てられるが、その実行には計算コスト（指数関数的）が伴う。

保守担当者向けメモ

  • 本関数はスキャン開始時に一度だけ実行される。
  • shim 系関数が割り当てられた場合、実行時のパフォーマンスに影響するため、複雑なクエリでは演算子クラス側での三値論理関数の実装が推奨される。
