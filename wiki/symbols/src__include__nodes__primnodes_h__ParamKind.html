ParamKind

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/primnodes.h:382

目的

クエリ実行時におけるパラメータノードのソース（由来）を識別するために使用されます。

シグネチャまたは形状

typedef enum ParamKind
{
	PARAM_EXTERN,
	PARAM_MULTIEXPR,
	PARAM_EXEC,
	PARAM_SUBLINK,
} ParamKind;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: Param ノード内の paramkind フィールドとして保持されます。

前提条件と不変条件

  • Param 構造体の一部として使用され、パラメータの評価方法を決定する識別子として機能します。

アルゴリズムまたは振る舞い

  • PARAM_EXTERN: 外部から提供されるパラメータ（例: プリペアドステートメントの引数）。
  • PARAM_EXEC: 実行プランのノードによって計算されるパラメータ。
  • PARAM_SUBLINK: サブクエリの結果として得られるパラメータ。
  • PARAM_MULTIEXPR: 複数式の結果を保持するパラメータ。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/nodes/primnodes.h 内の Param 構造体定義で参照されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数レベルの呼び出し元・呼び出し先は特定できません。

関連データ構造

  • Param: この列挙型をメンバとして持つ構造体。
  • src/include/nodes/params.h: パラメータ処理に関連する定義が含まれるモジュール。

失敗モードと境界条件

  • 定義外の値を paramkind に代入した場合の動作は未定義です。

保守担当者向けメモ

  • 本シンボルはクエリの実行段階におけるパラメータ解決の基盤です。新しいパラメータタイプを追加する場合は、関連する実行エンジン側の評価ロジック（ExecEvalParam 等）の更新が必要です。
