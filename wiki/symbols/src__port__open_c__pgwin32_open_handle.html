pgwin32_open_handle

定義

  • 種別: function
  • 定義箇所: src/port/open.c:64

目的

Windows 環境において、POSIX 互換のフラグ指定でファイルを開き、アンチウイルスソフト等による一時的なロック（共有違反）に対して再試行ロジックを適用した上で、Windows ネイティブの HANDLE を取得する。

シグネチャまたは形状

HANDLE pgwin32_open_handle(const char *fileName, int fileFlags, bool backup_semantics)

入力と出力

  • **入力**:
  • fileName: 開く対象のファイルパス。
  • fileFlags: POSIX 準拠のオープンフラグ（O_RDONLY, O_CREAT, O_CLOEXEC 等）。
  • backup_semantics: FILE_FLAG_BACKUP_SEMANTICS を適用するかどうかのフラグ。
  • **出力**:
  • 成功時: 開かれたファイルの HANDLE。
  • 失敗時: INVALID_HANDLE_VALUE。

前提条件と不変条件

  • initialize_ntdll() が成功していること。
  • fileFlags は、関数内で許可されたビットマスクの組み合わせであること（assert により検証）。
  • O_CLOEXEC が指定された場合、生成されるハンドルは非継承（non-inheritable）となる。

アルゴリズムまたは振る舞い

1. **初期化**: initialize_ntdll() を呼び出し、失敗した場合は即座に INVALID_HANDLE_VALUE を返す。 2. **ハンドル属性設定**: O_CLOEXEC の有無に基づき、SECURITY_ATTRIBUTES の bInheritHandle を設定する。 3. **ファイルオープン試行**: CreateFile を呼び出し、共有モードとして FILE_SHARE_READ | FILE_SHARE_WRITE | FILE_SHARE_DELETE を指定する。 4. **再試行ループ**: ERROR_SHARING_VIOLATION または ERROR_LOCK_VIOLATION が発生した場合、pg_usleep を用いて 100ms 待機し、最大 30 秒間（300 回）リトライする。 5. **エラー変換**: ERROR_ACCESS_DENIED かつ NT ステータスが STATUS_DELETE_PENDING の場合、状況に応じて ERROR_FILE_EXISTS または ERROR_FILE_NOT_FOUND に変換する。 6. **終了**: 最終的なエラーコードを _dosmaperr で errno にマッピングし、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/port/open.c (内部)
  • src/port/win32stat.c
  • **呼び出し先**:
  • initialize_ntdll
  • CreateFile (Windows API)
  • openFlagsToCreateFileFlags
  • pg_usleep
  • pg_RtlGetLastNtStatus
  • _dosmaperr
  • ereport (バックエンドビルド時のみ)

関連データ構造

  • HANDLE (Windows API 型)
  • SECURITY_ATTRIBUTES (Windows API 構造体)

失敗モードと境界条件

  • **共有違反**: アンチウイルスソフト等によるロックに対し、30 秒間の再試行を行う。
  • **削除保留**: STATUS_DELETE_PENDING 状態のファイルに対し、POSIX 互換のエラーコードを返す。
  • **無効なフラグ**: assert により、サポート外のフラグが渡された場合は異常終了する。

保守担当者向けメモ

  • O_CLOEXEC の実装において、SetHandleInformation ではなく CreateFile 時の SECURITY_ATTRIBUTES を利用しているのは、マルチスレッド環境での競合を避けるための設計である。
  • STATUS_DELETE_PENDING のハンドリングは、Windows 特有の「削除中だがまだファイルが存在する」状態を POSIX の期待通りに扱うための工夫である。
