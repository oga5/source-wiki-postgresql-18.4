JitProviderCallbacks

定義

  • 種別: struct
  • 定義箇所: src/include/jit/jit.h:65

目的

PostgreSQL の JIT プロバイダ（LLVM 等）が実装すべき関数ポインタを保持し、実行エンジンと JIT 実装間のインターフェースを抽象化するための構造体です。

シグネチャまたは形状

typedef struct JitProviderCallbacks JitProviderCallbacks; （構造体の詳細なメンバ構成は、ソースコードの当該箇所において不透明型として定義されています）

入力と出力

  • 入力: プロバイダ初期化関数 _PG_jit_provider_init(JitProviderCallbacks *cb) を通じて、プロバイダ側で各関数ポインタが設定されます。
  • 出力: 実行エンジンがこの構造体内の関数ポインタを呼び出すことで、JIT コンパイルや最適化処理が実行されます。

前提条件と不変条件

  • プロバイダのロード時に _PG_jit_provider_init が正常に呼び出され、コールバックが適切に埋められている必要があります。
  • JIT が無効な場合やプロバイダのロードに失敗した場合、呼び出し側は false が返される可能性を考慮する必要があります。

アルゴリズムまたは振る舞い

1. jit.c 内の provider_init が外部ライブラリから _PG_jit_provider_init をロードします。 2. ロードされたプロバイダが JitProviderCallbacks 構造体のメンバに関数ポインタを割り当てます。 3. 実行エンジンが JIT 処理を要求すると、この構造体を介してプロバイダ固有の実装が実行されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/jit/jit.c（プロバイダの管理および初期化）、src/include/jit/jit.h（インターフェース定義）。
  • 呼び出し先: src/backend/jit/llvm/llvmjit.c（_PG_jit_provider_init の実装例）。

関連データ構造

  • JitContext: JIT 処理の実行状態と統計情報を保持する構造体。
  • JitInstrumentation: JIT 処理のコスト計測用構造体。
  • JitProviderInit: JitProviderCallbacks を初期化するための関数ポインタ型。

失敗モードと境界条件

  • プロバイダのロード失敗: _PG_jit_provider_init が実行されない場合、コールバックが未設定のままとなり、呼び出し時にクラッシュする可能性があるため、呼び出し側でのガードが必要です。

保守担当者向けメモ

  • 本構造体はプロバイダ非依存の契約です。LLVM 特有の実装については llvmjit.h を参照してください。
  • 新しい JIT プロバイダを追加する際は、本構造体の全メンバを適切に実装する必要があります。
