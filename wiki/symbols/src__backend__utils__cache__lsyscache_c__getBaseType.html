getBaseType

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/lsyscache.c:2698

目的

指定された型（OID）がドメイン型である場合に、その基底型（Base Type）を再帰的に解決して返す。

シグネチャまたは形状

Oid getBaseType(Oid typid)

入力と出力

  • **入力**: typid (Oid) - 解決対象の型OID。
  • **出力**: Oid - 入力型がドメインであればその基底型、そうでなければ入力された typid そのもの。

前提条件と不変条件

  • 入力された typid はシステムカタログ pg_type に存在している必要がある。
  • 内部で getBaseTypeAndTypmod を呼び出すため、同関数の不変条件（キャッシュ検索の成功など）に依存する。

アルゴリズムまたは振る舞い

1. ローカル変数 typmod を -1 で初期化する。 2. getBaseTypeAndTypmod(typid, &typmod) を呼び出す。 3. getBaseTypeAndTypmod が解決した基底型の OID をそのまま呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し先**: getBaseTypeAndTypmod
  • **呼び出し元**:
  • src/backend/access/spgist/spgutils.c
  • src/backend/catalog/dependency.c
  • src/backend/catalog/heap.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/tablecmds.c
  • src/backend/executor/execExpr.c
  • src/backend/parser/parse_coerce.c

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • pg_type: 型情報を保持するシステムカタログ。

失敗モードと境界条件

  • 入力 typid が無効な場合、getBaseTypeAndTypmod 経由でシステムキャッシュ検索エラー（elog）が発生する可能性がある。

保守担当者向けメモ

  • 本関数は getBaseTypeAndTypmod の単なるラッパーであり、typmod を無視して基底型のみを取得したい場合に使用する。
  • 複雑な型解決ロジックは getBaseTypeAndTypmod に集約されているため、修正が必要な場合はそちらを確認すること。
