sockaddr

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/ifaddr.h:17

目的

ネットワークアドレス情報を保持するための汎用的なソケットアドレス構造体です。PostgreSQL のバックエンドにおいて、IPv4 および IPv6 を含むプロトコル非依存のアドレス操作を可能にします。

シグネチャまたは形状

標準的なソケットアドレス構造体であり、PgIfAddrCallback 型の定義において以下のように参照されます。

typedef void (*PgIfAddrCallback) (struct sockaddr *addr,
                                  struct sockaddr *netmask,
                                  void *cb_data);

入力と出力

本構造体自体はデータ保持用ですが、PgIfAddrCallback を介して以下の用途で使用されます。

  • **入力**: ネットワークインターフェースのアドレスおよびネットマスク情報。
  • **出力**: コールバック関数内でのアドレス比較や検証結果の処理。

前提条件と不変条件

  • libpq/pqcomm.h で定義されるソケット関連の基本構造体と互換性があります。
  • アドレスファミリー（AF_INET / AF_INET6）が一致していることが、比較操作を行う際の前提となります。

アルゴリズムまたは振る舞い

  • 本構造体は、pg_range_sockaddr 等の関数において、ビット演算によるサブネット判定や CIDR マスクの適用対象として利用されます。
  • pg_foreach_ifaddr を通じて、システム上のインターフェースから取得されたアドレス情報が本構造体形式でコールバックに渡されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/libpq/auth.c: 認証時のアドレスバインドやリモートアドレスの取得。
  • src/backend/libpq/hba.c: check_ip や check_network_callback における IP 範囲検証。
  • src/backend/libpq/ifaddr.c: インターフェース情報の走査およびマスク生成処理。
  • **呼び出し先**: 本構造体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • sockaddr_storage: より大きな領域を確保し、IPv4/IPv6 を透過的に扱うための構造体。
  • PgIfAddrCallback: 本構造体を引数に取るコールバック関数型。

失敗モードと境界条件

  • アドレスファミリーの不一致: 異なるファミリーのアドレス同士を比較しようとした場合、予期せぬ動作を引き起こす可能性があります。
  • メモリレイアウト: プラットフォーム依存のネットワーク API（getifaddrs 等）に依存するため、OS ごとの実装差異に注意が必要です。

保守担当者向けメモ

  • 本構造体は libpq サブシステム全体でネットワーク層の抽象化を担います。
  • 移植性に関する詳細は src/backend/libpq/ifaddr.c の実装を参照してください。
