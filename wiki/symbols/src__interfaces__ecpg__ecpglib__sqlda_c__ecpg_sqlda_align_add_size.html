ecpg_sqlda_align_add_size

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/sqlda.c:32

目的

C言語の構造体アライメント規則に従い、メモリ上のオフセットを調整し、指定されたデータサイズを加算して次のオフセットを算出する。

シグネチャまたは形状

static void ecpg_sqlda_align_add_size(long offset, int alignment, int size, long *current, long *next)

入力と出力

  • offset (long): 現在の基準オフセット。
  • alignment (int): 必要なアライメント境界（バイト単位）。
  • size (int): 加算するデータサイズ。
  • current (long*): 調整後のオフセットを格納するポインタ（NULL可）。
  • next (long*): 調整および加算後のオフセットを格納するポインタ（NULL可）。

前提条件と不変条件

  • alignment は正の整数であること。
  • current または next が NULL の場合、そのポインタへの書き込みは行われない。

アルゴリズムまたは振る舞い

1. offset が alignment の倍数でない場合、offset を次の境界まで切り上げる。 2. current が非 NULL なら、調整後の offset を *current に代入する。 3. offset に size を加算する。 4. next が非 NULL なら、加算後の offset を *next に代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/sqlda.c 内の各関数（sqlda_compat_total_size 等）から、SQLDA 構造体のメモリレイアウト計算のために呼び出される。
  • **呼び出し先**: 本関数はユーティリティであり、他の関数を呼び出さない。

関連データ構造

  • SQLDA 構造体（sqlda-compat.h / sqlda-native.h）のメモリ配置計算に使用される。

失敗モードと境界条件

  • alignment に 0 を指定した場合、除算エラーが発生する可能性がある。
  • offset が負の値の場合の挙動は未定義。

保守担当者向けメモ

  • 本関数は ecpglib における動的 SQLDA 構築時のメモリ管理の根幹を担う。
  • 修正時は、Informix 互換モードとネイティブモードの両方でアライメント規則が正しく適用されているか確認すること。
