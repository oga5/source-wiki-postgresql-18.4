TidOpExpr

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeTidrangescan.c:45

目的

TID範囲スキャンにおいて、ctid（物理的なタプル位置）に対する比較条件を保持し、実行時に評価するための構造体です。

シグネチャまたは形状

typedef struct TidOpExpr
{
	TidExprType exprtype;		/* 演算子の種類 (下限または上限) */
	ExprState  *exprstate;		/* TIDを生成するサブ式の状態 */
	bool		inclusive;		/* 演算子が包含的かどうか */
} TidOpExpr;

入力と出力

  • **入力**: TidExprType（下限/上限の識別）、ExprState（評価対象の式）、bool（包含性のフラグ）。
  • **出力**: 本構造体自体が TidRangeScanState 内のリスト（trss_tidexprs）の一部として保持され、TidRangeEval による範囲算出に使用されます。

前提条件と不変条件

  • exprstate は、実行時に ItemPointer を生成する有効な式である必要があります。
  • exprtype は、当該式がスキャン範囲の下限を定義するものか、上限を定義するものかを明確に区別する必要があります。

アルゴリズムまたは振る舞い

  • TidRangeEval 実行時に、保持している exprstate を評価して ctid 値を取得します。
  • inclusive フラグに基づき、比較演算子（<, <=, >, >=）の境界条件を適切に処理します。非包含（inclusive = false）の場合、ItemPointerInc や ItemPointerDec を用いた正規化処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeTidrangescan.c 内の式構築関数（MakeTidOpExpr 等）によってインスタンス化されます。
  • **呼び出し先**: TidRangeEval が本構造体を参照し、式評価を実行します。
  • **注記**: 呼び出し元・呼び出し先の詳細なコールグラフは提供されていません。

関連データ構造

  • TidExprType: exprtype フィールドで使用される列挙型。
  • ExprState: 式評価の状態を保持する構造体。
  • TidRangeScanState: 本構造体のリスト（trss_tidexprs）を保持する実行ノードの状態構造体。

失敗モードと境界条件

  • exprstate の評価が NULL または無効な ItemPointer を返した場合、スキャン範囲の算出に失敗する可能性があります。
  • 範囲が空（下限 > 上限）と判定された場合、スキャンはスキップされます。

保守担当者向けメモ

  • TidRangeEval 内での ItemPointer 比較ロジックを変更する際は、本構造体の inclusive フラグの解釈と整合性が取れているか確認してください。
  • src/include/nodes/execnodes.h の TidRangeScanState 定義と併せて参照してください。
