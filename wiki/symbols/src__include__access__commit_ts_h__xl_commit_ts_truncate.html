xl_commit_ts_truncate

定義

  • 種別: struct
  • 定義箇所: src/include/access/commit_ts.h:60

目的

コミット時刻管理用 SLRU（Simple Least Recently Used）キャッシュにおいて、古いトランザクションデータを切り捨てる（Truncate）際の WAL 記録用データ構造です。

シグネチャまたは形状

typedef struct xl_commit_ts_truncate
{
	int64		pageno;
	TransactionId oldestXid;
} xl_commit_ts_truncate;

入力と出力

  • **入力**:
  • pageno: 切り捨て対象となる SLRU ページ番号。
  • oldestXid: 切り捨て後に保持される最も古いトランザクション ID。
  • **出力**: なし（WAL レコードとして永続化されるデータ構造）。

前提条件と不変条件

  • 本構造体は track_commit_timestamp が有効な環境でのみ使用されます。
  • pageno は SLRU の論理ページ番号を指します。

アルゴリズムまたは振る舞い

  • チェックポイント処理等で古いコミット時刻データを削除する際、この構造体に情報を格納して WAL に書き込みます。
  • リカバリ時には、この WAL レコードを読み取ることで、どのページまでが切り捨てられたかを再構築し、一貫性を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/commit_ts.c 内の切り捨て処理関数（TruncateCommitTs 等）から、WAL 記録時にインスタンス化されます。
  • **呼び出し先**: src/backend/access/rmgrdesc/committsdesc.c が WAL レコードのデコード（記述）処理で本構造体を参照します。

関連データ構造

  • SizeOfCommitTsTruncate: 本構造体のサイズを計算するためのマクロ（offsetof を使用）。

失敗モードと境界条件

  • 呼び出し元および呼び出し先の詳細なエラーハンドリングについては、提供されたソース範囲内では言及されていません。

保守担当者向けメモ

  • 本構造体は WAL レコードのフォーマットを規定するため、フィールドの追加や変更を行う際は、リカバリ時の互換性に注意してください。
  • SizeOfCommitTsTruncate マクロが正しく定義されているか確認してください。
