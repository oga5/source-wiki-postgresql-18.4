bbsink_gzip_manifest_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_gzip.c:277

目的

gzip圧縮シンクにおいて、圧縮対象外であるマニフェストデータを後続のシンクへ透過的に転送する。

シグネチャまたは形状

static void bbsink_gzip_manifest_contents(bbsink *sink, size_t len)

入力と出力

  • sink: 現在の bbsink 構造体へのポインタ。
  • len: 転送するデータのバイト長。
  • 出力: なし（副作用として後続シンクのバッファへデータをコピーし、転送処理を呼び出す）。

前提条件と不変条件

  • sink および sink->bbs_next が有効なメモリ領域を指していること。
  • sink->bbs_buffer に少なくとも len バイトのデータが存在すること。
  • sink->bbs_next->bbs_buffer が len バイト以上の書き込みを許容するサイズであること。

アルゴリズムまたは振る舞い

1. sink->bbs_buffer から sink->bbs_next->bbs_buffer へ len バイトのデータを memcpy でコピーする。 2. bbsink_manifest_contents を呼び出し、後続のシンクに対してマニフェストデータの転送を指示する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink フレームワークのメソッドテーブル（bbsink_gzip 構造体の .manifest_contents フィールド）を通じて呼び出される。
  • 呼び出し先: bbsink_manifest_contents（後続のシンクに対する処理）。

関連データ構造

  • bbsink: バックアップデータのパイプライン処理を行うための基底構造体。

失敗モードと境界条件

  • len がバッファサイズを超過する場合、メモリ破壊が発生する可能性がある（本関数内でのチェックは行われていない）。

保守担当者向けメモ

  • 本関数はマニフェストファイルが圧縮対象外であるために実装されている。
  • HAVE_LIBZ が定義されている環境でのみ有効である。
