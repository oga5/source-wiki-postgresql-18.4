div100

定義

  • 種別: function
  • 定義箇所: src/common/d2s_intrinsics.h:162

目的

64ビット整数を100で除算した結果を高速に算出する。Ryu浮動小数点変換アルゴリズムにおいて、定数除算を最適化するために使用される。

シグネチャまたは形状

static inline uint64 div100(const uint64 x)

入力と出力

  • **入力**: x (uint64) - 除算対象の符号なし64ビット整数。
  • **出力**: uint64 - x / 100 の計算結果。

前提条件と不変条件

  • 本関数は、32ビット環境において高コストな除算命令を回避するため、マジックナンバーを用いた乗算とビットシフトに置き換えられる設計となっている。
  • umulh 関数が利用可能であること（src/common/d2s_intrinsics.h 内で定義）。

アルゴリズムまたは振る舞い

入力値 x に対して以下の演算を行う： 1. x を右に2ビットシフトする。 2. 定数 0x28F5C28F5C28F5C3 との積の上位64ビットを umulh で算出する。 3. 得られた結果をさらに右に2ビットシフトして返す。 これにより、ハードウェアの除算命令を使用せずに x / 100 を実現する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/d2s.c (行567, 568, 573) にて、変数 vp, vm, vr の除算に使用されている。
  • **呼び出し先**: umulh 関数を呼び出す。

関連データ構造

  • uint64: 64ビット符号なし整数型。

失敗モードと境界条件

  • 入力値が非常に大きい場合、マジックナンバーによる乗算の精度範囲内である必要がある。
  • 32ビット環境と64ビット環境で実装が分岐する可能性がある（HAS_64_BIT_INTRINSICS マクロ等の影響）。

保守担当者向けメモ

  • 本関数で使用されているマジックナンバー 0x28F5C28F5C28F5C3 は、Clang x64 が生成するコードと同等の定数である。数学的な正当性は検証済みだが、変更時には再検証が必須である。
  • umulh の実装がプラットフォーム間で一貫していることを前提としている。
