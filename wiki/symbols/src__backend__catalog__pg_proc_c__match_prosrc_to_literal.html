match_prosrc_to_literal

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_proc.c:1147

目的

SQL文字列リテラルとソース文字列を比較し、リテラル内のエスケープ（バックスラッシュや二重引用符）を考慮した上で、カーソル位置を調整する。

シグネチャまたは形状

static bool match_prosrc_to_literal(const char *prosrc, const char *literal, int cursorpos, int *newcursorpos)

入力と出力

  • prosrc: 比較対象のソース文字列。
  • literal: 比較対象のSQLリテラル文字列。
  • cursorpos: 現在のカーソル位置（文字単位）。
  • newcursorpos: （出力）一致した場合の調整後のカーソル位置。
  • 戻り値: 一致した場合は true、それ以外は false。

前提条件と不変条件

  • prosrc および literal はヌル終端文字列であること。
  • 文字単位の比較を行うため、マルチバイト文字に対応している。
  • 行を跨ぐリテラル構文には対応していない。

アルゴリズムまたは振る舞い

1. prosrc を終端まで走査し、literal と文字単位で比較する。 2. literal 内のバックスラッシュ（\）または二重引用符（''）を検出した場合、カーソル位置より前であれば newcp をインクリメントして調整する。 3. pg_mblen_cstr を使用してマルチバイト文字の長さを考慮しつつ比較を行う。 4. 比較中に不一致が発生した場合は fail ラベルへ遷移する。 5. 最後に literal が単一の引用符で終了していることを確認し、成功時に newcursorpos を更新して true を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_proc.c 内の関数（match_prosrc_to_query 等から参照されている）。
  • 呼び出し先: pg_mblen_cstr（マルチバイト文字長取得）、strncmp（文字列比較）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • literal が期待されるSQLリテラル形式（引用符のペアなど）でない場合、false を返す。
  • 比較中に prosrc と literal が一致しない場合、false を返す。
  • 失敗時にも *newcursorpos に newcp の値を代入し、コンパイラの警告を抑制する。

保守担当者向けメモ

  • 本関数はカーソル位置の計算に依存しているため、リテラル内のエスケープ処理を変更する際は、newcp のインクリメントロジックが正確であることを確認すること。
  • 行を跨ぐリテラルへの対応が必要な場合は、現在の while ループの構造を大幅に見直す必要がある。
