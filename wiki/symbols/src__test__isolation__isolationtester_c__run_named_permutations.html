run_named_permutations

定義

  • 種別: function
  • 定義箇所: src/test/isolation/isolationtester.c:485

目的

TestSpec 内に定義された名前付き順列（Permutation）を順次実行する。

シグネチャまたは形状

static void run_named_permutations(TestSpec *testspec)

入力と出力

  • 入力: TestSpec *testspec （実行対象のテスト仕様構造体へのポインタ）
  • 出力: なし（void）

前提条件と不変条件

  • testspec は有効なポインタであること。
  • testspec->permutations には実行可能な順列が格納されていること。

アルゴリズムまたは振る舞い

1. testspec->npermutations で指定された回数だけループする。 2. 各ループにおいて、testspec->permutations[i] から順列データを取り出す。 3. 取り出した順列のステップ数（nsteps）とステップ配列（steps）を引数として run_permutation を呼び出す。 4. すべての順列の実行が完了するまで繰り返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/isolation/isolationtester.c:397（run_testspec 関数内）
  • 呼び出し先: run_permutation

関連データ構造

  • TestSpec: テスト仕様全体を保持する構造体。
  • Permutation: 個別の順列定義を保持する構造体。

失敗モードと境界条件

  • testspec->npermutations が 0 の場合、ループは実行されず何も行われない。
  • testspec->permutations が NULL または不正なポインタである場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は run_testspec から呼び出されるヘルパー関数であり、テスト仕様に明示的な順列が定義されている場合にのみ実行される。
  • 順列の実行順序は testspec 内の配列順序に依存する。
