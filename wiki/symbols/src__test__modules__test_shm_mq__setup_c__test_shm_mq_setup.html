test_shm_mq_setup

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_shm_mq/setup.c:49

目的

共有メモリメッセージキュー（shm_mq）を用いたテスト環境の構築。動的共有メモリ（DSM）の確保、バックグラウンドワーカーの登録・起動、およびキューのアタッチを一括して行う。

シグネチャまたは形状

void test_shm_mq_setup(int64 queue_size, int32 nworkers, dsm_segment **segp,
                       shm_mq_handle **output, shm_mq_handle **input)

入力と出力

  • **入力**:
  • queue_size: 各キューのサイズ。
  • nworkers: 起動するバックグラウンドワーカーの数。
  • **出力**:
  • segp: 確保された dsm_segment へのポインタ。
  • output: 送信側メッセージキューのハンドル。
  • input: 受信側メッセージキューのハンドル。

前提条件と不変条件

  • nworkers は少なくとも1以上である必要がある（wstate->handle[nworkers - 1] へのアクセスが発生するため）。
  • 呼び出し元は、本関数が確保した dsm_segment を適切に管理・解放する責任を負う。

アルゴリズムまたは振る舞い

1. setup_dynamic_shared_memory を呼び出し、DSMセグメントの確保とキュー領域の初期化を行う。 2. setup_background_workers を呼び出し、指定された数のワーカーを登録・起動する。 3. shm_mq_attach を使用して、出力用および入力用のメッセージキューを現在のプロセスにアタッチする。 4. wait_for_workers_to_become_ready を呼び出し、全ワーカーが準備完了状態になるまで待機する。 5. cancel_on_dsm_detach を使用して、プロセス終了時の自動クリーンアップフック（cleanup_background_workers）を登録し、一時的な wstate 構造体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_shm_mq/test.c (テスト実行のメインロジック)
  • **呼び出し先**:
  • setup_dynamic_shared_memory
  • setup_background_workers
  • shm_mq_attach
  • wait_for_workers_to_become_ready
  • cancel_on_dsm_detach
  • cleanup_background_workers
  • pfree

関連データ構造

  • dsm_segment: 共有メモリセグメント。
  • shm_mq_handle: メッセージキューの操作ハンドル。
  • test_shm_mq_header: 共有メモリ上の制御ヘッダー。
  • worker_state: 起動したワーカーの情報を保持する内部構造体。

失敗モードと境界条件

  • ワーカーの起動失敗や準備完了までの待機タイムアウトが発生した場合、内部でエラーが発生しテストが中断される可能性がある。
  • nworkers が0以下の場合、配列アクセスで境界外参照が発生するリスクがある。

保守担当者向けメモ

  • 本関数は TopTransactionContext で実行されることを想定している。
  • ワーカーの生存期間はメッセージキューのシャットダウンに依存しており、本関数終了後に明示的な cleanup_background_workers がフックとして機能する設計となっている。
