brin_xlog_insert

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_xlog.c:123

目的

BRINインデックスへの挿入操作に関するWALレコードをリプレイし、インデックスページおよびREVMAP（範囲マップ）の整合性を復元します。

シグネチャまたは形状

static void brin_xlog_insert(XLogReaderState *record);

入力と出力

  • **入力**: XLogReaderState *record - リプレイ対象のWALレコード。
  • **出力**: なし（副作用としてインデックスページおよびREVMAPページを更新）。

前提条件と不変条件

  • record は XLOG_BRIN_INSERT 型のWALレコードを指している必要があります。
  • XLogRecGetData(record) が xl_brin_insert 構造体としてキャスト可能である必要があります。

アルゴリズムまたは振る舞い

1. XLogRecGetData を呼び出し、WALレコードから xl_brin_insert データを取得します。 2. 取得したデータを引数として brin_xlog_insert_update を呼び出し、実際のページ更新処理（REVMAPの更新およびインデックスページへのタプル追加）を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_redo（WALリプレイのディスパッチャ）から呼び出されます。
  • **呼び出し先**: brin_xlog_insert_update を呼び出します。

関連データ構造

  • xl_brin_insert: src/include/access/brin_xlog.h で定義される、挿入操作のWALレコード構造体。
  • XLogReaderState: WALレコードを読み取るための汎用状態構造体。

失敗モードと境界条件

  • WALレコードの形式が不正な場合、XLogRecGetData のキャストやその後の brin_xlog_insert_update 内でのバッファ操作においてエラーが発生する可能性があります。
  • 参照されるページが破損している場合、brin_xlog_insert_update が内部でエラーを発生させます。

保守担当者向けメモ

  • 本関数は brin_xlog_insert_update への単なるラッパーです。挿入ロジックの変更は、主に brin_xlog_insert_update 側で行う必要があります。
  • brin_xlog_update も同様に brin_xlog_insert_update を利用しており、挿入と更新で共通のREVMAP更新ロジックが共有されている点に留意してください。
