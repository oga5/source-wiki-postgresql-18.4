PublicationActions

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/pg_publication.h:78

目的

論理レプリケーションにおいて、特定のパブリケーションで許可されるデータ操作（DML）の種類を管理するためのフラグ構造体です。

シグネチャまたは形状

typedef struct PublicationActions
{
	bool		pubinsert;
	bool		pubupdate;
	bool		pubdelete;
	bool		pubtruncate;
} PublicationActions;

入力と出力

  • **入力**: 各フィールドは bool 型であり、対応する操作がパブリケーションで有効であれば true、無効であれば false を保持します。
  • **出力**: レプリケーションロジックが、特定の操作をパブリッシュすべきかどうかを判定するためのフラグとして利用されます。

前提条件と不変条件

  • 各フィールドは独立した操作の有効/無効を制御します。
  • パブリケーションの定義に従い、これらのフラグはシステムカタログ pg_publication の情報に基づいて初期化されます。

アルゴリズムまたは振る舞い

パブリケーションに関連付けられたテーブルに対して操作が発生した際、この構造体の各フラグを参照することで、変更内容をレプリケーションストリームに含めるかどうかが決定されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/publicationcmds.c（パブリケーションの作成や変更時）
  • src/backend/replication/pgoutput/pgoutput.c（レプリケーション出力プラグインによる判定時）
  • **呼び出し先**:
  • 提供されたソース情報には、本構造体から直接呼び出される関数や、本構造体を引数として受け取る具体的な関数の詳細なシグネチャは含まれていません。

関連データ構造

  • Publication: バックエンドでパブリケーション情報を保持するメモリ内表現。
  • pg_publication: システムカタログの行構造体（FormData_pg_publication）。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、フラグがすべて false に設定されている場合、そのパブリケーションは実質的にどのデータ変更もレプリケートしません。

保守担当者向けメモ

  • 本構造体は src/include/catalog/pg_publication.h で定義されており、論理レプリケーションの挙動を制御する重要なメタデータの一部です。
  • 新たなDML操作がレプリケーション対象として追加される場合は、本構造体へのフィールド追加を検討する必要があります。
