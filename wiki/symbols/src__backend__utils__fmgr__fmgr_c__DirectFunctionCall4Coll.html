DirectFunctionCall4Coll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:858

目的

4つの引数を取る関数を、FmgrInfo構造体の初期化を介さずに直接呼び出すための軽量なラッパー関数です。

シグネチャまたは形状

Datum DirectFunctionCall4Coll(PGFunction func, Oid collation, Datum arg1, Datum arg2, Datum arg3, Datum arg4)

入力と出力

  • **入力**:
  • func: 呼び出す関数のポインタ (PGFunction)。
  • collation: 照合順序 (Oid)。
  • arg1 ～ arg4: 関数に渡す4つの引数 (Datum)。
  • **出力**:
  • 関数の実行結果を Datum 型で返します。

前提条件と不変条件

  • 呼び出される関数は、NULLを返さないことが期待されています。
  • 引数 arg1 ～ arg4 は、呼び出し時に非NULLとして扱われます。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFO マクロを使用して、4つの引数を持つ FunctionCallInfoBaseData をスタック上に確保します。 2. InitFunctionCallInfoData を呼び出し、照合順序と引数情報を設定します。 3. 引数 arg1 ～ arg4 を fcinfo->args 配列に格納し、各 isnull フラグを false に設定します。 4. 関数ポインタ func を介して対象関数を実行します。 5. 実行後、fcinfo->isnull を確認し、関数がNULLを返した場合は elog(ERROR) を発生させます。 6. 正常終了時に result を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/extension.c:1344
  • src/include/fmgr.h:689 (マクロ経由の呼び出し)
  • **呼び出し先**:
  • 引数として渡された PGFunction 型の関数ポインタ。

関連データ構造

  • FunctionCallInfoBaseData: 実行時の引数やコンテキストを保持する構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • 関数がNULLを返した場合、elog(ERROR) によりエラーが発生します。
  • 呼び出し元がNULLを期待していない設計であるため、NULL許容の関数呼び出しには適していません。

保守担当者向けメモ

  • 本関数はパフォーマンスを重視した直接呼び出し用です。ポリモーフィック関数など、FmgrInfo の詳細なメタデータが必要な場合には使用できません。
  • src/include/fmgr.h に定義されたマクロ経由で利用されることが一般的です。
