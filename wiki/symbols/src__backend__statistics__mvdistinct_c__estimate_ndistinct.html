estimate_ndistinct

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mvdistinct.c:520

目的

Duj1推定器を用いて、サンプルデータから母集団全体における一意な値の数（NDistinct）を推定する。

シグネチャまたは形状

double estimate_ndistinct(double totalrows, int numrows, int d, int f1)

入力と出力

  • **入力**:
  • totalrows: 母集団の総行数。
  • numrows: サンプルデータの行数。
  • d: サンプル内での一意な値の数。
  • f1: サンプル内で1回だけ出現する値の数。
  • **出力**:
  • 推定されたNDistinct値（double型、四捨五入済み）。

前提条件と不変条件

  • totalrows は母集団の総行数として正の値である必要がある。
  • 推定結果は、観測された一意な値の数 d 以上、かつ totalrows 以下に収まるようクランプされる。

アルゴリズムまたは振る舞い

1. サンプルデータから得られた d（一意な値の数）と f1（1回のみ出現する値の数）を用い、以下の式で推定値を算出する。

  • numer = numrows * d
  • denom = (numrows - f1) + (f1 * numrows / totalrows)
  • ndistinct = numer / denom

    • 2. 丸め誤差を考慮し、結果を [d, totalrows] の範囲内にクランプする。 3. 最後に floor(ndistinct + 0.5) を行い、整数値に丸めて返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/mvdistinct.c 内の ndistinct_for_combination 関数から呼び出される。
  • **呼び出し先**: math.h の floor 関数。

関連データ構造

  • 特になし（プリミティブな数値型のみを使用）。

失敗モードと境界条件

  • numrows が 0 の場合、除算による不適切な結果が生じる可能性がある（呼び出し元でのガードが必要）。
  • totalrows が 0 の場合、denom の計算でゼロ除算が発生する可能性がある。

保守担当者向けメモ

  • 本関数は単一列の統計収集で用いられるDuj1推定器を多変量に拡張したものである。
  • 推定ロジックを変更する場合は、統計学的な妥当性と、クランプ処理による境界値の整合性に注意すること。
