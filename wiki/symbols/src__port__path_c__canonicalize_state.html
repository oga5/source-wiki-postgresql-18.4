canonicalize_state

定義

  • 種別: enum
  • 定義箇所: src/port/path.c:309

目的

パスの正規化処理における現在の走査状態を保持し、パスの構造（絶対パスか相対パスか、ディレクトリの深さ、親ディレクトリ参照の有無）を追跡するために使用されます。

シグネチャまたは形状

typedef enum
{
    ABSOLUTE_PATH_INIT,
    ABSOLUTE_WITH_N_DEPTH,
    RELATIVE_PATH_INIT,
    RELATIVE_WITH_N_DEPTH,
    RELATIVE_WITH_PARENT_REF,
} canonicalize_state;

入力と出力

  • 本シンボルは列挙型であり、関数ではありません。
  • 状態遷移の変数として、パス走査アルゴリズム内で使用されます。

前提条件と不変条件

  • ABSOLUTE_PATH_INIT: 絶対パスの先頭（ルートディレクトリおよびWindowsのドライブ名を含む）を通過した直後の状態。
  • ABSOLUTE_WITH_N_DEPTH / RELATIVE_WITH_N_DEPTH: 走査中に収集されたディレクトリの深さ（pathdepth）を保持している状態。
  • RELATIVE_WITH_PARENT_REF: 相対パスにおいて、..（親ディレクトリ参照）のみで構成されている状態。

アルゴリズムまたは振る舞い

パスの正規化プロセスにおいて、文字列を走査する際の状態マシンとして機能します。パスのコンポーネントを順次処理し、冗長な区切り文字や .. を解決する際の論理的な現在地を定義します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/path.c 内のパス正規化ロジック（canonicalize_path 等）から参照されます。
  • 呼び出し先: 本シンボル自体はデータ型であるため、呼び出し先はありません。

関連データ構造

  • canonicalize_path: 本列挙型を使用してパスの正規化を行う関数。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 本シンボルは src/port/path.c におけるパス正規化の内部状態を管理するものです。ロジックの変更時には、各状態がパスの走査位置と正しく同期しているかを確認してください。
