parse_dispatch_option

定義

  • 種別: function
  • 定義箇所: src/backend/main/main.c:239

目的

コマンドライン引数（プロセス名やオプション文字列）を解析し、PostgreSQL プロセスが実行すべき起動モード（DispatchOption）を特定する。

シグネチャまたは形状

DispatchOption parse_dispatch_option(const char *name);

入力と出力

  • **入力**: name (解析対象の文字列)
  • **出力**: DispatchOption 列挙型の値。一致するものがない場合はデフォルトとして DISPATCH_POSTMASTER を返す。

前提条件と不変条件

  • DispatchOptionNames 配列が定義されており、各 DispatchOption 値に対応する文字列が格納されていること。
  • EXEC_BACKEND マクロの定義有無により、DISPATCH_FORKCHILD の判定ロジックが変化する。

アルゴリズムまたは振る舞い

1. DispatchOptionNames 配列を順次走査する。 2. 現在のインデックスが DISPATCH_FORKCHILD の場合:

  • EXEC_BACKEND が定義されている環境でのみ、strncmp を用いて前方一致を確認し、一致すれば DISPATCH_FORKCHILD を返す。
  • それ以外の場合はスキップする。

    • 3. その他のオプションについては strcmp で完全一致を確認し、一致すれば対応する DispatchOption を返す。 4. ループ終了まで一致が見つからない場合、DISPATCH_POSTMASTER を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/main/main.c
  • src/backend/bootstrap/bootstrap.c
  • src/backend/postmaster/postmaster.c
  • src/backend/tcop/postgres.c
  • **呼び出し先**:
  • strcmp / strncmp / strlen (標準ライブラリ)

関連データ構造

  • DispatchOption (列挙型)
  • DispatchOptionNames (文字列配列)

失敗モードと境界条件

  • 引数 name が NULL の場合の挙動はソースコード上では明示的に保護されていない。
  • 一致するオプションが存在しない場合、エラーを返さず DISPATCH_POSTMASTER を返す仕様となっている。

保守担当者向けメモ

  • DISPATCH_FORKCHILD は他のオプションと異なり、前方一致（strncmp）で判定される特殊なケースである。このロジックを変更する際は、EXEC_BACKEND 環境への影響を考慮すること。
