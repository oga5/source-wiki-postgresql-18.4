EstimationInfo

定義

  • 種別: struct
  • 定義箇所: src/include/utils/selfuncs.h:80

目的

クエリプランナによるコスト見積もり処理において、推定の特性や状態を示すフラグを保持するために使用されます。

シグネチャまたは形状

typedef struct EstimationInfo
{
    uint32      flags;
} EstimationInfo;

入力と出力

  • **入力**: flags メンバに特定のビットフラグ（例: SELFLAG_USED_DEFAULT）を設定することで、推定ロジックの状態を伝達します。
  • **出力**: 構造体自体が参照渡しされ、呼び出し先でフラグの読み取りや更新が行われます。

前提条件と不変条件

  • flags は、推定処理中に発生した特殊な状況（デフォルト値の使用など）を記録するために使用されます。
  • 使用前に memset 等でゼロ初期化することが推奨されます。

アルゴリズムまたは振る舞い

1. プランナがコスト見積もり関数を呼び出す際、EstimationInfo 構造体のインスタンスを引数として渡します。 2. 推定ロジック内で統計情報が不足している場合、デフォルト値が使用され、その旨を示すフラグが flags に設定されます。 3. 呼び出し元は、このフラグを確認することで、推定結果が統計情報に基づいているか、あるいはフォールバック値によるものかを判断できます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/costsize.c 等のプランナ関連コード。
  • **呼び出し先**: src/backend/utils/adt/selfuncs.c 内の各種選択率推定関数。
  • 補足: 具体的な関数呼び出しの連鎖については、提供されたソース情報からは限定的な参照箇所しか特定できません。

関連データ構造

  • VariableStatData: 統計情報を保持する構造体。本構造体と併用されることが多い。

失敗モードと境界条件

  • 構造体が適切に初期化されていない場合、flags にゴミデータが含まれ、誤った推定状態が伝播する可能性があります。

保守担当者向けメモ

  • flags に追加のビット定義を行う場合は、selfuncs.h 内の既存のフラグ定義と重複しないよう注意してください。
  • SELFLAG_USED_DEFAULT のようなフラグがどのように解釈されるかは、selfuncs.c 内の各推定関数の実装に依存します。
