rerr

定義

  • 種別: struct
  • 定義箇所: src/backend/regex/regerror.c:40

目的

正規表現エンジンにおけるエラーコード、そのシンボル名、および人間が読み取り可能な説明文を保持するためのデータ構造です。

シグネチャまたは形状

struct rerr {
    int         code;    /* エラーコード */
    const char *name;    /* シンボル名 (例: "REG_NOMATCH") */
    const char *explain; /* 説明文 */
};

入力と出力

本シンボルは構造体定義であり、直接的な入力や出力は持ちません。この構造体を用いた静的配列 rerrs が、エラーコードの検索および変換のデータソースとして利用されます。

前提条件と不変条件

  • rerrs 配列の末尾には、番兵として {-1, "", "oops"} が配置されている必要があります。
  • code は一意な整数値である必要があります。

アルゴリズムまたは振る舞い

本構造体は、pg_regerror 関数内での線形探索の対象となります。エラーコードまたは名前による照合を行い、一致した要素から対応するメッセージを取得するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/regex/regerror.c 内の pg_regerror 関数が、この構造体型のポインタ変数 r を介して配列 rerrs を走査します。
  • 呼び出し先: なし（データ構造定義のため）。

関連データ構造

  • rerrs: rerr 型の要素を持つ静的な配列。regex/regerrs.h の内容を保持します。

失敗モードと境界条件

  • 該当するエラーコードが見つからない場合、pg_regerror は未知のエラーを示すテンプレート文字列を使用します。
  • 番兵（code == -1）に到達した時点で探索は終了します。

保守担当者向けメモ

  • rerrs 配列の定義は regex/regerrs.h に依存しています。エラーコードを追加・変更する場合は、ヘッダーファイルと本構造体の整合性を確認してください。
