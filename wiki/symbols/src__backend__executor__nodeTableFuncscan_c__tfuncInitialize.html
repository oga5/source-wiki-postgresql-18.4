tfuncInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTableFuncscan.c:339

目的

テーブル関数（XMLTABLE や JSON_TABLE 等）の実行に必要な名前空間、行フィルタ、および列フィルタの各式を評価し、解析エンジン（TableFuncRoutine）へ設定する。

シグネチャまたは形状

static void tfuncInitialize(TableFuncScanState *tstate, ExprContext *econtext, Datum doc)

入力と出力

  • tstate: テーブル関数スキャンノードの実行状態。
  • econtext: 式評価に使用するコンテキスト。
  • doc: 解析対象となる入力ドキュメント（Datum）。
  • 出力: なし（副作用として tstate->routine が保持する内部状態を更新する）。

前提条件と不変条件

  • tstate->routine が適切に初期化されていること。
  • econtext が式評価可能な状態であること。
  • doc が有効なデータ型であること。

アルゴリズムまたは振る舞い

1. **ドキュメント設定**: routine->SetDocument を呼び出し、入力ドキュメントをコンテキストに登録する。 2. **名前空間評価**: ns_uris と ns_names を反復し、各名前空間 URI を評価して routine->SetNamespace で登録する。 3. **行フィルタ設定**: routine->SetRowFilter が定義されている場合、rowexpr を評価し、その結果をフィルタとして登録する。 4. **列フィルタ設定**: colexprs を反復し、各列のフィルタ式を評価する。式が未定義の場合は列名自体をフィルタとして使用し、routine->SetColumnFilter で登録する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/executor/nodeTableFuncscan.c 内の tfuncFetchRows（306行目付近）。
  • 呼び出し先: ExecEvalExpr（式評価）、routine 構造体内の関数ポインタ（SetDocument, SetNamespace, SetRowFilter, SetColumnFilter）。

関連データ構造

  • TableFuncScanState: テーブル関数スキャンの実行状態を保持する構造体。
  • TableFuncRoutine: XML や JSON の解析ルーチンを定義する構造体。
  • ExprContext: 式評価のためのメモリ領域と状態。

失敗モードと境界条件

  • **NULL 値エラー**: 名前空間 URI、行フィルタ式、または列フィルタ式の評価結果が NULL の場合、ERRCODE_NULL_VALUE_NOT_ALLOWED でエラーを報告する。
  • **列インデックス**: ordinalitycol（順序列）に該当する列は、フィルタ設定の対象外となる。

保守担当者向けメモ

  • 本関数は tfuncFetchRows から呼び出され、解析の準備フェーズを担う。
  • routine 経由で呼び出される各関数は、解析対象のデータ形式（XML/JSON）に依存する動的な処理である。
  • colno のインクリメントと ordinalitycol のチェックは、列フィルタの適用順序と整合している必要がある。
