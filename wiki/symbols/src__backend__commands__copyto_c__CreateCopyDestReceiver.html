CreateCopyDestReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyto.c:1434

目的

COPY TO 操作において、クエリ結果を処理するための DestReceiver オブジェクトを生成および初期化します。

シグネチャまたは形状

DestReceiver *CreateCopyDestReceiver(void)

入力と出力

  • **入力**: なし
  • **出力**: DestReceiver 型へのポインタ。実際には DR_copy 構造体として割り当てられ、DestReceiver としてキャストされて返されます。

前提条件と不変条件

  • 呼び出し時に palloc を介してメモリが確保されます。
  • 返される DestReceiver の mydest フィールドは DestCopyOut に設定されます。
  • DR_copy 構造体内の cstate は初期状態で NULL に設定されます（後続の処理で設定されることが前提）。

アルゴリズムまたは振る舞い

1. DR_copy 構造体サイズ分のメモリを palloc で確保します。 2. DestReceiver インターフェースの関数ポインタを以下の通り設定します。

  • receiveSlot: copy_dest_receive
  • rStartup: copy_dest_startup
  • rShutdown: copy_dest_shutdown
  • rDestroy: copy_dest_destroy

    • 3. mydest を DestCopyOut に設定します。 4. cstate を NULL に、processed を 0 に初期化します。 5. 構築したオブジェクトを DestReceiver 型として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/dest.c:145
  • **呼び出し先**:
  • palloc (メモリ確保)
  • copy_dest_receive, copy_dest_startup, copy_dest_shutdown, copy_dest_destroy (関数ポインタへの代入先)

関連データ構造

  • DR_copy: COPY TO 用の DestReceiver 実装構造体。
  • DestReceiver: クエリ結果を受け取るための汎用インターフェース。
  • CopyToStateData: cstate フィールドを通じて間接的に関連します。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • cstate が NULL のまま使用された場合、後続の DestReceiver メソッド内で不具合が発生する可能性があります。

保守担当者向けメモ

  • 本関数は COPY TO の出力パイプラインの起点となります。DestReceiver の各メソッド（receiveSlot 等）の実装が変更された場合、本関数での初期化ロジックも整合性を確認してください。
