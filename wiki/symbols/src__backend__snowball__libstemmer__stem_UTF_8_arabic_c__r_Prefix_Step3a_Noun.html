目的

アラビア語のステミング処理において、名詞の特定の接頭辞を識別し、条件を満たす場合に除去する。

シグネチャまたは形状

static int r_Prefix_Step3a_Noun(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体）
  • 出力: int（処理が行われた場合は 1、それ以外は 0。内部エラー時は負の値を返す）

前提条件と不変条件

  • z->c が現在のカーソル位置を示す。
  • a_6 テーブル（among 構造体）が定義されている必要がある。
  • UTF-8 エンコーディングのバイト列を操作するため、len_utf8 による文字数判定が必須である。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c を z->bra に設定する。 2. 文字列の残り長さを確認し、特定のバイト値（132 または 167）が 4 文字目以降に存在するかを検証する。 3. find_among を使用して a_6 テーブルと照合し、一致した場合は among_var を取得する。 4. 一致した場合、z->ket を現在のカーソル位置に設定する。 5. among_var の値に応じて以下の条件で文字列を削除する:

  • case 1: 文字列長が 5 文字より大きい場合に slice_del を実行。
  • case 2: 文字列長が 4 文字より大きい場合に slice_del を実行。

    • 6. 成功時は 1 を返し、条件不一致や削除失敗時は 0 またはエラーコードを返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_arabic.c 内のメイン処理フローから参照されている。
  • 呼び出し先: find_among, len_utf8, slice_del（いずれも Snowball ランタイム関数）。

関連データ構造

  • SN_env: ステミングの実行状態を保持する構造体。
  • a_6: 接頭辞マッチングに使用される among 構造体の配列。

失敗モードと境界条件

  • 文字列長が閾値（4 または 5 文字）以下の場合は削除を行わず 0 を返す。
  • find_among が一致しない場合は 0 を返す。
  • slice_del が内部エラー（負の値）を返した場合は、その値をそのまま伝播させる。
