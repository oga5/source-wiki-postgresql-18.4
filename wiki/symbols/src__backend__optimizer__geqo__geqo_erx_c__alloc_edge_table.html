alloc_edge_table

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_erx.c:55

目的

GEQO（遺伝的クエリ最適化）におけるエッジ組み換え交叉（ERX）アルゴリズムで使用する、エッジ情報を格納するためのメモリ領域を確保する。

シグネチャまたは形状

Edge *alloc_edge_table(PlannerInfo *root, int num_gene)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体へのポインタ（使用されない）。
  • num_gene: 遺伝子（都市）の総数。
  • **出力**:
  • Edge *: 確保された Edge 構造体の配列へのポインタ。

前提条件と不変条件

  • 戻り値の配列は num_gene + 1 個の要素を持ち、インデックス 1 から num_gene までを直接参照可能である（インデックス 0 は使用されない）。

アルゴリズムまたは振る舞い

1. palloc を使用して、num_gene + 1 個の Edge 構造体分に相当するメモリを動的に確保する。 2. 確保したメモリ領域の先頭アドレスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_main.c:142
  • **呼び出し先**:
  • palloc（PostgreSQLのメモリ割り当て関数）

関連データ構造

  • Edge: GEQOにおいて都市間の接続関係を保持する構造体。

失敗モードと境界条件

  • palloc がメモリ不足に陥った場合、PostgreSQLの標準的なエラー処理によりアボートされる可能性がある。

保守担当者向けメモ

  • 本関数で確保されたメモリは、対応する free_edge_table 関数によって解放される必要がある。
  • root 引数は現在の実装では使用されていないが、将来的な拡張やインターフェースの整合性のために保持されている。
