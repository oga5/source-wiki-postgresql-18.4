entryLoadMoreItems

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginget.c:656

目的

GINインデックススキャンにおいて、現在のページで処理可能なアイテムを使い果たした際、次のアイテム群をインデックスから読み込み、GinScanEntry の状態を更新する。

シグネチャまたは形状

static void entryLoadMoreItems(GinState *ginstate, GinScanEntry entry, ItemPointerData advancePast)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの状態管理構造体。
  • entry: 走査対象の GinScanEntry。
  • advancePast: 現在の走査位置を示す ItemPointer。これより大きいアイテムを探す。
  • **出力**: なし（entry 構造体の buffer, list, offset, isFinished を直接更新）。

前提条件と不変条件

  • entry->buffer が有効な場合、そのページは共有ロック（GIN_SHARE）状態で保持されるか、処理終了時にアンロックされる。
  • advancePast は、次に取得すべきアイテムの探索開始点として使用される。

アルゴリズムまたは振る舞い

1. **初期チェック**: entry->buffer が無効であれば isFinished を true にして終了。 2. **ページ探索戦略**:

  • advancePast が現在の curItem と一致する場合、現在のページから右方向へ走査（stepright）。
  • 一致しない場合、ginFindLeafPage を用いてインデックスをルートから再降下し、該当するリーフページを特定。

    • 3. **ページ走査ループ**:

  • stepright が真の場合、右隣のページへ移動。右端ページであれば終了。
  • GIN_DELETED フラグが立っているページはスキップ。
  • ページ境界チェックを行い、advancePast がページの右境界より大きい場合はさらに右へ移動。
  • GinDataLeafPageGetItems を呼び出し、ページ内のアイテムリストを取得。
  • リストを走査し、advancePast より大きいアイテムが見つかった時点で entry->offset を設定してリターン。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginget.c 内の keyGetItem（955行目付近）。
  • **呼び出し先**:
  • ginCompareItemPointers, ginFindLeafPage, ginStepRight, GinDataLeafPageGetItems
  • LockBuffer, ReleaseBuffer, UnlockReleaseBuffer, BufferGetPage
  • pfree (メモリ解放)

関連データ構造

  • GinScanEntry: スキャン中のエントリ状態を保持。
  • GinBtreeStack: インデックス走査時のスタック構造。
  • ItemPointerData: TID（タプル識別子）。

失敗モードと境界条件

  • **右端ページ**: 走査中に右端ページに到達した場合、isFinished を true に設定し、バッファを解放して終了する。
  • **削除済みページ**: GIN_DELETED フラグを持つページは無視され、次のページへ進む。
  • **メモリ不足**: pfree によるリスト解放と再確保が行われる。

保守担当者向けメモ

  • 本関数は、インデックスのページ分割や同時実行によるバキューム（GIN_DELETED）を考慮した設計となっている。
  • advancePast を用いた再探索ロジックは、インデックスの構造変更に追従するための重要なパスである。
