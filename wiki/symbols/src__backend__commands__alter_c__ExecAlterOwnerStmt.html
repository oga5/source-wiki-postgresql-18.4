ExecAlterOwnerStmt

定義

  • 種別: function
  • 定義箇所: src/backend/commands/alter.c:836

目的

SQLの ALTER ... OWNER TO ステートメントを実行し、指定されたデータベースオブジェクトの所有者を変更する。

シグネチャまたは形状

ObjectAddress ExecAlterOwnerStmt(AlterOwnerStmt *stmt)

入力と出力

  • **入力**: AlterOwnerStmt *stmt（変更対象のオブジェクト型、識別子、および新しい所有者情報を含む構造体）
  • **出力**: ObjectAddress（変更されたオブジェクトのカタログ上の位置情報）

前提条件と不変条件

  • get_rolespec_oid により、新しい所有者のOIDが解決可能であること。
  • 対象オブジェクトがサポートされている型であること（未対応の型が渡された場合はエラーとなる）。

アルゴリズムまたは振る舞い

1. get_rolespec_oid を呼び出し、新しい所有者のOIDを取得する。 2. stmt->objectType に基づき、以下のいずれかの処理を行う：

  • **専用関数へのディスパッチ**: OBJECT_DATABASE, OBJECT_SCHEMA, OBJECT_TYPE 等の特定のオブジェクト型については、専用の Alter...Owner 関数を呼び出す。
  • **汎用的な処理**: OBJECT_AGGREGATE, OBJECT_FUNCTION 等の汎用的なオブジェクト型については、get_object_address で対象のOIDとクラスIDを特定し、AlterObjectOwner_internal を呼び出して所有者を更新する。

    • 3. 処理したオブジェクトの ObjectAddress を返す。 4. 未知のオブジェクト型が指定された場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（パーサーから渡されたステートメントの実行時）
  • **呼び出し先**:
  • get_rolespec_oid
  • 各種 Alter...Owner 関数（AlterDatabaseOwner, AlterSchemaOwner 等）
  • get_object_address
  • AlterObjectOwner_internal

関連データ構造

  • AlterOwnerStmt: 変更対象のオブジェクト情報と新しい所有者情報を保持するノード。
  • ObjectAddress: オブジェクトのカタログ識別子（classId, objectId, objectSubId）。

失敗モードと境界条件

  • 指定された新しい所有者が存在しない場合、get_rolespec_oid がエラーを発生させる。
  • サポートされていないオブジェクト型が渡された場合、elog(ERROR) により処理が中断される。
  • get_object_address が対象オブジェクトを見つけられない場合、エラーが発生する。

保守担当者向けメモ

  • 新しいオブジェクト型をサポートする場合、switch 文の汎用ケースに追加するか、専用の Alter...Owner 関数を実装してディスパッチ先に追加する必要がある。
  • 汎用ケースで処理する場合は、get_object_attnum_... 系関数が正しく実装されていることが前提となる。
