JsonManifestParseIncrementalState

定義

  • **種別**: struct
  • **定義箇所**: src/common/parse_manifest.c:95

目的

バックアップマニフェスト（JSON形式）を分割して解析（インクリメンタル解析）する際に、解析の進捗状態とチェックサム計算のコンテキストを保持するために使用されます。

シグネチャまたは形状

struct JsonManifestParseIncrementalState
{
    JsonLexContext lex;             /* JSON字句解析の状態 */
    JsonSemAction sem;              /* JSON解析時のアクション定義 */
    pg_cryptohash_ctx *manifest_ctx; /* マニフェスト全体のチェックサム計算用コンテキスト */
};

入力と出力

  • **入力**: json_parse_manifest_incremental_init 関数により初期化され、json_parse_manifest_incremental_chunk を通じてJSONデータの断片が入力されます。
  • **出力**: 解析終了時に json_parse_manifest_incremental_shutdown を呼び出すことで、リソースの解放と最終的な検証が行われます。

前提条件と不変条件

  • json_parse_manifest_incremental_init によって適切にメモリ確保および初期化されている必要があります。
  • インクリメンタル解析の最終チャンクには、マニフェスト全体の整合性を検証するためのチェックサムが含まれている必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: JsonLexContext と JsonSemAction を設定し、SHA-256計算用の manifest_ctx を準備します。 2. **チャンク処理**: json_parse_manifest_incremental_chunk を呼び出すたびに、提供されたデータがJSONパーサーへ渡され、同時に manifest_ctx を通じてチェックサム計算が更新されます。 3. **終了**: json_parse_manifest_incremental_shutdown により、解析状態のクリーンアップと、蓄積されたチェックサムを用いたマニフェスト全体の検証が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c
  • src/bin/pg_combinebackup/load_manifest.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • **呼び出し先**:
  • json_parse_manifest_incremental_init (初期化)
  • json_parse_manifest_incremental_chunk (データ処理)
  • json_parse_manifest_incremental_shutdown (終了処理)

関連データ構造

  • JsonLexContext: JSONの字句解析状態を保持。
  • JsonSemAction: JSON解析時のコールバック関数群を保持。
  • pg_cryptohash_ctx: チェックサム計算（SHA-256）の状態を保持。

失敗モードと境界条件

  • 解析中にエラーが発生した場合、json_manifest_parse_failure が呼び出され、pg_unreachable() により処理が停止します。呼び出し元のコールバックからは復帰しません。
  • メモリ不足時には palloc によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はフロントエンド（postgres_fe.h）での利用を前提としています。
  • マニフェストの解析ロジックを変更する際は、JsonSemAction に設定されているコールバック関数群との整合性に注意してください。
