LINE

定義

  • 種別: struct
  • 定義箇所: src/include/utils/geo_decls.h:128

目的

2次元平面上の直線を表現するためのデータ構造です。方程式 $Ax + By + C = 0$ の係数を保持します。

シグネチャまたは形状

typedef struct
{
	float8		A,
				B,
				C;
} LINE;

入力と出力

  • **入力**: 構造体初期化時に float8 型の係数 A, B, C を受け取ります。
  • **出力**: 幾何演算関数（line_sl, line_invsl 等）において、直線の傾きや切片などの計算結果を float8 として返します。

前提条件と不変条件

  • 係数 A, B, C は float8（倍精度浮動小数点数）として格納されます。
  • 浮動小数点演算の性質上、比較には geo_decls.h で定義された FPeq 等の許容誤差を伴う関数を使用する必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、src/backend/utils/adt/geo_ops.c における直線関連の幾何計算（交点計算、点と直線の距離、線分との交差判定など）の基盤として利用されます。
  • line_construct 関数により、点と傾きから係数が算出されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/utils/adt/geo_ops.c: line_decode, line_construct, line_interpt_line, line_contain_point, line_closept_point, lseg_interpt_line, lseg_closept_line 等の関数内で使用されています。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Point: 直線上の点や交点の計算に使用されます。
  • LSEG: 線分と直線の交差判定や距離計算において併用されます。

失敗モードと境界条件

  • **数値解析**: 本構造体を用いた計算は数値解析の専門的な手法に基づいているわけではないため、極端な値や計算誤差の蓄積により推移律が満たされない可能性があります。
  • **NaN**: 比較関数は NaN を考慮しておらず、NaN が含まれる場合は常に FALSE を返します。

保守担当者向けメモ

  • PATH や POLYGON と異なり、本構造体は固定長です。
  • src/backend/utils/adt/geo_ops.c 内の line_decode 等でメモリ確保（palloc）が行われるケースがあるため、ライフサイクル管理に注意してください。
