ProcArrayShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procarray.c:375

目的

PostgreSQLの共有メモリ初期化時に、ProcArray 構造体および関連するサブトランザクション管理用データ構造が必要とする合計バイト数を算出する。

シグネチャまたは形状

Size ProcArrayShmemSize(void)

入力と出力

  • 入力: なし（グローバル設定変数 MaxBackends, max_prepared_xacts, EnableHotStandby を参照）
  • 出力: 必要な共有メモリサイズ（Size 型）

前提条件と不変条件

  • PROCARRAY_MAXPROCS は MaxBackends + max_prepared_xacts として定義される。
  • TOTAL_MAX_CACHED_SUBXIDS は (PGPROC_MAX_CACHED_SUBXIDS + 1) * PROCARRAY_MAXPROCS として算出される。
  • EnableHotStandby が真の場合、サブトランザクション追跡用の領域が追加で確保される。

アルゴリズムまたは振る舞い

1. ProcArrayStruct 自体の基本サイズ（pgprocnos 配列のオフセットまで）を算出する。 2. PROCARRAY_MAXPROCS 分の int 配列サイズを加算する。 3. EnableHotStandby が有効な場合、TOTAL_MAX_CACHED_SUBXIDS 分の TransactionId 配列および bool 配列のサイズを合計に加算する。 4. 最終的な合計サイズを返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:131（共有メモリの総サイズ計算の一部として呼び出される）
  • 呼び出し先: add_size, mul_size（PostgreSQLのメモリサイズ計算用ユーティリティ）

関連データ構造

  • ProcArrayStruct: プロセス配列を管理するメイン構造体。
  • PGPROC: 各バックエンドプロセスの状態を保持する構造体。

失敗モードと境界条件

  • add_size および mul_size は、計算結果がオーバーフローする場合にエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数で算出されるサイズは、GetSnapshotData 等で利用されるローカルなデータ構造のサイズと一致させる必要がある。これは、共有メモリ上のデータをコピーして扱うケースがあるためである。
  • ホットスタンバイの有無に関わらず、共有メモリのセットアップ時点では EnableHotStandby の値に基づいて領域が確保される。
