SeqRecheck

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSeqscan.c:89

目的

EvalPlanQual（EPQ）実行時において、シーケンシャルスキャン中のタプルを再チェックするためのアクセスメソッドルーチンです。

シグネチャまたは形状

static bool SeqRecheck(SeqScanState *node, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • node: シーケンシャルスキャンの実行状態を保持する SeqScanState 構造体。
  • slot: 再チェック対象のタプルを格納する TupleTableSlot。
  • **出力**:
  • bool: 常に true を返します。

前提条件と不変条件

  • シーケンシャルスキャンは heap_beginscan においてキーを使用しないため、本関数内ではキーの整合性チェックを行いません。

アルゴリズムまたは振る舞い

  • 呼び出されると、追加の検証処理を行うことなく直ちに true を返します。これは、シーケンシャルスキャンがインデックススキャンとは異なり、スキャン開始時にフィルタ条件（キー）を適用してタプルを絞り込む仕組みを持たないことに起因します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSeqscan.c 内の各実行関数（ExecSeqScan 等）において、ExecScanRecheckMtd として参照されています。
  • **呼び出し先**: 本関数から直接呼び出している関数はありません。

関連データ構造

  • SeqScanState: シーケンシャルスキャンの実行状態を管理する構造体。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。常に成功（true）を返します。

保守担当者向けメモ

  • 本関数は IndexScan とは異なり、キーチェックを省略する設計となっています。将来的にシーケンシャルスキャンにキーベースの最適化が導入される場合、このロジックの再検討が必要になる可能性があります。
