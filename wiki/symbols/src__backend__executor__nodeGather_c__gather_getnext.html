gather_getnext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGather.c:262

目的

並列クエリ実行において、バックグラウンドワーカーからのタプル取得と、リーダープロセスによるローカル実行を統合し、単一のタプルストリームとして提供する。

シグネチャまたは形状

static TupleTableSlot *gather_getnext(GatherState *gatherstate)

入力と出力

  • **入力**: GatherState *gatherstate（Gatherノードの実行状態を保持する構造体）
  • **出力**: TupleTableSlot *（取得したタプルを格納したスロット。取得できない場合は ExecClearTuple により空のスロットが返される）

前提条件と不変条件

  • gatherstate は初期化済みであること。
  • gatherstate->nreaders がワーカーの読み取り状態を、gatherstate->need_to_scan_locally がリーダーによるローカル実行の必要性を正しく反映していること。

アルゴリズムまたは振る舞い

1. nreaders > 0 または need_to_scan_locally が真である限りループする。 2. 割り込みチェック (CHECK_FOR_INTERRUPTS) を実行する。 3. **ワーカーからの取得**: nreaders > 0 の場合、gather_readnext を呼び出す。タプルが取得できれば、fslot に格納して即座に返す。 4. **ローカル実行**: need_to_scan_locally が真の場合、estate->es_query_dsa に gatherstate->pei->area を一時的に設定し、ExecProcNode を呼び出して子プランをローカル実行する。タプルが取得できればそれを返す。取得できなければ need_to_scan_locally を false に更新する。 5. 全てのソースが枯渇した場合、空のスロットを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeGather.c 内の ExecGather から呼び出されることが確認されている。
  • **呼び出し先**:
  • gather_readnext（ワーカーからのタプル取得）
  • ExecStoreMinimalTuple（タプルの格納）
  • ExecProcNode（ローカル実行時）
  • ExecClearTuple（終了時）

関連データ構造

  • GatherState: 実行状態を管理する主要構造体。
  • TupleTableSlot: タプルを保持するスロット。
  • ParallelExecutorInfo (pei): 並列実行の共有メモリ領域情報。

失敗モードと境界条件

  • ワーカーがタプルを生成しない場合、リーダープロセスがローカル実行に切り替わる。
  • nreaders が 0 かつ need_to_scan_locally が偽の場合、ループを抜けて空のスロットを返す。

保守担当者向けメモ

  • gather_readnext は非ブロック読み取りを行うため、この関数はワーカーがタプルを生成するまで待機するのではなく、ローカル実行との間で適切に処理を切り替える役割を担う。
  • ローカル実行時に es_query_dsa を一時的に切り替える処理は、並列実行コンテキストの整合性を保つために不可欠である。
