fn

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/thread-alloc.c:119

目的

マルチスレッド環境におけるECPGのデータベース接続、SQL実行、および動的メモリ割り当ての安定性を検証するスレッドエントリポイント。

シグネチャまたは形状

static void* fn(void* arg)

入力と出力

  • **入力**: arg (スレッド識別子として使用される void* 型のポインタ)
  • **出力**: void* (常に 0 を返す)

前提条件と不変条件

  • arg は intptr_t にキャスト可能であること。
  • ECPGconnect が成功し、sqlca.sqlcode が正常であることを前提とする。
  • ループ内で ECPGdo により割り当てられたメモリ r は、各反復の終了時に free() で解放され、NULL にリセットされる必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: arg を整数値に変換し、接続名 name を生成する。 2. **接続**: ECPGconnect を呼び出し、データベース ecpg1_regression への接続を確立する。 3. **トランザクション設定**: ECPGsetcommit でコミットモードを on に設定する。 4. **ループ実行**: REPEATS 回数分、以下の処理を繰り返す。

  • ECPGdo を使用して pg_class からデータを取得し、ポインタ r に格納する。
  • 取得したメモリ領域 r を free() で解放し、r を NULL に初期化する。

    • 5. **終了処理**: ECPGdisconnect で接続を閉じ、0 を返して終了する。 6. **エラーハンドリング**: 各ECPG操作後に sqlca.sqlcode を確認し、エラー時は sqlprint() を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内で直接の呼び出し元は明示されていないが、main() 関数からスレッド生成関数（pthread_create 等）を通じて呼び出される設計となっている。
  • **呼び出し先**: ECPGconnect, ECPGsetcommit, ECPGdo, ECPGdisconnect, sqlprint, free, sprintf。

関連データ構造

  • sqlca: SQL通信領域（エラーハンドリングに使用）。
  • r: char ** 型の動的メモリ領域（SQL結果格納用）。

失敗モードと境界条件

  • ECPGconnect 失敗時: sqlca.sqlcode < 0 となり、sqlprint() が実行される。
  • ECPGdo 失敗時: ECPG_NOT_FOUND またはその他のエラーコードに応じて sqlprint() が実行される。
  • メモリ解放: r が NULL の状態で free() を呼び出す可能性があるが、標準Cライブラリの free() は NULL を許容するため安全である。

保守担当者向けメモ

  • 本関数はプリプロセッサの出力結果であるため、直接編集せず alloc.pgc を修正すること。
  • WindowsとPOSIXの両環境で動作するよう設計されているため、スレッド関連の変更を行う際は両方のプラットフォームへの影響を考慮すること。
  • r の解放漏れはメモリリークに直結するため、ループ内の free(r) と r = NULL のパターンは維持する必要がある。
