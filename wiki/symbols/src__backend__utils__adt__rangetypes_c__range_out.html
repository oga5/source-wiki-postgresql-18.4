range_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes.c:138

目的

PostgreSQL の範囲型（RangeType）を、SQL で表示可能なテキスト形式（例: [a, b)）に変換する出力関数です。

シグネチャまたは形状

Datum range_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される RangeType 構造体。
  • **出力**: PG_RETURN_CSTRING を介して、範囲を表す文字列（char *）を返します。

前提条件と不変条件

  • 入力された RangeType は、操作前にデトースト（detoast）されている必要があります。
  • check_stack_depth() が呼び出され、再帰的な型定義（範囲型の要素がさらに範囲型である場合など）によるスタックオーバーフローを防止します。

アルゴリズムまたは振る舞い

1. get_range_io_data を呼び出し、対象範囲型のメタデータ（要素型の出力関数など）を取得します。 2. range_deserialize を使用して、RangeType から境界値（lower, upper）と空フラグを抽出します。 3. range_get_flags で範囲のフラグを取得し、境界が存在する場合にのみ、要素型の出力関数（OutputFunctionCall）を呼び出して各境界値を文字列化します。 4. range_deparse を呼び出し、フラグと境界値の文字列を組み合わせて最終的な範囲表現文字列を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pseudotypes.c 内の関数から呼び出されることが確認されています。
  • **呼び出し先**: get_range_io_data, range_deserialize, range_get_flags, OutputFunctionCall, range_deparse。

関連データ構造

  • RangeType: 範囲型の内部表現。
  • RangeIOData: 範囲型の入出力に必要なキャッシュ情報。
  • RangeBound: 範囲の境界値を保持する構造体。

失敗モードと境界条件

  • スタック深度が制限を超えた場合、check_stack_depth() によりエラーが発生します。
  • 要素型の出力関数が失敗した場合、そのエラーが伝播します。

保守担当者向けメモ

  • 本関数は RangeType のシリアライズ形式に依存しています。内部構造の変更時には range_deserialize および range_deparse との整合性を確認してください。
  • 範囲型の要素型が再帰的である可能性があるため、スタック管理には注意が必要です。
