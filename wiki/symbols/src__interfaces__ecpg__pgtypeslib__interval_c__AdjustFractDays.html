AdjustFractDays

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/interval.c:40

目的

浮動小数点数で表現された端数（日数）を、struct tm の日フィールドと fsec_t（小数秒）へ正規化して加算する。

シグネチャまたは形状

static void AdjustFractDays(double frac, struct tm *tm, fsec_t *fsec, int scale)

入力と出力

  • **入力**:
  • frac: 加算する端数値（double）。
  • tm: 日付情報を保持する struct tm へのポインタ（更新対象）。
  • fsec: 小数秒を保持する fsec_t へのポインタ（更新対象）。
  • scale: frac を日数に換算するための倍率（例: DAYS_PER_MONTH や 7）。
  • **出力**: なし（引数の tm および fsec を直接更新）。

前提条件と不変条件

  • frac が 0 の場合、関数は即座にリターンする。
  • tm および fsec は有効なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. frac に scale を乗算し、総日数を算出する。 2. 整数部分（extra_days）を抽出し、tm->tm_mday に加算する。 3. 残りの端数（frac）を算出し、AdjustFractSeconds を呼び出して fsec および tm の秒単位の調整を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/pgtypeslib/interval.c 内の各所（162, 166, 208, 519 行目）。
  • **呼び出し先**: AdjustFractSeconds。

関連データ構造

  • struct tm: 標準的な日付・時刻構造体。
  • fsec_t: 小数秒を表現する型。

失敗モードと境界条件

  • frac が 0 の場合は何も行わない。
  • tm->tm_mday への加算により、月や年を跨ぐ繰り越しが発生する可能性がある（struct tm の仕様に依存）。

保守担当者向けメモ

  • 本関数は PostgreSQL バックエンドのロジックを ECPG 向けに移植したものである。
  • struct tm を使用しているため、バックエンドの struct pg_tm との差異に注意すること。
  • scale 引数には、文脈に応じて DAYS_PER_MONTH や 7（週）などが渡される。
