check_control_files

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:603

目的

複数のバックアップディレクトリに含まれる backup_label および control_file を検証し、バックアップチェーン全体でシステム識別子が一致していること、および制御ファイルのバージョンが適切であることを確認する。

シグネチャまたは形状

uint64 check_control_files(int n_backups, char **backup_dirs)

入力と出力

  • **入力**:
  • n_backups: 処理対象のバックアップディレクトリ数。
  • backup_dirs: 各バックアップディレクトリのパスを格納した文字列配列。
  • **出力**:
  • 成功時: チェーン全体で共通の system_identifier（uint64）。
  • 失敗時: pg_fatal によりプログラムを終了。

前提条件と不変条件

  • 各バックアップディレクトリ内に global/pg_control ファイルが存在すること。
  • すべての制御ファイルの pg_control_version が PG_CONTROL_VERSION と一致すること。
  • すべてのバックアップの system_identifier が一致すること。

アルゴリズムまたは振る舞い

1. 最新のバックアップから最古のバックアップへ向かって（n_backups - 1 から 0 まで）順次ループ処理を行う。 2. 各ディレクトリの global/pg_control を読み込み、CRCチェックを行う。CRCエラー時は致命的エラーとして終了する。 3. 制御ファイルのバージョンを検証し、不一致時は終了する。 4. 最初のループ（最新のバックアップ）で system_identifier を取得し、以降のバックアップで値が一致するかを比較する。不一致時は終了する。 5. 最新のバックアップの data_checksum_version を基準とし、それ以外のバックアップでチェックサム設定に不一致がある場合は警告フラグを立てる。 6. 最後に、チェックサム設定の不一致が検出された場合は警告メッセージを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_combinebackup/pg_combinebackup.c:278）。
  • **呼び出し先**: get_controlfile_by_exact_path（共通ユーティリティ）、psprintf、pg_log_debug、pg_fatal、pg_log_warning、pfree。

関連データ構造

  • ControlFileData: PostgreSQLの制御ファイル情報を保持する構造体。

失敗モードと境界条件

  • **CRCエラー**: 制御ファイルが破損している場合、pg_fatal で終了。
  • **バージョン不一致**: pg_control_version が現在のバージョンと異なる場合、pg_fatal で終了。
  • **識別子不一致**: 異なるデータベースクラスタのバックアップを混在させた場合、pg_fatal で終了。
  • **チェックサム不一致**: バックアップ間でチェックサム有効/無効の状態が混在している場合、警告を出力するが処理は継続する。

保守担当者向けメモ

  • 本関数は、バックアップチェーンの整合性を保証するための重要な検証ステップである。
  • data_checksum_mismatch の警告は、出力ディレクトリでのチェックサム再設定を推奨する内容となっている。
