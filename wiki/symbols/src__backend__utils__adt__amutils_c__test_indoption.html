test_indoption

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/amutils.c:116

目的

pg_index.indoption 配列内の特定のビットフラグを検証し、インデックス列のプロパティ（ソート順や NULL の扱いなど）を判定するヘルパー関数です。

シグネチャまたは形状

static bool test_indoption(HeapTuple tuple, int attno, bool guard, int16 iopt_mask, int16 iopt_expect, bool *res)

入力と出力

  • **入力**:
  • tuple: pg_index システムカタログのタプル。
  • attno: 検証対象の列番号（1 始まり）。
  • guard: 判定を行うための前提条件（アクセスメソッドの機能フラグ等）。
  • iopt_mask: indoption 値から抽出するビットマスク。
  • iopt_expect: iopt_mask を適用した結果期待される値。
  • **出力**:
  • res: 判定結果（bool へのポインタ）。
  • 戻り値: 常に true を返します。

前提条件と不変条件

  • guard が false の場合、res に false を設定して即座に終了します。
  • guard が true の場合、pg_index タプルから indoption 属性を読み込み、指定された列番号に対応する値を評価します。

アルゴリズムまたは振る舞い

1. guard が false ならば、*res = false として終了します。 2. SysCacheGetAttrNotNull を使用して、pg_index タプルから indoption（int2vector）を取得します。 3. attno - 1 をインデックスとして indoption 配列から値を取得します。 4. 取得した値に対して iopt_mask を適用し、iopt_expect と一致するかを判定して *res に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/amutils.c 内の indexam_property 関数から複数回呼び出されています（行 249, 256, 263, 270）。
  • **呼び出し先**: SysCacheGetAttrNotNull（システムキャッシュからの属性取得）。

関連データ構造

  • HeapTuple: システムカタログのタプル表現。
  • int2vector: pg_index.indoption を保持する可変長配列構造体。

失敗モードと境界条件

  • attno が無効な場合、int2vector へのアクセスでメモリ境界外参照が発生する可能性があります。
  • SysCacheGetAttrNotNull が INDEXRELID に対して失敗した場合、エラーが発生します。

保守担当者向けメモ

  • 本関数は pg_index の indoption ビットを直接操作するため、pg_index.h で定義されている INDOPTION_DESC などの定数と組み合わせて使用されます。
  • 呼び出し元で guard 引数にアクセスメソッドの能力（amcanorder 等）を渡すことで、カタログ値の評価を制御する設計になっています。
