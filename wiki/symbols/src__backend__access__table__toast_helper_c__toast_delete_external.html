toast_delete_external

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/toast_helper.c:317

目的

タプルに含まれる外部ストレージ上のTOASTデータを物理的に削除する。

シグネチャまたは形状

void toast_delete_external(Relation rel, const Datum *values, const bool *isnull, bool is_speculative)

入力と出力

  • rel: 対象リレーション。
  • values: タプルの属性値配列。
  • isnull: 各属性がNULLかどうかを示すフラグ配列。
  • is_speculative: 推測的挿入（speculative insertion）中かどうかを示すフラグ。
  • 戻り値: なし。

前提条件と不変条件

  • rel は有効なリレーション記述子であること。
  • values および isnull 配列は rel の属性数と一致していること。
  • 属性長が -1（可変長データ）である属性のみが処理対象となる。

アルゴリズムまたは振る舞い

1. リレーションの属性定義（TupleDesc）を走査する。 2. 各属性について、以下の条件を満たす場合に toast_delete_datum を呼び出す。

  • 属性がNULLではない。
  • 属性長が -1 である。
  • VARATT_IS_EXTERNAL_ONDISK マクロにより、データが外部ストレージ上に存在すると判定される。

    • 3. 外部データが存在する場合、toast_delete_datum(rel, value, is_speculative) を実行して物理削除を行う。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heaptoast.c:73 にて確認。
  • 呼び出し先: toast_delete_datum（access/toast_internals.h で定義される低レベル操作関数）。

関連データ構造

  • Relation: リレーションのメタデータ構造体。
  • TupleDesc: タプルの属性定義。
  • Datum: 属性値の表現。

失敗モードと境界条件

  • isnull 配列の要素が true の場合はスキップされる。
  • 外部ストレージ上にデータが存在しない（インライン化されている）場合は削除処理は行われない。

保守担当者向けメモ

  • 本関数は物理的な削除ロジックを toast_delete_datum に委譲している。
  • is_speculative フラグは、削除対象のデータが推測的挿入に関連しているかどうかを制御するために使用される。
