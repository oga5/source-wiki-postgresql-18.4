get_arg_by_name

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/stat_utils.c:250

目的

統計情報の更新処理において、指定された引数名に対応するインデックス番号を StatsArgInfo 配列から検索します。

シグネチャまたは形状

static int get_arg_by_name(const char *argname, struct StatsArgInfo *arginfo);

入力と出力

  • **入力**:
  • argname: 検索対象の引数名（文字列）。
  • arginfo: 検索対象となる StatsArgInfo 構造体の配列。argname フィールドが NULL である要素を終端とみなします。
  • **出力**:
  • 成功時: 見つかった引数のインデックス番号（0以上の整数）。
  • 失敗時: -1。

前提条件と不変条件

  • arginfo 配列は、argname が NULL である要素で適切に終端されている必要があります。
  • 比較は pg_strcasecmp を使用するため、大文字と小文字を区別しない名前照合が行われます。

アルゴリズムまたは振る舞い

1. arginfo 配列を先頭から順に走査します。 2. 各要素の argname が NULL に達するまで、pg_strcasecmp を用いて入力された argname と比較します。 3. 一致する名前が見つかった場合、そのインデックス番号を即座に返します。 4. 配列の終端まで一致するものが見つからなかった場合、WARNING レベルの ereport を発行し、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/stat_utils.c:353 (stats_fill_fcinfo_from_arg_pairs 内で呼び出されます)
  • **呼び出し先**: pg_strcasecmp (標準的な文字列比較関数), ereport (エラー報告用マクロ)

関連データ構造

  • StatsArgInfo: 引数名と関連情報を保持する構造体（本関数が走査対象とする配列の型）。

失敗モードと境界条件

  • **未定義の引数名**: 渡された argname が arginfo 配列内に存在しない場合、WARNING がログ出力され、-1 が返されます。
  • **空の配列**: arginfo の先頭要素が直ちに NULL である場合、ループは実行されず、常に -1 が返されます。

保守担当者向けメモ

  • 本関数は static であり、stat_utils.c 内の引数解析処理専用です。
  • pg_strcasecmp を使用しているため、SQLレベルでの引数名指定はケースインセンシティブです。将来的に引数名の厳密な照合が必要になった場合は、比較ロジックの変更を検討してください。
