PQsslInUse

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure.c:102

目的

現在の接続においてSSL/TLS暗号化が有効であるかどうかを判定します。

シグネチャまたは形状

int PQsslInUse(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn （接続状態を保持する構造体へのポインタ）
  • **出力**: int 型。SSLが使用中の場合は非ゼロ（真）、それ以外または接続がNULLの場合は0（偽）を返します。

前提条件と不変条件

  • conn が NULL の場合、関数は安全に 0 を返します。
  • 戻り値は conn->ssl_in_use の値を反映します。

アルゴリズムまたは振る舞い

1. 引数 conn が NULL かどうかを確認し、NULL であれば即座に 0 を返します。 2. conn が有効な場合、内部構造体メンバ ssl_in_use の値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c:821
  • src/bin/psql/command.c:4507
  • **呼び出し先**:
  • 本関数は他の関数を呼び出しません。

関連データ構造

  • PGconn: libpq の接続状態を管理する主要な構造体。本関数はこの構造体の ssl_in_use フィールドを参照します。

失敗モードと境界条件

  • conn が NULL の場合、エラーにはならず 0 を返します。
  • 接続確立前の PGconn オブジェクトに対して呼び出した場合、初期化状態に応じた値（通常は 0）を返します。

保守担当者向けメモ

  • 本関数は libpq-fe.h で公開されているパブリックAPIです。
  • ssl_in_use フラグは接続確立プロセス（fe-connect.c 等）で設定されます。暗号化プロトコルの追加や変更を行う際は、このフラグの更新ロジックと整合性を保つ必要があります。
