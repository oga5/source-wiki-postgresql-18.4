SocketBackend

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/postgres.c:352

目的

フロントエンド（クライアント）から送信されたメッセージタイプを識別し、その内容を読み取って後続の処理へ渡すためのインターフェースを提供します。

シグネチャまたは形状

static int SocketBackend(StringInfo inBuf)

入力と出力

  • **入力**: inBuf (StringInfo) - 受信したメッセージ本体を格納するためのバッファ。
  • **出力**: int - 受信したメッセージタイプコード（PqMsg_* 定数）、または接続終了時の EOF。

前提条件と不変条件

  • 呼び出し時に HOLD_CANCEL_INTERRUPTS() が実行され、メッセージ読み取り中の割り込みを抑制します。
  • 正常終了時には RESUME_CANCEL_INTERRUPTS() が呼ばれ、割り込み許可状態が復元されます。
  • プロトコルバージョン3を前提としており、すべてのメッセージにはタイプコードの直後に長さ情報が含まれている必要があります。

アルゴリズムまたは振る舞い

1. pq_startmsgread() を呼び出し、フロントエンドからのメッセージ読み取りを開始します。 2. pq_getbyte() でメッセージタイプコードを取得します。 3. EOF の場合、トランザクション状態を確認し、必要に応じてエラー報告を行い終了します。 4. メッセージタイプに基づき、maxmsglen（許容最大長）を設定し、doing_extended_query_message フラグを更新します。

  • PqMsg_Query 等は PQ_LARGE_MESSAGE_LIMIT、PqMsg_Sync 等は PQ_SMALL_MESSAGE_LIMIT を適用します。
  • 不明なタイプコードの場合は FATAL エラーを発生させます。

    • 5. pq_getmessage() を呼び出し、inBuf にメッセージ本体を読み込みます。 6. 成功時にメッセージタイプコードを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/postgres.c 内のメインループ（行486付近）。
  • **呼び出し先**:
  • pq_startmsgread, pq_getbyte, pq_getmessage（通信プロトコル層）
  • IsTransactionState（トランザクション管理）
  • ereport（エラー報告）

関連データ構造

  • StringInfo: 通信バッファを管理する構造体。
  • PqMsg_*: フロントエンドからのメッセージタイプを定義する定数群。

失敗モードと境界条件

  • **EOF**: クライアントの切断。トランザクションが開いている場合は COMMERROR を報告します。
  • **プロトコル違反**: 未定義のメッセージタイプを受信した場合、FATAL エラーとしてプロセスを終了します。
  • **メッセージ長超過**: pq_getmessage が失敗した場合、EOF を返します。

保守担当者向けメモ

  • 本関数は postgres.c のメインループの一部であり、プロトコル同期の維持に責任を持ちます。
  • doing_extended_query_message フラグの管理は、エラー発生時のスキップ処理（ignore_till_sync）と密接に関連しています。
  • メッセージタイプごとの maxmsglen の設定は、メモリ枯渇を防ぐための重要な防御的実装です。
