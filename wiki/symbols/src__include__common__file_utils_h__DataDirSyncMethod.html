DataDirSyncMethod

定義

  • 種別: enum
  • 定義箇所: src/include/common/file_utils.h:27

目的

データディレクトリの同期処理において、ファイルシステムレベルの整合性を確保するための同期戦略を選択するために使用されます。

シグネチャまたは形状

typedef enum DataDirSyncMethod
{
	DATA_DIR_SYNC_METHOD_FSYNC,
	DATA_DIR_SYNC_METHOD_SYNCFS,
} DataDirSyncMethod;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力を持つ関数ではありません。
  • 同期処理を行う関数（sync_pgdata 等）において、同期メソッドを指定する引数として利用されます。

前提条件と不変条件

  • DATA_DIR_SYNC_METHOD_SYNCFS を使用する場合、ビルド環境が syncfs システムコールをサポートしている必要があります（HAVE_SYNCFS マクロ等に依存）。

アルゴリズムまたは振る舞い

  • DATA_DIR_SYNC_METHOD_FSYNC: ディレクトリツリーを走査し、各ファイルに対して個別に fsync を発行します。
  • DATA_DIR_SYNC_METHOD_SYNCFS: ファイルシステム全体に対して syncfs を発行し、同期を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のツールにおいて、同期メソッドを保持する変数として参照されています。
  • initdb (src/bin/initdb/initdb.c)
  • pg_basebackup (src/bin/pg_basebackup/pg_basebackup.c)
  • pg_checksums (src/bin/pg_checksums/pg_checksums.c)
  • pg_combinebackup (src/bin/pg_combinebackup/pg_combinebackup.c)
  • pg_dump (src/bin/pg_dump/pg_dump.c, pg_backup_archiver.c)
  • pg_rewind (src/bin/pg_rewind/pg_rewind.c, pg_rewind.h)
  • **呼び出し先**: 情報なし。

関連データ構造

  • sync_pgdata(): 本列挙型を引数として受け取り、実際の同期処理を実行する関数。

失敗モードと境界条件

  • プラットフォームが syncfs をサポートしていない環境で DATA_DIR_SYNC_METHOD_SYNCFS を選択した場合の動作は、ビルド時の機能フラグ設定に依存します。

保守担当者向けメモ

  • 本列挙型は FRONTEND ツールでの利用が主です。サーバー側コードとの共有状況については、src/include/common/file_utils.h の FRONTEND マクロ定義を確認してください。
