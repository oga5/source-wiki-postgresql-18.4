FreePageManagerUpdateLargest

定義

  • 種別: static void 関数
  • 定義箇所: src/backend/utils/mmgr/freepage.c:365

目的

FreePageManager が保持する最大連続空きページ数（contiguous_pages）のキャッシュを最新の状態に更新します。

シグネチャまたは形状

static void FreePageManagerUpdateLargest(FreePageManager *fpm)

入力と出力

  • **入力**: FreePageManager *fpm（メモリ管理構造体へのポインタ）
  • **出力**: なし（副作用として fpm 構造体のメンバを更新）

前提条件と不変条件

  • fpm は有効な FreePageManager 構造体を指している必要があります。
  • fpm->contiguous_pages_dirty フラグが true の場合のみ、再計算が行われます。

アルゴリズムまたは振る舞い

1. fpm->contiguous_pages_dirty を確認し、false であれば何もせず終了します。 2. FreePageManagerLargestContiguous(fpm) を呼び出し、現在の最大連続空きページ数を再計算します。 3. 計算結果を fpm->contiguous_pages に格納します。 4. fpm->contiguous_pages_dirty を false に設定し、キャッシュが最新であることを示します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/freepage.c 内の複数箇所（FreePageManagerGet や FreePageManagerPut 等の内部処理から呼び出されています）。
  • **呼び出し先**:
  • FreePageManagerLargestContiguous

関連データ構造

  • FreePageManager: メモリ管理のメタデータを保持するメイン構造体。contiguous_pages（最大連続ページ数）および contiguous_pages_dirty（更新フラグ）をメンバに持ちます。

失敗モードと境界条件

  • fpm が NULL の場合、未定義の動作となります。
  • contiguous_pages_dirty が false の場合は計算をスキップするため、計算コストを抑制する設計となっています。

保守担当者向けメモ

  • 本関数は、メモリの確保や解放によって空き領域の構成が変化した際に、キャッシュの整合性を保つために呼び出されます。
  • FreePageManagerLargestContiguous は計算コストが高いため、必ず dirty フラグによるガードを確認してから呼び出すようにしてください。
