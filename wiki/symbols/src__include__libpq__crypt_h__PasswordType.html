PasswordType

定義

  • 種別: enum
  • 定義箇所: src/include/libpq/crypt.h:40

目的

PostgreSQLの認証プロセスにおいて、保存されたパスワード文字列のハッシュ形式を識別・分類するために使用されます。

シグネチャまたは形状

typedef enum PasswordType
{
	PASSWORD_TYPE_PLAINTEXT = 0,
	PASSWORD_TYPE_MD5,
	PASSWORD_TYPE_SCRAM_SHA_256,
} PasswordType;

入力と出力

  • 本シンボル自体は列挙型であり、入力や出力は持ちません。
  • get_password_type() 関数の戻り値として使用され、文字列から形式を判定する際の出力として機能します。
  • encrypt_password() 関数の引数として使用され、ターゲットとなるハッシュ形式を指定する入力として機能します。

前提条件と不変条件

  • PASSWORD_TYPE_PLAINTEXT は値 0 として定義されています。
  • 認証システムにおいて、パスワード文字列がどの形式に該当するかを判定する際の基準となります。

アルゴリズムまたは振る舞い

  • 認証フローにおいて、pg_authid から取得したパスワード文字列を解析し、本列挙型のいずれかの値にマッピングすることで、後続の検証処理（md5_crypt_verify 等）を決定します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/passwordcheck/passwordcheck.c
  • src/backend/libpq/auth.c
  • src/backend/libpq/crypt.c
  • src/include/commands/user.h
  • **呼び出し先**:
  • 本シンボルは列挙型であるため、直接的な呼び出し先はありません。関連する関数として get_password_type() および encrypt_password() が本型を利用します。

関連データ構造

  • pg_authid: パスワード情報が格納されるシステムカタログ。
  • Password_encryption: src/include/commands/user.h で定義される、本列挙型の値をとる設定変数。

失敗モードと境界条件

  • 判定対象の文字列が既知の形式（MD5, SCRAM-SHA-256, 平文）のいずれにも適合しない場合、get_password_type() がどのような値を返すかについては、本ソース excerpt には記載がありません。

保守担当者向けメモ

  • 新しい認証アルゴリズムを導入する際は、本列挙型に新しいメンバを追加し、get_password_type() および encrypt_password() の実装を更新する必要があります。
