int8out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int8.c:60

目的

PostgreSQLの64ビット整数型（int8 / bigint）の値を、クライアントへ送信するための文字列形式（C-string）に変換します。

シグネチャまたは形状

Datum int8out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される int64 型の値。
  • **出力**: Datum 型として返される、変換後の文字列（char *）。

前提条件と不変条件

  • 入力値は PG_GETARG_INT64(0) により取得されます。
  • 出力文字列は palloc を使用してメモリ確保されるため、呼び出し元で適切に管理される必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT64(0) で入力値を取得します。 2. pg_lltoa を使用して、64ビット整数を文字列バッファ buf に変換します。 3. palloc を用いて、文字列の長さに応じたメモリ領域を確保します。 4. memcpy を使用して、バッファの内容を確保したメモリ領域にコピーします。 5. PG_RETURN_CSTRING を介して、結果の文字列を返します。

  • pstrdup を使用せず、長さを事前に計算して memcpy を行うことで、冗長な strlen 呼び出しを回避しています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/formatting.c:6684（DirectFunctionCall1 を介した呼び出し）
  • **呼び出し先**:
  • pg_lltoa（整数から文字列への変換）
  • palloc（メモリ確保）
  • memcpy（メモリコピー）

関連データ構造

  • MAXINT8LEN: 64ビット整数を文字列化した際の最大長を定義する定数。

失敗モードと境界条件

  • palloc がメモリ不足となった場合、PostgreSQLの標準的なエラーハンドリングによりアボートされます。
  • 64ビット整数の全範囲（INT64_MIN から INT64_MAX）を正しく処理します。

保守担当者向けメモ

  • 本関数は int8 型の出力処理を担います。バイナリ形式での送信が必要な場合は int8send が別途使用されます。
  • pg_lltoa の戻り値を利用してメモリ確保量を最適化しているため、このロジックを変更する際はバッファサイズと len の整合性に注意してください。
