EnumValuesDelete

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_enum.c:223

目的

指定された列挙型（OID）に関連付けられたすべての列挙値（ラベル）をシステムカタログ pg_enum から削除します。

シグネチャまたは形状

void EnumValuesDelete(Oid enumTypeOid)

入力と出力

  • **入力**: enumTypeOid (削除対象の列挙型の OID)
  • **出力**: なし（副作用としてカタログを更新）

前提条件と不変条件

  • 呼び出し元は、対象の列挙型に対して適切なロックを保持している必要があります。
  • pg_enum カタログに対する RowExclusiveLock を取得して操作を行います。

アルゴリズムまたは振る舞い

1. table_open を使用して pg_enum を RowExclusiveLock モードで開きます。 2. ScanKeyInit を用いて、指定された enumTypeOid を検索キーとして設定します。 3. systable_beginscan を呼び出し、EnumTypIdLabelIndexId インデックスを使用してスキャンを開始します。 4. systable_getnext で該当するタプルを順次取得し、CatalogTupleDelete を用いて削除します。 5. systable_endscan でスキャンを終了し、table_close でカタログを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/typecmds.c:676 からの呼び出しが確認されています。
  • **呼び出し先**: table_open, ScanKeyInit, systable_beginscan, systable_getnext, CatalogTupleDelete, systable_endscan, table_close を呼び出します。

関連データ構造

  • pg_enum: 列挙型の定義を保持するシステムカタログ。
  • EnumTypIdLabelIndexId: pg_enum のインデックス。

失敗モードと境界条件

  • 指定された enumTypeOid に該当する値が存在しない場合、ループは実行されず何も削除されません。
  • カタログ操作中にエラーが発生した場合、PostgreSQL のトランザクションメカニズムによりアボートされます。

保守担当者向けメモ

  • 本関数は pg_enum のタプルを直接削除するため、依存関係の整合性については呼び出し元で担保する必要があります。
  • RowExclusiveLock を使用しているため、同時実行される他の DDL 操作との競合に注意してください。
