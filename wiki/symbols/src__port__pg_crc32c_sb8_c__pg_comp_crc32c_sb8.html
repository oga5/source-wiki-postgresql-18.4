pg_comp_crc32c_sb8

定義

  • 種別: function
  • 定義箇所: src/port/pg_crc32c_sb8.c:34

目的

ハードウェアアクセラレーションが利用できない環境において、Slicing-by-8アルゴリズムを用いて高速にCRC-32C（Castagnoli多項式: 0x1EDC6F41）を計算する。

シグネチャまたは形状

pg_crc32c pg_comp_crc32c_sb8(pg_crc32c crc, const void *data, size_t len)

入力と出力

  • **入力**:
  • crc: 初期CRC値。
  • data: 計算対象のデータポインタ。
  • len: データのバイト長。
  • **出力**:
  • 計算結果である pg_crc32c 型のCRC値。

前提条件と不変条件

  • 入力データは任意のバイト境界から開始可能。
  • 内部処理において、4バイト境界にアライメントされたデータアクセスを最適化の前提とする。
  • ビッグエンディアン環境では、テーブル内の値がバイト反転して格納されている。

アルゴリズムまたは振る舞い

1. **初期処理**: 入力ポインタが4バイト境界に揃うまで、CRC8 マクロを用いて1バイトずつ計算する。 2. **メインループ**: 8バイト単位でデータを読み込み、pg_crc32c_table を参照してXOR演算を行う。エンディアンに応じてバイト順序を調整し、8つのルックアップテーブルを組み合わせてCRCを更新する。 3. **終了処理**: 残余データ（0〜7バイト）に対して、再度 CRC8 マクロを用いて1バイトずつ計算し、最終的な値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/port/pg_crc32c_armv8_choose.c
  • src/port/pg_crc32c_sse42_choose.c
  • src/include/port/pg_crc32c.h（マクロ経由の利用を含む）
  • **呼び出し先**:
  • CRC8 マクロ（補助的な1バイト計算）
  • pg_crc32c_table（ルックアップテーブル）

関連データ構造

  • pg_crc32c_table: Slicing-by-8アルゴリズムで使用される8つのルックアップテーブル。

失敗モードと境界条件

  • len が0の場合、初期値 crc をそのまま返す。
  • データの長さが8バイト未満の場合、メインループはスキップされ、終了処理の CRC8 ループのみが実行される。

保守担当者向けメモ

  • 本実装はMichael E. Kounavis氏とFrank L. Berry氏の論文に基づく。
  • pg_crc32c_table は静的データ構造として巨大であるため、メモリ制約が厳しい環境ではバイナリサイズに注意すること。
  • ビッグエンディアン環境におけるバイト反転処理は、計算効率を維持するための設計である。
