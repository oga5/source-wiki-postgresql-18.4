PVIndStats

定義

  • **種別**: struct
  • **定義箇所**: src/backend/commands/vacuumparallel.c:137

目的

並列 VACUUM 処理において、各インデックスの処理状態および実行結果（統計情報）を保持・共有するための構造体です。

シグネチャまたは形状

typedef struct PVIndStats
{
    PVIndVacStatus status;
    bool           parallel_workers_can_process;
    bool           istat_updated;
    IndexBulkDeleteResult istat;
} PVIndStats;

入力と出力

  • **入力**: リーダープロセスが並列処理開始前に status および parallel_workers_can_process を設定します。
  • **出力**: 各ワーカーまたはリーダーがインデックス処理後に istat（IndexBulkDeleteResult）を更新し、istat_updated を true に設定します。

前提条件と不変条件

  • status および parallel_workers_can_process は、個別の並列インデックス VACUUM またはクリーンアップ処理の開始前にリーダーによって設定されます。
  • VACUUM 操作全体を通じて固定される値ではありません。

アルゴリズムまたは振る舞い

  • 本構造体は共有メモリ（DSM）上に配置され、リーダーとワーカー間でインデックスごとの進捗と結果を同期するために使用されます。
  • parallel_workers_can_process が true の場合、リーダーとワーカーの両方がそのインデックスを処理可能です。false の場合はリーダーのみが処理を行います。
  • 処理完了後、istat_updated フラグにより統計情報が回収可能であることを示します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/vacuumparallel.c 内の各関数（parallel_vacuum_process_all_indexes 等）が、共有メモリ上の PVIndStats 配列を参照・更新します。
  • **呼び出し先**: 具体的な呼び出し先（関数）の詳細は提供されたソース抜粋には含まれていません。

関連データ構造

  • PVIndVacStatus: インデックスの VACUUM 状態を示す列挙型。
  • IndexBulkDeleteResult: インデックスのバルク削除結果を保持する構造体。
  • ParallelVacuumState: 本構造体を配列として保持するメインの並列 VACUUM 状態構造体。

失敗モードと境界条件

  • 共有メモリの割り当てサイズ計算時（mul_size(sizeof(PVIndStats), nindexes)）に失敗する可能性があります。
  • 複数のワーカーが同時に同一インデックスを処理しようとした場合、PVIndVacStatus による排他制御が正しく機能しないと競合が発生します。

保守担当者向けメモ

  • 本構造体は DSM 上に配置されるため、ポインタメンバを含めることはできません。
  • istat フィールドは IndexBulkDeleteResult 型であり、そのサイズや内容が将来変更された場合、共有メモリレイアウトに影響を与える可能性があります。
