sepgsql_compute_create

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/selinux.c:841

目的

新規オブジェクト作成時に、SELinux ポリシーに基づいて適用すべきデフォルトのセキュリティコンテキストを計算する。

シグネチャまたは形状

char *sepgsql_compute_create(const char *scontext, const char *tcontext, uint16 tclass, const char *objname);

入力と出力

  • **入力**:
  • scontext: 主体（subject）のセキュリティコンテキスト。
  • tcontext: 対象（target）のセキュリティコンテキスト。
  • tclass: PostgreSQL 内部のオブジェクトクラスコード。
  • objname: 新規オブジェクト名（オプション）。
  • **出力**:
  • 計算されたセキュリティコンテキスト文字列（palloc 領域にコピー済み）。

前提条件と不変条件

  • tclass は SEPG_CLASS_MAX 未満であること（Assert による検証）。
  • libselinux が提供する malloc 領域を freecon で解放し、PostgreSQL 管理下のメモリへ複製する。

アルゴリズムまたは振る舞い

1. selinux_catalog を参照し、内部の tclass を SELinux の security_class_t 型へ変換する。 2. security_compute_create_name_raw を呼び出し、カーネルからデフォルトコンテキストを取得する。 3. 取得に失敗した場合は ERROR を発生させる。 4. PG_TRY/PG_FINALLY ブロック内で、libselinux が確保したメモリを pstrdup で PostgreSQL のメモリコンテキストへ複製し、直後に freecon で元のメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/database.c
  • contrib/sepgsql/proc.c
  • contrib/sepgsql/relation.c
  • contrib/sepgsql/schema.c
  • contrib/sepgsql/uavc.c
  • **呼び出し先**:
  • string_to_security_class
  • security_compute_create_name_raw
  • pstrdup
  • freecon

関連データ構造

  • selinux_catalog: オブジェクトクラスと SELinux クラス名のマッピングテーブル。

失敗モードと境界条件

  • security_compute_create_name_raw が負の値を返した場合、ereport(ERROR, ...) によりトランザクションが中断される。

保守担当者向けメモ

  • 本関数は libselinux のメモリ管理と PostgreSQL のメモリ管理を橋渡しするため、PG_TRY ブロックによる例外安全性が必須である。
  • tclass のマッピングは selinux_catalog に依存するため、ポリシー定義の変更時にはこのカタログの整合性に注意すること。
