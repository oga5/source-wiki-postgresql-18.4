do_lo_unlink

定義

  • 種別: function
  • 定義箇所: src/bin/psql/large_obj.c:238

目的

指定された OID を持つラージオブジェクトをデータベースから削除します。

シグネチャまたは形状

bool do_lo_unlink(const char *loid_arg);

入力と出力

  • **入力**: loid_arg (削除対象ラージオブジェクトの OID を表す文字列)
  • **出力**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • ラージオブジェクト操作はトランザクション内で行う必要があります。本関数は start_lo_xact を呼び出し、必要に応じてトランザクションを開始します。
  • pset.db が有効なデータベース接続を保持している必要があります。

アルゴリズムまたは振る舞い

1. atooid を使用して入力文字列を Oid 型に変換します。 2. start_lo_xact を呼び出し、トランザクション状態を確保します。 3. SetCancelConn(NULL) により割り込みを許可し、libpq の lo_unlink を実行します。 4. ResetCancelConn() で割り込み設定を戻します。 5. lo_unlink が失敗（戻り値が -1）した場合、PQerrorMessage をログ出力し、fail_lo_xact を経由してロールバック処理を行い false を返します。 6. 成功した場合、finish_lo_xact でトランザクションを終了（コミット）し、print_lo_result で結果を報告して true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:2437 (メタコマンド処理)
  • **呼び出し先**:
  • start_lo_xact, finish_lo_xact, fail_lo_xact (トランザクション管理)
  • SetCancelConn, ResetCancelConn (割り込み制御)
  • lo_unlink (libpq API)
  • print_lo_result (結果出力)
  • atooid (型変換)
  • pg_log_info, PQerrorMessage (エラーログ出力)

関連データ構造

  • pset: データベース接続情報および出力設定を保持するグローバル構造体。

失敗モードと境界条件

  • start_lo_xact が失敗した場合、即座に false を返します。
  • lo_unlink が -1 を返した場合、エラーメッセージをログ出力し、トランザクションをロールバックして false を返します。
  • finish_lo_xact が失敗した場合、false を返します。

保守担当者向けメモ

  • 本関数は psql の \lo_unlink メタコマンドの実装です。
  • トランザクションの自動管理（autocommit 考慮）は start_lo_xact および finish_lo_xact に依存しています。
