charin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/char.c:40

目的

文字列（cstring）をPostgreSQLの内部型 "char"（1バイト文字型）に変換します。

シグネチャまたは形状

Datum charin(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る cstring 型の文字列。
  • **出力**: Datum 型として返される1バイトの文字値。

前提条件と不変条件

  • 入力文字列が \ooo（o は8進数）の形式である場合、それはエスケープされた1バイト文字として解釈されます。
  • それ以外の入力は、文字列の先頭バイトがそのまま "char" 型の値として採用されます。
  • 長さ0の入力文字列に対しては、適切に動作するよう設計されています。

アルゴリズムまたは振る舞い

1. 入力文字列の長さを確認します。 2. 文字列が \ で始まり、続く3文字がすべて8進数（ISOCTAL）である場合、その3文字を8進数としてデコードし、得られた値を返します。 3. 上記の条件に合致しない場合、文字列の先頭バイト（ch[0]）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/char.c:211 にて、変換ルールの参照先として言及されています。
  • **呼び出し先**:
  • PG_GETARG_CSTRING マクロ（入力取得）
  • strlen（標準Cライブラリ）
  • ISOCTAL / FROMOCTAL マクロ（文字判定および変換）
  • PG_RETURN_CHAR マクロ（結果返却）

関連データ構造

  • Datum: PostgreSQLの汎用データ表現型。
  • "char": 本関数が生成する1バイトの内部データ型。

失敗モードと境界条件

  • 入力文字列が空の場合、ch[0] へのアクセスが発生しますが、ソースコードのコメントによればゼロ長入力に対しても正しく動作するよう考慮されています。
  • 8進数エスケープ形式の判定は、正確に4文字であることと、各文字が8進数であることを厳密にチェックします。

保守担当者向けメモ

  • 本関数はSQL標準の CHAR(n) 型ではなく、PostgreSQL固有の1バイト型 "char" を扱います。
  • 比較演算は uint8（符号なし）で行われますが、型変換ロジックとの間で符号の扱いに歴史的な不整合があるため、変更を加える際は注意してください。
