TidRangeEval

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidrangescan.c:138

目的

TID範囲スキャンにおいて、クエリの qual（比較条件）を評価し、スキャン対象となる ctid の下限値（trss_mintid）と上限値（trss_maxtid）を算出する。

シグネチャまたは形状

static bool TidRangeEval(TidRangeScanState *node)

入力と出力

  • **入力**: TidRangeScanState *node（スキャン状態を保持するノード構造体）
  • **出力**: bool（範囲が有効であれば true、評価の結果スキャン対象が空（NULL）と判定された場合は false）

前提条件と不変条件

  • node->trss_tidexprs には、ctid に対する比較演算式が格納されている必要がある。
  • 評価の結果、下限値が上限値を超えた場合や、評価式が NULL を返す場合はスキャン対象なしとみなされる。

アルゴリズムまたは振る舞い

1. **初期化**: 下限値を最小値（0, 0）、上限値を最大値（InvalidBlockNumber, PG_UINT16_MAX）に設定する。 2. **式評価**: node->trss_tidexprs リストを走査し、各 TidOpExpr を ExecEvalExprSwitchContext で評価する。 3. **範囲の絞り込み**:

  • TIDEXPR_LOWER_BOUND: 評価結果が現在の lowerBound より大きければ更新する。非包含（inclusive = false）の場合は ItemPointerInc で値を補正する。
  • TIDEXPR_UPPER_BOUND: 評価結果が現在の upperBound より小さければ更新する。非包含の場合は ItemPointerDec で値を補正する。

    • 4. **結果の格納**: 最終的な lowerBound と upperBound を node->trss_mintid および node->trss_maxtid にコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeTidrangescan.c 内の TidRangeNext（スキャン実行時）から呼び出される。
  • **呼び出し先**: ExecEvalExprSwitchContext（式評価）、ItemPointerCompare、ItemPointerInc、ItemPointerDec 等のユーティリティ関数。

関連データ構造

  • TidRangeScanState: スキャンノードの状態を保持。
  • TidOpExpr: ctid の比較条件と包含フラグを保持する構造体。
  • ItemPointerData: ctid（物理タプル位置）を表現するデータ構造。

失敗モードと境界条件

  • **NULL 評価**: 比較式が NULL を返した場合、即座に false を返しスキャンを終了する。
  • **非包含範囲**: inclusive = false の場合、ItemPointerInc/Dec による正規化が行われる。この際、結果が有効な ItemPointer 範囲外になる可能性がある。

保守担当者向けメモ

  • 本関数は TidRangeNext から繰り返し呼ばれる可能性があるため、効率的な評価が求められる。
  • ItemPointer の比較ロジックは、ストレージエンジンが提供する ctid の順序付けに依存している点に注意すること。
