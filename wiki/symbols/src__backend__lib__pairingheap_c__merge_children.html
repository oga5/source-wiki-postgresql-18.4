merge_children

定義

  • 種別: function
  • 定義箇所: src/backend/lib/pairingheap.c:233

目的

ペアリングヒープのルートノード削除時に、残された子ノードのリストを「2パス法」を用いて効率的にマージし、単一のヒープ構造を再構築する。

シグネチャまたは形状

static pairingheap_node *merge_children(pairingheap *heap, pairingheap_node *children)

入力と出力

  • **入力**:
  • heap: 操作対象の pairingheap 構造体へのポインタ。
  • children: マージ対象となる子ノードの連結リスト（next_sibling で接続）。
  • **出力**:
  • マージ後の新しいルートノードを指す pairingheap_node へのポインタ。

前提条件と不変条件

  • children が NULL または単一ノードの場合、マージは行われずそのまま返される。
  • merge 関数がヒープの比較ルールに従って2つのノードを統合できること。

アルゴリズムまたは振る舞い

2パス法によるマージを実行する。 1. **第1パス**: 子ノードのリストを左から順に走査し、隣接するペアごとに merge を呼び出す。生成されたサブヒープを新しいリスト（pairs）の先頭に順次追加する。リストの末尾にノードが1つ余る場合は、そのまま pairs リストに含める。 2. **第2パス**: 第1パスで生成された pairs リストを右から左へ順次マージする。リストの先頭ノードを初期ルートとし、残りのノードを順次 merge することで単一のヒープを形成する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pairingheap_remove_first (ルート削除時)
  • pairingheap_remove (ノード削除時)
  • **呼び出し先**:
  • merge (各ノードの統合処理)

関連データ構造

  • pairingheap: ヒープ全体の管理構造体。
  • pairingheap_node: ヒープを構成するノード。next_sibling ポインタを使用して子ノードのリストを保持する。

失敗モードと境界条件

  • children が NULL の場合、即座に NULL を返す。
  • children が単一ノードの場合、マージを行わずそのノードを返す。
  • メモリ不足等のエラーハンドリングは merge 関数の挙動に依存する。

保守担当者向けメモ

  • 本関数はペアリングヒープの削除操作における計算量（償却 O(log n)）を維持するための核心的な処理である。
  • 2パス法は、ヒープの平坦化を防ぎ、再帰的な構造を維持するために不可欠である。
