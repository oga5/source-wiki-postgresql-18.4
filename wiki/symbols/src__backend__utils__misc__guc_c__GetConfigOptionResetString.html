GetConfigOptionResetString

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/misc/guc.c:4404

目的

指定された設定パラメータの「リセット値（デフォルト値）」を文字列形式で取得する。

シグネチャまたは形状

const char *GetConfigOptionResetString(const char *name)

入力と出力

  • **入力**: name (設定パラメータ名)
  • **出力**: パラメータのリセット値を表す文字列。型に応じて適切に変換される。

前提条件と不変条件

  • name に対応するパラメータがシステム内に存在すること（find_option が失敗した場合は ERROR を投げる）。
  • 呼び出し元が pg_read_all_settings ロール等の適切な権限を持っていること（ConfigOptionIsVisible によるチェック）。

アルゴリズムまたは振る舞い

1. find_option を呼び出し、指定された名前のパラメータ定義（config_generic）を検索する。 2. ConfigOptionIsVisible により、現在のユーザーが当該パラメータを参照する権限があるか検証する。権限がない場合は ERROR を発生させる。 3. パラメータの型（vartype）に応じて以下の処理を行う：

  • PGC_BOOL: reset_val を "on" または "off" に変換。
  • PGC_INT: reset_val を整数文字列に変換。
  • PGC_REAL: reset_val を浮動小数点文字列に変換。
  • PGC_STRING: reset_val をそのまま返す（NULL の場合は空文字列）。
  • PGC_ENUM: config_enum_lookup_by_value を使用して列挙値に対応する文字列を検索。

    • 4. 変換後の文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/variable.c:151
  • **呼び出し先**:
  • find_option
  • ConfigOptionIsVisible
  • config_enum_lookup_by_value
  • snprintf (内部バッファへの書き込み用)

関連データ構造

  • config_generic: パラメータの基本構造体。
  • config_bool, config_int, config_real, config_string, config_enum: 各型に対応する派生構造体。

失敗モードと境界条件

  • **権限不足**: ConfigOptionIsVisible が偽を返すと ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了する。
  • **パラメータ不在**: find_option が見つからない場合、ERROR を発生させる。
  • **バッファ制限**: PGC_INT および PGC_REAL の変換には静的バッファ（256バイト）を使用しており、これを超える場合は切り詰められる可能性がある。

保守担当者向けメモ

  • 本関数は静的バッファ buffer を使用しているため、スレッドセーフではない（PostgreSQL のバックエンドプロセスモデル上は単一スレッドのため許容されている）。
  • 新しいパラメータ型を追加した場合は、switch 文に該当するケースを追加する必要がある。
