heap_prune_record_prunable

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/pruneheap.c:1200

目的

ヒープページ内の剪定（pruning）処理において、ページヘッダーに記録すべき最小のトランザクションID（prune_xid）を更新する。

シグネチャまたは形状

static void heap_prune_record_prunable(PruneState *prstate, TransactionId xid)

入力と出力

  • prstate: 剪定処理の状態を保持する PruneState 構造体へのポインタ。
  • xid: 剪定対象として記録するトランザクションID。
  • 出力: なし（prstate->new_prune_xid を副作用として更新）。

前提条件と不変条件

  • xid は TransactionIdIsNormal を満たす有効なトランザクションIDである必要がある（Assert による検証）。

アルゴリズムまたは振る舞い

1. prstate->new_prune_xid が未設定（InvalidTransactionId）であるか、あるいは指定された xid が現在の new_prune_xid よりも古い場合、prstate->new_prune_xid を xid で更新する。 2. この処理は PageSetPruning マクロの動作と一致するように設計されており、ページヘッダーへの直接書き込みができない段階で、作業用状態を保持するために使用される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/pruneheap.c 内の複数箇所（1426行目、1463行目など）。
  • 呼び出し先: なし。

関連データ構造

  • PruneState: 剪定処理中の状態を管理する構造体。new_prune_xid フィールドを保持する。

失敗モードと境界条件

  • xid が正常なトランザクションIDでない場合、アサーション違反により停止する。

保守担当者向けメモ

  • 本関数はページヘッダーの更新を遅延させるための補助的なロジックである。将来的にページヘッダーの更新タイミングが変更された場合、本関数の役割と PageSetPruning マクロとの整合性を再確認すること。
