assign_default_text_search_config

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/ts_cache.c:669

目的

GUC（Grand Unified Configuration）パラメータ default_text_search_config が変更された際に、キャッシュされている全文検索設定を無効化し、次回の使用時に再読み込みを強制する。

シグネチャまたは形状

void assign_default_text_search_config(const char *newval, void *extra)

入力と出力

  • 入力:
  • newval: 新しい設定値（文字列）。本関数内では使用されない。
  • extra: 追加情報。本関数内では使用されない。
  • 出力: なし（void）

前提条件と不変条件

  • TSCurrentConfigCache 変数が、現在キャッシュされている全文検索設定の OID を保持していること。

アルゴリズムまたは振る舞い

1. TSCurrentConfigCache に InvalidOid を代入する。 2. これにより、次回全文検索設定が必要になった際に、キャッシュミスが発生してシステムカタログから設定が再取得されるようになる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/guc_tables.c において、GUC パラメータの割り当てフックとして登録されている。
  • 呼び出し先: なし。

関連データ構造

  • TSCurrentConfigCache: 全文検索設定の OID を保持するグローバル変数。

失敗モードと境界条件

  • 特になし。本関数は単純な代入操作のみを行うため、失敗することはない。

保守担当者向けメモ

  • 本関数は default_text_search_config の値そのものを検証するものではない。検証は check_default_text_search_config が担当する。
  • キャッシュの無効化は、単に OID を無効値にリセットすることで実現している。
