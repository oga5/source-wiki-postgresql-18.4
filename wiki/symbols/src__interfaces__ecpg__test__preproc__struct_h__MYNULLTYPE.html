MYNULLTYPE

定義

  • 種別: typedef（struct mynulltype の別名）
  • 定義箇所: src/interfaces/ecpg/test/preproc/struct.h:19

目的

ECPG（Embedded SQL C Preprocessor）のテストにおいて、ホスト変数として使用される構造体の各フィールドに対応するNULLインジケータを保持するための型定義です。

シグネチャまたは形状

typedef struct mynulltype MYNULLTYPE;

※構造体本体のメンバー構成については、mynulltype の定義を参照してください。

入力と出力

  • 入力: ECPGプリプロセッサがSQL文のホスト変数として本型を読み込みます。
  • 出力: 実行時には、SQLのNULL状態を管理するためのメタデータとしてECPGランタイムライブラリへ渡されます。

前提条件と不変条件

  • 本型はECPGのテスト用であり、プロダクションコードでの使用は想定されていません。
  • struct mynulltype と対になる構造体（mytype等）の各フィールドに対応するNULL状態を管理する設計となっています。

アルゴリズムまたは振る舞い

本型自体はデータ定義であり、実行時のロジックは持ちません。ECPGプリプロセッサがSQL文を解析する際、この型を介してホスト変数のNULLインジケータのメモリレイアウトを特定します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/interfaces/ecpg/test/expected/preproc-outofscope.c: get_var1 関数の引数として使用。
  • 同ファイル内の malloc によるメモリ確保および ECPGget_var を用いた型情報のバインド処理。
  • 呼び出し先:
  • 実行時に ECPGt_int 等のECPGランタイム型定義と組み合わされ、SQLのホスト変数バインド処理が行われます。

関連データ構造

  • struct mynulltype: 本型の実体。
  • struct mytype: 本型がNULL状態を管理対象とする対応構造体。

失敗モードと境界条件

  • 構造体のメンバー構成と、対応するSQL文のホスト変数定義が一致しない場合、ECPGプリプロセッサによる型変換やメモリレイアウトの解釈が失敗する可能性があります。

保守担当者向けメモ

  • 本型はECPGの型変換ルールを検証するためのテスト用定義です。変更を加える際は、対応する .pgc ファイル内の EXEC SQL 文および生成された .c ファイル（expected ディレクトリ内）への影響を確認してください。
