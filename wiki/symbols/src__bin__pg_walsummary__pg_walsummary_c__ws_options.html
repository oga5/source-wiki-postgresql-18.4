ws_options

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_walsummary/pg_walsummary.c:24

目的

pg_walsummary コマンドの実行時オプションを保持し、プログラム全体で共有するための構造体です。

シグネチャまたは形状

typedef struct ws_options
{
	bool		individual;
	bool		quiet;
} ws_options;

入力と出力

  • 入力: コマンドライン引数から解析された値が格納されます。
  • 出力: dump_one_relation 関数などの処理ロジックにおいて、出力形式や動作の制御フラグとして参照されます。

前提条件と不変条件

  • main 関数内で memset を用いてゼロ初期化されることが想定されています。

アルゴリズムまたは振る舞い

  • individual: true の場合、ブロック番号を範囲（start..end）ではなく個別に表示します。
  • quiet: true の場合、パース処理は実行されますが、標準出力への結果表示が抑制されます。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（インスタンスの生成と初期化）、dump_one_relation 関数（引数として受け取り参照）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • RelFileLocator: dump_one_relation 関数内で本構造体と共に引数として使用されます。

失敗モードと境界条件

  • 構造体自体のメモリレイアウトや初期化に関する境界条件は特にありません。

保守担当者向けメモ

  • 新たなコマンドラインオプションを追加する場合は、この構造体にフィールドを追加し、main 関数内の解析ロジックを更新してください。
