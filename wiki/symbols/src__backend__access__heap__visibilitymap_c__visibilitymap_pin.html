visibilitymap_pin

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/visibilitymap.c:192

目的

指定されたヒープブロックに対応する可視性マップ（Visibility Map）のページをバッファプールに読み込み、ピン留め（pin）します。

シグネチャまたは形状

void visibilitymap_pin(Relation rel, BlockNumber heapBlk, Buffer *vmbuf)

入力と出力

  • rel: 対象のリレーション。
  • heapBlk: 可視性情報を取得したいヒープブロック番号。
  • vmbuf: 読み込んだマップページを格納するバッファへのポインタ。既存のバッファが渡された場合、再利用が試みられます。

前提条件と不変条件

  • vmbuf に有効なバッファが既に設定されている場合、そのバッファが対象のマップブロックと一致していれば、追加の処理を行わずに終了します。
  • マップブロックと一致しない場合は、既存のバッファを解放してから新しいバッファを読み込みます。

アルゴリズムまたは振る舞い

1. HEAPBLK_TO_MAPBLOCK マクロを使用して、heapBlk に対応する可視性マップのブロック番号を算出します。 2. vmbuf が有効なバッファを指しているか確認します。

  • 有効かつブロック番号が一致する場合、そのままリターンします。
  • 有効だがブロック番号が異なる場合、ReleaseBuffer で古いバッファを解放します。

    • 3. vm_readbuf を呼び出し、対象のマップブロックを読み込んで vmbuf に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c (複数箇所)
  • src/backend/access/heap/heapam_xlog.c (複数箇所)
  • contrib/pg_surgery/heap_surgery.c
  • **呼び出し先**:
  • vm_readbuf (内部関数)
  • BufferGetBlockNumber, BufferIsValid, ReleaseBuffer (バッファ管理関数)

関連データ構造

  • Buffer: バッファプール内のページを識別するハンドル。
  • Relation: リレーションのメタデータ構造。

失敗モードと境界条件

  • vm_readbuf が内部でエラーを発生させた場合、本関数も例外をスローします。
  • 無効な heapBlk が指定された場合、マップブロックの算出結果が不正になる可能性があります。

保守担当者向けメモ

  • 本関数は、可視性マップに対する一連の操作（セットやクリア）の開始点として機能します。
  • 呼び出し側は、操作終了後に ReleaseBuffer を呼び出してバッファを解放する責任を負います。
