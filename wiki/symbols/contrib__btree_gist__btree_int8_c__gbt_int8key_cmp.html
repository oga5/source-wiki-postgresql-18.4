gbt_int8key_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int8.c:55

目的

GiSTインデックス構築時における int64KEY 型のキー同士の比較を行い、ソート順序を決定する。

シグネチャまたは形状

static int gbt_int8key_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部の t メンバが int64KEY としてキャストされる。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • 0: 両者が等しい場合。
  • 1: a が b より大きい場合。
  • -1: a が b より小さい場合。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体を指している必要がある。
  • int64KEY 構造体は lower および upper メンバを持つ。

アルゴリズムまたは振る舞い

1. 入力された Nsrt 構造体から int64KEY へのポインタを取得する。 2. まず lower 値を比較する。ia->lower が ib->lower より大きければ 1 を、小さければ -1 を返す。 3. lower 値が等しい場合、upper 値を比較する。 4. upper 値も等しければ 0 を返し、そうでなければ大小関係に応じて 1 または -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int8.c:89（tinfo 構造体の初期化時に参照されている）。
  • **呼び出し先**: 直接的な関数呼び出しはなし。

関連データ構造

  • int64KEY: int8 型の範囲（lower, upper）を保持する構造体。
  • Nsrt: ソート処理時に使用される中間データ構造。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動は定義されていない。
  • int64 の範囲を超える比較は行われない（int8 型の範囲内に限定される）。

保守担当者向けメモ

  • 本関数は int8 型のソートサポート用比較関数として機能する。
  • 比較ロジックは lower を優先し、次に upper を見る辞書順比較である。
