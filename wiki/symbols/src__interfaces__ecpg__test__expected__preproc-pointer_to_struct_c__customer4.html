customer4

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:109

目的

ECPG（Embedded SQL for C）プリプロセッサの型解析能力を検証するための、構造体定義の一つです。構造体へのポインタをホスト変数として扱う際のデータマッピングの正当性を確認するために使用されます。

シグネチャまたは形状

struct customer4 {
  struct varchar_3 { int len; char arr[50]; } name;
  int phone;
} *custs4;

入力と出力

  • **入力**: SQLクエリの結果セット（name および phone フィールド）。
  • **出力**: 確保されたメモリ領域へのデータフェッチ結果。

前提条件と不変条件

  • malloc により sizeof(struct customer4) 分のメモリがヒープ上に確保されていること。
  • メンバ name は len（長さ）と arr（50バイトの文字配列）を持つ varchar 型として定義されていること。

アルゴリズムまたは振る舞い

1. malloc を用いて構造体インスタンスを動的に確保し、ポインタ custs4 に代入します。 2. ECPGの ECPGdo 関数を介して、データベースから取得した値を custs4->name および custs4->phone に格納します。 3. 実行時には sizeof(struct customer4) がメモリサイズ指定として使用され、型情報の整合性が検証されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c（メモリ確保およびデータフェッチ）
  • src/interfaces/ecpg/test/expected/preproc-array_of_struct.c（データマッピングの参照）
  • **呼び出し先**:
  • malloc（メモリ確保）
  • ECPGdo（SQL実行およびデータフェッチ）

関連データ構造

  • varchar_3: customer4 内にネストされた varchar 型構造体。
  • custs4: customer4 型を指すポインタ変数。

失敗モードと境界条件

  • malloc が失敗した場合、custs4 は NULL となり、その後の ECPGdo 呼び出しでセグメンテーション違反が発生する可能性があります。
  • name フィールドのデータが arr[50] のサイズを超える場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本構造体はプリプロセッサのテスト用生成コードの一部です。構造体のメンバ構成を変更する場合は、対応する .pgc ファイルおよび生成されるテストコードの整合性を確認してください。
  • ECPGdo 呼び出し時の型指定（ECPGt_varchar, ECPGt_int）と構造体メンバの型が一致していることを常に確認してください。
