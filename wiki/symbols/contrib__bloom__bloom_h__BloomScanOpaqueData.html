BloomScanOpaqueData

定義

  • 種別: struct
  • 定義箇所: contrib/bloom/bloom.h:166

目的

Bloom インデックスのスキャン実行時における状態を保持するための構造体です。

シグネチャまたは形状

typedef struct BloomScanOpaqueData
{
	BloomSignatureWord *sign;	/* Scan signature */
	BloomState	state;
} BloomScanOpaqueData;

入力と出力

  • 本構造体自体はデータ保持用であり、直接的な関数入出力は持ちません。
  • blscan.c にて palloc を用いてメモリ確保され、スキャン状態の管理に使用されます。

前提条件と不変条件

  • state フィールドには、インデックス操作に必要なハッシュ関数情報や列設定が格納されている必要があります。
  • sign はスキャン対象の署名を保持する領域を指す必要があります。

アルゴリズムまたは振る舞い

  • インデックススキャン中に、検索条件から生成された署名（sign）と、インデックス内の各タプルの署名を比較するための状態を維持します。
  • BloomState を介して、インデックスのメタ情報や実行時の設定にアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/bloom/blscan.c 内の関数（palloc によるインスタンス化を確認）。
  • 呼び出し先: 構造体内部で BloomState を保持しており、インデックススキャン処理の各段階で参照されます。

関連データ構造

  • BloomSignatureWord: スキャン署名の構成要素。
  • BloomState: インデックスの実行時状態を保持する構造体。
  • BloomScanOpaque: 本構造体へのポインタ型。

失敗モードと境界条件

  • メモリ不足時: palloc 呼び出し時にエラーが発生する可能性があります。
  • 構造体の初期化不備: BloomState が適切に初期化されていない状態でスキャンを実行した場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は blscan.c で動的に割り当てられるため、スキャン終了時に適切に解放されていることを確認してください。
  • BloomState の変更が本構造体の動作に直接影響するため、BloomState の定義変更時には整合性を確認してください。
