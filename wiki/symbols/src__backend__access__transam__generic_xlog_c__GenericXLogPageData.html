GenericXLogPageData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/generic_xlog.c:50

目的

汎用WAL（Write-Ahead Logging）フレームワークにおいて、変更対象となるバッファの状態と、その変更差分（デルタ）を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct
{
	Buffer		buffer;			/* 登録されたバッファ */
	int			flags;			/* バッファのフラグ */
	int			deltaLen;		/* デルタフィールドで消費されたサイズ */
	char	   *image;			/* 変更用ページイメージのコピー */
	char		delta[MAX_DELTA_SIZE];	/* ページイメージ間の差分 */
} GenericXLogPageData;

入力と出力

本構造体は関数ではなくデータ型です。GenericXLogState 構造体内の配列 pages の要素として管理され、WALレコード構築プロセスにおける各バッファの変更状態を保持します。

前提条件と不変条件

  • image メンバは、インプレース（直接）編集を避けるためにアライメントされたメモリ領域を指す必要があります。
  • delta メンバは MAX_DELTA_SIZE までの差分データを格納可能です。

アルゴリズムまたは振る舞い

  • **変更管理**: GenericXLogRegisterBuffer を通じて登録されたバッファに対し、image への編集が行われます。
  • **差分計算**: GenericXLogFinish 実行時に、computeDelta 関数が image と元のページを比較し、その結果を delta メンバに格納します。
  • **断片化処理**: writeFragment 関数により、計算されたデルタがWALレコードへ書き込まれます。

呼び出し元と呼び出し先

  • **呼び出し元**: GenericXLogStart で初期化された GenericXLogState を介して、GenericXLogRegisterBuffer、GenericXLogFinish、GenericXLogAbort 等から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出しません。内部的に computeDelta、computeRegionDelta、writeFragment などの静的関数に引数として渡されます。

関連データ構造

  • GenericXLogState: 本構造体を配列として保持し、WAL構築全体のライフサイクルを管理します。
  • Buffer: PostgreSQLのバッファ管理レイヤーにおけるバッファ識別子。

失敗モードと境界条件

  • delta のサイズが MAX_DELTA_SIZE を超える場合、汎用WALの仕組みでは処理できません。
  • 登録されたバッファが正しく管理されていない場合、GenericXLogFinish 実行時のデルタ計算で不整合が生じる可能性があります。

保守担当者向けメモ

  • image はページ編集の作業用コピーです。メモリリークを防ぐため、GenericXLogFinish または GenericXLogAbort で適切に解放されるライフサイクルを維持してください。
  • delta の計算ロジック（computeDelta）は、ページ内の「ホール（pd_lowerとpd_upperの間）」をゼロクリアする処理と密接に関係しています。変更を加える際は、物理的なページ構造の整合性に注意してください。
