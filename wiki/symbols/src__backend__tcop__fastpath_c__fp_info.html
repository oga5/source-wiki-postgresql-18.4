fp_info

定義

  • 種別: struct
  • 定義箇所: src/backend/tcop/fastpath.c:48

目的

PostgreSQL の Fastpath（PQfn）機能において、クライアントから直接呼び出されるサーバー側関数のメタデータを保持するために使用されます。

シグネチャまたは形状

struct fp_info
{
    Oid         funcid;
    FmgrInfo    flinfo;             /* 関数ルックアップ情報 */
    Oid         namespace;          /* pg_proc 由来のスキーマ情報 */
    Oid         rettype;            /* 戻り値の型 */
    Oid         argtypes[FUNC_MAX_ARGS]; /* 引数の型配列 */
    char        fname[NAMEDATALEN]; /* ログ用関数名 */
};

入力と出力

本構造体は関数呼び出しの準備段階で fetch_fp_info によってシステムカタログから値が設定されます。直接的な入力・出力関数ではありませんが、関数実行のメタデータコンテナとして機能します。

前提条件と不変条件

  • fetch_fp_info を呼び出す際、呼び出し元は fp_info 構造体へのポインタを渡し、初期化（MemSet 等）を行う必要があります。
  • トランザクション境界をまたいだキャッシュは行われない設計となっており、呼び出しごとにシステムカタログから再取得されます。

アルゴリズムまたは振る舞い

1. **初期化**: fetch_fp_info が pg_proc を検索し、対象関数の OID に基づいて各フィールドを埋めます。 2. **検証**: 保持されたメタデータは、parse_fcall_arguments における引数の解析や、実行権限チェックの基盤として利用されます。 3. **実行**: 最終的に FmgrInfo (flinfo) が FunctionCallInvoke に渡され、実際の関数実行が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • fetch_fp_info（情報の構築）
  • parse_fcall_arguments（引数解析の参照）
  • HandleFunctionRequest（リクエスト処理のメインループ内でのインスタンス化）
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • FmgrInfo: flinfo フィールドで使用される、関数実行のための内部管理構造体。
  • pg_proc: 本構造体の各フィールドの元となるシステムカタログ。

失敗モードと境界条件

  • FUNC_MAX_ARGS を超える引数を持つ関数はサポートされません。
  • Fastpath の制約上、照合順序（Collation）を必要とする関数や、集合を返す関数（proretset）の実行には対応していません。

保守担当者向けメモ

  • 過去にキャッシュの試行がありましたが、現在はトランザクションの整合性を保つため、都度カタログ検索を行う設計となっています。
  • 本構造体に関連するメモリ割り当ては MessageContext 上で行われるため、明示的な解放は不要です。
