ArrayIterator

定義

  • 種別: struct (不透明ポインタ ArrayIteratorData *)
  • 定義箇所: src/include/utils/array.h:258

目的

PostgreSQLの配列データ型を反復処理するための抽象化されたイテレータ構造体です。

シグネチャまたは形状

typedef struct ArrayIteratorData *ArrayIterator;

※ ArrayIteratorData の内部構造は本ソース抜粋には含まれていません。

入力と出力

  • **生成**: array_create_iterator(ArrayType *arr, int slice_ndim, ArrayMetaState *mstate) により生成されます。
  • **反復**: array_iterate(ArrayIterator iterator, Datum *value, bool *isnull) を通じて、現在の要素の Datum 値と NULL フラグを出力します。
  • **破棄**: array_free_iterator(ArrayIterator iterator) によりメモリが解放されます。

前提条件と不変条件

  • array_iterate を呼び出す前に、array_create_iterator で有効なイテレータを初期化する必要があります。
  • 反復処理の終了後、array_free_iterator を呼び出してリソースを解放する必要があります。

アルゴリズムまたは振る舞い

1. array_create_iterator で配列の走査状態を初期化します。 2. array_iterate を繰り返し呼び出すことで、配列内の要素を順次取得します。この関数は、次の要素が存在する場合は true を、走査が完了した場合は false を返します。 3. 走査完了後、array_free_iterator でイテレータを破棄します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/array_userfuncs.c
  • src/backend/utils/adt/arrayfuncs.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • palloc0 (イテレータ構造体のメモリ確保時)

関連データ構造

  • ArrayType: 反復対象となる標準的なフラット配列。
  • ArrayMetaState: 配列のメタデータ（要素型情報など）を保持する構造体。
  • ArrayIteratorData: ArrayIterator が指し示す実体。

失敗モードと境界条件

  • 配列が空の場合、array_iterate は即座に false を返します。
  • メモリ不足時には palloc0 等のメモリ管理関数経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは不透明ポインタとして定義されています。内部構造の変更が必要な場合は ArrayIteratorData の定義を確認してください。
  • array_iterate の戻り値が false になった後の挙動については、呼び出し側で適切に終了処理を行う必要があります。
