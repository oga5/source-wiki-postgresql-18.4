SnapBuildSetTwoPhaseAt

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/snapbuild.c:294

目的

SnapBuild 構造体において、2フェーズコミット（Two-Phase Commit）の準備が完了した、あるいは追跡を開始すべき WAL 位置（LSN）を設定します。

シグネチャまたは形状

void SnapBuildSetTwoPhaseAt(SnapBuild *builder, XLogRecPtr ptr)

入力と出力

  • builder: 操作対象の SnapBuild 構造体へのポインタ。
  • ptr: 設定する WAL 位置（XLogRecPtr）。
  • 戻り値: なし。

前提条件と不変条件

  • builder は有効な SnapBuild 構造体を指している必要があります。

アルゴリズムまたは振る舞い

引数として渡された ptr を、builder->two_phase_at フィールドに代入します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/logical.c:607 にて、論理デコーディングの開始時に start_lsn を設定するために呼び出されています。
  • 呼び出し先: なし（単純な代入操作のみ）。

関連データ構造

  • SnapBuild: 論理デコーディングにおけるスナップショット構築状態を保持する構造体。
  • XLogRecPtr: WAL 内のログ位置を示すデータ型。

失敗モードと境界条件

  • builder が NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は単なるセッターであり、状態遷移のロジックは含みません。2フェーズコミットのサポートに関連する重要な設定値であるため、変更時は SnapBuildGetTwoPhaseAt との整合性に注意してください。
