AtAbort_Portals

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/portalmem.c:780

目的

トランザクションアボート時に、全ポータルの状態を整理し、不要なリソースを解放してクリーンアップを行う。

シグネチャまたは形状

void AtAbort_Portals(void)

入力と出力

  • 入力: なし（グローバルな PortalHashTable を参照）
  • 出力: なし（副作用としてポータルの状態変更およびメモリ解放を行う）

前提条件と不変条件

  • PortalHashTable が初期化されていること。
  • トランザクションアボート処理の文脈で呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. PortalHashTable を走査し、各ポータルに対して以下の処理を行う。 2. FATAL エラー発生時、PORTAL_ACTIVE なポータルを FAILED 状態へ遷移させる。 3. 前トランザクション由来のポータル（createSubid == InvalidSubTransactionId）および autoHeld なポータルはスキップする。 4. PORTAL_READY 状態のポータルを FAILED 状態へ遷移させる。 5. ポータルの cleanup フックが存在すれば実行し、フックをクリアする。 6. キャッシュされたプランの参照を解放する。 7. resowner を NULL に設定する。 8. PORTAL_ACTIVE 以外のポータルについて、portalContext 配下のメモリコンテキストを削除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c（トランザクションアボート処理）
  • 呼び出し先: MarkPortalFailed, portal->cleanup (関数ポインタ), PortalReleaseCachedPlan, MemoryContextDeleteChildren

関連データ構造

  • PortalHashTable: 管理対象の全ポータルを保持するハッシュテーブル。
  • Portal: ポータルの状態（status, createSubid, autoHeld, cleanup, resowner, portalContext）を保持する構造体。

失敗モードと境界条件

  • shmem_exit_inprogress が真の場合、PORTAL_ACTIVE なポータルは FAILED にマークされる。
  • 既に FAILED や DONE になっているポータルに対しては、メモリ解放のみが試行される。

保守担当者向けメモ

  • 本関数はトランザクションアボート時のリソースリークを防ぐための重要なクリーンアップ処理である。
  • portal->cleanup フックの実行順序や、メモリコンテキストの削除タイミングは、ポータルの状態遷移と密接に関わっているため注意が必要。
