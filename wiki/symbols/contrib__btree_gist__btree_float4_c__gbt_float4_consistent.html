gbt_float4_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float4.c:129

目的

GiSTインデックスの検索時、特定のクエリ条件がインデックス内のノード（float4KEY）と一致するかを判定します。

シグネチャまたは形状

Datum gbt_float4_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY *entry（インデックスエントリ）
  • PG_GETARG_FLOAT4(1): float4 query（検索対象の値）
  • PG_GETARG_UINT16(2): StrategyNumber strategy（比較演算の戦略）
  • PG_GETARG_POINTER(4): bool *recheck（再チェックフラグ）
  • **出力**:
  • PG_RETURN_BOOL: 判定結果（bool）を返します。
  • *recheck: 常に false が設定されます（本関数による判定は完全であるため）。

前提条件と不変条件

  • entry->key は float4KEY 構造体として解釈可能です。
  • recheck は呼び出し元によって提供される有効なメモリ領域である必要があります。

アルゴリズムまたは振る舞い

1. entry->key から float4KEY を取得し、GBT_NUMKEY_R 構造体に変換します。 2. *recheck を false に設定します。 3. tinfo（gbtree_ninfo 型の静的変数）および GIST_LEAF(entry) を用いて、btree_utils_num.c で定義された汎用関数 gbt_num_consistent を呼び出し、検索条件との一致を判定します。 4. 判定結果を Datum 型として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのGiSTインデックス走査エンジンから呼び出されます（直接的な呼び出し元コードの詳細はソース内に明示されていません）。
  • **呼び出し先**: gbt_num_consistent（btree_utils_num.c 内の汎用関数）。

関連データ構造

  • float4KEY: インデックス内で保持される範囲（lower, upper）を定義する構造体。
  • GBT_NUMKEY_R: 数値範囲を扱うための内部表現構造体。
  • tinfo: float4 型の比較ロジックを保持する gbtree_ninfo 型の静的設定テーブル。

失敗モードと境界条件

  • recheck は常に false を返すため、インデックス検索後にヒープでの再チェックは発生しません。
  • float4 型の範囲外の値や不正な StrategyNumber が渡された場合の挙動は、依存先の gbt_num_consistent の実装に依存します。

保守担当者向けメモ

  • 本関数は btree_utils_num.c のラッパーです。ロジックの変更が必要な場合は、まず tinfo の定義や btree_utils_num 側の実装を確認してください。
  • PG_FUNCTION_INFO_V1(gbt_float4_consistent) により、PostgreSQLの関数マネージャに登録されています。
