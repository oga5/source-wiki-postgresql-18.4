pg_ltoa

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numutils.c:1119

目的

符号付き32ビット整数（int32）を文字列形式に変換し、指定されたバッファに格納します。

シグネチャまたは形状

int pg_ltoa(int32 value, char *a);

入力と出力

  • **入力**:
  • value: 変換対象の int32 値。
  • a: 変換結果を格納するための文字バッファ（十分なサイズが必要）。
  • **出力**:
  • 戻り値: 書き込まれた文字列の長さ（NUL終端文字を含まない）。

前提条件と不変条件

  • 引数 a が指すバッファは、負の符号と数値、および NUL 終端を格納するのに十分なサイズ（最低 12 バイト程度）である必要があります。
  • 戻り値は a に書き込まれた文字数と一致します。

アルゴリズムまたは振る舞い

1. value が負の場合、符号 - をバッファの先頭に書き込み、絶対値を計算して uvalue に代入します。 2. pg_ultoa_n を呼び出し、数値を文字列に変換してバッファの適切な位置へ書き込みます。 3. 変換された文字列の末尾に NUL 終端文字（\0）を付加します。 4. 最終的な文字列の長さを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printsimple.c:108
  • src/backend/utils/adt/int.c:332

  • **呼び出し先**:
  • pg_ultoa_n

関連データ構造

  • int32 (PostgreSQL の 32 ビット整数型)

失敗モードと境界条件

  • バッファ a のサイズが不足している場合、バッファオーバーフローが発生します。
  • int32 の最小値（-2147483648）を渡した場合、絶対値への変換時にオーバーフローが発生する可能性があるため、内部で uint32 へのキャストと演算を行っています。

保守担当者向けメモ

  • 本関数は NUL 終端を付加しますが、内部で呼び出している pg_ultoa_n は NUL 終端を付加しない仕様である点に注意してください。
  • パフォーマンス向上のため、標準ライブラリの sprintf ではなく pg_ultoa_n を介した最適化された変換ロジックを使用しています。
