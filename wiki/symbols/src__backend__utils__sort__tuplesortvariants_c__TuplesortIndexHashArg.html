TuplesortIndexHashArg

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:143

目的

ハッシュインデックスのソート処理において、ハッシュコードのビットマスクやバケット数といったハッシュ固有のパラメータを保持し、Tuplesortstate に注入するための構造体です。

シグネチャまたは形状

typedef struct {
    TuplesortIndexArg index;
    uint32 high_mask;
    uint32 low_mask;
    uint32 max_buckets;
} TuplesortIndexHashArg;

入力と出力

本構造体自体はデータ保持用であり、直接的な入出力関数を持ちません。tuplesort_begin_index_hash 等の初期化関数を通じてメモリ上に割り当てられ、ソート処理中のパラメータとして参照されます。

前提条件と不変条件

  • TuplesortIndexArg を継承（埋め込み）しており、インデックスソートに必要な共通情報が先頭に配置されています。
  • high_mask および low_mask は、ハッシュコードのソート可能部分を抽出するために使用されます。

アルゴリズムまたは振る舞い

ハッシュインデックスのソート時に、ハッシュコードの特定のビット範囲をマスクして比較を行うための設定値を保持します。ソートエンジン（tuplesort.c）が型固有の比較ロジックを実行する際、この構造体を参照してハッシュ値の正規化や比較を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesortvariants.c 内の tuplesort_begin_index_hash（推測される初期化箇所）および関連するソート処理関数。
  • **呼び出し先**: 本構造体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • TuplesortIndexArg: 基底となるインデックスソート用引数構造体。
  • Tuplesortstate: ソート処理の全状態を管理する構造体。

失敗モードと境界条件

  • メモリ不足時、palloc による割り当てに失敗する可能性があります。
  • max_buckets 等のパラメータが不正な場合、ハッシュコードのマスク処理において予期しないソート順序となる可能性があります。

保守担当者向けメモ

  • 本構造体は tuplesortvariants.c 内でハッシュインデックス専用のパラメータを管理するために使用されます。
  • TuplesortIndexHashArg のフィールドを変更する場合は、これらを参照している comparetup_index_hash 等の比較ロジックへの影響を考慮してください。
