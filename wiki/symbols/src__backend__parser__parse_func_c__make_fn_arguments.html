make_fn_arguments

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_func.c:1824

目的

関数呼び出しの引数リストに対し、宣言された型と実際の型が一致しない場合に、暗黙的な型変換（キャスト）を適用して構文木を修正する。

シグネチャまたは形状

void make_fn_arguments(ParseState *pstate, List *fargs, Oid *actual_arg_types, Oid *declared_arg_types)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。
  • fargs: 引数ノードのリスト。
  • actual_arg_types: 各引数の現在の型（OID）の配列。
  • declared_arg_types: 関数定義で要求される型（OID）の配列。
  • **出力**: なし（fargs リスト内のノードを直接書き換えることで副作用を及ぼす）。

前提条件と不変条件

  • fargs の長さと actual_arg_types / declared_arg_types 配列の要素数は一致している必要がある。
  • actual_arg_types と declared_arg_types の各インデックスにおいて、型が異なる場合は coerce_type が成功する互換性があることが前提となる。

アルゴリズムまたは振る舞い

1. fargs リストを走査する。 2. 各引数について actual_arg_types[i] と declared_arg_types[i] を比較する。 3. 型が一致しない場合、coerce_type を呼び出し、COERCION_IMPLICIT および COERCE_IMPLICIT_CAST を指定して型変換を行う。

  • 引数が NamedArgExpr の場合、その内部の arg フィールドを変換する。
  • それ以外の場合、リスト内のノード自体を変換後のノードで置換する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_func.c (関数解決後のノード構築時)
  • src/backend/parser/parse_oper.c (演算子解決時)
  • src/backend/optimizer/util/clauses.c
  • **呼び出し先**:
  • coerce_type (型変換の実行)

関連データ構造

  • ParseState: 解析中のコンテキスト情報。
  • NamedArgExpr: 名前付き引数構文を表現するノード。
  • List: 引数リスト。

失敗モードと境界条件

  • actual_arg_types や declared_arg_types が指す配列の範囲外アクセスについては、呼び出し側がリストの長さを正しく管理していることに依存する。
  • coerce_type が変換不能な型を検出した場合、内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、関数呼び出しの引数解決における最終的な型調整ステップである。
  • NamedArgExpr を特別扱いしているのは、名前付き引数の構造を維持しつつ、その値部分のみをキャストするためである。
