PostRestoreCommand

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:281

目的

外部コマンド（restore_command）の実行が終了したことをシステムに通知し、状態を更新する。

シグネチャまたは形状

void PostRestoreCommand(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • 本関数は PreRestoreCommand と対で使用されることを想定している。
  • 実行後、in_restore_command フラグは必ず false に設定される。

アルゴリズムまたは振る舞い

1. グローバル変数 in_restore_command を false に設定する。これにより、システムは現在外部コマンドが実行中ではないと判断する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogarchive.c 内のアーカイブ復旧処理フローから呼び出される。
  • 呼び出し先: なし。

関連データ構造

  • in_restore_command: src/backend/postmaster/startup.c で定義されているグローバルなブール型フラグ。SIGTERM受信時の挙動を制御するために使用される。

失敗モードと境界条件

  • 特筆すべき失敗モードはない。本関数は単純なフラグのクリアを行う。

保守担当者向けメモ

  • in_restore_command フラグは、アーカイブからの復旧中にSIGTERMが届いた際、安全にプロセスを終了させるためのガードとして機能する。本関数を呼び出すことで、そのガードを解除する。
