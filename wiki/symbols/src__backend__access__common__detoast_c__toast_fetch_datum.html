toast_fetch_datum

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/detoast.c:342

目的

TOASTテーブルに格納されている外部データ（varlena）を読み出し、メモリ上に復元します。

シグネチャまたは形状

struct varlena *toast_fetch_datum(struct varlena *attr)

入力と出力

  • **入力**: attr - 外部ストレージを指す varlena 構造体ポインタ。
  • **出力**: 復元されたデータを含む varlena 構造体へのポインタ。メモリは palloc によって確保されます。

前提条件と不変条件

  • 入力 attr は VARATT_IS_EXTERNAL_ONDISK を満たす必要があります。満たさない場合はエラーが発生します。

アルゴリズムまたは振る舞い

1. 入力データから外部ポインタ情報（varatt_external）を抽出します。 2. 復元後のデータサイズを計算し、メモリを確保します。 3. 圧縮フラグに応じて、結果の varlena ヘッダーを適切に設定します。 4. table_open を使用して対象のTOASTテーブルを AccessShareLock で開きます。 5. table_relation_fetch_toast_slice を呼び出し、TOASTテーブルから全チャンクを読み込んでメモリ上に展開します。 6. TOASTテーブルを閉じ、復元されたデータを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/detoast.c 内の複数箇所（detoast_attr 等の復元処理フロー）。
  • **呼び出し先**:
  • table_open / table_close (テーブルアクセス層)
  • table_relation_fetch_toast_slice (TOASTチャンク取得)
  • palloc (メモリ確保)

関連データ構造

  • varlena: 可変長データの基本構造体。
  • varatt_external: 外部ストレージ上のデータを指すポインタ構造体。

失敗モードと境界条件

  • VARATT_IS_EXTERNAL_ONDISK でないデータが渡された場合、elog(ERROR) により処理が中断されます。
  • attrsize が 0 の場合は、読み込みを行わず空の構造体を返します。

保守担当者向けメモ

  • 本関数が返すメモリは呼び出し元で pfree する必要があります。
  • VARATT_EXTERNAL_GET_POINTER を使用してアライメントを考慮したコピーを行っているため、入力ポインタの直接参照には注意してください。
