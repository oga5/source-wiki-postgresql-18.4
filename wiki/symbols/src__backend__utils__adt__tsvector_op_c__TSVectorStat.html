TSVectorStat

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/tsvector_op.c:58

目的

全文検索における語彙統計（ts_stat 関数など）を算出する際、文書集合内の語彙の出現頻度や重みを累積・保持するための作業用構造体です。

シグネチャまたは形状

typedef struct {
    int32       weight;
    uint32      maxdepth;
    StatEntry **stack;
    uint32      stackpos;
    StatEntry  *root;
} TSVectorStat;

入力と出力

  • **入力**: ts_accum 関数等を通じて、TSVector 型のデータおよび MemoryContext が供給されます。
  • **出力**: 累積された統計情報は root フィールドを起点とする StatEntry ツリー構造として保持されます。

前提条件と不変条件

  • root は統計情報のツリー構造の起点となります。
  • stack および stackpos は、ツリー走査や累積処理における現在の深さやパスを管理するために使用されます。
  • メモリ管理は MemoryContext に依存しており、ts_accum 等で初期化される際に MemoryContextAllocZero が使用されます。

アルゴリズムまたは振る舞い

  • ts_stat 関数がテーブルをスキャンする際、この構造体を用いて語彙を累積します。
  • insertStatEntry や chooseNextStatEntry を通じて、新しい語彙がツリーに追加されます。
  • walkStatEntryTree により、構築されたツリーを走査して最終的な統計結果を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: ts_accum（初期化および累積）、walkStatEntryTree（走査）、および ts_stat 関連の関数群から参照されます。
  • **呼び出し先**: StatEntry 構造体（ツリーのノードとして参照）。

関連データ構造

  • StatEntry: TSVectorStat が管理するツリーの各ノードを構成する構造体。
  • TSVector: 統計対象となる全文検索ベクトル型。

失敗モードと境界条件

  • メモリ不足時には MemoryContext の割り当てが失敗する可能性があります。
  • 非常に深いツリー構造や大量の語彙を処理する場合、再帰的な走査やスタック管理においてメモリ消費量が増大する可能性があります。

保守担当者向けメモ

  • 本構造体は tsvector_op.c 内の統計集計ロジックに特化しています。
  • stack 管理はツリーの深さに依存するため、極端なデータセットに対するスタックオーバーフローの可能性について考慮が必要です。
