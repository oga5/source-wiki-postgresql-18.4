dshash_table

定義

  • 種別: struct
  • 定義箇所: src/backend/lib/dshash.c:103

目的

動的共有メモリ（DSA）領域上に構築される、プロセス間で共有可能なオープンハッシュテーブルのバックエンドローカルな状態を保持する構造体です。

シグネチャまたは形状

struct dshash_table
{
    dsa_area   *area;           /* Backing dynamic shared memory area. */
    dshash_parameters params;   /* Parameters. */
    void       *arg;            /* User-supplied data pointer. */
    dshash_table_control *control;  /* Control object in DSM. */
    dsa_pointer *buckets;       /* Current bucket pointers in DSM. */
    size_t      size_log2;      /* log2(number of buckets) */
};

入力と出力

  • 本構造体は、ハッシュテーブルの操作関数（dshash_find 等）の第一引数として渡され、操作対象のハッシュテーブルの状態を特定するために使用されます。

前提条件と不変条件

  • dsa_pointer を介して共有メモリ上のデータにアクセスするため、直接的なメモリポインタの保持は避ける必要があります。
  • size_log2 はバケット数の対数値を保持し、リサイズ時に更新されます。

アルゴリズムまたは振る舞い

  • **状態管理**: 共有メモリ上の制御ブロック（dshash_table_control）とローカルなバケットポインタを同期させます。
  • **リサイズ対応**: ensure_valid_bucket_pointers を通じて、他プロセスによるリサイズが発生していないかを確認し、必要に応じてローカルの buckets ポインタを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: dshash.c 内の各種操作関数（dshash_find_or_insert, dshash_delete_key 等）から参照されます。
  • **呼び出し先**: 内部関数 delete_item, resize, ensure_valid_bucket_pointers, find_in_bucket, insert_item_into_bucket, insert_into_bucket, delete_key_from_bucket, delete_item_from_bucket, hash_key, equal_keys, copy_key 等が本構造体を引数として受け取ります。

関連データ構造

  • dshash_table_control: 共有メモリ上に配置されるハッシュテーブルの制御ブロック。
  • dshash_partition: ロックとエントリ数を管理するパーティション単位の構造体。
  • dshash_table_item: ハッシュテーブル内の個々のエントリ。
  • dsa_area: 基盤となる動的共有メモリ領域。

失敗モードと境界条件

  • **リサイズ時の競合**: リサイズ時は全パーティションのロックを取得するため、頻繁なリサイズはパフォーマンス低下を招きます。
  • **デッドロック**: 複数のパーティションロックを取得する際は、パーティション番号の昇順で取得する必要があります。
  • **スキャン中のブロック**: シーケンシャルスキャン中はパーティションロックを保持し続けるため、長時間のスキャンは他の操作をブロックする可能性があります。

保守担当者向けメモ

  • 本構造体はバックエンドローカルな状態を保持するものであり、共有メモリ上の実体は dshash_table_control です。
  • 共有メモリ上のデータアクセスには必ず dsa_pointer を使用してください。
