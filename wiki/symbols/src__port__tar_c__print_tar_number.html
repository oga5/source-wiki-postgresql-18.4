print_tar_number

定義

  • 種別: function
  • 定義箇所: src/port/tar.c:21

目的

tar ヘッダー内の固定長フィールドに対して、数値を適切な形式（8進数または GNU 拡張 base-256）でエンコードし、指定されたバッファに書き込みます。

シグネチャまたは形状

void print_tar_number(char *s, int len, uint64 val)

入力と出力

  • s: 書き込み先のバッファ（char *）。
  • len: バッファの長さ（int）。
  • val: エンコード対象の数値（uint64）。
  • 出力: なし（void）。バッファ s を直接更新します。

前提条件と不変条件

  • len は書き込み先のバッファサイズと一致している必要があります。
  • 8進数形式で表現可能な値（val < 8^(len-1)）の場合、末尾にスペースを付与した8進数文字列として書き込まれます。
  • 8進数で表現できない大きな値の場合、先頭バイトに \200 を付与した GNU 拡張 base-256 形式で書き込まれます。

アルゴリズムまたは振る舞い

1. val が 8 進数表現の範囲内か判定します。 2. 範囲内の場合:

  • バッファの末尾にスペース ' ' を設定します。
  • val を 3 ビットずつ右シフトし、下位 3 ビットを文字 '0' から '7' に変換してバッファの末尾から前方へ順次格納します。

    • 3. 範囲外の場合:

  • バッファの先頭 s[0] に \200 を設定します。
  • val を 8 ビットずつ右シフトし、下位 8 ビットをバッファの末尾から前方へ順次格納します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_basebackup/walmethods.c
  • src/port/tar.c 内の tarCreateHeader 等
  • 呼び出し先:
  • 特になし（標準的な算術演算とメモリ操作のみ）。

関連データ構造

  • pgtar.h: tar ヘッダーのオフセット定数（TAR_OFFSET_SIZE 等）が定義されています。

失敗モードと境界条件

  • len が小さすぎて値が収まらない場合、base-256 形式へのフォールバックが行われますが、len が 1 以下の場合の挙動についてはソースコード上での明示的な保護は確認できません。

保守担当者向けメモ

  • 本関数は POSIX 準拠の 8 進数表現と、GNU 拡張の base-256 表現の両方をサポートしています。
  • チェックサムフィールドの計算時には、この関数で書き込まれる値が「8 つのスペース」として扱われる必要がある点に注意してください。
