print_pool

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_misc.c:56

目的

遺伝的アルゴリズム（GEQO）の個体群（Pool）に含まれる個体データを、指定されたファイルストリームへデバッグ出力します。

シグネチャまたは形状

void print_pool(FILE *fp, Pool *pool, int start, int stop)

入力と出力

  • fp: 出力先のファイルストリーム。
  • pool: 出力対象の Pool 構造体へのポインタ。
  • start: 出力を開始する個体のインデックス。
  • stop: 出力を終了する個体のインデックス。
  • 戻り値: なし。

前提条件と不変条件

  • GEQO_DEBUG マクロが定義されている環境でのみ有効です。
  • pool は有効な Pool 構造体を指している必要があります。
  • 出力後、fflush(fp) が呼び出され、バッファがフラッシュされます。

アルゴリズムまたは振る舞い

1. 入力された start および stop の値が pool->size の範囲内に収まるよう補正します。

  • start < 0 の場合は 0 に設定。
  • stop > pool->size の場合は pool->size に設定。
  • start + stop > pool->size の場合は、全範囲（0 から pool->size）を出力対象とします。

    • 2. 指定された範囲の各個体について、インデックス、遺伝子配列（string）、および適応度（worth）を fp に書き出します。 3. 最後に fflush(fp) を実行し、出力を確定させます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/geqo/geqo_main.c:257 にて呼び出しを確認。
  • 呼び出し先: 標準ライブラリの fprintf および fflush を使用。

関連データ構造

  • Pool: 個体群を管理する構造体。data（個体配列）、size（個体数）、string_length（遺伝子長）を保持。

失敗モードと境界条件

  • start や stop に不適切な値が渡された場合、内部の補正ロジックにより安全な範囲に調整されます。
  • pool が NULL の場合、デリファレンスによりクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数はデバッグ専用です。本番環境のパフォーマンスへの影響を避けるため、必ず GEQO_DEBUG マクロによる条件付きコンパイル下で利用してください。
