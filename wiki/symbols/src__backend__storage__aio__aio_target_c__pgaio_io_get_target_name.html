pgaio_io_get_target_name

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_target.c:49

目的

指定された PgAioHandle に関連付けられたI/Oターゲットの名称（文字列）を取得します。主にデバッグメッセージやログ出力での利用を想定しています。

シグネチャまたは形状

const char *pgaio_io_get_target_name(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh （I/Oハンドルのポインタ）
  • **出力**: const char * （ターゲット名を表す静的文字列へのポインタ）

前提条件と不変条件

  • ioh->target が PGAIO_TID_INVALID 以上かつ PGAIO_TID_COUNT 未満であることを保証するアサーションが存在します。
  • INVALID 状態のターゲットも明示的に許可されています。

アルゴリズムまたは振る舞い

1. ioh->target の値が有効範囲内にあるか検証します。 2. pgaio_target_info 配列から、ioh->target に対応するターゲット情報を参照します。 3. 該当するターゲット構造体内の name フィールドを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio_funcs.c:197
  • **呼び出し先**:
  • 直接的な関数呼び出しはありませんが、グローバル配列 pgaio_target_info を参照します。

関連データ構造

  • PgAioHandle: I/O操作の状態を保持する構造体。
  • PgAioTargetInfo: ターゲットのメタデータ（名称等）を保持する内部構造体。

失敗モードと境界条件

  • ioh->target が範囲外の場合、アサーション違反によりプロセスが停止する可能性があります。
  • ioh 自体が NULL の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数はデバッグ用途を主眼としているため、ターゲットが INVALID であっても安全に処理できるよう設計されています。
  • 新しいI/Oターゲットを追加した際は、pgaio_target_info 配列への登録が正しく行われていることを確認してください。
