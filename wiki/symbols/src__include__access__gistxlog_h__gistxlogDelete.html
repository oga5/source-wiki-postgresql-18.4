gistxlogDelete

定義

  • 種別: struct
  • 定義箇所: src/include/access/gistxlog.h:50

目的

GiSTインデックスにおけるページ内のタプル削除操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct gistxlogDelete
{
    TransactionId snapshotConflictHorizon;
    uint16        ntodelete;
    bool          isCatalogRel;
    OffsetNumber  offsets[FLEXIBLE_ARRAY_MEMBER];
} gistxlogDelete;

入力と出力

  • **入力**: 削除対象のオフセット番号（offsets）および関連するトランザクション情報。
  • **出力**: WALレコードの一部としてシリアライズされ、リカバリ時にGiSTインデックスの整合性を再構築するために使用されます。

前提条件と不変条件

  • offsets は FLEXIBLE_ARRAY_MEMBER として定義されており、構造体の末尾に可変長で配置されます。
  • ntodelete は offsets 配列に格納された削除対象オフセットの数を保持します。

アルゴリズムまたは振る舞い

1. GiSTインデックスでタプル削除が発生した際、この構造体に削除対象のオフセット情報とトランザクションの競合情報を格納します。 2. XLogInsert を介してWALレコードとして書き込まれます。 3. リカバリ時には、gist_redo 関数がこの構造体を読み取り、指定されたオフセットのタプルをページから削除することでインデックスの状態を復元します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistxlog.c 内のGiST WAL記録処理。
  • **呼び出し先**: src/backend/access/rmgrdesc/gistdesc.c の out_gistxlogDelete 関数が、デバッグ出力のために本構造体を参照します。

関連データ構造

  • OffsetNumber: ページ内のタプル位置を特定する型。
  • TransactionId: トランザクション識別子。
  • SizeOfGistxlogDelete: offsetof(gistxlogDelete, offsets) として定義される、可変長配列を除いた構造体のサイズ。

失敗モードと境界条件

  • isCatalogRel が true の場合、スタンバイサーバーでの論理デコーディング中にリカバリ競合が発生する可能性があるため、特別なハンドリングが必要です。
  • 可変長配列 offsets を扱うため、メモリレイアウトの計算には SizeOfGistxlogDelete マクロを使用する必要があります。

保守担当者向けメモ

  • 本構造体のメンバを変更した場合は、必ず src/backend/access/gist/gistxlog.c のリプレイロジックおよび src/backend/access/rmgrdesc/gistdesc.c のデバッグ出力関数を更新してください。
