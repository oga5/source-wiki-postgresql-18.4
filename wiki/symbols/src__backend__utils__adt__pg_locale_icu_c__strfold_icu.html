strfold_icu

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_icu.c:444

目的

ICUライブラリを使用して、指定されたロケールに基づき文字列のケースフォールディング（大文字・小文字の区別をなくすための正規化）を行う。

シグネチャまたは形状

size_t strfold_icu(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: 結果を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換対象の文字列。
  • srclen: src の長さ（バイト単位）。
  • locale: ICUロケール情報を含む pg_locale_t 構造体。
  • **出力**:
  • 変換後の文字列長（バイト単位）を size_t で返す。

前提条件と不変条件

  • icu_to_uchar および icu_convert_case が内部でメモリを割り当てるため、呼び出し元は本関数が終了するまでにこれらのメモリが解放されることを期待できる。
  • locale は有効な ICU ロケール情報を含んでいる必要がある。

アルゴリズムまたは振る舞い

1. icu_to_uchar を呼び出し、データベースエンコーディングの文字列を ICU 内部表現（UChar）に変換する。 2. icu_convert_case を呼び出し、u_strFoldCase_default を用いてケースフォールディングを実行する。 3. icu_from_uchar を呼び出し、変換後の UChar 文字列をデータベースエンコーディングに戻して dest に格納する。 4. 変換過程で確保した buff_uchar および buff_conv を pfree で解放する。 5. 最終的な結果の長さを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c:1335 (確認済み)
  • **呼び出し先**:
  • icu_to_uchar
  • icu_convert_case
  • icu_from_uchar
  • pfree

関連データ構造

  • pg_locale_t: ロケール設定を保持する構造体。
  • UChar: ICUで使用されるUTF-16文字列型。

失敗モードと境界条件

  • ICUの変換処理でエラーが発生した場合、内部で ereport(ERROR, ...) が呼び出され、PostgreSQLのトランザクションメカニズムにより処理が中断される。

保守担当者向けメモ

  • 本関数は palloc を使用して一時バッファを確保するため、メモリ不足時にはPostgreSQLのメモリ管理機構によりエラーがスローされる。
  • ICUのバージョンやロケール設定により、ケースフォールディングの挙動が異なる可能性がある点に注意すること。
