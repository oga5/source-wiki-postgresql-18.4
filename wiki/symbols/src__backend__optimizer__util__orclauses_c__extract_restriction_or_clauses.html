extract_restriction_or_clauses

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/orclauses.c:74

目的

結合条件（Join Clause）に含まれるOR条件を解析し、特定のベースリレーション（Base Relation）に適用可能な制限条件（Restriction Clause）を抽出・生成します。これにより、結合前のスキャン段階で不要な行を排除し、プランの効率を向上させます。

シグネチャまたは形状

void extract_restriction_or_clauses(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（クエリ全体のプランナ情報）。
  • **出力**: なし（副作用として、対象リレーションの baserestrictinfo リストを更新します）。

前提条件と不変条件

  • root->simple_rel_array が初期化されていること。
  • 対象リレーションが RELOPT_BASEREL であること。
  • 抽出対象のOR条件は、join_clause_is_movable_to により対象リレーションへ移動可能と判定されていること。

アルゴリズムまたは振る舞い

1. root->simple_rel_array 内の全ベースリレーションを走査します。 2. 各リレーションの joininfo リストを反復し、restriction_is_or_clause でOR条件を特定します。 3. extract_or_clause を呼び出し、OR条件の各アームから対象リレーションのみを参照するサブ条件を抽出します。 4. 抽出に成功した場合、consider_new_or_clause を呼び出し、条件の妥当性を評価した上で baserestrictinfo への追加および選択率の補正を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planmain.c:266
  • **呼び出し先**:
  • restriction_is_or_clause
  • join_clause_is_movable_to
  • extract_or_clause
  • consider_new_or_clause

関連データ構造

  • PlannerInfo: クエリ全体の最適化情報を保持。
  • RelOptInfo: ベースリレーションの情報を保持。
  • RestrictInfo: 制限条件および結合条件のメタデータ構造。

失敗モードと境界条件

  • rel == NULL: 非ベースリレーションのRTE（Range Table Entry）はスキップされます。
  • reloptkind != RELOPT_BASEREL: ベースリレーション以外のRTEは無視されます。
  • extract_or_clause が NULL を返した場合: 抽出可能な条件がないと判断され、追加処理は行われません。

保守担当者向けメモ

  • 本関数で行われる選択率の補正は「Major Hack」と呼ばれ、冗長な条件追加による見積もりの歪みを防ぐためのものです。外部結合（Outer Join）などの複雑なケースでは完全ではない可能性があるため、修正時は注意が必要です。
