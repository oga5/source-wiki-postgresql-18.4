rewind_parseTimeLineHistory

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/timeline.c:27

目的

PostgreSQLのタイムライン履歴ファイル（.history）の内容を解析し、TimeLineHistoryEntry 構造体の配列としてメモリ上に構築します。

シグネチャまたは形状

TimeLineHistoryEntry *rewind_parseTimeLineHistory(char *buffer, TimeLineID targetTLI, int *nentries);

入力と出力

  • **入力**:
  • buffer: 履歴ファイルの内容を保持する文字列バッファ。
  • targetTLI: 対象となるタイムラインID。
  • nentries: 解析されたエントリ数を格納するためのポインタ（出力用）。
  • **出力**:
  • 構築された TimeLineHistoryEntry 構造体の配列へのポインタ。

前提条件と不変条件

  • 履歴ファイル内のタイムラインID（TLI）は昇順である必要があります。
  • 履歴ファイルに記述がない「現在のタイムラインの先端（tip）」が、配列の末尾に自動的に追加されます。
  • 解析エラーが発生した場合、pg_log_error を通じて詳細を出力し、exit(1) でプロセスを終了します。

アルゴリズムまたは振る舞い

1. バッファを1行ずつ走査し、sscanf でTLIとWAL切り替えポイント（LSN）を抽出します。 2. 空行やコメント行（#）をスキップします。 3. 抽出したTLIが昇順であることを検証します。 4. pg_realloc を用いて配列を動的に拡張し、各エントリの開始・終了位置を格納します。 5. 最後に、履歴ファイルに記述されていない「現在のタイムライン」用のエントリを末尾に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c:886
  • **呼び出し先**: pg_realloc, pg_malloc, pg_log_error, pg_log_error_detail（いずれも pg_rewind ユーティリティ関数）。

関連データ構造

  • TimeLineHistoryEntry: 各タイムラインの有効範囲（begin から end）を保持する構造体（src/include/access/timeline.h で定義）。

失敗モードと境界条件

  • **構文エラー**: 履歴ファイルの形式が不正な場合（数値以外のTLI、LSN形式の不一致など）、エラーを出力して終了します。
  • **順序エラー**: TLIが昇順でない場合、エラーを出力して終了します。
  • **ターゲットTLIの妥当性**: targetTLI が履歴内の最終TLI以下である場合、エラーを出力して終了します。

保守担当者向けメモ

  • 本関数はバックエンドの readTimeLineHistory をベースにしていますが、メモリ管理がフロントエンド向け（malloc/realloc）に調整されています。
  • 履歴ファイルの内容が正しいことを前提として処理が進むよう設計されています。
