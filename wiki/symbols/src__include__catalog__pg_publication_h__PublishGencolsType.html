PublishGencolsType

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/pg_publication.h:115

目的

論理レプリケーションにおいて、生成列（Generated Columns）をパブリケーションに含めるかどうかを制御するための列挙型です。

シグネチャまたは形状

typedef enum PublishGencolsType
{
    PUBLISH_GENCOLS_NONE = 'n',
    PUBLISH_GENCOLS_STORED = 's',
} PublishGencolsType;

入力と出力

  • 入力: 本列挙型の値は、パブリケーションの構成設定として利用されます。
  • 出力: レプリケーションプロトコルや列マッピング処理において、生成列の扱いを決定するフラグとして機能します。

前提条件と不変条件

  • PUBLISH_GENCOLS_NONE ('n'): 生成列をレプリケーション対象から除外します。
  • PUBLISH_GENCOLS_STORED ('s'): 生成列をレプリケーション対象に含めます。

アルゴリズムまたは振る舞い

本シンボルは、論理レプリケーションのデータ送信処理において、対象テーブルの列リストを構築する際のフィルタリング条件として参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • src/backend/catalog/pg_publication.c: pub_form_cols_map 関数にて列マッピングの制御に使用。
  • src/backend/replication/logical/proto.c: プロトコル定義およびデータ送信ロジックにて引数として使用。
  • src/backend/replication/pgoutput/pgoutput.c: pgoutput プラグインのレプリケーション設定として使用。
  • **呼び出し先**:
  • 本シンボルは列挙型であり、関数を呼び出す側ではありません。

関連データ構造

  • PublicationRelInfo: パブリケーションとリレーションの関連付けを保持する構造体であり、本列挙型の値を保持する可能性があります。

失敗モードと境界条件

  • 未定義の文字コードや範囲外の値が代入された場合の挙動は、各参照先の実装に依存します。

保守担当者向けメモ

  • 本シンボルは EXPOSE_TO_CLIENT_CODE ブロック内に定義されており、クライアント側でも利用可能です。
  • レプリケーションプロトコルの変更時には、src/backend/replication/logical/proto.c 内の各関数シグネチャとの整合性に注意してください。
