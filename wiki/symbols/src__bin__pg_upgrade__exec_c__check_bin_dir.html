check_bin_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/exec.c:382

目的

PostgreSQLのアップグレードに必要なバイナリが指定されたディレクトリ内に存在し、かつ実行可能であるかを検証する。

シグネチャまたは形状

static void check_bin_dir(ClusterInfo *cluster, bool check_versions)

入力と出力

  • **入力**:
  • cluster: 検証対象のクラスタ情報（ClusterInfo 構造体）。bindir メンバにパスが格納されている必要がある。
  • check_versions: バージョンチェックを実行するか否かを示すフラグ。
  • **出力**: なし（検証失敗時は report_status(PG_FATAL, ...) を通じてプロセスを終了する）。

前提条件と不変条件

  • cluster->bindir が指すパスがディレクトリとして存在すること。
  • get_bin_version を呼び出す前に pg_ctl の存在確認が完了していること。

アルゴリズムまたは振る舞い

1. stat を使用して cluster->bindir がディレクトリであることを確認する。 2. 必須バイナリ（postgres, pg_controldata, pg_ctl）に対して check_exec を実行し、存在とバージョンを検証する。 3. get_bin_version を呼び出し、クラスタのバージョン情報を取得する。 4. バージョンに基づき、pg_resetxlog（9.6以下）または pg_resetwal（10以上）のいずれかを検証する。 5. 対象が new_cluster である場合のみ、追加のバイナリ（initdb, pg_dump, pg_dumpall, pg_restore, psql, vacuumdb）の検証を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: verify_directories（src/bin/pg_upgrade/exec.c 内で old_cluster および new_cluster に対して呼び出される）。
  • **呼び出し先**:
  • stat (システムコール)
  • report_status
  • check_exec
  • get_bin_version

関連データ構造

  • ClusterInfo: クラスタのパスやバージョン情報を保持する構造体。

失敗モードと境界条件

  • bindir が存在しない、またはディレクトリでない場合に致命的エラー（PG_FATAL）となる。
  • check_exec が失敗した場合、指定されたバイナリが欠落またはバージョン不整合とみなされ、pg_upgrade は停止する。

保守担当者向けメモ

  • pg_resetxlog から pg_resetwal への名称変更に伴う条件分岐が含まれている。
  • new_cluster に対してのみ実行されるバイナリ群は、アップグレード後の環境構築に必須であるため、追加・削除の際は pg_upgrade の要件と照らし合わせること。
