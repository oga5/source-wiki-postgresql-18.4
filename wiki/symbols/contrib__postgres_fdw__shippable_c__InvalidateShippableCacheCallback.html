InvalidateShippableCacheCallback

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/shippable.c:64

目的

pg_foreign_server 等のシステムカタログが更新された際に、ShippableCacheHash に保持されているキャッシュエントリをすべて破棄し、整合性を維持する。

シグネチャまたは形状

static void InvalidateShippableCacheCallback(Datum arg, int cacheid, uint32 hashvalue)

入力と出力

  • arg: コールバック登録時に渡されたデータ（使用されていない）。
  • cacheid: 無効化のトリガーとなったキャッシュID。
  • hashvalue: 無効化のトリガーとなったハッシュ値。
  • 戻り値: なし。

前提条件と不変条件

  • ShippableCacheHash が初期化済みであること。
  • キャッシュの破棄は全エントリに対して行われる。特定のサーバーに関連するエントリのみを個別にフラッシュするロジックは実装されていない。

アルゴリズムまたは振る舞い

1. hash_seq_init を使用して ShippableCacheHash のシーケンシャルスキャンを開始する。 2. hash_seq_search でエントリを順次取得する。 3. 取得した各エントリに対して hash_search を HASH_REMOVE モードで呼び出し、キャッシュから削除する。 4. 削除中にハッシュテーブルの破損が検出された場合、elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL のシステムキャッシュ無効化メカニズムから呼び出される（contrib/postgres_fdw/shippable.c:103 で登録）。
  • 呼び出し先: hash_seq_init, hash_seq_search, hash_search, elog。

関連データ構造

  • ShippableCacheHash: 無効化対象となるハッシュテーブル。
  • ShippableCacheEntry: キャッシュ内の各エントリ構造体。

失敗モードと境界条件

  • キャッシュが空の場合、ループは実行されず即座に終了する。
  • hash_search が NULL を返した場合、ハッシュテーブルの破損とみなされエラー終了する。

保守担当者向けメモ

  • 現在の実装では、サーバー設定の変更時にキャッシュを全破棄する保守的なアプローチをとっている。個別のエントリのみを削除する最適化は可能だが、複雑さが増すため見送られている。
