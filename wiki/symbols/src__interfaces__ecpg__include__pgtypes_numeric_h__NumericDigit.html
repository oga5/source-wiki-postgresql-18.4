NumericDigit

定義

  • 種別: typedef
  • 定義箇所: src/interfaces/ecpg/include/pgtypes_numeric.h:17

目的

ECPG（Embedded SQL for C）において、NUMERIC型の数値を構成する個々の桁（基数ベースの数値単位）を表現するために使用されます。

シグネチャまたは形状

typedef unsigned char NumericDigit;

入力と出力

  • 入力: 符号なしの8ビット整数値。
  • 出力: 数値構造体（numericまたはdecimal）内の配列要素として格納されます。

前提条件と不変条件

  • unsigned char型として定義されており、負の値は保持しません。
  • 数値の各桁は、特定の基数（NBASE）に基づいた範囲内の値をとることが想定されています。

アルゴリズムまたは振る舞い

  • 数値構造体内部で、高精度数値を表現するための配列の要素型として機能します。
  • 算術演算や型変換の際、この型を介して数値の各桁が処理されます。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数関係は、本ヘッダーファイル単体では定義されていません。
  • ツールによるヒューリスティックな調査では、PostgreSQLバックエンドの src/backend/utils/adt/numeric.c において、同様の名称の型が数値データの保持に使用されていることが示唆されていますが、ECPGインターフェースにおける直接的な呼び出し関係は本ソースからは特定できません。

関連データ構造

  • numeric: 動的メモリ確保を行う数値構造体。
  • decimal: 固定長配列を持つ数値構造体。
  • pgtypes_numeric.h 内で定義されるこれら構造体のメンバとして利用されます。

失敗モードと境界条件

  • unsigned char の範囲（0〜255）を超える値は格納できません。
  • 構造体内の配列サイズ（decimalの場合はDECSIZE）を超えたアクセスは、メモリ破壊や未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本定義はECPGインターフェース用です。バックエンド側の numeric.c で定義されている NumericDigit（signed char や int16 等のバリエーションが存在）と混同しないよう注意してください。
  • 数値の精度や基数に関する変更を行う際は、pgtypes_numeric.h 内の関連する定数定義と整合性をとる必要があります。
