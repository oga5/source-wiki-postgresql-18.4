tlist_same_datatypes

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/tlist.c:256

目的

ターゲットリスト（tlist）内の各エントリのデータ型が、指定された型リスト（colTypes）と一致するかを検証する。

シグネチャまたは形状

bool tlist_same_datatypes(List *tlist, List *colTypes, bool junkOK)

入力と出力

  • **入力**:
  • tlist: TargetEntry ノードのリスト。
  • colTypes: 比較対象となる OID（データ型）のリスト。
  • junkOK: resjunk（ジャンク列）を許容するかどうかを示すフラグ。
  • **出力**:
  • bool: 全ての非ジャンク列の型が colTypes と順序通りに一致し、かつリストの長さが整合していれば true、そうでなければ false。

前提条件と不変条件

  • tlist 内の各 TargetEntry は有効な式（expr）を保持している必要がある。
  • colTypes は Oid のリストであること。

アルゴリズムまたは振る舞い

1. tlist を走査し、各 TargetEntry を確認する。 2. resjunk が true の場合:

  • junkOK が false ならば直ちに false を返す。
  • junkOK が true ならば無視して次のエントリへ進む。

    • 3. resjunk が false の場合:

  • colTypes の現在位置が空であれば、tlist が長いため false を返す。
  • exprType() を用いて式の型を取得し、colTypes の現在の OID と比較する。一致しなければ false を返す。

    • 4. 全走査終了後、colTypes に未処理の型が残っていれば（tlist が短い）、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepjointree.c:2277
  • src/backend/optimizer/prep/prepunion.c:296
  • **呼び出し先**:
  • exprType()（式ノードから型 OID を取得）
  • list_head(), lfirst(), lnext(), lfirst_oid()（リスト操作）

関連データ構造

  • TargetEntry: ターゲットリストの各要素を構成する構造体。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • tlist と colTypes の長さが不一致の場合、false を返す。
  • junkOK が false の状態で resjunk なエントリに遭遇した場合、false を返す。
  • 空の tlist と空の colTypes を渡した場合は true を返す。

保守担当者向けメモ

  • 本関数は主にプランナの準備段階（prep）において、サブクエリや UNION 操作の整合性チェックに使用される。
  • exprType() は式ノードの型を解決する標準的な関数であり、本関数の判定ロジックの根幹をなす。
