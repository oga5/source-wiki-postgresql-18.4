index_pages_fetched

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/costsize.c:907

目的

インデックススキャンにおいて、特定のタプル数（tuples_fetched）を取得するために読み込む必要のあるテーブルページ数を推定します。

シグネチャまたは形状

double index_pages_fetched(double tuples_fetched, BlockNumber pages, double index_pages, PlannerInfo *root);

入力と出力

  • **入力**:
  • tuples_fetched: 取得対象のタプル数。
  • pages: 対象テーブルの総ページ数。
  • index_pages: インデックスの総ページ数。
  • root: プランナのグローバル情報（effective_cache_size 等の参照に使用）。
  • **出力**:
  • 推定された読み込みページ数（double 型）。

前提条件と不変条件

  • pages が 0 の場合、内部的に 1.0 として扱われます。
  • total_pages（テーブルとインデックスの合計）は 1.0 以上であることが保証されます。
  • T（テーブルページ数）は total_pages 以下である必要があります（Assert による検証）。

アルゴリズムまたは振る舞い

Mackert & Lohman の公式に基づき、キャッシュ効率を考慮したページ読み込み数を算出します。 1. **キャッシュ共有の計算**: effective_cache_size を基に、テーブルが占有するキャッシュ領域 b を算出します。 2. **ページ数推定**:

  • T <= b の場合: キャッシュヒット率が高いとみなし、単純なモデルで算出します。
  • T > b の場合: lim（キャッシュ限界点）を境界として、取得タプル数に応じた線形補間または非線形モデルを適用します。

    • 3. **結果の調整**: 算出されたページ数は ceil によって整数値（double）に切り上げられます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/costsize.c 内の複数箇所（cost_index 等）。
  • src/backend/utils/adt/selfuncs.c 内の選択率計算処理。
  • **呼び出し先**:
  • ceil（標準数学ライブラリ）
  • Max（マクロ）
  • Assert（マクロ）

関連データ構造

  • PlannerInfo: プランナのコンテキスト情報。effective_cache_size や total_table_pages を保持。

失敗モードと境界条件

  • tuples_fetched が 0 の場合、結果は 0 になります。
  • pages が 1 以下の場合、T は 1.0 にクランプされます。
  • b が 1.0 未満の場合、1.0 に強制されます。
  • pages_fetched が T を超える場合は T に制限されます。

保守担当者向けメモ

  • 本関数は Mackert & Lohman のモデルを実装しており、統計情報に基づくコスト見積もりの精度に直結します。
  • effective_cache_size の値に大きく依存するため、システム設定変更時の影響範囲に含まれます。
