XLogRecordMatchesRelationBlock

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/pg_waldump.c:437

目的

指定された WAL レコード内に、特定のリレーション、フォーク、およびブロック番号に合致するブロックタグが含まれているかを判定する。

シグネチャまたは形状

static bool XLogRecordMatchesRelationBlock(XLogReaderState *record, RelFileLocator matchRlocator, BlockNumber matchBlock, ForkNumber matchFork)

入力と出力

  • **入力**:
  • record: 検査対象の XLogReaderState 構造体。
  • matchRlocator: 検索対象の RelFileLocator。emptyRelFileLocator の場合はリレーションの指定を無視する。
  • matchBlock: 検索対象の BlockNumber。InvalidBlockNumber の場合はブロック番号の指定を無視する。
  • matchFork: 検索対象の ForkNumber。InvalidForkNumber の場合はフォークの指定を無視する。
  • **出力**:
  • bool: 条件に合致するブロックがレコード内に存在すれば true、そうでなければ false を返す。

前提条件と不変条件

  • record は XLogReaderState として有効に初期化されている必要がある。
  • XLogRecMaxBlockId(record) を用いてレコード内の全ブロックを走査する。

アルゴリズムまたは振る舞い

1. レコードに含まれるブロック ID を 0 から XLogRecMaxBlockId(record) まで順次走査する。 2. XLogRecGetBlockTagExtended を呼び出し、各ブロックの rlocator、forknum、blk を取得する。取得に失敗した場合は次のブロックへ進む。 3. 以下の条件をすべて満たすブロックが見つかった時点で true を返す。

  • matchFork が InvalidForkNumber であるか、または forknum と一致する。
  • matchRlocator が emptyRelFileLocator であるか、または rlocator と一致する。
  • matchBlock が InvalidBlockNumber であるか、または blk と一致する。

    • 4. 全ブロックを走査しても条件を満たすものがなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_waldump/pg_waldump.c:438 (定義自体)
  • src/bin/pg_waldump/pg_waldump.c:1269
  • **呼び出し先**:

関連データ構造

  • XLogReaderState: WAL レコードの読み込み状態を保持する構造体。
  • RelFileLocator: リレーションを識別するための構造体。
  • ForkNumber, BlockNumber: PostgreSQL のストレージ層で使用される型。

失敗モードと境界条件

  • レコード内にブロック情報が含まれない場合、ループは実行されず false を返す。
  • XLogRecGetBlockTagExtended が失敗した場合、そのブロックはスキップされる。

保守担当者向けメモ

  • 本関数は pg_waldump におけるフィルタリング処理の一部として使用される。
  • RelFileLocatorEquals を使用してリレーションの比較を行っているため、リレーション定義の変更時には注意が必要である。
