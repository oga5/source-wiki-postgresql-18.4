output_set_descr

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:268

目的

SQLの SET DESCRIPTOR 文を、ECPGランタイムライブラリが実行可能なCコード（ECPGset_desc 呼び出し）へ変換して出力します。

シグネチャまたは形状

void output_set_descr(const char *desc_name, const char *index)

入力と出力

  • **入力**:
  • desc_name: 記述子名。
  • index: 記述子のインデックス（文字列）。
  • グローバル変数 assignments: 記述子に対して設定する項目リスト。
  • **出力**:
  • base_yyout: 生成されたCコードが書き込まれる出力ストリーム。

前提条件と不変条件

  • assignments リストには、設定対象の記述子項目が格納されている必要があります。
  • 関数終了時に drop_assignments() が呼び出され、assignments リストが解放されることが保証されています。

アルゴリズムまたは振る舞い

1. ECPGset_desc の開始コードを出力します。 2. グローバルな assignments リストを走査し、各項目を処理します。

  • ECPGd_cardinality 等の特定の項目は mmfatal によりエラー終了します。
  • ECPGd_key_name 等の読み取り専用項目は mmfatal によりエラー終了します。
  • ECPGd_data, ECPGd_indicator, ECPGd_length, ECPGd_type の場合は、ECPGdump_a_type を用いて型情報を出力します。

    • 3. drop_assignments() を呼び出してメモリをクリーンアップします。 4. ECPGd_EODT を出力して関数呼び出しを閉じ、whenever_action を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述子解析ロジック（具体的な呼び出し元関数名はソース抜粋からは特定できません）。
  • **呼び出し先**:
  • find_variable: 変数情報の取得。
  • descriptor_item_name: 項目名の取得。
  • get_dtype: データ型文字列の取得。
  • ECPGdump_a_type: 型情報の出力。
  • drop_assignments: リストの解放。
  • whenever_action: エラーハンドリングの出力。
  • mmfatal: エラー報告。

関連データ構造

  • struct assignment: 記述子への代入操作を保持するリスト構造。
  • struct variable: 変数情報を保持する構造体。

失敗モードと境界条件

保守担当者向けメモ

  • 本関数は assignments リストを消費する設計です。リストの解放漏れを防ぐため、drop_assignments の呼び出し位置を変更する際は注意してください。
  • 記述子項目ごとの詳細な処理ロジックは switch 文で管理されており、新しい項目を追加する際はここを拡張する必要があります。
