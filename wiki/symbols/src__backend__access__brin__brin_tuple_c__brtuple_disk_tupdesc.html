brtuple_disk_tupdesc

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_tuple.c:60

目的

BRINインデックスのオンディスク形式タプルを構築・操作するために必要な TupleDesc を提供します。BrinDesc 内にキャッシュを保持し、初回呼び出し時に生成することで効率化を図ります。

シグネチャまたは形状

TupleDesc brtuple_disk_tupdesc(BrinDesc *brdesc)

入力と出力

  • **入力**: BrinDesc *brdesc（BRINインデックスの記述子）
  • **出力**: TupleDesc（オンディスク形式のタプル記述子）

前提条件と不変条件

  • brdesc->bd_context が有効なメモリコンテキストである必要があります。
  • brdesc->bd_disktdesc が NULL の場合、brdesc->bd_context に切り替えて新しい TupleDesc を割り当てます。

アルゴリズムまたは振る舞い

1. brdesc->bd_disktdesc が既に存在するか確認し、存在すればそれを返します。 2. 存在しない場合、brdesc->bd_context へメモリコンテキストを切り替えます。 3. brdesc->bd_totalstored を基に CreateTemplateTupleDesc で新しい記述子を作成します。 4. brdesc->bd_tupdesc の各属性を走査し、各属性が保持する oi_typcache の型情報を基に TupleDescInitEntry を呼び出して記述子を構築します。 5. 生成した記述子を brdesc->bd_disktdesc にキャッシュし、元のメモリコンテキストに戻して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_tuple.c: brin_form_tuple（データサイズ計算およびタプル構築時）
  • src/backend/access/brin/brin_tuple.c: brin_deconstruct_tuple（ディスク上のタプル解析時）
  • **呼び出し先**:
  • CreateTemplateTupleDesc
  • TupleDescInitEntry
  • MemoryContextSwitchTo

関連データ構造

  • BrinDesc: インデックスのメタデータとキャッシュを保持する構造体。
  • TupleDesc: PostgreSQLのタプル記述子。

失敗モードと境界条件

  • brdesc が NULL の場合、または bd_context が無効な場合はメモリ保護違反が発生する可能性があります。
  • メモリ不足時には CreateTemplateTupleDesc がエラーをスローします。

保守担当者向けメモ

  • 本関数は BrinDesc のキャッシュを利用するため、BrinDesc のライフサイクルと整合性が取れている必要があります。
  • TupleDescInitEntry に渡される型情報は brdesc->bd_info から取得されるため、インデックスの定義変更時にはこのキャッシュが適切にクリアまたは更新されることを確認してください。
