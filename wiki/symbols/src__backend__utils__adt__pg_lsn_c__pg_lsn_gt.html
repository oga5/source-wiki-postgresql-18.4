pg_lsn_gt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:144

目的

2つの pg_lsn 型の値を比較し、第1引数が第2引数より大きいかどうかを判定する。

シグネチャまたは形状

Datum pg_lsn_gt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの pg_lsn 値を受け取る。
  • **出力**: bool 型（Datum 形式）。第1引数が第2引数より大きい場合は true、それ以外は false を返す。

前提条件と不変条件

  • 入力値は XLogRecPtr（64ビット整数）として扱われる。
  • PG_GETARG_LSN マクロにより、入力された Datum が XLogRecPtr に正しく変換されることが前提となる。

アルゴリズムまたは振る舞い

1. PG_GETARG_LSN(0) および PG_GETARG_LSN(1) を呼び出し、比較対象の2つの XLogRecPtr 値を取得する。 2. C言語の標準的な比較演算子 > を使用して、lsn1 が lsn2 より大きいかを評価する。 3. PG_RETURN_BOOL を使用して、評価結果を Datum 型として呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL演算子（>）の実行時に、PostgreSQLの関数マネージャを介して呼び出される。
  • **呼び出し先**: PG_GETARG_LSN および PG_RETURN_BOOL マクロ（PostgreSQLの関数呼び出し規約）。

関連データ構造

  • XLogRecPtr: LSNを保持する64ビット整数型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 入力引数が不足している場合や、型が pg_lsn ではない場合は、関数マネージャレベルでエラーが発生する可能性がある。
  • 比較対象が等しい場合は false を返す。

保守担当者向けメモ

  • 本関数は単純な数値比較であるため、pg_lsn_cmp と同様のロジックを共有している。
  • 比較ロジックに変更を加える際は、他の比較関数（pg_lsn_lt, pg_lsn_eq 等）との整合性に注意すること。
