seq_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/seqdesc.c:33

目的

WAL（Write-Ahead Log）レコードの info ビットから、シーケンス操作の種類を識別する文字列を返します。主に pg_waldump 等のWAL解析ツールで、レコードタイプを人間が読み取り可能な形式で表示するために使用されます。

シグネチャまたは形状

const char *seq_identify(uint8 info);

入力と出力

  • **入力**: info (uint8型) - WALレコードのフラグ情報を含むビットマスク。
  • **出力**: const char * - 識別された操作タイプを表す文字列（例: "LOG"）。該当しない場合は NULL を返します。

前提条件と不変条件

  • 入力された info ビットマスクから XLR_INFO_MASK を除去した値が、既知のシーケンス操作タイプ（XLOG_SEQ_LOG）と一致する必要があります。

アルゴリズムまたは振る舞い

1. 引数 info に対してビット演算 info & ~XLR_INFO_MASK を行い、操作タイプを抽出します。 2. switch 文を用いて抽出した値を評価します。 3. XLOG_SEQ_LOG に合致する場合、文字列 "LOG" を返します。 4. いずれのケースにも該当しない場合、初期値である NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h 内の PG_RMGR マクロ定義を通じて、PostgreSQLのリソースマネージャ（RmgrData）から参照されています。
  • **呼び出し先**: 内部で呼び出している関数はありません。

関連データ構造

  • XLR_INFO_MASK: WALレコードのフラグ部分をマスクするための定数。
  • XLOG_SEQ_LOG: シーケンス操作を識別するための定数。

失敗モードと境界条件

  • XLOG_SEQ_LOG 以外の値が渡された場合、関数は NULL を返します。呼び出し側はこの戻り値が NULL である可能性を考慮する必要があります。

保守担当者向けメモ

  • 本関数は表示専用のロジックであり、シーケンスの更新処理には関与しません。
  • 新しいシーケンス関連のWALレコードタイプが追加された場合は、switch 文に新しい case を追加してください。
