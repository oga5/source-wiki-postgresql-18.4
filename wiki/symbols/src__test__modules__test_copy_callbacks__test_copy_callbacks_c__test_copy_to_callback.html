test_copy_to_callback

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_copy_callbacks/test_copy_callbacks.c:33

目的

PostgreSQLの COPY TO コマンドにおけるカスタム出力先コールバック機構の動作を検証するためのSQLエントリポイント関数。

シグネチャまたは形状

Datum test_copy_to_callback(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_GETARG_OID(0) を介して対象テーブルのOIDを受け取る。
  • 出力: PG_RETURN_VOID() により値を返さない。処理行数を NOTICE レベルのログとして出力する。

前提条件と不変条件

  • 引数として渡されたOIDに対応するテーブルが存在し、AccessShareLock を取得可能であること。
  • BeginCopyTo に渡されるコールバック関数 to_cb が適切に定義されていること。

アルゴリズムまたは振る舞い

1. 指定されたテーブルを AccessShareLock でオープンする。 2. BeginCopyTo を呼び出し、カスタムコールバック to_cb を登録して CopyToState を初期化する。 3. DoCopyTo を呼び出し、テーブルスキャンとコールバック経由のデータ処理を実行する。 4. EndCopyTo を呼び出し、リソースを解放する。 5. 処理された行数を ereport で通知する。 6. テーブルをクローズし、処理を終了する。

呼び出し元と呼び出し先

  • 呼び出し元: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先: table_open, BeginCopyTo, DoCopyTo, EndCopyTo, RelationGetRelid, table_close, ereport。

関連データ構造

  • Relation: 対象テーブルのメタデータ構造。
  • CopyToState: COPY TO 処理の状態を保持する構造体。

失敗モードと境界条件

  • テーブルのオープン失敗や、COPY 処理中のエラーが発生した場合、PostgreSQLの標準的なエラーハンドリングによりトランザクションがアボートされる可能性がある。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、COPY コマンドの拡張インターフェースの利用例として機能する。
  • to_cb へのデータ受け渡しは COPY の出力形式に依存するため、将来的な仕様変更時には BeginCopyTo の引数構成を確認すること。
