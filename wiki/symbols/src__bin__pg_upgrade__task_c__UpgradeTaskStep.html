UpgradeTaskStep

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_upgrade/task.c:71

目的

pg_upgrade における並列タスク実行フレームワークにおいて、各データベースに対して順次実行される個別のクエリステップと、その結果処理ロジックを定義するために使用されます。

シグネチャまたは形状

typedef struct UpgradeTaskStep
{
    UpgradeTaskProcessCB process_cb; /* クエリ結果の処理用コールバック */
    bool        free_result;         /* 結果セットを解放すべきか */
    void       *arg;                 /* コールバックへ渡す引数 */
} UpgradeTaskStep;

入力と出力

  • **入力**: UpgradeTaskProcessCB 型の関数ポインタ、結果解放フラグ、およびコールバックに渡される任意のデータポインタ。
  • **出力**: なし（構造体定義のため）。

前提条件と不変条件

  • 複数の UpgradeTaskStep が登録された場合、それらは同一のデータベース接続上で順次実行されます。
  • process_cb は、クエリ実行後に呼び出され、結果の検証や処理を行います。

アルゴリズムまたは振る舞い

  • UpgradeTask 構造体の一部として保持され、upgrade_task_run の実行ループ内で参照されます。
  • process_slot 関数が現在のステップを特定し、クエリ送信後の結果処理において process_cb を実行します。
  • free_result が true の場合、コールバック実行後に PQclear 等による結果の解放が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: upgrade_task_add_step（ステップの追加時）、process_slot（実行時）。
  • **呼び出し先**: process_cb（コールバック関数として登録された任意の関数）。

関連データ構造

  • UpgradeTask: 本構造体の配列を保持する親構造体。
  • UpgradeTaskSlot: 実行中のステップインデックス（step_idx）を管理する構造体。

失敗モードと境界条件

  • ステップの追加時にメモリ確保（realloc）が失敗する可能性があります。
  • コールバック関数内での処理失敗は、pg_upgrade 全体のタスク実行失敗として扱われます。

保守担当者向けメモ

  • 本構造体は task.c 内のステートマシンによって制御されます。
  • process_cb に渡す arg は、呼び出し側で適切なライフサイクル管理を行う必要があります。
  • 最初のステップとして ALWAYS_SECURE_SEARCH_PATH_SQL が自動的に追加される仕様があるため、ステップのインデックス管理には注意してください。
