ApplySignedSortComparator

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/sortsupport.h:266

目的

符号付き64ビット整数として扱われる2つの Datum 値を比較し、NULL 処理とソート順序（昇順/降順）を考慮した比較結果を返します。

シグネチャまたは形状

static inline int ApplySignedSortComparator(Datum datum1, bool isNull1,
                                            Datum datum2, bool isNull2,
                                            SortSupport ssup)

入力と出力

  • **入力**:
  • datum1, datum2: 比較対象の Datum 値。
  • isNull1, isNull2: 各値が NULL かどうかを示すフラグ。
  • ssup: ソート設定を保持する SortSupport 構造体。
  • **出力**:
  • int: 比較結果。datum1 < datum2 なら -1、datum1 > datum2 なら 1、等価なら 0 を返します。

前提条件と不変条件

  • ssup は有効なポインタである必要があります。
  • datum1 および datum2 は、64ビット符号付き整数として解釈可能な値である必要があります。
  • ssup->ssup_nulls_first および ssup->ssup_reverse が適切に設定されていることを前提とします。

アルゴリズムまたは振る舞い

1. **NULL 判定**: どちらか一方が NULL の場合、ssup->ssup_nulls_first に基づいて NULL を先頭または末尾に配置するよう比較結果を決定します。両方 NULL の場合は 0 を返します。 2. **値の比較**: 両方非 NULL の場合、DatumGetInt64 を使用して値を64ビット整数として比較します。 3. **順序の反転**: ssup->ssup_reverse が真の場合、比較結果を反転させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesort.c:522 での利用が確認されています。
  • **呼び出し先**: DatumGetInt64 マクロおよび INVERT_COMPARE_RESULT マクロを使用します。

関連データ構造

  • SortSupportData (およびそのポインタ型 SortSupport): ソートの実行状態を保持します。

失敗モードと境界条件

  • datum が64ビット整数として解釈できない場合、誤った比較結果を返します。
  • ssup が NULL の場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数はインライン化されており、ソートのループ内でのオーバーヘッドを最小化するように設計されています。変更を加える際は、インライン化の効率に影響を与えないよう注意してください。
