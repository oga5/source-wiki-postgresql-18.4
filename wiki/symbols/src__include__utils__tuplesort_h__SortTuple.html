SortTuple

定義

  • 種別: struct
  • 定義箇所: src/include/utils/tuplesort.h:148

目的

ソート処理におけるタプルの最小管理単位。ソート対象のデータ本体と、比較演算を高速化するためのキー情報を保持する。

シグネチャまたは形状

typedef struct {
    void       *tuple;      /* タプル本体 */
    Datum       datum1;     /* 第1キー列の値 */
    bool        isnull1;    /* 第1キー列がNULLか */
    int         srctape;    /* ソーステープ番号 */
} SortTuple;

入力と出力

  • 本構造体は tuplesort モジュール内部で、メモリ上でのソート（memtuples 配列）や外部ソート時のテープ間マージ処理において、データ保持および受け渡しに使用される。

前提条件と不変条件

  • tuple フィールドはソート対象のデータを指す。
  • datum1 および isnull1 は、ソート比較ロジックがタプル本体をデコードせずに比較を行うためのキャッシュとして機能する。

アルゴリズムまたは振る舞い

  • tuplesort.c において、qsort を用いたメモリ内ソートや、ヒープ構造を用いたマージ処理の要素として利用される。
  • mergereadnext 等の関数を通じて、外部ソート時のテープからの読み込み単位として扱われる。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/utils/sort/tuplesort.c: memtuples 配列の要素型として定義。
  • mergereadnext: テープからの読み込み時に使用。
  • tuplesort_heap_insert / tuplesort_heap_replace_top: ヒープ操作時に使用。
  • qsort_tuple_..._compare: 比較関数群の引数として使用。
  • **呼び出し先**:
  • free_sort_tuple: 本構造体が保持するメモリの解放に使用。

関連データ構造

  • Tuplesortstate: ソート操作全体の状態を管理する構造体。SortTuple の配列を保持する。

失敗モードと境界条件

  • メモリ制限（work_mem）を超過した場合、SortTuple は一時ファイル（テープ）へ書き出される。この際、srctape フィールドがどのテープから読み込まれたかを追跡するために使用される。

保守担当者向けメモ

  • SortTuple はソートのパフォーマンスに直結するため、構造体のサイズ変更はメモリ消費量およびキャッシュ効率に影響を与える。
  • qsort 系の比較関数は SortTuple 型に特化して実装されているため、フィールドの追加や変更時には関連する比較ロジックの更新が必要となる。
