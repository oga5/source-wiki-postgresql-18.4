ExecInitMerge

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeModifyTable.c:3684

目的

MERGE 操作に必要な状態（MergeActionState）を初期化し、各アクション（INSERT, UPDATE, DELETE）の実行計画や投影情報を設定する。

シグネチャまたは形状

void ExecInitMerge(ModifyTableState *mtstate, EState *estate)

入力と出力

  • **入力**:
  • mtstate: ModifyTableState 構造体（実行ノードの状態）。
  • estate: EState 構造体（クエリ実行全体の実行状態）。
  • **出力**: なし（mtstate 内部の resultRelInfo や mt_partition_tuple_routing 等を副作用として更新）。

前提条件と不変条件

  • mtstate->mt_mergeActionLists が空（NIL）の場合、本関数は何もせず即座に終了する。
  • パーティションテーブルに対する INSERT アクションの場合、mtstate->mt_partition_tuple_routing が未初期化であれば本関数内で初期化される。

アルゴリズムまたは振る舞い

1. mtstate から mergeActionLists を取得し、各アクションに対してループ処理を行う。 2. 各リレーションに対して ExecInitMergeTupleSlots を呼び出し、フェッチ用スロットを準備する。 3. ExecInitQual を用いて結合条件（ri_MergeJoinCondition）を初期化する。 4. 各アクション（MergeAction）に対し、MergeActionState を生成し、when 句の条件（mas_whenqual）を初期化する。 5. アクションの種別（CMD_INSERT, CMD_UPDATE, CMD_DELETE）に応じて以下の処理を行う:

  • **INSERT**: ターゲットリストの検証を行い、パーティションテーブルの場合はルーティング構造をセットアップする。それ以外の場合は ri_newTupleSlot を初期化し、ExecBuildProjectionInfo で投影情報を構築する。
  • **UPDATE**: ExecBuildUpdateProjection を用いて更新用の投影情報を構築する。
  • **DELETE**: mt_merge_subcommands に MERGE_DELETE フラグを立てる。

    • 6. 継承テーブル（パーティションを除く）の場合、rootRelInfo に対する制約チェックや RETURNING 句の初期化を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeModifyTable.c:5129（ExecInitModifyTable 内から呼び出される）。
  • **呼び出し先**: ExecAssignExprContext, ExecInitMergeTupleSlots, ExecInitQual, makeNode, lappend, ExecCheckPlanOutput, table_slot_create, ExecSetupPartitionTupleRouting, ExecBuildProjectionInfo, ExecBuildUpdateProjection 等。

関連データ構造

  • ModifyTableState: 実行ノードの状態を保持。
  • MergeActionState: 個別のアクションの状態（条件式や投影情報）。
  • ResultRelInfo: ターゲットリレーションごとの情報。
  • MergeAction: プランナーが生成したアクション定義。

失敗モードと境界条件

  • action->commandType が未知の場合、elog(ERROR) により処理を中断する。
  • パーティションテーブルのルーティング初期化時にメモリ不足等のエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数のロジックは ExecInitPartitionInfo() と重複する部分があるため、変更時は両方の関数を確認すること。
  • INSERT アクションの投影は常にルートリレーションの記述子を使用する仕様となっている。
