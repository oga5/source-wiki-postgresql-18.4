leafSegmentInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/gindatapage.c:70

目的

GINポスティングツリーのリーフページ内における、各圧縮セグメントの変更状態（INSERT, REPLACE, DELETE等）をメモリ上で追跡・管理するために使用されます。

シグネチャまたは形状

typedef struct {
    dlist_node    node;
    char          action;
    ItemPointerData *modifieditems;
    uint16        nmodifieditems;
    GinPostingList *seg;
    ItemPointer   items;
    int           nitems;
} leafSegmentInfo;

入力と出力

本構造体は関数ではなくデータ構造であるため、入力と出力の概念は適用されません。各フィールドは、セグメントの変更内容（action）や、保持するTIDリスト（items）、圧縮済みデータ（seg）を保持します。

前提条件と不変条件

  • items（非圧縮）と seg（圧縮済み）の両方が設定されている場合、それらは同一のアイテムセットを表している必要があります。
  • action フィールドは GIN_SEGMENT_* のいずれかのコードである必要があります。
  • DELETE アクションの場合、seg および items フィールドは無視されます。

アルゴリズムまたは振る舞い

  • ページ内の圧縮セグメントをメモリ上で分解（disassemble）し、変更を適用する際の単位として機能します。
  • ADDITEMS アクション時は、削除を行わずに modifieditems を通じて追加されたアイテムの詳細を追跡します。
  • ページ再構築時、この構造体のリストを走査して各セグメントの状態に応じた処理（削除、置換、新規挿入）が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/gin/gindatapage.c 内の複数の関数（dataExecPlaceToPageLeaf、ginVacuumPostingTreeLeaf 等）が、dlist_container マクロを介して本構造体を参照しています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • dlist_node: 構造体をリスト管理するために使用。
  • GinPostingList: 圧縮されたセグメントデータ。
  • ItemPointerData: 個別のTID情報。
  • disassembledLeaf: 本構造体をリストとして保持する親構造体。

失敗モードと境界条件

  • メモリ不足時: items の割り当て（palloc）に失敗する可能性があります。
  • 整合性エラー: items と seg が異なるデータを指している場合、ページ再構築時にデータ破損を招く恐れがあります。

保守担当者向けメモ

  • 本構造体は gindatapage.c 内の複雑なページ操作ロジックの中核です。変更を加える際は、action コードと seg/items の同期が正しく保たれているかを確認してください。
  • dlist を使用しているため、リスト操作時のノードの整合性に注意してください。
