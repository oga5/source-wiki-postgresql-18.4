r_shortv

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_english.c:463

目的

単語の末尾が「短音節（short syllable）」構造であるかを判定する。Porter Stemmerアルゴリズムにおける接尾辞削除の条件判定に使用される。

シグネチャまたは形状

static int r_shortv(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（Snowballの実行環境構造体）
  • 出力: 判定結果を int で返す。条件を満たす場合は 1、満たさない場合は 0。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 内部で g_v_WXY（母音グループ）および g_v（母音グループ）のグループ化定義を参照する。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置から後方に向かって以下のパターンを検索する：

  • パターンA: [母音以外の文字] + [母音] + [母音以外の文字]（ただし末尾の文字は 'w', 'x', 'y' を除く）
  • パターンB: [母音] + [母音以外の文字]（単語の先頭位置に達した場合）

    • 2. パターンAに一致する場合、成功（1）を返す。 3. パターンAに一致せず、かつパターンB（単語先頭での「母音＋子音」）に一致する場合、成功（1）を返す。 4. いずれにも該当しない場合、失敗（0）を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_english.c 内の複数の箇所（r_Step_1b 等のステップ処理から参照されている）。
  • 呼び出し先: out_grouping_b_U, in_grouping_b_U（Snowballランタイムの共通関数）。

関連データ構造

  • struct SN_env: ステミング処理の現在の状態（カーソル位置、文字列バッファ等）を保持する。

失敗モードと境界条件

  • 文字列の先頭（z->lb）に達した際の処理が組み込まれており、単語の境界を越えて検索することはない。
  • 検索対象の文字グループ（g_v 等）が定義されていない場合、あるいは環境が初期化されていない場合は未定義の挙動となる。

保守担当者向けメモ

  • 本関数は自動生成コードの一部である。ロジックの変更が必要な場合は、直接編集せず Snowball のソース定義（.sbl）を修正すること。
  • g_v や g_v_WXY といったグループ定義は header.h を通じて提供される。
