attribute_stats_argnum

定義

  • 種別: enum
  • 定義箇所: src/backend/statistics/attribute_stats.c:39

目的

統計情報のインポートや操作を行う関数において、引数配列の各要素が指し示す統計項目を識別するためのインデックスを定義します。

シグネチャまたは形状

enum attribute_stats_argnum {
    ATTRELSCHEMA_ARG,
    ATTRELNAME_ARG,
    ATTNAME_ARG,
    ATTNUM_ARG,
    INHERITED_ARG,
    NULL_FRAC_ARG,
    AVG_WIDTH_ARG,
    N_DISTINCT_ARG,
    MOST_COMMON_VALS_ARG,
    MOST_COMMON_FREQS_ARG,
    HISTOGRAM_BOUNDS_ARG,
    CORRELATION_ARG,
    MOST_COMMON_ELEMS_ARG,
    MOST_COMMON_ELEM_FREQS_ARG,
    ELEM_COUNT_HISTOGRAM_ARG,
    RANGE_LENGTH_HISTOGRAM_ARG,
    RANGE_EMPTY_FRAC_ARG,
    RANGE_BOUNDS_HISTOGRAM_ARG,
    NUM_ATTRIBUTE_STATS_ARGS
};

入力と出力

  • 入力: なし（定数定義）
  • 出力: 0 から NUM_ATTRIBUTE_STATS_ARGS - 1 までの整数値。

前提条件と不変条件

  • NUM_ATTRIBUTE_STATS_ARGS は列挙型の末尾に配置されており、配列のサイズ定義として使用されます。
  • 各列挙子は、統計操作関数が受け取る引数配列のインデックスと一対一で対応している必要があります。

アルゴリズムまたは振る舞い

統計情報の更新処理において、特定の統計項目（例: NULL_FRAC_ARG や HISTOGRAM_BOUNDS_ARG）にアクセスする際のオフセットとして機能します。これにより、マジックナンバーを排除し、コードの可読性と保守性を向上させています。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース excerpt 内では明示的な呼び出し元は特定されていません。
  • 呼び出し先: なし（データ定義のため）。

関連データ構造

  • pg_statistic システムカタログ: 本列挙型が定義する各項目は、最終的にこのカタログの列またはスロットに格納されます。

失敗モードと境界条件

  • インデックスが NUM_ATTRIBUTE_STATS_ARGS を超える場合、配列外参照が発生する可能性があります。

保守担当者向けメモ

  • 新しい統計項目を pg_statistic に追加する場合は、必ずこの列挙型に新しい要素を追加し、NUM_ATTRIBUTE_STATS_ARGS が正しく更新されるようにしてください。
