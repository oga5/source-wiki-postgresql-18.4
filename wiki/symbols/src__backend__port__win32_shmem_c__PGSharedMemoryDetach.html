PGSharedMemoryDetach

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_shmem.c:504

目的

Windows 環境において、現在プロセスにマッピングされている共有メモリ領域の解放、アドレス空間の予約解除、および共有メモリハンドルをクローズする。

シグネチャまたは形状

void PGSharedMemoryDetach(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数のリセットと OS リソースの解放を行う）

前提条件と不変条件

  • ShmemProtectiveRegion が予約されている場合、VirtualFree で解放される。
  • UsedShmemSegAddr がマッピングされている場合、UnmapViewOfFile でアンマップされる。
  • UsedShmemSegID が有効なハンドルである場合、CloseHandle でクローズされる。
  • 処理完了後、各グローバル変数は初期状態（NULL または INVALID_HANDLE_VALUE）にリセットされる。

アルゴリズムまたは振る舞い

1. **保護領域の解放**: ShmemProtectiveRegion が NULL でなければ、VirtualFree を呼び出し、予約されたアドレス空間を解放する。成功後、変数を NULL に設定する。 2. **メモリのアンマップ**: UsedShmemSegAddr が NULL でなければ、UnmapViewOfFile を呼び出し、共有メモリビューをアンマップする。成功後、変数を NULL に設定する。 3. **ハンドルのクローズ**: UsedShmemSegID が INVALID_HANDLE_VALUE でなければ、CloseHandle を呼び出し、共有メモリハンドルを閉じる。成功後、変数を INVALID_HANDLE_VALUE に設定する。

失敗モードと境界条件

  • 各 Windows API (VirtualFree, UnmapViewOfFile, CloseHandle) が失敗した場合、elog(LOG, ...) を通じてエラーコードと共にログが出力される。
  • 既に解放・クローズ済みの状態（各変数が初期値）であれば、各条件分岐はスキップされ、何も行われない。

保守担当者向けメモ

  • 本関数はプロセス終了時のクリーンアップを目的としている。
  • ShmemProtectiveRegion の解放は、アドレス空間の断片化を防ぐための「整理」として行われる。
  • エラー発生時は LOG レベルで記録されるが、プロセスの終了処理自体は継続される。
