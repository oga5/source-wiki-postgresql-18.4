pg_lock_status

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:92

目的

システム内の現在保持中または待機中のロック情報を収集し、pg_locks ビューの基盤となるタプルセットとして出力する。

シグネチャまたは形状

Datum pg_lock_status(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出し規約に基づく引数）。
  • **出力**: Datum（SRFとして、ロック状態を示す16列のタプルを順次返却）。

前提条件と不変条件

  • SRF（Set Returning Function）として実装されており、初回呼び出し時に SRF_FIRSTCALL_INIT でコンテキストを初期化する必要がある。
  • 呼び出し間で状態を保持するため、funcctx->user_fctx に PG_Lock_Status 構造体を格納して永続化する。
  • ロックマネージャから取得したデータ（lockData）を破壊的に読み進めるため、呼び出しごとのインデックス管理が必須である。

アルゴリズムまたは振る舞い

1. **初回呼び出し**: GetLockStatusData() および GetPredicateLockStatusData() を呼び出し、全ロック状態のスナップショットを取得する。 2. **反復処理**: SRF_PERCALL_SETUP を経て、保持しているロック（holdMask）を優先的にタプル化する。 3. **状態更新**: holdMask 内の各ビットを順次処理し、完了したものは LOCKBIT_OFF でクリアする。 4. **待機ロック**: 保持中のロックがない場合、waitLockMode を確認し、待機中のロックがあればそれを報告する。 5. **終了判定**: 全てのロック要素を走査し終えるまで、currIdx を進めながらタプルを生成し続ける。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（pg_locks ビューのクエリ実行時）。
  • **呼び出し先**:
  • GetLockStatusData() / GetPredicateLockStatusData()（ロックマネージャからのデータ取得）。
  • SRF_FIRSTCALL_INIT / SRF_PERCALL_SETUP（SRF制御用マクロ）。
  • LockTagTypeNames（ロックタグの文字列変換）。

関連データ構造

  • PG_Lock_Status: 実行状態を保持するコンテキスト構造体。
  • LockData: ロックマネージャから取得したロック情報のスナップショット。
  • LockInstanceData: 個別のロックインスタンス情報。

失敗モードと境界条件

  • ロックタグのタイプが LOCKTAG_LAST_TYPE を超える場合、"unknown %d" という形式で文字列化される。
  • 述語ロック（Predicate Locks）は通常のロックとは管理体系が異なるため、predLockData を用いて別途処理される。

保守担当者向けメモ

  • 本関数はロックマネージャのデータを破壊的に読み進めるため、ロジック変更時は currIdx のインクリメントタイミングに注意すること。
  • タプルの列定義（NUM_LOCK_STATUS_COLUMNS）は pg_proc.h の定義と厳密に一致させる必要がある。
