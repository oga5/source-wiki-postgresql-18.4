array_iter_setup

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/arrayaccess.h:48

目的

配列データ（フラットな配列または展開された配列）を走査するためのイテレータ構造体 array_iter を初期化します。

シグネチャまたは形状

static inline void array_iter_setup(array_iter *it, AnyArrayType *a)

入力と出力

  • **入力**:
  • it: 初期化対象の array_iter 構造体へのポインタ。
  • a: 走査対象の AnyArrayType へのポインタ。
  • **出力**: なし（it が指す構造体の各フィールドが更新されます）。

前提条件と不変条件

  • it は有効なメモリ領域を指している必要があります。
  • a が展開済み配列（ExpandedArrayHeader）である場合、その内部状態に応じて datumptr または dataptr が適切に設定されます。

アルゴリズムまたは振る舞い

1. VARATT_IS_EXPANDED_HEADER(a) を用いて、入力が展開済み配列か否かを判定します。 2. **展開済み配列の場合**:

  • dvalues が存在すれば、datumptr と isnullptr に値をセットし、dataptr と bitmapptr を NULL に設定します。
  • dvalues が存在しない場合（フラットな配列を内包している場合）、dataptr と bitmapptr を内包する配列から取得し、datumptr と isnullptr を NULL に設定します。

    • 3. **フラットな配列の場合**:

  • ARR_DATA_PTR および ARR_NULLBITMAP を使用して、データ領域と NULL ビットマップへのポインタをセットします。

    • 4. 最後に、ビットマップ走査用の it->bitmask を 1 に初期化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arrayfuncs.c 内の複数の箇所（例: array_iter_setup(&iter, v); 等）から呼び出されています。
  • **呼び出し先**: VARATT_IS_EXPANDED_HEADER, ARR_DATA_PTR, ARR_NULLBITMAP などのマクロを使用します。

関連データ構造

  • array_iter: 走査状態を保持する構造体。
  • AnyArrayType: 配列の共通型定義。

失敗モードと境界条件

  • a が NULL の場合の挙動は定義されていません。
  • 展開済み配列の内部構造が不整合な場合、ポインタの割り当てが意図しない結果になる可能性があります。

保守担当者向けメモ

  • 本関数はインライン化されているため、変更時は src/include/utils/arrayaccess.h の再コンパイルが必要です。
  • it->bitmask の初期値 1 は、array_iter_next でのビット演算の開始点として固定されています。
