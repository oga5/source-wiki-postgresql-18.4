ProjectionInfo

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/execnodes.h:377

目的

クエリ実行エンジンにおいて、タプルの射影（Projection）処理を管理するための構造体です。ターゲットリストの評価命令と、その評価に使用するコンテキストを保持します。

シグネチャまたは形状

typedef struct ProjectionInfo
{
    NodeTag     type;
    ExprState   pi_state;
    ExprContext *pi_exprContext;
} ProjectionInfo;

入力と出力

  • **入力**: 評価対象となる式（pi_state 内に保持）および、式評価の基盤となるコンテキスト（pi_exprContext）。
  • **出力**: 射影の結果として生成される新しいタプル（実行時の ExecProject 等の関数を通じて出力）。

前提条件と不変条件

  • pi_state は ProjectionInfo 構造体内に埋め込まれており、別途メモリ確保（palloc）を行わない設計となっています。
  • 実行時専用の構造体であり、永続化やコピー（copyObject 等）はサポートされていません。

アルゴリズムまたは振る舞い

  • クエリ実行中にターゲットリストを評価し、新しいタプルを構築するために使用されます。
  • pi_exprContext に設定された現在のタプルやパラメータに基づき、pi_state が保持する命令列を実行することで射影を行います。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/executor/execExpr.c: 射影ノードの構築（makeNode(ProjectionInfo)）に使用。
  • src/backend/executor/execScan.c: スキャン処理における射影情報の保持。
  • src/backend/executor/nodeModifyTable.c: UPDATE 処理や RETURNING 句の処理における射影情報の構築・参照。
  • **呼び出し先**:
  • 提供されたソース excerpt には、本構造体が直接呼び出す関数や、本構造体を呼び出す具体的な実行関数（ExecProject 等）の内部ロジックは含まれていません。

関連データ構造

  • ExprState: 射影の評価命令を保持。
  • ExprContext: 評価時の変数や状態を保持。
  • PlanState: 実行ノードの基底構造体。

失敗モードと境界条件

  • 射影対象の式が複雑な場合や、メモリコンテキストの管理が不適切な場合に実行時エラーが発生する可能性があります。
  • 構造体自体の初期化が不完全な状態で ExecProject が呼び出された場合の挙動については、本ソースからは判断できません。

保守担当者向けメモ

  • pi_state は埋め込みフィールドであるため、ProjectionInfo を解放する際は ExprState 側のクリーンアップ処理と整合性を取る必要があります。
  • lefttree / righttree 等のプランノードとの関係性は PlanState を介して管理されるため、本構造体単体での操作には注意してください。
