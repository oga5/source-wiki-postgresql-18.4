NamespaceInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.h:184

目的

データベースのスキーマ（Namespace）情報をメモリ上で保持し、pg_dump におけるダンプ対象オブジェクトとして管理するための構造体です。

シグネチャまたは形状

typedef struct _namespaceInfo
{
	DumpableObject dobj;
	DumpableAcl dacl;
	bool		create;
	Oid			nspowner;
	const char *rolname;
} NamespaceInfo;

入力と出力

  • **入力**: システムカタログから取得したスキーマ情報（OID、所有者名、作成フラグ等）。
  • **出力**: pg_dump のダンプ処理プロセスにおいて、スキーマの定義生成や所有権設定の制御に使用されます。

前提条件と不変条件

  • dobj フィールドにより、DumpableObject としての共通属性（ID、依存関係など）を保持します。
  • pg_dump の実行期間中、メモリ上に生存します。

アルゴリズムまたは振る舞い

  • create フラグは、対象スキーマに対して CREATE SCHEMA 文を発行するか、あるいは所有権の設定のみを行うかを決定するために使用されます。
  • pg_dump.c 内の dumpNamespace 関数等によって、アーカイブへの書き出し処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/bin/pg_dump/pg_dump.c: findNamespace, dumpNamespace, selectDumpableNamespace 等で利用。
  • src/bin/pg_dump/common.c: 型キャストやオブジェクト管理の文脈で参照。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • DumpableObject: 基底構造体。
  • DumpableAcl: スキーマに関連付けられたアクセス制御リスト。
  • Archive: dumpNamespace 等の処理でアーカイブ管理のために使用されます。

失敗モードと境界条件

  • メモリ割り当て時（pg_malloc 等）の失敗が想定されます。
  • rolname は所有者名へのポインタであり、生存期間の管理に注意が必要です。

保守担当者向けメモ

  • 新しいスキーマ属性を追加する場合は、pg_dump.c 内の初期化ロジックおよびダンプ出力ロジックを更新してください。
  • dobj を介した依存関係管理が行われているため、スキーマのダンプ順序を変更する際は pg_dump_sort.c の整合性に留意してください。
