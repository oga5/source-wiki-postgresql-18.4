dispell_lexize

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_ispell.c:111

目的

Ispell辞書を用いて入力単語の正規化（語幹抽出）を行い、ストップワードを除外した語彙リストを生成する。

シグネチャまたは形状

Datum dispell_lexize(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): DictISpell * 型の辞書オブジェクト。
  • PG_GETARG_POINTER(1): char * 型の入力文字列。
  • PG_GETARG_INT32(2): int32 型の入力文字列長。
  • **出力**:
  • Datum: 正規化された TSLexeme * 配列（終端は lexeme が NULL）。結果がない場合は NULL を返す。

前提条件と不変条件

  • 入力文字列長が 0 以下の場合、直ちに NULL を返す。
  • 内部で str_tolower を使用し、入力文字列を小文字化して処理する。
  • 返却される TSLexeme 配列は動的に確保されており、呼び出し側で管理する必要がある。

アルゴリズムまたは振る舞い

1. 入力文字列を小文字化する。 2. NINormalizeWord を呼び出し、Ispell辞書に基づいて単語を正規化し、TSLexeme 配列を取得する。 3. 取得した配列を走査し、各語彙が searchstoplist に該当するか確認する。 4. ストップワードに該当する場合は pfree でメモリを解放し、該当しない場合は結果配列を詰め直す。 5. 最終的な語彙リストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に明示的な呼び出し元情報は記載されていないが、ts_lexize 関数から呼び出される辞書プラグインとして機能する。
  • **呼び出し先**:
  • str_tolower
  • NINormalizeWord
  • searchstoplist
  • pfree

関連データ構造

  • DictISpell: 辞書の状態（ストップリストと辞書オブジェクト）を保持。
  • TSLexeme: 形態素解析結果を格納する構造体。

失敗モードと境界条件

  • 入力長が 0 以下の場合、処理を中断し NULL を返す。
  • NINormalizeWord が NULL を返した場合、処理を中断し NULL を返す。

保守担当者向けメモ

  • 本関数は spell.c で定義された低レベル API のラッパーである。
  • メモリ管理（pfree）は dispell_lexize 内で行われるため、結果配列のライフサイクルに注意すること。
