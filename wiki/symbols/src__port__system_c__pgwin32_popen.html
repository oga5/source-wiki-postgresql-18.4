pgwin32_popen

定義

  • 種別: function
  • 定義箇所: src/port/system.c:85

目的

Windows環境において、コマンド文字列を二重引用符（"）で囲むことで、cmd.exe による引数解釈の誤りを防ぎ、安全に _popen を実行するためのラッパー関数です。

シグネチャまたは形状

FILE *pgwin32_popen(const char *command, const char *type)

入力と出力

  • **入力**:
  • command: 実行するコマンド文字列。
  • type: _popen に渡すモード文字列（"r" や "w" など）。
  • **出力**:
  • 成功時: プロセスへのパイプを表す FILE * ポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • 本関数は WIN32 かつ __CYGWIN__ ではない環境でのみ有効です。
  • 呼び出し側で既に引用符が含まれる複雑なコマンドを渡す場合、適切にエスケープされている必要があります。

アルゴリズムまたは振る舞い

1. command の長さを取得し、引用符2バイトと終端文字1バイトを加えたメモリを確保します。 2. 確保したバッファの先頭と末尾に " を挿入し、中央に元のコマンドをコピーします。 3. 変換後の文字列を標準の _popen に渡して実行します。 4. 実行時の errno を保存し、バッファを解放した後、保存していた errno を復元して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port.h にて popen マクロとして定義されており、標準の popen を呼び出す箇所から間接的に利用されます。
  • **呼び出し先**:
  • malloc, free, memcpy, strlen (標準ライブラリ)
  • _popen (Windows標準関数)

関連データ構造

  • FILE (標準ライブラリのファイルストリーム)

失敗モードと境界条件

  • malloc が失敗した場合、errno を ENOMEM に設定し、NULL を返します。
  • _popen 自体の実行失敗については、その戻り値と errno がそのまま呼び出し元に伝播します。

保守担当者向けメモ

  • cmd.exe の引用符ルールは複雑であるため、本実装は「文字列全体を引用符で囲む」という最小限の安全策をとっています。
  • src/port/system.c 内で popen を本関数に置換するマクロが定義されているため、Windows環境では標準の popen を直接使用しないよう注意してください。
