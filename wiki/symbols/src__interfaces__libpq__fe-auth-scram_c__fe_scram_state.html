fe_scram_state

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/fe-auth-scram.c:52

目的

PostgreSQL クライアント（libpq）における SCRAM-SHA-256 認証セッション中の状態を保持・管理する。

シグネチャまたは形状

typedef struct {
    fe_scram_state_enum state;
    PGconn     *conn;
    char       *password;
    char       *sasl_mechanism;
    pg_cryptohash_type hash_type;
    int         key_length;
    uint8       SaltedPassword[SCRAM_MAX_KEY_LEN];
    char       *client_nonce;
    char       *client_first_message_bare;
    char       *client_final_message_without_proof;
    char       *server_first_message;
    uint8      *salt;
    int         saltlen;
    int         iterations;
    char       *nonce;
    char       *server_final_message;
    uint8       ServerSignature[SCRAM_MAX_KEY_LEN];
} fe_scram_state;

入力と出力

  • 入力: 認証開始時に malloc で確保され、memset でゼロ初期化される。
  • 出力: 認証プロセス終了時に解放される（scram_free 等による管理を想定）。

前提条件と不変条件

  • state フィールドは fe_scram_state_enum に基づき、認証の進行状況を追跡する。
  • SaltedPassword および ServerSignature は SCRAM_MAX_KEY_LEN の固定長バッファを使用する。
  • 認証フローにおいて、サーバーからのメッセージ受信時に各フィールドが順次更新される。

アルゴリズムまたは振る舞い

  • 認証の各ステップ（FE_SCRAM_INIT → FE_SCRAM_NONCE_SENT → FE_SCRAM_PROOF_SENT）において、クライアントの nonce 生成、パスワードの正規化、HMAC を用いた証明計算、サーバー署名の検証を行う。
  • scram_exchange 関数が本構造体を介してステートマシンを駆動する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-auth-scram.c 内の scram_init、scram_exchange 等。
  • 呼び出し先: 本構造体を参照する関数として read_server_first_message、read_server_final_message、build_client_first_message、build_client_final_message、verify_server_signature、calculate_client_proof が存在する。

関連データ構造

  • fe_scram_state_enum: 状態遷移を定義する列挙型。
  • PGconn: クライアント接続情報。
  • pg_cryptohash_type: ハッシュアルゴリズムの指定。

失敗モードと境界条件

  • メモリ確保失敗時の挙動については、呼び出し側のエラーハンドリングに依存する。
  • input 文字列の解析（read_attr_value）は破壊的であるため、構造体内のポインタが指すメモリ領域の整合性に注意が必要。

保守担当者向けメモ

  • 本構造体は libpq のフロントエンド実装専用である。
  • サーバー側の実装（src/backend/libpq/auth-scram.c）と対になるため、プロトコル変更時は両方の整合性を確認すること。
  • チャネルバインディングが有効な場合、SSL 証明書のハッシュ値が認証プロセスに組み込まれる点に留意。
