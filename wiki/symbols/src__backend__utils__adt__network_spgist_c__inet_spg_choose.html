inet_spg_choose

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/network_spgist.c:67

目的

SP-GiSTインデックスにおいて、新しいネットワーク値（inet）を挿入する際、どのノードへ配置するか、あるいはインデックス構造の分割（split）が必要かを決定します。

シグネチャまたは形状

Datum inet_spg_choose(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgChooseIn（挿入対象データ、既存ノード情報）および spgChooseOut（決定結果）を受け取ります。
  • **出力**: spgChooseOut 構造体を更新し、spgMatchNode（既存ノードへの降下）または spgSplitTuple（ノード分割の実行）を指示します。

前提条件と不変条件

  • プレフィックスを持たないタプルは、必ず2ノード構成（IPv4/IPv6の分離）である必要があります。
  • プレフィックスを持つタプルは、4ノード構成であるか、allTheSame フラグが立っている必要があります。
  • 異なるアドレスファミリ（IPv4とIPv6）を同一の内部ノードに混在させることはできません。

アルゴリズムまたは振る舞い

1. **プレフィックスなし**: アドレスファミリに基づき、ノード0（IPv4）またはノード1（IPv6）を選択します。 2. **ファミリ不一致**: 既存のプレフィックスと新しい値のファミリが異なる場合、spgSplitTuple を発行して2ノード構成へ分割します。 3. **プレフィックス不一致**: 新しい値が既存のプレフィックスと一致しない場合、共通ビットを再計算し、spgSplitTuple を発行して4ノード構成へ分割します。 4. **正常系**: 上記のいずれにも該当しない場合、inet_spg_node_number を用いて適切な子ノードを特定し、spgMatchNode を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSP-GiSTインデックス管理コード（直接の呼び出し元情報はソースコード内に明示されていません）。
  • **呼び出し先**:
  • inet_spg_node_number: 子ノードの特定に使用。
  • cidr_set_masklen_internal: 分割時のプレフィックス生成に使用。
  • ip_family, ip_bits, bitncmp, bitncommon: ネットワークアドレスのビット演算に使用。

関連データ構造

  • spgChooseIn, spgChooseOut: SP-GiSTの選択ロジック用インターフェース構造体。
  • inet: PostgreSQLのネットワーク型内部表現。

失敗モードと境界条件

  • 異なるアドレスファミリが混在する挿入が発生した場合、強制的に分割処理が実行されます。
  • allTheSame が設定されている場合、ノード選択の結果は無視される可能性があります。

保守担当者向けメモ

  • 本関数はビット単位の比較を行うため、ip_bits や bitncommon の挙動変更はインデックス構造の整合性に直結します。
  • 4ノード構成のロジックは inet_spg_node_number と密結合しているため、ノード配置ルールを変更する際は両方の修正が必要です。
