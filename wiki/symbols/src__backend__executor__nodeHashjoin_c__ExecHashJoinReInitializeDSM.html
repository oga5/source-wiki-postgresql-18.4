ExecHashJoinReInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHashjoin.c:1713

目的

並列ハッシュ結合の再実行に備え、共有メモリ状態（DSM）を初期化する。

シグネチャまたは形状

void ExecHashJoinReInitializeDSM(HashJoinState *state, ParallelContext *pcxt)

入力と出力

  • state: ハッシュ結合の実行状態を保持する HashJoinState 構造体。
  • pcxt: 並列実行コンテキスト。
  • 出力なし（void）。

前提条件と不変条件

  • pcxt->seg が NULL の場合、本関数は何もせず即座に復帰する。
  • ParallelHashJoinState は pcxt->toc から plan_node_id を用いて取得される。

アルゴリズムまたは振る舞い

1. pcxt->seg が存在しない場合は処理を終了する。 2. shm_toc_lookup を使用して共有並列ハッシュ結合状態（pstate）を取得する。 3. state->hj_HashTable が存在する場合、ExecHashTableDetachBatch および ExecHashTableDetach を呼び出し、ハッシュテーブルの共有メモリを解放する。 4. SharedFileSetDeleteAll を呼び出し、共有バッチファイルを削除する。 5. BarrierInit を使用して pstate->build_barrier を PHJ_BUILD_ELECT にリセットし、再実行可能な状態にする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1012
  • **呼び出し先**:
  • shm_toc_lookup
  • ExecHashTableDetachBatch
  • ExecHashTableDetach
  • SharedFileSetDeleteAll
  • BarrierInit

関連データ構造

  • HashJoinState: 結合ノードの実行状態。
  • ParallelContext: 並列実行のコンテキスト。
  • ParallelHashJoinState: 共有メモリ上の並列ハッシュ結合状態。

失敗モードと境界条件

  • pcxt->seg が NULL の場合、安全に早期復帰する。
  • ハッシュテーブルが既に解放されている場合、state->hj_HashTable が NULL であれば解放処理はスキップされる。

保守担当者向けメモ

  • 現在の実装では、単一バッチのハッシュ結合であっても共有ハッシュテーブルを再利用せず、一度解放して再構築する方針をとっている。
  • finalize_plan() における並列ハッシュノードのダミー依存関係記録が残っているため、現時点ではリスカン最適化は行われない。
