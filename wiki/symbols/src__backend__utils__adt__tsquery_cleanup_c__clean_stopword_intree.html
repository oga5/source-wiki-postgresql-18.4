clean_stopword_intree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:237

目的

tsquery のクエリツリーからストップワード（QI_VALSTOP）を除去し、それに伴うフレーズ演算子（OP_PHRASE）の距離（distance）を再計算・調整します。

シグネチャまたは形状

static NODE *clean_stopword_intree(NODE *node, int *ladd, int *radd)

入力と出力

  • **入力**:
  • node: 処理対象のクエリツリーノード。
  • ladd, radd: 呼び出し元から渡される、親ノードへ伝播させる距離の累積値を受け取るポインタ。
  • **出力**:
  • 戻り値: 処理後の NODE ツリーのルート。ストップワード除去によりノードが削除された場合は NULL を返す。
  • ladd, radd: 削除されたサブツリーの距離情報が格納される。

前提条件と不変条件

  • 再帰的なツリー走査を行うため、スタックオーバーフローを防止する check_stack_depth() が呼び出されます。
  • フレーズ演算子以外の場合、ladd と radd は 0 に初期化されます。
  • フレーズ演算子において、子ノードから伝播される距離（lladd, lradd 等）は、ルールが正しく適用されていれば左右で等しいと仮定されます。

アルゴリズムまたは振る舞い

1. **ベースケース**: QI_VAL 型ノードはそのまま返します。QI_VALSTOP 型ノードは pfree で解放し NULL を返します。 2. **再帰処理**:

  • OP_NOT の場合: 右側のサブツリーを再帰的に処理します。
  • OP_PHRASE 以外の場合: 左右のサブツリーを再帰的に処理し、子ノードが削除された場合は距離を累積して親へ伝播させます。
  • OP_PHRASE の場合: 左右のサブツリーを再帰的に処理し、ストップワード除去によって生じた距離の欠損を node->valnode->qoperator.distance に加算します。

    • 3. **ノードの崩壊**: 子ノードが両方とも削除された場合、現在のノードも削除し、累積した距離を ladd/radd に設定します。片方のみ削除された場合は、残ったサブツリーを親へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: cleanup_tsquery_stopwords（src/backend/utils/adt/tsquery_cleanup.c:404）から呼び出されます。
  • **呼び出し先**: 再帰的に自身を呼び出すほか、freetree（ノード解放）、pfree（メモリ解放）、check_stack_depth（スタック保護）を呼び出します。

関連データ構造

  • NODE: クエリツリーを構成する構造体。
  • QueryItem: tsquery の要素を保持する構造体。

失敗モードと境界条件

  • **スタックオーバーフロー**: 深い入れ子構造を持つクエリに対しては check_stack_depth() によりエラーが発生します。
  • **全ノード削除**: ストップワードのみで構成されたツリーの場合、最終的に NULL が返されます。

保守担当者向けメモ

  • フレーズ演算子の距離調整ロジックは、ストップワード除去時に距離を親へ伝播させる仕組みに依存しています。特に isphrase の判定と距離の加算処理は、クエリの整合性を保つための重要な箇所です。
  • ladd と radd の伝播ルールは、TS_execute の動作と整合するように設計されています。変更時は距離の累積ロジックに注意してください。
