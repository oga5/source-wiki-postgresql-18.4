pg_hmac_init

定義

  • 種別: function
  • 定義箇所: src/common/hmac_openssl.c:170

目的

OpenSSL の HMAC_CTX を使用して、指定されたハッシュアルゴリズムと秘密鍵で HMAC 計算コンテキストを初期化します。

シグネチャまたは形状

int pg_hmac_init(pg_hmac_ctx *ctx, const uint8 *key, size_t len);

入力と出力

  • **入力**:
  • ctx: 初期化対象の pg_hmac_ctx 構造体へのポインタ。
  • key: HMAC 計算に使用する秘密鍵のバイト配列。
  • len: 秘密鍵の長さ（バイト単位）。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は -1 を返し、ctx->error に PG_HMAC_ERROR_OPENSSL が設定されます。

前提条件と不変条件

  • ctx が NULL でないこと。
  • ctx->type に有効なハッシュアルゴリズム（PG_MD5, PG_SHA1, PG_SHA224, PG_SHA256, PG_SHA384, PG_SHA512 のいずれか）が設定されていること。

アルゴリズムまたは振る舞い

1. ctx が NULL の場合は即座に -1 を返します。 2. ctx->type に基づき、対応する OpenSSL の EVP ダイジェスト関数（EVP_md5(), EVP_sha1() 等）を選択します。 3. HMAC_Init_ex を呼び出し、コンテキスト、鍵、鍵長、および選択したダイジェスト関数を渡して初期化を実行します。 4. OpenSSL の戻り値が 1（成功）以外の場合、ERR_get_error() でエラーコードを取得し、SSLerrmessage を用いて ctx->errreason にエラーメッセージを格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c
  • src/common/scram-common.c
  • src/interfaces/libpq/fe-auth-scram.c
  • **呼び出し先**:
  • HMAC_Init_ex (OpenSSL)
  • EVP_md5, EVP_sha1, EVP_sha224, EVP_sha256, EVP_sha384, EVP_sha512 (OpenSSL)
  • ERR_get_error (OpenSSL)
  • SSLerrmessage (内部関数)

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する構造体。

失敗モードと境界条件

  • ctx が NULL の場合: -1 を返します。
  • OpenSSL 内部でエラーが発生した場合: status <= 0 となり、エラー理由が ctx に記録された上で -1 を返します。

保守担当者向けメモ

  • 本関数は OpenSSL が有効なビルド環境でのみ使用されます。
  • エラー発生時の ctx->errreason は SSLerrmessage を介して取得されるため、OpenSSL のエラーキューの状態に依存します。
