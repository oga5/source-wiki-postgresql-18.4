checkcondition_arr

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsgistidx.c:275

目的

GiST インデックスのリーフページにおいて、tsquery のオペランド（ハッシュ値）が tsvector のインデックスキーに含まれているかをバイナリサーチで判定します。

シグネチャまたは形状

static TSTernaryValue checkcondition_arr(void *checkval, QueryOperand *val, ExecPhraseData *data)

入力と出力

  • **入力**:
  • checkval: CHKVAL 構造体へのポインタ。インデックス化されたハッシュ値の配列（arrb から arre）を保持します。
  • val: 検索対象の QueryOperand。valcrc（ハッシュ値）および prefix フラグを含みます。
  • data: ExecPhraseData 構造体（本関数内では未使用）。
  • **出力**:
  • TS_MAYBE: 条件に合致する可能性がある場合。
  • TS_NO: 条件に合致しない場合。

前提条件と不変条件

  • checkval は CHKVAL 型にキャスト可能である必要があります。
  • checkval 内の配列はソート済みであると想定されます。
  • ループ不変条件: StopLow <= val < StopHigh。

アルゴリズムまたは振る舞い

1. val->prefix が真の場合、ハッシュ値による前方一致検索は不可能なため、即座に TS_MAYBE を返します。 2. CHKVAL 内のハッシュ値配列に対してバイナリサーチを実行します。 3. val->valcrc と一致する値が見つかった場合、TS_MAYBE を返します。 4. 配列を走査し終えても一致が見つからない場合、TS_NO を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: gtsvector_consistent（src/backend/utils/adt/tsgistidx.c）から、リーフノードの判定ロジックとして呼び出されます。
  • **呼び出し先**: 外部関数への明示的な呼び出しはありません。
  • **注記**: contrib/intarray/_int_bool.c に同名の関数が存在しますが、本ページは tsgistidx.c 内の定義を対象としています。

関連データ構造

  • CHKVAL: tsgistidx.c 内で定義され、arrb（開始ポインタ）と arre（終了ポインタ）を介してハッシュ値配列を管理します。
  • QueryOperand: tsearch で使用されるクエリのオペランド構造体。
  • TSTernaryValue: 検索結果の適合性を示す列挙型。

失敗モードと境界条件

  • val->prefix が設定されている場合、ハッシュ値の照合を行わずに TS_MAYBE を返すため、インデックスによる絞り込みは行われません。
  • 空の配列（StopLow == StopHigh）が渡された場合、ループは実行されず TS_NO が返されます。

保守担当者向けメモ

  • 本関数は TS_execute のコールバックとして機能します。
  • インデックスが「不正確（inexact）」であるため、TS_MAYBE が返された場合は、後続のヒープ再チェックが必要です。
