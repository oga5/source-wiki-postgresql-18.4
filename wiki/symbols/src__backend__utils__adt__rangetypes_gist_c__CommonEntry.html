CommonEntry

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/rangetypes_gist.c:104

目的

GiSTインデックスのページ分割処理において、各エントリが2つの分割グループのどちらに近いかを評価し、ソートするための作業用構造体です。

シグネチャまたは形状

typedef struct {
    int    index;  /* 初期配列内でのエントリのインデックス */
    double delta;  /* 2つのグループに対する近接度の差分 */
} CommonEntry;

入力と出力

  • 本構造体は、インデックスの分割アルゴリズム内で一時的に生成・利用されます。
  • delta フィールドは、分割コストを最小化するためのヒューリスティックな評価値として計算されます。

前提条件と不変条件

  • index は、処理対象の元のエントリ配列に対する有効なインデックスである必要があります。
  • delta は、qsort 等を用いたソート処理において、分割の優先順位を決定するために使用されます。

アルゴリズムまたは振る舞い

  • インデックスページ分割時、エントリを2つのグループに分ける際の「コスト差分」を計算し、この構造体に格納します。
  • その後、delta 値に基づいて qsort を実行し、分割の境界を決定するロジックに供されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/rangetypes_gist.c 内の分割ロジック（range_gist_picksplit 等）から生成・利用されます。また、src/backend/access/gist/gistproc.c でも同様の構造体が利用されています。
  • **呼び出し先**: qsort 関数に渡される比較関数（common_entry_cmp 等）から参照されます。

関連データ構造

  • range_gist_picksplit: 本構造体を利用して分割候補を評価するメイン関数。

失敗モードと境界条件

  • palloc によるメモリ確保に失敗した場合、システム全体でメモリ不足エラーが発生します。
  • nentries が0の場合、本構造体を用いたソート処理は実行されません。

保守担当者向けメモ

  • 本構造体は src/backend/utils/adt/rangetypes_gist.c と src/backend/access/gist/gistproc.c の両方で定義・利用されています。変更を加える際は、両方の実装に影響がないか確認してください。
  • delta の計算ロジックは、インデックスの検索効率に直結するヒューリスティックであるため、変更時は性能への影響を評価してください。
