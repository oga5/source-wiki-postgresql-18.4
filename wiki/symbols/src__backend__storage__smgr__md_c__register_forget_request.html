register_forget_request

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/md.c:1568

目的

リレーションの特定のフォークおよびセグメントに関連付けられた、保留中の fsync 要求を破棄（忘却）するために使用されます。

シグネチャまたは形状

static void register_forget_request(RelFileLocatorBackend rlocator, ForkNumber forknum, BlockNumber segno)

入力と出力

  • **入力**:
  • rlocator: 対象リレーションを識別する RelFileLocatorBackend。
  • forknum: 対象のフォーク番号（ForkNumber）。
  • segno: 対象のセグメント番号（BlockNumber）。
  • **出力**: なし（void）。

前提条件と不変条件

  • INIT_MD_FILETAG マクロにより、指定されたリレーション、フォーク、セグメント情報から FileTag が初期化される必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った rlocator、forknum、segno を使用して FileTag 構造体を初期化します。 2. RegisterSyncRequest を呼び出し、SYNC_FORGET_REQUEST フラグを指定して、チェックポインタプロセス等に当該ファイルの同期要求を破棄するよう通知します。 3. retryOnError 引数には true が渡され、要求の登録失敗時に再試行が行われる設定となっています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/smgr/md.c:386
  • src/backend/storage/smgr/md.c:451
  • **呼び出し先**:
  • RegisterSyncRequest（storage/sync.h で定義される同期要求登録関数）

関連データ構造

  • RelFileLocatorBackend: リレーションの物理的な位置を特定する構造体。
  • FileTag: 同期要求の対象を識別するためのタグ構造体。
  • ForkNumber: リレーションのフォーク（メイン、FSM、VM等）を識別する列挙型。

失敗モードと境界条件

  • RegisterSyncRequest の内部実装に依存します。本関数自体は retryOnError を有効にして呼び出しているため、一時的なリソース不足等による登録失敗に対しては再試行が試みられます。

保守担当者向けメモ

  • 本関数は md.c 内の静的関数であり、主にリレーションの削除や切り詰め時に、不要となった同期要求をクリーンアップする目的で利用されます。
