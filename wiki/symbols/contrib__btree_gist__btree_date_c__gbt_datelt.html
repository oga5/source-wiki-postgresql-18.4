gbt_datelt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_date.c:62

目的

GiST インデックス内において、2つの date 型値の「より小さい（less than）」比較を行うためのラッパー関数です。

シグネチャまたは形状

static bool gbt_datelt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の DateADT 型値を指すポインタ。
  • flinfo: 関数マネージャ情報（DirectFunctionCall2 で使用）。
  • **出力**:
  • a < b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な DateADT 型の値を指している必要があります。
  • PostgreSQL の date_lt 関数が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. a と b をそれぞれ DateADT 型としてキャストし、DateADTGetDatum を用いて Datum 型に変換します。 2. DirectFunctionCall2 を介して PostgreSQL コアの date_lt 関数を呼び出します。 3. date_lt の戻り値（bool）を DatumGetBool で取得し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_date.c 内の構造体初期化（111行目付近）において、比較演算テーブルの一部として参照されています。
  • **呼び出し先**: PostgreSQL コアの date_lt 関数。

関連データ構造

  • DateADT: PostgreSQL の日付型表現。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、メモリ保護違反が発生する可能性があります。
  • date_lt 関数がエラーをスローした場合、その例外が伝播します。

保守担当者向けメモ

  • 本関数は btree_gist 拡張における date 型の比較ロジックの最小単位です。同様のパターンで gbt_dategt, gbt_datege, gbt_dateeq, gbt_datele が実装されています。
  • 比較ロジックを変更する場合は、コアの date_lt の挙動と整合性が取れていることを確認してください。
