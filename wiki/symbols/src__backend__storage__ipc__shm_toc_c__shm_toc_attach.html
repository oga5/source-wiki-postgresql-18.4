shm_toc_attach

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_toc.c:63

目的

既存の動的共有メモリ（DSM）セグメント内に配置された目次（Table of Contents）構造体へ接続し、検証を行う。

シグネチャまたは形状

shm_toc *shm_toc_attach(uint64 magic, void *address)

入力と出力

  • **入力**:
  • magic: セグメントの正当性を確認するための識別用マジックナンバー。
  • address: DSM セグメントの先頭アドレス。
  • **出力**:
  • 成功時: shm_toc 構造体へのポインタ。
  • 失敗時: NULL（マジックナンバーが不一致の場合）。

前提条件と不変条件

  • address は有効な shm_toc 構造体が配置されたメモリ領域を指している必要がある。
  • toc->toc_total_bytes は toc->toc_allocated_bytes 以上であること。
  • toc->toc_total_bytes は shm_toc 構造体内の toc_entry 配列のオフセットより大きいこと。

アルゴリズムまたは振る舞い

1. 引数 address を shm_toc 型のポインタとしてキャストする。 2. toc->toc_magic と引数 magic を比較する。不一致の場合は NULL を返して終了する。 3. 構造体の整合性（合計バイト数と割り当て済みバイト数の関係、および toc_entry の配置）を Assert で検証する。 4. 検証を通過した場合、toc ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/session.c
  • src/backend/access/transam/parallel.c
  • src/backend/replication/logical/applyparallelworker.c
  • src/test/modules/test_shm_mq/worker.c
  • **呼び出し先**:
  • 直接的な関数呼び出しはなし（マクロや標準ライブラリ関数を除く）。

関連データ構造

  • shm_toc: 目次を保持するメインの構造体。
  • shm_toc_entry: 目次に格納される各エントリの構造体。

失敗モードと境界条件

  • magic 値が一致しない場合、関数は NULL を返し、呼び出し側でエラーハンドリングが必要となる。
  • Assert が有効なビルドにおいて、メモリレイアウトの不整合（toc_total_bytes の異常など）が検出された場合はプロセスがアボートする。

保守担当者向けメモ

  • 本関数はロックを取得しない。接続後の構造体に対する操作は、shm_toc が提供するスピンロックやメモリバリアの規約に従う必要がある。
