ScalarItem

定義

  • 種別: struct
  • 定義箇所: src/include/statistics/extended_stats_internal.h:27

目的

統計情報の構築過程において、個別のデータ値と、その値が由来するタプルの位置情報をペアで保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	Datum		value;			/* a data value */
	int			tupno;			/* position index for tuple it came from */
} ScalarItem;

入力と出力

  • 入力: Datum 型の値およびタプルのインデックス（int）。
  • 出力: 構造体インスタンスとして、ソート処理や統計計算の入力データとして利用されます。

前提条件と不変条件

  • tupno は、元のデータセット内でのタプルの位置を指し示すインデックスとして機能します。
  • ソート処理において、ScalarItem の配列としてメモリ上に確保され、比較関数を通じて順序付けられます。

アルゴリズムまたは振る舞い

  • 統計構築時のソート処理において、値（value）に基づいた並び替えが行われます。
  • ソート後、tupno を参照することで、ソートされた値が元のどのタプルに属していたかを追跡可能です。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な使用例）: src/backend/commands/analyze.c において、サンプリングされた行のソートや統計計算の準備段階で利用されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、qsort_interruptible 等の標準的なソート関数に渡される比較関数内で、各フィールドが参照されます。

関連データ構造

  • StatsBuildData: 統計計算の入力となるデータセットを保持する構造体であり、その処理過程で ScalarItem が利用されます。

失敗モードと境界条件

  • メモリ確保失敗: palloc を用いて配列を確保するため、メモリ不足時にはエラーが発生します。
  • tupno の整合性: ソート前後で tupno が元のタプルを正しく指していることが前提となります。

保守担当者向けメモ

  • src/backend/commands/analyze.c 内の比較ロジック（value と tupno の両方を使用する箇所）を変更する際は、本構造体のメモリレイアウトと整合性が取れているか確認してください。
  • ソート後の tupno の追跡ロジックに依存しているコードがあるため、構造体のフィールド順序や型を変更する際は注意が必要です。
