wait_on_slots

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:195

目的

複数のデータベース接続を監視し、select() を用いてI/O多重化を行いながら、アクティブな接続からの入力を待機・消費して結果を処理する内部ループを提供します。

シグネチャまたは形状

static bool wait_on_slots(ParallelSlotArray *sa)

入力と出力

  • **入力**: ParallelSlotArray *sa （管理対象のスロット配列へのポインタ）
  • **出力**: bool （処理が正常に完了した場合は true、エラーが発生した場合は false）

前提条件と不変条件

  • sa->slots[i].connection が NULL でないこと（接続済みであること）。
  • select() の制限により、ファイルディスクリプタ数が FD_SETSIZE を超えないこと。
  • 呼び出し時に少なくとも1つの有効な接続が存在すること。

アルゴリズムまたは振る舞い

1. 全スロットのソケットを fd_set に登録し、select_loop を呼び出して入力を待機します。 2. SetCancelConn / ResetCancelConn を用いてキャンセル処理を統合します。 3. select() が入力を検知したスロットに対し、PQconsumeInput を実行します。 4. PQisBusy が偽になるまで PQgetResult をループし、結果を消費します。 5. processQueryResult を呼び出して各結果を処理します。 6. 結果が NULL になった場合、そのスロットをアイドル状態（inUse = false）に設定し、ハンドラをクリアします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/fe_utils/parallel_slot.c 内の ParallelSlotsGetIdle（スロットが全てビジーな場合）
  • src/bin/pg_upgrade/task.c（直接呼び出しの証拠あり）
  • **呼び出し先**:
  • select_loop
  • PQsocket, PQconsumeInput, PQisBusy, PQgetResult
  • SetCancelConn, ResetCancelConn
  • processQueryResult, ParallelSlotClearHandler

関連データ構造

  • ParallelSlotArray: スロットの集合を管理する構造体。
  • ParallelSlot: 個別の接続状態を保持する構造体。
  • PGconn: libpqの接続オブジェクト。

失敗モードと境界条件

  • cancelconn が取得できない場合、false を返します。
  • select_loop が負の値を返した場合、false を返します。
  • processQueryResult がエラーを返した場合、false を返します。
  • ソケットが負の値（無効）である場合、その接続は無視されます。

保守担当者向けメモ

  • 本関数はシングルスレッド環境を前提としています。
  • Windows環境では FD_SETSIZE の制限に注意してください。
  • processQueryResult の戻り値が処理の成否を決定するため、ハンドラの実装には注意が必要です。
