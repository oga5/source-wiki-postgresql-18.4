gbt_bytea_pf_match

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_utils_var.c:178

目的

bytea 型のデータにおいて、指定されたプレフィックス（pf）がクエリ値（query）の先頭部分と一致するかを判定します。

シグネチャまたは形状

static bool gbt_bytea_pf_match(const bytea *pf, const bytea *query, const gbtree_vinfo *tinfo)

入力と出力

  • **入力**:
  • pf: 比較対象のプレフィックス（bytea）。
  • query: 比較対象のクエリ値（bytea）。
  • tinfo: 型情報構造体（本関数内では未使用）。
  • **出力**:
  • bool: プレフィックスが一致する場合は true、それ以外（長さがクエリより長い場合や内容が不一致の場合）は false。

前提条件と不変条件

  • pf および query は有効な varlena 構造体である必要があります。
  • VARSIZE および VARDATA マクロを使用して、内部データ長とデータ本体にアクセスします。

アルゴリズムまたは振る舞い

1. pf と query それぞれのデータ長（ヘッダーサイズを除いた長さ）を算出します。 2. pf の長さが query の長さ以下である場合のみ比較を行います。 3. memcmp を使用して、query の先頭部分が pf と完全に一致するかを判定します。 4. 一致すれば true、不一致または長さ条件を満たさない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_utils_var.c 内の GiST インデックス操作ロジック（ノードの範囲判定やキーの比較処理）から呼び出されています。
  • **呼び出し先**:
  • PostgreSQL の標準マクロ VARSIZE, VARDATA および標準関数 memcmp を使用しています。

関連データ構造

  • bytea: PostgreSQL の可変長バイナリデータ型。
  • gbtree_vinfo: 可変長型を扱うための GiST ユーティリティ用構造体。

失敗モードと境界条件

  • pf の長さが query の長さより大きい場合、一致判定を行わず false を返します。
  • 入力ポインタが NULL の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数は gbt_var_node_pf_match 等のより上位のロジックから、インデックスの枝刈り判定の一部として利用されます。
  • tinfo 引数は現在の実装では使用されていませんが、将来的な型固有の比較ロジック拡張のためにシグネチャに含まれています。
