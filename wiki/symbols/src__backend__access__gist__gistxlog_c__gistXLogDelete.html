gistXLogDelete

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistxlog.c:669

目的

GiSTインデックスにおけるタプル削除操作をWAL（Write-Ahead Log）に記録し、クラッシュリカバリおよびスタンバイサーバーでの整合性維持を可能にする。

シグネチャまたは形状

XLogRecPtr gistXLogDelete(Buffer buffer, OffsetNumber *todelete, int ntodelete, TransactionId snapshotConflictHorizon, Relation heaprel)

入力と出力

  • **入力**:
  • buffer: 削除対象のインデックスページを含むバッファ。
  • todelete: 削除するタプルのオフセット番号配列。
  • ntodelete: 削除するタプルの数。
  • snapshotConflictHorizon: スタンバイサーバーでの競合解決に使用するトランザクションID。
  • heaprel: 関連するヒープリレーション。
  • **出力**:
  • XLogRecPtr: 書き込まれたWALレコードの終了位置（LSN）。

前提条件と不変条件

  • heaprel を用いて論理デコーディングの可否を判定するため、有効なリレーション情報が必要である。
  • XLogBeginInsert を呼び出した後に実行される必要がある。

アルゴリズムまたは振る舞い

1. gistxlogDelete 構造体に、論理デコーディングの可否、snapshotConflictHorizon、削除件数を格納する。 2. XLogBeginInsert を開始し、gistxlogDelete 構造体をWALデータとして登録する。 3. 削除対象のオフセット配列 todelete をWALデータとして登録する（スタンバイ側での競合解決に必須）。 4. XLogRegisterBuffer を使用して対象 buffer を登録する（REGBUF_STANDARD）。 5. XLogInsert を呼び出し、RM_GIST_ID リソースマネージャおよび XLOG_GIST_DELETE オペコードでWALレコードを生成・書き込む。

関連データ構造

  • gistxlogDelete: 削除操作のメタデータを保持する構造体。
  • Buffer: PostgreSQLのバッファ管理構造体。
  • OffsetNumber: インデックスページ内のタプル位置を示す型。

失敗モードと境界条件

  • XLogInsert が失敗した場合、例外が発生し処理が中断される。
  • ntodelete が0の場合の挙動については、本ソースコード内では明示的なガード条件は記述されていない。

保守担当者向けメモ

  • 本関数は、スタンバイサーバーが削除されたタプルに関連するスナップショットの競合を正しく解決できるよう、todelete 配列をWALに含める必要がある。この設計意図を維持すること。
