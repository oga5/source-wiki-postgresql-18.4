customer3

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:96

目的

ECPG（Embedded SQL for C）における構造体の配列をホスト変数として使用し、データベースとのデータ交換を検証するためのテスト用データ構造。

シグネチャまたは形状

struct customer3 {
    struct varchar_3 { int len; char arr[50]; } name;
    int phone;
} custs3[10];

入力と出力

  • **入力**: データベースからフェッチされた name（varchar）および phone（int）の値。
  • **出力**: C 言語の構造体配列 custs3 への格納。

前提条件と不変条件

  • 構造体メンバの順序は、対応する SQL 文のバインド順序と一致している必要がある。
  • custs3 は 10 要素の配列として定義されており、ECPG の操作対象として固定サイズで確保されている。

アルゴリズムまたは振る舞い

ECPG プリプロセッサによって生成されたコード内で、ECPGdo 関数を介して SQL の結果セットを構造体メンバにマッピングする。具体的には、name メンバに対して ECPGt_varchar 型、phone メンバに対して ECPGt_int 型としてバインドが行われる。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:215, 217: ECPGdo 呼び出し時にホスト変数として参照。
  • src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:106, 219, 221: 動的メモリ確保および ECPGdo 呼び出し時に参照。
  • **呼び出し先**:
  • 呼び出し先に関する情報は提供されていません。

関連データ構造

  • varchar_3: customer3 構造体内部で定義された、文字列格納用のネストされた構造体。

失敗モードと境界条件

  • name メンバの arr[50] を超えるデータがデータベースから返された場合、バッファオーバーフローや切り捨てが発生する可能性がある。
  • 配列サイズ（10）を超える行数がフェッチされた場合、メモリ破壊や未定義動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本構造体は ECPG のプリプロセッサテスト用であり、手動編集は行わないこと。
  • 構造体定義を変更する場合は、対応する ECPGdo のバインド引数（型、サイズ、オフセット）が整合しているか確認が必要。
