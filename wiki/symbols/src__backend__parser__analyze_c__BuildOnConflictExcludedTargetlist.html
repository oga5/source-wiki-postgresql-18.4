BuildOnConflictExcludedTargetlist

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:1276

目的

INSERT ... ON CONFLICT 構文において、競合時に参照される仮想的な EXCLUDED テーブルのターゲットリスト（tlist）を構築します。

シグネチャまたは形状

List *BuildOnConflictExcludedTargetlist(Relation targetrel, Index exclRelIndex)

入力と出力

  • **入力**:
  • targetrel: ターゲットとなるリレーション（Relation）。
  • exclRelIndex: EXCLUDED 疑似リレーションの範囲テーブル（RangeTable）インデックス。
  • **出力**:
  • List *: TargetEntry ノードのリスト。

前提条件と不変条件

  • リスト内の各 TargetEntry の resno は、対応するリレーションの属性番号（attno）と一致する必要があります。
  • 削除された列（attisdropped）に対しても、リスト内の位置を維持するためにエントリが必要です。
  • 最後に EXCLUDED.* 参照をサポートするための「行全体（whole-row）」Var エントリが追加されます。

アルゴリズムまたは振る舞い

1. targetrel の全属性をループ処理します。 2. 属性が削除されている場合、makeNullConst を使用してプレースホルダーの Var を作成します。 3. 属性が有効な場合、makeVar を使用して exclRelIndex を参照する Var を作成します。 4. 各 Var を makeTargetEntry でターゲットエントリに変換し、リストに追加します。 5. 最後に、InvalidAttrNumber を持つ行全体参照用の TargetEntry を作成し、リストの末尾に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c
  • src/backend/rewrite/rewriteHandler.c
  • **呼び出し先**:
  • RelationGetNumberOfAttributes, TupleDescAttr, makeNullConst, makeVar, pstrdup, NameStr, makeTargetEntry, lappend

関連データ構造

  • Relation: ターゲットテーブルのメタデータ。
  • TargetEntry: クエリツリーのターゲットリストを構成するノード。
  • Var: 式ツリー内の変数参照。

失敗モードと境界条件

  • 削除された列（attisdropped）が存在する場合、型情報が欠落するため makeNullConst で INT4OID を代入する特殊な処理が行われます。
  • EXCLUDED.* 用の行全体 Var は、通常のターゲットリストの慣習とは異なり InvalidAttrNumber を使用します。これは setrefs.c での参照解決において重要です。

保守担当者向けメモ

  • 本関数が生成するターゲットリストは、通常のクエリ実行用ではなく、ON CONFLICT 句のセマンティクスを解決するための特殊な構造です。変更を加える際は、setrefs.c での参照解決ロジックに影響がないか確認してください。
