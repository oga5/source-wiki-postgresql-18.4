pg_checksum_update

定義

  • 種別: function
  • 定義箇所: src/common/checksum_helper.c:144

目的

チェックサム計算のコンテキストに対し、入力データストリームを適用して内部状態を更新します。

シグネチャまたは形状

int pg_checksum_update(pg_checksum_context *context, const uint8 *input, size_t len);

入力と出力

  • context: 更新対象の pg_checksum_context 構造体へのポインタ。
  • input: 処理対象のデータバッファ。
  • len: input のバイト長。
  • 戻り値: 成功時は 0 を返し、SHA系アルゴリズムで更新処理が失敗した場合は -1 を返します。

前提条件と不変条件

  • context は pg_checksum_init によって初期化済みである必要があります。
  • CHECKSUM_TYPE_NONE の場合、本関数は何も行わず 0 を返します。

アルゴリズムまたは振る舞い

context->type に基づき、以下のいずれかの処理を実行します。 1. CHECKSUM_TYPE_NONE: 何もせず終了します。 2. CHECKSUM_TYPE_CRC32C: COMP_CRC32C マクロを使用して CRC-32C を計算し、context->raw_context.c_crc32c を更新します。 3. CHECKSUM_TYPE_SHA224 / SHA256 / SHA384 / SHA512: pg_cryptohash_update を呼び出し、context->raw_context.c_sha2 を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/bin/pg_combinebackup/backup_label.c
  • src/bin/pg_combinebackup/copy_file.c
  • src/bin/pg_combinebackup/reconstruct.c
  • src/bin/pg_combinebackup/write_manifest.c
  • src/bin/pg_verifybackup/astreamer_verify.c
  • **呼び出し先**:
  • COMP_CRC32C (マクロ)
  • pg_cryptohash_update

関連データ構造

  • pg_checksum_context: アルゴリズムの種類と状態を保持する構造体。

失敗モードと境界条件

  • SHA系アルゴリズムにおいて pg_cryptohash_update が負の値を返した場合、本関数も -1 を返します。
  • CHECKSUM_TYPE_NONE の場合は常に成功（0）となります。

保守担当者向けメモ

  • 本関数はストリーム処理を前提としており、複数回に分けてデータを渡すことが可能です。
  • SHA系アルゴリズムを使用している場合、内部で pg_cryptohash API に依存しているため、関連するリソース管理（pg_checksum_init および pg_checksum_final）が正しく行われていることを確認してください。
