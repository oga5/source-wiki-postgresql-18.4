get_collation_actual_version_builtin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_builtin.c:169

目的

組み込みロケールプロバイダ（C、C.UTF-8、PG_UNICODE_FAST）における照合順序のバージョンを返します。

シグネチャまたは形状

char *get_collation_actual_version_builtin(const char *collcollate)

入力と出力

  • **入力**: collcollate (ロケール名を表す文字列)
  • **出力**: 成功時はバージョン文字列 "1" を返します。無効なロケール名が指定された場合はエラーを発生させます。

前提条件と不変条件

  • 組み込みロケールは memcmp に基づく照合順序を使用しており、将来的に変更される予定がないため、バージョンは常に "1" と定義されています。
  • 照合順序のバージョンは、文字のセマンティクスではなく、ソート順序の変更のみを追跡します。

アルゴリズムまたは振る舞い

1. 引数 collcollate が "C"、"C.UTF-8"、または "PG_UNICODE_FAST" のいずれかと一致するかを strcmp で判定します。 2. 一致した場合は "1" を返します。 3. いずれにも一致しない場合は、ERRCODE_WRONG_OBJECT_TYPE エラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale.c 内の get_collation_actual_version_builtin 呼び出し箇所（collversion の取得に使用）。
  • **呼び出し先**: strcmp（標準ライブラリ）、ereport（PostgreSQL エラー報告マクロ）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • サポートされていないロケール名が渡された場合、ereport によりエラーが発生し、処理が中断されます。

保守担当者向けメモ

  • 本関数は組み込みプロバイダ専用です。ICU 等の外部プロバイダによるロケールについては、別の実装が参照されます。
