BackgroundWorkerShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/bgworker.c:161

目的

バックグラウンドワーカー管理用の共有メモリ領域を確保し、Postmasterのプライベートなワーカーリストの内容を共有メモリ上のスロット配列へ同期させる。

シグネチャまたは形状

void BackgroundWorkerShmemInit(void)

入力と出力

  • 入力: なし（グローバル変数 BackgroundWorkerList および max_worker_processes を参照）
  • 出力: なし（共有メモリ BackgroundWorkerData を初期化）

前提条件と不変条件

  • ShmemInitStruct を介して共有メモリを確保する。
  • Postmasterプロセス（!IsUnderPostmaster）においてのみ、共有メモリの初期化およびリストのコピーが実行される。
  • 子プロセス側では、既に確保済みの共有メモリ領域へのアタッチのみが行われる（found フラグが真であることを期待する）。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、BackgroundWorkerData 構造体を共有メモリ上に割り当てる。 2. Postmasterプロセスの場合:

  • total_slots を max_worker_processes に設定し、カウンタをリセットする。
  • BackgroundWorkerList を走査し、各ワーカー情報を共有メモリの BackgroundWorkerSlot 配列へコピーする。
  • 各スロットの in_use を true に設定し、rw_shmem_slot にインデックスを記録する。
  • 残りの未使用スロットについては in_use を false に設定する。

    • 3. 子プロセスの場合:

  • 共有メモリが既に初期化されていることをアサートする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c 内の CreateSharedMemoryAndSemaphores から呼び出される。
  • 呼び出し先: ShmemInitStruct, dlist_foreach, dlist_container, memcpy 等の共有メモリ管理およびリスト操作関数。

関連データ構造

  • BackgroundWorkerData: 共有メモリ上のメイン管理構造体。
  • BackgroundWorkerSlot: 各ワーカーの状態を保持する共有メモリ上のスロット。
  • BackgroundWorkerList: Postmasterが保持する登録済みワーカーのプライベートリスト。
  • RegisteredBgWorker: BackgroundWorkerList 内の各要素。

失敗モードと境界条件

  • max_worker_processes を超えるスロットへのアクセスは Assert によりガードされている。
  • 共有メモリ確保に失敗した場合、ShmemInitStruct の内部実装に従いパニックが発生する可能性がある。

保守担当者向けメモ

  • 本関数はPostgreSQLの起動シーケンスにおいて、共有メモリが初期化される初期段階で実行される必要がある。
  • in_use フラグを用いたロックレスな同期プロトコルに依存しているため、スロットの更新順序やメモリバリアの整合性に注意すること。
