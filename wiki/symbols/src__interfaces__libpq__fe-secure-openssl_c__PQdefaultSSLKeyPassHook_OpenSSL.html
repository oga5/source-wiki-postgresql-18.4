PQdefaultSSLKeyPassHook_OpenSSL

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-openssl.c:1903

目的

OpenSSL がクライアント証明書の秘密鍵を読み込む際に使用するパスワードを、PGconn 構造体から取得して提供する。

シグネチャまたは形状

int PQdefaultSSLKeyPassHook_OpenSSL(char *buf, int size, PGconn *conn)

入力と出力

  • **入力**:
  • buf: パスワードを格納するバッファ。
  • size: buf のサイズ。
  • conn: 接続情報を保持する PGconn 構造体へのポインタ。
  • **出力**:
  • 戻り値: buf にコピーされたパスワードの長さ（終端文字を含まない）。パスワードがない場合は 0。

前提条件と不変条件

  • conn が有効な場合、conn->sslpassword にパスワード文字列が保持されていることを想定している。
  • buf は size 分の書き込み領域を確保している必要がある。

アルゴリズムまたは振る舞い

1. conn が存在し、かつ conn->sslpassword が設定されているかを確認する。 2. パスワード文字列の長さが size を超える場合、標準エラー出力に警告メッセージを出力する。 3. strncpy を使用して conn->sslpassword を buf にコピーする。 4. buf の末尾を確実にヌル終端（\0）する。 5. コピーした文字列の長さを返す。 6. conn が無効またはパスワードが設定されていない場合は、buf[0] をヌル終端し、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure-openssl.c 内の内部処理および src/interfaces/libpq/fe-secure.c からの参照が確認されている。
  • **呼び出し先**: 標準ライブラリの strlen、strncpy、fprintf を呼び出す。

関連データ構造

  • PGconn: libpq の接続状態を管理する構造体。sslpassword メンバを介してパスワードを保持する。

失敗モードと境界条件

  • size がパスワード長よりも小さい場合、パスワードは切り詰められ、警告が標準エラー出力に出力される。
  • conn が NULL の場合、パスワードは空として扱われる。

保守担当者向けメモ

  • 本関数は libpq-fe.h でエクスポートされており、外部から利用可能なインターフェースである。
  • パスワードの切り詰めが発生した場合の挙動は、セキュリティ上の観点からパスワードの不完全な読み込みを意味するため、呼び出し元でのエラーハンドリングの検討が必要となる場合がある。
