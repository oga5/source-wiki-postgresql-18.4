pgp_load_cipher

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp.c:156

目的

OpenPGP の暗号アルゴリズムコードから、低レベル暗号抽象化レイヤー（px）の暗号実装をロードするファクトリ関数です。

シグネチャまたは形状

int pgp_load_cipher(int code, PX_Cipher **res);

入力と出力

  • **入力**:
  • code: ロード対象の暗号アルゴリズムを示す整数コード。
  • res: ロードされた暗号実装（PX_Cipher）を格納するためのポインタのポインタ。
  • **出力**:
  • 成功時は 0 を返し、res に初期化済みの PX_Cipher インスタンスが設定されます。
  • 失敗時はエラーコード（PXE_PGP_CORRUPT_DATA または PXE_PGP_UNSUPPORTED_CIPHER）を返します。

前提条件と不変条件

  • code に対応する暗号情報が get_cipher_info() によって解決可能である必要があります。
  • res は有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. get_cipher_info(code) を呼び出し、指定されたコードに対応する cipher_info 構造体を取得します。 2. 構造体が見つからない場合、PXE_PGP_CORRUPT_DATA を返します。 3. px_find_cipher() を呼び出し、取得した cipher_info 内の内部名称（int_name）を使用して暗号実装をロードします。 4. px_find_cipher() が成功すれば 0 を返し、失敗すれば PXE_PGP_UNSUPPORTED_CIPHER を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-cfb.c:59
  • **呼び出し先**:
  • get_cipher_info (同ファイル内)
  • px_find_cipher (px.h で定義された低レベル暗号ライブラリのインターフェース)

関連データ構造

  • PX_Cipher: 暗号処理の状態を保持する構造体。
  • cipher_info: 暗号アルゴリズムのメタデータ（内部名称等）を保持する構造体。

失敗モードと境界条件

  • PXE_PGP_CORRUPT_DATA: 指定された code が未知のアルゴリズムである場合に発生します。
  • PXE_PGP_UNSUPPORTED_CIPHER: アルゴリズムコードは存在するが、基盤となる px レイヤーでサポートされていない（またはロードに失敗した）場合に発生します。

保守担当者向けメモ

  • 本関数は px レイヤーへのブリッジとして機能します。新しい暗号アルゴリズムを追加する場合は、cipher_list の更新と併せて本関数が正しく px_find_cipher を呼び出せるか確認してください。
