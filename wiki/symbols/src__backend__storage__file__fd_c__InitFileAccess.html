InitFileAccess

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fd.c:902

目的

PostgreSQLのファイル管理サブシステムにおける仮想ファイル記述子（VFD）キャッシュの初期化を行う。

シグネチャまたは形状

void InitFileAccess(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルな VfdCache および SizeVfdCache を初期化する）

前提条件と不変条件

  • SizeVfdCache が 0 であること（二重初期化を防止するため Assert で検証）。
  • 本関数はバックエンド起動時に一度だけ呼び出される必要がある。

アルゴリズムまたは振る舞い

1. VfdCache 用のメモリを malloc で確保する。メモリ確保に失敗した場合は FATAL エラーを報告する。 2. 確保したメモリ領域を MemSet でゼロクリアする。 3. キャッシュのアンカー（番兵）として VfdCache[0] を設定し、そのファイル記述子を VFD_CLOSED に初期化する。 4. SizeVfdCache を 1 に設定し、キャッシュが初期化済みであることを示す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c:625
  • 呼び出し先: malloc, MemSet, ereport（PostgreSQL内部エラー報告関数）

関連データ構造

  • Vfd: 仮想ファイル記述子を保持する構造体。
  • VfdCache: VFDを管理するグローバルな配列（本関数で初期化される）。
  • SizeVfdCache: キャッシュ内のVFDスロット数。

失敗モードと境界条件

  • メモリ不足: malloc が NULL を返した場合、ereport(FATAL, ...) によりプロセスが終了する。
  • 二重呼び出し: SizeVfdCache が 0 でない状態で呼び出された場合、Assert によりデバッグビルド環境で停止する。

保守担当者向けメモ

  • 本関数は InitTemporaryFileAccess とは分離されている。これは一時ファイルのクリーンアップ処理が、メインのファイルアクセス初期化とは独立して動作する必要があるためである。
  • VfdCache[0] はLRUリングの番兵として機能するため、初期化時に必ず確保される。
