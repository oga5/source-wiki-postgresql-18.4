select_loop

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/task.c:331

目的

複数のファイル記述子（FD）に対する I/O 待機を、割り込みを考慮して安全に実行するためのラッパー関数です。

シグネチャまたは形状

static int select_loop(int maxFd, fd_set *input, fd_set *output);

入力と出力

  • **入力**:
  • maxFd: 監視対象の最大ファイル記述子番号。
  • input: 読み込み監視対象の fd_set。
  • output: 書き込み監視対象の fd_set。
  • **出力**:
  • select() システムコールの戻り値（準備が整った FD の数、またはエラー時の -1）。

前提条件と不変条件

  • maxFd が 0 の場合、直ちに 0 を返します。
  • select() 呼び出しのたびに input および output セットが破壊されるため、関数内で save_input および save_output を用いて状態を復元します。

アルゴリズムまたは振る舞い

1. 引数として渡された input および output の状態を保存します。 2. 無限ループに入り、select() を呼び出します。 3. プラットフォームに応じて以下の割り込み処理を行います。

  • **非 Windows**: errno == EINTR の場合、ループを継続して select() を再試行します。
  • **Windows**: WSAGetLastError() == WSAEINTR の場合、ループを継続して select() を再試行します。

    • 4. 割り込み以外の結果（成功またはエラー）が得られた時点で、その値を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/task.c:405（wait_on_slots 関数内から呼び出される）。
  • **呼び出し先**: select() システムコール。

関連データ構造

  • fd_set: POSIX/Windows の I/O 多重化用ビットマスク構造体。

失敗モードと境界条件

  • maxFd が 0 の場合は即座にリターンします。
  • select() がエラーを返した場合（EINTR 以外）、そのエラーコードを呼び出し元に伝播します。

保守担当者向けメモ

  • 本関数は src/bin/pg_dump/parallel.c や src/fe_utils/parallel_slot.c にも同名の関数が存在しますが、本ページは src/bin/pg_upgrade/task.c 内の定義に限定しています。
  • select() の引数 maxFd + 1 は、監視対象の範囲を正しく指定するために必須です。
