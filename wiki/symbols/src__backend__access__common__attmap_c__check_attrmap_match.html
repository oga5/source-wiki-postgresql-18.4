check_attrmap_match

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/attmap.c:287

目的

2つの TupleDesc 間で属性マッピング（AttrMap）が不要であるか、すなわち変換なしでデータが一致するかを判定する最適化関数です。

シグネチャまたは形状

static bool check_attrmap_match(TupleDesc indesc, TupleDesc outdesc, AttrMap *attrMap)

入力と出力

  • indesc: 入力側のタプル記述子。
  • outdesc: 出力側のタプル記述子。
  • attrMap: 属性番号の対応関係を保持する構造体。
  • 戻り値: 変換が不要な場合は true、変換が必要な場合は false を返します。

前提条件と不変条件

  • indesc と outdesc の属性数（natts）が一致している必要があります。
  • attrMap->attnums は出力側の各列に対応する入力側の列番号（1始まり）を保持しており、値が 0 の場合は削除済み列を指します。

アルゴリズムまたは振る舞い

1. indesc->natts と outdesc->natts を比較し、不一致であれば false を返します。 2. attrMap->maplen 分だけループし、各列を検証します。

  • 入力列に atthasmissing が設定されている場合は、変換が必要とみなして false を返します。
  • attrMap->attnums[i] == (i + 1) であれば、列の対応が維持されているため次の列へ進みます。
  • 削除済み列（attnums[i] == 0）の場合、入力側と出力側の attlen および attalignby が一致していれば変換不要とみなして継続します。

    • 3. 上記のいずれにも該当しない場合、変換が必要と判断し false を返します。 4. 全ての列で検証を通過した場合、true を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/common/attmap.c 内の build_attrmap_by_name_if_req および build_attrmap_by_name から呼び出されます。
  • 呼び出し先: TupleDescCompactAttr マクロを使用して属性情報にアクセスします。

関連データ構造

  • TupleDesc: タプルの構造を定義する記述子。
  • AttrMap: 属性番号の対応表。
  • CompactAttribute: 属性の物理的特性（長さ、アライメント、削除フラグ等）を保持する構造体。

失敗モードと境界条件

  • 属性数が異なる場合、直ちに false を返します。
  • 入力列に欠損値（atthasmissing）がある場合、マッピングの整合性に関わらず変換が必要と判断されます。
  • 削除済み列の比較において、attlen または attalignby が不一致の場合は false を返します。

保守担当者向けメモ

  • 本関数はタプル変換のオーバーヘッドを削減するための重要な最適化ポイントです。
  • 削除済み列の比較ロジックは、物理的なメモリレイアウトの互換性を保証するために不可欠です。変更時は attlen および attalignby の比較条件に注意してください。
