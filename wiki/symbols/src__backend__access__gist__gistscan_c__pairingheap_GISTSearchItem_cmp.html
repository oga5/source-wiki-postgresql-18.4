pairingheap_GISTSearchItem_cmp

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistscan.c:29

目的

GiSTインデックスの検索において、優先度付きキュー（pairingheap）内の探索項目を比較し、距離計算に基づく適切な探索順序を決定する。

シグネチャまたは形状

int pairingheap_GISTSearchItem_cmp(const pairingheap_node *a, const pairingheap_node *b, void *arg)

入力と出力

  • **入力**:
  • a, b: 比較対象の pairingheap_node（実際には GISTSearchItem 型）。
  • arg: IndexScanDesc 型のポインタ。スキャン状態および numberOfOrderBys を参照するために使用される。
  • **出力**:
  • a が b より優先される場合は正の整数、b が優先される場合は負の整数、等価な場合は 0 を返す。

前提条件と不変条件

  • arg として渡される IndexScanDesc は有効なスキャン状態を指している必要がある。
  • GISTSearchItem 内の distances 配列は、numberOfOrderBys に基づいて適切に初期化されていること。

アルゴリズムまたは振る舞い

1. **距離比較**: numberOfOrderBys に基づき、各距離を順に比較する。

  • NULL 値（距離が計算不能）を優先的に扱う（NULL でない方を優先）。
  • float8_cmp_internal を使用して距離を比較し、差がある場合はその結果を返す。

    • 2. **深さ優先探索の維持**: 距離が等しい場合、GISTSearchItemIsHeap を用いて判定を行い、インデックス内の内部ページよりもヒープ項目（リーフノードのデータ）を優先して処理することで、深さ優先探索を促進する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistscan.c 内の pairingheap_allocate 呼び出し時に、比較関数として登録される。
  • **呼び出し先**:
  • float8_cmp_internal（距離の比較）
  • GISTSearchItemIsHeap（項目の種別判定）

関連データ構造

  • GISTSearchItem: 検索キューに格納される項目。
  • IndexScanDesc: スキャン全体のメタデータ。
  • pairingheap: 優先度付きキューの実装。

失敗モードと境界条件

  • numberOfOrderBys が 0 の場合、距離比較はスキップされ、ヒープ項目かどうかの判定のみが行われる。
  • 距離が NULL の場合、NULL ではない項目が優先される。

保守担当者向けメモ

  • 本関数は pairingheap の比較ロジックを定義しており、GiSTの検索効率に直接影響する。
  • 距離が等しい場合のヒープ項目優先順位は、GiSTの探索アルゴリズム（深さ優先）を維持するための重要な不変条件である。
