pgstat_report_analyze

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/activity/pgstat_relation.c:281

目的

ANALYZE実行後の統計情報を共有メモリ上の統計エントリへ反映し、リレーションの生存タプル数および死滅タプル数の推定値を更新する。

シグネチャまたは形状

void pgstat_report_analyze(Relation rel,
                           PgStat_Counter livetuples, PgStat_Counter deadtuples,
                           bool resetcounter, TimestampTz starttime)

入力と出力

  • **入力**:
  • rel: 対象リレーションの構造体。
  • livetuples, deadtuples: ANALYZEにより推定されたタプル数。
  • resetcounter: true の場合、mod_since_analyze をリセットする。
  • starttime: ANALYZE開始時刻。
  • **出力**: なし（副作用として共有メモリ上の統計情報を更新）。

前提条件と不変条件

  • pgstat_track_counts が有効であること。
  • rel がパーティションテーブルではないこと（パーティションテーブルは対象外）。
  • 呼び出し時点で entry_ref が取得可能であること（Assert により検証）。

アルゴリズムまたは振る舞い

1. **統計の補正**: 現在のトランザクション内で実行された挿入・削除操作による影響を差し引く（コミット時の二重計上を防ぐため）。 2. **共有メモリの更新**: pgstat_get_entry_ref_locked を用いて対象リレーションの統計エントリをロックし、推定値を書き込む。 3. **メタデータの更新**: 実行プロセスがオートバキュームワーカか否かに応じ、last_autoanalyze_time または last_analyze_time 等の統計カウンタをインクリメントする。 4. **フラッシュ**: pgstat_flush_io および pgstat_flush_backend を呼び出し、統計情報の永続化を試みる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c
  • src/backend/access/heap/heapam_handler.c
  • **呼び出し先**:
  • pgstat_get_entry_ref_locked
  • pgstat_unlock_entry
  • pgstat_flush_io
  • pgstat_flush_backend
  • AmAutoVacuumWorkerProcess

関連データ構造

  • PgStat_TableXactStatus: トランザクションごとの統計変更分を保持。
  • PgStatShared_Relation: 共有メモリ上のリレーション統計エントリ。
  • PgStat_StatTabEntry: 統計情報の本体。

失敗モードと境界条件

  • pgstat_track_counts が無効な場合、何もせず即座に復帰する。
  • ANALYZEの推定値がトランザクション内の変更分より小さい場合、アンダーフローを防ぐため Max(..., 0) を適用する。
  • パーティションテーブルに対しては統計更新を行わない。

保守担当者向けメモ

  • 本関数はトランザクションのネストを考慮した補正を行うため、rel->pgstat_info->trans の階層構造を破壊しないよう注意すること。
  • 統計のフラッシュは pgstat_flush_backend を通じて行われるため、呼び出し後の即時反映が保証される。
