redirect_elem_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/rmgrdesc_utils.c:49

目的

WAL レコードに含まれるリダイレクト情報（オフセットのペア）を人間が読める文字列形式に変換し、StringInfo バッファへ出力する。

シグネチャまたは形状

void redirect_elem_desc(StringInfo buf, void *offset, void *data)

入力と出力

  • buf (入出力): 出力先の StringInfo バッファ。
  • offset (入力): OffsetNumber の配列（少なくとも2要素）を指すポインタ。
  • data (入力): 使用されない（NULL を渡すことが一般的）。
  • 戻り値: なし。

前提条件と不変条件

  • offset 引数は、少なくとも2つの OffsetNumber を保持するメモリ領域を指している必要がある。
  • buf は初期化済みの有効な StringInfo であること。

アルゴリズムまたは振る舞い

1. offset を OffsetNumber * 型にキャストする。 2. appendStringInfo を使用し、[1つ目のオフセット]->[2つ目のオフセット] という形式の文字列を buf に書き込む。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/rmgrdesc/heapdesc.c:324
  • 呼び出し先: appendStringInfo（PostgreSQL の文字列操作ユーティリティ）

関連データ構造

  • OffsetNumber: storage/off.h で定義される、ページ内のオフセットを指す型。
  • StringInfo: lib/stringinfo.h で定義される、動的文字列バッファ。

失敗モードと境界条件

  • offset が指すメモリ領域が不足している場合、メモリ破壊が発生する可能性がある。
  • buf が NULL の場合、appendStringInfo 内でセグメンテーション違反が発生する。

保守担当者向けメモ

  • 本関数は array_desc からコールバックとして呼び出されることを想定している。
  • 出力形式を変更する場合は、heapdesc.c 等の呼び出し側との整合性に注意すること。
