SecLabelSupportsObjectType

定義

  • 種別: function
  • 定義箇所: src/backend/commands/seclabel.c:36

目的

指定されたデータベースオブジェクト型に対して、セキュリティラベルの付与がサポートされているかどうかを判定する。

シグネチャまたは形状

static bool SecLabelSupportsObjectType(ObjectType objtype)

入力と出力

  • **入力**: ObjectType objtype（判定対象のオブジェクト型）
  • **出力**: bool（サポートされている場合は true、それ以外は false）

前提条件と不変条件

  • switch 文において、すべての ObjectType を網羅的に扱うことが意図されている。
  • default ケースは意図的に排除されており、新しい ObjectType が追加された際にコンパイラが警告を発するよう設計されている。

アルゴリズムまたは振る舞い

1. 引数 objtype を switch 文で評価する。 2. OBJECT_TABLE, OBJECT_FUNCTION, OBJECT_DATABASE など、セキュリティラベルをサポートする特定のオブジェクト型に対しては true を返す。 3. OBJECT_INDEX, OBJECT_TRIGGER, OBJECT_COLUMN（注: OBJECT_COLUMN はサポートリストに含まれる）など、サポート対象外の型に対しては false を返す。 4. 万が一 switch 文の網羅性から外れた値が渡された場合、安全策として false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/seclabel.c 内の SECURITY LABEL コマンド実行ロジック（ExecSecLabelStmt 等）から、対象オブジェクトがラベル付け可能かを確認するために呼び出される。
  • **呼び出し先**: 本関数は他の関数を呼び出さない。

関連データ構造

  • ObjectType: PostgreSQL のシステムカタログやコマンドで扱われるオブジェクト型を定義する列挙型。

失敗モードと境界条件

  • 未定義の ObjectType が渡された場合、コンパイル時の警告対象となるほか、実行時には false が返される。

保守担当者向けメモ

  • 新しいオブジェクト型を PostgreSQL に追加した際は、本関数の switch 文にその型を追加し、セキュリティラベルのサポート可否を明示的に定義すること。
