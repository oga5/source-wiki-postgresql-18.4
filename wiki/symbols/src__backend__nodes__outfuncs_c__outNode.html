outNode

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/outfuncs.c:729

目的

PostgreSQLの内部ノード構造を、デバッグやシリアライズに適したASCII文字列形式へ再帰的に変換します。

シグネチャまたは形状

void outNode(StringInfo str, const void *obj)

入力と出力

  • **入力**:
  • str: 文字列構築用バッファ (StringInfo)。
  • obj: シリアライズ対象のノードポインタ (const void *)。
  • **出力**: なし（str バッファに文字列が追記される）。

前提条件と不変条件

  • check_stack_depth() を呼び出し、再帰呼び出しによるスタックオーバーフローを防止しています。
  • obj が NULL の場合、バッファに <> を書き込みます。

アルゴリズムまたは振る舞い

1. **スタックチェック**: check_stack_depth() を実行。 2. **基本型処理**: List 系、Integer、Float、Boolean、String、BitString、Bitmapset の各型に対して専用の出力関数を呼び出します。 3. **構造体ノード処理**: 上記以外のノード型に対しては、{ を書き込んだ後、nodeTag(obj) に基づいて outfuncs.switch.c 内の switch 文で適切な出力関数へディスパッチし、最後に } を書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/outfuncs.c 内の各フィールド出力関数、および nodeToString 等の公開APIから再帰的に呼び出されます。
  • **呼び出し先**: _outList, _outInteger, _outFloat, _outBoolean, _outString, _outBitString, outBitmapset、および outfuncs.switch.c 内の各ノード型用出力関数。

関連データ構造

  • StringInfo: 文字列構築用バッファ。
  • Node: PostgreSQLのノード構造体（nodeTag を保持）。

失敗モードと境界条件

  • **未認識のノード型**: switch 文の default ケースに該当した場合、WARNING レベルのログを出力し、処理を継続します（エラー終了はしません）。
  • **スタック溢れ**: 非常に深いノードツリーを処理する場合、check_stack_depth() によりスタック保護が働きます。

保守担当者向けメモ

  • 新しいノード型を追加した際は、outfuncs.switch.c に対応するケースを追加する必要があります。
  • readfuncs.c との整合性を保つため、フィールドの出力順序や形式は対応する読み込み関数と一致させる必要があります。
