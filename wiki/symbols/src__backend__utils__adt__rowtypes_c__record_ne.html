record_ne

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rowtypes.c:1282

目的

2つのレコード（RECORD型）が等しくないかどうかを判定し、SQLの不一致演算子（<>）のロジックを提供します。

シグネチャまたは形状

Datum record_ne(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（fcinfo）を介して、比較対象となる2つのレコードデータを受け取ります。
  • **出力**: bool 型の Datum を返します。2つのレコードが等しくない場合は true、等しい場合は false を返します。

前提条件と不変条件

  • record_eq 関数が正常に動作し、比較可能な型定義（TupleDesc）が提供されている必要があります。
  • 内部的に DatumGetBool を使用するため、record_eq が有効なブール値を返すことが前提となります。

アルゴリズムまたは振る舞い

1. record_eq(fcinfo) を呼び出し、2つのレコードが等しいかどうかの判定結果を取得します。 2. その結果を DatumGetBool でブール値に変換します。 3. 変換された値の否定（!）を計算し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: record_eq
  • **呼び出し元**: ソースコード内に明示的な呼び出し元の記述はありません。SQLエンジンが演算子 != または <> を実行する際に呼び出されます。

関連データ構造

  • TupleDesc: レコードの構造定義。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • record_eq がエラーを発生させた場合、本関数もそのエラーを伝播させます。
  • 比較対象のレコードに NULL が含まれる場合、record_eq の仕様に従った比較が行われます。

保守担当者向けメモ

  • 本関数は record_eq のラッパーとして実装されています。比較ロジックに変更を加える場合は、record_eq を修正してください。
  • 呼び出し元に関する情報は、SQLの演算子カタログ（pg_operator）経由で動的に決定されるため、ソースコード上の静的解析では網羅できません。
