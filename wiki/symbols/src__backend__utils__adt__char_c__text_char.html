text_char

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/char.c:203

目的

text 型の値を PostgreSQL の内部的な 1 バイト文字型 ("char") に変換します。

シグネチャまたは形状

Datum text_char(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して text 型のデータを受け取ります。
  • **出力**: Datum 型として 1 バイトの文字（char）を返します。

前提条件と不変条件

  • 入力された text 型データは PG_GETARG_TEXT_PP により取得されます。
  • 変換ルールは charin() と同等ですが、空文字列の扱いが明示的に定義されています。

アルゴリズムまたは振る舞い

1. 入力された text データの先頭バイトを取得します。 2. 入力文字列が \ooo（8進数エスケープ形式）であるかを確認します。

  • \ooo 形式（長さ4バイト、先頭が \、続く3文字が8進数）の場合、その値を計算して result に格納します。

    • 3. それ以外で文字列の長さが 1 バイト以上ある場合、先頭の 1 バイトを result に格納します。 4. 文字列が空の場合、result にヌル文字 ('\0') を格納します。 5. 最終的に PG_RETURN_CHAR(result) を通じて変換結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**: PG_GETARG_TEXT_PP, VARDATA_ANY, VARSIZE_ANY_EXHDR, ISOCTAL, FROMOCTAL, PG_RETURN_CHAR などのマクロを利用しています。

関連データ構造

  • text: 可変長文字列型。
  • Datum: PostgreSQL の汎用データ表現型。

失敗モードと境界条件

  • 空文字列が入力された場合、エラーにはならず '\0' が返されます。
  • \ooo 形式の判定において、\ の後に続く文字が 8 進数でない場合は、通常の文字として先頭バイトが採用されます。

保守担当者向けメモ

  • 本関数は charin() と変換ロジックを共有していますが、空文字列の処理においてのみ挙動が異なります。修正時は charin() との整合性に注意してください。
  • ISOCTAL や FROMOCTAL マクロは、エスケープシーケンスの解析に依存しています。
