pgstat_get_entry_len

定義

  • 種別: function
  • 定義箇所: src/include/utils/pgstat_internal.h:894

目的

指定された統計種別（PgStat_Kind）に対応する、共有メモリ上の統計エントリのデータサイズ（バイト単位）を取得します。

シグネチャまたは形状

static inline size_t pgstat_get_entry_len(PgStat_Kind kind)

入力と出力

  • **入力**: kind (PgStat_Kind) - 統計情報の種類。
  • **出力**: size_t - 共有メモリ上で当該統計エントリが占有するデータサイズ。

前提条件と不変条件

  • kind は有効な統計種別である必要があります。
  • 内部で pgstat_get_kind_info(kind) を呼び出すため、対象の統計種別に対するメタデータ（PgStat_KindInfo）が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. pgstat_get_kind_info(kind) を呼び出し、指定された統計種別に関連付けられたメタデータ構造体を取得します。 2. そのメタデータ構造体内の shared_data_len フィールドを参照し、値を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: pgstat_get_kind_info
  • **呼び出し元**:
  • src/backend/utils/activity/pgstat.c (複数箇所)
  • src/backend/utils/activity/pgstat_shmem.c (複数箇所)
  • ※詳細はツールによる索引結果を参照。

関連データ構造

  • PgStat_Kind: 統計種別を識別する列挙型。
  • PgStat_KindInfo: 統計種別ごとのメタデータを保持する構造体。shared_data_len を定義します。

失敗モードと境界条件

  • 無効な kind が渡された場合の挙動は、pgstat_get_kind_info の実装に依存します（通常はアサーション失敗やセグメンテーション違反の可能性があります）。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、統計システム内の頻繁なアクセスを想定しています。
  • shared_data_len の値は、各統計種別の初期化時に PgStat_KindInfo へ登録されます。サイズ変更が必要な場合は、該当する pgstat_*.c ファイルのメタデータ定義を確認してください。
