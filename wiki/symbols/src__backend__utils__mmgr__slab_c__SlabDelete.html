SlabDelete

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/slab.c:484

目的

SLAB アロケータのメモリコンテキストを破棄し、関連するすべてのメモリブロックおよびコンテキストヘッダー自体を解放する。

シグネチャまたは形状

void SlabDelete(MemoryContext context)

入力と出力

  • **入力**: context (破棄対象の MemoryContext へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • context は有効な SLAB メモリコンテキストである必要がある。
  • 本関数呼び出し後、context が指すメモリ領域は無効となる。

アルゴリズムまたは振る舞い

1. SlabReset(context) を呼び出し、コンテキスト内のすべての SlabBlock を解放する。 2. free(context) を呼び出し、コンテキストヘッダー自体をシステムメモリから解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/mcxt.c:81 において、MemoryContextMethods 構造体の delete_context フィールドに登録されている。
  • **呼び出し先**:
  • SlabReset (内部関数)
  • free (標準ライブラリ関数)

関連データ構造

  • MemoryContext: SLAB アロケータの基底構造体。
  • SlabBlock: SlabReset を通じて解放されるブロック単位の管理構造体。

失敗モードと境界条件

  • context が NULL の場合、標準の free() の挙動に従う（通常は未定義動作または無視）。
  • SlabReset 内部でのメモリ解放処理に依存する。

保守担当者向けメモ

  • 本関数は MemoryContext の破棄インターフェースの一部として実装されている。
  • SlabRealloc はサポートされていないため、本関数による破棄後の再利用は考慮不要である。
