pg_cryptohash_final

定義

  • **種別**: 関数
  • **定義箇所**: src/common/cryptohash.c:171

目的

ハッシュ計算を完了させ、計算結果を呼び出し元が提供したバッファに書き出します。

シグネチャまたは形状

int pg_cryptohash_final(pg_cryptohash_ctx *ctx, uint8 *dest, size_t len);

入力と出力

  • **入力**:
  • ctx: ハッシュ計算の状態を保持する pg_cryptohash_ctx 構造体へのポインタ。
  • dest: 計算結果を格納するための出力バッファ。
  • len: dest バッファのサイズ（バイト単位）。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は -1 を返します。

前提条件と不変条件

  • ctx が NULL の場合、関数は直ちに -1 を返します。
  • len は、選択されたアルゴリズムのダイジェスト長（例: MD5_DIGEST_LENGTH 等）以上である必要があります。

アルゴリズムまたは振る舞い

1. ctx->type を参照し、現在使用中のハッシュアルゴリズムを特定します。 2. len が対象アルゴリズムの必要最小サイズを下回っている場合、ctx->error に PG_CRYPTOHASH_ERROR_DEST_LEN を設定し、-1 を返します。 3. 条件を満たす場合、各アルゴリズム固有の最終計算関数（pg_md5_final, pg_sha1_final, pg_sha224_final 等）を呼び出し、結果を dest に書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/libpq/auth-scram.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/checksum_helper.c
  • **呼び出し先**:
  • pg_md5_final, pg_sha1_final, pg_sha224_final, pg_sha256_final, pg_sha384_final, pg_sha512_final（各アルゴリズムの内部実装関数）。

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態とエラー情報を保持する不透明な構造体。

失敗モードと境界条件

  • **NULL ポインタ**: ctx が NULL の場合に失敗します。
  • **バッファ不足**: len が各アルゴリズムのダイジェスト長より小さい場合、ctx->error が更新され、失敗します。

保守担当者向けメモ

  • 本関数は src/common/cryptohash.c 内の抽象化レイヤーの一部です。新しいアルゴリズムを追加する際は、switch 文にケースを追加し、適切なダイジェスト長チェックを実装してください。
  • エラー発生時の詳細は pg_cryptohash_error 関数を通じて取得可能です。
