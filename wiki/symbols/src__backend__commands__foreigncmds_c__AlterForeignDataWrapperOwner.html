AlterForeignDataWrapperOwner

定義

  • 種別: function
  • 定義箇所: src/backend/commands/foreigncmds.c:285

目的

指定された外部データラッパー（FDW）の所有者を変更する。

シグネチャまたは形状

ObjectAddress AlterForeignDataWrapperOwner(const char *name, Oid newOwnerId)

入力と出力

  • **入力**:
  • name: 変更対象の外部データラッパー名（const char *）。
  • newOwnerId: 新しい所有者の OID（Oid）。
  • **出力**:
  • 変更されたオブジェクトの ObjectAddress を返す。

前提条件と不変条件

  • pg_foreign_data_wrapper システムカタログに対する RowExclusiveLock が保持される。
  • 指定された名前の FDW が存在しない場合、エラーが発生する。

アルゴリズムまたは振る舞い

1. ForeignDataWrapperRelationId を RowExclusiveLock でオープンする。 2. FOREIGNDATAWRAPPERNAME システムキャッシュを使用して、指定された名前の FDW タプルを検索・コピーする。 3. タプルが存在しない場合は ERRCODE_UNDEFINED_OBJECT でエラーを報告する。 4. 取得したタプルから FDW の OID を抽出する。 5. AlterForeignDataWrapperOwner_internal を呼び出し、所有者変更の内部処理を実行する。 6. ObjectAddressSet を使用して、対象オブジェクトの ObjectAddress を構築する。 7. タプルを解放し、リレーションをクローズして、構築した ObjectAddress を返す。

関連データ構造

  • ObjectAddress: 操作対象オブジェクトの識別子。
  • Form_pg_foreign_data_wrapper: pg_foreign_data_wrapper カタログのタプル構造体。

失敗モードと境界条件

  • 指定された名前の FDW が存在しない場合、ereport によりエラー終了する。

保守担当者向けメモ

  • 本関数は所有者変更のメインエントリポイントであり、実際のカタログ更新ロジックは AlterForeignDataWrapperOwner_internal に委譲されている。
