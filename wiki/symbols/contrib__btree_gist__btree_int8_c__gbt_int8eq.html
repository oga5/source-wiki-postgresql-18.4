gbt_int8eq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int8.c:39

目的

2つの int8（64ビット整数）値が等しいかどうかを判定する比較関数です。

シグネチャまたは形状

bool gbt_int8eq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の int64 値へのポインタ。
  • b: 比較対象の int64 値へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • bool: *a == *b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は、有効な int64 型の値を指している必要があります。

アルゴリズムまたは振る舞い

1. ポインタ a および b を int64 型のポインタとしてキャストします。 2. それぞれが指す値をデリファレンスし、比較演算子 == を用いて等価性を評価します。 3. 評価結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int8.c 内の静的構造体定義（tinfo）から参照されています。
  • **呼び出し先**: 直接的な関数呼び出しは行いません。

関連データ構造

  • int64: PostgreSQLの64ビット整数型。

失敗モードと境界条件

  • ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は int8 型の比較ロジックをカプセル化しており、GiSTインデックスの演算子クラス定義の一部として使用されます。
  • flinfo 引数は現在の実装では使用されていませんが、PostgreSQLの関数インターフェース規約に従い保持されています。
