MinMaxMultiOptions

定義

  • 種別: struct
  • 定義箇所: src/backend/access/brin/brin_minmax_multi.c:121

目的

BRIN インデックスの「Multi Min/Max」演算クラスにおいて、各ページ範囲で保持する値の数（valuesPerRange）を制御するためのオプション構造体です。

シグネチャまたは形状

typedef struct MinMaxMultiOptions
{
	int32		vl_len_;        /* varlena ヘッダー */
	int			valuesPerRange; /* 範囲あたりの値の数 */
} MinMaxMultiOptions;

入力と出力

  • **入力**: インデックス作成時や操作時に PG_GET_OPCLASS_OPTIONS() を介して取得されます。
  • **出力**: valuesPerRange フィールドを通じて、インデックスの圧縮アルゴリズムが保持すべき値の制限値を供給します。

前提条件と不変条件

  • vl_len_ は varlena ヘッダーであるため、直接操作してはなりません。
  • valuesPerRange は 0 以外の値を持つことが期待されます（マクロによるチェックあり）。

アルゴリズムまたは振る舞い

  • 本構造体は、インデックスのオプション設定を保持するコンテナとして機能します。
  • init_local_reloptions を用いて初期化され、インデックスのメタデータとして管理されます。
  • 実行時には、brin_minmax_multi_get_values 等の関数で valuesPerRange を参照し、動的な圧縮処理の閾値として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • brin_minmax_multi_get_values
  • PG_GET_OPCLASS_OPTIONS() を使用する関数群
  • init_local_reloptions
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • Ranges: 本オプションで指定された制限値に基づき、実際の値を保持する構造体。

失敗モードと境界条件

  • valuesPerRange が 0 の場合、インデックス操作において予期せぬ動作を引き起こす可能性があるため、マクロによるガード条件が存在します。

保守担当者向けメモ

  • 本構造体は varlena 型として扱われるため、メモリレイアウトやアライメントには注意してください。
  • valuesPerRange の値は、インデックスの圧縮効率とスキャン性能のトレードオフを決定する重要なパラメータです。変更時は brin_minmax_multi.c 内の圧縮ロジックへの影響を考慮してください。
