PLy_spi_execute

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_spi.c:149

目的

Python コードから SQL クエリまたはプリペアドステートメントを実行するためのエントリポイントを提供します。

シグネチャまたは形状

PyObject *PLy_spi_execute(PyObject *self, PyObject *args)

入力と出力

  • **入力**:
  • self: Python モジュールオブジェクト。
  • args: 引数タプル。以下のいずれかの形式を許容します。
  • (char *query, long limit): SQL クエリ文字列とオプションの行数制限。
  • (PyObject *plan, PyObject *list, long limit): プリペアドステートメントオブジェクト、引数リスト、および行数制限。
  • **出力**: 実行結果を格納した PyObject（通常は PLyResultObject）、またはエラー発生時に NULL を返します。

前提条件と不変条件

  • args は PyArg_ParseTuple による解析が可能である必要があります。
  • プリペアドステートメントを実行する場合、plan 引数は is_PLyPlanObject(plan) を満たす必要があります。

アルゴリズムまたは振る舞い

1. まず (query, limit) の形式で引数を解析し、成功すれば PLy_spi_execute_query を呼び出して終了します。 2. 失敗した場合、PyErr_Clear() で Python のエラー状態をリセットします。 3. 次に (plan, list, limit) の形式で引数を解析し、plan が有効な PLyPlanObject であれば PLy_spi_execute_plan を呼び出して終了します。 4. いずれの形式にも適合しない場合、PLy_exception_set を用いて plpy.execute が期待する引数形式ではない旨の例外を設定し、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_plpymodule.c 内の plpy モジュールメソッド定義から参照されています。
  • **呼び出し先**:
  • PLy_spi_execute_query
  • PLy_spi_execute_plan
  • PLy_exception_set

関連データ構造

  • PLyPlanObject: プリペアドステートメントを保持する構造体。
  • PyObject: Python の汎用オブジェクト。

失敗モードと境界条件

  • 引数がクエリ文字列でもプランオブジェクトでもない場合、PLy_exc_error 例外が発生します。
  • PLy_spi_execute_query または PLy_spi_execute_plan 内部で PostgreSQL のエラーが発生した場合、それらはサブトランザクションのメカニズムを通じて Python 例外へ変換されます。

保守担当者向けメモ

  • 本関数は単なるディスパッチャーであり、実際の実行ロジックは PLy_spi_execute_query および PLy_spi_execute_plan に委譲されています。
  • 引数解析の順序（クエリ優先）は固定されています。新しい実行モードを追加する場合はこの解析ロジックを拡張する必要があります。
