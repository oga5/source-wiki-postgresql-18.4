ShutdownSetExpr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execSRF.c:809

目的

集合を返す関数（SRF）の実行状態をクリーンアップし、関連するリソースを解放するコールバック関数です。

シグネチャまたは形状

static void ShutdownSetExpr(Datum arg);

入力と出力

  • **入力**: Datum arg - SetExprState 構造体へのポインタを Datum 型にキャストしたもの。
  • **出力**: なし（void）。

前提条件と不変条件

  • 入力 arg は SetExprState 構造体として有効である必要があります。
  • 実行終了時やエラー発生時に、リソースを確実に解放するために呼び出されます。

アルゴリズムまたは振る舞い

1. arg を SetExprState 型にキャストします。 2. funcResultSlot が存在する場合、ExecClearTuple を呼び出してスロットを解放します。 3. funcResultStore が存在する場合、tuplestore_end を呼び出してストアを破棄し、ポインタを NULL に設定します。 4. setArgsValid フラグを false に設定し、引数状態を無効化します。 5. shutdown_reg フラグを false に設定し、コールバックの登録状態を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、execSRF.c 内の SRF 実行管理フローにおいて、クリーンアップが必要なタイミングで登録・実行される設計です。
  • **呼び出し先**: ExecClearTuple、tuplestore_end。

関連データ構造

  • SetExprState: SRF の実行状態を保持する主要な構造体。

失敗モードと境界条件

  • funcResultSlot や funcResultStore が NULL の場合、それぞれの解放処理はスキップされます。

保守担当者向けメモ

  • 本関数はクエリの中断時にもリソースを解放する重要な役割を担います。SetExprState に新しいリソース管理項目を追加した場合は、本関数内での解放処理を必ず追加してください。
