XLogPrefetcherFree

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogprefetcher.c:389

目的

XLogPrefetcher 構造体に関連付けられたリソースを解放し、メモリをクリーンアップします。

シグネチャまたは形状

void XLogPrefetcherFree(XLogPrefetcher *prefetcher)

入力と出力

  • **入力**: prefetcher (XLogPrefetcher *) - 解放対象の先読み状態構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • prefetcher は有効な XLogPrefetcher 構造体を指している必要があります。
  • 内部の streaming_read および filter_table は、XLogPrefetcherAllocate 等によって適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. lrq_free を呼び出し、先読みキュー（streaming_read）に関連するリソースを解放します。 2. hash_destroy を呼び出し、フィルタテーブル（filter_table）を破棄します。 3. pfree を呼び出し、prefetcher 構造体自体のメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogrecovery.c:1641 にて呼び出されています。
  • **呼び出し先**:
  • lrq_free
  • hash_destroy
  • pfree

関連データ構造

  • XLogPrefetcher: 先読み状態を保持するメイン構造体。
  • LsnReadQueue: streaming_read フィールドが参照するキュー構造。
  • HTAB (推定): filter_table フィールドが参照するハッシュテーブル。

失敗モードと境界条件

  • prefetcher が NULL の場合、pfree 等の内部関数でセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は XLogPrefetcher のライフサイクルの終了時に必ず呼び出す必要があります。
  • XLogPrefetcherAllocate で追加のメモリ割り当てを行った場合は、本関数内でも対応する解放処理を追加してください。
