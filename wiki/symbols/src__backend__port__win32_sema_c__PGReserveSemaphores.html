PGReserveSemaphores

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_sema.c:46

目的

Windows環境におけるPostgreSQLのセマフォ管理機能を初期化し、セマフォハンドルを格納するためのメモリ領域を確保します。

シグネチャまたは形状

void PGReserveSemaphores(int maxSemas)

入力と出力

  • **入力**: maxSemas (int) - 確保するセマフォの最大数。
  • **出力**: なし。

前提条件と不変条件

  • maxSemas に基づいて mySemSet 配列が動的に確保されます。
  • 確保に失敗した場合、elog(PANIC, ...) によりプロセスが異常終了します。
  • プロセス終了時に ReleaseSemaphores が自動的に呼び出されるよう、on_shmem_exit に登録されます。

アルゴリズムまたは振る舞い

1. maxSemas 個の HANDLE を格納するためのメモリを malloc で確保し、静的変数 mySemSet に割り当てます。 2. 確保したメモリが NULL の場合は PANIC を発生させます。 3. 現在のセマフォ数 numSems を 0 に初期化し、最大数 maxSems を設定します。 4. on_shmem_exit を使用して、共有メモリ終了時に ReleaseSemaphores が実行されるよう登録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c からの呼び出しが確認されています。
  • **呼び出し先**: malloc (標準ライブラリ)、on_shmem_exit (PostgreSQL内部API)、ReleaseSemaphores (登録先)。

関連データ構造

  • mySemSet: セマフォハンドルを保持する静的配列。
  • numSems: 現在確保されているセマフォの数。
  • maxSems: 確保可能なセマフォの最大数。

失敗モードと境界条件

  • メモリ不足時: malloc が NULL を返した場合、PANIC レベルのログを出力してプロセスが停止します。

保守担当者向けメモ

  • 本関数はWindows環境専用の実装です。他プラットフォーム（POSIX/System V）にも同名の関数が存在しますが、実装内容は異なります。
  • on_shmem_exit を介したクリーンアップ登録により、プロセス終了時のハンドルリークを防いでいます。
