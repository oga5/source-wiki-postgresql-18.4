oauth_parse

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq-oauth/oauth-curl.c:456

目的

認可サーバーから受信した JSON レスポンスをパースする際の、解析状態とエラー情報を保持するためのコンテキスト構造体です。

シグネチャまたは形状

struct oauth_parse
{
    PQExpBuffer errbuf;             /* JSON_SEM_ACTION_FAILED 用の詳細メッセージ */
    int         nested;             /* ネストレベル（0がトップレベル） */
    const struct json_field *fields; /* フィールド定義配列 */
    const struct json_field *active; /* 現在処理中のフィールド定義へのポインタ */
};

入力と出力

  • **入力**: JSON 解析処理の開始時に、fields に定義済みの json_field 配列をセットして初期化されます。
  • **出力**: 解析中にエラーが発生した場合、errbuf に詳細なエラーメッセージが格納されます。

前提条件と不変条件

  • nested はトップレベルで 0 から開始されます。
  • fields は解析対象の JSON 構造に対応する json_field の配列を指している必要があります。

アルゴリズムまたは振る舞い

  • common/jsonapi.h を利用した JSON 解析プロセスにおいて、現在の解析位置（active）とネスト状態（nested）を追跡します。
  • 解析中に型不一致などの問題が発生した際、errbuf を介して libpq のエラーバッファへ情報を伝播させるための基盤として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq-oauth/oauth-curl.c 内の各解析関数（state 引数として渡される）および oauth_curl.c:833 での初期化。
  • **呼び出し先**: report_type_mismatch 関数が本構造体を引数として受け取り、エラー報告に使用します。

関連データ構造

  • json_field: 解析対象のフィールド定義を保持する構造体。
  • PQExpBuffer: エラーメッセージを蓄積するためのバッファ。

失敗モードと境界条件

  • nested の管理が不適切な場合、JSON の階層構造の追跡に失敗します。
  • fields が無効なポインタである場合、解析処理中にセグメンテーションフォールトが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は common/jsonapi.h を用いた汎用的な JSON 解析ロジックと密結合しています。フィールド定義を追加・変更する際は、json_field 配列の整合性に注意してください。
