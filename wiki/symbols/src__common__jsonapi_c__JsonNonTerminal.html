JsonNonTerminal

定義

  • 種別: enum
  • 定義箇所: src/common/jsonapi.c:113

目的

JSONのインクリメンタル（非再帰的）構文解析において、構文規則上の非終端記号を識別するために使用されます。

シグネチャまたは形状

enum JsonNonTerminal
{
	JSON_NT_JSON = 32,
	JSON_NT_ARRAY_ELEMENTS,
	JSON_NT_MORE_ARRAY_ELEMENTS,
	JSON_NT_KEY_PAIRS,
	JSON_NT_MORE_KEY_PAIRS,
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし

前提条件と不変条件

  • 値は 32 から開始されます。
  • インクリメンタルパーサーの遷移テーブル（td_parser_table）において、状態遷移のスタック要素として利用されます。

アルゴリズムまたは振る舞い

インクリメンタルパーサーがJSON構造を追跡する際、現在の解析位置がどの構文要素（配列の要素、キーと値のペアなど）にあるかを保持します。スタックベースの遷移処理において、次に期待されるトークンや構造を決定するための識別子として機能します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース excerpt には具体的な呼び出し元は記載されていません。
  • 呼び出し先: なし（列挙型のため）。

関連データ構造

  • JsonParserStack: 構文解析のスタック構造体。この列挙値がスタックに積まれることで解析状態を管理します。
  • td_entry: 遷移テーブルの各エントリ。

失敗モードと境界条件

  • 列挙値の範囲外の値をスタックに積んだ場合、パーサーの遷移テーブルが不正な状態となり、解析エラーが発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは「Dragon Book」のアルゴリズム4.3に基づくテーブル駆動型パーサーの一部です。遷移テーブルの変更を行う際は、この列挙値と整合性が取れていることを確認してください。
