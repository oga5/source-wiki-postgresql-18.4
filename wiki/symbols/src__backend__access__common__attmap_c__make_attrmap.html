make_attrmap

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/attmap.c:39

目的

属性番号の対応関係を保持する AttrMap 構造体のメモリを確保し、初期化を行う。

シグネチャまたは形状

AttrMap *make_attrmap(int maplen)

入力と出力

  • **入力**: maplen (int型) - マッピング対象となる列数。
  • **出力**: 初期化された AttrMap 構造体へのポインタ。

前提条件と不変条件

  • palloc0 を使用しているため、メモリ不足時はエラーが発生する。
  • res->attnums 配列は maplen 個の AttrNumber を格納するのに十分なサイズで確保される。

アルゴリズムまたは振る舞い

1. AttrMap 構造体自体のメモリを palloc0 で確保する。 2. 引数 maplen を res->maplen に設定する。 3. maplen に基づいて AttrNumber 配列を palloc0 で確保し、res->attnums に割り当てる。 4. 確保した構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/attmap.c (内部関数 build_attrmap_by_position 等)
  • src/backend/commands/tablecmds.c
  • src/backend/replication/logical/relation.c
  • **呼び出し先**:
  • palloc0 (PostgreSQLメモリ管理関数)

関連データ構造

  • AttrMap: 属性番号の対応関係を保持する構造体。attnums 配列は出力側の各列に対応する入力側の列番号を保持する。

失敗モードと境界条件

  • maplen が 0 の場合、attnums 配列は空の状態で確保される。
  • メモリ不足時には palloc0 がエラーをスローし、呼び出し元へは制御が戻らない。

保守担当者向けメモ

  • 本関数は単なるメモリ確保と初期化を行うユーティリティである。
  • attnums 配列のインデックスは出力側の列位置に対応し、値は入力側の列番号（1始まり）を格納する。値が 0 の場合は削除済み列を意味する。
