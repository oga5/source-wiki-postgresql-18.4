slashUsage

定義

  • 種別: function
  • 定義箇所: src/bin/psql/help.c:147

目的

psql のバックスラッシュコマンド（\?）の一覧を生成し、ユーザーに表示する。

シグネチャまたは形状

void slashUsage(unsigned short int pager);

入力と出力

  • **入力**: pager (ページング機能の有効/無効を制御するフラグ)
  • **出力**: なし（標準出力またはページャへヘルプテキストを出力）

前提条件と不変条件

  • pset 構造体が初期化されており、現在の接続状態や設定（pset.db, pset.popt 等）が有効であること。
  • PQExpBuffer を使用してメモリ上にヘルプテキストを構築するため、十分なメモリが利用可能であること。

アルゴリズムまたは振る舞い

1. PQExpBufferData を初期化し、バックスラッシュコマンドのカテゴリ（General, Help, Query Buffer, Input/Output, Conditional, Informational, Large Objects, Formatting, Connection, Operating System）ごとにヘルプ文字列を HELP0 および HELPN マクロを用いてバッファに追加する。 2. 現在の接続状態（currdb）や設定変数の値（pset.popt.topt 等）を動的に参照し、ヘルプテキスト内の現在値を更新する。 3. 構築したバッファを PageOutput() に渡し、ユーザー設定に従ってページャ経由または直接標準出力へ表示する。 4. ClosePager() を呼び出し、出力処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c:3609
  • src/bin/psql/command.c:3615
  • src/bin/psql/startup.c:709
  • **呼び出し先**:
  • PQdb (libpq)
  • initPQExpBuffer, termPQExpBuffer (内部ユーティリティ)
  • PageOutput, ClosePager (出力制御)
  • HELP0, HELPN (ヘルプ文字列構築マクロ)

関連データ構造

  • PQExpBufferData: ヘルプテキスト構築用バッファ。
  • pset: psql の現在の設定状態を保持するグローバル構造体。

失敗モードと境界条件

  • メモリ不足により PQExpBuffer の初期化や拡張が失敗した場合、プログラムが終了する可能性がある。
  • ページャが指定されている環境で外部コマンドの起動に失敗した場合、標準出力へフォールバックする挙動をとる（PageOutput の実装に依存）。

保守担当者向けメモ

  • ヘルプ内容を更新した際は、必ず ref/psql-ref.sgml（DocBook）と同期させること。
  • HELPN マクロを使用する際は、対応するフォーマット指定子が現在の設定値と正しく一致しているか確認すること。
