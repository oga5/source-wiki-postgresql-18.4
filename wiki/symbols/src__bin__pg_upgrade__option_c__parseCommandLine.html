parseCommandLine

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/option.c:38

目的

pg_upgrade の実行に必要なコマンドライン引数および環境変数を解析し、グローバルな設定構造体（user_opts, old_cluster, new_cluster, os_info 等）を初期化・設定します。

シグネチャまたは形状

void parseCommandLine(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。
  • 環境変数: PGPORTOLD, PGPORTNEW, PGUSER を参照。
  • **出力**:
  • なし（グローバル変数への副作用として設定を反映）。

前提条件と不変条件

  • 実行ユーザーが root 権限である場合、pg_fatal によりプログラムは強制終了します。
  • --help または --version が指定された場合、情報を出力して正常終了します。
  • ポート番号に 0 以下の値が指定された場合、エラーとして終了します。

アルゴリズムまたは振る舞い

1. **初期化**: user_opts のデフォルト値を設定し、プログラム名を取得します。 2. **環境変数適用**: PGPORTOLD, PGPORTNEW, PGUSER から初期値を読み込みます。 3. **引数解析**: getopt_long を使用し、以下のオプションを処理します。

  • ディレクトリパス (-d, -D, -b, -B)
  • ポート番号 (-p, -P)
  • 転送モード (-k, --clone, --copy, --copy-file-range)
  • 実行オプション (-o, -O)
  • その他フラグ (-c, -N, -j, -s, -U, -v, -r 等)

    • 4. **オプション結合**: -o や -O で指定されたオプションは、既存の設定値があれば連結されます。 5. **同期メソッド**: --sync-method が指定された場合、parse_sync_method を呼び出して検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/pg_upgrade.c の main 関数から呼び出されます。
  • **呼び出し先**:
  • getopt_long (標準ライブラリ)
  • get_progname, get_user_info, pg_strdup, pg_free, pg_fatal, psprintf (ユーティリティ関数)
  • usage (同ファイル内)
  • parse_sync_method (外部ユーティリティ)

関連データ構造

  • user_opts: 実行オプションを保持するグローバル構造体。
  • old_cluster, new_cluster: 各クラスタのパスやポート情報を保持する構造体。
  • os_info: 実行ユーザー情報やプログラム名を保持する構造体。
  • log_opts: ログ出力設定を保持する構造体。

失敗モードと境界条件

  • **root 実行**: 実行ユーザーが root の場合、エラー終了します。
  • **無効なポート**: ポート番号に数値以外や 0 以下が指定された場合、エラー終了します。
  • **メモリ不足**: pg_strdup 等のメモリ確保失敗時は内部でエラー処理されます。
  • **引数解析エラー**: getopt_long が未知のオプションを検出した場合、標準的なエラーメッセージを出力して終了します。

保守担当者向けメモ

  • 新しいコマンドラインオプションを追加する場合は、long_options 配列と switch 文の両方を更新する必要があります。
  • old_cluster.pgopts や new_cluster.pgopts の連結処理では、メモリリークを防ぐために古いポインタの解放 (free) が適切に行われているか注意してください。
