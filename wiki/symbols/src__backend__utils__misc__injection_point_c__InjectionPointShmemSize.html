InjectionPointShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/injection_point.c:232

目的

インジェクションポイント機能のために共有メモリ上で必要なバイト数を算出します。

シグネチャまたは形状

Size InjectionPointShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: Size 型のバイト数。--enable-injection-points でビルドされている場合は InjectionPointsCtl 構造体のサイズを返し、それ以外の場合は 0 を返します。

前提条件と不変条件

  • 本関数は、PostgreSQL の共有メモリ初期化プロセスの一部として呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. USE_INJECTION_POINTS マクロが定義されている場合：

  • sz を 0 で初期化します。
  • add_size 関数を用いて InjectionPointsCtl 構造体のサイズを加算します。
  • 計算結果を返します。

    • 2. USE_INJECTION_POINTS が定義されていない場合：

  • 0 を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:150（共有メモリの総サイズ計算時）
  • 呼び出し先: add_size（PostgreSQL のメモリサイズ計算ユーティリティ）

関連データ構造

  • InjectionPointsCtl: 共有メモリ上のインジェクションポイント管理構造体。

失敗モードと境界条件

  • add_size の内部仕様に依存しますが、通常はサイズ計算のオーバーフローを防止する設計となっています。

保守担当者向けメモ

  • 本機能は --enable-injection-points を指定してビルドした場合のみ有効です。ビルド設定が異なる環境では常に 0 を返すため、共有メモリの割り当てには影響しません。
