tstoreDestroyReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tstoreReceiver.c:228

目的

Tuplestore へのタプル格納用 DestReceiver オブジェクトを破棄し、関連するメモリリソースを解放する。

シグネチャまたは形状

static void tstoreDestroyReceiver(DestReceiver *self)

入力と出力

  • **入力**: self (DestReceiver *) - 破棄対象の DestReceiver 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • self は CreateTuplestoreDestReceiver によって割り当てられた有効なメモリ領域を指していること。
  • 本関数呼び出し後、self が指すメモリ領域は無効となるため、呼び出し元は当該ポインタを参照してはならない。

アルゴリズムまたは振る舞い

1. pfree(self) を呼び出し、DestReceiver 構造体（およびその派生構造体 TStoreState）に割り当てられたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（DestReceiver インターフェースの rDestroy メソッドとして呼び出される）。
  • **呼び出し先**: pfree（PostgreSQL のメモリ管理関数）。

関連データ構造

  • DestReceiver: 本関数が操作する抽象インターフェース。
  • TStoreState: DestReceiver を継承したプライベート構造体。

失敗モードと境界条件

  • self が NULL の場合、pfree の仕様に従いエラーとなる可能性がある。

保守担当者向けメモ

  • 本関数は DestReceiver の rDestroy フィールドに登録されるコールバックである。
  • tstoreShutdownReceiver とは異なり、本関数は構造体自体のメモリ解放のみを行う。リソースのクリーンアップ（変換マップの解放など）は tstoreShutdownReceiver が担当する。
