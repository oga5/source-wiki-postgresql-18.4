BarrierArriveAndWait

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/storage/ipc/barrier.c:124

目的

並列処理を行う複数のプロセス間で同期を取り、全参加者が特定の地点に到達するまで待機させる。また、バリア解除時にシリアルな処理を担当するプロセスを1つ選出する。

シグネチャまたは形状

bool BarrierArriveAndWait(Barrier *barrier, uint32 wait_event_info)

入力と出力

  • **入力**:
  • barrier: 同期状態を保持する Barrier 構造体へのポインタ。
  • wait_event_info: 待機時に使用する待機イベント情報。
  • **出力**:
  • bool: 呼び出し元プロセスが「シリアル作業担当」として選出された場合に true を返す。

前提条件と不変条件

  • 呼び出し元プロセスは、対象の barrier に参加（attach）している必要がある。
  • barrier 構造体内の操作は、内部のスピンロックによって保護されている。
  • 待機中、フェーズは start_phase または next_phase のいずれかであり、それ以上進むことはない（呼び出し元が参加しているため）。

アルゴリズムまたは振る舞い

1. **到達処理**: スピンロックを取得し、arrived カウンタをインクリメントする。 2. **フェーズ移行判定**: arrived が participants に達した場合、phase を進め、elected フラグを更新し、ConditionVariableBroadcast で待機中の全プロセスを起床させる。 3. **待機**: 最後の参加者でない場合、ConditionVariableSleep を使用してフェーズが更新されるまでループ待機する。 4. **選出**: 最後に到達したプロセス、または離脱によってフェーズが進んだ際に起床したプロセスのうち、elected フラグが未設定のものが選出され、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeHash.c 内の各所（nodeHash.c:261, 323, 598, 1219, 1338, 1351, 1362, 1419, 1668, 1697, 1732, 3168 など）。
  • **呼び出し先**: SpinLockAcquire, SpinLockRelease, ConditionVariableBroadcast, ConditionVariablePrepareToSleep, ConditionVariableSleep, ConditionVariableCancelSleep。

関連データ構造

  • Barrier: 同期状態（arrived, participants, phase, elected）と条件変数、スピンロックを保持する構造体。

失敗モードと境界条件

  • 参加者数が 0 の動的バリアモードでは、離脱によるフェーズ進行が発生し得る。
  • 待機中に他のプロセスが離脱した場合でも、フェーズが next_phase に進めば待機から復帰する。

保守担当者向けメモ

  • 本関数は、呼び出し元が CPU タイムスライスを保持している間にシリアル作業を開始できるよう、最後に到達したプロセスを優先的に選出する設計となっている。
  • 待機中は条件変数を使用するため、スピンロックを保持したまま長時間待機することはない。
