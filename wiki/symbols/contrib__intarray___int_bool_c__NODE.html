NODE

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int_bool.c:25

目的

クエリ文字列の構文解析中、一時的なスタック構造としてクエリのノード（演算子または値）を保持するために使用される連結リストの要素。

シグネチャまたは形状

typedef struct NODE
{
    int32       type;
    int32       val;
    struct NODE *next;
} NODE;

入力と出力

  • **入力**: palloc を介してメモリが割り当てられ、トークン化されたクエリ情報が格納される。
  • **出力**: 連結リストの各ノードとして、後続の makepol 関数等によるポーランド記法への変換処理に供される。

前提条件と不変条件

  • next ポインタは、リストの終端では NULL である必要がある。
  • type および val は、クエリのトークン種別および値を保持する。

アルゴリズムまたは振る舞い

クエリ文字列を解析する際、トークンを順次読み込み、NODE 構造体を生成してスタック（連結リスト）に積む。その後、このリストを走査して演算子の優先順位に基づいたポーランド記法（ITEM 配列）への変換が行われる。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_int_bool.c 内の構文解析ロジック（makepol 等）から palloc を通じてインスタンス化される。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。
  • **注記**: contrib/ltree/ltxtquery_io.c にも同名の構造体定義が存在するが、本ページは contrib/intarray/_int_bool.c の定義に限定している。

関連データ構造

  • ITEM: 最終的なポーランド記法ツリーを構成する構造体。
  • QUERYTYPE: クエリ全体を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗した場合、PostgreSQL のエラーハンドリングによりアボートされる。
  • スタック深度: 再帰的な処理を行う場合、check_stack_depth() によるスタック保護が適用される。

保守担当者向けメモ

  • 本構造体は _int_bool.c 内での一時的な解析用であり、永続的なクエリ保存には使用されない。
  • contrib/ltree と定義が重複しているため、修正時には両方のモジュールへの影響を考慮すること。
