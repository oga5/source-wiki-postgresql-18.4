GetConnection

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/connection.c:202

目的

リモートのPostgreSQLサーバーへの接続（PGconn）を取得します。接続キャッシュ（ConnectionHash）を管理し、必要に応じて新規接続の確立やトランザクションの同期を行います。

シグネチャまたは形状

PGconn *GetConnection(UserMapping *user, bool will_prep_stmt, PgFdwConnState **state)

入力と出力

  • **入力**:
  • user: 接続先サーバーと認証情報を特定するための UserMapping 構造体。
  • will_prep_stmt: 呼び出し元がプリペアドステートメントを使用するかどうかを示すフラグ。
  • state: 接続ごとの状態（PgFdwConnState）を受け取るためのポインタ（不要な場合は NULL 可）。
  • **出力**:
  • リモートサーバーへの有効な PGconn ポインタ。

前提条件と不変条件

  • ConnectionHash が未初期化の場合は、初回呼び出し時にハッシュテーブルの作成およびトランザクションコールバックの登録が行われます。
  • UserMapping の OID をキーとして接続がキャッシュされます。
  • 接続が「汚染」されている（pgfdw_reject_incomplete_xact_state_change で判定）場合、その接続は使用されません。

アルゴリズムまたは振る舞い

1. **キャッシュ検索**: user->umid をキーに ConnectionHash を検索。 2. **接続確立**: キャッシュに接続がない、または無効化されている場合は make_new_connection を呼び出し接続を確立。 3. **トランザクション同期**: begin_remote_xact を呼び出し、ローカルのトランザクション深度に合わせてリモート側でトランザクションまたはセーブポイントを開始。 4. **エラー回復**: 接続失敗が検出された場合、PG_CATCH ブロック内で接続を破棄し、再接続を試行。 5. **状態更新**: will_prep_stmt フラグをキャッシュエントリに反映し、必要に応じて state を返却。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/postgres_fdw.c 内の各実行関数（fsstate や dmstate の初期化時など）。
  • **呼び出し先**: hash_search, make_new_connection, begin_remote_xact, disconnect_pg_server, process_pending_request 等。

関連データ構造

  • ConnCacheEntry: 接続状態やトランザクション深度を保持するキャッシュエントリ。
  • ConnectionHash: 接続を管理するハッシュテーブル。
  • PgFdwConnState: 接続に関連付けられた内部状態。

失敗モードと境界条件

  • **接続失敗**: make_new_connection が失敗した場合、キャッシュエントリは conn == NULL のまま保持されます。
  • **トランザクションエラー**: リモート側でトランザクション開始に失敗した場合、PG_CATCH ブロックでエラーを捕捉し、接続が CONNECTION_BAD であれば再接続を試みます。再接続後も失敗する場合は例外が再スローされます。
  • **無効化**: invalidated フラグが立っている接続は、トランザクション深度が 0 の時に破棄されます。

保守担当者向けメモ

  • 接続キャッシュは UserMapping 単位であるため、ユーザーごとに認証情報が分離されます。
  • xact_depth によるトランザクション管理は、ローカルのサブトランザクションとリモートのセーブポイントを同期させるために重要です。
