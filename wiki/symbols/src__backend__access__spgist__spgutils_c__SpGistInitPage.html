SpGistInitPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgutils.c:707

目的

SP-GiSTインデックスのページを初期化し、SP-GiST固有のメタデータ構造（SpGistPageOpaqueData）をページ末尾に設定します。

シグネチャまたは形状

void SpGistInitPage(Page page, uint16 f)

入力と出力

  • page (Input/Output): 初期化対象となるメモリ上のページバッファ。
  • f (Input): ページに付与するフラグ（SPGIST_LEAF, SPGIST_NULLS, SPGIST_META 等）。
  • 戻り値: なし。

前提条件と不変条件

  • page は BLCKSZ サイズの有効なメモリ領域を指している必要があります。
  • PageInit を呼び出すため、既存のページ内容は破棄されます。

アルゴリズムまたは振る舞い

1. PageInit を呼び出し、ページヘッダーを初期化します。この際、sizeof(SpGistPageOpaqueData) 分の領域をページ末尾に確保します。 2. SpGistPageGetOpaque を使用して、ページ内の SpGistPageOpaque 構造体へのポインタを取得します。 3. 引数 f を opaque->flags に格納します。 4. opaque->spgist_page_id に定数 SPGIST_PAGE_ID を設定し、SP-GiSTページであることを識別可能にします。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/spgist/spginsert.c: 新規リーフページの作成時に使用。
  • src/backend/access/spgist/spgutils.c: SpGistInitBuffer やメタページ初期化関数から呼び出し。
  • 呼び出し先:
  • PageInit (PostgreSQLストレージ層)

関連データ構造

  • SpGistPageOpaqueData: SP-GiSTインデックスページ末尾に配置されるメタデータ構造。

失敗モードと境界条件

  • page が NULL または不正なメモリ領域である場合、PageInit 内でクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は物理的なページレイアウトを決定する低レベル関数です。フラグの定義（SPGIST_LEAF 等）を変更または追加する場合は、access/spgist_private.h の定義と整合性を確認してください。
