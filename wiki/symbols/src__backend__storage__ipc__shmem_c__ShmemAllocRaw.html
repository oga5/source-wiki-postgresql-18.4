ShmemAllocRaw

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:185

目的

共有メモリ領域から、キャッシュライン境界にアライメントされたメモリチャンクを低レベルで割り当てます。

シグネチャまたは形状

void *ShmemAllocRaw(Size size, Size *allocated_size)

入力と出力

  • **入力**:
  • size: 要求するメモリサイズ（バイト単位）。
  • allocated_size: 実際に割り当てられたサイズ（アライメント調整後）を格納するためのポインタ。
  • **出力**:
  • 成功時: 割り当てられたメモリ領域の先頭アドレス。
  • 失敗時: NULL（共有メモリの残量が不足している場合）。

前提条件と不変条件

  • ShmemSegHdr が初期化済みであること（Assert により検証）。
  • 割り当て後のアドレスがキャッシュライン境界に一致していること（Assert により検証）。
  • 共有メモリは一度割り当てられると解放できないため、本関数は永続的な領域確保に使用されます。

アルゴリズムまたは振る舞い

1. CACHELINEALIGN マクロを使用して、要求された size をキャッシュライン境界に切り上げます。 2. ShmemLock スピンロックを取得し、排他制御を行います。 3. 現在の共有メモリの空きオフセット（ShmemSegHdr->freeoffset）を確認します。 4. 残り容量が十分であれば、オフセットを更新し、ベースアドレスにオフセットを加算してポインタを算出します。 5. 容量不足の場合は NULL を返します。 6. SpinLockRelease でロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/shmem.c 内の ShmemAlloc および ShmemAllocNoError から呼び出されます。
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease（排他制御用）。
  • CACHELINEALIGN（アライメント計算用）。

関連データ構造

  • ShmemSegHdr: 共有メモリセグメントのヘッダー情報（空きオフセットや総サイズを保持）。
  • ShmemBase: 共有メモリ領域のベースアドレス。

失敗モードと境界条件

  • **容量不足**: 共有メモリの総サイズ（totalsize）を超える割り当て要求があった場合、NULL を返します。
  • **アライメント**: 割り当てられたアドレスは常にキャッシュライン境界に揃えられます。

保守担当者向けメモ

  • 本関数は共有メモリ管理の最下層に位置します。呼び出し側は、割り当てられた領域の内部構造（配列の要素など）についても、キャッシュライン境界を意識したパディングを行う必要があります。
  • ShmemLock を保持して処理を行うため、この関数内での処理は最小限に留める必要があります。
