ResetTupleHashTable

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execGrouping.c:279

目的

指定された TupleHashTable が保持するインメモリハッシュテーブルをリセットし、格納されているすべてのエントリを破棄して初期状態に戻します。

シグネチャまたは形状

void ResetTupleHashTable(TupleHashTable hashtable)

入力と出力

  • **入力**: hashtable (TupleHashTable) - リセット対象のハッシュテーブル構造体。
  • **出力**: なし（副作用としてハッシュテーブルの内容が消去されます）。

前提条件と不変条件

  • hashtable は BuildTupleHashTable 等によって適切に初期化されている必要があります。
  • 内部で使用されている lib/simplehash.h のハッシュテーブル構造が有効である必要があります。

アルゴリズムまたは振る舞い

1. hashtable 構造体内の hashtab メンバ（simplehash 実装）に対して、tuplehash_reset マクロを呼び出します。 2. これにより、ハッシュテーブル内の全エントリが解放・クリアされます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c (ハッシュ集約処理)
  • src/backend/executor/nodeRecursiveunion.c
  • src/backend/executor/nodeSetOp.c (重複排除処理)
  • src/backend/executor/nodeSubplan.c
  • **呼び出し先**:
  • tuplehash_reset (マクロ: lib/simplehash.h により生成)

関連データ構造

  • TupleHashTable: 本関数が操作する主要なデータ構造。src/include/nodes/execnodes.h に定義されています。

失敗モードと境界条件

  • hashtable が NULL の場合、セグメンテーション違反が発生する可能性があります。
  • メモリコンテキストの管理状況によっては、本関数呼び出し後にメモリリークが発生しないよう、呼び出し側で適切なコンテキストリセット（tempcxt 等）を併用する必要があります。

保守担当者向けメモ

  • 本関数は simplehash.h で生成された tuplehash_ 系関数のラッパーです。ハッシュテーブルの再利用時に呼び出されます。
  • 実行ノード（nodeAgg.c 等）がこの関数を呼び出す際、メモリコンテキストの切り替えやリセットが適切に行われているか確認してください。
