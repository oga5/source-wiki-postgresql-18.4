ExecForeignScanInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeForeignscan.c:374

目的

並列クエリ実行時において、外部テーブルスキャン（Foreign Scan）に必要な共有メモリ領域（DSM: Dynamic Shared Memory）を初期化し、共有メモリのテーブル・オブ・コンテント（TOC）に登録する。

シグネチャまたは形状

void ExecForeignScanInitializeDSM(ForeignScanState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: 実行中の ForeignScanState 構造体へのポインタ。
  • pcxt: 並列クエリのコンテキストを保持する ParallelContext 構造体へのポインタ。
  • **出力**: なし（副作用として共有メモリの割り当てと登録を行う）。

前提条件と不変条件

  • node->fdwroutine が有効な FdwRoutine を指していること。
  • node->pscan_len に、FDWが必要とする共有メモリサイズが事前に設定されていること。

アルゴリズムまたは振る舞い

1. node->fdwroutine から InitializeDSMForeignScan 関数ポインタを取得する。 2. 関数ポインタが定義されている場合のみ以下の処理を行う:

  • pcxt->toc から node->pscan_len 分の共有メモリ領域を shm_toc_allocate で確保する。
  • FDW固有の初期化関数 InitializeDSMForeignScan を呼び出し、確保した領域を初期化する。
  • shm_toc_insert を使用し、プランノードIDをキーとして確保した領域をTOCに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:492（並列クエリ実行の初期化フェーズ）。
  • **呼び出し先**: node->fdwroutine->InitializeDSMForeignScan（FDW固有の実装）。

関連データ構造

  • ForeignScanState: 外部スキャンの実行状態を保持する構造体。
  • FdwRoutine: FDWが提供する関数ポインタ群。
  • ParallelContext: 並列処理の共有メモリ管理用コンテキスト。

失敗モードと境界条件

  • fdwroutine->InitializeDSMForeignScan が定義されていない場合、本関数は何もしない（安全に終了する）。
  • 共有メモリの割り当て（shm_toc_allocate）が失敗した場合、PostgreSQLの標準的なエラー処理メカニズムにより例外が発生する。

保守担当者向けメモ

  • 本関数は ExecForeignScanEstimate で見積もられたサイズに基づいてメモリを確保します。サイズ計算と初期化ロジックの整合性に注意してください。
  • 実際のデータ構造の初期化はすべてFDW側の実装に委譲されています。
