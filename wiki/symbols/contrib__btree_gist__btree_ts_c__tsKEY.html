tsKEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_ts.c:15

目的

PostgreSQLのGiSTインデックスにおいて、timestamp および timestamptz 型の値を範囲として保持するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	Timestamp	lower;
	Timestamp	upper;
} tsKEY;

入力と出力

  • 本構造体は、GiSTインデックスのノード内におけるキーとして格納・参照されます。
  • lower および upper フィールドは、インデックス対象となるタイムスタンプの範囲の下限と上限を表します。

前提条件と不変条件

  • timestamptz 型を扱う場合、内部的にGMTへ正規化された値が格納されることが前提となります。
  • インデックスの整合性を保つため、通常は lower <= upper の関係が維持されます。

アルゴリズムまたは振る舞い

  • GiSTインデックスの圧縮処理（gbt_ts_compress）において、個別のタイムスタンプ値がこの tsKEY 構造体に変換されます。
  • 検索処理（gbt_ts_consistent）において、クエリ条件とこの構造体が保持する範囲を比較することで、インデックスノードを探索します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/btree_gist/btree_ts.c 内の各サポート関数（gbt_ts_compress, gbt_ts_consistent, gbt_ts_distance 等）から、型キャストを通じて頻繁に参照されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Timestamp: PostgreSQLのタイムスタンプ型。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。
  • Nsrt: ソート処理等で使用されるノード構造体。

失敗モードと境界条件

  • lower と upper の値が逆転している場合、インデックスの検索結果が正しく得られない可能性があります。
  • 浮動小数点数を用いたペナルティ計算時には、オーバーフローを防ぐための安全策（penalty_check_max_float マクロ等）が適用されます。

保守担当者向けメモ

  • 本構造体は btree_utils_num.c で定義された数値型GiSTインデックスの共通ロジックと互換性を持つように設計されています。
  • 構造体のサイズは sizeof(tsKEY) で取得可能であり、メモリ割り当て（palloc）の際にもこのサイズが使用されます。
