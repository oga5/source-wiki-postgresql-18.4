ReplicationSlotCtlData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/slot.h:234

目的

レプリケーションスロットを管理するための共有メモリ上のデータ構造です。システム全体で利用可能なすべてのレプリケーションスロットを保持・管理する基盤となります。

シグネチャまたは形状

typedef struct ReplicationSlotCtlData
{
    ReplicationSlot replication_slots[1];
} ReplicationSlotCtlData;

※ replication_slots は実際には可変長配列として機能するよう設計されています。

入力と出力

  • 入力: 共有メモリへの割り当て時に、スロット数に応じたサイズが計算されます。
  • 出力: 共有メモリ上の ReplicationSlot 配列へのアクセスを提供します。

前提条件と不変条件

  • 共有メモリ上に配置される必要があります。
  • replication_slots 配列の要素数は、サーバー起動時に設定された max_replication_slots に基づいて決定されます。

アルゴリズムまたは振る舞い

  • 共有メモリの初期化時に、offsetof(ReplicationSlotCtlData, replication_slots) を用いて構造体のオフセットが計算され、メモリ領域が確保されます。
  • 全スロットの管理は、この構造体内の配列を介して行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/slot.c 内の初期化処理（ReplicationSlotShmemInit 等）から参照されます。
  • 呼び出し先: 構造体内の各 ReplicationSlot 要素が、スロットの操作（作成、削除、状態更新）を行う各関数によって操作されます。

関連データ構造

  • ReplicationSlot: 本構造体内の配列要素として保持される、個別のレプリケーションスロットの状態管理構造体。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、システムは起動できません。
  • 配列のインデックス範囲外へのアクセスは、メモリ破壊を引き起こすため厳禁です。

保守担当者向けメモ

  • replication_slots[1] は、C言語の仕様上空の構造体に可変長配列を定義できないための回避策です。構造体サイズを計算する際は必ず offsetof を使用してください。
  • 本構造体へのアクセスは、通常 ReplicationSlotControlLock による保護を伴います。
