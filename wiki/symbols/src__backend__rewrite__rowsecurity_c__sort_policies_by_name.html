sort_policies_by_name

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rowsecurity.c:664

目的

行レベルセキュリティ（RLS）ポリシーのリストを、ポリシー名に基づいてソートする。

シグネチャまたは形状

static void sort_policies_by_name(List *policies)

入力と出力

  • **入力**: policies (List *) - ソート対象となる RowSecurityPolicy 構造体のリスト。
  • **出力**: なし（リストはインプレースでソートされる）。

前提条件と不変条件

  • policies リスト内の各要素は RowSecurityPolicy 構造体である必要がある。
  • ソート順序を決定するために row_security_policy_cmp 関数が使用される。

アルゴリズムまたは振る舞い

PostgreSQL の汎用リストソート関数 list_sort を呼び出し、row_security_policy_cmp を比較関数として使用してリストを昇順に並べ替える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/rewrite/rowsecurity.c 内の get_policies_for_relation 関数（612行目および630行目）。
  • **呼び出し先**:
  • list_sort（PostgreSQL 内部ユーティリティ）
  • row_security_policy_cmp（比較関数）

関連データ構造

  • List: PostgreSQL のリスト構造。
  • RowSecurityPolicy: src/include/rewrite/rowsecurity.h で定義されるポリシー情報を保持する構造体。

失敗モードと境界条件

  • policies が NIL（空リスト）の場合、list_sort の仕様に従い何も行われない。
  • policies が指すリストがメモリ上で不正な場合、セグメンテーションフォールトが発生する可能性がある。

保守担当者向けメモ

  • 本関数は static であり、rowsecurity.c 内でのみ使用される。
  • ソート順序を変更したい場合は、row_security_policy_cmp を修正すること。
