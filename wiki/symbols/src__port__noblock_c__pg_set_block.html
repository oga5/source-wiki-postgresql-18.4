pg_set_block

定義

  • 種別: function
  • 定義箇所: src/port/noblock.c:48

目的

指定されたソケット（ファイル記述子）をブロッキングモードへ設定するポータブルなラッパー関数です。

シグネチャまたは形状

bool pg_set_block(pgsocket sock)

入力と出力

  • **入力**: sock (pgsocket 型) - 操作対象のソケットまたはファイル記述子。
  • **出力**: bool - 操作が成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • pgsocket 型が c.h で適切に定義されていること。
  • プラットフォームに応じて、POSIX 環境では <fcntl.h> が利用可能であること。

アルゴリズムまたは振る舞い

プラットフォームに応じて以下の処理を行います。

  • **POSIX 環境 (!defined(WIN32))**:

    • 1. fcntl(sock, F_GETFL) で現在のファイルステータスフラグを取得します。取得に失敗した場合は false を返します。 2. fcntl(sock, F_SETFL, (flags & ~O_NONBLOCK)) を実行し、O_NONBLOCK フラグを解除してブロッキングモードへ移行します。 3. fcntl の結果が -1 であれば false を返し、それ以外は true を返します。

  • **Windows 環境 (WIN32)**:

    • 1. ioctlsocket(sock, FIONBIO, &ioctlsocket_ret) を呼び出します（ioctlsocket_ret は 0）。 2. ioctlsocket が 0 を返せば成功とみなし true を、それ以外は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には具体的な呼び出し元の関数名は記載されていません。src/include/port.h:45 に外部宣言が存在します。
  • **呼び出し先**:
  • POSIX: fcntl
  • Windows: ioctlsocket

関連データ構造

  • pgsocket: ソケットを表現する型（src/include/c.h で定義）。

失敗モードと境界条件

  • **POSIX**: fcntl によるフラグ取得または設定が失敗した場合（戻り値が -1）に false を返します。
  • **Windows**: ioctlsocket が非ゼロの値を返した場合に false を返します。

保守担当者向けメモ

  • 本関数は WIN32 マクロによる条件コンパイルで実装が完全に分岐しています。
  • 戻り値の判定基準（POSIX は -1、Windows は非ゼロ）が異なる点に注意してください。
