filehash_init

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/filemap.c:196

目的

pg_rewind がファイル同期の意思決定に使用するグローバルなハッシュテーブル filehash を初期化します。

シグネチャまたは形状

void filehash_init(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 filehash を初期化）

前提条件と不変条件

  • filehash が初期化される前に呼び出される必要があります。
  • FILEHASH_INITIAL_SIZE がハッシュテーブルの初期サイズとして使用されます。

アルゴリズムまたは振る舞い

filehash_create を呼び出し、FILEHASH_INITIAL_SIZE を指定してハッシュテーブルを生成し、その結果をグローバル変数 filehash に代入します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_rewind/pg_rewind.c:468
  • 呼び出し先: filehash_create（lib/simplehash.h で定義されたマクロ生成関数）

関連データ構造

  • filehash: simplehash.h を用いて生成されたファイルパスをキーとするハッシュテーブル。
  • file_entry_t: ハッシュテーブルに格納される各ファイルのメタデータ構造体。

失敗モードと境界条件

  • メモリ不足時には filehash_create 内部でエラーが発生し、プログラムが終了する可能性があります。

保守担当者向けメモ

  • 本関数は filemap.c 内の他のファイル操作関数が filehash を利用する前の必須の初期化ステップです。
  • simplehash.h の仕様変更や FILEHASH_INITIAL_SIZE の調整が必要な場合は、本関数および filemap.h の定義を確認してください。
