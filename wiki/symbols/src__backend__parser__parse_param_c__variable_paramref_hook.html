variable_paramref_hook

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_param.c:131

目的

クエリ解析中に遭遇したパラメータ参照（ParamRef）を Param ノードへ変換し、必要に応じてパラメータ型管理配列を動的に拡張・初期化します。

シグネチャまたは形状

static Node *variable_paramref_hook(ParseState *pstate, ParamRef *pref)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態（ParseState）。p_ref_hook_state に VarParamState が格納されていることを前提とします。
  • pref: 解析対象のパラメータ参照ノード（ParamRef）。
  • **出力**:
  • 変換された Param ノード（Node * 型として返却）。

前提条件と不変条件

  • pstate->p_ref_hook_state は VarParamState 構造体へのポインタであること。
  • pref->number が有効な範囲（1 以上かつ MaxAllocSize 以内）であること。
  • パラメータの型解決において、paramtypmod は常に -1 に設定されます。

アルゴリズムまたは振る舞い

1. **範囲チェック**: パラメータ番号が不正な場合、エラーを報告します。 2. **配列拡張**: パラメータ番号が現在の管理配列サイズを超える場合、repalloc0_array または palloc0_array を使用して VarParamState 内の paramTypes 配列を拡張し、numParams を更新します。 3. **型初期化**: 対象スロットの型が InvalidOid の場合、UNKNOWNOID で初期化します。 4. **特殊ケース処理**: プロシージャ呼び出しの引数（EXPR_KIND_CALL_ARGUMENT）かつ型が VOIDOID の場合、UNKNOWNOID に強制変換します。 5. **ノード生成**: Param ノードを作成し、PARAM_EXTERN として設定。型情報および照合順序（get_typcollation）を付与して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_param.c 内の setup_parse_variable_parameters 関数において、pstate->p_paramref_hook に関数ポインタとして登録されます。
  • **呼び出し先**:
  • palloc0_array / repalloc0_array（メモリ管理）
  • ereport（エラー報告）
  • get_typcollation（型照合順序の取得）
  • makeNode（ノード生成）

関連データ構造

  • VarParamState: パラメータの型配列と数を保持する構造体。
  • ParamRef: 入力となる構文解析ノード。
  • Param: 出力となる実行時パラメータノード。

失敗モードと境界条件

  • **パラメータ番号の範囲外**: paramno <= 0 または MaxAllocSize を超える場合、ERRCODE_UNDEFINED_PARAMETER でエラー終了します。
  • **メモリ不足**: 配列拡張時のメモリ確保に失敗した場合、PostgreSQL のメモリ管理機構によりエラーが発生します。

保守担当者向けメモ

  • 本関数は variable_coerce_param_hook と連携して型推論を行います。
  • paramtypmod を -1 に固定しているのは、実行時の型チェックを確実に行うための設計上の意図です。
  • UNKNOWNOID のまま解析が終了した場合、後続のフェーズでエラーとなる可能性があるため注意してください。
