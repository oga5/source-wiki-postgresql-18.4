ResourceOwnerDesc

定義

  • 種別: struct
  • 定義箇所: src/include/utils/resowner.h:91

目的

リソースオーナーが管理する特定のリソース種別に対する、解放ロジック、デバッグ情報、および解放順序を定義するための記述子です。

シグネチャまたは形状

typedef struct ResourceOwnerDesc
{
    const char *name;
    ResourceReleasePhase release_phase;
    ResourceReleasePriority release_priority;
    void (*ReleaseResource) (Datum res);
    char *(*DebugPrint) (Datum res);
} ResourceOwnerDesc;

入力と出力

  • **入力**: ReleaseResource および DebugPrint 関数は、管理対象リソースを識別する Datum 型の値を引数として受け取ります。
  • **出力**: DebugPrint はリソースの状態を説明する文字列を返します。ReleaseResource は戻り値を持ちません。

前提条件と不変条件

  • ReleaseResource は、リソースオーナーの解放時、または ResourceOwnerReleaseAllOfKind() の呼び出し時に実行されます。
  • ReleaseResource が呼び出されると、対象リソースは自動的にオーナーから削除されるため、コールバック内で ResourceOwnerForget を呼ぶ必要はありません。

アルゴリズムまたは振る舞い

1. **順序付け**: release_phase および release_priority に基づき、リソースの解放順序が決定されます。 2. **解放処理**: リソースオーナーのクリーンアップ時、定義された優先度に従って ReleaseResource が順次実行されます。 3. **デバッグ**: リソースがコミット時に解放されなかった場合、DebugPrint が呼び出され、警告メッセージとして出力されます。DebugPrint が NULL の場合は、汎用的な形式 [resource name]: %p が使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ResourceOwnerRelease および ResourceOwnerReleaseAllOfKind が本構造体のコールバックを呼び出します。
  • **呼び出し先**: 以下の各モジュールで静的インスタンスとして定義されています。
  • buffer_io_resowner_desc, buffer_pin_resowner_desc (bufmgr.c)
  • file_resowner_desc (fd.c)
  • catcache_resowner_desc, catlistref_resowner_desc (catcache.c)
  • planref_resowner_desc (plancache.c)
  • dsm_resowner_desc (dsm.c)
  • wait_event_set_resowner_desc (waiteventset.c)
  • jit_resowner_desc (llvmjit.c)
  • tupdesc_resowner_desc (tupdesc.c)
  • ossldigest_resowner_desc, osslcipher_resowner_desc (pgcrypto/openssl.c)

関連データ構造

  • ResourceOwner: リソースを保持する不透明なデータ構造。
  • ResourceReleasePhase: 解放フェーズを定義する列挙型。
  • ResourceReleasePriority: 同一フェーズ内での解放順序を定義する数値。

失敗モードと境界条件

  • ReleaseResource コールバックは、コミットまたはアボート後に実行されるため、失敗が許されない非クリティカルなクリーンアップ処理のみを実装する必要があります。

保守担当者向けメモ

  • 新しいリソース型を追加する場合は、既存の RELEASE_PRIO_* を参照し、適切なフェーズと優先度を割り当ててください。
  • DebugPrint を実装することで、リソースリーク発生時のデバッグが容易になります。
