目的

Unicode文字の分類カテゴリおよびプロパティ情報を保持するためのデータ構造です。PostgreSQLのUnicode処理において、文字の属性を高速に判定するための静的テーブルの要素として使用されます。

シグネチャまたは形状

typedef struct
{
	uint8		category;
	uint8		properties;
} pg_unicode_properties;

入力と出力

本シンボルはデータ構造の定義であり、関数ではありません。

  • category: Unicodeカテゴリを保持する8ビット整数。
  • properties: Unicodeプロパティ（Alphabetic, Lowercase, Uppercase等）を表現するビットフラグを保持する8ビット整数。

前提条件と不変条件

  • 本構造体は静的データテーブルの一部として定義されます。
  • properties フィールドの各ビットは、PG_U_PROP_* マクロによって定義されたフラグに対応します。

アルゴリズムまたは振る舞い

本構造体自体はロジックを持ちません。この構造体を用いたテーブル（unicode_opt_ascii 等）に対し、コードポイントをインデックスとして参照するか、二分探索を行うことで、特定の文字のカテゴリやプロパティを特定する処理フローで使用されます。
