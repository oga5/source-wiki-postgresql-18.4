PgAioUringCaps

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/aio/method_io_uring.c:107

目的

Linux カーネルの io_uring 機能、特に io_uring_queue_init_mem() のサポート状況を保持し、実行時の機能判定結果を記録するために使用されます。

シグネチャまたは形状

typedef struct PgAioUringCaps
{
    bool        checked;
    int         mem_init_size;
} PgAioUringCaps;

入力と出力

本構造体は関数ではなくデータ型であるため、直接的な入力や出力はありません。ただし、mem_init_size フィールドは、サポートされていない場合に -1 を保持する仕様となっています。

前提条件と不変条件

  • checked が true の場合、mem_init_size には判定結果（サポート状況または -1）が格納されているものとみなされます。

アルゴリズムまたは振る舞い

本構造体は、pgaio_uring_check_capabilities 関数によって初期化および更新されることを想定しています。この判定により、不要なメモリマッピングを回避し、パフォーマンスを最適化する判断材料として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/aio/method_io_uring.c:121 にて pgaio_uring_caps という静的変数としてインスタンス化されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • PgAioUringContext: バックエンドごとの io_uring インスタンスを管理する構造体。

失敗モードと境界条件

  • mem_init_size が -1 の場合、カーネルが io_uring_queue_init_mem() をサポートしていないことを示します。

保守担当者向けメモ

  • 本構造体は method_io_uring.c 内の静的変数 pgaio_uring_caps を通じて管理されます。機能判定ロジックを変更する際は、この構造体のフィールド定義と整合性を取る必要があります。
