heap_xlog_insert

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_xlog.c:416

目的

ヒープページへのタプル挿入操作をWALリプレイ時に再現し、ページ状態を更新する。

シグネチャまたは形状

static void heap_xlog_insert(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record (リプレイ対象のWALレコード)
  • **出力**: なし（副作用としてバッファおよび可視性マップ、FSMを更新）

前提条件と不変条件

  • xlrec->flags に XLH_INSERT_ALL_FROZEN_SET が含まれていないこと（ヒープ挿入パスではフリーズ処理を行わないため）。
  • ページが初期化される場合（XLOG_HEAP_INIT_PAGE）、XLogInitBufferForRedo を介してページが再構築される。

アルゴリズムまたは振る舞い

1. **可視性マップの更新**: XLH_INSERT_ALL_VISIBLE_CLEARED フラグがある場合、対象ブロックの可視性マップビットをクリアする。 2. **バッファの取得**: XLOG_HEAP_INIT_PAGE ならページを初期化し、それ以外は XLogReadBufferForRedo でバッファを取得する。 3. **タプルの復元**:

  • WALレコードからタプルヘッダーとデータを読み取る。
  • HeapTupleHeader を構築し、Xmin、Cmin、t_ctid を設定する。
  • PageAddItem を使用してページにタプルを挿入する。

    • 4. **ページ状態の更新**:

  • ページLSNを更新し、バッファをダーティとしてマークする。
  • XLH_INSERT_ALL_VISIBLE_CLEARED フラグがある場合、ページ上の PD_ALL_VISIBLE ビットをクリアする。

    • 5. **FSMの更新**: ページ内の空き容量が BLCKSZ / 5 未満になった場合、XLogRecordPageWithFreeSpace を呼び出しフリースペースマップを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: heap_redo（src/backend/access/heap/heapam_xlog.c:1193 での呼び出しが確認されている）。
  • **呼び出し先**: XLogRecGetData, XLogRecGetBlockTag, visibilitymap_pin, visibilitymap_clear, XLogInitBufferForRedo, XLogReadBufferForRedo, PageAddItem, PageSetLSN, MarkBufferDirty, XLogRecordPageWithFreeSpace 等。

関連データ構造

  • xl_heap_insert: WALレコードのデータ構造。
  • HeapTupleHeaderData: ページ上のタプルヘッダー構造。
  • Buffer: バッファマネージャの管理単位。

失敗モードと境界条件

  • PageGetMaxOffsetNumber(page) + 1 < xlrec->offnum: ページ内のオフセット番号が不正な場合に PANIC。
  • PageAddItem が InvalidOffsetNumber を返した場合に PANIC。

保守担当者向けメモ

  • 本関数は物理的なページ操作に特化している。論理的な整合性はWAL生成側（heapam.c）で担保されている前提である。
  • ページがフルページイメージから復元された場合、FSMの更新はスキップされる。
  • 新しいWALレコードタイプ追加時は、heap_mask 関数での非決定的なビットの除外を忘れないこと。
