locale_map

定義

  • 種別: struct
  • 定義箇所: src/port/win32setlocale.c:39

目的

Windows 環境におけるロケール名の文字列置換ルールを定義するためのデータ構造です。setlocale() に渡す引数や、その戻り値に対して正規化を行う際に使用されます。

シグネチャまたは形状

struct locale_map {
    const char *locale_name_start;
    const char *locale_name_end;
    const char *replacement;
};

入力と出力

  • locale_name_start: 検索対象の開始文字列。
  • locale_name_end: 検索対象の終了文字列（NULL の場合は単一文字列として扱われる）。
  • replacement: 置換後の文字列。

前提条件と不変条件

  • locale_name_end が指定されている場合、対象のロケール名には locale_name_start と locale_name_end の両方が含まれている必要があります。
  • 範囲指定（start と end）がある場合、その間の文字列が replacement に置換されます。

アルゴリズムまたは振る舞い

本構造体は map_locale 関数によって参照されます。map_locale はこの構造体の定義に基づき、ロケール名に対して「簡易的な正規表現検索」のような置換処理を実行します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/win32setlocale.c 内の locale_map_argument および locale_map_result 配列の定義において使用されています。
  • 呼び出し先: map_locale 関数が本構造体を引数として受け取り、ロケール名の変換処理を行います。

関連データ構造

  • locale_map_argument: 入力ロケール名変換用の locale_map 配列。
  • locale_map_result: 出力ロケール名変換用の locale_map 配列。

失敗モードと境界条件

  • locale_name_end が NULL の場合、単純な文字列置換として動作します。
  • 範囲置換において対象文字列が見つからない場合、置換は行われません。

保守担当者向けメモ

  • 本構造体は静的配列の定義に使用されるため、スレッドセーフな環境での利用には注意が必要です。
  • 範囲置換機能は、ロケール名の一部のみを動的に書き換える必要がある場合に利用されます。
