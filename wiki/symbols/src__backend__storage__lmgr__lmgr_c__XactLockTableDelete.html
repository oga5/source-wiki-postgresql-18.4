XactLockTableDelete

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lmgr.c:638

目的

指定されたトランザクションID（XID）に関連付けられたトランザクションロックを解放する。

シグネチャまたは形状

void XactLockTableDelete(TransactionId xid)

入力と出力

  • **入力**: xid (解放対象のトランザクションID)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • 指定された xid に対して、呼び出し元が既に排他ロック（ExclusiveLock）を保持している必要がある。

アルゴリズムまたは振る舞い

1. SET_LOCKTAG_TRANSACTION マクロを使用して、引数の xid からロックタグ（LOCKTAG）を構築する。 2. LockRelease 関数を呼び出し、構築したタグに対して ExclusiveLock を解放する。第3引数 recycle は false が指定される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c:5172 (トランザクション終了処理の一部として呼び出される)
  • **呼び出し先**: LockRelease (ロックマネージャの低レベル関数)

関連データ構造

  • LOCKTAG: ロック対象を識別するための構造体。本関数ではトランザクションIDを識別子として使用する。
  • TransactionId: ロック対象のトランザクションを特定する識別子。

失敗モードと境界条件

  • 呼び出し元が対象のロックを保持していない場合、LockRelease 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は lmgr.c 内のラッパーであり、実質的なロック解放処理は LockRelease に依存している。
  • トランザクション終了時のクリーンアップ処理として重要な役割を果たすため、変更時はトランザクション状態管理との整合性に注意すること。
