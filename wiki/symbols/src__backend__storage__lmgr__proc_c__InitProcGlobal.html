InitProcGlobal

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/proc.c:191

目的

PostgreSQLのバックエンドプロセス管理に必要な共有メモリ構造体（ProcGlobal）および全プロセス用 PGPROC 配列の初期化を行う。

シグネチャまたは形状

void InitProcGlobal(void)

入力と出力

  • 入力: なし（グローバル変数 MaxBackends, max_prepared_xacts 等を参照）
  • 出力: なし（共有メモリ上に ProcGlobal ヘッダーおよび PGPROC 配列を割り当て、初期化する）

前提条件と不変条件

  • 共有メモリの初期化フェーズで呼び出される必要がある。
  • ShmemInitStruct を使用するため、共有メモリが利用可能な状態であること。
  • ProcGlobal が未初期化であることを前提とする（Assert(!found) による検証）。

アルゴリズムまたは振る舞い

1. **ProcGlobalの確保**: 共有メモリ上に PROC_HDR 構造体を割り当て、各種フリーリスト（freeProcs 等）を初期化する。 2. **PGPROC配列の確保**: 全バックエンド数分（MaxBackends + NUM_AUXILIARY_PROCS + max_prepared_xacts）の PGPROC 構造体を一括確保し、ゼロクリアする。 3. **補助配列の割り当て**: トランザクションID、サブトランザクション状態、ステータスフラグ用の配列を PGPROC 配列の直後に配置する。 4. **Fast-Pathロック領域の確保**: 各バックエンド用のFast-Pathロック情報を格納する共有メモリ領域を確保する。 5. **各PGPROCの初期化**:

  • Fast-Pathロック用ポインタの割り当て。
  • セマフォ、ラッチ、fpInfoLock の初期化（準備済みトランザクション用を除く）。
  • プロセスタイプに応じたフリーリストへの登録（dlist_push_tail）。
  • ロック待ち行列およびアトミック変数の初期化。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c 内の共有メモリ初期化処理から呼び出される。
  • 呼び出し先: ShmemInitStruct, dlist_init, pg_atomic_init_u32, PGSemaphoreCreate, InitSharedLatch, LWLockInitialize 等。

関連データ構造

  • PROC_HDR: プロセス管理のグローバルヘッダー。
  • PGPROC: 個々のバックエンドプロセスの状態を保持する構造体。
  • dlist: プロセスを管理する双方向リスト。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct がパニックを引き起こす可能性がある。
  • TotalProcs の計算やメモリ配置において、想定サイズを超過した場合は Assert により停止する。

保守担当者向けメモ

  • PGPROC 配列は MaxBackends 等の構成パラメータに依存するため、共有メモリサイズ計算と整合性を保つ必要がある。
  • fpPtr を用いたメモリ配置は、Fast-Pathロックのメモリレイアウトを決定する重要な処理である。
  • 準備済みトランザクション（Prepared Xacts）用の PGPROC は、通常のプロセスとは異なりセマフォ等の初期化がスキップされる点に注意。
