tsearch_readline_callback

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_locale.c:184

目的

全文検索（tsearch）の設定ファイル読み込み中にエラーが発生した際、エラーコンテキスト（errcontext）にファイル名と行番号を付与し、デバッグ情報を補完する。

シグネチャまたは形状

static void tsearch_readline_callback(void *arg);
  • arg: tsearch_readline_state 構造体へのポインタ。

入力と出力

  • **入力**: arg を通じて tsearch_readline_state 構造体を受け取る。
  • **出力**: なし（戻り値は void）。副作用として errcontext を設定する。

前提条件と不変条件

  • arg は tsearch_readline_state 型の有効なポインタであること。
  • stp->filename および stp->lineno が適切に初期化されていること。

アルゴリズムまたは振る舞い

1. 引数 arg を tsearch_readline_state * にキャストする。 2. stp->curline が存在する場合、ファイル名、行番号、および現在の行内容を errcontext に出力する。 3. stp->curline が存在しない（読み込み中等の）場合は、ファイル名と行番号のみを errcontext に出力する。

  • エンコーディング違反等のエラーを考慮し、不正なエンコーディングを含む可能性のある行内容は、状況に応じて出力対象から除外する設計となっている。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/ts_locale.c 内の tsearch_readline_begin 関数において、エラーコンテキストスタックのコールバックとして登録される。
  • **呼び出し先**: PostgreSQL のエラー報告システムである errcontext マクロを呼び出す。

関連データ構造

  • tsearch_readline_state: ファイル読み込みの状態（ファイル名、現在の行番号、現在の行内容など）を保持する構造体。

失敗モードと境界条件

  • stp->curline が NULL の場合、行内容を表示せずにファイル名と行番号のみを表示する安全策が講じられている。
  • エンコーディング違反が発生している場合、行内容の表示を避けることで、不正なデータによるログ出力時の二次的なエラーを防ぐ。

保守担当者向けメモ

  • 本関数は tsearch_readline_begin で error_context_stack に登録されるため、直接呼び出すことは想定されていない。
  • tsearch_readline 実行中に発生するエラーのコンテキストを標準化する役割を担うため、エラーメッセージの形式を変更する際は注意が必要である。
