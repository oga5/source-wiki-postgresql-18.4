LauncherLastStartTimesEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/launcher.c:72

目的

論理レプリケーションの各サブスクリプションに対して、Apply Worker が最後に起動を試みた時刻を保持し、再起動間隔の制御を行うためのデータ構造です。

シグネチャまたは形状

typedef struct LauncherLastStartTimesEntry
{
    Oid            subid;            /* サブスクリプションのOID（ハッシュキー） */
    TimestampTz    last_start_time;  /* 最後にApply Workerが起動された時刻 */
} LauncherLastStartTimesEntry;

入力と出力

本構造体は共有メモリ上のハッシュテーブルのエントリとして使用されます。

  • 入力: subid をキーとして検索または挿入が行われます。
  • 出力: last_start_time を参照することで、前回の起動からの経過時間を判定します。

前提条件と不変条件

  • subid は論理レプリケーションのサブスクリプションを一意に識別する OID である必要があります。
  • 共有メモリ上のハッシュテーブル内で管理されるため、アクセスには適切なロック（LogicalRepWorkerLock 等）が必要です。

アルゴリズムまたは振る舞い

論理レプリケーションランチャーがワーカーを起動する際、この構造体を参照して wal_retrieve_retry_interval に基づく再起動制限を適用します。これにより、エラー発生時にワーカーが短期間で繰り返し起動され、リソースを過剰に消費する事態を防ぎます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/launcher.c 内のワーカー起動ロジック（ApplyLauncherGetWorkerStartTime 等）から参照されます。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • LogicalRepCtx: ワーカーの状態を管理する共有メモリコンテキスト。
  • dshash: 本構造体を格納するために使用される共有ハッシュテーブル。

失敗モードと境界条件

  • ハッシュテーブルのメモリ不足により、新しいサブスクリプションのエントリ作成が失敗する可能性があります。
  • subid が無効な場合、誤った時刻情報が保持される可能性があります。

保守担当者向けメモ

  • 本構造体は launcher.c 内の last_start_times ハッシュテーブルの要素として定義されています。
  • 起動間隔の制御ロジックを変更する場合は、last_start_time の更新タイミングと参照箇所を慎重に確認してください。
