hstore_to_plperl

定義

  • 種別: function
  • 定義箇所: contrib/hstore_plperl/hstore_plperl.c:65

目的

PostgreSQL の hstore 型データを、PL/Perl 環境で利用可能な Perl のハッシュリファレンス（HV）へ変換する。

シグネチャまたは形状

Datum hstore_to_plperl(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る hstore 型のデータ（PG_GETARG_HSTORE_P(0)）。
  • **出力**: Datum 型として返される、Perl のハッシュリファレンスへのリファレンス（newRV((SV *) hv)）。

前提条件と不変条件

  • dTHX マクロにより、Perl インタープリタのコンテキストが正しく取得されている必要がある。
  • 入力された hstore 構造体は有効なメモリ領域にあること。

アルゴリズムまたは振る舞い

1. newHV() を使用して新しい Perl ハッシュを作成する。 2. hstore 内の全エントリを走査する。 3. 各キーについて pnstrdup でメモリを確保し、Perl 文字列としてコピーする。 4. 各値について、NULL であれば newSV(0) を、そうでなければ cstr2sv を用いて Perl スカラーに変換する。 5. hv_store を使用して、キーと値を Perl ハッシュに格納する。 6. 最終的に、作成したハッシュへのリファレンスを Datum に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • hstore 操作用マクロ: HS_COUNT, STRPTR, ARRPTR, HSTORE_KEY, HSTORE_VAL 等。
  • Perl API: newHV, pnstrdup, newSV, cstr2sv, hv_store, newRV。

関連データ構造

  • HStore: PostgreSQL 側の hstore 内部構造体。
  • HV: Perl のハッシュ構造体。
  • SV: Perl のスカラー構造体。

失敗モードと境界条件

  • hstore 内のキーや値のメモリ確保に失敗した場合、PostgreSQL のエラーハンドリング機構によりアボートされる可能性がある。
  • Perl インタープリタのコンテキストが未初期化の場合、dTHX の使用により予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は hstore の内部表現を直接走査するため、hstore のデータ構造に変更があった場合は追従が必要。
  • Perl API (hv_store 等) の使用にあたっては、Perl のメモリ管理と PostgreSQL のメモリコンテキストの整合性に注意すること。
