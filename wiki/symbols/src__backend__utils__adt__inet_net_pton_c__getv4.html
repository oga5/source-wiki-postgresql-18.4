getv4

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/inet_net_pton.c:380

目的

IPv4 アドレス文字列（ドット区切り形式）をパースし、ネットワークバイトオーダーのバイナリ形式へ変換します。また、CIDR 表記が含まれる場合はプレフィックス長を抽出します。

シグネチャまたは形状

static int getv4(const char *src, u_char *dst, int *bitsp)

入力と出力

  • src: パース対象の IPv4 アドレス文字列（例: "192.168.1.0/24"）。
  • dst: 変換後のバイナリデータを格納するバッファ（u_char 配列）。
  • bitsp: CIDR 表記のプレフィックス長を格納する整数へのポインタ。
  • 戻り値: 成功時は 1、失敗時は 0 を返します。

前提条件と不変条件

  • dst は少なくとも 4 バイトの領域を確保している必要があります。
  • 先頭ゼロ（例: "192.168.01.1"）は許容されません。
  • 各オクテットの値は 0 から 255 の範囲内である必要があります。

アルゴリズムまたは振る舞い

1. 文字列を走査し、数字を読み取ってオクテット値（val）を計算します。 2. .（ドット）に遭遇した場合、現在の val を dst に書き込み、次のオクテットの処理へ移行します。 3. /（スラッシュ）に遭遇した場合、現在の val を書き込んだ後、getbits を呼び出してプレフィックス長を解析し、その結果を bitsp に格納して終了します。 4. 文字列終端に達した際、最後のオクテットを書き込み、成功として 1 を返します。 5. 以下のいずれかの場合、直ちに 0 を返して失敗します。

  • 先頭ゼロが含まれる場合。
  • オクテット値が 255 を超える場合。
  • オクテット数が 4 を超える場合。
  • 数字や区切り文字以外の不正な文字が含まれる場合。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/inet_net_pton.c 内の関数（inet_cidr_pton_ipv4 等から参照されていることが示唆されています）。
  • 呼び出し先: getbits（/ 以降のパース時）。

関連データ構造

  • u_char: ネットワークアドレスの各オクテットを保持するために使用される型。

失敗モードと境界条件

  • 不正なフォーマット（文字の混入、範囲外の数値、過剰なオクテット数）はすべて 0 を返します。
  • dst バッファのオーバーフローを防止するため、オクテット数が 4 を超える場合はエラーとなります。

保守担当者向けメモ

  • 本関数は inet_net_pton ファミリーの一部であり、IPv4 特有のパースロジックを担います。
  • getbits への依存があるため、CIDR 表記の仕様変更時には両方の関数を確認してください。
