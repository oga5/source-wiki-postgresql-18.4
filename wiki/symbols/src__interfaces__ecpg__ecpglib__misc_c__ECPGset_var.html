ECPGset_var

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/misc.c:537

目的

ECPGアプリケーションにおいて、ホスト変数（または関連する変数）の識別番号とメモリ上のアドレスを紐付け、グローバルな変数リスト（ivlist）に登録または更新する。

シグネチャまたは形状

void ECPGset_var(int number, void *pointer, int lineno)

入力と出力

  • number (int): 変数を識別するための番号。
  • pointer (void *): 変数のメモリ上のアドレス。
  • lineno (int): 呼び出し元のソースコード行番号（エラー報告用）。
  • 出力: なし（副作用として ivlist を更新、またはエラー時に sqlca を更新）。

前提条件と不変条件

  • ECPGget_sqlca() が有効な sqlca_t 構造体を返すこと。
  • ivlist は、登録された変数を管理する連結リストの先頭を指すグローバル変数であること。

アルゴリズムまたは振る舞い

1. ECPGget_sqlca() を呼び出し、現在のスレッドの sqlca を取得する。取得失敗時は ECPG_OUT_OF_MEMORY エラーを発生させて終了する。 2. ecpg_init_sqlca(sqlca) を呼び出し、SQLCAを初期化する。 3. 既存の ivlist を走査し、指定された number と一致するエントリがあるか確認する。

  • 一致する場合: そのエントリの pointer を新しい値に更新して終了する。

    • 4. 一致しない場合: 新しい var_list 構造体を calloc で確保する。

  • 確保失敗時: sqlca にメモリ不足エラー情報を設定し、ECPGfree_auto_mem() を呼び出してメモリを解放する。
  • 確保成功時: 新しいエントリに number と pointer を設定し、ivlist の先頭に挿入する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/compatlib/informix.c
  • ECPGプリプロセッサが生成するテストコード（preproc-cursor.c 等）
  • 呼び出し先:
  • ECPGget_sqlca()
  • ecpg_init_sqlca()
  • ecpg_raise()
  • ECPGfree_auto_mem()

関連データ構造

  • struct var_list: 変数番号とポインタ、次要素へのポインタを保持するリスト構造体。
  • struct sqlca_t: SQL通信領域（SQLCA）。

失敗モードと境界条件

  • メモリ不足: calloc が失敗した場合、sqlca に ECPG_OUT_OF_MEMORY エラーを設定し、エラー状態を記録する。
  • 呼び出し元が不正な lineno を渡した場合、エラーメッセージに誤った行番号が含まれる可能性がある。

保守担当者向けメモ

  • 本関数は ivlist というグローバル変数に依存しているため、マルチスレッド環境下での整合性について、呼び出し元が適切に排他制御を行っているか確認が必要である。
  • ECPGfree_auto_mem() の呼び出しは、メモリ不足発生時のクリーンアップとして機能する。
