pg_base64_enc_len

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:405

目的

Base64エンコード後のデータに必要なバッファサイズを計算します。

シグネチャまたは形状

static uint64 pg_base64_enc_len(const char *src, size_t srclen)

入力と出力

  • **入力**:
  • src: エンコード対象のデータ（本関数内では未使用）。
  • srclen: エンコード対象データのバイト長。
  • **出力**:
  • エンコード後のデータに必要なバイト数を uint64 型で返します。

前提条件と不変条件

  • 入力 srclen は非負の整数である必要があります。
  • 計算結果は、Base64エンコードの仕様（3バイトを4文字に変換）および76文字ごとの改行挿入ルールに基づきます。

アルゴリズムまたは振る舞い

1. 基本的なBase64変換サイズとして、(srclen + 2) / 3 * 4 を算出します。 2. 76文字ごとに改行を挿入するルールを考慮し、追加のバイト数を加算します。 3. 具体的には (uint64) srclen / (76 * 3 / 4) を加算することで、改行文字分を補正します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/encode.c 内の pg_base64_encode 等のエンコーディング処理フロー。
  • contrib/pgcrypto/pgp-armor.c:163
  • contrib/pgcrypto/pgp-armor.c:222
  • **呼び出し先**:
  • 本関数は計算のみを行うため、他の関数を呼び出しません。

関連データ構造

  • src/backend/utils/adt/encode.c 内の pg_encoding 構造体（エンコーディング方式を定義するテーブルの一部として参照されます）。

失敗モードと境界条件

  • srclen が非常に大きい場合、計算結果が uint64 の範囲を超える可能性がありますが、本関数自体はオーバーフローチェックを行いません。呼び出し側で MaxAllocSize 等を用いた検証が必要です。

保守担当者向けメモ

  • 本関数は76文字ごとの改行挿入を前提としています。将来的に改行ルールが変更された場合、計算式の定数（76）を更新する必要があります。
  • src 引数は現在の実装では使用されていませんが、将来的な拡張やインターフェースの一貫性のために保持されています。
