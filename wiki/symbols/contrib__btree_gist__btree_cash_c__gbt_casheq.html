gbt_casheq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_cash.c:39

目的

PostgreSQLの money 型の値が等しいかどうかを判定し、GiSTインデックスの比較操作をサポートします。

シグネチャまたは形状

static bool gbt_casheq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Cash 型へのポインタ（void * として渡される）。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: 両者の値が等しい場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な Cash 型の値を指している必要があります。

アルゴリズムまたは振る舞い

1. a と b をそれぞれ Cash 型のポインタにキャストします。 2. キャストしたポインタが指す値を直接比較演算子 == で比較し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_cash.c 内の tinfo 構造体（gbtree_ninfo）の初期化時に参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません（C言語の組み込み比較演算子を使用）。

関連データ構造

  • Cash: PostgreSQLの money 型を表現するデータ構造。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける money 型の比較アダプターの一部です。
  • flinfo 引数はインターフェースの互換性のために存在しますが、本実装では使用されていません。
