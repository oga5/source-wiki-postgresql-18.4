TS_phrase_output

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:1467

目的

2つの ExecPhraseData（全文検索における位置情報のリスト）をマージし、指定された条件（emit）に基づいて一致する位置情報を出力または判定する。

シグネチャまたは形状

static TSTernaryValue TS_phrase_output(
    ExecPhraseData *data,
    ExecPhraseData *Ldata,
    ExecPhraseData *Rdata,
    int emit,
    int Loffset,
    int Roffset,
    int max_npos
)

入力と出力

  • **入力**:
  • data: 結果を格納する構造体（NULL の場合は判定のみを行う）。
  • Ldata, Rdata: 比較対象となる位置情報リスト。
  • emit: 出力条件を示すビットフラグ（TSPO_L_ONLY, TSPO_R_ONLY, TSPO_BOTH）。
  • Loffset, Roffset: 各リストの位置情報に加算するオフセット値。
  • max_npos: data に格納可能な最大要素数。
  • **出力**:
  • TSTernaryValue: 一致が見つかった場合は TS_YES、見つからなかった場合は TS_NO を返す。

前提条件と不変条件

  • Ldata および Rdata の位置情報は昇順である必要がある（マージジョインの前提）。
  • data が非 NULL の場合、data->pos は max_npos 分のメモリを確保できる必要がある。
  • data->npos は max_npos を超えないことが Assert により保証される。

アルゴリズムまたは振る舞い

1. Ldata と Rdata をマージジョインの要領で走査する。 2. 各要素の WEP_GETPOS 値にオフセットを加算し、比較する。 3. emit フラグに従い、以下の条件で output_pos を決定する：

  • Lpos < Rpos: TSPO_L_ONLY が設定されていれば Lpos を出力。
  • Lpos == Rpos: TSPO_BOTH が設定されていれば Rpos を出力。
  • Lpos > Rpos: TSPO_R_ONLY が設定されていれば Rpos を出力。

    • 4. data が提供されている場合、結果を data->pos に格納する。提供されていない場合は、最初の一致が見つかった時点で TS_YES を即座に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsvector_op.c 内の複数の箇所から、フレーズ検索の評価ロジックとして呼び出される。
  • **呼び出し先**: palloc（メモリ確保）、WEP_GETPOS（マクロ）。

関連データ構造

  • ExecPhraseData: 位置情報リストとメタデータを保持する構造体。
  • WordEntryPos: tsvector 内の位置情報を表す型。
  • TSTernaryValue: 検索結果の三値論理値。

失敗モードと境界条件

  • data が NULL の場合、最初の一致で早期終了する。
  • Ldata または Rdata が空の場合、emit フラグの指定に従い、残りのリストを処理するか、あるいは即座に終了する。
  • data->npos が max_npos を超えることはない（Assert による保護）。

保守担当者向けメモ

  • 本関数は tsvector の位置情報操作における低レベルなマージ処理を担当する。
  • WEP_GETPOS マクロを使用しているため、tsvector の内部構造（WordEntryPosVector）に変更があった場合は注意が必要。
  • data が NULL の場合の早期リターンは、パフォーマンス最適化のために利用されている。
