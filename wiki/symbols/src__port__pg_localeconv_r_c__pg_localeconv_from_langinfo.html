pg_localeconv_from_langinfo

定義

  • 種別: function
  • 定義箇所: src/port/pg_localeconv_r.c:115

目的

指定されたロケール（数値用および通貨用）に基づき、nl_langinfo_l を使用して struct lconv 構造体の各メンバを埋める。

シグネチャまたは形状

static int pg_localeconv_from_langinfo(struct lconv *dst, locale_t monetary_locale, locale_t numeric_locale);

入力と出力

  • **入力**:
  • dst: 情報を格納する struct lconv へのポインタ。
  • monetary_locale: 通貨カテゴリのロケールオブジェクト。
  • numeric_locale: 数値カテゴリのロケールオブジェクト。
  • **出力**:
  • 成功時は 0 を返す。
  • メモリ確保失敗時は -1 を返し、errno に ENOMEM を設定する。

前提条件と不変条件

  • table[] に定義されたメタデータに基づき、lconv 構造体の全メンバを走査する。
  • 文字列メンバについては strdup() でメモリを確保するため、呼び出し元は後続の処理で pg_localeconv_free() を呼び出し、メモリを解放する責任がある。

アルゴリズムまたは振る舞い

1. table[] をループで走査する。 2. 各メンバのカテゴリに応じて monetary_locale または numeric_locale を選択する。 3. is_string フラグが真の場合:

  • nl_langinfo_l() で情報を取得し、strdup() でコピーする。
  • 確保に失敗した場合は pg_localeconv_free(dst) を呼び出してクリーンアップし、エラーを返す。

    • 4. is_string フラグが偽の場合:

  • nl_langinfo_l() の結果を直接キャストしてメンバに代入する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/pg_localeconv_r.c 内のロケール処理フローから呼び出される（行116および322）。
  • **呼び出し先**: nl_langinfo_l、strdup、pg_localeconv_free、lconv_string_member、lconv_char_member。

関連データ構造

  • struct lconv: 標準 C ライブラリのロケール情報構造体。
  • table[]: lconv 構造体の各メンバの属性（カテゴリ、型、nl_langinfo アイテム）を定義した内部テーブル。

失敗モードと境界条件

  • strdup() によるメモリ確保に失敗した場合、ENOMEM を返し、既に確保済みのメンバを解放するために pg_localeconv_free() を実行する。

保守担当者向けメモ

  • 本関数は nl_langinfo_l が利用可能な環境（主に POSIX）でのみ機能する。
  • table[] の定義と lconv 構造体のメンバ順序が一致していることが前提となっている。
