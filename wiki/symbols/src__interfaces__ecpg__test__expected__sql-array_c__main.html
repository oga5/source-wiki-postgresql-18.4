目的

ECPG（Embedded SQL for C）における配列型および複合データ型の取り扱いを検証するための回帰テストを実行する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム内で定義された静的データおよび PGTYPES ライブラリによる生成データを使用）。
  • 出力: データベース操作の結果を標準出力へ出力し、終了コードを返す。

前提条件と不変条件

  • ECPGプリプロセッサによって生成されたコードであること。
  • ecpglib および pgtypeslib が利用可能であること。
  • データベース ecpg1_regression への接続が可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: setlocale および ECPGdebug を設定。 2. **データ準備**: PGTYPES 関数群（PGTYPEStimestamp_from_asc 等）を用いて、テスト用の配列データ（ts, d, in, n）を初期化。 3. **データベース接続**: ECPGconnect で接続し、ECPGsetcommit でコミットモードを有効化。 4. **テーブル作成**: CREATE TABLE を実行し、配列型を含むスキーマを定義。 5. **データ操作**:

  • 複数の INSERT 文を実行し、C言語の配列をSQLのバインド変数として渡す。
  • SELECT 文を実行し、取得したデータを検証。

    • 6. **終了処理**: DROP TABLE を実行し、ECPGdisconnect で接続を終了。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムの実行起点）。
  • 呼び出し先: ECPGconnect, ECPGsetcommit, ECPGtrans, ECPGdo, PGTYPES... 関数群、malloc, strcpy, sprintf, setlocale。

関連データ構造

  • sqlca: SQL通信領域。各SQL実行後のエラーチェックに使用。
  • timestamp, date, interval, numeric: pgtypeslib で定義されたPostgreSQL互換のデータ型。

失敗モードと境界条件

  • 各SQL実行後に sqlca.sqlcode < 0 をチェックし、エラー発生時は sqlprint() を呼び出して診断情報を出力する。
  • 配列の境界（10要素）やメモリ確保（malloc）の成否がテストの整合性に影響する。

保守担当者向けメモ

  • 本関数は自動生成されたテストコードの期待値です。直接編集せず、プリプロセッサの出力結果との差分確認に使用してください。
  • ECPGdo マクロ内の引数はプリプロセッサが生成したバインド変数リストであり、SQL側の定義とC言語側の配列サイズ・型指定が厳密に一致している必要があります。
