add_variable_to_tail

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/variable.c:412

目的

SQL文の入出力引数リスト（arguments 構造体の連結リスト）の末尾に、新しい変数エントリを追加します。

シグネチャまたは形状

void add_variable_to_tail(struct arguments **list, struct variable *var, struct variable *ind)

入力と出力

  • **入力**:
  • list: 引数リストの先頭ポインタへのポインタ。
  • var: リストに追加するホスト変数へのポインタ。
  • ind: 対応するインジケータ変数へのポインタ（NULL可）。
  • **出力**: なし（list が指すリスト構造を直接更新）。

前提条件と不変条件

  • list は有効な arguments リストのポインタである必要があります。
  • mm_alloc を使用してメモリを確保するため、メモリ不足時には mmfatal によりプロセスが終了します。

アルゴリズムまたは振る舞い

1. mm_alloc を用いて新しい arguments 構造体を確保します。 2. リストが空でない場合、ループを使用してリストの末尾（next が NULL である要素）まで走査します。 3. 新しい構造体に var および ind を設定し、next を NULL に初期化します。 4. リストが空であれば list の先頭に新しい要素を配置し、空でなければ末尾の要素の next に新しい要素を接続します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/preproc_extern.h に宣言があり、プリプロセッサの構文解析フェーズで引数リストを構築する際に使用されます（具体的な呼び出し箇所はソース抜粋からは特定できません）。
  • **呼び出し先**: mm_alloc（メモリ確保）。

関連データ構造

  • struct arguments: SQL文の引数リストを構成する連結リスト構造体。
  • struct variable: ホスト変数やインジケータ変数の情報を保持する構造体。

失敗モードと境界条件

  • list が指す先が NULL（空リスト）の場合、新しい要素がリストの先頭として正しく設定されます。
  • ind が NULL の場合でも、変数エントリとして正しくリストに追加されます。

保守担当者向けメモ

  • 本関数は argsinsert や argsresult といった引数リストの構築に使用されます。
  • リストの末尾を走査する線形探索を行うため、引数リストが極端に長い場合はパフォーマンスに影響する可能性があります。
