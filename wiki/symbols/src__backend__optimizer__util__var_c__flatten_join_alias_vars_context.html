flatten_join_alias_vars_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/var.c:64

目的

JOINエイリアスを展開する際の再帰的な走査・置換処理において、現在のクエリ文脈や状態を保持するために使用されるコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    PlannerInfo *root;          /* 処理対象のPlannerInfo（NULLの可能性あり） */
    Query       *query;         /* 外側のQuery構造体 */
    int         sublevels_up;   /* 現在のネストレベル */
    bool        possible_sublink; /* エイリアスにSubLinkが含まれる可能性があるか */
    bool        inserted_sublink; /* SubLinkが挿入されたかどうかのフラグ */
} flatten_join_alias_vars_context;

入力と出力

本構造体は関数引数として渡され、flatten_join_alias_vars 等のミューテータ関数内で状態を共有するために使用されます。直接的な入力・出力値を持つ関数ではなく、状態保持のためのコンテナです。

前提条件と不変条件

  • root は NULL である可能性があります（パーサとプランナの両方から利用されるため）。
  • sublevels_up は、サブクエリや SubLink を扱う際に正しく管理される必要があります。

アルゴリズムまたは振る舞い

expression_tree_mutator を利用したツリー走査において、JOINエイリアスをベースリレーションの変数へ置換する際の制御状態を保持します。特に possible_sublink と inserted_sublink を用いて、置換過程で発生する SubLink の挿入状況を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/var.c 内の flatten_join_alias_vars および関連する内部関数（行791, 813, 974, 996付近）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • PlannerInfo: オプティマイザのプラン作成情報。
  • Query: SQLクエリのパースツリー。
  • SubLink: サブクエリを含む式ノード。

失敗モードと境界条件

  • root が NULL の場合、PlannerInfo に依存する特定の最適化処理が制限される可能性があります。
  • sublevels_up の管理が不適切な場合、変数のスコープ誤認が発生します。

保守担当者向けメモ

  • 本構造体は flatten_join_alias_vars の実行中にのみ有効な一時的な状態保持用です。
  • PlaceHolderVar が含まれる場合、オプティマイザの文脈では Var と同様に扱われるため、走査ロジックの変更時には注意が必要です。
