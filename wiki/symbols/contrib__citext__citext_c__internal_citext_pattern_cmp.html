internal_citext_pattern_cmp

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:71

目的

citext 型のパターンマッチング比較において、照合順序（Collation）を無視したバイナリレベルでの大小比較を行う。

シグネチャまたは形状

static int32 internal_citext_pattern_cmp(text *left, text *right, Oid collid)

入力と出力

  • **入力**:
  • left, right: 比較対象の text 型データ。
  • collid: 照合順序の OID（本関数内では DEFAULT_COLLATION_OID を使用するため、引数としては使用されない）。
  • **出力**:
  • int32: left < right なら負の値、left == right なら 0、left > right なら正の値を返す。

前提条件と不変条件

  • 入力された text 型データは str_tolower によって小文字に正規化される。
  • 比較は DEFAULT_COLLATION_OID を用いて行われる。
  • 処理の最後に、正規化のために確保されたメモリは pfree により解放される。

アルゴリズムまたは振る舞い

1. str_tolower を使用し、left と right を小文字に変換した文字列 lcstr, rcstr を生成する。 2. memcmp を使用し、短い方の文字列長までバイナリ比較を行う。 3. memcmp の結果が 0（一致）の場合、文字列長を比較し、長さが異なる場合は短い方を小さいと判定する。 4. 生成した一時文字列 lcstr, rcstr を解放し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • citext_pattern_lt (行323)
  • citext_pattern_le (行340)
  • citext_pattern_gt (行357)
  • citext_pattern_ge (行374)
  • その他、パターン比較用関数（行133等）
  • **呼び出し先**:
  • str_tolower
  • strlen
  • memcmp
  • Min
  • pfree

関連データ構造

  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • 入力文字列が空の場合、strlen は 0 を返し、長さ比較ロジックによって適切に処理される。
  • メモリ不足時には str_tolower がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は照合順序を無視する「パターン比較」用であるため、通常の citext_cmp（照合順序を考慮する）とは異なるロジックである点に注意すること。
  • collid 引数はシグネチャに含まれているが、実装内では DEFAULT_COLLATION_OID が固定で使用されている。
