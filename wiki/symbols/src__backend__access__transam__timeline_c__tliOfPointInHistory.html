tliOfPointInHistory

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/timeline.c:543

目的

指定されたWAL位置（XLogRecPtr）が、提供されたタイムライン履歴リストのどのタイムライン（TimeLineID）に属するかを特定します。

シグネチャまたは形状

TimeLineID tliOfPointInHistory(XLogRecPtr ptr, List *history)

入力と出力

  • **入力**:
  • ptr: 検索対象のWAL位置（XLogRecPtr）。
  • history: TimeLineHistoryEntry 構造体のリスト。
  • **出力**:
  • 該当する TimeLineID を返します。

前提条件と不変条件

  • history リストは、readTimeLineHistory 等によって構築された、タイムラインの履歴情報を含んでいる必要があります。
  • 履歴リスト内の各エントリは、begin から end までの範囲を保持しています。

アルゴリズムまたは振る舞い

1. history リストを先頭から順に走査します。 2. 各エントリについて、ptr が [begin, end) の範囲内にあるかを確認します。

  • begin が無効（XLogRecPtrIsInvalid）または ptr >= begin であること。
  • end が無効または ptr < end であること。

    • 3. 条件を満たすエントリが見つかった場合、その tli を返します。 4. リストの最後まで該当する範囲が見つからない場合、elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/access/transam/xlogutils.c
  • src/backend/replication/walsender.c
  • **呼び出し先**:
  • lfirst（リスト操作マクロ）
  • XLogRecPtrIsInvalid（WAL位置の検証マクロ）
  • elog（エラー報告）

関連データ構造

  • TimeLineHistoryEntry: タイムラインの開始位置、終了位置、および TimeLineID を保持する構造体。
  • List: PostgreSQLの標準的なリスト構造。

失敗モードと境界条件

  • **エラー**: どのタイムライン範囲にも該当しないWAL位置が渡された場合、elog(ERROR) により処理が中断されます。これは履歴情報の不整合を意味します。
  • **境界条件**: begin または end が無効な値（XLogRecPtrIsInvalid）である場合、その方向の範囲制限は無視されます。

保守担当者向けメモ

  • 本関数は履歴リストの線形探索を行うため、リストが非常に長い場合は性能に影響する可能性がありますが、通常はタイムラインの分岐回数は限定的であるため許容されています。
  • 戻り値の 0 はコンパイラの警告回避用であり、エラー発生時には到達しません。
