CollateStrength

定義

  • 種別: enum
  • 定義箇所: src/backend/parser/parse_collate.c:56

目的

式解析の過程において、照合順序（Collation）の派生強度を分類し、競合の有無や決定の根拠を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
    COLLATE_NONE,
    COLLATE_IMPLICIT,
    COLLATE_CONFLICT,
    COLLATE_EXPLICIT,
} CollateStrength;

入力と出力

本シンボルは列挙型であり、関数ではありません。式解析のコンテキスト内で、照合順序の解決状態を示す値として利用されます。

前提条件と不変条件

  • COLLATE_NONE: 対象のデータ型が照合順序を持たない場合に設定されます。
  • COLLATE_IMPLICIT: 照合順序が暗黙的に導出された状態を示します。
  • COLLATE_CONFLICT: 複数の暗黙的な照合順序が競合した状態を示します。
  • COLLATE_EXPLICIT: 照合順序が明示的に指定された状態を示します。

アルゴリズムまたは振る舞い

式解析の再帰的な走査において、各ノードの照合順序をマージする際に、この強度が比較・評価されます。競合が発生した場合は COLLATE_CONFLICT が設定され、後続の処理でエラー報告のトリガーとなります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_collate.c 内の assign_collations_context 構造体や、照合順序解決を行う各関数（merge_collation_state 等）で参照されています。
  • 呼び出し先: 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • assign_collations_context: 走査中に照合順序の状態を保持するためのコンテキスト構造体。本列挙型をメンバとして保持します。

失敗モードと境界条件

  • COLLATE_CONFLICT が設定された場合、照合順序の解決に失敗したとみなされ、クエリ解析時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 照合順序の解決ロジック（merge_collation_state 等）を変更する際は、この強度の優先順位と状態遷移が適切に維持されているか確認してください。
