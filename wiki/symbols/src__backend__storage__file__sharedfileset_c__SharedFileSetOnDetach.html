SharedFileSetOnDetach

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/sharedfileset.c:95

目的

DSM（動的共有メモリ）セグメントのデタッチ時に実行されるコールバック関数であり、共有ファイルセットの参照カウンタを管理し、最後の参照者がデタッチした際に物理的なファイル削除を実行します。

シグネチャまたは形状

static void SharedFileSetOnDetach(dsm_segment *segment, Datum datum);

入力と出力

  • segment: デタッチ対象の dsm_segment ポインタ。
  • datum: SharedFileSet 構造体へのポインタを保持する Datum 型。
  • 戻り値: なし（void）。

前提条件と不変条件

  • fileset->refcnt は常に 0 より大きい必要があります（Assert により検証）。
  • 参照カウンタの操作は fileset->mutex（スピンロック）によって保護されます。
  • 本関数はエラー発生時のクリーンアップパスでも呼び出されるため、関数内でエラーを発生させてはなりません。

アルゴリズムまたは振る舞い

1. datum から SharedFileSet 構造体を取得します。 2. fileset->mutex を獲得します。 3. refcnt をデクリメントし、その値が 0 になった場合は unlink_all フラグを true に設定します。 4. fileset->mutex を解放します。 5. unlink_all が true の場合、FileSetDeleteAll を呼び出し、関連するディレクトリとファイルを物理的に削除します。

呼び出し元と呼び出し先

  • 呼び出し元: on_dsm_detach を介して DSM デタッチ時に自動的に呼び出されます（SharedFileSetInit および SharedFileSetAttach 内で登録）。
  • 呼び出し先: FileSetDeleteAll（unlink_all が真の場合）。

関連データ構造

  • SharedFileSet: 共有ファイルセットを管理する構造体。
  • dsm_segment: 動的共有メモリセグメント。

失敗モードと境界条件

  • refcnt が 0 の状態で呼び出された場合、Assert により異常終了します。
  • 本関数内でのエラー発生は避ける必要があります。

保守担当者向けメモ

  • FileSetDeleteAll はファイルシステムレベルの削除を行うため、呼び出し時のコンテキストに注意してください。
  • 参照カウンタの整合性はスピンロックによって保証されています。
