DropObjectById

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/dependency.c:1188

目的

指定された ObjectAddress に基づき、システムカタログから特定のデータベースオブジェクトを物理的に削除します。

シグネチャまたは形状

static void DropObjectById(const ObjectAddress *object)

入力と出力

  • **入力**: object (const ObjectAddress *) - 削除対象となるオブジェクトのアドレス（クラスIDとオブジェクトID）。
  • **出力**: なし。成功時はカタログからタプルが削除されます。

前提条件と不変条件

  • 呼び出し元は、削除対象オブジェクトに対する適切なロック（RowExclusiveLock）を保持している必要があります。
  • 対象オブジェクトがシステムカタログ内に存在することが前提です。

アルゴリズムまたは振る舞い

1. get_object_catcache_oid を使用して、対象オブジェクトのクラスIDに対応するシステムキャッシュIDを取得します。 2. 対象のカタログテーブルを RowExclusiveLock モードで開きます。 3. **キャッシュ利用時**: cacheId が有効な場合、SearchSysCache1 でタプルを検索し、CatalogTupleDelete で削除します。 4. **キャッシュ非利用時**: cacheId が無効な場合、systable_beginscan を用いてインデックススキャンを行い、対象タプルを特定して CatalogTupleDelete で削除します。 5. 最後に table_close でカタログテーブルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c 内の関数から参照されています。
  • **呼び出し先**: table_open, get_object_catcache_oid, SearchSysCache1, CatalogTupleDelete, ReleaseSysCache, systable_beginscan, systable_getnext, systable_endscan, table_close 等のカタログ操作関数。

関連データ構造

  • ObjectAddress: 削除対象を識別するための構造体。
  • Relation: カタログテーブルへのアクセスに使用。
  • HeapTuple: カタログ内の行データ。

失敗モードと境界条件

  • **キャッシュ検索失敗**: SearchSysCache1 でタプルが見つからない場合、elog(ERROR) によりトランザクションがアボートされます。
  • **スキャン失敗**: systable_getnext でタプルが見つからない場合、同様に elog(ERROR) が発生します。

保守担当者向けメモ

  • 本関数は低レベルな削除処理を担当します。依存関係のチェックやカスケード削除のロジックは呼び出し元で行う必要があります。
  • 新しいオブジェクトタイプを追加する際は、get_object_catcache_oid 等が正しく対応しているか確認してください。
