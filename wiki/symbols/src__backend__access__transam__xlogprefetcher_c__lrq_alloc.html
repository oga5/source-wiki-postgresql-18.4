lrq_alloc

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogprefetcher.c:201

目的

先読みリクエストを管理するための循環キュー構造体（LsnReadQueue）を動的に割り当て、初期化する。

シグネチャまたは形状

static inline LsnReadQueue *lrq_alloc(uint32 max_distance, uint32 max_inflight, uintptr_t lrq_private, LsnReadQueueNextFun next)

入力と出力

  • **入力**:
  • max_distance: キューの最大容量（リングバッファのサイズ）。
  • max_inflight: 同時に処理可能な最大I/O数。
  • lrq_private: キューに関連付けるプライベートデータへのポインタ。
  • next: 次のブロックを取得するためのコールバック関数。
  • **出力**:
  • 初期化済みの LsnReadQueue 構造体へのポインタ。

前提条件と不変条件

  • max_distance >= max_inflight であること（アサーションにより検証）。
  • リングバッファとして機能させるため、内部的なキューサイズは max_distance + 1 として確保される。

アルゴリズムまたは振る舞い

1. max_distance + 1 のサイズで LsnReadQueue 構造体をメモリ割り当て（palloc）する。 2. 引数で渡された max_inflight、lrq_private、next コールバックを構造体に格納する。 3. キューのインデックス（head, tail）およびカウンタ（inflight, completed）を 0 に初期化する。 4. 初期化された構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogprefetcher.c 内の XLogPrefetcherAllocate 関数から呼び出される。
  • **呼び出し先**:
  • palloc（メモリ割り当て）。

関連データ構造

  • LsnReadQueue: 先読みI/Oリクエストを管理する循環キュー構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • max_distance が max_inflight より小さい場合、アサーション違反によりプロセスが停止する。

保守担当者向けメモ

  • 本関数は static inline で定義されており、xlogprefetcher.c 内でのみ使用される。
  • キューのサイズは「空き」を区別するために max_distance + 1 となっている点に注意すること。
