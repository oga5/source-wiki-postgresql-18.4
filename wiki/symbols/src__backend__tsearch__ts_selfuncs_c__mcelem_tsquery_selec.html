mcelem_tsquery_selec

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_selfuncs.c:208

目的

pg_statistic から取得した最頻要素（MCELEM）統計情報を基に、tsquery の選択率を推定する。

シグネチャまたは形状

static Selectivity mcelem_tsquery_selec(TSQuery query, Datum *mcelem, int nmcelem, float4 *numbers, int nnumbers)

入力と出力

  • **入力**:
  • query: 評価対象の TSQuery 構造体。
  • mcelem: 統計情報に含まれる最頻要素の配列。
  • nmcelem: mcelem 配列の要素数。
  • numbers: 各要素の頻度および最小/最大頻度を含む配列。
  • nnumbers: numbers 配列の要素数。
  • **出力**:
  • 推定された Selectivity（float8）。

前提条件と不変条件

  • nnumbers は nmcelem + 2 である必要がある（最後の2つは最小・最大頻度）。
  • mcelem 内の各要素は圧縮や外部格納されていないことが前提。

アルゴリズムまたは振る舞い

1. 統計情報の整合性チェックを行い、条件を満たさない場合は tsquery_opr_selec_no_stats を呼び出してデフォルト値を返す。 2. mcelem と numbers を TextFreq 構造体の配列に変換し、検索可能な形式に整える。 3. numbers 配列の最後から2番目の値を最小頻度（minfreq）として取得する。 4. tsquery_opr_selec を呼び出し、変換した統計情報と minfreq を用いてクエリの選択率を再帰的に計算する。 5. 割り当てた lookup 配列を解放し、計算結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: tsquerysel（src/backend/tsearch/ts_selfuncs.c 内）。
  • **呼び出し先**: tsquery_opr_selec。

関連データ構造

  • TextFreq: 語彙と頻度を保持する内部構造体。
  • TSQuery: 全文検索クエリの構造体。

失敗モードと境界条件

  • nnumbers != nmcelem + 2 の場合、統計情報が不完全とみなされ、tsquery_opr_selec_no_stats によるフォールバックが行われる。

保守担当者向けメモ

  • ts_typanalyze.c で生成された統計情報のレイアウトに強く依存している。
  • tsquery_opr_selec は再帰関数であるため、非常に複雑なクエリに対してはスタック深さの考慮が必要となる。
