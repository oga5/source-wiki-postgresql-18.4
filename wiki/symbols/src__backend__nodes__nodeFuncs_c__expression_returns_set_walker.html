expression_returns_set_walker

定義

  • 種別: static function
  • 定義箇所: src/backend/nodes/nodeFuncs.c:768

目的

式ツリーを再帰的に走査し、ツリー内に集合を返す関数（Set-Returning Function: SRF）が含まれているかどうかを判定する。

シグネチャまたは形状

static bool expression_returns_set_walker(Node *node, void *context);

入力と出力

  • **入力**:
  • node: 走査対象のノードツリー（Node *）。
  • context: 走査の文脈情報（本関数内では使用されていない）。
  • **出力**:
  • 集合を返すノードが見つかった場合は true、それ以外の場合は false を返す。

前提条件と不変条件

  • node が NULL の場合は false を返す。
  • 集合を返すノードとして FuncExpr（funcretset が真）および OpExpr（opretset が真）を認識する。
  • Aggref、GroupingFunc、WindowFunc は集合を返さないものとして扱い、これらが出現した枝の再帰走査は行わない。

アルゴリズムまたは振る舞い

1. ノードが NULL なら false を返す。 2. ノードが FuncExpr または OpExpr であり、かつそれぞれのフラグ（funcretset / opretset）が真であれば true を返す。 3. ノードが Aggref、GroupingFunc、WindowFunc のいずれかであれば、集合を返さないと判断し false を返す。 4. 上記以外の場合、expression_tree_walker を呼び出し、子ノードに対して再帰的に本関数を適用する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/nodeFuncs.c 内の expression_returns_set 関数から呼び出される。
  • **呼び出し先**:
  • expression_tree_walker（汎用的なツリー走査関数）。

関連データ構造

  • Node: 走査対象の汎用ノード構造体。
  • FuncExpr: 関数呼び出しノード。
  • OpExpr: 演算子式ノード。

失敗モードと境界条件

  • 非常に深いツリーを走査する場合、スタックオーバーフローのリスクがある（expression_tree_walker の特性に依存）。
  • 新たな集合を返すノード型を追加した場合は、本関数だけでなく clauses.c の expression_returns_set_rows および tlist.c の IS_SRF_CALL も更新する必要がある。

保守担当者向けメモ

  • 本関数は expression_tree_walker を介して再帰的に呼び出されるため、ノード型の追加・変更時には switch 文の代わりに IsA マクロを用いた条件分岐が適切か検討すること。
  • 集合を返すノードの判定ロジックは、他の関連関数と同期を保つ必要があるため、ソース内のコメントに従い関連箇所を必ず確認すること。
