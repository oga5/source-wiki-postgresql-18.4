heap_vacuum_rel

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/vacuumlazy.c:614

目的

指定されたヒープリレーションに対して「Lazy Vacuum（同時実行VACUUM）」を実行し、不要なタプルの削除、タプルの凍結、およびインデックスのクリーンアップを行うメインエントリポイントです。

シグネチャまたは形状

void heap_vacuum_rel(Relation rel, VacuumParams *params, BufferAccessStrategy bstrategy)

入力と出力

  • **入力**:
  • rel: VACUUM対象のリレーション。
  • params: VACUUMの動作を制御するパラメータ（index_cleanup, truncate 等）。
  • bstrategy: バッファアクセスの戦略（BufferAccessStrategy）。
  • **出力**: なし（副作用としてリレーションのメンテナンスと統計情報の更新を行う）。

前提条件と不変条件

  • params->index_cleanup および params->truncate は VACOPTVALUE_UNSPECIFIED ではないこと。
  • VacuumFailsafeActive は呼び出し前にリセットされている必要がある（再帰呼び出しを考慮し、関数内でもリセットされる）。
  • NewRelfrozenXid の追跡において、スキャン対象のページに含まれる未凍結のXIDを見逃さないよう、OldestXmin の取得とスキャン範囲の決定は厳密な順序で行われる必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: 統計情報の計測開始、進捗報告の開始、およびエラーコンテキストコールバックの設定を行う。 2. **状態管理**: LVRelState を割り当て、対象リレーションおよびインデックスの情報を初期化する。 3. **パラメータ適用**: params に基づき、インデックス掃除やテーブル切り詰めの可否を決定する。 4. **カットオフ計算**: vacuum_get_cutoffs を呼び出し、デッドタプルの判定基準（OldestXmin 等）とスキャン範囲を確定させる。 5. **スキャン準備**: 可視性マップ（VM）を用いたページスキップの可否を設定し、ヒープスキャン（lazy_scan_heap）へ移行する準備を整える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c:2656（relation_vacuum ハンドラ経由）
  • **呼び出し先**:
  • vacuum_error_callback（エラーコンテキスト用）
  • vac_open_indexes
  • vacuum_get_cutoffs
  • GlobalVisTestFor
  • その他、palloc 等のメモリ管理関数や pgstat 等の統計関数。

関連データ構造

  • LVRelState: VACUUM実行中の状態を保持する主要な構造体。
  • VacuumParams: VACUUMの実行オプション。
  • Relation: 対象リレーションのメタデータ。
  • IndexBulkDeleteResult: インデックス掃除の結果統計。

失敗モードと境界条件

  • **再帰呼び出し**: VacuumFailsafeActive のリセットにより、再帰的な呼び出し時にも安全に動作するよう設計されている。
  • **インデックス掃除の無効化**: params->index_cleanup が VACOPTVALUE_DISABLED の場合、インデックス掃除はスキップされる。
  • **エラーハンドリング**: error_context_stack を利用し、処理中のフェーズ情報を保持することで、エラー発生時に詳細なコンテキストを表示する。

保守担当者向けメモ

  • 本関数はVACUUMの初期化フェーズを担当する。実際のヒープスキャンやインデックス掃除のロジックは、この後に呼び出される lazy_scan_heap や lazy_vacuum に委譲される。
  • LVRelState の各フィールドは、後続のフェーズでエラー報告や統計更新に利用されるため、初期化漏れがないよう注意すること。
