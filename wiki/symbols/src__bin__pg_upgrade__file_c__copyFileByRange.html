copyFileByRange

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/file.c:150

目的

OSの copy_file_range システムコールを利用して、ソースファイルから宛先ファイルへデータを効率的に転送する。

シグネチャまたは形状

void copyFileByRange(const char *src, const char *dst, const char *schemaName, const char *relName)

入力と出力

  • **入力**:
  • src: コピー元ファイルのパス。
  • dst: コピー先ファイルのパス。
  • schemaName: エラーログ出力用のスキーマ名。
  • relName: エラーログ出力用のリレーション名。
  • **出力**: なし（戻り値は void）。成功時はファイルが複製され、失敗時は pg_fatal によりプロセスが終了する。

前提条件と不変条件

  • HAVE_COPY_FILE_RANGE マクロが定義されている環境でのみ有効。
  • src ファイルは読み取り可能であること。
  • dst ファイルは新規作成可能であること（O_CREAT | O_EXCL を使用）。

アルゴリズムまたは振る舞い

1. src を読み取り専用、dst を書き込み専用（新規作成）で開く。 2. copy_file_range を使用して、ファイル内容をループで転送する。 3. 転送中にエラーが発生した場合、pg_fatal を呼び出し、エラーメッセージと共に処理を中断する。 4. 転送完了後、両方のファイル記述子を閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/relfilenumber.c:635
  • **呼び出し先**:
  • open, close (標準ライブラリ)
  • copy_file_range (システムコール)
  • pg_fatal (エラーハンドリング)

関連データ構造

  • pg_file_create_mode: ファイル作成時のパーミッション定義。

失敗モードと境界条件

  • ファイルのオープン失敗（権限不足やパスの誤りなど）: pg_fatal により終了。
  • copy_file_range の実行失敗: pg_fatal により終了。
  • HAVE_COPY_FILE_RANGE が未定義の場合、本関数は何も行わない（空の関数となる）。

保守担当者向けメモ

  • 本関数はプラットフォーム依存の最適化機能であるため、OSが copy_file_range をサポートしている環境でのみ動作する。
  • エラーメッセージには %m を使用しており、システムエラーの詳細がログに出力される。
