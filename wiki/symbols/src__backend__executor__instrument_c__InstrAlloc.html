InstrAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/executor/instrument.c:30

目的

クエリ実行計画のノードで使用する Instrumentation 構造体の配列をメモリ上に確保し、指定されたオプションに基づいて初期化します。

シグネチャまたは形状

Instrumentation *InstrAlloc(int n, int instrument_options, bool async_mode)

入力と出力

  • **入力**:
  • n: 確保する Instrumentation 構造体の数。
  • instrument_options: 計測対象を指定するビットマスク（INSTRUMENT_BUFFERS, INSTRUMENT_TIMER, INSTRUMENT_WAL）。
  • async_mode: 非同期モードで動作するかどうかを示すフラグ。
  • **出力**:
  • 確保・初期化された Instrumentation 構造体配列へのポインタ。

前提条件と不変条件

  • palloc0 を使用するため、PostgreSQL のメモリコンテキスト内で呼び出される必要があります。
  • instrument_options に指定されたフラグに応じて、各構造体の need_bufusage, need_walusage, need_timer フィールドが適切に設定されます。

アルゴリズムまたは振る舞い

1. palloc0 を使用して、n 個分の Instrumentation 構造体領域をゼロクリアして確保します。 2. instrument_options に INSTRUMENT_BUFFERS, INSTRUMENT_TIMER, または INSTRUMENT_WAL が含まれている場合、配列内の全要素に対して以下のフラグを設定します。

  • need_bufusage: INSTRUMENT_BUFFERS が指定されているか。
  • need_walusage: INSTRUMENT_WAL が指定されているか。
  • need_timer: INSTRUMENT_TIMER が指定されているか。
  • async_mode: 引数 async_mode の値を設定。

    • 3. 確保した配列のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/auto_explain/auto_explain.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/backend/executor/execMain.c
  • src/backend/executor/execProcnode.c
  • **呼び出し先**:
  • palloc0 (PostgreSQL メモリ管理関数)

関連データ構造

  • Instrumentation: 実行統計を保持する主要構造体。
  • instrument_options: 計測の有効範囲を制御するビットマスク定数。

失敗モードと境界条件

  • n が 0 の場合、palloc0 は空のメモリ領域を返します。
  • メモリ不足時には palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本関数は Instrumentation の初期化のみを担当します。統計の集計や更新は InstrStartNode や InstrStopNode 等の別関数で行われます。
  • instrument_options のビットフラグ定義は src/include/executor/instrument.h を参照してください。
