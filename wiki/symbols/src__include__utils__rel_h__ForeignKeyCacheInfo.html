ForeignKeyCacheInfo

定義

  • 種別: struct
  • 定義箇所: src/include/utils/rel.h:273

目的

外部キー制約に関するメタデータをメモリ上で保持し、リレーションキャッシュ（RelCache）の一部として制約の検証や最適化を高速化するために使用される。

シグネチャまたは形状

typedef struct ForeignKeyCacheInfo
{
    NodeTag     type;
    Oid         conoid;
    Oid         conrelid;
    Oid         confrelid;
    int         nkeys;
    bool        conenforced;
    AttrNumber  conkey[INDEX_MAX_KEYS];
    AttrNumber  confkey[INDEX_MAX_KEYS];
    Oid         conpfeqop[INDEX_MAX_KEYS];
} ForeignKeyCacheInfo;

入力と出力

  • 本構造体は makeNode(ForeignKeyCacheInfo) を通じて生成される。
  • nkeys は、conkey、confkey、conpfeqop 配列内の有効なエントリ数を規定する。

前提条件と不変条件

  • conkey、confkey、conpfeqop の各配列は INDEX_MAX_KEYS のサイズを持ち、nkeys 分のデータのみが有効である。
  • pg_node_attr により、equal、read、query_jumble の自動生成対象から除外されている。

アルゴリズムまたは振る舞い

  • リレーションが制約される際、その外部キー情報をキャッシュとして保持する。
  • パーティションテーブルの制約比較や、プランナによる最適化（plancat.c）において、制約の定義を迅速に参照するために利用される。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/commands/tablecmds.c: パーティションテーブルの外部キー整合性チェックや比較処理。
  • src/backend/optimizer/util/plancat.c: クエリ最適化時の制約情報取得。
  • **呼び出し先（生成元）**:
  • src/backend/utils/cache/relcache.c: リレーションキャッシュ構築時に本構造体を生成し、リストとして返す。

関連データ構造

  • NodeTag: PostgreSQL のノードシステム識別子。
  • Oid: システムカタログ内のオブジェクト識別子。
  • AttrNumber: テーブルの属性（列）番号。

失敗モードと境界条件

  • nkeys が INDEX_MAX_KEYS を超えることはない（システム定義の制約による）。
  • キャッシュが無効化された場合、本構造体を含むリストも再構築が必要となる。

保守担当者向けメモ

  • 本構造体は RelationData の一部として管理されるメタデータである。
  • フィールドの追加や変更を行う際は、src/backend/utils/cache/relcache.c 内のキャッシュ構築ロジックと整合性をとること。
  • conkey 等の配列は nkeys に依存したサイズ制限があるため、アクセス時には必ず nkeys を確認すること。
