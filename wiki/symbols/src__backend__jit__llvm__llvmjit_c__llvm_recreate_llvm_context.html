llvm_recreate_llvm_context

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit.c:172

目的

LLVM コンテキストのメモリ消費を抑制するため、再利用回数が上限に達した際にコンテキストを破棄・再生成し、関連する型情報を再構築する。

シグネチャまたは形状

static void llvm_recreate_llvm_context(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルな llvm_context の更新と型キャッシュの再構築を行う）

前提条件と不変条件

  • llvm_context が存在していること（存在しない場合はエラー）。
  • JIT コンパイル中のコードが存在しないこと（llvm_jit_context_in_use_count == 0）。

アルゴリズムまたは振る舞い

1. **ガードチェック**: llvm_context の存在を確認し、JIT コンパイルが進行中であれば再利用回数をインクリメントして終了する。 2. **再利用判定**: 再利用回数が LLVMJIT_LLVM_CONTEXT_REUSE_MAX 以下であれば、回数をインクリメントして終了する。 3. **キャッシュリセット**: llvm_inline_reset_caches() を呼び出し、コンテキスト破棄によるダングリングポインタを防ぐ。 4. **コンテキスト再生成**: LLVMContextDispose で既存コンテキストを破棄し、LLVMContextCreate で新規作成する。 5. **状態リセット**: 再利用回数を 0 にリセットする。 6. **型再構築**: llvm_create_types() を呼び出し、LLVM IR 上の型情報を再定義する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/jit/llvm/llvmjit.c 内のコンテキスト管理フロー（詳細はソースコードの文脈に依存）。
  • 呼び出し先: llvm_inline_reset_caches(), LLVMContextDispose(), LLVMContextCreate(), llvm_create_types(), elog()。

関連データ構造

  • llvm_context: LLVM のコンテキストオブジェクト。
  • llvm_jit_context_in_use_count: 現在使用中の JIT コンテキスト数。
  • llvm_llvm_context_reuse_count: コンテキストの再利用回数。

失敗モードと境界条件

  • llvm_context が未初期化の状態で呼び出された場合、elog(ERROR) によりトランザクションがアボートされる。
  • JIT コンパイルが進行中の場合、コンテキストの再生成はスキップされる。

保守担当者向けメモ

  • 本関数は LLVM のメモリリーク対策として重要である。
  • llvm_create_types() の呼び出しは、生成されたコードが古い型定義への参照を持たないようにするために不可欠である。
