XLogPrefetchShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogprefetcher.c:314

目的

WAL先読み機能の統計情報を保持するための共有メモリ領域を確保および初期化します。

シグネチャまたは形状

void XLogPrefetchShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上に XLogPrefetchStats 構造体を割り当て、初期化する）

前提条件と不変条件

  • 本関数は、PostgreSQLの共有メモリ初期化プロセスの一部として呼び出される必要があります。
  • ShmemInitStruct により、XLogPrefetchStats 型の領域が確保されます。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、XLogPrefetchStats 構造体用の共有メモリ領域を取得します。 2. 既に領域が確保されている場合（found が真）、初期化をスキップします。 3. 初回呼び出し時（found が偽）の場合、pg_atomic_init_u64 を使用して、統計カウンタ（prefetch, hit, 各種 skip_*）および reset_time を 0 または現在時刻で初期化します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:289 （共有メモリ初期化シーケンス内）
  • 呼び出し先: ShmemInitStruct, pg_atomic_init_u64, GetCurrentTimestamp

関連データ構造

  • XLogPrefetchStats: 先読み処理の統計情報を格納する構造体。
  • SharedStats: 本関数内で初期化されるグローバルな統計情報ポインタ。

失敗モードと境界条件

  • 共有メモリの割り当てに失敗した場合、ShmemInitStruct の内部実装に従い、プロセスが異常終了する可能性があります。

保守担当者向けメモ

  • 本関数はシステム起動時の共有メモリ初期化フェーズで一度だけ実行されることを想定しています。
  • 統計カウンタはアトミック操作（pg_atomic_u64）で管理されているため、マルチスレッド環境下での更新の整合性が保証されています。
