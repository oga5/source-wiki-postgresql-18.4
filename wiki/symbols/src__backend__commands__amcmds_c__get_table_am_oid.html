get_table_am_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/amcmds.c:172

目的

指定された名前のテーブルアクセスメソッド（AM）に対応する OID を取得します。

シグネチャまたは形状

Oid get_table_am_oid(const char *amname, bool missing_ok);

入力と出力

  • **入力**:
  • amname: 検索対象のアクセスメソッド名。
  • missing_ok: true の場合、対象が見つからない場合にエラーを発生させず InvalidOid を返します。false の場合はエラーを発生させます。
  • **出力**:
  • 対象アクセスメソッドの Oid。

前提条件と不変条件

  • pg_am システムカタログにアクセス可能である必要があります。
  • 内部的に AMTYPE_TABLE を指定して検索を行うため、テーブル型として定義されたアクセスメソッドのみが対象となります。

アルゴリズムまたは振る舞い

1. 内部関数 get_am_type_oid を呼び出します。 2. その際、第2引数に AMTYPE_TABLE を固定値として渡し、テーブルアクセスメソッドに限定した検索を実行します。 3. 検索結果（OID または InvalidOid）をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/table/tableamapi.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • get_am_type_oid (同一ファイル内)

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。
  • pg_am: アクセスメソッドを管理するシステムカタログ。

失敗モードと境界条件

  • amname に該当するアクセスメソッドが存在しない場合:
  • missing_ok が false ならばエラーが発生します。
  • missing_ok が true ならば InvalidOid が返されます。
  • 指定された名前がテーブル型以外のアクセスメソッド（例: インデックス用）を指している場合、get_am_type_oid の仕様に従い、見つからないものとして扱われます。

保守担当者向けメモ

  • 本関数は get_am_type_oid のラッパーです。テーブルアクセスメソッドの検索ロジックを変更する場合は、get_am_type_oid の実装を確認してください。
