IpcSemaphoreInitialize

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_sema.c:156

目的

System V セマフォセット内の特定のセマフォに対して、初期値を設定します。

シグネチャまたは形状

static void IpcSemaphoreInitialize(IpcSemaphoreId semId, int semNum, int value)

入力と出力

  • **入力**:
  • semId (IpcSemaphoreId): 操作対象のセマフォセット ID。
  • semNum (int): セット内の対象セマフォのインデックス。
  • value (int): 設定する初期値。
  • **出力**: なし。成功時は値を設定し、失敗時は FATAL エラーでプロセスを終了します。

前提条件と不変条件

  • semId は有効な System V セマフォセット ID である必要があります。
  • カーネルの SEMVMX（セマフォの最大値）制限が、指定された value を許容している必要があります。

アルゴリズムまたは振る舞い

1. union semun 構造体に value を格納します。 2. semctl システムコールを SETVAL コマンドで呼び出し、指定されたセマフォの値を更新します。 3. semctl が負の値を返した場合、errno を保存し、FATAL レベルでエラーを報告します。

  • errno が ERANGE の場合、カーネルの SEMVMX パラメータ不足を示唆するヒントをエラーメッセージに付加します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/sysv_sema.c:294
  • src/backend/port/sysv_sema.c:412
  • src/backend/port/sysv_sema.c:425
  • **呼び出し先**:
  • semctl (システムコール)

関連データ構造

  • IpcSemaphoreId: セマフォセットを識別するための型。
  • union semun: semctl システムコールで使用される共用体。

失敗モードと境界条件

  • semctl が失敗した場合、FATAL エラーにより PostgreSQL バックエンドプロセスが停止します。
  • value がカーネルの許容範囲（SEMVMX）を超えている場合、ERANGE エラーが発生し、カーネル設定の変更を促すメッセージが出力されます。

保守担当者向けメモ

  • 本関数は semop ではなく semctl を使用して直接値を設定します。
  • エラー発生時の ereport は、カーネルパラメータの調整が必要な場合にユーザーへ具体的な指針を与えるよう設計されています。
