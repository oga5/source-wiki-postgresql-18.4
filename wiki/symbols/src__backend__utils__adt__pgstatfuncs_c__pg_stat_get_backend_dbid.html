pg_stat_get_backend_dbid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pgstatfuncs.c:709

目的

指定されたバックエンドプロセス番号に対応するデータベースのOID（Object Identifier）を取得します。

シグネチャまたは形状

Datum pg_stat_get_backend_dbid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT32(0) を介してバックエンドプロセス番号（int32）を受け取ります。
  • **出力**: 成功時は対象バックエンドが接続しているデータベースのOIDを Datum 型で返します。該当するバックエンドが存在しない場合は NULL を返します。

前提条件と不変条件

  • 統計情報システムが初期化されており、バックエンド状態情報にアクセス可能である必要があります。
  • 渡されるプロセス番号は、有効なバックエンドプロセスを特定するための識別子として機能します。

アルゴリズムまたは振る舞い

1. 引数として受け取ったプロセス番号を pgstat_get_beentry_by_proc_number に渡し、対応する PgBackendStatus 構造体を取得します。 2. 取得したポインタが NULL の場合、PG_RETURN_NULL() を呼び出し終了します。 3. ポインタが有効な場合、構造体内の st_databaseid フィールドを参照し、その値を PG_RETURN_OID マクロで返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（具体的なSQLクエリの文脈はソースコード上では明示されていません）。
  • **呼び出し先**:
  • pgstat_get_beentry_by_proc_number (統計情報システムからバックエンド情報を取得)

関連データ構造

  • PgBackendStatus: バックエンドの稼働状況を保持する構造体。st_databaseid フィールドを介してデータベースIDを保持します。

失敗モードと境界条件

  • 指定されたプロセス番号に対応するバックエンドが存在しない場合、関数は安全に NULL を返します。

保守担当者向けメモ

  • 本関数は pgstat システムの内部構造に直接依存しています。PgBackendStatus 構造体の定義が変更された場合、本関数の参照先フィールドを確認してください。
  • 権限チェックについては、本関数単体では明示的な HAS_PGSTAT_PERMISSIONS 呼び出しが見当たりません。呼び出し元または上位レイヤーでの権限管理状況を確認してください。
