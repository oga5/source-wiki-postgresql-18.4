InitPartitionPruneContext

定義

  • **種別**: function
  • **定義箇所**: src/backend/executor/execPartition.c:2238

目的

実行時パーティションプルーニングに必要なコンテキスト構造体（PartitionPruneContext）を初期化し、パーティションキーのメタデータおよび式評価のための状態を準備します。

シグネチャまたは形状

static void InitPartitionPruneContext(PartitionPruneContext *context,
                                      List *pruning_steps,
                                      PartitionDesc partdesc,
                                      PartitionKey partkey,
                                      PlanState *planstate,
                                      ExprContext *econtext)

入力と出力

  • **入力**:
  • context: 初期化対象の PartitionPruneContext 構造体へのポインタ。
  • pruning_steps: パーティションプルーニングの手順リスト。
  • partdesc: パーティション記述子。
  • partkey: パーティションキー情報。
  • planstate: 関連するプランノードの状態（NULL の場合あり）。
  • econtext: 式評価用のコンテキスト。
  • **出力**: なし（context 構造体の各フィールドが更新されます）。

前提条件と不変条件

  • partkey および partdesc は有効なポインタであること。
  • pruning_steps 内の各ステップは、PartitionPruneStepOp 型として処理されます。
  • planstate が NULL の場合、式評価には econtext に含まれる外部パラメータ情報が使用されます。

アルゴリズムまたは振る舞い

1. partkey および partdesc から戦略、パーティション数、境界情報、照合順序、サポート関数を context にコピーします。 2. n_steps と partnatts に基づき、stepcmpfuncs および exprstates 用のメモリを palloc0 で確保します。 3. pruning_steps を走査し、PartitionPruneStepOp 型のステップに対して以下を実行します。

  • 各キー属性について、NULL キーでないことを確認します。
  • 式が Const でない場合、planstate の有無に応じて ExecInitExpr または ExecInitExprWithParams を呼び出し、式評価状態を exprstates に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execPartition.c 内の InitPartitionPruneContext への参照（行 187, 2186, 2416）。
  • **呼び出し先**:
  • ExecInitExpr / ExecInitExprWithParams（式評価状態の初期化）
  • palloc0（メモリ確保）
  • list_length, list_head, lnext（リスト操作）
  • IsA, lfirst（ノード操作）
  • bms_is_member（ビットマップ操作）
  • PruneCxtStateIdx（インデックス計算）

関連データ構造

  • PartitionPruneContext: プルーニング実行時の状態を保持するメイン構造体。
  • PartitionPruneStepOp: プルーニングの各ステップを定義する構造体。
  • PartitionDesc, PartitionKey: パーティションの構造とキー定義。
  • ExprState: 式評価のための実行時状態。

失敗モードと境界条件

  • planstate が NULL かつ econtext に必要なパラメータ情報が不足している場合、式評価時にエラーが発生する可能性があります。
  • step->exprs の長さが partnatts を超える場合、Assert により停止します。

保守担当者向けメモ

  • 本関数は PartitionPruneContext の初期化に特化しており、メモリ管理は CurrentMemoryContext に依存します。
  • planstate が NULL の場合の挙動は、プランノードに依存しないプルーニング（初期プルーニングなど）を想定しています。
