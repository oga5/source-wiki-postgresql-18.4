DatumGetFullTransactionId

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/xid8.h:17

目的

PostgreSQL の汎用データ型 Datum を、64 ビットトランザクション ID を表す内部型 FullTransactionId へ変換する。

シグネチャまたは形状

static inline FullTransactionId DatumGetFullTransactionId(Datum X)

入力と出力

  • **入力**: Datum X（FullTransactionId を保持する Datum 型の値）
  • **出力**: FullTransactionId 型の変換済み値

前提条件と不変条件

  • 入力 Datum X は、FullTransactionId として正しく解釈可能な形式である必要がある。
  • access/transam.h で定義される FullTransactionId 型および関連関数が利用可能であること。

アルゴリズムまたは振る舞い

1. DatumGetUInt64(X) を呼び出し、Datum を 64 ビット整数値に変換する。 2. その結果を FullTransactionIdFromU64() に渡し、FullTransactionId 型へ変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/xid8.h:29 に定義されたマクロ PG_GETARG_FULLTRANSACTIONID(X) から直接呼び出される。
  • **呼び出し先**:
  • DatumGetUInt64(X)
  • FullTransactionIdFromU64(uint64)

関連データ構造

  • FullTransactionId: 64 ビットトランザクション ID を保持する型。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • 入力 Datum が FullTransactionId 以外の型を保持している場合、変換結果は不定となる。
  • 内部で使用する DatumGetUInt64 および FullTransactionIdFromU64 の制約に依存する。

保守担当者向けメモ

  • 本関数は xid8.h 内の軽量なラッパーであり、SQL 関数インターフェースと内部のトランザクション管理サブシステムを繋ぐ役割を持つ。
  • 32 ビットの TransactionId と混同しないよう注意すること。変更が必要な場合は access/transam.h の定義と整合性を取ること。
