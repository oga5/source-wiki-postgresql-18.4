GinTuple

定義

  • 種別: struct
  • 定義箇所: src/include/access/gin_tuple.h:20

目的

GIN（Generalized Inverted Index）インデックスにおいて、キー値とそれに関連するTID（Tuple Identifier）リストを保持する物理的なデータ格納構造を定義します。

シグネチャまたは形状

typedef struct GinTuple
{
    int         tuplen;         /* タプル全体の長さ */
    OffsetNumber attrnum;       /* インデックスキーの属性番号 */
    uint16      keylen;         /* キー値のデータ長（バイト） */
    int16       typlen;         /* キーの型長 */
    bool        typbyval;       /* キーの型が値渡し可能か */
    signed char category;       /* カテゴリ（通常またはNULL） */
    int         nitems;         /* データ内のTID数 */
    char        data[FLEXIBLE_ARRAY_MEMBER]; /* 可変長データ領域 */
} GinTuple;

入力と出力

  • 本構造体はインデックスページ上のデータをキャストして利用されます。
  • data 領域にはキー値および GinPostingList が格納されます。

前提条件と不変条件

  • FLEXIBLE_ARRAY_MEMBER を使用した可変長構造体であるため、メモリ確保やポインタ演算時には tuplen や keylen を用いた厳密な境界チェックが必須です。
  • GinTupleGetFirst() を用いて data 領域へアクセスする際は、keylen をスキップし、適切なメモリアライメント（SHORTALIGN 等）を遵守する必要があります。

アルゴリズムまたは振る舞い

  • GINインデックスの構築や検索時、キーのシリアライズ形式として機能します。
  • インデックスページから読み込まれたバイナリデータは、本構造体として解釈され、キー値の抽出やTIDリストの走査が行われます。
  • 比較処理（_gin_compare_tuples）において、キー値の大小関係やTIDの順序に基づいたソート順の決定に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/access/gin/gininsert.c: _gin_parse_tuple_items, _gin_parse_tuple_key, _gin_build_tuple, GinBufferKeyEquals, GinBufferShouldTrim, GinBufferStoreTuple, GinBufferCanAddKey 等で利用されています。
  • 呼び出し先:
  • 構造体定義のため、特定の関数呼び出しは行いません。

関連データ構造

  • GinPostingList: data 領域内に格納されるTIDリスト構造。
  • ItemPointer: data 領域から抽出されるTIDの型。
  • SortSupport: ソート時の比較ロジック最適化に使用。

失敗モードと境界条件

  • data 領域のオフセット計算を誤ると、メモリ破壊や不正なメモリアクセスが発生します。
  • tuplen が実際のデータサイズと一致しない場合、インデックスの破損を招く可能性があります。

保守担当者向けメモ

  • GinTuple は物理的な格納形式を規定するため、変更時は ginblock.h で定義されるページ構造との整合性に注意してください。
  • 直接ポインタを操作する際は、SHORTALIGN によるパディング規則を遵守してください。
