textregexreplace_extended

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regexp.c:699

目的

SQLの regexp_replace 関数を拡張し、開始位置、置換回数、およびフラグを指定した正規表現置換を実行する。

シグネチャまたは形状

Datum textregexreplace_extended(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_FUNCTION_ARGS: SQL関数呼び出し引数。
  • s (text): 対象文字列。
  • p (text): 正規表現パターン。
  • r (text): 置換後の文字列。
  • start (int, オプション): 置換を開始する位置（1始まり）。
  • n (int, オプション): 置換回数。
  • flags (text, オプション): 正規表現フラグ（'g', 'i' 等）。
  • **出力**:
  • Datum: 置換後の text 型データ。

前提条件と不変条件

  • start 引数が指定された場合、1以上である必要がある。
  • n 引数が指定された場合、0以上である必要がある。
  • flags が指定されない場合、デフォルトの動作が適用される。

アルゴリズムまたは振る舞い

1. 引数 s, p, r を取得する。 2. オプション引数 start と n が存在する場合、値を読み取り、負の値であればエラーを報告する。 3. parse_re_flags を呼び出し、flags 文字列を pg_re_flags 構造体に変換する。 4. n が指定されていない場合、re_flags.glob（'g'フラグ）の有無に基づき、0（すべて置換）または1（最初のみ置換）を決定する。 5. replace_text_regexp を呼び出し、実際の置換処理を実行する。その際、start - 1 を渡して0始まりのインデックスに調整する。 6. 置換結果を PG_RETURN_TEXT_P で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元として src/backend/utils/adt/regexp.c 内のラッパー関数群（textregexreplace_extended_no_n, textregexreplace_extended_no_flags 等）が確認されている。
  • **呼び出し先**: parse_re_flags, replace_text_regexp。

関連データ構造

  • pg_re_flags: 正規表現のコンパイルオプションを保持する構造体。
  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • start <= 0: ERRCODE_INVALID_PARAMETER_VALUE エラーを発生させる。
  • n < 0: ERRCODE_INVALID_PARAMETER_VALUE エラーを発生させる。

保守担当者向けメモ

  • 本関数は regexp_replace の多機能版インターフェースである。引数の数に応じて動的に挙動を切り替えるため、引数追加時の整合性に注意すること。
  • replace_text_regexp への引数として start - 1 を渡している点に留意（内部的なインデックス調整）。
