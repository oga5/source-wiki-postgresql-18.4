manifest_data

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_combinebackup/load_manifest.h:56

目的

PostgreSQL のバックアップマニフェスト（backup_manifest）の内容をメモリ上で保持し、バックアップ統合処理における構成情報の管理を行うためのルート構造体です。

シグネチャまたは形状

typedef struct manifest_data
{
	uint64		system_identifier;
	manifest_files_hash *files;
	manifest_wal_range *first_wal_range;
	manifest_wal_range *last_wal_range;
} manifest_data;

入力と出力

  • 入力: load_backup_manifest() 関数により、指定されたディレクトリ内のマニフェストファイルからデータが読み込まれ、本構造体に格納されます。
  • 出力: 解析結果として manifest_data 構造体へのポインタが呼び出し元へ返却されます。

前提条件と不変条件

  • files は simplehash を用いたハッシュテーブルであり、ファイル情報の検索に使用されます。
  • first_wal_range および last_wal_range は、WAL 範囲を管理する双方向リストの先頭と末尾を指します。

アルゴリズムまたは振る舞い

1. load_backup_manifest() がマニフェストファイルを読み込み、JSON 解析コンテキストを初期化します。 2. ファイル情報を files ハッシュテーブルに格納します。 3. WAL 範囲情報をリストとして連結し、first_wal_range と last_wal_range で管理します。 4. 最終的に構築された manifest_data 構造体を返却します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_combinebackup/load_manifest.c 内の各関数（load_backup_manifest 等）が本構造体を生成・操作します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出し先はありません。

関連データ構造

  • manifest_files_hash: ファイル検索用ハッシュテーブル。
  • manifest_wal_range: WAL のタイムラインと LSN 範囲を保持するリスト要素。

失敗モードと境界条件

  • load_backup_manifest 実行時にメモリ不足が発生した場合、pg_malloc0 等によりプロセスが終了する可能性があります。
  • 複数のバックアップを扱う場合、manifest_data の配列として管理されることが想定されており、メモリ管理の不備はリークの原因となります。

保守担当者向けメモ

  • simplehash を利用しているため、ハッシュテーブル操作関数は load_manifest.c で生成されます。
  • manifest_data は load_backup_manifests を通じて複数取得される設計であるため、メモリのライフサイクル管理に注意してください。
