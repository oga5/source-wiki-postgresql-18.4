statext_ndistinct_serialize

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mvdistinct.c:178

目的

メモリ上の MVNDistinct 構造体を、システムカタログ（pg_statistic_ext_data）へ格納可能なバイナリ形式（bytea）に変換します。

シグネチャまたは形状

bytea *statext_ndistinct_serialize(MVNDistinct *ndistinct)

入力と出力

  • **入力**: MVNDistinct *ndistinct （シリアライズ対象の多変量NDistinct統計情報構造体）
  • **出力**: bytea * （シリアライズされたデータを含むメモリ領域）

前提条件と不変条件

  • ndistinct->magic が STATS_NDISTINCT_MAGIC と一致すること。
  • ndistinct->type が STATS_NDISTINCT_TYPE_BASIC であること。
  • 各アイテムの属性数（nattributes）が2以上であること。
  • 処理終了時に、書き込みポインタ tmp が計算されたバッファ末尾と一致すること。

アルゴリズムまたは振る舞い

1. 構造体のヘッダーサイズおよび各アイテムの属性数に基づき、必要なメモリサイズを算出します。 2. palloc でメモリを確保し、SET_VARSIZE で bytea のサイズを設定します。 3. magic、type、nitems をバッファの先頭にコピーします。 4. 各アイテムについて、ndistinct 値、属性数、および属性番号の配列を順次バッファへコピーします。 5. 最後に、確保したメモリ領域を過不足なく使用したことをアサーションで検証し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/extended_stats.c:796
  • **呼び出し先**: 本関数内での直接的な関数呼び出しはありません（メモリ操作およびアサーションのみ）。

関連データ構造

  • MVNDistinct: シリアライズ対象の統計情報構造体。
  • MVNDistinctItem: MVNDistinct 内の各列組み合わせを保持する構造体。
  • bytea: PostgreSQLの可変長バイナリデータ型。

失敗モードと境界条件

  • nattributes < 2 の場合、アサーション違反により異常終了します。
  • メモリ割り当て（palloc）が失敗した場合は、PostgreSQLの標準的なメモリ管理機構によりエラーが発生します。
  • バッファの書き込みが計算サイズを超過した場合、アサーションにより検出されます。

保守担当者向けメモ

  • 本関数は MVNDistinct 構造体のメモリレイアウトに強く依存しています。構造体定義に変更を加える際は、本関数のシリアライズロジックおよび対応する statext_ndistinct_deserialize の双方を更新する必要があります。
