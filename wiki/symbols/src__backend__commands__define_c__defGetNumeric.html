defGetNumeric

定義

  • 種別: function
  • 定義箇所: src/backend/commands/define.c:67

目的

DefElem ノードから数値（double）を抽出するユーティリティ関数です。SQL コマンドのオプション引数解析において、数値指定が必要な箇所で利用されます。

シグネチャまたは形状

double defGetNumeric(DefElem *def)

入力と出力

  • **入力**: DefElem *def （解析対象の定義要素ノード）
  • **出力**: double 型の数値
  • **戻り値**: ノード内の値を double に変換した値。

前提条件と不変条件

  • def->arg が NULL であってはならない。
  • def->arg は T_Integer または T_Float 型のノードである必要がある。

アルゴリズムまたは振る舞い

1. def->arg が NULL の場合、ERRCODE_SYNTAX_ERROR を伴うエラーを報告する。 2. nodeTag(def->arg) を評価する。

  • T_Integer の場合: intVal() を呼び出し、その結果を double にキャストして返す。
  • T_Float の場合: floatVal() を呼び出し、その結果を返す。

    • 3. 上記以外のノード型の場合、ERRCODE_SYNTAX_ERROR を伴うエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/functioncmds.c （procost および prorows の取得時に使用）
  • **呼び出し先**:
  • intVal (マクロ)
  • floatVal (マクロ)
  • ereport (エラー報告用)

関連データ構造

  • DefElem: src/include/nodes/parsenodes.h で定義される、SQL コマンドのオプション引数を保持する構造体。

失敗モードと境界条件

  • def->arg が NULL の場合: ereport により処理が中断される。
  • def->arg が T_Integer や T_Float 以外の型（例: T_String など）である場合: ereport により処理が中断される。

保守担当者向けメモ

  • 本関数は純粋なデータ変換ユーティリティであり、データベースの状態変更は行いません。
  • パーサーが数値を Integer ではなく Float として解釈した場合（範囲外の値など）も考慮した実装になっています。
  • 新しい数値関連のオプションを追加する際は、本関数が利用可能か検討してください。
