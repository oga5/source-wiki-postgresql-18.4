heap_toast_insert_or_update

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heaptoast.c:95

目的

タプル挿入・更新時に、属性値がページサイズ制限を超過する場合に圧縮や外部保存（TOAST）を行い、タプルを目標サイズ以下に整形する。

シグネチャまたは形状

HeapTuple heap_toast_insert_or_update(Relation rel, HeapTuple newtup, HeapTuple oldtup, int options)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • newtup: 挿入または更新対象のヒープタプル。
  • oldtup: 更新前のタプル（新規挿入時は NULL）。
  • options: 処理オプション（HEAP_INSERT_SPECULATIVE は内部で除外される）。
  • **出力**:
  • 整形後の HeapTuple。必要に応じて palloc された新しいタプルを返す。

前提条件と不変条件

  • 対象リレーションは RELKIND_RELATION または RELKIND_MATVIEW であること（TOASTリレーション自体への再帰呼び出しは禁止）。
  • numAttrs は MaxHeapAttributeNumber 以下であること。

アルゴリズムまたは振る舞い

1. **初期化**: heap_deform_tuple でタプルを分解し、ToastTupleContext を初期化。 2. **圧縮・外部保存の段階的実行**:

  • **段階1**: EXTENDED 属性の圧縮、および巨大な EXTENDED/EXTERNAL 属性の外部保存。
  • **段階2**: 残りの EXTENDED/EXTERNAL 属性の外部保存。
  • **段階3**: MAIN 属性の圧縮。
  • **段階4**: MAIN 属性の外部保存（目標サイズを緩和して実施）。

    • 3. **制御**: 各段階で heap_compute_data_size が目標サイズ（maxDataLen）以下になるまで、toast_tuple_find_biggest_attribute で最大の属性を特定し処理を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/heap/rewriteheap.c
  • **呼び出し先**:
  • heap_deform_tuple
  • toast_tuple_init
  • toast_tuple_find_biggest_attribute
  • toast_tuple_try_compression
  • toast_tuple_externalize
  • heap_compute_data_size

関連データ構造

  • ToastTupleContext: TOAST処理中の状態管理用構造体。
  • ToastAttrInfo: 各属性のTOAST状態（サイズ、フラグ等）を保持。
  • Relation: リレーション情報（TOASTテーブルのOID等を参照）。

失敗モードと境界条件

  • TOASTテーブルが存在しない場合、EXTERNAL 属性の外部保存はスキップされる。
  • 属性が圧縮不能（TOASTCOL_INCOMPRESSIBLE）な場合は圧縮を試行しない。
  • 巨大な属性が単独で maxDataLen を超える場合、他の属性の圧縮を待たずに即座に外部保存される。

保守担当者向けメモ

  • 本関数は元のタプルを破壊せず、変更が必要な場合は新しいタプルを生成する。
  • TOAST_TUPLE_TARGET および TOAST_TUPLE_TARGET_MAIN が目標サイズの基準となる。
  • 処理の詳細は toast_internals.c で定義された低レベル関数に依存している。
