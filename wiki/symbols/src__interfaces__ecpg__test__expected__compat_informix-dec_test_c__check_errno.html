check_errno

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-dec_test.c:270

目的

グローバル変数 errno の値を解析し、対応するエラーメッセージを標準出力へ書き出すヘルパー関数です。

シグネチャまたは形状

void check_errno(void)

入力と出力

  • **入力**: グローバル変数 errno（システムエラーコードまたは ECPG/Informix 固有のエラーコード）。
  • **出力**: なし（標準出力への文字列書き出しのみ）。

前提条件と不変条件

  • errno が適切なエラーコードまたは 0 に設定されている状態で呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. errno の値を switch 文で評価します。 2. 既知の ECPG 関連エラー（ECPG_INFORMIX_NUM_OVERFLOW 等）および PGTYPES 関連エラー（PGTYPES_NUM_OVERFLOW 等）に対して、対応するエラー名を標準出力に表示します。 3. errno が 0 の場合は "(no errno set)" と表示します。 4. 上記のいずれにも該当しない場合、default ケースとして errno の数値と strerror(errno) による libc のエラー説明を表示します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/compat_informix-dec_test.c 内の main 関数から、数値変換や演算処理後のエラーチェックとして複数回呼び出されています（行 85, 107, 116, 120, 123, 126, 131, 135, 140, 147, 152, 163）。
  • **呼び出し先**: printf および strerror（標準ライブラリ）。

関連データ構造

  • errno: システムおよびライブラリのエラー状態を保持するグローバル変数。

失敗モードと境界条件

  • errno が定義済みのケースに含まれない場合、default ブロックが実行され、システム標準の strerror を用いた出力が行われます。

保守担当者向けメモ

  • 本関数はテストコード内の診断用ユーティリティです。新しいエラーコードが追加された場合は、switch 文にケースを追加してください。
