PreRestoreCommand

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:267

目的

外部コマンド（restore_command）の実行直前に呼び出され、SIGTERMシグナル受信時の挙動を制御するための状態遷移を行う。

シグネチャまたは形状

void PreRestoreCommand(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数を更新）

前提条件と不変条件

  • 本関数は、外部コマンド実行中であってもSIGTERMによる即時終了が安全なタイミングで呼び出されることを前提とする。

アルゴリズムまたは振る舞い

1. グローバル変数 in_restore_command を true に設定する。これにより、シグナルハンドラに対し、SIGTERM受信時に即時終了して良い状態であることを通知する。 2. shutdown_requested フラグを確認する。本関数の呼び出し直前にシャットダウン要求が届いていた場合、即座に proc_exit(1) を呼び出してプロセスを終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlogarchive.c 内のアーカイブ復旧処理フロー。
  • 呼び出し先: proc_exit（条件合致時）。

関連データ構造

  • in_restore_command (bool): 外部コマンド実行中であることを示すフラグ。
  • shutdown_requested (bool): シャットダウン要求の有無を示すフラグ。

失敗モードと境界条件

  • 本関数呼び出しの直前にSIGTERMが受信されていた場合、shutdown_requested のチェックにより即座に終了する。

保守担当者向けメモ

  • 本関数は PostRestoreCommand と対で使用される。in_restore_command フラグの管理は、リカバリプロセスが外部コマンド実行中にハングアップせず、かつ安全に終了できるようにするための重要なガードである。
