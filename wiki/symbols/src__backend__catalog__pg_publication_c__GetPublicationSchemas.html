GetPublicationSchemas

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_publication.c:924

目的

指定されたパブリケーションID（pubid）に関連付けられているすべてのスキーマのOIDリストを取得します。

シグネチャまたは形状

List *GetPublicationSchemas(Oid pubid)

入力と出力

  • **入力**: pubid (Oid) - 対象となるパブリケーションのOID。
  • **出力**: List * - パブリケーションに関連付けられたスキーマOIDのリスト。

前提条件と不変条件

  • pg_publication_namespace システムカタログに対する読み取り権限が必要です。
  • 呼び出し元は、返されたリストに対してメモリ管理（解放）を行う責任があります。

アルゴリズムまたは振る舞い

1. pg_publication_namespace システムカタログを AccessShareLock で開きます。 2. PublicationNamespacePnnspidPnpubidIndexId インデックスを使用して、指定された pubid に一致するタプルをスキャンします。 3. スキャンで見つかった各タプルから pnnspid（スキーマOID）を抽出し、結果リストに追加します。 4. スキャンを終了し、カタログリレーションを閉じます。 5. 収集したスキーマOIDのリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c (内部利用)
  • src/backend/commands/publicationcmds.c
  • **呼び出し先**:
  • table_open / table_close
  • systable_beginscan / systable_getnext / systable_endscan
  • lappend_oid

関連データ構造

  • pg_publication_namespace: パブリケーションとスキーマの紐付けを保持するシステムカタログ。
  • Form_pg_publication_namespace: カタログタプルの構造体。

失敗モードと境界条件

  • 指定された pubid に紐付くスキーマが存在しない場合、空のリスト（NIL）を返します。
  • カタログアクセス時にエラーが発生した場合、PostgreSQLの標準的なエラーハンドリングによりトランザクションがアボートされる可能性があります。

保守担当者向けメモ

  • 本関数は pg_publication_namespace カタログを直接スキャンします。キャッシュの利用状況については、モジュール概要の「キャッシュ」セクションを参照してください。
