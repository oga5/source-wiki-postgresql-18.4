format_procedure_extended

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/regproc.c:325

目的

関数OIDを、引数リストを含む人間が読み取り可能な文字列（例: schema.func(type, type)）に変換します。フラグ制御により、スキーマ修飾の強制や、無効なOIDに対する挙動の切り替えが可能です。

シグネチャまたは形状

char *format_procedure_extended(Oid procedure_oid, bits16 flags)

入力と出力

  • **入力**:
  • procedure_oid: 変換対象の関数OID。
  • flags: 動作を制御するビットマスク。
  • FORMAT_PROC_FORCE_QUALIFY: スキーマ修飾を強制する。
  • FORMAT_PROC_INVALID_AS_NULL: OIDがカタログに存在しない場合、エラーにせず NULL を返す。
  • **出力**:
  • 成功時: 関数名と引数型を含む文字列（palloc 済み）。
  • 失敗時（OID未検出かつフラグなし）: OIDの数値表現文字列。
  • 失敗時（OID未検出かつ FORMAT_PROC_INVALID_AS_NULL 指定時）: NULL。

前提条件と不変条件

  • ブートストラップモード（IsBootstrapProcessingMode()）では実行できません。
  • pg_proc システムカタログへのアクセスが必要です。

アルゴリズムまたは振る舞い

1. PROCOID システムキャッシュから procedure_oid を検索します。 2. **OIDが存在する場合**:

  • FunctionIsVisible を確認し、修飾が不要かつフラグで強制されていない場合は名前のみを使用します。それ以外は get_namespace_name でスキーマ名を付与します。
  • quote_qualified_identifier で名前を整形し、引数型リストを format_type_be（または修飾付きの format_type_be_qualified）で順次連結します。
  • ReleaseSysCache でキャッシュを解放します。

    • 3. **OIDが存在しない場合**:

  • FORMAT_PROC_INVALID_AS_NULL フラグがあれば NULL を返します。
  • なければ、OIDの数値を文字列として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/regproc.c 内の format_procedure および format_procedure_qualified。
  • src/backend/catalog/objectaddress.c（オブジェクトアドレスの文字列化）。
  • **呼び出し先**:
  • SearchSysCache1, ReleaseSysCache
  • get_namespace_name, FunctionIsVisible
  • quote_qualified_identifier, initStringInfo, appendStringInfo
  • format_type_be, format_type_be_qualified

関連データ構造

  • Form_pg_proc: pg_proc カタログの行構造体。
  • StringInfoData: 文字列構築用バッファ。

失敗モードと境界条件

  • **OID未検出**: カタログに存在しないOIDが渡された場合、デフォルトではOIDの数値文字列を返しますが、FORMAT_PROC_INVALID_AS_NULL 指定時は NULL を返します。
  • **ブートストラップモード**: Assert により強制終了します。

保守担当者向けメモ

  • 本関数は regprocedure 型の出力処理の核となります。
  • 引数型のフォーマットに format_type_be を使用しているため、型名の表示形式は現在の検索パスや設定に依存します。
