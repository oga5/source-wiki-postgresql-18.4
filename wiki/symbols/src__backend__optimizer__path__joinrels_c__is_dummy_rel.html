is_dummy_rel

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinrels.c:1275

目的

指定されたリレーション（RelOptInfo）が、クエリの制約により空の結果セットを返すことが証明されている（ダミーである）かどうかを判定する。

シグネチャまたは形状

bool is_dummy_rel(RelOptInfo *rel)

入力と出力

  • **入力**: RelOptInfo *rel（評価対象のリレーション）
  • **出力**: bool（ダミーであれば true、そうでなければ false）

前提条件と不変条件

  • rel->pathlist が空の場合、この関数は false を返す。
  • ダミーリレーションは、パスリストの先頭に子を持たない Append パスを持つことが期待される。

アルゴリズムまたは振る舞い

1. rel->pathlist が空であれば false を返す。 2. パスリストの先頭要素（linitial）を取得する。 3. 取得したパスが ProjectionPath または ProjectSetPath である場合、それらの subpath を再帰的に辿り、元のパスを特定する。 4. 最終的に到達したパスに対して IS_DUMMY_APPEND マクロを適用し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinrels.c 内の複数の箇所（結合の正当性や最適化の判断ロジック）から呼び出される。
  • **呼び出し先**: IS_DUMMY_APPEND マクロ（パスが空の Append かを判定）。

関連データ構造

  • RelOptInfo: 評価対象のリレーション構造体。
  • Path: パスリスト内のノード。
  • ProjectionPath, ProjectSetPath: パスをラップするノード。

失敗モードと境界条件

  • rel->pathlist が NIL の場合は false を返す。
  • パスが ProjectionPath や ProjectSetPath でラップされている場合でも、正しく内部の Append パスまで到達して判定を行う。

保守担当者向けメモ

  • 本関数は src/include/nodes/pathnodes.h で定義されている IS_DUMMY_REL(r) マクロの本体である。
  • 計画の後半段階で ProjectionPath 等が追加される可能性があるため、パスの構造を辿るループは必須である。
