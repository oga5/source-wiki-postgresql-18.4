gbt_intveq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_interval.c:43

目的

2つの interval 型の値が等しいかどうかを判定し、GiST インデックスの比較処理をサポートします。

シグネチャまたは形状

static bool gbt_intveq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の interval 型データへのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では直接使用されません）。
  • **出力**:
  • bool: a と b が等しい場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な interval 型のデータ構造を指している必要があります。

アルゴリズムまたは振る舞い

1. IntervalPGetDatum マクロを使用して、入力ポインタを PostgreSQL の Datum 形式に変換します。 2. DirectFunctionCall2 を介して、PostgreSQL のコア関数である interval_eq を呼び出します。 3. interval_eq の戻り値（bool）を DatumGetBool で取得し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_interval.c 内の tinfo 構造体（gbtree_ninfo）の初期化時に参照されています。
  • **呼び出し先**: interval_eq（PostgreSQL コアの比較関数）。

関連データ構造

  • interval: 比較対象となる PostgreSQL の時間間隔型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、IntervalPGetDatum の内部で不正なメモリアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.h で定義される gbtree_ninfo 構造体の一部として使用されるラッパー関数です。
  • interval_eq に依存しているため、interval 型の比較ロジックに変更があった場合、本関数もその影響を受けます。
