CreateStatistics

定義

  • 種別: function
  • 定義箇所: src/backend/commands/statscmds.c:61

目的

CREATE STATISTICS SQLコマンドを実行し、指定されたリレーションに対する拡張統計情報オブジェクトをカタログ（pg_statistic_ext）に登録し、依存関係を構築する。

シグネチャまたは形状

ObjectAddress CreateStatistics(CreateStatsStmt *stmt, bool check_rights)

入力と出力

  • **入力**:
  • stmt: CreateStatsStmt 構造体（統計対象のリレーション、列、式、オプション情報を含む）。
  • check_rights: スキーマ作成権限のチェックを行うかどうかのフラグ。
  • **出力**:
  • 作成された統計オブジェクトの ObjectAddress を返す。既に存在し IF NOT EXISTS が指定されている場合は InvalidObjectAddress を返す。

前提条件と不変条件

  • 対象リレーションは単一である必要がある。
  • 呼び出し元は対象リレーションの所有者である必要がある。
  • システムカタログに対する統計作成は、allowSystemTableMods が有効でない限り禁止される。
  • 統計対象の列数は STATS_MAX_DIMENSIONS 以下であること。
  • 対象リレーションは RELKIND_RELATION、MATVIEW、FOREIGN_TABLE、PARTITIONED_TABLE のいずれかであること。

アルゴリズムまたは振る舞い

1. **検証**: stmt->relations を走査し、単一のリレーションであることを確認。ShareUpdateExclusiveLock を取得し、リレーションの種類と所有権を検証する。 2. **名前解決**: stmt->defnames があればスキーマを特定し、なければリレーションと同じスキーマに自動生成名で統計オブジェクトを作成する。 3. **権限チェック**: check_rights が真の場合、対象スキーマへの CREATE 権限を確認する。 4. **重複確認**: STATEXTNAMENSP システムキャッシュを用いて、同名の統計オブジェクトが既に存在しないか確認する。 5. **列数制限**: stmt->exprs の長さが STATS_MAX_DIMENSIONS を超えていないか検証する。 6. **属性変換**: 式リストを属性番号（attnums）の配列に変換し、システム属性の除外やデータ型の妥当性検証を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c:9693
  • src/backend/tcop/utility.c:1904
  • **呼び出し先**:
  • relation_openrv, object_ownercheck, QualifiedNameGetCreationNamespace, ChooseExtendedStatisticName, SearchSysCacheExists2, ereport 等（ソースコードより推測）。

関連データ構造

  • CreateStatsStmt: SQL解析結果を保持するノード。
  • pg_statistic_ext: 統計情報が格納されるシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • list_length(stmt->relations) != 1: 単一リレーション以外はエラー。
  • object_ownercheck 失敗: 所有者でない場合はエラー。
  • SearchSysCacheExists2 で発見: 同名オブジェクトが存在する場合はエラー（IF NOT EXISTS 指定時を除く）。
  • numcols > STATS_MAX_DIMENSIONS: 列数が上限を超えるとエラー。

保守担当者向けメモ

  • 本関数はDDLの定義のみを扱い、実際の統計収集は ANALYZE 実行時に行われる。
  • 依存関係の記録には recordDependencyOn が使用されるため、テーブル削除時に統計情報も自動的にクリーンアップされる設計となっている。
  • 同時実行環境での名前衝突を避けるため、自動生成ロジックには注意が必要。
