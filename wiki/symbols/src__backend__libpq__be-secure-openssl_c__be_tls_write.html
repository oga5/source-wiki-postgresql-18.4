be_tls_write

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/libpq/be-secure-openssl.c:823

目的

OpenSSL を使用して、PostgreSQL のバックエンド接続（Port）を介したセキュアなデータ書き込みを実行します。

シグネチャまたは形状

ssize_t be_tls_write(Port *port, const void *ptr, size_t len, int *waitfor)

入力と出力

  • **入力**:
  • port: 通信状態を保持する Port 構造体へのポインタ。
  • ptr: 送信するデータバッファへのポインタ。
  • len: 送信するデータのバイト数。
  • waitfor: 非ブロッキング I/O 発生時に、呼び出し元が待機すべきイベント（WL_SOCKET_READABLE または WL_SOCKET_WRITEABLE）を格納する出力用ポインタ。
  • **出力**:
  • 成功時は書き込んだバイト数（ssize_t）、失敗時は -1 を返します。

前提条件と不変条件

  • port->ssl が有効な SSL セッションオブジェクトを保持している必要があります。
  • errno は関数呼び出しの直前にクリアされます。
  • OpenSSL のエラーキューは ERR_clear_error() により事前にクリアされます。

アルゴリズムまたは振る舞い

1. SSL_write を呼び出し、データを送信します。 2. SSL_get_error を用いて結果を判定します。 3. **成功時**: SSL_ERROR_NONE の場合、書き込みバイト数を返します。 4. **非ブロッキング I/O**: SSL_ERROR_WANT_READ または SSL_ERROR_WANT_WRITE の場合、waitfor に対応するイベントを設定し、errno を EWOULDBLOCK に設定して -1 を返します。 5. **エラー時**:

  • SSL_ERROR_SYSCALL: 読み込み EOF 等の状況を判定し、必要に応じて ECONNRESET を設定します。
  • SSL_ERROR_SSL: SSLerrmessage を使用してエラーをログ出力し、ECONNRESET を設定します。
  • SSL_ERROR_ZERO_RETURN: 接続切断として ECONNRESET を設定します。
  • その他: 未知のエラーとしてログ出力し、ECONNRESET を設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/be-secure.c 内の関数から呼び出されます。
  • **呼び出し先**: OpenSSL ライブラリの SSL_write、SSL_get_error、ERR_clear_error、ERR_get_error、および PostgreSQL の ereport マクロ。

関連データ構造

  • Port: src/include/libpq/libpq-be.h で定義される、接続状態を管理する構造体。

失敗モードと境界条件

  • SSL_write が失敗した場合、OpenSSL のエラーコードに基づき errno が ECONNRESET または EWOULDBLOCK に設定されます。
  • 予期しないエラーコードが発生した場合は COMMERROR レベルでログ出力されます。

保守担当者向けメモ

  • 本関数は SSL_write が内部で読み込みを行う可能性があることを考慮しています（SSL_ERROR_SYSCALL の処理を参照）。
  • waitfor を介した非ブロッキング制御は、PostgreSQL のイベントループと統合するために不可欠です。
