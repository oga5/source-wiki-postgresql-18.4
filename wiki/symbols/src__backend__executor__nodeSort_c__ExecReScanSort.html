ExecReScanSort

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSort.c:361

目的

SortState ノードのソート状態をリセットまたは再スキャンし、必要に応じてソート処理を再実行可能にする。

シグネチャまたは形状

void ExecReScanSort(SortState *node)

入力と出力

  • **入力**: SortState *node （ソートノードの状態構造体）
  • **出力**: なし（副作用として node 内のソート状態を更新）

前提条件と不変条件

  • node は初期化済みであること。

アルゴリズムまたは振る舞い

2. **タプルクリア**: 現在保持している結果タプルスロットを ExecClearTuple で解放する。 3. **再ソート判定**: 以下のいずれかの条件を満たす場合、ソート状態を破棄し再実行準備を行う。

  • 下位プランのパラメータが変更されている (outerPlan->chgParam != NULL)
  • バウンドソートのパラメータ（bounded または bound）が変更されている
  • randomAccess が選択されていない

    • 4. **再ソート実行**: 上記条件を満たす場合、tuplesort_end で既存のソート状態を破棄し、node->sort_Done を false に設定する。下位プランの chgParam が NULL であれば、ExecReScan を呼び出して下位プランを再スキャンする。 5. **単純再スキャン**: 上記条件を満たさない場合、tuplesort_rescan を呼び出し、既存のソート済みデータを巻き戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:262（直接的な呼び出しを確認）
  • **呼び出し先**:
  • ExecClearTuple
  • tuplesort_end
  • tuplesort_rescan
  • ExecReScan（条件付き）

関連データ構造

  • SortState: ソートノードの実行状態を保持する構造体。
  • Tuplesortstate: tuplesort.c が管理するソート処理の内部状態。
  • PlanState: 下位プランの状態。

失敗モードと境界条件

  • 下位プランの chgParam が NULL の場合、ExecReScan が再帰的に呼び出される。
  • randomAccess が設定されていない場合、パラメータ変更の有無に関わらず強制的に再ソートがトリガーされる。

保守担当者向けメモ

  • 本関数は tuplesort のライフサイクル管理と、実行エンジン側の再スキャン要求を同期させる重要な役割を持つ。
  • tuplesort_end を呼び出す際は、必ず node->tuplesortstate を NULL に初期化して二重解放を防ぐこと。
