pgstat_drop_function

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_function.c:59

目的

削除された関数の統計情報を統計サブシステムから除去し、整合性を維持する。

シグネチャまたは形状

void pgstat_drop_function(Oid proid)

入力と出力

  • **入力**: proid (Oid) - 削除対象となる関数のオブジェクト識別子。
  • **出力**: なし。

前提条件と不変条件

  • 呼び出し元は、対象の関数がシステムから削除される処理の過程で本関数を呼び出す必要がある。
  • MyDatabaseId が有効なデータベースコンテキストを指していること。

アルゴリズムまたは振る舞い

1. 統計サブシステムの汎用関数 pgstat_drop_transactional を呼び出す。 2. PGSTAT_KIND_FUNCTION を種別として指定し、現在のデータベースID (MyDatabaseId) と対象の関数ID (proid) を渡すことで、該当する関数統計の削除をトランザクション管理下で実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/functioncmds.c:1334 (関数削除処理の過程で呼び出される)。
  • **呼び出し先**: pgstat_drop_transactional (統計サブシステムの内部関数)。

関連データ構造

  • PGSTAT_KIND_FUNCTION: 関数統計であることを示す識別子。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • 統計サブシステムが初期化されていない場合や、共有メモリへのアクセスに失敗した場合の挙動は pgstat_drop_transactional の実装に依存する。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な削除ロジックは pgstat_drop_transactional に委譲されている。統計情報の削除タイミングや永続化の詳細は、統計サブシステム全体のトランザクション処理ルールに従う。
