HStore

定義

  • 種別: struct
  • 定義箇所: contrib/hstore/hstore.h:44

目的

PostgreSQL の hstore 型におけるメインのメモリレイアウトを定義し、キーと値のペアを格納するための基盤構造を提供します。

シグネチャまたは形状

typedef struct {
    int32    vl_len_;    /* varlena header */
    uint32   size_;      /* フラグおよびアイテム数 */
    /* HEntry 配列が続く */
} HStore;

入力と出力

  • **入力**: 可変長データ（varlena）としてメモリ上に配置されます。
  • **出力**: ARRPTR マクロにより後続の HEntry 配列の先頭ポインタが、STRPTR マクロによりデータ領域の先頭ポインタが取得可能です。

前提条件と不変条件

  • HStore 構造体の直後に HEntry 配列が配置されるメモリレイアウトが必須です。
  • vl_len_ は varlena ヘッダーとして扱われるため、直接操作してはなりません。
  • HEntry 配列の要素数は size_ フィールドから算出されます。

アルゴリズムまたは振る舞い

1. **構築**: HS_ADDITEM 等を用いてバッファへデータを書き込み、メタデータを HEntry に設定します。 2. **確定**: HS_FINALIZE を経て SET_VARSIZE で全体のサイズを確定させます。 3. **アクセス**: HEntry の情報（オフセットと長さ）を基に、HSTORE_KEY や HSTORE_VAL マクロを使用してデータ領域へアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元**: hstoreUpgrade (変換処理), hstorePairs (構築処理), hstoreFindKey (検索処理), hstoreValidNewFormat/hstoreValidOldFormat (検証処理) 等がこの構造体を引数として受け取ります。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • HEntry: 各キーと値のメタデータ（位置とフラグ）を保持する 32bit 構造体。
  • Pairs: hstore 構築時に使用されるキーと値のペア構造体。

失敗モードと境界条件

  • hstore のメモリレイアウトが不正な場合（HEntry のオフセットが範囲外を指す等）、アクセス時にメモリ破壊やクラッシュを引き起こす可能性があります。
  • 旧形式の hstore を扱う際は、hstoreUpgrade による変換が必要です。

保守担当者向けメモ

  • HEntry はキーと値に対して交互に配置されるため、配列操作時は HS_COUNT を用いたインデックス計算に注意してください。
  • マクロ（ARRPTR, STRPTR 等）は引数を複数回評価する可能性があるため、副作用のある式を渡さないでください。
