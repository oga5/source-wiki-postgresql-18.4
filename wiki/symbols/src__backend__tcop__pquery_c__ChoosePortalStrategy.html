ChoosePortalStrategy

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:209

目的

与えられたクエリリスト（List *stmts）に基づき、ポータル実行における最適な実行戦略（PortalStrategy）を判定する。

シグネチャまたは形状

PortalStrategy ChoosePortalStrategy(List *stmts)

入力と出力

  • **入力**: List *stmts - 実行対象の Query または PlannedStmt のリスト。
  • **出力**: PortalStrategy 型の列挙値。
  • PORTAL_ONE_SELECT: 単一の SELECT 文。
  • PORTAL_ONE_MOD_WITH: WITH 句でデータ変更を行う単一の SELECT 文。
  • PORTAL_ONE_RETURNING: RETURNING 句を持つ単一のデータ変更文。
  • PORTAL_UTIL_SELECT: 結果を返すユーティリティコマンド。
  • PORTAL_MULTI_QUERY: 上記に該当しない複雑なクエリや複数文。

前提条件と不変条件

  • 入力リスト内の各要素は Query または PlannedStmt である必要がある。
  • canSetTag フラグが真であるクエリが、戦略決定の主要な基準となる。

アルゴリズムまたは振る舞い

1. **単一文の判定**: リストの長さが 1 の場合、その文が SELECT か UTILITY かを判定し、PORTAL_ONE_SELECT、PORTAL_ONE_MOD_WITH、PORTAL_UTIL_SELECT のいずれかを返す。 2. **RETURNING 句の判定**: リスト内に canSetTag なクエリがちょうど 1 つ存在し、かつそれが RETURNING 句を持つ場合、PORTAL_ONE_RETURNING を返す。 3. **デフォルト**: 上記のいずれにも該当しない場合、PORTAL_MULTI_QUERY を返す。 4. **エラー処理**: Query または PlannedStmt 以外のノード型が渡された場合、elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/pquery.c (PortalStart 内)
  • src/backend/utils/cache/plancache.c
  • **呼び出し先**:
  • UtilityReturnsTuples (ユーティリティコマンドの判定に使用)
  • IsA / nodeTag (ノード型チェック)

関連データ構造

  • PortalStrategy (列挙型)
  • Query
  • PlannedStmt
  • List (PostgreSQL のリスト構造)

失敗モードと境界条件

  • **未認識のノード**: Query や PlannedStmt 以外のノードがリストに含まれるとエラー終了する。
  • **複数文**: canSetTag なクエリが複数含まれる場合、即座に PORTAL_MULTI_QUERY が選択される。

保守担当者向けメモ

  • 本関数はクエリの書き換えルール（Rewrite Rules）によって補助クエリが追加される可能性を考慮している。
  • PORTAL_ONE_RETURNING の判定ロジックは、canSetTag なクエリがリスト内に唯一存在することに依存している。
