ginAllocEntryAccumulator

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbulk.c:84

目的

GINインデックス構築時に使用する GinEntryAccumulator ノードを、メモリ効率を考慮したチャンク単位で割り当てる。

シグネチャまたは形状

static RBTNode *ginAllocEntryAccumulator(void *arg)

入力と出力

  • **入力**: arg (構築状態を保持する BuildAccumulator 構造体へのポインタ)
  • **出力**: 新たに割り当てられた RBTNode へのポインタ

前提条件と不変条件

  • arg は BuildAccumulator 型としてキャスト可能であること。
  • BuildAccumulator 内の entryallocator が初期化されているか、または eas_used が DEF_NENTRY に達している場合に新しいチャンクが割り当てられる。

アルゴリズムまたは振る舞い

1. 現在のチャンク（accum->entryallocator）が未割り当て、または使用済み数（eas_used）が DEF_NENTRY に達している場合、palloc を用いて DEF_NENTRY 個分の GinEntryAccumulator を含む大きなメモリブロックを確保する。 2. 確保したメモリサイズを accum->allocatedMemory に加算し、使用済みカウンタをリセットする。 3. 現在のチャンクから次の GinEntryAccumulator を取得し、使用済みカウンタをインクリメントする。 4. 取得したノードを RBTNode 型にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginbulk.c 内の赤黒木（RBT）構築処理（rbt_insert 等のコールバックとして使用される）。
  • **呼び出し先**: palloc（PostgreSQLメモリ管理関数）、GetMemoryChunkSpace。

関連データ構造

  • BuildAccumulator: 構築状態を管理する構造体。
  • GinEntryAccumulator: 赤黒木のノードとして機能するデータ構造。
  • RBTNode: src/include/lib/rbt.h で定義される赤黒木の汎用ノード型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、処理が中断される。
  • DEF_NENTRY 単位での割り当てにより、個別のノード解放は行われない（一括解放を前提としている）。

保守担当者向けメモ

  • 本関数は個別のノードを palloc するオーバーヘッドを避けるための最適化である。
  • BuildAccumulator のライフサイクル全体でメモリが管理されるため、個別のノードに対する pfree は不要である。
