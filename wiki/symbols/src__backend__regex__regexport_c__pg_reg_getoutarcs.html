pg_reg_getoutarcs

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:154

目的

コンパイル済み正規表現の特定の状態から遷移可能なアーク（遷移先情報）を、LACON（先読み・後読み制約）を透過的に処理した上で取得する。

シグネチャまたは形状

void pg_reg_getoutarcs(const regex_t *regex, int st, regex_arc_t *arcs, int arcs_len)

入力と出力

  • regex: 参照するコンパイル済み正規表現オブジェクト。
  • st: 遷移元となる状態のインデックス。
  • arcs: 取得したアーク情報を格納する呼び出し元提供のバッファ。
  • arcs_len: arcs バッファの要素数。

前提条件と不変条件

  • regex は NULL であってはならず、かつ REMAGIC で初期化されている必要がある。
  • 内部的に struct guts 構造体へキャストして search（cnfa）へアクセスする。

アルゴリズムまたは振る舞い

1. 指定された st が有効な状態範囲内（0 以上 cnfa->nstates 未満）かつ arcs_len > 0 であることを確認する。 2. arcs_count を 0 に初期化する。 3. traverse_lacons を呼び出し、LACON を再帰的に探索して実質的な遷移先を解決し、結果を arcs バッファに書き込む。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_regexp.c 内の複数箇所から呼び出されている。
  • **呼び出し先**: traverse_lacons を呼び出す。

関連データ構造

  • regex_t: 正規表現の公開構造体。
  • cnfa: 正規表現の内部的な非決定性有限オートマトン構造。
  • regex_arc_t: 遷移先情報を保持する構造体。

失敗モードと境界条件

  • st が範囲外の場合、または arcs_len が 0 以下の場合は何も行わず即座に復帰する。
  • assert により、regex が NULL または不正なマジックナンバーを持つ場合は異常終了する。

保守担当者向けメモ

  • 本関数は LACON をスキップし、実質的な遷移のみを公開する設計である。
  • traverse_lacons は再帰関数であるため、極端に深い LACON 構造を持つ正規表現に対してはスタック深さの制限に注意が必要である。
