ReindexIndexInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/indexcmds.c:3572

目的

REINDEX コマンド実行時、再構築対象となる個々のインデックスに関するメタデータおよび状態を保持するために使用されます。

シグネチャまたは形状

typedef struct ReindexIndexInfo
{
    Oid         indexId;
    Oid         tableId;
    Oid         amId;
    bool        safe;       /* set_indexsafe_procflags用 */
} ReindexIndexInfo;

入力と出力

本構造体はデータ保持用であり、関数ではありません。palloc_object(ReindexIndexInfo) を通じてメモリ確保が行われます。

前提条件と不変条件

  • indexId, tableId, amId は有効なシステムカタログの OID である必要があります。
  • safe フラグは、インデックスが式や述語を含まない場合に true に設定され、並行構築プロセスにおける安全性の判断に利用されます。

アルゴリズムまたは振る舞い

REINDEX 処理の過程で、対象インデックスの情報をリスト化し、後続の構築フェーズで参照するために利用されます。特に並行再構築（REINDEX CONCURRENTLY）において、各インデックスの属性を保持し、状態遷移を管理する役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/indexcmds.c 内の REINDEX 関連処理（ReindexRelationConcurrently 等）において、対象インデックスのリスト構築時にインスタンス化されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ReindexIndexCallbackState: 再構築処理のコールバック状態管理に使用されます。
  • ReindexErrorInfo: 再構築時のエラー情報保持に使用されます。

失敗モードと境界条件

  • メモリ不足時には palloc_object が失敗する可能性があります。
  • 保持する OID が無効な場合、後続のカタログアクセス処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は REINDEX の並行処理において重要な役割を果たすため、フィールドの追加や変更を行う際は、並行構築の各フェーズ（カタログ登録、構築、検証、スワップ）への影響を考慮してください。
  • safe フラグの判定ロジックについては、set_indexsafe_procflags の仕様と整合性を保つ必要があります。
