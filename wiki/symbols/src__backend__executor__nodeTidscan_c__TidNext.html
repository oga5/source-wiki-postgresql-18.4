TidNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidscan.c:311

目的

TidScan ノードにおいて、指定された TID（Tuple Identifier）リストに基づき、リレーションからタプルを順次フェッチする。

シグネチャまたは形状

TupleTableSlot *TidNext(TidScanState *node)

入力と出力

  • **入力**: TidScanState *node（スキャン状態を保持するノード構造体）
  • **出力**: フェッチに成功した場合は該当タプルを格納した TupleTableSlot、スキャン終了時は ExecClearTuple によりクリアされたスロット。

前提条件と不変条件

  • 初回呼び出し時に node->tss_TidList が NULL の場合、TidListEval が呼び出され、TID リストが生成・ソートされる。
  • スキャン方向（順方向/逆方向）に応じて node->tss_TidPtr が適切に管理される。

アルゴリズムまたは振る舞い

1. **初期化**: 初回実行時に TidListEval を呼び出し、対象 TID の配列を構築する。 2. **ポインタ更新**: スキャン方向に基づき、現在のインデックス tss_TidPtr を進める（または戻す）。 3. **フェッチループ**:

  • tss_TidPtr が有効範囲内である限りループする。
  • tss_isCurrentOf が真の場合、table_tuple_get_latest_tid を呼び出し、最新の TID を取得する。
  • table_tuple_fetch_row_version を呼び出し、タプルを取得する。成功すればそのスロットを返す。
  • 失敗した場合はインデックスを更新し、CHECK_FOR_INTERRUPTS() を実行して次の TID へ進む。

    • 4. **終了**: 全ての TID を走査してもタプルが見つからない場合、ExecClearTuple を実行して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は明示されていないが、ExecScanAccessMtd 経由で実行される設計となっている。
  • **呼び出し先**: TidListEval, table_tuple_get_latest_tid, table_tuple_fetch_row_version, ExecClearTuple, CHECK_FOR_INTERRUPTS。

関連データ構造

  • TidScanState: スキャン実行時の状態を保持する構造体。
  • ItemPointerData: TID を格納するデータ型。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • **TID 無効**: table_tuple_fetch_row_version が偽を返す場合、その TID はスキップされる。
  • **スキャン終了**: tss_TidPtr がリストの範囲外となった場合、スキャンは終了する。
  • **割り込み**: ループ内で CHECK_FOR_INTERRUPTS() が呼び出され、クエリキャンセル等の割り込みに対応する。

保守担当者向けメモ

  • WHERE CURRENT OF 句の処理が含まれており、カーソルに関連付けられた TID の動的な解決を行うため、通常の ctid 比較とは異なる挙動を示す。
  • TID リストのソートは I/O 効率に直結するため、リスト生成ロジックを変更する際は注意が必要である。
