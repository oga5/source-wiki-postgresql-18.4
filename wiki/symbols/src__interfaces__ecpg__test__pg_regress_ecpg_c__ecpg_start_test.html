ecpg_start_test

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/pg_regress_ecpg.c:147

目的

ECPG回帰テストにおいて、テスト対象プログラムの実行環境を準備し、プロセスを起動する。

シグネチャまたは形状

static PID_TYPE ecpg_start_test(const char *testname, _stringlist **resultfiles, _stringlist **expectfiles, _stringlist **tags)

入力と出力

  • **入力**:
  • testname: テストケース名。
  • resultfiles, expectfiles, tags: テスト結果の比較に使用するファイルリストおよびタグを格納するポインタ。
  • **出力**:
  • 戻り値: 起動したプロセスのプロセスID (PID_TYPE)。
  • 副作用: resultfiles 等のリストへの項目追加、ソースコードの正規化処理、およびテスト対象プロセスの起動。

前提条件と不変条件

  • inputdir, expecteddir, outputdir がグローバル変数として定義されていること。
  • testname に基づく入力ファイル（.c）が存在すること。

アルゴリズムまたは振る舞い

1. **パス生成**: testname 内の / を - に置換し、入出力ファイル（stdout, stderr, source）のパスを構築する。 2. **リスト登録**: add_stringlist_item を使用し、結果ファイル、期待値ファイル、およびタグを管理リストに登録する。 3. **ソース正規化**: ecpg_filter_source を呼び出し、ソースコード内の #line ディレクティブを正規化して出力ファイルに書き出す。 4. **環境設定**: PGAPPNAME 環境変数をテスト名に基づいて設定する。 5. **プロセス起動**: spawn_process を用いてテストプログラムを実行し、標準出力と標準エラー出力を指定のファイルへリダイレクトする。 6. **後処理**: PGAPPNAME を解除し、動的に確保したメモリを解放してプロセスIDを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/pg_regress_ecpg.c 内の regression_main（推測されるフローに基づく）。
  • **呼び出し先**: initStringInfo, appendStringInfoString, add_stringlist_item, ecpg_filter_source, psprintf, setenv, spawn_process, unsetenv, free。

関連データ構造

  • StringInfoData: テスト名変換用バッファ。
  • _stringlist: テスト結果管理用のリスト構造。

失敗モードと境界条件

  • spawn_process が INVALID_PID を返した場合、エラーメッセージを出力して exit(2) を実行し、プログラムを終了する。

保守担当者向けメモ

  • 本関数は pg_regress のコールバックとして機能する。
  • ecpg_filter_source による正規化ルールが変更された場合、本関数で生成される outfile_source の内容に影響を与えるため注意すること。
  • PGAPPNAME の設定はプロセス起動時のみ有効であり、終了後に unsetenv で確実にクリアされる設計となっている。
