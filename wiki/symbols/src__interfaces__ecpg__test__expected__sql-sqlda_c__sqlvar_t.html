sqlvar_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-sqlda.c:33
  • 宣言: typedef struct sqlvar_compat sqlvar_t;

目的

ECPG（Embedded SQL for C）において、SQLDA（SQL Descriptor Area）内の個々の列（カラム）に関するメタデータおよびデータ値を保持するために使用されます。

シグネチャまたは形状

sqlvar_compat 構造体の別名です。具体的なメンバ構成は、コンパイル時のモード（Informix 互換モードまたはネイティブモード）に応じて sqlda-compat.h または sqlda-native.h で定義されます。

入力と出力

  • **入力**: 動的 SQL の実行時、バインド変数として sqlvar_t 構造体をメモリ確保し、値をセットして使用します。
  • **出力**: FETCH 等の結果取得時、データベースから返されたデータ型、値、インジケータ値などが本構造体に格納されます。

前提条件と不変条件

  • sqlda_t 構造体の一部として管理されます。
  • 動的 SQL のバインドに使用する場合、malloc によるメモリ確保と memset によるゼロ初期化が推奨されます。

アルゴリズムまたは振る舞い

  • dump_sqlda 関数において、sqlvar_t 内の sqltype を参照し、型に応じたキャストを行うことで、格納されたデータを標準出力へ書き出す処理が行われます。
  • 動的 SQL の実行フローにおいて、sqlda->sqlvar を走査することで、結果セットの各列情報にアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/compat_informix-sqlda.c および src/interfaces/ecpg/test/expected/compat_oracle-char_array.c 等のテストコード内で、動的 SQL のバインドや結果の参照に使用されています。
  • **呼び出し先**: 直接的な関数呼び出し関係はありませんが、malloc や memset といった標準ライブラリ関数によってメモリ管理が行われます。

関連データ構造

  • sqlda_t: sqlvar_t を配列として保持する上位の記述子構造体。
  • sqlca_t: ECPG のエラー情報を保持する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗: malloc が失敗した場合、sqlvar_t へのアクセスはセグメンテーション違反を引き起こします。
  • 型不整合: sqltype が期待する型と異なる場合、dump_sqlda 等でのデータ解釈が不正な値を出力する可能性があります。

保守担当者向けメモ

  • 本構造体は _ECPG_INFORMIX_H マクロの定義有無により構造が切り替わるため、環境依存の挙動に注意してください。
  • プリプロセッサ生成コードであるため、手動修正時は ecpg プリプロセッサの仕様を優先してください。
