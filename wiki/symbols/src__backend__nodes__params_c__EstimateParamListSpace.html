EstimateParamListSpace

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/params.c:166

目的

ParamListInfo 構造体をシリアライズするために必要なメモリ領域のサイズ（バイト単位）を見積もる。

シグネチャまたは形状

Size EstimateParamListSpace(ParamListInfo paramLI)

入力と出力

  • **入力**: paramLI (シリアライズ対象の ParamListInfo 構造体へのポインタ)
  • **出力**: Size 型のバイト数。paramLI が NULL または numParams <= 0 の場合は sizeof(int) を返す。

前提条件と不変条件

  • paramLI が動的フック (paramFetch) を持つ場合、この関数はそのフックを呼び出してパラメータ値を取得する。
  • 型情報 (typeOid) が不明な場合は、Datum サイズかつ値渡し（by-value）として見積もる。

アルゴリズムまたは振る舞い

1. 初期サイズとして sizeof(int)（パラメータ数を格納する領域）を確保する。 2. パラメータリスト内の各要素について以下を繰り返す：

  • paramFetch フックがあれば呼び出し、なければ配列から直接 ParamExternData を取得する。
  • 型情報に基づき、Oid、uint16（フラグ）、および datumEstimateSpace で計算されるデータ本体のサイズを累積する。

    • 3. add_size を使用してオーバーフローを防止しつつ合計サイズを算出する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c (並列クエリ実行時のパラメータ転送準備)
  • **呼び出し先**: paramFetch (フックが設定されている場合)、get_typlenbyval、datumEstimateSpace

関連データ構造

  • ParamListInfo: パラメータの配列または取得フックを保持する構造体。
  • ParamExternData: 個々のパラメータの型、値、NULL状態を保持する構造体。

失敗モードと境界条件

  • paramLI が NULL の場合、安全に sizeof(int) を返す。
  • typeOid が無効な場合、typLen = sizeof(Datum) かつ typByVal = true として見積もる。

保守担当者向けメモ

  • 本関数は SerializeParamList と対になる見積もりロジックである。シリアライズ対象の構造に変更を加えた場合は、本関数の計算ロジックも同期させる必要がある。
  • paramValuesStr はシリアライズ対象外であるため、本見積もりには含まれない。
