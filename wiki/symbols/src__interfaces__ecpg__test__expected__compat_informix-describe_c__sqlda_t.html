sqlda_t

定義

  • 種別: typedef struct
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-describe.c:34
  • 実体: struct sqlda_compat の別名

目的

Informix 互換モードにおける SQL 記述子領域（SQLDA）を表現し、動的 SQL 文の実行結果やメタデータを保持するために使用されます。

シグネチャまたは形状

typedef struct sqlda_compat sqlda_t; （詳細は sqlda-compat.h に依存）

入力と出力

  • 入力: ECPGdescribe() 等の ECPG ライブラリ関数により、準備済みステートメントのメタデータが格納されます。
  • 出力: 保持されたメタデータ（列数、列名、型情報など）は、アプリケーション側で参照され、結果セットの処理や検証に使用されます。

前提条件と不変条件

  • Informix 互換モードでコンパイルされている必要があります。
  • 構造体を使用する前に、適切なメモリ割り当て（malloc 等）が必要です。

アルゴリズムまたは振る舞い

1. ECPGdescribe() を呼び出し、準備済みステートメントの情報を本構造体に格納します。 2. 構造体内のフィールド（sqld 等）を通じて、取得した列数や各列のメタデータにアクセスします。 3. sqlvar_t 型の配列を内部的に保持し、各列の詳細情報を管理します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/compat_informix-describe.c および compat_informix-sqlda.c 内のテストロジック。
  • 呼び出し先: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いませんが、ECPG ライブラリ関数（ECPGdescribe 等）によって操作されます。

関連データ構造

  • sqlvar_t: sqlda_t 内で各列の属性（型、名前、データポインタ等）を保持する構造体。
  • sqlda-compat.h: sqlda_compat 構造体の定義元。

失敗モードと境界条件

  • メモリ不足: malloc による動的確保に失敗した場合、構造体へのアクセスは未定義動作となります。
  • 記述子の不整合: DESCRIBE 文で取得した列数と、実際に確保した sqlda_t の容量が一致しない場合、バッファオーバーフローやデータ破損のリスクがあります。

保守担当者向けメモ

  • 本シンボルは Informix 互換モード専用です。非互換モードのテストコード（sql-describe.c 等）とは異なる定義が使用される点に注意してください。
  • 構造体のレイアウト変更は、ECPG ライブラリの Informix 互換機能全体に影響を及ぼします。
