nodeRead

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/read.c:319

目的

シリアライズされた文字列形式のデータから、PostgreSQLの内部ノード構造（Nodeツリー）を再帰的に復元する。

シグネチャまたは形状

void *nodeRead(const char *token, int tok_len)

入力と出力

  • **入力**:
  • token: 現在解析中のトークン文字列（NULL の場合は pg_strtok を用いて自動取得）。
  • tok_len: トークンの長さ。
  • **出力**: 復元された Node 構造体へのポインタ。入力が終了している場合は NULL を返す。

前提条件と不変条件

  • pg_strtok_ptr を介して解析状態が保持されるため、呼び出し元は一連の解析において整合性を保つ必要がある。
  • { で開始されたノードは、必ず対応する } で終了しなければならない。

アルゴリズムまたは振る舞い

1. token が NULL の場合、pg_strtok を呼び出して次のトークンを取得する。 2. nodeTokenType を使用してトークンの型を判定する。 3. トークンが { の場合、parseNodeString を呼び出してノードのプロパティを読み込み、直後に } が存在することを確認する。 4. その他の型（リスト、整数、文字列等）については、対応する解析処理へ分岐する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/read.c（自身を再帰的に呼び出し）
  • src/backend/nodes/readfuncs.c（各ノード型の読み込み処理から参照）
  • **呼び出し先**:
  • pg_strtok: トークンの切り出し。
  • nodeTokenType: トークンの型判定。
  • parseNodeString: { で始まるノードの解析。
  • elog: エラーハンドリング（} 不一致時）。

関連データ構造

  • Node: 復元対象となる基本構造体。
  • NodeTag: ノードの種類を識別する列挙型。

失敗モードと境界条件

  • 入力文字列の末尾で期待される } が見つからない場合、elog(ERROR) により処理を中断する。
  • token が NULL かつ入力ストリームが空の場合、NULL を返す。

保守担当者向けメモ

  • 本関数は再帰的に呼び出されるため、スタックオーバーフローのリスクがある深いノードツリーには注意が必要。
  • pg_strtok が静的変数 pg_strtok_ptr を使用しているため、マルチスレッド環境での同時実行は不可。
  • エスケープ文字の処理については、必要に応じて debackslash を併用すること。
