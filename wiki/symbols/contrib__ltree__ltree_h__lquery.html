lquery

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltree.h:113

目的

階層構造を持つラベルパス（ltree）に対する検索パターンを表現するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    int32       vl_len_;        /* varlena header */
    uint16      numlevel;       /* lquery_level の数 */
    uint16      firstgood;      /* 先頭の単純一致レベル数 */
    uint16      flag;           /* LQUERY_xxx フラグ */
    char        data[FLEXIBLE_ARRAY_MEMBER]; /* lquery_level 配列 */
} lquery;

入力と出力

  • **入力**: SQL クエリ等から渡される検索パターン文字列が、ltree_in 等の処理を経て本構造体に変換されます。
  • **出力**: ltree との照合結果（マッチングの成否）を判定するためのデータとして利用されます。

前提条件と不変条件

  • 可変長データ（varlena）であるため、サイズ計算には MAXALIGN を含む専用マクロ（LTREE_HDRSIZE 等）を使用する必要があります。
  • data フィールドは lquery_level 構造体の配列として配置されます。
  • 過去のディスクフォーマットとの互換性を維持するため、フィールドの配置やサイズには制約があります。

アルゴリズムまたは振る舞い

  • lquery_level の各レベルを走査し、lquery_variant のフラグ（LVAR_ANYEND 等）に基づいて ltree パスとの照合を行います。
  • lquery_op.c にて、lquery を ltree と比較するマッチングロジックが実装されています。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/ltree/_ltree_gist.c: GiST インデックス検索 (gist_qe) で使用。
  • contrib/ltree/_ltree_op.c: SQL 演算子実装 (PG_GETARG_LQUERY_P) で使用。
  • contrib/ltree/lquery_op.c: マッチングロジックの実装で使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • lquery_level: lquery の構成要素。
  • lquery_variant: 各レベルの検索条件を定義。
  • ltree: 検索対象となるラベルパス構造体。

失敗モードと境界条件

  • sizeof を直接使用するとアライメント違反が発生するため、必ず NEXTVAL マクロ等を用いて次の要素へアクセスする必要があります。
  • 構造体内の numlevel や flag の値が不正な場合、マッチング処理中にメモリ境界外アクセスが発生する可能性があります。

保守担当者向けメモ

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体の拡張や修正を行う際はメモリレイアウトに十分注意してください。
  • インデックス操作時は LTREE_GET_SIGLEN() マクロを介してシグネチャ長を取得する設計となっています。
