FindStreamingStart

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_receivewal.c:267

目的

ローカルの WAL 保存ディレクトリを走査し、既存の WAL セグメントファイルに基づいて、サーバーから WAL ストリーミングを再開すべき適切な位置（LSN）を決定する。

シグネチャまたは形状

static XLogRecPtr FindStreamingStart(uint32 *tli);

入力と出力

  • **入力**: uint32 *tli （タイムラインIDを格納するためのポインタ）
  • **出力**: ストリーミングを開始すべき位置を示す XLogRecPtr を返す。

前提条件と不変条件

  • basedir が有効なディレクトリとして設定されていること。
  • WalSegSz（WALセグメントサイズ）が事前に取得されていること。
  • 圧縮された WAL ファイル（GZIP/LZ4）の場合、展開後のサイズが WalSegSz と一致するか、あるいは部分書き込み状態でないことが検証される。

アルゴリズムまたは振る舞い

1. get_destination_dir を使用して対象ディレクトリを開く。 2. ディレクトリ内の各ファイルを is_xlogfilename で判定し、WAL セグメントであるかを確認する。 3. XLogFromFileName でファイル名からタイムラインとセグメント番号を解析する。 4. 圧縮なし、GZIP圧縮、LZ4圧縮の各形式に応じて、ファイルサイズまたは展開後のサイズを検証する。

  • GZIP の場合は末尾4バイトの ISIZE を確認。
  • LZ4 の場合は LZ4F_decompress を用いて展開サイズを計算。

    • 5. 検証に合格した最新のセグメントを特定し、そこからストリーミング開始位置を算出する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_basebackup/pg_receivewal.c:537）にて、stream.startpos の初期化に使用される。
  • **呼び出し先**: get_destination_dir, is_xlogfilename, XLogFromFileName, stat, open, read, close, LZ4F_decompress 等の標準ライブラリおよび内部ユーティリティ。

関連データ構造

  • XLogRecPtr: WAL の位置を示すポインタ型。
  • XLogSegNo: WAL セグメント番号。
  • pg_compress_algorithm: 圧縮アルゴリズムの列挙型。

失敗モードと境界条件

  • ディレクトリ内のファイルが破損している、またはサイズが WalSegSz と一致しない場合は警告を出力してスキップする。
  • ファイルのオープンや読み取りに失敗した場合は pg_fatal によりプロセスを終了する。
  • 圧縮コンテキストの生成や展開処理でエラーが発生した場合も pg_fatal を呼び出す。

保守担当者向けメモ

  • 圧縮アルゴリズムの検証ロジックは、将来的に新しい圧縮形式が追加される際に拡張が必要となる。
  • LZ4 展開処理では LZ4_CHUNK_SZ（64kB）単位でバッファ処理を行っているため、メモリ効率と整合性に注意すること。
  • 圧縮された WAL ファイルの末尾検証ロジックは、アーカイブコマンド等で生成されたファイルとの互換性に依存している。
