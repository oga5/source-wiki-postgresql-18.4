LookupTupleHashEntryHash

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execGrouping.c:355

目的

指定されたタプルに対して、ハッシュテーブル内の対応するエントリを検索または作成します。既に計算済みのハッシュ値を利用することで、効率的なグループ化処理を支援します。

シグネチャまたは形状

TupleHashEntry LookupTupleHashEntryHash(TupleHashTable hashtable, TupleTableSlot *slot, bool *isnew, uint32 hash)

入力と出力

  • **入力**:
  • hashtable: 操作対象の TupleHashTable。
  • slot: 検索対象のタプルを保持する TupleTableSlot。
  • isnew: 新規エントリが作成された場合に true が設定される出力用ポインタ。
  • hash: 事前に計算されたタプルのハッシュ値。
  • **出力**:
  • 検索または作成された TupleHashEntry へのポインタ。

前提条件と不変条件

  • hashtable->tempcxt が有効であること。
  • 呼び出し側は、ハッシュ関数および比較関数が hashtable に適切に設定されていることを期待します。
  • 戻り値が NULL でない場合、entry->hash == hash であることが保証されます。

アルゴリズムまたは振る舞い

1. hashtable->tempcxt にメモリコンテキストを切り替え、短命なメモリ割り当てを可能にします。 2. hashtable の inputslot、in_hash_expr、cur_eq_func を引数の slot およびテーブル内の設定値で更新します。 3. LookupTupleHashEntry_internal を呼び出し、ハッシュテーブル内を検索または挿入します。 4. 元のメモリコンテキストに復帰し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c:2766
  • **呼び出し先**:
  • LookupTupleHashEntry_internal (同ファイル内)
  • MemoryContextSwitchTo (PostgreSQL メモリ管理 API)

関連データ構造

  • TupleHashTable: グループ化のためのハッシュテーブル構造体。
  • TupleHashEntry: ハッシュテーブル内の個々のエントリ。
  • TupleTableSlot: タプルデータを保持するコンテナ。

失敗モードと境界条件

  • メモリ不足時には LookupTupleHashEntry_internal 経由でエラーが発生する可能性があります。
  • hashtable が適切に初期化されていない場合、ポインタ参照エラーが発生します。

保守担当者向けメモ

  • 本関数は、ハッシュ値を外部から受け取ることで、ハッシュ計算の重複を避ける最適化パスを提供します。
  • tempcxt の切り替えは、ハッシュ比較時のメモリリークを防ぐために不可欠です。
