tuplestore_begin_heap

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplestore.c:329

目的

ヒープタプルを格納するための Tuplestorestate を初期化し、ヒープタプル専用の操作関数を割り当てる。

シグネチャまたは形状

Tuplestorestate *tuplestore_begin_heap(bool randomAccess, bool interXact, int maxKBytes)

入力と出力

  • **入力**:
  • randomAccess (bool): 逆方向スキャンや巻き戻しを許可するかどうか。
  • interXact (bool): トランザクションをまたいで保持するかどうか。
  • maxKBytes (int): メモリ内保持の最大容量（KB）。
  • **出力**:
  • Tuplestorestate *: 初期化された状態構造体へのポインタ。

前提条件と不変条件

  • tuplestore_begin_common が正常にメモリコンテキストと初期読み取りポインタを生成できること。
  • ヒープタプル操作用の関数ポインタ（copytup_heap, writetup_heap, readtup_heap）が利用可能であること。

アルゴリズムまたは振る舞い

1. randomAccess の値に基づき、実行フラグ（EXEC_FLAG_BACKWARD, EXEC_FLAG_REWIND）を決定する。 2. tuplestore_begin_common を呼び出し、共通の初期化処理を実行する。 3. 戻り値の Tuplestorestate 構造体に対し、ヒープタプル専用の関数ポインタを代入する。

  • copytup = copytup_heap
  • writetup = writetup_heap
  • readtup = readtup_heap

    • 4. 初期化済みの state を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/tablefunc/tablefunc.c
  • src/backend/commands/trigger.c
  • src/backend/executor/execSRF.c
  • **呼び出し先**:
  • tuplestore_begin_common (内部関数)

関連データ構造

  • Tuplestorestate: タプル格納の全状態を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: tuplestore_begin_common 内でメモリ割り当てに失敗した場合、エラーが発生する。
  • maxKBytes が 0 以下の場合の挙動は tuplestore_begin_common の実装に依存する。

保守担当者向けメモ

  • 本関数はヒープタプル専用の抽象化レイヤーである。将来的に他のタプル形式をサポートする場合、同様のパターンで新しい tuplestore_begin_... 関数を追加する必要がある。
  • randomAccess の解釈は PostgreSQL 8.3 以前の互換性を維持している。
