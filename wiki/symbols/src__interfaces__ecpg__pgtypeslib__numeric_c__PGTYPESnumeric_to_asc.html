PGTYPESnumeric_to_asc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/numeric.c:342

目的

numeric 構造体で保持されている数値を、人間が読み取り可能な ASCII 文字列形式に変換します。

シグネチャまたは形状

char *PGTYPESnumeric_to_asc(numeric *num, int dscale)

入力と出力

  • **入力**:
  • num: 変換対象の numeric 構造体へのポインタ。
  • dscale: 出力文字列の小数点以下の桁数。負の値を指定した場合、num->dscale が使用されます。
  • **出力**:
  • 変換後の文字列へのポインタ。失敗した場合は NULL を返します。

前提条件と不変条件

  • num は有効な numeric 構造体を指している必要があります。
  • 内部で numeric 構造体のコピーを作成して処理を行うため、元の num の内容は変更されません。
  • 戻り値の文字列は動的に確保されるため、呼び出し元で適切に解放する必要があります。

アルゴリズムまたは振る舞い

1. PGTYPESnumeric_new() を使用して、作業用の numeric 構造体 (numcopy) を確保します。 2. PGTYPESnumeric_copy() を用いて、入力 num の内容を numcopy に複製します。 3. dscale が負の場合、num->dscale を適用します。 4. get_str_from_var() を呼び出し、numcopy を指定された dscale に基づいて文字列へ変換します。この際、必要に応じて丸め処理が行われます。 5. 作業用メモリ numcopy を PGTYPESnumeric_free() で解放し、生成された文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • src/interfaces/ecpg/test/expected/ 配下の各テストファイル
  • **呼び出し先**:
  • PGTYPESnumeric_new
  • PGTYPESnumeric_copy
  • get_str_from_var
  • PGTYPESnumeric_free

関連データ構造

  • numeric: 数値の桁、小数点位置、符号を保持する構造体。

失敗モードと境界条件

  • PGTYPESnumeric_new() によるメモリ確保に失敗した場合、NULL を返します。
  • PGTYPESnumeric_copy() が失敗した場合、確保したメモリを解放した上で NULL を返します。
  • dscale に負の値を指定すると、元の数値が持つ精度設定が優先されます。

保守担当者向けメモ

  • 本関数は内部で get_str_from_var を呼び出します。この関数は出力時に丸め処理を行うため、計算精度については get_str_from_var の仕様に依存します。
  • メモリ管理のため、PGTYPESnumeric_new と PGTYPESnumeric_free のペアが正しく機能していることを確認してください。
