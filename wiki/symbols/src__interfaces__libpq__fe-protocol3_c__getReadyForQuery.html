getReadyForQuery

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:1761

目的

PostgreSQL プロトコル 3.0 における ReadyForQuery メッセージを解析し、接続状態（トランザクション状態）を更新する。

シグネチャまたは形状

static int getReadyForQuery(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（PostgreSQL 接続オブジェクト）
  • **出力**: 成功時は 0 を返し、conn->xactStatus を更新する。入力バッファから文字を読み取れない場合は EOF を返す。

前提条件と不変条件

  • conn は有効な PGconn オブジェクトであること。
  • 呼び出し時点で、メッセージタイプ（'Z'）は既に消費済みであり、入力バッファにはトランザクション状態を示す 1 バイトの文字が残っていること。

アルゴリズムまたは振る舞い

1. pqGetc を使用して、入力バッファからトランザクション状態を示す文字を読み取る。 2. 読み取った文字に基づき、conn->xactStatus を以下のいずれかに更新する。

  • 'I': PQTRANS_IDLE（トランザクション外）
  • 'T': PQTRANS_INTRANS（トランザクション内）
  • 'E': PQTRANS_INERROR（トランザクション失敗状態）
  • その他: PQTRANS_UNKNOWN

    • 3. 正常終了時は 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c 内のメッセージ解析ループ（pqParseInput3 等）から呼び出される。
  • **呼び出し先**: pqGetc（低レベルバッファ読み込み関数）。

関連データ構造

  • PGconn: 接続状態を保持する構造体。xactStatus フィールドを更新する。

失敗モードと境界条件

  • pqGetc が失敗した場合（データ不足など）、即座に EOF を返し、状態更新は行われない。

保守担当者向けメモ

  • 本関数はプロトコル 3.0 専用である。
  • xactStatus の各値は libpq-int.h で定義されている。追加のトランザクション状態がプロトコルに追加された場合は、switch 文を拡張する必要がある。
