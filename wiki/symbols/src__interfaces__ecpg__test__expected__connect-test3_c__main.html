目的

ECPG（Embedded SQL for C）におけるデータベース接続の切断機能（DISCONNECT）の動作を検証する回帰テストの実行起点。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし
  • 出力: 正常終了時に 0 を返す。

前提条件と不変条件

  • ECPGプリプロセッサによって生成されたコードであること。
  • ECPGdebug(1, stderr) が呼び出され、デバッグログが標準エラー出力に設定されていること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を呼び出し、デバッグ出力を有効化する。 2. **接続**: ECPGconnect を使用して ecpg2_regression および ecpg1_regression データベースへ接続する。 3. **検証**: ECPGdo を介して SELECT current_database() を実行し、接続状態を確認する。 4. **切断テスト**: 以下の引数で ECPGdisconnect を順次実行し、接続管理の挙動を検証する。

  • CURRENT: 直近の接続を閉じる。
  • DEFAULT: デフォルト接続を閉じる。
  • ALL: すべての接続を閉じる。

    • 5. **終了**: 接続が存在しない状態での DISCONNECT 呼び出しを行い、エラーハンドリングを確認して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: 不明（テスト実行環境から呼び出される）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGdisconnect

関連データ構造

  • id (char[200]): 接続識別子用バッファ
  • res (char[200]): SQL実行結果格納用バッファ

失敗モードと境界条件

  • 接続が存在しない状態での DISCONNECT 呼び出し（CURRENT, DEFAULT, ALL）が含まれており、これに対するライブラリ側のエラーハンドリングを検証する。
