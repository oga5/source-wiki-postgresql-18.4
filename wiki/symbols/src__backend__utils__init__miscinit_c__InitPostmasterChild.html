InitPostmasterChild

定義

  • 種別: function
  • 定義箇所: src/backend/utils/init/miscinit.c:95

目的

Postmasterから起動された子プロセスにおいて、OSレベルの実行環境およびプロセス間通信の基盤を初期化する。

シグネチャまたは形状

void InitPostmasterChild(void)

入力と出力

  • 入力: なし（グローバル変数およびOS環境に依存）
  • 出力: なし（副作用としてプロセス状態を更新）

前提条件と不変条件

  • 本関数はPostmasterの子プロセスとして実行されること。
  • IsUnderPostmaster フラグが true に設定される。
  • proc_exit() ハンドラはリセットされ、親プロセスの設定を引き継がない。

アルゴリズムまたは振る舞い

1. **環境設定**: IsUnderPostmaster を有効化し、Windows環境ではシグナル初期化および stderr のバイナリモード設定を行う。 2. **プロセス管理**: on_exit_reset() により親プロセスの終了ハンドラを破棄する。EXEC_BACKEND 定義時はシグナルマスクを再初期化する。 3. **ラッチと待機**: InitializeWaitEventSupport 等を呼び出し、プロセスローカルなラッチと待機イベントを初期化する。 4. **セッション管理**: setsid() を呼び出し、プロセスグループリーダーとなる（HAVE_SETSID 定義時）。 5. **シグナル処理**: SIGQUIT ハンドラを SignalHandlerForCrashExit に設定し、BlockSig から SIGQUIT を除外して受信可能にする。 6. **監視**: PostmasterDeathSignalInit() を呼び出し、Postmasterの異常終了を検知する準備を行う。 7. **ファイル記述子**: Postmaster監視用パイプに FD_CLOEXEC を設定し、子プロセスへの漏洩を防ぐ（Windows以外）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/launch_backend.c (Backend起動時)
  • **呼び出し先**:
  • pgwin32_signal_initialize (Windowsのみ)
  • InitProcessGlobals
  • on_exit_reset
  • pqinitmask (EXEC_BACKEND 時)
  • InitializeWaitEventSupport, InitProcessLocalLatch, InitializeLatchWaitSet
  • setsid (HAVE_SETSID 時)
  • pqsignal, sigdelset, sigprocmask
  • PostmasterDeathSignalInit
  • fcntl (Windows以外)

関連データ構造

  • BlockSig: ブロック対象のシグナルセット。
  • postmaster_alive_fds: Postmasterの生存監視用パイプのファイル記述子。

失敗モードと境界条件

  • setsid() が失敗した場合、FATAL エラーでプロセスを終了する。
  • fcntl() による FD_CLOEXEC 設定が失敗した場合、FATAL エラーでプロセスを終了する。

保守担当者向けメモ

  • 本関数はプロセス起動の極めて早い段階で実行されるため、ここでのエラーはデータベース全体の可用性に直結する。
  • WindowsとUnix系でシグナル処理やファイル操作の挙動が大きく異なるため、修正時は両プラットフォームへの影響を考慮すること。
