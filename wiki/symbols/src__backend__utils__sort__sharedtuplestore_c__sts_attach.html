sts_attach

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sharedtuplestore.c:177

目的

共有タプルストア（SharedTuplestore）に対する操作用アクセサ（SharedTuplestoreAccessor）を初期化し、呼び出し元のバックエンドがストアへアクセスできるように接続を確立する。

シグネチャまたは形状

SharedTuplestoreAccessor *sts_attach(SharedTuplestore *sts, int my_participant_number, SharedFileSet *fileset)

入力と出力

  • **入力**:
  • sts: 共有メモリ上の SharedTuplestore 構造体へのポインタ。
  • my_participant_number: 現在のバックエンドに割り当てられた参加者番号。
  • fileset: 一時ファイルを管理する SharedFileSet へのポインタ。
  • **出力**:
  • 初期化された SharedTuplestoreAccessor 構造体へのポインタを返す。

前提条件と不変条件

  • my_participant_number は、sts->nparticipants 未満である必要がある（Assert により検証）。
  • 呼び出し元のメモリコンテキストが accessor->context に保持される。

アルゴリズムまたは振る舞い

1. palloc0 を使用して SharedTuplestoreAccessor 構造体用のメモリを確保し、ゼロ初期化する。 2. 引数として受け取った sts、my_participant_number、fileset を構造体の各フィールドに格納する。 3. 現在のメモリコンテキストを accessor->context に保存する。 4. 構築した accessor 構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHash.c: ParallelHashJoinBatchInner および ParallelHashJoinBatchOuter の結果に対して呼び出される。
  • **呼び出し先**:
  • palloc0（メモリ確保）

関連データ構造

  • SharedTuplestore: 共有メモリ上の制御構造体。
  • SharedTuplestoreAccessor: 各バックエンドが保持する操作用構造体。
  • SharedFileSet: 一時ファイルの管理用構造体。

失敗モードと境界条件

  • my_participant_number が不正な場合、Assert が有効なビルドではプロセスが停止する。
  • メモリ不足時には palloc0 がエラーを発生させる。

保守担当者向けメモ

  • 本関数は書き込みやスキャンの準備段階で呼び出される。
  • SharedTuplestoreAccessor は各バックエンドのローカルメモリ上に作成されるため、プロセス間での共有は行われない。
