PGSemaphoreUnlock

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_sema.c:195

目的

Windows環境において、指定されたセマフォのカウントを1つ増やし、待機中のプロセスを解放します。

シグネチャまたは形状

void PGSemaphoreUnlock(PGSemaphore sema)

入力と出力

  • **入力**: sema (PGSemaphore型) - 解放対象のセマフォハンドル。
  • **出力**: なし。

前提条件と不変条件

  • sema は PGSemaphoreCreate によって正常に作成された有効なWin32セマフォハンドルである必要があります。

アルゴリズムまたは振る舞い

1. Win32 APIの ReleaseSemaphore を呼び出し、セマフォのカウントを1増加させます。 2. ReleaseSemaphore が失敗した場合（戻り値が0）、ereport(FATAL, ...) を呼び出し、GetLastError() で取得したエラーコードと共にプロセスを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/clog.c
  • src/backend/storage/ipc/procarray.c
  • src/backend/storage/lmgr/lwlock.c
  • **呼び出し先**:
  • ReleaseSemaphore (Win32 API)
  • ereport (PostgreSQLエラー報告機能)

関連データ構造

  • PGSemaphore: セマフォを管理するための抽象型。Windows実装ではWin32のセマフォハンドルとして扱われます。

失敗モードと境界条件

  • ReleaseSemaphore が失敗した場合、致命的なエラーとしてプロセスが停止します。これはセマフォの解放に失敗することがシステム全体の整合性に重大な影響を与えるためです。

保守担当者向けメモ

  • 本関数はWindows固有の実装です。POSIXやSystem Vの実装とは異なり、カーネルオブジェクトの解放を直接Win32 APIに依存しています。
  • 呼び出し元で sema がNULLでないことを確認する設計になっていますが、関数内では ReleaseSemaphore の戻り値によるエラーハンドリングのみを行っています。
