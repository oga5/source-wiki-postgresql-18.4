WalSndCtlData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/walsender_private.h:84

目的

PostgreSQLの全WAL Senderプロセスを統括するグローバルな共有メモリ構造体です。同期レプリケーションの待機キュー管理、スタンバイの状態追跡、およびプロセス間通知のための同期プリミティブを提供します。

シグネチャまたは形状

typedef struct {
    dlist_head      SyncRepQueue[NUM_SYNC_REP_WAIT_MODE];
    XLogRecPtr      lsn[NUM_SYNC_REP_WAIT_MODE];
    bits8           sync_standbys_status;
    ConditionVariable wal_flush_cv;
    ConditionVariable wal_replay_cv;
    ConditionVariable wal_confirm_rcv_cv;
    WalSnd          walsnds[FLEXIBLE_ARRAY_MEMBER];
} WalSndCtlData;

入力と出力

  • **入力**: 同期レプリケーション待機中のバックエンドによるキュー登録、WAL Senderによる状態更新、チェックポインターによる sync_standbys_status の更新。
  • **出力**: 同期レプリケーションの待機解除通知（Condition Variable経由）、各WAL Senderの状態保持。

前提条件と不変条件

  • SyncRepQueue、lsn、sync_standbys_status へのアクセスは SyncRepLock による保護が必要です。
  • walsnds 配列は共有メモリ上に確保され、FLEXIBLE_ARRAY_MEMBER を用いて可変長で定義されています。
  • 待機中のバックエンドは設定ファイルの再読み込みを安全に行えないため、sync_standbys_status はチェックポインターによって更新されます。

アルゴリズムまたは振る舞い

1. **同期レプリケーション管理**: SyncRepQueue を用いて、コミット完了を待機するバックエンドを管理します。 2. **通知メカニズム**: ConditionVariable（wal_flush_cv, wal_replay_cv, wal_confirm_rcv_cv）を介して、WALのフラッシュやリプレイ、受信確認が発生した際に待機中のプロセスを起床させます。 3. **状態保持**: walsnds 配列により、個々のWAL Senderプロセスの実行状態を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c（初期化および操作）、src/backend/replication/syncrep.c（同期レプリケーション制御）。
  • **呼び出し先**: ConditionVariable 関連関数、dlist 操作関数。

関連データ構造

  • WalSnd: 個々のWAL Senderの状態を保持する構造体。
  • SyncRepQueue: 同期レプリケーションの待機リスト。
  • ConditionVariable: プロセス間同期のためのプリミティブ。

失敗モードと境界条件

  • walsnds は FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体全体のサイズは実行時に offsetof(WalSndCtlData, walsnds) を用いて計算する必要があります。
  • 共有メモリ上の構造体であるため、排他制御（スピンロック等）を欠いたアクセスはデータ破損を招きます。

保守担当者向けメモ

  • 本構造体は walsender.c の内部実装を共有メモリ経由で公開するものです。直接的な外部APIではないため、変更時は syncrep.c への影響を考慮してください。
  • sync_standbys_status のフラグ定義は同ヘッダーファイル内の定数に依存しています。
