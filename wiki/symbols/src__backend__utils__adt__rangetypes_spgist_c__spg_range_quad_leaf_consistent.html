spg_range_quad_leaf_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_spgist.c:916

目的

SP-GiSTインデックスのリーフノードにおいて、格納されている範囲データが検索条件（スキャンキー）を満たすかどうかを判定する。

シグネチャまたは形状

Datum spg_range_quad_leaf_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgLeafConsistentIn（リーフデータとスキャンキーを含む）および spgLeafConsistentOut を受け取る。
  • **出力**: PG_RETURN_BOOL(res) により、すべてのスキャンキー条件を満たす場合は true、それ以外は false を返す。また、out->recheck を false に設定し、結果が完全であることを示す。

前提条件と不変条件

  • in->leafDatum は有効な RangeType として解釈可能であること。
  • 検索条件の各 sk_strategy は、範囲型に対して定義された演算子に対応していること。

アルゴリズムまたは振る舞い

1. out->recheck を false に設定し、インデックスによる判定が確定的なものであることを示す。 2. range_get_typcache を使用して、対象範囲型の型キャッシュを取得する。 3. すべてのスキャンキー（in->nkeys）を順次評価する。 4. sk_strategy に基づき、range_before_internal や range_overlaps_internal などの対応する内部関数を呼び出し、リーフデータとキーを比較する。 5. いずれかの条件で false となった場合、残りのキー評価をスキップして即座に終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTインデックスの検索プロセスから呼び出される（直接的な呼び出し元情報はソースコード上では明示されていない）。
  • **呼び出し先**: range_get_typcache および各種 range_*_internal 関数（range_before_internal, range_overlaps_internal 等）。

関連データ構造

  • spgLeafConsistentIn: 検索対象のリーフデータおよびスキャンキーを保持する構造体。
  • spgLeafConsistentOut: 検索結果および再チェックフラグを格納する構造体。
  • RangeType: 範囲型の内部表現。
  • TypeCacheEntry: 範囲型の型情報キャッシュ。

失敗モードと境界条件

  • **未定義の戦略**: sk_strategy が RANGESTRAT のいずれにも該当しない場合、elog(ERROR) によりエラーが発生する。
  • **型不一致**: leafDatum が範囲型として正しく解析できない場合、range_get_typcache 等でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はすべてのテストが「完全（exact）」であると見なしているため、recheck は常に false となる。
  • 新しい範囲演算子を追加する場合は、switch 文に新しい case を追加する必要がある。
