Ranges

定義

  • 種別: struct
  • 定義箇所: src/backend/access/brin/brin_minmax_multi.c:169

目的

BRIN インデックスにおいて、ページ範囲内の値を複数の間隔（Interval）および単一ポイントの集合としてメモリ上で管理するための構造体です。外れ値を含むデータ分布を効率的に保持し、不要なページ範囲のスキップを可能にします。

シグネチャまたは形状

typedef struct Ranges {
    Oid         typid;
    Oid         colloid;
    AttrNumber  attno;
    FmgrInfo   *cmp;
    int         nranges;
    int         nsorted;
    int         nvalues;
    int         maxvalues;
    int         target_maxvalues;
    Datum       values[FLEXIBLE_ARRAY_MEMBER];
} Ranges;

入力と出力

  • **入力**: brin_minmax_multi_add_value 等を通じて新しい値が values 配列に追加されます。
  • **出力**: brin_range_serialize を介してディスク格納用の SerializedRanges へ変換されます。

前提条件と不変条件

  • (2 * nranges + nvalues) <= maxvalues という制約を満たす必要があります。
  • values 配列は、シリアライズ前の最終段階で指定された target_maxvalues に収まるよう圧縮される必要があります。
  • AssertCheckRanges により、データ構造の整合性（ソート順序や重なりの有無など）が検証されます。

アルゴリズムまたは振る舞い

  • **バッファリング**: 値の追加時にはソートや重複排除といった高コストな処理を避け、maxvalues までバッファに蓄積します。
  • **圧縮**: バッファが制限に達した際、間隔同士の距離計算を行い、最も近い間隔をマージすることでサイズを縮小します。
  • **効率化**: 頻繁な圧縮を避けるため、target_maxvalues よりも大きなバッファを確保して運用します。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_range_deserialize（デシリアライズ時）、brin_minmax_multi_add_value（値追加時）などで利用されます。
  • **呼び出し先**: AssertCheckRanges（整合性チェック）、brin_range_serialize（シリアライズ時）などが本構造体を操作します。
  • ※詳細なコールグラフは提供されたソース範囲外のため、これ以上の関係性は不明です。

関連データ構造

  • SerializedRanges: ディスク上の物理格納形式。
  • ExpandedRange: マージ処理時に使用される正規化された一時表現。
  • FmgrInfo: 比較関数を保持する PostgreSQL の標準構造体。

失敗モードと境界条件

  • maxvalues を超える値の追加は、圧縮処理（マージ）をトリガーします。
  • 距離計算は UUID や INET 型などの複雑な型では近似値が使用されるため、マージの最適化効率に影響を与える可能性があります。

保守担当者向けメモ

  • 圧縮処理は高コストであるため、MINMAX_BUFFER_FACTOR を用いたバッファ確保が重要です。
  • コード内の AssertCheckRanges は、構造体の不変条件を理解するための重要なリファレンスとなります。
