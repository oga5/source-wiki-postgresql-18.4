worker_spi_main

定義

  • 種別: function
  • 定義箇所: src/test/modules/worker_spi/worker_spi.c:133

目的

バックグラウンドワーカーとして動作し、データベース接続の確立、SPIを用いた定期的な集計クエリの実行、およびシグナル処理を行うメインループを提供します。

シグネチャまたは形状

void worker_spi_main(Datum main_arg)

入力と出力

  • **入力**: main_arg (Datum型): ワーカーのインデックス番号。MyBgworkerEntry->bgw_extra からデータベースOID、ロールOID、フラグ情報を取得します。
  • **出力**: なし（無限ループ内でプロセス終了まで実行）。

前提条件と不変条件

  • BackgroundWorkerUnblockSignals() が呼び出されるまでシグナルはブロックされている必要があります。
  • SPIを使用するため、StartTransactionCommand() と CommitTransactionCommand() でトランザクションを明示的に制御する必要があります。
  • WaitLatch を使用して待機し、CHECK_FOR_INTERRUPTS() を定期的に呼び出すことで、Postmasterの終了や設定変更（SIGHUP）に即座に応答する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: bgw_extra から接続情報を読み取り、シグナルハンドラ（SIGHUP, SIGTERM）を設定し、シグナルをアンブロックします。 2. **接続**: BackgroundWorkerInitializeConnection 等を用いてデータベースセッションを確立します。 3. **セットアップ**: initialize_worker_spi を呼び出し、対象テーブルを準備します。 4. **メインループ**:

  • WaitLatch で worker_spi_naptime 分だけ待機します。
  • ConfigReloadPending が真であれば設定を再読み込みします。
  • トランザクションを開始し、SPI_connect を経由して集計クエリ（DELETE/UPDATE）を実行します。
  • 実行結果をログ出力し、SPI_finish とトランザクションコミットを行います。
  • pgstat_report_activity で状態を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: バックグラウンドワーカーの起動プロセス（postmaster）。ソースコード内では worker_spi_main という文字列名で参照されています。
  • **呼び出し先**: initialize_worker_spi, BackgroundWorkerInitializeConnection, WaitLatch, SPI_execute, StartTransactionCommand, CommitTransactionCommand 等。

関連データ構造

  • worktable: スキーマ名とテーブル名を保持する構造体。
  • MyBgworkerEntry: バックグラウンドワーカーの登録情報を保持するグローバル変数。

失敗モードと境界条件

  • SPI_execute が期待した結果（SPI_OK_UPDATE_RETURNING）を返さない場合、elog(FATAL) によりプロセスが異常終了します。
  • SIGTERM を受信するとループを脱出し、プロセスが終了します。
  • WaitLatch の WL_EXIT_ON_PM_DEATH フラグにより、Postmasterが終了した場合は即座にプロセスが終了します。

保守担当者向けメモ

  • usleep 等の直接的な待機処理は避け、必ず WaitLatch を使用してください。
  • SPI 実行時は SetCurrentStatementStartTimestamp() を適切に呼び出し、トランザクションスナップショットを管理してください。
  • メモリ割り当てはプロセス終了時に解放されますが、長時間実行されるためメモリリークには注意が必要です。
