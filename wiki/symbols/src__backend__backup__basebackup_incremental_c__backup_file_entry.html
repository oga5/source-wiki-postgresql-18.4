backup_file_entry

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup_incremental.c:55

目的

増分バックアップ処理において、バックアップ対象となる個々のファイルのメタデータ（状態、パス、サイズ）を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	uint32		status;
	const char *path;
	uint64		size;
} backup_file_entry;

入力と出力

  • 本構造体はデータ保持用であり、関数ではないため直接的な入出力はありません。
  • メンバ変数 path はファイルパスを指し、size はファイルのサイズを保持します。

前提条件と不変条件

  • lib/simplehash.h を用いたハッシュテーブルの要素として利用されることが想定されています。

アルゴリズムまたは振る舞い

  • 本構造体自体はデータコンテナとして機能します。
  • src/backend/backup/basebackup_incremental.c 内で、ハッシュテーブルの要素型（SH_ELEMENT_TYPE）として定義されています。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/backup/basebackup_incremental.c 内のハッシュテーブル操作ロジック（985行目付近など）でポインタとして参照されています。
  • 呼び出し先: 構造体であるため、関数としての呼び出し先はありません。

関連データ構造

  • lib/simplehash.h: 本構造体を要素として管理するハッシュテーブルの実装に使用。
  • IncrementalBackupInfo: 本構造体を含む増分バックアップの主要なメタデータ構造体。

失敗モードと境界条件

  • 構造体定義自体に失敗モードはありませんが、path メンバが指す文字列のメモリ管理は、この構造体を保持する側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は basebackup_incremental.c 内のハッシュテーブルの要素として定義されています。ハッシュテーブルのキーや比較関数を変更する際は、この構造体の定義と整合性を取る必要があります。
