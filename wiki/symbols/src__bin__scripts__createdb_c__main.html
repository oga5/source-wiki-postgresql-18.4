目的

PostgreSQL のデータベース作成コマンド createdb のメインエントリポイントとして、コマンドライン引数の解析、サーバーへの接続確立、および CREATE DATABASE SQL 文の生成と実行を制御します。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**: コマンドライン引数 (argc, argv)。接続先ホスト、ポート、ユーザー名、データベース名、およびロケールや所有者などの各種オプション。
  • **出力**: 成功時は 0、エラー発生時は 1 などの終了ステータスを返します。

前提条件と不変条件

  • pg_logging_init および set_pglocale_pgservice による環境初期化が完了していること。
  • データベース名が指定されない場合、環境変数 PGDATABASE、PGUSER、または現在の OS ユーザー名がデフォルトとして使用されます。
  • maintenance_db が指定されていない場合で、作成対象が postgres データベースであるときは、接続先として template1 が自動選択されます。

アルゴリズムまたは振る舞い

1. **初期化**: ログ出力とロケール設定を初期化します。 2. **引数解析**: getopt_long を使用して、long_options に定義されたオプションを解析し、各変数に格納します。 3. **引数検証**: 引数の個数をチェックし、データベース名とオプションのコメントを取得します。 4. **接続**: connectMaintenanceDatabase を呼び出し、サーバーへの接続を確立します。 5. **SQL 構築**: PQExpBuffer を使用して、解析されたオプションに基づき CREATE DATABASE 文を構築します。 6. **実行**: PQexec を介してサーバーへ SQL を送信します。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（コマンド実行時）。
  • **呼び出し先**:
  • getopt_long: 引数解析に使用。
  • connectMaintenanceDatabase: サーバー接続に使用。
  • pg_strdup, pg_log_error, pg_fatal: ユーティリティ関数。
  • PQclientEncoding, PQexec, PQfinish: libpq 関数。
  • 注: ツールが提示した contrib/amcheck 等の参照は、本シンボルとは無関係な同名の識別子に対する言及であり、本関数の呼び出し関係を示すものではありません。

関連データ構造

  • struct option long_options[]: コマンドライン引数の定義。
  • ConnParams cparams: 接続パラメータ保持用。
  • PQExpBufferData sql: SQL 文構築用バッファ。
  • PGconn, PGresult: libpq の接続および結果保持用構造体。

失敗モードと境界条件

  • 引数が多すぎる場合、エラーメッセージを表示して終了します。
  • 指定されたエンコーディングが無効な場合、pg_fatal で異常終了します。
  • 接続失敗や SQL 実行エラーが発生した場合、libpq のエラーハンドリングを経て終了します。

保守担当者向けメモ

  • SQL インジェクションを防ぐため、SQL 構築時には fmtId や appendStringLiteralConn を使用してください。
  • 新しいロケール関連オプションを追加する場合は、main 内の switch 文と help 関数の両方を更新する必要があります。
