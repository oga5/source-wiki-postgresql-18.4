pg_sha256_update

定義

  • 種別: function
  • 定義箇所: src/common/sha2.c:475

目的

SHA-256 ハッシュ計算において、入力データストリームをバッファリングし、ブロック単位（64バイト）に達するごとに中間ハッシュ状態を更新します。

シグネチャまたは形状

void pg_sha256_update(pg_sha256_ctx *context, const uint8 *data, size_t len)

入力と出力

  • context: SHA-256 コンテキスト構造体（入力および更新対象）。
  • data: ハッシュ計算対象のバイト列。
  • len: data のバイト長。
  • 出力: なし（context 内部の状態が更新されます）。

前提条件と不変条件

  • len が 0 の場合、関数は何も行わず即座に復帰します。
  • context は事前に pg_sha256_init で初期化されている必要があります。
  • context->bitcount は処理されたデータの総ビット数を保持し、更新のたびに加算されます。

アルゴリズムまたは振る舞い

1. **バッファの埋め合わせ**: コンテキスト内の既存バッファに空きがある場合、入力データでバッファを埋めます。ブロック長（PG_SHA256_BLOCK_LENGTH）に達した時点で SHA256_Transform を呼び出し、バッファを処理します。 2. **ブロック単位の処理**: 入力データがブロック長以上残っている間、SHA256_Transform を直接呼び出してデータを順次処理します。 3. **残余データの保持**: ブロック長に満たない端数のデータは、次回の呼び出しのために context->buffer へコピーされます。 4. **状態更新**: 各ステップで context->bitcount を更新し、処理したデータ量分だけ len と data ポインタを進めます。

関連データ構造

  • pg_sha256_ctx: sha2_int.h で定義される SHA-256 用のコンテキスト構造体。

失敗モードと境界条件

  • len == 0: 正常終了（何もしない）。
  • 入力データがブロック境界で分割される場合: バッファリングと変換が適切に組み合わされ、データ損失なく処理されます。

保守担当者向けメモ

  • 本関数は SHA2_UNROLL_TRANSFORM が定義されている場合、変換ループのアンロールによる最適化が適用されます。
  • usedspace および freespace の計算は、bitcount を基に行われるため、コンテキストの整合性が維持されていることが重要です。
