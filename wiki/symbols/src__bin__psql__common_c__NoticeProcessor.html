NoticeProcessor

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:278

目的

データベースサーバーから送信された通知メッセージ（Notice）をログとして出力するコールバック関数です。

シグネチャまたは形状

void NoticeProcessor(void *arg, const char *message)

入力と出力

  • arg: void * 型。libpq の PQsetNoticeProcessor から渡される引数ですが、本関数内では使用されません。
  • message: const char * 型。サーバーから受信した通知メッセージの文字列。
  • 戻り値: なし（void）。

前提条件と不変条件

  • 本関数は libpq の通知処理用コールバックとして登録されることを前提としています。
  • message は有効なヌル終端文字列である必要があります。

アルゴリズムまたは振る舞い

1. 引数 arg を明示的に無視（キャストして破棄）します。 2. pg_log_info を使用して、受け取った message を標準出力またはログ出力先に書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/psql/command.c:4321 (PQsetNoticeProcessor 経由)
  • src/bin/psql/startup.c:328 (PQsetNoticeProcessor 経由)
  • **呼び出し先**:
  • pg_log_info (外部ライブラリ関数)

関連データ構造

  • libpq の通知処理インターフェースに関連します。

失敗モードと境界条件

  • message が NULL の場合、pg_log_info の実装に依存しますが、通常は未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は libpq の PQsetNoticeProcessor に渡すための標準的なハンドラです。
  • arg 引数は将来的な拡張のためにシグネチャに含まれていますが、現在は使用されていません。変更を加える際は、PQsetNoticeProcessor の仕様と整合性を保ってください。
