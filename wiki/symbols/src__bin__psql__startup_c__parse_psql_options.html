parse_psql_options

定義

  • 種別: function
  • 定義箇所: src/bin/psql/startup.c:487

目的

psql の起動時にコマンドライン引数を解析し、実行設定（adhoc_opts）およびグローバルな設定状態（pset）を初期化・更新する。

シグネチャまたは形状

static void parse_psql_options(int argc, char *argv[], struct adhoc_opts *options)

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数の数と配列。
  • options: 解析結果を格納する adhoc_opts 構造体へのポインタ。
  • **出力**: なし（副作用として options およびグローバル変数 pset を更新）。

前提条件と不変条件

  • options は呼び出し前にメモリ確保済みであること。
  • memset により、解析開始時に options はゼロクリアされる。
  • pset はグローバル変数として初期化済みであること。

アルゴリズムまたは振る舞い

1. getopt_long を使用し、定義されたロングオプションおよびショートオプションをループで順次処理する。 2. 各オプションに応じて以下の処理を行う：

  • **変数設定**: SetVariable 等を用いて pset.vars を更新（例: -a, -e, -q）。
  • **出力フォーマット**: pset.popt.topt の各フィールドを更新（例: -A, -H, -F, -T）。
  • **アクション登録**: simple_action_list_append を呼び出し、クエリ実行やファイル読み込みのタスクを options->actions に追加（例: -c, -f）。
  • **接続情報**: options 構造体のフィールドに値をコピー（例: -d, -h, -p, -U）。
  • **特殊処理**: -P（印刷パラメータ設定）や -v（変数設定）では、文字列を解析して do_pset や SetVariable/DeleteVariable を呼び出す。

    • 3. 解析中に致命的なエラーが発生した場合、exit(EXIT_FAILURE) を呼び出してプログラムを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/startup.c 内の main 関数から呼び出される。
  • **呼び出し先**:
  • SetVariable, SetVariableBool, DeleteVariable
  • simple_action_list_append
  • pg_strdup, pg_fatal
  • setQFout, do_pset
  • getopt_long (標準ライブラリ)

関連データ構造

  • struct adhoc_opts: コマンドライン引数から得られた実行オプションを保持する構造体。
  • pset (PsqlSettings): psql の実行状態を保持するグローバル変数。

失敗モードと境界条件

  • 不正なオプションや引数不足が発生した場合、getopt_long の挙動に従いエラーとなる。
  • -P や -v でのパラメータ解析時に形式が不正な場合、pg_fatal または exit(EXIT_FAILURE) により終了する。
  • メモリ不足時は pg_strdup 等の内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 新しいコマンドラインオプションを追加する場合は、long_options 配列への定義追加と、switch 文でのケース追加が必要。
  • pset はグローバル変数であるため、この関数内での変更はプログラム全体の動作に直結する点に注意すること。
