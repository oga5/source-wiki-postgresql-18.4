list_free_private

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/list.c:1519

目的

PostgreSQLの動的リスト構造体およびその要素が保持するメモリを解放する内部関数。

シグネチャまたは形状

static void list_free_private(List *list, bool deep)

入力と出力

  • list: 解放対象の List 構造体へのポインタ。
  • deep: true の場合、リスト内の各要素が指すメモリ領域も pfree で解放する。
  • 戻り値: なし。

前提条件と不変条件

  • list が NIL の場合は何もしない。
  • check_list_invariants(list) により、解放前にリストの整合性が検証される。

アルゴリズムまたは振る舞い

1. 引数 list が NIL であれば即座に終了する。 2. check_list_invariants を呼び出し、リストの整合性を確認する。 3. deep が true の場合、リスト内の全要素に対して pfree(lfirst(&list->elements[i])) を実行し、要素が指すメモリを解放する。 4. リストの要素配列 list->elements が初期領域 list->initial_elements と異なる場合、配列領域を pfree で解放する。 5. 最後にリストヘッダ自体を pfree で解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • list_free(List *list) (引数 deep=false で呼び出し)
  • list_free_deep(List *list) (引数 deep=true で呼び出し)
  • 呼び出し先:
  • check_list_invariants
  • pfree
  • lfirst (マクロ)

関連データ構造

  • List: リストのヘッダ構造体。elements 配列へのポインタと現在の長さを保持。

失敗モードと境界条件

  • list が NIL の場合、安全に無視される。
  • deep が true の場合、リスト内の各要素が有効なメモリ領域を指している必要がある。無効なポインタに対して pfree を実行するとメモリ破壊を引き起こす。

保守担当者向けメモ

  • 本関数は list_free および list_free_deep の共通実装である。
  • list->elements が list->initial_elements を指している場合、配列領域はヘッダと一体で確保されているため、個別の pfree は行われない。
