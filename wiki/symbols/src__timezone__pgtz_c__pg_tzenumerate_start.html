pg_tzenumerate_start

定義

  • 種別: function
  • 定義箇所: src/timezone/pgtz.c:396

目的

タイムゾーン列挙処理を開始するためのイテレータ構造体（pg_tzenum）を初期化し、タイムゾーンディレクトリの走査準備を行う。

シグネチャまたは形状

pg_tzenum *pg_tzenumerate_start(void)

入力と出力

  • **入力**: なし。
  • **出力**:
  • 成功時: 初期化された pg_tzenum 構造体へのポインタ。
  • 失敗時: ディレクトリのオープンに失敗した場合、ereport(ERROR, ...) により処理が中断される。

前提条件と不変条件

  • pg_TZDIR() が有効なタイムゾーンディレクトリパスを返すこと。
  • palloc0 によるメモリ確保が成功すること。

アルゴリズムまたは振る舞い

1. pg_tzenum 構造体メモリを確保し、ゼロクリアする。 2. pg_TZDIR() から取得したパスを startdir として保持する。 3. baselen に startdir の長さを設定する。 4. depth を 0 に初期化する。 5. AllocateDir を使用して startdir を開き、そのディレクトリ記述子を dirdesc[0] に格納する。 6. ディレクトリのオープンに失敗した場合はエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/datetime.c:5349
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • pstrdup (文字列複製)
  • pg_TZDIR (ディレクトリパス取得)
  • strlen (文字列長計算)
  • AllocateDir (ディレクトリオープン)
  • ereport (エラー通知)

関連データ構造

  • pg_tzenum: タイムゾーン列挙の状態を保持する構造体。

失敗モードと境界条件

  • AllocateDir が失敗した場合、ereport によりエラーが発生し、呼び出し元へは戻らない。

保守担当者向けメモ

  • 本関数は pg_tzenumerate_next と対で使用される。
  • palloc を使用しているため、呼び出し側のメモリコンテキストが適切に管理されている必要がある。
