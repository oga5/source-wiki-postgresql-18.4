bbsink_progress_archive_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_progress.c:149

目的

バックアップのデータ転送中に、転送済みバイト数を追跡し、システム統計ビュー pg_stat_progress_basebackup を更新する。

シグネチャまたは形状

static void bbsink_progress_archive_contents(bbsink *sink, size_t len)

入力と出力

  • sink: 処理対象の bbsink オブジェクト。
  • len: 今回転送されたデータのバイト数。
  • 出力: なし（副作用として bbsink_state の更新および pgstat への統計報告を行う）。

前提条件と不変条件

  • sink->bbs_state が有効な bbsink_state を指していること。
  • bbsink_forward_archive_contents を介して、後続のシンクへデータが透過的に転送されること。

アルゴリズムまたは振る舞い

1. state->bytes_done に len を加算する。 2. bbsink_forward_archive_contents(sink, len) を呼び出し、データを後続のシンクへ転送する。 3. pgstat_progress_update_multi_param を使用し、PROGRESS_BASEBACKUP_BACKUP_STREAMED を更新する。 4. state->bytes_total_is_valid が真かつ bytes_done が bytes_total を超過した場合、PROGRESS_BASEBACKUP_BACKUP_TOTAL を現在の bytes_done で上書きし、進捗率の不整合（100%超え）を回避する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink インターフェースの archive_contents コールバックとして登録されており、バックアップのデータ転送パイプラインから呼び出される。
  • 呼び出し先: bbsink_forward_archive_contents（後続シンクへの転送）、pgstat_progress_update_multi_param（統計更新）。

関連データ構造

  • bbsink: バックアップのデータ転送パイプラインを構成する構造体。
  • bbsink_state: 転送済みバイト数や合計バイト数などの状態を保持する構造体。

失敗モードと境界条件

  • bytes_done が bytes_total を超える場合、合計値を動的に補正するロジックが働く。
  • 統計更新時に nparam を動的に制御し、合計値の補正が必要な場合のみ PROGRESS_BASEBACKUP_BACKUP_TOTAL を更新する。

保守担当者向けメモ

  • 統計情報の更新は pgstat サブシステムに依存している。
  • 外部からの呼び出し元に関する詳細なスタックトレース情報は、本ソース抜粋からは特定できない。
