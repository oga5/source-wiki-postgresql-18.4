CalculateCopyStreamSleeptime

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:1241

目的

ストリーミングレプリケーションの受信ループにおいて、サーバーへのステータス更新（フィードバック）送信までの待機時間（ミリ秒）を計算します。

シグネチャまたは形状

static long CalculateCopyStreamSleeptime(TimestampTz now, int standby_message_timeout, TimestampTz last_status)

入力と出力

  • **入力**:
  • now: 現在のタイムスタンプ。
  • standby_message_timeout: ステータス更新の間隔（秒）。0の場合はタイムアウト無効。
  • last_status: 前回のステータス更新時刻。
  • **出力**:
  • long: 次のステータス更新までの待機時間（ミリ秒）。更新が不要な場合は -1 を返します。

前提条件と不変条件

  • standby_message_timeout が有効かつ still_sending が真である場合にのみ、待機時間が計算されます。
  • 待機時間は常に最低でも 1000 ミリ秒（1秒）以上となるよう補正されます。

アルゴリズムまたは振る舞い

1. standby_message_timeout が設定されており、かつ送信中であれば、前回のステータス更新時刻からタイムアウトまでの目標時刻 (status_targettime) を算出します。 2. 目標時刻が設定されている場合、feTimestampDifference を用いて現在時刻との差分を計算します。 3. 差分が 1 秒未満の場合は、強制的に 1 秒（1000 ミリ秒）を待機時間として設定します。 4. 目標時刻が設定されていない場合は -1 を返し、待機不要であることを示します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/receivelog.c:803 （HandleCopyStream 内のメインループ）
  • **呼び出し先**: feTimestampDifference（外部ユーティリティ関数）

関連データ構造

  • TimestampTz: PostgreSQLのタイムスタンプ型。
  • still_sending: 本関数が参照する静的変数（ファイルスコープ）。

失敗モードと境界条件

  • standby_message_timeout が 0 の場合、常に -1 を返し、待機処理をスキップさせます。
  • status_targettime が過去の時刻である場合、計算結果は 1 秒に切り上げられます。

保守担当者向けメモ

  • 本関数は HandleCopyStream のポーリング間隔を制御する重要なロジックです。
  • 待機時間の最小値（1秒）はハードコードされているため、将来的に動的な調整が必要な場合はこの定数を見直してください。
