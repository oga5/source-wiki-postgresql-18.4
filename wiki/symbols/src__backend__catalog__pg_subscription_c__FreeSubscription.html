FreeSubscription

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_subscription.c:185

目的

GetSubscription 等でメモリ上に確保された Subscription 構造体およびその内部メンバを解放し、メモリリークを防ぐ。

シグネチャまたは形状

void FreeSubscription(Subscription *sub)

入力と出力

  • **入力**: Subscription *sub - 解放対象のサブスクリプション構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 sub は有効な Subscription 構造体を指している必要がある。
  • sub 内の各メンバ（name, conninfo, slotname, publications）は、palloc または list_make 等で割り当てられたメモリ領域であること。

アルゴリズムまたは振る舞い

1. sub->name を解放する。 2. sub->conninfo を解放する。 3. sub->slotname が存在する場合（NULLでない場合）、これを解放する。 4. sub->publications リストを list_free_deep を用いて再帰的に解放する。 5. 最後に sub 構造体自体を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c:4071
  • **呼び出し先**:
  • pfree (メモリ解放)
  • list_free_deep (リスト構造の解放)

関連データ構造

  • Subscription 構造体（src/include/catalog/pg_subscription.h で定義）

失敗モードと境界条件

  • sub が NULL の場合、ポインタ参照によりクラッシュする可能性があるため、呼び出し側で NULL チェックを行う必要がある。

保守担当者向けメモ

  • Subscription 構造体の定義が変更され、新しい動的メモリメンバが追加された場合は、本関数で適切に pfree を追加すること。
