QPRS_STATE

定義

  • 種別: struct
  • 定義箇所: contrib/ltree/ltxtquery_io.c:37

目的

ltxtquery 型の解析処理において、現在の解析状態、演算子スタック、およびエラーコンテキストを保持するために使用される状態管理構造体です。

シグネチャまたは形状

typedef struct {
    char       *buf;            /* 入力バッファ */
    int32       state;          /* 現在の解析状態 */
    int32       count;          /* カウンタ */
    struct Node *escontext;     /* エラーコンテキスト */
    NODE       *str;            /* 逆ポーランド記法用の一時リスト */
    int32       num;            /* str 内の要素数 */
    int32       lenop;          /* オペランド長 */
    int32       sumlen;         /* 合計長 */
    char       *op;             /* オペランドバッファ */
    char       *curop;          /* 現在のオペランド位置 */
} QPRS_STATE;

入力と出力

本構造体は関数引数として渡され、解析の進行に伴い内部の str（ノードリスト）や op（オペランドバッファ）が更新されます。

前提条件と不変条件

  • makepol 関数による解析中、str メンバは逆ポーランド記法を構築するための一時的な連結リストとして使用されます。
  • メモリ管理において repalloc を用いた動的なバッファ拡張が行われるため、ポインタの整合性に注意が必要です。

アルゴリズムまたは振る舞い

  • ltxtq_in 関数内で初期化され、gettoken_query、pushquery、pushval_asis、makepol、findoprnd といった一連の解析関数間で状態を共有します。
  • クエリ文字列をトークン化し、演算子の優先順位に従ってツリー構造を構築する過程で、解析の進捗を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: ltxtq_in（解析の開始点）
  • **呼び出し先（参照・更新する関数）**:
  • gettoken_query
  • pushquery
  • pushval_asis
  • makepol
  • findoprnd

関連データ構造

  • NODE: 解析中に使用される連結リスト形式のツリーノード。
  • ITEM: ltxtquery 内部で保持される固定長配列要素。

失敗モードと境界条件

  • 再帰的な解析を行うため、スタックオーバーフローの危険性があります。これを防ぐため、関連関数内で check_stack_depth() が呼び出されます。
  • メモリ確保失敗時や構文エラー発生時には、escontext を通じてエラーが報告されます。

保守担当者向けメモ

  • 本構造体は ltxtquery の入出力処理の中核を成します。構造体のメンバを追加・変更する場合は、makepol および findoprnd におけるメモリ管理とポインタ操作への影響を十分に確認してください。
