pgoutput_begin_txn

定義

  • 種別: function
  • 定義箇所: src/backend/replication/pgoutput/pgoutput.c:593

目的

論理レプリケーションのトランザクション開始時に、プラグイン固有のトランザクション状態を初期化し、ReorderBufferTXN に関連付ける。

シグネチャまたは形状

static void pgoutput_begin_txn(LogicalDecodingContext *ctx, ReorderBufferTXN *txn)

入力と出力

  • **入力**:
  • ctx: 論理デコーディングのコンテキスト。メモリ割り当てに使用される。
  • txn: トランザクション情報を保持する ReorderBufferTXN 構造体。
  • **出力**: なし（副作用として txn->output_plugin_private に PGOutputTxnData が割り当てられる）。

前提条件と不変条件

  • ctx->context が有効なメモリコンテキストであること。
  • txn が有効なポインタであること。

アルゴリズムまたは振る舞い

1. ctx->context 上に PGOutputTxnData 構造体分のメモリをゼロクリアして割り当てる。 2. 割り当てた PGOutputTxnData へのポインタを、txn->output_plugin_private フィールドに格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: 論理デコーディングのフレームワーク（_PG_output_plugin_init で begin_cb として登録されているため、フレームワークからコールバックとして呼び出される）。
  • **呼び出し先**: MemoryContextAllocZero（PostgreSQLのメモリ管理関数）。

関連データ構造

  • LogicalDecodingContext: デコーディングセッションのコンテキスト。
  • ReorderBufferTXN: トランザクションのメタデータとプラグイン固有データを保持する構造体。
  • PGOutputTxnData: 本関数によって割り当てられる、トランザクションごとの状態保持用構造体。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocZero がエラーを発生させる。

保守担当者向けメモ

  • 本関数はプラグインの初期化フェーズで begin_cb に登録される。トランザクション終了時のメモリ解放については、対応するコミットまたはアボート処理のコールバックで行われる必要がある。
