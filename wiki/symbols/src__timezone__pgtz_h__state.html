state

定義

  • 種別: struct
  • 定義箇所: src/timezone/pgtz.h:41

目的

IANA タイムゾーンデータベース（tzfile）のバイナリ形式をメモリ上に展開し、タイムゾーンの遷移情報、閏秒情報、および時刻型情報を保持するための構造体です。

シグネチャまたは形状

struct state {
    int         leapcnt;
    int         timecnt;
    int         typecnt;
    int         charcnt;
    bool        goback;
    bool        goahead;
    pg_time_t   ats[TZ_MAX_TIMES];
    unsigned char types[TZ_MAX_TIMES];
    struct ttinfo ttis[TZ_MAX_TYPES];
    char        chars[...];
    struct lsinfo lsis[TZ_MAX_LEAPS];
    int         defaulttype;
};

入力と出力

  • **入力**: tzload() 関数がタイムゾーンファイル（tzfile）を読み込み、この構造体の各フィールドを構築します。
  • **出力**: 構築された state 構造体は pg_tz 構造体に保持され、セッションやログのタイムゾーン変換処理に使用されます。

前提条件と不変条件

  • TZ_MAX_TIMES、TZ_MAX_TYPES、TZ_MAX_LEAPS などの定数は tzfile.h で定義されており、配列サイズを決定します。
  • defaulttype は、最近の tzdb リリースでは常に 0 ですが、2018e 以前のデータでは非ゼロとなる可能性があります。

アルゴリズムまたは振る舞い

1. pg_open_tzfile() により対象のタイムゾーンファイルが特定されます。 2. tzload() がファイル内容を解析し、遷移時刻（ats）、型情報（ttis）、閏秒情報（lsis）を本構造体に格納します。 3. 時刻変換時には、これらの情報を参照して適切なタイムゾーンオフセットや夏時間フラグを決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/timezone/localtime.c 内の tzload() がこの構造体を初期化します。
  • **呼び出し先**: タイムゾーン変換処理を行う関数群（localtime.c 等）が、時刻計算のために本構造体の各フィールドを参照します。
  • **注意**: ツールが提示した contrib/amcheck 内の state は、本構造体とは無関係な別名シンボルです。

関連データ構造

  • struct ttinfo: 時刻の型情報（オフセット、夏時間フラグ等）を保持。
  • struct lsinfo: 閏秒情報を保持。
  • struct pg_tz: 本構造体を内部状態として保持する上位構造体。

失敗モードと境界条件

  • TZ_MAX_TIMES を超える遷移情報を持つファイルは読み込み時にエラーとなる可能性があります。
  • 破損したタイムゾーンファイルが入力された場合、構造体の各フィールドに不正な値が設定されるリスクがあります。

保守担当者向けメモ

  • 本構造体は IANA タイムゾーンデータベースのファイル形式と密接に結合しています。定義を変更する場合は tzfile.h との整合性に注意してください。
  • 外部モジュールからタイムゾーン機能を利用する場合は、本構造体を直接操作せず pgtime.h のインターフェースを使用してください。
