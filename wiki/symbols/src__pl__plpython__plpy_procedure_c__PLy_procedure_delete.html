PLy_procedure_delete

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_procedure.c:402

目的

PL/Python関数に関連付けられたリソース（Pythonオブジェクトおよびメモリコンテキスト）を解放し、キャッシュから削除されたプロシージャ構造体をクリーンアップする。

シグネチャまたは形状

void PLy_procedure_delete(PLyProcedure *proc)

入力と出力

  • 入力: PLyProcedure *proc - 削除対象のプロシージャ構造体へのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • 引数 proc は有効な PLyProcedure 構造体を指している必要がある。
  • proc が保持する mcxt（メモリコンテキスト）は、この関数が呼び出された後に無効化されるため、呼び出し元は以降このポインタを参照してはならない。

アルゴリズムまたは振る舞い

1. proc->code、proc->statics、proc->globals に対して Py_XDECREF を実行し、Pythonオブジェクトの参照カウントを減らす。 2. MemoryContextDelete(proc->mcxt) を呼び出し、プロシージャ専用のメモリコンテキストを破棄して関連するメモリを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/pl/plpython/plpy_main.c:324, 334
  • src/pl/plpython/plpy_procedure.c:106, 336
  • 呼び出し先:
  • Py_XDECREF (Python C API)
  • MemoryContextDelete (PostgreSQL MemoryContext API)

関連データ構造

  • PLyProcedure: 関数定義、コンパイル済みコード、実行環境を保持する主要構造体。

失敗モードと境界条件

  • proc が NULL の場合、Py_XDECREF は安全に処理されるが、proc->mcxt へのアクセスはセグメンテーション違反を引き起こす可能性があるため、呼び出し元で proc が非 NULL であることを保証する必要がある。

保守担当者向けメモ

  • 本関数は PLyProcedure が保持するメモリコンテキストを解放するため、呼び出し後に当該構造体へのポインタを保持し続けるとメモリリークや不正アクセスを招く。
  • Pythonオブジェクトの参照カウント管理とPostgreSQLのメモリコンテキスト管理が混在しているため、修正時には両方のライフサイクルを考慮すること。
