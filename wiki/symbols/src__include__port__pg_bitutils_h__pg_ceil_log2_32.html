pg_ceil_log2_32

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_bitutils.h:257

目的

与えられた 32 ビット符号なし整数 num に対し、ceil(log2(num)) を計算します。これは、num を格納するために必要な最小のビット幅（2 のべき乗の指数）を求める際に使用されます。

シグネチャまたは形状

static inline uint32 pg_ceil_log2_32(uint32 num)

入力と出力

  • **入力**: uint32 num （対象となる整数）
  • **出力**: uint32 （num をカバーする最小の 2 のべき乗の指数）

前提条件と不変条件

  • num が 2 未満（0 または 1）の場合、戻り値は 0 となります。
  • num > 1 の場合、結果は num - 1 の最上位ビット位置（0 始まり）に 1 を加えた値となります。

アルゴリズムまたは振る舞い

1. num < 2 の場合は 0 を返します。 2. それ以外の場合は、num - 1 を計算し、その値に対して pg_leftmost_one_pos32 を呼び出して最上位ビットの位置を取得します。 3. 取得した位置に 1 を加算して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_leftmost_one_pos32
  • **呼び出し元（直接的な証拠）**:
  • src/backend/access/hash/hashpage.c:661
  • src/backend/access/hash/hashutil.c:147
  • src/backend/utils/hash/dynahash.c:1807

関連データ構造

  • 特になし。

失敗モードと境界条件

  • num = 0 の場合、num - 1 はアンダーフローして 0xFFFFFFFF となりますが、関数内の条件分岐により 0 が返されるため安全です。
  • num = 1 の場合、条件分岐により 0 が返されます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • 内部で pg_leftmost_one_pos32 を利用しているため、プラットフォームごとの最適化（ハードウェア命令の利用など）はこの依存先関数を通じて行われます。
