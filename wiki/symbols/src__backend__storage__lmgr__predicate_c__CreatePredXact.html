CreatePredXact

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/predicate.c:581

目的

共有メモリ上の利用可能なリストから SERIALIZABLEXACT 構造体を確保し、アクティブなトランザクションリストへ登録する。

シグネチャまたは形状

static SERIALIZABLEXACT *CreatePredXact(void)

入力と出力

  • **入力**: なし
  • **出力**: 確保された SERIALIZABLEXACT 構造体へのポインタ。利用可能な構造体がない場合は NULL を返す。

前提条件と不変条件

  • PredXact グローバル構造体が初期化されていること。
  • 共有メモリ上の availableList および activeList に対する適切な排他制御が呼び出し元で確保されている必要がある（ソースコード単体ではロック取得の明示的な記述なし）。

アルゴリズムまたは振る舞い

1. PredXact->availableList が空かどうかを確認する。空であれば NULL を返して終了する。 2. availableList の先頭からノードを取り出し、dlist_container を用いて SERIALIZABLEXACT 構造体を取得する。 3. 取得した構造体の xactLink を使用して、PredXact->activeList の末尾に挿入する。 4. 確保した SERIALIZABLEXACT へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/predicate.c:1261
  • src/backend/storage/lmgr/predicate.c:1808
  • src/backend/storage/lmgr/predicate.c:4933
  • **呼び出し先**:
  • dlist_is_empty
  • dlist_pop_head_node
  • dlist_push_tail
  • dlist_container (マクロ)

関連データ構造

  • SERIALIZABLEXACT: 直列化可能トランザクションの状態を保持する構造体。
  • PredXact: 直列化可能トランザクションのリスト（availableList, activeList）を管理する共有メモリ上の制御構造体。

失敗モードと境界条件

  • availableList が空の場合、メモリ不足（または利用可能な SERIALIZABLEXACT の枯渇）として NULL を返す。呼び出し元はこの戻り値を適切に処理する必要がある。

保守担当者向けメモ

  • 本関数は static であり、predicate.c 内でのみ使用される。
  • 戻り値が NULL の場合の挙動は呼び出し元に依存するため、新規呼び出し箇所を追加する際は必ず NULL チェックを行うこと。
