pg_read_file_off_len_missing

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/genfile.c:300

目的

サーバー上の指定されたファイルから、オフセットと長さを指定してテキストデータを読み込みます。ファイルが存在しない場合にエラーを発生させるか、NULLを返すかを制御するフラグをサポートします。

シグネチャまたは形状

Datum pg_read_file_off_len_missing(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • filename_t (text): 読み込むファイルパス。
  • seek_offset (int64): 読み込み開始位置（バイト単位）。
  • bytes_to_read (int64): 読み込む最大バイト数。
  • missing_ok (bool): true の場合、ファイルが存在しなくてもエラーにせず NULL を返します。
  • **出力**:
  • Datum: 読み込んだファイル内容（text 型）。ファイルが存在しない場合に missing_ok が true であれば NULL を返します。

前提条件と不変条件

  • 呼び出し元は PostgreSQL の関数呼び出し規約（PG_FUNCTION_ARGS）に従う必要があります。
  • 内部で pg_read_file_common を呼び出すため、実行ユーザーは pg_read_server_files ロールの権限を有している必要があります。

アルゴリズムまたは振る舞い

1. 引数からファイルパス、オフセット、読み込みサイズ、および missing_ok フラグを取得します。 2. pg_read_file_common を呼び出し、テキスト形式での読み込みを実行します（バイナリ読み込みフラグは false に固定）。 3. pg_read_file_common が NULL を返した場合（ファイル不在かつ missing_ok が true の場合）、PG_RETURN_NULL() を実行します。 4. 読み込みに成功した場合、結果を text 型として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されることを想定しています（直接の呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: pg_read_file_common を呼び出します。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • **権限不足**: 実行ユーザーが pg_read_server_files ロールを持たない場合、pg_read_file_common 内でエラーが発生します。
  • **ファイル不在**: missing_ok が false の場合、ファイルが存在しないとエラーが発生します。true の場合は NULL が返されます。
  • **パスの不正**: convert_and_check_filename による検証により、データディレクトリ外へのアクセスは拒否されます。
