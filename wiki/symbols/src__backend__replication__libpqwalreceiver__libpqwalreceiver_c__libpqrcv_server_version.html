libpqrcv_server_version

定義

  • 種別: function
  • 定義箇所: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:469

目的

接続済みのレプリケーション接続（WalReceiverConn）を介して、接続先サーバーのバージョン番号を取得します。

シグネチャまたは形状

static int libpqrcv_server_version(WalReceiverConn *conn);

入力と出力

  • **入力**: WalReceiverConn *conn（アクティブなレプリケーション接続状態を保持する構造体）
  • **出力**: int（サーバーのバージョン番号。libpq の PQserverVersion が返す値に準拠）

前提条件と不変条件

  • 引数 conn は有効な WalReceiverConn オブジェクトを指している必要があります。
  • 内部の conn->streamConn（PGconn）が初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った WalReceiverConn 構造体から、内部の streamConn メンバを参照します。 2. libpq のライブラリ関数である PQserverVersion(conn->streamConn) を呼び出し、その結果を直接返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c 内の PQWalReceiverFunctions 構造体初期化時に、関数ポインタとして登録されています。
  • **呼び出し先**:
  • libpq ライブラリの PQserverVersion 関数。

関連データ構造

  • WalReceiverConn: 接続状態や PGconn を保持する内部構造体。

失敗モードと境界条件

  • conn が NULL の場合、あるいは conn->streamConn が無効な場合、PQserverVersion の挙動に依存してクラッシュまたは未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は WalReceiverFunctions インターフェースの一部として実装されています。
  • libpq のバージョン取得機能に対する単なるラッパーであるため、ロジックの変更は不要です。
