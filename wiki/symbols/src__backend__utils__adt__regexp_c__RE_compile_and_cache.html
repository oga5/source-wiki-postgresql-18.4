RE_compile_and_cache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regexp.c:140

目的

正規表現のコンパイル結果をキャッシュし、同一のパターン・フラグ・照合順序（collation）を持つ要求に対して再コンパイルを回避することで、正規表現処理のオーバーヘッドを削減する。

シグネチャまたは形状

regex_t *RE_compile_and_cache(text *text_re, int cflags, Oid collation)

入力と出力

  • **入力**:
  • text_re: コンパイル対象の正規表現パターン（text型）。
  • cflags: POSIX正規表現コンパイルフラグ。
  • collation: 照合順序（Oid）。
  • **出力**:
  • コンパイル済み正規表現オブジェクトへのポインタ（regex_t *）。

前提条件と不変条件

  • キャッシュは「自己組織化リスト」として実装されており、最も頻繁に使用されるエントリがリストの先頭に配置される。
  • キャッシュの最大保持数（MAX_CACHED_RES）はデフォルトで32。
  • コンパイルされた正規表現は RegexpCacheMemoryContext に保持され、バックエンドの生存期間中維持される。

アルゴリズムまたは振る舞い

1. **キャッシュ検索**: キャッシュ配列を先頭から走査し、パターン文字列、フラグ、照合順序が一致するエントリを探す。一致した場合はそのエントリをリストの先頭に移動し、即座に返す。 2. **初期化**: 初回呼び出し時に RegexpCacheMemoryContext を作成する。 3. **コンパイル**: 一致するものがない場合、入力パターンを pg_wchar に変換し、pg_regcomp を呼び出してコンパイルする。 4. **メモリ管理**: コンパイル失敗時はエラーを報告する。成功時は専用のメモリコンテキストを作成し、パターンを格納する。 5. **キャッシュ更新**: キャッシュが上限に達している場合、最も古い（末尾の）エントリを破棄する。新しいエントリをリストの先頭に挿入し、メモリコンテキストを RegexpCacheMemoryContext の配下に再配置する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/regexp.c 内の各正規表現関数（regexp_match, regexp_replace 等）。
  • src/backend/utils/adt/varlena.c（textregexreplace 等）。
  • **呼び出し先**:
  • pg_regcomp（正規表現コンパイル）。
  • pg_regerror（エラーメッセージ生成）。
  • MemoryContext 関連関数（AllocSetContextCreate, MemoryContextDelete 等）。

関連データ構造

  • regex_t: POSIX正規表現エンジンが使用するコンパイル済みパターン構造体。
  • cached_re_str: キャッシュされた正規表現、フラグ、照合順序、および関連するメモリコンテキストを保持する構造体。
  • RegexpCacheMemoryContext: キャッシュされた正規表現を格納するための永続的なメモリコンテキスト。

失敗モードと境界条件

  • **コンパイル失敗**: pg_regcomp が REG_OKAY 以外を返した場合、ereport(ERROR, ...) によりトランザクションが中断される。
  • **キャッシュ制限**: MAX_CACHED_RES を超える新しいパターンが要求された場合、最も古いエントリが削除される。

保守担当者向けメモ

  • 本関数は RegexpCacheMemoryContext を使用してメモリを管理しているため、キャッシュされた正規表現はバックエンド終了まで解放されない。
  • キャッシュミスが頻発する環境では、MAX_CACHED_RES の制限により頻繁な再コンパイルが発生する可能性がある。
