btinsert

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtree.c:201

目的

B-tree インデックスへ新しいタプルを挿入するための公開インターフェース関数です。

シグネチャまたは形状

bool btinsert(Relation rel, Datum *values, bool *isnull,
              ItemPointer ht_ctid, Relation heapRel,
              IndexUniqueCheck checkUnique,
              bool indexUnchanged,
              IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • rel: 挿入対象のインデックスリレーション。
  • values, isnull: インデックスキーのデータ配列とその NULL 状態。
  • ht_ctid: ヒープ内のタプル位置を示す ItemPointer。
  • heapRel: ヒープリレーション。
  • checkUnique: 一意性制約のチェック要否。
  • indexUnchanged: インデックスの更新が不要な場合のフラグ。
  • indexInfo: インデックス情報。
  • **出力**:
  • bool: 挿入が成功したか、あるいは一意性制約違反がなかったかを示す結果。

前提条件と不変条件

  • index_form_tuple を使用して、提供された values と isnull からインデックス用のタプル（IndexTuple）が生成されます。
  • 生成された itup->t_tid には ht_ctid が設定されます。

アルゴリズムまたは振る舞い

1. index_form_tuple を呼び出し、インデックスキーから IndexTuple を作成します。 2. 作成したタプルの t_tid フィールドに ht_ctid を格納します。 3. _bt_doinsert を呼び出し、実際の B-tree 構造への挿入処理を実行します。 4. 最後に pfree を呼び出し、一時的に作成した itup をメモリから解放します。 5. _bt_doinsert の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックスアクセスメソッドのルーチンとして登録されており、bthandler を通じて呼び出されます。
  • **呼び出し先**:
  • index_form_tuple (PostgreSQL 内部 API)
  • _bt_doinsert (src/backend/access/nbtree/nbtinsert.c)
  • pfree (メモリ管理 API)

関連データ構造

  • Relation: インデックスおよびヒープリレーションの構造体。
  • IndexTuple: B-tree インデックス内に格納されるタプル形式。
  • ItemPointer: ヒープタプルを指すポインタ。

失敗モードと境界条件

  • _bt_doinsert 内部で一意性制約違反が発生した場合、その振る舞いは checkUnique パラメータの設定に依存します。
  • メモリ不足時には index_form_tuple がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際の挿入ロジックは _bt_doinsert に委譲されています。
  • 挿入処理の詳細は src/backend/access/nbtree/nbtinsert.c を参照してください。
