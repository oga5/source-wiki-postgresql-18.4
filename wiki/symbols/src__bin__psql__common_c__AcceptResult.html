AcceptResult

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:415

目的

libpq から返された PGresult のステータスを評価し、処理が継続可能か否かを判定します。また、エラー発生時には必要に応じてエラーメッセージの出力と接続状態の確認を行います。

シグネチャまたは形状

bool AcceptResult(const PGresult *result, bool show_error)

入力と出力

  • **入力**:
  • result: 評価対象の PGresult ポインタ。
  • show_error: エラー時に pg_log_info を通じてメッセージを表示するかどうかを指定するフラグ。
  • **出力**:
  • bool: 処理が成功（true）か失敗（false）かを返します。

前提条件と不変条件

  • pset.db が有効なデータベース接続を保持していることが前提です（エラー時に PQerrorMessage(pset.db) を参照するため）。
  • result が NULL の場合、関数は即座に false を返します。

アルゴリズムまたは振る舞い

1. result が NULL であれば false を返します。 2. PQresultStatus(result) を評価します。

  • 以下のステータスは成功（true）とみなします: PGRES_COMMAND_OK, PGRES_TUPLES_OK, PGRES_TUPLES_CHUNK, PGRES_EMPTY_QUERY, PGRES_COPY_IN, PGRES_COPY_OUT, PGRES_PIPELINE_SYNC。
  • 以下のステータスは失敗（false）とみなします: PGRES_PIPELINE_ABORTED, PGRES_BAD_RESPONSE, PGRES_NONFATAL_ERROR, PGRES_FATAL_ERROR。
  • 上記以外のステータスは未知のエラーとして pg_log_error を出力し、false を返します。

    • 3. 判定結果が false かつ show_error が true の場合:

  • PQerrorMessage(pset.db) を取得し、内容が空でなければ pg_log_info で出力します。
  • CheckConnection() を呼び出し、接続状態の検証を行います。

    • 4. 最終的な判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/common.c 内の各所（ExecQueryAndProcessResults 等のクエリ実行フロー）。
  • **呼び出し先**: PQresultStatus, PQerrorMessage, pg_log_error, pg_log_info, CheckConnection。

関連データ構造

  • PGresult: libpq のクエリ結果構造体。
  • pset: psql のグローバル設定を保持する構造体（settings.h で定義）。

失敗モードと境界条件

  • result が NULL の場合、安全に false を返します。
  • 未知の PQresultStatus が渡された場合、ログを出力した上で false を返します。
  • show_error が true であっても、PQerrorMessage が空文字列の場合はログ出力を行いません。

保守担当者向けメモ

  • 本関数は psql のクエリ実行ループにおけるゲートキーパーです。新しい libpq のステータスコードが追加された場合は、本関数の switch 文に適切に追加する必要があります。
  • CheckConnection() の呼び出しは、接続断絶を検知して psql の状態を同期させるために重要です。
