utf8_to_gb18030

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c:217

目的

PostgreSQLの文字コード変換サブシステムにおいて、UTF-8エンコーディングからGB18030エンコーディングへの変換を実行する。

シグネチャまたは形状

Datum utf8_to_gb18030(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取る。
  • src: 変換元UTF-8文字列（CSTRING）
  • dest: 変換先バッファ（CSTRING）
  • len: 変換対象の長さ（INT32）
  • noError: エラーハンドリングのフラグ（BOOL）
  • **出力**: 変換されたバイト数（INT32）を PG_RETURN_INT32 で返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_UTF8, PG_GB18030) により、変換元がUTF-8、変換先がGB18030であることを検証する。

アルゴリズムまたは振る舞い

1. UtfToLocal 関数を呼び出し、変換処理を開始する。 2. 変換には gb18030_from_unicode_tree マッピングテーブルを使用する。 3. テーブル外の特殊な文字範囲については、補助関数 conv_utf8_to_18030 を介して処理する。 4. 変換されたバイト数を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの pg_conversion システムカタログ（SQL経由での呼び出し）。
  • **呼び出し先**: UtfToLocal、conv_utf8_to_18030。
  • **注記**: ソースコード内の PG_FUNCTION_INFO_V1(utf8_to_gb18030) マクロにより、PostgreSQLの関数管理インターフェースに登録されている。

関連データ構造

  • gb18030_from_unicode_tree: UTF-8からGB18030への変換に使用されるツリー構造のマッピングテーブル。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、エラーが発生する。
  • noError フラグの設定により、変換エラー時の挙動（例外発生または無視）が制御される。

保守担当者向けメモ

  • 本関数は基本的な変換をテーブルで行い、4バイト文字などの特殊な範囲を conv_utf8_to_18030 で処理する構成となっている。
  • 変換ロジックの詳細は src/backend/utils/mb/conversion_procs/utf8_and_gb18030/utf8_and_gb18030.c 内の補助関数を参照すること。
