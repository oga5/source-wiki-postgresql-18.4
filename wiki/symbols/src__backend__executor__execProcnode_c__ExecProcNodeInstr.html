ExecProcNodeInstr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execProcnode.c:478

目的

実行ノードの処理時間およびタプル取得状況を計測するためのラッパー関数です。

シグネチャまたは形状

TupleTableSlot *ExecProcNodeInstr(PlanState *node)

入力と出力

  • **入力**: PlanState *node - 実行対象のプランノード状態。
  • **出力**: TupleTableSlot * - ノードから取得されたタプル。

前提条件と不変条件

  • node->instrument が初期化されている必要があります。
  • node->ExecProcNodeReal に有効な実行関数が設定されている必要があります。

アルゴリズムまたは振る舞い

1. InstrStartNode を呼び出し、現在のノードの計測を開始します。 2. node->ExecProcNodeReal(node) を呼び出し、実際のタプル取得処理を実行します。 3. TupIsNull を用いて取得したタプルが空かどうかを判定し、InstrStopNode を呼び出して計測を終了します。この際、タプルが取得できた場合は 1.0、空の場合は 0.0 をカウントします。 4. 取得したタプルを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内で、node->ExecProcNode にこの関数が代入されることで実行されます。
  • **呼び出し先**: InstrStartNode, node->ExecProcNodeReal (関数ポインタ), InstrStopNode, TupIsNull。

関連データ構造

  • PlanState: 実行ノードの状態を保持する構造体。
  • TupleTableSlot: タプルを格納するスロット。
  • Instrumentation: ノードの実行統計情報を保持する構造体。

失敗モードと境界条件

  • node が NULL の場合、または node->instrument が NULL の場合は未定義の動作（セグメンテーション違反）を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は、実行統計の計測が有効な場合に ExecProcNode の代わりとして動的に割り当てられます。
  • 性能に直結するパスであるため、計測ロジックの変更はオーバーヘッドを考慮する必要があります。
