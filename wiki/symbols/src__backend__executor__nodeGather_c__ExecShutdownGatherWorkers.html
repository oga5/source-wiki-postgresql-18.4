ExecShutdownGatherWorkers

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGather.c:399

目的

Gatherノードに関連する並列実行ワーカーのリソースを解放し、リーダープロセスのメモリ状態をクリーンアップする。

シグネチャまたは形状

static void ExecShutdownGatherWorkers(GatherState *node)

入力と出力

  • **入力**: GatherState *node - 終了対象のGatherノードの実行状態構造体。
  • **出力**: なし（副作用としてリソース解放を行う）。

前提条件と不変条件

  • node が NULL でないこと。
  • node->pei が有効な場合、ExecParallelFinish が安全に呼び出せる状態であること。

アルゴリズムまたは振る舞い

1. node->pei が NULL でない場合、ExecParallelFinish(node->pei) を呼び出し、並列実行に関連するワーカーの終了処理を行う。 2. node->reader が存在する場合、pfree を使用してメモリを解放し、ポインタを NULL にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeGather.c 内の ExecEndGather および ExecReScanGather から呼び出されることが確認されている。
  • **呼び出し先**:
  • ExecParallelFinish (並列実行インフラの終了処理)
  • pfree (メモリ解放)

関連データ構造

  • GatherState: 並列実行の進捗やワーカーの状態を保持する構造体。
  • ParallelExecutorInfo (pei): 並列実行のコンテキスト情報。

失敗モードと境界条件

  • node->pei が NULL の場合、ワーカーの終了処理はスキップされる。
  • node->reader が NULL の場合、解放処理はスキップされる。

保守担当者向けメモ

  • 本関数は ExecEndGather（ノード終了時）および ExecReScanGather（再スキャン時）の両方から利用されるため、共有メモリ上のワーカー状態とローカルのリーダー側メモリ状態の整合性を保つ必要がある。
