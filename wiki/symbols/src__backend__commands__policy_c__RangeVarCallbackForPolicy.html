RangeVarCallbackForPolicy

定義

  • 種別: function
  • 定義箇所: src/backend/commands/policy.c:63

目的

ポリシー操作（CREATE POLICY 等）の対象となるリレーションが、操作に適したテーブルであるか、および実行ユーザーがその所有者であるかを検証するコールバック関数です。

シグネチャまたは形状

static void RangeVarCallbackForPolicy(const RangeVar *rv, Oid relid, Oid oldrelid, void *arg);

入力と出力

  • **入力**:
  • rv: 対象リレーションの識別情報（RangeVar）。
  • relid: 対象リレーションのOID。
  • oldrelid: 変更前のリレーションOID（未使用）。
  • arg: 追加の引数（未使用）。
  • **出力**: なし（成功時は正常終了、検証失敗時はエラーをスロー）。

前提条件と不変条件

  • pg_class システムカタログから対象リレーションの情報を取得できること。
  • 実行ユーザーが対象リレーションの所有者であること。
  • 対象リレーションがシステムカタログではないこと（allowSystemTableMods が真の場合を除く）。
  • 対象リレーションが通常のテーブルまたはパーティションテーブルであること。

アルゴリズムまたは振る舞い

1. RELOID システムキャッシュを使用して、relid に対応する pg_class タプルを取得する。 2. object_ownercheck を呼び出し、現在のユーザーが対象リレーションの所有者であることを確認する。所有者でない場合は ACLCHECK_NOT_OWNER エラーを発生させる。 3. IsSystemClass を使用して、対象がシステムカタログであるか判定する。システムカタログであり、かつ allowSystemTableMods が無効な場合はエラーを発生させる。 4. relkind を確認し、対象が RELKIND_RELATION（通常のテーブル）または RELKIND_PARTITIONED_TABLE（パーティションテーブル）のいずれかであることを検証する。それ以外の場合は ERRCODE_WRONG_OBJECT_TYPE エラーを発生させる。 5. 最後に ReleaseSysCache を呼び出し、取得したタプルを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/policy.c 内のポリシー関連コマンド処理（CreatePolicy 等）において、RangeVarGetRelidExtended のコールバックとして指定されている。
  • **呼び出し先**: SearchSysCache1, GETSTRUCT, object_ownercheck, aclcheck_error, get_relkind_objtype, get_rel_relkind, IsSystemClass, ereport, ReleaseSysCache。

関連データ構造

  • RangeVar: リレーションの識別に使用。
  • Form_pg_class: pg_class カタログの構造体。

失敗モードと境界条件

  • **所有権エラー**: ユーザーがテーブルの所有者でない場合、aclcheck_error によりエラーとなる。
  • **システムカタログ保護**: システムカタログに対してポリシーを定義しようとした場合、allowSystemTableMods が無効であればエラーとなる。
  • **型不一致**: ビューやインデックスなど、テーブル以外のリレーションを指定した場合、ERRCODE_WRONG_OBJECT_TYPE でエラーとなる。

保守担当者向けメモ

  • 本関数は RangeVarGetRelidExtended の引数として渡されるコールバック関数です。
  • relkind のチェックは、RLSが適用可能なオブジェクトをテーブルに限定するというPostgreSQLの仕様に基づいています。将来的に他のオブジェクトへのRLS適用が検討される場合は、この条件式を更新する必要があります。
