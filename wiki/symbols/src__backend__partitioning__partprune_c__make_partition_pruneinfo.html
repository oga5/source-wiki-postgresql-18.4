make_partition_pruneinfo

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partprune.c:224

目的

クエリのプランニング段階において、実行時にパーティションプルーニングを行うための情報（PartitionPruneInfo）を構築し、PlannerInfo に登録する。

シグネチャまたは形状

int make_partition_pruneinfo(PlannerInfo *root, RelOptInfo *parentrel, List *subpaths, List *prunequal)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • parentrel: 対象となるパーティション化されたリレーション。
  • subpaths: パーティションの子リレーションに対するスキャンパスのリスト。
  • prunequal: プルーニングに使用する条件式（RestrictInfo リスト）。
  • **出力**:
  • int: root->partPruneInfos リスト内でのインデックス。プルーニング情報が不要な場合は -1 を返す。

前提条件と不変条件

  • subpaths 内のパスは RELOPT_OTHER_MEMBER_REL である必要がある。
  • parentrel はパーティション化されたリレーションであること。
  • relid_subplan_map は root->simple_rel_array_size に基づく一時的なマッピング用配列として使用される。

アルゴリズムまたは振る舞い

1. subpaths を走査し、各パスが属するパーティション階層を特定する。 2. root->append_rel_array を辿り、parentrel に至るまでの階層構造を抽出する。 3. 抽出した階層ごとに make_partitionedrel_pruneinfo を呼び出し、プルーニングステップを生成する。 4. 有効なプルーニング情報が得られた場合、PartitionPruneInfo ノードを作成し、root->partPruneInfos に追加する。 5. プルーニング対象外のサブプラン（other_subplans）を特定し、実行時に誤って除外されないよう記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/createplan.c (複数箇所)
  • **呼び出し先**:
  • make_partitionedrel_pruneinfo
  • add_part_relids
  • bms_add_member, bms_join, bms_copy, bms_add_range, bms_del_members (ビットマップセット操作)
  • palloc0, pfree

関連データ構造

  • PartitionPruneInfo: 実行時に使用されるプルーニング情報のコンテナ。
  • PlannerInfo: プランナの全体状態を保持。
  • RelOptInfo: リレーションのプランニング情報。
  • Bitmapset: 生存するパーティションやサブプランのインデックス管理に使用。

失敗モードと境界条件

  • パーティション階層に有用なプルーニング条件（prunequal）が含まれない場合、prunerelinfos は NIL となり、関数は -1 を返す。
  • subpaths にパーティション化されていないテーブルが含まれる場合、それらは other_subplans として扱われ、プルーニング対象から除外される。

保守担当者向けメモ

  • relid_subplan_map は 1-based インデックスを使用している点に注意。
  • subplan という用語が使用されているが、実際には subpath を指している。
  • 複雑なパーティション階層（多段階パーティション）における parentrel への到達判定ロジックが重要である。
