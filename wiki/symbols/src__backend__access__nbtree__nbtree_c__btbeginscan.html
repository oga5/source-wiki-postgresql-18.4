btbeginscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtree.c:335

目的

B-tree インデックスに対するスキャン操作の初期化を行い、スキャン状態を管理するための構造体を割り当てて設定する。

シグネチャまたは形状

IndexScanDesc btbeginscan(Relation rel, int nkeys, int norderbys)

入力と出力

  • **入力**:
  • rel: スキャン対象のインデックスリレーション。
  • nkeys: スキャンで使用するキーの数。
  • norderbys: ORDER BY 句の数。
  • **出力**:
  • 初期化済みの IndexScanDesc 構造体。

前提条件と不変条件

  • norderbys は常に 0 であること（B-tree インデックスでは ORDER BY 演算子はサポートされないため）。

アルゴリズムまたは振る舞い

1. RelationGetIndexScan を呼び出し、汎用的な IndexScanDesc を取得する。 2. BTScanOpaqueData 構造体をメモリ上に割り当て、スキャン状態を初期化する。

  • currPos および markPos を無効化する。
  • nkeys > 0 の場合、keyData 用のメモリを割り当てる。
  • スキャンフラグ（skipScan, needPrimScan 等）をデフォルト値（false）に設定する。
  • killedItems を NULL に初期化する。

    • 3. インデックスのタプル記述子（xs_itupdesc）をリレーションから取得して設定する。 4. scan->opaque に作成した BTScanOpaque を格納し、スキャン記述子を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: bthandler を介してインデックスアクセスメソッドのインターフェースとして呼び出される（詳細はソースコード上の直接的な呼び出し元情報が不足しているため、これ以上の特定は不可）。
  • **呼び出し先**: RelationGetIndexScan、palloc、BTScanPosInvalidate、RelationGetDescr。

関連データ構造

  • IndexScanDesc: スキャンの汎用的な状態を保持する構造体。
  • BTScanOpaqueData: B-tree インデックススキャン固有の状態（キー、位置情報、削除済みアイテム等）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性がある。
  • norderbys != 0 の場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数はスキャンの初期化のみを行い、タプルワークスペース配列の割り当ては btrescan に委譲されている。
  • BTScanOpaque の各フィールドの初期化順序やメモリ割り当ての有無は、将来的なスキャン最適化の変更に影響を受ける可能性がある。
