compute_code_target

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/io.c:222

目的

コード行の適切なインデント開始カラム位置を計算し、戻り値として返す。

シグネチャまたは形状

int compute_code_target(void)

入力と出力

  • 入力: グローバル状態 ps（parser_state）、lineup_to_parens、lineup_to_parens_always、continuation_indent、paren_target、max_col、およびバッファ s_code。
  • 出力: 計算されたインデント位置を示す整数値（カラム数）。

前提条件と不変条件

  • indent_globs.h で定義されたグローバル状態が初期化されていること。
  • s_code バッファが現在のコード行を保持していること。

アルゴリズムまたは振る舞い

1. 基本インデントとして ps.ind_size * ps.ind_level + 1 を算出する。 2. 括弧内（ps.paren_level が真）の場合:

  • lineup_to_parens が偽であれば、continuation_indent を加算する。
  • lineup_to_parens_always が真であれば、paren_target を採用する。
  • それ以外の場合、paren_target を基準に max_col を考慮した調整を行い、必要に応じてインデントを決定する。

    • 3. 括弧内でなく、文の継続（ps.ind_stmt）がある場合は、基本インデントに continuation_indent を加算する。 4. 最終的な target_col を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/tools/pg_bsd_indent/indent.c:1034
  • src/tools/pg_bsd_indent/io.c:144
  • src/tools/pg_bsd_indent/pr_comment.c:133
  • 呼び出し先:

関連データ構造

  • parser_state (ps): インデントレベルや括弧の深さを保持するグローバル構造体。

失敗モードと境界条件

  • max_col を超えるインデントが発生する場合、count_spaces の結果に基づき target_col が調整される。
  • 括弧の深さや継続行のフラグ設定により、インデント値が動的に変化する。

保守担当者向けメモ

  • 本関数は indent_globs.h で定義されたグローバル変数に強く依存している。
  • lineup_to_parens 関連のロジックは、コードの整形スタイルを決定する重要な分岐点であるため、変更時は影響範囲に注意すること。
