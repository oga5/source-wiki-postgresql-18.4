box_out

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_ops.c:454

目的

BOX 型のデータを、外部表現（文字列形式）に変換します。

シグネチャまたは形状

Datum box_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される BOX 型のポインタ。
  • **出力**: Datum 型として返される、ボックスの座標を文字列化した cstring。

前提条件と不変条件

  • 入力された BOX ポインタは有効である必要があります。
  • 内部的に path_encode を利用するため、ボックスの座標データが path_encode の期待する形式（high 座標を起点とした構造）と整合している必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_BOX_P(0) を使用して、引数から BOX 構造体へのポインタを取得します。 2. path_encode 関数を呼び出し、PATH_NONE モードおよび 2 点の座標指定でボックスの座標を文字列化します。 3. 生成された文字列を PG_RETURN_CSTRING を介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: 不明（PostgreSQLの関数マネージャ経由でSQLから呼び出されることが想定されます）。
  • **呼び出し先**: path_encode（src/backend/utils/adt/geo_ops.c 内のユーティリティ関数）。

関連データ構造

  • BOX: src/include/utils/geo_decls.h で定義される、ボックスの座標を保持する構造体。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • メモリ不足時や、path_encode 内部での文字列生成失敗時にエラーが発生する可能性があります。
  • 座標値が float8 の範囲外である場合、path_encode の挙動に依存します。

保守担当者向けメモ

  • 本関数は box 型の出力用インターフェースです。座標のフォーマット変更が必要な場合は、path_encode の仕様を確認してください。
  • 呼び出し元情報の詳細は、PostgreSQLのシステムカタログ（pg_proc）を参照することで特定可能です。
