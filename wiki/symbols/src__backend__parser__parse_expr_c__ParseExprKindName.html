ParseExprKindName

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/parser/parse_expr.c:3142

目的

ParseExprKind 列挙型に対応する、人間が読み取り可能な文字列（エラーメッセージ等で使用される識別名）を返します。

シグネチャまたは形状

const char *ParseExprKindName(ParseExprKind exprKind)

入力と出力

  • **入力**: ParseExprKind 型の列挙値（式が使用されるコンテキストを示す）。
  • **出力**: コンテキスト名を表す const char * 型の文字列。

前提条件と不変条件

  • switch 文は網羅的であることを意図しており、新しい ParseExprKind が追加された際にコンパイラが警告を発するよう default ケースを意図的に設けていません。

アルゴリズムまたは振る舞い

1. 引数 exprKind を switch 文で評価します。 2. 各 ParseExprKind 値に対応する定数文字列を返します（例: EXPR_KIND_WHERE に対して "WHERE"）。 3. switch 文で一致しない値が渡された場合、関数末尾の "unrecognized expression kind" を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_agg.c
  • src/backend/parser/parse_clause.c
  • src/backend/parser/parse_func.c
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • ParseExprKind（列挙型）: 式がどの構文コンテキスト（WHERE句、SELECTリスト等）にあるかを識別するために使用されます。

失敗モードと境界条件

  • 定義されていない ParseExprKind 値が渡された場合、実行時に "unrecognized expression kind" が返されます。

保守担当者向けメモ

  • 新しい ParseExprKind を追加した際は、必ず本関数の switch 文にもケースを追加してください。追加を忘れるとコンパイラが警告を発しますが、実行時にはフォールスルーして「認識されない」文字列が返されます。
