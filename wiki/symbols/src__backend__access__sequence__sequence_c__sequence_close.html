sequence_close

定義

  • 種別: function
  • 定義箇所: src/backend/access/sequence/sequence.c:57

目的

シーケンスリレーションを閉じ、指定されたロックモードに従ってリソースを解放するためのラッパー関数です。

シグネチャまたは形状

void sequence_close(Relation relation, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relation: 閉じる対象の Relation 構造体ポインタ。
  • lockmode: 解放時に適用する LOCKMODE。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 relation は、有効なシーケンスリレーションを指している必要があります。
  • 本関数は relation_close のラッパーであり、内部的な状態整合性は relation_close の仕様に依存します。

アルゴリズムまたは振る舞い

1. 渡された relation と lockmode をそのまま relation_close 関数に引き渡します。 2. relation_close を実行し、リレーションのクローズおよびロックの解放処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/sequence.c 内の各シーケンス操作関数（nextval や setval 等の実行に関連する箇所で複数回呼び出されています）。
  • **呼び出し先**:
  • relation_close（PostgreSQLの汎用リレーション操作関数）。

関連データ構造

  • Relation: シーケンスリレーションを表現する構造体。
  • LOCKMODE: ロックの挙動を制御する列挙型。

失敗モードと境界条件

  • relation が NULL の場合や、既に閉じられたリレーションに対して呼び出された場合の挙動は relation_close の実装に準拠します。

保守担当者向けメモ

  • 本関数は非常に軽量なラッパーです。シーケンス特有の追加処理が必要になった場合を除き、基本的には relation_close への直接的な委譲を維持してください。
  • シーケンスに対するロックはトランザクション終了まで保持されることが一般的なため、呼び出し側で lockmode に NoLock を指定するケースが多く見られます。
