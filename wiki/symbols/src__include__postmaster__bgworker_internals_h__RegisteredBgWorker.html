RegisteredBgWorker

定義

  • 種別: struct
  • 定義箇所: src/include/postmaster/bgworker_internals.h:32

目的

Postmasterプロセスがバックグラウンドワーカーのライフサイクル（登録、実行状態、監視、終了）を管理するための内部的なデータ構造です。

シグネチャまたは形状

typedef struct RegisteredBgWorker
{
	BackgroundWorker rw_worker; /* 登録情報 */
	pid_t		rw_pid;			/* 実行中のPID（未実行時は0） */
	TimestampTz rw_crashed_at;	/* 最後にクラッシュした時刻（未クラッシュ時は0） */
	int			rw_shmem_slot;	/* 共有メモリ上のスロット位置 */
	bool		rw_terminate;	/* 終了フラグ */
	dlist_node	rw_lnode;		/* リスト管理用リンク */
} RegisteredBgWorker;

入力と出力

本構造体は関数ではなくデータ型です。Postmasterのメモリ空間内でワーカーの状態を保持し、BackgroundWorkerList（二重連結リスト）の要素として管理されます。

前提条件と不変条件

  • Postmasterプロセスのメモリ空間内でのみ有効です。
  • ワーカープロセス側から直接操作してはなりません。
  • rw_pid が 0 の場合、そのワーカーは現在実行中ではないことを示します。

アルゴリズムまたは振る舞い

  • **状態追跡**: rw_pid によりプロセスの生存状況を、rw_crashed_at によりクラッシュ履歴を保持します。
  • **リスト管理**: dlist_node を介して BackgroundWorkerList に組み込まれ、Postmasterによる一括管理を可能にします。
  • **終了制御**: rw_terminate フラグが設定されることで、Postmasterに対して当該ワーカーの終了処理を促します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/bgworker.c 内の各管理関数（ForgetBackgroundWorker, ReportBackgroundWorkerPID, ReportBackgroundWorkerExit など）が本構造体を引数として受け取ります。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • BackgroundWorker: 公開APIで定義されるワーカーの基本設定情報。
  • dlist_node: lib/ilist.h で定義される二重連結リストのノード。
  • BackgroundWorkerList: 本構造体を要素として保持するリスト。

失敗モードと境界条件

  • rw_pid が 0 の状態から起動に失敗した場合、rw_crashed_at が更新されます。
  • 同時に存在可能なワーカー数には MAX_PARALLEL_WORKER_LIMIT（1024）の上限があります。

保守担当者向けメモ

  • 本構造体はPostmaster内部専用です。拡張モジュール開発者は bgworker.h の公開APIを使用してください。
  • rw_shmem_slot は共有メモリとのマッピングに使用されるため、変更時は関連する共有メモリ管理ロジックとの整合性に注意してください。
