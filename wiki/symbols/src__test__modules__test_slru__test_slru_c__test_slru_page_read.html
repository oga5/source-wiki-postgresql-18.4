test_slru_page_read

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_slru.c:91

目的

SQLインターフェースを通じて、テスト用SLRU（Simple Least Recently Used）キャッシュから指定されたページのデータを読み取り、テキストとして返す。

シグネチャまたは形状

Datum test_slru_page_read(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • pageno (int64): 読み取るページ番号（第1引数）。
  • write_ok (bool): 読み取り時に書き込み許可を伴うかを示すフラグ（第2引数）。
  • **出力**:
  • text: 読み取ったページの内容を変換したテキストデータ。

前提条件と不変条件

  • 本関数は shared_preload_libraries でロードされた環境下で実行される必要がある。
  • TestSlruCtl が初期化済みであること。
  • 指定されたページ番号に対応するロック（SimpleLruGetBankLock）が取得可能であること。

アルゴリズムまたは振る舞い

1. SimpleLruGetBankLock を使用して、対象ページに対応する LWLock を取得する。 2. LWLockAcquire で排他ロック（LW_EXCLUSIVE）を確保する。 3. SimpleLruReadPage を呼び出し、ページをバッファに読み込む（必要に応じてディスクからロード）。 4. TestSlruCtl の共有バッファから該当スロットのデータを取得する。 5. LWLockRelease でロックを解放する。 6. 取得したデータを cstring_to_text で変換し、結果として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • SimpleLruGetBankLock
  • LWLockAcquire
  • SimpleLruReadPage
  • LWLockRelease
  • cstring_to_text

関連データ構造

  • TestSlruCtl: テスト用SLRUを制御する SlruCtlData 構造体。
  • SlruShared: TestSlruCtl 内の共有メモリ領域。

失敗モードと境界条件

  • ページ番号が不正な場合や、SLRUの初期化が完了していない場合はエラーが発生する可能性がある。
  • ロック取得の競合により、他のプロセスが対象ページを操作している間は待機が発生する。

保守担当者向けメモ

  • 本関数は SimpleLruReadPage をラップしており、SLRUの基本的な読み取りロジックをテストするために使用される。
  • write_ok 引数の扱いは SimpleLruReadPage の仕様に依存するため、変更時は src/backend/access/transam/slru.c の実装を参照すること。
