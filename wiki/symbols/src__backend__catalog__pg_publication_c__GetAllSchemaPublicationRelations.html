GetAllSchemaPublicationRelations

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_publication.c:1045

目的

指定されたパブリケーションに含まれるすべてのスキーマを特定し、それらのスキーマに属するリレーション（テーブル）のリストを取得する。

シグネチャまたは形状

List *GetAllSchemaPublicationRelations(Oid pubid, PublicationPartOpt pub_partopt)

入力と出力

  • **入力**:
  • pubid (Oid): 対象となるパブリケーションのOID。
  • pub_partopt (PublicationPartOpt): パーティションテーブルの扱いを制御するオプション。
  • **出力**:
  • List *: 該当するリレーションのOIDリスト。

前提条件と不変条件

  • pubid は有効なパブリケーションを参照している必要がある。
  • GetPublicationSchemas および GetSchemaPublicationRelations が正常に動作する環境であること。

アルゴリズムまたは振る舞い

1. GetPublicationSchemas(pubid) を呼び出し、パブリケーションに関連付けられたスキーマのリストを取得する。 2. 取得したスキーマリストをループで走査する。 3. 各スキーマに対し GetSchemaPublicationRelations(schemaid, pub_partopt) を呼び出し、そのスキーマ内のリレーションリストを取得する。 4. 各スキーマから得られたリレーションリストを list_concat で結合し、最終的なリストとして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c (複数箇所)
  • src/backend/commands/publicationcmds.c (複数箇所)
  • **呼び出し先**:
  • GetPublicationSchemas
  • GetSchemaPublicationRelations
  • list_concat

関連データ構造

  • List: PostgreSQLのリスト構造。
  • Oid: オブジェクト識別子。
  • PublicationPartOpt: パーティション処理のオプション定義。

失敗モードと境界条件

  • 指定された pubid にスキーマが一つも関連付けられていない場合、空のリスト (NIL) を返す。
  • 関連付けられたスキーマ内にリレーションが存在しない場合、そのスキーマ分は結果に含まれない。

保守担当者向けメモ

  • 本関数はスキーマ単位のパブリケーション解決を行うため、pg_publication_namespace カタログの変更に依存する。
  • パーティションテーブルの階層構造を考慮する必要がある場合は、pub_partopt の値が適切に伝播されていることを確認すること。
