prssyntaxerror

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_parser.c:141

目的

tsvector または tsquery の解析中に発生した構文エラーを報告し、エラーコンテキストを適切に処理する。

シグネチャまたは形状

static bool prssyntaxerror(TSVectorParseState state)

入力と出力

  • **入力**: TSVectorParseState 型の解析状態オブジェクト。
  • **出力**: 常に false を返す。これは呼び出し元がエラー発生時に処理を中断する際の便宜的な戻り値として利用される。

前提条件と不変条件

  • state 引数は有効な TSVectorParseState オブジェクトを指している必要がある。
  • state->escontext が初期化されており、エラー報告を受け取れる状態であること。

アルゴリズムまたは振る舞い

1. state->is_tsquery フラグを確認する。 2. フラグに応じて、tsquery または tsvector のいずれかに対する構文エラーメッセージを構築する。 3. errsave を呼び出し、state->escontext を通じてエラーを報告する。 4. 呼び出し元への便宜として false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsvector_parser.c 内の PRSSYNTAXERROR マクロ経由で呼び出される。
  • contrib/hstore/hstore_io.c 内の PRSSYNTAXERROR マクロ経由で呼び出される。
  • **呼び出し先**:
  • errsave (PostgreSQL のエラーハンドリング機構)

関連データ構造

  • TSVectorParseStateData: 解析状態を保持する構造体。本関数はこの構造体内の escontext（エラーコンテキスト）、is_tsquery（型フラグ）、bufstart（エラー対象の文字列）を参照する。

失敗モードと境界条件

  • state->escontext が NULL の場合、errsave の挙動は PostgreSQL のエラーハンドリング規約に従う。
  • state->bufstart が不正なメモリを指している場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は tsvector と tsquery の両方で共有されているため、エラーメッセージの切り替えロジックを保持している。
  • PRSSYNTAXERROR マクロを介して呼び出すことが推奨される。
  • contrib/hstore からも参照されているため、修正時には他モジュールへの影響を考慮すること。
