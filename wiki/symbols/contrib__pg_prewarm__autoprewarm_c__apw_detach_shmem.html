apw_detach_shmem

定義

  • 種別: function
  • 定義箇所: contrib/pg_prewarm/autoprewarm.c:894

目的

共有メモリ上の AutoPrewarmSharedState におけるプロセス固有の識別子（PID）をクリアし、リソースの解放や終了処理に伴う状態の整合性を維持する。

シグネチャまたは形状

static void apw_detach_shmem(int code, Datum arg)

入力と出力

  • **入力**:
  • code: before_shmem_exit コールバック等から渡される終了コード。
  • arg: 補助的な引数（現在は使用されていない）。
  • **出力**: なし（副作用として共有メモリの状態を更新）。

前提条件と不変条件

  • apw_state が初期化済みであること。
  • 共有メモリ上の apw_state->lock を使用して排他制御を行うため、呼び出し時にデッドロックが発生しない状態であること。

アルゴリズムまたは振る舞い

1. apw_state->lock を LW_EXCLUSIVE モードで取得する。 2. 現在のプロセス（MyProcPid）が pid_using_dumpfile に登録されている場合、これを InvalidPid にリセットする。 3. 現在のプロセスが bgworker_pid に登録されている場合、これを InvalidPid にリセットする。 4. LWLockRelease を呼び出し、ロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • before_shmem_exit への登録（autoprewarm.c:190）。
  • PG_ENSURE_ERROR_CLEANUP および PG_END_ENSURE_ERROR_CLEANUP マクロによるエラーハンドリング時（autoprewarm.c:853, 857）。
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease（PostgreSQL 共有メモリロック API）。

関連データ構造

  • AutoPrewarmSharedState: 共有メモリ上の制御構造体。本関数はこの構造体内の pid_using_dumpfile および bgworker_pid を管理する。

失敗モードと境界条件

  • ロック取得時に他のプロセスが既にロックを保持している場合、取得できるまで待機する。
  • 自身の PID がいずれのフィールドにも一致しない場合、何も変更せずロックを解放する。

保守担当者向けメモ

  • 本関数は shmem_exit 時のクリーンアップルーチンとして設計されている。
  • PG_ENSURE_ERROR_CLEANUP で使用される際は、エラー発生時に共有メモリの状態が不正なまま残らないようにする役割を果たす。
