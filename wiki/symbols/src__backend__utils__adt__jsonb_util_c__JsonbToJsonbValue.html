JsonbToJsonbValue

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_util.c:71

目的

Jsonb 型のバイナリデータを、メモリ上で操作可能な JsonbValue 構造体にラップします。

シグネチャまたは形状

void JsonbToJsonbValue(Jsonb *jsonb, JsonbValue *val)

入力と出力

  • **入力**:
  • jsonb: 変換対象となる Jsonb 構造体へのポインタ。
  • val: 変換結果を格納する JsonbValue 構造体へのポインタ。
  • **出力**: なし（val が指すメモリ領域を直接更新します）。

前提条件と不変条件

  • jsonb は有効な Jsonb 構造体である必要があります。
  • val は書き込み可能なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. val->type を jbvBinary に設定します。 2. val->val.binary.data に jsonb->root のアドレスを代入します。 3. val->val.binary.len に、jsonb の全サイズからヘッダサイズ（VARHDRSZ）を引いた値を代入します。 この処理により、JsonbValue は元の Jsonb バイナリを直接参照する状態になります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonbsubs.c:273
  • src/backend/utils/adt/jsonfuncs.c:4882
  • src/backend/utils/adt/jsonfuncs.c:5041
  • **呼び出し先**:
  • VARSIZE マクロ（postgres.h 由来）

関連データ構造

  • Jsonb: ディスク上の jsonb バイナリ表現。
  • JsonbValue: メモリ上での jsonb 操作用構造体。

失敗モードと境界条件

  • jsonb が NULL の場合、ポインタ参照によりクラッシュする可能性があります。
  • VARSIZE マクロが正しく機能しないような不正なメモリレイアウトの jsonb が渡された場合、len が不正になります。

保守担当者向けメモ

  • 本関数はデータのコピーを行わず、元の Jsonb へのポインタを保持するだけです。そのため、JsonbValue を使用する間は、元の Jsonb オブジェクトがメモリ上で生存している必要があります。
