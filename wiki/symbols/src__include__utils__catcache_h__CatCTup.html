CatCTup

定義

  • 種別: struct
  • 定義箇所: src/include/utils/catcache.h:88

目的

システムカタログのキャッシュエントリを保持する構造体。特定のシステムテーブル行（正のキャッシュ）または、特定のキーに対応する行が存在しないという事実（負のキャッシュ）をメモリ上に保持し、カタログ検索のオーバーヘッドを削減する。

シグネチャまたは形状

typedef struct catctup {
    int             ct_magic;
    uint32          hash_value;
    Datum           keys[CATCACHE_MAXKEYS];
    dlist_node      cache_elem;
    int             refcount;
    bool            dead;
    bool            negative;
    HeapTupleData   tuple;
    struct catclist *c_list;
    CatCache        *my_cache;
} CatCTup;

入力と出力

  • **入力**: キャッシュ検索時に生成されるハッシュ値、検索キー、およびシステムテーブルから取得した HeapTupleData。
  • **出力**: キャッシュヒット時に SearchCatCache 等を通じて呼び出し元へ返される CatCTup へのポインタ（参照カウントがインクリメントされる）。

前提条件と不変条件

  • ct_magic は 0x57261502 であること。
  • dead フラグが立っている場合、そのエントリは検索結果として返してはならない。
  • refcount が 0 になるまで、メモリからの物理的な削除は行われない。
  • 負のキャッシュエントリ（negative == true）の場合、tuple 以外の管理ロジックは正のキャッシュと同一である。

アルゴリズムまたは振る舞い

  • **ハッシュバケット管理**: cache_elem を用いてハッシュバケット内の双方向リストに格納され、LRU順で管理される。
  • **ライフサイクル**: システムテーブルの更新に伴い CatCacheInvalidate が呼ばれると dead マークが付与される。参照カウントが 0 に達した時点で物理的に解放される。
  • **リスト管理**: 最大で一つの CatCList に所属可能。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/catcache.c 内のキャッシュ検索および管理関数群（CatalogCacheCreateEntry, CatCacheRemoveCTup 等）。
  • **呼び出し先**: 構造体内部で CatCache および CatCList へのポインタを保持する。

関連データ構造

  • CatCache: このエントリを所有するキャッシュ全体。
  • CatCList: 部分キー検索の結果を保持するリスト（c_list フィールドで参照）。
  • HeapTupleData: キャッシュされた実際のタプルデータ。
  • dlist_node: ハッシュバケット管理用のリストノード。

失敗モードと境界条件

  • **メモリリーク**: ReleaseCatCache による参照カウントのデクリメントを怠ると、dead フラグが立ってもメモリから解放されず、キャッシュが肥大化する。
  • **整合性**: CatCList に所属するタプルを無効化する場合、対応するリスト自体も同時に dead マークを付与する必要がある。

保守担当者向けメモ

  • 本構造体は CacheMemoryContext 上で管理される。
  • 負のキャッシュエントリは、存在しないキーに対する検索を高速化する重要な役割を持つため、正のキャッシュと同様のライフサイクル管理が必要である。
  • CatCList との関連付けは、同一タプルに対して複数のキー数で検索が行われる場合に複雑化する可能性がある。
