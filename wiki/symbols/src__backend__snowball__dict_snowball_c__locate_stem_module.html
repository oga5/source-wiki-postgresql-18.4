locate_stem_module

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/dict_snowball.c:183

目的

指定された言語名に基づき、適切なSnowballステミングアルゴリズムを検索し、辞書インスタンス（DictSnowball）を初期化する。

シグネチャまたは形状

static void locate_stem_module(DictSnowball *d, const char *lang)

入力と出力

  • **入力**:
  • d: 初期化対象の DictSnowball 構造体へのポインタ。
  • lang: 検索対象の言語名（文字列）。
  • **出力**: なし（成功時は d のメンバを更新して終了）。

前提条件と不変条件

  • stemmer_modules 配列が有効なステマー定義を保持していること。
  • GetDatabaseEncoding() および GetDatabaseEncodingName() が現在のデータベース環境を正しく返すこと。

アルゴリズムまたは振る舞い

1. **完全一致検索**: stemmer_modules を走査し、現在のデータベースエンコーディング（または PG_SQL_ASCII）と一致し、かつ言語名が一致するステマーを探す。見つかれば d->stem と d->z を設定し、needrecode を false にして終了する。 2. **UTF-8フォールバック検索**: 1で見つからない場合、PG_UTF8 エンコーディングのステマーを検索する。見つかれば d->stem と d->z を設定し、needrecode を true にして終了する。 3. **エラー報告**: いずれも該当しない場合、ERRCODE_UNDEFINED_OBJECT でエラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: dsnowball_init（src/backend/snowball/dict_snowball.c:253）
  • **呼び出し先**:
  • pg_strcasecmp（言語名の比較）
  • m->create()（ステマーインスタンスの生成）
  • ereport（エラー発生時）

関連データ構造

  • DictSnowball: 辞書の状態を保持する構造体。
  • stemmer_module: サポートされる言語とエンコーディングの対応表。

失敗モードと境界条件

  • 指定された言語に対応するステマーが存在しない場合、ereport により処理が中断される。
  • PG_SQL_ASCII は任意のエンコーディングと互換性があるとみなされる。

保守担当者向けメモ

  • 新しい言語を追加する場合は、stemmer_modules 配列に定義を追加する必要がある。
  • needrecode フラグは、後のステミング実行時にエンコーディング変換が必要かどうかを判断するために使用される。
