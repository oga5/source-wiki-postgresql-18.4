numeric_trim_scale

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numeric.c:4325

目的

NUMERIC 値から末尾の不要なゼロを取り除き、スケール（小数点以下の桁数）を最小限に調整した新しい NUMERIC 値を生成します。

シグネチャまたは形状

Datum numeric_trim_scale(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して NUMERIC 型のデータを受け取ります。
  • **出力**: スケールが調整された NUMERIC 型の Datum を返します。

前提条件と不変条件

  • 入力値が NaN や Infinity などの特殊な値（NUMERIC_IS_SPECIAL）である場合、その値を複製してそのまま返します。
  • 演算中、NumericVar 構造体に対して init_var_from_num で初期化を行い、終了時には free_var を呼び出してメモリを解放する必要があります。

アルゴリズムまたは振る舞い

1. 入力値が特殊な値（NaN 等）か判定し、該当する場合は複製して即座に返します。 2. 入力値を NumericVar 構造体に展開します。 3. get_min_scale を呼び出し、値の有効桁数を維持できる最小のスケールを計算して result.dscale に設定します。 4. make_result を使用して、調整された NumericVar を Numeric 型のオンディスク形式に変換します。 5. 作業用メモリを解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（具体的な SQL レベルの呼び出し元はソースコード上では明示されていません）。
  • **呼び出し先**:
  • init_var_from_num
  • get_min_scale
  • make_result
  • free_var
  • duplicate_numeric

関連データ構造

  • Numeric: NUMERIC 型の内部表現。
  • NumericVar: 算術演算用の中間作業構造体。

失敗モードと境界条件

  • 入力値が NULL の場合、PG_GETARG_NUMERIC マクロの仕様に従い、呼び出し元で適切に処理される必要があります。
  • メモリ不足時には palloc 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は NUMERIC 型のスケール調整を行う標準的なユーティリティです。
  • get_min_scale の実装変更が、本関数の出力するスケール決定ロジックに直接影響します。
