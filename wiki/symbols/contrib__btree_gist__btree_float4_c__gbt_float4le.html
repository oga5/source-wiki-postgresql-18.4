gbt_float4le

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float4.c:43

目的

2つの float4 値を比較し、第一引数が第二引数以下であるかどうかを判定する。

シグネチャまたは形状

bool gbt_float4le(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の float4 値へのポインタ。
  • b: 比較対象の float4 値へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • bool: *a <= *b が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • a および b は、有効な float4 型の値を指している必要がある。
  • flinfo は NULL でも動作する（関数内で参照されていないため）。

アルゴリズムまたは振る舞い

1. a および b を float4 型のポインタとしてキャストする。 2. それぞれの値をデリファレンスし、C言語の <= 演算子を用いて比較を行う。 3. 比較結果をブール値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_float4.c 内の静的テーブル定義（tinfo 構造体の初期化時）において参照されている。
  • **呼び出し先**: 直接的な関数呼び出しはない。

関連データ構造

  • float4: PostgreSQLの単精度浮動小数点数型。

失敗モードと境界条件

  • a または b が NULL ポインタである場合、セグメンテーション違反が発生する可能性がある。
  • NaN（非数）が含まれる場合の比較動作は、標準的な浮動小数点数の比較規則に従う。

保守担当者向けメモ

  • 本関数は btree_gist における float4 型の比較サポート関数の一つである。
  • btree_utils_num.c で定義される汎用的な数値処理ロジックから利用されることを想定している。
