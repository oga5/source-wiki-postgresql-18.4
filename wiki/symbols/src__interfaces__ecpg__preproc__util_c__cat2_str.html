cat2_str

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:204

目的

2つの文字列をスペース区切りで連結し、一時メモリ領域に格納された新しい文字列を返します。

シグネチャまたは形状

char *cat2_str(const char *str1, const char *str2)

入力と出力

  • **入力**: 連結対象の2つの文字列 str1, str2。
  • **出力**: 連結後の文字列を指すポインタ。メモリは loc_alloc によって確保されます。

前提条件と不変条件

  • 戻り値の文字列は loc_alloc を使用して確保されるため、呼び出し側で free を行う必要はありません。
  • 解析サイクル終了時に reclaim_local_storage() が呼び出されることで、確保されたメモリが解放されることが前提となります。

アルゴリズムまたは振る舞い

1. str1 と str2 の長さの合計に、スペースと終端文字分を加えたサイズを loc_alloc で確保します。 2. str1 をコピーします。 3. str1 と str2 の双方が空文字列でない場合、間にスペース " " を挿入します。 4. str2 を連結します。 5. 生成された文字列のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し先**: loc_alloc（メモリ確保）、strlen, strcpy, strcat（標準Cライブラリ）。
  • **呼び出し元**: src/interfaces/ecpg/preproc/util.c 内の他の文字列操作関数（cat_str 等）から呼び出されていることが確認されています。

関連データ構造

  • loc_chunk: loc_alloc が使用する一時メモリ管理用のチャンク構造。

失敗モードと境界条件

  • loc_alloc がメモリ確保に失敗した場合、内部的にエラー処理（プロセス終了等）が行われます。
  • 入力文字列が空の場合、スペースは挿入されません。

保守担当者向けメモ

  • 本関数が返すメモリは reclaim_local_storage() で一括解放されるため、長期的な保持が必要な文字列には使用しないでください。
  • 非常に大きな文字列を連結する場合、loc_chunk のデフォルトサイズ（8192バイト）を超える可能性があるため注意が必要です。
