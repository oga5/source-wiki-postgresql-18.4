pg_logging_set_level

定義

  • 種別: function
  • 定義箇所: src/common/logging.c:175

目的

ログ出力フレームワークにおける現在のログ出力閾値を動的に変更します。

シグネチャまたは形状

void pg_logging_set_level(enum pg_log_level new_level)

入力と出力

  • **入力**: new_level (ログレベルを指定する pg_log_level 列挙型)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • 本関数はフロントエンドプログラムでの使用を想定しています。
  • 設定された new_level は、以降の pg_log_generic_v 呼び出しにおけるフィルタリング判定に使用されます。

アルゴリズムまたは振る舞い

1. 引数として渡された new_level を、モジュール内の静的変数 __pg_log_level に代入します。 2. これにより、以降のログ出力処理において、指定されたレベル未満の重要度を持つメッセージは破棄されるようになります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_createsubscriber.c:2109
  • src/bin/pg_dump/pg_dump.c:539
  • src/bin/pg_dump/pg_dumpall.c:213
  • src/bin/pg_dump/pg_restore.c:150
  • **呼び出し先**:
  • 直接的な関数呼び出し先はありません（変数への代入のみ）。

関連データ構造

  • enum pg_log_level: ログの重要度を定義する列挙型。
  • __pg_log_level: ログシステムの現在の閾値を保持する内部静的変数。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。無効な列挙値が渡された場合の挙動は、呼び出し側の型安全性に依存します。

保守担当者向けメモ

  • 本関数は __pg_log_level を直接更新するため、マルチスレッド環境でログレベルを動的に変更する場合には競合に注意が必要です（ただし、現在のPostgreSQLフロントエンドは主にシングルスレッドで動作します）。
