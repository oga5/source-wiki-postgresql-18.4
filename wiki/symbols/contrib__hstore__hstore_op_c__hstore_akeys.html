hstore_akeys

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_op.c:691

目的

hstore 型のデータからすべてのキーを抽出し、text 型の配列として返します。

シグネチャまたは形状

Datum hstore_akeys(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_HSTORE_P(0) により、操作対象の HStore 構造体を受け取ります。
  • **出力**: text 型の PostgreSQL 配列（ArrayType *）を Datum として返します。

前提条件と不変条件

  • 入力される HStore は、hstore.h で定義された内部レイアウト（ヘッダー、エントリ配列、文字列領域）に従っている必要があります。
  • HStore 内部のキーはソート済みであることが前提です。

アルゴリズムまたは振る舞い

1. HS_COUNT マクロを使用して hstore 内の要素数を取得します。 2. 要素数が 0 の場合、construct_empty_array(TEXTOID) を呼び出し、空の text 配列を返します。 3. 要素が存在する場合、要素数分の Datum 配列を palloc で確保します。 4. 各キーに対して HSTORE_KEY および HSTORE_KEYLEN マクロを用いて文字列領域からキーを抽出し、cstring_to_text_with_len で text 型に変換して Datum 配列に格納します。 5. 最後に construct_array_builtin を使用して、収集した Datum 配列から text 型の PostgreSQL 配列を構築して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: HSTORE_POLLUTE(hstore_akeys, akeys) マクロにより akeys 関数として公開されています。
  • **呼び出し先**: PG_GETARG_HSTORE_P, ARRPTR, STRPTR, HS_COUNT, construct_empty_array, palloc, cstring_to_text_with_len, HSTORE_KEY, HSTORE_KEYLEN, PointerGetDatum, construct_array_builtin, PG_RETURN_POINTER などのマクロおよび関数を呼び出します。

関連データ構造

  • HStore: 操作対象のデータ構造。
  • HEntry: hstore 内の各エントリのメタデータ。
  • text: 出力配列の要素型。

失敗モードと境界条件

  • count == 0 の場合、空の配列を正しく生成して終了します。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は hstore の内部構造（ARRPTR, STRPTR）に直接依存しています。hstore のバイナリレイアウトに変更を加える際は、本関数のキー抽出ロジックも確認してください。
  • PG_FUNCTION_INFO_V1(hstore_akeys) により、PostgreSQL の関数呼び出し規約に準拠しています。
