gather_merge_setup

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGatherMerge.c:394

目的

GatherMerge ノードの実行に必要なメモリ領域の確保およびデータ構造の初期化を行う。

シグネチャまたは形状

static void gather_merge_setup(GatherMergeState *gm_state)

入力と出力

  • **入力**: gm_state (GatherMergeState *) - 実行状態を保持する構造体。
  • **出力**: なし（副作用として gm_state 内の各フィールドを初期化する）。

前提条件と不変条件

  • gm_state->ps.plan が GatherMerge ノードとしてキャスト可能であること。
  • gm_state->ps.state および gm_state->tupDesc が初期化済みであること。
  • スロット 0 は常にリーダープロセス用として確保される。

アルゴリズムまたは振る舞い

1. GatherMerge プランからワーカー数 (num_workers) を取得する。 2. gm_slots を (ワーカー数 + 1) 分確保する。 3. 各ワーカー用に GMReaderTupleBuffer を確保し、各バッファ内に MAX_TUPLE_STORE 分の MinimalTuple 配列を割り当てる。 4. 各ワーカーに対して ExecInitExtraTupleSlot を呼び出し、TTSOpsMinimalTuple を用いたタプルスロットを生成する。 5. heap_compare_slots を比較関数として指定し、binaryheap_allocate でマージ用のバイナリヒープを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecInitGatherMerge (ソースコード内の参照より)
  • **呼び出し先**: palloc0, ExecInitExtraTupleSlot, binaryheap_allocate, heap_compare_slots

関連データ構造

  • GatherMergeState: 実行状態を管理。
  • GMReaderTupleBuffer: 各ワーカーからのタプルを保持するバッファ。
  • TupleTableSlot: タプルを格納するスロット。
  • binaryheap: 順序維持マージのための優先度付きキュー。

失敗モードと境界条件

  • ワーカー数が 0 の場合でも、リーダー用のスロット 1 つ分は確保される。
  • メモリ不足時には palloc0 がエラーを発生させる。
  • 実行中に利用可能なワーカー数が num_workers より少ない場合、確保された余剰スロットは未使用となる。

保守担当者向けメモ

  • 本関数は ExecInitGatherMerge の一部として呼び出される。
  • MAX_TUPLE_STORE はパフォーマンスに直結するため、変更時はメモリ消費量とのトレードオフを考慮すること。
  • リーダープロセスがローカルスキャンを行う場合、スロット 0 が使用される点に注意。
