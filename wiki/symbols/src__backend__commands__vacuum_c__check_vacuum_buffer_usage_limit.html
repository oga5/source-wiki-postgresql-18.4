check_vacuum_buffer_usage_limit

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuum.c:138

目的

GUC（Grand Unified Configuration）パラメータ vacuum_buffer_usage_limit の設定値が、許容される範囲内にあるかを検証する。

シグネチャまたは形状

bool check_vacuum_buffer_usage_limit(int *newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定しようとしている値（kB単位）。
  • extra: GUCフック用の補助ポインタ（未使用）。
  • source: 設定のソース元（未使用）。
  • **出力**:
  • true: 値が 0 であるか、または MIN_BAS_VAC_RING_SIZE_KB から MAX_BAS_VAC_RING_SIZE_KB の範囲内である場合。
  • false: 上記以外の値である場合。

前提条件と不変条件

  • newval が指す値が 0 の場合、無制限またはデフォルトの挙動を意味する有効な値として扱われる。
  • 0 以外の場合、MIN_BAS_VAC_RING_SIZE_KB 以上かつ MAX_BAS_VAC_RING_SIZE_KB 以下である必要がある。

アルゴリズムまたは振る舞い

1. newval が 0 であるか、定義された最小値と最大値の範囲内にあるかを確認する。 2. 条件を満たす場合は true を返す。 3. 条件を満たさない場合は GUC_check_errdetail を使用してエラー詳細を設定し、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc_tables.c において、GUCパラメータのチェックフックとして登録されている。
  • **呼び出し先**: GUC_check_errdetail（PostgreSQLのGUCエラー報告用マクロ）。

関連データ構造

  • MIN_BAS_VAC_RING_SIZE_KB および MAX_BAS_VAC_RING_SIZE_KB（定数として定義）。

失敗モードと境界条件

  • 0 以外の正の整数であっても、定義された最小値未満または最大値を超える場合は検証に失敗する。
  • 負の値は検証に失敗する。

保守担当者向けメモ

  • 本関数は GUC システムのフックとして動作するため、直接的なロジック変更は vacuum_buffer_usage_limit の制約変更を意味する。
  • extra および source 引数は現在の実装では使用されていない。
