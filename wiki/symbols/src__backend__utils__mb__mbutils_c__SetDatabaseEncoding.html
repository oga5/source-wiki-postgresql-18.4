SetDatabaseEncoding

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/mbutils.c:1286

目的

データベースのエンコーディング設定をグローバル変数に適用し、システム全体で使用するエンコーディング情報を初期化・更新します。

シグネチャまたは形状

void SetDatabaseEncoding(int encoding)

入力と出力

  • **入力**: encoding (int型) - 設定するエンコーディングID。
  • **出力**: なし（void）。

前提条件と不変条件

  • 入力された encoding は PG_VALID_BE_ENCODING マクロにより、有効なバックエンドエンコーディングであることが検証されます。
  • 実行後、グローバル変数 DatabaseEncoding が指し示すエンコーディングIDと、引数 encoding は一致しなければなりません（Assertによる検証）。

アルゴリズムまたは振る舞い

1. PG_VALID_BE_ENCODING を用いて、指定されたエンコーディングIDが妥当か確認します。無効な場合は elog(ERROR) を発生させます。 2. pg_enc2name_tbl テーブルから、指定されたIDに対応するエンコーディング情報を取得し、グローバル変数 DatabaseEncoding に代入します。 3. DatabaseEncoding が正しく設定されたことを Assert で確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/init/postinit.c:406
  • **呼び出し先**:
  • elog (エラー発生時)

関連データ構造

  • DatabaseEncoding: データベースのエンコーディング情報を保持するグローバル変数。
  • pg_enc2name_tbl: エンコーディングIDと名称・属性をマッピングする静的テーブル。

失敗モードと境界条件

  • 指定された encoding が PG_VALID_BE_ENCODING の範囲外である場合、エラーが発生し処理が中断されます。

保守担当者向けメモ

  • 本関数はデータベース初期化プロセスにおいて重要な役割を果たします。エンコーディングの定義を変更する場合は、pg_wchar.h の定義および pg_enc2name_tbl の整合性に注意してください。
