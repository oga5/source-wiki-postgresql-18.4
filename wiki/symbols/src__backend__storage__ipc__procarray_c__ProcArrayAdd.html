ProcArrayAdd

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procarray.c:467

目的

共有メモリ上のプロセス配列（ProcArray）に新しい PGPROC を登録し、トランザクション管理のための状態を初期化する。

シグネチャまたは形状

void ProcArrayAdd(PGPROC *proc)

入力と出力

  • **入力**: proc (登録対象の PGPROC 構造体へのポインタ)
  • **出力**: なし（副作用として共有メモリ上のプロセス配列が更新される）

前提条件と不変条件

  • ProcArrayLock および XidGenLock が排他モードで取得される必要がある。
  • プロセス配列は pgprocnos（プロセス番号）の昇順でソート状態が維持される。
  • 各 PGPROC の pgxactoff は、配列内での現在のインデックスと一致しなければならない。

アルゴリズムまたは振る舞い

1. ProcArrayLock と XidGenLock を排他モードで取得する。 2. 配列の空き容量を確認し、不足している場合は FATAL エラーを発生させる。 3. pgprocnos を走査し、新しい proc を挿入すべきインデックス位置を特定する。 4. 特定した位置以降の配列要素（pgprocnos, xids, subxidStates, statusFlags）を memmove で後方にずらす。 5. 空いた位置に新しい proc の情報を書き込み、proc->pgxactoff を更新する。 6. 配列内の全要素数（numProcs）をインクリメントする。 7. 挿入位置以降のすべての PGPROC について、pgxactoff を再計算して更新する。 8. XidGenLock、続いて ProcArrayLock の順でロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c
  • src/backend/storage/lmgr/proc.c
  • **呼び出し先**:
  • GetNumberFromPGProc (マクロまたはインライン関数)
  • LWLockAcquire / LWLockRelease
  • memmove

関連データ構造

  • ProcArrayStruct: 共有メモリ上のプロセス管理用メイン構造体。
  • PGPROC: 個々のバックエンドプロセスの状態を保持する構造体。
  • ProcGlobal: プロセス配列に関連するグローバルなトランザクション情報を保持する構造体。

失敗モードと境界条件

  • **接続数超過**: arrayP->numProcs >= arrayP->maxProcs の場合、ERRCODE_TOO_MANY_CONNECTIONS で FATAL エラーとなる。

保守担当者向けメモ

  • 本関数は ProcArrayLock を保持した状態で XidGenLock を取得する。デッドロックを避けるため、解放順序は取得と逆順（XidGenLock -> ProcArrayLock）である必要がある。
  • 配列のソート順序維持は、参照の局所性を高めるための設計である。挿入・削除頻度は低いため、memmove によるオーバーヘッドは許容されている。
