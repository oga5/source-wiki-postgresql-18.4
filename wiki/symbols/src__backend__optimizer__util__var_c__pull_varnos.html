pull_varnos

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/var.c:113

目的

クエリツリーまたは式ツリーを走査し、含まれるすべての Var ノード（および PlaceHolderVar 等）の varno を収集してビットマップセット（Relids）として返します。

シグネチャまたは形状

Relids pull_varnos(PlannerInfo *root, Node *node)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体へのポインタ。
  • node: 走査対象の Query または式ツリー。
  • **出力**:
  • Relids: 抽出された varno の集合。

前提条件と不変条件

  • sublevels_up は 0 から開始されます。
  • node が Query オブジェクトである場合、サブレベルのインクリメントを行わずに走査を開始します。

アルゴリズムまたは振る舞い

1. pull_varnos_context を初期化し、varnos を NULL に設定します。 2. query_or_expression_tree_walker を呼び出し、pull_varnos_walker を用いてツリーを再帰的に走査します。 3. 走査中に発見された varno を context.varnos に蓄積します。 4. 最終的に収集された Relids を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/clausesel.c
  • src/backend/optimizer/path/costsize.c
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/path/joinpath.c
  • **呼び出し先**:
  • query_or_expression_tree_walker
  • pull_varnos_walker (内部的な走査関数)

関連データ構造

  • Relids: varno を保持するビットマップセット。
  • pull_varnos_context: 走査状態を保持する内部構造体。
  • PlannerInfo: クエリ全体のプランニング情報を保持する構造体。

失敗モードと境界条件

  • 入力 node が NULL の場合、空の Relids を返します。
  • 走査対象に Var ノードが含まれない場合、NULL（空のビットマップ）を返します。

保守担当者向けメモ

  • PlaceHolderVar は Var と同様に扱われます。
  • sublevels_up の管理により、サブクエリ内の変数と外側の変数を適切に区別して収集します。
