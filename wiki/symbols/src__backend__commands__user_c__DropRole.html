DropRole

定義

  • 種別: function
  • 定義箇所: src/backend/commands/user.c:1089

目的

SQLコマンド DROP ROLE を実行し、指定されたロールをシステムから削除します。

シグネチャまたは形状

void DropRole(DropRoleStmt *stmt)

入力と出力

  • **入力**: DropRoleStmt *stmt（削除対象のロール名リストを含む構文解析ノード）
  • **出力**: なし（成功時はカタログ更新、失敗時はエラーをスロー）

前提条件と不変条件

  • 実行には CREATEROLE 属性が必要であり、かつ削除対象のロールに対して ADMIN オプションを保持している必要があります。
  • 削除対象がスーパーユーザーである場合、実行者自身もスーパーユーザーである必要があります。
  • 現在のセッションユーザー、または現在実行中のユーザー自身を削除することはできません。

アルゴリズムまたは振る舞い

1. **権限チェック**: 実行者が CREATEROLE 権限を持つか検証します。 2. **カタログオープン**: pg_authid および pg_auth_members を RowExclusiveLock で開きます。 3. **ロール検索**: pg_authid システムキャッシュから対象ロールを検索します。存在しない場合は missing_ok フラグに応じてエラーまたは通知を返します。 4. **安全性の検証**: 削除対象が現在のユーザーやセッションユーザーでないか確認し、スーパーユーザー削除の権限を検証します。 5. **ロック**: LockSharedObject を使用して、削除対象ロールに AccessExclusiveLock を取得します。 6. **メンバーシップのクリーンアップ**: pg_auth_members をスキャンし、対象ロールが関与するメンバーシップ情報を削除します。 7. **再帰的処理**: CommandCounterIncrement() を呼び出し、変更を確定させつつ、リスト内の全ロールに対して順次処理を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:922（DropRoleStmt を受け取って呼び出し）
  • **呼び出し先**:
  • table_open, systable_beginscan, systable_getnext, systable_endscan
  • SearchSysCache1, ReleaseSysCache
  • deleteSharedDependencyRecordsFor, CatalogTupleDelete
  • LockSharedObject, CommandCounterIncrement
  • InvokeObjectDropHook

関連データ構造

  • DropRoleStmt: 削除対象ロールのリストを保持するノード。
  • pg_authid: ロール定義を保持するシステムカタログ。
  • pg_auth_members: ロール間のメンバーシップ関係を保持するシステムカタログ。

失敗モードと境界条件

  • **権限不足**: CREATEROLE 属性がない、または対象ロールに対する ADMIN 権限がない場合に ERRCODE_INSUFFICIENT_PRIVILEGE を発生。
  • **オブジェクト不在**: ロールが存在せず missing_ok が偽の場合に ERRCODE_UNDEFINED_OBJECT を発生。
  • **使用中**: 現在のユーザーやセッションユーザーを削除しようとした場合に ERRCODE_OBJECT_IN_USE を発生。
  • **不正な指定**: ROLESPEC_CSTRING 以外のロール指定子を使用した場合に ERRCODE_INVALID_PARAMETER_VALUE を発生。

保守担当者向けメモ

  • pg_auth_members の削除処理において、GRANT/REVOKE の依存関係を考慮した二段階の処理が行われています。
  • CommandCounterIncrement() は、同一トランザクション内で複数のロールを削除する際に、カタログの整合性を保つために不可欠です。
