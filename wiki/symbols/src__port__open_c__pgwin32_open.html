pgwin32_open

定義

  • 種別: function
  • 定義箇所: src/port/open.c:167

目的

Windows 環境において、POSIX 準拠の open() をエミュレートします。Windows ネイティブの CreateFile を利用してファイルハンドルを取得し、それを C ランタイムのファイル記述子（fd）に変換することで、PostgreSQL がプラットフォーム間で共通のファイル操作インターフェースを使用できるようにします。

シグネチャまたは形状

int pgwin32_open(const char *fileName, int fileFlags, ...);

入力と出力

  • **入力**:
  • fileName: 開くファイルのパス。
  • fileFlags: POSIX 互換のオープンフラグ（O_CREAT, O_TRUNC, O_BINARY, O_TEXT 等）。
  • **出力**:
  • 成功時: POSIX 互換のファイル記述子（int）。
  • 失敗時: -1（errno が適切に設定されます）。

前提条件と不変条件

  • 本関数は WIN32 マクロが定義されている環境でのみ有効です。
  • pgwin32_open_handle が返す Windows ハンドルが有効である必要があります。
  • FRONTEND 定義時は、O_BINARY または O_TEXT が指定されていない場合、デフォルトで O_TEXT が適用されます。

アルゴリズムまたは振る舞い

1. pgwin32_open_handle を呼び出し、再試行ロジックを含む Windows ネイティブのファイルオープンを実行します。 2. ハンドル取得に失敗した場合は -1 を返します。 3. FRONTEND ビルドの場合、フラグのデフォルト値を調整します。 4. _open_osfhandle を使用して、取得した Windows ハンドルを C ランタイムのファイル記述子に変換します。 5. 変換に失敗した場合はハンドルをクローズします。 6. O_TEXT または O_BINARY フラグが指定されている場合、_setmode を用いてファイル記述子のモードを設定します。設定に失敗した場合は記述子をクローズして -1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/port.h（open マクロとして定義）
  • src/bin/pg_upgrade/pg_upgrade.h（言及あり）
  • src/port/open.c 内の pgwin32_fopen（推測される関連性）
  • **呼び出し先**:
  • pgwin32_open_handle
  • _open_osfhandle
  • _setmode
  • _close
  • CloseHandle

関連データ構造

  • HANDLE（Windows API）
  • fileFlags（fcntl.h 等で定義される POSIX フラグ）

失敗モードと境界条件

  • pgwin32_open_handle が INVALID_HANDLE_VALUE を返した場合、即座に -1 を返します。
  • _open_osfhandle が失敗した場合、ハンドルをクローズし、エラーを伝播します。
  • _setmode が失敗した場合、ファイル記述子をクローズし、-1 を返します。

保守担当者向けメモ

  • 本関数は open() のラッパーとして機能するため、src/include/port.h でのマクロ置換を通じて呼び出されます。
  • O_CLOEXEC のサポートは、CreateFile のセキュリティ属性を通じて行われており、SetHandleInformation は使用されていません。
  • STATUS_DELETE_PENDING への対応など、Windows 特有のファイルシステム挙動を吸収するロジックが含まれています。
