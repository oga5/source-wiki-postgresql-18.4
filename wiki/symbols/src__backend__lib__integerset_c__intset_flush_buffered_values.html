intset_flush_buffered_values

定義

  • 種別: function
  • 定義箇所: src/backend/lib/integerset.c:394

目的

IntegerSet 内のバッファに一時蓄積された整数値を Simple-8b アルゴリズムで圧縮し、B-tree のリーフノードへ永続的に書き出す。

シグネチャまたは形状

static void intset_flush_buffered_values(IntegerSet *intset)

入力と出力

  • **入力**: IntegerSet *intset （操作対象の整数集合構造体）
  • **出力**: なし（副作用として intset 内のバッファがクリアされ、B-tree 構造が更新される）

前提条件と不変条件

  • intset->buffered_values に格納された整数値は昇順であること。
  • 呼び出し時点で intset は有効なメモリコンテキストに存在すること。

アルゴリズムまたは振る舞い

1. **初期化**: 集合が空の場合、最初のリーフノード（兼ルート）を生成する。 2. **圧縮ループ**: バッファ内の値が MAX_VALUES_PER_LEAF_ITEM 以上ある限り、simple8b_encode を用いて leaf_item を生成する。 3. **ノードへの追加**:

  • 現在のリーフノードが満杯（MAX_LEAF_ITEMS）であれば、新しいリーフノードを割り当ててリンクし、intset_update_upper で上位ノードを更新する。
  • 生成した leaf_item をリーフノードに追加する。

    • 4. **バッファの整理**: 処理しきれなかった（MAX_VALUES_PER_LEAF_ITEM 未満の）残りの値をバッファの先頭へ memmove で移動し、num_buffered_values を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/lib/integerset.c 内の intset_add_member 等から呼び出される。
  • **呼び出し先**:
  • intset_new_leaf_node
  • simple8b_encode
  • intset_update_upper
  • memmove

関連データ構造

  • IntegerSet: 集合全体を管理する構造体。
  • intset_leaf_node: 圧縮された値を保持するリーフノード。
  • leaf_item: リーフノード内の最小単位（プレーン値と圧縮コードワード）。

失敗モードと境界条件

  • **空のバッファ**: num_values が MAX_VALUES_PER_LEAF_ITEM 未満の場合、ループは実行されずバッファはそのまま維持される。
  • **ノードの溢れ**: リーフノードが満杯になると自動的に新しいリーフノードが割り当てられ、ツリー構造が拡張される。

保守担当者向けメモ

  • 本関数は intset_add_member の挿入処理の一部として機能する。
  • MAX_VALUES_PER_LEAF_ITEM 未満の値をバッファに残すことで、エンコーダが入力不足に陥ることを防いでいる。
  • 挿入順序の制約（昇順）は本関数でも維持される必要がある。
