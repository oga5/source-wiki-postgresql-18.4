get_attr_stat_type

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/attribute_stats.c:594

目的

指定されたリレーション属性（カラム）の統計情報取得に必要な型情報、照合順序、および比較演算子（等価・不等価）を導出します。

シグネチャまたは形状

static void get_attr_stat_type(Oid reloid, AttrNumber attnum,
                               Oid *atttypid, int32 *atttypmod,
                               char *atttyptype, Oid *atttypcoll,
                               Oid *eq_opr, Oid *lt_opr)

入力と出力

  • **入力**:
  • reloid: 対象リレーションのOID。
  • attnum: 対象属性の番号。
  • **出力**:
  • atttypid: 属性のデータ型OID（マルチレンジ型の場合はレンジ型に変換される）。
  • atttypmod: 属性の型修飾子。
  • atttyptype: 型のカテゴリ（typcacheより取得）。
  • atttypcoll: 属性の照合順序OID。
  • eq_opr: 等価演算子OID。
  • lt_opr: 不等価（小なり）演算子OID。

前提条件と不変条件

  • 対象のリレーションは AccessShareLock でオープンされます。
  • システムカタログ pg_attribute に該当する属性が存在し、かつ削除（dropped）されていない必要があります。
  • tsvector 型の場合、照合順序は強制的に DEFAULT_COLLATION_OID に設定されます。

アルゴリズムまたは振る舞い

1. SearchSysCache2 を使用して pg_attribute から属性情報を取得し、存在確認を行います。 2. get_attr_expr を呼び出し、式インデックスの場合はその式の型情報を、通常の列の場合は列定義の型情報を優先して取得します。 3. 取得した型がマルチレンジ型である場合、get_multirange_range を用いて基底のレンジ型へ変換します。 4. lookup_type_cache を使用して、型に対応する演算子（eq_opr, lt_opr）および型カテゴリを解決します。 5. 最後に relation_close でリレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/attribute_stats.c 内の attribute_statistics_update から呼び出されます。
  • **呼び出し先**: relation_open, SearchSysCache2, get_attr_expr, exprType, exprTypmod, exprCollation, ReleaseSysCache, type_is_multirange, get_multirange_range, lookup_type_cache, relation_close。

関連データ構造

  • Form_pg_attribute: 属性の定義情報。
  • TypeCacheEntry: 型のキャッシュ情報（演算子情報を含む）。
  • Node: 式インデックスの表現。

失敗モードと境界条件

  • 属性が存在しない、または attisdropped が真である場合、ERRCODE_UNDEFINED_COLUMN エラーを発生させます。
  • システム列（attnum < 0）に対する動作は本関数内では明示的に制限されていませんが、pg_attribute の検索結果に依存します。

保守担当者向けメモ

  • 式インデックスの型解決ロジックは examine_attribute() の挙動を模倣しています。修正時は当該関数との整合性に注意してください。
  • tsvector の照合順序のハードコードは compute_tsvector_stats() との整合性を保つための特殊処理です。
