PQsocketPoll

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-misc.c:1140

目的

指定されたソケットに対して、読み込みまたは書き込みの準備が整うまで、あるいはタイムアウトに達するまで待機する抽象化レイヤーを提供します。

シグネチャまたは形状

int PQsocketPoll(int sock, int forRead, int forWrite, pg_usec_time_t end_time)

入力と出力

  • sock: 監視対象のソケットファイル記述子。
  • forRead: 読み込み可能状態を監視する場合に非ゼロを指定。
  • forWrite: 書き込み可能状態を監視する場合に非ゼロを指定。
  • end_time: 待機終了時刻（マイクロ秒単位）。-1 は無期限待機、0 は即時復帰を意味します。
  • 戻り値: poll(2) または select(2) のシステムコール結果を返します。

前提条件と不変条件

  • ソケットは非ブロッキングモードで動作していることを前提としています。
  • end_time が指定された場合、内部で PQgetCurrentTimeUSec() を呼び出し、現在時刻との差分からタイムアウト値を算出します。

アルゴリズムまたは振る舞い

1. HAVE_POLL が定義されている環境では poll(2) を使用し、それ以外では select(2) を使用します。 2. forRead および forWrite が両方とも偽の場合、待機せずに 0 を返します。 3. タイムアウト計算:

  • end_time == -1: タイムアウトなし。
  • end_time == 0: タイムアウトを 0 に設定。
  • end_time > 0: 現在時刻と比較し、経過していれば 0、そうでなければ残り時間を算出します。

    • 4. システムコールを実行し、ソケットの状態変化を待機します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/psql/command.c:4419
  • src/interfaces/libpq-oauth/oauth-curl.c:1556
  • src/interfaces/libpq/fe-misc.c:1114
  • 呼び出し先:
  • PQgetCurrentTimeUSec() (タイムアウト計算用)
  • poll(2) または select(2) (システム依存)

関連データ構造

  • pollfd (HAVE_POLL時)
  • fd_set (非HAVE_POLL時)
  • pg_usec_time_t (時刻表現)

失敗モードと境界条件

  • sock が無効な場合、システムコールがエラーを返し、その結果がそのまま戻り値となります。
  • end_time が現在時刻より過去の場合、タイムアウト値は 0 として扱われ、即座に復帰します。

保守担当者向けメモ

  • 本関数は poll と select の両方の実装を保持しており、プラットフォームごとの差異を吸収しています。修正時は両方のパスに影響がないか確認してください。
  • libpq-fe.h にて公開 API として宣言されています。
