hashint4extended

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:77

目的

int4 型の値を、指定されたシード値を用いてハッシュ化し、ハッシュ結合等の処理で利用可能な 64 ビットのハッシュ値を生成します。

シグネチャまたは形状

Datum hashint4extended(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • 第1引数: ハッシュ対象の int4 値。
  • 第2引数: ハッシュ計算に使用する int8 型のシード値。
  • **出力**: 計算されたハッシュ値を Datum 型として返します。

前提条件と不変条件

  • 入力値は PG_FUNCTION_ARGS マクロを通じて正しく渡される必要があります。
  • 内部で呼び出される hash_uint32_extended の仕様に従い、同一の入力とシード値に対しては常に決定論的な結果を返します。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT32(0) により、第1引数から int4 値を取得します。 2. PG_GETARG_INT64(1) により、第2引数からシード値を取得します。 3. hash_uint32_extended 関数を呼び出し、取得した値とシード値を渡してハッシュ計算を実行します。 4. 計算結果を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し先**: hash_uint32_extended
  • **呼び出し元**: 提供されたソースコード内には、本関数を明示的に呼び出している他の関数に関する情報はありません。

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • PG_FUNCTION_ARGS: SQL 関数インターフェースのための引数コンテナ。

失敗モードと境界条件

  • PG_GETARG_* マクロが期待する型の引数が渡されない場合、PostgreSQL の関数呼び出し規約に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は ...extended バージョンのハッシュ関数であり、ハッシュ結合のパフォーマンス最適化のためにシード値を受け取る設計となっています。
  • int4 型のハッシュ計算ロジックを変更する場合は、対応する非拡張版の hashint4 との整合性に注意してください。
