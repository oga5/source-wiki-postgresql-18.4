DetermineSleepTime

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/postmaster.c:1544

目的

Postmaster プロセスが次のイベント処理まで待機すべき時間をミリ秒単位で算出する。

シグネチャまたは形状

int DetermineSleepTime(void)

入力と出力

  • **入力**: グローバル変数 Shutdown, StartWorkerNeeded, HaveCrashedWorker, AbortStartTime, BackgroundWorkerList を参照。
  • **出力**: 待機時間（ミリ秒単位の int 値）。

前提条件と不変条件

  • 戻り値は常に 0 以上である。
  • シャットダウンシーケンス中、またはバックグラウンドワーカーが不要な場合は、デフォルトで 60,000 ミリ秒（1分）を返す。

アルゴリズムまたは振る舞い

1. **シャットダウン/通常時**: シャットダウン中またはワーカー起動が不要な場合、AbortStartTime が設定されていれば強制終了までの残り時間を返し、そうでなければ 60,000 ミリ秒を返す。 2. **ワーカー起動要求**: StartWorkerNeeded が真であれば、即時起動のため 0 を返す。 3. **クラッシュしたワーカーの再起動**: HaveCrashedWorker が真の場合、BackgroundWorkerList を走査し、各ワーカーの再起動予定時刻（rw_crashed_at + bgw_restart_time）を計算する。 4. **待機時間の決定**: 最も早い再起動時刻までの時間を算出し、最大 60,000 ミリ秒に制限して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c 内のメインループ（ServerLoop 等）から呼び出される。
  • **呼び出し先**: ForgetBackgroundWorker（ワーカーの破棄）、TimestampTzPlusMilliseconds、TimestampDifferenceMilliseconds 等のユーティリティ関数。

関連データ構造

  • BackgroundWorkerList: 再起動待ちのバックグラウンドワーカーを保持するリスト。
  • RegisteredBgWorker: 各バックグラウンドワーカーの状態を保持する構造体。

失敗モードと境界条件

  • AbortStartTime が設定されている場合、残り時間が負にならないよう Max(seconds * 1000, 0) でクランプされる。
  • TimestampDifferenceMilliseconds の結果は [0, INT_MAX] の範囲に収まることが保証されている。
  • BGW_NEVER_RESTART が設定されているワーカーは再起動対象から除外される。

保守担当者向けメモ

  • 本関数は ServerLoop の待機時間を決定する重要なロジックである。変更時はバックグラウンドワーカーの再起動タイミングに影響を与えるため注意すること。
  • 戻り値の最大値（60秒）は、Postmaster が定期的に状態を確認するための上限値として機能している。
