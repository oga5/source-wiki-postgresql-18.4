WALAvailability

定義

  • 種別: enum
  • 定義箇所: src/include/access/xlog.h:189

目的

特定の WAL セグメントがシステム内でどのような保持状態にあるか（利用可能か、削除済みか等）を分類し、管理するための列挙型です。

シグネチャまたは形状

typedef enum WALAvailability
{
    WALAVAIL_INVALID_LSN,
    WALAVAIL_RESERVED,
    WALAVAIL_EXTENDED,
    WALAVAIL_UNRESERVED,
    WALAVAIL_REMOVED,
} WALAvailability;

入力と出力

  • 入力: GetWALAvailability(XLogRecPtr targetLSN) 関数に対し、対象となる LSN を渡すことで、対応するセグメントの状態を本列挙型として取得します。

前提条件と不変条件

  • 各列挙値は、WAL セグメントの保持ポリシー（max_wal_size や wal_keep_size、レプリケーションスロットによる予約）に基づいた状態を反映します。

アルゴリズムまたは振る舞い

  • WALAVAIL_INVALID_LSN: パラメータエラー。
  • WALAVAIL_RESERVED: セグメントが max_wal_size の範囲内に存在。
  • WALAVAIL_EXTENDED: レプリケーションスロットまたは wal_keep_size により明示的に保持されている。
  • WALAVAIL_UNRESERVED: 保持期間を過ぎているが、物理的にはまだ削除されていない。
  • WALAVAIL_REMOVED: セグメントは既に削除済み。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c および src/backend/replication/slotfuncs.c から参照されていることが確認されています。
  • 呼び出し先: 本シンボル自体はデータ型であり、関数呼び出し先はありません。

関連データ構造

  • XLogRecPtr: WAL 内の論理的な位置を示す型。本列挙型の判定基準として使用されます。

失敗モードと境界条件

  • WALAVAIL_INVALID_LSN は、無効な LSN が指定された場合の境界条件として定義されています。

保守担当者向けメモ

  • 本列挙型は GetWALAvailability 関数の戻り値として利用されます。状態遷移のロジックを変更する場合は、src/backend/access/transam/xlog.c の実装と整合性を確認してください。
