hash_create

定義

  • 種別: function
  • 定義箇所: src/backend/utils/hash/dynahash.c:351

目的

PostgreSQLの動的ハッシュテーブル（HTAB）を初期化し、メモリ割り当て、ハッシュ関数、比較関数、およびキーコピー関数の設定を行う。

シグネチャまたは形状

HTAB *hash_create(const char *tabname, long nelem, const HASHCTL *info, int flags)

入力と出力

  • **入力**:
  • tabname: ハッシュテーブルを識別するための名称。
  • nelem: 初期要素数の目安。
  • info: ハッシュテーブルの構成情報（HASHCTL）。
  • flags: 動作を指定するビットフラグ（HASH_ELEM, HASH_SHARED_MEM, HASH_PARTITION 等）。
  • **出力**:
  • 初期化された HTAB 構造体へのポインタ。

前提条件と不変条件

  • flags に HASH_ELEM が含まれていること。
  • info->keysize > 0 であること。
  • info->entrysize >= info->keysize であること。
  • 共有メモリテーブルの場合、HASH_DIRSIZE および HASH_ALLOC フラグが適切に設定されている必要がある。

アルゴリズムまたは振る舞い

1. **メモリコンテキストの決定**: HASH_SHARED_MEM 指定時は TopMemoryContext を使用し、それ以外は専用のメモリコンテキストを作成する。 2. **構造体割り当て**: HTAB 構造体とテーブル名をメモリ上に確保し、初期化する。 3. **関数ポインタの設定**:

  • ハッシュ関数: HASH_FUNCTION 指定時は指定関数、HASH_BLOBS 指定時は uint32_hash または tag_hash、それ以外は string_hash を選択。
  • 比較関数: HASH_COMPARE 指定時は指定関数、それ以外は string_compare または memcmp を選択。
  • キーコピー関数: HASH_KEYCOPY 指定時は指定関数、それ以外は strlcpy または memcpy を選択。

    • 4. **共有メモリの処理**: HASH_SHARED_MEM 指定時は既存の共有メモリ構造にアタッチし、HASH_ATTACH フラグがあればここで終了する。 5. **初期化の完了**: hdefault() を呼び出し、ハッシュテーブルのデフォルト値を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/dblink, contrib/postgres_fdw, src/backend/access/heap/rewriteheap.c 等の各機能モジュールから多数の呼び出しがある。
  • **呼び出し先**: AllocSetContextCreate, MemoryContextAlloc, DynaHashAlloc, hdefault, strcpy, strlen, MemSet 等。

関連データ構造

  • HTAB: ハッシュテーブルの制御構造体。
  • HASHHDR: 共有メモリ上に配置されるヘッダー情報。
  • HASHCTL: 設定パラメータを保持する構造体。

失敗モードと境界条件

  • メモリ不足時: alloc 関数が失敗した場合、ereport(ERROR, ...) により処理を中断する。
  • 不適切なフラグ指定: HASH_STRINGS を指定せずに keysize が小さい場合や、HASH_FUNCTION と HASH_BLOBS を同時に指定した場合などに Assert 違反が発生する。

保守担当者向けメモ

  • 共有メモリ上のハッシュテーブルはディレクトリの動的拡張ができないため、nelem の見積もりが重要である。
  • string_hash の直接指定は非推奨であり、HASH_STRINGS フラグの使用が推奨される。
