ProcessMainLoopInterrupts

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/interrupt.c:33

目的

PostgreSQLのバックグラウンドプロセスにおいて、メインループ内で保留中のシグナル割り込みや要求を安全に処理する。

シグネチャまたは形状

void ProcessMainLoopInterrupts(void)

入力と出力

  • 入力: なし（グローバルなフラグ変数を参照）
  • 出力: なし（副作用として設定再読み込み、プロセス終了、メモリログ出力などを実行）

前提条件と不変条件

  • 本関数は、シグナルハンドラによって設定された sig_atomic_t 型のフラグ（ConfigReloadPending 等）を監視する。
  • メインループの反復ごとに呼び出されることを想定している。

アルゴリズムまたは振る舞い

以下の順序で保留中のフラグを確認し、該当する場合は処理を実行する： 1. ProcSignalBarrierPending: ProcessProcSignalBarrier() を呼び出す。 2. ConfigReloadPending: フラグを false にリセットし、ProcessConfigFile(PGC_SIGHUP) を実行して設定を再読み込みする。 3. ShutdownRequestPending: proc_exit(0) を呼び出し、プロセスを正常終了させる。 4. LogMemoryContextPending: ProcessLogMemoryContextInterrupt() を呼び出し、メモリコンテキストのログを出力する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/postmaster/bgwriter.c
  • src/backend/postmaster/walwriter.c
  • 呼び出し先:
  • ProcessProcSignalBarrier
  • ProcessConfigFile
  • proc_exit
  • ProcessLogMemoryContextInterrupt

関連データ構造

  • ConfigReloadPending (グローバルフラグ)
  • ShutdownRequestPending (グローバルフラグ)
  • ProcSignalBarrierPending (グローバルフラグ)
  • LogMemoryContextPending (グローバルフラグ)

失敗モードと境界条件

  • ShutdownRequestPending が真の場合、本関数は proc_exit を呼び出すため、呼び出し元へは戻らない。
  • 各フラグは非同期シグナルハンドラから設定されるため、本関数はメインループの適切なタイミングで実行される必要がある。

保守担当者向けメモ

  • 本関数は、シグナルハンドラ内での非同期安全性を維持するために、ハンドラ側ではフラグ設定のみを行い、実処理を本関数に集約する設計となっている。
  • 新しいバックグラウンドプロセスを追加する際は、メインループ内で本関数を定期的に呼び出すことで、標準的なシグナル処理を継承できる。
