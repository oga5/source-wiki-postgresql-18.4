test_slru_page_truncate

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_slru.c:175

目的

SQLインターフェースを通じて、テスト用SLRU（Simple Least Recently Used）の指定ページ以降を切り詰める（Truncate）操作を実行する。

シグネチャまたは形状

Datum test_slru_page_truncate(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT64(0) により、切り詰め開始位置となるページ番号（int64）を受け取る。
  • **出力**: なし（PG_RETURN_VOID()）。

前提条件と不変条件

  • 本関数はPostgreSQLの拡張モジュールとしてロードされている必要がある。
  • TestSlruCtl が SimpleLruInit によって初期化済みであること。

アルゴリズムまたは振る舞い

1. 引数からページ番号 pageno を取得する。 2. TestSlruCtl を制御構造体として、SimpleLruTruncate を呼び出し、指定されたページ番号以降のSLRUセグメントを切り詰める。 3. 処理完了後、voidを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: SimpleLruTruncate（access/slru.h で定義されたPostgreSQLコアのSLRU API）。

関連データ構造

  • TestSlruCtl: テスト用SLRUを制御する SlruCtlData 構造体。

失敗モードと境界条件

  • 指定されたページ番号が不正な場合、SimpleLruTruncate 内部のロジックに従いエラーが発生する可能性がある。
  • 共有メモリ上のSLRU状態が初期化されていない場合、実行時に予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、実運用環境のコードではなく、SLRUの切り詰めロジックの検証を目的としている。
  • SimpleLruTruncate の挙動変更は本関数の動作に直接影響を与える。
