RBTNode

定義

  • 種別: struct
  • 定義箇所: src/include/lib/rbtree.h:23

目的

PostgreSQL の赤黒木（Red-Black Tree）実装において、木構造を構成する各ノードの内部状態を保持するための構造体です。

シグネチャまたは形状

typedef struct RBTNode
{
    char color;                 /* ノードの色（赤または黒） */
    struct RBTNode *left;       /* 左の子、または RBTNIL */
    struct RBTNode *right;      /* 右の子、または RBTNIL */
    struct RBTNode *parent;     /* 親、または NULL */
} RBTNode;

入力と出力

本構造体はデータ構造の構成要素であり、関数ではありません。ユーザー定義の構造体の先頭に埋め込むことで使用されます。

前提条件と不変条件

  • **侵入型設計**: ユーザー定義構造体の先頭メンバとして配置する必要があります。
  • **直接操作の禁止**: color, left, right, parent フィールドはライブラリ内部で管理されるため、呼び出し側が直接変更してはなりません。
  • **NULL の扱い**: parent が NULL の場合は親が存在しないことを示しますが、left および right は存在しない場合に RBTNIL（センチネルノード）を指す必要があります。

アルゴリズムまたは振る舞い

赤黒木の平衡維持アルゴリズム（回転や再着色）において、ノード間のリンクを保持します。ポインタのキャストにより、RBTNode* からそれを包含するユーザー定義構造体への変換が可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginbulk.c 内の ginCombineData や cmpEntryAccumulator などが、RBTNode を含む構造体を操作する際に参照しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTreeIterator: 走査状態を保持する構造体。

失敗モードと境界条件

  • parent フィールドはルートノードにおいて NULL となりますが、left および right は NULL ではなく RBTNIL を指すという境界条件があります。

保守担当者向けメモ

  • 本構造体はメモリ管理を呼び出し側に委譲しています。ノードの確保・解放には rbt_create で指定された allocfunc および freefunc が使用される点に留意してください。
