AggStatePerHashData

定義

  • **種別**: struct
  • **定義箇所**: src/include/executor/nodeAgg.h:309

目的

ハッシュ集約（Hash Aggregation）モードにおいて、特定のハッシュテーブルおよびその操作に必要なメタデータと実行状態を保持する。

シグネチャまたは形状

typedef struct AggStatePerHashData
{
    TupleHashTable hashtable;       /* グループごとのエントリを持つハッシュテーブル */
    TupleHashIterator hashiter;     /* ハッシュテーブル走査用イテレータ */
    TupleTableSlot *hashslot;       /* ハッシュテーブル読み込み用スロット */
    FmgrInfo   *hashfunctions;      /* グループ化フィールドごとのハッシュ関数 */
    Oid        *eqfuncoids;         /* グループ化フィールドごとの等価比較関数 */
    int         numCols;            /* ハッシュキー列の数 */
    int         numhashGrpCols;     /* ハッシュテーブル内の列数 */
    int         largestGrpColIdx;   /* ハッシュに必要な最大列インデックス */
    AttrNumber *hashGrpColIdxInput; /* 入力スロット内のハッシュ列インデックス */
    AttrNumber *hashGrpColIdxHash;  /* ハッシュテーブルタプル内のインデックス */
    Agg        *aggnode;            /* 元の Agg ノード（グループ数等の参照用） */
} AggStatePerHashData;

入力と出力

  • **入力**: Agg ノードのプラン情報および入力タプル。
  • **出力**: ハッシュテーブルに格納された集約状態。

前提条件と不変条件

  • 本構造体はハッシュ集約実行時に AggState 内で管理される。
  • numCols、numhashGrpCols、largestGrpColIdx は、ハッシュテーブルの構築およびアクセスにおいて整合性が保たれている必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: ExecInitAgg 時に palloc0 を使用してメモリが割り当てられ、プランノードの情報に基づいて各フィールドが設定される。 2. **ハッシュ計算**: hashfunctions を用いて入力タプルからハッシュ値を算出し、hashtable への挿入または検索を行う。 3. **グループ照合**: eqfuncoids を用いて、既存のグループとの等価性を判定する。 4. **走査**: hashiter を使用して、集約完了後のハッシュテーブルを順次走査し、結果を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeAgg.c 内の集約初期化処理（palloc0 によるメモリ確保）。
  • **呼び出し先**: 実行時のハッシュテーブル操作関数（TupleHashTable 等の API）および FmgrInfo で指定された関数群。

関連データ構造

  • AggState: 本構造体を保持する上位の実行状態構造体。
  • TupleHashTable: ハッシュ集約の基盤となるテーブル構造。
  • Agg: プランノード。

失敗モードと境界条件

  • メモリ不足: palloc0 失敗時にエラーが発生する可能性がある。
  • ハッシュキーの不一致: hashGrpColIdxInput と hashGrpColIdxHash のインデックス設定が不適切な場合、ハッシュテーブルの検索・挿入が正しく行われない。

保守担当者向けメモ

  • AggStatePerHashData はハッシュ集約専用である。ソートベースの集約には使用されない。
  • numhashGrpCols と numCols の違いに注意すること。前者はハッシュテーブル内の物理的な列数、後者は論理的なキー列数を指す。
