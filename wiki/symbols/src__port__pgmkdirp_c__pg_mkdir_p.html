pg_mkdir_p

定義

  • 種別: function
  • 定義箇所: src/port/pgmkdirp.c:56

目的

指定されたパスに対して再帰的にディレクトリを作成する。UNIXの mkdir -p コマンドと同等の機能を提供し、既存のディレクトリが存在してもエラーとしない。

シグネチャまたは形状

int pg_mkdir_p(char *path, int omode);

入力と出力

  • **入力**:
  • path: 作成対象のディレクトリパス（破壊的に変更される）。
  • omode: 作成するディレクトリのパーミッション（int 型）。
  • **出力**:
  • 成功時: 0
  • 失敗時: -1（errno が設定される）

前提条件と不変条件

  • path 引数は関数内部で / を \0 に置き換えるため、呼び出し元で文字列を保持する必要がある場合は注意が必要。
  • Windows環境では、ドライブレターやUNCパスのプレフィックスを適切にスキップする。
  • 親ディレクトリ作成時には、一時的に umask を変更して書き込み・実行権限（u+wx）を確保する。

アルゴリズムまたは振る舞い

1. **パスの正規化**: Windows環境の場合、ネットワークドライブやローカルドライブの指定をスキップする。 2. **umaskの調整**: 親ディレクトリ作成時に必要な権限を確保するため、現在の umask を保存し、u+wx を許可するように変更する。 3. **再帰的作成**: パスを / で分割し、各階層を順次走査する。

  • stat で存在確認を行い、ディレクトリでない場合はエラーとする。
  • 存在しない場合は mkdir を実行する。

    • 4. **復元**: 処理終了時またはエラー発生時に、保存していた元の umask を必ず復元する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c
  • src/backend/commands/tablespace.c
  • src/bin/initdb/initdb.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_combinebackup/pg_combinebackup.c
  • src/bin/pg_upgrade/relfilenumber.c
  • **呼び出し先**:
  • stat, mkdir, umask, strlen, strstr（標準ライブラリ関数）

関連データ構造

  • struct stat: ファイルの状態情報を保持するために使用。

失敗モードと境界条件

  • 指定したパスの階層にディレクトリ以外のファイルが存在する場合、ENOTDIR または EEXIST エラーを返す。
  • mkdir システムコールが失敗した場合、-1 を返す。
  • エラー発生時、path 引数は「問題が発生した階層」までが書き換えられた状態となる。

保守担当者向けメモ

  • omode は port.h への依存を避けるため mode_t ではなく int で定義されている。
  • Windows環境におけるパス処理の特殊性については、src/port/path.c のユーティリティ関数群との整合性に注意すること。
