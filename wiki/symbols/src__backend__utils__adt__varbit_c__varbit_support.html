varbit_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:701

目的

BIT および BIT VARYING 型に対するクエリ最適化（簡略化）をサポートします。具体的には、型修飾子（typmod）の変更が不要または安全な場合に、式を再ラベル付けして最適化します。

シグネチャまたは形状

Datum varbit_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して SupportRequestSimplify 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、簡略化された Node（成功時）または NULL（最適化不可時）を返します。

前提条件と不変条件

  • 入力された FuncExpr は少なくとも2つの引数を持つ必要があります。
  • typmod 引数は Const ノードであり、かつ非 NULL である必要があります。
  • varbit 型において typmod 0 は無効な値として扱われます。

アルゴリズムまたは振る舞い

1. SupportRequestSimplify リクエストであるかを確認します。 2. 式の第2引数から新しい typmod を取得します。 3. 以下の条件を満たす場合、relabel_to_typmod を呼び出して式を再ラベル付けします。

  • 新しい typmod が 0 より大きい。
  • 元の typmod が 0 以下であるか、または元の typmod が新しい typmod 以下である（縮小が発生しない）。

    • 4. 条件を満たさない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のプランナ/オプティマイザ（直接の呼び出し元詳細はソースコード内に明示されていません）。
  • **呼び出し先**: IsA (マクロ), PG_GETARG_POINTER, list_length, lsecond, linitial, exprTypmod, relabel_to_typmod, PG_RETURN_POINTER。

関連データ構造

  • SupportRequestSimplify: クエリ簡略化リクエストを保持するノード。
  • FuncExpr: 関数呼び出し式。
  • Const: 定数ノード。

失敗モードと境界条件

  • typmod が 0 以下の場合、最適化は行われません。
  • typmod が NULL の場合、最適化は行われません。
  • 引数が不足している場合、Assert により停止する可能性があります。

保守担当者向けメモ

  • 本関数は varbit 型の型修飾子変更に伴うキャストの最適化を担います。
  • relabel_to_typmod の挙動に依存しているため、型システムの変更時には注意が必要です。
