BuildEventTriggerCache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/evtcache.c:76

目的

システムカタログ pg_event_trigger をスキャンし、イベントトリガーの定義情報をメモリ上のハッシュテーブルとして再構築する。

シグネチャまたは形状

static void BuildEventTriggerCache(void)

入力と出力

  • **入力**: なし（システムカタログ pg_event_trigger および pg_event_trigger_name_index を直接参照）
  • **出力**: なし（副作用としてグローバル変数 EventTriggerCache および EventTriggerCacheState を更新）

前提条件と不変条件

  • EventTriggerCacheContext が初期化済みであるか、本関数内で作成される必要がある。
  • CacheMemoryContext が存在しない場合は作成される。
  • 再構築中は EventTriggerCacheState が ETCS_REBUILD_STARTED に設定され、完了時に ETCS_VALID へ遷移する。

アルゴリズムまたは振る舞い

1. **メモリ管理**: EventTriggerCacheContext をリセットまたは作成し、メモリコンテキストを切り替える。 2. **無効化登録**: 初回実行時に InvalidateEventCacheCallback をシステムキャッシュのコールバックとして登録する。 3. **ハッシュテーブル構築**: EventTriggerCacheHash を作成し、pg_event_trigger を名前順でスキャンする。 4. **データ抽出**: 各タプルからイベント名（evtevent）、関数OID、コマンドタグを抽出し、EventTriggerCacheItem を生成する。 5. **ハッシュ格納**: イベントタイプをキーとして EventTriggerCacheEntry にトリガーリストを格納する。 6. **状態更新**: スキャン終了後、キャッシュを EventTriggerCache に割り当て、状態を ETCS_VALID に更新する。ただし、再構築中に無効化通知が届いていた場合は、次回再構築を促すために状態を維持する。

呼び出し元と呼び出し先

  • **呼び出し元**: EventCacheLookup（src/backend/utils/cache/evtcache.c 内）から呼び出される。
  • **呼び出し先**: MemoryContextReset, AllocSetContextCreate, CacheRegisterSyscacheCallback, MemoryContextSwitchTo, hash_create, systable_beginscan_ordered, systable_getnext_ordered, DecodeTextArrayToBitmapset, hash_search, lappend, list_make1 等。

関連データ構造

  • EventTriggerCache: キャッシュの実体であるハッシュテーブル。
  • EventTriggerCacheContext: キャッシュ専用のメモリコンテキスト。
  • EventTriggerCacheEntry: ハッシュテーブルの各エントリ。
  • EventTriggerCacheItem: 個別のトリガー定義情報。

失敗モードと境界条件

  • **無効化の競合**: 再構築中に無効化通知が発生した場合、現在の再構築処理を完了させた上で、キャッシュを ETCS_VALID ではなく ETCS_NEEDS_REBUILD 相当の状態に留めることで整合性を維持する。
  • **無効なタプル**: evtenabled が TRIGGER_DISABLED のトリガーはキャッシュから除外される。
  • **メモリ不足**: palloc0 等のメモリ確保失敗時には標準のメモリ管理例外が発生する。

保守担当者向けメモ

  • 本関数は EventTriggerCacheContext を使用するため、戻り値のリストを長期間保持する場合は呼び出し側でコピーを作成すること。
  • pg_event_trigger のスキーマ変更や新しいイベントタイプの追加時には、evtevent の判定ロジックを更新する必要がある。
