accesstype_arg_to_string

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_oat_hooks/test_oat_hooks.c:456

目的

PostgreSQLのオブジェクトアクセスフック（ObjectAccessType）に関連付けられた追加情報（arg）を、デバッグやログ出力のために人間が読み取り可能な文字列へ変換する。

シグネチャまたは形状

static char *accesstype_arg_to_string(ObjectAccessType access, void *arg)

入力と出力

  • **入力**:
  • access: オブジェクトアクセスの種別（ObjectAccessType）。
  • arg: 各アクセス種別に対応する構造体へのポインタ。
  • **出力**:
  • 変換された文字列を格納したメモリ領域へのポインタ（pstrdup または psprintf で割り当て）。

前提条件と不変条件

  • arg が NULL の場合は "extra info null" を返す。
  • OAT_TRUNCATE および OAT_FUNCTION_EXECUTE は引数を取らないため、これらが渡された場合はエラーを示す文字列を返す。
  • 認識できない access 型が渡された場合は、デフォルトの警告文字列を返す。

アルゴリズムまたは振る舞い

1. arg が NULL かどうかを確認し、該当すれば即座に文字列を返す。 2. access の値に基づき switch 文で分岐する。

  • OAT_POST_CREATE: ObjectAccessPostCreate にキャストし、is_internal フラグに応じて "internal" または "explicit" を返す。
  • OAT_DROP: ObjectAccessDrop にキャストし、dropflags のビットマスクを評価して、設定されているフラグ（PERFORM_DELETION_*）をカンマ区切りの文字列として連結する。
  • OAT_POST_ALTER: ObjectAccessPostAlter にキャストし、is_internal および auxiliary_id の有無を文字列に含める。
  • OAT_NAMESPACE_SEARCH: ObjectAccessNamespaceSearch にキャストし、ereport_on_violation と result の状態を文字列化する。

    • 3. 処理終了後、生成された文字列を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_oat_hooks/test_oat_hooks.c 内の監査ログ出力処理（emit_audit_message 等）から呼び出される。
  • **呼び出し先**: pstrdup および psprintf（PostgreSQLのメモリ管理関数）。

関連データ構造

  • ObjectAccessType（列挙型）
  • ObjectAccessPostCreate
  • ObjectAccessDrop
  • ObjectAccessPostAlter
  • ObjectAccessNamespaceSearch

失敗モードと境界条件

  • メモリ不足時には pstrdup / psprintf 経由でシステムエラーが発生する可能性がある。
  • 予期しない access 型が渡された場合、安全策として "cannot parse extra info..." という文字列を返す。

保守担当者向けメモ

  • 本関数はテスト用モジュール内でのログ出力専用である。
  • 新しい ObjectAccessType が追加された場合は、本関数の switch 文にも対応するケースを追加する必要がある。
  • psprintf を使用しているため、戻り値の文字列は呼び出し元で適切に解放（pfree）される必要がある。
