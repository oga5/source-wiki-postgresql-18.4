ReadNextXLogRecord

定義

  • 種別: function
  • 定義箇所: contrib/pg_walinspect/pg_walinspect.c:149

目的

XLogReaderState を使用して、WALの次のレコードを読み取り、デコードして返します。

シグネチャまたは形状

static XLogRecord *ReadNextXLogRecord(XLogReaderState *xlogreader)

入力と出力

  • **入力**: XLogReaderState *xlogreader（WAL読み取り状態を保持する構造体）
  • **出力**: 読み取りに成功した場合は XLogRecord *、WALの終端に達した場合は NULL を返します。読み取りエラーが発生した場合はエラーを報告し、処理を中断します。

前提条件と不変条件

  • xlogreader は初期化済みである必要があります。
  • xlogreader->private_data は ReadLocalXLogPageNoWaitPrivate 型の構造体としてキャスト可能である必要があります。

アルゴリズムまたは振る舞い

1. XLogReadRecord を呼び出し、次のWALレコードを読み取ります。 2. 読み取り結果が NULL の場合:

  • private_data->end_of_wal を確認し、真であれば NULL を返して終了します。
  • それ以外の場合、エラーメッセージの有無に応じて ereport(ERROR, ...) を発行し、読み取り失敗を通知します。

    • 3. 読み取りに成功した場合は、取得した XLogRecord へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_walinspect/pg_walinspect.c 内の各関数（pg_get_wal_records_info 等）のループ処理内で呼び出されます。
  • **呼び出し先**:
  • XLogReadRecord（access/xlogreader.h で定義された基盤関数）

関連データ構造

  • XLogReaderState: WAL読み取りの制御状態を保持。
  • XLogRecord: デコードされたWALレコードの構造体。
  • ReadLocalXLogPageNoWaitPrivate: WALの終端状態を判定するために private_data として参照される構造体。

失敗モードと境界条件

  • **WAL終端**: 正常に終端に達した場合は NULL を返します。
  • **読み取りエラー**: ファイルアクセスエラーや破損などにより XLogReadRecord が失敗し、かつ終端ではない場合、ereport によりPostgreSQLのトランザクションがアボートされます。

保守担当者向けメモ

  • 本関数は pg_walinspect モジュール内の走査ループの基盤です。
  • XLogReadRecord の挙動変更や、エラーハンドリングの要件変更がある場合は、本関数のエラー報告ロジックと同期させる必要があります。
