GetWALRecordInfo

定義

  • 種別: function
  • 定義箇所: contrib/pg_walinspect/pg_walinspect.c:191

目的

XLogReaderState からデコードされたWALレコードの情報を抽出し、SQLインターフェースで利用可能な Datum 配列形式に変換します。

シグネチャまたは形状

static void GetWALRecordInfo(XLogReaderState *record, Datum *values, bool *nulls, uint32 ncols)

入力と出力

  • **入力**:
  • record: 解析済みのWALレコードを保持する XLogReaderState へのポインタ。
  • ncols: 期待されるカラム数。
  • **出力**:
  • values: 抽出されたレコード情報（LSN、XID、レコードタイプ、統計情報等）が格納される Datum 配列。
  • nulls: 各カラムがNULLかどうかを示すフラグ配列。

前提条件と不変条件

  • record は有効な XLogReaderState を指している必要があります。
  • values および nulls 配列は、少なくとも ncols 分の領域が確保されている必要があります。
  • 関数終了時に i == ncols であることが Assert により保証されます。

アルゴリズムまたは振る舞い

1. リソースマネージャ（Rmgr）の識別子を取得し、レコードタイプを特定します。 2. desc.rm_desc を呼び出し、レコードの詳細説明を生成します。 3. レコードがブロック参照を持つ場合、XLogRecGetBlockRefInfo を用いてブロック参照情報を展開します。 4. 以下の情報を values 配列に順次格納します：

  • LSN（読み取り位置、終了位置、前レコード位置）
  • トランザクションID
  • リソースマネージャ名およびレコードタイプ
  • レコードの総長、データ長、FPI（Full Page Image）長
  • レコードの詳細説明およびブロック参照情報（存在する場合）

    • 5. 該当する情報がない場合は nulls フラグを立てます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_walinspect/pg_walinspect.c 内の pg_get_wal_record_info (497行目) および GetWALRecordsInfo (568行目) から呼び出されています。
  • **呼び出し先**:
  • GetRmgr: リソースマネージャ情報の取得。
  • initStringInfo / desc.rm_desc: レコード詳細の生成。
  • XLogRecHasAnyBlockRefs / XLogRecGetBlockRefInfo: ブロック参照の解析。
  • 各種 Datum 変換マクロ（LSNGetDatum, CStringGetTextDatum 等）。

関連データ構造

  • XLogReaderState: WAL読み取り状態を保持する構造体。
  • RmgrData: リソースマネージャの定義情報。
  • StringInfoData: 可変長文字列バッファ。

失敗モードと境界条件

  • レコードタイプが不明な場合、UNKNOWN としてフォーマットされます。
  • レコード詳細やブロック参照が存在しない場合、対応する nulls フラグが設定されます。
  • ncols と実際の書き込み数が一致しない場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は pg_get_wal_records_info 等の出力形式と密接に関連しています。カラムの追加や変更を行う際は、呼び出し側の ncols 定義と同期させる必要があります。
  • palloc を使用したメモリ確保が行われるため、呼び出し元で適切な MemoryContext の管理が行われていることを前提としています。
