dsm_pin_mapping

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm.c:914

目的

指定された動的共有メモリ（DSM）セグメントの自動デタッチ管理を解除し、現在のリソースオーナーによる自動クリーンアップの対象から外す。

シグネチャまたは形状

void dsm_pin_mapping(dsm_segment *seg)

入力と出力

  • **入力**: seg (操作対象の dsm_segment 構造体へのポインタ)
  • **出力**: なし（副作用として seg->resowner を更新）

前提条件と不変条件

  • seg は有効な dsm_segment 構造体を指している必要がある。
  • 本関数を呼び出すことで、当該セグメントは現在のリソースオーナーの管理下から離れ、セッション終了時やトランザクション終了時に自動的にデタッチされなくなる。

アルゴリズムまたは振る舞い

1. seg->resowner が NULL でないかを確認する。 2. NULL でない場合、ResourceOwnerForgetDSM を呼び出して、現在のリソースオーナーから当該セグメントの追跡を解除する。 3. seg->resowner を NULL に設定し、管理状態を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/session.c
  • src/backend/storage/ipc/dsm_registry.c
  • src/backend/utils/mmgr/dsa.c
  • **呼び出し先**:
  • ResourceOwnerForgetDSM (リソースオーナー管理機能)

関連データ構造

  • dsm_segment: DSM セグメントを管理する構造体。resowner フィールドを保持する。

失敗モードと境界条件

  • seg->resowner が既に NULL の場合、本関数は何も行わず正常に終了する。

保守担当者向けメモ

  • 本関数は、セグメントの生存期間を現在のリソースオーナーのライフサイクルから切り離すために使用される。
  • セッション全体で維持する必要があるマッピングに対して使用すること。
  • 各プロセスで個別に呼び出す必要がある点に注意。
