ConfigVariable

定義

  • 種別: struct
  • 定義箇所: src/include/utils/guc.h:139

目的

設定ファイル（postgresql.conf 等）の解析時に生成される、設定パラメータの名前と値のペアを保持する連結リスト構造体です。

シグネチャまたは形状

typedef struct ConfigVariable
{
	char	   *name;        /* 設定パラメータ名 */
	char	   *value;       /* 設定値 */
	char	   *errmsg;      /* 解析エラー時のメッセージ */
	char	   *filename;    /* 定義元のファイル名 */
	int			sourceline;  /* 定義元の行番号 */
	bool		ignore;      /* 無視フラグ */
	bool		applied;     /* 適用済みフラグ */
	struct ConfigVariable *next; /* リストの次要素へのポインタ */
} ConfigVariable;

入力と出力

  • 入力: 設定ファイル解析プロセスにより、ファイル名、行番号、パラメータ名、値が格納されます。
  • 出力: 連結リストとして保持され、設定の適用や postgresql.auto.conf への書き出し処理に利用されます。

前提条件と不変条件

  • next ポインタにより連結リストを構成します。
  • applied フラグは、設定がシステムに適用されたかどうかを追跡するために使用されます。

アルゴリズムまたは振る舞い

設定ファイルの読み込み時に、各行が ConfigVariable インスタンスとして生成され、リストに追加されます。その後、システムはリストを走査して設定を適用し、必要に応じて applied フラグを更新します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/guc.c 内の write_auto_conf_file や replace_auto_config_value など、設定ファイルの読み書きを行う関数群から参照されます。また、src/backend/commands/extension.c において拡張機能の制御ファイル解析時にも利用されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • GucSource: 設定値の優先順位を決定する際に併用されることがあります。

失敗モードと境界条件

  • 解析エラーが発生した場合、errmsg フィールドにエラー内容が格納されます。
  • リストの末尾は next が NULL であることで示されます。

保守担当者向けメモ

  • 本構造体は GUC システムのフロントエンドとバックエンドの橋渡しを行う重要なデータ構造です。
  • postgresql.auto.conf の更新ロジック（replace_auto_config_value 等）を変更する際は、リストの整合性が保たれるよう注意してください。
