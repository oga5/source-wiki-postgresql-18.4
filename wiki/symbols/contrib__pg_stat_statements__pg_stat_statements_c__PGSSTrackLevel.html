PGSSTrackLevel

定義

  • 種別: enum
  • 定義箇所: contrib/pg_stat_statements/pg_stat_statements.c:280

目的

pg_stat_statements 拡張機能において、どの範囲の SQL 文を統計対象として追跡するかを制御するための列挙型です。

シグネチャまたは形状

typedef enum
{
    PGSS_TRACK_NONE,    /* 統計を追跡しない */
    PGSS_TRACK_TOP,     /* トップレベルの文のみ追跡 */
    PGSS_TRACK_ALL,     /* ネストされた文を含むすべての文を追跡 */
} PGSSTrackLevel;

入力と出力

  • 入力: なし（設定値としての列挙型）
  • 出力: なし

前提条件と不変条件

  • pg_stat_statements の設定パラメータ（GUC）として利用され、実行時の統計収集動作を決定します。

アルゴリズムまたは振る舞い

  • PGSS_TRACK_NONE: 統計収集を無効化します。
  • PGSS_TRACK_TOP: クエリのネストレベルを考慮し、トップレベルのクエリのみを統計対象とします。
  • PGSS_TRACK_ALL: 関数内などで実行されるネストされたクエリを含め、すべてのクエリを統計対象とします。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは、具体的な関数呼び出し元は特定できません。
  • 呼び出し先: なし（データ型定義のため）。

関連データ構造

  • pgssSharedState: 共有メモリ上の状態管理構造体。本列挙型の設定値に基づいて統計収集の可否が判断されます。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 本列挙型は、pg_stat_statements の統計収集範囲を切り替えるための重要な設定値です。新しい追跡レベルを追加する場合は、関連するフック関数（planner_hook や ExecutorStart_hook 等）での条件分岐ロジックを更新する必要があります。
