diag4

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/io.c:555

目的

整形処理中に発生した警告またはエラーメッセージを、出力先（標準出力または標準エラー出力）に応じて整形して報告する。

シグネチャまたは形状

void diag4(int level, const char *msg, int a, int b)

入力と出力

  • **入力**:
  • level: 診断レベル（0以外の場合はエラーとみなす）。
  • msg: printf 形式のフォーマット文字列。
  • a, b: msg に渡す引数。
  • **出力**: なし（副作用としてストリームへの書き出しを行う）。

前提条件と不変条件

  • グローバル変数 line_no が現在の行番号を保持していること。
  • グローバル変数 output が出力ストリームを指していること。
  • level が0以外の場合、グローバル変数 found_err が 1 に設定される。

アルゴリズムまたは振る舞い

1. level が真であれば、エラーフラグ found_err を 1 にセットする。 2. output が stdout と等しい場合:

  • /**INDENT** [Warning|Error]@[line_no]: という形式で stdout に出力する。
  • msg を a, b と共にフォーマットして出力する。
  • 最後に */\n を付加する。

    • 3. output が stdout 以外の場合:

  • [Warning|Error]@[line_no]: という形式で stderr に出力する。
  • msg を a, b と共にフォーマットして出力し、改行する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。
  • **呼び出し先**: fprintf（標準ライブラリ）。

関連データ構造

  • found_err（グローバル変数）
  • line_no（グローバル変数）
  • output（グローバル変数）

失敗モードと境界条件

  • msg に不正なフォーマット文字列が渡された場合、fprintf の動作に依存する。
  • output がクローズされている場合や書き込み権限がない場合、fprintf は失敗する可能性がある。

保守担当者向けメモ

  • 本関数は indent.h で宣言されています。
  • stdout への出力時に /**INDENT** という特殊なコメント形式で出力されるため、この出力を解析するツールが存在する可能性があります。変更時は注意してください。
