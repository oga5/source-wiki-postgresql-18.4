oidrecv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:59

目的

PostgreSQLの Oid 型の外部バイナリ形式を、内部的な Oid 値に変換します。

シグネチャまたは形状

Datum oidrecv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される StringInfo 型のバッファ（PG_GETARG_POINTER(0)）。
  • **出力**: 変換された Oid 値を Datum 型として返却します。

前提条件と不変条件

  • 入力バッファは libpq のバイナリフォーマットに従っている必要があります。
  • Oid のサイズ（sizeof(Oid)）がバッファから正しく読み取れる必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、関数引数から StringInfo 型のバッファを取得します。 2. pq_getmsgint を呼び出し、バッファから sizeof(Oid) バイト分のデータを読み取ります。 3. 読み取った値を Oid 型にキャストし、PG_RETURN_OID マクロを介して Datum として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/regproc.c 内の複数の関数（regprocrecv, regprocedurein 等のバイナリ入力処理）から、コード共有の目的で呼び出されています。
  • **呼び出し先**:
  • pq_getmsgint（libpq/pqformat.h）

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子型。
  • StringInfo: バイナリメッセージを保持する構造体。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • バッファの残量が sizeof(Oid) に満たない場合、pq_getmsgint がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は Oid 型専用のバイナリ入力ルーチンですが、regproc.c 内の類似型でも共通して利用されています。修正を行う際は、これらの依存先への影響を考慮してください。
