SharedInvalidationMessage

定義

  • 種別: typedef (共用体)
  • 定義箇所: src/include/storage/sinval.h:124

目的

PostgreSQLの共有キャッシュ無効化メカニズムにおいて、バックエンド間でやり取りされる各種無効化メッセージを包含し、単一の型として扱うための共用体です。

シグネチャまたは形状

typedef union {
    int8 id; /* 型識別子（必須） */
    SharedInvalCatcacheMsg cc;
    SharedInvalCatalogMsg cat;
    SharedInvalRelcacheMsg rc;
    SharedInvalSmgrMsg sm;
    SharedInvalRelmapMsg rm;
    SharedInvalSnapshotMsg sn;
    SharedInvalRelSyncMsg rs;
} SharedInvalidationMessage;

入力と出力

  • **入力**: 共有メモリ上の無効化キューへ書き込まれる際、各キャッシュ無効化イベントの内容が各メンバに格納されます。
  • **出力**: 受信側バックエンドがキューから読み取り、id フィールドに基づいて適切なキャッシュ無効化関数へ渡されます。

前提条件と不変条件

  • id フィールドは共用体の先頭に配置されなければなりません。
  • メッセージの型識別は、この id フィールドの値（SHAREDINVALCATALOG_ID 等）によって行われます。

アルゴリズムまたは振る舞い

1. **識別**: 受信側は id を確認し、どのキャッシュ（Catcache, Relcache等）を無効化すべきかを判断します。 2. **処理**: 識別された型に応じて、対応する構造体メンバが参照され、キャッシュの整合性を保つための無効化処理が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/rmgrdesc/standbydesc.c
  • src/backend/access/rmgrdesc/xactdesc.c
  • src/backend/access/transam/twophase.c
  • **呼び出し先**:
  • 本シンボル自体はデータ構造であり、直接関数を呼び出しません。受信処理において ReceiveSharedInvalidMessages 等の関数が本構造体の中身を解析・処理します。

関連データ構造

  • SharedInvalCatcacheMsg, SharedInvalCatalogMsg, SharedInvalRelcacheMsg, SharedInvalSmgrMsg, SharedInvalRelmapMsg, SharedInvalSnapshotMsg, SharedInvalRelSyncMsg

失敗モードと境界条件

  • **キューの溢れ**: 共有メモリ上の無効化キューが満杯になった場合、PROCSIG_CATCHUP_INTERRUPT を介した強制的な同期処理が発生します。
  • **ハッシュ衝突**: Catcacheの無効化はハッシュ値ベースで行われるため、稀に無関係なエントリが誤って無効化される可能性がありますが、これはキャッシュの再ロードにより許容されます。

保守担当者向けメモ

  • 本構造体はトランザクションコミット時や特定のストレージ操作時に共有メモリを介して受け渡されます。新しい無効化メッセージ型を追加する場合は、本共用体にメンバを追加し、id による識別ロジックを更新してください。
