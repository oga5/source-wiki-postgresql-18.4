PQsendQueryParams

定義

  • **種別**: 関数
  • **定義箇所**: src/interfaces/libpq/fe-exec.c:1508

目的

パラメータ化されたクエリを非同期にバックエンドへ送信します。名前のないステートメントを使用し、クライアントからバックエンドへパラメータ値を渡すためのインターフェースを提供します。

シグネチャまたは形状

int PQsendQueryParams(PGconn *conn,
                      const char *command,
                      int nParams,
                      const Oid *paramTypes,
                      const char *const *paramValues,
                      const int *paramLengths,
                      const int *paramFormats,
                      int resultFormat);

入力と出力

  • **入力**:
  • conn: 接続オブジェクト。
  • command: 実行する SQL 文字列。
  • nParams: パラメータ数。
  • paramTypes: パラメータの OID 配列（NULL 可）。
  • paramValues: パラメータ値の配列。
  • paramLengths: 各パラメータの長さ（バイナリ形式の場合）。
  • paramFormats: 各パラメータの形式（0: テキスト, 1: バイナリ）。
  • resultFormat: 結果の形式（0: テキスト, 1: バイナリ）。
  • **出力**: 成功時に 1、失敗時に 0 を返します。

前提条件と不変条件

  • conn が有効であること。
  • nParams は 0 以上 PQ_QUERY_PARAM_MAX_LIMIT 以下であること。
  • command は NULL であってはならない。

アルゴリズムまたは振る舞い

1. PQsendQueryStart を呼び出し、接続状態がクエリ送信可能か確認します。 2. 引数の妥当性チェック（command の NULL チェック、nParams の範囲チェック）を行います。 3. チェックを通過した場合、内部関数 PQsendQueryGuts を呼び出し、名前のないステートメントとしてクエリとパラメータをバックエンドへ送信します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/postgres_fdw/deparse.c、contrib/postgres_fdw/postgres_fdw.c、src/bin/pgbench/pgbench.c、src/bin/psql/common.c、src/include/libpq/libpq-be-fe-helpers.h、src/test/modules/libpq_pipeline/libpq_pipeline.c 等から参照されています。
  • **呼び出し先**: PQsendQueryStart および PQsendQueryGuts を呼び出します。

関連データ構造

  • PGconn: 接続状態を保持する構造体。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • command が NULL の場合、エラーメッセージを記録し 0 を返します。
  • nParams が範囲外の場合、エラーメッセージを記録し 0 を返します。
  • PQsendQueryStart が失敗した場合、その戻り値（0）をそのまま返します。

保守担当者向けメモ

  • 本関数は名前のないステートメントを強制的に使用します。名前付きステートメントが必要な場合は PQsendQueryPrepared を検討してください。
  • エラー発生時は libpq_append_conn_error を通じて接続オブジェクトにエラー情報が付与されます。
