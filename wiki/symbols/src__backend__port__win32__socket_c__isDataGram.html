isDataGram

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:168

目的

指定されたソケットがデータグラム型（UDP）であるかどうかを判定する。

シグネチャまたは形状

static int isDataGram(SOCKET s)

入力と出力

  • **入力**: SOCKET s（判定対象のWinsockソケットハンドル）
  • **出力**: int 型。データグラムソケットである場合は 1、それ以外（ストリームソケット等）または判定不能な場合は 0 を返す。

前提条件と不変条件

  • 引数 s は有効なWinsockソケットハンドルであること。
  • getsockopt が失敗した場合、安全側に倒して 1 を返す仕様となっている。

アルゴリズムまたは振る舞い

1. getsockopt を呼び出し、ソケットオプション SO_TYPE を取得する。 2. 取得したソケットタイプが SOCK_DGRAM と一致するかを比較する。 3. 一致すれば 1 を、一致しなければ 0 を返す。 4. getsockopt 自体がエラーを返した場合は、判定不能とみなして 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/win32/socket.c 内の関数（例: pgwin32_waitforsinglesocket 等のソケット操作関数）から参照されている。
  • **呼び出し先**: Winsock API の getsockopt を呼び出す。

関連データ構造

  • SOCKET: Windowsのソケットハンドル型。

失敗モードと境界条件

  • getsockopt が失敗した場合、関数は 1 を返す。これは、エラー発生時にデータグラムとして扱う（あるいはUDP特有の処理を適用する）という設計上の判断による。

保守担当者向けメモ

  • 本関数は static であり、src/backend/port/win32/socket.c 内でのみ使用される。
  • UDP通信時の高負荷下でのロック問題を回避するための特殊なタイムアウト処理（pgwin32_waitforsinglesocket 内）の判定ロジックとして利用されている。
