oauth_init

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-oauth.c:101

目的

SASL OAUTHBEARER 認証セッションを開始し、認証コンテキストを初期化する。

シグネチャまたは形状

static void *oauth_init(Port *port, const char *selected_mech, const char *shadow_pass);

入力と出力

  • **入力**:
  • port: 現在の接続情報を保持する Port 構造体。
  • selected_mech: クライアントが選択したSASLメカニズム名。
  • shadow_pass: 使用されない（NULL または無視される）。
  • **出力**:
  • 成功時: 初期化された oauth_ctx 構造体へのポインタ。
  • 失敗時: ereport(ERROR, ...) によりプロセスが中断される。

前提条件と不変条件

  • selected_mech は OAUTHBEARER_NAME と一致しなければならない。
  • port->hba が有効であり、OAuth関連の設定（oauth_issuer, oauth_scope, oauth_validator）が保持されていること。

アルゴリズムまたは振る舞い

1. selected_mech が OAUTHBEARER であるか検証し、不一致の場合はプロトコル違反としてエラーを報告する。 2. oauth_ctx 構造体をメモリ確保し、ゼロ初期化する。 3. コンテキストの状態を OAUTH_STATE_INIT に設定し、port 情報を保持する。 4. HBA設定から issuer および scope をコンテキストにコピーする。 5. load_validator_library を呼び出し、HBA設定で指定された検証用ライブラリをロードする。 6. 初期化済みコンテキストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c 内の pg_be_oauth_mech 構造体（認証メカニズムの初期化関数として登録）。
  • **呼び出し先**:
  • palloc0 (メモリ確保)
  • load_validator_library (外部検証モジュールのロード)
  • ereport (エラー報告)

関連データ構造

  • oauth_ctx: セッションごとの状態を保持する構造体。
  • Port: 接続情報およびHBA設定を保持する構造体。

失敗モードと境界条件

  • selected_mech が OAUTHBEARER 以外の場合、ERRCODE_PROTOCOL_VIOLATION で終了する。
  • load_validator_library が失敗した場合、その内部でエラーが発生し、本関数は復帰しない。

保守担当者向けメモ

  • 本関数はサーバー側の初期化処理です。src/interfaces/libpq/fe-auth-oauth.c に同名の関数が存在しますが、クライアント側の実装であり、本関数とは別物です。
  • shadow_pass 引数は現在の実装では使用されていません。
