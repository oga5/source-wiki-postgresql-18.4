FileSetInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fileset.c:51

目的

FileSet 構造体を初期化し、一時ファイルの配置先となるテーブルスペース設定を固定します。これにより、同一の FileSet を使用するすべてのバックエンド間で、パス解決の一貫性を保証します。

シグネチャまたは形状

void FileSetInit(FileSet *fileset)

入力と出力

  • **入力**: fileset (FileSet *) - 初期化対象の FileSet 構造体へのポインタ。
  • **出力**: なし（fileset が指す構造体のメンバが更新されます）。

前提条件と不変条件

  • fileset は有効なメモリ領域を指している必要があります。
  • MyProcPid および MyDatabaseTableSpace が現在のプロセスコンテキストで正しく設定されている必要があります。
  • FileSet の生存期間中、ここで取得されたテーブルスペース情報は変更されません。

アルゴリズムまたは振る舞い

1. **識別子の割り当て**: MyProcPid を creator_pid に設定し、静的カウンタを用いて一意な number を割り当てます。 2. **テーブルスペースの取得**: PrepareTempTablespaces() を呼び出し、現在の temp_tablespaces 設定をキャプチャします。 3. **設定の正規化**:

  • 設定が空の場合は、現在のデータベースのデフォルトテーブルスペース (MyDatabaseTableSpace) を使用します。
  • 設定内に InvalidOid が含まれる場合は、それを MyDatabaseTableSpace に置換し、すべてのバックエンドが同一のパス解決を行えるようにします。

関連データ構造

  • FileSet: 一時ファイルの集合を管理する構造体。creator_pid、number、およびテーブルスペースの配列を保持します。

失敗モードと境界条件

  • counter は INT_MAX でラップアラウンドします。
  • temp_tablespaces が空の場合、自動的にデフォルトのテーブルスペースへフォールバックします。

保守担当者向けメモ

  • FileSet はトランザクション終了時に自動削除されません。呼び出し側で FileSetDelete または FileSetDeleteAll を呼び出し、リソースを明示的に解放する必要があります。
  • FileSetInit で固定されたテーブルスペース情報は、後続の FileSetCreate 等でパスを生成する際の基盤となります。
