pg_preadv

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/port/pg_iovec.h:53

目的

プラットフォーム間で一貫したベクトル読み込み（scatter input）操作を提供するための抽象化レイヤーです。OS が preadv をサポートしている場合はそれを利用し、未サポートの場合は pg_pread を用いたエミュレーションを行います。

シグネチャまたは形状

ssize_t pg_preadv(int fd, const struct iovec *iov, int iovcnt, off_t offset);

入力と出力

  • **入力**:
  • fd: 読み込み対象のファイル記述子。
  • iov: 読み込み先バッファを記述する iovec 構造体の配列。
  • iovcnt: iov 配列の要素数。
  • offset: 読み込みを開始するファイル内のオフセット。
  • **出力**:
  • 成功時: 読み込んだ合計バイト数を ssize_t で返します。
  • 失敗時: -1 を返します。

前提条件と不変条件

  • iovcnt が 1 の場合、カーネルのオーバーヘッドを避けるため単一バッファの読み込みとして処理されます。
  • エミュレーション環境では、各 pg_pread 呼び出しごとに offset が読み込みバイト数分だけ進められます。
  • Windows 環境で使用する場合、ファイルポインタが移動する副作用があることに注意が必要です。

アルゴリズムまたは振る舞い

1. **ネイティブサポート時 (HAVE_DECL_PREADV が真)**:

  • iovcnt == 1 の場合: pread() を直接呼び出します。
  • それ以外の場合: システムの preadv() を呼び出します。

    • 2. **エミュレーション時**:

  • iov 配列をループで走査し、各要素に対して pg_pread() を実行します。
  • pg_pread() がエラーを返した場合、最初の要素であれば -1 を返し、それ以外であればその時点までの合計バイト数を返します。
  • 部分的な読み込みが発生した場合、その時点までの合計バイト数を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio_io.c:128
  • src/backend/storage/file/fd.c:2186
  • **呼び出し先**:
  • pread (ネイティブサポート時)
  • preadv (ネイティブサポート時)
  • pg_pread (エミュレーション時)

関連データ構造

  • struct iovec: 読み込み先バッファのベースアドレスと長さを保持する構造体。

失敗モードと境界条件

  • pg_pread がエラーを返した場合、処理は中断されます。

保守担当者向けメモ

  • PG_IOV_MAX を超える iovec 配列を扱う場合は、スタックオーバーフローを避けるためヒープメモリの使用を検討してください。
  • 本関数は static inline で定義されているため、ヘッダーをインクルードするすべての翻訳単位で展開されます。
