varbit_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varbit.c:635

目的

外部バイナリ表現からPostgreSQLの BIT および BIT VARYING 型の内部表現（VarBit）へ変換します。

シグネチャまたは形状

Datum varbit_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取ります。
  • buf (StringInfo): 受信したバイナリデータ。
  • atttypmod (int32): 型修飾子（ビット長制限）。
  • **出力**: Datum 型として変換後の VarBit 構造体へのポインタを返します。

前提条件と不変条件

  • bitlen は 0 以上 VARBITMAXLEN 以下である必要があります。
  • atttypmod が指定されている場合、入力された bitlen はその値を超えてはなりません。
  • VarBit 構造体の最後のバイトの未使用ビットは、VARBIT_PAD マクロにより常にゼロでなければなりません。

アルゴリズムまたは振る舞い

1. pq_getmsgint を使用して、バッファからビット長（bitlen）を読み取ります。 2. bitlen の妥当性を検証し、範囲外であればエラーを報告します。 3. atttypmod が指定されている場合、bitlen が制限を超えていないか確認し、超えていればエラーを報告します。 4. VARBITTOTALLEN を用いて必要なメモリサイズを計算し、palloc で領域を確保します。 5. SET_VARSIZE および VARBITLEN でヘッダ情報を設定します。 6. pq_copymsgbytes を使用して、バッファからビットデータをコピーします。 7. VARBIT_PAD を呼び出し、最後のバイトの余剰ビットをゼロクリアします。 8. 構築した VarBit を PG_RETURN_VARBIT_P で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は明示されていませんが、PostgreSQLの型システム（recv 関数）から呼び出されることを想定しています。
  • **呼び出し先**: pq_getmsgint, ereport, palloc, SET_VARSIZE, VARBITTOTALLEN, VARBITLEN, pq_copymsgbytes, VARBIT_PAD, PG_RETURN_VARBIT_P 等のユーティリティ関数およびマクロ。

関連データ構造

  • VarBit: ビット列を保持する構造体。
  • StringInfo: バイナリメッセージの読み取りに使用する構造体。

失敗モードと境界条件

  • bitlen が負の値、または VARBITMAXLEN を超える場合は ERRCODE_INVALID_BINARY_REPRESENTATION でエラーとなります。
  • bitlen が atttypmod を超える場合は ERRCODE_STRING_DATA_RIGHT_TRUNCATION でエラーとなります。

保守担当者向けメモ

  • 本関数は varbit_in と同様の検証ロジックを共有しています。
  • VARBIT_PAD によるゼロクリアは、比較演算（bit_cmp）の正確性を維持するために不可欠です。変更を加える際は、この不変条件が維持されていることを確認してください。
