wildcard_certificate_match

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-common.c:44

目的

SSL/TLS証明書のホスト名検証において、ワイルドカードパターン（例: *.example.com）が対象のホスト名と一致するかを判定する。RFC 2818に準拠したルールを適用する。

シグネチャまたは形状

static bool wildcard_certificate_match(const char *pattern, const char *string)

入力と出力

  • **入力**:
  • pattern: 証明書に含まれるワイルドカードパターン文字列。
  • string: 接続先のホスト名。
  • **出力**:
  • bool: パターンがホスト名に適合する場合は true、それ以外は false を返す。

前提条件と不変条件

  • pattern は * で始まり、その直後に . が続く必要がある（例: *.example.com）。
  • ワイルドカードは単一のパス名コンポーネントのみにマッチし、ドットを跨ぐことはできない。

アルゴリズムまたは振る舞い

1. **形式チェック**: パターンが * で始まらない、または *. で始まらない場合は false を返す。 2. **長さチェック**: パターンが対象文字列より長い場合は false を返す。 3. **末尾一致**: パターンのワイルドカード部分を除いた残りの文字列が、対象文字列の末尾と一致するかを大文字小文字を区別せず比較する。 4. **ドットの存在確認**: パターンのマッチ開始位置より左側にドットが存在する場合、ドットを跨ぐマッチングとみなして false を返す。 5. 上記をすべて通過した場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure-common.c 内の証明書検証ロジックから呼び出される。
  • **呼び出し先**: strlen, pg_strcasecmp, strchr などの標準ライブラリ関数およびユーティリティ関数。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • pattern が * のみ、あるいは * で始まらない場合は即座に失敗する。
  • 対象文字列の中にドットが複数含まれる場合、ワイルドカードがドットを跨ごうとすると失敗する。
  • 大文字小文字の差異は pg_strcasecmp により許容される。

保守担当者向けメモ

  • 本関数は CVE-2009-4034 対策として、証明書内の名前に埋め込まれた NULL バイトを拒否する文脈で利用される。
  • RFC 2818 の厳密な解釈に基づいているため、仕様変更がない限りロジックの変更は不要。
