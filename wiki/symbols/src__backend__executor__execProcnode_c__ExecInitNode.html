ExecInitNode

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/executor/execProcnode.c:141

目的

プランナーが生成したプランツリーを走査し、実行エンジンが利用可能な PlanState ツリーを再帰的に構築する。

シグネチャまたは形状

PlanState *ExecInitNode(Plan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象のプランノード（Plan 構造体）。
  • estate: 実行状態を保持する EState。
  • eflags: 実行オプションを示すビットフラグ。
  • **出力**:
  • 構築された PlanState 構造体へのポインタ。node が NULL の場合は NULL を返す。

前提条件と不変条件

  • node が NULL の場合、何もせず NULL を返す。
  • 再帰的な初期化中にスタックオーバーフローが発生しないよう、check_stack_depth() によるスタック深さの検証を行う。

アルゴリズムまたは振る舞い

1. node が NULL であれば即座に終了する。 2. check_stack_depth() を呼び出し、スタックの安全性を確認する。 3. nodeTag(node) を使用してノードタイプを判定し、対応する ExecInit<Node> 関数（例: ExecInitSeqScan, ExecInitNestLoop 等）を呼び出して PlanState を生成する。 4. 制御ノード、スキャンノード、結合ノード、マテリアライズノードなど、各カテゴリに応じた初期化処理をディスパッチする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execMain.c（メインクエリツリーの初期化）
  • src/backend/executor/execParallel.c
  • src/backend/executor/execPartition.c
  • その他、プランツリーを再帰的に初期化する必要がある箇所。
  • **呼び出し先**:
  • 各ノードタイプに対応する ExecInit<Node> 関数群（例: ExecInitResult, ExecInitSeqScan, ExecInitNestLoop 等）。

関連データ構造

  • Plan: プランナーが生成したプランツリーのノード。
  • PlanState: 実行時に使用されるプランノードの状態管理構造体。
  • EState: クエリ実行全体の実行状態を保持する構造体。

失敗モードと境界条件

  • スタック深さが制限を超えた場合、check_stack_depth() によりエラーが発生する可能性がある。
  • 未知のノードタイプが渡された場合、switch 文のデフォルト動作（またはコンパイラの警告）に依存する。

保守担当者向けメモ

  • 本関数は巨大な switch 文で構成されている。新しいプランノードタイプを PostgreSQL に追加する際は、必ず本関数の switch 文に新しいケースを追加し、対応する ExecInit<Node> 関数を呼び出す必要がある。
