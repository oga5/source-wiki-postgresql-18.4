List

定義

  • 種別: typedef
  • 定義箇所: src/include/foreign/fdwapi.h:78

目的

PostgreSQL のノード構造体やクエリ計画において、複数の要素を保持するための汎用的なリスト構造を定義します。

シグネチャまたは形状

ソースコード上では List 型として定義されています。具体的な内部構造（セルやポインタの構成）は本ヘッダーファイル内では非公開です。

入力と出力

  • **入力**: 任意のデータポインタを格納するリストとして使用されます。
  • **出力**: リスト内の各要素へのアクセスや、リスト全体の走査に使用されます。

前提条件と不変条件

  • PostgreSQL のメモリ管理システム（palloc 等）によって割り当てられたメモリ領域で管理されることが想定されています。
  • リストの操作には PostgreSQL の nodes/pg_list.h 等で提供される標準的なリスト操作関数を使用する必要があります。

アルゴリズムまたは振る舞い

  • 複数の要素を順序付けて保持するコンテナとして機能します。
  • fdwapi.h 内では、特に BeginForeignModify_function の引数 fdw_private として、FDW（外部データラッパー）固有の計画情報を保持するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/file_fdw や contrib/dblink などの各 FDW 実装において、オプションリストやターゲットリストの保持に使用されています。
  • **呼び出し先**: 本シンボル自体はデータ構造の型定義であるため、直接的な関数呼び出し関係はありません。

関連データ構造

  • FdwRoutine: FDW のインターフェースを定義する構造体。
  • ModifyTableState: BeginForeignModify 等の関数で List を受け取る際のコンテキスト情報。

失敗モードと境界条件

  • メモリ不足時には、リスト操作関数がエラーを発生させる可能性があります。
  • 空のリスト（NIL）を渡すべき箇所に不正なポインタを渡すと、セグメンテーション違反や予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本シンボルは PostgreSQL のコアライブラリ（nodes/pg_list.h）で定義されている汎用的なリスト型を指しています。
  • FDW 開発において fdw_private として使用する場合、その中身は各 FDW 実装が独自に定義する構造体や定数のリストとなります。シリアライズが必要なデータが含まれる場合は注意してください。
