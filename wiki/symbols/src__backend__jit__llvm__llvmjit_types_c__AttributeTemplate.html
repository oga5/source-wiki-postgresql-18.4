AttributeTemplate

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_types.c:80

目的

PostgreSQL の JIT コンパイルにおいて、関数属性（呼び出し規約や最適化ヒントなど）を LLVM に伝達するためのテンプレートとして機能します。実行時に呼び出される関数ではなく、LLVM IR 生成時の型情報および属性の参照元として使用されます。

シグネチャまたは形状

Datum AttributeTemplate(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（PostgreSQL の標準的な関数呼び出し規約に従う引数群）
  • 出力: Datum 型（常に PG_RETURN_NULL() により NULL を返す）

前提条件と不変条件

  • AssertVariableIsOfType(&AttributeTemplate, PGFunction) により、自身の型が PGFunction であることが実行時に検証されます。
  • 本関数は、LLVM モジュール内で属性をコピーするための「雛形」としてのみ存在するため、実際の実行ロジックは持ちません。

アルゴリズムまたは振る舞い

1. 自身の型が PGFunction であることをアサーションで確認します。 2. PG_RETURN_NULL() を実行して終了します。 3. LLVM 側では、この関数の属性（llvm_copy_attributes 等を介して取得される情報）が、JIT コンパイルされた他の関数に適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit.c: LLVMGetFunctionType の引数として使用。
  • src/backend/jit/llvm/llvmjit_deform.c および llvmjit_expr.c: llvm_copy_attributes の引数として使用。
  • **呼び出し先**:
  • 本関数自体が実行時に他の関数を呼び出すことはありません。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約を定義するマクロ。
  • LLVMValueRef: src/include/jit/llvmjit.h で定義される、LLVM 上の関数表現。

失敗モードと境界条件

  • 本関数は実行されることを意図していないため、通常の処理フローにおける失敗モードはありません。ただし、Assert が有効なビルドにおいて、型定義が期待と異なる場合はアサーション失敗が発生します。

保守担当者向けメモ

  • 本関数は static にしないでください。static にすると Clang が不要なコードと判断し、ビットコードから削除してしまうため、LLVM が属性情報を取得できなくなります。
  • JIT コンパイルされた関数に新しい属性を適用したい場合は、このテンプレート関数に属性を追加することで、LLVM 側へ伝播させることが可能です。
