file_action_t

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_rewind/filemap.h:16

目的

pg_rewind において、ソースディレクトリとターゲットディレクトリのファイル比較結果に基づき、ターゲット側で実行すべき同期操作を定義するために使用されます。

シグネチャまたは形状

typedef enum {
    FILE_ACTION_UNDECIDED = 0,
    FILE_ACTION_CREATE,
    FILE_ACTION_COPY,
    FILE_ACTION_COPY_TAIL,
    FILE_ACTION_NONE,
    FILE_ACTION_TRUNCATE,
    FILE_ACTION_REMOVE,
} file_action_t;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: file_entry_t 構造体の action メンバとして保持され、同期処理の制御に使用されます。

前提条件と不変条件

  • 列挙値の順序は重要です。decide_file_actions() はこの順序を利用して、ディレクトリ作成から削除まで安全な順序で同期が実行されるようソートを行います。

アルゴリズムまたは振る舞い

各値は以下の操作に対応します：

  • FILE_ACTION_UNDECIDED: 初期状態。
  • FILE_ACTION_CREATE: ディレクトリまたはシンボリックリンクの作成。
  • FILE_ACTION_COPY: ファイル全体のコピー（既存ファイルは上書き）。
  • FILE_ACTION_COPY_TAIL: ファイルの末尾部分のみをコピー。
  • FILE_ACTION_NONE: 操作なし（ただしWALに基づき変更ブロックのコピーが発生する場合あり）。
  • FILE_ACTION_TRUNCATE: 指定サイズへの切り詰め。
  • FILE_ACTION_REMOVE: ファイル、ディレクトリ、またはシンボリックリンクの削除。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_rewind/filemap.c 内の decide_file_actions() 等でアクションの決定に使用されます。
  • 呼び出し先: action_to_str() 関数がこの型を引数として受け取り、デバッグやログ出力用の文字列へ変換します。

関連データ構造

  • file_entry_t: 各ファイルのアクションを保持する構造体。
  • filemap_t: 同期計画全体を管理する構造体。

失敗モードと境界条件

  • FILE_ACTION_UNDECIDED のまま同期プロセスに移行した場合、意図しない動作を引き起こす可能性があります。

保守担当者向けメモ

  • 新しいアクションを追加する場合は、decide_file_actions() におけるソート順序への影響を考慮してください。
  • filemap.c 内の action_to_str 関数も併せて更新する必要があります。
