xlog_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/xlogdesc.c:172

目的

WAL（Write-Ahead Log）レコードの info フィールドに基づき、そのレコードが表す種類を人間が判読可能な文字列ラベルとして特定する。

シグネチャまたは形状

const char *xlog_identify(uint8 info);

入力と出力

  • **入力**: uint8 info - WALレコードの info ビットフィールド。
  • **出力**: const char * - 特定されたレコードタイプを示す文字列（例: "CHECKPOINT_SHUTDOWN"）。該当するものがない場合は NULL を返す。

前提条件と不変条件

  • 入力された info は、XLR_INFO_MASK を用いてマスク処理された後に判定に使用される。
  • 新しいXLOGレコードタイプを追加した場合は、本関数の switch 文にケースを追加する必要がある。

アルゴリズムまたは振る舞い

1. 入力された info に対して ~XLR_INFO_MASK を適用し、不要なビットを除去する。 2. マスク後の値に基づき switch 文で分岐する。 3. XLOG_CHECKPOINT_SHUTDOWN から XLOG_CHECKPOINT_REDO までの各定数に対応する文字列を id に代入する。 4. 確定した文字列ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/access/rmgrlist.h において、PG_RMGR マクロの引数として登録されており、リソースマネージャのインターフェースの一部として呼び出される。
  • **呼び出し先**:
  • 提供されたソースコード内では、他の関数を呼び出していない。

関連データ構造

  • XLR_INFO_MASK: info フィールドからレコードタイプを抽出するために使用されるビットマスク。
  • XLOG_* 定数群: access/xlog.h 等で定義されるWALレコードタイプ識別子。

失敗モードと境界条件

  • info が既知の XLOG_* 定数のいずれにも一致しない場合、id は初期値の NULL のまま返される。

保守担当者向けメモ

  • 本関数はWALの「解釈」のみを目的としており、WALの書き込みやリカバリ処理には関与しない。
  • pg_waldump などのツールがWALレコードを可視化する際に利用されるため、レコードタイプを追加した際は必ず本関数を更新すること。
