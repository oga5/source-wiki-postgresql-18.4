libpqsrv_disconnect

定義

  • 種別: function
  • 定義箇所: src/include/libpq/libpq-be-fe-helpers.h:106

目的

PostgreSQL バックエンドから外部データベースへの接続を安全に終了し、関連するリソース（ファイル記述子）を解放します。

シグネチャまたは形状

static inline void libpqsrv_disconnect(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（終了対象の接続ハンドル。NULL も許容される）
  • **出力**: なし（void）

前提条件と不変条件

  • conn が NULL の場合、本関数は何も行わず即座に復帰します。
  • 接続が確立されている場合、本関数は ReleaseExternalFD() を呼び出してバックエンドのファイル記述子制限を更新します。
  • PG_CATCH() ブロック内での呼び出しを容易にするため、接続が確立されていない（または既に解放済み）状態でも安全に呼び出せる設計となっています。

アルゴリズムまたは振る舞い

1. conn が NULL かどうかを確認し、NULL であれば終了します。 2. ReleaseExternalFD() を呼び出し、バックエンドが管理する外部接続用のファイル記述子枠を解放します。 3. PQfinish(conn) を呼び出し、libpq レベルでの接続終了処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/postgres_fdw/connection.c
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c
  • **呼び出し先**:
  • ReleaseExternalFD()（バックエンドの FD 管理関数）
  • PQfinish()（libpq の接続終了関数）

関連データ構造

  • PGconn: libpq が使用する接続ハンドル。

失敗モードと境界条件

  • conn が NULL の場合、エラーにはならず正常に終了します。
  • PQfinish は内部で接続のクローズを試みますが、本関数自体は戻り値を持たないため、クローズ失敗時のエラーハンドリングは呼び出し元に依存します。

保守担当者向けメモ

  • 本関数は libpq-be-fe-helpers.h 内で static inline として定義されており、バックエンドの FD 管理ルール（AcquireExternalFD との対）を遵守するために必須です。直接 PQfinish を呼び出すのではなく、必ず本関数を使用してください。
