BufferGetPage

定義

  • 種別: function (static inline)
  • 定義箇所: src/include/storage/bufmgr.h:416

目的

バッファIDから、対応するメモリ上のページ（Page）へのポインタを取得する。

シグネチャまたは形状

static inline Page BufferGetPage(Buffer buffer)

入力と出力

  • **入力**: buffer (Buffer 型) - 取得対象のバッファID。
  • **出力**: Page 型のポインタ - バッファが保持するページデータの先頭アドレス。

前提条件と不変条件

  • 入力された buffer は有効なバッファIDである必要がある。
  • 本関数は単なる型変換のラッパーであり、バッファのピン留めやロック状態の検証は行わない。

アルゴリズムまたは振る舞い

1. 内部関数 BufferGetBlock(buffer) を呼び出し、バッファに対応するブロックのメモリ位置を取得する。 2. 取得したブロックポインタを Page 型にキャストして返却する。

呼び出し元と呼び出し先

  • **呼び出し先**: BufferGetBlock
  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • contrib/amcheck/verify_heapam.c
  • contrib/amcheck/verify_nbtree.c
  • contrib/bloom/blinsert.c
  • contrib/bloom/blscan.c
  • contrib/bloom/blutils.c

    • (上記はツールによる直接的な参照の証拠に基づく)

関連データ構造

  • Buffer: バッファを識別する整数型。
  • Page: ページレイアウトを定義するデータ構造（storage/bufpage.h で定義）。

失敗モードと境界条件

  • 無効な buffer IDが渡された場合の挙動は BufferGetBlock の実装に依存する。本関数自体はエラーチェックを行わない。

保守担当者向けメモ

  • 本関数はインライン化されており、パフォーマンス重視のパスで使用される。
  • ページ内容へのアクセス前に、呼び出し側で適切なロック（LockBuffer）が保持されていることを確認すること。
