ExecReScanUnique

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeUnique.c:174

目的

Unique ノードの状態をリセットし、サブプランの再スキャンを可能にします。

シグネチャまたは形状

void ExecReScanUnique(UniqueState *node)

入力と出力

  • **入力**: UniqueState *node （再スキャン対象の Unique ノードの状態構造体）
  • **出力**: なし（副作用としてノードの状態を更新）

前提条件と不変条件

  • 入力される UniqueState は初期化済みであること。
  • Unique ノードは入力がソート済みであることを前提として動作します。

アルゴリズムまたは振る舞い

1. ExecClearTuple を呼び出し、node->ps.ps_ResultTupleSlot をクリアします。これにより、再スキャン後の最初の呼び出しで、入力ストリームの最初のタプルが確実に処理されるようにします。 2. outerPlan->chgParam を確認します。

  • chgParam が NULL の場合、ExecReScan(outerPlan) を呼び出し、サブプランを明示的に再スキャンします。
  • chgParam が NULL でない場合、サブプランは後続の ExecProcNode 呼び出し時に自動的に再スキャンされるため、ここでは何もしません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:282
  • **呼び出し先**:
  • ExecClearTuple
  • outerPlanState (マクロ)
  • ExecReScan (条件付き)

関連データ構造

  • UniqueState: Unique ノードの実行時状態を保持する構造体。
  • PlanState: サブプランの状態を保持する構造体。

失敗モードと境界条件

  • node が NULL の場合の動作は定義されていません。
  • サブプランが NULL の場合、outerPlanState(node) の呼び出しで異常終了する可能性があります。

保守担当者向けメモ

  • 本関数は Unique ノードの再スキャン処理を完結させるためのものです。
  • ExecClearTuple による結果スロットのクリアは、重複排除ロジックが「直前に返したタプル」を正しくリセットするために不可欠です。
