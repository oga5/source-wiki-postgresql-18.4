TmToChar

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/formatting.c:508

目的

日時データ型を文字列へ変換（TO_CHAR）する際、変換対象となる日時情報を保持するための構造体です。

シグネチャまたは形状

typedef struct TmToChar
{
	struct fmt_tm tm;			/* almost the classic 'tm' struct */
	fsec_t		fsec;			/* fractional seconds */
	const char *tzn;			/* timezone */
} TmToChar;

入力と出力

本構造体は、日時変換関数における入力データとして使用されます。

  • tm: 日時要素（年、月、日、時、分、秒など）を格納する構造体。
  • fsec: 秒の小数部（fsec_t 型）。
  • tzn: タイムゾーン名を示す文字列ポインタ。

前提条件と不変条件

  • tm フィールドは fmt_tm 構造体として定義されており、標準的な tm 構造体に準拠した日時情報を保持する必要があります。

アルゴリズムまたは振る舞い

TO_CHAR 系の関数において、入力された日時データを FormatNode のリストに従って解析・整形するためのコンテナとして機能します。DCH_to_char や datetime_to_char_body などの関数内で、この構造体に格納された値がフォーマット文字列に基づいて出力バッファへ書き出されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/utils/adt/formatting.c:2502 (コメント内での言及)
  • src/backend/utils/adt/formatting.c:2507 (DCH_to_char の引数)
  • src/backend/utils/adt/formatting.c:3931 (datetime_to_char_body の引数)
  • src/backend/utils/adt/formatting.c:4005, 4040, 4081 (ローカル変数としての宣言)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • fmt_tm: TmToChar 内部で日時情報を保持するために使用される構造体。
  • FormatNode: TO_CHAR 変換時に適用されるフォーマット解析ツリー。

失敗モードと境界条件

  • tzn が指す文字列が有効なメモリ領域を指していない場合、または tm フィールドに不正な日時値が含まれている場合、変換処理の結果が未定義となる可能性があります。

保守担当者向けメモ

  • 本構造体は formatting.c 内の TO_CHAR 処理に特化しています。日時フォーマットの拡張や変更を行う際は、fmt_tm 構造体との整合性に注意してください。
