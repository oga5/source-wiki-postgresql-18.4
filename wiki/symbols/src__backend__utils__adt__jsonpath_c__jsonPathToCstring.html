jsonPathToCstring

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonpath.c:212

目的

バイナリ形式の JsonPath 構造体を、人間が読める文字列形式（C文字列）へ変換します。

シグネチャまたは形状

char *jsonPathToCstring(StringInfo out, JsonPath *in, int estimated_len)

入力と出力

  • **入力**:
  • out: 出力用バッファ（StringInfo）。NULL の場合は内部で新規作成されます。
  • in: 変換対象の JsonPath 構造体へのポインタ。
  • estimated_len: バッファ拡張時に使用する推定長。
  • **出力**:
  • 変換後の文字列データ（out->data）。

前提条件と不変条件

  • in は有効な JsonPath 構造体を指している必要があります。
  • out が NULL の場合、内部で initStringInfo が呼び出され、呼び出し元で解放が必要なバッファが生成されます。

アルゴリズムまたは振る舞い

1. out が NULL の場合、StringInfoData を初期化します。 2. estimated_len に基づき、enlargeStringInfo でバッファを拡張します。 3. in->header を確認し、JSONPATH_LAX フラグが立っていない場合は、文字列の先頭に "strict " を付与します。 4. jspInit を使用してバイナリバッファから JsonPathItem を初期化します。 5. printJsonPathItem を呼び出し、バイナリ表現を文字列として out に書き出します。 6. 最終的な文字列データ out->data を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • jsonpath_out (src/backend/utils/adt/jsonpath.c:138)
  • jsonpath_send (src/backend/utils/adt/jsonpath.c:155)
  • **呼び出し先**:
  • initStringInfo
  • enlargeStringInfo
  • appendStringInfoString
  • jspInit
  • printJsonPathItem

関連データ構造

  • JsonPath: パス式のバイナリ表現を保持する構造体。
  • StringInfo: 文字列構築用の可変長バッファ構造体。
  • JsonPathItem: バイナリから復元されたパス式の各要素を扱う構造体。

失敗モードと境界条件

  • out が NULL の場合、メモリ不足が発生すると initStringInfo 内でエラーが発生する可能性があります。
  • estimated_len が不適切な場合、バッファの再割り当てが発生しますが、機能的な失敗には直結しません。

保守担当者向けメモ

  • 本関数は jsonpath 型の出力処理の主要なエントリポイントです。
  • 文字列の再構築ロジックを変更する場合は、printJsonPathItem の実装と整合性を取る必要があります。
