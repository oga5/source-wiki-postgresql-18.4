remove_target

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:132

目的

file_entry_t で指定されたターゲットファイルシステム上のオブジェクト（ディレクトリ、ファイル、またはシンボリックリンク）を削除します。

シグネチャまたは形状

void remove_target(file_entry_t *entry)

入力と出力

  • **入力**: file_entry_t *entry - 削除対象のファイル情報を含む構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • entry->action が FILE_ACTION_REMOVE であること。
  • entry->target_exists が真であること。
  • 上記条件が満たされない場合、アサーションエラーが発生します。

アルゴリズムまたは振る舞い

1. entry->target_type に基づき、以下のいずれかの関数を呼び出して削除を実行します。

  • FILE_TYPE_DIRECTORY: remove_target_dir(entry->path) を呼び出します。
  • FILE_TYPE_REGULAR: remove_target_file(entry->path, false) を呼び出します。
  • FILE_TYPE_SYMLINK: remove_target_symlink(entry->path) を呼び出します。

    • 2. FILE_TYPE_UNDEFINED の場合は pg_fatal を呼び出し、エラーメッセージを出力してプロセスを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c:615 から呼び出されていることが確認されています。
  • **呼び出し先**:
  • remove_target_dir
  • remove_target_file
  • remove_target_symlink
  • pg_fatal

関連データ構造

  • file_entry_t: 削除対象のパス、アクション、タイプ、存在フラグを保持する構造体。

失敗モードと境界条件

  • entry->target_type が FILE_TYPE_UNDEFINED の場合、pg_fatal によりプログラムが異常終了します。
  • 呼び出し先の各削除関数（remove_target_dir 等）が失敗した場合の挙動は、それらの関数の実装に依存します。

保守担当者向けメモ

  • 本関数は file_entry_t の型に応じて適切な削除関数へディスパッチする役割を担います。新しいファイルタイプを追加する場合は、switch 文にケースを追加してください。
  • remove_target_file の第2引数に false を渡している点は、この関数が通常のファイル削除を意図していることを示しています。
