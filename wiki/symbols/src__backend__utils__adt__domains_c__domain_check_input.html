domain_check_input

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/domains.c:137

目的

ドメイン型に対して定義された制約（NOT NULL および CHECK 制約）を検証し、入力値がドメインの定義に適合しているかを判定する。

シグネチャまたは形状

static void domain_check_input(Datum value, bool isnull, DomainIOData *my_extra, Node *escontext)

入力と出力

  • value: 検証対象のデータ（Datum）。
  • isnull: 値が NULL かどうかを示すフラグ。
  • my_extra: ドメインの制約情報や実行コンテキストを保持する DomainIOData 構造体。
  • escontext: エラー報告用のノード（Node *）。NULL の場合は例外を発生させ、非 NULL の場合はエラー情報を格納する。
  • 戻り値: なし（成功時は正常終了、失敗時はエラー報告または例外発生）。

前提条件と不変条件

  • my_extra は初期化済みであること。
  • UpdateDomainConstraintRef により、制約情報が最新の状態に更新されていること。
  • CHECK 制約の評価には ExprContext を使用し、評価終了後には ReScanExprContext を呼び出してリソースを解放すること。

アルゴリズムまたは振る舞い

1. UpdateDomainConstraintRef を呼び出し、最新の制約リストを取得する。 2. 制約リストを順次走査する。

  • DOM_CONSTRAINT_NOTNULL: isnull が真であればエラーを報告し、検証を終了する。
  • DOM_CONSTRAINT_CHECK:
  • ExprContext が未作成であれば、my_extra->mcxt 上で作成する。
  • value を読み取り専用の展開オブジェクトとして econtext->domainValue_datum に設定する。
  • ExecCheck を実行し、式が偽を返せばエラーを報告し、検証を終了する。

    • 3. 最後に ReScanExprContext を呼び出し、評価に使用したメモリコンテキストをリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/domains.c 内の domain_in、domain_recv、および domain_check 等から呼び出される。
  • **呼び出し先**:
  • UpdateDomainConstraintRef
  • MemoryContextSwitchTo
  • CreateStandaloneExprContext
  • MakeExpandedObjectReadOnly
  • ExecCheck
  • errsave
  • ReScanExprContext

関連データ構造

  • DomainIOData: ドメインの制約情報、キャッシュされた式状態、実行コンテキストを保持。
  • DomainConstraintState: 個別の制約（NOT NULL または CHECK）の定義と実行状態。
  • ExprContext: 式評価のための実行環境。

失敗モードと境界条件

  • NOT NULL 制約違反: ERRCODE_NOT_NULL_VIOLATION を発生させる。
  • CHECK 制約違反: ERRCODE_CHECK_VIOLATION を発生させる。
  • 未知の制約タイプ: elog(ERROR) によりシステムエラーとして処理される。
  • escontext が提供されている場合、例外を投げずにエラー情報を記録して処理を継続（または中断）する。

保守担当者向けメモ

  • econtext は DomainIOData 内でキャッシュされるが、評価ごとに ReScanExprContext を呼び出すことでメモリリークを防止している。
  • domainValue_datum への設定時に MakeExpandedObjectReadOnly を使用しており、入力値の不変性を保証している。
