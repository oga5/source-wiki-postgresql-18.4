SerializedActiveRelMaps

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/relmapper.c:101

目的

並列ワーカーに対して、トランザクション内で保留中（pending）となっている共有およびローカルのリレーションマッピング状態をシリアライズ・転送するために使用されます。

シグネチャまたは形状

typedef struct SerializedActiveRelMaps
{
	RelMapFile	active_shared_updates;
	RelMapFile	active_local_updates;
} SerializedActiveRelMaps;

入力と出力

  • **入力**: 本構造体はメモリ上の領域を指し示すポインタとして扱われ、シリアライズされたデータが書き込まれるか、あるいは既存のデータから読み取られます。
  • **出力**: sizeof(SerializedActiveRelMaps) を通じて、シリアライズに必要なメモリサイズが提供されます。

前提条件と不変条件

  • 本構造体は、トランザクション内で変更されたマッピング状態を保持します。
  • 含まれる RelMapFile 構造体は、有効なマジック番号と整合性チェックのための CRC を保持している必要があります。

アルゴリズムまたは振る舞い

1. **サイズ計算**: sizeof(SerializedActiveRelMaps) を用いて、並列処理用の共有メモリ領域のサイズを決定します。 2. **シリアライズ/リストア**: ポインタキャスト（SerializedActiveRelMaps *relmaps = (SerializedActiveRelMaps *) startAddress）を通じて、メモリ上の特定アドレスから構造体へアクセスし、マッピング状態のコピーまたは復元を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/relmapper.c 内の並列クエリ実行に関連する関数群（シリアライズおよびリストア処理）。
  • **呼び出し先**: 直接的な関数呼び出しは行わず、データ保持のためのコンテナとして機能します。

関連データ構造

  • RelMapFile: 本構造体が保持する、共有およびローカルマッピングの実際のデータ構造。

失敗モードと境界条件

  • **メモリ配置**: startAddress が SerializedActiveRelMaps のアライメント要件を満たしていない場合、アクセス時に問題が発生する可能性があります。
  • **データ破損**: 復元された RelMapFile 内の CRC が一致しない場合、マッピングの整合性が保証されません。

保守担当者向けメモ

  • 本構造体は並列クエリのインフラに依存しています。RelMapFile の定義が変更された場合、本構造体のサイズやレイアウトも影響を受ける可能性があるため注意してください。
