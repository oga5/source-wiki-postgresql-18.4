DCHCacheEntry

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/formatting.c:394

目的

日時フォーマット文字列の解析結果（FormatNode のリスト）をメモリ上に保持し、再利用することで、TO_CHAR や TO_DATE 等の変換処理における解析コストを削減するためのキャッシュエントリ構造体です。

シグネチャまたは形状

typedef struct {
    FormatNode  format[DCH_CACHE_SIZE + 1];
    char        str[DCH_CACHE_SIZE + 1];
    bool        std;
    bool        valid;
    int         age;
} DCHCacheEntry;

入力と出力

本構造体自体はデータ保持用であり、直接的な入出力関数を持ちません。キャッシュの操作は DCH_cache_fetch 等の関数を通じて行われます。

前提条件と不変条件

  • format 配列および str 配列のサイズは DCH_CACHE_SIZE + 1 で固定されています。
  • valid フラグがキャッシュエントリの有効性を示します。
  • age フィールドはキャッシュの置換アルゴリズム（LRU 等）の管理に使用されます。

アルゴリズムまたは振る舞い

  • フォーマット文字列がキャッシュサイズを超える場合、キャッシュは使用されず毎回解析が行われます。
  • キャッシュ検索時に一致する文字列と std フラグが見つかった場合、解析済みの FormatNode ツリーが再利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/formatting.c 内のキャッシュ管理関数群（DCH_cache_fetch, DCH_cache_search, DCH_cache_getnew）から参照されます。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、特定の関数を呼び出すことはありません。

関連データ構造

  • FormatNode: フォーマット文字列の解析結果を格納するノード。
  • DCHCache: DCHCacheEntry を保持する静的配列（DCHCacheEntry *DCHCache[DCH_CACHE_ENTRIES]）。

失敗モードと境界条件

  • DCH_CACHE_SIZE を超えるフォーマット文字列はキャッシュされません。
  • メモリ不足時には MemoryContextAllocZero による割り当てが失敗する可能性があります。

保守担当者向けメモ

  • 新しいフォーマット指定子を追加する場合は、本構造体自体ではなく、DCH_keywords テーブルや DCH_poz 列挙型の更新が必要です。
  • キャッシュのヒット率が低い場合は、DCH_CACHE_SIZE や DCH_CACHE_ENTRIES の定数を見直す必要があります。
