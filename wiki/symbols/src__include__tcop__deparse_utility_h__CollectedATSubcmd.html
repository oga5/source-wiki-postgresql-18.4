CollectedATSubcmd

定義

  • 種別: struct
  • 定義箇所: src/include/tcop/deparse_utility.h:38

目的

ALTER TABLE コマンド実行時に発生する個別のサブコマンド（列の変更、制約の追加・削除など）を追跡・記録するためのデータ構造です。

シグネチャまたは形状

typedef struct CollectedATSubcmd
{
	ObjectAddress address;
	Node	   *parsetree;
} CollectedATSubcmd;

入力と出力

  • **入力**: ObjectAddress（影響を受けるオブジェクトの識別子）および Node（サブコマンドに対応するパースツリー）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • address フィールドには、列、制約、インデックスなど、ALTER TABLE によって影響を受ける対象オブジェクトの情報が格納されます。
  • parsetree は、PostgreSQLのパースツリーノードを指す必要があります。

アルゴリズムまたは振る舞い

本構造体はロジックを持たず、ユーティリティコマンドの実行過程で生成される情報を保持するコンテナとして機能します。イベントトリガーや監査ログ等の目的で、コマンドの構造を後続処理で再利用可能な形式（Deparse可能）で保持するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/commands/event_trigger.c: サブコマンドのインスタンス生成および割り当てに使用。
  • src/test/modules/test_ddl_deparse/test_ddl_deparse.c: テストコード内での参照。
  • **呼び出し先**: なし（データ構造のため）。

関連データ構造

  • ObjectAddress: データベースオブジェクトを一意に特定するために使用。
  • Node: パースツリーの基底型。
  • CollectedCommand: 本構造体を内部で保持し、コマンド全体を管理する上位の構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc 失敗時）に例外が発生する可能性があります。
  • parsetree が NULL の場合、後続の Deparse 処理で不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は共用体の一部として使用されることが多いため、アクセス時には必ず親となる CollectedCommand の type フィールドを確認してください。
  • ALTER TABLE の複雑なネスト構造を扱うため、拡張機能や監査ログの実装において、この構造体が正しく初期化されているかを確認することが重要です。
