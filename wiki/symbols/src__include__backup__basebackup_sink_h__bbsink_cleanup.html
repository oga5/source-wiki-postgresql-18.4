bbsink_cleanup

定義

  • 種別: function
  • 定義箇所: src/include/backup/basebackup_sink.h:264

目的

バックアップ処理の終了時、またはエラー発生時に、bbsink オブジェクトおよびそのチェーンに関連付けられたリソースを解放・クリーンアップする。

シグネチャまたは形状

static inline void bbsink_cleanup(bbsink *sink)

入力と出力

  • **入力**: bbsink *sink - クリーンアップ対象のシンク構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • sink は NULL であってはならない（Assert による検証）。
  • sink->bbs_ops が有効であり、かつ cleanup コールバック関数が実装されている必要がある。

アルゴリズムまたは振る舞い

1. 引数 sink が NULL でないことを確認する。 2. sink が保持する bbs_ops テーブル内の cleanup 関数ポインタを呼び出し、具体的なシンク実装に応じた解放処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c:1063
  • src/backend/backup/basebackup_sink.c:124（再帰的なチェーン解放時）
  • **呼び出し先**:
  • sink->bbs_ops->cleanup(sink)（各シンク実装固有のクリーンアップ関数）

関連データ構造

  • bbsink: バックアップデータの転送チェーンを構成する基本構造体。
  • bbsink_ops: 各シンクの振る舞いを定義する関数テーブル。

失敗モードと境界条件

  • sink が NULL の場合、アサーション違反によりプロセスが停止する。
  • sink->bbs_ops が初期化されていない場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、シンクチェーンの末端から順次呼び出されることで、チェーン全体のメモリ解放を確実に行う設計となっている。
  • 新しいシンク型を追加する際は、必ず bbsink_ops 内の cleanup 関数を適切に実装し、割り当てたリソースを確実に解放すること。
