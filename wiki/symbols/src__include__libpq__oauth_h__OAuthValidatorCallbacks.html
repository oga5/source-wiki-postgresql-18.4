OAuthValidatorCallbacks

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/oauth.h:76

目的

外部のOAuthバリデータモジュールがPostgreSQLの認証プロセスと統合するために実装すべきコールバック関数のインターフェースを定義します。

シグネチャまたは形状

typedef struct OAuthValidatorCallbacks
{
    uint32        magic;            /* PG_OAUTH_VALIDATOR_MAGIC に設定必須 */
    ValidatorStartupCB startup_cb;
    ValidatorShutdownCB shutdown_cb;
    ValidatorValidateCB validate_cb;
} OAuthValidatorCallbacks;

入力と出力

  • **入力**: 構造体の各フィールドには、モジュールが提供する関数ポインタを格納します。
  • **出力**: なし（構造体自体が関数ポインタのコンテナとして機能します）。

前提条件と不変条件

  • magic フィールドには、互換性検証のために PG_OAUTH_VALIDATOR_MAGIC を設定する必要があります。
  • 構造体は、モジュール初期化関数（OAuthValidatorModuleInit）を通じてコアシステムへ提供されます。

アルゴリズムまたは振る舞い

  • startup_cb: モジュールの初期化時に呼び出されます。
  • shutdown_cb: モジュールの終了時やアンロード時に呼び出されます。
  • validate_cb: 実際のOAuthトークン検証処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c 内のバリデータ管理ロジックが、ロードされたモジュールからこの構造体を取得します。
  • **呼び出し先**: 構造体内の各コールバック関数は、認証フローの各段階でコアシステムから呼び出されます。
  • **テスト実装**: src/test/modules/oauth_validator/ 配下の各テストモジュール（fail_validator.c, magic_validator.c, validator.c）が本構造体を実装しています。

関連データ構造

  • ValidatorStartupCB, ValidatorShutdownCB, ValidatorValidateCB: 各コールバックの関数型定義。
  • OAuthValidatorModuleInit: 本構造体を返す初期化関数の型定義。

失敗モードと境界条件

  • magic 値が一致しない場合、コアシステムはモジュールのロードを拒否します。
  • 各コールバック関数が NULL である場合の挙動は、実装側の初期化ロジックに依存します。

保守担当者向けメモ

  • 本構造体はABIの安定性を維持するために magic フィールドによるバージョン管理を行っています。フィールドの追加や変更を行う際は、PG_OAUTH_VALIDATOR_MAGIC の更新を検討してください。
