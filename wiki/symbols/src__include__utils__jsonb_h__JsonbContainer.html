JsonbContainer

定義

  • 種別: struct
  • 定義箇所: src/include/utils/jsonb.h:190

目的

PostgreSQLの jsonb 型における、オンディスク上のデータコンテナ構造を定義します。JSONオブジェクトや配列の要素を効率的に格納・走査するためのバイナリ表現を提供します。

シグネチャまたは形状

typedef struct JsonbContainer
{
    uint32      header;
    JEntry      children[FLEXIBLE_ARRAY_MEMBER];
} JsonbContainer;

入力と出力

  • **入力**: header フィールドにより要素数やフラグを保持し、children 配列を通じて各要素のメタデータ（型、位置、長さ）を管理します。
  • **出力**: 構造体直後のメモリ領域に、各子ノードの実際のデータが格納されます。

前提条件と不変条件

  • Jsonb 構造体の一部として保持されます。
  • children 配列は可変長メンバー（FLEXIBLE_ARRAY_MEMBER）として定義されており、header で指定された数だけ要素が存在します。
  • 構造体全体は4バイト境界でアライメントされます。

アルゴリズムまたは振る舞い

  • header フィールドには、コンテナ内の要素数またはキー/値ペアの数、およびコンテナの属性を示すフラグが格納されます。
  • 各子ノードのデータは、children 配列内の JEntry が指し示すオフセットと長さを基に、コンテナの末尾領域から読み取られます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照）**:
  • contrib/jsonb_plperl/jsonb_plperl.c: Jsonb_to_SV
  • contrib/jsonb_plpython/jsonb_plpython.c: PLyObject_FromJsonbContainer
  • src/backend/utils/adt/jsonb.c: JsonbToCStringWorker, JsonbContainerTypeName, JsonbToCString, JsonbToCStringIndent, JsonbExtractScalar
  • **呼び出し先**:
  • JEntry 構造体（children 配列の要素として利用）。

関連データ構造

  • Jsonb: JsonbContainer を保持するルート構造体。
  • JEntry: 各要素の型と位置情報を管理するヘッダー。
  • JsonbValue: メモリ内での操作に使用される汎用構造体。

失敗モードと境界条件

  • JEntry のオフセット計算ロジックが複雑であり、誤ったオフセット参照はメモリ破壊を引き起こす可能性があります。
  • JGIN_MAXLENGTH を超える値はハッシュ化されるため、直接的な値の取得には注意が必要です。

保守担当者向けメモ

  • 可変長データ領域へのアクセス時には、パディングを考慮したアライメントの確認が必須です。
  • JsonbContainer.header フィールドのビット定義については、src/backend/utils/adt/jsonb_util.c の実装詳細を参照してください。
