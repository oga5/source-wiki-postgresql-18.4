test_dsa_resowners

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_dsa/test_dsa.c:65

目的

PostgreSQLのDSA（動的共有メモリ領域）において、ResourceOwnerの切り替えおよび解放に伴うメモリ管理の整合性を検証する。

シグネチャまたは形状

Datum test_dsa_resowners(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（SQL関数として呼び出し可能）
  • 出力: PG_RETURN_VOID() により値を返さない。

前提条件と不変条件

  • LWLockNewTrancheId() によって確保されたロックトランチは、テスト終了後もプロセス内でリークする。
  • dsa_allocate() で確保したメモリは、ResourceOwner の解放プロセスを通じて適切に処理される必要がある。

アルゴリズムまたは振る舞い

1. LWLockNewTrancheId() でロックトランチを確保し、dsa_create() でDSA領域を生成する。 2. ResourceOwnerCreate() で子リソースオーナーを作成し、CurrentResourceOwner を一時的に切り替える。 3. 子オーナーのコンテキスト下で dsa_allocate() を10,000回実行し、データを書き込む。 4. 最初の500個の割り当てを dsa_free() で明示的に解放する。 5. CurrentResourceOwner を元のオーナーに戻し、子オーナーに対して ResourceOwnerRelease を各フェーズ（BEFORE_LOCKS, LOCKS, AFTER_LOCKS）で実行し、最後に ResourceOwnerDelete() で削除する。 6. dsa_detach() でDSA領域を破棄する。

呼び出し元と呼び出し先

  • 呼び出し元: SQLインターフェース経由（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先: LWLockNewTrancheId, LWLockRegisterTranche, dsa_create, ResourceOwnerCreate, dsa_allocate, dsa_get_address, dsa_free, ResourceOwnerRelease, ResourceOwnerDelete, dsa_detach。

関連データ構造

  • dsa_area: DSA管理構造体。
  • dsa_pointer: DSA領域内のメモリ参照用ポインタ。
  • ResourceOwner: リソースの生存期間を管理するオブジェクト。

失敗モードと境界条件

  • 10,000回の割り当てと500回の解放という大量の操作を行うため、DSAのセグメント拡張ロジックに負荷がかかる。
  • ResourceOwner の解放順序が正しくない場合、メモリリークや不正なメモリ参照が発生する可能性がある。

保守担当者向けメモ

  • 本関数はテスト専用であり、LWLockNewTrancheId() のリークは意図的な仕様である。
  • ResourceOwner の解放フェーズは、PostgreSQLのバックエンドにおけるリソース管理の標準的な手順を模倣している。
