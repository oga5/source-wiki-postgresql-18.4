bool_int4

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int.c:400

目的

boolean 型の値を int4 型（32ビット整数）に変換します。

シグネチャまたは形状

Datum bool_int4(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される boolean 型の Datum。
  • **出力**: int4 型の Datum。true の場合は 1 を、false の場合は 0 を返します。

前提条件と不変条件

  • 入力引数は PG_GETARG_BOOL(0) を介して取得可能であること。
  • 戻り値は PG_RETURN_INT32 マクロを使用して構築されること。

アルゴリズムまたは振る舞い

1. PG_GETARG_BOOL(0) を呼び出し、入力値が false かどうかを判定します。 2. 入力値が false であれば 0 を返します。 3. 入力値が true であれば 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c:5214 において、式評価の過程で DirectFunctionCall1 を介して呼び出されています。
  • **呼び出し先**:
  • PG_GETARG_BOOL および PG_RETURN_INT32 マクロ（PostgreSQLの関数マネージャインターフェース）。

関連データ構造

  • Datum: PostgreSQLの汎用データ表現型。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。入力が有効な boolean 型であれば常に成功します。

保守担当者向けメモ

  • 本関数は単純な型変換を行うユーティリティです。将来的に bool から他の整数型への変換が必要になった場合、同様のパターンに従うことが推奨されます。
