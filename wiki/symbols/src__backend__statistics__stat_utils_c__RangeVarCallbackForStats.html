RangeVarCallbackForStats

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/stat_utils.c:130

目的

統計情報の更新操作において、対象リレーションの解決、権限チェック、およびデッドロックを回避するための適切なロック取得を行うコールバック関数です。

シグネチャまたは形状

void RangeVarCallbackForStats(const RangeVar *relation, Oid relId, Oid oldRelId, void *arg)

入力と出力

  • relation: 解決対象の RangeVar 構造体。
  • relId: 現在のルックアップで特定されたリレーションのOID。
  • oldRelId: 前回のルックアップで特定されたリレーションのOID。
  • arg: ロック済みリレーションのOIDを保持する Oid * 型のポインタ。
  • 出力: なし（副作用としてロックの取得/解放およびエラー報告を行う）。

前提条件と不変条件

  • arg は有効な Oid へのポインタであること。
  • 統計情報の更新対象は、RELKIND_RELATION, RELKIND_MATVIEW, RELKIND_FOREIGN_TABLE, RELKIND_PARTITIONED_TABLE のいずれかであること。
  • 共有リレーションの統計更新は許可されない。

アルゴリズムまたは振る舞い

1. **ロックの解放**: relId が oldRelId と異なり、かつ *locked_oid が有効な場合、以前取得した ShareUpdateExclusiveLock を解放する。 2. **リレーション解決**: relId がインデックス（RELKIND_INDEX 等）である場合、IndexGetRelation を用いて親テーブルのOIDを取得する。 3. **並行操作の検出**: relId == oldRelId の場合、インデックスの削除や再作成といった並行操作が発生していないか検証し、矛盾があればエラーを投げる。 4. **権限チェック**: 対象リレーションの所有者であるか、または MAINTAIN 権限を有しているかを確認する。 5. **ロック取得**: インデックスの親テーブルに対して ShareUpdateExclusiveLock を取得し、*locked_oid を更新する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/statistics/attribute_stats.c
  • src/backend/statistics/relation_stats.c
  • 呼び出し先:
  • UnlockRelationOid, LockRelationOid
  • IndexGetRelation
  • SearchSysCache1, ReleaseSysCache
  • object_ownercheck, pg_class_aclcheck

関連データ構造

  • RangeVar: リレーション名とスキーマ情報を保持。
  • Form_pg_class: システムカタログ pg_class の行構造。

失敗モードと境界条件

  • 対象リレーションが存在しない場合、処理を中断する。
  • サポート外の relkind（インデックス単体など）や共有リレーションに対しては ERROR を発行する。
  • 権限不足時は aclcheck_error を発行する。
  • 並行操作によりインデックスが削除・再作成された場合、デッドロック回避のためエラーを投げる。

保守担当者向けメモ

  • 本関数は RangeVarGetRelidExtended 等のコールバックとして使用されることを想定しています。
  • デッドロックを避けるため、インデックスではなく必ず親テーブルに対してロックを取得する設計になっています。
  • relId == oldRelId 時のチェックは、カタログの不整合を検知するための安全装置です。
