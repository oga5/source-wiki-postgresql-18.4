xl_standby_locks

定義

  • **種別**: struct
  • **定義箇所**: src/include/storage/standbydefs.h:38

目的

ホットスタンバイモードにおいて、プライマリサーバーからスタンバイサーバーへ、競合解決に必要なロック情報を伝播させるためのデータ構造です。

シグネチャまたは形状

typedef struct xl_standby_locks
{
    int            nlocks;
    xl_standby_lock locks[FLEXIBLE_ARRAY_MEMBER];
} xl_standby_locks;

入力と出力

  • **入力**: WAL レコードとして書き出される際、nlocks にロックの個数が設定され、locks 配列に具体的なロック情報が格納されます。
  • **出力**: スタンバイ側で WAL を再生（Redo）する際、この構造体を参照して共有メモリ上のロックテーブルが更新されます。

前提条件と不変条件

  • FLEXIBLE_ARRAY_MEMBER を使用した可変長構造体です。
  • メモリレイアウトを扱う際は、offsetof(xl_standby_locks, locks) 等を用いてオフセットを計算する必要があります。

アルゴリズムまたは振る舞い

1. **記録**: プライマリ側でロック情報を本構造体に詰め、WAL レコードとして出力します。 2. **再生**: スタンバイ側で XLogReader がレコードを読み込み、standby_redo 関数が本構造体を参照してスタンバイ側の共有メモリ状態を同期します。

呼び出し元と呼び出し先

  • **呼び出し先**:

  • XLogRecGetData (WAL 再生時)
  • ※具体的な関数呼び出しの連鎖については、提供されたソース情報からはこれ以上の詳細は不明です。

関連データ構造

  • xl_standby_lock: locks 配列の各要素の型。
  • XLOG_STANDBY_LOCK: 本構造体を使用する WAL レコードタイプ。

失敗モードと境界条件

  • 可変長構造体であるため、メモリ確保やポインタ演算の誤りはバッファオーバーランを引き起こす可能性があります。
  • nlocks の値と実際の配列サイズが不整合を起こさないよう、WAL 記録時のデータ構築に依存します。

保守担当者向けメモ

  • 本構造体は src/include/storage/standbydefs.h で定義されています。
  • 構造体のサイズ計算には offsetof マクロを適切に使用してください。
  • スタンバイ側のリカバリ処理（standby_redo）において、共有メモリの整合性を保つための重要なインターフェースです。
