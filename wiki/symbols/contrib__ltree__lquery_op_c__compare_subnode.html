compare_subnode

定義

  • 種別: function
  • 定義箇所: contrib/ltree/lquery_op.c:43

目的

LVAR_SUBLEXEME フラグが有効な場合に、ltree のレベル内の各ラベルと lquery の部分文字列（レキセム）を照合し、すべてのクエリ要素がツリーノード内に存在するかを判定する。

シグネチャまたは形状

bool compare_subnode(ltree_level *t, char *qn, int len, bool prefix, bool ci)

入力と出力

  • **入力**:
  • t: 照合対象の ltree_level 構造体。
  • qn: クエリ文字列のポインタ。
  • len: クエリ文字列の長さ。
  • prefix: プレフィックスマッチングを行うかどうかのフラグ。
  • ci: 大文字小文字を区別しない（case-insensitive）比較を行うかどうかのフラグ。
  • **出力**:
  • bool: すべてのクエリレキセムがツリーノード内のいずれかのラベルと一致した場合は true、それ以外は false。

前提条件と不変条件

  • t は有効な ltree_level を指している必要がある。
  • getlexeme 関数を用いて、クエリおよびツリーノード内のラベルを順次抽出する。

アルゴリズムまたは振る舞い

1. クエリ文字列 qn から getlexeme を使用してレキセムを一つずつ抽出する。 2. 抽出した各クエリレキセムに対し、ツリーノード t 内の全ラベルを走査する。 3. ltree_label_match を呼び出し、現在のクエリレキセムとツリーラベルが一致するか判定する。 4. クエリレキセムがツリー内のいずれかのラベルと一致すれば、次のクエリレキセムの評価へ進む。 5. 一致するものがない場合、直ちに false を返す。 6. すべてのクエリレキセムが一致した場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/lquery_op.c:183
  • contrib/ltree/ltxtquery_op.c:68
  • **呼び出し先**:
  • getlexeme (ソース内での直接参照)
  • ltree_label_match (ソース内での直接参照)

関連データ構造

  • ltree_level: ltree の階層構造を保持する構造体。

失敗モードと境界条件

  • クエリレキセムがツリーノード内のどのラベルとも一致しない場合、false を返す。
  • getlexeme が NULL を返した時点でループが終了する。

保守担当者向けメモ

  • 本関数は LVAR_SUBLEXEME 処理のコアであり、ltree_label_match によるラベル比較の成否に依存する。
  • パフォーマンスは ltree_label_match の計算コストに左右される。
