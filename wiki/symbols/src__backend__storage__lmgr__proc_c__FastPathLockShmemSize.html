FastPathLockShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/proc.c:114

目的

PostgreSQLの共有メモリにおいて、ファストパスロック（Fast-path lock）機構をサポートするために必要なメモリサイズを算出する。

シグネチャまたは形状

Size FastPathLockShmemSize(void)

入力と出力

  • **入力**: なし（グローバルな設定値 MaxBackends, NUM_AUXILIARY_PROCS, max_prepared_xacts を参照）
  • **出力**: Size 型のバイト数。全プロセス（バックエンド、補助プロセス、準備済みトランザクション）のファストパスロック用配列に必要な合計メモリ量。

前提条件と不変条件

  • MaxBackends、NUM_AUXILIARY_PROCS、max_prepared_xacts が初期化されていること。
  • 各バックエンドのロックビット配列とリレーションID配列は、MAXALIGN によって適切にアライメントされていること。

アルゴリズムまたは振る舞い

1. TotalProcs を MaxBackends + NUM_AUXILIARY_PROCS + max_prepared_xacts として算出する。 2. バックエンドごとのファストパスロックビット配列サイズ (fpLockBitsSize) を FastPathLockGroupsPerBackend * sizeof(uint64) で計算し、アライメントする。 3. バックエンドごとのリレーションIDスロットサイズ (fpRelIdSize) を FastPathLockSlotsPerBackend() * sizeof(Oid) で計算し、アライメントする。 4. TotalProcs に、上記2つのサイズの合計を乗算し、必要な総バイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/proc.c 内の共有メモリサイズ計算ロジック（add_size 呼び出し箇所）。
  • **呼び出し先**:
  • add_size (PostgreSQLのメモリ計算用ユーティリティ)
  • mul_size (PostgreSQLのメモリ計算用ユーティリティ)
  • FastPathLockSlotsPerBackend() (インライン関数またはマクロと推測されるが、本ソース内での定義詳細は不明)

関連データ構造

  • PGPROC（共有メモリ上のプロセス管理構造体）
  • Oid（リレーションID）

失敗モードと境界条件

  • add_size や mul_size はオーバーフローをチェックする設計となっている。
  • TotalProcs が 0 の場合、サイズは 0 を返す。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定する重要な計算の一部である。FastPathLockGroupsPerBackend や FastPathLockSlotsPerBackend() の定義が変更された場合、必ず本関数の計算結果に影響が出るため注意すること。
