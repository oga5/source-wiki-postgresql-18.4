ReadReplicationSlotCmd

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/replnodes.h:106

目的

レプリケーションスロットの情報を読み取るためのコマンドパラメータを保持するパースノードです。

シグネチャまたは形状

typedef struct ReadReplicationSlotCmd
{
	NodeTag		type;
	char	   *slotname;
} ReadReplicationSlotCmd;

入力と出力

  • **入力**: 構文解析器によって、レプリケーションプロトコルから抽出されたスロット名（slotname）が格納されます。
  • **出力**: 実行エンジン（walsender.c）が本構造体を参照し、指定されたスロットの情報を取得します。

前提条件と不変条件

  • type フィールドは、PostgreSQLのノードシステムの一部として適切なタグ値を持つ必要があります。
  • slotname は、有効なレプリケーションスロット名を指す文字列である必要があります。

アルゴリズムまたは振る舞い

本構造体はデータ保持専用のノードです。構文解析器がプロトコルからコマンドを解釈する際にインスタンス化され、その後、実行エンジンがこの構造体からスロット名を取り出し、対応するスロットの読み取り処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c 内のコマンド処理フローにおいて、ReadReplicationSlot 関数へ渡される引数として使用されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • NodeTag: PostgreSQLのノードシステムにおける型識別子。
  • src/include/nodes/replnodes.h 内の他のコマンド構造体（CreateReplicationSlotCmd 等）と共通の設計思想に基づいています。

失敗モードと境界条件

  • slotname が NULL または空文字列の場合、実行エンジン側での検証処理に依存します。
  • メモリ不足により構造体の割り当てが失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は walsender.c の ReadReplicationSlot 関数で直接参照されています。構造体のフィールドを変更する場合は、当該関数の実装も併せて確認してください。
