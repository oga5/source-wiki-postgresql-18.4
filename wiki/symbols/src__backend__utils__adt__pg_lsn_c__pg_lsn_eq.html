pg_lsn_eq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:117

目的

2つの pg_lsn 型の値が等しいかどうかを判定する。

シグネチャまたは形状

Datum pg_lsn_eq(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS を介して2つの pg_lsn 型の値（XLogRecPtr）を受け取る。
  • 出力: 2つの値が等しい場合は true、等しくない場合は false を Datum 型として返す。

前提条件と不変条件

  • 入力される pg_lsn は、内部的には uint64 として表現される XLogRecPtr である。
  • PG_GETARG_LSN マクロを使用して、引数から正しく XLogRecPtr を抽出できる必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_LSN(0) および PG_GETARG_LSN(1) を呼び出し、2つの XLogRecPtr 値を取得する。 2. 取得した2つの値を比較演算子 == で比較する。 3. PG_RETURN_BOOL を使用して、比較結果をブール値として返す。

呼び出し元と呼び出し先

  • 呼び出し元: SQLレベルの等価演算子（=）から呼び出される。
  • 呼び出し先: PG_GETARG_LSN および PG_RETURN_BOOL マクロを使用する。
  • 注: 具体的な呼び出し元関数のリストは、提供されたソース excerpt には含まれていない。

関連データ構造

  • XLogRecPtr: LSNの内部表現である64ビット整数。src/include/access/xlogdefs.h で定義されている。

失敗モードと境界条件

  • 引数が正しく渡されない場合、PG_GETARG_LSN マクロの内部でエラーが発生する可能性がある。
  • 64ビット整数としての比較であるため、値の範囲による境界条件は存在しない。

保守担当者向けメモ

  • 本関数は pg_lsn 型の等価性チェックを行う基本的な関数である。
  • 比較ロジックは単純な uint64 の比較であるため、計算コストは極めて低い。
