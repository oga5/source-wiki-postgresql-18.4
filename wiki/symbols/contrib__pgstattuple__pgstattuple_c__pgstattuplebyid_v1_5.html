pgstattuplebyid_v1_5

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:227

目的

リレーションの OID を引数として受け取り、対象リレーションの物理的な統計情報（タプル数、デッドタプル数、空き容量など）を算出・返却する SQL 呼び出し可能な関数です。

シグネチャまたは形状

Datum pgstattuplebyid_v1_5(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、対象リレーションの OID (relid) を受け取ります。
  • **出力**: Datum 型として、統計情報を格納した複合型を返却します。

前提条件と不変条件

  • 対象リレーションに対して AccessShareLock を取得します。
  • 実行ユーザーには、対象リレーションに対する適切なアクセス権限が必要です。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID(0) を使用して、引数からリレーションの OID を取得します。 2. relation_open を呼び出し、AccessShareLock モードでリレーションを開きます。 3. pgstat_relation を呼び出し、リレーションの種別（テーブルまたはインデックス）に応じた統計情報の集計処理へ委譲します。 4. pgstat_relation の戻り値を PG_RETURN_DATUM を介して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから呼び出されます（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • relation_open
  • pgstat_relation

関連データ構造

  • Relation: 開かれたリレーション情報を保持する構造体。
  • pgstattuple_type: 統計結果を保持するために内部で使用される構造体。

失敗モードと境界条件

  • 指定された OID が存在しない場合、relation_open によりエラーが発生します。
  • 権限がない場合、relation_open 時に権限エラーが発生します。
  • 大規模なリレーションに対して実行した場合、全ページ走査に伴う高い I/O 負荷が発生します。

保守担当者向けメモ

  • 本関数は pgstattuple 関数のバージョン 1.5 以降のインターフェースです。古いバージョンとの互換性を維持するために提供されています。
  • 統計情報の集計ロジックの詳細は pgstat_relation に依存しています。
