get_relation_constraints

定義

  • **種別**: function
  • **定義箇所**: src/backend/optimizer/util/plancat.c:1282

目的

リレーションに関連付けられたチェック制約、NOT NULL 制約、およびパーティション制約を収集し、プランナが利用可能な式リスト（List *）として返します。

シグネチャまたは形状

static List *get_relation_constraints(PlannerInfo *root, Oid relationObjectId, RelOptInfo *rel, bool include_noinherit, bool include_notnull, bool include_partition)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • relationObjectId: 対象リレーションの OID。
  • rel: 対象リレーションの RelOptInfo 構造体。
  • include_noinherit: NO INHERIT 制約を含めるかどうかのフラグ。
  • include_notnull: NOT NULL 制約を含めるかどうかのフラグ。
  • include_partition: パーティション制約を含めるかどうかのフラグ。
  • **出力**: 制約を表す式のリスト（List *）。

前提条件と不変条件

  • 対象リレーションは、本関数呼び出し前に既に安全にロックされている必要があります。
  • NOT ENFORCED な制約は無視されます。

アルゴリズムまたは振る舞い

1. table_open でリレーションを開きます。 2. **チェック制約の処理**: rd_att->constr からチェック制約を走査し、stringToNode で変換後、eval_const_expressions と canonicalize_qual を適用して正規化します。その後、ChangeVarNodes で varno を現在のクエリの varno に合わせます。 3. **NOT NULL 制約の処理**: include_notnull が真の場合、有効な NOT NULL 属性を持つ列に対して NullTest ノードを生成し、リストに追加します。 4. **パーティション制約の処理**: include_partition が真かつリレーションがパーティションである場合、set_baserel_partition_constraint を呼び出し、rel->partition_qual をリストに結合します。 5. **生成列の展開**: 最後に expand_generated_columns_in_expr を呼び出し、仮想生成列を式内で展開します。 6. table_close でリレーションを閉じ、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/util/plancat.c 内の他の関数から呼び出されています（行1737など）。
  • **呼び出し先**: table_open, table_close, stringToNode, eval_const_expressions, canonicalize_qual, ChangeVarNodes, make_ands_implicit, TupleDescCompactAttr, makeNode, set_baserel_partition_constraint, expand_generated_columns_in_expr 等。

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • RelOptInfo: リレーションの最適化情報。
  • TupleConstr: リレーションの制約情報。
  • NullTest: NOT NULL 制約を表現するノード。

失敗モードと境界条件

  • リレーションに制約が存在しない場合、空のリストを返します。
  • varno が 1 以外の場合、式内の varno を適切に書き換える処理が必須となります。

保守担当者向けメモ

  • 本関数は、制約式をクエリの varno に適合させるために ChangeVarNodes を使用しています。この処理はプランナが式を正しく評価するために不可欠です。
  • NOT NULL 制約の生成において、複合型列に対する IS NOT NULL は SQL 標準の IS NOT NULL とは異なる挙動（IS DISTINCT FROM NULL）を示すため、argisrow = false を明示的に設定しています。
