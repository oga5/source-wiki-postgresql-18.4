AioHandleShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_init.c:54

目的

AIO（非同期I/O）サブシステムにおいて、全バックエンドプロセスが使用する PgAioHandle 構造体群を格納するために必要な共有メモリサイズを算出します。

シグネチャまたは形状

Size AioHandleShmemSize(void)

入力と出力

  • **入力**: なし（グローバル変数 io_max_concurrency および AioProcs() の戻り値に依存）
  • **出力**: Size 型のバイト数

前提条件と不変条件

  • io_max_concurrency > 0 であること（アサーションにより検証）。
  • AioChooseMaxConcurrency() が実行済みであり、io_max_concurrency が適切に設定されていること。

アルゴリズムまたは振る舞い

1. AioProcs()（全バックエンド数）と io_max_concurrency（バックエンドあたりの最大同時I/O数）の積を計算します。 2. その結果に sizeof(PgAioHandle) を乗算し、必要な総メモリサイズを算出します。 3. 計算には mul_size を使用し、オーバーフローを防止します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio_init.c 内の AioShmemSize（合計サイズの算出時）
  • src/backend/storage/aio/aio_init.c 内の AioShmemInit（共有メモリ確保時）
  • **呼び出し先**:
  • AioProcs()
  • mul_size()

関連データ構造

  • PgAioHandle: 各バックエンドが保持するI/Oハンドル構造体。

失敗モードと境界条件

  • io_max_concurrency が 0 以下の場合、アサーション違反によりプロセスが停止します。
  • mul_size の内部で計算結果が Size 型の最大値を超える場合、PostgreSQL のメモリ割り当て規則に従いエラーが発生します。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定する重要な関数です。PgAioHandle 構造体のサイズが変更された場合、自動的に計算結果に反映されます。
  • io_max_concurrency はシステムリソースに応じて動的に決定されるため、メモリ見積もりの際は本関数が返す値が変動することに注意してください。
