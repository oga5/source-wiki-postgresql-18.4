astreamer_recovery_injector_content

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/astreamer_inject.c:84

目的

アーカイブストリーム処理パイプラインにおいて、リカバリ設定ファイル（recovery.conf または postgresql.auto.conf）の動的な挿入、編集、および standby.signal の注入を行う。

シグネチャまたは形状

static void astreamer_recovery_injector_content(
    astreamer *streamer,
    astreamer_member *member,
    const char *data,
    int len,
    astreamer_archive_context context
)

入力と出力

  • **入力**:
  • streamer: 状態を保持する astreamer_recovery_injector 構造体。
  • member: 現在処理中のアーカイブメンバー情報。
  • data, len: メンバーのコンテンツデータとその長さ。
  • context: 現在の処理フェーズ（ヘッダー、コンテンツ、トレーラー、アーカイブ終了）。
  • **出力**: なし（副作用として後続のストリーマーへデータを転送、または注入を行う）。

前提条件と不変条件

  • member が NULL の場合、context は必ず ASTREAMER_ARCHIVE_TRAILER でなければならない。
  • astreamer_recovery_injector 構造体としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. **ヘッダー処理 (ASTREAMER_MEMBER_HEADER)**:

  • 対象ファイル（recovery.conf, postgresql.auto.conf, standby.signal）を識別し、スキップフラグを立てる。
  • postgresql.auto.conf の場合、注入する設定内容の分だけサイズを拡張し、ヘッダー情報を無効化（data=NULL, len=0）する。

    • 2. **コンテンツ処理 (ASTREAMER_MEMBER_CONTENTS)**:

  • スキップ対象であれば転送を停止する。

    • 3. **トレーラー処理 (ASTREAMER_MEMBER_TRAILER)**:

  • postgresql.auto.conf の場合、保持していたリカバリ設定内容を後続ストリーマーへ送信する。

    • 4. **アーカイブ終了処理 (ASTREAMER_ARCHIVE_TRAILER)**:

  • 未注入の設定ファイルがあれば astreamer_inject_file を用いて新規追加する。
  • PostgreSQL v12 以降では standby.signal を強制的に注入する。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer フレームワークのパイプライン（astreamer_recovery_injector 構造体の .content フィールド経由）。
  • **呼び出し先**:
  • astreamer_content（後続ストリーマーへの転送）
  • astreamer_inject_file（リカバリ設定や信号ファイルの注入）

関連データ構造

  • astreamer_recovery_injector: 状態管理用構造体。
  • astreamer_member: アーカイブメンバーのメタデータ。
  • astreamer_archive_context: 処理フェーズを定義する列挙型。

失敗モードと境界条件

  • 未定義の context が渡された場合、pg_fatal を呼び出しプロセスを終了させる。
  • バージョン判定（is_recovery_guc_supported）により、処理対象ファイルが動的に切り替わる。

保守担当者向けメモ

  • 本関数はアーカイブヘッダーを直接操作しないため、注入されたデータは後続のストリーマーによって再パッケージ化される必要がある。
  • postgresql.auto.conf の編集時、ヘッダーの整合性を維持するために data を NULL に設定する処理が含まれている点に注意すること。
