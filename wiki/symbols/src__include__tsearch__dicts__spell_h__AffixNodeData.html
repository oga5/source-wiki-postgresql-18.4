AffixNodeData

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/dicts/spell.h:130

目的

接辞（affix）を格納するためのトライ（Trie）構造を構成するノードのデータ部分を保持します。

シグネチャまたは形状

typedef struct {
    uint32      val:8,
                naff:24;
    AFFIX     **aff;
    struct AffixNode *node;
} AffixNodeData;

入力と出力

  • val: ノードが保持する文字値（8ビット）。
  • naff: このノードに関連付けられた接辞の数（24ビット）。
  • aff: AFFIX 構造体へのポインタ配列。
  • node: 子ノードを指す AffixNode 構造体へのポインタ。

前提条件と不変条件

  • AffixNode 構造体の一部として、トライ構造の階層を形成するために使用されます。
  • naff は接辞の数を表し、aff 配列の要素数と整合する必要があります。

アルゴリズムまたは振る舞い

  • 辞書構築時にトライ構造を形成し、接辞の検索効率を最適化するために利用されます。
  • src/include/tsearch/dicts/spell.h 内で FLEXIBLE_ARRAY_MEMBER を用いた配列の一部として定義されるケースがあります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/spell.c 内の辞書構築および検索処理（cpalloc0 や palloc0 を用いたメモリ確保時など）。
  • 呼び出し先: AFFIX 構造体および AffixNode 構造体。

関連データ構造

  • AffixNode: 本構造体を保持する親構造体。
  • AFFIX: 接辞ルールを定義する構造体。

失敗モードと境界条件

  • メモリ確保時、ANHRDSZ や sizeof(AffixNodeData) を用いたサイズ計算が必須です。不適切なサイズ計算はメモリ破壊を招く可能性があります。

保守担当者向けメモ

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリレイアウトを直接操作する際は SPNHDRSZ などのサイズ計算マクロを必ず参照してください。
  • 辞書構築フェーズと実行フェーズでメモリのライフサイクルが異なるため、NIFinishBuild による適切なメモリ解放を確認してください。
