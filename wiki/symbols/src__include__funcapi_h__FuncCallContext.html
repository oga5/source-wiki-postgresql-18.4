FuncCallContext

定義

  • 種別: struct
  • 定義箇所: src/include/funcapi.h:57

目的

集合を返す関数（SRF: Set Returning Functions）において、複数回の呼び出し間で状態を保持・管理するための構造体です。

シグネチャまたは形状

typedef struct FuncCallContext {
    uint64          call_cntr;
    uint64          max_calls;
    void           *user_fctx;
    AttInMetadata  *attinmeta;
    MemoryContext   multi_call_memory_ctx;
    TupleDesc       tuple_desc;
} FuncCallContext;

入力と出力

  • **入力**: SRF_FIRSTCALL_INIT() マクロにより初期化されます。
  • **出力**: 複数呼び出しにまたがる状態（カウンタ、ユーザー定義データ、メモリコンテキスト、型情報）を保持し、SRF_RETURN_DONE() でクリーンアップされます。

前提条件と不変条件

  • call_cntr は SRF_FIRSTCALL_INIT() で 0 に初期化され、SRF_RETURN_NEXT() が呼ばれるたびに自動的にインクリメントされます。
  • multi_call_memory_ctx は複数呼び出し間でデータを生存させるために必須であり、SRF_FIRSTCALL_INIT() によって設定されます。
  • attinmeta は BuildTupleFromCStrings() を使用する場合にのみ必要です。
  • tuple_desc は heap_form_tuple() を使用する場合に必要であり、事前に BlessTupleDesc() を適用しておくことが推奨されます。

アルゴリズムまたは振る舞い

SRF の実行フローにおいて、初回呼び出し時に FuncCallContext を作成し、以降の呼び出しでこの構造体を通じて状態を維持します。max_calls は任意設定であり、設定しない場合は関数側で終了条件を管理する必要があります。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/dblink/dblink.c、contrib/hstore/hstore_op.c、contrib/pageinspect/ 配下の各モジュール（brinfuncs.c, btreefuncs.c, ginfuncs.c, hashfuncs.c, heapfuncs.c）などで参照されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出し先はありません。操作には SRF_* マクロ群が使用されます。

関連データ構造

  • AttInMetadata: 複合型を返す際の型入力メタデータ。
  • MemoryContext: 複数呼び出しにまたがるメモリ領域の管理。
  • TupleDesc: タプルの記述子。

失敗モードと境界条件

  • max_calls を設定しない場合、終了条件を適切に実装しないと無限ループに陥る可能性があります。

保守担当者向けメモ

  • multi_call_memory_ctx 以外のメモリ領域は、呼び出しごとにリセットされる可能性があるため注意してください。
  • 永続的なリソース管理が必要な場合は、マテリアライズドモード（InitMaterializedSRF()）の検討が必要です。
