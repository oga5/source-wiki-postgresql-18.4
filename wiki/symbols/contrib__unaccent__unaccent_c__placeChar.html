placeChar

定義

  • 種別: function
  • 定義箇所: contrib/unaccent/unaccent.c:56

目的

アクセント除去辞書のルールを保持する Trie 木構造に対し、置換対象の文字列（ソース文字列）と置換後の文字列をバイト単位で登録する。

シグネチャまたは形状

static TrieChar *placeChar(TrieChar *node, const unsigned char *str, int lenstr, const char *replaceTo, int replacelen)

入力と出力

  • **入力**:
  • node: 現在の Trie ノード（または NULL）。
  • str: 登録対象のソース文字列。
  • lenstr: str の長さ。
  • replaceTo: 置換後の文字列。
  • replacelen: replaceTo の長さ。
  • **出力**:
  • 登録済みの Trie 木のルートノードを返す。

前提条件と不変条件

  • lenstr > 0 であること（アサーションにより検証）。
  • node が NULL の場合、新しい TrieChar 配列（256要素）がメモリ確保される。

アルゴリズムまたは振る舞い

1. node が NULL ならば、palloc0 で 256 個の TrieChar を確保する。 2. str の先頭バイトをインデックスとして、現在のノード curnode を特定する。 3. lenstr が 1 の場合（末尾文字に到達）:

  • すでに replaceTo が設定されている場合は警告を発行する。
  • 未設定であれば、replacelen と replaceTo をノードにコピーして保存する。

    • 4. lenstr が 1 より大きい場合:

  • curnode->nextChar に対して再帰的に placeChar を呼び出し、残りの文字列を登録する。

    • 5. 最終的に node を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/unaccent/unaccent.c 内の initTrie（設定ファイル読み込み時）。
  • **呼び出し先**:
  • placeChar（再帰呼び出し）。
  • palloc0 / palloc（PostgreSQL メモリ管理関数）。
  • memcpy（標準ライブラリ）。
  • ereport（PostgreSQL エラー報告関数）。

関連データ構造

  • TrieChar: 置換ルールを保持する Trie 木のノード構造体。

失敗モードと境界条件

  • **重複登録**: 同一のソース文字列が複数回登録された場合、WARNING を発行し、最初の登録が優先される。
  • **メモリ不足**: palloc が失敗した場合は PostgreSQL の標準的なエラー処理（メモリ不足例外）が発生する。

保守担当者向けメモ

  • 本関数はバイト単位で Trie を構築するため、マルチバイト文字のエンコーディング整合性は呼び出し元で保証する必要がある。
  • 再帰呼び出しを行っているため、非常に長いソース文字列を登録する場合はスタック深さに注意が必要である。
