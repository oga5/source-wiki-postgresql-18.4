remove_rel_from_joinlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/analyzejoins.c:821

目的

結合ツリー（joinlist）から指定されたリレーション（relid）に対応する RangeTblRef を再帰的に探索し、削除します。

シグネチャまたは形状

List *remove_rel_from_joinlist(List *joinlist, int relid, int *nremoved)

入力と出力

  • **入力**:
  • joinlist: 結合ツリーを表す List。
  • relid: 削除対象のリレーションID。
  • nremoved: 削除されたノード数をカウントするためのポインタ。
  • **出力**:
  • 指定された relid を除外した新しい結合ツリーの List。

前提条件と不変条件

  • joinlist は RangeTblRef または List（サブリスト）のみで構成されている必要があります。
  • 削除の結果、空になったサブリストは結果から除外されます。

アルゴリズムまたは振る舞い

1. joinlist の各要素を走査します。 2. 要素が RangeTblRef の場合:

  • rtindex が relid と一致すれば、nremoved をインクリメントし、その要素を結果リストに追加しません。
  • 一致しなければ、そのまま結果リストに追加します。

    • 3. 要素が List（サブリスト）の場合:

  • 関数自身を再帰的に呼び出し、サブリスト内から relid を削除します。
  • 削除後のサブリストが空でなければ、結果リストに追加します。

    • 4. 上記以外のノード型に遭遇した場合は elog(ERROR) を発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/analyzejoins.c 内の複数箇所（remove_useless_joins 等の最適化ルーチンから呼び出される）。
  • **呼び出し先**:
  • 自身（再帰呼び出し）。

関連データ構造

  • List: 結合ツリーの構造を保持。
  • RangeTblRef: 結合ツリー内の各リレーション参照。

失敗モードと境界条件

  • RangeTblRef および List 以外のノード型が joinlist に含まれている場合、エラーとなります。
  • 削除の結果、すべての要素が取り除かれた場合、空のリスト（NIL）が返されます。

保守担当者向けメモ

  • 本関数はクエリの結合ツリーを直接書き換えるため、呼び出し後に PlannerInfo 内の他のデータ構造（EquivalenceClass や RestrictInfo 等）との整合性が崩れないよう注意が必要です。
  • 再帰的な構造を扱うため、結合ツリーの深さが極端に深い場合の影響に留意してください。
