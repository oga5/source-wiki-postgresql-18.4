get_comma_elts

定義

  • 種別: function
  • 定義箇所: contrib/oid2name/oid2name.c:260

目的

eary 構造体に格納された文字列リストを、SQL の IN 句などで利用可能な、カンマ区切りかつシングルクォートで囲まれたエスケープ済みの文字列に変換する。

シグネチャまたは形状

char *get_comma_elts(eary *eary)

入力と出力

  • **入力**: eary *eary （検索対象の文字列リストを保持する構造体）
  • **出力**: char * （変換後の文字列。呼び出し元で解放が必要）

前提条件と不変条件

  • eary が有効なポインタであること。
  • eary->num が 0 の場合、空文字列 "" を返す。
  • 戻り値のメモリは pg_malloc で確保される。

アルゴリズムまたは振る舞い

1. eary->num が 0 の場合、空文字列を返して終了する。 2. 全要素の文字列長を合計し、エスケープ処理およびクォート付与に必要なバッファサイズを計算する。 3. pg_malloc で必要なメモリを確保する。 4. 各要素をループで処理する：

  • 2 番目以降の要素の前にカンマ , を挿入する。
  • 各要素をシングルクォート ' で囲む。
  • PQescapeString を使用して、各要素の文字列を SQL 安全な形式にエスケープしてバッファに書き込む。

    • 5. 完成した文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/oid2name/oid2name.c 内のメイン処理（513行目、514行目、515行目）にて、opts->oids、opts->tables、opts->filenumbers の変換に使用。
  • **呼び出し先**:
  • pg_strdup
  • pg_malloc
  • PQescapeString
  • strlen
  • sprintf

関連データ構造

  • eary: 文字列の動的配列を管理する構造体。

失敗モードと境界条件

  • メモリ不足時: pg_malloc が失敗した場合、プログラムは終了する（PostgreSQL の標準的なメモリ割り当て動作）。
  • 空のリスト: eary->num == 0 の場合、空文字列を返す。

保守担当者向けメモ

  • PQescapeString の仕様に基づき、バッファサイズとして length * 2 + 4 * eary->num を確保している。この計算式は、各要素が最大で元の 2 倍の長さになり、かつクォートとカンマの分を考慮した安全な見積もりである。
