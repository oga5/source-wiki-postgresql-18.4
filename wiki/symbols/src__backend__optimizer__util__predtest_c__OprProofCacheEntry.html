OprProofCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/predtest.c:2080

目的

演算子間の論理包含（implication）および矛盾（refutation）の証明結果をキャッシュするためのデータ構造です。

シグネチャまたは形状

typedef struct OprProofCacheEntry
{
    OprProofCacheKey key;           /* ハッシュルックアップキー（先頭必須） */
    bool        have_implic;        /* 包含判定結果の有無 */
    bool        have_refute;        /* 矛盾判定結果の有無 */
    bool        same_subexprs_implies; /* 包含関係の真偽 */
    bool        same_subexprs_refutes; /* 矛盾関係の真偽 */
    Oid         implic_test_op;     /* 包含テスト用演算子OID（なしの場合は0） */
    Oid         refute_test_op;     /* 矛盾テスト用演算子OID（なしの場合は0） */
} OprProofCacheEntry;

入力と出力

  • 本構造体は OprProofCacheHash ハッシュテーブルの要素として使用されます。
  • hash_search を介してキー（OprProofCacheKey）に基づいた検索・挿入が行われます。

前提条件と不変条件

  • key フィールドは構造体の先頭に配置される必要があります。
  • implic_test_op および refute_test_op は、演算子が存在しない場合 0 に設定されます。

アルゴリズムまたは振る舞い

  • 演算子比較の証明結果を保持し、再計算を回避するために使用されます。
  • have_implic および have_refute フラグにより、各証明の試行結果がキャッシュ済みであるかを管理します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/util/predtest.c 内のキャッシュ検索および更新ロジック（hash_search を使用する関数群）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • OprProofCacheKey: キャッシュの検索キー。
  • OprProofCacheHash: 本構造体を格納するハッシュテーブル。

失敗モードと境界条件

  • キャッシュは pg_amop の変更時に InvalidateOprProofCacheCallBack によってフラッシュされます。
  • 構造体サイズは sizeof(OprProofCacheEntry) としてハッシュテーブル制御構造に渡されます。

保守担当者向けメモ

  • 本構造体は predtest.c における演算子推論のパフォーマンスを左右します。フィールドを追加・変更する場合は、ハッシュキーの整合性とキャッシュの無効化ロジックに注意してください。
