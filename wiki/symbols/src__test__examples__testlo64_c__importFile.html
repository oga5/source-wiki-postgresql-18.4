importFile

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlo64.c:34

目的

ローカルの Unix ファイルの内容を読み込み、PostgreSQL データベース内の新しいラージオブジェクトとしてインポートする。

シグネチャまたは形状

static Oid importFile(PGconn *conn, char *filename)

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • filename: インポート対象のファイルパスを示す文字列。
  • **出力**:
  • 成功時: 作成されたラージオブジェクトの Oid を返す。
  • 失敗時: lo_creat が失敗した場合は 0 を返す。

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • 指定された filename は読み取り可能であること。
  • lo_creat および lo_open が成功し、ラージオブジェクトの書き込み権限が確保されていること。

アルゴリズムまたは振る舞い

1. open を使用して指定されたファイルを読み取り専用で開く。 2. lo_creat を呼び出し、INV_READ | INV_WRITE モードで新しいラージオブジェクトを作成する。 3. lo_open を呼び出し、作成したラージオブジェクトを書き込み用に開く。 4. read を使用してローカルファイルから BUFSIZE 単位でデータを読み込み、lo_write を使用してラージオブジェクトへ順次書き込む。 5. ファイルの読み込みが完了した後、close でローカルファイルを閉じ、lo_close でラージオブジェクトを閉じる。 6. 作成したラージオブジェクトの Oid を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/examples/testlo.c および src/test/examples/testlo64.c 内で参照されているが、ソース内のコメントによれば main 関数内ではより簡潔な lo_import が優先的に使用されており、本関数は直接呼び出されていない可能性がある。
  • **呼び出し先**:
  • open, read, close (標準 C ライブラリ)
  • lo_creat, lo_open, lo_write, lo_close (libpq-fs API)

関連データ構造

  • PGconn: PostgreSQL 接続ハンドル。
  • Oid: ラージオブジェクトを一意に識別するオブジェクト識別子。

失敗モードと境界条件

  • ファイルが開けない場合、標準エラー出力にメッセージを出力する。
  • lo_creat が失敗した場合、標準エラー出力にメッセージを出力する。
  • lo_write による書き込みバイト数が期待値より少ない場合、標準エラー出力にエラーメッセージを出力する。

保守担当者向けメモ

  • 本関数は libpq が提供する標準的な lo_import 関数を自作で再実装したものである。
  • main 関数内での利用状況を鑑みると、テスト目的の補助関数として位置付けられている。
