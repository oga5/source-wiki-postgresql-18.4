ExecIndexScanInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:1760

目的

並列クエリ実行時、各ワーカープロセスにおいてインデックススキャンノードの状態を共有メモリ（DSM）から復元し、初期化を行う。

シグネチャまたは形状

void ExecIndexScanInitializeWorker(IndexScanState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の IndexScanState 構造体。
  • pwcxt: 並列ワーカーのコンテキスト情報。
  • **出力**: なし（副作用として node 内のフィールドを更新）。

前提条件と不変条件

  • node->ss.ps.plan が有効であること。
  • pwcxt->toc に当該プランノードの共有メモリ情報が含まれていること。

アルゴリズムまたは振る舞い

1. **初期化判定**: 計測（instrument）または並列実行（parallel_aware）が不要な場合は即座に終了する。 2. **DSM検索**: shm_toc_lookup を使用し、プランノードIDに基づいて共有メモリ上の ParallelIndexScanDesc を取得する。 3. **計測情報の紐付け**: 計測が有効な場合、piscan からオフセット計算を行い node->iss_SharedInfo を設定する。 4. **並列スキャン開始**: parallel_aware が真の場合、index_beginscan_parallel を呼び出し、インデックススキャン記述子を生成する。 5. **スキャンキーの適用**: ランタイムキーが存在しない、または準備完了状態であれば、index_rescan を呼び出してスキャンキーとオーダーキーをインデックスAMに適用する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1347 （直接的な呼び出しを確認）。
  • **呼び出し先**: shm_toc_lookup, OffsetToPointer, index_beginscan_parallel, index_rescan。

関連データ構造

  • IndexScanState: インデックススキャンの実行状態を保持する構造体。
  • ParallelIndexScanDesc: 並列スキャンにおける共有状態。
  • SharedIndexScanInstrumentation: 並列実行時の統計情報。

失敗モードと境界条件

  • shm_toc_lookup が対象ノードを見つけられない場合、エラーが発生する可能性がある。
  • ランタイムキーが未準備の状態で index_rescan が呼び出されない場合、スキャンキーは適用されない（後続の ExecReScanIndexScan 等で処理される必要がある）。

保守担当者向けメモ

  • 本関数は並列ワーカーの起動時に一度だけ実行されることを想定している。
  • parallel_aware フラグが偽の場合でも、計測情報（SharedInfo）のセットアップのために本関数が通過する点に注意すること。
