AutoVacuumShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/autovacuum.c:3377

目的

自動バキュームシステムで使用される共有メモリ領域（AutoVacuumShmem）の割り当ておよび初期化を行う。

シグネチャまたは形状

void AutoVacuumShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 AutoVacuumShmem が初期化される）

前提条件と不変条件

  • ShmemInitStruct を介して共有メモリを確保する。
  • IsUnderPostmaster が偽（Postmaster プロセス自身による初期化時）の場合、共有メモリ構造体およびワーカー情報リストの初期化が必須である。
  • IsUnderPostmaster が真（子プロセスによる参照時）の場合、共有メモリが既に存在している必要がある。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、AutoVacuum Data という名前で共有メモリを確保する。 2. IsUnderPostmaster が偽の場合（初期起動時）:

  • AutoVacuumShmem の各フィールド（PID、リスト、ワークアイテム）をゼロクリアまたは初期化する。
  • AutoVacuumShmemStruct の直後に配置された WorkerInfo 配列を計算する。
  • autovacuum_worker_slots 分のワーカー情報をループで走査し、av_freeWorkers リストへ追加し、各ワーカーのバランスフラグを初期化する。
  • av_nworkersForBalance を 0 に初期化する。

    • 3. IsUnderPostmaster が真の場合:

  • 共有メモリが既に存在していることを Assert(found) で確認する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:328（PostgreSQL の共有メモリ初期化シーケンスの一部として呼び出される）
  • 呼び出し先: ShmemInitStruct, dclist_init, dlist_init, memset, dclist_push_head, pg_atomic_init_flag, pg_atomic_init_u32

関連データ構造

  • AutoVacuumShmemStruct: 自動バキュームの制御情報を保持するメイン構造体。
  • WorkerInfo: 個々のワーカーの状態を管理する構造体。
  • AutoVacuumWorkItem: 共有メモリ上の作業項目配列。

失敗モードと境界条件

  • ShmemInitStruct が失敗した場合、PostgreSQL の共有メモリ割り当てルールに従いプロセスは異常終了する。
  • autovacuum_worker_slots の数に応じて WorkerInfo 配列が確保されるため、この値が極端に大きい場合はメモリ不足のリスクがある。

保守担当者向けメモ

  • 本関数は Postmaster の起動プロセスにおいて一度だけ実行されることを想定している。
  • WorkerInfo 配列は AutoVacuumShmemStruct の直後にメモリ配置されるため、構造体のサイズ変更時にはメモリレイアウトの整合性に注意すること。
