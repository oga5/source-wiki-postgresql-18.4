positionjoinsel

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_selfuncs.c:72

目的

幾何演算子（左、右、上、下など）を用いた結合クエリにおける選択率（Selectivity）を推定し、クエリプランナに提供します。

シグネチャまたは形状

Datum positionjoinsel(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（クエリプランナからの引数群）。
  • **出力**: float8 型の固定値 0.1 を PG_RETURN_FLOAT8 マクロ経由で返却します。

前提条件と不変条件

  • 入力されたクエリの統計情報やデータ分布は一切参照されません。
  • 常に固定値 0.1 を返却する静的な実装です。

アルゴリズムまたは振る舞い

1. 関数が呼び出されると、統計計算を行わずに即座に 0.1 を返却します。 2. この値は、プランナに対して幾何インデックス（GiST等）の利用を促すためのヒューリスティックな値として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: クエリプランナ（詳細な呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: なし。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約で使用される引数構造体。

失敗モードと境界条件

  • 特になし。入力内容に関わらず常に一定の値を返却します。

保守担当者向けメモ

  • 本実装は統計情報に基づかない「完全にデタラメ（totally bogus）」な固定値です。
  • 幾何学的なデータ分布を考慮した真の選択率推定アルゴリズムが実装されるまでの暫定的な処置です。
