ExecAlterExtensionContentsStmt

定義

  • 種別: function
  • 定義箇所: src/backend/commands/extension.c:3712

目的

ALTER EXTENSION ... ADD/DROP SQLコマンドを処理し、指定されたオブジェクトを拡張機能の構成要素として追加または削除する。

シグネチャまたは形状

ObjectAddress ExecAlterExtensionContentsStmt(AlterExtensionContentsStmt *stmt, ObjectAddress *objAddr)

入力と出力

  • **入力**:
  • stmt: 実行対象の AlterExtensionContentsStmt 構造体（拡張機能名、対象オブジェクトの型と識別子を含む）。
  • objAddr: 処理対象オブジェクトのアドレスを格納するためのポインタ（NULL可）。
  • **出力**:
  • 処理対象の拡張機能の ObjectAddress を返す。

前提条件と不変条件

  • 拡張機能の所有者である必要がある。
  • 対象オブジェクトの所有者である必要がある。
  • OBJECT_DATABASE や OBJECT_EXTENSION など、拡張機能に追加できない特定のオブジェクト型が指定された場合はエラーとなる。
  • object.objectSubId は 0 である必要がある。

アルゴリズムまたは振る舞い

1. **型チェック**: stmt->objtype が拡張機能に追加可能な型か検証し、不可であればエラーを投げる。 2. **拡張機能の特定**: get_object_address を呼び出し、拡張機能の ObjectAddress を取得し、AccessShareLock を獲得する。 3. **権限確認**: object_ownercheck を使用し、現在のユーザーが拡張機能の所有者であることを確認する。 4. **対象オブジェクトの特定**: get_object_address を呼び出し、対象オブジェクトの ObjectAddress を取得し、ShareUpdateExclusiveLock を獲得する。 5. **所有権確認**: check_object_ownership を使用し、対象オブジェクトの所有権を確認する。 6. **更新実行**: ExecAlterExtensionContentsRecurse を呼び出し、依存関係を含めた更新処理を実行する。 7. **フック実行**: InvokeObjectPostAlterHook を呼び出し、変更後のフック処理を行う。 8. **リソース解放**: 取得したリレーションをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:1585 (ProcessUtility内)
  • **呼び出し先**:
  • get_object_address
  • object_ownercheck
  • check_object_ownership
  • ExecAlterExtensionContentsRecurse
  • InvokeObjectPostAlterHook
  • relation_close

関連データ構造

  • AlterExtensionContentsStmt: SQLコマンドの解析結果を保持する構造体。
  • ObjectAddress: データベースオブジェクトを一意に特定する構造体。

失敗モードと境界条件

  • 拡張機能に追加不可能なオブジェクト型が指定された場合、ERRCODE_INVALID_OBJECT_DEFINITION でエラーとなる。
  • 拡張機能または対象オブジェクトが存在しない場合、get_object_address によりエラーとなる。
  • 権限がない場合、aclcheck_error によりエラーとなる。

保守担当者向けメモ

  • 本関数は ShareUpdateExclusiveLock を使用して対象オブジェクトをロックし、同時実行される DROP や ALTER EXTENSION 操作から保護している。
  • ExecAlterExtensionContentsRecurse への依存関係の再帰処理が含まれている点に注意すること。
