PgChecksumMode

定義

  • 種別: enum
  • 定義箇所: src/bin/pg_checksums/pg_checksums.c:48

目的

pg_checksums ツールにおけるデータディレクトリの処理モード（検証、無効化、有効化）を定義します。

シグネチャまたは形状

typedef enum
{
	PG_MODE_CHECK,
	PG_MODE_DISABLE,
	PG_MODE_ENABLE,
} PgChecksumMode;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: PG_MODE_CHECK, PG_MODE_DISABLE, PG_MODE_ENABLE のいずれかの値。

前提条件と不変条件

  • 本列挙型は、pg_checksums ツールがオフライン状態の PostgreSQL データディレクトリに対して実行されることを前提としています。

アルゴリズムまたは振る舞い

  • PG_MODE_CHECK: データファイル内のページチェックサムを検証します。
  • PG_MODE_DISABLE: データファイル内のページチェックサムを無効化します。
  • PG_MODE_ENABLE: データファイル内のページチェックサムを有効化（計算および書き込み）します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_checksums/pg_checksums.c 内の静的変数 mode の初期化（55行目）に使用されています。
  • 呼び出し先: 情報なし。

関連データ構造

  • ControlFileData: クラスタのメタデータを保持する構造体。本モード設定に基づき、最終的に update_controlfile() を通じてチェックサムの状態が更新されます。

失敗モードと境界条件

  • 稼働中のクラスタに対して本モード設定に基づく処理を実行した場合、チェックサムの不整合やデータ破損の誤検知が発生する可能性があります。

保守担当者向けメモ

  • 新しい処理モードを追加する場合は、本列挙型に値を追加し、scan_file() およびメインの処理フローにおける条件分岐を更新してください。
