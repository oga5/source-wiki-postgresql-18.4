check_compressed_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_io.c:218

目的

指定されたパスに特定の拡張子を付与したファイルが存在するかを確認し、そのファイルパスを生成・保持します。

シグネチャまたは形状

static bool check_compressed_file(const char *path, char **fname, char *ext);

入力と出力

  • **入力**:
  • path: ベースとなるファイルパス。
  • fname: 生成されたフルパス（path.ext）を格納するためのポインタ。
  • ext: 付与する拡張子文字列。
  • **出力**:
  • 戻り値: ファイルが存在する場合は true、存在しない場合は false。
  • fname: 呼び出し元で提供されたメモリが解放され、新しく割り当てられたパス文字列が格納されます。

前提条件と不変条件

  • *fname は free() 可能なメモリ領域を指している必要があります（free_keep_errno により解放されます）。
  • path および ext は有効な文字列である必要があります。

アルゴリズムまたは振る舞い

1. free_keep_errno(*fname) を呼び出し、既存の fname が指すメモリを解放します。 2. psprintf を使用して、path に . と ext を連結した新しい文字列を生成し、*fname に代入します。 3. access(*fname, F_OK) を実行し、ファイルシステム上に当該ファイルが存在するかを確認します。 4. 存在確認の結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_io.c 内の圧縮形式判定ロジック（gz, lz4, zst の各拡張子確認時）。
  • **呼び出し先**: free_keep_errno（メモリ解放）、psprintf（文字列生成）、access（ファイル存在確認）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • psprintf がメモリ不足で失敗した場合、プログラムは終了します。
  • access がファイルシステムエラー（権限不足等）を返した場合、ファイルが存在しないものとして false を返します。

保守担当者向けメモ

  • 本関数は errno を保持したままメモリ解放を行うため、呼び出し元のエラーハンドリング状態を破壊しません。
  • fname に渡すポインタは、必ず NULL または malloc 等で確保された領域である必要があります。
