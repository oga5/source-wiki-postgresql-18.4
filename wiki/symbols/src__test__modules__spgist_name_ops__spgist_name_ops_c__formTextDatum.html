formTextDatum

定義

  • 種別: function
  • 定義箇所: src/test/modules/spgist_name_ops/spgist_name_ops.c:51

目的

指定されたデータバッファから、PostgreSQLの text 型として有効な Datum を構築する内部ヘルパー関数です。

シグネチャまたは形状

Datum formTextDatum(const char *data, int datalen)

入力と出力

  • **入力**:
  • data: 文字列データへのポインタ（NULL許容）。
  • datalen: データの長さ（バイト単位）。
  • **出力**: 構築された text 型の Datum。

前提条件と不変条件

  • palloc を使用してメモリを割り当てるため、呼び出し元は適切なメモリコンテキストにいる必要があります。
  • datalen が0の場合、データ部を持たない空の text 型として構築されます。

アルゴリズムまたは振る舞い

1. datalen + VARHDRSZ 分のメモリを palloc で確保します。 2. データ長に応じて、varlena ヘッダーの形式を決定します。

  • datalen + VARHDRSZ_SHORT が VARATT_SHORT_MAX 以下の場合、ショート形式のヘッダー（SET_VARSIZE_SHORT）を使用し、データをコピーします。
  • それ以外の場合は、標準のロング形式ヘッダー（SET_VARSIZE）を使用し、データをコピーします。

    • 3. 構築したメモリ領域を PointerGetDatum に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/spgist_name_ops/spgist_name_ops.c 内の各関数（spgist_name_choose 等）。
  • src/backend/access/spgist/spgtextproc.c 内の各関数。
  • **呼び出し先**:
  • palloc (PostgreSQLメモリ管理)
  • SET_VARSIZE_SHORT / SET_VARSIZE (varlenaマクロ)
  • memcpy
  • PointerGetDatum

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • varlena: 可変長データ型（text 型の内部表現）。

失敗モードと境界条件

  • palloc がメモリ不足（OOM）を検知した場合、PostgreSQLの標準的なエラーハンドリングによりアボートします。
  • data が NULL で datalen が 0 より大きい場合、memcpy で未定義の動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は spgtextproc.c と spgist_name_ops.c の両方から利用される共通ロジックです。変更時は両方のモジュールへの影響を考慮してください。
  • reconstructedValue を扱う際の標準的な構築手法として利用されています。
