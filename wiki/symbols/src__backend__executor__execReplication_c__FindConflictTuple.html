FindConflictTuple

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execReplication.c:467

目的

論理レプリケーションにおいて、指定されたインデックスに基づき制約違反を引き起こしている既存のタプルを特定し、ロックを取得する。

シグネチャまたは形状

static bool FindConflictTuple(ResultRelInfo *resultRelInfo, EState *estate, Oid conflictindex, TupleTableSlot *slot, TupleTableSlot **conflictslot)

入力と出力

  • **入力**:
  • resultRelInfo: 対象リレーション情報。
  • estate: 実行状態。
  • conflictindex: 競合チェックに使用するインデックスのOID。
  • slot: 競合の原因となっている入力タプル。
  • **出力**:
  • conflictslot: 競合タプルが格納されるスロット（見つからない場合は NULL）。
  • 戻り値: 競合タプルが正常にロックされた場合は true、競合が存在しない場合は false。

前提条件と不変条件

  • BuildConflictIndexInfo が事前に呼び出され、インデックス情報が準備されている必要がある。
  • 呼び出し元は、競合が発生した際に conflictslot を適切に解放する責任を持つ。

アルゴリズムまたは振る舞い

1. BuildConflictIndexInfo を呼び出し、制約チェックに必要な情報を構築する。 2. ExecCheckIndexConstraints を使用して、指定されたインデックスで制約違反が発生しているか確認する。違反がない場合は false を返す。 3. 違反がある場合、table_slot_create でスロットを確保する。 4. table_tuple_lock を呼び出し、競合タプルに対して LockTupleShare モードでロックを取得する。 5. 同時実行制御によりタプルが更新された場合（should_refetch_tuple が真）、手順2へ戻り再試行する。 6. ロック成功後、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execReplication.c 内のレプリケーション適用ロジックから呼び出される。
  • **呼び出し先**: BuildConflictIndexInfo, ExecCheckIndexConstraints, ExecDropSingleTupleTableSlot, table_slot_create, table_tuple_lock, should_refetch_tuple 等。

関連データ構造

  • ResultRelInfo: リレーションの実行時情報。
  • TupleTableSlot: タプルを保持するメモリ構造。
  • TM_FailureData: タプルロック失敗時の詳細情報。

失敗モードと境界条件

  • インデックス制約に違反しない場合、false を返す。
  • table_tuple_lock が同時実行による競合を検出した場合、再試行ループ（retry）により解決を試みる。

保守担当者向けメモ

  • 本関数は goto retry を用いた再試行ロジックを含んでおり、同時実行環境下での整合性を維持している。
  • conflictslot のメモリ管理は呼び出し元に依存するため、リークを防ぐよう注意すること。
