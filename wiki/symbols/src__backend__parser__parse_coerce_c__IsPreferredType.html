IsPreferredType

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_coerce.c:2994

目的

指定されたデータ型が、特定の型カテゴリ（TYPCATEGORY）において「優先される型（Preferred Type）」であるかどうかを判定します。これは主に、暗黙的な型変換の解決において、複数の候補から最適な型を選択するために使用されます。

シグネチャまたは形状

bool IsPreferredType(TYPCATEGORY category, Oid type)

入力と出力

  • **入力**:
  • category: 判定対象の型カテゴリ（TYPCATEGORY 型）。TYPCATEGORY_INVALID を指定した場合は、任意のカテゴリにおける優先型であるかを判定します。
  • type: 判定対象のデータ型の OID。
  • **出力**:
  • bool: 指定された型が該当カテゴリの優先型であれば true、そうでなければ false を返します。

前提条件と不変条件

  • get_type_category_preferred 関数が正常に動作し、型情報がシステムカタログから取得可能である必要があります。

アルゴリズムまたは振る舞い

1. get_type_category_preferred を呼び出し、指定された type のカテゴリ（typcategory）と、それが優先型であるか（typispreferred）を取得します。 2. 引数の category が TYPCATEGORY_INVALID であるか、または取得した typcategory と一致する場合に、typispreferred の値を返します。 3. それ以外の場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/indexcmds.c:2401
  • src/backend/parser/parse_func.c:1131
  • **呼び出し先**:
  • get_type_category_preferred（システムカタログ検索関数）

関連データ構造

  • TYPCATEGORY: 型カテゴリを識別するための列挙型または定数群。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • type が無効な OID である場合、get_type_category_preferred の挙動に依存します。
  • category に TYPCATEGORY_INVALID を渡すと、その型が「いずれかのカテゴリ」において優先型としてマークされている場合に true を返します。

保守担当者向けメモ

  • 本関数は src/include/parser/parse_coerce.h で公開されています。
  • 型の優先順位は pg_type システムカタログの typispreferred 列によって定義されています。変更が必要な場合はカタログ定義を確認してください。
