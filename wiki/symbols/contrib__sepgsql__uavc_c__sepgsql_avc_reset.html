sepgsql_avc_reset

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:77

目的

ユーザー空間アクセスベクタキャッシュ（AVC）の全エントリを破棄し、キャッシュの状態を初期化する。

シグネチャまたは形状

static void sepgsql_avc_reset(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてキャッシュ状態をリセット）

前提条件と不変条件

  • キャッシュは avc_mem_cxt という専用のメモリコンテキストで管理されており、本関数はこのコンテキストをリセットする。
  • 本関数実行後、キャッシュに関連するグローバル変数は初期状態（空）にリセットされる。

アルゴリズムまたは振る舞い

1. MemoryContextReset(avc_mem_cxt) を呼び出し、キャッシュ用メモリコンテキストを解放する。 2. avc_slots 配列を memset によりゼロクリアし、ハッシュテーブルの各スロットを空にする。 3. キャッシュエントリ数 avc_num_caches を 0 に設定する。 4. LRU（Least Recently Used）ヒント avc_lru_hint を 0 にリセットする。 5. 未ラベル状態を示す avc_unlabeled ポインタを NULL に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/sepgsql/uavc.c:156（文脈上、キャッシュの無効化や再構築が必要なタイミングで呼び出される）
  • 呼び出し先: MemoryContextReset（PostgreSQL 内部関数）

関連データ構造

  • avc_mem_cxt: キャッシュエントリを保持するメモリコンテキスト。
  • avc_slots: キャッシュエントリを格納するハッシュテーブルのリスト配列。
  • avc_num_caches: 現在のキャッシュエントリ数。
  • avc_lru_hint: LRU 管理用のヒント値。
  • avc_unlabeled: 未ラベルオブジェクトのキャッシュエントリ。

失敗モードと境界条件

  • メモリコンテキストが正しく初期化されていない状態で呼び出された場合、MemoryContextReset の挙動に依存する。
  • 本関数はキャッシュを完全にクリアするため、呼び出し直後の権限チェックはキャッシュミスとなり、再計算が発生する。

保守担当者向けメモ

  • 本関数はキャッシュの全フラッシュを行うため、パフォーマンスに影響を与える。ポリシー変更時など、キャッシュの妥当性が失われた場合にのみ使用すること。
  • avc_mem_cxt のライフサイクルと整合性が取れていることを確認すること。
