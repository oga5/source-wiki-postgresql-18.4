gin_mask

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginxlog.c:792

目的

GINインデックスページの内容をマスクし、リカバリ後のページ整合性チェック（wal_consistency_checking）において、チェックサムやLSNの差異、および非決定的なビット情報を無視して検証を可能にする。

シグネチャまたは形状

void gin_mask(char *pagedata, BlockNumber blkno)

入力と出力

  • pagedata: マスク対象となるページデータのポインタ。
  • blkno: 対象ページのブロック番号（本関数内では未使用）。
  • 戻り値: なし（pagedata が直接変更される）。

前提条件と不変条件

  • pagedata は有効なGINインデックスページを指している必要がある。
  • ページヘッダーの pd_lower が適切に設定されている場合、未使用領域のマスクが適用される。

アルゴリズムまたは振る舞い

1. mask_page_lsn_and_checksum(page) を呼び出し、ページ全体のLSNとチェックサムをマスクする。 2. mask_page_hint_bits(page) を呼び出し、ヒントビットをマスクする。 3. ページが GIN_DELETED フラグを持つ場合、ページ全体を mask_page_content でマスクする。 4. それ以外の場合で、pd_lower がページヘッダーサイズより大きい場合、mask_unused_space を呼び出し、ページ内の未使用領域をマスクする。

呼び出し元と呼び出し先

  • 呼び出し元: GINリソースマネージャ（RM_GIN_ID）の定義において、リカバリ時の検証用関数として登録されている（src/include/access/rmgrlist.h）。
  • 呼び出し先: mask_page_lsn_and_checksum, mask_page_hint_bits, mask_page_content, mask_unused_space（いずれも access/bufmask.h 由来）。

関連データ構造

  • PageHeader: ページヘッダー情報。
  • GinPageOpaque: GINインデックス特有のページメタデータ（GIN_DELETED フラグの確認に使用）。

失敗モードと境界条件

  • GIN_DELETED フラグが立っているページは、内容が空とみなされ全体がマスクされる。
  • pd_lower が不正な値（SizeOfPageHeaderData 以下）である場合、未使用領域のマスクはスキップされる。

保守担当者向けメモ

  • 新しいWALレコードタイプを追加した際は、本関数で適切にマスク対象が定義されているか確認すること。
  • ページ構造の変更時には、mask_unused_space が正しく機能するよう pd_lower の管理に注意が必要。
