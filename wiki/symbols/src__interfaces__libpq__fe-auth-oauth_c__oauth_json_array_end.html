oauth_json_array_end

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-oauth.c:282

目的

JSON 解析プロセスにおいて、配列の終了を通知し、現在のネストレベルを更新するために使用されます。

シグネチャまたは形状

JsonParseErrorType oauth_json_array_end(void *state);

入力と出力

  • **入力**: void *state - json_ctx 構造体へのポインタ。
  • **出力**: JsonParseErrorType - 常に JSON_SUCCESS を返します。

前提条件と不変条件

  • state は struct json_ctx 型としてキャスト可能である必要があります。
  • 呼び出しのたびに ctx->nested がデクリメントされます。

アルゴリズムまたは振る舞い

1. 引数 state を struct json_ctx *ctx にキャストします。 2. ctx->nested をデクリメントします。 3. JSON_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq-oauth/oauth-curl.c:640
  • src/interfaces/libpq/fe-auth-oauth.c:283 (内部参照)
  • **呼び出し先**: なし。

関連データ構造

  • json_ctx: JSON 解析の状態を保持する構造体。nested メンバを介して階層構造を追跡します。

失敗モードと境界条件

  • ctx が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • 配列の開始（oauth_json_array_start）と終了の整合性が取れていない場合、nested 値が不整合を起こす可能性があります。

保守担当者向けメモ

  • 本関数は json_ctx の状態管理を行うコールバック関数として設計されています。
  • src/interfaces/libpq/fe-auth-oauth.c:553 および src/interfaces/libpq-oauth/oauth-curl.c:866 にて、JSON 解析用構造体 sem の array_end フィールドに本関数が割り当てられています。
