AssertArrayOrder

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_minmax_multi.c:278

目的

指定されたデータ配列が、提供された比較関数に基づいて昇順（または非降順）にソートされていることを検証するデバッグ用アサーション関数です。

シグネチャまたは形状

static void AssertArrayOrder(FmgrInfo *cmp, Oid colloid, Datum *values, int nvalues)

入力と出力

  • cmp: 比較に使用する FmgrInfo 構造体へのポインタ。
  • colloid: 比較に使用する照合順序（Collation）の OID。
  • values: 比較対象の Datum 配列。
  • nvalues: 配列内の要素数。
  • 戻り値: なし。

前提条件と不変条件

  • values 配列の各隣接要素 values[i] と values[i+1] について、cmp 関数が values[i] <= values[i+1]（または values[i] < values[i+1]）を真と判定すること。
  • アサーションが有効なビルド環境であること。

アルゴリズムまたは振る舞い

1. 0 から nvalues - 2 までのインデックスでループを実行します。 2. 各ステップで FunctionCall2Coll を呼び出し、隣接する2つの要素を比較します。 3. 比較結果が偽（false）の場合、Assert マクロによりプログラムが停止します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/brin/brin_minmax_multi.c 内の AssertCheckRanges および AssertCheckExpandedRanges から呼び出されています。
  • 呼び出し先:
  • FunctionCall2Coll（PostgreSQL の関数呼び出し基盤）。

関連データ構造

  • FmgrInfo: 関数呼び出し情報を保持する構造体。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • nvalues が 0 または 1 の場合、ループは実行されず、検証は行われません。
  • 比較関数が正しくソート順を判定できない場合、Assert によりプロセスがアボートします。

保守担当者向けメモ

  • 本関数はデバッグ専用です。本番環境のビルドでは最適化により除去される可能性があるため、実行時のロジックとして依存しないでください。
  • brin_minmax_multi.c 内のデータ構造（Ranges 等）の整合性チェックにおいて、ソート順序の不変条件を保証するために使用されています。
