Jsonb

定義

  • 種別: struct
  • 定義箇所: src/include/utils/jsonb.h:212

目的

PostgreSQLの jsonb データ型におけるルート構造体であり、バイナリ形式のJSONデータをメモリおよびディスク上で保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	int32		vl_len_;		/* varlena header (do not touch directly!) */
	JsonbContainer root;
} Jsonb;

入力と出力

  • **入力**: jsonb 型のデータは、JsonbParseState を介して構築され、JsonbValueToJsonb 関数によってこの構造体にシリアライズされます。
  • **出力**: DatumGetJsonbP 等のアクセサを介して、内部の JsonbContainer が後続の走査や検索処理に提供されます。

前提条件と不変条件

  • vl_len_ は varlena ヘッダーであり、直接操作してはなりません。
  • 構造体全体は4バイト境界でアライメントされます。
  • 可変長データ領域へのアクセス時には、パディングを考慮する必要があります。

アルゴリズムまたは振る舞い

Jsonb は、圧縮効率とランダムアクセス性能を両立させるため、JsonbContainer 内で JEntry 配列と実際のデータ領域を分離して保持します。この構造により、JsonbIterator を用いた再帰的な走査や、特定のキーに対する高速な検索が可能です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/jsonb_plperl および contrib/jsonb_plpython (データ変換)
  • src/backend/executor/execExprInterp.c (DatumGetJsonbP 経由)
  • src/backend/tsearch/to_tsany.c (jsonb_to_tsvector_worker 等)
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。操作には JsonbIterator や JsonbContainer を扱うAPI群が使用されます。

関連データ構造

  • JsonbContainer: Jsonb の実体となるコンテナ構造。
  • JEntry: JsonbContainer 内の各要素の型と位置を管理するヘッダー。
  • JsonbValue: メモリ上での操作用汎用構造体。
  • JsonbIterator: 構造を走査するためのイテレータ。

失敗モードと境界条件

  • JGIN_MAXLENGTH を超える値はハッシュ化されて格納されるため、インデックス検索時にはハッシュ値の衝突や一致判定のロジックが適用されます。
  • 不正な varlena ヘッダーやアライメント違反は、データ破損やセグメンテーション違反を引き起こす可能性があります。

保守担当者向けメモ

  • JEntry のオフセット計算ロジックは非常に繊細であるため、変更時には src/backend/utils/adt/jsonb.c の実装と整合性を確認してください。
  • GINインデックスの戦略番号（JsonbContainsStrategyNumber 等）は本ヘッダーで定義されており、インデックス演算子クラスの実装時に参照されます。
