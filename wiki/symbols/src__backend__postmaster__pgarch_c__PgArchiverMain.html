PgArchiverMain

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pgarch.c:217

目的

WAL（Write-Ahead Log）アーカイバプロセスのメインエントリポイントとして、プロセスの初期化、シグナルハンドラの設定、およびアーカイブ処理のメインループ実行を行う。

シグネチャまたは形状

void PgArchiverMain(const void *startup_data, size_t startup_data_len)

入力と出力

  • **入力**:
  • startup_data: 使用されない（startup_data_len == 0 である必要がある）。
  • startup_data_len: 0 であること。
  • **出力**: なし（プロセス終了時に proc_exit を呼び出す）。

前提条件と不変条件

  • XLogArchivingActive() が真であること。
  • Postmasterからフォークされた補助プロセスとして実行されること。

アルゴリズムまたは振る舞い

1. **初期化**: MyBackendType を B_ARCHIVER に設定し、AuxiliaryProcessMainCommon() を呼び出す。 2. **シグナル設定**: SIGHUP, SIGTERM, SIGUSR1, SIGUSR2, SIGQUIT 以外のシグナルを無視またはデフォルト設定にし、シグナルマスクを解除する。 3. **リソース確保**: on_shmem_exit に終了時処理（pgarch_die）を登録し、共有メモリ上の PgArch にプロセス番号を通知する。 4. **データ構造構築**: アーカイブ対象ファイル管理用の arch_files_state 構造体と優先度付きヒープを割り当てる。 5. **ライブラリロード**: LoadArchiveLibrary() を呼び出し、アーカイブモジュールをロードする。 6. **メインループ**: pgarch_MainLoop() を呼び出し、アーカイブ処理を継続する。 7. **終了**: ループ終了後、proc_exit(0) でプロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c（B_ARCHIVER プロセス起動時）。
  • **呼び出し先**: AuxiliaryProcessMainCommon, pqsignal, on_shmem_exit, binaryheap_allocate, AllocSetContextCreate, LoadArchiveLibrary, pgarch_MainLoop, proc_exit。

関連データ構造

  • PgArch (共有メモリ構造体)
  • arch_files_state (アーカイブ対象ファイル管理用)
  • binaryheap (アーカイブ優先度管理用)

失敗モードと境界条件

  • startup_data_len が 0 でない場合、Assert により異常終了する。
  • XLogArchivingActive() が偽の場合、Assert により異常終了する。
  • アーカイブライブラリのロード失敗時は LoadArchiveLibrary 内でエラーが発生する。

保守担当者向けメモ

  • 本関数はプロセスのライフサイクル管理を担当する。アーカイブ処理の詳細なロジックは pgarch_MainLoop およびその配下の関数群に委譲されている。
  • sigsetjmp によるエラーハンドリングは pgarch_MainLoop 内部で行われるため、本関数自体はプロセス終了を直接制御する。
