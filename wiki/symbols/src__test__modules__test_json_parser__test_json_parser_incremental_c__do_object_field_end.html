do_object_field_end

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_json_parser/test_json_parser_incremental.c:286

目的

JSONオブジェクトのフィールド処理終了時に呼び出され、フィールド名文字列のメモリ解放を管理するコールバック関数です。

シグネチャまたは形状

static JsonParseErrorType do_object_field_end(void *state, char *fname, bool isnull)

入力と出力

  • **入力**:
  • state: 状態保持用ポインタ（使用されません）。
  • fname: 終了したフィールド名の文字列ポインタ。
  • isnull: フィールド値がNULLかどうかを示すフラグ（使用されません）。
  • **出力**:
  • JSON_SUCCESS: 常に成功を返します。

前提条件と不変条件

  • lex_owns_tokens グローバル変数が、トークンのメモリ管理責任をパーサー側が負うか否かを決定します。

アルゴリズムまたは振る舞い

1. lex_owns_tokens が false の場合、fname が指すメモリ領域を free() で解放します。 2. lex_owns_tokens が true の場合、メモリ解放を行わずにそのまま終了します。 3. 常に JSON_SUCCESS を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: JSONパーサーのセマンティックアクションテーブル（JsonSemAction）を介して、パース処理中に呼び出されます。
  • **呼び出し先**: 標準ライブラリの free() を呼び出す可能性があります。

関連データ構造

  • JsonSemAction: 本関数が登録されるコールバックテーブル。
  • lex_owns_tokens: メモリ管理方針を決定するフラグ。

失敗モードと境界条件

  • fname が NULL の場合、lex_owns_tokens が false であれば free(NULL) が実行されます（標準Cライブラリの仕様により安全です）。

保守担当者向けメモ

  • 本関数はテストモジュール専用のコールバックです。lex_owns_tokens の状態に応じてメモリ管理を切り替えることで、メモリリークの有無を検証するテストをサポートしています。
