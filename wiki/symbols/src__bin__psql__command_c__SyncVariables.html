SyncVariables

定義

  • 種別: function
  • 定義箇所: src/bin/psql/command.c:4568

目的

データベース接続の状態（エンコーディング、バージョン、接続情報など）を psql の内部変数および設定構造体（pset）と同期させます。

シグネチャまたは形状

void SyncVariables(void)

入力と出力

  • **入力**: グローバル変数 pset を介して現在のデータベース接続（pset.db）を参照します。
  • **出力**: なし（副作用として pset の各フィールドおよび pset.vars に値を設定します）。

前提条件と不変条件

  • pset.db が有効なデータベース接続を保持している必要があります。

アルゴリズムまたは振る舞い

1. **接続情報の取得**: PQclientEncoding および PQserverVersion を使用して、現在の接続のエンコーディングとサーバーバージョンを pset に反映します。 2. **フォーマット設定**: setFmtEncoding を呼び出し、出力エンコーディングを更新します。 3. **変数同期**: 以下の情報を pset.vars に設定します。

  • DBNAME, USER, HOST, PORT, ENCODING
  • SERVICE（接続情報から取得）
  • SERVER_VERSION_NAME（サーバーから取得、または数値からフォーマット）
  • SERVER_VERSION_NUM（数値形式）

    • 4. **エラー設定**: PQsetErrorVerbosity および PQsetErrorContextVisibility を使用して、接続側のエラー報告設定を pset の設定と同期させます。

関連データ構造

  • pset (グローバルな psql 設定構造体)
  • pset.vars (変数管理用データ構造)

失敗モードと境界条件

  • service_name が取得できない場合、SERVICE 変数は設定されません。
  • PQparameterStatus がサーバーバージョンを返さない場合、pset.sversion を基にしたフォールバック処理が行われます。

保守担当者向けメモ

  • 本関数は接続確立直後や再接続時に呼び出されることを想定しています。
  • connection_warnings 関数と併用されることが多いため、変更時は呼び出し順序に注意してください。
