heap_get_latest_tid

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam.c:1868

目的

指定されたタプルの t_ctid リンクを辿り、更新チェーンの末尾にある最新のタプル TID を取得する。

シグネチャまたは形状

void heap_get_latest_tid(TableScanDesc sscan, ItemPointer tid)

入力と出力

  • **入力**:
  • sscan: テーブルスキャン記述子。対象リレーションとスナップショット情報を含む。
  • tid: 検索対象の初期 TID。関数終了時には、発見された最新の TID で上書きされる。
  • **出力**: なし（tid 引数を通じて結果を返す）。

前提条件と不変条件

  • 入力 tid は有効な ItemPointer であること。
  • sscan が参照するリレーションおよびスナップショットは有効であること。
  • t_ctid リンクはテーブル内で有効であると仮定される。

アルゴリズムまたは振る舞い

1. tid を開始点として、t_ctid リンクを辿るループを開始する。 2. 各ステップで対象ページを ReadBuffer で読み込み、共有ロック (BUFFER_LOCK_SHARE) を取得する。 3. ページ内のオフセットが不正な場合や、タプルが正常でない場合はループを終了する。 4. priorXmax を用いて、辿った先のタプルが期待される更新チェーン上のものか（XMIN の一致）を確認する。 5. HeapTupleSatisfiesVisibility を用いてタプルの可視性を判定し、可視であれば *tid を更新する。 6. t_infomask のフラグ（HEAP_XMAX_INVALID 等）や、t_ctid が自身を指している場合はチェーンの終端とみなし、ループを終了する。 7. 終端でない場合は t_ctid を次の対象とし、priorXmax を更新してループを継続する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c: heap_tableam_ops の .tuple_get_latest_tid として登録されている。
  • **呼び出し先**:
  • ReadBuffer, LockBuffer, BufferGetPage, UnlockReleaseBuffer (バッファ管理)
  • HeapTupleSatisfiesVisibility (可視性判定)
  • HeapCheckForSerializableConflictOut (シリアライザブル競合チェック)

関連データ構造

  • TableScanDesc: スキャン状態を保持。
  • ItemPointer: タプルの物理位置（ブロック番号とオフセット）。
  • HeapTupleHeader: タプルのヘッダー情報（t_ctid, t_infomask 等）。

失敗モードと境界条件

  • **不正なオフセット**: ページ内のオフセットが範囲外の場合、エラーにはせず、直前の有効な TID を保持して終了する。
  • **リンクの不一致**: priorXmax と XMIN が一致しない場合、チェーンの追跡を停止する。
  • **可視性**: スナップショットに基づき可視でないタプルは、最新 TID の候補として採用されない。

保守担当者向けメモ

  • 本関数は Table Access Method インターフェースの一部であり、ヒープテーブルの更新履歴を追跡する重要な役割を持つ。
  • t_ctid リンクを辿る際、ページロックの取得と解放をループ内で行うため、デッドロックやバッファ管理の整合性に注意すること。
