ConstrCheck

定義

  • 種別: struct
  • 定義箇所: src/include/access/tupdesc.h:28

目的

テーブルのチェック制約（CHECK constraint）に関するメタデータを保持する構造体です。

シグネチャまたは形状

typedef struct ConstrCheck
{
	char	   *ccname;
	char	   *ccbin;			/* nodeToString representation of expr */
	bool		ccenforced;
	bool		ccvalid;
	bool		ccnoinherit;	/* this is a non-inheritable constraint */
} ConstrCheck;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。ただし、リレーションのキャッシュ構築時やタプル記述子のコピー時に、この構造体の配列がメモリ確保およびコピーの対象となります。

前提条件と不変条件

  • ccbin は、制約式を nodeToString 形式で表現した文字列を保持します。
  • 制約チェックの整合性を保つため、ConstrCheck のエントリは特定の順序でソートされていることが期待される場合があります（src/backend/access/common/tupdesc.c:729 の記述に基づく）。

アルゴリズムまたは振る舞い

  • **メモリ管理**: TupleDesc が保持する TupleConstr 構造体の一部として、palloc を用いて動的に確保されます。
  • **ソート**: relcache.c 等において、制約の比較を行うために qsort を用いて CheckConstraintCmp 関数によるソートが行われます。
  • **コピー**: TupleDesc の複製時、memcpy を用いて ConstrCheck の配列全体が新しいメモリ領域へコピーされます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/common/tupdesc.c: タプル記述子のコピーや制約の比較処理。
  • src/backend/commands/tablecmds.c: テーブルコマンド処理。
  • src/backend/executor/execMain.c: 実行エンジンによる制約確認。
  • src/backend/utils/cache/relcache.c: リレーションキャッシュの構築およびソート処理。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • TupleConstr: ConstrCheck の配列を保持する親構造体。
  • TupleDesc: TupleConstr を介して本構造体を参照するタプル記述子。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保に失敗した場合、システムはエラーを発生させます。
  • 不整合: ConstrCheck の配列がソートされていない場合、制約の比較や検証処理において予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • ccbin は文字列として保持されるため、メモリリークや不正なポインタ参照に注意してください。
  • ConstrCheck の配列を操作する際は、必ず TupleConstr 内の num_check との整合性を維持してください。
  • ソート順序に依存するロジック（CheckConstraintCmp）を変更する場合は、関連するすべての参照箇所でソート要件が満たされているか確認してください。
