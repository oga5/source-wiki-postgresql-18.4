ExecScanReScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execScan.c:107

目的

スキャンノードの再実行（ReScan）時に、スキャン状態のクリアおよび EvalPlanQual (EPQ) 状態の同期を行う。

シグネチャまたは形状

void ExecScanReScan(ScanState *node)

入力と出力

  • **入力**: ScanState *node（再スキャン対象のスキャンノード状態）
  • **出力**: なし（副作用として node の状態および EState 内の EPQ 状態を更新する）

前提条件と不変条件

  • node->ss_ScanTupleSlot が有効なスロットであること。
  • EPQ 再チェック中の場合、node->ps.state->es_epq_active が適切に設定されていること。

アルゴリズムまたは振る舞い

1. **スキャンタプルのクリア**: ExecClearTuple を呼び出し、スキャンノードが現在タプルを保持していない状態にする。 2. **EPQ 状態の同期**: estate->es_epq_active が存在する場合、以下の処理を行う。

  • 現在のスキャンノードが参照するリレーション（scanrelid）を特定する。
  • ForeignScan または CustomScan の場合は、複数のリレーション ID を走査する。
  • 特定されたリレーションの relsubs_done フラグを、対応する relsubs_blocked フラグの値で更新し、ブロック状態を維持しつつ再スキャンを準備する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nodeBitmapHeapscan.c, nodeCtescan.c, nodeForeignscan.c, nodeFunctionscan.c, nodeIndexonlyscan.c, nodeIndexscan.c, nodeNamedtuplestorescan.c, nodeSamplescan.c, nodeSeqscan.c, nodeSubqueryscan.c 等の各スキャンノード実装における ReScan 関数。
  • **呼び出し先**:
  • ExecClearTuple
  • bms_next_member（複数リレーションの場合）
  • nodeTag / IsA（ノード種別判定）

関連データ構造

  • ScanState: スキャンノードの実行状態を保持する構造体。
  • EState: 実行エンジン全体の状態。
  • EPQState: EvalPlanQual 実行時の状態管理構造体。

失敗モードと境界条件

  • node->ps.plan が ForeignScan または CustomScan 以外の場合、elog(ERROR) により異常終了する。
  • scanrelid が 0 以下で、かつ対象が ForeignScan/CustomScan でない場合も同様にエラーとなる。

保守担当者向けメモ

  • 本関数は、スキャンノードの再実行時に「現在位置」をリセットする重要な役割を担う。
  • EPQ 実行中のフラグ管理（relsubs_done）は、同時実行制御の整合性に直結するため、変更時は nodeModifyTable.c 等の EPQ 関連ロジックとの整合性に注意すること。
