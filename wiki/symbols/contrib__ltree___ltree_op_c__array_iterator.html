array_iterator

定義

  • 種別: function
  • 定義箇所: contrib/ltree/_ltree_op.c:37

目的

ltree 型の配列を走査し、各要素に対して指定されたコールバック関数を適用して条件判定を行う。

シグネチャまたは形状

static bool array_iterator(ArrayType *la, PGCALL2 callback, void *param, ltree **found)

入力と出力

  • **入力**:
  • la: 走査対象の ltree 配列 (ArrayType*)。
  • callback: 各要素に対して実行する関数ポインタ (PGCALL2)。
  • param: コールバック関数に渡す追加パラメータ (void*)。
  • found: 条件に合致した要素へのポインタを格納するアドレス（不要な場合は NULL を指定）。
  • **出力**:
  • 戻り値: コールバックが真を返した場合は true、それ以外は false。
  • found: 条件に合致した最初の要素へのポインタ（found が非 NULL の場合）。

前提条件と不変条件

  • 入力配列 la は1次元配列でなければならない。
  • 入力配列 la は NULL 値を含んではならない。
  • 上記条件に違反した場合、ereport によりエラーが発生する。

アルゴリズムまたは振る舞い

1. 配列の次元数と要素数を取得する。 2. 配列の次元数が1を超えているか、または NULL を含んでいる場合にエラーを報告する。 3. found が指定されている場合、初期値を NULL に設定する。 4. 配列の各要素に対してループ処理を行う：

  • callback 関数を DirectFunctionCall2 を介して呼び出し、現在の要素と param を渡す。
  • コールバックが真を返した場合、found に現在の要素ポインタを格納し、即座に true を返す。
  • 次の要素へ移動する。

    • 5. 全要素を走査しても条件を満たさない場合、false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/_ltree_op.c 内の各関数（ltree_isparent, ltree_risparent, ltq_regex, ltxtq_exec 等を用いた検索処理）。
  • **呼び出し先**: DirectFunctionCall2 を介して引数 callback に渡された関数。

関連データ構造

  • ArrayType: PostgreSQLの配列構造体。
  • ltree: ltree モジュールの主要データ型。

失敗モードと境界条件

  • 配列が多次元の場合: ERRCODE_ARRAY_SUBSCRIPT_ERROR でエラー。
  • 配列に NULL が含まれる場合: ERRCODE_NULL_VALUE_NOT_ALLOWED でエラー。

保守担当者向けメモ

  • 本関数は ltree 配列に対する汎用的なイテレータとして機能する。
  • src/backend/utils/adt/array_userfuncs.c に存在する同名のシンボルとの混同に注意すること（本関数は static であり、contrib/ltree モジュール内でのみ使用される）。
