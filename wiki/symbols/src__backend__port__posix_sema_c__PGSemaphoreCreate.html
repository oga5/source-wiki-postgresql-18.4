PGSemaphoreCreate

定義

  • 種別: function
  • 定義箇所: src/backend/port/posix_sema.c:260

目的

PostgreSQL のプロセス間同期用セマフォ（PGSemaphore）を新規に作成し、初期化します。

シグネチャまたは形状

PGSemaphore PGSemaphoreCreate(void);

入力と出力

  • **入力**: なし
  • **出力**: 作成されたセマフォへのポインタ（PGSemaphore 型）を返します。

前提条件と不変条件

  • IsUnderPostmaster が false であること（Postmaster 起動後のバックエンドプロセス内では呼び出し不可）。
  • numSems が maxSems 未満であること。

アルゴリズムまたは振る舞い

1. IsUnderPostmaster を確認し、違反している場合はアサーションエラーを発生させます。 2. 現在のセマフォ数が上限に達していないか確認し、超過している場合は PANIC で終了します。 3. コンパイル時フラグ USE_NAMED_POSIX_SEMAPHORES の設定に応じて分岐します。

  • **有効時**: PosixSemaphoreCreate() を呼び出し、名前付きセマフォを生成して mySemPointers に記録します。
  • **無効時**: 共有メモリ上の sharedSemas 配列からスロットを確保し、PG_SEM_REF で取得したアドレスに対して PosixSemaphoreCreate() を実行します。

    • 4. numSems をインクリメントし、作成したセマフォを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/proc.c: proc->sem = PGSemaphoreCreate();
  • **呼び出し先**:
  • PosixSemaphoreCreate (内部関数)
  • elog (エラー発生時)

関連データ構造

  • PGSemaphore: sem_t * を抽象化したポインタ型。
  • sharedSemas: 名前なしセマフォ使用時に利用される共有メモリ上のセマフォ配列。
  • mySemPointers: 名前付きセマフォ使用時に管理されるポインタ配列。

失敗モードと境界条件

  • **上限超過**: numSems >= maxSems の場合、PANIC によりプロセスが異常終了します。
  • **アサーション違反**: IsUnderPostmaster が真の場合、アサーションにより停止します。

保守担当者向けメモ

  • 本関数は共有メモリの初期化フェーズで呼び出されることを想定しています。
  • USE_NAMED_POSIX_SEMAPHORES の有無により、セマフォの管理場所（共有メモリ内か、OS 管理の名前付きセマフォか）が大きく異なるため、プラットフォームごとの挙動に注意してください。
