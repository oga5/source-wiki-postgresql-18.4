RectBox

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/geo_spgist.c:115

目的

SP-GiSTインデックスを用いた空間検索において、4次元空間（x_low, x_high, y_low, y_high）における現在の探索範囲を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	RangeBox	range_box_x;
	RangeBox	range_box_y;
} RectBox;

入力と出力

  • 本構造体は、探索中の再帰的な木降下プロセスにおいて、現在の境界条件を保持するコンテナとして機能します。

前提条件と不変条件

  • 2次元の矩形（BOX）を4次元空間の点として扱い、インデックス構築および検索時のフィルタリングに使用されます。
  • 探索中に動的に生成されるため、traversalMemoryContext を通じて管理されます。

アルゴリズムまたは振る舞い

  • 検索時、inner_consistent 関数内で再帰的に木を降下する際、nextRectBox 関数によって次の探索範囲が計算されます。
  • overlap4D, contain4D, contained4D などの関数において、クエリ条件との適合性を判定するために参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/geo_spgist.c 内の nextRectBox, overlap4D, contain4D, contained4D 等の関数。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • RangeBox: RectBox を構成するX軸およびY軸の範囲情報。

失敗モードと境界条件

  • 動的にメモリ確保（palloc）されるため、不要になった RectBox はメモリリークを防ぐために pfree する必要があります。

保守担当者向けメモ

  • 本実装は、R-Treeが苦手とする重なりが多いデータに対して、4次元四分木としてマッピングを行うための重要なデータ構造です。
  • 検索の正確性は is_bounding_box_test_exact に依存しており、近似検索を行う場合は recheck フラグの管理に注意してください。
