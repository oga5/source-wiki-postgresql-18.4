injection_stats_fixed_init_shmem_cb

定義

  • **種別**: 関数
  • **定義箇所**: src/test/modules/injection_points/injection_stats_fixed.c:72

目的

固定数統計（fixed-numbered statistics）用の共有メモリ領域を初期化し、統計データ保護のための軽量ロック（LWLock）をセットアップする。

シグネチャまたは形状

static void injection_stats_fixed_init_shmem_cb(void *stats)

入力と出力

  • **入力**: void *stats - 初期化対象となる共有メモリ上の PgStatShared_InjectionPointFixed 構造体へのポインタ。
  • **出力**: なし（副作用として共有メモリ内のロックが初期化される）。

前提条件と不変条件

  • 引数 stats は PgStatShared_InjectionPointFixed 構造体としてキャスト可能であること。
  • PostgreSQLの共有メモリ初期化フェーズにおいて、pgstat フレームワークから呼び出されることを前提とする。

アルゴリズムまたは振る舞い

1. 引数 stats を PgStatShared_InjectionPointFixed * 型にキャストする。 2. キャストした構造体内の lock メンバに対し、LWLockInitialize を実行する。 3. ロックのトランシェ（Tranche）として LWTRANCHE_PGSTATS_DATA を割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat フレームワークの統計登録プロセス（injection_stats_fixed 構造体の .init_shmem_cb フィールド経由で参照）。
  • **呼び出し先**: LWLockInitialize（PostgreSQLコアのロック管理関数）。

関連データ構造

  • PgStatShared_InjectionPointFixed: 統計カウンタとロックを保持する共有メモリ構造体。

失敗モードと境界条件

  • 引数が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • 共有メモリが適切に割り当てられていない状態で呼び出された場合、メモリ破壊を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は pgstat のカスタム統計登録時に一度だけ実行される。
  • ロックの競合が発生する場合は、LWTRANCHE_PGSTATS_DATA の使用状況を確認すること。
