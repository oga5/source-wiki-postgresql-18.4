CopyToTextLikeOutFunc

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyto.c:232

目的

COPY TO 操作において、特定の属性（カラム）をテキストまたはCSV形式で出力するための関数情報を取得・設定します。

シグネチャまたは形状

static void CopyToTextLikeOutFunc(CopyToState cstate, Oid atttypid, FmgrInfo *finfo)

入力と出力

  • **入力**:
  • cstate: COPY TO 操作の状態を保持する構造体。
  • atttypid: 出力対象の属性のデータ型OID。
  • finfo: 取得した出力関数情報を格納するための FmgrInfo 構造体へのポインタ。
  • **出力**: なし（finfo を介して副作用として出力関数情報を設定）。

前提条件と不変条件

  • atttypid は有効なデータ型OIDである必要があります。
  • finfo は書き込み可能なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. getTypeOutputInfo を呼び出し、指定された atttypid に対応する出力関数（func_oid）と、その型が可変長かどうか（is_varlena）をシステムカタログから取得します。 2. fmgr_info を呼び出し、取得した func_oid に基づいて finfo 構造体を初期化します。これにより、後続のデータ変換処理でこの関数ポインタが利用可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyto.c 内の CopyToTextLikeStart 等から参照されています。
  • **呼び出し先**:
  • getTypeOutputInfo（システムカタログ検索）
  • fmgr_info（関数マネージャによる情報設定）

関連データ構造

  • CopyToState: COPY TO 操作の全状態を保持する構造体。
  • FmgrInfo: PostgreSQLの関数マネージャが使用する、関数呼び出しに必要な情報を保持する構造体。

失敗モードと境界条件

  • 無効な atttypid が渡された場合、getTypeOutputInfo 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はテキスト形式およびCSV形式の出力ルーチンで共通して利用されます。
  • CopyToRoutine 構造体の CopyToOutFunc フィールドに割り当てられる関数です。
