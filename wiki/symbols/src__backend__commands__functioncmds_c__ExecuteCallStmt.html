ExecuteCallStmt

定義

  • 種別: function
  • 定義箇所: src/backend/commands/functioncmds.c:2205

目的

SQLの CALL 文を実行し、指定されたプロシージャを呼び出す。

シグネチャまたは形状

void ExecuteCallStmt(CallStmt *stmt, ParamListInfo params, bool atomic, DestReceiver *dest)

入力と出力

  • **入力**:
  • stmt: 実行対象の CallStmt 構造体。
  • params: 実行時のパラメータリスト。
  • atomic: トランザクション制御を許可するかどうかを示すフラグ。
  • dest: 結果を受け取るための DestReceiver。
  • **出力**: なし（副作用としてプロシージャの実行および結果の送信を行う）。

前提条件と不変条件

  • stmt->funcexpr は FuncExpr である必要がある。
  • プロシージャの引数が FUNC_MAX_ARGS を超えてはならない。
  • proconfig が設定されている、または prosecdef（セキュリティ定義者）である場合、atomic は強制的に true となる。

アルゴリズムまたは振る舞い

1. **権限チェック**: object_aclcheck を使用し、対象プロシージャの実行権限を検証する。 2. **コンテキスト準備**: CallContext を作成し、atomic フラグを設定する。 3. **実行準備**: fmgr_info を用いて関数情報を取得し、FunctionCallInfoData を初期化する。 4. **引数評価**: ExecutorState を作成し、ExecPrepareExpr と ExecEvalExprSwitchContext を用いて引数を評価する。非アトミックな場合はトランザクションスナップショットを一時的にプッシュする。 5. **実行**: FunctionCallInvoke を通じてプロシージャを呼び出す。 6. **結果処理**: 戻り値が RECORDOID の場合、DestReceiver を介して結果をクライアントへ送信する。 7. **クリーンアップ**: ExecutorState を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c:854 (直接の呼び出し)
  • **呼び出し先**:
  • object_aclcheck, fmgr_info, FunctionCallInvoke, ExecEvalExprSwitchContext, CreateExecutorState 等の PostgreSQL 内部 API。

関連データ構造

  • CallStmt: 実行対象の構文木。
  • CallContext: プロシージャ実行時のコンテキスト情報。
  • FunctionCallInfoData: 関数呼び出しに必要なメタデータ。
  • EState: 実行状態を保持する構造体。

失敗モードと境界条件

  • **権限エラー**: 実行権限がない場合は aclcheck_error を発生させる。
  • **キャッシュエラー**: PROCOID のシステムキャッシュ検索に失敗した場合は elog(ERROR) を発生させる。
  • **引数過多**: 引数が FUNC_MAX_ARGS を超える場合はエラーを報告する。
  • **戻り値エラー**: 予期しない戻り値型や、RECORDOID で NULL が返された場合はエラーを発生させる。

保守担当者向けメモ

  • atomic 引数は、トランザクション制御（COMMIT/ROLLBACK）の可否を決定する重要なフラグである。
  • プロシージャが RECORDOID を返す際の TOAST ポインタの扱いに注意が必要（EnsurePortalSnapshotExists を使用しているが、制約が残っている）。
