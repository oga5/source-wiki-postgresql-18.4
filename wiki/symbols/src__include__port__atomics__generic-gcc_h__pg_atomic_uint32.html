pg_atomic_uint32

定義

  • 種別: struct
  • 定義箇所: src/include/port/atomics/generic-gcc.h:90

目的

原子操作（Atomic Operations）の対象となる32ビット符号なし整数値を保持し、コンパイラやハードウェアによる最適化の影響を受けずに安全な読み書きを保証するためのラッパー構造体です。

シグネチャまたは形状

typedef struct pg_atomic_uint32
{
	volatile uint32 value;
} pg_atomic_uint32;

入力と出力

  • **入力**: 構造体内の value メンバに uint32 型の値を格納します。
  • **出力**: volatile 修飾子により、コンパイラによる最適化（レジスタへのキャッシュ等）を抑制し、メモリ上の最新値を参照します。

前提条件と不変条件

  • atomics.h を経由してインクルードされる必要があります（INSIDE_ATOMICS_H マクロのガードが必要）。
  • 本構造体に対する操作は、pg_atomic_compare_exchange_u32_impl 等の専用の原子操作関数を介して行うことが前提です。

アルゴリズムまたは振る舞い

  • volatile 修飾子により、コンパイラに対してこのメモリ領域が外部から変更される可能性があることを通知します。
  • GCCの組み込み関数（__atomic または __sync）と組み合わせて使用することで、マルチスレッド環境下でのデータ競合を防ぐ原子的な読み取り・書き込み・演算を実現します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/commands/vacuum.c
  • src/backend/commands/vacuumparallel.c
  • src/backend/nodes/tidbitmap.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/pgarch.c
  • src/backend/storage/buffer/freelist.c
  • src/backend/storage/ipc/procsignal.c
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。操作には pg_atomic_*_u32_impl 系の関数が使用されます。

関連データ構造

  • pg_atomic_uint64: 64ビット版の原子操作用構造体。
  • pg_atomic_flag: フラグ操作用の原子構造体。

失敗モードと境界条件

  • 構造体のアライメントが不適切な場合、プラットフォームによっては原子操作が効率的に行われない、あるいは失敗する可能性があります。
  • 64ビット環境と32ビット環境で uint32 のサイズが異なることはありませんが、ハードウェアが原子操作をサポートしていない場合は、コンパイラが提供するフォールバック実装に依存します。

保守担当者向けメモ

  • 本構造体にはインクルードガードが意図的に設定されていません。必ず atomics.h を通じて利用してください。
  • volatile 修飾子はコンパイラの最適化を抑制しますが、メモリバリア（pg_memory_barrier_impl）の代わりにはなりません。原子操作を行う際は必ず適切なバリア関数を併用してください。
