ReplicationSlotDropAtPubNode

定義

  • 種別: function
  • 定義箇所: src/backend/commands/subscriptioncmds.c:1914

目的

リモートのパブリッシャーノード上で、指定された論理レプリケーションスロットを削除します。

シグネチャまたは形状

void ReplicationSlotDropAtPubNode(WalReceiverConn *wrconn, char *slotname, bool missing_ok)

入力と出力

  • **入力**:
  • wrconn: パブリッシャーへの有効な WalReceiverConn 接続。
  • slotname: 削除対象のスロット名。
  • missing_ok: true の場合、スロットが存在しなくてもエラーにせず LOG レベルで記録します。
  • **出力**: なし（成功時は NOTICE を発行）。

前提条件と不変条件

  • wrconn は非 NULL である必要があります（Assert で検証）。
  • libpqwalreceiver モジュールがロード可能である必要があります。

アルゴリズムまたは振る舞い

1. libpqwalreceiver をロードします。 2. DROP_REPLICATION_SLOT <slotname> WAIT コマンド文字列を構築します。 3. walrcv_exec を使用してリモートノードでコマンドを実行します。 4. 実行結果のステータスに応じて以下の処理を行います：

  • WALRCV_OK_COMMAND: 削除成功を NOTICE として報告します。
  • WALRCV_ERROR かつ missing_ok が true かつエラーコードが ERRCODE_UNDEFINED_OBJECT: 削除対象が存在しない旨を LOG として記録します。
  • 上記以外: ERRCODE_CONNECTION_FAILURE を伴うエラーを発生させます。

    • 5. PG_FINALLY ブロックにより、コマンド文字列のメモリを確実に解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/subscriptioncmds.c 内のサブスクリプション管理処理。
  • src/backend/replication/logical/tablesync.c 内のテーブル同期処理。
  • **呼び出し先**:
  • walrcv_exec（リモートコマンド実行）
  • walrcv_clear_result（結果セットの解放）

関連データ構造

  • WalReceiverConn: パブリッシャーとの接続状態を保持する構造体。
  • WalRcvExecResult: リモートコマンドの実行結果を保持する構造体。

失敗モードと境界条件

  • リモートノードとの接続が切断されている場合、walrcv_exec がエラーを返します。
  • missing_ok が false の状態でスロットが存在しない場合、エラーが発生します。
  • quote_identifier を使用しているため、スロット名に特殊文字が含まれる場合も適切に処理されます。

保守担当者向けメモ

  • 本関数は PG_TRY/PG_FINALLY を使用して、リモート接続に関連するリソースのリークを防いでいます。修正を加える際は、この例外安全性を維持してください。
