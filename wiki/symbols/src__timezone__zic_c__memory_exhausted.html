memory_exhausted

定義

  • 種別: function
  • 定義箇所: src/timezone/zic.c:410

目的

メモリ割り当ての失敗やリソース不足が発生した際に、エラーメッセージを標準エラー出力へ表示し、プログラムを異常終了させるためのユーティリティ関数です。

シグネチャまたは形状

static void memory_exhausted(const char *msg)

入力と出力

  • **入力**: msg (const char *) - 発生したエラーの詳細を示すメッセージ文字列。
  • **出力**: なし（関数は呼び出し元へ戻らず、プロセスを終了します）。

前提条件と不変条件

  • プログラムのグローバル変数 progname が適切に初期化されている必要があります。
  • 本関数が呼び出された場合、プログラムは EXIT_FAILURE を返して即座に終了します。

アルゴリズムまたは振る舞い

1. progname（プログラム名）と引数 msg を含むエラーメッセージを stderr に出力します。 2. exit(EXIT_FAILURE) を呼び出し、プロセスを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/timezone/zic.c 内のメモリ確保失敗時や、サイズ・整数オーバーフロー検出箇所から呼び出されます。
  • 具体的な呼び出し箇所: size_product 関数内、およびメモリ確保に関連するエラーハンドリング箇所（strerror(errno) を伴う呼び出しなど）。
  • **呼び出し先**:
  • fprintf (標準ライブラリ)
  • exit (標準ライブラリ)

関連データ構造

  • なし。

失敗モードと境界条件

  • msg が NULL の場合、fprintf の動作は未定義となります。
  • stderr への書き込みが失敗した場合でも、exit は実行されます。

保守担当者向けメモ

  • 本関数は zic ユーティリティ全体で共通のエラー終了処理として機能します。
  • メッセージには _() マクロが適用されており、国際化（i18n）に対応しています。
