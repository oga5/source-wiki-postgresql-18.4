TableScanDescData

定義

  • 種別: struct
  • 定義箇所: src/include/access/relscan.h:33

目的

テーブルスキャン実行時の状態を保持する基底データ構造。特定のアクセスメソッド（AM）に依存しない汎用的なスキャンパラメータを管理し、テーブルAMがスキャンの進捗や設定を追跡するための共通インターフェースを提供する。

シグネチャまたは形状

typedef struct TableScanDescData {
    Relation rs_rd;
    struct SnapshotData *rs_snapshot;
    int rs_nkeys;
    struct ScanKeyData *rs_key;
    union {
        TBMIterator rs_tbmiterator;
        struct {
            ItemPointerData rs_mintid;
            ItemPointerData rs_maxtid;
        } tidrange;
    } st;
    uint32 rs_flags;
    struct ParallelTableScanDescData *rs_parallel;
} TableScanDescData;

入力と出力

  • **入力**: スキャン開始時に Relation、Snapshot、およびオプションの ScanKey が設定される。
  • **出力**: スキャン実行中、st（共用体）内のイテレータやTID範囲、および rs_flags がスキャンの進捗に応じて更新される。

前提条件と不変条件

  • 本構造体は、特定のAM固有の構造体（例: HeapScanDescData）の先頭メンバとして配置されることが想定されている。
  • rs_rd は有効なリレーション記述子を指している必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: エグゼキュータが本構造体を割り当て、スナップショットやスキャンキーを設定する。 2. **スキャン実行**: テーブルAM（heapam 等）が本構造体を参照し、rs_flags を確認しながらタプルをフェッチする。 3. **並列処理**: 並列スキャン時には rs_parallel を介して共有メモリ上の並列スキャン状態にアクセスし、ワーカー間で処理ブロックを調整する。

呼び出し元と呼び出し先

  • **呼び出し元**: エグゼキュータ（src/include/nodes/execnodes.h の ss_currentScanDesc 等）がスキャン管理のために参照する。
  • **呼び出し先**: 構造体内部で SnapshotData、ScanKeyData、ParallelTableScanDescData を参照する。具体的な関数呼び出し関係については、提供されたソース範囲内では詳細な情報が不足している。

関連データ構造

  • Relation: スキャン対象のリレーション。
  • SnapshotData: 可視性判定に使用するスナップショット。
  • ScanKeyData: スキャン条件。
  • ParallelTableScanDescData: 並列スキャン時の共有状態。
  • TBMIterator: ビットマップスキャン用イテレータ。

失敗モードと境界条件

  • rs_parallel が設定されている場合、共有メモリ上の並列スキャン状態が正しく初期化されていないと、並列ワーカー間でのブロック割り当てで不整合が生じる可能性がある。

保守担当者向けメモ

  • 本構造体は多くのAMで「継承」的に利用されるため、メンバの追加や順序変更は、これを利用する全AMの実装に影響を与える可能性がある。
  • 並列スキャン関連のメンバは共有メモリ配置を考慮する必要がある。
