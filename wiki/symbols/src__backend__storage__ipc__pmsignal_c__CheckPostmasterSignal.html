CheckPostmasterSignal

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:181

目的

Postmaster（親プロセス）が受信した特定のシグナル理由（PMSignalReason）を確認し、該当するフラグが立っている場合にそれをクリアして通知を受け取る。

シグネチャまたは形状

bool CheckPostmasterSignal(PMSignalReason reason)

入力と出力

  • **入力**: reason (確認対象のシグナル理由を示す列挙型)
  • **出力**: bool (指定された理由のフラグがセットされていた場合は true、それ以外は false)

前提条件と不変条件

  • PMSignalState が初期化済みであること。
  • PMSignalFlags は sig_atomic_t であり、アトミックな読み書きが保証されていること。

アルゴリズムまたは振る舞い

1. 共有メモリ上の PMSignalState->PMSignalFlags 配列から、指定された reason に対応するフラグを確認する。 2. フラグが true であれば、そのフラグを false にリセット（クリア）し、true を返す。 3. フラグが false であれば、何も変更せず false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c 内のメインループ（PMSIGNAL_RECOVERY_STARTED, PMSIGNAL_ROTATE_LOGFILE, PMSIGNAL_START_WALRECEIVER 等の各理由に対して呼び出される）。
  • **呼び出し先**: 本関数は直接他の関数を呼び出さない。

関連データ構造

  • PMSignalData: 共有メモリ上に配置され、PMSignalFlags を保持する構造体。
  • PMSignalReason: シグナルの理由を定義する列挙型。

失敗モードと境界条件

  • reason が PMSignalFlags 配列の範囲外である場合、メモリ破壊の可能性がある（本関数内での境界チェックは行われていない）。
  • フラグがセットされていない状態で呼び出した場合、単に false を返す。

保守担当者向けメモ

  • 本関数は SIGUSR1 ハンドラ等でセットされたフラグを消費するために使用される。
  • フラグをクリアする前に状態を確認する設計となっており、一度の通知に対して一度だけ処理が実行されるよう制御されている。
