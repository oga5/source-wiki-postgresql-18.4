IndexTupleHasVarwidths

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/itup.h:83

目的

インデックスタプル（IndexTupleData）内に可変長属性が含まれているかどうかを判定します。

シグネチャまたは形状

static inline bool IndexTupleHasVarwidths(const IndexTupleData *itup)

入力と出力

  • **入力**: itup - 検査対象の IndexTupleData 構造体へのポインタ。
  • **出力**: bool - 可変長属性が含まれている場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • itup は有効な IndexTupleData を指している必要があります。
  • IndexTupleData の t_info フィールドが正しく初期化されていることを前提とします。

アルゴリズムまたは振る舞い

itup->t_info（16ビットのフラグ領域）に対して INDEX_VAR_MASK をビット単位の論理積（AND）で適用し、その結果が非ゼロであるかどうかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • contrib/pageinspect/btreefuncs.c
  • src/backend/access/common/indextuple.c
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（ビット演算のみを実行）。

関連データ構造

  • IndexTupleData: インデックスタプルのヘッダー構造体。
  • INDEX_VAR_MASK: t_info 内で可変長属性の有無を示すビットマスク。

失敗モードと境界条件

  • itup が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、インライン関数として定義されています。
  • t_info のビット定義を変更する場合は、INDEX_VAR_MASK の整合性に注意してください。
