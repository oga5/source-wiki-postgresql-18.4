IpcMemoryId

定義

  • 種別: typedef int
  • 定義箇所: src/backend/port/sysv_shmem.c:71

目的

System V 共有メモリセグメントを識別するための整数型です。shmget(2) システムコールによって返される共有メモリ識別子を保持するために使用されます。

シグネチャまたは形状

typedef int IpcMemoryId;

入力と出力

  • 入力: shmget(2) の戻り値として生成されます。
  • 出力: PGSharedMemoryAttach などの共有メモリ操作関数に引数として渡されます。

前提条件と不変条件

  • System V 共有メモリ API をサポートするプラットフォームでのみ有効です。
  • PostgreSQL プロセスが特定のデータディレクトリに関連付けられた共有メモリセグメントを操作する際の識別子として機能します。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、具体的なロジックは持ちません。この型を使用して保持される値は、PGSharedMemoryAttach 等の関数内で、対象のセグメントが現在のデータディレクトリに関連しているか、あるいは他プロセスによって使用中であるかを判定するためのキーとして利用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/port/sysv_shmem.c: PGSharedMemoryAttach の引数として使用。
  • src/backend/port/sysv_shmem.c: ローカル変数 shmid の型として使用。
  • 呼び出し先:
  • 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • IpcMemoryState: IpcMemoryId で識別されるセグメントの状態（SHMSTATE_ATTACHED, SHMSTATE_UNATTACHED 等）を分類する列挙型。

失敗モードと境界条件

  • shmget(2) が失敗した場合、無効な IpcMemoryId が生成される可能性があります。
  • 既存の共有メモリセグメントを再利用する際、異なるデータディレクトリ間でのキーの競合を避けるため、識別子の管理には慎重な分析が求められます。

保守担当者向けメモ

  • 本シンボルは sysv_shmem.c 内でのみ使用される内部的な型です。
  • IpcMemoryId を扱う際は、常に IpcMemoryState を用いた状態確認を併用し、意図しないセグメントへのアタッチを防ぐ設計となっています。
