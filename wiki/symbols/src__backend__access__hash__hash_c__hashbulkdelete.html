hashbulkdelete

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:464

目的

ハッシュインデックスに対する VACUUM 処理の一環として、不要となったタプルを削除し、インデックスの統計情報を更新する。

シグネチャまたは形状

IndexBulkDeleteResult *hashbulkdelete(IndexVacuumInfo *info, IndexBulkDeleteResult *stats, IndexBulkDeleteCallback callback, void *callback_state)

入力と出力

  • **入力**:
  • info: インデックスの関連情報（リレーションや戦略など）。
  • stats: 既存の削除統計情報（NULL の場合は新規作成）。
  • callback: 不要タプル判定用のコールバック関数。
  • callback_state: コールバック関数に渡す状態データ。
  • **出力**:
  • 更新された IndexBulkDeleteResult 構造体へのポインタ。

前提条件と不変条件

  • cachedmetap はハッシュインデックスのメタページ情報を保持している必要がある。
  • 処理中にバケットの分割（split）が発生した場合、メタページ情報を再取得して整合性を維持する。
  • hashbucketcleanup を呼び出す際、対象バケットのプライマリページに対してクリーンアップロックを保持し、並行スキャンとの競合を回避する。

アルゴリズムまたは振る舞い

1. メタページをキャッシュし、現在の最大バケット数を特定する。 2. 全バケットを順次走査する。

  • 各バケットのプライマリページをロックし、分割の必要性や状態を確認する。
  • hashbucketcleanup を呼び出し、バケット内の不要タプルを削除する。

    • 3. 走査中にバケットの分割が発生した場合は、追加されたバケットを含めて走査を継続する。 4. 最後にメタページを排他ロックし、インデックス全体のタプル数を更新する。

  • 走査中に分割や挿入がなかった場合は正確な値を設定し、あった場合は削除数に基づく推計値で更新する。

    • 5. 必要に応じて WAL ログを記録し、メタページを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックス AM インターフェース（IndexAmRoutine 経由）。
  • **呼び出し先**:
  • _hash_getcachedmetap: メタページ情報の取得。
  • hashbucketcleanup: バケット単位の削除処理。
  • _hash_getbuf / _hash_relbuf / _hash_dropbuf: バッファ管理。
  • LockBuffer / LockBufferForCleanup: ロック制御。
  • XLogInsert 等: WAL 記録。

関連データ構造

  • IndexBulkDeleteResult: 削除結果の統計情報。
  • HashMetaPage: ハッシュインデックスのメタページ構造。
  • HashPageOpaque: バケットページのメタデータ。

失敗モードと境界条件

  • 走査中にバケットが分割された場合、メタページ情報を再取得して処理を継続する（loop_top へのジャンプ）。
  • 統計情報の更新時、走査中にインデックスの変更があった場合は正確なカウントが困難なため、削除数に基づく推計値で補正する。

保守担当者向けメモ

  • hashbucketcleanup でのロック連鎖は、並行スキャンとの整合性を保つために重要である。
  • 本関数は「損失あり（Lossy）」インデックスの特性を考慮し、統計情報の更新において慎重な判断を行っている。
