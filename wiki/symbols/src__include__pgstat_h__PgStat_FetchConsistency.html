PgStat_FetchConsistency

定義

  • 種別: enum
  • 定義箇所: src/include/pgstat.h:45

目的

統計情報の取得時における一貫性のレベルを定義し、統計データへのアクセス制御や整合性保証のモードを指定するために使用されます。

シグネチャまたは形状

typedef enum PgStat_FetchConsistency
{
    PGSTAT_FETCH_CONSISTENCY_NONE,
    PGSTAT_FETCH_CONSISTENCY_CACHE,
    PGSTAT_FETCH_CONSISTENCY_SNAPSHOT,
} PgStat_FetchConsistency;

入力と出力

  • 入力: 統計取得関数等の引数として、取得モードを指定するために渡されます。
  • 出力: 該当なし（列挙型定数）。

前提条件と不変条件

  • 統計システムが初期化されている必要があります。
  • 各定数は、統計取得時の整合性要件（キャッシュ利用の有無やスナップショットの適用）に対応しています。

アルゴリズムまたは振る舞い

  • PGSTAT_FETCH_CONSISTENCY_NONE: 一貫性を特に保証しない、またはデフォルトの取得モード。
  • PGSTAT_FETCH_CONSISTENCY_CACHE: キャッシュされた統計情報を利用して取得を行うモード。
  • PGSTAT_FETCH_CONSISTENCY_SNAPSHOT: スナップショットに基づいて一貫性のある統計情報を取得するモード。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/utils/pgstat_internal.h:512 にて mode フィールドの型として参照されていることが確認されています。
  • 呼び出し先: 列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • PgStat_BackendSubEntry（pgstat_internal.h 内で本型を使用）

失敗モードと境界条件

  • 統計システムが未初期化の状態で本モードを指定して取得を試みた場合の挙動については、本ソース excerpt からは不明です。

保守担当者向けメモ

  • 本列挙型は統計取得の整合性ポリシーを決定する重要なインターフェースです。新しい取得モードを追加する場合は、pgstat_internal.h 側の処理ロジックも併せて更新する必要があります。
