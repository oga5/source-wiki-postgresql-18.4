statement

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:64

目的

ECPG（Embedded SQL for C）において、SQL実行のライフサイクル全体を管理するための中心的なデータ構造です。プリプロセッサが生成したSQL文の実行環境、パラメータ、および結果を保持します。

シグネチャまたは形状

struct statement {
    int lineno;
    char *command;
    char *name;
    struct connection *connection;
    enum COMPAT_MODE compat;
    bool force_indicator;
    enum ECPG_statement_type statement_type;
    bool questionmarks;
    struct variable *inlist;
    struct variable *outlist;
    /* ロケール情報 (プラットフォーム依存) */
    int nparams;
    char **paramvalues;
    int *paramlengths;
    int *paramformats;
    PGresult *results;
};

入力と出力

  • **入力**: ECPGプリプロセッサによって生成されたSQLコマンド文字列、ホスト変数リスト（inlist）、および接続情報。
  • **出力**: 実行結果として PGresult オブジェクトを保持し、必要に応じてホスト変数（outlist）へデータを転送します。

前提条件と不変条件

  • connection メンバは、有効なデータベース接続を指している必要があります。
  • ecpg_do_prologue によって初期化され、ecpg_do_epilogue によってメモリ解放されるライフサイクルを前提とします。

アルゴリズムまたは振る舞い

1. **初期化**: ecpg_do_prologue により構造体が生成され、実行環境が準備されます。 2. **実行**: ecpg_execute が libpq を介してクエリを送信し、結果を results メンバに格納します。 3. **変換**: ecpg_get_data 等の関数が、results からホスト変数への型変換を行います。 4. **終了**: ecpg_do_epilogue が呼び出され、関連するメモリのクリーンアップが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/execute.c 内の ecpg_do 関数が、この構造体を使用してSQL実行フローを制御します。
  • **呼び出し先**: libpq ライブラリ（PGresult を介した通信）およびメモリ管理関数（ecpg_alloc 等）。
  • **注記**: ツールが検出した contrib/ 配下の statement への言及は、本構造体とは無関係な一般的なSQL文を指しており、本シンボルの呼び出し元ではありません。

関連データ構造

  • struct connection: データベース接続状態を保持。
  • struct variable: ホスト変数とSQL変数のマッピング。
  • PGresult: libpq によるクエリ実行結果。

失敗モードと境界条件

  • メモリ不足時: ecpg_alloc 失敗による異常終了の可能性があります。
  • 接続エラー: connection が無効な場合、実行時にエラーが発生します。
  • 型変換エラー: inlist や outlist の型定義と実際のデータが不整合な場合、実行時エラーとなります。

保守担当者向けメモ

  • 本構造体は ecpglib 内部でのみ使用されます。
  • ロケール処理（oldlocale）はプラットフォーム（WIN32 等）によって定義が異なるため、移植性を考慮する際は注意してください。
  • メモリ管理には標準の malloc ではなく、ecpglib 規定のメモリ管理関数を使用してください。
