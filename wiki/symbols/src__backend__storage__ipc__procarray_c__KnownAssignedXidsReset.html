KnownAssignedXidsReset

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procarray.c:5255

目的

共有メモリ上の KnownAssignedXids（ホットスタンバイ時に追跡される実行中トランザクションのリスト）を初期化し、空の状態にリセットする。

シグネチャまたは形状

static void KnownAssignedXidsReset(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の ProcArrayStruct を更新）

前提条件と不変条件

  • ProcArrayLock を排他モードで取得する必要がある。
  • 本関数実行後、numKnownAssignedXids、tailKnownAssignedXids、headKnownAssignedXids はすべて 0 に設定される。

アルゴリズムまたは振る舞い

1. ProcArrayLock を LW_EXCLUSIVE モードで取得する。 2. 共有メモリ上の ProcArrayStruct を参照し、numKnownAssignedXids（保持数）、tailKnownAssignedXids（末尾インデックス）、headKnownAssignedXids（先頭インデックス）をすべて 0 にリセットする。 3. ProcArrayLock を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/procarray.c 内の ProcArrayInitRecovery（1115行目）から呼び出される。
  • 呼び出し先: LWLockAcquire および LWLockRelease（storage/lwlock.h）。

関連データ構造

  • ProcArrayStruct: 共有メモリ上でプロセス配列と KnownAssignedXids を管理する構造体。
  • ProcArrayLock: 本関数が操作する共有メモリ領域を保護するロック。

失敗モードと境界条件

  • ProcArrayLock が取得できない場合、デッドロックまたは待機が発生する可能性がある。
  • 本関数はリセット処理のみを行うため、呼び出し時点で他のプロセスが KnownAssignedXids を参照している場合、一時的に不整合な状態が見える可能性がある（ロックにより保護されているため、通常は安全）。

保守担当者向けメモ

  • 本関数はホットスタンバイの初期化プロセスの一部として使用される。
  • KnownAssignedXids はスタンバイ側でのトランザクション可視性判定に不可欠であるため、リセットのタイミングには注意が必要である。
