ssl_client_dn_field

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:237

目的

現在のSSLセッションにおけるクライアント証明書の識別名（DN）から、指定されたフィールド（例: CN, O, OUなど）の値を抽出してPostgreSQLの text 型として返します。

シグネチャまたは形状

Datum ssl_client_dn_field(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、抽出対象のフィールド名（text 型）を受け取ります。
  • **出力**: 成功時は抽出されたフィールド値を Datum（text 型）として返します。SSLが使用されていない、証明書が無効、または指定フィールドが見つからない場合は NULL を返します。

前提条件と不変条件

  • MyProcPort が有効であり、かつ ssl_in_use が真であること。
  • peer_cert_valid が真であること（クライアント証明書が検証済みであること）。

アルゴリズムまたは振る舞い

1. MyProcPort を参照し、SSL接続が確立されていない、または証明書が無効な場合は直ちに NULL を返します。 2. X509_get_subject_name を使用して、現在のセッションのピア証明書からサブジェクト名を取得します。 3. X509_NAME_field_to_text を呼び出し、指定されたフィールド名を検索・抽出します。 4. 抽出結果が NULL であれば NULL を返し、値が存在すればその Datum を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSQLインターフェースから呼び出されます（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: X509_get_subject_name（OpenSSL API）、X509_NAME_field_to_text（内部関数）。

関連データ構造

  • MyProcPort: 現在のセッションのSSL状態を保持するグローバル変数。
  • X509: OpenSSLの証明書構造体。

失敗モードと境界条件

  • **SSL未接続/証明書無効**: PG_RETURN_NULL() により安全に終了します。
  • **フィールド不在**: X509_NAME_field_to_text が NULL を返す場合、本関数も NULL を返します。
  • **エンコーディング**: X509_NAME_field_to_text 内部でデータベースエンコーディングへの変換が行われます。

保守担当者向けメモ

  • 本関数は MyProcPort に強く依存しています。
  • OpenSSLのAPI呼び出し結果が NULL の場合の挙動は X509_NAME_field_to_text の実装に依存するため、変更時は当該関数の仕様を確認してください。
