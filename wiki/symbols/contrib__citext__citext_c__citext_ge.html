citext_ge

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:299

目的

citext 型の2つの値が「以上（Greater than or equal to）」であるかを判定するSQL関数です。

シグネチャまたは形状

Datum citext_ge(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの text 型（citext として扱われる）引数を受け取ります。
  • **出力**: bool 型の Datum を返します。左辺が右辺以上であれば true、そうでなければ false です。

前提条件と不変条件

  • 入力引数は PG_GETARG_TEXT_PP を使用して取得されるため、TOAST されたデータは必要に応じて展開されます。
  • 比較には PG_GET_COLLATION() で取得される現在の照合順序が使用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を用いて2つの入力引数を取得します。 2. citextcmp 関数を呼び出し、照合順序を考慮した比較を行います。 3. citextcmp の戻り値が 0 以上であるかを確認し、その結果を result に格納します。 4. PG_FREE_IF_COPY を呼び出し、必要に応じて入力引数のメモリを解放します。 5. PG_RETURN_BOOL を使用して判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジンから演算子として呼び出されます。ソースコード内では PG_FUNCTION_INFO_V1(citext_ge) による登録が確認できます。
  • **呼び出し先**: citextcmp を呼び出します。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • 入力値が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で NULL 判定が行われることが期待されます（本関数内での明示的な NULL チェックは省略されています）。
  • 照合順序の設定によっては、予期しない比較結果となる可能性があります。

保守担当者向けメモ

  • 本関数は citext 型の >= 演算子を実装しています。比較ロジックを変更する場合は citextcmp を参照してください。
  • メモリリークを防ぐため、PG_GETARG_TEXT_PP で取得したポインタは必ず PG_FREE_IF_COPY で解放する必要があります。
