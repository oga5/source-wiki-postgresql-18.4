r_prelude

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_french.c:439

目的

フランス語のステミング処理における前処理として、単語内の特殊な文字や特定の文字列パターンを正規化する。

シグネチャまたは形状

static int r_prelude(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体（ステミング対象の単語データを含む）。
  • 出力: 成功時に 1、メモリ不足等のエラー発生時に負の整数を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • g_v（母音グループ）が定義されている環境で実行される。

アルゴリズムまたは振る舞い

単語全体を走査し、以下のパターンを置換する： 1. 母音に挟まれた 'u' を 'U' に、'i' を 'I' に、'y' を 'Y' に置換する。 2. 特殊文字 0xEB (ë) を "e" に、0xEF (ï) を "i" に置換する。 3. 'y' が母音に続く場合に 'Y' に置換する。 4. "qu" を "qU" に置換する。 これらの処理は slice_from_s を用いて単語バッファを直接書き換えることで行われる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_ISO_8859_1_french.c 内のメイン処理フローから参照されている。
  • 呼び出し先: in_grouping（母音判定）、slice_from_s（文字列置換）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • g_v: 母音を定義するグループ。
  • s_1 ～ s_7: 置換対象の文字列定数。

失敗モードと境界条件

  • slice_from_s が失敗した場合、その戻り値をそのまま返す。
  • 単語の末尾に達した場合は走査を終了する。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接編集せず元の .sbl ファイルを修正すること。
  • 0xEB や 0xEF などの文字コードは ISO-8859-1 エンコーディングに基づいている。
