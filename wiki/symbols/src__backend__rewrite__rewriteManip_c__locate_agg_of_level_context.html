locate_agg_of_level_context

定義

  • 種別: struct
  • 定義箇所: src/backend/rewrite/rewriteManip.c:33

目的

クエリツリーの走査中に、特定のネストレベルに存在する集約関数（Aggref）の位置を特定するためのコンテキスト情報を保持します。

シグネチャまたは形状

typedef struct
{
	int			agg_location;
	int			sublevels_up;
} locate_agg_of_level_context;

入力と出力

  • agg_location: 走査中に発見された集約関数の位置情報（出力用）。
  • sublevels_up: 探索対象とするクエリのネストレベル（入力用）。

前提条件と不変条件

  • sublevels_up は、現在のクエリツリーの深さに応じて適切に管理される必要があります。
  • 走査アルゴリズム（Walker）は、サブクエリに潜る際にこの値をインクリメントし、戻る際にデクリメントする不変条件に従うことが期待されます。

アルゴリズムまたは振る舞い

  • 本構造体は、expression_tree_walker 等を用いた再帰的なツリー走査において、状態保持用コンテキストとして使用されます。
  • 走査中に Aggref ノードに遭遇した場合、そのノードのレベルが sublevels_up と一致するかを確認し、一致すれば agg_location に情報を記録します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/rewrite/rewriteManip.c 内の関数（locate_agg_of_level_context 型の変数を宣言・初期化している箇所）。
  • 呼び出し先: 具体的な関数呼び出し関係は提供されたソースからは特定できません。

関連データ構造

  • Aggref: 集約関数を表すノード。
  • Query: クエリツリーのルートノード。

失敗モードと境界条件

  • 探索対象のレベルが深すぎる場合や、ツリー内に該当する集約関数が存在しない場合、agg_location は初期値のままとなります。

保守担当者向けメモ

  • 本構造体は rewriteManip.c 内の特定の走査ロジックに密結合しています。走査ロジックを変更する際は、sublevels_up の管理ルールが維持されているか確認してください。
