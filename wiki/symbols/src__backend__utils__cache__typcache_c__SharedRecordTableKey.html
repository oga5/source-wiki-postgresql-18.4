SharedRecordTableKey

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/typcache.c:197

目的

レコード型（record）のキャッシュにおいて、共有メモリまたはローカルメモリ上のタプル記述子（TupleDesc）を一意に識別するためのキーとして使用されます。

シグネチャまたは形状

typedef struct SharedRecordTableKey
{
    union
    {
        TupleDesc    local_tupdesc;
        dsa_pointer  shared_tupdesc;
    }            u;
    bool         shared;
} SharedRecordTableKey;

入力と出力

本構造体は関数ではなくデータ構造です。ハッシュテーブルの検索キーとして利用される際、shared フラグの値に応じて u.local_tupdesc または u.shared_tupdesc が参照されます。

前提条件と不変条件

  • shared が true の場合、u.shared_tupdesc が有効な dsa_pointer である必要があります。
  • shared が false の場合、u.local_tupdesc が有効な TupleDesc を指している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、ハッシュテーブルのキーとして比較およびハッシュ化されます。
  • typcache.c 内で定義された比較関数およびハッシュ関数により、shared フラグと対応するポインタ値（local_tupdesc または dsa_pointer）が等しいかどうかが判定されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/typcache.c 内のレコード型キャッシュ検索処理（lookup_type_cache 等に関連する箇所）で、ローカル変数としてインスタンス化されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。比較関数やハッシュ関数が本構造体を引数として受け取ります。

関連データ構造

  • TupleDesc: ローカルメモリ上のタプル記述子。
  • dsa_pointer: 共有メモリ上のタプル記述子を指すポインタ。
  • SharedRecordTableEntry: 本キーに対応するハッシュテーブルのエントリ。

失敗モードと境界条件

  • shared フラグと共用体内のポインタの不整合が発生した場合、ハッシュテーブルの検索失敗やメモリ不正アクセスのリスクがあります。

保守担当者向けメモ

  • 本構造体は dshash（共有ハッシュテーブル）とローカルキャッシュの両方で利用される設計です。共用体 u の切り替えロジックを変更する際は、比較関数およびハッシュ関数の実装も併せて確認してください。
