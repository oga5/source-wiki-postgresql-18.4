目的

ECPG（Embedded SQL for C）プリプロセッサの回帰テスト用プログラム。C言語の複雑な初期化式と、EXEC SQL WHENEVER 文によるエラーハンドリングの動作検証を行う。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム内部で定義された変数およびスタブ関数を使用）。
  • 出力: 標準出力へ計算結果および変数の値を表示。エラー発生時には WHENEVER で指定されたスタブ関数が呼び出される。

前提条件と不変条件

  • sqlca 構造体が正しく初期化されていること。
  • ECPGdo() がデータベース接続を確立し、SQL 文を実行可能な状態であること。

アルゴリズムまたは振る舞い

1. **変数宣言と初期化**: EXEC SQL BEGIN DECLARE SECTION 内で、算術演算、構造体メンバ参照、三項演算子等を含む変数の初期化を行う。 2. **デバッグ設定**: ECPGdebug(1, stderr) を呼び出し、デバッグログを有効化する。 3. **結果表示**: 初期化した変数の値を printf で出力する。 4. **SQL 実行とエラーハンドリング**: select now() を複数回実行し、各呼び出しの直後に sqlca.sqlcode をチェックする。エラー（sqlcode < 0）が発生した場合、WHENEVER 文で登録された関数（fa, fb, fc, fd, fe, sqlnotice）をそれぞれ実行する。 5. **終了**: return 0 で正常終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）。
  • 呼び出し先: ECPGdebug, printf, ECPGdo, fa, fb, fc, fd, fe, sqlnotice。

関連データ構造

  • sqlca_t: SQL 実行結果やエラー情報を保持する構造体。
  • struct sa: テスト用の構造体。

失敗モードと境界条件

  • ECPGdo 実行時に SQL エラーが発生した場合、sqlca.sqlcode < 0 の条件が真となり、対応するスタブ関数が実行される。
  • _cplusplus マクロが定義されている場合、C++ 互換の宣言セクションが処理される。

保守担当者向けメモ

  • 本関数は自動生成されたテストコードの期待値であるため、直接編集しないこと。
  • WHENEVER 文の展開ロジックを確認する際は、ECPGdo 直後の if (sqlca.sqlcode < 0) ブロックを参照すること。
