IsThereCollationInNamespace

定義

  • 種別: function
  • 定義箇所: src/backend/commands/collationcmds.c:395

目的

指定された名前空間（スキーマ）内に、特定の照合順序（Collation）が既に存在するかを確認し、存在する場合はエラーを発生させる。

シグネチャまたは形状

void IsThereCollationInNamespace(const char *collname, Oid nspOid)

入力と出力

  • collname (const char *): 確認対象の照合順序名。
  • nspOid (Oid): 確認対象の名前空間のOID。
  • 出力: なし。正常時は何も返さず終了する。

前提条件と不変条件

  • COLLNAMEENCNSP システムキャッシュが利用可能であること。
  • データベースのエンコーディング情報が GetDatabaseEncoding() を通じて取得可能であること。

アルゴリズムまたは振る舞い

1. 現在のデータベースエンコーディングと指定された名前空間において、collname が既に存在するかを SearchSysCacheExists3 で検索する。存在すれば ERRCODE_DUPLICATE_OBJECT でエラーを投げる。 2. 続いて、エンコーディングに依存しない（エンコーディング値が -1 の）エントリとして collname が存在するかを検索する。存在すれば同様にエラーを投げる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/alter.c:277
  • src/backend/commands/alter.c:781
  • 呼び出し先:
  • SearchSysCacheExists3 (システムキャッシュ検索)
  • GetDatabaseEncoding / GetDatabaseEncodingName

  • ereport (エラー報告)

関連データ構造

  • pg_collation カタログ（COLLNAMEENCNSP インデックス経由で参照）

失敗モードと境界条件

  • 指定された名前空間内に同名の照合順序が既に存在する場合、ERRCODE_DUPLICATE_OBJECT を伴うエラーで処理が中断される。
  • エンコーディング固有の照合順序と、エンコーディング非依存の照合順序の両方が重複チェックの対象となる。

保守担当者向けメモ

  • 本関数は照合順序の作成や名称変更時の重複チェックとして機能する。
  • COLLNAMEENCNSP インデックスの定義変更時には、本関数の検索ロジックも整合性を確認する必要がある。
