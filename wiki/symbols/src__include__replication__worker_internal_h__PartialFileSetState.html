PartialFileSetState

定義

  • 種別: enum
  • 定義箇所: src/include/replication/worker_internal.h:126

目的

論理レプリケーションにおける並列適用ワーカーのストリーミングトランザクション処理において、FileSet を用いたシリアライズ状態を管理するために使用されます。

シグネチャまたは形状

typedef enum PartialFileSetState
{
    FS_EMPTY,
    FS_SERIALIZE_IN_PROGRESS,
    FS_SERIALIZE_DONE,
    FS_READY,
} PartialFileSetState;

入力と出力

  • 本シンボルは列挙型であり、状態遷移の値を保持します。
  • src/backend/replication/logical/applyparallelworker.c 内の関数（pa_get_fileset_state 等）において、現在の状態を判定または取得するための戻り値として利用されます。

前提条件と不変条件

  • リーダーワーカーと並列ワーカー間で共有される ParallelApplyWorkerShared 構造体の一部として管理されます。
  • 状態遷移は、トランザクションのコミット順序の整合性を維持するために、リーダーと並列ワーカー間で適切に同期される必要があります。

アルゴリズムまたは振る舞い

  • FS_EMPTY: 初期状態。
  • FS_SERIALIZE_IN_PROGRESS: トランザクションデータのシリアライズ処理が実行中。
  • FS_SERIALIZE_DONE: シリアライズ処理が完了。
  • FS_READY: 読み取り可能な状態。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/applyparallelworker.c 内の各処理関数。
  • **呼び出し先**: 本シンボル自体は列挙型であり、直接の呼び出し先はありません。

関連データ構造

  • ParallelApplyWorkerShared: 本状態を保持する共有メモリ構造体。
  • FileSet: ストリーミングトランザクションの永続化に使用されるストレージ構造。

失敗モードと境界条件

  • 状態遷移の不整合は、並列適用ワーカー間でのコミット順序の崩壊を招く可能性があるため、同期制御には注意が必要です。

保守担当者向けメモ

  • ParallelApplyWorkerShared 内の fileset_state 遷移は、リーダーと並列ワーカー間の複雑な同期を制御します。変更時はコミット順序の整合性に細心の注意を払ってください。
