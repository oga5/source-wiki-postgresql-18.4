PageGetExactFreeSpace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/bufpage.c:956

目的

PostgreSQLのページ内における、現在の正確な空き領域サイズ（バイト単位）を算出します。

シグネチャまたは形状

Size PageGetExactFreeSpace(const PageData *page)

入力と出力

  • **入力**: page (対象となるページへのポインタ)
  • **出力**: Size 型の空き領域サイズ。

前提条件と不変条件

  • 入力された page は PageHeaderData として解釈可能である必要があります。
  • ページヘッダー内の pd_lower（アイテムID配列の末尾）と pd_upper（タプルデータの先頭）の値に基づきます。

アルゴリズムまたは振る舞い

1. ページヘッダーから pd_upper と pd_lower を取得します。 2. 符号付き整数として pd_upper - pd_lower を計算します。 3. 計算結果が負（pd_lower > pd_upper の状態）であれば、空き領域は存在しないものとして 0 を返します。 4. 正であれば、その差分を Size 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgstattuple（統計情報取得用）
  • src/backend/access/brin/brin_pageops.c
  • src/backend/access/gin/ginfast.c
  • src/backend/access/nbtree/nbtdedup.c
  • src/backend/access/nbtree/nbtsplitloc.c
  • **呼び出し先**: 外部関数呼び出しなし。

関連データ構造

  • PageHeaderData: ページヘッダー構造体。pd_lower および pd_upper フィールドを参照します。

失敗モードと境界条件

  • pd_lower > pd_upper の場合、計算結果が負になりますが、関数内で 0 に補正されるため、呼び出し側で負のサイズを扱うリスクはありません。

保守担当者向けメモ

  • 本関数は、ページ内の断片化を考慮した「物理的な」空き領域を返します。PageRepairFragmentation 等でデフラグが行われていない場合、この値は必ずしも「連続した」空き領域の最大サイズとは一致しない点に注意してください。
