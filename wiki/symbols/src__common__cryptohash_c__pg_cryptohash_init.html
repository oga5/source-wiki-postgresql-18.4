pg_cryptohash_init

定義

  • src/common/cryptohash.c:99

目的

指定された暗号学的ハッシュコンテキストを初期化し、選択されたアルゴリズムの計算準備を整える。

シグネチャまたは形状

int pg_cryptohash_init(pg_cryptohash_ctx *ctx);

入力と出力

  • **入力**: ctx (初期化対象の pg_cryptohash_ctx 構造体へのポインタ)
  • **出力**: 成功時は 0、ctx が NULL の場合は -1 を返す。

前提条件と不変条件

  • ctx は事前に pg_cryptohash_create 等で割り当てられ、有効なハッシュ型（PG_MD5, PG_SHA1, PG_SHA224, PG_SHA256, PG_SHA384, PG_SHA512 のいずれか）が設定されている必要がある。

アルゴリズムまたは振る舞い

1. ctx が NULL かどうかを検証し、該当する場合は即座に -1 を返す。 2. ctx->type を参照し、設定されたアルゴリズムに対応する初期化関数（pg_md5_init, pg_sha1_init, pg_sha224_init, pg_sha256_init, pg_sha384_init, pg_sha512_init のいずれか）を呼び出す。 3. 内部状態をリセットし、計算開始可能な状態にして 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/libpq/auth-scram.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/checksum_helper.c
  • **呼び出し先**:
  • pg_md5_init, pg_sha1_init, pg_sha224_init, pg_sha256_init, pg_sha384_init, pg_sha512_init（各アルゴリズム固有の初期化関数）

関連データ構造

  • pg_cryptohash_ctx: ハッシュアルゴリズムの種類と計算状態を保持する不透明な構造体。

失敗モードと境界条件

  • ctx が NULL の場合、関数は何もせず -1 を返す。
  • ctx->type がサポート対象外の値である場合、本関数は何も実行せず 0 を返す（デフォルトの switch 動作）。

保守担当者向けメモ

  • 本関数は OpenSSL が利用できない環境でのフォールバック実装として機能する。
  • 新しいハッシュアルゴリズムを追加する場合は、switch 文に新しい case を追加し、対応する _init 関数を呼び出す必要がある。
