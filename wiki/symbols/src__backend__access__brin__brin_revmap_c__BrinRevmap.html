BrinRevmap

定義

  • 種別: struct
  • 定義箇所: src/backend/access/brin/brin_revmap.c:46

目的

BRIN（Block Range Index）インデックスにおいて、ヒープのページ範囲と、その範囲を要約したインデックス内のタプル（BrinTuple）を対応付けるための翻訳テーブル（リバースレンジマップ）へのアクセス状態を保持・キャッシュする構造体です。

シグネチャまたは形状

struct BrinRevmap
{
    Relation    rm_irel;            /* インデックスのリレーション */
    BlockNumber rm_pagesPerRange;   /* 1つの要約タプルがカバーするブロック数 */
    BlockNumber rm_lastRevmapPage;  /* メタページからキャッシュされた最終revmapページ番号 */
    Buffer      rm_metaBuf;         /* メタページのバッファ */
    Buffer      rm_currBuf;         /* 現在アクセス中のrevmapページのバッファ */
};

入力と出力

本構造体自体はデータ保持用であり、関数呼び出しの引数として渡されることで、特定のインデックス操作における状態管理を行います。

前提条件と不変条件

  • 本構造体はスレッドセーフではなく、単一のバックエンド内での効率化を目的としています。
  • rm_lastRevmapPage はメタページの内容と整合している必要があります。
  • revmapは常にヒープブロックの範囲と要約タプルを正しく指し示す必要があり、不整合はインデックス破損とみなされます。

アルゴリズムまたは振る舞い

  • **キャッシュ管理**: メタページおよび現在処理中のrevmapページをバッファとして保持し、インデックス検索や挿入時の高速なアクセスを支援します。
  • **ライフサイクル**: brinRevmapInitialize で初期化され、brinRevmapTerminate で終了処理が行われます。
  • **拡張**: インデックスの成長に伴い、メタページを更新しつつ物理的なページ拡張を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c（インデックス操作の各所）、src/backend/access/brin/brin_pageops.c（ページ退避や更新処理）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、revmap_get_blkno などの内部関数が本構造体のフィールドを参照して処理を行います。

関連データ構造

  • Relation: インデックスのリレーション情報。
  • Buffer: PostgreSQLのバッファ管理構造。
  • BrinTuple: revmapによって指し示される要約タプル。

失敗モードと境界条件

  • **競合**: revmapの拡張時にメタページを排他ロックするため、他のインデックス操作と競合する可能性があります。
  • **破損**: ヒープブロックと要約タプルの対応関係に不整合がある場合、エラーを報告します。

保守担当者向けメモ

  • 本構造体は単一バックエンド内でのキャッシュ用であり、共有メモリ上には存在しません。
  • 拡張時には brin_evacuate_page を介して既存ページとの競合を回避するロジックが組み込まれています。
