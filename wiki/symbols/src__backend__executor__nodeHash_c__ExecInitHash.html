ExecInitHash

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:369

目的

ハッシュ結合のビルドフェーズで使用される Hash ノードの実行状態（HashState）を初期化し、子ノードの準備を行う。

シグネチャまたは形状

HashState *ExecInitHash(Hash *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Hash プランノード。
  • estate: 実行状態を保持する EState オブジェクト。
  • eflags: 実行フラグ（EXEC_FLAG_BACKWARD および EXEC_FLAG_MARK はサポート外）。
  • **出力**:
  • 初期化済みの HashState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと。
  • node->plan.qual が NIL であること。
  • ハッシュテーブルの構築（ExecHashTableCreate）は、本関数内では行われず、実行時の後続処理へ遅延される。
  • hash_expr の初期化は、結合タイプが確定する ExecInitHashJoin まで遅延される。

アルゴリズムまたは振る舞い

1. HashState 構造体を割り当て、プランノードと実行状態を関連付ける。 2. ps.ExecProcNode に ExecHash を設定する。 3. ExecAssignExprContext を呼び出し、式コンテキストを作成する。 4. ExecInitNode を呼び出し、子ノード（outerPlan）を再帰的に初期化する。 5. ExecInitResultTupleSlotTL を使用して、結果タプルスロットを TTSOpsMinimalTuple で初期化する。 6. hashtable および hash_expr を NULL に初期化し、後続の実行フェーズでの設定に備える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execProcnode.c:366
  • **呼び出し先**:
  • makeNode (マクロ)
  • ExecAssignExprContext
  • ExecInitNode
  • ExecInitResultTupleSlotTL

関連データ構造

  • HashState: 本関数が生成する主要な状態構造体。
  • Hash: 入力となるプランノード。
  • EState: 実行状態。
  • TTSOpsMinimalTuple: 結果スロットのタプル操作関数群。

失敗モードと境界条件

  • Assert により、サポート外のフラグ（EXEC_FLAG_BACKWARD, EXEC_FLAG_MARK）や qual の存在が検出された場合にアボートする。

保守担当者向けメモ

  • 本関数はハッシュテーブルの作成を明示的に行わない「遅延初期化」設計となっている。
  • hash_expr の構築には結合タイプ（join type）の情報が必要であり、それが確定する ExecInitHashJoin 以前に本関数が呼ばれるため、この設計が維持されている。
