SPNodeData

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/dicts/spell.h:27

目的

ISpell/Hunspell 辞書における接頭辞・接尾辞木（Trie）の各ノードのデータを保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	uint32		val:8,
				isword:1,
				compoundflag:4,
				affix:19;
	struct SPNode *node;
} SPNodeData;

入力と出力

本構造体はデータ保持用であり、関数ではないため入力・出力の概念は適用されません。メモリ上のデータ表現として機能します。

前提条件と不変条件

  • SPNode 構造体と組み合わせて使用され、Trie 構造を構成します。
  • SPNode 構造体内の data フィールド（FLEXIBLE_ARRAY_MEMBER）として配置されることが想定されています。
  • サイズ計算には SPNHDRSZ マクロを使用する必要があります。

アルゴリズムまたは振る舞い

  • val (8bit): ノードが表す文字コードを保持します。
  • isword (1bit): このノードが単語の終端であるかを示します。
  • compoundflag (4bit): 複合語処理のためのフラグを保持します。
  • affix (19bit): AffixData フィールドへの参照インデックスとして機能します。
  • node: 子ノードへのポインタを保持します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tsearch/spell.c 内の辞書構築および検索ロジック（1657, 1670 行付近など）で、Trie の構築や走査時に参照・確保されます。
  • 呼び出し先: 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • SPNode: SPNodeData を要素として含む Trie ノード構造体。
  • IspellDict: 辞書全体の状態を管理する構造体。

失敗モードと境界条件

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、メモリ確保時に SPNHDRSZ を含めた適切なサイズ計算を行わない場合、バッファオーバーフローやメモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • SPNodeData はビットフィールドを使用してメモリ効率を最適化しています。フィールドのビット幅を変更する場合は、合計が uint32 (32bit) に収まるよう注意してください。
  • 辞書構築フェーズと実行フェーズでメモリレイアウトが異なる可能性があるため、spell.c 内の操作ロジックを確認する際は、現在の処理フェーズを考慮してください。
