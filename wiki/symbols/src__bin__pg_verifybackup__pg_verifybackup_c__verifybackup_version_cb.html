verifybackup_version_cb

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:523

目的

JSON形式のバックアップマニフェストからバージョン番号を抽出し、manifest_data 構造体に格納するコールバック関数です。

シグネチャまたは形状

static void verifybackup_version_cb(JsonManifestParseContext *context, int manifest_version);

入力と出力

  • **入力**:
  • context: マニフェスト解析のコンテキストを保持する JsonManifestParseContext 構造体へのポインタ。
  • manifest_version: 解析されたマニフェストのバージョン番号（整数）。
  • **出力**: なし（void）。

前提条件と不変条件

  • context->private_data は manifest_data 構造体として初期化されている必要があります。

アルゴリズムまたは振る舞い

1. context->private_data を通じて manifest_data 構造体への参照を取得します。 2. 引数として渡された manifest_version を manifest->version に代入します。 3. バージョンの検証処理は本関数内では行わず、後のステージで実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: マニフェスト解析処理（parse_manifest_file 内でコールバックとして登録・実行される）。
  • **呼び出し先**: なし。

関連データ構造

  • JsonManifestParseContext: マニフェスト解析の制御構造体。
  • manifest_data: バックアップマニフェストの情報を保持する構造体。

失敗モードと境界条件

  • context->private_data が適切に設定されていない場合、メモリ不正アクセスの可能性があります。

保守担当者向けメモ

  • 本関数は JsonManifestParseContext の version_cb フィールドに割り当てられるコールバックです。
  • バージョン番号の妥当性チェックが必要な場合は、本関数内ではなく、マニフェスト解析完了後の検証フェーズで行う設計となっています。
