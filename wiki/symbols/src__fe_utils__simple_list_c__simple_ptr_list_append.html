simple_ptr_list_append

定義

  • 種別: function
  • 定義箇所: src/fe_utils/simple_list.c:161

目的

汎用ポインタを保持する単方向連結リストの末尾に、新しい要素を追加します。

シグネチャまたは形状

void simple_ptr_list_append(SimplePtrList *list, void *ptr)

入力と出力

  • list: 操作対象の SimplePtrList 構造体へのポインタ。
  • ptr: リストに追加する汎用ポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • list は初期化済みである必要があります。
  • リストの末尾（tail）が正しく維持されている必要があります。

アルゴリズムまたは振る舞い

1. pg_malloc を使用して新しい SimplePtrListCell を確保します。 2. 新しいセルの next を NULL に、ptr を引数の ptr に設定します。 3. リストが空でない場合（list->tail が存在する場合）、現在の末尾セルの next を新しいセルに接続します。 4. リストが空の場合、list->head を新しいセルに設定します。 5. リストの tail を新しいセルに更新します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な証拠）:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_dump/common.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • 呼び出し先:
  • pg_malloc（メモリ確保）

関連データ構造

  • SimplePtrList: リストの head と tail を保持する構造体。
  • SimplePtrListCell: リストの各要素（next ポインタと保持する ptr を含む）。

失敗モードと境界条件

  • メモリ不足時: pg_malloc が失敗した場合、プログラムは終了します。
  • 空リストへの追加: head と tail の両方が正しく新しいセルを指すように処理されます。

保守担当者向けメモ

  • 本関数はポインタの参照のみを保持します。リスト破棄時に、保持されているポインタが指す先のメモリは解放されません。
  • 単方向リストであるため、要素追加は $O(1)$ ですが、検索は $O(N)$ となる点に留意してください。
