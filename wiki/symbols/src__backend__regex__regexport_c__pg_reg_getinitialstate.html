pg_reg_getinitialstate

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:49

目的

コンパイル済みの正規表現オブジェクトから、内部的な NFA（非決定性有限オートマトン）の初期状態番号を取得する。

シグネチャまたは形状

int pg_reg_getinitialstate(const regex_t *regex)

入力と出力

  • **入力**: regex_t *regex （初期化済みの正規表現オブジェクトへのポインタ）
  • **出力**: int （NFA の初期状態を表す整数値）

前提条件と不変条件

  • regex は NULL であってはならない。
  • regex->re_magic は REMAGIC と一致していなければならない（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. 引数として渡された regex オブジェクトの re_guts フィールドを struct guts * 型にキャストする。 2. 内部の search 構造体（cnfa 型）にアクセスする。 3. cnfa 構造体内に保持されている初期状態フィールド pre を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_trgm/trgm_regexp.c:925
  • contrib/pg_trgm/trgm_regexp.c:2167
  • **呼び出し先**:
  • 本関数は他の関数を呼び出さない。

関連データ構造

  • regex_t: 正規表現の公開用構造体。
  • struct guts: 正規表現エンジンの内部状態を保持する構造体。
  • struct cnfa: NFA の構造を定義する内部構造体。

失敗モードと境界条件

  • regex が無効な場合、アサーションによりプログラムが停止する可能性がある。
  • 正規表現が正しくコンパイルされていない場合、re_guts の参照先が不正である可能性がある。

保守担当者向けメモ

  • 本関数は regex/regguts.h で定義された内部構造体 cnfa に直接依存している。
  • pg_reg_getinitialstate は、正規表現の検索開始位置を特定するために使用されるアクセサである。
