XLogRegisterData

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xloginsert.c:363

目的

WALレコードの構築中に、レコード本体となる任意のデータチャンクを登録します。

シグネチャまたは形状

void XLogRegisterData(const void *data, uint32 len)

入力と出力

  • data: 登録するデータへのポインタ。
  • len: 登録するデータのバイト数。
  • 出力: なし（内部的なデータチェーン rdatas に情報を追加）。

前提条件と不変条件

  • XLogBeginInsert() が既に呼び出されていること（begininsert_called が真であること）。
  • 登録可能なデータセグメント数（max_rdatas）を超過していないこと。

アルゴリズムまたは振る舞い

1. begininsert_called をアサートで確認します。 2. 現在の登録済みデータ数（num_rdatas）が上限（max_rdatas）に達していないかチェックし、超過している場合はエラーを報告します。 3. rdatas 配列から次の空きスロットを取得し、引数で渡された data ポインタと len を格納します。 4. mainrdata_last が指す現在のチェーン末尾に、新しい rdata を連結します。 5. mainrdata_last を新しい末尾に更新します。 6. mainrdata_len に len を加算し、レコード全体のデータ長を更新します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pg_visibility/pg_visibility.c、src/backend/access/brin/ 配下の各ファイル、src/backend/access/gin/ 配下の各ファイルなど、リソースマネージャによる WAL 構築処理から呼び出されます。
  • 呼び出し先: 直接的な関数呼び出しはありませんが、グローバル変数 rdatas、num_rdatas、mainrdata_last、mainrdata_len を操作します。

関連データ構造

  • XLogRecData: WALレコードのデータ断片を保持する構造体。
  • rdatas: 登録された XLogRecData を保持する配列。

失敗モードと境界条件

  • num_rdatas >= max_rdatas: 登録可能なデータセグメント数を超過した場合、ERROR レベルで処理を中断します。

保守担当者向けメモ

  • 本関数は XLogBeginInsert で初期化された作業領域を使用します。
  • 登録されたデータは XLogRecordAssemble が呼び出されるまでメモリ上に保持されます。
  • mainrdata_last を介してチェーンを構築しているため、個別の next ポインタのクリアは不要です。
