macaddr8_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac8.c:286

目的

macaddr8 型（EUI-64）のデータを、ネットワーク通信用のバイナリ形式に変換して送信する。

シグネチャまたは形状

Datum macaddr8_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して macaddr8 型のポインタを受け取る。
  • **出力**: bytea 型のバイナリデータ（Datum）を返す。

前提条件と不変条件

  • 入力引数として有効な macaddr8 構造体へのポインタが提供されていること。
  • libpq/pqformat.h で定義された StringInfo インターフェースが利用可能であること。

アルゴリズムまたは振る舞い

1. pq_begintypsend を呼び出し、送信用の StringInfoData バッファを初期化する。 2. macaddr8 構造体のメンバ a から h までを、順次 pq_sendbyte を用いてバッファに書き込む。 3. pq_endtypsend を呼び出してバッファを bytea 形式に変換し、PG_RETURN_BYTEA_P を通じて結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（Fmgr）から呼び出される。
  • **呼び出し先**:
  • pq_begintypsend
  • pq_sendbyte
  • pq_endtypsend

関連データ構造

  • macaddr8: 8バイトのMACアドレスを保持する構造体。
  • StringInfoData: バイナリ送受信用のバッファ構造体。

失敗モードと境界条件

  • メモリ不足時など、pq_begintypsend や pq_sendbyte 内部でエラーが発生した場合、PostgreSQL のエラーハンドリング機構により処理が中断される。

保守担当者向けメモ

  • 本関数は macaddr8 構造体のメモリレイアウト（メンバ a〜h）に直接依存している。構造体の定義に変更を加える場合は、本関数の送信順序と整合性が取れているか確認すること。
