parent_offset

定義

  • 種別: function
  • 定義箇所: src/common/binaryheap.c:101

目的

バイナリヒープ（配列ベースの二分木）において、指定されたインデックス i にあるノードの親ノードのインデックスを算出する。

シグネチャまたは形状

static inline int parent_offset(int i)

入力と出力

  • **入力**: int i （対象ノードのインデックス）
  • **出力**: int （親ノードのインデックス）

前提条件と不変条件

  • 入力 i は 0 以上の整数であること。
  • ヒープのルート（インデックス 0）に対して呼び出した場合、(0 - 1) / 2 の計算結果として 0 が返される（整数除算の特性による）。

アルゴリズムまたは振る舞い

  • 以下の数式に基づき、親のインデックスを算出する。

    • return (i - 1) / 2;

  • 二分ヒープの構造上、インデックス i の左の子は 2*i + 1、右の子は 2*i + 2 に配置されるため、その逆変換を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/binaryheap.c: sift_up 処理およびヒープ構築時のループ内で使用。
  • src/backend/utils/sort/logtape.c: ヒープ操作に関連する箇所で参照。
  • **呼び出し先**: なし（算術演算のみを実行）。

関連データ構造

  • binaryheap 構造体（src/include/lib/binaryheap.h で定義）が管理するノード配列のインデックス計算に使用される。

失敗モードと境界条件

  • i = 0 の場合、計算結果は 0 となる。ルートノードには親が存在しないため、呼び出し側でルートノードに対する処理を適切に制御する必要がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、コンパイル時に展開されることを意図している。
  • src/backend/utils/sort/logtape.c では uint64 型の引数で呼び出されているため、型変換に伴うオーバーフローの可能性については呼び出し側の文脈を確認すること。
