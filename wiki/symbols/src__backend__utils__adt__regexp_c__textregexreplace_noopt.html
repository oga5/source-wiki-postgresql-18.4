textregexreplace_noopt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regexp.c:641

目的

SQLの regexp_replace 関数において、フラグ指定なし（デフォルト動作）の置換処理を実行します。

シグネチャまたは形状

Datum textregexreplace_noopt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の3つの text 型引数を受け取ります。
  • 第1引数: 対象文字列 (s)
  • 第2引数: 正規表現パターン (p)
  • 第3引数: 置換後の文字列 (r)
  • **出力**: 置換後の結果を保持する text 型の Datum を返します。

前提条件と不変条件

  • 入力引数は PG_GETARG_TEXT_PP を用いて取得されるため、TOAST展開済みのデータとして扱われます。
  • 内部で REG_ADVANCED フラグが固定的に使用されます。

アルゴリズムまたは振る舞い

1. 引数 s, p, r を取得します。 2. replace_text_regexp 関数を呼び出し、以下の引数を渡して置換を実行します。

  • REG_ADVANCED: POSIX拡張正規表現エンジンを指定。
  • PG_GET_COLLATION(): 現在の照合順序を使用。
  • cflags = 0: 追加のフラグなし。
  • n = 1: 置換回数の制限なし（デフォルトの挙動）。

    • 3. replace_text_regexp の戻り値を PG_RETURN_TEXT_P で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は特定されていません。
  • **呼び出し先**: replace_text_regexp を呼び出します。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に基づく引数コンテナ。

失敗モードと境界条件

  • replace_text_regexp 内部での正規表現コンパイルエラーやメモリ不足が発生した場合、PostgreSQLの標準的なエラーハンドリングによりアボートされます。

保守担当者向けメモ

  • 本関数は regexp_replace の特定の呼び出しパターン（フラグなし）を最適化・簡略化するためのラッパーです。
  • 動作を変更する場合は、呼び出し先の replace_text_regexp の仕様を確認してください。
