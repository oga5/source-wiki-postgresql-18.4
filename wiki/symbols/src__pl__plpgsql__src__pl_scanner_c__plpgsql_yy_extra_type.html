plpgsql_yy_extra_type

定義

  • 種別: struct
  • 定義箇所: src/pl/plpgsql/src/pl_scanner.c:103

目的

PL/pgSQL スキャナの状態を保持する構造体です。字句解析器がソースコードを解析する際のコンテキスト（ソース位置、トークン情報、プッシュバックスタックなど）を管理します。

シグネチャまたは形状

struct plpgsql_yy_extra_type {
    core_yy_extra_type core_yy_extra;
    const char *scanorig;
    int         plpgsql_yyleng;
    int         plpgsql_yytoken;
    int         num_pushbacks;
    int         pushback_token[MAX_PUSHBACKS];
    TokenAuxData pushback_auxdata[MAX_PUSHBACKS];
    const char *cur_line_start;
    const char *cur_line_end;
    int         cur_line_num;
};

入力と出力

  • **入力**: 字句解析の対象となるソース文字列（scanorig）を保持します。
  • **出力**: 構文解析器（Bison）が消費するトークンコード（plpgsql_yytoken）や、エラー報告用の行番号情報（cur_line_num）を提供します。

前提条件と不変条件

  • MAX_PUSHBACKS（4）までのトークンをスタックに保持可能です。
  • scanorig は解析対象のソースコード全体を指す必要があります。
  • cur_line_start および cur_line_end は、エラー位置報告のために現在の行の範囲を正確に指し示す必要があります。

アルゴリズムまたは振る舞い

  • **トークン管理**: plpgsql_yylex が取得した現在のトークン長とコードを保持します。
  • **先読み（プッシュバック）**: 構文解析の都合上、一度読み取ったトークンをスタックに戻す機能をサポートします。
  • **行番号追跡**: plpgsql_location_to_lineno() の呼び出し時に、保持している行の開始・終了位置情報を使用して現在の行番号を特定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_scanner.c 内の plpgsql_scanner_init（または類似の初期化処理）にて palloc0_object を通じてインスタンス化されます。
  • **呼び出し先**: plpgsql_yylex などの字句解析関数が、スキャナの状態を読み書きするためにこの構造体を参照します。

関連データ構造

  • core_yy_extra_type: PostgreSQL コアの字句解析器が必要とする状態。
  • TokenAuxData: プッシュバックされたトークンに関連付けられた補助データ。

失敗モードと境界条件

  • num_pushbacks が MAX_PUSHBACKS を超える場合、スタックオーバーフローが発生する可能性があります。
  • scanorig が無効なポインタである場合、エラー報告機能がクラッシュする可能性があります。

保守担当者向けメモ

  • 本構造体は plpgsql_yylex の実行状態を完全に保持するため、スキャナの再入可能性や状態遷移をデバッグする際の主要な参照先となります。
  • TokenAuxData の変更は、プッシュバックスタックの整合性に影響を与えるため注意してください。
