SpGistSpecialOffsetNumbers

定義

  • 種別: enum
  • 定義箇所: src/backend/access/spgist/spgscan.c:755

目的

SP-GiST インデックスの走査処理において、通常のページ内オフセット番号（OffsetNumber）の範囲外で特別な意味を持つ状態を識別するために使用されます。

シグネチャまたは形状

enum SpGistSpecialOffsetNumbers
{
    SpGistBreakOffsetNumber = InvalidOffsetNumber,
    SpGistRedirectOffsetNumber = MaxOffsetNumber + 1,
    SpGistErrorOffsetNumber = MaxOffsetNumber + 2,
};

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: SpGistBreakOffsetNumber, SpGistRedirectOffsetNumber, SpGistErrorOffsetNumber の各定数。

前提条件と不変条件

  • SpGistBreakOffsetNumber は InvalidOffsetNumber と等価です。
  • SpGistRedirectOffsetNumber および SpGistErrorOffsetNumber は、有効なページ内オフセットの最大値（MaxOffsetNumber）を超える値として定義されています。

アルゴリズムまたは振る舞い

  • SpGistBreakOffsetNumber: 走査の停止や中断を示すために使用されます。
  • SpGistRedirectOffsetNumber: SP-GiST インデックスにおけるタプルのリダイレクト状態を表現します。
  • SpGistErrorOffsetNumber: 走査中に発生したエラー状態を表現します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース抜粋からは特定できません。

関連データ構造

  • OffsetNumber（PostgreSQL の標準的なページ内オフセット型）

失敗モードと境界条件

  • 本シンボル自体は定数定義であるため、直接的な失敗モードはありません。ただし、これらを使用するロジックにおいて、MaxOffsetNumber を超える値が通常のオフセットとして誤って解釈されないよう注意が必要です。

保守担当者向けメモ

  • 本列挙型は SP-GiST のスキャンロジックにおける状態管理用です。新しい特殊状態を追加する場合は、既存の MaxOffsetNumber との衝突を避ける必要があります。
