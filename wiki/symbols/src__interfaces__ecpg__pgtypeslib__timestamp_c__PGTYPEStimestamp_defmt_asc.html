PGTYPEStimestamp_defmt_asc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/timestamp.c:805

目的

指定されたフォーマット文字列に従って、文字列形式のタイムスタンプを timestamp 型へ変換します。

シグネチャまたは形状

int PGTYPEStimestamp_defmt_asc(const char *str, const char *fmt, timestamp *d)

入力と出力

  • **入力**:
  • str: 解析対象のタイムスタンプ文字列。
  • fmt: 解析に使用するフォーマット文字列（NULL の場合はデフォルトの "%Y-%m-%d %H:%M:%S" が使用される）。
  • d: 変換結果を格納する timestamp 型へのポインタ。
  • **出力**:
  • 成功時は 0、失敗時は非ゼロの整数値を返します。

前提条件と不変条件

  • fmt が空文字列（""）の場合、関数は即座に 1 を返します。
  • 内部的に pgtypes_strdup を使用して入力文字列を複製するため、呼び出し元でメモリを解放する必要はありません。

アルゴリズムまたは振る舞い

1. fmt が NULL の場合、デフォルトのフォーマットを適用します。 2. str と fmt を複製します。 3. 年、月、日、時、分、秒、タイムゾーンの各フィールドを初期化します（年・月・日・分・秒は未指定を判定するため -1 に設定）。 4. PGTYPEStimestamp_defmt_scan を呼び出し、実際の解析処理を実行します。 5. 複製した文字列メモリを解放し、スキャン結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c
  • **呼び出し先**:
  • pgtypes_strdup
  • PGTYPEStimestamp_defmt_scan
  • free

関連データ構造

  • timestamp: int64（マイクロ秒単位）として定義されたデータ型。

失敗モードと境界条件

  • fmt が空文字列の場合、変換を行わず 1 を返します。
  • PGTYPEStimestamp_defmt_scan の内部処理で解析に失敗した場合、その戻り値がそのまま返されます。
  • タイムスタンプの計算結果が有効範囲外の場合、timestamp 型の仕様に基づきエラーが発生する可能性があります。

保守担当者向けメモ

  • year 等の初期値に -1 を使用してフィールドの指定有無を判定していますが、BC（紀元前）年との曖昧さについて注意が必要です。
  • PGTYPEStimestamp_defmt_scan が解析の主軸を担っているため、詳細なフォーマット仕様の変更はそちらを確認してください。
