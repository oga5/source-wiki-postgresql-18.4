tuplesort_begin_index_btree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:358

目的

B-treeインデックス構築のためのソート処理を初期化し、インデックスのキー属性に基づいた比較ロジックや入出力関数を Tuplesortstate に登録する。

シグネチャまたは形状

Tuplesortstate *tuplesort_begin_index_btree(Relation heapRel,
                                            Relation indexRel,
                                            bool enforceUnique,
                                            bool uniqueNullsNotDistinct,
                                            int workMem,
                                            SortCoordinate coordinate,
                                            int sortopt)

入力と出力

  • **入力**:
  • heapRel, indexRel: 対象となるヒープおよびインデックスのリレーション。
  • enforceUnique: 一意性制約を強制するかどうか。
  • uniqueNullsNotDistinct: NULL値を区別しない一意性制約かどうか。
  • workMem: ソートに使用可能なメモリ量。
  • coordinate: 並列ソート等の実行環境情報。
  • sortopt: ソートオプション（ランダムアクセス可否など）。
  • **出力**:
  • 初期化済みの Tuplesortstate 構造体へのポインタ。

前提条件と不変条件

  • base->maincontext が有効であること。
  • indexRel からインデックスのキー属性数が取得可能であること。
  • SortSupport データがインデックスの各キー属性に対して正しく準備されること。

アルゴリズムまたは振る舞い

1. tuplesort_begin_common を呼び出し、共通のソート状態を初期化する。 2. TuplesortIndexBTreeArg を割り当て、インデックス固有の引数を格納する。 3. comparetup_index_btree（比較関数）、writetup_index（書き込み）、readtup_index（読み込み）等の関数ポインタを Tuplesortstate に設定する。 4. _bt_mkscankey を用いてインデックスの走査キーを作成し、各属性に対して PrepareSortSupportFromIndexRel を実行して比較サポート情報を構築する。 5. 構築した走査キーを解放し、初期化済みの状態を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtsort.c (複数箇所)
  • **呼び出し先**:
  • tuplesort_begin_common
  • TuplesortstateGetPublic
  • IndexRelationGetNumberOfKeyAttributes
  • _bt_mkscankey
  • PrepareSortSupportFromIndexRel
  • MemoryContextSwitchTo

関連データ構造

  • Tuplesortstate: ソート処理の全状態を保持。
  • TuplesortIndexBTreeArg: B-treeインデックスソート固有の引数構造体。
  • SortSupportData: 各キー属性の比較を高速化するためのサポート構造体。
  • BTScanInsert: B-treeインデックスの走査キー情報。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性がある。
  • indexRel が無効な場合や、キー属性の取得に失敗した場合は予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は tuplesort.c のエンジンが利用するプラグインとして機能する。
  • インデックスのキー属性に対する比較ロジック（comparetup_index_btree）の変更は、本関数での SortSupport の設定と密接に関連しているため注意が必要。
