ProcessUtilitySlow

定義

  • **種別**: function
  • **定義箇所**: src/backend/tcop/utility.c:1091

目的

イベントトリガーのサポートが必要な複雑なユーティリティコマンド（DDL等）の実行を制御します。ProcessUtility から呼び出され、コマンドの実行前後でイベントトリガーを管理し、必要に応じてコマンドを再帰的にディスパッチします。

シグネチャまたは形状

static void ProcessUtilitySlow(
    ParseState *pstate,
    PlannedStmt *pstmt,
    const char *queryString,
    ProcessUtilityContext context,
    ParamListInfo params,
    QueryEnvironment *queryEnv,
    DestReceiver *dest,
    QueryCompletion *qc
)

入力と出力

  • **入力**: 実行対象の PlannedStmt、SQL文字列、実行コンテキスト、パラメータ情報、結果受信先など。
  • **出力**: なし（副作用としてデータベース状態を変更し、イベントトリガーを起動する）。

前提条件と不変条件

  • isCompleteQuery が真の場合、イベントトリガーのライフサイクル（EventTriggerBeginCompleteQuery / EndCompleteQuery）が保証される。
  • 実行中のコマンドがイベントトリガーの対象である場合、EventTriggerDDLCommandStart が呼び出される。

アルゴリズムまたは振る舞い

1. **イベントトリガー初期化**: EventTriggerBeginCompleteQuery を呼び出し、PG_TRY ブロックで囲むことで、異常終了時も確実に EventTriggerEndCompleteQuery が呼ばれるようにする。 2. **コマンドディスパッチ**: nodeTag(parsetree) に基づき、switch 文で適切なハンドラへ分岐する。

  • T_CreateSchemaStmt: CreateSchemaCommand を実行。
  • T_CreateStmt / T_CreateForeignTableStmt:
  • transformCreateStmt で解析し、生成された複数のステートメントを順次処理する。
  • DefineRelation 等でオブジェクトを作成し、EventTriggerCollectSimpleCommand でイベントトリガーに収集する。
  • TableLikeClause が含まれる場合は expandTableLikeClause で展開し、再帰的に処理する。
  • その他のステートメントは ProcessUtility を再帰呼び出しして処理する。

    • 3. **状態更新**: コマンド間や処理の節目で CommandCounterIncrement() を呼び出し、変更を可視化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c 内の ProcessUtility から呼び出される。
  • **呼び出し先**:
  • EventTrigger 関連関数（EventTriggerBeginCompleteQuery, EventTriggerDDLCommandStart 等）
  • 各種コマンドハンドラ（CreateSchemaCommand, DefineRelation, CreateForeignTable, transformCreateStmt 等）
  • ProcessUtility（再帰呼び出し）

関連データ構造

  • PlannedStmt: 実行計画およびユーティリティ文のツリーを保持。
  • ObjectAddress: 作成されたデータベースオブジェクトの識別子。
  • EventTrigger: DDL コマンドのライフサイクル管理に使用。

失敗モードと境界条件

  • PG_TRY ブロックにより、コマンド実行中にエラーが発生してもイベントトリガーの終了処理が保証される。
  • CreateStmt 処理において、stmts リストを走査中にリストが変更される可能性があるため、foreach ではなく linitial と list_delete_first を用いた安全な反復処理を行っている。

保守担当者向けメモ

  • 本関数はイベントトリガーをサポートするコマンド専用です。新しい DDL コマンドを追加し、イベントトリガーのフックが必要な場合は、この switch 文にケースを追加してください。
  • CreateStmt のような複雑なコマンドは、内部で複数のサブコマンドを生成するため、個別の EventTriggerCollectSimpleCommand 呼び出しと CommandCounterIncrement のタイミングに注意してください。
