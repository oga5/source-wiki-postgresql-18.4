heapam_tuple_insert

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_handler.c:243

目的

テーブルアクセス手法（Table AM）の抽象化レイヤーにおいて、ヒープストレージへのタプル挿入処理を仲介する。

シグネチャまたは形状

static void heapam_tuple_insert(Relation relation, TupleTableSlot *slot, CommandId cid, int options, BulkInsertState bistate)

入力と出力

  • **入力**:
  • relation: 挿入対象のテーブルリレーション。
  • slot: 挿入するタプルデータを保持する TupleTableSlot。
  • cid: 現在のコマンドID。
  • options: 挿入オプション（HEAP_INSERT_SKIP_WAL 等）。
  • bistate: 一括挿入用の状態管理構造体。
  • **出力**: なし（副作用として slot の tts_tid および tts_tableOid が更新される）。

前提条件と不変条件

  • slot から抽出された HeapTuple は、挿入後にメモリ解放が必要な場合がある。
  • 挿入成功後、slot->tts_tid には物理的なタプル位置（ItemPointer）が格納される。

アルゴリズムまたは振る舞い

1. ExecFetchSlotHeapTuple を呼び出し、スロットから HeapTuple を取得する。 2. リレーションの OID を取得し、スロットおよびタプル構造体の t_tableOid に設定する。 3. heap_insert を呼び出し、実際のヒープストレージへの挿入を実行する。 4. 挿入されたタプルの物理位置（t_self）を ItemPointerCopy によりスロットの tts_tid にコピーする。 5. 取得したタプルが一時的なものであれば pfree でメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: TableAmRoutine 構造体（heapam_methods）の tuple_insert フィールドとして登録されている。
  • **呼び出し先**:
  • ExecFetchSlotHeapTuple
  • RelationGetRelid
  • heap_insert（src/backend/access/heap/heapam.c）
  • ItemPointerCopy
  • pfree

関連データ構造

  • Relation: テーブルのメタデータ。
  • TupleTableSlot: 実行エンジンとストレージ間でタプルをやり取りするためのコンテナ。
  • HeapTuple: ヒープ上のタプル表現。
  • TableAmRoutine: テーブルAMの関数ポインタテーブル。

失敗モードと境界条件

  • heap_insert が内部でエラーを発生させた場合、トランザクションの異常終了に伴い処理が中断される可能性がある。
  • メモリ不足時には pfree 以前の段階でエラーが発生する場合がある。

保守担当者向けメモ

  • 本関数は heapam.c の heap_insert をラップするブリッジ関数である。ロジックの変更が必要な場合は、まず heap_insert の挙動を確認すること。
  • slot の管理状態（shouldFree）に注意し、メモリリークが発生しないよう実装されている。
