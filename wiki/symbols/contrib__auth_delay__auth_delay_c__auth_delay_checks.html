auth_delay_checks

定義

  • 種別: function
  • 定義箇所: contrib/auth_delay/auth_delay.c:33

目的

認証失敗時に意図的な遅延を挿入し、ブルートフォース攻撃に対する簡易的な防御を行う。

シグネチャまたは形状

static void auth_delay_checks(Port *port, int status)

入力と出力

  • 入力:
  • port: 接続情報を保持する Port 構造体へのポインタ。
  • status: 認証結果を示す整数値（STATUS_OK 等）。
  • 出力: なし（void）。

前提条件と不変条件

  • ClientAuthentication_hook に登録されるコールバック関数として動作する。
  • auth_delay_milliseconds 変数に設定されたミリ秒数に基づいて待機時間が決定される。

アルゴリズムまたは振る舞い

1. **フックの連鎖**: original_client_auth_hook が設定されている場合、まずそれを呼び出し、他の認証プラグインの処理を優先させる。 2. **認証ステータスの判定**: status が STATUS_OK ではない（認証失敗）場合、以下の処理を行う。 3. **遅延の実行**: pg_usleep を使用し、auth_delay_milliseconds に基づく時間（ミリ秒単位）だけプロセスを停止させる。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の認証プロセス（ClientAuthentication_hook を介して呼び出される）。
  • 呼び出し先:
  • original_client_auth_hook（存在する場合）。
  • pg_usleep（認証失敗時）。

関連データ構造

  • Port: 接続状態を管理する構造体。
  • original_client_auth_hook: 本モジュール読み込み前に設定されていた既存のフック関数を保持する変数。
  • auth_delay_milliseconds: 遅延時間をミリ秒単位で保持する GUC 変数。

失敗モードと境界条件

  • **ブロッキング**: pg_usleep はプロセスをブロックするため、認証失敗が短時間に大量発生した場合、バックエンドプロセスが長時間占有され、リソース枯渇を招く可能性がある。
  • **フックの競合**: 他の認証プラグインと併用する場合、読み込み順序によりフックの実行順序が決定される。

保守担当者向けメモ

  • 本関数は ClientAuthentication_hook に代入されることで実行される。
  • pg_usleep の引数はマイクロ秒単位であるため、1000L * auth_delay_milliseconds による単位変換を行っている点に注意すること。
