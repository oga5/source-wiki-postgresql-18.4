ineq_histogram_selectivity

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/selfuncs.c:1048

目的

統計情報のヒストグラムを使用して、不等号演算子（<, <=, >, >=）を含むクエリ条件の選択率を推定する。

シグネチャまたは形状

double ineq_histogram_selectivity(PlannerInfo *root,
                                  VariableStatData *vardata,
                                  Oid opoid, FmgrInfo *opproc, bool isgt, bool iseq,
                                  Oid collation,
                                  Datum constval, Oid consttype)

入力と出力

  • **入力**:
  • vardata: 対象列の統計情報（pg_statistic 等）。
  • opoid, opproc: 比較演算子の OID および関数情報。
  • isgt: 比較が「より大きい（>）」方向かを示すフラグ。
  • iseq: 比較に等号が含まれるかを示すフラグ。
  • constval, consttype: 比較対象の定数値とその型。
  • **出力**:
  • 0.0 から 1.0 の範囲の double 型値（推定選択率）。ヒストグラムが利用できない場合は -1.0 を返す。

前提条件と不変条件

  • vardata->statsTuple が有効であること。
  • 比較演算子がヒストグラムのソート順序と互換性を持つこと（comparison_ops_are_compatible で検証）。
  • 戻り値は CLAMP_PROBABILITY マクロ等で 0.0〜1.0 に収める必要がある（本関数内では -1.0 が未計算を示す）。

アルゴリズムまたは振る舞い

1. **ヒストグラム取得**: get_attstatsslot を使用して STATISTIC_KIND_HISTOGRAM を取得する。 2. **二分探索**: ヒストグラムの各ビンに対して比較演算子を適用し、定数値がどの位置にあるかを特定する。

  • 探索中、必要に応じて get_actual_variable_range を呼び出し、ヒストグラムの端点を最新の列の最小値・最大値で更新する。

    • 3. **範囲外判定**: 定数値がヒストグラムの範囲外にある場合、選択率を 0.0 または 1.0 とみなす。 4. **補間**: 定数値がヒストグラムのビン内にある場合、線形補間を用いて選択率を算出する。

  • 等号が含まれる場合、get_variable_numdistinct を用いて等価条件の選択率を別途推定し、加算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/selfuncs.c (scalarineqsel 等の内部)
  • src/backend/utils/adt/like_support.c
  • **呼び出し先**:
  • get_attstatsslot, free_attstatsslot
  • comparison_ops_are_compatible
  • get_actual_variable_range
  • FunctionCall2Coll
  • get_variable_numdistinct

関連データ構造

  • VariableStatData: 列の統計情報とメタデータを保持する構造体。
  • AttStatsSlot: pg_statistic から抽出された統計データ（ヒストグラム値や MCV）を保持する構造体。

失敗モードと境界条件

  • **統計情報欠落**: vardata->statsTuple が無効、またはヒストグラムが存在しない場合は -1.0 を返す。
  • **型不一致**: 演算子と列の型が互換性を持たない場合、推定をスキップする。
  • **端点更新**: get_actual_variable_range が失敗した場合、古いヒストグラム値を使用して推定を続行する。

保守担当者向けメモ

  • 本関数は scalarineqsel の中核を担う。ヒストグラムの端点更新は、ANALYZE 以降のデータ変動による推定誤差を軽減するための重要な処理である。
  • 等価条件の推定（eq_selec）は、定数が MCV である可能性を考慮できないため、論理的に厳密ではない点に注意すること。
