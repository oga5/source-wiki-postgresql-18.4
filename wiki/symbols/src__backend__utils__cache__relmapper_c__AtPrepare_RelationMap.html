AtPrepare_RelationMap

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relmapper.c:587

目的

トランザクションのプリペア（PREPARE TRANSACTION）実行時に、リレーションマッピングの変更が保留されていないかを検証する。

シグネチャまたは形状

void AtPrepare_RelationMap(void)

入力と出力

  • 入力: なし（グローバルな更新状態を参照）
  • 出力: なし（正常時は復帰、異常時はエラーをスロー）

前提条件と不変条件

  • リレーションマッピングの変更は、プリペアされるトランザクション内では許可されない。

アルゴリズムまたは振る舞い

1. active_shared_updates、active_local_updates、pending_shared_updates、pending_local_updates の各構造体における num_mappings を確認する。 2. いずれかの値が 0 でない場合、リレーションマッピングが変更されていると判断し、ERRCODE_FEATURE_NOT_SUPPORTED を伴うエラーを発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xact.c:2674
  • 呼び出し先: なし（本関数から直接呼び出される関数はソース内に記載なし）

関連データ構造

  • active_shared_updates / active_local_updates
  • pending_shared_updates / pending_local_updates

    • （これらは relmapper.c 内で管理されるマッピング更新状態を保持する構造体）

失敗モードと境界条件

  • トランザクション内でリレーションマッピングの変更（RelationMapUpdateMap 等による）が行われている場合、ereport(ERROR, ...) によりトランザクションのプリペアが拒否される。

保守担当者向けメモ

  • 本関数は、2フェーズコミットにおいてリレーションマッピングの変更をサポートしていない制約を強制するために存在する。将来的にサポートが必要な場合は、このチェックを解除し、プリペアされたトランザクションのコミット/ロールバック時のマッピング永続化ロジックを実装する必要がある。
