gss_buffer_desc

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-int.h:61

目的

GSSAPI 認証プロセスにおいて、データトークン（入力および出力）を保持するためのバッファ構造を提供します。

シグネチャまたは形状

typedef struct
{
	void	   *value;
	int			length;
} gss_buffer_desc;

入力と出力

  • value: データ本体を指すポインタ。
  • length: value が指すデータのバイト長。

前提条件と不変条件

  • value は有効なメモリ領域を指している必要があります。
  • length は負の値であってはなりません（具体的な検証ロジックはソース抜粋外）。

アルゴリズムまたは振る舞い

GSSAPI 認証の送受信において、トークンデータをカプセル化するために使用されます。認証ルーチンは、この構造体を通じてバッファのポインタとサイズを GSSAPI ライブラリへ受け渡します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/libpq/auth.c
  • src/backend/libpq/be-gssapi-common.c
  • src/backend/libpq/be-secure-gssapi.c
  • src/include/libpq/libpq-be.h
  • src/interfaces/libpq/fe-auth.c
  • src/interfaces/libpq/fe-gssapi-common.c
  • **呼び出し先**:
  • ソース抜粋からは、本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • libpq-be.h: バックエンド側の GSSAPI 関連定義で使用。
  • fe-gssapi-common.h: フロントエンド側の GSSAPI 関連定義で使用。

失敗モードと境界条件

  • value が NULL の状態で length が 0 でない場合、メモリ参照エラーが発生する可能性があります。
  • 認証トークンのサイズが許容範囲を超えた場合の挙動については、本構造体単体では規定されていません。

保守担当者向けメモ

  • 本構造体は libpq-int.h で定義されていますが、src/include/libpq/libpq-be.h でも定義されており、クライアントとサーバーの両方で共通のデータ形式として利用されています。変更時は両方のコンテキストへの影響を考慮してください。
