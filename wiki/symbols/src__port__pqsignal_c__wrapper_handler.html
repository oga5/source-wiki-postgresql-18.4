wrapper_handler

定義

  • 種別: function
  • 定義箇所: src/port/pqsignal.c:85

目的

シグナル発生時に実行される中間ハンドラとして、errno の保護およびプロセス ID の検証を行い、安全に実際のシグナルハンドラへ処理を委譲する。

シグネチャまたは形状

static void wrapper_handler(SIGNAL_ARGS) ※ SIGNAL_ARGS は libpq/pqsignal.h で定義されるシグナルハンドラの引数型。

入力と出力

  • 入力: SIGNAL_ARGS（シグナル番号を含む）
  • 出力: なし（void）

前提条件と不変条件

  • postgres_signal_arg が 0 より大きく PG_NSIG 未満であること。
  • FRONTEND が定義されていない場合、MyProcPid が設定済みであること。
  • FRONTEND が定義されていない場合、MyProcPid が PostmasterPid と一致し、かつ IsUnderPostmaster が偽である状態（ポストマスタープロセス自身がシグナルを受け取る状況）ではないこと。

アルゴリズムまたは振る舞い

1. 現在の errno を save_errno に退避する。 2. FRONTEND が定義されていない場合、現在のプロセス ID (getpid()) と MyProcPid を比較する。

  • 不一致の場合、当該シグナルのデフォルト動作 (SIG_DFL) を設定し、自身に対して raise() を実行して終了する。

    • 3. 一致する場合、pqsignal_handlers 配列からシグナル番号に対応する関数を取り出し、実行する。 4. 最後に save_errno を errno に復元する。

呼び出し元と呼び出し先

  • 呼び出し元: OS のシグナルサブシステム（sigaction 経由で登録）。
  • 呼び出し先: pqsignal_handlers に登録された各シグナルハンドラ関数。

関連データ構造

  • pqsignal_handlers[]: 登録されたシグナルハンドラを保持する内部配列。
  • MyProcPid: 現在のプロセスの PID（miscadmin.h 由来）。

失敗モードと境界条件

  • MyProcPid が getpid() と一致しない場合、不正なプロセスでのシグナル処理を回避するため、ハンドラをデフォルトに戻してシグナルを再送する。
  • postgres_signal_arg が範囲外の場合、Assert により停止する。

保守担当者向けメモ

  • 本関数は fork() 後の子プロセスが誤って親プロセスのシグナルハンドラを実行するのを防ぐガードレールとして機能する。
  • FRONTEND ビルド時にはプロセス検証ロジックがスキップされるため、環境ごとの挙動の違いに注意すること。
