pgstat_wal_reset_all_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_wal.c:156

目的

共有メモリ上に保持されているWAL統計情報をリセットし、リセット時刻を更新します。

シグネチャまたは形状

void pgstat_wal_reset_all_cb(TimestampTz ts)

入力と出力

  • **入力**: ts (TimestampTz) - リセット時刻として記録するタイムスタンプ。
  • **出力**: なし。共有メモリ上の PgStatShared_Wal 構造体を直接更新します。

前提条件と不変条件

  • 共有メモリ上のWAL統計領域（pgStatLocal.shmem->wal）が初期化済みであること。
  • 本関数は pgstat モジュールのコールバックとして呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. 共有メモリ上のWAL統計構造体へのポインタを取得します。 2. LWLockAcquire を使用して、対象の統計構造体に対して排他ロック（LW_EXCLUSIVE）を取得します。 3. memset を使用して、統計構造体内の stats フィールドをゼロクリアします。 4. 引数 ts を stat_reset_timestamp に代入し、リセット時刻を更新します。 5. LWLockRelease を使用してロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:480 にて、統計システムのコールバックテーブル（reset_all_cb）に登録されています。
  • **呼び出し先**: LWLockAcquire および LWLockRelease（共有メモリの排他制御用）。

関連データ構造

  • PgStatShared_Wal: 共有メモリ上のWAL統計を保持する構造体。
  • pgStatLocal: 統計システムのローカル状態を保持するグローバル変数。

失敗モードと境界条件

  • ロック取得時に他のプロセスが共有メモリを操作している場合、ロックが解放されるまで待機します。
  • 共有メモリが未初期化の状態で呼び出された場合の挙動については、本ソースコード内では定義されていません。

保守担当者向けメモ

  • 本関数は pgstat モジュールから動的に呼び出されるコールバックです。シグネチャを変更する場合は、pgstat 側のコールバック定義および登録箇所との整合性に注意してください。
