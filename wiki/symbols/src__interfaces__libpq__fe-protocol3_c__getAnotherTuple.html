getAnotherTuple

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:776

目的

PostgreSQL プロトコル 3.0 におけるデータ行（"D" メッセージ）を解析し、受信バッファからフィールド値を抽出して行処理関数へ渡す。

シグネチャまたは形状

static int getAnotherTuple(PGconn *conn, int msgLength)

入力と出力

  • **入力**:
  • conn: 現在の接続状態を保持する PGconn 構造体。
  • msgLength: 受信した "D" メッセージの長さ。
  • **出力**:
  • 成功時は 0 を返す。エラー発生時も 0 を返すが、内部的に conn->errorMessage が更新され、エラー状態が設定される。

前提条件と不変条件

  • conn->result が有効な PGresult オブジェクトを指していること。
  • conn->inBuffer にメッセージデータが格納されており、conn->inCursor が現在の読み込み位置を指していること。
  • rowbuf[i].value は、NULL 値であっても常にデータバッファ内の有効なアドレスを指すこと。

アルゴリズムまたは振る舞い

1. メッセージからフィールド数（tupnfields）を読み取り、期待される数と一致するか検証する。 2. 必要に応じて conn->rowBuf を再割り当て（realloc）し、フィールド情報を格納する領域を確保する。 3. 各フィールドについて、4バイトの長さ（vlen）を読み取り、rowbuf に格納する。 4. vlen が正の場合、pqSkipnchar を使用してバッファのカーソルを該当データ分だけ進める。 5. 全フィールドの走査後、pqRowProcessor を呼び出して行データを処理する。 6. 処理中にエラーが発生した場合、pqClearAsyncResult で既存の結果を破棄し、エラーメッセージを構築して pqSaveErrorResult を呼び出す。 7. エラー時は conn->inCursor をメッセージの末尾まで強制的に進め、後続の解析への影響を防ぐ。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c 内のメッセージ解析ループ（pqParseInput3 等から間接的に呼び出される）。
  • **呼び出し先**: pqGetInt, pqSkipnchar, pqRowProcessor, pqClearAsyncResult, pqSaveErrorResult, appendPQExpBuffer, realloc, libpq_gettext。

関連データ構造

  • PGconn: 接続状態および入力バッファを管理。
  • PGresult: 構築中のクエリ結果。
  • PGdataValue: 各フィールドの長さと値のポインタを保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: realloc が失敗した場合、エラーメッセージを生成して終了する。
  • **フィールド数不一致**: 受信したフィールド数が期待値と異なる場合、エラーとして処理する。

保守担当者向けメモ

  • 本関数は pqParseInput3 の解析ループの一部として動作する。
  • エラー発生時に inCursor を適切に進めることで、不正なメッセージが後続の解析を阻害しないように設計されている。
  • pqRowProcessor が失敗した場合、既存の PGresult をクリアしてエラー結果に置き換えるため、呼び出し側は結果の整合性に注意すること。
