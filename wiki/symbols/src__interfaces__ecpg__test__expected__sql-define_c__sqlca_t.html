sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-define.c:45

目的

ECPG（Embedded SQL for C）において、SQL文の実行結果やエラー情報を保持する「SQL通信領域（SQL Communication Area）」を定義する構造体です。

シグネチャまたは形状

struct sqlca_t {
    char    sqlcaid[8];
    long    sqlabc;
    long    sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    } sqlerrm;
    char    sqlerrp[8];
    long    sqlerrd[6];
    char    sqlwarn[8];
    char    sqlstate[5];
};

入力と出力

本構造体は、ECPGランタイムライブラリによって実行されたSQL文のステータスを格納する出力先として機能します。

  • **主な出力フィールド**:
  • sqlcode: SQL実行結果コード（0は成功、負の値はエラー）。
  • sqlerrm: エラーメッセージ情報。
  • sqlerrd[2]: 直近の INSERT, UPDATE, DELETE で処理された行数。
  • sqlwarn: 警告フラグ。
  • sqlstate: SQL標準準拠のエラーコード。

前提条件と不変条件

  • sqlca.h をインクルードすることで利用可能です。
  • ECPGプリプロセッサが生成するコードにおいて、各 EXEC SQL 文の直後に sqlca.sqlcode をチェックするエラーハンドリングパターンが定型的に挿入されます。

アルゴリズムまたは振る舞い

SQL文の実行結果を反映するコンテナとして動作します。特に sqlerrd[2] はデータ操作文の実行後に処理行数を保持し、sqlwarn は文字列の切り捨てや非致命的な通知が発生した際に特定のフラグ（'W'）が設定されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/compatlib/informix.c: sqlca_init の初期化および ECPGget_sqlca() を介した取得。
  • src/interfaces/ecpg/ecpglib/connect.c, data.c, descriptor.c: ECPGget_sqlca() を通じてランタイムライブラリが状態を更新するために使用。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しは行いません。

関連データ構造

  • sqlda_t, sqlvar_t: ECPGにおける動的SQL記述子領域。
  • sqlca.h: 本構造体の定義元ヘッダー。

失敗モードと境界条件

  • sqlcode < 0: SQL文の実行失敗を示します。
  • sqlwarn: 警告が発生した場合、インデックス0に 'W' が設定され、他のインデックスで詳細な警告内容が示されます。
  • sqlerrmc: メッセージ長は SQLERRMC_LEN に制限されます。

保守担当者向けメモ

  • 本構造体はECPGランタイムの根幹をなすため、フィールドの順序やサイズを変更するとバイナリ互換性が損なわれます。
  • テストコード（src/interfaces/ecpg/test/expected/ 配下）では、この構造体を通じてプリプロセッサが正しくエラーハンドリングを生成しているかを検証しています。
