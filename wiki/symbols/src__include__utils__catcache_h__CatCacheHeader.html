CatCacheHeader

定義

  • 種別: struct
  • 定義箇所: src/include/utils/catcache.h:184

目的

システム全体で管理されるすべての CatCache 構造体を追跡し、カタログキャッシュシステム全体の統計情報（タプル総数）を保持するためのグローバルなヘッダー構造体です。

シグネチャまたは形状

typedef struct catcacheheader
{
	slist_head	ch_caches;		/* キャッシュ構造体のリストの先頭 */
	int			ch_ntup;		/* 全キャッシュ内のタプル総数 */
} CatCacheHeader;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。ただし、src/backend/utils/cache/catcache.c 内の静的変数 CacheHdr を通じて、キャッシュの初期化や管理操作の起点として利用されます。

前提条件と不変条件

  • ch_caches は slist_head を使用しており、双方向リストとして管理されることが前提です。
  • ch_ntup は、システム内のすべてのカタログキャッシュに格納されているタプルの合計数と常に一致している必要があります。

アルゴリズムまたは振る舞い

  • カタログキャッシュシステムが初期化される際、この構造体が palloc によって割り当てられ、グローバルな管理ポインタに保持されます。
  • 新しい CatCache が作成されるたびに、そのエントリが ch_caches リストに追加されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/catcache.c 内の初期化処理（CacheHdr への割り当て）が確認されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • CatCache: ch_caches リストの各要素として格納される、個別のシステムカタログキャッシュ構造体。
  • slist_head: lib/ilist.h で定義されるリスト管理用構造体。

失敗モードと境界条件

  • メモリ不足時: palloc による割り当て失敗が発生する可能性があります。
  • 整合性エラー: ch_ntup の更新が各キャッシュの操作と同期していない場合、統計情報の不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体はカタログキャッシュのメタデータを集約する重要なポイントです。デバッグ時にシステム全体のキャッシュ状態を確認する際の起点となります。
  • CacheHdr は src/backend/utils/cache/catcache.c 内で静的変数として定義されているため、外部モジュールからの直接参照は制限されています。
