dsm_impl_unpin_segment

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/dsm_impl.c:1013

目的

動的共有メモリ（DSM）セグメントの生存期間を終了させ、関連するOSリソース（Windows環境におけるハンドル等）を解放する。

シグネチャまたは形状

void dsm_impl_unpin_segment(dsm_handle handle, void **impl_private)

入力と出力

  • handle: 解除対象のDSMセグメントを識別するハンドル。
  • impl_private: 実装固有の識別子（ハンドルやファイル記述子）を保持するポインタ。関数終了後、この値は NULL に設定される。

前提条件と不変条件

  • dynamic_shared_memory_type に基づいて動作する。
  • Windows環境下では、IsUnderPostmaster が真である場合にのみハンドル操作が行われる。

アルゴリズムまたは振る舞い

1. 現在の dynamic_shared_memory_type を確認する。 2. Windows環境（DSM_IMPL_WINDOWS）の場合:

  • IsUnderPostmaster が真であれば、DuplicateHandle を DUPLICATE_CLOSE_SOURCE フラグ付きで呼び出し、対象ハンドルを閉じる。
  • 成功後、*impl_private を NULL に更新する。

    • 3. その他のプラットフォームでは、特段の処理を行わず終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/dsm.c 内のDSM管理ロジックから呼び出される。
  • 呼び出し先: Windows環境において DuplicateHandle および _dosmaperr を使用する。

関連データ構造

  • dsm_handle: セグメント識別子。
  • dynamic_shared_memory_type: 使用中のDSM実装方式を決定するGUC変数。

失敗モードと境界条件

  • Windows環境において DuplicateHandle が失敗した場合、ereport(ERROR, ...) によりエラーを報告し、処理を中断する。その際、対象セグメント名を含むエラーメッセージが生成される。

保守担当者向けメモ

  • 本関数はプラットフォーム抽象化レイヤーの一部であり、Windows以外のプラットフォームでは現在のところ空の操作（no-op）として実装されている。
  • Windowsにおけるハンドル管理の詳細は、Postmasterプロセスとの生存期間の同期に依存している。
