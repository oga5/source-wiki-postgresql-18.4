md_readv_report

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/md.c:2045

目的

非同期 I/O (PgAio) によるブロック読み込み操作の結果を報告し、エラー発生時または部分的な読み込みが発生した際にログ出力を行う。

シグネチャまたは形状

static void md_readv_report(PgAioResult result, const PgAioTargetData *td, int elevel)

入力と出力

  • result: 実行された非同期 I/O の結果（エラーコードまたは読み込みバイト数）。
  • td: 読み込み対象のストレージ情報（リレーションのロケータ、ブロック番号、ブロック数など）。
  • elevel: ereport に渡すエラーレベル。
  • 出力: なし（ereport を通じたログ出力のみ）。

前提条件と不変条件

  • td が有効な PgAioTargetData を指していること。
  • result.error_data が 0 の場合、result.result は読み込まれたブロック数を示す。

アルゴリズムまたは振る舞い

1. relpathbackend を使用して、対象リレーションの物理ファイルパスを取得する。 2. result.error_data が非ゼロの場合、ファイルアクセスエラーとして errno を設定し、該当ブロック範囲とパスを含むエラーメッセージを elevel で報告する。 3. result.error_data が 0 の場合（部分読み込み発生時）、データ破損エラーとして、要求されたブロック範囲と実際に読み込めたバイト数を報告する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/smgr/md.c 内の非同期 I/O 処理フロー（PgAioTargetData の .report フィールド経由で参照）。
  • 呼び出し先: relpathbackend（パス取得）、ereport（ログ出力）。

関連データ構造

  • PgAioResult: 非同期 I/O の結果を保持する構造体。
  • PgAioTargetData: I/O 対象のメタデータ（リレーション情報、ブロック範囲）。
  • RelPathStr: ファイルパス文字列を保持する構造体。

失敗モードと境界条件

  • ファイルアクセスエラー: result.error_data が設定されている場合に発生。
  • 部分読み込み（Short Read）: result.error_data が 0 であっても、要求したブロック数と実際の読み込み結果が一致しない場合に発生。

保守担当者向けメモ

  • 本関数は非同期 I/O のコールバックとして機能する。
  • 部分読み込みの報告は主にデバッグ目的で利用される。
  • td->smgr.is_temp の判定により、一時ファイルと通常ファイルでパス生成ロジックが切り替わる点に注意すること。
