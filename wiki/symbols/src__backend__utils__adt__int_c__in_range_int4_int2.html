in_range_int4_int2

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int.c:685

目的

int4 型の値を対象として、指定された範囲内にあるかを判定する関数です。

シグネチャまたは形状

Datum in_range_int4_int2(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して5つの引数を受け取ります。
  • 第1引数: 比較対象の値 (int4)
  • 第2引数: 範囲の下限または上限 (int4)
  • 第3引数: オフセット値 (int2)
  • 第4引数: include_lower (boolean)
  • 第5引数: include_upper (boolean)
  • **出力**: Datum 型として判定結果（boolean）を返します。

前提条件と不変条件

  • 本関数は in_range_int4_int4 のラッパーとして動作します。
  • 第3引数（オフセット）は int2 型として受け取った後、int32 にキャストされて処理されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT16(2) を使用して第3引数を int2 として取得します。 2. Int32GetDatum((int32) ...) により、取得したオフセット値を int32 に変換します。 3. DirectFunctionCall5 を呼び出し、引数を in_range_int4_int4 関数へ転送します。 4. in_range_int4_int4 の戻り値をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に明示的な呼び出し元は記載されていません（SQLインターフェース経由での呼び出しが想定されます）。
  • **呼び出し先**: in_range_int4_int4

関連データ構造

  • Datum: PostgreSQLの汎用データ表現型。
  • PG_FUNCTION_ARGS: SQL関数呼び出し規約のためのマクロ。

失敗モードと境界条件

  • in_range_int4_int4 に依存するため、同関数の失敗モード（オーバーフローや不正な引数など）に準じます。
  • int2 から int32 へのキャストは安全に行われます。

保守担当者向けメモ

  • コードの重複を避けるため、in_range_int4_int4 を直接呼び出す実装となっています。将来的にロジックを変更する場合は、呼び出し先の関数を確認してください。
