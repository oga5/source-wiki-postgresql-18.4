float8_mi

定義

  • 種別: function
  • 定義箇所: src/include/utils/float.h:181

目的

2つの float8 型の値の減算を行い、結果がオーバーフローした場合にはPostgreSQLのエラー報告メカニズムを介して処理を中断する。

シグネチャまたは形状

static inline float8 float8_mi(const float8 val1, const float8 val2)

入力と出力

  • **入力**:
  • val1: 被減数 (float8)
  • val2: 減数 (float8)
  • **出力**:
  • 減算結果 (float8)

前提条件と不変条件

  • 入力値が有限数である場合、結果が無限大 (inf) になることは許容されない。
  • 演算結果が inf であり、かつ入力値 val1 および val2 のいずれも inf ではない場合、オーバーフローとみなされる。

アルゴリズムまたは振る舞い

1. val1 - val2 を計算し、result に代入する。 2. result が無限大 (isinf(result)) かつ、入力値 val1 と val2 が共に無限大ではない場合、float_overflow_error() を呼び出してエラーを発生させる。 3. 正常な場合は result を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistproc.c 内の複数の箇所（box 構造体の座標計算や範囲計算、gist インデックスのペナルティ計算等）。
  • **呼び出し先**:
  • isinf() (標準ライブラリ <math.h>)
  • float_overflow_error() (定義箇所は src/backend/utils/adt/float.c)

関連データ構造

  • float8: PostgreSQLの8バイト浮動小数点型。

失敗モードと境界条件

  • **オーバーフロー**: 演算結果が表現可能な範囲を超えて無限大となった場合、float_overflow_error() により処理が中断される。
  • **NaNの扱い**: 入力にNaNが含まれる場合の挙動は、標準的なC言語の浮動小数点演算の仕様に従う。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、パフォーマンスを重視している。
  • エラー報告関数 float_overflow_error は pg_noinline 属性を持つため、エラー発生時のみスタックフレームが生成される設計となっている。
