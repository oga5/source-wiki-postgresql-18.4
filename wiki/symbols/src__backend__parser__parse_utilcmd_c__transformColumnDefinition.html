transformColumnDefinition

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_utilcmd.c:590

目的

テーブル定義における個別の列定義（ColumnDef）を解析・変換し、CreateStmtContext に格納して、制約の検証やシリアル型に関連する補助的なコマンド生成を行う。

シグネチャまたは形状

static void transformColumnDefinition(CreateStmtContext *cxt, ColumnDef *column)

入力と出力

  • **入力**:
  • cxt: 変換処理の状態を保持する CreateStmtContext へのポインタ。
  • column: 変換対象の ColumnDef 構造体。
  • **出力**: なし（副作用として cxt->columns への追加や、column 構造体内部の制約リストの更新を行う）。

前提条件と不変条件

  • cxt は有効な CreateStmtContext を指している必要がある。
  • column は構文解析後の ColumnDef ノードであること。
  • SERIAL 型の処理において、配列型（serial[] 等）はサポートされていない。

アルゴリズムまたは振る舞い

1. **列の登録**: cxt->columns リストに現在の column を追加する。 2. **SERIAL 型の判定と変換**: smallserial, serial, bigserial 等の疑似型を検出し、対応する整数型（INT2OID, INT4OID, INT8OID）へ置換する。 3. **補助コマンド生成**: SERIAL 型の場合、generateSerialExtraStmts を呼び出してシーケンス生成コマンドを構築し、nextval() を呼び出す DEFAULT 制約を列に追加する。 4. **型変換**: transformColumnType を呼び出し、列の型定義を確定させる。 5. **制約の処理**:

  • transformConstraintAttrs を呼び出し、制約属性を処理する。
  • PRIMARY KEY や IDENTITY 等の制約をスキャンし、NO INHERIT 属性の整合性を確認する。
  • NULL/NOT NULL 制約の競合をチェックし、パーティションテーブルにおける NO INHERIT 指定を禁止する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_utilcmd.c 内のテーブル定義解析処理（transformCreateStmt 等の文脈から呼び出される）。
  • **呼び出し先**:
  • generateSerialExtraStmts
  • transformColumnType
  • transformConstraintAttrs
  • ereport (エラー発生時)

関連データ構造

  • CreateStmtContext: 変換中の状態管理用構造体。
  • ColumnDef: 列定義を保持するノード。
  • Constraint: 列に付与された制約ノード。

失敗モードと境界条件

  • **SERIAL 配列**: serial[] 構文が指定された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーとなる。
  • **制約の競合**: 同一列に対して NULL と NOT NULL が混在する場合、ERRCODE_SYNTAX_ERROR でエラーとなる。
  • **パーティション制約**: パーティションテーブルにおいて NOT NULL 制約に NO INHERIT が指定された場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラーとなる。

保守担当者向けメモ

  • 本関数は SERIAL 型の特殊処理と一般的な制約処理を混在させている。SERIAL 関連のロジックは generateSerialExtraStmts に依存しているため、シーケンス生成の仕様変更時は両方の確認が必要。
  • AddRelationNotNullConstraints の制限により、PRIMARY KEY 等の制約との整合性をここで強制している。将来的にこの制限が緩和された場合、本関数のチェックロジックも更新可能である。
