ExprEvalStep

定義

  • 種別: struct
  • 定義箇所: src/include/executor/execExpr.h:300

目的

クエリ実行エンジンにおいて、式評価の最小単位である「命令（ステップ）」を保持するための構造体です。プランナーが生成した式ツリーを、実行時に効率的に評価可能なフラットな命令配列へ変換した際の各要素として機能します。

シグネチャまたは形状

typedef struct ExprEvalStep {
    intptr_t opcode;
    Datum *resvalue;
    bool *resnull;
    union { ... } d;
} ExprEvalStep;

入力と出力

  • **入力**: opcode（実行する操作の種類）および union d 内の操作固有の引数。
  • **出力**: resvalue（結果の Datum）および resnull（結果が NULL かどうかを示すフラグ）への書き込み。

前提条件と不変条件

  • 構造体全体のサイズは、一般的なシステムにおけるキャッシュライン（64バイト）を超えないよう設計されています。
  • union d 内のデータは、64ビットシステムにおいて40バイト以下に抑える必要があります。
  • opcode は実行時に intptr_t として扱われ、計算された goto（computed goto）のポインタ等に置換される可能性があります。

アルゴリズムまたは振る舞い

1. 実行エンジンは、ExprEvalStep の配列を先頭から順に走査します。 2. opcode に基づいて、execExprInterp.c 等で定義された適切な評価ロジックへディスパッチされます。 3. 各ステップは、自身の計算結果を resvalue と resnull に格納し、後続のステップへ引き継ぎます。 4. EEOP_JUMP 系の命令により、配列内の特定のインデックスへ制御を移すことで、条件分岐やループを実現します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExpr.c 内の ExecInitFunc、ExecInitWholeRowVar、ExecInitSubscriptingRef 等の初期化関数が、この構造体を生成・設定します。
  • **呼び出し先**: 実行時には src/backend/executor/execExprInterp.c のインタープリタロジックが各ステップを実行します。

関連データ構造

  • ExprEvalOp: opcode に指定される操作の種類を定義する列挙型。
  • ExprState: ExprEvalStep の配列を保持する実行状態構造体。
  • Datum: 式評価の結果値。

失敗モードと境界条件

  • union 内のデータ構造が大きすぎる場合、メモリ効率やキャッシュ効率が低下します。
  • opcode と実行ロジック（ディスパッチテーブル）の順序が一致していない場合、実行時に不正な命令が呼び出されるリスクがあります。

保守担当者向けメモ

  • 新しい命令を追加する際は、union のサイズがキャッシュライン制限を超えないよう注意してください。
  • 複雑な状態を持つ命令（SubscriptingRefState 等）は、インラインではなくポインタ経由で保持する設計を採用してください。
