heap_page_items_state

定義

  • 種別: struct
  • 定義箇所: contrib/pageinspect/heapfuncs.c:122

目的

PostgreSQL のヒープページ内のアイテムを走査するセットリターン関数（SRF）である heap_page_items において、関数呼び出し間で走査状態を保持するために使用されます。

シグネチャまたは形状

typedef struct heap_page_items_state
{
    TupleDesc    tupd;    /* タプルの記述子 */
    Page         page;    /* 走査対象のページへのポインタ */
    uint16       offset;  /* 現在の走査位置（オフセット番号） */
} heap_page_items_state;

入力と出力

本構造体は関数内部のメモリ管理（palloc）を通じて生成・保持されます。直接的な入力引数や戻り値を持つ関数ではありません。

前提条件と不変条件

  • heap_page_items 関数の初回呼び出し時に palloc によってメモリが割り当てられ、以降の呼び出しで状態が更新されます。
  • offset は、ページ内の有効な行ポインタを指し示す必要があります。

アルゴリズムまたは振る舞い

1. heap_page_items の初回呼び出し時に、この構造体のインスタンスが生成されます。 2. ページヘッダの検証後、offset を用いてページ内の各行ポインタを順次走査します。 3. 各呼び出しにおいて、現在の offset に基づいてタプル情報を抽出し、次の呼び出しのために offset をインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pageinspect/heapfuncs.c 内の heap_page_items 関数。
  • **呼び出し先**: 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • TupleDesc: タプルの属性定義を保持します。
  • Page: ヒープページのメモリレイアウトを指します。

失敗モードと境界条件

  • ページが破損している場合、offset がページ境界を超えて参照される可能性があります。heap_page_items はこの構造体の値を用いて境界チェックを行い、不正な場合は NULL を返してクラッシュを回避します。

保守担当者向けメモ

  • 本構造体は heap_page_items の実行コンテキストに強く依存しています。構造体のメンバを変更する場合は、heap_page_items 内の初期化および更新ロジックも併せて修正してください。
