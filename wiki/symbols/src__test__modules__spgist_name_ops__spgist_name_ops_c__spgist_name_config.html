spgist_name_config

定義

  • 種別: function
  • 定義箇所: src/test/modules/spgist_name_ops/spgist_name_ops.c:33

目的

PostgreSQLのSP-GiSTインデックスにおいて、name 型を扱うためのインデックス構成メタデータを設定します。

シグネチャまたは形状

Datum spgist_name_config(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgConfigIn（使用せず）および spgConfigOut 構造体を受け取ります。
  • **出力**: spgConfigOut 構造体の各フィールドを更新し、PG_RETURN_VOID() を返します。

前提条件と不変条件

  • spgConfigOut 構造体へのポインタが有効である必要があります。
  • name 型のインデックス化において、内部ストレージとして text 型を使用する前提で構成されます。

アルゴリズムまたは振る舞い

1. cfg->prefixType を TEXTOID に設定し、プレフィックスにテキスト型を使用することを指定します。 2. cfg->labelType を INT2OID に設定し、ノードラベルに2バイト整数を使用します。 3. cfg->leafType を TEXTOID に設定し、リーフノードにテキスト型を使用します。 4. cfg->canReturnData を true に設定し、インデックスから元の値を復元可能にします。 5. cfg->longValuesOK を true に設定し、長い値に対してサフィックスによる短縮を許可します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSP-GiSTインデックス構築プロセス（直接的な呼び出し元コードはソース内に明示されていませんが、PG_FUNCTION_INFO_V1 マクロにより登録されています）。
  • **呼び出し先**: なし。

関連データ構造

  • spgConfigOut: SP-GiSTインデックスの構成情報を保持する構造体。
  • TEXTOID, INT2OID: PostgreSQLの型識別子。

失敗モードと境界条件

  • 供給されたソースコード内には、明示的なエラーハンドリングや境界条件のチェックは含まれていません。

保守担当者向けメモ

  • 本関数は spgtextproc.c の構成ロジックを模倣しています。
  • 照合順序（Collation）は考慮されておらず、バイナリ比較に基づいた構成となっています。
