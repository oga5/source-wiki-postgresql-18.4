LexizeData

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/ts_parse.c:41

目的

全文検索における辞書処理の現在の状態を保持する構造体です。辞書チェーンの走査状況、処理中のトークンリスト、および正規化結果を管理します。

シグネチャまたは形状

typedef struct {
    TSConfigCacheEntry *cfg;
    Oid            curDictId;
    int            posDict;
    DictSubState   dictState;
    ParsedLex     *curSub;
    ListParsedLex  towork;
    ListParsedLex  waste;
    ParsedLex     *lastRes;
    TSLexeme      *tmpRes;
} LexizeData;

入力と出力

本構造体は関数間でポインタとして受け渡され、状態の更新に使用されます。

  • **入力**: LexizeInit により TSConfigCacheEntry を用いて初期化されます。
  • **出力**: 辞書処理の結果（正規化された語彙）は、towork および waste リスト、または tmpRes を通じて後続の処理へ提供されます。

前提条件と不変条件

  • towork リストには現在処理中のトークンが格納され、waste リストには既に正規化済みの語彙が格納されます。
  • シソーラスのように複数トークンを跨ぐ正規化を行う場合、lastRes および tmpRes を使用して一時的な状態を保持します。

アルゴリズムまたは振る舞い

辞書チェーンを順次適用する際の「状態マシン」として機能します。 1. LexizeInit で設定を読み込みます。 2. LexizeExec が辞書を走査し、トークンを towork から waste へ移動させながら正規化を行います。 3. 辞書が複数トークンを必要とする場合、tmpRes を介してスタックのようにトークンを積み上げ、正規化が完了するまで状態を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: parsetext（ts_parse.c 内のメイン関数）が本構造体をインスタンス化し、辞書処理のコンテキストとして使用します。
  • **呼び出し先**: LexizeInit, LexizeAddLemm, RemoveHead, setCorrLex, moveToWaste, setNewTmpRes, LexizeExec などの各関数が、引数として LexizeData *ld を受け取り、その内部状態を操作します。

関連データ構造

  • TSConfigCacheEntry: 辞書設定のキャッシュ。
  • ParsedLex: 解析対象のトークン単位。
  • ListParsedLex: ParsedLex のリスト構造（towork, waste）。
  • TSLexeme: 正規化された最終的な語彙データ。

失敗モードと境界条件

  • **メモリ管理**: palloc を多用するため、tmpRes や lastRes の操作においてメモリリークの可能性があります。
  • **制限事項**: MAXSTRLEN を超えるトークンは、設定に応じて無視されるかエラーとなります。

保守担当者向けメモ

  • towork（作業中）と waste（処理済み）リストの状態遷移が辞書処理の核心です。デバッグ時はこれらのリストの内容を追跡してください。
  • 複数単語を跨ぐ正規化（シソーラス等）のロジックは LexizeExec 内で複雑な分岐を行うため、変更時は tmpRes のライフサイクルに注意してください。
