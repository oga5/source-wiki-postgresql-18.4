ProcessPgArchInterrupts

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pgarch.c:860

目的

アーカイバプロセスにおいて、保留中のシグナルや設定変更要求などの割り込みイベントを処理する。

シグネチャまたは形状

static void ProcessPgArchInterrupts(void)

入力と出力

  • 入力: なし（グローバルなフラグ変数を参照）
  • 出力: なし（副作用としてプロセス終了や設定再読み込みを実行）

前提条件と不変条件

  • 本関数は、アーカイバプロセスのメインループ内から呼び出されることを想定している。
  • ConfigReloadPending が真の場合、ProcessConfigFile(PGC_SIGHUP) を実行して設定を更新する。

アルゴリズムまたは振る舞い

1. **シグナル処理**: ProcSignalBarrierPending が真であれば ProcessProcSignalBarrier() を実行する。 2. **メモリコンテキストログ**: LogMemoryContextPending が真であれば ProcessLogMemoryContextInterrupt() を実行する。 3. **設定再読み込み**: ConfigReloadPending が真の場合：

  • ProcessConfigFile(PGC_SIGHUP) を呼び出す。
  • archive_command と archive_library が同時に設定されている場合はエラーを報告する。
  • archive_library の値が変更された場合、ログを出力し proc_exit(0) を呼び出してプロセスを終了させる（再起動を促す）。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/pgarch.c 内のメインループおよび待機処理（pgarch_MainLoop 等）。
  • 呼び出し先: ProcessProcSignalBarrier, ProcessLogMemoryContextInterrupt, ProcessConfigFile, ereport, proc_exit。

関連データ構造

  • XLogArchiveLibrary (GUC変数)
  • XLogArchiveCommand (GUC変数)
  • ConfigReloadPending (グローバルフラグ)
  • ProcSignalBarrierPending (グローバルフラグ)
  • LogMemoryContextPending (グローバルフラグ)

失敗モードと境界条件

  • archive_command と archive_library が同時に設定されている場合、ERROR を発生させプロセスを停止させる。
  • archive_library の変更検知時は、現在のプロセスを終了させることで設定の反映を保証する。

保守担当者向けメモ

  • アーカイブライブラリの動的なアンロード機構が存在しないため、ライブラリ変更時はプロセス再起動という設計を採用している。
  • sigsetjmp による例外キャッチの文脈で呼び出される可能性があるため、本関数内でのエラーハンドリングは慎重に行うこと。
