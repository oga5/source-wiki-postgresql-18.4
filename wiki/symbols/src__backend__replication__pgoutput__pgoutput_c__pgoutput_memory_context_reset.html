pgoutput_memory_context_reset

定義

  • 種別: function
  • 定義箇所: src/backend/replication/pgoutput/pgoutput.c:435

目的

論理レプリケーションの出力プラグインに関連付けられたリレーション同期キャッシュ（RelationSyncCache）を破棄し、メモリを解放する。

シグネチャまたは形状

static void pgoutput_memory_context_reset(void *arg)

入力と出力

  • **入力**: void *arg（使用されていない引数）
  • **出力**: なし（void）

前提条件と不変条件

  • RelationSyncCache が初期化されている場合、本関数によって破棄され、ポインタは NULL にリセットされる。
  • 本関数実行後、RelationSyncCache は無効な状態となる。

アルゴリズムまたは振る舞い

1. グローバル変数 RelationSyncCache が NULL でないかを確認する。 2. NULL でない場合、hash_destroy を呼び出してハッシュテーブルを破棄する。 3. RelationSyncCache を NULL に設定し、二重解放や不正な参照を防ぐ。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/pgoutput/pgoutput.c:1788（直接呼び出し）
  • src/backend/replication/pgoutput/pgoutput.c:474（コールバック関数として登録）
  • **呼び出し先**:
  • hash_destroy（PostgreSQLのハッシュテーブル管理関数）

関連データ構造

  • RelationSyncCache: パブリケーションとテーブルの同期状態を保持するハッシュテーブル。

失敗モードと境界条件

  • RelationSyncCache が既に NULL の場合、本関数は何も行わず安全に終了する。

保守担当者向けメモ

  • 本関数はメモリコンテキストのリセット時にキャッシュをクリーンアップするために使用される。
  • arg 引数は現在の実装では使用されていないが、コールバックのシグネチャ適合のために保持されている。
