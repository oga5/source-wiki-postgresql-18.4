bms_copy

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/bitmapset.c:121

目的

既存の Bitmapset のメモリ上にコピーを作成し、新しいポインタを返します。

シグネチャまたは形状

Bitmapset *bms_copy(const Bitmapset *a)

入力と出力

  • **入力**: a (コピー対象の Bitmapset へのポインタ。NULL も許容される)
  • **出力**: コピーされた Bitmapset へのポインタ。a が NULL の場合は NULL を返す。

前提条件と不変条件

  • a が非 NULL の場合、bms_is_valid_set(a) が真であること（内部的な整合性が保たれていること）。
  • 戻り値は palloc によって割り当てられるため、呼び出し元は不要になった時点で pfree を行う責任がある。

アルゴリズムまたは振る舞い

1. a が NULL かどうかを確認し、NULL であればそのまま NULL を返す。 2. BITMAPSET_SIZE マクロを使用して、入力セットのワード数に基づいた必要なメモリサイズを計算する。 3. palloc を用いて新しい Bitmapset 用のメモリ領域を確保する。 4. memcpy を使用して、入力セットの内容を新しい領域へコピーする。 5. 新しい Bitmapset へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/dml.c
  • src/backend/commands/trigger.c
  • src/backend/executor/execMain.c
  • src/backend/executor/execPartition.c
  • src/backend/executor/nodeAgg.c
  • src/backend/lib/knapsack.c
  • src/backend/nodes/bitmapset.c
  • **呼び出し先**:
  • bms_is_valid_set (アサーション内)
  • palloc (メモリ確保)
  • memcpy (メモリコピー)

関連データ構造

  • Bitmapset: 非負整数の集合を表現する構造体。nwords と words 配列を保持する。

失敗モードと境界条件

  • a が NULL の場合、安全に NULL を返す。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は入力セットを破壊せず、完全に独立したメモリ領域を確保する。
  • Bitmapset の内部構造（nwords や words）に変更があった場合は、BITMAPSET_SIZE マクロの定義と併せて確認すること。
