ExecWindowAgg

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWindowAgg.c:2210

目的

SQLのウィンドウ関数（OVER 句）を実行する実行ノードのメイン処理関数。パーティション内のタプルをバッファリングし、ウィンドウフレームに基づいた関数評価と結果の投影を行う。

シグネチャまたは形状

static TupleTableSlot *ExecWindowAgg(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（WindowAggState としてキャストされる実行ノードの状態）。
  • **出力**: TupleTableSlot *（計算結果を含む投影されたタプル、または処理終了時の NULL）。

前提条件と不変条件

  • pstate は WindowAggState 型であること。
  • 初回呼び出し時（またはリキャン時）に calculate_frame_offsets が実行され、フレームオフセットが確定していること。
  • tuplestore に現在のパーティションの行が保持されていること。

アルゴリズムまたは振る舞い

1. **割り込みチェック**: CHECK_FOR_INTERRUPTS() を実行。 2. **状態確認**: WINDOWAGG_DONE なら直ちに終了。 3. **パーティション管理**:

  • next_partition が真であれば begin_partition を呼び出し初期化。
  • それ以外の場合は currentpos を進め、フレーム境界フラグを無効化。
  • spool_tuples で必要な行を読み込み、パーティション末尾に達した場合は release_partition を実行。

    • 4. **タプル取得**: tuplestore から現在の行を ScanTupleSlot に読み込む。GROUPS モード等の場合はピアグループの判定を行う。 5. **関数評価**: WINDOWAGG_RUN 状態であれば、ウィンドウ関数および集約関数を順次評価。 6. **フレーム更新**: フレーム境界ポインタ（framehead_ptr 等）が有効な場合、各位置を更新。 7. **メモリ管理**: tuplestore_trim により不要になった行を削除。 8. **投影**: ExecProject を呼び出し、結果を格納したスロットを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeWindowAgg.c 内の ps.ExecProcNode に登録されている（直接の呼び出し元関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: begin_partition, spool_tuples, release_partition, tuplestore_select_read_pointer, tuplestore_gettupleslot, are_peers, eval_windowfunction, eval_windowaggregates, update_frameheadpos, update_frametailpos, update_grouptailpos, tuplestore_trim, ExecProject。

関連データ構造

  • WindowAggState: 実行時状態を保持。
  • TupleTableSlot: タプルデータの保持に使用。
  • ExprContext: 式評価のコンテキスト。
  • tuplestore: パーティションデータのバッファリングに使用。

失敗モードと境界条件

  • tuplestore からの読み込み失敗時に elog(ERROR) を発生させる。
  • パーティションが空の場合、適切に処理を終了する。
  • work_mem を超えるパーティションデータはディスクにスピルされる。

保守担当者向けメモ

  • ウィンドウ関数APIは WindowObject を介して行データにアクセスするため、tuplestore の読み取りポインタ管理が重要。
  • GROUPS モードや EXCLUDE 句を使用する場合、ピアグループの判定ロジックが currentpos の進行と連動している点に注意。
