目的

ECPG（Embedded SQL for C）における WHENEVER SQLERROR CONTINUE 指令の挙動を検証する回帰テストの実行起点。SQLエラー発生時にプログラムを停止させず、処理を継続させるフローを実証する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（コマンドライン引数は使用しない）
  • 出力: 成功時に 0、接続やテーブル作成等の初期化処理でエラーが発生した場合は 1 を返して終了する。

前提条件と不変条件

  • ECPGプリプロセッサによって生成されたコードであること。
  • sqlca 構造体がECPGライブラリによって適切に更新され、sqlcode がエラー状態（< 0）を正しく反映していること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGconnect でデータベースに接続し、emp テーブルを作成する。 2. **データ準備**: INSERT 文でサンプルデータを投入する。 3. **カーソル操作**: DECLARE でカーソルを定義し、ループ内で FETCH を実行する。 4. **エラーハンドリング**: WHENEVER SQLERROR CONTINUE が設定されているため、FETCH 実行時にエラーが発生しても sqlca.sqlcode < 0 のチェックを経て continue 文が実行され、ループが継続される。 5. **終了処理**: CLOSE および DROP TABLE を実行し、exit(0) で正常終了する。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムの実行起点）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, printf, exit, strcpy

関連データ構造

  • sqlca: SQL通信領域。ECPGの実行結果（sqlcode）を保持し、エラー判定に使用される。
  • emp: データベースのレコードを保持するための構造体（ename, sal, comm を含む）。

失敗モードと境界条件

  • ECPGconnect や CREATE TABLE 等の初期化ステップで sqlca.sqlcode < 0 となった場合、即座に exit(1) が呼び出される。
  • FETCH 実行時にエラーが発生した場合、WHENEVER SQLERROR CONTINUE の影響によりプログラムは停止せず、次のイテレーションへ移行する。
  • FETCH でデータが見つからない場合（ECPG_NOT_FOUND）、ループを break して終了する。

保守担当者向けメモ

  • 本ファイルはプリプロセッサの出力結果であるため、直接編集しないこと。
  • WHENEVER 文は、プリプロセッサがソースコードを走査する際に、後続のすべてのSQL文に対してエラーハンドリングコードを自動挿入する仕組みである。
  • ECPGdebug(1, stderr) が有効なため、実行時の詳細なログは stderr に出力される。
