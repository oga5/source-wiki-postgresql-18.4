px_combo

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/px.h:163

目的

暗号化および復号化処理を抽象化し、共通のインターフェースを通じて実行するための構造体です。

シグネチャまたは形状

struct px_combo {
    int (*init) (PX_Combo *cx, const uint8 *key, unsigned klen, const uint8 *iv, unsigned ivlen);
    int (*encrypt) (PX_Combo *cx, const uint8 *data, unsigned dlen, uint8 *res, unsigned *rlen);
    int (*decrypt) (PX_Combo *cx, const uint8 *data, unsigned dlen, uint8 *res, unsigned *rlen);
    unsigned (*encrypt_len) (PX_Combo *cx, unsigned dlen);
    unsigned (*decrypt_len) (PX_Combo *cx, unsigned dlen);
    void (*free) (PX_Combo *cx);

    PX_Cipher *cipher;
    unsigned padding;
};

入力と出力

  • **入力**: 各関数ポインタは PX_Combo 構造体へのポインタを第一引数として受け取ります。init は鍵 (key) と初期化ベクトル (iv) を受け取り、encrypt/decrypt はデータ (data) とその長さを入力として受け取ります。
  • **出力**: init, encrypt, decrypt は成功時に 0 を、失敗時にエラーコードを返します。encrypt_len と decrypt_len は処理後のデータ長を返します。

前提条件と不変条件

  • PX_Combo 構造体は、使用前に適切な初期化関数によって関数ポインタが設定されている必要があります。
  • free 関数ポインタを呼び出すことで、構造体が保持するリソースを解放する必要があります。

アルゴリズムまたは振る舞い

  • init: 暗号化コンテキストを初期化します。
  • encrypt/decrypt: 指定されたデータに対して暗号化または復号化を実行し、結果を res に格納します。
  • encrypt_len/decrypt_len: 入力データ長に基づき、出力に必要なバッファサイズを計算します。
  • free: 構造体に関連付けられたメモリやコンテキストを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/px.h 内で typedef struct px_combo PX_Combo; として定義されており、モジュール内の上位レイヤーから利用されます。
  • 呼び出し先: 具体的な実装は px.c やバックエンド（OpenSSL等）のコードに依存しますが、本ソース excerpt からは特定の呼び出し先は特定できません。

関連データ構造

  • PX_Cipher: 暗号化アルゴリズムのコンテキストを保持する構造体。
  • uint8: データ処理に使用される符号なし8ビット整数型。

失敗モードと境界条件

  • 各関数ポインタは、処理失敗時に px.h で定義された PXE_* エラーコードを返すことが想定されています。
  • バッファサイズが不足している場合や、無効な鍵・IVが渡された場合の挙動は、各実装に依存します。

保守担当者向けメモ

  • 本構造体は関数ポインタによるポリモーフィズムを実現しています。デバッグ時には cipher メンバが指す具体的な実装コンテキストを確認してください。
  • padding メンバはパディング方式の管理に使用されます。
