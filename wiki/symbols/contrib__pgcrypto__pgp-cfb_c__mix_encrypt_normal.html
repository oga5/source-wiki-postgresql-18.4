mix_encrypt_normal

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-cfb.c:93

目的

CFB（Cipher Feedback）モードにおける標準的な暗号化処理を実行します。入力データと暗号化されたフィードバックレジスタ（fre）との XOR 演算を行い、その結果を暗号文として出力しつつ、内部バッファを更新します。

シグネチャまたは形状

static int mix_encrypt_normal(PGP_CFB *ctx, const uint8 *data, int len, uint8 *dst)

入力と出力

  • ctx: 暗号化コンテキスト（PGP_CFB 構造体）。現在の位置（pos）、暗号化済みバッファ（encbuf）、およびフィードバックレジスタの暗号化結果（fre）を保持。
  • data: 暗号化対象の入力データ。
  • len: 入力データの長さ。
  • dst: 暗号化結果を格納するバッファ。
  • 戻り値: 処理したデータの長さ（len を返す）。

前提条件と不変条件

  • ctx->fre には、ブロック暗号によって暗号化された現在のフィードバックレジスタの値が格納されている必要があります。
  • ctx->pos は、現在のブロック内でのオフセットを指しており、処理後に len 分だけインクリメントされます。

アルゴリズムまたは振る舞い

1. ctx->pos から開始し、len 分だけループを実行します。 2. 各バイトについて、入力データ data と ctx->fre の対応する位置の値を XOR 演算します。 3. XOR の結果を dst に書き込み、同時に ctx->encbuf に保存します。 4. 処理終了後、ctx->pos を len だけ進めます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-cfb.c 内のロジック（ctx->resync フラグが偽の場合に選択される）。
  • 呼び出し先: 直接的な関数呼び出しはありません（演算のみ）。

関連データ構造

  • PGP_CFB: 暗号化コンテキスト。pos（現在位置）、encbuf（暗号化バッファ）、fre（暗号化されたフィードバックレジスタ）を保持します。

失敗モードと境界条件

  • len が ctx->pos との合計でバッファサイズを超過する場合の安全性については、本関数単体では制御していません（呼び出し元に依存します）。

保守担当者向けメモ

  • 本関数は「通常」の暗号化パスです。PGP の古い仕様に関連する再同期処理が必要な場合は、mix_encrypt_resync が使用されます。
  • ctx->pos の更新は呼び出し元でブロック境界を管理するための重要な状態遷移です。変更時は cfb_process 等の整合性に注意してください。
