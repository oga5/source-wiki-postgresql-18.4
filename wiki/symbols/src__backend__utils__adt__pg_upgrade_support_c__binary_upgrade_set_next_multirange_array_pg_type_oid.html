binary_upgrade_set_next_multirange_array_pg_type_oid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_upgrade_support.c:85

目的

pg_upgrade 実行中に、次に作成されるマルチレンジ配列型の OID をシステムカタログへ強制的に割り当てるための予約値を設定する。

シグネチャまたは形状

Datum binary_upgrade_set_next_multirange_array_pg_type_oid(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して、次に割り当てるべき OID を受け取る。
  • **出力**: PG_RETURN_VOID() により、戻り値なし（void）。

前提条件と不変条件

  • サーバーが binary_upgrade モードで起動されている必要がある。CHECK_IS_BINARY_UPGRADE マクロにより、この条件が満たされない場合はエラーが発生する。

アルゴリズムまたは振る舞い

1. 引数から OID を取得する。 2. CHECK_IS_BINARY_UPGRADE マクロを実行し、アップグレードモードであることを検証する。 3. グローバル変数 binary_upgrade_next_mrng_array_pg_type_oid に取得した OID を代入する。 4. 処理を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c において、アップグレード時のダンプ出力の一部として SQL 経由で呼び出される。
  • **呼び出し先**: 内部的な関数呼び出しはない。

関連データ構造

  • binary_upgrade_next_mrng_array_pg_type_oid: src/include/catalog/binary_upgrade.h で宣言されているグローバル変数。

失敗モードと境界条件

  • binary_upgrade モード以外で呼び出された場合、CHECK_IS_BINARY_UPGRADE により実行が中断される。

保守担当者向けメモ

  • 本関数はシステムカタログの整合性を直接操作する強力なツールである。誤用はカタログの破損を招くため、pg_upgrade 以外のコンテキストでの利用は想定されていない。
