CheckPointReplicationOrigin

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/origin.c:595

目的

論理レプリケーションの進捗状況（レプリケーション起点）を共有メモリからディスクへ永続化し、チェックポイント完了時に整合性を保証する。

シグネチャまたは形状

void CheckPointReplicationOrigin(void)

入力と出力

  • **入力**: なし（共有メモリ上の replication_states 配列および max_active_replication_origins を参照）。
  • **出力**: なし（副作用として pg_replorigin_checkpoint ファイルへの書き出しおよび durable_rename による永続化を行う）。

前提条件と不変条件

  • max_active_replication_origins > 0 であること。
  • 呼び出し時に他のチェックポイント処理が並行して実行されていないこと。
  • 共有メモリ上の各 ReplicationState は、個別の lock によって保護されていること。

アルゴリズムまたは振る舞い

1. 一時ファイル（pg_replorigin_checkpoint.tmp）を作成し、マジックナンバーを書き込む。 2. ReplicationOriginLock を共有モードで取得し、起点情報の作成・削除を抑止する。 3. 全ての有効な ReplicationState を走査し、以下の処理を繰り返す：

  • 各状態の lock を取得し、remote_lsn と local_lsn をコピーする。
  • XLogFlush(local_lsn) を呼び出し、該当するローカルのコミットレコードがディスクにフラッシュされていることを保証する。
  • 状態データを一時ファイルへ書き出す。

    • 4. CRCを計算・書き込み、ファイルを閉じる。 5. durable_rename を使用して、一時ファイルを永続的なチェックポイントファイルへ置き換える。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c（チェックポイント処理の一部として呼び出される）。
  • **呼び出し先**: XLogFlush, LWLockAcquire, LWLockRelease, OpenTransientFile, CloseTransientFile, durable_rename 等のストレージおよびWAL管理関数。

関連データ構造

  • ReplicationState: 共有メモリ上の進捗管理構造体。
  • ReplicationStateOnDisk: ディスク書き出し用の固定長構造体。
  • ReplicationOriginLock: 起点テーブル全体を保護するロック。

失敗モードと境界条件

  • ファイル作成、書き込み、またはクローズに失敗した場合、PANIC を発生させてプロセスを停止する。
  • ディスク容量不足（ENOSPC）が発生した場合も PANIC となる。
  • max_active_replication_origins が 0 の場合は何もせず即座に復帰する。

保守担当者向けメモ

  • 本関数はチェックポイントの整合性を維持するため、XLogFlush を介してローカルLSNの永続化を強制している。
  • 内部ID（roident）が InvalidRepOriginId のエントリはスキップされる。
  • 8バイトのLSN書き込みのアトミック性を保証するため、必ず各 ReplicationState のロックを取得してから読み出すこと。
