pg_popcount_neon

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_aarch64.c:323

目的

AArch64アーキテクチャにおいて、Neon命令セットを活用してバッファ内のビットカウント（popcount）を高速に実行する。

シグネチャまたは形状

static uint64 pg_popcount_neon(const char *buf, int bytes)

入力と出力

  • **入力**:
  • buf: カウント対象のデータが格納されたバッファへのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • **出力**:
  • バッファ内の全ビットの合計（uint64）。

前提条件と不変条件

  • POPCNT_AARCH64 が定義されている環境で利用可能。
  • buf は有効なメモリ領域を指している必要がある。
  • 命令レベルの並列性を最大化するため、内部で4つのアキュムレータ（accum1〜accum4）を使用する。

アルゴリズムまたは振る舞い

1. **SIMDループ**: 128ビット（uint8x16_t）レジスタを4つ並列に処理するループを展開し、vcntq_u8 でビットカウントを行う。 2. **端数処理**:

  • 4レジスタ単位で処理しきれない残余データに対し、2レジスタ単位の処理を行う。
  • さらに残った8バイト単位のデータは pg_popcount64 を使用して処理する。
  • 最後に残った1バイト単位のデータは pg_number_of_ones テーブルを参照して逐次処理する。

    • 3. **集計**: 各アキュムレータの値を加算し、最終的な合計を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/port/pg_popcount_aarch64.c 内の pg_popcount_choose（関数ポインタへの代入）。
  • **呼び出し先**:
  • pg_popcount64（8バイト単位の処理時）。
  • vld1q_u8, vcntq_u8, vpaddlq_u8, vpaddlq_u16, vpadalq_u32, vaddq_u64, vaddvq_u64 等のNeon組み込み関数。

関連データ構造

  • uint8x16_t, uint64x2_t: Neon SIMDレジスタ型。
  • pg_number_of_ones: 逐次処理用ビットカウントテーブル。

失敗モードと境界条件

  • bytes が0以下の場合、ループは実行されず0が返される。
  • バッファサイズが16バイト未満の場合、SIMDループはスキップされ、逐次処理のみが行われる。

保守担当者向けメモ

  • 本関数は pg_popcount_optimized 関数ポインタ経由で呼び出されることを想定している。
  • コンパイラによるインライン化を促進する設計となっている。
  • SVE対応版（pg_popcount_sve）との使い分けについては、同ファイルの動的ディスパッチロジックを参照すること。
