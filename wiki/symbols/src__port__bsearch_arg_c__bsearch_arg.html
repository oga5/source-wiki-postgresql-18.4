bsearch_arg

定義

  • 種別: function
  • 定義箇所: src/port/bsearch_arg.c:54

目的

標準ライブラリの bsearch を拡張し、比較関数にユーザー定義のコンテキストポインタ（arg）を渡せるようにした二分探索の実装です。状態を保持する必要がある検索処理において、グローバル変数を使用せずに検索を実行可能にします。

シグネチャまたは形状

void *bsearch_arg(const void *key, const void *base0, size_t nmemb, size_t size,
                  int (*compar) (const void *, const void *, void *), void *arg);

入力と出力

  • **入力**:
  • key: 検索対象のキーへのポインタ。
  • base0: 検索対象となるソート済み配列の先頭ポインタ。
  • nmemb: 配列の要素数。
  • size: 各要素のバイトサイズ。
  • compar: 比較関数。引数として key、配列要素、および arg を受け取り、比較結果（負、0、正）を返す。
  • arg: 比較関数に渡されるユーザー定義のコンテキストポインタ。
  • **出力**:
  • 検索対象が見つかった場合はその要素へのポインタ。
  • 見つからない場合は NULL。

前提条件と不変条件

  • base0 が指す配列は、compar 関数に基づき昇順にソートされている必要があります。
  • compar 関数は、第1引数に key、第2引数に配列要素、第3引数に arg を受け取る仕様である必要があります。

アルゴリズムまたは振る舞い

1. lim（残りの要素数）を nmemb で初期化し、lim が 0 になるまでループします。 2. lim >> 1 を用いて中央の要素 p を特定します。 3. compar(key, p, arg) を呼び出し、結果 cmp を取得します。 4. cmp == 0 の場合、p を返します。 5. cmp > 0 の場合、探索範囲を右半分に絞り込みます（base を更新し、lim を調整）。 6. cmp < 0 の場合、探索範囲を左半分に絞り込みます（lim を半分にします）。 7. ループ終了後、見つからなければ NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のソースコードから呼び出されていることが確認されています。
  • src/backend/access/brin/brin_minmax_multi.c
  • src/backend/statistics/mcv.c
  • **呼び出し先**: 引数として渡された関数ポインタ compar を実行します。

関連データ構造

  • c.h: 基本型定義やマクロを提供します。
  • qsort_arg.c: 本関数と同様の設計思想（arg を受け取る比較関数）を持つクイックソートの実装であり、対になる操作として併用されます。

失敗モードと境界条件

  • nmemb が 0 の場合、ループは実行されず直ちに NULL を返します。
  • 比較関数が不適切な順序を返す場合、探索結果は保証されません。

保守担当者向けメモ

  • lim が奇数か偶数かによって探索範囲を調整する古典的な二分探索アルゴリズムを採用しています。
  • 比較関数 compar の実装者は、arg を通じて外部の状態に安全にアクセスできるため、グローバル変数の使用を避ける設計を推奨します。
