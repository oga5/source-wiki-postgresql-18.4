GrantRoleOptions

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/user.c:72

目的

GRANT コマンド実行時に指定されるオプション（ADMIN, INHERIT, SET）を保持し、ロールメンバーシップの付与・剥奪処理に伝達するための構造体です。

シグネチャまたは形状

typedef struct
{
	unsigned	specified;
	bool		admin;
	bool		inherit;
	bool		set;
} GrantRoleOptions;

入力と出力

  • **入力**: GRANT 文の構文解析結果から、各オプションの有無および値が設定されます。
  • **出力**: ロール管理関数（GrantRole 等）へ渡され、カタログ更新ロジックの制御に使用されます。

前提条件と不変条件

  • specified フィールドは、どのオプションが明示的に指定されたかを追跡するために使用されます。
  • admin, inherit, set は、対応するオプションが有効な場合に true となります。

アルゴリズムまたは振る舞い

  • InitGrantRoleOptions 関数によって初期化されます。
  • GrantRole や AddRoleMems などの関数において、この構造体の値を参照して pg_auth_members カタログの更新内容を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/user.c 内の GrantRole 関数および関連するメンバーシップ更新関数が、この構造体を引数として受け取ります。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • pg_auth_members: ロールメンバーシップ情報を保持するシステムカタログ。
  • GrantRole: 本構造体を利用してメンバーシップを更新する主要関数。

失敗モードと境界条件

  • 構造体内のフラグが矛盾する組み合わせ（例: 互いに排他的なオプションの同時指定）については、呼び出し側の関数で検証されます。

保守担当者向けメモ

  • 本構造体は src/include/commands/user.h にも関連付けられています。
  • オプションの追加や変更を行う際は、InitGrantRoleOptions の初期化ロジックと、GrantRole 内のフラグ参照箇所を併せて更新してください。
