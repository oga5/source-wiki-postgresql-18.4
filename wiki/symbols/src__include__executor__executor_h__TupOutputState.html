TupOutputState

定義

  • 種別: struct
  • 定義箇所: src/include/executor/executor.h:606

目的

タプル出力処理における状態管理を行うための構造体です。主にクエリ結果やバックアップデータなどの出力を、指定された DestReceiver を介して制御するために使用されます。

シグネチャまたは形状

typedef struct TupOutputState
{
	TupleTableSlot *slot;
	DestReceiver *dest;
} TupOutputState;

入力と出力

  • **入力**:
  • slot: タプルデータを保持するための TupleTableSlot。
  • dest: タプルの出力先を制御する DestReceiver。
  • **出力**: 本構造体自体が、出力処理のコンテキストとして後続の出力関数へ渡されます。

前提条件と不変条件

  • slot および dest は、出力処理を開始する前に適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

本構造体は、以下のライフサイクル関数で利用されます。 1. do_tup_output: TupOutputState を使用してタプル（values, isnull）を出力します。 2. do_text_output_multiline: テキスト形式の出力を処理します。 3. end_tup_output: 出力処理を終了し、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/backup/basebackup_copy.c
  • src/backend/commands/explain.c
  • src/backend/commands/functioncmds.c
  • src/backend/executor/execTuples.c
  • src/backend/replication/walsender.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。操作は execTuples.c 内の関数群によって行われます。

関連データ構造

  • TupleTableSlot: タプルデータを格納するスロット。
  • DestReceiver: タプルの出力先（宛先）を定義するインターフェース。

失敗モードと境界条件

  • slot または dest が NULL の状態で出力関数に渡された場合、セグメンテーションフォールト等の未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/executor/execTuples.c で定義される一連の出力ユーティリティと密接に関連しています。変更を加える際は、これらのユーティリティ関数への影響を確認してください。
