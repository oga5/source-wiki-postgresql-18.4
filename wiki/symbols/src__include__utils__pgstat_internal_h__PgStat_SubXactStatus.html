PgStat_SubXactStatus

定義

  • 種別: struct
  • 定義箇所: src/include/utils/pgstat_internal.h:172

目的

サブトランザクション単位での統計情報の収集と、トランザクションのコミット/アボート確定までの統計更新の一時保持を行うための構造体です。

シグネチャまたは形状

typedef struct PgStat_SubXactStatus
{
    int nest_level;
    struct PgStat_SubXactStatus *prev;
    dclist_head pending_drops;
    PgStat_TableXactStatus *first;
} PgStat_SubXactStatus;

入力と出力

  • **入力**: トランザクション処理中に発生するテーブルの挿入/削除カウントや、オブジェクトの削除イベント。
  • **出力**: トランザクションのコミットまたはアボート時に、確定した統計情報が上位レベルへ伝播または破棄されます。

前提条件と不変条件

  • 本構造体は TopTransactionContext 上に配置されます。
  • サブトランザクションは通常、多くのテーブルを同時に変更しないという前提で設計されています。
  • 統計情報の伝播は、トランザクションのコミット/アボートが確定するまで保留されます。

アルゴリズムまたは振る舞い

1. **統計の蓄積**: トランザクション実行中のタプル挿入/削除カウントを PgStat_TableXactStatus リスト（first）に保持します。 2. **削除の管理**: トランザクション的に削除されたオブジェクトの統計情報を pending_drops リストで管理し、コミット/アボート時に適切に処理します。 3. **階層管理**: prev ポインタにより、上位レベルのサブトランザクション状態を指すスタック構造を形成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_relation.c および src/backend/utils/activity/pgstat_xact.c 内のトランザクション管理関数（AtEOXact_PgStat_Relations, AtEOSubXact_PgStat_Relations, AtEOXact_PgStat_DroppedStats 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出し先はありません。

関連データ構造

  • PgStat_TableXactStatus: テーブルごとのトランザクション内統計情報を保持する構造体。
  • dclist_head: pending_drops を管理するための二重連結リスト。

失敗モードと境界条件

  • サブトランザクションがネストしている場合、nest_level と prev ポインタによるスタックの整合性が維持される必要があります。
  • メモリ管理は TopTransactionContext に依存しており、トランザクション終了時の適切なクリーンアップが必須です。

保守担当者向けメモ

  • 本構造体は共有メモリではなく、バックエンドローカルなメモリ（TopTransactionContext）上で動作します。
  • 統計のフラッシュタイミングについては pgstat_report_stat() のロジックと連動しているため、変更時は pgstat_xact.c のトランザクションフックを確認してください。
