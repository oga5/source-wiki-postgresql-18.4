TransactionTreeSetCommitTsData

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/commit_ts.c:140

目的

トランザクションおよびそのサブトランザクション群のコミット時刻とレプリケーション元ノードIDを、SLRU（Simple Least Recently Used）キャッシュおよび共有メモリへ一括記録する。

シグネチャまたは形状

void TransactionTreeSetCommitTsData(TransactionId xid, int nsubxids,
                                   TransactionId *subxids, TimestampTz timestamp,
                                   RepOriginId nodeid)

入力と出力

  • **入力**:
  • xid: 親トランザクションID。
  • nsubxids: サブトランザクションの数。
  • subxids: サブトランザクションIDの配列。
  • timestamp: 記録するコミット時刻。
  • nodeid: レプリケーション元のノードID。
  • **出力**: なし（副作用としてSLRUページおよび共有メモリを更新）。

前提条件と不変条件

  • commitTsShared->commitTsActive が真であること（偽の場合は即座にリターンする）。
  • SLRUページ境界を跨ぐトランザクションID群は、適切にグループ化されて処理される必要がある。

アルゴリズムまたは振る舞い

1. **アクティブ確認**: commitTsActive が無効な場合は何もしない。 2. **範囲特定**: バッチ内の最新トランザクションID（newestXact）を特定する。 3. **SLRUページへの書き込み**:

  • トランザクションID群をSLRUページ単位に分割する。
  • 各ページごとに SetXidCommitTsInPage を呼び出し、該当するXID範囲のデータを書き込む。
  • ページ境界を跨ぐ場合は、次のグループのヘッドを更新してループを継続する。

    • 4. **共有メモリ更新**: CommitTsLock を排他ロックし、最後にコミットされたトランザクション情報を共有メモリ上の commitTsShared に反映する。 5. **最新XIDの更新**: TransamVariables->newestCommitTsXid が newestXact より古い場合、これを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xact.c
  • **呼び出し先**:
  • TransactionIdToCTsPage
  • SetXidCommitTsInPage
  • LWLockAcquire / LWLockRelease
  • TransactionIdPrecedes

関連データ構造

  • CommitTimestampShared (共有メモリ上の制御構造体)
  • CommitTsLock (排他制御用ロック)
  • TransamVariables (トランザクション関連のグローバル変数)

失敗モードと境界条件

  • nsubxids が 0 の場合、親トランザクションのみが処理対象となる。
  • commitTsActive が偽の場合、処理はスキップされる（スタンバイモードでの設定変更時に発生）。
  • SLRUバッファの競合が発生した場合、SetXidCommitTsInPage 内部で待機が発生する可能性がある。

保守担当者向けメモ

  • 本関数はトランザクションコミットのクリティカルパスで実行されるため、SLRUのバッファ設定（commit_timestamp_buffers）が性能に直結する。
  • スタンバイ側ではリカバリプロセスのみが本関数を呼び出すため、commitTsActive のチェックはロックなしで行っても安全である。
