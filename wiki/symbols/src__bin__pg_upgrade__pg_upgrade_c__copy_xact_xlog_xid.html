copy_xact_xlog_xid

定義

  • **種別**: function
  • **定義箇所**: src/bin/pg_upgrade/pg_upgrade.c:749

目的

旧クラスタから新クラスタへ、トランザクション状態（XID）、コミットログ、およびマルチトランザクション（MultiXact）のメタデータを移行し、新クラスタの pg_control を適切に初期化する。

シグネチャまたは形状

static void copy_xact_xlog_xid(void)

入力と出力

  • **入力**: グローバル変数 old_cluster および new_cluster が保持するクラスタ情報および制御データ（controldata）。
  • **出力**: なし（副作用として新クラスタのデータディレクトリへのファイルコピーおよび pg_resetwal の実行）。

前提条件と不変条件

  • old_cluster および new_cluster が初期化済みであること。
  • 新クラスタのデータディレクトリが作成済みであること。

アルゴリズムまたは振る舞い

1. **コミットログのコピー**: copy_subdir_files を使用し、バージョンに応じて pg_clog または pg_xact を新クラスタへコピーする。 2. **XID の設定**: pg_resetwal を呼び出し、旧クラスタのチェックポイント情報に基づいて、新クラスタの「最古の XID」「次回の XID」「エポック」「コミットタイムスタンプ制限」を再設定する。 3. **MultiXact の移行**:

  • 両クラスタが MULTIXACT_FORMATCHANGE_CAT_VER 以降の場合、pg_multixact（offsets/members）をコピーし、pg_resetwal でマルチトランザクション ID とオフセットを同期する。
  • 新クラスタのみが該当する場合、pg_multixact/offsets を削除し、pg_resetwal でマルチトランザクション ID を再設定する。

    • 4. **WAL アーカイブの初期化**: pg_resetwal -l を使用し、WAL タイムラインを 1 にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_upgrade/pg_upgrade.c:160）。
  • **呼び出し先**: copy_subdir_files, prep_status, exec_prog, check_ok, remove_new_subdir。

関連データ構造

  • old_cluster, new_cluster: クラスタのメタデータおよび制御データを保持する構造体。

失敗モードと境界条件

  • pg_resetwal の実行に失敗した場合、exec_prog により処理が中断される可能性がある。
  • バージョン判定（906 や MULTIXACT_FORMATCHANGE_CAT_VER）により、実行されるファイル操作やコマンド引数が動的に変化する。

保守担当者向けメモ

  • pg_clog から pg_xact への名称変更など、PostgreSQL のバージョンアップに伴うディレクトリ構造の変化を考慮した実装となっている。
  • pg_resetwal の引数は、旧クラスタの controldata に強く依存しているため、構造体の定義変更時には注意が必要である。
