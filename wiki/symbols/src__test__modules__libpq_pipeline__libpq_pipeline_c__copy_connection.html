copy_connection

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:202

目的

既存のデータベース接続（PGconn）の設定情報を抽出し、それを用いて新しいデータベース接続を確立して複製を作成します。

シグネチャまたは形状

static PGconn *copy_connection(PGconn *conn)

入力と出力

  • **入力**: conn (既存の PGconn オブジェクトへのポインタ)
  • **出力**: 新たに確立された PGconn オブジェクトへのポインタ。接続失敗時は pg_fatal によりプロセスが終了します。

前提条件と不変条件

  • 入力 conn は有効な PGconn オブジェクトである必要があります。
  • PQconninfo を介して接続パラメータが取得可能である必要があります。

アルゴリズムまたは振る舞い

1. PQconninfo(conn) を呼び出し、現在の接続パラメータを取得します。 2. 取得したパラメータから、値が設定されているもののみを抽出し、keywords および vals 配列に格納します。 3. PQconnectdbParams を使用して、抽出したパラメータで新しい接続を試行します。 4. PQstatus を確認し、接続が CONNECTION_OK でない場合は pg_fatal を呼び出してエラーメッセージを出力し、プログラムを終了します。 5. 成功した場合、新しい接続オブジェクトを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/libpq_pipeline/libpq_pipeline.c 内の monitorConn = copy_connection(conn); （260行目）にて呼び出されています。
  • **呼び出し先**: PQconninfo, pg_malloc, PQconnectdbParams, PQstatus, pg_fatal, PQerrorMessage を呼び出します。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PQconninfoOption: 接続パラメータのキーワードと値のペアを保持する構造体。

失敗モードと境界条件

  • 接続パラメータの取得に失敗した場合や、新しい接続先への接続が確立できない場合、pg_fatal によりテストプログラム自体が異常終了します。

保守担当者向けメモ

  • 本関数はテスト用ユーティリティであり、接続パラメータを動的に複製して別セッションを確立するために使用されます。
  • pg_malloc を使用しているため、メモリ管理には注意してください。
  • 接続パラメータの配列構築時に NULL 終端を適切に処理しています。
