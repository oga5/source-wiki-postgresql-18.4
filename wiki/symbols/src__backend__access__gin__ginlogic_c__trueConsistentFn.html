trueConsistentFn

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginlogic.c:49

目的

GINインデックススキャンにおいて、検索条件が常に真（TRUE）であるとみなされるケース（例: GIN_SEARCH_MODE_EVERYTHING）に対して、整合性チェックの成功を強制するためのダミー関数として機能します。

シグネチャまたは形状

static bool trueConsistentFn(GinScanKey key)

入力と出力

  • **入力**: GinScanKey key（スキャンキーの状態を保持する構造体へのポインタ）
  • **出力**: bool 型の true を常に返します。

前提条件と不変条件

  • key 引数は有効な GinScanKey 構造体を指している必要があります。
  • 呼び出し後、key->recheckCurItem が false に設定されることが保証されます。

アルゴリズムまたは振る舞い

1. key->recheckCurItem を false に設定します。これは、このエントリに対して再チェック（Recheck）が不要であることを示します。 2. true を返して、整合性チェックが成功したことをスキャンエンジンに伝えます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginlogic.c 内の ginInitConsistentFunction において、key->boolConsistentFn に関数ポインタとして代入されます。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • GinScanKey: GINスキャン中の状態を保持し、本関数によって recheckCurItem フィールドが操作されます。

失敗モードと境界条件

  • 本関数自体は常に成功（true を返却）するため、関数内部での失敗モードはありません。

保守担当者向けメモ

  • 本関数は GIN_SEARCH_MODE_EVERYTHING モード時に使用されるプレースホルダーです。ロジックの変更は、GINスキャン全体の整合性判定フローに影響を与えるため注意してください。
