BlockRefTableBuffer

定義

  • 種別: struct
  • 定義箇所: src/common/blkreftable.c:171

目的

ブロック参照テーブルのシリアライズおよびデシリアライズ処理において、ディスク I/O を抽象化し、バッファリングと整合性チェック（CRC）を行うための状態保持構造体です。

シグネチャまたは形状

typedef struct BlockRefTableBuffer
{
	io_callback_fn io_callback;
	void	   *io_callback_arg;
	char		data[BUFSIZE];
	int			used;
	int			cursor;
	pg_crc32c	crc;
} BlockRefTableBuffer;

入力と出力

  • **入力**: io_callback を介して外部ストレージやストリームからデータを読み込み、または書き出します。
  • **出力**: data バッファ内のデータ、および crc による整合性検証結果。

前提条件と不変条件

  • data 配列のサイズは BUFSIZE に固定されています。
  • used はバッファ内の有効なデータ量、cursor は現在の読み書き位置を保持します。
  • 読み込み時は、BlockRefTableReader がこのバッファを使用して CRC を更新しながらデータを処理します。

アルゴリズムまたは振る舞い

  • **バッファリング**: io_callback を通じて BUFSIZE 単位でデータの読み書きを行います。
  • **整合性管理**: データの読み書き時に pg_crc32c を計算・更新し、ファイル形式の整合性を保証します。
  • **I/O 抽象化**: コールバック関数を使用することで、ファイルシステムだけでなくメモリバッファやネットワークストリームに対しても透過的に動作します。

呼び出し元と呼び出し先

  • **呼び出し元**: BlockRefTableReader および BlockRefTableWriter の内部で使用されます。
  • **呼び出し先**: BlockRefTableFlush（バッファの書き出し）、BlockRefTableWrite（データ書き込み）、BlockRefTableFileTerminate（終了処理）などの内部関数から操作されます。

関連データ構造

  • BlockRefTableReader: 本構造体をメンバとして保持し、読み込み状態を管理します。
  • BlockRefTableWriter: 本構造体を使用してシリアライズデータを出力します。

失敗モードと境界条件

  • バッファのオーバーフローや io_callback がエラーを返した場合の挙動については、本構造体単体では定義されておらず、呼び出し側のエラーハンドリングに依存します。
  • BUFSIZE を超えるデータ処理は、複数回のバッファフラッシュまたは読み込みによって行われます。

保守担当者向けメモ

  • io_callback の実装を変更することで、I/O 先を容易に切り替え可能です。
  • CRC 計算はデータ処理と同期して行われるため、バッファ操作関数を直接呼び出す際は CRC の整合性が崩れないよう注意してください。
