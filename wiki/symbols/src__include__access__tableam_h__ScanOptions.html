ScanOptions

定義

  • 種別: enum
  • 定義箇所: src/include/access/tableam.h:45

目的

テーブルスキャン操作の挙動を制御するためのビットマスク定義です。スキャンの種類、アクセス戦略、同期スキャン、ページモード、およびスナップショット管理の要否を指定するために使用されます。

シグネチャまたは形状

typedef enum ScanOptions
{
    SO_TYPE_SEQSCAN = 1 << 0,
    SO_TYPE_BITMAPSCAN = 1 << 1,
    SO_TYPE_SAMPLESCAN = 1 << 2,
    SO_TYPE_TIDSCAN = 1 << 3,
    SO_TYPE_TIDRANGESCAN = 1 << 4,
    SO_TYPE_ANALYZE = 1 << 5,
    SO_ALLOW_STRAT = 1 << 6,
    SO_ALLOW_SYNC = 1 << 7,
    SO_ALLOW_PAGEMODE = 1 << 8,
    SO_TEMP_SNAPSHOT = 1 << 9,
} ScanOptions;

入力と出力

  • **入力**: スキャン開始関数（table_beginscan 等）の引数としてビットマスク値を受け取ります。
  • **出力**: 該当しません（列挙型定義のため）。

前提条件と不変条件

  • SO_TYPE_* に分類される値は、スキャンタイプを識別するために使用され、通常は単一の指定が想定されます。
  • SO_ALLOW_* に分類される値は、ビット演算により複数組み合わせることが可能です。

アルゴリズムまたは振る舞い

  • **スキャンタイプ指定**: SO_TYPE_SEQSCAN から SO_TYPE_ANALYZE までのフラグにより、実行するスキャンの種類を決定します。
  • **動作制御**:
  • SO_ALLOW_STRAT: アクセス戦略の使用を許可/禁止します。
  • SO_ALLOW_SYNC: 同期スキャン（SyncScan）ロジックへの位置報告を行うかを制御します。
  • SO_ALLOW_PAGEMODE: ページ単位での可視性検証を行うかを制御します。
  • **リソース管理**: SO_TEMP_SNAPSHOT が指定された場合、スキャン終了時（table_endscan）にスナップショットの登録解除が必要であることを示します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/relscan.h 等で参照されています。具体的な関数呼び出しの連鎖については、提供されたソース情報からは特定できません。
  • **呼び出し先**: 該当しません。

関連データ構造

  • TableScanDesc: スキャン状態を保持する構造体であり、このフラグを内部的に保持・参照します。

失敗モードと境界条件

  • 複数の SO_TYPE_* フラグを同時に設定した場合の動作は、各テーブルアクセス手法（AM）の実装に依存します。

保守担当者向けメモ

  • 新しいスキャンタイプやオプションを追加する場合は、既存のビットマスクと重複しないように注意してください。
  • table_beginscan 系の関数を呼び出す際は、このフラグが正しく設定されているか確認してください。
