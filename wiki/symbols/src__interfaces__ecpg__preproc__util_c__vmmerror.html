vmmerror

定義

  • 種別: static function
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:15

目的

ECPGプリプロセッサにおけるエラーおよび警告メッセージを標準エラー出力へ整形して出力し、エラー発生時には終了コードを記録する内部関数です。

シグネチャまたは形状

static void vmmerror(int error_code, enum errortype type, const char *error, va_list ap);

入力と出力

  • **入力**:
  • error_code: エラー発生時に設定される整数値。
  • type: ET_WARNING または ET_ERROR を指定する列挙型。
  • error: フォーマット文字列（国際化対応済み）。
  • ap: 可変長引数リスト。
  • **出力**: なし（戻り値は void）。副作用として標準エラー出力への書き込みおよびグローバル変数 ret_value の更新を行います。

前提条件と不変条件

  • input_filename および base_yylineno が有効な値として設定されている必要があります。
  • error 引数は gettext 等の国際化マクロ _() を通して処理されます。

アルゴリズムまたは振る舞い

1. input_filename と base_yylineno を用いてファイル名と行番号を標準エラー出力に出力します。 2. type に応じて "WARNING: " または "ERROR: " という接頭辞を出力します。 3. vfprintf を使用して、渡されたフォーマット文字列と引数リストを標準エラー出力に書き出します。 4. type が ET_ERROR の場合、グローバル変数 ret_value に error_code を代入します。

呼び出し元と呼び出し先

  • **呼び出し元**: mmerror 関数（src/interfaces/ecpg/preproc/util.c 内）から呼び出されます。
  • **呼び出し先**: fprintf, vfprintf などの標準ライブラリ関数。

関連データ構造

  • enum errortype: エラーの種類を定義する列挙型。
  • ret_value: プリプロセッサの終了ステータスを保持するグローバル変数。

失敗モードと境界条件

  • type が ET_WARNING の場合、ret_value の更新は行われません。
  • type が ET_ERROR の場合のみ、ret_value が更新されます。

保守担当者向けメモ

  • 本関数は mmerror を介して呼び出される内部的なユーティリティです。直接呼び出すのではなく、mmerror を使用してください。
  • input_filename や base_yylineno は preproc_extern.h で定義されたグローバル変数に依存しています。
