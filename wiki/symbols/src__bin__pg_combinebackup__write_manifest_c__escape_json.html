escape_json

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/write_manifest.c:194

目的

与えられた文字列を JSON 形式の文字列リテラルとしてエスケープし、StringInfo バッファに追加する。

シグネチャまたは形状

static void escape_json(StringInfo buf, const char *str)

入力と出力

  • **入力**:
  • buf: 文字列を追加する対象の StringInfo バッファ。
  • str: エスケープ対象のヌル終端文字列。
  • **出力**: なし（buf の内容が更新される）。

前提条件と不変条件

  • buf は初期化済みであること。
  • str は有効なヌル終端文字列であること。
  • 出力される文字列は、開始と終了に必ずダブルクォーテーション（"）で囲まれる。

アルゴリズムまたは振る舞い

1. バッファに開始の " を追加する。 2. str の各文字を走査し、以下の制御文字および特殊文字をエスケープする。

  • \b, \f, \n, \r, \t は対応するエスケープシーケンスに変換。
  • " は \" に、\ は \\ に変換。
  • ASCII 制御文字（' ' 未満）は \u00xx 形式の Unicode エスケープに変換。
  • 上記以外の文字はそのままバッファに追加。

    • 3. バッファに終了の " を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコードのコンテキスト内では、本関数を直接呼び出している箇所は特定されていない。
  • **呼び出し先**: appendStringInfoCharMacro および appendStringInfoString を使用してバッファ操作を行う。
  • **注意**: ツールが検出した src/backend/... 等の呼び出し元は、本関数と同名の別実装である可能性が高く、本モジュール内での直接的な呼び出し関係は未確認である。

関連データ構造

  • StringInfo: PostgreSQL の可変長文字列バッファ構造体。

失敗モードと境界条件

  • str が空文字列の場合、"" がバッファに追加される。
  • 制御文字を含む文字列に対しては、JSON 仕様に準拠したエスケープが行われる。

保守担当者向けメモ

  • 本関数は pg_combinebackup ツール内で JSON 形式のバックアップマニフェストを構築するために使用される。
  • バックエンド側の同名関数と類似しているが、フロントエンドツールとして動作するため StringInfo を介したバッファリングを行う設計となっている。
