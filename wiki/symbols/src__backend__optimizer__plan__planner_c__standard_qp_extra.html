standard_qp_extra

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/planner.c:123

目的

クエリプランニングの過程で、grouping_planner 等の関数間で共有される追加的なコンテキスト情報を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	List	   *activeWindows;	/* active windows, if any */
	grouping_sets_data *gset_data;	/* grouping sets data, if any */
	SetOperationStmt *setop;	/* parent set operation or NULL */
} standard_qp_extra;

入力と出力

本構造体はデータ保持用であり、関数ではありません。プランニング処理中に standard_qp_extra 型のインスタンスが生成され、プランニングの各段階で必要なメタデータ（ウィンドウ関数、グループ化セット、集合演算の親情報）を格納するために使用されます。

前提条件と不変条件

  • 各フィールドは、該当する機能（ウィンドウ関数、グループ化セット、集合演算）がクエリ内に存在しない場合、NULL または空のリストとして初期化されることが想定されます。

アルゴリズムまたは振る舞い

  • 本構造体は、プランニングの制御フローにおいて、特定のクエリノードに関連する補助情報を一括して受け渡すためのコンテナとして機能します。
  • grouping_planner 内でインスタンス化され、サブクエリや集約処理の文脈を維持するために参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c 内の grouping_planner 関数がこの構造体をインスタンス化し、利用しています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • grouping_sets_data: グループ化セットに関する詳細情報を保持します。
  • SetOperationStmt: 集合演算（UNION, INTERSECT, EXCEPT）のノード定義です。
  • List: PostgreSQL の汎用リスト構造であり、activeWindows の保持に使用されます。

失敗モードと境界条件

  • 構造体内のポインタが適切に初期化されていない場合、プランニング処理中にヌルポインタ参照が発生する可能性があります。
  • 集合演算がネストしている場合、setop フィールドが正しく親ノードを指している必要があります。

保守担当者向けメモ

  • 本構造体は planner.c 内のプランニングロジックに密結合しています。フィールドを追加・変更する場合は、grouping_planner における初期化および参照箇所をすべて確認してください。
