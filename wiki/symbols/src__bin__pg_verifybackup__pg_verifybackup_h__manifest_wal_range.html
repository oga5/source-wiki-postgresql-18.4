manifest_wal_range

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.h:63

目的

バックアップマニフェストに含まれる WAL（Write-Ahead Log）の範囲情報を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct manifest_wal_range
{
	TimeLineID	tli;
	XLogRecPtr	start_lsn;
	XLogRecPtr	end_lsn;
	struct manifest_wal_range *next;
	struct manifest_wal_range *prev;
} manifest_wal_range;

入力と出力

  • **入力**: 構造体メンバとしてタイムライン ID (tli)、開始 LSN (start_lsn)、終了 LSN (end_lsn) を保持します。
  • **出力**: 双方向リストを構成するためのポインタ (next, prev) を保持し、リスト構造として管理されます。

前提条件と不変条件

  • next および prev ポインタにより、WAL 範囲のリストを双方向に辿ることが可能です。
  • 各インスタンスは、特定のタイムラインにおける連続した WAL セグメントの範囲を定義します。

アルゴリズムまたは振る舞い

  • 本構造体は、バックアップマニフェスト内の WAL 範囲をメモリ上でリストとして管理するために使用されます。
  • リストの先頭および末尾は、上位の manifest_data 構造体などで保持されることが想定されています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_verifybackup/pg_verifybackup.c および src/bin/pg_combinebackup/load_manifest.c でインスタンスの生成や参照が行われています。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • manifest_data: マニフェスト全体の情報を保持するルート構造体であり、本構造体のリストを管理します。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc 等の失敗）に本構造体の生成が失敗する可能性があります。
  • リストの終端では next が NULL に、先頭では prev が NULL になることが想定されます。

保守担当者向けメモ

  • 本構造体は pg_verifybackup だけでなく pg_combinebackup でも利用されています。変更を加える際は両方のツールへの影響を考慮してください。
  • リスト操作を行う際は、next および prev ポインタの整合性が維持されていることを確認してください。
