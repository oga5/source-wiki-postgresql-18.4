CIRCLE

定義

  • 種別: struct
  • 定義箇所: src/include/utils/geo_decls.h:162

目的

PostgreSQLにおける幾何データ型「円」を表現するためのデータ構造です。SQLレベルの circle 型とC言語のメモリ表現を橋渡しします。

シグネチャまたは形状

typedef struct
{
	Point		center;
	float8		radius;
} CIRCLE;

入力と出力

  • **入力**: Point 型の座標（中心点）と float8 型の半径。
  • **出力**: 構造体としてメモリ上に保持される円の幾何情報。

前提条件と不変条件

  • center は Point 構造体（x, y座標）で定義されます。
  • radius は float8（倍精度浮動小数点数）で定義されます。
  • 幾何計算において、本構造体は math.h の関数や geo_decls.h で定義される浮動小数点比較関数（FPeq 等）と組み合わせて使用されます。

アルゴリズムまたは振る舞い

  • PostgreSQLの関数マネージャ（fmgr）を介して Datum から変換される際、PG_GETARG_CIRCLE_P マクロ等を用いてポインタとして取得されます。
  • src/backend/utils/adt/geo_ops.c 内の関数群により、面積計算（circle_ar）、多角形から円への変換（poly_to_circle）、および距離計算（dist_cpoly_internal）などの操作が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/gist/gistproc.c: GiSTインデックス処理におけるキーの取得。
  • src/backend/utils/adt/geo_ops.c: 幾何演算の実装関数内での引数取得や一時変数としての利用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。操作は geo_ops.c 等の外部関数によって実行されます。

関連データ構造

  • Point: 円の中心座標を保持するために使用。
  • POLYGON: poly_to_circle や dist_cpoly_internal 等で相互変換や距離計算の対象として関連。

失敗モードと境界条件

  • **浮動小数点演算**: 比較関数（FPeq 等）はNaNを考慮しておらず、NaNが含まれる場合は常に FALSE を返します。
  • **数値精度**: 本構造体を用いた計算ルーチンは数値解析の専門家によるものではないため、推移律を満たさない可能性がある点に注意が必要です。

保守担当者向けメモ

  • 本構造体は固定長データとして扱われます。
  • geo_ops.c 内で palloc を用いて動的にメモリ確保されるケースがあるため、メモリ管理には注意してください。
