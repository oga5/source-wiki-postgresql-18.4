pg_get_replica_identity_index

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/misc.c:1100

目的

指定されたリレーション（テーブル）に対して設定されているレプリカアイデンティティインデックスのOIDを取得します。

シグネチャまたは形状

Datum pg_get_replica_identity_index(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介してリレーションのOIDを受け取ります。
  • **出力**: インデックスが存在する場合はそのOIDを PG_RETURN_OID で返し、存在しない場合は PG_RETURN_NULL を返します。

前提条件と不変条件

  • 入力されたOIDに対応するリレーションが存在する必要があります。
  • table_open 呼び出し時に AccessShareLock を取得し、処理終了後に解放します。

アルゴリズムまたは振る舞い

1. table_open を使用して、指定されたOIDのリレーションを AccessShareLock モードで開きます。 2. RelationGetReplicaIndex を呼び出し、当該リレーションのレプリカアイデンティティとして設定されているインデックスのOIDを取得します。 3. table_close を使用してリレーションを閉じます。 4. 取得したOIDが有効（OidIsValid）であればその値を返し、無効であればNULLを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/tablesync.c:985 でクエリの一部として参照されています。
  • **呼び出し先**:
  • table_open / table_close (テーブルアクセス)
  • RelationGetReplicaIndex (インデックスOID取得)
  • OidIsValid (OID検証)

関連データ構造

  • Relation: リレーション記述子。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • 指定されたOIDがシステム内に存在しない場合、table_open がエラーを発生させます。
  • レプリカアイデンティティが設定されていない場合、RelationGetReplicaIndex は InvalidOid を返し、本関数はNULLを返します。

保守担当者向けメモ

  • 本関数はSQLインターフェースとして公開されています。
  • RelationGetReplicaIndex の挙動変更は、本関数の戻り値に直接影響します。
