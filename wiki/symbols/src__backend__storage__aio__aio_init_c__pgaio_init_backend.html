pgaio_init_backend

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_init.c:217

目的

各バックエンドプロセスにおいて、非同期 I/O (AIO) サブシステムを利用するためのコンテキストを初期化し、共有メモリ上の状態と紐付けます。

シグネチャまたは形状

void pgaio_init_backend(void)

入力と出力

  • 入力: なし（グローバル変数 MyProcNumber および pgaio_ctl を参照）
  • 出力: なし（副作用として pgaio_my_backend を設定）

前提条件と不変条件

  • pgaio_my_backend が既に初期化されていないこと（Assert による検証）。
  • MyBackendType が B_IO_WORKER である場合、本関数は何もせず復帰する。
  • MyProc が存在し、かつ MyProcNumber が有効な範囲内（AioProcs() 未満）であること。

アルゴリズムまたは振る舞い

1. 二重初期化を防止するためのアサーションを実行。 2. B_IO_WORKER 型のバックエンドであれば処理を終了。 3. MyProc の有効性を確認し、無効な場合はエラーをスロー。 4. pgaio_ctl->backend_state から現在のプロセス番号に対応するバックエンド状態を取得し、pgaio_my_backend に割り当て。 5. pgaio_method_ops->init_backend が定義されている場合、そのメソッド固有の初期化関数を呼び出し。 6. プロセス終了時に pgaio_shutdown が実行されるよう before_shmem_exit に登録。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/postinit.c:639
  • 呼び出し先:
  • pgaio_method_ops->init_backend() (メソッド定義に依存)
  • before_shmem_exit()

関連データ構造

  • pgaio_ctl: 共有メモリ上の AIO 制御構造体。
  • pgaio_my_backend: 現在のバックエンドの AIO 状態を保持するポインタ。
  • PgAioBackend: backend_state 配列の各要素の型。

失敗モードと境界条件

  • MyProc == NULL または MyProcNumber >= AioProcs() の場合、elog(ERROR) により異常終了します。
  • pgaio_my_backend が既に設定されている状態で呼び出された場合、アサーション違反となります。

保守担当者向けメモ

  • 本関数はバックエンド起動時の重要なセットアップ手順です。共有メモリレイアウトの変更や AIO メソッドの追加時には、本関数内での初期化順序に注意してください。
