PrepareSortSupportFromGistIndexRel

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/sortsupport.c:184

目的

GiSTインデックスリレーションに基づき、SortSupportData 構造体を初期化してソート処理を準備します。

シグネチャまたは形状

void PrepareSortSupportFromGistIndexRel(Relation indexRel, SortSupport ssup)

入力と出力

  • **入力**:
  • indexRel: 対象となるGiSTインデックスのリレーション情報。
  • ssup: 初期化対象の SortSupportData 構造体（ssup_attno が設定されている必要がある）。
  • **出力**: なし（ssup 構造体のフィールドが更新される）。

前提条件と不変条件

  • ssup->comparator が NULL であること（Assert により検証）。
  • indexRel がGiSTアクセスメソッドを使用していること。

アルゴリズムまたは振る舞い

1. indexRel から対象属性の演算子ファミリー（opfamily）と入力型を特定します。 2. ssup->ssup_reverse を false に設定します。 3. get_opfamily_proc を使用して、当該演算子ファミリーから GIST_SORTSUPPORT_PROC を検索します。 4. 該当するサポート関数が見つからない場合はエラーを発生させます。 5. 見つかった関数を OidFunctionCall1 で実行し、ssup 構造体の初期化を完了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/sort/tuplesortvariants.c:546
  • **呼び出し先**:
  • get_opfamily_proc (PostgreSQL内部関数)
  • OidFunctionCall1 (PostgreSQL内部関数)
  • elog (エラー報告用マクロ)

関連データ構造

  • Relation: インデックスのメタデータ保持に使用。
  • SortSupportData (SortSupport): ソートの比較ロジックや状態を保持する中心的な構造体。

失敗モードと境界条件

  • indexRel がGiSTインデックスではない場合、elog(ERROR) を発生させます。
  • 指定された演算子ファミリーに GIST_SORTSUPPORT_PROC が定義されていない場合、elog(ERROR) を発生させます。

保守担当者向けメモ

  • 本関数はB-treeとは異なり、古いスタイルの比較関数（BTORDER_PROC）へのフォールバックをサポートしていません。GiSTインデックスでは GIST_SORTSUPPORT_PROC の実装が必須です。
