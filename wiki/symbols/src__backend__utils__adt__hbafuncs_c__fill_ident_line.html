fill_ident_line

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/hbafuncs.c:486

目的

pg_ident_file_mappings ビューの1行分を構築し、結果保持用の Tuplestore に格納します。

シグネチャまたは形状

static void fill_ident_line(Tuplestorestate *tuple_store, TupleDesc tupdesc,
                            int map_number, char *filename, int lineno, 
                            IdentLine *ident, const char *err_msg)

入力と出力

  • **入力**:
  • tuple_store: 結果を蓄積する Tuplestorestate。
  • tupdesc: 行の構造を定義する TupleDesc。
  • map_number: マッピング番号。
  • filename: 設定ファイル名。
  • lineno: 行番号。
  • ident: パース済みの IdentLine 構造体（パース失敗時は NULL）。
  • err_msg: パースエラー発生時のエラーメッセージ（正常時は NULL）。
  • **出力**: なし（tuple_store への書き込みのみ）。

前提条件と不変条件

  • tupdesc->natts が NUM_PG_IDENT_FILE_MAPPINGS_ATTS と一致している必要があります（Assert で検証）。

アルゴリズムまたは振る舞い

1. values 配列と nulls 配列を初期化します。 2. err_msg が存在する場合、map_number を NULL に設定し、エラーメッセージを最終カラムに格納します。 3. err_msg が存在しない場合、map_number、filename、lineno を設定します。 4. ident が有効な場合、usermap、system_user、pg_user を Datum に変換して格納します。 5. ident が NULL の場合、関連するフィールドを NULL に設定します。 6. heap_form_tuple を使用してタプルを生成し、tuplestore_puttuple で tuple_store に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/hbafuncs.c:577（fill_ident_view 内のループから呼び出されます）。
  • **呼び出し先**: heap_form_tuple, tuplestore_puttuple, CStringGetTextDatum, Int32GetDatum 等のPostgreSQL内部API。

関連データ構造

  • Tuplestorestate: 結果セットを保持する一時ストレージ。
  • IdentLine: pg_ident.conf の各行のパース結果を保持する構造体。
  • TupleDesc: タプルの属性定義。

失敗モードと境界条件

  • ident が NULL の場合、パース結果に関連するフィールド（usermap, system_user, pg_user）が NULL として出力されます。
  • err_msg が提供された場合、map_number は NULL となり、エラー内容がビューの末尾カラムに反映されます。

保守担当者向けメモ

  • 本関数は pg_ident_file_mappings ビューのスキーマ定義と密接に結合しています。ビューの列定義を変更する場合は、NUM_PG_IDENT_FILE_MAPPINGS_ATTS 定数および本関数のインデックス操作を同期させる必要があります。
