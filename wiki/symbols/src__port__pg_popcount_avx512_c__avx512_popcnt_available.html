avx512_popcnt_available

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_avx512.c:77

目的

実行環境のCPUがAVX-512命令セット（VPOPCNTDQおよびBW拡張）をサポートしているかを判定します。

シグネチャまたは形状

static inline bool avx512_popcnt_available(void)

入力と出力

  • 入力: なし
  • 出力: bool 型。AVX-512の対象命令セットが利用可能な場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • USE_AVX512_POPCNT_WITH_RUNTIME_CHECK マクロが定義されている環境でのみ有効です。
  • CPUID命令が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. CPUID 命令（__get_cpuid_count または __cpuidex）を呼び出し、機能フラグを取得します。 2. leaf 7, subleaf 0 の結果から以下のビットを確認します。

  • EBX レジスタの第30ビット（AVX512-BW）
  • ECX レジスタの第14ビット（AVX512-VPOPCNTDQ）

    • 3. 両方のビットがセットされている場合にのみ true を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/pg_popcount_avx512.c 内の他の関数から実行時チェックとして参照されています。
  • 呼び出し先: __get_cpuid_count または __cpuidex（コンパイル環境に依存）。

関連データ構造

  • unsigned int exx[4]: CPUIDの結果を格納する一時的な配列。

失敗モードと境界条件

  • CPUID命令が利用できない環境ではコンパイルエラー（#error）が発生します。
  • 実行環境がAVX-512をサポートしていない場合、本関数は正しく false を返します。

保守担当者向けメモ

  • 本関数はインライン化を前提としています。
  • 新しいCPUアーキテクチャでAVX-512の判定基準が変更された場合は、ビットマスクの値を更新する必要があります。
