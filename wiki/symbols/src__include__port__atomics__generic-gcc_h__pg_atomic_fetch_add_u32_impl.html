pg_atomic_fetch_add_u32_impl

定義

  • 種別: function
  • 定義箇所: src/include/port/atomics/generic-gcc.h:206

目的

pg_atomic_uint32 型の変数に対して、指定した値を原子的に加算し、加算前の値を返します。

シグネチャまたは形状

static inline uint32 pg_atomic_fetch_add_u32_impl(volatile pg_atomic_uint32 *ptr, int32 add_)

入力と出力

  • **入力**:
  • ptr: 原子操作対象となる pg_atomic_uint32 構造体へのポインタ（volatile 修飾あり）。
  • add_: 加算する値（int32）。
  • **出力**:
  • 加算操作が行われる直前の ptr->value の値（uint32）。

前提条件と不変条件

  • ptr は有効な pg_atomic_uint32 構造体を指している必要があります。
  • 本関数は atomics.h を経由して呼び出されることを前提としています。

アルゴリズムまたは振る舞い

GCCの組み込み関数 __sync_fetch_and_add を利用して、メモリ上の値を原子的に更新します。 1. ptr->value の現在値を読み取り、それに add_ を加算した結果をメモリに書き戻します。 2. この一連の操作は不可分（atomic）として実行されます。 3. 操作前の値を戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/atomics.h:369 および src/include/port/atomics/generic.h 内の複数の箇所から呼び出されています。
  • **呼び出し先**: コンパイラ組み込み関数 __sync_fetch_and_add を直接呼び出します。

関連データ構造

  • pg_atomic_uint32: 原子操作対象の値を保持する構造体。

失敗モードと境界条件

  • ptr が NULL の場合、未定義の動作を引き起こします。
  • 整数オーバーフローが発生した場合、標準的な符号付き整数演算のラップアラウンド挙動に従います。

保守担当者向けメモ

  • 本関数は __sync 系APIを使用しています。より新しい __atomic API を利用可能な環境での最適化については、generic-gcc.h 内の他の条件分岐やコンパイラフラグの定義状況を確認してください。
  • アーキテクチャ固有の実装（arch-x86.h 等）が存在する場合、そちらが優先される可能性があります。
