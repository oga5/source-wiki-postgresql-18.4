pg_base64_decode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/encode.c:334

目的

Base64 エンコードされた文字列をバイナリデータへデコードします。

シグネチャまたは形状

static uint64 pg_base64_decode(const char *src, size_t len, char *dst)

入力と出力

  • **入力**:
  • src: デコード対象の Base64 文字列。
  • len: 入力文字列の長さ。
  • dst: デコード結果を格納するバッファ（十分なサイズが確保されている必要がある）。
  • **出力**:
  • 戻り値: デコードされたバイナリデータのバイト数（uint64）。

前提条件と不変条件

  • dst バッファは、デコード後のデータサイズを保持するのに十分な領域を確保している必要があります。
  • 入力文字列に含まれる空白文字（スペース、タブ、改行、復帰）は無視されます。

アルゴリズムまたは振る舞い

1. 入力文字列を 1 文字ずつ走査します。 2. 空白文字はスキップし、=（パディング）は終了シーケンスとして処理します。 3. Base64 文字を b64lookup テーブルを用いて 6 ビットの値に変換し、内部バッファ buf に蓄積します。 4. 4 文字分（24 ビット）蓄積されるごとに、3 バイトのバイナリデータとして dst に書き出します。 5. パディングの有無に応じて、書き出すバイト数を調整します。 6. 処理終了時に pos（蓄積された文字数）が 0 でない場合、不正なシーケンスとしてエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/encode.c 内のエンコーディング管理テーブル（pg_base64_dec_len 等と併用）。
  • contrib/pgcrypto/pgp-armor.c（直接的な呼び出しが確認されています）。
  • **呼び出し先**:
  • ereport: エラー発生時にトランザクションを中断するために使用。
  • pg_mblen_range: エラーメッセージ生成時に使用。

関連データ構造

  • b64lookup: Base64 文字から値への変換テーブル（ソースコード内で参照）。

失敗モードと境界条件

  • **不正な文字**: Base64 範囲外の文字が含まれる場合、ereport(ERROR, ...) を発行します。

保守担当者向けメモ

  • 本関数は pg_encoding インターフェースの一部として利用されることを想定しています。
  • contrib/pgcrypto からの呼び出しにおいて、引数の型が uint8 * となっている箇所がありますが、これは char * へのキャストを伴う利用です。
