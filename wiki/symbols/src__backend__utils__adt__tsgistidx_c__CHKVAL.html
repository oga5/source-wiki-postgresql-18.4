CHKVAL

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsgistidx.c:266

目的

全文検索のインデックス照合処理において、検索対象となる配列の範囲（開始位置と終了位置）を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	int32	   *arrb;
	int32	   *arre;
} CHKVAL;

入力と出力

  • arrb: 検索対象配列の開始ポインタ。
  • arre: 検索対象配列の終了ポインタ。

前提条件と不変条件

  • arrb は配列の先頭を指し、arre は配列の末尾（または終端）を指す必要があります。
  • 呼び出し側で CHKVAL 型にキャストして使用されることが前提となっています。

アルゴリズムまたは振る舞い

本構造体は、TS_execute 等の検索ロジックにおいて、インデックス内の配列データに対するバイナリサーチや走査を行う際のコンテキストとして利用されます。tsgistidx.c 内では、checkcondition_arr 関数がこの構造体を経由して配列データにアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsgistidx.c 内の checkcondition_arr 関数が、CHKVAL 型にキャストされたポインタから arrb および arre を参照しています。
  • **その他**: contrib/intarray/_int_bool.c および contrib/ltree/ltxtquery_op.c においても同名の構造体が定義・利用されており、同様のパターンで配列やノードの走査に使用されています。

関連データ構造

  • SignTSVector: tsgistidx.c で定義される GiST インデックスのキー構造体。

失敗モードと境界条件

  • arrb または arre が NULL の状態でアクセスした場合、セグメンテーションフォールトが発生します。
  • arrb > arre となるような不正な範囲が渡された場合、走査ロジックが期待通りに動作しません。

保守担当者向けメモ

  • 本構造体は tsgistidx.c 以外（contrib モジュール等）でも同名で定義されています。修正を行う際は、他のモジュールへの影響範囲を考慮してください。
  • tsgistidx.c における利用は、主に tsvector のインデックス検索時の配列走査に限定されています。
