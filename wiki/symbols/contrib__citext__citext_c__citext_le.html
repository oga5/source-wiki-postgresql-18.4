citext_le

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:265

目的

2つの citext 型の値を受け取り、左辺が右辺以下であるか（<=）を判定する比較関数です。

シグネチャまたは形状

Datum citext_le(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの text 型（citext として扱われる）引数を受け取ります。
  • **出力**: PG_RETURN_BOOL を介して bool 型の判定結果を返します。

前提条件と不変条件

  • 入力引数は PG_GETARG_TEXT_PP を使用して取得されるため、TOAST されたデータが自動的に展開されます。
  • 処理終了時には PG_FREE_IF_COPY を呼び出し、必要に応じてメモリを解放する必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して、2つの入力引数 left と right を取得します。 2. citextcmp 関数を呼び出し、現在の照合順序（PG_GET_COLLATION()）に基づいて比較を行います。 3. citextcmp の戻り値が 0 以下であれば true、そうでなければ false を result に格納します。 4. PG_FREE_IF_COPY を使用して、入力引数のメモリを適切に解放します。 5. PG_RETURN_BOOL を使用して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(citext_le) マクロにより、PostgreSQL の関数マネージャから呼び出されるインターフェースとして登録されています。
  • **呼び出し先**: citextcmp（比較ロジックの実行）、PG_GETARG_TEXT_PP、PG_GET_COLLATION、PG_FREE_IF_COPY、PG_RETURN_BOOL。

関連データ構造

  • text: PostgreSQL の可変長文字列型。citext はこの型をベースに実装されています。

失敗モードと境界条件

  • 入力引数が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で NULL 処理が行われることが想定されます。
  • citextcmp の内部で照合順序に基づいた比較が行われるため、照合順序の設定によっては予期せぬ比較結果となる可能性があります。

保守担当者向けメモ

  • 本関数は citext 型の比較演算子として SQL レベルで利用されます。
  • citextcmp は大文字・小文字を区別しない比較を行うための共通関数であり、変更時は他の比較関数（citext_lt 等）への影響を考慮してください。
