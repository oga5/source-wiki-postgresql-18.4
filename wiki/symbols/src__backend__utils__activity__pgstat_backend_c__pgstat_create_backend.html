pgstat_create_backend

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_backend.c:321

目的

特定のバックエンドプロセスに関連する統計情報を初期化し、共有メモリ上の統計エントリおよびプロセスローカルな統計バッファをリセットする。

シグネチャまたは形状

void pgstat_create_backend(ProcNumber procnum)

入力と出力

  • **入力**: procnum (ProcNumber) - 初期化対象のプロセス番号。
  • **出力**: なし。副作用として共有メモリおよび静的変数が更新される。

前提条件と不変条件

  • 呼び出し時に pgstat_get_entry_ref_locked を介して共有メモリ上の統計エントリがロックされる。
  • プロセスローカルな統計バッファ PendingBackendStats は、メモリ割り当てを行わない静的領域に配置されている必要がある。

アルゴリズムまたは振る舞い

1. pgstat_get_entry_ref_locked を呼び出し、指定された procnum に対応する共有メモリ上のバックエンド統計エントリを取得・ロックする。 2. 共有メモリ上の統計構造体 (shstatent->stats) を memset で 0 クリアする（以前のプロセスが使用していた古い統計情報を破棄）。 3. pgstat_unlock_entry で共有メモリのロックを解除する。 4. プロセスローカルな統計バッファ PendingBackendStats を MemSet で 0 クリアする。 5. I/O 統計フラグ backend_has_iostats を false に設定する。 6. WAL 統計の差分計算用基準値 prevBackendWalUsage を現在の pgWalUsage で初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/backend_status.c:498 (MyProcNumber を引数として呼び出し)
  • **呼び出し先**:
  • pgstat_get_entry_ref_locked
  • pgstat_unlock_entry
  • memset / MemSet

関連データ構造

  • PgStatShared_Backend: 共有メモリ上に保持されるバックエンド統計構造体。
  • PendingBackendStats: プロセスローカルな統計保持用構造体。
  • pgWalUsage: WAL 使用量計測用のグローバル変数。
  • prevBackendWalUsage: WAL 統計の差分計算に使用される静的変数。

失敗モードと境界条件

  • 共有メモリが初期化されていない、または procnum が無効な場合、pgstat_get_entry_ref_locked 内でエラーが発生する可能性がある。
  • 以前のプロセスが同じ procnum を使用していた場合、共有メモリ上の統計は本関数によって強制的にリセットされる。

保守担当者向けメモ

  • 本関数はプロセス起動時に実行されることを想定している。
  • prevBackendWalUsage の初期化は、以降の pgstat_backend_flush_cb における差分更新の正確性を担保するために不可欠である。
