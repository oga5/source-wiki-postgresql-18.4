pg_index_has_property

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/amutils.c:420

目的

インデックスのプロパティ（例: orderable, unique 等）を SQL レベルで照会するためのインターフェースを提供します。

シグネチャまたは形状

Datum pg_index_has_property(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_OID(0): 対象インデックスの OID (relid)。
  • PG_GETARG_TEXT_PP(1): 照会するプロパティ名を表す文字列。
  • **出力**:
  • Datum: プロパティの有無を示す真偽値（bool）。

前提条件と不変条件

  • 入力された relid は pg_class に存在する有効なインデックスである必要があります。
  • プロパティ名は IndexAMProperty 列挙型および am_propnames 配列で定義された既知の名称である必要があります。

アルゴリズムまたは振る舞い

1. 第1引数からインデックスの OID を取得します。 2. 第2引数の text 型を C 文字列に変換します。 3. 内部関数 indexam_property を呼び出し、以下の引数を渡して処理を委譲します。

  • fcinfo: 関数呼び出し情報。
  • propname: 変換済みのプロパティ名。
  • InvalidOid: 列番号を指定しない（インデックス全体または AM レベルの照会）。
  • relid: 対象インデックスの OID。
  • 0: 列番号のデフォルト値。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（pg_index_has_property SQL 関数）。
  • **呼び出し先**: indexam_property（内部関数）。

関連データ構造

  • IndexAmRoutine: アクセスメソッドの能力を定義する構造体。
  • IndexAMProperty: 照会可能なプロパティを定義する列挙型。

失敗モードと境界条件

  • 無効なインデックス OID が指定された場合、システムカタログ参照時にエラーが発生する可能性があります。
  • 未定義のプロパティ名が指定された場合、indexam_property 内でエラーがスローされます。

保守担当者向けメモ

  • 本関数は pg_index_column_has_property と異なり、列番号を 0 として固定的に呼び出します。
  • プロパティの判定ロジックを変更する場合は、indexam_property を参照してください。
