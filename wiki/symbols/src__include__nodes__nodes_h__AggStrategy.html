AggStrategy

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/nodes.h:358

目的

クエリ実行時における集約（Aggregation）処理の実行戦略を定義する列挙型です。

シグネチャまたは形状

typedef enum AggStrategy
{
    AGG_PLAIN,
    AGG_SORTED,
    AGG_HASHED,
    AGG_MIXED,
} AggStrategy;

入力と出力

  • 本シンボル自体は列挙型であり、プランナーやエグゼキュータにおいて集約ノードの動作モードを決定するフラグとして使用されます。

前提条件と不変条件

  • AGG_SORTED を選択する場合、入力データはグループ化キーに基づいてソート済みである必要があります。
  • AGG_HASHED を選択する場合、内部ハッシュテーブルを使用してグループ化が行われます。

アルゴリズムまたは振る舞い

  • AGG_PLAIN: 全入力行に対して単一の集約計算を行います。
  • AGG_SORTED: 入力行がソートされていることを前提に、グループ単位で集約を行います。
  • AGG_HASHED: ハッシュテーブルを用いてグループ単位で集約を行います。
  • AGG_MIXED: ハッシュとソートの両方の手法を組み合わせて集約を行います。

  • 呼び出し先: 情報なし。

関連データ構造

失敗モードと境界条件

  • 実行時に選択された戦略が入力データの特性（ソート順序の欠如など）と一致しない場合、誤った集約結果や実行時エラーが発生する可能性があります。

保守担当者向けメモ

  • 本列挙型の順序や値の変更は、プランナーおよびエグゼキュータのロジックに直接影響するため注意が必要です。
  • 新しい戦略を追加する場合は、関連する costsize.c や createplan.c 等のロジックも併せて更新してください。
