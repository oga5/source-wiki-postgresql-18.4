gimme_pool_size

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_main.c:319

目的

遺伝的アルゴリズム（GEQO）で使用する個体群サイズ（pool size）を、クエリ内のリレーション数に基づいて動的に決定する。

シグネチャまたは形状

static int gimme_pool_size(int nr_rel)

入力と出力

  • **入力**: nr_rel (int) - クエリに含まれるリレーションの数。
  • **出力**: (int) - 決定された個体群サイズ。

前提条件と不変条件

  • グローバル変数 Geqo_pool_size が 2 以上の場合、その値が優先的に返される。
  • Geqo_effort は個体群サイズの範囲（最小 10〜100、最大 50〜500）を決定する係数として使用される。

アルゴリズムまたは振る舞い

1. Geqo_pool_size が 2 以上であればその値を返す。 2. それ以外の場合、2^(nr_rel + 1) を計算し、基本サイズとする。 3. 50 * Geqo_effort を上限（maxsize）とし、計算値がこれを超える場合は上限値を返す。 4. 10 * Geqo_effort を下限（minsize）とし、計算値がこれを下回る場合は下限値を返す。 5. 上記のいずれにも該当しない場合、計算値の切り上げ（ceil）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_main.c 内の geqo() 関数から呼び出される。
  • **呼び出し先**: 数学関数 pow および ceil を使用する。

関連データ構造

  • Geqo_pool_size (グローバル変数): ユーザー定義の個体群サイズ設定。
  • Geqo_effort (グローバル変数): GEQOの探索努力量を制御する設定値。

失敗モードと境界条件

  • Geqo_pool_size が 1 以下の場合は無視され、計算ロジックが適用される。
  • nr_rel が小さい場合、計算値が minsize に切り上げられる。

保守担当者向けメモ

  • 本関数は geqo() の初期化フェーズで呼び出される。
  • Geqo_effort の値が変更されると、算出される個体群サイズの範囲も連動して変化する点に注意すること。
