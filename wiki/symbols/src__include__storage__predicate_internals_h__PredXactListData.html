PredXactListData

定義

  • 種別: struct
  • 定義箇所: src/include/storage/predicate_internals.h:144

目的

SSI（Serializable Snapshot Isolation）における直列化可能トランザクションの管理状態を保持する構造体です。共有メモリ上でアクティブなトランザクションのリスト管理およびグローバルな統計情報の追跡を行います。

シグネチャまたは形状

typedef struct PredXactListData
{
    dlist_head    availableList;
    dlist_head    activeList;
    TransactionId SxactGlobalXmin;
    int           SxactGlobalXminCount;
    int           WritableSxactCount;
    SerCommitSeqNo LastSxactCommitSeqNo;
    SerCommitSeqNo CanPartialClearThrough;
    SerCommitSeqNo HavePartialClearedThrough;
    SERIALIZABLEXACT *OldCommittedSxact;
    SERIALIZABLEXACT *element;
} PredXactListData;

入力と出力

  • **入力**: トランザクションの開始、コミット、および競合発生時に本構造体の各フィールドが更新されます。
  • **出力**: SxactGlobalXmin や LastSxactCommitSeqNo などのグローバルな状態値を提供し、SSIサブシステム全体の判断基準となります。

前提条件と不変条件

  • SerializableXactHashLock によって保護される必要があります（一部フィールドを除く）。
  • LastSxactCommitSeqNo は、直列化可能トランザクションのコミットごとに単調増加する不変条件を持ちます。
  • SxactGlobalXmin はアクティブな直列化可能トランザクションのグローバルな最小Xminを維持します。

アルゴリズムまたは振る舞い

  • **リスト管理**: availableList（未使用）と activeList（実行中）の二重連結リストを用いて、SERIALIZABLEXACT オブジェクトを管理します。
  • **部分クリア**: CanPartialClearThrough および HavePartialClearedThrough を用いて、コミット済みトランザクションに関連する述語ロックや競合情報のクリーンアップ範囲を制御します。
  • **グローバル統計**: 書き込み可能なトランザクション数や、グローバルな最小Xminを追跡し、SSIの整合性判定に利用します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数名は、提供されたソース抜粋には含まれていません。
  • predicate.c 内のSSIサブシステムによって操作されることが示唆されています。

関連データ構造

  • SERIALIZABLEXACT: 本構造体で管理される個別のトランザクション状態。
  • SerCommitSeqNo: コミット順序を追跡するためのシーケンス番号型。
  • dlist_head: lib/ilist.h で定義される共有メモリ上のリスト管理構造。

失敗モードと境界条件

  • 共有メモリ上のロック（SerializableXactHashLock 等）の競合により、アクセスがブロックされる可能性があります。
  • availableList が空の場合、新しいトランザクションの登録が制限される可能性があります。

保守担当者向けメモ

  • 本構造体は全バックエンドで共有されるグローバルな状態を保持します。
  • フィールドの更新時には、コメントに記載された適切なロック（SerializableXactHashLock または SerializableFinishedListLock）を必ず取得してください。
  • OldCommittedSxact はダミーのトランザクション状態を指す共有コピーとして使用されます。
