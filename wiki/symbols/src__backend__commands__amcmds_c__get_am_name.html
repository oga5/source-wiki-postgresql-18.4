get_am_name

定義

  • 種別: function
  • 定義箇所: src/backend/commands/amcmds.c:191

目的

指定されたアクセスメソッド（AM）の OID に対応する名前を取得する。

シグネチャまたは形状

char *get_am_name(Oid amOid)

入力と出力

  • **入力**: amOid (アクセスメソッドの OID)
  • **出力**: アクセスメソッド名（pstrdup によりメモリ確保された文字列）。該当する OID が存在しない場合は NULL を返す。

前提条件と不変条件

  • AMOID システムキャッシュを使用して検索を行うため、システムカタログ pg_am が読み取り可能であること。
  • 戻り値は呼び出し元で pfree する必要がある（pstrdup を使用しているため）。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用し、AMOID キャッシュから amOid に一致する pg_am カタログのタプルを検索する。 2. タプルが存在する場合、GETSTRUCT で Form_pg_am 構造体へキャストし、amname フィールドを抽出する。 3. pstrdup で名前のコピーを作成し、ReleaseSysCache でタプルの参照を解放する。 4. 取得した文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/opclasscmds.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • SearchSysCache1
  • HeapTupleIsValid
  • GETSTRUCT
  • pstrdup
  • ReleaseSysCache

関連データ構造

  • Form_pg_am: pg_am システムカタログの行構造体。
  • HeapTuple: システムカタログから取得されるタプル。

失敗モードと境界条件

  • 指定された amOid が pg_am に存在しない場合、SearchSysCache1 は無効なタプルを返し、関数は NULL を返す。

保守担当者向けメモ

  • 本関数は pg_am カタログの amname を取得する標準的な手段である。
  • 呼び出し元でメモリリークを防ぐため、戻り値の解放が適切に行われているか注意すること。
