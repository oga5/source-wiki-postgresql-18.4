pg_checksum_page

定義

  • 種別: function
  • 定義箇所: src/include/storage/checksum_impl.h:186

目的

PostgreSQLのデータページに対するチェックサムを計算します。ページ内の既存のチェックサム値を一時的に無視し、ページ内容とブロック番号に基づいて整合性を検証するための値を生成します。

シグネチャまたは形状

uint16 pg_checksum_page(char *page, BlockNumber blkno)

入力と出力

  • **入力**:
  • page: チェックサム計算対象となるページデータへのポインタ。
  • blkno: ページが属するブロック番号。
  • **出力**:
  • 戻り値: 計算された16ビットのチェックサム値（1〜65535の範囲）。

前提条件と不変条件

  • page は適切に初期化されたページである必要があります（PageIsNew マクロによる検証で偽となること）。
  • 本関数はページヘッダーのチェックサムフィールドを一時的に変更しますが、計算終了後に元の値へ復元します。

アルゴリズムまたは振る舞い

1. ページヘッダーの pd_checksum を退避し、一時的に0で上書きします。 2. pg_checksum_block を呼び出し、ページ全体に対して並列ハッシュ計算を行います。 3. 退避していた元の pd_checksum を復元します。 4. 計算結果のハッシュ値に対して blkno をXOR演算し、ページが誤った場所に配置されていないかを確認します。 5. 最終的な値を (checksum % 65535) + 1 により16ビットの範囲に収め、0以外の値を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: pg_checksum_block
  • **呼び出し元**:

関連データ構造

  • PGChecksummablePage: ページデータを32列の行列として扱う共用体。
  • PageHeaderData: ページヘッダー構造。

失敗モードと境界条件

  • PageIsNew なページに対しては Assert により停止します。
  • 戻り値は必ず1〜65535の範囲に収まるよう設計されており、0は返りません。

保守担当者向けメモ

  • 本関数はチェックサムの計算のみを行い、ページへの書き込みは行いません。
  • N_SUMS や checksumBaseOffsets の変更はデータベースの互換性を損なうため注意してください。
  • 外部ツールから利用する際は、Assert マクロの定義環境に留意してください。
