ExecAppend

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:302

目的

複数の子プラン（サブプラン）を順次または並列に実行し、結果を統合して上位ノードへタプルを提供します。

シグネチャまたは形状

TupleTableSlot *ExecAppend(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（AppendState 構造体としてキャストして使用）
  • **出力**: TupleTableSlot *（取得したタプル、または終了時は NULL を含むスロット）

前提条件と不変条件

  • 初回呼び出し時、node->as_begun が false であること。
  • as_begun が true の場合、as_whichplan は有効なサブプランインデックス（0 以上 as_nplans 未満）であること。
  • 非同期サブプランが存在する場合、ExecAppendAsyncGetNext がタプルを返すか、as_needrequest が空になるまで処理が継続されること。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時、as_begun が偽であれば、非同期サブプランの開始（ExecAppendAsyncBegin）および最初の同期サブプランの選択（choose_next_subplan）を行う。 2. **ループ処理**:

  • 非同期サブプランが有効な場合、ExecAppendAsyncGetNext を試行し、タプルがあれば即座に返す。
  • 現在の同期サブプランから ExecProcNode を呼び出し、タプルを取得する。
  • タプルが取得できた場合はそれを返す。
  • タプルが空（TupIsNull）の場合、非同期イベントの待機（ExecAppendAsyncEventWait）を行い、choose_next_subplan を呼び出して次のサブプランへ移行する。
  • 全てのサブプランが終了し、かつ非同期サブプランも残っていない場合は ExecClearTuple を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン内の ExecProcNode インターフェース経由で呼び出される（直接的な呼び出し元関数の詳細はソース抜粋からは不明）。
  • **呼び出し先**:
  • ExecAppendAsyncBegin
  • ExecAppendAsyncGetNext
  • ExecAppendAsyncEventWait
  • ExecProcNode
  • node->choose_next_subplan（関数ポインタ）

関連データ構造

  • AppendState: ノードの実行状態を保持する主要構造体。
  • PlanState: 汎用的なプランノード状態。

失敗モードと境界条件

  • **サブプランなし**: as_nplans == 0 の場合、即座に空スロットを返す。
  • **全プラン終了**: 同期・非同期サブプランがすべて枯渇した場合、ExecClearTuple を返して終了する。
  • **割り込み**: CHECK_FOR_INTERRUPTS() により、実行中に割り込みが発生した場合は処理を中断する。

保守担当者向けメモ

  • 本関数は AppendState の as_begun フラグを用いて初期化状態を管理しています。
  • choose_next_subplan は関数ポインタであり、実行モード（ローカル、リーダー、ワーカー）に応じて動的に切り替わります。
  • 非同期実行の制御は as_nasyncremain および as_needrequest ビットマップの状態に依存しています。
