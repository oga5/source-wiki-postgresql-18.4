DirectFunctionCall2Coll

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/fmgr.c:811

目的

2つの引数をとる関数を、FmgrInfo構造体の初期化を介さずに直接呼び出すためのオーバーヘッドを最小化したラッパー関数です。

シグネチャまたは形状

Datum DirectFunctionCall2Coll(PGFunction func, Oid collation, Datum arg1, Datum arg2)

入力と出力

  • **入力**:
  • func: 呼び出す関数ポインタ（PGFunction）。
  • collation: 照合順序（Oid）。
  • arg1, arg2: 関数に渡す2つの引数（Datum）。
  • **出力**:
  • 関数の実行結果（Datum）。

前提条件と不変条件

  • 引数 arg1 および arg2 は非NULLとして扱われます。
  • 呼び出される関数はNULLを返さないことが期待されます。NULLが返された場合、elog(ERROR)により処理が中断されます。
  • ポリモーフィック関数など、FmgrInfoの完全な初期化を必要とする関数には使用できません。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFOマクロを使用して、2引数用のFunctionCallInfoBaseDataをスタック上に確保します。 2. InitFunctionCallInfoDataを呼び出し、照合順序と引数情報を設定します。 3. arg1およびarg2をfcinfo->argsに格納し、isnullフラグをfalseに設定します。 4. 関数ポインタfuncをfcinfoと共に実行します。 5. 実行結果のfcinfo->isnullを確認し、NULLであればエラーを発生させます。 6. 正常終了時に結果のDatumを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_text.c 内の各関数（text_gt, text_ge, texteq, text_le, text_lt, bttextcmp, bpchargt, bpcharge, bpchareq, bpcharle, bpcharlt, bpcharcmp 等）。
  • **呼び出し先**: 引数として渡された PGFunction 型の関数ポインタ。

関連データ構造

  • FunctionCallInfoBaseData: 実行時の引数やコンテキストを保持する構造体。
  • Datum: PostgreSQLの汎用データ型。
  • PGFunction: 関数ポインタの型定義。

失敗モードと境界条件

  • 呼び出された関数がNULLを返した場合、elog(ERROR)により強制終了します。
  • 渡された関数ポインタが不正な場合、セグメンテーションフォールト等の未定義動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はパフォーマンスが重要なパスで使用されます。FmgrInfoを必要としない単純な関数呼び出しに限定してください。
  • 呼び出し先がNULLを返す可能性がある場合は、本関数ではなく通常のFunctionCall系インターフェースを使用してください。
