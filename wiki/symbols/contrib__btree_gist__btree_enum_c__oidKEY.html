oidKEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_enum.c:15

目的

PostgreSQL の ENUM 型を GiST インデックス内で表現するための範囲（レンジ）構造体です。ENUM 型は内部的に OID として扱われるため、その順序関係を保持する下限値と上限値を格納します。

シグネチャまたは形状

typedef struct
{
	Oid			lower;
	Oid			upper;
} oidKEY;

入力と出力

  • 入力: ENUM 型の値を OID として保持する lower および upper フィールド。
  • 出力: GiST インデックスのキーとして、範囲検索やインデックス構築の際に利用されます。

前提条件と不変条件

  • lower は範囲の下限、upper は範囲の上限を表します。
  • ENUM 型の順序付けに基づき、lower <= upper であることが期待されます。

アルゴリズムまたは振る舞い

GiST インデックスのノードにおいて、ENUM 値の範囲を保持します。この構造体は contrib/btree_gist/btree_utils_num.c で定義された数値ユーティリティ関数（gbt_num_*）によって操作され、インデックスの圧縮、一致判定、分割などの処理で利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/btree_gist/btree_enum.c 内の各 GiST サポート関数（gbt_enum_consistent, gbt_enum_compress, gbt_enum_union 等）がこの構造体をキャストして使用しています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • gbtree_ninfo: ENUM 型の比較関数やサイズ情報を保持する設定テーブル。
  • GISTENTRY: PostgreSQL の GiST インデックスエントリ構造体。

失敗モードと境界条件

  • DatumGetPointer を介して entry->key からキャストする際、無効なポインタや不適切なデータ型が渡された場合はメモリ破壊やクラッシュの原因となります。
  • ENUM 型の OID が予期せず変更された場合、インデックス内の範囲整合性が損なわれる可能性があります。

保守担当者向けメモ

  • 本構造体は btree_oid.c で使用される oidKEY と同一の定義です。
  • ENUM 型の比較は内部的な OID の順序に依存するため、この構造体は OID 型のインデックス実装と非常に近い挙動を示します。
