copy_replication_slot

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:603

目的

既存のレプリケーションスロットの設定と状態を複製し、新しいスロットを作成する内部ヘルパー関数です。

シグネチャまたは形状

static Datum copy_replication_slot(FunctionCallInfo fcinfo, bool logical_slot)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • logical_slot: 論理スロットとしてコピーするかどうかを示すフラグ。
  • **出力**:
  • Datum: 処理結果（行型）。

前提条件と不変条件

  • 呼び出し元で CheckSlotPermissions() が実行され、権限が検証されていること。
  • 論理スロットの場合は CheckLogicalDecodingRequirements()、物理スロットの場合は CheckSlotRequirements() が満たされていること。
  • コピー元のスロットがWALを予約していること（restart_lsn が有効であること）。
  • コピー元のスロットが無効化（invalidated）されていないこと。
  • コピー元とコピー先の型（論理/物理）が一致していること。

アルゴリズムまたは振る舞い

1. ReplicationSlotControlLock を共有モードで取得し、ソーススロットを検索してその内容をコピーします。 2. コピーした情報に基づき、create_logical_replication_slot または create_physical_replication_slot を呼び出して新しいスロットを作成します。 3. 作成後、再度ソーススロットのロックを取得し、ソーススロットが削除や変更を受けていないか（restart_lsn の後退や型変更など）を検証します。 4. 検証に失敗した場合はエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/slotfuncs.c 内の各SQL関数エントリポイント（論理/物理スロットのコピー用関数）。
  • **呼び出し先**:
  • create_logical_replication_slot
  • create_physical_replication_slot
  • CheckSlotPermissions
  • CheckLogicalDecodingRequirements / CheckSlotRequirements

関連データ構造

  • ReplicationSlot: スロットの内部状態を保持する構造体。
  • ReplicationSlotCtl: 共有メモリ上のスロット制御構造体。

失敗モードと境界条件

  • 指定されたソーススロットが存在しない場合、ERRCODE_UNDEFINED_OBJECT を発生させます。
  • 型の不一致（物理を論理としてコピーしようとする等）が発生した場合、ERRCODE_FEATURE_NOT_SUPPORTED を発生させます。
  • WALを予約していないスロットや、無効化されたスロットをコピーしようとした場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE を発生させます。
  • コピー中にソーススロットの状態が不整合になった場合、エラーを発生させます。

保守担当者向けメモ

  • 本関数は、スロットのコピー中にソーススロットのWALが削除されるのを防ぐため、二段階のロックと検証を行っています。
  • 論理スロットのコピー時、フェイルオーバーオプションは同期の問題を避けるためコピーされません。
