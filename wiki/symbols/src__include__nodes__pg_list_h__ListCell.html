ListCell

定義

  • 種別: union
  • 定義箇所: src/include/nodes/pg_list.h:45

目的

PostgreSQLの汎用リスト構造（List）において、各要素を保持するための共用体です。

シグネチャまたは形状

typedef union ListCell
{
	void	   *ptr_value;
	int			int_value;
	Oid			oid_value;
	TransactionId xid_value;
} ListCell;

入力と出力

  • **入力**: 保持する値として void*（ポインタ）、int、Oid、TransactionId のいずれかを受け入れます。
  • **出力**: lfirst などのマクロを通じて、格納されている値を各型として取り出します。

前提条件と不変条件

  • List 構造体の elements 配列の一部として存在します。
  • リストが空でない限り、ListCell のメモリ配置は安定しており、再配置されません。

アルゴリズムまたは振る舞い

  • 複数のデータ型を単一のメモリ領域で表現する共用体として機能します。
  • foreach マクロによるリスト走査時に、現在の要素を指すポインタとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のソースコードで ListCell 型のポインタ変数が宣言されていることが確認されています（直接的な呼び出し関係はソースコードの文脈に依存します）。
  • contrib/amcheck/verify_heapam.c
  • contrib/bloom/blvalidate.c
  • contrib/dblink/dblink.c
  • contrib/dict_int/dict_int.c
  • contrib/dict_xsyn/dict_xsyn.c
  • contrib/file_fdw/file_fdw.c
  • contrib/pg_trgm/trgm_regexp.c
  • **呼び出し先**: 情報なし。

関連データ構造

  • List: ListCell の配列を管理するヘッダ構造体。
  • nodes/nodes.h: ノード基盤定義。

失敗モードと境界条件

  • **型安全性**: ListCell 自体は型情報を保持しません。誤った型として値を取り出した場合の動作は未定義です。
  • **走査中の削除**: foreach ループ内で要素を削除する場合、通常の削除関数を使用するとイテレータの整合性が崩れるため、foreach_delete_current を使用する必要があります。

保守担当者向けメモ

  • lfirst マクロは void * を返すため、ノードを扱う際は lfirst_node(type, lc) を使用して型キャストを行うことが推奨されます。
  • 歴史的な経緯からLispのコンスセルを模倣していますが、現在は拡張可能な配列として実装されている点に注意してください。
