ExecGroup

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGroup.c:35

目的

ソート済み入力ストリームに対してグループ化処理を行い、グループ単位での射影（Projection）および HAVING 句の評価を実行する。

シグネチャまたは形状

TupleTableSlot *ExecGroup(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（GroupState 構造体としてキャストされる）。
  • **出力**: グループ条件（HAVING 句）を満たすグループの射影結果である TupleTableSlot *、または処理終了時は NULL。

前提条件と不変条件

  • 入力となる子プランのタプルは、グループ化キーに従ってソート済みであること。
  • node->ss.ss_ScanTupleSlot は、各グループの最初のタプルを保持するために使用される。

アルゴリズムまたは振る舞い

1. **初期化**: 初回呼び出し時、子プランから最初のタプルを取得し、firsttupleslot に格納する。 2. **HAVING 評価**: ExecQual を用いて HAVING 句を評価する。合格すれば ExecProject で射影結果を返す。 3. **グループ走査**: ループ内で子プランから順次タプルを取得し、eqfunction を用いて firsttupleslot と比較する。 4. **グループ境界**: キーが一致する間は読み飛ばし、不一致となった時点で新しいグループの開始とみなす。 5. **終了処理**: 子プランからタプルが取得できなくなった場合、node->grp_done を true に設定し、NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（node->ss.ps.ExecProcNode に関数ポインタとして登録される）。
  • **呼び出し先**: ExecProcNode（子プランの実行）、ExecCopySlot、ExecQual、ExecProject、ExecQualAndReset。

関連データ構造

  • GroupState: 実行状態を保持する構造体。
  • PlanState: 実行ノードの共通状態。
  • TupleTableSlot: タプルを保持するスロット。
  • ExprContext: 式評価用のコンテキスト。

失敗モードと境界条件

  • 入力が空の場合、即座に grp_done を true にし NULL を返す。
  • HAVING 句の条件を満たさないグループはスキップされ、次のグループの走査へ移行する。
  • CHECK_FOR_INTERRUPTS() により、処理中に割り込みチェックを行う。

保守担当者向けメモ

  • 本関数は ExecQualAndReset を使用しており、ループ内でのメモリコンテキスト管理が最適化されている。
  • nodeGroup.c の他の関数（ExecInitGroup, ExecEndGroup, ExecReScanGroup）と連携してライフサイクルが管理される。
