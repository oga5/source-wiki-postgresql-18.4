init_timezone_hashtable

定義

  • 種別: function
  • 定義箇所: src/timezone/pgtz.c:201

目的

タイムゾーン情報をキャッシュするためのハッシュテーブル timezone_cache を初期化します。

シグネチャまたは形状

static bool init_timezone_hashtable(void)

入力と出力

  • 入力: なし
  • 出力: 初期化に成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • timezone_cache グローバル変数が本関数によって初期化されます。
  • ハッシュテーブルのキーサイズは TZ_STRLEN_MAX + 1、エントリサイズは sizeof(pg_tz_cache) に固定されます。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を設定し、キーサイズとエントリサイズを定義します。 2. hash_create を呼び出し、"Timezones" という名前で初期サイズ 4 のハッシュテーブルを作成します。 3. 作成されたハッシュテーブルのポインタを timezone_cache に格納します。 4. 作成結果を判定し、成功・失敗を呼び出し元に返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/timezone/pgtz.c 内のタイムゾーン初期化処理（pg_timezone_initialize 等）から呼び出されることが示唆されています。
  • 呼び出し先: hash_create (PostgreSQL のハッシュテーブル管理ユーティリティ)。

関連データ構造

  • timezone_cache: タイムゾーン情報を保持するハッシュテーブル。
  • pg_tz_cache: ハッシュテーブルのエントリとして使用される構造体。
  • HASHCTL: ハッシュテーブルの作成パラメータを保持する構造体。

失敗モードと境界条件

  • hash_create がメモリ不足等で NULL を返した場合、本関数は false を返し、初期化失敗として扱われます。

保守担当者向けメモ

  • 本関数は静的関数であり、モジュール内でのみ使用されます。
  • ハッシュテーブルの初期サイズは 4 と小さく設定されていますが、必要に応じて調整可能です。
