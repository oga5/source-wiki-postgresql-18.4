PQcancelSocket

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:312

目的

キャンセル要求用接続（PGcancelConn）に関連付けられたソケット記述子を取得します。

シグネチャまたは形状

int PQcancelSocket(const PGcancelConn *cancelConn);

入力と出力

  • **入力**: cancelConn - キャンセル要求の状態を保持する PGcancelConn 構造体へのポインタ。
  • **出力**: 成功した場合はソケット記述子（整数）、失敗した場合は PQsocket の仕様に従う値（通常は無効なソケットを示す値）を返します。

前提条件と不変条件

  • cancelConn は有効なポインタである必要があります。
  • 内部の PGconn 構造体が初期化されていることを前提とします。

アルゴリズムまたは振る舞い

内部的に PQsocket 関数を呼び出し、cancelConn 構造体内に保持されている conn フィールドからソケット記述子を抽出して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/libpq/libpq-be-fe-helpers.h:439
  • src/test/modules/libpq_pipeline/libpq_pipeline.c:303
  • src/test/modules/libpq_pipeline/libpq_pipeline.c:360
  • **呼び出し先**:
  • PQsocket（libpq 内部関数）

関連データ構造

  • PGcancelConn: キャンセル要求の通信状態を管理する構造体。

失敗モードと境界条件

  • cancelConn が NULL の場合の挙動は、内部の PQsocket 実装に依存します。
  • ソケットがまだ作成されていない、または既に閉じられている場合、無効なソケット記述子が返される可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーです。ソケット取得ロジックの変更は PQsocket 側で行う必要があります。
  • 非同期キャンセル処理において、ソケットの監視（select や poll 等）を行うために利用されます。
