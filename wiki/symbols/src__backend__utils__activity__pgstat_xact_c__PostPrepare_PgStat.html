PostPrepare_PgStat

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/activity/pgstat_xact.c:210

目的

2相コミット（PREPARE）が実行された後のトランザクション状態において、統計情報の後処理を行い、現在のトランザクション統計スタックをリセットする。

シグネチャまたは形状

void PostPrepare_PgStat(void)

入力と出力

  • **入力**: なし（グローバル変数 pgStatXactStack を参照）
  • **出力**: なし（副作用として pgStatXactStack のクリアおよび統計スナップショットの破棄を行う）

前提条件と不変条件

  • 本関数は、トランザクションが PREPARE された直後に呼び出されることを想定している。
  • pgStatXactStack が存在する場合、そのネストレベルは 1 であり、かつスタックの先頭であることが保証される必要がある（Assert による検証）。

アルゴリズムまたは振る舞い

1. 現在のトランザクション統計スタック (pgStatXactStack) を取得する。 2. スタックが存在する場合、PostPrepare_PgStat_Relations(xact_state) を呼び出し、リレーションに関連する統計の後処理を実行する。 3. pgStatXactStack を NULL に設定し、スタックを破棄する（メモリは TopTransactionContext 上にあるため明示的な解放は不要）。 4. pgstat_clear_snapshot() を呼び出し、保持されている統計スナップショットを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c:2735
  • **呼び出し先**:
  • PostPrepare_PgStat_Relations
  • pgstat_clear_snapshot

関連データ構造

  • pgStatXactStack: 現在のトランザクションおよびサブトランザクションの統計状態を保持するスタック。
  • PgStat_SubXactStatus: スタック内の各要素の型。

失敗モードと境界条件

  • pgStatXactStack が NULL の場合、スタック操作はスキップされるが、pgstat_clear_snapshot() は実行される。
  • Assert に違反する場合（ネストレベルが 1 でない等）、デバッグビルドではプロセスが停止する。

保守担当者向けメモ

  • 本関数は TopTransactionContext 上のメモリを利用しているため、明示的な pfree は行わない設計となっている。
  • 2相コミットのフローにおいて、統計情報の整合性を維持するための重要な終了処理ポイントである。
