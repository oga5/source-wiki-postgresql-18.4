PQExpBuffer

定義

  • 種別: typedef
  • 定義箇所: src/interfaces/libpq/pqexpbuffer.h:51

目的

動的に拡張可能な文字列バッファ（PQExpBufferData）を操作するためのポインタ型を提供し、libpq におけるメモリ管理を抽象化します。

シグネチャまたは形状

typedef PQExpBufferData *PQExpBuffer;

入力と出力

  • **入力**: PQExpBufferData 構造体へのポインタ。
  • **出力**: なし（型定義）。

前提条件と不変条件

  • malloc を使用してメモリを確保するため、使用後は必ず destroyPQExpBuffer() または termPQExpBuffer() による解放が必要です。
  • メモリ不足（OOM）が発生した場合、バッファは「broken」状態（maxlen = 0）に遷移します。

アルゴリズムまたは振る舞い

  • PQExpBufferData 構造体への参照として機能し、文字列の追記、フォーマット出力、バイナリデータの構築をサポートします。
  • 内部で enlargePQExpBuffer を呼び出し、必要に応じて動的にメモリを再確保します。
  • PQExpBufferBroken(str) マクロにより、バッファが破壊状態にあるかを判定可能です。

呼び出し元と呼び出し先

  • **呼び出し元（例）**:
  • src/bin/initdb/initdb.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_basebackup/astreamer_inject.c
  • **呼び出し先**:
  • createPQExpBuffer() / destroyPQExpBuffer()（初期化・破棄）
  • appendPQExpBuffer() / appendBinaryPQExpBuffer()（データ操作）
  • enlargePQExpBuffer()（内部メモリ拡張）

関連データ構造

  • PQExpBufferData: バッファの実体（data、len、maxlen を保持）。

失敗モードと境界条件

  • メモリ確保失敗時: バッファは maxlen = 0 の「broken」状態となり、以降の操作は安全に無視されます。

保守担当者向けメモ

  • バックエンドの StringInfo と概念的に同等ですが、palloc ではなく標準の malloc を使用する点に注意してください。
  • ループ内で文字列を構築する際は、必ず PQExpBufferBroken をチェックし、エラー状態での処理継続を回避してください。
