log_pre_callback

定義

  • 種別: function
  • 定義箇所: src/bin/psql/startup.c:91

目的

ログ出力処理の直前に、クエリ出力先ファイルストリームのバッファをフラッシュし、出力の整合性を確保する。

シグネチャまたは形状

static void log_pre_callback(void)

入力と出力

  • 入力: なし（グローバル変数 pset を参照）
  • 出力: なし（副作用としてファイルストリームのフラッシュを行う）

前提条件と不変条件

  • グローバル変数 pset が初期化されていること。
  • pset.queryFout が有効なファイルポインタであるか、あるいは NULL であること。

アルゴリズムまたは振る舞い

1. pset.queryFout が NULL でなく、かつ標準出力 (stdout) と異なる場合、そのファイルストリームに対して fflush() を実行する。 2. これにより、ログメッセージが出力される前に、現在進行中のクエリ結果の書き出しを完了させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/logging.c 内のログ出力処理（log_pre_callback() として実行される）。
  • 呼び出し先: fflush()（標準ライブラリ関数）。
  • 登録元: src/bin/psql/startup.c にて pg_logging_set_pre_callback(log_pre_callback) を介して登録される。

関連データ構造

  • PsqlSettings (グローバル変数 pset): psql の実行状態を保持する構造体。queryFout メンバが参照される。

失敗モードと境界条件

  • pset.queryFout が stdout と等しい場合はフラッシュをスキップする。
  • pset.queryFout が NULL の場合は何もしない。

保守担当者向けメモ

  • 本関数は pg_logging_set_pre_callback を通じてログシステムのコールバックとして登録されるため、直接呼び出すのではなく、ログ出力のフックとして機能する点に注意すること。
