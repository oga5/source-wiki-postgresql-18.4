fe_recvint64

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/streamutil.c:867

目的

ネットワークバイトオーダー（ビッグエンディアン）でエンコードされた 64 ビット整数を、ホストのバイトオーダーに変換して読み取ります。

シグネチャまたは形状

int64 fe_recvint64(char *buf)

入力と出力

  • **入力**: buf (ネットワークバイトオーダーの 8 バイトデータを含むバッファへのポインタ)
  • **出力**: ホストバイトオーダーに変換された int64 値

前提条件と不変条件

  • buf は少なくとも 8 バイトの有効なメモリ領域を指している必要があります。
  • 入力データはネットワークバイトオーダー（ビッグエンディアン）であると想定されます。

アルゴリズムまたは振る舞い

1. memcpy を使用して、buf から 8 バイトのデータを uint64 型の変数 n64 にコピーします。 2. pg_ntoh64 マクロを適用し、ネットワークバイトオーダーからホストバイトオーダーへ変換します。 3. 変換後の値を int64 として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_recvlogical.c
  • src/bin/pg_basebackup/receivelog.c
  • **呼び出し先**:
  • pg_ntoh64 (マクロ)
  • memcpy (標準ライブラリ)

関連データ構造

  • int64 / uint64 (PostgreSQL の固定長整数型)

失敗モードと境界条件

  • buf が NULL または 8 バイト未満の領域を指している場合、未定義の動作（セグメンテーション違反やメモリ破壊）が発生します。
  • 変換処理自体は常に成功しますが、入力データの整合性は呼び出し元で保証する必要があります。

保守担当者向けメモ

  • 本関数はフロントエンド（クライアント）側でレプリケーションプロトコルのバイナリデータを解析するために使用されます。
  • サーバーとの通信プロトコル仕様が変更されない限り、本関数のロジックを変更する必要はありません。
