member_verify_checksum

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/astreamer_verify.c:284

目的

アーカイブメンバーのストリーム処理終了時に、読み込まれたデータ量と計算済みのチェックサムをバックアップマニフェストの情報と照合し、整合性を検証する。

シグネチャまたは形状

static void member_verify_checksum(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer（検証状態を保持する astreamer_verify 構造体へのポインタ）
  • **出力**: なし（検証失敗時は report_backup_error を通じてエラーを報告する）

前提条件と不変条件

  • mystreamer->verify_checksum が真であること（Assert により保証）。
  • mystreamer->checksum_ctx が有効なチェックサム計算コンテキストとして初期化されていること。

アルゴリズムまたは振る舞い

1. **サイズ検証**: 実際に読み込まれたバイト数（mystreamer->checksum_bytes）と、マニフェスト上の期待サイズ（m->size）を比較する。不一致の場合はエラーを報告して終了する。 2. **チェックサム確定**: pg_checksum_final を呼び出し、計算された最終チェックサムを取得する。失敗した場合はエラーを報告して終了する。 3. **チェックサム比較**:

  • チェックサムの長さがマニフェストの定義と一致するか確認する。
  • memcmp を用いて、計算されたチェックサム値とマニフェストのペイロード（m->checksum_payload）を比較する。
  • 不一致がある場合は、エラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_verifybackup/astreamer_verify.c:117（astreamer_verify_content 関数内のトレーラー処理フェーズから呼び出される）。
  • **呼び出し先**:
  • pg_checksum_final（チェックサム計算の終了処理）
  • report_backup_error（エラー発生時の通知）

関連データ構造

  • astreamer_verify: 検証コンテキストを保持する構造体。
  • manifest_file: マニフェスト内のファイル情報を保持する構造体。

失敗モードと境界条件

  • **サイズ不一致**: アーカイブ内のデータ長がマニフェストの期待値と異なる場合。
  • **チェックサム計算失敗**: pg_checksum_final が負の値を返した場合。
  • **長さ不一致**: 計算されたチェックサムの長さがマニフェストの期待値と異なる場合。
  • **値の不一致**: 計算されたチェックサム値がマニフェストの記録と一致しない場合。

保守担当者向けメモ

  • 本関数は astreamer インターフェースの一部として、アーカイブの各メンバーの処理完了時に呼び出される。
  • チェックサムのアルゴリズム自体は pg_checksum_final に依存している。
