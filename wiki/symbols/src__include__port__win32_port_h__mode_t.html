mode_t

定義

  • 種別: typedef
  • 定義箇所: src/include/port/win32_port.h:561

目的

Windows 環境において POSIX 準拠のファイルモード表現を可能にするための型定義です。

シグネチャまたは形状

typedef unsigned short mode_t;

入力と出力

  • 入力: 16ビットの符号なし整数。
  • 出力: ファイルのパーミッションや属性を示すビットマスクとして利用されます。

前提条件と不変条件

  • Windows 環境（MSVC または MinGW）でのビルド時にのみ定義されます。
  • POSIX 互換のファイルアクセス権限（S_IRUSR, S_IWUSR 等）を保持するためのコンテナとして機能します。

アルゴリズムまたは振る舞い

Windows のネイティブなファイル属性と POSIX の mode_t の差異を吸収するため、ファイル操作関数（open 等）の引数として渡される際に、PostgreSQL の移植層を通じて適切な Windows API 呼び出しへ変換されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/file/fd.c (BasicOpenFilePerm, PathNameOpenFilePerm, OpenTransientFilePerm)
  • src/backend/postmaster/syslogger.c (umask 呼び出し時)
  • src/bin/pg_test_fsync/pg_test_fsync.c (open_direct)
  • その他、src/backend/commands/copyto.c 等で変数型として使用。
  • 呼び出し先:
  • 本シンボル自体は型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • struct stat: Windows 環境下で 64 ビット対応として再定義される構造体と組み合わせて使用されます。

失敗モードと境界条件

  • unsigned short 型であるため、値の範囲は 0 から 65535 に制限されます。
  • Windows のファイルシステム（NTFS 等）のアクセス制御リスト（ACL）モデルと POSIX のビットマスクモデルの完全な互換性はないため、変換ロジックに依存した挙動となります。

保守担当者向けメモ

  • 本定義は Cygwin ビルドでは使用されません。
  • sys/stat.h 等の標準ヘッダーとの競合を避けるため、インクルード順序に依存する可能性があります。変更時は win32_port.h 内の定義順序を維持してください。
