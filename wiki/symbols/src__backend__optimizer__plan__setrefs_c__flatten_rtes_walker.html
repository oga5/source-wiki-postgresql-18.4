flatten_rtes_walker

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:496

目的

クエリツリーを走査し、レンジテーブルエントリ（RTE）のうち、リレーション（RTE_RELATION）または有効な OID を持つサブクエリ（RTE_SUBQUERY）を抽出し、フラット化されたレンジテーブルへ保存する。

シグネチャまたは形状

static bool flatten_rtes_walker(Node *node, flatten_rtes_walker_context *cxt);

入力と出力

  • **入力**:
  • node: 走査対象のノード（Node）。
  • cxt: 走査のコンテキスト（flatten_rtes_walker_context）。PlannerGlobal 構造体および現在の Query 構造体への参照を保持する。
  • **出力**:
  • 戻り値: 走査を継続する場合は false、中断する場合は true。

前提条件と不変条件

  • cxt->query は、現在走査中の Query ノードと常に同期している必要がある。
  • Query ノードを走査する際、cxt->query を一時的に更新し、走査終了後に元の値へ復元する不変条件が守られる。

アルゴリズムまたは振る舞い

1. **ノード判定**:

  • RangeTblEntry の場合: rtekind が RTE_RELATION であるか、RTE_SUBQUERY かつ relid が有効な場合に add_rte_to_flat_rtable を呼び出し、フラットなレンジテーブルに追加する。
  • Query の場合: 現在の cxt->query を保存し、対象ノードを新たな cxt->query に設定する。その後、query_tree_walker を用いて再帰的に走査し、終了後に cxt->query を復元する。
  • その他のノードの場合: expression_tree_walker を用いて子ノードを再帰的に走査する。

呼び出し元と呼び出し先

  • **呼び出し元**: query_tree_walker（再帰呼び出し）、および expression_tree_walker（再帰呼び出し）。直接的な外部からの呼び出し元はソースコード上では明示されていない。
  • **呼び出し先**: add_rte_to_flat_rtable、query_tree_walker、expression_tree_walker。

関連データ構造

  • RangeTblEntry: 走査対象となるレンジテーブルエントリ。
  • Query: 走査対象のクエリ構造。
  • flatten_rtes_walker_context: 走査状態を保持するコンテキスト構造体。

失敗モードと境界条件

  • node が NULL の場合は即座に false を返し、何も行わない。
  • サブクエリの走査において、cxt->query の更新に失敗すると、レンジテーブルとパーミッション情報の整合性が失われる可能性がある。

保守担当者向けメモ

  • 本関数は query_tree_walker および expression_tree_walker のコールバックとして機能する。
  • RTE_SUBQUERY の処理において OidIsValid(rte->relid) をチェックしているのは、特定の条件下で生成されるサブクエリのみを対象とするためである。
  • 呼び出し元情報の詳細は、現在のソースコード解析範囲では限定的である。
