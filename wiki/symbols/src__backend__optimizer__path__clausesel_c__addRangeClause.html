addRangeClause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/clausesel.c:426

目的

同一変数に対する複数の範囲条件（例: x > 10 AND x < 20）を収集し、論理的に最も制約の強い選択率を保持するデータ構造を構築・更新します。

シグネチャまたは形状

static void addRangeClause(RangeQueryClause **rqlist, Node *clause, bool varonleft, bool isLTsel, Selectivity s2)

入力と出力

  • **入力**:
  • rqlist: 範囲クエリを保持するリストへのポインタ。
  • clause: 評価対象の式ノード。
  • varonleft: 変数が式の左側にあるかを示すフラグ。
  • isLTsel: 選択率が「より小さい（Less Than）」条件に基づくかを示すフラグ。
  • s2: 計算済みの選択率。
  • **出力**: なし（rqlist が指すリストを直接更新）。

前提条件と不変条件

  • rqlist は RangeQueryClause 構造体の連結リストを指す必要があります。
  • 同一変数に対する複数の条件が入力された場合、常に最も制約の強い（値が小さい）選択率が保持されます。

アルゴリズムまたは振る舞い

1. 式から変数（var）を特定し、条件が上限（high bound）か下限（low bound）かを判定します。 2. rqlist 内を走査し、同一の var を持つ既存の RangeQueryClause を検索します。 3. 一致する要素が見つかった場合:

  • 該当する境界（lobound または hibound）が未設定であれば値を代入します。
  • 既に設定済みであれば、新しい選択率 s2 と比較し、より小さい値を採用します。

    • 4. 一致する要素がない場合:

  • palloc を使用して新しい RangeQueryClause を割り当て、リストの先頭に追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/clausesel.c 内の clauselist_selectivity（推測される文脈上の主要な呼び出し元）。
  • **呼び出し先**: get_leftop, get_rightop, equal, palloc。

関連データ構造

  • RangeQueryClause: 範囲クエリの境界情報（var, lobound, hibound, have_lobound, have_hibound 等）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • equal() 関数による変数比較が失敗した場合、同一変数であっても別エントリとして扱われる可能性があります。

保守担当者向けメモ

  • 本関数は clauselist_selectivity における範囲条件の重複排除と最適化の要です。
  • equal() を使用しているため、関数呼び出しを含む複雑な式に対しても正しく動作する設計となっています。
  • 拡張統計が適用される前の、個別の条件に対する基本的な選択率補正ロジックとして機能します。
