UnlockRelation

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lmgr.c:313

目的

指定されたリレーションに対して保持している特定のロックモードを解放します。

シグネチャまたは形状

void UnlockRelation(Relation relation, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relation: 解放対象のリレーション記述子（Relation）。
  • lockmode: 解放するロックのモード（LOCKMODE）。
  • **出力**: なし（void）。

前提条件と不変条件

  • relation 引数は有効なリレーション記述子であり、rd_lockInfo が初期化されている必要があります。
  • 指定された lockmode は、現在のプロセスが当該リレーションに対して保持しているロックと一致している必要があります。

アルゴリズムまたは振る舞い

1. relation->rd_lockInfo からデータベースIDとリレーションIDを取得し、SET_LOCKTAG_RELATION マクロを使用して LOCKTAG を構築します。 2. 構築した tag を引数として LockRelease を呼び出し、共有メモリ上のロックテーブルから該当するロックを解放します。第3引数には false が渡され、セッションロックではなく通常のトランザクションロックとして処理されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c (VACUUM処理中のロック解放)
  • **呼び出し先**:
  • LockRelease (低レベルなロック解放関数)

関連データ構造

  • Relation: リレーションのメタデータおよびロック情報を保持する構造体。
  • LOCKTAG: ロック対象を一意に識別するためのタグ。
  • LOCKMODE: ロックの強度や種類を定義する列挙型。

失敗モードと境界条件

  • 保持していないロックを解放しようとした場合、LockRelease 内部で警告やエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は LockRelation の対となる操作です。
  • rd_lockInfo に依存しているため、リレーションキャッシュが正しく構築されていない状態で呼び出すと予期せぬ動作を引き起こす可能性があります。
