pg_set_regex_collation

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_pg_locale.c:227

目的

正規表現エンジンが使用するロケールおよび文字判定戦略を、指定された照合順序（Collation）に基づいて初期化・設定します。

シグネチャまたは形状

void pg_set_regex_collation(Oid collation)

入力と出力

  • **入力**: collation (Oid) - 使用する照合順序のOID。
  • **出力**: なし（副作用としてグローバル変数 pg_regex_strategy および pg_regex_locale を更新）。

前提条件と不変条件

  • collation が有効なOIDである必要があります。
  • 照合順序が非決定論的（nondeterministic）である場合、エラーが発生します。
  • PG_REGEX_STRATEGY_BUILTIN が選択される場合、データベースエンコーディングが PG_UTF8 である必要があります。

アルゴリズムまたは振る舞い

1. **検証**: collation が無効な場合、エラーを報告します。 2. **Cロケール判定**: C_COLLATION_OID または locale->ctype_is_c が真の場合、戦略を PG_REGEX_STRATEGY_C に設定します。 3. **戦略選択**:

  • COLLPROVIDER_BUILTIN: PG_REGEX_STRATEGY_BUILTIN を選択。
  • COLLPROVIDER_ICU: PG_REGEX_STRATEGY_ICU を選択（USE_ICU 定義時）。
  • COLLPROVIDER_LIBC: データベースエンコーディングが PG_UTF8 なら PG_REGEX_STRATEGY_LIBC_WIDE、それ以外なら PG_REGEX_STRATEGY_LIBC_1BYTE を選択。

    • 4. **適用**: 決定した戦略とロケールオブジェクトをグローバル変数に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regcomp.c:403
  • src/backend/regex/regexec.c:216
  • src/backend/regex/regprefix.c:65
  • **呼び出し先**:
  • pg_newlocale_from_collation (ロケールオブジェクトの取得)
  • GetDatabaseEncoding (エンコーディング確認)
  • ereport (エラー報告)

関連データ構造

  • PG_Locale_Strategy (列挙型: 戦略の定義)
  • pg_locale_t (ロケール情報)

失敗モードと境界条件

  • **ERRCODE_INDETERMINATE_COLLATION**: 照合順序が解決できない場合に発生。
  • **ERRCODE_FEATURE_NOT_SUPPORTED**: 非決定論的な照合順序が指定された場合に発生。

保守担当者向けメモ

  • 本関数は regcomp.c 等から呼び出され、正規表現処理のコンテキストを確立します。
  • pg_regex_strategy および pg_regex_locale はグローバル変数として保持されるため、並行処理や再入時の整合性に注意してください。
