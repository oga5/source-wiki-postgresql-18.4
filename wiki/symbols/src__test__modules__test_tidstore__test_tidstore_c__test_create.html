test_create

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_tidstore/test_tidstore.c:97

目的

テストモジュール内で使用する TidStore インスタンスおよび検証用データ構造を初期化し、メモリ上に確保する。

シグネチャまたは形状

Datum test_create(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BOOL(0): shared（真の場合、共有メモリ上に TidStore を作成する）。
  • **出力**:
  • PG_RETURN_VOID(): 戻り値なし。

前提条件と不変条件

  • 静的変数 tidstore が NULL であること（Assert(tidstore == NULL)）。
  • TopMemoryContext 上で実行され、プロセス終了まで生存するメモリ領域を確保する。

アルゴリズムまたは振る舞い

1. TopMemoryContext へメモリコンテキストを切り替える。 2. shared 引数に基づき TidStore を生成する：

  • true の場合: LWLock トランシェを登録し、TidStoreCreateShared で作成後、dsa_pin_mapping を実行。
  • false の場合: TidStoreCreateLocal で作成。

    • 3. tidstore_empty_size に初期メモリ使用量を記録する。 4. 検証用構造体 items（insert_tids, lookup_tids, iter_tids）を初期化し、メモリを割り当てる。 5. 元のメモリコンテキストへ復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出される（PG_FUNCTION_INFO_V1(test_create) による登録）。
  • **呼び出し先**: LWLockNewTrancheId, LWLockRegisterTranche, TidStoreCreateShared, TidStoreGetDSA, dsa_pin_mapping, TidStoreCreateLocal, TidStoreMemoryUsage, palloc0, MemoryContextSwitchTo。

関連データ構造

  • TidStore: テスト対象のデータ構造。
  • ItemArray (グローバル変数 items): テストの整合性検証に使用する補助構造体。

失敗モードと境界条件

  • tidstore が既に初期化されている場合、Assert により異常終了する。
  • メモリ不足時には palloc0 等がエラーを発生させる。

保守担当者向けメモ

  • 本関数は単一プロセスでのテストを想定している。
  • TidStore のライフサイクル管理の起点であり、test_destroy と対になる。
  • items 構造体の初期化サイズは 1024 バイト固定で確保される。
