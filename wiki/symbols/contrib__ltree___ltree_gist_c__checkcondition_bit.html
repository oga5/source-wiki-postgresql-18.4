checkcondition_bit

定義

  • 種別: function
  • 定義箇所: contrib/ltree/_ltree_gist.c:417

目的

GISTインデックスの検索条件において、特定のビットがシグネチャ内に存在するかを判定する。

シグネチャまたは形状

static bool checkcondition_bit(void *cxt, ITEM *val)

入力と出力

  • **入力**:
  • cxt: LtreeSignature 構造体へのポインタ。シグネチャデータと長さを保持。
  • val: 評価対象の ITEM 構造体。フラグとハッシュ値を含む。
  • **出力**:
  • bool: 条件を満たす場合は true、満たさない場合は false。

前提条件と不変条件

  • cxt は LtreeSignature 型としてキャスト可能であること。
  • val は有効な ITEM 構造体を指していること。

アルゴリズムまたは振る舞い

1. cxt を LtreeSignature 型の sig としてキャストする。 2. val->flag に対して FLG_CANLOOKSIGN マクロを適用し、シグネチャによる判定が可能かを確認する。 3. 判定可能な場合、val->val から AHASHVAL を用いてハッシュ値を算出し、sig->sign 内の該当ビットが立っているかを GETBIT マクロで確認して返す。 4. 判定不可能な場合は、常に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/ltree/_ltree_gist.c および contrib/ltree/ltree_gist.c 内で参照されている。
  • **呼び出し先**: FLG_CANLOOKSIGN, GETBIT, AHASHVAL マクロを使用。

関連データ構造

  • LtreeSignature: シグネチャデータ (sign) とその長さ (siglen) を保持する構造体。
  • ITEM: 検索条件の各要素を保持する構造体。

失敗モードと境界条件

  • cxt が NULL の場合、あるいは val が不正なメモリ領域を指している場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は ltree モジュールのGISTインデックス検索において共通的に利用される。
  • contrib/intarray や src/backend/utils/adt/tsgistidx.c にも同名の関数が存在するが、本ページは contrib/ltree/_ltree_gist.c の実装を対象としている。
