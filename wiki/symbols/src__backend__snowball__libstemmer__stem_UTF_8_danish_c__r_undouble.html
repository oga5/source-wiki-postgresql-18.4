r_undouble

定義

  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_danish.c:261

目的

単語内の重複した子音を検出し、その一方を削除することで語幹を正規化する。

シグネチャまたは形状

static int r_undouble(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体）
  • 出力: 成功時に 1、処理対象外または条件不一致時に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z->I[1] に格納された境界位置以降の処理を行う。
  • z->c（現在のカーソル位置）が z->I[1] より小さい場合、処理は直ちに終了する。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置が z->I[1] 以上であることを確認する。 2. 処理範囲を z->I[1] から現在のカーソル位置までに制限する。 3. in_grouping_b_U を使用し、g_c グループ（子音）に含まれる文字を後方から検索する。 4. 該当文字を z->S[0] に保存し、その文字が直前にも存在するか（eq_v_b）を判定する。 5. 重複が確認された場合、slice_del を呼び出して該当する文字を削除する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_danish.c 内のメイン処理フローから呼び出される。また、stem_ISO_8859_1_danish.c、stem_ISO_8859_1_dutch.c、stem_ISO_8859_2_hungarian.c 等の他言語実装からも参照されている。
  • 呼び出し先: in_grouping_b_U（子音判定）、slice_to（文字列保存）、eq_v_b（文字列比較）、slice_del（削除）。

関連データ構造

  • SN_env: ステミング処理の状態（カーソル位置、境界、作業用文字列バッファ）を保持する構造体。

失敗モードと境界条件

  • z->c < z->I[1] の場合、何もせず 0 を返す。
  • slice_to または slice_del が内部エラー（負の値）を返した場合、その値をそのまま伝播させる。
