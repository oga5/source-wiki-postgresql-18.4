ssl_issuer_field

定義

  • 種別: function
  • 定義箇所: contrib/sslinfo/sslinfo.c:272

目的

現在のSSL接続におけるクライアント証明書の発行者（Issuer）情報から、指定されたフィールド（例: CN, Oなど）の値を抽出してテキスト形式で返します。

シグネチャまたは形状

Datum ssl_issuer_field(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、抽出対象のフィールド名を表す text 型の引数を受け取ります。
  • **出力**: 指定されたフィールドの値を含む text 型の Datum を返します。該当するフィールドが存在しない場合や、SSL接続が確立されていない場合は NULL を返します。

前提条件と不変条件

  • MyProcPort が有効であり、かつ peer（クライアント証明書）が保持されている必要があります。
  • 内部で X509_NAME_field_to_text を呼び出し、OpenSSLのASN.1形式からPostgreSQLのテキスト形式への変換を行います。

アルゴリズムまたは振る舞い

1. MyProcPort->peer を確認し、証明書が存在しない場合は NULL を返します。 2. X509_get_issuer_name を使用して、証明書から発行者名の情報を取得します。 3. X509_NAME_field_to_text を呼び出し、指定されたフィールド名に対応する値を抽出・変換します。 4. 抽出結果が NULL であれば NULL を返し、値があればその Datum を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSQLインターフェースから呼び出されます（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • X509_get_issuer_name (OpenSSL API)
  • X509_NAME_field_to_text (同ファイル内のヘルパー関数)
  • PG_GETARG_TEXT_PP / PG_RETURN_NULL (PostgreSQL関数API)

関連データ構造

  • Port (グローバル変数 MyProcPort を介してアクセス)
  • X509 (OpenSSLの証明書構造体)

失敗モードと境界条件

  • SSL接続が確立されていない場合、またはクライアント証明書が提供されていない場合は NULL を返します。
  • 指定したフィールド名が証明書内に存在しない場合、X509_NAME_field_to_text の結果に基づき NULL を返します。
  • OpenSSLの内部処理でエラーが発生した場合、PostgreSQLの ereport(ERROR, ...) によりトランザクションが中断される可能性があります。

保守担当者向けメモ

  • 本関数は ssl_client_dn_field と同様のロジックで実装されています。フィールド抽出の共通処理に変更を加える際は、両方の関数への影響を確認してください。
  • エンコーディング変換は X509_NAME_field_to_text 内で行われるため、変換エラー時の挙動については当該関数の実装を参照してください。
