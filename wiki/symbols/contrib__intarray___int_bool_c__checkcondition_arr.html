checkcondition_arr

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_bool.c:232

目的

ソート済み整数配列内において、指定された値（ITEM）が存在するかを二分探索を用いて判定する。

シグネチャまたは形状

static bool checkcondition_arr(void *checkval, ITEM *item, void *options)

入力と出力

  • **入力**:
  • checkval: CHKVAL 構造体へのポインタ。検索対象の配列範囲（arrb から arre）を保持する。
  • item: 検索対象の値を持つ ITEM 構造体。
  • options: 使用されていない（void *）。
  • **出力**:
  • bool: 値が見つかった場合は true、それ以外は false を返す。

前提条件と不変条件

  • checkval が指す配列は、昇順にソートされている必要がある。ソートされていない場合、結果は保証されない。
  • ループ不変条件: StopLow <= val < StopHigh。

アルゴリズムまたは振る舞い

1. CHKVAL 構造体から配列の開始ポインタ（arrb）と終了ポインタ（arre）を取得する。 2. 二分探索アルゴリズムを用いて、item->val と一致する要素を探索する。 3. StopMiddle を計算し、中央値が item->val と一致すれば true を返す。 4. 中央値が item->val より小さい場合は探索範囲を後半へ、大きい場合は前半へ絞り込む。 5. 探索範囲が空になった時点で false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_bool.c 内のクエリ実行エンジンからコールバックとして参照されている。
  • src/backend/utils/adt/tsgistidx.c からの参照が確認されている（ツールによるヒューリスティックな検出）。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない。

関連データ構造

  • CHKVAL: 検索対象の配列の境界情報を保持する構造体。
  • ITEM: クエリツリーのノードであり、検索対象の値（val）を保持する。

失敗モードと境界条件

  • 配列が空の場合、StopLow < StopHigh が直ちに偽となり false を返す。
  • 配列がソートされていない場合、二分探索の特性上、値が存在しても見つからない可能性がある。

保守担当者向けメモ

  • 本関数は execute() 関数からコールバックとして呼び出される。
  • options 引数は将来的な拡張のために用意されているが、現時点では使用されていない。
