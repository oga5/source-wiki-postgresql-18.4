Pg_abi_values

定義

  • 種別: struct
  • 定義箇所: src/include/fmgr.h:466

目的

PostgreSQL のバックエンドと動的にロードされるモジュール間での ABI（Application Binary Interface）互換性を検証するための構造体です。

シグネチャまたは形状

typedef struct
{
	int			version;		/* PostgreSQL major version */
	int			funcmaxargs;	/* FUNC_MAX_ARGS */
	int			indexmaxkeys;	/* INDEX_MAX_KEYS */
	int			namedatalen;	/* NAMEDATALEN */
	int			float8byval;	/* FLOAT8PASSBYVAL */
	char		abi_extra[32];	/* see pg_config_manual.h */
} Pg_abi_values;

入力と出力

本構造体は関数ではなくデータ構造ですが、memcmp() を用いた比較対象として使用されます。

前提条件と不変条件

  • 構造体の内容は memcmp() で比較されるため、パディングを含めて一貫性がある必要があります。
  • abi_extra フィールドのサイズは FMGR_ABI_EXTRA マクロのサイズ以下である必要があります（StaticAssertDecl による検証あり）。

アルゴリズムまたは振る舞い

動的ライブラリのロード時に、バックエンド側が保持する値とロード対象のモジュールが保持する値を比較し、不一致がある場合にロードを拒否することで、ABI の不整合によるクラッシュや未定義動作を防止します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**: src/backend/utils/fmgr/dfmgr.c において、モジュールのマジックデータ検証に使用されています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • Pg_magic_struct: Pg_abi_values を含む、モジュール互換性チェック用の上位構造体。

失敗モードと境界条件

  • version や namedatalen 等の各フィールドがバックエンドの定義と一致しない場合、モジュールのロードが失敗します。
  • 構造体の定義を変更する際は、関連するマジックデータ定数の調整が必須です。

保守担当者向けメモ

  • 本構造体のフィールドを変更した場合は、必ず PG_MODULE_ABI_DATA マクロおよび関連する検証ロジックを更新してください。
  • memcmp() による比較が行われるため、構造体のメンバ順序やパディングには細心の注意を払ってください。
