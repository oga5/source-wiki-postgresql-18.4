ExecReScanTableFuncScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTableFuncscan.c:236

目的

テーブル関数スキャンノード（TableFuncScanState）の再スキャン処理を実行し、必要に応じて保持しているタプルストアをリセットまたは再初期化する。

シグネチャまたは形状

void ExecReScanTableFuncScan(TableFuncScanState *node)

入力と出力

  • **入力**: TableFuncScanState *node（スキャンノードの実行状態）
  • **出力**: なし（副作用としてノードの状態を更新）

前提条件と不変条件

  • node は初期化済みであること。
  • node->ss.ps.chgParam が設定されている場合、パラメータ変更に伴う再評価が必要となる。

アルゴリズムまたは振る舞い

1. ps_ResultTupleSlot が存在する場合、ExecClearTuple を呼び出してスロットをクリアする。 2. ExecScanReScan を呼び出し、共通のスキャン再スキャン処理を実行する。 3. chgParam が存在する場合（パラメータが変更された場合）、既存の tupstore を tuplestore_end で破棄し、NULL にリセットする。 4. tupstore が NULL でない場合、tuplestore_rescan を呼び出して現在のストアを先頭から再スキャン可能にする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:214 からの呼び出しが確認されている。
  • **呼び出し先**:
  • ExecClearTuple
  • ExecScanReScan
  • tuplestore_end
  • tuplestore_rescan

関連データ構造

  • TableFuncScanState: テーブル関数スキャンの実行状態を保持する構造体。
  • tuplestore: 抽出された行を保持するメモリ/ディスク上のストレージ。

失敗モードと境界条件

  • tupstore が既に NULL の場合、tuplestore_end や tuplestore_rescan は呼び出されないため、安全に処理される。

保守担当者向けメモ

  • 本関数はマテリアライズ型スキャンの再実行を制御する。パラメータ変更時に tupstore を破棄することで、次回の TableFuncNext 呼び出し時に tfuncFetchRows が再実行されるトリガーとなる。
