ConversionLocation

定義

  • 種別: struct
  • 定義箇所: contrib/postgres_fdw/postgres_fdw.c:307

目的

データ変換処理中にエラーが発生した際、どの属性（カラム）やテーブル、プランノードで問題が起きたかを特定するためのコンテキスト情報を保持する構造体です。

シグネチャまたは形状

typedef struct ConversionLocation
{
	AttrNumber	cur_attno;		/* 処理中の属性番号、または 0 */
	Relation	rel;			/* 処理中の外部テーブル、または NULL */
	ForeignScanState *fsstate;	/* 処理中のプランノード、または NULL */
} ConversionLocation;

入力と出力

本構造体は関数引数として渡されるコンテキストとして機能します。直接的な入力・出力関数はありませんが、エラーハンドリングのコールバック関数内で参照されます。

前提条件と不変条件

  • cur_attno が 0 の場合、特定の属性に紐付かないエラーであることを示唆します。
  • rel および fsstate は、処理の段階に応じて NULL になる可能性があります。

アルゴリズムまたは振る舞い

エラー発生時に、この構造体に格納された情報を利用して、ユーザーに対して詳細なエラー位置（どのテーブルのどのカラムで変換失敗したか等）を報告するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/postgres_fdw/postgres_fdw.c 内のデータ変換処理を行う関数群（例: 7609 行目付近で errpos として宣言）。
  • 呼び出し先: エラー報告用コールバック関数（例: 7755 行目付近で arg としてキャストされ参照）。

関連データ構造

  • Relation: PostgreSQL のリレーション（テーブル）定義。
  • ForeignScanState: 外部スキャン実行時の状態を保持する構造体。

失敗モードと境界条件

  • rel や fsstate が NULL の状態でエラーが発生した場合、エラーメッセージ生成時にこれらのフィールドを参照しないようなロジックが呼び出し側に求められます。

保守担当者向けメモ

  • 本構造体は主にエラー報告の精度を高めるために使用されます。新しい変換ロジックを追加する際は、この構造体を適切に更新してエラーコンテキストを維持してください。
