ReplicationSlotSetInactiveSince

定義

  • 種別: static inline function
  • 定義箇所: src/include/replication/slot.h:246

目的

レプリケーションスロットが最後にアクティブでなくなった時刻（inactive_since）を更新します。

シグネチャまたは形状

static inline void ReplicationSlotSetInactiveSince(ReplicationSlot *s, TimestampTz ts, bool acquire_lock)

入力と出力

  • s: 更新対象の ReplicationSlot 構造体へのポインタ。
  • ts: 設定する時刻（TimestampTz）。
  • acquire_lock: スピンロックを取得・解放するかどうかを指定するフラグ。

前提条件と不変条件

  • スロットが既に無効化されている場合（s->data.invalidated != RS_INVAL_NONE）、時刻の更新は行われません。
  • acquire_lock が true の場合、関数内で s->mutex を使用して排他制御が行われます。

アルゴリズムまたは振る舞い

1. acquire_lock が true の場合、s->mutex を取得します。 2. スロットの data.invalidated フィールドが RS_INVAL_NONE であることを確認します。 3. 条件を満たす場合、s->inactive_since に引数 ts を代入します。 4. acquire_lock が true の場合、s->mutex を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/slotsync.c
  • src/backend/replication/slot.c
  • **呼び出し先**:
  • SpinLockAcquire (マクロ)
  • SpinLockRelease (マクロ)

関連データ構造

  • ReplicationSlot: スロットのメモリ内状態を保持する構造体。
  • TimestampTz: 時刻を表す型。

失敗モードと境界条件

  • スロットが既に無効化されている場合、時刻の更新は無視されます。
  • acquire_lock を false で呼び出す場合、呼び出し元で適切なロック（s->mutex）が既に保持されている必要があります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、頻繁に呼び出されるスロット状態管理のオーバーヘッドを最小限に抑える設計となっています。
  • ロックの取得・解放を呼び出し元で制御できるため、一連の操作をアトミックに行いたい場合に acquire_lock = false を指定して利用されます。
