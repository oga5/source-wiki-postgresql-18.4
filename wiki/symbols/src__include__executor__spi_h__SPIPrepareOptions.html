SPIPrepareOptions

定義

  • 種別: struct
  • 定義箇所: src/include/executor/spi.h:37

目的

SPI（Server Programming Interface）において、SQL文をプリペアドプランとして準備する際の詳細な設定オプションを保持するために使用されます。

シグネチャまたは形状

typedef struct SPIPrepareOptions
{
	ParserSetupHook parserSetup;
	void	   *parserSetupArg;
	RawParseMode parseMode;
	int			cursorOptions;
} SPIPrepareOptions;

入力と出力

  • **入力**: 構造体の各フィールドを通じて、パーサーのフック関数、その引数、解析モード、およびカーソルオプションを指定します。
  • **出力**: 本構造体は設定値のコンテナであり、SPIの準備関数（SPI_prepare_extended 等）へ引数として渡されます。

前提条件と不変条件

  • parserSetup が指定されている場合、parserSetupArg はそのフック関数が期待する形式のデータである必要があります。
  • cursorOptions は、PostgreSQLの実行エンジンが解釈可能なビットフラグである必要があります。

アルゴリズムまたは振る舞い

本構造体は、SQLの解析および計画作成プロセスにおいて、標準的な動作をカスタマイズするためのパラメータを提供します。特に、カスタムパーサーフックや特定の解析モード（RawParseMode）を適用することで、動的なSQL生成や特殊な構文解析を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_exec.c:4177 にてインスタンス化および利用の形跡があります。
  • **呼び出し先**: src/backend/executor/spi.c:903 および src/include/executor/spi.h:137 にて、SPI関連関数の引数として参照されています。

関連データ構造

  • ParserSetupHook: パーサーのセットアップを行う関数ポインタ型。
  • RawParseMode: SQLの解析モードを定義する列挙型。

失敗モードと境界条件

  • 構造体の各フィールドに不正な値（特にNULLポインタや無効なビットフラグ）が設定された場合、後続の SPI_prepare 処理においてセグメンテーションフォールトや解析エラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はSPIの拡張機能の一部です。新しい解析オプションやカーソル制御フラグを追加する場合は、本構造体の定義と、それを利用する spi.c 内の処理の両方を更新してください。
