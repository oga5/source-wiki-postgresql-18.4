CreateDBRelInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/dbcommands.c:104

目的

データベース作成時（CREATE DATABASE）に、テンプレートデータベースからコピー対象となるリレーションの情報を保持するために使用される構造体です。

シグネチャまたは形状

typedef struct CreateDBRelInfo
{
	RelFileLocator rlocator;	/* 物理リレーション識別子 */
	Oid			reloid;			/* リレーションOID */
	bool		permanent;		/* 永続リレーションか、アンログドか */
} CreateDBRelInfo;

入力と出力

  • 本構造体は、主に ScanSourceDatabasePgClassTuple 等の関数によって、システムカタログ pg_class のタプルから情報を抽出・格納するために生成されます。
  • palloc を用いてメモリ確保されることが一般的です。

前提条件と不変条件

  • reloid は有効なリレーションOIDである必要があります。
  • rlocator は物理的なリレーションの特定に必要な情報を含みます。

アルゴリズムまたは振る舞い

  • データベース複製プロセスにおいて、コピーすべきリレーションのリストを構築するために利用されます。
  • テンプレートデータベースの pg_class をスキャンする際、各リレーションに対してこの構造体が生成され、後続のコピー処理（WAL_LOG または FILE_COPY 戦略）に渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ScanSourceDatabasePgClassTuple がこの構造体を生成し、情報を設定します。
  • **呼び出し先**: データベース作成処理のメインロジック（CreateDatabaseUsingWalLog 等）が、この構造体を参照して物理的なコピー操作を実行します。

関連データ構造

  • RelFileLocator: 物理ファイル識別用。
  • HeapTupleData: pg_class から情報を抽出する際のソース。

失敗モードと境界条件

  • メモリ不足時には palloc によりエラーが発生します。
  • テンプレートデータベースのスキャン中に無効なリレーション情報が渡された場合、後続のコピー処理で失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は dbcommands.c 内のデータベース複製ロジックに密結合しています。構造体のフィールドを変更する場合は、ScanSourceDatabasePgClassTuple での初期化処理および、これを利用するコピー戦略の実装をすべて更新する必要があります。
