SS_attach_initplans

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/subselect.c:2352

目的

クエリの最適化プロセスにおいて収集された InitPlan のリストを、実行計画の最上位ノードに付与する。

シグネチャまたは形状

void SS_attach_initplans(PlannerInfo *root, Plan *plan)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体へのポインタ。計画生成中に収集された init_plans を保持する。
  • plan: Plan 構造体へのポインタ。InitPlan を格納する対象の実行計画ノード。
  • **出力**: なし（副作用として plan->initPlan を更新する）。

前提条件と不変条件

  • root->init_plans には、クエリ実行前に評価されるべき SubPlan のリストが格納されている必要がある。
  • plan は有効な Plan ノードを指している必要がある。

アルゴリズムまたは振る舞い

root->init_plans に格納されている InitPlan のリストを、指定された plan ノードの initPlan フィールドに代入する。これにより、実行エンジンはクエリのメイン処理開始前にこれらのサブプランを評価できるようになる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/createplan.c:368
  • **呼び出し先**: なし（直接的な関数呼び出しは行わない）。

関連データ構造

  • PlannerInfo: 計画生成中の状態を保持する構造体。
  • Plan: 実行計画のノード構造体。initPlan フィールドを通じて InitPlan を保持する。

失敗モードと境界条件

  • root->init_plans が空（NULL）の場合、plan->initPlan にも NULL が代入される。これは正常な挙動である。

保守担当者向けメモ

  • 本関数は、計画生成の最終段階で InitPlan を計画ツリーに統合するために使用される。
  • SS_charge_for_initplans や SS_compute_initplan_cost とは異なり、コスト計算ではなく構造的なアタッチメントのみを行う。
