NamedLWLockTranche

定義

  • 種別: struct
  • 定義箇所: src/include/storage/lwlock.h:77

目的

LWLock（軽量ロック）のトランチ（Tranche: ロックのグループ）に対して、動的に割り当てられた識別子と名前を関連付けるための構造体です。

シグネチャまたは形状

typedef struct NamedLWLockTranche
{
    int         trancheId;    /* トランチの識別子 */
    char       *trancheName;  /* トランチの名前 */
} NamedLWLockTranche;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではないため直接的な入出力はありません。
  • src/backend/storage/lmgr/lwlock.c において、NamedLWLockTrancheArray として配列形式で管理されます。

前提条件と不変条件

  • trancheId は、システム内で一意である必要があります。
  • trancheName は、デバッグや統計情報で識別可能な文字列を指す必要があります。

アルゴリズムまたは振る舞い

  • NamedLWLockTrancheArray を介して、実行時に動的に定義されたトランチのメタデータを保持します。
  • ロックの初期化や統計情報の収集時に、この構造体を通じてトランチ名が参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/lwlock.c 内のロック初期化処理において、トランチ配列のサイズ計算およびメモリ確保に使用されます。
  • **呼び出し先**: 情報は提供されていません。

関連データ構造

  • NamedLWLockTrancheArray: 本構造体の配列を指すグローバル変数。
  • LWLock: 本構造体で定義された trancheId を保持し、ロックの所属先を特定します。

失敗モードと境界条件

  • NamedLWLockTrancheArray が NULL の場合、トランチ情報の参照は失敗します。
  • メモリ確保時、NamedLWLockTrancheRequests の数に応じて適切なサイズが確保されない場合、バッファオーバーフローや不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は lwlock.c で管理される動的トランチの基盤です。新しいトランチを追加する際は、NamedLWLockTrancheArray の初期化ロジックと整合性を取る必要があります。
