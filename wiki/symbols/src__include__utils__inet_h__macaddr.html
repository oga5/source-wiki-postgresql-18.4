macaddr

定義

  • 種別: struct
  • 定義箇所: src/include/utils/inet.h:94

目的

PostgreSQLにおいて、MACアドレス（Media Access Control address）を格納するための固定長データ構造を提供します。

シグネチャまたは形状

typedef struct macaddr
{
	unsigned char a;
	unsigned char b;
	unsigned char c;
	unsigned char d;
	unsigned char e;
	unsigned char f;
} macaddr;

入力と出力

  • 本構造体は固定長であり、Datum との相互変換には DatumGetMacaddrP や MacaddrPGetDatum マクロが使用されます。

前提条件と不変条件

  • 固定長データ型であり、inet 型のような可変長ヘッダー（varlena）は持ちません。
  • 6バイトの符号なし文字型（unsigned char）で構成され、MACアドレスの各オクテットを保持します。

アルゴリズムまたは振る舞い

  • データベースのタプルとしてネットワークアドレスを格納・処理するために使用されます。
  • src/backend/utils/adt/mac.c 等の関数群により、比較やソート、変換操作が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • contrib/btree_gin/btree_gin.c: palloc0 によるメモリ確保。
  • contrib/btree_gist/btree_macaddr.c: 比較演算や mac_2_uint64 変換での利用。
  • src/backend/access/brin/brin_minmax_multi.c: PG_GETARG_MACADDR_P を介した引数取得。
  • **呼び出し先**:
  • 提供されたソース情報には、本構造体が直接呼び出す関数やシンボルの情報は含まれていません。

関連データ構造

  • macaddr8: 8バイト版のMACアドレス構造体。
  • inet: ネットワークアドレスを扱うための可変長構造体。

失敗モードと境界条件

  • 構造体自体は固定長であるため、メモリレイアウト上の境界条件は単純ですが、ポインタ経由でアクセスする際は PG_GETARG_MACADDR_P 等の適切なインターフェースを通す必要があります。

保守担当者向けメモ

  • 本構造体は inet.h で定義されており、ネットワーク関連のデータ型操作の基盤となります。
  • 変更を加える際は、src/backend/utils/adt/mac.c にあるソートサポートや比較関数への影響を確認してください。
