ValidIOData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/misc.c:54

目的

pg_input_is_valid 系の関数において、型変換処理に必要な情報をキャッシュし、再利用するために使用される構造体です。

シグネチャまたは形状

typedef struct ValidIOData
{
    Oid         typoid;           /* 対象の型ID */
    int32       typmod;           /* 型修飾子 */
    bool        typname_constant; /* 型名が定数かどうか */
    Oid         typiofunc;        /* 入力関数ID */
    Oid         typioparam;       /* 入力関数のパラメータID */
    FmgrInfo    inputproc;        /* 関数マネージャ情報 */
} ValidIOData;

入力と出力

本構造体は関数呼び出し間で状態を保持するためのコンテナとして機能します。入力として各フィールドの初期化値を受け取り、出力として FmgrInfo を含む型変換に必要なメタデータを保持します。

前提条件と不変条件

  • fcinfo->flinfo->fn_extra を介してキャッシュとして利用されます。
  • 使用前に fn_extra が NULL であるかを確認し、初期化されていない場合は構造体の割り当てと設定を行う必要があります。

アルゴリズムまたは振る舞い

1. pg_input_is_valid_common 等の関数内で、fn_extra が指すメモリ領域として利用されます。 2. 初回呼び出し時に ValidIOData 構造体が割り当てられ、型情報や入力関数情報が格納されます。 3. 以降の呼び出しでは、キャッシュされた ValidIOData を参照することで、型情報の再検索コストを削減します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/misc.c 内の pg_input_is_valid 関連関数。
  • 呼び出し先: FmgrInfo を介して、PostgreSQL の関数マネージャ（fmgr）が呼び出されます。

関連データ構造

  • FmgrInfo: 関数呼び出しに必要な情報を保持する構造体。
  • fcinfo: 関数呼び出しのコンテキスト情報。

失敗モードと境界条件

  • メモリ割り当て失敗時（palloc 失敗時）には例外が発生します。
  • fn_extra が正しく初期化されていない状態でアクセスすると、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は pg_input_is_valid 系の関数におけるパフォーマンス最適化の要です。フィールドを追加・変更する場合は、pg_input_is_valid_common における初期化ロジックと整合性を取ってください。
