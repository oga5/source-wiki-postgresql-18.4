SHA512_Last

定義

  • 種別: function
  • 定義箇所: src/common/sha2.c:854

目的

SHA-512 ハッシュ計算の最終処理として、メッセージのパディングとビット長情報の埋め込みを行い、最後のブロック変換を実行する。

シグネチャまたは形状

static void SHA512_Last(pg_sha512_ctx *context)

入力と出力

  • **入力**: pg_sha512_ctx *context（ハッシュ計算のコンテキスト構造体）
  • **出力**: なし（副作用として context 内のバッファおよびハッシュ状態を更新）

前提条件と不変条件

  • context は初期化済みであり、データが更新されていること。
  • WORDS_BIGENDIAN が未定義の場合、内部で REVERSE64 マクロを使用してビットカウントのバイト順序を変換する。

アルゴリズムまたは振る舞い

1. **ビットカウントの調整**: context->bitcount を必要に応じてバイト順序変換する。 2. **パディング処理**:

  • 未使用領域に 0x80 を付与する。
  • ブロック境界に応じてゼロ埋めを行う。
  • ブロック長を超える場合は、中間的な SHA512_Transform を呼び出す。

    • 3. **長さ情報の埋め込み**: バッファの末尾に元の入力データのビット長を格納する。 4. **最終変換**: SHA512_Transform を呼び出し、最終的なダイジェストを確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/sha2.c 内の pg_sha512_final 等から呼び出される（直接の呼び出し元関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: SHA512_Transform を呼び出す。

関連データ構造

  • pg_sha512_ctx: SHA-512 の計算状態を保持する構造体（sha2_int.h で定義）。

失敗モードと境界条件

  • usedspace（バッファ内の使用済み領域）がブロック長に近い場合、パディングのために追加の SHA512_Transform が実行される。
  • メモリ操作は PG_SHA512_BLOCK_LENGTH および PG_SHA512_SHORT_BLOCK_LENGTH に基づいて安全に行われる。

保守担当者向けメモ

  • 本関数は pg_sha512_final の内部処理の一部として機能する。
  • エンディアン依存の処理が含まれているため、移植時には REVERSE64 マクロの定義を確認すること。
