datapagemap_next

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/datapagemap.c:86

目的

ビットマップデータ構造から、変更済みとして記録された次のブロック番号を順次取得する。

シグネチャまたは形状

bool datapagemap_next(datapagemap_iterator_t *iter, BlockNumber *blkno);

入力と出力

  • **入力**:
  • iter: 走査状態を保持する datapagemap_iterator_t 構造体へのポインタ。
  • blkno: 見つかったブロック番号を格納するための BlockNumber 型へのポインタ。
  • **出力**:
  • 戻り値: 次のブロックが見つかった場合は true、ビットマップの終端に達した場合は false を返す。
  • blkno: true が返された場合、発見されたブロック番号が書き込まれる。

前提条件と不変条件

  • iter は datapagemap_iterate() によって初期化されている必要がある。
  • iter->map が指すビットマップ構造体は、走査中に変更されないことが期待される。

アルゴリズムまたは振る舞い

1. イテレータの nextblkno を起点に、ビットマップを順次走査する。 2. 現在のブロック番号からビットマップ上のオフセット（バイト単位）とビット位置を計算する。 3. オフセットがビットマップのサイズを超えた場合、走査を終了し false を返す。 4. ビットが立っている（変更済みである）場合、そのブロック番号を blkno に代入し、イテレータの nextblkno をインクリメントして true を返す。 5. ビットが立っていない場合は、次のブロックへ進み走査を継続する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/datapagemap.c (イテレータ使用箇所)
  • src/bin/pg_rewind/filemap.c:532
  • src/bin/pg_rewind/pg_rewind.c:586
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、iter 構造体のメンバを参照してビット演算を行う。

関連データ構造

  • datapagemap_t: ビットマップ本体を保持する構造体。
  • datapagemap_iterator_t: 走査状態（現在のブロック番号と対象マップ）を保持する構造体。

失敗モードと境界条件

  • ビットマップの末尾に達した場合は false を返し、安全に終了する。
  • iter が指すマップの範囲外へのアクセスは、nextoff >= map->bitmapsize のチェックにより防止される。

保守担当者向けメモ

  • 本関数は datapagemap_iterate() で生成されたイテレータを消費する。イテレータのメモリ解放は呼び出し側で行う必要がある。
  • ビット演算は 8 ビット単位で行われるため、BlockNumber の増分とビットマップのサイズ整合性に注意すること。
