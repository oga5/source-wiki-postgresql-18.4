UnsyncVariables

定義

  • 種別: function
  • 定義箇所: src/bin/psql/command.c:4617

目的

データベース接続に関連する psql のセッション変数をクリアし、接続状態と変数状態の同期を解除します。

シグネチャまたは形状

void UnsyncVariables(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として pset.vars を更新）

前提条件と不変条件

  • グローバル変数 pset が初期化されており、pset.vars が有効な変数リストを保持している必要があります。

アルゴリズムまたは振る舞い

SetVariable 関数を呼び出し、以下の変数を NULL に設定することで削除します。

  • DBNAME, SERVICE, USER, HOST, PORT, ENCODING, SERVER_VERSION_NAME, SERVER_VERSION_NUM

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/psql/command.c:4302
  • src/bin/psql/common.c:386
  • 呼び出し先:
  • SetVariable (変数管理用関数)

関連データ構造

  • pset: psql の設定を保持するグローバル構造体。
  • pset.vars: 変数リストを管理するデータ構造。

失敗モードと境界条件

  • pset.vars が未初期化の場合、SetVariable 内部で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 接続情報に関連する新しい変数を psql に追加した場合は、この関数にも忘れずに削除処理を追加してください。
