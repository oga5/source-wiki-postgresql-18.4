system_time_nextsampletuple

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_time/tsm_system_time.c:289

目的

TABLESAMPLE SYSTEM_TIME メソッドにおいて、現在スキャン中のページ内にある次のタプルオフセットを決定する。

シグネチャまたは形状

static OffsetNumber system_time_nextsampletuple(SampleScanState *node, BlockNumber blockno, OffsetNumber maxoffset)

入力と出力

  • **入力**:
  • node: サンプリングスキャンの状態を保持する SampleScanState 構造体。
  • blockno: 現在処理中のブロック番号。
  • maxoffset: 現在のページにおける有効な最大オフセット番号。
  • **出力**:
  • 次に読み込むべき OffsetNumber。ページ内の全タプルを走査し終えた場合は InvalidOffsetNumber を返す。

前提条件と不変条件

  • node->tsm_state は SystemTimeSamplerData 構造体としてキャスト可能であること。
  • sampler->lt には、前回の呼び出しで返されたオフセット（または初期状態の InvalidOffsetNumber）が保持されていること。

アルゴリズムまたは振る舞い

1. node->tsm_state からサンプラー状態を取得する。 2. sampler->lt（前回のオフセット）を確認し、初回呼び出し時（InvalidOffsetNumber）は FirstOffsetNumber を、それ以外は +1 して次のオフセットを算出する。 3. 算出したオフセットが maxoffset を超える場合、走査終了とみなして InvalidOffsetNumber を設定する。 4. 更新されたオフセットを sampler->lt に保存し、その値を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のテーブルサンプリング実行エンジン（tsmapi.h のインターフェース経由）。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • SampleScanState: スキャン全体の実行状態を保持する。
  • SystemTimeSamplerData: tsm_system_time 固有のサンプリング状態（lt フィールドで現在のオフセットを管理）。

失敗モードと境界条件

  • maxoffset が 0 の場合、直ちに InvalidOffsetNumber を返す。
  • ページ内の全タプルを走査し終えた場合、適切に InvalidOffsetNumber を返すことでスキャンループの終了を通知する。

保守担当者向けメモ

  • 本関数はページ内のタプルを順次走査する役割を持つ。ブロックの切り替えは system_time_nextsampleblock が担当する。
  • sampler->lt の状態管理がスキャンの継続性を保証しているため、この変数の更新ロジックを変更する際は注意が必要である。
