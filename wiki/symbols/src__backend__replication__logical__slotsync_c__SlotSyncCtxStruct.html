SlotSyncCtxStruct

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/slotsync.c:99

目的

論理レプリケーションスロットの同期プロセスにおける共有メモリ上の状態を保持・管理するために使用されます。

シグネチャまたは形状

typedef struct SlotSyncCtxStruct
{
	pid_t		pid;            /* 同期プロセスのプロセスID */
	bool		stopSignaled;   /* 停止要求フラグ */
	bool		syncing;        /* 同期実行中フラグ */
	time_t		last_start_time;/* 前回の開始時刻 */
	slock_t		mutex;          /* 状態保護用スピンロック */
} SlotSyncCtxStruct;

入力と出力

  • 本構造体自体はデータ保持用であり、直接的な入出力関数を持ちません。
  • 共有メモリ上のインスタンスとして、同期プロセスの状態を外部（SQL関数やバックグラウンドワーカー）へ公開します。

前提条件と不変条件

  • 構造体内の各フィールドへのアクセスは、mutex フィールドを用いたスピンロックによって保護される必要があります。
  • syncing フラグは、現在のプロセスが同期処理を実行中である場合にのみ true となります。

アルゴリズムまたは振る舞い

  • 同期プロセスのライフサイクル管理（起動、停止要求の伝達、実行状態の追跡）をサポートします。
  • 複数の同期プロセスが競合しないよう、共有メモリ上で単一のインスタンスとして管理されます。

呼び出し元と呼び出し先

  • 呼び出し元: SlotSyncShmemSize（サイズ計算）、SlotSyncShmemInit（初期化）、および同期ワーカーのメインループ。
  • 呼び出し先: 本構造体はデータ構造であり、特定の関数を呼び出すことはありません。

関連データ構造

  • SlotSyncCtx: SlotSyncCtxStruct 型の静的ポインタ変数。共有メモリ上の実体を指します。

失敗モードと境界条件

  • 共有メモリの初期化前や、同期プロセスが異常終了した場合の整合性については、mutex によるロック制御に依存します。
  • 構造体サイズは SlotSyncShmemSize で計算され、共有メモリ確保時に使用されます。

保守担当者向けメモ

  • 本構造体は src/backend/replication/logical/slotsync.c 内でのみ使用される内部的な状態管理用です。
  • 変更を加える際は、共有メモリのレイアウトおよび SlotSyncShmemSize の計算式に影響がないか確認してください。
