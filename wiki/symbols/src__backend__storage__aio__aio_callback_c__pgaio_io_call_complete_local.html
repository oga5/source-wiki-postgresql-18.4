pgaio_io_call_complete_local

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_callback.c:284

目的

非同期I/O（AIO）操作の完了フェーズにおいて、ローカルバックエンド固有の処理を実行し、結果を最終的に加工する。

シグネチャまたは形状

PgAioResult pgaio_io_call_complete_local(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh（対象のI/Oハンドル）
  • **出力**: PgAioResult（ローカルコールバックによって加工された最終的なI/O結果）

前提条件と不変条件

  • START_CRIT_SECTION() によりクリティカルセクション内で実行される。
  • ioh->target および ioh->op が有効な範囲内であること。
  • 入力となる ioh->distilled_result.status が PGAIO_RS_UNKNOWN ではないこと。
  • 各コールバック実行後、結果のステータスが PGAIO_RS_UNKNOWN に遷移しないこと。

アルゴリズムまたは振る舞い

1. ioh->distilled_result を初期結果として取得する。 2. ハンドルに登録されたコールバックを、登録順序の逆順（スタック的）に反復処理する。 3. 各コールバックについて、complete_local 関数が定義されていればそれを呼び出し、結果を更新する。 4. 処理終了後、更新された結果を呼び出し元へ返す。この結果は ioh->distilled_result には保存されない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/aio.c:693
  • **呼び出し先**: 登録された各コールバックの complete_local 関数（aio_handle_cbs 配列経由）

関連データ構造

  • PgAioHandle: I/O操作の状態とコールバック情報を保持する構造体。
  • PgAioResult: I/O操作の結果ステータスやエラー情報を保持する構造体。
  • PgAioHandleCallbacksEntry: コールバック関数ポインタを保持するエントリ。
  • aio_handle_cbs: 利用可能な全コールバックを定義する静的配列。

失敗モードと境界条件

  • コールバックが未定義（complete_local が NULL）の場合はスキップされる。
  • 登録されたコールバックが 0 個の場合は、初期結果がそのまま返される。

保守担当者向けメモ

  • 本関数はローカルバックエンドの完了処理に特化しており、共有メモリ上の状態更新は行わない。
  • 実行順序は登録の逆順であるため、後から登録された処理が優先的に結果を加工する。
