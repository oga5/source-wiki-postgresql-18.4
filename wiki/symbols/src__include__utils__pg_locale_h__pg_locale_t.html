pg_locale_t

定義

  • 種別: typedef (構造体 pg_locale_struct へのポインタ)
  • 定義箇所: src/include/utils/pg_locale.h:70

目的

PostgreSQLにおけるロケールおよび照合順序（Collation）処理の抽象化レイヤーを担うオブジェクトです。libc、ICU、および組み込みの各ロケールプロバイダーを統一的に扱い、実行時のポリモーフィズムを実現します。

シグネチャまたは形状

typedef struct pg_locale_struct *pg_locale_t;

入力と出力

  • 入力: pg_newlocale_from_collation(Oid collid) 等のファクトリ関数を通じて生成されます。
  • 出力: 文字列操作関数（pg_strcoll 等）に渡され、プロバイダー固有のロジックへディスパッチするためのコンテキストとして利用されます。

前提条件と不変条件

  • データベースの照合順序プロバイダー（libc/ICU/builtin）に応じたメソッドテーブルが割り当てられている必要があります。
  • collate_is_c や ctype_is_c フラグが true の場合、ロケール処理をスキップする高速パスが適用される不変条件があります。

アルゴリズムまたは振る舞い

  • 実行時にプロバイダーを切り替えるためのポリモーフィズムを提供します。
  • 文字列の大小文字変換、照合（coll）、変換（xfrm）の共通インターフェースを保持します。
  • GUC設定（LC_COLLATE, LC_CTYPE 等）の変更に伴い、再キャッシュされる対象となります。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/ltree/crc32.c
  • contrib/ltree/lquery_op.c
  • src/backend/access/hash/hashfunc.c
  • src/backend/regex/regc_pg_locale.c
  • src/backend/tsearch/ts_locale.c
  • src/backend/tsearch/wparser_def.c
  • src/backend/utils/adt/formatting.c
  • 呼び出し先:
  • collate_methods（メソッドテーブル）を参照し、strncoll_icu や strncoll_libc 等の実装へディスパッチします。

関連データ構造

  • pg_locale_struct: 実体となる構造体。
  • collate_methods: 照合順序の挙動を定義する関数ポインタテーブル。
  • UCollator: ICUライブラリを使用する場合の内部構造体。

失敗モードと境界条件

  • ロケール情報のロード失敗や、無効な Oid からの生成時には適切にエラーハンドリングされる必要があります。
  • ICU使用時は U_SHOW_CPLUSPLUS_API を無効化し、C APIのみを露出させることでC++ヘッダーとの競合を回避しています。

保守担当者向けメモ

  • 本シンボルは単なるデータ保持ではなく、ロケール処理のディスパッチャーとして機能します。
  • パフォーマンス最適化のため、C ロケール判定フラグの整合性に注意してください。
