BTShared

定義

  • 種別: struct
  • 定義箇所: src/backend/access/nbtree/nbtsort.c:93

目的

並列インデックス構築時において、リーダープロセスとワーカープロセス間で共有される状態を保持するための構造体です。インデックス構築のメタ情報および、ワーカーによるスキャン結果の集約に使用されます。

シグネチャまたは形状

typedef struct BTShared {
    Oid         heaprelid;
    Oid         indexrelid;
    bool        isunique;
    bool        nulls_not_distinct;
    bool        isconcurrent;
    int         scantuplesortstates;
    int64       queryid;
    ConditionVariable workersdonecv;
    slock_t     mutex;
    int         nparticipantsdone;
    double      reltuples;
    bool        havedead;
    double      indtuples;
    bool        brokenhotchain;
} BTShared;

入力と出力

  • **入力**: 並列構築の初期化時にリーダーが共有メモリ上に割り当て、各ワーカーが参照します。
  • **出力**: 構築終了時に、ワーカーが収集した統計情報（reltuples, indtuples 等）がリーダーへ報告されます。

前提条件と不変条件

  • mutex フィールドは、heapdesc 以降の可変状態（nparticipantsdone 等）へのアクセスを保護します。
  • workersdonecv は、すべてのワーカーがスキャンを完了したことをリーダーが確認するために使用されます。
  • ソート開始前には、すべての並列参加者が完了通知を行う必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: リーダーが共有メモリを確保し、インデックスの基本属性を格納します。 2. **同期**: ワーカーは自身の作業状況を mutex で保護されたフィールドに書き込みます。 3. **集約**: リーダーは workersdonecv を通じてワーカーの終了を待機し、nparticipantsdone 等の統計情報を集計して最終的なインデックス構築に反映させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtsort.c 内の並列構築関連関数（_bt_parallel_build_main 等）から参照されます。
  • **呼び出し先**: 直接的な関数呼び出しは行いませんが、共有メモリ上のデータ構造として shm_toc_allocate 等を介して管理されます。

関連データ構造

  • BTLeader: 並列構築のリーダー側の状態を管理する構造体。
  • ParallelTableScanDescData: 本構造体の直後に配置される並列スキャン記述子。

失敗モードと境界条件

  • 共有メモリの割り当て失敗時に構築が中断される可能性があります。
  • 複数のワーカーが同時に統計情報を更新する際、mutex による排他制御が正しく行われない場合、統計情報の不整合が発生します。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、ポインタを含めることはできません。
  • ParallelTableScanDescData が直後に続くため、メモリレイアウトの計算には BUFFERALIGN を使用する必要があります。
