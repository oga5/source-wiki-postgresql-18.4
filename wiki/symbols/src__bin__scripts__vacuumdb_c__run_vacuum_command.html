run_vacuum_command

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/vacuumdb.c:1311

目的

PostgreSQL サーバーに対して VACUUM または ANALYZE コマンドを非同期に送信し、実行結果の成否をログ出力する。

シグネチャまたは形状

static void run_vacuum_command(PGconn *conn, const char *sql, bool echo, const char *table)

入力と出力

  • conn: データベース接続ハンドル (PGconn*)。
  • sql: 実行する SQL コマンド文字列。
  • echo: true の場合、実行前に SQL を標準出力に表示する。
  • table: 処理対象のテーブル名（指定がない場合は NULL）。
  • 出力: なし（副作用としてサーバーへのクエリ送信とエラーログ出力を行う）。

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • sql はサーバーが解釈可能な有効な SQL 文であること。

アルゴリズムまたは振る舞い

1. echo が true の場合、sql を標準出力に書き出す。 2. PQsendQuery を使用して、サーバーへ非同期に SQL を送信する。 3. 送信が失敗した場合（PQsendQuery が 1 を返さない場合）、pg_log_error を使用してエラーメッセージを記録する。

  • table が指定されている場合は、テーブル名とデータベース名を含む詳細なエラーを出力する。
  • table が NULL の場合は、データベース全体に対するエラーとして出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/scripts/vacuumdb.c 内の以下の箇所から呼び出される。
  • 740行目: 並列実行時のテーブル処理。
  • 765行目: データベース全体または個別の処理。
  • 呼び出し先:
  • printf (標準ライブラリ)
  • PQsendQuery (libpq)
  • PQdb (libpq)
  • PQerrorMessage (libpq)
  • pg_log_error (内部ユーティリティ)

関連データ構造

  • PGconn: libpq のデータベース接続構造体。

失敗モードと境界条件

  • PQsendQuery が失敗した場合、関数はエラーログを出力するが、プログラムの実行は継続される。
  • table 引数が NULL の場合、ログ出力時にテーブル名を含まない形式でエラーが報告される。

保守担当者向けメモ

  • 本関数は非同期送信 (PQsendQuery) を行うのみであり、クエリの完了待機や結果の取得は行わない。呼び出し側で適切なタイミングで結果を確認する必要がある。
