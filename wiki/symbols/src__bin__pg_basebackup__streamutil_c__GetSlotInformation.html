GetSlotInformation

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/streamutil.c:489

目的

指定された物理レプリケーションスロットの情報をサーバーから取得し、再開位置（LSN）とタイムラインID（TLI）を抽出する。

シグネチャまたは形状

bool GetSlotInformation(PGconn *conn, const char *slot_name,
                        XLogRecPtr *restart_lsn, TimeLineID *restart_tli)

入力と出力

  • **入力**:
  • conn: サーバーへの有効な PGconn 接続。
  • slot_name: 情報を取得するレプリケーションスロット名。
  • restart_lsn: 取得した再開LSNを格納するポインタ（NULL可）。
  • restart_tli: 取得したタイムラインIDを格納するポインタ（NULL可）。
  • **出力**:
  • 成功時に true、失敗時に false を返す。成功時は指定されたポインタに値が書き込まれる。

前提条件と不変条件

  • conn はレプリケーション接続として確立されている必要がある。
  • READ_REPLICATION_SLOT コマンドは物理スロットのみをサポートするため、論理スロットを指定した場合はエラーとなる。

アルゴリズムまたは振る舞い

1. READ_REPLICATION_SLOT <slot_name> コマンドを構築し、サーバーへ送信する。 2. サーバーからの応答が1行3列のタプルであることを検証する。 3. 応答の第1フィールド（スロットタイプ）が "physical" であることを確認する。 4. 第2フィールド（restart_lsn）を XLogRecPtr 形式（%X/%X）からパースし、第3フィールド（restart_tli）を数値に変換する。 5. 取得した値を呼び出し元が指定したポインタに格納する。 6. 処理の過程で発生したエラーは pg_log_error を通じて出力される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_receivewal.c
  • **呼び出し先**: libpq 関数群（PQexec, PQntuples, PQnfields, PQgetisnull, PQgetvalue, PQclear, PQerrorMessage）、PQExpBuffer 関連関数、標準ライブラリ（sscanf, atoll, strcmp）。

関連データ構造

  • PGconn: libpqの接続ハンドル。
  • XLogRecPtr: WALのログシーケンス番号。
  • TimeLineID: タイムライン識別子。
  • PQExpBuffer: コマンド文字列構築用バッファ。

失敗モードと境界条件

  • 接続エラーまたはコマンド実行失敗時に false を返す。
  • スロットが存在しない場合、または物理スロットではない場合にエラーログを出力し false を返す。
  • 応答の形式が期待と異なる場合（行数・列数の不一致）にエラーログを出力し false を返す。
  • LSNのパースに失敗した場合にエラーログを出力し false を返す。

保守担当者向けメモ

  • 本関数は物理スロット専用である。将来的に論理スロットのサポートが必要な場合は、READ_REPLICATION_SLOT の挙動とパースロジックの拡張が必要となる。
  • restart_lsn や restart_tli が NULL の場合、値の代入はスキップされるが、コマンド実行自体は行われる。
