mcelem_array_contained_selec

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/array_selfuncs.c:695

目的

PostgreSQLのクエリプランナにおいて、配列の包含演算子（<@）の選択率を、最頻要素（MCELEM）と要素数ヒストグラムを用いて推定します。

シグネチャまたは形状

static Selectivity mcelem_array_contained_selec(
    Datum *mcelem, int nmcelem,
    float4 *numbers, int nnumbers,
    Datum *array_data, int nitems,
    float4 *hist, int nhist,
    Oid operator, TypeCacheEntry *typentry
)

入力と出力

  • **入力**:
  • mcelem, nmcelem: 最頻要素の配列とその個数。
  • numbers: 統計情報（各要素の頻度、最小/最大頻度、NULL頻度）。
  • array_data, nitems: 比較対象の定数配列データとその要素数。
  • hist, nhist: 要素数ヒストグラム。
  • operator: 評価対象の演算子OID。
  • typentry: 型情報のキャッシュ。
  • **出力**: Selectivity（0.0〜1.0の範囲の推定値）。

前提条件と不変条件

  • numbers 配列は nmcelem + 3 のサイズである必要があります（末尾3セルに最小頻度、最大頻度、NULL頻度が格納されているため）。
  • hist 配列は少なくとも3要素以上である必要があります。
  • 上記条件を満たさない場合、デフォルト値 DEFAULT_CONTAIN_SEL を返します。

アルゴリズムまたは振る舞い

1. **統計情報の抽出**: numbers から最小頻度、NULL頻度、平均要素数を取得します。 2. **頻度計算**: 定数配列の各要素とMCELEMを照合し、出現確率を推定します。MCELEMに含まれない要素は minfreq / 2 を用いて近似します。 3. **確率モデル**: ポアソン分布を用いて、各要素が列に含まれない確率を積算（mult）し、包含関係の確率を補正します。 4. **計算コスト制限**: EFFORT マクロに基づき、計算量が過大になる場合は unique_nitems を削減して近似計算を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_selfuncs.c 内の calc_arraycontsel 等から呼び出されます。
  • **呼び出し先**: element_compare（要素比較）、palloc（メモリ確保）、qsort（ソート）、exp、sqrt 等の数学関数。

関連データ構造

  • ArrayType: 配列の内部表現。
  • TypeCacheEntry: 型固有の比較関数等を含むキャッシュ構造体。
  • pg_statistic: 統計情報が格納されるシステムカタログ。

失敗モードと境界条件

  • 統計情報（numbers や hist）が不足している場合、計算を中断し DEFAULT_CONTAIN_SEL を返します。
  • nitems が0の場合や、計算コストが EFFORT を超える場合は、精度を犠牲にした近似計算に切り替わります。

保守担当者向けメモ

  • 本関数は <@ 演算子特有の補正ロジックを含みます。
  • EFFORT 定数はプランニング時間の制約と推定精度のトレードオフを制御します。
  • 統計情報が更新されていない場合、推定値が不正確になる可能性があるため、ANALYZE の実行状況を確認してください。
