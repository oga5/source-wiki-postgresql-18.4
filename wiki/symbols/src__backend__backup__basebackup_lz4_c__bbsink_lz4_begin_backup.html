bbsink_lz4_begin_backup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_lz4.c:92

目的

LZ4圧縮シンクの初期化を行い、圧縮に必要なバッファの確保および後続シンクへのバックアップ開始通知を行う。

シグネチャまたは形状

static void bbsink_lz4_begin_backup(bbsink *sink)

入力と出力

  • **入力**: bbsink *sink （bbsink_lz4 構造体としてキャストして使用）
  • **出力**: なし（副作用として sink 内のバッファ確保および後続シンクの begin_backup を呼び出す）

前提条件と不変条件

  • sink は bbsink_lz4 型のインスタンスであること。
  • mysink->base.bbs_buffer_length が適切に設定されていること。
  • BLCKSZ が定義されていること。

アルゴリズムまたは振る舞い

1. LZ4F_preferences_t をゼロクリアし、ブロックサイズを LZ4F_max256KB、圧縮レベルを mysink->compresslevel に設定する。 2. mysink->base.bbs_buffer を bbs_buffer_length のサイズで確保する。 3. LZ4F_compressBound を用いて、入力バッファサイズから圧縮後の最大サイズを算出する。 4. 算出されたサイズを BLCKSZ の倍数に切り上げる。 5. 後続のシンク（sink->bbs_next）に対して、算出されたバッファサイズを引数として bbsink_begin_backup を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_lz4_ops 構造体を通じて、バックアップパイプラインの開始時に呼び出される。
  • **呼び出し先**: bbsink_begin_backup（後続のシンクに対して実行）。

関連データ構造

  • bbsink_lz4: 圧縮コンテキスト、設定、バッファを保持する構造体。
  • LZ4F_preferences_t: LZ4フレーム圧縮の設定構造体。

失敗モードと境界条件

  • palloc によるメモリ確保失敗時にエラーが発生する可能性がある。
  • LZ4F_compressBound の計算結果が BLCKSZ の倍数でない場合、切り上げ処理が行われる。

保守担当者向けメモ

  • 本関数は USE_LZ4 が定義されている環境でのみコンパイルされる。
  • 圧縮後のデータサイズは入力より大きくなる可能性があるため、LZ4F_compressBound によるバッファサイズの再計算は必須である。
