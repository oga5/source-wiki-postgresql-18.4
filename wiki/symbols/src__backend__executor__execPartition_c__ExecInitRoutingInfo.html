ExecInitRoutingInfo

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execPartition.c:993

目的

パーティションテーブルへのタプルルーティングにおいて、特定のパーティションに対する ResultRelInfo の初期化および管理を行う。

シグネチャまたは形状

static void ExecInitRoutingInfo(ModifyTableState *mtstate, EState *estate, PartitionTupleRouting *proute, PartitionDispatch dispatch, ResultRelInfo *partRelInfo, int partidx, bool is_borrowed_rel)

入力と出力

  • **入力**:
  • mtstate: 実行中の ModifyTable ノードの状態。
  • estate: クエリ実行状態。
  • proute: パーティションルーティング全体の管理構造体。
  • dispatch: パーティション階層情報。
  • partRelInfo: 初期化対象のパーティションの ResultRelInfo。
  • partidx: パーティションのインデックス。
  • is_borrowed_rel: リレーションが借用されたものかを示すフラグ。
  • **出力**: なし（副作用として partRelInfo の設定および proute の配列更新を行う）。

前提条件と不変条件

  • partRelInfo->ri_BatchSize は初期化後に 1 以上であることが保証される。
  • dispatch->indexes[partidx] は初期化前には -1 である必要がある。
  • proute->memcxt コンテキスト内でメモリ割り当てが行われる。

アルゴリズムまたは振る舞い

1. **メモリコンテキストの切り替え**: proute->memcxt に切り替えて処理を行う。 2. **タプル変換の準備**: 親テーブルとパーティションの行型が異なる場合、ExecGetRootToChildMap を用いて変換マップを確認し、必要であれば ri_PartitionTupleSlot を作成する。 3. **FDWの初期化**: 外部テーブルの場合、BeginForeignInsert を呼び出し、バッチサイズを決定する（サポート外の場合は 1）。 4. **ルーティング配列への登録**: proute->partitions 配列を必要に応じて拡張し、対象の partRelInfo を登録する。 5. **インデックスの更新**: dispatch->indexes[partidx] に割り当てられた rri_index を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execPartition.c 内のルーティング初期化処理から呼び出される（具体的な関数名はソース内の参照として ExecInitPartitionInfo 等が示唆されるが、明示的なコールグラフは提供されていない）。
  • **呼び出し先**: table_slot_create, partRelInfo->ri_FdwRoutine の各メソッド、palloc/repalloc 等。

関連データ構造

  • PartitionTupleRouting: ルーティング状態を保持。
  • ResultRelInfo: パーティションごとの実行時情報。
  • PartitionDispatch: パーティション階層の探索用データ。

失敗モードと境界条件

  • proute->max_partitions が 0 の場合、初期サイズ 8 で配列を確保する。
  • 配列が満杯になった場合、サイズを 2 倍に拡張する。
  • FDW がバッチ挿入をサポートしていない場合、バッチサイズは 1 に固定される。

保守担当者向けメモ

  • 本関数は ResultRelInfo のオンデマンド初期化の一部を担う。
  • ri_PartitionTupleSlot は制約チェック等の後続処理で利用されるため、変換が必要な場合は必ず初期化される必要がある。
