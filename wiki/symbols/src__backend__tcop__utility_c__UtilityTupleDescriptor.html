UtilityTupleDescriptor

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/utility.c:2083

目的

ユーティリティコマンド（parsetree）が実行結果としてタプルを返す場合に、その構造（TupleDesc）を特定して返却する。

シグネチャまたは形状

TupleDesc UtilityTupleDescriptor(Node *parsetree)

入力と出力

  • **入力**: Node *parsetree （解析済みのユーティリティ文の構文木）
  • **出力**: TupleDesc （タプルの構造情報）。タプルを返さないコマンドや、対象が見つからない場合は NULL を返す。

前提条件と不変条件

  • 入力された parsetree は有効なノードである必要がある。
  • FetchStmt や ExecuteStmt の場合、対象となるポータルやプリペアドステートメントが存在しない場合はエラーを発生させず NULL を返す。

アルゴリズムまたは振る舞い

nodeTag(parsetree) に基づき、以下のコマンド種別に応じて結果の TupleDesc を取得する：

  • T_CallStmt: CallStmtResultDesc を呼び出す。
  • T_FetchStmt: MOVE でない場合、指定されたポータルから CreateTupleDescCopy を用いてコピーを生成する。
  • T_ExecuteStmt: FetchPreparedStatement で取得したステートメントから FetchPreparedStatementResultDesc を呼び出す。
  • T_ExplainStmt: ExplainResultDesc を呼び出す。
  • T_VariableShowStmt: GetPGVariableResultDesc を呼び出す。
  • その他: NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/pquery.c:573
  • src/backend/utils/cache/plancache.c:2081
  • **呼び出し先**:
  • CallStmtResultDesc, GetPortalByName, CreateTupleDescCopy, FetchPreparedStatement, FetchPreparedStatementResultDesc, ExplainResultDesc, GetPGVariableResultDesc

関連データ構造

  • TupleDesc: 返却されるタプルの列定義情報。
  • Node: ユーティリティ文の抽象構文木。
  • Portal, PreparedStatement: 実行状態や準備済みステートメントの管理構造。

失敗モードと境界条件

  • FetchStmt において ismove が真の場合、またはポータルが無効な場合は NULL を返す。
  • ExecuteStmt においてプリペアドステートメントが見つからない場合は NULL を返す。
  • 未対応のノードタグが渡された場合は NULL を返す。

保守担当者向けメモ

  • 新しいユーティリティコマンドがタプルを返すようになった場合、本関数の switch 文に新しい case を追加する必要がある。
  • FetchStmt の処理では、ポータルの有効性チェックを呼び出し元に委ねる設計となっている点に注意すること。
