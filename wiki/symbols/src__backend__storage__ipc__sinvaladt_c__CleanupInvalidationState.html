CleanupInvalidationState

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/sinvaladt.c:327

目的

共有メモリ上のキャッシュ無効化セグメントから、終了するバックエンドプロセスの状態をクリーンアップし、アクティブなプロセスリストから当該プロセスを除外する。

シグネチャまたは形状

static void CleanupInvalidationState(int status, Datum arg)

入力と出力

  • status: on_shmem_exit コールバックによって渡される終了ステータス。
  • arg: 共有メモリ上の SISeg 構造体へのポインタ（Datum 型）。
  • 出力: なし（副作用として共有メモリの状態を更新）。

前提条件と不変条件

  • arg は有効な SISeg 構造体を指している必要がある。
  • SInvalWriteLock を排他モードで取得し、共有メモリ上のデータ構造を保護する。
  • 終了するプロセスは、segP->pgprocnos 配列内に必ず存在しなければならない。

アルゴリズムまたは振る舞い

1. SInvalWriteLock を排他取得する。 2. 現在のプロセスの ProcState を取得し、nextLXID を保存する。 3. プロセスの状態をリセットする（procPid = 0, nextMsgNum = 0, resetState = false, signaled = false）。 4. segP->pgprocnos 配列をスキャンし、自身の MyProcNumber を検索する。 5. 見つかった場合、配列の末尾の要素を当該位置に移動し、numProcs をデクリメントして配列から削除する。 6. 配列内にエントリが見つからない場合は PANIC を発生させる。 7. SInvalWriteLock を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: on_shmem_exit を介してプロセス終了時に呼び出される（src/backend/storage/ipc/sinvaladt.c:316）。
  • 呼び出し先: LWLockAcquire, LWLockRelease, DatumGetPointer, elog（PANIC時）。

関連データ構造

  • SISeg: 共有メモリ上のメイン構造体。
  • ProcState: 各バックエンドの進捗状態を保持する構造体。

失敗モードと境界条件

  • segP->pgprocnos 内に MyProcNumber が存在しない場合、PANIC によりプロセスが異常終了する。

保守担当者向けメモ

  • 本関数は on_shmem_exit コールバックとして登録されるため、直接呼び出すことは想定されていない。
  • pgprocnos 配列の要素削除時に末尾の要素で上書きする手法をとっているため、配列内の順序は維持されない。
