shortest

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:203

目的

正規表現エンジンにおいて、指定された範囲内で最短一致（shortest match）を探索するメインループを実行します。

シグネチャまたは形状

static chr *shortest(struct vars *v, struct dfa *d, chr *start, chr *min, chr *max, chr **coldp, int *hitstopp)

入力と出力

  • **入力**:
  • v: 正規表現の実行状態を保持する vars 構造体。
  • d: DFA の実行状態を保持する dfa 構造体。
  • start: マッチング開始位置。
  • min: マッチング終了の最小位置。
  • max: マッチング終了の最大位置。
  • coldp: lastcold を格納するためのポインタ（NULL 可）。
  • hitstopp: 停止条件に到達したか記録するためのポインタ（NULL 可）。
  • **出力**:
  • マッチした文字列の終端位置を指す chr *。マッチ失敗時は NULL を返します。

前提条件と不変条件

  • d->backno >= 0 の場合、バックリファレンスとして dfa_backref を呼び出します。
  • d->cnfa->flags に MATCHALL が含まれる場合、専用の高速パスで処理されます。
  • POSTSTATE フラグを持つ状態セットに到達することがマッチ成功の条件です。

アルゴリズムまたは振る舞い

1. **初期化**: initialize を呼び出し、開始状態セットを作成します。 2. **スキャン**: 入力文字列を 1 文字ずつ進め、GETCOLOR で文字を色に変換します。 3. **遷移**: 現在の状態セットから outs 配列を参照し、遷移先を決定します。キャッシュミス時は miss を呼び出して新しい状態セットを計算します。 4. **終了判定**: POSTSTATE フラグを持つ状態セットに到達し、かつ cp >= realmin を満たした時点でループを終了します。 5. **後処理**: 終端条件（EOL）の確認を行い、最終的なマッチ位置を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は、提供された情報からは特定できません。
  • **呼び出し先**: initialize, dfa_backref, miss, lastcold, GETCOLOR マクロ。

関連データ構造

  • struct vars: 正規表現実行時の変数群。
  • struct dfa: DFA の実行状態。
  • struct sset: DFA の状態セット（キャッシュエントリ）。
  • struct colormap: 文字クラスの定義。

失敗モードと境界条件

  • initialize または miss が NULL を返した場合、マッチ失敗として NULL を返します。
  • max を超えてスキャンが進んだ場合、または POSTSTATE に到達せずに終端に達した場合はマッチ失敗となります。
  • MATCHALL フラグが立っている場合、min および max の制約に基づき早期リターンする場合があります。

保守担当者向けメモ

  • REG_DEBUG が定義されている場合、REG_FTRACE フラグに応じてトレース出力の有無を切り替える二重ループ構造になっています。
  • lacon（先読み/後読み）の評価が必要な場合、miss を通じて再帰的に DFA が呼び出される可能性があるため、スタック深さに注意してください。
  • coldp や hitstopp が NULL でない場合、それぞれ適切な値が代入されます。
