parallel_worker_main_type

定義

  • 種別: typedef (関数ポインタ型)
  • 定義箇所: src/include/access/parallel.h:23

目的

並列クエリ実行において、バックグラウンドワーカープロセスが実行するメイン関数のシグネチャを定義します。

シグネチャまたは形状

typedef void (*parallel_worker_main_type) (dsm_segment *seg, shm_toc *toc);

入力と出力

  • **入力**:
  • dsm_segment *seg: ワーカーがアタッチした動的共有メモリセグメントへのポインタ。
  • shm_toc *toc: 共有メモリ内のデータ構造を検索するためのテーブル・オブ・コンテント（TOC）。
  • **出力**: なし（void）。

前提条件と不変条件

  • ワーカープロセスが起動され、共有メモリセグメントへのアタッチが完了した後に呼び出されます。
  • リーダープロセスが作成した共有メモリ内のコンテキスト（ParallelContext）が有効であることが前提です。

アルゴリズムまたは振る舞い

1. ワーカープロセスは、この型を持つ関数をエントリポイントとして実行を開始します。 2. 引数として渡された seg と toc を使用して、リーダーから共有されたトランザクション状態やスナップショット、実行計画などの情報を取得します。 3. 取得した情報に基づき、ワーカー固有の処理（クエリの断片実行など）を行います。 4. 処理完了後、またはエラー発生時に終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/bgworker.c のバックグラウンドワーカー起動メカニズムから、ParallelWorkerMain を経由して呼び出されます。
  • **呼び出し先**: 実行される具体的な関数（例: ParallelWorkerMain）は、クエリ実行層のロジックを呼び出します。

関連データ構造

  • dsm_segment: 動的共有メモリセグメント。
  • shm_toc: 共有メモリ内のオブジェクト配置を管理するテーブル。
  • InternalParallelWorkers[]: この型を保持する関数ポインタの配列（src/backend/access/transam/parallel.c 内）。

失敗モードと境界条件

  • 共有メモリの初期化に失敗した場合や、toc から必要なデータ構造が見つからない場合、ワーカーはエラーを報告して終了します。
  • エラー発生時は shm_mq を介してリーダーに通知されます。

保守担当者向けメモ

  • 本型は src/include/access/parallel.h で定義されています。
  • EXEC_BACKEND ビルド環境（Windows等）では、関数ポインタを直接渡せないため、LookupParallelWorkerFunction を使用して名前から関数アドレスを解決する仕組みが併用されています。
