NDBOX

定義

  • 種別: struct
  • 定義箇所: contrib/cube/cubedata.h:9

目的

PostgreSQL の cube 拡張モジュールにおいて、多次元ボックス（または点）を表現するための可変長データ構造。

シグネチャまたは形状

typedef struct NDBOX
{
    int32        vl_len_;
    unsigned int header;
    double       x[FLEXIBLE_ARRAY_MEMBER];
} NDBOX;

入力と出力

  • **入力**: PostgreSQL の fmgr インターフェース（PG_GETARG_NDBOX_P 等）を介して Datum から変換されたポインタとして取得される。
  • **出力**: 構造体内の x 配列に格納された double 型の座標値。

前提条件と不変条件

  • header フィールドは直接操作せず、提供されているマクロ（DIM, IS_POINT 等）を使用すること。
  • header のビット 0-7 は次元数、ビット 31 はポイントフラグ（1 の場合、右上座標は左下座標と同一とみなされる）を保持する。
  • CUBE_MAX_DIM は 100 に制限されている。
  • FLEXIBLE_ARRAY_MEMBER を使用した可変長構造体であるため、メモリ確保時には CUBE_SIZE マクロによるサイズ計算が必要。

アルゴリズムまたは振る舞い

  • header のポイントフラグが設定されている場合、右上座標はメモリ上に格納されず、左下座標と同一として扱われる。
  • 座標データは x 配列の先頭から左下座標が並び、ポイントフラグがオフの場合はその後に右上座標が続く。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • contrib/cube/cube.c: cube_cmp_v0, cube_contains_v0, cube_overlap_v0, cube_union_v0, rt_cube_size, g_cube_binary_union, g_cube_leaf_consistent, g_cube_internal_consistent, cube_is_point_internal 等の関数で引数または戻り値として使用。
  • **呼び出し先**:
  • PostgreSQL のメモリ管理関数（palloc0 等）が NDBOX のインスタンス生成時に呼び出される。

関連データ構造

  • header マクロ群（DIM, IS_POINT 等）
  • LL_COORD, UR_COORD（座標アクセサ）

失敗モードと境界条件

  • CUBE_MAX_DIM（100）を超える次元数を扱う際のオーバーフローリスク。
  • header を直接操作した場合のバイナリ互換性の喪失。

保守担当者向けメモ

  • NDBOX は可変長構造体であるため、ポインタ演算やメモリ確保時には必ず cubedata.h で定義されたマクロを使用すること。
  • UR_COORD マクロはポイントフラグの状態に応じて参照先を動的に切り替えるため、実装の際は注意が必要。
