worktable

定義

  • 種別: struct
  • 定義箇所: src/test/modules/worker_spi/worker_spi.c:58

目的

バックグラウンドワーカーが操作対象とするデータベースのスキーマ名とテーブル名を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct worktable
{
	const char *schema;
	const char *name;
} worktable;

入力と出力

  • 入力: schema（スキーマ名）、name（テーブル名）の各文字列ポインタ。
  • 出力: なし（構造体定義）。

前提条件と不変条件

  • schema および name は、有効な文字列へのポインタである必要があります。

アルゴリズムまたは振る舞い

本構造体は、initialize_worker_spi 関数において、対象となるテーブルの存在確認や初期化を行う際の識別子として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • initialize_worker_spi (引数として受け取る)
  • worker_spi_main (ローカル変数としてインスタンス化し、palloc でメモリを確保)
  • 呼び出し先:
  • 情報なし。

*注: ツールが提示した src/backend/executor/nodeWorktablescan.c 等の参照は、PostgreSQLの再帰クエリ処理に関連する同名の概念であり、本モジュール内の worktable 構造体とは直接関係のない別個のシンボルである可能性が高いです。*

関連データ構造

  • なし。

失敗モードと境界条件

  • palloc によるメモリ確保失敗時には、PostgreSQLの標準的なエラーハンドリング（メモリ不足エラー）が発生します。

保守担当者向けメモ

  • 本構造体は worker_spi モジュール専用の定義です。PostgreSQLコアの再帰クエリ用 worktable と混同しないよう注意してください。
