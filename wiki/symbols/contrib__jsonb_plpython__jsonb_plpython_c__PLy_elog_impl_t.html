PLy_elog_impl_t

定義

  • 種別: typedef
  • 定義箇所: contrib/jsonb_plpython/jsonb_plpython.c:19

目的

plpython 拡張モジュールが提供するエラーログ出力関数への関数ポインタ型を定義します。

シグネチャまたは形状

typedef void (*PLy_elog_impl_t) (int elevel, const char *fmt, ...);

入力と出力

  • **入力**:
  • int elevel: PostgreSQL のエラーレベル（ERROR, WARNING 等）。
  • const char *fmt: 書式指定文字列。
  • ...: 可変長引数。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本型は、plpython 本体がロードされた後に解決される関数ポインタを保持するために使用されます。
  • 呼び出し時には、対象の関数が適切に初期化（_PG_init 等での解決）されている必要があります。

アルゴリズムまたは振る舞い

  • この型は、PostgreSQL のエラー報告メカニズムを plpython 経由で間接的に呼び出すためのインターフェースとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/jsonb_plpython/jsonb_plpython.c 内の PLy_elog_impl_p 変数を通じて利用されます。
  • **呼び出し先**: plpython 拡張モジュール内のエラーログ出力関数（PLy_elog 等）を指すことが想定されています。

関連データ構造

  • PLy_elog_impl_p: 本型を使用して定義された静的変数。

失敗モードと境界条件

  • 初期化前に本型を用いた関数ポインタを呼び出した場合、未定義の動作（セグメンテーション違反等）が発生する可能性があります。

保守担当者向けメモ

  • 本モジュールは plpython に動的に依存しているため、_PG_init での関数ポインタ解決が正しく行われているか確認してください。
  • AssertVariableIsOfType を用いて、代入される関数ポインタの型整合性が検証されています。
