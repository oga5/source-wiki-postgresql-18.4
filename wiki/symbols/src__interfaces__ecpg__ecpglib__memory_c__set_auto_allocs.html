set_auto_allocs

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/memory.c:94

目的

スレッドローカルストレージ（TLS）に、ECPGの自動メモリ管理用リスト（auto_mem 構造体）を登録または更新します。

シグネチャまたは形状

static void set_auto_allocs(struct auto_mem *am)

入力と出力

  • **入力**: am - スレッドに関連付ける auto_mem 構造体へのポインタ。NULL を指定することで、現在のスレッドの追跡リストを解除できます。
  • **出力**: なし（void）。

前提条件と不変条件

  • auto_mem_key が初期化済みである必要があります。
  • 本関数は pthread_setspecific を利用するため、スレッド環境下で実行されることを前提としています。

アルゴリズムまたは振る舞い

1. pthread_setspecific を呼び出し、グローバルな auto_mem_key に引数 am を関連付けます。 2. これにより、以降そのスレッド内で pthread_getspecific(auto_mem_key) を呼び出すと、指定した am が取得可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/memory.c 内のメモリ管理ロジック（ecpg_auto_alloc や ECPGfree_auto_mem 等の内部処理）から呼び出されます。
  • **呼び出し先**:
  • pthread_setspecific（外部ライブラリ関数）。

関連データ構造

  • struct auto_mem: 自動解放対象のメモリブロックを保持するリスト構造体。
  • auto_mem_key: スレッドローカルストレージを識別するためのキー。

失敗モードと境界条件

  • pthread_setspecific が失敗した場合の挙動については、本関数内では明示的なエラーハンドリングを行っていません。

保守担当者向けメモ

  • 本関数は内部的なヘルパー関数であり、スレッドごとのメモリ追跡状態を切り替えるために使用されます。
  • NULL を渡すことで追跡リストをクリアできるため、メモリ管理の所有権移譲やリセット時に利用されます。
