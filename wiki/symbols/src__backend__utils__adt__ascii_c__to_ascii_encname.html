to_ascii_encname

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ascii.c:118

目的

SQL関数 to_ascii のバックエンド実装の一つであり、指定されたエンコーディング名に基づいて入力テキストをASCII近似表現に変換する。

シグネチャまたは形状

Datum to_ascii_encname(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_P_COPY(0): 変換対象の text 型データ。
  • PG_GETARG_NAME(1): エンコーディング名を指定する name 型データ。
  • **出力**:
  • Datum: 変換後の text 型データ。

前提条件と不変条件

  • 入力されたエンコーディング名が pg_char_to_encoding によって有効なエンコーディングとして解決される必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_P_COPY を使用して入力テキストのコピーを取得する。 2. PG_GETARG_NAME を使用してエンコーディング名を取得し、pg_char_to_encoding で内部IDに変換する。 3. エンコーディング名が無効な場合（enc < 0）、ERRCODE_UNDEFINED_OBJECT エラーを発生させる。 4. 有効な場合、encode_to_ascii を呼び出してインプレース変換を実行し、その結果を PG_RETURN_TEXT_P で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（直接的なC言語レベルの呼び出し元はソース内に明示されていない）。
  • **呼び出し先**:
  • pg_char_to_encoding
  • encode_to_ascii
  • ereport

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 指定されたエンコーディング名がシステムでサポートされていない場合、ereport(ERROR) によりトランザクションが中断される。

保守担当者向けメモ

  • 本関数は encode_to_ascii を利用しているため、変換によって文字列長が変化しない前提で動作する。
  • pg_char_to_encoding の結果が負数であるかどうかが唯一のバリデーションポイントである。
