CatalogTupleUpdate

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/indexing.c:312

目的

システムカタログのタプル更新処理を抽象化し、ヒープ更新とそれに伴うインデックスの整合性維持を自動的に実行する。

シグネチャまたは形状

void CatalogTupleUpdate(Relation heapRel, ItemPointer otid, HeapTuple tup)

入力と出力

  • **入力**:
  • heapRel: 更新対象のシステムカタログリレーション記述子。
  • otid: 更新対象タプルの物理位置（ItemPointer）。
  • tup: 更新後の新しいタプルデータ。
  • **出力**: なし（副作用としてカタログおよびインデックスを更新）。

前提条件と不変条件

  • システムカタログに対する操作であることを前提とする。
  • システムカタログでは式インデックス、部分インデックス、除外制約はサポートされない（CatalogIndexInsert 内の Assert により検証される）。

アルゴリズムまたは振る舞い

1. CatalogTupleCheckConstraints を呼び出し、カタログ制約（NOT NULL 等）を検証する。 2. CatalogOpenIndexes を呼び出し、対象リレーションのインデックス情報を取得する。 3. simple_heap_update を実行し、ヒープ上のタプルを更新する。この際、HOT (Heap Only Tuple) 更新の可否が判定され、updateIndexes に反映される。 4. CatalogIndexInsert を呼び出し、updateIndexes の指示に従ってインデックスを更新する。 5. CatalogCloseIndexes を呼び出し、インデックスリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c（権限管理）、src/backend/catalog/heap.c（属性定義更新など）の各関数から参照されている。
  • **呼び出し先**:
  • CatalogTupleCheckConstraints
  • CatalogOpenIndexes
  • simple_heap_update
  • CatalogIndexInsert
  • CatalogCloseIndexes

関連データ構造

  • Relation: 操作対象のリレーション記述子。
  • HeapTuple: 更新対象のタプル。
  • CatalogIndexState: インデックス更新に必要な情報を保持する不透明な構造体。
  • TU_UpdateIndexes: インデックス更新の要否を示すフラグ（TU_All 等）。

失敗モードと境界条件

  • カタログ制約違反が発生した場合、CatalogTupleCheckConstraints によりエラーがスローされる。
  • HOT更新が可能な場合、インデックス更新はスキップされる。

保守担当者向けメモ

  • 複数行を連続して更新する場合は、CatalogTupleUpdate ではなく CatalogTupleUpdateWithInfo を使用することで、インデックスオープン処理のオーバーヘッドを削減できる。
  • 本関数は単一タプルの更新用であり、一括更新には適さない。
