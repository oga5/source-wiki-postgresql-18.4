TwoPhaseLockRecord

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/lmgr/lock.c:158

目的

2フェーズコミット（2PC）において、準備状態（Prepared）のトランザクションが保持するロック情報を永続化または転送するためのデータ構造です。

シグネチャまたは形状

typedef struct TwoPhaseLockRecord
{
	LOCKTAG		locktag;
	LOCKMODE	lockmode;
} TwoPhaseLockRecord;

入力と出力

  • 入力: 構造体メンバとして LOCKTAG（ロック対象の識別子）および LOCKMODE（ロックの強度）を保持します。
  • 出力: 2PCの準備処理時や、準備済みトランザクションの再構築時に、ロック状態をシリアライズ/デシリアライズするためのデータとして利用されます。

前提条件と不変条件

  • sizeof(TwoPhaseLockRecord) は、シリアライズされたデータの読み書き時に、期待されるサイズと一致している必要があります（Assert による検証が行われています）。

アルゴリズムまたは振る舞い

  • 準備済みトランザクションのロック情報を共有メモリから取り出し、ディスクへの書き出しや、再起動後のリカバリ時にロック状態を復元するために使用されます。
  • データの読み込み時には、recdata ポインタを TwoPhaseLockRecord * にキャストしてアクセスします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/lock.c 内の複数の関数（行番号: 3459, 4330, 4511, 4543）で、準備済みトランザクションのロック情報の処理に使用されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • LOCKTAG: ロック対象を特定するためのタグ。
  • LOCKMODE: ロックの競合レベルを定義する列挙型。

失敗モードと境界条件

  • データの読み込み時に、提供されたデータの長さ（len）が sizeof(TwoPhaseLockRecord) と一致しない場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本構造体は 2PC 関連の処理に特化しています。ロックマネージャのメインロジック（LockAcquire 等）とは異なり、トランザクションの準備状態における永続化を目的としている点に注意してください。
