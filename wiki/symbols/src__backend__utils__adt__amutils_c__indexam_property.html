indexam_property

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/amutils.c:150

目的

インデックス・アクセスメソッド（AM）の機能や特性（プロパティ）を、AM、インデックス、または特定のインデックス列のレベルで照会する。

シグネチャまたは形状

static Datum indexam_property(FunctionCallInfo fcinfo, const char *propname, Oid amoid, Oid index_oid, int attno)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • propname: 照会するプロパティ名（文字列）。
  • amoid: AM の OID（インデックス指定時は InvalidOid）。
  • index_oid: インデックスの OID（AM 指定時は InvalidOid）。
  • attno: 列番号（インデックス全体の場合は 0）。
  • **出力**:
  • Datum: プロパティの真偽値（bool）、または該当なし・不明の場合は NULL。

前提条件と不変条件

  • index_oid が有効な場合、amoid は InvalidOid でなければならない。
  • attno は 0 以上、かつ対象インデックスの列数以下である必要がある。
  • amoid または index_oid から取得される IndexAmRoutine が存在しなければならない。

アルゴリズムまたは振る舞い

1. **プロパティ変換**: propname を IndexAMProperty 列挙型に変換する。 2. **コンテキスト特定**: index_oid が指定されている場合、システムカタログから AM の OID と列数を取得する。 3. **AM ルーチン取得**: GetIndexAmRoutineByAmId を呼び出し、対象 AM のルーチン情報を取得する。 4. **AM 優先判定**: AM 固有の amproperty 関数が定義されている場合、その結果を優先して返す。 5. **汎用ロジック**: AM 側で解決されない場合、以下の処理を行う：

  • **列レベル**: pg_index を参照し、test_indoption を用いてソート順（ASC/DESC）や NULL の扱いを判定する。
  • **非キー列**: amcaninclude が有効な場合、キー列外の属性に対してはプロパティを false または NULL として扱う。
  • **その他**: amcanorder や amcanreturn などのフラグに基づき、汎用的な判定を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pg_indexam_has_property
  • pg_index_has_property
  • pg_index_column_has_property
  • **呼び出し先**:
  • lookup_prop_name
  • SearchSysCache1
  • GetIndexAmRoutineByAmId
  • test_indoption
  • index_can_return
  • AM 固有の amproperty 関数（IndexAmRoutine 経由）

関連データ構造

  • IndexAmRoutine: AM の機能フラグやメソッドを保持する構造体。
  • IndexAMProperty: 照会可能なプロパティの列挙型。
  • Form_pg_index, Form_pg_class: システムカタログの行構造。

失敗モードと境界条件

  • index_oid が無効、または pg_class 上でインデックスではない場合、NULL を返す。
  • amoid が無効、または対応する AM ルーチンが見つからない場合、NULL を返す。
  • attno が範囲外の場合、NULL を返す。
  • iskey（キー列か否か）の判定により、一部のプロパティは false を返すか、判定不能として NULL を返す。

保守担当者向けメモ

  • 新しいプロパティを追加する場合は、IndexAMProperty 列挙型と am_propnames 配列の更新が必須である。
  • AM 固有のロジックを優先する設計のため、汎用ロジックはあくまでフォールバックとして機能する。
