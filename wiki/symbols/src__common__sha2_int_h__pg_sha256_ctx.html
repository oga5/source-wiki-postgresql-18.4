pg_sha256_ctx

定義

  • 種別: struct
  • 定義箇所: src/common/sha2_int.h:55

目的

SHA-224およびSHA-256ハッシュ計算アルゴリズムの実行に必要な内部状態（コンテキスト）を保持する。

シグネチャまたは形状

typedef struct pg_sha256_ctx
{
    uint32      state[8];
    uint64      bitcount;
    uint8       buffer[PG_SHA256_BLOCK_LENGTH];
} pg_sha256_ctx;

入力と出力

  • **入力**: state（ハッシュ計算の内部変数）、bitcount（処理済みビット数）、buffer（未処理の入力データブロック）。
  • **出力**: なし（構造体として状態を保持）。

前提条件と不変条件

  • buffer のサイズは PG_SHA256_BLOCK_LENGTH に固定される。
  • state は8つの32ビット符号なし整数で構成される。

アルゴリズムまたは振る舞い

1. **初期化**: pg_sha256_init によりコンテキストがリセットされる。 2. **更新**: pg_sha256_update に入力データが渡され、内部バッファが満たされるごとに SHA256_Transform が呼び出されてハッシュ計算が進行する。 3. **終了**: pg_sha256_final により残りのバッファが処理され、最終的なハッシュ値が生成される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash.c および src/common/sha2.c 内の各関数（pg_sha256_init, pg_sha256_update, pg_sha256_final, SHA256_Transform, SHA256_Last）。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはない。

関連データ構造

  • pg_sha224_ctx: 本構造体のエイリアスとして定義されている。

失敗モードと境界条件

  • 構造体自体に失敗モードはないが、pg_sha256_update 等の操作関数において、バッファオーバーフローや不正なポインタ参照が発生する可能性がある。

保守担当者向けメモ

  • 本構造体は内部実装用であり、外部モジュールから直接利用すべきではない。暗号学的ハッシュが必要な場合は cryptohash.h を介すること。
  • SHA-224 は SHA-256 の構造体を流用している。
  • 著作権表示にある通り、Aaron D. Gifford氏による実装をベースとしている。
