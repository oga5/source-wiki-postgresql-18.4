fix_scan_expr_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/setrefs.c:58

目的

プランツリーの式ツリー（TargetList や Qual）を走査・変異させる際、スキャンノードのコンテキスト情報を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	PlannerInfo *root;
	int			rtoffset;
	double		num_exec;
} fix_scan_expr_context;

入力と出力

本構造体は関数引数として渡されるコンテキストオブジェクトであり、以下のフィールドを持ちます。

  • root: プラン全体の PlannerInfo 構造体へのポインタ。
  • rtoffset: レンジテーブルのオフセット値。
  • num_exec: 実行回数の推定値。

前提条件と不変条件

  • root は有効な PlannerInfo を指している必要があります。
  • 式ツリーの走査中に、このコンテキストを通じて変数の参照解決が行われます。

アルゴリズムまたは振る舞い

fix_scan_expr_mutator および fix_scan_expr_walker が式ツリーを走査する際、この構造体を介して現在のプランのメタデータ（rtoffset 等）を参照し、変数のインデックス変換や依存関係の記録を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/setrefs.c 内の関数（fix_scan_expr_mutator を呼び出す箇所など）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、fix_scan_expr_mutator および fix_scan_expr_walker に引数として渡されます。

関連データ構造

  • PlannerInfo: プラン作成時のグローバル情報。
  • fix_scan_expr_mutator: 本コンテキストを利用して式ツリーを変換する関数。
  • fix_scan_expr_walker: 本コンテキストを利用して式ツリーを走査する関数。

失敗モードと境界条件

  • root が NULL の場合、あるいは rtoffset が不正な値である場合、式ツリーの変換処理が誤ったインデックスを生成する可能性があります。

保守担当者向けメモ

  • 本構造体は setrefs.c における式変換処理の基盤です。フィールドの追加や変更を行う際は、fix_scan_expr_mutator および fix_scan_expr_walker の両方の実装に影響が及ぶことに留意してください。
