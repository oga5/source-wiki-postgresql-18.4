_param

定義

  • 種別: struct
  • 定義箇所: contrib/vacuumlo/vacuumlo.c:41

目的

vacuumlo ユーティリティの実行時オプションおよび接続パラメータを保持するための構造体です。

シグネチャまたは形状

struct _param
{
    char       *pg_user;           /* データベースユーザー名 */
    enum trivalue pg_prompt;       /* パスワードプロンプトの要否 */
    char       *pg_port;           /* 接続ポート番号 */
    char       *pg_host;           /* 接続ホスト名 */
    const char *progname;          /* プログラム名 */
    int         verbose;           /* 詳細出力フラグ */
    int         dry_run;           /* ドライラン（削除なし）フラグ */
    long        transaction_limit; /* トランザクションあたりの削除制限数 */
};

入力と出力

  • **入力**: main 関数にてコマンドライン引数から解析された値が格納されます。
  • **出力**: vacuumlo 関数へ読み取り専用のパラメータとして渡され、データベース接続や削除処理の制御に使用されます。

前提条件と不変条件

  • transaction_limit は、ロックテーブルの枯渇を防ぐために適切な正の整数である必要があります。
  • pg_prompt は trivalue 型の列挙値に従う必要があります。

アルゴリズムまたは振る舞い

本構造体は、プログラムの実行設定をカプセル化します。main 関数で初期化された後、vacuumlo 関数に渡され、接続先情報の指定や、削除処理におけるトランザクションのコミット頻度（transaction_limit）の制御、およびドライランモードの判定に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/vacuumlo/vacuumlo.c:454 (main 関数内でインスタンス化)
  • **呼び出し先**:
  • contrib/vacuumlo/vacuumlo.c:53 および 62 (vacuumlo 関数の引数として参照)

関連データ構造

  • enum trivalue: pg_prompt フィールドで使用される列挙型。

失敗モードと境界条件

  • 構造体の各ポインタフィールド（pg_user, pg_port, pg_host）が NULL の場合、libpq のデフォルト接続設定が適用されます。
  • transaction_limit が不適切な値（0以下など）に設定された場合、削除処理のループ制御に影響を及ぼす可能性があります。

保守担当者向けメモ

  • 本構造体は vacuumlo の設定を集中管理する役割を持つため、新しいコマンドラインオプションを追加する場合はこの構造体にフィールドを追加し、main 関数での解析処理を更新してください。
