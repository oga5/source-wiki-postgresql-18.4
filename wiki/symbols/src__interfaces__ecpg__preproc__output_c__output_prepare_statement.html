output_prepare_statement

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/output.c:167

目的

SQLの PREPARE 文に対応するCコードを生成し、ECPGランタイムライブラリの ECPGprepare 関数を呼び出すコードを出力します。

シグネチャまたは形状

void output_prepare_statement(const char *name, const char *stmt)

入力と出力

  • **入力**:
  • name: プリペアードステートメントの名前（文字列）。
  • stmt: 実行されるSQL文の文字列。
  • **出力**:
  • なし（副作用として base_yyout ストリームへCコードを書き出します）。

前提条件と不変条件

  • グローバル変数 base_yyout が有効なファイルストリームとして初期化されていること。
  • グローバル変数 connection および questionmarks が適切に設定されていること。

アルゴリズムまたは振る舞い

1. base_yyout に対して、ECPGprepare 関数の呼び出しを開始する定型句を出力します。この際、現在の行番号（__LINE__）と接続情報（connection）、および questionmarks フラグを引数として渡します。 2. output_escaped_str を呼び出し、ステートメント名とSQL文をC言語の文字列リテラルとして安全にエスケープして出力します。 3. ECPGprepare の呼び出しを閉じます。 4. whenever_action(2) を呼び出し、エラーハンドリング用のコードを挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の具体的な呼び出し元は、提供された抜粋からは特定できません。
  • **呼び出し先**:
  • output_escaped_str: 文字列のエスケープ処理。
  • whenever_action: エラーハンドリングコードの生成。
  • fprintf, fputs: 標準ライブラリによる出力。

関連データ構造

  • base_yyout: 出力先のファイルストリーム。
  • connection: 接続先を指定するグローバル変数。
  • questionmarks: プレースホルダーの扱いを制御するグローバル変数。

失敗モードと境界条件

  • name や stmt が空文字列の場合、output_escaped_str のエスケープ処理に従って出力されます。
  • connection が NULL の場合、出力されるCコード内では "NULL" として扱われます。

保守担当者向けメモ

  • 本関数は ECPGprepare を生成するため、プリプロセッサが生成するコードの実行時挙動に直接影響します。
  • whenever_action(2) の引数 2 は、このステートメントに対するエラー処理のスコープを制御するフラグです。変更時は whenever_action の仕様を確認してください。
