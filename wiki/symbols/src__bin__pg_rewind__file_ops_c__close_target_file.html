close_target_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:77

目的

現在開かれているターゲットファイル（dstfd）を閉じ、ファイル記述子をリセットします。

シグネチャまたは形状

void close_target_file(void)

入力と出力

  • 入力: なし（グローバル変数 dstfd および dstpath を参照）
  • 出力: なし（成功時）

前提条件と不変条件

  • dstfd が有効なファイル記述子であるか、または -1 である必要があります。
  • dstpath には現在開いているファイルのパスが格納されている必要があります。

アルゴリズムまたは振る舞い

1. dstfd が -1 の場合は何もせず終了します。 2. close(dstfd) を呼び出し、OS レベルでファイルを閉じます。 3. close が失敗した場合、pg_fatal を呼び出してエラーメッセージを出力し、プロセスを終了します。 4. 成功した場合、dstfd を -1 に更新してクローズ済みであることを示します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_rewind/file_ops.c
  • src/bin/pg_rewind/pg_rewind.c
  • 呼び出し先:
  • close (システムコール)
  • pg_fatal (エラーハンドリング)

関連データ構造

  • dstfd (ファイル記述子を保持するグローバル変数)
  • dstpath (ファイルパスを保持するグローバル変数)

失敗モードと境界条件

  • close システムコールが失敗した場合、pg_fatal によりプログラムが異常終了します。
  • dstfd が既に -1 の場合は安全に無視されます。

保守担当者向けメモ

  • 本関数はグローバルな状態（dstfd, dstpath）に依存しています。マルチスレッド環境での利用は想定されていません。
  • pg_fatal を使用しているため、呼び出し元でエラーを捕捉して回復することはできません。
