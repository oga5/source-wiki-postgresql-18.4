mbuf_create_from_data

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/mbuf.c:130

目的

既存のメモリ領域をラップし、読み取り専用の MBuf 構造体を生成する。

シグネチャまたは形状

MBuf *mbuf_create_from_data(uint8 *data, int len)

入力と出力

  • **入力**:
  • data: ラップ対象のデータポインタ。
  • len: データの長さ（バイト単位）。
  • **出力**:
  • 初期化された MBuf 構造体へのポインタ。

前提条件と不変条件

  • MBuf 構造体は palloc によりメモリ確保される。
  • 生成された MBuf は書き込み不可（no_write = true）として設定される。
  • MBuf は渡された data の所有権を持たない（own_data = false）。

アルゴリズムまたは振る舞い

1. palloc を使用して MBuf 構造体用のメモリを確保する。 2. mbuf->data を入力 data で初期化する。 3. buf_end および data_end を data + len に設定する。 4. read_pos を data の先頭に設定する。 5. 書き込み禁止フラグを立て、データ所有権フラグをオフにする。 6. 初期化済みの MBuf ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c 内の関数（pgp_sym_encrypt 等の文脈で利用）。
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理関数)

関連データ構造

  • MBuf: データを保持する可変長バッファ構造体。

失敗モードと境界条件

  • palloc がメモリ不足により失敗した場合、PostgreSQL のエラーハンドリングにより処理が中断される。
  • len が負の値である場合の挙動は定義されていない。

保守担当者向けメモ

  • 本関数は既存のメモリを直接参照するため、MBuf の生存期間中に元の data が解放されないよう注意が必要である。
  • own_data = false であるため、mbuf_free を呼び出しても元の data バッファは解放されない。
