finalize_primnode_context

定義

  • **種別**: struct
  • **定義箇所**: src/backend/optimizer/plan/subselect.c:54

目的

計画生成の最終段階において、式ツリー内のノードを走査し、外部パラメータ（PARAM_EXEC）の依存関係を収集するためのコンテキスト情報を保持します。

シグネチャまたは形状

typedef struct finalize_primnode_context
{
    PlannerInfo *root;        /* プランナーのグローバル情報 */
    Bitmapset  *paramids;     /* 検出された非ローカルな PARAM_EXEC ID の集合 */
} finalize_primnode_context;

入力と出力

本構造体は、finalize_primnode および finalize_agg_primnode 関数へコンテキストとして渡されます。

  • **入力**: root（プランナー情報へのポインタ）
  • **出力**: paramids（走査中に特定されたパラメータIDがビットマップとして蓄積される）

前提条件と不変条件

  • root は有効な PlannerInfo 構造体を指している必要があります。
  • paramids は、走査開始前に初期化（または NULL）されている必要があります。

アルゴリズムまたは振る舞い

この構造体は、式ツリーを再帰的に探索するウォーカー関数（finalize_primnode 等）の引数として使用されます。走査中に PARAM_EXEC 型のパラメータが見つかると、そのIDが paramids ビットマップに追加されます。これにより、計画ノードがどの外部パラメータに依存しているかを特定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/subselect.c 内の各箇所でインスタンス化されます（例: finalize_primnode を呼び出す直前の関数群）。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。本構造体を引数として受け取る関数として finalize_primnode および finalize_agg_primnode が存在します。

関連データ構造

  • PlannerInfo: プランナーの全体的な状態を保持する構造体。
  • Bitmapset: パラメータIDの集合を効率的に管理するために使用されます。
  • Node: 走査対象となる式ツリーのノード。

失敗モードと境界条件

  • root が NULL の場合、参照時にクラッシュする可能性があります。
  • paramids が適切に管理されていない場合、メモリリークや不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は SS_finalize_plan の処理過程で利用されます。パラメータの依存関係を追跡する重要な役割を担っているため、PARAM_EXEC の扱いを変更する際は、このコンテキストが正しく更新されているか確認してください。
