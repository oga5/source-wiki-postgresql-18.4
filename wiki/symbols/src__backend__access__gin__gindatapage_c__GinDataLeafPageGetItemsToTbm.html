GinDataLeafPageGetItemsToTbm

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gindatapage.c:181

目的

GINポスティングツリーのリーフページからTID（Tuple Identifier）を抽出し、指定されたビットマップ（TIDBitmap）に追加する。

シグネチャまたは形状

int GinDataLeafPageGetItemsToTbm(Page page, TIDBitmap *tbm)

入力と出力

  • **入力**:
  • page: 走査対象のGINデータリーフページ。
  • tbm: TIDを格納するためのビットマップ構造体。
  • **出力**:
  • 戻り値: ページから取得・処理されたTIDの総数（int）。

前提条件と不変条件

  • page は有効なGINデータリーフページであること。
  • tbm は初期化済みであること。

アルゴリズムまたは振る舞い

1. ページが圧縮されているか判定（GinPageIsCompressed）。 2. **圧縮されている場合**:

  • ページ内のポスティングリストセグメントを取得し、ginPostingListDecodeAllSegmentsToTbm を呼び出して全セグメントをデコードし、tbm に追加する。

    • 3. **圧縮されていない場合**:

  • dataLeafPageGetUncompressed を呼び出し、非圧縮状態のTID配列と個数を取得する。
  • TIDが存在する場合、tbm_add_tuples を使用してビットマップへ追加する。

    • 4. 処理したTIDの総数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginget.c:93
  • **呼び出し先**:
  • GinPageIsCompressed
  • GinDataLeafPageGetPostingList
  • GinDataLeafPageGetPostingListSize
  • ginPostingListDecodeAllSegmentsToTbm
  • dataLeafPageGetUncompressed
  • tbm_add_tuples

関連データ構造

  • Page: GINインデックスの物理ページ。
  • TIDBitmap: 抽出したTIDを保持するビットマップ。
  • GinPostingList: 圧縮されたTIDリストのデータ構造。

失敗モードと境界条件

  • ページ内にTIDが存在しない場合、戻り値は0となり、ビットマップへの追加は行われない。
  • 圧縮・非圧縮のいずれの形式であっても、ページ内のデータ構造が破損している場合は予期せぬ動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は ginget.c におけるスキャン処理の主要なデータ取得経路である。
  • 圧縮形式の変更や新しいセグメント処理の追加を行う際は、ginPostingListDecodeAllSegmentsToTbm の挙動と整合性を保つ必要がある。
