定義

  • 定義箇所: src/bin/pg_rewind/pg_rewind.c:120

目的

pg_rewind プログラムのメインエントリポイントであり、コマンドライン引数の解析、実行環境の検証、および同期処理の初期化を行う。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**: コマンドライン引数 (argc, argv)。ターゲットデータディレクトリ、ソースデータディレクトリまたは接続文字列、および各種オプションフラグ（--dry-run, --write-recovery-conf 等）を受け取る。
  • **出力**: 正常終了時に 0 を返し、エラー発生時や不正な引数指定時には 1 を返して終了する。

前提条件と不変条件

  • 実行ユーザーが root であってはならない（WIN32 以外）。
  • ターゲットデータディレクトリ（--target-pgdata）の指定が必須。
  • ソースデータディレクトリ（--source-pgdata）またはソースサーバー接続文字列（--source-server）のいずれか一方が必須であり、両方の指定は不可。
  • --write-recovery-conf を指定する場合、--source-server の指定が必須。

アルゴリズムまたは振る舞い

1. **初期化**: ロケール設定、プログラム名の取得、ログ初期化を行う。 2. **引数解析**: getopt_long を使用してオプションを解析し、グローバル変数に状態を格納する。 3. **バリデーション**: 必須引数の有無、排他オプションの競合、実行ユーザー権限をチェックする。 4. **環境設定**: get_restricted_token による権限制限、GetDataDirectoryCreatePerm によるターゲットディレクトリのパーミッション確認、および umask の設定を行う。 5. **リカバリ設定**: getRestoreCommand を呼び出し、リカバリコマンドの準備を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（プログラムのエントリポイント）。
  • **呼び出し先**:
  • usage, exit, getopt_long, pg_logging_init, set_pglocale_pgservice, get_progname, pg_strdup, parse_sync_method, pg_log_error, pg_log_error_hint, geteuid, get_restricted_token, GetDataDirectoryCreatePerm, umask, getRestoreCommand。
  • ※注: ツールによる解析では、contrib/amcheck 等の他の main 関数との混同が見られるため、本シンボルの具体的な呼び出し元・呼び出し先はソースコードの文脈に依存する。

関連データ構造

  • struct option long_options[]: コマンドライン引数の定義。
  • filemap_t: 同期計画を保持する構造体（本関数内で宣言）。

失敗モードと境界条件

  • 引数が不足または過剰な場合、エラーメッセージを表示して終了する。
  • root ユーザーで実行された場合、エラーを表示して終了する。
  • 必須の引数（--target-pgdata 等）が欠落している場合、エラーを表示して終了する。
  • sync-method の解析に失敗した場合、終了する。

保守担当者向けメモ

  • 本関数はプログラムの初期化を担当する。同期処理の核心部分は、この後の処理（perform_rewind 等）に引き継がれる。
  • 新しいコマンドラインオプションを追加する場合は、long_options 配列と getopt_long の switch 文を更新すること。
