ExecReScanMemoize

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMemoize.c:1139

目的

Memoize ノードの再スキャン処理を実行し、キャッシュ状態のリセットおよび必要に応じたキャッシュの全破棄を行う。

シグネチャまたは形状

void ExecReScanMemoize(MemoizeState *node)

入力と出力

  • **入力**: MemoizeState *node （再スキャン対象の Memoize 実行ノード状態）
  • **出力**: なし（副作用として node の内部状態を更新）

前提条件と不変条件

  • node は初期化済みであること。
  • outerPlanState(node) を介してサブプランの状態にアクセス可能であること。

アルゴリズムまたは振る舞い

1. **状態リセット**: ノードの状態 mstatus を MEMO_CACHE_LOOKUP に設定し、前回のスキャンで使用した entry および last_tuple ポインタを NULL に初期化する。 2. **サブプランの再スキャン**: サブプランの chgParam が NULL の場合、ExecReScan を呼び出してサブプランを再スキャンする。 3. **キャッシュの妥当性検証**: キャッシュキーに含まれないパラメータが変更された場合（outerPlan->chgParam と node->keyparamids の差分がある場合）、cache_purge_all(node) を呼び出し、キャッシュ全体を破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:258 （ExecReScan 経由での呼び出しが確認されている）
  • **呼び出し先**:
  • outerPlanState(node)
  • ExecReScan(outerPlan)
  • cache_purge_all(node)
  • bms_nonempty_difference(...)

関連データ構造

  • MemoizeState: ノードの実行状態を保持する主要構造体。
  • PlanState: サブプランの実行状態。

失敗モードと境界条件

  • outerPlan->chgParam が NULL の場合、サブプランの再スキャンは条件付きで実行される。
  • キャッシュキー以外のパラメータ変更が検出された場合、キャッシュの全破棄が発生し、次回のスキャン時にキャッシュの再充填が必要となる。

保守担当者向けメモ

  • 本関数は Memoize ノードのライフサイクル管理の一部であり、ExecReScan インターフェースを通じて呼び出される。
  • キャッシュのパージ条件は keyparamids に依存しているため、キャッシュキーの定義変更時には本関数のロジックも影響を受ける可能性がある。
