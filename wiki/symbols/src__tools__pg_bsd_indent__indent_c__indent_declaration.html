indent_declaration

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/indent.c:1242

目的

宣言文において、識別子の位置を揃えるためのインデント計算を行い、コードバッファ (e_code) へ適切なタブまたは空白を挿入する。

シグネチャまたは形状

static void indent_declaration(int cur_dec_ind, int tabs_to_var)

入力と出力

  • **入力**:
  • cur_dec_ind: 識別子を配置すべき目標のインデント位置。
  • tabs_to_var: タブ文字の使用を許可するかどうかを示すフラグ。
  • **出力**: なし（副作用としてグローバルなコードバッファ e_code を更新）。

前提条件と不変条件

  • グローバル変数 ps (parser_state) が初期化されており、現在のインデントレベル (ps.ind_level) が有効であること。
  • e_code および s_code が有効なコードバッファを指していること。

アルゴリズムまたは振る舞い

1. **オフセット調整**: 現在のインデントレベルがタブサイズの倍数でない場合、その端数分を pos および cur_dec_ind に加算する。 2. **タブ挿入**: tabs_to_var が真の場合、tabsize を用いて可能な限りタブ文字 (\t) を挿入する。この際、postgres_tab_rules に基づき、特定の条件下ではタブの代わりに空白が選択される。 3. **空白補完**: 残りの必要なインデント幅を埋めるために、空白文字 (' ') を e_code に順次追加する。 4. **空白の補完**: バッファが空で、かつ ps.want_blank が真である場合、先頭に空白を1つ追加し ps.want_blank を偽に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/indent.c 内の各所（547, 628, 732, 974, 1026 行目付近）。
  • **呼び出し先**: CHECK_SIZE_CODE マクロ（バッファサイズ確認用）。

関連データ構造

  • parser_state (ps): インデントレベルや状態を保持するグローバル構造体。
  • e_code, s_code: コードバッファの現在位置および開始位置を示すポインタ。

失敗モードと境界条件

  • CHECK_SIZE_CODE マクロにより、バッファ溢れが発生する場合は内部的にエラー処理またはメモリ拡張が行われる。
  • cur_dec_ind が現在の pos より小さい場合、ループは実行されずインデントは追加されない。

保守担当者向けメモ

  • 本関数は indent.c 内の宣言処理において、型名と変数名の間の空白を制御する重要な役割を担う。
  • postgres_tab_rules によるタブと空白の切り替えロジックは、PostgreSQL 特有の整形ルールに依存しているため、変更時は注意が必要。
