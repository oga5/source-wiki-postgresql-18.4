encrypt_free

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-encrypt.c:213

目的

暗号化処理で使用された EncStat 構造体のリソースを解放し、メモリをクリーンアップします。

シグネチャまたは形状

static void encrypt_free(void *priv)

入力と出力

  • **入力**: void *priv - 解放対象となる EncStat 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • priv は EncStat 型の有効なポインタである必要があります。
  • pgp_cfb_free は st->ciph が非 NULL の場合にのみ呼び出されます。

アルゴリズムまたは振る舞い

1. 引数 priv を EncStat *st にキャストします。 2. st->ciph が存在する場合、pgp_cfb_free を呼び出して暗号化コンテキストを解放します。 3. px_memset を使用して st が指すメモリ領域をゼロクリアします。 4. pfree を呼び出して st 自体のメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: フィルタチェーンの終了処理において、フィルタの破棄プロセスから呼び出されます。
  • **呼び出し先**: pgp_cfb_free（暗号化コンテキスト解放）、px_memset（メモリゼロクリア）、pfree（メモリ解放）。

関連データ構造

  • EncStat: 暗号化の状態を保持する構造体。本関数はこの構造体の破棄を担当します。

失敗モードと境界条件

  • st->ciph が NULL の場合、pgp_cfb_free の呼び出しはスキップされます。
  • priv が NULL の場合に発生する動作については、本ソースコード内では明示的なチェックは行われていません。

保守担当者向けメモ

  • 本関数は PushFilter の破棄用コールバックとして設計されています。
  • メモリ解放後に pfree を使用しているため、PostgreSQL のメモリコンテキスト管理下で動作することが前提です。
