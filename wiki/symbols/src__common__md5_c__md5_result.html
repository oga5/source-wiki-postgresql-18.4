md5_result

定義

  • 種別: function
  • 定義箇所: src/common/md5.c:347

目的

MD5 ハッシュ計算の最終段階において、コンテキスト内に保持された内部状態を、エンディアンを考慮した形式で出力バッファへ書き出す。

シグネチャまたは形状

static void md5_result(uint8 *digest, pg_md5_ctx *ctx)

入力と出力

  • digest: 16バイトのハッシュ値を受け取るための出力先バッファ（uint8 *）。
  • ctx: ハッシュ計算の状態を保持する pg_md5_ctx 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • ctx は pg_md5_init および pg_md5_update によって適切に初期化・更新されていること。
  • digest は少なくとも16バイトのメモリ領域を確保していること。

アルゴリズムまたは振る舞い

1. コンパイル時のエンディアン設定（WORDS_BIGENDIAN）に基づき、以下のいずれかの処理を行う。

  • **リトルエンディアン環境**: ctx->md5_st8 の内容を memmove を用いてそのまま digest にコピーする。
  • **ビッグエンディアン環境**: ctx->md5_st8 の各バイトを、MD5 アルゴリズムが期待するバイト順序（リトルエンディアン形式）に並べ替えて digest に格納する。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/md5.c 内の pg_md5_final 関数から呼び出される。
  • 呼び出し先: memmove（標準ライブラリ関数）。

関連データ構造

  • pg_md5_ctx: src/common/md5_int.h で定義される、ハッシュ計算の状態を保持する構造体。

失敗モードと境界条件

  • digest が NULL ポインタである場合や、十分な領域がない場合はメモリ破壊を引き起こす可能性がある。本関数自体は境界チェックを行わない。

保守担当者向けメモ

  • 本関数は RFC 1321 に準拠したバイト順序の調整を行うための低レベル関数である。
  • ビッグエンディアン環境での手動バイト入れ替え処理は、MD5 の仕様に基づく固定的な操作であるため、変更の必要性は低い。
