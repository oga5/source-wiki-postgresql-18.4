sepgsql_compute_avd

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/selinux.c:738

目的

指定されたソースコンテキストとターゲットコンテキスト、およびオブジェクトクラスに基づき、SELinux ポリシーからアクセス権限（av_decision）を計算し、PostgreSQL 内部形式へ変換して取得する。

シグネチャまたは形状

void sepgsql_compute_avd(const char *scontext, const char *tcontext, uint16 tclass, struct av_decision *avd)

入力と出力

  • **入力**:
  • scontext: ソースのセキュリティコンテキスト文字列。
  • tcontext: ターゲットのセキュリティコンテキスト文字列。
  • tclass: PostgreSQL 内部のオブジェクトクラスコード。
  • **出力**:
  • avd: 計算結果を格納する struct av_decision 構造体（呼び出し元で確保）。

前提条件と不変条件

  • tclass は SEPG_CLASS_MAX 未満であること。
  • tclass は selinux_catalog 内の class_code と一致すること。
  • SELinux ポリシーがロードされている環境下で実行されること。

アルゴリズムまたは振る舞い

1. **クラス変換**: selinux_catalog を参照し、PostgreSQL 内部クラスコードを SELinux の外部クラス名に変換。 2. **ポリシー判定**: security_compute_av_flags_raw を呼び出し、カーネルから生（raw）のアクセス判定結果を取得。失敗時は ERROR をスロー。 3. **マッピング**: 取得した外部ビットマスクを、selinux_catalog の定義に基づき PostgreSQL 内部の権限ビットへ再マッピング。 4. **未知の権限処理**: ポリシーに存在しない権限については、security_deny_unknown() の設定に従い、許可または拒否のデフォルト値を適用。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/uavc.c: ユーザー空間 AVC のキャッシュ判定処理から呼び出される。
  • **呼び出し先**:
  • security_deny_unknown()
  • string_to_security_class()
  • security_compute_av_flags_raw()
  • string_to_av_perm()

関連データ構造

  • struct av_decision: SELinux のアクセス判定結果（allowed, auditallow, auditdeny, flags）を保持する構造体。
  • selinux_catalog: PostgreSQL 内部クラス/権限と SELinux 外部コードの対応表。

失敗モードと境界条件

  • **ポリシー未定義**: 指定された tclass が現在の SELinux ポリシーに存在しない場合、ダミーデータ（deny_unknown 設定に基づく）を avd に設定して正常終了する。
  • **計算エラー**: security_compute_av_flags_raw が負の値を返した場合、ereport(ERROR, ...) によりトランザクションを中断する。

保守担当者向けメモ

  • 本関数は実行時のポリシーに依存するため、キャッシュ機構（uavc.c）と組み合わせて使用することが推奨される。
  • selinux_catalog の定義を変更した際は、本関数のマッピングループが正しく動作するか確認が必要。
