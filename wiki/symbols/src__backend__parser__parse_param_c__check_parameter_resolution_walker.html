check_parameter_resolution_walker

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_param.c:286

目的

クエリツリーを走査し、PARAM_EXTERN 型のパラメータが正しく解決されているか（型が確定しているか）を検証する。

シグネチャまたは形状

static bool check_parameter_resolution_walker(Node *node, ParseState *pstate);

入力と出力

  • **入力**:
  • node: 走査対象のノード。
  • pstate: 現在の解析状態を保持する ParseState 構造体。
  • **出力**:
  • bool: 走査を継続する場合は false、中断する場合は true を返す。

前提条件と不変条件

  • pstate->p_ref_hook_state は VarParamState 構造体としてキャスト可能であること。
  • パラメータの型解決が完了した状態で呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. **ノードの判定**:

  • Param ノードの場合: paramkind が PARAM_EXTERN であれば、VarParamState を参照してパラメータ番号の妥当性と型の一致を確認する。型が未定（不一致）の場合はエラーを報告する。
  • Query ノードの場合: query_tree_walker を用いてサブクエリ内を再帰的に走査する。
  • その他のノードの場合: expression_tree_walker を用いて式ツリーを再帰的に走査する。

    • 2. **検証**: パラメータ番号が範囲外であるか、あるいは param->paramtype が VarParamState に記録された型と一致しない場合に ereport(ERROR, ...) を発行する。

呼び出し元と呼び出し先

  • **呼び出し元**: check_variable_parameters（ソースコード内の直接的な呼び出し元として確認）。
  • **呼び出し先**:
  • query_tree_walker（サブクエリ走査用）
  • expression_tree_walker（式ツリー走査用）
  • ereport（エラー報告用）

関連データ構造

  • Param: パラメータ情報を保持するノード。
  • ParseState: 解析状態を保持する構造体。
  • VarParamState: 可変パラメータの状態を保持する構造体（p_ref_hook_state 経由でアクセス）。

失敗モードと境界条件

  • **ERRCODE_UNDEFINED_PARAMETER**: パラメータ番号が 0 以下、または定義されたパラメータ数を超えている場合に発生。
  • **ERRCODE_AMBIGUOUS_PARAMETER**: パラメータの型が決定できない（paramtype が期待される型と一致しない）場合に発生。

保守担当者向けメモ

  • 本関数は expression_tree_walker のコールバックとして機能するため、戻り値の false は「走査を中断しない」ことを意味する。
  • paramtypmod が -1 に設定されている場合、実行時の型チェックが優先される設計となっている。
