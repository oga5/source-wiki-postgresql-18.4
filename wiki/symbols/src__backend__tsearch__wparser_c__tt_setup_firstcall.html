tt_setup_firstcall

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/wparser.c:46

目的

全文検索パーサーのトークン型定義を取得し、SRF（Set-Returning Function）の初回呼び出しに必要な初期化処理を行う。

シグネチャまたは形状

static void tt_setup_firstcall(FuncCallContext *funcctx, FunctionCallInfo fcinfo, Oid prsid)

入力と出力

  • **入力**:
  • funcctx: SRFの状態を保持するコンテキスト。
  • fcinfo: 関数呼び出し情報。
  • prsid: 対象となるテキスト検索パーサーのOID。
  • **出力**: なし（funcctx の内部状態を更新する）。

前提条件と不変条件

  • 指定された prsid に対応するパーサーがキャッシュに存在すること。
  • パーサーの lextype メソッドが定義されていること（未定義の場合はエラー）。
  • 戻り値の型が複合型（TYPEFUNC_COMPOSITE）であること。

アルゴリズムまたは振る舞い

1. lookup_ts_parser_cache を用いてパーサー定義を取得する。 2. パーサーの lextypeOid が有効か検証し、無効な場合はエラーを投げる。 3. funcctx->multi_call_memory_ctx へメモリコンテキストを切り替える。 4. TSTokenTypeStorage 構造体を割り当て、cur を 0 に初期化する。 5. パーサーの lextype 関数を呼び出し、トークン型のリストを st->list に格納する。 6. funcctx->user_fctx に上記構造体を保存する。 7. get_call_result_type で戻り値の型情報を取得し、funcctx->tuple_desc および funcctx->attinmeta を設定する。 8. 元のメモリコンテキストへ復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ts_token_type_byid (src/backend/tsearch/wparser.c:114)
  • ts_token_type_byname (src/backend/tsearch/wparser.c:137)
  • **呼び出し先**:
  • lookup_ts_parser_cache
  • MemoryContextSwitchTo
  • palloc
  • OidFunctionCall1 (パーサーの lextype 関数を実行)
  • get_call_result_type
  • TupleDescGetAttInMetadata

関連データ構造

  • FuncCallContext: SRFの状態保持用。
  • TSTokenTypeStorage: トークン型情報のリストと現在のカーソル位置を保持する構造体。
  • TSParserCacheEntry: パーサーのキャッシュ定義。
  • LexDescr: トークン型の記述子。

失敗モードと境界条件

  • lextype メソッドが定義されていないパーサーOIDが渡された場合、elog(ERROR) で中断する。
  • 戻り値の型が複合型でない場合、elog(ERROR) で中断する。

保守担当者向けメモ

  • 本関数はSRFの初回呼び出し専用です。後続のタプル生成は tt_process_call が担当します。
  • メモリ管理のため、funcctx->multi_call_memory_ctx を使用して割り当てを行っています。修正時はメモリコンテキストの切り替え漏れに注意してください。
