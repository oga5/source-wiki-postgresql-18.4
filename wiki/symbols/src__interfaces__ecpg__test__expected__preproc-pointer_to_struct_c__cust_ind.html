cust_ind

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:44

目的

ECPG（Embedded SQL for C）において、SQLクエリの結果として返される値がNULLであるかどうかを判定するためのインジケータ変数用構造体。

シグネチャまたは形状

typedef struct ind {
    short name_ind;
    short phone_ind;
} cust_ind;

入力と出力

  • 入力: SQLのフェッチ操作において、対応するホスト変数の状態（NULLか否か）を受け取る。
  • 出力: 負の値（通常は-1）が設定された場合、対応するSQLカラムがNULLであることを示す。

前提条件と不変条件

  • 構造体の各メンバは short 型であり、ECPGのインジケータ変数として期待されるメモリレイアウトに従う必要がある。

アルゴリズムまたは振る舞い

  • SQLの FETCH や SELECT 文において、ホスト変数とペアで使用される。
  • プリプロセッサは、この構造体を通じてデータベースから返されるNULL値を検出し、アプリケーション側で適切に処理できるようにする。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/interfaces/ecpg/test/expected/preproc-array_of_struct.c:49, 90
  • src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:50, 94
  • 呼び出し先: 情報なし。

関連データ構造

  • customer: 本構造体と対になって使用されるデータ格納用構造体。

失敗モードと境界条件

  • インジケータ変数のメモリ領域が確保されていない場合、または malloc 等によるメモリ割り当てが不十分な場合、SQL実行時にメモリ破壊や不正な値の読み取りが発生する可能性がある。

保守担当者向けメモ

  • 本構造体はECPGのテスト生成物の一部であり、手動編集は推奨されない。
  • 構造体のメンバ構成を変更する場合は、対応するSQLクエリのインジケータ指定も整合性を取る必要がある。
