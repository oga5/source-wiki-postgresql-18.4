xl_replorigin_drop

定義

  • 種別: struct
  • 定義箇所: src/include/replication/origin.h:25

目的

論理レプリケーションにおける「レプリケーション起点（Replication Origin）」の削除操作をWAL（Write-Ahead Log）に記録するためのデータ構造です。

シグネチャまたは形状

typedef struct xl_replorigin_drop
{
	RepOriginId node_id;
} xl_replorigin_drop;

入力と出力

  • **入力**: 削除対象となるレプリケーション起点の識別子 node_id (RepOriginId)。
  • **出力**: 本構造体を含むWALレコードが生成され、リカバリ時にレプリケーション起点の削除を再実行するために使用されます。

前提条件と不変条件

  • node_id は有効な RepOriginId である必要があります。
  • 本構造体は、レプリケーション起点の削除処理において、共有メモリ上の状態変更と同期してWALに書き込まれる必要があります。

アルゴリズムまたは振る舞い

1. レプリケーション起点の削除処理において、本構造体に削除対象の node_id を格納します。 2. WALレコードとして書き込まれることで、クラッシュリカバリ時に replorigin_redo 関数がこのレコードを読み取り、対応するレプリケーション起点の情報をシステムから削除します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/origin.c: レプリケーション起点の削除処理を行う関数内で、WAL記録用として使用されます。
  • **呼び出し先**:
  • src/backend/access/rmgrdesc/replorigindesc.c: WALレコードのデコードおよび説明（description）生成時に参照されます。

関連データ構造

  • RepOriginId: レプリケーション起点を識別する2バイトの内部ID。

失敗モードと境界条件

  • 存在しない node_id を指定して削除を試みた場合、リカバリ処理や削除処理において不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体はWALのフォーマットの一部であるため、構造を変更する場合はWALの互換性に注意してください。
  • src/backend/replication/logical/origin.c の削除ロジックと、replorigindesc.c によるWAL解析ロジックが常に同期していることを確認してください。
