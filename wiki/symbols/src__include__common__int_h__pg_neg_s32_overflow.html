pg_neg_s32_overflow

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:204

目的

符号付き32ビット整数（int32）の符号反転時に発生するオーバーフローを検出し、安全に演算を行う。

シグネチャまたは形状

static inline bool pg_neg_s32_overflow(int32 a, int32 *result)

入力と出力

  • **入力**:
  • a: 符号反転対象の int32 値。
  • result: 演算結果を格納する int32 型ポインタ。
  • **出力**:
  • 戻り値: オーバーフローが発生した場合は true、それ以外は false を返す。
  • *result: 演算成功時は反転した値、オーバーフロー時はダミー値 0x5EED が格納される。

前提条件と不変条件

  • int32 型の範囲内での演算を前提とする。
  • PG_INT32_MIN を反転しようとした場合、結果は表現可能な範囲を超えるためオーバーフローとみなされる。

アルゴリズムまたは振る舞い

1. コンパイラが __builtin_sub_overflow をサポートしている場合、それを利用して 0 - a を計算し、オーバーフロー判定を行う。 2. サポートしていない場合:

  • a が PG_INT32_MIN と等しいかを確認する。
  • 等しい場合、*result に 0x5EED を代入し true を返す。
  • 等しくない場合、*result に -a を代入し false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/date.c:263 にて呼び出しを確認。
  • **呼び出し先**:
  • コンパイラ組み込み関数 __builtin_sub_overflow（環境依存）。
  • 明示的な関数呼び出し先は他に存在しない。

関連データ構造

  • int32: PostgreSQL で定義される符号付き32ビット整数型。

失敗モードと境界条件

  • **境界条件**: a が PG_INT32_MIN のとき、符号反転は int32 の範囲外となるため、本関数はオーバーフローを報告する。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、ヘッダーファイル src/include/common/int.h に配置されている。
  • 0x5EED はオーバーフロー発生時のダミー値として一貫して使用されている。
