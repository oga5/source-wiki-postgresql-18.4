tuplesort_begin_index_brin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:554

目的

BRIN（Block Range Index）インデックス構築のためのソート処理を初期化し、Tuplesortstate を構築する。

シグネチャまたは形状

Tuplesortstate *tuplesort_begin_index_brin(int workMem, SortCoordinate coordinate, int sortopt)

入力と出力

  • **入力**:
  • workMem: ソートに使用可能なメモリ量（KB）。
  • coordinate: ソートの実行コンテキスト（並列処理等の情報）。
  • sortopt: ソートのオプションフラグ（TUPLESORT_RANDOMACCESS 等）。
  • **出力**:
  • 初期化済みの Tuplesortstate 構造体へのポインタ。

前提条件と不変条件

  • tuplesort_begin_common が正常に終了し、Tuplesortstate が割り当てられていること。
  • BRINインデックスの特性上、ソートキーはブロック番号の1つのみに固定される。

アルゴリズムまたは振る舞い

1. tuplesort_begin_common を呼び出し、汎用的なソート状態を初期化する。 2. trace_sort が有効な場合、ログを出力する。 3. TuplesortPublic 構造体の nKeys を 1 に設定する。 4. BRINインデックス専用の関数ポインタを登録する:

  • removeabbrev: removeabbrev_index_brin
  • comparetup: comparetup_index_brin
  • writetup: writetup_index_brin
  • readtup: readtup_index_brin

    • 5. haveDatum1 を true に設定し、arg を NULL に初期化して状態を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c (複数箇所)
  • **呼び出し先**:
  • tuplesort_begin_common (内部関数)
  • TuplesortstateGetPublic (マクロ)
  • elog (ログ出力)

関連データ構造

  • Tuplesortstate: ソート処理の全状態を保持する。
  • TuplesortPublic: Tuplesortstate の公開インターフェース部分。

失敗モードと境界条件

  • メモリ不足時には tuplesort_begin_common 内でエラーが発生する可能性がある。
  • sortopt のフラグ設定が不適切な場合、後続のソート処理で予期せぬ挙動を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はBRINインデックス専用のプラグインとして機能する。比較ロジックやシリアライズ処理を変更する場合は、対応する comparetup_index_brin 等の関数群も併せて確認すること。
