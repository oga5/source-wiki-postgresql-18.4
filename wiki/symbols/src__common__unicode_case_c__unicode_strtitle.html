unicode_strtitle

定義

  • 種別: function
  • 定義箇所: src/common/unicode_case.c:137

目的

入力された UTF-8 文字列を Unicode 規格に基づきタイトルケース（各単語の先頭を大文字、それ以外を小文字）に変換します。

シグネチャまたは形状

size_t unicode_strtitle(char *dst, size_t dstsize, const char *src, ssize_t srclen,
                        bool full, WordBoundaryNext wbnext, void *wbstate);

入力と出力

  • **入力**:
  • dst: 変換結果を格納するバッファ。
  • dstsize: dst バッファのサイズ（バイト単位）。0 の場合は必要なサイズを計算します。
  • src: 変換対象の UTF-8 文字列。
  • srclen: src の長さ（バイト単位）。-1 の場合はヌル終端文字列として扱います。
  • full: 特殊ケースマッピング（ギリシャ語の最終シグマ等）を適用するかどうかのフラグ。
  • wbnext: 単語境界を判定するためのコールバック関数。
  • wbstate: wbnext に渡す状態データ。
  • **出力**: 変換後の文字列に必要なバイト数（ヌル終端文字を含まない）。

前提条件と不変条件

  • src は有効な UTF-8 文字列である必要があります。
  • wbnext コールバックは、タイトルケース変換における単語境界の判定ロジックを提供します。

アルゴリズムまたは振る舞い

内部関数 convert_case を呼び出し、CaseTitle モードで変換を実行します。入力文字列を走査し、wbnext を用いて単語境界を特定しながら、各文字に対して適切なケースマッピングを適用します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale_builtin.c
  • src/common/unicode/case_test.c
  • **呼び出し先**:
  • convert_case（内部関数）

関連データ構造

  • CaseMapResult（変換結果の状態を示す列挙型）
  • WordBoundaryNext（単語境界判定用関数ポインタ型）

失敗モードと境界条件

  • dstsize が 0 の場合、変換後の文字列を格納するために必要なバイト数を返します。
  • srclen が -1 の場合、src がヌル終端されていることを前提に処理します。

保守担当者向けメモ

  • 本関数は convert_case のラッパーであり、タイトルケース変換のロジックは convert_case および casemap に依存しています。
  • ギリシャ語の最終シグマのような文脈依存の変換が必要な場合は、full フラグを true に設定してください。
