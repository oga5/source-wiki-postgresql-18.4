write_reconstructed_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/reconstruct.c:552

目的

増分バックアップの再構築計画（sourcemap および offsetmap）に基づき、物理的なデータファイルを生成または再構築する。

シグネチャまたは形状

static void write_reconstructed_file(char *input_filename,
                                     char *output_filename,
                                     unsigned block_length,
                                     rfile **sourcemap,
                                     off_t *offsetmap,
                                     pg_checksum_context *checksum_ctx,
                                     CopyMethod copy_method,
                                     bool debug,
                                     bool dry_run);

入力と出力

  • **入力**:
  • input_filename, output_filename: 入出力ファイルパス。
  • block_length: 出力ファイルの総ブロック数。
  • sourcemap: 各ブロックのソース元を保持する rfile 構造体の配列。NULL はゼロ埋めブロックを意味する。
  • offsetmap: 各ブロックのソースファイル内オフセット。
  • checksum_ctx: チェックサム計算用コンテキスト。
  • copy_method: ブロック転送手法（COPY_METHOD_COPY_FILE_RANGE 等）。
  • debug, dry_run: デバッグ出力およびドライラン実行のフラグ。
  • **出力**: なし（副作用としてファイルシステムへの書き込みを行う）。

前提条件と不変条件

  • dry_run が false の場合、出力ファイルは新規作成（O_CREAT | O_EXCL）される必要がある。
  • sourcemap 内の各 rfile は有効なファイル記述子を保持していること。
  • BLCKSZ 単位での読み書きが前提。

アルゴリズムまたは振る舞い

1. **デバッグ出力**: debug が有効な場合、再構築計画を StringInfo に集約しログ出力する。 2. **ファイルオープン**: dry_run でない場合、output_filename を書き込みモードで開く。 3. **ブロック処理ループ**: block_length 分だけ以下の処理を繰り返す。

  • sourcemap[i] が NULL の場合: ゼロ埋めブロックとして書き込む。
  • copy_method が COPY_METHOD_COPY_FILE_RANGE の場合: copy_file_range を使用して効率的に転送し、必要に応じてチェックサムを更新する。
  • それ以外の場合: read_block で読み込み、write_block で書き出す。

    • 4. **会計情報更新**: 各 rfile の読み込み回数と最大オフセットを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: reconstruct_from_incremental_file（src/bin/pg_combinebackup/reconstruct.c:344）。
  • **呼び出し先**: read_block, write_block, pg_checksum_update, copy_file_range（プラットフォーム依存）。

関連データ構造

  • rfile: ソースファイルのメタデータとファイル記述子を保持。
  • pg_checksum_context: チェックサム計算の状態管理。
  • CopyMethod: ファイルコピー戦略の列挙型。

失敗モードと境界条件

  • **ファイルオープン失敗**: open() が失敗した場合、pg_fatal で終了する。
  • **コピー失敗**: copy_file_range がエラーを返した場合、pg_fatal で終了する。
  • **プラットフォーム制限**: copy_file_range がサポートされていない環境で当該メソッドが指定された場合、pg_fatal で終了する。
  • **チェックサム更新失敗**: pg_checksum_update がエラーを返した場合、pg_fatal で終了する。

保守担当者向けメモ

  • dry_run モード時はファイルオープンおよび書き込み処理がスキップされるため、計画の検証に利用可能。
  • copy_file_range 使用時はオフセットが更新されるため、ループ内でのオフセット管理に注意すること。
