TSLexeme

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_public.h:115

目的

辞書による単語分割処理の結果を保持し、単語の正規化や分割バリエーションを表現するために使用されます。

シグネチャまたは形状

typedef struct {
    uint16      nvariant;
    uint16      flags;
    char       *lexeme;
} TSLexeme;

入力と出力

  • **入力**: 辞書関数（lexize）が処理した単語の分割結果を格納します。
  • **出力**: 呼び出し元に対して、正規化された単語（lexeme）と、その分割グループ（nvariant）、および制御フラグ（flags）を提供します。

前提条件と不変条件

  • nvariant は、単語の分割バリエーションをグループ化するために使用されます。
  • 連続する TSLexeme 要素において、nvariant の値が同じであれば、それらは同一の分割バリエーションに属するとみなされます。
  • nvariant の絶対値そのものよりも、隣接する要素間での値の変化が重要です。

アルゴリズムまたは振る舞い

辞書関数は入力単語を解析し、TSLexeme の配列を返します。例えば、「fotballklubber」という単語を「fotball, klubb」と「fot, ball, klubb」の2通りに分割する場合、辞書は nvariant を切り替えることでこれらを表現します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/tsearch/dict.c
  • contrib/dict_int/dict_int.c
  • contrib/dict_xsyn/dict_xsyn.c
  • contrib/unaccent/unaccent.c
  • src/backend/snowball/dict_snowball.c
  • **呼び出し先**: 構造体であるため、関数呼び出しの対象ではありません。

関連データ構造

  • TSL_ADDPOS, TSL_PREFIX, TSL_FILTER: flags フィールドで使用される制御フラグ。
  • DictSubState: シソーラス処理などで状態を保持するために併用される構造体。

失敗モードと境界条件

  • lexeme は C 文字列として扱われるため、メモリ管理（palloc 等）が適切に行われない場合、メモリリークや不正なメモリ参照が発生する可能性があります。
  • nvariant の値が適切に設定されていない場合、後続の全文検索処理において分割バリエーションが正しく認識されません。

保守担当者向けメモ

  • nvariant は単なる識別子であり、値の大小関係に意味はありません。
  • flags ビットフィールドの定義については、本ヘッダーファイル内の関連定数定義を参照してください。
