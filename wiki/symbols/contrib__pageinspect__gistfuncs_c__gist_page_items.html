gist_page_items

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/gistfuncs.c:193

目的

GiSTインデックスの物理ページ内容を解析し、ページ内の各インデックス項目（タプル）をデコードして人間が読み取り可能な形式で出力する。

シグネチャまたは形状

Datum gist_page_items(PG_FUNCTION_ARGS)

  • 引数: bytea（対象ページデータ）、Oid（対象インデックスのOID）
  • 戻り値: Datum（tuplestore を介したセット形式）

入力と出力

  • 入力:
  • raw_page: 解析対象のGiSTページデータ。
  • indexRelid: 対象インデックスのOID。
  • 出力:
  • ページ内の各アイテムについて、オフセット番号、TID、タプルサイズ、削除フラグ、およびデコードされたカラム値を含む行セットを返す。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 指定された indexRelid がGiSTインデックスであること。
  • ページがGiSTページとして有効であること（verify_gist_page による検証）。

アルゴリズムまたは振る舞い

1. スーパーユーザー権限を確認し、対象インデックスを AccessShareLock で開く。 2. verify_gist_page でページを検証し、空ページの場合は NULL を返す。 3. ページがリーフページか否かに応じて TupleDesc を決定し、インデックス定義を取得する。 4. ページ内の各アイテム（OffsetNumber）を走査する。 5. index_deform_tuple を使用してタプルをデコードし、属性値を文字列化する。 6. 構築した行を tuplestore に格納し、最後にインデックスを閉じて終了する。

呼び出し元と呼び出し先

  • 呼び出し元: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • 呼び出し先: verify_gist_page, index_open, index_close, index_deform_tuple, tuplestore_putvalues 等。

関連データ構造

  • GISTPageOpaqueData: GiSTページヘッダーのメタデータ。
  • IndexTuple: ページ内のインデックス項目。
  • TupleDesc: インデックスの列定義情報。

失敗モードと境界条件

  • スーパーユーザーでない場合、エラーを発生させる。
  • GiSTインデックスではないリレーションが指定された場合、エラーを発生させる。
  • ページが破損している、またはGiSTページではない場合、verify_gist_page がエラーを発生させる。
  • ページが削除済み（GistPageIsDeleted）の場合、警告を発行する。

保守担当者向けメモ

  • 本関数は pg_get_indexdef_columns_extended を使用して列定義を取得しており、インデックスのスキーマ変更に追従する設計となっている。
  • record_out に類似した文字列変換ロジックを含んでおり、特殊文字のエスケープ処理が行われている。
