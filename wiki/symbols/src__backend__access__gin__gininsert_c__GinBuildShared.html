GinBuildShared

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/gininsert.c:49

目的

GINインデックスの並列構築において、リーダープロセスとワーカープロセス間で共有される状態を保持するために使用されます。

シグネチャまたは形状

typedef struct GinBuildShared {
    Oid         heaprelid;
    Oid         indexrelid;
    bool        isconcurrent;
    int         scantuplesortstates;
    ConditionVariable workersdonecv;
    slock_t     mutex;
    int         nparticipantsdone;
    double      reltuples;
    double      indtuples;
} GinBuildShared;

入力と出力

  • **入力**: 並列構築開始時に共有メモリ上に割り当てられ、各プロセスから参照されます。
  • **出力**: 構築終了時に、リーダープロセスがワーカーから報告された統計情報（reltuples, indtuples）を読み取ります。

前提条件と不変条件

  • mutex フィールドは、nparticipantsdone、reltuples、indtuples へのアクセスを保護します。
  • heaprelid、indexrelid、isconcurrent、scantuplesortstates は構築中に変更されません。
  • 並列参加者は、リーダーが結果をインデックスに書き込む前に、workersdonecv を通じて完了を通知する必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: リーダープロセスが共有メモリ上に本構造体を割り当てます。 2. **並列処理**: 各ワーカーは自身の処理状況を mutex で保護されたフィールドに更新します。 3. **同期**: リーダーは workersdonecv を監視し、全ワーカーの完了を待機します。 4. **集約**: 全ワーカー完了後、リーダーは共有された統計情報を集計し、インデックス構築を完了させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gininsert.c 内の並列構築関連関数（shm_toc_allocate を用いた割り当てや、並列構築の初期化処理）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ConditionVariable: ワーカーの完了監視に使用。
  • slock_t: 共有フィールドの排他制御に使用。
  • ParallelTableScanDescData: 本構造体の直後に配置される（構造体定義には含まれないが、メモリレイアウト上後続する）。

失敗モードと境界条件

  • 共有メモリの割り当て失敗時に構築が中断される可能性があります。
  • ParallelTableScanDescData のアライメント要件のため、本構造体単体ではなく BUFFERALIGN を用いた計算でメモリ配置を行う必要があります。

保守担当者向けメモ

  • 本構造体は並列構築のリーダー/ワーカー間の通信基盤です。フィールドを追加する際は、mutex による保護が必要かどうかを慎重に検討してください。
  • ParallelTableScanDescData が直後に続くため、メモリレイアウトの変更には注意が必要です。
