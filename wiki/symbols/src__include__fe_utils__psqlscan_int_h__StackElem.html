StackElem

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/psqlscan_int.h:69

目的

psqlの変数展開時など、字句解析器（Flex）が使用する入力バッファを管理するためのスタック要素を保持します。

シグネチャまたは形状

typedef struct StackElem
{
	YY_BUFFER_STATE buf;
	char	   *bufstring;
	char	   *origstring;
	char	   *varname;
	struct StackElem *next;
} StackElem;

入力と出力

  • **入力**: Flexのバッファ状態 (YY_BUFFER_STATE)、解析対象文字列、変数名。
  • **出力**: スタック構造のリンク (next) を介した、ネストされた解析状態の管理。

前提条件と不変条件

  • buf はFlexの入力制御構造体として有効である必要があります。
  • next ポインタにより、単方向連結リストとしてスタックを形成します。

アルゴリズムまたは振る舞い

  • psqlscan_push_new_buffer 等の関数により、新しいバッファがスタックに積まれます。
  • 解析器はスタックの先頭にあるバッファを順次処理し、終了後に psqlscan_pop_buffer_stack を用いて元のバッファへ復帰します。
  • varname は、そのバッファがどの変数に由来するかを追跡するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: PsqlScanStateData 構造体内の buffer_stack メンバとして参照されます。
  • **呼び出し先**: 提供されたソース excerpt 内に具体的な関数呼び出しの記述はありません。

関連データ構造

  • PsqlScanStateData: 本構造体をスタックとして保持する上位の解析状態構造体。
  • YY_BUFFER_STATE: Flexが提供するバッファ制御型。

失敗モードと境界条件

  • スタックが空の状態でポップ操作が行われた場合の挙動については、本ソース excerpt に記述がありません。
  • メモリ割り当て失敗時の挙動については、本ソース excerpt に記述がありません。

保守担当者向けメモ

  • 本構造体は psqlscan.l 等の再入可能な字句解析器において、バッファの切り替えを安全に行うために不可欠です。
  • bufstring や origstring のメモリ管理（解放タイミング）については、psqlscan_pop_buffer_stack の実装を確認してください。
