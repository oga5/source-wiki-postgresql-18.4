superuser_arg

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/superuser.c:55

目的

指定されたロールIDがスーパーユーザー権限を持つかどうかを判定します。システムカタログ pg_authid を直接参照せず、キャッシュ機構を介して効率的に権限情報を取得します。

シグネチャまたは形状

bool superuser_arg(Oid roleid)

入力と出力

  • **入力**: roleid (Oid) - 判定対象のロールID。
  • **出力**: bool - スーパーユーザーであれば true、そうでなければ false。

前提条件と不変条件

  • プロセスローカルなキャッシュ（last_roleid）が有効な場合、カタログ検索をスキップして結果を返します。
  • 初回実行時に、システムカタログの変更を監視するためのコールバック（RoleidCallback）が登録されます。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: last_roleid が引数と一致する場合、キャッシュされた結果を即座に返します。 2. **ブートストラップ例外**: シングルユーザーモード（!IsUnderPostmaster）かつ BOOTSTRAP_SUPERUSERID の場合は、カタログ検索を行わず true を返します。 3. **カタログ検索**: SearchSysCache1 を使用して pg_authid から情報を取得します。 4. **結果抽出**: rolsuper 属性を確認し、取得したタプルを解放します。 5. **コールバック登録**: 初回実行時のみ CacheRegisterSyscacheCallback を呼び出し、キャッシュ無効化の監視を設定します。 6. **キャッシュ更新**: 判定結果を last_roleid および last_roleid_is_super に保存し、次回以降の高速化に備えます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/connection.c
  • src/backend/access/transam/twophase.c
  • src/backend/catalog/aclchk.c
  • **呼び出し先**:
  • SearchSysCache1 (システムキャッシュ検索)
  • ReleaseSysCache (タプル解放)
  • CacheRegisterSyscacheCallback (コールバック登録)
  • RoleidCallback (キャッシュ無効化用コールバック)

関連データ構造

  • pg_authid: ロール情報を保持するシステムカタログ。
  • last_roleid / last_roleid_is_super: 判定結果を保持する静的キャッシュ変数。
  • roleid_callback_registered: コールバック登録状態を保持するフラグ。

失敗モードと境界条件

  • roleid が無効な場合、SearchSysCache1 が無効なタプルを返すため、結果は false となります。
  • キャッシュはプロセスローカルであるため、他プロセスでの権限変更は RoleidCallback による無効化イベントを介して同期されます。

保守担当者向けメモ

  • pg_authid.rolsuper を直接参照することは禁止されています。必ず本関数を経由してください。
  • シングルユーザーモードでのロックアウトを防ぐため、BOOTSTRAP_SUPERUSERID の例外処理が組み込まれています。
  • デバッグ時は、キャッシュの不整合がないか last_roleid の状態を確認してください。
