PQdisplayTuples

定義

  • **種別**: function
  • **定義箇所**: src/interfaces/libpq/fe-print.c:604

目的

PGresult オブジェクトに含まれるクエリ結果を、指定された出力ストリームに対して表形式で整形出力するレガシーなユーティリティ関数です。

シグネチャまたは形状

void PQdisplayTuples(const PGresult *res, FILE *fp, int fillAlign, const char *fieldSep, int printHeader, int quiet);

入力と出力

  • **入力**:
  • res: 出力対象の PGresult オブジェクト。
  • fp: 出力先ファイルポインタ（NULL の場合は stdout を使用）。
  • fillAlign: 非ゼロの場合、フィールドをスペースで埋めてアライメントを行う。
  • fieldSep: フィールド間の区切り文字（NULL の場合は " " を使用）。
  • printHeader: 非ゼロの場合、ヘッダーと区切り線を出力する。
  • quiet: 非ゼロの場合、末尾の「Query returned X rows.」メッセージを抑制する。
  • **出力**: なし（void）。指定されたストリームへ整形済みテキストを書き出す。

前提条件と不変条件

  • res は有効な PGresult オブジェクトである必要があります。
  • fillAlign が有効な場合、内部で malloc を使用して列幅計算用のメモリを確保します。メモリ確保失敗時はエラーメッセージを出力して早期リターンします。

アルゴリズムまたは振る舞い

1. **初期化**: fieldSep が NULL ならデフォルトの " " を適用し、fp が NULL なら stdout に設定します。 2. **アライメント計算**: fillAlign が真の場合、全列のフィールド名とデータ値を走査し、各列の最大幅を fLength 配列に格納します。 3. **ヘッダー出力**: printHeader が真の場合、列名を表示し、fillAlign が有効なら fill 関数で整形して区切り文字を挿入します。その後、ハイフンによる下線行を出力します。 4. **データ出力**: 全タプルを走査し、各フィールド値を PQgetvalue で取得して出力します。fillAlign が有効な場合は fill 関数でスペース埋めを行います。 5. **終了処理**: quiet が偽の場合、結果行数を表示します。最後に fflush(fp) を実行し、確保した fLength メモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は提供されていません。
  • **呼び出し先**:
  • PQnfields, PQntuples, PQfname, PQgetlength, PQgetvalue (libpq API)
  • fill (同ファイル内の補助関数)
  • malloc, free, fprintf, fputs, strlen, fflush (標準ライブラリ)

関連データ構造

  • PGresult: クエリ結果を保持する構造体。

失敗モードと境界条件

  • fillAlign が有効な環境でメモリ不足が発生した場合、stderr にエラーを出力して処理を中断します。
  • res が空の結果セットの場合、ヘッダーのみ（または何も）出力され、行数メッセージのみが表示されます。

保守担当者向けメモ

  • 本関数はレガシーなインターフェースであり、現代のアプリケーションでは推奨されません。
  • fillAlign を有効にした場合、大規模な結果セットに対しては全データを走査するためパフォーマンスに影響を与える可能性があります。
