issue_warnings_and_set_wal_level

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/check.c:791

目的

アップグレード後の新クラスタに対して、WALレベルの調整、旧バージョン由来のインデックス再構築、および拡張機能の更新確認を行う。

シグネチャまたは形状

void issue_warnings_and_set_wal_level(void)

入力と出力

  • 入力: なし（グローバル変数 new_cluster および old_cluster を参照）
  • 出力: なし（副作用として新クラスタの起動・停止およびログ出力を行う）

前提条件と不変条件

  • new_cluster および old_cluster が初期化されていること。
  • pg_resetwal によって新クラスタの wal_level が一時的に minimum に設定されている状態を前提とする。

アルゴリズムまたは振る舞い

1. start_postmaster を呼び出し、新クラスタを起動する。 2. 旧クラスタのメジャーバージョンが 9.6 以下の場合、old_9_6_invalidate_hash_indexes を呼び出し、ハッシュインデックスの無効化（再構築準備）を行う。 3. report_extension_updates を呼び出し、拡張機能の更新が必要かを確認・報告する。 4. stop_postmaster を呼び出し、新クラスタを停止する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/pg_upgrade.c:231
  • 呼び出し先:
  • start_postmaster
  • old_9_6_invalidate_hash_indexes
  • report_extension_updates
  • stop_postmaster

関連データ構造

  • new_cluster (グローバル変数)
  • old_cluster (グローバル変数)

失敗モードと境界条件

  • 新クラスタの起動（start_postmaster）に失敗した場合、処理は中断される。
  • 旧クラスタのバージョン判定（GET_MAJOR_VERSION）は、old_cluster.major_version の値に依存する。

保守担当者向けメモ

  • 本関数は、スタンバイサーバーのアップグレード時に wal_level を replica に昇格させるための最終的なWALレコード書き込みを確実にするために、新サーバーの起動・停止を強制的に行っている。
  • old_9_6_invalidate_hash_indexes の呼び出し条件は、PostgreSQL 10.0 未満の旧クラスタからのアップグレードに特化した互換性維持のための処理である。
