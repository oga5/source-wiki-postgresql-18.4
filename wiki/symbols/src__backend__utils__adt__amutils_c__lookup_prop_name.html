lookup_prop_name

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/amutils.c:89

目的

文字列として与えられたインデックスプロパティ名を、内部的な列挙型 IndexAMProperty に変換します。

シグネチャまたは形状

static IndexAMProperty lookup_prop_name(const char *name)

入力と出力

  • **入力**: name (検索対象のプロパティ名文字列)
  • **出力**: 一致する IndexAMProperty 値。一致しない場合は AMPROP_UNKNOWN を返します。

前提条件と不変条件

  • am_propnames 配列が定義されており、プロパティ名と列挙型のマッピングが保持されている必要があります。
  • 比較は pg_strcasecmp を使用するため、大文字小文字を区別しません。

アルゴリズムまたは振る舞い

1. am_propnames 配列を先頭から順に走査します。 2. 各要素の name フィールドと引数の name を pg_strcasecmp で比較します。 3. 一致した場合は、対応する prop フィールド（IndexAMProperty）を即座に返します。 4. 配列の最後まで一致が見つからなかった場合、エラーを発生させずに AMPROP_UNKNOWN を返します。これにより、個別のアクセスメソッド（AM）が独自のプロパティを定義する余地を残しています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/amutils.c 内の indexam_property 関数から呼び出されます。
  • **呼び出し先**: pg_strcasecmp (文字列比較関数)。

関連データ構造

  • IndexAMProperty: 照会可能なプロパティを定義する列挙型。
  • am_propnames: プロパティ名文字列と IndexAMProperty を対応付ける静的配列。

失敗モードと境界条件

  • 引数 name に一致するプロパティが存在しない場合、エラーにはならず AMPROP_UNKNOWN を返します。
  • am_propnames 配列の範囲外アクセスは発生しません（lengthof マクロによる安全なループ）。

保守担当者向けメモ

  • 新しいプロパティを追加する場合は、IndexAMProperty 列挙型と am_propnames 配列の両方を更新する必要があります。
  • 検索は線形探索であるため、プロパティ数が増加した場合はパフォーマンスに留意してください。
