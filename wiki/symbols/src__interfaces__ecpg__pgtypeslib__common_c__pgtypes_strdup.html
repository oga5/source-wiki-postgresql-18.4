pgtypes_strdup

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/common.c:19

目的

標準の strdup をラップし、メモリ確保失敗時に errno を ENOMEM に設定することで、エラーハンドリングを統一する。

シグネチャまたは形状

char *pgtypes_strdup(const char *str);

入力と出力

  • **入力**: str (複製対象のヌル終端文字列)
  • **出力**: 複製された文字列へのポインタ。メモリ確保に失敗した場合は NULL を返す。

前提条件と不変条件

  • 入力 str は有効なヌル終端文字列であること。
  • 戻り値が NULL でない場合、呼び出し元は後続の PGTYPESchar_free を使用してメモリを解放する責任を負う。

アルゴリズムまたは振る舞い

1. 標準ライブラリの strdup(str) を呼び出し、メモリを確保して文字列をコピーする。 2. 確保されたポインタが NULL の場合、errno に ENOMEM を代入する。 3. 確保されたポインタ（または NULL）を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/pgtypeslib/datetime.c
  • src/interfaces/ecpg/pgtypeslib/dt_common.c
  • src/interfaces/ecpg/pgtypeslib/interval.c
  • src/interfaces/ecpg/pgtypeslib/timestamp.c
  • **呼び出し先**:
  • 標準ライブラリ関数 strdup

関連データ構造

  • なし

失敗モードと境界条件

  • メモリ不足により strdup が失敗した場合、errno が ENOMEM に設定され、関数は NULL を返す。

保守担当者向けメモ

  • 本関数で確保したメモリは、Windows 環境での DLL 境界を跨いだ解放の整合性を保つため、必ず PGTYPESchar_free を通じて解放すること。
