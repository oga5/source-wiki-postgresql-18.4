Datum

定義

  • 種別: typedef
  • 定義箇所: contrib/hstore_plpython/hstore_plpython.c:20（および postgres.h, fmgr.h）

目的

PostgreSQL における汎用的なデータ表現型です。SQL レベルのデータ値（数値、文字列、ポインタ等）を、PostgreSQL の関数インターフェースを通じて受け渡しするために使用されます。

シグネチャまたは形状

typedef uintptr_t Datum;

※ postgres.h 等のコア定義に基づく。

入力と出力

  • **入力**: 任意の PostgreSQL データ型（値そのもの、またはメモリ上のデータへのポインタ）。
  • **出力**: 関数呼び出しの結果として返されるデータ値、または関数引数として渡されるデータ値。

前提条件と不変条件

  • Datum は、値が 4 バイトまたは 8 バイト（プラットフォーム依存）の整数として表現可能であるか、あるいはメモリ上のデータ構造を指すポインタであることを前提とします。
  • ポインタとして使用される場合、そのメモリ領域は PostgreSQL のメモリコンテキスト管理下にある必要があります。

アルゴリズムまたは振る舞い

  • Datum 自体は単なる型定義であり、具体的な振る舞いはそれを扱う関数（fmgr インターフェース等）に依存します。
  • hstore_plpython モジュール内では、hstore 型のデータや Python オブジェクトへの変換過程において、データ値を抽象化して保持するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のコアエンジン、および hstore_plpython 内の変換関数（hstore_to_plpython 等）。
  • **呼び出し先**: Datum を引数として受け取る PostgreSQL の内部関数、または Datum を返す関数。
  • 注: 提供されたソース抜粋内では、具体的な関数呼び出しの連鎖は明示されていません。

関連データ構造

  • HStore: hstore 型の内部構造体。
  • PyObject: Python 側のデータ構造。
  • fmgr インターフェース: Datum を介して関数を呼び出すための PostgreSQL の関数管理機構。

失敗モードと境界条件

  • **型不整合**: Datum が指すデータ型と、それを解釈する関数が期待する型が一致しない場合、メモリ破壊やセグメンテーション違反が発生する可能性があります。
  • **ポインタの有効性**: Datum がポインタとして使用される場合、参照先のメモリが解放済み（pfree 後など）であると未定義動作を引き起こします。

保守担当者向けメモ

  • Datum は PostgreSQL 全体で多用される基盤型です。本モジュール内での使用は、主に hstore 構造体へのポインタや、型変換関数間のデータ受け渡しに限定されています。
  • 変更を加える際は、postgres.h での定義と整合性が取れていることを確認してください。
