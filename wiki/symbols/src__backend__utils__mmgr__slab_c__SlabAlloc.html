SlabAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/slab.c:630

目的

SLAB メモリコンテキストから、指定された固定サイズのメモリチャンクを割り当てる。

シグネチャまたは形状

void *SlabAlloc(MemoryContext context, Size size, int flags)

入力と出力

  • **入力**:
  • context: 対象の SlabContext。
  • size: 要求するメモリサイズ（コンテキスト作成時の chunkSize と一致する必要がある）。
  • flags: 割り当てフラグ。
  • **出力**:
  • 割り当てられたメモリ領域へのポインタ。

前提条件と不変条件

  • context は有効な SlabContext であること。
  • size はコンテキスト作成時に定義された chunkSize と一致すること。
  • slab->curBlocklistIndex は常に有効な範囲（0 から chunksPerBlock まで）にあること。

アルゴリズムまたは振る舞い

1. 要求サイズが chunkSize と一致するか検証し、不一致の場合は SlabAllocInvalidSize を呼び出す。 2. curBlocklistIndex が 0 の場合（利用可能な部分充填ブロックがない）、SlabAllocFromNewBlock を呼び出して新規ブロックから割り当てる。 3. curBlocklistIndex が 0 以外の場合、該当リストの先頭ブロックからチャンクを取得する。 4. チャンク取得後、ブロック内の空き数（nfree）に基づき、ブロックを適切な blocklist へ移動する。 5. ブロックが空になった場合、SlabFindNextBlockListIndex を用いて curBlocklistIndex を更新する。 6. 最後に SlabAllocSetupNewChunk を呼び出し、チャンクの初期化とメタデータ設定を行ってポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/mcxt.c: MCTX_SLAB_ID の alloc 関数ポインタとして登録。
  • **呼び出し先**:
  • SlabAllocInvalidSize
  • SlabAllocFromNewBlock
  • SlabGetNextFreeChunk
  • SlabBlocklistIndex
  • SlabFindNextBlockListIndex
  • SlabAllocSetupNewChunk

関連データ構造

  • SlabContext: アロケータのメイン構造体。
  • SlabBlock: メモリブロック管理構造体。
  • MemoryChunk: 割り当てられるメモリのヘッダー情報。

失敗モードと境界条件

  • **サイズ不一致**: 要求サイズが chunkSize と異なる場合、SlabAllocInvalidSize が実行される。
  • **ブロック枯渇**: 部分的に空きがあるブロックが存在しない場合、SlabAllocFromNewBlock により新規ブロックの確保が試行される。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、パフォーマンス上の理由からロジックの一部がヘルパー関数に分離されている。
  • SlabRealloc はサポートされていないため、サイズ変更は考慮不要。
  • MEMORY_CONTEXT_CHECKING が有効な場合、内部で整合性チェックが実行される。
