PQcancelBlocking

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:189

目的

PostgreSQLサーバーに対して、実行中のクエリをキャンセルするためのリクエストをブロッキング（同期）方式で送信します。

シグネチャまたは形状

int PQcancelBlocking(PGcancelConn *cancelConn);

入力と出力

  • **入力**: cancelConn (PGcancelConn *) - キャンセル要求に使用する接続オブジェクト。
  • **出力**: int - 成功した場合は 1、失敗した場合は 0 を返します。

前提条件と不変条件

  • cancelConn は PQcancelCreate 等で適切に初期化されている必要があります。
  • 本関数はブロッキング動作を行うため、呼び出し元のスレッドは処理が完了するまで待機します。

アルゴリズムまたは振る舞い

1. PQcancelStart(cancelConn) を呼び出し、キャンセル用接続の確立を開始します。 2. PQcancelStart が失敗した場合は直ちに 0 を返します。 3. 成功した場合、内部の pqConnectDBComplete を呼び出し、接続およびキャンセルリクエストの送信が完了するまで待機します。 4. 最終的な処理結果（成功なら 1、失敗なら 0）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/fe_utils/connect_utils.c:166
  • src/test/isolation/isolationtester.c:951
  • src/test/modules/libpq_pipeline/libpq_pipeline.c:284
  • **呼び出し先**:
  • PQcancelStart
  • pqConnectDBComplete

関連データ構造

  • PGcancelConn: キャンセル要求の状態や接続情報を保持する構造体。

失敗モードと境界条件

  • PQcancelStart が失敗した場合、本関数は 0 を返します。
  • ネットワークエラーやサーバー側の応答拒否など、pqConnectDBComplete が失敗する要因がある場合、本関数は 0 を返します。
  • エラーの詳細は PQcancelErrorMessage(cancelConn) を通じて取得可能です。

保守担当者向けメモ

  • 本関数はブロッキングAPIです。シグナルハンドラ内での安全性が求められる場合は、代わりに PQcancel の使用を検討してください。
  • 非同期処理が必要な新規開発では、PQcancelStart と PQcancelPoll を組み合わせた非同期APIの使用が推奨されます。
