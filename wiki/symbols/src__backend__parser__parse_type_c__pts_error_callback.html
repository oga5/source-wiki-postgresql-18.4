pts_error_callback

定義

  • **種別**: function
  • **定義箇所**: src/backend/parser/parse_type.c:718

目的

型名の解決失敗時に、エラーメッセージのコンテキストとして「無効な型名」であることを報告するためのコールバック関数です。

シグネチャまたは形状

static void pts_error_callback(void *arg);

入力と出力

  • **入力 (arg)**: 型名を表す文字列 (const char *) へのポインタ。
  • **出力**: なし（void）。内部で errcontext を呼び出し、エラー情報をスタックに積みます。

前提条件と不変条件

  • arg は有効なヌル終端文字列を指している必要があります。
  • PostgreSQL のエラー報告メカニズム（errcontext）がアクティブな状態で呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. 引数 arg を const char * 型にキャストします。 2. errcontext マクロを使用して、"invalid type name \"%s\"" という形式でエラーコンテキスト情報を設定します。これにより、エラー発生時に SQL 文の解析位置と併せて、どの型名が問題であったかが報告されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_type.c 内の型解決処理において、エラー発生時のコンテキスト設定用として参照されています。
  • **呼び出し先**: errcontext（PostgreSQL のエラー報告用マクロ）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • arg が NULL の場合、errcontext 内で文字列フォーマット時にクラッシュする可能性があります。

保守担当者向けメモ

  • 本関数は ErrorContextCallback として使用されることを想定しています。
  • エラーメッセージの文言を変更する場合は、PostgreSQL の翻訳ガイドラインに従ってください。
