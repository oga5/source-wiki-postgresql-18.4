LLVMGetFunctionType

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_wrap.cpp:43

目的

LLVMのC APIでは直接提供されていない、LLVMValueRef（関数）からその関数型（LLVMTypeRef）を抽出する機能を提供します。

シグネチャまたは形状

LLVMTypeRef LLVMGetFunctionType(LLVMValueRef r)

入力と出力

  • **入力**: LLVMValueRef r - LLVMの関数を表す不透明ポインタ。
  • **出力**: LLVMTypeRef - 入力された関数に対応するLLVMの関数型。

前提条件と不変条件

  • 入力 r は、内部的に llvm::Function オブジェクトへキャスト可能である必要があります。

アルゴリズムまたは振る舞い

1. llvm::unwrap<llvm::Function>(r) を呼び出し、C APIの不透明ポインタをLLVM C++ APIの llvm::Function オブジェクトに変換します。 2. 変換されたオブジェクトに対して getFunctionType() を呼び出し、関数型を取得します。 3. 取得した型を llvm::wrap() を用いてC API形式の LLVMTypeRef に変換して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit.c
  • src/backend/jit/llvm/llvmjit_deform.c
  • src/backend/jit/llvm/llvmjit_expr.c
  • **呼び出し先**:
  • llvm::unwrap<llvm::Function>
  • llvm::Function::getFunctionType
  • llvm::wrap

関連データ構造

  • LLVMValueRef (C API) / llvm::Function (C++ API)
  • LLVMTypeRef (C API) / llvm::FunctionType (C++ API)

失敗モードと境界条件

  • 入力 r が llvm::Function ではない場合、llvm::unwrap の内部でアサーション失敗や未定義動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数はLLVM C APIの不足を補うラッパーです。LLVMのバージョンアップによりAPIが変更された場合、llvm::unwrap や llvm::wrap の使用方法が影響を受ける可能性があるため注意してください。
