ReportApplyConflict

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/conflict.c:102

目的

論理レプリケーション中に発生した競合をログへ記録し、統計情報を更新する。

シグネチャまたは形状

void ReportApplyConflict(EState *estate, ResultRelInfo *relinfo, int elevel,
                        ConflictType type, TupleTableSlot *searchslot,
                        TupleTableSlot *remoteslot, List *conflicttuples)

入力と出力

  • **入力**:
  • estate: 実行状態。
  • relinfo: 競合が発生したリレーション情報。
  • elevel: ログレベル（ERROR や LOG など）。
  • type: 競合の種類（ConflictType）。
  • searchslot, remoteslot: 競合に関与するタプルスロット。
  • conflicttuples: 競合したタプルの詳細リスト（ConflictTupleInfo のリスト）。
  • **出力**: なし（副作用としてログ出力および統計更新を行う）。

前提条件と不変条件

  • conflicttuples リストが空でない場合、各要素に対して詳細な競合情報が生成される。
  • MySubscription が有効である必要がある（pgstat_report_subscription_conflict 内部で使用）。

アルゴリズムまたは振る舞い

1. initStringInfo で詳細メッセージ用のバッファを初期化する。 2. conflicttuples リストを走査し、errdetail_apply_conflict を呼び出して各競合タプルの詳細情報をバッファに追記する。 3. pgstat_report_subscription_conflict を呼び出し、サブスクリプションの競合統計を更新する。 4. ereport を使用し、指定された elevel で競合内容（リレーション名、競合タイプ）と構築済みの詳細情報をログに出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execReplication.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • errdetail_apply_conflict
  • pgstat_report_subscription_conflict
  • errcode_apply_conflict
  • ereport
  • initStringInfo

関連データ構造

  • ConflictType: 競合の種類を定義する列挙型。
  • ConflictTupleInfo: 競合したタプルのメタデータ（xmin, origin, ts, indexoid 等）を保持する構造体。
  • ResultRelInfo: ターゲットリレーションの記述子。

失敗モードと境界条件

  • track_commit_timestamp が無効な場合、競合ログ内のタイムスタンプ情報は欠落する。
  • elevel に ERROR が指定された場合、ログ出力後にトランザクションが中断される。

保守担当者向けメモ

  • 本関数は競合の「報告」のみを行い、「解決」ロジックは含まない。
  • StringInfo を使用して詳細メッセージを構築しているため、メモリ管理に留意すること。
  • 競合タイプごとのメッセージ整形は errdetail_apply_conflict に依存している。
