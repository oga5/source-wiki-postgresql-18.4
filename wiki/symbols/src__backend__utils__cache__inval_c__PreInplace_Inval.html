PreInplace_Inval

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/inval.c:1249

目的

インプレース更新（トランザクションを伴わないカタログ更新など）の実行前に、リレーションキャッシュの初期化ファイルを無効化する必要があるかどうかを判定し、必要に応じて処理を実行する。

シグネチャまたは形状

void PreInplace_Inval(void)

入力と出力

  • 入力: なし（グローバルな inplaceInvalInfo を参照）
  • 出力: なし

前提条件と不変条件

  • CritSectionCount == 0 であること（クリティカルセクション内ではないこと）。

アルゴリズムまたは振る舞い

1. CritSectionCount が 0 であることをアサートで確認する。 2. グローバル変数 inplaceInvalInfo が存在し、かつそのメンバ RelcacheInitFileInval が真である場合、RelationCacheInitFilePreInvalidate() を呼び出してリレーションキャッシュの初期化ファイルを無効化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/heapam.c:6538
  • 呼び出し先: RelationCacheInitFilePreInvalidate()

関連データ構造

  • inplaceInvalInfo: インプレース更新に関連する無効化情報を保持する構造体。

失敗モードと境界条件

  • inplaceInvalInfo が NULL の場合、何も実行されない。
  • クリティカルセクション内で呼び出された場合、アサートにより異常終了する。

保守担当者向けメモ

  • 本関数は、relmapper 等のトランザクション管理外の更新処理において、キャッシュの整合性を保つために使用される。
  • AtInplace_Inval() と対で使用されることが想定されている。
