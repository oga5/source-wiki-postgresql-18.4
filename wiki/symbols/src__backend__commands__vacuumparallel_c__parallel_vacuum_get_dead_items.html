parallel_vacuum_get_dead_items

定義

  • 種別: function
  • 定義箇所: src/backend/commands/vacuumparallel.c:466

目的

並列VACUUM処理において、共有メモリ上の削除対象タプル情報（TidStore）およびそのメタデータへのポインタを取得する。

シグネチャまたは形状

TidStore *parallel_vacuum_get_dead_items(ParallelVacuumState *pvs, VacDeadItemsInfo **dead_items_info_p)

入力と出力

  • **入力**:
  • pvs: 並列VACUUMの状態を保持する ParallelVacuumState 構造体へのポインタ。
  • dead_items_info_p: VacDeadItemsInfo 構造体へのポインタを格納するための出力用引数。
  • **出力**:
  • 戻り値: pvs->dead_items が保持する TidStore へのポインタ。
  • 副作用: *dead_items_info_p に pvs->shared->dead_items_info のアドレスが代入される。

前提条件と不変条件

  • pvs が有効な ParallelVacuumState を指していること。
  • pvs->shared が初期化済みであること。

アルゴリズムまたは振る舞い

1. pvs->shared->dead_items_info のアドレスを、引数 dead_items_info_p を介して呼び出し元に渡す。 2. pvs->dead_items を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/vacuumlazy.c: vacrel->dead_items の初期化時に呼び出される。
  • **呼び出し先**:
  • なし（直接的な関数呼び出しは行わない）。

関連データ構造

  • ParallelVacuumState: 並列VACUUMの全体状態を管理する構造体。
  • TidStore: 削除対象のタプルIDを格納するデータ構造。
  • VacDeadItemsInfo: 削除対象タプルのメタデータ。

失敗モードと境界条件

  • pvs が NULL の場合、デリファレンスによりクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は、並列VACUUMのリーダープロセスおよびワーカープロセスが共有メモリ上の削除対象タプル情報にアクセスするためのアクセサとして機能する。
  • 戻り値の TidStore は ParallelVacuumState のライフサイクルに依存するため、解放タイミングに注意すること。
