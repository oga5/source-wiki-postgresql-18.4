NamedLWLockTrancheRequest

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/lmgr/lwlock.c:221

目的

動的に割り当てられる軽量ロック（LWLock）のグループ（Tranche）を登録するためのリクエスト情報を保持する。

シグネチャまたは形状

typedef struct NamedLWLockTrancheRequest
{
    char        tranche_name[NAMEDATALEN];
    int         num_lwlocks;
} NamedLWLockTrancheRequest;

入力と出力

  • tranche_name: トランチを識別するための名称（最大 NAMEDATALEN バイト）。
  • num_lwlocks: 当該トランチに割り当てる LWLock の個数。

前提条件と不変条件

  • tranche_name は一意である必要がある（システム全体で重複しないことが期待される）。
  • num_lwlocks は正の整数である必要がある。

アルゴリズムまたは振る舞い

本構造体は、共有メモリ上に配置される LWLock トランチの配列（NamedLWLockTrancheRequestArray）を構築するための定義体として機能する。システム初期化時にこの構造体の配列を用いて、必要な数の LWLock を確保し、名前を割り当てる処理が行われる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/lwlock.c 内の初期化処理（CreateLWLocks 等）において、トランチの割り当てサイズ計算や配列の初期化に使用される。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはない。

関連データ構造

  • NamedLWLockTrancheRequestArray: 本構造体の配列を指すポインタ。
  • LWLock: 本構造体で定義された数だけ実際に生成されるロック本体。

失敗モードと境界条件

  • tranche_name が NAMEDATALEN を超える場合、切り詰められる可能性がある。
  • 共有メモリの不足により、num_lwlocks で指定された数のロックを確保できない場合、システム起動時にエラーが発生する可能性がある。

保守担当者向けメモ

  • 本構造体は lwlock.c 内で静的に管理される配列の要素として使用される。
  • 新しいトランチを追加する場合は、この構造体を通じて登録を行う必要がある。
  • 呼び出し元の詳細なコールグラフは、lwlock.c 内の CreateLWLocks 周辺のロジックを参照すること。
