pgstattuple_v1_5

定義

  • 種別: function
  • 定義箇所: contrib/pgstattuple/pgstattuple.c:194

目的

指定されたリレーション（テーブルまたはインデックス）の物理的な統計情報を算出するための SQL 呼び出し可能なエントリポイントです。バージョン 1.5 以降のインターフェースを提供します。

シグネチャまたは形状

Datum pgstattuple_v1_5(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、対象リレーション名（text 型）を受け取ります。
  • **出力**: Datum 型として、統計情報を含む複合型を返却します。

前提条件と不変条件

  • 対象リレーションに対して AccessShareLock を取得します。
  • 実行ユーザーには、対象リレーションに対する適切なアクセス権限が必要です。

アルゴリズムまたは振る舞い

1. 引数からリレーション名を解析し、RangeVar を生成します。 2. relation_openrv を使用して、AccessShareLock モードで対象リレーションを開きます。 3. pgstat_relation 関数を呼び出し、リレーションの種類に応じた統計情報の集計処理へ委譲します。 4. 集計結果を Datum に変換して呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロによる登録経由）。
  • **呼び出し先**: makeRangeVarFromNameList、textToQualifiedNameList、relation_openrv、pgstat_relation。

関連データ構造

  • Relation: 開かれたリレーションのメタデータを保持する構造体。
  • RangeVar: リレーション名を特定するための構造体。

失敗モードと境界条件

  • 指定されたリレーション名が存在しない場合、relation_openrv によりエラーが発生します。
  • 権限がない場合、アクセス拒否エラーが発生します。

保守担当者向けメモ

  • 本関数は pgstattuple 関数のバージョン 1.5 以降のインターフェースです。古いバージョンとの互換性維持のため、サフィックスなしの関数とは別に定義されています。
  • 統計情報の算出は物理的な全ページ走査を伴うため、大規模なリレーションに対しては I/O 負荷が高い点に注意してください。
