pgstat_reset_of_kind

定義

  • src/backend/utils/activity/pgstat.c:874

目的

指定された統計種別（PgStat_Kind）に関連する統計情報をリセットする。

シグネチャまたは形状

void pgstat_reset_of_kind(PgStat_Kind kind)

入力と出力

  • **入力**: kind (リセット対象の統計種別)
  • **出力**: なし（副作用として共有メモリ上の統計情報がリセットされる）

前提条件と不変条件

  • pgstat_get_kind_info(kind) が有効な PgStat_KindInfo 構造体を返すこと。
  • 呼び出し時に GetCurrentTimestamp() により現在時刻が取得可能であること。

アルゴリズムまたは振る舞い

1. pgstat_get_kind_info(kind) を呼び出し、対象種別のメタ情報を取得する。 2. 現在時刻を GetCurrentTimestamp() で取得する。 3. kind_info->fixed_amount が真の場合:

  • kind_info->reset_all_cb(ts) を呼び出し、種別全体を一括リセットする。

    • 4. kind_info->fixed_amount が偽の場合:

  • pgstat_reset_entries_of_kind(kind, ts) を呼び出し、該当種別の各エントリを個別にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c 内の統計リセット関数群（ARCHIVER, BGWRITER, CHECKPOINTER, IO, SLRU, WAL 等の種別に対して呼び出し）。
  • **呼び出し先**:
  • pgstat_get_kind_info
  • GetCurrentTimestamp
  • kind_info->reset_all_cb (関数ポインタ)
  • pgstat_reset_entries_of_kind

関連データ構造

  • PgStat_Kind: 統計種別を識別する列挙型。
  • PgStat_KindInfo: 統計種別の振る舞いを定義する構造体。

失敗モードと境界条件

  • 無効な kind が渡された場合の挙動については、pgstat_get_kind_info の実装に依存する。
  • 共有メモリへのアクセス権限やロック状態に関する境界条件は、内部で呼び出される reset_all_cb または pgstat_reset_entries_of_kind の実装に依存する。

保守担当者向けメモ

  • 本関数は統計システムの抽象化レイヤーであり、具体的なリセット処理は各統計種別のコールバックまたは pgstat_reset_entries_of_kind に委譲される。
  • 新しい統計種別を追加する際は、PgStat_KindInfo の fixed_amount フラグを適切に設定し、必要に応じて reset_all_cb を定義すること。
