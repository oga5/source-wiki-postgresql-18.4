SharedTypmodTableEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/typcache.c:220

目的

共有メモリ上で、特定の型修飾子（typmod）に対応するタプル記述子（TupleDesc）を管理するためのエントリ構造体です。

シグネチャまたは形状

typedef struct SharedTypmodTableEntry
{
	uint32		typmod;
	dsa_pointer shared_tupdesc;
} SharedTypmodTableEntry;

入力と出力

  • typmod (uint32): 識別対象となる型修飾子。
  • shared_tupdesc (dsa_pointer): 共有メモリ領域（DSA）上のタプル記述子を指すポインタ。

前提条件と不変条件

  • 本構造体は共有メモリ上に配置されることを前提としています。
  • shared_tupdesc は有効な DSA ポインタである必要があります。

アルゴリズムまたは振る舞い

  • 共有ハッシュテーブル内のエントリとして使用され、特定の typmod に関連付けられたタプル記述子を検索・保持するために利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/typcache.c 内の共有レコード型レジストリ関連関数（SharedRecordTypmodRegistry 関連）から参照されています。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • SharedRecordTypmodRegistry: 本エントリを格納するハッシュテーブルの管理構造体。
  • dsa_pointer: 共有メモリ領域を指すためのポインタ型。

失敗モードと境界条件

  • 共有メモリの枯渇により、shared_tupdesc の割り当てが失敗する可能性があります。
  • 検索時に該当する typmod が存在しない場合、ハッシュテーブルの検索処理は失敗します。

保守担当者向けメモ

  • 本構造体は並列クエリ環境下でのレコード型キャッシュ共有に使用されます。
  • dsa_pointer を扱うため、メモリのライフサイクル管理には注意が必要です。
