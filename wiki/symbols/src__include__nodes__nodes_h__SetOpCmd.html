SetOpCmd

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/nodes.h:402

目的

SQLの集合演算（INTERSECTおよびEXCEPT）の種類を識別するために使用される列挙型です。

シグネチャまたは形状

typedef enum SetOpCmd
{
    SETOPCMD_INTERSECT,
    SETOPCMD_INTERSECT_ALL,
    SETOPCMD_EXCEPT,
    SETOPCMD_EXCEPT_ALL,
} SetOpCmd;

入力と出力

  • 入力: 該当なし（列挙型定数）
  • 出力: 集合演算の種別を示す識別子

前提条件と不変条件

  • 集合演算ノード（SetOp）の構成要素として使用されます。
  • ALL サフィックスが付与された値は、重複行を保持する演算に対応します。

アルゴリズムまたは振る舞い

  • SETOPCMD_INTERSECT: 共通する行を抽出します。
  • SETOPCMD_INTERSECT_ALL: 重複を考慮して共通する行を抽出します。
  • SETOPCMD_EXCEPT: 左側の集合から右側の集合に含まれる行を除外します。
  • SETOPCMD_EXCEPT_ALL: 重複を考慮して左側の集合から行を除外します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/optimizer/plan/createplan.c: make_setop 関数内で使用。
  • src/backend/optimizer/prep/prepunion.c: 集合演算の型選択に使用。
  • src/backend/optimizer/util/pathnode.c: 関連するパスノード生成に使用。
  • 呼び出し先: 情報なし。

関連データ構造

  • SetOp (プランノード): src/include/nodes/plannodes.h で定義され、SetOpCmd 型のフィールド cmd を保持します。
  • SetOpPath (パスノード): src/include/nodes/pathnodes.h で定義され、同様に cmd フィールドを保持します。

失敗モードと境界条件

  • 該当なし（列挙型のため、定義外の値が代入された場合の挙動は未定義）。

保守担当者向けメモ

  • 本列挙型は SetOpStrategy と組み合わせて使用されることが一般的です。
  • 新しい集合演算を追加する場合は、関連する copyfuncs.c や equalfuncs.c 等のノードサポート関数も更新する必要があります。
