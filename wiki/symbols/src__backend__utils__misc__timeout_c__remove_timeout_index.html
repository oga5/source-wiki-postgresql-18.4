remove_timeout_index

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:136

目的

現在有効なタイムアウト管理配列（active_timeouts）から、指定されたインデックスのタイムアウトエントリを削除し、配列を詰め直す。

シグネチャまたは形状

static void remove_timeout_index(int index)

入力と出力

  • **入力**: index (削除対象の active_timeouts 配列内インデックス)
  • **出力**: なし（副作用として active_timeouts 配列の内容と num_active_timeouts が更新される）

前提条件と不変条件

  • index は 0 以上 num_active_timeouts - 1 の範囲内であること。
  • 削除対象の active_timeouts[index] は active 状態（active == true）であること。

アルゴリズムまたは振る舞い

1. index が範囲外の場合、elog(FATAL) を発生させる。 2. 対象エントリの active フラグを false に設定する。 3. index より後ろにあるすべての要素を一つずつ前方にシフトし、配列の隙間を埋める。 4. num_active_timeouts をデクリメントし、有効な要素数を更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/timeout.c 内の各所（find_active_timeout の戻り値や、先頭要素の削除時に直接呼び出される）。
  • **呼び出し先**:
  • elog (エラー発生時)

関連データ構造

  • active_timeouts: 有効なタイムアウトを保持する静的配列。
  • num_active_timeouts: 現在の有効なタイムアウト数を示すグローバル変数。

失敗モードと境界条件

  • index が範囲外の場合、プロセスは FATAL エラーで停止する。
  • num_active_timeouts が 0 の状態で呼び出された場合、範囲外チェックによりエラーとなる。

保守担当者向けメモ

  • 本関数は配列のシフト操作を行うため、active_timeouts の要素数が多い場合には計算量に注意が必要だが、通常タイムアウトの登録数はごく少数であるため問題とならない。
  • 呼び出し元で find_active_timeout を使用する場合、対象が存在しない（-1 が返る）可能性を考慮する必要がある。
