coerce_record_to_complex

定義

  • **種別**: function
  • **定義箇所**: src/backend/parser/parse_coerce.c:1009

目的

RECORD 型の式を、指定された複合型（またはそのドメイン型）へ変換する。

シグネチャまたは形状

Node *coerce_record_to_complex(ParseState *pstate, Node *node, Oid targetTypeId, CoercionContext ccontext, CoercionForm cformat, int location)

入力と出力

  • **入力**:
  • pstate: パーサーの状態。
  • node: 変換対象の RECORD 型ノード（RowExpr または Var）。
  • targetTypeId: 変換先の複合型またはドメイン型の OID。
  • ccontext, cformat, location: 変換の文脈、形式、およびエラー位置情報。
  • **出力**: 変換後の RowExpr ノード、またはドメイン型の場合は CoerceToDomain ノード。

前提条件と不変条件

  • 入力 node は RowExpr または Var（varattno == InvalidAttrNumber）である必要がある。
  • 変換先の targetTypeId は複合型、または複合型上のドメイン型でなければならない。
  • 変換後の各フィールドは、ターゲット型の属性型と互換性がある必要がある。

アルゴリズムまたは振る舞い

1. **入力の展開**: RowExpr から引数リストを取得するか、Var の場合は expandNSItemVars を使用して列を展開する。 2. **型情報の取得**: getBaseTypeAndTypmod でターゲットの基底型を特定し、lookup_rowtype_tupdesc でタプル記述子を取得する。 3. **フィールドの変換**: ターゲット型の各属性に対してループ処理を行う。

  • 削除された列（attisdropped）には NULL 定数を挿入する。
  • 各フィールドに対し coerce_to_target_type を再帰的に呼び出し、型変換を行う。

    • 4. **検証**: 入力列数とターゲットの列数が一致しない場合はエラーを報告する。 5. **ノード構築**: RowExpr を生成し、ターゲットがドメイン型であれば coerce_to_domain を適用して制約チェックを追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_coerce.c 内の型変換ロジック（coerce_to_target_type 等から間接的に呼び出される）。
  • **呼び出し先**: GetNSItemByRangeTablePosn, expandNSItemVars, getBaseTypeAndTypmod, lookup_rowtype_tupdesc, coerce_to_target_type, coerce_to_domain, ReleaseTupleDesc 等。

関連データ構造

  • RowExpr: 変換対象および結果の行式ノード。
  • TupleDesc: ターゲット複合型の構造定義。
  • ParseState: パーサーのコンテキスト情報。

失敗モードと境界条件

  • **列数不一致**: 入力列数がターゲットの列数と異なる場合、ERRCODE_CANNOT_COERCE でエラー。
  • **型変換不可**: 各フィールドの型変換が失敗した場合、ERRCODE_CANNOT_COERCE でエラー。
  • **不正な入力**: RowExpr でも Var でもないノードが渡された場合、エラー。

保守担当者向けメモ

  • 本関数は RECORD 型の動的な性質を静的な複合型に適合させるための重要な処理です。
  • 削除された列（attisdropped）の処理において、makeNullConst を使用して型を INT4OID としていますが、これは単なるプレースホルダーであり、型自体は重要ではありません。
  • ターゲットがドメイン型の場合、最後に coerce_to_domain を通すことで制約が適用される設計になっています。
