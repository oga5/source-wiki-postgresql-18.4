relatt_cache_syshash

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/attoptcache.c:83

目的

属性オプションキャッシュ（AttoptCacheHash）において、AttoptCacheKey からハッシュ値を計算する。システムキャッシュ（syscache）とハッシュ計算ロジックを共有することで、無効化処理の効率化を図る。

シグネチャまたは形状

static uint32 relatt_cache_syshash(const void *key, Size keysize)

入力と出力

  • **入力**:
  • key: AttoptCacheKey 構造体へのポインタ。
  • keysize: キーのサイズ（sizeof(AttoptCacheKey) である必要がある）。
  • **出力**:
  • uint32: 計算されたハッシュ値。

前提条件と不変条件

  • keysize は sizeof(AttoptCacheKey) と一致しなければならない（Assert により検証）。
  • key は AttoptCacheKey 型としてキャスト可能であること。

アルゴリズムまたは振る舞い

1. 引数 key を AttoptCacheKey 型としてキャストする。 2. keysize が期待されるサイズと一致するかを確認する。 3. ATTNUM システムキャッシュの識別子と、キーに含まれる attrelid および attnum を使用し、GetSysCacheHashValue2 を呼び出してハッシュ値を算出・返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/attoptcache.c: キャッシュ初期化時のハッシュテーブル制御構造体（ctl.hash）の設定に使用。
  • **呼び出し先**:
  • GetSysCacheHashValue2: システムキャッシュのハッシュ値計算関数。

関連データ構造

  • AttoptCacheKey: キャッシュ検索のキーとなる構造体。
  • AttoptCacheHash: 本関数をハッシュ関数として利用するハッシュテーブル。

失敗モードと境界条件

  • keysize が sizeof(AttoptCacheKey) と異なる場合、アサーション違反によりプロセスが停止する可能性がある。

保守担当者向けメモ

  • 本関数は syscache と同じハッシュ計算ロジックを共有しているため、ATTNUM システムキャッシュのハッシュ計算仕様に変更があった場合は本関数も影響を受ける可能性がある。
