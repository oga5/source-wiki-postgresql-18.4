PLyNumber_ToJsonbValue

定義

  • 種別: function
  • 定義箇所: contrib/jsonb_plpython/jsonb_plpython.c:359

目的

Python オブジェクトを PostgreSQL の jsonb 型で扱える数値形式（jbvNumeric）に変換する。

シグネチャまたは形状

JsonbValue *PLyNumber_ToJsonbValue(PyObject *obj, JsonbValue *jbvNum)

入力と出力

  • **入力**:
  • obj: 変換対象の Python オブジェクト。
  • jbvNum: 変換結果を格納する JsonbValue 構造体へのポインタ。
  • **出力**:
  • 変換後の JsonbValue 構造体へのポインタを返す。

前提条件と不変条件

  • obj は PLyObject_AsString を通じて文字列として解釈可能であること。
  • JSON 仕様に基づき、NaN（非数）および無限大（Infinity）は変換不可であり、これらが含まれる場合はエラーを発生させる。

アルゴリズムまたは振る舞い

1. PLyObject_AsString を使用して Python オブジェクトを文字列に変換する。 2. numeric_in 関数を呼び出し、文字列を PostgreSQL の Numeric 型に変換する。この際、エラー発生時は PG_TRY/PG_CATCH ブロックで捕捉し、変換失敗を報告する。 3. 変換された Numeric 値が NaN または Infinity であるかを確認し、該当する場合は ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE でエラーを投げる。 4. jbvNum->type を jbvNumeric に設定し、jbvNum->val.numeric に変換済み数値を格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/jsonb_plpython/jsonb_plpython.c 内の PLyObject_ToJsonbValue から呼び出される（ソースコード上の直接的な参照に基づく）。
  • **呼び出し先**:
  • PLyObject_AsString
  • DirectFunctionCall3 (numeric_in)
  • numeric_is_nan, numeric_is_inf
  • ereport

関連データ構造

  • JsonbValue: JSONB の各要素を保持する構造体。
  • Numeric: PostgreSQL の数値型。
  • PyObject: Python のオブジェクト。

失敗モードと境界条件

  • **変換失敗**: Python オブジェクトが数値として解釈できない文字列の場合、ERRCODE_DATATYPE_MISMATCH でエラーとなる。
  • **範囲外エラー**: NaN または Infinity の場合、ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE でエラーとなる。

保守担当者向けメモ

  • 本関数は plpython 拡張のメモリ管理と PostgreSQL のエラーハンドリングを統合している。PG_TRY ブロック内で確保されたメモリやリソースの解放順序に注意すること。
  • numeric_in の呼び出しには InvalidOid と -1 が渡されており、型修飾子なしの標準的な数値変換が行われる。
