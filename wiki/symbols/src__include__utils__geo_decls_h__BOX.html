BOX

定義

  • 種別: struct
  • 定義箇所: src/include/utils/geo_decls.h:140

目的

PostgreSQLにおける矩形領域（Box）を表現するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	Point		high,
				low;
} BOX;

入力と出力

  • 入力: 2つの Point 型（high および low）を保持します。
  • 出力: 幾何演算関数において、矩形の範囲を示すデータとして利用されます。

前提条件と不変条件

  • high および low は Point 構造体です。
  • 幾何演算の文脈において、high が矩形の右上端、low が左下端の座標として扱われることが一般的ですが、構造体自体には座標値の大小関係を強制する制約は定義されていません。

アルゴリズムまたは振る舞い

  • 幾何データ型として、空間インデックス（GiSTなど）や矩形同士の包含・交差判定に使用されます。
  • src/backend/access/gist/gistproc.c 等の関数において、矩形の面積計算や、2つの矩形を包含する最小矩形の算出（rt_box_union）などに利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/gist/gistproc.c: gist_box_leaf_consistent, rtree_internal_consistent, rt_box_union, size_box, box_penalty, adjustBox 等で利用されています。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • Point: BOX を構成する頂点データとして使用されます。

失敗モードと境界条件

  • 数値演算の特性上、NaN が含まれる場合の挙動は定義されていません（比較関数 FPeq 等は NaN を FALSE と判定します）。
  • 面積計算において、結果が +Infinity になる可能性があります。

保守担当者向けメモ

  • PATH や POLYGON とは異なり、固定長の構造体です。
  • 座標の順序（high と low）に関する厳密なバリデーションは本構造体定義には含まれていないため、操作関数側で適切に処理する必要があります。
