throttle

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_throttle.c:133

目的

ベースバックアップのデータ転送レートを監視し、設定された最大レートを超過しないよう待機時間を挿入してスロットリングを行う。

シグネチャまたは形状

static void throttle(bbsink_throttle *sink, size_t increment)

入力と出力

  • sink: スロットリング状態を保持する bbsink_throttle 構造体へのポインタ。
  • increment: 今回転送されたデータ量（バイト単位）。
  • 戻り値: なし。

前提条件と不変条件

  • sink->throttling_counter は常に 0 以上であること。
  • 整数演算で処理されるため、throttling_sample 未満の端数は次回の呼び出しに持ち越される。

アルゴリズムまたは振る舞い

1. increment を throttling_counter に加算する。 2. throttling_counter が throttling_sample に達していない場合は即座に終了する。 3. 達している場合、経過時間と転送量から必要な待機時間を算出する。 4. 転送が速すぎる場合、WaitLatch を用いてループ内で待機する。

  • 待機中、CHECK_FOR_INTERRUPTS() を呼び出し、停止要求に対して応答可能にする。
  • WL_LATCH_SET が発生した場合は割り込みチェックを行う。

    • 5. 待機完了後、throttling_counter を throttling_sample で除算した余りに更新する。 6. throttled_last を現在のタイムスタンプで更新する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • bbsink_throttle_archive_contents (src/backend/backup/basebackup_throttle.c:112)
  • bbsink_throttle_manifest_contents (src/backend/backup/basebackup_throttle.c:123)
  • 呼び出し先:
  • GetCurrentTimestamp()
  • ResetLatch()
  • CHECK_FOR_INTERRUPTS()
  • WaitLatch()

関連データ構造

  • bbsink_throttle: スロットリング状態（カウンタ、サンプル値、最終計測時刻など）を保持する構造体。

失敗モードと境界条件

  • 転送が設定レートより遅い場合、待機は発生しない。
  • WaitLatch のタイムアウトにより、待機ループから正常に抜ける。
  • プロセス停止シグナル受信時は CHECK_FOR_INTERRUPTS() により処理が中断される。

保守担当者向けメモ

  • 本関数は「デコレータ」として機能し、データ転送のパイプラインに挿入される。
  • 整数演算による端数処理を行っているため、高頻度な呼び出しでも累積誤差を最小限に抑える設計となっている。
  • WaitLatch の引数 sleep / 1000 はミリ秒単位への変換を意図している。
