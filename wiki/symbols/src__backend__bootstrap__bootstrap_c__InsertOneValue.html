InsertOneValue

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/bootstrap/bootstrap.c:656

目的

ブートストラップモードにおいて、現在開いているリレーション（boot_reldesc）の指定された属性位置に、文字列形式の値を変換して挿入するための準備を行う。

シグネチャまたは形状

void InsertOneValue(char *value, int i)

入力と出力

  • **入力**:
  • value (char *): 挿入する値の文字列表現。
  • i (int): 挿入対象の属性インデックス（0始まり）。
  • **出力**: なし（副作用としてグローバルな values 配列を更新）。

前提条件と不変条件

  • i は 0 以上 MAXATTR 未満であること（Assert により検証）。
  • boot_reldesc が有効なリレーション記述子として初期化されていること。

アルゴリズムまたは振る舞い

1. 指定された属性インデックス i に対応する型情報を boot_reldesc から取得する。 2. boot_get_type_io_data を呼び出し、当該型の入出力関数（typinput, typoutput）および属性情報を取得する。 3. OidInputFunctionCall を使用して、文字列 value を該当型の内部表現に変換し、グローバルな values 配列の i 番目に格納する。 4. デバッグ用に、変換後の値を OidOutputFunctionCall で文字列に戻し、DEBUG4 レベルでログ出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、BKIコマンド解析処理の一部として呼び出されることが示唆されている。
  • **呼び出し先**:
  • TupleDescAttr
  • boot_get_type_io_data
  • OidInputFunctionCall
  • OidOutputFunctionCall
  • elog / ereport

関連データ構造

  • boot_reldesc: 現在操作中のリレーション記述子。
  • values: 挿入対象のタプルを構築するために使用されるグローバルな値配列。

失敗モードと境界条件

  • i が範囲外の場合、Assert によりプロセスが停止する可能性がある。
  • 型の入力関数が不正な文字列形式を処理できない場合、例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数はタプルの各属性を個別に設定する役割を持つ。タプル全体の書き込みについては InsertOneTuple が担当する。
  • values 配列はグローバル変数であるため、本関数の呼び出し順序はリレーションのスキーマ定義と一致している必要がある。
