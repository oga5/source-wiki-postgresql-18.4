EvalOrderByExpressions

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:361

目的

インデックススキャン実行時に、ORDER BY 句に含まれる各式の値を評価し、その結果を IndexScanState ノード内に格納する。

シグネチャまたは形状

static void EvalOrderByExpressions(IndexScanState *node, ExprContext *econtext)

入力と出力

  • **入力**:
  • node: IndexScanState 構造体へのポインタ。評価結果を保持する iss_OrderByValues および iss_OrderByNulls 配列を持つ。
  • econtext: 式評価に使用する ExprContext。
  • **出力**: なし（副作用として node 内の配列を更新する）。

前提条件と不変条件

  • node->indexorderbyorig に評価対象の ExprState リストが格納されていること。
  • econtext が現在のタプルコンテキストに対して適切に初期化されていること。
  • node->iss_OrderByValues および node->iss_OrderByNulls が、indexorderbyorig の要素数分確保されていること。

アルゴリズムまたは振る舞い

1. econtext->ecxt_per_tuple_memory にメモリコンテキストを切り替え、評価結果のメモリ管理を行う。 2. node->indexorderbyorig リストを反復処理する。 3. 各 ExprState に対して ExecEvalExpr を呼び出し、式を評価する。 4. 評価結果（値と NULL フラグ）を node->iss_OrderByValues[i] および node->iss_OrderByNulls[i] に格納する。 5. 処理終了後、元のメモリコンテキストに復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeIndexscan.c 内のインデックススキャン実行ロジックから呼び出される（例: IndexNextWithReorder 等での再評価時）。
  • **呼び出し先**: ExecEvalExpr（式評価用）。

関連データ構造

  • IndexScanState: インデックススキャンの実行状態を保持する構造体。
  • ExprContext: 式評価のための実行コンテキスト。
  • ExprState: 評価対象の式。

失敗モードと境界条件

  • indexorderbyorig が空の場合、ループは実行されず何も行われない。
  • 式評価中にエラーが発生した場合、ExecEvalExpr の挙動に従い例外がスローされる。

保守担当者向けメモ

  • 本関数はタプルごとの評価を行うため、メモリリークを防ぐために econtext->ecxt_per_tuple_memory を使用している。このコンテキストの管理方針を変更する場合は注意が必要。
