SyncRepGetSyncRecPtr

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:585

目的

現在の同期レプリケーション設定に基づき、同期スタンバイ群のWAL到達位置（Write/Flush/Apply）を計算し、呼び出し元が同期スタンバイとして機能しているかを判定する。

シグネチャまたは形状

static bool SyncRepGetSyncRecPtr(XLogRecPtr *writePtr, XLogRecPtr *flushPtr, XLogRecPtr *applyPtr, bool *am_sync)

入力と出力

  • **入力**:
  • writePtr, flushPtr, applyPtr: 各種WAL位置を格納するためのポインタ（出力用）。
  • am_sync: 呼び出し元が同期スタンバイとして認識されているかを格納するためのポインタ（出力用）。
  • **出力**:
  • 戻り値: 同期スタンバイとして有効な位置が計算できた場合は true、それ以外（未設定、または同期スタンバイ数が不足している場合）は false。

前提条件と不変条件

  • SyncRepConfig が初期化されていること。
  • 呼び出し時に SyncRepLock 等の適切な排他制御が外部で行われている必要がある（関数内ではロック取得を行わない）。
  • 計算に使用する sync_standbys 配列は、関数終了時に pfree で解放される。

アルゴリズムまたは振る舞い

1. 出力引数を InvalidXLogRecPtr および false で初期化する。 2. SyncRepConfig が未設定であれば false を返す。 3. SyncRepGetCandidateStandbys を呼び出し、現在の同期候補スタンバイを取得する。 4. 候補リスト内に自身のプロセスが含まれるかを確認し、am_sync を更新する。 5. 自身のプロセスが同期対象外であるか、または同期スタンバイ数が SyncRepConfig->num_sync に満たない場合は false を返す。 6. 同期方式（SYNC_REP_PRIORITY または SYNC_REP_QUORUM）に応じて、以下のいずれかを呼び出し位置を計算する。

  • SyncRepGetOldestSyncRecPtr（Priority方式）
  • SyncRepGetNthLatestSyncRecPtr（Quorum方式）

    • 7. 成功時に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/syncrep.c:516（SyncRepReleaseWaiters 内）
  • **呼び出し先**:
  • SyncRepGetCandidateStandbys
  • SyncRepGetOldestSyncRecPtr
  • SyncRepGetNthLatestSyncRecPtr
  • pfree

関連データ構造

  • SyncRepConfig: 同期レプリケーションの構成設定。
  • SyncRepStandbyData: 同期スタンバイの各状態を保持する構造体。
  • XLogRecPtr: WALのログ位置を示すデータ型。

失敗モードと境界条件

  • SyncRepConfig が NULL の場合、計算を行わず false を返す。
  • 同期スタンバイ数が SyncRepConfig->num_sync 未満の場合、同期位置の計算を中断し false を返す。

保守担当者向けメモ

  • 本関数は SyncRepReleaseWaiters から呼び出され、同期コミットの待機解除条件を判定する重要な役割を担う。
  • Quorum方式において、現在の同期スタンバイ数と要求数が一致する場合、SyncRepGetOldestSyncRecPtr を使用して効率化できる可能性がある（ソース内の XXX コメント参照）。
