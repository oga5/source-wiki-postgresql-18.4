Instrumentation

定義

  • 種別: struct
  • 定義箇所: src/include/executor/instrument.h:69

目的

クエリ実行計画の各ノードにおける実行時間、バッファアクセス、WAL生成量、タプル処理数などの統計情報を収集・管理する。

シグネチャまたは形状

typedef struct Instrumentation {
    /* 設定パラメータ */
    bool need_timer;
    bool need_bufusage;
    bool need_walusage;
    bool async_mode;
    /* 現在のサイクル情報 */
    bool running;
    instr_time starttime;
    instr_time counter;
    double firsttuple;
    double tuplecount;
    BufferUsage bufusage_start;
    WalUsage walusage_start;
    /* 累積統計 */
    double startup;
    double total;
    double ntuples;
    double ntuples2;
    double nloops;
    double nfiltered1;
    double nfiltered2;
    BufferUsage bufusage;
    WalUsage walusage;
} Instrumentation;

入力と出力

  • **入力**: ノード実行時の開始時刻、バッファ/WAL使用量のスナップショット、および各ループ終了時の累積加算値。
  • **出力**: EXPLAIN ANALYZE 等で利用される、ノードごとの実行コストおよびリソース消費の集計結果。

前提条件と不変条件

  • BufferUsage および WalUsage は累積カウンタであり、リセットされない。
  • 特定の処理コストを算出するには、開始時と終了時の差分（current - start）を計算する必要がある。
  • InstrumentOption に基づき、必要な計測項目のみが有効化される。

アルゴリズムまたは振る舞い

1. **初期化**: InstrAlloc により構造体が確保され、オプションが設定される。 2. **ノード開始**: InstrStartNode が呼び出され、開始時刻と現在のリソース使用量が記録される。 3. **ノード終了**: InstrStopNode が経過時間を計算し、累積値に加算する。 4. **サイクル集計**: InstrEndLoop により、ノードの実行サイクルごとの統計が合計値に加算される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/explain.c、src/backend/commands/trigger.c、contrib/postgres_fdw/postgres_fdw.c 等の実行エンジンおよびコマンド処理系。
  • **呼び出し先**: portability/instr_time.h（時間計測用マクロ）。
  • **補足**: 特定の関数呼び出し関係の詳細は、提供されたソース範囲内では網羅されていません。

関連データ構造

  • BufferUsage: バッファI/O統計を保持。
  • WalUsage: WAL生成統計を保持。
  • instr_time: 高精度な時間計測用型。
  • InstrumentOption: 計測対象を指定するビットマスク。

失敗モードと境界条件

  • 計測はクエリ実行のオーバーヘッドとなるため、不要な項目を有効にするとパフォーマンスが低下する。
  • パラレルクエリや非同期スキャンノードでは、async_mode フラグによる特殊な計測ロジックが必要となる。

保守担当者向けメモ

  • 本構造体は src/backend/executor/instrument.c で実装される関数群によって操作される。
  • 統計の累積値は常に増加するため、差分計算のロジックを誤らないよう注意すること。
