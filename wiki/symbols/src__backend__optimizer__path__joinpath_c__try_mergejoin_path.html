try_mergejoin_path

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinpath.c:1028

目的

マージ結合（Merge Join）を実行するためのパスを生成し、コスト評価を経て有効なパスであれば結合リレーション（joinrel）に追加する。

シグネチャまたは形状

static void try_mergejoin_path(
    PlannerInfo *root,
    RelOptInfo *joinrel,
    Path *outer_path,
    Path *inner_path,
    List *pathkeys,
    List *mergeclauses,
    List *outersortkeys,
    List *innersortkeys,
    JoinType jointype,
    JoinPathExtraData *extra,
    bool is_partial
)

入力と出力

  • **入力**: 結合対象のパス（outer_path, inner_path）、結合条件（mergeclauses）、ソートキー、結合タイプ、および追加メタデータ（extra）。
  • **出力**: なし（成功時は joinrel->pathlist にパスが追加される）。

前提条件と不変条件

  • is_partial が true の場合、try_partial_mergejoin_path に処理を委譲し即時終了する。
  • 外部結合（Outer Join）において、入力パスがその結合自体に依存するパラメータ化を行っている場合はパス生成を拒否する。
  • パラメータ化されたパスの場合、そのパラメータが extra->param_source_rels に含まれている必要がある。

アルゴリズムまたは振る舞い

1. **ソートの最適化**: 入力パスが既に必要な順序を満たしている場合、outersortkeys または innersortkeys を NIL に設定し、不要なソート処理を回避する。 2. **コスト計算**: initial_cost_mergejoin を呼び出し、マージ結合の起動コストと総コストを算出する。 3. **パスの検証と追加**: add_path_precheck でコストやパラメータ化の妥当性を確認し、合格した場合は create_mergejoin_path でパスノードを作成し、add_path で joinrel に登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinpath.c 内の各所（add_paths_to_joinrel 等から呼び出される）。
  • **呼び出し先**: try_partial_mergejoin_path, initial_cost_mergejoin, add_path_precheck, create_mergejoin_path, add_path。

関連データ構造

  • RelOptInfo: 結合対象のリレーション情報。
  • Path: 結合パスの基本構造。
  • JoinPathExtraData: 結合処理に必要なメタデータ（SpecialJoinInfo や restrictlist 等）。
  • JoinCostWorkspace: コスト計算結果を保持する作業領域。

失敗モードと境界条件

  • パラメータ化が不適切な場合、またはコスト評価の結果として既存のパスより劣ると判断された場合は、パスを追加せずに終了する。
  • メモリリークを防ぐため、パスを拒否する際には required_outer のメモリを解放する。

保守担当者向けメモ

  • 本関数は try_nestloop_path と同様のコスト評価およびパス追加のパターンに従う。
  • インクリメンタルソートの適用可否判定が含まれており、outer_presorted_keys の計算が重要となる。
