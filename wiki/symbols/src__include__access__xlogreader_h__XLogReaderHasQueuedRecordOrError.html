XLogReaderHasQueuedRecordOrError

定義

  • 種別: function
  • 定義箇所: src/include/access/xlogreader.h:324

目的

XLogReaderState が保持するデコード済みレコードのキューに未処理のレコードが存在するか、あるいは処理中に発生したエラーメッセージが保留されているかを判定します。

シグネチャまたは形状

static inline bool XLogReaderHasQueuedRecordOrError(XLogReaderState *state)

入力と出力

  • **入力**: XLogReaderState *state（WAL 読み込みセッションの状態を保持する構造体へのポインタ）
  • **出力**: bool 型。以下のいずれかが真であれば true を返し、それ以外は false を返します。

前提条件と不変条件

  • state は有効な XLogReaderState 構造体を指している必要があります。
  • 本関数は static inline で定義されており、ヘッダーファイル経由で直接呼び出されます。

アルゴリズムまたは振る舞い

以下の論理和（OR）を評価します。 1. state->decode_queue_head != NULL: デコード済みレコードのキューに要素が存在するか。 2. state->errormsg_deferred: 読み込み処理中に発生したエラーメッセージが保留されているか。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogprefetcher.c: 先読み処理の非ブロッキング判定に使用。
  • src/backend/access/transam/xlogreader.c: 内部的な状態チェックに使用。
  • **呼び出し先**:
  • 本関数は state 構造体のメンバへ直接アクセスするのみであり、他の関数を呼び出しません。

関連データ構造

  • XLogReaderState: WAL 読み込みのセッション状態を管理する構造体。
  • DecodedXLogRecord: キューイングされるレコードの型。

失敗モードと境界条件

  • state が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性があります。
  • キューが空であっても、errormsg_deferred が設定されていれば true を返すため、エラー状態の検知にも利用されます。

保守担当者向けメモ

  • 本関数はフロントエンド（pg_waldump 等）でも利用されるため、バックエンド固有の依存関係を持ち込まないように注意してください。
  • 状態の判定ロジックに変更を加える場合は、XLogReaderState の構造体定義と整合性を保つ必要があります。
