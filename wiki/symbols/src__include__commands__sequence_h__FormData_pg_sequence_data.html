FormData_pg_sequence_data

定義

  • **種別**: struct
  • **定義箇所**: src/include/commands/sequence.h:25

目的

PostgreSQLのシーケンスオブジェクトにおける現在の状態（最終値、ログ記録用カウンタ、呼び出しフラグ）を保持し、ディスク上のシーケンスリレーションのデータ構造として機能します。

シグネチャまたは形状

typedef struct FormData_pg_sequence_data
{
    int64       last_value;
    int64       log_cnt;
    bool        is_called;
} FormData_pg_sequence_data;

入力と出力

本構造体は関数ではなくデータ型です。シーケンス操作の過程で、システムカタログやディスク上のシーケンスリレーションから読み出され、あるいは書き込まれるデータセットとして扱われます。

前提条件と不変条件

  • last_value: シーケンスの現在の値。
  • log_cnt: 次のWALログ出力が必要になるまでの残りの呼び出し回数。
  • is_called: シーケンスが一度でも呼び出されたかどうかを示すフラグ。

アルゴリズムまたは振る舞い

シーケンスのキャッシュ機構において、nextval 呼び出し時に last_value が更新されます。パフォーマンス向上のため、一定回数（log_cnt に基づく）の呼び出しごとにディスクへの書き込みが行われる仕組みです。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/sequence.c 内のシーケンス管理ロジック（nextval_internal 等）で、シーケンスデータの読み書きに使用されます。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • Form_pg_sequence_data: FormData_pg_sequence_data へのポインタ型。
  • xl_seq_rec: シーケンスの変更をWALに記録するための構造体。

失敗モードと境界条件

  • システムクラッシュ時、キャッシュされた値がディスクに書き込まれる前に失われる可能性があるため、シーケンス値に「飛び」が生じる仕様です。

保守担当者向けメモ

  • 本構造体はシーケンスの物理的なデータ形式を定義しています。変更を加える際は、src/backend/commands/sequence.c での読み書き処理およびWALリプレイ関数（seq_redo）への影響を考慮してください。
