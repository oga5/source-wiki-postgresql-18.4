findObjectByDumpId

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/common.c:764

目的

DumpId をキーとして、管理されている DumpableObject を dumpIdMap 配列から高速に検索・取得する。

シグネチャまたは形状

DumpableObject *findObjectByDumpId(DumpId dumpId)

入力と出力

  • **入力**: dumpId (検索対象の DumpId)
  • **出力**: 指定されたIDに対応する DumpableObject 構造体へのポインタ。該当しない場合は NULL を返す。

前提条件と不変条件

  • dumpIdMap 配列が初期化され、有効なオブジェクトが格納されていること。
  • allocedDumpIds 変数が現在の dumpIdMap の有効範囲（サイズ）を正しく保持していること。

アルゴリズムまたは振る舞い

1. 引数 dumpId が有効範囲内（0 < dumpId < allocedDumpIds）にあるかを確認する。 2. 範囲外であれば NULL を返す。 3. 範囲内であれば、dumpIdMap 配列の該当インデックスに格納されている DumpableObject ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c (依存関係解決やオブジェクト参照時に複数箇所で使用)
  • src/bin/pg_dump/pg_dump_sort.c (ダンプ順序決定時の依存関係走査)
  • **呼び出し先**:
  • 本関数は直接的な関数呼び出しを行わず、配列アクセスのみを行う。

関連データ構造

  • DumpableObject: データベースオブジェクトを抽象化した構造体。
  • dumpIdMap: DumpId をインデックスとして DumpableObject を保持するグローバルな配列。

失敗モードと境界条件

  • dumpId <= 0: 無効なIDとして NULL を返す。
  • dumpId >= allocedDumpIds: 確保済みの配列範囲外として NULL を返す。

保守担当者向けメモ

  • 本関数は pg_dump におけるオブジェクト参照の頻出パスであるため、インライン化や最適化が考慮されている。
  • dumpIdMap のサイズ変更（AssignDumpId 等による拡張）と本関数の参照タイミングに競合がないことを確認すること。
