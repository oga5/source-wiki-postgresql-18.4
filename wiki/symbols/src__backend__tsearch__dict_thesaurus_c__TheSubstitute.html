TheSubstitute

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/dict_thesaurus.c:45

目的

シソーラス辞書において、マッチしたフレーズを置換するための結果データを保持する構造体です。

シグネチャまたは形状

typedef struct {
    uint16      lastlexeme;     /* 置換対象の語彙数 */
    uint16      reslen;         /* 結果の長さ */
    TSLexeme   *res;            /* 準備された置換後の結果 */
} TheSubstitute;

入力と出力

  • 入力: シソーラス定義ファイルから読み込まれた置換ルール。
  • 出力: thesaurus_lexize 実行時に、マッチしたフレーズに対する置換後の TSLexeme 配列を提供します。

前提条件と不変条件

  • res フィールドは、メモリ割り当て後に有効な TSLexeme 配列を指している必要があります。
  • lastlexeme は置換されるフレーズのトークン数を表します。

アルゴリズムまたは振る舞い

シソーラス辞書が入力トークン列とマッチした際、この構造体を通じて置換後のトークン列（res）が参照されます。辞書のコンパイル時に compileTheSubstitute 等によって構築され、検索時に利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: DictThesaurus 構造体内の配列として保持され、thesaurus_lexize 等から参照されます。
  • 呼び出し先: copyTSLexeme 関数がこの構造体を引数として受け取ります。
  • 補足: 呼び出し元・呼び出し先の詳細なコールグラフは、提供されたソース抜粋からは網羅的に特定できません。

関連データ構造

  • DictThesaurus: 本構造体を保持するメインの辞書構造体。
  • TSLexeme: 置換後のトークン情報を格納する型。

失敗モードと境界条件

  • メモリ不足時: palloc や repalloc によるメモリ確保失敗の可能性があります。
  • 辞書定義の不整合: reslen と実際の res 配列のサイズが一致しない場合、不正なメモリ参照が発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は DictThesaurus の一部として動的に確保・再確保（repalloc）されるため、ライフサイクル管理に注意してください。
  • res フィールドが指すメモリ領域の所有権と解放タイミングについては、dict_thesaurus.c 内のメモリ管理ロジックを確認してください。
