ReleaseSemaphores

定義

  • 種別: function
  • 定義箇所: src/backend/port/posix_sema.c:238

目的

プロセス終了時に、確保済みのすべての POSIX セマフォリソースを解放し、クリーンアップを行う。

シグネチャまたは形状

static void ReleaseSemaphores(int status, Datum arg)

入力と出力

  • **入力**:
  • status: on_shmem_exit から渡される終了ステータス。
  • arg: on_shmem_exit から渡される補助引数（未使用）。
  • **出力**: なし（副作用としてセマフォリソースを解放）。

前提条件と不変条件

  • numSems が現在確保されているセマフォの総数として正しく保持されていること。
  • USE_NAMED_POSIX_SEMAPHORES または USE_UNNAMED_POSIX_SEMAPHORES のいずれかが定義されている環境で実行されること。

アルゴリズムまたは振る舞い

1. コンパイル時フラグに応じて以下のいずれかの処理を実行する:

  • USE_NAMED_POSIX_SEMAPHORES が有効な場合: mySemPointers 配列内の各セマフォに対して PosixSemaphoreKill を呼び出し、最後に mySemPointers 配列自体を free する。
  • USE_UNNAMED_POSIX_SEMAPHORES が有効な場合: sharedSemas 配列内の各セマフォに対して PosixSemaphoreKill を呼び出す。

    • 2. PosixSemaphoreKill は内部で sem_close または sem_destroy を実行し、OS レベルのリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: on_shmem_exit を介してプロセス終了時に登録・実行される。
  • **呼び出し先**: PosixSemaphoreKill。

関連データ構造

  • numSems: 確保されたセマフォの数。
  • mySemPointers: 名前付きセマフォのポインタ配列。
  • sharedSemas: 名前なしセマフォの共有メモリ上の配列。

失敗モードと境界条件

  • numSems が 0 の場合、ループは実行されず何も行われない。
  • PosixSemaphoreKill 内部でエラーが発生した場合の挙動は、当該関数の実装に依存する。

保守担当者向けメモ

  • 本関数は on_shmem_exit のコールバックとして登録されるため、直接呼び出すことは想定されていない。
  • 名前付きセマフォと名前なしセマフォで解放対象のデータ構造が異なるため、修正時は両方の #ifdef ブロックを確認すること。
