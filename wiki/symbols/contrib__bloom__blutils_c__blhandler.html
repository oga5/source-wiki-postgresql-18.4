blhandler

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blutils.c:102

目的

PostgreSQL のインデックスアクセスメソッド（AM）として、Bloom フィルタインデックスの機能構造（IndexAmRoutine）を初期化し、システムへ登録するためのエントリポイントを提供します。

シグネチャまたは形状

Datum blhandler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQL の関数呼び出し規約に基づく引数）
  • **出力**: Datum 型として IndexAmRoutine 構造体へのポインタを返します。

前提条件と不変条件

  • 本関数は PostgreSQL のインデックスハンドラとして呼び出されることを前提としています。
  • IndexAmRoutine 構造体の各フィールドには、Bloom インデックスの特性（マルチカラム対応、順序付け不可など）が静的に設定されます。

アルゴリズムまたは振る舞い

1. makeNode(IndexAmRoutine) を使用してインデックスルーチン構造体をメモリ上に確保します。 2. インデックスの特性（amcanmulticol = true 等）を定義します。 3. Bloom インデックスの各操作に対応する関数ポインタ（ambuild, aminsert, ambulkdelete, amgetbitmap 等）を登録します。 4. 構築された構造体を PG_RETURN_POINTER マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のインデックス登録メカニズム（PG_FUNCTION_INFO_V1(blhandler) により登録）。
  • **呼び出し先**: blbuild, blbuildempty, blinsert, blbulkdelete, blvacuumcleanup, blcostestimate, bloptions, blvalidate, blbeginscan, blrescan, blgetbitmap, blendscan（各インデックス操作の実装関数）。

関連データ構造

  • IndexAmRoutine: インデックスアクセスメソッドのインターフェース定義構造体。

失敗モードと境界条件

  • メモリ不足により makeNode が失敗した場合、PostgreSQL の標準的なエラーハンドリングによりアボートされます。

保守担当者向けメモ

  • 本関数で設定される IndexAmRoutine のフラグや関数ポインタを変更する場合、contrib/bloom 内の各実装ファイル（blinsert.c, blscan.c 等）との整合性に注意してください。
  • amparallelvacuumoptions には VACUUM_OPTION_PARALLEL_BULKDEL および VACUUM_OPTION_PARALLEL_CLEANUP が設定されており、並列バキュームをサポートしています。
