PGTYPESnumeric_new

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/numeric.c:41

目的

ECPG（PostgreSQL クライアントサイド）において、新しい numeric 構造体を動的に割り当て、初期化するために使用されます。

シグネチャまたは形状

numeric *PGTYPESnumeric_new(void)

入力と出力

  • **入力**: なし。
  • **出力**: 成功した場合は初期化された numeric 構造体へのポインタを返し、メモリ割り当てに失敗した場合は NULL を返します。

前提条件と不変条件

  • 戻り値が NULL でない場合、呼び出し元は最終的に PGTYPESnumeric_free を使用してメモリを解放する責任を負います。
  • 内部で alloc_var(var, 0) を呼び出し、数値バッファの初期化を試みます。

アルゴリズムまたは振る舞い

1. pgtypes_alloc を使用して numeric 構造体分のメモリを確保します。 2. 確保に失敗した場合は NULL を返します。 3. alloc_var を呼び出し、数値保持用の内部バッファを初期化します。 4. alloc_var が失敗した場合は、確保した numeric 構造体のメモリを解放し、NULL を返します。 5. 成功した場合は、初期化済みの numeric 構造体ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/compatlib/informix.c 内の複数の箇所（例: 54, 57, 100, 103, 109, 255, 277, 299, 390, 435, 456, 483 行目）から呼び出されています。
  • **呼び出し先**: pgtypes_alloc（メモリ確保）、alloc_var（内部バッファ初期化）、free（エラー時のメモリ解放）。

関連データ構造

  • numeric: 数値の桁、重み、符号を保持する主要構造体。

失敗モードと境界条件

  • システムメモリが不足している場合、pgtypes_alloc または alloc_var が失敗し、関数は NULL を返します。

保守担当者向けメモ

  • 本関数は numeric 型のライフサイクル管理の起点です。メモリリークを防ぐため、本関数で生成されたインスタンスは必ず対応する解放関数で処理されているか確認してください。
