SH_UPDATE_PARAMETERS

定義

  • 種別: static inline function
  • 定義箇所: src/include/lib/simplehash.h:336

目的

ハッシュテーブルのサイズ変更に伴い、テーブルの容量パラメータ（size、sizemask）および拡張閾値（grow_threshold）を更新する。

シグネチャまたは形状

static inline void SH_UPDATE_PARAMETERS(SH_TYPE * tb, uint64 newsize)

入力と出力

  • **入力**:
  • tb: 更新対象のハッシュテーブル構造体へのポインタ。
  • newsize: 新しいテーブルサイズとして要求される値。
  • **出力**: なし（tb が指す構造体のメンバを直接更新する）。

前提条件と不変条件

  • newsize は SH_COMPUTE_SIZE マクロによって有効なハッシュテーブルサイズ（2のべき乗）に変換される。
  • tb->size は常に SH_MAX_SIZE 以下である必要がある。

アルゴリズムまたは振る舞い

1. SH_COMPUTE_SIZE(newsize) を呼び出し、実際のメモリ割り当てサイズを計算する。 2. tb->size に計算結果を代入し、tb->sizemask に size - 1 を設定する（高速な剰余演算のため）。 3. tb->size が SH_MAX_SIZE に達している場合は SH_MAX_FILLFACTOR を、それ以外の場合は SH_FILLFACTOR を用いて、次の拡張タイミングである tb->grow_threshold を再計算する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/lib/simplehash.h:466
  • src/include/lib/simplehash.h:515
  • **呼び出し先**:
  • SH_COMPUTE_SIZE (マクロ)

関連データ構造

  • SH_TYPE: 生成されるハッシュテーブルの構造体。size、sizemask、grow_threshold メンバを持つ。

失敗モードと境界条件

  • newsize が極端に大きい場合、SH_COMPUTE_SIZE の挙動に依存する。
  • tb->size が SH_MAX_SIZE の場合、それ以上の拡張は行われないよう閾値計算が分岐する。

保守担当者向けメモ

  • 本関数は simplehash.h 内で生成されるテンプレート関数の一部であり、マクロ展開によって特定の型に対してインスタンス化される。
  • SH_FILLFACTOR および SH_MAX_FILLFACTOR は、ハッシュテーブルの性能（衝突率とメモリ効率のトレードオフ）を決定する定数である。
