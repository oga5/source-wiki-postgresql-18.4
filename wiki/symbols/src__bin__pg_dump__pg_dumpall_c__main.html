main

定義

  • **種別**: function
  • **定義箇所**: src/bin/pg_dump/pg_dumpall.c:127

目的

pg_dumpall ユーティリティのメインエントリーポイントであり、コマンドライン引数の解析、環境の初期化、およびクラスタ全体のダンプ処理（ロール、テーブルスペース、各データベースのダンプ）をオーケストレートします。

シグネチャまたは形状

int main(int argc, char *argv[])

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。ダンプ対象や接続先、各種オプション（--data-only, --clean, --file 等）を指定。
  • **出力**:
  • 正常終了時は 0 を返します。
  • 異常終了時やエラー発生時は pg_fatal 等を通じてエラーメッセージを出力し、終了します。

前提条件と不変条件

  • 実行環境において pg_dump バイナリが利用可能である必要があります。
  • find_other_exec により、pg_dumpall と同一ディレクトリ（またはパス内）にある pg_dump のバージョン整合性が検証されます。

アルゴリズムまたは振る舞い

1. **初期化**: ログ出力の初期化、ロケール設定、プログラム名の取得を行います。 2. **引数解析**: --help や --version の即時処理を行い、getopt_long を用いて詳細なオプションを解析します。 3. **バイナリ検証**: find_other_exec を呼び出し、サブプロセスとして使用する pg_dump の存在とバージョンを確認します。 4. **オプション蓄積**: pgdumpopts バッファを作成し、解析したオプションを pg_dump 実行時に渡すための文字列として保持します。 5. **実行**: 以降の処理（ソースコード抜粋外）で、接続の確立、共有オブジェクト（ロール等）のダンプ、各データベースに対する pg_dump のサブプロセス起動を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: OS（プログラムの起動時）。
  • **呼び出し先**:
  • help(): ヘルプ表示。
  • exit_nicely(): 終了処理。
  • find_other_exec(): pg_dump バイナリの検索。
  • pg_logging_init() / pg_logging_set_level(): ログ設定。
  • createPQExpBuffer(): オプション保持用バッファの生成。
  • getopt_long(): 引数解析。
  • appendPQExpBufferStr() / appendShellString(): オプション文字列の構築。

関連データ構造

  • struct option long_options[]: getopt_long 用のコマンドライン引数定義。
  • PGconn: データベース接続用構造体。
  • PQExpBuffer (pgdumpopts): pg_dump に渡す引数文字列を格納するバッファ。

失敗モードと境界条件

  • pg_dump バイナリが見つからない場合、またはバージョンが不一致の場合は pg_fatal によりエラー終了します。
  • 不正な引数が渡された場合、getopt_long の挙動に従いエラーとなります。

保守担当者向けメモ

  • pg_dumpall は pg_dump をサブプロセスとして呼び出すため、引数の追加や変更を行う際は pgdumpopts への追加ロジックと getopt_long の定義を同期させる必要があります。
  • 呼び出し元および呼び出し先の情報は、提供されたソースコードの範囲内で解析したものです。
