CollAliasData

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/collationcmds.c:41

目的

libcのロケール名とそれに対応する短縮エイリアス、およびエンコーディング情報を保持し、システム照合順序のインポート処理を管理するために使用されます。

シグネチャまたは形状

typedef struct
{
	char	   *localename;		/* name of locale, as per "locale -a" */
	char	   *alias;			/* shortened alias for same */
	int			enc;			/* encoding */
} CollAliasData;

入力と出力

本構造体はデータ保持用であり、関数としての入出力は持ちません。主に pg_import_system_collations 関数内で、システムから取得したロケール情報を格納する配列の要素として使用されます。

前提条件と不変条件

  • localename および alias は、メモリ確保された文字列へのポインタである必要があります。
  • enc はPostgreSQLのエンコーディングIDを保持します。

アルゴリズムまたは振る舞い

  • pg_import_system_collations 内で、システムから検出されたロケールリストを格納するために動的に配列が割り当てられます。
  • qsort を用いてソートされる際、cmpaliases 関数が比較ロジックとして利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/collationcmds.c 内の pg_import_system_collations 関数が、この構造体の配列を割り当て、値を設定します。
  • **呼び出し先**: qsort 関数が cmpaliases を介して本構造体のインスタンスを比較します。

関連データ構造

  • cmpaliases: 本構造体の配列をソートするための比較関数。

失敗モードと境界条件

  • メモリ不足時: palloc または repalloc が失敗した場合、PostgreSQLの標準的なエラー処理によりアボートします。
  • データの不整合: localename がシステム上の有効なロケール名と一致しない場合、後続の照合順序作成処理で失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/commands/collationcmds.c 内でのみ使用される内部的なデータ保持用構造体です。
  • プラットフォーム固有のロケール名形式に依存するため、OS間での移植性を考慮する際は pg_import_system_collations のロジックと併せて確認してください。
