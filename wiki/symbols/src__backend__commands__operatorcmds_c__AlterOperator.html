AlterOperator

定義

  • **種別**: function
  • **定義箇所**: src/backend/commands/operatorcmds.c:494

目的

SQLの ALTER OPERATOR 文を実行し、既存の演算子の属性（推定関数、commutator、negator、およびハッシュ/マージ結合の可否）を更新します。

シグネチャまたは形状

ObjectAddress AlterOperator(AlterOperatorStmt *stmt)

入力と出力

  • **入力**: AlterOperatorStmt *stmt（解析済みの ALTER OPERATOR 構文木）
  • **出力**: ObjectAddress（更新された演算子のオブジェクト識別情報）

前提条件と不変条件

  • 呼び出し元は、対象の演算子が既に存在することを前提としています。
  • 実行ユーザーは、対象演算子の所有者である必要があります。
  • 演算子の引数型、関数本体、および一度設定された commutator や negator などの属性は変更できません。

アルゴリズムまたは振る舞い

1. LookupOperWithArgs を使用して対象演算子の OID を特定し、pg_operator カタログからタプルをコピー取得します。 2. stmt->options を走査し、指定されたオプション（restrict, join, commutator, negator, merges, hashes）を解析します。 3. object_ownercheck により、実行ユーザーが演算子の所有者であることを検証します。 4. 指定された各オプションに対し、ValidateRestrictionEstimator、ValidateJoinEstimator、ValidateOperatorReference を呼び出し、妥当性を確認します。 5. 既存の属性と競合する変更（例: 既に設定済みの commutator の変更）を検知した場合はエラーを発生させます。 6. 最後に OperatorValidateParams を呼び出し、変更後のパラメータ整合性を確認します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（AlterOperatorStmt の実行時）
  • **呼び出し先**:
  • LookupOperWithArgs, table_open, SearchSysCacheCopy1
  • ValidateRestrictionEstimator, ValidateJoinEstimator, ValidateOperatorReference
  • object_ownercheck, aclcheck_error, OperatorValidateParams

関連データ構造

  • AlterOperatorStmt: ALTER OPERATOR 文の構文解析結果を保持する構造体。
  • Form_pg_operator: pg_operator システムカタログのタプル形式。

失敗モードと境界条件

  • **権限エラー**: 所有者でない場合に ACLCHECK_NOT_OWNER を発生させます。
  • **属性変更制限**: leftarg, rightarg, function, procedure の変更を試みた場合、または既に設定済みの commutator/negator を変更しようとした場合にエラーとなります。
  • **自己否定**: 演算子自身を negator に指定することはできません。
  • **キャッシュミス**: 演算子が見つからない場合、elog(ERROR) で停止します。

保守担当者向けメモ

  • 本関数は CREATE OPERATOR と異なり、プランナに影響を与える可能性のあるコア属性の変更を厳格に禁止しています。
  • 演算子の属性更新ロジックを追加する際は、OperatorValidateParams による検証が適切に行われるよう注意してください。
