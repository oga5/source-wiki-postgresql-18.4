JsonbContainerTypeName

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb.c:158

目的

JsonbContainer が保持するデータの型を文字列として特定する。

シグネチャまたは形状

const char *JsonbContainerTypeName(JsonbContainer *jbc)

入力と出力

  • **入力**: JsonbContainer *jbc（解析対象の JSONB コンテナ）
  • **出力**: "scalar"、"array"、"object" のいずれかの文字列リテラル。型が不明な場合はエラーを発生させる。

前提条件と不変条件

  • 引数 jbc は有効な JsonbContainer 構造体を指している必要がある。
  • 内部のヘッダー情報が破損している場合、elog(ERROR) により処理が中断される。

アルゴリズムまたは振る舞い

1. JsonbExtractScalar を呼び出し、コンテナがスカラー値であるかを確認する。スカラーであれば JsonbTypeName を呼び出し、その型名を返す。 2. スカラーでない場合、JsonContainerIsArray を確認し、真であれば "array" を返す。 3. JsonContainerIsObject を確認し、真であれば "object" を返す。 4. 上記のいずれにも該当しない場合、jbc->header の値と共にエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonb.c 内の複数箇所（JsonbTypeName の内部実装や、型情報を取得する他の関数から参照されている）。
  • **呼び出し先**:
  • JsonbExtractScalar
  • JsonbTypeName
  • JsonContainerIsArray
  • JsonContainerIsObject
  • elog

関連データ構造

  • JsonbContainer: JSONB のバイナリ表現を保持する構造体。
  • JsonbValue: JSONB のノード表現。

失敗モードと境界条件

  • jbc->header が定義されたいずれの型（スカラー、配列、オブジェクト）にも一致しない場合、elog(ERROR) が発生し、トランザクションがアボートされる可能性がある。

保守担当者向けメモ

  • 本関数は JsonbContainer の型を判別する主要なインターフェースである。新しい JSONB 型が追加された場合は、本関数の条件分岐を更新する必要がある。
