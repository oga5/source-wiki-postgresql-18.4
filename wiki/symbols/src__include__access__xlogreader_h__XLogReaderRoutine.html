XLogReaderRoutine

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlogreader.h:72

目的

WAL（Write-Ahead Log）読み込み処理を抽象化するためのコールバック関数群を保持する構造体です。ストレージやネットワークからの物理的なデータ取得と、WAL リーダーのロジックを分離するために使用されます。

シグネチャまたは形状

typedef struct XLogReaderRoutine
{
    XLogPageReadCB page_read;
    WALSegmentOpenCB segment_open;
    WALSegmentCloseCB segment_close;
} XLogReaderRoutine;

入力と出力

  • **page_read**: WAL ページを読み込むコールバック。
  • 入力: targetPagePtr（読み込み開始位置）、reqLen（要求バイト数）等。
  • 出力: 読み込んだバイト数（成功時）、または -1（失敗時）。
  • **segment_open**: WAL セグメントを開くコールバック。
  • 入力: nextSegNo（セグメント番号）、tli_p（タイムライン ID）。
  • 出力: 成功時はファイル記述子を ->seg.ws_file に設定。失敗時はエラーを発生させ、戻らない。
  • **segment_close**: WAL セグメントを閉じるコールバック。
  • 出力: ->seg.ws_file を負の値に設定。

前提条件と不変条件

  • page_read は、要求されたバイト数が利用可能になるまで待機（スリープ）する必要があります。
  • page_read は、一度の呼び出しで最大 XLOG_BLCKSZ バイトまで読み込み可能です。
  • segment_open は、成功時に必ず有効なファイル記述子を保持する必要があります。

アルゴリズムまたは振る舞い

1. **ページ読み込み**: page_read は指定された位置からデータを取得し、readBuf に格納します。必要に応じてタイムライン（seg.ws_tli）を更新します。 2. **セグメント管理**: segment_open および segment_close を通じて、WAL セグメントのファイルハンドルを適切に管理します。 3. **抽象化**: リーダー本体は、これらのコールバックを呼び出すことで、物理的なファイルシステムやネットワークの差異を意識せずに WAL を順次読み取ります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogreader.c（リーダーの初期化時）、src/backend/access/transam/xlogrecovery.c、src/backend/replication/logical/logical.c 等の各コンテキストで本構造体がインスタンス化され、コールバックが登録されます。
  • **呼び出し先**: 各コールバックの実装（例: xlogutils.c 内のローカルファイル読み込み関数など）が呼び出されます。

関連データ構造

  • XLogReaderState: 本構造体を保持し、読み込みセッションの状態を管理するメイン構造体。
  • XLogPageReadCB, WALSegmentOpenCB, WALSegmentCloseCB: 各コールバックの関数型定義。

失敗モードと境界条件

  • page_read が失敗した場合、-1 を返します。
  • segment_open が失敗した場合、エラーを発生させ、呼び出し元へは戻りません。
  • page_read は、要求されたバイト数よりも少ないデータしか得られない場合、再度呼び出される可能性があります。

保守担当者向けメモ

  • 本構造体はフロントエンド（pg_waldump 等）でも使用されるため、実装するコールバック関数内でバックエンド専用の ereport 等を使用しないよう注意してください。
  • XLogReaderState の各フィールドを直接操作せず、提供されている関数やマクロ経由でアクセスしてください。
