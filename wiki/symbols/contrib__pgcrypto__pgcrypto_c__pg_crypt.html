pg_crypt

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:212

目的

SQL インターフェースから提供されたパスワードとソルトを受け取り、crypt(3) 互換のハッシュ化処理を実行して結果を返す。

シグネチャまたは形状

Datum pg_crypt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの text 型引数（パスワードおよびソルト）を受け取る。
  • **出力**: text 型のハッシュ化された文字列を返す。

前提条件と不変条件

  • 入力引数は text 型である必要がある。
  • px_crypt が内部で PX_MAX_CRYPT バイトのバッファを必要とする。
  • PostgreSQL のメモリ管理ルールに従い、palloc0 で確保されたメモリは処理終了前に pfree される必要がある。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して入力引数を取得し、text_to_cstring で C 文字列に変換する。 2. palloc0 で PX_MAX_CRYPT サイズの作業用バッファを確保する。 3. px_crypt を呼び出し、パスワードとソルトからハッシュ値を生成する。 4. px_crypt が NULL を返した場合は ERRCODE_EXTERNAL_ROUTINE_INVOCATION_EXCEPTION でエラーを報告する。 5. 生成された C 文字列を cstring_to_text で text 型に変換する。 6. 作業用バッファを解放し、PG_FREE_IF_COPY で入力引数のメモリを適切に解放した後、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(pg_crypt) により登録）。
  • **呼び出し先**: px_crypt（内部暗号化ライブラリ）。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • PX_MAX_CRYPT: px-crypt.h で定義される、crypt 処理に必要な最大バッファサイズ。

失敗モードと境界条件

  • px_crypt が NULL を返した場合、外部ルーチン呼び出し例外としてエラー終了する。
  • メモリ確保失敗時は PostgreSQL の標準的なメモリ管理機構によりエラーが発生する。

保守担当者向けメモ

  • 本関数は crypt(3) 系のアルゴリズムをラップするブリッジである。アルゴリズム自体の実装については px-crypt.c を参照すること。
  • PG_FREE_IF_COPY の使用により、TOAST されたデータやコピーされた引数のメモリリークを防いでいる。変更時はメモリ管理の整合性に注意が必要。
