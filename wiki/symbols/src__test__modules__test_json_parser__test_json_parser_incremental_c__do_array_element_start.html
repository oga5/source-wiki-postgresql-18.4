do_array_element_start

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_json_parser/test_json_parser_incremental.c:317

目的

JSON 配列の要素開始イベントを処理し、配列構造を再構築（エコーバック）する際の区切り文字（カンマと改行）を出力する。

シグネチャまたは形状

static JsonParseErrorType do_array_element_start(void *state, bool isnull)

入力と出力

  • **入力**:
  • state: DoState 構造体へのポインタ（void * として渡される）。
  • isnull: 要素が NULL かどうかを示すフラグ（本関数内では未使用）。
  • **出力**:
  • 戻り値: JSON_SUCCESS を返す。

前提条件と不変条件

  • state は DoState 構造体としてキャスト可能であること。
  • DoState 内の elem_is_first フラグが、配列の最初の要素かどうかを正しく追跡していること。

アルゴリズムまたは振る舞い

1. 引数の state を DoState 型にキャストする。 2. _state->elem_is_first を確認し、最初の要素でない場合は、前の要素との区切りとして ",\n" を標準出力に書き出す。 3. _state->elem_is_first を false に更新する。 4. JSON_SUCCESS を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: JsonSemAction 構造体のコールバックテーブルを通じて、パーサーから間接的に呼び出される。
  • **呼び出し先**: 標準出力への printf を実行する。

関連データ構造

  • DoState: パース中の状態（要素の順序フラグなど）を保持する構造体。
  • JsonSemAction: 本関数をメンバ .array_element_start として保持するコールバックテーブル。

失敗モードと境界条件

  • 常に JSON_SUCCESS を返すため、関数内部での失敗モードはない。
  • 配列の最初の要素については区切り文字を出力しないという境界条件を elem_is_first フラグで制御している。

保守担当者向けメモ

  • 本関数はテストプログラムの一部であり、パース結果の可視化を目的としている。
  • isnull 引数は現在の実装では使用されていないが、将来的な拡張や API 互換性のためにシグネチャに含まれている。
