print_pathkeys

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/print.c:429

目的

クエリプランの最適化過程で使用される PathKey リストを、人間が読みやすい形式で標準出力にダンプするデバッグ用ユーティリティです。

シグネチャまたは形状

void print_pathkeys(const List *pathkeys, const List *rtable)

入力と出力

  • **入力**:
  • pathkeys: 表示対象の PathKey ノードのリスト。
  • rtable: 式の表示（print_expr）に使用されるレンジテーブル（RangeTblEntry のリスト）。
  • **出力**: なし（標準出力へ直接書き出されます）。

前提条件と不変条件

  • pathkeys は PathKey ノードのリストである必要があります。
  • rtable は、式に含まれる Var ノードの参照解決のために提供される必要があります。
  • PathKey が非正規（non-canonical）な場合、ec_merged を辿って正規の EquivalenceClass を参照します。

アルゴリズムまたは振る舞い

1. 全体として (...) で囲まれた形式で出力します。 2. pathkeys リストを走査し、各 PathKey が保持する EquivalenceClass を取得します。 3. ec_merged を再帰的に辿り、正規化された EquivalenceClass を特定します。 4. そのクラス内の各メンバー（EquivalenceMember）について、print_expr を呼び出して式の内容を表示します。 5. 複数のメンバーがある場合はカンマで区切り、PathKey 間もカンマで区切って整形します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は、提供された抜粋からは特定できません。
  • **呼び出し先**:
  • print_expr: 各メンバーの式を表示するために呼び出されます。
  • lfirst, lnext: リスト走査に使用されます。

関連データ構造

  • PathKey: ソート順序を定義するノード。
  • EquivalenceClass: 同値関係にある式の集合。
  • EquivalenceMember: EquivalenceClass 内の個別の式。
  • List, ListCell: PostgreSQL のリスト構造。

失敗モードと境界条件

  • pathkeys が空の場合、単に () と出力されます。
  • rtable が不適切な場合、print_expr 内部で式の内容が正しく解決・表示されない可能性があります。

保守担当者向けメモ

  • 本関数はデバッグ専用です。本番環境のコードからは呼び出さないでください。
  • print_expr に依存しているため、式表示のフォーマット変更は print_expr の仕様に準じます。
