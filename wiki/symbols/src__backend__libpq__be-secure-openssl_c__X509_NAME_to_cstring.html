X509_NAME_to_cstring

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure-openssl.c:1644

目的

OpenSSL の X509_NAME 構造体を、PostgreSQL 内部で利用可能な形式の C 文字列（/field=value 形式の連結文字列）に変換します。

シグネチャまたは形状

static char *X509_NAME_to_cstring(X509_NAME *name)

入力と出力

  • **入力**: X509_NAME *name （変換対象の X.509 名構造体）
  • **出力**: char * （変換された文字列。メモリは pstrdup により割り当てられるため、呼び出し元で解放が必要）

前提条件と不変条件

  • 入力 name は有効な X509_NAME 構造体である必要があります。
  • 内部で使用する BIO メモリバッファの確保に失敗した場合、ereport(ERROR, ...) により処理が中断されます。

アルゴリズムまたは振る舞い

1. メモリベースの BIO を生成します。 2. X509_NAME 内の各エントリを順次走査します。 3. 各エントリの NID（Name ID）を特定し、対応するフィールド名（短縮名またはロング名）を取得します。 4. BIO_printf を用いて /フィールド名= をバッファに書き込みます。 5. ASN1_STRING_print_ex を使用し、値を RFC 2253 準拠かつ UTF-8 変換を適用してバッファに書き込みます。 6. バッファの末尾にヌル文字を付与し、pg_any_to_server を介して PostgreSQL のサーバーエンコーディングに変換します。 7. 変換後の文字列を pstrdup で複製して返却し、一時的なバッファとリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/be-secure-openssl.c 内の複数箇所（証明書のサブジェクト名や発行者名の取得時）。
  • **呼び出し先**:
  • OpenSSL API: BIO_new, BIO_s_mem, X509_NAME_entry_count, X509_NAME_get_entry, OBJ_obj2nid, X509_NAME_ENTRY_get_object, X509_NAME_ENTRY_get_data, OBJ_nid2sn, OBJ_nid2ln, BIO_printf, ASN1_STRING_print_ex, BIO_write, BIO_get_mem_data, BIO_free
  • PostgreSQL 内部関数: ereport, pg_any_to_server, pstrdup, pfree, elog

関連データ構造

  • X509_NAME: OpenSSL の証明書名構造体。
  • BIO: OpenSSL の入出力抽象化構造体。

失敗モードと境界条件

  • BIO_new が失敗した場合、メモリ不足エラーとして ereport されます。
  • OBJ_obj2nid が NID_undef を返した場合、またはフィールド名が特定できない場合、エラーとして ereport されます。
  • BIO_free が正常終了（1 を返却）しない場合、elog によりエラーが記録されます。

保守担当者向けメモ

  • 本関数は UTF-8 変換を伴うため、エンコーディング変換後のメモリ管理（pfree の要否判定）が重要です。
  • ASN1_STRFLGS_RFC2253 フラグを使用しており、出力形式は RFC 2253 に準拠しています。
