ExecInitParallelPlan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execParallel.c:598

目的

並列クエリ実行のための初期化処理を行い、リーダープロセスとワーカープロセス間で共有される動的共有メモリ（DSM）のセットアップとプランのシリアライズを担う。

シグネチャまたは形状

ParallelExecutorInfo *ExecInitParallelPlan(PlanState *planstate, EState *estate,
                                           Bitmapset *sendParams, int nworkers,
                                           int64 tuples_needed)

入力と出力

  • **入力**:
  • planstate: 実行するプランツリーのルートノード。
  • estate: 現在のクエリ実行状態。
  • sendParams: ワーカーへ転送するパラメータのセット。
  • nworkers: 起動するワーカープロセスの数。
  • tuples_needed: ワーカーが生成すべきタプル数の目安。
  • **出力**:
  • 並列実行の管理情報を含む ParallelExecutorInfo 構造体へのポインタ。

前提条件と不変条件

  • GetActiveSnapshot() == estate->es_snapshot であること（スナップショットの一貫性）。
  • ExecSetParamPlanMulti により、ワーカーへ渡す initplan の出力が評価済みであること。

アルゴリズムまたは振る舞い

1. **プランのシリアライズ**: ExecSerializePlan を呼び出し、プランツリーをワーカーが再構築可能な形式に変換する。 2. **コンテキスト作成**: CreateParallelContext で並列実行コンテキストを生成する。 3. **空間の推定**: shm_toc_estimate_chunk を使用し、固定状態、クエリ文字列、シリアライズ済みプラン、パラメータ、統計情報（BufferUsage/WalUsage）、タプルキュー、およびDSA領域に必要なメモリ量を算出する。 4. **ノードごとの推定**: ExecParallelEstimate を呼び出し、並列対応ノードが必要とする追加メモリ量を加算する。 5. **DSMの確保**: InitializeParallelDSM を呼び出し、算出されたサイズに基づいて共有メモリセグメントを確保する。 6. **データ格納**: shm_toc_allocate と shm_toc_insert を使用し、各データをDSM上の適切なキー（PARALLEL_KEY_*）に配置する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeGather.c
  • src/backend/executor/nodeGatherMerge.c
  • **呼び出し先**:
  • ExecSetParamPlanMulti, ExecSerializePlan, CreateParallelContext, shm_toc_estimate_chunk, shm_toc_estimate_keys, ExecParallelEstimate, InitializeParallelDSM, shm_toc_allocate, shm_toc_insert 等。

関連データ構造

  • ParallelExecutorInfo: 並列実行の全体状態を保持。
  • ParallelContext: 並列実行の基盤コンテキスト。
  • FixedParallelExecutorState: DSM内に格納される固定サイズの実行状態。
  • SharedExecutorInstrumentation: 実行統計情報の共有領域。

失敗モードと境界条件

  • nworkers が 0 の場合や、メモリ確保が失敗した際の挙動は shm_toc および dsa の実装に依存する。
  • estate->es_instrument が有効な場合、ノード数に応じた統計情報領域の計算が必須となる。

保守担当者向けメモ

  • 本関数はプランツリーを再帰的に走査する ExecParallelEstimate を含むため、新しい並列対応ノードを追加する際は、推定ロジックの更新が必要。
  • PARALLEL_KEY_* 定数はDSM内のオフセット管理に直結するため、キーの追加・変更時には競合を避けること。
  • ExecSetParamPlanMulti の呼び出しによるメモリリークのリスクについては、将来的なAPI変更の検討対象である。
