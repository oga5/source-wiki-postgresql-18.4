LimitAdditionalLocalPins

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/localbuf.c:321

目的

一時テーブル用バッファに対して要求された追加のピン留め数（additional_pins）を、利用可能なバッファの残数内に制限します。

シグネチャまたは形状

void LimitAdditionalLocalPins(uint32 *additional_pins)

入力と出力

  • **入力**: additional_pins（制限対象となるピン留め数のポインタ）
  • **出力**: なし（引数のポインタ経由で値を更新）

前提条件と不変条件

  • additional_pins が指す値が 1 以下の場合は、制限を行わず即座に復帰します。
  • ローカルバッファの総数（num_temp_buffers）と現在のピン留め数（NLocalPinnedBuffers）に基づき、許容される最大値が算出されます。

アルゴリズムまたは振る舞い

1. *additional_pins が 1 以下であれば処理を終了します。 2. 利用可能な最大ピン留め数 max_pins を num_temp_buffers - NLocalPinnedBuffers として計算します。 3. 要求された *additional_pins が max_pins を超える場合、*additional_pins を max_pins に切り詰めます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/localbuf.c:360（ExtendBufferedRelLocal 内での呼び出しが確認されています）
  • **呼び出し先**: 直接的な関数呼び出しはソース内にありません。

関連データ構造

  • num_temp_buffers: 一時バッファの総数。
  • NLocalPinnedBuffers: 現在ピン留めされているローカルバッファの数。

失敗モードと境界条件

  • num_temp_buffers が未初期化の場合、その時点での値が使用されます。
  • additional_pins が 0 または 1 の場合は変更されません。

保守担当者向けメモ

  • 本関数はプロセスローカルなバッファ管理専用です。共有バッファマネージャの LimitAdditionalPins とは異なり、他バックエンドの状態を考慮する必要はありません。
