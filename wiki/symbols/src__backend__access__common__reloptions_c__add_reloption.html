add_reloption

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/reloptions.c:710

目的

カスタムリレーションオプション（relopt_gen）をシステム内のグローバルなオプションリストに追加し、管理対象として登録します。

シグネチャまたは形状

static void add_reloption(relopt_gen *newoption)

入力と出力

  • **入力**: newoption (relopt_gen *) - 追加するオプションの定義構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • custom_options 配列は TopMemoryContext 上で管理されます。
  • num_custom_options は現在登録されているカスタムオプションの数を保持するグローバル変数です。
  • need_initialization フラグが true に設定され、後続の初期化処理が必要であることを示します。

アルゴリズムまたは振る舞い

1. 現在の登録数 (num_custom_options) が配列の容量 (max_custom_options) に達している場合、TopMemoryContext に切り替えて配列を拡張（初回は8要素、以降は倍増）します。 2. newoption を custom_options 配列の末尾に追加し、num_custom_options をインクリメントします。 3. need_initialization を true に設定し、オプション定義の再構築を促します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c 内の以下の関数から呼び出されます。
  • add_bool_reloption
  • add_local_reloption
  • add_local_bool_reloption
  • add_int_reloption (推測される関連関数)
  • add_real_reloption (推測される関連関数)
  • add_string_reloption (推測される関連関数)
  • **呼び出し先**: MemoryContextSwitchTo, palloc, repalloc（いずれもメモリ管理用）。

関連データ構造

  • relopt_gen: オプションの基底構造体。
  • custom_options: 登録されたカスタムオプションを保持する静的配列。

失敗モードと境界条件

  • メモリ不足時には palloc/repalloc がエラーを発生させます。
  • max_custom_options が 0 の状態から開始し、動的に拡張される設計になっています。

保守担当者向けメモ

  • 本関数は reloptions.c 内の内部的な登録処理です。新しいオプションタイプを追加する際は、この関数を直接呼ぶのではなく、公開されている add_*_reloption 系関数を利用してください。
  • need_initialization を立てることで、次回 initialize_reloptions が呼ばれた際に新しいオプションが反映される仕組みになっています。
