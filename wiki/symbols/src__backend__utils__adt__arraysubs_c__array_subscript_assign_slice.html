array_subscript_assign_slice

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arraysubs.c:344

目的

配列型に対するスライス代入操作を実行し、更新された配列を生成する。

シグネチャまたは形状

static void array_subscript_assign_slice(ExprState *state, ExprEvalStep *op, ExprContext *econtext)

入力と出力

  • **入力**:
  • op->d.sbsref.state: 添字操作の状態を保持する SubscriptingRefState。
  • op->resvalue: 代入対象の配列（入力）および更新後の配列（出力）。
  • op->resnull: 配列が NULL かどうかを示すフラグ。
  • **出力**:
  • op->resvalue を通じて更新後の配列を返す。

前提条件と不変条件

  • ArraySubWorkspace を通じて、配列の型情報（要素の長さ、アライメント等）が提供されている必要がある。
  • 固定長配列の場合、元の配列または代入値が NULL であれば操作を中断する。

アルゴリズムまたは振る舞い

1. **固定長配列のチェック**: refattrlength > 0 の場合、元の配列または代入値が NULL ならば何もせず終了する。 2. **可変長配列（varlena）の NULL 処理**: 元の配列が NULL の場合、空の配列（0次元）を生成して代入のベースとする。 3. **スライス代入の実行**: array_set_slice を呼び出し、指定された範囲（upperindex, lowerindex）に対して値を代入する。 4. **結果の格納**: array_set_slice の戻り値を op->resvalue に格納する。結果は NULL にならないため、op->resnull は変更しない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arraysubs.c 内の methods->sbs_assign に関数ポインタとして登録されている。
  • **呼び出し先**: array_set_slice（utils/array.h 関連）、construct_empty_array。

関連データ構造

  • SubscriptingRefState: 添字操作の実行時状態。
  • ArraySubWorkspace: 添字の境界値や要素の型情報を保持する構造体。

失敗モードと境界条件

  • 固定長配列において NULL が入力された場合、代入は行われず元の値が維持される。
  • 可変長配列において NULL が入力された場合、空の配列として初期化される。

保守担当者向けメモ

  • 本関数は SubscriptingRef ノードの実行ステップとして呼び出される。
  • workspace 内の refattrlength が 0 より大きい場合は固定長配列として扱われるため、NULL 許容範囲に注意すること。
