repairViewRuleLoop

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump_sort.c:970

目的

ビューとルール間の循環依存関係を解消するために、ルールからビューへの依存関係を削除する。

シグネチャまたは形状

static void repairViewRuleLoop(DumpableObject *viewobj, DumpableObject *ruleobj)

入力と出力

  • viewobj: 循環に含まれるビューの DumpableObject へのポインタ。
  • ruleobj: 循環に含まれるルールの DumpableObject へのポインタ。
  • 戻り値: なし（void）。

前提条件と不変条件

  • viewobj と ruleobj は循環依存関係の一部として特定されていること。
  • 両オブジェクトのフラグは、この修復処理に適した状態に既に設定されていること。

アルゴリズムまたは振る舞い

1. ruleobj が保持する viewobj への依存関係を removeObjectDependency 関数を用いて削除する。 2. これにより、トポロジカルソートにおける循環参照を解消する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_dump/pg_dump_sort.c:1205
  • src/bin/pg_dump/pg_dump_sort.c:1217
  • 呼び出し先:
  • removeObjectDependency

関連データ構造

  • DumpableObject: pg_dump におけるダンプ対象オブジェクトを表現する構造体。

失敗モードと境界条件

  • 循環依存の検出ロジックが誤ったオブジェクトペアを渡した場合、意図しない依存関係の削除が発生する可能性がある。

保守担当者向けメモ

  • 本関数は findDependencyLoops 等によって循環が検出された際に呼び出される修復ロジックの一部である。
  • 依存関係の削除は removeObjectDependency に依存しているため、依存関係管理の仕様変更時には注意が必要である。
