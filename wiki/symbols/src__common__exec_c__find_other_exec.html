find_other_exec

定義

  • 種別: function
  • 定義箇所: src/common/exec.c:309

目的

現在の実行ファイルと同じディレクトリ内に存在する別の実行ファイルを検索し、そのバージョンが期待値と一致するかを検証する。

シグネチャまたは形状

int find_other_exec(const char *argv0, const char *target, const char *versionstr, char *retpath);

入力と出力

  • argv0: 現在のプロセスの起動パス（argv[0]）。
  • target: 検索対象の実行ファイル名。
  • versionstr: 期待されるバージョン文字列。
  • retpath: 検索結果の絶対パスを格納するバッファ（MAXPGPATH以上のサイズが必要）。
  • 戻り値:
  • 0: 成功。
  • -1: 実行ファイルの特定失敗、または検証失敗。
  • -2: バージョン不一致。

前提条件と不変条件

  • retpath は MAXPGPATH バイト以上のメモリ領域を指していること。
  • target はディレクトリパスを含まないファイル名であること。
  • 実行環境において EXE マクロ（Windows用の .exe 等）が適切に定義されていること。

アルゴリズムまたは振る舞い

1. find_my_exec を呼び出し、現在のプロセスの絶対パスを retpath に取得する。 2. retpath からファイル名部分を除去し、ディレクトリパスのみにする。 3. target と EXE 拡張子をディレクトリパスに結合し、フルパスを構築する。 4. validate_exec を呼び出し、対象ファイルが存在し、かつ実行可能であることを確認する。 5. 対象ファイルを "-V" オプション付きで実行し、出力を pipe_read_line で取得する。 6. 取得した出力文字列と versionstr を strcmp で比較し、一致すれば成功として終了する。

関連データ構造

  • MAXPGPATH: パス文字列の最大長を規定する定数。

失敗モードと境界条件

  • find_my_exec が失敗した場合、-1 を返す。
  • validate_exec による権限チェックまたは存在確認に失敗した場合、-1 を返す。
  • pipe_read_line がコマンド実行に失敗または出力を取得できなかった場合、-1 を返す。
  • 取得したバージョン文字列が versionstr と一致しない場合、-2 を返す。

保守担当者向けメモ

  • 本関数は外部コマンドのバージョン整合性を保証するために使用される。
  • pipe_read_line で確保されたメモリは pfree で解放されるが、エラー発生時も適切に解放されるよう設計されている。
  • Windows環境では EXE マクロにより拡張子が自動補完されるため、移植性を維持するために直接ファイル名を指定しないこと。
