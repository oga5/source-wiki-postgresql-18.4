ExecReScanWindowAgg

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWindowAgg.c:2821

目的

WindowAgg ノードの実行状態をリセットし、ウィンドウ集約処理を最初から再実行可能にする。

シグネチャまたは形状

void ExecReScanWindowAgg(WindowAggState *node)

入力と出力

  • **入力**: WindowAggState *node（ウィンドウ集約ノードの実行時状態）
  • **出力**: なし（副作用としてノード内部状態を初期化）

前提条件と不変条件

  • node は初期化済みであること。
  • 呼び出し後、ノードの状態は WINDOWAGG_RUN に設定され、all_first フラグが true になる。

アルゴリズムまたは振る舞い

1. ノードの状態を WINDOWAGG_RUN に設定し、all_first を true にする。 2. release_partition(node) を呼び出し、現在のパーティションに関連する tuplestore 等のリソースを解放する。 3. ExecClearTuple を使用して、スキャン用タプルスロット、パーティション先頭スロット、集約行スロット、および一時作業用スロット（temp_slot_1, temp_slot_2）をクリアする。フレーム境界用スロットが存在する場合はそれらもクリアする。 4. ExprContext 内の集約値（ecxt_aggvalues）と NULL フラグ（ecxt_aggnulls）を MemSet でゼロクリアし、現在のウィンドウ関数値を破棄する。 5. 下位プランノードの chgParam が NULL の場合、ExecReScan を呼び出して下位プランを再スキャンする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:278
  • **呼び出し先**: release_partition, ExecClearTuple, MemSet, ExecReScan（下位プランに対して）

関連データ構造

  • WindowAggState: ウィンドウ集約ノードの実行時状態を保持。
  • ExprContext: 式評価コンテキスト。
  • PlanState: 下位プランノードの状態。

失敗モードと境界条件

  • node が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • 下位プランの chgParam が設定されている場合、この関数内では下位プランの再スキャンは行われない（呼び出し側で処理される）。

保守担当者向けメモ

  • 本関数は tuplestore の解放とスロットのクリアを確実に行う必要がある。新しいスロットやリソースを WindowAggState に追加した場合は、本関数でのクリア処理漏れに注意すること。
