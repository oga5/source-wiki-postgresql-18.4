astreamer_zstd_compressor_content

定義

  • src/fe_utils/astreamer_zstd.c:145

目的

入力データストリームをZstandardアルゴリズムで圧縮し、後続のストリーマーへ転送する。

シグネチャまたは形状

static void astreamer_zstd_compressor_content(
    astreamer *streamer,
    astreamer_member *member,
    const char *data,
    int len,
    astreamer_archive_context context
)

入力と出力

  • **入力**:
  • streamer: 圧縮コンテキストを保持する astreamer_zstd_frame 構造体。
  • data, len: 圧縮対象の入力データバッファとその長さ。
  • **出力**: なし（副作用として後続ストリーマーの astreamer_content を呼び出す）。

前提条件と不変条件

  • streamer は astreamer_zstd_frame 型としてキャスト可能であること。
  • mystreamer->zstd_outBuf が有効な出力バッファを指していること。

アルゴリズムまたは振る舞い

1. 入力データがすべて処理されるまでループする。 2. 出力バッファの空き容量が ZSTD_compressBound で算出される最大必要量に満たない場合、既存の圧縮済みデータを mystreamer->base.bbs_next（後続ストリーマー）へ転送し、出力バッファをリセットする。 3. ZSTD_compressStream2 を呼び出し、データを圧縮して出力バッファに書き込む。 4. ZSTD_isError を用いて圧縮処理のエラーを監視し、エラー発生時は pg_fatal で異常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer インフラストラクチャのパイプライン処理（直接の呼び出し元はソースコード上では明示的な関数名として特定不可）。
  • **呼び出し先**:
  • astreamer_content (後続ストリーマーへのデータ転送)
  • ZSTD_compressStream2 (libzstd)
  • pg_fatal (エラー発生時)

関連データ構造

  • astreamer_zstd_frame: 圧縮コンテキスト（cctx）および出力バッファ（zstd_outBuf）を保持する構造体。

失敗モードと境界条件

  • **圧縮エラー**: ZSTD_compressStream2 がエラーを返した場合、pg_fatal によりプロセスが終了する。
  • **バッファ溢れ**: 出力バッファが満杯になるたびに後続ストリーマーへフラッシュされるため、メモリ不足による境界条件は適切に管理されている。

保守担当者向けメモ

  • 本関数は USE_ZSTD が定義されている環境でのみコンパイルされる。
  • ZSTD_e_continue を使用してストリーム処理を継続している。フラッシュが必要な最終段階については astreamer_zstd_compressor_finalize が担当する。
