intset_subtract

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_op.c:391

目的

2つの整数配列（集合）の差分（第1引数から第2引数に含まれる要素を除外したもの）を計算し、結果を新しい配列として返します。

シグネチャまたは形状

Datum intset_subtract(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの ArrayType を受け取ります。
  • **出力**: Datum 型として、差分結果を格納した新しい ArrayType を返します。

前提条件と不変条件

  • 入力配列は CHECKARRVALID マクロにより整合性が検証されます。
  • 内部処理において、入力配列は QSORT でソートされ、_int_unique で重複排除されます。
  • 処理終了時に、入力としてコピーされた配列は pfree によりメモリ解放されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_ARRAYTYPE_P_COPY を使用して、入力配列 a と b のコピーを作成します。 2. 両方の配列をソートし、重複要素を削除して正規化します。 3. ソート済みの2つの配列を走査し、a に存在し b に存在しない要素のみを抽出して結果配列 result に格納します。 4. resize_intArrayType を呼び出し、実際に格納された要素数に合わせて結果配列のサイズを調整します。 5. 作業用メモリを解放し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(intset_subtract) マクロにより、PostgreSQLの関数インターフェースとして登録されています。
  • **呼び出し先**:
  • CHECKARRVALID (マクロ)
  • QSORT (マクロ)
  • _int_unique
  • new_intArrayType
  • resize_intArrayType
  • pfree
  • PG_RETURN_POINTER (マクロ)

関連データ構造

  • ArrayType: PostgreSQLの配列データ型。
  • int32: 配列要素の型。

失敗モードと境界条件

  • 入力配列が不正な形式である場合、CHECKARRVALID によりエラーが発生する可能性があります。
  • メモリ不足時には、new_intArrayType 等のメモリ割り当て関数がエラーをスローします。

保守担当者向けメモ

  • 本関数は入力配列をコピーして操作するため、元のデータには影響を与えません。
  • 効率的な差分計算のため、入力が事前にソートおよび重複排除されていることを前提としています。
  • _int.h で定義されたマクロやヘルパー関数に強く依存しているため、変更時は同ヘッダーの定義を確認してください。
