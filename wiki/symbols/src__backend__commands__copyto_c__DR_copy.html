DR_copy

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/copyto.c:101

目的

クエリ実行結果を COPY TO コマンドの出力として受け取るためのブリッジとなる DestReceiver 構造体です。

シグネチャまたは形状

typedef struct {
    DestReceiver pub;        /* 公開関数ポインタ */
    CopyToState cstate;      /* コマンドの実行状態 */
    uint64     processed;    /* 処理済みタプル数 */
} DR_copy;

入力と出力

  • **入力**: DestReceiver インターフェースを介してクエリ実行エンジンからタプルを受け取ります。
  • **出力**: 内部の CopyToState を通じて、ファイル、外部プログラム、またはクライアントへデータを出力します。

前提条件と不変条件

  • pub フィールドは DestReceiver として初期化されている必要があります。
  • cstate は COPY TO 操作の有効な状態を指している必要があります。

アルゴリズムまたは振る舞い

クエリ実行エンジンがタプルを生成するたびに、DR_copy を介して COPY の出力処理が呼び出されます。processed カウンタは、処理されたタプル数を追跡するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: CreateCopyDestReceiver 関数内でインスタンス化されます。
  • **呼び出し先**: 実行エンジンがタプルを渡す際に、DestReceiver の関数ポインタを通じて間接的に呼び出されます。また、DoCopyTo 等の関数から cstate や processed フィールドが直接参照されます。

関連データ構造

  • DestReceiver: 本構造体が継承する基底インターフェース。
  • CopyToStateData: cstate が参照する、COPY TO 操作の全状態を保持する構造体。

失敗モードと境界条件

  • cstate が NULL の状態でタプル処理が行われた場合、セグメンテーション違反が発生する可能性があります。
  • processed カウンタは uint64 型であり、非常に大規模なデータセットではオーバーフローの可能性がありますが、通常の COPY 操作の範囲内では十分な容量を持ちます。

保守担当者向けメモ

  • 本構造体は DestReceiver の派生型であるため、DestReceiver の仕様変更時には pub フィールドの整合性に注意してください。
  • processed フィールドは統計情報として利用されるため、タプル処理の正確なカウントを維持してください。
