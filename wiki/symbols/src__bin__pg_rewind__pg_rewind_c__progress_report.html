progress_report

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/pg_rewind.c:792

目的

データ同期処理の進捗状況を標準エラー出力（stderr）へ報告する。

シグネチャまたは形状

void progress_report(bool finished)

入力と出力

  • **入力**:
  • finished (bool): 処理が完了したかどうかを示すフラグ。
  • **出力**: なし（副作用として stderr に進捗状況を出力）。

前提条件と不変条件

  • グローバル変数 showprogress が真である場合にのみ動作する。
  • 処理完了時（finished が真）を除き、前回の報告から1秒以上経過していない場合は何も出力しない。

アルゴリズムまたは振る舞い

1. showprogress を確認し、偽であれば即座にリターンする。 2. 現在時刻を取得し、前回の報告時刻と比較して1秒未満であればリターンする。 3. fetch_done（完了量）と fetch_size（総量）から進捗率（%）を算出する。 4. 進捗率が100%を超えないよう制限し、fetch_done が fetch_size を超えた場合は fetch_size を更新する。 5. 進捗状況を「完了量/総量 kB (進捗率%) copied」の形式でフォーマットする。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内では明示的な呼び出し元は特定されていません。
  • **呼び出し先**: time(), snprintf(), fprintf(), fputc(), isatty(), fileno()。

関連データ構造

  • fetch_done (グローバル変数と推測される)
  • fetch_size (グローバル変数と推測される)
  • showprogress (グローバル変数と推測される)

失敗モードと境界条件

  • fetch_size が0の場合、進捗率は0%として扱われる。
  • 端末以外（パイプ等）への出力時は、進捗状況が改行を伴って逐次出力される。

保守担当者向けメモ

  • 本関数は pg_rewind 固有の進捗報告ロジックです。他ツール（pg_amcheck や pg_basebackup）にも同名の関数が存在しますが、シグネチャや実装が異なるため混同しないよう注意してください。
  • fetch_done および fetch_size の単位はバイト単位で管理されている前提で、出力時に kB へ変換しています。
