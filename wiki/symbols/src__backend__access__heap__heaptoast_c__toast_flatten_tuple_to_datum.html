toast_flatten_tuple_to_datum

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heaptoast.c:448

目的

TOAST（外部保存または圧縮）された属性を含むタプルを、すべての属性がインライン化された（デトーストされた）状態の新しいタプルへ変換し、Datum として返します。

シグネチャまたは形状

Datum toast_flatten_tuple_to_datum(HeapTupleHeader tup, uint32 tup_len, TupleDesc tupleDesc)

入力と出力

  • **入力**:
  • tup: 処理対象の HeapTupleHeader。
  • tup_len: タプルの長さ。
  • tupleDesc: タプルの記述子（属性定義）。
  • **出力**:
  • 変換後のタプルを指す Datum。メモリは palloc されており、呼び出し元で管理する必要があります。

前提条件と不変条件

  • numAttrs（属性数）は MaxTupleAttributeNumber 以下であること。
  • 外部保存または圧縮された属性は、detoast_attr を介してメモリ上に展開されます。
  • 変換後のタプルは、元のタプルのヘッダー情報を継承しつつ、natts、t_hoff、および複合型としてのヘッダーフィールド（型ID、型モディファイア等）が適切に再設定されます。

アルゴリズムまたは振る舞い

1. **分解**: heap_deform_tuple を使用して、入力タプルを属性値の配列に分解します。 2. **デトースト**: 各属性を走査し、varlena 型かつ外部保存または圧縮されている属性を detoast_attr で展開します。展開された値は一時的に保持され、後で解放されます。 3. **サイズ計算**: 展開後の属性に基づき、新しいタプルのヘッダーサイズとデータサイズを計算します。 4. **メモリ確保**: 計算されたサイズ分を palloc0 で確保します。 5. **構築**: memcpy でヘッダーをコピーし、heap_fill_tuple でデータを書き込みます。 6. **クリーンアップ**: デトースト処理で一時的に確保したメモリを pfree します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/heaptuple.c:1090
  • src/backend/executor/execTuples.c:2427
  • **呼び出し先**:
  • heap_deform_tuple
  • detoast_attr
  • heap_compute_data_size
  • heap_fill_tuple
  • palloc0 / pfree

関連データ構造

  • HeapTupleHeader: タプルの物理的なヘッダー構造。
  • TupleDesc: タプルのスキーマ定義。
  • varlena: 可変長データの内部表現。

失敗モードと境界条件

  • 属性数が MaxTupleAttributeNumber を超える場合、Assert により停止します。
  • メモリ不足時には palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本関数は heap_toast_insert_or_update 内の再構築ロジックと整合性を保つ必要があります。
  • 戻り値の Datum は palloc されたメモリを指しているため、呼び出し側でのメモリリークに注意してください。
