ClusterParams

定義

  • 種別: struct
  • 定義箇所: src/include/commands/cluster.h:29

目的

CLUSTER コマンドおよび VACUUM FULL 操作における実行オプションを保持し、バックエンドの処理ロジックへ伝達するための構造体です。

シグネチャまたは形状

typedef struct ClusterParams
{
	bits32		options;		/* bitmask of CLUOPT_* */
} ClusterParams;

入力と出力

  • **入力**: options フィールドに CLUOPT_* ビットフラグを格納します。
  • **出力**: 構造体自体が cluster_rel や cluster_multiple_rels などの関数へ引数として渡され、処理の挙動を制御します。

前提条件と不変条件

  • options フィールドは CLUOPT_* で定義されたビットマスク値である必要があります。

アルゴリズムまたは振る舞い

  • CLUSTER コマンドの解析フェーズで生成され、実行フェーズにおいて VERBOSE モードの有無などの制御フラグとして参照されます。
  • 処理の実行中、この構造体を通じて一貫したオプション設定が各サブ関数（cluster_rel 等）に共有されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/cluster.c: cluster_multiple_rels および cluster_rel の呼び出し元で初期化されます。
  • src/backend/commands/vacuum.c: VACUUM FULL 実行時に初期化されます。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • CLUOPT_*: options フィールドで使用されるビットフラグ群。

失敗モードと境界条件

  • 構造体自体の失敗モードは定義されていません。ただし、options に無効なビットフラグが設定された場合、呼び出し先の関数が予期しない挙動を示す可能性があります。

保守担当者向けメモ

  • 本構造体は CLUSTER 操作の拡張に伴い、新しいオプションフラグが追加される可能性があります。
  • cluster.c および vacuum.c での初期化パターン（{0} によるゼロ初期化）が一般的です。
