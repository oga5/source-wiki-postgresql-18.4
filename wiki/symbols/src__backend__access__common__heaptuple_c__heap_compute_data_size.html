heap_compute_data_size

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/heaptuple.c:218

目的

タプルを物理的に構築する際に必要となる、データ領域の合計サイズを計算します。

シグネチャまたは形状

Size heap_compute_data_size(TupleDesc tupleDesc, const Datum *values, const bool *isnull)

入力と出力

  • **入力**:
  • tupleDesc: タプルの属性定義情報。
  • values: 属性値の配列（Datum）。
  • isnull: 各属性がNULLかどうかを示すフラグ配列。
  • **出力**:
  • 計算されたデータ領域の合計サイズ（Size）。

前提条件と不変条件

  • isnull[i] が真の場合、その属性は計算対象から除外されます。
  • 属性の型に応じたアライメント（attalignby）が考慮されます。

アルゴリズムまたは振る舞い

各属性を順次走査し、以下のいずれかの処理で data_length を加算します。 1. **短縮可能（Packable）な可変長データ**: 1バイトヘッダーへの変換が見込まれる場合、変換後のサイズを加算します（アライメント調整は行いません）。 2. **展開済み（Expanded）データ**: 外部展開されたデータの場合、フラット化したサイズを加算し、アライメント調整を行います。 3. **標準的なデータ**: 型定義に基づき、アライメント調整を行った上でデータサイズを加算します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_tuple.c
  • src/backend/access/common/heaptuple.c
  • src/backend/access/common/indextuple.c
  • src/backend/access/heap/heaptoast.c

  • EOH_get_flat_size

  • att_datum_alignby
  • att_addlength_datum

関連データ構造

  • TupleDesc: タプルの属性記述子。
  • CompactAttribute: 属性の物理レイアウト情報。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • isnull 配列の要素数が tupleDesc->natts と一致しない場合、メモリ破壊や不正なアクセスが発生する可能性があります。
  • values 配列内のポインタが指すデータが、tupleDesc の定義と不整合である場合、計算結果が不正になります。

保守担当者向けメモ

  • 本関数はタプル構築の初期段階で呼ばれるため、ここでの計算結果と実際の heap_fill_tuple での書き込みロジックに乖離が生じないよう注意してください。
  • 圧縮や展開に関するロジックが変更された場合、本関数のサイズ計算式も追従する必要があります。
