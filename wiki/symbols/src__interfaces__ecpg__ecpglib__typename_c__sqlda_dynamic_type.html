sqlda_dynamic_type

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/typename.c:106

目的

PostgreSQLのシステムカタログOIDを、ECPGライブラリ内部で使用される型識別子（ECPGttype）へ変換します。主にSQLDA（SQL Descriptor Area）処理において、データベースから取得したデータの型をECPG側で解釈可能な形式にマッピングするために使用されます。

シグネチャまたは形状

int sqlda_dynamic_type(Oid type, enum COMPAT_MODE compat);

入力と出力

  • **入力**:
  • type (Oid): PostgreSQLの型OID。
  • compat (enum COMPAT_MODE): 互換モード（Informixモード等の判定に使用）。
  • **出力**:
  • int: ECPG内部の型列挙子（ECPGt_*）。

前提条件と不変条件

  • INT8OID の変換には、プラットフォームの long または long long のサイズに依存する条件コンパイルが含まれており、8バイトの整数型が利用可能であることが前提となります。

アルゴリズムまたは振る舞い

1. 入力された type (OID) に基づき switch 文で分岐します。 2. 文字列系（CHAROID, VARCHAROID 等）は ECPGt_char を返します。 3. 数値系（INT2OID, INT4OID, FLOAT8OID 等）は対応する ECPGt_* 型を返します。 4. NUMERICOID は compat モードが INFORMIX_MODE の場合に ECPGt_decimal を、それ以外は ECPGt_numeric を返します。 5. INT8OID はコンパイル時の環境（SIZEOF_LONG 等）に応じて適切な整数型を返します。 6. 上記のいずれにも該当しない未知の型は、デフォルトとして ECPGt_char を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/sqlda.c: PQftype の結果を変換するために複数箇所で呼び出されています。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません（純粋な変換ロジック）。

関連データ構造

  • ECPGttype (ECPG内部の型定義)
  • COMPAT_MODE (互換性フラグ)
  • Oid (PostgreSQLの型識別子)

失敗モードと境界条件

  • 未知のOIDが渡された場合、エラーを返さず ECPGt_char を返すことで安全性を確保しています。
  • INT8OID を処理する際、環境が8バイトの整数型をサポートしていない場合はコンパイルエラーとなります。

保守担当者向けメモ

  • 新しいPostgreSQLの型をサポートする必要がある場合は、switch 文に新しい case を追加してください。
  • INT8OID の処理はプラットフォーム依存であるため、移植性を確認する際は注意が必要です。
