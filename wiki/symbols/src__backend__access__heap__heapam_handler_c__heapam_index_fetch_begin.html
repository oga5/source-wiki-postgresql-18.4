heapam_index_fetch_begin

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_handler.c:81

目的

インデックススキャンを開始するためのヒープAM固有のデータ構造を初期化し、割り当てます。

シグネチャまたは形状

IndexFetchTableData *heapam_index_fetch_begin(Relation rel)

入力と出力

  • **入力**: rel (対象となるリレーション)
  • **出力**: IndexFetchTableData 構造体へのポインタ（実際には IndexFetchHeapData のベース部分）。

前提条件と不変条件

  • 呼び出し時に rel は有効なリレーションである必要があります。
  • palloc0 を使用してメモリを確保するため、メモリコンテキストが適切に設定されている必要があります。

アルゴリズムまたは振る舞い

1. IndexFetchHeapData 構造体分のメモリをゼロ初期化して確保します。 2. 確保した構造体の xs_base.rel に引数の rel を設定します。 3. バッファ管理用の xs_cbuf を InvalidBuffer に初期化します。 4. 構造体のベース部分である xs_base へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_handler.c 内の heapam_methods 構造体（TableAmRoutine）の .index_fetch_begin フィールドとして登録されています。
  • **呼び出し先**: 外部の呼び出し元情報は提供されていません。

関連データ構造

  • IndexFetchHeapData: ヒープAMにおけるインデックススキャン状態を保持する構造体。
  • IndexFetchTableData: テーブルAM共通のインデックススキャン状態構造体。
  • Relation: 操作対象のリレーション情報。

失敗モードと境界条件

  • メモリ不足時には palloc0 によりエラーが発生します。

保守担当者向けメモ

  • 本関数は TableAmRoutine の一部として呼び出されるインターフェースです。
  • IndexFetchHeapData 構造体の定義は heapam_handler.c 内に存在し、ヒープ特有のバッファ管理状態を保持します。
