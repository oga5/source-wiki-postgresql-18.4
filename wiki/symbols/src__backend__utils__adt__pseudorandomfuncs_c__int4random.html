int4random

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pseudorandomfuncs.c:125

目的

指定された範囲（下限値から上限値まで）の整数を生成し、SQL層へ返却します。

シグネチャまたは形状

Datum int4random(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_INT32(0): 乱数生成範囲の下限値 (rmin)
  • PG_GETARG_INT32(1): 乱数生成範囲の上限値 (rmax)
  • **出力**:
  • PG_RETURN_INT32(result): rmin 以上 rmax 以下の範囲で生成された int32 型の乱数

前提条件と不変条件

  • rmin <= rmax である必要があります。これに違反する場合、エラーが発生します。
  • 乱数生成器の状態 (prng_state) は、関数呼び出し前に初期化されているか、関数内で initialize_prng() を通じて初期化されます。

アルゴリズムまたは振る舞い

1. 引数 rmin と rmax を取得します。 2. rmin > rmax の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告します。 3. initialize_prng() を呼び出し、乱数生成器の状態を準備します。 4. pg_prng_int64_range を使用して、指定範囲内の乱数を生成します。 5. 生成された値を int32 にキャストし、PG_RETURN_INT32 マクロで返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（詳細は提供されていません）。
  • **呼び出し先**:
  • initialize_prng()
  • pg_prng_int64_range()

関連データ構造

  • prng_state: プロセス単位で保持される乱数生成器の状態構造体。

失敗モードと境界条件

  • rmin > rmax: ereport によりエラーが発生し、処理が中断されます。

保守担当者向けメモ

  • 本関数は int32 型の範囲を扱います。より大きな範囲が必要な場合は int8random の利用を検討してください。
  • 乱数生成器の状態はプロセス内で共有されるため、並列クエリ間での同期は行われません。
