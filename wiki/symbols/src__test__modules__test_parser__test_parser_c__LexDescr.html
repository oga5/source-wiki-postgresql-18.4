LexDescr

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_parser/test_parser.c:31

目的

PostgreSQLの全文検索パーサーがサポートするトークン型を定義し、システムに通知するためのメタデータ構造体です。

シグネチャまたは形状

typedef struct
{
	int			lexid;
	char	   *alias;
	char	   *descr;
} LexDescr;

入力と出力

  • lexid: トークンを識別する整数ID。
  • alias: トークンの短い別名（文字列）。
  • descr: トークンの説明文（文字列）。

前提条件と不変条件

  • testprs_lextype 関数において、トークン型のリストとして配列形式で返されることを想定しています。
  • 配列の末尾には、通常、番兵（sentinel）となる要素が含まれる必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用であり、ロジックは持ちません。パーサーの実装（testprs_lextype）において、この構造体の配列をメモリ確保し、各トークン型情報を格納して呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_parser/test_parser.c:115 (testprs_lextype 内で配列を割り当て)
  • src/backend/tsearch/wparser_def.c:1881 (標準パーサーの実装内で配列を割り当て)
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。
  • **参照元（ツールによる推論）**:
  • src/backend/commands/tsearchcmds.c および src/backend/tsearch/wparser.c において、パーサーのトークン型リストを受け取る変数として参照されています。

関連データ構造

  • ParserState: 解析中の状態を管理する構造体。

失敗モードと境界条件

  • palloc によるメモリ確保失敗時にエラーが発生する可能性があります。
  • 配列のサイズ指定（LASTNUM + 1 等）が不適切である場合、メモリ破壊や読み取りエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/tsearch/ts_public.h にも定義が存在します。変更を加える際は、PostgreSQLの全文検索インターフェースとの互換性に注意してください。
  • lexid の値は、PostgreSQLの標準パーサーと合わせることで、既存のハイライト機能などを再利用可能になります。
