conditional_stack_set_query_len

定義

  • 種別: function
  • 定義箇所: src/fe_utils/conditional.c:150

目的

\if ブロックのネスト管理において、現在のスタックトップ要素にクエリバッファの長さ（query_len）を記録する。これは、非アクティブなブロック内で蓄積されたバッファを後で破棄・復元するために使用されるメタデータである。

シグネチャまたは形状

void conditional_stack_set_query_len(ConditionalStack cstack, int len)

入力と出力

  • cstack: 操作対象の ConditionalStack 構造体。
  • len: 設定するクエリバッファの長さ（整数値）。
  • 戻り値: なし。

前提条件と不変条件

  • cstack が空であってはならない（Assert により検証される）。

アルゴリズムまたは振る舞い

1. conditional_stack_empty(cstack) を呼び出し、スタックが空でないことを確認する。 2. スタックの先頭要素（cstack->head）の query_len フィールドに、引数 len を代入する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/command.c:3804 にて、query_buf->len を引数として呼び出されている。
  • 呼び出し先: conditional_stack_empty（マクロまたは関数）を呼び出し、スタックの状態を検証する。

関連データ構造

  • ConditionalStack: \if ブロックのネスト状態を管理するスタック構造体。
  • head: スタックの最上位要素を指すポインタ。

失敗モードと境界条件

  • スタックが空の状態で呼び出された場合、アサーション違反によりプログラムが停止する（デバッグビルド時）。

保守担当者向けメモ

  • 本関数は純粋なデータ構造の更新を行う。構文解析ロジックは含まない。
  • query_len は、\if ブロックが非アクティブな際に誤って蓄積されたバッファを適切に管理するための重要なメタデータである。
