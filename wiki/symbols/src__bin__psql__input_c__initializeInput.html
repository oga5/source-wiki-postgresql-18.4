initializeInput

定義

  • 種別: function
  • 定義箇所: src/bin/psql/input.c:343

目的

psql 起動時に、readline ライブラリの初期化および履歴ファイルのパス決定と読み込みを行う。

シグネチャまたは形状

void initializeInput(int flags)

入力と出力

  • **入力**: flags (int型)。ビット0（flags & 1）がセットされている場合、readline を有効化する。
  • **出力**: なし（副作用としてグローバル状態および履歴ファイルを操作する）。

前提条件と不変条件

  • USE_READLINE が定義されている環境でのみ、readline 関連の初期化が実行される。
  • atexit(finishInput) が登録され、プログラム終了時に履歴の保存が保証される。

アルゴリズムまたは振る舞い

1. flags & 1 が真の場合:

  • useReadline および useHistory を true に設定。
  • initialize_readline() を呼び出し、readline のグローバル変数を設定。
  • rl_variable_bind で comment-begin を -- に設定し、rl_initialize() を実行。
  • 履歴ファイルパスを以下の優先順位で決定:

    • 1. pset.vars 内の HISTFILE 変数。 2. 環境変数 PSQL_HISTORY。 3. ホームディレクトリ下のデフォルトパス（.psql_history）。

  • 決定したパスから read_history() で履歴を読み込み、decode_history() で内容を復元する。

    • 2. 終了処理として finishInput を atexit に登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/startup.c:466
  • **呼び出し先**: initialize_readline, rl_variable_bind, rl_initialize, using_history, GetVariable, getenv, get_home_path, psprintf, pg_strdup, expand_tilde, read_history, decode_history, atexit(finishInput)

関連データ構造

  • pset.vars: 履歴ファイルパス取得に使用する変数格納構造体。
  • psql_history: 履歴ファイルのパスを保持するグローバル変数。

失敗モードと境界条件

  • HISTFILE や PSQL_HISTORY が未設定かつホームディレクトリが特定できない場合、履歴の読み込みはスキップされる。
  • psql_history が空または無効なパスの場合、履歴の読み込みは行われない。

保守担当者向けメモ

  • readline の初期化順序（rl_variable_bind の後に rl_initialize）は、.inputrc の読み込みに関連するため重要である。
  • flags & 1 の判定は startup.c で options.no_readline の値に基づいて決定される。
