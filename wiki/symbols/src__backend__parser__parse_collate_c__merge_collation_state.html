merge_collation_state

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_collate.c:779

目的

式解析の過程で、ノードの照合順序（Collation）情報を親コンテキストにマージし、照合順序の伝播、競合検出、および解決を行う。

シグネチャまたは形状

static void merge_collation_state(Oid collation,
                                  CollateStrength strength,
                                  int location,
                                  Oid collation2,
                                  int location2,
                                  assign_collations_context *context)

入力と出力

  • **入力**:
  • collation, strength, location: 現在処理中のノードの照合順序OID、派生強度、および構文上の位置。
  • collation2, location2: 競合発生時に使用される補助的な照合順序情報。
  • context: 照合順序の解決状態を保持する assign_collations_context 構造体。
  • **出力**: なし（context 構造体の内容を直接更新する）。

前提条件と不変条件

  • context は有効なポインタであること。
  • strength は CollateStrength 列挙型（NONE, IMPLICIT, CONFLICT, EXPLICIT）のいずれかであること。

アルゴリズムまたは振る舞い

1. **強度比較**: 現在のノードの strength が context->strength より大きい場合、親のコンテキストを現在のノードの情報で上書きする。 2. **マージ処理**: 強度が等しい場合、以下のルールで解決する。

  • COLLATE_NONE: 何もしない。
  • COLLATE_IMPLICIT: 異なるOIDを持つ場合、デフォルト照合順序（DEFAULT_COLLATION_OID）であれば上書きし、それ以外で競合する場合は COLLATE_CONFLICT 状態へ移行する。
  • COLLATE_CONFLICT: 状態を維持する。
  • COLLATE_EXPLICIT: 異なるOIDを持つ場合、直ちに ERRCODE_COLLATION_MISMATCH エラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_collate.c 内の各箇所（assign_expr_collations 等から呼び出される）。
  • **呼び出し先**: ereport（エラー発生時）、get_collation_name（エラーメッセージ生成時）。

関連データ構造

  • assign_collations_context: 走査中の照合順序状態を保持する構造体。
  • CollateStrength: 照合順序の派生強度を定義する列挙型。

失敗モードと境界条件

  • **明示的な競合**: COLLATE_EXPLICIT 同士で異なる照合順序が指定された場合、ereport(ERROR, ...) により処理が中断される。
  • **暗黙的な競合**: COLLATE_IMPLICIT 同士でデフォルト以外の異なる照合順序が指定された場合、エラーを即座に投げず、context に競合情報を記録して後続の処理に委ねる。

保守担当者向けメモ

  • 本関数は、SQL標準に従い、明示的な照合順序の競合に対しては厳格なエラーチェックを行う。
  • COLLATE_IMPLICIT の競合解決は、後続の CollateExpr による解決の可能性を考慮し、遅延評価される設計となっている。
