BloomFilter

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/brin/brin_bloom.c:241

目的

BRIN（Block Range Index）において、ページ範囲内の値を要約し、等価性検索（=）の効率的な絞り込みを行うためのブルームフィルタ構造体です。

シグネチャまたは形状

typedef struct BloomFilter
{
    int32       vl_len_;        /* varlenaヘッダー */
    uint16      flags;          /* 未使用 */
    uint8       nhashes;        /* ハッシュ関数数 */
    uint32      nbits;          /* ビットマップのサイズ（ビット数） */
    uint32      nbits_set;      /* セットされているビット数 */
    char        data[FLEXIBLE_ARRAY_MEMBER]; /* フィルタ本体 */
} BloomFilter;

入力と出力

  • **入力**: bloom_add_value を通じて uint32 型のハッシュ値を受け取り、フィルタ内のビットを更新します。
  • **出力**: bloom_contains_value を通じて、指定された値がフィルタ内に存在する可能性があるか（bool）を返します。

前提条件と不変条件

  • varlena ヘッダーを含むため、直接的なメモリ操作は避ける必要があります。
  • フィルタデータは data メンバに可変長配列として格納されます。
  • フィルタサイズは1つのインデックスページ（8kB）に収まる必要があります。

アルゴリズムまたは振る舞い

  • 「Less Hashing, Same Performance」論文に基づき、2つのハッシュ関数から複数のハッシュ値を生成する手法を採用しています。
  • bloom_add_value は、ハッシュ値に基づいてビットマップの該当ビットを立てます。
  • bloom_contains_value は、すべての該当ビットが立っているかを確認し、包含判定を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_bloom_add_value、brin_bloom_consistent、brin_bloom_union 等のBRIN演算子クラス関数から参照されます。
  • **呼び出し先**: palloc0（メモリ確保時）、PG_DETOAST_DATUM（データ取得時）が使用されます。

関連データ構造

  • BloomOptions: 誤判定率などの設定を保持。
  • BloomOpaque: インデックスのメタデータ保持に使用。

失敗モードと境界条件

  • BloomMaxFilterSize を超えるサイズのフィルタはエラーとなります。
  • 誤判定率（false_positive_rate）の設定により、フィルタのサイズと検索精度がトレードオフの関係にあります。

保守担当者向けメモ

  • 本構造体は varlena 型であるため、メモリ確保時には offsetof(BloomFilter, data) + nbytes を使用して適切なサイズを計算してください。
  • flags フィールドは現在未使用ですが、将来的な拡張のために確保されています。
