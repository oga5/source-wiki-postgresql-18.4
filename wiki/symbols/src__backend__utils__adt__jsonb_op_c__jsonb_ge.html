jsonb_ge

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_op.c:207

目的

2つの jsonb 値を比較し、左側の値が右側の値以上であるか（>=）を判定します。

シグネチャまたは形状

Datum jsonb_ge(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの jsonb 型データ（jba, jbb）を受け取ります。
  • **出力**: bool 型の判定結果を Datum として返します。

前提条件と不変条件

  • 入力された jsonb ポインタは PG_GETARG_JSONB_P マクロにより適切に取得される必要があります。
  • 比較処理の終了後、PG_FREE_IF_COPY を使用してメモリの解放を行う必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_JSONB_P を用いて2つの jsonb オブジェクトを取得します。 2. compareJsonbContainers 関数を呼び出し、両者の root 構造を比較します。 3. 比較結果が 0 以上（>= 0）であれば true、そうでなければ false を返します。 4. PG_FREE_IF_COPY により、必要に応じて入力引数のメモリを解放します。 5. PG_RETURN_BOOL を介して結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（fmgr）を介してSQL演算子から呼び出されます。具体的な呼び出し元のコード箇所は提供されていません。
  • **呼び出し先**: compareJsonbContainers（src/backend/utils/adt/jsonb_util.c で定義されている補助関数）を呼び出します。

関連データ構造

  • Jsonb: jsonb 型の内部表現を保持する構造体。
  • JsonbContainer: Jsonb 構造体内の root フィールドとして参照されるデータ構造。

失敗モードと境界条件

  • 入力が NULL の場合、PG_GETARG_JSONB_P の挙動に従い、呼び出し元で適切に処理される必要があります。
  • compareJsonbContainers が内部的に行う比較ロジックに依存しており、JSONBの型順序や値の比較規則に従います。

保守担当者向けメモ

  • 本関数はSQLレベルの >= 演算子に対応しています。
  • 比較ロジックの変更が必要な場合は、compareJsonbContainers の実装を確認してください。
