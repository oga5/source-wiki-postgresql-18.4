ordered_set_startup

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/orderedsetaggs.c:112

目的

順序集合集約関数（Ordered-set aggregate）の実行に必要な状態管理構造を初期化し、ソート処理（tuplesort）の準備を行う。

シグネチャまたは形状

OSAPerGroupState *ordered_set_startup(FunctionCallInfo fcinfo, bool use_tuples)

入力と出力

  • **入力**:
  • fcinfo: 関数呼び出し情報。
  • use_tuples: 複数カラムのソートを行うか（true）、単一のデータムをソートするか（false）を指定するフラグ。
  • **出力**:
  • 戻り値: グループ単位の実行状態を保持する OSAPerGroupState 構造体へのポインタ。

前提条件と不変条件

  • AggCheckCallContext により、集約コンテキスト内から呼び出されている必要がある。
  • fn_extra を介してクエリ単位の状態（OSAPerQueryState）をキャッシュし、再利用する。
  • AGGKIND_HYPOTHETICAL（仮説的集約）の場合、ソート対象にフラグカラムを追加する。

アルゴリズムまたは振る舞い

1. **コンテキスト確認**: 集約コンテキストであることを検証する。 2. **クエリ状態の初期化**: fn_extra が未設定の場合、OSAPerQueryState を作成し、集約対象のソート順序、演算子、型情報を抽出してキャッシュする。

  • use_tuples が true の場合、Aggref からタプル記述子（TupleDesc）を生成し、必要に応じてフラグカラムを追加する。
  • use_tuples が false の場合、単一データム用の型情報を保存する。

    • 3. **グループ状態の作成**: OSAPerGroupState を割り当て、クエリ状態への参照と現在のグループコンテキストを保持する。 4. **ソート準備**: rescan_needed の判定に基づき、tuplesort のオプション（TUPLESORT_RANDOMACCESS 等）を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/orderedsetaggs.c 内の各集約遷移関数（ordered_set_transition 等）から呼び出される。
  • **呼び出し先**:
  • AggCheckCallContext, AggGetAggref, MemoryContextSwitchTo, palloc0, ExecTypeFromTL, MakeSingleTupleTableSlot, get_typlenbyvalalign 等のPostgreSQL内部API。

関連データ構造

  • OSAPerQueryState: クエリ全体で共有されるメタデータ（ソート順、型情報）。
  • OSAPerGroupState: グループ単位の実行状態（tuplesort オブジェクトを保持）。
  • Aggref: 集約関数のノード情報。

失敗モードと境界条件

  • **エラー**:
  • 集約コンテキスト以外から呼び出された場合、elog(ERROR) を発生させる。
  • AGGKIND_HYPOTHETICAL でない場合に複数カラムのソートを要求するなど、サポート外の構成が検出された場合、エラーとなる。
  • **境界条件**:
  • tuplesort は work_mem を使用するため、大量データ時には一時ファイルが生成される可能性がある。

保守担当者向けメモ

  • OSAPerQueryState はクエリのライフサイクルに依存し、OSAPerGroupState はグループのライフサイクルに依存する。メモリリークを防ぐため、集約終了時には必ず ordered_set_shutdown を介したクリーンアップを行うこと。
  • 仮説的集約（AGGKIND_HYPOTHETICAL）では、ソート対象にフラグカラムを追加する特殊な処理が含まれているため、変更時は注意が必要。
