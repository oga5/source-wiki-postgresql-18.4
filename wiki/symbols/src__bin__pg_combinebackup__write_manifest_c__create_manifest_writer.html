create_manifest_writer

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/write_manifest.c:47

目的

バックアップマニフェスト生成用の manifest_writer 構造体を初期化し、JSON 形式のヘッダー情報を構築する。

シグネチャまたは形状

manifest_writer *create_manifest_writer(char *directory, uint64 system_identifier);

入力と出力

  • **入力**:
  • directory: マニフェストファイル (backup_manifest) を作成するディレクトリパス。
  • system_identifier: バックアップ対象のシステム識別子。
  • **出力**:
  • 初期化済みの manifest_writer 構造体へのポインタ。

前提条件と不変条件

  • 指定された directory に書き込み権限があること。
  • pg_malloc によるメモリ確保が成功すること。
  • manifest_writer の fd は初期状態として -1 に設定される。

アルゴリズムまたは振る舞い

1. manifest_writer 構造体用のメモリを確保する。 2. directory パスを基に backup_manifest ファイルのフルパスを生成する。 3. StringInfo バッファを初期化し、JSON の開始タグ、バージョン情報、システム識別子をバッファに書き込む。 4. SHA-256 チェックサム計算用のコンテキストを初期化する。 5. first_file フラグを true に設定し、後続のファイル追加処理に備える。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/pg_combinebackup.c:338
  • **呼び出し先**:
  • pg_malloc
  • initStringInfo
  • pg_checksum_init
  • appendStringInfo

関連データ構造

  • manifest_writer: マニフェスト生成の状態（ファイルパス、ファイル記述子、バッファ、チェックサムコンテキスト）を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には pg_malloc がエラーを発生させる。
  • パス名が MAXPGPATH を超える場合、snprintf による切り詰めが発生する可能性がある。

保守担当者向けメモ

  • 本関数はマニフェストの初期化のみを行い、実際のファイル作成（open）は後続の flush_manifest 等の処理で行われる設計となっている。
  • system_identifier は JSON の System-Identifier フィールドに埋め込まれるため、正確な値が渡されていることを確認すること。
