r_standard_suffix

目的

オランダ語のステミング処理において、単語末尾の標準的な接尾辞を特定し、ルールに基づいた削除または置換を行う。

シグネチャまたは形状

static int r_standard_suffix(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（Snowball ステミング環境）
  • 出力: 成功時は 1、失敗時は 0、エラー発生時は負の整数を返す。

前提条件と不変条件

  • z が有効な Snowball 環境であること。
  • r_mark_regions によって単語内の R1, R2 領域が正しくマークされていること。
  • 内部状態（z->c, z->l, z->bra, z->ket）が適切に管理されていること。

アルゴリズムまたは振る舞い

1. **接尾辞マッチング**: find_among_b を使用し、後方から複数の接尾辞テーブル（a_3, a_4 等）を検索する。 2. **ルール適用**:

  • a_3 のマッチ結果に応じて、r_R1 の条件確認や slice_from_s による置換、r_en_ending の呼び出しを行う。
  • r_e_ending を適用し、必要に応じて削除を行う。
  • s_12（接尾辞）の存在を確認し、r_R2 領域内であれば削除する。
  • a_4 のマッチ結果に応じて、r_R2 条件下での削除、r_undouble による二重子音の解消、r_e_ending の再適用を行う。

    • 3. **状態復元**: 各ステップで m1 から m9 までのマーカーを使用し、マッチング失敗時にはカーソル位置をバックトラックする。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内では明示的な呼び出し元は特定されていません。
  • 呼び出し先: r_R1, r_R2, r_en_ending, r_e_ending, r_undouble, slice_from_s, slice_del, find_among_b, eq_s_b, out_grouping_b_U, skip_b_utf8。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_3, a_4, a_5: 接尾辞マッチング用の静的テーブル。
  • g_v_j, g_v_I, g_v: 文字グループ定義。

失敗モードと境界条件

  • z->c が境界（z->lb）を超えた場合、またはマッチング対象の文字が期待と異なる場合は lab ラベルへジャンプし、処理をスキップする。
  • slice_ 系関数が負の値を返した場合、エラーとして即座に終了する。
