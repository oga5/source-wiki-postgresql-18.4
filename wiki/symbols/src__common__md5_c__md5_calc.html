md5_calc

定義

  • 種別: function
  • 定義箇所: src/common/md5.c:153

目的

RFC 1321 に準拠した MD5 メッセージダイジェストアルゴリズムのコアとなる変換関数を実行し、64バイトのデータブロックを用いてハッシュ状態を更新します。

シグネチャまたは形状

static void md5_calc(const uint8 *b64, pg_md5_ctx *ctx)

入力と出力

  • **入力**:
  • b64: 処理対象となる64バイトのデータブロックへのポインタ。
  • ctx: ハッシュ計算の状態を保持する pg_md5_ctx 構造体へのポインタ。
  • **出力**: なし（副作用として ctx 内の内部状態 md5_sta ～ md5_std を更新）。

前提条件と不変条件

  • b64 は少なくとも64バイトの有効なメモリ領域を指している必要があります。
  • ctx は pg_md5_init によって初期化済みである必要があります。

アルゴリズムまたは振る舞い

1. **エンディアン調整**: WORDS_BIGENDIAN が定義されている場合、入力データ b64 をバイト単位で並び替え、リトルエンディアン形式の uint32 配列 X に変換します。それ以外の場合は直接キャストします。 2. **変換処理**: 4つのラウンド関数（ROUND1 ～ ROUND4）を順次適用し、内部状態 A, B, C, D を計算します。 3. **状態更新**: 計算結果を ctx の各メンバ（md5_sta ～ md5_std）に加算します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/md5.c 内の pg_md5_update および関連する内部処理から呼び出されます。
  • **呼び出し先**: ROUND1, ROUND2, ROUND3, ROUND4 マクロを呼び出します。

関連データ構造

  • pg_md5_ctx: ハッシュ計算の状態（中間値およびバッファ）を保持する構造体。

失敗モードと境界条件

  • 入力データが64バイト未満の場合、本関数を直接呼び出すとバッファオーバーランが発生する可能性があります（呼び出し側の pg_md5_update 等で適切に制御されることが前提です）。

保守担当者向けメモ

  • 本関数はパフォーマンスよりも移植性を重視した実装です。
  • ビッグエンディアン環境でのバイト順序変換は、明示的なバイト配置によって行われています。修正時は RFC 1321 の仕様との整合性に注意してください。
