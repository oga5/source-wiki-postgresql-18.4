spg_kd_picksplit

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgkdtreeproc.c:107

目的

SP-GiSTインデックスにおいて、Point 型データの挿入時に空間を再帰的に二分割し、k-d 木のノード構成を決定する。

シグネチャまたは形状

Datum spg_kd_picksplit(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgPickSplitIn 構造体（入力タプル群、現在のレベル等）を受け取る。
  • **出力**: spgPickSplitOut 構造体を介して、分割に使用するプレフィックス（float8）、ノード数（2）、および各タプルの所属ノード（0または1）を返す。

前提条件と不変条件

  • in->level が偶数の場合は x 座標、奇数の場合は y 座標でソートを行う。
  • k-d 木の性質上、allTheSame（すべての値が同一）という状態は発生しない設計である。
  • 分割境界値（中央値）と一致する座標を持つ点は、ソート順序に応じて左右いずれかのノードに分類される。

アルゴリズムまたは振る舞い

1. 入力タプルを SortedPoint 配列に格納する。 2. in->level に基づき x_cmp または y_cmp を用いてソートする。 3. 中央値（nTuples >> 1）を算出し、これをプレフィックスとして設定する。 4. nNodes を 2 に設定し、ノードラベルは使用しない（NULL）。 5. 各タプルを中央値と比較し、小さい場合はノード 0、それ以外はノード 1 に割り当てる。 6. 割り当て結果を mapTuplesToNodes に格納し、PG_RETURN_VOID() で終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述なし。
  • **呼び出し先**: palloc（メモリ確保）、qsort（ソート）、x_cmp / y_cmp（比較関数）、DatumGetPointP / PointPGetDatum（データ変換）。

関連データ構造

  • spgPickSplitIn, spgPickSplitOut: SP-GiSTの分割処理用インターフェース構造体。
  • SortedPoint: ソート処理のためにタプルとインデックスを保持する補助構造体。

失敗モードと境界条件

  • nTuples が 0 の場合の挙動については本ソース内では明示されていない。
  • 境界値（中央値）と等しい値を持つデータは、ソート順序に依存して左右いずれかに振り分けられるため、検索側（spg_kd_inner_consistent）が両方のノードを探索する必要がある。

保守担当者向けメモ

  • 本関数は allTheSame ロジックをトリガーしない設計である。
  • 検索時の枝刈りロジックと整合性を保つため、分割境界の扱いは spg_kd_inner_consistent の実装とセットで考慮すること。
