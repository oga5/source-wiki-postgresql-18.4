logstreamer_param

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:533

目的

WAL（Write-Ahead Log）ストリーミング処理を実行するプロセスまたはスレッドに対し、必要な接続情報およびストリーミングパラメータを渡すためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	PGconn	   *bgconn;
	XLogRecPtr	startptr;
	char		xlog[MAXPGPATH];
	char	   *sysidentifier;
	int			timeline;
	pg_compress_algorithm wal_compress_algorithm;
	int			wal_compress_level;
} logstreamer_param;

入力と出力

本構造体は、LogStreamerMain 関数への入力パラメータとして使用されます。出力としての役割は持ちません。

前提条件と不変条件

  • bgconn は有効な PostgreSQL 接続である必要があります。
  • xlog は、WAL データの保存先となるディレクトリまたは tar ファイルのパスを保持します。

アルゴリズムまたは振る舞い

pg_basebackup において、WAL ストリーミングを開始する際にこの構造体がメモリ確保（pg_malloc0）され、各フィールドが初期化されます。その後、この構造体へのポインタが LogStreamerMain に渡され、バックグラウンドでの WAL 受信処理の制御に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_basebackup.c 内の WAL ストリーミング開始ロジック（StartLogStreamer 関連箇所）。
  • 呼び出し先: LogStreamerMain(logstreamer_param *param) が本構造体を受け取ります。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • XLogRecPtr: WAL の位置を示すポインタ型。
  • pg_compress_algorithm: WAL 圧縮アルゴリズムの列挙型。

失敗モードと境界条件

  • pg_malloc0 によるメモリ確保失敗時にプログラムが終了する可能性があります。
  • xlog フィールドは MAXPGPATH の制限を受けるため、パス長がこれを超えないことが前提となります。

保守担当者向けメモ

  • 本構造体は LogStreamerMain の引数としてのみ使用されるため、フィールドを追加・変更する場合は、LogStreamerMain 側の処理も併せて更新してください。
  • xlog フィールドの解釈（ディレクトリかファイルか）は、実行時のモードに依存するため、変更時には walmethods.c 側の実装との整合性に注意してください。
