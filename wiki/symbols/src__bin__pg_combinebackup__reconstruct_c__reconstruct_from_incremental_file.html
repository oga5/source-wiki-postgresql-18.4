reconstruct_from_incremental_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/reconstruct.c:87

目的

増分バックアップファイルと過去のバックアップチェーンを統合し、特定のデータファイルの完全な状態を再構築する。

シグネチャまたは形状

void reconstruct_from_incremental_file(
    char *input_filename, char *output_filename, char *relative_path,
    char *bare_file_name, int n_prior_backups, char **prior_backup_dirs,
    manifest_data **manifests, char *manifest_path,
    pg_checksum_type checksum_type, int *checksum_length,
    uint8 **checksum_payload, CopyMethod copy_method,
    bool debug, bool dry_run
);

入力と出力

  • **入力**: 最新の増分ファイルパス、出力先パス、バックアップチェーン情報（ディレクトリリスト、マニフェスト）、チェックサム設定、コピー手法、デバッグ/ドライランフラグ。
  • **出力**: なし（副作用としてファイルシステム上に再構築されたファイルを作成する）。

前提条件と不変条件

  • relative_path は空であってはならず、末尾が / である必要がある。
  • 再構築される各ブロックは、最新の増分ファイルまたは過去のバックアップチェーンのいずれかに存在しなければならない。
  • WAL に記録されていないブロックは、再構築時にゼロ埋めされる。

アルゴリズムまたは振る舞い

1. 最新の増分ファイルからヘッダーを読み込み、再構築後のファイルサイズ（ブロック数）を特定する。 2. sourcemap と offsetmap を初期化し、各ブロックの供給元を記録する計画を立てる。 3. 最新の増分ファイルに含まれるブロックを sourcemap に登録する。 4. バックアップチェーンを遡り、まだソースが特定されていないブロックを過去のバックアップから順次埋める。 5. 全ブロックが単一の過去のバックアップから取得可能か判定し、最適化（フルコピー）の可能性を評価する。 6. 計画に基づき、ブロック単位の読み込み・書き込み、またはファイル全体のコピーを実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/pg_combinebackup.c
  • **呼び出し先**: make_incremental_rfile, find_reconstructed_block_length, make_rfile, pg_malloc0, fstat, pg_fatal 等。

関連データ構造

  • rfile: 再構築に使用するソースファイルのメタデータとファイル記述子を保持。
  • sourcemap / offsetmap: 出力ファイルの各ブロックの供給元とオフセットを定義する配列。

失敗モードと境界条件

  • fstat 失敗時に pg_fatal を呼び出しプロセスを終了する。
  • 過去のバックアップにファイルが存在しない場合、増分ファイルを探すフォールバック処理を行う。
  • ゼロ埋めブロックの検証は不可能であり、サーバー側の未初期化ブロックの挙動に依存する。

保守担当者向けメモ

  • full_copy_possible フラグは、最新の増分ファイルにブロックが含まれる時点で false に設定される。
  • dry_run が有効な場合、実際の書き込み処理はスキップされる。
  • copy_file_range 等の効率的なコピー手法が利用可能な場合は、copy_method に基づき最適化される。
