sqlda_t

目的

ECPG（Embedded SQL for C）における Informix 互換モードでの動的 SQL 実行時に、SQL 文の入力パラメータや出力結果を保持・伝達するための記述子領域（SQL Descriptor Area）を提供します。

シグネチャまたは形状

sqlda_compat 構造体の別名です。具体的なメンバ構成は _ECPG_INFORMIX_H マクロの定義有無に依存し、sqlda-compat.h または sqlda-native.h に定義されます。

入力と出力

  • **入力**: EXECUTE 文の USING DESCRIPTOR 句において、バインド変数として使用されます。
  • **出力**: FETCH 文の USING DESCRIPTOR 句において、データベースからの取得結果を格納するために使用されます。

前提条件と不変条件

  • 使用前に malloc 等でメモリを確保し、memset 等でゼロ初期化することが一般的です。
  • 記述子の構造は、実行する SQL 文の列数や型と整合している必要があります。

アルゴリズムまたは振る舞い

動的 SQL の実行フローにおいて、以下の役割を担います。 1. **構築**: malloc によりメモリを確保し、必要に応じて各列の属性を設定します。 2. **バインド**: ECPGdo() 関数を通じて、SQL 文の実行時に記述子として渡されます。 3. **解放**: 使用後は free() を呼び出し、メモリを適切に解放する必要があります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/compat_informix-sqlda.c 内の main 関数において、inp_sqlda および outp_sqlda として動的に割り当てられ、SQL 実行の仲介に使用されます。
  • **呼び出し先**: dump_sqlda() 関数が引数として受け取り、記述子内の情報を標準出力へダンプします。

関連データ構造

  • sqlvar_t: sqlda_t 内で各列のデータ型、データポインタ、インジケータを保持するために使用される構造体。

失敗モードと境界条件

  • メモリ割り当て失敗: malloc が失敗した場合、記述子へのアクセスによりセグメンテーション違反が発生する可能性があります。
  • 構造不整合: SQL 文の列数と sqlda_t が保持する列定義が一致しない場合、実行時エラーが発生します。
