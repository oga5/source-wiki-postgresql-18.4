pg_try_advisory_lock_int8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:693

目的

ユーザーが指定した64ビット整数キーを使用して、セッションスコープのアドバイザリーロックを非ブロック的に取得します。

シグネチャまたは形状

Datum pg_try_advisory_lock_int8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT64(0) により、ロック識別子として使用する int64 型のキーを受け取ります。
  • **出力**: ロックが正常に取得できた場合は true、既に他のセッションによってロックされており取得できなかった場合は false を返します。

前提条件と不変条件

  • ロックのスコープは現在のセッションに限定されます。
  • ロックキーは MyDatabaseId と組み合わされて管理されるため、データベースを跨いだロックは行えません。

アルゴリズムまたは振る舞い

1. SET_LOCKTAG_INT64 マクロを使用して、入力された int64 キーを LOCKTAG 構造体に変換します。 2. LockAcquire 関数を呼び出し、以下の引数を渡します。

  • tag: 変換したロックタグ。
  • ExclusiveLock: 排他ロックモードを指定。
  • true: dontWait 引数に true を指定し、ロックが即座に取得できない場合に待機せず制御を戻すようにします。
  • true: report_prio 引数に true を指定します。

    • 3. LockAcquire の戻り値が LOCKACQUIRE_NOT_AVAIL でないことを確認し、その結果（ブール値）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます（具体的な呼び出し元コードの詳細は提供されていません）。
  • **呼び出し先**:
  • LockAcquire (PostgreSQLロックマネージャのコア機能)

関連データ構造

  • LOCKTAG: ロック対象を識別するための構造体。
  • LockAcquireResult: LockAcquire 関数の戻り値型。

失敗モードと境界条件

  • ロックが既に他のプロセスによって保持されている場合、待機せずに false を返します。
  • メモリ不足や内部的なロックマネージャの制限により LockAcquire が失敗する可能性があります。

保守担当者向けメモ

  • 本関数は非ブロック的なロック取得を行うため、デッドロックを回避したいアプリケーションロジックでの利用に適しています。
  • 述語ロック（Predicate Locks）とは管理体系が異なるため、本関数は通常のロックマネージャを介して処理されます。
