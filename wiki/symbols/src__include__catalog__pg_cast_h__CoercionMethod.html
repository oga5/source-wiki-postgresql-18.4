CoercionMethod

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/pg_cast.h:87

目的

データ型間のキャスト（型変換）を実行する際の手法を識別するために使用されます。

シグネチャまたは形状

typedef enum CoercionMethod
{
    COERCION_METHOD_FUNCTION = 'f',
    COERCION_METHOD_BINARY   = 'b',
    COERCION_METHOD_INOUT    = 'i',
} CoercionMethod;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: char 型の値（'f', 'b', 'i'）としてシステムカタログ pg_cast に格納されます。

前提条件と不変条件

  • pg_cast カタログの castmethod 列に対応します。
  • COERCION_METHOD_BINARY が選択された場合、通常 castfunc は 0 となります。

アルゴリズムまたは振る舞い

  • COERCION_METHOD_FUNCTION: 変換関数を呼び出して型変換を行います。
  • COERCION_METHOD_BINARY: 変換関数を介さず、バイナリ互換性に基づき直接変換します。
  • COERCION_METHOD_INOUT: 型の入出力関数（I/O関数）を使用して変換します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは特定の関数呼び出し関係は特定できません。
  • 呼び出し先: なし（列挙型定義のため）。

関連データ構造

  • FormData_pg_cast: pg_cast テーブルの行構造。本列挙型の値が castmethod フィールドに保持されます。

失敗モードと境界条件

  • データベース内では char 型として保存されるため、無効な文字が代入された場合の整合性はカタログ操作ロジック（src/backend/catalog/pg_cast.c）に依存します。

保守担当者向けメモ

  • Catalog.pm が本ファイルを解析してカタログ初期化スクリプトを生成するため、列挙子の追加や変更には注意が必要です。
  • クライアントコード向けには列挙型として公開されていますが、ストレージ上は単一の文字として扱われます。
