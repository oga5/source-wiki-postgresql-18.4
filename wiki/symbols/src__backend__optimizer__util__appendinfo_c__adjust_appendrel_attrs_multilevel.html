adjust_appendrel_attrs_multilevel

定義

  • src/backend/optimizer/util/appendinfo.c:544

目的

階層的な継承関係（多段階の親子関係）を持つリレーション間において、式ツリー内の属性参照を親リレーションから子リレーションへ再帰的に変換する。

シグネチャまたは形状

Node *adjust_appendrel_attrs_multilevel(PlannerInfo *root, Node *node,
                                        RelOptInfo *childrel,
                                        RelOptInfo *parentrel)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • node: 変換対象の式ツリー（Node）。
  • childrel: 変換先となる子リレーションの RelOptInfo。
  • parentrel: 変換元となる親リレーションの RelOptInfo。
  • **出力**:
  • 変換後の式ツリー（Node）。

前提条件と不変条件

  • childrel が parentrel の直接的または間接的な子リレーションであること。
  • childrel->parent が存在しない（かつ childrel != parentrel である）場合、エラーが発生する。

アルゴリズムまたは振る舞い

1. **再帰的走査**: childrel->parent が parentrel と一致しない場合、childrel->parent を新たな子リレーションとして自身を再帰的に呼び出し、階層を遡る。 2. **変換情報の取得**: 現在の childrel に対応する AppendRelInfo 配列を find_appinfos_by_relids により取得する。 3. **属性変換**: adjust_appendrel_attrs を呼び出し、取得した AppendRelInfo を用いて式ツリー内の属性参照を変換する。 4. **メモリ解放**: 取得した appinfos 配列を pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/equivclass.c
  • src/backend/optimizer/plan/planner.c
  • **呼び出し先**:
  • adjust_appendrel_attrs_multilevel（再帰呼び出し）
  • find_appinfos_by_relids
  • adjust_appendrel_attrs

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • RelOptInfo: リレーションの最適化情報。
  • AppendRelInfo: 親子間の属性変換マッピングを保持する構造体。

失敗モードと境界条件

  • childrel が parentrel の子孫関係にない場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は、単一レベルの変換を行う adjust_appendrel_attrs を多段階の継承ツリーに対応させるためのラッパーとして機能する。
  • 変換対象の式ツリーが複雑な場合、再帰の深さに応じてスタック消費が発生する可能性がある点に留意すること。
