IsCatalogRelationOid

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/catalog.c:120

目的

指定されたOIDがシステムカタログ（システムテーブル、そのインデックス、TOASTテーブル、TOASTインデックスを含む）に該当するかを高速に判定する。

シグネチャまたは形状

bool IsCatalogRelationOid(Oid relid)

入力と出力

  • **入力**: relid (判定対象のOID)
  • **出力**: true（システムカタログである場合）、false（それ以外の場合）

前提条件と不変条件

  • information_schema 内のリレーションは、システムカタログとはみなされない（ユーザー定義テーブルと同様に扱われる）。
  • OIDのラップアラウンド発生時、システムカタログのOID範囲はスキップされるため、この判定ロジックは信頼性が保証されている。

アルゴリズムまたは振る舞い

  • 引数 relid が FirstUnpinnedObjectId 未満であるかを比較する。
  • カタログアクセスを一切行わないため、ロック取得中や再帰的な呼び出しにおいても安全に使用可能。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/dml.c
  • src/backend/access/common/tupdesc.c
  • src/backend/access/transam/varsup.c
  • src/backend/catalog/catalog.c（内部の他関数から参照）
  • src/backend/catalog/pg_publication.c
  • src/backend/commands/indexcmds.c
  • **呼び出し先**:
  • 本関数は定数比較のみを行うため、他の関数を呼び出さない。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子型。

失敗モードと境界条件

  • information_schema 内のオブジェクトに対しては false を返す。
  • 比較は符号なし整数として行われる（varsup.c の記述に基づく）。

保守担当者向けメモ

  • 本関数は IsSystemRelation 等の判定ロジックの基礎となるため、変更時はシステム全体のカタログ判定に影響を与える点に注意すること。
  • 処理速度が優先される箇所で使用されるため、ロジックの複雑化は避けること。
