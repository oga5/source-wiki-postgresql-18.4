pg_malloc_internal

定義

  • 種別: function
  • 定義箇所: src/common/fe_memutils.c:28

目的

PostgreSQLフロントエンド環境において、標準の malloc をラップし、メモリ確保の失敗時における一貫したエラーハンドリングと、移植性を考慮したメモリ確保処理を提供します。

シグネチャまたは形状

static inline void *pg_malloc_internal(size_t size, int flags)

入力と出力

  • **入力**:
  • size: 確保するメモリサイズ（バイト単位）。
  • flags: 挙動を制御するビットフラグ（MCXT_ALLOC_NO_OOM, MCXT_ALLOC_ZERO）。
  • **出力**:
  • 確保されたメモリ領域へのポインタ。確保失敗時は flags に応じてプロセスを終了するか、NULL を返します。

前提条件と不変条件

  • size が 0 の場合、移植性の観点から内部的に size を 1 として扱います。
  • MCXT_ALLOC_NO_OOM フラグが指定されていない場合、メモリ確保失敗は致命的なエラーとみなされ、プロセスは exit(EXIT_FAILURE) で終了します。

アルゴリズムまたは振る舞い

1. size が 0 の場合、size を 1 に補正します。 2. malloc(size) を実行します。 3. 確保に失敗した場合:

  • MCXT_ALLOC_NO_OOM フラグが未設定であれば、stderr にエラーメッセージを出力し、exit(EXIT_FAILURE) を呼び出します。
  • MCXT_ALLOC_NO_OOM フラグが設定されていれば、NULL を返します。

    • 4. 確保に成功した場合:

  • MCXT_ALLOC_ZERO フラグが設定されていれば、MemSet を使用して確保領域を 0 で初期化します。

    • 5. 確保したメモリ領域へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の pg_malloc, pg_malloc0, pg_malloc_extended およびそれらに関連するラッパー関数群から呼び出されます。
  • **呼び出し先**: 標準ライブラリの malloc、およびメモリ初期化のための MemSet を呼び出します。

関連データ構造

  • MCXT_ALLOC_NO_OOM: メモリ不足時に終了させないためのフラグ。
  • MCXT_ALLOC_ZERO: 確保後にメモリをゼロクリアするためのフラグ。

失敗モードと境界条件

  • malloc が NULL を返す場合、メモリ不足（OOM）として扱われます。
  • size が 0 の入力は、malloc(1) として処理されます。

保守担当者向けメモ

  • 本関数は FRONTEND マクロが定義されている環境でのみ使用可能です。
  • バックエンドのメモリ管理との互換性を維持するため、サイズ計算には add_size や mul_size を併用し、オーバーフローを未然に防ぐ設計となっています。
