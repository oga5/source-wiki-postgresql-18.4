ConditionVariablePrepareToSleep

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/condition_variable.c:55

目的

プロセスを特定の条件変数（ConditionVariable）の待機リストに登録し、待機状態へ移行するための準備を行う。

シグネチャまたは形状

void ConditionVariablePrepareToSleep(ConditionVariable *cv)

入力と出力

  • **入力**: cv - 待機対象となる ConditionVariable 構造体へのポインタ。
  • **出力**: なし（void）。副作用として、スレッドローカルな cv_sleep_target が更新され、対象の条件変数の待機リストに現在のプロセスが追加される。

前提条件と不変条件

  • 呼び出し元は、条件変数の待機リストを保護するスピンロックを保持していないこと。
  • 1つのプロセスは一度に1つの条件変数に対してのみ待機準備が可能である。

アルゴリズムまたは振る舞い

1. **既存の待機状態の解消**: 静的変数 cv_sleep_target が既に設定されている場合、ConditionVariableCancelSleep() を呼び出して既存の待機準備をキャンセルする。 2. **ターゲットの記録**: 引数 cv を cv_sleep_target に代入する。 3. **待機リストへの追加**:

  • cv->mutex スピンロックを取得する。
  • proclist_push_tail を使用し、現在のプロセス（MyProcNumber）を cv->wakeup リストの末尾に追加する。
  • cv->mutex スピンロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/checkpointer.c
  • src/backend/replication/logical/origin.c
  • src/backend/replication/slot.c
  • src/backend/replication/walreceiverfuncs.c
  • src/backend/replication/walsender.c
  • src/backend/storage/aio/aio.c
  • src/backend/storage/buffer/bufmgr.c
  • **呼び出し先**:
  • ConditionVariableCancelSleep（条件による）
  • SpinLockAcquire / SpinLockRelease
  • proclist_push_tail

関連データ構造

  • ConditionVariable: 待機リストと保護用スピンロックを保持する構造体。
  • PGPROC: プロセス情報を保持し、cvWaitLink を介して待機リストを構成する。
  • cv_sleep_target: 待機中の条件変数を追跡するスレッドローカルな静的変数。

失敗モードと境界条件

  • 既に別の条件変数に対して待機準備がなされている場合、本関数は自動的に古い準備をキャンセルしてから新しい準備を行う。これは、各プロセスが保持できる cvWaitLink が1つであるという制約に基づく。

保守担当者向けメモ

  • 本関数は ConditionVariableSleep の呼び出し前に実行されることを想定している。
  • 待機リストへの追加はスピンロックで保護されているため、共有メモリ上での競合に対して安全である。
