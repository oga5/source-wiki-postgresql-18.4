simple_string_list_append

定義

  • 種別: function
  • 定義箇所: src/fe_utils/simple_list.c:62

目的

SimpleStringList 構造体で管理される単方向連結リストの末尾に、新しい文字列要素を追加します。

シグネチャまたは形状

void simple_string_list_append(SimpleStringList *list, const char *val)

入力と出力

  • list: 操作対象の SimpleStringList 構造体へのポインタ。
  • val: リストに追加する文字列（const char *）。
  • 戻り値: なし。

前提条件と不変条件

  • list は初期化済みである必要があります。
  • val は NULL であってはなりません。
  • リスト内の各セルは、val を保持するために offsetof(SimpleStringListCell, val) + strlen(val) + 1 のサイズで動的に確保されます。

アルゴリズムまたは振る舞い

1. pg_malloc を使用して、文字列の長さに応じたメモリを確保し SimpleStringListCell を作成します。 2. 新しいセルの next を NULL に、touched フラグを false に初期化します。 3. strcpy を使用して、引数 val をセルの末尾（FLEXIBLE_ARRAY_MEMBER 領域）にコピーします。 4. リストの tail が存在する場合は、現在の末尾の next を新しいセルに接続します。 5. リストが空の場合は、head を新しいセルに設定します。 6. リストの tail を新しいセルに更新します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_dump/pg_dump.c
  • 呼び出し先:
  • pg_malloc (メモリ確保)
  • strlen, strcpy (文字列処理)

関連データ構造

  • SimpleStringList: リストの head と tail を保持する構造体。
  • SimpleStringListCell: リストの各要素を構成する構造体。

失敗モードと境界条件

  • メモリ不足時: pg_malloc が失敗した場合、プロセスは終了します。
  • 空リストへの追加: list->head が NULL の場合、正しく head と tail の両方が新しいセルを指すように更新されます。

保守担当者向けメモ

  • 本関数は文字列をコピーして保持するため、呼び出し元で渡した文字列のメモリ管理を気にする必要はありません。
  • 単方向リストであるため、要素数が増加すると末尾への追加は $O(1)$ ですが、検索は $O(N)$ となります。大量の要素を扱う用途には適していません。
