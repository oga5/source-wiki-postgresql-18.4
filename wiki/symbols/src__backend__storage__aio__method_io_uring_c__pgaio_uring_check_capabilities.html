pgaio_uring_check_capabilities

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/method_io_uring.c:140

目的

Linux カーネルが io_uring_queue_init_mem をサポートしているかを実行時に判定し、io_uring インスタンスのメモリマッピングを最適化するための情報を収集する。

シグネチャまたは形状

static void pgaio_uring_check_capabilities(void)

入力と出力

  • 入力: なし（グローバルな pgaio_uring_caps および io_max_concurrency を参照）
  • 出力: なし（副作用として pgaio_uring_caps の状態を更新）

前提条件と不変条件

  • pgaio_uring_caps.checked が true の場合、関数は即座にリターンする。
  • HAVE_IO_URING_QUEUE_INIT_MEM および IORING_SETUP_NO_MMAP が定義されている環境でのみ、カーネルのプローブ処理が実行される。

アルゴリズムまたは振る舞い

1. pgaio_uring_caps.checked を確認し、既にチェック済みであれば終了する。 2. カーネルサポートのプローブが必要な場合、1MB の一時メモリを mmap で確保する。 3. io_uring_queue_init_mem を呼び出し、カーネルがユーザー提供メモリによるリング作成をサポートしているか試行する。 4. 成功した場合、必要なメモリサイズを pgaio_uring_caps.mem_init_size に記録し、テスト用のリングを破棄する。 5. 失敗した場合はエラーログを出力し、サポート不可として扱う。 6. 最後に pgaio_uring_caps.checked を true に設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/aio/method_io_uring.c 内の初期化処理（pgaio_uring_shmem_init 等から呼び出されることが示唆されている）。
  • 呼び出し先:
  • mmap, munmap, sysconf (システムコール)
  • io_uring_queue_init_mem, io_uring_queue_exit (liburing API)
  • elog (PostgreSQL エラーログ出力)

関連データ構造

  • pgaio_uring_caps: サポート状況とメモリ要件を保持するグローバル構造体。
  • struct io_uring: liburing が使用するリング管理構造体。

失敗モードと境界条件

  • mmap が失敗した場合、elog(ERROR) によりプロセスが中断される。
  • io_uring_queue_init_mem が失敗した場合、errno をログに出力し、機能が利用できないものとして処理を継続する。
  • ring_size がページサイズの倍数でない場合、sysconf(_SC_PAGESIZE) を用いて調整される。

保守担当者向けメモ

  • 本関数は io_uring のメモリマッピング効率化のために重要である。
  • io_uring_queue_init_mem のサポート状況はカーネルバージョンに依存するため、将来的なカーネル更新時に挙動が変わる可能性がある。
  • ring_size の 1MB という値は、io_max_concurrency の範囲内であれば十分であるという前提に基づいている。
