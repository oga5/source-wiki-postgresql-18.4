commitTable

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-declare.c:581

目的

ECPG テストスイートにおいて、複数のデータベース接続（con1 および con2）に対して明示的に COMMIT トランザクションを発行し、リソースの確定を行う。

シグネチャまたは形状

void commitTable(void)

入力と出力

  • 入力: なし。
  • 出力: なし（副作用としてデータベースの状態を更新）。

前提条件と不変条件

  • con1 および con2 という名称のデータベース接続が既に確立されていること。
  • 各 ECPGtrans 呼び出し後、sqlca.sqlcode を通じてエラーチェックが行われる。

アルゴリズムまたは振る舞い

1. con1 に対して COMMIT を実行する。 2. con1 の実行結果（sqlca.sqlcode）が負の値（エラー）である場合、sqlprint() を呼び出してエラー情報を出力する。 3. con2 に対して COMMIT を実行する。 4. con2 の実行結果（sqlca.sqlcode）が負の値（エラー）である場合、sqlprint() を呼び出してエラー情報を出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/sql-declare.c 内のテスト実行フロー（182行目および199行目）。
  • 呼び出し先:
  • ECPGtrans: ECPG ライブラリのトランザクション制御関数。
  • sqlprint: エラー発生時に標準エラー出力へ情報を書き出す関数。

関連データ構造

  • sqlca: ECPG の通信領域構造体。エラーコードの判定に使用。

失敗モードと境界条件

  • データベース接続が確立されていない場合、ECPGtrans はエラーを返し、sqlca.sqlcode が負の値となる。
  • トランザクションが既に終了している、あるいは無効な接続名が指定された場合も同様にエラーとして扱われる。

保守担当者向けメモ

  • 本関数は sql-declare.c のテストケース内で、一連の SQL 操作を確定させるために使用される。
  • sqlca.sqlcode < 0 のチェックは、ECPG における標準的なエラーハンドリングパターンである。
