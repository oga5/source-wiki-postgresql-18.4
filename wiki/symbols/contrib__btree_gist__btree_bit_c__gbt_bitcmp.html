gbt_bitcmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:65

目的

BIT 型および VARBIT 型の値を比較し、その大小関係を整数値で返すための比較関数です。

シグネチャまたは形状

int32 gbt_bitcmp(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のビット列データへのポインタ。
  • collation: 照合順序（本関数では未使用）。
  • flinfo: 関数呼び出し情報（本関数では未使用）。
  • **出力**:
  • a が b より小さい場合は負の整数、等しい場合は 0、大きい場合は正の整数を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効なビット列データ（bytea 形式と互換性のある構造）を指している必要があります。

アルゴリズムまたは振る舞い

1. DirectFunctionCall2 を使用して、PostgreSQL 標準の byteacmp 関数を呼び出します。 2. byteacmp の結果（Datum 型）を DatumGetInt32 で int32 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_bit.c 内の tinfo 構造体（gbtree_vinfo）の初期化時に参照されています（118行目）。
  • **呼び出し先**: byteacmp（PostgreSQL 標準のバイト列比較関数）。

関連データ構造

  • gbtree_vinfo: btree_gist がインデックス操作に使用する設定テーブル。本関数はこのテーブルの一部として登録されます。

失敗モードと境界条件

  • byteacmp の仕様に依存します。入力が NULL の場合や不正なメモリ領域を指している場合は、基底関数側でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は byteacmp をラップしているため、ビット列の比較ロジックに変更が必要な場合は、標準の byteacmp の挙動を確認してください。
  • collation および flinfo 引数は現在の実装では使用されていませんが、シグネチャの互換性維持のために保持されています。
