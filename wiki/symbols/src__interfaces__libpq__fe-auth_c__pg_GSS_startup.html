pg_GSS_startup

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth.c:168

目的

GSSAPI認証プロセスの初期化を行い、認証コンテキストの準備と最初の認証パケットの処理を開始します。

シグネチャまたは形状

static int pg_GSS_startup(PGconn *conn, int payloadlen)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体へのポインタ。
  • payloadlen: サーバーから受信した認証ペイロードの長さ。
  • **出力**:
  • 成功時は pg_GSS_continue の戻り値（通常は STATUS_OK 等）、失敗時は STATUS_ERROR を返します。

前提条件と不変条件

  • conn->connhost に有効なホスト名が設定されている必要があります。
  • 既に GSSAPI 認証が開始されている（conn->gctx が非 NULL）場合、重複要求としてエラーになります。

アルゴリズムまたは振る舞い

1. ホスト名が指定されているか検証し、空であればエラーを返します。 2. conn->gctx が既に存在する場合は、重複した認証要求としてエラーを返します。 3. pg_GSS_load_servicename を呼び出し、サービス名をロードします。失敗した場合はその戻り値を返します。 4. conn->gctx を GSS_C_NO_CONTEXT に初期化します。 5. pg_GSS_continue を呼び出し、認証の継続処理へ移行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth.c 内の認証ディスパッチャー（サーバーからの認証要求ハンドラ）。
  • **呼び出し先**:
  • pg_GSS_load_servicename
  • pg_GSS_continue
  • libpq_append_conn_error

関連データ構造

  • PGconn: 接続状態を管理する構造体。gctx（GSSAPI コンテキスト）および connhost を保持します。

失敗モードと境界条件

  • ホスト名が未指定の場合: STATUS_ERROR を返し、エラーメッセージを記録します。
  • GSSAPI コンテキストが既に存在する場合: STATUS_ERROR を返し、エラーメッセージを記録します。
  • サービス名のロードに失敗した場合: pg_GSS_load_servicename のエラーコードを返します。

保守担当者向けメモ

  • 本関数は GSSAPI 認証の開始点です。認証フローの非同期的な中断・再開については pg_GSS_continue の実装を確認してください。
