Zstd_eof

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_zstd.c:496

目的

Zstandard 圧縮されたファイルハンドルが終端（EOF）に達しているかを判定します。

シグネチャまたは形状

static bool Zstd_eof(CompressFileHandle *CFH)

入力と出力

  • **入力**: CompressFileHandle *CFH - 圧縮処理のコンテキストを保持するハンドル。
  • **出力**: bool - 内部のファイルポインタが EOF に達していれば true、そうでなければ false を返します。

前提条件と不変条件

  • CFH は有効な CompressFileHandle 構造体を指している必要があります。
  • CFH->private_data には、初期化済みの ZstdCompressorState 構造体が格納されている必要があります。

アルゴリズムまたは振る舞い

1. CFH->private_data を ZstdCompressorState 型にキャストして取得します。 2. 内部保持されているファイルポインタ zstdcs->fp に対して標準ライブラリの feof() を呼び出し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_zstd.c 内で CFH->eof_func に代入されており、圧縮抽象化層を通じて間接的に呼び出されます。
  • **呼び出し先**: 標準ライブラリ関数 feof() を呼び出します。

関連データ構造

  • CompressFileHandle: 圧縮処理の抽象化インターフェース。
  • ZstdCompressorState: Zstd 圧縮・解凍のコンテキストを保持する構造体。

失敗モードと境界条件

  • CFH または CFH->private_data が NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は CompressFileHandle のコールバック関数として登録されることを前提としています。
  • feof() の戻り値に直接依存しているため、ファイルストリームの状態管理が正しく行われていることが前提となります。
