fe_oauth_state

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/fe-auth-oauth.h:35

目的

SASL OAUTHBEARER 認証メカニズムにおける、クライアント側の認証セッションコンテキストを保持する構造体です。

シグネチャまたは形状

typedef struct
{
	enum fe_oauth_step step;
	PGconn	   *conn;
	void	   *async_ctx;
	void	   *builtin_flow;
} fe_oauth_state;

入力と出力

  • 入力: 認証プロセス開始時に初期化され、PGconn への参照を保持します。
  • 出力: 認証の進捗に応じて step が更新され、非同期処理の結果が async_ctx に格納されます。

前提条件と不変条件

  • conn は有効な PGconn 構造体を指している必要があります。
  • async_ctx は認証成功または失敗時に適切に解放される必要があります。

アルゴリズムまたは振る舞い

1. oauth_init により構造体が生成され、初期状態 FE_OAUTH_INIT が設定されます。 2. 認証の進行に伴い、step が FE_OAUTH_BEARER_SENT 等へ遷移します。 3. async_ctx を介して外部の OAuth フロー（ビルトインまたはプラグイン）と非同期に通信を行います。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-auth-oauth.c 内の認証関数群、および src/interfaces/libpq-oauth/oauth-curl.c の OAuth 処理ルーチン。
  • 呼び出し先: 構造体自体はデータ保持用であり、具体的な処理は fe-auth-oauth.c の関数（use_builtin_flow 等）が本構造体を参照して実行します。

関連データ構造

  • fe_oauth_step: 認証プロセスの段階を管理する列挙型。
  • PGconn: libpq の接続状態を保持する構造体。

失敗モードと境界条件

  • async_ctx の解放漏れによるメモリリークが発生する可能性があるため、oauth_free との整合性が必須です。
  • 外部プラグイン利用時の非同期通信失敗は、step のエラー状態への遷移を通じて管理されます。

保守担当者向けメモ

  • 本構造体は外部公開される可能性があるため、変更時は ABI 互換性に注意してください。
  • oauth_unsafe_debugging_enabled() は開発用であり、本番環境での利用には注意が必要です。
