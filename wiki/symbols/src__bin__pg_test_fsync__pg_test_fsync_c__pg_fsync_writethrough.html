pg_fsync_writethrough

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:612

目的

プラットフォーム固有の「ライトスルー（Write-through）」同期操作をファイル記述子に対して実行し、データの物理的な書き込みを保証する。

シグネチャまたは形状

int pg_fsync_writethrough(int fd);

入力と出力

  • **入力**: fd (同期対象のファイル記述子)
  • **出力**: 成功時は 0、失敗時は -1 を返す。

前提条件と不変条件

  • F_FULLFSYNC マクロが定義されている環境（主に macOS 等）でのみ、実際の fcntl 操作が実行される。
  • F_FULLFSYNC が定義されていない環境では、常に errno に ENOSYS が設定され、-1 が返される。

アルゴリズムまたは振る舞い

1. F_FULLFSYNC が定義されている場合、fcntl(fd, F_FULLFSYNC, 0) を呼び出す。 2. fcntl の戻り値が -1 であれば、エラーとして -1 を返す。 3. F_FULLFSYNC が定義されていない場合、errno を ENOSYS に設定し、-1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c
  • src/backend/storage/file/fd.c
  • src/bin/pg_test_fsync/pg_test_fsync.c
  • **呼び出し先**:
  • fcntl (システムコール)

関連データ構造

  • なし

失敗モードと境界条件

  • fcntl が失敗した場合、-1 を返す。
  • F_FULLFSYNC がサポートされていないプラットフォームでは、呼び出し自体が機能せず、常に ENOSYS エラーとなる。

保守担当者向けメモ

  • 本関数はプラットフォーム依存の同期手法を抽象化するラッパーである。
  • src/include/storage/fd.h に外部宣言が存在する。
  • pg_test_fsync ツール内では、この関数が利用可能かどうかを検証する目的でも使用される。
