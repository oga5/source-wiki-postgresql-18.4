pq_sendint

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/libpq/pqformat.h:170

目的

指定された整数値をネットワークバイトオーダー（ビッグエンディアン）に変換し、通信バッファ StringInfo に追加します。バイトサイズを動的に指定可能な汎用インターフェースを提供します。

シグネチャまたは形状

static inline void pq_sendint(StringInfo buf, uint32 i, int b)

入力と出力

  • **入力**:
  • buf: 送信データを保持する StringInfo 構造体へのポインタ。
  • i: 書き込む整数値（uint32）。
  • b: 書き込むバイト数（1, 2, または 4）。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • buf は初期化済みである必要があります。
  • b に指定できる値は 1, 2, 4 のいずれかです。それ以外の値が渡された場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. 引数 b の値に基づき、以下のいずれかの関数を呼び出します。

  • b == 1: pq_sendint8(buf, (uint8) i)
  • b == 2: pq_sendint16(buf, (uint16) i)
  • b == 4: pq_sendint32(buf, (uint32) i)

    • 2. b が 1, 2, 4 以外の場合は elog(ERROR) を呼び出し、処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/pqformat.c:36
  • src/backend/replication/logical/proto.c:839
  • src/backend/utils/adt/varlena.c:5566
  • **呼び出し先**:
  • pq_sendint8
  • pq_sendint16
  • pq_sendint32
  • elog

関連データ構造

  • StringInfo: 通信データのバッファを管理する構造体。

失敗モードと境界条件

  • b に 1, 2, 4 以外の値が指定された場合、elog(ERROR) によりエラーがスローされます。
  • バッファの容量が不足している場合、内部で呼び出される pq_sendintN 系関数がバッファの拡張を試みます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • 整数サイズが固定されている場合は、オーバーヘッドを避けるために直接 pq_sendint8/16/32 を使用することが推奨されます。
