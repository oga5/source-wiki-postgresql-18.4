update_leaves

定義

  • 種別: function
  • 定義箇所: contrib/fuzzystrmatch/daitch_mokotoff.c:331

目的

Daitch-Mokotoff Soundex アルゴリズムの処理過程において、現在の音韻ノードの状態を更新し、次の文字に対する音韻コードの分岐を生成する。

シグネチャまたは形状

static void update_leaves(dm_node *first_node[2], int *ix_node, int letter_no,
                          const dm_codes *codes, const dm_codes *next_codes,
                          ArrayBuildState *soundex)

入力と出力

  • first_node[2]: 現在および次のノードリストを保持する配列。
  • ix_node: 現在のノードリストのインデックス（0または1）。関数終了時に更新される。
  • letter_no: 現在処理中の文字位置。
  • codes: 現在の文字に対応する音韻コードの候補。
  • next_codes: 次の文字に対応する音韻コードの候補。
  • soundex: 最終的な Soundex コードを蓄積する ArrayBuildState。
  • 出力: なし（first_node と ix_node を介して状態を更新）。

前提条件と不変条件

  • first_node は dm_node のリンクリストを指す必要がある。
  • ix_node は 0 または 1 の値をとる。
  • codes および next_codes は、有効な音韻コードを含む構造体であること。

アルゴリズムまたは振る舞い

1. ix_node_next を計算し、次の状態のためのリストを初期化する。 2. 現在のノードリスト（first_node[*ix_node]）を走査する。 3. 現在の文字の音韻コード（codes）と、次の文字の音韻コード（next_codes）の組み合わせをループ処理する。 4. letter_no が 0（先頭文字）か、次の文字が母音か、それ以外かに応じて code_index を決定する。 5. update_node を呼び出し、新しいノードの生成または既存ノードの更新を行う。 6. 最後に *ix_node を ix_node_next に更新する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/fuzzystrmatch/daitch_mokotoff.c 内の daitch_mokotoff_coding 関数から呼び出される。
  • 呼び出し先: update_node 関数を呼び出す。

関連データ構造

  • dm_node: 音韻コード生成のツリー構造を保持するノード。
  • dm_codes: 音韻ルールに基づくコードの集合。
  • ArrayBuildState: PostgreSQL の配列構築用状態管理構造体。

失敗モードと境界条件

  • letter_no == 0 の場合、先頭文字として特別なコード選択ロジックが適用される。
  • codes や next_codes が空の場合、ループは実行されない。
  • ix_node の交互切り替えにより、メモリ上のノードリストを効率的に再利用する。

保守担当者向けメモ

  • 本関数は dm_node のリンクリストを走査して状態を遷移させるため、ノードのメモリ管理とポインタの整合性に注意すること。
  • code_index の決定ロジックは Daitch-Mokotoff の音韻ルールに直結しているため、変更時は daitch_mokotoff.h のルール定義と照合すること。
