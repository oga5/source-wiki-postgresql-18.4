MemoryContextAllocationFailure

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/mcxt.c:1156

目的

メモリ割り当て要求が失敗した際の共通エラーハンドリングおよび統計情報の出力を行う。

シグネチャまたは形状

void *MemoryContextAllocationFailure(MemoryContext context, Size size, int flags)

入力と出力

  • **入力**:
  • context: 割り当てに失敗したメモリコンテキスト。
  • size: 要求されたメモリサイズ。
  • flags: 割り当てフラグ（MCXT_ALLOC_NO_OOM 等）。
  • **出力**:
  • flags に MCXT_ALLOC_NO_OOM が含まれる場合は NULL を返す。それ以外の場合は ereport(ERROR, ...) により処理が中断される。

前提条件と不変条件

  • flags に MCXT_ALLOC_NO_OOM が含まれていない場合、本関数は呼び出し元へ戻らず、PostgreSQL のエラー処理機構によって例外が発生する。

アルゴリズムまたは振る舞い

1. flags を確認し、MCXT_ALLOC_NO_OOM ビットが立っていないか判定する。 2. ビットが立っていない場合:

  • TopMemoryContext が存在すれば MemoryContextStats を呼び出し、現在のメモリ使用状況をログに出力する。
  • ereport(ERROR, ...) を発行し、メモリ不足（ERRCODE_OUT_OF_MEMORY）としてエラーを報告する。エラーメッセージには要求サイズとコンテキスト名が含まれる。

    • 3. ビットが立っている場合、何もせず NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/alignedalloc.c
  • src/backend/utils/mmgr/aset.c
  • src/backend/utils/mmgr/bump.c
  • src/backend/utils/mmgr/generation.c
  • **呼び出し先**:
  • MemoryContextStats (条件付き)
  • ereport (条件付き)

関連データ構造

  • MemoryContext: メモリコンテキストの基底構造体。

失敗モードと境界条件

  • context->name が NULL の場合、ereport 内のフォーマット指定で問題が発生する可能性がある（コンテキストの初期化状態に依存）。
  • TopMemoryContext が初期化されていない段階で呼び出された場合、MemoryContextStats の呼び出しはスキップされる。

保守担当者向けメモ

  • 本関数はメモリ不足時の最終的なエラー報告ポイントであるため、ここでの追加処理はスタック消費を最小限に抑える必要がある。
