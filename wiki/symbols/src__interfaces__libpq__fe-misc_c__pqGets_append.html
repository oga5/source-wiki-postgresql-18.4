pqGets_append

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-misc.c:141

目的

通信バッファから文字列を読み出し、既存の PQExpBuffer に追記します。

シグネチャまたは形状

int pqGets_append(PQExpBuffer buf, PGconn *conn);

入力と出力

  • **入力**:
  • buf: 読み出した文字列を格納（追記）するための PQExpBuffer 構造体へのポインタ。
  • conn: 接続状態を保持する PGconn 構造体へのポインタ。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は EOF を返します。

前提条件と不変条件

  • conn が有効な接続状態であること。
  • buf が初期化済みであること。
  • 本関数は内部的に pqGets_internal を呼び出しており、その引数 false により、バッファのクリアを行わずに追記する動作が保証されます。

アルゴリズムまたは振る舞い

1. pqGets_internal(buf, conn, false) を呼び出します。 2. pqGets_internal の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-connect.c:4059 で確認されています。
  • **呼び出し先**: pqGets_internal を呼び出します。

関連データ構造

  • PQExpBuffer: 動的文字列バッファ。
  • PGconn: libpq の接続状態を管理する構造体。

失敗モードと境界条件

  • 接続が切断された場合や、読み込み中にエラーが発生した場合は EOF を返します。
  • PQExpBuffer へのメモリ割り当てに失敗した場合の挙動は、内部の pqGets_internal および pqexpbuffer.c の実装に依存します。

保守担当者向けメモ

  • 本関数は pqGets_internal のラッパーです。ロジックの変更が必要な場合は pqGets_internal を参照してください。
  • false を渡しているため、既存のバッファ内容を保持したまま追記を行う点に注意してください。
