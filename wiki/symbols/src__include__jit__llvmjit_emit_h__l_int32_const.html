l_int32_const

定義

  • 種別: static inline function
  • 定義箇所: src/include/jit/llvmjit_emit.h:65

目的

LLVM IR 生成において、指定された 32 ビット整数値を表す定数ノード（LLVMValueRef）を生成するヘルパー関数です。

シグネチャまたは形状

LLVMValueRef l_int32_const(LLVMContextRef lc, int32 i)

入力と出力

  • **入力**:
  • lc (LLVMContextRef): 定数を作成する対象の LLVM コンテキスト。
  • i (int32): 生成する定数値。
  • **出力**:
  • LLVMValueRef: LLVM IR 上の 32 ビット整数定数。

前提条件と不変条件

  • lc は有効な LLVMContextRef である必要があります。
  • 本関数は USE_LLVM が定義されている環境でのみ利用可能です。

アルゴリズムまたは振る舞い

1. LLVMInt32TypeInContext(lc) を呼び出し、指定されたコンテキストにおける 32 ビット整数型を取得します。 2. LLVMConstInt を使用して、値 i を持つ定数ノードを生成します。この際、符号付き整数として扱われないよう false が指定されます。 3. 生成された LLVMValueRef を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit_deform.c
  • src/backend/jit/llvm/llvmjit_expr.c

    • (主に JIT コンパイル時のパラメータ設定や属性値の定数化に使用されています)

  • **呼び出し先**:
  • LLVMInt32TypeInContext (LLVM C API)
  • LLVMConstInt (LLVM C API)

関連データ構造

  • LLVMValueRef: LLVM IR の値を表す不透明なポインタ型。
  • LLVMContextRef: LLVM のコンパイルコンテキストを管理する型。

失敗モードと境界条件

  • LLVM C API の仕様に基づき、無効なコンテキストが渡された場合の動作は未定義です。

保守担当者向けメモ

  • 本関数はインライン関数であり、呼び出し側のコンテキストで展開されます。
  • LLVM C API のバージョンアップに伴い、定数生成の仕様が変更された場合は本関数を修正してください。
  • 構造体アクセス等でフィールド番号を指定する際によく利用されます。
