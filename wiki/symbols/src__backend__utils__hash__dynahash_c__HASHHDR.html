HASHHDR

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/hash/dynahash.c:168

目的

ハッシュテーブルの制御情報を保持するヘッダー構造体です。共有メモリ上に配置されることを前提としており、ハッシュテーブルのディレクトリ構造や空きリスト、構成パラメータを管理します。

シグネチャまたは形状

struct HASHHDR {
    FreeListData freeList[NUM_FREELISTS];
    long         dsize;
    long         nsegs;
    uint32       max_bucket;
    uint32       high_mask;
    uint32       low_mask;
    Size         keysize;
    Size         entrysize;
    long         num_partitions;
    long         max_dsize;
    long         ssize;
    int          sshift;
    int          nelem_alloc;
#ifdef HASH_STATISTICS
    long         accesses;
    long         collisions;
#endif
};

入力と出力

  • 本構造体は関数ではなくデータ構造であるため、入力および出力の概念は適用されません。
  • 共有メモリ上のハッシュテーブル作成時、hash_create を通じて初期化されます。

前提条件と不変条件

  • num_partitions が 0 でない場合、ハッシュテーブルはパーティション分割されており、freeList の各要素が独立したスピンロックとエントリ数管理を持ちます。
  • 非パーティション分割テーブルの場合、freeList[0] のみが使用され、呼び出し元によるロックが十分であると仮定されます。
  • ssize（セグメントサイズ）は 2 のべき乗である必要があります。
  • 共有メモリ上のハッシュテーブルでは、dsize（ディレクトリサイズ）は変更できません。

アルゴリズムまたは振る舞い

  • **空きリスト管理**: 高並行環境での競合を避けるため、複数の空きリストを保持します。必要に応じて、デフォルト以外の空きリストからエントリを回収（scavenge）します。
  • **統計情報**: HASH_STATISTICS が定義されている場合、アクセス数と衝突数を記録しますが、パーティション分割されたテーブルではスピンロックを使用しないため、値にわずかな不整合が生じる可能性があります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/shmem.c
  • src/backend/utils/hash/dynahash.c（hash_create 等の内部関数）
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • FreeListData: パーティションごとの空きリスト管理構造体。
  • HTAB: 各バックエンドが保持するハッシュテーブルの制御構造体。
  • HASHSEGMENT: ハッシュテーブルのディレクトリを構成するセグメント。

失敗モードと境界条件

  • 共有メモリ上のハッシュテーブルでは、初期化時に指定した nelem（要素数）の見積もりが不正確であると、動的なディレクトリ拡張ができないため制限に達する可能性があります。

保守担当者向けメモ

  • HASHHDR は共有メモリ上に配置されるため、構造体のレイアウト変更はバイナリ互換性に影響を与える可能性があります。
  • パーティション分割されたテーブルにおける統計情報の競合は、意図的な設計上のトレードオフです。
