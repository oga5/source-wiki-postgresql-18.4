get_snapbuild_state_desc

定義

  • 種別: function
  • 定義箇所: contrib/pg_logicalinspect/pg_logicalinspect.c:30

目的

SnapBuildState 列挙型の内部状態値を、人間が判読可能な文字列形式に変換する。

シグネチャまたは形状

static const char *get_snapbuild_state_desc(SnapBuildState state)

入力と出力

  • **入力**: SnapBuildState state（論理デコーディングにおけるスナップショット構築の状態を示す列挙値）
  • **出力**: const char *（状態に対応する文字列ラベル）

前提条件と不変条件

  • 入力される state は SnapBuildState 型の有効な列挙値であること。
  • 定義された列挙値（SNAPBUILD_START, SNAPBUILD_BUILDING_SNAPSHOT, SNAPBUILD_FULL_SNAPSHOT, SNAPBUILD_CONSISTENT）のいずれかに該当しない場合、デフォルト値として "unknown state" を返す。

アルゴリズムまたは振る舞い

1. 初期値を "unknown state" に設定する。 2. switch 文を用いて state を評価する。 3. 各ケースに応じて以下の文字列を割り当てる：

  • SNAPBUILD_START → "start"
  • SNAPBUILD_BUILDING_SNAPSHOT → "building"
  • SNAPBUILD_FULL_SNAPSHOT → "full"
  • SNAPBUILD_CONSISTENT → "consistent"

    • 4. 決定した文字列ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_logicalinspect/pg_logicalinspect.c 内の pg_get_logical_snapshot_info 関数（ondisk.builder.state を引数として渡している）。
  • **呼び出し先**:
  • 本関数は外部関数を呼び出さない。

関連データ構造

  • SnapBuildState: 論理デコーディングのビルド状態を定義する列挙型（replication/snapbuild_internal.h で定義）。

失敗モードと境界条件

  • 定義されていない SnapBuildState 値が渡された場合、エラーにはならず "unknown state" を返す。

保守担当者向けメモ

  • src/include/replication/snapbuild.h に記載がある通り、SnapBuildState の定義が変更された場合は、本関数の switch 文も追従して更新する必要がある。
