bbsink_server_archive_contents

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_server.c:159

目的

バックアップストリームから受信したデータを、サーバー上のローカルファイルへ書き込む。

シグネチャまたは形状

static void bbsink_server_archive_contents(bbsink *sink, size_t len)

入力と出力

  • sink: bbsink 構造体へのポインタ。内部で bbsink_server 型にキャストして使用される。
  • len: 書き込むデータのバイト数。
  • 出力: なし（成功時は内部状態の更新とデータの書き込み、失敗時は ereport によるエラー終了）。

前提条件と不変条件

  • sink は bbsink_server 型として初期化されていること。
  • mysink->file が有効なファイル記述子として開かれていること。
  • mysink->base.bbs_buffer に書き込むべきデータが格納されていること。

アルゴリズムまたは振る舞い

1. FileWrite を呼び出し、bbs_buffer から指定された長さ len のデータを現在のファイルオフセット mysink->filepos に書き込む。 2. 書き込みバイト数が len と一致しない場合、エラーを報告する。

  • nbytes < 0 の場合はファイルアクセスエラーとして報告。
  • nbytes < len の場合はディスクフルとして報告。

    • 3. 成功した場合、mysink->filepos を書き込んだバイト数分進める。 4. bbsink_forward_archive_contents を呼び出し、後続のシンクへ処理を転送する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink_server_ops 構造体を通じて、バックアップ処理のメインフローから間接的に呼び出される。
  • 呼び出し先:
  • FileWrite (storage/fd.h)
  • FilePathName (storage/fd.h)
  • bbsink_forward_archive_contents (basebackup_sink.c)

関連データ構造

  • bbsink_server: ローカルファイル保存用の状態を保持する構造体。
  • bbsink: バックアップシンクの基底インターフェース。

失敗モードと境界条件

  • ディスク容量不足: FileWrite が要求したバイト数より少ない値を返した場合、ERRCODE_DISK_FULL でエラー終了する。
  • ファイル書き込みエラー: FileWrite が負の値を返した場合、ファイルアクセスエラーとして終了する。

保守担当者向けメモ

  • 本関数は bbsink インターフェースの archive_contents メソッドとして登録されている。
  • fsync は本関数内では実行されず、アーカイブ終了時に bbsink_server_end_archive で行われる設計である。
