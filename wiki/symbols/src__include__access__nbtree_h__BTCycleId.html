BTCycleId

定義

  • 種別: typedef
  • 定義箇所: src/include/access/nbtree.h:30

目的

B-tree インデックスの VACUUM 処理における「サイクル ID」を保持するための識別子です。主にインデックスのページ分割と VACUUM 処理の整合性を管理するために使用されます。

シグネチャまたは形状

typedef uint16 BTCycleId;

入力と出力

  • 入力: 符号なし 16 ビット整数。
  • 出力: 該当なし（型定義）。

前提条件と不変条件

  • uint16 型であるため、値の範囲は 0 から 65535 です。
  • src/backend/access/nbtree/nbtutils.c 内で time(NULL) の結果をキャストして生成されるなど、時間的な一意性を確保する目的で利用される場合があります。

アルゴリズムまたは振る舞い

  • インデックスページ（BTPageOpaqueData）の btpo_cycleid フィールドに格納され、そのページで最後に発生した分割がどの VACUUM サイクルに関連しているかを記録します。
  • VACUUM 処理中、インデックスの各ページが現在のサイクル ID と一致するかを確認することで、並行して行われるページ分割操作との競合を制御します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/include/access/nbtree.h: BTPageOpaqueData 構造体内の btpo_cycleid として定義。
  • src/backend/access/nbtree/nbtutils.c: cycle_ctr として最新の ID を保持・生成。
  • src/backend/access/nbtree/nbtree.c: ページ分割や検索処理の引数として使用。
  • contrib/pageinspect/btreefuncs.c: インデックスの内部構造を検査する際に使用。
  • **呼び出し先**:
  • 本シンボルは型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • BTPageOpaqueData: B-tree ページの末尾に配置される構造体。btpo_cycleid フィールドを通じて本型を利用します。

失敗モードと境界条件

  • **オーバーフロー**: uint16 型のため、値が 65535 を超えると 0 にラップアラウンドします。このため、サイクル ID の比較は単純な大小比較ではなく、循環的な順序を考慮したロジックが必要となる場合があります。

保守担当者向けメモ

  • 本型は BTPageOpaqueData の一部としてディスク上に永続化される可能性があるため、将来的なサイズ変更には注意が必要です。
  • サイクル ID の生成ロジック（time(NULL) の利用など）を変更する場合、既存のインデックスページとの互換性に影響がないか確認してください。
