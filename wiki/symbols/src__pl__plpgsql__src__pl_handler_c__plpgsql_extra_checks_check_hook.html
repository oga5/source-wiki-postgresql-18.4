plpgsql_extra_checks_check_hook

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_handler.c:62

目的

PL/pgSQL の設定パラメータ plpgsql.extra_checks の値を検証し、内部的なビットマスク表現へ変換する GUC（Grand Unified Configuration）チェックフック関数です。

シグネチャまたは形状

static bool plpgsql_extra_checks_check_hook(char **newvalue, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newvalue: 設定される文字列値へのポインタ。
  • extra: 変換後のビットマスクを格納するための出力用ポインタ。
  • source: 設定のソース（GUC 内部で使用）。
  • **出力**:
  • 成功時は true を返し、extra に割り当てられた int 型のビットマスクを設定。
  • 失敗時は false を返し、GUC_check_errdetail を通じてエラー詳細を報告。

前提条件と不変条件

  • newvalue は有効な文字列である必要があります。
  • all または none が指定された場合、他のキーワードとの組み合わせは許可されません。
  • 成功時に extra へ割り当てられるメモリは guc_malloc によって確保されます。

アルゴリズムまたは振る舞い

1. 入力文字列が all または none の場合、対応するビットマスク（PLPGSQL_XCHECK_ALL または PLPGSQL_XCHECK_NONE）を即座に決定します。 2. それ以外の場合、文字列をカンマ区切りで解析し、個々の識別子を検証します。

  • shadowed_variables、too_many_rows、strict_multi_assignment のいずれかであれば、対応するビットを extrachecks に加算します。
  • 不明なキーワードや、all/none と他のキーワードの混在が検出された場合は、エラー詳細を設定して false を返します。

    • 3. 解析完了後、guc_malloc で確保したメモリにビットマスクを格納し、extra にセットして true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GUC 管理システム（src/pl/plpgsql/src/pl_handler.c 内の定義構造体から参照）。
  • **呼び出し先**:
  • pstrdup, pfree, list_free, guc_malloc（メモリ管理）
  • SplitIdentifierString（文字列解析）
  • pg_strcasecmp（文字列比較）
  • GUC_check_errdetail（エラー報告）

関連データ構造

  • PLPGSQL_XCHECK_* 定数（ビットマスク値）
  • GucSource（GUC システム定義）

失敗モードと境界条件

  • **構文エラー**: SplitIdentifierString が失敗した場合、または未知のキーワードが含まれる場合に false を返します。
  • **不正な組み合わせ**: all や none を他のキーワードと併用した場合に false を返します。
  • **メモリ不足**: guc_malloc が失敗した場合に false を返します。

保守担当者向けメモ

  • 本関数は plpgsql.extra_checks の設定変更時にのみ実行されます。
  • 新しいチェック項目を追加する場合は、解析ループ内の if-else ブロックに新しいキーワードとビットフラグを追加してください。
  • pstrdup で確保したメモリは、解析終了後に必ず pfree される設計となっています。
