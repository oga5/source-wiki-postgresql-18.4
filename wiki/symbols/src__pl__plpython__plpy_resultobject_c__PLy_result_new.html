PLy_result_new

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_resultobject.c:88

目的

クエリ実行結果を保持する新しい PLyResultObject インスタンスを生成し、初期化するファクトリ関数です。

シグネチャまたは形状

PyObject *PLy_result_new(void)

入力と出力

  • **入力**: なし
  • **出力**: 成功時は初期化された PLyResultObject へのポインタ（PyObject *）、失敗時は NULL。

前提条件と不変条件

  • PLy_ResultType が Python 型システムに登録済みであること。
  • Python 3.8 未満の環境では、PLy_ResultType の参照カウントが適切に管理されること。

アルゴリズムまたは振る舞い

1. PyObject_New を使用して PLyResultObject をメモリ確保します。 2. Python 3.8 未満の場合、PLy_ResultType の参照カウントをインクリメントします。 3. 内部フィールドを初期化します：

  • status を Py_None に設定（参照カウントをインクリメント）。
  • nrows を -1 を表す Python Long オブジェクトとして設定。
  • rows を空の Python リストとして作成。
  • tupdesc を NULL に設定。

    • 4. rows リストの作成に失敗した場合は、確保したオブジェクトを解放して NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_cursorobject.c:427
  • src/pl/plpython/plpy_spi.c:342
  • **呼び出し先**:
  • PyObject_New (Python C API)
  • Py_INCREF / Py_DECREF (Python C API)
  • PyLong_FromLong (Python C API)
  • PyList_New (Python C API)

関連データ構造

  • PLyResultObject: クエリ結果を保持するC構造体。

失敗モードと境界条件

  • メモリ不足により PyObject_New が失敗した場合、NULL を返します。
  • PyList_New が失敗した場合、確保した PLyResultObject を解放し NULL を返します。

保守担当者向けメモ

  • Python 3.8 以降では Py_INCREF(PLy_ResultType) が不要となるよう条件付きコンパイルが適用されています。
  • tupdesc は初期状態では NULL です。必要に応じて後続の処理で設定されることを想定しています。
