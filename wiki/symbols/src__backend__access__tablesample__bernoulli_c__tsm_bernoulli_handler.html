tsm_bernoulli_handler

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/bernoulli.c:64

目的

PostgreSQL の TABLESAMPLE BERNOULLI メソッドを実装するための TsmRoutine 構造体を初期化し、サンプリング処理に必要なコールバック関数を登録するハンドラ関数です。

シグネチャまたは形状

Datum tsm_bernoulli_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (PostgreSQL の関数呼び出し規約に基づく引数)
  • **出力**: TsmRoutine 構造体へのポインタを Datum 型として返します。

前提条件と不変条件

  • makeNode(TsmRoutine) によりメモリが確保されること。
  • 戻り値の TsmRoutine は、TABLESAMPLE メソッドとして必要な関数ポインタが適切に設定されていること。

アルゴリズムまたは振る舞い

1. TsmRoutine ノードを生成します。 2. パラメータ型として FLOAT4OID を設定します。 3. クエリ間およびスキャン間でのサンプリングの再現性を true に設定します。 4. 以下のコールバック関数を登録します:

  • SampleScanGetSampleSize: bernoulli_samplescangetsamplesize
  • InitSampleScan: bernoulli_initsamplescan
  • BeginSampleScan: bernoulli_beginsamplescan
  • NextSampleTuple: bernoulli_nextsampletuple

    • 5. NextSampleBlock および EndSampleScan は NULL に設定されます。 6. 初期化された構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内に直接的な呼び出し元の記述はありません。
  • **呼び出し先**: makeNode マクロ、list_make1_oid、および bernoulli_ で始まる各サンプリング用関数を登録します。

関連データ構造

  • TsmRoutine: サンプリングメソッドのインターフェースを定義する構造体。

失敗モードと境界条件

  • メモリ不足時には makeNode によりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は TABLESAMPLE BERNOULLI のエントリポイントです。新しいサンプリングロジックを追加または変更する場合は、この関数で登録されるコールバック関数群を確認してください。
