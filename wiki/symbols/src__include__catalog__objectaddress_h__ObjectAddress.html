ObjectAddress

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/objectaddress.h:24

目的

PostgreSQL 内のあらゆるデータベースオブジェクトを、カタログテーブルの物理構造に依存せず一意に識別するための抽象化構造体です。

シグネチャまたは形状

typedef struct ObjectAddress
{
	Oid			classId;		/* Class Id from pg_class */
	Oid			objectId;		/* OID of the object */
	int32		objectSubId;	/* Subitem within object (eg column), or 0 */
} ObjectAddress;

入力と出力

  • 本構造体は、get_object_address 関数によってパーサー出力から生成されます。
  • getObjectIdentity や getObjectDescription 関数への入力として使用され、人間が読める形式の文字列に変換されます。

前提条件と不変条件

  • objectSubId は、テーブルの列（Column）など、親オブジェクト内のサブ要素を指す場合にのみ使用されます。それ以外の場合は常に 0 である必要があります。
  • カタログ操作の際は、適切な LOCKMODE を指定して同時実行制御を行う必要があります。

アルゴリズムまたは振る舞い

  • システムカタログ上のオブジェクトを classId（カタログの OID）と objectId（オブジェクト自身の OID）の組み合わせで特定します。
  • 物理的なカタログテーブルを直接参照する代わりに、この構造体を介して所有権確認やメタデータ取得を行う汎用的なインターフェースを提供します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql モジュール（database.c, dml.c, label.c, proc.c）において、セキュリティラベルの管理や権限チェックのために使用されています。
  • **呼び出し先**: 構造体自体はデータ保持用ですが、関連する操作関数として get_object_address、check_object_ownership、getObjectIdentity 等が定義されています。

関連データ構造

  • ObjectAddresses: 複数の ObjectAddress を保持するための構造体。
  • ObjectType: オブジェクトの種類を定義する列挙型（本ヘッダーファイル内で定義）。
  • HeapTuple: access/htup.h で定義され、カタログ検索時に使用されます。

失敗モードと境界条件

  • 存在しない OID や無効な classId を指定した場合、カタログ検索関数が失敗する可能性があります。
  • objectSubId の不適切な使用（親オブジェクトが存在しない場合のサブ要素指定など）は、後続の処理でエラーを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は DDL コマンド、イベントトリガー、権限管理システムの中核を成します。
  • 実装の詳細は src/backend/catalog/objectaddress.c にあり、ObjectPropertyType 構造体によるカタログ属性管理と密接に関連しています。
