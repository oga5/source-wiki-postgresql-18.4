PGProcShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/proc.c:96

目的

PostgreSQLのプロセス管理用共有メモリ領域（PGPROC配列および関連する状態フラグやXID配列）に必要な合計バイト数を計算する。

シグネチャまたは形状

static Size PGProcShmemSize(void)

入力と出力

  • **入力**: なし（グローバル設定値 MaxBackends, NUM_AUXILIARY_PROCS, max_prepared_xacts を参照）
  • **出力**: Size 型の共有メモリ所要バイト数

前提条件と不変条件

  • MaxBackends、NUM_AUXILIARY_PROCS、max_prepared_xacts の合計値が TotalProcs として算出される。
  • 共有メモリの計算には add_size および mul_size を使用し、オーバーフローを防止している。

アルゴリズムまたは振る舞い

1. TotalProcs を MaxBackends + NUM_AUXILIARY_PROCS + max_prepared_xacts として算出する。 2. 以下の各要素の合計サイズを計算する：

  • PGPROC 構造体配列 (TotalProcs 分)
  • ProcGlobal->xids 配列 (TotalProcs 分)
  • ProcGlobal->subxidStates 配列 (TotalProcs 分)
  • ProcGlobal->statusFlags 配列 (TotalProcs 分)

    • 3. 計算された合計サイズを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/proc.c 内の共有メモリサイズ見積もりロジック（147行目、236行目など）。
  • **呼び出し先**:
  • add_size
  • mul_size

関連データ構造

  • PGPROC: プロセスごとの状態管理構造体。
  • ProcGlobal: プロセス管理のグローバルヘッダー。

失敗モードと境界条件

  • add_size および mul_size を使用しているため、計算結果が SIZE_MAX を超える場合はエラーとなる。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウト変更や、PGPROC 構造体のサイズ変更時に必ず更新する必要がある。
  • TotalProcs の算出ロジックは、プロセス管理のメモリ設計と密接に結びついているため注意すること。
