XLogLongPageHeaderData

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlog_internal.h:61

目的

WAL（先行書き込みログ）のページヘッダーが拡張形式である場合に、システム識別子やセグメントサイズなどの追加情報を含めるための構造体です。

シグネチャまたは形状

typedef struct XLogLongPageHeaderData
{
    XLogPageHeaderData std;        /* 標準ヘッダーフィールド */
    uint64        xlp_sysid;       /* pg_control からのシステム識別子 */
    uint32        xlp_seg_size;    /* セグメントサイズのクロスチェック用 */
    uint32        xlp_xlog_blcksz; /* ブロックサイズのクロスチェック用 */
} XLogLongPageHeaderData;

入力と出力

  • **入力**: WALページ書き込み時に、システムIDや設定値が本構造体に格納されます。
  • **出力**: WAL読み込み時、XLogReaderState が本構造体を介してページヘッダーのメタデータを検証します。

前提条件と不変条件

  • std.xlp_info に XLP_LONG_HEADER フラグが設定されている場合にのみ、この構造体が使用されます。
  • xlp_seg_size および xlp_xlog_blcksz は、現在のデータベース設定との整合性を確認するためのクロスチェック値として機能します。

アルゴリズムまたは振る舞い

WALの物理レイアウトにおいて、ページヘッダーが標準的な XLogPageHeaderData よりも多くの情報（システムID等）を保持する必要がある場合、この構造体がページ先頭に配置されます。WAL読み込み処理では、このヘッダーを解析することで、ログが現在のデータベースインスタンスと互換性があるかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: WAL書き込み処理（src/backend/access/transam/xlog.c）およびWAL読み込み処理（XLogReaderState）。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、SizeOfXLogLongPHD マクロを通じてサイズ計算に利用されます。
  • 注: 呼び出し元・呼び出し先の詳細な関数レベルの依存関係は、提供されたソース範囲内では明示されていません。

関連データ構造

  • XLogPageHeaderData: 本構造体の先頭に含まれる標準ヘッダー。
  • XLogLongPageHeader: 本構造体へのポインタ型（XLogLongPageHeaderData *）。

失敗モードと境界条件

  • xlp_sysid や xlp_seg_size が現在のデータベースの pg_control 情報と一致しない場合、WALの読み込みやリカバリ処理において不整合としてエラーが発生します。

保守担当者向けメモ

  • 本構造体はフロントエンドツール（pg_receivewal 等）からも参照されるため、バックエンド専用のヘッダーファイルや関数を含めないように注意してください。
  • 構造体のサイズ変更を行う場合は、SizeOfXLogLongPHD マクロが MAXALIGN を考慮していることを確認してください。
