newhicolorrow

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:420

目的

hicolormap において新しい行を割り当て、既存の行の内容を複製することで、高位文字コードの彩色管理領域を拡張・初期化します。

シグネチャまたは形状

static int newhicolorrow(struct colormap *cm, int oldrow)

入力と出力

  • **入力**:
  • cm: カラーマップ構造体へのポインタ。
  • oldrow: 複製元となる既存の行インデックス。
  • **出力**:
  • 成功時: 新しく割り当てられた行のインデックス（int）。
  • 失敗時: 0（エラー発生時）。

前提条件と不変条件

  • cm は有効な colormap 構造体を指している必要があります。
  • oldrow は現在の cm->hiarrayrows の範囲内である必要があります。
  • メモリ拡張が必要な場合、cm->maxarrayrows が INT_MAX を超えないことが保証されます。

アルゴリズムまたは振る舞い

1. **領域確保**: 現在の行数が最大容量（maxarrayrows）に達している場合、REALLOC_ARRAY を使用して hicolormap の容量を2倍に拡張します。 2. **行の追加**: hiarrayrows をインクリメントし、新しい行を確保します。 3. **データ複製**: memcpy を使用し、oldrow の内容を新しい行へコピーします。 4. **参照カウント更新**: 新しい行に含まれる各色の参照カウント（nuchrs）をインクリメントし、カラーマップの整合性を維持します。 5. **エラー処理**: メモリ確保に失敗した場合は CERR(REG_ESPACE) を呼び出し、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_color.c 内の複数箇所（行 679, 705, 714, 798, 829, 838, 856）。
  • **呼び出し先**: REALLOC_ARRAY（マクロ）、CERR（マクロ）、memcpy（標準ライブラリ）。

関連データ構造

  • colormap: 彩色情報を保持する主要構造体。hicolormap（配列）、hiarrayrows、maxarrayrows、hiarraycols を含みます。
  • colordesc: cd 配列を通じて参照される色情報。nuchrs（参照カウント）を保持します。

失敗モードと境界条件

  • **メモリ不足**: REALLOC_ARRAY が NULL を返した場合、REG_ESPACE エラーを報告し 0 を返します。
  • **整数オーバーフロー**: maxarrayrows が INT_MAX を超える計算になる場合、REG_ESPACE エラーを報告し 0 を返します。

保守担当者向けメモ

  • 本関数は hicolormap のポインタを移動させる可能性があるため、呼び出し元で cm->hicolormap を直接参照し続けることは避けてください。
  • hiarraycols は固定長として扱われるため、行の複製は memcpy によるメモリブロック単位のコピーで完結します。
