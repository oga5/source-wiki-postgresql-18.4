test_read_multixact

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_multixact.c:41

目的

指定されたMultiXact IDのメンバー情報を読み取り、SLRU（Simple Least Recently Used）キャッシュ機構の動作を検証する。

シグネチャまたは形状

Datum test_read_multixact(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して MultiXactId を受け取る。
  • **出力**: void。成功時は何も返さない。指定されたIDが見つからない場合はエラーを発生させる。

前提条件と不変条件

  • PostgreSQLの関数管理インターフェース（fmgr）を通じて呼び出されることを前提とする。
  • 実行時に AtEOXact_MultiXact() が呼び出され、セッション内のMultiXactキャッシュが強制的に破棄される。

アルゴリズムまたは振る舞い

1. PG_GETARG_TRANSACTIONID(0) により、引数から対象の MultiXactId を取得する。 2. AtEOXact_MultiXact() を実行し、ローカルキャッシュをクリアする。 3. GetMultiXactIdMembers() を呼び出し、SLRU層からメンバー情報を取得する。 4. 戻り値が -1 の場合、elog(ERROR) を発行して処理を中断する。 5. 正常終了時は PG_RETURN_VOID() を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由での呼び出しを想定（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • AtEOXact_MultiXact()
  • GetMultiXactIdMembers()
  • elog()

関連データ構造

  • MultiXactId: 複数トランザクションを識別するためのID。
  • MultiXactMember: MultiXactのメンバー情報を保持する構造体。

失敗モードと境界条件

  • 指定された MultiXactId が存在しない場合、elog(ERROR) によりトランザクションがアボートされる。

保守担当者向けメモ

  • 本関数はテスト専用であり、キャッシュを意図的に破棄して物理的な読み取りを強制する副作用を持つ。実運用コードでの直接利用は推奨されない。
