output_escaped_str

定義

  • **種別**: function
  • **定義箇所**: src/interfaces/ecpg/preproc/output.c:195

目的

入力文字列をC言語の文字列リテラルとして安全に出力するためのエスケープ処理を行います。

シグネチャまたは形状

static void output_escaped_str(const char *str, bool quoted)

入力と出力

  • **入力**:
  • str: エスケープ対象の文字列。
  • quoted: 文字列が引用符で囲まれているかを示すフラグ。
  • **出力**: なし（副作用としてグローバル変数 base_yyout が指すストリームへ書き出します）。

前提条件と不変条件

  • base_yyout が有効なファイルストリームとして初期化されている必要があります。
  • quoted が true の場合、str の先頭と末尾が " であることが期待されます。

アルゴリズムまたは振る舞い

1. **引用符の処理**: quoted が true かつ文字列が " で囲まれている場合、先頭の " を出力し、処理対象から除外します。 2. **文字単位の走査**: 文字列を1文字ずつ走査し、以下の変換を行います。

  • " → \"
  • \n → \\\n
  • \ → \\ （ただし、直後に改行が続く場合はエスケープをスキップ）
  • \r\n → \\\r\n
  • その他: そのまま出力。

    • 3. **末尾の処理**: quoted が true の場合、末尾の " を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/output.c 内の各出力関数（output_simple_statement, output_prepare_statement, output_deallocate_prepare_statement 等）から呼び出されます。
  • **呼び出し先**: 標準ライブラリ関数 strlen, fputs, fputc を使用します。

関連データ構造

  • base_yyout: 出力先となるファイルストリーム（preproc_extern.h で宣言）。

失敗モードと境界条件

  • str が空文字列の場合、ループは実行されず、quoted フラグに応じた引用符のみが出力される可能性があります。
  • \ の直後に空白やタブが続く場合、それらをスキップして改行の有無を判定するロジックが含まれています。

保守担当者向けメモ

  • 本関数はC言語の文字列リテラル生成に特化しています。新しいエスケープルールを追加する場合は、for ループ内の条件分岐を更新してください。
  • quoted フラグの判定ロジックは、入力文字列の構造に依存しているため、呼び出し側で渡す文字列が期待通りの形式であることを確認してください。
