oauth_ctx

定義

  • 種別: struct
  • 定義箇所: src/backend/libpq/auth-oauth.c:65

目的

SASL OAUTHBEARER 認証プロセスにおける、セッションごとの状態および関連情報を保持するために使用されます。

シグネチャまたは形状

struct oauth_ctx
{
    enum oauth_state state;
    Port       *port;
    const char *issuer;
    const char *scope;
};

入力と出力

本構造体は関数間で受け渡されるコンテキストオブジェクトとして機能します。

  • state: 現在の認証状態（oauth_state 型）。
  • port: 接続中のクライアントポート情報。
  • issuer, scope: 認証エラー応答生成時に使用されるメタデータ。

前提条件と不変条件

  • 認証フローの各ステップにおいて、有効な oauth_ctx インスタンスが維持される必要があります。
  • state は RFC 7628 に基づく状態遷移（INIT, ERROR, FINISHED 等）に従う必要があります。

アルゴリズムまたは振る舞い

  • 認証ハンドシェイク中の状態管理を行います。
  • 認証失敗時には、保持している issuer や scope を用いて generate_error_response がエラー詳細を含む JSON を構築します。

呼び出し元と呼び出し先

  • 呼び出し元: oauth_exchange 等の認証処理関数。
  • 呼び出し先: generate_error_response が本構造体を引数として受け取ります。

関連データ構造

  • oauth_state: 認証状態を定義する列挙型。
  • Port: クライアント接続情報を保持する構造体。

失敗モードと境界条件

  • 認証失敗時、state が OAUTH_STATE_ERROR に遷移し、再試行フローをサポートします。
  • 外部検証モジュールによる検証失敗時、本構造体の情報を用いてクライアントへ適切なエラー応答が生成されます。

保守担当者向けメモ

  • 本構造体はセッションごとの状態を保持するため、メモリ管理には注意してください。
  • 認証フローの拡張（RFC 7628 Sec 3.2.3 の再試行フロー等）を行う際は、state の遷移ロジックと本構造体のフィールド整合性を確認してください。
