LocalBufferAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/localbuf.c:117

目的

指定されたブロック（forkNum, blockNum）に対応するローカルバッファを取得または割り当てます。一時テーブル用のバッファ管理において、ハッシュテーブル検索または新規バッファの確保を行い、該当するバッファ記述子を返します。

シグネチャまたは形状

BufferDesc *LocalBufferAlloc(SMgrRelation smgr, ForkNumber forkNum, BlockNumber blockNum, bool *foundPtr)

入力と出力

  • **入力**:
  • smgr: 対象リレーションのストレージマネージャ情報。
  • forkNum: フォーク番号。
  • blockNum: ブロック番号。
  • **出力**:
  • 戻り値: 取得されたバッファの BufferDesc 構造体へのポインタ。
  • foundPtr: バッファが既に存在した場合は true、新規割り当ての場合は false が設定されます。

前提条件と不変条件

  • 初回呼び出し時に InitLocalBuffers() が実行され、LocalBufHash が初期化されている必要があります。
  • CurrentResourceOwner が有効である必要があります。
  • 割り当てられたバッファは、呼び出し元でピン留めされた状態となります。

アルゴリズムまたは振る舞い

1. InitBufferTag を使用してリクエストされたブロックのタグを生成します。 2. LocalBufHash を検索し、既にバッファが存在する場合は PinLocalBuffer を呼び出してピン留めし、foundPtr を true に設定して終了します。 3. バッファが存在しない場合:

  • GetLocalVictimBuffer() を呼び出し、置換対象のバッファ ID を取得します。
  • LocalBufHash に新しいタグを登録します。
  • バッファ記述子のタグを更新し、状態フラグを BM_TAG_VALID および BUF_USAGECOUNT_ONE に設定します。
  • foundPtr を false に設定して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c:1148
  • **呼び出し先**:
  • InitLocalBuffers()
  • hash_search()
  • GetLocalBufferDescriptor()
  • PinLocalBuffer()
  • GetLocalVictimBuffer()
  • pg_atomic_read_u32() / pg_atomic_unlocked_write_u32()

関連データ構造

  • BufferDesc: バッファのメタデータと状態を保持。
  • LocalBufferLookupEnt: ハッシュテーブルのキーとバッファ ID を対応付ける構造体。
  • LocalBufHash: ローカルバッファ検索用のハッシュテーブル。

失敗モードと境界条件

  • LocalBufHash が既に存在する場合に HASH_ENTER で衝突が発生した場合は、ハッシュテーブルの破損とみなして elog(ERROR) を発生させます。

保守担当者向けメモ

  • 本関数はプロセスローカルなバッファを扱うため、共有バッファのような LWLock は不要ですが、アトミック操作による状態更新が行われています。
  • 並列クエリのワーカープロセスからは呼び出せない設計となっています（InitLocalBuffers 内で制限）。
