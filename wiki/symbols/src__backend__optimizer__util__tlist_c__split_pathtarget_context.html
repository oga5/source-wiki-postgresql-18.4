split_pathtarget_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/util/tlist.c:47

目的

SRF（セットを返す関数）を含む式をプランナが実行可能な複数のレベルに分解する際、再帰的な走査状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    PlannerInfo *root;
    bool        is_grouping_target;
    List       *input_target_exprs;
    List       *level_srfs;
    List       *level_input_vars;
    List       *level_input_srfs;
    List       *current_input_vars;
    List       *current_input_srfs;
    int         current_depth;
    Index       current_sgref;
} split_pathtarget_context;

入力と出力

  • **入力**: PlannerInfo へのポインタ、グループ化ターゲットかどうかのフラグ。
  • **出力**: 走査過程で蓄積された SRF の評価レベル情報（level_srfs, level_input_vars, level_input_srfs）が格納されます。

前提条件と不変条件

  • split_pathtarget_walker による式ツリーの走査中に使用されます。
  • current_depth は現在のサブ式における SRF の最大ネスト深さを追跡します。

アルゴリズムまたは振る舞い

1. split_pathtarget_walker が式ツリーを再帰的に走査する際、この構造体を通じて現在の SRF 依存関係を追跡します。 2. level_srfs 等のリストに、各実行レベルで必要な SRF や入力変数を蓄積します。 3. 走査終了後、蓄積された情報に基づき、複数の PathTarget を生成して実行計画を再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: split_pathtarget_at_srfs（src/backend/optimizer/util/tlist.c:947）にてインスタンス化されます。
  • **呼び出し先**: split_pathtarget_walker（src/backend/optimizer/util/tlist.c:1144）に引数として渡され、走査中に更新されます。

関連データ構造

  • PlannerInfo: プランナのグローバル情報。
  • PathTarget: 本構造体によって生成・管理されるターゲットリスト表現。
  • split_pathtarget_item: level_srfs 等のリスト要素として使用される補助構造体。

失敗モードと境界条件

  • 供給されたソース情報には、特定の失敗モードや境界条件に関する明示的な記述はありません。

保守担当者向けメモ

  • 本構造体は SRF のネストを分解するための状態保持に特化しています。SRF の評価順序や依存関係のロジックを変更する場合は、current_depth や各 level_* リストの更新ロジックを慎重に確認してください。
