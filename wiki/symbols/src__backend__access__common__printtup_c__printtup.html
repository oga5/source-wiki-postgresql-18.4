printtup

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/printtup.c:303

目的

クエリ結果のタプルをフロントエンド（クライアント）へ送信するために、PostgreSQL の DataRow メッセージ形式へ変換・シリアライズする。

シグネチャまたは形状

static bool printtup(TupleTableSlot *slot, DestReceiver *self);

入力と出力

  • **入力**:
  • slot: 送信対象のタプルを含む TupleTableSlot。
  • self: DestReceiver インターフェースを実装した DR_printtup 構造体。
  • **出力**:
  • 常に true を返す。
  • 副作用として、フロントエンドへ DataRow メッセージが送信される。

前提条件と不変条件

  • self は DR_printtup 型としてキャスト可能であること。
  • slot には有効な TupleDesc が設定されていること。
  • myState->tmpcontext が初期化されていること。

アルゴリズムまたは振る舞い

1. **準備**: myState->attrinfo が現在のタプル記述子と一致しない場合、printtup_prepare_info を呼び出し、各属性の出力関数をキャッシュする。 2. **デコード**: slot_getallattrs(slot) を呼び出し、タプル内の全属性を展開する。 3. **メモリ管理**: 行ごとの処理用メモリコンテキスト (tmpcontext) に切り替え、処理後にリセットすることでメモリリークを防ぐ。 4. **メッセージ構築**:

  • pq_beginmessage_reuse で DataRow メッセージを開始。
  • 各属性をループ処理し、NULL 値であれば -1 を送信。
  • 非 NULL 値の場合、キャッシュされた関数（テキスト形式なら typoutput、バイナリ形式なら typsend）を用いて変換し、バッファへ書き込む。

    • 5. **送信**: pq_endmessage_reuse でメッセージを確定・送信し、メモリコンテキストをリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（DestReceiver インターフェース経由で呼び出される）。
  • **呼び出し先**:
  • printtup_prepare_info
  • slot_getallattrs
  • MemoryContextSwitchTo / MemoryContextReset
  • pq_beginmessage_reuse / pq_endmessage_reuse
  • OutputFunctionCall / SendFunctionCall
  • VALGRIND_CHECK_MEM_IS_DEFINED

関連データ構造

  • DR_printtup: printtup の状態を保持する構造体。
  • TupleTableSlot: 送信対象のタプルデータ。
  • PrinttupAttrInfo: 各カラムの型情報や出力関数を保持するキャッシュ。

失敗モードと境界条件

  • **NULL 値**: tts_isnull が真の場合、値の代わりに -1 を送信する。
  • **可変長データ**: typisvarlena が真の場合、Valgrind を用いてメモリ定義チェックを行う。
  • **メモリ**: tmpcontext を使用することで、大量のタプル処理時もメモリ消費を一定に保つ。

保守担当者向けメモ

  • 本関数は DestReceiver の receiveSlot に登録される。
  • パフォーマンス向上のため、行ごとのメモリ確保を避けて tmpcontext のリセットを行っている点に注意すること。
  • format フィールドが 0 の場合はテキスト出力、それ以外はバイナリ出力として扱われる。
