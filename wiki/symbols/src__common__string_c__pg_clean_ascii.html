pg_clean_ascii

定義

  • 種別: function
  • 定義箇所: src/common/string.c:84

目的

入力文字列に含まれる非 ASCII 文字（ASCII 範囲外の文字）をエスケープシーケンス \xXX に置換し、ログ出力等に適した安全な文字列を生成する。

シグネチャまたは形状

char *pg_clean_ascii(const char *str, int alloc_flags)

入力と出力

  • **入力**:
  • str: 変換対象のヌル終端文字列。
  • alloc_flags: メモリ確保時のフラグ（バックエンド実行時のみ使用）。
  • **出力**:
  • 変換後の新しい文字列へのポインタ。メモリ確保に失敗した場合は NULL を返す。

前提条件と不変条件

  • str は有効なヌル終端文字列であること。
  • 戻り値のメモリ所有権は呼び出し元に帰属する。
  • FRONTEND マクロの定義に応じて、malloc または palloc_extended が使用される。

アルゴリズムまたは振る舞い

1. 入力文字列の最大長を strlen(str) * 4 + 1 と見積もり、メモリを確保する。 2. 入力文字列を 1 バイトずつ走査する。 3. 各文字が ASCII 範囲内（32 以上 126 以下）であれば、そのまま出力バッファにコピーする。 4. 範囲外であれば、snprintf を使用して \xXX 形式の 16 進数表現に変換し、出力バッファに書き込む。 5. 最後にヌル終端文字を付与して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/variable.c
  • src/backend/libpq/be-secure-openssl.c
  • src/backend/tcop/backend_startup.c
  • **呼び出し先**:
  • strlen (標準ライブラリ)
  • malloc または palloc_extended (環境依存)
  • snprintf (標準ライブラリ)
  • Assert (デバッグ用マクロ)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • メモリ確保失敗時: dst が NULL となり、関数は NULL を返す。
  • 境界条件: 入力文字列が空の場合、ヌル終端のみの文字列を返す。

保守担当者向けメモ

  • 本関数はログ出力の安全性を確保するための限定的なツールであり、汎用的な文字列フィルタリングとして使用しないこと。
  • FRONTEND マクロの有無によりメモリ確保戦略が切り替わるため、リンク環境に注意すること。
