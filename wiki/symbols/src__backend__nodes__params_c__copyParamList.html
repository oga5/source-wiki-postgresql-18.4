copyParamList

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/params.c:77

目的

動的フック（paramFetch）によって提供されるパラメータを含む ParamListInfo 構造体から、すべての値を実体化した静的なコピーを作成します。

シグネチャまたは形状

ParamListInfo copyParamList(ParamListInfo from)

入力と出力

  • **入力**: from (コピー元の ParamListInfo 構造体)
  • **出力**: 複製された ParamListInfo 構造体。入力が NULL またはパラメータ数が 0 以下の場合、NULL を返します。

前提条件と不変条件

  • CurrentMemoryContext に新しい構造体が確保されるため、呼び出し側は適切なメモリコンテキストが選択されていることを保証する必要があります。
  • 参照渡し（pass-by-reference）のデータ型については、datumCopy を用いてメモリ上の実体が複製されます。

アルゴリズムまたは振る舞い

1. from->numParams を用いて makeParamList で新しい構造体を確保します。 2. 各パラメータについて以下を順次実行します。

  • from->paramFetch が設定されている場合はそれを呼び出し、動的な値を解決します。設定されていない場合は from->params 配列から直接取得します。
  • 取得した ParamExternData を新しい構造体にフラットコピーします。
  • 値が NULL でなく、かつ型 OID が有効な場合、get_typlenbyval で型情報を取得し、datumCopy を使用して値を複製します。

    • 3. 構築された新しい ParamListInfo を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/portalcmds.c:132
  • src/backend/executor/spi.c:1766
  • **呼び出し先**:
  • makeParamList (コンストラクタ)
  • from->paramFetch (動的フックが存在する場合)
  • get_typlenbyval (型情報の取得)
  • datumCopy (値の複製)

関連データ構造

  • ParamListInfo: パラメータリストを保持する主要構造体。
  • ParamExternData: 個々のパラメータのメタデータと値を保持する構造体。

失敗モードと境界条件

  • from が NULL または numParams <= 0 の場合、処理を行わず NULL を返します。
  • ptype が無効（OidIsValid が偽）な場合、datumCopy はスキップされます。

保守担当者向けメモ

  • 本関数は動的フックを解決して値を実体化するため、実行エンジンにおけるパラメータの永続化やプロセス間通信の準備として利用されます。
  • paramValuesStr（ログ出力用キャッシュ）はコピーされません。
