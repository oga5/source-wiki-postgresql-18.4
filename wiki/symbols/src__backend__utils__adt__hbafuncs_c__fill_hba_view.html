fill_hba_view

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/hbafuncs.c:392

目的

pg_hba.conf ファイルを読み込み、パースした認証ルールを Tuplestore に格納して、SQL経由で参照可能な形式（ビュー）に変換します。

シグネチャまたは形状

static void fill_hba_view(Tuplestorestate *tuple_store, TupleDesc tupdesc)

入力と出力

  • **入力**:
  • tuple_store: 結果を蓄積するための Tuplestorestate へのポインタ。
  • tupdesc: 生成するタプルの記述子。
  • **出力**: なし（tuple_store を直接更新する）。

前提条件と不変条件

  • HbaFileName で指定されたファイルが存在し、読み取り可能であること。
  • Tuplestore は呼び出し元で適切に初期化されていること。
  • パース処理は専用のメモリコンテキスト（hba parser context）内で実行され、終了時に解放されること。

アルゴリズムまたは振る舞い

1. open_auth_file を呼び出し、pg_hba.conf を開く（失敗時はエラーを投げる）。 2. tokenize_auth_file を使用してファイル内容をトークンリストに変換する。 3. AllocSetContextCreate でパース専用のメモリコンテキストを作成し、切り替える。 4. トークンリストを走査し、各行に対して以下の処理を行う：

  • エラーがない行は parse_hba_line で構造体に変換し、ルール番号をインクリメントする。
  • fill_hba_line を呼び出し、パース結果またはエラー情報を tuple_store に書き込む。

    • 5. free_auth_file でファイルを閉じ、メモリコンテキストを破棄して元のコンテキストに戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/hbafuncs.c 内の pg_hba_file_rules 関数から呼び出される。
  • **呼び出し先**:
  • open_auth_file
  • tokenize_auth_file
  • AllocSetContextCreate / MemoryContextSwitchTo / MemoryContextDelete
  • parse_hba_line
  • fill_hba_line
  • free_auth_file

関連データ構造

  • Tuplestorestate: 結果保持用構造体。
  • HbaLine: パースされた認証ルールを保持する構造体。
  • TokenizedAuthLine: トークン化された行データ。

失敗モードと境界条件

  • ファイルが開けない場合、open_auth_file がエラーを発生させ、関数は中断されます。
  • パースエラーが発生した行については、tok_line->err_msg が設定され、fill_hba_line を通じてエラー情報としてビューに出力されます。

保守担当者向けメモ

  • 本関数は pg_hba_file_rules ビューのバックエンドロジックです。
  • メモリ管理のため、パース処理は一時的なコンテキストで行われます。修正を加える際は、メモリリークを防ぐため MemoryContext の切り替えが適切に行われているか確認してください。
