left_oper

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_oper.c:517

目的

単項（前置）演算子の解決を行い、システムカタログ pg_operator から対応する演算子定義を検索・取得する。

シグネチャまたは形状

Operator left_oper(ParseState *pstate, List *op, Oid arg, bool noError, int location)

入力と出力

  • **入力**:
  • pstate: パーサーの状態（キャッシュキー生成に使用）。
  • op: 演算子名（List）。
  • arg: 演算子の右辺の型（Oid）。
  • noError: 検索失敗時にエラーを発生させるかどうかのフラグ。
  • location: 構文解析上の位置情報。
  • **出力**:
  • 成功時: 解決された演算子の Operator（HeapTuple へのポインタ）。
  • 失敗時: noError が true の場合は NULL、false の場合はエラーを報告して終了。

前提条件と不変条件

  • 演算子キャッシュ（OprCacheHash）が有効な場合、検索パスと引数型に基づき高速に解決される。
  • 候補リスト取得時、OpernameGetCandidates が返すリストの引数構造を一時的に書き換える（args[0] = args[1]）ため、リストの破壊的変更を許容する。

アルゴリズムまたは振る舞い

1. **キャッシュ検索**: make_oper_cache_key を使用し、キャッシュにヒットすればその演算子を返す。 2. **完全一致検索**: OpernameGetOprid を使用し、型が完全に一致する演算子を検索する。 3. **候補探索**: 一致しない場合、OpernameGetCandidates で候補リストを取得し、oper_select_candidate を用いて型適合性を評価する。 4. **キャッシュ更新**: 解決に成功した場合、結果をキャッシュに保存する。 5. **エラー処理**: 解決できず noError が false の場合、op_error を呼び出しエラーを報告する。

関連データ構造

  • Operator: pg_operator カタログのタプルを指す型。
  • OprCacheKey: 演算子検索のキャッシュキー。
  • FuncCandidateList: 演算子の候補リスト。

失敗モードと境界条件

  • 演算子が見つからない場合、noError が false であれば op_error によりエラーが発生する。
  • 候補が複数あり、型適合性で絞り込めない（曖昧な）場合は oper_select_candidate の結果に基づきエラーとなる。

保守担当者向けメモ

  • 本関数は単項演算子専用である。oper_select_candidate に渡す前に引数リストの調整を行っている点に注意。
  • OprCacheHash の無効化コールバック（InvalidateOprCacheCallBack）が呼ばれると、キャッシュされた演算子定義は破棄される。
