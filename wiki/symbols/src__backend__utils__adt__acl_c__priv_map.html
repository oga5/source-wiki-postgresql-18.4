priv_map

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/acl.c:55

目的

権限名（文字列）とそれに対応する内部的な権限ビットマスク（AclMode）を関連付けるためのマッピング構造体です。

シグネチャまたは形状

typedef struct
{
	const char *name;
	AclMode		value;
} priv_map;

入力と出力

  • **入力**: 権限名を表す文字列 (name) および権限ビット (value)。
  • **出力**: なし（データ構造定義のため）。

前提条件と不変条件

  • name は権限を識別する一意の文字列である必要があります。
  • value は AclMode 型のビットマスク値です。

アルゴリズムまたは振る舞い

本構造体は、特定のデータベースオブジェクト（テーブル、関数、データベース等）に対して許可される権限のリストを定義する静的配列の要素として使用されます。システムはこれらの配列を走査することで、文字列形式の権限名を内部的なビットマスクへ変換したり、その逆を行ったりします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/acl.c 内の各権限マップ定義（any_priv_map, table_priv_map, sequence_priv_map, column_priv_map, database_priv_map, foreign_data_wrapper_priv_map, function_priv_map）が本構造体を利用しています。
  • **呼び出し先**: 権限名とビットマスクの照合を行う関数（例: aclparse 等の権限名を受け取る関数）から参照されます。

関連データ構造

  • AclMode: 権限ビットを保持する型。
  • AclItem: 権限付与の最小単位を構成する構造体。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、この構造体を用いた検索処理において、定義されていない権限名が指定された場合、一致するエントリが見つからない可能性があります。

保守担当者向けメモ

  • 新しい権限タイプを追加する際は、対応する priv_map 配列を定義し、この構造体を用いて権限名とビット値をマッピングしてください。
  • 参照箇所は src/backend/utils/adt/acl.c 内の各 *_priv_map 定義に限定されています。
