ExecEndGather

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGather.c:250

目的

Gather ノードの実行を終了し、並列クエリ実行に関連するリソースを解放する。

シグネチャまたは形状

void ExecEndGather(GatherState *node)

入力と出力

  • **入力**: GatherState *node - 終了対象の Gather ノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は有効な GatherState を指している必要がある。
  • 本関数は、Gather ノードの実行終了時に呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. outerPlanState(node) を引数として ExecEndNode を呼び出し、子プランノードのクリーンアップを実行する。 2. ExecShutdownGather(node) を呼び出し、並列ワーカーの停止および共有メモリリソースの解放を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:632
  • **呼び出し先**:
  • ExecEndNode (子ノードの終了処理)
  • ExecShutdownGather (並列リソースの解放)

関連データ構造

  • GatherState: Gather ノードの実行状態を保持する構造体。

失敗モードと境界条件

  • node が NULL の場合の挙動については、本ソース抜粋からは判断できない。

保守担当者向けメモ

  • 本関数は、子ノードの終了処理を優先的に実行する順序を保証している。
  • 並列実行インフラのクリーンアップは ExecShutdownGather に委譲されているため、並列処理特有の終了ロジックを変更する場合は ExecShutdownGather を参照すること。
