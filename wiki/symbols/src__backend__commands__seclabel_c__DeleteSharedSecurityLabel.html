DeleteSharedSecurityLabel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/seclabel.c:490

目的

共有データベースオブジェクト（データベース、テーブルスペース、ロール等）に関連付けられたセキュリティラベルをシステムカタログ pg_shseclabel から削除します。

シグネチャまたは形状

void DeleteSharedSecurityLabel(Oid objectId, Oid classId)

入力と出力

  • **入力**:
  • objectId (Oid): 削除対象オブジェクトの OID。
  • classId (Oid): 対象オブジェクトが属するカタログの OID。
  • **出力**: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • pg_shseclabel システムカタログに対する RowExclusiveLock を取得して操作を行います。
  • 指定された objectId と classId に一致するすべてのラベルエントリを削除します。

アルゴリズムまたは振る舞い

1. pg_shseclabel カタログを RowExclusiveLock モードで開きます。 2. ScanKey を使用して、指定された objectId および classId に一致するエントリを検索するためのスキャンキーを初期化します。 3. systable_beginscan を用いてインデックススキャンを開始します。 4. systable_getnext で該当するタプルを順次取得し、CatalogTupleDelete を呼び出して削除します。 5. スキャンを終了し、カタログテーブルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c
  • src/backend/commands/seclabel.c (内部ヘルパーとして)
  • src/backend/commands/tablespace.c
  • src/backend/commands/user.c
  • **呼び出し先**:
  • table_open / table_close
  • systable_beginscan / systable_getnext / systable_endscan
  • CatalogTupleDelete

関連データ構造

  • pg_shseclabel: 共有オブジェクトのセキュリティラベルを保持するシステムカタログ。
  • ScanKeyData: カタログ検索条件を保持する構造体。

失敗モードと境界条件

  • 対象オブジェクトにラベルが存在しない場合、ループは実行されず、エラーにはならず正常終了します。
  • カタログ操作中にロック競合が発生した場合、PostgreSQL の標準的なロック待機動作に従います。

保守担当者向けメモ

  • 本関数は共有カタログ専用です。通常のオブジェクト（テーブルやカラム等）のラベル削除には DeleteSecurityLabel を使用してください。
  • pg_shseclabel のインデックス SharedSecLabelObjectIndexId を利用した効率的な検索を行っています。カタログ定義に変更を加える際は、このインデックスとの整合性に注意してください。
