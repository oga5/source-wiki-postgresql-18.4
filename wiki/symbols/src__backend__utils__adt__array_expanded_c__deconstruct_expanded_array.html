deconstruct_expanded_array

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_expanded.c:423

目的

フラットな配列（ArrayType）を保持している展開済み配列（ExpandedArrayHeader）に対し、要素へのアクセスを可能にするために、内部の Datum 配列および NULL フラグ配列を生成（分解）する。

シグネチャまたは形状

void deconstruct_expanded_array(ExpandedArrayHeader *eah)

入力と出力

  • **入力**: eah - 分解対象の ExpandedArrayHeader 構造体へのポインタ。
  • **出力**: なし（副作用として eah 内部のフィールドを更新）。

前提条件と不変条件

  • eah->dvalues が NULL である場合のみ分解処理が実行される（遅延評価）。
  • 分解処理は eah->hdr.eoh_context（展開済みオブジェクトのメモリコンテキスト）内で行われる。

アルゴリズムまたは振る舞い

1. eah->dvalues が既に存在するか確認し、存在しない場合のみ以下の処理を行う。 2. eah->hdr.eoh_context にメモリコンテキストを切り替える。 3. deconstruct_array を呼び出し、eah->fvalue（フラットな配列）から要素の Datum 配列と NULL フラグ配列を生成する。 4. 生成された配列ポインタ、要素数、NULL フラグ情報を eah ヘッダーに格納する。 5. 元のメモリコンテキストへ復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/extended_stats.c
  • src/backend/utils/adt/arrayfuncs.c
  • **呼び出し先**:
  • deconstruct_array（utils/array.h）
  • MemoryContextSwitchTo（utils/memutils.h）

関連データ構造

  • ExpandedArrayHeader: 展開済み配列のヘッダー。
  • ArrayType: 分解元のフラットな配列形式。

失敗モードと境界条件

  • deconstruct_array が失敗した場合、メモリリークが発生する可能性があるが、eah の状態は整合性を保つよう設計されている。
  • 入力が既に分解済み（dvalues != NULL）の場合は何も行わない。

保守担当者向けメモ

  • 本関数は遅延評価を行うため、要素アクセスが必要になる直前に呼び出される。
  • dvalues が NULL でない場合は処理をスキップするため、複数回呼び出しても安全である。
