PageIndexTupleDeleteNoCompact

定義

  • 種別: function
  • 定義箇所: src/backend/storage/page/bufpage.c:1293

目的

ページ内の指定されたタプルを削除し、その領域を解放します。PageRepairFragmentation と異なり、ページ全体の完全なデフラグは行わず、削除対象タプルの領域を詰める操作のみを行います。

シグネチャまたは形状

void PageIndexTupleDeleteNoCompact(Page page, OffsetNumber offnum)

入力と出力

  • **入力**:
  • page: 操作対象のページへのポインタ。
  • offnum: 削除するタプルのオフセット番号。
  • **出力**: なし（副作用としてページ内容を直接変更）。

前提条件と不変条件

  • ページヘッダーの pd_lower, pd_upper, pd_special が整合していること。
  • offnum が有効な範囲内であること。
  • 削除対象のタプルが ItemIdHasStorage を満たしていること。
  • ページ内のデータ構造が破損していないこと（破損時は ereport(ERROR) を発行）。

アルゴリズムまたは振る舞い

1. **検証**: ページヘッダーおよび削除対象アイテムのオフセット・サイズが妥当か確認します。 2. **アイテムIDの更新**:

  • 削除対象が最後のアイテムであれば、pd_lower を減らしてアイテムID配列を縮小します。
  • それ以外の場合は、該当するアイテムIDを unused に設定します。

    • 3. **データ領域の移動**: 削除されたタプルよりページ先頭側（pd_upper 側）にあるタプルを memmove で前方へ詰め、空いた領域を統合します。 4. **境界ポインタの更新**: pd_upper を削除サイズ分だけ増加させ、空き領域を広げます。 5. **オフセットの修正**: ページ内の残存アイテムのうち、削除されたタプルより物理的に前方にあったすべてのアイテムのオフセット（lp_off）を、削除サイズ分だけ加算して補正します。

関連データ構造

  • PageHeaderData (ページヘッダー)
  • ItemIdData (アイテムID)

失敗モードと境界条件

  • **データ破損**: ページヘッダーのポインタ値が範囲外の場合、またはアイテムのオフセットがページ境界を逸脱している場合は ERRCODE_DATA_CORRUPTED でエラー終了します。
  • **無効なオフセット**: offnum がページ内の有効な範囲外である場合、elog(ERROR) を発行します。

保守担当者向けメモ

  • 本関数は、削除対象のタプルより前方に存在するタプルのオフセットをすべて更新するため、ページ内のアイテム数が多い場合にはコストがかかります。
  • 削除対象がページ末尾のタプルである場合、memmove によるデータ移動は発生しません。
