l_int16_const

定義

  • 種別: function
  • 定義箇所: src/include/jit/llvmjit_emit.h:56

目的

LLVM IR 生成において、指定された int16 型の値を保持する定数ノード（LLVMValueRef）を生成する。

シグネチャまたは形状

static inline LLVMValueRef l_int16_const(LLVMContextRef lc, int16 i)

入力と出力

  • **入力**:
  • lc: 定数を作成する対象の LLVMContextRef。
  • i: 定数として保持する int16 型の値。
  • **出力**:
  • 指定された値と型（LLVMInt16Type）を持つ LLVMValueRef。

前提条件と不変条件

  • lc は有効な LLVMContextRef であること。
  • 本関数はインライン関数であり、呼び出し側のコンテキストで展開される。

アルゴリズムまたは振る舞い

1. LLVMInt16TypeInContext(lc) を呼び出し、LLVM コンテキスト内での 16 ビット整数型を取得する。 2. LLVMConstInt を使用して、値 i を持つ定数ノードを生成する。この際、符号付き整数として扱われる（signExtend 引数は false）。 3. 生成された LLVMValueRef を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/jit/llvm/llvmjit_deform.c (複数箇所: HEAP_HASNULL, HEAP_NATTS_MASK, natts, attnum, TTS_FLAG_SLOW 等の定数生成に使用)
  • src/backend/jit/llvm/llvmjit_expr.c (op->d.fetch.last_var の定数生成に使用)
  • **呼び出し先**:
  • LLVMInt16TypeInContext (LLVM C API)
  • LLVMConstInt (LLVM C API)

関連データ構造

  • LLVMValueRef: LLVM IR 上の値を表す不透明なポインタ。
  • LLVMContextRef: LLVM の型や定数を管理するコンテキスト。

失敗モードと境界条件

  • LLVM C API の仕様に依存する。無効なコンテキストが渡された場合、LLVM 側のエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数は USE_LLVM が定義されている環境でのみ利用可能。
  • LLVM C API のラッパーであるため、API のバージョンアップにより挙動が変更される可能性がある。定数生成の簡略化を目的としているため、ロジックの変更は最小限に留めること。
