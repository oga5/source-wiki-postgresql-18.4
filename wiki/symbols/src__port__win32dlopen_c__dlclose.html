dlclose

定義

  • 種別: function
  • 定義箇所: src/port/win32dlopen.c:48

目的

Windows環境において、POSIX標準の dlclose 関数をエミュレートし、ロード済みの共有ライブラリ（DLL）の参照カウントを減らして解放します。

シグネチャまたは形状

int dlclose(void *handle);

入力と出力

  • **入力**: handle - dlopen によって返された、解放対象のDLLハンドル（void *）。
  • **出力**: 成功時は 0 を返し、失敗時は 1 を返します。

前提条件と不変条件

  • handle は有効なDLLハンドルである必要があります。
  • 内部エラー管理用の静的バッファ last_dyn_error を使用するため、スレッドセーフではありません。

アルゴリズムまたは振る舞い

1. Windows APIの FreeLibrary を呼び出し、指定された handle の参照カウントを減らします。 2. FreeLibrary が失敗した場合（戻り値が0）、set_dl_error() を呼び出してエラー情報を記録し、1 を返します。 3. FreeLibrary が成功した場合、静的エラーバッファ last_dyn_error の先頭をヌル文字でクリアし、0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/fmgr/dfmgr.c
  • src/interfaces/libpq/fe-auth-oauth.c
  • **呼び出し先**:
  • FreeLibrary (Windows API)
  • set_dl_error (内部ヘルパー関数)

関連データ構造

  • last_dyn_error: 直近のエラーメッセージを保持する静的バッファ。

失敗モードと境界条件

  • FreeLibrary が失敗した場合、関数は 1 を返し、dlerror() を通じて取得可能なエラー情報が更新されます。

保守担当者向けメモ

  • 本関数はWindowsビルド時のみコンパイルされます。
  • last_dyn_error が静的変数であるため、マルチスレッド環境での同時呼び出しには注意が必要です。
