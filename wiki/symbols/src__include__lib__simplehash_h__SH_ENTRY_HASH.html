SH_ENTRY_HASH

定義

  • 種別: static inline function
  • 定義箇所: src/include/lib/simplehash.h:394

目的

ハッシュテーブル内の指定された要素からハッシュ値を算出または取得する。

シグネチャまたは形状

static inline uint32 SH_ENTRY_HASH(SH_TYPE * tb, SH_ELEMENT_TYPE * entry)

入力と出力

  • **入力**:
  • tb: ハッシュテーブル構造体へのポインタ。
  • entry: ハッシュ値を算出対象とする要素へのポインタ。
  • **出力**:
  • uint32: 計算されたハッシュ値。

前提条件と不変条件

  • SH_TYPE および SH_ELEMENT_TYPE は、本ヘッダーファイルがインクルードされる前にマクロ定義されている必要がある。
  • SH_STORE_HASH マクロの定義有無により、動作が分岐する。

アルゴリズムまたは振る舞い

1. SH_STORE_HASH が定義されている場合:

  • SH_GET_HASH マクロを呼び出し、要素内に格納済みのハッシュ値を直接取得して返す。

    • 2. SH_STORE_HASH が定義されていない場合:

  • 要素のキー（entry->SH_KEY）を SH_HASH_KEY マクロに渡し、ハッシュ値を再計算して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/lib/simplehash.h 内の複数の箇所（行 549, 572, 679, 899, 931, 962, 1101）で、テーブルのリサイズ、挿入、検索、削除処理の過程で呼び出される。
  • **呼び出し先**:
  • SH_GET_HASH（SH_STORE_HASH 定義時）
  • SH_HASH_KEY（SH_STORE_HASH 未定義時）

関連データ構造

  • SH_TYPE: 生成されるハッシュテーブルの型。
  • SH_ELEMENT_TYPE: ハッシュテーブルに格納される要素の型。

失敗モードと境界条件

  • entry が NULL の場合、entry->SH_KEY へのアクセスにより未定義動作が発生する可能性がある。

保守担当者向けメモ

  • 本関数はマクロによるテンプレート生成の一部であり、インライン展開されることを前提としている。
  • SH_STORE_HASH を利用すると、キーの再計算を回避できるため、検索性能が向上する。
