Oid

定義

  • 種別: typedef
  • 定義箇所: src/include/catalog/objectaccess.h:127

目的

PostgreSQLのシステムカタログ内のオブジェクトを一意に識別するための識別子型です。

シグネチャまたは形状

typedef により定義された整数型（通常は unsigned int）です。

入力と出力

  • 入力: データベース内のオブジェクト（テーブル、関数、型など）を指し示す識別子として使用されます。
  • 出力: 多くの関数において、特定のオブジェクトを特定するための引数として渡されます。

前提条件と不変条件

  • システムカタログ内の各オブジェクトに対して一意な値が割り当てられます。
  • データベースクラスタ全体で一意性が保証されるものと、データベース単位で一意なものがあります。

アルゴリズムまたは振る舞い

  • オブジェクトの検索、参照、および操作の際のキーとして機能します。
  • object_access_hook などのフック機構において、操作対象のオブジェクトを特定するために classId や objectId として渡されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/amcheck モジュール（verify_common.c, verify_gin.c, verify_heapam.c, verify_nbtree.c 等）において、リレーションやインデックスの識別に使用されています。
  • 呼び出し先: 提供されたソース excerpt には、本シンボルを呼び出す具体的な関数や、本シンボルが呼び出す関数に関する情報は含まれていません。

関連データ構造

  • ObjectAccessType: フック呼び出し時のイベント種別。
  • object_access_hook_type: Oid を引数に含むフック関数の型定義。

失敗モードと境界条件

  • 無効な Oid（InvalidOid）が指定された場合、システムはオブジェクトを見つけることができず、エラーまたは予期しない動作を引き起こす可能性があります。

保守担当者向けメモ

  • Oid は PostgreSQL のコア全体で多用される基本型です。変更はシステム全体に影響を及ぼすため注意してください。
  • フック機構で使用される際は、Oid が指すオブジェクトがトランザクション内で有効であることを確認する必要があります。
