rbound_bsearch

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_selfuncs.c:627

目的

範囲型の境界値ヒストグラム配列から、指定された値（RangeBound）を二分探索し、その位置を特定します。

シグネチャまたは形状

static int rbound_bsearch(TypeCacheEntry *typcache, const RangeBound *value, const RangeBound *hist, int hist_length, bool equal)

入力と出力

  • **入力**:
  • typcache: 型キャッシュ情報。
  • value: 検索対象の範囲境界値。
  • hist: 検索対象となる範囲境界のヒストグラム配列。
  • hist_length: ヒストグラムの要素数。
  • equal: 比較時に等価な値を含めるかどうかを指定するフラグ。
  • **出力**:
  • 検索条件を満たすヒストグラム内のインデックス（int）。見つからない場合は -1 を返します。

前提条件と不変条件

  • hist 配列は、range_cmp_bounds 関数によって定義される順序でソートされている必要があります。
  • typcache は有効な型情報を含んでいる必要があります。

アルゴリズムまたは振る舞い

1. lower を -1、upper を hist_length - 1 に初期化します。 2. lower < upper である間、以下の処理を繰り返します。

  • 中間インデックス middle を計算します。
  • range_cmp_bounds を用いて hist[middle] と value を比較します。
  • equal が true の場合、cmp <= 0 となる範囲を探索し、それ以外の場合は cmp < 0 となる範囲を探索します。

    • 3. 条件を満たす最大のインデックスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rangetypes_selfuncs.c 内の各選択率計算関数。
  • src/backend/utils/adt/multirangetypes_selfuncs.c 内の各選択率計算関数。
  • **呼び出し先**:
  • range_cmp_bounds（型固有の境界比較関数）。

関連データ構造

  • RangeBound: 範囲型の境界値を保持する構造体。
  • TypeCacheEntry: 型の比較関数やキャッシュ情報を保持する構造体。

失敗モードと境界条件

  • hist_length が 0 の場合、lower は -1 のまま返されます。
  • 検索対象の値がヒストグラムの全要素よりも小さい場合、-1 が返されます。

保守担当者向けメモ

  • 本関数は rangetypes_selfuncs.c と multirangetypes_selfuncs.c の両方から利用されています。修正時は両方のモジュールへの影響を考慮してください。
  • 比較ロジックは range_cmp_bounds に完全に依存しているため、カスタム範囲型を定義する際は当該関数の実装が重要です。
