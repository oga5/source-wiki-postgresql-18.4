basebackup_progress_estimate_backup_size

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_progress.c:195

目的

ベースバックアップの進捗状況において、バックアップサイズの見積もりフェーズへ遷移したことを統計システムに通知する。

シグネチャまたは形状

void basebackup_progress_estimate_backup_size(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として pgstat の状態を更新）

前提条件と不変条件

  • pgstat_progress_start_command が既に呼び出され、進捗追跡が開始されている必要がある。

アルゴリズムまたは振る舞い

1. pgstat_progress_update_param を呼び出す。 2. PROGRESS_BASEBACKUP_PHASE パラメータを PROGRESS_BASEBACKUP_PHASE_ESTIMATE_BACKUP_SIZE に設定し、現在のバックアップフェーズを更新する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/backup/basebackup.c:302
  • 呼び出し先: pgstat_progress_update_param（pgstat.h で定義）

関連データ構造

  • PROGRESS_BASEBACKUP_PHASE: 進捗ビューにおけるフェーズ識別子。
  • PROGRESS_BASEBACKUP_PHASE_ESTIMATE_BACKUP_SIZE: フェーズ定数。

失敗モードと境界条件

  • 統計システムが初期化されていない場合、pgstat 関連の関数呼び出しが無視される可能性がある。

保守担当者向けメモ

  • 本関数は pg_stat_progress_basebackup ビューのフェーズ列を更新する役割を持つ。
  • バックアップのメインロジック（basebackup.c）から、サイズ見積もり処理の開始時に呼び出されることを前提としている。
