brin_page_type

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/brinfuncs.c:43

目的

指定された bytea 形式のページデータが、どの種類の BRIN (Block Range Index) ページであるかを判定し、その型をテキストとして返します。

シグネチャまたは形状

Datum brin_page_type(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (SQL 経由で渡される bytea 型のページデータ)
  • **出力**: text 型（"meta", "revmap", "regular", または "unknown (XX)" のいずれか）

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要です。
  • 入力ページは BrinSpecialSpace 構造体として有効なサイズ（MAXALIGN(sizeof(BrinSpecialSpace))）を持つ必要があります。

アルゴリズムまたは振る舞い

1. superuser() を呼び出し、権限を確認します。失敗した場合はエラーを報告します。 2. get_page_from_raw() を使用して bytea を Page 構造体に変換します。 3. PageIsNew() でページが空か確認し、空であれば NULL を返します。 4. PageGetSpecialSize() でページサイズを検証し、BRIN ページとして不正な場合はエラーを報告します。 5. BrinPageType() マクロを使用してページ種別を判定し、以下のいずれかの文字列を返します。

  • BRIN_PAGETYPE_META → "meta"
  • BRIN_PAGETYPE_REVMAP → "revmap"
  • BRIN_PAGETYPE_REGULAR → "regular"
  • その他 → "unknown (XX)" (XX は 16 進数表記の型コード)

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: superuser(), get_page_from_raw(), PageIsNew(), PageGetSpecialSize(), BrinPageType(), PG_RETURN_TEXT_P(), cstring_to_text(), psprintf()。

関連データ構造

  • BrinSpecialSpace: BRIN ページの特殊領域の構造体。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーとなります。
  • **不正なページサイズ**: BRIN ページとして期待される特殊領域サイズと一致しない場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーとなります。
  • **空ページ**: PageIsNew の場合、NULL を返します。

保守担当者向けメモ

  • 本関数は生のページデータを直接扱うため、デバッグ目的での使用を想定しています。
  • ページ種別の判定ロジックは access/brin_page.h で定義されている BrinPageType マクロに依存しています。将来的に新しいページ型が追加された場合は、switch 文の更新が必要です。
