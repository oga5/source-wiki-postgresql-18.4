DeconstructFkConstraintRow

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_constraint.c:1535

目的

pg_constraint カタログのタプルから、外部キー制約に関連する複雑な属性（列番号、演算子、削除時の動作設定）を抽出し、呼び出し元が利用可能な形式に分解する。

シグネチャまたは形状

void DeconstructFkConstraintRow(HeapTuple tuple, int *numfks,
                                 AttrNumber *conkey, AttrNumber *confkey,
                                 Oid *pf_eq_oprs, Oid *pp_eq_oprs, Oid *ff_eq_oprs,
                                 int *num_fk_del_set_cols, AttrNumber *fk_del_set_cols)

入力と出力

  • **入力**:
  • tuple: pg_constraint の行データ（HeapTuple）。
  • pf_eq_oprs, pp_eq_oprs, ff_eq_oprs, fk_del_set_cols: 各種メタデータを受け取るためのバッファポインタ（NULL許容）。
  • **出力**:
  • numfks: 制約に含まれる列数。
  • conkey, confkey: 参照元および参照先の列番号配列。
  • pf_eq_oprs, pp_eq_oprs, ff_eq_oprs: 演算子 OID 配列。
  • num_fk_del_set_cols, fk_del_set_cols: 削除時に影響を受ける列の数と列番号配列。

前提条件と不変条件

  • 入力タプルは pg_constraint カタログの有効な行であること。
  • conkey および confkey は、少なくとも INDEX_MAX_KEYS 分の領域を確保した配列であること。
  • 抽出対象の配列属性（conkey, confkey, conpfeqop 等）は、1次元の配列としてカタログに格納されていること。

アルゴリズムまたは振る舞い

1. SysCacheGetAttrNotNull を使用して pg_constraint から各属性を取得する。 2. DatumGetArrayTypeP で配列をデトーストし、次元数、NULLの有無、要素型（INT2OID または OIDOID）を検証する。 3. memcpy を用いて、カタログ上の配列データを呼び出し元が提供したバッファへコピーする。 4. confdelsetcols 属性については NULL の可能性があるため、SysCacheGetAttr で確認し、NULL の場合は num_fk_del_set_cols を 0 に設定する。 5. デトーストによってコピーが生成された場合、pfree でメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/utils/adt/ri_triggers.c
  • src/backend/utils/cache/relcache.c
  • **呼び出し先**:
  • SysCacheGetAttr / SysCacheGetAttrNotNull
  • DatumGetArrayTypeP
  • pfree

関連データ構造

  • HeapTuple: カタログ行の表現。
  • pg_constraint: 外部キー制約のメタデータを保持するシステムカタログ。

失敗モードと境界条件

  • **エラー**: 属性が期待される型（INT2OID や OIDOID）でない場合、または配列が1次元でない場合、elog(ERROR) を発生させる。
  • **境界**: 列数が 0 以下、または INDEX_MAX_KEYS を超える場合はエラーとなる。

保守担当者向けメモ

  • 本関数は pg_constraint のスキーマ変更（特に配列属性の型や順序）に強く依存している。
  • TryReuseForeignKey 等の関連ロジックを変更する際は、本関数の検証ロジックと整合性が取れているか確認すること。
