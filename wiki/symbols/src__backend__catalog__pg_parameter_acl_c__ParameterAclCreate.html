ParameterAclCreate

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_parameter_acl.c:67

目的

システムカタログ pg_parameter_acl に、指定されたGUCパラメータ名に対応する新しいACLエントリを挿入します。

シグネチャまたは形状

Oid ParameterAclCreate(const char *parameter)

入力と出力

  • **入力**: parameter (GUCパラメータ名を表す文字列)
  • **出力**: 作成されたエントリのOID (Oid)

前提条件と不変条件

  • 呼び出し側は、対象のパラメータが既に存在しないことを確認している必要があります。
  • pg_parameter_acl は共有カタログであるため、マルチテナント環境や共有メモリに関連する挙動に注意が必要です。
  • パラメータ名は check_GUC_name_for_parameter_acl による妥当性検証を通過する必要があります。

アルゴリズムまたは振る舞い

1. check_GUC_name_for_parameter_acl を呼び出し、パラメータ名の妥当性を検証します。 2. convert_GUC_name_for_parameter_acl を呼び出し、カタログ格納用の形式に変換します。 3. pg_parameter_acl を RowExclusiveLock モードで開きます。 4. GetNewOidWithIndex を使用して新しいOIDを生成します。 5. heap_form_tuple を用いて、OIDと変換後のパラメータ名を格納し、ACLフィールドをNULLに設定したタプルを構築します。 6. CatalogTupleInsert でタプルを挿入します。 7. テーブルを閉じ、OIDを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c:764
  • **呼び出し先**:
  • check_GUC_name_for_parameter_acl
  • convert_GUC_name_for_parameter_acl
  • table_open / table_close
  • GetNewOidWithIndex
  • heap_form_tuple / heap_freetuple
  • CatalogTupleInsert

関連データ構造

  • pg_parameter_acl: 操作対象のシステムカタログ。
  • FormData_pg_parameter_acl: カタログのスキーマ定義。

失敗モードと境界条件

  • パラメータ名が不正な場合、check_GUC_name_for_parameter_acl によりエラーが発生します。
  • 同名のパラメータが既に存在する場合、一意インデックス制約により挿入が失敗する可能性があります（本関数は排他ロックによる競合防止を行わず、インデックスの制約に依存しています）。

保守担当者向けメモ

  • 本関数は、呼び出し側が同名のパラメータが存在しないことを確認している前提で設計されています。
  • テーブルのロックは table_close 後もコミットまで保持されます。
