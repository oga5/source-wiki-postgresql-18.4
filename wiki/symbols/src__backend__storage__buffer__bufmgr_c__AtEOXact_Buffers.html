AtEOXact_Buffers

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/bufmgr.c:3990

目的

トランザクション終了時におけるバッファ管理のクリーンアップ処理。

シグネチャまたは形状

void AtEOXact_Buffers(bool isCommit)

入力と出力

  • **入力**:
  • isCommit (bool): トランザクションがコミットされた場合は true、アボートされた場合は false を指定する。
  • **出力**: なし（void）。

前提条件と不変条件

  • PrivateRefCountOverflowed が 0 であること（Assert により検証）。

アルゴリズムまたは振る舞い

1. CheckForBufferLeaks() を呼び出し、現在のバックエンドでピン留めされたままのバッファがないか確認する。 2. AtEOXact_LocalBuffers(isCommit) を呼び出し、ローカルバッファの終了処理を実行する。 3. 最後に PrivateRefCountOverflowed が 0 であることを検証する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c (トランザクション終了処理)
  • src/backend/postmaster/autovacuum.c
  • src/backend/postmaster/bgwriter.c
  • src/backend/postmaster/checkpointer.c
  • src/backend/postmaster/walwriter.c
  • **呼び出し先**:
  • CheckForBufferLeaks()
  • AtEOXact_LocalBuffers(bool)

関連データ構造

  • PrivateRefCountEntry (バックエンドごとのピン留め回数追跡に使用)

失敗モードと境界条件

  • ピン留めされたバッファが残っている場合、CheckForBufferLeaks() 内でエラーが発生する可能性がある。
  • PrivateRefCountOverflowed が 0 でない場合、Assert 違反により異常終了する。

保守担当者向けメモ

  • 本関数はトランザクション境界でのクリーンアップを担うため、共有バッファのピン留めリークを検出する重要な役割を持つ。
  • ローカルバッファの処理については AtEOXact_LocalBuffers に委譲されている。
