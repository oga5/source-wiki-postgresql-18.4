gen_partprune_steps

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partprune.c:743

目的

クエリの WHERE 句（clauses）を解析し、パーティションプルーニングを実行するためのステップリストを生成する。

シグネチャまたは形状

static void gen_partprune_steps(RelOptInfo *rel, List *clauses, PartClauseTarget target, GeneratePruningStepsContext *context)

入力と出力

  • **入力**:
  • rel: 対象となるパーティションテーブルの RelOptInfo。
  • clauses: プルーニングの判断材料となる RestrictInfo リスト。
  • target: プルーニングの対象フェーズ（プランニング時か実行時かを示す PartClauseTarget）。
  • context: ステップ生成の状態を保持する GeneratePruningStepsContext 構造体（出力先）。
  • **出力**:
  • context 引数を通じて、生成されたプルーニングステップのリストおよびメタデータが書き込まれる。

前提条件と不変条件

  • context は呼び出し前にメモリ確保されている必要がある。
  • rel->boundinfo が有効であること。

アルゴリズムまたは振る舞い

1. context をゼロクリアし、rel と target を設定する。 2. 対象テーブルがデフォルトパーティションを持ち、かつ partition_qual（親テーブルからの制約）が存在する場合、clauses に partition_qual を追加する（デフォルトパーティションのプルーニングを可能にするため）。 3. gen_partprune_steps_internal を呼び出し、再帰的に条件式を走査してプルーニングステップを構築する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/partitioning/partprune.c 内の複数箇所（プランニング時および実行時のプルーニング準備プロセス）。
  • **呼び出し先**:
  • gen_partprune_steps_internal（直接の内部呼び出し）。

関連データ構造

  • RelOptInfo: パーティションテーブルのプランニング情報。
  • GeneratePruningStepsContext: ステップ生成中の状態を保持する構造体。
  • PartClauseTarget: プルーニングの適用フェーズを定義する列挙型。

失敗モードと境界条件

  • clauses が空の場合、プルーニングステップは生成されない。
  • partition_bound_has_default が真でない場合、partition_qual の追加は行われない。

保守担当者向けメモ

  • 本関数はプルーニングの入り口であり、実際の論理構築は gen_partprune_steps_internal に委譲されている。
  • partition_qual の結合は list_concat_copy を使用しており、元のリストを破壊しない設計となっている。
