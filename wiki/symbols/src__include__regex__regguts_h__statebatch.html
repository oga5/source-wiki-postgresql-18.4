statebatch

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regguts.h:345

目的

正規表現のコンパイル時における NFA（非決定性有限オートマトン）の「状態（state）」を、個別に確保するのではなく、一括してメモリ割り当てを行うためのバッファ構造体です。

シグネチャまたは形状

struct statebatch {
    struct statebatch *next;
    size_t nstates;
    struct state s[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: STATEBATCHSIZE(n) マクロを用いて、確保したい状態数 n に応じたメモリサイズを計算し、MALLOC で割り当てます。
  • **出力**: 割り当てられたメモリブロックの先頭アドレスが struct statebatch * として返されます。

前提条件と不変条件

  • s メンバはフレキシブル配列メンバとして定義されており、構造体の末尾に可変長の state 配列が続きます。
  • next メンバにより、複数のバッチが連結されたリスト構造を形成します。

アルゴリズムまたは振る舞い

1. regc_nfa.c 内で、新しい状態が必要になった際に既存のバッチに空きがあるかを確認します。 2. 空きがない場合、STATEBATCHSIZE を用いて必要なメモリ量を算出し、新しい statebatch を確保します。 3. 確保されたバッチは、guts 構造体内の lastsb 等で管理されるリストに連結されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_nfa.c において、状態の動的確保時に使用されます。
  • **呼び出し先**: MALLOC マクロ（メモリ確保）、および struct state（配列要素として）。

関連データ構造

  • struct state: バッチ内に格納される個々の状態オブジェクト。
  • struct nfa: statebatch を用いて状態を管理する上位の構造体。
  • struct guts: lastsb メンバを通じて、確保された statebatch のリストを保持します。

失敗モードと境界条件

  • MALLOC が失敗した場合、メモリ不足により正規表現のコンパイルが中断されます。
  • STATEBATCHSIZE の計算時にオーバーフローが発生しないよう、nstates の値は適切に制御される必要があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、sizeof(struct statebatch) は配列部分を含まないサイズとなります。メモリ確保時には必ず STATEBATCHSIZE マクロを使用してください。
  • next ポインタによる連結リストは、コンパイル終了後のメモリ解放時に全バッチを辿るために使用されます。
