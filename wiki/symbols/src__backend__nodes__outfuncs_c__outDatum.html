outDatum

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/outfuncs.c:346

目的

PostgreSQLの Datum 値を、デバッグやシリアライズに適したASCII文字列形式（length [ byte1 byte2 ... ]）に変換し、StringInfo バッファへ書き込みます。

シグネチャまたは形状

void outDatum(StringInfo str, Datum value, int typlen, bool typbyval)

入力と出力

  • **入力**:
  • str: 出力先の StringInfo バッファ。
  • value: シリアライズ対象の Datum。
  • typlen: 型の長さ（バイト単位）。
  • typbyval: 値が値渡し（by-value）かどうかを示すフラグ。
  • **出力**: なし（str が更新される）。

前提条件と不変条件

  • str は初期化済みであること。
  • typlen および typbyval は、対象となる Datum の型定義と一致していること。

アルゴリズムまたは振る舞い

1. datumGetSize を用いて value のバイト長を取得します。 2. typbyval が真の場合:

  • value をメモリ上のバイト列として扱い、sizeof(Datum) 分のバイトを str に書き込みます。

    • 3. typbyval が偽の場合:

  • DatumGetPointer でポインタを取得します。
  • ポインタが無効な場合は "0 [ ]" を書き込みます。
  • 有効な場合は、length 分のバイトを str に書き込みます。

    • 4. いずれの場合も、形式は 長さ [ バイト1 バイト2 ... ] となります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/outfuncs.c 内のノード出力処理（例: Const ノードのシリアライズ時）。
  • **呼び出し先**:
  • datumGetSize
  • appendStringInfo
  • appendStringInfoChar
  • appendStringInfoString
  • DatumGetPointer
  • PointerIsValid

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • StringInfo: 文字列構築用バッファ。

失敗モードと境界条件

  • typbyval が偽でポインタが NULL（PointerIsValid が偽）の場合、0 [ ] として安全に処理されます。
  • typlen が不正確な場合、メモリ上の不正な領域を読み取る可能性があります。

保守担当者向けメモ

  • 本関数は readfuncs.c 側の対応する読み込み関数と整合性を保つ必要があります。
  • typbyval の判定ロジックは、PostgreSQLの型システムにおける値の格納方式に依存しているため、変更時は注意してください。
