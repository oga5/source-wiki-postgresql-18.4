func_get_detail

定義

  • **種別**: function
  • **定義箇所**: src/backend/parser/parse_func.c:1394

目的

SQLクエリ内の関数呼び出しに対し、システムカタログから関数定義を検索し、型解決および候補の絞り込みを行う。関数として解決できない場合は、型変換（キャスト）としての解釈を試みる。

シグネチャまたは形状

FuncDetailCode func_get_detail(List *funcname, List *fargs, List *fargnames, int nargs, Oid *argtypes, bool expand_variadic, bool expand_defaults, bool include_out_arguments, Oid *funcid, Oid *rettype, bool *retset, int *nvargs, Oid *vatype, Oid **true_typeids, List **argdefaults);

入力と出力

  • **入力**: 関数名、引数リスト、引数型配列、および検索オプション（可変長引数やデフォルト値の展開フラグ等）。
  • **出力**:
  • 戻り値: FuncDetailCode（関数、型変換、またはエラー状態を示す列挙型）。
  • 引数ポインタ経由: 関数OID、戻り値型、セット返しフラグ、可変長引数情報、真の型ID配列、デフォルト引数リスト。

前提条件と不変条件

  • 出力引数（funcid, rettype 等）は、関数呼び出し前に呼び出し元で初期化される必要がある（本関数内でも初期化を実行）。
  • nargs == 0 の場合、argtypes は NULL であってもよい。
  • FUNCDETAIL_COERCION を返す場合、呼び出し側は coerce_type を適用する。無限再帰を避けるため、coerce_type が扱えないケースでこの値を返してはならない。

アルゴリズムまたは振る舞い

1. **初期化**: 出力引数をデフォルト値（InvalidOid 等）で初期化。 2. **候補検索**: FuncnameGetCandidates を呼び出し、名前空間から候補リストを取得。 3. **完全一致検索**: 引数型が完全に一致する候補を優先的に探索。 4. **型変換の検討**: 完全一致がない場合、単一引数の関数呼び出しであれば、関数名を型名と見なして型変換（RELABELTYPE または COERCEVIAIO）が可能か判定。 5. **候補絞り込み**: 候補が複数ある場合、func_match_argtypes で型適合性を確認し、func_select_candidate で最適な一つを選択。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_func.c (ParseFuncOrColumn 等)
  • src/backend/catalog/pg_aggregate.c
  • **呼び出し先**:
  • FuncnameGetCandidates
  • FuncNameAsType
  • find_coercion_pathway
  • func_match_argtypes
  • func_select_candidate

関連データ構造

  • FuncDetailCode: 検索結果の状態を示す列挙型。
  • FuncCandidateList: 関数候補のリスト構造。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • **曖昧な呼び出し**: 複数の候補があり、かつ func_select_candidate で解決できない場合は FUNCDETAIL_MULTIPLE を返す。
  • **型変換の制限**: COERCEVIAIO において、複合型から文字列型への変換など、誤用されやすいケースは明示的に拒否する。
  • **未知の型**: UNKNOWNOID を持つ定数引数は、型変換の文脈で特別に扱われる。

保守担当者向けメモ

  • 本関数は ParseFuncOrColumn の中心的なロジックであり、型解決の優先順位（関数 vs キャスト）を決定する重要な箇所です。
  • func_select_candidate のヒューリスティックを変更する際は、既存のクエリの解決結果に影響を与える可能性があるため注意してください。
