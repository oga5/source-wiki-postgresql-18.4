readDatum

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/readfuncs.c:599

目的

シリアライズされた文字列形式から Const ノード内の Datum 値を復元します。

シグネチャまたは形状

Datum readDatum(bool typbyval)

入力と出力

  • **入力**: typbyval（値が値渡し（by-value）される型かどうかを示すフラグ）
  • **出力**: 復元された Datum 値

前提条件と不変条件

  • pg_strtok を使用して、シリアライズされた文字列からトークンを順次読み取ります。
  • 文字列形式は length [ byte1 byte2 ... ] の構造に従っている必要があります。
  • [ で開始し ] で終了することが期待されます。

アルゴリズムまたは振る舞い

1. pg_strtok でデータ長を読み取ります。 2. [ トークンを確認します。 3. typbyval が真の場合:

  • Datum 型のサイズ（sizeof(Datum)）を超えないことを確認します。
  • 続くバイト列を読み取り、Datum 変数に直接格納します。

    • 4. typbyval が偽の場合:

  • length が 0 以下なら (Datum) NULL を返します。
  • それ以外の場合、palloc でメモリを確保し、バイト列を読み込んで PointerGetDatum でラップして返します。

    • 5. 最後に ] トークンを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/readfuncs.c:281（Const ノードの読み込み処理内）
  • **呼び出し先**: pg_strtok（字句解析用）、atoui、atoi、palloc、PointerGetDatum、elog

関連データ構造

  • Datum: PostgreSQL の汎用データ表現型。
  • Const: 本関数が値を復元する対象となるノード構造体。

失敗モードと境界条件

  • 期待されるトークン（[ や ]）が存在しない、または形式が不正な場合は elog(ERROR) を発生させます。
  • typbyval が真であるにもかかわらず、指定された length が sizeof(Datum) を超える場合はエラーとなります。

保守担当者向けメモ

  • 本関数は readfuncs.c 内の他の _read 関数と同様、pg_strtok の呼び出し順序に強く依存しています。シリアライズ形式（outfuncs.c 側）を変更する場合は、必ず本関数の読み込みロジックと整合性を取ってください。
