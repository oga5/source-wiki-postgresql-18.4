IpcMemoryState

定義

  • 種別: enum
  • 定義箇所: src/backend/port/sysv_shmem.c:84

目的

System V 共有メモリセグメントの状態を分類し、現在のデータディレクトリ（DataDir）との関連性およびプロセスのアタッチ状況を判定するために使用されます。

シグネチャまたは形状

typedef enum
{
    SHMSTATE_ANALYSIS_FAILURE, /* IDの解析に失敗 */
    SHMSTATE_ATTACHED,         /* DataDirに関連し、アタッチ中のPIDが存在 */
    SHMSTATE_ENOENT,           /* 指定IDのセグメントが存在しない */
    SHMSTATE_FOREIGN,          /* 存在はするがDataDirとは無関係 */
    SHMSTATE_UNATTACHED        /* DataDirに関連するが、アタッチ中のPIDは存在しない */
} IpcMemoryState;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: なし（列挙型定義）

前提条件と不変条件

  • 各列挙値は、shmctl(IPC_STAT) を用いたセグメントのメタデータ確認結果に基づいています。
  • SHMSTATE_ATTACHED および SHMSTATE_UNATTACHED は、対象セグメントが現在の DataDir に関連付けられていることを前提とします。

アルゴリズムまたは振る舞い

共有メモリセグメントのライフサイクル管理において、以下の判定基準として機能します。 1. SHMSTATE_ANALYSIS_FAILURE: システムコールエラー等により状態を特定できない場合。 2. SHMSTATE_ATTACHED: shm_nattch が 0 より大きく、かつ DataDir との整合性が取れている状態。 3. SHMSTATE_ENOENT: shmget または shmctl が ENOENT を返す場合。 4. SHMSTATE_FOREIGN: セグメントは存在するが、現在の DataDir とは一致しない場合。 5. SHMSTATE_UNATTACHED: セグメントは存在するが、アタッチ中のプロセスが存在しない場合。

呼び出し元と呼び出し先

  • **呼び出し元**: PGSharedMemoryAttach および共有メモリの生存確認を行う内部関数（src/backend/port/sysv_shmem.c 内の各所）。
  • **呼び出し先**: 本シンボルは列挙型であるため、直接的な呼び出し先はありません。

関連データ構造

  • IpcMemoryId: セグメント識別子。
  • IpcMemoryKey: セグメントのキー。

失敗モードと境界条件

  • SHMSTATE_ANALYSIS_FAILURE は、OS 側の権限不足やリソース枯渇により shmctl が失敗した場合に発生します。

保守担当者向けメモ

  • 新しい状態を追加する場合は、PGSharedMemoryAttach 内の判定ロジックを更新し、すべての分岐で網羅的に処理されていることを確認してください。
