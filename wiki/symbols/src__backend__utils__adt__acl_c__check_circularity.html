check_circularity

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/acl.c:1222

目的

権限（GRANT OPTION）の付与において、循環参照が発生しないかを検証する。付与者（grantor）が、自身が持たない権限を被付与者（grantee）に付与しようとする不正な操作を防止する。

シグネチャまたは形状

static void check_circularity(const Acl *old_acl, const AclItem *mod_aip, Oid ownerId)

入力と出力

  • old_acl: 変更前の ACL 配列。
  • mod_aip: 追加・変更しようとしている AclItem。
  • ownerId: オブジェクト所有者の OID。
  • 戻り値: なし。
  • 副作用: 循環参照が検出された場合、ereport(ERROR, ...) によりトランザクションを中断する。

前提条件と不変条件

  • mod_aip->ai_grantee は ACL_ID_PUBLIC であってはならない。
  • mod_aip->ai_grantor が ownerId と一致する場合、所有者は常に権限を持つためチェックをスキップする。

アルゴリズムまたは振る舞い

1. check_acl を呼び出し、old_acl の整合性を検証する。 2. old_acl の作業用コピーを作成する。 3. cc_restart ループにて、対象の被付与者が持つ既存の権限（GRANT OPTION を含む）を aclupdate を用いて再帰的に剥奪（DROP_CASCADE）し、影響範囲をシミュレーションする。 4. 権限剥奪後の ACL を基に aclmask を呼び出し、付与者が現在保持している権限を計算する。 5. 付与者が付与しようとしている権限が、自身の保持する権限の範囲を超えている場合、ERRCODE_INVALID_GRANT_OPERATION エラーを発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/acl.c 内の ACL 更新処理（aclupdate 関連のフロー）。
  • 呼び出し先: check_acl, allocacl, ACL_NUM, ACL_DAT, aclupdate, aclmask, pfree, ereport。

関連データ構造

  • Acl: アクセス制御リストの配列型。
  • AclItem: 権限付与の最小単位（被付与者、付与者、権限ビット）。

失敗モードと境界条件

  • 付与者が自分自身に対して、自分が持っていない権限を付与しようとした場合、エラーとなる。
  • mod_aip->ai_grantee が PUBLIC の場合はアサーション違反となる。

保守担当者向けメモ

  • 本関数は aclupdate による権限変更の過程で呼び出される。
  • DROP_CASCADE を伴う再帰的な権限剥奪のシミュレーションを行うため、大規模な ACL では計算コストに注意が必要。
  • goto 文による cc_restart ループは、依存関係の連鎖的な解決を簡潔に記述するために使用されている。
