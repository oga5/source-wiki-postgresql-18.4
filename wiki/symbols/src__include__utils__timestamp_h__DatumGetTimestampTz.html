DatumGetTimestampTz

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/timestamp.h:33

目的

PostgreSQLの内部データ型 Datum を、タイムゾーン付きタイムスタンプ型である TimestampTz に変換します。

シグネチャまたは形状

static inline TimestampTz DatumGetTimestampTz(Datum X)

入力と出力

  • **入力**: Datum X（変換対象のデータ）
  • **出力**: TimestampTz（int64 として表現されたタイムスタンプ値）

前提条件と不変条件

  • TimestampTz は内部的に int64 として実装されています。
  • 本関数は DatumGetInt64 を介して値を抽出するため、入力 Datum は int64 相当のデータとして格納されている必要があります。

アルゴリズムまたは振る舞い

1. 引数 X を DatumGetInt64 マクロにより int64 型の値として取り出します。 2. 取り出した値を TimestampTz 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_ts.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/libpq/crypt.c
  • src/backend/utils/adt/json.c
  • src/backend/utils/adt/jsonpath_exec.c
  • src/backend/utils/adt/selfuncs.c
  • src/backend/utils/adt/uuid.c
  • **呼び出し先**:
  • DatumGetInt64（マクロ）

関連データ構造

  • TimestampTz: タイムゾーン付きタイムスタンプを表す内部型（int64）。
  • Datum: PostgreSQLの汎用データ表現型。

失敗モードと境界条件

  • 入力 Datum が int64 以外の型を保持している場合、ビット列の解釈が不正となります。本関数自体は型チェックを行いません。

保守担当者向けメモ

  • 本関数はインライン化されており、オーバーヘッドを最小限に抑える設計です。
  • 算術演算は int64 の整数演算として行われるため、オーバーフロー等の制約は int64 の仕様に依存します。
