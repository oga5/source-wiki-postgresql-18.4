insert_timeout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/timeout.c:113

目的

指定されたタイムアウト識別子（TimeoutId）を、現在有効なタイムアウト管理配列（active_timeouts）の指定位置に挿入し、アクティブ状態に設定する。

シグネチャまたは形状

static void insert_timeout(TimeoutId id, int index)

入力と出力

  • **入力**:
  • id: 挿入するタイムアウトの識別子。
  • index: active_timeouts 配列内での挿入位置（0 から num_active_timeouts の間）。
  • **出力**: なし（副作用としてグローバルな管理状態を更新）。

前提条件と不変条件

  • 挿入対象のタイムアウト（all_timeouts[id]）が、挿入時点で非アクティブ（active == false）であること。
  • index が 0 以上 num_active_timeouts 以下であること。

アルゴリズムまたは振る舞い

1. index が範囲外の場合、elog(FATAL) を発生させる。 2. all_timeouts[id].active が true であることを Assert で確認する。 3. all_timeouts[id].active を true に設定する。 4. active_timeouts 配列内の index 以降の要素を一つずつ後ろにずらし、空いた index 位置に &all_timeouts[id] を格納する。 5. num_active_timeouts をインクリメントする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/timeout.c 内のタイムアウト有効化処理（enable_timeout_after 等の内部ロジック）から呼び出される。
  • **呼び出し先**: 外部関数は呼び出さないが、elog マクロおよび Assert マクロを使用する。

関連データ構造

  • all_timeouts: 全てのタイムアウト設定を保持する静的配列。
  • active_timeouts: 現在有効なタイムアウトを期限順に保持する配列。
  • num_active_timeouts: 現在アクティブなタイムアウトの総数。

失敗モードと境界条件

  • index が 0 未満または num_active_timeouts を超える場合、致命的エラー（FATAL）としてプロセスが終了する。
  • すでにアクティブなタイムアウトを挿入しようとした場合、Assert が有効なビルドではアサーション失敗となる。

保守担当者向けメモ

  • 本関数は active_timeouts 配列の要素をシフトさせるため、タイムアウト数が多い場合には計算量に注意が必要。
  • シグナルハンドラ（handle_sig_alarm）と競合しないよう、呼び出し側で適切にシグナル抑制（disable_alarm 等）が行われていることを前提としている。
