ExecSubPlan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:61

目的

式評価の一部としてサブクエリ（SubPlan）を実行し、その結果を算出する。

シグネチャまたは形状

Datum ExecSubPlan(SubPlanState *node, ExprContext *econtext, bool *isNull)

入力と出力

  • **入力**:
  • node: 実行対象の SubPlanState 構造体。
  • econtext: 式評価コンテキスト。
  • isNull: 結果が NULL かどうかを格納するポインタ。
  • **出力**:
  • Datum: サブクエリの実行結果。
  • isNull: 呼び出し元で NULL 判定に使用されるフラグ。

前提条件と不変条件

  • CTE_SUBLINK 型のサブプランは実行不可（エラーをスロー）。
  • setParam が設定されているサブプランは、MULTIEXPR_SUBLINK でない限り実行不可（エラーをスロー）。
  • 実行中、EState のスキャン方向は ForwardScanDirection に強制される。

アルゴリズムまたは振る舞い

1. CHECK_FOR_INTERRUPTS() を呼び出し、割り込みを確認する。 2. isNull をデフォルトで false に設定する。 3. サブプランの型チェックを行い、不正な場合はエラーを発生させる。 4. 実行時のスキャン方向を ForwardScanDirection に一時変更する。 5. subplan->useHashTable フラグに基づき、以下のいずれかの戦略を選択する:

  • true: ExecHashSubPlan を呼び出し、ハッシュテーブルを用いて結果を取得する。
  • false: ExecScanSubPlan を呼び出し、サブクエリをスキャンして評価する。

    • 6. 変更したスキャン方向を元の状態に復元する。 7. 算出した Datum を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExprInterp.c (式評価エンジン内)
  • **呼び出し先**:
  • ExecHashSubPlan (ハッシュ化戦略時)
  • ExecScanSubPlan (逐次スキャン戦略時)
  • elog (エラーハンドリング時)

関連データ構造

  • SubPlanState: サブプランの実行状態を保持する構造体。
  • SubPlan: サブプランの定義情報。
  • EState: クエリ実行全体のステート。
  • ExprContext: 式評価のコンテキスト。

失敗モードと境界条件

  • **エラー**: CTE_SUBLINK を実行しようとした場合、または許可されていない setParam を持つサブプランを実行しようとした場合に elog(ERROR) が発生する。
  • **割り込み**: 実行中に CHECK_FOR_INTERRUPTS() により割り込みが検知された場合、処理が中断される。

保守担当者向けメモ

  • 本関数は式評価エンジンとプラン実行ノードの橋渡しを行う。
  • useHashTable フラグによる分岐は、サブクエリの最適化戦略（ハッシュ化 vs スキャン）を決定する重要な箇所である。
  • スキャン方向の変更と復元は、プラン実行の整合性を保つために必須である。
