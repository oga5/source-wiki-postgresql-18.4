fill_buffer

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/io.c:345

目的

入力ストリームから1行分のデータを読み込み、内部バッファ（in_buffer）へ格納する。また、/*INDENT ON/OFF*/ ディレクティブの検出と処理を行う。

シグネチャまたは形状

void fill_buffer(void)

入力と出力

  • **入力**: グローバル変数 input（FILE *）、および lookahead バッファ。
  • **出力**: グローバル変数 in_buffer、buf_ptr、buf_end を更新。整形無効化時は output ストリームへ直接書き出しを行う。

前提条件と不変条件

  • in_buffer は動的に拡張可能であること。
  • bp_save が設定されている場合、読み込みを行わずに保存されたバッファへ切り替える。
  • inhibit_formatting が真の場合、読み込んだ行をそのまま出力する。

アルゴリズムまたは振る舞い

1. **バッファ復帰**: bp_save が存在すればそれを優先し、処理を終了する。 2. **読み込み**: getc(input) を使用して改行文字まで読み込む。バッファ容量が不足した場合は realloc で拡張する。 3. **ディレクティブ解析**: 行内に /*INDENT ON*/ または /*INDENT OFF*/ が含まれるか判定する。

  • 検出時、必要に応じて dump_line() を呼び出し、inhibit_formatting フラグを更新する。

    • 4. **整形無効化処理**: inhibit_formatting が有効な場合、読み込んだ行を即座に output へ書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/indent.c の各所（メインループおよび初期化処理）。
  • **呼び出し先**: dump_line()（整形済み行のフラッシュ）、realloc()（バッファ拡張）、getc()（入力読み込み）。

関連データ構造

  • in_buffer, in_buffer_limit: 入力データを保持するグローバルバッファ。
  • lookahead_buf, lookahead_start, lookahead_end: 先読み用バッファ。
  • inhibit_formatting: 整形処理の有効/無効を制御するフラグ。

失敗モードと境界条件

  • **メモリ不足**: realloc が失敗した場合、errx(1, "input line too long") を呼び出しプログラムを終了する。
  • **EOF**: 入力終了時に had_eof を真に設定し、バッファに改行を付与して終了する。
  • **再帰**: /**INDENT** エラーメッセージ検出時に自身を再帰呼び出ししてフラッシュする。

保守担当者向けメモ

  • 本関数はグローバルな状態（ps やバッファポインタ）に強く依存している。
  • in_buffer のポインタ操作は indent_globs.h で定義されたグローバル変数に基づいているため、修正時は関連するポインタの整合性に注意すること。
