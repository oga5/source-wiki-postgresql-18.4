in_grouping

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/utilities.c:167

目的

ステミング環境内の現在の文字が、指定されたビットマップ形式の文字集合（グループ）に含まれるかを判定し、必要に応じてカーソルを進める。

シグネチャまたは形状

int in_grouping(struct SN_env * z, const unsigned char * s, int min, int max, int repeat)

入力と出力

  • **入力**:
  • z: ステミング環境構造体（SN_env）。
  • s: 文字集合を表すビットマップ配列。
  • min, max: 判定対象の文字コード範囲。
  • repeat: 0 以外の場合、グループに一致し続ける限りカーソルを前進させる。
  • **出力**:
  • 0: 成功（一致した）。
  • 1: 失敗（現在の文字がグループに含まれない）。
  • -1: 失敗（カーソルがバッファ末尾に到達している）。

前提条件と不変条件

  • z->c は現在の処理位置（カーソル）を指し、z->l はバッファの長さを指す。
  • s は min から max までの範囲をカバーする十分なサイズのビットマップであること。

アルゴリズムまたは振る舞い

1. カーソル z->c がバッファ末尾 z->l に達している場合、-1 を返す。 2. 現在の文字 ch = z->p[z->c] を取得する。 3. ch が min 未満、または max を超える場合、グループ外とみなして 1 を返す。 4. ch が範囲内の場合、s のビットマップをチェックする（s[ch >> 3] & (0X1 << (ch & 0X7))）。 5. ビットが立っていない（グループに含まれない）場合、1 を返す。 6. ビットが立っている場合、repeat が真であればカーソルをインクリメントして 1. に戻り、偽であれば 0 を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: 各言語のステミング実装（例: stem_ISO_8859_1_basque.c, stem_ISO_8859_1_dutch.c 等）。
  • **呼び出し先**: 外部関数呼び出しなし（SN_env 構造体のメンバアクセスのみ）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • among: 辞書検索に使用される構造体（本関数では直接使用しないが、同モジュール内で関連）。

失敗モードと境界条件

  • z->c >= z->l: バッファの終端に達した場合は -1 を返す。
  • ch < min または ch > max: 指定範囲外の文字はグループ外として 1 を返す。
  • repeat が 0 の場合、一致してもカーソルは進まない。

保守担当者向けメモ

  • 本関数は非 UTF-8（シングルバイト）環境用である。UTF-8 環境では in_grouping_U を使用すること。
  • s のビットマップ操作は、文字コードを 8 ビット単位でインデックス化しているため、min と max の範囲設定が正確である必要がある。
