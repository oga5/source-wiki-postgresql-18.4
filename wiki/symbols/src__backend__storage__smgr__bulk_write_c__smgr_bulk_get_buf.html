smgr_bulk_get_buf

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/bulk_write.c:346

目的

バルク書き込み操作において、ページデータを保持するためのメモリ領域を確保する。

シグネチャまたは形状

BulkWriteBuffer smgr_bulk_get_buf(BulkWriteState *bulkstate)

入力と出力

  • **入力**: BulkWriteState *bulkstate - バルク書き込みの状態を管理する構造体へのポインタ。
  • **出力**: BulkWriteBuffer - bulkstate->memcxt 上に確保された、BLCKSZ サイズかつ PG_IO_ALIGN_SIZE にアライメントされたメモリ領域。

前提条件と不変条件

  • bulkstate は有効な BulkWriteState を指している必要がある。
  • 確保されたメモリは bulkstate のメモリコンテキストに属する。

アルゴリズムまたは振る舞い

1. bulkstate が保持するメモリコンテキスト (memcxt) を使用する。 2. MemoryContextAllocAligned を呼び出し、PostgreSQL のページサイズ (BLCKSZ) 分のメモリを、I/O アライメント (PG_IO_ALIGN_SIZE) を考慮して確保する。 3. 確保したメモリ領域を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuild.c
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/nbtree/nbtree.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/access/spgist/spginsert.c
  • src/backend/catalog/storage.c
  • **呼び出し先**:
  • MemoryContextAllocAligned (PostgreSQL メモリ管理 API)

関連データ構造

  • BulkWriteState: 書き込み操作の状態とメモリコンテキストを保持する。
  • BulkWriteBuffer: 本関数が返すメモリ領域の型。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocAligned の挙動に従い、エラーが発生する。

保守担当者向けメモ

  • 本関数で確保されたバッファの所有権は BulkWriteState に帰属します。呼び出し側で pfree を実行してはいけません。
  • 確保されたバッファは、後続の smgr_bulk_write に渡すことで書き込みキューへ登録されます。
