目的

PostgreSQL の埋め込み SQL (ECPG) におけるデータベース接続および切断機能の回帰テストを実行するエントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム実行時に環境変数や設定に依存）
  • 出力: 正常終了時に 0 を返します。

前提条件と不変条件

  • ECPGdebug が呼び出され、デバッグログが stderr に出力される設定で開始されます。
  • 接続文字列、ユーザー名、パスワードの組み合わせを変えた複数のテストケースが順次実行されます。

アルゴリズムまたは振る舞い

1. **デバッグ有効化**: ECPGdebug(1, stderr) を実行。 2. **接続テスト**: ECPGconnect を使用し、TCP、Unix ドメインソケット、認証情報付き接続など、多様な接続形式を試行します。 3. **SQL 実行**: 接続直後に ALTER USER などの SQL 文を ECPGdo で実行し、接続の有効性を確認します。 4. **切断処理**: ECPGdisconnect を呼び出し、接続状態をクリーンアップします。 5. **異常系テスト**: 存在しないデータベース、不正なポート、誤ったパスワードを指定し、接続失敗時の挙動を確認します。 6. **終了**: 全てのテストケース終了後、0 を返して終了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（テスト実行プロセス）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGdisconnect（いずれも ECPG ライブラリ関数）

関連データ構造

  • db (char[200]): 接続文字列を保持するバッファ。
  • pw (char[200]): パスワードを保持するバッファ。

失敗モードと境界条件

  • 接続失敗（存在しない DB、誤ったポート、誤ったパスワード）が意図的にテストケースに含まれています。
  • 接続が確立されなかった場合、ECPGdisconnect は呼び出されません。

保守担当者向けメモ

  • 本ファイルは ecpg プリプロセッサによって生成された「期待される出力」であり、直接編集しないでください。
  • 接続名が NULL の場合、デフォルトの接続が使用されます。
  • 接続失敗のテストケースでは、接続が確立されないため ECPGdisconnect が省略されています。
