get_partition_ancestors_worker

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/partition.c:152

目的

指定されたパーティションの祖先リレーション（親、祖父母など）を再帰的に探索し、リストとして構築します。

シグネチャまたは形状

static void get_partition_ancestors_worker(Relation inhRel, Oid relid, List **ancestors)

入力と出力

  • inhRel: pg_inherits カタログを参照するためのオープンされたリレーション。
  • relid: 祖先を探索する対象のパーティションの OID。
  • ancestors: 探索結果の OID リストを格納するポインタ。再帰呼び出しを通じて蓄積されます。

前提条件と不変条件

  • inhRel は pg_inherits カタログを指している必要があります。
  • 探索は最上位の親に到達するか、パーティションがデタッチ処理中（detach_pending）である場合に終了します。

アルゴリズムまたは振る舞い

1. get_partition_parent_worker を呼び出し、現在の relid の直接の親 OID とデタッチ状態を取得します。 2. 親が存在しない、またはデタッチ処理中の場合は探索を終了します。 3. 親が存在する場合、その OID を ancestors リストの末尾に追加します。 4. 取得した親 OID を引数として、自身を再帰的に呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: get_partition_ancestors（src/backend/catalog/partition.c:141）から呼び出されます。
  • **呼び出し先**:
  • get_partition_parent_worker（直接の親取得）
  • lappend_oid（リストへの追加）
  • get_partition_ancestors_worker（自身を再帰呼び出し）

関連データ構造

  • Relation: リレーション記述子。
  • Oid: オブジェクト識別子。
  • List: PostgreSQLのリスト構造（ancestors の保持に使用）。

失敗モードと境界条件

  • 最上位のパーティション（親を持たない）に達した場合、正常に終了します。
  • デタッチ処理中のパーティションに遭遇した場合、それ以上の祖先探索は行われません。

保守担当者向けメモ

  • 本関数は再帰的な探索を行うため、パーティション階層が極端に深い場合でもスタックオーバーフローが発生しない設計であることを確認してください。
  • get_partition_parent_worker の挙動に依存しているため、カタログスキャンロジックの変更時には注意が必要です。
