fatal_llvm_new_handler

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit_error.cpp:122

目的

LLVMのメモリ割り当てエラー（bad_alloc）を捕捉し、PostgreSQLのプロセス管理と整合する FATAL エラーとして報告する。

シグネチャまたは形状

static void fatal_llvm_new_handler(void *user_data, const char *reason, bool gen_crash_diag);

入力と出力

  • user_data: LLVMから渡されるユーザー定義データ（未使用）。
  • reason: エラーの詳細を示す文字列。
  • gen_crash_diag: クラッシュ診断を生成するかどうかのフラグ。
  • 出力: なし（ereport(FATAL, ...) によりプロセスが終了するため戻り値を持たない）。

前提条件と不変条件

  • 本関数は、LLVMの bad_alloc_error_handler として登録されることを前提としている。
  • ereport(FATAL, ...) が呼び出されるため、呼び出し元のトランザクションやプロセスは終了する。

アルゴリズムまたは振る舞い

1. ereport(FATAL, ...) を呼び出し、PostgreSQLの標準エラーログに ERRCODE_OUT_OF_MEMORY を記録する。 2. エラーメッセージに「out of memory」を、詳細情報にLLVMから提供された reason を含める。 3. プロセスを安全に終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: LLVMライブラリ内部（メモリ割り当て失敗時）。
  • 呼び出し先: ereport（PostgreSQLのコアエラー報告関数）。
  • 参照元: src/backend/jit/llvm/llvmjit_error.cpp:60 にて llvm::install_bad_alloc_error_handler の引数として登録されている。

関連データ構造

  • なし。

失敗モードと境界条件

  • LLVM内部でメモリ不足が発生した場合にのみ呼び出される。
  • FATAL エラーとして処理されるため、回復は試みられず、プロセスが終了する。

保守担当者向けメモ

  • 本関数はC++で記述されており、LLVMのハンドラインターフェースに適合させている。
  • 拡張機能が独自のメモリハンドラを使用している場合、競合が発生する可能性があるため注意が必要である。
