check_archive_directory

定義

  • 種別: function
  • 定義箇所: contrib/basic_archive/basic_archive.c:95

目的

GUCパラメータ basic_archive.archive_directory に設定されたパスの妥当性を検証する。

シグネチャまたは形状

static bool check_archive_directory(char **newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定されるディレクトリパスへのポインタ。
  • extra, source: GUCコールバック用の標準引数。
  • **出力**:
  • bool: パスが許容範囲内であれば true、長すぎる場合は false を返す。

前提条件と不変条件

  • 空文字列または NULL は、設定値として許容される（アーカイブ処理自体は別途 check_configured で制御される）。
  • パス長は MAXPGPATH を超えてはならない。

アルゴリズムまたは振る舞い

1. newval が NULL または空文字列の場合、検証をパスし true を返す。 2. newval の長さに、アーカイブされるファイル名の最大長（64文字）とパス区切り文字等の余裕分（2文字）を加算する。 3. 合計値が MAXPGPATH 以上の場合、GUC_check_errdetail を通じてエラー詳細を設定し、false を返す。 4. 上記以外の場合は true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: _PG_init 関数内でのGUC登録時に参照される。
  • **呼び出し先**: GUC_check_errdetail（PostgreSQLコアのGUC管理関数）。

関連データ構造

  • GucSource: GUC設定のソースを識別する列挙型。

失敗モードと境界条件

  • **失敗モード**: パス長が MAXPGPATH の制限を超える場合、設定の適用が拒否される。
  • **境界条件**: 空文字列は有効な値として扱われる。

保守担当者向けメモ

  • 本関数はパスの長さのみを検証する。ディレクトリの存在確認や書き込み権限のチェックは行わないため、必要に応じて呼び出し側で考慮すること。
