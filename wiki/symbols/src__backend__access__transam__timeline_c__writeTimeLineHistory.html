writeTimeLineHistory

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/timeline.c:303

目的

新しいタイムラインへの切り替え時に、親タイムラインの履歴情報を継承しつつ、新しい分岐点情報を追記した履歴ファイルを作成する。

シグネチャまたは形状

void writeTimeLineHistory(TimeLineID newTLI, TimeLineID parentTLI, XLogRecPtr switchpoint, char *reason)

入力と出力

  • **入力**:
  • newTLI: 作成する新しいタイムラインID。
  • parentTLI: 分岐元の親タイムラインID。
  • switchpoint: 分岐が発生したWAL位置（LSN）。
  • reason: 分岐理由を示す文字列。
  • **出力**: なし（副作用としてファイルシステム上に履歴ファイルを作成）。

前提条件と不変条件

  • newTLI > parentTLI であること（アサーションにより検証）。
  • 履歴ファイルの更新はアトミックに行われる（一時ファイル作成、書き込み、fsync、durable_rename の順序を遵守）。

アルゴリズムまたは振る舞い

1. 一時ファイル（xlogtemp.<pid>）を作成する。 2. parentTLI に対応する既存の履歴ファイルを読み込み、その内容を一時ファイルにコピーする（アーカイブリカバリ時は RestoreArchivedFile を使用）。 3. 分岐情報（parentTLI、switchpoint、reason）をタブ区切り形式で一時ファイルの末尾に追記する。 4. 一時ファイルを fsync し、durable_rename を用いて正式な履歴ファイル名（newTLI に基づく）へリネームする。 5. アーカイブが有効な場合、XLogArchiveNotify を呼び出し、作成した履歴ファイルをアーカイブ対象として通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c:6020
  • **呼び出し先**:
  • TLHistoryFileName, TLHistoryFilePath (マクロ)
  • RestoreArchivedFile, OpenTransientFile, CloseTransientFile, pg_fsync, durable_rename
  • pgstat_report_wait_start, pgstat_report_wait_end
  • XLogArchiveNotify, XLogArchivingActive

関連データ構造

  • TimeLineID: タイムラインを識別する数値型。
  • XLogRecPtr: WAL上の位置を示すポインタ型。

失敗モードと境界条件

  • ファイル作成・読み込み・書き込み・クローズ時にエラーが発生した場合、ereport(ERROR, ...) により処理を中断する。
  • 書き込み失敗時には一時ファイルを unlink してディスク領域を解放する。
  • parentTLI の履歴ファイルが存在しない場合、親は存在しないものとして処理を続行する。

保守担当者向けメモ

  • 履歴ファイルは <parentTLI> <switchpoint> <reason> の形式を持つ。
  • durable_rename を使用しているため、クラッシュリカバリ時にも履歴ファイルの整合性が保たれる設計となっている。
  • タイムライン1は初期状態であり、履歴ファイルを持たない。
