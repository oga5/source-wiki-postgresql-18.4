mbuf_avail

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/mbuf.c:49

目的

MBuf 構造体内に保持されている、現在読み取り可能なデータ量（バイト数）を算出する。

シグネチャまたは形状

int mbuf_avail(MBuf *mbuf);

入力と出力

  • **入力**: MBuf *mbuf （対象となるメモリバッファへのポインタ）
  • **出力**: int （読み取り可能な残存バイト数）

前提条件と不変条件

  • mbuf は有効な MBuf 構造体を指している必要がある。
  • mbuf->data_end はバッファ内のデータ末尾を、mbuf->read_pos は現在の読み取り位置を指す。
  • 常に data_end >= read_pos であることが期待される。

アルゴリズムまたは振る舞い

バッファのデータ終了位置 (data_end) から現在の読み取り位置 (read_pos) を減算し、その差分を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/mbuf.c: バッファの残量確認に使用。
  • contrib/pgcrypto/pgp-encrypt.c: mbuf_grab 呼び出し時の引数として使用。
  • **呼び出し先**:
  • 本関数は他の関数を呼び出さない。

関連データ構造

  • MBuf: データを保持する可変長バッファ構造体。

失敗モードと境界条件

  • mbuf が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • バッファが空の場合、戻り値は 0 となる。

保守担当者向けメモ

  • 本関数は単純な算術演算のみを行うため、副作用はない。
  • mbuf_grab 等のデータ消費関数と組み合わせて使用されることが多いため、バッファの状態変更を伴う処理の前後で呼び出す際は、読み取り位置の整合性に注意すること。
