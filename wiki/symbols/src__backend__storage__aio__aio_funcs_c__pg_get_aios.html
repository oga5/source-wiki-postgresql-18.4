pg_get_aios

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_funcs.c:47

目的

PostgreSQLの非同期I/O（AIO）サブシステムの稼働状況をSQLインターフェース経由で可視化し、現在実行中または完了したI/O操作の状態をテーブル形式（SRF）で返却します。

シグネチャまたは形状

Datum pg_get_aios(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数呼び出しの引数）
  • **出力**: tuplestore に格納された15列のレコードセット（SRF）。各レコードにはPID、I/O ID、世代番号、状態、操作種別、オフセット、サイズ、ターゲット名、結果などが含まれます。

前提条件と不変条件

  • ロックフリーで共有メモリ上の PgAioHandle を読み取るため、pg_read_barrier() を使用してメモリ整合性を保証しています。
  • 世代番号（generation）の不一致は、I/Oの再利用（完了後の新規開始）を意味し、その場合は表示をスキップします。
  • 状態（state）の変更は、楽観的並行制御により再試行（retry）で解決されます。

アルゴリズムまたは振る舞い

1. InitMaterializedSRF で結果セットを初期化します。 2. pgaio_ctl->io_handles を走査し、各ハンドルに対して以下を実行します。

  • 世代番号と状態を記録し、ハンドルと iovec をローカルメモリにコピーします。
  • 読み取り後に世代番号が変化した場合はスキップし、状態が変化した場合は retry ラベルへ戻ります。

    • 3. 取得したデータを Datum 配列に変換し、tuplestore_putvalues で結果セットに追加します。 4. PGAIO_HS_HANDED_OUT 状態の場合は、一部の列をNULLとして出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（システムビュー等からの呼び出しが想定されますが、ソースコード上での具体的な呼び出し元は明示されていません）。
  • **呼び出し先**: InitMaterializedSRF, pgaio_io_get_id, GetPGProcByNumber, pg_read_barrier, memcpy, pgaio_io_get_state_name, pgaio_io_get_op_name, iov_byte_length, pgaio_io_get_target_name, pgaio_result_status_string, pgaio_io_get_target_description, tuplestore_putvalues。

関連データ構造

  • PgAioHandle: I/O操作の管理構造体。
  • pgaio_ctl: AIOサブシステム全体の制御構造体。
  • tuplestore: SRFの結果を保持するメモリ構造。

失敗モードと境界条件

  • **ライブロックの回避**: I/O状態の変化に対して再試行を行いますが、状態遷移は一方向であるため、無限ループの危険性は低い設計となっています。
  • **IDの再利用**: 世代番号のチェックにより、I/O完了後に同じIDで開始された新しい操作を誤って表示することを防ぎます。
  • **PIDの欠落**: プロセスが終了している場合など、PIDが取得できない場合はNULLを返します。

保守担当者向けメモ

  • 本関数はロックを使用しないため、共有メモリへのアクセスには常に pg_read_barrier() を伴う必要があります。
  • PG_GET_AIOS_COLS（15列）の定義を変更する場合は、values および nulls 配列のサイズと、tuplestore_putvalues への入力を整合させる必要があります。
