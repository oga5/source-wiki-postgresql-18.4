tsq_mcontained

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_op.c:353

目的

tsquery 型の包含関係判定において、引数の順序を入れ替えて tsq_mcontains を呼び出すためのラッパー関数です。SQL における <@ 演算子をサポートします。

シグネチャまたは形状

Datum tsq_mcontained(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの tsquery 型の Datum を受け取ります。
  • **出力**: tsq_mcontains の実行結果（boolean）を返します。

前提条件と不変条件

  • 入力された Datum は有効な tsquery 型である必要があります。
  • 内部的に tsq_mcontains を呼び出すため、同関数の前提条件（単語集合としての包含判定ロジック）に依存します。

アルゴリズムまたは振る舞い

1. PG_GETARG_DATUM(0) と PG_GETARG_DATUM(1) を取得します。 2. DirectFunctionCall2 を使用して、引数の順序を入れ替えた状態で tsq_mcontains を呼び出します。

  • 具体的には tsq_mcontains(arg1, arg0) として実行されます。

    • 3. tsq_mcontains の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: tsq_mcontains
  • **呼び出し元**: ソースコード内での直接的な呼び出し元は確認されていません。SQL インターフェース経由で演算子 <@ として呼び出されることが想定されます。

関連データ構造

  • tsquery: PostgreSQL の全文検索用データ型。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • tsq_mcontains が失敗する場合（メモリ不足や不正な tsquery 構造など）、本関数も同様に失敗します。
  • 引数が NULL の場合の挙動は、PostgreSQL の関数呼び出し規約（strict 設定）に依存します。

保守担当者向けメモ

  • 本関数は単なる引数入れ替えのプロキシです。包含判定ロジックの変更が必要な場合は、tsq_mcontains を修正してください。
  • tsq_mcontains は構造的な包含ではなく、単語集合としての包含を判定する点に注意してください。
