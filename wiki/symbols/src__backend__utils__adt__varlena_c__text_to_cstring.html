text_to_cstring

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varlena.c:225

目的

PostgreSQLの内部的な可変長文字列型（text）を、C言語で扱えるヌル終端文字列（char *）に変換します。

シグネチャまたは形状

char *text_to_cstring(const text *t)

入力と出力

  • **入力**: const text *t - 変換対象のPostgreSQL text 型データへのポインタ。
  • **出力**: char * - 呼び出し元が pfree で解放すべき、ヌル終端されたC文字列。

前提条件と不変条件

  • 入力 t はTOAST（圧縮または外部化）されている可能性があるため、関数内部で透過的に展開されます。
  • 戻り値は palloc によって現在のメモリコンテキストに割り当てられます。

アルゴリズムまたは振る舞い

1. pg_detoast_datum_packed を使用して、入力データが圧縮されている場合は展開（またはコピー）します。 2. 展開後のデータ長を VARSIZE_ANY_EXHDR で取得します。 3. palloc を用いて、データ長 + 1 バイトのメモリを確保します。 4. memcpy でデータをコピーし、末尾にヌル文字（\0）を付与します。 5. 展開のために一時的なメモリ確保が行われた場合（tunpacked != t）、そのメモリを pfree で解放します。 6. 生成したC文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck/verify_heapam.c および contrib/dblink/dblink.c 等の拡張モジュールから呼び出されています。
  • **呼び出し先**: pg_detoast_datum_packed, palloc, memcpy, pfree を呼び出します。

関連データ構造

  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、トランザクションがアボートされる可能性があります。
  • 入力がNULLの場合の挙動は本関数内では定義されておらず、呼び出し元で適切に処理する必要があります。

保守担当者向けメモ

  • unconstify マクロを使用して const 修飾を外していますが、これは pg_detoast_datum_packed のインターフェース上の制約によるものです。
  • 戻り値は呼び出し元で必ず解放（pfree）してください。
