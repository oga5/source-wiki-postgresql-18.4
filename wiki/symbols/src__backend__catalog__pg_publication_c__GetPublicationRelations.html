GetPublicationRelations

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_publication.c:779

目的

指定されたパブリケーションに関連付けられているリレーション（テーブル）のOIDリストを取得します。

シグネチャまたは形状

List *GetPublicationRelations(Oid pubid, PublicationPartOpt pub_partopt)

入力と出力

  • **入力**:
  • pubid (Oid): 対象となるパブリケーションのOID。
  • pub_partopt (PublicationPartOpt): パーティションテーブルをどのように扱うか（ルートのみ、全パーティション等）を指定するオプション。
  • **出力**:
  • List *: パブリケーションに含まれるリレーションOIDのリスト。

前提条件と不変条件

  • pg_publication_rel システムカタログに対する AccessShareLock を取得してスキャンを行います。
  • 返されるリストは、関数内でソートおよび重複排除が行われるため、一意なOIDのリストであることが保証されます。

アルゴリズムまたは振る舞い

1. pg_publication_rel システムカタログを prpubid インデックスを用いてスキャンします。 2. パブリケーションID (pubid) に一致するすべてのタプルを走査します。 3. 各タプルからリレーションOID (prrelid) を抽出し、GetPubPartitionOptionRelations を呼び出して、pub_partopt に基づいた適切なリレーションを結果リストに追加します。 4. スキャン終了後、リストに対して list_sort（list_oid_cmp 使用）および list_deduplicate_oid を実行し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c 内の複数箇所
  • src/backend/commands/publicationcmds.c 内の複数箇所
  • **呼び出し先**:
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan
  • GetPubPartitionOptionRelations
  • list_sort, list_deduplicate_oid

関連データ構造

  • pg_publication_rel: パブリケーションとリレーションの紐付けを保持するシステムカタログ。
  • PublicationPartOpt: パーティションの公開範囲を制御する列挙型。

失敗モードと境界条件

  • 指定した pubid が存在しない場合、空のリストが返されます。
  • pub_partopt の指定により、パーティション階層の解決結果が空になる場合があります。

保守担当者向けメモ

  • 本関数は pg_publication_rel のスキャン結果を直接加工するため、カタログのスキーマ変更やインデックス定義の変更時には注意が必要です。
  • パーティションの解決ロジックは GetPubPartitionOptionRelations に依存しているため、パーティション関連の挙動を調査する際は併せて確認してください。
