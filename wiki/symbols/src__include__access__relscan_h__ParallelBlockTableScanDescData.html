ParallelBlockTableScanDescData

定義

  • 種別: struct
  • 定義箇所: src/include/access/relscan.h:92

目的

ブロック指向ストレージにおける並列テーブルスキャンの共有状態を保持し、複数のバックエンド（ワーカー）間でスキャン対象ブロックを動的に割り当てるための制御構造体です。

シグネチャまたは形状

typedef struct ParallelBlockTableScanDescData
{
    ParallelTableScanDescData base;     /* 基底構造体 */
    BlockNumber phs_nblocks;            /* スキャン開始時のリレーション総ブロック数 */
    slock_t     phs_mutex;              /* 開始ブロック設定用の排他制御用スピンロック */
    BlockNumber phs_startblock;         /* スキャン開始ブロック番号 */
    pg_atomic_uint64 phs_nallocated;    /* ワーカーに割り当て済みのブロック数（アトミック） */
} ParallelBlockTableScanDescData;

入力と出力

  • **入力**: スキャン開始時にリレーションの総ブロック数や開始位置が設定されます。
  • **出力**: phs_nallocated をアトミックにインクリメントすることで、各ワーカーが次に処理すべきブロック番号を決定します。

前提条件と不変条件

  • 本構造体は共有メモリ上に配置される必要があります。
  • phs_nallocated はアトミック操作によって更新され、並列ワーカー間での重複のないブロック割り当てを保証します。
  • phs_mutex は、スキャン開始時の初期化処理において phs_startblock を保護するために使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: スキャン開始時に phs_nblocks が設定されます。 2. **ブロック割り当て**: 各ワーカーは phs_nallocated をアトミックに取得・加算し、その値を基に処理対象のブロックを計算します。 3. **終了判定**: 割り当てられたブロック番号が phs_nblocks に達した時点で、そのワーカーのスキャンは終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/table/tableam.c 内の関数が、この構造体のサイズを参照してメモリ確保を行っています。
  • **呼び出し先**: 提供されたソース情報からは、具体的な関数呼び出し関係は特定できません。

関連データ構造

  • ParallelTableScanDescData: 本構造体の先頭メンバであり、並列スキャンの共通基底クラス。
  • ParallelBlockTableScanDesc: 本構造体へのポインタ型。

失敗モードと境界条件

  • リレーションが空の場合、phs_nblocks が 0 となり、スキャンは即座に終了します。
  • 並列ワーカー数がブロック数を超過する場合、一部のワーカーには処理対象が割り当てられません。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、メンバの順序変更や追加を行う際は、並列処理の整合性とアトミック操作の要件を十分に考慮してください。
  • phs_mutex はスキャン開始時の初期化にのみ使用されるため、スキャン実行中のブロック割り当てには影響しません。
