ExtendBufferedFlags

定義

  • 種別: enum
  • 定義箇所: src/include/storage/bufmgr.h:68

目的

バッファの拡張操作（Relationの拡張）を行う際の動作を制御するためのフラグセットを定義します。

シグネチャまたは形状

typedef enum ExtendBufferedFlags
{
    EB_SKIP_EXTENSION_LOCK = (1 << 0),
    EB_PERFORMING_RECOVERY = (1 << 1),
    EB_CREATE_FORK_IF_NEEDED = (1 << 2),
    EB_LOCK_FIRST = (1 << 3),
    EB_CLEAR_SIZE_CACHE = (1 << 4),
    EB_LOCK_TARGET = (1 << 5),
} ExtendBufferedFlags;

入力と出力

本シンボルはビットマスクとして使用されます。各フラグは、バッファマネージャの拡張関連関数に対する引数として渡され、処理の挙動を修飾します。

前提条件と不変条件

  • EB_SKIP_EXTENSION_LOCK を使用する場合、対象のリレーションが共有されていないこと、排他ロックが保持されていること、あるいはスタートアッププロセス中であることのいずれかが保証されている必要があります。

アルゴリズムまたは振る舞い

各フラグは以下の制御を行います：

  • EB_SKIP_EXTENSION_LOCK: 拡張ロックの取得をスキップします。
  • EB_PERFORMING_RECOVERY: リカバリ処理中であることを示します。
  • EB_CREATE_FORK_IF_NEEDED: 存在しない場合にフォークを作成します。
  • EB_LOCK_FIRST: 最初の戻りバッファをロック状態で返します。
  • EB_CLEAR_SIZE_CACHE: ストレージマネージャのサイズキャッシュをクリアします。
  • EB_LOCK_TARGET: 内部的なロック制御フラグです。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数名は、提供されたソース抜粋には含まれていません。

関連データ構造

  • Buffer: バッファを識別する整数型。
  • BufferDesc: バッファ記述子（src/include/storage/buf_internals.h で定義）。

失敗モードと境界条件

  • EB_CREATE_FORK_IF_NEEDED は、リレーションフォークに対してのみ適用されることが想定されています。
  • フラグの誤った組み合わせ（例：排他ロックが必要な状況での EB_SKIP_EXTENSION_LOCK の不適切な使用）は、データ整合性の欠如を招く可能性があります。

保守担当者向けメモ

  • 本列挙型はビット演算によるフラグ制御を前提としています。新しいフラグを追加する際は、既存のビット位置と重複しないように注意してください。
