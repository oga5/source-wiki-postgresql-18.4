TS_execute_locations

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:2006

目的

全文検索クエリ（tsquery）のツリーを走査し、マッチングした位置情報のリストを抽出する。

シグネチャまたは形状

List *TS_execute_locations(QueryItem *curitem, void *arg, uint32 flags, TSExecuteCallback chkcond)

入力と出力

  • **入力**:
  • curitem: 評価対象のクエリツリーノード。
  • arg: 照合対象のデータ（tsvector 等）。
  • flags: 実行オプション（現在は TS_EXEC_EMPTY のみサポート）。
  • chkcond: 語彙の照合を行うコールバック関数。
  • **出力**:
  • マッチした位置情報の List。マッチしない場合は NIL を返す。

前提条件と不変条件

  • flags は TS_EXEC_EMPTY である必要がある（Assert により検証）。
  • chkcond コールバックは、arg と curitem を用いて適切に照合可能であること。

アルゴリズムまたは振る舞い

1. flags が TS_EXEC_EMPTY であることを確認する。 2. TS_execute_locations_recurse を呼び出し、クエリツリーを再帰的に走査して位置情報を収集する。 3. 再帰処理が成功した場合は収集した List を返し、失敗した場合は NIL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/wparser_def.c 内の複数箇所で、クエリマッチング結果の取得や再チェックに使用されている。
  • **呼び出し先**:
  • TS_execute_locations_recurse（再帰的な走査を実行）。

関連データ構造

  • QueryItem: tsquery の構成要素。
  • List: PostgreSQL のリスト構造（位置情報の格納に使用）。
  • TSExecuteCallback: 語彙照合用の関数ポインタ型。

失敗モードと境界条件

  • TS_execute_locations_recurse がマッチングに失敗した場合、NIL が返される。
  • フレーズ検索の OR 条件など、一部の複雑なクエリ構造に対しては、返される位置情報が不正確または不完全になる可能性がある（ソースコード内のコメントによる指摘）。

保守担当者向けメモ

  • 本関数は TS_execute_locations_recurse へのラッパーとして機能している。
  • 現在のフラグ引数は拡張性を考慮しているが、実装上は TS_EXEC_EMPTY 以外は許可されていない。
