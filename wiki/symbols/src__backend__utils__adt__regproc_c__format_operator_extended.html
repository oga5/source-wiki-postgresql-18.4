format_operator_extended

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regproc.c:721

目的

演算子の OID を、人間が読み取り可能な「スキーマ修飾された名前(引数型1, 引数型2)」形式の文字列に変換します。

シグネチャまたは形状

char *format_operator_extended(Oid operator_oid, bits16 flags)

入力と出力

  • **入力**:
  • operator_oid: 変換対象の演算子の OID。
  • flags: 動作を制御するビットフラグ。
  • FORMAT_OPERATOR_FORCE_QUALIFY: 演算子を強制的にスキーマ修飾する。
  • FORMAT_OPERATOR_INVALID_AS_NULL: OID が無効な場合に NULL を返す。
  • **出力**:
  • 成功時: 演算子名と引数型を含む文字列（palloc 済み）。
  • 失敗時: flags に FORMAT_OPERATOR_INVALID_AS_NULL が含まれる場合は NULL、それ以外の場合は OID の数値文字列。

前提条件と不変条件

  • IsBootstrapProcessingMode() が真である環境では実行できません（Assert により停止）。
  • システムカタログ pg_operator へのアクセスが必要です。

アルゴリズムまたは振る舞い

1. OPEROID システムキャッシュを使用して pg_operator エントリを検索します。 2. エントリが見つかった場合:

  • flags の指定または OperatorIsVisible の結果に基づき、必要であれば quote_identifier を用いてスキーマ名を付与します。
  • 演算子名に続き、左引数および右引数の型を format_type_be（または format_type_be_qualified）で解決して括弧内に連結します。

    • 3. エントリが見つからない場合:

  • FORMAT_OPERATOR_INVALID_AS_NULL フラグがあれば NULL を返します。
  • それ以外の場合は、OID を文字列としてフォーマットして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c (複数箇所)
  • src/backend/utils/adt/regproc.c (内部の format_operator 等)
  • **呼び出し先**:
  • SearchSysCache1, ReleaseSysCache
  • get_namespace_name, quote_identifier
  • format_type_be, format_type_be_qualified
  • initStringInfo, appendStringInfo 等の StringInfo API

関連データ構造

  • pg_operator: 演算子の定義を保持するシステムカタログ。
  • StringInfoData: 文字列構築用バッファ。

失敗モードと境界条件

  • 指定された operator_oid が pg_operator に存在しない場合、デフォルトでは数値文字列が返されます。
  • 引数型が NONE（単項演算子など）の場合、該当箇所は NONE と出力されます。

保守担当者向けメモ

  • 本関数はブートストラップモードをサポートしていません。
  • flags の追加や変更を行う際は、src/include/utils/regproc.h に定義されているフラグとの整合性に注意してください。
