gbt_int2_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int2.c:127

目的

GiST インデックス内の int2 型エントリから、元のデータを取得（フェッチ）します。

シグネチャまたは形状

Datum gbt_int2_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、インデックスエントリから復元された値を返します。

前提条件と不変条件

  • tinfo（gbtree_ninfo 型の静的構造体）が、int2 型のメタデータとして正しく初期化されている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から GISTENTRY 構造体を取得します。 2. 取得した entry と、int2 型の定義情報を持つ静的変数 tinfo を引数として、gbt_num_fetch 関数を呼び出します。 3. gbt_num_fetch の戻り値を PG_RETURN_POINTER マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構（直接的な呼び出し元コードはソース抜粋内には存在しません）。
  • **呼び出し先**: gbt_num_fetch（btree_utils_num.c で定義された汎用数値型フェッチ関数）。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • tinfo: int2 型の比較ロジックやサイズ情報を保持する gbtree_ninfo 型の静的構造体。

失敗モードと境界条件

  • entry が NULL の場合、または tinfo が適切に設定されていない場合、gbt_num_fetch の挙動に依存します。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の gbt_num_fetch を呼び出すだけの単純なラッパーです。
  • int2 型固有のロジックを変更する場合は、tinfo 構造体の定義を確認してください。
