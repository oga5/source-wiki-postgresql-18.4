r_prelude

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_german.c:155

目的

ドイツ語のステミング処理の初期段階として、入力単語内の特殊文字（ウムラウト等）の正規化および特定の接尾辞パターンの置換を行う。

シグネチャまたは形状

static int r_prelude(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: 成功時に 1、エラー発生時に負の整数を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • header.h で定義された in_grouping_U、slice_from_s、find_among、skip_utf8 などのランタイム関数が利用可能であること。

アルゴリズムまたは振る舞い

1. **特殊文字の正規化**: 単語全体を走査し、特定の母音に続く 'u' や 'y' を正規化された形式に置換する。 2. **接尾辞の置換**: find_among を使用して定義済みのテーブル a_0 に基づき、単語末尾のパターンを検索する。一致したパターンに応じて slice_from_s で文字列を置換、または skip_utf8 でポインタを進める処理を繰り返す。 3. 処理は単語の末尾から前方に向かって適用される。

呼び出し元と呼び出し先

  • 呼び出し元: german_UTF_8_stem（ファイル内のメインエントリポイント）から呼び出される。
  • 呼び出し先: in_grouping_U, slice_from_s, find_among, skip_utf8（いずれも header.h 由来）。

関連データ構造

  • struct SN_env: ステミング対象の文字列、現在のカーソル位置、領域境界情報などを保持する。
  • a_0: find_among で使用される接尾辞マッチングテーブル。

失敗モードと境界条件

  • slice_from_s が負の値を返した場合、そのエラーコードをそのまま呼び出し元へ伝播させる。
  • 文字列の末尾に達した場合や、マッチング対象が見つからない場合はループを終了する。
