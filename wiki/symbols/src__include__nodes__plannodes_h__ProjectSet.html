ProjectSet

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/plannodes.h:260

目的

クエリ実行計画において、セットを返す関数（SRF: Set-Returning Functions）を評価するためにプランナによって挿入されるノードです。

シグネチャまたは形状

typedef struct ProjectSet
{
    Plan    plan;
} ProjectSet;

Plan 構造体を最初のメンバとして保持する継承構造をとります。

入力と出力

  • **入力**: 子ノード（plan.lefttree）から供給されるタプル。
  • **出力**: SRF を評価し、射影を行った結果のタプル。

前提条件と不変条件

  • SRF を含むクエリ実行時にのみ使用されます。SRF が存在しない場合は使用されません。
  • ProjectSet ノード自体にはフィルタ条件（qual）を適用することはサポートされていません。

アルゴリズムまたは振る舞い

プランナによって実行計画ツリーに挿入され、エグゼキュータがこのノードを処理する際に ExecInitProjectSet を通じて実行時状態（PlanState）が初期化されます。子ノードから受け取ったデータに対して SRF の評価を行い、結果を射影します。

呼び出し元と呼び出し先

  • **呼び出し元（生成側）**: src/backend/optimizer/plan/createplan.c 内の create_project_set_plan および make_project_set 関数。
  • **呼び出し先（実行側）**: src/backend/executor/execProcnode.c の ExecInitProjectSet が本ノードを初期化します。
  • **その他**: src/backend/commands/explain.c がデバッグや計画表示のために本シンボルを参照します。

関連データ構造

  • Plan: 基底構造体。
  • PlanState: エグゼキュータ側で対応する実行時状態構造体。

失敗モードと境界条件

  • SRF が含まれないクエリに対して本ノードを使用することは想定されていません。
  • フィルタ条件（qual）を本ノードに設定しようとするとサポート外となります。

保守担当者向けメモ

  • 本ノードは Plan 構造体を継承しているため、Plan* へのキャストが可能です。
  • 実行時の挙動については src/backend/executor/nodeProjectSet.c を参照してください。
