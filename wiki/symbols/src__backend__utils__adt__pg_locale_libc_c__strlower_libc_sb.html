strlower_libc_sb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:152

目的

シングルバイトエンコーディング環境において、OSのロケール設定に基づき文字列を小文字に変換します。

シグネチャまたは形状

static size_t strlower_libc_sb(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: 変換結果を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換対象の文字列。
  • srclen: src の長さ（負の場合は strlen(src) が使用される）。
  • locale: ロケール情報を保持する pg_locale_t 構造体。
  • **出力**:
  • 変換後の文字列長（srclen）を返します。

前提条件と不変条件

  • srclen が負の場合、src は NUL 終端されている必要があります。
  • dest バッファは srclen + 1 バイト以上の容量が必要です。

アルゴリズムまたは振る舞い

1. srclen が負の場合、strlen で長さを算出します。 2. srclen + 1 が destsize 以下であることを確認します。 3. src を dest にコピーし、末尾に NUL 文字を付加します。 4. 文字列内の各文字に対して以下の変換を行います。

  • locale->is_default が真の場合: pg_tolower を使用（ASCIIベースの変換）。
  • それ以外の場合: tolower_l を使用し、指定されたロケールハンドル（locale->info.lt）に基づき変換。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale_libc.c 内の strlower_libc 関数から呼び出されます。
  • **呼び出し先**: strlen, memcpy, pg_tolower, tolower_l を呼び出します。

関連データ構造

  • pg_locale_t: ロケール情報およびOSのロケールハンドル（locale_t）を保持する構造体。

失敗モードと境界条件

  • srclen + 1 > destsize の場合、変換処理はスキップされ、元の srclen が返されます。
  • tolower_l の挙動は、OSが提供するロケール定義に依存します。

保守担当者向けメモ

  • 本関数はシングルバイト文字専用です。マルチバイト文字には strlower_libc_mb が使用されます。
  • tolower_l は isupper_l によるガードなしで呼び出されます。
  • デフォルトロケール時は pg_tolower を使用し、PostgreSQL 伝統の ASCII 処理を維持しています。
