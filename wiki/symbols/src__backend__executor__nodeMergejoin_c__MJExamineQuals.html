MJExamineQuals

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMergejoin.c:174

目的

マージ結合で使用される結合条件（mergeclauses）を解析し、実行可能な形式（MergeJoinClauseData の配列）に変換・準備します。

シグネチャまたは形状

MergeJoinClause MJExamineQuals(List *mergeclauses,
                               Oid *mergefamilies,
                               Oid *mergecollations,
                               bool *mergereversals,
                               bool *mergenullsfirst,
                               PlanState *parent)

入力と出力

  • **入力**:
  • mergeclauses: 結合条件のリスト（OpExpr）。
  • mergefamilies, mergecollations, mergereversals, mergenullsfirst: 各結合条件に対応する演算子族、照合順序、反転フラグ、NULL順序の配列。
  • parent: 実行ノードの PlanState。
  • **出力**:
  • MergeJoinClause（MergeJoinClauseData 構造体の配列）へのポインタ。

前提条件と不変条件

  • 入力される mergeclauses の各要素は OpExpr である必要があります。
  • 結合演算子は等価比較（COMPARE_EQ）である必要があります。
  • 結合条件の数は、提供される各属性配列の長さと一致している必要があります。

アルゴリズムまたは振る舞い

1. palloc0 を使用して、結合条件数分の MergeJoinClauseData 領域を確保します。 2. 各結合条件に対し、ExecInitExpr を呼び出して入力式（lexpr, rexpr）を初期化します。 3. SortSupport 構造体（ssup）に照合順序、反転フラグ、NULL順序を設定します。 4. get_op_opfamily_properties で演算子の戦略を取得し、等価比較であることを検証します。 5. get_opfamily_proc を使用して、BTSORTSUPPORT_PROC（ソートサポート関数）または BTORDER_PROC（比較関数）を取得し、ssup.comparator を設定します。

  • BTORDER_PROC の場合は PrepareSortSupportComparisonShim を介して比較用シムを準備します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeMergejoin.c 内の初期化処理（ExecInitMergeJoin 等）から呼び出されます。
  • **呼び出し先**: ExecInitExpr, get_op_opfamily_properties, get_opfamily_method, IndexAmTranslateStrategy, get_opfamily_proc, OidFunctionCall1, PrepareSortSupportComparisonShim。

関連データ構造

  • MergeJoinClauseData: 結合キーの比較に必要な式やソートサポート情報を保持する構造体。
  • SortSupport: ソートおよび比較のためのサポートデータ。
  • OpExpr: 結合条件を表すノード。

失敗モードと境界条件

  • OpExpr 以外のノードが渡された場合、elog(ERROR) を発生させます。
  • 等価比較ではない演算子が指定された場合、elog(ERROR) を発生させます。
  • 必要なサポート関数（BTORDER_PROC）が見つからない場合、elog(ERROR) を発生させます。

保守担当者向けメモ

  • 本関数はマージ結合の初期化フェーズで一度だけ実行されます。
  • 略称最適化（abbreviation optimization）はマージ結合ではサポートされないため、ssup.abbreviate は明示的に false に設定されています。
