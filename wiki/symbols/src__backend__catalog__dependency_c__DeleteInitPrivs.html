DeleteInitPrivs

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/dependency.c:2799

目的

指定されたデータベースオブジェクトに関連付けられた初期権限情報（pg_init_privs）をカタログから削除する。

シグネチャまたは形状

static void DeleteInitPrivs(const ObjectAddress *object)

入力と出力

  • **入力**: object (const ObjectAddress *) - 削除対象のオブジェクトを特定するためのアドレス情報。
  • **出力**: なし（副作用として pg_init_privs カタログを更新）。

前提条件と不変条件

  • pg_init_privs システムカタログに対する RowExclusiveLock を取得可能であること。
  • object 引数は有効な ObjectAddress を指していること。

アルゴリズムまたは振る舞い

1. pg_init_privs カタログを RowExclusiveLock モードで開く。 2. object の objectId および classId をキーとしてスキャン条件を初期化する。 3. objectSubId が 0 でない場合、それもスキャン条件に含める。 4. systable_beginscan を使用して InitPrivsObjIndexId インデックス経由で該当レコードを検索する。 5. 検索されたすべてのタプルに対し、CatalogTupleDelete を実行して削除する。 6. スキャンを終了し、カタログテーブルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c 内のオブジェクト削除処理フローから呼び出される。
  • **呼び出し先**: table_open, ScanKeyInit, systable_beginscan, systable_getnext, CatalogTupleDelete, systable_endscan, table_close（すべてPostgreSQL内部のカタログアクセス関数）。

関連データ構造

  • ObjectAddress: 削除対象オブジェクトを識別する構造体。
  • pg_init_privs: 初期権限情報を保持するシステムカタログ。

失敗モードと境界条件

  • 該当する初期権限レコードが存在しない場合、何も削除されず正常に終了する。
  • objectSubId が 0 の場合はオブジェクト全体を対象とし、0 以外の場合は特定のサブオブジェクト（列など）のみを対象とする。

保守担当者向けメモ

  • 本関数は pg_init_privs のクリーンアップ専用である。新しいオブジェクトタイプが初期権限を保持するようになった場合、本関数が正しく対象を捕捉できるか確認すること。
