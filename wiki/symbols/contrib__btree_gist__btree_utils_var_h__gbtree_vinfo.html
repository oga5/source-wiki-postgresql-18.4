gbtree_vinfo

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_utils_var.h:23

目的

可変長データ型（text, bytea, numeric 等）を GiST 索引で扱うための型固有のメソッドおよび属性を保持する抽象化テーブルです。各データ型モジュールがこの構造体を静的に定義し、btree_utils_var.c が提供する汎用的な GiST メソッドへ型固有のロジックを委譲するために使用されます。

シグネチャまたは形状

typedef struct {
    enum gbtree_type t;
    int32 eml;
    bool trnc;
    bool (*f_gt) (const void *, const void *, Oid, FmgrInfo *);
    bool (*f_ge) (const void *, const void *, Oid, FmgrInfo *);
    bool (*f_eq) (const void *, const void *, Oid, FmgrInfo *);
    bool (*f_le) (const void *, const void *, Oid, FmgrInfo *);
    bool (*f_lt) (const void *, const void *, Oid, FmgrInfo *);
    int32 (*f_cmp) (const void *, const void *, Oid, FmgrInfo *);
    GBT_VARKEY *(*f_l2n) (GBT_VARKEY *, FmgrInfo *flinfo);
} gbtree_vinfo;

入力と出力

  • **入力**: 各データ型モジュールによる関数ポインタの割り当て。
  • **出力**: GiST 索引操作（consistent, union 等）の実行時に、登録された関数ポインタを通じて比較結果や変換後のキーを返します。

前提条件と不変条件

  • eml はデータベースのエンコーディング最大長を保持し、0 は未定義を意味します。
  • trnc が true の場合、キーの圧縮（切り詰め）が有効であることを示します。
  • 登録される関数ポインタは、btree_utils_var.c 内の汎用関数が期待するシグネチャと一致している必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: 各データ型（btree_text.c 等）が gbtree_vinfo インスタンスを静的に定義し、比較関数や変換関数を登録します。 2. **委譲**: GiST 索引の操作時、btree_utils_var.c の関数がこの構造体を参照し、f_cmp や f_gt などの関数ポインタを呼び出すことで、型固有の比較ロジックを実行します。 3. **変換**: f_l2n はリーフノードのキーを内部ノード用のキーへ変換する際に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_utils_var.c（gbt_var_leaf2node, gbt_var_node_cp_len, gbt_var_node_pf_match 等）
  • **呼び出し先**:
  • 各データ型モジュール（btree_text.c, btree_bytea.c, btree_numeric.c, btree_bit.c）で定義された具体的な比較・変換関数。

関連データ構造

  • GBT_VARKEY: 可変長キーの型定義。
  • GBT_VARKEY_R: 索引内のキーを読み取るための構造体。
  • gbtree_type: データ型を識別する列挙型。

失敗モードと境界条件

  • eml が 0 の場合、マルチバイト文字の処理で予期しない動作をする可能性があります。
  • 関数ポインタが NULL の場合、btree_utils_var.c 側の呼び出しでセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 新しい可変長データ型を追加する場合は、本構造体を実装し、必要な比較関数をすべて提供してください。
  • f_l2n の実装は、リーフノードから内部ノードへの昇格時にキーの整合性を保つよう慎重に設計してください。
