pg_lsn_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:91

目的

クライアントから送信されたバイナリ形式のデータを受け取り、PostgreSQLの pg_lsn 型（内部表現 XLogRecPtr）に変換します。

シグネチャまたは形状

Datum pg_lsn_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して StringInfo 形式のバッファを受け取ります。
  • **出力**: Datum 型として XLogRecPtr（64ビット整数）を返します。

前提条件と不変条件

  • 入力バッファには、ネットワークバイトオーダーでエンコードされた64ビット整数が含まれている必要があります。
  • PG_RETURN_LSN マクロを使用して、変換後の値を適切に Datum 型へラップして返却します。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、引数から StringInfo バッファを取得します。 2. pq_getmsgint64(buf) を呼び出し、バッファから64ビット整数を読み取ります。 3. 読み取った値を XLogRecPtr 型の変数 result に格納します。 4. PG_RETURN_LSN(result) を実行し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのバイナリプロトコル処理層（詳細はソースコード内に明示的な記述なし）。
  • **呼び出し先**: pq_getmsgint64（libpq/pqformat.h で定義）。

関連データ構造

  • XLogRecPtr: LSNを保持する64ビット整数型。
  • StringInfo: バイナリメッセージを保持するための構造体。
  • Datum: PostgreSQLの汎用データ表現型。

失敗モードと境界条件

  • pq_getmsgint64 がバッファの不足や不正な形式を検出した場合、エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は pg_lsn 型のバイナリ入力関数です。LSNの内部表現（uint64）に変更があった場合は、pq_getmsgint64 の使用箇所と整合性を確認してください。
