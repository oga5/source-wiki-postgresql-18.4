AlterRoleSet

定義

  • 種別: function
  • 定義箇所: src/backend/commands/user.c:999

目的

ロールまたはデータベースに対する設定パラメータ（SET 句）の変更を実行します。

シグネチャまたは形状

Oid AlterRoleSet(AlterRoleSetStmt *stmt)

入力と出力

  • **入力**: AlterRoleSetStmt *stmt（変更対象のロール、データベース、および設定値を含む構文解析ノード）。
  • **出力**: Oid（操作対象となったロールのOID。ロールが指定されていない場合は InvalidOid）。

前提条件と不変条件

  • ロールが指定されている場合、そのロールはシステムに存在しなければならない。
  • データベースが指定されている場合、そのデータベースはシステムに存在しなければならない。
  • 権限チェックは、操作対象（ロールまたはデータベース）および操作の種類（グローバル設定か否か）に応じて厳格に行われる。

アルゴリズムまたは振る舞い

1. **ロールの検証とロック**: stmt->role が指定されている場合、get_rolespec_tuple でロールを取得し、shdepLockAndCheckObject でロックをかける。 2. **権限チェック**:

  • 対象ロールがスーパーユーザーの場合、実行者もスーパーユーザーである必要がある。
  • それ以外の場合、実行者は CREATEROLE 権限を持ち、かつ対象ロールに対する ADMIN オプションを持つか、または自分自身のロール設定を変更する場合のみ許可される。

    • 3. **データベースの検証とロック**: stmt->database が指定されている場合、get_database_oid でOIDを取得し、shdepLockAndCheckObject でロックをかける。ロールが未指定の場合は、データベースの所有者であるかを確認する。 4. **グローバル設定の検証**: ロールもデータベースも指定されていない場合、実行者はスーパーユーザーでなければならない。 5. **設定の適用**: AlterSetting を呼び出し、指定された設定変更を永続化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:917（パーサーからのディスパッチ）。
  • **呼び出し先**: check_rolespec_name, get_rolespec_tuple, shdepLockAndCheckObject, superuser, have_createrole_privilege, is_admin_of_role, GetUserId, get_database_oid, object_ownercheck, aclcheck_error, AlterSetting。

関連データ構造

  • AlterRoleSetStmt: 変更対象と設定内容を保持する構造体。
  • Form_pg_authid: pg_authid カタログの行データ構造。

失敗モードと境界条件

  • 予約済みロールの変更を試みた場合、エラーが発生する。
  • 権限不足の場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラー終了する。
  • 指定されたロールやデータベースが存在しない場合、エラーが発生する。

保守担当者向けメモ

  • 本関数は pg_db_role_setting カタログを操作する AlterSetting を呼び出すため、設定の永続化ロジックを変更する際は AlterSetting の実装と併せて確認すること。
  • 権限チェックのロジックはセキュリティの根幹であるため、変更時には superuser 属性や ADMIN オプションの継承ルールに十分注意すること。
