plpgsql_ns_lookup

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_funcs.c:129

目的

PL/pgSQL の名前空間チェーン（PLpgSQL_nsitem のリンクリスト）から、指定された識別子（変数名やラベル）を検索します。

シグネチャまたは形状

PLpgSQL_nsitem *plpgsql_ns_lookup(PLpgSQL_nsitem *ns_cur, bool localmode,
                                  const char *name1, const char *name2, const char *name3,
                                  int *names_used)

入力と出力

  • **入力**:
  • ns_cur: 検索を開始する名前空間アイテムのポインタ。
  • localmode: true の場合、現在のブロックレベルのみを検索し、上位スコープを探索しません。
  • name1, name2, name3: 検索対象の識別子名（修飾名に対応）。
  • names_used: 検索に使用された名前の数（1〜3）を格納する出力用ポインタ。
  • **出力**:
  • 成功時: 見つかった PLpgSQL_nsitem へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • ns_cur は PLpgSQL_nsitem のリンクリストの先頭、または NULL である必要があります。
  • 名前空間チェーンは、ブロックレベルごとに構成されており、各レベルの終端には PLPGSQL_NSTYPE_LABEL 型のアイテムが存在します。

アルゴリズムまたは振る舞い

1. ns_cur を起点に、ブロックレベルごとにループ処理を行います。 2. **非修飾検索**: name1 と一致する変数を現在のブロックレベル内で探索します。見つかった場合、names_used に 1 を設定し、そのアイテムを返します。 3. **修飾検索**: name2 が指定されている場合、現在のブロックレベルのラベル名が name1 と一致するか確認し、一致すれば name2 を変数名として探索します。見つかった場合、names_used に 2 を設定し、そのアイテムを返します。 4. localmode が false の場合、ns_cur を上位のブロックレベル（nsitem->prev）へ更新して探索を継続します。 5. 探索対象が尽きた場合、names_used を 0 に初期化し NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpgsql/src/pl_comp.c 内の複数の箇所（コンパイル時の識別子解決に使用）。
  • **呼び出し先**: strcmp（標準Cライブラリ）。

関連データ構造

  • PLpgSQL_nsitem: 名前空間チェーンを構成するノード。itemtype（PLPGSQL_NSTYPE_VAR や PLPGSQL_NSTYPE_LABEL など）を保持します。

失敗モードと境界条件

  • 指定された名前がどのスコープにも存在しない場合、NULL を返します。
  • localmode が true の場合、現在のブロックレベル外は検索されません。
  • names_used が NULL の場合、そのポインタへの書き込みはスキップされます。

保守担当者向けメモ

  • 本関数はコンパイル時の名前解決の要です。新しい名前空間アイテムの種類を追加する際は、検索ロジックが PLPGSQL_NSTYPE_LABEL を境界として正しく動作するか確認してください。
