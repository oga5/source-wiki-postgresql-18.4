InvalidatePubRelSyncCache

定義

  • 種別: function
  • 定義箇所: src/backend/commands/publicationcmds.c:505

目的

パブリケーションの定義変更に伴い、論理レプリケーションで使用されるリレーション同期キャッシュ（RelSyncCache）を無効化し、レプリケーションワーカーが最新の定義を取得できるようにする。

シグネチャまたは形状

void InvalidatePubRelSyncCache(Oid pubid, bool puballtables)

入力と出力

  • pubid (Oid): 対象となるパブリケーションのOID。
  • puballtables (bool): パブリケーションがすべてのテーブルを含む設定かどうか。
  • 戻り値: なし。

前提条件と不変条件

  • puballtables が真の場合、システム全体の全リレーション同期キャッシュを無効化する。
  • puballtables が偽の場合、指定されたパブリケーションに関連付けられたテーブルおよびスキーマ内のテーブルを特定し、それらのキャッシュを個別に無効化する。

アルゴリズムまたは振る舞い

1. puballtables が真の場合、CacheInvalidateRelSyncAll() を呼び出し、全キャッシュをクリアする。 2. puballtables が偽の場合：

  • GetPublicationRelations および GetAllSchemaPublicationRelations を使用して、パブリケーションに含まれるリレーションのOIDリストを取得する。
  • list_concat_unique_oid で重複を除去したリレーションリストを作成する。
  • リスト内の各リレーションに対して CacheInvalidateRelSync(relid) を呼び出し、個別に無効化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/alter.c:354
  • 呼び出し先: CacheInvalidateRelSyncAll, GetPublicationRelations, GetAllSchemaPublicationRelations, list_concat_unique_oid, CacheInvalidateRelSync

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • List: PostgreSQLのリスト構造（リレーションOIDの保持に使用）。

失敗モードと境界条件

  • パーティションテーブルが含まれる場合、PUBLICATION_PART_ALL フラグを使用して、ルートテーブルだけでなくパーティション（リーフテーブル）もキャッシュ無効化の対象とする必要がある。

保守担当者向けメモ

  • 本関数は論理レプリケーションの整合性に直結するため、パブリケーションの定義変更時には必ず呼び出す必要がある。
  • キャッシュ無効化の漏れは、レプリケーションワーカーが古い定義を参照し続ける原因となる。
