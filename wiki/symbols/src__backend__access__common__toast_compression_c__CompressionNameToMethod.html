CompressionNameToMethod

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/toast_compression.c:284

目的

圧縮アルゴリズムの名称（文字列）を、内部で利用される圧縮方式ID（char）に変換します。

シグネチャまたは形状

char CompressionNameToMethod(const char *compression)

入力と出力

  • **入力**: compression (圧縮アルゴリズム名を表す文字列)
  • **出力**: 対応する圧縮方式ID（TOAST_PGLZ_COMPRESSION または TOAST_LZ4_COMPRESSION）。一致しない場合は InvalidCompressionMethod を返します。

前提条件と不変条件

  • compression が "lz4" であり、かつビルド時に USE_LZ4 が定義されていない場合、NO_LZ4_SUPPORT() マクロによりエラーが発生します。

アルゴリズムまたは振る舞い

1. 入力文字列が "pglz" と一致する場合、TOAST_PGLZ_COMPRESSION を返します。 2. 入力文字列が "lz4" と一致する場合、環境が LZ4 をサポートしているか確認し、TOAST_LZ4_COMPRESSION を返します。 3. 上記いずれにも該当しない場合、InvalidCompressionMethod を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c:22082
  • **呼び出し先**:
  • strcmp (標準ライブラリ)
  • NO_LZ4_SUPPORT (マクロ)

関連データ構造

  • TOAST_PGLZ_COMPRESSION / TOAST_LZ4_COMPRESSION: 圧縮方式を識別する定数。
  • InvalidCompressionMethod: 不正な圧縮方式を示す定数。

失敗モードと境界条件

  • 未サポートのアルゴリズム名が渡された場合、InvalidCompressionMethod が返されます。
  • LZ4 がビルド時に無効化されている環境で "lz4" が指定された場合、関数は値を返さずエラー終了します。

保守担当者向けメモ

  • ToastCompressionId は varlena ヘッダー内の2ビットのみを使用するため、将来的に新しいアルゴリズムを追加する際は、最大4種類までの制限に留意してください。
