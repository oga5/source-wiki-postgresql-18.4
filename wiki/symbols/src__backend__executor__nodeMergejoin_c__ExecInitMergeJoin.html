ExecInitMergeJoin

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMergejoin.c:1438

目的

マージ結合（Merge Join）を実行するための実行ノード（MergeJoinState）を初期化し、必要な式コンテキスト、子ノード、および結合条件の準備を行う。

シグネチャまたは形状

MergeJoinState *ExecInitMergeJoin(MergeJoin *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の MergeJoin プランノード。
  • estate: クエリ実行状態を保持する EState。
  • eflags: 実行フラグ。EXEC_FLAG_BACKWARD および EXEC_FLAG_MARK は指定不可。
  • **出力**:
  • 初期化済みの MergeJoinState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと。
  • node->join.joinqual が存在する場合、node->skip_mark_restore は偽でなければならない。
  • RIGHT JOIN、RIGHT ANTI JOIN、FULL JOIN の場合、結合条件は定数でなければならない（定数でない場合はエラー）。

アルゴリズムまたは振る舞い

1. MergeJoinState 構造体を割り当て、ExecMergeJoin を実行関数として設定する。 2. 結合式評価用の ExprContext を作成する（ノード共通のものとは別に、左右入力用に各1つずつ作成）。 3. 子ノードを初期化し、mj_SkipMarkRestore フラグに基づき EXEC_FLAG_MARK の要否を決定する。 4. Material ノードが子にある場合、条件に応じて mj_ExtraMarks を有効化する。 5. 結合タイプ（JOIN_INNER, JOIN_LEFT 等）に応じて、NULL補完用スロットの初期化やフラグ設定を行う。 6. MJExamineQuals を呼び出し、マージ結合用の結合句を前処理する。 7. ステートマシンを EXEC_MJ_INITIALIZE_OUTER に初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:303
  • **呼び出し先**:
  • makeNode, CreateExprContext, ExecInitNode, ExecInitResultTupleSlotTL, ExecAssignProjectionInfo, ExecInitExtraTupleSlot, ExecInitQual, ExecInitNullTupleSlot, MJExamineQuals, check_constant_qual, ereport 等。

関連データ構造

  • MergeJoinState: 実行時の状態を保持する主要構造体。
  • MergeJoin: プランノード。
  • EState: 実行状態。
  • ExprContext: 式評価コンテキスト。

失敗モードと境界条件

  • RIGHT JOIN / FULL JOIN において、結合条件が定数でない場合に ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。
  • 未知の結合タイプが指定された場合、elog によりエラー終了する。
  • IndexScan 等でサポートされない mj_ExtraMarks を設定しようとした場合、実行時に不整合が生じる可能性がある（ソース内の警告による）。

保守担当者向けメモ

  • 本関数は複雑なステートマシンの初期化を担当している。デバッグ時は MJ1_printf によるトレースが利用可能。
  • mj_SkipMarkRestore の扱いは、結合条件の性質に強く依存するため、変更時は joinqual の制約を確認すること。
