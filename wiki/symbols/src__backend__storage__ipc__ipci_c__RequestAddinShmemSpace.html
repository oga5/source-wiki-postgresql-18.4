RequestAddinShmemSpace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipci.c:73

目的

拡張モジュールが起動時に必要とする共有メモリ領域のサイズを予約するために使用されます。

シグネチャまたは形状

void RequestAddinShmemSpace(Size size)

入力と出力

  • **入力**: size (Size型) - 予約を要求する共有メモリのバイト数。
  • **出力**: なし。内部変数 total_addin_request に加算されます。

前提条件と不変条件

  • process_shmem_requests_in_progress が真である必要があります。このフラグが偽の場合、関数は FATAL エラーで停止します。

アルゴリズムまたは振る舞い

1. process_shmem_requests_in_progress を確認し、許可されていないタイミングであればエラーを発生させます。 2. add_size 関数を用いて、現在の total_addin_request に要求された size を加算します。この際、オーバーフローチェックが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/test/modules/injection_points/injection_points.c
  • src/test/modules/test_aio/test_aio.c
  • src/test/modules/test_slru/test_slru.c
  • **呼び出し先**:
  • add_size (内部的な加算およびオーバーフローチェック用関数)

関連データ構造

  • total_addin_request: 予約された共有メモリの総量を保持する静的変数。
  • process_shmem_requests_in_progress: 予約が許可されている期間を示すフラグ。

失敗モードと境界条件

  • process_shmem_requests_in_progress が偽の状態で呼び出された場合、elog(FATAL) によりプロセスが終了します。
  • add_size 内部でオーバーフローが検出された場合、PostgreSQL の標準的なエラー処理に従いエラーが発生します。

保守担当者向けメモ

  • 本関数は shmem_request_hook が実行されている期間中にのみ呼び出されることを想定しています。
  • 拡張モジュールが共有メモリを必要とする場合、必ずこの関数を介してサイズを通知する必要があります。
