AdjustIntervalForTypmod

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/timestamp.c:1349

目的

Interval 型の値に対し、指定された型修飾子（typmod）に基づいてフィールド範囲の制限および秒以下の精度調整（丸め）を行う。

シグネチャまたは形状

static bool AdjustIntervalForTypmod(Interval *interval, int32 typmod, Node *escontext)

入力と出力

  • **入力**:
  • interval: 調整対象の Interval 構造体へのポインタ。
  • typmod: 型修飾子。負の値の場合は調整を行わない。
  • escontext: エラー報告用コンテキスト（Node*）。
  • **出力**:
  • 成功時は true を返し、interval を直接更新する。
  • 失敗時は false を返す（escontext が設定されている場合）。

前提条件と不変条件

  • interval が無限大（INTERVAL_NOT_FINITE）である場合、調整は行われず即座に true を返す。
  • typmod が負の場合、調整は行われない。
  • precision が 0 未満または MAX_INTERVAL_PRECISION を超える場合はエラーとなる。
  • 算術演算（丸め処理）においてオーバーフローが発生した場合、エラーとして報告される。

アルゴリズムまたは振る舞い

1. **範囲制限**: typmod から抽出された range に基づき、不要なフィールド（month, day, time）を 0 にクリアする。例えば YEAR 指定時は day と time を 0 に設定する。 2. **精度調整**: precision が指定されている場合、IntervalScales および IntervalOffsets を用いて interval->time を丸める。

  • 正の値の場合: time + offset を計算し、scale で剰余演算を行って切り捨てる。
  • 負の値の場合: time - offset を計算し、同様に切り捨てる。

    • 3. **エラーハンドリング**: pg_add_s64_overflow または pg_sub_s64_overflow を使用して、丸め処理中のオーバーフローを検知する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/timestamp.c 内の複数の箇所から、Interval 型の生成や変換時に呼び出される。
  • **呼び出し先**: 内部で pg_add_s64_overflow、pg_sub_s64_overflow、elog、ereturn を使用する。

関連データ構造

  • Interval: month, day, time を保持する構造体。
  • IntervalScales / IntervalOffsets: 秒以下の精度調整に使用される静的定数配列。

失敗モードと境界条件

  • typmod が認識できない値の場合: elog(ERROR, ...) により中断される。
  • precision が範囲外の場合: ERRCODE_INVALID_PARAMETER_VALUE でエラーとなる。
  • 丸め処理で値が範囲を超えた場合: ERRCODE_DATETIME_VALUE_OUT_OF_RANGE でエラーとなる。

保守担当者向けメモ

  • 本関数は PostgreSQL 8.4 以降、高位フィールドのモジュロ演算を廃止し、低位フィールドの切り捨て・丸めのみを行う仕様となっている。
  • interval_support 関数等で将来的に範囲制限のロジックを強化する場合、本関数の実装と整合性を取る必要がある。
