DECountItem

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/array_typanalyze.c:77

目的

配列型の統計解析（ANALYZE）において、配列内に含まれる「一意な要素数」の分布を記録するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    int count;      /* 配列内の一意な要素数 */
    int frequency;  /* その要素数を持つ配列の出現頻度 */
} DECountItem;

入力と出力

本構造体は、compute_array_stats 内でハッシュテーブル（count_tab）のエントリとして使用されます。

  • **入力**: 配列のデコード結果から得られた一意な要素数（distinct_count）。
  • **出力**: 統計情報収集プロセスにおいて、pg_statistic の DECHIST（要素数ヒストグラム）を生成するための集計データ。

前提条件と不変条件

  • count_tab ハッシュテーブルの entrysize として sizeof(DECountItem) が指定されている必要があります。
  • 統計収集の最終段階で、DECountItem のポインタ配列が作成され、頻度に基づいたソートが行われます。

アルゴリズムまたは振る舞い

1. compute_array_stats 内で、各配列の一意な要素数をキーとしてハッシュテーブルを検索します。 2. 該当する DECountItem が存在しない場合は新規作成し、存在する場合は frequency をインクリメントします。 3. 収集終了後、hash_seq_search を用いて全エントリを走査し、ポインタ配列に格納してソート（countitem_compare_count 等を使用）することで、要素数の分布を確定させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_typanalyze.c 内の compute_array_stats 関数。
  • **呼び出し先**: 比較関数 countitem_compare_count が本構造体のソート時に呼び出されます。

関連データ構造

  • count_tab: DECountItem を格納するハッシュテーブル。
  • ArrayAnalyzeExtraData: 配列解析の補助データ構造。

失敗モードと境界条件

  • メモリ不足時: palloc によるポインタ配列の確保に失敗する可能性があります。
  • 境界条件: 配列が空の場合や、要素数が 0 の場合の挙動は compute_array_stats のロジックに依存します。

保守担当者向けメモ

  • 本構造体は pg_statistic の DECHIST スロットに書き込まれる統計情報の基礎となります。
  • countitem_compare_count によるソート順序を変更する場合は、統計情報の生成ロジックへの影響を確認してください。
