pg_ceil_log2_64

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_bitutils.h:270

目的

与えられた 64 ビット整数 num に対し、ceil(log2(num)) を計算します。これは、num を格納するために必要な最小のビット数（2 のべき乗の指数）を求める際に使用されます。

シグネチャまたは形状

static inline uint64 pg_ceil_log2_64(uint64 num)

入力と出力

  • **入力**: uint64 num（対象となる 64 ビット符号なし整数）
  • **出力**: uint64（num を超えない最小の 2 のべき乗の指数。num <= 1 の場合は 0 を返す）

前提条件と不変条件

  • num が 0 または 1 の場合、戻り値は 0 となります。
  • 内部で pg_leftmost_one_pos64 を呼び出しているため、num - 1 が 0 になる場合（num=1）の挙動は、条件分岐によって保護されています。

アルゴリズムまたは振る舞い

1. num < 2 の場合、即座に 0 を返します。 2. それ以外の場合、num - 1 の最上位ビット位置（MSB）を pg_leftmost_one_pos64 で取得し、それに 1 を加算した値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/sysv_shmem.c:552
  • src/backend/utils/hash/dynahash.c:1809
  • **呼び出し先**:
  • pg_leftmost_one_pos64（num > 1 の場合）

関連データ構造

  • 特になし（プリミティブ型 uint64 のみを使用）。

失敗モードと境界条件

  • num = 0: 戻り値は 0 です。
  • num = 1: 戻り値は 0 です。
  • num が非常に大きい場合: pg_leftmost_one_pos64 の実装に依存しますが、64 ビット整数の範囲内で正しく動作します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • pg_leftmost_one_pos64 の実装がプラットフォームごとに最適化されているため、本関数のパフォーマンスもそれに準じます。
