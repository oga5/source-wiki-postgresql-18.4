verify_tar_backup

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:789

目的

バックアップディレクトリ内の tar ファイルを走査し、各アーカイブの内容をストリーミング形式で検証する。

シグネチャまたは形状

static void verify_tar_backup(verifier_context *context, DIR *dir)

入力と出力

  • **入力**:
  • context: 検証設定および状態を保持する verifier_context 構造体へのポインタ。
  • dir: 走査対象のディレクトリを示す DIR ポインタ。
  • **出力**: なし（副作用として検証結果を報告）。

前提条件と不変条件

  • context->format が 'p'（plain形式）ではないこと。
  • dir は有効なディレクトリポインタであること。

アルゴリズムまたは振る舞い

1. **第1パス（ディレクトリ走査）**:

  • 指定されたディレクトリを読み込み、. および .. を除外する。
  • should_ignore_relpath で無視対象でないファイルに対し、precheck_tar_backup_file を実行して tarfiles リストに登録する。
  • 走査終了後、ディレクトリを閉じる。

    • 2. **第2パス（検証実行）**:

  • tarfiles リストを順次処理する。
  • create_archive_verifier で圧縮形式に応じた astreamer スタックを構築する。
  • verify_tar_file を呼び出し、アーカイブの読み込み、解凍、および検証を行う。
  • 各ファイル処理後に astreamer_finalize および astreamer_free でリソースを解放する。

    • 3. **終了処理**:

  • simple_ptr_list_destroy でリストを破棄し、進捗報告を完了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_verifybackup/pg_verifybackup.c 内の main 関数（行353）。
  • **呼び出し先**:
  • progress_report, readdir, should_ignore_relpath, psprintf, precheck_tar_backup_file, pfree, closedir, report_backup_error, create_archive_verifier, verify_tar_file, astreamer_finalize, astreamer_free, simple_ptr_list_destroy。

関連データ構造

  • verifier_context: 検証の全体状態を管理。
  • SimplePtrList: 検証対象の tar ファイルを保持するリスト。
  • tar_file: 個別の tar ファイル情報を保持する構造体。
  • astreamer: アーカイブのストリーム処理用スタック。

失敗モードと境界条件

  • ディレクトリのクローズに失敗した場合、report_backup_error を通じてエラーを報告し、処理を中断する。
  • should_ignore_relpath により、バックアップに含まれる不要なファイルは検証対象から除外される。

保守担当者向けメモ

  • 本関数は tar 形式のバックアップ検証におけるメインループである。
  • astreamer スタックの構築と破棄が各ファイルごとに行われるため、メモリリークを防ぐために astreamer_free の呼び出しを維持すること。
