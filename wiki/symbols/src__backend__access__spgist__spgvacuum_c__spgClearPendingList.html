spgClearPendingList

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgvacuum.c:89

目的

SP-GiST の VACUUM 処理中に保持された保留リスト（pending list）を解放し、メモリをクリーンアップします。

シグネチャまたは形状

static void spgClearPendingList(spgBulkDeleteState *bds)

入力と出力

  • **入力**: spgBulkDeleteState *bds - VACUUM 実行中の状態を保持する構造体。
  • **出力**: なし（副作用として bds->pendingList が NULL に更新され、リスト内の全要素がメモリ解放されます）。

前提条件と不変条件

  • リスト内のすべての spgVacPendingItem 要素は、この関数が呼び出される前に処理済み（pitem->done が真）である必要があります。これに違反するとアサーションエラーが発生します。

アルゴリズムまたは振る舞い

1. bds->pendingList を先頭から順に走査します。 2. 各要素（pitem）に対して Assert(pitem->done) を実行し、処理済みであることを確認します。 3. pfree(pitem) を呼び出し、各要素のメモリを解放します。 4. 最後に bds->pendingList を NULL に設定し、リストを空にします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgvacuum.c 内の spgbulkdelete 関数から呼び出されます。
  • **呼び出し先**: pfree（メモリ解放関数）。

関連データ構造

  • spgBulkDeleteState: 保留リストの起点（pendingList）を保持します。
  • spgVacPendingItem: リストを構成する個々の要素。

失敗モードと境界条件

  • リストが空（bds->pendingList == NULL）の場合、ループは実行されず、単に bds->pendingList が NULL に再設定されます。
  • リスト内に done フラグが偽の要素が含まれている場合、アサーションが有効なビルドではプログラムが停止します。

保守担当者向けメモ

  • 本関数は spgbulkdelete の終了処理の一部として実行されます。リスト内のアイテムが適切に処理されていない状態で呼び出すことは想定されていません。
