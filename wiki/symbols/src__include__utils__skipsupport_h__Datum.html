Datum

定義

  • 種別: typedef
  • 定義箇所: src/include/utils/skipsupport.h:51

目的

PostgreSQL において、任意のデータ型を汎用的に扱うための不透明なデータ表現型です。本モジュール（skipsupport.h）においては、インデックススキャン時の値の受け渡しや、演算子クラスによる値の増減処理における引数および戻り値として利用されます。

シグネチャまたは形状

typedef uintptr_t Datum;

*(注: PostgreSQL の基盤定義に基づく一般的な形状であり、本ソース excerpt 内では SkipSupportIncDec 関数の引数および戻り値の型として現れます)*

入力と出力

  • **入力**: データベース内の任意のデータ型（整数、日付、文字列等）を Datum 型として受け取ります。
  • **出力**: 演算の結果や、特定のデータ型に対応する値を Datum 型として返します。

前提条件と不変条件

  • SkipSupportIncDec 関数において、引数 existing に NULL が渡されることはありません。
  • Datum は値そのもの、あるいは値へのポインタとして扱われるため、対象となるデータ型の特性に応じた適切な解釈が必要です。

アルゴリズムまたは振る舞い

Datum は、B-Tree スキップスキャンにおいて、現在のインデックス値から次の値を計算する際の「値のコンテナ」として機能します。SkipSupportIncDec 関数は、この Datum を入力として受け取り、型固有のロジックで増減させた新しい Datum を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck モジュール内の各検証関数（verify_gin.c, verify_heapam.c, verify_nbtree.c 等）で、インデックスキーやタプル値の参照として広範に使用されています。
  • **呼び出し先**: 本シンボル自体は型定義であるため、直接の呼び出し先はありません。Datum を操作する関数（SkipSupportIncDec 等）が、各データ型固有の演算関数を呼び出します。

関連データ構造

  • SkipSupportIncDec: Datum を引数および戻り値に持つ関数ポインタ型。
  • SkipSupportData: スキップスキャンに必要な状態を保持する構造体。

失敗モードと境界条件

  • **境界条件**: SkipSupportIncDec 関数内で値の増減を行った結果、型の最大値・最小値を超えた場合、overflow フラグが true に設定されます。
  • **失敗モード**: Datum 自体の型定義に起因する失敗はありませんが、Datum を解釈する関数側で型不整合が発生した場合、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • Datum は PostgreSQL 全体で多用される基盤型です。本ヘッダーファイルにおける利用は、あくまでスキップスキャンという特定の文脈における値の抽象化であることを理解してください。
  • SkipSupportIncDec を実装する際は、Datum が指すメモリ領域のライフサイクルに注意し、メモリリークが発生しないよう設計してください。
