pg_cryptohash_errno

定義

  • 種別: enum
  • 定義箇所: src/common/cryptohash.c:44

目的

暗号学的ハッシュ関数（MD5, SHA1, SHA2系）の処理中に発生したエラー状態を識別するために使用されます。

シグネチャまたは形状

typedef enum pg_cryptohash_errno
{
    PG_CRYPTOHASH_ERROR_NONE = 0,
    PG_CRYPTOHASH_ERROR_DEST_LEN,
} pg_cryptohash_errno;

入力と出力

  • **出力**: 列挙型定数（PG_CRYPTOHASH_ERROR_NONE または PG_CRYPTOHASH_ERROR_DEST_LEN）。

前提条件と不変条件

  • PG_CRYPTOHASH_ERROR_NONE はエラーが発生していない正常な状態を示します。
  • PG_CRYPTOHASH_ERROR_DEST_LEN は、主に pg_cryptohash_final において、出力バッファのサイズが不足している場合に発生します。

アルゴリズムまたは振る舞い

ハッシュ計算処理の各段階でエラーが発生した際、この列挙型を用いて状態が記録されます。呼び出し元は、この値を参照することでエラーの種別を特定できます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash.c および src/common/cryptohash_openssl.c 内の関数が、エラー状態を保持する変数としてこの型を参照しています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算のコンテキストを保持する構造体。本エラー型は、このコンテキストに関連するエラー状態を管理するために利用されます。

失敗モードと境界条件

  • PG_CRYPTOHASH_ERROR_DEST_LEN: pg_cryptohash_final 実行時に、結果を格納するバッファサイズがアルゴリズムの要求を満たさない場合に発生します。

保守担当者向けメモ

  • 本列挙型は src/common/cryptohash.c と src/common/cryptohash_openssl.c の両方で利用されています。エラーコードを追加する際は、両方の実装で整合性が取れるように注意してください。
