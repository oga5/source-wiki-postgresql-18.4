ExecFunctionScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeFunctionscan.c:264

目的

SQL クエリの FROM 句に記述されたテーブル関数（RangeFunctions）をスキャンし、結果セットをクエリ実行エンジンへ提供する。

シグネチャまたは形状

TupleTableSlot *ExecFunctionScan(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（実行中のプランノードの状態）
  • **出力**: TupleTableSlot *（スキャンされたタプルを格納するスロット）

前提条件と不変条件

  • pstate は FunctionScanState 型にキャスト可能であること。
  • ExecScan を介して呼び出される実行ノードとして機能する。

アルゴリズムまたは振る舞い

1. pstate を FunctionScanState にキャストする。 2. ExecScan を呼び出し、スキャン処理を委譲する。 3. ExecScan の内部で、タプル取得のために FunctionNext を、再チェックのために FunctionRecheck を使用する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecProcNode インターフェースを介してクエリ実行エンジンから呼び出される（scanstate->ss.ps.ExecProcNode = ExecFunctionScan; による登録）。
  • **呼び出し先**:
  • castNode (マクロ)
  • ExecScan
  • FunctionNext (関数ポインタとして渡される)
  • FunctionRecheck (関数ポインタとして渡される)

関連データ構造

  • FunctionScanState: ノードの実行時状態を保持する構造体。
  • PlanState: 実行プランの共通状態構造体。
  • TupleTableSlot: タプルを保持するスロット構造体。

失敗モードと境界条件

  • 実行ノードの初期化が不完全な場合、castNode による型変換でエラーが発生する可能性がある。
  • FunctionNext および FunctionRecheck の内部ロジックに依存する失敗モード（メモリ不足や関数実行エラー等）が存在する。

保守担当者向けメモ

  • 本関数は ExecScan のラッパーであり、実質的なロジックは FunctionNext に集約されている。
  • ExecProcNode への登録を通じて呼び出されるため、直接的な呼び出し箇所を追跡する際は ExecProcNode の呼び出し元を確認すること。
