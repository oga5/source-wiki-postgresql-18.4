g_int_compress

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_gist.c:162

目的

GiST インデックス挿入時に、整数配列を範囲（range）の集合に変換してインデックスサイズを削減する。

シグネチャまたは形状

Datum g_int_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（GISTENTRY 構造体を含む）
  • **出力**: Datum（圧縮済み、または元の GISTENTRY を指すポインタ）

前提条件と不変条件

  • entry->leafkey が真の場合、入力配列の要素数が 2 * G_INT_GET_NUMRANGES() 未満であること。
  • 内部ノードの圧縮時、結果のスパース性が MAXNUMELTS を超えないこと。

アルゴリズムまたは振る舞い

1. **リーフノード**: 入力配列をコピーし、サイズ制限を超えていないか検証して返却する。 2. **内部ノード**:

  • 配列が空であればそのまま返却する。
  • 要素数が 2 * G_INT_GET_NUMRANGES() 以上の場合、連続する整数を範囲表現（開始値と終了値のペア）にマージする。
  • マージ後も要素数が上限を超える場合、範囲間の距離が最小となる箇所を順次削除してサイズを縮小する。
  • 最終的なスパース性を internal_size で確認し、許容範囲内であれば圧縮後の配列を格納した GISTENTRY を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセス（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: resize_intArrayType, internal_size, gistentryinit, palloc, pfree 等のユーティリティ関数。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • ArrayType: PostgreSQL の配列型データ構造。

失敗モードと境界条件

  • **ERRCODE_PROGRAM_LIMIT_EXCEEDED**:
  • リーフノードにおいて入力配列が大きすぎる場合。
  • 内部ノードにおいて圧縮後のデータがスパースすぎる場合。
  • いずれの場合も gist__intbig_ops オペレータクラスの使用を推奨するエラーメッセージが発行される。

保守担当者向けメモ

  • 本関数は標準的な gist__int_ops 用です。大規模またはスパースなデータセットに対しては、_intbig_gist.c で定義される gist__intbig_ops が適切である点に留意してください。
