FPne

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/geo_decls.h:52

目的

浮動小数点数 A と B が、許容誤差 EPSILON を考慮した上で「等しくない」かどうかを判定する。

シグネチャまたは形状

bool FPne(double A, double B)

入力と出力

  • **入力**: 比較対象の浮動小数点数 A および B。
  • **出力**: A と B が厳密に異なり、かつその差の絶対値が EPSILON を超える場合に true を返す。それ以外の場合は false を返す。

前提条件と不変条件

  • math.h の fabs 関数に依存する。
  • EPSILON 定数による曖昧な比較を行う。
  • NaN（非数）が含まれる場合、比較結果は常に false となる。

アルゴリズムまたは振る舞い

以下の論理式を評価する: A != B && fabs(A - B) > EPSILON

1. まず A != B を評価し、厳密に異なるかを確認する。 2. 次に fabs(A - B) が EPSILON より大きいかを確認する。 3. 両方の条件を満たす場合にのみ true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/geo_ops.c:4917 において、円の面積比較に使用されている。
  • **呼び出し先**: math.h の fabs 関数。

関連データ構造

  • 本関数は double 型のプリミティブ値を操作対象とする。

失敗モードと境界条件

  • **NaN**: NaN を含む比較では常に false を返す。
  • **推移律**: 浮動小数点演算の性質上、推移律（a != b かつ b != c ならば a != c）を満たさない可能性がある。

保守担当者向けメモ

  • 数値解析の専門的な実装ではないため、厳密な精度が求められる計算での使用には注意が必要である。
  • 同一ヘッダー内で #define FPne(A,B) ((A) != (B)) と定義されている箇所（84行目）があるため、ビルド環境やマクロの有効範囲に留意すること。
