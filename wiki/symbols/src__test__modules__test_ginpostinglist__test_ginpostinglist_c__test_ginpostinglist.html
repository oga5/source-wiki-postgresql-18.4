test_ginpostinglist

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_ginpostinglist/test_ginpostinglist.c:87

目的

GINインデックスのポスティングリストにおける可変長バイトエンコーディング（varbyte-encoding）の正確性を検証するためのテストエントリポイントです。

シグネチャまたは形状

Datum test_ginpostinglist(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（PostgreSQLの関数呼び出し規約に基づく引数）
  • 出力: void（PG_RETURN_VOID() を介して返却）

前提条件と不変条件

  • 本関数はPostgreSQLの拡張モジュールとして実行される必要があります。
  • 内部で呼び出される test_itemptr_pair が、指定されたブロック番号とオフセット番号のペアに対して圧縮・展開のラウンドトリップテストを成功させることが期待されます。

アルゴリズムまたは振る舞い

1. 異なるブロック番号およびオフセット番号の組み合わせを用いて、test_itemptr_pair を4回呼び出します。

  • (0, 2) をサイズ制限 14 でテスト
  • (0, MaxHeapTuplesPerPage) をサイズ制限 14 でテスト
  • (MaxBlockNumber, MaxHeapTuplesPerPage) をサイズ制限 14 でテスト
  • (MaxBlockNumber, MaxHeapTuplesPerPage) をサイズ制限 16 でテスト

    • 2. 各テストケースにおいて、圧縮アルゴリズムの整合性とサイズ制限の遵守を検証します。 3. 処理完了後、PG_RETURN_VOID() を返します。

呼び出し元と呼び出し先

  • 呼び出し元: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録済み）。
  • 呼び出し先: test_itemptr_pair（各テストケースの実行用）。

関連データ構造

  • ItemPointer: GINインデックスで管理されるTID（Tuple Identifier）。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • test_itemptr_pair 内部で圧縮結果が期待値と一致しない場合、またはサイズ制限を超過した場合は elog(ERROR) が発行され、トランザクションがアボートされます。
  • MaxBlockNumber や MaxHeapTuplesPerPage といった境界値を用いたテストにより、エンコーディングのオーバーフローや不正なデータ変換を検出します。

保守担当者向けメモ

  • 本関数は単体テスト用です。新しいテストケースを追加する場合は、test_itemptr_pair を用いて同様のパターンで呼び出しを追加してください。
  • 実行には CREATE EXTENSION test_ginpostinglist; が必要です。
