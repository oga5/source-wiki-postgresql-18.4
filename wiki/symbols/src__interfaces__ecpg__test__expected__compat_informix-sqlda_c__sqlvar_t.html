sqlvar_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_informix-sqlda.c:35
  • 別名定義: typedef struct sqlvar_compat sqlvar_t;

目的

ECPG（Embedded SQL for C）の Informix 互換モードにおいて、動的 SQL の実行時に個々の列のデータ型、データポインタ、およびインジケータ情報を保持するために使用されます。

シグネチャまたは形状

sqlvar_compat 構造体への型定義です。具体的なメンバ構成は sqlda-compat.h に依存します。

入力と出力

  • **入力**: 動的 SQL の実行時、malloc でメモリを確保し、memset で初期化した上で sqlda_t 構造体のメンバとしてバインドされます。
  • **出力**: FETCH 文や EXECUTE 文の結果セットとして、データベースから取得された列データがこの構造体を介してアプリケーションに渡されます。

前提条件と不変条件

  • sqlda_t 構造体の一部として管理される必要があります。
  • 使用前には malloc によるメモリ確保と、memset 等によるゼロ初期化が推奨されます。

アルゴリズムまたは振る舞い

動的 SQL の実行フローにおいて以下の役割を果たします。 1. **構築**: inp_sqlda->sqlvar = malloc(sizeof(sqlvar_t)) によりメモリが割り当てられます。 2. **初期化**: memset(inp_sqlda->sqlvar, 0, sizeof(sqlvar_t)) により、構造体内の各フィールドが安全な状態にリセットされます。 3. **データ保持**: データベースから取得された値や、クエリに渡すパラメータ値を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/compat_informix-sqlda.c 内の動的 SQL 実行処理（EXECUTE 文の準備フェーズなど）。
  • **呼び出し先**: 構造体内部のメンバ変数（データポインタや型情報）が ECPG ライブラリ関数によって参照されます。

関連データ構造

  • sqlda_t: sqlvar_t を保持する親構造体。
  • sqlda-compat.h: 本構造体の定義元。

失敗モードと境界条件

  • メモリ割り当て失敗: malloc が失敗した場合、sqlvar ポインタが NULL となり、その後のアクセスでセグメンテーション違反が発生する可能性があります。
  • 初期化不足: memset を行わずに使用した場合、不定な値がデータ型やポインタとして解釈され、予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は _ECPG_INFORMIX_H マクロの定義状況により、sqlda-native.h 側の定義と切り替わる可能性があります。
  • 構造体のメンバ詳細を確認する場合は、sqlda-compat.h を参照してください。
