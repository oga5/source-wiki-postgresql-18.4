gbt_int4_dist

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int4.c:70

目的

GiST インデックス内において、2つの int4 値間の距離を計算します。

シグネチャまたは形状

static float8 gbt_int4_dist(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象となる int4 値へのポインタ。
  • flinfo: 関数管理情報（現在は使用されていません）。
  • **出力**: 2つの値の差の絶対値を float8 型で返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な int4 値を指している必要があります。
  • GET_FLOAT_DISTANCE マクロの仕様に基づき、オーバーフローを考慮した安全な距離計算が行われます。

アルゴリズムまたは振る舞い

1. GET_FLOAT_DISTANCE マクロを呼び出します。 2. マクロ内で int32 型として値をキャストし、その差分を計算して float8 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の参照情報に基づくと、contrib/btree_gist/btree_int4.c 内の GiST サポート関数テーブル等から参照されている可能性がありますが、具体的な直接の呼び出し元はソース抜粋からは特定できません。
  • **呼び出し先**: GET_FLOAT_DISTANCE マクロ（btree_utils_num.h で定義）を呼び出します。

関連データ構造

  • int4: PostgreSQL の 4 バイト整数型。
  • float8: 距離計算結果の型。

失敗モードと境界条件

  • 入力値が int4 の範囲外である場合や、ポインタが NULL の場合の挙動については、本関数単体では制御していません。

保守担当者向けメモ

  • 本関数は GiST インデックスの近傍検索（距離計算）をサポートするための内部関数です。
  • SQL から直接呼び出される int4_dist 関数とは別物であることに注意してください。
  • ロジックの詳細は btree_utils_num.h の GET_FLOAT_DISTANCE マクロに依存しています。
