ApplySortAbbrevFullComparator

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/sortsupport.h:340

目的

省略キー（Abbreviated Key）を用いた比較処理において、結果が不確定（等価とみなされるが、実際には異なる可能性がある場合）となった際に、信頼できる完全な比較を行うためのフォールバック関数。

シグネチャまたは形状

int ApplySortAbbrevFullComparator(Datum datum1, bool isNull1,
                                  Datum datum2, bool isNull2,
                                  SortSupport ssup);

入力と出力

  • **入力**:
  • datum1, datum2: 比較対象のデータ。
  • isNull1, isNull2: 各データが NULL かどうかを示すフラグ。
  • ssup: ソート状態を保持する SortSupportData 構造体へのポインタ。
  • **出力**:
  • 比較結果を整数で返す（datum1 < datum2 なら負、datum1 == datum2 なら 0、datum1 > datum2 なら正）。

前提条件と不変条件

  • ssup は初期化済みであること。
  • ssup->abbrev_full_comparator が有効な関数ポインタとして設定されていること。
  • NULL 処理は ssup->ssup_nulls_first に基づいて正規化される。

アルゴリズムまたは振る舞い

1. **NULL 判定**: 両方の NULL 状態を比較し、NULL の配置設定（ssup_nulls_first）に従って結果を決定する。 2. **完全比較**: 両者が非 NULL の場合、ssup->abbrev_full_comparator を呼び出して比較を実行する。 3. **順序反転**: ssup->ssup_reverse が真の場合、比較結果を反転させる。 4. **戻り値**: 最終的な比較結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesortvariants.c 内のソート処理ループ（行 1268, 1414, 1630, 2015）。
  • **呼び出し先**: ssup->abbrev_full_comparator（SortSupportData 内の関数ポインタ）。

関連データ構造

  • SortSupportData: ソートの実行状態、比較関数、NULL 処理設定を保持する中心的な構造体。

失敗モードと境界条件

  • ssup->abbrev_full_comparator が未設定の場合、呼び出し時にクラッシュする可能性がある。
  • NULL 比較において、ssup->ssup_nulls_first の設定が正しくない場合、期待されるソート順序と矛盾する結果を返す。

保守担当者向けメモ

  • 本関数はインライン化されており、ソートのパフォーマンスに直結するため、ロジックの変更は慎重に行うこと。
  • 省略キーの比較が不確定な場合にのみ呼び出される設計であるため、この関数自体が高速である必要はないが、正確性は必須である。
