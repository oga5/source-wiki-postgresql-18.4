SpGistUpdateMetaPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgutils.c:449

目的

SP-GiSTインデックスのメタページ（SPGIST_METAPAGE_BLKNO）を、メモリ上のキャッシュ状態（SpGistCache）と同期させるために使用されます。

シグネチャまたは形状

void SpGistUpdateMetaPage(Relation index)

入力と出力

  • **入力**: Relation index - 対象となるSP-GiSTインデックスのリレーション構造体。
  • **出力**: なし（副作用としてメタページを更新）。

前提条件と不変条件

  • index->rd_amcache が SpGistCache 型として初期化されていること。
  • メタページの pd_lower は、メタデータ構造体の直後に設定される必要がある（WAL圧縮時のデータ消失を防ぐため）。

アルゴリズムまたは振る舞い

1. index->rd_amcache からキャッシュ情報を取得する。 2. SPGIST_METAPAGE_BLKNO を読み込み、条件付きロック（ConditionalLockBuffer）を試みる。 3. ロック取得に成功した場合：

  • キャッシュ内の lastUsedPages をメタページのメタデータに反映する。
  • メタページの pd_lower を SpGistMetaPageData の末尾位置に更新する。
  • バッファをダーティとしてマークし、ロックを解除して解放する。

    • 4. ロック取得に失敗した場合：

  • バッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spginsert.c
  • src/backend/access/spgist/spgvacuum.c
  • **呼び出し先**:
  • ReadBuffer, ConditionalLockBuffer, BufferGetPage, SpGistPageGetMeta, MarkBufferDirty, UnlockReleaseBuffer, ReleaseBuffer

関連データ構造

  • SpGistCache: インデックスの実行時キャッシュ。
  • SpGistMetaPageData: メタページ内に格納されるSP-GiST固有のメタデータ。

失敗モードと境界条件

  • ConditionalLockBuffer が失敗した場合、メタページの更新はスキップされ、バッファは単に解放される。
  • rd_amcache が NULL の場合、何も行われない。

保守担当者向けメモ

  • 本関数はWALログを最小限にする設計となっている。
  • pd_lower の更新は、古いバージョンのPostgreSQL（v11未満）で作成されたインデックスとの互換性を維持するために必須である。
