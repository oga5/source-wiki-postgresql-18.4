ExecReScanAppend

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeAppend.c:420

目的

Append ノードの再スキャン（ReScan）を実行し、子プランの状態をリセットして、クエリ実行を最初からやり直せる状態にする。

シグネチャまたは形状

void ExecReScanAppend(AppendState *node)

入力と出力

  • **入力**: AppendState *node - 再スキャン対象の Append ノードの実行時状態。
  • **出力**: なし（副作用として node の内部状態および各子プランの状態が更新される）。

前提条件と不変条件

  • node が有効な AppendState 構造体を指していること。
  • パーティションプルーニングに使用される PARAM_EXEC パラメータが変更された場合、as_valid_subplans が適切にクリアされ、再選択が必要な状態になること。

アルゴリズムまたは振る舞い

1. **パーティションプルーニングの再評価**: as_prune_state が存在し、chgParam に関連するパラメータが含まれる場合、有効なサブプランのリスト（as_valid_subplans, as_valid_asyncplans）を破棄し、再選択フラグをリセットする。 2. **子プランの再スキャン**: すべてのサブプランに対して UpdateChangedParamSet を呼び出し、パラメータ変更を伝播させる。その後、chgParam が空のサブプランに対して ExecReScan を実行する。 3. **非同期状態のリセット**: 非同期サブプランが存在する場合、各 AsyncRequest のフラグ（callback_pending, request_complete）をクリアし、結果バッファをリセットする。 4. **実行状態の初期化**: as_whichplan を INVALID_SUBPLAN_INDEX に戻し、as_syncdone および as_begun を false に設定して、次回の ExecAppend 呼び出しで最初のサブプランから処理が開始されるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:146（ExecReScan 経由）。
  • **呼び出し先**: UpdateChangedParamSet, ExecReScan（各子プランに対して）、bms_next_member（非同期リクエストの走査）。

関連データ構造

  • AppendState: Append ノードの実行時状態を保持。
  • AsyncRequest: 非同期実行のリクエスト状態。
  • PlanState: 各子プランの実行状態。

失敗モードと境界条件

  • as_nasyncplans が 0 の場合、非同期状態のリセット処理はスキップされる。
  • chgParam が NULL の場合、パラメータ変更の伝播は行われない。

保守担当者向けメモ

  • 本関数は ExecReScan インターフェースの一部であり、Append ノード特有の「子プランの切り替え」と「非同期リクエストの管理」をリセットする責任を持つ。
  • パーティションプルーニングの動的再評価は、クエリ実行中にパラメータが変化するシナリオ（例: ネストされたループ内でのスキャン）で重要となる。
