change_directory

定義

  • 種別: function
  • 定義箇所: src/timezone/zic.c:561

目的

指定されたディレクトリへ作業ディレクトリを変更します。ディレクトリが存在しない場合は作成を試みます。

シグネチャまたは形状

static void change_directory(char const *dir)

入力と出力

  • **入力**: dir (移動先ディレクトリパスを示す文字列)
  • **出力**: なし（成功時は作業ディレクトリが変更される。失敗時はエラーメッセージを出力してプロセスを終了する）

前提条件と不変条件

  • chdir システムコールが利用可能であること。
  • mkdirs 関数がディレクトリ作成のために利用可能であること。

アルゴリズムまたは振る舞い

1. chdir(dir) を実行し、成功すれば終了する。 2. 失敗した場合、エラーコードが ENOENT（ディレクトリが存在しない）であれば、mkdirs(dir, false) を呼び出してディレクトリの作成を試みる。 3. 作成後、再度 chdir(dir) を実行する。 4. いずれの段階でもディレクトリ変更に失敗した場合、stderr にエラーメッセージを出力し、exit(EXIT_FAILURE) でプログラムを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/timezone/zic.c 内のメイン処理フロー（コマンドライン引数で指定された出力先ディレクトリへの移動など）。
  • **呼び出し先**: chdir (標準ライブラリ), mkdirs (本ファイル内のディレクトリ作成関数)。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 指定されたパスがディレクトリではない、または権限不足により chdir が失敗する場合、プログラムは即座に終了する。
  • mkdirs によるディレクトリ作成が失敗した場合も、同様に終了する。

保守担当者向けメモ

  • 本関数は zic ユーティリティがバイナリ出力先を制御するために使用されます。
  • エラー発生時の終了処理は EXIT_FAILURE を使用しており、呼び出し元でエラーハンドリングを継続する設計にはなっていません。
