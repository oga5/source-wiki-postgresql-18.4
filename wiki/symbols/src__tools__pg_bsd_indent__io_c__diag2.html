diag2

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/io.c:589

目的

整形処理中に発生した警告またはエラーメッセージを、現在の行番号とともに標準出力または標準エラー出力へ報告する。

シグネチャまたは形状

void diag2(int level, const char *msg)

入力と出力

  • **入力**:
  • level: 0 の場合は「Warning」、それ以外の場合は「Error」として分類される。
  • msg: 出力するメッセージ文字列。
  • **出力**:
  • output が stdout の場合: /**INDENT** [Warning|Error]@[line_no]: [msg] */ を標準出力に書き出す。
  • それ以外の場合: [Warning|Error]@[line_no]: [msg] を標準エラー出力に書き出す。

前提条件と不変条件

  • グローバル変数 line_no が現在の行番号を保持していること。
  • グローバル変数 found_err がエラー発生の追跡に使用されること。

アルゴリズムまたは振る舞い

1. level が 0 でない場合、グローバル変数 found_err を 1 に設定する。 2. output ストリームが stdout と等しいかを確認する。 3. stdout の場合は、整形コメント形式（/**INDENT** ... */）でメッセージを整形して出力する。 4. それ以外の場合は、標準エラー出力（stderr）へ直接メッセージを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/tools/pg_bsd_indent/indent.c 内の各所（バッファオーバーフロー、行の断絶、ファイル末尾の欠損、括弧の不整合、#if スタックエラー等の検出時）。
  • **呼び出し先**: fprintf（標準ライブラリ）。

関連データ構造

  • found_err（グローバル変数）
  • line_no（グローバル変数）
  • output（グローバル変数）

失敗モードと境界条件

  • level が 0 か否かで出力されるラベルが切り替わる。
  • output の設定状態によって出力先が標準出力か標準エラー出力に分岐する。

保守担当者向けメモ

  • 本関数は indent.c から頻繁に呼び出されるエラー報告の共通インターフェースである。
  • stdout への出力時に /**INDENT** という特殊な形式を採用しているため、出力先の変更やフォーマットの修正には注意が必要である。
