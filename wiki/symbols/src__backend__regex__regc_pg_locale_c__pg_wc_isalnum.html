pg_wc_isalnum

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_pg_locale.c:352

目的

指定されたワイド文字（pg_wchar）が英数字（alphanumeric）であるかを、現在のロケール戦略に基づいて判定します。

シグネチャまたは形状

int pg_wc_isalnum(pg_wchar c)

入力と出力

  • **入力**: pg_wchar c（判定対象の文字コード）
  • **出力**: int（英数字であれば非ゼロ、そうでなければ 0 を返す）

前提条件と不変条件

  • pg_regex_strategy が適切に初期化されている必要があります。
  • pg_wchar は符号なし型として扱われます。

アルゴリズムまたは振る舞い

グローバル変数 pg_regex_strategy の値に応じて、以下のいずれかの戦略で判定を行います。

1. **PG_REGEX_STRATEGY_C**: pg_char_properties テーブルを参照し、ASCII範囲内（127以下）の英数字判定を行います。 2. **PG_REGEX_STRATEGY_BUILTIN**: PostgreSQL組み込みの pg_u_isalnum を使用してUnicodeベースの判定を行います。 3. **PG_REGEX_STRATEGY_LIBC_WIDE**: iswalnum_l を使用します。wchar_t が4バイト未満の場合は、文字コードが 0xFFFF 以下であるかを確認します。 4. **PG_REGEX_STRATEGY_LIBC_1BYTE**: isalnum_l を使用します。文字コードが UCHAR_MAX 以下である必要があります。 5. **PG_REGEX_STRATEGY_ICU**: ICUライブラリの u_isalnum を使用します（USE_ICU 定義時）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_locale.c: pg_ctype_get_cache 内および文字クラス判定ロジックから呼び出されます。
  • src/include/regex/regcustom.h: iscalnum マクロとして定義されています。
  • **呼び出し先**:
  • 戦略に応じて pg_u_isalnum, iswalnum_l, isalnum_l, u_isalnum を呼び出します。

関連データ構造

  • pg_wchar: 文字コードを表す型。
  • PG_Locale_Strategy: ロケール判定戦略を定義する列挙型。

失敗モードと境界条件

  • PG_REGEX_STRATEGY_LIBC_1BYTE において、c が UCHAR_MAX を超える場合は判定不能（0を返す）となります。
  • PG_REGEX_STRATEGY_ICU は USE_ICU が定義されていない環境では利用できません。
  • どの戦略にも該当しない場合、関数は 0 を返します。

保守担当者向けメモ

  • 本関数は regcomp.c からインクルードされることを前提としています。
  • pg_regex_locale が指すロケール情報が、選択された戦略と整合していることを確認してください。
