r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_danish.c:136

目的

単語内のステミング処理対象領域（R1, R2等）を決定し、SN_env 構造体の内部配列 I に境界位置を格納する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z （ステミング環境の状態を保持する構造体）
  • 出力: 成功時に 1、失敗時に 0 を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • z->I 配列のインデックス 0 および 1 が領域境界の保持に使用される。

アルゴリズムまたは振る舞い

1. z->I[1] を文字列の末尾 z->l で初期化する。 2. 文字列の先頭から3文字進んだ位置を z->I[0] に設定する。 3. out_grouping を使用して、母音（g_v）以外の文字をスキップする。 4. in_grouping を使用して、母音（g_v）の連続する箇所を特定し、その直後の位置を z->I[1] に更新する。 5. z->I[1] が z->I[0] より小さい場合、z->I[1] を z->I[0] に調整する。

呼び出し元と呼び出し先

  • 呼び出し元: danish_ISO_8859_1_stem （本ファイル内のメインエントリーポイント）。
  • 呼び出し先: out_grouping, in_grouping （Snowballランタイムのユーティリティ関数）。
  • 注記: 他の言語実装（Basque, Catalan, Dutch等）でも同名の関数が定義されているが、本ページは stem_ISO_8859_1_danish.c の文脈に限定している。

関連データ構造

  • struct SN_env: ステミング環境の状態（カーソル位置 c、文字列長 l、整数配列 I など）を保持する。

失敗モードと境界条件

  • z->c + 3 が文字列長 z->l を超える場合、処理を中断し 0 を返す。
  • out_grouping または in_grouping が負の値を返した場合、処理を中断し 0 を返す。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接編集せず元のSnowballスクリプト（.sbl）を修正すること。
  • g_v は母音グループを定義する定数であり、言語ごとに異なる可能性があるため注意が必要。
