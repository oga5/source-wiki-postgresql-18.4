shm_mq_get_sender

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shm_mq.c:256

目的

共有メモリメッセージキュー（shm_mq）に現在登録されている送信側プロセス（PGPROC）のポインタを取得します。

シグネチャまたは形状

PGPROC *shm_mq_get_sender(shm_mq *mq)

入力と出力

  • **入力**: mq (共有メモリメッセージキュー構造体へのポインタ)
  • **出力**: 送信側として登録されている PGPROC 構造体へのポインタ。送信側が未設定の場合は NULL を返します。

前提条件と不変条件

  • mq は有効な共有メモリ領域を指している必要があります。
  • 内部状態の整合性を保つため、mq->mq_mutex スピンロックを使用してアクセスを保護します。

アルゴリズムまたは振る舞い

1. mq->mq_mutex スピンロックを獲得します。 2. mq->mq_sender フィールドから送信側プロセスのポインタを読み取ります。 3. mq->mq_mutex スピンロックを解放します。 4. 読み取ったポインタを呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c: 並列クエリ実行時の状態確認に使用。
  • src/backend/storage/ipc/shm_mq.c: キューの内部状態チェックやアサーションに使用。
  • **呼び出し先**:
  • SpinLockAcquire
  • SpinLockRelease

関連データ構造

  • shm_mq: メッセージキュー本体。
  • PGPROC: PostgreSQL のプロセス制御ブロック。

失敗モードと境界条件

  • mq が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • 送信側が設定されていない場合、戻り値は NULL となります。これはエラーではなく正常な状態として扱われます。

保守担当者向けメモ

  • 本関数はスピンロックを使用してアトミックな読み取りを保証しています。呼び出し元でロックを保持する必要はありません。
