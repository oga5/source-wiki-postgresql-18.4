dsm_segment_detach_callback

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/dsm.c:58

目的

動的共有メモリ（DSM）セグメントがデタッチされる際に実行されるコールバック関数とその引数を保持するための構造体です。

シグネチャまたは形状

typedef struct dsm_segment_detach_callback
{
    on_dsm_detach_callback function;
    Datum        arg;
    slist_node   node;
} dsm_segment_detach_callback;

入力と出力

  • function: デタッチ時に呼び出される on_dsm_detach_callback 型の関数ポインタ。
  • arg: コールバック関数に渡される Datum 型の引数。
  • node: dsm_segment 内のリスト管理用ノード。

前提条件と不変条件

  • 本構造体は dsm_segment に関連付けられたリストの一部として管理されます。
  • slist_node を介して単方向リストに連結されます。

アルゴリズムまたは振る舞い

  • セグメントのデタッチ処理中に、この構造体で登録されたコールバック関数が順次実行されます。
  • slist_container マクロを使用して、リストノードから本構造体インスタンスを特定し、格納された関数を呼び出します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/dsm.c 内のセグメントデタッチ処理（dsm_detach 等）において、登録されたコールバックを反復処理する際に参照されます。
  • 呼び出し先: 本構造体が保持する function フィールドに設定された関数が実行されます。

関連データ構造

  • dsm_segment: 本構造体をリストとして保持する親構造体。
  • slist_node: lib/ilist.h で定義される単方向リストノード。

失敗モードと境界条件

  • コールバック関数の登録時にメモリ不足が発生した場合、本構造体の確保に失敗する可能性があります。
  • 実行時のコールバック関数内でのエラー処理については、dsm_detach がエラーを投げない設計であるため、コールバック側でも適切にハンドリングされる必要があります。

保守担当者向けメモ

  • 本構造体は dsm.c 内部でのみ使用されるプライベートな構造体です。
  • リストの反復処理には slist_container を使用するため、node フィールドのオフセット変更には注意が必要です。
