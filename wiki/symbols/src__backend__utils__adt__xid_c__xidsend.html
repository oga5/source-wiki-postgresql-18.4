xidsend

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/xid.c:66

目的

TransactionId（xid）をPostgreSQLのバイナリ転送形式（bytea）に変換する。

シグネチャまたは形状

Datum xidsend(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TRANSACTIONID(0) を介して TransactionId 型の引数を受け取る。
  • **出力**: bytea 型の Datum を返す。

前提条件と不変条件

  • libpq/pqformat.h で定義された StringInfo インターフェースを介してバイナリデータを構築する。
  • ネットワークバイトオーダー（ビッグエンディアン）での送信を前提とする。

アルゴリズムまたは振る舞い

1. pq_begintypsend を呼び出し、バイナリ送信用の StringInfoData バッファを初期化する。 2. pq_sendint32 を使用して、引数の TransactionId を4バイトの整数としてバッファに書き込む。 3. pq_endtypsend を呼び出してバッファを bytea 形式に変換し、その結果を PG_RETURN_BYTEA_P で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 不明（PostgreSQLの型システムを介して呼び出される）。
  • **呼び出し先**:
  • pq_begintypsend
  • pq_sendint32
  • pq_endtypsend

関連データ構造

  • TransactionId: トランザクション識別子。
  • StringInfoData: バイナリデータ構築用バッファ。

失敗モードと境界条件

  • メモリ不足時、pq_begintypsend 等の内部関数がエラーを発生させる可能性がある。
  • TransactionId は32ビット値として処理されるため、値の範囲は uint32 に準拠する。

保守担当者向けメモ

  • 本関数は xid 型のバイナリ送信関数である。xid8（64bitトランザクションID）を扱う場合は、別の関数（xid8send 等）を使用する必要がある。
  • ネットワークバイトオーダーの変換は pq_sendint32 が内部で処理するため、呼び出し側でのエンディアン変換は不要である。
