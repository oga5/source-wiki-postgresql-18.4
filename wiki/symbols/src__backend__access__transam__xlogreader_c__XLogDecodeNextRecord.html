XLogDecodeNextRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogreader.c:528

目的

WAL（Write-Ahead Log）のストリームから次のレコードを読み取り、デコード可能な状態まで準備する。ページ境界をまたぐレコードの再構築や、ヘッダーの検証を行う。

シグネチャまたは形状

static XLogPageReadResult XLogDecodeNextRecord(XLogReaderState *state, bool nonblocking)

入力と出力

  • **入力**:
  • state: WAL読み取り状態を保持する XLogReaderState 構造体。
  • nonblocking: ブロックせずに読み込みを行うかを示すフラグ。
  • **出力**:
  • XLogPageReadResult: 処理結果（成功、ブロック発生、エラーなど）。

前提条件と不変条件

  • state->NextRecPtr が読み取り開始位置を指していること。
  • ReadPageInternal がページヘッダーの整合性を検証済みであること。
  • レコードのヘッダーがページをまたぐ場合、total_len が SizeOfXLogRecord 以上であること。

アルゴリズムまたは振る舞い

1. **初期化**: エラーバッファをリセットし、読み取り位置（RecPtr）を特定する。 2. **ページ読み込み**: ReadPageInternal を呼び出し、レコードヘッダーを含むページを state->readBuf に取得する。 3. **ヘッダー検証**: ページヘッダーをスキップし、レコードヘッダーが現在のページ内に収まっている場合は ValidXLogRecordHeader で検証を行う。 4. **メモリ確保**: XLogReadRecordAlloc を使用して、レコード全体を格納するためのデコード用バッファを確保する。 5. **再構築**: レコードがページ境界をまたぐ場合、後続のページからデータを連結する準備（assembled = true）を行う。 6. **エラー処理**: 検証失敗時や無効なオフセット検出時は err ラベルへ遷移し、エラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: XLogReadRecord（src/backend/access/transam/xlogreader.c:984）から呼び出される。
  • **呼び出し先**: ReadPageInternal、XLogPageHeaderSize、ValidXLogRecordHeader、XLogReadRecordAlloc、report_invalid_record。

関連データ構造

  • XLogReaderState: リーダーの現在の状態、バッファ、読み取り位置を保持。
  • XLogRecord: WALレコードの物理ヘッダー構造。
  • DecodedXLogRecord: 解析後のレコード情報を格納する構造体。

失敗モードと境界条件

  • **XLREAD_WOULDBLOCK**: nonblocking が真で、かつバッファ不足やI/O待ちが発生した場合。
  • **無効なオフセット**: targetRecOff がページヘッダーサイズより小さい場合。
  • **ヘッダー検証失敗**: ValidXLogRecordHeader が偽を返した場合。
  • **長さ異常**: total_len が SizeOfXLogRecord 未満の場合。

保守担当者向けメモ

  • 本関数は XLogReaderState の内部状態を直接操作するため、呼び出し順序に注意すること。
  • nonblocking モード時は、XLogReadRecordAlloc が NULL を返した場合に即座に XLREAD_WOULDBLOCK を返す必要がある。
  • ページをまたぐレコードの再構築処理は、この関数の後半部分で継続される。
