CHKVAL

定義

  • 種別: struct
  • 定義箇所: contrib/intarray/_int_bool.c:223

目的

クエリ評価エンジンにおいて、検索対象となる整数配列の範囲（開始位置と終了位置）を保持し、条件判定関数へ渡すためのコンテナとして機能します。

シグネチャまたは形状

typedef struct
{
	int32	   *arrb;
	int32	   *arre;
} CHKVAL;

入力と出力

  • **入力**: 構造体初期化時に、対象となる整数配列の先頭ポインタ（arrb）と末尾ポインタ（arre）が設定されます。
  • **出力**: checkcondition_arr 等の関数において、この構造体を介して配列の範囲が参照されます。

前提条件と不変条件

  • arrb は配列の開始位置、arre は配列の終了位置を指す必要があります。
  • checkcondition_arr で使用される場合、対象の整数配列はソート済みである必要があります（ソートされていない場合、二分探索による判定が失敗します）。

アルゴリズムまたは振る舞い

execute 関数等のクエリ評価プロセスにおいて、評価対象のデータセットを抽象化するために使用されます。void * 型の引数として渡された後、内部で CHKVAL * にキャストされ、arrb および arre を通じて配列の要素にアクセスします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/intarray/_int_bool.c 内の checkcondition_arr 等の関数で、クエリ評価のコンテキストとしてインスタンス化されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。
  • **注記**: contrib/ltree/ltxtquery_op.c や src/backend/utils/adt/tsgistidx.c にも同名の構造体定義が存在しますが、本ページは contrib/intarray/_int_bool.c の定義を対象としています。

関連データ構造

  • ITEM: クエリツリーの各ノードを構成する構造体。
  • QUERYTYPE: クエリ全体を保持する構造体。

失敗モードと境界条件

  • arrb または arre が不正なメモリ領域を指している場合、評価時にセグメンテーション違反が発生する可能性があります。
  • 配列が空の場合、arrb と arre の関係性が適切に管理されていないと、二分探索で境界外参照が発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は intarray モジュール以外（ltree や tsgistidx）でも類似の目的で定義されています。変更を加える際は、他モジュールとの名前衝突や定義の差異に注意してください。
  • checkcondition_arr がこの構造体を利用する際、配列のソート状態が保証されていることを前提としている点に留意してください。
