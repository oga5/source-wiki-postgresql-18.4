HashPageStat

定義

  • 種別: struct
  • 定義箇所: contrib/pageinspect/hashfuncs.c:38

目的

PostgreSQL のハッシュインデックスページから抽出した統計情報およびページヘッダーの不透明（opaque）データを保持するために使用されます。

シグネチャまたは形状

typedef struct HashPageStat
{
    int         live_items;
    int         dead_items;
    int         page_size;
    int         free_size;
    BlockNumber hasho_prevblkno;
    BlockNumber hasho_nextblkno;
    Bucket      hasho_bucket;
    uint16      hasho_flag;
    uint16      hasho_page_id;
} HashPageStat;

入力と出力

本構造体は関数への引数として渡され、解析結果を格納するコンテナとして機能します。

  • **入力**: GetHashPageStatistics 関数等において、初期化された本構造体のポインタが渡されます。
  • **出力**: ページ内のタプル数、空き容量、およびページヘッダーのメタデータが各フィールドに書き込まれます。

前提条件と不変条件

  • verify_hash_page によるページ検証が完了していることが前提です。
  • ページ内のアイテム走査結果に基づき、live_items と dead_items が算出されます。

アルゴリズムまたは振る舞い

  • ページ内の各アイテムを走査し、ItemIdIsDead を用いてタプルの生存状態を判定して live_items および dead_items をカウントします。
  • ページヘッダーの HashPageOpaqueData から hasho_prevblkno、hasho_nextblkno、hasho_bucket、hasho_flag、hasho_page_id をコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • GetHashPageStatistics (contrib/pageinspect/hashfuncs.c:156)
  • hash_page_stats (contrib/pageinspect/hashfuncs.c:244)
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • HashPageOpaqueData: ハッシュインデックスページの末尾に配置される構造体。本構造体の hasho_ で始まるフィールドの直接的なソースとなります。

失敗モードと境界条件

  • ページが破損している場合、または verify_hash_page を通過しない不正なページデータが渡された場合、統計情報の算出結果は信頼できません。

保守担当者向けメモ

  • 本構造体は pageinspect モジュール専用のデータ保持用であり、PostgreSQL コアのインデックスアクセス手法（AM）の内部構造体とは異なります。
  • フィールドを追加する場合は、GetHashPageStatistics 内での値の割り当て処理を更新してください。
