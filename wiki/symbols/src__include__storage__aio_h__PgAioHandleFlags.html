PgAioHandleFlags

定義

  • 種別: enum
  • 定義箇所: src/include/storage/aio.h:48

目的

非同期 I/O (AIO) 操作の挙動を制御するためのビットフラグを定義する。

シグネチャまたは形状

typedef enum PgAioHandleFlags
{
    PGAIO_HF_SYNCHRONOUS = 1 << 0,
    PGAIO_HF_REFERENCES_LOCAL = 1 << 1,
    PGAIO_HF_BUFFERED = 1 << 2,
} PgAioHandleFlags;

入力と出力

  • 入力: pgaio_io_set_flag(PgAioHandle *ioh, PgAioHandleFlags flag) 関数を通じて、PgAioHandle 構造体に設定される。
  • 出力: 各フラグはビットフィールドとして保持され、I/O メソッドの実行戦略やヒューリスティックの制御に使用される。

前提条件と不変条件

  • PGAIO_HF_REFERENCES_LOCAL: プロセスローカルメモリを参照する I/O には必須。一部の I/O メソッドはローカルメモリを直接扱えないため、このフラグが設定されている場合は同期実行へフォールバックする必要がある。

アルゴリズムまたは振る舞い

  • PGAIO_HF_SYNCHRONOUS: 同期実行のヒント。AIO インターフェース経由での同期 I/O 実行を最適化し、コードの重複を避けるために使用される。
  • PGAIO_HF_BUFFERED: バッファード I/O を使用することを示す。一部の I/O メソッドにおけるヒューリスティック制御に利用される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/aio/aio.c 内の pgaio_io_set_flag がこのフラグを引数として受け取る。
  • 呼び出し先: 内部的なビットフィールドとして保持されるため、特定の関数呼び出し先はソースから特定できない。

関連データ構造

  • PgAioHandle: このフラグが適用される主要な構造体。
  • src/include/storage/aio_internal.h: ビットフィールドとしての利用箇所が定義されている。

失敗モードと境界条件

  • PGAIO_HF_REFERENCES_LOCAL が設定されているにもかかわらず、非同期実行を強制する I/O メソッドが選択された場合、正当性が損なわれる可能性がある。

保守担当者向けメモ

  • 本フラグは pgaio_io_set_flag を介して設定される。
  • PGAIO_HF_SYNCHRONOUS および PGAIO_HF_BUFFERED は最適化のためのヒントであり、設定しなくても正当性には影響しない。
