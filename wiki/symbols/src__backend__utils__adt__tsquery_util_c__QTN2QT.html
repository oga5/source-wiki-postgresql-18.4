QTN2QT

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_util.c:362

目的

QTNode ツリー構造を、PostgreSQL の全文検索用シリアライズ形式である TSQuery（フラットな配列形式）へ変換します。

シグネチャまたは形状

TSQuery QTN2QT(QTNode *in)

入力と出力

  • **入力**: QTNode *in（変換対象の QTNode ツリーのルートノード）
  • **出力**: TSQuery（シリアライズされた tsquery 構造体へのポインタ）

前提条件と不変条件

  • 入力された QTNode ツリーは有効な構造である必要があります。
  • 変換後のサイズが TSQUERY_TOO_BIG マクロの制限を超える場合、エラーが発生します。
  • palloc0 を使用してメモリを確保するため、呼び出し元はメモリコンテキストの管理に注意する必要があります。

アルゴリズムまたは振る舞い

1. cntsize を呼び出し、ツリー全体を走査して必要なノード数（nnode）とオペランドの合計長（sumlen）を算出します。 2. COMPUTESIZE マクロを用いて、シリアライズ後の TSQuery に必要なメモリサイズを計算します。 3. palloc0 でメモリを確保し、SET_VARSIZE でサイズを設定します。 4. QTN2QTState 構造体を初期化し、GETQUERY および GETOPERAND を使用して、シリアライズ後のデータ領域へのポインタをセットアップします。 5. fillQT を呼び出し、ツリー構造をフラットな配列形式へ再帰的に書き込みます。 6. 構築された TSQuery を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery_op.c
  • src/backend/utils/adt/tsquery_rewrite.c
  • **呼び出し先**:
  • cntsize
  • fillQT
  • palloc0
  • ereport

関連データ構造

  • QTNode: 入力となるツリー構造体。
  • TSQuery: 出力となるシリアライズされた構造体。
  • QTN2QTState: 変換処理中の状態（現在のアイテム位置およびオペランド位置）を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: palloc0 が失敗した場合、PostgreSQL のエラーハンドリングによりアボートします。
  • **サイズ超過**: TSQUERY_TOO_BIG に該当する場合、ERRCODE_PROGRAM_LIMIT_EXCEEDED でエラーを報告します。

保守担当者向けメモ

  • 本関数は再帰的な処理を行うため、深いツリー構造を扱う際はスタックオーバーフローのリスクがあります。
  • cntsize および fillQT と密接に結合しているため、TSQuery の内部フォーマットに変更を加える際は、これら3つの関数を同時に更新する必要があります。
