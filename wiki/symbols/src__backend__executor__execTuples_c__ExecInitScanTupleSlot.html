ExecInitScanTupleSlot

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execTuples.c:1999

目的

スキャン操作を行うプランノード（ScanState）に対して、タプルを保持するための TupleTableSlot を初期化し、関連するメタデータを設定する。

シグネチャまたは形状

void ExecInitScanTupleSlot(EState *estate, ScanState *scanstate, TupleDesc tupledesc, const TupleTableSlotOps *tts_ops)

入力と出力

  • **入力**:
  • estate: 実行状態を管理する EState 構造体。
  • scanstate: 初期化対象の ScanState 構造体。
  • tupledesc: スロットが扱うタプルの記述子（TupleDesc）。
  • tts_ops: スロットの振る舞いを定義する TupleTableSlotOps 構造体。
  • **出力**: なし（scanstate のメンバを直接更新する）。

前提条件と不変条件

  • estate および scanstate は有効なポインタであること。
  • tts_ops は、使用するタプル形式に対応した適切な関数テーブルであること。

アルゴリズムまたは振る舞い

1. ExecAllocTableSlot を呼び出し、estate->es_tupleTable に新しいスロットを割り当てて scanstate->ss_ScanTupleSlot に格納する。 2. scanstate->ps.scandesc に tupledesc を設定する。 3. scanstate->ps.scanopsfixed を tupledesc != NULL に基づいて設定する。 4. scanstate->ps.scanops に tts_ops を設定する。 5. scanstate->ps.scanopsset を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ExecInitSeqScan
  • ExecInitBitmapHeapScan
  • ExecInitCteScan
  • ExecInitCustomScan
  • ExecInitForeignScan
  • ExecInitFunctionScan
  • ExecInitIndexOnlyScan
  • その他、各スキャンノードの初期化関数から呼び出される。
  • **呼び出し先**:
  • ExecAllocTableSlot（src/backend/executor/execTuples.c 内）

関連データ構造

  • ScanState: スキャンノードの実行状態を保持する構造体。
  • TupleTableSlot: タプルを保持する汎用コンテナ。
  • TupleTableSlotOps: スロットの操作関数テーブル。
  • TupleDesc: タプルの属性定義。

失敗モードと境界条件

  • tupledesc が NULL の場合、scanopsfixed は false となる。
  • メモリ不足時には ExecAllocTableSlot 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、スキャンノードの初期化プロセスにおける標準的なスロット設定手順をカプセル化している。
  • scanstate->ps（PlanState）のフィールドを直接操作するため、PlanState の構造変更時には本関数の更新が必要となる。
