get_tables_to_cluster

定義

  • 種別: function
  • 定義箇所: src/backend/commands/cluster.c:1642

目的

システムカタログ pg_index をスキャンし、indisclustered（クラスタリング対象としてマーク）が真であり、かつ現在のユーザーが操作権限を持つテーブルとインデックスのリストを収集します。

シグネチャまたは形状

List *get_tables_to_cluster(MemoryContext cluster_context)

入力と出力

  • **入力**: cluster_context（結果リストを保持するためのメモリコンテキスト）
  • **出力**: RelToCluster 構造体のリスト（List *）。各要素には対象テーブルの OID とインデックスの OID が格納される。

前提条件と不変条件

  • IndexRelationId（pg_index）に対する AccessShareLock を取得してスキャンを行う。
  • リストの生成には、呼び出し元から渡された cluster_context を使用する。
  • 権限チェック関数 cluster_is_permitted_for_relation を通過したリレーションのみがリストに含まれる。

アルゴリズムまたは振る舞い

1. pg_index を indisclustered 列でスキャンする。 2. 各エントリについて、cluster_is_permitted_for_relation を呼び出し、現在のユーザーが対象テーブルをクラスタリングする権限を持つか検証する。 3. 権限がある場合、cluster_context にメモリコンテキストを切り替え、RelToCluster 構造体を割り当ててリストに追加する。 4. スキャン終了後、AccessShareLock を解放してリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/cluster.c 内の cluster() 関数（全テーブル対象のクラスタリング処理時）。
  • **呼び出し先**:
  • table_open, table_beginscan_catalog, heap_getnext, table_endscan, relation_close（テーブルアクセス）
  • ScanKeyInit（スキャンキー初期化）
  • cluster_is_permitted_for_relation（権限チェック）
  • MemoryContextSwitchTo, palloc, lappend（メモリ管理とリスト操作）

関連データ構造

  • RelToCluster: テーブル OID とインデックス OID を保持する構造体。
  • pg_index: スキャン対象のシステムカタログ。

失敗モードと境界条件

  • クラスタリング対象のインデックスが存在しない場合、空のリストを返す。
  • 権限不足のテーブルはリストから除外されるが、エラーにはならない。
  • table_open 等の低レベル関数が失敗した場合、PostgreSQL の標準的なエラーハンドリングによりトランザクションがアボートされる可能性がある。

保守担当者向けメモ

  • 本関数は CLUSTER コマンドの全テーブル対象モードでのみ使用される。
  • cluster_context のメモリ管理は呼び出し元に依存しているため、リストの生存期間は呼び出し元のコンテキストに準拠する。
