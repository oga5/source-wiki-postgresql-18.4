CommandTagBehavior

定義

  • 種別: struct
  • 定義箇所: src/backend/tcop/cmdtag.c:20

目的

SQLコマンドの実行結果をクライアントへ通知する際、各コマンドタグ（SELECT, INSERT 等）が持つ属性や振る舞いを保持・管理するために使用されます。

シグネチャまたは形状

typedef struct CommandTagBehavior
{
	const char *name;			/* タグ名 (例: "SELECT") */
	const uint8 namelen;		/* name の長さ */
	const bool	event_trigger_ok;
	const bool	table_rewrite_ok;
	const bool	display_rowcount;	/* 完了文字列に行数を含めるか */
} CommandTagBehavior;

入力と出力

本構造体はデータ保持用であり、関数ではありません。tag_behavior 配列の要素として静的に定義され、BuildQueryCompletionString 等の関数によって読み取られます。

前提条件と不変条件

  • namelen は name の文字列長と一致している必要があります。
  • tag_behavior 配列は cmdtaglist.h を通じて定義され、システム全体で不変のメタデータとして扱われます。

アルゴリズムまたは振る舞い

  • 各コマンドのメタデータを保持し、実行エンジンがクライアントへの完了メッセージを構築する際の判断基準を提供します。
  • display_rowcount が true の場合、BuildQueryCompletionString は処理行数を完了文字列に付与します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/cmdtag.c 内の tag_behavior 配列定義および、この配列を参照する関数群（BuildQueryCompletionString 等）。
  • **呼び出し先**: 本構造体はデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • CommandTag (列挙型): コマンドを識別するための列挙値。
  • tag_behavior: CommandTagBehavior 型の全コマンドタグ属性を保持する静的配列。
  • cmdtaglist.h: 本構造体のインスタンスを生成するためのマスターリスト。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、BuildQueryCompletionString で本構造体を参照する際、バッファサイズ（COMPLETION_TAG_BUFSIZE: 64バイト）を超過しないよう注意が必要です。

保守担当者向けメモ

  • 新しいコマンドを追加する場合は、cmdtaglist.h を編集してください。本構造体の定義を変更する必要はありません。
  • INSERT コマンドの完了タグに 0 が含まれるのは、過去の OID 埋め込み仕様との互換性維持のためです。
