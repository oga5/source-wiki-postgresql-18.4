pg_wc_isprint

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_pg_locale.c:469

目的

指定された文字（pg_wchar）が印刷可能文字であるかどうかを、現在のロケール戦略に基づいて判定します。

シグネチャまたは形状

int pg_wc_isprint(pg_wchar c)

入力と出力

  • **入力**: pg_wchar c（判定対象の文字コード）
  • **出力**: int（印刷可能文字であれば非ゼロ、そうでなければ 0 を返す）

前提条件と不変条件

  • pg_regex_strategy が適切に設定されている必要があります。
  • pg_wchar は符号なし型として扱われます。

アルゴリズムまたは振る舞い

現在の pg_regex_strategy に応じて以下のいずれかの判定ロジックを実行します。

1. **PG_REGEX_STRATEGY_C**: pg_char_properties 配列を参照し、ASCII範囲内（127以下）かつ PG_ISPRINT フラグが立っているかを確認します。 2. **PG_REGEX_STRATEGY_BUILTIN**: pg_u_isprint(c) を呼び出し、組み込みのUnicode判定を行います。 3. **PG_REGEX_STRATEGY_LIBC_WIDE**: wchar_t のサイズが4バイト以上、または文字コードが 0xFFFF 以下の場合、iswprint_l を使用してロケール依存の判定を行います。 4. **PG_REGEX_STRATEGY_LIBC_1BYTE**: isprint_l を使用して、1バイト文字としてロケール依存の判定を行います。 5. **PG_REGEX_STRATEGY_ICU**: u_isprint(c) を呼び出し、ICUライブラリによる判定を行います（USE_ICU 定義時）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_locale.c:597 (pg_ctype_get_cache への引数として使用)
  • src/backend/regex/regc_locale.c:682
  • **呼び出し先**:
  • pg_u_isprint (BUILTIN戦略時)
  • iswprint_l / isprint_l (LIBC戦略時)
  • u_isprint (ICU戦略時)

関連データ構造

  • pg_regex_strategy: 判定戦略を決定する列挙型変数。
  • pg_regex_locale: ロケール情報を含む構造体。
  • pg_char_properties: Cロケール用の文字プロパティテーブル。

失敗モードと境界条件

  • pg_regex_strategy が予期しない値の場合、関数は 0 を返します。
  • LIBC_WIDE 戦略において、wchar_t が小さく、かつ文字コードが 0xFFFF を超える場合の挙動は、フォールスルーにより LIBC_1BYTE 戦略へ移行します。

保守担当者向けメモ

  • 本関数は regcomp.c からインクルードされることを前提としています。
  • ICUサポートはコンパイル時の USE_ICU 定義に依存します。
  • キャッシュ機構（pg_ctype_get_cache）と組み合わせて使用されるため、判定ロジックの変更はキャッシュの整合性に影響を与える可能性があります。
