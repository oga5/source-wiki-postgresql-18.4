tfunc_upper

定義

  • 種別: function
  • 定義箇所: src/common/unicode/case_test.c:316

目的

PostgreSQL の Unicode 大文字変換関数 unicode_strupper をテスト用インターフェースとしてラップし、test_convert 関数から呼び出し可能な形式で提供する。

シグネチャまたは形状

static size_t tfunc_upper(char *dst, size_t dstsize, const char *src, ssize_t srclen);

入力と出力

  • **入力**:
  • dst: 変換結果を格納するバッファへのポインタ。
  • dstsize: dst バッファのサイズ。
  • src: 変換元の文字列。
  • srclen: 変換対象の長さ（バイト単位）。
  • **出力**:
  • 変換後の文字列長（バイト単位）を返す。

前提条件と不変条件

  • unicode_strupper の第5引数に true を渡し、大文字変換を実行する。
  • 内部的に unicode_strupper を呼び出すため、同関数の仕様（NULL終端の扱い等）に依存する。

アルゴリズムまたは振る舞い

1. 引数として受け取った dst, dstsize, src, srclen をそのまま unicode_strupper に渡す。 2. unicode_strupper の第5引数（is_upper フラグ）を true に固定して呼び出す。 3. unicode_strupper が返した変換後の長さをそのまま呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: test_convert 関数（src/common/unicode/case_test.c 内）。
  • **呼び出し先**: unicode_strupper（common/unicode_case.h で定義されていると推測される PostgreSQL 内部関数）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • unicode_strupper の仕様に準拠する。
  • test_convert を通じて、バッファサイズが不足する場合や srclen が不適切な場合の挙動が検証される。

保守担当者向けメモ

  • 本関数はテスト専用のラッパーである。変換ロジック自体を変更する場合は unicode_strupper を修正すること。
  • テストケースを追加する際は、test_convert(tfunc_upper, ...) の形式で case_test.c 内のテストスイートに追加する。
