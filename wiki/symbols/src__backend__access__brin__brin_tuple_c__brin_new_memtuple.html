brin_new_memtuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_tuple.c:481

目的

BRINインデックス操作用のメモリ内タプル構造体（BrinMemTuple）を新規に割り当て、初期化する。

シグネチャまたは形状

BrinMemTuple *brin_new_memtuple(BrinDesc *brdesc)

入力と出力

  • **入力**: brdesc (BRINインデックスの記述子。属性数や格納データ総数の算出に使用)
  • **出力**: 初期化済みの BrinMemTuple 構造体へのポインタ

前提条件と不変条件

  • brdesc は有効な BrinDesc 構造体を指している必要がある。
  • 戻り値の BrinMemTuple は、bt_empty_range が true に設定され、各属性のNULL状態（bt_allnulls, bt_hasnulls）が初期化された状態で返される。
  • bt_context には、CurrentMemoryContext を親とする新しいメモリコンテキストが割り当てられる。

アルゴリズムまたは振る舞い

1. BrinDesc から必要なメモリサイズを計算し、BrinMemTuple 本体および bt_values 配列、NULLビットマップ領域を palloc0 で確保する。 2. bt_values、bt_allnulls、bt_hasnulls の各配列を個別に palloc で確保する。 3. bt_empty_range を true に設定する。 4. AllocSetContextCreate を呼び出し、タプル専用のメモリコンテキストを作成して bt_context に格納する。 5. brin_memtuple_initialize を呼び出し、タプルの初期状態を確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c: brin_build_empty, brin_begin_parallel_scan, brin_getinsertstate 等から呼び出される。
  • **呼び出し先**:
  • palloc0, palloc: メモリ確保。
  • AllocSetContextCreate: メモリコンテキスト作成。
  • brin_memtuple_initialize: タプルの初期化。

関連データ構造

  • BrinMemTuple: メモリ上でBRINタプルを保持する構造体。
  • BrinDesc: BRINインデックスのメタデータ。
  • BrinValues: BrinMemTuple 内で各属性の値を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • brdesc が無効な場合、メモリ確保時に不正なサイズ計算が行われる可能性がある。

保守担当者向けメモ

  • 本関数で作成される BrinMemTuple は、不要になった際に brin_free_tuple を使用して適切にメモリを解放する必要がある。
  • bt_context はタプル固有のメモリ管理に使用されるため、ライフサイクル管理に注意すること。
