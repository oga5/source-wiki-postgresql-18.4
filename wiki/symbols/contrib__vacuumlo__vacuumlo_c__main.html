main

定義

  • 種別: function
  • 定義箇所: contrib/vacuumlo/vacuumlo.c:436

目的

vacuumlo ユーティリティの実行エントリポイントとして、コマンドライン引数の解析、実行パラメータの設定、および指定された各データベースに対するクリーンアップ処理の制御を行う。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • argc: 引数の数。
  • argv: 引数文字列の配列（接続オプション、フラグ、および対象データベース名）。
  • **出力**:
  • 戻り値: 処理に失敗したデータベースの総数（成功時は 0）。

前提条件と不変条件

  • transaction_limit のデフォルト値は 1000。
  • 負の transaction_limit は指定不可。
  • ポート番号は 1 から 65535 の範囲内であること。
  • 少なくとも 1 つのデータベース名が引数として提供される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: pg_logging_init を呼び出し、_param 構造体にデフォルト値を設定する。 2. **引数解析**: getopt_long を使用して、ホスト、ポート、ユーザー名、トランザクション制限、ドライラン設定などを解析・格納する。 3. **ヘルプ・バージョン表示**: --help または --version が指定された場合、情報を出力して終了する。 4. **バリデーション**: データベース名が指定されていない場合はエラーを出力して終了する。 5. **実行**: optind 以降の引数をデータベース名とみなし、各データベースに対して vacuumlo() を呼び出す。 6. **集計**: 各 vacuumlo() の戻り値を累積し、最終的な終了ステータスとして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（実行バイナリのエントリポイント）。
  • **呼び出し先**:
  • usage(): ヘルプ表示用。
  • vacuumlo(): データベースごとの処理実行用。
  • getopt_long(): 引数解析用。
  • pg_fatal / pg_log_error: エラー報告用。
  • **注記**: ツールが提示した contrib/amcheck 関連の呼び出し元情報は、本シンボルとは無関係なコードベース内の文字列一致による誤検知であり、本関数を呼び出すものではない。

関連データ構造

  • struct _param: 接続情報や実行オプションを保持する。
  • struct option: getopt_long 用のコマンドライン引数定義。

失敗モードと境界条件

  • **引数エラー**: 必須のデータベース名がない場合、または無効なオプション・ポート番号が指定された場合はエラーメッセージを出力して終了する。
  • **処理失敗**: vacuumlo() が 0 以外の値を返した場合、その回数が戻り値に加算される。

保守担当者向けメモ

  • 本関数は vacuumlo ユーティリティの制御フローを司る。新しいコマンドラインオプションを追加する場合は、long_options 配列と switch 文の両方を更新する必要がある。
  • transaction_limit の変更は、共有メモリのロックテーブル枯渇を防ぐための重要な調整項目である。
