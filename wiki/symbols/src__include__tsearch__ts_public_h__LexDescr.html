LexDescr

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_public.h:25

目的

全文検索パーサーがサポートするトークン型を記述するための構造体です。各トークン型に対して識別子、別名、および説明を提供します。

シグネチャまたは形状

typedef struct
{
	int			lexid;
	char	   *alias;
	char	   *descr;
} LexDescr;

入力と出力

  • **入力**: パーサーの lextype 関数がこの構造体の配列を生成し、呼び出し元へ返します。
  • **出力**: lexid（整数識別子）、alias（短縮名）、descr（人間が読める説明文）の各フィールドを保持します。

前提条件と不変条件

  • lextype 関数によって動的に割り当てられた配列として扱われることが一般的です。
  • 配列の末尾には、通常、番兵となる要素（lexid が 0 など）が含まれることが期待されます。

アルゴリズムまたは振る舞い

パーサーが定義するトークン型をシステムに通知するために使用されます。OidFunctionCall1 を介してパーサーの lextypeOid を呼び出し、返されたポインタを LexDescr 型にキャストして利用します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • src/backend/commands/tsearchcmds.c
  • src/backend/tsearch/wparser.c
  • src/backend/tsearch/wparser_def.c
  • src/test/modules/test_parser/test_parser.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • TSParserCacheEntry（関連するパーサー定義）

失敗モードと境界条件

  • メモリ割り当て失敗: palloc を使用しているため、メモリ不足時にはシステム全体でエラーが発生します。
  • 配列の終端判定: 配列のサイズが正しく管理されていない場合、後続の処理でバッファオーバーランが発生する可能性があります。

保守担当者向けメモ

  • lexid はパーサー固有の識別子であり、システム全体での一意性は保証されません。
  • alias および descr は palloc されたメモリを指すため、ライフサイクル管理に注意してください。
