StartReplicationCmd

定義

  • **種別**: struct
  • **定義箇所**: src/include/nodes/replnodes.h:91

目的

ストリーミングレプリケーションを開始するためのコマンドパラメータを保持するパースノード構造体です。フロントエンドから送信されたレプリケーション開始要求をバックエンドが解釈するために使用されます。

シグネチャまたは形状

typedef struct StartReplicationCmd
{
    NodeTag     type;
    ReplicationKind kind;
    char       *slotname;
    TimeLineID  timeline;
    XLogRecPtr  startpoint;
    List       *options;
} StartReplicationCmd;

入力と出力

  • **入力**: 構文解析器によって、レプリケーションプロトコルのコマンドから各フィールドがインスタンス化されます。
  • **出力**: 実行エンジンが本構造体を参照し、物理または論理レプリケーションの開始処理を実行します。

前提条件と不変条件

  • NodeTag を先頭に持ち、PostgreSQLのノードシステムに従う必要があります。
  • kind フィールドにより、物理レプリケーションか論理レプリケーションかが識別されます。

アルゴリズムまたは振る舞い

本構造体はデータ保持専用のノードです。構文解析器が受信したコマンドを本構造体に格納し、実行エンジンがこの情報を読み取って walsender プロセス内の適切な関数（物理または論理レプリケーション開始関数）へ引き渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c において、コマンドノードのキャスト先として参照されています。
  • **呼び出し先**: src/backend/replication/walsender.c 内の StartReplication および StartLogicalReplication 関数が、本構造体を引数として受け取り処理を実行します。

関連データ構造

  • ReplicationKind: レプリケーションの種類を定義する列挙型。
  • List: コマンド固有のオプションを保持するために使用。
  • TimeLineID, XLogRecPtr: access/xlogdefs.h で定義されたWAL関連の型。

失敗モードと境界条件

  • 構造体内の各フィールド（特に slotname や options）が適切に初期化されていない場合、実行エンジン側でエラーが発生する可能性があります。

保守担当者向けメモ

  • 新しいレプリケーションオプションを追加する場合は、本構造体の options リストを活用するか、必要に応じてフィールドを追加してください。
  • 構造体を変更した際は、src/backend/nodes/copyfuncs.c および outfuncs.c への対応が必要になる場合があります。
