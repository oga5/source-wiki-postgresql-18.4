WorkerInstrumentation

定義

  • 種別: struct
  • 定義箇所: src/include/executor/instrument.h:96

目的

パラレルクエリ実行時において、各ワーカープロセスが個別に収集した実行統計情報（Instrumentation）を保持・管理するために使用されます。

シグネチャまたは形状

typedef struct WorkerInstrumentation
{
    int         num_workers;    /* 続く構造体の数 */
    Instrumentation instrument[FLEXIBLE_ARRAY_MEMBER];
} WorkerInstrumentation;

入力と出力

  • **入力**: パラレル実行計画のノード数やワーカー数に基づき、必要なメモリサイズを計算して確保されます。
  • **出力**: 各ワーカーの統計情報が instrument 配列内に格納されます。

前提条件と不変条件

  • instrument フィールドは FLEXIBLE_ARRAY_MEMBER として定義されており、構造体自体は可変長です。
  • 構造体の確保時には、offsetof(WorkerInstrumentation, instrument) を用いてオフセットを考慮したメモリ割り当てが必要です。

アルゴリズムまたは振る舞い

  • パラレルクエリ実行時に、各ワーカーが自身の実行統計をこの構造体の対応するインデックスに書き込みます。
  • EXPLAIN ANALYZE 実行時など、メインプロセスが各ワーカーの統計情報を集約または表示するために参照します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c: palloc を用いてメモリを動的に確保する際に参照。
  • src/backend/commands/explain.c: EXPLAIN ANALYZE の出力生成時に参照。
  • **呼び出し先**:
  • Instrumentation 構造体（各ワーカーの統計情報を保持）。

関連データ構造

  • Instrumentation: 各ワーカーの個別の統計情報を保持する構造体。
  • PlanState (src/include/nodes/execnodes.h): worker_instrument フィールドを通じて本構造体を保持します。

失敗モードと境界条件

  • メモリ確保時に num_workers の数と Instrumentation 構造体のサイズを正しく計算しない場合、バッファオーバーランが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は可変長配列を含むため、直接インスタンス化せず、必ず palloc 等で動的にメモリを確保してください。
  • Instrumentation 構造体の定義変更が本構造体のメモリレイアウトに影響を与える点に注意してください。
