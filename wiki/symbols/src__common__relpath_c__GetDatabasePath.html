GetDatabasePath

定義

  • 種別: function
  • 定義箇所: src/common/relpath.c:109

目的

データベースのOIDとテーブルスペースのOIDに基づき、データディレクトリ内におけるデータベース固有のディレクトリパス文字列を生成します。

シグネチャまたは形状

char *GetDatabasePath(Oid dbOid, Oid spcOid)

入力と出力

  • **入力**:
  • dbOid (Oid): 対象データベースの識別子。
  • spcOid (Oid): 対象テーブルスペースの識別子。
  • **出力**:
  • 構築されたパス文字列（char *）。pstrdup または psprintf を使用してメモリが割り当てられるため、呼び出し側で解放する必要があります。

前提条件と不変条件

  • spcOid が GLOBALTABLESPACE_OID の場合、dbOid は 0 である必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

テーブルスペースの種類に応じてパスを構築します。 1. **Global**: spcOid が GLOBALTABLESPACE_OID の場合、"global" を返します。 2. **Default**: spcOid が DEFAULTTABLESPACE_OID の場合、"base/{dbOid}" を返します。 3. **Custom**: 上記以外の場合、PG_TBLSPC_DIR、spcOid、TABLESPACE_VERSION_DIRECTORY、dbOid を組み合わせたパス（pg_tblspc/{spcOid}/{version}/{dbOid}）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dbcommands.c 内の複数の箇所（データベース作成、削除、コピー、リカバリ処理など）から呼び出されています。
  • **呼び出し先**: pstrdup および psprintf を使用してメモリを確保します。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • GLOBALTABLESPACE_OID, DEFAULTTABLESPACE_OID: catalog/pg_tablespace_d.h で定義される定数。

失敗モードと境界条件

  • メモリ不足時には pstrdup/psprintf の内部実装によりエラーが発生します。
  • spcOid が GLOBALTABLESPACE_OID であり、かつ dbOid != 0 の場合はアサーション違反となります。

保守担当者向けメモ

  • 本関数は palloc を使用するため、呼び出し側で必ずメモリ解放を行ってください。
  • パスフォーマットは外部ツールやストレージマネージャが強く依存しているため、変更には細心の注意が必要です。特に TABLESPACE_VERSION_DIRECTORY はメジャーバージョンごとに変化します。
