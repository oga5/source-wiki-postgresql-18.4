AuthToken

定義

  • **種別**: struct
  • **定義箇所**: src/include/libpq/hba.h:88

目的

pg_hba.conf および pg_ident.conf ファイル内の各トークンを保持し、必要に応じて正規表現によるマッチングをサポートするためのデータ構造です。

シグネチャまたは形状

typedef struct AuthToken
{
    char       *string;
    bool        quoted;
    regex_t    *regex;
} AuthToken;

入力と出力

  • **入力**: palloc0 を使用してメモリを確保する際、トークン文字列の長さに応じたメモリ領域が動的に割り当てられます。
  • **出力**: 構造体内の string フィールドは、構造体自身に続くメモリ領域を指すように配置されることがあります（src/backend/libpq/hba.c の実装による）。

前提条件と不変条件

  • regex フィールドが設定されている場合、正規表現のコンパイルおよび実行処理が必要です。
  • メモリ管理は PostgreSQL のメモリコンテキストに依存しており、free_auth_token を通じて適切に解放する必要があります。

アルゴリズムまたは振る舞い

  • 設定ファイルのパース時にトークンを保持します。
  • regcomp_auth_token および regexec_auth_token を介して、トークンに対する正規表現のコンパイルとマッチングを行います。
  • copy_auth_token により、既存のトークン構造体の複製が可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/hba.c 内のパース処理およびトークン管理関数群。
  • **呼び出し先**: regex/regex.h で定義される正規表現ライブラリ関数。

関連データ構造

  • TokenizedAuthLine: 設定ファイルの行をトークン化したリストを保持する構造体。
  • HbaLine: AuthToken を含む認証ルール定義構造体。

失敗モードと境界条件

  • メモリ確保失敗時（palloc0 失敗時）の挙動は PostgreSQL の標準的なエラーハンドリングに従います。
  • 正規表現が不正な場合、regcomp_auth_token でエラーが発生する可能性があります。

保守担当者向けメモ

  • AuthToken を破棄する際は、regex フィールドが指すメモリも適切に解放されるよう free_auth_token を使用してください。
  • string フィールドのメモリ配置が構造体と連続している実装（palloc0(sizeof(AuthToken) + toklen + 1)）があるため、構造体のサイズ変更時には注意が必要です。
