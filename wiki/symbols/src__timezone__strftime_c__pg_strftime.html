pg_strftime

定義

  • 種別: function
  • 定義箇所: src/timezone/strftime.c:134

目的

struct pg_tm 形式の時刻データを、指定されたフォーマット文字列に従って文字列へ変換し、呼び出し元が提供するバッファへ安全に書き込みます。

シグネチャまたは形状

size_t pg_strftime(char *s, size_t maxsize, const char *format, const struct pg_tm *t);

入力と出力

  • **入力**:
  • s: 書き込み先のバッファへのポインタ。
  • maxsize: バッファの最大サイズ。
  • format: 時刻フォーマットを指定する文字列。
  • t: 変換対象の時刻データ (struct pg_tm)。
  • **出力**:
  • 成功時: バッファに書き込まれたバイト数（ヌル終端文字を含まない）。
  • 失敗時: 0 を返し、errno を設定する。

前提条件と不変条件

  • C ロケールのみをサポートします。
  • バッファオーバーランを防止するため、常にヌル終端された文字列を返します。
  • 内部処理において ptlim（バッファ終端）を監視し、書き込み制限を厳守します。

アルゴリズムまたは振る舞い

1. 内部関数 _fmt を呼び出し、フォーマット文字列を再帰的に解析してバッファ s へ書き込みます。 2. _fmt が NULL を返した場合、errno を EOVERFLOW に設定し、バッファが空でなければヌル終端して 0 を返します。 3. 書き込み結果がバッファの終端に達した場合、errno を ERANGE に設定し、バッファが空でなければヌル終端して 0 を返します。 4. 正常終了時は、書き込まれた文字列の末尾にヌル文字を付加し、書き込みバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:

関連データ構造

  • struct pg_tm: 時刻情報の保持に使用。
  • enum warn: 変換時の警告状態管理に使用。

失敗モードと境界条件

  • **EOVERFLOW**: フォーマット処理がバッファ容量を超過した場合。
  • **ERANGE**: 書き込み結果がバッファサイズと完全に一致し、ヌル終端のための余地がない場合。
  • maxsize が 0 の場合、書き込みは行われず、失敗時はヌル終端も行われません。

保守担当者向けメモ

  • 本関数はロケール非依存です。
  • %Z 指定子の出力は tm_zone フィールドに依存しており、マルチバイト文字が含まれる可能性があるため、バッファ境界チェックは厳密に行う必要があります。
  • KITCHEN_SINK マクロ定義時にはデバッグ用フォーマット指定子が有効になります。
