BumpAllocLarge

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/bump.c:291

目的

BumpContext において、通常のバンプ割り当てでは収まらない巨大なメモリ要求に対し、専用のメモリブロックを確保して割り当てる。

シグネチャまたは形状

static void *BumpAllocLarge(MemoryContext context, Size size, int flags)

入力と出力

  • **入力**:
  • context: 割り当て対象の BumpContext。
  • size: 要求するメモリサイズ（バイト単位）。
  • flags: 割り当てオプション（MemoryContextCheckSize で検証に使用）。
  • **出力**:
  • 成功時: 確保されたメモリ領域へのポインタ。
  • 失敗時: MemoryContextAllocationFailure を呼び出し、その戻り値を返す。

前提条件と不変条件

  • size は MemoryContextCheckSize により、現在のコンテキストの制限内であることが検証される。
  • 確保されるブロックには Bump_BLOCKHDRSZ（ブロックヘッダー）および Bump_CHUNKHDRSZ（チャンクヘッダー、MEMORY_CONTEXT_CHECKING 有効時）のオーバーヘッドが含まれる。
  • 割り当てられたメモリは、BumpReset または BumpDelete が呼ばれるまで解放されない。

アルゴリズムまたは振る舞い

1. **サイズ計算**: 要求サイズにヘッダー分を加算し、MAXALIGN でアライメントを調整する。 2. **ブロック確保**: malloc を使用して必要なサイズのメモリブロックを確保する。失敗時はエラー処理へ移行する。 3. **メタデータ更新**: コンテキストの mem_allocated を加算し、ブロックの freeptr と endptr をブロック末尾に設定する（このブロックは即座に満杯とみなされる）。 4. **デバッグ情報付与** (MEMORY_CONTEXT_CHECKING 有効時):

  • チャンクヘッダーに MCTX_BUMP_ID を設定。
  • センチネルバイトの設定やメモリのランダム化（RANDOMIZE_ALLOCATED_MEMORY 有効時）を行う。

    • 5. **リスト管理**: 確保したブロックを BumpContext の blocks リストの末尾に追加する。 6. **返却**: チャンクのデータ領域へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/bump.c 内の割り当てロジック（BumpAlloc からの委譲）。
  • **呼び出し先**:
  • MemoryContextCheckSize
  • malloc
  • MemoryContextAllocationFailure
  • dlist_push_tail
  • MemoryChunkSetHdrMaskExternal (デバッグ時)
  • set_sentinel (デバッグ時)
  • randomize_mem (デバッグ時)
  • VALGRIND_MAKE_MEM_NOACCESS (デバッグ時)

関連データ構造

  • BumpContext: コンテキストのメタデータとブロックリストを保持。
  • BumpBlock: 確保されるメモリブロックの構造体。
  • MemoryChunk: デバッグビルド時に使用されるチャンクヘッダー。

失敗モードと境界条件

  • malloc が NULL を返した場合、MemoryContextAllocationFailure を経由してエラーを報告する。
  • size がコンテキストの許容範囲を超えている場合、MemoryContextCheckSize によりエラーが発生する。

保守担当者向けメモ

  • 本関数は pg_noinline であり、インライン化されない。
  • MEMORY_CONTEXT_CHECKING が有効な場合、チャンクヘッダーが付与されるため、メモリレイアウトが通常時と異なる点に注意すること。
  • この関数で確保されたブロックは、通常のバンプ割り当て（BumpAlloc）の空き領域とは独立して管理される。
