oper_select_candidate

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_oper.c:311

目的

演算子の候補リストから、指定された入力型に適合する最適な演算子を特定する。

シグネチャまたは形状

static FuncDetailCode oper_select_candidate(
    int nargs,
    Oid *input_typeids,
    FuncCandidateList candidates,
    Oid *operOid
)

入力と出力

  • **入力**:
  • nargs: 引数の数。
  • input_typeids: 入力型のOid配列。
  • candidates: 演算子の候補リスト。
  • **出力**:
  • operOid: 選択された演算子のOid（失敗時は InvalidOid）。
  • 戻り値: FuncDetailCode（FUNCDETAIL_NORMAL, FUNCDETAIL_NOTFOUND, FUNCDETAIL_MULTIPLE のいずれか）。

前提条件と不変条件

  • candidates は func_match_argtypes によるフィルタリング前のリストである必要がある。
  • operOid は結果を格納するための有効なポインタであること。

アルゴリズムまたは振る舞い

1. func_match_argtypes を呼び出し、入力型と直接または型キャストによって適合可能な候補のみに絞り込む。 2. 候補が0件の場合、FUNCDETAIL_NOTFOUND を返し operOid を InvalidOid に設定する。 3. 候補が1件の場合、その演算子のOidを返し FUNCDETAIL_NORMAL を返す。 4. 候補が複数残った場合、func_select_candidate を呼び出し、曖昧さを解消するヒューリスティックを適用して最適な1つを選択する。 5. 最終的に候補が特定できれば FUNCDETAIL_NORMAL を、特定できなければ FUNCDETAIL_MULTIPLE を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_oper.c 内の演算子解決処理（make_op 等）。
  • **呼び出し先**:
  • func_match_argtypes
  • func_select_candidate

関連データ構造

  • FuncCandidateList: 演算子の候補を保持するリスト構造。
  • FuncDetailCode: 解決結果を示す列挙型。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • 適合する演算子が存在しない場合、FUNCDETAIL_NOTFOUND を返す。
  • 複数の候補が残り、かつヒューリスティックでも解決できない場合、FUNCDETAIL_MULTIPLE を返す。

保守担当者向けメモ

  • 本関数は func_select_candidate と同様のヒューリスティックを使用して曖昧さを解消する。
  • 演算子の適合性判定には型キャストの可否が含まれるため、pg_cast の変更が影響を与える可能性がある。
