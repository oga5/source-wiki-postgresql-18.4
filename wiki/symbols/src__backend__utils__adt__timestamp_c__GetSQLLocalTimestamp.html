GetSQLLocalTimestamp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/timestamp.c:1676

目的

SQLの LOCALTIMESTAMP および LOCALTIMESTAMP(n) 構文を実装し、現在のトランザクション開始時刻をローカルタイムスタンプとして取得します。

シグネチャまたは形状

Timestamp GetSQLLocalTimestamp(int32 typmod)

入力と出力

  • **入力**:
  • typmod (int32): タイムスタンプの精度（小数点以下の桁数）。負の値の場合は精度制限なし。
  • **出力**:
  • Timestamp: トランザクション開始時刻を基にしたローカルタイムスタンプ値。

前提条件と不変条件

  • GetCurrentTransactionStartTimestamp() が有効なトランザクション開始時刻を返す環境で呼び出される必要があります。
  • 返される Timestamp はマイクロ秒単位の int64 です。

アルゴリズムまたは振る舞い

1. GetCurrentTransactionStartTimestamp() を呼び出し、現在のトランザクション開始時刻（TimestampTz）を取得します。 2. timestamptz2timestamp() を使用して、タイムゾーン情報を除いた Timestamp 型へ変換します。 3. typmod が 0 以上の場合、AdjustTimestampForTypmod() を呼び出し、指定された精度に合わせて値を丸めます。 4. 最終的な Timestamp 値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execExprInterp.c: 式評価の過程で呼び出されます。
  • **呼び出し先**:
  • GetCurrentTransactionStartTimestamp()
  • timestamptz2timestamp()
  • AdjustTimestampForTypmod()

関連データ構造

  • Timestamp: int64 型（マイクロ秒単位）。
  • TimestampTz: タイムゾーン付きのタイムスタンプ。

失敗モードと境界条件

  • typmod が負の値の場合、精度調整はスキップされます。
  • AdjustTimestampForTypmod 内部での丸め処理により、値が変更される可能性があります。

保守担当者向けメモ

  • 本関数は LOCALTIMESTAMP 構文のバックエンド実装です。
  • タイムゾーン変換を伴わないローカル時刻の取得であるため、timestamptz2timestamp を介してタイムゾーン情報を除去する設計となっています。
