new_intArrayType

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_tool.c:223

目的

PostgreSQL の intarray モジュールで使用される、指定された要素数を持つ新しい ArrayType 構造体を動的に確保および初期化します。

シグネチャまたは形状

ArrayType *new_intArrayType(int num)

入力と出力

  • **入力**: num (作成する整数配列の要素数)
  • **出力**: 初期化済みの ArrayType 構造体へのポインタ。num <= 0 の場合は空の配列を返します。

前提条件と不変条件

  • num が 0 以下の場合、construct_empty_array(INT4OID) を使用して空の配列を生成します。
  • num > 0 の場合、palloc0 によりメモリが確保され、INT4 型の配列としてメタデータ（次元数、要素型、要素数、下限値）が設定されます。
  • 生成される配列は NULL ビットマップを持たない形式（dataoffset = 0）で初期化されます。

アルゴリズムまたは振る舞い

1. num <= 0 の場合、空の配列を返して終了します。 2. num > 0 の場合、ARR_OVERHEAD_NONULLS(1) と要素分のサイズを加算して必要バイト数を算出します。 3. palloc0 でメモリを確保し、SET_VARSIZE で全体のサイズを記録します。 4. 配列のヘッダ情報（次元数 1、要素型 INT4OID、要素数 num、下限値 1）を設定します。 5. 確保したメモリ領域を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c
  • contrib/intarray/_int_op.c
  • contrib/intarray/_int_tool.c (内部の集合演算関数など)
  • **呼び出し先**:
  • construct_empty_array (PostgreSQL 内部関数)
  • palloc0 (PostgreSQL メモリ管理関数)

関連データ構造

  • ArrayType: PostgreSQL の標準的な配列構造体。
  • INT4OID: 整数型を示す OID。

失敗モードと境界条件

  • num が負の値の場合、Assert(num == 0) によりデバッグビルドでは停止します。
  • メモリ不足時には palloc0 がエラーを発生させます。

保守担当者向けメモ

  • 本関数が返すメモリは palloc されているため、呼び出し側で不要になった際に適切に pfree するか、PostgreSQL のメモリコンテキスト管理に委ねる必要があります。
  • NULL ビットマップをサポートしない設計であるため、NULL を含む配列を扱う用途には適していません。
