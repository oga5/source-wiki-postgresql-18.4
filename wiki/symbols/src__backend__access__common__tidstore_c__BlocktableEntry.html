BlocktableEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/access/common/tidstore.c:44

目的

TidStore において、特定のブロックに含まれるタプルのオフセット情報をメモリ効率よく保持するためのデータ構造です。オフセット数が少ない場合はヘッダー領域に直接格納し、多い場合はビットマップを使用することで領域を最適化します。

シグネチャまたは形状

typedef struct BlocktableEntry
{
    struct {
        /* エンディアンに応じて flags と nwords の順序が入れ替わる */
        uint8 flags;
        int8 nwords;
        OffsetNumber full_offsets[NUM_FULL_OFFSETS];
    } header;
    bitmapword words[FLEXIBLE_ARRAY_MEMBER];
} BlocktableEntry;

入力と出力

  • 入力: TidStoreSetBlockOffsets 等を通じて、ブロック番号に対応するオフセット情報が書き込まれます。
  • 出力: TidStoreIsMember 等を通じて、特定のオフセットが存在するかどうかの判定に使用されます。

前提条件と不変条件

  • header 内の flags は、ラディックツリーがポインタの最下位ビットをタグ付けするために予約されています。
  • words は可変長配列として定義されており、ビットマップとして機能します。
  • 新規エントリ作成時、words までの領域はゼロクリアされる必要があります。

アルゴリズムまたは振る舞い

1. **最適化格納**: 少数のオフセットは full_offsets 配列に直接格納されます。 2. **ビットマップ拡張**: オフセット数が増加した場合、words 配列を使用してビットマップ形式で管理されます。 3. **メモリレイアウト**: header 構造体は、エンディアン（WORDS_BIGENDIAN）に応じて flags と nwords の配置が切り替わります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/common/tidstore.c 内の TidStore 管理関数群（TidStoreSetBlockOffsets など）。
  • 呼び出し先: 直接的な関数呼び出しではなく、lib/radixtree.h のデータ構造の一部として利用されます。
  • 注記: 具体的な呼び出し元関数の詳細なスタックトレースは提供されていません。

関連データ構造

  • TidStore: 本構造体を管理するメインのラディックツリー構造体。
  • OffsetNumber: full_offsets に格納されるオフセット型。
  • bitmapword: words 配列を構成するビットマップの単位。

失敗モードと境界条件

  • full_offsets の容量（NUM_FULL_OFFSETS）を超過した場合、ビットマップ形式への移行が必要です。
  • 共有メモリ上で使用する場合、呼び出し側で適切なロック（TidStoreLockExclusive 等）を保持していないと競合が発生します。

保守担当者向けメモ

  • header 構造体のパディングには注意が必要です。コード内で offsetof(BlocktableEntry, words) を使用してサイズ計算を行う箇所があるため、構造変更時はこのオフセット計算に影響がないか確認してください。
  • words は FLEXIBLE_ARRAY_MEMBER であるため、動的なメモリ確保時には適切なサイズ計算が必須です。
