format_procedure_parts

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/regproc.c:397

目的

指定された OID に対応するプロシージャ（関数）の名前空間、関数名、および引数の型リストを分解して取得します。

シグネチャまたは形状

void format_procedure_parts(Oid procedure_oid, List **objnames, List **objargs, bool missing_ok)

入力と出力

  • **入力**:
  • procedure_oid: 対象となるプロシージャの OID。
  • missing_ok: true の場合、OID が見つからなくてもエラーを発生させず正常終了します。
  • **出力**:
  • objnames: {スキーマ名, 関数名} を含むリストへのポインタ。
  • objargs: 引数の型名（修飾済み）のリストへのポインタ。

前提条件と不変条件

  • pg_proc システムカタログへのアクセスが必要です。
  • objnames および objargs は、呼び出し元で確保された List ** 型の領域である必要があります。

アルゴリズムまたは振る舞い

1. SearchSysCache1 を使用して PROCOID キャッシュから対象の pg_proc タプルを検索します。 2. タプルが見つからない場合、missing_ok が false ならば elog(ERROR) を発生させます。 3. タプルが見つかった場合、以下の処理を行います。

  • get_namespace_name_or_temp を用いてスキーマ名を取得し、関数名と共に objnames に格納します。
  • pronargs（引数の数）に基づきループし、各引数の型を format_type_be_qualified で修飾名に変換して objargs リストに追加します。

    • 4. 最後に ReleaseSysCache でタプルを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c:4906
  • **呼び出し先**:
  • SearchSysCache1
  • get_namespace_name_or_temp
  • format_type_be_qualified
  • ReleaseSysCache
  • list_make2 / lappend

関連データ構造

  • pg_proc: 関数情報を保持するシステムカタログ。
  • List: PostgreSQL のリスト構造体。

失敗モードと境界条件

  • 指定された procedure_oid が pg_proc に存在しない場合、missing_ok が false であればエラーが発生します。

保守担当者向けメモ

  • 本関数は、オブジェクトの識別情報を分解して取得するユーティリティであり、主にカタログオブジェクトの参照解決に使用されます。
  • 引数の型解決には format_type_be_qualified を使用しているため、型名の表示形式は現在の検索パス設定に依存します。
