ecpg_postprocess_result

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/pg_regress_ecpg.c:238

目的

ECPG回帰テストの実行結果ファイルに対して、環境依存の差異を除去するための後処理を行います。

シグネチャまたは形状

static void ecpg_postprocess_result(const char *filename)

入力と出力

  • **入力**: filename (処理対象のファイルパスを示す文字列)
  • **出力**: なし（副作用として、対象ファイルが .stderr 拡張子を持つ場合に正規化された内容で更新されます）

前提条件と不変条件

  • 引数 filename は有効なファイルパスである必要があります。
  • .stderr 拡張子を持つファイルのみが処理対象となります。

アルゴリズムまたは振る舞い

1. ファイル名の末尾が .stderr であるかを確認します。 2. 条件に合致する場合、psprintf を使用して一時ファイル名（.tmp を付与）を生成します。 3. ecpg_filter_stderr を呼び出し、元のファイルの内容をフィルタリングして一時ファイルへ書き出します。 4. 一時ファイル名を保持していたメモリを pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、pg_regress フレームワークのコールバックとして呼び出されることが示唆されています。
  • **呼び出し先**:
  • ecpg_filter_stderr (フィルタリング処理の実行)
  • psprintf (一時ファイル名生成)
  • pfree (メモリ解放)

関連データ構造

  • 特になし。

失敗モードと境界条件

  • ファイル名が7文字未満の場合、拡張子の判定処理はスキップされます。
  • ecpg_filter_stderr の内部実装に依存しますが、ファイル操作権限やディスク容量不足により処理が失敗する可能性があります。

保守担当者向けメモ

  • 本関数は現在 .stderr ファイルの正規化のみをサポートしています。将来的に他の拡張子（例: .stdout）の正規化が必要になった場合は、条件分岐の拡張を検討してください。
  • ecpg_filter_stderr はインプレースに近い処理を行うため、ファイルシステム上の競合やエラーハンドリングには注意が必要です。
