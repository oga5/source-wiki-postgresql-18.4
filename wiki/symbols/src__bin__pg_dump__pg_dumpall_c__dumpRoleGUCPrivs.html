dumpRoleGUCPrivs

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dumpall.c:1291

目的

データベースクラスタ内の設定パラメータ（GUC）に対して定義された非デフォルトのアクセス制御リスト（ACL）を抽出し、SQL形式で出力する。

シグネチャまたは形状

static void dumpRoleGUCPrivs(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（データベース接続ハンドル）
  • **出力**: なし（直接グローバル変数 OPF にSQLコマンドを書き出す）

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • pg_catalog.pg_parameter_acl テーブルが存在し、クエリ可能であること。
  • グローバル変数 OPF が書き込み可能な出力ストリームとして初期化されていること。

アルゴリズムまたは振る舞い

1. pg_catalog.pg_parameter_acl からパラメータ名、所有者、ACL情報、およびデフォルトACLを取得するクエリを実行する。 2. 取得したタプルが1件以上存在する場合、出力ストリームにヘッダーコメントを書き出す。 3. 各タプルに対して以下の処理を繰り返す:

  • パラメータ名を fmtId でクォートし、buildACLCommands を呼び出してACL設定用のSQLコマンドを生成する。
  • 生成されたSQLコマンドを OPF に書き出す。
  • 使用したメモリ（PQExpBuffer 等）を解放する。

    • 4. 最後にクエリ結果を解放し、改行を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dumpall.c 内のメイン処理フロー（main 関数から間接的に呼び出される）。
  • **呼び出し先**: executeQuery, PQntuples, PQgetvalue, createPQExpBuffer, pg_strdup, fmtId, buildACLCommands, pg_log_error, PQfinish, exit_nicely, destroyPQExpBuffer, PQclear。

関連データ構造

  • PGconn: データベース接続状態を保持。
  • PGresult: クエリ結果セットを保持。
  • PQExpBuffer: SQLコマンド生成用のバッファ。

失敗モードと境界条件

  • buildACLCommands が失敗した場合、エラーログを出力し、接続を閉じて exit_nicely(1) によりプログラムを終了する。
  • クエリ結果が0件の場合、何も出力せず正常終了する。

保守担当者向けメモ

  • 本関数は pg_dumpall におけるクラスタレベルの権限ダンプの一部である。
  • buildACLCommands への引数として server_version が使用されているため、接続先サーバーのバージョンに依存したSQL生成が行われる点に注意すること。
