ResourceOwnerDelete

定義

  • 種別: function
  • 定義箇所: src/backend/utils/resowner/resowner.c:870

目的

指定された ResourceOwner オブジェクトをメモリから解放し、リソースオーナー階層から切り離す。

シグネチャまたは形状

void ResourceOwnerDelete(ResourceOwner owner)

入力と出力

  • **入力**: owner (解放対象の ResourceOwner 構造体へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • owner は CurrentResourceOwner と同一であってはならない。
  • owner が保持するリソース（配列、ハッシュテーブル、ロック）はすべて解放済みであること（narr == 0, nhash == 0, nlocks が 0 または MAX_RESOWNER_LOCKS + 1 であること）。

アルゴリズムまたは振る舞い

1. **子オーナーの再帰的削除**: owner->firstchild が存在する場合、再帰的に ResourceOwnerDelete を呼び出し、すべての子を削除する。 2. **親からの切り離し**: ResourceOwnerNewParent(owner, NULL) を呼び出し、親リソースオーナーとのリンクを解除する。 3. **メモリ解放**: 内部のハッシュテーブル (owner->hash) が存在する場合は pfree し、最後に owner 自体を pfree する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c (トランザクション終了時)
  • src/backend/utils/mmgr/portalmem.c (ポータル破棄時)
  • src/pl/plpgsql/src/pl_handler.c (PL/pgSQL プロシージャ終了時)
  • **呼び出し先**:
  • ResourceOwnerDelete (再帰呼び出し)
  • ResourceOwnerNewParent
  • pfree

関連データ構造

  • ResourceOwnerData (引数 owner の型)

失敗モードと境界条件

  • **アサーション失敗**: 前提条件（CurrentResourceOwner との不一致、リソースの未解放）が満たされない場合、プログラムは異常終了する。
  • **メモリ管理**: owner が適切に解放されていない状態で本関数を呼ぶと、リソースリークや不正なメモリ参照を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数はリソースの解放処理そのものではなく、リソースが空になった後のオブジェクト破棄を担う。リソースの解放が必要な場合は、事前に ResourceOwnerRelease を実行すること。
  • 階層構造の削除は再帰的に行われるため、深い階層を持つオーナーを削除する際はスタックオーバーフローに注意が必要である。
