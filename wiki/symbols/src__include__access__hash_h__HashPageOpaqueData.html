HashPageOpaqueData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/hash.h:77

目的

PostgreSQL ハッシュインデックスの各物理ページ末尾に配置されるメタデータ構造体です。ページの種類（メタ、バケット、オーバーフロー等）を識別し、ハッシュインデックスの構造維持に必要なリンク情報を提供します。

シグネチャまたは形状

typedef struct HashPageOpaqueData
{
    BlockNumber hasho_prevblkno;    /* 前方ページへのリンク */
    BlockNumber hasho_nextblkno;    /* 後方ページへのリンク */
    Bucket      hasho_bucket;       /* 所属するバケット番号 */
    uint16      hasho_flag;         /* ページタイプおよびフラグビット */
    uint16      hasho_page_id;      /* ハッシュインデックス識別用ID */
} HashPageOpaqueData;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入出力はありません。ただし、PageInit 等のページ初期化処理において、sizeof(HashPageOpaqueData) が特殊領域（Special Space）のサイズとして渡されます。

前提条件と不変条件

  • 各ハッシュインデックスページは、末尾に本構造体を保持する必要があります。
  • ページサイズは MAXALIGN(sizeof(HashPageOpaqueData)) でアライメントされている必要があります。
  • hasho_page_id はハッシュインデックスの識別に使用されます。

アルゴリズムまたは振る舞い

  • **ページ識別**: hasho_flag を用いて、そのページがメタページ、バケットページ、あるいはオーバーフローページであるかを判別します。
  • **リンク管理**: hasho_prevblkno および hasho_nextblkno を使用して、オーバーフローチェーンを辿るための物理的なリンクを維持します。
  • **バケット所属**: hasho_bucket により、特定のバケットに属するページ群を追跡します。

関連データ構造

  • HashMetaPageData: インデックス全体のメタ情報を保持するルートページ構造体。
  • HashScanOpaqueData: インデックススキャン中の状態管理に使用される構造体。
  • HashPageOpaque: HashPageOpaqueData へのポインタ型。

失敗モードと境界条件

  • PageGetSpecialSize によるチェックで、期待されるサイズと一致しない場合、ページ破損とみなされる可能性があります。
  • 物理ブロック番号の範囲外アクセスや、リンクの不整合が発生した場合、インデックスの走査が失敗します。

保守担当者向けメモ

  • 本構造体のサイズを変更する場合、既存のインデックスファイルとの互換性に注意してください。
  • hasho_flag のビット定義は src/include/access/hash.h 内の LH_PAGE_TYPE 等を参照してください。
  • ページ初期化時には PageInit を使用し、必ず sizeof(HashPageOpaqueData) を指定してください。
