WALDumpReadPage

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/pg_waldump.c:388

目的

XLogReader が WAL データを読み込む際に使用するコールバック関数であり、指定されたオフセットから WAL ページを読み取り、バッファへ格納します。

シグネチャまたは形状

static int WALDumpReadPage(XLogReaderState *state, XLogRecPtr targetPagePtr, int reqLen, XLogRecPtr targetPtr, char *readBuff)

入力と出力

  • **入力**:
  • state: XLogReaderState 構造体へのポインタ。
  • targetPagePtr: 読み込みを開始する WAL 上の LSN。
  • reqLen: 要求された読み込みバイト数。
  • targetPtr: 現在処理中のレコードの LSN。
  • readBuff: 読み込んだデータを格納するバッファ。
  • **出力**:
  • 成功時: 実際に読み込んだバイト数を返す。
  • 失敗時: pg_fatal を呼び出し、プロセスを終了させる。

前提条件と不変条件

  • state->private_data は XLogDumpPrivate 構造体としてキャスト可能である必要があります。
  • private->endptr が設定されている場合、読み込み範囲は endptr を超えないよう制限されます。

アルゴリズムまたは振る舞い

1. XLogDumpPrivate から現在の読み込み終了位置（endptr）を取得します。 2. 読み込み範囲が endptr を超える場合、endptr_reached フラグを立てて -1 を返します。 3. WALRead を呼び出し、指定されたタイムラインとオフセットからデータを読み込みます。 4. WALRead が失敗した場合、WALReadError 構造体からエラー情報を取得し、ファイル名とオフセットを含むエラーメッセージを pg_fatal で出力して終了します。 5. 成功した場合は、読み込んだバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: XLogReader インターフェース（XL_ROUTINE 構造体の .page_read フィールド経由で間接的に呼び出されます）。
  • **呼び出し先**: WALRead（データ読み込みの実行）、XLogFileName（エラー時のファイル名生成）、pg_fatal（致命的エラーの報告）。

関連データ構造

  • XLogReaderState: WAL 読み込みの状態管理構造体。
  • XLogDumpPrivate: pg_waldump 固有の読み込み状態（タイムラインや終了位置など）を保持。
  • WALReadError: WALRead 実行時のエラー詳細情報。

失敗モードと境界条件

  • **読み込み範囲外**: targetPagePtr が endptr を超える場合、-1 を返して終了します。
  • **ファイル読み込みエラー**: WALRead が失敗した場合、errno が設定されていればその値を報告し、そうでなければ読み込みバイト数の不一致を報告して pg_fatal で終了します。

保守担当者向けメモ

  • 本関数は XLogReader のコールバックとして登録されるため、シグネチャの変更には access/xlogreader.h の定義との整合性が必要です。
  • pg_waldump はフロントエンドツールであるため、エラー処理には pg_fatal を使用してユーザーに分かりやすいメッセージを表示するように設計されています。
