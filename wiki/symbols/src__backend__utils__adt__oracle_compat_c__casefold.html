casefold

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:129

目的

入力された文字列に対して大文字小文字の区別を排除した正規化（ケースフォールディング）を行い、結果を text 型として返します。Oracle データベースとの互換性を目的とした関数です。

シグネチャまたは形状

Datum casefold(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される text 型の文字列。
  • **出力**: 正規化された新しい text 型のデータ（Datum）。

前提条件と不変条件

  • 入力文字列は PG_GETARG_TEXT_PP を介して取得されるため、TOAST 展開済みのデータとして扱われます。
  • 内部で str_casefold を呼び出す際、現在のセッションの照合順序（Collation）が PG_GET_COLLATION() により適用されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP(0) により入力文字列を取得します。 2. str_casefold を呼び出し、入力文字列のデータポインタとサイズ、および現在の照合順序を渡して変換を実行します。 3. str_casefold が生成した char* 文字列を cstring_to_text で PostgreSQL の text 型に変換します。 4. 変換に使用した一時的な char* バッファを pfree で解放します。 5. PG_RETURN_TEXT_P を使用して、変換後の text データを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQL インターフェース経由で呼び出されます。
  • src/bin/pg_upgrade/check.c:1980 にて、システムカタログ内の関数名として参照されていることが確認できます。
  • **呼び出し先**:
  • str_casefold (文字列変換のコアロジック)
  • cstring_to_text (メモリ管理および型変換)
  • pfree (メモリ解放)

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • 入力文字列が NULL の場合、PG_GETARG_TEXT_PP の仕様に従いエラーまたは NULL 処理が行われます。
  • メモリ不足時には palloc（cstring_to_text 内部で使用）によりエラーが発生します。

保守担当者向けメモ

  • 本関数は utils/formatting.h で定義されている str_casefold に依存しています。
  • 照合順序（Collation）に依存した動作を行うため、ロケール設定によって結果が異なる可能性があります。
  • str_casefold が返すバッファは palloc されている前提で pfree を行っています。実装を変更する際はメモリ管理の整合性に注意してください。
