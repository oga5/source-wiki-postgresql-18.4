ScanKeyEntryInitializeWithInfo

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/scankey.c:100

目的

既存の FmgrInfo 構造体を再利用して ScanKey エントリを初期化し、関数ルックアップのオーバーヘッドを回避する。

シグネチャまたは形状

void ScanKeyEntryInitializeWithInfo(
    ScanKey entry,
    int flags,
    AttrNumber attributeNumber,
    StrategyNumber strategy,
    Oid subtype,
    Oid collation,
    FmgrInfo *finfo,
    Datum argument
)

入力と出力

  • **入力**:
  • entry: 初期化対象の ScanKey 構造体へのポインタ。
  • flags, attributeNumber, strategy, subtype, collation: スキャン条件を定義する各属性値。
  • finfo: 比較関数情報を含む FmgrInfo 構造体へのポインタ。
  • argument: 比較対象となる Datum 値。
  • **出力**: なし（entry が指すメモリ領域を直接更新）。

前提条件と不変条件

  • entry は有効なメモリ領域を指していること。
  • FmgrInfo のコピー先として CurrentMemoryContext が使用されるため、entry の生存期間中、このメモリコンテキストが有効であること。

アルゴリズムまたは振る舞い

1. entry の各フィールド（sk_flags, sk_attno, sk_strategy, sk_subtype, sk_collation, sk_argument）に引数の値を代入する。 2. fmgr_info_copy を呼び出し、finfo の内容を entry->sk_func にコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtsearch.c:1411
  • src/backend/access/nbtree/nbtutils.c:158
  • **呼び出し先**:
  • fmgr_info_copy (内部関数)

関連データ構造

  • ScanKeyData: スキャン条件を保持する構造体。
  • FmgrInfo: 関数呼び出しに必要なメタデータを保持する構造体。

失敗モードと境界条件

  • finfo が無効なポインタである場合、fmgr_info_copy 内で不正なメモリ参照が発生する可能性がある。
  • CurrentMemoryContext が不適切な場合、メモリリークや生存期間の不整合を招く可能性がある。

保守担当者向けメモ

  • 頻繁に実行されるスキャンにおいて、関数ルックアップを事前に行うことでパフォーマンスを最適化するために使用する。
  • ScanKeyInit と異なり、フラグや照合順序を詳細に制御できる。
