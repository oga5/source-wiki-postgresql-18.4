PQsetInstanceData

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/libpq-events.c:97

目的

PGconn オブジェクトに関連付けられた、特定のイベントプロシージャに対応するインスタンスデータ（void *data）を更新します。

シグネチャまたは形状

int PQsetInstanceData(PGconn *conn, PGEventProc proc, void *data);

入力と出力

  • **入力**:
  • conn: 対象となる PGconn 構造体へのポインタ。
  • proc: データを更新する対象のイベントプロシージャ（関数ポインタ）。
  • data: 設定する任意のデータへのポインタ。
  • **出力**:
  • int: 成功した場合は true (1)、失敗した場合は false (0) を返します。

前提条件と不変条件

  • conn および proc が NULL でないこと。
  • 指定された proc が、事前に PQregisterEventProc を通じて当該 PGconn に登録済みであること。

アルゴリズムまたは振る舞い

1. conn または proc が NULL の場合、直ちに false を返します。 2. conn->events 配列を走査し、登録されているイベントプロシージャの中から proc と一致するエントリを検索します。 3. 一致するエントリが見つかった場合、そのエントリの data フィールドを引数 data で更新し、true を返します。 4. 配列内に一致する proc が存在しない場合、何も変更せず false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内では、具体的な呼び出し元は特定されていません。
  • **呼び出し先**: 内部で呼び出される関数はありません。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。
  • PGEventProc: イベント処理関数の型定義。

失敗モードと境界条件

  • conn が NULL の場合、または proc が NULL の場合は失敗します。
  • 登録されていない proc を指定した場合、検索が失敗し false を返します。

保守担当者向けメモ

  • 本関数は、イベントプロシージャのアドレスをキーとして検索を行うため、同一プロシージャを複数回登録することはできません。
  • PGresult に対する同様の操作には PQresultSetInstanceData を使用してください。
