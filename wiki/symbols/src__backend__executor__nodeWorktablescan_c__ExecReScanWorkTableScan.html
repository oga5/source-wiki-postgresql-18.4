ExecReScanWorkTableScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWorktablescan.c:190

目的

再帰的クエリ（WITH RECURSIVE）の実行中に、作業テーブル（Work Table）のスキャン状態をリセットし、再スキャンを可能にする。

シグネチャまたは形状

void ExecReScanWorkTableScan(WorkTableScanState *node)

入力と出力

  • **入力**: WorkTableScanState *node（スキャン対象の実行状態構造体）
  • **出力**: なし（副作用としてスキャン状態を初期化する）

前提条件と不変条件

  • node は有効な WorkTableScanState を指している必要がある。
  • 後方スキャン（Backward Scan）はサポートされない。
  • rustate（RecursiveUnionState への参照）は、初回実行時に遅延解決されるため、NULL の可能性がある。

アルゴリズムまたは振る舞い

1. **結果スロットのクリア**: node->ss.ps.ps_ResultTupleSlot が存在する場合、ExecClearTuple を呼び出してスロットを空にする。 2. **スキャン基盤のリセット**: ExecScanReScan(&node->ss) を呼び出し、共通のスキャン状態をリセットする。 3. **作業テーブルの再スキャン**: node->rustate が解決済みであれば、tuplestore_rescan を呼び出し、親ノード（RecursiveUnion）が保持する作業テーブルの読み取り位置を先頭に戻す。未解決の場合は何もしない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:230 からの呼び出しが確認されている。
  • **呼び出し先**:
  • ExecClearTuple
  • ExecScanReScan
  • tuplestore_rescan

関連データ構造

  • WorkTableScanState: 本ノードの実行状態を保持する構造体。
  • RecursiveUnionState: rustate を介して参照される、作業テーブルを管理する親ノードの状態。

失敗モードと境界条件

  • ExecWorkTableScan が一度も呼び出されていない場合、rustate が NULL である可能性がある。この場合、tuplestore_rescan はスキップされる。

保守担当者向けメモ

  • 本関数は RecursiveUnion ノードと密接に結合している。作業テーブルの読み取りポインタ管理は、このノードが唯一のリーダーであることを前提としている。
  • rustate の遅延解決メカニズムにより、初期化直後の再スキャン要求に対しても安全に動作するよう設計されている。
