Working_State

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/twophase.c:696

目的

準備済みトランザクション（GXACT）のリストを走査・管理するための作業用状態を保持します。主に準備済みトランザクションの一覧を取得する関数内で、メモリ上の情報を一時的に保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	GlobalTransaction array;
	int			ngxacts;
	int			currIdx;
} Working_State;

入力と出力

  • **入力**: palloc を介してメモリが割り当てられ、currIdx 等のフィールドが初期化されます。
  • **出力**: 準備済みトランザクションの走査結果を保持し、関数呼び出し元へ返されます。

前提条件と不変条件

  • ngxacts は、現在システム内に存在する準備済みトランザクションの総数を表します。
  • currIdx は、走査中の現在のインデックスを指します。
  • 構造体内の array は、GlobalTransaction 型の配列を指すポインタとして機能します。

アルゴリズムまたは振る舞い

1. palloc により Working_State 構造体分のメモリが確保されます。 2. currIdx を用いて、array に格納された準備済みトランザクションを順次参照します。 3. funcctx->user_fctx にこの構造体を格納することで、複数回にわたる関数呼び出し（セットを返す関数など）の間で状態を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/twophase.c 内の準備済みトランザクション一覧を取得する関数（pg_prepared_xact 等）から参照されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • GlobalTransaction: 準備済みトランザクションの個別の状態を保持するデータ構造。
  • TwoPhaseStateData: 共有メモリ上で準備済みトランザクションを管理するグローバルな構造体。

失敗モードと境界条件

  • palloc が失敗した場合、メモリ不足により処理が中断されます。
  • ngxacts が 0 の場合、走査は即座に終了します。
  • currIdx が ngxacts に達した際、走査の終了条件として適切に処理される必要があります。

保守担当者向けメモ

  • 本構造体は、主に準備済みトランザクションの情報を外部（SQLインターフェース等）へ公開する際のイテレータとして機能します。
  • 共有メモリ上の TwoPhaseState を直接操作するものではなく、あくまで読み取り専用の作業用コピーとして利用してください。
