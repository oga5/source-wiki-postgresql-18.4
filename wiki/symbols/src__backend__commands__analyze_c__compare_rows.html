compare_rows

定義

  • 種別: function
  • 定義箇所: src/backend/commands/analyze.c:1360

目的

HeapTuple の物理的な格納位置（ブロック番号およびオフセット番号）に基づいて、2つのタプルを比較する比較関数です。主にソート処理のコールバックとして使用されます。

シグネチャまたは形状

static int compare_rows(const void *a, const void *b, void *arg)

入力と出力

  • **入力**:
  • a, b: 比較対象の HeapTuple へのポインタを指すポインタ。
  • arg: 使用されない汎用引数。
  • **出力**:
  • a が b より小さい場合は -1、大きい場合は 1、等しい場合は 0 を返します。

前提条件と不変条件

  • a および b は有効な HeapTuple を指している必要があります。
  • HeapTuple 構造体内の t_self（ItemPointer）が初期化されていることが前提です。

アルゴリズムまたは振る舞い

1. a と b からそれぞれ BlockNumber と OffsetNumber を抽出します。 2. まず BlockNumber を比較し、値が異なる場合はその結果を返します。 3. BlockNumber が等しい場合、OffsetNumber を比較し、その結果を返します。 4. 両者が等しい場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/analyze.c 内のソート処理（qsort 等）から呼び出されます。
  • **呼び出し先**: ItemPointerGetBlockNumber および ItemPointerGetOffsetNumber マクロを使用します。

関連データ構造

  • HeapTuple: 比較対象となるタプル構造体。
  • ItemPointer: タプルの物理位置を特定するデータ構造。

失敗モードと境界条件

  • a または b が NULL の場合の動作は定義されていません。
  • t_self が無効な値（InvalidBlockNumber 等）である場合、比較結果は物理的な格納順序に基づいた値となります。

保守担当者向けメモ

  • 本関数は qsort 互換のシグネチャを持っています。
  • 物理的な順序（ディスク上の位置）のみを比較対象としており、タプルの内容（データ値）は考慮しません。
