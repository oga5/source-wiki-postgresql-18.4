AlterForeignServerOwner_oid

定義

  • 種別: function
  • 定義箇所: src/backend/commands/foreigncmds.c:460

目的

指定されたOIDを持つ外部サーバーの所有者を変更します。

シグネチャまたは形状

void AlterForeignServerOwner_oid(Oid srvId, Oid newOwnerId)

入力と出力

  • srvId (Oid): 変更対象の外部サーバーのOID。
  • newOwnerId (Oid): 新しい所有者のロールOID。
  • 出力: なし（成功時はシステムカタログを更新し、失敗時はエラーをスロー）。

前提条件と不変条件

  • pg_foreign_server システムカタログに対する RowExclusiveLock が取得されます。
  • 対象の srvId が存在しない場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. pg_foreign_server テーブルを RowExclusiveLock でオープンします。 2. FOREIGNSERVEROID システムキャッシュを使用して、対象サーバーのタプルをコピーとして取得します。 3. タプルが有効でない場合は ERRCODE_UNDEFINED_OBJECT エラーを発生させます。 4. AlterForeignServerOwner_internal を呼び出し、所有者変更の内部処理（権限チェックやカタログ更新）を実行します。 5. 取得したタプルを解放し、テーブルをクローズします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_shdepend.c:1683
  • 呼び出し先: AlterForeignServerOwner_internal

関連データ構造

  • HeapTuple: システムカタログの行データ。
  • Relation: pg_foreign_server システムカタログの記述子。

失敗モードと境界条件

  • 指定したOIDの外部サーバーが存在しない場合、ereport(ERROR, ...) により処理が中断されます。

保守担当者向けメモ

  • 本関数はOIDベースの所有者変更処理を提供します。名前ベースの変更が必要な場合は AlterForeignServerOwner を参照してください。
  • 実際のカタログ更新ロジックは AlterForeignServerOwner_internal に集約されています。
