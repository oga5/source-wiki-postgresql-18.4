lsinfo

定義

  • 種別: struct
  • 定義箇所: src/timezone/pgtz.h:35

目的

タイムゾーンデータにおける閏秒（leap second）の遷移情報と、その際の補正値を保持するために使用されます。

シグネチャまたは形状

struct lsinfo {
    pg_time_t ls_trans; /* 遷移時刻 */
    int64     ls_corr;  /* 適用する補正値 */
};

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。tzload() 等の処理において、タイムゾーンファイルから読み込まれたバイナリデータが本構造体のフィールドにマッピングされます。

前提条件と不変条件

  • ls_trans は pg_time_t 型であり、システム時刻の遷移時点を表します。
  • ls_corr は int64 型であり、当該遷移時点での時刻補正量を保持します。

アルゴリズムまたは振る舞い

本構造体は、IANA タイムゾーンデータベースのファイル形式（tzfile）に基づいたメモリ表現の一部です。時刻変換処理において、特定の時刻が閏秒の遷移を跨ぐかどうかを判定し、適切な補正値を適用するための参照データとして利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/timezone/localtime.c 内の関数（lp ポインタとして参照されるケースが確認されています）。
  • 呼び出し先: 本構造体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • struct state: タイムゾーンの遷移情報全体を保持する構造体であり、そのメンバとして lsinfo の配列（lsis[TZ_MAX_LEAPS]）を保持します。
  • tzfile.h: 本構造体が準拠するバイナリフォーマットの定義元です。

失敗モードと境界条件

  • TZ_MAX_LEAPS で定義された最大数を超える閏秒情報は保持できません。
  • 構造体の各フィールドは、読み込み元のタイムゾーンファイルが破損している場合、不正な値を持つ可能性があります。

保守担当者向けメモ

  • 本構造体は src/timezone/localtime.c での時刻計算ロジックと密接に結合しています。変更を加える際は、tzfile.h の定義および localtime.c でのポインタ操作への影響を確認してください。
