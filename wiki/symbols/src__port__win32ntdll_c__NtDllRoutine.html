NtDllRoutine

定義

  • 種別: struct
  • 定義箇所: src/port/win32ntdll.c:24

目的

Windowsの ntdll.dll が提供する低レベルなシステム関数を、実行時に動的に解決して保持するためのマッピング構造体です。

シグネチャまたは形状

typedef struct NtDllRoutine
{
    const char *name;
    pg_funcptr_t *address;
} NtDllRoutine;

入力と出力

  • name: 解決対象となるNT関数の名前（文字列）。
  • address: 解決された関数ポインタを格納する pg_funcptr_t 型の変数へのポインタ。

前提条件と不変条件

  • address は、ntdll.dll から取得した関数ポインタを格納するメモリ領域を指している必要があります。
  • この構造体の配列は、initialize_ntdll() 内で GetProcAddress を用いて関数アドレスを解決するために使用されます。

アルゴリズムまたは振る舞い

initialize_ntdll() 関数において、この構造体の配列（routines）をループ処理し、各エントリの name を用いて ntdll.dll から関数アドレスを検索します。解決されたアドレスは、対応する address ポインタに代入されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/win32ntdll.c 内の initialize_ntdll() 関数がこの構造体の配列を参照します。
  • 呼び出し先: 情報なし。

関連データ構造

  • pg_funcptr_t: 関数ポインタを保持するための型定義。
  • routines (配列): NtDllRoutine 型の静的配列。src/port/win32ntdll.c:30 で定義されています。

失敗モードと境界条件

  • GetProcAddress が指定された name の関数を見つけられない場合、address には NULL が格納される可能性があります。
  • 呼び出し元である initialize_ntdll() は、解決失敗時にエラーハンドリングを行う設計となっています。

保守担当者向けメモ

  • 新しいNT関数を追加する場合は、win32ntdll.h に型定義と extern 宣言を追加し、win32ntdll.c の routines 配列に新しいエントリを追加してください。
  • 本構造体は静的リンクを避けるための動的ロード機構の一部であり、スレッドセーフなアクセスが必要な場合は呼び出し元での排他制御を検討してください。
