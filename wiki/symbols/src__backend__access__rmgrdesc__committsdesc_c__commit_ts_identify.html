commit_ts_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/committsdesc.c:42

目的

WAL（Write-Ahead Log）レコードの info ビットから、Commit Timestamp 機能に関連する操作の種類を識別し、人間が読み取り可能な文字列を返します。

シグネチャまたは形状

const char *commit_ts_identify(uint8 info)

入力と出力

  • **入力**: uint8 info （WALレコードの info フィールド）
  • **出力**: const char * （操作名を表す文字列、または不明な場合は NULL）

前提条件と不変条件

  • info は COMMIT_TS_ZEROPAGE または COMMIT_TS_TRUNCATE のいずれかである必要があります。
  • 戻り値は静的な文字列リテラルへのポインタであり、呼び出し元で解放する必要はありません。

アルゴリズムまたは振る舞い

1. info 引数に対して switch 文を実行します。 2. COMMIT_TS_ZEROPAGE の場合は "ZEROPAGE" を返します。 3. COMMIT_TS_TRUNCATE の場合は "TRUNCATE" を返します。 4. 上記以外の場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h で定義されているリソースマネージャインターフェース（RM_COMMIT_TS_ID）を通じて、WAL解析ツール（pg_waldump 等）から間接的に呼び出されます。
  • **呼び出し先**: なし。

関連データ構造

  • COMMIT_TS_ZEROPAGE, COMMIT_TS_TRUNCATE: access/commit_ts.h で定義される定数。

失敗モードと境界条件

  • 未定義の info 値が渡された場合、NULL を返します。

保守担当者向けメモ

  • 本関数は Commit Timestamp 機能の WAL 操作のみを対象としています。将来的に新しい WAL 操作が追加された場合は、switch 文に新しい case を追加してください。
