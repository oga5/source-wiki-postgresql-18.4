stop_postmaster_atexit

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/server.c:192

目的

プログラム終了時に実行中のデータベースサーバー（Postmaster）を安全に停止させるためのクリーンアップハンドラです。

シグネチャまたは形状

static void stop_postmaster_atexit(void)

入力と出力

  • 入力: なし
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • atexit(3) を介して登録されることを前提としています。
  • 内部で呼び出される stop_postmaster が、現在のプロセス状態に基づいてサーバーを停止できる必要があります。

アルゴリズムまたは振る舞い

1. stop_postmaster(true) を呼び出します。 2. 引数 true は、サーバー停止処理においてクリーンアップを伴う実行であることを示唆します。

呼び出し元と呼び出し先

  • 呼び出し元: atexit() 関数に登録されることで、プログラム終了時に呼び出されます。
  • 呼び出し先: stop_postmaster(bool)

関連データ構造

  • os_info.running_cluster: stop_postmaster 内部で参照される、停止対象のクラスター情報を保持する構造体（server.c の文脈より）。

失敗モードと境界条件

  • stop_postmaster の内部実装に依存します。サーバーが既に停止している場合や、停止コマンドが失敗した場合の挙動は stop_postmaster の仕様に従います。

保守担当者向けメモ

  • 本関数は atexit フック専用です。直接呼び出すのではなく、終了時の自動停止が必要な場合に atexit(stop_postmaster_atexit) を使用してください。
