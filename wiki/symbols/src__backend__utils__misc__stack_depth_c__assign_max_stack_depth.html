assign_max_stack_depth

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/stack_depth.c:158

目的

GUC（Grand Unified Configuration）パラメータ max_stack_depth の更新を反映し、内部的なスタック制限値（バイト単位）を同期させるために使用されます。

シグネチャまたは形状

void assign_max_stack_depth(int newval, void *extra)

入力と出力

  • **入力**:
  • newval: 設定される新しいスタック深さの制限値（KB単位）。
  • extra: 使用されていません。
  • **出力**: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • newval は KB 単位の値として渡されることが期待されています。
  • 内部変数 max_stack_depth_bytes が、この関数によって newval * 1024 に更新されます。

アルゴリズムまたは振る舞い

1. 引数 newval を ssize_t 型にキャストし、1024 を乗算してバイト単位の値を算出します。 2. 算出された値をグローバル変数 max_stack_depth_bytes に代入します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc_tables.c における GUC パラメータ定義内で参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • max_stack_depth_bytes: 本関数によって更新される、スタック深さ判定に使用されるグローバル変数。

失敗モードと境界条件

  • newval が非常に大きい場合、ssize_t への変換時にオーバーフローが発生する可能性がありますが、GUC システム側でのバリデーションが前提となります。

保守担当者向けメモ

  • 本関数は GUC フックとして機能します。設定値の妥当性検証は check_max_stack_depth が担当するため、本関数は値の適用のみに専念しています。
