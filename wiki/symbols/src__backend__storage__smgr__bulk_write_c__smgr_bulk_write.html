smgr_bulk_write

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/bulk_write.c:322

目的

リレーションの新規作成や初期ロード時に、ページを書き込みキューに追加し、バッファマネージャを介さずに効率的なディスク書き込みを行うためのインターフェースを提供します。

シグネチャまたは形状

void smgr_bulk_write(BulkWriteState *bulkstate, BlockNumber blocknum, BulkWriteBuffer buf, bool page_std)

入力と出力

  • bulkstate (入力): 書き込み操作の状態を保持する BulkWriteState 構造体。
  • blocknum (入力): 書き込み対象のブロック番号。
  • buf (入力): 書き込むページデータを含む BulkWriteBuffer。
  • page_std (入力): 標準的なページフォーマットかどうかを示すフラグ。
  • 出力: なし（副作用として bulkstate 内のキューを更新）。

前提条件と不変条件

  • smgr_bulk_write に渡されたバッファの所有権は BulkWriteState に移譲されます。呼び出し側で pfree を行ってはなりません。
  • 対象リレーションに対して、他のバックエンドがアクセスしていない（排他制御が担保されている）ことが前提です。

アルゴリズムまたは振る舞い

1. bulkstate->pending_writes 配列の現在のインデックス位置に、引数で受け取ったバッファ、ブロック番号、ページフラグを格納します。 2. bulkstate->npending をインクリメントします。 3. 保留中の書き込み数が MAX_PENDING_WRITES に達した場合、smgr_bulk_flush(bulkstate) を呼び出し、キュー内のページを物理的にフラッシュします。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/gist/gistbuild.c
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/nbtree/nbtree.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/access/spgist/spginsert.c
  • src/backend/catalog/storage.c
  • 呼び出し先:
  • smgr_bulk_flush (保留数が上限に達した場合)

関連データ構造

  • BulkWriteState: 書き込み操作の状態を管理する構造体。
  • PendingWrite: 保留中の書き込み情報を保持する構造体。

失敗モードと境界条件

  • MAX_PENDING_WRITES に達した際の自動フラッシュ処理が境界条件となります。

保守担当者向けメモ

  • 本関数はバッファの所有権を移譲するため、呼び出し側でのメモリ管理に注意してください。
  • smgr_bulk_finish との組み合わせで、チェックポイントとの競合を回避するロジックが重要となります。
