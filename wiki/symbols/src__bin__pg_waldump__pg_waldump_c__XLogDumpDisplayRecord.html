XLogDumpDisplayRecord

定義

  • 種別: function
  • 定義箇所: src/bin/pg_waldump/pg_waldump.c:545

目的

WAL レコードの内容を解析し、人間が読み取れる形式で標準出力（stdout）に整形表示する。

シグネチャまたは形状

static void XLogDumpDisplayRecord(XLogDumpConfig *config, XLogReaderState *record)

入力と出力

  • **入力**:
  • config: 表示設定（ブロック詳細の有無など）を保持する XLogDumpConfig 構造体へのポインタ。
  • record: 表示対象の WAL レコード状態を保持する XLogReaderState 構造体へのポインタ。
  • **出力**: なし（void）。副作用として標準出力へレコード情報を書き出す。

前提条件と不変条件

  • record は有効な XLogReaderState を指している必要がある。
  • GetRmgrDesc により、レコードのリソースマネージャ ID に対応する RmgrDescData が取得可能であること。

アルゴリズムまたは振る舞い

1. GetRmgrDesc を使用して、レコードのリソースマネージャ（RMGR）情報を取得する。 2. XLogRecGetLen 等を用いてレコードの長さ、トランザクション ID、LSN、直前の LSN を取得し、基本情報を出力する。 3. rm_identify を呼び出し、レコードのサブタイプ情報を特定して出力する。 4. rm_desc を呼び出し、リソースマネージャ固有のレコード詳細を StringInfo バッファに生成して出力する。 5. XLogRecGetBlockRefInfo を使用し、設定に応じてブロック参照情報を取得・出力する。 6. 使用した StringInfo バッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_waldump/pg_waldump.c 内のメイン読み込みループ（1291行目付近）。
  • **呼び出し先**:
  • GetRmgrDesc
  • XLogRecGetRmid, XLogRecGetInfo, XLogRecGetPrev, XLogRecGetLen, XLogRecGetTotalLen, XLogRecGetXid, XLogRecGetBlockRefInfo
  • initStringInfo, resetStringInfo, pfree
  • desc->rm_identify, desc->rm_desc

関連データ構造

  • XLogDumpConfig: 表示オプションを制御。
  • XLogReaderState: WAL レコードの読み込み状態を保持。
  • RmgrDescData: リソースマネージャごとの説明用関数ポインタを保持。

失敗モードと境界条件

  • rm_identify が NULL を返す場合、未知のレコードタイプとして 16 進数で出力する。
  • StringInfo のメモリ確保に失敗した場合、PostgreSQL のメモリ管理機構によりエラーがスローされる可能性がある。

保守担当者向けメモ

  • 本関数は pg_waldump の出力形式を決定する主要な箇所である。出力フォーマットを変更する場合は、rmgrdesc.c 内の各リソースマネージャの rm_desc 実装と整合性を保つこと。
  • XLogRecGetBlockRefInfo の第 2 引数に true を渡しており、常にブロック参照情報の取得を試みる仕様となっている。
