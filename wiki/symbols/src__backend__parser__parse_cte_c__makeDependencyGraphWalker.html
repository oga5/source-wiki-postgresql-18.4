makeDependencyGraphWalker

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_cte.c:669

目的

CTE（共通テーブル式）間の依存関係を特定し、再帰的CTEの自己参照を検出するためのツリーウォーカーです。

シグネチャまたは形状

static bool makeDependencyGraphWalker(Node *node, CteState *cstate);

入力と出力

  • **入力**:
  • node: 解析対象の構文木ノード。
  • cstate: CTEの解析状態を保持する CteState 構造体。
  • **出力**:
  • bool: 走査を継続する場合は false、中断する場合は true を返します。

前提条件と不変条件

  • cstate は初期化済みである必要があります。
  • WithClause ノード自体は、この関数が直接走査するのではなく、WalkInnerWith を介して制御される必要があります。

アルゴリズムまたは振る舞い

1. **RangeVarの処理**:

  • スキーマ修飾のない名前の場合、cstate->innerwiths を確認し、内側の WITH 句で隠蔽されているか判定します。
  • 隠蔽されていない場合、現在のクエリレベルのCTEリストと照合します。
  • 一致するCTEが見つかった場合、依存関係（depends_on）を追加するか、自身への参照であれば cterecursive フラグを立てます。

    • 2. **ステートメントの処理**:

  • SelectStmt, InsertStmt, DeleteStmt, UpdateStmt, MergeStmt に withClause が含まれる場合、WalkInnerWith を呼び出してネストされた WITH 句を処理します。

    • 3. **再帰的走査**:

  • 上記以外のノードに対しては raw_expression_tree_walker を使用して再帰的に走査を続行します。

呼び出し元と呼び出し先

  • **呼び出し元**: makeDependencyGraph および WalkInnerWith から呼び出されます。
  • **呼び出し先**: WalkInnerWith および raw_expression_tree_walker を呼び出します。

関連データ構造

  • CteState: 解析中のCTEリストや依存関係の状態を保持します。
  • CommonTableExpr: CTEの定義情報。
  • RangeVar: 参照されているリレーション名。

失敗モードと境界条件

  • node が NULL の場合は直ちに false を返します。
  • WithClause ノードに遭遇した際は、無限再帰や不適切な走査を防ぐため、即座に false を返して走査を停止します。

保守担当者向けメモ

  • 本関数は raw_expression_tree_walker を利用して構文木を探索しますが、WITH 句のネスト構造を正しく扱うために、特定のステートメントノードでは明示的に WalkInnerWith を経由するよう制御されています。変更時はこの可視性制御が維持されているか確認してください。
