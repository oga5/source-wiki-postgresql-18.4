BufferCacheNumaContext

定義

  • 種別: struct
  • 定義箇所: contrib/pg_buffercache/pg_buffercache_pages.c:82

目的

pg_buffercache_numa_pages 関数が実行中に保持する状態（コンテキスト）を管理するための構造体です。NUMA ノード情報の取得および SRF（Set-Returning Function）としての反復処理に必要なメタデータを保持します。

シグネチャまたは形状

typedef struct {
    TupleDesc           tupdesc;          /* 結果セットのタプル記述子 */
    int                 buffers_per_page; /* ページあたりのバッファ数 */
    int                 pages_per_buffer; /* バッファあたりのページ数 */
    int                 os_page_size;     /* OS のページサイズ */
    BufferCacheNumaRec *record;           /* NUMA 記録データの配列 */
} BufferCacheNumaContext;

入力と出力

  • 本構造体自体はデータ保持用であり、直接的な入出力関数ではありません。
  • pg_buffercache_pages.c 内で palloc を介してメモリ確保され、SRF の呼び出し間で状態を維持するために使用されます。

前提条件と不変条件

  • record フィールドは、バッファキャッシュの NUMA 情報を格納するために動的に割り当てられる必要があります。
  • tupdesc は、SRF が返す行形式と一致している必要があります。

アルゴリズムまたは振る舞い

  • SRF の初回呼び出し時に初期化され、以降の呼び出しで record に格納された NUMA 情報を順次参照するために利用されます。
  • 内部的に OS のページサイズやバッファ構成情報を保持することで、NUMA ノード情報の計算を効率化します。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_buffercache_numa_pages 関数内で palloc され、コンテキストとして利用されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • BufferCacheNumaRec: 個別の NUMA ノード情報を保持する構造体。本コンテキスト内の record フィールドがこの配列を指します。

失敗モードと境界条件

  • メモリ不足時: palloc 失敗によりエラーが発生します。
  • NUMA 非対応環境: OS が NUMA をサポートしていない場合、本構造体を利用した情報取得処理が適切に機能しない可能性があります。

保守担当者向けメモ

  • 本構造体は pg_buffercache_numa_pages の実行コンテキストとしてのみ使用されます。
  • フィールドの追加や変更を行う際は、SRF の初期化処理および反復処理（SRF_FIRSTCALL_INIT 等）との整合性に注意してください。
