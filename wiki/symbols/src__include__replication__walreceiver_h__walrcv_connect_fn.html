walrcv_connect_fn

定義

  • 種別: typedef
  • 定義箇所: src/include/replication/walreceiver.h:242

目的

プライマリサーバーへの接続を確立するための関数ポインタ型です。通信プロトコルの詳細を抽象化し、WALレシーバーが動的に接続実装をロードできるようにします。

シグネチャまたは形状

typedef WalReceiverConn *(*walrcv_connect_fn) (const char *conninfo,
                                               bool replication,
                                               bool logical,
                                               bool must_use_password,
                                               const char *appname,
                                               char **err);

入力と出力

  • **入力**:
  • conninfo: 接続文字列。
  • replication: レプリケーション接続であることを示すフラグ。
  • logical: 論理レプリケーションを使用するかどうかのフラグ。
  • must_use_password: パスワード認証を強制するかどうかのフラグ。
  • appname: 接続に使用するアプリケーション名。
  • **出力**:
  • 戻り値: 成功時は WalReceiverConn 構造体へのポインタ、失敗時は NULL。
  • err: 失敗時にエラーメッセージが格納される文字列ポインタへのポインタ。

前提条件と不変条件

  • 呼び出し元は、戻り値が NULL の場合に err を通じて返されるエラーメッセージを適切に処理する必要があります。
  • 接続先サーバーがレプリケーションプロトコルをサポートしている必要があります。

アルゴリズムまたは振る舞い

1. 指定された conninfo を使用してプライマリサーバーへの接続を試行します。 2. replication および logical フラグに基づき、適切なレプリケーションモードで接続を初期化します。 3. 認証が必要な場合、must_use_password に従って認証プロセスを実行します。 4. 接続成功時は WalReceiverConn オブジェクトを生成して返します。失敗時はエラー詳細を err に書き込みます。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/replication/walreceiver.h 内の WalReceiverFunctions 構造体を通じて参照されます。具体的な呼び出し元コードの詳細は提供されていません。
  • 呼び出し先: 実装（libpqwalreceiver 等）に依存します。

関連データ構造

  • WalReceiverConn: 接続状態を保持する不透明な構造体。

失敗モードと境界条件

  • 接続文字列が不正な場合、またはサーバーが到達不能な場合に NULL を返します。
  • 認証失敗やプロトコル不一致が発生した場合、err にエラーメッセージが設定されます。

保守担当者向けメモ

  • 本シンボルは関数ポインタ型です。具体的な実装は src/backend/replication/libpqwalreceiver/ などの通信バックエンドモジュールを確認してください。
