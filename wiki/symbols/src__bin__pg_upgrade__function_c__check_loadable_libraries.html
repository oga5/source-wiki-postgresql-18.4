check_loadable_libraries

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/function.c:164

目的

旧クラスタで使用されている共有ライブラリが、新クラスタ環境においてもロード可能であることを検証する。

シグネチャまたは形状

void check_loadable_libraries(void)

入力と出力

  • **入力**: グローバル変数 os_info.libraries（検証対象のライブラリリスト）および new_cluster（接続先クラスタ情報）。
  • **出力**: なし（成功時）。失敗時は loadable_libraries.txt にエラー詳細を書き出し、pg_fatal を呼び出してプロセスを終了する。

前提条件と不変条件

  • os_info.libraries に有効なライブラリ情報が格納されていること。
  • new_cluster への接続が確立可能であること。
  • ライブラリのロード順序は、依存関係を考慮して library_name_compare によりソートされていること。

アルゴリズムまたは振る舞い

1. connectToServer を使用して新クラスタの template1 に接続する。 2. os_info.libraries を library_name_compare に基づきソートする。 3. リスト内の各ライブラリに対し、重複を除去しつつ LOAD コマンドを発行する。 4. LOAD が失敗した場合、loadable_libraries.txt を作成（または追記）し、エラーメッセージと該当データベース名を記録する。 5. 全ての検証終了後、ファイルが作成されていれば pg_fatal で終了し、そうでなければ check_ok() を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c:716
  • **呼び出し先**:
  • connectToServer (新クラスタへの接続)
  • qsort (ライブラリのソート)
  • library_name_compare (比較関数)
  • PQexec, PQescapeStringConn, PQresultStatus, PQclear, PQfinish (libpq関数)
  • fopen_priv, fprintf, fclose (ファイル操作)
  • pg_fatal, pg_log, prep_status, check_ok (pg_upgradeユーティリティ)

関連データ構造

  • os_info: ライブラリ情報を含むグローバル構造体。
  • LibraryInfo: ライブラリ名とデータベース番号を保持する構造体。
  • PGconn, PGresult: libpq の接続および結果オブジェクト。

失敗モードと境界条件

  • **ロード失敗**: LOAD コマンドが PGRES_COMMAND_OK を返さない場合、致命的エラーとして処理される。
  • **ファイル操作失敗**: loadable_libraries.txt のオープンに失敗した場合、pg_fatal で終了する。
  • **重複ライブラリ**: 同一ライブラリ名が連続する場合、2回目以降の LOAD 試行はスキップされる。

保守担当者向けメモ

  • 本関数はライブラリのロード失敗を「致命的」と見なすため、アップグレード実行前に環境の整合性を確認すること。
  • library_name_compare によるソート順序は、ライブラリ間の依存関係解決に不可欠であるため、変更時は注意が必要。
