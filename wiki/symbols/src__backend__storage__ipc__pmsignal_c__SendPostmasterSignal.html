SendPostmasterSignal

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:164

目的

バックエンドプロセスからPostmaster（親プロセス）に対し、特定のイベント発生を非同期に通知する。

シグネチャまたは形状

void SendPostmasterSignal(PMSignalReason reason)

入力と出力

  • **入力**: reason (PMSignalReason 型) - 通知するイベントの種類（例: PMSignalReason 列挙値）。
  • **出力**: なし（void）。

前提条件と不変条件

  • IsUnderPostmaster が真であること。スタンドアロンバックエンドで呼び出された場合は何もしない。
  • PMSignalState が初期化済みであること。

アルゴリズムまたは振る舞い

1. IsUnderPostmaster を確認し、偽であれば即座にリターンする。 2. 共有メモリ上の PMSignalState->PMSignalFlags 配列の指定された reason インデックスを true に設定する。この操作はアトミックに行われる。 3. kill(PostmasterPid, SIGUSR1) を実行し、Postmasterプロセスに対して SIGUSR1 シグナルを送信する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/multixact.c (オートバキュームランチャー起動要求)
  • src/backend/access/transam/varsup.c (オートバキュームランチャー起動要求)
  • src/backend/access/transam/xlogrecovery.c (リカバリ状態遷移通知)
  • src/backend/postmaster/autovacuum.c (オートバキュームワーカー起動要求)
  • **呼び出し先**:
  • kill() (OSシステムコール)

関連データ構造

  • PMSignalState: 共有メモリ上のシグナル状態を保持するグローバル構造体。
  • PMSignalFlags: PMSignalState 内の各イベントの発生有無を保持するフラグ配列。

失敗モードと境界条件

  • IsUnderPostmaster が偽の場合、副作用なしで正常終了する。
  • kill() が失敗した場合の挙動については、本関数内では明示的なエラーハンドリングを行っていない。

保守担当者向けメモ

  • PMSignalFlags は sig_atomic_t を使用しており、シグナルハンドラ内での更新を想定した設計となっている。
  • 本関数は SIGUSR1 を使用するため、Postmaster側では CheckPostmasterSignal() を用いてフラグを確認する必要がある。
