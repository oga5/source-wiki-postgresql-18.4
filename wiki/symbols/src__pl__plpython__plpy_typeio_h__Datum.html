Datum

定義

  • 種別: typedef
  • 定義箇所: src/pl/plpython/plpy_typeio.h:28
  • 補足: fmgr.h:26 にも定義が存在します。

目的

PostgreSQL における汎用的なデータ表現型です。SQL のデータ型を C 言語レベルで統一的に扱うための型として使用されます。

シグネチャまたは形状

typedef として定義されており、通常は uintptr_t と同等のサイズを持つ整数型として実装されます。

入力と出力

  • 入力: PostgreSQL の関数や演算子において、任意のデータ型の値として渡されます。
  • 出力: 関数や演算子の戻り値として、あるいはデータ構造内の値として保持されます。

前提条件と不変条件

  • 値の解釈は、対応する Oid（型識別子）に依存します。
  • Datum 自体には型情報が含まれないため、呼び出し側が正しい型としてキャストまたは変換を行う必要があります。

アルゴリズムまたは振る舞い

  • plpy_typeio.h の文脈では、PLyDatumToObFunc などの変換関数において、PostgreSQL 側のデータを Python オブジェクトへ変換する際の引数として渡されます。
  • 変換処理の過程で、Datum は PLy_input_convert を通じて PyObject へマーシャリングされます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/amcheck などのモジュールや、PostgreSQL のコアエンジン全般。
  • 呼び出し先: Datum を引数に取る各種の型変換関数（PLyDatumToObFunc など）や、PostgreSQL の関数マネージャ（fmgr）。
  • 注記: 提供されたソース excerpt には、特定の呼び出し元・呼び出し先の直接的な関係性は記述されていません。

関連データ構造

  • PLyDatumToOb: PostgreSQL から Python への変換状態を保持する構造体。
  • PyObject: Python 側のデータ表現。
  • Oid: Datum の型を識別するために併用される識別子。

失敗モードと境界条件

  • 不適切な型として解釈された場合、メモリ破壊や不正なデータ読み取りが発生する可能性があります。
  • Datum がポインタとして扱われる場合（byValue が false の型）、その有効期間は PostgreSQL のメモリコンテキストに依存します。

保守担当者向けメモ

  • 本シンボルは PostgreSQL 全体で共有される基盤型です。plpy_typeio.h 内での使用は、主に SQL 型から Python 型への変換インターフェースの一部として行われます。
  • Datum の操作を行う際は、対象のデータ型が byValue なのか byReference なのかを常に意識してください。
