sync_queue_sync_all

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/relfilenumber.c:59

目的

swap モードの移行処理において、キューに蓄積されたファイルパスに対して一括で fsync を実行し、永続化を保証する。

シグネチャまたは形状

static inline void sync_queue_sync_all(void)

入力と出力

  • 入力: なし（グローバル変数 sync_queue、sync_queue_len、sync_queue_inited を参照）
  • 出力: なし（副作用としてファイルシステムへの同期を実行）

前提条件と不変条件

  • sync_queue_inited が真であること（偽の場合は何もせず終了する）。
  • sync_queue に格納されているファイルパスが有効であること。

アルゴリズムまたは振る舞い

1. sync_queue_inited を確認し、初期化されていなければ即座に復帰する。 2. sync_queue_len に基づき、キュー内の全ファイルパスに対して fsync_fname(path, false) を呼び出す。 3. fsync_fname が 0 以外の値を返した場合、pg_fatal を呼び出して処理を中断する。 4. 処理完了後、sync_queue_len を 0 にリセットし、キューを空にする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_upgrade/relfilenumber.c:38
  • src/bin/pg_upgrade/relfilenumber.c:83
  • src/bin/pg_upgrade/relfilenumber.c:217
  • 呼び出し先:
  • fsync_fname (共通ユーティリティ)
  • pg_fatal (エラーハンドリング)

関連データ構造

  • sync_queue: 同期対象のファイルパスを保持する配列。
  • sync_queue_len: 現在キューに保持されているファイル数。
  • sync_queue_inited: キューの初期化状態を示すフラグ。

失敗モードと境界条件

  • fsync_fname が失敗した場合、pg_fatal によりプログラムが異常終了する。
  • キューが空（sync_queue_len == 0）の場合、ループは実行されず、即座に終了する。

保守担当者向けメモ

  • 本関数は swap モードにおけるパフォーマンス最適化のためのバッファリング機構の一部です。
  • sync_queue_push で追加されたファイルは、本関数が呼ばれるまでディスクへの同期が保証されないため、呼び出しタイミングの整合性に注意してください。
