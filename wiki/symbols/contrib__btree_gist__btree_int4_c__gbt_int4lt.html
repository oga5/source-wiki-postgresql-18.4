gbt_int4lt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int4.c:47

目的

2つの int4（4バイト整数）値を比較し、左辺が右辺より小さいかどうかを判定する。

シグネチャまたは形状

bool gbt_int4lt(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の int4 値へのポインタ。
  • b: 比較対象の int4 値へのポインタ。
  • flinfo: 関数呼び出し情報（本関数内では未使用）。
  • **出力**:
  • bool: *a < *b が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • a および b は、有効な int32 値を指すポインタである必要がある。
  • 本関数は int4 型の標準的な比較演算子 < に準拠する。

アルゴリズムまたは振る舞い

1. a が指す値を int32 としてキャストし、デリファレンスする。 2. b が指す値を int32 としてキャストし、デリファレンスする。 3. 両者の値を比較し、その結果を論理値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int4.c 内の静的構造体（tinfo 等）から参照されている。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • int32: 比較対象となる PostgreSQL の 4 バイト整数型。

失敗モードと境界条件

  • ポインタ a または b が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • 整数値の範囲外（オーバーフロー等）については、C 言語の標準的な比較動作に従う。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された汎用 GiST ユーティリティから呼び出されることを想定した比較関数である。
  • 同様のパターンで gbt_int4gt, gbt_int4ge, gbt_int4le, gbt_int4eq が定義されており、これらと一貫性を保つこと。
