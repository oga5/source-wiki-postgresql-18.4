nfa

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regguts.h:356

目的

正規表現のコンパイルフェーズにおいて、非決定性有限オートマトン（NFA）の状態とアーク（弧）の集合を保持・管理するための構造体です。

シグネチャまたは形状

struct nfa {
    struct state *pre, *init, *final, *post;
    int nstates;
    struct state *states, *slast, *freestates;
    struct arc *freearcs;
    struct statebatch *lastsb;
    struct arcbatch *lastab;
    size_t lastsbused, lastabused;
    struct colormap *cm;
    color bos[2], eos[2];
    int flags, minmatchall, maxmatchall;
    struct vars *v;
    struct nfa *parent;
};

入力と出力

  • **入力**: コンパイルプロセスにおいて、regcomp が本構造体を構築し、colormap を用いて文字クラスを色（color）に変換します。
  • **出力**: 最終的に cnfa（コンパクト化された NFA）へ変換され、実行時のマッチング処理に利用されます。

前提条件と不変条件

  • cm（カラーマップ）が適切に初期化されている必要があります。
  • pre（前初期状態）、init（初期状態）、final（終了状態）、post（後終了状態）の各ポインタは、NFA の構造を定義する基点となります。
  • freestates および freearcs は、メモリ再利用のための空きリストとして機能します。

アルゴリズムまたは振る舞い

  • コンパイル中に状態やアークを動的に追加・管理します。
  • statebatch および arcbatch を介してメモリをバッチ確保し、効率的なメモリ管理を行います。
  • bos/eos フィールドにより、行頭・行末（BOL/EOL）等の特殊なマッチング条件を色として保持します。
  • parent ポインタにより、ネストされた NFA 構造をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_color.c 内の各関数（newarc, okcolors, rainbow, colorcomplement 等）が、NFA 構造体に対してアークの追加や色の割り当てを行います。
  • **呼び出し先**: 構造体内部の各フィールドは state や arc、colormap などの関連データ構造を参照します。

関連データ構造

  • struct state: NFA の状態。
  • struct arc: 状態間の遷移。
  • struct colormap: 文字コードと色のマッピング。
  • struct statebatch / struct arcbatch: メモリ確保用のバッチ構造。
  • struct vars: コンパイルエラー報告用。

失敗モードと境界条件

  • メモリ不足時には statebatch や arcbatch の確保が失敗する可能性があります。
  • nstates の管理や lastsbused/lastabused のカウンタが正しく更新されない場合、メモリリークや不正なメモリ参照が発生するリスクがあります。

保守担当者向けメモ

  • 本構造体はコンパイル時専用です。実行時のマッチングには cnfa が使用されるため、両者の変換ロジックを変更する際は整合性に注意してください。
  • bos および eos 配列は、BOS/BOL および EOS/EOL の色を保持するために使用されます。インデックス 0 と 1 の使い分けについては、正規表現エンジンの仕様（regguts.h のコメント）を参照してください。
