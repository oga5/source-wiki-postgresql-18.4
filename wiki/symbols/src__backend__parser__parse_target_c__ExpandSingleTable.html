ExpandSingleTable

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_target.c:1374

目的

指定された名前空間項目（ParseNamespaceItem）から、テーブル内の全列に対応するリスト（TargetEntry または Var）を展開する。

シグネチャまたは形状

static List *ExpandSingleTable(ParseState *pstate, ParseNamespaceItem *nsitem,
                               int sublevels_up, int location, bool make_target_entry)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState。
  • nsitem: 展開対象の ParseNamespaceItem。
  • sublevels_up: 外部クエリレベルからの階層数。
  • location: 構文解析上の位置情報。
  • make_target_entry: true の場合 TargetEntry リストを、false の場合 Var リストを返す。
  • **出力**: 展開された列の List。

前提条件と不変条件

  • make_target_entry が true の場合、expandNSItemAttrs が呼び出され、権限チェックを含む TargetEntry が生成される。
  • make_target_entry が false の場合、expandNSItemVars を使用して Var ノードのリストが生成される。
  • RTE_RELATION 型のテーブルの場合、perminfo が存在し、ACL_SELECT 権限が付与される。

アルゴリズムまたは振る舞い

1. make_target_entry が真であれば、expandNSItemAttrs を呼び出しその結果を返す。 2. 偽であれば、expandNSItemVars を呼び出し Var ノードのリストを取得する。 3. 対象が RTE_RELATION であれば、perminfo->requiredPerms に ACL_SELECT を追加する。 4. 取得した各 Var ノードに対して markVarForSelectPriv を呼び出し、列単位の読み取り権限をマークする。 5. 最終的に Var ノードのリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_target.c 内の * 展開ロジック（ExpandColumnRefStar 等）から呼び出される。
  • **呼び出し先**:
  • expandNSItemAttrs
  • expandNSItemVars
  • markVarForSelectPriv

関連データ構造

  • ParseState: 解析状態。
  • ParseNamespaceItem: 名前空間内の項目。
  • RangeTblEntry: 範囲テーブルエントリ。
  • RTEPermissionInfo: 権限情報。
  • Var: 変数ノード。

失敗モードと境界条件

  • テーブルが 0 列の場合でも、markVarForSelectPriv 等の処理を通じて適切な権限チェックが行われるよう設計されている。
  • 結合（Join）対象の nsitem の場合、権限チェックは既に範囲テーブル追加時に行われているため、本関数内ではスキップされる。

保守担当者向けメモ

  • 本関数は SELECT * や table.* の展開時に使用される。
  • make_target_entry フラグにより、TargetEntry（SELECT 句用）と Var（式評価用）のどちらを生成するかを切り替えるため、修正時は両方のパスを確認すること。
