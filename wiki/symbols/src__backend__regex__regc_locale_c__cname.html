cname

定義

  • 種別: struct
  • 定義箇所: src/backend/regex/regc_locale.c:55

目的

正規表現における文字名（例: [[:NUL:]]）と、それに対応する文字コードをマッピングするためのデータ構造を定義します。

シグネチャまたは形状

struct cname {
    const char *name;
    const char  code;
};

入力と出力

  • 入力: 文字名を表す文字列ポインタ (name) および対応する文字コード (code)。
  • 出力: 本構造体の配列 cnames[] として定義され、正規表現コンパイル時の文字名解決に使用されます。

前提条件と不変条件

  • cnames[] 配列は、正規表現エンジンが文字名を解決する際の参照テーブルとして機能します。
  • 構造体内の code は char 型であり、単一の文字コードを保持します。

アルゴリズムまたは振る舞い

正規表現コンパイル中に文字名が指定された際、この構造体を用いて name 文字列を検索し、対応する code を取得することで、正規表現内の特殊な文字指定を解決します。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソース情報からは、本構造体を利用する具体的な関数呼び出しの証拠は得られていません。
  • 呼び出し先: 本構造体はデータ定義であり、関数を呼び出しません。
  • 注意: ツールが検出した pg_constraint.c 等の cname は、変数名としての一致であり、本構造体定義とは無関係です。

関連データ構造

  • cnames[]: 本構造体を用いた静的配列。
  • element(): 本構造体で定義されたマッピングを利用して文字名から chr 型への変換を行う関数。

失敗モードと境界条件

  • 該当する文字名が cnames[] 配列内に存在しない場合、文字名解決は失敗します。
  • code が char 型であるため、マルチバイト文字や拡張文字コードの表現には適していません。

保守担当者向けメモ

  • 本構造体は regc_locale.c 内で定義されており、regcomp.c からインクルードされるコンテキストで利用されます。
  • 構造体の定義を変更する場合は、element() 関数による検索ロジックとの整合性に注意してください。
