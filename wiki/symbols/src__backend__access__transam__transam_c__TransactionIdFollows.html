TransactionIdFollows

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:313

目的

2つのトランザクションID（XID）を比較し、id1 が id2 よりも論理的に後続（大きい）かどうかを判定します。

シグネチャまたは形状

bool TransactionIdFollows(TransactionId id1, TransactionId id2)

入力と出力

  • **入力**:
  • id1: 比較対象のトランザクションID。
  • id2: 比較基準となるトランザクションID。
  • **出力**:
  • bool: id1 が id2 より後続であれば true、そうでなければ false を返します。

前提条件と不変条件

  • XIDは32ビットの循環値であるため、通常の比較演算子（>）はラップアラウンド発生時に誤動作します。本関数はラップアラウンドを考慮した論理比較を行います。
  • id1 または id2 が「正規の（Normal）」XIDではない場合、単純な比較演算子による判定にフォールバックします。

アルゴリズムまたは振る舞い

1. id1 および id2 が TransactionIdIsNormal を満たすか確認します。 2. いずれかが正規でない場合、id1 > id2 の結果を返します。 3. 両者が正規の場合、差分 diff = (int32)(id1 - id2) を計算します。 4. diff > 0 であれば true を返します。これにより、XIDの循環を考慮した前後関係が判定されます。

関連データ構造

  • TransactionId: 32ビットのトランザクション識別子。

失敗モードと境界条件

  • XIDのラップアラウンド境界付近では、int32 へのキャストによる差分計算が正しく機能するように設計されています。
  • 正規ではないXID（BootstrapTransactionId や FrozenTransactionId 等）が入力された場合、循環を考慮しない単純比較が行われます。

保守担当者向けメモ

  • XIDの比較には必ず本関数または関連する TransactionIdPrecedes 等の専用関数を使用してください。直接的な比較演算子（> や <）を正規のXIDに対して使用することは避けてください。
