validate_kvpair

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/auth-oauth.c:326

目的

RFC 7628 で規定される OAUTHBEARER メッセージ内のキーと値のペアが、プロトコル仕様に準拠しているかを検証する。

シグネチャまたは形状

static void validate_kvpair(const char *key, const char *val)

入力と出力

  • **入力**:
  • key: 検証対象のキー文字列。
  • val: 検証対象の値文字列。
  • **出力**: なし（戻り値なし）。検証失敗時は ereport(ERROR, ...) を通じて例外を発生させる。

前提条件と不変条件

  • key は空であってはならない。
  • key はアルファベット（A-Z, a-z）のみで構成される必要がある。
  • val は RFC 7628 Sec 3.1 に基づき、VCHAR（可視文字）、スペース、水平タブ、CR、LF のいずれかのみで構成される必要がある。

アルゴリズムまたは振る舞い

1. **キーの検証**:

  • key が空文字列でないことを確認する。
  • strspn を使用し、key がアルファベットのみで構成されているかを確認する。

    • 2. **値の検証**:

  • val の各文字を走査する。
  • 文字が 0x21 から 0x7E の範囲（VCHAR）にあるか、またはスペース、タブ、CR、LF のいずれかであれば継続する。
  • 上記以外が含まれる場合は ERRCODE_PROTOCOL_VIOLATION でエラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-oauth.c 内のメッセージ解析処理（parse_kvpairs_for_auth 等）から呼び出される。
  • **呼び出し先**: ereport マクロ（PostgreSQLの標準エラー報告機構）。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **プロトコル違反**: キーが空、キーに非アルファベットが含まれる、または値に許可されていない制御文字が含まれる場合、ERRCODE_PROTOCOL_VIOLATION を伴うエラーが発生する。

保守担当者向けメモ

  • 本関数は RFC 7628 の構文規則を厳密に適用する。仕様変更や拡張が必要な場合は、key_allowed_set や値の走査ループを修正すること。
  • val の検証において、セキュリティ上の理由からトークン内容がログに出力されないよう注意が払われている。修正時はこの方針を維持すること。
