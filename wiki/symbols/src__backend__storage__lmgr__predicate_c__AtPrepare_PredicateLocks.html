AtPrepare_PredicateLocks

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/predicate.c:4789

目的

2フェーズコミット（2PC）の準備フェーズにおいて、現在のトランザクションが保持している述語ロック（Predicate Lock）の状態を永続化可能な形式で記録します。

シグネチャまたは形状

void AtPrepare_PredicateLocks(void)

入力と出力

  • **入力**: なし（現在のバックエンドの MySerializableXact を参照）。
  • **出力**: なし。副作用として RegisterTwoPhaseRecord を通じて共有メモリ上の2フェーズ状態ファイルにレコードを書き込みます。

前提条件と不変条件

  • MySerializableXact が有効であること。
  • 呼び出し時に並列ワーカーが存在しないこと（!IsParallelWorker() && !ParallelContextActive()）。
  • 述語ロックのリストを走査するため、SerializablePredicateListLock を共有モードで取得する必要があります。

アルゴリズムまたは振る舞い

1. MySerializableXact が InvalidSerializableXact の場合は直ちに終了します。 2. トランザクション自体の情報（xmin や flags）を TWOPHASEPREDICATERECORD_XACT 型のレコードとして登録します。 3. SerializablePredicateListLock を取得し、現在のトランザクションが保持する述語ロックリスト（sxact->predicateLocks）を反復処理します。 4. 各述語ロックについて、その対象（target）を TWOPHASEPREDICATERECORD_LOCK 型のレコードとして登録します。 5. 最後にロックを解放します。

  • 注: 競合リストは準備後にも追加される可能性があるため、状態ファイルには保存せず、リカバリ時に保守的な仮定を置く設計となっています。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c:2671
  • **呼び出し先**: RegisterTwoPhaseRecord, LWLockAcquire, LWLockRelease

関連データ構造

  • SERIALIZABLEXACT: 直列化可能トランザクションの状態管理構造体。
  • TwoPhasePredicateRecord: 2PC 用の記録形式。
  • PREDICATELOCK: 述語ロックの内部表現。

失敗モードと境界条件

  • MySerializableXact が無効な場合は何もしません。
  • 並列クエリ実行中（ParallelContextActive() が真）に呼び出された場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は、ローカルのハッシュテーブルではなく、sxact に直接紐付いたリストを走査します。これはローカルテーブルの不正確さを回避するためです。
  • 競合情報の保存を省略しているため、リカバリ時の挙動が保守的（安全側）であることを確認してください。
