pgp_cfb_encrypt

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-cfb.c:251

目的

PGP暗号化処理において、CFB（Cipher Feedback）モードを用いたデータの暗号化を行う。

シグネチャまたは形状

int pgp_cfb_encrypt(PGP_CFB *ctx, const uint8 *data, int len, uint8 *dst);

入力と出力

  • ctx: 暗号化コンテキスト（PGP_CFB 構造体）。
  • data: 暗号化対象の入力データバッファ。
  • len: 入力データの長さ。
  • dst: 暗号化結果を格納する出力バッファ。
  • 戻り値: cfb_process の実行結果（通常は成功時に 0、失敗時にエラーコード）。

前提条件と不変条件

  • ctx は pgp_cfb_create によって適切に初期化されている必要がある。
  • ctx->resync フラグの状態に基づき、適切な暗号化ロジックが選択される。

アルゴリズムまたは振る舞い

1. ctx->resync フラグを確認し、mix_encrypt_resync または mix_encrypt_normal のいずれかの関数ポインタを選択する。 2. 選択した関数を mix 引数として cfb_process を呼び出し、実際の暗号化処理（ブロック暗号の適用とXOR演算）を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-encrypt.c 内の各箇所（セッションキーの暗号化やデータストリームの処理）。
  • **呼び出し先**:
  • cfb_process（内部処理の実行）。
  • mix_encrypt_normal または mix_encrypt_resync（暗号化ロジックの切り替え）。

関連データ構造

  • PGP_CFB: 暗号化コンテキストを保持する構造体。

失敗モードと境界条件

  • cfb_process が内部で返すエラーコードに依存する。
  • 入力データ長 len が負の値である場合や、バッファの境界条件については cfb_process の実装に準拠する。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実際のロジックは cfb_process および mix 関数群に集約されている。
  • 古い PGP 仕様に対応するための resync モードの切り替えがここで行われるため、再同期ロジックの変更が必要な場合は mix_encrypt_resync を参照すること。
