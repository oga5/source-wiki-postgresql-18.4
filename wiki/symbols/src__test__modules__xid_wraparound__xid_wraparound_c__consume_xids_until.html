consume_xids_until

定義

  • 種別: function
  • 定義箇所: src/test/modules/xid_wraparound/xid_wraparound.c:52

目的

指定されたターゲットトランザクションID（XID）に到達するまで、システム上のXIDを強制的に消費するSQL呼び出し可能な関数です。

シグネチャまたは形状

Datum consume_xids_until(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_FULLTRANSACTIONID(0) を介してターゲットとなる FullTransactionId を受け取ります。
  • **出力**: 最終的に到達した FullTransactionId を PG_RETURN_FULLTRANSACTIONID を介して返します。

前提条件と不変条件

  • 入力された targetxid は FullTransactionIdIsNormal を満たす（正常なXIDである）必要があります。満たさない場合はエラーとなります。

アルゴリズムまたは振る舞い

1. 引数として受け取った targetxid が正常な範囲内であるか検証します。 2. consume_xids_common 関数を呼び出し、ターゲットXIDに到達するまでXIDの消費処理を実行します。 3. 消費処理の結果として得られた最終的なXIDを呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロによる登録経由）。
  • **呼び出し先**: consume_xids_common（XID消費ロジックの中核）。

関連データ構造

  • FullTransactionId: 64bitトランザクションIDを保持するデータ構造。

失敗モードと境界条件

  • targetxid が正常なXIDではない場合、elog(ERROR, ...) により処理が中断されます。
  • consume_xids_common 内部のロジックにより、SLRU境界やXID周回に関連する制約が適用されます。

保守担当者向けメモ

  • 本関数はテスト専用です。本番環境での実行はXID周回を早め、データベース停止のリスクを伴うため避けてください。
  • consume_xids_common の実装変更が本関数の挙動に直接影響します。
