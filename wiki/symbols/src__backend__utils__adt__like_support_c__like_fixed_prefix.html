like_fixed_prefix

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_support.c:988

目的

LIKE パターンから、インデックススキャンに利用可能な固定接頭辞（prefix）を抽出します。

シグネチャまたは形状

Pattern_Prefix_Status like_fixed_prefix(Const *patt_const, bool case_insensitive, Oid collation,
                                        Const **prefix_const, Selectivity *rest_selec);

入力と出力

  • **入力**:
  • patt_const: LIKE パターンを含む Const ノード（TEXT または BYTEA 型）。
  • case_insensitive: 大文字小文字を区別しない検索かどうか。
  • collation: 照合順序。
  • **出力**:
  • prefix_const: 抽出された固定接頭辞を格納する Const ノードへのポインタ。
  • rest_selec: 接頭辞を除いた残りのパターン部分に対する選択率（NULL でない場合）。
  • **戻り値**: Pattern_Prefix_Status（Pattern_Prefix_Exact, Pattern_Prefix_Partial, Pattern_Prefix_None のいずれか）。

前提条件と不変条件

  • patt_const は TEXTOID または BYTEAOID である必要があります。
  • case_insensitive が true の場合、BYTEA 型はサポートされずエラーとなります。
  • case_insensitive が true で collation が無効な場合、エラーとなります。

アルゴリズムまたは振る舞い

1. パターン文字列を走査し、ワイルドカード（%, _）またはエスケープ文字（\）が出現するまでを固定接頭辞として抽出します。 2. case_insensitive が有効な場合、アルファベット文字が出現した時点で走査を停止します。 3. 抽出した接頭辞を prefix_const として構築します。 4. rest_selec が指定されている場合、like_selectivity を呼び出して残りのパターンに対する選択率を計算します。 5. パターン全体が固定文字列であれば Pattern_Prefix_Exact を、一部のみが固定であれば Pattern_Prefix_Partial を、それ以外は Pattern_Prefix_None を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/like_support.c 内の textlike_support および texticlike_support から呼び出されます。
  • **呼び出し先**:
  • string_to_const / string_to_bytea_const
  • like_selectivity
  • pg_newlocale_from_collation
  • pattern_char_isalpha

関連データ構造

  • Pattern_Prefix_Status: 抽出結果の状態を示す列挙型。
  • Const: PostgreSQL の定数ノード。

失敗モードと境界条件

  • BYTEA 型に対する大文字小文字区別なし検索の要求はエラーとなります。
  • 照合順序が解決できない場合、エラーとなります。
  • パターンが空の場合、Pattern_Prefix_Exact が返されます。

保守担当者向けメモ

  • 本関数は LIKE 演算子のプランナサポートの一部です。非Cロケールにおける pattern_char_isalpha の挙動はロケール依存であるため、変更時は注意してください。
