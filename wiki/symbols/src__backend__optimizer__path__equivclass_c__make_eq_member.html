make_eq_member

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/equivclass.c:591

目的

等価クラス（EquivalenceClass）の新しいメンバー（EquivalenceMember）を生成し、初期化する。

シグネチャまたは形状

static EquivalenceMember *make_eq_member(EquivalenceClass *ec, Expr *expr, Relids relids, JoinDomain *jdomain, EquivalenceMember *parent, Oid datatype)

入力と出力

  • **入力**:
  • ec: メンバーを追加する対象の等価クラス。
  • expr: メンバーとなる式。
  • relids: 式に含まれるリレーションの集合。
  • jdomain: 結合ドメイン情報。
  • parent: 親メンバー（子メンバーの場合）。
  • datatype: 式のデータ型。
  • **出力**:
  • 初期化済みの EquivalenceMember 構造体へのポインタ。

前提条件と不変条件

  • parent が指定されている場合、そのメンバーは子メンバーとして扱われる。
  • relids が空の場合、その式は擬似定数（pseudoconstant）とみなされ、ec->ec_has_const が true に設定される。

アルゴリズムまたは振る舞い

1. makeNode(EquivalenceMember) を使用してメモリを確保する。 2. 引数で渡された expr, relids, datatype, jdomain, parent を構造体の各フィールドに代入する。 3. em_is_child を parent != NULL に基づいて設定する。 4. relids が空であるかを確認し、空であれば em_is_const を true に設定し、親クラス ec の ec_has_const フラグを立てる。 5. 初期化された EquivalenceMember を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/equivclass.c 内の関数（add_eq_member や add_child_eq_member など）から呼び出される。
  • **呼び出し先**:
  • makeNode（PostgreSQL のノード生成マクロ）。
  • bms_is_empty（ビットマップセットの空判定）。

関連データ構造

  • EquivalenceClass: 等価関係を保持する親構造体。
  • EquivalenceMember: 本関数が生成するノード構造体。
  • Relids: リレーション識別子の集合。
  • JoinDomain: 結合ドメイン情報。

失敗モードと境界条件

  • relids が空の場合、parent が存在しないことが前提となる（Assert(!parent)）。

保守担当者向けメモ

  • 本関数は EquivalenceMember の生成に特化しており、クラスへの実際の追加処理は呼び出し側で行う必要がある。
  • 擬似定数の判定ロジックは、process_equivalence() からの呼び出しを想定しており、揮発性や集約関数等のチェックは呼び出し前に行われている前提である。
