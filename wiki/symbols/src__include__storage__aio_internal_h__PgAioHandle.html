PgAioHandle

定義

  • 種別: struct
  • 定義箇所: src/include/storage/aio_internal.h:102

目的

PostgreSQL の非同期 I/O (AIO) サブシステムにおいて、個別の I/O 操作のライフサイクル、状態、および実行結果を追跡・管理するための主要な構造体です。

シグネチャまたは形状

uint8 型のフィールドを多用し、メモリ効率を考慮した構造になっています。状態管理用の dlist_node や完了待ち合わせ用の ConditionVariable、I/O 実行に必要なデータ構造（PgAioOpData, PgAioTargetData）を含みます。

入力と出力

  • **入力**: pgaio_io_set_handle_data_*() を介して関連データが設定されます。
  • **出力**: result フィールドに I/O 操作の生の結果が格納され、distilled_result には共有コールバックの結果が保持されます。

前提条件と不変条件

  • すべての状態更新は必ず pgaio_io_update_state() を経由する必要があります。
  • state に応じて、ハンドルは特定のリスト（PgAioBackend のキューや発行元の in_flight リスト）に登録されます。
  • generation フィールドは、ハンドルが再利用されるたびにインクリメントされます。

アルゴリズムまたは振る舞い

1. **初期化**: pgaio_io_start_* により HANDED_OUT から DEFINED 状態へ移行。 2. **ステージング**: pgaio_io_stage を経て STAGED 状態となり、バックエンドのステージング配列に格納。 3. **実行**: submit コールバックにより SUBMITTED 状態へ移行。 4. **完了**: pgaio_io_process_completion により COMPLETED_LOCAL へ遷移し、最終的に IDLE 状態へ戻り再利用可能となります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/aio.c 内の pgaio_io_update_state, pgaio_io_reclaim, pgaio_io_resowner_register, pgaio_io_wait 等から参照されます。
  • **呼び出し先**: 構造体内部で ConditionVariable の待機や IoMethodOps の関数ポインタを介した操作が行われます。

関連データ構造

  • PgAioHandleState: ハンドルの状態遷移を定義。
  • PgAioBackend: バックエンドごとの I/O 管理。
  • PgAioOpData, PgAioTargetData: I/O 実行および対象識別用データ。
  • dlist_node: リスト管理用。
  • ConditionVariable: 完了待ち合わせ用。

失敗モードと境界条件

  • SUBMITTED 状態のハンドルを待機する場合、IoMethodOps->wait_one() を通じて進捗を確認する必要があります。
  • generation の不一致は、ハンドルが再利用されたことを示唆します。

保守担当者向けメモ

  • PgAioHandle 内のフィールドはメモリ節約のため uint8 を多用しています。
  • owner_procno はポインタ演算等で最適化可能な可能性があります。
  • iovec_off は現在 PgAioCtl 内の配列インデックスとして使用されています。
