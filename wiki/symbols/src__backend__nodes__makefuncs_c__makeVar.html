makeVar

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/makefuncs.c:65

目的

Var ノードを生成し、必須フィールドを引数で初期化しつつ、その他のフィールドに安全なデフォルト値を設定することで、クエリツリー構築時のコード重複とエラーを低減する。

シグネチャまたは形状

Var *makeVar(int varno, AttrNumber varattno, Oid vartype, int32 vartypmod, Oid varcollid, Index varlevelsup)

入力と出力

  • **入力**:
  • varno: 変数の参照先（Range Table Entryのインデックス）。
  • varattno: 属性番号。
  • vartype: 型のOID。
  • vartypmod: 型の修飾子。
  • varcollid: 照合順序（Collation）のOID。
  • varlevelsup: 上位クエリレベルへの参照数。
  • **出力**:
  • 初期化済みの Var 構造体へのポインタ。

前提条件と不変条件

  • makeNode(Var) を通じて CurrentMemoryContext にメモリが確保されるため、呼び出し側は適切なメモリコンテキストがアクティブであることを保証する必要がある。

アルゴリズムまたは振る舞い

1. makeNode(Var) を呼び出し、メモリを確保する。 2. 引数で渡された varno, varattno, vartype, vartypmod, varcollid, varlevelsup を構造体の対応するフィールドに代入する。 3. 以下のフィールドをデフォルト値で初期化する:

  • varreturningtype: VAR_RETURNING_DEFAULT
  • varnullingrels: NULL
  • varnosyn: varno と同じ値
  • varattnosyn: varattno と同じ値
  • location: -1（不明）

    • 4. 構築された Var ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/commands/tablecmds.c
  • src/backend/nodes/makefuncs.c（内部の他の関数から呼び出される）
  • **呼び出し先**:
  • makeNode（マクロ）

関連データ構造

  • Var（src/include/nodes/parsenodes.h で定義）

失敗モードと境界条件

  • メモリ不足時には makeNode 経由でエラーが発生する可能性がある。
  • 特殊な varreturningtype や varnullingrels 等が必要な場合は、本関数の戻り値に対して呼び出し側で手動設定を行う必要がある。

保守担当者向けメモ

  • 本関数は「一般的なケース」を想定している。特殊なフィールド設定が必要な場合でも、本関数をベースにフィールドを上書きする設計となっている。
  • varnosyn や varattnosyn は引数として提供されていないため、これらを変更する必要がある場合は呼び出し後に直接代入すること。
