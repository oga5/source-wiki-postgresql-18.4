genericPickSplit

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistsplit.c:343

目的

ユーザー定義の picksplit 関数が不適切な分割（片側に全タプルを寄せる等）を行った際のフォールバックとして、GiSTインデックスのページ分割を強制的に実行する。

シグネチャまたは形状

static void genericPickSplit(GISTSTATE *giststate, GistEntryVector *entryvec, GIST_SPLITVEC *v, int attno)

入力と出力

  • **入力**:
  • giststate: GiSTインデックスの状態情報。
  • entryvec: 分割対象となるエントリのベクトル。
  • v: 分割結果を格納する GIST_SPLITVEC 構造体。
  • attno: 分割に使用する列インデックス。
  • **出力**:
  • v の各フィールド（spl_left, spl_right, spl_nleft, spl_nright, spl_ldatum, spl_rdatum）が更新される。

前提条件と不変条件

  • entryvec は有効なエントリを含んでいること。
  • attno に対応するユニオン関数が giststate 内に存在すること。
  • 分割後の各ページについて、指定された列のユニオンキーが再計算される。

アルゴリズムまたは振る舞い

1. entryvec 内のエントリを単純に半分に分割し、v->spl_left と v->spl_right に割り当てる。 2. 分割された各グループに対して、giststate->unionFn[attno] を呼び出し、各側のユニオンキー（spl_ldatum, spl_rdatum）を計算する。 3. 計算には GistEntryVector を一時的に構築して使用する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistsplit.c 内の分割ロジックから呼び出される（詳細はソースコードの文脈に依存）。
  • **呼び出し先**: FunctionCall2Coll を介して giststate->unionFn[attno] を実行する。

関連データ構造

  • GISTSTATE: インデックスの操作状態を保持。
  • GistEntryVector: 分割対象のエントリ群。
  • GIST_SPLITVEC: 分割結果（左右のオフセットとユニオンキー）を保持。

失敗モードと境界条件

  • エントリ数が極端に少ない場合でも、単純な半分分割を行う。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は、ユーザー定義の picksplit が失敗した際の安全装置として機能する。
  • 常に単純な二等分を行うため、インデックスの品質は最適化されない可能性がある。
  • attno を用いて特定の列に対するユニオン計算を行うため、マルチカラムインデックスの整合性を維持する役割を持つ。
