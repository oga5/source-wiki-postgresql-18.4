RemoveExtensionById

定義

  • 種別: function
  • 定義箇所: src/backend/commands/extension.c:2279

目的

指定された OID を持つ拡張機能を pg_extension カタログから削除します。

シグネチャまたは形状

void RemoveExtensionById(Oid extId)

入力と出力

  • **入力**: extId (削除対象の拡張機能の OID)
  • **出力**: なし（成功時はカタログからタプルを削除、失敗時はエラーを投げる）

前提条件と不変条件

  • 削除対象の拡張機能が現在挿入（インストール/更新）中ではないこと。extId が CurrentExtensionObject と一致する場合、依存関係の不整合を防ぐため削除は拒否されます。

アルゴリズムまたは振る舞い

1. CurrentExtensionObject との照合を行い、現在処理中の拡張機能であればエラーを発生させます。 2. pg_extension リレーションを RowExclusiveLock で開きます。 3. ExtensionOidIndexId インデックスを使用して、対象の OID に一致するタプルを検索します。 4. タプルが見つかった場合、CatalogTupleDelete を呼び出してカタログから削除します。 5. スキャンを終了し、リレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c:1428 からの呼び出しが確認されています。
  • **呼び出し先**: table_open, ScanKeyInit, systable_beginscan, systable_getnext, CatalogTupleDelete, systable_endscan, table_close 等のカタログ操作関数。

関連データ構造

  • pg_extension: 拡張機能のメタデータを保持するシステムカタログ。
  • Relation, SysScanDesc, HeapTuple: カタログ操作に使用される PostgreSQL の内部構造体。

失敗モードと境界条件

  • **エラー**: extId == CurrentExtensionObject の場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー終了します。
  • **境界条件**: 対象の OID が存在しない場合、タプルは取得されず、何も削除されずに正常終了します。

保守担当者向けメモ

  • 本関数は DROP EXTENSION コマンドの実行時だけでなく、依存関係の再帰的な削除処理からも呼び出される可能性があるため、排他制御のチェックが重要です。
  • 拡張機能のスクリプト実行中に発生する依存関係の不整合を回避するため、このレベルでのチェックが必須となっています。
