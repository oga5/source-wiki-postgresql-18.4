spg_kd_choose

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgkdtreeproc.c:53

目的

SP-GiSTインデックスへのデータ挿入時、k-d 木の構造に基づき、入力された Point をどのノードへ振り分けるかを決定します。

シグネチャまたは形状

Datum spg_kd_choose(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgChooseIn 構造体を受け取ります。これには挿入対象の datum（Point 型）、現在のツリーレベル、プレフィックス値が含まれます。
  • **出力**: spgChooseOut 構造体を介して、選択されたノード番号 (nodeN)、レベルの増分 (levelAdd)、および次のステップへ渡すデータ (restDatum) を返します。

前提条件と不変条件

  • in->allTheSame は false であること（k-d 木では中央値分割が前提のため）。
  • in->hasPrefix は true であること。
  • in->nNodes は 2 であること（k-d 木は二分木であるため）。

アルゴリズムまたは振る舞い

1. in->prefixDatum から分割基準となる座標値を取得します。 2. 現在のレベル (in->level) に基づき、x 座標または y 座標のいずれを比較対象とするかを選択します。 3. getSide 関数を呼び出し、入力ポイントがプレフィックス値に対してどちらの側に位置するかを判定します。 4. 判定結果に基づき、out->result.matchNode.nodeN に 0 または 1 を設定します。 5. levelAdd を 1 に設定し、挿入対象のポイントをそのまま restDatum として引き継ぎます。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTの挿入処理プロセス（具体的な呼び出し元関数名はソースコード内に明示されていません）。
  • **呼び出し先**: getSide 関数を呼び出します。

関連データ構造

  • spgChooseIn: 挿入時の入力パラメータを保持。
  • spgChooseOut: 選択結果を格納。
  • Point: 挿入対象の幾何データ型。

失敗モードと境界条件

  • allTheSame が true の場合、エラーを発生させます。
  • nNodes が 2 でない場合、アサーション違反となります。

保守担当者向けメモ

  • 本関数は k-d 木の二分分割ロジックに特化しています。level % 2 を用いて次元を交互に切り替えるため、インデックスの深さと次元の対応関係を変更する際は注意してください。
