hashcharextended

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:53

目的

char 型の値に対して、ハッシュ結合用のシード値を用いた拡張ハッシュ計算を行う。

シグネチャまたは形状

Datum hashcharextended(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_CHAR(0): ハッシュ対象の char 型値。
  • PG_GETARG_INT64(1): ハッシュ計算に使用するシード値（int64）。
  • **出力**:
  • Datum: 計算されたハッシュ値。

前提条件と不変条件

  • 入力された char 型の値は int32 にキャストされて処理される。
  • ハッシュ結合のパフォーマンス向上のため、シード値を受け取る必要がある。

アルゴリズムまたは振る舞い

1. 第1引数から char 値を取得し、int32 型に変換する。 2. 第2引数から int64 型のシード値を取得する。 3. hash_uint32_extended 関数を呼び出し、変換後の値とシード値からハッシュ値を生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/jsonb_util.c:1394 （DirectFunctionCall2 を介した呼び出し）
  • **呼び出し先**:
  • hash_uint32_extended

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • PG_FUNCTION_ARGS: SQL関数呼び出しの引数コンテキスト。

失敗モードと境界条件

  • 特筆すべき失敗モードは定義されていない。入力値の範囲は char 型の仕様に依存する。

保守担当者向けメモ

  • 本関数は hashchar の拡張版であり、ハッシュ結合の最適化を目的としている。新しい型を追加する際は、同様に ...extended バージョンを実装すること。
