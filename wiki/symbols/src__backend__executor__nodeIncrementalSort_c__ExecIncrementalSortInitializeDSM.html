ExecIncrementalSortInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIncrementalSort.c:1193

目的

パラレルクエリ実行時において、インクリメンタルソートノードの統計情報を共有メモリ（DSM）上に初期化し、各ワーカープロセスが統計を記録できるようにする。

シグネチャまたは形状

void ExecIncrementalSortInitializeDSM(IncrementalSortState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の IncrementalSortState 構造体。
  • pcxt: 並列実行コンテキスト (ParallelContext)。
  • **出力**: なし（副作用として node->shared_info が設定され、共有メモリ領域が確保される）。

前提条件と不変条件

  • node->ss.ps.instrument が有効であり、かつ pcxt->nworkers > 0 である場合にのみ処理が実行される。
  • 確保される共有メモリ領域は SharedIncrementalSortInfo 構造体と、ワーカー数分の IncrementalSortInfo 配列を保持するのに十分なサイズであること。

アルゴリズムまたは振る舞い

1. インストゥルメンテーションが不要、またはワーカーが存在しない場合は即座にリターンする。 2. 必要な共有メモリサイズを計算する（SharedIncrementalSortInfo のヘッダーサイズ + ワーカー数 × IncrementalSortInfo のサイズ）。 3. shm_toc_allocate を使用して共有メモリ領域を割り当てる。 4. 割り当てた領域を memset でゼロクリアする。 5. shared_info->num_workers にワーカー数を設定する。 6. shm_toc_insert を呼び出し、プランノード ID をキーとして共有メモリ領域を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:525
  • **呼び出し先**:
  • shm_toc_allocate (共有メモリ領域の確保)
  • shm_toc_insert (共有メモリ領域の登録)
  • memset (メモリの初期化)

関連データ構造

  • IncrementalSortState: インクリメンタルソートの実行状態を保持する構造体。
  • SharedIncrementalSortInfo: 共有メモリ上に配置される統計情報のヘッダー。
  • IncrementalSortInfo: 各ワーカーごとの統計情報。
  • ParallelContext: 並列実行のコンテキスト情報。

失敗モードと境界条件

  • pcxt->nworkers が 0 の場合、共有メモリの確保は行われない。
  • node->ss.ps.instrument が NULL の場合、統計情報の初期化はスキップされる。

保守担当者向けメモ

  • 本関数は ExecIncrementalSort が並列環境で動作する際の準備フェーズで呼び出される。
  • 共有メモリのレイアウト変更や、統計情報の項目追加を行う場合は、size の計算ロジックと SharedIncrementalSortInfo の定義を同期させる必要がある。
