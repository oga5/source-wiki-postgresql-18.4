get_default_oid_from_partdesc

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partdesc.c:500

目的

パーティション記述子（PartitionDesc）から、デフォルトパーティションに対応するリレーションの OID を取得します。

シグネチャまたは形状

Oid get_default_oid_from_partdesc(PartitionDesc partdesc)

入力と出力

  • **入力**: partdesc (PartitionDesc) - パーティション構造を保持する記述子。
  • **出力**: Oid - デフォルトパーティションが存在する場合はその OID、存在しない場合は InvalidOid。

前提条件と不変条件

  • partdesc が NULL の場合、関数は安全に InvalidOid を返します。
  • partdesc->boundinfo が存在し、かつ partition_bound_has_default が真を返す場合にのみ、有効な OID が返されます。

アルゴリズムまたは振る舞い

1. partdesc が有効であり、かつ boundinfo を保持しているか確認します。 2. partition_bound_has_default(partdesc->boundinfo) を呼び出し、デフォルトパーティションが定義されているか判定します。 3. 定義されている場合、boundinfo->default_index をインデックスとして partdesc->oids 配列から該当する OID を取得して返します。 4. 条件を満たさない場合は InvalidOid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c
  • src/backend/catalog/partition.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • partition_bound_has_default (マクロまたは関数)

関連データ構造

  • PartitionDesc: パーティションの OID 配列および境界情報を保持する構造体。
  • PartitionBoundInfo: パーティションの境界定義とデフォルトパーティションのインデックスを保持する構造体。

失敗モードと境界条件

  • partdesc が NULL の場合、またはデフォルトパーティションが定義されていない場合、InvalidOid を返します。これはエラーではなく、デフォルトパーティションが存在しない正常な状態として扱われます。

保守担当者向けメモ

  • 本関数は PartitionDesc の内部構造（特に boundinfo と oids の関係）に強く依存しています。PartitionDesc の定義が変更された場合は、インデックス参照の整合性を確認してください。
