SetSharedSecurityLabel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/seclabel.c:328

目的

共有システムカタログ pg_shseclabel におけるデータベースオブジェクトのセキュリティラベルを永続化、更新、または削除する。

シグネチャまたは形状

static void SetSharedSecurityLabel(const ObjectAddress *object, const char *provider, const char *label)

入力と出力

  • object: 操作対象のオブジェクトアドレス（objectId, classId）。
  • provider: ラベルプロバイダー名。
  • label: 設定するラベル文字列。NULL の場合は既存のラベルを削除する。
  • 出力: なし（副作用として pg_shseclabel を更新）。

前提条件と不変条件

  • 呼び出し元で適切な権限チェックが完了していること。
  • pg_shseclabel テーブルに対する RowExclusiveLock が内部で取得される。

アルゴリズムまたは振る舞い

1. 指定された object と provider をキーとして pg_shseclabel をスキャンする。 2. 既存のタプルが見つかった場合:

  • label が NULL ならば CatalogTupleDelete で削除する。
  • label が指定されていれば heap_modify_tuple で内容を更新し CatalogTupleUpdate を実行する。

    • 3. 既存のタプルが見つからず、かつ label が NULL でない場合:

  • heap_form_tuple で新規タプルを作成し CatalogTupleInsert を実行する。

    • 4. 最後に table_close を呼び出し、ロックを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: SetSecurityLabel（src/backend/commands/seclabel.c 内のヘルパー関数として参照）。
  • 呼び出し先: table_open, systable_beginscan, systable_getnext, CatalogTupleDelete, heap_modify_tuple, CatalogTupleUpdate, systable_endscan, heap_form_tuple, CatalogTupleInsert, heap_freetuple, table_close。

関連データ構造

  • ObjectAddress: 操作対象オブジェクトの識別子。
  • pg_shseclabel: 共有セキュリティラベルを保持するシステムカタログ。

失敗モードと境界条件

  • 指定されたオブジェクトやプロバイダーの組み合わせがカタログに存在しない場合、新規挿入が行われる。
  • label が NULL で、かつ該当する既存ラベルが存在しない場合は何も行われない。

保守担当者向けメモ

  • 本関数は共有オブジェクト専用である。通常オブジェクトのラベル操作には SetSecurityLabel を使用すること。
  • pg_shseclabel のインデックス SharedSecLabelObjectIndexId を使用して検索を行うため、カタログ定義変更時は注意が必要。
