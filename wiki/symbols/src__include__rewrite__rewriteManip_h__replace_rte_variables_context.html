replace_rte_variables_context

定義

  • 種別: struct
  • 定義箇所: src/include/rewrite/rewriteManip.h:23

目的

クエリ書き換え（Query Rewriter）プロセスにおいて、特定の範囲テーブルエントリ（RTE）に関連する変数（Var ノード）を置換する際の走査状態およびコールバック情報を保持するために使用されます。

シグネチャまたは形状

typedef struct replace_rte_variables_context replace_rte_variables_context;

※構造体の内部メンバ定義は本ソース抜粋には含まれていません。

入力と出力

  • 本構造体は、ツリー走査関数（walker/mutator）の引数として渡されるコンテキストオブジェクトとして機能します。
  • 走査中のネストレベル（sublevels_up）や、置換対象となる RTE の識別情報、および置換ルールを保持します。

前提条件と不変条件

  • rewriteManip.c 内の走査関数において、ツリーの再帰的な探索とノードの置換を行うための状態管理に使用されます。
  • サブクエリを跨ぐ際の sublevels_up の調整が、このコンテキストを通じて一貫して行われる必要があります。

アルゴリズムまたは振る舞い

  • expression_tree_walker や expression_tree_mutator と組み合わせて使用されます。
  • ツリーを探索し、特定の varno に合致する Var ノードを検出した際、コンテキスト内の設定に基づいてノードの置換や属性番号の調整を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepjointree.c
  • src/backend/rewrite/rewriteManip.c
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、関数ではありません。具体的な操作は rewriteManip.c 内の関数群によって実行されます。

関連データ構造

  • Var: 置換対象となるノード。
  • RangeTblEntry: 変数が参照する範囲テーブルエントリ。
  • ReplaceVarsNoMatchOption: 変数置換時に一致するものが見つからなかった場合の挙動を制御する列挙型。

失敗モードと境界条件

  • 構造体内部のメンバ定義が不足しているため、特定の失敗モード（例：NULL ポインタ参照や無効な varno 指定時の挙動）については、実装ファイル rewriteManip.c の詳細な確認が必要です。

保守担当者向けメモ

  • 本構造体は rewriteManip.h で前方宣言されており、詳細は rewriteManip.c に依存します。
  • 新しいノード型を追加した際は、このコンテキストを利用する mutator 関数が適切にノードを処理できるよう更新してください。
  • 破壊的変更を行う関数で使用される場合があるため、呼び出し元でツリーのコピーが必要か否かを慎重に判断してください。
