gbtreekey_in

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_gist.c:25

目的

GiST 索引の仕様上要求される入力関数（in）のスタブとして機能し、ユーザーが本モジュールの内部型を直接操作することを防ぐためのエラー報告を行います。

シグネチャまたは形状

Datum gbtreekey_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（PostgreSQL の関数呼び出し規約に従う引数群）。内部で PG_GETARG_OID(1) を使用して型情報を取得します。
  • **出力**: なし。関数は常にエラーを発生させて終了します。

前提条件と不変条件

  • 本関数は、GiST 索引のフレームワークから呼び出されることを想定していますが、本モジュールで定義される型はユーザーによる直接的な入力（in 操作）をサポートしていません。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID(1) を呼び出し、入力された型情報の OID を取得します。 2. ereport を使用して ERRCODE_FEATURE_NOT_SUPPORTED エラーを発生させます。 3. エラーメッセージには、format_type_extended を用いて、受け入れ不可能な型名を表示します。 4. PG_RETURN_VOID() を呼び出しますが、実際にはエラーにより制御が戻ることはありません。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。具体的な実行時の呼び出し元は、GiST 索引の型定義に関連するシステムカタログ操作やユーザークエリに依存しますが、本ソースコード内での直接的な呼び出し元は確認できません。
  • **呼び出し先**: ereport（エラー報告）、format_type_extended（型名の文字列化）。

関連データ構造

  • Datum: PostgreSQL の汎用データ型。
  • PG_FUNCTION_ARGS: 関数引数コンテキスト。

失敗モードと境界条件

  • 本関数は常に失敗（エラー発生）するように設計されています。
  • ユーザーがこの関数を直接呼び出そうとした場合、またはシステムがこの型の入力関数を必要とする操作を行った場合に、ERRCODE_FEATURE_NOT_SUPPORTED が発生します。

保守担当者向けメモ

  • 本関数はスタブであり、意図的にエラーを投げることで、ユーザーが GiST 内部のキー型を直接操作することを禁止しています。ロジックの変更は不要です。
