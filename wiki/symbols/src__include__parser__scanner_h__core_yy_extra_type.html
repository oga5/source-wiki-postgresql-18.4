core_yy_extra_type

定義

  • 種別: struct
  • 定義箇所: src/include/parser/scanner.h:66

目的

PostgreSQLの字句解析器（スキャナ）における内部状態を保持するための構造体です。解析対象のバッファ、キーワードリスト、GUC設定、およびリテラル解析中の一時的な状態を管理します。

シグネチャまたは形状

typedef struct core_yy_extra_type {
    char *scanbuf;
    Size scanbuflen;
    const ScanKeywordList *keywordlist;
    const uint16 *keyword_tokens;
    int backslash_quote;
    bool escape_string_warning;
    bool standard_conforming_strings;
    char *literalbuf;
    int literallen;
    int literalalloc;
    int state_before_str_stop;
    int xcdepth;
    char *dolqstart;
    YYLTYPE save_yylloc;
    int32 utf16_first_part;
    bool warn_on_first_escape;
    bool saw_non_ascii;
} core_yy_extra_type;

入力と出力

  • **入力**: scanner_init() を通じて、解析対象文字列やGUC設定値が本構造体に初期化されます。
  • **出力**: 字句解析の過程で literalbuf に文字列が蓄積され、yylval 等を通じてパーサーへトークン情報が渡されます。

前提条件と不変条件

  • literalbuf は必ずしもヌル終端ではありませんが、literallen の位置にヌル文字を格納するための領域が常に確保されています。
  • 呼び出し側のパーサーは、本構造体を先頭に含む独自の構造体を定義し、拡張することが可能です。

アルゴリズムまたは振る舞い

  • **バッファ管理**: scanbuf を保持し、現在解析中のトークン（yytext）のオフセットを計算します。
  • **リテラル構築**: 複数ルールにまたがるリテラル解析時、startlit() で初期化し addlit() で文字を追加します。
  • **状態保持**: コメントのネスト深度（xcdepth）や、ドル引用符の開始文字列（dolqstart）、Unicodeエスケープのサロゲートペア情報などを保持します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/gramparse.h
  • src/pl/plpgsql/src/pl_scanner.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • **呼び出し先**:
  • scanner_init()（初期化時に使用）
  • startlit() / addlit()（リテラル処理時に使用）

関連データ構造

  • ScanKeywordList: キーワードリストの定義。
  • YYLTYPE: トークンの位置情報（バイトオフセット）。

失敗モードと境界条件

  • literalbuf は palloc された動的バッファであり、メモリ不足時にはエラーが発生する可能性があります。
  • literalbuf のヌル終端は必要に応じて明示的に行われるため、バッファを直接参照する際は literallen を使用する必要があります。

保守担当者向けメモ

  • 本構造体は scanner_init() 後に呼び出し側でGUC設定を上書き可能です。
  • 位置情報は行/列番号ではなく、ソース文字列の先頭からのバイトオフセットとして管理されています。
