log_locus_callback

定義

  • 種別: function
  • 定義箇所: src/bin/psql/startup.c:98

目的

ログ出力時に現在処理中のファイル名と行番号を特定するためのコールバック関数です。

シグネチャまたは形状

static void log_locus_callback(const char **filename, uint64 *lineno)

入力と出力

  • **入力**:
  • filename: ファイル名を格納するポインタへのポインタ。
  • lineno: 行番号を格納するポインタ。
  • **出力**: なし（引数経由で値を書き戻す）。

前提条件と不変条件

  • グローバル変数 pset が初期化されている必要があります。
  • pset.inputfile が設定されている場合、その値と pset.lineno が出力されます。

アルゴリズムまたは振る舞い

1. pset.inputfile が存在するか確認する。 2. 存在する場合、*filename に pset.inputfile を、*lineno に pset.lineno を代入する。 3. 存在しない場合、*filename を NULL に、*lineno を 0 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/logging.c 内のログ出力処理から、登録されたコールバックとして間接的に呼び出されます。
  • **呼び出し先**: 自身の内部で呼び出す関数はありません。
  • **登録**: src/bin/psql/startup.c 内で pg_logging_set_locus_callback(log_locus_callback) を介して登録されます。

関連データ構造

  • pset (PsqlSettings): psql の実行状態を保持するグローバル構造体。

失敗モードと境界条件

  • pset.inputfile が NULL の場合、ファイル名は NULL、行番号は 0 として報告されます。

保守担当者向けメモ

  • 本関数は libpq 側のログ出力機構と psql の内部状態を橋渡しする役割を持ちます。
  • pset 構造体の定義変更に伴い、参照するフィールドに変更が必要になる可能性があります。
