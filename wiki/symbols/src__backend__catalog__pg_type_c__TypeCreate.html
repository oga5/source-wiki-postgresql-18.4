TypeCreate

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_type.c:194

目的

システムカタログ pg_type に新しいデータ型定義を永続化します。型の属性（サイズ、アライメント、I/O関数等）を検証し、カタログへの行挿入または更新を行います。

シグネチャまたは形状

ObjectAddress TypeCreate(Oid newTypeOid, const char *typeName, Oid typeNamespace, Oid relationOid, char relationKind, Oid ownerId, int16 internalSize, char typeType, char typeCategory, bool typePreferred, char typDelim, Oid inputProcedure, Oid outputProcedure, Oid receiveProcedure, Oid sendProcedure, Oid typmodinProcedure, Oid typmodoutProcedure, Oid analyzeProcedure, Oid subscriptProcedure, Oid elementType, bool isImplicitArray, Oid arrayType, Oid baseType, const char *defaultTypeValue, char *defaultTypeBin, bool passedByValue, char alignment, char storage, int32 typeMod, int32 typNDims, bool typeNotNull, Oid typeCollation)

入力と出力

  • **入力**: 型の識別子、名前、所有者、サイズ、アライメント、I/O関数OID、デフォルト値、および型属性フラグ等。
  • **出力**: 作成された型の ObjectAddress を返します。

前提条件と不変条件

  • internalSize は 1 以上の正数、-1 (varlena)、または -2 (cstring) である必要があります。
  • passedByValue が真の場合、internalSize と alignment の組み合わせがプラットフォームの制約（fetch_att 等）に適合している必要があります。
  • storage が TYPSTORAGE_PLAIN 以外の場合、internalSize は -1 (varlena) である必要があります。

アルゴリズムまたは振る舞い

1. 引数の妥当性検証（サイズ、アライメント、ストレージ制約）。 2. isDependentType の判定（暗黙的な配列型、マルチレンジ型、または非複合型の関係行型であるか）。 3. pg_type カタログ行を構築するためのデータ配列（values, nulls, replaces）の初期化。 4. 提供された引数に基づき、pg_type の各属性（typname, typlen, typbyval, typtype 等）をセット。 5. （ソース抜粋以降の処理）カタログへの挿入/更新実行および GenerateTypeDependencies による依存関係の記録。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/heap.c および src/backend/commands/typecmds.c から呼び出されます。
  • **呼び出し先**: カタログ操作関数（heap_form_tuple 等）および依存関係管理関数（GenerateTypeDependencies）が呼び出されます。

関連データ構造

  • pg_type: 型定義を保持するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトの識別情報。
  • Natts_pg_type: pg_type の属性数。

失敗モードと境界条件

  • 不正なサイズやアライメントが指定された場合、ERRCODE_INVALID_OBJECT_DEFINITION でエラーを報告します。
  • 固定長型に対して PLAIN 以外のストレージが指定された場合、エラーを報告します。

保守担当者向けメモ

  • 本関数は TypeShellMake で作成されたシェル型を完全な型定義へ昇格させる際にも使用されます。
  • isDependentType の判定ロジックは GenerateTypeDependencies と密接に関連しており、変更時は両方の整合性を確認してください。
