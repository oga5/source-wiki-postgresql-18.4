pgstat_end_function_usage

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_function.c:145

目的

関数実行終了時の統計情報を計算し、累積統計データ構造を更新する。

シグネチャまたは形状

void pgstat_end_function_usage(PgStat_FunctionCallUsage *fcu, bool finalize)

入力と出力

  • fcu: 関数実行開始時に pgstat_init_function_usage で初期化された計測用コンテキスト。
  • finalize: true の場合、関数の呼び出し回数（numcalls）をインクリメントする。

前提条件と不変条件

  • fcu->fs が NULL の場合、本関数は何も行わず即座に復帰する。
  • 再帰呼び出しによる二重計上を避けるため、バックエンド全体の累積時間（total_func_time）との差分を用いて自己時間を算出する。

アルゴリズムまたは振る舞い

1. 現在時刻を取得し、関数開始時からの経過時間（total）を算出する。 2. total_func_time の現在値と開始時の値（fcu->save_total）の差分を求め、経過時間から差し引くことで、当該関数固有の実行時間（self）を算出する。 3. 算出した self をバックエンド全体の累積時間 total_func_time に加算する。 4. 関数実行前の累積時間（fcu->save_f_total_time）に経過時間を加算し、関数の総実行時間として更新する。 5. finalize が true ならば numcalls をインクリメントする。 6. 算出した total および self を fcu->fs（PgStat_FunctionCounts）に反映させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/event_trigger.c
  • src/backend/commands/functioncmds.c
  • src/backend/commands/trigger.c
  • src/backend/executor/execExprInterp.c
  • src/backend/executor/execSRF.c
  • src/backend/utils/fmgr/fmgr.c
  • src/pl/tcl/pltcl.c
  • **呼び出し先**:
  • 外部関数としての呼び出し先情報は提供されていません。

関連データ構造

  • PgStat_FunctionCallUsage: 実行計測のコンテキストを保持する構造体。
  • PgStat_FunctionCounts: 関数の統計情報（呼び出し回数、総実行時間、自己実行時間）を保持する構造体。

失敗モードと境界条件

  • fcu->fs が NULL の場合、統計収集はスキップされる。
  • 再帰呼び出しが発生した場合でも、total_func_time の差分計算により、親関数の自己時間が正しく算出されるよう設計されている。

保守担当者向けメモ

  • 本関数は pgstat_init_function_usage と対で使用される。
  • 統計収集のオーバーヘッドを避けるため、GUCパラメータ pgstat_track_functions による制御が前提となっている。
