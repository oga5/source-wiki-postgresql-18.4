DropRelationFiles

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/md.c:1600

目的

指定された複数のリレーションに関連する物理ファイルを一括で削除する。

シグネチャまたは形状

void DropRelationFiles(RelFileLocator *delrels, int ndelrels, bool isRedo)

入力と出力

  • delrels: 削除対象のリレーションを特定する RelFileLocator 配列。
  • ndelrels: delrels 配列の要素数。
  • isRedo: リカバリ（REDO）処理中かどうかを示すフラグ。

前提条件と不変条件

  • isRedo が true の場合、各リレーションの全フォーク（MAX_FORKNUM まで）に対して XLogDropRelation が呼び出される。
  • 処理対象のリレーションは smgropen を介して一時的にオープンされ、処理後に smgrclose でクローズされる。

アルゴリズムまたは振る舞い

1. ndelrels 分の SMgrRelation 構造体をメモリ確保する。 2. 各リレーションに対して smgropen を実行し、isRedo が真であれば XLogDropRelation を呼び出してログ記録を行う。 3. smgrdounlinkall を呼び出し、物理的なファイル削除を実行する。 4. 最後にすべての SMgrRelation を smgrclose で閉じ、確保したメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c:1607
  • src/backend/access/transam/xact.c:6239
  • src/backend/access/transam/xact.c:6350
  • **呼び出し先**:
  • smgropen
  • XLogDropRelation (条件付き)
  • smgrdounlinkall
  • smgrclose

関連データ構造

  • RelFileLocator: リレーションの物理的な位置を特定する識別子。
  • SMgrRelation: ストレージマネージャが管理するリレーションのハンドル。

失敗モードと境界条件

  • ndelrels が 0 の場合、ループは実行されず smgrdounlinkall が空のリストで呼び出される。
  • ファイルシステムレベルでの削除失敗時の挙動は、呼び出される smgrdounlinkall の実装に依存する。

保守担当者向けメモ

  • 本関数は一括削除を行うため、個別の削除処理よりも効率的である。
  • isRedo フラグの扱いは、WAL リカバリ時の整合性維持に直結するため注意が必要。
