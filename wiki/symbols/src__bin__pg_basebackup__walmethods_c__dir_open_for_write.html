dir_open_for_write

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/walmethods.c:116

目的

WAL（Write-Ahead Log）データをディレクトリ形式で保存するためのファイルを開き、必要に応じて圧縮設定や事前パディング、同期処理を適用する。

シグネチャまたは形状

static Walfile *dir_open_for_write(WalWriteMethod *wwmethod, const char *pathname, const char *temp_suffix, size_t pad_to_size)

入力と出力

  • **入力**:
  • wwmethod: 抽象基底構造体（DirectoryMethodData としてキャストして使用）。
  • pathname: 書き込み対象のファイルパス。
  • temp_suffix: 一時ファイル用のサフィックス。
  • pad_to_size: 非圧縮時にファイルをゼロ埋めするサイズ。
  • **出力**: 成功時は初期化済みの Walfile 構造体（DirectoryMethodFile）、失敗時は NULL を返す。

前提条件と不変条件

  • wwmethod は DirectoryMethodData 型として安全にキャスト可能であること。
  • 圧縮アルゴリズムが PG_COMPRESSION_GZIP または PG_COMPRESSION_LZ4 の場合、対応するライブラリが有効であること。
  • sync が有効な場合、ファイルおよび親ディレクトリの fsync が成功すること。

アルゴリズムまたは振る舞い

1. dir_get_file_name を用いてファイル名を生成し、ベースディレクトリと結合する。 2. open() でファイルを作成・オープンする。 3. 圧縮設定（Gzip/LZ4）に応じて、圧縮コンテキストの初期化とヘッダー書き込みを行う。 4. 非圧縮かつ pad_to_size が指定されている場合、pg_pwrite_zeros でファイルを事前パディングし、ファイルポインタを先頭に戻す。 5. wwmethod->sync が真の場合、fsync_fname および fsync_parent_path を実行して永続性を保証する。 6. DirectoryMethodFile 構造体を割り当て、ファイルディスクリプタや圧縮コンテキストを格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/walmethods.c 内の WalWriteMethodOps 構造体（open_for_write コールバックとして登録）。
  • **呼び出し先**: dir_get_file_name, open, pg_pwrite_zeros, lseek, fsync_fname, fsync_parent_path, pg_malloc0 等。

関連データ構造

  • WalWriteMethod: 抽象基底。
  • DirectoryMethodData: ディレクトリ保存用メソッドのデータ構造。
  • DirectoryMethodFile: 本関数が生成する、ファイルハンドルと圧縮コンテキストを保持する構造体。

失敗モードと境界条件

  • ファイルオープン失敗時、errno を wwmethod->lasterrno に設定し NULL を返す。
  • 圧縮コンテキストの初期化やヘッダー書き込み失敗時、リソースを解放し NULL を返す。
  • sync 失敗時、開いたファイルをクローズし NULL を返す。
  • ディスク容量不足等のエラーは errno または ENOSPC を通じて報告される。

保守担当者向けメモ

  • gzflush() は fsync を行わないため、dir_sync() メソッドでの同期処理が必須である。
  • LZ4 圧縮時は LZ4F_compressBegin でヘッダーを書き込む必要がある。
  • win32pwrite.c の挙動に依存するため、パディング後の lseek による位置リセットは重要である。
