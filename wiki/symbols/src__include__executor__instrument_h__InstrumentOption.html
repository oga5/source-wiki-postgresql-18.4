InstrumentOption

定義

  • 種別: enum
  • 定義箇所: src/include/executor/instrument.h:60

目的

クエリ実行時の統計情報収集において、どの項目（タイマー、バッファ、WAL、行数）を計測対象とするかを指定するためのビットマスクを提供します。

シグネチャまたは形状

typedef enum InstrumentOption
{
	INSTRUMENT_TIMER = 1 << 0,
	INSTRUMENT_BUFFERS = 1 << 1,
	INSTRUMENT_ROWS = 1 << 2,
	INSTRUMENT_WAL = 1 << 3,
	INSTRUMENT_ALL = PG_INT32_MAX
} InstrumentOption;

入力と出力

  • 入力: ビットフラグの組み合わせ。
  • 出力: 実行エンジンが計測のオーバーヘッドを制御するための設定値として利用されます。

前提条件と不変条件

  • 各フラグはビット演算による組み合わせが可能です。
  • INSTRUMENT_ALL はすべての計測項目を有効にします。

アルゴリズムまたは振る舞い

  • 実行エンジンは、このオプションに基づいて Instrumentation 構造体の各フィールドを更新するかどうかを判断します。
  • 計測はクエリ実行のオーバーヘッドとなるため、必要な項目のみを有効にすることが推奨されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • src/include/executor/execdesc.h:44 (instrument_options)
  • src/include/nodes/execnodes.h:715 (es_instrument)
  • 呼び出し先: 情報なし。

関連データ構造

  • Instrumentation: 統計情報を保持するメイン構造体。
  • BufferUsage: バッファI/O統計。
  • WalUsage: WAL生成量統計。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • 本シンボルはビットマスクとして使用されるため、新しい計測項目を追加する場合はビット位置の重複に注意してください。
  • INSTRUMENT_TIMER を指定すると、行数カウントも自動的に有効になる仕様です。
