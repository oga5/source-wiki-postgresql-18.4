ExplainResultDesc

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:253

目的

EXPLAIN コマンドの出力結果を保持するための TupleDesc（タプル記述子）を生成します。出力形式（TEXT, XML, JSON）に応じて、適切なデータ型を定義します。

シグネチャまたは形状

TupleDesc ExplainResultDesc(ExplainStmt *stmt)

入力と出力

  • **入力**: ExplainStmt *stmt（EXPLAIN コマンドの解析済み構文木）。
  • **出力**: TupleDesc（単一カラム「QUERY PLAN」を持つタプル記述子）。

前提条件と不変条件

  • stmt->options に含まれる format オプションを走査します。
  • 複数の format オプションが存在する場合、最後に指定された値が採用されます。
  • 戻り値の TupleDesc は、QUERY PLAN という名称の単一カラムを持ちます。

アルゴリズムまたは振る舞い

1. stmt->options リストを反復処理し、format オプションを探します。 2. format の値が xml なら XMLOID、json なら JSONOID を選択し、それ以外はデフォルトの TEXTOID を使用します。 3. CreateTemplateTupleDesc(1) で 1 カラム分の領域を確保します。 4. TupleDescInitEntry を呼び出し、カラム名を "QUERY PLAN"、型を上記で決定した OID に設定して初期化します。 5. 生成した TupleDesc を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c:238
  • src/backend/tcop/utility.c:2116
  • **呼び出し先**:
  • defGetString
  • CreateTemplateTupleDesc
  • TupleDescInitEntry

関連データ構造

  • ExplainStmt: EXPLAIN コマンドの構文木構造体。
  • TupleDesc: PostgreSQL のタプル構造を定義する記述子。
  • DefElem: オプション指定を保持する構造体。

失敗モードと境界条件

  • format オプションが指定されていない場合、デフォルトで TEXTOID が選択されます。
  • format に未知の文字列が指定された場合、TEXTOID にフォールバックします。

保守担当者向けメモ

  • 本関数は EXPLAIN の出力形式を決定する重要な箇所です。新しい出力形式を追加する場合は、この関数の switch 的な条件分岐を拡張する必要があります。
  • TupleDesc のメモリ管理は呼び出し元に依存するため、メモリリークが発生しないよう注意してください。
