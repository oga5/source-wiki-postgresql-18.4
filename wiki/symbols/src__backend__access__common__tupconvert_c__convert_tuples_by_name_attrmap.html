convert_tuples_by_name_attrmap

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tupconvert.c:123

目的

提供された属性マッピング情報（AttrMap）に基づき、異なる物理構造を持つタプル間での変換を行うための TupleConversionMap 構造体を初期化・構築する。

シグネチャまたは形状

TupleConversionMap *convert_tuples_by_name_attrmap(TupleDesc indesc, TupleDesc outdesc, AttrMap *attrMap)

入力と出力

  • **入力**:
  • indesc: 入力タプルの記述子（TupleDesc）。
  • outdesc: 出力タプルの記述子（TupleDesc）。
  • attrMap: 属性の対応関係を定義する AttrMap 構造体。
  • **出力**:
  • 変換に必要なワークスペースとメタデータを保持する TupleConversionMap 構造体へのポインタ。

前提条件と不変条件

  • attrMap は NULL であってはならない（Assert により検証）。
  • 戻り値の構造体には、変換実行時に使用される Datum 配列と bool 配列（outvalues, outisnull, invalues, inisnull）が事前に確保される。
  • invalues[0] および inisnull[0] は、NULL 値を表現するためのエントリとして初期化される。

アルゴリズムまたは振る舞い

1. TupleConversionMap 構造体用のメモリを確保する。 2. 入力および出力の TupleDesc と attrMap を構造体に格納する。 3. 出力タプルの属性数に基づき、変換結果を格納するための outvalues および outisnull 配列を確保する。 4. 入力タプルの属性数に基づき、変換作業用の invalues および inisnull 配列を確保する。 5. invalues[0] に (Datum) 0 を、inisnull[0] に true を設定し、NULL 参照用のインデックスを準備する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tupconvert.c:116（convert_tuples_by_name 内）
  • src/backend/executor/execUtils.c:1351
  • **呼び出し先**:
  • palloc（メモリ確保）

関連データ構造

  • TupleConversionMap: 変換に必要な情報を保持するメイン構造体。
  • TupleDesc: タプルの構造定義。
  • AttrMap: 属性の対応関係を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • attrMap が NULL の場合、アサーション違反によりプロセスが停止する。

保守担当者向けメモ

  • 本関数は変換マップの「構築」のみを担当する。実際のタプル変換処理は execute_attr_map_tuple や execute_attr_map_slot が行う。
  • 確保されたメモリは free_conversion_map を使用して解放する必要がある。
  • TupleDesc 自体の生存期間管理は本関数では行わないため、呼び出し側で適切に管理すること。
