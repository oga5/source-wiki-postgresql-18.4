pg_getaddrinfo_all

定義

  • 種別: function
  • 定義箇所: src/common/ip.c:52

目的

ホスト名とサービス名からネットワークアドレス情報を取得するためのプロトコル非依存の抽象化関数です。標準の getaddrinfo をラップし、IPv4/IPv6 に加えて Unix ドメインソケットを透過的に扱います。

シグネチャまたは形状

int pg_getaddrinfo_all(const char *hostname, const char *servname,
                       const struct addrinfo *hintp, struct addrinfo **result);

入力と出力

  • **入力**:
  • hostname: 解決対象のホスト名（NULL または空文字は getaddrinfo の仕様に従い処理される）。
  • servname: サービス名またはポート番号。
  • hintp: 解決の条件を指定する addrinfo 構造体。
  • **出力**:
  • result: 成功時に確保された addrinfo 構造体のリストへのポインタが格納される。
  • 戻り値: 成功時に 0、失敗時にエラーコードを返す。

前提条件と不変条件

  • *result は関数内で必ず NULL に初期化されます。
  • hintp->ai_family が AF_UNIX の場合、Unix ドメインソケット用の解決処理が行われます。

アルゴリズムまたは振る舞い

1. *result を NULL に初期化します。 2. hintp->ai_family が AF_UNIX であるかを確認します。

  • AF_UNIX の場合: getaddrinfo_unix を呼び出し、その結果を返して終了します。

    • 3. それ以外の場合: hostname が NULL または空文字であれば NULL を、そうでなければ hostname を引数として標準の getaddrinfo を呼び出します。 4. getaddrinfo の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth.c
  • src/backend/libpq/hba.c
  • src/backend/libpq/pqcomm.c
  • src/interfaces/libpq/fe-connect.c
  • **呼び出し先**:
  • getaddrinfo (標準ライブラリ)
  • getaddrinfo_unix (src/common/ip.c 内の内部関数)

関連データ構造

  • struct addrinfo: ネットワークアドレス情報を保持する標準構造体。

失敗モードと境界条件

  • getaddrinfo が失敗した場合、そのエラーコードがそのまま返されます。
  • getaddrinfo の実装によっては失敗時に *result がゼロクリアされない可能性があるため、本関数冒頭で明示的に NULL 代入を行っています。

保守担当者向けメモ

  • 本関数で取得した addrinfo リストを解放する際は、必ず pg_freeaddrinfo_all を使用してください。これは、Unix ドメインソケット用と標準 API 用で解放ロジックが異なるためです。
