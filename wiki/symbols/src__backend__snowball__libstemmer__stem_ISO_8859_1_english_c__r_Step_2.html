r_Step_2

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_english.c:644

目的

Porterステミングアルゴリズムの第2段階として、特定の語尾パターンをR1領域内の単語に対して置換または削除する。

シグネチャまたは形状

static int r_Step_2(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: 成功時に 1、失敗または条件不一致時に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z->c（現在のカーソル位置）が単語の末尾を指していること。
  • 処理対象の単語がR1領域（r_R1）の条件を満たしていること。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置に設定する。 2. 単語末尾の文字が特定のビットマスク条件に合致するか確認し、一致しなければ終了する。 3. find_among_b を使用し、テーブル a_5（24エントリ）に基づき後方から語尾を検索する。 4. z->bra を検索開始位置に設定し、r_R1 を呼び出して対象がR1領域内にあるか検証する。 5. 検索された語尾の種類（among_var）に応じて、以下の処理を行う：

  • ケース1〜12: slice_from_s を使用して特定の文字列に置換する。
  • ケース13: 末尾が 'l' であることを確認後、slice_from_s で置換する。
  • ケース14: slice_from_s で置換する。
  • ケース15: in_grouping_b で特定の文字グループ（g_valid_LI）を確認し、条件を満たせば slice_del で削除する。

呼び出し元と呼び出し先

  • 呼び出し元: english_ISO_8859_1_stem（直接の呼び出し元はソース内の制御フローより推測されるが、本抜粋では明示されていない）。
  • 呼び出し先: find_among_b, r_R1, slice_from_s, slice_del, in_grouping_b。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。
  • a_5: 語尾検索に使用される among 構造体のテーブル。
  • g_valid_LI: ケース15の判定に使用される文字グループ。

失敗モードと境界条件

  • 検索対象の語尾が見つからない場合、0 を返す。
  • R1領域の判定に失敗した場合、r_R1 の戻り値をそのまま返す。
  • 文字列操作（slice_from_s 等）でメモリ不足やエラーが発生した場合、負の値を返す。
  • 単語の境界（lb）を超えた操作は行われない。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによる自動生成コードであるため、直接編集せず、元の .sbl スクリプトを修正すること。
  • among_var の各ケースは、Porterアルゴリズムの特定の語尾置換ルールに対応している。
