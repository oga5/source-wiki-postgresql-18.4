plpgsql_ns_lookup_label

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_funcs.c:194

目的

指定された名前空間チェーン（PLpgSQL_nsitem のリンクリスト）を走査し、特定の名前を持つラベル項目を検索します。

シグネチャまたは形状

PLpgSQL_nsitem *plpgsql_ns_lookup_label(PLpgSQL_nsitem *ns_cur, const char *name)

入力と出力

  • **入力**:
  • ns_cur: 検索を開始する名前空間アイテムのポインタ（チェーンの末尾または現在のスコープ）。
  • name: 検索対象のラベル名（文字列）。
  • **出力**:
  • 見つかった場合は該当する PLpgSQL_nsitem へのポインタ。
  • 見つからない場合は NULL。

前提条件と不変条件

  • ns_cur は NULL であるか、有効な PLpgSQL_nsitem のリンクリストを指している必要があります。
  • 検索対象の itemtype は PLPGSQL_NSTYPE_LABEL である必要があります。

アルゴリズムまたは振る舞い

1. ns_cur が NULL になるまで、以下の処理を繰り返します。 2. 現在のアイテムの itemtype が PLPGSQL_NSTYPE_LABEL かつ、name が一致するかを strcmp で確認します。 3. 一致した場合はそのアイテムを返します。 4. 一致しない場合は、ns_cur->prev を辿って親スコープへ移動します。 5. リストの終端に達した場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報からは具体的な呼び出し元関数は特定できません。
  • **呼び出し先**: strcmp（標準ライブラリ関数）を呼び出します。

関連データ構造

  • PLpgSQL_nsitem: 名前空間チェーンを構成する構造体。itemtype および name フィールドを持ちます。

失敗モードと境界条件

  • ns_cur が NULL の場合、直ちに NULL を返します。
  • ラベル名が一致しない場合、リスト全体を走査した後に NULL を返します。

保守担当者向けメモ

  • 本関数はラベルの検索に特化しています。変数など他の名前空間アイテムを検索する場合は、plpgsql_ns_lookup 等の他の関数を使用してください。
  • plpgsql.h にて宣言されています。
