SS_finalize_plan

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/subselect.c:2367

目的

計画生成の最終段階において、実行計画ツリー内の各ノードに対するパラメータ依存関係（extParam, allParam）を確定させるための処理を行います。

シグネチャまたは形状

void SS_finalize_plan(PlannerInfo *root, Plan *plan)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo 構造体。
  • plan: 最終化対象となる Plan ノードのツリー。
  • **出力**: なし（副作用として plan ツリー内のパラメータ関連フィールドが更新されます）。

前提条件と不変条件

  • 本関数が呼び出される前に、対象となる InitPlan やサブクエリの計画は既に SS_finalize_plan による処理が完了している必要があります。
  • setrefs.c による参照解決処理の前に実行されることが想定されています。

アルゴリズムまたは振る舞い

1. 内部関数 finalize_plan を呼び出し、計画ツリー全体を再帰的に巡回します。 2. 巡回中に各ノードのパラメータ依存関係（extParam および allParam）を計算・確定させます。 3. root->outer_params を初期値として使用し、ツリー構造に沿ってパラメータ情報を伝播させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c: サブクエリの計画化およびトップレベルの計画生成時に呼び出されます。
  • **呼び出し先**:
  • finalize_plan (内部関数): 実際の再帰的な巡回とパラメータ計算を実行します。

関連データ構造

  • PlannerInfo: クエリ全体の最適化情報を保持する構造体。
  • Plan: 実行計画ツリーの各ノード。
  • Param: 実行時に評価されるパラメータノード。

失敗モードと境界条件

  • 計画ツリーが空（NULL）の場合の挙動については、finalize_plan の実装に依存します。
  • 既にパラメータが確定済みのノードに対して再実行した場合の整合性については、本関数の責務外となります。

保守担当者向けメモ

  • 本関数は setrefs.c が実行される前にパラメータ情報を確定させる重要な役割を担っています。
  • 計画ツリーの構造変更やパラメータ管理のロジックを変更する際は、finalize_plan の再帰処理が正しくパラメータを伝播しているか確認してください。
  • aggParams など、一部のフィールドは本関数の実行後に埋められることが想定されているため、呼び出し順序には注意が必要です。
