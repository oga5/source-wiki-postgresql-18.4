TheLexeme

定義

  • 種別: struct
  • 定義箇所: src/backend/tsearch/dict_thesaurus.c:39

目的

シソーラス辞書において、特定の語彙（lexeme）とそれに関連するエントリ情報を保持し、フレーズ置換ルールを管理するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    char       *lexeme;
    LexemeInfo *entries;
} TheLexeme;

入力と出力

  • lexeme: 置換対象となる単語文字列へのポインタ。
  • entries: その単語に関連付けられた LexemeInfo 構造体の配列（エントリ情報）。

前提条件と不変条件

  • DictThesaurus 構造体の一部として、メモリ上に動的に確保される配列として使用されます。
  • palloc または repalloc を介してメモリ管理が行われます。

アルゴリズムまたは振る舞い

  • シソーラス辞書のコンパイル時、addCompiledLexeme 関数によってルールが追加・拡張されます。
  • 辞書内のフレーズ探索において、cmpLexeme 関数を用いて TheLexeme 同士の比較が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: addCompiledLexeme、thesaurus_init（辞書初期化時のメモリ確保）、cmpLexeme（比較処理）。
  • **呼び出し先**: LexemeInfo 構造体（entries フィールドを通じて参照）。

関連データ構造

  • LexemeInfo: TheLexeme が保持する詳細なエントリ情報。
  • DictThesaurus: TheLexeme の配列を保持するメインの辞書構造体。

失敗モードと境界条件

  • メモリ不足時に palloc / repalloc が失敗する可能性があります。
  • repalloc による配列の再配置時に、既存のポインタが無効化される可能性があるため、参照の整合性に注意が必要です。

保守担当者向けメモ

  • TheLexeme はシソーラスの置換ルールを検索可能にするためのインデックスの一部です。
  • cmpLexeme は qsort や二分探索の比較関数として利用されるため、比較ロジックを変更する際は一貫性を維持してください。
