ConnectionUp

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:335

目的

現在のデータベース接続が有効（CONNECTION_BAD 状態ではない）かどうかを判定する。

シグネチャまたは形状

static bool ConnectionUp(void)

入力と出力

  • 入力: なし（グローバル変数 pset.db を参照）
  • 出力: bool 型。接続が確立されている場合は true、それ以外（切断状態など）は false を返す。

前提条件と不変条件

  • pset.db が有効な PGconn オブジェクトを指していること。

アルゴリズムまたは振る舞い

1. libpq の PQstatus(pset.db) を呼び出す。 2. 戻り値が CONNECTION_BAD と等しくないことを確認し、その結果を論理値として返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/psql/common.c 内の複数箇所（CheckConnection 等の接続状態確認処理）。
  • src/bin/psql/common.c:1251（トランザクション状態の判定ロジック）。
  • 呼び出し先:
  • libpq の PQstatus 関数。

関連データ構造

  • pset: psql のグローバル設定を保持する構造体。pset.db は現在のデータベース接続を保持する PGconn 型のポインタ。

失敗モードと境界条件

  • pset.db が NULL の場合、PQstatus の仕様に依存する（通常はセグメンテーション違反等の未定義動作を引き起こす可能性がある）。

保守担当者向けメモ

  • 本関数は static であり、common.c 内でのみ使用される。
  • 接続状態の判定には PQstatus を直接使用しているため、libpq の接続状態定義に変更があった場合は本関数の挙動も影響を受ける。
