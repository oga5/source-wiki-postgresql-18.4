cash_in

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/cash.c:172

目的

通貨型（money）の文字列表現を、内部表現である int64（Cash 型）へ変換します。

シグネチャまたは形状

Datum cash_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して受け取った通貨文字列。
  • **出力**: Cash 型（int64）の内部表現。エラー時は ereport により処理を中断します。

前提条件と不変条件

  • 現在のロケール設定（lc_monetary）に強く依存します。
  • PGLC_localeconv() から取得した frac_digits、mon_decimal_point、currency_symbol 等の情報を基にパースを行います。
  • 内部的には負の数として値を蓄積し、最後に符号を適用する設計です。

アルゴリズムまたは振る舞い

1. **ロケール情報の取得**: PGLC_localeconv() を呼び出し、小数点や通貨記号などのフォーマット情報を取得します。 2. **前処理**: 文字列の先頭にある空白や通貨記号をスキップします。 3. **符号の判定**: 先頭のマイナス記号、括弧 ()、またはプラス記号を検出し、符号を決定します。 4. **数値のパース**: 文字列を走査し、数字を int64 に累積します。

  • 小数点（dsymbol）を検出した場合は、それ以降の桁数をカウントします。
  • 桁区切り文字（ssymbol）は無視します。
  • オーバーフローが発生した場合はエラーを報告します。

    • 5. **後処理**: 小数点以下の桁数がロケール設定に満たない場合は、値をスケーリング（10のべき乗を乗算）して調整します。最後に符号を適用します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルでの型変換時に呼び出されます（具体的な呼び出し元関数名はソース内に明示されていません）。
  • **呼び出し先**: PGLC_localeconv()（ロケール情報取得）、pg_mul_s64_overflow / pg_sub_s64_overflow（オーバーフローチェック付き算術演算）、ereport（エラー報告）。

関連データ構造

  • Cash: int64 の別名。通貨型の内部表現。
  • struct lconv: ロケール固有のフォーマット情報（frac_digits 等）を保持する構造体。

失敗モードと境界条件

  • **オーバーフロー**: 数値が int64 の範囲を超える場合、ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE でエラーとなります。
  • **ロケール依存**: ロケール設定により、同じ文字列でも解釈される数値が異なる可能性があります。
  • **パース不備**: 括弧による負数表現のバランスチェックなど、一部のヒューリスティックな処理には制限があります。

保守担当者向けメモ

  • frac_digits の値が CHAR_MAX になる環境（Cロケール等）を考慮し、範囲チェックを行っています。
  • 負の数のオーバーフローを避けるため、計算は負の方向で行い、最後に符号を反転させる手法をとっています。
  • 補助関数 append_num_word 等と同様、ロケール設定の変更が結果に直結するため、テスト時はロケール環境に注意してください。
