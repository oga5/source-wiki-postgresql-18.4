namesend

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/name.c:105

目的

PostgreSQL の name 型データを、クライアントとの通信に適したバイナリ形式（bytea）に変換します。

シグネチャまたは形状

Datum namesend(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して name 型のデータを受け取ります。
  • **出力**: bytea 型の Datum を返します。

前提条件と不変条件

  • 入力された name 型の値は、NAMEDATALEN バイト以内の固定長文字列です。
  • pq_begintypsend および pq_endtypsend を使用したバイナリプロトコル変換の規約に従う必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_NAME(0) を使用して、入力された name 型の値を s に取得します。 2. pq_begintypsend を呼び出し、バイナリ送信用のバッファ buf を初期化します。 3. NameStr(*s) を用いて name 型から文字列を取り出し、strlen で長さを算出します。 4. pq_sendtext を呼び出し、文字列データをバッファに書き込みます。 5. pq_endtypsend を呼び出してバッファを bytea 形式に変換し、その結果を PG_RETURN_BYTEA_P で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の型システム（バイナリ送信時）。
  • **呼び出し先**: pq_begintypsend, pq_sendtext, pq_endtypsend（libpq/pqformat.h で定義されたバイナリプロトコル関数）。

関連データ構造

  • Name: name 型の内部表現。
  • StringInfoData: バイナリ送信用のバッファ構造体。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • メモリ不足が発生した場合、palloc 関連の内部関数によりエラーがスローされます。
  • 入力値が NULL の場合、PG_GETARG_NAME のマクロ定義に従った挙動となります。

保守担当者向けメモ

  • 本関数は name 型のバイナリ送信用インターフェースです。プロトコル定義に変更がない限り、ロジックの変更は不要です。
  • NameStr マクロによる文字列アクセスと strlen による長さ計算は、name 型がヌル終端文字列であることを前提としています。
