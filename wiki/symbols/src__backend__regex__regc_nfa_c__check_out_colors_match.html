check_out_colors_match

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_nfa.c:3414

目的

指定された状態 s から出る遷移において、色 co1 を持つ遷移先集合と、色 co2 を持つ遷移先集合が完全に一致するかを判定します。

シグネチャまたは形状

bool check_out_colors_match(struct state *s, color co1, color co2)

入力と出力

  • **入力**:
  • s: 調査対象の state 構造体。
  • co1, co2: 比較対象となる color 型の値。
  • **出力**:
  • true: co1 で到達可能な状態集合と co2 で到達可能な状態集合が一致する場合。
  • false: 一致しない場合。

前提条件と不変条件

  • NFA 内に重複する遷移（同じ色で同じ状態へ向かう複数の弧）が存在しないことを前提としています。
  • 処理の過程で state->tmp フィールドを一時的に使用するため、呼び出し前後でこのフィールドが NULL であることが期待されます。

アルゴリズムまたは振る舞い

線形時間で以下の手順を実行します。 1. s から出る遷移を走査し、co1 を持つ遷移先の tmp フィールドに自身（a->to）をマークします。 2. 再度 s から出る遷移を走査し、co2 を持つ遷移先を確認します。

  • tmp がマークされていれば、対応する co1 遷移が存在するためマークを解除します。
  • tmp がマークされていなければ、co2 側にのみ存在する遷移とみなし false を返します。

    • 3. 最後に s から出る遷移を再度走査し、まだマークが残っている（co1 側にのみ存在する）遷移がないか確認します。残っていれば false を返します。 4. 処理終了時にすべての tmp フィールドを NULL にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_nfa.c:3161, 3162 (NFA の事前状態チェック)
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。state 構造体の outchain を辿るポインタ操作を行います。

関連データ構造

  • struct state: 状態を保持する構造体。tmp フィールドを一時的な作業領域として利用します。
  • struct arc: 遷移を保持する構造体。co（色）および to（遷移先状態）を含みます。

失敗モードと境界条件

  • s->outs が NULL の場合、両方の色が空集合とみなされ true を返します。
  • 遷移先状態の tmp フィールドが呼び出し時に NULL でない場合、assert によりエラーとなります。

保守担当者向けメモ

  • check_in_colors_match は本関数と対になるアルゴリズムであり、入力遷移（ins）に対して同様の比較を行います。
  • 処理の効率化のため tmp フィールドを破壊的に利用するため、並列実行や再帰的な呼び出しを行う場合は注意が必要です。
