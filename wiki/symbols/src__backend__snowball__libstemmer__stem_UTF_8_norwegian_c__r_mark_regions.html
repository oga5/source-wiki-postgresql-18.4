r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_norwegian.c:133

目的

ノルウェー語のステミング処理において、接尾辞の削除範囲を制限するための領域（R1, R2等）を計算し、SN_env 構造体の整数配列 z->I に記録します。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（ステミング実行環境）
  • 出力: 成功時に 1、失敗時に 0 を返します。
  • 副作用: z->I[0] および z->I[1] に計算された境界位置が格納されます。

前提条件と不変条件

  • z は有効な SN_env 構造体である必要があります。
  • UTF-8 エンコーディングを前提としており、文字単位ではなくバイト単位でのカーソル操作が行われます。

アルゴリズムまたは振る舞い

1. z->I[1] を文字列の末尾 z->l で初期化します。 2. 文字列の先頭から3文字分スキップした位置を z->I[0] に記録します。 3. out_grouping_U を使用して母音（g_v）以外の文字を検索し、その後の母音の開始位置を z->I[1] に設定します。 4. z->I[1] が z->I[0] より小さい場合、z->I[1] を z->I[0] に更新します（領域の最小範囲を保証）。

呼び出し元と呼び出し先

  • 呼び出し元: norwegian_UTF_8_stem（ファイル概要より、ステミング実行の初期段階で呼び出されます）。
  • 呼び出し先: skip_utf8, out_grouping_U, in_grouping_U（いずれも header.h で定義されたランタイム関数）。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。z->I 配列（整数値格納用）および z->c（カーソル）、z->l（文字列長）を使用します。

失敗モードと境界条件

  • skip_utf8 が失敗した場合、または out_grouping_U / in_grouping_U が条件を満たさない場合に 0 を返して終了します。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードです。直接編集せず、元の .sbl スクリプトを修正してください。
  • g_v は母音グループを定義する外部変数です。
  • UTF-8 処理のため、バイト単位のオフセット操作が正しく行われているか注意してください。
