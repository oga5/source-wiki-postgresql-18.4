close_destination_dir

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_receivewal.c:251

目的

開いているディレクトリストリームを閉じ、関連するリソースを解放する。

シグネチャまたは形状

static void close_destination_dir(DIR *dest_dir, char *dest_folder)

入力と出力

  • dest_dir: 閉じる対象のディレクトリポインタ（DIR *）。
  • dest_folder: エラー発生時にログ出力で使用するディレクトリパス（char *）。
  • 戻り値: なし（void）。

前提条件と不変条件

  • dest_dir および dest_folder は NULL であってはならない（Assert による検証）。

アルゴリズムまたは振る舞い

1. dest_dir と dest_folder が非 NULL であることを確認する。 2. closedir(dest_dir) を呼び出し、ディレクトリを閉じる。 3. closedir が失敗した場合、pg_fatal を呼び出してエラーメッセージを表示し、プログラムを終了させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_receivewal.c 内の以下の箇所から呼び出されている。
  • 473行目
  • 825行目
  • 呼び出し先: closedir（標準ライブラリ）、pg_fatal（PostgreSQL ログユーティリティ）。

関連データ構造

  • DIR: POSIX ディレクトリストリーム型。

失敗モードと境界条件

  • closedir が失敗した場合（例: ファイル記述子が不正な場合など）、pg_fatal によりプロセスが強制終了される。

保守担当者向けメモ

  • 本関数は pg_receivewal におけるディレクトリ操作の終了処理を標準化するために使用されている。エラー発生時の挙動は致命的（fatal）であるため、呼び出し側でディレクトリが正常に開かれていることを保証する必要がある。
