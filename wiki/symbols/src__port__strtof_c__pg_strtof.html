pg_strtof

定義

  • 種別: function
  • 定義箇所: src/port/strtof.c:29

目的

プラットフォーム固有の strtof 実装における精度不足やエラーハンドリングの不備を補完し、移植性を確保するためのラッパー関数です。

シグネチャまたは形状

float pg_strtof(const char *nptr, char **endptr)

入力と出力

  • nptr: 変換対象の文字列へのポインタ。
  • endptr: 変換終了位置を格納するポインタ（指定された場合）。
  • 戻り値: 変換された float 型の値。

前提条件と不変条件

  • 呼び出し元の errno 状態を保持し、関数終了時に復元します。
  • 内部で strtod をフォールバックとして使用し、float 型の範囲外やサブノーマル数の判定を補完します。

アルゴリズムまたは振る舞い

1. 呼び出し元の errno を保存し、errno を 0 にリセットします。 2. strtof を呼び出し、結果を fresult に格納します。 3. strtof がエラーを返した場合は、そのまま fresult を返します。 4. 解析不能、NaN、または通常の有限値（FLT_MIN 以上かつ isinf でない）であれば、保存していた errno を復元して fresult を返します。 5. 上記以外の場合、strtod を使用して double 型で再計算を行います。

  • strtod でエラーが発生した場合は fresult を返します。
  • 0 または無限大で値が一致する場合は、errno を復元して fresult を返します。
  • サブノーマル数（非ゼロ）の場合は、errno を復元して double からキャストした値を返します。
  • それ以外は errno を ERANGE に設定し、fresult を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/port.h にて strtof としてマクロ定義されており、PostgreSQL の各所から標準ライブラリの代わりとして呼び出されます。
  • 呼び出し先: 標準ライブラリの strtof および strtod。

関連データ構造

  • float 型および double 型の浮動小数点数。
  • <float.h> で定義される FLT_MIN。

失敗モードと境界条件

  • strtof がエラーを報告した場合、そのエラーをそのまま伝播させます。
  • 変換結果が範囲外の場合、errno を ERANGE に設定します。
  • nptr が解析不能な場合、元の errno を復元して fresult を返します。

保守担当者向けメモ

  • errno の操作が複雑であるため、すべてのリターンパスで caller_errno が正しく復元されていることを確認してください。
  • 特定のプラットフォーム（Cygwin, Mingw 等）での浮動小数点変換の挙動不一致を吸収するための実装です。
