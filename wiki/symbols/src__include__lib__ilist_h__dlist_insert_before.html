dlist_insert_before

定義

  • 種別: static inline function
  • 定義箇所: src/include/lib/ilist.h:392

目的

二重連結リストにおいて、指定したノード（before）の直前に新しいノード（node）を挿入します。

シグネチャまたは形状

static inline void dlist_insert_before(dlist_node *before, dlist_node *node)

入力と出力

  • **入力**:
  • before: 挿入位置の基準となる既存の dlist_node へのポインタ。
  • node: リストに挿入する新しい dlist_node へのポインタ。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • before および node は有効な dlist_node を指している必要があります。
  • before は既に何らかのリストに含まれている必要があります（before->prev が有効なノードを指しているため）。
  • 挿入後、node は before の直前に配置され、リストの整合性が維持されます。

アルゴリズムまたは振る舞い

1. node->prev を before->prev に設定し、node の前方を特定します。 2. node->next を before に設定し、node の後方を before に繋ぎます。 3. before->prev の next ポインタを node に更新します。 4. before->prev を node に更新し、リンクを確定させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/reorderbuffer.c:1191
  • src/include/lib/ilist.h:750（内部的なヘルパー関数経由の可能性あり）
  • **呼び出し先**:
  • 本関数はポインタ操作のみを行うため、他の関数を呼び出しません。

関連データ構造

  • dlist_node: リストのリンクを保持する構造体。
  • dlist_head: リストの起点（本関数は dlist_head を直接引数に取りませんが、リスト構造の一部として機能します）。

失敗モードと境界条件

  • before がリストの先頭ノードである場合、before->prev はリストの末尾を指している必要があります（循環リストの性質）。
  • before が NULL の場合、または before がリストに正しくリンクされていない場合、セグメンテーション違反が発生します。

保守担当者向けメモ

  • 本関数はインライン関数であり、パフォーマンスを重視して設計されています。
  • リストの整合性を維持するため、ノードの挿入・削除時には ILIST_DEBUG を有効にして整合性チェックを行うことが推奨されます。
  • 侵入型リストであるため、node が既に他のリストに含まれている状態で挿入を行うと、リスト構造が破壊される可能性があります。
