be_lo_lseek64

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-fsstubs.c:230

目的

ラージオブジェクト（LO）の読み書き位置を64ビットのオフセット値で指定・変更する。

シグネチャまたは形状

Datum be_lo_lseek64(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を受け取る。
  • fd (int32): ラージオブジェクトディスクリプタ。
  • offset (int64): 移動先のオフセット値。
  • whence (int32): シークの基準位置（SEEK_SET, SEEK_CUR, SEEK_END 等）。
  • **出力**: int64 型のシーク後の現在位置を PG_RETURN_INT64 で返す。

前提条件と不変条件

  • cookies 配列が初期化されており、指定された fd が有効な範囲内（0 <= fd < cookies_size）であること。
  • cookies[fd] が NULL でないこと（オープン済みであること）。

アルゴリズムまたは振る舞い

1. 引数から fd、offset、whence を取得する。 2. fd の妥当性を検証し、範囲外または未オープンであれば ERRCODE_UNDEFINED_OBJECT エラーを発生させる。 3. inv_seek を呼び出し、対象のラージオブジェクトに対してシーク操作を実行する。 4. inv_seek の戻り値（新しいオフセット位置）を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されることを想定。
  • **呼び出し先**: inv_seek（src/backend/storage/large_object/inv_api.c で定義されるラージオブジェクトAPI）。

関連データ構造

  • cookies: 現在開いているラージオブジェクトのディスクリプタを保持する動的配列。

失敗モードと境界条件

  • fd が負の値、または cookies_size 以上の場合、エラーとなる。
  • cookies[fd] が NULL の場合、エラーとなる。
  • inv_seek 内部で発生するエラー（無効なシーク位置など）は、そのまま上位へ伝播する。

保守担当者向けメモ

  • 本関数は64ビットオフセットを扱うため、大規模なラージオブジェクトの操作に適している。
  • トランザクション終了時には AtEOXact_LargeObject によってリソースがクリーンアップされるため、本関数で開いたディスクリプタのライフサイクルは現在のトランザクションに依存する。
