varattrib_4b

定義

  • 種別: union
  • 定義箇所: src/include/varatt.h:111

目的

PostgreSQLにおいて、4バイトのヘッダーを持つ可変長データ（varlena）の物理レイアウトを定義する共用体です。通常のデータと、インラインで圧縮されたデータの両形式を表現するために使用されます。

シグネチャまたは形状

typedef union {
    struct {                /* 通常のvarlena（4バイト長） */
        uint32      va_header;
        char        va_data[FLEXIBLE_ARRAY_MEMBER];
    } va_4byte;
    struct {                /* インライン圧縮形式 */
        uint32      va_header;
        uint32      va_tcinfo;
        char        va_data[FLEXIBLE_ARRAY_MEMBER];
    } va_compressed;
} varattrib_4b;

入力と出力

  • 入力: 本構造体へのポインタ（PTR）を介して、va_header や va_data にアクセスします。
  • 出力: マクロ（VARDATA_4B 等）を通じて、ヘッダーを除いたデータ本体へのポインタや、圧縮情報の取得が可能です。

前提条件と不変条件

  • 本構造体はアライメントが保証されない場合があります。構造体へのキャストを行う際は、適切にコピーしてからアクセスすることが推奨されます。
  • va_header のビット配置はエンディアンに依存するため、直接操作せず提供されているマクロを使用する必要があります。

アルゴリズムまたは振る舞い

  • va_4byte: 通常のデータ格納用。va_header にサイズ情報が含まれます。
  • va_compressed: 圧縮データ格納用。va_tcinfo に元のデータサイズと圧縮手法が格納され、va_data に圧縮後のデータ本体が保持されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/amcheck/verify_heapam.c
  • src/include/utils/expandeddatum.h
  • src/include/varatt.h 内の各種マクロ（VARDATA_4B, VARHDRSZ_COMPRESSED 等）
  • 呼び出し先: 実行コードではなくデータ構造定義であるため、関数呼び出しは発生しません。

関連データ構造

  • varattrib_1b: 1バイトヘッダーを持つ可変長データ構造。
  • varatt_external: TOASTテーブルに格納された外部データ用構造体。

失敗モードと境界条件

  • 1バイトヘッダーのデータに対して本構造体（4バイトヘッダー前提）を適用した場合、メモリレイアウトの不整合によりデータ破損や誤った読み取りが発生します。
  • アライメントが不適切な環境で直接キャストを行うと、プラットフォーム依存のクラッシュや不正なメモリアクセスを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を使用しているため、サイズ計算には offsetof マクロを使用してください。
  • ビット操作が必要な場合は、src/include/varatt.h で定義されている既存の述語マクロ（VARATT_IS_...）を必ず利用してください。
