GetAdditionalLocalPinLimit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/localbuf.c:313

目的

現在利用可能なローカルバッファのピン留め可能残数を取得します。

シグネチャまたは形状

uint32 GetAdditionalLocalPinLimit(void)

入力と出力

  • 入力: なし
  • 出力: uint32 型の残りピン留め可能数。

前提条件と不変条件

  • NLocalPinnedBuffers <= num_temp_buffers であること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. 現在ピン留めされているローカルバッファ数 (NLocalPinnedBuffers) が、設定された一時バッファの総数 (num_temp_buffers) を超えていないことを確認します。 2. 総数から現在ピン留めされている数を減算し、その結果を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/aio/read_stream.c:314 (buffer_limit の算出に使用)
  • 呼び出し先:
  • なし（本関数内で他の関数を呼び出していません）

関連データ構造

  • NLocalPinnedBuffers: 現在ピン留めされているローカルバッファの数。
  • num_temp_buffers: 設定された一時バッファの総数。

失敗モードと境界条件

  • NLocalPinnedBuffers が num_temp_buffers と等しい場合、戻り値は 0 となります。
  • アサーション違反が発生した場合、プロセスは異常終了します。

保守担当者向けメモ

  • 本関数は、非同期I/Oストリーム等のバッファ制限を計算する際に利用されます。
  • num_temp_buffers はセッション内で管理される一時バッファの最大数です。
