bytea_3

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-bytea.c:60

目的

PostgreSQL の ECPG における bytea 型のバイナリデータ送受信を検証するための、C 言語用データ構造体。

シグネチャまたは形状

struct bytea_3 {
    int len;
    char arr[DATA_SIZE];
};

入力と出力

  • 入力: SQL の bytea 型データ。
  • 出力: C 言語の構造体メンバ len（データ長）および arr（データ本体）。

前提条件と不変条件

  • len メンバには、arr に格納された有効なバイナリデータの長さが正しく設定されている必要がある。
  • arr のサイズは DATA_SIZE マクロによって固定されている。

アルゴリズムまたは振る舞い

ECPG プリプロセッサによって生成されるコードにおいて、SQL 文の実行結果を格納するホスト変数として利用される。ECPGdo() 関数を通じて、データベースから取得したバイナリデータを本構造体のメンバへマッピングする。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/sql-bytea.c:250 において、ECPGt_bytea 型として ECPGdo() 関数に渡されている。
  • 呼び出し先: 情報なし。

関連データ構造

  • bytea_1, bytea_2, bytea_4: 同様の目的で定義された他の bytea 構造体。
  • recv_vlen_buf: 本構造体型へのポインタ変数。

失敗モードと境界条件

  • DATA_SIZE を超えるバイナリデータが転送された場合、バッファオーバーフローやデータの切り捨てが発生する可能性がある。
  • len が不適切な値である場合、後続の dump_binary() 等の処理でメモリ破壊や不正な出力が発生する可能性がある。

保守担当者向けメモ

  • 本構造体は ECPG の回帰テスト用コードの一部であり、手動編集は推奨されない。
  • 構造体の定義変更は、対応する ECPG プリプロセッサの生成ロジックと整合させる必要がある。
