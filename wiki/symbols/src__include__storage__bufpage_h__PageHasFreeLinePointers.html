PageHasFreeLinePointers

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/bufpage.h:397

目的

ページヘッダー内のフラグを検査し、ページ内に再利用可能な空きラインポインタ（ItemIdData）が存在するかどうかを判定する。

シグネチャまたは形状

static inline bool PageHasFreeLinePointers(const PageData *page)

入力と出力

  • **入力**: page - 検査対象のページへのポインタ（PageData *）。
  • **出力**: bool - PD_HAS_FREE_LINES フラグがセットされていれば true、そうでなければ false を返す。

前提条件と不変条件

  • 入力される page は、有効な PageHeaderData 構造体として解釈可能であること。
  • 本関数が参照する PD_HAS_FREE_LINES は「ヒントビット」であり、パフォーマンス最適化のために使用される。WALログには記録されず、常に正確であるとは限らない。

アルゴリズムまたは振る舞い

1. 引数 page を PageHeaderData 型のポインタへキャストする。 2. 構造体内の pd_flags メンバに対してビット単位の AND 演算を行い、PD_HAS_FREE_LINES フラグが立っているかを確認する。 3. その結果を真偽値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/page/bufpage.c:250
  • src/backend/storage/page/bufpage.c:1006
  • **呼び出し先**:
  • 特筆すべき関数呼び出しなし（インライン展開される単純なビット演算）。

関連データ構造

  • PageData: ページ全体を指す抽象的な型。
  • PageHeaderData: ページ先頭のメタデータ構造体。pd_flags を保持する。
  • ItemIdData: ページ内のタプルを指すラインポインタ。

失敗モードと境界条件

  • page が NULL の場合、あるいはページヘッダーとして初期化されていないメモリ領域を指している場合、未定義の動作を引き起こす。

保守担当者向けメモ

  • 本関数はヒントビットを読み取るのみである。フラグの更新には PageSetHasFreeLinePointers や PageClearHasFreeLinePointers を使用すること。
  • ページレイアウトの整合性維持のため、フラグの状態と実際のラインポインタ配列の空き状況が乖離していないか注意が必要である。
