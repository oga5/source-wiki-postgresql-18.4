Clump

定義

  • **種別**: struct
  • **定義箇所**: src/backend/optimizer/geqo/geqo_eval.c:36

目的

GEQO（遺伝的クエリ最適化）において、結合済みのリレーションの集合を管理し、プラン構築過程で結合順序を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
    RelOptInfo *joinrel; /* 結合済みリレーションの集合 */
    int         size;    /* クランプに含まれる入力リレーションの数 */
} Clump;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではないため入出力はありません。
  • palloc を用いてメモリ確保され、merge_clump 等の処理を通じてリスト構造の一部として操作されます。

前提条件と不変条件

  • joinrel は有効な RelOptInfo 構造体を指している必要があります。
  • size はクランプに含まれるリレーションの数を正しく反映している必要があります。

アルゴリズムまたは振る舞い

  • gimme_tree 関数がツアーを走査する際、結合可能なリレーションを Clump としてまとめます。
  • merge_clump 関数により、既存の Clump リストに対して新しい Clump を再帰的にマージし、結合順序を構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_eval.c 内の gimme_tree および merge_clump でインスタンス化および参照されます。
  • **呼び出し先**: 本構造体はデータ構造であるため、特定の関数を呼び出すことはありません。

関連データ構造

  • RelOptInfo: 結合対象のリレーション情報を保持します。
  • List: Clump オブジェクトを管理するためのリスト構造として使用されます。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • merge_clump において、結合不可能なリレーションの組み合わせが発生した場合の挙動は、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • geqo_eval の評価プロセスで生成される一時的な構造体であるため、評価終了後のメモリ解放（MemoryContext の管理）が適切に行われていることを確認してください。
  • root->join_rel_list を変更する副作用を伴う処理で使用されるため、リストの整合性に注意してください。
