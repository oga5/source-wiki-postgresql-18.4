pgstat_discard_stats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat.c:517

目的

永続化された統計ファイル（pg_stat/pgstat.stat）を削除し、メモリ上の統計情報をリセットすることで、統計システムを初期状態へ戻す。

シグネチャまたは形状

void pgstat_discard_stats(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてファイルシステム上の統計ファイル削除と、メモリ上の統計状態リセットを行う）

前提条件と不変条件

  • シングルユーザーモードを含む、あらゆる実行モードで呼び出し可能であること。
  • 統計ファイルの削除失敗はログ記録されるが、処理全体を停止させる致命的なエラーとは見なされない。

アルゴリズムまたは振る舞い

1. PGSTAT_STAT_PERMANENT_FILENAME で定義される統計ファイルを unlink() で削除する。

  • ファイルが存在しない場合（ENOENT）は DEBUG2 レベルで記録する。
  • その他のエラーが発生した場合は LOG レベルで報告する。

    • 2. pgstat_reset_after_failure() を呼び出し、メモリ上の統計内容をリセットする。これにより、固定番号の統計のリセットタイムスタンプが現在時刻に更新される。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xlog.c:5721
  • 呼び出し先:
  • unlink() (システムコール)
  • pgstat_reset_after_failure()

関連データ構造

  • PGSTAT_STAT_PERMANENT_FILENAME (ファイルパス定義)

失敗モードと境界条件

  • ファイル削除失敗時: unlink が失敗した場合、errno に応じて DEBUG2 または LOG レベルのメッセージを出力する。
  • ファイル不在時: ENOENT の場合は正常なフローとして処理される。

保守担当者向けメモ

  • 本関数は統計システムのクリーンアップを行うため、リカバリ処理や初期化失敗時のリカバリパスから呼び出されることを想定している。
  • 統計ファイル削除の成否に関わらず、メモリ上のリセット処理（pgstat_reset_after_failure）は必ず実行される。
