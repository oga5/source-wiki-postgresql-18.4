pg_GSS_write

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-secure-gssapi.c:92

目的

GSSAPI を用いてデータを暗号化し、ネットワーク経由で安全に送信する。

シグネチャまたは形状

ssize_t pg_GSS_write(PGconn *conn, const void *ptr, size_t len)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体。
  • ptr: 送信対象のデータバッファ。
  • len: 送信データのバイト数。
  • **出力**:
  • 成功時: 送信したバイト数（len と一致）。
  • 失敗時: -1（エラー詳細は conn->errorMessage に格納）。

前提条件と不変条件

  • conn->gctx が有効な GSSAPI コンテキストであること。
  • 処理完了時、PqGSSSendConsumed は 0 にリセットされる。

アルゴリズムまたは振る舞い

1. **再送チェック**: len < PqGSSSendConsumed の場合、エラーとして終了する。 2. **送信バッファ処理**: PqGSSSendLength にデータが残っている場合、pqsecure_raw_write を呼び出し、バッファ内の暗号化済みデータを優先して送信する。 3. **暗号化ループ**:

  • 未暗号化データがある限り、gss_wrap を用いてデータを暗号化する。
  • 暗号化パケットに uint32 のネットワークバイトオーダーの長さヘッダーを付与し、PqGSSSendBuffer に格納する。
  • pqsecure_raw_write を呼び出し、暗号化データを送信する。

    • 4. **バッファ管理**: gss_release_buffer を呼び出し、GSSAPI が割り当てた一時バッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-secure.c（pg_GSS_write を呼び出す上位のセキュリティ抽象化層）。
  • **呼び出し先**: gss_wrap（GSSAPI）、pqsecure_raw_write（低レベルソケット送信）、gss_release_buffer（GSSAPI）。

関連データ構造

  • PGconn: 接続状態および GSSAPI コンテキスト（gctx）を保持。
  • PqGSSSendBuffer, PqGSSSendLength, PqGSSSendConsumed: 送信用の内部バッファおよび状態管理変数。

失敗モードと境界条件

  • **ハードエラー**: gss_wrap の失敗、暗号化後のパケットサイズ超過、または機密性（confidentiality）が保証されない場合は -1 を返す。
  • **入力検証**: 送信データが PqGSSSendConsumed より小さい場合は EINVAL を設定して失敗する。

保守担当者向けメモ

  • 本関数は非同期 I/O を前提としており、部分的な書き込みが発生した場合は呼び出し元が同じデータで再試行する必要がある。
  • パケット境界を維持するため、uint32 の長さヘッダー付与は必須であり、変更不可。
  • 過去のバグ（commit d053a879b）を避けるため、部分的な書き込み完了の報告は行わず、全データの送信完了まで待機する設計となっている。
