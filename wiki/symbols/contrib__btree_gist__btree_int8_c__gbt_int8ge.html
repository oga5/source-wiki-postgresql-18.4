gbt_int8ge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int8.c:34

目的

int8（64ビット整数）型同士の「以上（greater than or equal to）」比較をGiSTインデックスの検索処理等で実行する。

シグネチャまたは形状

static bool gbt_int8ge(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の int64 値へのポインタ。
  • b: 比較対象の int64 値へのポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a >= *b が真であれば true、そうでなければ false を返す。

前提条件と不変条件

  • a および b は、有効な int64 型の値を指すポインタである必要がある。

アルゴリズムまたは振る舞い

1. ポインタ a が指す値を int64 としてキャストし取得する。 2. ポインタ b が指す値を int64 としてキャストし取得する。 3. C言語の標準的な >= 演算子を用いて比較を行い、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_int8.c:85 において、静的構造体（tinfo 等）のメンバとして参照されている。
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない（組み込みの比較演算子のみを使用）。

関連データ構造

  • int64: PostgreSQLの64ビット整数型。

失敗モードと境界条件

  • ポインタが NULL の場合、セグメンテーション違反が発生する可能性がある。
  • int64 の範囲外の値や不正なメモリ領域を参照した場合の挙動は未定義。

保守担当者向けメモ

  • 本関数は int8 型専用の比較ラッパーである。GiSTインデックスの演算子クラス定義において、比較ロジックを共通化するために使用される。
