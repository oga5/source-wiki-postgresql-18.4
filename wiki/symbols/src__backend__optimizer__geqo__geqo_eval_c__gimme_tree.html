gimme_tree

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_eval.c:162

目的

遺伝的アルゴリズム（GEQO）で生成された結合順序（ツアー）に基づき、リレーションを結合してプランツリー（RelOptInfo）を構築する。

シグネチャまたは形状

RelOptInfo *gimme_tree(PlannerInfo *root, Gene *tour, int num_gene)

入力と出力

  • **入力**:
  • root: クエリのプランナ情報。
  • tour: 結合順序を示す遺伝子配列。
  • num_gene: ツアー内の遺伝子数。
  • **出力**:
  • 構築された結合リレーション（RelOptInfo *）。結合に失敗した場合は NULL を返す。

前提条件と不変条件

  • root->join_search_private に GeqoPrivateData が格納されている必要がある。
  • 評価プロセスで生成されるプラン情報は、呼び出し元で適切にメモリ管理（解放）される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: 結合済みリレーションの集合を管理する clumps リストを空で初期化する。 2. **ツアー走査**: tour を順に走査し、各リレーションを単一の Clump として merge_clump を呼び出す。この際、ヒューリスティックに基づいた結合（force=false）を試みる。 3. **強制結合**: 全てのツアー走査後、clumps が1つにまとまっていない場合、ヒューリスティックを無視して強制的に結合（force=true）を試みる。 4. **結果判定**: 最終的に clumps が単一の要素であれば、その joinrel を返す。それ以外は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_eval.c (内部評価ロジック)
  • src/backend/optimizer/geqo/geqo_main.c:272
  • **呼び出し先**:
  • merge_clump: 結合の試行とリスト管理。
  • list_nth, palloc, list_length, linitial: PostgreSQLのリストおよびメモリ管理ユーティリティ。

関連データ構造

  • RelOptInfo: 結合結果を保持するプランナの基本構造体。
  • Clump: 結合済みのリレーション集合を管理する構造体。
  • GeqoPrivateData: GEQO固有のプランナプライベートデータ。

失敗モードと境界条件

  • ツアー内の全ての結合を完了できず、最終的な clumps の長さが1にならない場合、NULL を返す。
  • num_gene が0の場合や、ツアーの内容が不正な場合は未定義の動作となる可能性がある。

保守担当者向けメモ

  • root->join_rel_list に副作用を与えるため、呼び出し元での状態復元（リストの切り詰め等）が必須である。
  • merge_clump の第5引数（force）の切り替えにより、ヒューリスティックな探索と強制的な結合の2段階で処理が行われている点に注意すること。
