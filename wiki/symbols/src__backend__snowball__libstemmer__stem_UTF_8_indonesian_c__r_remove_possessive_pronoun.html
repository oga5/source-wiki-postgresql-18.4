r_remove_possessive_pronoun

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_indonesian.c:135

目的

インドネシア語のステミング処理において、単語末尾から所有代名詞（-ku, -mu, -nya）を除去し、語幹を正規化する。

シグネチャまたは形状

static int r_remove_possessive_pronoun(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング対象の単語バッファおよび状態を保持する環境構造体）
  • **出力**: int 型。除去が成功した場合は 1 を返し、失敗または条件不適合の場合は 0 を返す。内部エラー発生時は負の値を返す。

前提条件と不変条件

  • z->c が現在の処理位置を指していること。
  • z->lb がバッファの境界（下限）であること。
  • 処理対象の単語末尾が 'a' (97) または 'u' (117) である必要がある。

アルゴリズムまたは振る舞い

1. 現在位置 z->c を z->ket（接尾辞の終了位置）に設定する。 2. バッファの末尾から1文字前が 'a' または 'u' でない場合、処理を中断し 0 を返す。 3. a_1 配列に定義された所有代名詞パターンを find_among_b で後方検索する。 4. マッチした場合、その位置を z->bra（接尾辞の開始位置）に設定し、slice_del で該当部分を削除する。 5. 削除成功後、環境内の整数配列 z->I[1] をデクリメントし、1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_indonesian.c および src/backend/snowball/libstemmer/stem_ISO_8859_1_indonesian.c 内のメインステミング処理フローから呼び出される。
  • **呼び出し先**: find_among_b（パターンマッチング）、slice_del（バッファ操作）。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。
  • a_1: 所有代名詞のパターンを格納する among 構造体の配列。

失敗モードと境界条件

  • 単語の長さが不足している場合（z->c - 1 <= z->lb）、処理をスキップする。
  • 末尾文字が 'a' または 'u' 以外の場合、処理をスキップする。
  • find_among_b がパターンに一致しない場合、何も変更せず 0 を返す。
  • slice_del が内部エラー（負の値）を返した場合、その値をそのまま伝播させる。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接編集せず、元の .sbl スクリプトを修正すること。
  • z->I[1] の更新は、ステミングの進行状態を追跡するために重要である。
