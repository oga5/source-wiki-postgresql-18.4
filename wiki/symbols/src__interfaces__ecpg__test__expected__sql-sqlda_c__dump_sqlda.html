dump_sqlda

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-sqlda.c:130

目的

ECPGのSQLDA（SQL Descriptor Area）構造体に含まれる各列の情報を解析し、型に応じて標準出力へダンプするデバッグ用関数です。

シグネチャまたは形状

static void dump_sqlda(sqlda_t *sqlda);

入力と出力

  • **入力**: sqlda_t *sqlda（ダンプ対象のSQLDA構造体へのポインタ）
  • **出力**: なし（標準出力へのフォーマット済み文字列出力のみ）

前提条件と不変条件

  • sqlda が NULL の場合、エラーメッセージを出力して早期リターンします。
  • sqlda->sqld に格納された列数分だけループ処理を行います。

アルゴリズムまたは振る舞い

1. sqlda が NULL でないことを確認します。 2. sqlda->sqld（列数）までループし、各列の sqlvar を走査します。 3. sqlind（インジケータ）が -1 の場合、値が NULL である旨を出力します。 4. それ以外の場合、sqltype に基づいて以下の型を判定し、値をキャストして出力します。

  • ECPGt_char: 文字列として出力。
  • ECPGt_int: int として出力。
  • ECPGt_long: long int として出力。
  • ECPGt_long_long: long long int として出力。
  • ECPGt_double: double として出力。
  • ECPGt_numeric: PGTYPESnumeric_to_asc を使用して文字列変換し、出力後に PGTYPESchar_free でメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/sql-sqlda.c および src/interfaces/ecpg/test/expected/compat_informix-sqlda.c 内の各テストケースから呼び出されています。
  • **呼び出し先**: printf（標準ライブラリ）、PGTYPESnumeric_to_asc、PGTYPESchar_free（pgtypesライブラリ）。

関連データ構造

  • sqlda_t: SQLDAの全体構造を保持。
  • sqlvar_t: SQLDA内の個別の列情報を保持。

失敗モードと境界条件

  • sqlda が NULL の場合、安全に処理を中断します。
  • 未定義の sqltype が渡された場合、その列の出力はスキップされます。

保守担当者向けメモ

  • 新しいデータ型をサポートする場合は、switch 文に新しい case を追加してください。
  • sqlda_t の定義は環境（Informix互換モードの有無）により異なるため、変更時は sqlda-compat.h および sqlda-native.h の定義を確認してください。
