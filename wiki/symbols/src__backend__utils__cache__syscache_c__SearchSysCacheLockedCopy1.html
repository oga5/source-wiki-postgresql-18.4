SearchSysCacheLockedCopy1

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/syscache.c:403

目的

システムカタログキャッシュからタプルを検索し、ロックを取得した状態でそのコピーを返します。呼び出し側がタプルを安全に変更できるようにすることを目的としています。

シグネチャまたは形状

HeapTuple SearchSysCacheLockedCopy1(int cacheId, Datum key1)

入力と出力

  • **入力**:
  • cacheId: 検索対象のシステムキャッシュID。
  • key1: 検索に使用する単一のキー値（Datum型）。
  • **出力**:
  • HeapTuple: 見つかったタプルのコピー。キャッシュ内のタプルは解放されるため、呼び出し側で heap_freetuple 等によるメモリ管理が必要です。見つからない場合は NULL または無効なタプルが返されます。

前提条件と不変条件

  • SearchSysCacheLocked1 が内部で呼び出されるため、呼び出し側はインプレース更新などの特定の整合性が必要な状況であることを前提とします。
  • 返されるタプルはキャッシュの直接参照ではなくコピーであるため、呼び出し側で変更可能です。

アルゴリズムまたは振る舞い

1. SearchSysCacheLocked1(cacheId, key1) を呼び出し、キャッシュからタプルを検索し、ロックを取得します。 2. 検索結果が有効でない場合、そのまま結果を返します。 3. 有効なタプルが取得できた場合、heap_copytuple を使用してタプルのコピーを作成します。 4. ReleaseSysCache を呼び出し、元のキャッシュエントリの参照カウントを減らし、ロックを解放します。 5. 作成したコピーを呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c
  • src/backend/commands/dbcommands.c
  • src/backend/commands/event_trigger.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/tablecmds.c
  • src/backend/utils/cache/relcache.c
  • **呼び出し先**:
  • SearchSysCacheLocked1
  • heap_copytuple
  • ReleaseSysCache

関連データ構造

  • HeapTuple: PostgreSQLのタプル表現。
  • CatCache: syscache.c 内で管理されるシステムカタログキャッシュ構造体。

失敗モードと境界条件

  • 指定された cacheId や key1 に該当するタプルが存在しない場合、HeapTupleIsValid チェックにより無効なタプルが返されます。
  • メモリ不足時には heap_copytuple がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は SearchSysCacheLocked1 のロックシーケンスをラップし、コピーを返すことで安全性を高めています。
  • キャッシュの直接変更を避けるためのユーティリティであるため、インプレース更新が必要なケース以外での使用は推奨されません。
