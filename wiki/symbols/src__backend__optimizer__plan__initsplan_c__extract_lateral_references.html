extract_lateral_references

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/initsplan.c:756

目的

LATERAL 指定されたサブクエリや関数が外部リレーションを参照している場合、その依存関係を特定し、必要な変数を抽出してプランナの管理構造に登録する。

シグネチャまたは形状

static void extract_lateral_references(PlannerInfo *root, RelOptInfo *brel, Index rtindex)

入力と出力

  • root: プランナのグローバル情報。
  • brel: 処理対象となるリレーションの最適化情報 (RelOptInfo)。
  • rtindex: 対象リレーションのレンジテーブルインデックス。
  • 出力: なし（副作用として brel->lateral_vars の更新および root 内のターゲットリストへの変数の追加を行う）。

前提条件と不変条件

  • 対象の RangeTblEntry が lateral フラグを保持していること。
  • rtindex が root->simple_rte_array に存在すること。

アルゴリズムまたは振る舞い

1. **LATERALチェック**: 対象RTEが lateral でない場合は即座に終了する。 2. **変数抽出**: RTEの種類（RTE_RELATION, RTE_SUBQUERY, RTE_FUNCTION 等）に応じて、pull_vars_of_level を使用し、外部レベルを参照する変数（Var または PlaceHolderVar）を抽出する。 3. **正規化**: 抽出した各ノードをコピーし、varlevelsup を 0 に調整する。PlaceHolderVar の場合は、必要に応じて preprocess_phv_expression を呼び出し、式を前処理する。 4. **登録**: 抽出した変数を add_vars_to_targetlist を通じてソースリレーションのターゲットリストへ追加する。 5. **保持**: 最終的に brel->lateral_vars に抽出した変数リストを格納する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/plan/initsplan.c 内のロジックから呼び出される（詳細はソースコードのフローを参照）。
  • 呼び出し先: pull_vars_of_level, copyObject, IncrementVarSublevelsUp, preprocess_phv_expression, add_vars_to_targetlist。

関連データ構造

  • PlannerInfo: プランナの全体状態。
  • RelOptInfo: リレーションの最適化情報。
  • RangeTblEntry: レンジテーブルエントリ。
  • Var, PlaceHolderVar: 抽出対象となるノード。

失敗モードと境界条件

  • lateral フラグが偽の場合は何もしない。
  • 抽出された変数が空（NIL）の場合は何もしない。
  • 未対応の rtekind が渡された場合は Assert(false) により停止する。

保守担当者向けメモ

  • 本関数は LATERAL 参照を解決するための重要なステップであり、ここで抽出された変数は後続の rebuild_lateral_attr_needed や create_lateral_join_info で利用される。
  • where_needed を rtindex のみに限定する実装は簡略化されたアプローチであるため、将来的な変更時には注意が必要。
