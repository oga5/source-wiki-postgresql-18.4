newcolor

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_color.c:184

目的

正規表現コンパイル時に使用する新しい「色（color）」を割り当て、カラーマップ（colormap）の管理領域を必要に応じて拡張する。

シグネチャまたは形状

color newcolor(struct colormap *cm)

入力と出力

  • **入力**: cm (カラーマップ構造体へのポインタ)
  • **出力**: 割り当てられた新しい色の識別子（color 型）。エラー発生時または色の上限に達した場合は COLORLESS を返す。

前提条件と不変条件

  • cm が有効な colormap 構造体を指していること。
  • MAX_COLOR を超える色の割り当ては許可されない。
  • 内部の cd 配列が拡張される際、ポインタの再配置が発生する可能性がある。

アルゴリズムまたは振る舞い

1. **エラーチェック**: CISERR() が真であれば直ちに COLORLESS を返す。 2. **再利用**: cm->free に未使用の色インデックスが存在する場合、それを優先的に再利用する。 3. **新規割り当て**: 未使用領域がある場合は cm->max をインクリメントして新しい色を確保する。 4. **拡張**: 領域が不足している場合、MALLOC または REALLOC を用いて cm->cd 配列のサイズを倍増させる。

  • MAX_COLOR を超える拡張は REG_ECOLORS エラーとして処理される。
  • メモリ確保失敗時は REG_ESPACE エラーとして処理される。

    • 5. **初期化**: 確保された colordesc 構造体の各フィールド（nschrs, nuchrs, sub, arcs, flags 等）を初期状態にリセットする。 6. **返却**: 確保したインデックスを color 型にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_color.c 内の他の関数（subcolor 等）。
  • src/backend/regex/regcomp.c
  • **呼び出し先**:
  • MALLOC, REALLOC, memcpy（メモリ管理用）
  • CERR（エラー報告用）

関連データ構造

  • colormap: 色の割り当て状態を管理するメイン構造体。
  • colordesc: 個々の色の属性を保持する構造体。

失敗モードと境界条件

  • **メモリ不足**: MALLOC/REALLOC が失敗した場合、REG_ESPACE をセットし COLORLESS を返す。
  • **色数上限**: MAX_COLOR に達した場合、REG_ECOLORS をセットし COLORLESS を返す。
  • **エラー状態**: 既にエラーフラグが立っている場合、処理を行わず COLORLESS を返す。

保守担当者向けメモ

  • cm->cd 配列の拡張時にポインタが移動するため、この関数を呼び出す前後で colormap 内の他のポインタ参照が壊れていないか注意すること。
  • MAX_COLOR + 1 までの制限により、配列サイズのオーバーフローは防止されている。
