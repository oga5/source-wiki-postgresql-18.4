Query

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/parsenodes.h:117

目的

SQL文の解析（parse_analyze）によって生成される、意味的に整理された「クエリツリー」のルートノードです。リライターやプランナーが処理を行うための中心的なデータ構造として機能します。

シグネチャまたは形状

typedef struct Query {
    NodeTag     type;
    CmdType     commandType;
    QuerySource querySource;
    int64       queryId;
    bool        canSetTag;
    Node       *utilityStmt;
    int         resultRelation;
    /* ... 多数のフラグおよびリストフィールド ... */
    ParseLoc    stmt_location;
    ParseLoc    stmt_len;
} Query;

入力と出力

  • **入力**: パーサーが生成した RawStmt を受け取り、意味解析器が Query 構造体を構築します。
  • **出力**: 最終的にプランナーによって PlannedStmt（実行計画）へ変換されます。

前提条件と不変条件

  • commandType が CMD_UTILITY の場合、utilityStmt フィールドが非NULLである必要があります。
  • stmt_location および stmt_len は、トップレベルのクエリでのみ設定されることが一般的です。
  • queryId はクエリの同一性判定（Jumble）の結果であり、equal 比較やクエリのハッシュ計算からは除外されます。

アルゴリズムまたは振る舞い

1. **解析**: parse_analyze() により、生の構文木から意味的な情報を付与された Query 構造体が生成されます。 2. **変換**: リライターやプランナーがこの構造体を読み取り、最適化や書き換えを行います。 3. **メタデータ管理**: hasAggs や hasSubLinks などのフラグ群により、クエリの特性（集約関数の有無やサブクエリの存在など）を効率的に判定可能です。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • contrib/pg_stat_statements/pg_stat_statements.c: pgss_post_parse_analyze および pgss_planner で解析後のクエリを処理。
  • contrib/postgres_fdw/deparse.c および postgres_fdw.c: リモートクエリ生成のために root->parse から参照。
  • **呼び出し先**:
  • 本構造体は List 型のフィールドを多用しており、TargetEntry、CommonTableExpr、SortGroupClause などのノードを保持します。

関連データ構造

  • CmdType: クエリの種類（SELECT, INSERT等）。
  • FromExpr: jointree を構成するFROM/WHERE句のツリー。
  • TargetEntry: targetList や returningList を構成する要素。
  • RangeTblEntry: rtable に格納されるテーブル参照情報。

失敗モードと境界条件

  • stmt_location や stmt_len が不明な場合、それぞれ -1 や 0 が設定されることがあります。
  • queryId は int64 で保持されますが、uint64 の全範囲を表現できないため、表示上の制約として符号付きで扱われます。

保守担当者向けメモ

  • pg_node_attr マクロが付与されたフィールド（query_jumble_ignore 等）は、クエリの同一性判定時に自動的に無視されます。変更時はこの属性の整合性に注意してください。
  • Query は非常に巨大な構造体です。特定の機能（MERGEやON CONFLICT等）に関するフィールドを追加・変更する場合は、関連する List や Node の定義も併せて確認してください。
