tqueueShutdownReceiver

定義

  • 種別: function
  • 定義箇所: src/backend/executor/tqueue.c:91

目的

並列クエリ実行において使用される共有メモリキュー（shm_mq）のデタッチ処理を行い、送信側レシーバーの終了処理を担います。

シグネチャまたは形状

static void tqueueShutdownReceiver(DestReceiver *self)

入力と出力

  • **入力**: self (DestReceiver *) - TQueueDestReceiver 構造体としてキャストされるレシーバーインスタンス。
  • **出力**: なし（副作用として共有メモリキューのデタッチを実行）。

前提条件と不変条件

  • self は TQueueDestReceiver 型の構造体として安全にキャスト可能であること。
  • tqueue->queue が NULL でない場合、有効な共有メモリキューハンドルを保持していること。

アルゴリズムまたは振る舞い

1. 引数 self を TQueueDestReceiver 型にキャストします。 2. tqueue->queue が NULL でない場合、shm_mq_detach を呼び出して共有メモリキューからデタッチします。 3. 処理完了後、tqueue->queue を NULL に設定し、二重解放や無効なポインタへのアクセスを防ぎます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/tqueue.c:127 にて、DestReceiver インターフェースの rShutdown フィールドに関数ポインタとして登録されています。
  • **呼び出し先**: shm_mq_detach（共有メモリキューのデタッチ処理）。

関連データ構造

  • TQueueDestReceiver: DestReceiver を継承した送信側構造体。shm_mq_handle を保持します。
  • DestReceiver: PostgreSQL のタプル受信インターフェース。

失敗モードと境界条件

  • tqueue->queue が既に NULL の場合、デタッチ処理はスキップされ、安全に終了します。

保守担当者向けメモ

  • 本関数は DestReceiver インターフェースの一部として呼び出されます。
  • 共有メモリ上のリソースを管理しているため、並列実行の終了シーケンスにおいて適切に呼び出される必要があります。
