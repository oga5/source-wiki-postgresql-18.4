VarParamState

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_param.c:49

目的

SQLクエリの解析過程において、型が未定のパラメータ（$1, $2 など）の型情報を動的に管理・保持するために使用されます。

シグネチャまたは形状

typedef struct VarParamState
{
    Oid        **paramTypes;    /* パラメータ型OIDの配列 */
    int         *numParams;     /* 配列の要素数 */
} VarParamState;

入力と出力

  • 入力: ParseState の p_ref_hook_state を介して参照されます。
  • 出力: 解析中に決定されたパラメータの型情報（Oid）を paramTypes 配列に格納し、管理します。

前提条件と不変条件

  • setup_parse_variable_parameters によって初期化され、ParseState に関連付けられている必要があります。
  • 解析終了時に check_variable_parameters によって、すべてのパラメータの型が矛盾なく解決されているか検証されることが想定されています。

アルゴリズムまたは振る舞い

1. パーサーが ParamRef を検出すると、登録されたフック（variable_paramref_hook）が本構造体を参照します。 2. variable_coerce_param_hook がクエリの文脈からパラメータの型を推論し、本構造体の paramTypes 配列を更新します。 3. 解析完了後、ツリーを走査して型が UNKNOWNOID のまま残っていないかを確認します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_param.c 内の setup_parse_variable_parameters, variable_paramref_hook, variable_coerce_param_hook, check_variable_parameters 等が、pstate->p_ref_hook_state をキャストして本構造体を利用しています。
  • 呼び出し先: 直接的な関数呼び出し関係は定義されておらず、データ保持用の構造体として機能します。

関連データ構造

  • ParseState: 本構造体を保持し、解析の文脈を提供します。
  • ParamRef: パーサーが処理する対象のノード。

失敗モードと境界条件

  • 型が UNKNOWNOID のまま解析が終了した場合、後のフェーズでエラーとなる可能性があります。
  • paramtypmod は実行時の型チェックを確実にするため -1 に設定される設計となっています。

保守担当者向けメモ

  • 本構造体は「明示的な $n 形式のパラメータ」のみを対象としています。
  • paramTypes はポインタのポインタとして定義されており、動的な配列拡張を考慮した設計です。
