LogicalRepBeginData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/logicalproto.h:127

目的

論理レプリケーションにおけるトランザクション開始メッセージ（BEGIN）のデータを保持するための構造体です。

シグネチャまたは形状

typedef struct LogicalRepBeginData
{
    XLogRecPtr    final_lsn;
    TimestampTz   committime;
    TransactionId xid;
} LogicalRepBeginData;

入力と出力

本構造体は、論理レプリケーションストリームから読み取られたトランザクション開始情報を格納するために使用されます。

前提条件と不変条件

  • final_lsn: トランザクションの終了位置を示すログシーケンス番号。
  • committime: トランザクションのコミット時刻。
  • xid: トランザクションID。

アルゴリズムまたは振る舞い

論理レプリケーションのデコードプロセスにおいて、ストリームから受信した BEGIN メッセージの内容をデシリアライズする際のデータコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c: 受信したトランザクション情報を処理する際に本構造体をインスタンス化します。
  • **呼び出し先**:
  • src/backend/replication/logical/proto.c: logicalrep_read_begin 関数が、ストリームから読み取った値を本構造体に格納します。

関連データ構造

  • XLogRecPtr (WAL位置)
  • TimestampTz (タイムスタンプ)
  • TransactionId (トランザクション識別子)

失敗モードと境界条件

  • ストリームから読み取る際、データが不完全な場合はデシリアライズ関数側でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は論理レプリケーションプロトコルの BEGIN メッセージと直接対応しています。プロトコル仕様に変更を加える際は、本構造体のフィールド構成と logicalrep_read_begin / logicalrep_write_begin の実装を同期させる必要があります。
