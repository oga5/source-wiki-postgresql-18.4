DropAllPreparedStatements

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:539

目的

現在のセッションで保持されているすべてのプリペアドステートメントを破棄し、関連するリソースを解放する。

シグネチャまたは形状

void DropAllPreparedStatements(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてハッシュテーブルの状態を変更）

前提条件と不変条件

  • prepared_queries ハッシュテーブルが初期化されているか、または NULL であること。
  • 本関数はセッションローカルなスコープで動作し、他セッションのステートメントには影響を与えない。

アルゴリズムまたは振る舞い

1. prepared_queries が NULL の場合、何もせず終了する。 2. hash_seq_init を使用して、prepared_queries ハッシュテーブルの全エントリを走査する。 3. 各エントリに対して以下の処理を行う：

  • DropCachedPlan(entry->plansource) を呼び出し、計画キャッシュのエントリを解放する。
  • hash_search を HASH_REMOVE モードで呼び出し、ハッシュテーブルからエントリを削除する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/discard.c:72
  • src/backend/commands/prepare.c:510
  • 呼び出し先:
  • DropCachedPlan
  • hash_seq_init
  • hash_seq_search
  • hash_search

関連データ構造

  • prepared_queries: セッション内のプリペアドステートメントを管理するハッシュテーブル。
  • PreparedStatement: ハッシュテーブルに格納される各ステートメントの構造体。
  • CachedPlanSource: DropCachedPlan を通じて管理される計画ソース。

失敗モードと境界条件

  • prepared_queries が未初期化（NULL）の場合、安全に早期リターンする。
  • ハッシュテーブルが空の場合、ループは実行されず正常終了する。

保守担当者向けメモ

  • 本関数は DISCARD ALL コマンドやセッション終了時のクリーンアップ処理の一部として利用される。
  • DropCachedPlan の呼び出しにより、計画キャッシュの参照カウントが適切に減少されることを前提としている。
