CreateTransform

定義

  • 種別: function
  • 定義箇所: src/backend/commands/functioncmds.c:1831

目的

SQLの CREATE TRANSFORM 文を処理し、特定のデータ型とプログラミング言語間の変換関数（FROM SQL / TO SQL）をシステムカタログ pg_transform に登録または更新する。

シグネチャまたは形状

ObjectAddress CreateTransform(CreateTransformStmt *stmt)

入力と出力

  • **入力**: CreateTransformStmt *stmt（解析済みの CREATE TRANSFORM 構文木）
  • **出力**: ObjectAddress（作成または更新された変換オブジェクトの識別子）

前提条件と不変条件

  • 対象のデータ型が擬似型（pseudo-type）やドメイン型であってはならない。
  • 呼び出し元は対象の型および言語に対する USAGE 権限、および変換関数に対する EXECUTE 権限を保持している必要がある。
  • FROM SQL 関数は戻り値の型が internal であること、TO SQL 関数は戻り値の型が変換対象の型と一致していることが必須。

アルゴリズムまたは振る舞い

1. **検証**: 指定された型、言語、および変換関数の存在と権限をチェックする。 2. **関数検証**: check_transform_function を呼び出し、変換関数のシグネチャが適切か検証する。 3. **カタログ操作**: pg_transform を検索し、既存の変換が存在する場合は REPLACE 指定の有無を確認する。

  • 存在しない場合: CatalogTupleInsert で新規登録。
  • 存在する場合: REPLACE 指定があれば CatalogTupleUpdate で更新、なければエラーを投げる。

    • 4. **依存関係の構築**: pg_transform エントリと、関連する型、言語、関数との間に依存関係（ObjectAddress）を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c:1743（ProcessUtility 経由で呼び出される）
  • **呼び出し先**: typenameTypeId, get_typtype, object_ownercheck, get_language_oid, LookupFuncWithArgs, check_transform_function, table_open, CatalogTupleInsert, CatalogTupleUpdate, deleteDependencyRecordsFor, add_exact_object_address 等。

関連データ構造

  • CreateTransformStmt: SQL構文木。
  • pg_transform: 変換情報を保持するシステムカタログ。
  • ObjectAddress: オブジェクトの識別子。

失敗モードと境界条件

  • 指定した型がドメインや擬似型である場合に ERRCODE_WRONG_OBJECT_TYPE を発生させる。
  • 変換関数が期待される戻り値型と一致しない場合に ERRCODE_INVALID_OBJECT_DEFINITION を発生させる。
  • 既に存在する変換に対して OR REPLACE が指定されていない場合に ERRCODE_DUPLICATE_OBJECT を発生させる。
  • 権限不足時に aclcheck_error を発生させる。

保守担当者向けメモ

  • pg_transform は型と言語のペアで一意性を管理している。
  • 変換関数を更新する際は、既存の依存関係を deleteDependencyRecordsFor で一度削除してから再構築する設計となっている。
