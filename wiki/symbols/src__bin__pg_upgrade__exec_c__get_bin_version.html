get_bin_version

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/exec.c:32

目的

指定されたクラスタのバイナリディレクトリから pg_ctl を実行し、そのバージョン情報を取得して ClusterInfo 構造体に格納する。

シグネチャまたは形状

static void get_bin_version(ClusterInfo *cluster)

入力と出力

  • **入力**: ClusterInfo *cluster （bindir メンバにバイナリパスが設定されていること）
  • **出力**: なし（副作用として cluster->bin_version を更新する）

前提条件と不変条件

  • cluster->bindir が有効なディレクトリパスを指していること。
  • pg_ctl が実行可能であり、--version オプションに応答すること。

アルゴリズムまたは振る舞い

1. cluster->bindir を使用して pg_ctl --version コマンド文字列を構築する。 2. popen を使用してコマンドを実行し、標準出力を読み取る。 3. 出力文字列からメジャーバージョンおよびマイナーバージョンを解析する。 4. バージョン番号に基づき、以下のルールで cluster->bin_version を算出する:

  • バージョン 10 未満: v1 * 10000 + v2 * 100 （例: 9.6.1 → 90600）
  • バージョン 10 以上: v1 * 10000 （例: 10.1 → 100000）

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/exec.c 内の関数から呼び出される（直接の呼び出し箇所として src/bin/pg_upgrade/exec.c:404 が確認されている）。
  • **呼び出し先**: popen, fgets, pclose, sscanf, pg_fatal （エラー時）。

関連データ構造

  • ClusterInfo: pg_upgrade.h で定義。バイナリパス (bindir) とバージョン情報 (bin_version) を保持する。

失敗モードと境界条件

  • popen が失敗した場合、または pg_ctl の実行結果が空の場合、pg_fatal によりプロセスが終了する。
  • pclose の戻り値が 0 以外の場合、pg_fatal によりプロセスが終了する。
  • sscanf によるバージョン解析に失敗した場合、pg_fatal によりプロセスが終了する。

保守担当者向けメモ

  • 本関数は PostgreSQL のバージョン表記形式の変更（10以降のメジャーバージョンのみの扱い）を吸収するロジックを含んでいる。
  • Windows 環境では popen の挙動に注意が必要だが、本関数自体は標準的なパイプ処理に依存している。
