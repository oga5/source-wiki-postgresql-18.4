varatt_indirect

定義

  • 種別: struct
  • 定義箇所: src/include/varatt.h:57

目的

メモリ上に存在する varlena 構造体への間接参照を保持する「TOASTポインタ」の一種です。データがインラインではなく、メモリ上の別の場所に格納されていることを示すために使用されます。

シグネチャまたは形状

typedef struct varatt_indirect
{
    struct varlena *pointer;    /* Pointer to in-memory varlena */
} varatt_indirect;

入力と出力

  • **入力**: varlena 構造体へのポインタ。
  • **出力**: なし（構造体としてメモリ上に保持される）。

前提条件と不変条件

  • pointer メンバは、有効なメモリ上の varlena 構造体を指している必要があります。
  • 本構造体は、TOASTポインタとして扱われる際、適切なタグ（VARTAG_INDIRECT）を伴って識別される必要があります。

アルゴリズムまたは振る舞い

本構造体はデータそのものではなく、データへの参照を保持します。システムがこの構造体に遭遇した際、pointer が指すメモリ位置へアクセスすることで、実際の可変長データ（varlena）を取得します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/common/detoast.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/test/regress/regress.c
  • src/include/access/detoast.h
  • src/include/varatt.h
  • **呼び出し先**:
  • struct varlena（pointer メンバを通じて参照）

関連データ構造

  • varlena: 可変長データの基本構造体。
  • vartag_external: TOASTポインタの種類を識別する列挙型（VARTAG_INDIRECT を含む）。

失敗モードと境界条件

  • **無効なポインタ**: pointer が指すメモリ領域が解放済みである場合、アクセス時にセグメンテーション違反が発生します。
  • **アライメント**: varatt_indirect 自体は構造体ですが、これを格納するコンテキストにおいてアライメントが保証されない場合、直接的なキャストアクセスは避ける必要があります（memcpy 等の利用が推奨されます）。

保守担当者向けメモ

  • 本構造体は src/include/varatt.h で定義されており、TOAST機構のメモリ内間接参照を担います。
  • 構造体へのアクセス時は、必ず VARTAG_INDIRECT タグによる形式確認を経てから行うようにしてください。
