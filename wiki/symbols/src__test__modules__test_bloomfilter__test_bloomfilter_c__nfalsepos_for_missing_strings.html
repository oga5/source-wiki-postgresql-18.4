nfalsepos_for_missing_strings

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_bloomfilter/test_bloomfilter.c:51

目的

ブルームフィルタ内に存在しないことが保証された一連の文字列を検査し、フィルタが誤って「存在あり」と判定した回数（偽陽性数）をカウントする。

シグネチャまたは形状

static int64 nfalsepos_for_missing_strings(bloom_filter *filter, int64 nelements)

入力と出力

  • **入力**:
  • filter: 検査対象の bloom_filter 構造体へのポインタ。
  • nelements: 検査する文字列の総数。
  • **出力**:
  • int64: 偽陽性として検出された回数の合計。

前提条件と不変条件

  • filter は初期化済みであること。
  • 検査対象の文字列は "M" + 数値 という形式で生成される。
  • ループ実行中に CHECK_FOR_INTERRUPTS() が呼び出されるため、PostgreSQLの割り込み処理が有効であること。

アルゴリズムまたは振る舞い

1. 0 から nelements - 1 までの範囲でループを実行する。 2. 各イテレーションで CHECK_FOR_INTERRUPTS() を呼び出し、処理の中断要求を確認する。 3. snprintf を使用して "M" + i という形式の文字列を生成する。 4. bloom_lacks_element を呼び出し、フィルタ内に要素が存在しないかを確認する。 5. bloom_lacks_element が偽（要素が存在すると判定）を返した場合、nfalsepos をインクリメントする。 6. 全ループ終了後、集計した nfalsepos を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: create_and_test_bloom (同ファイル内)
  • **呼び出し先**:
  • CHECK_FOR_INTERRUPTS()
  • snprintf()
  • bloom_lacks_element()
  • strlen()

関連データ構造

  • bloom_filter: src/include/lib/bloomfilter.h で定義されているブルームフィルタの構造体。

失敗モードと境界条件

  • nelements が非常に大きい場合、ループ処理に時間がかかるが、CHECK_FOR_INTERRUPTS() によりSQL側からのキャンセルが可能。
  • element バッファ（MAX_ELEMENT_BYTES）を超える文字列が生成された場合、snprintf の挙動に依存する。

保守担当者向けメモ

  • 本関数はテスト専用であり、ブルームフィルタの統計的精度を検証するために使用される。
  • 偽陽性率は確率的な事象であるため、テスト結果が閾値付近で変動する可能性がある。
