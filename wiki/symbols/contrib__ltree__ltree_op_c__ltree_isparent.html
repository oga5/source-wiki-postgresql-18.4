ltree_isparent

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:236

目的

2つの ltree 型の値を受け取り、最初の引数が2番目の引数の親（または祖先）であるかどうかを判定する SQL 呼び出し可能な関数です。

シグネチャまたは形状

Datum ltree_isparent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの ltree ポインタを受け取ります。
  • 第1引数（インデックス 0）: 親パス (p)
  • 第2引数（インデックス 1）: 子パス (c)
  • **出力**: bool 型の Datum を返します。親パスが子パスの先頭部分と一致する場合は true、それ以外は false です。

前提条件と不変条件

  • 入力される ltree は有効な構造体である必要があります。
  • PostgreSQL のメモリ管理規則に従い、PG_GETARG_LTREE_P で取得したポインタは、必要に応じて PG_FREE_IF_COPY で解放されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_LTREE_P マクロを使用して、引数から ltree 構造体へのポインタを取得します。 2. inner_isparent(c, p) を呼び出し、子パス c が親パス p で始まるかを判定します。 3. PG_FREE_IF_COPY を使用して、コピーされた引数メモリを適切に解放します。 4. PG_RETURN_BOOL を介して判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/_ltree_op.c 内の array_iterator から呼び出されます。
  • **呼び出し先**:
  • inner_isparent (親子判定のコアロジック)

関連データ構造

  • ltree: 階層的なラベルパスを保持する PostgreSQL の可変長データ型。

失敗モードと境界条件

  • 入力が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で処理される必要があります。
  • inner_isparent の内部ロジックに依存し、パスの長さやラベルの不一致がある場合は false を返します。

保守担当者向けメモ

  • 本関数は SQL インターフェース層であり、実際の比較ロジックは inner_isparent に集約されています。ロジックの変更が必要な場合は inner_isparent を参照してください。
  • PG_FREE_IF_COPY の使用は、メモリリークを防ぐために必須です。修正を加える際は、引数のコピー有無を考慮してください。
