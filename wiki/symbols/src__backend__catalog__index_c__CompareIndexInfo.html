CompareIndexInfo

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/index.c:2536

目的

2つの IndexInfo 構造体が、指定された属性マップ（attmap）および照合順序・演算子族の定義において論理的に等価であるかを判定します。

シグネチャまたは形状

bool CompareIndexInfo(const IndexInfo *info1, const IndexInfo *info2,
                      const Oid *collations1, const Oid *collations2,
                      const Oid *opfamilies1, const Oid *opfamilies2,
                      const AttrMap *attmap)

入力と出力

  • **入力**: 比較対象の IndexInfo 構造体2つ、それぞれの照合順序（collations）と演算子族（opfamilies）の配列、および属性番号の変換用 attmap。
  • **出力**: 両者が等価であれば true、そうでなければ false を返します。

前提条件と不変条件

  • info1 と info2 は有効な IndexInfo 構造体を指している必要があります。
  • attmap は、info2 の属性番号を info1 のコンテキストに変換するために使用されます。
  • ii_ExclusionOps が設定されているインデックスの比較には対応していません。

アルゴリズムまたは振る舞い

1. 基本属性（一意性、Null非区別、アクセスメソッド、属性数、キー属性数）を比較し、不一致があれば直ちに false を返します。 2. 各インデックス属性を走査し、attmap を用いて属性番号を照合します。式（InvalidAttrNumber）が含まれる場合は、両者が共に式であるかを確認します。 3. キー属性については、照合順序と演算子族が一致するかを確認します。 4. インデックス式（ii_Expressions）が存在する場合、map_variable_attnos で変数を変換した上で equal() を用いて構造的な一致を確認します。 5. 部分インデックスの述語（ii_Predicate）が存在する場合も同様に、変換後に equal() で一致を確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • map_variable_attnos
  • equal
  • elog（エラー発生時）

関連データ構造

  • IndexInfo: インデックスの定義情報を保持する構造体。
  • AttrMap: 属性番号の対応関係を保持する構造体。

失敗モードと境界条件

  • attmap->maplen が info2 の属性番号より小さい場合、elog(ERROR) が発生します。
  • どちらか一方のみが式インデックスである場合、または述語の有無が異なる場合は false を返します。
  • ii_ExclusionOps が設定されている場合、比較を放棄し false を返します。
  • map_variable_attnos で found_whole_row が真となった場合、比較不能として false を返します。

保守担当者向けメモ

  • 本関数は、インデックスの論理的な等価性を検証する重要な関数です。特に CONCURRENTLY なインデックス作成やパーティションテーブルのインデックス整合性チェックで使用されます。
  • 比較ロジックに新しいインデックス属性やプロパティを追加した際は、本関数にも反映が必要です。
