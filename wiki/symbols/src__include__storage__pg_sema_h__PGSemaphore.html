PGSemaphore

定義

  • 種別: typedef（struct PGSemaphoreData * へのポインタ）
  • 定義箇所: src/include/storage/pg_sema.h:34

目的

PostgreSQLにおけるプロセス間同期のための、プラットフォーム非依存なセマフォ抽象化ハンドル。OS固有の実装（POSIXセマフォ、System Vセマフォ、Windowsハンドル等）を隠蔽し、バックエンド全体で一貫した同期インターフェースを提供します。

シグネチャまたは形状

typedef struct PGSemaphoreData *PGSemaphore;

※Windows環境ではネイティブな HANDLE にマッピングされます。

入力と出力

  • 入力: 各セマフォ操作関数（PGSemaphoreLock, PGSemaphoreUnlock 等）の引数として渡されます。
  • 出力: PGSemaphoreCreate 等の生成関数により、初期化されたハンドルが返されます。

前提条件と不変条件

  • PGSemaphoreData は不透明な構造体として扱われ、プラットフォーム非依存コードは内部構造に直接アクセスしてはなりません。
  • Windows環境では他のプラットフォームとABI互換性がありません。
  • 使用前に PGReserveSemaphores によるシステム全体のリソース確保と、PGSemaphoreCreate によるインスタンスの初期化が必要です。

アルゴリズムまたは振る舞い

  • **抽象化**: OS固有のセマフォ実装をラップし、統一されたAPIを提供します。
  • **同期操作**:
  • PGSemaphoreLock: カウントをデクリメントし、0未満の場合はブロックします（P操作）。
  • PGSemaphoreUnlock: カウントをインクリメントします（V操作）。
  • PGSemaphoreTryLock: ブロックせずにロックを試み、成功時に true を返します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • src/backend/port/posix_sema.c: セマフォの初期化、リセット、ロック、アンロックの実装内で使用。
  • 呼び出し先:
  • 供給されたソース情報には、本シンボルが呼び出す具体的な関数や、本シンボルを呼び出す上位モジュールの詳細なリストは含まれていません。

関連データ構造

  • PGSemaphoreData: セマフォの実体。
  • LWLock: 軽量ロック。内部的にセマフォの概念を利用することがあります。
  • Latch: 高レベルな待機イベント管理。現代のPostgreSQLではセマフォよりも優先される傾向があります。

失敗モードと境界条件

  • デッドロックのリスク: 低レベルなプリミティブであるため、不適切な使用はデッドロックを招く可能性があります。
  • プラットフォーム依存性: Windows環境における HANDLE へのマッピングにより、プラットフォーム間でのバイナリ互換性が欠如しています。

保守担当者向けメモ

  • セマフォは低レベルな同期機構です。可能な限り LWLock や Latch の使用を検討してください。
  • プラットフォーム固有の実装は src/backend/storage/ipc/pg_sema.c に集約されています。修正時はこのファイルを確認してください。
