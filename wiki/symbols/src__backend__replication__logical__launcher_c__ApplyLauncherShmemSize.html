ApplyLauncherShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/launcher.c:908

目的

論理レプリケーションのランチャープロセスが共有メモリ上で使用する領域の合計サイズを算出する。

シグネチャまたは形状

Size ApplyLauncherShmemSize(void)

入力と出力

  • 入力: なし（グローバル変数 max_logical_replication_workers を参照）
  • 出力: 必要な共有メモリサイズ（Size 型）

前提条件と不変条件

  • max_logical_replication_workers が正しく設定されていること。
  • LogicalRepCtxStruct および LogicalRepWorker 構造体のサイズが固定されていること。

アルゴリズムまたは振る舞い

1. LogicalRepCtxStruct のサイズを確保する。 2. MAXALIGN を用いてアライメントを調整する。 3. max_logical_replication_workers 分の LogicalRepWorker 配列に必要なサイズを add_size および mul_size を用いて加算する。 4. 計算された合計サイズを返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/ipc/ipci.c: 共有メモリ全体のサイズ計算時に呼び出される。
  • src/backend/replication/logical/launcher.c: ApplyLauncherShmemInit 内で初期化サイズとして使用される。
  • 呼び出し先:
  • add_size, mul_size: PostgreSQLのメモリサイズ計算ユーティリティ。

関連データ構造

  • LogicalRepCtxStruct: ランチャーの共有メモリコンテキスト。
  • LogicalRepWorker: 個別の論理レプリケーションワーカーの状態管理構造体。

失敗モードと境界条件

  • max_logical_replication_workers が極端に大きい場合、mul_size によりオーバーフローが発生する可能性があるが、PostgreSQLのメモリ計算関数がこれを保護する。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウト変更（構造体の追加など）を行う際に必ず更新する必要がある。
  • LogicalRepWorker のサイズ変更時も同様に影響を受ける。
