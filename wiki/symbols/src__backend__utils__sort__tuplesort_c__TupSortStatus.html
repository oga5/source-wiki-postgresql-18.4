TupSortStatus

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/sort/tuplesort.c:154

目的

PostgreSQL のタプルソート処理における現在の進行状況およびソート状態を管理・追跡するために使用されます。

シグネチャまたは形状

typedef enum
{
    TSS_INITIAL,        /* メモリ制限内でタプルを読み込み中 */
    TSS_BOUNDED,        /* 境界付きヒープにタプルを読み込み中 */
    TSS_BUILDRUNS,      /* タプルを読み込み中、テープへ書き出し中 */
    TSS_SORTEDINMEM,    /* メモリ内でのソートが完了 */
    TSS_SORTEDONTAPE,   /* ソート完了、最終ランがテープ上にある */
    TSS_FINALMERGE,     /* オンザフライで最終マージを実行中 */
} TupSortStatus;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: Tuplesortstate 構造体内の status フィールドおよび maxSpaceStatus フィールドとして保持されます。

前提条件と不変条件

  • Tuplesortstate のライフサイクルを通じて、ソートのフェーズ遷移を正確に反映する必要があります。
  • 外部ソートへ移行する際（TSS_BUILDRUNS）は、メモリ制限（work_mem）の超過がトリガーとなります。

アルゴリズムまたは振る舞い

  • TSS_INITIAL: 初期状態。メモリ内でソート可能なデータ量を処理します。
  • TSS_BOUNDED: tuplesort_begin_common 等で指定された制限付きソートを実行中。
  • TSS_BUILDRUNS: メモリ制限を超過し、一時ファイル（論理テープ）への書き出しを行っている状態。
  • TSS_SORTEDINMEM / TSS_SORTEDONTAPE: ソート処理が完了した状態。
  • TSS_FINALMERGE: 外部マージソートの最終段階。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/sort/tuplesort.c 内の各関数（Tuplesortstate を操作する関数群）。
  • 呼び出し先: なし（列挙型のため）。
  • 注記: 具体的な関数呼び出しの依存関係については、提供されたソース excerpt には詳細な記述がありません。

関連データ構造

  • Tuplesortstate: この列挙型を保持するメインのソート状態構造体。

失敗モードと境界条件

  • 状態遷移の不整合: ソートの進行と現在の TupSortStatus が一致しない場合、メモリ管理やテープ読み書きでエラーが発生する可能性があります。
  • 境界条件: maxSpaceStatus は、メモリ制限に達した瞬間の状態を記録するために使用されます。

保守担当者向けメモ

  • ソートアルゴリズムの変更や新しいソートモードの追加時には、この列挙型に新しい状態を追加し、Tuplesortstate を操作する各関数でのスイッチ文を更新する必要があります。
