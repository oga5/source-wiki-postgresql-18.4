cx

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_cx.c:49

目的

遺伝的アルゴリズム（GEQO）におけるサイクル交叉（Cycle Crossover: CX）を実行し、2つの親個体から子個体を生成する。

シグネチャまたは形状

int cx(PlannerInfo *root, Gene *tour1, Gene *tour2, Gene *offspring, int num_gene, City *city_table)

入力と出力

  • **入力**:
  • root: PlannerInfo 構造体（乱数生成に使用）。
  • tour1, tour2: 親個体の遺伝子配列。
  • num_gene: 遺伝子の総数。
  • city_table: 遺伝子の位置管理用作業領域。
  • **出力**:
  • offspring: 生成された子個体（ポインタ経由で書き込み）。
  • 戻り値: 親個体（tour1）と生成された子個体の間の遺伝子配置の差異数。

前提条件と不変条件

  • city_table は num_gene 分のメモリが確保されていること。
  • tour1 および tour2 は有効な遺伝子配列であること。

アルゴリズムまたは振る舞い

1. **初期化**: city_table を使用して、両親の各遺伝子の位置をマッピングする。 2. **サイクル探索**: ランダムに開始位置を決定し、tour1 の遺伝子を子に継承する。その後、tour2 の対応する位置にある遺伝子を辿り、閉路（サイクル）が完成するまで tour1 の遺伝子を子にコピーし続ける。 3. **補完**: サイクルに含まれなかった残りの遺伝子を tour2 からコピーし、個体を完成させる。 4. **評価**: 生成された子個体と tour1 を比較し、差異の数をカウントして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は本抜粋からは特定できません（ツールが提示した pgcrypto 関連の cx は本関数とは無関係な別シンボルです）。
  • **呼び出し先**: geqo_randint（乱数生成）。

関連データ構造

  • Gene: 遺伝子を表す型。
  • City: 遺伝子の位置情報（tour1_position, tour2_position）および使用フラグ（used）を保持する構造体。

失敗モードと境界条件

  • サイクル探索後に count < num_gene となる場合、残りの遺伝子を tour2 から補完する処理を行う。
  • 補完後も count < num_gene の場合は、差異のカウント処理へ進む。

保守担当者向けメモ

  • 本関数は #if defined(CX) で囲まれており、ビルド時に有効化されている場合のみコンパイルされる。
  • Oliver et al. によるサイクル交叉アルゴリズムの実装である。
