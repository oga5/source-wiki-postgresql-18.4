buffers_to_iovec

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/md.c:784

目的

複数のバッファポインタを、連続するメモリ領域ごとにグループ化し、ベクトルI/O（iovec）構造体の配列に変換する。

シグネチャまたは形状

static int buffers_to_iovec(struct iovec *iov, void **buffers, int nblocks)

入力と出力

  • **入力**:
  • iov: iovec 構造体の配列（呼び出し元で十分なサイズを確保する必要がある）。
  • buffers: バッファポインタの配列。
  • nblocks: 処理するバッファの数。
  • **出力**:
  • 戻り値: 使用された iovec 構造体の数（iovcnt）。

前提条件と不変条件

  • nblocks >= 1 であること。
  • ダイレクトI/Oが有効かつ PG_IO_ALIGN_SIZE <= BLCKSZ の場合、すべてのバッファポインタは PG_IO_ALIGN_SIZE にアライメントされている必要がある。

アルゴリズムまたは振る舞い

1. 最初のバッファを最初の iovec 要素として設定する。 2. 残りのバッファを順次走査し、前のバッファとメモリ上で連続しているかを確認する。 3. 連続している場合は、現在の iovec 要素の長さを BLCKSZ 分だけ加算する。 4. 連続していない場合は、新しい iovec 要素を作成し、ポインタと長さを設定する。 5. 最終的な iovec の個数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/smgr/md.c 内の複数箇所（mdreadv や mdwritev 等の文脈で呼び出される）。
  • **呼び出し先**:
  • 特になし（標準的なポインタ演算とアサーションのみ）。

関連データ構造

  • struct iovec: POSIX で定義されたベクトルI/O用構造体。
  • BLCKSZ: PostgreSQL のブロックサイズ定数。

失敗モードと境界条件

  • nblocks が 1 未満の場合、Assert により異常終了する。
  • ダイレクトI/O環境下でアライメントが不適切な場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は BLCKSZ 単位での連続性を前提としている。
  • iov 配列のサイズが nblocks 以上であることを呼び出し元が保証する必要がある。
