pg_re_flags

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/regexp.c:45

目的

SQLレベルの正規表現関数において、コンパイルオプションおよび実行時のグローバルマッチングフラグを保持するために使用されます。

シグネチャまたは形状

typedef struct pg_re_flags
{
	int			cflags;			/* compile flags for Spencer's regex code */
	bool		glob;			/* do it globally (for each occurrence) */
} pg_re_flags;

入力と出力

  • cflags: Spencer's regexパッケージで使用されるコンパイルフラグ（整数）。
  • glob: マッチングをグローバル（全出現箇所）に行うかどうかを示す真偽値。

前提条件と不変条件

  • cflags は regex/regex.h で定義されたPOSIX準拠の正規表現エンジンが解釈可能な値である必要があります。

アルゴリズムまたは振る舞い

本構造体は、SQLのフラグ文字列（'g', 'i' 等）を解析する parse_re_flags 関数によって初期化され、正規表現のコンパイルやマッチング処理を行う各関数へ渡されます。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_re_flags および正規表現を利用する各種SQL関数（regexp_matches 等のバックエンド実装）内でローカル変数として宣言・利用されます。
  • **呼び出し先**: setup_regexp_matches 等の正規表現実行関数へ引数として渡されます。

関連データ構造

  • regex/regex.h: cflags が参照する正規表現エンジンの定義元。

失敗モードと境界条件

  • 構造体自体に失敗モードはありませんが、不正な cflags が設定された場合、後続の pg_regcomp 等の正規表現エンジン呼び出しが失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は regexp.c 内の正規表現処理関数のインターフェースとして多用されています。フラグの追加や変更を行う際は、parse_re_flags による解析ロジックとの整合性に注意してください。
