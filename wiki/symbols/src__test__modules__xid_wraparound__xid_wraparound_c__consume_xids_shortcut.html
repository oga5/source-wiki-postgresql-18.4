consume_xids_shortcut

定義

  • 種別: function
  • 定義箇所: src/test/modules/xid_wraparound/xid_wraparound.c:199

目的

XidGenLock を排他取得し、トランザクションID（XID）カウンタをSLRUのページ境界を考慮した安全な範囲で一括して進めることで、XID消費を高速化します。

シグネチャまたは形状

static int64 consume_xids_shortcut(void)

入力と出力

  • 入力: なし
  • 出力: int64 型。実際にスキップ（消費）されたXIDの数を返します。

前提条件と不変条件

  • XidGenLock を排他ロックするため、呼び出し中は他のトランザクションの進行がブロックされます。
  • XidSkip 関数により、SLRUページ境界（CLOGやSUBTRANS等）をまたがない安全な範囲でのみ消費が行われます。

アルゴリズムまたは振る舞い

1. XidGenLock を LW_EXCLUSIVE モードで取得します。 2. 現在の nextXid を取得します。 3. XidSkip(nextXid) を呼び出し、SLRU境界を考慮してスキップ可能な距離を計算します。 4. 計算された距離が0より大きい場合、TransamVariables->nextXid をその分だけ加算します。 5. XidGenLock を解放し、消費したXID数を返します。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/xid_wraparound/xid_wraparound.c 内の consume_xids_common 関数（122行目）から呼び出されます。
  • 呼び出し先: LWLockAcquire, LWLockRelease, XidSkip を呼び出します。

関連データ構造

  • TransamVariables: nextXid を保持するグローバルなトランザクション変数構造体。
  • XidGenLock: XID生成を制御するための軽量ロック。

失敗モードと境界条件

  • XidSkip が0を返した場合、SLRU境界に近いためスキップは行われません。
  • 本関数はテスト専用であり、本番環境で実行するとXID周回を早め、データベースの停止リスクを伴います。

保守担当者向けメモ

  • XidSkip 内の定数はPostgreSQLの内部構造（SLRUページサイズ等）に依存しているため、バージョンアップ時にロジックの妥当性を再確認してください。
  • ショートカット実行時は排他ロックを保持するため、長時間実行されないよう注意が必要です。
