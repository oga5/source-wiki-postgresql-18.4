gbt_numeric_compress

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:96

目的

GiST インデックス構築時、numeric 型のデータをインデックス格納に適した形式に圧縮します。

シグネチャまたは形状

Datum gbt_numeric_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、圧縮されたインデックスエントリを返します。

前提条件と不変条件

  • tinfo（gbtree_vinfo 構造体）が、numeric 型の比較関数群を保持する有効な設定情報として定義されている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を使用して、入力された GISTENTRY を取得します。 2. btree_utils_var.c で定義されている汎用関数 gbt_var_compress を呼び出します。 3. その際、numeric 型固有の比較ロジックを保持する tinfo を引数として渡し、可変長データ型の圧縮処理を委譲します。 4. 処理結果を PG_RETURN_POINTER で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセス（直接的な呼び出し元コードはソース内に明示されていません）。
  • **呼び出し先**: gbt_var_compress（btree_utils_var.c で定義）。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • tinfo: gbtree_vinfo 型。numeric 型の比較関数群を格納し、汎用関数へ渡すための設定情報。

失敗モードと境界条件

  • 汎用関数 gbt_var_compress の仕様に依存します。

保守担当者向けメモ

  • 本関数は btree_utils_var.h で定義された汎用関数を呼び出すためのアダプターです。
  • numeric 型特有の圧縮ロジックを追加する必要がある場合は、tinfo の設定内容を確認してください。
