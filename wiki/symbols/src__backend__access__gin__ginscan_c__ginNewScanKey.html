ginNewScanKey

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginscan.c:268

目的

GINインデックススキャンにおいて、ユーザーから提供された ScanKey を解析し、インデックス走査に必要な内部構造体 GinScanKey へ変換・最適化する。

シグネチャまたは形状

void ginNewScanKey(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan（インデックススキャンの状態記述子）
  • **出力**: なし（副作用として scan->opaque が指す GinScanOpaque 構造体内の keys および entries 配列を構築・更新する）

前提条件と不変条件

  • scan->opaque は GinScanOpaque 型として初期化済みであること。
  • so->keyCtx（メモリコンテキスト）が有効であること。
  • GINインデックスのバージョンが古い場合、全インデックススキャンやNULL検索は拒否される（ソース内のコメントによる）。

アルゴリズムまたは振る舞い

1. **初期化**: so->keyCtx に切り替え、GinScanKey 配列および GinScanEntry ポインタ配列を確保する。 2. **クエリ解析**: 各 ScanKey に対して extractQueryFn を呼び出し、検索値、検索モード、およびメタデータを取得する。 3. **検索モードの適用**: GIN_SEARCH_MODE_DEFAULT 以外のモード（ALL 等）が指定された場合、インデックス走査用のプレースホルダが必要と判断する。 4. **キー構築**: ginFillScanKey を呼び出し、各属性に対する GinScanEntry を生成する。 5. **後処理（再スキャン）**: GIN_SEARCH_MODE_ALL を持つキーに対し、必要に応じて GIN_CAT_EMPTY_QUERY を持つ隠しエントリを追加する。 6. **順序調整**: excludeOnly なキーを配列の末尾に移動し、通常のキーが先に処理されるよう再配置する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginget.c
  • **呼び出し先**: extractQueryFn（ginstate 経由）、ginFillScanKey、ginScanKeyAddHiddenEntry

関連データ構造

  • IndexScanDesc: インデックススキャンの全体状態。
  • GinScanOpaque: GIN固有のスキャン状態（keys, entries, keyCtx 等を保持）。
  • GinScanKey: 解析された検索キーの内部表現。
  • GinScanEntry: 検索キーから分解された個別のインデックスエントリ。

失敗モードと境界条件

  • sk_flags に SK_ISNULL が含まれる場合、検索結果は空（isVoidRes = true）として処理される。
  • extractQueryFn が無効な値を返した場合、検索モードを GIN_SEARCH_MODE_ALL に強制補正する。
  • 検索キーが存在しない場合、デフォルトモードであれば検索結果を空とみなす。

保守担当者向けメモ

  • extractQueryFn がメモリリークを起こした場合、そのメモリはスキャン終了または再スキャン時まで解放されない。
  • excludeOnly キーの再配置ロジックは、インデックススキャンの正しい実行順序を保証するために不可欠である。
  • numExcludeOnly < so->nkeys のアサーションは、少なくとも1つの通常キーが存在することを前提としている。
