fsm_get_location

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/freespace.c:500

目的

指定されたヒープブロック番号に対応する FSM（Free Space Map）の論理アドレス（FSMAddress）と、そのページ内でのスロット位置を算出する。

シグネチャまたは形状

static FSMAddress fsm_get_location(BlockNumber heapblk, uint16 *slot)

入力と出力

  • **入力**:
  • heapblk: 対象となるヒープのブロック番号。
  • slot: スロット番号を格納するためのポインタ（出力用）。
  • **出力**:
  • 戻り値: FSMAddress 構造体。level は常に FSM_BOTTOM_LEVEL に設定され、logpageno には計算された論理ページ番号が格納される。

前提条件と不変条件

  • FSM_BOTTOM_LEVEL が定義されていること。
  • SlotsPerFSMPage がゼロでないこと（除算に使用されるため）。

アルゴリズムまたは振る舞い

1. 戻り値の addr.level を FSM_BOTTOM_LEVEL に初期化する。 2. heapblk を SlotsPerFSMPage で除算し、その商を addr.logpageno に代入する。 3. heapblk を SlotsPerFSMPage で剰余演算し、その結果を *slot に書き込む。 4. 計算された addr を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: freespace.c 内の複数の関数（RecordPageWithFreeSpace, GetRecordedFreeSpace, FreeSpaceMapPrepareTruncateRel 等）から、ヒープブロックに対応する FSM 情報を特定するために呼び出される。
  • **呼び出し先**: 外部関数への直接的な呼び出しはない（算術演算のみ）。

関連データ構造

  • FSMAddress: FSM 内の階層レベルと論理ページ番号を保持する構造体。
  • BlockNumber: PostgreSQL のブロック番号型。

失敗モードと境界条件

  • SlotsPerFSMPage が 0 の場合、ゼロ除算が発生する可能性がある（通常は定数として定義されているため発生しない）。

保守担当者向けメモ

  • 本関数は FSM の最下層（リーフ）レベルのアドレスを計算する専用のロジックである。
  • 階層構造の変更や SlotsPerFSMPage の定義変更時には、この計算ロジックが整合しているか確認が必要である。
