目的

pg_test_timing ツールによる計測結果（時刻取得にかかった時間の分布）を、ヒストグラムとして標準出力に整形・表示する。

シグネチャまたは形状

static void output(uint64 loop_count)

入力と出力

  • **入力**: loop_count（計測ループの総実行回数）。
  • **出力**: なし（標準出力へのフォーマット済みテキスト出力のみ）。

前提条件と不変条件

  • グローバル変数 histogram[] が計測ループによって適切に更新されていること。
  • loop_count が 0 でないこと（0 の場合、除算により計算結果が不正になる可能性がある）。

アルゴリズムまたは振る舞い

1. **範囲決定**: histogram 配列を走査し、値が格納されている最大のビット位置（max_bit）を特定する。 2. **ヘッダー出力**: 「< us」「% of total」「count」の各列見出しを表示する。 3. **データ出力**: 0 から max_bit までの各バケットについて以下を表示する。

  • 1l << i: マイクロ秒単位の閾値。
  • (double) histogram[i] * 100 / loop_count: 全体に対する割合（パーセント）。
  • histogram[i]: そのバケットに該当した回数。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の文脈から、main 関数から呼び出されることが示唆される（ただし、提供されたソース excerpt には明示的な呼び出し箇所は含まれていない）。
  • **呼び出し先**: printf、strlen、Max マクロ。
  • **注意**: ツールが提示した他の output という名称のシンボル（contrib/ 配列内のものなど）は、本関数とは無関係な別個のシンボルである。

関連データ構造

  • histogram[]: 時刻差分を 2 のべき乗単位で分類して保持するグローバル配列。

失敗モードと境界条件

  • loop_count が 0 の場合、割合の計算でゼロ除算が発生する可能性がある。
  • max_bit が 0 の場合、ループは 1 回のみ実行される。

保守担当者向けメモ

  • 本関数は pg_test_timing.c 専用のローカル関数である。
  • 出力フォーマットの調整が必要な場合は、Max マクロによる幅指定と printf の書式指定子を確認すること。
  • histogram 配列のインデックスは 2 のべき乗（マイクロ秒）に対応しているため、変更時は計測側のロジックとの整合性に注意が必要。
