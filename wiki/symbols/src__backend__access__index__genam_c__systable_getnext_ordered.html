systable_getnext_ordered

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:731

目的

順序付きシステムカタログスキャンにおいて、指定された方向に次のタプルを取得する。

シグネチャまたは形状

HeapTuple systable_getnext_ordered(SysScanDesc sysscan, ScanDirection direction)

入力と出力

  • **入力**:
  • sysscan: スキャン状態を保持する SysScanDesc 構造体。
  • direction: スキャン方向（ForwardScanDirection 等）。
  • **出力**:
  • 次の HeapTuple。スキャンが終了した場合は NULL を返す。

前提条件と不変条件

  • sysscan->irel（インデックスリレーション）が有効であること（Assert により検証）。
  • 損失のあるインデックス条件（lossy index conditions）はサポートされていない。該当する場合、エラーが発生する。

アルゴリズムまたは振る舞い

1. index_getnext_slot を呼び出し、インデックススキャンを通じて次のタプルをスロットに取得する。 2. 取得に成功した場合、ExecFetchSlotHeapTuple を用いてスロットから HeapTuple を抽出する。 3. インデックスの再チェックが必要な場合（xs_recheck が真）、エラーを投げる。 4. HandleConcurrentAbort を呼び出し、論理ストリーミング中の同時実行アボートを処理する。 5. 取得したタプルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • src/backend/access/common/toast_internals.c
  • src/backend/access/heap/heaptoast.c
  • src/backend/storage/large_object/inv_api.c
  • src/backend/utils/adt/enum.c
  • **呼び出し先**:
  • index_getnext_slot
  • ExecFetchSlotHeapTuple
  • HandleConcurrentAbort

関連データ構造

  • SysScanDesc: システムカタログスキャンの状態を管理する構造体。
  • HeapTuple: 返されるタプルの型。

失敗モードと境界条件

  • **エラー**: インデックス条件が損失（lossy）である場合、elog(ERROR) により処理が中断される。
  • **終了条件**: スキャン対象がなくなった場合、NULL を返す。

保守担当者向けメモ

  • 本関数は順序付きスキャン専用である。順序が不要な場合は systable_getnext を使用すること。
  • HandleConcurrentAbort の呼び出しは、論理レプリケーション等のストリーミング処理における整合性維持のために必須である。
