combo_decrypt

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px.c:225

目的

PX_Combo オブジェクトを使用して、指定されたデータに対して復号処理を実行する。

シグネチャまたは形状

static int combo_decrypt(PX_Combo *cx, const uint8 *data, unsigned dlen, uint8 *res, unsigned *rlen);

入力と出力

  • cx: 復号に使用する PX_Combo 構造体へのポインタ。
  • data: 復号対象の入力データバッファ。
  • dlen: 入力データの長さ。
  • res: 復号結果を格納する出力バッファ。
  • rlen: 復号されたデータの実際の長さが書き込まれる出力用ポインタ。
  • 戻り値: 成功時に 0、失敗時にエラーコードを返す。

前提条件と不変条件

  • cx は初期化済みである必要がある。
  • res バッファは、復号後のデータを受け取るために十分なサイズを確保している必要がある。

アルゴリズムまたは振る舞い

PX_Combo 構造体内部の cipher（暗号アルゴリズム）と padding（パディング方式）を参照し、px_cipher_decrypt を呼び出して復号処理を委譲する。

呼び出し元と呼び出し先

  • 呼び出し先: px_cipher_decrypt
  • 呼び出し元: contrib/pgcrypto/px.c 内の PX_Combo 構造体初期化処理（cx->decrypt = combo_decrypt;）にて関数ポインタとして割り当てられる。これ以外の直接的な呼び出し元はソースコードから特定できない。

関連データ構造

  • PX_Combo: 暗号アルゴリズムとパディング方式を保持する構造体。

失敗モードと境界条件

  • 復号処理中に発生したエラーは px_cipher_decrypt の戻り値として伝播される。
  • 入力データ長やパディングの不整合が発生した場合、px_cipher_decrypt の仕様に従いエラーコードを返す。

保守担当者向けメモ

  • 本関数は PX_Combo インターフェースの一部であり、新しい暗号アルゴリズムやパディング方式を追加する際は、このインターフェースとの整合性を維持すること。
