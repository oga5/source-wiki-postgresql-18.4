hstoreUpgrade

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_compat.c:235

目的

hstore 型の値を読み込む際、旧形式（hstore-old）から現在の形式（hstore-new）へのインプレース変換を行い、整合性を確保した上で正しい形式の HStore ポインタを返します。

シグネチャまたは形状

HStore *hstoreUpgrade(Datum orig)

入力と出力

  • **入力**: Datum orig（hstore 型のデータ）。
  • **出力**: 変換済み、または検証済みの HStore *。

前提条件と不変条件

  • 旧形式と新形式のエントリサイズが同一である必要があります。
  • HS_FLAG_NEWVERSION フラグが設定されている場合は、変換を行わず即座に返却されます。
  • 変換が必要な場合、書き込み可能なメモリ領域が確保されます。

アルゴリズムまたは振る舞い

1. **デトースト**: PG_DETOAST_DATUM を実行し、必要に応じて書き込み可能なコピーを作成します。 2. **高速パス**: HS_FLAG_NEWVERSION が確認できればそのまま返却します。 3. **検証**: hstoreValidNewFormat および hstoreValidOldFormat を使用してデータの妥当性を判定します。 4. **変換処理**:

  • 旧形式と判定された場合、HOldEntry から HEntry へのマッピングを再計算し、メモリ上で上書き変換を行います。
  • 曖昧なケース（旧形式と新形式の両方として解釈可能）では、ビルド設定（HSTORE_IS_HSTORE_NEW）に基づき解決し、警告を出力します。

    • 5. **完了**: HS_SETCOUNT および HS_FIXSIZE を適用し、正しい形式として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore.h（DatumGetHStoreP マクロ経由）
  • contrib/hstore_plperl/hstore_plperl.c
  • contrib/hstore_plpython/hstore_plpython.c
  • **呼び出し先**:
  • hstoreValidNewFormat
  • hstoreValidOldFormat
  • PG_DETOAST_DATUM / PG_DETOAST_DATUM_COPY
  • HS_SETCOUNT / HS_FIXSIZE（マクロ）

関連データ構造

  • HStore: メインのデータ構造。
  • HEntry: 新形式のエントリ構造体。
  • HOldEntry: 旧形式のエントリ構造体。

失敗モードと境界条件

  • **エラー**: valid_old が偽かつ valid_new も偽の場合、elog(ERROR, "invalid hstore value found") を発生させます。
  • **境界条件**: hs->size_ == 0 の場合、変換処理をスキップして即座に返却します。

保守担当者向けメモ

  • 本関数は hstore のアップグレードにおけるゲートウェイです。
  • 曖昧なケースに対する WARNING は、将来的に DEBUG1 へ格下げされる可能性があります。
  • 変換ロジックは、旧形式と新形式のエントリサイズが一致するという前提に強く依存しています。
