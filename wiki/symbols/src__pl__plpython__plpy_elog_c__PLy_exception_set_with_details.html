PLy_exception_set_with_details

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:521

目的

PostgreSQL の ErrorData 構造体に含まれる詳細なエラー情報を、Python の例外オブジェクトの属性としてマッピングし、Python インタプリタ上で例外として送出する。

シグネチャまたは形状

void PLy_exception_set_with_details(PyObject *excclass, ErrorData *edata)

入力と出力

  • **入力**:
  • excclass: 送出する Python 例外クラス（PyObject*）。
  • edata: PostgreSQL のエラー情報を含む ErrorData 構造体へのポインタ。
  • **出力**: なし（戻り値は void）。成功時は Python の例外状態がセットされ、失敗時は elog(ERROR, ...) により PostgreSQL 側でエラーが報告される。

前提条件と不変条件

  • edata は有効な ErrorData 構造体を指している必要がある。
  • excclass は Python の例外クラスとして有効である必要がある。
  • 内部で呼び出される set_string_attr が失敗した場合、メモリリークを防ぐために failure ラベルへジャンプし、確保したオブジェクトを解放する。

アルゴリズムまたは振る舞い

1. edata->message を引数として Python 例外オブジェクトを生成する。 2. sqlstate、detail、hint、query、および各種データベースオブジェクト名（schema_name, table_name 等）を、set_string_attr を用いて例外オブジェクトの属性として順次設定する。 3. すべての属性設定が成功した場合、PyErr_SetObject を呼び出して Python 側に例外をセットする。 4. 最後に args および error オブジェクトの参照カウントを減らす。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_plpymodule.c:547（直接的な呼び出しを確認）
  • **呼び出し先**:
  • Py_BuildValue
  • PyObject_CallObject
  • set_string_attr
  • unpack_sql_state
  • PyErr_SetObject
  • Py_DECREF / Py_XDECREF
  • elog

関連データ構造

  • ErrorData: PostgreSQL のエラー報告用構造体。
  • PyObject: Python のオブジェクト表現。

失敗モードと境界条件

  • Py_BuildValue や PyObject_CallObject が NULL を返した場合、または set_string_attr が false を返した場合は、failure ラベルへ遷移し、PostgreSQL 側で ERROR を発生させる。これは Python 側のメモリ不足や属性設定失敗が原因となる。

保守担当者向けメモ

  • 本関数は PLy_exception_set の拡張版であり、より詳細なエラーコンテキストを Python 側に渡す必要がある場合に利用する。
  • set_string_attr の戻り値チェックを省略するとメモリリークや不正な状態遷移を招くため、新規属性を追加する際は必ずチェックを行うこと。
