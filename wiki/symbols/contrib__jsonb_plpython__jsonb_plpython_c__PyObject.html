PyObject

定義

  • 種別: typedef
  • 定義箇所: contrib/jsonb_plpython/jsonb_plpython.c:16

目的

Python C API におけるすべての Python オブジェクトを表現するための基底型です。PostgreSQL の jsonb 型と Python 型の相互変換において、Python 側のデータ構造を保持するために使用されます。

シグネチャまたは形状

Python C API で定義される不透明な構造体（opaque structure）です。

入力と出力

  • **入力**: Python C API の関数群（PyDict_New, PyList_New 等）を通じて生成されます。
  • **出力**: plpython 拡張の変換関数（PLyObject_FromJsonbContainer 等）が、変換結果としてこの型を返します。

前提条件と不変条件

  • plpython 拡張がロードされている環境下でのみ有効です。
  • Python のメモリ管理ルール（参照カウント）に従う必要があり、使用後は Py_DECREF または Py_XDECREF による解放が必須です。

アルゴリズムまたは振る舞い

本モジュール内では、主に以下の役割を担います。

  • jsonb から変換された Python オブジェクト（辞書、リスト、数値等）のコンテナとして機能します。
  • PLyObject_FromJsonbContainer などの関数を通じて、jsonb の階層構造を Python のオブジェクトツリーへ再帰的にマッピングします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/jsonb_plpython/jsonb_plpython.c 内の変換関数群（PLyObject_FromJsonbContainer 等）がこの型を生成・操作します。
  • **呼び出し先**: Python C API の各種関数。
  • **注記**: 外部モジュール（contrib/hstore_plpython）からの参照も確認されていますが、本ファイル内での具体的な呼び出し関係はソースコードの文脈に依存します。

関連データ構造

  • JsonbContainer: jsonb データを保持する PostgreSQL 側の構造体。
  • JsonbValue: jsonb の個々の要素を表現する構造体。

失敗モードと境界条件

  • **メモリ不足**: Python オブジェクト生成時にメモリが不足した場合、Python C API は例外を発生させます。
  • **参照カウントの不整合**: Py_DECREF の呼び忘れによるメモリリーク、または過剰な解放によるセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本シンボルは Python C API の一部であり、PostgreSQL 側のメモリ管理（palloc 等）と Python 側のメモリ管理（PyObject）が混在する箇所では、特に例外発生時のクリーンアップ（PG_CATCH ブロック内での Py_XDECREF）に注意してください。
