get_collation_actual_version_icu

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_icu.c:569

目的

指定されたICU照合順序（Collation）の実際のバージョン情報を文字列として取得します。

シグネチャまたは形状

char *get_collation_actual_version_icu(const char *collcollate)

入力と出力

  • **入力**: collcollate (ICUロケール名を表す文字列)
  • **出力**: ICUのバージョン番号を含む動的に割り当てられた文字列（呼び出し元で解放が必要）。

前提条件と不変条件

  • collcollate は有効なICUロケール名である必要があります。
  • ICUライブラリが初期化されている必要があります。

アルゴリズムまたは振る舞い

1. pg_ucol_open を呼び出し、指定されたロケールに対応する UCollator オブジェクトを生成します。 2. ucol_getVersion を使用して、照合器のバージョン情報を UVersionInfo 構造体に抽出します。 3. ucol_close を呼び出し、照合器を解放します。 4. u_versionToString を使用して、バージョン情報を文字列バッファに変換します。 5. pstrdup を使用して、結果の文字列をメモリにコピーして返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale.c 内のロケール管理ロジックから呼び出されます。
  • **呼び出し先**:
  • pg_ucol_open (ICU照合器の生成)
  • ucol_getVersion (ICU API)
  • ucol_close (ICU API)
  • u_versionToString (ICU API)
  • pstrdup (PostgreSQLメモリ管理)

関連データ構造

  • UCollator: ICUの照合順序を管理する構造体。
  • UVersionInfo: ICUのバージョン情報を保持する型。

失敗モードと境界条件

  • pg_ucol_open が失敗した場合、内部でエラーが発生し処理が中断される可能性があります。
  • バージョン文字列が U_MAX_VERSION_STRING_LENGTH を超えることはありません。

保守担当者向けメモ

  • 本関数はICUのバージョン管理に依存しています。ICUのアップグレード時にバージョン文字列の形式が変更された場合、影響を受ける可能性があります。
  • 戻り値は palloc されたメモリ領域であるため、呼び出し側でのメモリリークに注意してください。
