OnCommitItem

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/tablecmds.c:115

目的

トランザクション終了時に実行すべきリレーションに対するアクション（OnCommitAction）を管理するためのデータ構造です。

シグネチャまたは形状

typedef struct OnCommitItem
{
    Oid             relid;          /* リレーションのOID */
    OnCommitAction  oncommit;       /* トランザクション終了時のアクション */
    SubTransactionId creating_subid; /* 作成したサブトランザクションID */
    SubTransactionId deleting_subid; /* 削除対象としてマークしたサブトランザクションID */
} OnCommitItem;

入力と出力

  • 入力: relid（対象リレーション）、oncommit（実行アクション）、およびサブトランザクションID。
  • 出力: なし（構造体として状態を保持）。

前提条件と不変条件

  • creating_subid: 現在のトランザクションで作成された場合はそのサブトランザクションID、以前のトランザクションで作成された場合は 0。
  • deleting_subid: 削除要求が保留中の場合はそのサブトランザクションID、それ以外は 0。

アルゴリズムまたは振る舞い

  • トランザクションの進行に伴い、リレーションに対する変更（作成や削除の予約）を追跡します。
  • tablecmds.c 内のリスト管理を通じて、コミット時に不要になったエントリの削除や、保留中のアクションの適用が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/tablecmds.c 内の複数の箇所で、palloc を用いたメモリ確保およびリスト操作（lfirst 等）を通じて参照・更新されています。
  • 呼び出し先: 具体的な関数呼び出し関係は提供されたソース情報からは特定できません。

関連データ構造

  • OnCommitAction（列挙型、アクションの種類を定義）
  • SubTransactionId（サブトランザクション識別子）
  • Oid（リレーション識別子）

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保失敗の可能性があります。
  • 状態不整合: creating_subid と deleting_subid の値が、現在のトランザクション状態と矛盾する場合の挙動については、本構造体定義からは判断できません。

保守担当者向けメモ

  • 本構造体は tablecmds.c 内でリストとして管理されており、トランザクションのコミット/アボート処理と密接に関連しています。
  • 削除処理のロジック（deleting_subid の設定など）を変更する際は、トランザクションのネスト構造との整合性に注意してください。
