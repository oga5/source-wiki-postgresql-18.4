TwoPhaseRecordOnDisk

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/twophase.c:984

目的

準備済みトランザクション（2PC）のデータをディスク上の状態ファイルに書き出す際、各リソースマネージャ（rmgr）が生成したレコードのヘッダーとして機能します。

シグネチャまたは形状

typedef struct TwoPhaseRecordOnDisk
{
    uint32      len;            /* rmgrデータの長さ */
    TwoPhaseRmgrId rmid;        /* このレコードのリソースマネージャID */
    uint16      info;           /* rmgrが使用するフラグビット */
} TwoPhaseRecordOnDisk;

入力と出力

  • 本構造体自体はデータ保持用であり、関数ではありません。
  • len フィールドは、このヘッダー直後に続くrmgrデータのサイズのみを表し、ヘッダー自身のサイズは含みません。

前提条件と不変条件

  • len はヘッダーサイズを含まないデータ長である必要があります。
  • 状態ファイル内のレコード列の最後には、rmid が TWOPHASE_RM_END_ID である終了用レコード（センチネル）が配置されます。

アルゴリズムまたは振る舞い

  • 準備済みトランザクションをディスクへ保存する際、各リソースマネージャのデータは本構造体をヘッダーとして直列化されます。
  • 読み込み時は、bufptr を使用して本構造体をキャストし、MAXALIGN(sizeof(TwoPhaseRecordOnDisk)) を加算することで次のレコード位置を特定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/twophase.c 内の準備済みトランザクションの永続化処理（save_state_data 等）およびリカバリ時の読み込み処理。
  • **呼び出し先**: 特定の関数呼び出し関係はソースから直接特定できません。

関連データ構造

  • TwoPhaseFileHeader: 状態ファイル全体のヘッダー。
  • TwoPhaseRmgrId: リソースマネージャを識別するID。

失敗モードと境界条件

  • MAXALIGN を考慮しないポインタ操作を行うと、アライメント違反やデータ読み込みエラーが発生する可能性があります。
  • 終了用レコード（TWOPHASE_RM_END_ID）が正しく配置されていない場合、リカバリ時の読み込みループが終了せずメモリ破壊やクラッシュを招く恐れがあります。

保守担当者向けメモ

  • len フィールドがヘッダーサイズを含まない点に注意してください。
  • 構造体のサイズを変更する場合は、MAXALIGN を使用したポインタ計算箇所（1325行目、1692行目など）に影響がないか確認が必要です。
