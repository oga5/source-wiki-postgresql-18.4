send_cancellable_query_impl

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:171

目的

テスト対象の接続 (conn) に対して、キャンセル可能な長時間クエリ（pg_sleep）を非同期に送信し、そのクエリが実際に実行状態に入るまで待機する。

シグネチャまたは形状

static void send_cancellable_query_impl(int line, PGconn *conn, PGconn *monitorConn);

入力と出力

  • line: エラー発生時に報告するソースコードの行番号。
  • conn: クエリを送信する対象の PGconn オブジェクト。
  • monitorConn: 状態監視に使用する別の PGconn オブジェクト。
  • 戻り値: なし。クエリ送信失敗時は pg_fatal_impl を呼び出しプロセスを終了する。

前提条件と不変条件

  • monitorConn は、conn のバックエンド状態を監視可能な有効な接続であること。
  • 環境変数 PG_TEST_TIMEOUT_DEFAULT が設定されていない場合、デフォルトの待機時間として "180" 秒が使用される。

アルゴリズムまたは振る舞い

1. wait_for_connection_state を呼び出し、conn が "idle" 状態になるまで待機する。 2. PQsendQueryParams を使用して SELECT pg_sleep($1) を送信する。パラメータ $1 には環境変数から取得した待機秒数が渡される。 3. 送信に失敗した場合は pg_fatal_impl でエラーを報告する。 4. 再度 wait_for_connection_state を呼び出し、conn のバックエンドで "PgSleep" が実行中であることを確認するまで待機する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/libpq_pipeline/libpq_pipeline.c 内のテストケース（test_cancel など）。
  • 呼び出し先:
  • wait_for_connection_state
  • PQbackendPID
  • getenv
  • PQsendQueryParams
  • pg_fatal_impl
  • PQerrorMessage

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • Oid: PostgreSQL の型識別子（INT4OID を使用）。

失敗モードと境界条件

  • PQsendQueryParams が 1 以外を返した場合、PQerrorMessage を伴って pg_fatal_impl が呼び出され、テストが強制終了される。
  • wait_for_connection_state がタイムアウトまたは接続エラーを検出した場合、その内部実装に従いテストが中断される可能性がある。

保守担当者向けメモ

  • 本関数は、キャンセル操作のテストにおいて「クエリが確実に実行中である」という前提条件を整えるためのユーティリティである。
  • monitorConn を使用した監視は、テスト対象の接続状態を外部から観測するために不可欠である。
