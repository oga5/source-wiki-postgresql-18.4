RestoreLibraryState

定義

  • 種別: function
  • 定義箇所: src/backend/utils/fmgr/dfmgr.c:740

目的

シリアライズされたライブラリパスのリストを読み込み、それらをプロセス空間に再ロードする。主に並列クエリ実行時など、バックエンド間でロード済みライブラリの状態を同期するために使用される。

シグネチャまたは形状

void RestoreLibraryState(char *start_address)

入力と出力

  • **入力**: start_address - ヌル文字で区切られたライブラリパス文字列が連続して格納されているメモリ領域の先頭ポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • start_address が指すメモリ領域は、SerializeLibraryState によって生成された形式（パス文字列が \0 で区切られ、全体が二重の \0 で終端される形式）に従っている必要がある。
  • 渡される各ライブラリパスは、現在の環境でロード可能な有効なパスであること。

アルゴリズムまたは振る舞い

1. start_address が指す文字列がヌル文字（\0）である間、ループを継続する。 2. 現在の start_address を引数として internal_load_library を呼び出し、ライブラリをロードする。 3. strlen を用いて現在の文字列の長さを取得し、ポインタを次の文字列の開始位置へ進める。 4. すべてのパスを処理し終えるまで繰り返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/parallel.c:1461
  • **呼び出し先**: internal_load_library

関連データ構造

  • DynamicFileList: ロード済みライブラリを管理する連結リスト。internal_load_library を通じて更新される。

失敗モードと境界条件

  • internal_load_library が失敗した場合（ライブラリが見つからない、ABI 互換性がない等）、その時点でエラーが発生し処理が中断される可能性がある。
  • 入力ポインタが不正なメモリ領域を指している場合、未定義動作を引き起こす。

保守担当者向けメモ

  • 本関数は SerializeLibraryState と対になる処理である。シリアライズ形式に変更を加える場合は、本関数の走査ロジックも併せて更新すること。
  • ロード処理は internal_load_library に依存しているため、ライブラリの重複ロード防止や初期化処理の挙動については同関数の仕様に従う。
