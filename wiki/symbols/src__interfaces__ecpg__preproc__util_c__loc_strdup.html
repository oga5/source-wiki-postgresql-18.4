loc_strdup

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/util.c:169

目的

入力文字列の複製を一時メモリ領域（loc_chunk）上に作成し、そのポインタを返します。

シグネチャまたは形状

char *loc_strdup(const char *string);

入力と出力

  • **入力**: string (複製対象のヌル終端文字列)
  • **出力**: 複製された文字列を指すポインタ。

前提条件と不変条件

  • 戻り値のメモリ領域は loc_alloc によって確保されます。
  • 呼び出し側で free を呼び出す必要はありません。解析サイクル終了時に reclaim_local_storage() を呼び出すことで一括解放されることが前提です。

アルゴリズムまたは振る舞い

1. strlen(string) + 1 バイトのメモリを loc_alloc を用いて確保します。 2. strcpy を使用して、入力文字列の内容を確保したメモリ領域へコピーします。 3. 複製された文字列の先頭ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/descriptor.c:354
  • src/interfaces/ecpg/preproc/parser.c:209, 259, 263
  • **呼び出し先**:
  • loc_alloc (メモリ確保)
  • strlen, strcpy (標準ライブラリ)

関連データ構造

  • loc_chunk: 一時メモリ管理のためのチャンク構造体。

失敗モードと境界条件

  • loc_alloc がメモリ確保に失敗した場合、loc_alloc 内部でエラー処理（プロセス終了等）が行われます。
  • 入力文字列が NULL の場合の挙動については、本ソース excerpt 内では定義されていません。

保守担当者向けメモ

  • 本関数は一時メモリを使用するため、解析の節目で必ず reclaim_local_storage() が呼ばれていることを確認してください。
  • loc_alloc は 8192 バイト単位でメモリを管理しているため、非常に大きな文字列を扱う場合は注意が必要です。
