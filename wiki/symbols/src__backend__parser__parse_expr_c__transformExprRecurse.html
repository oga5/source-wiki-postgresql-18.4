transformExprRecurse

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_expr.c:136

目的

構文解析器が生成した「生の式（raw expression）」ノードを、意味解析済みの「式ツリー（expression tree）」へ再帰的に変換・ディスパッチする。

シグネチャまたは形状

static Node *transformExprRecurse(ParseState *pstate, Node *expr);

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体へのポインタ。
  • expr: 変換対象の生の式ノード（Node）。
  • **出力**:
  • 意味解析および型変換が完了した Node 構造体。expr が NULL の場合は NULL を返す。

前提条件と不変条件

  • check_stack_depth() を呼び出し、再帰によるスタックオーバーフローを防止する。
  • T_SetToDefault ノードが渡された場合、コンテキストエラーとして ereport(ERROR, ...) を発生させる。

アルゴリズムまたは振る舞い

1. 入力が NULL であれば即座に NULL を返す。 2. nodeTag(expr) を使用してノードの種類を判定し、対応する transform* 関数（例: transformColumnRef, transformFuncCall, transformAExprOp など）へ処理を委譲する。 3. T_NamedArgExpr や T_NullTest のように、ノード内部の引数を再帰的に処理する必要がある場合は、自身を再帰呼び出しして変換を行う。 4. A_Expr のような複合ノードについては、内部の kind に基づいてさらに詳細な変換関数へディスパッチする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • transformExpr（エントリーポイント）
  • transformExprRecurse（自身による再帰呼び出し）
  • その他、parse_expr.c 内の各変換関数（transformIndirection 等）
  • **呼び出し先**:
  • 各種 transform* 関数（transformColumnRef, transformFuncCall, transformSubLink 等）
  • make_const
  • check_stack_depth
  • elog / ereport

関連データ構造

  • ParseState: 解析状態を管理する主要構造体。
  • Node: 構文解析ツリーの基本単位。
  • 各種 Node 派生構造体（ColumnRef, A_Expr, FuncCall 等）。

失敗モードと境界条件

  • **スタックオーバーフロー**: 複雑すぎる式に対しては check_stack_depth() がエラーを発生させる。
  • **未知のノード**: A_Expr の kind が認識できない場合、elog(ERROR, ...) を発生させる。
  • **コンテキスト違反**: DEFAULT キーワードが許可されていない場所で使用された場合、エラーを報告する。

保守担当者向けメモ

  • 本関数は switch 文による巨大なディスパッチャーである。新しい式ノードタイプを追加する際は、必ず本関数の switch 文にケースを追加すること。
  • transformIndirection や transformAExpr などの複雑な変換ロジックは別関数に分離されているため、詳細な変換ルールを変更する場合は各サブ関数を参照すること。
