binaryheap_remove_first

定義

  • 種別: function
  • 定義箇所: src/common/binaryheap.c:191

目的

バイナリヒープのルート要素（最小値または最大値）を削除し、ヒープ構造を再構成して整合性を維持します。

シグネチャまたは形状

bh_node_type binaryheap_remove_first(binaryheap *heap)

入力と出力

  • **入力**: binaryheap *heap（操作対象のヒープ構造体へのポインタ）
  • **出力**: bh_node_type（削除されたルート要素の値）

前提条件と不変条件

  • ヒープが空であってはならない（!binaryheap_empty(heap)）。
  • ヒープは有効なヒーププロパティを保持していなければならない（heap->bh_has_heap_property）。
  • 上記条件は Assert により検証されます。

アルゴリズムまたは振る舞い

1. ルート要素（bh_nodes[0]）を結果として保持します。 2. ヒープの要素数が1の場合、サイズを減らして即座にルート要素を返します。 3. 要素数が1より大きい場合：

  • ヒープの末尾要素をルート位置（bh_nodes[0]）に移動します。
  • ヒープサイズを1減らします。
  • sift_down(heap, 0) を呼び出し、新しいルート要素を適切な位置まで沈下させてヒーププロパティを再構築します。

    • 4. 保持していたルート要素を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeGatherMerge.c
  • src/backend/executor/nodeMergeAppend.c
  • src/backend/postmaster/pgarch.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/storage/buffer/bufmgr.c
  • src/bin/pg_dump/pg_dump_sort.c
  • **呼び出し先**:
  • sift_down（内部関数）

関連データ構造

  • binaryheap: ヒープの状態（要素配列 bh_nodes、サイズ bh_size、プロパティフラグ bh_has_heap_property）を保持する構造体。

失敗モードと境界条件

  • **空ヒープ**: binaryheap_empty が真の場合、Assert によりアボートします。
  • **単一要素**: 要素数が1のときは sift_down をスキップし、サイズを0にして終了します。

保守担当者向けメモ

  • 本関数は O(log n) の計算量で動作します。
  • bh_node_type は環境により Datum または void * に解決されるため、比較関数との整合性に注意してください。
