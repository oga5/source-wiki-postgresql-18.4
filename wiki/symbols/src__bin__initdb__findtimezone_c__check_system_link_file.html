check_system_link_file

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/findtimezone.c:543

目的

システム上のシンボリックリンク（主に /etc/localtime）のリンク先パスを解析し、そのパスに含まれる文字列からPostgreSQLで利用可能なタイムゾーン名を特定する。

シグネチャまたは形状

static bool check_system_link_file(const char *linkname, struct tztry *tt, char *bestzonename);

入力と出力

  • **入力**:
  • linkname: 解析対象のシンボリックリンクパス（例: "/etc/localtime"）。
  • tt: タイムゾーンの妥当性を検証するためのサンプリングデータ構造（struct tztry）。
  • bestzonename: 成功時に特定されたタイムゾーン名が格納されるバッファ。
  • **出力**:
  • bool: タイムゾーン名が特定できた場合は true、それ以外（リンクが存在しない、解析不能、一致するゾーン名が見つからない等）は false。

前提条件と不変条件

  • HAVE_READLINK が定義されている環境（Unix系）でのみ有効。
  • bestzonename は、十分な長さのバッファを指している必要がある。
  • リンク先パスの各セグメントが、PostgreSQLのタイムゾーンデータベースと一致するかを perfect_timezone_match で検証する。

アルゴリズムまたは振る舞い

1. readlink を使用してシンボリックリンクのターゲットパスを取得する。 2. 取得したパスを / で分割し、各セグメントを順次走査する。 3. 各セグメントが . で始まらず、かつ perfect_timezone_match によって tt との整合性が確認できた場合、そのセグメントを bestzonename にコピーして true を返す。 4. パス全体を走査しても一致が見つからない場合、または readlink が失敗した場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/findtimezone.c 内の identify_system_timezone（src/bin/initdb/findtimezone.c:406 付近）。
  • **呼び出し先**: readlink（システムコール）、perfect_timezone_match。

関連データ構造

  • struct tztry: タイムゾーンの妥当性検証に使用されるサンプリングデータ構造。

失敗モードと境界条件

  • readlink が失敗した場合、またはパスがバッファサイズ（MAXPGPATH）を超える場合は false を返す。
  • リンク先パスに有効なタイムゾーン名が含まれていない場合は false を返す。
  • HAVE_READLINK が未定義の環境では、常に false を返す。

保守担当者向けメモ

  • 本関数は「ブルートフォース」的な探索を行うため、リンク先パスの構造に依存する。OSのディレクトリ構成が変更された場合、探索ロジックの調整が必要になる可能性がある。
  • perfect_timezone_match の結果に強く依存しているため、タイムゾーン定義の不整合が発生した場合は同関数を確認すること。
