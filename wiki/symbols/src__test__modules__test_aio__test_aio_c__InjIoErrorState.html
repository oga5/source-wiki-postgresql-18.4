InjIoErrorState

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_aio/test_aio.c:38

目的

PostgreSQL の非同期 I/O (AIO) サブシステムのテストにおいて、意図的なエラー注入状態を保持するための共有メモリ構造体です。

シグネチャまたは形状

typedef struct InjIoErrorState
{
	bool		enabled_short_read;    /* 短縮読み込み（short read）の有効化フラグ */
	bool		enabled_reopen;        /* 再オープン失敗の有効化フラグ */

	bool		short_read_result_set; /* 読み込み結果設定の有無 */
	int			short_read_result;     /* 読み込み結果の戻り値 */
} InjIoErrorState;

入力と出力

本構造体はデータ保持用であり、直接的な関数入力・出力は持ちません。各フィールドは InjectionPoint を介したテスト実行時に、エラー発生ロジックの制御パラメータとして参照されます。

前提条件と不変条件

  • 共有メモリ上に配置されることを前提としています。
  • test_aio_shmem_startup 等の初期化プロセスにおいて、RequestAddinShmemSpace を通じて本構造体サイズ分のメモリが確保される必要があります。

アルゴリズムまたは振る舞い

  • enabled_short_read が真の場合、I/O 処理中に短縮読み込みをシミュレートします。
  • short_read_result_set が真の場合、short_read_result に格納された値を I/O 操作の結果として強制的に適用します。
  • enabled_reopen が真の場合、I/O 処理中の再オープン操作を失敗させるトリガーとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_aio/test_aio.c 内の RequestAddinShmemSpace 呼び出し箇所（メモリ確保時）。
  • inj_io_short_read および inj_io_reopen（注入ポイントの実行ロジック）。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • InjectionPoint: 本構造体のフラグをトリガーとして利用するテスト用インフラ。

失敗モードと境界条件

  • 共有メモリの初期化に失敗した場合、本構造体へのアクセスは不正なメモリ参照を引き起こす可能性があります。
  • 注入されたエラー値（short_read_result）が不適切な場合、AIO サブシステムが予期しない動作をする可能性があります。

保守担当者向けメモ

  • 本構造体はテスト専用です。本番環境のコードには影響を与えませんが、共有メモリを使用しているため、構造体のサイズ変更時には RequestAddinShmemSpace の引数に影響が出る点に注意してください。
