startScan

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginget.c:604

目的

GINインデックススキャンにおいて、各検索キーの出現頻度を推定し、スキャンキーの初期化を完了させる。

シグネチャまたは形状

static void startScan(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan（スキャン記述子）。内部の opaque フィールドから GinScanOpaque 構造体を参照する。
  • **出力**: なし（副作用として scan->opaque 内の各エントリおよびキーの状態を更新する）。

前提条件と不変条件

  • scan->opaque が GinScanOpaque 型として正しく初期化されていること。
  • so->entries および so->keys が適切に割り当てられていること。

アルゴリズムまたは振る舞い

1. **エントリの初期化**: so->entries に含まれる全エントリに対して startScanEntry を呼び出し、各エントリの統計情報を取得する。 2. **ファジー検索の最適化**: GinFuzzySearchLimit が有効な場合、全エントリの推定結果数（predictNumberResult）を評価する。すべてのエントリが閾値を超えていると判断された場合、各エントリの推定結果数を削減し、reduceResult フラグを立てる。 3. **キーの最終化**: so->keys に含まれる全キーに対して startScanKey を呼び出し、頻度情報に基づいたスキャンキーの最終的な初期化を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginget.c 内の関数（具体的な関数名はソース抜粋からは不明だが、スキャン開始時に呼び出される）。
  • **呼び出し先**:
  • startScanEntry
  • startScanKey

関連データ構造

  • IndexScanDesc: PostgreSQLの標準的なインデックススキャン記述子。
  • GinScanOpaque: GINインデックス固有のスキャン状態を保持する構造体。
  • GinState: GINインデックスの操作状態を管理する構造体。

失敗モードと境界条件

  • GinFuzzySearchLimit が 0 以下の場合、削減ロジックはスキップされる。
  • so->totalentries が 0 の場合、ループは実行されない。

保守担当者向けメモ

  • 本関数は、頻度ベースの最適化を行うための重要なフェーズである。
  • reduceResult の適用は、インデックス検索のパフォーマンスを向上させるためのヒューリスティックな処理であり、厳密な結果を保証するものではない点に注意すること。
