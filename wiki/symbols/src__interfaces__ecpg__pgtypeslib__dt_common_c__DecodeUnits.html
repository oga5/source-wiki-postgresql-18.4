DecodeUnits

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/dt_common.c:535

目的

日付・時刻の解析において、文字列トークン（単位）を対応する内部型および数値へ変換する。

シグネチャまたは形状

int DecodeUnits(int field, char *lowtoken, int *val)

入力と出力

  • **入力**:
  • field: キャッシュインデックスとして使用される整数。
  • lowtoken: 解析対象の文字列トークン。
  • val: 変換後の値を格納する整数ポインタ。
  • **出力**:
  • 戻り値: トークンの型（int）。一致しない場合は UNKNOWN_FIELD を返す。
  • *val: トークンに対応する数値が書き込まれる。

前提条件と不変条件

  • lowtoken は strncmp による比較が可能な有効な文字列であること。
  • deltacache 配列が field インデックスに対してアクセス可能であること。

アルゴリズムまたは振る舞い

1. deltacache[field] を確認し、キャッシュされたトークンと lowtoken が一致すればそれを使用する。 2. 一致しない場合、datebsearch を用いて deltatktbl（時間単位テーブル）から lowtoken をバイナリサーチする。 3. 検索結果を deltacache[field] に保存する。 4. トークンが見つかった場合、その型と値を返し、見つからなかった場合は UNKNOWN_FIELD を返し *val に 0 を設定する。

呼び出し元と呼び出し先

  • **呼び出し先**: datebsearch（src/interfaces/ecpg/pgtypeslib/dt_common.c 内で定義）。
  • **呼び出し元**: ツールによる解析では、src/backend/utils/adt/ 配下の date.c, datetime.c, timestamp.c 等からの参照が示唆されているが、これらはバックエンド側のコードであり、本ライブラリ（ECPG）との直接的な呼び出し関係についてはソースコードの文脈からは断定できない。

関連データ構造

  • datetkn: トークンの型と値を保持する構造体。
  • deltatktbl: 時間単位を定義するテーブル。
  • deltacache: 検索結果を保持するキャッシュ配列。

失敗モードと境界条件

  • トークンがテーブル内に存在しない場合、tp は NULL となり、型として UNKNOWN_FIELD が返される。
  • field インデックスが deltacache の範囲外である場合の挙動については、本ソース excerpt からは判断できない。

保守担当者向けメモ

  • 本関数は deltacache を使用して検索を最適化している。キャッシュの整合性は呼び出し側の field 管理に依存する。
  • strncmp を使用しているため、トークンが切り詰められていてもマッチする可能性がある。
