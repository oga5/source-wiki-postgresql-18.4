pg_num_nonnulls

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/misc.c:177

目的

可変長引数リスト（VARIADIC）または関数に渡された引数群のうち、NULL ではない値の個数をカウントして返します。

シグネチャまたは形状

Datum pg_num_nonnulls(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される可変長引数。
  • **出力**: int32 型の非NULL引数の個数。引数リストが解析できない場合は NULL を返します。

前提条件と不変条件

  • count_nulls 関数が正常に引数リストを解析できる必要があります。

アルゴリズムまたは振る舞い

1. count_nulls(fcinfo, &nargs, &nulls) を呼び出し、全引数数（nargs）とNULLである引数の数（nulls）を取得します。 2. count_nulls が失敗（false を返却）した場合は、PG_RETURN_NULL() を実行して終了します。 3. 成功した場合は、全引数数からNULL数を減算した値（nargs - nulls）を PG_RETURN_INT32 を用いて返却します。

呼び出し元と呼び出し先

  • **呼び出し先**: count_nulls（src/backend/utils/adt/misc.c 内で定義）
  • **呼び出し元**: ソースコード上での直接的な呼び出し元は確認されていません（SQLインターフェースとして外部から呼び出されることを想定しています）。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • PG_FUNCTION_ARGS: SQL関数呼び出し時の引数コンテキスト。

失敗モードと境界条件

  • count_nulls が内部的に失敗した場合、本関数はSQLレベルの NULL を返します。

保守担当者向けメモ

  • 本関数は pg_num_nulls と対になる実装です。ロジックの変更が必要な場合は、count_nulls の仕様変更が両関数に影響を与える点に留意してください。
