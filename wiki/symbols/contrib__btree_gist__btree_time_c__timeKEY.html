timeKEY

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_time.c:13

目的

PostgreSQLのGiSTインデックスにおいて、time型およびtimetz型の値を範囲（下限と上限）として保持し、B-treeのような順序付けられた検索を可能にするためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	TimeADT		lower;
	TimeADT		upper;
} timeKEY;

入力と出力

  • **入力**: TimeADT 型の lower（範囲の下限）および upper（範囲の上限）。
  • **出力**: GiSTインデックスのノードに格納される構造体として、インデックスの圧縮や比較処理の単位となります。

前提条件と不変条件

  • lower は常に upper 以下である必要があります。
  • timetz 型を扱う際は、タイムゾーンを考慮して正規化された TimeADT 値が格納されます。

アルゴリズムまたは振る舞い

GiSTインデックスのノード内で、時間データの範囲を表現するために使用されます。gbt_time_compress などの関数を通じて、元の時間データがこの構造体に変換され、インデックスの木構造に挿入されます。検索時には、この範囲情報を用いてクエリ値との適合判定が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_time.c 内の各GiSTサポート関数（gbt_time_compress, gbt_time_consistent, gbt_time_dist 等）から、インデックスエントリのキャストやメモリ確保の対象として参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • TimeADT: PostgreSQLの内部的な時間表現型。
  • GISTENTRY: GiSTインデックスの各エントリを保持する構造体。
  • gbtree_ninfo: 本構造体を用いた比較・距離計算関数のマッピングテーブル。

失敗モードと境界条件

  • lower > upper となるような不正な範囲が生成された場合、インデックス検索の結果が正しく得られない可能性があります。
  • メモリ割り当て（palloc）失敗時に例外が発生する可能性があります。

保守担当者向けメモ

  • btree_utils_num.c で定義された汎用的な数値処理ロジックと密接に連携しています。
  • timetz 型の処理では、タイムゾーンオフセットの正規化が正しく行われているかを確認してください。
  • recheck フラグが必要な検索条件については、gbt_time_consistent の実装を参照してください。
