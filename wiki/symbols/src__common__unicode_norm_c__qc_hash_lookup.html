qc_hash_lookup

定義

  • **種別**: 関数
  • **定義箇所**: src/common/unicode_norm.c:561

目的

Unicode 正規化プロパティの静的ハッシュテーブルから、指定されたコードポイントに対応する情報を高速に検索します。

シグネチャまたは形状

static const pg_unicode_normprops *qc_hash_lookup(pg_wchar ch, const pg_unicode_norminfo *norminfo)

入力と出力

  • **入力**:
  • ch: 検索対象のコードポイント (pg_wchar)。
  • norminfo: 検索対象のハッシュテーブルおよび関連情報を含む構造体 (pg_unicode_norminfo)。
  • **出力**:
  • 成功時: 対応する pg_unicode_normprops 構造体へのポインタ。
  • 失敗時: NULL。

前提条件と不変条件

  • norminfo は有効な初期化済み構造体である必要があります。
  • ハッシュテーブルは「完全ハッシュ（perfect hash）」として設計されており、インデックス計算後の衝突回避処理は不要です。

アルゴリズムまたは振る舞い

1. pg_hton32 を使用してコードポイントをネットワークバイトオーダーに変換し、ハッシュキーを生成します。 2. norminfo->hash 関数を呼び出し、ハッシュテーブルのインデックス h を算出します。 3. インデックス h が範囲外（h < 0 または h >= num_normprops）の場合は NULL を返します。 4. インデックス h に格納されているエントリのコードポイントが、入力 ch と一致するか確認します。一致しない場合は NULL を返します。 5. 一致した場合、該当エントリへのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode_norm.c 内の正規化処理において、UnicodeNormInfo_NFC_QC および UnicodeNormInfo_NFKC_QC を引数として呼び出されます。
  • **呼び出し先**:
  • norminfo->hash（関数ポインタ経由でハッシュ関数を呼び出し）。

関連データ構造

  • pg_unicode_norminfo: ハッシュ関数、プロパティ配列、およびテーブルサイズを保持する構造体。
  • pg_unicode_normprops: 正規化プロパティを保持する構造体。

失敗モードと境界条件

  • 入力コードポイントがハッシュテーブルに存在しない場合、NULL を返します。
  • ハッシュ計算の結果がテーブル範囲外となる場合、NULL を返します。

保守担当者向けメモ

  • 本関数はバックエンドでの高速なクイックチェックに使用されます。
  • 完全ハッシュの性質上、インデックスが一致してもコードポイントの照合（ch != norminfo->normprops[h].codepoint）が必須です。
