SyncRepWaitForLSN

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:147

目的

トランザクションのコミット時、指定されたLSN（Log Sequence Number）が同期スタンバイサーバーへ確実にレプリケートされるまで、ユーザーバックエンドを待機させる。

シグネチャまたは形状

void SyncRepWaitForLSN(XLogRecPtr lsn, bool commit)

入力と出力

  • **入力**:
  • lsn (XLogRecPtr): 待機対象となるWALの終了位置。
  • commit (bool): トランザクションのコミットであるか否か。false の場合、待機レベルは SYNC_REP_WAIT_FLUSH を上限として制限される。
  • **出力**: なし（完了時に復帰）。

前提条件と不変条件

  • 呼び出し時に InterruptHoldoffCount > 0 であること（割り込みを禁止した状態で呼び出す必要がある）。
  • MyProc->syncRepLinks がデタッチ状態であること。
  • WalSndCtl が初期化されていること。
  • 待機キューは常にLSN順にソートされていること。

アルゴリズムまたは振る舞い

1. **早期終了判定**: 同期レプリケーションが要求されていない場合、または同期スタンバイが未定義の場合は即座に復帰する。 2. **待機モード設定**: commit 引数に基づき、待機レベル（Write/Flush/Apply）を決定する。 3. **キューイング**: SyncRepLock を排他獲得し、MyProc の waitLSN を設定して SyncRepQueueInsert を実行し、待機状態へ移行する。 4. **待機ループ**: MyLatch を使用して、syncRepState が SYNC_REP_WAIT_COMPLETE になるまで待機する。 5. **割り込み処理**: 待機中に ProcDiePending や QueryCancelPending が発生した場合、警告を発行して SyncRepCancelWait を呼び出し、待機を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c (prepare/commit処理)
  • src/backend/access/transam/xact.c (トランザクションコミット処理)
  • **呼び出し先**:
  • SyncRepQueueInsert
  • SyncRepCancelWait
  • LWLockAcquire / LWLockRelease
  • SetLatch / ResetLatch
  • set_ps_display_suffix

関連データ構造

  • WalSndCtl: 同期レプリケーションの状態を保持する共有メモリ構造体。
  • PGPROC (MyProc): 現在のバックエンドのプロセス情報と待機状態（syncRepState）。
  • SyncRepLock: 待機キューを保護するLWLock。

失敗モードと境界条件

  • **管理者の介入**: 待機中に管理者が接続を終了させた場合、警告を発して待機を中断する。
  • **クエリキャンセル**: ユーザーがキャンセルを要求した場合、警告を発して待機を中断する。
  • **競合状態**: SyncRepUpdateSyncStandbysDefined との競合を避けるため、ロック取得前後で sync_standbys_status を慎重に検証する。

保守担当者向けメモ

  • 本関数はコミットのたびに呼び出されるため、同期レプリケーションが不要な場合の早期終了パスは極めて重要である。
  • 待機中に ProcDiePending が発生しても、ローカルでのコミットは完了しているため、エラーではなく警告を発行して処理を継続する設計となっている。
