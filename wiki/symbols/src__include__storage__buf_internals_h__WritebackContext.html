WritebackContext

定義

  • 種別: struct
  • 定義箇所: src/include/storage/buf_internals.h:312

目的

バッファの書き戻し（writeback）要求を蓄積し、効率的に処理するために使用されるコンテキスト構造体です。複数の書き戻し要求をまとめて（coalesce）発行することを目的としています。

シグネチャまたは形状

typedef struct WritebackContext
{
    int        *max_pending;
    int         nr_pending;
    PendingWriteback pending_writebacks[WRITEBACK_MAX_PENDING_FLUSHES];
} WritebackContext;

入力と出力

  • **入力**: WritebackContextInit を介して max_pending（書き戻し要求の最大数）が設定されます。
  • **出力**: ScheduleBufferTagForWriteback により pending_writebacks 配列に要求が蓄積され、IssuePendingWritebacks により蓄積された要求が物理的な書き込み操作へと発行されます。

前提条件と不変条件

  • nr_pending は常に 0 以上 WRITEBACK_MAX_PENDING_FLUSHES 以下である必要があります。
  • max_pending は有効な整数へのポインタである必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: WritebackContextInit でコンテキストを準備します。 2. **蓄積**: ScheduleBufferTagForWriteback を呼び出し、書き戻すべきバッファタグを pending_writebacks 配列に追加します。 3. **発行**: IssuePendingWritebacks を呼び出すことで、蓄積された要求を実際に実行します。この際、要求が上限に達した場合や、明示的なフラッシュ要求時に処理が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: bgwriter.c（wb_context）、bufmgr.c（BgBufferSync, SyncOneBuffer 等）で利用されています。
  • **呼び出し先**: WritebackContextInit, ScheduleBufferTagForWriteback, IssuePendingWritebacks といった bufmgr.c 内の関数群が本構造体を操作します。

関連データ構造

  • PendingWriteback: 個別の書き戻し要求を保持する構造体。
  • WRITEBACK_MAX_PENDING_FLUSHES: pending_writebacks 配列のサイズを規定する定数。

失敗モードと境界条件

  • pending_writebacks の容量（WRITEBACK_MAX_PENDING_FLUSHES）を超えて要求を追加しようとした場合の挙動については、ソースコードの断片からは明示されていません。

保守担当者向けメモ

  • 本構造体はバッファマネージャの書き戻し処理の最適化に直結しています。構造体のメンバを変更する際は、bgwriter や bufmgr における書き戻しフローへの影響を考慮してください。
