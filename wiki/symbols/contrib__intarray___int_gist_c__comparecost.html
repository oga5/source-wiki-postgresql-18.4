comparecost

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_gist.c:428

目的

GiST インデックスのページ分割処理において、SPLITCOST 構造体の配列をコスト値に基づいて昇順にソートするための比較関数です。

シグネチャまたは形状

static int comparecost(const void *a, const void *b)

入力と出力

  • **入力**: a, b (比較対象となる SPLITCOST 構造体へのポインタ)
  • **出力**:
  • 0: a と b のコストが等しい場合
  • 1: a のコストが b より大きい場合
  • -1: a のコストが b より小さい場合

前提条件と不変条件

  • 入力ポインタ a および b は、SPLITCOST 型の構造体を指している必要があります。
  • SPLITCOST 構造体には cost メンバが存在することが前提です。

アルゴリズムまたは振る舞い

1. 引数 a と b を SPLITCOST* にキャストします。 2. 両者の cost メンバを比較します。 3. cost が等しければ 0 を返し、a->cost が大きければ 1 を、小さければ -1 を返します。 4. この振る舞いは標準ライブラリの qsort 関数が期待する比較関数の仕様に準拠しています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/intarray/_int_gist.c 内の qsort 呼び出し。
  • 他のモジュール（contrib/hstore, contrib/ltree, contrib/pg_trgm, src/backend/utils/adt/tsgistidx.c）でも同名の関数が qsort の引数として利用されています。
  • **呼び出し先**: なし。

関連データ構造

  • SPLITCOST: インデックス分割時のコストを保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、または SPLITCOST 型以外のポインタが渡された場合の動作は未定義です。

保守担当者向けメモ

  • 本関数は複数の contrib モジュールおよびコア機能で重複して定義されています。修正を行う際は、他のファイルとの整合性に注意してください。
  • qsort の比較関数として汎用的に利用されているため、ロジックの変更は広範囲に影響を及ぼす可能性があります。
