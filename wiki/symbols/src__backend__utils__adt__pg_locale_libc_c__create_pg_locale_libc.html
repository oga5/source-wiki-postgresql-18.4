create_pg_locale_libc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:420

目的

指定された照合順序（Collation）OIDに基づき、OSのlibcロケール機能を利用するための pg_locale_t 構造体を初期化および構築する。

シグネチャまたは形状

pg_locale_t create_pg_locale_libc(Oid collid, MemoryContext context)

入力と出力

  • **入力**:
  • collid: 照合順序のOID。DEFAULT_COLLATION_OID の場合はデータベースのデフォルトロケールを使用する。
  • context: ロケール構造体を割り当てるためのメモリコンテキスト。
  • **出力**:
  • 初期化済みの pg_locale_t 構造体へのポインタ。

前提条件と不変条件

  • collid が DEFAULT_COLLATION_OID の場合、MyDatabaseId が有効であること。
  • それ以外の場合、指定された collid が pg_collation システムカタログに存在すること。
  • make_libc_collator がOSのロケールハンドルを正常に生成できること。

アルゴリズムまたは振る舞い

1. **ロケール情報の取得**: collid に応じて pg_database または pg_collation システムカタログから datcollate/collcollate および datctype/collctype を取得する。 2. **OSロケールハンドルの生成**: make_libc_collator を呼び出し、取得したロケール文字列からOS固有のロケールハンドル（locale_t）を生成する。 3. **構造体の初期化**: MemoryContextAllocZero を使用して pg_locale_struct を確保し、以下のフィールドを設定する。

  • provider: COLLPROVIDER_LIBC
  • deterministic: true
  • collate_is_c / ctype_is_c: ロケール名が "C" または "POSIX" かどうかを判定。

    • 4. **メソッドの割り当て**: "C" ロケール以外の場合、プラットフォームに応じて適切な照合メソッドテーブル（collate_methods_libc または Windows用の collate_methods_libc_win32_utf8）を割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c (複数箇所で呼び出しを確認)
  • **呼び出し先**:
  • SearchSysCache1, SysCacheGetAttrNotNull, ReleaseSysCache
  • make_libc_collator
  • MemoryContextAllocZero
  • TextDatumGetCString

関連データ構造

  • pg_locale_struct: ロケール情報を保持するメイン構造体。
  • pg_collation, pg_database: システムカタログ。
  • collate_methods_libc: libcプロバイダーの照合メソッドテーブル。

失敗モードと境界条件

  • SearchSysCache1 が失敗した場合、elog(ERROR) により処理が中断される。
  • システムがサポートしていないロケール名が指定された場合、make_libc_collator の内部実装に依存してエラーが発生する可能性がある。

保守担当者向けメモ

  • Windows環境では、データベースエンコーディングがUTF-8の場合に専用のメソッドテーブル（collate_methods_libc_win32_utf8）が選択される点に注意すること。
  • collate_is_c の判定は、ロケール名が "C" または "POSIX" であるかどうかに基づく。
