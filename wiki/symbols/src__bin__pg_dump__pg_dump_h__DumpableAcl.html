DumpableAcl

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_dump.h:168

目的

データベースオブジェクトに関連付けられたアクセス制御リスト（ACL）情報を保持し、ダンプ処理時にオブジェクトの権限設定を適切に再現するためのデータ構造を提供します。

シグネチャまたは形状

typedef struct _dumpableAcl
{
    char       *acl;            /* オブジェクトの実際のACL文字列 */
    char       *acldefault;     /* オブジェクトの型と所有者に対するデフォルトACL */
    char        privtype;       /* エントリ型: 'i' (init), 'e' (extension), 0 (なし) */
    char       *initprivs;      /* オブジェクトの初期ACL文字列、またはNULL */
} DumpableAcl;

入力と出力

  • **入力**: システムカタログ（pg_init_privs 等）から取得された権限情報。
  • **出力**: ダンプ生成時に使用されるACL文字列および初期権限情報。

前提条件と不変条件

  • ACLを持つオブジェクトは、必ずこの構造体をサブ構造体として保持する必要があります。
  • privtype は、pg_init_privs エントリが存在する場合にその型を示します。

アルゴリズムまたは振る舞い

  • オブジェクトの現在のACL状態と、システムカタログ上の初期状態（initprivs）を保持します。
  • ダンプ実行時、これらのフィールドを参照して GRANT/REVOKE 文を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の各処理関数が、オブジェクトのACL情報を操作・参照するために利用しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • DumpableObject: ACLを持つオブジェクトの基底構造体。
  • DumpableObjectWithAcl: DumpableAcl をメンバとして含む、ACL対応オブジェクトの派生構造体。

失敗モードと境界条件

  • initprivs が NULL の場合、そのオブジェクトには初期権限エントリが存在しないことを示します。
  • privtype が 0 の場合、initprivs フィールドは無視されるべきです。

保守担当者向けメモ

  • 新しいオブジェクト型にACLサポートを追加する場合は、必ず DumpableAcl を構造体に含めてください。
  • pg_init_privs の変更に伴い、privtype の列挙値や initprivs の解釈に変更がないか確認してください。
