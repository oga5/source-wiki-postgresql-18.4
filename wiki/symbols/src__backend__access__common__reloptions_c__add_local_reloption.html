add_local_reloption

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/reloptions.c:767

目的

特定のローカルリレーションオプション構造体（local_relopts）に対し、新しいオプション定義（relopt_gen）を登録し、その格納オフセットを関連付ける。

シグネチャまたは形状

static void add_local_reloption(local_relopts *relopts, relopt_gen *newoption, int offset)

入力と出力

  • **入力**:
  • relopts: オプション定義を保持する local_relopts 構造体へのポインタ。
  • newoption: 追加するオプションのメタデータを含む relopt_gen 構造体へのポインタ。
  • offset: ターゲット構造体内での当該オプションの格納オフセット。
  • **出力**: なし（副作用として relopts->options リストが更新される）。

前提条件と不変条件

  • offset は、relopts->relopt_struct_size 未満でなければならない（Assert による検証）。

アルゴリズムまたは振る舞い

1. palloc を使用して local_relopt 構造体用のメモリを確保する。 2. 引数で受け取った newoption と offset を確保した構造体に格納する。 3. lappend を使用して、当該オプションを relopts->options リストの末尾に追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/reloptions.c 内の以下の関数から呼び出されることが確認されている。
  • add_local_bool_reloption (行883, 937, 992, 1056, 1140)
  • **呼び出し先**:
  • palloc (メモリ確保)
  • lappend (リスト操作)

関連データ構造

  • local_relopts: オプションのリストと構造体サイズを管理するコンテナ。
  • relopt_gen: オプションの基本メタデータ（名前、型、ロックレベル等）を保持する基底構造体。
  • local_relopt: relopt_gen とオフセットをペアにする内部管理用構造体。

失敗モードと境界条件

  • offset が relopts->relopt_struct_size 以上の場合、アサーション違反によりプロセスが停止する可能性がある。
  • メモリ不足時には palloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数は static であり、モジュール内でのローカルなオプション定義の構築に特化している。
  • relopts->options はリスト形式で管理されるため、オプション数が増加した場合の検索コストに留意すること。
