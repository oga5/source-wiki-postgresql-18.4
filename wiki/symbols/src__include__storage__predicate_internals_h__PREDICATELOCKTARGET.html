PREDICATELOCKTARGET

定義

  • 種別: struct
  • 定義箇所: src/include/storage/predicate_internals.h:284

目的

PostgreSQLのSSI（Serializable Snapshot Isolation）において、述語ロックの対象となるリレーション、ページ、またはタプルを識別し、それに関連する述語ロックのリストを保持するために使用されます。

シグネチャまたは形状

typedef struct PREDICATELOCKTARGET
{
    PREDICATELOCKTARGETTAG tag;
    dlist_head    predicateLocks;
} PREDICATELOCKTARGET;

入力と出力

  • **入力**: PREDICATELOCKTARGETTAG 型の tag フィールドにより、ロック対象を一意に識別します。
  • **出力**: dlist_head 型の predicateLocks フィールドを通じて、当該対象に関連付けられた PREDICATELOCK オブジェクトのリストを管理します。

前提条件と不変条件

  • 共有メモリ上のハッシュテーブル（PredicateLockTargetHash）に格納されるエントリとして使用されます。
  • predicateLocks は lib/ilist.h で定義される双方向リストとして管理されます。

アルゴリズムまたは振る舞い

  • 述語ロックの取得時に、対象がハッシュテーブルに存在しない場合は本構造体が新規作成されます。
  • 関連付けられたすべての PREDICATELOCK が解放された際、RemoveTargetIfNoLongerUsed 等の関数によって不要になったターゲットが削除されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/storage/lmgr/predicate.c: ハッシュテーブルの初期化、検索、および削除処理において参照されます。
  • **呼び出し先**:
  • dlist_head（lib/ilist.h）: リスト管理機能を利用します。
  • PREDICATELOCKTARGETTAG: 識別子として利用されます。

関連データ構造

  • PREDICATELOCKTARGETTAG: ロック対象の物理的・論理的識別子。
  • PREDICATELOCK: 本構造体にリンクされる個別のロック状態。
  • dlist_head: lib/ilist.h で定義されるリスト構造。

失敗モードと境界条件

  • 共有メモリ上のハッシュテーブルが満杯の場合、新規ターゲットの割り当てに失敗する可能性があります。
  • ターゲット削除時の競合（他のプロセスが同時に参照している場合など）については、呼び出し側の predicate.c 内のロック制御に依存します。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、フィールドの追加や変更を行う際はアライメントとパディングに注意してください。
  • predicateLocks リストの操作は、SSIサブシステムの共有ロック（LWLock）によって保護されている必要があります。
