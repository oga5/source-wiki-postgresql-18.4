heap_create_with_catalog

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/heap.c:1135

目的

PostgreSQLにおいて、カタログ情報（pg_class等）への登録を伴う新しいヒープリレーション（テーブル）を作成するメインエントリポイントです。

シグネチャまたは形状

Oid heap_create_with_catalog(
    const char *relname, Oid relnamespace, Oid reltablespace, Oid relid,
    Oid reltypeid, Oid reloftypeid, Oid ownerid, Oid accessmtd,
    TupleDesc tupdesc, List *cooked_constraints, char relkind,
    char relpersistence, bool shared_relation, bool mapped_relation,
    OnCommitAction oncommit, Datum reloptions, bool use_user_acl,
    bool allow_system_table_mods, bool is_internal, Oid relrewrite,
    ObjectAddress *typaddress
);

入力と出力

  • **入力**: リレーション名、名前空間、テーブルスペース、OID、所有者ID、アクセスメソッド、タプル記述子（tupdesc）、制約リスト、リレーション種別（relkind）等の属性情報。
  • **出力**: 作成されたリレーションの Oid を返します。

前提条件と不変条件

  • トランザクション内での実行が必須です。
  • IsNormalProcessingMode() または IsBootstrapProcessingMode() である必要があります。
  • 共有リレーション（shared_relation）を作成する場合、reltablespace は GLOBALTABLESPACE_OID である必要があります。

アルゴリズムまたは振る舞い

1. **検証**: CheckAttributeNamesTypes を呼び出し、列定義の妥当性を検証します。 2. **重複チェック**: 同名の既存リレーションや型名との衝突を確認し、存在する場合はエラーを発生させます。 3. **OID/RelFileNumberの決定**: IsBinaryUpgrade モード時は指定されたOIDを使用し、それ以外は GetNewRelFileNumber を用いて新しい識別子を割り当てます。 4. **ロック**: AccessExclusiveLock を取得し、他のセッションからのアクセスを排他します。 5. **権限設定**: use_user_acl が有効な場合、get_user_default_acl を用いて初期アクセス権限を決定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/toasting.c、src/backend/commands/cluster.c、src/backend/commands/tablecmds.c 等から呼び出されます。
  • **呼び出し先**: table_open、CheckAttributeNamesTypes、get_relname_relid、GetSysCacheOid2、moveArrayTypeName、GetNewRelFileNumber、LockRelationOid、get_user_default_acl 等。

関連データ構造

  • TupleDesc: リレーションの列定義を保持。
  • Relation: pg_class 等のカタログリレーションへのアクセスに使用。
  • ObjectAddress: 作成されたオブジェクトの識別情報を格納。

失敗モードと境界条件

  • **重複エラー**: 同名のテーブルや型が既に存在する場合、ERRCODE_DUPLICATE_TABLE または ERRCODE_DUPLICATE_OBJECT で失敗します。
  • **バイナリアップグレードエラー**: アップグレードモード時に必要なOIDやファイル番号が設定されていない場合、ERRCODE_INVALID_PARAMETER_VALUE で失敗します。
  • **制約違反**: 共有リレーションがグローバルテーブルスペース以外に配置されようとした場合、エラーとなります。

保守担当者向けメモ

  • 本関数は pg_class への挿入を含むカタログ操作の起点です。
  • binary_upgrade_next_... 変数を用いたバイナリアップグレード時のOID固定処理が含まれており、変更には注意が必要です。
  • システム属性の定義は歴史的な経緯によりハードコーディングされている箇所があるため、修正時は関連する heap.c 内の定義を確認してください。
