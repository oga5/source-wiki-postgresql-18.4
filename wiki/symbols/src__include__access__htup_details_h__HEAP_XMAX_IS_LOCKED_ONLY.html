HEAP_XMAX_IS_LOCKED_ONLY

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/htup_details.h:229

目的

ヒープタプルの t_infomask を評価し、タプルがデータ更新ではなくロックのみの状態にあるかを判定する。

シグネチャまたは形状

static inline bool HEAP_XMAX_IS_LOCKED_ONLY(uint16 infomask)

入力と出力

  • **入力**: infomask (タプルの t_infomask フィールド)
  • **出力**: bool (ロックのみの状態であれば true、それ以外は false)

前提条件と不変条件

  • 入力される infomask は、有効なヒープタプルのヘッダーから取得されたものであること。

アルゴリズムまたは振る舞い

以下のいずれかの条件を満たす場合に true を返す： 1. HEAP_XMAX_LOCK_ONLY ビットがセットされている。 2. HEAP_XMAX_IS_MULTI ビットと HEAP_LOCK_MASK の論理積が HEAP_XMAX_EXCL_LOCK と一致する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • src/backend/access/heap/heapam.c
  • **呼び出し先**:
  • 本関数はインライン関数であり、特定の関数を呼び出すのではなく、ビット演算のみを実行する。

関連データ構造

  • HeapTupleHeaderData (タプルヘッダー)
  • t_infomask (タプルの状態を保持するビットマスク)

失敗モードと境界条件

  • infomask が不正な値（ビット定義外）である場合、誤った判定結果を返す可能性がある。

保守担当者向けメモ

  • 本関数は、タプルの可視性やロック状態を判定する低レベルな基盤であるため、変更時は t_infomask のビット定義との整合性に十分注意すること。
