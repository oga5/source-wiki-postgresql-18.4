GetRealCmax

定義

  • 種別: function
  • 定義箇所: src/backend/utils/time/combocid.c:285

目的

Combo CID（コマンドIDのペアを抽象化したID）から、対応する実際の cmax 値を復元するために使用されます。

シグネチャまたは形状

static CommandId GetRealCmax(CommandId combocid)

入力と出力

  • **入力**: combocid (CommandId型) - 変換対象のCombo CID。
  • **出力**: CommandId - 該当するCombo CIDに関連付けられた実際の cmax 値。

前提条件と不変条件

  • combocid は、現在有効な usedComboCids（使用中のCombo CID数）よりも小さい値である必要があります。この条件は Assert によって検証されます。

アルゴリズムまたは振る舞い

1. 引数 combocid が usedComboCids 未満であることをアサートします。 2. comboCids 配列の指定されたインデックス（combocid）に格納されている cmax 値を直接参照し、その値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/time/combocid.c:134（HeapTupleHeaderGetCmax 内での参照が確認されています）
  • **呼び出し先**:
  • 外部関数への呼び出しはありません。

関連データ構造

  • comboCids: Combo CIDをインデックスとして cmin/cmax のペアを保持する配列。
  • CommandId: トランザクション内でのコマンドの順序を示す識別子。

失敗モードと境界条件

  • combocid が usedComboCids 以上の場合、アサートが有効なビルドではプログラムが停止します。

保守担当者向けメモ

  • 本関数は static であり、combocid.c モジュール内でのみ使用されます。
  • comboCids 配列のインデックス参照は高速ですが、インデックスの正当性は呼び出し側で保証されている必要があります。
