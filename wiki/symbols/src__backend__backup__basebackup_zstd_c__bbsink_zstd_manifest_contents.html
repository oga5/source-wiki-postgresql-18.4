bbsink_zstd_manifest_contents

定義

  • src/backend/backup/basebackup_zstd.c:301

目的

Zstandard圧縮シンクにおいて、マニフェストデータを圧縮処理を介さずに後続のシンクへ透過的に転送する。

シグネチャまたは形状

static void bbsink_zstd_manifest_contents(bbsink *sink, size_t len)

入力と出力

  • sink: 現在のZstd圧縮シンクの状態を保持する bbsink 構造体へのポインタ。
  • len: 転送するデータのバイト長。
  • 出力: なし（副作用として後続シンクのバッファへデータがコピーされ、転送処理が実行される）。

前提条件と不変条件

  • sink および sink->bbs_next が有効であること。
  • sink->bbs_buffer に少なくとも len バイトのデータが存在すること。
  • マニフェストデータは圧縮対象外であるため、本関数は圧縮を行わない。

アルゴリズムまたは振る舞い

1. sink->bbs_buffer から sink->bbs_next->bbs_buffer へ、指定された長さ len 分のデータを memcpy でコピーする。 2. bbsink_manifest_contents を呼び出し、後続のシンクに対してマニフェストデータの転送を委譲する。

呼び出し元と呼び出し先

  • 呼び出し元: bbsink_zstd_ops 構造体（関数ポインタ経由での呼び出し）。
  • 呼び出し先: bbsink_manifest_contents（後続のシンク）。

関連データ構造

  • bbsink: バックアップパイプラインの各段階を定義する構造体。
  • bbsink_zstd_ops: bbsink インターフェースを実装する関数ポインタテーブル。

失敗モードと境界条件

  • sink->bbs_next が NULL の場合、セグメンテーション違反が発生する可能性がある（呼び出し元で保証される前提）。
  • len がバッファサイズを超過した場合の挙動は、memcpy の仕様およびバッファ確保状況に依存する。

保守担当者向けメモ

  • 本関数はマニフェストファイルを圧縮パイプラインからバイパスするために使用される。圧縮ロジック（ZSTD_compressStream2 等）は一切含まれない。
  • bbsink_zstd_ops の .manifest_contents フィールドに登録されている。
