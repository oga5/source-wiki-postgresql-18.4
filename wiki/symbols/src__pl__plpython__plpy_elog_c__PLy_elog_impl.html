PLy_elog_impl

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:43

目的

Python の例外状態を PostgreSQL のエラー報告システム（ereport）へ変換し、ログ出力やクライアントへの通知を行うためのブリッジ機能を提供します。

シグネチャまたは形状

void PLy_elog_impl(int elevel, const char *fmt, ...)

入力と出力

  • **入力**:
  • elevel: PostgreSQL のエラーレベル（ERROR, FATAL 等）。
  • fmt: オプションのフォーマット文字列（printf 形式）。
  • ...: フォーマット引数。
  • **出力**: なし（void）。副作用として ereport を介したエラー報告が行われます。

前提条件と不変条件

  • Python の例外状態（PyErr）が設定されていることを前提とします。
  • PG_TRY / PG_FINALLY ブロック内で実行され、Python オブジェクトの参照カウント管理とメモリ解放が保証されます。

アルゴリズムまたは振る舞い

1. PyErr_Fetch で現在の Python 例外情報を取得します。 2. PyErr_NormalizeException を実行し、例外の種類を判定します。 3. PLy_exc_spi_error または PLy_exc_error に該当する場合、関連するエラー詳細（SQLSTATE、ヒント、位置情報等）を抽出します。 4. PLy_traceback を呼び出し、Python のスタックトレースを整形します。 5. fmt が指定されている場合、StringInfo を用いてメッセージを構築します。 6. ereport を呼び出し、抽出した情報を PostgreSQL のエラー診断フィールドにマッピングして報告します。 7. PG_FINALLY ブロックにて、取得した Python オブジェクトの参照カウントを減らし、動的に確保した文字列バッファを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_elog.h 内の PLy_elog マクロ経由で広く利用されます。
  • contrib/jsonb_plpython/jsonb_plpython.c からの参照が確認されています。
  • **呼び出し先**:
  • PyErr_Fetch, PyErr_NormalizeException, PyErr_GivenExceptionMatches (Python C API)
  • PLy_get_spi_error_data, PLy_get_error_data, PLy_traceback (PL/Python 内部関数)
  • ereport, initStringInfo, appendStringInfoVA, pfree (PostgreSQL 内部関数)

関連データ構造

  • StringInfoData: メッセージ構築用バッファ。
  • PyObject: Python の例外オブジェクト（exc, val, tb）。

失敗モードと境界条件

  • Python 例外が設定されていない場合、デフォルトの SQLSTATE (ERRCODE_EXTERNAL_ROUTINE_EXCEPTION) が使用されます。
  • fmt が NULL の場合、Python の例外メッセージが優先的に使用されます。
  • メモリ不足時は ereport が PostgreSQL のエラー処理機構により適切にハンドリングします。

保守担当者向けメモ

  • 本関数は PG_TRY ブロック内で volatile なポインタ変数を使用しており、例外発生時のスタック巻き戻しによるメモリリークを防ぐ設計となっています。修正時は PG_FINALLY ブロックでの解放漏れに注意してください。
