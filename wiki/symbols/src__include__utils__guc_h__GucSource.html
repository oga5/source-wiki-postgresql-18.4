GucSource

定義

  • 種別: enum
  • 定義箇所: src/include/utils/guc.h:111

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおいて、設定パラメータがどのソースから提供されたかを識別し、設定値の優先順位を決定するために使用されます。

シグネチャまたは形状

typedef enum {
    PGC_S_DEFAULT,
    PGC_S_DYNAMIC_DEFAULT,
    PGC_S_ENV_VAR,
    PGC_S_FILE,
    PGC_S_ARGV,
    PGC_S_GLOBAL,
    PGC_S_DATABASE,
    PGC_S_USER,
    PGC_S_DATABASE_USER,
    PGC_S_CLIENT,
    PGC_S_OVERRIDE,
    PGC_S_INTERACTIVE,
    PGC_S_TEST,
    PGC_S_SESSION
} GucSource;

入力と出力

  • 入力: 設定変更を試みる際、その変更元を示す値として関数（check_* フック等）に渡されます。
  • 出力: 設定の優先順位判定ロジックにおいて、現在の設定値が上書き可能かどうかを判断する基準となります。

前提条件と不変条件

  • 列挙型の順序は、設定の優先順位や適用可否を決定するロジックに直結しています。
  • PGC_S_INTERACTIVE は、エラー報告における境界線として機能します。

アルゴリズムまたは振る舞い

GUCシステムは、この列挙値を用いて設定の適用優先度を管理します。例えば、postgresql.conf (PGC_S_FILE) からの設定よりも、SET コマンド (PGC_S_SESSION) による設定が優先されるといった制御が行われます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/basic_archive/basic_archive.c
  • src/backend/access/table/tableamapi.c
  • src/backend/access/transam/clog.c
  • src/backend/access/transam/commit_ts.c
  • src/backend/access/transam/multixact.c
  • src/backend/access/transam/subtrans.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogprefetcher.c
  • 呼び出し先: 本シンボル自体は列挙型であり、直接的な呼び出し先はありません。

関連データ構造

  • GucContext: 設定変更が許可されるタイミングを定義する列挙型。
  • ConfigVariable: 設定ファイル解析時に生成される名前・値ペアのリスト構造体。

失敗モードと境界条件

  • 列挙値の順序を変更する場合、GUCシステムの優先順位判定ロジック全体に影響を及ぼすため、慎重な検討が必要です。

保守担当者向けメモ

  • 本シンボルはGUCサブシステムのフロントエンドとして機能します。
  • フック関数（check_*）内でこの値を使用する際は、設定の適用コンテキストを正しく理解する必要があります。
