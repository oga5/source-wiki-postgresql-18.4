clone_file

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/copydir.c:237

目的

OSが提供する高速なファイルクローン機能（Reflink等）を利用して、ファイルシステムレベルで効率的にファイルの複製を行う。

シグネチャまたは形状

static void clone_file(const char *fromfile, const char *tofile)

入力と出力

  • fromfile: コピー元ファイルのパス（入力）。
  • tofile: コピー先ファイルのパス（出力）。
  • 戻り値: なし。成功時は処理を完了し、失敗時は ereport(ERROR, ...) により例外を発生させる。

前提条件と不変条件

  • プラットフォームが HAVE_COPYFILE (かつ COPYFILE_CLONE_FORCE) または HAVE_COPY_FILE_RANGE をサポートしていること。
  • サポートされていない環境で呼び出された場合は pg_unreachable() が実行される。

アルゴリズムまたは振る舞い

1. **プラットフォーム別の分岐**:

  • HAVE_COPYFILE が定義されている場合: copyfile() を COPYFILE_CLONE_FORCE フラグ付きで呼び出す。
  • HAVE_COPY_FILE_RANGE が定義されている場合:
  • OpenTransientFile を使用してソースを読み取り専用、宛先を書き込み専用（新規作成）で開く。
  • copy_file_range を使用し、最大 1MB 単位でループ処理を行う。
  • 各ループで CHECK_FOR_INTERRUPTS() を実行し、シグナルによる中断を可能にする。
  • pgstat_report_wait_start/end を呼び出し、待機イベント WAIT_EVENT_COPY_FILE_COPY を記録する。
  • 完了後、両方のファイル記述子を閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/file/copydir.c 内のディレクトリ走査処理から呼び出される。
  • **呼び出し先**: OpenTransientFile, CloseTransientFile, copy_file_range, CHECK_FOR_INTERRUPTS, pgstat_report_wait_start, pgstat_report_wait_end, ereport。

関連データ構造

  • O_RDONLY, O_WRONLY, O_CREAT, O_EXCL, PG_BINARY: ファイルオープン時のフラグ。

失敗モードと境界条件

  • ファイルのオープン失敗、クローン操作の失敗、またはクローズ失敗時に ereport(ERROR, ...) を発行する。
  • copy_file_range が EINTR 以外のエラーを返した場合、処理を中断してエラーを報告する。

保守担当者向けメモ

  • 本関数は OS の機能に依存するため、移植性を考慮して条件付きコンパイルが適用されている。
  • copy_file_range を使用する場合、ループ内で CHECK_FOR_INTERRUPTS() を呼び出しているため、長時間かかるコピー処理でもシグナルによる停止が可能である。
