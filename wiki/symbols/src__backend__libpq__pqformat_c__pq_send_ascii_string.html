pq_send_ascii_string

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqformat.c:226

目的

NULL終端文字列を、エンコーディング変換を行わずに StringInfo バッファへ追加します。非ASCII文字が含まれる場合は、安全のために ? に置換します。

シグネチャまたは形状

void pq_send_ascii_string(StringInfo buf, const char *str)

入力と出力

  • buf (入力/出力): データを追加する対象の StringInfo バッファ。
  • str (入力): 追加するNULL終端文字列。
  • 戻り値: なし。

前提条件と不変条件

  • str はNULL終端されている必要があります。
  • buf は初期化済みの有効な StringInfo である必要があります。
  • 文字列の各文字に対して IS_HIGHBIT_SET マクロによる判定が行われ、上位ビットが立っている（ASCII範囲外の）文字は ? に置換されます。

アルゴリズムまたは振る舞い

1. str が指す文字列を終端まで走査します。 2. 各文字 ch について、IS_HIGHBIT_SET(ch) が真であれば ch を '?' に置き換えます。 3. 処理した文字を appendStringInfoCharMacro を用いて buf に追加します。 4. ループ終了後、appendStringInfoChar を用いてNULL終端文字 \0 を buf に追加します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/error/elog.c:3524 からの呼び出しが確認されています。
  • 呼び出し先: appendStringInfoCharMacro および appendStringInfoChar を呼び出します。

関連データ構造

  • StringInfo: PostgreSQLの動的バッファ管理構造体。

失敗モードと境界条件

  • buf がメモリ不足に陥った場合、appendStringInfoChar 内部でエラーが発生し処理が中断される可能性があります。
  • 入力文字列に非ASCII文字が含まれる場合、データは破壊的（? への置換）に変換されます。

保守担当者向けメモ

  • エンコーディング変換が必要な場合は pq_sendstring を使用してください。本関数は変換を意図的に回避し、ASCII互換性を強制する場合にのみ使用されます。
