SplitIdentifierString

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varlena.c:3580

目的

カンマ等の区切り文字で連結された文字列を解析し、個別の識別子（名前）のリストに分割します。二重引用符で囲まれた識別子や、引用符内のエスケープ（二重引用符の連続）をサポートします。

シグネチャまたは形状

bool SplitIdentifierString(char *rawstring, char separator, List **namelist);

入力と出力

  • **入力**:
  • rawstring: 解析対象の文字列。**破壊的に変更される**ため、呼び出し元でコピーが必要な場合があります。
  • separator: 識別子を区切る文字。
  • namelist: 分割された識別子を格納する List へのポインタ。
  • **出力**:
  • 成功時は true を返し、namelist に識別子のリストが格納されます。
  • 構文エラー（引用符の不整合や空の識別子など）時は false を返します。

前提条件と不変条件

  • rawstring は書き込み可能なメモリ領域である必要があります。
  • 引用符で囲まれていない識別子は、小文字に変換（downcase_truncate_identifier）され、長さ制限が適用されます。
  • 引用符で囲まれた識別子は、引用符の連続（""）が単一の引用符に置換されますが、大文字小文字の変換は行われません。

アルゴリズムまたは振る舞い

1. 先頭の空白をスキップします。 2. 文字列を走査し、識別子を順次抽出します。

  • 引用符で始まる場合: 終了の引用符まで読み込み、内部の "" を " に置換します。
  • 引用符以外の場合: 区切り文字または空白までを識別子とし、小文字化と切り詰めを行います。

    • 3. 抽出した識別子を lappend で namelist に追加します。 4. 区切り文字をヌル文字（\0）で上書きして文字列を分割し、次の識別子へ進みます。 5. 文字列の末尾に到達するまで繰り返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/option.c
  • src/backend/access/transam/xlog.c
  • src/backend/catalog/namespace.c
  • src/backend/commands/extension.c
  • src/backend/commands/publicationcmds.c
  • src/backend/commands/tablespace.c
  • **呼び出し先**:
  • downcase_truncate_identifier, truncate_identifier, lappend, pfree, strchr, memmove, scanner_isspace 等。

関連データ構造

  • List: PostgreSQLのリスト構造体（namelist として使用）。

失敗モードと境界条件

  • 引用符が閉じられていない場合、false を返します。
  • 引用符なしの識別子が空（例: ,,）の場合、false を返します。
  • 不正な構文（識別子以外の予期せぬ文字など）が検出された場合、false を返します。

保守担当者向けメモ

  • 本関数は入力文字列を破壊的に変更します。呼び出し元で pstrdup 等を使用して元の文字列を保護しているか確認してください。
  • 引用符なしの識別子に対する小文字変換は、現在の実装では入力文字列の長さを超える変換をサポートしていません。将来的な拡張時には注意が必要です。
