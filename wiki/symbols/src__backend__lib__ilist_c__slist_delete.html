slist_delete

定義

  • 種別: function
  • 定義箇所: src/backend/lib/ilist.c:30

目的

単方向連結リスト（slist_head）から、指定されたノード（slist_node）を検索して削除します。

シグネチャまたは形状

void slist_delete(slist_head *head, const slist_node *node)

入力と出力

  • **入力**:
  • head: 操作対象のリストのヘッドポインタ。
  • node: リストから削除する対象のノード。
  • **出力**: なし（void）。

前提条件と不変条件

  • 指定された node は、head が指すリスト内に存在している必要があります。存在しない場合、アサーションエラーが発生します。
  • リストの要素は一度に複数のリストに所属させることはできません。

アルゴリズムまたは振る舞い

1. リストの先頭から順にノードを走査します。 2. 対象の node と一致する要素が見つかった場合、その直前のノードの next ポインタを、削除対象の next ポインタで更新することでリストから切り離します。 3. 処理の最後に slist_check(head) を呼び出し、リストの整合性を検証します。 4. 計算量はリストの長さに比例する O(n) です。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc.c 内の複数箇所（guc_stack_list および guc_report_list の操作）。
  • **呼び出し先**:
  • slist_check（リストの整合性検証用）。

関連データ構造

  • slist_head: 単方向リストのヘッド構造体。
  • slist_node: リストの各要素を構成するノード構造体。

失敗モードと境界条件

  • **リスト内にノードが存在しない場合**: Assert(found) によりアサーション失敗となります。
  • **パフォーマンス**: O(n) の線形探索を行うため、リストが長い場合はパフォーマンス上のボトルネックとなる可能性があります。

保守担当者向けメモ

  • 頻繁に削除操作を行う場合は、ノード自体から削除可能な slist_delete_current の使用を検討してください。
  • リストの破損が疑われる場合は、ILIST_DEBUG を有効にしてビルドすることで詳細な検証が可能です。
