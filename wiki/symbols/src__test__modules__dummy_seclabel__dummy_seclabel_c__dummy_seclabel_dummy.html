dummy_seclabel_dummy

定義

  • 種別: function
  • 定義箇所: src/test/modules/dummy_seclabel/dummy_seclabel.c:56

目的

PostgreSQLの拡張モジュールとしてロード可能な状態を維持し、モジュールが空ではないことを保証するためのダミー関数です。

シグネチャまたは形状

Datum dummy_seclabel_dummy(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（PostgreSQLの関数呼び出し規約に基づく引数）
  • 出力: Datum 型（PG_RETURN_VOID() により、戻り値なしとして終了）

前提条件と不変条件

  • PostgreSQLの関数マネージャ（fmgr）経由で呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. 関数が呼び出されると、即座に PG_RETURN_VOID() を実行して制御を戻します。 2. 内部的な処理や状態変更は一切行いません。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録済み）。
  • 呼び出し先: なし。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数引数構造体。

失敗モードと境界条件

  • 特になし。常に正常終了します。

保守担当者向けメモ

  • 本関数はテストモジュールのロード可能性を担保するためのスタブです。実運用上のロジックは含まれていません。
