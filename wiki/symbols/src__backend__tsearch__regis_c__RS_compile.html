RS_compile

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/regis.c:84

目的

正規表現文字列を解析し、Regis 構造体およびその連結リストである RegisNode を構築する。

シグネチャまたは形状

void RS_compile(Regis *r, bool issuffix, const char *str)

入力と出力

  • **入力**:
  • r: 初期化対象の Regis 構造体へのポインタ。
  • issuffix: 接尾辞として扱うかを示すフラグ。
  • str: 解析対象の正規表現文字列。
  • **出力**: なし（r が指す構造体が更新される）。

前提条件と不変条件

  • r は有効なメモリ領域を指している必要がある。
  • str は本モジュールがサポートする限定的な正規表現形式（[abc] や [^abc] 等）であること。
  • RS_compile 実行後、Regis 構造体は RS_free による解放が必要となる。

アルゴリズムまたは振る舞い

1. memset で r をゼロ初期化し、issuffix フラグを設定する。 2. 文字列 str を走査し、状態遷移マシン（RS_IN_WAIT, RS_IN_ONEOF, RS_IN_ONEOF_IN, RS_IN_NONEOF）を用いて解析する。 3. newRegisNode を呼び出し、RegisNode の連結リストを生成する。

  • RSF_ONEOF: 指定文字のいずれかに一致。
  • RSF_NONEOF: 指定文字以外に一致。

    • 4. 各ノードの data に文字集合を格納する。 5. 最後に、構築された連結リストを走査して r->nchar（ノード数）をカウントする。

関連データ構造

  • Regis: 正規表現エンジンが使用するメイン構造体。
  • RegisNode: パターンを保持する連結リストのノード。

失敗モードと境界条件

  • サポート外の構文や不正な形式が入力された場合、elog(ERROR, ...) を呼び出し、トランザクションを中断する。
  • 内部状態遷移で予期せぬ値が発生した場合、内部エラーとして elog を発行する。

保守担当者向けメモ

  • 本関数は汎用的な正規表現エンジンではないため、構文拡張を行う際は RS_isRegis との整合性に注意すること。
  • マルチバイト文字の処理には pg_mblen_cstr を使用しているため、文字単位の走査ロジックを変更する際は注意が必要。
