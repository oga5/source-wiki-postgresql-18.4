FreePageManagerGet

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/freepage.c:209

目的

指定されたページ数（npages）のメモリ領域を FreePageManager から確保し、その開始ページ番号を返します。

シグネチャまたは形状

bool FreePageManagerGet(FreePageManager *fpm, Size npages, Size *first_page)

入力と出力

  • **入力**:
  • fpm: メモリ管理構造体へのポインタ。
  • npages: 確保したいページ数。
  • first_page: 確保された領域の開始ページ番号を格納する出力用ポインタ。
  • **出力**:
  • bool: 確保に成功した場合は true、失敗した場合は false。

前提条件と不変条件

  • FPM_EXTRA_ASSERTS が有効な場合、以下の不変条件が検証されます。
  • 確保成功時、fpm->free_pages が要求された npages 以上であること。
  • 確保後の fpm->free_pages が、管理下の全空きページの合計と一致すること。
  • fpm->contiguous_pages が、管理下の最大連続空きページ数と一致すること。

アルゴリズムまたは振る舞い

1. FreePageManagerGetInternal を呼び出し、メモリ確保を試みます。 2. 確保処理に伴う B-tree の構造変化を整理するため、FreePageBtreeCleanup を実行します。 3. FreePageBtreeCleanup の結果に基づき、fpm->contiguous_pages を更新します。 4. FreePageManagerUpdateLargest を呼び出し、必要に応じて最大連続ページ数を再計算します。 5. 成功時は fpm->free_pages を減算し、結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/ipc/dsm.c
  • src/backend/utils/mmgr/dsa.c
  • **呼び出し先**:
  • FreePageManagerGetInternal
  • FreePageBtreeCleanup
  • FreePageManagerUpdateLargest
  • FreePageManagerLargestContiguous (デバッグ時のみ)

関連データ構造

  • FreePageManager: メモリ管理のメイン構造体。

失敗モードと境界条件

  • メモリ不足により要求された npages を確保できない場合、false を返します。
  • 確保処理により B-tree のノードが再利用される際、再帰的なメモリ確保が発生する可能性があります。

保守担当者向けメモ

  • 本関数は、メモリ確保後のクリーンアップ処理を含んでおり、確保操作が結果的に連続領域を増やす可能性があるという副作用を考慮した設計になっています。
  • デバッグ時には FPM_EXTRA_ASSERTS を有効にすることで、整合性チェックが強化されます。
