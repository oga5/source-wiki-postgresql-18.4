ecpg_filter_source

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/pg_regress_ecpg.c:33

目的

ECPGの回帰テストにおいて、ソースコード内の #line ディレクティブに含まれる環境依存のパス情報を除去し、ファイル名のみに短縮することで、テスト結果の比較可能性を確保します。

シグネチャまたは形状

static void ecpg_filter_source(const char *sourcefile, const char *outfile)

入力と出力

  • **入力**:
  • sourcefile: 読み込み対象のソースファイルパス。
  • outfile: 正規化後の内容を書き出すファイルパス。
  • **出力**: なし（副作用としてファイルシステムへ書き出し）。

前提条件と不変条件

  • sourcefile は読み込み可能である必要があります。
  • outfile は書き込み可能である必要があります。
  • ファイルオープンに失敗した場合、プログラムは exit(2) を呼び出し終了します。

アルゴリズムまたは振る舞い

1. sourcefile を読み込みモード、outfile を書き込みモードで開きます。 2. pg_get_line_buf を使用して、ソースファイルを1行ずつ読み込みます。 3. 各行の先頭が #line で始まるか確認します。 4. #line 行の場合、ダブルクォーテーション " 内のパス部分を走査し、. や / が続くプレフィックス部分を特定します。 5. 特定したプレフィックス部分を memmove で削除し、パスをファイル名のみの状態に書き換えます。 6. 処理後の行を outfile に書き出します。 7. 最後にバッファを解放し、両ファイルを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/pg_regress_ecpg.c 内のテスト実行フロー（ecpg_start_test 相当の処理）から呼び出されます。
  • **呼び出し先**: fopen, fprintf, exit, initStringInfo, pg_get_line_buf, strstr, strchr, memmove, strlen, fputs, pfree, fclose（すべて標準ライブラリまたはPostgreSQLのユーティリティ関数）。

関連データ構造

  • StringInfoData: 行単位のバッファリングに使用される動的文字列構造体。

失敗モードと境界条件

  • ファイルが開けない場合、標準エラー出力にメッセージを出力して異常終了します。
  • #line 行の形式が想定と異なる場合、パスの削除処理はスキップされ、元の行がそのまま出力されます。

保守担当者向けメモ

  • 本関数はインプレースでの正規化ではなく、別ファイルへの書き出しを行う設計です。
  • memmove によるパス削除処理は、#line ディレクティブの構文が厳密であることを前提としています。将来的にプリプロセッサの出力形式が変更された場合は、このパースロジックの修正が必要です。
