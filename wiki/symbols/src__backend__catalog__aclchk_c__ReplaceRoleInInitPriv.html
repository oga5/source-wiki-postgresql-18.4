ReplaceRoleInInitPriv

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/aclchk.c:4758

目的

システムカタログ pg_init_privs に格納されている初期権限情報において、特定のロール（oldroleid）を別のロール（newroleid）に置換する。

シグネチャまたは形状

void ReplaceRoleInInitPriv(Oid oldroleid, Oid newroleid, Oid classid, Oid objid, int32 objsubid)

入力と出力

  • **入力**:
  • oldroleid: 置換対象のロールOID。
  • newroleid: 置換後のロールOID。
  • classid: 対象オブジェクトのカタログOID。
  • objid: 対象オブジェクトのOID。
  • objsubid: 対象オブジェクトのサブID（列番号等）。
  • **出力**: なし（副作用として pg_init_privs カタログおよび共有依存関係を更新）。

前提条件と不変条件

  • pg_init_privs テーブルに対する RowExclusiveLock を保持して実行される。
  • 対象オブジェクトに対応する pg_init_privs エントリは最大で1つ存在することを前提とする。

アルゴリズムまたは振る舞い

1. pg_init_privs をスキャンし、指定された objid, classid, objsubid に一致するタプルを取得する。 2. タプルが存在しない場合は、何もせず終了する。 3. 既存の ACL データを取得し、aclnewowner を用いてロールを置換した新しい ACL を生成する。 4. 新しい ACL が空であれば、CatalogTupleDelete でエントリを削除する。 5. そうでなければ、CatalogTupleUpdate で pg_init_privs を更新する。 6. updateInitAclDependencies を呼び出し、共有依存関係を新しい ACL に合わせて同期する。 7. CommandCounterIncrement を実行し、同一トランザクション内での後続処理に備える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_shdepend.c（ロール削除や変更時の依存関係処理に関連）
  • src/backend/catalog/aclchk.c 内の関連処理
  • **呼び出し先**:
  • table_open / table_close
  • systable_beginscan / systable_getnext / systable_endscan
  • aclnewowner
  • CatalogTupleUpdate / CatalogTupleDelete
  • updateInitAclDependencies
  • CommandCounterIncrement

関連データ構造

  • pg_init_privs: 初期権限を保持するシステムカタログ。
  • Acl: アクセス制御リスト構造体。

失敗モードと境界条件

  • 対象オブジェクトのエントリが存在しない場合、エラーにはならず静かに終了する。
  • 置換の結果 ACL が空になった場合、エントリ自体が削除される。

保守担当者向けメモ

  • aclnewowner を使用しているが、本来の所有者変更以外のロール置換にも適用している点に注意すること。
  • CommandCounterIncrement が含まれているため、本関数を呼び出す際はトランザクション内の他の操作との順序関係に留意が必要である。
