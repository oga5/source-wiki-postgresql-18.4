ExecReScanProjectSet

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeProjectSet.c:336

目的

ProjectSet 実行ノードの状態をリセットし、再スキャンを可能にする。特に、集合を返す関数（SRF）の評価状態を初期化する。

シグネチャまたは形状

void ExecReScanProjectSet(ProjectSetState *node)

入力と出力

  • **入力**: ProjectSetState *node - 再スキャン対象の ProjectSet ノードの実行時状態。
  • **出力**: なし（副作用として node の状態を更新）。

前提条件と不変条件

  • node は有効な ProjectSetState を指している必要がある。
  • outerPlanState(node) を介して子ノード（outerPlan）にアクセス可能であること。

アルゴリズムまたは振る舞い

2. 子ノードの chgParam を確認する。

  • chgParam が NULL の場合、子ノードに対して ExecReScan を呼び出し、子ノード全体を再スキャンする。
  • chgParam が NULL でない場合、子ノードは最初の ExecProcNode 呼び出し時に自動的に再スキャンされるため、ここでは何もしない。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execAmi.c:138
  • **呼び出し先**:
  • ExecReScan (子ノードに対して条件付きで実行)

関連データ構造

  • ProjectSetState: ノードの実行時状態を保持する構造体。pending_srf_tuples フラグを含む。
  • PlanState: 子ノードの実行状態。chgParam を保持する。

失敗モードと境界条件

  • node が NULL の場合、セグメンテーション違反が発生する可能性がある（呼び出し元でのガードはソースからは不明）。
  • 子ノードが NULL の場合、outerPlanState(node) の呼び出しで失敗する可能性がある。

保守担当者向けメモ

  • 本関数は SRF の反復評価を中断・リセットする役割を持つ。pending_srf_tuples のクリアは、SRF の評価サイクルを強制的に最初からやり直すために不可欠である。
