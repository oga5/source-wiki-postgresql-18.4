helpSQL

定義

  • 種別: function
  • 定義箇所: src/bin/psql/help.c:592

目的

psql において SQL コマンドの構文や説明を表示します。引数なしの場合は利用可能な全コマンドの一覧を、引数ありの場合は該当するコマンドの詳細情報を出力します。

シグネチャまたは形状

void helpSQL(const char *topic, unsigned short int pager)

入力と出力

  • **入力**:
  • topic: 検索対象の SQL コマンド名（文字列）。空または NULL の場合は全コマンド一覧を表示。
  • pager: ページャの使用を制御するフラグ。
  • **出力**: 標準出力（または PageOutput を介したページャ）へヘルプテキストを出力。

前提条件と不変条件

  • sql_help.h に定義された QL_HELP 配列が有効であること。
  • pset（現在の psql 設定）が初期化されていること。
  • ページングが必要な場合、PageOutput および ClosePager を介して出力が管理されること。

アルゴリズムまたは振る舞い

1. **引数なしの場合**:

  • 端末幅を ioctl（TIOCGWINSZ）で取得し、列数を計算。
  • QL_HELP 配列を走査し、コマンド名を整形して出力。

    • 2. **引数ありの場合**:

  • topic に基づき、最大3段階のパスで部分一致検索を実行（完全一致、先頭2単語、先頭1単語の順）。
  • 一致したコマンドについて、syntaxfunc を呼び出して構文情報を取得。
  • PostgreSQL 公式ドキュメントへの URL を生成し、コマンド名、説明、構文、URL を出力。
  • 完全一致が見つかった時点で検索を終了。

    • 3. **共通**:

  • 検索結果が見つからない場合は、その旨を通知。
  • ページャが有効な場合、PageOutput で開始し ClosePager で終了。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c:2031（\h コマンド実行時）。
  • **呼び出し先**:
  • PageOutput, ClosePager（fe_utils/print.c 関連）
  • initPQExpBuffer, termPQExpBuffer（libpq 関連）
  • QL_HELP（sql_help.h）

関連データ構造

  • QL_HELP: SQL コマンドのヘルプ情報を含む配列。
  • PQExpBufferData: 構文文字列を構築するための一時バッファ。
  • pset: psql の現在の設定状態を保持する構造体。

失敗モードと境界条件

  • topic が見つからない場合: 「No help available」というメッセージを表示。
  • ioctl が失敗した場合: 端末幅を 80 カラムと仮定して動作。
  • ページャが指定されていても出力内容が空の場合: ページャは起動されない。

保守担当者向けメモ

  • ヘルプ内容を更新した際は、必ず ref/psql-ref.sgml（DocBook）との同期を確認すること。
  • helpSQL は部分一致検索を行うため、\h SELECT のような正確な指定だけでなく、曖昧な検索にも対応している。
  • ページャの行数計算ロジックは、出力ループのロジックと厳密に一致させる必要がある。
