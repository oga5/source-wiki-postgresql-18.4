ECPGget_desc_header

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/descriptor.c:83

目的

指定された名前のSQL記述子に関連付けられた結果セットの列数（属性数）を取得し、アプリケーションへ返します。

シグネチャまたは形状

bool ECPGget_desc_header(int lineno, const char *desc_name, int *count)

入力と出力

  • **入力**:
  • lineno: 呼び出し元の行番号（エラーログ用）。
  • desc_name: 取得対象の記述子名。
  • **出力**:
  • count: 記述子に含まれる列数へのポインタ。
  • **戻り値**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • sqlca 構造体が取得可能である必要があります。
  • 記述子名に対応する PGresult が存在する必要があります。

アルゴリズムまたは振る舞い

1. ECPGget_sqlca() を呼び出し、SQLCA 構造体を初期化します。取得失敗時はエラーを発生させ false を返します。 2. ecpg_result_by_descriptor() を使用して、指定された desc_name に対応する PGresult を検索します。見つからない場合は false を返します。 3. PQnfields() を呼び出し、結果セットの列数を取得して count に格納します。 4. sqlca->sqlerrd[2] を 1 に設定し、成功を記録します。 5. 成功時に ecpg_log で列数を出力し、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGプリプロセッサが生成するコード（src/interfaces/ecpg/preproc/descriptor.c）およびテストコード（compat_informix-describe.c, preproc-describe.c など）。
  • **呼び出し先**:
  • ECPGget_sqlca()
  • ecpg_raise()
  • ecpg_init_sqlca()
  • ecpg_result_by_descriptor()
  • PQnfields() (libpq)
  • ecpg_log()

関連データ構造

  • sqlca_t: SQL通信領域。
  • PGresult: libpq のクエリ結果構造体。

失敗モードと境界条件

  • ECPGget_sqlca() が NULL を返した場合、メモリ不足エラーとして処理されます。
  • ecpg_result_by_descriptor() が NULL を返した場合（記述子が存在しない等）、関数はそのまま false を返します。

保守担当者向けメモ

  • 本関数は va_list を使用しない単純な属性取得関数です。
  • sqlca->sqlerrd[2] への値代入は、SQL標準の動作を模倣するためのものです。
