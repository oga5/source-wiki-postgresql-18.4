relmap_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/relmapdesc.c:34

目的

WAL（Write-Ahead Log）レコードの info フィールドに基づき、リレーションマッピングに関連する操作の種類を識別し、人間が読み取り可能な文字列として返します。

シグネチャまたは形状

const char *relmap_identify(uint8 info);

入力と出力

  • **入力**: uint8 info - WALレコードの info ビットフィールド。
  • **出力**: const char * - 操作タイプを表す文字列（例: "UPDATE"）。該当しない場合は NULL。

前提条件と不変条件

  • 入力された info ビットフィールドから XLR_INFO_MASK を除去した値が、既知の操作タイプと一致する必要があります。

アルゴリズムまたは振る舞い

1. 入力された info に対してビットマスク ~XLR_INFO_MASK を適用し、フラグ部分を除去して純粋な操作タイプを抽出します。 2. switch 文を用いて、抽出された値が XLOG_RELMAP_UPDATE と一致するか判定します。 3. 一致する場合、文字列 "UPDATE" を返します。 4. 一致しない場合、初期値である NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h において、リソースマネージャ（RM_RELMAP_ID）のコールバック関数として登録されています。
  • **呼び出し先**: 直接的な関数呼び出しは行いません。

関連データ構造

  • XLOG_RELMAP_UPDATE: 識別対象となるWALレコードの操作タイプ定数。
  • XLR_INFO_MASK: 操作タイプを特定するために info から除去されるビットマスク。

失敗モードと境界条件

  • info が既知の操作タイプ（XLOG_RELMAP_UPDATE）に該当しない場合、関数は NULL を返します。

保守担当者向けメモ

  • 新しいWALレコードタイプが relmapper.c に追加された場合、本関数の switch 文に新しい case を追加する必要があります。
