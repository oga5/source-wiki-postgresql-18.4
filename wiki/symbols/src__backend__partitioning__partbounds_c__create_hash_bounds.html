create_hash_bounds

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partbounds.c:346

目的

ハッシュパーティションの定義（PartitionBoundSpec）を、実行時に効率的な検索が可能な内部表現（PartitionBoundInfo）へ変換します。

シグネチャまたは形状

static PartitionBoundInfo create_hash_bounds(PartitionBoundSpec **boundspecs, int nparts, PartitionKey key, int **mapping)

入力と出力

  • **入力**:
  • boundspecs: パーティション境界定義の配列。
  • nparts: パーティション数。
  • key: パーティションキー情報。
  • mapping: パーティションのインデックスを再配置するためのポインタ。
  • **出力**:
  • PartitionBoundInfo: 構築されたハッシュパーティションの境界情報構造体。

前提条件と不変条件

  • 入力される boundspecs の各要素の戦略は PARTITION_STRATEGY_HASH である必要があります。
  • greatest_modulus（最大モジュラス）が boundinfo->nindexes として設定され、indexes 配列は初期値 -1 で確保されます。

アルゴリズムまたは振る舞い

1. PartitionBoundInfoData を初期化し、ハッシュパーティション用のメタデータを設定します。 2. 各 PartitionBoundSpec からモジュラスと剰余（remainder）を抽出し、PartitionHashBound 配列に格納します。 3. qsort を用いてハッシュ境界をモジュラスの昇順にソートします。 4. 最大モジュラスに基づき indexes 配列を確保し、-1 で初期化します。 5. 各パーティションの剰余から開始し、最大モジュラスに達するまで remainder += modulus を繰り返して indexes 配列を埋めます。この際、重複が発生しないことを Assert で検証します。 6. mapping 配列を更新し、一時的な hbounds 配列を解放して結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: partition_bounds_create（src/backend/partitioning/partbounds.c 内）から呼び出されます。
  • **呼び出し先**: palloc0, palloc, pfree, qsort, qsort_partition_hbound_cmp, Int32GetDatum 等のメモリ管理およびユーティリティ関数。

関連データ構造

  • PartitionBoundInfo: 変換後の境界情報。
  • PartitionHashBound: ソートおよび処理用の一時的な境界構造体。
  • PartitionBoundSpec: 入力となるカタログ定義。

失敗モードと境界条件

  • spec->strategy が PARTITION_STRATEGY_HASH でない場合、elog(ERROR) により処理が中断されます。
  • 剰余の計算過程で indexes 配列の重複が発生した場合、Assert により検証されます。

保守担当者向けメモ

  • 本関数はハッシュパーティションの検索最適化のために indexes 配列を構築します。indexes は greatest_modulus のサイズを持ち、ハッシュ値から対応するパーティションインデックスを高速に特定するために使用されます。
  • mapping 配列の更新は、ソート後の順序を呼び出し元に反映させるために不可欠です。
