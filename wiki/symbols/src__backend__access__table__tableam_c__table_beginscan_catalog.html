table_beginscan_catalog

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableam.c:112

目的

システムカタログテーブルに対するスキャンを開始するための初期化処理を行います。カタログ専用のスナップショットを取得・登録し、適切なスキャンフラグを設定してテーブルアクセスメソッド（AM）の scan_begin を呼び出します。

シグネチャまたは形状

TableScanDesc table_beginscan_catalog(Relation relation, int nkeys, struct ScanKeyData *key)

入力と出力

  • **入力**:
  • relation: スキャン対象のリレーション（Relation）。
  • nkeys: スキャンキーの数。
  • key: スキャンキーの配列（ScanKeyData）。
  • **出力**:
  • TableScanDesc: 初期化されたテーブルスキャン記述子。

前提条件と不変条件

  • relation は有効なカタログリレーションである必要があります。
  • 内部で GetCatalogSnapshot を呼び出すため、カタログスナップショットが利用可能なコンテキストで実行される必要があります。

アルゴリズムまたは振る舞い

1. スキャンフラグ flags を定義します（SO_TYPE_SEQSCAN, SO_ALLOW_STRAT, SO_ALLOW_SYNC, SO_ALLOW_PAGEMODE, SO_TEMP_SNAPSHOT）。 2. RelationGetRelid を用いてリレーションIDを取得します。 3. GetCatalogSnapshot でカタログスナップショットを取得し、RegisterSnapshot で登録します。 4. relation->rd_tableam->scan_begin を呼び出し、AM固有の初期化処理に委譲します。その際、取得したスナップショットと定義したフラグを渡します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/bootstrap/bootstrap.c
  • src/backend/catalog/aclchk.c
  • src/backend/catalog/pg_db_role_setting.c
  • src/backend/catalog/pg_publication.c
  • src/backend/catalog/pg_subscription.c
  • src/backend/commands/cluster.c
  • src/backend/commands/dbcommands.c
  • **呼び出し先**:
  • RelationGetRelid
  • GetCatalogSnapshot
  • RegisterSnapshot
  • relation->rd_tableam->scan_begin（AM固有の実装）

関連データ構造

  • TableScanDesc: スキャン状態を保持する構造体。
  • Relation: テーブルのメタデータ構造体。
  • ScanKeyData: スキャン条件を定義する構造体。
  • TableAmRoutine: rd_tableam を介して参照されるAMインターフェース定義。

失敗モードと境界条件

  • relation が無効な場合、または rd_tableam が適切に初期化されていない場合はクラッシュする可能性があります。
  • スナップショットの登録に失敗した場合、メモリ管理上のエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はカタログスキャン専用のフラグセットを固定で適用します。カタログ以外のテーブルスキャンには使用しないでください。
  • rd_tableam を介したディスパッチを行っているため、具体的な挙動はリレーションのストレージ形式（Heap等）に依存します。
