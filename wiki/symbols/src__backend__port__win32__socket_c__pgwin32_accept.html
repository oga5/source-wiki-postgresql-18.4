pgwin32_accept

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/socket.c:336

目的

Windows環境において、POSIX準拠の accept() 関数をWinsock2 API上でエミュレートし、PostgreSQLのシグナル処理と統合する。

シグネチャまたは形状

SOCKET pgwin32_accept(SOCKET s, struct sockaddr *addr, int *addrlen)

入力と出力

  • **入力**:
  • s: 接続待ち状態のソケットハンドル。
  • addr: 接続先アドレス情報を格納する構造体へのポインタ。
  • addrlen: addr が指す構造体のサイズを格納する整数へのポインタ。
  • **出力**:
  • 成功時: 新しく作成されたソケットハンドル。
  • 失敗時: INVALID_SOCKET。

前提条件と不変条件

  • pgwin32_poll_signals() を呼び出し、保留中のシグナルを処理する。
  • pqcomm.c 側の仕様により、シグナル割り込みが発生しても EINTR を返さず、処理を継続する。

アルゴリズムまたは振る舞い

1. pgwin32_poll_signals() を呼び出し、シグナル割り込みを処理する。 2. WSAAccept() を呼び出し、接続要求を受け入れる。 3. 戻り値が INVALID_SOCKET の場合、TranslateSocketError() を呼び出してWinsockエラーコードをPOSIX準拠の errno に変換する。 4. 成功した場合はソケットハンドルを返し、失敗した場合は INVALID_SOCKET を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/win32_port.h にて accept マクロとして定義されており、PostgreSQLのネットワーク通信層から呼び出される。
  • **呼び出し先**:
  • pgwin32_poll_signals()
  • WSAAccept() (Winsock2 API)
  • TranslateSocketError()

関連データ構造

  • SOCKET: Winsockのソケットハンドル型。
  • struct sockaddr: ソケットアドレス情報。

失敗モードと境界条件

  • WSAAccept が INVALID_SOCKET を返した場合、TranslateSocketError によりエラー状態が更新される。
  • シグナル処理中および接続受け入れ処理中に発生するエラーは、標準的な errno にマッピングされる。

保守担当者向けメモ

  • 本関数は accept のラッパーであり、Windows特有のシグナル処理を挿入している。
  • pqcomm.c が EINTR を想定していないため、シグナル発生時もエラーを返さず処理を続行する設計となっている。
