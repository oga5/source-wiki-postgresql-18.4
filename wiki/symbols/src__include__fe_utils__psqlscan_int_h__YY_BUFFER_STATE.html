YY_BUFFER_STATE

定義

  • 種別: typedef (構造体へのポインタ)
  • 定義箇所: src/include/fe_utils/psqlscan_int.h:56

目的

Flex字句解析器が使用する入力バッファの状態を管理するための不透明なハンドルです。

シグネチャまたは形状

typedef struct yy_buffer_state *YY_BUFFER_STATE;

入力と出力

  • 入力: Flexスキャナが読み込む入力ストリームの制御情報。
  • 出力: スキャナの現在の読み込み位置やバッファ内容を保持する構造体へのポインタ。

前提条件と不変条件

  • Flexによって生成されたスキャナの内部構造に依存します。
  • psqlscan_int.h 内で定義される PsqlScanStateData 構造体の一部として保持され、解析器の再入可能性を支えます。

アルゴリズムまたは振る舞い

  • Flexのバッファ切り替え機能（yy_switch_to_buffer 等）と連携し、psql変数展開などの際に解析対象の入力を動的に切り替えるために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/include/fe_utils/psqlscan_int.h:71 (StackElem 構造体内のメンバ)
  • src/include/fe_utils/psqlscan_int.h:96 (PsqlScanStateData 構造体内のメンバ)
  • 呼び出し先:
  • psqlscan_prepare_buffer 関数がこの型を戻り値として返します。

関連データ構造

  • StackElem: psql変数展開時のバッファスタック要素。
  • PsqlScanStateData: 解析器の全状態を保持する構造体。

失敗モードと境界条件

  • 外部のFlexライブラリのバージョンに依存するため、異なるバージョン間で混在させた場合、互換性の問題が発生するリスクがあります。

保守担当者向けメモ

  • 本シンボルはFlexの内部構造体へのポインタです。直接メンバにアクセスせず、FlexのAPIまたは psqlscan モジュールが提供するラッパー関数を介して操作してください。
