ResetCancelConn

定義

  • 種別: function
  • 定義箇所: src/fe_utils/cancel.c:106

目的

現在保持しているキャンセル用オブジェクト（PGcancel）を破棄し、関連するポインタを安全にリセットします。

シグネチャまたは形状

void ResetCancelConn(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルな cancelConn ポインタをクリアし、メモリを解放する）

前提条件と不変条件

  • Windows環境では、cancelConnLock クリティカルセクションを使用して排他制御が行われます。
  • cancelConn ポインタは、解放処理中にシグナルハンドラ（handle_sigint）から参照されないよう、解放前に NULL に設定されます。

アルゴリズムまたは振る舞い

1. Windows環境の場合、cancelConnLock を取得します。 2. 現在の cancelConn を一時変数 oldCancelConn に退避させます。 3. cancelConn を NULL に設定し、シグナルハンドラが解放済みのメモリを参照しないようにします。 4. oldCancelConn が NULL でなければ、PQfreeCancel を呼び出してメモリを解放します。 5. Windows環境の場合、cancelConnLock を解放します。

呼び出し元と呼び出し先

  • 呼び出し元: pgbench（pgbench.c）、psql（command.c, common.c, large_obj.c）の各所から呼び出されます。
  • 呼び出し先: PQfreeCancel（libpq ライブラリ関数）。

関連データ構造

  • PGcancel: libpq が提供するキャンセル要求用構造体。
  • cancelConn: 本モジュール内で管理される、現在の接続に対応する PGcancel へのポインタ。
  • cancelConnLock: Windows環境での排他制御用クリティカルセクション。

失敗モードと境界条件

  • cancelConn が既に NULL の場合、PQfreeCancel は呼び出されず、安全に終了します。

保守担当者向けメモ

  • 本関数は SetCancelConn と対で使用されます。
  • シグナルハンドラ内での安全性を確保するため、ポインタのクリアと解放の順序は厳守してください。
  • CancelRequested フラグは本関数ではリセットされません。必要に応じて呼び出し側で管理してください。
