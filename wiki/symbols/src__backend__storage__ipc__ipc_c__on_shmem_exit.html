on_shmem_exit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipc.c:371

目的

プロセス終了時の共有メモリ関連のクリーンアップ処理（低レベルリソースの解放など）を実行するためのコールバック関数を登録します。

シグネチャまたは形状

void on_shmem_exit(pg_on_exit_callback function, Datum arg);

入力と出力

  • function: 終了時に実行される関数ポインタ。
  • arg: コールバック関数に渡される Datum 型の引数。
  • 出力: なし（戻り値なし）。

前提条件と不変条件

  • 登録可能なコールバックの数には MAX_ON_EXITS という上限があります。
  • コールバックは登録された順序とは逆の順序（LIFO）で実行されます。

アルゴリズムまたは振る舞い

1. 現在の登録数（on_shmem_exit_index）が MAX_ON_EXITS に達していないか確認し、超過している場合は FATAL エラーを発生させます。 2. 指定された function と arg を on_shmem_exit_list 配列に格納します。 3. on_shmem_exit_index をインクリメントします。 4. atexit_callback_setup フラグを確認し、未設定であれば atexit(atexit_callback) を呼び出してプロセス終了時のバックストップ処理を有効化し、フラグを true に設定します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の例）:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/backend/port/posix_sema.c
  • src/backend/port/sysv_sema.c
  • src/backend/port/sysv_shmem.c
  • 呼び出し先:
  • atexit()（標準ライブラリ関数）
  • ereport()（エラー発生時）

関連データ構造

  • on_shmem_exit_list: コールバック関数と引数を保持する配列。
  • on_shmem_exit_index: 現在の登録位置を示すインデックス。
  • atexit_callback_setup: atexit が既に登録済みかを示すフラグ。

失敗モードと境界条件

  • on_shmem_exit_index >= MAX_ON_EXITS: 登録スロットが不足している場合、FATAL エラーとなりプロセスが終了します。

保守担当者向けメモ

  • 本関数は「共有メモリ関連」の終了処理を対象としています。高レベルなクリーンアップには before_shmem_exit を使用してください。
  • コールバック実行中にエラーが発生した場合、再帰的な呼び出しを防ぐためにリストから該当コールバックが削除される設計になっています。
