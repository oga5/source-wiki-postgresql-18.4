to_chars_df

定義

  • 種別: function
  • 定義箇所: src/common/d2s.c:630

目的

floating_decimal_64 構造体で表現された仮数と指数を、固定小数点形式の文字列としてバッファへ書き込みます。

シグネチャまたは形状

static inline int to_chars_df(const floating_decimal_64 v, const uint32 olength, char *const result)

入力と出力

  • **入力**:
  • v: 変換対象の floating_decimal_64 構造体（仮数と指数を保持）。
  • olength: 出力する数値の桁数。
  • result: 書き込み先の文字バッファ。
  • **出力**:
  • 戻り値: 書き込まれた文字列の総長（int）。

前提条件と不変条件

  • 呼び出し元は、必要に応じて負符号（-）の書き込みを完了していること。
  • mantissa * 10^exponent が出力すべき値であること。
  • 固定小数点表記として出力可能な範囲（指数が 16 未満など）であることを前提とする。

アルゴリズムまたは振る舞い

1. **初期化**: 指数と出力長に基づき、小数点位置や先行ゼロの配置を決定する。 2. **数値変換**: DIGIT_TABLE を使用し、uint32 単位で高速に数値を文字列へ変換する。 3. **フォーマット**:

  • 指数が負の場合、0.000... 形式のゼロ埋めを行う。
  • 指数が正の場合、必要に応じて末尾にゼロを補完する。
  • 小数点が必要な位置に挿入されるよう、memmove を用いて文字列をシフトする。

    • 4. **終了**: 最終的な文字列長を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/d2s.c 内の関数（to_chars などから呼び出される）。
  • **呼び出し先**: div1e8（64ビット除算用）、memcpy、memset、memmove。

関連データ構造

  • floating_decimal_64: 変換対象の数値を保持する構造体。
  • DIGIT_TABLE: 数値を文字列へ高速変換するためのルックアップテーブル。

失敗モードと境界条件

  • 指数が 16 以上の場合、本関数は固定小数点形式として適切に処理できない（科学的表記への切り替えが必要）。
  • nexp（指数と長さの合計）が負の場合、Assert により -3 までの範囲が検証される。

保守担当者向けメモ

  • 本関数は to_chars からの呼び出しを想定した内部関数です。
  • 128ビット演算が利用できない環境でのパフォーマンス低下については、d2s.c 全体のマクロ定義を確認してください。
  • 小数点挿入時の memmove は、小規模なシフトに最適化されています。変更時は性能への影響を考慮してください。
