get_raw_page

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/rawpage.c:70

目的

データベースの物理ストレージから指定されたリレーションのページを読み出し、そのバイナリイメージを bytea 形式で取得する。

シグネチャまたは形状

Datum get_raw_page(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relname (text): 対象リレーション名。
  • blkno (uint32): 読み出すブロック番号。
  • **出力**:
  • bytea: ページの内容を含むバイナリデータ。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要（物理データへの直接アクセスのため）。
  • 引数は正確に2つである必要がある（過去の互換性問題への対策）。

アルゴリズムまたは振る舞い

1. PG_NARGS() を使用して引数の数が2であることを検証し、不一致の場合はエラーを報告する。 2. get_raw_page_internal を呼び出し、MAIN_FORKNUM を指定して共有バッファからページイメージを取得する。 3. 取得したページイメージを bytea 型として呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • SQL インターフェース経由でユーザーから直接呼び出される。
  • contrib/pageinspect 内の各解析関数（brin_page_items, bt_page_items, hash_page_type 等）から、解析対象のページを取得するために呼び出される。
  • **呼び出し先**:
  • get_raw_page_internal: 実際のページ読み込み処理を実行。

関連データ構造

  • bytea: ページイメージを格納する PostgreSQL の可変長バイナリ型。
  • MAIN_FORKNUM: 読み込み対象のフォーク（メインデータ）。

失敗モードと境界条件

  • 引数の数が2でない場合、エラーが発生する。
  • 指定されたリレーションが存在しない、またはブロック番号が範囲外である場合、get_raw_page_internal 経由でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は pageinspect 拡張機能の基盤であり、他の解析関数がページデータを取得するための主要なエントリーポイントである。
  • 過去のバージョンとの互換性維持のため、引数の数に関するチェックが明示的に行われている点に注意すること。
