parseqatom

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/regex/regcomp.c:838

目的

正規表現のコンパイルプロセスにおいて、単一の量指定子付きアトム（atom）または制約（constraint）を解析し、NFA（非決定性有限オートマトン）の状態遷移を構築する。

シグネチャまたは形状

struct subre *parseqatom(struct vars *v, int stopper, int type, struct state *lp, struct state *rp, struct subre *top)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体。
  • stopper: 解析を終了させる文字（EOS または )）。
  • type: 解析対象の種別（LACON または PLAIN）。
  • lp, rp: アトムを配置する開始状態と終了状態。
  • top: 現在のサブ式ツリーのルート。
  • **出力**: 更新されたサブ式ツリーのルート（struct subre *）。

前提条件と不変条件

  • lp->nouts == 0 であること（新しいコードを lp と rp の間に接続するため）。
  • rp->nins == 0 であること。
  • 呼び出し時に v->nexttype が解析対象のトークンを指していること。

アルゴリズムまたは振る舞い

1. **初期化**: atom を NULL に初期化し、状態の前提条件を検証する。 2. **トークン分岐**: v->nexttype に基づき処理を分岐する。

  • **制約**: ^, $, SBEGIN, SEND, <, >, WBDRY, NWBDRY 等のアンカーや境界条件を処理し、NFA にアークを追加して即座に top を返す。
  • **ルックアラウンド (LACON)**: parse() を再帰的に呼び出し、内部構造を構築後、processlacon で制約を適用する。
  • **エラー**: 量指定子（*, + 等）が単独で現れた場合は REG_BADRPT を報告する。
  • **文字・クラス**: PLAIN 文字、[ (ブラケット式)、CCLASSS/CCLASSC (文字クラス)、. (ワイルドカード) を処理し、対応する NFA 遷移を生成する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regcomp.c 内の解析ループから呼び出される。
  • **呼び出し先**: newstate, newarc, parse, onechr, bracket, cbracket, charclass, charclasscomplement, rainbow, processlacon, word, nonword, wordchrs 等の内部関数。

関連データ構造

  • struct vars: コンパイル全体のコンテキスト。
  • struct subre: 正規表現の構造を表現するツリーノード。
  • struct state: NFA の状態。

失敗モードと境界条件

  • ERR() マクロにより、不正な構文（REG_BADRPT 等）やアサーション失敗時にエラー状態を設定し、呼び出し元へ制御を戻す。
  • NOERRN() マクロにより、エラー発生時に即座に中断する。
  • 括弧の不一致や不正なトークン配置に対してエラーを報告する。

保守担当者向けメモ

  • 本関数は parse() から呼び出され、正規表現の構文解析における「アトム」単位の処理を担う。
  • vars 構造体の nexttype を消費しながら進むため、再帰呼び出し時の状態管理に注意すること。
  • REG_DEBUG を有効にすると、コンパイル過程のダンプ出力が可能。
