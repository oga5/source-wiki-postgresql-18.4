FixedParamState

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_param.c:37

目的

SQLクエリ解析時において、事前に型が確定しているパラメータ（$1, $2 等）の型情報を保持し、パーサーが参照できるようにするための状態管理構造体です。

シグネチャまたは形状

typedef struct FixedParamState
{
	const Oid  *paramTypes;		/* array of parameter type OIDs */
	int			numParams;		/* number of array entries */
} FixedParamState;

入力と出力

  • **入力**: パラメータの型情報の配列（Oid）およびその要素数。
  • **出力**: ParseState のフック状態として保持され、解析中に ParamRef が出現した際の型解決に使用されます。

前提条件と不変条件

  • paramTypes は、解析対象のクエリに含まれるパラメータ数以上の要素を持つ配列である必要があります。
  • numParams は、paramTypes 配列の有効な要素数と一致している必要があります。

アルゴリズムまたは振る舞い

1. setup_parse_fixed_parameters 関数によって初期化され、ParseState の p_ref_hook_state に格納されます。 2. パーサーが ParamRef ノードを処理する際、この構造体を参照して対応するパラメータの型（Oid）を特定します。 3. 特定された型情報は、生成される Param ノードに反映されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_param.c:71（初期化時）
  • src/backend/parser/parse_param.c:102（フック処理内での参照）
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ParseState: 本構造体がフック状態として登録される解析コンテキスト。
  • ParamRef: パーサーがクエリ内でパラメータを検出した際に生成するノード。

失敗モードと境界条件

  • numParams を超えるインデックスのパラメータがクエリ内で参照された場合、型解決に失敗する可能性があります。
  • paramTypes が NULL の状態でフックが呼び出された場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は「明示的な $n 形式のパラメータ」の型解決に特化しています。
  • 可変パラメータを扱う VarParamState とは役割が異なるため、混同しないよう注意してください。
