ExtractExtensionList

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/option.c:453

目的

カンマ区切りの拡張機能名文字列を解析し、対応する PostgreSQL の拡張機能 OID のリストを抽出する。

シグネチャまたは形状

List *ExtractExtensionList(const char *extensionsString, bool warnOnMissing)

入力と出力

  • **入力**:
  • extensionsString: 拡張機能名をカンマで区切った文字列。
  • warnOnMissing: 存在しない拡張機能名が含まれていた場合に警告を発するかどうかを指定するフラグ。
  • **出力**:
  • 成功時: 見つかった拡張機能の OID を保持する List。
  • 失敗時: 文字列の形式が不正な場合、ereport(ERROR, ...) により処理が中断される。

前提条件と不変条件

  • extensionsString はカンマ区切りのリスト形式である必要がある。
  • SplitIdentifierString が内部で入力文字列を破壊的に変更するため、関数内で pstrdup を使用してコピーを作成している。

アルゴリズムまたは振る舞い

1. pstrdup で入力文字列を複製し、SplitIdentifierString を用いてカンマ区切りのリストに分割する。 2. 分割に失敗した場合は、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告する。 3. 分割された各名前に対して get_extension_oid を呼び出し、OID を取得する。 4. OID が有効な場合は結果リストに追加する。 5. OID が無効（拡張機能が存在しない）かつ warnOnMissing が true の場合、ERRCODE_UNDEFINED_OBJECT で警告を報告する。 6. 最後に一時的なリストを解放し、OID リストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/option.c:162
  • contrib/postgres_fdw/postgres_fdw.c:6266
  • **呼び出し先**:
  • SplitIdentifierString
  • get_extension_oid
  • lappend_oid
  • list_free
  • ereport

関連データ構造

  • List (PostgreSQL のリスト構造)
  • Oid (オブジェクト識別子)

失敗モードと境界条件

  • **構文エラー**: extensionsString がリストとして解析できない場合、エラー終了する。
  • **存在しない拡張機能**: warnOnMissing が false の場合、存在しない拡張機能は単に無視される。true の場合は警告が出力される。

保守担当者向けメモ

  • 本関数は postgres_fdw における拡張機能の依存関係管理に使用される。
  • SplitIdentifierString の仕様により、入力文字列のコピーが必須である点に注意すること。
