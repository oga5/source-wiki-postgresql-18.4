pltcl_init_tuple_store

定義

  • 種別: function
  • 定義箇所: src/pl/tcl/pltcl.c:3350

目的

Set-Returning Function (SRF) の実行結果を格納するための tuplestore および関連するメタデータ（attinmeta）を初期化します。

シグネチャまたは形状

static void pltcl_init_tuple_store(pltcl_call_state *call_state)

入力と出力

  • **入力**: call_state (pltcl_call_state *) - 現在の関数呼び出しの状態を保持する構造体。
  • **出力**: なし（call_state 内の tuple_store および attinmeta フィールドが更新されます）。

前提条件と不変条件

  • call_state->rsi が存在すること（SRF であること）。
  • call_state->tuple_store および call_state->attinmeta が未初期化であること。
  • rsi->expectedDesc が提供されていること。

アルゴリズムまたは振る舞い

1. rsi->expectedDesc を call_state->ret_tupdesc に設定します。 2. call_state->tuple_store_cxt および call_state->tuple_store_owner を使用して、メモリコンテキストとリソースオーナーを一時的に切り替えます。これにより、サブトランザクションの影響を受けずに永続的なストレージを確保します。 3. tuplestore_begin_heap を呼び出し、work_mem を用いて tuple_store を作成します。 4. TupleDescGetAttInMetadata を呼び出し、タプル変換用のメタデータを生成して call_state->attinmeta に格納します。 5. 元のメモリコンテキストとリソースオーナーに復帰します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/tcl/pltcl.c:2288 にて呼び出されています。
  • **呼び出し先**: tuplestore_begin_heap および TupleDescGetAttInMetadata を呼び出します。

関連データ構造

  • pltcl_call_state: 実行状態を管理する構造体。
  • ReturnSetInfo: SRF の実行情報を保持する PostgreSQL の構造体。
  • Tuplestorestate: tuplestore_begin_heap によって生成される結果格納用構造体。

失敗モードと境界条件

  • rsi が NULL の場合、または expectedDesc が欠如している場合はアサーションエラーが発生します。
  • tuple_store が既に初期化されている場合、アサーションエラーが発生します。

保守担当者向けメモ

  • 本関数は、SRF の結果を蓄積するコンテキストの生存期間を正しく管理するために、リソースオーナーの切り替えを行っています。例外ブロックなどでサブトランザクションが使用される場合でも、結果が適切に保持されるよう設計されています。
