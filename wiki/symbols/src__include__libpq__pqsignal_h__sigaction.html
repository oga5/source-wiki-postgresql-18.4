sigaction

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/pqsignal.h:25

目的

POSIX準拠のシグナルハンドラ設定を、プラットフォーム間で統一的に扱うための構造体です。特にWindows環境において、POSIXシグナルAPIをエミュレートする役割を担います。

シグネチャまたは形状

struct sigaction
{
    void        (*sa_handler) (int);
    sigset_t    sa_mask;
    int         sa_flags;
};

入力と出力

  • sa_handler: シグナル発生時に呼び出される関数ポインタ。
  • sa_mask: シグナルハンドラ実行中にブロックするシグナルのセット。
  • sa_flags: シグナルの挙動を制御するフラグ。

前提条件と不変条件

  • sa_sigaction フィールドは本構造体に含まれますが、現時点では実装されていません。
  • Windows環境では、sigaction マクロを通じて pqsigaction 関数へマッピングされます。

アルゴリズムまたは振る舞い

  • 本構造体はシグナルハンドラの登録および設定情報の保持に使用されます。
  • Windows環境では、pqsigaction を介して内部的なシグナル処理ロジックへ変換されます。
  • シグナル発生時には、pqsignal.c 内のラッパーハンドラが介在し、プロセスの正当性チェックを行った後に sa_handler が実行されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/win32/signal.c、src/interfaces/libpq/legacy-pqsignal.c、src/port/pqsignal.c などで、シグナルハンドラの設定や取得のために使用されています。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しを直接行うものではありません。関連する関数として pqsigaction が存在します。

関連データ構造

  • sigset_t: シグナルセットを表現する型。
  • pqsigaction: 本構造体を利用してシグナル設定を行う関数。

失敗モードと境界条件

  • Windows環境において sa_sigaction フィールドを使用しようとした場合、期待した動作は得られません。
  • pqsigaction を介した呼び出しにおいて、シグナル番号が不正な場合やシステムコールが失敗した場合は、戻り値によりエラーが通知されます。

保守担当者向けメモ

  • 本構造体はPOSIX環境とWindows環境の差異を吸収するための抽象化レイヤーの一部です。
  • Windows環境でのシグナル処理を拡張する際は、src/backend/port/win32/signal.c の実装と整合性を取る必要があります。
  • sa_sigaction の実装が必要な場合は、プラットフォーム固有のシグナル処理ロジックの大幅な見直しが必要となる可能性があります。
