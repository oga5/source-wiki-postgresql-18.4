xpath_string

定義

  • 種別: function
  • 定義箇所: contrib/xml2/xpath.c:264

目的

XMLドキュメントに対してXPath式を適用し、その評価結果を文字列（text型）として抽出するSQL呼び出し可能な関数です。

シグネチャまたは形状

Datum xpath_string(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): 対象となるXMLドキュメント（text型）。
  • PG_GETARG_TEXT_PP(1): 評価するXPath式（text型）。
  • **出力**:
  • 評価結果を格納したtext型。結果が空または無効な場合はNULLを返します。

前提条件と不変条件

  • 入力されたXPath式は、内部で string(...) でラップされ、libxml2 のXPathエンジンに渡されます。
  • xpath_workspace 構造体を使用して、評価に必要なリソース（ドキュメントやコンテキスト）を管理します。
  • libxml2 のメモリ領域は、関数終了前に cleanup_workspace を通じて明示的に解放される必要があります。

アルゴリズムまたは振る舞い

1. 入力されたXPath式を string() 関数で囲み、評価用の文字列を構築します。 2. pgxml_xpath を呼び出し、XMLドキュメントのパースおよびXPathのコンパイル・評価を実行します。 3. pgxml_result_to_text を使用して、xmlXPathObjectPtr 型の評価結果をPostgreSQLの text 型に変換します。 4. cleanup_workspace を呼び出し、評価に使用したワークスペースをクリーンアップします。 5. 変換された結果を返します。結果が NULL の場合はSQLの NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • pgxml_xpath
  • pgxml_result_to_text
  • cleanup_workspace
  • palloc / pfree

関連データ構造

  • xpath_workspace: 評価中のXMLドキュメント、XPathコンテキスト、結果オブジェクトを保持する管理構造体。

失敗モードと境界条件

  • XPath式の評価が失敗した場合や、結果が空の場合は NULL を返します。
  • メモリ不足時には palloc がエラーを発生させます。
  • XMLのパースエラーやXPathの構文エラーが発生した場合、pgxml_xpath 内部の PG_TRY/CATCH ブロックによりPostgreSQLの elog(ERROR) へ遷移します。

保守担当者向けメモ

  • 本関数は libxml2 を直接操作するため、メモリリークを防ぐために cleanup_workspace の呼び出しを必ず維持してください。
  • XPath式を string() でラップする処理は、XPathの戻り値を強制的に文字列型へキャストするための仕様です。
