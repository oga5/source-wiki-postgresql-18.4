px_set_debug_handler

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/px.c:143

目的

pgcrypto モジュール内でのデバッグログ出力に使用されるコールバック関数を登録または解除します。

シグネチャまたは形状

void px_set_debug_handler(void (*handler) (const char *))

入力と出力

  • **入力**: handler - デバッグメッセージ（const char *）を受け取る関数ポインタ。NULL を指定することでデバッグ出力を無効化できます。
  • **出力**: なし（void）。

前提条件と不変条件

  • 登録された handler は、px_debug 関数が呼び出された際に実行されます。
  • handler に NULL を渡すと、以降の px_debug による出力は抑制されます。

アルゴリズムまたは振る舞い

1. 引数として受け取った関数ポインタ handler を、モジュール内の静的変数 debug_handler に代入します。 2. これにより、後続の px_debug 呼び出しがこのハンドラを経由してログを出力するようになります。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pgsql.c 内の各関数（pgp_decrypt 等の処理前後でデバッグハンドラの有効化/無効化が行われています）。
  • **呼び出し先**: なし。

関連データ構造

  • debug_handler: 本関数によって設定される、モジュールスコープの静的関数ポインタ変数。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。NULL を渡すことは正常な操作として扱われ、デバッグ出力の無効化として機能します。

保守担当者向けメモ

  • 本関数はグローバルな状態（debug_handler）を変更するため、マルチスレッド環境や再入可能なコンテキストでの使用には注意が必要です。
  • px_debug とセットで利用されるため、デバッグ機能の拡張や変更を行う際は両方の実装を確認してください。
