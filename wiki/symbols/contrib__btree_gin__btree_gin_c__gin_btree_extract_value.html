gin_btree_extract_value

定義

  • 種別: function
  • 定義箇所: contrib/btree_gin/btree_gin.c:32

目的

GINインデックス構築時に、単一の値をインデックスキーとして抽出します。B-treeの比較ロジックをGINインターフェースに適合させるためのブリッジ関数の一つです。

シグネチャまたは形状

static Datum gin_btree_extract_value(FunctionCallInfo fcinfo, bool is_varlena)

入力と出力

  • **入力**:
  • fcinfo: PostgreSQLの関数呼び出し情報。第1引数に抽出対象のDatum、第2引数にnentries（出力用ポインタ）が含まれます。
  • is_varlena: 対象データ型が可変長（varlena）かどうかを示すフラグ。
  • **出力**:
  • 抽出されたキーを格納したDatum配列へのポインタを返します。
  • nentries引数を通じて、抽出されたキーの数（常に1）を返します。

前提条件と不変条件

  • nentriesは有効なint32ポインタである必要があります。
  • is_varlenaがtrueの場合、入力datumはデトースト（detoast）処理が行われます。
  • 常に単一のキーを抽出するため、nentriesは必ず1に設定されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_DATUM(0)により入力値を取得します。 2. is_varlenaが真であれば、PG_DETOAST_DATUMを用いてデータを展開します。 3. pallocによりDatum配列を確保し、抽出した値を格納します。 4. nentriesに1を書き込みます。 5. 確保した配列のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gin/btree_gin.c 内の各型固有の抽出関数（例: gin_extract_value_[type]）から呼び出されます。
  • **呼び出し先**: PostgreSQLのメモリ管理関数（palloc）およびデータ型変換マクロ（PG_DETOAST_DATUM等）を呼び出します。

関連データ構造

  • Datum: PostgreSQLの汎用データ表現型。

失敗モードと境界条件

  • メモリ不足時にはpallocがエラーを発生させます。
  • is_varlenaの指定が実際のデータ型と一致しない場合、不正なメモリ参照やデータ破損の可能性があります。

保守担当者向けメモ

  • 本関数は、GINインデックスがサポートする各データ型に対して共通の抽出ロジックを提供します。
  • 新しいデータ型を追加する際は、この関数を再利用し、型固有のextract_value関数から呼び出す構成を維持してください。
