auto_mem_key_init

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/memory.c:81

目的

スレッドローカルストレージ（TLS）でメモリ追跡リストを管理するための pthread キーを初期化します。

シグネチャまたは形状

static void auto_mem_key_init(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として auto_mem_key を初期化）

前提条件と不変条件

  • pthread_key_create が正常に実行可能であること。
  • auto_mem_destructor がスレッド終了時のクリーンアップ関数として有効であること。

アルゴリズムまたは振る舞い

1. pthread_key_create を呼び出し、グローバル変数 auto_mem_key にスレッド固有のデータキーを割り当てます。 2. その際、スレッド終了時にメモリ追跡リストを解放するためのデストラクタとして auto_mem_destructor を登録します。

呼び出し元と呼び出し先

  • 呼び出し元: pthread_once(&auto_mem_once, auto_mem_key_init) を介して呼び出されます（src/interfaces/ecpg/ecpglib/memory.c:90）。
  • 呼び出し先: pthread_key_create（システムライブラリ関数）、auto_mem_destructor（デストラクタとして登録）。

関連データ構造

  • auto_mem_key: スレッドごとのメモリ追跡リストを特定するためのキー。
  • auto_mem_destructor: スレッド終了時に呼び出される解放関数。

失敗モードと境界条件

  • pthread_key_create が失敗した場合の挙動は、基盤となるスレッドライブラリの仕様に依存します。

保守担当者向けメモ

  • 本関数は pthread_once を通じて一度だけ実行されることを前提としています。
  • スレッド環境下でのメモリリークを防ぐための重要な初期化処理です。変更時は auto_mem_destructor との整合性に注意してください。
