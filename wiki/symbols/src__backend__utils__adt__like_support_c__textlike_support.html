textlike_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like_support.c:114

目的

LIKE 演算子に対するプランナのサポート関数として機能します。プランナがインデックススキャンを利用可能か判断する際や、選択率を推定する際に呼び出されます。

シグネチャまたは形状

Datum textlike_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して Node *rawreq（プランナからの要求情報）を受け取ります。
  • **出力**: like_regex_support の実行結果である Datum を返します。

前提条件と不変条件

  • rawreq はプランナから渡される有効な Node である必要があります。
  • Pattern_Type_Like を指定して like_regex_support を呼び出すため、本関数は LIKE 演算子専用のサポートロジックとして動作します。

アルゴリズムまたは振る舞い

1. 引数から rawreq を取得します。 2. Pattern_Type_Like を指定して like_regex_support を呼び出し、処理を委譲します。 3. like_regex_support が生成したインデックス条件または選択率推定結果をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: プランナ（インデックス適合性評価時）。
  • **呼び出し先**: like_regex_support。

関連データ構造

  • Pattern_Type: パターンマッチングの種類を定義する列挙型（Pattern_Type_Like を使用）。
  • Node: プランナの要求を保持する構造体。

失敗モードと境界条件

  • like_regex_support が内部で処理できないパターンや無効な入力を受け取った場合、その関数に準じたエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は like_regex_support への単なるラッパーです。LIKE 演算子に関するプランナの挙動を変更する場合は、like_regex_support またはその下位関数（match_pattern_prefix 等）を確認してください。
