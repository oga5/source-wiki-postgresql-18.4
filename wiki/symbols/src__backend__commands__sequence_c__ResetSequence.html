ResetSequence

定義

  • 種別: function
  • 定義箇所: src/backend/commands/sequence.c:261

目的

指定されたシーケンスを初期状態（start_value）にリセットします。

シグネチャまたは形状

void ResetSequence(Oid seq_relid)

入力と出力

  • **入力**: seq_relid (リセット対象のシーケンスのOID)
  • **出力**: なし（副作用としてシーケンスの物理ストレージが再作成され、値が初期化される）

前提条件と不変条件

  • 対象の seq_relid が有効なシーケンスであること。
  • 呼び出し元で適切なロック（AccessExclusiveLock 等）が取得されていること。
  • シーケンスの relfrozenxid が InvalidTransactionId、relminmxid が InvalidMultiXactId であること。

アルゴリズムまたは振る舞い

1. init_sequence を呼び出し、対象シーケンスのキャッシュ状態とリレーション情報を取得する。 2. システムカタログ（SEQRELID）からシーケンスの定義情報を検索し、開始値（seqstart）を取得する。 3. 現在のシーケンスデータをコピーし、古いバッファを解放する。 4. コピーしたデータ構造の last_value を開始値に、is_called を false に、log_cnt を 0 に設定する。 5. RelationSetNewRelfilenumber を呼び出し、シーケンスの物理ファイルを再作成する。 6. fill_seq_with_data を使用して、初期化されたデータを新しいストレージに書き込む。 7. セッション内のローカルキャッシュ（elm->cached）を更新し、キャッシュされた値が古いことを反映させる。

関連データ構造

  • SeqTable: セッション内のシーケンスキャッシュ状態を保持。
  • Form_pg_sequence: シーケンスの定義情報（システムカタログ）。
  • Form_pg_sequence_data: シーケンスの現在の値状態。

失敗モードと境界条件

  • 指定されたOIDがシーケンスでない場合、またはキャッシュ検索に失敗した場合は elog(ERROR) を発生させる。
  • 物理ファイルの再作成やデータ書き込み時にストレージ層のエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は物理ファイルを再作成することでリセットを実現しており、トランザクションの原子性が保証されます。
  • currval() の状態には影響を与えない仕様です。
  • relfrozenxid および relminmxid の不変条件は Assert で検証されています。
