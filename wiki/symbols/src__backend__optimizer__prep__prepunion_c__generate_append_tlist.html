generate_append_tlist

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepunion.c:1484

目的

Append ノード（UNION ALL 等）の実行プランにおいて、入力サブクエリ間でデータ型や照合順序（Collation）を整合させたターゲットリスト（tlist）を生成する。

シグネチャまたは形状

List *generate_append_tlist(List *colTypes, List *colCollations, List *input_tlists, List *refnames_tlist)

入力と出力

  • **入力**:
  • colTypes: 各列の期待される OID 型リスト。
  • colCollations: 各列の照合順序 OID リスト。
  • input_tlists: 各入力サブクエリのターゲットリストのリスト。
  • refnames_tlist: 参照名（列名）を保持するターゲットリスト。
  • **出力**:
  • 整合化された TargetEntry のリスト（tlist）。

前提条件と不変条件

  • input_tlists 内の各 TargetEntry は resjunk であってはならない。
  • 生成される各 TargetEntry の ressortgroupref は、自身の resno と等しく設定される。
  • 出力列の Var ノードは varno 0 を使用する。

アルゴリズムまたは振る舞い

1. **Typmod の決定**: 全入力サブクエリの各列の型と typmod を走査する。すべての入力で型が一致し、かつ typmod も一致する場合のみその typmod を採用し、それ以外は -1 とする。 2. **ターゲットリストの構築**: colTypes、colCollations、refnames_tlist を並行して走査する。 3. **Var ノードの生成**: 各列に対して makeVar(0, resno, ...) を使用して新しい Var ノードを作成する。 4. **TargetEntry の作成**: makeTargetEntry を呼び出し、列名と resno を割り当ててリストに追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepunion.c 内の plan_set_operations 等の集合演算プランニング処理から呼び出される。
  • **呼び出し先**: palloc, list_length, list_head, lfirst, exprType, exprTypmod, lnext, makeVar, makeTargetEntry, pstrdup, lappend, pfree 等の PostgreSQL 内部ユーティリティ関数。

関連データ構造

  • TargetEntry: ターゲットリストの各要素を構成する構造体。
  • List: PostgreSQL のリスト構造。

失敗モードと境界条件

  • 入力リストの長さが不一致の場合、foreach や forthree マクロの挙動により予期せぬ動作やアサーション違反が発生する可能性がある。
  • メモリ不足時には palloc がエラーを投げる。

保守担当者向けメモ

  • 本関数は UNION ALL のような単純な結合における列の整合性を担保する。
  • ressortgroupref の設定は、後続のソートやグループ化処理でこの tlist が利用されることを前提とした慣習である。
  • varno 0 を持つ Var ノードは、集合演算の出力列を指すための特殊な表現であることに注意すること。
