ExecParallelInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execParallel.c:1333

目的

並列クエリ実行において、各ワーカープロセスが実行を開始する前に、プランツリー内の各ノードに対して必要な初期化処理（共有メモリ状態のセットアップ等）を再帰的に実行する。

シグネチャまたは形状

static bool ExecParallelInitializeWorker(PlanState *planstate, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • planstate: 初期化対象のプランツリーのルートノード。
  • pwcxt: ワーカープロセスの実行コンテキストを保持する構造体。
  • **出力**:
  • bool: 常に true を返す（planstate_tree_walker の戻り値に依存）。

前提条件と不変条件

  • planstate が NULL の場合は即座に false を返す。
  • 並列実行に対応したノード（SeqScan, HashJoin 等）に対しては、planstate->plan->parallel_aware が真である場合にのみ初期化関数が呼び出される。
  • EXPLAIN ANALYZE 実行時には、並列非対応のノード（Sort, Agg 等）であっても統計情報収集のために初期化関数が呼び出される。

アルゴリズムまたは振る舞い

1. planstate が NULL であれば終了する。 2. nodeTag(planstate) を使用してノードの種類を判定し、対応する Exec*InitializeWorker 関数を呼び出す。 3. planstate_tree_walker を使用して、プランツリーの全ノードに対して再帰的に本関数を適用する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c 内の並列実行初期化プロセスから呼び出される。
  • **呼び出し先**:
  • 各種ノード固有の初期化関数（例: ExecSeqScanInitializeWorker, ExecHashJoinInitializeWorker, ExecAggInitializeWorker など）。
  • planstate_tree_walker（ツリー走査用）。

関連データ構造

  • PlanState: 実行プランのノード状態。
  • ParallelWorkerContext: ワーカーの並列実行コンテキスト。

失敗モードと境界条件

  • planstate が NULL の場合、安全に終了する。
  • 未対応のノードタグが渡された場合、default ケースにより何も行わず、ツリーの走査のみを継続する。

保守担当者向けメモ

  • 新しい並列対応ノードを追加した際は、本関数の switch 文に新しい case を追加し、適切な Exec*InitializeWorker を呼び出す必要がある。
  • EXPLAIN ANALYZE 用の初期化が必要なノードについては、parallel_aware フラグの有無に関わらず呼び出される点に注意すること。
