BTPageStat

定義

  • 種別: struct
  • 定義箇所: contrib/pageinspect/btreefuncs.c:58

目的

PostgreSQLのB-treeインデックスページに関する統計情報およびページヘッダーの不透明領域（opaque）データを保持するために使用されます。

シグネチャまたは形状

typedef struct BTPageStat
{
    uint32      blkno;
    uint32      live_items;
    uint32      dead_items;
    uint32      page_size;
    uint32      max_avail;
    uint32      free_size;
    uint32      avg_item_size;
    char        type;

    /* opaque data */
    BlockNumber btpo_prev;
    BlockNumber btpo_next;
    uint32      btpo_level;
    uint16      btpo_flags;
    BTCycleId   btpo_cycleid;
} BTPageStat;

入力と出力

本構造体は関数への出力（ポインタ渡し）として利用されます。

  • **入力**: GetBTPageStatistics 等の関数において、統計情報を格納するためのメモリ領域として提供されます。
  • **出力**: ページ内のタプル数、空き容量、およびB-tree特有のメタデータ（レベルや前後ページ番号など）がフィールドに書き込まれます。

前提条件と不変条件

  • contrib/pageinspect モジュールの関数によって、対象のブロックが有効なB-treeページであることが検証されている必要があります。
  • 構造体の各フィールドは、対象ページを読み取った時点の物理的な状態を反映します。

アルゴリズムまたは振る舞い

  • ページ内の各タプルを走査し、ライブ/デッドタプルのカウント、平均サイズ、空き容量を計算して各フィールドを埋めます。
  • ページヘッダーの不透明領域から、B-treeの階層構造（btpo_level）や隣接ページ情報（btpo_prev, btpo_next）を抽出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • GetBTPageStatistics (内部ヘルパー関数)
  • bt_page_stats (SQL関数)
  • bt_multi_page_stats (SQL関数)
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • access/nbtree.h: B-treeの内部構造（BTPageOpaqueData 等）の定義元。
  • Buffer: ページ読み取りに使用されるバッファ管理構造体。

失敗モードと境界条件

  • 対象ブロックがB-treeインデックスページでない場合、不透明領域の解釈が不正になる可能性があります。
  • 破損したページに対しては、計算された統計値が物理的な整合性を欠く場合があります。

保守担当者向けメモ

  • 本構造体は contrib/pageinspect モジュール専用のデバッグ用データ構造です。
  • B-treeの内部構造に変更があった場合、opaque data セクションのフィールド定義を access/nbtree.h の BTPageOpaqueData と同期させる必要があります。
