test_slru_page_delete

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_slru/test_slru.c:160

目的

指定されたページ番号に対応するSLRUセグメントを削除するためのSQL呼び出し可能関数です。

シグネチャまたは形状

Datum test_slru_page_delete(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT64(0) を介してページ番号（int64）を受け取ります。
  • **出力**: PG_RETURN_VOID() により値を返しません。

前提条件と不変条件

  • TestSlruCtl が初期化されている必要があります。
  • ページ番号からセグメント番号への変換は pageno / SLRU_PAGES_PER_SEGMENT の計算式に基づきます。

アルゴリズムまたは振る舞い

1. 引数からページ番号を取得します。 2. ページ番号を SLRU_PAGES_PER_SEGMENT で除算し、対応するセグメント番号を算出します。 3. SlruDeleteSegment を呼び出し、TestSlruCtl が管理する対象セグメントを削除します。 4. 削除したセグメント番号を NOTICE レベルのログとして出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: SlruDeleteSegment（PostgreSQLコアのSLRU API）。

関連データ構造

  • TestSlruCtl: テスト用SLRUを制御する SlruCtlData 構造体。
  • FileTag: セグメント番号を保持するために使用される構造体。

失敗モードと境界条件

  • ページ番号が負の値の場合や、無効なセグメントを指定した場合の挙動は SlruDeleteSegment の実装に依存します。

保守担当者向けメモ

  • 本関数はテスト専用です。運用環境での直接的なセグメント操作には使用しないでください。
  • SlruDeleteSegment の引数として TestSlruCtl を渡しているため、モジュール内の共有メモリ状態と整合していることを確認してください。
