TransactionIdPrecedes

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/transam.c:279

目的

2つのトランザクションID（XID）の論理的な前後関係を判定します。XIDのラップアラウンド（循環）を考慮した比較を行うための関数です。

シグネチャまたは形状

bool TransactionIdPrecedes(TransactionId id1, TransactionId id2)

入力と出力

  • **入力**:
  • id1: 比較対象のトランザクションID
  • id2: 比較対象のトランザクションID
  • **出力**:
  • bool: id1 が id2 よりも論理的に先行している場合に true を返します。

前提条件と不変条件

  • XIDは32ビットの循環値です。直接的な比較演算子（< など）を使用すると、ラップアラウンド発生時に誤動作するため、必ず本関数を使用する必要があります。
  • どちらか一方のIDが「正規（Normal）」ではない（予約済みIDなど）場合、単純な符号なし整数比較が行われます。

アルゴリズムまたは振る舞い

1. id1 または id2 のいずれかが正規のXIDでない場合、id1 < id2 の結果を返します。 2. 両者が正規のXIDである場合、id1 - id2 を int32 にキャストして差分を計算します。 3. 計算された差分が負（diff < 0）であれば、id1 が id2 より先行していると判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • contrib/amcheck/verify_nbtree.c
  • contrib/pg_visibility/pg_visibility.c
  • src/backend/access/heap/heapam.c
  • **呼び出し先**:
  • 本関数は TransactionIdIsNormal マクロ（src/include/access/transam.h で定義）を利用してIDの性質を判定します。

関連データ構造

  • TransactionId: トランザクションを識別する32ビットの型。

失敗モードと境界条件

  • 循環するXID空間において、距離が $2^{31}$ を超える比較は正しく判定できません。

保守担当者向けメモ

  • 本関数は、TransactionIdPrecedesOrEquals、TransactionIdFollows、TransactionIdFollowsOrEquals と同様のロジック群の一部です。変更時はこれら関連関数との整合性に注意してください。
