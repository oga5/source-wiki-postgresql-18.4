r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_irish.c:223

目的

アイルランド語のステミング処理において、単語内の特定の領域（R1, R2, RV）を特定し、SN_env 構造体の整数配列 I にその開始位置を記録する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z（ステミング実行環境）
  • **出力**: 常に 1 を返す。
  • **副作用**: z->I[0] (R2), z->I[1] (R1), z->I[2] (RV) の各値が更新される。

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。
  • 処理対象の文字列は UTF-8 エンコーディングであること。

アルゴリズムまたは振る舞い

1. 初期状態として、I[0], I[1], I[2] を文字列の終端位置 z->l に設定する。 2. 母音と子音の出現パターンに基づき、以下の順序で領域をマークする：

  • 母音の直後の子音位置を探索し、I[2] (RV) を更新。
  • さらに母音の直後の子音位置を探索し、I[1] (R1) を更新。
  • 同様の探索を繰り返し、I[0] (R2) を更新。

    • 3. 探索が失敗した場合は、初期値（文字列終端）を保持する。

呼び出し元と呼び出し先

  • **呼び出し元**: irish_UTF_8_stem（ソースコード内の制御フローより）。
  • **呼び出し先**: out_grouping_U, in_grouping_U（Snowball ランタイムのユーティリティ関数）。

関連データ構造

  • struct SN_env: ステミング処理の状態を保持する構造体。I 配列（整数領域）および c（現在のカーソル位置）、l（文字列長）を使用する。

失敗モードと境界条件

  • 文字列が短い場合や、母音・子音のパターンが適合しない場合、領域の開始位置は文字列の終端（z->l）に留まる。
  • out_grouping_U または in_grouping_U が負の値を返した場合、探索は中断され、その時点での z->c が復元される。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接の編集は避けること。ロジックの変更が必要な場合は、対応する .sbl ファイルを修正し、再生成を行うこと。
  • I[0]～I[2] は後続の接尾辞削除処理（r_noun_sfx 等）における境界条件として必須である。
