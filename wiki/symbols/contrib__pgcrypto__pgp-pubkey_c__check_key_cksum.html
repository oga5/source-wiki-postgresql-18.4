check_key_cksum

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubkey.c:298

目的

PGP 秘密鍵パケットの末尾に付与された 2 バイトのチェックサムを検証し、復号された鍵データの整合性を確認します。

シグネチャまたは形状

int check_key_cksum(PullFilter *src, PGP_PubKey *pk)

入力と出力

  • **入力**:
  • src: 鍵データを含む PullFilter。
  • pk: 復号済みの鍵情報が格納された PGP_PubKey 構造体。
  • **出力**:
  • 成功時: 0
  • 失敗時: PXE_PGP_KEYPKT_CORRUPT（チェックサム不一致）、または pullf_read_fixed が返すエラーコード。

前提条件と不変条件

  • pk->algo に基づき、対応する MPI（多倍長整数）データが pk->sec 内に正しく格納されている必要があります。
  • src から 2 バイトのデータが読み取り可能である必要があります。

アルゴリズムまたは振る舞い

1. pullf_read_fixed を使用して、src から 2 バイトのチェックサム値を読み取ります。 2. pk->algo（アルゴリズム種別）に応じて、以下の MPI 値を対象に pgp_mpi_cksum を用いてチェックサムを再計算します。

  • PGP_PUB_ELG_ENCRYPT: x
  • PGP_PUB_RSA_*: d, p, q, u の順に累積計算。
  • PGP_PUB_DSA_SIGN: x

    • 3. 再計算した値と読み取った値を比較し、一致しない場合はエラーを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pubkey.c 内の process_secret_key から呼び出されます（行 439）。
  • **呼び出し先**:
  • pullf_read_fixed
  • pgp_mpi_cksum
  • px_debug

関連データ構造

  • PGP_PubKey: 鍵のアルゴリズム情報および秘密鍵の MPI を保持します。
  • PullFilter: 鍵データのストリーム読み込みに使用されます。

失敗モードと境界条件

  • pullf_read_fixed が 2 バイト未満しか読み込めない場合、そのエラーコードを伝播します。
  • 鍵のアルゴリズムがサポート対象外（switch 文のケース外）の場合、my_cksum は初期値 0 のままとなり、チェックサム検証が失敗する可能性があります。
  • チェックサムが一致しない場合、px_debug にログを出力し PXE_PGP_KEYPKT_CORRUPT を返します。

保守担当者向けメモ

  • 本関数は PGP バージョン 4 の秘密鍵パケット形式に依存しています。
  • 新しいアルゴリズムを追加する場合は、switch 文に該当する MPI の計算ロジックを追加する必要があります。
