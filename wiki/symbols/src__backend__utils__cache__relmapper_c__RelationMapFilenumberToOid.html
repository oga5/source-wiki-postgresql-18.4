RelationMapFilenumberToOid

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/relmapper.c:217

目的

物理ファイル番号（RelFileNumber）から対応するリレーションの OID を逆引きする。

シグネチャまたは形状

Oid RelationMapFilenumberToOid(RelFileNumber filenumber, bool shared)

入力と出力

  • **入力**:
  • filenumber: 検索対象の物理ファイル番号。
  • shared: 共有カタログを検索する場合は true、ローカルカタログの場合は false。
  • **出力**:
  • 対応する Oid。見つからない場合は InvalidOid を返す。

前提条件と不変条件

  • 検索は、まずトランザクション内で保留中の更新（active_shared_updates または active_local_updates）を優先し、次にメインのマップ（shared_map または local_map）を検索する。
  • マッピングの変更はサブトランザクション内では許可されない。

アルゴリズムまたは振る舞い

1. shared 引数に基づき、対象となる「保留中更新マップ」と「メインマップ」を選択する。 2. まず保留中更新マップを走査し、filenumber が一致するエントリがあればその mapoid を即座に返す。 3. 保留中更新マップに存在しない場合、メインマップを走査し、一致するエントリがあればその mapoid を返す。 4. どちらにも存在しない場合、InvalidOid を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/relfilenumbermap.c:184
  • src/backend/utils/cache/relfilenumbermap.c:235
  • **呼び出し先**:
  • 直接的な関数呼び出しは行わない。メモリ上のグローバル変数（active_shared_updates, shared_map, active_local_updates, local_map）を参照する。

関連データ構造

  • RelMapFile: マッピング情報を保持する構造体。
  • RelFileNumber: 物理ファイル識別子。

失敗モードと境界条件

  • 指定された filenumber がどのマップにも存在しない場合、InvalidOid が返される。これはエラーではなく、該当するマッピングが存在しないことを示す正常な戻り値として扱われる。

保守担当者向けメモ

  • 本関数は relmapper.c 内の他の関数（RelationMapUpdateMap 等）と連携して動作する。
  • 検索順序（保留中更新が優先）は、トランザクションの分離レベルと整合性を保つために重要である。
