SlotSyncShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/slotsync.c:1800

目的

論理レプリケーションスロット同期機能で使用する共有メモリ領域（SlotSyncCtx）の割り当ておよび初期化を行う。

シグネチャまたは形状

void SlotSyncShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上の SlotSyncCtx が初期化される）

前提条件と不変条件

  • SlotSyncShmemSize() が返すサイズ分の共有メモリが確保可能であること。
  • 本関数は共有メモリの初期化プロセス中に一度だけ呼び出される必要がある。

アルゴリズムまたは振る舞い

1. SlotSyncShmemSize() を呼び出し、必要な共有メモリサイズを取得する。 2. ShmemInitStruct を使用して "Slot Sync Data" という名前の共有メモリ領域を確保または取得する。 3. 共有メモリが新規に作成された場合（found が false の場合）のみ、以下の初期化を行う：

  • 領域全体をゼロクリアする。
  • SlotSyncCtx->pid を InvalidPid に設定する。
  • SlotSyncCtx->mutex に対して SpinLockInit を実行し、スピンロックを初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:336（共有メモリ初期化シーケンスの一部）
  • 呼び出し先: SlotSyncShmemSize（サイズ計算）、ShmemInitStruct（メモリ確保）、memset、SpinLockInit

関連データ構造

  • SlotSyncCtxStruct: 同期プロセスの状態を保持する共有メモリ構造体。本関数によりこの構造体のインスタンスが初期化される。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct の内部実装に従い、PostgreSQL のエラーハンドリングによりプロセスが終了する可能性がある。

保守担当者向けメモ

  • 本関数は PostgreSQL の起動時の共有メモリ初期化フェーズで実行される。
  • SlotSyncCtx 構造体のフィールドを追加・変更した場合は、必ず SlotSyncShmemSize の計算ロジックも併せて更新すること。
