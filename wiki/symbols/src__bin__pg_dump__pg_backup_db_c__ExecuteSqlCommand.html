ExecuteSqlCommand

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_db.c:266

目的

データベースに対して SQL コマンドを実行し、結果のステータスを検証する。エラー発生時にはプロセスを終了させるラッパー関数。

シグネチャまたは形状

void ExecuteSqlCommand(ArchiveHandle *AH, const char *qry, const char *desc)

入力と出力

  • **入力**:
  • AH: データベース接続情報を含む ArchiveHandle 構造体。
  • qry: 実行する SQL クエリ文字列。
  • desc: エラー発生時にログ出力される操作の説明文。
  • **出力**: なし（成功時は正常終了、失敗時は warn_or_exit_horribly によりプロセス終了）。

前提条件と不変条件

  • AH->connection が有効な PGconn オブジェクトを保持していること。
  • COPY モードへの遷移が予期される場合、PQRES_COPY_IN が返されることを許容する。

アルゴリズムまたは振る舞い

1. PQexec を使用してクエリを実行する。 2. PQresultStatus で結果を確認する。

  • PGRES_COMMAND_OK, PGRES_TUPLES_OK, PGRES_EMPTY_QUERY の場合は正常終了。
  • PGRES_COPY_IN の場合は、AH->pgCopyIn フラグを true に設定する。
  • 上記以外の場合は、desc と PQerrorMessage を含むエラーメッセージを出力し、プロセスを終了する。

    • 3. PQclear を呼び出し、結果オブジェクトを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_db.c 内の各所（SQL実行、トランザクション制御など）。
  • **呼び出し先**:
  • libpq の PQexec, PQresultStatus, PQerrorMessage, PQclear。
  • pg_backup_db.c 内の warn_or_exit_horribly。

関連データ構造

  • ArchiveHandle: データベース接続およびアーカイブ状態を管理する構造体。
  • PGconn, PGresult: libpq が提供する接続および結果保持用構造体。

失敗モードと境界条件

  • クエリ実行が PGRES_COMMAND_OK 等の期待されるステータス以外を返した場合、warn_or_exit_horribly によりプログラムが強制終了される。
  • COPY モード開始時、AH->pgCopyIn が更新されるが、その後の終了処理については本関数単体では完結しない。

保守担当者向けメモ

  • 本関数は libpq のエラーハンドリングを抽象化している。複雑な SQL 解析は行わず、単純なコマンド実行に適している。
  • NOT_USED マクロで囲まれたデバッグ用出力が存在する。
