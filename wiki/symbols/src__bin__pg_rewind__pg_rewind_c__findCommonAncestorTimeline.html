findCommonAncestorTimeline

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/pg_rewind.c:920

目的

2つのクラスタ（ソースとターゲット）のタイムライン履歴を比較し、両者が分岐した共通の祖先ポイントを特定する。

シグネチャまたは形状

static void findCommonAncestorTimeline(
    TimeLineHistoryEntry *a_history, int a_nentries,
    TimeLineHistoryEntry *b_history, int b_nentries,
    XLogRecPtr *recptr, int *tliIndex
)

入力と出力

  • **入力**:
  • a_history, b_history: 比較対象となるタイムライン履歴エントリの配列。
  • a_nentries, b_nentries: 各履歴配列の要素数。
  • **出力**:
  • recptr: 分岐点となる WAL 位置（XLogRecPtr）。
  • tliIndex: 共通祖先が見つかった履歴配列上のインデックス。

前提条件と不変条件

  • 履歴配列は TimeLineHistoryEntry 構造体で構成されている必要がある。
  • 共通の祖先が見つからない場合、関数は pg_fatal を呼び出しプログラムを終了させる。

アルゴリズムまたは振る舞い

1. 両方の履歴配列を先頭から順に走査し、tli（タイムラインID）と begin（開始位置）が一致する限り進む。 2. 一致しなくなった直前のエントリを共通祖先とみなす。 3. 共通祖先が見つかった場合、そのエントリの end 位置のうち小さい方を recptr に設定し、該当するインデックスを tliIndex に格納して終了する。 4. 履歴が全く一致しない場合はエラーとして終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c:392（main 関数内から呼び出される）。
  • **呼び出し先**: Min マクロおよび MinXLogRecPtr 関数。

関連データ構造

  • TimeLineHistoryEntry: タイムラインの履歴情報を保持する構造体。
  • XLogRecPtr: WAL 内のログ位置を示すポインタ型。

失敗モードと境界条件

  • **失敗モード**: 両クラスタの履歴に共通点がない場合、pg_fatal によりプロセスが異常終了する。
  • **境界条件**: 履歴配列の要素数が異なる場合、Min(a_nentries, b_nentries) を用いて最小の範囲で比較を行う。

保守担当者向けメモ

  • タイムライン履歴の比較において、単なるIDの一致だけでなく begin 位置の検証も行っているため、履歴ファイルが異なるプロセスで取得された場合でも正確な分岐点を特定できるよう設計されている。
