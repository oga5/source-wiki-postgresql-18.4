StartupDecodingContext

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logical.c:151

目的

論理デコーディング処理に必要なメモリコンテキスト、WALリーダー、再構築バッファ、および出力プラグインのコールバックラッパーを初期化し、LogicalDecodingContext を構築する。

シグネチャまたは形状

LogicalDecodingContext *StartupDecodingContext(
    List *output_plugin_options,
    XLogRecPtr start_lsn,
    TransactionId xmin_horizon,
    bool need_full_snapshot,
    bool fast_forward,
    bool in_create,
    XLogReaderRoutine *xl_routine,
    LogicalOutputPluginWriterPrepareWrite prepare_write,
    LogicalOutputPluginWriterWrite do_write,
    LogicalOutputPluginWriterUpdateProgress update_progress
)

入力と出力

  • **入力**:
  • output_plugin_options: プラグインに渡すオプションリスト。
  • start_lsn, xmin_horizon: デコーディング開始位置とトランザクションの整合性境界。
  • fast_forward: 変更を生成せずWALをスキップするモードのフラグ。
  • xl_routine: WAL読み取り用のルーチン定義。
  • prepare_write, do_write, update_progress: 出力処理用のコールバック関数。
  • **出力**: 初期化済みの LogicalDecodingContext 構造体へのポインタ。

前提条件と不変条件

  • MyReplicationSlot が有効であること。
  • fast_forward が false の場合、出力プラグインがロード可能であること。
  • トランザクションブロック外で呼び出された場合、現在のプロセスが PROC_IN_LOGICAL_DECODING としてマークされる。

アルゴリズムまたは振る舞い

1. 専用のメモリコンテキストを作成し、LogicalDecodingContext を割り当てる。 2. fast_forward モードでなければ LoadOutputPlugin を実行し、プラグインの存在を確認する。 3. トランザクション外であれば、ProcArrayLock を取得して現在のプロセスを論理デコーディング中として登録する。 4. XLogReaderAllocate でWALリーダーを初期化する。 5. ReorderBufferAllocate で変更蓄積用バッファを確保し、SnapshotBuilder を初期化する。 6. begin, change, commit 等の各コールバックを、エラーコンテキストを付与するラッパー関数（*_cb_wrapper）でラップして設定する。 7. ストリーミングおよび2フェーズコミットのサポート状況を判定し、対応するラッパーを設定する。 8. 出力用コールバックとオプションをコンテキストに格納し、作成したコンテキストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • CreateInitDecodingContext
  • CreateDecodingContext
  • **呼び出し先**:
  • AllocSetContextCreate, palloc0, MemoryContextSwitchTo
  • LoadOutputPlugin
  • XLogReaderAllocate
  • ReorderBufferAllocate
  • AllocateSnapshotBuilder
  • makeStringInfo
  • 各種ラッパー関数（begin_cb_wrapper 等）

関連データ構造

  • LogicalDecodingContext: デコーディングの状態を保持するメイン構造体。
  • ReplicationSlot: 現在のレプリケーションスロット情報。
  • ReorderBuffer: トランザクション変更の蓄積用バッファ。
  • XLogReaderRoutine: WAL読み取りインターフェース。

失敗モードと境界条件

  • メモリ不足時: XLogReaderAllocate が失敗した場合、ereport(ERROR, ...) により中断される。
  • プラグイン不在時: LoadOutputPlugin が失敗した場合、エラーが発生する。

保守担当者向けメモ

  • 本関数は ReorderBuffer のコールバックをラッパー関数で上書きすることで、エラー発生時にどのプラグインのどの処理で失敗したかを特定可能にしている。
  • streaming および twophase のフラグは、コールバックの有無に基づいて動的に決定される。
  • fast_forward モード時はプラグインのロードをスキップするため、プラグインの検証は行われない点に注意すること。
