CreateReplicationSlotCmd

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/replnodes.h:52

目的

レプリケーションスロット作成コマンド（CREATE_REPLICATION_SLOT）のパラメータを保持し、構文解析器から実行エンジンへ情報を伝達するためのパースノードです。

シグネチャまたは形状

typedef struct CreateReplicationSlotCmd
{
	NodeTag		type;
	char	   *slotname;
	ReplicationKind kind;
	char	   *plugin;
	bool		temporary;
	List	   *options;
} CreateReplicationSlotCmd;

入力と出力

  • **入力**: フロントエンドから送信されるレプリケーションコマンドの各要素（スロット名、種類、プラグイン名、一時フラグ、オプションリスト）。
  • **出力**: 実行エンジン（walsender.c 等）が参照する構造体インスタンス。

前提条件と不変条件

  • NodeTag 型の type フィールドを先頭に持ち、PostgreSQLのノードシステムに準拠している必要があります。
  • options フィールドは List 構造体を使用し、コマンド固有の柔軟なパラメータを保持します。

アルゴリズムまたは振る舞い

1. **パース時**: レプリケーションプロトコルのコマンド受信に伴い、構文解析器が本構造体をインスタンス化し、各フィールドに値を格納します。 2. **実行時**: 実行エンジンがこの構造体を参照し、CreateReplicationSlot 関数等のバックエンド処理へパラメータを渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c 内の各処理（CreateReplicationSlot 関数への引数として利用）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ReplicationKind: レプリケーションの種類（物理/論理）を識別する列挙型。
  • List: コマンドオプションを保持するためのリスト構造。

失敗モードと境界条件

  • 構造体内の各ポインタ（slotname, plugin）が NULL の場合の挙動は、呼び出し側の実行エンジン側の実装に依存します。

保守担当者向けメモ

  • 新しいレプリケーションオプションを追加する場合は、本構造体の options リストを活用するか、必要に応じてフィールドを追加してください。
  • 構造体の変更時は、src/backend/nodes/copyfuncs.c および outfuncs.c での対応するコピー・出力処理の更新を検討してください。
