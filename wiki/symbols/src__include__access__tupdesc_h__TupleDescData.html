TupleDescData

定義

  • 種別: struct
  • 定義箇所: src/include/access/tupdesc.h:135

目的

リレーションやクエリ結果の行構造（列数、各列の型、制約など）を保持するメタデータ構造体。実行エンジンやアクセスメソッドがメモリ上のタプルを解釈するための基盤となる。

シグネチャまたは形状

typedef struct TupleDescData
{
    int         natts;
    Oid         tdtypeid;
    int32       tdtypmod;
    int         tdrefcount;
    TupleConstr *constr;
    CompactAttribute compact_attrs[FLEXIBLE_ARRAY_MEMBER];
} TupleDescData;

入力と出力

  • 入力: 属性数（natts）に基づきメモリ確保される。
  • 出力: compact_attrs を通じて、タプル変形（Deformation）に必要な属性の型やアライメント情報を提供する。

前提条件と不変条件

  • tdrefcount が -1 の場合、キャッシュ管理対象外（スタック上や一時的メモリ）であることを示す。
  • FormData_pg_attribute（カタログ定義）が変更された場合、populate_compact_attribute() を呼び出して compact_attrs を同期させる必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: CreateTupleDesc 等により生成され、natts に応じたメモリ領域が確保される。 2. **アクセス**: 実行エンジンは TupleDescCompactAttr() を介して属性情報を取得し、タプルのバイト列を解析する。 3. **解放**: 参照カウントが 0 になると FreeTupleDesc() によりメモリが解放される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/common/tupdesc.c（メモリ確保）、src/backend/jit/llvm/llvmjit_types.c、src/include/access/relscan.h（スキャン状態での参照）。
  • 呼び出し先: TupleConstr 構造体、CompactAttribute 構造体。

関連データ構造

  • TupleDesc: TupleDescData へのポインタ型。
  • CompactAttribute: タプル変形を高速化するための軽量メタデータ。
  • TupleConstr: 制約情報（NOT NULL、デフォルト値など）。

失敗モードと境界条件

  • compact_attrs と FormData_pg_attribute の同期を怠ると、タプル解析時に不整合が発生する。
  • tdrefcount が -1 の場合、Pin/Release マクロによる参照カウント操作は無効となる。

保守担当者向けメモ

  • CompactAttribute は 16 バイトに制限されている。拡張時はタプル変形処理の性能に直結するため注意が必要。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズは動的に決定される。
