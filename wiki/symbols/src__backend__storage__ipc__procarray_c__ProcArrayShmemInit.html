ProcArrayShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/procarray.c:417

目的

PostgreSQL の共有メモリ上に、アクティブなバックエンドプロセスを管理するための ProcArrayStruct およびホットスタンバイ用の KnownAssignedXids 配列を確保またはアタッチする。

シグネチャまたは形状

void ProcArrayShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上のグローバル変数 procArray、allProcs、およびホットスタンバイ有効時は KnownAssignedXids と KnownAssignedXidsValid が初期化またはアタッチされる）

前提条件と不変条件

  • 共有メモリの初期化フェーズで呼び出される必要がある。
  • EnableHotStandby の値に基づき、必要な共有メモリ領域が動的に確保される。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を使用して "Proc Array" という名前の共有メモリ領域を確保またはアタッチする。 2. 領域が新規作成された場合（found == false）、ProcArrayStruct の各メンバ（numProcs、maxProcs、numKnownAssignedXids 等）をゼロまたはデフォルト値で初期化する。 3. allProcs ポインタを ProcGlobal->allProcs に設定する。 4. EnableHotStandby が真の場合、"KnownAssignedXids" および "KnownAssignedXidsValid" という名前の共有メモリ領域を確保またはアタッチする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:312 (共有メモリ初期化シーケンスの一部)
  • 呼び出し先: ShmemInitStruct (共有メモリ管理関数)

関連データ構造

  • ProcArrayStruct: プロセス配列のメイン管理構造体。
  • PGPROC: 各バックエンドプロセスの状態を保持する構造体。
  • KnownAssignedXids: ホットスタンバイ時に使用されるトランザクションID配列。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct 内部でエラーが発生し、プロセスが終了する可能性がある。
  • PROCARRAY_MAXPROCS を超えるプロセス数はサポートされない。

保守担当者向けメモ

  • 本関数はシステム起動時の共有メモリ初期化プロセスにおいて重要な役割を果たす。
  • ホットスタンバイに関連するメモリ確保は EnableHotStandby フラグに依存するため、スタンバイ構成の変更時にはこの初期化ロジックが正しく動作することを確認すること。
