update_ps_display_precheck

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/ps_status.c:344

目的

プロセス名の更新処理を実行可能かどうかを判定するガード関数です。

シグネチャまたは形状

static bool update_ps_display_precheck(void)

入力と出力

  • 入力: なし（グローバル変数およびプロセス状態を参照）
  • 出力: bool 型。更新処理を続行すべき場合は true、中断すべき場合は false を返す。

前提条件と不変条件

  • update_process_title GUC変数が有効であること。
  • 現在のプロセスが postmaster の配下で動作していること。
  • PS_USE_CLOBBER_ARGV が定義されている環境では、ps_buffer が初期化済み（非NULL）であること。

アルゴリズムまたは振る舞い

1. update_process_title が false の場合、false を返す。 2. IsUnderPostmaster が false（スタンドアロンバックエンド）の場合、false を返す。 3. PS_USE_CLOBBER_ARGV が定義されている場合、ps_buffer が NULL であれば false を返す。 4. 上記の条件をすべて通過した場合、true を返す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/misc/ps_status.c 内の set_ps_display_with_len、set_ps_display_suffix、set_ps_display_remove_suffix から呼び出される。
  • 呼び出し先:
  • 直接的な関数呼び出しはない。グローバル変数 update_process_title、IsUnderPostmaster、および ps_buffer を参照する。

関連データ構造

  • update_process_title (GUC変数)
  • IsUnderPostmaster (プロセス状態フラグ)
  • ps_buffer (OSのargv領域を指すポインタ)

失敗モードと境界条件

  • update_process_title が無効な場合、更新はスキップされる。
  • スタンドアロンモードで起動されたプロセスでは、プロセス名の更新は行われない。
  • PS_USE_CLOBBER_ARGV 環境下で ps_buffer が確保される前に呼び出された場合、更新は行われない。

保守担当者向けメモ

  • 本関数は、プロセス名更新の各APIの冒頭でガードとして使用されます。
  • PS_USE_CLOBBER_ARGV 以外のプラットフォーム固有の制約が将来追加された場合、この関数に条件を追加してください。
