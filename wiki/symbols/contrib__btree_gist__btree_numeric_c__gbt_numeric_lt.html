gbt_numeric_lt

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:60

目的

PostgreSQL の numeric 型に対する「より小さい（less than）」比較演算を、GiST インデックスのサポート関数として提供します。

シグネチャまたは形状

static bool gbt_numeric_lt(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の numeric 型データへのポインタ（void *）。
  • collation: 照合順序（本関数では使用されません）。
  • flinfo: 関数呼び出し情報（本関数では使用されません）。
  • **出力**:
  • a < b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な numeric 型のデータ構造を指している必要があります。
  • PostgreSQL コアの numeric_lt 関数が正常に動作する環境である必要があります。

アルゴリズムまたは振る舞い

1. a と b を PointerGetDatum を用いて Datum 型に変換します。 2. PostgreSQL コアの numeric_lt 関数を DirectFunctionCall2 を介して呼び出します。 3. 戻り値を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_numeric.c 内の tinfo 構造体（gbtree_vinfo）の初期化時に参照されています。
  • **呼び出し先**: PostgreSQL コアの numeric_lt 関数。

関連データ構造

  • gbtree_vinfo: numeric 型の比較関数群を保持し、GiST インデックス操作に提供するための構造体。

失敗モードと境界条件

  • numeric 型の比較ルールに従い、NaN（Not a Number）が含まれる場合の挙動はコアの numeric_lt 実装に依存します。
  • 入力ポインタが NULL の場合の挙動については、コアの numeric_lt の仕様に準拠します。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける numeric 型用のアダプター関数です。ロジックの変更が必要な場合は、コアの numeric_lt の仕様を確認してください。
