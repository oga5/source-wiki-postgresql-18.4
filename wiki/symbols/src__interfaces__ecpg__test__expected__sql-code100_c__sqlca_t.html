sqlca_t

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-code100.c:30

目的

SQL通信領域（SQL Communication Area）として、ECPG（Embedded SQL for C）で実行されたSQL文のステータスコード、エラーメッセージ、および実行結果の詳細を保持する。

シグネチャまたは形状

struct sqlca_t {
    char        sqlcaid[8];
    long        sqlabc;
    long        sqlcode;
    struct {
        int     sqlerrml;
        char    sqlerrmc[SQLERRMC_LEN];
    }           sqlerrm;
    char        sqlerrp[8];
    long        sqlerrd[6];
    char        sqlwarn[8];
    char        sqlstate[5];
};

入力と出力

  • **入力**: ECPGランタイムライブラリ（ECPGdo 等）がSQL実行後に本構造体の各フィールドを更新する。
  • **出力**: アプリケーションは sqlcode や sqlstate を参照し、SQL文の成功・失敗・警告（例: 100 = データなし）を判定する。

前提条件と不変条件

  • sqlcode が 100 の場合、SQL標準における「データが見つからない（No data found）」状態を示す。
  • sqlerrd[2] は INSERT、UPDATE、DELETE 文の実行後に処理された行数を保持する。
  • sqlwarn[0] は他の警告フラグがセットされた場合に 'W' となる。

アルゴリズムまたは振る舞い

ECPGプリプロセッサが生成するコードにおいて、各SQL操作の直後に本構造体の内容が更新される。アプリケーションはこれを確認することで、後続の処理（エラーハンドリングや条件分岐）を決定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/compatlib/informix.c および src/interfaces/ecpg/ecpglib/ 配下の各ファイル（connect.c, data.c, descriptor.c 等）が ECPGget_sqlca() を通じて本構造体を参照・更新する。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはない。

関連データ構造

  • sqlda_t: 動的SQL記述領域。
  • sqlvar_t: sqlda_t 内で使用される列定義構造体。

失敗モードと境界条件

  • sqlerrmc のサイズは SQLERRMC_LEN に制限されており、長いエラーメッセージは切り詰められる可能性がある。
  • sqlcode が 0 以外の場合、エラーまたは警告が発生していることを示す。

保守担当者向けメモ

  • 本構造体はECPGランタイムの根幹であり、定義の変更は全ECPGアプリケーションに影響する。
  • sql-code100.c は、この構造体が正しくエラー状態（特にコード100）を反映するかを検証する回帰テスト用である。
