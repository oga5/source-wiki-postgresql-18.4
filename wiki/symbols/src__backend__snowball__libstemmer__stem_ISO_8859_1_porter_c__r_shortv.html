r_shortv

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_porter.c:225

目的

単語の末尾が「子音-母音-子音（CVC）」のパターンに該当するかを判定する。ポーター・ステミングアルゴリズムにおいて、特定の接尾辞削除条件を検証するために使用される。

シグネチャまたは形状

static int r_shortv(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング処理の状態を保持する構造体）
  • **出力**: int 型。条件を満たす場合は 1 を返し、満たさない場合は 0 を返す。

前提条件と不変条件

  • SN_env 構造体が初期化済みであること。
  • 内部で g_v_WXY（母音およびW, X, Y）および g_v（母音）のグループ定義を参照する。

アルゴリズムまたは振る舞い

以下の順序で後方から文字を検査し、すべて一致した場合に成功（1）を返す： 1. out_grouping_b(z, g_v_WXY, 89, 121, 0): 末尾が母音・W・X・Yのいずれでもないこと（子音C）。 2. in_grouping_b(z, g_v, 97, 121, 0): 次の文字が母音であること（母音V）。 3. out_grouping_b(z, g_v, 97, 121, 0): さらにその前の文字が母音ではないこと（子音C）。

呼び出し元と呼び出し先

  • **呼び出し元**:

  • **呼び出し先**:

関連データ構造

  • SN_env: ステミング処理のカーソル位置やバッファを管理する構造体。

失敗モードと境界条件

  • 文字列の長さが3文字未満の場合、条件を満たせず 0 を返す。
  • out_grouping_b や in_grouping_b が境界外や不一致を検出した場合、即座に 0 を返して終了する。

保守担当者向けメモ
