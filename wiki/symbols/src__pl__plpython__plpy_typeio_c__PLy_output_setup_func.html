PLy_output_setup_func

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_typeio.c:295

目的

SQL型からPythonオブジェクトへの変換を行うための変換構造体（PLyObToDatum）を初期化し、型に応じた適切な変換関数ポインタとメタデータを設定する。

シグネチャまたは形状

void PLy_output_setup_func(PLyObToDatum *arg, MemoryContext arg_mcxt, Oid typeOid, int32 typmod, PLyProcedure *proc)

入力と出力

  • **入力**:
  • arg: 初期化対象の PLyObToDatum 構造体へのポインタ。
  • arg_mcxt: 変換情報を割り当てるメモリコンテキスト。
  • typeOid: 変換対象のSQL型のOID。
  • typmod: 型修飾子。
  • proc: 現在のPL/Pythonプロシージャ情報。
  • **出力**: なし（arg が指す構造体を直接更新する）。

前提条件と不変条件

  • 再帰的な型定義（ドメインや配列など）を扱うため、スタック深さのチェック（check_stack_depth()）が実行される。
  • RECORDOID は特殊な複合型として扱われ、キャッシュを介さずハードコードされた属性値が設定される。

アルゴリズムまたは振る舞い

1. **メタデータ設定**: arg に型OID、修飾子、メモリコンテキストを格納する。 2. **型情報の取得**: lookup_type_cache を使用して型情報を取得し、typbyval、typlen、typalign を設定する。 3. **変換メソッドの決定**:

  • **ドメイン**: PLyObject_ToDomain を設定し、基底型に対して再帰的に PLy_output_setup_func を呼び出す。
  • **配列**: PLySequence_ToArray を設定し、要素型に対して再帰的に PLy_output_setup_func を呼び出す。
  • **変換関数**: get_transform_tosql が定義されていれば PLyObject_ToTransform を使用する。
  • **複合型**: PLyObject_ToComposite を設定する。
  • **スカラー**: BOOLOID や BYTEAOID の特殊ケースを除き、PLyObject_ToScalar とその入力関数を設定する。

関連データ構造

  • PLyObToDatum: 変換ロジックと型情報を保持する構造体。
  • TypeCacheEntry: PostgreSQLの型キャッシュ情報。
  • PLyProcedure: PL/Pythonプロシージャの実行コンテキスト。

失敗モードと境界条件

  • **スタックオーバーフロー**: 再帰的な型定義が深すぎる場合、check_stack_depth() によりエラーが発生する。
  • **メモリ不足**: MemoryContextAllocZero 呼び出し時にメモリ不足が発生する可能性がある。

保守担当者向けメモ

  • 本関数は再帰的であるため、メモリコンテキストの管理には注意が必要。
  • ドメインや配列のネスト構造を走査する際、中間レベルの型情報をスキップする仕様があるため、変換関数の追加時には注意すること。
