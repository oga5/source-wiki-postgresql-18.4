PgArchCanRestart

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pgarch.c:197

目的

アーカイバプロセスが再起動可能かどうかを判定します。短期間に繰り返される再起動を防ぐためのスロットリング機能を提供します。

シグネチャまたは形状

bool PgArchCanRestart(void)

入力と出力

  • **入力**: なし（内部で time(NULL) を使用して現在時刻を取得）
  • **出力**: bool 型。再起動が許可される場合は true、前回の起動から PGARCH_RESTART_INTERVAL 秒が経過していない場合は false を返します。

前提条件と不変条件

  • 静的変数 last_pgarch_start_time により、前回の起動時刻が保持されます。
  • PGARCH_RESTART_INTERVAL 定数に基づき、再起動の間隔が制限されます。

アルゴリズムまたは振る舞い

1. 現在時刻 curtime を取得します。 2. curtime と last_pgarch_start_time の差分を計算します。 3. 差分が PGARCH_RESTART_INTERVAL 未満であれば false を返します。 4. 差分が PGARCH_RESTART_INTERVAL 以上であれば、last_pgarch_start_time を curtime で更新し、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/postmaster.c:3329 からの呼び出しが確認されています。
  • **呼び出し先**: 標準ライブラリ関数 time() を呼び出します。

関連データ構造

  • last_pgarch_start_time (関数内静的変数)

失敗モードと境界条件

  • time() が失敗した場合の挙動は定義されていません。
  • 起動直後の初回呼び出し時は last_pgarch_start_time が 0 で初期化されているため、経過時間の計算はエポック時刻からの差分となります。

保守担当者向けメモ

  • 本関数はアーカイバプロセスの過度な再起動ループを防止するために重要です。設定値 PGARCH_RESTART_INTERVAL を変更する際は、システム全体のリカバリ性能への影響を考慮してください。
