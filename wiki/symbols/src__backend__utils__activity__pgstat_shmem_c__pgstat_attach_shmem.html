pgstat_attach_shmem

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_shmem.c:243

目的

現在のバックエンドプロセスを、PostgreSQLの統計情報管理用共有メモリ領域に接続します。

シグネチャまたは形状

void pgstat_attach_shmem(void)

入力と出力

  • 入力: なし（pgStatLocal.shmem を介して共有メモリの制御構造を参照）
  • 出力: なし（副作用として pgStatLocal の状態を更新）

前提条件と不変条件

  • pgStatLocal.dsa が NULL であること（二重接続の防止）。
  • pgStatLocal.shmem が有効な共有メモリ制御構造を指していること。

アルゴリズムまたは振る舞い

1. TopMemoryContext にメモリコンテキストを切り替え、接続情報をバックエンドの生存期間中保持するように設定します。 2. dsa_attach_in_place を呼び出し、共有メモリ上の動的共有メモリ（DSA）領域を現在のプロセスにアタッチします。 3. dsa_pin_mapping を呼び出し、DSAのマッピングを固定します。 4. dshash_attach を呼び出し、共有メモリ上の統計情報ハッシュテーブルをアタッチします。 5. 元のメモリコンテキストへ復帰します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/activity/pgstat.c:644
  • 呼び出し先: dsa_attach_in_place, dsa_pin_mapping, dshash_attach, MemoryContextSwitchTo

関連データ構造

  • pgStatLocal: バックエンドローカルな統計情報管理構造体。
  • dsa_area: 動的共有メモリ領域。
  • dshash_table: 共有メモリ上のハッシュテーブル。

失敗モードと境界条件

  • 共有メモリが初期化されていない状態で呼び出された場合、pgStatLocal.shmem へのアクセスによりクラッシュする可能性があります。
  • dsa_attach_in_place や dshash_attach が失敗した場合、PostgreSQLの標準的なエラーハンドリングによりプロセスが終了します。

保守担当者向けメモ

  • 本関数はバックエンド起動時に一度だけ実行されることを想定しています。
  • TopMemoryContext を使用しているため、接続情報はバックエンドの終了まで解放されません。
