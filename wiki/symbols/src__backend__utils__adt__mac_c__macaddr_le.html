macaddr_le

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac.c:218

目的

2つの macaddr 型の値を受け取り、第1引数が第2引数以下であるか（<=）を判定する。

シグネチャまたは形状

Datum macaddr_le(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの macaddr * 型の引数を受け取る。
  • **出力**: Datum 型として、比較結果が真であれば true、偽であれば false を返す。

前提条件と不変条件

  • 入力される macaddr ポインタは有効なメモリ領域を指している必要がある。
  • 内部比較ロジックとして macaddr_cmp_internal を使用する。

アルゴリズムまたは振る舞い

1. PG_GETARG_MACADDR_P マクロを使用して、2つの入力引数 a1 および a2 を取得する。 2. macaddr_cmp_internal(a1, a2) を呼び出し、6バイトのMACアドレス値を比較する。 3. 比較結果が 0 以下（<= 0）である場合、true を返し、それ以外の場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_macaddr.c:50 （直接的な呼び出しの証拠あり）
  • **呼び出し先**:
  • macaddr_cmp_internal （定義箇所 src/backend/utils/adt/mac.c 内の関数）

関連データ構造

  • macaddr: 6バイトのEUI-48 MACアドレスを保持する構造体。

失敗モードと境界条件

  • 引数が NULL の場合、PostgreSQLの関数呼び出し規約に従い、呼び出し元で適切に処理される必要がある（本関数内では明示的なNULLチェックは行われていない）。

保守担当者向けメモ

  • 本関数は macaddr 型の標準的な比較演算子の一部である。比較ロジックの変更が必要な場合は、macaddr_cmp_internal を修正することで、本関数を含むすべての比較関数に一貫した変更が適用される。
