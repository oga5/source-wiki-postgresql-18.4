JoinTreeItem

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/plan/initsplan.c:60

目的

クエリのジョインツリーを解析する際、各ノードの構造や依存関係、制約条件の分配状況を一時的に保持するためのデータ構造です。

シグネチャまたは形状

typedef struct JoinTreeItem
{
    /* 解析フェーズ（deconstruct_recurse）で設定 */
    Node       *jtnode;
    JoinDomain *jdomain;
    struct JoinTreeItem *jti_parent;
    Relids      qualscope;
    Relids      inner_join_rels;
    Relids      left_rels;
    Relids      right_rels;
    Relids      nonnullable_rels;
    /* 分配フェーズ（deconstruct_distribute）で設定 */
    SpecialJoinInfo *sjinfo;
    List       *oj_joinclauses;
    List       *lateral_clauses;
} JoinTreeItem;

入力と出力

  • **入力**: deconstruct_recurse によるジョインツリーの走査結果。
  • **出力**: deconstruct_distribute による制約条件の分配先情報（sjinfo, oj_joinclauses, lateral_clauses）。

前提条件と不変条件

  • deconstruct_jointree の実行中にのみ生存する一時的な構造体です。
  • jti_parent はツリーのルートノードでは NULL となります。
  • deconstruct_jointree の終了時に解放されます。

アルゴリズムまたは振る舞い

1. **ツリー走査**: deconstruct_recurse によりジョインツリーを深さ優先探索し、各ノードのスコープ（qualscope）やリレーションの包含関係を記録します。 2. **制約分配**: deconstruct_distribute がこの構造体を反復処理し、WHERE句やJOIN/ON句の条件を適切なリレーションへ振り分けます。 3. **外部結合管理**: 外部結合ノードの場合、SpecialJoinInfo を生成して制約や順序付けルールを管理します。

呼び出し元と呼び出し先

  • **呼び出し元**: deconstruct_jointree およびその再帰的な呼び出し（deconstruct_recurse）。
  • **呼び出し先**: 構造体内部で直接関数を呼び出すことはありませんが、deconstruct_distribute 等の関数がこの構造体を引数として受け取り、内部フィールドを操作します。

関連データ構造

  • SpecialJoinInfo: 外部結合の制約情報を保持。
  • Relids: リレーションの集合を表現するビットマップ。
  • JoinDomain: ジョインのON/WHERE句のドメイン情報。

失敗モードと境界条件

  • 構造体自体はメモリ確保に失敗しない限り安定していますが、LATERAL 参照や PlaceHolderVar が含まれる場合、依存関係の解決において複雑な制約条件の分配が発生します。

保守担当者向けメモ

  • 本構造体は initsplan.c 内でのみ使用される一時的な解析用オブジェクトです。
  • LATERAL 参照や PlaceHolderVar の評価レベルを追跡するロジックが本構造体の利用において最も複雑な部分です。
  • 修正を行う際は、deconstruct_recurse（構造構築）と deconstruct_distribute（条件分配）の二段階のライフサイクルを考慮してください。
