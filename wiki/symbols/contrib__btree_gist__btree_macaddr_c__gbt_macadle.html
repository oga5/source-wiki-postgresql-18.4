gbt_macadle

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr.c:47

目的

macaddr 型の2つの値が「以下（less than or equal to）」の関係にあるかを判定し、GiSTインデックスの比較処理をサポートします。

シグネチャまたは形状

bool gbt_macadle(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**: 比較対象となる2つの macaddr 値へのポインタ（a, b）および関数管理情報（flinfo）。
  • **出力**: a <= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な macaddr 型のデータとして扱える必要があります。
  • PostgreSQLの内部関数 macaddr_le が呼び出し可能である必要があります。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力ポインタをPostgreSQLの Datum 型に変換します。 2. DirectFunctionCall2 を介して、PostgreSQL標準の macaddr_le 関数を呼び出します。 3. 戻り値を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: macaddr_le（PostgreSQL標準の比較関数）。
  • **呼び出し元**: contrib/btree_gist/btree_macaddr.c 内の静的構造体 tinfo（gbtree_ninfo 型）から参照されています。

関連データ構造

  • macaddr: 比較対象となるPostgreSQLのネットワークアドレス型。
  • tinfo: macaddr 型の比較関数を保持する設定テーブル。

失敗モードと境界条件

  • 入力が NULL の場合、DirectFunctionCall2 の挙動に依存します。
  • macaddr_le が内部的にエラーを発生させた場合、呼び出し元に例外が伝播します。

保守担当者向けメモ

  • 本関数は btree_utils_num.h のパターンに従ったラッパー関数です。
  • macaddr 型の比較ロジックを変更する場合は、src/backend/utils/adt/mac.c の macaddr_le の定義を確認してください。
