ReplaceWrapOption

定義

  • 種別: enum
  • 定義箇所: src/backend/optimizer/prep/prepjointree.c:61

目的

サブクエリ引き上げ処理において、式を PlaceHolderVar (PHV) でラップする必要があるかどうかを制御するためのオプションを定義します。

シグネチャまたは形状

typedef enum ReplaceWrapOption
{
    REPLACE_WRAP_NONE,      /* ラップ不要 */
    REPLACE_WRAP_ALL,       /* 全ての式をラップする */
    REPLACE_WRAP_VARFREE,   /* 変数を含まない式のみをラップする */
} ReplaceWrapOption;

入力と出力

本シンボルは列挙型であり、関数ではありません。主に pullup_replace_vars_context 構造体の一部として保持され、引き上げ処理の挙動を決定するフラグとして使用されます。

前提条件と不変条件

  • 外結合の null 伝播を正しく扱うために、特定の条件下で式を PHV に変換する必要があります。
  • REPLACE_WRAP_VARFREE は、変数を含まない定数や関数呼び出しなどを PHV で保護する際に使用されます。

アルゴリズムまたは振る舞い

  • REPLACE_WRAP_NONE: 式のラップを行いません。
  • REPLACE_WRAP_ALL: 対象となる全ての式を PHV でラップします。
  • REPLACE_WRAP_VARFREE: 式の中に変数（Var ノード）が含まれていない場合のみ、その式を PHV でラップします。

呼び出し元と呼び出し先

  • **呼び出し元**: pullup_replace_vars_context 構造体内で定義され、pull_up_subqueries 関連の処理において、再帰的な引き上げ処理中に現在のラップ状態を保存・復元するために参照されます（src/backend/optimizer/prep/prepjointree.c:2444, 2611）。
  • **呼び出し先**: 本シンボル自体はデータ型であり、関数を呼び出しません。

関連データ構造

  • pullup_replace_vars_context: この列挙型をメンバ wrap_option として保持します。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • 本オプションは、外結合の null 伝播を考慮した式置換において極めて重要です。pullup_replace_vars のロジックを変更する際は、このオプションがどのように PHV の挿入判断に影響するかを確認してください。
