hashchar

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:47

目的

SQLの char 型（1バイト文字）をハッシュインデックスやハッシュ結合で使用可能な32ビットのハッシュ値に変換します。

シグネチャまたは形状

Datum hashchar(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される char 型のデータ。
  • **出力**: Datum 型として返される32ビットのハッシュ値。

前提条件と不変条件

  • 入力値は PG_GETARG_CHAR(0) を介して取得可能であること。
  • ハッシュ関数としての性質上、同一の入力値に対しては常に同一のハッシュ値を返すこと。

アルゴリズムまたは振る舞い

1. PG_GETARG_CHAR(0) を呼び出し、入力された char 型の値を抽出します。 2. 抽出した値を int32 型にキャストします。 3. hash_uint32 関数を呼び出し、キャストした値からハッシュ値を計算して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのハッシュアクセスメソッド（pg_amproc に登録された演算子クラス経由で呼び出される）。
  • **呼び出し先**: hash_uint32（src/include/common/hashfn.h で定義された整数用ハッシュ計算関数）。

関連データ構造

  • Datum: PostgreSQLの汎用データ型表現。
  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に従う引数コンテナ。

失敗モードと境界条件

  • 入力値が char 型の範囲外である場合や、関数呼び出しの引数が不正な場合は、PostgreSQLの標準的なエラーハンドリング機構（elog 等）により処理されます。

保守担当者向けメモ

  • 本関数は単純な型変換と hash_uint32 への委譲のみを行っています。
  • ハッシュ結合のパフォーマンス最適化が必要な場合は、シード値を受け取る hashcharextended の実装と整合性を保つ必要があります。
