map_locale

定義

  • 種別: function
  • 定義箇所: src/port/win32setlocale.c:110

目的

Windows 環境において、setlocale() が正しく認識できないロケール名や、非 ASCII 文字を含むロケール名を、システムが許容する形式へ置換・正規化する。

シグネチャまたは形状

static const char *map_locale(const struct locale_map *map, const char *locale);

入力と出力

  • **入力**:
  • map: 置換ルールを定義した locale_map 構造体の配列。
  • locale: 変換対象のロケール名文字列。
  • **出力**:
  • 置換後のロケール名（静的バッファ aliasbuf へのポインタ）、または置換が不要な場合は元の locale 文字列を返す。
  • バッファサイズを超える場合は NULL を返す。

前提条件と不変条件

  • map 配列は locale_name_start が NULL の要素で終端されている必要がある。
  • aliasbuf は静的領域であるため、呼び出しごとに内容が上書きされる。

アルゴリズムまたは振る舞い

1. map 配列を順に走査し、locale 文字列内に locale_name_start に一致する部分があるか検索する。 2. 一致した場合、locale_name_end が定義されていれば、その範囲を含めて置換対象とする。 3. 置換後の文字列が MAX_LOCALE_NAME_LEN を超えないか検証する。 4. memcpy を用いて、置換前後の文字列を aliasbuf に組み立てて返す。 5. 一致するルールがない場合は、元の locale 文字列をそのまま返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/port/win32setlocale.c 内の pgwin32_setlocale 関数から、引数の正規化および戻り値の正規化のために呼び出される。
  • **呼び出し先**:
  • 標準ライブラリ関数: strstr, strlen, memcpy。

関連データ構造

  • struct locale_map: 置換ルールを保持する構造体。
  • aliasbuf: 変換結果を格納する静的バッファ。

失敗モードと境界条件

  • 置換後の文字列長が MAX_LOCALE_NAME_LEN を超える場合、NULL を返す。
  • map 配列の終端条件が満たされていない場合、未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • aliasbuf は静的領域を使用しているため、マルチスレッド環境での安全性には注意が必要である。
  • ロケール名の一部のみを動的に書き換える範囲置換をサポートしているため、ルール追加時には locale_name_end の指定が適切か確認すること。
