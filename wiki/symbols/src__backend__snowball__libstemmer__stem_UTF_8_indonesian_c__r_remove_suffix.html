r_remove_suffix

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_indonesian.c:170

目的

インドネシア語のステミング処理において、単語末尾から特定の接尾辞を除去し、語幹を正規化する。

シグネチャまたは形状

static int r_remove_suffix(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング対象の単語バッファおよび状態管理構造体）
  • 出力: int（成功時に 1、条件不一致やエラー時に 0 または負の値を返す）

前提条件と不変条件

  • z->c は現在の処理位置を指す。
  • z->lb はバッファの境界を示す。
  • z->I[1] はステミングの進行状態を保持する整数値であり、除去成功時にデクリメントされる。

アルゴリズムまたは振る舞い

1. 現在位置 z->c を z->ket に設定する。 2. バッファの境界チェックを行い、末尾の文字が 'i' (105) または 'n' (110) でない場合は終了する。 3. a_2 配列で定義された接尾辞パターンを find_among_b で後方検索する。 4. マッチした場合、その位置を z->bra に設定し、slice_del で該当部分を削除する。 5. 状態管理変数 z->I[1] を 1 減算し、成功を示す 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: stem_UTF_8_indonesian.c および stem_ISO_8859_1_indonesian.c 内のステミングロジックから呼び出される。
  • 呼び出し先: find_among_b（パターンマッチング）、slice_del（バッファ操作）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_2: 接尾辞のパターンを格納する among 構造体の配列。

失敗モードと境界条件

  • バッファの先頭に達している場合や、末尾文字が 'i'/'n' 以外の場合は 0 を返す。
  • find_among_b がマッチしない場合は 0 を返す。
  • slice_del が内部エラー（負の値）を返した場合は、その値をそのまま伝播させる。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接編集せず、元の Snowball スクリプト（.sbl）を修正すること。
  • z->I[1] の更新は、インドネシア語のステミングルールにおける接尾辞除去の回数制限や状態遷移と密接に関連している。
