plpython_inline_error_callback

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_main.c:361

目的

PostgreSQL のエラー発生時に、PL/Python の匿名コードブロック（DO 文）実行中であることを示すコンテキスト情報をエラースタックに追加する。

シグネチャまたは形状

static void plpython_inline_error_callback(void *arg)

入力と出力

  • **入力**: void *arg（現在は使用されていない）
  • **出力**: なし（errcontext を通じて PostgreSQL のエラー報告メカニズムに情報を渡す）

前提条件と不変条件

  • PostgreSQL のエラーハンドリング機構（ErrorContextCallback）から呼び出されることを前提とする。
  • 匿名コードブロックの実行コンテキスト内でエラーが発生した際にのみ有効に機能する。

アルゴリズムまたは振る舞い

1. errcontext マクロを呼び出す。 2. 固定文字列 "PL/Python anonymous code block" をエラーコンテキストとして出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_main.c 内の plpython3_inline_handler 関数において、ErrorContextCallback 構造体の callback フィールドに本関数が設定されている。
  • **呼び出し先**: PostgreSQL の errcontext 関数（マクロ経由）。

関連データ構造

  • ErrorContextCallback: PostgreSQL のエラー報告時に実行されるコールバックを保持する構造体。

失敗モードと境界条件

  • 本関数自体は単純な文字列出力を行うのみであり、特有の失敗モードはない。
  • arg 引数は現在使用されていないため、NULL を渡しても動作に影響しない。

保守担当者向けメモ

  • 本関数は plpython3_inline_handler でのみ使用される。
  • 将来的に実行コンテキストの詳細（行番号やソースコードの断片など）を報告する必要が生じた場合、arg 引数を使用して PLyExecutionContext を渡すように拡張可能である。
