PLyExceptionEntry

定義

  • 種別: struct
  • 定義箇所: src/pl/plpython/plpy_spi.h:18

目的

PostgreSQL のエラーコード（SQLSTATE）と、それに対応する Python の例外オブジェクトをマッピングするためのデータ構造です。

シグネチャまたは形状

typedef struct PLyExceptionEntry
{
	int			sqlstate;		/* hash key, must be first */
	PyObject   *exc;			/* corresponding exception */
} PLyExceptionEntry;

入力と出力

  • **入力**: sqlstate（ハッシュキーとして使用される整数値）および exc（対応する Python 例外オブジェクトへのポインタ）。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • sqlstate フィールドは構造体の先頭に配置される必要があります（ハッシュテーブルのキーとして機能するため）。

アルゴリズムまたは振る舞い

本構造体は、PostgreSQL 側で発生したエラーを Python 側で適切な例外として再送出するためのルックアップテーブルの一部として利用されます。ハッシュテーブルの各エントリとして格納され、エラー発生時に sqlstate をキーとして対応する Python 例外オブジェクトを特定します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/pl/plpython/plpy_plpymodule.c: ハッシュテーブルのサイズ定義およびエントリの参照。
  • src/pl/plpython/plpy_spi.c: エラー処理時の例外マッピング参照。
  • **呼び出し先**: 情報なし。

関連データ構造

  • PyObject (Python C API): Python の例外オブジェクトを保持するために使用。

失敗モードと境界条件

  • ハッシュテーブルのキーとして sqlstate を使用するため、一意性が保証されている必要があります。
  • exc ポインタが NULL の場合、例外の特定に失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は plpy_plpymodule.c 内のハッシュテーブル管理と密接に関連しています。構造体のメンバ順序を変更する場合は、ハッシュテーブルのオフセット計算に影響がないか確認してください。
