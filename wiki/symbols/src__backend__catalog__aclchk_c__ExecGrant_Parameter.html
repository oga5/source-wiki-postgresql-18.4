ExecGrant_Parameter

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/aclchk.c:2423

目的

システムパラメータ（GUC）に対する GRANT / REVOKE 文の実行結果を処理し、pg_parameter_acl カタログを更新する。

シグネチャまたは形状

static void ExecGrant_Parameter(InternalGrant *istmt)

入力と出力

  • **入力**: InternalGrant *istmt（実行対象の権限操作情報を含む構造体）
  • **出力**: なし（副作用としてシステムカタログ pg_parameter_acl を更新）

前提条件と不変条件

  • すべてのパラメータは BOOTSTRAP_SUPERUSERID が所有するものとして扱われる。
  • istmt->all_privs が真かつ権限が未指定の場合、ACL_ALL_RIGHTS_PARAMETER_ACL が適用される。

アルゴリズムまたは振る舞い

1. pg_parameter_acl テーブルを RowExclusiveLock で開く。 2. istmt->objects に含まれる各パラメータに対し、以下の処理を繰り返す：

  • PARAMETERACLOID システムキャッシュから対象のタプルを取得。
  • 既存の ACL を取得（存在しない場合は acldefault を使用）。
  • select_best_grantor で権限付与者と付与可能な権限を特定。
  • restrict_and_check_grant で権限の妥当性を検証。
  • merge_acl_with_grant で新しい ACL を生成。
  • 新しい ACL がデフォルト値と等しい場合は CatalogTupleDelete でエントリを削除し、それ以外の場合は CatalogTupleUpdate で更新。
  • recordExtensionInitPriv で拡張機能の初期権限を記録。
  • updateAclDependencies で共有依存関係を更新。

    • 3. 最後にテーブルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c 内の ExecuteGrantStmt から呼び出されることが示唆されている。
  • **呼び出し先**: table_open, SearchSysCache1, SysCacheGetAttr, acldefault, select_best_grantor, restrict_and_check_grant, merge_acl_with_grant, CatalogTupleDelete, CatalogTupleUpdate, recordExtensionInitPriv, updateAclDependencies, table_close 等。

関連データ構造

  • InternalGrant: 権限操作の定義。
  • pg_parameter_acl: ターゲットとなるシステムカタログ。

失敗モードと境界条件

  • PARAMETERACLOID キャッシュルックアップに失敗した場合、elog(ERROR) を発生させる。
  • 重複オブジェクト処理時にエラーを回避するため、ループ内で CommandCounterIncrement() を実行する。

保守担当者向けメモ

  • 本関数はパラメータ固有の ACL ロジックを実装している。列レベルやテーブルレベルの権限操作とは異なり、パラメータはすべてスーパーユーザー所有として固定されている点に注意すること。
