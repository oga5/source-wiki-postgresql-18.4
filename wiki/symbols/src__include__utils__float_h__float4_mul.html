float4_mul

定義

  • 種別: static inline function
  • 定義箇所: src/include/utils/float.h:193

目的

2つの float4 型の値を乗算し、結果に対してPostgreSQLの要件に基づいたオーバーフローおよびアンダーフローのチェックを行う。

シグネチャまたは形状

static inline float4 float4_mul(const float4 val1, const float4 val2)

入力と出力

  • **入力**: val1 (float4), val2 (float4)
  • **出力**: 乗算結果 (float4)

前提条件と不変条件

  • 演算結果が無限大（isinf）になった場合、入力値のいずれもが無限大でない限りエラーとみなす。
  • 演算結果が 0.0f になった場合、入力値のいずれもが 0.0f でない限りエラーとみなす。

アルゴリズムまたは振る舞い

1. val1 と val2 を乗算し、結果を result に格納する。 2. result が無限大であり、かつ val1 と val2 が共に無限大ではない場合、float_overflow_error() を呼び出す。 3. result が 0.0f であり、かつ val1 と val2 が共に 0.0f ではない場合、float_underflow_error() を呼び出す。 4. 最終的な result を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/float.c:751 (PG_RETURN_FLOAT4 マクロ内)
  • **呼び出し先**: float_overflow_error(), float_underflow_error()（いずれも src/backend/utils/adt/float.c で実装）

関連データ構造

  • float4: PostgreSQLの単精度浮動小数点型。

失敗モードと境界条件

  • **オーバーフロー**: 演算結果が浮動小数点の表現可能な最大値を超えた場合に発生。
  • **アンダーフロー**: 演算結果が非ゼロの入力からゼロにアンダーフローした場合に発生。

保守担当者向けメモ

  • 本関数はインライン化されており、パフォーマンスを重視している。
  • エラー報告関数（float_overflow_error 等）は pg_noinline 属性を持つため、エラー発生時のみ呼び出される設計となっている。
  • 比較演算には標準の == ではなく、NaNを考慮した専用の比較関数を使用する必要がある点に注意すること。
