SendSharedInvalidMessages

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/sinval.c:46

目的

ローカルのキャッシュ無効化メッセージを共有メモリ上のメッセージキューへ書き込み、他のバックエンドへ通知を行うためのエントリポイントです。

シグネチャまたは形状

void SendSharedInvalidMessages(const SharedInvalidationMessage *msgs, int n)

入力と出力

  • **入力**:
  • msgs: 送信する SharedInvalidationMessage 構造体の配列へのポインタ。
  • n: 送信するメッセージの数。
  • **出力**: なし（void）。

前提条件と不変条件

  • 呼び出し元は、有効な SharedInvalidationMessage の配列と正しい要素数 n を提供する必要があります。
  • 共有メモリ上のメッセージキューが初期化されている必要があります。

アルゴリズムまたは振る舞い

1. 受け取ったメッセージ配列 msgs とその数 n を、共有メモリ管理関数 SIInsertDataEntries に渡します。 2. SIInsertDataEntries を介して、共有メモリ上のリングバッファへメッセージを投入します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/twophase.c
  • src/backend/utils/cache/inval.c
  • **呼び出し先**:
  • SIInsertDataEntries（src/backend/storage/ipc/sinvaladt.c で定義）

関連データ構造

  • SharedInvalidationMessage: 送信されるメッセージの型定義（src/include/storage/sinval.h）。

失敗モードと境界条件

  • 共有メモリのキューが満杯の場合、SIInsertDataEntries の内部実装に従い、キャッシュの全リセットが必要となる可能性があります。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な共有メモリ操作は SIInsertDataEntries に依存しています。
  • キャッシュ無効化のフローにおいて、送信側が共有メモリへ書き込む際の唯一のインターフェースです。
