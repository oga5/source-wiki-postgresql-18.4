PromptInterruptContext

定義

  • 種別: struct
  • 定義箇所: src/include/common/string.h:17

目的

対話型入力処理において、シグナル割り込み（Ctrl+C等）が発生した際に、安全に制御フローを中断および復帰させるためのコンテキストを保持します。

シグネチャまたは形状

typedef struct PromptInterruptContext
{
	void	   *jmpbuf;
	volatile sig_atomic_t *enabled;
	bool		canceled;
} PromptInterruptContext;

入力と出力

  • jmpbuf: setjmp/longjmp で使用するバッファへのポインタ（void * として保持）。
  • enabled: 割り込み時の longjmp を許可するかを制御するフラグへのポインタ。
  • canceled: 割り込みが発生し、キャンセルされたかどうかを示すフラグ。

前提条件と不変条件

  • jmpbuf は setjmp.h の依存を避けるため void * で定義されており、使用時には適切な型へのキャストが必要です。
  • enabled は sig_atomic_t 型を指す必要があり、割り込みハンドラとの間で安全に読み書きされることが想定されています。

アルゴリズムまたは振る舞い

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/bin/psql/command.c (複数箇所)
  • **呼び出し先（利用関数）**:
  • pg_get_line (src/common/pg_get_line.c)
  • simple_prompt (src/common/sprompt.c)

関連データ構造

  • sig_atomic_t (signal.h で定義): 割り込みハンドラ内で安全に更新可能な型。

失敗モードと境界条件

  • jmpbuf が初期化されていない状態で longjmp が試行された場合、未定義の動作を引き起こします。
  • enabled が指すメモリ領域が有効でない場合、割り込み制御が正しく機能しません。

保守担当者向けメモ

  • 本構造体はフロントエンドおよびバックエンドの両方で使用されるため、変更時は両方のコンテキストへの影響を考慮してください。
  • jmpbuf のキャスト忘れによる型安全性の欠如に注意してください。
