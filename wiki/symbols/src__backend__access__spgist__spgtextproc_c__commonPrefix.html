commonPrefix

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgtextproc.c:137

目的

2つの文字列（a と b）を先頭から比較し、一致するバイト数の長さを算出します。SP-GiST 索引における文字列の共通接頭辞抽出や、ノード分岐の判定に使用されます。

シグネチャまたは形状

static int commonPrefix(const char *a, const char *b, int lena, int lenb)

入力と出力

  • **入力**:
  • a, b: 比較対象の文字列ポインタ。
  • lena, lenb: それぞれの文字列の長さ。
  • **出力**:
  • 戻り値: 両方の文字列が先頭から一致しているバイト数（int）。

前提条件と不変条件

  • lena および lenb は、それぞれの文字列の有効な長さである必要があります。
  • 比較はバイト単位で行われます。

アルゴリズムまたは振る舞い

1. インデックス i を 0 に初期化します。 2. i が lena および lenb 未満であり、かつ a[i] と b[i] が等しい間、ループを継続します。 3. ループ内で各ポインタをインクリメントし、i を加算します。 4. 一致が途切れた時点、またはどちらかの文字列の終端に達した時点での i を返します。

関連データ構造

  • text 型（VARDATA_ANY を介してアクセスされる可変長データ）

失敗モードと境界条件

  • 空文字列が渡された場合、戻り値は 0 となります。
  • 文字列の長さが異なる場合、短い方の長さを超えて比較されることはありません。
