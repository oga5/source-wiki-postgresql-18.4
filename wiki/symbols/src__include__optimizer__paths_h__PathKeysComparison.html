PathKeysComparison

定義

  • 種別: enum
  • 定義箇所: src/include/optimizer/paths.h:210

目的

2つのパスキー（PathKeys）リスト間の包含関係や同一性を判定するための列挙型です。プランナがソート順序の最適化やマージ結合の互換性を評価する際に使用されます。

シグネチャまたは形状

typedef enum
{
    PATHKEYS_EQUAL,     /* パスキーは同一 */
    PATHKEYS_BETTER1,   /* パスキー1がパスキー2のスーパーセット */
    PATHKEYS_BETTER2,   /* パスキー2がパスキー1のスーパーセット */
    PATHKEYS_DIFFERENT, /* 互いに包含関係にない */
} PathKeysComparison;

入力と出力

  • 本シンボル自体は列挙値であり、compare_pathkeys(List *keys1, List *keys2) 関数の戻り値として使用されます。

前提条件と不変条件

  • PATHKEYS_BETTER1 は、パスキー1がパスキー2のすべての要素を順序通りに含んでいることを示します。
  • PATHKEYS_DIFFERENT は、どちらのパスキーも相手を完全に包含していない状態を指します。

アルゴリズムまたは振る舞い

  • プランナは compare_pathkeys を呼び出し、得られた PathKeysComparison の値に基づいて、特定のパスが要求されたソート順序を満たしているか、あるいは結合条件に適しているかを判断します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/optimizer/path/pathkeys.c:303
  • src/backend/optimizer/util/pathnode.c:490, 708, 821, 943
  • **呼び出し先**:
  • 本シンボルは列挙型であるため、呼び出し先はありません。

関連データ構造

  • List: パスキーの保持に使用されます。
  • compare_pathkeys: 本列挙型を返す関数として src/include/optimizer/paths.h で宣言されています。

失敗モードと境界条件

  • パスキーが空リストである場合や、等価クラス（EquivalenceClass）の不一致がある場合、比較結果は PATHKEYS_DIFFERENT となるのが一般的です。

保守担当者向けメモ

  • compare_pathkeys の実装（pathkeys.c）を修正する際は、この列挙値の定義が網羅的であることを前提としています。新しい比較ロジックが必要な場合は、列挙値の追加と全参照箇所の更新を検討してください。
