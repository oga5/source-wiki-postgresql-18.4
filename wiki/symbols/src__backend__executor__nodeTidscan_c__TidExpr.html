TidExpr

定義

  • 種別: struct
  • 定義箇所: src/backend/executor/nodeTidscan.c:49

目的

TidScan ノードにおいて、スキャン対象となるタプル識別子（TID）を生成するための式を保持する構造体です。

シグネチャまたは形状

typedef struct TidExpr
{
    ExprState  *exprstate;      /* TIDを生成するサブ式の状態 */
    bool        isarray;        /* trueの場合、単一のTIDではなくTID配列を生成する */
    CurrentOfExpr *cexpr;       /* CURRENT OF 句による指定 */
} TidExpr;

入力と出力

  • **入力**: プランナーが生成した式ツリー（ExprState または CurrentOfExpr）。
  • **出力**: TidListEval 等の評価関数を通じて、物理的なタプル位置を示す ItemPointerData の配列として提供されます。

前提条件と不変条件

  • exprstate または cexpr のいずれかが有効である必要があります。
  • isarray が true の場合、exprstate は tid[] 型の値を返す式である必要があります。

アルゴリズムまたは振る舞い

1. TidExprListCreate により、クエリ内の ctid 関連式から TidExpr 構造体が構築されます。 2. 実行時、TidListEval がこの構造体を参照し、exprstate を評価して TID を抽出します。 3. isarray フラグに基づき、単一の TID か配列かを判別して処理を分岐します。 4. CurrentOfExpr が設定されている場合は、カーソルに関連付けられた TID を動的に解決します。

呼び出し元と呼び出し先

  • **呼び出し元**: TidExprListCreate（構築時）、TidListEval（評価時）。
  • **呼び出し先**: ExecInitTidScan 等の初期化処理からメモリ確保（palloc0）が行われます。

関連データ構造

  • ExprState: 式評価の実行状態。
  • CurrentOfExpr: WHERE CURRENT OF 句の解析結果。
  • ItemPointerData: 実際の物理的なタプル位置（TID）。

失敗モードと境界条件

  • exprstate が NULL かつ cexpr が NULL の場合、不正な状態となります。
  • isarray が false であるにもかかわらず配列が返された場合、型不整合による実行時エラーが発生する可能性があります。

保守担当者向けメモ

  • TidExpr は TidScan ノードの実行計画の一部として tidexprs リストに格納されます。
  • CURRENT OF 処理は通常の ctid 比較とは異なるコードパスを辿るため、変更時は execCurrent.c との整合性に注意してください。
