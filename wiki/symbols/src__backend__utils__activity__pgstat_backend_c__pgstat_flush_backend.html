pgstat_flush_backend

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_backend.c:269

目的

プロセスローカルなバックエンド統計（I/O および WAL 使用量）を、共有メモリ上の統計領域へフラッシュ（書き出し）します。

シグネチャまたは形状

bool pgstat_flush_backend(bool nowait, bits32 flags);

入力と出力

  • **入力**:
  • nowait (bool): true の場合、共有メモリのロック取得に失敗した際に待機せず即座に制御を戻します。
  • flags (bits32): フラッシュ対象を指定するビットマスク（PGSTAT_BACKEND_FLUSH_IO, PGSTAT_BACKEND_FLUSH_WAL 等）。
  • **出力**:
  • bool: 処理が完了したか、あるいはフラッシュすべきデータが存在しなかった場合に false を返します。ロック取得失敗によりフラッシュがスキップされた場合は true を返します。

前提条件と不変条件

  • pgstat_tracks_backend_bktype(MyBackendType) が真である必要があります。
  • 統計更新はクリティカルセクション内でも実行可能な設計となっており、メモリ割り当てを行いません。

アルゴリズムまたは振る舞い

1. **対象判定**: 現在のバックエンド種別が統計収集対象外であれば false を返します。 2. **保留データ確認**: flags で指定されたカテゴリにおいて、ローカルバッファに未反映のデータが存在するか確認します。存在しない場合は false を返します。 3. **ロック取得**: pgstat_get_entry_ref_locked を呼び出し、共有メモリ上のバックエンド統計エントリをロックします。 4. **フラッシュ実行**:

  • PGSTAT_BACKEND_FLUSH_IO が指定されていれば pgstat_flush_backend_entry_io を実行。
  • PGSTAT_BACKEND_FLUSH_WAL が指定されていれば pgstat_flush_backend_entry_wal を実行。

    • 5. **解放**: pgstat_unlock_entry でロックを解除し、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/walsender.c
  • src/backend/utils/activity/pgstat_relation.c
  • src/backend/utils/activity/pgstat_wal.c
  • src/backend/utils/activity/pgstat_backend.c (内部ラッパー)
  • **呼び出し先**:
  • pgstat_tracks_backend_bktype
  • pgstat_backend_wal_have_pending
  • pgstat_get_entry_ref_locked
  • pgstat_flush_backend_entry_io
  • pgstat_flush_backend_entry_wal
  • pgstat_unlock_entry

関連データ構造

  • PendingBackendStats: プロセスローカルな統計保持用構造体。
  • PgStat_EntryRef: 共有メモリ上の統計エントリへの参照およびロック状態を保持。

失敗モードと境界条件

  • nowait が true で、かつ共有メモリのロックが既に保持されている場合、関数は true を返してフラッシュをスキップします。
  • 統計収集対象外のバックエンド種別では、常に false を返します。

保守担当者向けメモ

  • 本関数は、プロセス終了時ではなく、統計の整合性が必要なタイミングで適宜呼び出されることを想定しています。
  • WAL 統計の計算には prevBackendWalUsage を用いた差分更新方式が採用されているため、呼び出し順序や頻度には注意が必要です。
