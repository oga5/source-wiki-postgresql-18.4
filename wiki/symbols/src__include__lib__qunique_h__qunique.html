qunique

定義

  • 種別: function
  • 定義箇所: src/include/lib/qunique.h:20

目的

ソート済みの配列から重複する要素をインプレース（その場で）削除し、ユニークな要素のみで構成される配列の新しいサイズを返します。

シグネチャまたは形状

static inline size_t qunique(void *array, size_t elements, size_t width, int (*compare) (const void *, const void *))

入力と出力

  • **入力**:
  • array: 操作対象のソート済み配列へのポインタ。
  • elements: 配列内の要素数。
  • width: 各要素のサイズ（バイト単位）。
  • compare: 2つの要素を比較する関数ポインタ。
  • **出力**:
  • 重複排除後のユニークな要素数（size_t）。

前提条件と不変条件

  • 入力配列は、提供された compare 関数に従って事前にソートされている必要があります。ソートされていない場合、重複は完全には排除されません。
  • 配列はインプレースで書き換えられるため、呼び出し元は元のデータが不要であることを保証する必要があります。

アルゴリズムまたは振る舞い

1. elements が 1 以下の場合は、そのまま elements を返します。 2. 配列を先頭から走査し、現在の要素と直前のユニークな要素を compare 関数で比較します。 3. 重複していない要素が見つかった場合、その要素をユニーク領域の末尾（j）の直後に memcpy で移動させます。 4. 最終的に、ユニークな要素数である j + 1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な証拠）**:
  • contrib/pg_trgm/trgm_op.c
  • src/backend/access/nbtree/nbtinsert.c
  • src/backend/executor/nodeTidscan.c
  • src/backend/utils/adt/acl.c
  • **呼び出し先**:
  • compare（引数として渡された関数ポインタ）
  • memcpy（標準ライブラリ関数）

関連データ構造

  • 特になし（汎用的な配列操作関数として設計されています）。

失敗モードと境界条件

  • elements が 0 または 1 の場合、比較を行わずに即座に値を返します。
  • width が不適切な場合、memcpy によるメモリ破壊や比較時の不正なメモリアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されているため、呼び出しコストは最小限ですが、memcpy を使用しているため、要素サイズ（width）が大きい場合はパフォーマンスに影響を与える可能性があります。
  • qunique_arg と対になっており、比較関数にユーザーデータを渡す必要がある場合はそちらを使用してください。
