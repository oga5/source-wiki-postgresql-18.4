copy_file_blocks

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/copy_file.c:173

目的

ソースファイルを読み込み、ブロック単位でデスティネーションファイルへ書き出すとともに、転送データのチェックサムを計算・更新する。

シグネチャまたは形状

static void copy_file_blocks(const char *src, const char *dst, pg_checksum_context *checksum_ctx)

入力と出力

  • **入力**:
  • src: コピー元ファイルパス（文字列）
  • dst: コピー先ファイルパス（文字列）
  • checksum_ctx: チェックサム計算用のコンテキスト構造体へのポインタ
  • **出力**: なし（成功時は正常終了、失敗時は pg_fatal によりプロセス終了）

前提条件と不変条件

  • src は読み取り可能であること。
  • dst は書き込み可能であり、かつ既存のファイルと競合しないこと（O_EXCL フラグを使用）。
  • checksum_ctx は初期化済みであること。

アルゴリズムまたは振る舞い

1. src を読み取り専用、dst を書き込み専用（新規作成・排他）でオープンする。 2. 50ブロック分（50 * BLCKSZ）のバッファを確保する。 3. ループ内で read を行い、読み込んだデータを write で dst に書き出す。 4. 書き込みと同時に pg_checksum_update を呼び出し、転送データでチェックサムを更新する。 5. 読み込みが完了したらバッファを解放し、両方のファイルディスクリプタをクローズする。 6. 各システムコール（open, read, write）およびチェックサム更新処理においてエラーが発生した場合、pg_fatal を呼び出してエラーメッセージを出力し終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/copy_file.c 内の copy_file 関数において、コピー戦略として本関数が選択された場合に呼び出される。
  • **呼び出し先**: open, read, write, close などの標準システムコール、および pg_malloc, pg_free, pg_checksum_update などの PostgreSQL 共通関数。

関連データ構造

  • pg_checksum_context: チェックサム計算の状態を保持する構造体。
  • BLCKSZ: PostgreSQL の標準ブロックサイズ。

失敗モードと境界条件

  • ファイルのオープン失敗（権限不足、パス不正など）: pg_fatal で終了。
  • 書き込み時のサイズ不一致: 転送バイト数が期待値と異なる場合、pg_fatal で終了。
  • 読み込みエラー: read が負の値を返した場合、pg_fatal で終了。
  • O_EXCL 指定により、dst が既に存在する場合はオープンに失敗する。

保守担当者向けメモ

  • 本関数はブロック単位の汎用的なコピー実装であり、他の最適化手法（clone や copy_file_range）が利用できない場合のフォールバックとして機能する。
  • チェックサム計算をコピー処理と同期して行うため、他の手法と比較して I/O 効率が異なる点に留意すること。
