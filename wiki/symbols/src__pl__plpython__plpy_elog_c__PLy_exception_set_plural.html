PLy_exception_set_plural

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_elog.c:503

目的

国際化対応された単数形または複数形のメッセージを用いて、Python の例外オブジェクトをセットします。

シグネチャまたは形状

void PLy_exception_set_plural(PyObject *exc, const char *fmt_singular, const char *fmt_plural, unsigned long n, ...)

入力と出力

  • **入力**:
  • exc: セットする Python 例外クラス（PyObject*）。
  • fmt_singular: 単数形のメッセージフォーマット文字列。
  • fmt_plural: 複数形のメッセージフォーマット文字列。
  • n: 数値（単数・複数の判定に使用）。
  • ...: フォーマット引数。
  • **出力**: なし（void）。副作用として Python のエラー状態が更新されます。

前提条件と不変条件

  • n に基づき dngettext が適切なメッセージを選択します。
  • フォーマット後のメッセージは 1024 バイト以内に収まる必要があります。

アルゴリズムまたは振る舞い

1. 可変長引数リスト ap を初期化します。 2. dngettext を使用して、n に応じた適切なフォーマット文字列を選択します。 3. vsnprintf を使用して、固定長バッファ buf (1024バイト) にフォーマット済みメッセージを書き込みます。 4. PyErr_SetString を呼び出し、指定された例外 exc にメッセージ buf をセットします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_cursorobject.c:195
  • src/pl/plpython/plpy_spi.c:202
  • **呼び出し先**:
  • dngettext (PostgreSQL/システムライブラリ)
  • vsnprintf (標準ライブラリ)
  • PyErr_SetString (Python C API)

関連データ構造

  • PyObject: Python のオブジェクト表現。

失敗モードと境界条件

  • メッセージが 1024 バイトを超える場合、vsnprintf の仕様に従い切り詰められます。
  • PyErr_SetString が失敗した場合、Python のエラー状態が適切に設定されない可能性があります。

保守担当者向けメモ

  • 本関数は plpy_elog.h で公開されています。
  • バッファサイズが固定（1024バイト）であるため、非常に長いエラーメッセージを生成する場合は注意が必要です。
