strupper_libc_mb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:372

目的

マルチバイトエンコーディング環境において、OSのlibcロケール機能を利用して文字列を大文字に変換する。

シグネチャまたは形状

size_t strupper_libc_mb(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: 変換後の文字列を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換元のマルチバイト文字列。
  • srclen: src のバイト長（負の場合は strlen(src) を使用）。
  • locale: ロケール情報を含む pg_locale_t 構造体。
  • **出力**:
  • 変換後の文字列のバイト長を size_t で返す。

前提条件と不変条件

  • srclen が INT_MAX / sizeof(wchar_t) を超える場合、メモリ不足エラー（ERRCODE_OUT_OF_MEMORY）を報告する。
  • destsize が変換後のサイズより小さい場合、dest へのコピーは行われない。

アルゴリズムまたは振る舞い

1. srclen が負の場合、strlen で長さを取得する。 2. srclen に基づき、ワイド文字変換用のワークスペースを palloc で確保する。 3. char2wchar を使用して、入力文字列をワイド文字配列に変換する。 4. 各ワイド文字に対し、towupper_l を用いてロケール依存の大文字変換を適用する。 5. 変換後の最大バイト数を計算し、結果格納用のメモリを palloc で確保する。 6. wchar2char を使用して、ワイド文字配列を元のエンコーディングに戻す。 7. destsize が十分な場合、dest に結果をコピーし、NUL終端する。 8. 確保したワークスペースと結果用メモリを pfree で解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/pg_locale_libc.c 内のロケール処理フローから参照されている（具体的な関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: strlen, palloc, char2wchar, towupper_l, pg_database_encoding_max_length, wchar2char, memcpy, pfree。

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。内部の info.lt（locale_t 型）がOSのロケールハンドルとして使用される。

失敗モードと境界条件

  • **メモリ不足**: 入力長が極端に大きい場合、palloc 失敗または ereport によるエラー終了が発生する。
  • **バッファ不足**: destsize が変換後の文字列長（result_size）に対して不足している場合、dest への書き込みはスキップされる。

保守担当者向けメモ

  • 本関数はマルチバイト文字列専用である。シングルバイト用には strupper_libc_sb が存在する。
  • towupper_l の挙動はOSのロケール実装に依存する。
  • pg_database_encoding_max_length() を使用して変換後の最大サイズを予測しているため、エンコーディングごとの特性に依存する。
