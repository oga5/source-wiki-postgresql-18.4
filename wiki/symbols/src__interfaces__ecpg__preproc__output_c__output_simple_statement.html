output_simple_statement

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/output.c:17

目的

ECPGプリプロセッサにおいて、SQLステートメント文字列をエスケープ処理した上で出力し、必要に応じてエラーハンドリングコードと行番号情報を付与する。

シグネチャまたは形状

void output_simple_statement(const char *stmt, int whenever_mode)

入力と出力

  • **入力**:
  • stmt: 出力対象となるSQLステートメントの文字列。
  • whenever_mode: エラーハンドリングの有効性を制御するフラグ（0以外の場合に whenever_action を呼び出す）。
  • **出力**: なし（base_yyout ストリームへの書き込みを行う）。

前提条件と不変条件

  • output_escaped_str がC言語の文字列リテラルとして安全な形式に変換できること。
  • whenever_mode が有効な場合、whenever_action が現在のエラー処理状態に基づいて適切なコードを生成できること。

アルゴリズムまたは振る舞い

1. output_escaped_str(stmt, false) を呼び出し、SQLステートメントをエスケープして出力する。 2. whenever_mode が真であれば、whenever_action(whenever_mode) を呼び出し、エラーハンドリング用のコードを挿入する。 3. output_line_number() を呼び出し、現在のソースコードの行番号情報を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は特定されていませんが、src/interfaces/ecpg/preproc/preproc_extern.h にて外部公開されています。
  • **呼び出し先**:
  • output_escaped_str
  • whenever_action
  • output_line_number

関連データ構造

  • whenever_mode: WHENEVER 文の状態を管理する整数フラグ。

失敗モードと境界条件

  • stmt がNULLの場合の挙動については、本関数内では明示的なチェックを行っていません。
  • whenever_mode が0の場合、エラーハンドリングコードは生成されません。

保守担当者向けメモ

  • 本関数は、生成されるCコードの構造を決定する一連の出力処理の一部です。
  • #line ディレクティブの挿入順序が、生成されたコードのデバッグ可能性に直結するため、変更時には注意が必要です。
