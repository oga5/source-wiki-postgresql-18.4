TidStoreIter

定義

  • 種別: struct
  • 定義箇所: src/include/access/tidstore.h:21

目的

TidStore に格納された ItemPointerData (TID) の集合を効率的に走査するためのイテレータ状態を保持します。

シグネチャまたは形状

typedef struct TidStoreIter TidStoreIter;

不透明な構造体として定義されており、内部メンバは公開されていません。

入力と出力

  • **入力**: TidStoreBeginIterate 関数に TidStore ポインタを渡すことで生成されます。
  • **出力**: TidStoreIterateNext 関数に渡すことで、走査結果である TidStoreIterResult を取得します。

前提条件と不変条件

  • TidStoreBeginIterate によって初期化された後に使用する必要があります。
  • 走査終了後は TidStoreEndIterate を呼び出し、リソースを解放する必要があります。

アルゴリズムまたは振る舞い

1. TidStoreBeginIterate でイテレータを生成し、呼び出し元のメモリコンテキストに確保します。 2. TidStoreIterateNext を繰り返し呼び出すことで、ストア内の TID をブロック単位で順次取得します。 3. 走査が完了した時点で TidStoreEndIterate を呼び出し、イテレータに関連付けられたメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/tidstore.c (イテレータの生成・操作・解放)
  • src/backend/access/heap/vacuumlazy.c
  • src/test/modules/test_tidstore/test_tidstore.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。操作は tidstore.c 内の関数群を介して行われます。

関連データ構造

  • TidStore: 走査対象となる TID ストア本体。
  • TidStoreIterResult: 走査によって得られる結果データ。

失敗モードと境界条件

  • 走査中に TidStore が変更された場合の挙動については、ソース内に明示的な記述はありません。
  • メモリ不足時には palloc0 等のメモリ割り当て関数がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本構造体は不透明型として設計されています。内部構造に依存せず、必ず tidstore.h で提供される API を通じて操作してください。
  • 共有メモリ上の TidStore を走査する場合でも、イテレータ自体のメモリは呼び出し元のローカルメモリコンテキストに確保されます。
