pg_stat_get_progress_info

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pgstatfuncs.c:254

目的

指定されたコマンド名（VACUUM, ANALYZE等）に対応する、現在実行中のバックエンドプロセスの進捗情報を取得し、結果セットとして返却します。

シグネチャまたは形状

Datum pg_stat_get_progress_info(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): 検索対象のコマンド名（文字列）。
  • **出力**:
  • tuplestore を介した行セット。各行にはプロセスID、データベースID、ターゲットID、およびコマンド固有の進捗パラメータが含まれます。

前提条件と不変条件

  • 入力されたコマンド名がサポートされていること（VACUUM, ANALYZE, CLUSTER, CREATE INDEX, BASEBACKUP, COPY）。
  • 呼び出し元が統計情報へのアクセス権限を持つ場合のみ、詳細な進捗パラメータが返されます。権限がない場合は、PIDとDBID以外のカラムはNULLとなります。

アルゴリズムまたは振る舞い

1. 入力されたコマンド名を内部の ProgressCommandType に変換します。無効な場合はエラーを報告します。 2. InitMaterializedSRF を呼び出し、結果セット構築の準備を行います。 3. 全バックエンドを走査し、pgstat_get_local_beentry_by_index で各プロセスの状態を取得します。 4. プロセスの st_progress_command が指定されたコマンドと一致する場合のみ、情報を抽出します。 5. HAS_PGSTAT_PERMISSIONS を用いて権限を検証し、許可されている場合は進捗パラメータを、そうでない場合はNULLを tuplestore_putvalues に渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（システムビュー等）。
  • **呼び出し先**:
  • pgstat_fetch_stat_numbackends()
  • pgstat_get_local_beentry_by_index()
  • tuplestore_putvalues()
  • InitMaterializedSRF()

関連データ構造

  • ProgressCommandType: コマンド種別の列挙型。
  • LocalPgBackendStatus: バックエンドの統計状態を保持する構造体。
  • ReturnSetInfo: SRF（Set Returning Function）の結果格納用構造体。

失敗モードと境界条件

  • サポートされていないコマンド名が指定された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了します。
  • 権限チェックに失敗した場合、機密性の高い進捗パラメータはNULLとして出力されます。

保守担当者向けメモ

  • 新しい進捗追跡コマンドを追加する場合は、コマンド名変換の if-else ブロックおよび ProgressCommandType の定義を更新する必要があります。
  • PG_STAT_GET_PROGRESS_COLS マクロは PGSTAT_NUM_PROGRESS_PARAM + 3 で定義されており、パラメータ数に変更がある場合はこのマクロとループ処理の整合性に注意してください。
