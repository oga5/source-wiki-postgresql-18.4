create_lateral_join_info

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/initsplan.c:895

目的

クエリ内の LATERAL 参照および PlaceHolderVar に関連する依存関係を解析し、各リレーションの lateral_relids および direct_lateral_relids を構築・更新する。これにより、プランナが適切なジョイン順序を決定できるようにする。

シグネチャまたは形状

void create_lateral_join_info(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root (クエリのプランニング情報。simple_rel_array および placeholder_list を含む)
  • **出力**: なし（副作用として root->simple_rel_array 内の各 RelOptInfo および root->hasLateralRTEs を更新する）

前提条件と不変条件

  • root->simple_rel_array が初期化済みであること。
  • root->placeholdersFrozen が true であること（PlaceHolderVar の評価位置が確定している必要がある）。

アルゴリズムまたは振る舞い

1. **初期チェック**: root->hasLateralRTEs が偽であれば即座に終了する。 2. **ベースリレーションの走査**: 各 baserel の lateral_vars を走査し、直接的な Var 参照および PlaceHolderVar 依存関係を direct_lateral_relids および lateral_relids に記録する。 3. **プレースホルダの解析**: root->placeholder_list を走査し、PlaceHolderVar の評価位置 (ph_eval_at) に応じて、関連するベースリレーションの lateral_relids を更新する。 4. **推移的閉包の計算**: ワーシャル（Warshall）アルゴリズムの変種を用い、リレーション間の間接的な LATERAL 依存関係を lateral_relids に伝播させる。 5. **終了処理**: LATERAL 参照が一つも存在しなかった場合、root->hasLateralRTEs を false にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planmain.c:252
  • **呼び出し先**:
  • find_placeholder_info
  • find_base_rel
  • find_base_rel_ignore_join
  • bms_* 系関数（ビットマップセット操作）

関連データ構造

  • PlannerInfo: プランニング全体のコンテキスト。
  • RelOptInfo: 各リレーションの最適化情報。lateral_relids および direct_lateral_relids を保持。
  • PlaceHolderInfo: PlaceHolderVar の評価位置や依存関係を保持。

失敗モードと境界条件

  • LATERAL 参照が存在しない場合、早期リターンする。
  • PlaceHolderVar の評価位置がジョインである場合、そのジョインに含まれる各ベースリレーションに対して間接的な依存関係として記録する。
  • brel が NULL または RELOPT_BASEREL 以外の場合はスキップされる。

保守担当者向けメモ

  • 本関数は、ジョイン順序の制約を決定するための重要な前処理ステップである。
  • ワーシャルアルゴリズムによる推移的閉包の計算は、リレーション数が増加すると計算量が増大する可能性がある。
  • PlaceHolderVar の評価位置が外部結合を跨ぐ場合の挙動は、root->all_baserels との積集合をとることで制御されている。
