bbsink_end_manifest

定義

  • 種別: function
  • 定義箇所: src/include/backup/basebackup_sink.h:245

目的

バックアップマニフェストの生成終了処理を、シンクチェーンの現在のノードから実行する。

シグネチャまたは形状

static inline void bbsink_end_manifest(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - 操作対象のシンク構造体。
  • **出力**: なし。

前提条件と不変条件

  • sink は NULL であってはならない（Assert により検証）。
  • sink->bbs_ops が有効であり、かつ end_manifest コールバック関数が実装されている必要がある。

アルゴリズムまたは振る舞い

1. 引数として渡された sink が NULL でないことを確認する。 2. sink が保持する bbs_ops（操作テーブル）内の end_manifest 関数を呼び出し、マニフェスト終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/backup_manifest.c:374
  • src/backend/backup/basebackup_sink.c:104
  • **呼び出し先**:
  • sink->bbs_ops->end_manifest(sink)

関連データ構造

  • bbsink: シンクの基本構造体。
  • bbsink_ops: シンクの動作を定義するコールバックテーブル。

失敗モードと境界条件

  • sink が NULL の場合、アサーション違反によりプロセスが停止する。
  • sink->bbs_ops または sink->bbs_ops->end_manifest が初期化されていない場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、シンクチェーンの各段階で処理を委譲または終了させるために使用される。
  • src/backend/backup/basebackup_sink.c 内の呼び出しは、チェーンの次段（bbs_next）への転送を目的としている。
