dsm_impl_mmap

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/ipc/dsm_impl.c:791

目的

mmap を使用して、ファイルシステム上のファイルを共有メモリセグメントとしてマッピングする低レベルな動的共有メモリ（DSM）実装を提供します。

シグネチャまたは形状

static bool dsm_impl_mmap(dsm_op op, dsm_handle handle, Size request_size,
                          void **impl_private, void **mapped_address, Size *mapped_size,
                          int elevel);

入力と出力

  • **入力**:
  • op: 操作種別 (DSM_OP_CREATE, DSM_OP_ATTACH, DSM_OP_DETACH, DSM_OP_DESTROY)
  • handle: セグメント識別子
  • request_size: 作成時の要求サイズ
  • elevel: エラー報告レベル
  • **出力**:
  • impl_private: 実装固有のデータ（本関数では未使用）
  • mapped_address: マッピングされたメモリの開始アドレス
  • mapped_size: マッピングされたサイズ
  • **戻り値**: 成功時に true、失敗時に false

前提条件と不変条件

  • PG_DYNSHMEM_DIR ディレクトリが利用可能であること。
  • op が DSM_OP_CREATE の場合、ファイルは新規作成され、指定サイズまでゼロ埋めされる。
  • op が DSM_OP_ATTACH の場合、既存のファイルサイズが自動的に取得される。

アルゴリズムまたは振る舞い

1. **パス生成**: handle に基づき PG_DYNSHMEM_DIR 内のファイルパスを生成。 2. **終了処理**: DETACH または DESTROY の場合、munmap を実行し、DESTROY ならば unlink でファイルを削除。 3. **ファイルオープン**: OpenTransientFile を使用してファイルを開く。 4. **初期化/アタッチ**:

  • CREATE: ファイルを request_size 分だけゼロ埋めして領域を確保。
  • ATTACH: fstat で既存ファイルのサイズを取得。

    • 5. **マッピング**: mmap を MAP_SHARED | MAP_HASSEMAPHORE | MAP_NOSYNC フラグで呼び出し、メモリ空間に配置。 6. **後処理**: ファイル記述子を閉じ、成功を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/dsm_impl.c 内のディスパッチャ関数（dsm_impl_op）から呼び出される。
  • **呼び出し先**: munmap, unlink, OpenTransientFile, fstat, palloc0, write, pgstat_report_wait_start/end, mmap, CloseTransientFile 等のシステムコールおよびPostgreSQL内部関数。

関連データ構造

  • dsm_op: 操作種別を定義する列挙型。
  • dsm_handle: セグメントを一意に識別するID。

失敗モードと境界条件

  • ファイルのオープン、サイズ取得、ゼロ埋め書き込み、または mmap 自体が失敗した場合、エラーを報告し false を返す。
  • ゼロ埋め中にディスク容量が不足した場合、作成したファイルを削除し errno を設定して終了する。
  • munmap や unlink の失敗時もエラーとして報告される。

保守担当者向けメモ

  • 本実装はファイルシステムを利用するため、ディスクI/Oが発生します。パフォーマンス要件が厳しい場合は、PG_DYNSHMEM_DIR をRAMディスク上に配置することを検討してください。
  • ゼロ埋め処理は ZBUFFER_SIZE 単位でループ実行されるため、非常に大きなセグメント作成時には待機イベント WAIT_EVENT_DSM_FILL_ZERO_WRITE が発生します。
