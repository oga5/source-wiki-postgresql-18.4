test_lfind8_internal

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_lfind/test_lfind.c:29

目的

pg_lfind8 関数の正確性を検証するための内部ヘルパー関数。ベクトル演算（SIMD）が適用される範囲と、末尾の端数処理（テール処理）が発生する範囲の両方において、検索対象の有無をテストする。

シグネチャまたは形状

static void test_lfind8_internal(uint8 key)

入力と出力

  • **入力**: key (検索対象とする8ビット値)
  • **出力**: なし（成功時は正常終了、失敗時は elog(ERROR) によりトランザクションをアボート）

前提条件と不変条件

  • pg_lfind8 が正しく実装されていること。
  • LEN_NO_TAIL および LEN_WITH_TAIL マクロが Vector8 に対して正しくバッファ長を計算できること。

アルゴリズムまたは振る舞い

1. LEN_WITH_TAIL(Vector8) サイズのバッファを確保し、0xFF で初期化する。 2. **テール処理のテスト**: バッファの末尾に key を配置し、pg_lfind8 を用いて key-1（存在しない）、key（存在する）、key+1（存在しない）の検索を行い、結果を検証する。 3. **ベクトル演算のテスト**: バッファの len_no_tail - 1 位置に key を配置し、同様に key-1、key、key+1 の検索を行い、結果を検証する。 4. 期待値と異なる結果が得られた場合、elog(ERROR) を発行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_lfind/test_lfind.c 内のテスト関数（test_lfind8 などから、引数に異なる key を指定して複数回呼び出される）。
  • **呼び出し先**: pg_lfind8（検証対象）、memset、elog。

関連データ構造

  • Vector8: 8ビット値のベクトル演算用型。

失敗モードと境界条件

  • key が 0x00 の場合、key-1 の検索はスキップされる。
  • key が 0xFF の場合、key+1 の検索はスキップされる。
  • ベクトル境界（len_no_tail）とバッファ末尾（len_with_tail）の双方で、検索対象が正しく検出されない、あるいは存在しない値が検出された場合にエラーとなる。

保守担当者向けメモ

  • 本関数はユニットテストの一部であり、CI環境での回帰テストを目的としている。
  • テストケースを追加する際は、key の境界値（0, 1, 0x7F, 0x80, 0x81, 0xFD, 0xFE, 0xFF）が網羅されていることを確認すること。
