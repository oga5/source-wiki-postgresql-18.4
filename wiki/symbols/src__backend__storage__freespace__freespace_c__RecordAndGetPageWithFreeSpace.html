RecordAndGetPageWithFreeSpace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/freespace.c:153

目的

特定のページの空き容量を更新しつつ、同時に指定された空き容量を満たす別のページを効率的に検索する。

シグネチャまたは形状

BlockNumber RecordAndGetPageWithFreeSpace(Relation rel, BlockNumber oldPage, Size oldSpaceAvail, Size spaceNeeded)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • oldPage: 更新対象のブロック番号。
  • oldSpaceAvail: oldPage の現在の空き容量。
  • spaceNeeded: 検索対象の最小空き容量。
  • **出力**:
  • 条件を満たすブロック番号（BlockNumber）。条件を満たすページが見つからない場合は、fsm_search の結果を返す。

前提条件と不変条件

  • oldSpaceAvail および spaceNeeded は、内部的にカテゴリ（0〜255）へ変換される。
  • 返されるブロック番号は、リレーション内に物理的に存在することが fsm_does_block_exist によって検証される。

アルゴリズムまたは振る舞い

1. fsm_space_avail_to_cat および fsm_space_needed_to_cat を用いて、入力サイズを FSM 管理用のカテゴリ値に変換する。 2. fsm_get_location を呼び出し、oldPage に対応する FSM 上の論理アドレスとスロットを取得する。 3. fsm_set_and_search を実行し、oldPage の情報を更新しつつ、同じ FSM ページ内で条件を満たす別のスロットを探索する。 4. 探索結果が有効な場合、fsm_get_heap_blk でブロック番号を特定し、そのブロックがリレーション内に存在すればそれを返す。 5. 上記で見つからない場合、fsm_search を呼び出してリレーション全体から条件を満たすページを再検索する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_pageops.c:865
  • src/backend/access/heap/hio.c:759
  • **呼び出し先**:
  • fsm_space_avail_to_cat, fsm_space_needed_to_cat
  • fsm_get_location
  • fsm_set_and_search
  • fsm_get_heap_blk
  • fsm_does_block_exist
  • fsm_search

関連データ構造

  • Relation: リレーション情報。
  • BlockNumber: 物理ブロック番号。
  • FSMAddress: FSM 木構造上の論理アドレス。

失敗モードと境界条件

  • fsm_set_and_search が条件を満たすブロックを見つけられない場合、通常の fsm_search にフォールバックする。
  • fsm_does_block_exist が偽を返した場合、そのブロックは無視され、fsm_search による再検索が行われる。

保守担当者向けメモ

  • 本関数は、更新と検索を単一の操作に統合することで、FSM ページへのアクセス回数を削減しパフォーマンスを向上させる目的がある。
  • FSM はヒント情報であるため、返されたブロックが必ずしも要求された空き容量を保持しているとは限らない（リーフページでの再確認が必要な場合がある）。
