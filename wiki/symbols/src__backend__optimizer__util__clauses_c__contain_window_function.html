contain_window_function

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/clauses.c:215

目的

与えられた式ツリー内にウィンドウ関数（WindowFunc ノード）が含まれているかどうかを判定します。

シグネチャまたは形状

bool contain_window_function(Node *clause)

入力と出力

  • **入力**: clause (ウィンドウ関数の有無を調査する式ツリーのルートノード)
  • **出力**: 式ツリー内にウィンドウ関数が存在する場合は true、存在しない場合は false を返します。

前提条件と不変条件

  • 入力 clause は有効な Node ツリーである必要があります。
  • この関数は、式ツリーの構造を破壊したり変更したりすることはありません。

アルゴリズムまたは振る舞い

内部的に contain_windowfuncs(clause) を呼び出し、その結果をそのまま返します。式ツリーを再帰的に走査し、WindowFunc 型のノードが検出された時点で探索を終了し、真を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/equivclass.c:859
  • src/backend/optimizer/util/var.c:1149
  • **呼び出し先**:
  • contain_windowfuncs (同モジュール内の関数)

関連データ構造

  • Node: PostgreSQLの式ツリーを構成する汎用ノード構造体。
  • WindowFunc: 式ツリー内でウィンドウ関数を表す特定のノード型。

失敗モードと境界条件

  • clause が NULL の場合、あるいはウィンドウ関数を含まない式ツリー（定数、変数、通常の関数呼び出しのみなど）が渡された場合は false を返します。

保守担当者向けメモ

  • 本関数は単なるラッパーです。詳細な走査ロジックについては contain_windowfuncs を参照してください。
  • src/include/optimizer/clauses.h にプロトタイプが定義されており、プランナの各所で式がウィンドウ関数を含むかどうかのガード条件として利用されています。
