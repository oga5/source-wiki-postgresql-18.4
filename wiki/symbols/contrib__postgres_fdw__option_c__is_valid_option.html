is_valid_option

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/option.c:382

目的

指定されたキーワード（オプション名）が、特定のコンテキスト（カタログOID）において有効な postgres_fdw オプションであるかを判定する。

シグネチャまたは形状

static bool is_valid_option(const char *keyword, Oid context)

入力と出力

  • **入力**:
  • keyword: 検証対象のオプション名（文字列）。
  • context: オプションが適用されるカタログの OID。
  • **出力**:
  • bool: 指定されたコンテキストでオプションが定義されていれば true、そうでなければ false を返す。

前提条件と不変条件

  • postgres_fdw_options 配列が初期化済みであること（Assert により検証）。
  • postgres_fdw_options は PgFdwOption 型の構造体配列であり、末尾は keyword が NULL の要素で終端されている必要がある。

アルゴリズムまたは振る舞い

1. 初期化済みの静的配列 postgres_fdw_options を先頭から順に走査する。 2. 各要素について、以下の条件を両方満たすかを確認する。

  • opt->optcontext が引数の context と一致する。
  • opt->keyword が引数の keyword と文字列として一致する（strcmp を使用）。

    • 3. 一致する要素が見つかった場合は直ちに true を返し、配列の終端まで見つからなかった場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/option.c:88（postgres_fdw_validator 内でオプション検証時に呼び出し）。
  • **呼び出し先**:
  • strcmp（標準Cライブラリ）。
  • **注記**: contrib/file_fdw/file_fdw.c にも同名の関数が存在するが、本シンボルとは別個の実装である。

関連データ構造

  • PgFdwOption: オプション名、適用対象コンテキスト、libpq 依存フラグを保持する構造体。

失敗モードと境界条件

  • postgres_fdw_options が初期化されていない状態で呼び出された場合、Assert により異常終了する。
  • keyword が NULL の場合、strcmp によるセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は postgres_fdw_options 配列の線形探索を行うため、オプション数が増加した場合は性能に影響する可能性があるが、現状のオプション数では許容範囲内である。
  • 新しいオプションを追加する場合は、postgres_fdw_options 配列の定義を更新するだけで本関数は自動的に対応する。
