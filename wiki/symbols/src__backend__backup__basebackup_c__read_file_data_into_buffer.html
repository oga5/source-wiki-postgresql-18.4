read_file_data_into_buffer

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup.c:1849

目的

ファイルからデータを読み込み、bbsink バッファに格納すると同時に、必要に応じてページチェックサムの検証を行う。

シグネチャまたは形状

off_t read_file_data_into_buffer(bbsink *sink, const char *readfilename, int fd,
                                 off_t offset, size_t length, BlockNumber blkno,
                                 bool verify_checksum, int *checksum_failures)

入力と出力

  • **入力**:
  • sink: データ転送先およびバッファ管理を行う bbsink オブジェクト。
  • readfilename: 読み込み対象のファイル名。
  • fd: 読み込み対象のファイル記述子。
  • offset: 読み込み開始位置。
  • length: 読み込み要求サイズ。
  • blkno: 読み込み開始ブロック番号。
  • verify_checksum: チェックサム検証を行うかどうかのフラグ。
  • checksum_failures: 検証失敗回数を保持するカウンタへのポインタ。
  • **出力**: 実際に読み込まれたバイト数（off_t）。

前提条件と不変条件

  • sink->bbs_buffer が有効なメモリ領域を指していること。
  • チェックサム検証を行う場合、読み込みサイズは BLCKSZ の倍数である必要がある。

アルゴリズムまたは振る舞い

1. basebackup_read_file を呼び出し、指定された length 分のデータを sink->bbs_buffer に読み込む。 2. チェックサム検証が不要、または読み込みサイズが BLCKSZ の倍数でない場合は即座に読み込みバイト数を返す。 3. BLCKSZ 単位で各ブロックを走査し、verify_page_checksum で検証を行う。 4. 検証に失敗した場合、一度だけ再読み込みを試行する（書き込み中のページによる誤検知を考慮）。 5. 再読み込み後も検証に失敗した場合は、checksum_failures をインクリメントし、警告ログを出力する（最大5回まで報告）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c 内の sendFileWithContent 等から呼び出される。
  • **呼び出し先**: basebackup_read_file、verify_page_checksum、ereport。

関連データ構造

  • bbsink: データ転送パイプラインを定義する構造体。
  • PageHeader: ページチェックサム確認に使用される構造体。

失敗モードと境界条件

  • **EOF 到達**: 再読み込み時に EOF に達した場合、読み込みサイズを調整して処理を終了する。
  • **チェックサム失敗**: 5回目までは警告ログが出力され、それ以降は報告が抑制される。
  • **読み込みサイズ**: BLCKSZ の倍数でない場合、チェックサム検証はスキップされる。

保守担当者向けメモ

  • 本関数は、バックアップ中に発生しうる「書き込み中のページ（torn write）」を考慮した再試行ロジックを含んでいる。
  • 現在の設計では、書き込み完了を待機するインターロックが存在しないため、再試行はあくまで暫定的な対策である点に注意すること。
