check_key_sha1

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubkey.c:252

目的

OpenPGP 秘密鍵パケットの末尾に付与された SHA-1 チェックサムを検証し、鍵データの整合性を確認する。

シグネチャまたは形状

static int check_key_sha1(PullFilter *src, PGP_PubKey *pk);

入力と出力

  • **入力**:
  • src: 鍵データを含む PullFilter。
  • pk: 整合性チェック対象の PGP_PubKey 構造体。
  • **出力**:
  • 成功時は 0 を返す。
  • 失敗時は負の整数（エラーコード）を返す。

前提条件と不変条件

  • src から 20 バイトのチェックサムデータが読み取り可能であること。
  • pk->algo に基づき、対応する秘密鍵の MPI（多倍長整数）が pk->sec に格納されていること。

アルゴリズムまたは振る舞い

1. src から 20 バイトのチェックサム（got_sha1）を読み込む。 2. SHA-1 ダイジェストコンテキストを初期化する。 3. pk->algo に応じて、以下の秘密鍵要素をダイジェストに追加する：

  • ELG_ENCRYPT: x
  • RSA_SIGN/ENCRYPT/ENCRYPT_SIGN: d, p, q, u
  • DSA_SIGN: x

    • 4. 計算されたハッシュ値（my_sha1）と got_sha1 を比較する。 5. 比較結果が一致しない場合、PXE_PGP_KEYPKT_CORRUPT を返す。 6. 終了時に使用したバッファをゼロクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pubkey.c 内の process_secret_key から呼び出される。
  • **呼び出し先**:
  • pullf_read_fixed
  • pgp_load_digest
  • pgp_mpi_hash
  • px_md_finish
  • px_md_free
  • px_memset

関連データ構造

  • PGP_PubKey: 鍵情報を保持する構造体。
  • PullFilter: 入力ストリームの抽象化。
  • PX_MD: ハッシュ計算用コンテキスト。

失敗モードと境界条件

  • pullf_read_fixed が失敗した場合、そのエラーコードをそのまま返す。
  • pgp_load_digest が失敗した場合、エラーを返す。
  • 鍵のハッシュ値が一致しない場合、PXE_PGP_KEYPKT_CORRUPT を返す。

保守担当者向けメモ

  • 本関数は PGP バージョン 4 の秘密鍵パケット形式に依存している。
  • 秘密鍵の MPI 構成に変更があった場合、switch 文内のハッシュ対象フィールドを更新する必要がある。
