ExecIndexOnlyScanInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexonlyscan.c:757

目的

パラレルクエリ実行時において、インデックスのみスキャン（Index-Only Scan）に必要な動的共有メモリ（DSM）領域の割り当てと初期化を行う。

シグネチャまたは形状

void ExecIndexOnlyScanInitializeDSM(IndexOnlyScanState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: IndexOnlyScanState 構造体へのポインタ。スキャンノードの状態を保持。
  • pcxt: ParallelContext 構造体へのポインタ。共有メモリのコンテキスト。
  • **出力**: なし（副作用としてDSM領域の確保と初期化を行う）。

前提条件と不変条件

  • node->ss.ps.instrument が NULL かつ node->ss.ps.plan->parallel_aware が false の場合、DSMは不要であり、関数は即座にリターンする。
  • parallel_aware が true の場合、node->ioss_ScanDesc が初期化され、ioss_VMBuffer が InvalidBuffer に設定される。

アルゴリズムまたは振る舞い

1. **DSM割り当て**: shm_toc_allocate を使用して ioss_PscanLen 分の領域を確保する。 2. **初期化**: index_parallelscan_initialize を呼び出し、インデックススキャンの並列状態を初期化する。 3. **登録**: shm_toc_insert を使用して、プランノードIDに関連付けられたDSM領域を登録する。 4. **並列スキャン設定**: parallel_aware が true の場合、index_beginscan_parallel を呼び出してスキャン記述子を生成し、xs_want_itup を true に設定する。 5. **再スキャン判定**: ランタイムキーが存在しない、または準備完了している場合、index_rescan を呼び出してスキャンキーをインデックスAMに渡す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:483（直接的な証拠あり）。
  • **呼び出し先**: shm_toc_allocate, index_parallelscan_initialize, shm_toc_insert, index_beginscan_parallel, index_rescan（ソースコードより直接的な呼び出しを確認）。

関連データ構造

  • IndexOnlyScanState: スキャンノードの実行状態を管理。
  • ParallelContext: 並列実行の共有メモリコンテキスト。
  • ParallelIndexScanDesc: 並列インデックススキャンの共有状態。

失敗モードと境界条件

  • DSM領域の確保に失敗した場合、shm_toc_allocate 経由でエラーが発生する可能性がある。
  • parallel_aware が false の場合、SharedInfo の初期化のみを行い、スキャン記述子の生成はスキップされる。

保守担当者向けメモ

  • 本関数は IndexOnlyScan の並列実行におけるDSMのセットアップを担う。
  • ioss_VMBuffer は初期化時に必ず InvalidBuffer にリセットされる点に注意すること。
  • index_rescan の呼び出し条件は、ランタイムキーの評価タイミングに依存している。
