ECPGfree_auto_mem

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/memory.c:130

目的

スレッドローカルな自動メモリ追跡リストに登録されたすべてのメモリ領域を解放し、リスト構造体自体を破棄する。

シグネチャまたは形状

void ECPGfree_auto_mem(void)

入力と出力

  • 入力: なし（内部的に get_auto_allocs() を介してスレッド固有のリストを取得）
  • 出力: なし（副作用としてメモリ解放を実行）

前提条件と不変条件

  • get_auto_allocs() が返すリストは、ecpg_auto_alloc によって確保・登録された有効なメモリ領域を指している必要がある。
  • 呼び出し後、スレッドの自動メモリ追跡リストは NULL にリセットされる。

アルゴリズムまたは振る舞い

1. get_auto_allocs() を呼び出し、現在のスレッドのメモリ追跡リストの先頭を取得する。 2. リストが空でない場合、do-while ループでリストを走査する。 3. 各ノードについて、ecpg_free(act->pointer) を呼び出してユーザーが使用していたメモリを解放する。 4. 続いて ecpg_free(act) を呼び出し、リストのノード構造体自体を解放する。 5. 全ノードの処理完了後、set_auto_allocs(NULL) を呼び出し、追跡リストを空に初期化する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/ecpglib/error.c (複数箇所)
  • src/interfaces/ecpg/ecpglib/memory.c
  • src/interfaces/ecpg/ecpglib/misc.c
  • 各種テストコード (sql-dynalloc.c, sql-dynalloc2.c)
  • 呼び出し先:
  • get_auto_allocs()
  • ecpg_free()
  • set_auto_allocs()

関連データ構造

  • struct auto_mem: メモリ追跡リストのノード構造体。確保されたポインタと次のノードへの参照を保持する。

失敗モードと境界条件

  • リストが空（am == NULL）の場合、何も行わずに正常終了する。
  • 既に解放済みのポインタがリストに含まれている場合、ecpg_free の挙動に依存する（二重解放等のリスク）。

保守担当者向けメモ

  • 本関数はエラーハンドリングやスコープ終了時のクリーンアップとして重要である。
  • メモリの所有権を移譲したい場合は、本関数ではなく ecpg_clear_auto_mem を使用すること。
  • auto_mem_destructor を通じてスレッド終了時にも自動的に呼び出される設計となっている。
