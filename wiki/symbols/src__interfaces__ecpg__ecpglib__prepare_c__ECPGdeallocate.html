ECPGdeallocate

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:314

目的

ECPGにおける EXEC SQL DEALLOCATE 文のフロントエンド実装であり、指定された名前のプリペアドステートメントを接続先から破棄します。

シグネチャまたは形状

bool ECPGdeallocate(int lineno, int c, const char *connection_name, const char *name)

入力と出力

  • **入力**:
  • lineno: 呼び出し元の行番号（エラー報告用）。
  • c: 互換性モード（INFORMIX_MODE 等の判定に使用）。
  • connection_name: 接続名（NULLの場合はデフォルト接続）。
  • name: 破棄対象のプリペアドステートメント名。
  • **出力**:
  • 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • 接続オブジェクトが ecpg_get_connection により取得可能であること。
  • ecpg_init による初期化が成功すること。

アルゴリズムまたは振る舞い

1. ecpg_get_connection を呼び出し、指定された接続を取得します。 2. ecpg_init で接続状態を検証します。 3. ecpg_find_prepared_statement を使用して、指定された名前のステートメントを接続のリストから検索します。 4. ステートメントが見つかった場合、deallocate_one を呼び出してバックエンドへの DEALLOCATE コマンド発行およびメモリ解放を行います。 5. ステートメントが見つからなかった場合:

  • INFORMIX_MODE が有効であれば true を返します。
  • それ以外の場合は ecpg_raise を呼び出し、ECPG_INVALID_STMT エラーを発生させて false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPGプリプロセッサが生成するコード（src/interfaces/ecpg/preproc/output.c）および各テストケース。
  • **呼び出し先**: ecpg_get_connection, ecpg_init, ecpg_find_prepared_statement, deallocate_one, ecpg_raise。

関連データ構造

  • struct connection: 接続情報を保持する構造体。
  • struct prepared_statement: プリペアドステートメントの管理用構造体。

失敗モードと境界条件

  • **ステートメント未検出**: 指定された名前のステートメントが存在しない場合、Informixモード以外ではエラー（ECPG_INVALID_STMT）となります。
  • **接続エラー**: ecpg_init が失敗した場合、関数は直ちに false を返します。

保守担当者向けメモ

  • deallocate_one 内部でのメモリ解放処理と同期しているため、ステートメントリストの操作には注意が必要です。
  • Informixモードにおける挙動（ステートメント未検出をエラーとしない）は、既存の互換性要件に基づいています。
