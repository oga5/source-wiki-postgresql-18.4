PosixSemaphoreKill

定義

  • 種別: function
  • 定義箇所: src/backend/port/posix_sema.c:146

目的

POSIX セマフォのリソースを解放し、システムから破棄する。

シグネチャまたは形状

static void PosixSemaphoreKill(sem_t *sem)

入力と出力

  • 入力: sem_t *sem （解放対象のセマフォへのポインタ）
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • sem は有効なセマフォを指している必要がある。
  • コンパイル時のフラグ USE_NAMED_POSIX_SEMAPHORES の状態に応じて、適切な解放関数（sem_close または sem_destroy）が選択される。

アルゴリズムまたは振る舞い

1. USE_NAMED_POSIX_SEMAPHORES が定義されている場合:

  • sem_close(sem) を呼び出し、名前付きセマフォの参照を閉じる。

    • 2. USE_NAMED_POSIX_SEMAPHORES が定義されていない場合:

  • sem_destroy(sem) を呼び出し、名前なしセマフォを破棄する。

    • 3. いずれの関数も失敗（戻り値が 0 未満）した場合は、elog(LOG, ...) を使用してエラーログを出力する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/port/posix_sema.c:245
  • src/backend/port/posix_sema.c:251
  • 呼び出し先:
  • sem_close（USE_NAMED_POSIX_SEMAPHORES 定義時）
  • sem_destroy（USE_NAMED_POSIX_SEMAPHORES 未定義時）
  • elog

関連データ構造

  • sem_t: POSIX セマフォの型。

失敗モードと境界条件

  • sem_close または sem_destroy が失敗した場合、プロセスは終了せずログ出力のみを行う。

保守担当者向けメモ

  • 本関数はセマフォのライフサイクル管理の最終段階で使用される。
  • sem_unlink は本関数内では呼び出されないため、名前付きセマフォを使用する場合は別途適切にアンリンクされていることを確認すること。
