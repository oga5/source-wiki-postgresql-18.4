Expr

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/primnodes.h:188

目的

PostgreSQLのクエリ処理パイプライン（解析・計画・実行）において、すべての実行可能な式（Expression）ノードの基底となる抽象構造体です。

シグネチャまたは形状

typedef struct Expr
{
	pg_node_attr(abstract)

	NodeTag		type;
} Expr;

入力と出力

  • **入力**: 構造体自体は NodeTag を保持するのみです。
  • **出力**: 本構造体を継承する具体的な式ノード（Var, Const, FuncExpr 等）が、実行時に評価結果を生成します。

前提条件と不変条件

  • pg_node_attr(abstract) が付与されており、本構造体自体が直接インスタンス化されることは想定されていません。
  • すべての式ノードは、先頭メンバとして NodeTag を持つ必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータ構造の定義のみであり、実行時のロジックは持ちません。実行時には、ExecEvalExpr 等の関数がこの型を基底として再帰的に走査し、具体的なノード型に応じた評価を行います。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**: contrib/postgres_fdw/deparse.c において、式ツリーのデパース（SQL文字列への変換）処理で頻繁に参照されています。
  • **呼び出し先**: 本構造体は基底クラスであるため、特定の関数を呼び出すことはありません。

関連データ構造

  • NodeTag: ノードの種類を識別するための列挙型。
  • 派生構造体: Var, Const, FuncExpr, OpExpr 等、Expr を継承する具体的な式ノード群。

失敗モードと境界条件

  • 構造体自体に失敗モードはありません。ただし、Expr ポインタを派生型へキャストして利用する際、type フィールドによる型チェックが不適切な場合、実行時エラーや未定義動作の原因となります。

保守担当者向けメモ

  • pg_node_attr マクロが付与されているため、クエリのハッシュ化（Jumbling）や等価性判定の対象となる可能性があります。
  • 本構造体は抽象型であるため、新しい式ノードを追加する際は必ず Expr を継承するように定義してください。
