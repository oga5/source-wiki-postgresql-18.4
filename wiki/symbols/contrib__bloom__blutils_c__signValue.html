signValue

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blutils.c:264

目的

特定の属性値（Datum）をハッシュ化し、Bloom フィルタの署名ビット列（BloomSignatureWord 配列）に反映させる。

シグネチャまたは形状

void signValue(BloomState *state, BloomSignatureWord *sign, Datum value, int attno)

入力と出力

  • **入力**:
  • state: インデックスの実行時状態（ハッシュ関数やオプションを含む）。
  • sign: 更新対象の署名ビット列。
  • value: ハッシュ化対象の属性値。
  • attno: 属性番号。
  • **出力**: なし（sign が指すメモリ領域を直接更新する）。

前提条件と不変条件

  • state->hashFn[attno] に有効なハッシュ関数が設定されていること。
  • attno に対応する state->opts.bitSize[attno] が定義されていること。
  • myRand および mySrand は、インデックスのオンディスク構造の安定性を保つため、PostgreSQL 標準の乱数生成器とは独立して動作する。

アルゴリズムまたは振る舞い

1. mySrand(attno) を呼び出し、属性番号に基づいて乱数生成器を初期化する。これにより、異なる列の同一値が異なるビットにマッピングされることを保証する。 2. FunctionCall1Coll を使用して value をハッシュ化する。 3. ハッシュ値と myRand() の XOR をシードとして再度 mySrand を呼び出し、乱数生成器を再初期化する。 4. state->opts.bitSize[attno] 回ループし、myRand() を用いて生成したビット位置に対して SETBIT マクロを実行し、署名ビットを立てる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blscan.c:105
  • contrib/bloom/blutils.c:312
  • **呼び出し先**:
  • mySrand
  • FunctionCall1Coll
  • myRand
  • SETBIT（マクロ）

関連データ構造

  • BloomState: インデックスの実行時状態を保持。
  • BloomSignatureWord: 署名を構成するビット配列の要素型。

失敗モードと境界条件

  • attno が無効な場合、state->hashFn[attno] へのアクセスでクラッシュする可能性がある。
  • state->opts.bloomLength * SIGNWORDBITS を超えるビット位置が生成されないよう、myRand() の剰余演算で制御されている。

保守担当者向けメモ

  • myRand は PostgreSQL 標準の乱数生成器とは独立しているため、変更時はインデックスのオンディスク互換性に注意すること。
  • SETBIT マクロは複数回評価を避ける必要があるため、ループ内で nBit 変数に一度格納してから使用している。
