pg_prng_uint64_range

定義

  • 種別: function
  • 定義箇所: src/common/pg_prng.c:143

目的

指定された範囲 [rmin, rmax] 内で一様な擬似乱数（uint64）を生成します。

シグネチャまたは形状

uint64 pg_prng_uint64_range(pg_prng_state *state, uint64 rmin, uint64 rmax)

入力と出力

  • **入力**:
  • state: 乱数生成器の状態を保持する pg_prng_state 構造体へのポインタ。
  • rmin: 範囲の最小値（含む）。
  • rmax: 範囲の最大値（含む）。
  • **出力**:
  • rmin 以上 rmax 以下の uint64 型の乱数値を返します。rmax <= rmin の場合は常に rmin を返します。

前提条件と不変条件

  • state は初期化済みである必要があります。
  • 内部で xoroshiro128ss を使用しており、暗号学的な安全性は保証されません。

アルゴリズムまたは振る舞い

1. rmax > rmin の場合、範囲 range = rmax - rmin を算出します。 2. pg_leftmost_one_pos64 を用いて range の最上位ビット位置からシフト量 rshift を決定します。 3. xoroshiro128ss(state) で生成した乱数値を rshift 分右シフトし、その結果が range を超える場合は再試行する「ビットマスク・リジェクションサンプリング」を行います。 4. 最終的に rmin + val を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgdoinsert.c
  • src/backend/optimizer/geqo/geqo_random.c
  • src/backend/storage/file/fd.c
  • src/backend/utils/adt/array_userfuncs.c
  • src/backend/utils/adt/numeric.c
  • src/bin/pgbench/pgbench.c
  • **呼び出し先**:
  • xoroshiro128ss (内部生成関数)
  • pg_leftmost_one_pos64 (ビット操作ユーティリティ)

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • rmax <= rmin: 範囲指定が無効な場合、val は 0 となり、結果として rmin が返されます。
  • rmax - rmin が大きい場合: リジェクションサンプリングの性質上、平均して2回程度の試行で値が決定されます。

保守担当者向けメモ

  • 本関数は暗号学的に安全ではないため、パスワードや暗号鍵の生成には使用しないでください。
  • 内部で使用している xoroshiro128ss の状態がすべて0になると、生成器は0を返し続けるため、初期化時の pg_prng_seed_check によるガードが重要です。
