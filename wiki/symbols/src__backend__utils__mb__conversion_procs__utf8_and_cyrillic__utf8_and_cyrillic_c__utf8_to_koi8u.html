utf8_to_koi8u

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_cyrillic/utf8_and_cyrillic.c:89

目的

UTF-8エンコーディングの文字列をKOI8-Uエンコーディングへ変換するPostgreSQLのロード可能な変換プロシージャです。

シグネチャまたは形状

Datum utf8_to_koi8u(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して以下の引数を受け取ります。
  • src (CSTRING): 変換元のUTF-8文字列。
  • dest (CSTRING): 変換結果を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 変換エラー発生時にエラーを抑制するかどうか。
  • **出力**: Datum 型として変換されたバイト数を返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_UTF8, PG_KOI8U) により、ソースエンコーディングがUTF-8、ターゲットエンコーディングがKOI8-Uであることを検証します。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングIDの妥当性を確認します。 3. UtfToLocal 関数を呼び出し、koi8u_from_unicode_tree 変換マップを使用してUTF-8からKOI8-Uへの変換を実行します。 4. 変換されたバイト数を PG_RETURN_INT32 で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのエンコーディング変換サブシステム（直接的な関数呼び出し元はソースコード上では明示されていません）。
  • **呼び出し先**: UtfToLocal（PostgreSQLのマルチバイト変換汎用関数）。

関連データ構造

  • koi8u_from_unicode_tree: ../../Unicode/utf8_to_koi8u.map から提供される変換テーブル。

失敗モードと境界条件

  • noError 引数が false の場合、変換不可能な文字が含まれるとエラーが発生します。
  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQLの標準的なエラーハンドリングにより処理が中断されます。

保守担当者向けメモ

  • 本関数はテンプレート的な実装であり、他のエンコーディング変換関数と構造が共通しています。
  • 変換ロジックは ../../Unicode/ 配下のマップファイルに依存しているため、変換テーブルの更新が必要な場合は該当ファイルを確認してください。
