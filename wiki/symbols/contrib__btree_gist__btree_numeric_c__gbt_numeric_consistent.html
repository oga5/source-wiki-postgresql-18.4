gbt_numeric_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:104

目的

GiST インデックス検索において、numeric 型のインデックスキーが指定されたクエリ条件を満たすかどうかを判定します。

シグネチャまたは形状

Datum gbt_numeric_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY 構造体（インデックスキーを含む）。
  • PG_GETARG_DATUM(1): 検索対象の numeric 値。
  • PG_GETARG_UINT16(2): 比較戦略を示す StrategyNumber。
  • PG_GETARG_POINTER(4): recheck フラグへのポインタ。
  • **出力**:
  • Datum (bool): 条件に一致する場合は true、それ以外は false を返します。
  • *recheck: 常に false が設定されます（本関数による判定が完全であることを示します）。

前提条件と不変条件

  • 本関数は numeric 型専用の比較ロジックを提供します。
  • *recheck を false に設定することで、呼び出し元に対して再チェックが不要であることを保証します。

アルゴリズムまたは振る舞い

1. GISTENTRY からキーを取り出し、gbt_var_key_readable を用いて読み取り可能な形式に変換します。 2. *recheck を false に設定します。 3. gbt_var_consistent を呼び出し、tinfo（numeric 型の比較関数群を保持する構造体）を使用して、インデックスキーとクエリ値の比較を汎用的な可変長型処理関数に委譲します。 4. 判定結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス検索エンジン（直接の呼び出し箇所はソースコード上では明示されていませんが、PG_FUNCTION_INFO_V1 マクロにより登録されています）。
  • **呼び出し先**:
  • gbt_var_key_readable
  • gbt_var_consistent

関連データ構造

  • GISTENTRY: GiST インデックスの各ノード情報を保持する構造体。
  • GBT_VARKEY: 可変長データ型を格納するための GiST インデックスキー構造体。
  • tinfo: numeric 型の比較関数群を保持する gbtree_vinfo 構造体。

失敗モードと境界条件

  • 本実装は完全な一致判定を行うため、境界条件による再チェック（recheck = true）は発生しません。

保守担当者向けメモ

  • 本関数は btree_utils_var.c で定義された汎用関数 gbt_var_consistent のアダプターとして機能しています。
  • 比較ロジックの変更が必要な場合は、tinfo 構造体が参照している比較関数群を確認してください。
