ExecEndBitmapAnd

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapAnd.c:177

目的

BitmapAnd 実行ノードの終了処理を行い、保持しているすべての子ノードのリソースを解放する。

シグネチャまたは形状

void ExecEndBitmapAnd(BitmapAndState *node)

入力と出力

  • **入力**: BitmapAndState *node（終了対象となる BitmapAnd ノードの状態構造体）
  • **出力**: なし（void）

前提条件と不変条件

  • node は ExecInitBitmapAnd によって初期化済みであること。
  • node->bitmapplans に格納された各サブプランに対して ExecEndNode を呼び出す。

アルゴリズムまたは振る舞い

1. node からサブプランの配列 (bitmapplans) とその数 (nplans) を取得する。 2. ループを使用して、各サブプランを順次走査する。 3. 各サブプランが存在する場合、ExecEndNode を呼び出して個別の終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:613（ExecEndNode 経由での呼び出しが確認されている）
  • **呼び出し先**: ExecEndNode（各サブプランに対して呼び出される）

関連データ構造

  • BitmapAndState: nodeBitmapAnd.c で管理される実行状態構造体。
  • PlanState: bitmapplans 配列の各要素の型。

失敗モードと境界条件

  • bitmapplans[i] が NULL の場合、その要素に対する ExecEndNode の呼び出しはスキップされる。

保守担当者向けメモ

  • 本関数は ExecProcNode 呼び出し規約をサポートしていないため、実行時には MultiExecProcNode を使用すること。
  • ExecReScanBitmapAnd と同様に、サブプランの管理において BitmapAndState の構造と整合性を保つ必要がある。
