new_variable

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/variable.c:9

目的

ECPGプリプロセッサにおいて、新しいホスト変数エントリを生成し、管理用の静的リンクリスト allvariables に追加します。

シグネチャまたは形状

struct variable *new_variable(const char *name, struct ECPGtype *type, int brace_level)

入力と出力

  • **入力**:
  • name: 変数名（文字列）。
  • type: 変数の型情報（ECPGtype 構造体へのポインタ）。
  • brace_level: 変数が宣言されたスコープ階層を示す整数値。
  • **出力**:
  • 生成された variable 構造体へのポインタを返します。

前提条件と不変条件

  • mm_alloc を使用してメモリを確保するため、メモリ不足時はプロセスが終了する可能性があります。
  • allvariables は静的なリンクリストであり、本関数は常に新しい変数をリストの先頭に追加します。

アルゴリズムまたは振る舞い

1. mm_alloc を用いて variable 構造体分のメモリを確保します。 2. 引数 name を mm_strdup で複製し、構造体の name フィールドに格納します。 3. 引数 type および brace_level を構造体の各フィールドに設定します。 4. 現在の allvariables を新しい変数の next ポインタに代入し、allvariables を新しい変数で更新します。 5. 生成した構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/variable.c 内の各関数（構造体や配列の型解決を行う箇所など、複数箇所から参照）。
  • **呼び出し先**: mm_alloc（メモリ確保）、mm_strdup（文字列複製）。

関連データ構造

  • variable: 変数情報を保持する構造体。allvariables リストの要素。
  • ECPGtype: 変数の型定義を保持する構造体。
  • allvariables: 宣言された全変数を管理する静的なリンクリスト。

失敗モードと境界条件

  • メモリ割り当て失敗時: mm_alloc 内部でエラーが発生し、プリプロセッサが終了します。
  • name が NULL の場合: mm_strdup 呼び出し時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は allvariables リストの先頭挿入のみを行います。リストからの削除や検索については remove_variables や find_variable を参照してください。
  • brace_level はスコープ管理に必須であり、呼び出し側で現在の構文解析レベルと整合性が取れていることを確認してください。
