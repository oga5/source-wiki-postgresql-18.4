PGSharedMemoryAttach

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_shmem.c:346

目的

指定された System V 共有メモリセグメントが現在のデータディレクトリに関連するものかを確認し、その状態（アタッチ可能か、他プロセスが使用中か等）を判定する。

シグネチャまたは形状

static IpcMemoryState PGSharedMemoryAttach(
    IpcMemoryId shmId, 
    void *attachAt, 
    PGShmemHeader **addr
)

入力と出力

  • **入力**:
  • shmId: 調査対象の共有メモリセグメント ID。
  • attachAt: アタッチを試みるメモリアドレス（NULL 可）。
  • addr: 成功時にアタッチされた共有メモリヘッダーのアドレスを格納するポインタ。
  • **出力**:
  • IpcMemoryState 型の列挙値（SHMSTATE_ENOENT, SHMSTATE_FOREIGN, SHMSTATE_ANALYSIS_FAILURE, SHMSTATE_UNATTACHED, SHMSTATE_ATTACHED）。

前提条件と不変条件

  • DataDir（グローバル変数）が有効なデータディレクトリを指していること。
  • shmat 呼び出し時に使用される PG_SHMAT_FLAGS がシステム環境に応じて適切に定義されていること。

アルゴリズムまたは振る舞い

1. **セグメントの存在確認**: shmctl(IPC_STAT) を呼び出し、セグメントのメタデータを取得する。失敗時はエラーコード（EINVAL, EACCES, EIDRM）に基づき、存在しないか外部セグメントと判定する。 2. **データディレクトリの照合**: stat(DataDir) を実行し、現在のデータディレクトリのデバイス ID と inode を取得する。 3. **アタッチ**: shmat を用いてセグメントをプロセス空間にマッピングする。失敗時はエラーコードに基づき状態を返す。 4. **検証**: マッピングされた PGShmemHeader の magic 番号、device、inode が現在のデータディレクトリと一致するか確認する。不一致の場合は SHMSTATE_FOREIGN を返す。 5. **使用状況の判定**: shmctl で取得した shm_nattch を参照し、他プロセスがアタッチ中であれば SHMSTATE_ATTACHED、そうでなければ SHMSTATE_UNATTACHED を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/sysv_shmem.c 内の複数箇所（PGSharedMemoryCreate や共有メモリ再アタッチ処理など）。
  • **呼び出し先**:
  • shmctl, stat, shmat（OS システムコール）。

関連データ構造

  • IpcMemoryState: 共有メモリの状態を示す列挙型。
  • PGShmemHeader: 共有メモリセグメントの先頭に配置されるヘッダー構造体。
  • shmid_ds: System V 共有メモリのメタデータ構造体。

失敗モードと境界条件

  • **セグメント消滅**: shmctl や shmat で EINVAL が返る場合、セグメントが既に削除されたとみなす。
  • **権限不足**: EACCES が返る場合、PostgreSQL 以外のセグメントとみなす。
  • **Linux のバグ**: HAVE_LINUX_EIDRM_BUG が定義されている環境では、EIDRM を EINVAL と同等に扱う。
  • **保守的判定**: 予期せぬエラー発生時は SHMSTATE_ANALYSIS_FAILURE を返し、安全側に倒す。

保守担当者向けメモ

  • shmctl による shm_nattch の値は、現在のプロセスがアタッチする前の時点での値である点に注意すること。
  • PGSharedMemoryCreate から呼び出される場合、stat 呼び出しが重複するが、頻繁に発生するパスではないため最適化は不要と判断されている。
