HeapKeyTest

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/valid.h:27

目的

ヒープタプルが指定された複数の検索条件（スキャンキー）すべてに合致するかを判定する。

シグネチャまたは形状

bool HeapKeyTest(HeapTuple tuple, TupleDesc tupdesc, int nkeys, ScanKey keys)

入力と出力

  • **入力**:
  • tuple: 検査対象の HeapTuple。
  • tupdesc: タプルの属性情報を保持する TupleDesc。
  • nkeys: 評価するキーの数。
  • keys: 評価対象の ScanKey 配列。
  • **出力**:
  • すべてのキー条件を満たす場合は true、一つでも不一致または NULL 判定で失敗した場合は false を返す。

前提条件と不変条件

  • keys 配列は nkeys 個の有効な要素を持つ必要がある。
  • NULL 値の扱いは厳格であり、キーが NULL を期待していない場合に属性値が NULL であれば、即座に不一致とみなされる。

アルゴリズムまたは振る舞い

1. nkeys 分だけ ScanKey 配列を順次走査する。 2. 各キーについて以下の処理を行う：

  • SK_ISNULL フラグが立っている場合は false を返す。
  • heap_getattr を用いてタプルから属性値を抽出する。
  • 抽出した属性値が NULL の場合は false を返す。
  • FunctionCall2Coll を呼び出し、比較関数を実行して条件を評価する。
  • 評価結果が false であれば即座に終了し false を返す。

    • 3. すべてのキーで条件を通過した場合に true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c (ヒープスキャン処理内)
  • **呼び出し先**:
  • heap_getattr
  • FunctionCall2Coll

関連データ構造

  • HeapTuple: 検査対象のタプル構造体。
  • TupleDesc: タプルの記述子。
  • ScanKey: 検索条件を保持する構造体。

失敗モードと境界条件

  • nkeys が 0 の場合、ループは実行されず常に true を返す。
  • 属性値が NULL の場合、キーの期待値に関わらず不一致として扱われる。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、パフォーマンスに直結する。インライン展開によるバイナリサイズへの影響を考慮すること。
  • catcache.c 等で sk_attno の初期化が正しく行われていることが前提となる。
