GzipCompressorState

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:36

目的

pg_dump および pg_restore における GZIP 圧縮・解凍処理において、圧縮ストリームの状態を保持するために使用される内部構造体です。

シグネチャまたは形状

typedef struct GzipCompressorState
{
	z_streamp	zp;         /* zlib 圧縮ストリーム */
	void	   *outbuf;     /* 出力バッファ */
	size_t		outsize;    /* 出力バッファサイズ */
} GzipCompressorState;

入力と出力

  • 本構造体は pg_malloc0 を介してメモリ確保され、CompressorState の private_data メンバとして保持されます。
  • z_streamp を通じて zlib ライブラリの圧縮・解凍状態を管理します。

前提条件と不変条件

  • HAVE_LIBZ が定義されている環境で利用可能です。
  • CompressorState の private_data としてキャストして使用されます。

アルゴリズムまたは振る舞い

  • zlib の deflate および inflate 処理におけるバッファリング状態を保持します。
  • 圧縮データは outbuf を経由してアーカイブへ書き出されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/compress_gzip.c 内の各関数（InitCompressorGzip 等）が本構造体をインスタンス化し、cs->private_data に格納します。
  • 呼び出し先: zlib ライブラリの関数群（deflate, inflate 等）。

関連データ構造

  • CompressorState: 本構造体を内部状態として保持する上位の圧縮インターフェース構造体。

失敗モードと境界条件

  • メモリ確保失敗時: pg_malloc0 によりプロセスが終了する可能性があります。
  • zlib 関連の処理失敗: zlib のエラーコードが返され、pg_dump のエラー報告メカニズムへ変換されます。

保守担当者向けメモ

  • gzgetc マクロは、ライブラリとヘッダー間の構造体レイアウトの不一致を避けるため、本ファイル内で意図的に #undef されています。
  • ゼロ長チャンクの書き出しは、アーカイブ形式の EOF マーカーとの衝突を避けるためガードロジックが必要です。
