r_R1

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_english.c:479

目的

Porterステミングアルゴリズムにおいて、接尾辞の削除や置換が許可される領域「R1」の範囲内にあるかを判定する。

シグネチャまたは形状

static int r_R1(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（Snowballの実行環境状態）
  • 出力: int（判定結果。領域R1内であれば非ゼロ、そうでなければ0を返す）

前提条件と不変条件

  • z->I[1] には、事前に r_mark_regions 関数によって計算されたR1領域の開始位置が格納されている必要がある。
  • z->c は現在のカーソル位置を示す。

アルゴリズムまたは振る舞い

1. z->I[1]（R1領域の開始インデックス）と z->c（現在のカーソル位置）を比較する。 2. z->I[1] <= z->c が真であれば、現在のカーソル位置はR1領域内（または境界上）にあると判断し、成功（非ゼロ）を返す。 3. それ以外の場合は失敗（0）を返す。

呼び出し元と呼び出し先

  • 呼び出し元: stem_ISO_8859_1_basque.c、stem_ISO_8859_1_catalan.c、stem_ISO_8859_1_dutch.c 等の各言語実装ファイルから、接尾辞処理の条件判定として呼び出される。
  • 呼び出し先: 直接的な関数呼び出しはない。

関連データ構造

  • struct SN_env: Snowballランタイムの環境構造体。I 配列（整数型ワークスペース）および c（カーソル位置）を保持する。

失敗モードと境界条件

  • z が NULL の場合、あるいは z->I が初期化されていない場合は未定義動作となる。
  • カーソル位置が文字列の末尾を超えている場合や、R1領域が定義されていない（r_mark_regions が未実行）場合は、意図しない判定結果となる可能性がある。
