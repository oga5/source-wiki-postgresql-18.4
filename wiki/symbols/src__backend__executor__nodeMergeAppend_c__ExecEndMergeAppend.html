ExecEndMergeAppend

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMergeAppend.c:334

目的

MergeAppend 実行ノードの終了処理を行い、関連するすべてのサブプラン（サブスキャン）をクリーンアップします。

シグネチャまたは形状

void ExecEndMergeAppend(MergeAppendState *node)

入力と出力

  • **入力**: MergeAppendState *node - 終了対象となる MergeAppend ノードの実行時状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • 引数 node は有効な MergeAppendState 構造体を指している必要があります。
  • node->mergeplans には、初期化済みの PlanState 配列が格納されている必要があります。

アルゴリズムまたは振る舞い

1. node からサブプランのリスト (mergeplans) とサブプラン数 (nplans) を取得します。 2. 0 から nplans - 1 までのインデックスでループを実行します。 3. 各ループ内で ExecEndNode(mergeplans[i]) を呼び出し、個々のサブプランの終了処理を再帰的に実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:605 にて呼び出されていることが確認されています。
  • **呼び出し先**: ExecEndNode を呼び出します。

関連データ構造

  • MergeAppendState: MergeAppend ノードの実行時状態を保持する構造体。
  • PlanState: 各サブプランの実行状態を保持する構造体。

失敗モードと境界条件

  • nplans が 0 の場合、ループは実行されず、何も行われません。
  • mergeplans 内の各 PlanState が適切に初期化されていない場合、ExecEndNode の呼び出しにより未定義の動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は MergeAppend ノードの終了処理の責任を負います。サブプランの追加や変更を行った際は、本関数がすべてのサブプランを正しく終了させているか確認してください。
