analyze_rel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/analyze.c:108

目的

指定されたリレーション（テーブル、マテリアライズドビュー、外部テーブル、またはパーティションテーブル）に対して統計情報を生成し、pg_statistic カタログを更新するためのエントリポイントとして機能します。

シグネチャまたは形状

void analyze_rel(Oid relid, RangeVar *relation, VacuumParams *params, List *va_cols, bool in_outer_xact, BufferAccessStrategy bstrategy)

入力と出力

  • **入力**:
  • relid: 対象リレーションの OID。
  • relation: 対象リレーションの識別情報（RangeVar）。
  • params: VACUUM オプションを含むパラメータ構造体。
  • va_cols: 解析対象とする列のリスト。
  • in_outer_xact: 外部トランザクション内で実行されているかを示すフラグ。
  • bstrategy: バッファアクセスの戦略。
  • **出力**: なし（副作用として統計情報の更新および進捗報告を行う）。

前提条件と不変条件

  • ShareUpdateExclusiveLock を取得し、対象リレーションに対する同時実行の ANALYZE を防止します。
  • pg_statistic テーブル自体は解析対象外です。
  • 一時テーブル（他バックエンドのもの）は解析対象外としてスキップされます。

アルゴリズムまたは振る舞い

1. **初期化**: ログレベルの設定とバッファ戦略の適用。 2. **ロックと検証**: vacuum_open_relation でリレーションを開き、権限チェックを実行。 3. **リレーション種別の判定**:

  • 通常テーブル/マテリアライズドビュー: acquire_sample_rows を使用してサンプル行を取得。
  • 外部テーブル: FDW（Foreign Data Wrapper）の AnalyzeForeignTable フックを呼び出し、解析可否と取得関数を決定。
  • パーティションテーブル: 再帰的な解析のためにスキップ。

    • 4. **進捗報告**: pgstat_progress_start_command を開始。 5. **統計生成**:

  • 非パーティションテーブルの場合: do_analyze_rel を呼び出し、統計を計算・更新。
  • 子テーブルが存在する場合: do_analyze_rel を再帰的に呼び出し。

    • 6. **終了処理**: リレーションをクローズし、進捗報告を終了。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuum.c (直接の呼び出しを確認)
  • **呼び出し先**:
  • vacuum_open_relation
  • vacuum_is_permitted_for_relation
  • acquire_sample_rows (通常テーブルの場合)
  • GetFdwRoutineForRelation (外部テーブルの場合)
  • do_analyze_rel
  • pgstat_progress_start_command / pgstat_progress_end_command

関連データ構造

  • VacuumParams: 解析オプションを保持。
  • Relation: 対象リレーションのメタデータ。
  • FdwRoutine: 外部テーブル解析用の関数ポインタ群。

失敗モードと境界条件

  • リレーションがオープンできない、またはロックできない場合は即座に終了。
  • 権限がない場合、または解析対象外の特殊なシステムテーブルの場合はスキップ。
  • 外部テーブルで解析がサポートされていない場合は警告を出して終了。

保守担当者向けメモ

  • 本関数は ANALYZE の高レベルな制御フローを担います。実際の統計計算ロジックは do_analyze_rel に委譲されています。
  • 外部テーブルの解析は FDW の実装に依存するため、fdwroutine->AnalyzeForeignTable の戻り値チェックが重要です。
  • リレーションのクローズ後もロックを保持することで、コミットまでの整合性を担保しています。
