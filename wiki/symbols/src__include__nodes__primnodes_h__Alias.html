Alias

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/primnodes.h:48

目的

クエリ内のリレーション（テーブル等）やサブクエリに付与された別名（エイリアス）および、オプションとして指定される列名の別名を保持するために使用されます。

シグネチャまたは形状

typedef struct Alias
{
	NodeTag		type;
	char	   *aliasname;		/* aliased rel name (never qualified) */
	List	   *colnames;		/* optional list of column aliases */
} Alias;

入力と出力

  • **入力**: aliasname（文字列）および colnames（List）。
  • **出力**: 構造体自体がノードとしてクエリツリーの一部を構成し、解析・計画段階で参照されます。

前提条件と不変条件

  • aliasname は修飾名（スキーマ名等）を含まない単純な名前である必要があります。
  • colnames はオプションであり、指定されない場合は NIL となります。

アルゴリズムまたは振る舞い

本構造体はデータ保持に特化しており、実行時の計算ロジックは持ちません。クエリの解析段階で生成され、以降の計画作成や説明（EXPLAIN）出力において、リレーションの識別や列名のマッピングに使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（生成）**: src/backend/nodes/makefuncs.c 内の関数（makeNode(Alias) を使用）がノード生成の起点となります。
  • **呼び出し先（参照）**:
  • contrib/pg_overexplain/pg_overexplain.c: エイリアス情報の出力に使用。
  • src/backend/commands/explain.c: EXPLAIN コマンドによるクエリ計画表示時に参照。
  • src/backend/nodes/queryjumblefuncs.c: クエリのハッシュ化処理で参照。
  • src/backend/optimizer/plan/setrefs.c および src/backend/parser/parse_cte.c: 計画最適化やCTE解析の過程で参照。

関連データ構造

  • NodeTag: ノードの種類を識別するためのタグ。
  • List: colnames を保持するために使用される PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • メモリ不足により Alias ノードの割り当てが失敗する可能性があります。
  • aliasname が適切に設定されていない場合、クエリの識別や EXPLAIN 出力において不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は primnodes.h で定義されており、クエリ処理の全段階で共通して利用されます。変更を加える際は、解析ツリー（parsenodes.h）および計画ツリー（plannodes.h）への影響を考慮してください。
  • colnames のリスト操作については、nodes/pg_list.h で定義された標準的なリスト操作関数を使用してください。
