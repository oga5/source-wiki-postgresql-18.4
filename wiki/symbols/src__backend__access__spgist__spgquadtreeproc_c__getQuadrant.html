getQuadrant

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgquadtreeproc.c:54

目的

SP-GiST四分木において、基準点（重心）に対する対象点の位置関係に基づき、1から4のいずれかの象限番号を決定する。

シグネチャまたは形状

static int16 getQuadrant(Point *centroid, Point *tst)

入力と出力

  • **入力**:
  • centroid: 基準となる重心の Point 構造体へのポインタ。
  • tst: 判定対象となる Point 構造体へのポインタ。
  • **出力**:
  • int16: 象限番号（1, 2, 3, 4）。

前提条件と不変条件

  • 象限は重心を原点として、1（右上）、2（右下）、3（左下）、4（左上）の順に反時計回りで定義される。
  • 軸上の点は、より番号の小さい象限に分類される。
  • どの象限にも該当しない状態は論理的に不可能であり、発生した場合はエラーとなる。

アルゴリズムまたは振る舞い

1. SPTEST マクロを用いて、tst が centroid に対してどの方向（上下左右）にあるかを判定する。 2. 以下の条件で象限を特定する:

  • **象限1**: 上または水平、かつ右または垂直。
  • **象限2**: 下、かつ右または垂直。
  • **象限3**: 下または水平、かつ左。
  • **象限4**: 上、かつ左。

    • 3. いずれにも該当しない場合、elog(ERROR) を発行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgquadtreeproc.c 内の spg_quad_choose、spg_quad_picksplit、spg_quad_inner_consistent などから複数回参照されている。
  • src/backend/utils/adt/geo_spgist.c からも参照されている（ただし同名で引数が BOX* の別関数が存在する可能性に注意）。
  • **呼び出し先**:
  • SPTEST マクロ（point_above, point_right 等の幾何学演算関数をラップ）。
  • elog（エラー発生時）。

関連データ構造

  • Point: utils/geo_decls.h で定義される座標データ構造。

失敗モードと境界条件

  • 基準点と対象点が完全に一致する場合や軸上に重なる場合、定義された優先順位に従い特定の象限に割り当てられる。
  • 内部ロジックで網羅できないケースが発生した場合は ERROR をスローし、処理を中断する。

保守担当者向けメモ

  • 本関数は Point 型専用である。BOX 型を扱う同名の関数が src/backend/utils/adt/geo_spgist.c に存在するため、修正時は対象ファイルを確認すること。
  • 象限の定義（反時計回り、軸上の点の扱い）は SP-GiST のインデックス構造に直結しているため、変更には注意が必要である。
