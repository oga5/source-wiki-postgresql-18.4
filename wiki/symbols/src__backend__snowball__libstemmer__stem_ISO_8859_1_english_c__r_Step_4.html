r_Step_4

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_english.c:785

目的

Porterステミングアルゴリズムの第4段階として、特定の語尾（接尾辞）をR2領域内で削除または置換する。

シグネチャまたは形状

static int r_Step_4(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体）
  • 出力: int（成功時に1、失敗または条件不一致時に0、エラー時に負の値を返す）

前提条件と不変条件

  • z が有効な SN_env 構造体を指していること。
  • r_R2(z) が呼び出され、現在のカーソル位置がR2領域内にあることが条件となる。

アルゴリズムまたは振る舞い

1. find_among_b を使用し、テーブル a_7 に基づいて語尾を後方から検索する。 2. 検索対象の文字が特定のビットマスク条件を満たさない場合は即座に終了する。 3. r_R2(z) を呼び出し、対象の語尾がR2領域内にあるか検証する。 4. among_var の値に応じて以下の処理を行う:

  • case 1: slice_del(z) を実行し、語尾を削除する。
  • case 2: 語尾が 's' または 't' で終わるかを確認し、条件を満たせば slice_del(z) を実行する。

    • 5. 処理完了後、1を返す。

呼び出し元と呼び出し先

  • 呼び出し元: stem_ISO_8859_1_english.c および関連する porter.c、UTF_8_english.c 等のメインステミング処理フローから呼び出される。
  • 呼び出し先: find_among_b（語尾検索）、r_R2（領域判定）、slice_del（文字列操作）。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_7: ステミングルールで使用される among 構造体のテーブル。

失敗モードと境界条件

  • find_among_b が一致する語尾を見つけられない場合、0を返す。
  • r_R2 が失敗した場合、その戻り値をそのまま返す。
  • slice_del がメモリ操作等で失敗した場合、負の値を返す。
  • カーソル位置が lb（下限）を超えた場合、安全に終了する。
