element_hash

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_typanalyze.c:709

目的

配列要素のハッシュ値を計算し、ANALYZE 処理中の要素頻度集計用ハッシュテーブルにおけるキーとして利用可能にする。

シグネチャまたは形状

static uint32 element_hash(const void *key, Size keysize)

入力と出力

  • **入力**:
  • key: Datum 型の値を指すポインタ。
  • keysize: キーのサイズ（Size）。
  • **出力**:
  • uint32: 計算されたハッシュ値。

前提条件と不変条件

  • グローバル変数またはコンテキスト変数 array_extra_data が初期化されており、有効なハッシュ関数 (hash) と照合用照合順序 (coll_id) が設定されていること。
  • key は Datum 型の値を指している必要がある。

アルゴリズムまたは振る舞い

1. key が指す Datum 値を取り出す。 2. array_extra_data->hash に格納された関数を FunctionCall1Coll を介して呼び出し、ハッシュ計算を実行する。 3. 計算結果を DatumGetUInt32 で uint32 型に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/array_typanalyze.c:281: elem_hash_ctl.hash の初期化時に参照されている。
  • **呼び出し先**:
  • FunctionCall1Coll: array_extra_data->hash に登録された関数を実行するために使用。

関連データ構造

  • ArrayAnalyzeExtraData: array_extra_data の型。要素型の型情報やハッシュ関数を保持する。

失敗モードと境界条件

  • array_extra_data が未初期化の場合、ヌルポインタ参照によりクラッシュする可能性がある。
  • 登録されたハッシュ関数がエラーをスローした場合、そのエラーが伝播する。

保守担当者向けメモ

  • 本関数は hash_create 等で作成されるハッシュテーブルのコールバックとして使用される。
  • src/backend/utils/adt/rowtypes.c に同名のシンボルが存在するが、本ページは array_typanalyze.c 内の定義を対象としている。混同しないよう注意すること。
