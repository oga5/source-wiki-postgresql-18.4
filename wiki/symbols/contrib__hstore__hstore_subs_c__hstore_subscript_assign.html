hstore_subscript_assign

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_subs.c:142

目的

hstore 型のデータに対して、添字構文（hstore_val['key'] = 'value'）を用いた値の挿入または更新を行う。

シグネチャまたは形状

static void hstore_subscript_assign(ExprState *state, ExprEvalStep *op, ExprContext *econtext);

入力と出力

  • **入力**:
  • state: 式の評価状態。
  • op: 実行ステップ情報。op->d.sbsref.state から添字情報、op->resvalue から対象の hstore を取得。
  • econtext: 式の実行コンテキスト。
  • **出力**:
  • op->resvalue: 更新後の新しい hstore を指す Datum。
  • op->resnull: 更新結果が NULL かどうかを示すフラグ（常に false に設定）。

前提条件と不変条件

  • 添字（キー）は text 型である必要がある。
  • hstore は不変（immutable）なデータ構造として扱われるため、代入操作は常に新しいメモリ領域を確保して新しい hstore を構築する。
  • 添字が NULL の場合、エラーが発生する。

アルゴリズムまたは振る舞い

1. **添字の検証**: 添字が NULL でないことを確認する。 2. **キーと値の準備**: 添字からキーを抽出し、代入する値が NULL かどうかを確認する。 3. **新規構築**:

  • op->resnull が真の場合、単一要素の hstore を作成する。
  • op->resnull が偽の場合、既存の hstore を走査し、新しいキーと値をマージして新しい hstore を構築する。

    • 4. **マージ処理**: キーの順序を維持しながら、既存のキーと新しいキーを比較し、新しい hstore のデータ領域にコピーする。 5. **結果の格納**: 生成された hstore を op->resvalue に格納し、op->resnull を false に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_subs.c 内の hstore_subscript_handler が本関数を SubscriptRoutines 構造体に登録しており、実行エンジンから呼び出される。
  • **呼び出し先**: hstoreCheckKeyLen, hstoreCheckValLen, hstorePairs, DatumGetHStoreP, HS_COUNT, HS_SETCOUNT, HS_ADDITEM, HS_COPYITEM, HS_FINALIZE 等の hstore 内部操作関数。

関連データ構造

  • HStore: 操作対象となる hstore データ構造。
  • SubscriptingRefState: 添字演算の状態を保持する構造体。
  • Pairs: キーと値のペアを一時的に保持する構造体。

失敗モードと境界条件

  • **添字が NULL**: ERRCODE_NULL_VALUE_NOT_ALLOWED エラーを発生させる。
  • **メモリ不足**: palloc によるメモリ確保時に失敗する可能性がある。

保守担当者向けメモ

  • 本関数は hstore の不変性を維持するため、既存の hstore を直接変更せず、常に新しい hstore を再構築する。
  • 頻繁な更新操作はメモリ割り当てコストが高くなるため、パフォーマンス上の注意が必要である。
