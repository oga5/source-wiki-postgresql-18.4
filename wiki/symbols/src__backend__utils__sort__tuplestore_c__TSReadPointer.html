TSReadPointer

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/sort/tuplestore.c:91

目的

Tuplestorestate 内で、タプル読み取りの現在位置および状態を個別に管理するための構造体です。複数の読み取りポインタをサポートすることで、カーソル操作やマーク/リストア機能を可能にします。

シグネチャまたは形状

typedef struct {
    int         eflags;         /* capability flags */
    bool        eof_reached;    /* read has reached EOF */
    int         current;        /* next array index to read */
    int         file;           /* temp file# */
    off_t       offset;         /* byte offset in file */
} TSReadPointer;

入力と出力

本構造体はデータ構造であり、関数ではありません。各フィールドは tuplestore_gettupleslot 等の読み取り関数によって、メモリ上のインデックス（current）またはディスク上の位置（file, offset）として参照・更新されます。

前提条件と不変条件

  • Tuplestorestate 内の readptrs 配列の一部として管理されます。
  • eof_reached は、読み取り操作がデータの末尾に到達したことを示し、並行する書き込み操作との同期効率化に利用されます。

アルゴリズムまたは振る舞い

  • **メモリ読み取り時**: current フィールドを使用して、メモリ内のタプル配列の次読み取り位置を追跡します。
  • **ディスク読み取り時**: file および offset を使用して、BufFile 上の物理的な読み取り位置を保持します。
  • **状態遷移**: tuplestore_gettupleslot 実行時に、現在の状態（メモリかディスクか）に応じてこれらのフィールドが更新され、必要に応じて BufFileSeek が呼び出されます。

呼び出し元と呼び出し先

  • **呼び出し元**: tuplestore_begin_heap（初期化）、tuplestore_alloc_read_pointer（動的確保）、tuplestore_gettupleslot（読み取り）、tuplestore_select_read_pointer（アクティブポインタの切り替え）等から参照・操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • Tuplestorestate: 本構造体を配列として保持する親構造体。
  • TupStoreStatus: Tuplestorestate の状態（メモリ/ディスク）を定義し、TSReadPointer がどの領域を参照すべきかを決定します。

失敗モードと境界条件

  • readptrs 配列のサイズを超えてアクセスしないよう、Tuplestorestate 側の管理が必要です。
  • eof_reached が true の状態で読み取りを継続した場合の挙動は、tuplestore_gettupleslot の実装に依存します。

保守担当者向けメモ

  • eflags は読み取りポインタの機能を制御するフラグであり、特定の読み取り要件（逆方向スキャンなど）をサポートするために使用されます。
  • 複数の読み取りポインタが存在する場合、tuplestore_trim を実行する際は、すべてのポインタが参照している位置を考慮する必要があります。
