PGEventConnReset

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/libpq-events.h:42

目的

libpq のイベントシステムにおいて、接続のリセットイベントが発生した際に、対象となる PGconn オブジェクトをイベントハンドラへ伝達するための構造体です。

シグネチャまたは形状

typedef struct
{
	PGconn	   *conn;
} PGEventConnReset;

入力と出力

  • **入力**: イベント発生時に libpq 内部から PGconn ポインタが本構造体の conn メンバに格納されます。
  • **出力**: 本構造体自体はイベントハンドラ（PGEventProc）の引数として渡され、ハンドラ内での接続状態の確認やカスタムデータの管理に使用されます。

前提条件と不変条件

  • conn メンバは有効な PGconn オブジェクトを指している必要があります。
  • 本構造体は、イベント ID が PGEVT_CONNRESET であるイベントが発生した際に使用されることが想定されています。

アルゴリズムまたは振る舞い

1. libpq 内部で接続のリセット処理が発生します。 2. 登録済みのイベントハンドラが呼び出される際、イベント固有のデータとして PGEventConnReset 構造体へのポインタが渡されます。 3. イベントハンドラは、この構造体を通じてリセット対象の接続を特定し、必要に応じて関連付けられたカスタムデータの再初期化やクリーンアップを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-connect.c 内の接続リセット処理箇所から参照されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しを直接行うものではありません。

関連データ構造

  • PGconn: 接続状態を保持する libpq の主要構造体。
  • PGEventId: イベントの種類を識別する列挙型。
  • PGEventProc: 本構造体を受け取るイベントハンドラの関数ポインタ型。

失敗モードと境界条件

  • conn が NULL の場合、ハンドラ側で適切なエラーハンドリングが必要です。
  • イベントハンドラ内で PQsetInstanceData 等を用いてカスタムデータを操作する際、メモリ管理の不備があるとリークの原因となります。

保守担当者向けメモ

  • 本構造体は libpq-events.h で定義されており、libpq-events.c と連携して動作します。
  • 接続リセット時のイベント処理は、接続の再確立に伴う状態の整合性を保つために重要です。変更を加える際は、既存のイベントハンドラへの影響を考慮してください。
