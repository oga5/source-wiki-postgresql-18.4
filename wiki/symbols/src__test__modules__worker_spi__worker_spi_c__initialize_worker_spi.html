initialize_worker_spi

定義

  • 種別: function
  • 定義箇所: src/test/modules/worker_spi/worker_spi.c:68

目的

バックグラウンドワーカーの実行に必要なスキーマおよびテーブル構造の存在を確認し、存在しない場合にこれらを構築する初期化処理を行う。

シグネチャまたは形状

static void initialize_worker_spi(worktable *table)

入力と出力

  • **入力**: worktable *table - スキーマ名およびテーブル名を含む構造体へのポインタ。
  • **出力**: なし（副作用としてデータベース内にスキーマとテーブルを作成する）。

前提条件と不変条件

  • 本関数はトランザクションの外部から呼び出されることを想定している（内部で StartTransactionCommand を実行するため）。
  • SPI_connect を呼び出すため、適切なバックグラウンドワーカーの接続状態が必要である。
  • worktable 構造体は有効なスキーマ名とテーブル名を含んでいる必要がある。

アルゴリズムまたは振る舞い

1. トランザクションを開始し、SPI接続を確立してアクティブなスナップショットをプッシュする。 2. pgstat_report_activity を使用して状態を更新する。 3. 指定されたスキーマが pg_namespace に存在するかを SPI_execute で確認する。 4. スキーマが存在しない場合:

  • CREATE SCHEMA および CREATE TABLE（type 列と value 列を持つ）を実行する。
  • type が 'total' である行を対象としたユニークインデックスを作成する。

    • 5. SPI接続を終了し、スナップショットをポップしてトランザクションをコミットする。 6. 最後に pgstat_report_activity を呼び出し、状態を STATE_IDLE に戻す。

呼び出し元と呼び出し先

  • **呼び出し元**: worker_spi_main（src/test/modules/worker_spi/worker_spi.c:174）。
  • **呼び出し先**: SetCurrentStatementStartTimestamp, StartTransactionCommand, SPI_connect, PushActiveSnapshot, GetTransactionSnapshot, pgstat_report_activity, initStringInfo, appendStringInfo, SPI_execute, elog, SPI_getbinval, DatumGetInt64, resetStringInfo, SPI_finish, PopActiveSnapshot, CommitTransactionCommand。

関連データ構造

  • worktable: スキーマ名とテーブル名を保持する構造体。
  • StringInfoData: SQLクエリ文字列を構築するためのバッファ。

失敗モードと境界条件

  • SPI_execute が SPI_OK_SELECT 以外の値を返した場合、FATAL エラーで停止する。
  • クエリ結果が単一の結果セットでない場合、または結果が NULL の場合、FATAL エラーで停止する。
  • スキーマ作成時の SPI_execute が SPI_OK_UTILITY 以外を返した場合、FATAL エラーで停止する。

保守担当者向けメモ

  • 本関数は debug_query_string を一時的に書き換えるため、実行中に他のプロセスから現在のクエリ状態が参照される可能性がある。
  • CREATE SCHEMA 実行時に IF NOT EXISTS を使用する代替案がコメントで示唆されているが、現状は pg_namespace のカウントによる判定を行っている。
