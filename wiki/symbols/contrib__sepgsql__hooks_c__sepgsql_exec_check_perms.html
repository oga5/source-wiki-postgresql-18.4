sepgsql_exec_check_perms

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/hooks.c:292

目的

DML操作（SELECT/INSERT等）実行時の権限チェックを行うためのフック関数です。PostgreSQLコアの ExecutorCheckPerms_hook に登録され、SE-PostgreSQLのセキュリティポリシーに基づいたアクセス制御を適用します。

シグネチャまたは形状

static bool sepgsql_exec_check_perms(List *rangeTbls, List *rteperminfos, bool abort)

入力と出力

  • **入力**:
  • rangeTbls: 範囲テーブルのリスト。
  • rteperminfos: RTE（RangeTblEntry）の権限情報リスト。
  • abort: 権限チェック失敗時にエラーを発生させるかどうかを示すフラグ。
  • **出力**:
  • bool: 権限チェックが許可された場合は true、拒否された場合は false を返します。

前提条件と不変条件

  • 他のセキュリティモジュールがフックを登録している場合、それらのチェックが優先されます。
  • next_exec_check_perms_hook が存在する場合、その結果が false であれば、本関数は即座に false を返します。

アルゴリズムまたは振る舞い

1. **フックの連鎖**: next_exec_check_perms_hook が設定されている場合、まずそれを呼び出します。その戻り値が false の場合、処理を中断し false を返します。 2. **SE-PostgreSQL権限チェック**: sepgsql_dml_privileges を呼び出し、DML操作に対する権限を検証します。 3. **結果の返却**: sepgsql_dml_privileges の結果をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLコアのExecutor（ExecutorCheckPerms_hook を介して呼び出されます）。
  • **呼び出し先**:
  • next_exec_check_perms_hook（存在する場合）
  • sepgsql_dml_privileges（contrib/sepgsql/dml.c 内の関数）

関連データ構造

  • List (PostgreSQLコアのリスト構造)
  • RangeTblEntry (RTE)

失敗モードと境界条件

  • 権限チェックで拒否された場合、abort 引数の値に応じてエラーが発生するか、単に false が返されます。
  • フックの連鎖において、先行するフックが false を返した場合、本関数のロジックは実行されません。

保守担当者向けメモ

  • 本関数は ExecutorCheckPerms_hook に代入されることで機能します。他のセキュリティ拡張との共存のため、必ず next_exec_check_perms_hook を経由する連鎖構造を維持してください。
