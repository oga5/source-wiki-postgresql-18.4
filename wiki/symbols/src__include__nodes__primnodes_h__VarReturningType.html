VarReturningType

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/primnodes.h:254

目的

クエリの RETURNING 句において、どの行データ（OLD または NEW）を参照すべきかを指定するために使用されます。

シグネチャまたは形状

typedef enum VarReturningType
{
	VAR_RETURNING_DEFAULT,
	VAR_RETURNING_OLD,
	VAR_RETURNING_NEW,
} VarReturningType;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: VAR_RETURNING_DEFAULT、VAR_RETURNING_OLD、VAR_RETURNING_NEW のいずれかの値。

前提条件と不変条件

  • VAR_RETURNING_DEFAULT: DELETE 操作時は OLD を返し、それ以外（INSERT/UPDATE）は NEW を返します。
  • VAR_RETURNING_OLD: DELETE または UPDATE 操作時は OLD を返し、それ以外は NULL を返します。
  • VAR_RETURNING_NEW: INSERT または UPDATE 操作時は NEW を返し、それ以外は NULL を返します。

アルゴリズムまたは振る舞い

この列挙型は、クエリ解析および書き換えの過程で、RETURNING 句に含まれる変数がどのタプルソースを参照すべきかを決定するフラグとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/parser/analyze.c
  • src/backend/parser/parse_relation.c
  • src/backend/rewrite/rewriteManip.c
  • src/include/executor/execExpr.h
  • src/include/parser/parse_node.h
  • **呼び出し先**:
  • このシンボル自体は列挙型定義であり、関数ではないため、直接的な呼び出し先はありません。

関連データ構造

  • Var (同ファイル内で定義): クエリ内の変数参照を表現する構造体。
  • ParseState (関連ヘッダーで定義): 解析状態を保持する構造体。

失敗モードと境界条件

  • 列挙型として定義されているため、定義外の値を代入した場合の動作は未定義です。
  • 操作の種類（INSERT/UPDATE/DELETE）とこのフラグの組み合わせが不適切な場合、実行時に意図しない NULL や誤ったタプルが返される可能性があります。

保守担当者向けメモ

  • この列挙型は、主に RETURNING 句の処理における OLD/NEW の解決ロジックを制御します。変更を加える際は、src/backend/rewrite/rewriteManip.c 等の書き換えロジックへの影響を確認してください。
