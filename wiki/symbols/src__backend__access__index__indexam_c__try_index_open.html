try_index_open

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/indexam.c:151

目的

指定された OID に基づいてインデックスリレーションをオープンします。対象のインデックスが存在しない場合はエラーを発生させずに NULL を返します。

シグネチャまたは形状

Relation try_index_open(Oid relationId, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relationId (Oid): オープン対象のインデックスの OID。
  • lockmode (LOCKMODE): リレーション取得時に適用するロックモード。
  • **出力**:
  • 成功時: オープンされた Relation 構造体へのポインタ。
  • 失敗時: インデックスが存在しない場合は NULL。

前提条件と不変条件

  • try_relation_open が正常にリレーションを返した場合、validate_relation_kind(r) が呼び出され、対象がインデックスリレーションであることを検証します。

アルゴリズムまたは振る舞い

1. try_relation_open(relationId, lockmode) を呼び出し、リレーションの取得を試みます。 2. リレーションが存在しない（NULL が返された）場合は、即座に NULL を返して終了します。 3. リレーションが存在する場合、validate_relation_kind(r) を実行してインデックスとしての妥当性を確認します。 4. 検証を通過した Relation ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/index.c:3677
  • **呼び出し先**:
  • try_relation_open
  • validate_relation_kind

関連データ構造

  • Relation: オープンされたリレーション情報を保持する構造体。

失敗モードと境界条件

  • 指定された relationId が存在しない場合、try_relation_open の挙動に従い NULL を返します。
  • validate_relation_kind がインデックス以外のリレーション種別を検出した場合、内部でエラーが発生し処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は index_open と異なり、存在しない場合に例外を投げない安全なラッパーとして機能します。
  • インデックス以外のリレーションを誤って開こうとした場合の検証は validate_relation_kind に依存しています。
