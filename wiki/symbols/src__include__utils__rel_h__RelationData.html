RelationData

定義

  • 種別: struct
  • 定義箇所: src/include/utils/rel.h:55

目的

PostgreSQLのリレーションキャッシュ（RelCache）において、テーブル、インデックス、ビューなどの物理的・論理的属性をメモリ上で管理するための中心的なデータ構造です。

シグネチャまたは形状

typedef struct RelationData {
    RelFileLocator rd_locator;
    SMgrRelation   rd_smgr;
    int            rd_refcnt;
    /* ... 多数のメタデータフィールドおよびポインタ ... */
} RelationData;

入力と出力

  • **入力**: リレーションのOIDや物理識別子に基づき、relcache.c によって構築されます。
  • **出力**: リレーションのメタデータ（タプル記述子、インデックスリスト、パーティション情報、アクセス権限等）へのアクセスを提供します。

前提条件と不変条件

  • rd_refcnt はリレーションの生存期間を管理します。直接操作せず、専用の関数（RelationIncrementReferenceCount 等）を介する必要があります。
  • rd_options は bytea として格納されており、直接アクセスは避け、提供されているマクロ（RelationGetFillFactor 等）を使用してください。
  • 特定のフィールド（rd_indexlist や rd_partkey 等）は遅延評価されるため、アクセス時には対応する RelationGet... 関数を使用する必要があります。

アルゴリズムまたは振る舞い

  • **遅延初期化**: RelationGetSmgr() が呼び出された際に、必要に応じてストレージマネージャ（smgropen）が初期化されます。
  • **無効化**: リレーションキャッシュの無効化イベントが発生すると、rd_smgr が閉じられ、キャッシュデータがリセットされます。
  • **状態管理**: rd_isvalid 等のフラグにより、キャッシュエントリの有効性が制御されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/relcache.c が主にこの構造体のメモリ確保および構築を行います。また、xlogutils.c 等で参照されます。
  • **呼び出し先**: 物理ストレージ操作のために storage/smgr.h の関数群を呼び出します。

関連データ構造

  • Relation: RelationData へのポインタ型。
  • LockInfoData: ロックマネージャ用情報。
  • TupleDesc: タプル記述子。
  • SMgrRelation: ストレージマネージャのファイルハンドル。
  • StdRdOptions: ヒープリレーションのオプション構造体。

失敗モードと境界条件

  • rd_smgr が NULL の場合、ストレージマネージャが未初期化であることを示します。
  • キャッシュ無効化メッセージにより、rd_amcache 等のキャッシュデータがリセットされる可能性があります。
  • rd_newRelfilelocatorSubid は、単一トランザクション内で複数の TRUNCATE 等が発生し、サブトランザクションがロールバックされた場合に情報が失われる可能性があります。

保守担当者向けメモ

  • 非常に巨大な構造体であるため、変更時はメモリレイアウトとキャッシュの整合性に注意してください。
  • rd_amcache はリセット可能なキャッシュ領域であるため、永続的な状態保持には使用しないでください。
  • rd_rel フィールドから直接 rd_options を読み取ることはできません。必ず rd_options フィールドを参照してください。
