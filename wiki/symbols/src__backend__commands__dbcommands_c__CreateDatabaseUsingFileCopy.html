CreateDatabaseUsingFileCopy

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dbcommands.c:550

目的

テンプレートデータベースの物理ファイルをコピーすることで、新しいデータベースを構築する。

シグネチャまたは形状

static void CreateDatabaseUsingFileCopy(Oid src_dboid, Oid dst_dboid, Oid src_tsid, Oid dst_tsid)

入力と出力

  • **入力**:
  • src_dboid: コピー元データベースのOID
  • dst_dboid: 作成するデータベースのOID
  • src_tsid: コピー元テーブルスペースのOID
  • dst_tsid: コピー先テーブルスペースのOID
  • **出力**: なし（副作用として物理ファイルシステム上にデータベースディレクトリを作成）

前提条件と不変条件

  • IsBinaryUpgrade が真でない場合、処理の開始前および終了前に RequestCheckpoint を実行し、ディスク上の整合性を確保する。
  • pg_tablespace カタログをスキャンし、GLOBALTABLESPACE_OID を除くすべてのテーブルスペースディレクトリを対象とする。

アルゴリズムまたは振る舞い

1. **事前チェックポイント**: IsBinaryUpgrade でない場合、全バッファをフラッシュし、保留中の削除要求を処理する。 2. **テーブルスペース走査**: pg_tablespace をスキャンし、各テーブルスペースのデータベースディレクトリを特定する。 3. **ディレクトリコピー**: copydir() を使用して、ソースからターゲットへディレクトリを再帰的にコピーする。 4. **WAL記録**: XLOG_DBASE_CREATE_FILE_COPY レコードを挿入し、ファイルシステム操作をWALに記録する。 5. **事後チェックポイント**: IsBinaryUpgrade でない場合、コミット前に再度チェックポイントを実行し、リカバリ時の不整合（未ログのファイル作成やPITR時の競合）を防止する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dbcommands.c 内のデータベース作成ロジックから呼び出される。
  • **呼び出し先**: RequestCheckpoint, table_open, table_beginscan_catalog, heap_getnext, GetDatabasePath, copydir, XLogBeginInsert, XLogRegisterData, XLogInsert 等。

関連データ構造

  • xl_dbase_create_file_copy_rec: WAL記録に使用される構造体。
  • Form_pg_tablespace: pg_tablespace カタログのタプル構造。

失敗モードと境界条件

  • stat() でディレクトリが確認できない、または空の場合はスキップされる。
  • copydir() の失敗はエラーとして伝播する。
  • IsBinaryUpgrade モード時はチェックポイントをスキップする最適化が行われる。

保守担当者向けメモ

  • 本関数は CreateDatabaseUsingWalLog() と比較して、ファイルシステムレベルのコピーを行う「安直な」戦略である。
  • 過去のリリース（8.0.3以前）で発生したリカバリ時の不整合を防ぐため、コミット前のチェックポイントが必須となっている。
  • copydir() はサブディレクトリの再帰的コピーを行わない仕様である点に注意すること。
