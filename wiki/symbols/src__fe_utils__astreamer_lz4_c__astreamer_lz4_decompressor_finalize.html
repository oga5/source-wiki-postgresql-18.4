astreamer_lz4_decompressor_finalize

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_lz4.c:389

目的

LZ4伸長ストリーマーの終了処理を行い、バッファ内に残存するデータを後続のストリーマーへ転送してパイプラインを完結させる。

シグネチャまたは形状

static void astreamer_lz4_decompressor_finalize(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer - 終了対象となるLZ4伸長ストリーマーのインスタンス。
  • **出力**: なし（void）。

前提条件と不変条件

  • streamer は astreamer_lz4_frame 構造体としてキャスト可能であること。
  • streamer->bbs_next が有効な後続ストリーマーを指していること。

アルゴリズムまたは振る舞い

1. 引数の streamer を astreamer_lz4_frame 型へキャストする。 2. mystreamer->bytes_written が 0 より大きい場合、出力バッファ (mystreamer->base.bbs_buffer.data) に残存しているデータを astreamer_content を介して後続のストリーマーへ転送する。 3. 後続ストリーマーの finalize メソッドを astreamer_finalize 経由で呼び出し、パイプラインの終了処理を伝播させる。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer インターフェースの finalize 関数ポインタを通じて呼び出される（直接的な呼び出し元はソースコード上では明示されていない）。
  • **呼び出し先**:
  • astreamer_content（後続ストリーマーへのデータ転送）
  • astreamer_finalize（後続ストリーマーの終了処理）

関連データ構造

  • astreamer_lz4_frame: 伸長処理の内部状態を保持する構造体。
  • astreamer: ストリーマーの基底インターフェース。

失敗モードと境界条件

  • bytes_written が 0 の場合、バッファのフラッシュ処理はスキップされる。
  • 後続のストリーマーが NULL である場合、astreamer_content および astreamer_finalize の呼び出しにより未定義動作が発生する可能性がある（ソースコード上でのガードは確認できない）。

保守担当者向けメモ

  • 本関数は astreamer フレームワークの終了シーケンスの一部である。
  • USE_LZ4 が定義されている環境でのみコンパイルされる。
  • バッファのフラッシュ後に後続ストリーマーの finalize を呼ぶ順序は、パイプラインの整合性を保つために重要である。
