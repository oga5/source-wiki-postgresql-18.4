RemoveStatisticsDataById

定義

  • 種別: function
  • 定義箇所: src/backend/commands/statscmds.c:760

目的

拡張統計情報に関連付けられたデータ行をシステムカタログ pg_statistic_ext_data から削除します。

シグネチャまたは形状

void RemoveStatisticsDataById(Oid statsOid, bool inh)

入力と出力

  • **入力**:
  • statsOid: 削除対象の統計オブジェクトのOID。
  • inh: 継承された統計データを含めるかどうかを示すフラグ。
  • **出力**: なし（void）。

前提条件と不変条件

  • StatisticExtDataRelationId（pg_statistic_ext_data）に対する RowExclusiveLock を取得して操作を行います。
  • 統計データが存在しない場合でもエラーにはならず、安全に終了します。

アルゴリズムまたは振る舞い

1. table_open を使用して pg_statistic_ext_data を RowExclusiveLock モードで開きます。 2. SearchSysCache2 を用いて、指定された statsOid と inh フラグに一致するタプルを検索します。 3. タプルが見つかった場合（HeapTupleIsValid が真）、CatalogTupleDelete を呼び出して該当行を削除します。 4. 検索に使用したシステムキャッシュを ReleaseSysCache で解放します。 5. table_close を呼び出してリレーションを閉じます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/statscmds.c 内の統計削除処理（RemoveStatisticsById 等から呼び出される）。
  • src/backend/statistics/extended_stats.c:826。
  • **呼び出し先**:
  • table_open, table_close
  • SearchSysCache2, ReleaseSysCache
  • CatalogTupleDelete

関連データ構造

  • pg_statistic_ext_data: 拡張統計情報の実際のデータ値を保持するカタログ。
  • STATEXTDATASTXOID: pg_statistic_ext_data を検索するためのシステムキャッシュID。

失敗モードと境界条件

  • 指定された statsOid に対応するデータが存在しない場合、SearchSysCache2 は無効なタプルを返し、削除処理はスキップされます。

保守担当者向けメモ

  • 本関数は統計情報の「データ」部分のみを削除します。統計情報の定義そのもの（pg_statistic_ext）の削除は、呼び出し元である RemoveStatisticsById 等で別途管理されています。
