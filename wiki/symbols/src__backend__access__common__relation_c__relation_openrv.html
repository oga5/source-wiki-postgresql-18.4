relation_openrv

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/relation.c:136

目的

RangeVar（名前）で指定されたリレーションを特定し、適切なロックを取得した上で、リレーション記述子（Relation）を開いて返します。

シグネチャまたは形状

Relation relation_openrv(const RangeVar *relation, LOCKMODE lockmode);

入力と出力

  • **入力**:
  • relation: 開く対象のリレーションを指定する RangeVar 構造体。
  • lockmode: 取得するロックモード（LOCKMODE）。
  • **出力**:
  • 成功時: 開かれたリレーションの記述子（Relation）。
  • 失敗時: リレーションが見つからない場合、RangeVarGetRelid によりエラーが発生します。

前提条件と不変条件

  • lockmode が NoLock でない場合、リレーションキャッシュの整合性を保つために AcceptInvalidationMessages() が呼び出されます。
  • RangeVarGetRelid の第3引数に false が渡されているため、リレーションが存在しない場合はエラーとなります。

アルゴリズムまたは振る舞い

1. **無効化メッセージの処理**: lockmode が NoLock でない場合、AcceptInvalidationMessages() を呼び出し、最新のカタログ情報（ACL等）を反映させます。 2. **名前解決とロック**: RangeVarGetRelid を使用して、RangeVar からリレーションの OID を取得し、指定された lockmode でロックを取得します。 3. **リレーションのオープン**: 取得した OID を用いて relation_open(relOid, NoLock) を呼び出し、リレーション記述子を取得して返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • AcceptInvalidationMessages()
  • RangeVarGetRelid()
  • relation_open()
  • **呼び出し元**:
  • contrib/amcheck/verify_common.c
  • contrib/pageinspect/btreefuncs.c
  • contrib/pageinspect/rawpage.c
  • contrib/pgrowlocks/pgrowlocks.c
  • contrib/pgstattuple/pgstatindex.c
  • contrib/pgstattuple/pgstattuple.c

関連データ構造

  • RangeVar: リレーションの名前やスキーマ修飾情報を保持する構造体。
  • Relation: リレーションのメタデータや状態を保持する記述子。
  • LOCKMODE: ロックの種類を定義する列挙型。

失敗モードと境界条件

  • 指定された RangeVar に該当するリレーションが存在しない場合、RangeVarGetRelid がエラーを発生させます。
  • lockmode が NoLock の場合、無効化メッセージの確認はスキップされます。

保守担当者向けメモ

  • 本関数は relation_openrv_extended と異なり、missing_ok フラグを false に固定して呼び出します。
  • NoLock 指定時の無効化メッセージスキップは歴史的な経緯によるものであり、将来的な設計変更の余地があります。
