EventTriggerCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/evtcache.c:40

目的

イベントトリガーのキャッシュ用ハッシュテーブルにおいて、特定のイベントに関連付けられたトリガー関数のリストを保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	EventTriggerEvent event;
	List	   *triggerlist;
} EventTriggerCacheEntry;

入力と出力

  • **入力**: キャッシュ構築時に pg_event_trigger カタログから抽出されたイベント種別とトリガーリスト。
  • **出力**: EventCacheLookup を通じて、特定のイベントに対応する triggerlist が呼び出し元へ提供されます。

前提条件と不変条件

  • キャッシュは EventTriggerCacheContext という専用のメモリコンテキストで管理されます。
  • triggerlist はキャッシュの再構築時に更新されるため、呼び出し側で長期間保持する場合はコピーが必要です。

アルゴリズムまたは振る舞い

  • ハッシュテーブルの各エントリとして、イベント種別をキーに、そのイベントで実行すべきトリガーのリストを保持します。
  • キャッシュの再構築（BuildEventTriggerCache）時に、カタログから読み込まれた情報に基づいてこの構造体がインスタンス化されます。

呼び出し元と呼び出し先

  • **呼び出し元**: BuildEventTriggerCache（キャッシュ構築時）、EventCacheLookup（キャッシュ検索時）。
  • **呼び出し先**: 直接的な関数呼び出しは行いません。データ構造として List 型を保持します。

関連データ構造

  • EventTriggerEvent: キーとして使用されるイベント種別の列挙型。
  • List: triggerlist フィールドが保持する、トリガー情報を含むリスト。

失敗モードと境界条件

  • キャッシュが無効化された場合、本構造体を含むハッシュテーブル全体が破棄されます。
  • メモリコンテキストのリセットにより、保持している triggerlist へのポインタが無効になる可能性があります。

保守担当者向けメモ

  • 本構造体は src/backend/utils/cache/evtcache.c 内のハッシュテーブル管理（ctl.entrysize）においてサイズが参照されています。
  • 構造体の変更時は、ハッシュテーブルの初期化ロジックと整合性が取れているか確認してください。
