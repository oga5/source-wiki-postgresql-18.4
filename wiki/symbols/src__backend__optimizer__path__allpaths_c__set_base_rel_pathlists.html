set_base_rel_pathlists

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/allpaths.c:332

目的

クエリ内のすべてのベースリレーション（RELOPT_BASEREL）に対して、スキャンパスの生成をトリガーし、各リレーションのパスリストを構築する。

シグネチャまたは形状

static void set_base_rel_pathlists(PlannerInfo *root)

入力と出力

  • **入力**: PlannerInfo *root（クエリ全体のプランナ情報）。
  • **出力**: なし（副作用として、root->simple_rel_array 内の各 RelOptInfo にパスが格納される）。

前提条件と不変条件

  • root->simple_rel_array が初期化されていること。
  • 各 RelOptInfo の relid と配列インデックス rti が一致していること（Assert による検証）。

アルゴリズムまたは振る舞い

1. root->simple_rel_array_size を上限として、rti を 1 から順に走査する。 2. rel が NULL の場合はスキップする（ベースリレーション以外の RTE 用スロット）。 3. rel->reloptkind が RELOPT_BASEREL であるもののみを対象とする。 4. 対象リレーションごとに set_rel_pathlist を呼び出し、具体的なパス生成を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/allpaths.c:221（make_one_rel 内から呼び出される）。
  • **呼び出し先**: set_rel_pathlist。

関連データ構造

  • PlannerInfo: クエリ全体の最適化状態を保持する構造体。
  • RelOptInfo: 個別のリレーションのパスやコスト情報を保持する構造体。

失敗モードと境界条件

  • rel == NULL のスロットは安全に無視される。
  • RELOPT_BASEREL 以外のリレーション（結合リレーションやその他）は処理対象外としてスキップされる。

保守担当者向けメモ

  • 本関数はベースリレーションのパス生成フェーズの起点です。各リレーションの具体的なパス生成ロジックは set_rel_pathlist に委譲されています。
  • root->simple_rel_array の走査順序は rti（RangeTableIndex）に依存します。
