CopySource

定義

  • 種別: enum
  • 定義箇所: src/include/commands/copyfrom_internal.h:25

目的

COPY FROM コマンドにおいて、データの読み込み元（ソース）を識別するために使用されます。

シグネチャまたは形状

typedef enum CopySource
{
	COPY_FILE,
	COPY_FRONTEND,
	COPY_CALLBACK,
} CopySource;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: CopyFromStateData 構造体内の copy_src フィールドとして保持されます。

前提条件と不変条件

  • COPY FROM 実行時に、いずれか一つのソースタイプが必ず選択される必要があります。

アルゴリズムまたは振る舞い

  • COPY_FILE: ファイルまたはパイプ経由のプログラムからデータを読み込みます。
  • COPY_FRONTEND: クライアント（フロントエンド）からデータを読み込みます。
  • COPY_CALLBACK: 登録されたコールバック関数を通じてデータを読み込みます。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/commands/copyfrom_internal.h:65 にて CopyFromStateData 構造体のメンバとして参照されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数呼び出し関係は特定できません。

関連データ構造

  • CopyFromStateData: COPY FROM の実行状態を保持する構造体であり、本列挙型をメンバとして含みます。

失敗モードと境界条件

  • 該当なし（列挙型定義のため）。

保守担当者向けメモ

  • COPY サブシステムのデータソース拡張を行う際は、この列挙型に新しいソースタイプを追加し、それに対応する読み込みロジックを copyfrom.c 等に実装する必要があります。
