io_callback_fn

定義

  • 種別: typedef
  • 定義箇所: src/include/common/blkreftable.h:46

目的

BlockRefTable のシリアライズ（書き出し）およびデシリアライズ（読み込み）処理において、外部ストレージとの間でデータ転送を行うための汎用的なコールバックインターフェースを提供します。

シグネチャまたは形状

typedef int (*io_callback_fn) (void *callback_arg, void *data, int length);

入力と出力

  • **入力**:
  • callback_arg: コールバック関数に渡されるコンテキスト情報（呼び出し元で定義）。
  • data: 読み込みまたは書き込み対象のデータバッファ。
  • length: 転送するデータのバイト数。
  • **出力**:
  • 戻り値: 実際に読み込みまたは書き込みが行われたバイト数を返します。

前提条件と不変条件

  • 戻り値は、要求されたバイト数（length）と一致するか、エラー発生時にはそれ以外の値となることが期待されます。
  • BlockRefTableReader および BlockRefTableWriter の初期化時に、この関数ポインタを介して具体的な I/O 実装が注入されます。

アルゴリズムまたは振る舞い

この関数は、BlockRefTable のストリーミング処理中に呼び出されます。

  • 書き込み時: BlockRefTableWriter を通じて、メモリ上の変更ブロック情報を外部ストレージへ出力するために使用されます。
  • 読み込み時: BlockRefTableReader を通じて、外部ストレージからデータを読み込み、BlockRefTable を再構築するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/blkreftable.c 内の CreateBlockRefTableReader および CreateBlockRefTableWriter で設定され、内部の I/O 処理から呼び出されます。
  • **呼び出し先**:
  • 提供されたソースコード内には、この関数ポインタが指す具体的な実装関数は定義されていません。呼び出し元で提供される具体的な I/O 関数が実行されます。

関連データ構造

  • BlockRefTableReader: 読み込みコンテキスト。
  • BlockRefTableWriter: 書き込みコンテキスト。

失敗モードと境界条件

  • 戻り値が length と一致しない場合、I/O エラーまたは予期せぬ終端（EOF）として扱われます。エラーハンドリングの詳細は、このコールバックを呼び出す側のロジックに依存します。

保守担当者向けメモ

  • 本シンボルは関数ポインタ型です。実装を追加する際は、BlockRefTableReader や BlockRefTableWriter の利用者が適切なエラー処理を実装していることを確認してください。
  • 読み込み処理において、戻り値が読み込んだバイト数であることを保証する必要があります。
