transformJoinUsingClause

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:305

目的

SQLの JOIN ... USING 句を解析し、結合条件となる等価演算子の式ツリーを生成する。

シグネチャまたは形状

Node *transformJoinUsingClause(ParseState *pstate, List *leftVars, List *rightVars)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態。
  • leftVars, rightVars: 結合対象となる左右のテーブルから抽出された Var ノードのリスト。
  • **出力**:
  • 結合条件を表す Node（単一の OpExpr または BoolExpr）。

前提条件と不変条件

  • leftVars と rightVars は同じ長さである必要がある。
  • 入力された Var ノードは既に変換済みである。
  • 呼び出し元は、結合される列に対して SELECT 権限のチェックを必要とする（本関数内で markVarForSelectPriv を呼び出し、権限をマークする）。

アルゴリズムまたは振る舞い

1. leftVars と rightVars を順次走査し、各ペアに対して markVarForSelectPriv を実行する。 2. 各ペアについて lvar = rvar という等価演算子（A_Expr）を生成する。 3. 複数の結合列がある場合は AND で結合し、単一の場合はそのままの式を保持する。 4. 生成した式ツリーに対し transformExpr を呼び出し、演算子の解決（型解決等）を行う。 5. 最後に coerce_to_boolean を呼び出し、結果が論理値型であることを保証する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_clause.c:1398（transformFromClauseItem 内の JoinExpr 処理）。
  • **呼び出し先**:
  • markVarForSelectPriv
  • makeSimpleA_Expr
  • makeBoolExpr
  • transformExpr
  • coerce_to_boolean

関連データ構造

  • ParseState: 解析状態を保持する構造体。
  • Var: 結合対象の列を指すノード。
  • A_Expr: 構文解析中の未解決な演算子式。

失敗モードと境界条件

  • 結合列のリストが空の場合の挙動については、ソースコード上では明示的なガード句は見当たらない。
  • transformExpr が型解決に失敗した場合、エラーがスローされる可能性がある。

保守担当者向けメモ

  • 本関数は、既に変換済みの Var を使用して未変換の演算子ツリーを構築するという特殊な手法をとっている。そのため、transformExpr 呼び出し時に既存のノードに対する警告が発生しないよう注意が必要である。
  • transformJoinOnClause と異なり、入力が既に Var であるため、複雑な名前解決のステップを省略している。
