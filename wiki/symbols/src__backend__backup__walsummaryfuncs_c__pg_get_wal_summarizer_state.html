pg_get_wal_summarizer_state

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummaryfuncs.c:176

目的

WALサマライザープロセスの現在の進捗状況（タイムライン、LSN）およびプロセスIDを取得し、SQLから参照可能な単一行のレコードとして返します。

シグネチャまたは形状

Datum pg_get_wal_summarizer_state(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL関数インターフェース経由の引数）
  • **出力**: Datum（WALサマライザーの状態を示す複合型タプル）

前提条件と不変条件

  • 呼び出し元は、戻り値の型として複合型（row type）を期待している必要があります。
  • summarizer_pid が負の値である場合、出力タプルのPIDフィールドはNULLとして扱われます。

アルゴリズムまたは振る舞い

1. GetWalSummarizerState を呼び出し、現在の summarized_tli（タイムライン）、summarized_lsn、pending_lsn、および summarizer_pid を取得します。 2. get_call_result_type を使用して、呼び出し元の期待する戻り値の型（TupleDesc）を確認します。複合型でない場合はエラーを発生させます。 3. 取得した各値を Datum 形式に変換し、values 配列に格納します。 4. summarizer_pid が負の場合は nulls 配列の該当箇所を true に設定し、それ以外の場合は値を格納します。 5. heap_form_tuple を用いてタプルを生成し、PG_RETURN_DATUM を通じて結果を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • GetWalSummarizerState（状態取得用）
  • get_call_result_type
  • heap_form_tuple
  • **呼び出し元**: SQLインターフェース経由でユーザーから呼び出されます。

関連データ構造

  • NUM_STATE_ATTS: タプルの属性数。
  • TupleDesc: 戻り値の型定義。
  • HeapTuple: 生成されるタプルデータ。

失敗モードと境界条件

  • 戻り値の型が複合型ではない場合、elog(ERROR) により処理が中断されます。
  • WALサマライザーが稼働していない場合、PIDは負の値として取得され、結果セットのPID列はNULLとなります。

保守担当者向けメモ

  • 本関数は postmaster/walsummarizer.h で定義された状態取得関数に依存しています。サマライザーの内部状態構造に変更があった場合は、本関数のフィールドマッピングを更新してください。
