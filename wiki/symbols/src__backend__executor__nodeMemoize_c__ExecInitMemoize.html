ExecInitMemoize

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeMemoize.c:951

目的

Memoize ノードの実行状態を初期化し、パラメータ化されたサブプランの結果をキャッシュするための環境を構築する。

シグネチャまたは形状

MemoizeState *ExecInitMemoize(Memoize *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の Memoize プランノード。
  • estate: 実行状態を保持する EState。
  • eflags: 実行フラグ（EXEC_FLAG_BACKWARD および EXEC_FLAG_MARK はサポート外）。
  • **出力**:
  • 初期化済みの MemoizeState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと。
  • キャッシュ用のハッシュテーブルは、メモリ割り当ての負荷を考慮し、この関数内では生成されず実行時に遅延生成される。

アルゴリズムまたは振る舞い

1. MemoizeState 構造体を割り当て、プランノードと実行状態を紐付ける。 2. ExecInitNode を呼び出し、サブプランを初期化する。 3. キャッシュ検索用のハッシュキー記述子とタプルスロット（tableslot, probeslot）を作成する。 4. パラメータ式を評価するための ExprState 配列と、ハッシュ関数情報を初期化する。 5. ExecBuildParamSetEqual を使用して、キャッシュキー比較用の等価性評価式を構築する。 6. キャッシュ専用のメモリコンテキスト（MemoizeHashTable）を作成し、LRUリストを初期化する。 7. 統計情報カウンタをゼロクリアし、ステートマシンを MEMO_CACHE_LOOKUP に設定して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内の ExecInitNode から呼び出される。
  • **呼び出し先**: ExecInitNode（サブプラン用）、ExecAssignExprContext、ExecInitExpr、ExecBuildParamSetEqual、AllocSetContextCreate 等。

関連データ構造

  • MemoizeState: ノードの実行状態を管理するメイン構造体。
  • Memoize: プランノード定義。
  • ExprState: パラメータ評価用。
  • dlist: LRUリスト管理用。

失敗モードと境界条件

  • ハッシュ演算子に対応するハッシュ関数が見つからない場合、elog(ERROR) を発生させる。
  • メモリ制限（mem_limit）は get_hash_memory_limit() から取得される。

保守担当者向けメモ

  • ハッシュテーブルの構築は実行時まで遅延されるため、hashtable フィールドは初期状態で NULL である。
  • singlerow フラグや binary_mode はプランノードから直接コピーされ、キャッシュの挙動を制御する。
