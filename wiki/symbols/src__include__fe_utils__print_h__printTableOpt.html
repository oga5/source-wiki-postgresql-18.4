printTableOpt

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/print.h:111

目的

クエリ結果の整形出力に関する設定（フォーマット、罫線、ページャ、エンコーディング等）を一元管理し、出力関数へ渡すための構造体です。

シグネチャまたは形状

typedef struct printTableOpt
{
    enum printFormat format;
    unsigned short int expanded;
    printXheaderWidthType expanded_header_width_type;
    int         expanded_header_exact_width;
    unsigned short int border;
    unsigned short int pager;
    int         pager_min_lines;
    bool        tuples_only;
    bool        start_table;
    bool        stop_table;
    bool        default_footer;
    unsigned long prior_records;
    const printTextFormat *line_style;
    struct separator fieldSep;
    struct separator recordSep;
    char        csvFieldSep[2];
    bool        numericLocale;
    char       *tableAttr;
    int         encoding;
    int         env_columns;
    int         columns;
    unicode_linestyle unicode_border_linestyle;
    unicode_linestyle unicode_column_linestyle;
    unicode_linestyle unicode_header_linestyle;
} printTableOpt;

入力と出力

本構造体は出力関数（printTable 等）への入力として使用されます。出力自体は行わず、出力処理の挙動を制御するパラメータ群を保持します。

前提条件と不変条件

  • 構造体をゼロ初期化することで、デフォルト設定が適用されます。
  • pager 設定は、出力先が標準出力であり、かつ端末（tty）である場合に有効となります。
  • fieldSep および recordSep は、主に非整列（unaligned）テキストモードで使用されます。

アルゴリズムまたは振る舞い

  • 出力フォーマット（format）に基づき、テーブルの罫線や配置を決定します。
  • expanded モードでは、レコードを垂直方向に展開して表示します（0=無効, 1=有効, 2=自動）。
  • pager 設定により、出力行数が pager_min_lines を超える場合にページャを起動するかどうかを制御します。
  • numericLocale が有効な場合、ロケールに基づいた数値フォーマット（桁区切りや小数点記号）が適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/describe.c（pset.popt.topt からのコピー等）
  • **呼び出し先**: src/fe_utils/print.c 内の各関数（printTableInit, PageOutput, get_line_style, refresh_utf8format, print_aligned_vertical_line 等）が、本構造体を参照して処理を分岐・実行します。

関連データ構造

  • printFormat: 出力形式を定義する列挙型。
  • printTextFormat: 罫線スタイルを定義する構造体。
  • separator: フィールドおよびレコードの区切り文字を保持する構造体。
  • unicode_linestyle: Unicode罫線のスタイル定義。
  • printTableContent: 出力対象のデータを保持する構造体（printTableInit で本構造体と紐付けられる）。

失敗モードと境界条件

  • env_columns が 0 の場合、環境変数 $COLUMNS が未設定であることを示します。
  • line_style が NULL の場合、デフォルトのスタイルが使用されます。
  • ページャの起動条件は出力先が tty であることに依存するため、パイプ経由の出力では無視される場合があります。

保守担当者向けメモ

  • 本構造体にフィールドを追加した際は、printTableInit や関連する初期化処理で適切にデフォルト値が設定されるか確認してください。
  • printTableOpt を変更した際は、src/fe_utils/print.c 内の参照箇所（特にページャ制御やスタイル取得ロジック）への影響を考慮してください。
