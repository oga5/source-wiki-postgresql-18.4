check_locale_monetary

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:365

目的

GUC（Grand Unified Configuration）パラメータとして設定されたロケール名が、通貨フォーマット（LC_MONETARY）としてシステム上で有効かどうかを検証します。

シグネチャまたは形状

bool check_locale_monetary(char **newval, void **extra, GucSource source);

入力と出力

  • **入力**:
  • newval: 検証対象のロケール名文字列へのポインタ。
  • extra: 未使用。
  • source: 設定のソース（GucSource）。
  • **出力**:
  • 戻り値: 指定されたロケールが有効であれば true、無効であれば false を返します。

前提条件と不変条件

  • check_locale 関数がシステム環境におけるロケール妥当性の判定を担います。
  • 本関数は GUC フックとして呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. LC_MONETARY カテゴリを指定して内部関数 check_locale を呼び出します。 2. check_locale の結果（ロケール名の妥当性）をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc_tables.c において、lc_monetary パラメータの検証用フックとして登録されています。
  • **呼び出し先**: check_locale（src/backend/utils/adt/pg_locale.c 内）。

関連データ構造

  • GucSource: 設定の由来を示す列挙型。

失敗モードと境界条件

  • 指定されたロケール名が OS でサポートされていない場合、check_locale が false を返し、本関数も false を返します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、ロケール検証の詳細は check_locale に依存しています。
  • src/include/utils/guc_hooks.h に外部宣言が存在します。
