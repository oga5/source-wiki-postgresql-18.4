clause_selectivity_ext

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/clausesel.c:683

目的

クエリプランナがコスト見積もりのために使用する、個別の式（clause）に対する選択率（Selectivity）を計算します。

シグネチャまたは形状

Selectivity clause_selectivity_ext(PlannerInfo *root,
                                   Node *clause,
                                   int varRelid,
                                   JoinType jointype,
                                   SpecialJoinInfo *sjinfo,
                                   bool use_extended_stats);

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • clause: 評価対象の式（RestrictInfo または Expr）。
  • varRelid: 対象リレーションのID（0の場合は全リレーション）。
  • jointype: 結合タイプ（JOIN_INNER 等）。
  • sjinfo: 特殊な結合情報。
  • use_extended_stats: 拡張統計の使用可否。
  • **出力**:
  • Selectivity 型（double）の推定値（0.0〜1.0）。

前提条件と不変条件

  • clause が RestrictInfo の場合、キャッシュ可能な条件（varRelid が一致するか、ベースリレーションを含まない等）であれば、RestrictInfo 内の norm_selec または outer_selec を優先的に使用します。
  • 擬似定数（pseudoconstant）かつ定数 FALSE 以外の場合は、選択率 1.0 を返します。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: RestrictInfo であれば、計算済み結果をキャッシュから取得します。 2. **式タイプ別の分岐**:

  • Var: boolvarsel を呼び出し。
  • Const: 値に基づき 0.0 または 1.0 を返却。
  • Param: estimate_expression_value で定数化を試行。
  • NOT: 再帰的に呼び出し、結果を 1.0 から減算。
  • AND: clauselist_selectivity_ext を呼び出し。
  • OR: clauselist_selectivity_or を呼び出し。
  • OpExpr / DistinctExpr: treat_as_join_clause の判定に基づき、join_selectivity または restriction_selectivity を呼び出し。DistinctExpr の場合は結果を反転。

    • 3. **デフォルト**: 未処理の式タイプに対しては 0.5 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: clauselist_selectivity（clausesel.c 内）、src/backend/statistics/extended_stats.c 等。
  • **呼び出し先**: boolvarsel, estimate_expression_value, clauselist_selectivity_ext, clauselist_selectivity_or, treat_as_join_clause, join_selectivity, restriction_selectivity 等。

関連データ構造

  • RestrictInfo: 条件式とキャッシュされた選択率を保持。
  • PlannerInfo: プランナのコンテキスト情報。

失敗モードと境界条件

  • clause が NULL の場合はデフォルト値 0.5 を返します。
  • DistinctExpr の場合、NULL 値に対する正確な挙動は保証されません。
  • Param が定数に評価できない場合、デフォルト値のまま処理されます。

保守担当者向けメモ

  • RestrictInfo のキャッシュは jointype に依存するため、JOIN_INNER かそれ以外かで別々のキャッシュ変数（norm_selec, outer_selec）を使用しています。
  • 拡張統計の適用可否は use_extended_stats 引数で制御されます。
