gist_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/gistdesc.c:60

目的

GiSTインデックスのWALレコードを解析し、人間が読める形式の文字列として StringInfo バッファに出力する。

シグネチャまたは形状

void gist_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 解析結果を書き込むための StringInfo バッファ。
  • record: 解析対象のWALレコード状態を保持する XLogReaderState 構造体。
  • **出力**: なし（buf を通じて文字列が更新される）。

前提条件と不変条件

  • record は有効なGiST WALレコードを指している必要がある。
  • XLogRecGetInfo で取得した情報ビットから XLR_INFO_MASK を除外した値が、GiSTの既知のWAL操作タイプと一致する必要がある。

アルゴリズムまたは振る舞い

1. XLogRecGetData を呼び出し、レコードのデータ部分を取得する。 2. XLogRecGetInfo を呼び出し、レコードの操作タイプ（info）を特定する。 3. switch 文により info に基づいて以下のいずれかの整形関数を呼び出す：

  • XLOG_GIST_PAGE_UPDATE -> out_gistxlogPageUpdate
  • XLOG_GIST_PAGE_REUSE -> out_gistxlogPageReuse
  • XLOG_GIST_DELETE -> out_gistxlogDelete
  • XLOG_GIST_PAGE_SPLIT -> out_gistxlogPageSplit
  • XLOG_GIST_PAGE_DELETE -> out_gistxlogPageDelete

    • 4. XLOG_GIST_ASSIGN_LSN の場合は詳細を出力せず終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h にて RM_GIST_ID リソースマネージャのメソッドとして登録されており、WAL再生やダンプ処理の過程で呼び出される。
  • **呼び出し先**: out_gistxlogPageUpdate, out_gistxlogPageReuse, out_gistxlogDelete, out_gistxlogPageSplit, out_gistxlogPageDelete。

関連データ構造

  • StringInfo: 文字列バッファ。
  • XLogReaderState: WALレコードの読み取り状態。
  • gistxlog* 各構造体（gistxlogPageUpdate 等）: GiSTのWALレコードのバイナリレイアウト。

失敗モードと境界条件

  • 未知の info ビットが渡された場合、switch 文で何も実行されず、バッファには何も書き込まれない。
  • XLOG_GIST_ASSIGN_LSN は意図的に詳細出力をスキップする仕様となっている。

保守担当者向けメモ

  • 新しいGiST WALレコードタイプを追加した際は、必ず本関数の switch 文にケースを追加すること。
  • out_gistxlogPageUpdate は現在空の実装であるため、必要に応じてデバッグ情報を追加すること。
