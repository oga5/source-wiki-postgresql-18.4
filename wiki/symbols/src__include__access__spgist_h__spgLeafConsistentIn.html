spgLeafConsistentIn

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist.h:167

目的

SP-GiSTインデックスのリーフノードにおいて、格納されたデータが検索条件（ScanKey）に合致するかを判定するための入力パラメータを保持する。

シグネチャまたは形状

typedef struct spgLeafConsistentIn
{
    ScanKey     scankeys;
    ScanKey     orderbys;
    int         nkeys;
    int         norderbys;
    Datum       reconstructedValue;
    void       *traversalValue;
    int         level;
    bool        returnData;
    Datum       leafDatum;
} spgLeafConsistentIn;

入力と出力

  • **入力**: 検索条件（scankeys, orderbys）、親ノードから引き継がれた値（reconstructedValue, traversalValue）、現在のツリーレベル、およびリーフタプル内のデータ（leafDatum）。
  • **出力**: 本構造体自体は入力専用であり、判定結果は対応する spgLeafConsistentOut 構造体に格納される。

前提条件と不変条件

  • nkeys および norderbys は、それぞれ scankeys および orderbys 配列の有効な長さを表す。
  • traversalValue は、演算子クラス固有の検索状態を保持するポインタであり、NULLの場合もある。

アルゴリズムまたは振る舞い

SP-GiSTの検索処理の最終段階で呼び出される。リーフノードに到達した際、leafDatum と scankeys を照合し、条件を満たすかを確認する。また、returnData が真の場合、インデックスから取得したデータを結果として返す必要がある。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgscan.c
  • 各演算子クラスの実装（spgquadtreeproc.c, spgtextproc.c, geo_spgist.c, network_spgist.c, rangetypes_spgist.c, spgist_name_ops.c）において、PG_GETARG_POINTER(0) を介して取得される。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • spgLeafConsistentOut: 判定結果（合致の有無、再チェック要否、戻り値）を格納する対となる構造体。
  • ScanKey: インデックス検索条件を保持する標準的なPostgreSQL構造体。

失敗モードと境界条件

  • scankeys が空（nkeys == 0）の場合、すべてのリーフデータが合致するとみなされる可能性がある。
  • traversalValue が不適切な型やNULLである場合、演算子クラスの実装側でセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本構造体はSP-GiSTの演算子クラスAPIの一部である。変更を加える際は、すべての既存の演算子クラス実装（spgquadtreeproc.c 等）との整合性を確認すること。
  • reconstructedValue は親ノードから引き継がれるため、ツリーの深さに応じた状態管理が必要な場合に利用される。
