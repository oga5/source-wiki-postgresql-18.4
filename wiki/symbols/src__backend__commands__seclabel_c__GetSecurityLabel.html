GetSecurityLabel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/seclabel.c:271

目的

指定されたデータベースオブジェクトおよびセキュリティプロバイダーに関連付けられたセキュリティラベル文字列を取得します。

シグネチャまたは形状

char *GetSecurityLabel(const ObjectAddress *object, const char *provider)

入力と出力

  • **入力**:
  • object: 対象オブジェクトを特定する ObjectAddress 構造体へのポインタ。
  • provider: セキュリティラベルプロバイダー名を示す文字列。
  • **出力**:
  • 成功時: 該当するセキュリティラベル文字列（メモリは呼び出し元で管理）。
  • 該当なし: NULL。

前提条件と不変条件

  • object が共有リレーション（IsSharedRelation）に該当する場合、本関数は内部的に GetSharedSecurityLabel を呼び出し、その結果を返します。
  • 共有リレーション以外の場合は pg_seclabel システムカタログを検索します。

アルゴリズムまたは振る舞い

1. IsSharedRelation を判定し、共有オブジェクトであれば GetSharedSecurityLabel に委譲して終了します。 2. 非共有オブジェクトの場合、pg_seclabel カタログに対して以下のキーでスキャンを実行します。

  • objoid (オブジェクトID)
  • classoid (クラスID)
  • objsubid (サブID)
  • provider (プロバイダー名)

    • 3. table_open(SecLabelRelationId, AccessShareLock) を使用してカタログを開き、systable_beginscan で検索します。 4. ヒットしたタプルが存在する場合、Anum_pg_seclabel_label 属性からラベル文字列を抽出し、TextDatumGetCString で変換して返します。 5. 最後に systable_endscan および table_close を実行し、リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/label.c:454
  • contrib/sepgsql/uavc.c:425, 455
  • **呼び出し先**:
  • GetSharedSecurityLabel (共有オブジェクトの場合)
  • table_open / table_close
  • systable_beginscan / systable_getnext / systable_endscan
  • ScanKeyInit

関連データ構造

  • ObjectAddress: 操作対象のオブジェクトを識別する構造体。
  • pg_seclabel: セキュリティラベルを保持するシステムカタログ。

失敗モードと境界条件

  • 指定されたプロバイダーやオブジェクトが存在しない場合、NULL を返します。
  • AccessShareLock を取得するため、カタログに対する同時実行制御の制約を受けます。

保守担当者向けメモ

  • 本関数はラベルの「保存場所」を管理する低レベル API です。ラベルの妥当性検証は行いません。
  • 共有カタログと通常カタログの分岐がハードコードされているため、新しいカタログタイプを追加する際は注意が必要です。
