ConditionVariable

定義

  • 種別: struct
  • 定義箇所: src/include/storage/condition_variable.h:28

目的

PostgreSQLにおけるプロセス間同期のための条件変数を提供します。特定の条件が満たされるまでプロセスを待機させ、条件が満たされた際にシグナルまたはブロードキャストによって待機中のプロセスを起床させるために使用されます。

シグネチャまたは形状

typedef struct
{
	slock_t		mutex;			/* スピンロック（待機リストを保護） */
	proclist_head wakeup;		/* 起床可能なプロセスのリスト */
} ConditionVariable;

入力と出力

本構造体はデータ保持を目的としており、関数呼び出しの入力（引数）や出力（戻り値）としては、ConditionVariableSleep や ConditionVariableSignal などの関連関数を通じて操作されます。

前提条件と不変条件

  • 共有メモリ内での配置をサポートします。
  • 待機プロセスリスト（wakeup）へのアクセスは、必ず内部の mutex（スピンロック）によって保護されなければなりません。
  • ConditionVariableSleep を呼び出す際は、必ず述語ループ（条件チェック）とセットで使用する必要があります。

アルゴリズムまたは振る舞い

1. **待機**: ConditionVariablePrepareToSleep でリストに追加し、ConditionVariableSleep でスリープ状態へ移行します。 2. **通知**: ConditionVariableSignal または ConditionVariableBroadcast が呼び出されると、スピンロックで保護された wakeup リストからプロセスが取り出され、起床されます。 3. **終了**: 待機ループを抜けた後、ConditionVariableCancelSleep を呼び出し、リストから自身を確実に除去します。

関連データ構造

  • proclist_head: 待機プロセスを管理するリスト構造。
  • slock_t: 構造体内のスピンロック実装。
  • ConditionVariableMinimallyPadded: キャッシュライン競合を避けるためのパディング付き共用体。

失敗モードと境界条件

  • 待機中に割り込み（キャンセル）が発生する可能性があります（LWLockとの相違点）。
  • 述語ループを伴わない待機は、起床シグナルの見落としやデッドロックを招く恐れがあります。

保守担当者向けメモ

  • 共有メモリ上で使用されるため、構造体のサイズやアライメントには注意が必要です。
  • 最初の条件チェックで成功する可能性が高い場合は、ConditionVariablePrepareToSleep を省略することでオーバーヘッドを削減可能です。
