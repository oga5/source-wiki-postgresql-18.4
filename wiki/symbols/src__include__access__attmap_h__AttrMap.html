AttrMap

定義

  • 種別: struct
  • 定義箇所: src/include/access/attmap.h:34

目的

異なる行型（TupleDesc）間における属性（列）の対応関係を管理し、物理的な列順序の差異や削除済み列を吸収するためのマッピング構造体です。

シグネチャまたは形状

typedef struct AttrMap
{
    AttrNumber *attnums;
    int         maplen;
} AttrMap;

入力と出力

  • attnums: 出力側の各属性に対応する入力側の属性番号（1始まり）を格納する配列。
  • maplen: マッピングの長さ（通常は出力側の属性数）。

前提条件と不変条件

  • attnums 配列のインデックスは、出力側の属性位置に対応します。
  • attnums[i] == 0 の場合、その列は削除済みであるか、対応する入力列が存在しないことを示します。

アルゴリズムまたは振る舞い

タプル変換処理において、入力タプルから出力タプルへデータをコピーする際、この構造体を参照して各列のソース位置を特定します。build_attrmap_by_position() や build_attrmap_by_name() によって構築され、列の型や名前の整合性を検証した結果が格納されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/common/attmap.c 内の関数群（構築および解放処理）が本構造体を利用しています。
  • 呼び出し先: 具体的な関数呼び出し関係は、提供されたソース excerpt には含まれていません。

関連データ構造

  • AttrNumber: access/attnum.h で定義される属性番号型。
  • TupleDesc: access/tupdesc.h で定義される行型記述子。

失敗モードと境界条件

  • attnums[i] == 0 のケースは、変換先の列が入力側に存在しない（または削除済み）という境界条件として扱われます。

保守担当者向けメモ

  • 本構造体は継承やパーティションの DDL 操作における物理的な列のズレを吸収するために不可欠です。
  • 変換処理の実装については src/include/access/tupconvert.h を参照してください。
