DomainConstraintCache

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/typcache.c:139

目的

ドメイン型に関連付けられた制約情報をキャッシュし、システムカタログへのアクセスを最小化するための構造体です。

シグネチャまたは形状

struct DomainConstraintCache
{
    List       *constraints;    /* DomainConstraintState ノードのリスト */
    MemoryContext dccContext;   /* 関連データを保持するメモリコンテキスト */
    long        dccRefCount;    /* この構造体への参照カウント */
};

入力と出力

  • 入力: ドメイン型の制約評価に必要な情報。
  • 出力: DomainConstraintState ノードのリストを含むキャッシュオブジェクト。

前提条件と不変条件

  • dccContext は、キャッシュされたデータ（constraints リスト内のノード等）の生存期間を管理するために使用されます。
  • dccRefCount は、このキャッシュオブジェクトの生存期間を制御するために使用されます。

アルゴリズムまたは振る舞い

  • lookup_type_cache 等の処理において、必要に応じて MemoryContextAlloc を用いてメモリを確保し、インスタンス化されます。
  • キャッシュされた制約情報は、ドメイン型の検証時に再利用されます。
  • 参照カウントがゼロになった際の解放処理については、ソースコード内の decr_dcc_refcount 関数が関与していることが示唆されています。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/typcache.c 内のキャッシュ生成ロジック（1177行目、1269行目付近）。
  • 呼び出し先: MemoryContextAlloc（メモリ確保）、decr_dcc_refcount（参照カウント管理）。

関連データ構造

  • DomainConstraintState: constraints リスト内に格納されるノード。
  • MemoryContext: 関連データのメモリ管理に使用。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc が失敗する可能性があります。
  • 参照カウントの不整合が発生した場合、メモリリークまたは早期解放のリスクがあります。

保守担当者向けメモ

  • 本構造体自体は式プランツリーを保持しません（プランツリーは別途管理されます）。
  • dccContext 内に構築されるノードツリーは、キャッシュのライフサイクルと同期させる必要があります。
