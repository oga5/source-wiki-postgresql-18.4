LogicalTapeRewindForRead

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/logtape.c:845

目的

論理テープを読み込み用に巻き戻し、バッファを再構成します。書き込みフェーズの終了処理（フラッシュ）や、凍結済みテープの再読み込み準備を行います。

シグネチャまたは形状

void LogicalTapeRewindForRead(LogicalTape *lt, size_t buffer_size)

入力と出力

  • lt: 操作対象の LogicalTape 構造体へのポインタ。
  • buffer_size: 読み込み用に確保するバッファサイズ（バイト単位）。
  • 出力: なし（副作用として lt の状態が更新されます）。

前提条件と不変条件

  • buffer_size は BLCKSZ の倍数に切り捨てられます。
  • テープが凍結（frozen）されている場合、buffer_size は強制的に BLCKSZ となります。
  • テープが凍結されていない場合、buffer_size は BLCKSZ 以上かつ lt->max_size 以下に調整されます。
  • 凍結されていないテープに対しては、書き込み状態（writing）から読み込み状態への遷移が許可されます。
  • 凍結済みのテープに対しては、writing が false であることが前提となります。

アルゴリズムまたは振る舞い

1. **バッファサイズの調整**: 指定された buffer_size を BLCKSZ 境界に合わせて正規化します。 2. **書き込み終了処理**: lt->writing が真の場合、未フラッシュのデータ（dirty な場合）を物理ファイルへ書き出し、writing フラグを false に設定します。 3. **状態検証**: lt->writing が偽の場合、テープが frozen であることをアサートします。 4. **メモリ解放**: 現在の lt->buffer を解放し、NULL に初期化します。 5. **リソース解放**: lt->prealloc リストが存在する場合、保持していたブロックを ltsReleaseBlock で解放し、リスト自体をメモリから解放します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/executor/nodeAgg.c
  • src/backend/utils/sort/tuplesort.c
  • 呼び出し先:
  • ltsWriteBlock (書き込みフェーズ終了時)
  • ltsReleaseBlock (プリエロケーションリスト解放時)
  • pfree
  • VALGRIND_MAKE_MEM_DEFINED

関連データ構造

  • LogicalTape: テープの状態（writing, frozen, dirty 等）を保持。
  • LogicalTapeSet: テープが属するセット。ブロック管理に使用。

失敗モードと境界条件

  • buffer_size が BLCKSZ 未満の場合、最小値として BLCKSZ が適用されます。
  • lt->writing が偽かつ lt->frozen が偽の場合、アサーションエラーが発生します。
  • 物理ファイルへの書き込み失敗時は ltsWriteBlock 経由でエラーが伝播します。

保守担当者向けメモ

  • 本関数は、書き込みから読み込みへの遷移時にバッファを再割り当てするため、呼び出し後に LogicalTapeRead を実行する際はバッファが NULL であることに留意してください。
  • prealloc リストの解放は、読み込み開始時に不要なブロックをフリーリストへ戻すための重要な処理です。
