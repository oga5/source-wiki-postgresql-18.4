RemoveAttrDefaultById

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_attrdef.c:206

目的

指定された OID に対応する pg_attrdef エントリを削除し、関連するテーブル属性のデフォルト値フラグを更新します。

シグネチャまたは形状

void RemoveAttrDefaultById(Oid attrdefId)

入力と出力

  • **入力**: attrdefId (削除対象の pg_attrdef エントリの OID)
  • **出力**: なし（副作用としてカタログおよびシステムキャッシュを更新）

前提条件と不変条件

  • pg_attrdef カタログに対する RowExclusiveLock を保持する必要があります。
  • 対象の属性を所有するテーブルに対して AccessExclusiveLock を取得します。
  • 削除対象の OID が pg_attrdef に存在しない場合、エラーが発生します。

アルゴリズムまたは振る舞い

1. pg_attrdef を開き、指定された attrdefId を検索します。 2. 該当するタプルから、対象のテーブル OID (adrelid) と属性番号 (adnum) を抽出します。 3. 対象テーブルに対して AccessExclusiveLock を取得します。 4. CatalogTupleDelete を用いて pg_attrdef から該当行を削除します。 5. pg_attribute カタログを開き、対象属性の atthasdef フラグを false に更新します。 6. CatalogTupleUpdate を実行し、リレーションキャッシュ（relcache）の再構築をトリガーします。 7. 最後にテーブルのロックを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c:1400 (performDeletion 経由)
  • **呼び出し先**: table_open, systable_beginscan, systable_getnext, relation_open, CatalogTupleDelete, SearchSysCacheCopy2, CatalogTupleUpdate, table_close, relation_close

関連データ構造

  • pg_attrdef (システムカタログ)
  • pg_attribute (システムカタログ)
  • Form_pg_attrdef (タプル構造体)
  • Form_pg_attribute (タプル構造体)

失敗モードと境界条件

  • 指定した attrdefId が存在しない場合、elog(ERROR) により処理が中断されます。
  • pg_attribute のキャッシュ検索に失敗した場合、整合性エラーとして elog(ERROR) が発生します。

保守担当者向けメモ

  • 本関数は performDeletion から呼び出されることを前提としています。
  • pg_attribute の更新はリレーションキャッシュの再構築を伴うため、呼び出し側で追加のキャッシュ無効化処理を行う必要はありません。
  • 生成列（Generated Column）の削除については、通常のデフォルト値とは異なる依存関係管理が必要となるため、呼び出し元の文脈に注意してください。
