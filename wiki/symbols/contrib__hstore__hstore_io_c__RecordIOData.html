RecordIOData

定義

  • 種別: struct
  • 定義箇所: contrib/hstore/hstore_io.c:823

目的

hstore 型と PostgreSQL のレコード型（record）との相互変換を行う際に、対象となるレコードの型情報および各カラムの入出力処理に必要なメタデータを保持するために使用されます。

シグネチャまたは形状

typedef struct RecordIOData
{
    Oid         record_type;
    int32       record_typmod;
    void       *domain_info;
    int         ncolumns;
    ColumnIOData columns[FLEXIBLE_ARRAY_MEMBER];
} RecordIOData;

入力と出力

  • **入力**: hstore_from_record や hstore_populate_record 等の関数において、fcinfo->flinfo->fn_extra を介してキャッシュとして提供されます。
  • **出力**: 構造体内の columns 配列を通じて、各カラムの型変換に必要な情報（ColumnIOData）が後続の処理へ渡されます。

前提条件と不変条件

  • columns フィールドは可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、インスタンス生成時に ncolumns に基づくメモリ確保が必要です。
  • domain_info は、対象型が複合型上のドメインである場合にのみ使用される不透明なキャッシュです。

アルゴリズムまたは振る舞い

  • hstore_io.c 内の関数において、fn_extra に格納された本構造体をキャストして取得し、レコードの各カラムに対する入出力処理を効率化します。
  • メモリ確保時には offsetof(RecordIOData, columns) + (ncolumns * sizeof(ColumnIOData)) のような計算式を用いて、可変長部分を含めたサイズを算出します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_io.c 内の hstore_from_record および hstore_populate_record が fn_extra から本構造体を読み取ります。
  • **呼び出し先**: src/backend/utils/adt/jsonfuncs.c において、本構造体へのポインタが利用されていることが示唆されています。

関連データ構造

  • ColumnIOData: 各カラムの型変換情報を保持する構造体。
  • HStore: hstore 型の内部表現。

失敗モードと境界条件

  • ncolumns が実際のレコード構造と一致しない場合、columns 配列へのアクセスでメモリ破壊が発生する可能性があります。
  • fn_extra が初期化されていない場合、キャスト後のアクセスは未定義動作となります。

保守担当者向けメモ

  • 本構造体は可変長であるため、palloc 等で確保する際はサイズ計算に注意してください。
  • domain_info のライフサイクルは fn_extra の生存期間に依存します。
