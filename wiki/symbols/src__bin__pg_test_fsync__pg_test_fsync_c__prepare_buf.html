prepare_buf

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:228

目的

ベンチマークテストで使用するバッファ領域を、ランダムなデータで初期化し、アライメントを調整する。

シグネチャまたは形状

static void prepare_buf(void)

入力と出力

  • 入力: なし（グローバル変数 full_buf および pg_global_prng_state を参照）
  • 出力: なし（グローバル変数 buf を更新）

前提条件と不変条件

  • full_buf が DEFAULT_XLOG_SEG_SIZE 以上のサイズで確保されていること。
  • pg_global_prng_state が初期化済みであること。
  • buf は TYPEALIGN マクロにより XLOG_BLCKSZ にアライメントされたアドレスを保持する。

アルゴリズムまたは振る舞い

1. DEFAULT_XLOG_SEG_SIZE の範囲でループし、pg_prng_int32 を用いて full_buf の各要素にランダムな値を書き込む。 2. TYPEALIGN マクロを使用して、full_buf の先頭アドレスを XLOG_BLCKSZ 境界に切り上げたアドレスを buf に格納する。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（src/bin/pg_test_fsync/pg_test_fsync.c:124）
  • 呼び出し先: pg_prng_int32（乱数生成）

関連データ構造

  • full_buf: バッファの生データ領域。
  • buf: アライメント調整後の書き込み用バッファポインタ。
  • pg_global_prng_state: 乱数生成器の状態。

失敗モードと境界条件

  • full_buf のサイズが不足している場合、バッファオーバーフローが発生する可能性がある。
  • TYPEALIGN によるアライメント調整後、buf が full_buf の確保領域を逸脱しないことが前提となる。

保守担当者向けメモ

  • 本関数はプログラムの初期化フェーズで一度だけ呼び出される。
  • XLOG_BLCKSZ の定義変更に伴い、アライメントの挙動が変化するため注意すること。
