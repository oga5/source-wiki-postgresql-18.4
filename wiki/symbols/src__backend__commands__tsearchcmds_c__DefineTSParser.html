DefineTSParser

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/commands/tsearchcmds.c:183

目的

SQLの CREATE TEXT SEARCH PARSER コマンドを実行し、新しいテキスト検索パーサーをシステムカタログ pg_ts_parser に登録する。

シグネチャまたは形状

ObjectAddress DefineTSParser(List *names, List *parameters)

入力と出力

  • **入力**:
  • names: パーサー名を指定する List。
  • parameters: パーサーの各メソッド（start, gettoken, end, headline, lextypes）を定義する DefElem のリスト。
  • **出力**:
  • ObjectAddress: 作成されたパーサーオブジェクトのカタログ上の位置情報。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • start, gettoken, end, lextypes メソッドの定義が必須。これらが欠けている場合はエラーとなる。

アルゴリズムまたは振る舞い

1. **権限確認**: superuser() を呼び出し、非特権ユーザーによる実行を拒否する。 2. **カタログ準備**: pg_ts_parser を RowExclusiveLock で開く。 3. **名前解決**: QualifiedNameGetCreationNamespace で名前空間を特定する。 4. **パラメータ解析**: parameters リストを走査し、各メソッドに対応する関数 OID を get_ts_parser_func で取得してタプルフィールドに格納する。 5. **検証**: 必須メソッドがすべて設定されているか確認する。 6. **永続化**: heap_form_tuple でタプルを作成し、CatalogTupleInsert でカタログへ挿入する。 7. **依存関係構築**: makeParserDependencies を呼び出し、関連するオブジェクトとの依存関係を記録する。 8. **フック実行**: InvokeObjectPostCreateHook を呼び出し、イベントトリガーやキャッシュ無効化を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c (SQL解析後の実行フェーズ)
  • **呼び出し先**:
  • superuser, table_open, QualifiedNameGetCreationNamespace, GetNewOidWithIndex, get_ts_parser_func, heap_form_tuple, CatalogTupleInsert, makeParserDependencies, InvokeObjectPostCreateHook, heap_freetuple, table_close

関連データ構造

  • pg_ts_parser: パーサー定義を保持するシステムカタログ。
  • DefElem: SQLコマンドから渡されるパラメータ定義。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • **権限不足**: スーパーユーザーでない場合に ERRCODE_INSUFFICIENT_PRIVILEGE でエラー。
  • **定義不備**: 必須メソッドが指定されていない場合に ERRCODE_INVALID_OBJECT_DEFINITION でエラー。
  • **構文エラー**: 未知のパラメータが指定された場合に ERRCODE_SYNTAX_ERROR でエラー。

保守担当者向けメモ

  • 本関数は pg_ts_parser カタログへの書き込みを行うため、カタログスキーマの変更時には values 配列のインデックスや Natts_pg_ts_parser の整合性に注意すること。
  • 依存関係の管理は makeParserDependencies に委譲されているため、依存関係のロジックを変更する場合はそちらを確認すること。
