destroy_tablespace_directories

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tablespace.c:685

目的

指定されたテーブルスペースに関連する物理ディレクトリ構造（バージョンディレクトリおよびデータベースサブディレクトリ）およびシンボリックリンクを削除する。

シグネチャまたは形状

static bool destroy_tablespace_directories(Oid tablespaceoid, bool redo)

入力と出力

  • **入力**:
  • tablespaceoid (Oid): 削除対象のテーブルスペースの OID。
  • redo (bool): WAL リプレイ中であれば true、通常の DDL 実行中であれば false。
  • **出力**:
  • bool: 成功した場合は true、失敗した場合は false。

前提条件と不変条件

  • 呼び出し元は TablespaceCreateLock を保持している必要がある。
  • 通常運用時（redo == false）は、ディレクトリが空でない場合に削除を中止する。
  • WAL リプレイ時（redo == true）は、ディレクトリが存在しない場合（ENOENT）を許容する。

アルゴリズムまたは振る舞い

1. PG_TBLSPC_DIR/<tablespaceoid>/<version> ディレクトリを開く。 2. ディレクトリ内の各サブディレクトリ（データベース別）を走査し、rmdir を試みる。

  • 通常運用時、ディレクトリが空でない場合は false を返す。
  • 削除失敗時は、redo モードに応じてエラーまたはログを出力する。

    • 3. バージョンディレクトリ自体を rmdir で削除する。 4. 最後に、PG_TBLSPC_DIR 内の対象テーブルスペースに対応するシンボリックリンク（またはディレクトリ）を unlink または rmdir で削除する。

  • 削除対象がシンボリックリンクかディレクトリかを確認し、適切に処理する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablespace.c 内の DDL 処理ロジック（DROP TABLESPACE 等）。
  • tblspc_redo（WAL リプレイ時）。
  • **呼び出し先**:
  • AllocateDir, ReadDir, FreeDir（ディレクトリ走査）
  • directory_is_empty（空チェック）
  • rmdir, unlink, lstat（ファイルシステム操作）
  • psprintf, pstrdup, pfree（メモリ管理）

関連データ構造

  • Oid: テーブルスペース識別子。
  • DIR, dirent: ディレクトリ操作用構造体。
  • struct stat: ファイル属性取得用。

失敗モードと境界条件

  • **ディレクトリが存在しない**: redo が true の場合は無視、false の場合は警告を発行し、シンボリックリンクの削除へ進む。
  • **ディレクトリが空でない**: 通常運用時は削除を中断し false を返す。
  • **権限不足またはファイルシステムエラー**: rmdir や unlink の失敗は、redo モードに応じて ERROR または LOG として報告される。
  • **対象がディレクトリ/シンボリックリンク以外**: 削除を拒否しエラーを報告する。

保守担当者向けメモ

  • 本関数は TablespaceCreateLock に依存して並行性を制御している。
  • redo モード時の ENOENT は正常なパスとして扱われるが、それ以外のエラーはログに記録される。
  • シンボリックリンクの削除処理は、WAL リプレイ時にディレクトリとして作成されている可能性を考慮した実装となっている。
