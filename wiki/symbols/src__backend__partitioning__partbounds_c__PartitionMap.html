PartitionMap

定義

  • **種別**: struct
  • **定義箇所**: src/backend/partitioning/partbounds.c:76

目的

パーティションワイズ結合（Partitionwise Join）処理において、結合対象となる2つのパーティション境界の対応関係やマージ状態を追跡するために使用される構造体です。

シグネチャまたは形状

typedef struct PartitionMap
{
    int         nparts;          /* パーティション数 */
    int        *merged_indexes;  /* マージされたパーティションのインデックス */
    bool       *merged;          /* 非ダミーパーティションとマージ済みかを示すフラグ */
    bool        did_remapping;   /* 再マッピングの有無 */
    int        *old_indexes;     /* 再マッピング時の旧インデックス */
} PartitionMap;

入力と出力

本構造体は、partition_bounds_merge 処理の過程で状態を保持するために利用されます。入力として init_partition_map を介して初期化され、結合処理の各ステップで各フィールドが更新されます。

前提条件と不変条件

  • nparts は対象となるパーティションの総数を保持します。
  • merged_indexes および merged 配列は、nparts に基づくサイズで確保される必要があります。
  • did_remapping が true の場合、old_indexes に有効なインデックス情報が格納されていることが期待されます。

アルゴリズムまたは振る舞い

パーティションワイズ結合において、結合対象のパーティションが一致または包含関係にあるかを判定し、その結果を merged フラグや merged_indexes に記録します。再マッピングが必要な場合は did_remapping を通じて追跡し、インデックスの整合性を維持します。

呼び出し元と呼び出し先

  • **呼び出し元**: partition_bounds_merge（間接的な参照を含む）
  • **呼び出し先**:
  • init_partition_map (初期化)
  • free_partition_map (解放)
  • merge_matching_partitions, process_outer_partition, process_inner_partition, merge_null_partitions, merge_default_partitions (状態更新および処理)

関連データ構造

  • PartitionBoundInfoData: パーティション境界の情報を保持する主要構造体。
  • RelOptInfo: init_partition_map の引数として使用されます。

失敗モードと境界条件

  • メモリ割り当て失敗: merged_indexes 等の配列確保時に発生する可能性があります。
  • インデックスの不整合: nparts と実際のパーティション数が一致しない場合、配列アクセス時に境界外参照が発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は結合処理の複雑な状態遷移を管理するため、デバッグ時には merged フラグの変化と did_remapping の遷移を追跡することが推奨されます。
  • Rangeパーティション等の複雑な境界条件を持つ結合において、インデックスの再マッピングが正しく行われているかを確認してください。
