A_Const

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/parsenodes.h:381

目的

SQLのパースツリーにおいて、リテラル定数を表現するためのノードです。

シグネチャまたは形状

typedef struct A_Const
{
    pg_node_attr(custom_copy_equal, custom_read_write, custom_query_jumble)

    NodeTag     type;
    union ValUnion val;
    bool        isnull;
    ParseLoc    location;
} A_Const;

入力と出力

  • **入力**: SQL文中のリテラル値。
  • **出力**: パーサーによって生成されるパースツリーの一部として、後続の解析フェーズ（parse_analyze）へ渡されます。

前提条件と不変条件

  • isnull が true の場合、val の内容は無視されます。
  • location はSQL文字列内のバイトオフセットを保持します。不明な場合は -1 が設定されます。
  • pg_node_attr により、コピー、比較、読み書き、およびクエリの同一性判定（Jumbling）においてカスタム処理が適用されます。

アルゴリズムまたは振る舞い

  • パーサーがSQLを読み込む際に生成されます。
  • copyfuncs.c や equalfuncs.c などのノード操作関数群により、ツリーの複製や比較が行われます。
  • クエリの同一性判定（queryjumblefuncs.c）において、定数値として扱われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/gram.y（パーサー）が主に生成します。また、src/backend/nodes/makefuncs.c を通じてノードが作成されます。
  • **呼び出し先**: copyfuncs.c, equalfuncs.c, outfuncs.c, readfuncs.c, nodeFuncs.c, queryjumblefuncs.c などのノード処理ユーティリティが本構造体を操作します。

関連データ構造

  • union ValUnion: 定数の値を保持する共用体。
  • NodeTag: ノードの種類を識別するタグ。
  • ParseLoc: SQL文字列内の位置情報。

失敗モードと境界条件

  • location が -1 の場合、エラーメッセージ等で正確な位置を表示できません。
  • isnull フラグの不整合は、意味解析フェーズでの型チェックエラーを引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は gen_node_support.pl による自動生成の対象です。フィールドの追加や変更を行う際は、関連する funcs.c ファイル群の更新が必要になる場合があります。
  • query_jumble 属性が付与されているため、クエリの正規化処理において定数値がどのように扱われるか（定数置換の有無など）に注意してください。
