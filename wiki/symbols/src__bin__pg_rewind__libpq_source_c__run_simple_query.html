run_simple_query

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/libpq_source.c:162

目的

リモートの PostgreSQL サーバーに対して単一の結果セットを返す SQL クエリを実行し、その結果を文字列として取得する。

シグネチャまたは形状

static char *run_simple_query(PGconn *conn, const char *sql)

入力と出力

  • **入力**:
  • conn: libpq の接続ハンドル (PGconn *)。
  • sql: 実行する SQL クエリ文字列。
  • **出力**:
  • クエリ結果の最初の行・最初の列の値をコピーした文字列 (char *) を返す。呼び出し元で解放が必要。

前提条件と不変条件

  • クエリは必ず「1行かつ1列」の結果を返す必要がある。
  • 結果が NULL であってはならない。

アルゴリズムまたは振る舞い

1. PQexec を使用して SQL を実行する。 2. クエリの実行結果が PGRES_TUPLES_OK でない場合、pg_fatal を呼び出してエラー終了する。 3. 結果セットのフィールド数およびタプル数が 1 であり、かつ値が NULL でないことを検証する。条件を満たさない場合は pg_fatal で終了する。 4. PQgetvalue で取得した値を pg_strdup で複製する。 5. PQclear で結果セットを解放し、複製した文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/libpq_source.c 内の以下の箇所から呼び出されている。
  • SHOW full_page_writes の実行時。
  • SELECT pg_current_wal_insert_lsn() の実行時。
  • **呼び出し先**:
  • libpq ライブラリ関数 (PQexec, PQresultStatus, PQnfields, PQntuples, PQgetisnull, PQgetvalue, PQclear)。
  • pg_fatal (エラーハンドリング用)。
  • pg_strdup (メモリ確保用)。

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。
  • PGresult: クエリ実行結果を保持する構造体。

失敗モードと境界条件

  • クエリ実行自体が失敗した場合、または結果セットが期待する形式（1行1列かつ非NULL）でない場合、pg_fatal によりプログラムが異常終了する。

保守担当者向けメモ

  • 本関数は単一値を取得するためのユーティリティであり、複雑なデータセットの取得には適さない。
  • 戻り値の文字列は pg_strdup で確保されているため、呼び出し側で適切に free する必要がある。
