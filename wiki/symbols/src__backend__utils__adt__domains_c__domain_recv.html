domain_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/domains.c:286

目的

ドメイン型のバイナリ形式入力（recv）を処理し、基底型の変換後にドメイン制約（NOT NULL や CHECK 制約）の検証を行う。

シグネチャまたは形状

Datum domain_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): StringInfo 型のバイナリデータバッファ。
  • PG_GETARG_OID(1): 対象となるドメイン型の OID。
  • **出力**:
  • 制約を通過した Datum 値。入力が NULL の場合は PG_RETURN_NULL() を返す。

前提条件と不変条件

  • domain_recv は非厳密（non-strict）関数であり、NULL 入力を許容する。
  • fcinfo->flinfo->fn_extra を使用して DomainIOData をキャッシュし、カタログ検索を最適化する。
  • ドメイン型が変更されない限り、キャッシュされた DomainIOData を再利用する。

アルゴリズムまたは振る舞い

1. **入力確認**: 引数が NULL か判定し、バッファを取得する。 2. **キャッシュ初期化**: fn_extra が未設定または型が不一致の場合、domain_state_setup を呼び出して DomainIOData を構築・保存する。 3. **基底型変換**: ReceiveFunctionCall を呼び出し、基底型のバイナリ入力関数を実行して Datum を生成する。 4. **制約検証**: domain_check_input を呼び出し、生成された値がドメイン定義の制約を満たすか検証する。 5. **戻り値**: 検証成功時に値を返し、入力が NULL であれば NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの型変換処理（直接的な呼び出し元情報はソースコード内に明示なし）。
  • **呼び出し先**:
  • domain_state_setup
  • ReceiveFunctionCall
  • domain_check_input

関連データ構造

  • DomainIOData: ドメインの制約情報や基底型の I/O 情報を保持する構造体。
  • StringInfo: バイナリデータバッファ。

失敗モードと境界条件

  • **制約違反**: domain_check_input 内で CHECK 制約や NOT NULL 制約に違反した場合、エラーが発生する。
  • **型不一致**: キャッシュされた domain_type と現在の domainType が一致しない場合、再セットアップが行われる。
  • **NULL 入力**: PG_ARGISNULL(0) の場合、バッファは NULL として扱われ、制約チェックをスキップして NULL を返す。

保守担当者向けメモ

  • 本関数は domain_in と同様の制約検証レイヤーをバイナリ入力に対して提供する。
  • fn_extra のメモリ管理は fcinfo->flinfo->fn_mcxt に依存しているため、コンテキストの生存期間に注意すること。
