inet_net_pton_ipv4

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/inet_net_pton.c:258

目的

IPv4 アドレス（およびオプションの CIDR プレフィックス）を、人間が読みやすい文字列形式からネットワークバイトオーダーのバイナリ形式へ変換します。

シグネチャまたは形状

static int inet_net_pton_ipv4(const char *src, u_char *dst);

入力と出力

  • **入力**:
  • src: パース対象の IPv4 アドレス文字列（例: "192.168.1.0" または "192.168.1.0/24"）。
  • dst: 変換後のバイナリ（4バイト）を格納するバッファ。
  • **出力**:
  • 成功時: 確定したプレフィックス長（ビット数、0〜32）。
  • 失敗時: -1 を返し、errno を ENOENT（不正な形式）または EMSGSIZE（バッファ不足や範囲外）に設定。

前提条件と不変条件

  • dst は少なくとも 4 バイトの領域を指している必要があります。
  • プレフィックス長が指定された場合、その値は 32 以下である必要があります。
  • プレフィックス長が指定されない場合、4 オクテットすべてが記述されていればデフォルトで 32 とみなされます。

アルゴリズムまたは振る舞い

1. **オクテットのパース**: . で区切られた 10 進数文字列を走査し、各オクテットをバイナリに変換して dst に書き込みます。各値は 255 を超えてはなりません。 2. **CIDR 処理**: / が出現した場合、後続の数値をプレフィックス長として読み取ります。 3. **バリデーション**:

  • プレフィックス長が 32 を超える場合は EMSGSIZE を返します。
  • プレフィックス長が指定されていない場合、4 オクテットすべてが揃っていないと ENOENT を返します。
  • プレフィックス長がオクテット数（バイト数）を超えて指定されている場合（例: 1 バイトしか指定がないのに /24 など）、ENOENT を返します。

    • 4. **パディング**: 4 オクテットに満たない場合、残りのバイトを 0 で埋めます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/inet_net_pton.c 内の関数（pg_inet_net_pton 等）から呼び出されることが示唆されていますが、具体的な呼び出し階層の詳細は提供されていません。
  • **呼び出し先**: 標準ライブラリの isdigit, strchr を使用します。

関連データ構造

  • u_char: ネットワークバイトオーダーのバイナリ表現に使用される型。

失敗モードと境界条件

  • ENOENT: 文字列が IPv4 として不正、オクテットが 255 を超える、プレフィックス長がオクテット数と整合しない、または入力が空の場合。
  • EMSGSIZE: プレフィックス長が 32 を超える場合。

保守担当者向けメモ

  • 本関数は ISC ライセンスに基づく外部由来のコードであり、PostgreSQL 固有のメモリ管理ではなく標準の errno を使用します。
  • プレフィックス長が指定されない場合のデフォルト値（32）の適用条件は、4 オクテットすべてが記述されていることに依存しています。
