recordAdditionalCatalogID

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/common.c:718

目的

指定された CatalogId を catalogIdHash ハッシュテーブルに登録し、対応する DumpableObject への参照を保持する。

シグネチャまたは形状

void recordAdditionalCatalogID(CatalogId catId, DumpableObject *dobj)

入力と出力

  • **入力**:
  • catId: 登録対象の CatalogId。
  • dobj: 関連付ける DumpableObject へのポインタ。
  • **出力**: なし（副作用としてハッシュテーブルを更新）。

前提条件と不変条件

  • catalogIdHash が初期化済みであること（Assert(catalogIdHash != NULL) により検証）。
  • 登録対象の CatalogId がハッシュテーブル内に存在しないか、存在しても dobj が未設定（NULL）であること（Assert(entry->dobj == NULL) により検証）。

アルゴリズムまたは振る舞い

1. catalogIdHash が存在することを検証する。 2. catalogid_insert を呼び出し、catId に対応するエントリを取得または作成する。 3. 新規作成されたエントリであれば、dobj および ext フィールドを NULL で初期化する。 4. エントリの dobj フィールドが NULL であることを確認し、引数の dobj を代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_dump.c:3947
  • **呼び出し先**:
  • catalogid_insert (ハッシュテーブル操作関数)

関連データ構造

  • CatalogId: オブジェクトを識別するOIDのペア。
  • DumpableObject: ダンプ対象オブジェクトの抽象構造体。
  • CatalogIdMapEntry: ハッシュテーブルの要素。dobj および ext ポインタを保持する。

失敗モードと境界条件

  • catalogIdHash が NULL の場合、アサーション違反により異常終了する。
  • 既に dobj が設定されている CatalogId に対して呼び出した場合、アサーション違反により異常終了する。

保守担当者向けメモ

  • 本関数は pg_dump におけるオブジェクト検索の基盤の一部であり、simplehash.h を利用したハッシュテーブル管理に依存している。
  • CatalogIdMapEntry のメモリ管理は simplehash の実装に委ねられているため、ポインタの生存期間に注意すること。
