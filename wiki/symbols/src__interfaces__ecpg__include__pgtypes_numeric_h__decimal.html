decimal

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/include/pgtypes_numeric.h:29

目的

ECPG（Embedded SQL for C）において、固定サイズのメモリ領域で高精度数値を保持・操作するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
    int           ndigits;
    int           weight;
    int           rscale;
    int           dscale;
    int           sign;
    NumericDigit  digits[DECSIZE];
} decimal;

入力と出力

  • **入力**: 構造体の各メンバ（ndigits, weight, rscale, dscale, sign, digits）に値を設定することで、数値の表現を定義します。
  • **出力**: 構造体内の各メンバを通じて、数値の属性（符号、スケール、重み）および実際の桁データが保持されます。

前提条件と不変条件

  • DECSIZE で定義された固定長配列 digits を使用するため、保持できる精度には上限があります。
  • ndigits は 0 を取り得ます。
  • sign メンバは NUMERIC_POS、NUMERIC_NEG、または NUMERIC_NAN のいずれかである必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、動的メモリ確保を行わないスタック配置に適した設計です。
  • numeric 構造体とは異なり、サイズが固定されているため、DECSIZE を超える精度が必要な場合には使用できません。
  • 数値の操作は、PGTYPESnumeric_* 系の関数を介して行われることが推奨されます。

呼び出し元と呼び出し先

  • ツールによる解析では、本構造体を直接参照する具体的な呼び出し元・呼び出し先は特定できませんでした。
  • 関連する実装コードは src/interfaces/ecpg/pgtypeslib/numeric.c に存在します。

関連データ構造

  • NumericDigit: digits 配列の要素型。
  • numeric: 動的メモリ確保を行う、可変長の高精度数値構造体。

失敗モードと境界条件

  • DECSIZE を超える桁数の数値を格納しようとした場合、データが切り捨てられるか、構造体の制限により表現不能となります。
  • sign に不正な値が設定された場合の動作は未定義です。

保守担当者向けメモ

  • numeric との主な違いはメモリ管理戦略です。スタック上に配置したい場合は decimal を、動的なサイズ変更が必要な場合は numeric を選択してください。
  • 構造体のメンバを直接操作せず、提供されている API を通じてアクセスしてください。
