plpython_error_callback

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_main.c:345

目的

PostgreSQL のエラー発生時に、PL/Python 実行コンテキストから関数名や手続き名を抽出し、エラーメッセージにコンテキスト情報（errcontext）を付与するために使用されます。

シグネチャまたは形状

static void plpython_error_callback(void *arg)

入力と出力

  • **入力**: void *arg - PLyExecutionContext 構造体へのポインタ。
  • **出力**: なし（副作用として PostgreSQL のエラースタックに情報を追加）。

前提条件と不変条件

  • 引数 arg は有効な PLyExecutionContext 構造体を指している必要があります。
  • exec_ctx->curr_proc が存在する場合、その手続きまたは関数名は有効である必要があります。

アルゴリズムまたは振る舞い

1. 引数 arg を PLyExecutionContext * にキャストします。 2. exec_ctx->curr_proc が存在するか確認します。 3. is_procedure フラグに基づき、現在の実行対象が「プロシージャ」か「関数」かを判定します。 4. PLy_procedure_name を呼び出して対象の名前を取得し、errcontext を用いてエラーコンテキストを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/pl/plpython/plpy_main.c:228 にて、エラーコンテキストのコールバックとして登録されています。
  • **呼び出し先**: PLy_procedure_name（名前の取得用）。

関連データ構造

  • PLyExecutionContext: 実行中の手続きやメモリコンテキストを保持するスタック構造体。
  • PLyProcedure: 実行される関数やプロシージャの情報を保持する構造体。

失敗モードと境界条件

  • exec_ctx が NULL の場合、または curr_proc が NULL の場合、コンテキスト情報の追加はスキップされます。

保守担当者向けメモ

  • 本関数は PostgreSQL のエラー報告メカニズムの一部として機能します。
  • PLy_procedure_name の実装変更が本関数の出力形式に直接影響するため、変更時は注意してください。
