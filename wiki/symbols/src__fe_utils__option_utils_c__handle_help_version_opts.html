handle_help_version_opts

定義

  • 種別: function
  • 定義箇所: src/fe_utils/option_utils.c:23

目的

PostgreSQLフロントエンドツールにおいて、コマンドライン引数に --help (-?) または --version (-V) が指定された場合に、標準的な情報を出力してプロセスを終了させるための共通処理を提供します。

シグネチャまたは形状

void handle_help_version_opts(int argc, char *argv[], const char *fixed_progname, help_handler hlp);

入力と出力

  • argc, argv: main 関数から渡されるコマンドライン引数。
  • fixed_progname: バージョン表示時に使用するプログラム名。
  • hlp: ヘルプ表示を実行するための関数ポインタ（help_handler 型）。
  • 出力: 該当するオプションが指定された場合、標準出力に情報を書き出し、exit(0) によりプログラムを終了します。該当しない場合は何も行わず呼び出し元へ戻ります。

前提条件と不変条件

  • argc > 1 の場合にのみ引数のチェックが行われます。
  • 該当オプションが指定された場合、本関数は呼び出し元へ戻ることはありません。

アルゴリズムまたは振る舞い

1. 引数の数が1より大きい場合、argv[1] を確認します。 2. argv[1] が --help または -? と一致する場合、hlp 関数を呼び出し、プログラム名（get_progname(argv[0])）を渡してヘルプを表示し、exit(0) を実行します。 3. argv[1] が --version または -V と一致する場合、fixed_progname と PG_VERSION を含むバージョン情報を出力し、exit(0) を実行します。 4. いずれにも一致しない場合は、そのまま呼び出し元へ制御を戻します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • pg_amcheck, pg_combinebackup, pg_walsummary
  • clusterdb, createdb, createuser, dropdb, dropuser, pg_isready, reindexdb, vacuumdb
  • 呼び出し先:
  • hlp (引数として渡された関数ポインタ)
  • get_progname (標準ライブラリ/ユーティリティ)
  • printf (標準ライブラリ)
  • exit (標準ライブラリ)

関連データ構造

  • help_handler: ヘルプ表示を行う関数の型定義。

失敗モードと境界条件

  • 引数が1つのみ（argc <= 1）の場合、本関数は何も行いません。
  • argv[1] 以外の引数は無視されます。

保守担当者向けメモ

  • 本関数は main 関数の冒頭で呼び出されることを想定しています。
  • バージョン表示には PG_VERSION マクロが使用されます。
  • 新しいツールを追加する際は、適切な help_handler を実装して本関数に渡すことで、標準的なヘルプ表示挙動を継承できます。
