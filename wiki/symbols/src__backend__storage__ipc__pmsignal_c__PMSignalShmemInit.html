PMSignalShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:144

目的

Postmasterとバックエンドプロセス間でのシグナル通知および状態管理に使用する共有メモリ領域（PMSignalData）を初期化またはアタッチします。

シグネチャまたは形状

void PMSignalShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 PMSignalState および num_child_flags を更新）

前提条件と不変条件

  • 共有メモリの初期化フェーズで呼び出される必要があります。
  • PMSignalShmemSize() が返すサイズ分の共有メモリが確保可能である必要があります。

アルゴリズムまたは振る舞い

1. ShmemInitStruct を呼び出し、"PMSignalState" という名前で共有メモリ領域を確保または既存領域へアタッチします。 2. 領域が新規作成された場合（found が false の場合）:

  • MemSet を使用して共有メモリ領域全体を 0 で初期化します。
  • MaxLivePostmasterChildren() を呼び出し、システムで許容される最大子プロセス数を取得します。
  • 取得した値を共有メモリ内の num_child_flags およびローカル変数 num_child_flags に格納します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:325（共有メモリ初期化シーケンス内）
  • 呼び出し先: ShmemInitStruct, PMSignalShmemSize, MemSet, MaxLivePostmasterChildren

関連データ構造

  • PMSignalData: 共有メモリ上に配置されるメインの管理構造体。
  • PMSignalState: PMSignalData を指すグローバルポインタ。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、ShmemInitStruct 内部でエラーが発生しプロセスが終了します。

保守担当者向けメモ

  • 本関数は共有メモリの初期化時に一度だけ実行されることを想定しています。
  • unvolatize マクロを使用して PMSignalState の型修飾を一時的に解除し、MemSet を適用しています。
