apw_dump_now

定義

  • 種別: function
  • 定義箇所: contrib/pg_prewarm/autoprewarm.c:665

目的

現在の共有バッファの状態（ブロック情報）を一時ファイルへ書き出し、アトミックに永続化ファイル（autoprewarm.blocks）へ置き換えることで、サーバー再起動後のプリウォーム用データを保存します。

シグネチャまたは形状

int apw_dump_now(bool is_bgworker, bool dump_unlogged)

入力と出力

  • **入力**:
  • is_bgworker: バックグラウンドワーカーとして実行中かを示すフラグ。
  • dump_unlogged: unlogged テーブルのブロックを含めるかを示すフラグ。
  • **出力**:
  • 戻り値: 書き出したブロックの総数（int）。
  • 副作用: autoprewarm.blocks.tmp を作成し、最終的に autoprewarm.blocks へリネームする。

前提条件と不変条件

  • apw_state->lock を使用して、ダンプ処理が重複しないよう排他制御されます。
  • 共有バッファの全スキャンを行うため、NBuffers 分のメモリ（BlockInfoRecord 配列）を確保します。
  • 処理中に CHECK_FOR_INTERRUPTS() を呼び出し、割り込み要求に応答します。

アルゴリズムまたは振る舞い

1. **排他制御**: apw_state->pid_using_dumpfile を確認し、既に別のプロセスがダンプ中であれば、バックグラウンドワーカーの場合はスキップ、それ以外はエラーを発生させます。 2. **バッファスキャン**: 全共有バッファを走査し、有効なブロック（BM_TAG_VALID）かつ永続的（または dump_unlogged が真）なブロックのメタデータを収集します。 3. **一時ファイル書き出し**: autoprewarm.blocks.tmp を作成し、ヘッダー（ブロック数）と収集したブロック情報を CSV 形式で書き込みます。 4. **アトミック更新**: 書き込み完了後、durable_rename を用いて autoprewarm.blocks へリネームし、状態を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • autoprewarm.c 内のバックグラウンドワーカーメインループ（autoprewarm_main 等）。
  • ユーザーからの手動実行要求（pg_prewarm 経由の呼び出し）。
  • **呼び出し先**:
  • GetBufferDescriptor, LockBufHdr, UnlockBufHdr（バッファ管理）。
  • AllocateFile, FreeFile, durable_rename（ファイル操作）。
  • palloc_extended（メモリ確保）。

関連データ構造

  • BlockInfoRecord: ダンプされるブロックのメタデータ（DB, テーブルスペース, ファイル番号, フォーク番号, ブロック番号）を保持。
  • AutoPrewarmSharedState: 共有メモリ上の制御構造体。ダンプ処理の実行状態を管理。

失敗モードと境界条件

  • **ファイル操作失敗**: 一時ファイルのオープン、書き込み、クローズ、またはリネームに失敗した場合、エラーを報告し、一時ファイルを削除します。
  • **メモリ不足**: NBuffers が非常に大きい場合、palloc_extended による巨大なメモリ確保が必要です。
  • **重複実行**: 既に別のプロセスがダンプを実行中の場合、呼び出し元に応じてスキップまたはエラーとなります。

保守担当者向けメモ

  • 現在、NBuffers に比例したメモリを確保する実装となっており、大規模な shared_buffers 環境ではメモリ効率が悪化する可能性があります。将来的なデータ構造の再設計が推奨されています。
  • durable_rename を使用しているため、クラッシュ時にもダンプファイルの一貫性が保証されます。
