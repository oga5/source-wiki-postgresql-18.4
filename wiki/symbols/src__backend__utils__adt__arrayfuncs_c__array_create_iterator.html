array_create_iterator

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:4602

目的

配列要素を効率的に走査するための ArrayIterator 構造体を初期化し、割り当てます。

シグネチャまたは形状

ArrayIterator array_create_iterator(ArrayType *arr, int slice_ndim, ArrayMetaState *mstate);

入力と出力

  • **入力**:
  • arr: 走査対象の ArrayType ポインタ。
  • slice_ndim: スライスとして扱う次元数（0の場合は要素単位の走査）。
  • mstate: 型情報のキャッシュ（ArrayMetaState）。NULLの場合は内部で型情報を取得します。
  • **出力**:
  • 初期化済みの ArrayIterator 構造体へのポインタを返します。

前提条件と不変条件

  • arr は有効なポインタである必要があります。
  • slice_ndim は 0 以上かつ配列の次元数以下である必要があります。
  • mstate が提供される場合、その element_type は arr の要素型と一致している必要があります。

アルゴリズムまたは振る舞い

1. ArrayIteratorData 用のメモリを palloc0 で確保します。 2. 配列の基本情報（NULLビットマップ、総要素数）を構造体に格納します。 3. mstate があればその情報を利用し、なければ get_typlenbyvalalign で型情報を取得します。 4. slice_ndim > 0 の場合、スライス用の次元情報、下限値、およびスライス構築用のワークスペース（slice_values, slice_nulls）を割り当てます。 5. 配列のデータポインタと現在のインデックスを初期化し、走査準備を完了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/array_userfuncs.c
  • src/pl/plpgsql/src/pl_exec.c
  • **呼び出し先**:
  • palloc0, palloc: メモリ確保。
  • ArrayGetNItems: 要素数計算。
  • get_typlenbyvalalign: 型属性の取得。
  • elog: エラー報告。

関連データ構造

  • ArrayType: 配列の内部表現。
  • ArrayIteratorData: 走査状態を保持する構造体。
  • ArrayMetaState: 型情報のキャッシュ構造体。

失敗モードと境界条件

  • slice_ndim が負の値、または配列の次元数を超えている場合、elog(ERROR) により処理を中断します。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は ArrayIterator のライフサイクルの開始点です。対応する解放関数 array_free_iterator が適切に呼び出されているか確認してください。
  • slice_ndim が 0 の場合、スライス用のメモリ割り当てはスキップされます。
