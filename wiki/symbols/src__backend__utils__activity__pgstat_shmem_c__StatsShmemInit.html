StatsShmemInit

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/activity/pgstat_shmem.c:154

目的

PostgreSQLの統計情報管理システムで使用する共有メモリ領域（DSAおよびハッシュテーブル）の確保と初期化を行います。

シグネチャまたは形状

void StatsShmemInit(void)

入力と出力

  • **入力**: なし（内部状態および共有メモリ設定に依存）
  • **出力**: なし（副作用として共有メモリ上に統計制御構造体を構築）

前提条件と不変条件

  • IsUnderPostmaster の状態により挙動が分岐します。
  • IsUnderPostmaster が偽（Postmaster起動時）の場合、共有メモリ領域が未確保であることを前提とします。
  • IsUnderPostmaster が真の場合、既に共有メモリが初期化済みであることを前提とします。

アルゴリズムまたは振る舞い

1. StatsShmemSize() を用いて必要な共有メモリサイズを算出します。 2. ShmemInitStruct を呼び出し、pgStatLocal.shmem に共有メモリ領域を割り当てます。 3. Postmaster起動時（!IsUnderPostmaster）の場合:

  • 共有メモリ内に dsa_area を作成し、ピン留めします。
  • dshash（共有ハッシュテーブル）を作成し、そのハンドルを ctl->hash_handle に格納します。
  • 統計情報のGC要求カウンタを初期化します。
  • PGSTAT_KIND_MIN から PGSTAT_KIND_MAX までの各統計種別に対し、fixed_amount が設定されている場合は init_shmem_cb を呼び出して初期化します。
  • 最後に、Postmasterプロセスがこれらを参照しないよう、ローカルの dsa/dshash 参照をデタッチします。

    • 4. それ以外の場合、found フラグが真であることを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c:344
  • **呼び出し先**:
  • StatsShmemSize, ShmemInitStruct
  • dsa_create_in_place, dsa_pin, dsa_set_size_limit, dsa_detach
  • dshash_create, dshash_get_hash_table_handle, dshash_detach
  • pg_atomic_init_u64
  • pgstat_get_kind_info, pgstat_is_kind_builtin

関連データ構造

  • PgStat_ShmemControl: 共有メモリ上の統計管理のルート構造体。
  • dsa_area: 動的共有メモリ領域。
  • dshash_table: 統計情報を格納する共有ハッシュテーブル。

失敗モードと境界条件

  • ShmemInitStruct が失敗した場合、PostgreSQLの共有メモリ確保ルールに従いプロセスが終了する可能性があります。
  • IsUnderPostmaster が真の状態で共有メモリが未初期化の場合、Assert(found) によりアボートします。

保守担当者向けメモ

  • 本関数はPostmaster起動時に一度だけ実行されることを想定しています。
  • dshash の作成時に dsa のサイズ制限を一時的に設ける必要があるため、実装順序には注意が必要です。
  • カスタム統計種別（PGSTAT_KIND_CUSTOM）のメモリ配置は、kind_info->shared_size に基づいて動的に計算されます。
