local_queue_fetch_file

定義

  • **種別**: 関数
  • **定義箇所**: src/bin/pg_rewind/local_source.c:74

目的

ローカルファイルシステム上のソースデータディレクトリから指定されたファイルを読み込み、ターゲットディレクトリへコピーする。

シグネチャまたは形状

static void local_queue_fetch_file(rewind_source *source, const char *path, size_t len)

入力と出力

  • **入力**:
  • source: rewind_source 構造体へのポインタ（local_source としてキャストして使用）。
  • path: コピー対象ファイルの相対パス。
  • len: 期待されるファイルサイズ（バイト単位）。
  • **出力**: なし（成功時はターゲットへの書き込みが完了し、失敗時は pg_fatal によりプロセスが終了する）。

前提条件と不変条件

  • source は local_source 型としてキャスト可能であること。
  • 読み込み中にソースファイルが変更されないこと（変更された場合は pg_fatal を発生させる）。

アルゴリズムまたは振る舞い

1. source からデータディレクトリのパスを取得し、対象ファイルのフルパスを構築する。 2. ソースファイルを読み込み専用で開く。 3. open_target_file(path, true) を呼び出し、ターゲットファイルを書き込み用に開く（既存内容は切り詰められる）。 4. PGIOAlignedBlock をバッファとして使用し、ソースからターゲットへデータを逐次読み書きする。 5. 全データのコピー完了後、読み込んだ総バイト数と引数 len を比較し、不一致があればエラーとする。 6. ソースファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: init_local_source 内で rewind_source インターフェースの関数ポインタとして登録される。
  • **呼び出し先**:
  • open_target_file (外部関数)
  • write_target_range (外部関数)
  • read, open, close (標準ライブラリ)
  • pg_fatal (エラーハンドリング)

関連データ構造

  • rewind_source: 抽象インターフェース。
  • local_source: datadir を保持する具象構造体。
  • PGIOAlignedBlock: I/O 操作用のバッファ構造体。

失敗モードと境界条件

  • ソースファイルが開けない場合、読み込みに失敗した場合、または閉じる際にエラーが発生した場合は pg_fatal で終了する。
  • 読み込み完了時のファイルサイズが引数 len と一致しない場合、同時変更とみなして pg_fatal で終了する。

保守担当者向けメモ

  • 本関数は同期的に動作する。
  • open_target_file および write_target_range は file_ops.c で定義されている。
  • 非同期的なフェッチを行う他の実装との互換性を保つため、インターフェース設計上この名前が付けられている。
