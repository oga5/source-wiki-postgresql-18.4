dclist_head

定義

  • 種別: struct
  • 定義箇所: src/include/lib/ilist.h:212

目的

計数付き二重連結リスト（Doubly Linked List with Count）の起点となる構造体です。標準的な二重連結リストに要素数の管理機能を追加し、リスト内のアイテム数を定数時間で取得可能にします。

シグネチャまたは形状

typedef struct dclist_head
{
    dlist_head    dlist;    /* 実際のリストヘッダー */
    uint32        count;    /* リスト内のアイテム数 */
} dclist_head;

入力と出力

  • **入力**: リスト操作を行う関数（dlist 関連の操作）に対して、この構造体へのポインタが渡されます。
  • **出力**: count フィールドにより、リストの現在の要素数が常に保持されます。

前提条件と不変条件

  • dlist フィールドは、ilist.h で定義される二重連結リストの規則に従います。
  • count は、リストへのノード追加・削除操作と同期して適切に更新される必要があります。
  • リストノードとして使用される構造体は、dlist_node を内部に保持している必要があります。

アルゴリズムまたは振る舞い

  • dlist_head をラップすることで、双方向の走査と定数時間でのノード削除をサポートしつつ、リストの長さを追跡します。
  • メモリ割り当ては行わず、呼び出し側が提供するメモリ領域を使用します。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/heap/rewriteheap.c
  • src/backend/access/transam/multixact.c
  • src/backend/postmaster/autovacuum.c
  • src/backend/storage/lmgr/deadlock.c
  • src/backend/storage/lmgr/lock.c
  • src/backend/storage/lmgr/proc.c
  • src/backend/utils/adt/ri_triggers.c
  • src/backend/utils/mmgr/slab.c
  • **呼び出し先**:
  • 構造体定義のため、特定の関数呼び出し先はありません。リスト操作を行う際は ilist.h で定義された dlist_* 系マクロや関数が利用されます。

関連データ構造

  • dlist_head: 本構造体の基盤となる二重連結リストのヘッダー。
  • dlist_node: リストに格納される各要素が内部に持つリンク構造体。

失敗モードと境界条件

  • リストが空の場合、dlist は空の状態を示し、count は 0 となります。
  • リストノードがリストに含まれている間、そのノードを含む構造体を free() するとメモリ破壊を引き起こします。

保守担当者向けメモ

  • count の更新は自動的には行われないため、リスト操作を行うコード内で整合性を維持する必要があります。
  • デバッグ時には ILIST_DEBUG を定義することで、リストの整合性チェックを有効にできます。
