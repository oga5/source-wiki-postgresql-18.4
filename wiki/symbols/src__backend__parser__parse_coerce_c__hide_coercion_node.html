hide_coercion_node

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_coerce.c:808

目的

指定された型変換ノードの表示形式（Coercion Format）を COERCE_IMPLICIT_CAST に強制的に変更し、明示的なキャストとして表示されないように隠蔽する。

シグネチャまたは形状

static void hide_coercion_node(Node *node)

入力と出力

  • **入力**: Node *node - 変換対象のノード。
  • **出力**: なし（副作用として入力ノードの内部フラグを更新）。

前提条件と不変条件

  • 入力 node は、サポートされている特定の型変換ノード（FuncExpr, RelabelType, CoerceViaIO, ArrayCoerceExpr, ConvertRowtypeExpr, RowExpr, CoerceToDomain のいずれか）である必要がある。

アルゴリズムまたは振る舞い

1. node の型を判定する。 2. 対応するノード型に応じて、以下のフィールドを COERCE_IMPLICIT_CAST に設定する。

  • FuncExpr: funcformat
  • RelabelType: relabelformat
  • CoerceViaIO: coerceformat
  • ArrayCoerceExpr: coerceformat
  • ConvertRowtypeExpr: convertformat
  • RowExpr: row_format
  • CoerceToDomain: coercionformat

    • 3. 上記のいずれにも該当しない場合、elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_coerce.c 内の関数（行690および765で呼び出しを確認）。
  • **呼び出し先**: elog（エラー発生時）。

関連データ構造

  • FuncExpr, RelabelType, CoerceViaIO, ArrayCoerceExpr, ConvertRowtypeExpr, RowExpr, CoerceToDomain などのノード構造体。

失敗モードと境界条件

  • サポートされていないノード型が渡された場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は、特定の変換ノードを「暗黙的」なものとして扱う必要がある場合にのみ使用すること。
  • 新しい変換ノード型が追加された場合は、本関数の if-else チェーンに該当する型を追加する必要がある。
