IsSystemClass

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/catalog.c:85

目的

指定されたリレーションがシステムカタログであるか、またはTOASTテーブルであるかを判定します。

シグネチャまたは形状

bool IsSystemClass(Oid relid, Form_pg_class reltuple)

入力と出力

  • **入力**:
  • relid (Oid): 対象リレーションのOID。
  • reltuple (Form_pg_class): 対象リレーションの pg_class 行データ。
  • **出力**:
  • bool: システムカタログまたはTOASTテーブルであれば true、それ以外は false を返します。

前提条件と不変条件

  • 本関数はカタログアクセスを行いません。
  • IsCatalogRelationOid を優先的に評価し、高速な判定を試みます。

アルゴリズムまたは振る舞い

1. IsCatalogRelationOid(relid) を呼び出し、OIDに基づいてシステムカタログかどうかを判定します。 2. 1が偽の場合、IsToastClass(reltuple) を呼び出し、TOASTテーブルかどうかを判定します。 3. いずれかが真であれば true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/aclchk.c
  • src/backend/catalog/catalog.c
  • src/backend/commands/cluster.c
  • src/backend/commands/indexcmds.c
  • src/backend/commands/policy.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • IsCatalogRelationOid
  • IsToastClass

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子。
  • Form_pg_class: pg_class システムカタログの行構造体。

失敗モードと境界条件

  • reltuple が無効なポインタである場合、IsToastClass の内部実装に依存して未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はカタログアクセスを発生させない設計であるため、ロック取得中や再帰的な呼び出しにおいても安全に使用可能です。
  • 判定ロジックを変更する場合は、IsCatalogRelationOid および IsToastClass の定義と整合性を保つ必要があります。
