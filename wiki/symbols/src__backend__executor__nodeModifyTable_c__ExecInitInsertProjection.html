ExecInitInsertProjection

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeModifyTable.c:639

目的

INSERT 操作において、サブプランの出力タプルをターゲットリレーションの形式に適合させるための投影（Projection）情報を初期化する。

シグネチャまたは形状

static void ExecInitInsertProjection(ModifyTableState *mtstate, ResultRelInfo *resultRelInfo)

入力と出力

  • **入力**:
  • mtstate: ModifyTable ノードの実行状態。
  • resultRelInfo: 挿入先リレーションの情報。
  • **出力**: なし（resultRelInfo 内の ri_newTupleSlot および ri_projectNew を更新）。

前提条件と不変条件

  • mtstate->ps.plan が ModifyTable ノードとしてキャスト可能であること。
  • resultRelInfo->ri_RelationDesc が有効であること。

アルゴリズムまたは振る舞い

1. サブプランのターゲットリストから resjunk 属性（ジャンク列）を除外したリストを作成する。 2. ジャンク列が存在する場合、投影が必要と判断する。 3. ExecCheckPlanOutput を呼び出し、ジャンク列を除いたリストがターゲットリレーションの形式と適合するか検証する。 4. table_slot_create を使用し、ターゲットリレーションの形式に合わせた新しいタプルスロットを作成し、resultRelInfo->ri_newTupleSlot に格納する。 5. 投影が必要な場合、ExecBuildProjectionInfo を用いて投影情報を構築し、resultRelInfo->ri_projectNew に設定する。この際、必要に応じて ExprContext を割り当てる。 6. 最後に resultRelInfo->ri_projectNewInfoValid を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeModifyTable.c 内の ModifyTable ノード初期化処理（ExecInitModifyTable 等から参照される）。
  • **呼び出し先**: ExecCheckPlanOutput, table_slot_create, ExecAssignExprContext, ExecBuildProjectionInfo。

関連データ構造

  • ModifyTableState: 実行ノードの状態管理。
  • ResultRelInfo: ターゲットリレーションごとの実行時情報。
  • Plan, TargetEntry: クエリプランおよびターゲットリストの定義。

失敗モードと境界条件

  • サブプランの出力形式がターゲットリレーションの期待する形式と一致しない場合、ExecCheckPlanOutput によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は INSERT 実行時のタプル変換を担う。投影が不要な場合（need_projection が false）は ri_projectNew は設定されない。
  • ri_projectNewInfoValid フラグは、この初期化が完了したことを示すために使用される。
