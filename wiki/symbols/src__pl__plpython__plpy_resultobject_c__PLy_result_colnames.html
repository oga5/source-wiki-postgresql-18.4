PLy_result_colnames

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_resultobject.c:139

目的

PLyResultObject が保持するクエリ結果の列名リストを、Pythonのリストオブジェクトとして取得する。

シグネチャまたは形状

static PyObject *PLy_result_colnames(PyObject *self, PyObject *unused)

入力と出力

  • **入力**:
  • self: PLyResultObject へのポインタ（Pythonオブジェクトとして渡される）。
  • unused: 使用されない引数（METH_NOARGS 呼び出し用）。
  • **出力**:
  • 成功時: 列名が格納された PyListObject へのポインタ。
  • 失敗時: NULL（例外が設定される）。

前提条件と不変条件

  • self は PLyResultObject 型である必要がある。
  • ob->tupdesc が有効（NULLでない）であること。NULLの場合、結果セットを持たないとみなされエラーとなる。

アルゴリズムまたは振る舞い

1. self を PLyResultObject にキャストする。 2. ob->tupdesc が存在するか確認し、存在しない場合は PLy_exc_error を発生させて NULL を返す。 3. ob->tupdesc->natts のサイズで新しい Python リストを作成する。 4. TupleDesc 内の各属性（Form_pg_attribute）を反復処理する。 5. 各属性の列名（attname）を PLyUnicode_FromString で Python 文字列に変換し、リストに格納する。 6. 完成したリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_resultobject.c 内のメソッド定義テーブル（PLyResult_spec）から、colnames メソッドとして参照されている。
  • **呼び出し先**:
  • PLy_exception_set (エラー発生時)
  • PyList_New
  • TupleDescAttr
  • PLyUnicode_FromString
  • NameStr
  • PyList_SetItem

関連データ構造

  • PLyResultObject: クエリ結果を保持するC構造体。
  • TupleDesc: PostgreSQLの行記述子。列名や型情報のソースとなる。

失敗モードと境界条件

  • ob->tupdesc が NULL の場合: PLy_exception_set により例外が送出される。
  • PyList_New がメモリ不足等で失敗した場合: NULL を返す。
  • PyList_SetItem が失敗した場合: 内部で適切に処理される。

保守担当者向けメモ

  • 本関数は Python の METH_NOARGS メソッドとして登録されている。
  • tupdesc はクエリ結果のメタデータであり、結果セットを伴わない実行（例: INSERT のみ）では NULL になる可能性があるため、NULL チェックは必須である。
