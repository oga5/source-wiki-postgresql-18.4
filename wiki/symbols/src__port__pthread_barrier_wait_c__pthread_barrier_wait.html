pthread_barrier_wait

定義

  • 種別: function
  • 定義箇所: src/port/pthread_barrier_wait.c:37

目的

pthread_barrier_t をネイティブサポートしないプラットフォームにおいて、指定された数のスレッドが到達するまで同期待機を行う。

シグネチャまたは形状

int pthread_barrier_wait(pthread_barrier_t *barrier);

入力と出力

  • **入力**: barrier (初期化済みの pthread_barrier_t 構造体へのポインタ)
  • **出力**:
  • 成功時: バリアを解除した最後の一つのスレッドには PTHREAD_BARRIER_SERIAL_THREAD を返し、それ以外のスレッドには 0 を返す。

前提条件と不変条件

  • barrier は事前に pthread_barrier_init で初期化されていること。
  • barrier->arrived は常に barrier->count 以下であること（Assert により検証）。

アルゴリズムまたは振る舞い

「センス反転（Sense Reversal）」アルゴリズムを採用し、バリアの再利用を効率化している。 1. ミューテックス barrier->mutex をロックし、barrier->arrived をインクリメントする。 2. 到達数が目標数に達した場合:

  • arrived を 0 にリセットし、sense を反転させる。
  • ミューテックスをアンロックし、pthread_cond_broadcast で待機中の全スレッドを起床させる。
  • PTHREAD_BARRIER_SERIAL_THREAD を返す。

    • 3. 到達数が目標数に達していない場合:

  • 現在の sense を記憶し、sense が変化するまで pthread_cond_wait で待機する。
  • 待機終了後、ミューテックスをアンロックし 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pgbench/pgbench.c において THREAD_BARRIER_WAIT マクロ経由で呼び出される。
  • **呼び出し先**: pthread_mutex_lock, pthread_mutex_unlock, pthread_cond_wait, pthread_cond_broadcast（いずれもプラットフォーム固有の pthread 実装に依存）。

関連データ構造

  • pthread_barrier_t: バリアの状態（到達数、目標数、条件変数、ミューテックス、センスフラグ）を保持する構造体。

失敗モードと境界条件

  • 内部で使用するミューテックスや条件変数の操作が失敗した場合の挙動は、基盤となる pthread 実装に依存する。
  • barrier が正しく初期化されていない場合、未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本実装は Windows 等の環境における移植性確保を目的としている。
  • PTHREAD_BARRIER_SERIAL_THREAD の定義については port/pg_pthread.h を参照のこと。
