stats_check_arg_pair

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/stat_utils.c:99

目的

統計情報の更新関数において、関連する2つの引数が「両方とも指定されている」か「両方とも省略されている（NULL）」かのいずれかであることを検証します。

シグネチャまたは形状

bool stats_check_arg_pair(FunctionCallInfo fcinfo, struct StatsArgInfo *arginfo, int argnum1, int argnum2)

入力と出力

  • **入力**:
  • fcinfo: 呼び出し元の関数情報。
  • arginfo: 引数名などのメタデータを含む StatsArgInfo 構造体の配列。
  • argnum1, argnum2: 検証対象となる2つの引数のインデックス。
  • **出力**:
  • true: 両方の引数がNULLであるか、両方が非NULLである場合。
  • false: 片方のみがNULLである場合。

前提条件と不変条件

  • argnum1 および argnum2 は、fcinfo 内の有効な引数インデックスである必要があります。
  • arginfo 配列は、指定されたインデックスに対応する引数名情報を提供できる必要があります。

アルゴリズムまたは振る舞い

1. PG_ARGISNULL を使用して、指定された2つの引数が両方ともNULLかどうかを確認します。両方NULLであれば true を返します。 2. 片方のみがNULLである場合、ereport(WARNING, ...) を発行し、不足している引数名をエラーメッセージに含めて false を返します。 3. 両方が非NULLである場合は true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/attribute_stats.c (行267, 274, 282)
  • **呼び出し先**:
  • ereport (PostgreSQLの標準エラー報告マクロ)
  • PG_ARGISNULL (PostgreSQLの引数チェックマクロ)

関連データ構造

  • FunctionCallInfo: 関数呼び出しの引数情報。
  • StatsArgInfo: 引数のメタデータ（引数名など）を保持する構造体。

失敗モードと境界条件

  • 片方の引数のみが指定された場合、WARNING レベルのログを出力し、処理を継続せずに false を返します。
  • 両方の引数がNULLの場合、検証は成功とみなされます。

保守担当者向けメモ

  • 本関数は統計更新のバリデーション層の一部です。エラー発生時に WARNING を発行するため、呼び出し側で戻り値を確認し、必要に応じて処理を中断する設計になっています。
