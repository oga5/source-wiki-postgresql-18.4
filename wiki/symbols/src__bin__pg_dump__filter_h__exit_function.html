exit_function

定義

  • 種別: typedef
  • 定義箇所: src/bin/pg_dump/filter.h:20

目的

フィルタ解析中にエラーが発生した際、呼び出し元のアプリケーション（pg_dump や pg_restore 等）が適切にリソース解放やログ出力を行い、終了処理を実行するためのコールバック関数型です。

シグネチャまたは形状

typedef void (*exit_function) (int status);

入力と出力

  • 入力: int status（終了ステータスコード）
  • 出力: なし（void）

前提条件と不変条件

  • filter_init を呼び出す際に、有効な関数ポインタとして登録される必要があります。
  • 呼び出される関数は、アプリケーション固有の終了処理を完遂し、プロセスを終了させる責任を持ちます。

アルゴリズムまたは振る舞い

フィルタ処理モジュールがエラーを検知した際、登録されたこの関数を呼び出すことで、モジュール内部のクリーンアップだけでなく、アプリケーション全体の整合性を保った終了を可能にします。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/filter.c 内の解析処理（エラー発生時）。
  • 呼び出し先: filter_init を呼び出す側のアプリケーション（pg_dump.c や pg_restore.c など）で定義された関数。

関連データ構造

  • FilterStateData: フィルタ解析状態を保持する構造体。filter_init を通じて exit_function が登録されます。

失敗モードと境界条件

  • 登録された関数ポインタが NULL の場合、呼び出し時に未定義の動作を引き起こす可能性があるため、初期化時に適切な関数が渡されていることが前提となります。

保守担当者向けメモ

  • 本モジュールは pg_dump と pg_restore の両方で利用されるため、本関数を実装する際は、両アプリケーションの終了要件（リソース解放の順序等）を考慮してください。
