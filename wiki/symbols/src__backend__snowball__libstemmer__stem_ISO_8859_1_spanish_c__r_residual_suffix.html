r_residual_suffix

定義

  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_spanish.c:940

目的

単語の末尾に残存する特定の接尾辞を検索し、条件（RV領域内であること等）を満たす場合に削除を行う。

シグネチャまたは形状

static int r_residual_suffix(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境を保持する構造体）
  • 出力: int（処理が成功した場合は 1、マッチしなかった場合は 0、エラー発生時は負の値を返す）

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • find_among_b を使用するため、z->c（カーソル位置）が単語の末尾付近にあることが前提となる。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置に設定する。 2. a_9 配列を用いて、単語末尾から接尾辞を後方検索（find_among_b）する。 3. マッチした among_var に応じて以下の処理を行う：

  • case 1: r_RV を呼び出し、RV領域内であれば slice_del で削除する。
  • case 2: r_RV を呼び出し、RV領域内であれば slice_del で削除する。さらに、直前に「ug」という文字列が存在するかを確認し、条件を満たせば追加で削除を行う。

    • 4. 処理完了後、成功値 1 を返す。

呼び出し元と呼び出し先

  • 呼び出し元: spanish_ISO_8859_1_stem（直接の呼び出し元であることはソース内のフローから示唆されるが、明示的なコールグラフは未解析）。
  • 呼び出し先: find_among_b, r_RV, slice_del。

関連データ構造

  • SN_env: ステミング環境を管理する構造体。
  • a_9: 接尾辞マッチングに使用される among 構造体の配列。

失敗モードと境界条件

  • マッチする接尾辞がない場合、0 を返す。
  • r_RV や slice_del がエラー（負の値）を返した場合、その値をそのまま伝播させる。
  • case 2 における「ug」の確認時に、バッファ境界（z->lb）や文字不一致が発生した場合は lab0 へジャンプし、追加削除をスキップする。
