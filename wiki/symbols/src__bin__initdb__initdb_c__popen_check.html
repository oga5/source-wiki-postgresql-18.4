popen_check

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/initdb.c:744

目的

標準ライブラリの popen をラップし、コマンド実行失敗時のエラーログ出力を一元化する。

シグネチャまたは形状

static FILE *popen_check(const char *command, const char *mode);

入力と出力

  • **入力**:
  • command: 実行するコマンド文字列。
  • mode: popen に渡すモード文字列（"r" または "w"）。
  • **出力**:
  • 成功時: コマンドのパイプストリームへのポインタ (FILE *)。
  • 失敗時: NULL。

前提条件と不変条件

  • popen を呼び出す前に、すべての出力ストリームをフラッシュするために fflush(NULL) が実行される。
  • 失敗時には pg_log_error を介してエラーメッセージが出力される。

アルゴリズムまたは振る舞い

1. fflush(NULL) を呼び出し、バッファリングされているすべてのストリームをフラッシュする。 2. errno を 0 にリセットする。 3. popen(command, mode) を実行する。 4. 戻り値が NULL の場合、pg_log_error を使用してコマンド実行失敗の旨とエラー詳細（%m）をログに出力する。 5. 取得したファイルポインタを呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/initdb.c 内の bootstrap_template1 等（行323付近で確認）。
  • **呼び出し先**: popen (標準ライブラリ), fflush (標準ライブラリ), pg_log_error (PostgreSQL 共通ユーティリティ)。

関連データ構造

  • FILE: 標準入出力ストリーム。

失敗モードと境界条件

  • popen が失敗した場合、NULL が返される。この際、pg_log_error によりエラーが記録されるため、呼び出し元は戻り値が NULL であることを確認し、必要に応じて終了処理を行う必要がある。

保守担当者向けメモ

  • 本関数は initdb プロセス内でのコマンド実行の安全性を高めるためのラッパーである。
  • 呼び出し元で exit(1) を行う際は、本関数が既にエラーメッセージを出力済みであることを前提としている。
