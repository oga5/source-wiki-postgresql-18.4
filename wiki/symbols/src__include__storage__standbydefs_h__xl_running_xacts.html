xl_running_xacts

定義

  • 種別: struct
  • 定義箇所: src/include/storage/standbydefs.h:47

目的

ホットスタンバイモードにおいて、プライマリサーバー上の実行中トランザクション情報をスタンバイサーバーへ伝播し、スタンバイ側でスナップショットを構築・維持するために使用されます。

シグネチャまたは形状

typedef struct xl_running_xacts
{
    int         xcnt;           /* xids[] 内のトランザクションID数 */
    int         subxcnt;        /* xids[] 内のサブトランザクションID数 */
    bool        subxid_overflow; /* スナップショット溢れ発生フラグ */
    TransactionId nextXid;      /* 次のトランザクションID */
    TransactionId oldestRunningXid; /* 最古の実行中トランザクションID */
    TransactionId latestCompletedXid; /* 最新の完了済みトランザクションID */
    TransactionId xids[FLEXIBLE_ARRAY_MEMBER]; /* 可変長配列 */
} xl_running_xacts;

入力と出力

  • **入力**: プライマリ側で生成された実行中トランザクションの状態。
  • **出力**: WAL レコードとして書き出され、スタンバイ側で読み込まれることで ProcArray の状態更新に使用されます。

前提条件と不変条件

  • FLEXIBLE_ARRAY_MEMBER を使用した可変長構造体であるため、メモリレイアウトを扱う際はオフセット計算用マクロ（MinSizeOfInvalidations 等）が必要です。
  • oldestRunningXid は oldestXmin とは異なります。

アルゴリズムまたは振る舞い

1. **記録**: プライマリ側で定期的なチェックポイントやスタンバイ起動時に生成され、WAL レコードとして出力されます。 2. **再生**: スタンバイ側で XLogReader がレコードを読み込み、standby_redo 関数がこの構造体を参照してスタンバイ側の共有メモリ状態を更新します。 3. **整合性**: スタンバイ側の ProcArray に反映されることで、ホットスタンバイの整合性が維持されます。

呼び出し元と呼び出し先

  • **呼び出し元（生成・参照）**:
  • src/backend/postmaster/bgwriter.c: 定期的な WAL 記録。
  • src/backend/replication/logical/snapbuild.c: スナップショット構築および整合性確認。
  • **呼び出し先（処理）**:
  • src/backend/access/rmgrdesc/standbydesc.c: standby_desc_running_xacts によるデバッグ出力。
  • src/backend/replication/logical/snapbuild.c: SnapBuildFindSnapshot, SnapBuildWaitSnapshot, SnapBuildProcessRunningXacts による状態遷移。

関連データ構造

  • TransactionId: トランザクション識別子。
  • ProcArray: スタンバイ側で更新されるトランザクション管理用共有メモリ構造。

失敗モードと境界条件

  • subxid_overflow が true の場合、サブトランザクション情報が欠落していることを示します。
  • 可変長配列 xids のサイズは xcnt および subxcnt に依存するため、メモリ確保およびアクセス時に境界チェックが必要です。

保守担当者向けメモ

  • 本構造体はホットスタンバイの整合性に直結します。変更時は standby_redo および SnapBuild 関連のロジックへの影響を考慮してください。
  • FLEXIBLE_ARRAY_MEMBER を含むため、構造体のサイズ計算には注意が必要です。
