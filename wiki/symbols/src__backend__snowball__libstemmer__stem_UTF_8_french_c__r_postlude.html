r_postlude

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/snowball/libstemmer/stem_UTF_8_french.c:610

目的

ステミング処理の最終段階として、単語の末尾に残存する特定の文字パターンやアクセント記号の置換・削除を行い、語幹を正規化する。

シグネチャまたは形状

static int r_postlude(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（ステミング環境構造体。現在の文字列バッファ p、カーソル位置 c、文字列長 l を含む）。
  • **出力**: 成功時に 1 を返し、メモリ操作等の失敗時に負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • z->c は処理対象の文字列内を指すカーソルとして機能する。

アルゴリズムまたは振る舞い

1. 文字列の末尾に向かってループ処理を実行する。 2. find_among を使用して、定義された配列 a_1 に基づき、現在のカーソル位置からマッチする接尾辞を検索する。 3. マッチしたパターンに応じて以下の処理を行う：

  • case 1～5: slice_from_s を呼び出し、該当部分を特定の文字列に置換する。
  • case 6: slice_del を呼び出し、該当部分を削除する。
  • case 7: マッチしない場合、skip_utf8 を用いてカーソルを1文字分進める。

    • 4. 処理が完了するまでループを継続し、最終的に 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: french_UTF_8_stem（直接の呼び出し関係はソースコードの制御フローより推論）。
  • **呼び出し先**: find_among、slice_from_s、slice_del、skip_utf8。
  • **注記**: 他の言語のステマー（stem_ISO_8859_1_dutch.c 等）にも同名の関数が存在するが、本ページは stem_UTF_8_french.c 内の定義に限定している。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_1: among 構造体の配列（置換・削除対象のパターン定義）。

失敗モードと境界条件

  • slice_from_s や slice_del がメモリ不足等の理由で負の値を返した場合、その値をそのまま呼び出し元へ伝播させる。
  • 文字列の終端に達した際、skip_utf8 が失敗すると lab0 ラベルへジャンプし、ループを終了する。
