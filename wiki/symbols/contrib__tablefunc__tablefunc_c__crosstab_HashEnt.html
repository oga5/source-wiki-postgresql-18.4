crosstab_HashEnt

定義

  • 種別: struct
  • 定義箇所: contrib/tablefunc/tablefunc.c:161

目的

crosstab_hash 関数において、カテゴリ名とそれに対応する属性記述子（crosstab_cat_desc）をマッピングするためのハッシュテーブルエントリを保持します。

シグネチャまたは形状

typedef struct crosstab_hashent
{
    char        internal_catname[MAX_CATNAME_LEN];
    crosstab_cat_desc *catdesc;
} crosstab_HashEnt;

入力と出力

  • **入力**: 構造体メンバとして internal_catname（カテゴリ名）および catdesc（crosstab_cat_desc へのポインタ）を保持します。
  • **出力**: ハッシュテーブル検索時のエントリとして利用されます。

前提条件と不変条件

  • internal_catname は MAX_CATNAME_LEN の長さ制限に従います。
  • ハッシュテーブルの初期化時に sizeof(crosstab_HashEnt) がエントリサイズとして指定されている必要があります。

アルゴリズムまたは振る舞い

  • hash_search を用いたハッシュテーブル操作において、キーとなるカテゴリ名と、それに関連付けられたメタデータ（catdesc）を紐付けるためのコンテナとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/tablefunc/tablefunc.c 内のハッシュテーブル検索処理（hash_search）にて参照されています。
  • **呼び出し先**: 構造体内部で crosstab_cat_desc を参照しています。

関連データ構造

  • crosstab_cat_desc: カテゴリの属性記述を保持する構造体。
  • HASHTAB: 本構造体をエントリとして格納するハッシュテーブル。

失敗モードと境界条件

  • internal_catname が MAX_CATNAME_LEN を超える場合、バッファオーバーフローのリスクがあります。
  • ハッシュテーブルのメモリ割り当てに失敗した場合、hash_search がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本構造体は crosstab_hash の実行コンテキスト内でのみ使用されます。
  • 構造体のサイズ変更を行う場合は、ctl.entrysize = sizeof(crosstab_HashEnt) を設定している箇所（contrib/tablefunc/tablefunc.c:715）との整合性に注意してください。
