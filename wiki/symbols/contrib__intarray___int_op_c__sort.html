sort

定義

  • 種別: function
  • 定義箇所: contrib/intarray/_int_op.c:197

目的

整数配列を昇順または降順にソートし、結果を新しい配列として返します。

シグネチャまたは形状

Datum sort(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: ソート対象の ArrayType（PG_GETARG_ARRAYTYPE_P_COPY によりコピーを取得）。
  • 第2引数（任意）: ソート順序を指定する text 型（"ASC" または "DESC"）。
  • **出力**: ソート済みの ArrayType を Datum として返却。

前提条件と不変条件

  • CHECKARRVALID(a) マクロにより、入力配列の整合性が検証されます。
  • 配列の要素数が2未満の場合、ソートを行わず入力配列をそのまま返します。
  • 第2引数が指定された場合、"ASC"（昇順）または "DESC"（降順）のいずれかである必要があります。それ以外の場合はエラーとなります。

アルゴリズムまたは振る舞い

1. 入力配列のコピーを作成します。 2. 第2引数が省略された場合はデフォルトで昇順（dir = 1）とみなします。 3. 第2引数の文字列を解析し、"ASC" なら昇順、"DESC" なら降順（dir = 0）を設定します。 4. QSORT マクロを使用して配列をソートします。 5. ソート済みの配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出されます。ソースコード内の直接的な呼び出し元は、提供された情報からは特定できません。
  • **呼び出し先**:
  • CHECKARRVALID (マクロ)
  • QSORT (マクロ)
  • ereport (エラー報告)

関連データ構造

  • ArrayType: PostgreSQLの配列データ型。
  • text: ソート順序指定のための文字列型。

失敗モードと境界条件

  • **無効なパラメータ**: 第2引数に "ASC" または "DESC" 以外の文字列が渡された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーが発生します。
  • **配列の整合性**: CHECKARRVALID が失敗した場合、処理は中断されます。

保守担当者向けメモ

  • 本関数は入力配列をコピーして操作するため、元の配列データは変更されません。
  • QSORT マクロの実装は _int.h に依存しています。
  • 呼び出し元情報の詳細は、現在のソース解析範囲では特定できていません。
