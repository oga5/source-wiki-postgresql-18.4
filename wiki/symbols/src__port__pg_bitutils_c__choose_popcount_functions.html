choose_popcount_functions

定義

  • 種別: function
  • 定義箇所: src/port/pg_bitutils.c:160

目的

実行時のCPU機能判定に基づき、ポピュレーションカウント（ビット数カウント）処理に最適な実装関数を選択し、関数ポインタを更新する。

シグネチャまたは形状

void choose_popcount_functions(void)

入力と出力

  • 入力: なし（内部的なCPU機能フラグを参照）
  • 出力: なし（副作用としてグローバルな関数ポインタを書き換える）

前提条件と不変条件

  • 本関数は、ポピュレーションカウント関連の関数が最初に呼び出された際に実行される必要がある。
  • pg_popcount_available の戻り値に基づき、ハードウェア命令（POPCNT等）が利用可能かどうかの判定が行われる。

アルゴリズムまたは振る舞い

1. pg_popcount_available() を呼び出し、CPUが POPCNT 命令をサポートしているか判定する。 2. サポートされている場合、pg_popcount32、pg_popcount64、pg_popcount_optimized、pg_popcount_masked_optimized の各関数ポインタを _fast 接尾辞を持つ高速なアセンブラ実装へ割り当てる。 3. サポートされていない場合、上記ポインタを _slow 接尾辞を持つ汎用的な実装へ割り当てる。 4. USE_AVX512_POPCNT_WITH_RUNTIME_CHECK が定義されている環境では、追加で pg_popcount_avx512_available() を確認し、利用可能であれば AVX-512 用の実装へポインタを上書きする。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/pg_bitutils.c 内の各 _choose 関数、および src/port/pg_popcount_aarch64.c 内の初期化処理から参照されている。
  • 呼び出し先: pg_popcount_available()、および USE_AVX512_POPCNT_WITH_RUNTIME_CHECK 定義時は pg_popcount_avx512_available() を呼び出す。

関連データ構造

  • pg_popcount32, pg_popcount64, pg_popcount_optimized, pg_popcount_masked_optimized（関数ポインタ変数）

失敗モードと境界条件

  • CPU機能判定が正しく行えない環境では、フォールバックとして _slow 実装が選択される。
  • コンパイル時のマクロ定義（USE_AVX512_POPCNT_WITH_RUNTIME_CHECK 等）により、実行されるコードパスが変化する。

保守担当者向けメモ

  • 本関数は動的ディスパッチの起点である。新しいハードウェア命令セットに対応する実装を追加する際は、本関数内の条件分岐を拡張する必要がある。
