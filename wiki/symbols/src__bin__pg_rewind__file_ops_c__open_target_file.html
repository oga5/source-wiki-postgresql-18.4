open_target_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:46

目的

ターゲットデータディレクトリ内のファイルを書き込み用に開き、書き込みセッションを管理します。

シグネチャまたは形状

void open_target_file(const char *path, bool trunc)

入力と出力

  • path: 開く対象のファイルパス（ターゲットディレクトリからの相対パス）。
  • trunc: true の場合、ファイルを開く際に O_TRUNC を適用して内容を切り詰めます。
  • 出力: なし（成功時はグローバル変数 dstfd にファイル記述子が格納されます）。

前提条件と不変条件

  • path_is_safe_for_extraction により、指定されたパスがターゲットディレクトリ外へ脱出しないことが保証されている必要があります。
  • dry_run が有効な場合、ファイルシステムへの変更は行われません。

アルゴリズムまたは振る舞い

1. パスが安全か検証し、安全でない場合は pg_fatal で終了します。 2. dry_run モードであれば即座に復帰します。 3. すでに同じファイルが dstfd で開かれている場合、かつ trunc が不要であれば、そのまま処理を終了します。 4. 既存のファイルが開かれている場合は close_target_file() を呼び出して閉じます。 5. 新しいパスを構築し、O_WRONLY | O_CREAT | PG_BINARY モードでファイルを開きます。trunc が指定されていれば O_TRUNC を追加します。 6. 開くことに失敗した場合は pg_fatal で終了します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_rewind/libpq_source.c
  • src/bin/pg_rewind/local_source.c
  • src/bin/pg_rewind/pg_rewind.c
  • 呼び出し先:
  • path_is_safe_for_extraction
  • close_target_file
  • pg_fatal
  • open (システムコール)

関連データ構造

  • dstfd: 現在開いているターゲットファイルのファイル記述子（グローバル変数）。
  • dstpath: 現在開いているターゲットファイルのパス（グローバル変数）。
  • datadir_target: ターゲットデータディレクトリのベースパス（グローバル変数）。

失敗モードと境界条件

  • パスが安全でない場合: pg_fatal によりプロセスが終了します。
  • ファイルオープン失敗時: pg_fatal によりプロセスが終了します。

保守担当者向けメモ

  • 本関数は pg_rewind における書き込みセッションの起点です。
  • dstfd を介した状態管理を行っているため、呼び出し側で明示的に close_target_file を呼ぶ必要はありませんが、セッションの切り替えは本関数が自動的に行います。
