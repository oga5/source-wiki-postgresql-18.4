pg_ndistinct_recv

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mvdistinct.c:391

目的

pg_ndistinct 型のバイナリ形式での入力（受信）を処理する関数ですが、本型は内部的なバイナリ表現でありユーザーによる直接操作を想定していないため、明示的にエラーを発生させることを目的とします。

シグネチャまたは形状

Datum pg_ndistinct_recv(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（標準的なPostgreSQL関数呼び出し引数）
  • 出力: なし（常にエラーをスローして終了する）

前提条件と不変条件

  • 本関数は pg_ndistinct 型の受信関数として登録されていますが、実行時には常に ERRCODE_FEATURE_NOT_SUPPORTED を伴うエラーが発生します。

アルゴリズムまたは振る舞い

1. ereport を呼び出し、ERRCODE_FEATURE_NOT_SUPPORTED エラーを発生させます。 2. エラーメッセージとして「cannot accept a value of type pg_ndistinct」を出力します。 3. PG_RETURN_VOID() を呼び出しますが、実際にはエラーにより制御が戻ることはありません。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLの型システム（pg_type カタログ経由で受信関数として呼び出される可能性があります）。
  • 呼び出し先: ereport（PostgreSQLのエラー報告メカニズム）。

関連データ構造

  • pg_ndistinct: 本関数が対象とする内部統計用データ型。

失敗モードと境界条件

  • 本関数は呼び出された時点で常に失敗（エラー発生）するように設計されています。

保守担当者向けメモ

  • 本関数は pg_ndistinct 型の入出力インターフェースを埋めるためのプレースホルダーであり、機能的な実装は意図的に行われていません。将来的にこの型をユーザーが直接操作可能にする必要がない限り、変更は不要です。
