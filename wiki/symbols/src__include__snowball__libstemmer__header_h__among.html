among

定義

  • 種別: struct
  • 定義箇所: src/include/snowball/libstemmer/header.h:15

目的

Snowball ステミングアルゴリズムにおける辞書引きやパターンマッチングのための検索対象リストを表現する構造体です。

シグネチャまたは形状

struct among {
    int s_size;                 /* 文字列の文字数 */
    const symbol * s;           /* 検索対象文字列 */
    int substring_i;            /* 最長一致部分文字列へのインデックス */
    int result;                 /* ルックアップの結果 */
    int (* function)(struct SN_env *); /* 一致時に実行される関数ポインタ */
};

入力と出力

  • 入力: find_among または find_among_b 関数に対し、この構造体の配列が検索対象として渡されます。
  • 出力: マッチング成功時に result フィールドの値が返されるか、指定された function が実行されます。

前提条件と不変条件

  • s は symbol 型の文字列を指す必要があります。
  • function は SN_env 構造体へのポインタを引数に取る関数である必要があります。

アルゴリズムまたは振る舞い

1. find_among（または find_among_b）が、現在のステミング環境（SN_env）内の文字列と among 配列を照合します。 2. 文字列が一致した場合、構造体内の function ポインタが呼び出されます。 3. 処理結果として result が返されます。

呼び出し元と呼び出し先

  • 呼び出し元: 各言語固有のステミング実装（stem_*.h ファイル群）が、この構造体の配列を定義し、検索関数に渡します。
  • 呼び出し先: among 構造体内の function ポインタは、ステミング環境（SN_env）を操作する関数を指します。
  • 注記: ツールが検出した contrib/ や src/backend/ 内の「among」という単語を含むコードは、本構造体とは無関係な自然言語としての使用例であり、本シンボルの呼び出し元ではありません。

関連データ構造

  • symbol: ステミング処理で用いられる文字列型。
  • SN_env: ステミングアルゴリズムの実行環境を保持する構造体。

失敗モードと境界条件

  • 検索対象の文字列が among 配列内に存在しない場合、マッチングは失敗します。
  • function ポインタが NULL の場合、一致しても関数呼び出しは行われません。

保守担当者向けメモ

  • 本構造体は libstemmer の中核となる検索ロジックで使用されます。変更時は find_among および find_among_b の実装への影響を確認してください。
