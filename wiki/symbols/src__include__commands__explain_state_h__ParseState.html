ParseState

定義

  • 種別: struct
  • 定義箇所: src/include/commands/explain_state.h:82（注: parser/parse_node.h:23 にも同名の定義が存在します）

目的

SQL解析プロセスにおいて、クエリの構文解析および意味解析の過程で生成される状態情報を保持します。EXPLAIN コマンド実行時には、オプションの検証処理においてコンテキストとして利用されます。

シグネチャまたは形状

src/include/commands/explain_state.h 内では、主に explain_validate_options_hook_type の引数として前方宣言的に参照されています。

入力と出力

  • **入力**: 解析対象のクエリ情報や、EXPLAIN コマンドのオプションリストと組み合わせて使用されます。
  • **出力**: 解析結果の反映や、構文解析器が後続の処理へ引き継ぐ状態情報として機能します。

前提条件と不変条件

  • EXPLAIN コマンドの実行時、オプション検証フック（explain_validate_options_hook_type）に渡される際、有効な ParseState インスタンスが提供されている必要があります。

アルゴリズムまたは振る舞い

  • EXPLAIN コマンドのオプション検証処理において、ユーザーが指定したオプションが現在のクエリコンテキスト（ParseState）において妥当であるかを判定するための参照情報として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_overexplain などの拡張機能や、src/backend/catalog/heap.c 等のカタログ操作関数から、解析コンテキストとして渡されます。
  • **呼び出し先**: 本構造体自体はデータ保持を目的としており、特定の関数を呼び出すものではありません。

関連データ構造

  • ExplainState: EXPLAIN コマンドの実行状態を管理する構造体。
  • List: 解析オプションの保持に使用。
  • DefElem: オプション定義の保持に使用。

失敗モードと境界条件

  • ParseState が NULL の状態で解析処理に渡された場合、後続の transformExpr 等の処理でヌルポインタ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は parser/parse_node.h で定義されているものと同一の型を指すことが想定されています。explain_state.h 内での参照はあくまで EXPLAIN コマンドのフック定義のためのものです。修正を行う際は、パーサー側の定義との整合性に注意してください。
