RenameTableSpace

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tablespace.c:929

目的

既存のテーブルスペースの名前を変更する。

シグネチャまたは形状

ObjectAddress RenameTableSpace(const char *oldname, const char *newname)

入力と出力

  • **入力**:
  • oldname: 変更対象の既存テーブルスペース名。
  • newname: 新しいテーブルスペース名。
  • **出力**:
  • 変更されたテーブルスペースの ObjectAddress を返す。

前提条件と不変条件

  • 呼び出し元は適切なトランザクションコンテキストにある必要がある。
  • oldname で指定されたテーブルスペースが存在しなければならない。
  • 呼び出し元は対象テーブルスペースの所有者でなければならない。
  • newname は既存のテーブルスペース名と重複してはならない。
  • allowSystemTableMods が無効な場合、newname に pg_ で始まる予約語を使用することはできない。

アルゴリズムまたは振る舞い

1. pg_tablespace システムカタログを RowExclusiveLock で開く。 2. oldname をキーにスキャンを行い、対象のテーブルスペースを特定する。存在しない場合はエラーを投げる。 3. object_ownercheck を呼び出し、現在のユーザーが所有者であることを確認する。 4. newname の妥当性を検証し、既存のテーブルスペース名と重複していないか確認する。 5. pg_tablespace の該当タプルを更新し、新しい名前を反映させる。 6. InvokeObjectPostAlterHook を呼び出し、変更を通知する。 7. 最後に ObjectAddress を構築して返し、テーブルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/alter.c:391
  • **呼び出し先**:
  • table_open, table_beginscan_catalog, table_endscan, table_close
  • ScanKeyInit, heap_getnext, heap_copytuple, CatalogTupleUpdate
  • object_ownercheck, IsReservedName, InvokeObjectPostAlterHook, ObjectAddressSet

関連データ構造

  • pg_tablespace システムカタログ
  • ObjectAddress
  • HeapTuple

失敗モードと境界条件

  • 指定した oldname が存在しない場合、ERRCODE_UNDEFINED_OBJECT エラー。
  • ユーザーが所有者でない場合、aclcheck_error によりエラー。
  • newname が予約語（pg_）で、システム変更が許可されていない場合、ERRCODE_RESERVED_NAME エラー。
  • newname が既に存在する場合、ERRCODE_DUPLICATE_OBJECT エラー。

保守担当者向けメモ

  • ENFORCE_REGRESSION_TEST_NAME_RESTRICTIONS が定義されている場合、regress_ で始まらない名前に対して警告を発する。
  • 物理的なディレクトリ名の変更は行わず、カタログ上の名前変更のみを扱う。
