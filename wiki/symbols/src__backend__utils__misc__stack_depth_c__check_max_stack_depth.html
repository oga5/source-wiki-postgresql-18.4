check_max_stack_depth

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/stack_depth.c:141

目的

GUCパラメータ max_stack_depth の設定値が、OSが規定するスタックリソース制限（RLIMIT_STACK）を超過していないかを検証する。

シグネチャまたは形状

bool check_max_stack_depth(int *newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定しようとしている新しいスタック深さ（KB単位）。
  • extra, source: GUCフックの標準引数。
  • **出力**:
  • true: 設定値が許容範囲内である場合。
  • false: 設定値がOSの制限を超過している場合。

前提条件と不変条件

  • get_stack_depth_rlimit() がOSから有効なスタック制限値を取得できること。
  • STACK_DEPTH_SLOP（安全のためのマージン）が考慮されること。

アルゴリズムまたは振る舞い

1. newval をバイト単位に変換する。 2. get_stack_depth_rlimit() を呼び出し、OSの物理的なスタック制限を取得する。 3. newval_bytes が stack_rlimit - STACK_DEPTH_SLOP を超える場合、エラー詳細とヒント（ulimit -s の調整推奨）をGUCエラー報告機構経由で出力し、false を返す。 4. 上記以外の場合は true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc_tables.c:2628（GUC設定のフックとして登録されている）。
  • **呼び出し先**: get_stack_depth_rlimit()。

関連データ構造

  • max_stack_depth: 本関数で検証対象となるグローバル設定値。

失敗モードと境界条件

  • OSのスタック制限が取得できない（stack_rlimit <= 0）場合、検証はスキップされ true を返す。
  • 設定値がOSの制限値に対してマージン（STACK_DEPTH_SLOP）を考慮した範囲内に収まらない場合、設定変更が拒否される。

保守担当者向けメモ

  • 本関数はGUCのチェックフックとして機能するため、設定変更のトランザクション内で実行される。
  • STACK_DEPTH_SLOP は、スタックオーバーフローを確実に防ぐためのバッファとして機能する。
