btrim

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:358

目的

入力文字列の先頭および末尾から、指定された文字セットに含まれる文字をすべて除去した新しい文字列を返します。Oracle データベースの BTRIM 関数との互換性を提供します。

シグネチャまたは形状

Datum btrim(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TEXT_PP(0): 対象となる文字列 (text 型)。
  • PG_GETARG_TEXT_PP(1): 除去対象の文字セット (text 型)。
  • **出力**:
  • PG_RETURN_TEXT_P(ret): 除去処理後の新しい text 型データ。

前提条件と不変条件

  • 入力引数は PostgreSQL の text 型として渡される必要があります。
  • 内部的に dotrim を呼び出すため、マルチバイト文字の境界を正しく認識した処理が行われます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP マクロを使用して、入力文字列と除去対象セットを text 構造体として取得します。 2. dotrim 関数を呼び出し、以下の引数を渡してトリミングを実行します。

  • 文字列のデータポインタとサイズ。
  • 除去セットのデータポインタとサイズ。
  • 先頭除去フラグ (true) および末尾除去フラグ (true)。

    • 3. dotrim が返した結果を PG_RETURN_TEXT_P を介して呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出されます（例: src/bin/psql/describe.c 等からの参照を確認）。
  • **呼び出し先**: dotrim (共通実装関数)。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ表現型。

失敗モードと境界条件

  • 入力文字列または除去セットが NULL の場合、PostgreSQL の関数呼び出し規約に従い、通常は NULL が返されます。
  • メモリ不足時には palloc がエラーを発生させます。
  • 除去セットが空の場合、元の文字列がそのまま返されます。

保守担当者向けメモ

  • 本関数は dotrim のラッパーです。ロジックの変更が必要な場合は、まず dotrim の実装を確認してください。
  • マルチバイト文字の処理については、src/backend/utils/adt/oracle_compat.c の概要にある通り、pg_mblen_range を活用した走査ロジックに依存しています。
