pgp_get_keyid

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-info.c:111

目的

PGP データパケットストリームを解析し、暗号化に使用された鍵の識別子（Key ID）を特定して文字列として取得します。

シグネチャまたは形状

int pgp_get_keyid(MBuf *pgp_data, char *dst)

入力と出力

  • **入力**:
  • pgp_data: 解析対象の PGP データを含む MBuf。
  • dst: 結果の Key ID 文字列を格納するバッファ。
  • **出力**:
  • 成功時: 書き込まれた文字列の長さ（バイト数）。
  • 失敗時: 負の整数（エラーコード）。

前提条件と不変条件

  • dst 引数には、Key ID 文字列（16 文字）と終端文字を格納できる 17 バイト以上のメモリ領域が必要です。
  • 複数の鍵が含まれる場合、整合性チェックによりエラーとなる可能性があります。

アルゴリズムまたは振る舞い

1. MBuf からストリームを生成し、パケットヘッダーを順次読み取ります。 2. パケットタグを識別し、以下の処理を行います。

  • 公開鍵/秘密鍵パケット: 最初のメイン鍵はスキップし、それ以降はエラーとします。
  • 公開鍵/秘密鍵サブキーパケット: read_pubkey_keyid を呼び出し Key ID を抽出します。
  • 公開鍵暗号化セッション鍵パケット: read_pubenc_keyid を呼び出し Key ID を抽出します。
  • 暗号化データパケット: 読み取りを終了します。
  • その他: 署名やユーザー ID 等の関連パケットはスキップします。

    • 3. 抽出された Key ID が any_key と一致する場合は "ANYKEY" を、対称鍵暗号の場合は "SYMKEY" を dst に書き込みます。それ以外の場合は print_key を使用して 16 進数文字列に変換します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgcrypto/pgp-pgsql.c:995
  • **呼び出し先**:
  • read_pubkey_keyid
  • read_pubenc_keyid
  • print_key
  • pgp_parse_pkt_hdr
  • pgp_create_pkt_reader
  • pgp_skip_packet
  • pullf_create_mbuf_reader
  • pullf_free

関連データ構造

  • MBuf: PGP データのメモリバッファ。
  • PullFilter: ストリーム読み込み用フィルタ。

失敗モードと境界条件

  • PXE_PGP_MULTIPLE_KEYS: 複数の鍵が混在している場合に発生。
  • PXE_PGP_CORRUPT_DATA: データが破損している、または公開鍵と公開鍵暗号化セッション鍵が混在している場合に発生。
  • PXE_PGP_NO_USABLE_KEY: 使用可能な鍵が見つからない場合に発生。
  • pullf_create_mbuf_reader の失敗時、そのエラーコードを返します。

保守担当者向けメモ

  • 本関数は暗号化や復号そのものは行わず、メタデータの抽出のみを行います。
  • any_key はワイルドカード的な Key ID を表す定数として扱われます。
