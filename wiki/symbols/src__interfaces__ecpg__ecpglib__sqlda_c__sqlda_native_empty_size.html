sqlda_native_empty_size

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/sqlda.c:170

目的

ECPGネイティブモードにおいて、PGresult から構築される SQLDA 構造体のメタデータ領域および初期オフセットに必要なメモリサイズを算出します。

シグネチャまたは形状

static long sqlda_native_empty_size(const PGresult *res)

入力と出力

  • **入力**: const PGresult *res（SQLクエリの結果セット）
  • **出力**: long 型のバイトサイズ。構造体本体、フィールド定義配列、および最初のデータフィールド用のアライメント調整済みオフセットの合計値を返します。

前提条件と不変条件

  • res は PQnfields を呼び出し可能な有効な PGresult ポインタである必要があります。
  • 戻り値は、ecpg_sqlda_align_add_size によるアライメント調整を含んだ値となります。

アルゴリズムまたは振る舞い

1. PQnfields(res) を呼び出し、結果セットの列数（sqld）を取得します。 2. sqlda_struct のサイズと、列数分（sqld - 1）の sqlvar_struct のサイズを加算して初期オフセットを算出します。 3. ecpg_sqlda_align_add_size を呼び出し、最初のデータフィールド配置のために int 型のアライメントを考慮したパディングをオフセットに加算します。 4. 最終的なオフセット値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/sqlda.c 内の関数（ecpg_build_native_sqlda 等）から呼び出されています。
  • **呼び出し先**:
  • PQnfields (libpq)
  • ecpg_sqlda_align_add_size (内部ユーティリティ)

関連データ構造

  • sqlda_struct: ネイティブモードの SQLDA メイン構造体。
  • sqlvar_struct: 各列のメタデータを保持する構造体。
  • PGresult: libpq のクエリ結果保持構造体。

失敗モードと境界条件

  • res が NULL の場合、PQnfields の挙動に依存します。
  • 列数が 0 の場合でも、構造体自体のサイズ計算は行われます。

保守担当者向けメモ

  • 本関数はネイティブモード専用です。Informix 互換モードについては sqlda_compat_empty_size を参照してください。
  • メモリレイアウトの変更（構造体のメンバ追加等）を行った場合は、sizeof の計算式を更新する必要があります。
