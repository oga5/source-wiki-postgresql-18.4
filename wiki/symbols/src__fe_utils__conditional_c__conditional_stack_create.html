conditional_stack_create

定義

  • 種別: function
  • 定義箇所: src/fe_utils/conditional.c:17

目的

ネストされた \if 文の制御フローを管理するための、新しい空の ConditionalStack 構造体を生成・初期化します。

シグネチャまたは形状

ConditionalStack conditional_stack_create(void)

入力と出力

  • **入力**: なし
  • **出力**: 新たに割り当てられた ConditionalStack 構造体へのポインタ。

前提条件と不変条件

  • pg_malloc を使用してメモリを確保するため、メモリ不足時にはプロセスが終了します。
  • 生成されたスタックの head は NULL に初期化され、空の状態であることが保証されます。

アルゴリズムまたは振る舞い

1. ConditionalStackData 構造体のサイズ分のメモリを pg_malloc で確保します。 2. 確保した構造体の head メンバを NULL に設定します。 3. 初期化済みのスタックポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pgbench/pgbench.c (複数箇所)
  • src/bin/psql/mainloop.c
  • src/bin/psql/startup.c
  • **呼び出し先**:
  • pg_malloc (メモリ確保用)

関連データ構造

  • ConditionalStack: スタック全体を管理する構造体（ConditionalStackData へのポインタ）。
  • fe_utils/conditional.h: 本関数および関連するスタック構造体の定義元。

失敗モードと境界条件

  • メモリ不足時には pg_malloc がエラーを発生させ、呼び出し元へは戻りません。

保守担当者向けメモ

  • 本関数で生成されたスタックは、使用後に conditional_stack_destroy を呼び出してメモリを解放する必要があります。
  • スタックの要素（ノード）は本関数では生成されず、後続の conditional_stack_push 等によって追加されます。
