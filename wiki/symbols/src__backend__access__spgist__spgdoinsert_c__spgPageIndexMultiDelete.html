spgPageIndexMultiDelete

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgdoinsert.c:130

目的

SP-GiSTインデックスページ内の指定された複数のタプルを削除し、それらをリダイレクトまたはプレースホルダーとして再配置することで、ページ内の整合性を維持します。

シグネチャまたは形状

void spgPageIndexMultiDelete(SpGistState *state, Page page, OffsetNumber *itemnos, int nitems, int firststate, int reststate, BlockNumber blkno, OffsetNumber offnum)

入力と出力

  • state: SP-GiSTのインデックス状態を保持する構造体。
  • page: 操作対象のインデックスページ。
  • itemnos: 削除対象のオフセット番号の配列。
  • nitems: 削除対象の数。
  • firststate: 最初のアイテムに適用するタプル状態（SPGIST_REDIRECT または SPGIST_PLACEHOLDER）。
  • reststate: 残りのアイテムに適用するタプル状態。
  • blkno, offnum: 置き換え用タプル生成に使用するブロック番号とオフセット番号。

前提条件と不変条件

  • nitems > 0 の場合のみ処理を実行します。
  • 削除対象のオフセット番号は、内部でソートされて処理されます。
  • ページ内の不透明領域（Opaque）にある nRedirection および nPlaceholder カウンタは、挿入されたタプルの状態に応じて適切に更新されます。

アルゴリズムまたは振る舞い

1. nitems が0の場合は即座に終了します。 2. itemnos 配列をコピーし、cmpOffsetNumbers を用いて昇順にソートします。 3. PageIndexMultiDelete を呼び出し、対象のタプルをページから削除します。 4. ソートされた各オフセット番号に対し、firststate または reststate に基づいて spgFormDeadTuple で新しいタプルを生成します。 5. PageAddItem を使用して、生成したタプルを元のオフセット位置に挿入します。 6. 挿入したタプルの状態に応じて、ページヘッダーのカウンタをインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgdoinsert.c 内の挿入ロジック
  • src/backend/access/spgist/spgvacuum.c 内のバキューム処理
  • src/backend/access/spgist/spgxlog.c 内のWALリプレイ処理
  • **呼び出し先**:
  • PageIndexMultiDelete (PostgreSQLコア関数)
  • qsort
  • spgFormDeadTuple
  • PageAddItem
  • SpGistPageGetOpaque

関連データ構造

  • SpGistState: インデックス全体のメタデータ。
  • SpGistDeadTuple: 削除後に配置されるリダイレクトまたはプレースホルダー用タプル。
  • Page: PostgreSQLのページ構造。

失敗モードと境界条件

  • PageAddItem が指定されたオフセットへの追加に失敗した場合、elog(ERROR) を発生させます。
  • 削除対象が0件の場合は何も行わず正常終了します。

保守担当者向けメモ

  • 本関数は PageIndexMultiDelete を利用するため、呼び出し元の itemnos 配列を破壊しないようコピーを作成しています。
  • 挿入するタプルの状態（tupstate）が変更されるたびに spgFormDeadTuple を再実行する最適化が行われています。
