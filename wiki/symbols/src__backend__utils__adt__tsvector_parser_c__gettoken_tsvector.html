gettoken_tsvector

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_parser.c:175

目的

tsvector または tsquery の入力文字列を解析し、トークン（単語）、位置情報、および重み情報を逐次的に抽出する。

シグネチャまたは形状

bool gettoken_tsvector(TSVectorParseState state,
                       char **strval, int *lenval,
                       WordEntryPos **pos_ptr, int *poslen,
                       char **endptr)

入力と出力

  • **入力**:
  • state: 解析状態を保持する TSVectorParseState 構造体。
  • **出力**:
  • strval: 抽出されたトークン文字列へのポインタ。
  • lenval: トークンの長さ。
  • pos_ptr: 抽出された位置情報配列へのポインタ。
  • poslen: 位置情報の個数。
  • endptr: 未使用（ソースコード上では引数として定義されているが、本体での使用は確認できない）。
  • **戻り値**: トークンが正常に抽出された場合は true、入力文字列の終端に達した場合は false を返す。

前提条件と不変条件

  • state は init_tsvector_parser によって初期化済みであること。
  • state->prsbuf は解析対象の文字列バッファを指していること。
  • 内部状態遷移において、WAITWORD から開始し、トークン確定時に RETURN_TOKEN マクロが呼び出されることで呼び出し元へ制御が戻る。

アルゴリズムまたは振る舞い

有限オートマトンとして動作し、以下の状態間を遷移する： 1. **WAITWORD**: トークンの開始を待機。引用符やエスケープ文字を処理し、単語の読み取りを開始する。 2. **WAITENDWORD**: トークンの終了を待機。空白、演算子、または位置情報開始文字（:）を検知する。 3. **WAITENDCMPLX / WAITCHARCMPLX**: 引用符で囲まれた複雑なトークンの解析。 4. **WAITPOSINFO / INPOSINFO / WAITPOSDELIM**: トークンに続く位置情報（: で始まる数値と重み）を解析する。

  • 状態遷移中に構文エラーを検知した場合は PRSSYNTAXERROR マクロによりエラーを報告する。
  • RESIZEPRSBUF マクロにより、必要に応じて単語バッファを動的に拡張する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery.c
  • src/backend/utils/adt/tsvector.c
  • **呼び出し先**:
  • palloc / repalloc (メモリ管理)
  • ereturn (エラーハンドリング)
  • ts_copychar_cstr (文字コピー)
  • ISOPERATOR, PRSSYNTAXERROR, RETURN_TOKEN, RESIZEPRSBUF (マクロ)

関連データ構造

  • TSVectorParseStateData: 解析の進捗、バッファ、フラグを保持する。
  • WordEntryPos: 位置情報と重みを格納する構造体。

失敗モードと境界条件

  • **構文エラー**: 不正なエスケープ、位置情報の欠落、または予期しない文字が出現した場合、escontext を介してエラーを報告する。
  • **バッファ不足**: トークンや位置情報がバッファサイズを超える場合、動的拡張を試みる。
  • **終端**: 入力文字列が空の場合、false を返して終了する。

保守担当者向けメモ

  • 状態遷移ロジックは複雑であるため、新しい構文ルールを追加する際は WAIT 系状態コードの遷移図を更新すること。
  • is_tsquery フラグは主にエラーメッセージの文言切り替えに使用される。
  • escontext を使用したソフトエラー処理は、バリデーション関数などで例外を発生させずにエラーを検知する際に重要である。
