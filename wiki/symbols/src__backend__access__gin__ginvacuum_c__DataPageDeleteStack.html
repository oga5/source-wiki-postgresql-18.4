DataPageDeleteStack

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/ginvacuum.c:113

目的

GINインデックスのポストティングツリー削除処理において、削除対象のページ階層を追跡するためのスタック構造を提供します。

シグネチャまたは形状

typedef struct DataPageDeleteStack
{
    struct DataPageDeleteStack *child;
    struct DataPageDeleteStack *parent;
    BlockNumber blkno;
    Buffer      leftBuffer;
    bool        isRoot;
} DataPageDeleteStack;

入力と出力

  • **入力**: 構造体生成時に blkno（現在のブロック番号）や isRoot フラグが設定されます。
  • **出力**: 削除処理の再帰的な走査において、親ノードから子ノードへのポインタ（child）および逆方向のポインタ（parent）を保持します。

前提条件と不変条件

  • leftBuffer は、削除対象ページの左側にある、削除されていない最も右側のページを指し、ピン留めおよびロックされている必要があります。
  • 階層構造を維持するため、parent と child のポインタは整合性を保つ必要があります。

アルゴリズムまたは振る舞い

  • ポストティングツリーの削除プロセスにおいて、ツリーの階層を辿る際に動的に割り当てられ、スタックとして機能します。
  • ginScanToDelete などの関数内で、削除対象のページを特定し、ツリー構造を再帰的に処理するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: ginScanToDelete 等の関数が、この構造体を palloc0 で割り当てて使用しています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • ginDeletePage: 本構造体で追跡されるページを物理的に削除するために使用されます。
  • ginScanToDelete: 本構造体を使用してポストティングツリーを走査します。

失敗モードと境界条件

  • メモリ不足時に palloc0 が失敗する可能性があります。
  • 階層の深さが極端に深い場合、スタック操作に伴うメモリ消費に注意が必要です。

保守担当者向けメモ

  • 本構造体は ginvacuum.c 内のポストティングツリー削除ロジックに特化しています。
  • ロックの順序（親から子へ）が厳密に管理されているため、このスタックを操作する際はデッドロックを回避する設計を維持してください。
