ParamListInfo

定義

  • 種別: typedef struct ParamListInfoData *ParamListInfo;
  • 定義箇所: src/include/nodes/params.h:98

目的

クエリ実行時に使用されるパラメータ（Paramノード）の集合を保持・管理するためのポインタ型です。プランナーが生成したパラメータ付きプランと、エグゼキュータ間の橋渡しを行います。

シグネチャまたは形状

ParamListInfoData 構造体へのポインタです。

入力と出力

  • 入力: クエリ実行に必要なパラメータ値の集合。
  • 出力: 実行エンジンによるパラメータ値の参照および取得。

前提条件と不変条件

  • ParamListInfo は実際には柔軟な配列（FLEXIBLE_ARRAY_MEMBER）を持つ構造体です。
  • paramFetch フックが設定されている場合、内部のパラメータ配列は必ずしも割り当てられている必要はありません。

アルゴリズムまたは振る舞い

  • **静的アクセス**: 構造体内の配列を直接参照してパラメータを取得します。
  • **動的アクセス**: paramFetch フックが定義されている場合、エグゼキュータは配列の代わりにこの関数を呼び出し、値を遅延評価または外部から取得します。

呼び出し元と呼び出し先

  • **呼び出し元（例）**:
  • contrib/pg_stat_statements/pg_stat_statements.c
  • src/backend/commands/explain.c
  • src/backend/commands/createas.c
  • contrib/sepgsql/hooks.c
  • **呼び出し先**:
  • 提供されたソース excerpt には具体的な呼び出し先関数は含まれていません。

関連データ構造

  • ParamListInfoData: 実体となる構造体。
  • ParamExternData: 外部パラメータの値を保持する構造体。
  • ParamExecData: エグゼキュータ内部で一時的に保持されるパラメータ。
  • ParamFetchHook: 動的パラメータ取得のためのコールバック関数型。

失敗モードと境界条件

  • paramFetch フックを使用しない場合、パラメータ配列の範囲外アクセスに注意が必要です。
  • PARAM_EXEC（内部計算用）と PARAM_EXTERN（外部入力用）で用途が異なるため、混同すると実行時エラーの原因となります。

保守担当者向けメモ

  • 「リスト」という名称ですが、実際には柔軟な配列構造を持つ点に留意してください。
  • パラメータのシリアライズや複製を行う際は、src/backend/nodes/params.c にある makeParamList や copyParamList を参照してください。
