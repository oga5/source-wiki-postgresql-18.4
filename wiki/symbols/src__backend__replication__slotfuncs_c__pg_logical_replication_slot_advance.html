pg_logical_replication_slot_advance

定義

  • 種別: function
  • 定義箇所: src/backend/replication/slotfuncs.c:500

目的

論理レプリケーションスロットのWAL読み取り位置（restart_lsn）を、指定されたLSNまで強制的に進めるための内部関数です。

シグネチャまたは形状

static XLogRecPtr pg_logical_replication_slot_advance(XLogRecPtr moveto)

入力と出力

  • **入力**: moveto (XLogRecPtr) - スロットを進める目標となるWAL位置。
  • **出力**: XLogRecPtr - 処理の結果、到達した最終的なWAL位置。

前提条件と不変条件

  • 本関数は論理スロットのコンテキスト内でのみ呼び出されることを前提としています。
  • 呼び出し元で適切な権限チェックおよびスロットのロックが確保されている必要があります。

アルゴリズムまたは振る舞い

1. 引数として受け取った moveto を、内部関数 LogicalSlotAdvanceAndCheckSnapState に渡します。 2. LogicalSlotAdvanceAndCheckSnapState の第2引数には NULL を渡します。 3. 同関数の戻り値（到達したLSN）をそのまま呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/slotfuncs.c 内の pg_replication_slot_advance 関数から呼び出されることが確認されています。
  • **呼び出し先**: LogicalSlotAdvanceAndCheckSnapState を呼び出します。

関連データ構造

  • XLogRecPtr: WAL上の位置を指し示すデータ型。

失敗モードと境界条件

  • 内部関数 LogicalSlotAdvanceAndCheckSnapState がエラーを発生させた場合、そのエラーがそのまま伝播します。
  • 指定された moveto が現在のスロット位置より過去である場合や、無効なLSNである場合の挙動は、依存先の LogicalSlotAdvanceAndCheckSnapState の実装に依存します。

保守担当者向けメモ

  • 本関数は pg_replication_slot_advance の論理スロット用ラッパーです。論理スロット特有の挙動変更が必要な場合は、本関数ではなく LogicalSlotAdvanceAndCheckSnapState の修正を検討してください。
