clean_NOT

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:189

目的

tsquery 構造体から NOT 演算子を含むノードを除去し、最適化されたクエリを再構築します。

シグネチャまたは形状

QueryItem *clean_NOT(QueryItem *ptr, int *len)

入力と出力

  • **入力**:
  • ptr: 処理対象となる QueryItem 配列のポインタ。
  • len: 結果として生成される QueryItem 配列の長さを格納するための整数ポインタ。
  • **出力**:
  • 処理後の QueryItem 配列へのポインタを返します。

前提条件と不変条件

  • 入力された ptr は有効な tsquery のフラットな配列である必要があります。
  • 内部で maketree を使用するため、メモリ管理は PostgreSQL のメモリコンテキストに依存します。

アルゴリズムまたは振る舞い

1. maketree(ptr) を呼び出し、フラットな QueryItem 配列を NODE 構造体のツリー形式に変換します。 2. clean_NOT_intree(root) を呼び出し、ツリーを再帰的に走査して NOT 演算子を持つノードを特定・除去します。 3. plaintree(...) を呼び出し、クリーンアップ後のツリーを再びフラットな QueryItem 配列に変換します。 4. 変換後の配列の長さを len に書き込み、結果の配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsquery.c:1377
  • **呼び出し先**:
  • maketree
  • clean_NOT_intree
  • plaintree

関連データ構造

  • QueryItem: tsquery を構成するフラットな要素。
  • NODE: tsquery をツリー形式で表現するための内部構造体。

失敗モードと境界条件

  • 入力ツリーが空の場合や、すべてのノードが NOT 演算子によって除去される場合の挙動は、plaintree および clean_NOT_intree の実装に依存します。
  • スタックオーバーフロー防止のため、内部で check_stack_depth() が呼び出される可能性があります（tsquery_cleanup.c の共通仕様に基づく）。

保守担当者向けメモ

  • 本関数は tsquery の最適化パイプラインの一部です。NOT 演算子の除去ロジックを変更する場合は、clean_NOT_intree の再帰的な走査ロジックを確認してください。
  • plaintree による再構築時にメモリ確保が行われるため、呼び出し元でのメモリ解放の責任範囲に注意してください。
