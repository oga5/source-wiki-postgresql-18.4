PgAioResult

定義

  • **種別**: struct
  • **定義箇所**: src/include/storage/aio_types.h:99

目的

非同期I/O（AIO）操作の完了ステータス、エラー情報、およびメタデータをコンパクトに集約して保持するための構造体です。

シグネチャまたは形状

typedef struct PgAioResult
{
    uint32      id:PGAIO_RESULT_ID_BITS;
    uint32      status:PGAIO_RESULT_STATUS_BITS;
    uint32      error_data:PGAIO_RESULT_ERROR_BITS;
    int32       result;
} PgAioResult;

入力と出力

  • **入力**: I/O操作完了時に、各コールバック関数や完了処理ルーチンによって各フィールドが設定されます。
  • **出力**: PgAioReturn 構造体の一部として上位コンポーネントに渡され、エラーハンドリングや後続処理の判断材料として利用されます。

前提条件と不変条件

  • **サイズ制約**: メモリ効率を考慮し、合計32ビット（ビットフィールド部）＋32ビット（result部）の構成で、全体としてコンパクトに設計されています。
  • **型定義**: ビットフィールドの符号解釈の差異を避けるため、uint32 を明示的に使用しています。
  • **整合性**: 変更を加える際は、StaticAssertDecl によるサイズチェックを維持する必要があります。

アルゴリズムまたは振る舞い

I/O完了後に PgAioResult が生成され、buffer_readv_decode_error や buffer_readv_report などの関数を通じて、I/Oの結果（成功/失敗）やエラーの詳細が伝達されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/aio.c および src/backend/storage/aio/aio_callback.c 内のI/O完了処理ルーチン。
  • **呼び出し先**: src/backend/storage/buffer/bufmgr.c における buffer_readv_decode_error、buffer_readv_encode_error、buffer_readv_complete、buffer_readv_report 等が本構造体を参照・操作します。

関連データ構造

  • PgAioReturn: 本構造体とターゲット情報を組み合わせた上位構造体。
  • PgAioTargetData: I/O対象を特定するための情報。
  • PgAioResultStatus: status フィールドの型定義。

失敗モードと境界条件

  • ビットフィールドの各ビット数（PGAIO_RESULT_ID_BITS 等）を超過する値を代入した場合の挙動は未定義です。
  • コンパイラによるビットフィールドのパッキング順序に依存するため、プラットフォーム間でのバイナリ互換性には注意が必要です。

保守担当者向けメモ

  • 本構造体はサイズが固定されているため、フィールドの追加やビット幅の変更を行う際は、全体のサイズが意図した範囲に収まっているか確認してください。
  • error_data の意味は、使用されるコールバックの report 関数によって定義されます。
