namenlike

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/like.c:262

目的

SQLの NOT LIKE 演算子を name 型に対して実装し、パターンマッチングの結果を反転させて返します。

シグネチャまたは形状

Datum namenlike(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_NAME(0): 検索対象の name 型データ。
  • PG_GETARG_TEXT_PP(1): パターン文字列（text 型）。
  • **出力**:
  • bool: パターンに一致しない場合は true、一致する場合は false を返します。

前提条件と不変条件

  • PG_FUNCTION_ARGS マクロを通じて渡される引数は、PostgreSQLの関数呼び出し規約に従っている必要があります。
  • データベースの照合順序（Collation）が PG_GET_COLLATION() を通じて正しく取得できる必要があります。

アルゴリズムまたは振る舞い

1. NameStr を使用して name 型から文字列ポインタを取得し、その長さを算出します。 2. VARDATA_ANY および VARSIZE_ANY_EXHDR を使用して、パターン文字列のデータポインタと長さを取得します。 3. GenericMatchText を呼び出し、対象文字列とパターンを照合します。 4. GenericMatchText の戻り値が LIKE_TRUE でない場合（つまり一致しない場合）、結果を true とします。 5. PG_RETURN_BOOL を介して判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • GenericMatchText: パターンマッチングの実行。
  • PG_GETARG_NAME, PG_GETARG_TEXT_PP, PG_GET_COLLATION, PG_RETURN_BOOL: PostgreSQLの関数インターフェースマクロ。
  • **呼び出し元**:
  • ソースコード内に直接的な呼び出し元の記述はありません。SQLエンジンから演算子として呼び出されることが想定されます。

関連データ構造

  • Name: name 型の内部表現。
  • text: 可変長文字列型。
  • Datum: PostgreSQLの汎用データ型。

失敗モードと境界条件

  • 入力パターンが不正な形式である場合、GenericMatchText 内部でエラーが発生する可能性があります。
  • メモリ不足等のシステムリソース制約により、関数実行が中断される場合があります。

保守担当者向けメモ

  • 本関数は name 型専用の NOT LIKE 実装です。text 型や bytea 型に対する同様の処理は、それぞれ別の関数（textnlike や byteanlike）で定義されています。
  • GenericMatchText への依存が強いため、マッチングロジックの変更は like_match.c および GenericMatchText の実装を確認してください。
