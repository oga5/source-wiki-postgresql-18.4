ghstore_decompress

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_gist.c:212

目的

GiST インデックスにおける hstore 型の圧縮データの伸長（デコンプレス）処理を提供します。

シグネチャまたは形状

Datum ghstore_decompress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される GISTTYPE 構造体へのポインタ。
  • **出力**: 入力されたポインタをそのまま Datum 型として返します。

前提条件と不変条件

  • 入力された Datum は GISTTYPE 構造体として解釈可能である必要があります。
  • 本関数はデータの変換を行わず、入力値をそのまま返すパススルー実装です。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を呼び出し、第1引数からポインタを取得します。 2. 取得したポインタを PG_RETURN_POINTER マクロを通じてそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックスインターフェースから呼び出されます。
  • **呼び出し先**: 該当なし。
  • **補足**: contrib/hstore/hstore_gist.c:140 にて PG_FUNCTION_INFO_V1(ghstore_decompress) が宣言されており、PostgreSQL の関数マネージャに登録されています。

関連データ構造

  • GISTTYPE: インデックスに格納されるシグネチャを保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、PostgreSQL の標準的な関数呼び出し規約に従い動作します。本関数自体に特有の境界条件チェックは含まれていません。

保守担当者向けメモ

  • 本関数は現在、実質的な伸長処理を行わないパススルー関数として実装されています。将来的に hstore の GiST インデックス構造に変更が生じ、データが圧縮形式で格納されるようになった場合には、この関数内で適切な伸長ロジックを実装する必要があります。
