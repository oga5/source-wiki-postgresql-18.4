ParallelTableScanDescData

定義

  • 種別: struct
  • 定義箇所: src/include/access/relscan.h:80

目的

テーブルスキャンを並列実行する際に、複数のバックエンド（ワーカー）間で共有されるスキャン状態を保持するために使用されます。

シグネチャまたは形状

typedef struct ParallelTableScanDescData
{
	RelFileLocator phs_locator;     /* スキャン対象の物理リレーション */
	bool		phs_syncscan;       /* 同期スキャンロジックへ位置を報告するか */
	bool		phs_snapshot_any;   /* SnapshotAnyを使用するか */
	Size		phs_snapshot_off;   /* スナップショット用データ */
} ParallelTableScanDescData;

入力と出力

  • **入力**: 並列スキャン開始時に、リーダープロセスによって初期化されます。
  • **出力**: 共有メモリ上に配置され、各ワーカープロセスがスキャン対象の物理リレーション情報やスナップショット状態を参照するために利用されます。

前提条件と不変条件

  • 本構造体は共有メモリ上に配置されることを前提としています。
  • メンバの追加や変更を行う際は、共有メモリ上のレイアウトに影響を与えるため注意が必要です。

アルゴリズムまたは振る舞い

  • 並列スキャン実行時、各ワーカーは共有メモリ上の本構造体を参照します。
  • phs_syncscan が有効な場合、スキャン位置を同期スキャン（SyncScan）ロジックに報告します。
  • phs_snapshot_any および phs_snapshot_off は、スキャンに使用するスナップショットの状態をワーカー間で共有するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 具体的な呼び出し元はソースコードから特定できません。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。
  • **参照**: src/include/access/relscan.h 内の TableScanDescData 構造体が rs_parallel メンバとして本構造体へのポインタを保持します。

関連データ構造

  • ParallelTableScanDesc: ParallelTableScanDescData へのポインタ型。
  • TableScanDescData: テーブルスキャンの基底構造体。
  • RelFileLocator: スキャン対象の物理リレーションを識別する構造体。

失敗モードと境界条件

  • 共有メモリ上の配置に関する制約があるため、構造体のサイズ変更やメンバ順序の変更は、並列処理の整合性に影響を及ぼす可能性があります。

保守担当者向けメモ

  • 本構造体は多くのアクセスメソッドで並列スキャンの基盤として利用されます。
  • 構造体定義の変更は、共有メモリのレイアウトに依存する他のモジュールとの互換性に注意してください。
