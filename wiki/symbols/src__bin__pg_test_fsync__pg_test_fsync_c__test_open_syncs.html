test_open_syncs

定義

  • 種別: function
  • 定義箇所: src/bin/pg_test_fsync/pg_test_fsync.c:449

目的

open_sync を使用した際の書き込みサイズの違いによるコストを比較・評価する。

シグネチャまたは形状

static void test_open_syncs(void)

入力と出力

  • 入力: なし
  • 出力: なし（標準出力へテスト結果を表示）

前提条件と不変条件

  • 実行環境が open_sync をサポートしている必要がある。
  • 呼び出し前にテスト用の一時ファイルが準備されている必要がある。

アルゴリズムまたは振る舞い

1. テストの目的を説明するメッセージを標準出力に表示する。 2. 合計 16kB の書き込みを行うために、以下のブロックサイズと回数の組み合わせで test_open_sync を順次呼び出す。

  • 16kB × 1回
  • 8kB × 2回
  • 4kB × 4回
  • 2kB × 8回
  • 1kB × 16回

呼び出し元と呼び出し先

  • 呼び出し元: main 関数（src/bin/pg_test_fsync/pg_test_fsync.c:134）
  • 呼び出し先: test_open_sync

関連データ構造

  • なし

失敗モードと境界条件

  • OS やファイルシステムが open_sync をサポートしていない場合、内部で呼び出される test_open_sync が失敗またはスキップされる可能性がある。

保守担当者向けメモ

  • 本関数は open_sync のパフォーマンス特性を評価するための固定的なテストスイートである。書き込みサイズのバリエーションを追加・変更する場合は、test_open_sync の呼び出し引数を調整すること。
