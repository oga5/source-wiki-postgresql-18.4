pq_writeint32

定義

  • 種別: function
  • 定義箇所: src/include/libpq/pqformat.h:73

目的

32ビット整数値をネットワークバイトオーダー（ビッグエンディアン）に変換し、通信バッファ（StringInfoData）の末尾に書き込みます。

シグネチャまたは形状

static inline void pq_writeint32(StringInfoData *pg_restrict buf, uint32 i)

入力と出力

  • **入力**:
  • buf: 書き込み対象の StringInfoData 構造体へのポインタ。
  • i: 書き込む uint32 型の整数値。
  • **出力**: なし（戻り値なし）。副作用として buf の内容が更新されます。

前提条件と不変条件

  • buf が指すバッファには、sizeof(uint32) 分の書き込みを行うための十分な空き容量（buf->maxlen - buf->len）が事前に確保されている必要があります。
  • 本関数はバッファの自動拡張を行いません。

アルゴリズムまたは振る舞い

1. pg_hton32 を使用して、入力値 i をホストバイトオーダーからネットワークバイトオーダーへ変換します。 2. Assert により、バッファの容量不足がないことを検証します。 3. memcpy を使用して、変換後の値を buf->data の現在の末尾位置（buf->len）にコピーします。 4. buf->len を sizeof(uint32) 分だけ増加させ、書き込み位置を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/common/printtup.c（複数の箇所で呼び出し）
  • src/include/libpq/pqformat.h（内部的な再利用の可能性あり）
  • **呼び出し先**:
  • pg_hton32（バイトオーダー変換）
  • memcpy（メモリコピー）

関連データ構造

  • StringInfoData: 通信データを保持するバッファ構造体。

失敗モードと境界条件

  • バッファの容量が不足している場合、アサーションが有効なビルドではプログラムが停止（Assert 失敗）します。
  • バッファの動的拡張が必要な場合は、本関数ではなく pq_sendint32 を使用する必要があります。

保守担当者向けメモ

  • static inline 関数であるため、ヘッダーファイル内で定義されています。
  • パフォーマンス最適化のため、バッファの拡張チェックを省略しています。呼び出し側で enlargeStringInfo 等を用いて事前に十分な領域を確保していることを確認してください。
