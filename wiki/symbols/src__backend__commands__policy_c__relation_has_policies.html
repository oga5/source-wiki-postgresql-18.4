relation_has_policies

定義

  • 種別: function
  • 定義箇所: src/backend/commands/policy.c:1255

目的

指定されたリレーション（テーブル）に対して、行レベルセキュリティ（RLS）ポリシーが1つ以上定義されているかどうかを判定します。

シグネチャまたは形状

bool relation_has_policies(Relation rel)

入力と出力

  • **入力**: rel (Relation) - ポリシーの有無を確認する対象のリレーション。
  • **出力**: bool - ポリシーが存在する場合は true、存在しない場合は false を返します。

前提条件と不変条件

  • rel は有効なリレーション記述子である必要があります。
  • 関数実行中、pg_policy カタログに対して AccessShareLock が取得されます。

アルゴリズムまたは振る舞い

1. pg_policy システムカタログを AccessShareLock モードで開きます。 2. Anum_pg_policy_polrelid 属性（ポリシーが適用されるリレーションのOID）をキーとして、PolicyPolrelidPolnameIndexId インデックスを使用したスキャンを開始します。 3. systable_getnext を呼び出し、該当するポリシーエントリが1件でも存在するかを確認します。 4. エントリが見つかった場合（HeapTupleIsValid が真）、戻り値を true に設定します。 5. スキャンを終了し、カタログを閉じて結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には具体的な呼び出し元の関数名は記載されていません。
  • **呼び出し先**: table_open, ScanKeyInit, systable_beginscan, systable_getnext, systable_endscan, table_close などのカタログアクセス用APIを呼び出します。

関連データ構造

  • Relation: リレーション記述子。
  • pg_policy: ポリシー情報を保持するシステムカタログ。
  • ScanKeyData: カタログスキャン時の検索条件を保持する構造体。

失敗モードと境界条件

  • カタログスキャン中にエラーが発生した場合、PostgreSQLの標準的なエラーハンドリングによりトランザクションがアボートされる可能性があります。
  • rel が無効な場合や、カタログアクセス権限がない場合は予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数はポリシーの存在確認のみを行う軽量な関数です。ポリシーの詳細な定義が必要な場合は RelationBuildRowSecurity 等の利用を検討してください。
  • カタログスキャンには AccessShareLock を使用しているため、同時実行性への影響は最小限に抑えられています。
