connectby_text_serial

定義

  • 種別: function
  • 定義箇所: contrib/tablefunc/tablefunc.c:1054

目的

階層データ探索関数 connectby のラッパーであり、シリアル番号（連番）を含む形式で階層構造を再帰的に抽出して結果セットを返します。

シグネチャまたは形状

Datum connectby_text_serial(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relname, key_fld, parent_key_fld, orderby_fld, start_with (各 text)
  • max_depth (int32)
  • branch_delim (text, オプション)
  • **出力**: Datum (常に (Datum) 0)。実際のデータは ReturnSetInfo 構造体の setResult に格納された Tuplestore を介して返却されます。

前提条件と不変条件

  • 呼び出し元が SFRM_Materialize モードをサポートしている必要があります。
  • expectedDesc が提供されている必要があります。
  • 戻り値のタプル記述子は validateConnectbyTupleDesc による検証を通過する必要があります。

アルゴリズムまたは振る舞い

1. 引数を C 文字列に変換し、ReturnSetInfo のコンテキストを検証します。 2. branch_delim が指定されていない場合はデフォルト値 ~ を使用します。 3. per_query_ctx にメモリコンテキストを切り替え、タプル記述子をコピーします。 4. validateConnectbyTupleDesc を呼び出し、出力形式の整合性を確認します。 5. connectby 関数を呼び出し、再帰的な探索結果を Tuplestore に格納します。 6. rsinfo->setResult に Tuplestore を、rsinfo->setDesc にタプル記述子を設定し、メモリコンテキストを復元して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(connectby_text_serial) マクロにより登録されています。具体的な SQL クエリからの呼び出しについては、ソースコード内に直接的な記述はありません。
  • **呼び出し先**: connectby, text_to_cstring, PG_GETARG_*, CreateTupleDescCopy, validateConnectbyTupleDesc, TupleDescGetAttInMetadata, MemoryContextSwitchTo, pstrdup。

関連データ構造

  • ReturnSetInfo: セットを返す関数の実行情報を保持。
  • TupleDesc: タプルの構造定義。
  • AttInMetadata: タプル構築用のメタデータ。
  • Tuplestore: 構築された結果セットを保持するストレージ。

失敗モードと境界条件

  • rsinfo が NULL または ReturnSetInfo 型でない場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了します。
  • SFRM_Materialize モードが許可されていない場合、エラー終了します。
  • 階層が深すぎる場合や循環参照がある場合、内部の connectby 呼び出しによりパフォーマンス低下やスタックオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本関数は connectby のシリアル番号表示オプションを有効（show_serial = true）にして呼び出す専用のインターフェースです。
  • branch_delim の処理において、引数が 7 個ある場合は第 7 引数を使用し、それ以外はデフォルト値を使用するロジックになっています。
