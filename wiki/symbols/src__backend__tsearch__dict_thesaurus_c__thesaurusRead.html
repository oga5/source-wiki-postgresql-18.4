thesaurusRead

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_thesaurus.c:167

目的

シソーラス定義ファイル（.ths）を読み込み、その内容を解析して DictThesaurus 構造体にコンパイル済みのルールとして格納する。

シグネチャまたは形状

static void thesaurusRead(const char *filename, DictThesaurus *d)

入力と出力

  • **入力**:
  • filename: 読み込む設定ファイルのパス。
  • d: 読み込んだルールを格納する DictThesaurus 構造体へのポインタ。
  • **出力**: なし（d の内部状態を更新する）。

前提条件と不変条件

  • filename で指定されたファイルは、シソーラス定義形式に従っている必要がある。
  • ファイル内の各行は、: を境界として「置換対象フレーズ」と「置換後フレーズ」に分割される。
  • 置換後の各単語は、*（そのまま使用）、\（エスケープ）、または通常の文字列として指定される。

アルゴリズムまたは振る舞い

1. get_tsearch_config_filename を用いてファイルパスを解決し、tsearch_readline_begin で読み込みを開始する。 2. ファイルを1行ずつ読み込み、コメントや空行をスキップする。 3. 各行を状態マシン（TR_WAITLEX, TR_INLEX, TR_WAITSUBS, TR_INSUBS）で解析する。

  • TR_WAITLEX / TR_INLEX: 置換対象の単語を抽出し、newLexeme を呼び出す。
  • TR_WAITSUBS / TR_INSUBS: 置換後の単語を抽出し、addWrd を呼び出す。

    • 4. 各行の処理終了時に、置換ルール数（idsubst）をインクリメントし、整合性を検証する。 5. 最後に tsearch_readline_end でリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: thesaurus_init（src/backend/tsearch/dict_thesaurus.c:617）
  • **呼び出し先**: newLexeme, addWrd, tsearch_readline_begin, tsearch_readline, tsearch_readline_end, get_tsearch_config_filename, ereport

関連データ構造

  • DictThesaurus: 辞書全体の情報を保持する構造体。
  • tsearch_readline_state: ファイル読み込みの状態管理用構造体。

失敗モードと境界条件

  • ファイルが開けない場合、ERROR を発生させる。
  • 行のフォーマットが不正（区切り文字の欠落、単語の欠落など）な場合、ERROR を発生させる。
  • 置換ルール内の単語数が uint16 の範囲を超える場合、ERROR を発生させる。

保守担当者向けメモ

  • 本関数は辞書の初期化時にのみ実行される。
  • pg_mblen_cstr を使用してマルチバイト文字に対応している。
  • d->nsubst に最終的なルール数が格納されるため、後続の検索処理で参照される。
