get_qual_from_partbound

定義

  • 種別: function
  • 定義箇所: src/backend/partitioning/partbounds.c:248

目的

パーティション境界定義（PartitionBoundSpec）に基づき、そのパーティションが受け入れる値の範囲を規定する実行可能な制約式（List *）を生成します。

シグネチャまたは形状

List *get_qual_from_partbound(Relation parent, PartitionBoundSpec *spec)

入力と出力

  • **入力**:
  • parent: パーティション化された親テーブルの Relation。
  • spec: 生成対象となるパーティションの境界仕様を保持する PartitionBoundSpec。
  • **出力**:
  • パーティションの制約を表す式ツリーのリスト（List *）。

前提条件と不変条件

  • parent から取得される PartitionKey が存在すること（Assert(key != NULL)）。
  • spec のパーティション戦略が、親テーブルの PartitionKey の戦略と一致すること。

アルゴリズムまたは振る舞い

1. parent からパーティションキーを取得します。 2. キーの戦略（PARTITION_STRATEGY_HASH, LIST, RANGE）を判定します。 3. 戦略に応じて以下のいずれかの関数を呼び出し、制約式を構築します。

  • get_qual_for_hash
  • get_qual_for_list
  • get_qual_for_range

    • 4. 生成された制約式のリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c:20472
  • src/backend/utils/cache/partcache.c:381
  • **呼び出し先**:
  • get_qual_for_hash
  • get_qual_for_list
  • get_qual_for_range

関連データ構造

  • Relation: 親テーブルの情報を保持。
  • PartitionBoundSpec: パーティション境界の仕様。
  • PartitionKey: パーティションキーの定義。

失敗モードと境界条件

  • key->strategy が未知の場合の挙動は定義されていません。
  • spec の戦略が key->strategy と不一致の場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は、カタログ定義から実行時の制約式を生成するブリッジとして機能します。
  • 新しいパーティション戦略を追加する場合は、本関数の switch 文に新しいケースを追加し、対応する get_qual_for_* 関数を実装する必要があります。
