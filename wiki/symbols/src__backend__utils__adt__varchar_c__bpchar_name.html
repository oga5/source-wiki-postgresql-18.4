bpchar_name

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varchar.c:370

目的

BpChar（char(n)）型の値を PostgreSQL の Name 型（システムカタログ等で使用される固定長識別子）に変換します。

シグネチャまたは形状

Datum bpchar_name(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して BpChar 型のデータを受け取ります。
  • **出力**: Name 型（NAMEDATALEN バイトの固定長文字列）を Datum として返します。

前提条件と不変条件

  • 入力データは BpChar 型として正当である必要があります。
  • 出力される Name 型は、palloc0 によりゼロ初期化され、末尾の空白が除去された状態で格納されます。

アルゴリズムまたは振る舞い

1. 入力データの長さを取得し、NAMEDATALEN を超える場合は pg_mbcliplen を用いて切り詰めます。 2. 文字列の末尾から空白（' '）を走査し、空白を除いた実質的な長さを算出します。 3. palloc0(NAMEDATALEN) でメモリを確保し、算出した長さ分だけデータをコピーします。 4. ゼロ初期化されたメモリ領域により、結果は自動的にヌル終端およびパディングされます。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の記述から、型変換処理の文脈で呼び出されることが示唆されていますが、具体的な呼び出し箇所は提供されたソース内では特定されていません。
  • **呼び出し先**: VARSIZE_ANY_EXHDR、VARDATA_ANY、pg_mbcliplen、palloc0、memcpy などのメモリ操作および文字列処理ユーティリティを呼び出します。

関連データ構造

  • BpChar: char(n) 型の内部表現。
  • Name: システム識別子用の固定長文字列型（NAMEDATALEN バイト）。

失敗モードと境界条件

  • 入力長が NAMEDATALEN を超える場合、マルチバイト文字の整合性を維持しつつ切り詰めが行われます。
  • 入力がすべて空白の場合、長さ 0 の文字列として処理され、Name 型の領域はすべてゼロで埋められます。

保守担当者向けメモ

  • 本関数は char(n) から name 型へのキャストを実装しています。
  • Name 型の仕様（NAMEDATALEN）に強く依存しているため、将来的に識別子の長さ制限を変更する場合は注意が必要です。
