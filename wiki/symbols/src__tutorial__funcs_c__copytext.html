copytext

定義

  • 種別: function
  • 定義箇所: src/tutorial/funcs.c:64

目的

入力された PostgreSQL の text 型データを複製し、新しいメモリ領域にコピーして返却する。

シグネチャまたは形状

Datum copytext(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介して取得される text 型のポインタ。
  • **出力**: 複製された text 型データを含む Datum。

前提条件と不変条件

  • PostgreSQL の V1 呼び出し規約に従う関数であること。
  • palloc を使用してメモリを確保するため、呼び出し元のメモリコンテキストが有効であること。
  • 入力データが圧縮されているか、ショートヘッダーである可能性があるため、VARDATA_ANY および VARSIZE_ANY_EXHDR を使用して安全にアクセスすること。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して入力テキストを取得する。 2. VARSIZE_ANY_EXHDR を用いてデータ本体のサイズを計算し、フルサイズのヘッダー（VARHDRSZ）を付加した領域を palloc で確保する。 3. SET_VARSIZE を使用して、新しいテキスト構造体のヘッダーにサイズ情報を設定する。 4. VARDATA_ANY から VARDATA へ、memcpy を用いてデータ本体をコピーする。 5. PG_RETURN_TEXT_P を使用して、新しく作成したテキスト構造体を Datum として返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(copytext) により登録）。
  • **呼び出し先**: PostgreSQL のメモリ管理関数（palloc）およびデータ操作マクロ（PG_GETARG_TEXT_PP, SET_VARSIZE, VARDATA, VARDATA_ANY, VARSIZE_ANY_EXHDR, PG_RETURN_TEXT_P）。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • 入力データが NULL の場合、PG_GETARG_TEXT_PP の挙動に従い、NULL 許容設定に応じて処理される。

保守担当者向けメモ

  • 本関数は text 型のコピーにおける標準的な実装例である。
  • memcpy を使用する際は、ソースとデスティネーションのヘッダー形式の違い（ショートヘッダーの有無など）を考慮する必要があるため、マクロ経由のアクセスを維持すること。
