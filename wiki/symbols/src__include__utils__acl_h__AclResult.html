AclResult

定義

  • 種別: enum
  • 定義箇所: src/include/utils/acl.h:181

目的

データベースオブジェクトに対する権限チェックの結果を表現するための列挙型です。

シグネチャまたは形状

typedef enum
{
    ACLCHECK_OK = 0,
    ACLCHECK_NO_PRIV,
    ACLCHECK_NOT_OWNER,
} AclResult;

入力と出力

  • 出力: 権限チェック関数（aclmask 等）が、判定結果としてこの列挙値のいずれかを返します。

前提条件と不変条件

  • ACLCHECK_OK は権限が許可されていることを示します。
  • 権限チェック関数は、呼び出し元に対してこの列挙値を用いて判定結果を伝達します。

アルゴリズムまたは振る舞い

  • ACLCHECK_OK: 権限チェックが成功。
  • ACLCHECK_NO_PRIV: 対象ロールに必要な権限が不足している。
  • ACLCHECK_NOT_OWNER: 操作に必要な所有者権限が不足している。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/dblink/dblink.c
  • contrib/pg_prewarm/pg_prewarm.c
  • contrib/pgrowlocks/pgrowlocks.c
  • src/backend/access/index/genam.c
  • src/backend/catalog/aclchk.c（aclcheck_error 等の関数で結果を処理）
  • 呼び出し先: 情報なし。

関連データ構造

  • AclItem: 権限付与の最小単位。
  • Acl: AclItem の配列。
  • AclMode: 権限ビットを保持する型。

失敗モードと境界条件

  • src/backend/catalog/aclchk.c 内で、未定義の AclResult 値が渡された場合に elog(ERROR, ...) を発生させるガードが存在します。

保守担当者向けメモ

  • 新しい権限チェック結果の状態を追加する場合は、aclchk.c 内の elog によるエラーハンドリング箇所も更新してください。
