stop_postmaster

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/server.c:332

目的

実行中のPostgreSQLサーバー（Postmaster）を停止し、管理状態を更新します。

シグネチャまたは形状

void stop_postmaster(bool in_atexit)

入力と出力

  • **入力**:
  • in_atexit (bool): atexit フックから呼び出されているかどうかを示すフラグ。true の場合、強制停止（-m fast）が選択され、ログ出力の抑制が行われます。
  • **出力**: なし（void）。

前提条件と不変条件

  • os_info.running_cluster が old_cluster または new_cluster を指している必要があります。
  • 実行対象のクラスターが os_info.running_cluster に正しく設定されている必要があります。

アルゴリズムまたは振る舞い

1. os_info.running_cluster を参照し、停止対象のクラスター（old_cluster または new_cluster）を特定します。対象が設定されていない場合は何もせず終了します。 2. pg_ctl コマンドを構築し、exec_prog を介して実行します。

  • in_atexit が true の場合、-m fast モードで停止し、ログ出力は抑制されます。
  • in_atexit が false の場合、-m smart モードで停止し、ログ出力が行われます。

    • 3. 停止コマンドの実行後、os_info.running_cluster を NULL にリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c
  • src/bin/pg_upgrade/pg_upgrade.c
  • src/bin/pg_upgrade/server.c (内部の stop_postmaster_atexit 等)
  • **呼び出し先**:
  • exec_prog (外部コマンド実行用)

関連データ構造

  • ClusterInfo: 停止対象のクラスター情報（パス、設定等）を保持。
  • os_info: 実行中のクラスター状態を管理するグローバル構造体。

失敗モードと境界条件

  • os_info.running_cluster が NULL の場合、関数は即座にリターンし、何も実行しません。
  • exec_prog が失敗した場合、その内部実装に従いプログラムが終了する可能性があります。

保守担当者向けメモ

  • 本関数は pg_ctl を使用してサーバーを停止します。停止モード（smart/fast）は in_atexit フラグによって切り替えられます。
  • os_info.running_cluster のリセットは、コマンド実行の成否に関わらず行われる点に注意してください。
