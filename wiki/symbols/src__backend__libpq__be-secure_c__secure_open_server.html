secure_open_server

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/be-secure.c:111

目的

SSL/TLSハンドシェイクを開始し、クライアントとのセキュアな通信接続を確立する。

シグネチャまたは形状

int secure_open_server(Port *port)

入力と出力

  • **入力**: Port *port（通信状態や暗号化フラグを保持する構造体）
  • **出力**: 成功時は 0、失敗時は STATUS_ERROR（または be_tls_open_server が返す値）を返す。

前提条件と不変条件

  • USE_SSL が定義されている環境で動作する。
  • SSLハンドシェイク開始前にバッファリングされていた未処理データ（raw_buf）は、ハンドシェイク完了後にすべて消費されている必要がある。

アルゴリズムまたは振る舞い

1. **バッファ退避**: SSLハンドシェイク開始前に読み込まれていた未処理の平文データを port->raw_buf に一時退避する。 2. **注入ポイント**: backend-ssl-startup インジェクションポイントを実行する。 3. **ハンドシェイク**: be_tls_open_server(port) を呼び出し、TLSセッションを確立する。 4. **検証**: ハンドシェイク後に未処理の平文データが残っていないことを確認する。残存している場合はエラーとする。 5. **クリーンアップ**: 退避していた raw_buf を解放する。 6. **ログ出力**: 接続成功時にピアのDN/CN情報を DEBUG2 レベルで記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/backend_startup.c
  • **呼び出し先**:
  • be_tls_open_server（TLS実装への委譲）
  • pq_buffer_remaining_data, pq_startmsgread, pq_getbytes, pq_endmsgread（バッファ操作）
  • palloc, pfree（メモリ管理）
  • elog, ereport（ログ出力）

関連データ構造

  • Port: 通信状態、暗号化フラグ、およびバッファ（raw_buf 等）を保持する。

失敗モードと境界条件

  • pq_getbytes が EOF を返した場合、STATUS_ERROR を返す。
  • ハンドシェイク後に port->raw_buf_remaining > 0 となった場合（セッション確立前に暗号化データが送られてきたとみなされる）、ログを出力して STATUS_ERROR を返す。

保守担当者向けメモ

  • 本関数は USE_SSL が有効な場合にのみ機能するラッパーである。
  • raw_buf の処理は、SSLハンドシェイク開始前に読み込まれてしまった平文データを保持し、セッション確立後に再送または処理するための仕組みである。
  • 具体的なTLS実装については be-secure-openssl.c を参照すること。
