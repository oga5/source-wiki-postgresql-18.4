RenameRewriteRule

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/rewrite/rewriteDefine.c:792

目的

指定されたリレーションに紐付く書き換えルール（Rewrite Rule）の名前を変更します。

シグネチャまたは形状

ObjectAddress RenameRewriteRule(RangeVar *relation, const char *oldName, const char *newName)

入力と出力

  • **入力**:
  • relation: 対象リレーションを指定する RangeVar。
  • oldName: 変更対象のルール名。
  • newName: 新しいルール名。
  • **出力**:
  • 変更されたルールの ObjectAddress を返します。

前提条件と不変条件

  • 対象リレーションに対して AccessExclusiveLock が取得されている必要があります。
  • ON SELECT ルールは名前が _RETURN に固定されているため、名前変更は許可されません。
  • 新しい名前 (newName) が既に存在する場合はエラーとなります。

アルゴリズムまたは振る舞い

1. RangeVarGetRelidExtended を呼び出し、対象リレーションの OID を取得し、AccessExclusiveLock を獲得します。 2. SearchSysCacheCopy2 を使用して、pg_rewrite カタログから対象のルールタプルを検索します。存在しない場合はエラーを報告します。 3. IsDefinedRewriteRule を用いて、新しい名前のルールが既に存在しないか確認します。 4. ルールが ON SELECT 型でないことを検証します。 5. namestrcpy でルール名を更新し、CatalogTupleUpdate でカタログを更新します。 6. InvokeObjectPostAlterHook を呼び出し、変更を通知します。 7. CacheInvalidateRelcache を呼び出し、リレーションキャッシュを無効化して全バックエンドに通知します。 8. 最後にリレーションをクローズしますが、ロックは保持したまま終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/alter.c:406
  • **呼び出し先**:
  • RangeVarGetRelidExtended, relation_open, table_open, SearchSysCacheCopy2, IsDefinedRewriteRule, CatalogTupleUpdate, InvokeObjectPostAlterHook, heap_freetuple, table_close, CacheInvalidateRelcache, ObjectAddressSet, relation_close

関連データ構造

  • ObjectAddress: ルールを特定するための識別子。
  • Form_pg_rewrite: pg_rewrite カタログの構造体。
  • RangeVar: リレーション指定用構造体。

失敗モードと境界条件

  • **ERRCODE_UNDEFINED_OBJECT**: 指定した oldName のルールが存在しない場合。
  • **ERRCODE_DUPLICATE_OBJECT**: 指定した newName のルールが既に存在する場合。
  • **ERRCODE_INVALID_OBJECT_DEFINITION**: ON SELECT ルールの名前を変更しようとした場合。

保守担当者向けメモ

  • 本関数はトランザクション終了まで AccessExclusiveLock を保持し続けます。
  • CacheInvalidateRelcache の呼び出しは、他バックエンドとの整合性を保つために必須です。
  • ON SELECT ルールの制約（_RETURN という名前）は、PostgreSQLのビュー実装における不変条件です。
