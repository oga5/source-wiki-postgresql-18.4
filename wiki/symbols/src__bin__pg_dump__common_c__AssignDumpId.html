AssignDumpId

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/common.c:656

目的

データベースオブジェクト（DumpableObject）に一意なダンプIDを割り当て、管理用のグローバル配列およびカタログIDハッシュテーブルに登録する。

シグネチャまたは形状

void AssignDumpId(DumpableObject *dobj)

入力と出力

  • **入力**: dobj（初期化対象の DumpableObject 構造体へのポインタ）
  • **出力**: なし（副作用として dobj のメンバ更新およびグローバルな管理構造体への登録を行う）

前提条件と不変条件

  • dobj は有効なメモリ領域を指していること。
  • lastDumpId がグローバルに管理されており、呼び出しごとにインクリメントされる。
  • dumpIdMap 配列は、必要に応じて pg_realloc_array により動的に拡張される。
  • catalogIdHash は、最初の呼び出し時に catalogid_create で初期化される。

アルゴリズムまたは振る舞い

1. **ID割り当てと初期化**: lastDumpId をインクリメントして dobj->dumpId に代入し、name, namespace, dependencies などのデフォルト値を設定する。 2. **配列登録**: dobj->dumpId が現在の allocedDumpIds を超える場合、dumpIdMap を拡張し、新しい領域をゼロクリアする。その後、dumpIdMap[dobj->dumpId] に dobj を格納する。 3. **ハッシュ登録**: dobj->catId.tableoid が有効な場合、catalogIdHash に dobj->catId をキーとして登録する。既に存在しないエントリであれば、dobj を紐付ける。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/common.c 内のオブジェクト生成処理（attachinfo 等）、および src/bin/pg_dump/pg_dump.c 内の各種オブジェクト（tdinfo, loinfo, polinfo, pubinfo 等）の生成処理から呼び出される。
  • **呼び出し先**: pg_malloc_array, pg_realloc_array, memset, catalogid_create, catalogid_insert, OidIsValid, Assert。

関連データ構造

  • DumpableObject: ダンプ対象オブジェクトの基本構造。
  • dumpIdMap: 全オブジェクトをIDで検索するための配列。
  • catalogIdHash: CatalogId から DumpableObject を検索するためのハッシュテーブル。
  • CatalogIdMapEntry: ハッシュテーブル内のエントリ構造。

失敗モードと境界条件

  • メモリ不足時: pg_malloc_array / pg_realloc_array 内部でエラーが発生する可能性がある。
  • dobj->catId.tableoid が無効な場合: ハッシュテーブルへの登録はスキップされる。
  • catalogIdHash が未初期化の場合: 最初の登録時に自動的に作成される。

保守担当者向けメモ

  • dobj->name や dobj->namespace はこの関数内では NULL に初期化されるため、呼び出し側で後続の処理として適切に設定する必要がある。
  • CatalogIdMapEntry は simplehash.h を使用しており、ハッシュテーブル内のエントリはポインタ経由で管理される設計となっている。
