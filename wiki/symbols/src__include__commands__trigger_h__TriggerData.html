TriggerData

定義

  • 種別: struct
  • 定義箇所: src/include/commands/trigger.h:31

目的

トリガー関数実行時に、対象テーブルのコンテキスト情報、操作対象のタプル、およびトリガー定義を保持し、実行エンジンとトリガー関数間で受け渡すための構造体です。

シグネチャまたは形状

typedef struct TriggerData
{
	NodeTag		type;
	TriggerEvent tg_event;
	Relation	tg_relation;
	HeapTuple	tg_trigtuple;
	HeapTuple	tg_newtuple;
	Trigger    *tg_trigger;
	TupleTableSlot *tg_trigslot;
	TupleTableSlot *tg_newslot;
	Tuplestorestate *tg_oldtable;
	Tuplestorestate *tg_newtable;
	const Bitmapset *tg_updatedcols;
} TriggerData;

入力と出力

  • **入力**: 実行エンジン（nodeModifyTable.c 等）によって構築され、トリガー関数へ渡されます。
  • **出力**: トリガー関数は、この構造体内の tg_trigtuple や tg_newtuple を参照・変更し、実行結果を返します。

前提条件と不変条件

  • type フィールドは T_TriggerData である必要があります。
  • 実行中のトリガー関数内では、tg_relation や tg_trigger などのポインタが有効であることが期待されます。

アルゴリズムまたは振る舞い

1. 実行エンジンがトリガー実行関数（ExecCallTriggerFunc 等）を呼び出す際に、TriggerData を初期化します。 2. トリガー関数は fcinfo->context を通じてこの構造体を取得します。 3. 遷移テーブル（REFERENCING 句）が有効な場合、tg_oldtable および tg_newtable に変更前後のタプルが蓄積されます。 4. tg_updatedcols は、UPDATE 操作時に変更された列のビットマップを提供します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/trigger.c (ExecCallTriggerFunc)
  • src/backend/commands/tablecmds.c
  • 各種 contrib モジュール（lo, spi/autoinc, spi/refint 等）のトリガー関数。
  • **呼び出し先**:
  • 本構造体自体はデータコンテナであり、直接関数を呼び出すことはありません。

関連データ構造

  • TriggerEvent: トリガーの発生要因とタイミングを定義。
  • Trigger: トリガーの定義情報。
  • TupleTableSlot: タプルデータへのアクセス用スロット。
  • Tuplestorestate: 遷移テーブルの保持用ストア。

失敗モードと境界条件

  • 構造体が適切に初期化されていない場合、トリガー関数内でのヌルポインタ参照が発生する可能性があります。
  • tg_newtuple は INSERT や UPDATE 以外の操作では無効（NULL）となる場合があります。

保守担当者向けメモ

  • TriggerEvent のビットフラグと pg_trigger.tgtype の混同を避けてください。
  • 遷移テーブル（tg_oldtable, tg_newtable）のライフサイクルは、トリガー実行のスコープに依存するため注意が必要です。
