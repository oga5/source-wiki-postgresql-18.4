getRightMostDot

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/compatlib/informix.c:749

目的

文字列内において、最後に出現するドット（.）のインデックス位置を特定します。

シグネチャまたは形状

static int getRightMostDot(const char *str)

入力と出力

  • **入力**: str (ヌル終端文字列へのポインタ)
  • **出力**:
  • ドットが見つかった場合: 文字列の先頭を 0 としたドットのインデックス（int）
  • ドットが見つからなかった場合: -1

前提条件と不変条件

  • str は有効なヌル終端文字列である必要があります。
  • 戻り値は、文字列の長さが int の範囲内であることを前提としています。

アルゴリズムまたは振る舞い

1. 文字列の長さ len を取得します。 2. 文字列の末尾から先頭に向かって（len - 1 から 0 まで）ループします。 3. 現在の文字が . である場合、その位置（len - j - 1）を即座に返します。 4. ドットが見つからないままループが終了した場合、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/compatlib/informix.c 内の rfmtlong 関数（推測される文脈）にて、フォーマット文字列 fmt を引数として呼び出されています。
  • **呼び出し先**: 標準ライブラリの strlen を呼び出します。

関連データ構造

  • 特になし（標準的な C 文字列を使用）。

失敗モードと境界条件

  • **空文字列**: len が 0 となり、ループが実行されず -1 を返します。
  • **ドットなし**: 文字列全体を走査し、-1 を返します。
  • **複数のドット**: 最も右側（末尾に近い方）のドット位置を返します。

保守担当者向けメモ

  • 本関数は static であり、informix.c 内でのみ使用されるユーティリティです。
  • 文字列の長さが非常に大きい場合、int 型へのキャストによるオーバーフローの可能性について考慮が必要ですが、現在の用途では rfmtlong 等のフォーマット処理に限定されています。
