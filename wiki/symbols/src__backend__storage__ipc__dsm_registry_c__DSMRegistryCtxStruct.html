DSMRegistryCtxStruct

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/ipc/dsm_registry.c:35

目的

共有メモリ上に配置される、名前付き動的共有メモリ（DSM）レジストリのメタデータを保持するための構造体です。

シグネチャまたは形状

typedef struct DSMRegistryCtxStruct
{
	dsa_handle	dsah;        /* 動的共有メモリ領域のハンドル */
	dshash_table_handle dshh; /* 共有ハッシュテーブルのハンドル */
} DSMRegistryCtxStruct;

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。共有メモリの初期化時やレジストリへのアクセス時に、この構造体を通じて dsa_handle および dshash_table_handle が参照されます。

前提条件と不変条件

  • 共有メモリ上に配置される必要があるため、DSMRegistryShmemInit 等を通じて適切に初期化されている必要があります。
  • 本構造体に含まれるハンドルは、dshash および dsa サブシステムによって管理される有効なリソースを指している必要があります。

アルゴリズムまたは振る舞い

  • レジストリのコンテキストとして機能し、名前付きDSMセグメントを検索・管理するためのハッシュテーブル（dshh）と、そのメモリ管理領域（dsah）へのアクセスポイントを提供します。

呼び出し元と呼び出し先

  • 呼び出し元: DSMRegistryShmemInit（初期化時）、init_dsm_registry（レジストリ接続時）にて参照されます。
  • 呼び出し先: 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • DSMRegistryEntry: ハッシュテーブル内に格納される個別のセグメント管理情報。
  • dshash_table_handle: 共有ハッシュテーブルのハンドル。
  • dsa_handle: 動的共有メモリ領域のハンドル。

失敗モードと境界条件

  • 共有メモリの確保に失敗している場合、本構造体へのポインタ（DSMRegistryCtx）が初期化されず、アクセス時にクラッシュする可能性があります。

保守担当者向けメモ

  • 本構造体は DSMRegistryCtx という静的変数を通じてモジュール内でグローバルに参照されます。
  • 構造体のサイズは DSMRegistryShmemSize 関数内で MAXALIGN を用いて計算されており、共有メモリ確保時の整合性に注意してください。
