WalSndShutdown

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walsender.c:379

目的

WAL 送信プロセス（walsender）を安全に終了させるためのクリーンアップ処理を実行します。

シグネチャまたは形状

static void WalSndShutdown(void)

入力と出力

  • 入力: なし
  • 出力: なし（プロセスを終了させるため呼び出し元には戻らない）

前提条件と不変条件

  • whereToSendOutput がグローバル変数として存在すること。
  • プロセス終了時に proc_exit(0) を呼び出すことで、PostgreSQL の標準的な終了処理シーケンスに移行する。

アルゴリズムまたは振る舞い

1. 現在の出力先がリモート接続（DestRemote）である場合、whereToSendOutput を DestNone に変更します。これにより、終了処理中に ereport 等がスタンバイ側へメッセージを送信しようとするのを防止します。 2. proc_exit(0) を呼び出し、プロセスを正常終了させます。 3. コンパイラの警告を抑制するため、到達不能な abort() を呼び出します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/walsender.c 内の複数箇所（行 1590, 1648, 1703, 1932, 2800, 2861, 4144）。主にエラー発生時や通信切断時の終了処理として呼び出されます。
  • 呼び出し先: proc_exit（PostgreSQL のプロセス終了関数）。

関連データ構造

  • whereToSendOutput: 出力先を制御するグローバル変数。

失敗モードと境界条件

  • 本関数はプロセスを終了させるため、呼び出し後の制御は戻りません。
  • whereToSendOutput の状態に関わらず、proc_exit が実行されることでプロセスは終了します。

保守担当者向けメモ

  • 本関数は pg_noreturn として宣言されています。終了処理のロジックを変更する場合は、proc_exit の前後で必要なリソース解放が漏れていないか確認してください。
