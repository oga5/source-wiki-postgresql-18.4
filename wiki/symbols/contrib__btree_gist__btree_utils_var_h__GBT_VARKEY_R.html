GBT_VARKEY_R

定義

  • 種別: struct
  • 定義箇所: contrib/btree_gist/btree_utils_var.h:14

目的

GiST 索引内に格納された可変長データ型のキーを、比較や範囲判定が可能な「下限値（lower）」と「上限値（upper）」のペアとして展開・保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	bytea	   *lower,
			   *upper;
} GBT_VARKEY_R;

入力と出力

  • 本構造体は、gbt_var_key_readable 関数によって生成されます。
  • 入力として GiST 索引のキー（GBT_VARKEY）を受け取り、展開された lower および upper ポインタを保持する構造体を返します。

前提条件と不変条件

  • リーフノードにおいては、通常 lower と upper は同一の値となります。
  • 保持される lower および upper は bytea 型へのポインタです。

アルゴリズムまたは振る舞い

  • 索引内のバイナリ形式のキーを、プログラムからアクセス可能な形式に変換する抽象化層として機能します。
  • GiST の consistent や union などのメソッド内で、範囲比較を行う際の基点となります。

呼び出し元と呼び出し先

  • 呼び出し元: btree_bit.c, btree_bytea.c, btree_numeric.c, btree_text.c 等の各データ型実装ファイル。
  • 呼び出し先: gbt_var_key_readable 関数（本構造体を生成する関数）が主要なインターフェースです。

関連データ構造

  • GBT_VARKEY: 索引内に格納される可変長キーの物理的な型定義。
  • gbtree_vinfo: 本構造体を利用して比較や変換を行うためのメソッド群を定義する構造体。

失敗モードと境界条件

  • 索引データが破損している場合、gbt_var_key_readable による展開時に不正なメモリ参照が発生する可能性があります。
  • マルチバイト文字を含むデータ型の場合、エンコーディングの最大長（eml）の設定が不適切だと、範囲判定が正しく行われない可能性があります。

保守担当者向けメモ

  • 新しい可変長データ型を btree_gist に追加する際は、本構造体を利用して型固有の比較ロジックを実装してください。
  • メモリ管理については、GBT_FREE_IF_COPY マクロを使用して、コピーされたデータのみを適切に解放するように注意してください。
