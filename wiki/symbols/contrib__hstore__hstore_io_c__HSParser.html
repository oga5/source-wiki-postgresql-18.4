HSParser

定義

  • 種別: struct
  • 定義箇所: contrib/hstore/hstore_io.c:33

目的

hstore 型の文字列形式（key=>value）を解析する際のパース状態を保持する構造体です。

シグネチャまたは形状

typedef struct {
    char       *begin;      /* 入力文字列の開始位置 */
    char       *ptr;        /* 現在の解析位置 */
    char       *cur;        /* 現在処理中のトークン開始位置 */
    char       *word;       /* 現在のキーまたは値のバッファ */
    int         wordlen;    /* 現在の単語長 */
    Node       *escontext;  /* エラーコンテキスト */
    Pairs      *pairs;      /* 解析されたキーと値のペア配列 */
    int         pcur;       /* 現在のペアインデックス */
    int         plen;       /* ペア配列の総容量 */
} HSParser;

入力と出力

  • **入力**: 解析対象の文字列およびエラー報告用の escontext を保持します。
  • **出力**: 解析結果として Pairs 配列を構築し、パースの進行状況を更新します。

前提条件と不変条件

  • pairs は解析開始前に適切なサイズで確保されている必要があります。
  • escontext を介したソフトエラー処理が想定されており、パース失敗時に即座にプロセスを終了させない設計となっています。

アルゴリズムまたは振る舞い

  • parse_hstore() が有限オートマトンを用いて文字列を走査する際、この構造体を通じて現在の解析位置やトークン情報を管理します。
  • get_val() 等の補助関数が、この構造体の ptr や word を更新しながらトークン化を行います。
  • hstoreCheckKeyLength() および hstoreCheckValLength() が、この構造体を参照してキーと値の長さ制限を検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: parse_hstore がこの構造体を初期化し、解析のメインループで使用します。
  • **呼び出し先**: hstoreCheckKeyLength および hstoreCheckValLength がこの構造体を引数として受け取り、検証を行います。また、prssyntaxerror や prseof などの内部関数が状態参照のために使用します。

関連データ構造

  • Pairs: 解析されたキーと値のペアを格納する構造体。
  • Node: エラーコンテキスト（escontext）として使用。

失敗モードと境界条件

  • キーや値が HSTORE_MAX_KEY_LEN 等の制限を超えた場合、hstoreCheckKeyLength/hstoreCheckValLength を通じてエラーが報告されます。
  • パース中に構文エラーが発生した場合、prssyntaxerror が escontext を使用して通知を行います。

保守担当者向けメモ

  • 本構造体は hstore_in 等の入出力関数でパース状態を維持するために使用されます。
  • escontext を使用したエラーハンドリングの実装により、SQL レベルでの安全なエラー報告が可能です。修正時はこのコンテキストの整合性を維持してください。
