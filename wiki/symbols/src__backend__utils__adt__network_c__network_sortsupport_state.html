network_sortsupport_state

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/network.c:46

目的

ネットワーク型（inet / cidr）のソート処理において、省略キー（Abbreviated Key）の生成およびカーディナリティ（値の多様性）推定を行うための状態を保持します。

シグネチャまたは形状

typedef struct {
    int64               input_count;    /* 非NULL値のカウント */
    bool                estimating;     /* カーディナリティ推定中か否か */
    hyperLogLogState    abbr_card;      /* カーディナリティ推定用状態 */
} network_sortsupport_state;

入力と出力

本構造体自体はデータ保持用であり、直接的な入出力関数ではありません。ソートサポートの過程で palloc を介してメモリ確保され、SortSupport 構造体の ssup_extra フィールドに格納されます。

前提条件と不変条件

  • input_count は、ソート対象の非NULL値の処理数に応じてインクリメントされる必要があります。
  • abbr_card は hyperLogLogState を利用しており、ソートの最適化判断のためにカーディナリティの推定値を提供します。

アルゴリズムまたは振る舞い

ソート処理中に network_sortsupport から利用されます。 1. ソート開始時に palloc で確保され、ssup_extra に割り当てられます。 2. network_abbrev_convert などの関数内で、現在の処理状況（input_count）や推定状態（estimating）を追跡するために参照されます。 3. hyperLogLogState を用いて、省略キーによるソートが有効かどうかを判断するための統計情報を蓄積します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network.c: network_sortsupport 関数内でインスタンスが生成（palloc）されます。
  • src/backend/utils/adt/network.c: network_abbrev_convert 等のソート関連関数内で ssup->ssup_extra を通じて参照されます。
  • **呼び出し先**:
  • hyperLogLog 関連のライブラリ関数（abbr_card フィールド経由）。

関連データ構造

  • SortSupport: ソート最適化のための共通インターフェース。本構造体はこの ssup_extra に格納されます。
  • hyperLogLogState: カーディナリティ推定に使用されるデータ構造。

失敗モードと境界条件

  • メモリ不足: palloc 失敗時にエラーが発生します。
  • 推定精度の限界: hyperLogLogState の特性上、非常に大規模なデータセットでは推定値に誤差が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は network_sortsupport の実行コンテキストを維持するためのものです。
  • abbr_card の更新ロジックを変更する場合は、lib/hyperloglog.h の仕様と network_abbrev_convert での利用方法を併せて確認してください。
