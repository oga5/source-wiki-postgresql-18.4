verify_hash_page

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/hashfuncs.c:58

目的

ハッシュインデックスの物理ページ（bytea）が有効な構造を持っているかを検証し、必要に応じてページタイプを制限・確認するゲートキーパー関数です。

シグネチャまたは形状

static Page verify_hash_page(bytea *raw_page, int flags)

入力と出力

  • **入力**:
  • raw_page: get_raw_page で取得された生のページデータ。
  • flags: 期待されるページタイプを示すビットマスク（LH_META_PAGE, LH_BUCKET_PAGE, LH_OVERFLOW_PAGE 等）。0 の場合はタイプ制限を行いません。
  • **出力**:
  • 正常時は検証済みの Page ポインタを返します。
  • 不正なページや期待するタイプと一致しない場合はエラーを発生させます。

前提条件と不変条件

  • 入力ページが PageIsNew でない場合、PageGetSpecialSize が HashPageOpaqueData のサイズと一致する必要があります。
  • ページヘッダー内の hasho_page_id が HASHO_PAGE_ID と一致する必要があります。
  • メタページの場合、hashm_magic が HASH_MAGIC、hashm_version が HASH_VERSION と一致する必要があります。

アルゴリズムまたは振る舞い

1. get_page_from_raw を呼び出し、bytea から Page を抽出します。 2. ページが新規作成済みであれば、HashPageOpaque を取得し、ページIDと特殊領域サイズを検証します。 3. ページタイプ（hasho_flag）を特定し、有効なハッシュインデックスのページタイプ（メタ、バケット、オーバーフロー、ビットマップ、未使用）のいずれかであることを確認します。 4. flags が指定されている場合、特定されたページタイプがそのマスクに含まれているかを確認し、不一致ならエラーを報告します。 5. ページタイプがメタページである場合、さらにマジックナンバーとバージョン番号を検証します。 6. 検証を通過した Page ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/hashfuncs.c 内の各関数（hash_page_type, hash_page_stats, hash_page_items, hash_metapage_info 等）。
  • **呼び出し先**:
  • get_page_from_raw（pageinspect.h 経由）
  • HashPageGetOpaque
  • HashPageGetMeta
  • ereport / elog

関連データ構造

  • HashPageOpaqueData: ハッシュインデックスのページ末尾に配置されるメタデータ。
  • HashMetaPage: ハッシュインデックスのメタページ構造体。

失敗モードと境界条件

  • **ERRCODE_INVALID_PARAMETER_VALUE**: ページIDの不一致、特殊領域サイズの不整合、無効なページタイプ、または指定した flags と実際のページタイプが一致しない場合に発生します。
  • **ERRCODE_INDEX_CORRUPTED**: メタページのマジックナンバーやバージョンが期待値と異なる場合に発生します。

保守担当者向けメモ

  • 本関数は contrib/pageinspect のハッシュインデックス関連関数における唯一の検証レイヤーです。新しいハッシュインデックス関連の解析関数を追加する際は、必ず本関数を呼び出してページを検証してください。
  • flags 引数による検証は、特定の関数が特定のページタイプ（例：メタページのみ）を期待する場合に利用されます。
