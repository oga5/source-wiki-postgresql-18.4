plpgsql_extra_errors_assign_hook

定義

  • 種別: function
  • 定義箇所: src/pl/plpgsql/src/pl_handler.c:135

目的

PL/pgSQL の設定パラメータ plpgsql.extra_errors が変更された際に、その値を内部変数 plpgsql_extra_errors に反映させるための GUC（Grand Unified Configuration）割り当てフックです。

シグネチャまたは形状

static void plpgsql_extra_errors_assign_hook(const char *newvalue, void *extra)

入力と出力

  • newvalue: 設定される新しい値の文字列（本関数内では未使用）。
  • extra: 解析済みの設定値へのポインタ。int 型としてキャストして使用されます。
  • 戻り値: なし。

前提条件と不変条件

  • extra 引数は、有効な int 型の値へのポインタであることが前提です。
  • PostgreSQL の GUC フレームワークによって呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. extra ポインタを int * にキャストします。 2. キャストしたポインタが指す値を参照し、グローバル変数 plpgsql_extra_errors に代入します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQL の GUC 管理システム（src/pl/plpgsql/src/pl_handler.c:200 の設定定義内で参照）。
  • 呼び出し先: なし。

関連データ構造

  • plpgsql_extra_errors: 本関数によって更新されるグローバル変数。

失敗モードと境界条件

  • extra が NULL または不正なメモリ領域を指している場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は GUC の assign_hook として登録されています。設定値のバリデーションは別途 check_hook で行われるため、本関数内では代入処理のみに専念しています。
