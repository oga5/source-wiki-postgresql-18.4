table_tuple_get_latest_tid

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableam.c:235

目的

指定されたタプルID（TID）が指すタプルの、現在のトランザクションから見える最新のバージョンを特定し、引数の tid を更新します。

シグネチャまたは形状

void table_tuple_get_latest_tid(TableScanDesc scan, ItemPointer tid)

入力と出力

  • **入力**:
  • scan: テーブルスキャン記述子。
  • tid: 検索対象のタプルID（入力時に使用され、出力時に最新のTIDで上書きされる）。
  • **出力**: なし（tid ポインタ経由で結果を返却）。

前提条件と不変条件

  • scan は有効な TableScanDesc であること。
  • 論理デコード中（CheckXidAlive が有効かつ bsysscan が偽）の呼び出しは禁止されており、エラーが発生します。

アルゴリズムまたは振る舞い

1. **論理デコードチェック**: 論理デコード実行中の不正な呼び出しを検出し、エラーをスローします。 2. **TID検証**: tableam->tuple_tid_valid を呼び出し、入力された tid がリレーションに対して有効か確認します。無効な場合は ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告します。 3. **ディスパッチ**: リレーションのアクセスメソッド（rd_tableam）の tuple_get_latest_tid 関数を呼び出し、実際の最新TID取得処理を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/executor/nodeTidscan.c
  • src/backend/utils/adt/tid.c
  • **呼び出し先**:
  • rel->rd_tableam->tuple_tid_valid
  • rel->rd_tableam->tuple_get_latest_tid

関連データ構造

  • TableScanDesc: スキャン状態を保持する構造体。
  • TableAmRoutine: テーブルアクセスメソッドの関数ポインタ群を保持する構造体。
  • ItemPointer: タプルID（TID）を表現する型。

失敗モードと境界条件

  • **論理デコード中の呼び出し**: elog(ERROR) によりトランザクションを中断します。
  • **無効なTID**: 入力された tid がリレーションの範囲外である場合、ereport(ERROR) によりエラーを報告します。

保守担当者向けメモ

  • 本関数は rd_tableam を介したディスパッチャです。具体的な動作は各ストレージ実装（heapam.c など）の tuple_get_latest_tid 実装に依存します。
  • tid はユーザー入力を含む可能性があるため、検証ステップを省略しないでください。
