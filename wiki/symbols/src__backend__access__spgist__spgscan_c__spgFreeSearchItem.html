spgFreeSearchItem

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgscan.c:83

目的

SP-GiST スキャン中に使用される SpGistSearchItem 構造体およびその内部で保持されている動的メモリを解放します。

シグネチャまたは形状

void spgFreeSearchItem(SpGistScanOpaque so, SpGistSearchItem *item)

入力と出力

  • **入力**:
  • so: スキャン状態を保持する SpGistScanOpaque 構造体。属性の型情報（attType または attLeafType）の参照に使用されます。
  • item: 解放対象の SpGistSearchItem 構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • item が指すメモリ領域は、palloc 等で確保された有効な領域である必要があります。
  • item->value がポインタ型（attbyval が偽）の場合、そのデータは palloc で確保されている必要があります。

アルゴリズムまたは振る舞い

1. **値の解放**: item->isLeaf の状態に応じて so->state.attType または so->state.attLeafType を参照し、attbyval が偽であれば item->value を pfree します。 2. **リーフタプルの解放**: item->leafTuple が存在する場合、pfree を実行します。 3. **トラバーサル値の解放**: item->traversalValue が存在する場合、pfree を実行します。 4. **構造体自体の解放**: 最後に item ポインタ自体を pfree します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgscan.c:919 にて呼び出されていることが確認されています。
  • **呼び出し先**: pfree（PostgreSQL のメモリ解放関数）を呼び出します。

関連データ構造

  • SpGistSearchItem: 検索キューで管理される項目。
  • SpGistScanOpaque: スキャン中の状態を保持する構造体。

失敗モードと境界条件

  • item が NULL の場合、未定義の動作を引き起こす可能性があります（ソースコード上でのチェックは行われていません）。
  • item->value が NULL であっても DatumGetPointer の結果が NULL であれば解放はスキップされます。

保守担当者向けメモ

  • 本関数は、SpGistSearchItem が保持する複数の動的メモリ領域を適切にクリーンアップする役割を担っています。新しいフィールドが SpGistSearchItem に追加された場合は、本関数での解放処理も更新してください。
