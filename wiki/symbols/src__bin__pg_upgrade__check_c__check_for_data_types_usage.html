check_for_data_types_usage

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/check.c:463

目的

旧クラスタ内の全データベースをスキャンし、アップグレードの互換性を損なう可能性のあるデータ型の使用状況を検証する。

シグネチャまたは形状

static void check_for_data_types_usage(ClusterInfo *cluster)

入力と出力

  • **入力**: ClusterInfo *cluster（検証対象の旧クラスタ情報）
  • **出力**: なし（検証失敗時は pg_fatal によりプロセスを終了する）

前提条件と不変条件

  • data_types_usage_checks 配列が有効なチェック定義を保持していること。
  • upgrade_task_create() がタスク管理用のメモリを確保できること。

アルゴリズムまたは振る舞い

1. data_types_usage_checks 配列を走査し、現在のクラスタバージョンに適用可能なチェック項目を特定する。 2. 各チェック項目に対し、data_type_check_query() で生成されたSQLクエリを UpgradeTask に登録する。 3. upgrade_task_run() を呼び出し、各データベースに対して順次クエリを実行する。 4. 各ステップの実行結果（process_data_type_check）を収集し、エラーが報告された場合は pg_fatal で処理を中断する。 5. 最後に確保したメモリ（クエリ文字列、状態管理用構造体）を解放し、check_ok() を呼び出して完了を通知する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c:637 (check_and_dump_old_cluster 内)
  • **呼び出し先**:
  • upgrade_task_create, upgrade_task_add_step, upgrade_task_run, upgrade_task_free
  • data_type_check_query
  • process_data_type_check (コールバック)
  • check->version_hook (条件付き)
  • pg_malloc0, pg_free, pg_fatal, check_ok

関連データ構造

  • ClusterInfo: クラスタのメタデータ。
  • DataTypesUsageChecks: チェック対象のデータ型やバージョン閾値を定義する構造体。
  • UpgradeTask: データベースに対する一連の検証タスクを管理する構造体。
  • data_type_check_state: 各チェックの実行状態とレポート用バッファを保持する構造体。

失敗モードと境界条件

  • チェック対象のデータ型が検出された場合、report バッファに詳細が蓄積され、関数終了時に pg_fatal が発生する。
  • threshold_version が MANUAL_CHECK の場合、version_hook の判定結果に従ってチェックがスキップされることがある。

保守担当者向けメモ

  • 本関数は、将来のバージョンアップで発生する破壊的変更に対応するため、data_types_usage_checks 配列を拡張することで容易にチェック項目を追加できるよう設計されている。
  • 再帰的なCTEを含むクエリが実行されるため、複雑な型依存関係を持つデータベースでは実行時間に注意が必要である。
