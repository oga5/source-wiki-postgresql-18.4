leftmostvalue_int4

定義

  • 種別: function
  • 定義箇所: contrib/btree_gin/btree_gin.c:217

目的

int4 型（4バイト整数）における最小値を返し、GINインデックスを用いた範囲検索においてスキャン開始点を決定するために使用されます。

シグネチャまたは形状

static Datum leftmostvalue_int4(void)

入力と出力

  • 入力: なし
  • 出力: Datum 型として表現された INT_MIN（int32 の最小値）。

前提条件と不変条件

  • 本関数は int4 型の値を扱うコンテキストでのみ呼び出されることを前提としています。
  • 返される値は、int4 型の定義域における絶対的な最小値である必要があります。

アルゴリズムまたは振る舞い

1. INT_MIN（limits.h で定義される int32 の最小値）を取得します。 2. Int32GetDatum マクロを使用して、その値を PostgreSQL の内部データ表現である Datum 型に変換して返します。

呼び出し元と呼び出し先

  • 呼び出し元: GIN_SUPPORT(int4, false, leftmostvalue_int4, btint4cmp) マクロ内で参照されています。
  • 呼び出し先: Int32GetDatum（PostgreSQL 内部マクロ）。

関連データ構造

  • Datum: PostgreSQL の汎用データ表現型。
  • GIN_SUPPORT: GINインデックスのサポート関数を登録するためのマクロ。

失敗モードと境界条件

  • 境界条件: int4 型の最小値（-2,147,483,648）を正確に返すことが本関数の責務です。
  • 失敗モード: 特筆すべき失敗モードはありません。

保守担当者向けメモ

  • 本関数は GIN_SUPPORT マクロを通じて、int4 型のインデックススキャン範囲を決定する際に利用されます。
  • 新しい整数型を追加する際は、同様のパターンで最小値を返す関数を定義し、GIN_SUPPORT に登録してください。
