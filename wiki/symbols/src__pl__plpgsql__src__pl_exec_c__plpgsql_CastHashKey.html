plpgsql_CastHashKey

定義

  • 種別: struct
  • 定義箇所: src/pl/plpgsql/src/pl_exec.c:152

目的

PL/pgSQL 実行エンジンにおいて、型変換（キャスト）情報を検索するためのハッシュキーとして使用されます。

シグネチャまたは形状

typedef struct {
    Oid     srctype;    /* ソース型 */
    Oid     dsttype;    /* 変換先型 */
    int32   srctypmod;  /* ソース型の typmod */
    int32   dsttypmod;  /* 変換先型の typmod */
} plpgsql_CastHashKey;

入力と出力

  • 入力: 変換元および変換先の OID と typmod の組み合わせ。
  • 出力: ハッシュテーブル検索用のキー構造体。

前提条件と不変条件

  • 構造体内にパディングバイトが含まれないことが前提とされています。
  • ハッシュテーブルのキーとして使用する場合、構造体の先頭に配置する必要があります。

アルゴリズムまたは振る舞い

  • 型変換のキャッシュ検索において、特定の型ペアと typmod の組み合わせを一意に識別するために使用されます。
  • ハッシュテーブルのサイズ設定（ctl.keysize）において、sizeof(plpgsql_CastHashKey) が参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/pl_exec.c 内のハッシュテーブル初期化処理および検索処理（4041, 4054, 4068, 7946 行付近）。
  • 呼び出し先: 情報なし。

関連データ構造

  • plpgsql_CastHashEntry: このキーを保持するハッシュエントリ構造体。
  • plpgsql_CastExprHashEntry: キャスト式を保持するハッシュエントリ構造体。

失敗モードと境界条件

  • 構造体内にパディングが発生する環境では、ハッシュ検索の整合性が損なわれるリスクがあります。

保守担当者向けメモ

  • 本構造体はハッシュキーとして機能するため、メンバの追加や変更を行う際は、パディングが発生しないよう注意してください。
  • plpgsql_CastHashEntry や plpgsql_CastExprHashEntry の先頭メンバとして定義されているため、構造体のレイアウト変更はこれらのデータ構造にも影響を与えます。
