Offset

定義

  • 種別: typedef
  • 定義箇所: src/include/c.h:603

目的

メモリ上の位置やデータ構造内の相対的な距離を表現するための整数型です。

シグネチャまたは形状

typedef signed int Offset;

入力と出力

  • 入力: 符号付き整数値。
  • 出力: メモリ上のオフセット値またはインデックス値として利用されます。

前提条件と不変条件

  • signed int 型として定義されており、プラットフォーム依存の整数サイズに従います。

アルゴリズムまたは振る舞い

本シンボルは型定義であり、実行時のアルゴリズムは持ちません。メモリ上のバイト単位の距離や、構造体内の相対位置を保持するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • src/backend/storage/page/bufpage.c: ページヘッダー内のオフセット管理（upper, copy_tail, copy_head, lastoff）。
  • src/backend/storage/ipc/shm_toc.c: 共有メモリのTOC（Table of Contents）における開始位置からのバイトオフセット。
  • contrib/amcheck/verify_nbtree.c: インデックスブロック内のオフセット管理。
  • 呼び出し先: なし（プリミティブ型へのエイリアス）。

関連データ構造

  • PageHeaderData（bufpage.c 等で使用）: ページ内のデータ配置を管理する構造体。
  • shm_toc（shm_toc.c で使用）: 共有メモリの管理構造体。

失敗モードと境界条件

  • 符号付き整数であるため、負の値が許容される文脈と、正の値のみを想定する文脈（メモリ上の絶対位置など）が存在します。使用箇所においてオーバーフローや負の値の妥当性を検証する必要があります。

保守担当者向けメモ

  • int 型のサイズはプラットフォームに依存するため、移植性を考慮したコードでは Size 型（size_t）との混同に注意してください。
  • 本型は c.h で定義されているため、PostgreSQL のほぼすべてのソースファイルから参照可能です。変更時は広範囲への影響を考慮してください。
