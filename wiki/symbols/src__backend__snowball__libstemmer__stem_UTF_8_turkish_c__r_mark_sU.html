r_mark_sU

目的

トルコ語のステミング処理において、特定の接尾辞（sU）のパターンマッチングと除去を制御する。

シグネチャまたは形状

static int r_mark_sU(struct SN_env * z)

入力と出力

  • **入力**: z (Snowball ステミング環境を保持する SN_env 構造体へのポインタ)
  • **出力**: 成功時に 1、失敗または処理対象外の場合に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 内部で呼び出される r_check_vowel_harmony および r_mark_suffix_with_optional_s_consonant が正常に動作する環境であること。

アルゴリズムまたは振る舞い

1. r_check_vowel_harmony(z) を呼び出し、母音調和のルールを検証する。結果が 0 以下であればその値を返して終了する。 2. in_grouping_b_U(z, g_U, 105, 305, 0) を使用し、現在のカーソル位置で g_U グループに含まれる文字を後方検索する。マッチしない場合は 0 を返して終了する。 3. r_mark_suffix_with_optional_s_consonant(z) を呼び出し、接尾辞の処理を継続する。その戻り値をそのまま返す。

  • g_U: 母音グループを定義する静的データ（詳細はソースファイル内の定義を参照）。

失敗モードと境界条件

  • r_check_vowel_harmony が 0 以下を返した場合、処理は中断される。
  • in_grouping_b_U で対象文字が見つからない場合、処理は 0 を返し終了する。
  • r_mark_suffix_with_optional_s_consonant がエラー（負の値）を返した場合、そのエラーコードが伝播する。
