appendStringInfo

定義

  • 種別: function
  • 定義箇所: src/common/stringinfo.c:144

目的

printf 形式のフォーマット文字列を StringInfo バッファの末尾に追加する。バッファ容量が不足している場合は自動的に拡張を行う。

シグネチャまたは形状

void appendStringInfo(StringInfo str, const char *fmt, ...)

入力と出力

  • **入力**:
  • str: 操作対象の StringInfo 構造体へのポインタ。
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（str が指すバッファの内容が更新される）。

前提条件と不変条件

  • str は初期化済みであること。
  • maxlen == 0 の読み取り専用バッファに対して呼び出した場合、内部の enlargeStringInfo が Assert により停止する可能性がある。

アルゴリズムまたは振る舞い

1. 現在の errno を保存する。 2. ループ内で appendStringInfoVA を呼び出し、フォーマット済みデータの書き込みを試みる。 3. appendStringInfoVA が 0 を返せば成功として終了する。 4. 0 以外（必要な追加バッファサイズ）が返された場合、enlargeStringInfo を呼び出してバッファを拡張し、再度ループの先頭に戻って書き込みを再試行する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/dblink/dblink.c 等の各関数から、SQL クエリの動的構築のために頻繁に呼び出される。
  • **呼び出し先**:
  • appendStringInfoVA: フォーマット処理と書き込みの実行。
  • enlargeStringInfo: バッファの再確保（容量不足時）。

関連データ構造

  • StringInfoData: バッファの実体。data（文字列）、len（現在長）、maxlen（最大容量）を保持する。

失敗モードと境界条件

  • メモリ不足時: palloc/repalloc が失敗した場合、PostgreSQL のメモリ管理機構によりエラーがスローされる。
  • 読み取り専用バッファ: maxlen が 0 の場合、拡張不能のため異常終了する。

保守担当者向けメモ

  • va_list の再走査を避けるため、内部で appendStringInfoVA をループ呼び出しする設計となっている。
  • 頻繁な追加が発生する場合は、事前に enlargeStringInfo で十分なサイズを確保することでオーバーヘッドを削減可能である。
