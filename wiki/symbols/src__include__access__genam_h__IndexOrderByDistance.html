IndexOrderByDistance

定義

  • 種別: struct
  • 定義箇所: src/include/access/genam.h:148

目的

インデックススキャンにおいて、検索対象とインデックス項目間の「距離」を保持するために使用される構造体です。主に距離に基づいた順序付け（Nearest Neighbor 検索など）を行う際に利用されます。

シグネチャまたは形状

typedef struct IndexOrderByDistance
{
	double		value;
	bool		isnull;
} IndexOrderByDistance;

入力と出力

  • **入力**: 距離計算の結果（double 型の value）および値が有効か否かを示すフラグ（bool 型の isnull）。
  • **出力**: インデックススキャン実行中に、距離情報として呼び出し元へ返されます。

前提条件と不変条件

  • isnull が true の場合、value の内容は無視されます。
  • 距離計算が不可能な場合や値が欠損している場合に isnull を使用して表現します。

アルゴリズムまたは振る舞い

インデックススキャン（特に GiST や SP-GiST 等の距離計算をサポートする AM）において、各インデックス項目に対する距離を格納するコンテナとして機能します。スキャン実行時に、計算された距離値がこの構造体に書き込まれ、呼び出し元へ渡されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/gist/gistget.c
  • src/backend/access/index/indexam.c
  • src/backend/access/spgist/spgscan.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出しは含みません。

関連データ構造

  • gist_private.h: IndexOrderByDistance の配列（distances）を保持する構造体定義が含まれます。
  • spgist_private.h: MaxIndexTuplesPerPage サイズの IndexOrderByDistance 配列を定義しています。

失敗モードと境界条件

  • 距離計算がサポートされていないインデックス型に対して本構造体を使用した場合の挙動は、各 AM の実装に依存します。
  • 境界条件に関する明示的な制約はソースコード上には記載されていません。

保守担当者向けメモ

  • 本構造体は、インデックススキャンにおける距離情報の受け渡しという特定の目的に特化しています。
  • gist_private.h や spgist_private.h で配列として利用されているため、メモリレイアウトやアライメントを変更する際はこれらのファイルへの影響を考慮してください。
