toast_tuple_find_biggest_attribute

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/toast_helper.c:180

目的

TOAST処理において、圧縮または外部化の候補となる属性のうち、サイズが最大である属性のインデックスを特定します。

シグネチャまたは形状

int toast_tuple_find_biggest_attribute(ToastTupleContext *ttc, bool for_compression, bool check_main)

入力と出力

  • **入力**:
  • ttc: 処理対象のタプル状態を保持する ToastTupleContext へのポインタ。
  • for_compression: true の場合、圧縮対象の属性を探します。
  • check_main: true の場合、TYPSTORAGE_MAIN の属性を対象とします。false の場合は TYPSTORAGE_EXTENDED または TYPSTORAGE_EXTERNAL を対象とします。
  • **出力**:
  • 候補として見つかった属性のインデックス（0から始まる整数）。見つからない場合は -1 を返します。

前提条件と不変条件

  • ttc は toast_tuple_init によって適切に初期化されている必要があります。
  • 戻り値のインデックスは ttc->ttc_values および ttc->ttc_attr 配列の有効な範囲内です。

アルゴリズムまたは振る舞い

1. for_compression が指定された場合、TOASTCOL_INCOMPRESSIBLE フラグを持つ属性をスキップ対象に追加します。 2. タプルの全属性を走査し、以下の条件に該当する属性を候補から除外します。

  • TOASTCOL_IGNORE フラグが設定されている。
  • 既に外部化されている（VARATT_IS_EXTERNAL）。
  • 圧縮済みであり、かつ for_compression が真である。
  • check_main の指定と属性のストレージ戦略（attstorage）が一致しない。

    • 3. 除外されなかった属性のうち、tai_size が現在保持している最大サイズ（初期値は MAXALIGN(TOAST_POINTER_SIZE)）を超えるものを探索し、そのインデックスを更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heaptoast.c (複数の箇所で、圧縮や外部化の判断ロジックとして呼び出されています)
  • **呼び出し先**:
  • 直接的な関数呼び出しはソース内にありませんが、TupleDescAttr マクロや VARATT_IS_* マクロを使用して属性情報を参照します。

関連データ構造

  • ToastTupleContext: 処理中のタプル属性情報（サイズ、フラグ、値）を保持します。
  • TupleDesc: タプルの属性定義情報。

失敗モードと境界条件

  • 候補となる属性が存在しない場合、-1 を返します。
  • check_main の設定により、対象となるストレージ戦略を持つ属性が一つも存在しない場合も -1 を返します。

保守担当者向けメモ

  • 本関数は、どの属性を優先的に処理すべきかを決定するヒューリスティックを提供します。実際の物理的な圧縮や外部化処理は行いません。
  • check_main フラグの切り替えにより、TOASTの段階的な処理（まず EXTENDED を処理し、次に MAIN を処理する等）を制御しています。
