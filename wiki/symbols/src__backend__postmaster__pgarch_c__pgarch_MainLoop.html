pgarch_MainLoop

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/pgarch.c:310

目的

WALアーカイバプロセスのメイン実行ループを制御し、アーカイブ対象ファイルの監視、転送処理の実行、およびプロセス終了シグナルの管理を行う。

シグネチャまたは形状

static void pgarch_MainLoop(void)

入力と出力

  • **入力**: グローバル変数 ready_to_stop、ShutdownRequestPending、last_sigterm_time、および共有メモリ上のラッチ MyLatch を参照。
  • **出力**: なし（副作用としてWALファイルのアーカイブを実行し、プロセスを終了させる）。

前提条件と不変条件

  • 本関数はアーカイバプロセス内で実行されることを前提とする。
  • MyLatch はループの各反復開始時にリセットされる。

アルゴリズムまたは振る舞い

1. **ループ開始**: ready_to_stop を確認し、終了フラグを初期化。 2. **割り込み処理**: ProcessPgArchInterrupts() を呼び出し、設定変更やバリアイベントを処理。 3. **シャットダウン監視**: ShutdownRequestPending が真の場合、SIGTERM受信から60秒経過した時点でループを脱出する。 4. **アーカイブ実行**: pgarch_ArchiverCopyLoop() を呼び出し、アーカイブ対象のWALファイルを処理。 5. **待機**: WaitLatch() を使用し、以下のいずれかまで待機する。

  • シグナルの受信（WL_LATCH_SET）
  • PGARCH_AUTOWAKE_INTERVAL の経過（WL_TIMEOUT）
  • Postmasterの終了（WL_POSTMASTER_DEATH）

    • 6. **終了判定**: Postmasterの終了またはSIGUSR2受信後の最終サイクル完了時にループを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: PgArchiverMain（src/backend/postmaster/pgarch.c:272）から呼び出される。
  • **呼び出し先**:
  • ProcessPgArchInterrupts()
  • pgarch_ArchiverCopyLoop()
  • ResetLatch() / WaitLatch()

関連データ構造

  • MyLatch: プロセス間通信および待機制御に使用。
  • ready_to_stop, ShutdownRequestPending, last_sigterm_time: プロセスの状態管理用グローバル変数。

失敗モードと境界条件

  • **Postmaster終了**: WaitLatch が WL_POSTMASTER_DEATH を検知した場合、即座にループを終了する。
  • **SIGTERM受信**: 受信から60秒経過すると、アーカイブ処理の完了を待たずにプロセスを終了する。
  • **アーカイブ失敗**: pgarch_ArchiverCopyLoop 内で例外処理が行われるため、本ループ自体は継続してリトライを試みる。

保守担当者向けメモ

  • 本ループは PGARCH_AUTOWAKE_INTERVAL ごとに自動起床し、プロアクティブなアーカイブを保証する設計となっている。
  • SIGTERM受信時の60秒待機は、Postmasterが新しいアーカイバを起動できるようにするための安全装置である。
