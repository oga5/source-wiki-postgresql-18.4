Node

定義

  • 種別: typedef
  • 定義箇所: src/include/nodes/nodeFuncs.h:40

目的

PostgreSQLのクエリツリーや式ツリーを構成する基本単位です。ツリー構造の走査、変形、および特定のノード型の判定や属性抽出を行うための汎用的なインターフェースを提供します。

シグネチャまたは形状

Node は構造体へのポインタとして扱われることが一般的です。具体的な構造定義は src/include/nodes/nodes.h に存在します。

入力と出力

  • **入力**: walker や mutator 関数において、処理対象のノードとして Node * 型が渡されます。
  • **出力**: 走査結果や変形後のノードが返されます。

前提条件と不変条件

  • Node はツリー構造のノードとして一貫性を保つ必要があります。
  • walker や mutator を使用する場合、コールバック関数は Node *node を引数として受け取ります。

アルゴリズムまたは振る舞い

  • **走査**: expression_tree_walker や query_tree_walker を用いて、ツリーを再帰的に探索します。
  • **変形**: expression_tree_mutator 等を用いて、ツリーの構造を再帰的に書き換えます。
  • **判定**: IsA マクロや、is_funcclause などのインライン関数を使用して、ノードの具体的な型を識別します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/file_fdw/file_fdw.c（makeDefElem や pull_varattnos の引数として使用）、contrib/hstore/hstore_io.c など、PostgreSQLの拡張モジュールやコア機能の多くから参照されています。
  • **呼び出し先**: 本シンボル自体はデータ構造の型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • Query: クエリツリーのルートノード。
  • Expr: 式ノードの基底。
  • List: ノードの集合を保持するリスト構造。

失敗モードと境界条件

  • **スタックオーバーフロー**: ツリー構造が極端に深い場合、再帰的な走査や変形によってスタックオーバーフローが発生するリスクがあります。
  • **型不整合**: Node * から特定のノード型へキャストする際、IsA マクロによる型チェックを怠ると実行時エラーの原因となります。

保守担当者向けメモ

  • Node は PostgreSQL のノードシステムの根幹です。変更を加える際は src/include/nodes/nodes.h および src/include/nodes/nodeFuncs.h の両方の整合性に注意してください。
  • walker/mutator のコールバックは void *context を受け取るため、キャスト時の型安全性には十分注意を払う必要があります。
