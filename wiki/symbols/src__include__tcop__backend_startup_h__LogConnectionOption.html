LogConnectionOption

定義

  • 種別: enum
  • 定義箇所: src/include/tcop/backend_startup.h:74

目的

バックエンドプロセスにおける接続関連イベントのログ出力を制御するためのビットマスクフラグを定義します。

シグネチャまたは形状

typedef enum LogConnectionOption
{
    LOG_CONNECTION_RECEIPT = (1 << 0),
    LOG_CONNECTION_AUTHENTICATION = (1 << 1),
    LOG_CONNECTION_AUTHORIZATION = (1 << 2),
    LOG_CONNECTION_SETUP_DURATIONS = (1 << 3),
    LOG_CONNECTION_ON = ...,
    LOG_CONNECTION_ALL = ...,
} LogConnectionOption;

入力と出力

  • **入力**: GUC設定等を通じて本列挙型の値が参照され、接続確立フェーズにおけるログ出力の可否を決定します。
  • **出力**: 接続確立の各段階（受信、認証、認可、所要時間計測）におけるログ出力の制御フラグとして機能します。

前提条件と不変条件

  • 各メンバはビット演算による組み合わせを前提としています。
  • LOG_CONNECTION_ON は受信、認証、認可の論理和です。
  • LOG_CONNECTION_ALL は上記に加え、所要時間計測（SETUP_DURATIONS）を含む全フラグの論理和です。

アルゴリズムまたは振る舞い

  • 各フラグはビットシフトにより定義されており、特定の接続フェーズに対応します。
  • 接続確立プロセスにおいて、現在の設定値と各フラグをビット単位で比較することで、該当フェーズのログを出力するか否かを判定します。

呼び出し元と呼び出し先

  • 呼び出し元および呼び出し先の具体的な関数情報は、提供されたソース抜粋からは特定できません。

関連データ構造

  • ConnectionTiming: 接続確立の各段階の時間を記録する構造体であり、本列挙型と組み合わせてログ出力の判断に使用されます。

失敗モードと境界条件

  • 該当する情報はありません。

保守担当者向けメモ

  • 本列挙型はビットマスクとして使用されるため、新しいログカテゴリを追加する際は既存のビット位置と重複しないように注意してください。
  • LOG_CONNECTION_ON および LOG_CONNECTION_ALL の定義変更時は、論理和の構成要素に漏れがないか確認してください。
