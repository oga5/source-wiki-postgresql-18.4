xl_hash_init_meta_page

定義

  • 種別: struct
  • 定義箇所: src/include/access/hash_xlog.h:217

目的

ハッシュインデックスのメタページを初期化する際のWAL（Write-Ahead Logging）レコードのデータ構造を定義します。

シグネチャまたは形状

typedef struct xl_hash_init_meta_page
{
    double      num_tuples;
    RegProcedure procid;
    uint16      ffactor;
} xl_hash_init_meta_page;

入力と出力

  • **入力**: ハッシュインデックス作成時に設定されるタプル数（num_tuples）、プロシージャID（procid）、およびフィルファクタ（ffactor）。
  • **出力**: WALレコードのペイロードとして書き込まれる上記構造体のバイナリデータ。

前提条件と不変条件

  • 本構造体は、ハッシュインデックスのメタページ初期化操作に関連付けられたWALレコードのペイロードとして使用されます。
  • ffactor はメタページのフィルファクタ設定を保持します。

アルゴリズムまたは振る舞い

1. **書き込み**: ハッシュインデックスの初期化時、この構造体に値を格納し、WALレコードとして記録します。 2. **リカバリ**: リカバリ処理（hash_redo）において、WALレコードから本構造体を読み出し、メタページの初期化状態を再現するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hashpage.c: メタページ初期化時に本構造体へ値を設定。
  • **呼び出し先（参照元）**:
  • src/backend/access/hash/hash_xlog.c: リカバリ処理中に XLogRecGetData を通じて本構造体を読み取り。
  • src/backend/access/rmgrdesc/hashdesc.c: WALレコードのログ出力（デバッグ・解析用）にて参照。

関連データ構造

  • src/include/access/hash.h: ハッシュインデックスのメタページ定義。

失敗モードと境界条件

  • WALレコードの読み取り時に、レコードサイズが本構造体のサイズと一致しない場合、リカバリ処理が失敗する可能性があります。
  • offsetof を用いたサイズ計算（src/include/access/hash_xlog.h:225）が、構造体のパディングやアライメントに依存するため、プラットフォーム間で一貫性がある必要があります。

保守担当者向けメモ

  • 本構造体を変更する場合は、hash_redo および hash_desc の実装も併せて更新し、リカバリ時の互換性を維持してください。
