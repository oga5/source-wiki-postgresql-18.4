TupStoreStatus

定義

  • 種別: enum
  • 定義箇所: src/backend/utils/sort/tuplestore.c:72

目的

Tuplestorestate における現在のタプル格納状態を識別し、メモリと一時ファイル（BufFile）間のデータ移動および読み書きの制御を管理する。

シグネチャまたは形状

typedef enum
{
    TSS_INMEM,      /* メモリ内にタプルが収まっている状態 */
    TSS_WRITEFILE,  /* 一時ファイルへ書き出し中の状態 */
    TSS_READFILE,   /* 一時ファイルから読み込み中の状態 */
} TupStoreStatus;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: Tuplestorestate 構造体内の status フィールドとして保持される。

前提条件と不変条件

  • TSS_INMEM から TSS_WRITEFILE への遷移は、メモリ制限（maxKBytes）を超過した際に発生する。
  • TSS_WRITEFILE と TSS_READFILE の切り替えは、書き込み操作と読み取り操作が混在する際に BufFileTell および BufFileSeek を伴って行われる。

アルゴリズムまたは振る舞い

  • **TSS_INMEM**: タプルはメモリ上の memtuples 配列に格納される。
  • **TSS_WRITEFILE**: メモリ制限を超えた場合、dumptuples が呼び出され、全タプルが一時ファイルへ書き出される。以降の追加タプルも直接ファイルへ書き込まれる。
  • **TSS_READFILE**: 一時ファイルからタプルを読み出す状態。必要に応じて readtup を用いてメモリへ復元される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/sort/tuplestore.c 内の Tuplestorestate 構造体（105行目）にて状態保持用として参照されている。
  • 呼び出し先: 情報なし。

関連データ構造

  • Tuplestorestate: 本状態を保持するメインの構造体。
  • BufFile: TSS_WRITEFILE および TSS_READFILE 状態で使用される一時ファイル管理構造体。

失敗モードと境界条件

  • メモリ不足時に TSS_INMEM から TSS_WRITEFILE へ遷移する際、一時ファイル作成に失敗した場合はエラーとなる。

保守担当者向けメモ

  • 本列挙型は tuplestore.c の内部状態管理に特化している。状態遷移のロジックは tuplestore_puttupleslot および tuplestore_gettupleslot 内の制御フローに依存しているため、変更時はこれらの関数における BufFile 操作との整合性に注意すること。
