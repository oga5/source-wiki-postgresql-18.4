gbt_macad8_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr8.c:113

目的

GiST インデックス内の macaddr8 型のインデックスエントリから、元の値をフェッチ（取得）します。

シグネチャまたは形状

Datum gbt_macad8_fetch(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体へのポインタを受け取ります。
  • **出力**: Datum 型として、フェッチされた値を返します。

前提条件と不変条件

  • PG_GETARG_POINTER(0) により、有効な GISTENTRY 構造体が取得できる必要があります。
  • 内部的に tinfo（gbtree_ninfo 型の静的変数）が初期化されている必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINTER(0) を呼び出し、引数から GISTENTRY 構造体を取得します。 2. 取得した entry と、モジュール内で定義された静的変数 tinfo を引数として gbt_num_fetch 関数を呼び出します。 3. gbt_num_fetch の戻り値を PG_RETURN_POINTER マクロで返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス管理機構から呼び出されます（直接的なコード上の呼び出し箇所はソース内に明示されていませんが、PG_FUNCTION_INFO_V1 マクロにより登録されています）。
  • **呼び出し先**: gbt_num_fetch（btree_utils_num.c で定義された汎用関数）。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • tinfo: macaddr8 型の比較ロジックや型情報を保持する gbtree_ninfo 構造体。

失敗モードと境界条件

  • entry が NULL の場合や、tinfo が正しく設定されていない場合は、gbt_num_fetch の内部挙動に従いエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で提供される汎用的なフェッチ処理のラッパーです。
  • macaddr8 型固有の複雑な処理は含まれておらず、主に tinfo を介した型情報の橋渡しを行っています。
