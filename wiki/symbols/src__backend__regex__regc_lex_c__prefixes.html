prefixes

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_lex.c:98

目的

正規表現の先頭に付与される特殊なフラグ（*** 形式や (?...) 形式の埋め込みオプション）を解析し、コンパイルフラグ（v->cflags）を動的に更新する。

シグネチャまたは形状

static void prefixes(struct vars *v)

入力と出力

  • **入力**: struct vars *v（正規表現のコンパイル状態を保持する構造体）。v->now が指す文字列の先頭部分を読み取る。
  • **出力**: なし（副作用として v->cflags および v->now を更新する）。

前提条件と不変条件

  • v は有効な struct vars 構造体を指している必要がある。
  • v->cflags に REG_QUOTE が設定されている場合、本関数は何も行わず即座に終了する。

アルゴリズムまたは振る舞い

1. ***** 形式の解析**:

  • ***?: REG_BADPAT エラーを発生させる。
  • ***=: REG_QUOTE を有効にし、他の高度なフラグを無効化する。
  • ***:: REG_ADVANCED を有効にする。

    • 2. **埋め込みオプション（(?...)）の解析**:

  • REG_ADVANCED が有効な場合のみ実行される。
  • (? で始まり、その後にアルファベットが続く場合に解析を開始する。
  • 内部の各文字（b, c, e, i, m, n, p, q, s, t, w, x）に応じて v->cflags をビット演算で更新する。
  • 閉じ括弧 ) が見つからない場合は REG_BADOPT エラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: lexstart（src/backend/regex/regc_lex.c:72）から呼び出される。
  • **呼び出し先**: ERR（エラー報告マクロ）、NOTE（通知マクロ）、HAVE/NEXT 系マクロ（入力確認用）、ATEOS（終端判定）を使用する。

関連データ構造

  • struct vars: 正規表現コンパイルの全状態を保持する主要構造体。cflags（コンパイルオプション）と now（現在の入力ポインタ）が本関数で直接操作される。

失敗モードと境界条件

  • **エラー発生**: 不正なオプション文字や構文（***? や閉じ括弧の欠落など）を検出した場合、ERR() マクロを介してエラー状態をセットし、処理を中断する。
  • **境界条件**: 入力文字列が短すぎて *** や (? を構成できない場合、何も変更せずに終了する。

保守担当者向けメモ

  • 本関数は regcomp.c にインクルードされるため、v は暗黙的に共有される。
  • v->now を進める際は、解析した文字数分だけ正確にインクリメントする必要がある。
  • 新しい正規表現オプションを追加する場合は、switch 文のケースを追加し、対応する v->cflags のビット操作を定義すること。
