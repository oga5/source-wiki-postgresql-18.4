tarPaddingBytesRequired

定義

  • 種別: static inline function
  • 定義箇所: src/include/pgtar.h:83

目的

tar アーカイブ内のファイルデータ末尾から、次の TAR_BLOCK_SIZE（512バイト）境界に到達するために必要なパディングバイト数を算出します。

シグネチャまたは形状

static inline size_t tarPaddingBytesRequired(size_t len)

入力と出力

  • **入力**: len (size_t) - 現在のファイルデータのサイズ。
  • **出力**: size_t - 境界調整のために追加が必要なパディングバイト数。

前提条件と不変条件

  • c.h で定義されている TYPEALIGN マクロを利用するため、本関数を使用する環境では c.h がインクルードされている必要があります。
  • TAR_BLOCK_SIZE は 512 バイトとして定義されています。

アルゴリズムまたは振る舞い

1. TYPEALIGN(TAR_BLOCK_SIZE, len) を使用して、len を TAR_BLOCK_SIZE の倍数に切り上げた値を計算します。 2. 切り上げた値から元の len を減算し、不足しているバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/bin/pg_basebackup/walmethods.c
  • src/bin/pg_dump/pg_backup_tar.c
  • src/fe_utils/astreamer_tar.c
  • **呼び出し先**:
  • TYPEALIGN マクロ（c.h 由来）

関連データ構造

  • TAR_BLOCK_SIZE: パディング計算の基準となるブロックサイズ（512バイト）。

失敗モードと境界条件

  • len が 0 の場合、パディングは 0 を返します。
  • len が TAR_BLOCK_SIZE の倍数である場合、パディングは 0 を返します。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、ヘッダーファイル内で完結しています。
  • TYPEALIGN の挙動に依存しているため、将来的に TAR_BLOCK_SIZE の定義を変更する場合は、アライメントの整合性に注意してください。
