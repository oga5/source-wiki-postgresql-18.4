typenameType

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_type.c:263

目的

TypeName 構造体からシステムカタログ上の型情報（Type / HeapTuple）を検索し、その型が完全に定義済みであることを検証して返します。

シグネチャまたは形状

Type typenameType(ParseState *pstate, const TypeName *typeName, int32 *typmod_p)

入力と出力

  • **入力**:
  • pstate: 構文解析状態（エラー報告用）。
  • typeName: 解決対象の型名構造体。
  • typmod_p: 型修飾子（typmod）を格納するポインタ。不要な場合は NULL を指定可能。
  • **出力**:
  • 成功時: pg_type システムカタログの HeapTuple（Type 型）。
  • 失敗時: エラーをスローし、制御を戻さない。

前提条件と不変条件

  • 呼び出し元は、返された Type（HeapTuple）に対して必ず ReleaseSysCache を実行しなければなりません。

アルゴリズムまたは振る舞い

1. LookupTypeName を呼び出し、型名から Type を検索します。 2. 検索結果が NULL の場合、ERRCODE_UNDEFINED_OBJECT でエラーを報告します。 3. 取得したタプルの typisdefined フラグを確認し、false（シェル型）であれば ERRCODE_UNDEFINED_OBJECT でエラーを報告します。 4. 検証を通過した有効な Type を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • LookupTypeName
  • TypeNameToString
  • parser_errposition
  • ereport
  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/commands/typecmds.c
  • src/backend/parser/parse_type.c（typenameTypeId および typenameTypeIdAndMod の内部実装として利用）

関連データ構造

  • TypeName: 構文解析中の型指定を保持する構造体。
  • Form_pg_type: pg_type システムカタログの行構造。
  • ParseState: 構文解析のコンテキスト。

失敗モードと境界条件

  • **型未定義**: 指定された名前の型がシステムカタログに存在しない場合、エラーとなります。
  • **シェル型**: 型名は存在するが、typisdefined が偽である（型定義が完了していない）場合、エラーとなります。
  • **メモリ管理**: 呼び出し側で ReleaseSysCache を怠るとキャッシュリークが発生します。

保守担当者向けメモ

  • 本関数は型解決の標準的なエントリポイントです。型情報の完全性を保証する必要がある箇所では、LookupTypeName ではなく本関数を使用してください。
  • エラーメッセージには TypeNameToString を使用して、ユーザーが理解可能な形式で型名を表示します。
