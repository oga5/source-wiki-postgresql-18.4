check_default_table_access_method

定義

  • 種別: function
  • 定義箇所: src/backend/access/table/tableamapi.c:101

目的

GUC（Grand Unified Configuration）設定 default_table_access_method の値がシステム内で有効なテーブルアクセス手法（TAM）であるかを検証するフック関数です。

シグネチャまたは形状

bool check_default_table_access_method(char **newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定されるTAM名の文字列ポインタ。
  • extra: 未使用。
  • source: 設定のソース（GucSource 型）。
  • **出力**:
  • 成功時は true を返し、検証失敗時は false を返します。

前提条件と不変条件

  • newval が空文字列の場合、検証は失敗します。
  • newval の長さが NAMEDATALEN 以上の場合、検証は失敗します。
  • トランザクション内かつデータベース接続済みである場合のみ、システムカタログを検索してTAMの存在を確認します。それ以外の場合は値をそのまま受け入れます。

アルゴリズムまたは振る舞い

1. **基本バリデーション**: 文字列が空でないこと、および長さが NAMEDATALEN 未満であることを確認します。 2. **存在確認**: トランザクション状態（IsTransactionState()）かつデータベース接続済み（MyDatabaseId != InvalidOid）であれば、get_table_am_oid を呼び出して指定されたTAMのOIDを取得します。 3. **エラーハンドリング**:

  • TAMが存在しない場合、source が PGC_S_TEST であれば NOTICE を発行して true を返します。
  • それ以外の場合は GUC_check_errdetail でエラー詳細を設定し、false を返します。

    • 4. **正常終了**: 検証を通過した場合は true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc_tables.c: GUC設定のフックとして登録されています。
  • **呼び出し先**:
  • IsTransactionState()
  • get_table_am_oid()
  • GUC_check_errdetail()
  • ereport()

関連データ構造

  • GucSource: 設定の由来を示す列挙型。
  • NAMEDATALEN: 名前識別子の最大長を定義する定数。

失敗モードと境界条件

  • **空文字列**: default_table_access_method に空文字が指定された場合、エラーとなります。
  • **名前の長さ**: NAMEDATALEN 以上の文字列が指定された場合、エラーとなります。
  • **存在しないTAM**: システムカタログに存在しないTAM名が指定された場合、PGC_S_TEST 以外ではエラーとなります。
  • **カタログアクセス不可**: トランザクション外またはデータベース未接続時は、存在確認を行わずに true を返します。

保守担当者向けメモ

  • 本関数はGUCのフックとして動作するため、エラー報告には ereport ではなく GUC_check_errdetail を使用して設定変更を拒否する設計になっています。
  • PGC_S_TEST 時の挙動は、設定ファイルのテスト読み込み時に過度なエラーを防ぐためのものです。
