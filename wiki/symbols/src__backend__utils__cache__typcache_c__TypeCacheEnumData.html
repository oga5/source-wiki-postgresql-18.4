TypeCacheEnumData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/typcache.c:153

目的

列挙型（Enum）の値をキャッシュし、システムカタログへのアクセスを最小限に抑えつつ、列挙値の順序比較や検索を高速化するために使用されます。

シグネチャまたは形状

typedef struct TypeCacheEnumData
{
    Oid         bitmap_base;    /* ビットマップセットのビット0に対応するOID */
    Bitmapset  *sorted_values;  /* 順序が確定しているOIDのセット */
    int         num_values;     /* 列挙型の総数 */
    EnumItem    enum_values[FLEXIBLE_ARRAY_MEMBER]; /* 可変長配列による列挙値の保持 */
} TypeCacheEnumData;

入力と出力

  • **入力**: palloc を用いて動的にメモリ確保される際、offsetof(TypeCacheEnumData, enum_values) を基準としたサイズ計算が行われます。
  • **出力**: find_enumitem などの関数を通じて、特定の列挙値（EnumItem）を検索・取得するために利用されます。

前提条件と不変条件

  • enum_values は可変長配列（FLEXIBLE_ARRAY_MEMBER）として定義されており、構造体自体のサイズは確保時に計算される必要があります。
  • sorted_values は、列挙値の順序関係が既知である場合にのみ有効な情報を保持します。

アルゴリズムまたは振る舞い

  • 列挙型のキャッシュ生成時に、カタログから取得した値をこの構造体に格納します。
  • find_enumitem 関数は、この構造体内の enum_values を走査して、指定された Oid に一致する項目を特定します。
  • enum_known_sorted 関数は、この構造体を参照して、指定された値が順序付けされたセットに含まれるかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: typcache.c 内の列挙型キャッシュ管理ロジック（enum_known_sorted、find_enumitem 等）。
  • **呼び出し先**: 構造体内部で EnumItem 型を使用しています。
  • **注記**: 外部モジュールからは不透明な構造体として扱われます（typcache.h にて宣言）。

関連データ構造

  • EnumItem: 列挙型の個々の要素を保持する構造体。
  • Bitmapset: sorted_values で使用されるビットマップデータ構造。

失敗モードと境界条件

  • メモリ確保時、FLEXIBLE_ARRAY_MEMBER のサイズ計算を誤るとバッファオーバーフローや不正なメモリ参照が発生する可能性があります。
  • num_values が実際の enum_values の要素数と一致しない場合、検索処理で境界外アクセスが発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は typcache.c 内でのみ定義・利用されるプライベートな構造体です。
  • 構造体のサイズが可変であるため、インスタンス化の際は必ず offsetof を用いた動的確保を行ってください。
