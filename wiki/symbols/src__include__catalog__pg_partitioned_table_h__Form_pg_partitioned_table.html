Form_pg_partitioned_table

定義

  • **種別**: typedef
  • **定義箇所**: src/include/catalog/pg_partitioned_table.h:65

目的

システムカタログ pg_partitioned_table の各行（タプル）をプログラムから参照するためのポインタ型を提供します。

シグネチャまたは形状

typedef FormData_pg_partitioned_table *Form_pg_partitioned_table;

FormData_pg_partitioned_table 構造体へのポインタとして定義されています。

入力と出力

  • **入力**: GETSTRUCT(tuple) マクロ等を通じて取得された、pg_partitioned_table カタログの物理的なタプルデータ。
  • **出力**: パーティション戦略、キー列、式などのメタデータを含む構造体へのアクセス手段。

前提条件と不変条件

  • 本型は FormData_pg_partitioned_table のメモリレイアウトに依存します。
  • システムキャッシュ（PARTRELID）やカタログスキャンを通じて取得された有効なタプルに対してキャストして使用する必要があります。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、実行時の振る舞いは持ちません。実運用では、タプルから抽出されたデータをこの型にキャストし、partstrat（戦略）、partattrs（属性番号）、partexprs（式）といったフィールドを読み取るために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/catalog/partition.c
  • src/backend/utils/adt/ruleutils.c
  • src/backend/utils/cache/partcache.c
  • **呼び出し先**:
  • FormData_pg_partitioned_table（構造体定義）

関連データ構造

  • FormData_pg_partitioned_table: カタログの物理的な行構造。
  • PARTRELID: パーティション情報を保持するシステムキャッシュ。

失敗モードと境界条件

  • **無効なポインタ**: GETSTRUCT が NULL を返すような不正なタプルに対してキャストを行うと、後続のフィールドアクセスでセグメンテーション違反が発生します。
  • **アライメント**: CATALOG_VARLEN フィールドが含まれる場合、ポインタ操作時に適切なアライメントが考慮されている必要があります。

保守担当者向けメモ

  • 本型は pg_partitioned_table のスキーマ変更（列の追加・削除）に伴い、FormData_pg_partitioned_table と共に更新する必要があります。
  • partattrs 配列内の値が 0 の場合、対応するキーは属性ではなく式（partexprs）であることを示すため、ロジック実装時にはこの不変条件に注意してください。
