sepgsql_proc_drop

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/proc.c:154

目的

関数（プロシージャ）の削除操作において、SELinuxポリシーに基づいた権限チェックを実行します。

シグネチャまたは形状

void sepgsql_proc_drop(Oid functionId)

入力と出力

  • **入力**: functionId (削除対象の関数のOID)
  • **出力**: なし（権限チェックに失敗した場合は例外が発生し、処理が中断されます）

前提条件と不変条件

  • 呼び出し元は、対象の関数が削除される直前のコンテキストで本関数を呼び出す必要があります。
  • getObjectIdentity を使用して監査用のオブジェクト識別子を生成するため、システムカタログへのアクセスが可能です。

アルゴリズムまたは振る舞い

1. **スキーマ権限の確認**: 対象関数が属する名前空間（スキーマ）に対し、db_schema:{remove_name} 権限を sepgsql_avc_check_perms で検証します。 2. **関数権限の確認**: 対象関数自体に対し、db_procedure:{drop} 権限を sepgsql_avc_check_perms で検証します。 3. **監査情報の生成**: 各チェックにおいて getObjectIdentity を呼び出し、権限拒否時に詳細な監査ログを出力できるようにします。 4. **メモリ解放**: 生成した監査用文字列 audit_name を pfree で解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:183
  • **呼び出し先**:
  • get_func_namespace
  • getObjectIdentity
  • sepgsql_avc_check_perms
  • pfree

関連データ構造

  • ObjectAddress: 権限チェック対象のオブジェクト（スキーマおよび関数）を特定するために使用。
  • NamespaceRelationId, ProcedureRelationId: システムカタログの識別子。

失敗モードと境界条件

  • 権限チェックが拒否された場合、sepgsql_avc_check_perms 内でエラーが発生し、トランザクションがアボートされます。
  • get_func_namespace が無効なOIDに対して呼び出された場合の挙動は、PostgreSQLのコアAPIの仕様に依存します。

保守担当者向けメモ

  • 本関数は pg_proc カタログに対するフックとして機能します。
  • 監査ログの正確性を保つため、getObjectIdentity の戻り値は必ず pfree で解放してください。
