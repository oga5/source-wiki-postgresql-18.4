pg_hmac

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:124

目的

SQL インターフェース経由で HMAC（Hash-based Message Authentication Code）計算を実行し、結果を bytea 型として返却する。

シグネチャまたは形状

Datum pg_hmac(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_BYTEA_PP(0): メッセージ本体（bytea）
  • PG_GETARG_BYTEA_PP(1): 秘密鍵（bytea）
  • PG_GETARG_TEXT_PP(2): アルゴリズム名（text）
  • **出力**:
  • PG_RETURN_BYTEA_P(res): 計算された HMAC 値（bytea）

前提条件と不変条件

  • find_provider が指定されたアルゴリズム名に対応するプロバイダーを見つけられない場合、エラーが発生する。
  • 入力引数は PostgreSQL のメモリ管理ルールに従い、必要に応じて PG_FREE_IF_COPY で解放される。

アルゴリズムまたは振る舞い

1. find_provider を呼び出し、指定されたアルゴリズム名で PX_HMAC コンテキストを取得する。 2. px_hmac_result_size で結果のサイズを特定し、メモリを確保する。 3. px_hmac_init で鍵をセットし、px_hmac_update でメッセージを処理する。 4. px_hmac_finish で計算を完了し、結果をバッファに書き込む。 5. px_hmac_free でコンテキストを解放する。 6. 入力引数のコピーが存在する場合はメモリを解放し、結果を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1(pg_hmac) により登録）。
  • **呼び出し先**: find_provider, px_hmac_result_size, px_hmac_init, px_hmac_update, px_hmac_finish, px_hmac_free（いずれも px レイヤーの関数）。

関連データ構造

  • PX_HMAC: HMAC 計算のための抽象化コンテキスト。
  • bytea, text: PostgreSQL のデータ型。

失敗モードと境界条件

  • アルゴリズム名が不正な場合、find_provider がエラーをスローする。
  • メモリ確保（palloc）失敗時は PostgreSQL の標準的なエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数は px レイヤーのラッパーであり、実際の暗号化ロジックは px レイヤー（px.c 等）に依存している。
  • PG_FREE_IF_COPY の使用により、関数呼び出し時にコピーされた引数メモリのリークを防いでいる。
