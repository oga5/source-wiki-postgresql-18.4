DomainIOData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/domains.c:50

目的

ドメイン型の入出力処理および制約検証に必要な情報を保持し、カタログアクセスを最小化するためのキャッシュ構造体です。

シグネチャまたは形状

typedef struct DomainIOData
{
    Oid            domain_type;
    Oid            typiofunc;
    Oid            typioparam;
    int32          typtypmod;
    FmgrInfo       proc;
    DomainConstraintRef constraint_ref;
    ExprContext   *econtext;
    MemoryContext  mcxt;
} DomainIOData;

入力と出力

本構造体は直接的な関数ではありませんが、domain_state_setup によって初期化され、domain_in、domain_recv、domain_check_input 等の関数において、基底型のI/O処理および制約評価のコンテキストとして利用されます。

前提条件と不変条件

  • FmgrInfo proc は基底型の入力関数を呼び出すために使用されます。
  • ExprContext *econtext は CHECK 制約の評価に使用され、使用後は ReScanExprContext によるクリーンアップが必要です。
  • 本構造体は FmgrInfo の fn_extra に保持されることで、同一セッション内での再利用が可能です。

アルゴリズムまたは振る舞い

1. **初期化**: domain_state_setup が TypeCache から情報を取得し、本構造体に格納します。 2. **I/O実行**: 基底型の入力関数を proc を介して実行し、生の Datum を生成します。 3. **検証**: constraint_ref を通じて制約リストを走査し、econtext 上で CHECK 制約を評価します。 4. **メモリ管理**: mcxt に保持されたメモリコンテキスト上で動作し、検証失敗時にはエラーを報告します。

呼び出し元と呼び出し先

  • **呼び出し元**: domain_in、domain_recv、domain_check_input が fn_extra から本構造体を取得して利用します。
  • **呼び出し先**: ExecCheck（式評価用）、MemoryContextAlloc（メモリ確保用）などが利用されます。
  • 補足: src/backend/utils/adt/jsonfuncs.c:196 にも同名の構造体定義が存在しますが、本ページは domains.c の定義を対象としています。

関連データ構造

  • DomainConstraintRef: 制約情報のキャッシュリスト。
  • ExprContext: CHECK 制約評価用の実行コンテキスト。
  • FmgrInfo: 基底型の関数呼び出し情報。

失敗モードと境界条件

  • 制約違反時にはエラーが発生します。
  • econtext のクリーンアップを怠るとメモリリークの原因となります。
  • fn_extra が初期化されていない場合、または不正な型情報が渡された場合の挙動は、呼び出し側の実装に依存します。

保守担当者向けメモ

  • DomainIOData は FmgrInfo の fn_extra に格納されるため、ライフサイクルは関数呼び出しの生存期間と一致します。
  • domain_check_safe を使用することで、例外を発生させずに検証結果をブール値で取得可能です。
