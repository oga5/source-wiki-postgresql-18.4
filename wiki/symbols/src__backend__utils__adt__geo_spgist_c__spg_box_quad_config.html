spg_box_quad_config

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/geo_spgist.c:400

目的

SP-GiSTインデックスにおける、矩形（BOX）データ型のインデックス設定（構成情報）を定義する。

シグネチャまたは形状

Datum spg_box_quad_config(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SP-GiSTの構成出力構造体 spgConfigOut へのポインタを含む）。
  • **出力**: void（引数として渡された cfg 構造体の各フィールドを更新することで設定を反映する）。

前提条件と不変条件

  • PG_GETARG_POINTER(1) を介して spgConfigOut 構造体が正しく取得できること。

アルゴリズムまたは振る舞い

1. cfg->prefixType を BOXOID に設定し、プレフィックスとして矩形型を使用することを指定する。 2. cfg->labelType を VOIDOID に設定し、ノードラベルが不要であることを示す。 3. cfg->canReturnData を true に設定し、インデックスからデータが取得可能であることを示す。 4. cfg->longValuesOK を false に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLのSP-GiSTインデックスマネージャ（インデックス構築・初期化時）。
  • 呼び出し先: 提供されたソースコード内では明示的な呼び出し先は確認できない。

関連データ構造

  • spgConfigOut: SP-GiSTインデックスの構成を保持する構造体。
  • BOXOID: 矩形型を識別するOID。
  • VOIDOID: ラベルなしを示すOID。

失敗モードと境界条件

  • 特筆すべき失敗モードや境界条件はソースコード内に定義されていない。

保守担当者向けメモ

  • 本関数はSP-GiSTの必須インターフェースの一部であり、box_ops インデックス演算クラスの構成を決定する。変更時はSP-GiSTのインデックス構造への影響を考慮すること。
