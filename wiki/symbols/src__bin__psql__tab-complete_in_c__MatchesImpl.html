MatchesImpl

定義

  • 種別: function
  • 定義箇所: src/bin/psql/tab-complete.in.c:1785

目的

入力された単語リストが、指定されたパターン（可変長引数）と一致するかを判定し、タブ補完の文脈マッチングを行う。

シグネチャまたは形状

bool MatchesImpl(bool case_sensitive, int previous_words_count, char **previous_words, int narg, ...);

入力と出力

  • **入力**:
  • case_sensitive: 大文字小文字を区別するかどうかのフラグ。
  • previous_words_count: previous_words 配列の要素数。
  • previous_words: 比較対象となる入力単語の配列。
  • narg: 可変長引数で渡されるパターンの数。
  • ...: 比較対象のパターン文字列（最大64個まで）。
  • **出力**:
  • bool: パターンと一致した場合は true、それ以外は false。

前提条件と不変条件

  • narg は 64 以下である必要がある（Assert により検証）。
  • previous_words_count が narg - 1 未満の場合、パターンマッチングは不可能と判断され直ちに false を返す。

アルゴリズムまたは振る舞い

1. 可変長引数からパターン文字列を argarray に格納する。 2. 入力単語数がパターン数に対して不足していないか確認する。 3. MatchesArray 関数を呼び出し、実際の比較処理を委譲する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/tab-complete.in.c 内の補完ロジック（MatchesImpl(false, ...) および MatchesImpl(true, ...) として呼び出されている）。
  • **呼び出し先**: MatchesArray

関連データ構造

  • argarray: パターンを一時保持するための固定長配列（const char *[64]）。

失敗モードと境界条件

  • narg が 64 を超える場合、Assert によりプログラムが停止する可能性がある。
  • 入力単語数がパターン数に対して不足している場合は、比較を行わず false を返す。

保守担当者向けメモ

  • 本関数は MatchesArray へのラッパーとして機能している。
  • gen_tabcomplete.pl による自動生成コードの一部として利用されることを想定している。
