InstrStartNode

定義

  • 種別: function
  • 定義箇所: src/backend/executor/instrument.c:67

目的

クエリ実行計画ノードの計測を開始し、実行開始時点の経過時間およびリソース使用量（バッファ、WAL）のスナップショットを記録します。

シグネチャまたは形状

void InstrStartNode(Instrumentation *instr)

入力と出力

  • **入力**: Instrumentation *instr（計測対象の統計構造体へのポインタ）
  • **出力**: なし（副作用として instr 構造体内の開始時刻、バッファ使用量、WAL使用量の各フィールドを更新）

前提条件と不変条件

  • instr は有効な Instrumentation 構造体を指している必要があります。
  • instr->need_timer が真の場合、タイマーの初期化が成功する必要があります。

アルゴリズムまたは振る舞い

1. instr->need_timer が真の場合、INSTR_TIME_SET_CURRENT_LAZY を用いて現在の時刻を instr->starttime に保存します。既にタイマーが設定されている場合はエラーを報告します。 2. instr->need_bufusage が真の場合、グローバル変数 pgBufferUsage の現在の値を instr->bufusage_start にコピーします。 3. instr->need_walusage が真の場合、グローバル変数 pgWalUsage の現在の値を instr->walusage_start にコピーします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/trigger.c
  • src/backend/executor/execAsync.c
  • src/backend/executor/execMain.c
  • src/backend/executor/execProcnode.c
  • **呼び出し先**:
  • elog（エラー発生時）
  • マクロ INSTR_TIME_SET_CURRENT_LAZY

関連データ構造

  • Instrumentation: 計測データを保持する構造体。
  • BufferUsage: バッファアクセス統計を保持するグローバル構造体 pgBufferUsage の型。
  • WalUsage: WAL生成統計を保持するグローバル構造体 pgWalUsage の型。

失敗モードと境界条件

  • INSTR_TIME_SET_CURRENT_LAZY が失敗した場合、または連続して二度呼び出された場合に ERROR を発生させます。

保守担当者向けメモ

  • 本関数は InstrStopNode と対で使用されます。ノード実行の開始と終了の差分を計算することで、特定のノードが消費したリソースを算出する仕組みです。
  • pgBufferUsage および pgWalUsage はプロセス全体で累積される値であるため、本関数で開始時の値を保存しておくことが正確な差分計算に不可欠です。
