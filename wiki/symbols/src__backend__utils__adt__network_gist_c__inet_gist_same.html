inet_gist_same

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_gist.c:796

目的

2つの GistInetKey がGiSTインデックスの観点で同一であるかを判定します。

シグネチャまたは形状

Datum inet_gist_same(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • 第1引数: GistInetKey へのポインタ（Datum として渡される）
  • 第2引数: GistInetKey へのポインタ（Datum として渡される）
  • 第3引数: 結果を格納する bool 型へのポインタ
  • **出力**:
  • 第3引数で指定されたメモリ領域に、2つのキーが同一であれば true、そうでなければ false を書き込みます。

前提条件と不変条件

  • 入力される2つの GistInetKey は有効なポインタである必要があります。
  • GistInetKey は varlena 形式の構造体です。

アルゴリズムまたは振る舞い

以下のすべての条件が真である場合にのみ、同一とみなして true を返します。 1. アドレスファミリー（gk_ip_family）が一致する。 2. 最小ネットマスク長（gk_ip_minbits）が一致する。 3. 共通プレフィックス長（gk_ip_commonbits）が一致する。 4. アドレス本体（gk_ip_addr）の内容が、そのサイズ（gk_ip_addrsize）分だけ完全に一致する。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスの管理フレームワークから呼び出されます（具体的な呼び出し元関数の詳細は提供されていません）。
  • **呼び出し先**:
  • DatumGetInetKeyP
  • gk_ip_family, gk_ip_minbits, gk_ip_commonbits, gk_ip_addr, gk_ip_addrsize（マクロ）
  • memcmp

関連データ構造

  • GistInetKey: ネットワーク型に対するGiSTインデックスのキー構造体。

失敗モードと境界条件

  • 入力ポインタがNULLの場合、セグメンテーション違反が発生する可能性があります。
  • アドレスファミリーが異なる場合、他のフィールドの比較を待たずに false と判定されます。

保守担当者向けメモ

  • 本関数はGiSTの same メソッドとして実装されています。インデックスの重複排除やノードの同一性判定に使用されます。
  • GistInetKey の内部構造に変更があった場合、比較ロジックも追従する必要があります。
