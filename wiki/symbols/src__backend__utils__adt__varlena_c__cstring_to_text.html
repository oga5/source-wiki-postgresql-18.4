cstring_to_text

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/varlena.c:192

目的

C言語のヌル終端文字列（const char *）を、PostgreSQLの内部可変長文字列型（text *）に変換します。

シグネチャまたは形状

text *cstring_to_text(const char *s)

入力と出力

  • **入力**: s (ヌル終端文字列へのポインタ)
  • **出力**: 変換された text 型データへのポインタ。メモリは現在のメモリコンテキストに割り当てられます。

前提条件と不変条件

  • 入力文字列 s はヌル終端である必要があります。
  • 内部で strlen(s) を呼び出すため、s が有効なメモリ領域を指していることが前提です。

アルゴリズムまたは振る舞い

1. 入力文字列 s の長さを strlen(s) で算出します。 2. 算出された長さを用いて cstring_to_text_with_len を呼び出し、text 型の構造体を作成して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: cstring_to_text_with_len (直接呼び出し)
  • **呼び出し元**: contrib/dblink/dblink.c 内の複数の関数（dblink_connect, dblink_exec 等の戻り値生成やステータス設定に使用）。

関連データ構造

  • text: PostgreSQLの可変長文字列型。

失敗モードと境界条件

  • 入力 s が NULL の場合、strlen の呼び出しによりセグメンテーション違反が発生します。
  • メモリ不足時には、PostgreSQLのメモリ管理機構によりエラーがスローされます。

保守担当者向けメモ

  • 本関数は cstring_to_text_with_len のラッパーです。文字列の長さが既に判明している場合は、cstring_to_text_with_len を直接使用することで strlen のオーバーヘッドを回避できます。
