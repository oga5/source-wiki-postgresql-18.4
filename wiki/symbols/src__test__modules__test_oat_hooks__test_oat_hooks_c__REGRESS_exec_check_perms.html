REGRESS_exec_check_perms

定義

  • **種別**: function
  • **定義箇所**: src/test/modules/test_oat_hooks/test_oat_hooks.c:346

目的

DML実行時のパーミッションチェックに介入し、GUC設定に基づいたアクセス制御のテストおよび監査ログの記録を行う。

シグネチャまたは形状

static bool REGRESS_exec_check_perms(List *rangeTabls, List *rteperminfos, bool do_abort)

入力と出力

  • **入力**:
  • rangeTabls: 範囲テーブルのリスト。
  • rteperminfos: RTE（Range Table Entry）権限情報のリスト。
  • do_abort: 権限チェック失敗時にエラーを発生させるかを示すフラグ。
  • **出力**:
  • bool: 権限チェックが許可された場合は true、拒否された場合は false を返す。

前提条件と不変条件

  • モジュール初期化時に ExecutorCheckPerms_hook に本関数が登録されていること。
  • next_exec_check_perms_hook が存在する場合、チェーンを維持するために必ず呼び出されること。

アルゴリズムまたは振る舞い

1. audit_attempt を呼び出し、実行試行を記録する。 2. REGRESS_deny_exec_perms（GUC）が有効かつ非スーパーユーザーである場合、権限を拒否する。 3. do_abort が true でかつ権限が拒否された場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーを発生させる。 4. next_exec_check_perms_hook が設定されている場合、それを呼び出し、その結果を allow に反映させる。 5. 最終的な allow の結果に基づき、audit_success または audit_failure を呼び出して監査ログを記録する。 6. 判定結果を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLコアの ExecutorCheckPerms フックポイント（ExecutorCheckPerms_hook 経由）。
  • **呼び出し先**:
  • audit_attempt, audit_success, audit_failure（監査ログ出力）。
  • next_exec_check_perms_hook（フックチェーンの次段）。
  • ereport（エラー発生時）。

関連データ構造

  • List: PostgreSQLのリスト構造。
  • REGRESS_deny_exec_perms: 権限拒否を制御するGUC変数。
  • next_exec_check_perms_hook: 以前に登録されていたフック関数を保持するポインタ。

失敗モードと境界条件

  • do_abort が true の場合、権限チェック失敗時にプロセスが中断される。
  • next_exec_check_perms_hook が false を返した場合、自身の許可状態に関わらず最終的な戻り値は false となる。

保守担当者向けメモ

  • 本関数はテスト用モジュールの一部であり、フックチェーンを壊さないよう next_exec_check_perms_hook の呼び出しが必須である。
  • 監査ログの出力は audit_* 関数群に委譲されているため、ログ形式の変更はそちらを参照すること。
