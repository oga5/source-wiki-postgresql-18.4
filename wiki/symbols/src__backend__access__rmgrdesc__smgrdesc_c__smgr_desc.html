smgr_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/smgrdesc.c:20

目的

ストレージマネージャ（smgr）に関連するWALレコードの内容を人間が読める形式に変換し、指定されたバッファへ書き出す。

シグネチャまたは形状

void smgr_desc(StringInfo buf, XLogReaderState *record)

入力と出力

  • **入力**:
  • buf: 情報を書き込むための StringInfo バッファ。
  • record: 解析対象のWALレコードを保持する XLogReaderState 構造体。
  • **出力**: なし（buf を通じて文字列が追記される）。

前提条件と不変条件

  • record は有効なWALレコードを指している必要がある。
  • XLogRecGetInfo(record) で取得される情報は、XLR_INFO_MASK を除外した状態で XLOG_SMGR_CREATE または XLOG_SMGR_TRUNCATE のいずれかと一致する必要がある。

アルゴリズムまたは振る舞い

1. XLogRecGetData を用いてレコードの生データを取得する。 2. XLogRecGetInfo とビットマスク演算により、レコードのサブタイプ（info）を特定する。 3. info が XLOG_SMGR_CREATE の場合:

  • データを xl_smgr_create 構造体にキャストする。
  • relpathperm を使用してリレーションのパス情報を取得し、buf に追記する。

    • 4. info が XLOG_SMGR_TRUNCATE の場合:

  • データを xl_smgr_truncate 構造体にキャストする。
  • relpathperm で取得したパス情報、ブロック数、およびフラグ情報を整形し、buf に追記する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h 内の PG_RMGR マクロ定義を通じて、WAL解析システムから呼び出される。
  • **呼び出し先**:
  • XLogRecGetData, XLogRecGetInfo (WAL解析用API)
  • appendStringInfo, appendStringInfoString (文字列操作)
  • relpathperm (リレーションパス生成)

関連データ構造

  • StringInfo: 文字列バッファ。
  • XLogReaderState: WALレコードの状態管理構造体。
  • xl_smgr_create: 作成操作のWALデータ構造。
  • xl_smgr_truncate: 切り詰め操作のWALデータ構造。

失敗モードと境界条件

  • info が定義されたサブタイプ以外の場合、関数は何も追記せずに終了する。
  • relpathperm が返すパス情報が空の場合や、無効なレコードデータが渡された場合の挙動については、本ソースコード内では明示的なエラーハンドリングは行われていない。

保守担当者向けメモ

  • 本関数はWALの表示専用であり、実際のストレージ操作ロジックは含まない。
  • XLR_INFO_MASK を用いたビット演算は、WALレコードのサブタイプを分離するための標準的な手法である。
  • relpathperm の使用は、ストレージ層の物理的な配置に依存している点に注意すること。
