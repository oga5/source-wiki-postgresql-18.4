brin_mask

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_xlog.c:341

目的

BRINインデックスページの検証（WALリプレイ後のページ内容比較など）において、非決定的なデータや無視すべき領域をマスクし、ページ内容の比較を可能にする。

シグネチャまたは形状

void brin_mask(char *pagedata, BlockNumber blkno)

入力と出力

  • pagedata: マスク対象となるページデータのポインタ。
  • blkno: 対象ページのブロック番号（関数内では未使用）。
  • 出力: なし（pagedata が指すメモリ領域が直接変更される）。

前提条件と不変条件

  • pagedata は有効なBRINインデックスページを指している必要がある。
  • メタページの場合、pd_lower が SizeOfPageHeaderData より大きい場合にのみ未使用領域のマスクが行われる。

アルゴリズムまたは振る舞い

1. mask_page_lsn_and_checksum を呼び出し、LSNとチェックサムをマスクする。 2. mask_page_hint_bits を呼び出し、ページヒントビットをマスクする。 3. 通常のBRINページ、または適切に初期化されたメタページに対して mask_unused_space を実行し、未使用領域をクリアする。 4. ページフラグから BRIN_EVACUATE_PAGE ビットを強制的に除去する（このフラグはWALログに記録されないため）。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/access/rmgrlist.h で定義されたリソースマネージャのインターフェースとして登録されており、テストや検証フレームワークから呼び出される。
  • 呼び出し先: mask_page_lsn_and_checksum, mask_page_hint_bits, mask_unused_space（いずれも access/bufmask.h 由来）。

関連データ構造

  • PageHeader: ページヘッダー情報。
  • BRIN_IS_REGULAR_PAGE, BRIN_IS_META_PAGE: ページ種別判定マクロ。
  • BrinPageFlags: ページフラグ操作マクロ。

失敗モードと境界条件

  • pagedata が不正なページ構造を指している場合、キャストやマクロ実行時に未定義の動作を引き起こす可能性がある。
  • メタページにおいて pd_lower が異常値である場合、未使用領域のマスクがスキップされる。

保守担当者向けメモ

  • 本関数は主にテストおよび検証用である。ページレイアウトやフラグ定義に変更を加えた際は、本関数が適切に非決定的な領域をカバーしているか確認すること。
  • BRIN_EVACUATE_PAGE の除去は、リカバリ時に不要なフラグが残存することを防ぐための重要な処理である。
