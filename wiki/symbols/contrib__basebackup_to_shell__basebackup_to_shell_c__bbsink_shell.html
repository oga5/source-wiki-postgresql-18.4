bbsink_shell

定義

  • 種別: struct
  • 定義箇所: contrib/basebackup_to_shell/basebackup_to_shell.c:26

目的

PostgreSQLのベースバックアップデータを外部シェルコマンドへ転送するための状態管理構造体です。bbsink インターフェースを実装し、バックアップストリームと外部プロセス間のパイプ接続を保持します。

シグネチャまたは形状

typedef struct bbsink_shell
{
    bbsink      base;             /* 共通のシンク情報 */
    char       *target_detail;    /* ユーザー指定のターゲット詳細文字列 */
    char       *shell_command;    /* 使用するシェルコマンドのパターン */
    char       *current_command;  /* 現在実行中のコマンド文字列 */
    FILE       *pipe;             /* 実行中コマンドへのパイプ */
} bbsink_shell;

入力と出力

  • **入力**: shell_get_sink 等で割り当てられ、target_detail や shell_command が設定されます。
  • **出力**: pipe メンバを通じて外部プロセスへデータが供給されます。

前提条件と不変条件

  • base メンバにより bbsink フレームワークのインターフェース要件を満たす必要があります。
  • target_detail はセキュリティ上の理由から英数字のみに制限されます。
  • pipe は shell_run_command で開かれ、shell_finish_command で閉じられるライフサイクルを持ちます。

アルゴリズムまたは振る舞い

1. **初期化**: shell_get_sink によりメモリが割り当てられ、コマンド文字列の検証が行われます。 2. **コマンド実行**: begin_archive / begin_manifest イベント時に shell_run_command が呼ばれ、popen を介してパイプが開かれます。 3. **データ転送**: archive_contents / manifest_contents イベント時に shell_send_data が呼ばれ、バッファの内容がパイプへ書き込まれます。 4. **終了処理**: end_archive / end_manifest イベント時に shell_finish_command が呼ばれ、パイプが閉じられ終了ステータスが検証されます。

呼び出し元と呼び出し先

  • **呼び出し元**: shell_get_sink (初期化), shell_finish_command, shell_run_command, shell_send_data, および各 bbsink コールバック関数（bbsink_shell_begin_archive 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • bbsink: 基底となるバックアップシンク構造体。
  • bbsink_shell_ops: bbsink インターフェースを実装する関数ポインタテーブル。

失敗モードと境界条件

  • 外部コマンドが非ゼロの終了コードを返した場合、バックアップ処理全体がエラーとして扱われます。
  • target_detail にメタ文字が含まれる場合、バリデーションエラーが発生します。

保守担当者向けメモ

  • 本構造体は bbsink フレームワークの一部として機能します。
  • pipe の管理は storage/fd.h の関数（OpenPipeStream 等）に依存しています。
  • コマンド文字列内の %f や %d の置換ロジックは common/percentrepl.h を利用しています。
