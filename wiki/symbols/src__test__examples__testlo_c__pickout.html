pickout

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlo.c:77

目的

データベース内のラージオブジェクト（Large Object）の指定された範囲を読み取り、標準エラー出力へ表示する。

シグネチャまたは形状

static void pickout(PGconn *conn, Oid lobjId, int start, int len)

入力と出力

  • **入力**:
  • conn: PostgreSQL データベース接続ハンドル (PGconn *)。
  • lobjId: 対象ラージオブジェクトの OID (Oid)。
  • start: 読み取り開始位置のオフセット (int)。
  • len: 読み取るバイト数 (int)。
  • **出力**: なし（標準エラー出力への書き出しのみ）。

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • ラージオブジェクト操作はトランザクション内で行われる必要がある。
  • len バイト分のメモリ確保が成功すること。

アルゴリズムまたは振る舞い

1. lo_open を使用して、指定された lobjId を読み取り専用モード (INV_READ) で開く。 2. lo_lseek を使用し、start で指定された位置までシークする。 3. len + 1 バイトのバッファを確保する。 4. len バイトに達するまで lo_read をループ実行し、読み取ったデータを >>> というプレフィックスと共に標準エラー出力へ書き出す。 5. 読み取り終了後、バッファを解放し、lo_close でラージオブジェクトを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/examples/testlo.c:255
  • src/test/examples/testlo64.c:79（注: testlo64.c では引数の型が異なる可能性がある）
  • src/test/examples/testlo64.c:280
  • **呼び出し先**:
  • lo_open, lo_lseek, lo_read, lo_close (libpq ラージオブジェクト API)
  • malloc, free (標準 C ライブラリ)
  • fprintf (標準 C ライブラリ)

関連データ構造

  • PGconn: libpq のデータベース接続構造体。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • lo_open が失敗した場合（戻り値が負）、エラーメッセージを出力するが処理は継続される。
  • lo_read が 0 以下の値を返した場合、ループを中断する。
  • 読み取り中に len に達する前にデータが終了した場合、その時点までの内容が表示される。

保守担当者向けメモ

  • 本関数はエラー発生時に fprintf で通知を行うのみであり、呼び出し元へのエラー伝播は行わない。
  • testlo64.c でも同名の関数が使用されているが、オフセットの型が異なる可能性があるため、修正時は両方のファイルを確認すること。
