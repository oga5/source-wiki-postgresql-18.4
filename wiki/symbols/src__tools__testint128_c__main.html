目的

PostgreSQL の int128.h で定義された 128 ビット整数演算の実装を、コンパイラがネイティブサポートする int128 型と比較検証するためのテストプログラムのエントリポイント。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • argc, argv: 第1引数にテストの反復回数を指定可能（省略時は 10 億回）。
  • **出力**:
  • 成功時: 0 を返す。
  • 失敗時: 不一致を検出した時点でエラー情報を標準出力に書き出し、1 を返して終了する。

前提条件と不変条件

  • test128 共用体が、ネイティブな int128 型と構造体表現（hi/lo）の間でメモリレイアウトの互換性を持つこと。
  • 実行環境のエンディアン設定（WORDS_BIGENDIAN）が int128.h の実装と整合していること。

アルゴリズムまたは振る舞い

1. pg_prng_seed を用いて乱数生成器を固定シードで初期化する。 2. 指定された回数分、以下の検証を繰り返す：

  • 乱数から test128 構造体を構築する。
  • **加算検証**: 符号なしおよび符号付きの加算結果を、ネイティブ演算と int128_add_uint64/int128_add_int64 の結果で比較する。
  • **乗算検証**: int128_add_int64_mul_int64 の結果をネイティブな乗算結果と比較する。
  • **比較検証**: my_int128_compare と int128_compare の結果を比較する。

    • 3. 全ての反復が成功すれば 0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェルまたはテスト実行環境（ソースコード上の直接の呼び出し元は特定不可）。
  • **呼び出し先**:
  • pg_prng_seed, pg_prng_uint64
  • int128_add_uint64, int128_add_int64, int128_add_int64_mul_int64
  • my_int128_compare, int128_compare

関連データ構造

  • test128: ネイティブな int128 型と INT128 構造体を保持する共用体。

失敗モードと境界条件

  • 演算結果の不一致（ビット単位の比較）が発生した場合、即座にエラーメッセージを出力して終了する。
  • 比較検証において、上位ビット（hi）が同一であるケースなど、特定の数値パターンで不一致が発生する可能性がある。

保守担当者向けメモ

  • 本関数はスタンドアロンのテストツールであり、ライブラリ関数ではない。
  • 実行時間が非常に長いため、デバッグ時は引数で count を減らして実行することを推奨する。
  • find_callers で検出された amcheck 関連の参照は、本シンボル main とは無関係な別コンテキストのコードである。
