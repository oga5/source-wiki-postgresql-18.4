pg_wait_until_termination

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/signalfuncs.c:168

目的

指定されたPIDを持つバックエンドプロセスが終了するまで、指定されたタイムアウト時間まで待機します。

シグネチャまたは形状

static bool pg_wait_until_termination(int pid, int64 timeout)

入力と出力

  • **入力**:
  • pid: 監視対象のプロセスID。
  • timeout: 待機する最大ミリ秒数。
  • **出力**:
  • bool: プロセスが正常に終了した場合は true、タイムアウトした場合は false を返します。

前提条件と不変条件

  • 呼び出し元は、ターゲットプロセスに対してシグナルを送信済みであることを想定しています。
  • 待機中、CHECK_FOR_INTERRUPTS() が呼び出されるため、クエリキャンセル等の割り込みが発生した場合は処理が中断されます。

アルゴリズムまたは振る舞い

1. remainingtime を timeout で初期化し、100ミリ秒単位のループを開始します。 2. kill(pid, 0) を実行し、プロセスの生存を確認します。

  • errno が ESRCH の場合、プロセスは既に終了していると判断し true を返します。
  • その他のエラーが発生した場合は ereport(ERROR) で処理を中断します。

    • 3. CHECK_FOR_INTERRUPTS() を呼び出し、保留中の割り込みを処理します。 4. WaitLatch を使用して、指定された waittime（最大100ms）の間、プロセス終了またはタイムアウトまで待機します。 5. タイムアウトに達するまで上記を繰り返し、終了を確認できない場合は WARNING を発行して false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/signalfuncs.c 内の pg_terminate_backend 関数から呼び出されます。
  • **呼び出し先**:
  • kill() (システムコール)
  • CHECK_FOR_INTERRUPTS()
  • WaitLatch()
  • ResetLatch()

関連データ構造

  • MyLatch: 待機処理に使用される現在のプロセスのラッチ。

失敗モードと境界条件

  • **ESRCH**: プロセスが存在しない場合、正常終了とみなして true を返します。
  • **タイムアウト**: 指定時間内にプロセスが終了しない場合、WARNING を出力し false を返します。
  • **システムエラー**: kill が ESRCH 以外のエラーを返した場合、内部エラーとして ereport(ERROR) を発行します。

保守担当者向けメモ

  • 本関数は pg_terminate_backend の待機ロジックをカプセル化したものです。
  • WaitLatch の引数に WL_EXIT_ON_PM_DEATH が指定されており、Postmasterが終了した場合には待機が解除される設計になっています。
