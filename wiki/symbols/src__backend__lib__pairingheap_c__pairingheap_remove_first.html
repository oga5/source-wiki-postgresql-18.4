pairingheap_remove_first

定義

  • 種別: function
  • 定義箇所: src/backend/lib/pairingheap.c:144

目的

ペアリングヒープから最小（または最大）のルートノードを削除し、ヒープを再構築して返却する。

シグネチャまたは形状

pairingheap_node *pairingheap_remove_first(pairingheap *heap)

入力と出力

  • **入力**: heap (ペアリングヒープ構造体へのポインタ)
  • **出力**: 削除されたルートノード (pairingheap_node *)

前提条件と不変条件

  • ヒープが空であってはならない (Assert(!pairingheap_is_empty(heap)) により検証)。
  • 削除後、ヒープが空でない場合、新しいルートノードの prev_or_parent および next_sibling は NULL に設定される。

アルゴリズムまたは振る舞い

1. 現在のルートノードを result として保持する。 2. ルートノードの子ノード群 (result->first_child) を取得する。 3. merge_children を呼び出し、子ノード群をマージして新しいヒープを形成し、それを heap->ph_root に設定する。 4. 新しいルートが存在する場合、そのポインタを適切に初期化（NULL化）する。 5. 削除した元のルートノード result を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistget.c
  • src/backend/access/spgist/spgscan.c
  • src/backend/executor/nodeIndexscan.c
  • src/backend/lib/pairingheap.c (内部的な再帰的利用の可能性あり)
  • **呼び出し先**:
  • merge_children (子ノード群の再構築に使用)
  • pairingheap_is_empty (アサーション内)

関連データ構造

  • pairingheap: ヒープ全体を管理する構造体。
  • pairingheap_node: ヒープ内の各要素。

失敗モードと境界条件

  • **空のヒープ**: pairingheap_is_empty が真の場合、アサーションによりプログラムが停止する。
  • **子ノードなし**: ルートノードに子が存在しない場合、merge_children は NULL を返し、ヒープは空の状態となる。

保守担当者向けメモ

  • 本関数はルートノードのメモリ解放を行わない。ノードのライフサイクル管理は呼び出し側に依存する。
  • 償却計算量は O(log n) である。
  • merge_children で採用されている「2パス法」がヒープの再構築効率を左右する。
