set_descriptors

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/descriptor.c:46

目的

スレッドローカルストレージ（TLS）上の記述子リストの先頭ポインタを更新し、現在のスレッドに関連付けられた記述子管理状態を保持する。

シグネチャまたは形状

static void set_descriptors(struct descriptor *value)

入力と出力

  • 入力: value (struct descriptor *) - スレッドローカルに格納する記述子リストのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • descriptor_key が初期化済みであること。
  • 本関数は pthread_setspecific を使用するため、マルチスレッド環境下で呼び出される必要がある。

アルゴリズムまたは振る舞い

1. pthread_setspecific を呼び出し、グローバルな descriptor_key に対応する現在のスレッドの値を value で上書きする。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/interfaces/ecpg/ecpglib/descriptor.c:47
  • src/interfaces/ecpg/ecpglib/descriptor.c:777
  • src/interfaces/ecpg/ecpglib/descriptor.c:834
  • 呼び出し先:
  • pthread_setspecific (外部ライブラリ関数)

関連データ構造

  • struct descriptor: ECPGにおけるSQL記述子を保持する構造体。
  • descriptor_key: スレッドローカルストレージのキーとして使用される pthread_key_t 型の変数。

失敗モードと境界条件

  • pthread_setspecific が失敗した場合の挙動については、本ソースコード内では明示的なエラーハンドリングは行われていない。

保守担当者向けメモ

  • 本関数は static であり、descriptor.c 内でのみ使用される。
  • スレッドローカルな記述子リストの管理において、リストの先頭を差し替える重要な役割を持つため、呼び出し箇所でのポインタの整合性に注意すること。
