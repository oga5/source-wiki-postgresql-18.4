config_real

定義

  • 種別: struct
  • 定義箇所: src/include/utils/guc_tables.h:246

目的

PostgreSQLのGUC（Grand Unified Configuration）システムにおいて、浮動小数点数（double）型の設定変数を定義・管理するための構造体です。

シグネチャまたは形状

struct config_real
{
    struct config_generic gen;      /* 基底構造体 */
    double       *variable;         /* 現在値を保持する変数へのポインタ */
    double        boot_val;         /* 起動時のデフォルト値 */
    double        min;              /* 許容最小値 */
    double        max;              /* 許容最大値 */
    GucRealCheckHook check_hook;    /* 値の検証用フック */
    GucRealAssignHook assign_hook;  /* 値の適用用フック */
    GucShowHook show_hook;          /* 値の表示用フック */
    double        reset_val;        /* リセット時の値 */
    void         *reset_extra;      /* リセット時の追加情報 */
};

入力と出力

  • **入力**: GUCシステムによる設定値の読み込みや変更要求（SQLの SET コマンド等）。
  • **出力**: variable ポインタが指すメモリ領域への値の反映、および show_hook を介した現在の設定値の出力。

前提条件と不変条件

  • gen フィールド（config_generic）が構造体の先頭に配置されており、型キャストによる基底クラスとしての利用が前提となっています。
  • boot_val、min、max などの定数フィールドは、初期化時に正しく設定される必要があります。

アルゴリズムまたは振る舞い

  • GUCシステムは、本構造体を通じて浮動小数点設定の検証（check_hook）と適用（assign_hook）を行います。
  • reset_val および reset_extra は、トランザクションのロールバックや設定のリセット時に、変数を以前の状態へ復元するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc.c 内の各関数（call_real_check_hook 等）が、設定値の検証や更新処理において本構造体を操作します。
  • **呼び出し先**: check_hook、assign_hook、show_hook に登録された関数が、設定変更のライフサイクルに応じて呼び出されます。

関連データ構造

  • config_generic: 全てのGUC変数構造体の基底。
  • GucRealCheckHook, GucRealAssignHook, GucShowHook: 設定値の操作をカスタマイズするための関数ポインタ型。

失敗モードと境界条件

  • min および max を超える値を設定しようとした場合、check_hook による検証プロセスで拒否される可能性があります。
  • variable ポインタが正しく初期化されていない場合、設定値の更新時にメモリ破壊やセグメンテーション違反が発生するリスクがあります。

保守担当者向けメモ

  • 本構造体は config_generic を継承しているため、gconf 型のポインタから config_real へキャストして使用するコードが guc.c に多数存在します。修正時はキャストの整合性に注意してください。
  • reset_extra は、check_hook で生成された複雑な状態を assign_hook に引き継ぐためのブリッジとして機能します。
