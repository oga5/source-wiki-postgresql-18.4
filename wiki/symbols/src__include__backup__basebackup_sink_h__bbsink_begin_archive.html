bbsink_begin_archive

定義

  • 種別: function
  • 定義箇所: src/include/backup/basebackup_sink.h:190

目的

バックアップ処理において、新しいアーカイブの開始をシンク（bbsink）チェーンに通知する。

シグネチャまたは形状

static inline void bbsink_begin_archive(bbsink *sink, const char *archive_name)

入力と出力

  • sink: 操作対象の bbsink オブジェクトへのポインタ。
  • archive_name: 開始するアーカイブの名前を示す文字列。
  • 出力: なし（void）。

前提条件と不変条件

  • sink は NULL であってはならない（Assert による検証）。
  • sink->bbs_ops が初期化されており、begin_archive コールバックが実装されている必要がある。

アルゴリズムまたは振る舞い

1. sink が NULL でないことを確認する。 2. sink が保持する bbs_ops テーブル内の begin_archive 関数を呼び出し、archive_name を渡す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/backend/backup/basebackup_gzip.c
  • src/backend/backup/basebackup_lz4.c
  • src/backend/backup/basebackup_sink.c
  • src/backend/backup/basebackup_zstd.c
  • **呼び出し先**:
  • sink->bbs_ops->begin_archive（bbsink の具体的な実装におけるコールバック関数）。

関連データ構造

  • bbsink: バックアップデータの転送先となる抽象インターフェース構造体。
  • bbsink_ops: 各種バックアップイベントに対するコールバック関数を保持するテーブル。

失敗モードと境界条件

  • sink が NULL の場合、アサーション違反によりプロセスが停止する可能性がある。
  • sink->bbs_ops が適切に設定されていない場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数はインライン関数であり、シンクチェーンの各段階で bbs_next を介して再帰的または順次的に呼び出される設計となっている。
  • 新しいシンクタイプを追加する際は、bbsink_ops 構造体で begin_archive を適切に定義すること。
