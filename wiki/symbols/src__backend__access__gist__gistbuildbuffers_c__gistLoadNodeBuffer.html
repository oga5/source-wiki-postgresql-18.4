gistLoadNodeBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistbuildbuffers.c:220

目的

一時ファイルに退避されているノードバッファの内容をメモリ上に読み込み、アクティブな状態へ復元します。

シグネチャまたは形状

static void gistLoadNodeBuffer(GISTBuildBuffers *gfbb, GISTNodeBuffer *nodeBuffer)

入力と出力

  • **入力**:
  • gfbb: バッファリングビルド全体の管理構造体。
  • nodeBuffer: 読み込み対象のノードバッファ構造体。
  • **出力**: なし（副作用としてメモリ割り当てとファイル読み込みを実行）。

前提条件と不変条件

  • nodeBuffer->pageBuffer が NULL であること。
  • nodeBuffer->blocksCount > 0 であること（読み込むべきデータが存在すること）。
  • 読み込み完了後、nodeBuffer->pageBlocknum は InvalidBlockNumber に設定されます。

アルゴリズムまたは振る舞い

1. pageBuffer が未割り当てかつデータが存在する場合にのみ処理を実行します。 2. gistAllocateNewPageBuffer を呼び出し、メモリ領域を確保します。 3. ReadTempFileBlock を使用して、一時ファイルから該当するブロックをメモリへ読み込みます。 4. gistBuffersReleaseBlock を呼び出し、一時ファイル上の当該ブロックを解放済みとしてマークします。 5. gistAddLoadedBuffer を呼び出し、読み込み済みバッファのリストへ登録します。 6. nodeBuffer->pageBlocknum を InvalidBlockNumber に更新し、ディスク上の位置情報をクリアします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistbuildbuffers.c 内の他の関数（gistPushItupToNodeBuffer や gistPopItupFromNodeBuffer 等から呼び出されることが示唆されています）。
  • **呼び出し先**:
  • gistAllocateNewPageBuffer
  • ReadTempFileBlock
  • gistBuffersReleaseBlock
  • gistAddLoadedBuffer

関連データ構造

  • GISTBuildBuffers: ビルド全体の状態管理。
  • GISTNodeBuffer: 個別のノードバッファの状態（メモリ上のページやディスク上のブロック番号を保持）。

失敗モードと境界条件

  • メモリ不足時や一時ファイルの読み込み失敗時は、内部で呼び出される関数が elog(ERROR) を発生させ、処理を停止します。
  • nodeBuffer->pageBuffer が既に存在する場合や blocksCount が 0 の場合は、何も行わず正常終了します。

保守担当者向けメモ

  • 本関数は、GiST バッファリングビルドにおける「スワップイン」処理を担います。
  • gistBuffersReleaseBlock によるブロック解放は、一時ファイル領域の再利用を目的としています。
  • 呼び出し元で個別のエラーハンドリングは不要です。
