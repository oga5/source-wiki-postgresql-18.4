Oid

定義

  • 種別: typedef
  • 定義箇所: src/include/utils/inval.h:24

目的

PostgreSQLのシステムカタログやリレーションなどのデータベースオブジェクトを一意に識別するための識別子（Object Identifier）です。

シグネチャまたは形状

typedef unsigned int Oid;

*(注: src/include/utils/inval.h 内での直接的な型定義は typedef ですが、PostgreSQLの標準的な定義に基づき unsigned int として扱われます)*

入力と出力

  • 本シンボルは型定義であるため、関数としての入力・出力はありません。
  • データベース内のオブジェクト参照において、引数や戻り値として頻繁に使用されます。

前提条件と不変条件

  • データベース内の特定のオブジェクトに対して一意であることが期待されます。
  • システムカタログの主キーとして機能します。

アルゴリズムまたは振る舞い

  • オブジェクトの識別子として、キャッシュのルックアップやリレーションの特定に使用されます。
  • inval.h の文脈では、キャッシュ無効化イベントが発生した際に、どのオブジェクト（relid 等）が影響を受けたかを特定するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/amcheck モジュール（verify_common.c, verify_gin.c, verify_heapam.c, verify_nbtree.c）などで、リレーションやインデックスの識別子として参照されています。
  • **呼び出し先**: 本シンボルは型定義であるため、呼び出し先はありません。

関連データ構造

  • RelcacheCallbackFunction: Oid relid を引数に取るコールバック関数型定義で使用されます。
  • Relation: リレーションキャッシュ内でオブジェクトを特定するために Oid が保持されます。

失敗モードと境界条件

  • InvalidOid（通常 0）は、有効なオブジェクトを指さない識別子として予約されています。
  • 存在しない Oid を指定した場合、キャッシュ検索やカタログアクセス時にエラーまたは不整合が発生する可能性があります。

保守担当者向けメモ

  • Oid は PostgreSQL 全体で多用される基本型です。変更はシステム全体に影響を及ぼすため、慎重に行う必要があります。
  • src/include/utils/inval.h においては、主にキャッシュ無効化の対象を特定するための識別子として機能している点に留意してください。
