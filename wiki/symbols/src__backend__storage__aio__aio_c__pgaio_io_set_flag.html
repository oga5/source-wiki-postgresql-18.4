pgaio_io_set_flag

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio.c:329

目的

非同期I/Oハンドル（PgAioHandle）に対して、特定のフラグを設定する。

シグネチャまたは形状

void pgaio_io_set_flag(PgAioHandle *ioh, PgAioHandleFlags flag)

入力と出力

  • ioh: 操作対象の PgAioHandle 構造体へのポインタ。
  • flag: 設定する PgAioHandleFlags 型のビットフラグ。
  • 戻り値: なし。

前提条件と不変条件

  • ioh->state が PGAIO_HS_HANDED_OUT であること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. ioh の現在の状態が PGAIO_HS_HANDED_OUT であることを確認する。 2. ioh->flags に対して、ビット論理和（OR）演算を用いて指定された flag を追加する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/buffer/bufmgr.c:1942
  • src/backend/storage/smgr/md.c:1020
  • src/test/modules/test_aio/test_aio.c:403
  • 呼び出し先:
  • 外部関数呼び出しなし。

関連データ構造

  • PgAioHandle: I/O操作の状態を保持する構造体。
  • PgAioHandleFlags: フラグ定義に使用される列挙型またはビットマスク。

失敗モードと境界条件

  • ioh が PGAIO_HS_HANDED_OUT 以外の状態である場合、アサーション違反によりプロセスが停止する。

保守担当者向けメモ

  • 本関数はフラグの追加のみを行う。フラグの削除や状態遷移を伴う操作が必要な場合は、別の関数を使用すること。
