skipfile

定義

  • 種別: function
  • 定義箇所: src/bin/pg_checksums/pg_checksums.c:156

目的

指定されたファイル名が、チェックサム処理から除外すべきシステムファイル（pg_control 等）のリストに含まれているかを判定します。

シグネチャまたは形状

static bool skipfile(const char *fn)

入力と出力

  • **入力**: fn (処理対象のファイル名文字列)
  • **出力**: true（除外対象の場合）、false（それ以外の場合）

前提条件と不変条件

  • skip 配列が exclude_list_item 構造体のリストとして定義されており、末尾が name フィールド NULL の要素で終端されている必要があります。

アルゴリズムまたは振る舞い

1. skip 配列を順次走査します。 2. 各除外項目について、match_prefix フラグを確認します。

  • match_prefix が真の場合、ファイル名の先頭が除外名と一致するかを比較します。
  • match_prefix が偽の場合、ファイル名が除外名と完全に一致するかを比較します（比較長を調整）。

    • 3. 一致が見つかった時点で直ちに true を返し、リストの最後まで一致がなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: scan_directory 関数（src/bin/pg_checksums/pg_checksums.c:345）から呼び出され、ディレクトリ走査時に処理対象外のファイルをフィルタリングするために使用されます。
  • **呼び出し先**: 標準ライブラリ関数 strlen および strncmp を使用します。

関連データ構造

  • exclude_list_item: 除外対象のファイル名と、前方一致判定を行うかどうかのフラグを保持する構造体。

失敗モードと境界条件

  • skip 配列が正しく終端されていない場合、メモリ境界外アクセスが発生する可能性があります。
  • ファイル名が NULL の場合、strlen 等でセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 新しいシステムファイルを除外対象に追加する必要がある場合は、skip 配列を更新してください。
  • match_prefix の設定は、特定のファイルだけでなく、特定のプレフィックスを持つファイル群を一括除外する際に重要となります。
