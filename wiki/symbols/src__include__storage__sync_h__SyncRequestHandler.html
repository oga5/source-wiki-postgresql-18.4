SyncRequestHandler

定義

  • 種別: enum
  • 定義箇所: src/include/storage/sync.h:35

目的

PostgreSQLのストレージサブシステムにおいて、ファイル同期（fsync）や削除などの要求を処理するハンドラを識別するために使用されます。

シグネチャまたは形状

typedef enum SyncRequestHandler
{
	SYNC_HANDLER_MD = 0,
	SYNC_HANDLER_CLOG,
	SYNC_HANDLER_COMMIT_TS,
	SYNC_HANDLER_MULTIXACT_OFFSET,
	SYNC_HANDLER_MULTIXACT_MEMBER,
	SYNC_HANDLER_NONE,
} SyncRequestHandler;

入力と出力

  • **入力**: 本列挙型は、同期要求の登録時や処理時に、どのサブシステム（MD, CLOG等）が要求を処理すべきかを指定する識別子として機能します。
  • **出力**: 該当しません。

前提条件と不変条件

  • sync.c 内の関数ポインタテーブル（syncsw）のインデックスと、本列挙型の値は一致している必要があります。

アルゴリズムまたは振る舞い

  • チェックポイント処理中、ProcessSyncRequests はこの列挙値に基づいて適切なハンドラへ同期処理を委譲します。
  • SYNC_HANDLER_MD は物理ストレージ（md.c）の操作に対応し、他の値は各アクセスメソッド（CLOG, COMMIT_TS, MULTIXACT等）の同期処理に対応します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/slru.c および src/include/access/slru.h で参照されています。
  • **呼び出し先**: 本シンボル自体は列挙型であり、関数呼び出しは行いません。

関連データ構造

  • FileTag: 同期対象ファイルを識別する構造体。
  • syncsw: sync.c 内で定義される、本列挙型をキーとする関数ポインタテーブル。

失敗モードと境界条件

  • SYNC_HANDLER_NONE は、有効なハンドラが割り当てられていない状態を示します。
  • 未定義の列挙値が渡された場合の挙動については、ソースコードの文脈からは不明です。

保守担当者向けメモ

  • 新しいストレージ対象を追加する場合は、本列挙型に値を追加し、sync.c の syncsw テーブルを更新する必要があります。
  • int16 型のフィールド（src/include/storage/sync.h:52）に格納される際、領域節約のために本列挙値がキャストされて利用される点に注意してください。
