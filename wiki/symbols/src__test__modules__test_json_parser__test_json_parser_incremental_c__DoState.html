DoState

定義

  • 種別: struct
  • 定義箇所: src/test/modules/test_json_parser/test_json_parser_incremental.c:51

目的

インクリメンタルな JSON パース処理中における、現在のパース状態およびセマンティック処理（エコーバック等）のためのコンテキストを保持する。

シグネチャまたは形状

typedef struct DoState
{
	JsonLexContext *lex;
	bool		elem_is_first;
	StringInfo	buf;
} DoState;

入力と出力

  • **入力**: パース処理の初期化時に JsonLexContext が割り当てられ、StringInfo バッファが生成される。
  • **出力**: パースイベントのコールバック関数内で、現在の状態（elem_is_first 等）の更新や、buf への文字列追加が行われる。

前提条件と不変条件

  • sem.semstate として palloc によりメモリが割り当てられている必要がある。
  • lex フィールドは有効な JsonLexContext を指している必要がある。

アルゴリズムまたは振る舞い

  • パースイベント（オブジェクト開始、配列要素開始など）が発生するたびに、コールバック関数内で state 引数から DoState へキャストされ、状態管理に使用される。
  • elem_is_first は、コンマの挿入制御など、構造の再構築における区切り判定に利用される。
  • buf は、パース結果を再構成するための動的文字列バッファとして機能する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（初期化時）および各セマンティックコールバック関数（do_object_start, do_scalar 等）。
  • **呼び出し先**: makeStringInfo（初期化時）。

関連データ構造

  • JsonLexContext: パース対象の字句解析コンテキスト。
  • StringInfo: パース結果を蓄積するための動的文字列バッファ。
  • JsonSemAction: 本構造体を semstate として保持するコールバックテーブル。

失敗モードと境界条件

  • palloc によるメモリ確保失敗時、プログラムは異常終了する可能性がある。
  • buf の操作においてメモリ不足が発生した場合、StringInfo の内部実装に従いエラーが発生する。

保守担当者向けメモ

  • 本構造体はテストモジュール専用であり、パース結果の検証（エコーバック）を目的としている。
  • sem.semstate へのキャストを伴うため、型安全性を維持するために DoState へのポインタ変換が正しく行われているか注意すること。
