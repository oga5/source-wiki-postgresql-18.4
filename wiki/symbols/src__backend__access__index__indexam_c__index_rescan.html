index_rescan

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/indexam.c:355

目的

既存のインデックススキャンを再開（リセット）し、新しい検索キーや順序付け条件を適用します。

シグネチャまたは形状

void index_rescan(IndexScanDesc scan, ScanKey keys, int nkeys, ScanKey orderbys, int norderbys)

入力と出力

  • **入力**:
  • scan: 再開対象のインデックススキャン記述子。
  • keys: 新しい検索キーの配列。
  • nkeys: 検索キーの数。
  • orderbys: 新しい順序付け条件の配列。
  • norderbys: 順序付け条件の数。
  • **出力**: なし（副作用としてスキャン状態を更新）。

前提条件と不変条件

  • SCAN_CHECKS マクロにより、スキャン記述子が有効であることが検証されます。
  • CHECK_SCAN_PROCEDURE(amrescan) により、対象のインデックスアクセス手法が amrescan 関数を実装している必要があります。
  • nkeys および norderbys は、スキャン記述子作成時に定義された数と一致しなければなりません（Assert による検証）。

アルゴリズムまたは振る舞い

1. **リソース解放**: scan->xs_heapfetch が存在する場合、table_index_fetch_reset を呼び出してテーブルアクセスに関連するバッファピン等のリソースを解放します。 2. **状態リセット**: kill_prior_tuple および xs_heap_continue フラグを false に設定し、安全な初期状態に戻します。 3. **AM 固有処理の実行**: インデックスリレーションの amrescan 関数を呼び出し、各インデックス型（B-tree 等）固有の再スキャンロジックを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/index/genam.c
  • src/backend/executor/execIndexing.c
  • src/backend/executor/execReplication.c
  • src/backend/executor/nodeBitmapIndexscan.c
  • src/backend/executor/nodeIndexonlyscan.c
  • **呼び出し先**:
  • table_index_fetch_reset
  • scan->indexRelation->rd_indam->amrescan（各インデックス AM の実装関数）

関連データ構造

  • IndexScanDesc: インデックススキャンの状態を保持する構造体。
  • ScanKey: 検索条件を定義する構造体。

失敗モードと境界条件

  • インデックス AM が amrescan を実装していない場合、CHECK_SCAN_PROCEDURE によりエラーが発生します。
  • nkeys や norderbys がスキャン記述子の期待値と一致しない場合、デバッグビルドでは Assert により停止します。

保守担当者向けメモ

  • 本関数はスキャンを再利用するための重要なインターフェースです。リソースリークを防ぐため、AM 固有の amrescan 実装において、以前のスキャンで確保したリソースが適切に解放されることを確認してください。
