logicalrep_get_attrs_str

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/relation.c:226

目的

論理レプリケーションにおいて、特定の属性（列）セットを人間が読み取り可能なカンマ区切りの文字列形式に変換し、エラーメッセージやログ出力用に提供する。

シグネチャまたは形状

static char *logicalrep_get_attrs_str(LogicalRepRelation *remoterel, Bitmapset *atts)

入力と出力

  • **入力**:
  • remoterel: リモートリレーション情報を保持する LogicalRepRelation 構造体へのポインタ。属性名（attnames）の参照に使用。
  • atts: 変換対象の属性インデックスを示す Bitmapset。
  • **出力**:
  • 属性名をダブルクォーテーションで囲み、カンマで連結した文字列（char *）。

前提条件と不変条件

  • atts は空であってはならない（Assert(!bms_is_empty(atts)) により検証）。
  • remoterel->attnames が有効な文字列配列を保持していること。

アルゴリズムまたは振る舞い

1. StringInfoData を初期化する。 2. bms_next_member を使用して、atts に含まれるすべての属性インデックスを順次走査する。 3. 走査した各属性名に対し、"%s" の形式で attsbuf に追加する。 4. 2つ目以降の属性については、追加前に ", " を挿入する。 5. 構築された文字列バッファのデータ部分を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/relation.c 内の関数（logicalrep_report_missing_or_gen_attrs 等での利用が確認されている）。
  • **呼び出し先**: initStringInfo, bms_next_member, appendStringInfoString, appendStringInfo 等のユーティリティ関数。

関連データ構造

  • LogicalRepRelation: リモートテーブルのメタデータ（属性名等）を保持する構造体。
  • Bitmapset: 属性の選択状態を管理するビットマップ。
  • StringInfoData: 文字列構築用バッファ。

失敗モードと境界条件

  • atts が空の場合、アサーション違反によりプロセスが停止する可能性がある。
  • メモリ不足時には appendStringInfo 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は palloc されたメモリを返すため、呼び出し側で適切に解放する必要がある。
  • 属性名がダブルクォーテーションで囲まれるため、識別子としてそのまま利用可能な形式で出力される。
