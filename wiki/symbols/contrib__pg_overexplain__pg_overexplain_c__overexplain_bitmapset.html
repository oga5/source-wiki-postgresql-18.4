overexplain_bitmapset

定義

  • 種別: function
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:714

目的

Bitmapset 構造体の内容を文字列に変換し、EXPLAIN 出力の一部として ExplainState にプロパティとして追加する。

シグネチャまたは形状

static void overexplain_bitmapset(const char *qlabel, Bitmapset *bms, ExplainState *es)

入力と出力

  • qlabel (入力): 出力時のプロパティ名。
  • bms (入力): 表示対象の Bitmapset。
  • es (入力): EXPLAIN の状態を管理する ExplainState。
  • 出力: なし（ExplainPropertyText を介して es に書き込まれる）。

前提条件と不変条件

  • bms が空の場合、プロパティ値として "none" が出力される。
  • bms が空でない場合、各メンバーがスペース区切りの数値文字列として出力される。
  • buf.data の先頭には必ずスペースが含まれることが Assert で保証されている。

アルゴリズムまたは振る舞い

1. bms_is_empty で空か判定し、空であれば "none" を出力して終了する。 2. initStringInfo でバッファを初期化する。 3. bms_next_member を用いて Bitmapset 内の全メンバーを走査し、各数値を " %d" 形式でバッファに追記する。 4. ExplainPropertyText を呼び出し、バッファの先頭スペースを除いた文字列を qlabel に紐付けて出力する。 5. pfree でバッファメモリを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pg_overexplain/pg_overexplain.c 内の各ハンドラ関数（extParam, allParam, Scan RTIs, Append RTIs, Subplans Needing Rewind, Unprunable RTIs の出力時に使用）。
  • 呼び出し先:
  • bms_is_empty, bms_next_member (PostgreSQL ビットマップセット操作関数)
  • initStringInfo, appendStringInfo, pfree (PostgreSQL 文字列操作・メモリ管理関数)
  • ExplainPropertyText (PostgreSQL EXPLAIN 出力関数)

関連データ構造

  • Bitmapset: 整数集合を保持するビットマップ構造。
  • ExplainState: EXPLAIN コマンドの実行状態と出力先を保持する構造体。
  • StringInfoData: 動的文字列バッファ。

失敗モードと境界条件

  • bms が NULL の場合の挙動はソースコード上では明示されていない（bms_is_empty の実装に依存する）。
  • メモリ不足時には initStringInfo や appendStringInfo がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は ExplainPropertyText を使用しているため、出力フォーマット（TEXT, JSON 等）に応じた適切なエスケープ処理は EXPLAIN サブシステム側に委ねられている。
  • bms_next_member のループ終了条件は x < 0 である。
