defGetObjectId

定義

  • 種別: function
  • 定義箇所: src/backend/commands/define.c:205

目的

DefElem ノードから Oid（オブジェクト識別子）を抽出する。SQL コマンドのオプション引数として渡された数値を、内部的な Oid 型へ変換するために使用される。

シグネチャまたは形状

Oid defGetObjectId(DefElem *def)

入力と出力

  • **入力**: DefElem *def（解析対象の定義要素ノード）
  • **出力**: Oid 型の値
  • **副作用**: 変換に失敗した場合、ereport により ERRCODE_SYNTAX_ERROR を発生させ処理を中断する。

前提条件と不変条件

  • def および def->arg が NULL でないこと（def->arg が NULL の場合はエラーとなる）。
  • def->arg は T_Integer または T_Float のいずれかのノードタグを持つ必要がある。

アルゴリズムまたは振る舞い

1. def->arg が NULL の場合、エラーを報告する。 2. nodeTag(def->arg) を判定する。

  • T_Integer: intVal() マクロを用いて整数値を抽出し、Oid にキャストして返す。
  • T_Float: レクサーが大きな数値を Float として解釈した場合を考慮し、fval 文字列を oidin 関数（DirectFunctionCall1 経由）で Oid に変換して返す。

    • 3. 上記以外の型が渡された場合、エラーを報告する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dbcommands.c:849 での呼び出しが確認されている。
  • **呼び出し先**: intVal マクロ、DirectFunctionCall1、oidin 関数、castNode マクロ。

関連データ構造

  • DefElem: src/include/nodes/parsenodes.h で定義される、SQL コマンドのオプション引数を保持する構造体。
  • Oid: PostgreSQL のオブジェクト識別子型。

失敗モードと境界条件

  • def->arg が NULL の場合: ERRCODE_SYNTAX_ERROR を発生させる。
  • def->arg が T_Integer や T_Float 以外の型（例: T_String 等）である場合: ERRCODE_SYNTAX_ERROR を発生させる。
  • T_Float の値が oidin で変換できない形式である場合: oidin 内部のロジックによりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は define.c 内の他のアクセサ関数と同様、型変換の安全性を担保する役割を持つ。
  • T_Float の処理は、パーサーが数値を Integer として扱えないほど大きい場合に備えたフォールバックである。変更時は oidin の仕様に注意すること。
