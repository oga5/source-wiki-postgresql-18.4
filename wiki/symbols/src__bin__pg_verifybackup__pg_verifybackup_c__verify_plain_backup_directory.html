verify_plain_backup_directory

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/pg_verifybackup.c:615

目的

Plain形式のバックアップディレクトリを再帰的に走査し、各ファイルがマニフェストと整合しているかを検証する。

シグネチャまたは形状

static void verify_plain_backup_directory(verifier_context *context, char *relpath, char *fullpath, DIR *dir);

入力と出力

  • context: 検証状態を保持する verifier_context 構造体へのポインタ。
  • relpath: バックアップルートからの相対パス。
  • fullpath: ファイルシステム上の絶対パスまたは相対パス。
  • dir: 既に開かれているディレクトリポインタ（NULLの場合は内部で opendir を実行）。
  • 出力: なし（検証エラーは context を通じて記録される）。

前提条件と不変条件

  • fullpath は有効なディレクトリパスであること。
  • context は初期化済みであること。
  • . および .. ディレクトリは走査対象から除外される。

アルゴリズムまたは振る舞い

1. dir が NULL の場合、opendir でディレクトリを開く。失敗時はエラーを記録し、ignore_list に relpath を追加して終了する。 2. readdir を使用してディレクトリ内の全エントリをループ処理する。 3. 各エントリに対し、パスを構築し should_ignore_relpath で無視対象か判定する。 4. 無視対象でない場合、verify_plain_backup_file を呼び出し、個別のファイル検証を行う。 5. ループ終了後、closedir でディレクトリを閉じる。失敗時はエラーを記録する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_verifybackup/pg_verifybackup.c 内のメイン処理および再帰呼び出し箇所。
  • 呼び出し先:
  • report_backup_error
  • simple_string_list_append
  • should_ignore_relpath
  • verify_plain_backup_file

関連データ構造

  • verifier_context: 検証のコンテキスト、無視リスト、エラー状態を管理。
  • dirent: ディレクトリ内のエントリ情報。

失敗モードと境界条件

  • ディレクトリのオープン失敗: report_backup_error を呼び出し、当該パスを無視リストに追加して処理を中断する。
  • ディレクトリのクローズ失敗: report_backup_error を呼び出し、エラーを記録する。
  • 権限不足やパスの不整合: readdir や verify_plain_backup_file の実行結果に基づきエラーが報告される。

保守担当者向けメモ

  • 本関数は再帰的に呼び出される可能性がある設計となっている。
  • relpath が NULL の場合はルートディレクトリの走査とみなされる。
  • errno をクリアしてから readdir を呼び出すことで、エラー発生時の判定を確実に行っている。
