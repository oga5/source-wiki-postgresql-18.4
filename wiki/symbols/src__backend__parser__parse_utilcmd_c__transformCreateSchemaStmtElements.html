transformCreateSchemaStmtElements

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_utilcmd.c:4110

目的

CREATE SCHEMA 文に含まれる複数の要素（テーブル、シーケンス、ビュー、インデックス、トリガー、権限付与）を解析し、実行順序に従ってソートされたコマンドリストを生成する。

シグネチャまたは形状

List *transformCreateSchemaStmtElements(List *schemaElts, const char *schemaName)

入力と出力

  • **入力**:
  • schemaElts: スキーマ内で定義される要素（CreateSeqStmt, CreateStmt, ViewStmt, IndexStmt, CreateTrigStmt, GrantStmt）のリスト。
  • schemaName: 対象となるスキーマ名。
  • **出力**:
  • 変換およびソート済みのユーティリティコマンドの List。

前提条件と不変条件

  • 入力された各要素のスキーマ名が未設定の場合、schemaName で指定された値が設定される。
  • 処理順序は、シーケンス、テーブル、ビュー、インデックス、トリガー、権限付与の順に固定される。

アルゴリズムまたは振る舞い

1. CreateSchemaStmtContext を初期化し、各要素タイプ用のリストを空にする。 2. schemaElts を走査し、各ノードの型に応じて setSchemaName を呼び出してスキーマ名を適用し、対応するコンテキスト内のリストへ追加する。 3. 未対応のノード型が検出された場合は elog(ERROR) を発生させる。 4. 最後に、各リストをシーケンス、テーブル、ビュー、インデックス、トリガー、権限付与の順序で連結し、単一のリストとして返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/schemacmds.c:197
  • **呼び出し先**: setSchemaName（各要素のスキーマ名設定用）

関連データ構造

  • CreateSchemaStmtContext: 変換処理中に各要素のリストを保持するための内部構造体。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • schemaElts 内にサポートされていないノード型が含まれている場合、elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数はスキーマ内の要素を型ごとに分類して順序付ける役割に特化している。
  • CreateStmt や ViewStmt の内部的な依存関係解決が必要な場合は、本関数の該当箇所を拡張する必要がある。
