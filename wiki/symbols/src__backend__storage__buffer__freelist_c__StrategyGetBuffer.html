StrategyGetBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/freelist.c:195

目的

バッファマネージャが新しいバッファを必要とする際に、再利用可能なバッファを特定して取得する。

シグネチャまたは形状

BufferDesc *StrategyGetBuffer(BufferAccessStrategy strategy, uint32 *buf_state, bool *from_ring)

入力と出力

  • **入力**:
  • strategy: バッファアクセス戦略（BufferAccessStrategy）。NULLの場合は共有フリーリストまたはクロック掃引を使用。
  • buf_state: 取得したバッファの現在の状態を格納するポインタ。
  • from_ring: リングバッファから取得したかどうかを示すフラグを格納するポインタ。
  • **出力**:
  • 成功時: 再利用可能なバッファの記述子（BufferDesc *）。
  • 失敗時: エラー（elog(ERROR, ...)）によりプロセスが中断される。

前提条件と不変条件

  • strategy が提供されている場合、そのリングバッファからバッファの選択を試みる。
  • フリーリストの操作は buffer_strategy_lock によって保護される。
  • 取得されたバッファは、ピンされておらず、かつ usage_count が 0 である必要がある。

アルゴリズムまたは振る舞い

1. **リングバッファ確認**: strategy が指定されていれば GetBufferFromRing を呼び出し、利用可能なバッファがあれば即座に返す。 2. **バックグラウンドライター通知**: 必要に応じて StrategyControl->bgwprocno を確認し、バックグラウンドライターのラッチをセットする。 3. **フリーリスト探索**: 共有フリーリストに空きがあればロックを取得してバッファを取り出し、状態を確認する。使用可能であればリングバッファに追加して返す。 4. **クロック掃引**: フリーリストが空の場合、ClockSweepTick() を用いて全バッファを巡回する。ピンされていないバッファの usage_count を減らしながら、再利用可能なバッファを探索する。 5. **終了**: 全バッファを走査しても再利用可能なものが見つからない場合はエラーを投げる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/bufmgr.c（bufmgr.c:2366 等）
  • **呼び出し先**:
  • GetBufferFromRing
  • SetLatch
  • SpinLockAcquire / SpinLockRelease
  • GetBufferDescriptor
  • LockBufHdr / UnlockBufHdr
  • AddBufferToRing
  • ClockSweepTick

関連データ構造

  • BufferDesc: バッファ記述子。
  • BufferStrategyControl: 共有メモリ上の戦略管理構造体。
  • BufferAccessStrategy: プロセス固有のバッファリング戦略。

失敗モードと境界条件

  • **エラー**: 全バッファがピンされている場合、"no unpinned buffers available" というエラーでプロセスが終了する。
  • **競合**: フリーリストから取得したバッファが、取得直前に他プロセスによって使用された場合、再試行を行う。

保守担当者向けメモ

  • buffer_strategy_lock はフリーリスト操作時のみ取得され、クロック掃引のメインループではロックを避ける設計となっている。
  • bgwriter への通知は、StrategyControl のアトミックな読み込みと SetLatch を用いて行われる。
  • usage_count の減算は、クロック掃引中にピンされていないバッファに対して行われる。
