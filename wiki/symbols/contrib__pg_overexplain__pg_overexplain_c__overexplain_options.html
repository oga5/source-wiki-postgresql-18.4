overexplain_options

定義

  • 種別: struct
  • 定義箇所: contrib/pg_overexplain/pg_overexplain.c:29

目的

EXPLAIN コマンドの拡張オプション（debug および range_table）の状態を保持し、ExplainState にアタッチして管理するための構造体です。

シグネチャまたは形状

typedef struct
{
	bool		debug;
	bool		range_table;
} overexplain_options;

入力と出力

  • **入力**: EXPLAIN コマンド実行時にユーザーが指定したオプションに基づき、overexplain_ensure_options 関数を通じて初期化されます。
  • **出力**: ExplainState 内に格納され、各フック関数（overexplain_per_node_hook 等）が追加情報を出力する際の判断基準として参照されます。

前提条件と不変条件

  • ExplainState に関連付けられた拡張データとして管理されます。
  • palloc0 を使用してメモリ確保されるため、初期状態では各フラグは false です。

アルゴリズムまたは振る舞い

1. overexplain_ensure_options が ExplainState から本構造体を取得または新規作成します。 2. ユーザーが指定したオプションに応じて、debug または range_table フラグが true に設定されます。 3. 各フック関数は、この構造体のフラグをチェックし、有効な場合にのみ詳細な出力処理（overexplain_debug や overexplain_range_table）を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • overexplain_ensure_options (構造体の生成・取得)
  • overexplain_per_node_hook (状態の参照)
  • overexplain_per_plan_hook (状態の参照)
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • ExplainState: PostgreSQL の EXPLAIN 実行状態を保持する構造体。本構造体はこの内部にキャッシュされます。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させます。
  • ExplainState が適切に初期化されていない場合、本構造体の取得に失敗する可能性があります。

保守担当者向けメモ

  • ExplainState にデータをキャッシュする手法（GetExplainExtensionState / SetExplainExtensionState）の典型的な実装例です。
  • 新しいオプションを追加する場合は、この構造体にフィールドを追加し、overexplain_ensure_options での初期化ロジックを更新してください。
