fsync_fname

定義

  • 種別: function
  • 定義箇所: src/common/file_utils.c:399

目的

指定されたファイルまたはディレクトリを fsync し、永続化を保証する。

シグネチャまたは形状

int fsync_fname(const char *fname, bool isdir)

入力と出力

  • fname: 対象のファイルまたはディレクトリのパス（const char *）。
  • isdir: 対象がディレクトリである場合は true、ファイルである場合は false を指定。
  • 戻り値: 成功時は 0。失敗時は -1 を返し、エラーログを出力する（ただし、特定の無視可能なエラーを除く）。

前提条件と不変条件

  • isdir が false の場合、ファイルは読み書き可能（O_RDWR）として開かれる。
  • isdir が true の場合、ディレクトリは読み取り専用（O_RDONLY）として開かれる。
  • 呼び出し元は、本関数が失敗した際にプロセスを終了させるか、エラーを適切に処理する必要がある。

アルゴリズムまたは振る舞い

1. isdir に応じて適切なフラグ（O_RDWR または O_RDONLY）を設定し、open を実行する。 2. open 失敗時、EACCES またはディレクトリに対する EISDIR の場合はエラーを無視して 0 を返す。それ以外はエラーログを出力し -1 を返す。 3. fsync を実行する。 4. fsync 失敗時、ディレクトリに対して EBADF または EINVAL が発生した場合は、OSがディレクトリの同期をサポートしていないと判断し、エラーを無視する。 5. 上記以外の fsync エラーはログ出力し、exit(EXIT_FAILURE) でプロセスを終了する。 6. 最後に close を実行し、成功時は 0 を返す。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **権限不足**: EACCES は無視される。
  • **ディレクトリ操作の制限**: open 時の EISDIR や、fsync 時の EBADF/EINVAL は、OSの仕様としてディレクトリ同期がサポートされていない環境と見なし、正常系として扱う。
  • **致命的エラー**: fsync が上記以外の理由で失敗した場合、データ整合性を守るためプロセスを強制終了する。

保守担当者向けメモ

  • 本関数は FRONTEND 定義時にのみコンパイルされるコードパスに含まれる可能性がある。
  • O_RDWR でファイルを開くため、ユーザーIDに書き込み権限がないファイルに対しては open が失敗する可能性がある。
