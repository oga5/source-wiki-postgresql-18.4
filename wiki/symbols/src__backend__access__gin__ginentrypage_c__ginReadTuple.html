ginReadTuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginentrypage.c:161

目的

GINインデックスのリーフページに格納されたインデックスタプルから、ポスティングリスト（ItemPointerの配列）を抽出し、デコードしてメモリ上に展開する。

シグネチャまたは形状

ItemPointer ginReadTuple(GinState *ginstate, OffsetNumber attnum, IndexTuple itup, int *nitems)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの状態管理構造体。
  • attnum: 属性番号。
  • itup: 読み取り対象のインデックスタプル。
  • nitems: ポスティングリスト内のアイテム数を格納するための出力用ポインタ。
  • **出力**:
  • 復元された ItemPointer 配列の先頭アドレス。呼び出し元で pfree する必要がある。

前提条件と不変条件

  • itup は有効なGINインデックスタプルであること。
  • nitems は有効な整数へのポインタであること。
  • 圧縮されている場合、ヘッダーに記録されたアイテム数とデコード後のアイテム数が一致しなければならない。

アルゴリズムまたは振る舞い

1. タプルからポスティングリストのデータ領域（ptr）とアイテム数（nipd）を取得する。 2. タプルが圧縮されている場合（GinItupIsCompressed）:

  • nipd > 0 ならば ginPostingListDecode を呼び出し、圧縮データを展開する。
  • 展開後のアイテム数がヘッダーの nipd と一致するか検証し、不一致ならエラーを投げる。
  • nipd == 0 ならば空のメモリを割り当てる。

    • 3. 圧縮されていない場合:

  • nipd 分のメモリを確保し、memcpy でデータをコピーする。

    • 4. nitems に nipd を書き込み、復元した配列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginget.c
  • src/backend/access/gin/gininsert.c
  • **呼び出し先**:
  • ginPostingListDecode (圧縮時)
  • palloc (メモリ確保)
  • memcpy (非圧縮時)
  • elog (エラー発生時)

関連データ構造

  • IndexTuple: GINインデックスの物理的な格納形式。
  • ItemPointer: 抽出されるポスティングリストの各要素（TID）。
  • GinPostingList: 圧縮されたポスティングリストの構造体。

失敗モードと境界条件

  • **不整合**: 圧縮データのヘッダー情報と実際のデコード結果のアイテム数が一致しない場合、elog(ERROR) により処理が中断される。
  • **メモリ不足**: palloc が失敗した場合はPostgreSQLのメモリ管理機構によりエラーが発生する。

保守担当者向けメモ

  • 本関数は圧縮・非圧縮の両方の形式を透過的に扱う。圧縮形式の変更や新しい圧縮アルゴリズムの導入時は、ginPostingListDecode の呼び出し部分を確認すること。
  • 戻り値の ItemPointer 配列は呼び出し元でメモリ解放の責任を負うため、メモリリークに注意が必要。
