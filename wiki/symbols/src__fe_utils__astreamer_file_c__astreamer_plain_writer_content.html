astreamer_plain_writer_content

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_file.c:106

目的

astreamer フレームワークにおいて、ストリームデータを物理ファイルへ書き出すためのコンテンツ処理関数です。

シグネチャまたは形状

static void astreamer_plain_writer_content(
    astreamer *streamer,
    astreamer_member *member,
    const char *data,
    int len,
    astreamer_archive_context context
)

入力と出力

  • **入力**:
  • streamer: astreamer_plain_writer 型にキャスト可能なストリーマーコンテキスト。
  • data: 書き込み対象のデータバッファ。
  • len: 書き込むデータの長さ。
  • **出力**: なし（戻り値なし）。書き込み失敗時は pg_fatal によりプロセスを終了します。

前提条件と不変条件

  • streamer は astreamer_plain_writer 構造体として初期化されている必要があります。
  • mystreamer->file は書き込み可能なファイルポインタとして有効である必要があります。

アルゴリズムまたは振る舞い

1. streamer を astreamer_plain_writer 型へキャストします。 2. len が 0 の場合は何もせず終了します。 3. fwrite を使用して、data を mystreamer->file に書き込みます。 4. 書き込みに失敗した場合（戻り値が 1 でない場合）:

  • errno が 0 であれば ENOSPC（ディスク容量不足）をセットします。
  • pg_fatal を呼び出し、ファイルパスとともにエラーメッセージを出力して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_plain_writer 構造体の .content メンバとして登録されており、ストリーム処理の過程で呼び出されます。
  • **呼び出し先**: fwrite (標準ライブラリ)、pg_fatal (PostgreSQL ログ出力)。

関連データ構造

  • astreamer_plain_writer: ファイル書き込み用のコンテキスト構造体。
  • astreamer: 基底となるストリーマー構造体。

失敗モードと境界条件

  • **書き込み失敗**: ディスクフルや権限エラー等で fwrite が失敗した場合、pg_fatal によりプログラムが異常終了します。
  • **空データ**: len が 0 の場合は処理をスキップします。

保守担当者向けメモ

  • 本関数は astreamer_plain_writer の状態遷移におけるデータ書き込みフェーズを担当します。
  • ファイルのオープンやクローズは本関数ではなく、それぞれ new 関数や finalize 関数で行われるため、本関数内ではファイルハンドルが既に有効であることを前提としています。
