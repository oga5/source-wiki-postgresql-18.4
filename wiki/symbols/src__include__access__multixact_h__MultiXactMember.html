MultiXactMember

定義

  • 種別: struct
  • 定義箇所: src/include/access/multixact.h:56

目的

マルチトランザクション（MultiXact）において、特定のタプルに対してロックを保持する個々のトランザクションID（XID）と、そのロック状態をペアとして保持するために使用されます。

シグネチャまたは形状

typedef struct MultiXactMember
{
	TransactionId xid;
	MultiXactStatus status;
} MultiXactMember;

入力と出力

  • **入力**: 構造体メンバとして TransactionId（トランザクション識別子）および MultiXactStatus（ロックモード）を受け取ります。
  • **出力**: 構造体として、特定のマルチトランザクションを構成するメンバー情報の集合体を提供します。

前提条件と不変条件

  • 本構造体は、pg_multixact SLRU領域に格納されるメンバー情報の基本単位です。
  • 複数の MultiXactMember が配列として扱われることで、単一の MultiXactId に紐づく複数のトランザクションのロック状態を表現します。

アルゴリズムまたは振る舞い

  • heapam.c 等において、複数のトランザクションが同一タプルに対してロックを要求した際、これらを統合して管理するために本構造体の配列が動的に割り当てられ（palloc）、メンバー情報が格納されます。
  • MultiXactId を介して参照される際、GetMultiXactIdMembers 等の関数を通じて、この構造体の配列としてロック状態が復元されます。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照・利用）**:
  • src/backend/access/heap/heapam.c: 行ロックの競合解決やマルチトランザクションの生成・解析時に頻繁に使用されます。
  • contrib/pgrowlocks/pgrowlocks.c: 行ロック情報の可視化に使用されます。
  • src/backend/access/rmgrdesc/mxactdesc.c: WALレコードのデバッグ出力（out_member）に使用されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数呼び出しを直接行うものではありません。

関連データ構造

  • MultiXactId: 本構造体の集合を識別するためのID。
  • MultiXactStatus: 本構造体内でロックの種類（FOR SHARE, FOR UPDATE等）を定義する列挙型。
  • xl_multixact_create: マルチトランザクション作成時のWALレコード構造体。

失敗モードと境界条件

  • メモリ不足: palloc による配列割り当て時に失敗する可能性があります。
  • 整合性: pg_multixact SLRU領域の破損や切り詰め（Truncate）により、参照すべきメンバー情報が取得できない場合があります。

保守担当者向けメモ

  • 本構造体は、PostgreSQLの行ロック管理における「共有ロック」の核心部分です。
  • MultiXactStatus の変更や TransactionId の扱いに変更がある場合、本構造体のメモリレイアウトがWAL記録やSLRUのディスクフォーマットに直接影響するため注意が必要です。
