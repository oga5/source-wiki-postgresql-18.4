verify_directories

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/exec.c:262

目的

アップグレード処理を開始する前に、カレントディレクトリの権限および新旧両方のPostgreSQLクラスタのディレクトリ構造とバイナリの整合性を一括検証する。

シグネチャまたは形状

void verify_directories(void)

入力と出力

  • 入力: なし（グローバル変数 old_cluster および new_cluster を参照）
  • 出力: なし（検証失敗時は pg_fatal によりプロセスを終了）

前提条件と不変条件

  • カレントディレクトリに対して、読み取り・書き込み・実行の権限が必要である。
  • old_cluster および new_cluster が初期化済みであること。

アルゴリズムまたは振る舞い

1. **ディレクトリ権限確認**:

  • Windows環境以外では access(".", R_OK | W_OK | X_OK) を実行。
  • Windows環境では win32_check_directory_write_permissions() を実行。
  • 権限がない場合、pg_fatal でエラー終了する。

    • 2. **クラスタ検証**:

  • check_bin_dir(&old_cluster, false): 旧クラスタのバイナリを検証。
  • check_data_dir(&old_cluster): 旧クラスタのデータディレクトリを検証。
  • check_bin_dir(&new_cluster, true): 新クラスタのバイナリを検証。
  • check_data_dir(&new_cluster): 新クラスタのデータディレクトリを検証。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_upgrade/pg_upgrade.c (main関数内)
  • 呼び出し先:
  • access (標準ライブラリ)
  • win32_check_directory_write_permissions
  • pg_fatal
  • check_bin_dir
  • check_data_dir

関連データ構造

  • ClusterInfo (グローバル変数 old_cluster, new_cluster の型)

失敗モードと境界条件

  • カレントディレクトリに書き込み権限がない場合、即座に致命的エラーとして終了する。
  • check_bin_dir または check_data_dir が内部でエラーを検出した場合、同様に pg_fatal により終了する。

保守担当者向けメモ

  • Windows環境における権限チェックは win32_check_directory_write_permissions に抽象化されているため、プラットフォーム固有の挙動変更時は同関数を確認すること。
  • check_bin_dir の第2引数は、新クラスタか否か（バージョンチェックの厳密さ）を制御している。
