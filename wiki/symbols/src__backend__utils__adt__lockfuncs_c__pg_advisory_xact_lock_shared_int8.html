pg_advisory_xact_lock_shared_int8

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/adt/lockfuncs.c:675

目的

ユーザーが指定した64ビット整数値をキーとして、現在のトランザクションスコープで共有アドバイザリーロックを取得します。

シグネチャまたは形状

Datum pg_advisory_xact_lock_shared_int8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して int64 型のキーを受け取ります。
  • **出力**: void（成功時はロックが保持され、失敗時はエラーが発生します）。

前提条件と不変条件

  • ロックは現在のトランザクションに紐付きます。トランザクション終了時に自動的に解放されます。
  • データベース単位のロックとして管理されるため、MyDatabaseId がロックタグに含まれます。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT64(0) により、引数からロックキーを取得します。 2. SET_LOCKTAG_INT64 マクロを使用して、取得したキーを LOCKTAG 構造体に変換します。 3. LockAcquire を呼び出し、ShareLock モードでロックを要求します。

  • 第3引数（exclusive）は false です。
  • 第4引数（dontWait）は false であり、ロックが取得できるまで待機します。

    • 4. 処理完了後、PG_RETURN_VOID() を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されます（具体的な呼び出し元コードの詳細は提供されていません）。
  • **呼び出し先**:
  • LockAcquire（src/backend/storage/lmgr/lock.c 内のロックマネージャ機能）

関連データ構造

  • LOCKTAG: ロック対象を識別するための構造体。
  • ShareLock: ロックモードの定数。

失敗モードと境界条件

  • ロック取得時にデッドロックが検出された場合、LockAcquire によりエラーが発生し、トランザクションがアボートされる可能性があります。
  • データベースを跨いだロックは行えません。

保守担当者向けメモ

  • 本関数はトランザクションスコープのロックであるため、セッションスコープの pg_advisory_lock_shared_int8 とはライフサイクルが異なります。
  • LockAcquire の第4引数が false であるため、ロック競合時には呼び出し元が待機状態になる点に注意してください。
