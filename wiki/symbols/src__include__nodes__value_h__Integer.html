Integer

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/value.h:28

目的

SQLクエリの構文解析ツリー（Parse Tree）において、整数リテラル値を保持するためのノード構造体です。

シグネチャまたは形状

typedef struct Integer
{
	pg_node_attr(special_read_write)

	NodeTag		type;
	int			ival;
} Integer;

入力と出力

  • **入力**: 構文解析時に整数値が検出された際、makeInteger 関数（src/backend/nodes/value.c）を介して初期化されます。
  • **出力**: intVal() マクロを使用することで、ノード内に格納された int 型の値を抽出可能です。

前提条件と不変条件

  • NodeTag 型の type フィールドを含み、PostgreSQLのノードシステムと統合されています。
  • 構文解析ツリー内でリテラル値を表現するコンテナとして機能します。

アルゴリズムまたは振る舞い

1. **生成**: パーサーが字句解析で整数リテラルを検出すると、makeInteger を呼び出し、ヒープ上に Integer 構造体を割り当てます。 2. **保持**: 生成されたノードは List 構造などに格納され、解析フェーズを通じて受け渡されます。 3. **抽出**: 意味解析や計画作成の段階で、intVal() マクロを用いて ival フィールドから実際の整数値を取り出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/dbcommands.c: IsA(encodingEl->arg, Integer) による型チェックで使用。
  • contrib/postgres_fdw/postgres_fdw.c: 属性番号のリストや fetch_size の保持に使用。
  • src/backend/commands/aggregatecmds.c: 集約関数の引数カウントの保持に使用。
  • **呼び出し先**:
  • 本構造体はデータコンテナであり、特定の関数を呼び出すロジックは含みません。

関連データ構造

  • NodeTag: ノードの型識別子。
  • List: Integer ノードを格納するために使用されるリスト構造。
  • makeInteger: 本構造体を生成するコンストラクタ関数。
  • intVal: 本構造体から値を抽出するアクセスマクロ。

失敗モードと境界条件

  • int 型の範囲を超える数値については、本構造体では表現できません。
  • castNode を介さない不正なポインタキャストによる値の抽出は、型安全性を損なう可能性があります。

保守担当者向けメモ

  • Value 構造体から分離された個別のノード型の一つです。
  • pg_node_attr(special_read_write) 属性が付与されており、ノードシステムの自動生成ツールによる処理対象となっています。
