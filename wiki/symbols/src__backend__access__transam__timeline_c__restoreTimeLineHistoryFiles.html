restoreTimeLineHistoryFiles

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/timeline.c:49

目的

指定された範囲のタイムライン履歴ファイルをアーカイブから取得し、ローカル環境へ復元します。

シグネチャまたは形状

void restoreTimeLineHistoryFiles(TimeLineID begin, TimeLineID end)

入力と出力

  • **入力**:
  • begin (TimeLineID): 復元を開始するタイムラインID。
  • end (TimeLineID): 復元を終了するタイムラインID（この値自体は含まない）。
  • **出力**: なし。

前提条件と不変条件

  • begin から end までの範囲で、タイムラインID 1 は初期状態として扱われ、アーカイブからの復元対象外となります。
  • アーカイブからのファイル取得には RestoreArchivedFile を使用し、成功した場合は KeepFileRestoredFromArchive を呼び出してファイルを保持します。

アルゴリズムまたは振る舞い

1. begin から end - 1 までの各 TimeLineID についてループ処理を行います。 2. tli == 1 の場合はスキップします。 3. TLHistoryFileName を使用して、対象タイムラインの履歴ファイル名（.history）を生成します。 4. RestoreArchivedFile を呼び出し、アーカイブから履歴ファイルを検索・取得します。 5. 取得に成功した場合、KeepFileRestoredFromArchive を呼び出し、当該ファイルをリカバリに必要な領域へ保存します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlog.c:5697
  • src/backend/access/transam/xlogrecovery.c:4218
  • **呼び出し先**:
  • TLHistoryFileName
  • RestoreArchivedFile
  • KeepFileRestoredFromArchive

関連データ構造

  • TimeLineID: タイムラインを識別するための数値型。

失敗モードと境界条件

  • RestoreArchivedFile が失敗した場合、そのタイムラインの履歴ファイルは復元されません。
  • begin >= end の場合、ループは実行されません。

保守担当者向けメモ

  • 本関数はリカバリプロセスにおいて、必要なタイムライン履歴をアーカイブからローカルへ同期するために使用されます。
  • TLI 1 は履歴ファイルを持たない特別なIDであるため、明示的に除外されています。
