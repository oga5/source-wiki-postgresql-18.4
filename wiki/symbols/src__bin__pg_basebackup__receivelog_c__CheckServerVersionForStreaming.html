CheckServerVersionForStreaming

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/receivelog.c:373

目的

接続先のPostgreSQLサーバーのバージョンが、ストリーミングレプリケーション機能を利用可能な範囲内にあるかを確認する。

シグネチャまたは形状

bool CheckServerVersionForStreaming(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（確立済みのデータベース接続）
  • **出力**: 互換性がある場合は true、互換性がない場合は false を返す。

前提条件と不変条件

  • conn は有効な接続オブジェクトである必要がある。
  • ストリーミングレプリケーションのプロトコル互換性に基づき、サーバーバージョン 9.3 以上をサポートする。
  • クライアント自身のバージョン（PG_VERSION_NUM）を超えるサーバーバージョンはサポート対象外とする。

アルゴリズムまたは振る舞い

1. サーバーのメジャーバージョン（PQserverVersion(conn) / 100）を取得する。 2. 最小サポートバージョン（9.3）および最大サポートバージョン（クライアントのメジャーバージョン）と比較する。 3. 範囲外の場合は pg_log_error を使用してエラーメッセージを出力し、false を返す。 4. 範囲内の場合は true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • src/bin/pg_basebackup/receivelog.c
  • **呼び出し先**:
  • PQserverVersion (libpq)
  • PQparameterStatus (libpq)
  • pg_log_error (common/logging)

関連データ構造

  • PGconn (libpqの接続ハンドル)

失敗モードと境界条件

  • **バージョン不足**: サーバーが 9.3 未満の場合、エラーを出力して失敗する。
  • **バージョン超過**: サーバーがクライアントのメジャーバージョンより新しい場合、安全のためエラーを出力して失敗する。
  • **情報取得失敗**: PQparameterStatus がサーバーバージョンを取得できない場合、エラーメッセージには 'unknown' と表示される。

保守担当者向けメモ

  • 本関数はストリーミング開始前のガード節として機能する。
  • 新しいPostgreSQLメジャーバージョンをリリースする際は、クライアント側の PG_VERSION_NUM が更新されることで、本関数の最大バージョン判定も自動的に追従する設計となっている。
