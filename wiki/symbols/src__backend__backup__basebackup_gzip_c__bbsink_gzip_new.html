bbsink_gzip_new

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_gzip.c:61

目的

gzip圧縮を行うための bbsink（バックアップシンク）インスタンスを生成し、初期化するコンストラクタ関数です。

シグネチャまたは形状

bbsink *bbsink_gzip_new(bbsink *next, pg_compress_specification *compress);

入力と出力

  • **入力**:
  • next: 圧縮後のデータを受け取る後続の bbsink オブジェクト。
  • compress: 圧縮レベルを含む pg_compress_specification 構造体。
  • **出力**:
  • 成功時: 初期化された bbsink オブジェクトへのポインタ。
  • 失敗時: HAVE_LIBZ が未定義の場合、エラーを報告し終了します。

前提条件と不変条件

  • next は NULL であってはなりません。
  • compress->level は 1 から 9 の範囲、または Z_DEFAULT_COMPRESSION である必要があります。
  • 本関数は HAVE_LIBZ が定義されている環境でのみ動作します。

アルゴリズムまたは振る舞い

1. HAVE_LIBZ の定義を確認し、未定義であればエラーを発生させます。 2. bbsink_gzip 構造体用にメモリを確保します。 3. bbsink_gzip_ops を用いて bbsink の操作テーブルをセットアップします。 4. 後続のシンク (next) と指定された圧縮レベルを構造体に格納します。 5. 初期化済みの bbsink ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:1044
  • **呼び出し先**: palloc0（PostgreSQLメモリ管理関数）

関連データ構造

  • bbsink: バックアップデータのパイプライン処理を行う基底構造体。
  • bbsink_gzip: 圧縮状態（z_stream 等）を保持する派生構造体。
  • pg_compress_specification: 圧縮設定を保持する構造体。

失敗モードと境界条件

  • HAVE_LIBZ が定義されていないビルドでは、呼び出し時に ERRCODE_FEATURE_NOT_SUPPORTED エラーが発生します。
  • next が NULL の場合、Assert により異常終了します。
  • 圧縮レベルが指定範囲外の場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は zlib を使用した圧縮パイプラインの入り口です。
  • bbsink_gzip_ops が正しく定義されていることを前提としています。
  • HAVE_LIBZ がない環境でのビルドを考慮し、条件付きコンパイルが適用されています。
