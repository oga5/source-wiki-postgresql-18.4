ExecInitExpr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExpr.c:142

目的

解析・計画された式ツリー（Expr）を、実行時に効率的に評価可能なフラットな命令配列（ExprEvalSteps）を持つ ExprState へと変換（コンパイル）します。

シグネチャまたは形状

ExprState *ExecInitExpr(Expr *node, PlanState *parent)

入力と出力

  • **入力**:
  • node: 変換対象の式ツリー（Expr）。NULL の場合は NULL を返す。
  • parent: 式に関連付けられる親の PlanState。
  • **出力**:
  • 構築された ExprState 構造体へのポインタ。

前提条件と不変条件

  • node が NULL でない場合、有効な ExprState が生成されます。
  • 戻り値の ExprState は、実行準備が完了した状態（ExecReadyExpr 呼び出し済み）となります。

アルゴリズムまたは振る舞い

1. **初期化**: ExprState ノードを生成し、expr と parent を設定します。 2. **セットアップ**: ExecCreateExprSetupSteps を呼び出し、タプルデフォーム等の必要な準備ステップを挿入します。 3. **再帰的コンパイル**: ExecInitExprRec を呼び出し、式ツリーを走査して ExprEvalStep のリストを構築します。 4. **終了処理**: 命令配列の末尾に EEOP_DONE_RETURN を追加し、ExecReadyExpr を実行して JIT コンパイルの試行やインタプリタの準備を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c
  • src/backend/commands/tablecmds.c
  • src/backend/executor/execExpr.c 内の各関数（ExecInitQual 等）
  • **呼び出し先**:
  • ExecCreateExprSetupSteps
  • ExecInitExprRec
  • ExprEvalPushStep
  • ExecReadyExpr

関連データ構造

  • ExprState: 式評価の実行状態を保持する構造体。
  • ExprEvalStep: 式評価の各ステップを定義する命令構造体。
  • Expr: 入力となる式ツリーのノード。
  • PlanState: 親となるプランの状態。

失敗モードと境界条件

  • node が NULL の場合、関数は即座に NULL を返します。
  • 式ツリーが非常に深い場合、ExecInitExprRec 内でスタックオーバーフローが発生する可能性があります（check_stack_depth() による保護あり）。

保守担当者向けメモ

  • 本関数はコンパイルのみを行い、実行は行いません。
  • ExprEvalStep の順序は execExprInterp.c のインタプリタと密接に関連しているため、変更時は整合性に注意してください。
