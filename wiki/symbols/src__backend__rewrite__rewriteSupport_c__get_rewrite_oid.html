get_rewrite_oid

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteSupport.c:91

目的

指定されたリレーション（テーブル）に関連付けられたルール名から、対応する pg_rewrite カタログの OID を取得します。

シグネチャまたは形状

Oid get_rewrite_oid(Oid relid, const char *rulename, bool missing_ok)

入力と出力

  • **入力**:
  • relid (Oid): 対象リレーションの OID。
  • rulename (const char *): 検索対象のルール名。
  • missing_ok (bool): true の場合、ルールが見つからなくてもエラーにせず InvalidOid を返します。
  • **出力**:
  • 成功時: 対象ルールの OID。
  • 失敗時: missing_ok が false の場合はエラーをスローし、true の場合は InvalidOid を返します。

前提条件と不変条件

  • RULERELNAME システムキャッシュを使用して検索を行います。
  • 取得したタプルから抽出された ruleform->ev_class が、入力された relid と一致することを Assert で検証しています。

アルゴリズムまたは振る舞い

1. SearchSysCache2 を用いて、relid と rulename をキーに pg_rewrite カタログを検索します。 2. タプルが見つからない場合、missing_ok に応じてエラーを報告するか InvalidOid を返します。 3. タプルが見つかった場合、GETSTRUCT で Form_pg_rewrite 構造体を取得し、その oid フィールドを保持します。 4. ReleaseSysCache を呼び出してタプルの参照を解放します。 5. 保持した ruleoid を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c:1453
  • **呼び出し先**:
  • SearchSysCache2 (システムキャッシュ検索)
  • HeapTupleIsValid (タプル有効性判定)
  • ereport (エラー報告)
  • get_rel_name (エラーメッセージ用リレーション名取得)
  • GETSTRUCT (タプル構造体変換)
  • ReleaseSysCache (キャッシュ解放)

関連データ構造

  • pg_rewrite (カタログ定義)
  • HeapTuple (システムキャッシュタプル)
  • Form_pg_rewrite (ルール定義構造体)

失敗モードと境界条件

  • 指定した relid と rulename の組み合わせが存在しない場合、missing_ok が false であれば ERRCODE_UNDEFINED_OBJECT エラーが発生します。

保守担当者向けメモ

  • 本関数はシステムキャッシュを利用するため、カタログへの直接アクセスよりも高速です。
  • Assert による relid の検証は、キャッシュの整合性を確認するための重要なガードレールです。
