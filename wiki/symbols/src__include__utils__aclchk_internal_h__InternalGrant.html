InternalGrant

定義

  • 種別: struct
  • 定義箇所: src/include/utils/aclchk_internal.h:31

目的

SQLの GRANT および REVOKE 文の解析結果を保持し、パーサーから実行エンジン（システムカタログ更新処理）へ情報を伝達するための中間データ構造です。

シグネチャまたは形状

typedef struct {
    bool        is_grant;
    ObjectType  objtype;
    List       *objects;
    bool        all_privs;
    AclMode     privileges;
    List       *col_privs;
    List       *grantees;
    bool        grant_option;
    DropBehavior behavior;
} InternalGrant;

入力と出力

  • **入力**: パーサーが生成した構文木から、名前解決（OID変換）を経て本構造体に値が格納されます。
  • **出力**: 実行エンジンが本構造体の情報を参照し、pg_class 等のシステムカタログ内のACLエントリを更新します。

前提条件と不変条件

  • col_privs（列レベルの権限）は、objtype が OBJECT_TABLE である場合にのみ有効です。
  • all_privs が true の場合、privileges フィールドはオブジェクト型に応じた適切なビットマスクに正規化される必要があります。

アルゴリズムまたは振る舞い

1. **変換**: パーサーが読み取った構文木内の文字列がOIDに解決され、本構造体に詰め込まれます。 2. **正規化**: all_privs が指定されている場合、privileges フィールドがオブジェクト型に基づき更新されます。 3. **実行**: 構築された InternalGrant を基に、システムカタログの権限情報が更新されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c 内の関数群（ExecGrantStmt_oids 等）が、GrantStmt を InternalGrant 形式へ変換して利用しています。
  • **呼び出し先**: 本構造体はデータ保持用であり、直接関数を呼び出すことはありません。src/backend/catalog/aclchk.c 内の ExecGrant_Relation や ExecGrant_Language_check 等の関数が、本構造体を引数として受け取り処理を実行します。

関連データ構造

  • ObjectType: 対象オブジェクトの種類を定義。
  • AclMode: 権限の集合をビットマスクで保持。
  • AccessPriv: 列レベルの権限指定を保持するノード。
  • List: オブジェクトや権限付与対象（grantees）のリスト管理に使用。

失敗モードと境界条件

  • all_privs フラグの操作時に privileges フィールドが書き換えられる副作用があるため、構造体の取り扱いには注意が必要です。

保守担当者向けメモ

  • 本構造体は GRANT/REVOKE 処理の橋渡し役であり、関数宣言は含まれていません。
  • 権限更新ロジックの変更時は、src/backend/catalog/aclchk.c での消費箇所と整合性を取る必要があります。
