decide_k_bits

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-mpi-openssl.c:91

目的

ElGamal 暗号化処理において、乱数 $k$ のビット長を決定する。GnuPG の挙動を模倣したヒューリスティックな計算を行う。

シグネチャまたは形状

int decide_k_bits(int p_bits)

入力と出力

  • **入力**: p_bits (ElGamal 素数 $p$ のビット長)
  • **出力**: 決定された乱数 $k$ のビット長（整数）

前提条件と不変条件

  • 入力 p_bits は正の整数であることが想定される。

アルゴリズムまたは振る舞い

入力された素数 $p$ のビット長に基づき、以下の条件分岐で計算を行う：

  • p_bits <= 5120 の場合: p_bits / 10 + 160 を返す。
  • p_bits > 5120 の場合: (p_bits / 8 + 200) * 3 / 2 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-mpi-openssl.c 内の pgp_elgamal_encrypt 関数（BN_num_bits(p) の結果を引数として渡す）。
  • **呼び出し先**: なし。

関連データ構造

  • なし。

失敗モードと境界条件

  • 負数や極端に小さい p_bits が入力された場合の挙動は定義されていない。

保守担当者向けメモ

  • 本関数はセキュリティとパフォーマンスのバランスを考慮した経験的な計算式である。変更を行う際は、GnuPG 等の参照実装との整合性に注意すること。
