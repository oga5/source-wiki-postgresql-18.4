SlotInvalidationCauseMap

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/slot.c:106

目的

レプリケーションスロットの無効化理由（ReplicationSlotInvalidationCause）と、それに対応する人間が判読可能な名称（文字列）をマッピングするために使用されます。

シグネチャまたは形状

typedef struct SlotInvalidationCauseMap
{
	ReplicationSlotInvalidationCause cause;
	const char *cause_name;
} SlotInvalidationCauseMap;

入力と出力

  • 入力: ReplicationSlotInvalidationCause 型の列挙値および対応する文字列リテラル。
  • 出力: なし（データ構造の定義のみ）。

前提条件と不変条件

  • cause フィールドは ReplicationSlotInvalidationCause 型の有効な列挙値である必要があります。
  • cause_name は静的な文字列リテラルを指すことが想定されています。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のための型定義です。この構造体の配列（SlotInvalidationCauses）を通じて、システム内で発生したスロット無効化イベントをログ出力や統計情報として可視化する際の変換テーブルとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/slot.c 内の SlotInvalidationCauses 配列定義において使用されています。
  • 呼び出し先: 提供されたソース情報からは、本構造体を利用して特定の関数が呼び出されている証拠は確認できません。

関連データ構造

  • ReplicationSlotInvalidationCause: 無効化理由を定義する列挙型。
  • SlotInvalidationCauses: SlotInvalidationCauseMap 型の静的配列。

失敗モードと境界条件

  • 本構造体はデータ定義であるため、直接的な失敗モードはありません。ただし、SlotInvalidationCauses 配列を走査する際に、定義されていない cause 値が渡された場合の挙動は、その走査ロジックの実装に依存します。

保守担当者向けメモ

  • 本構造体は、新しい無効化理由が追加された際に SlotInvalidationCauses 配列を更新するための基盤となります。変更時は ReplicationSlotInvalidationCause 列挙型との整合性を確認してください。
