pg_buffercache_evict_all

定義

  • 種別: function
  • 定義箇所: contrib/pg_buffercache/pg_buffercache_pages.c:760

目的

共有バッファキャッシュ内のすべてのピン留めされていない（unpinned）バッファを強制的に解放（エビクション）し、その結果（解放数、フラッシュ数、スキップ数）を SQL レイヤーへ報告する。

シグネチャまたは形状

Datum pg_buffercache_evict_all(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (SQL インターフェース経由の呼び出し引数)
  • **出力**: Datum (解放されたバッファ数、フラッシュされたバッファ数、スキップされたバッファ数を含む複合型レコード)

前提条件と不変条件

  • 呼び出し元はスーパーユーザー権限を有している必要がある（pg_buffercache_superuser_check による検証）。
  • 関数は複合型を返す必要がある（get_call_result_type による検証）。

アルゴリズムまたは振る舞い

1. 戻り値の型が複合型であることを確認する。 2. pg_buffercache_superuser_check を呼び出し、実行権限を検証する。 3. EvictAllUnpinnedBuffers を呼び出し、共有バッファキャッシュ全体のスキャンと解放処理を実行する。 4. 解放処理の結果（buffers_evicted, buffers_flushed, buffers_skipped）を values 配列に格納する。 5. heap_form_tuple を使用して結果タプルを構築し、PG_RETURN_DATUM で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 マクロによる登録）。
  • **呼び出し先**:
  • get_call_result_type
  • pg_buffercache_superuser_check
  • EvictAllUnpinnedBuffers
  • heap_form_tuple

関連データ構造

  • TupleDesc: 返却するタプルの構造定義。
  • HeapTuple: SQL レイヤーへ返すためのタプルデータ。
  • NUM_BUFFERCACHE_EVICT_ALL_ELEM: 返却する要素数（3要素）。

失敗モードと境界条件

  • 戻り値の型が複合型でない場合、elog(ERROR) により処理が中断される。
  • スーパーユーザー権限がない場合、pg_buffercache_superuser_check によりエラーが発生する。

保守担当者向けメモ

  • 本関数は EvictAllUnpinnedBuffers のラッパーとして機能する。
  • 共有バッファの操作を伴うため、実行時にはシステム全体に影響を与える可能性がある。
