pg_popcount64_fast

定義

  • src/port/pg_bitutils.c:236

目的

64ビット整数に含まれる「1」のビット数を、ハードウェア命令を利用して高速にカウントする。

シグネチャまたは形状

static inline int pg_popcount64_fast(uint64 word)

入力と出力

  • **入力**: uint64 word（カウント対象の64ビット整数）
  • **出力**: int（word 内の1のビット数）

前提条件と不変条件

  • 本関数は、実行環境が POPCNT 命令をサポートしていることを前提として呼び出される必要がある。
  • コンパイル環境が _MSC_VER の場合は __popcnt64 組み込み関数を使用し、それ以外の場合は GCC/Clang 互換のインラインアセンブラ（popcntq）を使用する。

アルゴリズムまたは振る舞い

1. _MSC_VER が定義されている場合、__popcnt64 を呼び出して結果を返す。 2. それ以外の場合、インラインアセンブラを用いて popcntq 命令を実行し、結果を res に格納して int 型にキャストして返す。 3. アセンブラブロックでは cc（条件コード）レジスタが変更されることを明示している。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/port/pg_bitutils.c 内の動的ディスパッチ処理（pg_popcount64 ポインタへの代入）。
  • src/port/pg_bitutils.c 内のバッファ走査処理（pg_popcount64_fast(*words++) 等の呼び出し）。
  • **呼び出し先**:
  • ハードウェア命令（__popcnt64 または popcntq）。

関連データ構造

  • なし。

失敗モードと境界条件

  • CPUが POPCNT 命令をサポートしていない環境で本関数を直接実行した場合、不正命令例外が発生する可能性がある。そのため、必ず pg_popcount_available 等による事前チェックを経て呼び出す必要がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、最適化レベルに応じてインライン展開される。
  • インラインアセンブラの構文は GCC/Clang に依存しているため、他のコンパイラへの移植時には対応が必要となる可能性がある。
