smgrnblocks_cached

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/smgr.c:846

目的

指定されたリレーションおよびフォークにおける、キャッシュされたブロック数を取得します。

シグネチャまたは形状

BlockNumber smgrnblocks_cached(SMgrRelation reln, ForkNumber forknum)

入力と出力

  • **入力**:
  • reln: 対象となる SMgrRelation オブジェクト。
  • forknum: 取得対象のフォーク番号。
  • **出力**:
  • キャッシュされたブロック数（BlockNumber）。キャッシュが無効な場合やリカバリ中でない場合は InvalidBlockNumber を返します。

前提条件と不変条件

  • 現在の設計では、ファイルサイズの変更に対する共有無効化メカニズムが存在しないため、リカバリ中以外でのキャッシュ利用は制限されています。
  • リカバリ中（InRecovery が真）であり、かつ対象のキャッシュ値が InvalidBlockNumber でない場合にのみ、キャッシュされた値が返されます。

アルゴリズムまたは振る舞い

1. InRecovery フラグを確認します。 2. リカバリ中かつ reln->smgr_cached_nblocks[forknum] が有効な値であれば、その値を返します。 3. 上記条件を満たさない場合、InvalidBlockNumber を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c: nForkBlock の取得および block 配列の初期化に使用。
  • src/backend/storage/smgr/smgr.c: 内部的な呼び出し（行824）。
  • **呼び出し先**:
  • 外部関数への呼び出しは含まれていません。

関連データ構造

  • SMgrRelation: ストレージマネージャのリレーションハンドル。smgr_cached_nblocks 配列を保持します。
  • BlockNumber: ブロック数を表す型。
  • ForkNumber: フォークを識別する列挙型。

失敗モードと境界条件

  • キャッシュが更新されていない（InvalidBlockNumber）場合、本関数は常に InvalidBlockNumber を返します。
  • リカバリ中以外では、キャッシュの状態に関わらず常に InvalidBlockNumber を返す仕様となっています。

保守担当者向けメモ

  • 本関数が返す値は「キャッシュ」であるため、呼び出し側は古いデータ（stale data）が含まれる可能性があることを前提に処理を行う必要があります。
  • 共有無効化メカニズムが将来的に実装された場合、リカバリ中以外の条件分岐が見直される可能性があります。
