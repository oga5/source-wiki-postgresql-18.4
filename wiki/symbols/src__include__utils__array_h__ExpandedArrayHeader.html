ExpandedArrayHeader

定義

  • 種別: struct
  • 定義箇所: src/include/utils/array.h:115

目的

メモリ上で配列要素を個別に保持し、変更操作（要素の追加や更新など）を高速化するための展開済み配列（Expanded Object）構造体です。

シグネチャまたは形状

typedef struct ExpandedArrayHeader {
    ExpandedObjectHeader hdr;
    int         ea_magic;
    int         ndims;
    int        *dims;
    int        *lbound;
    Oid         element_type;
    int16       typlen;
    bool        typbyval;
    char        typalign;
    Datum      *dvalues;
    bool       *dnulls;
    int         dvalueslen;
    int         nelems;
    Size        flat_size;
    ArrayType  *fvalue;
    char       *fstartptr;
    char       *fendptr;
} ExpandedArrayHeader;

入力と出力

  • **入力**: 構築時または変換時に Datum 配列や既存の ArrayType を受け取り、メモリ上に展開します。
  • **出力**: get_flat_size 等の操作を通じて、最終的に標準的な ArrayType（フラット形式）へ変換可能です。

前提条件と不変条件

  • 配列要素は常に完全に detoast された状態で保持されます。
  • dvalues および dnulls 配列は、オブジェクトのプライベートコンテキスト内で palloc されます。
  • fstartptr および fendptr は、fvalue が存在する場合にそのデータ領域の境界を指し、dvalues 内のポインタがフラット表現を指しているか、個別に割り当てられた領域を指しているかを判別するために使用されます。

アルゴリズムまたは振る舞い

  • 展開済みオブジェクトインフラを利用し、配列の変更操作を効率化します。
  • dvalues 配列のサイズは動的に変更可能です。
  • flat_size を保持することで、フラット化が必要な際の計算コストを削減します。
  • ea_magic はデバッグ目的の識別子として使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/extended_stats.c および src/backend/utils/adt/array_expanded.c 内の関数群から参照されています。
  • **呼び出し先**: 展開済みオブジェクトの基底構造である ExpandedObjectHeader を含みます。具体的な関数呼び出し関係は、本ソース抜粋からは特定できません。

関連データ構造

  • ExpandedObjectHeader: 展開済みオブジェクトの共通ヘッダー。
  • ArrayType: 標準的なフラット形式の配列構造体。
  • Datum: 配列要素の保持に使用される汎用データ型。

失敗モードと境界条件

  • dnulls は、NULL要素が存在しない場合は NULL になる可能性があります。
  • fvalue が存在しない場合、fvalue は NULL となります。
  • flat_size が不明な場合は 0 が格納されます。

保守担当者向けメモ

  • 配列要素のアクセスには、型ごとのアライメント要件（typalign）を考慮する必要があります。
  • dvalues 内のポインタがフラット表現を指しているか、個別に割り当てられた領域を指しているかの判定には fstartptr と fendptr を使用してください。
