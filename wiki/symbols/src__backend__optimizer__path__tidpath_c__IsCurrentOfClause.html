IsCurrentOfClause

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/tidpath.c:211

目的

指定された RestrictInfo が WHERE CURRENT OF 句に対応する式であり、かつ対象のリレーションを参照しているかを判定する。

シグネチャまたは形状

static bool IsCurrentOfClause(RestrictInfo *rinfo, RelOptInfo *rel)

入力と出力

  • **入力**:
  • rinfo: 評価対象の RestrictInfo 構造体。
  • rel: 比較対象の RelOptInfo 構造体。
  • **出力**:
  • bool: 条件が CurrentOfExpr であり、かつ rel->relid と一致する場合は true、それ以外は false。

前提条件と不変条件

  • rinfo は有効なポインタである必要がある。
  • rinfo->clause が NULL の場合、関数は false を返す。

アルゴリズムまたは振る舞い

1. rinfo->clause が存在し、かつその型が CurrentOfExpr であるかを確認する。一致しない場合は false を返す。 2. CurrentOfExpr ノード内の cvarno（対象リレーションの識別子）が、引数 rel の relid と一致するかを比較する。 3. 一致すれば true、一致しなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/tidpath.c 内の TidQualFromRestrictInfoList 等から呼び出されている（ツールによる直接的な参照確認）。
  • **呼び出し先**:
  • IsA マクロ（ノード型判定）。

関連データ構造

  • RestrictInfo: クエリの制約条件を保持する構造体。
  • CurrentOfExpr: WHERE CURRENT OF 句を表すノード。
  • RelOptInfo: リレーションの最適化情報を保持する構造体。

失敗モードと境界条件

  • rinfo->clause が NULL の場合、安全に false を返す。
  • rinfo->clause が CurrentOfExpr 以外の型である場合、false を返す。

保守担当者向けメモ

  • 本関数は WHERE CURRENT OF 句を識別するための専用ロジックである。この句が検出された場合、プランナは他のパス生成をスキップし、このパスを優先的に採用する挙動をとるため、変更時には注意が必要である。
