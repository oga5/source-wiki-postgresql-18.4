getnameinfo_unix

定義

  • 種別: function
  • 定義箇所: src/common/ip.c:227

目的

Unix ドメインソケット（AF_UNIX）のアドレス情報を、ホスト名およびサービス名として文字列形式に変換します。標準の getnameinfo が対応していない Unix ソケット特有のパス表現を解決するために使用されます。

シグネチャまたは形状

static int getnameinfo_unix(const struct sockaddr_un *sa, int salen,
                            char *node, int nodelen,
                            char *service, int servicelen,
                            int flags);

入力と出力

  • **入力**:
  • sa: sockaddr_un 構造体へのポインタ。
  • salen: アドレス構造体のサイズ。
  • node, nodelen: ホスト名格納用バッファとそのサイズ。
  • service, servicelen: サービス名（パス）格納用バッファとそのサイズ。
  • flags: 動作フラグ（本実装では未使用）。
  • **出力**: 成功時は 0 を返します。失敗時は EAI_FAIL または EAI_MEMORY を返します。

前提条件と不変条件

  • sa は NULL であってはならず、sa->sun_family は AF_UNIX である必要があります。
  • node と service の両方が NULL である場合はエラーとなります。

アルゴリズムまたは振る舞い

1. 引数の妥当性を検証し、不適合な場合は EAI_FAIL を返します。 2. node が指定されている場合、固定文字列 "[local]" を書き込みます。書き込みがバッファサイズを超える場合は EAI_MEMORY を返します。 3. service が指定されている場合、sa->sun_path を評価します。

  • 先頭がヌル文字で、かつ2文字目がヌル文字でない場合、Linux 抽象名前空間ソケットと見なし、@ を付与してパスを書き込みます。
  • それ以外の場合は、通常のパスとして書き込みます。

    • 4. 書き込み時にバッファ不足が発生した場合は EAI_MEMORY を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/ip.c 内の pg_getnameinfo_all から呼び出されます。
  • **呼び出し先**: 標準ライブラリの snprintf を使用します。

関連データ構造

  • struct sockaddr_un: Unix ドメインソケットのアドレス情報を保持する構造体。

失敗モードと境界条件

  • sa == NULL または sun_family != AF_UNIX: EAI_FAIL を返します。
  • node および service が両方 NULL: EAI_FAIL を返します。
  • バッファサイズ不足 (snprintf の戻り値が負または len 以上): EAI_MEMORY を返します。

保守担当者向けメモ

  • 本関数は pg_getnameinfo_all の内部ヘルパーとして設計されています。
  • 抽象名前空間ソケットの判定ロジック（sun_path[0] == '\0'）は、Linux 固有の挙動に依存しています。他プラットフォームへの移植時にはこの挙動の差異に注意してください。
