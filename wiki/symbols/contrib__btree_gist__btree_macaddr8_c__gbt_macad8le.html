gbt_macad8le

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr8.c:46

目的

macaddr8 型の2つの値に対して「以下（less than or equal to）」の比較を行い、その結果を真偽値で返します。

シグネチャまたは形状

bool gbt_macad8le(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の macaddr8 値（void *）。
  • b: 比較対象の macaddr8 値（void *）。
  • flinfo: 関数呼び出し情報（FmgrInfo *）。
  • **出力**: a <= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な macaddr8 型のデータである必要があります。
  • PostgreSQL の DirectFunctionCall2 を使用するため、macaddr8_le 関数がシステムに登録されている必要があります。

アルゴリズムまたは振る舞い

1. PointerGetDatum を使用して、入力された void * 型のポインタを PostgreSQL の Datum 形式に変換します。 2. DirectFunctionCall2 を介して、PostgreSQL のネイティブ関数 macaddr8_le を呼び出します。 3. macaddr8_le の戻り値を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_macaddr8.c 内の静的構造体（tinfo）の初期化において参照されています。
  • **呼び出し先**:
  • macaddr8_le（PostgreSQL のコア関数）。

関連データ構造

  • macaddr8: PostgreSQL のネットワークアドレス型。
  • FmgrInfo: 関数マネージャ情報構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、PointerGetDatum の挙動に依存してクラッシュする可能性があります。
  • macaddr8_le 関数が内部でエラーを発生させた場合、PostgreSQL のエラー処理メカニズムにより処理が中断されます。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける macaddr8 型の比較アダプターです。
  • 比較ロジック自体はコアの macaddr8_le に委譲しているため、比較仕様の変更はコア側の関数を確認してください。
