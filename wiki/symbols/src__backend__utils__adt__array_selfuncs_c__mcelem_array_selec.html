mcelem_array_selec

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_selfuncs.c:427

目的

配列型に対する演算子（@>、<@、&&）の選択率（Selectivity）を、統計情報（MCELEM、ヒストグラム）を用いて推定する中核関数です。

シグネチャまたは形状

Selectivity mcelem_array_selec(ArrayType *array, TypeCacheEntry *typentry,
                               Datum *mcelem, int nmcelem,
                               float4 *numbers, int nnumbers,
                               float4 *hist, int nhist,
                               Oid operator)

入力と出力

  • **入力**:
  • array: 評価対象の定数配列。
  • typentry: 配列要素の型情報キャッシュ。
  • mcelem, nmcelem: 最頻要素のデータ配列とその個数。
  • numbers, nnumbers: 各最頻要素の出現頻度データとその個数。
  • hist, nhist: 要素数ヒストグラムデータとその個数。
  • operator: 評価する演算子のOID（OID_ARRAY_CONTAINS_OP, OID_ARRAY_OVERLAP_OP, OID_ARRAY_CONTAINED_OP）。
  • **出力**:
  • Selectivity (double): 推定された選択率（0.0〜1.0）。

前提条件と不変条件

  • array は deconstruct_array により分解可能であること。
  • operator はサポートされている配列演算子であること。
  • NULL要素を含む配列に対し、@> 演算子が指定された場合は常に 0.0 を返す。

アルゴリズムまたは振る舞い

1. **配列の分解**: deconstruct_array を使用して配列を Datum 配列に変換し、NULL要素を除去します。 2. **NULLチェック**: operator が @> で、かつ配列内にNULLが含まれる場合、一致する行は存在しないと見なし 0.0 を返します。 3. **ソート**: 抽出した要素を element_compare を用いてソートし、重複排除とマージを効率化します。 4. **演算子別の推定**:

  • @> または &&: mcelem_array_contain_overlap_selec を呼び出し、包含または重複の選択率を計算します。
  • <@: mcelem_array_contained_selec を呼び出し、ヒストグラムを用いた補正を含む選択率を計算します。

    • 5. **後処理**: 割り当てたメモリを解放し、計算結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/array_selfuncs.c 内の calc_arraycontsel 等から呼び出されます。
  • **呼び出し先**:
  • mcelem_array_contain_overlap_selec
  • mcelem_array_contained_selec
  • deconstruct_array, qsort_arg, pfree, elog

関連データ構造

  • ArrayType: PostgreSQLの配列型内部表現。
  • TypeCacheEntry: 型の比較関数やサイズ情報を含むキャッシュ構造体。

失敗モードと境界条件

  • **未対応演算子**: サポート外の operator が渡された場合、elog(ERROR) により処理を中断します。
  • **メモリ不足**: deconstruct_array 等の内部処理でメモリ不足が発生した場合は例外がスローされます。

保守担当者向けメモ

  • 本関数は統計情報を用いた推定のディスパッチャーとして機能します。新しい配列演算子の選択率推定を追加する場合は、演算子分岐の条件を追加し、適切な計算関数を呼び出す必要があります。
