ECPGgeneric_varchar

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/ecpglib/ecpglib_extern.h:35

目的

ECPGライブラリ内において、可変長文字列（VARCHAR）データを保持するための汎用的な構造体です。

シグネチャまたは形状

struct ECPGgeneric_varchar
{
    int         len;
    char        arr[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • len: 文字列の長さを保持する整数。
  • arr: 実際の文字列データを格納するフレキシブル配列。

前提条件と不変条件

  • arr はフレキシブル配列メンバーとして定義されており、構造体の末尾に可変長のデータ領域を確保することを前提としています。
  • 文字列操作時には len が実際のデータ長と整合している必要があります。

アルゴリズムまたは振る舞い

  • ECPGの実行時ライブラリにおいて、SQLのVARCHAR型とC言語のホスト変数間でのデータ受け渡しに使用されます。
  • misc.c 等では、この構造体へのポインタをキャストして len を 0 にリセットしたり、arr の先頭を終端文字（0x00）で初期化する操作が行われます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/interfaces/ecpg/ecpglib/data.c: データ変換処理で使用。
  • src/interfaces/ecpg/ecpglib/descriptor.c: 記述子操作で使用。
  • src/interfaces/ecpg/ecpglib/execute.c: SQL実行時の変数マッピングで使用。
  • src/interfaces/ecpg/ecpglib/misc.c: 構造体の初期化および状態確認で使用。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • struct variable: ECPGにおけるホスト変数管理構造体。本構造体は variable を介して参照されることが多いです。

失敗モードと境界条件

  • arr のサイズは動的に決定されるため、メモリ割り当てが不十分な場合にバッファオーバーフローのリスクがあります。
  • len の値が実際の arr の内容と一致しない場合、データの読み取り時に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は ecpglib 内部で広く利用されています。構造を変更する場合は、data.c や execute.c 等のキャスト処理に影響がないか確認してください。
  • フレキシブル配列を使用しているため、インスタンス化の際は適切なメモリサイズ計算が必要です。
