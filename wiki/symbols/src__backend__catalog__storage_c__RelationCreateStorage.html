RelationCreateStorage

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/storage.c:121

目的

リレーション（テーブルやインデックス）の物理的なストレージファイルを生成し、必要に応じてトランザクションアボート時の削除予約や同期管理を行う。

シグネチャまたは形状

SMgrRelation RelationCreateStorage(RelFileLocator rlocator, char relpersistence, bool register_delete)

入力と出力

  • **入力**:
  • rlocator: 作成対象リレーションの識別子。
  • relpersistence: リレーションの永続性（RELPERSISTENCE_TEMP, UNLOGGED, PERMANENT）。
  • register_delete: トランザクションアボート時に物理ファイルを削除する予約を行うかどうかのフラグ。
  • **出力**:
  • 作成されたストレージマネージャリレーション（SMgrRelation）のポインタ。

前提条件と不変条件

  • 並列モード（Parallel Mode）では実行不可（pendingSyncHash の更新ができないため）。
  • relpersistence には有効な値（TEMP, UNLOGGED, PERMANENT）が指定される必要がある。

アルゴリズムまたは振る舞い

1. **永続性判定**: relpersistence に基づき、WAL 記録の要否とプロセス番号を決定する。 2. **ストレージ生成**: smgropen でハンドルを取得し、smgrcreate で物理ファイルを作成する。 3. **WAL 記録**: 永続的なリレーションの場合、log_smgrcreate を呼び出し WAL に記録する。 4. **削除予約**: register_delete が真の場合、TopMemoryContext に PendingRelDelete 構造体を確保し、現在のトランザクションネストレベルと共に pendingDeletes リストへ追加する。 5. **同期予約**: 永続的かつ WAL ログ出力が不要な場合（!XLogIsNeeded()）、AddPendingSync を呼び出しコミット時の同期を予約する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c
  • src/backend/catalog/heap.c
  • src/backend/commands/tablecmds.c
  • src/backend/storage/buffer/bufmgr.c

  • **呼び出し先**:

関連データ構造

  • SMgrRelation: ストレージマネージャが管理するリレーションの物理状態。
  • PendingRelDelete: トランザクション終了時の削除待ちリストを構成する構造体。
  • RelFileLocator: リレーションの物理的な場所を特定する識別子。

失敗モードと境界条件

  • relpersistence が不正な値の場合、elog(ERROR) を発生させる。
  • 並列モードで呼び出された場合、Assert により異常終了する。

保守担当者向けメモ

  • pendingDeletes リストは TopMemoryContext に保持されるため、トランザクション終了後も生存する。
  • TRUNCATE 等の操作と組み合わさる際、WAL ログと物理ファイルの同期順序が整合性を保つよう設計されている。
