decode_history

定義

  • 種別: function
  • 定義箇所: src/bin/psql/input.c:318

目的

readline の履歴リスト内に保存されているエスケープ済みの改行文字（NL_IN_HISTORY）を、実際の改行コード（\n）に復元します。

シグネチャまたは形状

static void decode_history(void)

入力と出力

  • 入力: なし（グローバルな履歴リストを直接操作）
  • 出力: なし（副作用として履歴リスト内の文字列を書き換える）

前提条件と不変条件

  • readline ライブラリが初期化されていること。
  • 履歴リスト内の各エントリが、以前に encode_history によって NL_IN_HISTORY に置換されていること。

アルゴリズムまたは振る舞い

1. BEGIN_ITERATE_HISTORY マクロを使用して、履歴リストの全エントリを走査します。 2. 各エントリの文字列（line）を先頭から順に走査します。 3. 文字が NL_IN_HISTORY と一致する場合、それを \n に置換します。 4. END_ITERATE_HISTORY マクロで走査を終了します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/input.c:394（具体的な関数名はソース抜粋からは不明）
  • 呼び出し先: BEGIN_ITERATE_HISTORY および END_ITERATE_HISTORY マクロ（readline ライブラリのイテレーション機能）。

関連データ構造

  • HIST_ENTRY: readline が使用する履歴エントリ構造体。line メンバを介して文字列にアクセスします。

失敗モードと境界条件

  • HIST_ENTRY.line が const char * として宣言されているプラットフォームでは、キャストによる書き込みが発生するため、環境依存の挙動に注意が必要です。

保守担当者向けメモ

  • 本関数は readline と libedit の差異を吸収するマクロに依存しています。履歴の走査順序やイテレーションの仕組みを変更する場合は、関連するマクロ定義を確認してください。
