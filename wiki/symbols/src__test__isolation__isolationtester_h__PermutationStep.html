PermutationStep

定義

  • 種別: struct
  • 定義箇所: src/test/isolation/isolationtester.h:60

目的

分離レベルテストの順列（Permutation）内において、実行される個別のステップをラップし、実行時のブロック条件を管理するためのデータ構造です。

シグネチャまたは形状

typedef struct {
    char *name;
    PermutationStepBlocker **blockers;
    int nblockers;
    Step *step;
} PermutationStep;

入力と出力

  • **入力**:
  • name: 参照する Step の名前。
  • blockers: 実行を阻止する条件の配列。
  • nblockers: ブロック条件の数。
  • **出力**:
  • step: check_testspec() によって解決された、対応する Step 構造体へのポインタ。

前提条件と不変条件

  • check_testspec() が呼び出されるまで、step フィールドは NULL です。
  • step フィールドは、check_testspec() 実行後に name に対応する Step オブジェクトを指すようリンクされます。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持を目的としています。実行エンジンは、この構造体を走査して blockers を確認し、条件が満たされた場合に step が指す SQL コマンドを実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/isolation/isolationtester.c 内の各関数（try_complete_steps, try_complete_step, step_has_blocker など）が、テスト実行中に本構造体を参照します。
  • **呼び出し先**: 実行時に step フィールドを介して Step 構造体を参照します。

関連データ構造

  • Step: 本構造体が参照する実際の SQL コマンド定義。
  • PermutationStepBlocker: 実行を制御するブロック条件。
  • Permutation: 本構造体をリストとして保持する親構造体。

失敗モードと境界条件

  • check_testspec() 実行前に step フィールドへアクセスすると、未解決のポインタ（NULL）を参照するリスクがあります。

保守担当者向けメモ

  • PermutationStep は静的な Step 定義と実行時のコンテキストを橋渡しする役割を持ちます。
  • 新しいブロック条件を追加する場合は、PermutationStepBlockerType 列挙型を拡張し、blockers 配列の処理ロジックを isolationtester.c に追加してください。
