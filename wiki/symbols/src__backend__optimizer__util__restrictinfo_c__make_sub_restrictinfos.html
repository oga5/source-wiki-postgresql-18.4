make_sub_restrictinfos

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:261

目的

論理演算子（OR または AND）を含む式ツリーを再帰的に走査し、各末端の条件式に対して RestrictInfo ノードを生成・付与する。

シグネチャまたは形状

static Expr *make_sub_restrictinfos(PlannerInfo *root,
                                    Expr *clause,
                                    bool is_pushed_down,
                                    bool has_clone,
                                    bool is_clone,
                                    bool pseudoconstant,
                                    Index security_level,
                                    Relids required_relids,
                                    Relids incompatible_relids,
                                    Relids outer_relids)

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報。
  • clause: 処理対象の式（Expr）。
  • 各種フラグおよび Relids: RestrictInfo に設定されるメタデータ。
  • **出力**:
  • RestrictInfo でラップされた、または再帰的に構築された Expr ツリー。

前提条件と不変条件

  • clause が OR 句の場合、再帰呼び出し時に required_relids は NULL として渡される。
  • AND 句および OR 句の各引数は、本関数によって再帰的に処理される。

アルゴリズムまたは振る舞い

1. **OR 句の場合**: clause の各引数に対して再帰的に make_sub_restrictinfos を呼び出し、結果のリストを make_orclause で再構築する。その後、全体を make_plain_restrictinfo でラップして返す。 2. **AND 句の場合**: clause の各引数に対して再帰的に make_sub_restrictinfos を呼び出し、結果のリストを make_andclause で再構築して返す。 3. **その他の場合**: make_plain_restrictinfo を呼び出し、単一の RestrictInfo ノードを生成して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/restrictinfo.c 内の make_restrictinfo（直接の呼び出しを確認）。
  • 本関数自身（再帰呼び出し）。
  • **呼び出し先**:
  • make_plain_restrictinfo
  • make_orclause
  • make_andclause

関連データ構造

  • RestrictInfo: 条件式にメタデータを付与したノード。
  • PlannerInfo: プランナのグローバル状態。
  • BoolExpr: OR や AND 句を表現するノード。

失敗モードと境界条件

  • clause が OR 句の場合、再帰呼び出しの引数として required_relids が NULL にリセットされるため、呼び出し元で指定された制約がサブ句には引き継がれない点に注意が必要。

保守担当者向けメモ

  • 本関数は RestrictInfo の階層構造を構築するための再帰的なユーティリティである。
  • OR 句の処理において required_relids が NULL になる仕様が、意図した動作であるか（結合条件の移動可能性判定等に影響しないか）を確認すること。
