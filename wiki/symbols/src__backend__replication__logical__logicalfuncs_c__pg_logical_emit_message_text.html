pg_logical_emit_message_text

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/logicalfuncs.c:381

目的

ユーザー定義のテキストメッセージをWAL（Write-Ahead Log）に挿入するためのSQLインターフェースを提供します。

シグネチャまたは形状

Datum pg_logical_emit_message_text(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して、メッセージの送信先（レプリケーションスロット名など）およびメッセージ内容（テキスト）を受け取ります。
  • **出力**: Datum 型を返します。内部的には pg_logical_emit_message_bytea の戻り値をそのまま返却します。

前提条件と不変条件

  • bytea 型と text 型のデータ表現が互換性を持つことを前提としています。
  • 実行には適切な権限が必要です（詳細は pg_logical_emit_message_bytea の仕様に依存）。

アルゴリズムまたは振る舞い

1. 受け取った fcinfo（関数呼び出し情報）をそのまま pg_logical_emit_message_bytea に引き渡します。 2. pg_logical_emit_message_bytea の処理結果を呼び出し元に返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で本関数を呼び出すクライアント（SQLクエリ）。
  • **呼び出し先**: pg_logical_emit_message_bytea。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に従う引数構造体。

失敗モードと境界条件

  • 内部的に呼び出される pg_logical_emit_message_bytea が失敗した場合、そのエラーがそのまま伝播します。
  • 引数が不正な場合や、論理デコーディングコンテキストが適切でない場合はエラーが発生します。

保守担当者向けメモ

  • 本関数は pg_logical_emit_message_bytea への単なるラッパーです。ロジックの変更が必要な場合は、原則として pg_logical_emit_message_bytea を修正してください。
