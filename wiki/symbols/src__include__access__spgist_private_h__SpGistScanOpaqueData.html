SpGistScanOpaqueData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/spgist_private.h:189

目的

SP-GiSTインデックスのスキャン実行時における動的な状態管理を行うための構造体です。検索キュー、メモリコンテキスト、演算子クラス情報、およびスキャン結果の一時保持領域を提供します。

シグネチャまたは形状

typedef struct SpGistScanOpaqueData {
    SpGistState state;
    pairingheap *scanQueue;
    MemoryContext tempCxt;
    MemoryContext traversalCxt;
    bool        searchNulls;
    bool        searchNonNulls;
    int         numberOfKeys;
    ScanKey     keyData;
    int         numberOfOrderBys;
    int         numberOfNonNullOrderBys;
    ScanKey     orderByData;
    Oid        *orderByTypes;
    int        *nonNullOrderByOffsets;
    Oid         indexCollation;
    FmgrInfo    innerConsistentFn;
    FmgrInfo    leafConsistentFn;
    double     *zeroDistances;
    double     *infDistances;
    TIDBitmap  *tbm;
    int64       ntids;
    bool        want_itup;
    TupleDesc   reconTupDesc;
    int         nPtrs;
    int         iPtr;
    ItemPointerData heapPtrs[MaxIndexTuplesPerPage];
    bool        recheck[MaxIndexTuplesPerPage];
    bool        recheckDistances[MaxIndexTuplesPerPage];
    HeapTuple   reconTups[MaxIndexTuplesPerPage];
    IndexOrderByDistance *distances[MaxIndexTuplesPerPage];
} SpGistScanOpaqueData;

入力と出力

  • **入力**: スキャン開始時に SpGistState を介して初期化され、演算子クラスの関数（innerConsistentFn, leafConsistentFn）や検索条件（keyData）が設定されます。
  • **出力**: amgetbitmap スキャン時は TIDBitmap に結果が格納され、amgettuple スキャン時は heapPtrs や reconTups を通じてタプル情報が返されます。

前提条件と不変条件

  • SP-GiSTのタプルは MAXALIGN に従う必要があります。
  • heapPtrs 等の配列サイズは MaxIndexTuplesPerPage に依存しており、これは SpGistLeafTuple が通常の IndexTuple より大きいことを利用した安全な境界として設計されています。

アルゴリズムまたは振る舞い

  • **スキャン管理**: pairingheap を用いて次に訪問すべきインデックス項目を管理します。
  • **メモリ管理**: tempCxt（短寿命）と traversalCxt（スキャン生存期間）の2つのメモリコンテキストを使い分け、メモリリークを防ぎます。
  • **検索制御**: searchNulls および searchNonNulls フラグにより、NULL値および非NULL値の検索対象を制御します。
  • **再構築**: want_itup が真の場合、reconTupDesc を使用してインデックスからタプルを再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgscan.c にて palloc0 を用いてインスタンス化されることが確認されています。
  • **呼び出し先**: 演算子クラスの関数（innerConsistentFn, leafConsistentFn）を呼び出し、ツリー探索を行います。

関連データ構造

  • SpGistState: インデックス操作の共通状態。
  • SpGistScanOpaque: 本構造体へのポインタ型定義。
  • pairingheap: スキャンキューの実装に使用。
  • TIDBitmap: ビットマップスキャン時の結果格納先。

失敗モードと境界条件

  • MaxIndexTuplesPerPage を超えるタプルがページ内に存在する場合、配列のオーバーフローが発生する可能性があります（ただし、SP-GiSTの設計上、この定数は安全な上限として機能する前提です）。

保守担当者向けメモ

  • 本構造体は amgetbitmap と amgettuple の両方のスキャンモードで共用されています。特定のフィールド（tbm や reconTups など）は特定のモードでのみ使用されるため、変更時は各スキャンパスへの影響を考慮してください。
