CreateExprContextInternal

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execUtils.c:236

目的

クエリ実行状態（EState）に関連付けられた式評価コンテキスト（ExprContext）を生成し、初期化する内部関数です。

シグネチャまたは形状

static ExprContext *CreateExprContextInternal(EState *estate, Size minContextSize, Size initBlockSize, Size maxBlockSize)

入力と出力

  • **入力**:
  • estate: 関連付けるクエリ実行状態。
  • minContextSize, initBlockSize, maxBlockSize: 式評価用メモリコンテキスト（ecxt_per_tuple_memory）のサイズ設定。
  • **出力**:
  • 初期化済みの ExprContext 構造体へのポインタ。

前提条件と不変条件

  • estate->es_query_cxt が有効であること。
  • 生成された ExprContext は、estate->es_exprcontexts リストに登録され、EState の解放時に自動的にクリーンアップされる不変条件を持つ。

アルゴリズムまたは振る舞い

1. estate->es_query_cxt にメモリコンテキストを切り替え、ExprContext ノードを確保する。 2. タプルポインタ（ecxt_scantuple 等）を NULL に初期化する。 3. AllocSetContextCreate を呼び出し、指定されたサイズで式評価用のメモリコンテキストを作成する。 4. EState からパラメータ情報（ecxt_param_exec_vals, ecxt_param_list_info）をコピーする。 5. 集約用データポインタやドメイン値の初期値を設定する。 6. estate->es_exprcontexts リストの先頭に作成したコンテキストを lcons で追加する（これにより、生成順の逆順でシャットダウンが保証される）。 7. 元のメモリコンテキストに戻し、作成した ExprContext を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execUtils.c 内の CreateExprContext、CreateWorkExprContext、CreateStandaloneExprContext から呼び出される。
  • **呼び出し先**:
  • MemoryContextSwitchTo
  • makeNode
  • AllocSetContextCreate
  • lcons

関連データ構造

  • ExprContext: 式評価の状態を保持する主要構造体。
  • EState: クエリ実行全体の状態を管理する構造体。

失敗モードと境界条件

  • メモリ不足時には AllocSetContextCreate がエラーを発生させる可能性がある。
  • estate が NULL の場合、estate->es_query_cxt へのアクセスによりクラッシュする。

保守担当者向けメモ

  • 本関数は static であり、execUtils.c 内の公開関数からのみ利用される。
  • es_exprcontexts への登録は、FreeExecutorState によるクリーンアップ漏れを防ぐために必須である。
