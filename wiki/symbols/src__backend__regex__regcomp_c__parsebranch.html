parsebranch

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regcomp.c:785

目的

正規表現の1つの「ブランチ（枝）」を解析します。|（選択演算子）で区切られた正規表現の構成要素を処理し、NFAの状態遷移とサブ式ツリー（subre）を構築します。

シグネチャまたは形状

static struct subre *parsebranch(struct vars *v, int stopper, int type, struct state *left, struct state *right, int partial);

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体。
  • stopper: 解析を終了させる文字（EOS または )）。
  • type: LACON（先読みサブ式）または PLAIN。
  • left, right: ブランチの開始および終了状態。
  • partial: このブランチが全体の一部であるかを示すフラグ。
  • **出力**:
  • 構築された subre 構造体へのポインタ。

前提条件と不変条件

  • NOERRN() マクロにより、エラー発生時には即座に NULL を返します。
  • v->nfa が有効である必要があります。
  • 空のブランチの場合、partial が偽であれば REG_UUNSPEC を通知し、left から right へ直接 EMPTYARC を追加します。

アルゴリズムまたは振る舞い

1. subre を演算子 =（仮の連結）で初期化します。 2. |、stopper、または EOS に到達するまでループし、parseqatom() を呼び出してアトムを順次解析します。 3. 2つ目以降のアトムが解析される際、暗黙の連結演算子として newstate() で新しい状態を作成し、moveins() で遷移を繋ぎます。 4. 内容が空のブランチであれば、left と right の間に空アークを追加します。

関連データ構造

  • struct vars: コンパイル全体のコンテキスト。
  • struct subre: 正規表現の構造を表現するツリーノード。
  • struct state: NFAの状態。

失敗モードと境界条件

  • stopper や EOS に遭遇すると解析を停止します。
  • seencontent フラグにより、空のブランチを検出し、必要に応じて警告（REG_UUNSPEC）を発行します。
  • エラー発生時は NOERRN() により呼び出し元へエラー状態が伝播します。

保守担当者向けメモ

  • 本関数は parseqatom() と密接に連携しており、再帰的な解析を行うため、スタックの深さや状態遷移の整合性に注意してください。
  • partial フラグの扱いは、複雑な正規表現構造（特に先読みやグループ化）において重要です。
