目的

PostgreSQL におけるインデックス（索引）や配列の添字、あるいは特定の識別子を表現するための汎用的な符号なし整数型を定義します。

シグネチャまたは形状

typedef unsigned int Index;

入力と出力

  • 入力: 符号なし整数値。
  • 出力: unsigned int 型のエイリアスとして機能します。

前提条件と不変条件

  • unsigned int 型のサイズおよび表現範囲に依存します。
  • 本シンボルは c.h で定義されており、PostgreSQL の全ソースファイルから利用可能です。

アルゴリズムまたは振る舞い

本シンボルは型定義であり、実行時のアルゴリズムは持ちません。コンパイル時に unsigned int と同等の型として扱われます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • contrib/amcheck/verify_common.c
  • contrib/amcheck/verify_gin.c
  • contrib/amcheck/verify_nbtree.c
  • 上記のソースコードにおいて、主にエラーメッセージの出力やデータ構造の記述に使用されています。
  • 呼び出し先: なし（型定義のため）。

関連データ構造

  • unsigned int: 本型の基底となる型。

失敗モードと境界条件

  • unsigned int の最大値を超える値を代入した場合、オーバーフローが発生します。
  • プラットフォームごとの unsigned int のビット幅（通常 32 ビット）に依存するため、移植性を考慮した設計が必要です。

保守担当者向けメモ

  • 本型は PostgreSQL 全体で広く使用される汎用型です。変更を行う際は、システム全体への影響範囲が広いため注意してください。
  • contrib/amcheck 等のモジュールで多用されていることから、データベースの内部構造（インデックスやタプル）を扱う際の標準的な型として定着しています。
