目的

Porterステミングアルゴリズムの第3段階として、特定の語尾（接尾辞）を正規化または削除する。

シグネチャまたは形状

static int r_Step_3(struct SN_env * z)

入力と出力

  • 入力: SN_env 構造体（現在のステミング環境、文字列データ、カーソル位置を含む）。
  • 出力: 成功時に 1、失敗または適用対象外の場合に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z->c（現在のカーソル位置）が有効な範囲内にあること。
  • r_R1 領域内での処理が必須条件となっている。

アルゴリズムまたは振る舞い

1. **後方検索**: 現在のカーソル位置から後方に向かって a_6 テーブル（9つのエントリ）を find_among_b で検索する。 2. **条件判定**: 検索対象の文字が特定のビットマスク条件を満たさない場合、即座に終了する。 3. **領域チェック**: 検索がヒットした場合、r_R1 を呼び出し、現在の位置が R1 領域内にあることを確認する。 4. **置換処理**: ヒットした among_var の値に応じて以下のいずれかを行う：

  • case 1-4: slice_from_s を使用して特定の文字列に置換する。
  • case 5: slice_del を使用して語尾を削除する。
  • case 6: r_R2 を呼び出し、R2 領域内であることを確認した上で slice_del を実行する。

呼び出し元と呼び出し先

  • 呼び出し元: stem_ISO_8859_1_english.c および stem_ISO_8859_1_porter.c 等のステミングメイン関数から呼び出される。
  • 呼び出し先: find_among_b, r_R1, r_R2, slice_from_s, slice_del。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_6: ステミングルールで使用される among 構造体のテーブル。

失敗モードと境界条件

  • find_among_b が一致を見つけられない場合は 0 を返す。
  • r_R1 または r_R2 のチェックに失敗した場合は、その戻り値（0または負の値）を伝播させる。
  • slice 操作中にメモリ不足等のエラーが発生した場合は負の値を返す。
