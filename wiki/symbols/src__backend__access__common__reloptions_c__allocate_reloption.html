allocate_reloption

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/reloptions.c:785

目的

リレーションオプション（ストレージパラメータ）のメタデータを保持する構造体 relopt_gen を動的に割り当て、初期化します。

シグネチャまたは形状

relopt_gen *allocate_reloption(bits32 kinds, int type, const char *name, const char *desc, LOCKMODE lockmode);

入力と出力

  • **入力**:
  • kinds: オプションの種類（RELOPT_KIND_LOCAL 等）。
  • type: オプションのデータ型（RELOPT_TYPE_BOOL, INT, REAL, ENUM, STRING）。
  • name: オプション名。
  • desc: オプションの説明文（NULL 可）。
  • lockmode: オプション変更時に必要なロックレベル。
  • **出力**: 初期化済みの relopt_gen 構造体へのポインタ。

前提条件と不変条件

  • type に指定された値がサポートされている型（RELOPT_TYPE_*）である必要があります。
  • kinds が RELOPT_KIND_LOCAL 以外の場合、メモリは TopMemoryContext に割り当てられます。

アルゴリズムまたは振る舞い

1. kinds が RELOPT_KIND_LOCAL でない場合、メモリコンテキストを TopMemoryContext に切り替えます。 2. type に基づいて必要なメモリサイズを決定します。サポート外の型の場合は elog(ERROR) を発生させます。 3. palloc でメモリを確保します。 4. name および desc を pstrdup でコピーし、構造体の各フィールド（kinds, namelen, type, lockmode）をセットします。 5. 切り替えたメモリコンテキストを元に戻し、作成した構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下の関数から呼び出されています。
  • add_bool_reloption
  • add_int_reloption
  • add_real_reloption
  • add_enum_reloption
  • add_string_reloption
  • **呼び出し先**: MemoryContextSwitchTo, palloc, pstrdup, strlen, elog。

関連データ構造

  • relopt_gen: オプションの基底構造体。
  • relopt_bool, relopt_int, relopt_real, relopt_enum, relopt_string: 各型に対応する派生構造体。

失敗モードと境界条件

  • 未サポートの type が渡された場合、elog(ERROR) により処理が中断されます。
  • メモリ不足時は palloc がエラーを発生させます。

保守担当者向けメモ

  • 新しいオプション型を追加する場合は、switch 文にケースを追加し、対応する構造体サイズを定義してください。
  • TopMemoryContext への切り替えは、オプション定義がプロセス生存期間中永続することを意図しています。
