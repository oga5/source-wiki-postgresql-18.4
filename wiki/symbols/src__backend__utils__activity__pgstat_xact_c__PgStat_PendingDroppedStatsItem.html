PgStat_PendingDroppedStatsItem

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/activity/pgstat_xact.c:21

目的

トランザクション内での統計情報の作成または削除操作を追跡し、トランザクションのコミットまたはアボート時に、それらの操作を物理的な統計システムへ反映させるための管理用構造体です。

シグネチャまたは形状

typedef struct PgStat_PendingDroppedStatsItem
{
	xl_xact_stats_item item;
	bool		is_create;
	dlist_node	node;
} PgStat_PendingDroppedStatsItem;

入力と出力

  • **入力**: xl_xact_stats_item 型の統計アイテム情報、および操作種別を示す bool フラグ。
  • **出力**: なし（構造体定義のため）。

前提条件と不変条件

  • TopTransactionContext 上に割り当てられる必要があります。
  • dlist_node を介して、トランザクションごとの統計変更スタックにリンクされます。

アルゴリズムまたは振る舞い

  • トランザクション内で統計オブジェクトの作成や削除が要求されると、本構造体がインスタンス化され、現在のトランザクション状態に関連付けられます。
  • トランザクション終了時（AtEOXact_PgStat 等）、is_create フラグに基づき、コミット時には「削除予定」の統計を物理削除し、アボート時には「作成予定」の統計を物理削除する処理の制御に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: pgstat_create_transactional 等のトランザクション統計管理APIが、本構造体を MemoryContextAlloc で生成します。
  • **呼び出し先**: dlist 操作関数（dclist_container 等）によって、統計変更スタックのイテレーション中に参照されます。

関連データ構造

  • xl_xact_stats_item: 統計アイテムの識別情報。
  • dlist_node: 統計変更リストを構成するための双方向リストノード。
  • TopTransactionContext: 本構造体のメモリ割り当てに使用されるメモリコンテキスト。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc がエラーを発生させます。
  • サブトランザクションのネスト構造において、スタックの整合性が崩れると、統計の適用順序や破棄対象に不整合が生じる可能性があります。

保守担当者向けメモ

  • 本構造体は統計情報の「作成」と「削除」をトランザクションの成否と逆の論理で処理するため、ロジック変更時は is_create フラグの判定条件を慎重に確認してください。
  • 2PC（PREPARE）時には AtEOXact_PgStat がスキップされるため、AtPrepare_PgStat を介した特殊なライフサイクル管理が行われている点に注意してください。
