inet_spg_picksplit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_spgist.c:164

目的

SP-GiSTインデックスのページ分割時、入力されたタプル群を共通プレフィックスに基づいて分類し、適切なノードへ振り分ける。

シグネチャまたは形状

Datum inet_spg_picksplit(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgPickSplitIn（入力タプル群）および spgPickSplitOut（出力構造）を受け取る。
  • **出力**: void。out 引数を通じて以下の値を設定する。
  • nodeLabels: NULL（ラベルは使用しない）。
  • mapTuplesToNodes: 各タプルが属するノード番号の配列。
  • leafTupleDatums: 各タプルのデータ表現。
  • nNodes: 分割後のノード数（2または4）。
  • hasPrefix: プレフィックスの有無。
  • prefixDatum: 共通プレフィックス（4ノード構成時のみ）。

前提条件と不変条件

  • 入力タプル群は inet 型または cidr 型であること。
  • 異なるアドレスファミリ（IPv4とIPv6）が混在する場合、ファミリによる2分割が優先される。
  • 同一ファミリ内の場合、共通ビット数に基づいた4分割が行われる。

アルゴリズムまたは振る舞い

2. **2ノード分割（ファミリ混在時）**:

  • nNodes を 2 に設定。
  • PGSQL_AF_INET か否かでノード 0 または 1 に振り分ける。

    • 3. **4ノード分割（ファミリ統一時）**:

  • nNodes を 4 に設定。
  • 算出した共通プレフィックスを prefixDatum に格納。
  • inet_spg_node_number を呼び出し、各タプルを 0〜3 のいずれかのノードに割り当てる。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのSP-GiSTインデックスアクセスメソッド（直接の呼び出し元情報はソースコード上では未定義）。
  • **呼び出し先**:
  • ip_bits, ip_family, ip_addr, bitncommon: ネットワークアドレスのビット演算用関数。
  • inet_spg_node_number: ノード割り当て判定用関数。
  • cidr_set_masklen_internal: プレフィックス長の設定用関数。
  • palloc: メモリ確保。

関連データ構造

  • spgPickSplitIn, spgPickSplitOut: SP-GiSTの分割処理用構造体。
  • inet: PostgreSQLのネットワーク型内部表現。

失敗モードと境界条件

  • 入力タプルが空の場合の挙動は本ソースコードからは不明。
  • 共通ビット数が 0 の場合、直ちにループを抜けて分割処理へ移行する。

保守担当者向けメモ

  • 本関数は network_spgist.c におけるインデックス構築の核心部分であり、ビット演算の正確性がインデックスの検索効率に直結する。
  • 4ノード構成時の inet_spg_node_number のロジック変更は、本関数の分割戦略と整合させる必要がある。
