pg_popcount_avx512

定義

  • 種別: function
  • 定義箇所: src/port/pg_popcount_avx512.c:110

目的

Intel AVX-512 命令セット（VPOPCNTDQ）を活用し、メモリバッファ内のビット数を高速にカウントする。

シグネチャまたは形状

uint64 pg_popcount_avx512(const char *buf, int bytes)

入力と出力

  • **入力**:
  • buf: カウント対象のデータが格納されたメモリバッファへのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • **出力**:
  • バッファ内の全ビットの合計数（uint64）。

前提条件と不変条件

  • 本関数は、実行環境がAVX-512命令セットをサポートしている場合にのみ呼び出されることを前提とする（実行時の判定は呼び出し元で行う）。
  • USE_AVX512_POPCNT_WITH_RUNTIME_CHECK マクロが定義された環境でコンパイルされている必要がある。
  • 内部で _mm512_load_si512 を使用するため、アライメント調整後のバッファアクセスは 64 バイト境界に準拠する。

アルゴリズムまたは振る舞い

1. **初期アライメント**: 入力バッファの開始位置を 64 バイト境界に合わせるため、最初のチャンクに対してマスク処理を行い、余分なバイトを無視してカウントする。 2. **メインループ**: 512 ビット（64 バイト）単位でデータをロードし、_mm512_popcnt_epi64 命令を用いてビット数を集計する。 3. **終了処理**: バッファの末尾が 64 バイトの倍数でない場合、残りのバイトに対してマスクを適用し、正確にビット数をカウントする。 4. **集計**: _mm512_reduce_add_epi64 を使用して、ZMM レジスタ内の累積値を最終的な 64 ビット整数に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/pg_bitutils.c 内の最適化関数選択ロジックから参照される。
  • **呼び出し先**: AVX-512 イントリンシック関数（_mm512_setzero_si512, _mm512_maskz_loadu_epi8, _mm512_popcnt_epi64, _mm512_add_epi64, _mm512_load_si512, _mm512_reduce_add_epi64）を直接呼び出す。

関連データ構造

  • __m512i: AVX-512 用の 512 ビットベクトル型。
  • __mmask64: 64 ビットのマスク型。

失敗モードと境界条件

  • バッファサイズが 0 の場合、または 64 バイト未満の場合でも、マスク処理により正しくビット数を算出する。
  • 実行環境が AVX-512 をサポートしていない環境で本関数が実行された場合の動作は未定義（呼び出し元でのガードが必須）。

保守担当者向けメモ

  • 本関数は pg_attribute_target を使用してコンパイル時に AVX-512 命令セットを有効化している。
  • 修正を行う際は、src/port/pg_bitutils.h の宣言と整合性を保つこと。
