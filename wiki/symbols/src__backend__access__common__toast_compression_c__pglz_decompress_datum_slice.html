pglz_decompress_datum_slice

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/toast_compression.c:108

目的

PGLZアルゴリズムで圧縮されたTOASTデータから、指定された長さ（スライス）のみを伸長して取得する。

シグネチャまたは形状

struct varlena *pglz_decompress_datum_slice(const struct varlena *value, int32 slicelength)

入力と出力

  • **入力**:
  • value: 圧縮されたデータを含む varlena 構造体へのポインタ。
  • slicelength: 伸長するデータの長さ（バイト単位）。
  • **出力**:
  • 伸長されたデータを含む varlena 構造体へのポインタ。

前提条件と不変条件

  • value は有効な圧縮済み varlena データであること。
  • slicelength は伸長後のデータサイズを超えないことが期待される。

アルゴリズムまたは振る舞い

1. slicelength に VARHDRSZ を加えたサイズでメモリを確保する。 2. pglz_decompress を呼び出し、value のヘッダー直後からデータを読み取り、確保したメモリ領域へ伸長を行う。 3. 伸長処理の結果（rawsize）が負の場合、データ破損とみなしてエラーを報告する。 4. 伸長後のデータサイズをヘッダーに設定し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/common/detoast.c:528
  • **呼び出し先**: pglz_decompress（common/pg_lzcompress.h で定義）

関連データ構造

  • struct varlena: PostgreSQLの可変長データ型。
  • VARHDRSZ: varlena ヘッダーサイズ。
  • VARHDRSZ_COMPRESSED: 圧縮済み varlena のヘッダーサイズ。

失敗モードと境界条件

  • pglz_decompress が負の値を返した場合、ERRCODE_DATA_CORRUPTED を伴うエラーが発生する。
  • メモリ確保失敗時は palloc の標準的な挙動（メモリ不足エラー）に従う。

保守担当者向けメモ

  • 本関数はPGLZ専用である。LZ4形式のスライス伸長には lz4_decompress_datum_slice が使用される。
  • 伸長後のサイズが slicelength と一致しない場合や、入力データが不正な場合の挙動については pglz_decompress の実装に依存する。
