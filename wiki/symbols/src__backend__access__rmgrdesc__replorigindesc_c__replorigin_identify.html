replorigin_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/replorigindesc.c:50

目的

論理レプリケーションのWALレコードに含まれる info ビットを解析し、対応する操作の種類（SETまたはDROP）を人間が判読可能な文字列として特定します。

シグネチャまたは形状

const char *replorigin_identify(uint8 info);

入力と出力

  • **入力**: uint8 info （WALレコードの info フィールド）
  • **出力**: const char * （識別された操作名 "SET" または "DROP"、該当しない場合は NULL）

前提条件と不変条件

  • 入力される info は、XLR_INFO_MASK 等のフラグビットが除去された状態である必要があります。
  • 戻り値の文字列は静的領域を指しており、呼び出し側で解放する必要はありません。

アルゴリズムまたは振る舞い

1. 引数 info を switch 文で評価します。 2. XLOG_REPLORIGIN_SET の場合は "SET" を返します。 3. XLOG_REPLORIGIN_DROP の場合は "DROP" を返します。 4. 上記いずれにも一致しない場合、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/access/rmgrlist.h において、RM_REPLORIGIN_ID リソースマネージャの記述子として登録されています。
  • **呼び出し先**: 本関数は制御フローの末端に位置し、他の関数を呼び出しません。

関連データ構造

  • XLOG_REPLORIGIN_SET, XLOG_REPLORIGIN_DROP: replication/origin.h で定義された定数。

失敗モードと境界条件

  • 未定義の info ビットが渡された場合、NULL を返します。呼び出し側はこの戻り値が NULL になる可能性を考慮する必要があります。

保守担当者向けメモ

  • 新しいレプリケーションオリジンのWAL操作を追加した場合は、必ず本関数の switch 文に新しいケースを追加してください。
