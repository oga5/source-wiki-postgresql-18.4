access_renamed

定義

  • 種別: typedef
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-type.c:42

目的

既存の型 access に対する別名（エイリアス）を定義し、ECPGプリプロセッサによる型変換およびコード生成の検証を行う。

シグネチャまたは形状

typedef access access_renamed;

入力と出力

  • 入力: access 型
  • 出力: access_renamed 型（access と同等の型として扱われる）

前提条件と不変条件

  • access 型が事前に定義されている必要がある。
  • 本シンボルはプリプロセッサによって生成されたテストコードの一部であり、C言語の型システムにおいて access と access_renamed は互換性を持つ。

アルゴリズムまたは振る舞い

  • typedef 文により、コンパイル時に access_renamed を access の別名として識別させる。

呼び出し元と呼び出し先

  • 呼び出し元: 不明（ソースコード上の使用箇所については提供された情報に記載なし）
  • 呼び出し先: なし

関連データ構造

  • access: 本シンボルの基底となる型。

失敗モードと境界条件

  • access 型が未定義の場合、コンパイルエラーとなる。

保守担当者向けメモ

  • 本ファイルはプリプロセッサの出力結果を検証するための回帰テスト用ソースコードであるため、直接編集しないこと。
  • EXEC SQL TYPE 指令に関連する型変換のテストケースとして使用されている。
