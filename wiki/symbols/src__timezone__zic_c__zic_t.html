zic_t

定義

  • 種別: typedef
  • 定義箇所: src/timezone/zic.c:25

目的

タイムゾーン定義の計算およびコンパイル処理において、時刻やオフセット値を保持するための汎用的な整数型として定義されています。

シグネチャまたは形状

typedef int64 zic_t;

入力と出力

  • **入力**: 64ビット符号付き整数値。
  • **出力**: 64ビット符号付き整数値。

前提条件と不変条件

  • int64 型として定義されており、プラットフォームに依存せず64ビットの精度で時刻計算を行うことを前提としています。

アルゴリズムまたは振る舞い

  • タイムゾーンの遷移時刻、オフセット値、およびルールに関連する期間の計算において、標準的な整数型として使用されます。
  • zic.c 内の計算ロジックにおいて、時刻演算や比較操作の基盤となります。

呼び出し元と呼び出し先

  • **呼び出し元**: zic.c 内の構造体定義（rule, zone）および関数（addtt, addtype, leapadd, gethms）がこの型を参照しています。
  • **呼び出し先**: 組み込みの int64 型に依存しています。

関連データ構造

  • struct rule: r_loyear, r_hiyear, r_tod, r_save, r_temp フィールドで使用。
  • struct zone: z_stdoff, z_save, z_untiltime フィールドで使用。

失敗モードと境界条件

  • 64ビット整数の範囲を超える値が代入された場合、オーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本型は PostgreSQL の移植性確保のため、IANA の zic ユーティリティのフォークにおいて int64 を明示的に使用するように調整されています。
  • 時刻演算関数（rpytime() や oadd() など）を扱う際は、この型が保持する値がエポックや閏年を考慮した計算結果であることを意識してください。
