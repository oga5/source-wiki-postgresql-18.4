ReadCommand

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/postgres.c:480

目的

クライアントからのコマンド入力を取得するための抽象化レイヤーとして機能します。接続先（リモートまたは対話型）に応じて適切な入力取得メソッドへ処理を委譲します。

シグネチャまたは形状

static int ReadCommand(StringInfo inBuf)

入力と出力

  • **入力**: inBuf (StringInfo) - 受信したコマンドデータを格納するためのバッファ。
  • **出力**: int - 読み取ったコマンドの種別またはステータスを示す整数値。

前提条件と不変条件

  • whereToSendOutput グローバル変数が適切に設定されている必要があります。
  • inBuf は呼び出し元によって適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. whereToSendOutput の値を確認します。 2. 値が DestRemote であれば SocketBackend(inBuf) を呼び出し、ソケット経由でコマンドを読み取ります。 3. それ以外の場合は InteractiveBackend(inBuf) を呼び出し、対話型入力からコマンドを読み取ります。 4. 呼び出した関数の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/postgres.c 内のメインループ（PostgresMain 等）から呼び出されます。
  • **呼び出し先**:
  • SocketBackend
  • InteractiveBackend

関連データ構造

  • StringInfo: PostgreSQL で可変長文字列を扱うためのデータ構造。

失敗モードと境界条件

  • 接続が切断された場合や入力エラーが発生した場合の挙動は、呼び出し先の SocketBackend または InteractiveBackend の実装に依存します。

保守担当者向けメモ

  • 本関数は単なるディスパッチャです。具体的な通信プロトコルの処理ロジックを変更する場合は、呼び出し先の SocketBackend または InteractiveBackend を参照してください。
