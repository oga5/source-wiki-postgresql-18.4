pg_hmac_ctx

定義

  • 種別: struct
  • 定義箇所: src/include/common/hmac.h:21

目的

RFC 2104 で規定される HMAC（Keyed-Hashing for Message Authentication）計算における状態を保持するための不透明な（opaque）構造体です。

シグネチャまたは形状

typedef struct pg_hmac_ctx pg_hmac_ctx;

※構造体の内部メンバは src/common/hmac.c に隠蔽されており、外部からは直接アクセスできません。

入力と出力

本構造体自体は、HMAC 計算のライフサイクル（作成、初期化、更新、終了、解放）を通じて状態を保持するコンテナとして機能します。

前提条件と不変条件

  • pg_hmac_create() によってメモリが確保され、初期化されている必要があります。
  • 内部の k_ipad や k_opad などの計算状態は、pg_hmac_init 以降の関数によって管理されます。
  • 呼び出し側は、本構造体の内部メンバに直接触れてはなりません。

アルゴリズムまたは振る舞い

1. **作成**: pg_hmac_create でコンテキストが確保されます。 2. **初期化**: pg_hmac_init で秘密鍵が処理されます（鍵がブロックサイズより長い場合はハッシュ化されます）。 3. **更新**: pg_hmac_update を通じてメッセージデータが順次入力されます。 4. **終了**: pg_hmac_final で内部的な ipad/opad を用いた計算が完了し、結果が書き出されます。 5. **解放**: pg_hmac_free でメモリが解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/auth-scram.c（SCRAM 認証プロセスにおいて利用）。
  • **呼び出し先**: src/common/hmac.c 内の各関数（pg_hmac_create, pg_hmac_init, pg_hmac_update, pg_hmac_final, pg_hmac_free）が本構造体を操作します。

関連データ構造

  • pg_cryptohash_type: src/include/common/cryptohash.h で定義されるハッシュアルゴリズムの型。

失敗モードと境界条件

  • メモリ割り当て失敗時（ALLOC 失敗時）の挙動は hmac.c の実装に依存します。
  • 計算中のエラー状態は pg_hmac_error() を通じて取得可能です。
  • explicit_bzero によるメモリのゼロクリアが pg_hmac_free 等で行われ、機密情報の漏洩を防ぐ設計となっています。

保守担当者向けメモ

  • 本構造体は不透明な型として定義されているため、実装を変更する際は src/common/hmac.c のみを修正し、ヘッダー側の定義を変更する必要はありません。
  • バックエンドでは palloc、フロントエンドでは malloc が使用されるよう hmac.c 内で抽象化されています。
