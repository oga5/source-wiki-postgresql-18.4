EnumTypeUncommitted

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_enum.c:689

目的

指定された OID を持つ列挙型が、現在のトランザクション内で作成され、まだコミットされていない状態であるかを判定する。

シグネチャまたは形状

static bool EnumTypeUncommitted(Oid typ_id)

入力と出力

  • **入力**: typ_id (Oid) - 検査対象の列挙型の OID。
  • **出力**: bool - 未コミットの列挙型テーブルに存在する場合は true、それ以外は false。

前提条件と不変条件

  • uncommitted_enum_types ハッシュテーブルが初期化されているか、あるいは NULL であることを前提とする。
  • トランザクション内で作成された列挙型は、この関数を通じて追跡されることで、インデックス破損等の不整合を防ぐ。

アルゴリズムまたは振る舞い

1. uncommitted_enum_types が NULL の場合、未コミットの型は存在しないと見なし false を返す。 2. uncommitted_enum_types が存在する場合、hash_search を用いて typ_id をキーとして検索を行う。 3. 検索結果（found）に基づき、テーブル内に存在すれば true を、存在しなければ false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_enum.c 内の列挙型操作に関連する関数（AddEnumLabel 等）から参照されている。
  • **呼び出し先**: hash_search（ハッシュテーブル検索関数）。

関連データ構造

  • uncommitted_enum_types: 未コミットの列挙型を追跡するために使用されるハッシュテーブル。

失敗モードと境界条件

  • uncommitted_enum_types が初期化されていない場合、安全に false を返す。
  • typ_id が無効な OID である場合、ハッシュテーブル内に見つからないため false を返す。

保守担当者向けメモ

  • 本関数はトランザクションの整合性維持に不可欠である。列挙型の追加や変更ロジックを修正する際は、この追跡メカニズムが正しく機能しているか確認すること。
