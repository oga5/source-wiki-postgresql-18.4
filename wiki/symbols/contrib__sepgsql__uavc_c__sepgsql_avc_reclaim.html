sepgsql_avc_reclaim

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:91

目的

ユーザー空間アクセスベクタキャッシュ（AVC）のメモリ使用量を削減し、キャッシュエントリ数が閾値を超えた際に古いエントリを解放する。

シグネチャまたは形状

void sepgsql_avc_reclaim(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてキャッシュエントリの削除およびメモリ解放を行う）

前提条件と不変条件

  • avc_num_caches が avc_threshold - AVC_NUM_RECLAIM を下回るまでループが継続される。
  • avc_lru_hint を用いてハッシュスロットを巡回し、LRU（Least Recently Used）に近い順序で解放候補を探索する。

アルゴリズムまたは振る舞い

1. キャッシュエントリ数が閾値に達している間、avc_lru_hint が指すスロットから順に処理を行う。 2. 各キャッシュエントリを走査し、hot_cache フラグを確認する。

  • hot_cache が false の場合: エントリをリストから削除し、scontext、tcontext、および存在する場合は ncontext を pfree で解放した後、エントリ自体を解放する。avc_num_caches をデクリメントする。
  • hot_cache が true の場合: hot_cache を false にリセットし、次回の解放候補とする。

    • 3. スロットの走査が終わるごとに avc_lru_hint をインクリメントし、循環させる。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/sepgsql/uavc.c 内のキャッシュ更新処理（sepgsql_avc_lookup 等の内部で呼び出される可能性あり）。
  • 呼び出し先: pfree（PostgreSQL メモリ管理関数）、foreach_delete_current。

関連データ構造

  • avc_cache: キャッシュエントリを保持する構造体。
  • avc_slots: キャッシュエントリを格納するハッシュスロットの配列。
  • avc_num_caches: 現在のキャッシュエントリ総数。
  • avc_threshold: キャッシュ解放を開始する閾値。
  • avc_lru_hint: 次に走査すべきハッシュスロットのインデックス。

失敗モードと境界条件

  • キャッシュが空の場合や、すべてのエントリが hot_cache である場合、ループが終了するまでスロットを巡回し続ける。
  • ncontext はオプションであるため、存在する場合のみ解放される。

保守担当者向けメモ

  • 本関数は avc_mem_cxt メモリコンテキスト内で動作することを前提としている。
  • hot_cache フラグは、最近アクセスされたエントリを即座に解放しないための保護機構として機能している。
