WaitEventCustomShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/wait_event.c:118

目的

カスタム待機イベントを管理するための共有メモリ領域（カウンタおよびハッシュテーブル）の初期化またはアタッチを行います。

シグネチャまたは形状

void WaitEventCustomShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として共有メモリ上のグローバル変数 WaitEventCustomCounter、WaitEventCustomHashByInfo、WaitEventCustomHashByName が初期化されます）

前提条件と不変条件

  • PostgreSQLの共有メモリ初期化フェーズで呼び出される必要があります。
  • 共有メモリ上のカウンタが未初期化の場合、nextId は WAIT_EVENT_CUSTOM_INITIAL_ID に設定され、スピンロックが初期化されます。

アルゴリズムまたは振る舞い

1. WaitEventCustomCounterData 構造体を共有メモリから取得（または作成）します。 2. 初回作成時のみ、nextId の初期化と mutex の SpinLockInit を実行します。 3. 待機イベントIDによる検索用ハッシュテーブル WaitEventCustomHashByInfo を作成またはアタッチします。 4. 待機イベント名による検索用ハッシュテーブル WaitEventCustomHashByName を作成またはアタッチします。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:345 (PostgreSQLの共有メモリ初期化プロセスの一部として呼び出されます)
  • 呼び出し先: ShmemInitStruct, SpinLockInit, ShmemInitHash

関連データ構造

  • WaitEventCustomCounterData: カスタムイベントIDの割り当てカウンタと排他制御用スピンロックを保持。
  • WaitEventCustomEntryByInfo: IDベースのハッシュテーブルエントリ。
  • WaitEventCustomEntryByName: 名前ベースのハッシュテーブルエントリ。

失敗モードと境界条件

  • 共有メモリの割り当てに失敗した場合、ShmemInitStruct や ShmemInitHash 内部でパニックが発生する可能性があります。
  • ハッシュテーブルのサイズは WAIT_EVENT_CUSTOM_HASH_INIT_SIZE から WAIT_EVENT_CUSTOM_HASH_MAX_SIZE (128) に制限されています。

保守担当者向けメモ

  • 本関数はバックエンド起動時に一度だけ実行されることを想定しています。
  • カスタム待機イベントの登録上限はハッシュテーブルの最大サイズに依存するため、拡張モジュールでの利用数には注意が必要です。
