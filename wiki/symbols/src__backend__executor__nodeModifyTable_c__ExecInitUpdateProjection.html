ExecInitUpdateProjection

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeModifyTable.c:709

目的

UPDATE 操作において、サブプランから供給されるタプルと既存のタプルを組み合わせて新しいタプルを構築するための投影（Projection）処理を初期化します。

シグネチャまたは形状

static void ExecInitUpdateProjection(ModifyTableState *mtstate, ResultRelInfo *resultRelInfo)

入力と出力

  • **入力**:
  • mtstate: ModifyTableState 構造体へのポインタ。実行状態とプラン情報を保持。
  • resultRelInfo: ResultRelInfo 構造体へのポインタ。ターゲットとなるリレーション情報。
  • **出力**: なし（副作用として resultRelInfo 内の投影状態を更新）。

前提条件と不変条件

  • mtstate->ps.plan が ModifyTable ノードとしてキャスト可能であること。
  • resultRelInfo が有効なリレーション記述子（ri_RelationDesc）を保持していること。
  • mtstate->mt_updateColnosLists が初期化済みであること。

アルゴリズムまたは振る舞い

1. mtstate->mt_lastResultIndex を用いて、現在のターゲットリレーションのインデックスを特定します。 2. mtstate->mt_updateColnosLists から、該当リレーションの更新対象列番号リストを取得します。 3. table_slot_create を呼び出し、古いタプル用および新しいタプル用のスロットを estate->es_tupleTable に作成し、resultRelInfo に格納します。 4. mtstate->ps.ps_ExprContext が未割り当ての場合、ExecAssignExprContext で作成します。 5. ExecBuildUpdateProjection を呼び出し、サブプランのターゲットリストと更新列リストを基に投影式を構築し、resultRelInfo->ri_projectNew に設定します。 6. resultRelInfo->ri_projectNewInfoValid を true に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeModifyTable.c 内の複数箇所（行 693, 2043, 2616, 4499 付近）から呼び出されています。
  • **呼び出し先**: table_slot_create, ExecAssignExprContext, ExecBuildUpdateProjection 等。

関連データ構造

  • ModifyTableState: 実行ノードの状態。
  • ResultRelInfo: ターゲットリレーションごとの実行時情報。
  • TupleDesc: タプルの構造定義。

失敗モードと境界条件

  • mtstate->mt_lastResultIndex が不正な場合、インデックスの再計算が行われますが、範囲外であれば Assert により停止します。
  • メモリ不足時には table_slot_create 等でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は UPDATE 操作の投影準備に特化しています。INSERT 用の投影初期化については ExecInitInsertProjection を参照してください。
  • ri_projectNew は、サブプランが生成したタプルに不足している値を古いタプルから補完して新しいタプルを生成するために使用されます。
