remove_target_file

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/file_ops.c:189

目的

ターゲットデータディレクトリ内の指定されたファイルを安全に削除します。

シグネチャまたは形状

void remove_target_file(const char *path, bool missing_ok)

入力と出力

  • path: 削除対象ファイルの相対パス。
  • missing_ok: true の場合、ファイルが存在しなくてもエラーとせず正常終了します。
  • 戻り値: なし（成功時）。失敗時は pg_fatal によりプロセスを終了します。

前提条件と不変条件

  • path は path_is_safe_for_extraction によってターゲットディレクトリ内に収まっていることが検証されなければなりません。
  • グローバル変数 datadir_target がターゲットディレクトリのパスとして設定されている必要があります。

アルゴリズムまたは振る舞い

1. path_is_safe_for_extraction(path) を呼び出し、パスが安全でない場合は pg_fatal で終了します。 2. グローバル変数 dry_run が true の場合、何もせず即座に復帰します。 3. datadir_target と path を結合し、絶対パス（またはターゲット基準のフルパス）を生成します。 4. unlink() を実行してファイルを削除します。 5. unlink() が失敗し、かつ errno が ENOENT（ファイル不在）で missing_ok が true の場合は、エラーとせず復帰します。 6. 上記以外で unlink() が失敗した場合は、pg_fatal でエラーメッセージを出力して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_rewind/file_ops.c (内部的なファイル操作処理)
  • src/bin/pg_rewind/libpq_source.c
  • **呼び出し先**:
  • path_is_safe_for_extraction
  • unlink (システムコール)
  • pg_fatal

関連データ構造

  • なし（文字列パスとフラグのみを使用）。

失敗モードと境界条件

  • **パスの不正**: ターゲットディレクトリ外を指すパスが渡された場合、pg_fatal で停止します。
  • **ファイル不在**: missing_ok が false の状態でファイルが存在しない場合、pg_fatal で停止します。
  • **権限不足・I/Oエラー**: unlink が失敗した場合、pg_fatal で停止します。

保守担当者向けメモ

  • 本関数は dry_run モードをサポートしています。新しいファイル操作関数を追加する際も、同様に dry_run チェックを実装してください。
  • path_is_safe_for_extraction によるセキュリティチェックは必須です。これをバイパスしないでください。
