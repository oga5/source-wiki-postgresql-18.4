r_standard_suffix

定義

  • **種別**: function
  • **定義箇所**: src/backend/snowball/libstemmer/stem_ISO_8859_1_french.c:665

目的

フランス語のステミング処理において、単語末尾の標準的な接尾辞を検索し、特定の領域（R1, R2, RV）条件に基づいて削除または置換を行う。

シグネチャまたは形状

static int r_standard_suffix(struct SN_env * z)

入力と出力

  • **入力**: struct SN_env * z （Snowballの実行環境状態）
  • **出力**: int （成功時に1以上、失敗または処理不要時に0、エラー時に負の値を返す）

前提条件と不変条件

  • z->c（現在のカーソル位置）が単語の末尾を指していること。
  • find_among_b を使用して、a_4 テーブル（43個のパターン）から後方マッチングを行う。
  • 領域判定関数（r_R1, r_R2, r_RV）が、接尾辞削除の可否を決定する不変条件として機能する。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置に設定し、find_among_b で接尾辞を検索する。 2. マッチした接尾辞の種類（among_var）に応じて switch 分岐を実行する。 3. 各ケースにおいて、以下の処理を行う：

  • r_R2 等の領域判定を行い、条件を満たせば slice_del（削除）または slice_from_s（置換）を実行する。
  • ケース6のように、接尾辞削除後にさらに別の接尾辞（a_2 テーブル）を再帰的に検索・処理する入れ子構造を持つ。
  • ケース7のように、特定の文字コード条件（z->p[z->c - 1]）を検証した上で、a_3 テーブルを用いた追加の検索を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_ISO_8859_1_french.c 内のメインステミング関数から呼び出される（直接の呼び出し元関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: find_among_b, r_R1, r_R2, r_RV, slice_del, slice_from_s, eq_s_b。
  • **注記**: 他言語のソースファイル（catalan.c, dutch.c 等）にも同名のシンボルが存在するが、これらは各言語固有の実装であり、本関数とは別個の定義である。

関連データ構造

  • struct SN_env: ステミング環境を保持する構造体。
  • a_2, a_3, a_4: 接尾辞検索用の among 構造体配列。

失敗モードと境界条件

  • find_among_b がマッチしない場合、即座に 0 を返して終了する。
  • 領域判定（r_R1, r_R2 等）が失敗した場合、または slice 操作が失敗した場合は、その戻り値をそのまま伝播させる。
  • ケース6や7において、内部の goto ラベル（lab0 ～ lab8）を用いたバックトラック処理が行われ、条件不一致時にカーソル位置を復元する。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接編集せず、対応する .sbl ファイルを修正すること。
  • among_var の各ケースは、フランス語の文法規則に基づく複雑な条件分岐を含んでいるため、デバッグ時は find_among_b の戻り値と switch 文の対応を厳密に追跡すること。
