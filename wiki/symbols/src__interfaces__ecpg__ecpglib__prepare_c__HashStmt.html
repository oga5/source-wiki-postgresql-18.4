HashStmt

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/prepare.c:384

目的

ECPGの自動プリペア機能において、SQLクエリ文字列をハッシュ化し、ステートメントキャッシュ内の格納位置（インデックス）を算出するために使用されます。

シグネチャまたは形状

static int HashStmt(const char *ecpgQuery)

入力と出力

  • **入力**: ecpgQuery (キャッシュ対象のSQLクエリ文字列)
  • **出力**: int 型のキャッシュインデックス。stmtCacheNBuckets と stmtCacheEntPerBucket に基づいて算出されます。

前提条件と不変条件

  • クエリ文字列の先頭最大50文字を使用してハッシュ値を計算します。
  • 戻り値は必ず 1 以上となります（配列のインデックス0を使用しないためのオフセット処理）。

アルゴリズムまたは振る舞い

1. クエリ文字列の長さ（最大50文字）を決定します。 2. 各文字の値を加算し、32ビットハッシュ値を13ビット左回転させる処理を繰り返します。 3. 得られた hashVal を stmtCacheNBuckets で剰余演算し、バケット番号を特定します。 4. bucketNo * stmtCacheEntPerBucket + 1 を計算し、キャッシュエントリの開始位置を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/ecpglib/prepare.c 内のキャッシュ検索および管理ロジックから呼び出されます。
  • **呼び出し先**:
  • C標準ライブラリの strlen を使用します。

関連データ構造

  • stmtCacheNBuckets: キャッシュのバケット数。
  • stmtCacheEntPerBucket: 各バケット内のエントリ数。

失敗モードと境界条件

  • クエリ文字列が空の場合、ハッシュ値は0となり、インデックスは1を返します。
  • ハッシュ衝突が発生した場合、本関数は同一のインデックスを返します（衝突解決は呼び出し側のキャッシュ管理ロジックに依存します）。

保守担当者向けメモ

  • 本関数はハッシュ計算のみを担当します。キャッシュの置換アルゴリズムや衝突時の挙動については、呼び出し元のキャッシュ管理関数を参照してください。
  • ハッシュアルゴリズムを変更する場合は、既存のキャッシュエントリとの整合性に注意してください。
