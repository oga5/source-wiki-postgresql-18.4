sepgsql_xact_callback

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/label.c:164

目的

トランザクションのコミットまたはアボートイベントに応じ、保留中のクライアントセキュリティラベル（client_label_pending）を確定または破棄し、セッションのラベル状態を整合させる。

シグネチャまたは形状

static void sepgsql_xact_callback(XactEvent event, void *arg)

入力と出力

  • event: XactEvent 型。発生したトランザクションイベント（XACT_EVENT_COMMIT または XACT_EVENT_ABORT）。
  • arg: void * 型。未使用。

前提条件と不変条件

  • client_label_pending は CurTransactionContext 上で管理されているため、トランザクション終了時にメモリが暗黙的に解放されることを前提としている。
  • コミット時には、client_label_pending の末尾要素を TopMemoryContext にコピーし、永続的な client_label_committed として保持する。

アルゴリズムまたは振る舞い

1. event が XACT_EVENT_COMMIT の場合:

  • client_label_pending が空でない場合、リストの末尾（llast）にあるラベル情報を取得する。
  • ラベルが存在すれば TopMemoryContext に複製し、既存の client_label_committed があれば解放した上で更新する。
  • client_label_pending を NIL にリセットする。

    • 2. event が XACT_EVENT_ABORT の場合:

  • client_label_pending を NIL にリセットし、保留中の変更を破棄する。

呼び出し元と呼び出し先

  • 呼び出し元: RegisterXactCallback(sepgsql_xact_callback, NULL); により、トランザクションシステムからコールバックとして登録される（contrib/sepgsql/label.c:433）。
  • 呼び出し先: MemoryContextStrdup, pfree, llast（PostgreSQL 内部 API）。

関連データ構造

  • pending_label: 保留中のラベル情報を保持する構造体。
  • client_label_pending: 保留中のラベルリスト。
  • client_label_committed: コミット済みのラベルを保持するグローバル変数。

失敗モードと境界条件

  • client_label_pending が空の場合、コミット処理は何も行わない。
  • client_label_committed が既に設定されている場合、pfree で解放してから新しいラベルで上書きする。

保守担当者向けメモ

  • 本関数は CurTransactionContext のメモリ管理に依存している。pending_label の要素がトランザクション終了時に自動解放されることを利用しているため、メモリリークの懸念は低いが、TopMemoryContext へのコピー処理は必須である。
