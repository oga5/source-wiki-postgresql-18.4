SHA256_Transform

定義

  • 種別: function
  • 定義箇所: src/common/sha2.c:314

目的

SHA-256 アルゴリズムにおけるメッセージブロック単位の圧縮処理を実行し、ハッシュコンテキストの中間状態を更新します。

シグネチャまたは形状

static void SHA256_Transform(pg_sha256_ctx *context, const uint8 *data)

入力と出力

  • **入力**:
  • context: pg_sha256_ctx 構造体へのポインタ。現在のハッシュ状態（state）と入力バッファを保持します。
  • data: 処理対象となる 64 バイトのメッセージブロック。
  • **出力**: なし（副作用として context->state を更新します）。

前提条件と不変条件

  • context は初期化済みである必要があります。
  • data は 64 バイトのデータブロックを指している必要があります。
  • 内部レジスタ a から h は、処理開始時に context->state から読み込まれ、処理終了時に context->state へ加算されます。

アルゴリズムまたは振る舞い

1. context->state の値を 8 つの内部レジスタ（a〜h）にロードします。 2. 64 バイトの入力データを 32 ビットワードの配列として扱います。 3. 64 ラウンドの圧縮処理を実行します。

  • 前半 16 ラウンド（0〜15）は ROUND256_0_TO_15 マクロを使用して実行されます。
  • 残りの 48 ラウンド（16〜63）は ROUND256 マクロを使用して実行されます。

    • 4. 最終的なレジスタ値を context->state に加算して中間ハッシュ値を更新します。 5. セキュリティ上の配慮から、使用したレジスタ変数をゼロクリアします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/sha2.c 内の pg_sha256_update および pg_sha256_final から呼び出されます。
  • **呼び出し先**: ROUND256_0_TO_15 および ROUND256 マクロを呼び出します。

関連データ構造

  • pg_sha256_ctx: sha2_int.h で定義される SHA-256 用のコンテキスト構造体。

失敗モードと境界条件

  • 本関数自体はエラーコードを返しません。
  • 入力データが 64 バイト未満の場合の挙動は定義されていません（呼び出し元でブロック単位のパディングが保証される必要があります）。

保守担当者向けメモ
