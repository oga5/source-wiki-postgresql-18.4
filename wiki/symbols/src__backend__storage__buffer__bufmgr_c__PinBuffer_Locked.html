PinBuffer_Locked

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/bufmgr.c:3177

目的

バッファヘッダーが既にロックされている状態で、指定されたバッファのピン留め（Pin）を行い、参照カウントを更新する。

シグネチャまたは形状

static void PinBuffer_Locked(BufferDesc *buf)

入力と出力

  • **入力**: buf (対象バッファの記述子 BufferDesc*)
  • **出力**: なし（副作用としてバッファ状態の更新とリソース所有権の登録を行う）

前提条件と不変条件

  • 呼び出し時点で、対象バッファのヘッダーがロックされていること（BM_LOCKED フラグが立っていること）。
  • 呼び出し元において、当該バッファに対する既存のピンが存在しないこと（GetPrivateRefCountEntry が NULL を返すこと）。

アルゴリズムまたは振る舞い

1. **検証**: Assert により、既存のピンが存在しないことを確認する。 2. **メモリ定義**: VALGRIND_MAKE_MEM_DEFINED を呼び出し、バッファブロックを Valgrind に対して定義済みとしてマークする。 3. **状態更新**: バッファの state から BM_LOCKED を確認し、BUF_REFCOUNT_ONE を加算して UnlockBufHdr を呼び出し、ロックを解放する。 4. **参照管理**: NewPrivateRefCountEntry を使用してプライベート参照カウントエントリを新規作成し、カウントをインクリメントする。 5. **リソース追跡**: ResourceOwnerRememberBuffer を呼び出し、現在のリソースオーナーにバッファを登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/bufmgr.c 内の各所（行 516, 738, 2372, 3963, 5005, 5102, 5330, 6603）。
  • **呼び出し先**:
  • GetPrivateRefCountEntry
  • BufHdrGetBlock
  • pg_atomic_read_u32
  • UnlockBufHdr
  • BufferDescriptorGetBuffer
  • NewPrivateRefCountEntry
  • ResourceOwnerRememberBuffer

関連データ構造

  • BufferDesc: バッファの状態を保持する記述子。
  • PrivateRefCountEntry: バックエンドごとのピン留め回数を追跡する構造体。

失敗モードと境界条件

  • 前提条件（ロックの保持、既存ピンの不在）が満たされない場合、Assert により異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は、呼び出し元が既にバッファヘッダーのロックを保持していることを前提とする特殊なピン留め処理である。
  • ロックの解放（UnlockBufHdr）は本関数内で行われるため、呼び出し元で二重にロックを解放しないよう注意が必要である。
