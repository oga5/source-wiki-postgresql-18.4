PLy_push_execution_context

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_main.c:391

目的

PL/Python の実行コンテキストスタックに新しい PLyExecutionContext を追加し、ネストされた Python 呼び出しの追跡とメモリ管理を可能にする。

シグネチャまたは形状

PLyExecutionContext *PLy_push_execution_context(bool atomic_context)

入力と出力

  • **入力**:
  • atomic_context (bool): true の場合、TopTransactionContext をメモリ割り当てに使用する。false の場合、PortalContext を使用する。
  • **出力**:
  • 新規作成された PLyExecutionContext 構造体へのポインタ。

前提条件と不変条件

  • PLy_execution_contexts グローバル変数により、実行コンテキストのスタックが維持される。
  • 割り当てられたコンテキストの curr_proc および scratch_ctx は初期値として NULL に設定される。

アルゴリズムまたは振る舞い

1. atomic_context の値に基づき、TopTransactionContext または PortalContext から PLyExecutionContext サイズのメモリを割り当てる。 2. 割り当てた構造体の curr_proc と scratch_ctx を NULL に初期化する。 3. 現在のスタック先頭（PLy_execution_contexts）を新しいコンテキストの next ポインタに設定する。 4. PLy_execution_contexts を新しいコンテキストで更新し、スタックの先頭として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/pl/plpython/plpy_main.c:215 (ハンドラ実行時)
  • src/pl/plpython/plpy_main.c:303 (DO ブロック実行時)
  • **呼び出し先**:
  • MemoryContextAlloc (PostgreSQL メモリ管理関数)

関連データ構造

  • PLyExecutionContext: 実行中の手続きやメモリコンテキストを保持するスタック構造体。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAlloc がエラーを発生させる。
  • atomic_context の選択が不適切な場合、メモリコンテキストの生存期間と実行のライフサイクルが一致せず、メモリリークや不正アクセスを引き起こす可能性がある。

保守担当者向けメモ

  • scratch_ctx は遅延生成されるため、この関数で割り当てられたコンテキストは、対応する PLy_pop_execution_context を呼び出して適切に解放する必要がある。
