FilterWalSummaries

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummary.c:99

目的

WAL要約ファイルのリストから、指定されたタイムラインおよびLSN範囲に合致するファイルのみを抽出します。

シグネチャまたは形状

List *FilterWalSummaries(List *wslist, TimeLineID tli, XLogRecPtr start_lsn, XLogRecPtr end_lsn)

入力と出力

  • **入力**:
  • wslist: WalSummaryFile 構造体のリスト。
  • tli: フィルタリング対象のタイムラインID（0の場合はタイムラインによるフィルタリングを行わない）。
  • start_lsn: フィルタリングの開始LSN（InvalidXLogRecPtr の場合は無視）。
  • end_lsn: フィルタリングの終了LSN（InvalidXLogRecPtr の場合は無視）。
  • **出力**:
  • 条件に合致した WalSummaryFile 構造体を含む新しい List。

前提条件と不変条件

  • 入力リスト wslist は WalSummaryFile 構造体のポインタを保持している必要があります。
  • tli が 0 でない場合、ws->tli が一致するファイルのみが選択されます。

アルゴリズムまたは振る舞い

1. 入力された wslist を順次走査します。 2. 各 WalSummaryFile に対して以下の条件判定を行います：

  • tli が指定されており、かつ ws->tli と一致しない場合は除外。
  • start_lsn が有効で、かつ start_lsn > ws->end_lsn の場合は除外。
  • end_lsn が有効で、かつ end_lsn < ws->start_lsn の場合は除外。

    • 3. 上記の除外条件に該当しない要素を、新しいリスト result に追加して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c:510
  • **呼び出し先**:
  • 本関数はリスト操作関数（lfirst, lappend）および XLogRecPtr 比較マクロを使用します。

関連データ構造

  • WalSummaryFile: WAL要約ファイルのメタデータ（tli, start_lsn, end_lsn 等）を保持する構造体。

失敗モードと境界条件

  • wslist が空の場合、空のリストを返します。
  • start_lsn や end_lsn に InvalidXLogRecPtr を指定した場合、その軸でのフィルタリングは無効化されます。
  • タイムラインIDに 0 を指定した場合、タイムラインによる絞り込みは行われません。

保守担当者向けメモ

  • 本関数はタイムラインの整合性チェックを行いません。呼び出し側で適切なタイムラインを指定する必要があります。
  • 戻り値のリストは新しいリストとして生成されるため、呼び出し側で不要になった際に適切に解放してください。
