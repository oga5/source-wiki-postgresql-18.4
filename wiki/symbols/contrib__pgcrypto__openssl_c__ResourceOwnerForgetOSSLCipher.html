ResourceOwnerForgetOSSLCipher

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/openssl.c:285

目的

OSSLCipher コンテキストを ResourceOwner の管理対象から除外する。これにより、トランザクション終了時やエラー発生時に、当該リソースが自動的に解放されることを防ぐ。

シグネチャまたは形状

static inline void ResourceOwnerForgetOSSLCipher(ResourceOwner owner, OSSLCipher *od)

入力と出力

  • owner: リソースを管理している ResourceOwner オブジェクト。
  • od: 管理対象から除外する OSSLCipher 構造体へのポインタ。
  • 戻り値: なし。

前提条件と不変条件

  • od は、事前に ResourceOwnerRememberOSSLCipher を通じて owner に登録されている必要がある。
  • osslcipher_resowner_desc がリソースの記述子として定義されていること。

アルゴリズムまたは振る舞い

1. PointerGetDatum(od) を使用して OSSLCipher ポインタを Datum 型に変換する。 2. ResourceOwnerForget 関数を呼び出し、指定された owner から当該リソースの追跡を解除する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/pgcrypto/openssl.c 内の暗号化コンテキストの破棄処理において呼び出される（例: od->owner を引数として使用）。
  • 呼び出し先:
  • ResourceOwnerForget (PostgreSQL コアのリソース管理関数)

関連データ構造

  • OSSLCipher: 暗号化/復号用のコンテキストを保持する構造体。
  • ResourceOwner: PostgreSQL のリソース管理機構。
  • osslcipher_resowner_desc: ResourceOwner がリソースを識別・管理するための記述子。

失敗モードと境界条件

  • owner が NULL の場合、ResourceOwnerForget の内部実装に依存する（通常はエラーとなる）。
  • od が既に登録解除されている場合や、無効なポインタである場合の挙動は未定義。

保守担当者向けメモ

  • 本関数は static inline で定義されており、pgcrypto モジュール内でのリソース管理の整合性を保つために使用される。
  • ResourceOwner への登録・解除のペアが正しく維持されていることを確認すること。特にエラー発生時のメモリリークを防ぐため、ResourceOwnerRememberOSSLCipher との対で使用される必要がある。
