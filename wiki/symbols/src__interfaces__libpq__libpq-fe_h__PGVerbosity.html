PGVerbosity

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-fe.h:154

目的

libpq が生成するエラーメッセージの冗長性（詳細度）を制御するための列挙型です。

シグネチャまたは形状

typedef enum
{
    PQERRORS_TERSE,    /* 単一行のエラーメッセージ */
    PQERRORS_DEFAULT,  /* 推奨されるスタイル */
    PQERRORS_VERBOSE,  /* すべての情報を出力 */
    PQERRORS_SQLSTATE  /* エラーの深刻度と SQLSTATE コードのみ */
} PGVerbosity;

入力と出力

  • 入力: PQsetErrorVerbosity 関数等の引数として、エラー出力レベルを指定するために使用されます。
  • 出力: 該当なし（列挙型定義のため）。

前提条件と不変条件

  • libpq のエラー報告設定として使用されます。
  • 列挙値は PQERRORS_TERSE から PQERRORS_SQLSTATE まで定義されており、これらはエラーメッセージのフォーマットを決定します。

アルゴリズムまたは振る舞い

  • PQERRORS_TERSE: メッセージを単一行に制限します。
  • PQERRORS_DEFAULT: 標準的なエラー報告を行います。
  • PQERRORS_VERBOSE: エラーに関する詳細な情報をすべて出力します。
  • PQERRORS_SQLSTATE: エラーの深刻度と SQLSTATE コードのみに情報を絞ります。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/bin/psql/settings.h: エラー冗長性レベルの保持に使用。
  • src/interfaces/libpq/fe-connect.c, fe-exec.c, fe-protocol3.c: エラー処理ロジック内で使用。
  • src/interfaces/libpq/libpq-int.h: 内部的なエラーメッセージ冗長性の保持に使用。
  • 呼び出し先: 該当なし（列挙型定義のため）。

関連データ構造

  • PGconn: データベース接続状態を管理する構造体。PQsetErrorVerbosity を通じてこの設定が適用されます。

失敗モードと境界条件

  • 定義外の値を設定した場合の挙動は未定義です。
  • 接続が確立されていない状態での設定変更がどのように反映されるかは、各実装関数（PQsetErrorVerbosity 等）の仕様に依存します。

保守担当者向けメモ

  • 本シンボルは libpq の公開 API の一部です。列挙値の追加は可能ですが、既存の順序や値の変更は後方互換性を損なう可能性があるため避けてください。
