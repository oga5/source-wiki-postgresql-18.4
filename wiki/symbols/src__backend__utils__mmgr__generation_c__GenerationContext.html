GenerationContext

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/mmgr/generation.c:59

目的

PostgreSQL のメモリ管理における「世代別（Generational）」メモリコンテキストを保持するための構造体です。寿命が近いオブジェクトをグループ化して管理し、FIFO 的な割り当て・解放パターンにおいてメモリの断片化を抑制しつつ効率的な再利用を実現します。

シグネチャまたは形状

typedef struct GenerationContext
{
    MemoryContextData header;
    uint32      initBlockSize;
    uint32      maxBlockSize;
    uint32      nextBlockSize;
    uint32      allocChunkLimit;
    GenerationBlock *block;
    GenerationBlock *freeblock;
    dlist_head  blocks;
} GenerationContext;

入力と出力

本構造体はメモリコンテキストの管理状態を保持するコンテナであり、関数ではありません。メモリ割り当て要求（palloc 等）のコンテキストとして使用されます。

前提条件と不変条件

  • header フィールドにより MemoryContext インターフェースと互換性を持ちます。
  • blocks は dlist_head を用いて管理される双方向リストであり、コンテキスト内の全メモリブロックを追跡します。
  • freeblock は、全チャンクが解放された空のブロックを再利用目的で保持するか、該当がない場合は NULL となります。

アルゴリズムまたは振る舞い

  • **ブロック管理**: 割り当て要求に対し、現在の block に空きがあれば使用し、なければ freeblock の再利用または新規ブロックの確保を行います。
  • **リセット**: GenerationReset 実行時、KeeperBlock 以外の全ブロックを走査し、メモリを解放または初期化します。
  • **解放**: GenerationFree によりチャンクが解放され、ブロック内の全チャンクが空になった場合、そのブロックを freeblock として保持するか、free() を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/reorderbuffer.c 等で、可変長トランザクションデータの管理用として参照されています。
  • **呼び出し先**: GenerationBlock 構造体（ブロック管理用）を内部で参照します。

関連データ構造

  • MemoryContextData: 基底となるメモリコンテキスト情報。
  • GenerationBlock: メモリ割り当ての物理的な単位。
  • dlist_head: ブロックリスト管理用の双方向リストヘッダー。

失敗モードと境界条件

  • freeblock が NULL の場合、新規ブロックの確保が必要です。
  • KeeperBlock（コンテキスト作成時に確保される最初のブロック）は、コンテキストが削除されるまで生存し、通常のブロック解放ロジックの対象外となります。

保守担当者向けメモ

  • 本コンテキストは、割り当て順と解放順が近い場合に最適化されています。ランダムな解放が頻発する用途には適していません。
  • Valgrind 連携マクロが多用されているため、メモリの有効性チェックを行う際は、これらのマクロによる状態遷移に注意してください。
