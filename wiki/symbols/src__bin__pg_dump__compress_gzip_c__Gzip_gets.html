Gzip_gets

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:331

目的

GZIP 圧縮されたファイルストリームから、指定されたサイズまでの文字列を読み込みます。

シグネチャまたは形状

char *Gzip_gets(char *ptr, int size, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • ptr: 読み込んだデータを格納するバッファへのポインタ。
  • size: 読み込む最大バイト数。
  • CFH: 圧縮ファイルハンドル構造体。
  • **出力**: 成功した場合は ptr を返し、エラーまたは EOF の場合は NULL を返します。

前提条件と不変条件

  • CFH->private_data に有効な gzFile ハンドルが格納されている必要があります。

アルゴリズムまたは振る舞い

1. CFH->private_data から gzFile ハンドルを取得します。 2. zlib の gzgets 関数を呼び出し、圧縮ストリームからデータを解凍して ptr に格納します。 3. gzgets の戻り値をそのまま呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**: zlib ライブラリの gzgets 関数。
  • **呼び出し元**: ソースコード内の直接的な呼び出し元は確認されていませんが、CompressFileHandle 構造体の gets_func フィールドに本関数が割り当てられています（src/bin/pg_dump/compress_gzip.c:426）。

関連データ構造

  • CompressFileHandle: 圧縮 I/O の抽象化に使用される構造体。
  • gzFile: zlib が提供する GZIP ファイル操作用ハンドル。

失敗モードと境界条件

  • zlib の gzgets が失敗した場合や EOF に到達した場合は NULL が返されます。
  • 詳細は zlib の gzgets の仕様に依存します。

保守担当者向けメモ

  • 本関数は CompressFileHandle インターフェースの一部として実装されています。
  • gzgetc マクロはライブラリ間の不一致を避けるため #undef されていますが、本関数は gzgets を直接利用しています。
