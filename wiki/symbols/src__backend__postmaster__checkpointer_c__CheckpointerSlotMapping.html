CheckpointerSlotMapping

定義

  • 種別: struct
  • 定義箇所: src/backend/postmaster/checkpointer.c:1222

目的

チェックポインタープロセスにおいて、ファイル同期要求（CheckpointerRequest）と、それに対応するバッファ管理上のスロット番号を関連付けるためのデータ構造です。

シグネチャまたは形状

struct CheckpointerSlotMapping
{
    CheckpointerRequest request;
    int                 slot;
};

入力と出力

  • 入力: CheckpointerRequest 型の要求データおよび int 型のスロット識別子。
  • 出力: なし（データ保持用構造体）。

前提条件と不変条件

  • CheckpointerRequest が有効な同期要求を保持していること。
  • slot はバッファ管理システムにおける有効なインデックスを指していること。

アルゴリズムまたは振る舞い

本構造体は、ハッシュテーブル等のデータ構造の要素として利用されます。チェックポインターがバックエンドからの fsync 要求を処理する際、要求内容と対象バッファスロットをペアとして管理するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照）:
  • src/backend/postmaster/checkpointer.c:1247: ハッシュテーブルの entrysize 設定時に sizeof(struct CheckpointerSlotMapping) として参照。
  • src/backend/postmaster/checkpointer.c:1270: ハッシュテーブル操作用のポインタ変数 slotmap の型として参照。
  • 呼び出し先: なし。

関連データ構造

  • CheckpointerRequest: 同期要求の内容を保持する構造体。
  • ハッシュテーブル（ctl.entrysize で指定される構造体）: 本構造体をエントリとして格納する管理テーブル。

失敗モードと境界条件

  • メモリ不足: ハッシュテーブルの初期化やエントリ追加時にメモリ確保が失敗する可能性があります。
  • 無効なスロット: slot に不正な値が設定された場合、後続のバッファ同期処理で不整合が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は checkpointer.c 内のハッシュテーブル管理に特化しています。ハッシュテーブルのサイズやエントリ構造を変更する場合は、entrysize の定義箇所（1247行目）と整合性を取る必要があります。
