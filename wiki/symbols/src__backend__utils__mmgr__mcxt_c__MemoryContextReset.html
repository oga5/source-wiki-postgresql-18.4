MemoryContextReset

定義

  • src/backend/utils/mmgr/mcxt.c:388

目的

指定されたメモリコンテキスト内の全メモリ割り当てを解放し、コンテキストを初期状態に戻す。

シグネチャまたは形状

void MemoryContextReset(MemoryContext context)

入力と出力

  • **入力**: context (操作対象の MemoryContext 構造体へのポインタ)
  • **出力**: なし（副作用としてメモリが解放される）

前提条件と不変条件

  • context は MemoryContextIsValid マクロにより有効性が検証されている必要がある。

アルゴリズムまたは振る舞い

1. **子コンテキストの処理**: context->firstchild が存在する場合、MemoryContextDeleteChildren(context) を呼び出し、すべての子コンテキストを削除する。 2. **メモリのリセット**: context->isReset が偽（未リセット状態）である場合、MemoryContextResetOnly(context) を呼び出し、当該コンテキスト内のメモリ割り当てを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • contrib/bloom/blinsert.c
  • contrib/dblink/dblink.c
  • contrib/file_fdw/file_fdw.c
  • contrib/pg_walinspect/pg_walinspect.c
  • contrib/postgres_fdw/postgres_fdw.c
  • contrib/sepgsql/uavc.c
  • **呼び出し先**:
  • MemoryContextDeleteChildren (条件付き)
  • MemoryContextResetOnly (条件付き)

関連データ構造

  • MemoryContext: メモリコンテキストの階層構造を保持する構造体。firstchild フィールドおよび isReset フラグを参照する。

失敗モードと境界条件

  • 無効な context ポインタが渡された場合、Assert により異常終了する。

保守担当者向けメモ

  • 本関数は、子コンテキストが存在しない場合や、既にリセット済みの場合は不要な関数呼び出しを回避する最適化が行われている。
  • MemoryContextResetOnly は、コンテキスト自体は保持したまま、その内部のメモリ領域のみを解放する。
