sql_exec_dumpalldbs

定義

  • 種別: function
  • 定義箇所: contrib/oid2name/oid2name.c:451

目的

PostgreSQL データベース内の全データベース名、対応する OID、およびデフォルトのテーブルスペース名をシステムカタログから取得し、一覧表示する。

シグネチャまたは形状

void sql_exec_dumpalldbs(PGconn *conn, struct options *opts)

入力と出力

  • **入力**:
  • conn: データベース接続を保持する PGconn ポインタ。
  • opts: 実行時オプションを保持する struct options ポインタ（quiet フラグを参照）。
  • **出力**: なし（標準出力への結果表示のみ）。

前提条件と不変条件

  • conn は有効なデータベース接続状態であること。
  • pg_catalog.pg_database および pg_catalog.pg_tablespace へのアクセス権限が必要。

アルゴリズムまたは振る舞い

1. pg_catalog.pg_database と pg_catalog.pg_tablespace を dattablespace = t.oid で結合する SQL クエリを構築する。 2. 取得対象は d.oid（OID）、datname（データベース名）、spcname（テーブルスペース名）であり、データベース名で昇順ソートされる。 3. 構築したクエリ文字列を sql_exec 関数に渡し、結果を標準出力へ整形表示する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/oid2name/oid2name.c:646（main 関数内と推測される箇所）。
  • **呼び出し先**: sql_exec（クエリ実行および出力用）。

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • struct options: 実行時の設定（quiet モードなど）を管理する構造体。

失敗モードと境界条件

  • データベース接続が切断されている場合、sql_exec 内部でエラーが発生する可能性がある。
  • システムカタログへのアクセス権限がない場合、クエリ実行時にエラーが発生する。
  • snprintf のバッファサイズ（1024バイト）を超えるクエリは構築されないが、現在のクエリ長は十分に収まる範囲である。

保守担当者向けメモ

  • 本関数は pg_database と pg_tablespace の結合に依存している。カタログ定義に変更があった場合はクエリの整合性を確認すること。
  • 出力形式を変更する場合は、sql_exec の仕様と併せて検討が必要。
