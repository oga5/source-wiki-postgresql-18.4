pg_timezone_initialize

定義

  • 種別: function
  • 定義箇所: src/timezone/pgtz.c:360

目的

システム起動時および初期化フェーズにおいて、セッションおよびログ出力用のデフォルトタイムゾーンを「GMT」に設定します。

シグネチャまたは形状

void pg_timezone_initialize(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 session_timezone および log_timezone を更新）

前提条件と不変条件

  • PGSHAREDIR の場所が不明な環境（EXEC_BACKEND サブプロセスなど）でも安全に実行可能です。
  • pg_tzset("GMT") を呼び出すことで、ファイルシステムに依存せずにタイムゾーンを初期化します。

アルゴリズムまたは振る舞い

1. pg_tzset("GMT") を呼び出し、GMT タイムゾーン情報を取得します。この関数はファイルシステムを参照せずに GMT を解釈するよう強制されます。 2. 取得したタイムゾーン情報を session_timezone グローバル変数に代入します。 3. log_timezone グローバル変数にも同じ値を代入します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/guc.c:1539
  • 呼び出し先: pg_tzset

関連データ構造

  • pg_tz: タイムゾーン情報を保持する構造体。

失敗モードと境界条件

  • 本関数はファイルシステムに依存しない「GMT」を明示的に指定するため、ファイルシステム関連の初期化エラーの影響を受けません。

保守担当者向けメモ

  • 本関数は guc_tables.c で定義されたブートストラップ時のデフォルト値と整合させる必要があります。
  • pg_tzset が内部でキャッシュを利用するため、この初期化は効率的に行われます。
