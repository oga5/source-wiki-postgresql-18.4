SharedDependencyType

定義

  • 種別: enum
  • 定義箇所: src/include/catalog/dependency.h:78

目的

データベースを跨いで共有されるオブジェクト（ロールやテーブルスペースなど）に対する依存関係のセマンティクスを定義します。

シグネチャまたは形状

typedef enum SharedDependencyType
{
    SHARED_DEPENDENCY_OWNER = 'o',
    SHARED_DEPENDENCY_ACL = 'a',
    SHARED_DEPENDENCY_INITACL = 'i',
    SHARED_DEPENDENCY_POLICY = 'r',
    SHARED_DEPENDENCY_TABLESPACE = 't',
    SHARED_DEPENDENCY_INVALID = 0,
} SharedDependencyType;

入力と出力

  • 入力: 各列挙値は char 型の文字コード（'o', 'a', 'i', 'r', 't'）に対応します。
  • 出力: pg_shdepend カタログにおける依存関係の種別を識別するために使用されます。

前提条件と不変条件

  • 共有オブジェクトの依存関係は、通常のオブジェクト依存関係（DependencyType）とは異なる制約ルールに従います。
  • SHARED_DEPENDENCY_INVALID は無効な状態を示します。

アルゴリズムまたは振る舞い

  • pg_shdepend カタログへのレコード追加・削除・更新時に、依存関係の性質を分類するために利用されます。
  • オブジェクトの削除処理において、共有オブジェクトに対する依存関係の整合性を維持するための判断基準となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/catalog/pg_shdepend.c 内の複数の関数（pg_shdepend カタログ操作関数）で引数として使用されています。
  • 呼び出し先: 本シンボル自体は列挙型定義であり、直接的な関数呼び出しは行いません。

関連データ構造

  • pg_shdepend: 共有依存関係を格納するシステムカタログ。
  • ObjectAddress: オブジェクトを一意に識別するために併用されます。

失敗モードと境界条件

  • SHARED_DEPENDENCY_INVALID を使用した操作は、依存関係の不整合やエラーを引き起こす可能性があります。

保守担当者向けメモ

  • 本型は共有オブジェクト専用であるため、通常の pg_depend を対象とする処理と混同しないよう注意してください。
  • 新たな共有依存関係の種別を追加する場合は、pg_shdepend.c 側のロジックも併せて更新する必要があります。
