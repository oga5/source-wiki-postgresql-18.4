SetPGVariable

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc_funcs.c:314

目的

CコードからPostgreSQLのGUC（Grand Unified Configuration）パラメータを設定するための汎用インターフェースを提供します。SQLの SET コマンドと同等の機能をプログラムから実行可能にします。

シグネチャまたは形状

void SetPGVariable(const char *name, List *args, bool is_local)

入力と出力

  • name (const char *): 設定するパラメータ名。
  • args (List *): パラメータに設定する値のリスト。
  • is_local (bool): true の場合、現在のトランザクション内のみ有効な設定（SET LOCAL）として適用されます。
  • 戻り値: なし（void）。

前提条件と不変条件

  • args が NULL の場合、SET DEFAULT と同様の振る舞い（デフォルト値へのリセット）となります。
  • 呼び出し元の権限に応じて、PGC_SUSET（スーパーユーザー用）または PGC_USERSET（一般ユーザー用）のコンテキストが自動的に選択されます。

アルゴリズムまたは振る舞い

1. flatten_set_variable_args を呼び出し、args リストをGUCが処理可能な文字列形式（argstring）に変換します。 2. set_config_option を呼び出し、以下の引数で設定を適用します。

  • コンテキスト: superuser() の判定に基づき PGC_SUSET または PGC_USERSET を指定。
  • ソース: PGC_S_SESSION（セッションレベルの設定）。
  • アクション: is_local が true なら GUC_ACTION_LOCAL、false なら GUC_ACTION_SET を指定。
  • その他: apply=true、scontext=0、scontext_arg=false を固定値として渡します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/discard.c
  • src/backend/tcop/utility.c
  • src/backend/utils/misc/guc_funcs.c（自身の内部および関連関数）
  • 呼び出し先:
  • flatten_set_variable_args
  • set_config_option（guc.c 内のコアロジック）
  • superuser()

関連データ構造

  • List: SQL構文解析結果を保持するPostgreSQLのリスト構造。

失敗モードと境界条件

  • name が無効なパラメータ名である場合、または args の値が不正な場合、set_config_option 内部でエラーが発生し、トランザクションがアボートする可能性があります。
  • 権限不足の場合、set_config_option によりエラーがスローされます。

保守担当者向けメモ

  • 本関数は SET コマンドの実行エントリポイントである ExecSetVariableStmt とは異なり、C言語レベルでの直接的な設定変更を目的としています。
  • set_config_option の引数構成を変更する際は、本関数が想定している PGC_S_SESSION ソースおよび権限判定ロジックとの整合性に注意してください。
