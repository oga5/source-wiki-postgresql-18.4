pg_cryptohash_ctx

定義

  • 種別: struct
  • 定義箇所: src/common/cryptohash_openssl.c:63

目的

OpenSSLの EVP_MD_CTX をラップし、PostgreSQLにおける暗号学的ハッシュ計算（MD5, SHA1, SHA2系列）の状態を保持するコンテキスト構造体です。

シグネチャまたは形状

struct pg_cryptohash_ctx
{
	pg_cryptohash_type type;
	pg_cryptohash_errno error;
	const char *errreason;
	EVP_MD_CTX *evpctx;
#ifndef FRONTEND
	ResourceOwner resowner;
#endif
};

入力と出力

  • **入力**: pg_cryptohash_create 等のAPIを通じて初期化され、pg_cryptohash_update でデータが投入されます。
  • **出力**: pg_cryptohash_final を通じてハッシュ値がバッファに書き出されます。エラー発生時は error フィールドおよび errreason に詳細が格納されます。

前提条件と不変条件

  • バックエンド環境では、ResourceOwner が割り当てられ、トランザクション終了時やエラー時の自動解放が保証されます。
  • コンテキスト解放時には explicit_bzero を使用し、機密情報がメモリ上に残らないよう配慮されます。

アルゴリズムまたは振る舞い

  • OpenSSLの EVP APIを抽象化し、ハッシュ計算のライフサイクル（作成、初期化、更新、終了、解放）を管理します。
  • OpenSSL固有のエラーコードをPostgreSQL標準の形式へ変換する仲介役を果たします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/uuid-ossp/uuid-ossp.c、src/backend/libpq/auth-scram.c、src/backend/utils/adt/cryptohashfuncs.c、および src/common/cryptohash.c から参照されています。
  • **呼び出し先**: OpenSSLの EVP_MD_CTX 型および関連するEVP関数群を内部的に利用します。

関連データ構造

  • pg_cryptohash_type: ハッシュアルゴリズムの種別。
  • pg_cryptohash_errno: エラーコード。
  • EVP_MD_CTX: OpenSSLのハッシュコンテキスト。
  • ResourceOwner: バックエンド環境でのリソース管理用構造体。

失敗モードと境界条件

  • OpenSSLの内部エラー（FIPSモード等でのエラーキューの競合など）が発生する可能性があります。
  • メモリ確保失敗やOpenSSLの初期化失敗時には、error フィールドに適切な値が設定されます。

保守担当者向けメモ

  • 本構造体はフロントエンドとバックエンドで定義が異なります（FRONTEND マクロによる条件分岐）。
  • バックエンド環境では TopMemoryContext を使用したメモリ確保が前提となっています。
