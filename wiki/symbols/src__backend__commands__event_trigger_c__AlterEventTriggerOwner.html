AlterEventTriggerOwner

定義

  • 種別: function
  • 定義箇所: src/backend/commands/event_trigger.c:478

目的

指定されたイベントトリガーの所有者を変更します。

シグネチャまたは形状

ObjectAddress AlterEventTriggerOwner(const char *name, Oid newOwnerId)

入力と出力

  • **入力**:
  • name: 変更対象のイベントトリガー名（const char *）。
  • newOwnerId: 新しい所有者のOID（Oid）。
  • **出力**:
  • 変更されたイベントトリガーの ObjectAddress を返します。

前提条件と不変条件

  • pg_event_trigger システムカタログに対する RowExclusiveLock を保持する必要があります。
  • 指定された名前のイベントトリガーがシステムカタログに存在しなければなりません。

アルゴリズムまたは振る舞い

1. pg_event_trigger テーブルを RowExclusiveLock でオープンします。 2. EVENTTRIGGERNAME システムキャッシュを使用して、対象イベントトリガーのタプルを検索・コピーします。 3. タプルが存在しない場合は ERRCODE_UNDEFINED_OBJECT エラーを発生させます。 4. AlterEventTriggerOwner_internal を呼び出し、所有者変更の内部処理（カタログ更新や権限チェック等）を実行します。 5. ObjectAddressSet を使用して、対象イベントトリガーのオブジェクトアドレスを構築します。 6. コピーしたタプルを解放し、テーブルをクローズしてアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c:863
  • **呼び出し先**:
  • table_open / table_close
  • SearchSysCacheCopy1
  • AlterEventTriggerOwner_internal
  • heap_freetuple
  • ObjectAddressSet

関連データ構造

  • ObjectAddress: 操作対象のオブジェクト識別子。
  • Form_pg_event_trigger: pg_event_trigger カタログの行構造体。
  • HeapTuple: システムカタログから取得したタプル。

失敗モードと境界条件

  • 指定した名前のイベントトリガーが存在しない場合、ereport によりエラー終了します。

保守担当者向けメモ

  • 本関数はラッパーであり、実際のカタログ更新ロジックは AlterEventTriggerOwner_internal に委譲されています。
  • 呼び出し元情報の詳細は src/backend/commands/alter.c を参照してください。
