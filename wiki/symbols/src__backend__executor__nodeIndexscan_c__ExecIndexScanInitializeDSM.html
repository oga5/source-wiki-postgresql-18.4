ExecIndexScanInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeIndexscan.c:1693

目的

並列クエリ実行時において、インデックススキャンノードのための共有メモリ（DSM）領域を確保・初期化し、ワーカー間で共有されるスキャン状態をセットアップする。

シグネチャまたは形状

void ExecIndexScanInitializeDSM(IndexScanState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の IndexScanState 構造体。
  • pcxt: 並列実行コンテキスト（ParallelContext）。
  • **出力**: なし（副作用としてDSM領域の確保とノード状態の更新を行う）。

前提条件と不変条件

  • node->ss.ps.plan が有効であること。
  • instrument（統計情報取得）または parallel_aware（並列認識）のいずれかが有効な場合にのみDSM操作が行われる。

アルゴリズムまたは振る舞い

1. **DSM領域の確保**: shm_toc_allocate を使用して、必要なサイズの並列スキャン記述子（ParallelIndexScanDesc）を確保する。 2. **初期化**: index_parallelscan_initialize を呼び出し、インデックススキャンの共有情報を初期化する。 3. **登録**: shm_toc_insert を使用して、プランノードIDをキーにDSM領域を登録する。 4. **並列スキャン開始**: parallel_aware が真の場合、index_beginscan_parallel を呼び出し、並列スキャン記述子を node->iss_ScanDesc に格納する。 5. **スキャンキーの適用**: ランタイムキーが存在しない、または準備完了状態であれば、index_rescan を呼び出してスキャンキーとオーダーキーをインデックスAMに適用する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:479（ExecIndexScanInitializeDSM を呼び出していることが確認されている）。
  • **呼び出し先**:
  • shm_toc_allocate, shm_toc_insert
  • index_parallelscan_initialize
  • index_beginscan_parallel
  • index_rescan

関連データ構造

  • IndexScanState: インデックススキャンの実行状態を保持する構造体。
  • ParallelIndexScanDesc: 並列スキャンにおける共有状態を管理する構造体。
  • ParallelContext: 並列実行全体のコンテキスト。

失敗モードと境界条件

  • instrument も parallel_aware も偽である場合、本関数は何もせず即座にリターンする。
  • parallel_aware が偽の場合、DSM上の SharedInfo の初期化のみを行い、スキャン記述子の生成はスキップされる。

保守担当者向けメモ

  • 本関数は並列実行の初期化フェーズで呼び出される。インデックススキャンの並列化に関するDSM管理の要所であるため、ParallelIndexScanDesc のレイアウト変更時には注意が必要。
