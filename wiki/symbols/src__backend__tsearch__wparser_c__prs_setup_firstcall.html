prs_setup_firstcall

定義

  • **種別**: function
  • **定義箇所**: src/backend/tsearch/wparser.c:161

目的

SQLインターフェース（SRF）を介したテキスト解析の初回呼び出し時に、指定されたパーサーを使用してテキストを全トークンに分解し、メモリ上にキャッシュ（PrsStorage）として保持する。

シグネチャまたは形状

static void prs_setup_firstcall(FuncCallContext *funcctx, FunctionCallInfo fcinfo, Oid prsid, text *txt)

入力と出力

  • **入力**:
  • funcctx: SRFの状態を保持するコンテキスト。
  • fcinfo: 関数呼び出し情報。
  • prsid: 使用するパーサーのOID。
  • txt: 解析対象のテキストデータ。
  • **出力**: なし（funcctx->user_fctx に解析結果の PrsStorage を格納し、funcctx->tuple_desc を設定する）。

前提条件と不変条件

  • funcctx->multi_call_memory_ctx が有効であること。
  • lookup_ts_parser_cache(prsid) が有効なパーサー定義を返すこと。
  • 戻り値の型が複合型（TYPEFUNC_COMPOSITE）であること。

アルゴリズムまたは振る舞い

1. funcctx->multi_call_memory_ctx へメモリコンテキストを切り替える。 2. PrsStorage 構造体を初期化し、初期バッファを確保する。 3. lookup_ts_parser_cache で取得したパーサーの prsstart を呼び出し、解析用データ（prsdata）を取得する。 4. prstoken をループ呼び出しし、テキストをトークンに分解して PrsStorage のリストに格納する。必要に応じて repalloc でリストを拡張する。 5. prsend を呼び出し、解析処理を終了する。 6. funcctx->user_fctx に PrsStorage を保存し、タプルのメタデータを設定して元のメモリコンテキストへ戻る。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ts_parse_byid (src/backend/tsearch/wparser.c:252)
  • ts_parse_byname (src/backend/tsearch/wparser.c:277)
  • **呼び出し先**:
  • lookup_ts_parser_cache
  • prsstart, prstoken, prsend (パーサー定義内の関数ポインタ)
  • get_call_result_type
  • TupleDescGetAttInMetadata

関連データ構造

  • FuncCallContext: SRFの状態管理用。
  • PrsStorage: 解析済みトークンリストを保持する構造体。
  • TSParserCacheEntry: パーサーの関数ポインタ群を保持するキャッシュエントリ。
  • LexemeEntry: 個別のトークン情報を保持する構造体。

失敗モードと境界条件

  • get_call_result_type が複合型を返さない場合、elog(ERROR) により処理を中断する。
  • メモリ不足時には palloc / repalloc がエラーを発生させる。

保守担当者向けメモ

  • 本関数はSRFの初回実行時に全トークンをメモリに展開するため、非常に長いテキストを解析する場合、メモリ消費量が増大する点に注意が必要。
  • prsstart, prstoken, prsend は TSParserCacheEntry を介して動的に呼び出されるため、パーサーの実装に依存する。
