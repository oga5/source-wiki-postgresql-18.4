in_range_int2_int8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int.c:779

目的

int2 型の値が、指定された範囲内にあるかどうかを判定する関数です。

シグネチャまたは形状

Datum in_range_int2_int8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して5つの引数を受け取ります。
  • 第1引数: int2 型の値（比較対象）。
  • 第2引数: int2 型の値（範囲の基準値）。
  • 第3引数以降: 範囲判定に必要なオフセットやフラグ等の Datum 値。
  • **出力**: Datum 型（範囲内であれば true、そうでなければ false を示す論理値）。

前提条件と不変条件

  • 入力された int2 型の値は、内部で int32 にキャストされて処理されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT16 を使用して第1引数および第2引数を取得します。 2. 取得した値を Int32GetDatum により int32 型へキャストします。 3. in_range_int4_int8 関数を呼び出し、その結果をそのまま返却します。コードの重複を避けるため、int4 向けのロジックへ処理を委譲しています。

呼び出し元と呼び出し先

  • **呼び出し先**: in_range_int4_int8
  • **呼び出し元**: 提供されたソース情報からは特定の呼び出し元は特定できません。

関連データ構造

  • Datum: PostgreSQLの汎用データ表現型。
  • int2 / int32: PostgreSQLの整数型。

失敗モードと境界条件

  • in_range_int4_int8 の実装に依存します。
  • 引数の型変換時にオーバーフローが発生する可能性は低いですが、int2 から int32 への変換は安全に行われます。

保守担当者向けメモ

  • 本関数は in_range_int4_int8 への単なるラッパーです。ロジックの変更が必要な場合は、呼び出し先の in_range_int4_int8 を修正してください。
