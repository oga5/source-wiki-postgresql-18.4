LexizeAddLemm

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_parse.c:99

目的

辞書処理の作業中リスト（towork）に新しい語彙（ParsedLex）を追加し、現在の処理対象ポインタを更新します。

シグネチャまたは形状

static void LexizeAddLemm(LexizeData *ld, int type, char *lemm, int lenlemm)

入力と出力

  • **入力**:
  • ld: 辞書処理の状態を保持する LexizeData 構造体へのポインタ。
  • type: 追加する語彙のトークン型。
  • lemm: 語彙の文字列データ。
  • lenlemm: 語彙の長さ。
  • **出力**: なし（副作用として ld の内部状態を更新）。

前提条件と不変条件

  • ld は有効な LexizeData 構造体を指している必要があります。
  • palloc によるメモリ確保が成功することが前提です。

アルゴリズムまたは振る舞い

1. palloc を使用して新しい ParsedLex 構造体をメモリ上に確保します。 2. 引数で受け取った type、lemm、lenlemm を新しい構造体に設定します。 3. LPLAddTail を呼び出し、ld->towork リストの末尾に新しい構造体を追加します。 4. ld->curSub を、追加されたばかりの末尾要素（ld->towork.tail）に更新します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/ts_parse.c:400
  • src/backend/tsearch/ts_parse.c:586
  • **呼び出し先**:
  • LPLAddTail（リスト操作ユーティリティ）
  • palloc（PostgreSQL メモリ管理）

関連データ構造

  • LexizeData: 辞書処理の現在の状態を保持する構造体。
  • ParsedLex: 語彙情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、処理が中断されます。

保守担当者向けメモ

  • 本関数は LexizeData の towork リストを操作する低レベルなヘルパーです。curSub の更新が towork リストの末尾に固定される点に注意してください。
