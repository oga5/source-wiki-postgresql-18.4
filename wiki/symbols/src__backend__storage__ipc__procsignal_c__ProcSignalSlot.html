ProcSignalSlot

定義

  • **種別**: struct
  • **定義箇所**: src/backend/storage/ipc/procsignal.c:64

目的

各バックエンドプロセスまたは補助プロセスのシグナル状態および同期バリア情報を保持し、プロセス間での非同期イベント伝達を管理する共有メモリ上のデータ構造です。

シグネチャまたは形状

typedef struct {
    pg_atomic_uint32 pss_pid;
    int              pss_cancel_key_len;
    uint8            pss_cancel_key[MAX_CANCEL_KEY_LENGTH];
    volatile sig_atomic_t pss_signalFlags[NUM_PROCSIGNALS];
    slock_t          pss_mutex;
    pg_atomic_uint64 pss_barrierGeneration;
    pg_atomic_uint32 pss_barrierCheckMask;
    ConditionVariable pss_barrierCV;
} ProcSignalSlot;

入力と出力

  • **入力**: プロセス間シグナル送信時、pss_signalFlags に特定のシグナル理由が設定されます。
  • **出力**: 受信側プロセスが pss_signalFlags を読み取り、対応する割り込み処理を実行します。

前提条件と不変条件

  • pss_mutex は pss_signalFlags およびキャンセルキー関連フィールドの保護に使用されます。
  • バリア関連フィールド（pss_barrierGeneration 等）は pss_mutex の保護対象外であり、アトミック操作で管理されます。
  • pss_cancel_key_len が 0 の場合、そのプロセスに対するキャンセルは不可とみなされます。

アルゴリズムまたは振る舞い

1. **シグナル通知**: 送信側が対象スロットの pss_signalFlags を更新し、kill(pid, SIGUSR1) を発行します。 2. **シグナル受信**: 受信側が procsignal_sigusr1_handler を経由して pss_signalFlags を確認し、必要な処理（クエリキャンセルやバリア同期など）をトリガーします。 3. **バリア同期**: pss_barrierGeneration を用いて全プロセスが特定の同期点に到達したかを追跡し、pss_barrierCV を介して待機・通知を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/procsignal.c 内の各関数（SendProcSignal, EmitProcSignalBarrier, WaitForProcSignalBarrier 等）がこの構造体を参照・操作します。
  • **呼び出し先**: 特定の関数呼び出し関係はソース抜粋からは直接特定できませんが、MyProcSignalSlot を通じて各プロセスが自身のスロットを操作します。

関連データ構造

  • ProcSignalHeader: 全プロセスの ProcSignalSlot を管理する共有メモリ上のヘッダー。
  • ConditionVariable: バリア同期のための待機キュー。
  • ProcSignalReason: procsignal.h で定義されるシグナル理由の列挙型。

失敗モードと境界条件

  • **シグナルの上書き**: pss_signalFlags は配列として保持されますが、同一理由のシグナルが連続して届いた場合、古い通知が上書きされる可能性があります。
  • **バリアの競合**: EmitProcSignalBarrier は全プロセスを中断させるため、高頻度な呼び出しはシステム全体のパフォーマンスを低下させる境界条件となります。

保守担当者向けメモ

  • pss_mutex による保護とアトミック操作の使い分けが重要です。バリア関連フィールドはアトミック操作のみで保護されている点に注意してください。
  • pss_signalFlags の操作は SIGUSR1 ハンドラ内で行われるため、非同期シグナル安全性を維持する必要があります。
