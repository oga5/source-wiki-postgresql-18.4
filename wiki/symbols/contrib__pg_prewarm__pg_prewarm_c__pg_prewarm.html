pg_prewarm

定義

  • 種別: function
  • 定義箇所: contrib/pg_prewarm/pg_prewarm.c:59

目的

指定されたリレーションのブロックを、OS キャッシュまたは共有バッファへ事前に読み込み（プリウォーム）、アクセスを高速化する。

シグネチャまたは形状

Datum pg_prewarm(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • regclass (リレーションOID)
  • text (プリウォーム手法: "prefetch", "read", "buffer")
  • text (フォーク名: "main", "fsm", "vm" 等)
  • int64 (開始ブロック番号、省略時は0)
  • int64 (終了ブロック番号、省略時は最終ブロック)
  • **出力**: 処理したブロック数 (int64) を返す。

前提条件と不変条件

  • 対象リレーションに対する SELECT 権限が必要。インデックスの場合は親テーブルの権限を確認する。
  • リレーションはストレージを持つ必要がある。
  • 指定されたフォークが存在しなければならない。
  • 実行中、長時間処理を考慮して CHECK_FOR_INTERRUPTS() が呼び出される。

アルゴリズムまたは振る舞い

1. 引数を解析し、プリウォーム手法 (PrewarmType) を決定する。 2. リレーションをオープンし、権限チェックを行う。インデックスの場合はデッドロック回避のため親テーブルを先にロックする。 3. 指定されたブロック範囲を検証し、NULL の場合はデフォルト値を適用する。 4. 手法に応じたループ処理を実行する:

  • PREFETCH: PrefetchBuffer を使用し、OS に読み込みヒントを送信する（非同期）。
  • READ: smgrread を使用し、物理的に読み込む（同期）。
  • BUFFER: ReadStream を使用し、共有バッファへロードする。

    • 5. 処理したブロック数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェースから直接呼び出される。autoprewarm.c は本関数を直接呼び出すのではなく、バックグラウンドワーカーとしてプリウォーム状態を管理する。
  • **呼び出し先**: relation_open, IndexGetRelation, smgrexists, PrefetchBuffer, smgrread, read_stream_begin_relation 等。

関連データ構造

  • PrewarmType (列挙型: PREWARM_PREFETCH, PREWARM_READ, PREWARM_BUFFER)
  • Relation
  • ForkNumber

失敗モードと境界条件

  • 無効なプリウォーム手法や NULL 引数が渡された場合はエラーとなる。
  • リレーションがストレージを持たない場合や、指定したフォークが存在しない場合はエラーとなる。
  • 権限不足時は aclcheck_error を発生させる。
  • インデックスの親テーブルがロック取得前に削除された場合、エラーとなる。
  • PREFETCH モードがビルド時にサポートされていない場合、エラーとなる。

保守担当者向けメモ

  • BUFFER モードは共有バッファを消費するため、メモリ圧迫に注意が必要。
  • インデックスのプリウォーム時は、親テーブルのロックを優先することでデッドロックを回避している。
  • 処理のキャンセルは CHECK_FOR_INTERRUPTS() により行われるため、ループ内での呼び出しを維持すること。
