dropTablespaces

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dumpall.c:1346

目的

PostgreSQL クラスタから、組み込みテーブルスペース（pg_ で始まる名前）を除くすべてのテーブルスペースを削除するための DROP TABLESPACE SQL 文を生成し、出力ストリーム（OPF）へ書き出す。

シグネチャまたは形状

static void dropTablespaces(PGconn *conn)

入力と出力

  • **入力**: PGconn *conn（データベース接続ハンドル）
  • **出力**: なし（副作用としてグローバル変数 OPF が指すストリームへ SQL を出力）

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • グローバル変数 OPF が書き込み可能な出力ストリームとして初期化されていること。
  • グローバル変数 if_exists が定義されており、DROP TABLESPACE 文に IF EXISTS 句を付与するかどうかの制御に使用される。

アルゴリズムまたは振る舞い

1. pg_catalog.pg_tablespace システムカタログをクエリし、spcname が pg_ で始まらないテーブルスペースの一覧を取得する。 2. 取得したテーブルスペースが存在する場合、出力ストリームにヘッダーコメント -- Drop tablespaces を書き出す。 3. 取得した各テーブルスペース名に対し、fmtId() で識別子をエスケープし、DROP TABLESPACE 文を生成して出力する。 4. if_exists が真であれば、各文に IF EXISTS 句を挿入する。 5. 最後に PQclear(res) を呼び出し、クエリ結果を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dumpall.c 内のメイン処理フロー（行640付近）。
  • **呼び出し先**: executeQuery（クエリ実行）、PQntuples / PQgetvalue / PQclear（libpq 関数）、fmtId（識別子整形）。

関連データ構造

  • PGconn: データベース接続状態を保持する libpq の構造体。
  • PGresult: クエリ結果を保持する libpq の構造体。

失敗モードと境界条件

  • データベース接続が切断されている場合、executeQuery 内部でエラーが発生する可能性がある。
  • テーブルスペースが存在しない場合、ループは実行されず、ヘッダーも出力されない。
  • fmtId による識別子のエスケープに失敗するような特殊なテーブルスペース名が含まれる場合、生成される SQL が不正になる可能性がある。

保守担当者向けメモ

  • 本関数は pg_dumpall のダンプ出力の冒頭部分で呼び出されることを想定している。
  • 組み込みテーブルスペースの除外条件（spcname !~ '^pg_'）はハードコードされているため、将来的にシステムテーブルスペースの命名規則が変更された場合は修正が必要となる。
