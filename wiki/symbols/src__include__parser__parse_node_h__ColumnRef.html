ColumnRef

定義

  • 種別: typedef
  • 定義箇所: src/include/parser/parse_node.h:91
  • 構造体定義: nodes/parsenodes.h:307

目的

SQL 文の解析過程において、クエリ内の列参照（テーブル名や列名など）を表現するための構文木ノードです。

シグネチャまたは形状

nodes/parsenodes.h で定義される構造体であり、解析対象の列名や修飾子、ソーステキスト上の位置情報を保持します。

入力と出力

  • **入力**: SQL のパース時に生成される構文木の一部として入力されます。
  • **出力**: 解析フェーズにおいて、ParseState を通じて解決され、最終的に Var ノードや他のクエリ木要素へ変換されます。

前提条件と不変条件

  • location フィールドには、ソーステキスト内の有効なオフセットが格納されている必要があります。
  • PreParseColumnRefHook を介して処理される場合、ParseState が有効なコンテキストを提供している必要があります。

アルゴリズムまたは振る舞い

  • パーサーは ColumnRef を走査し、現在の ParseState が持つ範囲テーブル（Range Table）や名前空間（Namespace）と照合して、参照先の列を特定します。
  • PreParseColumnRefHook が定義されている場合、標準的な解決ロジックの前にフック関数が呼び出され、カスタムの解決処理が実行可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copy.c 等でノードの生成が行われます。また、src/backend/nodes/nodeFuncs.c でノードの走査時に参照されます。
  • **呼び出し先**: PreParseColumnRefHook 型の定義により、列参照解決のためのフック関数が呼び出される可能性があります。
  • *注: 具体的な呼び出しチェーンの詳細は、提供されたソース範囲内では網羅されていません。*

関連データ構造

  • ParseState: 解析状態を保持し、ColumnRef の解決コンテキストを提供します。
  • Node: PostgreSQL の構文木ノードの基底型。
  • PreParseColumnRefHook: ColumnRef を処理するための関数ポインタ型。

失敗モードと境界条件

  • 参照された列名が現在の名前空間で見つからない場合、解析エラーが発生します。
  • location が無効な場合、エラーメッセージの発生箇所特定に失敗する可能性があります。

保守担当者向けメモ

  • ColumnRef は nodes/parsenodes.h で定義されているため、構造体のフィールド構成を変更する場合はそちらを参照してください。
  • parse_node.h で定義されている PreParseColumnRefHook は、拡張モジュールや特定のコマンド（COPY 等）で列解決ロジックをカスタマイズするために使用されます。
