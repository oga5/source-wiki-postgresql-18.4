cube_a_f8_f8

定義

  • 種別: function
  • 定義箇所: contrib/cube/cube.c:142

目的

2つの浮動小数点数配列（右上座標と左下座標）から、PostgreSQL の cube 型である NDBOX 構造体を生成します。

シグネチャまたは形状

Datum cube_a_f8_f8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_ARRAYTYPE_P(0): 右上座標（UR）を表す ArrayType
  • PG_GETARG_ARRAYTYPE_P(1): 左下座標（LL）を表す ArrayType
  • **出力**:
  • PG_RETURN_NDBOX_P(result): 生成された NDBOX 構造体へのポインタ

前提条件と不変条件

  • 入力配列に NULL を含めることはできません。
  • 両方の配列の次元数は一致している必要があります。
  • 次元数は CUBE_MAX_DIM を超えてはなりません。

アルゴリズムまたは振る舞い

1. 入力配列の妥当性（NULLチェック、次元数の一致、上限チェック）を検証します。 2. 全ての軸において dur[i] == dll[i] であるかを確認し、ポイント（点）かボックスかを判定します。 3. 判定結果に基づき、必要なメモリサイズ（POINT_SIZE または CUBE_SIZE）を計算し、palloc0 でメモリを確保します。 4. NDBOX のヘッダー情報を設定し、座標データをコピーします。

  • ポイントの場合は SET_POINT_BIT を使用してフラグを立てます。
  • ボックスの場合は、前半に右上座標、後半に左下座標を格納します。

    • 5. 構築した NDBOX を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数呼び出しインターフェース（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: PostgreSQL のメモリ管理関数（palloc0）および配列操作マクロ（ARRNELEMS, ARRPTR 等）。

関連データ構造

  • NDBOX: contrib/cube/cubedata.h で定義される多次元ボックス構造体。
  • ArrayType: PostgreSQL の配列型。

失敗モードと境界条件

  • **ERRCODE_ARRAY_ELEMENT_ERROR**: 入力配列に NULL が含まれる場合、または2つの配列の次元数が異なる場合に発生します。
  • **ERRCODE_PROGRAM_LIMIT_EXCEEDED**: 次元数が CUBE_MAX_DIM を超える場合に発生します。

保守担当者向けメモ

  • NDBOX の内部レイアウトは、ポイントか否かでメモリ配置が異なります。座標アクセスには cubedata.h で定義されたマクロを使用してください。
  • palloc0 を使用しているため、メモリリークの心配はありません。
