r_mark_regions

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_ISO_8859_1_italian.c:586

目的

イタリア語のステミング処理において、単語を語幹抽出のルール適用範囲である RV, R1, R2 領域に分割し、その境界位置を SN_env 構造体に記録する。

シグネチャまたは形状

static int r_mark_regions(struct SN_env * z)

入力と出力

  • 入力: struct SN_env * z（Snowball の実行環境。現在の単語データやカーソル位置 c を保持）
  • 出力: int（常に 1 を返す）
  • 副作用: z->I[0] (R2), z->I[1] (R1), z->I[2] (RV) に、計算された領域の開始位置（オフセット）が格納される。

前提条件と不変条件

  • z は有効な SN_env 構造体を指していること。
  • g_v（母音グループ）が定義されていること。
  • 領域の計算は、単語内の母音と子音の出現パターンに基づいて行われる。

アルゴリズムまたは振る舞い

1. 初期状態として、すべての領域（RV, R1, R2）の開始位置を単語の末尾（z->l）に設定する。 2. **RV 領域の特定**:

  • 母音の直後の子音、または単語の先頭から数えて2番目の母音以降の開始位置を探索する。
  • 複数の条件分岐（lab ラベル）を用いて、イタリア語の音韻規則に基づいた RV 領域の境界を決定し、z->I[2] に保存する。

    • 3. **R1, R2 領域の特定**:

  • 母音と子音の交互の出現パターンを追跡する。
  • out_grouping（子音）と in_grouping（母音）を交互に呼び出し、母音の直後の子音位置を特定する。
  • 最初の母音-子音遷移の後に現れる母音-子音遷移を R1 (z->I[1]) とし、さらにその後に現れるものを R2 (z->I[0]) として記録する。

    • 4. 処理終了後、カーソル位置 z->c を元の位置に復元して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: italian_ISO_8859_1_stem（直接の呼び出し関係はソースコードの制御フローから推論可能）。
  • 呼び出し先: in_grouping, out_grouping（Snowball ランタイムのユーティリティ関数）。

関連データ構造

  • struct SN_env: ステミング処理の状態を保持する構造体。I[] 配列に領域境界、c にカーソル位置、l に単語長を保持する。

失敗モードと境界条件

  • 単語が短すぎる場合や、母音・子音のパターンが領域定義を満たさない場合、領域の開始位置はデフォルト値（単語末尾）のまま維持される。
  • in_grouping / out_grouping が失敗した場合は、各ラベル（lab0〜lab7）へジャンプし、部分的な計算をスキップする。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによる自動生成コードであるため、直接編集せず、元の .sbl ファイルを修正すること。
  • z->I[0] (R2), z->I[1] (R1), z->I[2] (RV) のインデックスは、他の言語のステマーと共通の規約に従っている。デバッグ時は z->I の値が期待する境界位置と一致しているかを確認すること。
