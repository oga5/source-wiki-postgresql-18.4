cleanup_path

定義

  • 種別: function
  • 定義箇所: src/port/path.c:257

目的

Windows 環境において、ファイルパスを短い形式（Short Path Name）に変換し、パス区切り文字を標準化することで、パスの整合性を確保する。

シグネチャまたは形状

void cleanup_path(char *path)

入力と出力

  • **入力**: path (変更可能な文字列バッファへのポインタ)。
  • **出力**: なし（引数 path が直接書き換えられる）。

前提条件と不変条件

  • Windows 環境でのみ動作する（#ifdef WIN32）。
  • path は MAXPGPATH 以上の十分なバッファサイズを持つ必要がある。

アルゴリズムまたは振る舞い

1. GetShortPathName を呼び出し、指定されたパスを短い形式に変換する。

  • パスが存在しない場合や、ファイルシステムで短い名前がサポートされていない場合は、元のパスを維持する。

    • 2. debackslash_path を呼び出し、パス内の \ を / に置換する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/config_info.c 内の複数の箇所から呼び出されている。
  • **呼び出し先**:
  • GetShortPathName (Windows API)
  • debackslash_path (同一ファイル src/port/path.c 内の関数)

関連データ構造

  • なし。

失敗モードと境界条件

  • GetShortPathName が失敗した場合、エラーを報告せず、元のパスをそのまま保持して処理を継続する。これは、--sysconfdir のようにパスがまだ存在しないケースを許容するためである。

保守担当者向けメモ

  • 本関数は Windows 特有のパス処理を抽象化するものである。
  • debackslash_path 呼び出し時に PG_SQL_ASCII を指定しているのは、サーバーで安全とされるエンコーディングにおいて \ の扱いが共通しているためである。
