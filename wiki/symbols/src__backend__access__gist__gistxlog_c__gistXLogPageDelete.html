gistXLogPageDelete

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistxlog.c:551

目的

GiSTインデックスのページ削除操作をWAL（Write-Ahead Log）に記録する。

シグネチャまたは形状

XLogRecPtr gistXLogPageDelete(Buffer buffer, FullTransactionId xid, Buffer parentBuffer, OffsetNumber downlinkOffset)

入力と出力

  • **入力**:
  • buffer: 削除対象のインデックスページ。
  • xid: 削除を実行したトランザクションID。
  • parentBuffer: 削除対象ページへのダウンリンクを持つ親ページ。
  • downlinkOffset: 親ページ内でのダウンリンクのオフセット。
  • **出力**:
  • XLogRecPtr: 生成されたWALレコードの挿入位置（LSN）。

前提条件と不変条件

  • XLogBeginInsert が呼び出された後に実行される必要がある。
  • buffer および parentBuffer は、WAL記録のために XLogRegisterBuffer を介して登録される。

アルゴリズムまたは振る舞い

1. gistxlogPageDelete 構造体に xid と downlinkOffset を格納する。 2. XLogBeginInsert を呼び出し、WAL挿入シーケンスを開始する。 3. XLogRegisterData を使用して、gistxlogPageDelete 構造体の内容をWALデータとして登録する。 4. XLogRegisterBuffer を使用し、buffer（インデックスページ）と parentBuffer（親ページ）を標準的な形式（REGBUF_STANDARD）で登録する。 5. XLogInsert を呼び出し、RM_GIST_ID リソースマネージャおよび XLOG_GIST_PAGE_DELETE オペコードでWALレコードを書き込む。 6. 書き込まれたレコードのLSNを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gistvacuum.c:708
  • **呼び出し先**:
  • XLogBeginInsert
  • XLogRegisterData
  • XLogRegisterBuffer
  • XLogInsert

関連データ構造

  • gistxlogPageDelete: 削除操作の詳細（トランザクションIDとオフセット）を保持する構造体。

失敗モードと境界条件

  • WAL書き込み時にディスクフルやI/Oエラーが発生した場合、XLogInsert 内部でエラーがスローされる可能性がある。

保守担当者向けメモ

  • 本関数はGiSTインデックスのバキューム処理に関連しており、親ページと対象ページの両方をWALに登録することで、クラッシュリカバリ時の整合性を保証している。
  • gistxlogPageDelete 構造体の定義は src/include/access/gistxlog.h を参照のこと。
