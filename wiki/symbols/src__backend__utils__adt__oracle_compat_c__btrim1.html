btrim1

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oracle_compat.c:378

目的

入力文字列の先頭および末尾から空白文字（' '）を除去した新しい文字列を生成する。

シグネチャまたは形状

Datum btrim1(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_TEXT_PP(0) を介して取得される text 型の文字列。
  • **出力**: 先頭と末尾の空白が除去された text 型のデータ（Datum）。

前提条件と不変条件

  • 入力文字列は text 型として有効であること。
  • 内部的に dotrim 関数を呼び出すため、マルチバイト文字環境下では文字境界が正しく処理される。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して入力文字列を取得する。 2. dotrim 関数を呼び出し、以下の引数を渡す。

  • 入力文字列のデータ本体とサイズ。
  • 除去対象文字として " "（空白）、その長さ 1。
  • 先頭除去フラグ true、末尾除去フラグ true。

    • 3. dotrim が返した結果を PG_RETURN_TEXT_P で返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内のコメントによれば、btrim の「セットを空白に固定した版」として利用される。具体的な呼び出し箇所はソースコード上では明示されていない。
  • **呼び出し先**: dotrim（src/backend/utils/adt/oracle_compat.c 内で定義された共通トリミング関数）。

関連データ構造

  • text: PostgreSQL の可変長文字列型。

失敗モードと境界条件

  • 入力文字列が空の場合、dotrim の仕様に従い空文字列が返される。
  • メモリ不足時には palloc によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は dotrim のラッパーである。トリミングロジックの変更が必要な場合は dotrim を参照すること。
  • 除去対象文字を動的に変更したい場合は、btrim1 ではなく btrim を使用する設計となっている。
