SPLITCOST

定義

  • 種別: struct
  • 定義箇所: contrib/hstore/hstore_gist.c:351

目的

GiST インデックスのノード分割処理において、各エントリの分割コストを保持し、ソートおよび比較を行うためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	OffsetNumber pos;
	int32		cost;
} SPLITCOST;

入力と出力

  • pos: インデックス内のエントリ位置（OffsetNumber）。
  • cost: 分割に伴うコスト計算値（int32）。

前提条件と不変条件

  • ghstore_picksplit 関数内で、インデックスノードの分割効率を評価するために使用されます。
  • cost フィールドは、分割アルゴリズムにおけるハミング距離などの指標に基づいて算出されます。

アルゴリズムまたは振る舞い

  • ghstore_picksplit 内で palloc により maxoff 分の配列として確保されます。
  • qsort を用いて cost 値に基づいた昇順ソートが行われ、最適な分割点を選択するための基準として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/hstore/hstore_gist.c 内の ghstore_picksplit 関数が、インデックス分割の計算過程で本構造体の配列を生成・操作します。
  • **呼び出し先**: qsort に渡される比較関数（comparecost 等）から、cost フィールドが参照されます。
  • 注: contrib/intarray/_int_gist.c にも同名の構造体定義が存在しますが、本ページは hstore モジュール内の定義を対象としています。

関連データ構造

  • OffsetNumber: PostgreSQL のインデックスページ内でのエントリ位置を示す型。

失敗モードと境界条件

  • メモリ不足時: palloc によるメモリ確保が失敗した場合、PostgreSQL のエラーハンドリングにより中断されます。
  • maxoff が 0 の場合: 配列の確保やソート処理において境界条件が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は contrib/intarray と定義が重複しています。将来的なリファクタリングで共通化の検討が可能です。
  • cost の計算ロジックを変更する場合は、ghstore_picksplit 内のコスト評価関数と併せて修正する必要があります。
