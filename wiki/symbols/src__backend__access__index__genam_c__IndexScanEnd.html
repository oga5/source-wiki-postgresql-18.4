IndexScanEnd

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:144

目的

インデックススキャン記述子（IndexScanDesc）に関連付けられたメモリリソースを解放し、スキャンを終了する。

シグネチャまたは形状

void IndexScanEnd(IndexScanDesc scan)

入力と出力

  • **入力**: scan (IndexScanDesc) - 終了対象となるインデックススキャン記述子へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • scan は有効な IndexScanDesc 構造体を指している必要がある。
  • scan が指す構造体内の keyData および orderByData は、pfree 可能であるか、または NULL である必要がある。

アルゴリズムまたは振る舞い

1. scan->keyData が NULL でない場合、pfree を呼び出してメモリを解放する。 2. scan->orderByData が NULL でない場合、pfree を呼び出してメモリを解放する。 3. 最後に scan 自体に対して pfree を呼び出し、記述子構造体全体を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/index/indexam.c:404 からの呼び出しが確認されている。
  • **呼び出し先**: pfree（メモリ管理関数）。

関連データ構造

  • IndexScanDesc: インデックススキャンの状態を保持する構造体。本関数により破棄される。

失敗モードと境界条件

  • scan が NULL の場合、pfree の仕様に依存する（通常はセグメンテーション違反を引き起こす可能性がある）。
  • keyData や orderByData が不正なポインタである場合、メモリ解放時にエラーが発生する。

保守担当者向けメモ

  • 本関数はリソースの解放のみを行う。スキャンに関連する他のリソース（ロックやバッファのピンなど）が適切に処理されていることを呼び出し側で保証する必要がある。
