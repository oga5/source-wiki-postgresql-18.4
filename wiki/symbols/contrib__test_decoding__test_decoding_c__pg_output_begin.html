pg_output_begin

定義

  • 種別: function
  • 定義箇所: contrib/test_decoding/test_decoding.c:309

目的

論理デコーディングの出力プラグインにおいて、トランザクション開始イベント（BEGIN）をテキスト形式で出力バッファに書き込む。

シグネチャまたは形状

static void pg_output_begin(LogicalDecodingContext *ctx, TestDecodingData *data, ReorderBufferTXN *txn, bool last_write)

入力と出力

  • ctx (LogicalDecodingContext*): 論理デコーディングのコンテキスト。出力バッファ ctx->out を含む。
  • data (TestDecodingData*): プラグインの設定情報。include_xids フラグにより XID 出力の有無を制御する。
  • txn (ReorderBufferTXN*): トランザクション情報。xid を保持する。
  • last_write (bool): 出力ストリームの終了フラグ。OutputPluginPrepareWrite および OutputPluginWrite に渡される。

前提条件と不変条件

  • ctx および txn は有効なポインタであること。
  • data->include_xids が真の場合、txn->xid が有効であること。

アルゴリズムまたは振る舞い

1. OutputPluginPrepareWrite(ctx, last_write) を呼び出し、出力の準備を行う。 2. data->include_xids が真であれば、ctx->out に "BEGIN %u"（XID付き）を書き込む。 3. data->include_xids が偽であれば、ctx->out に "BEGIN" を書き込む。 4. OutputPluginWrite(ctx, last_write) を呼び出し、バッファの内容を書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/test_decoding/test_decoding.c 内の各トランザクション処理関数（pg_decode_change 等のコンテキスト内）から、トランザクション開始時に呼び出される。
  • **呼び出し先**:
  • OutputPluginPrepareWrite
  • appendStringInfo / appendStringInfoString
  • OutputPluginWrite

関連データ構造

  • LogicalDecodingContext: 出力プラグインの基盤となるコンテキスト。
  • TestDecodingData: プラグインの設定（include_xids を保持）。
  • ReorderBufferTXN: トランザクションのメタデータ（xid を保持）。

失敗モードと境界条件

  • ctx->out への書き込み時にメモリ不足が発生した場合、PostgreSQL のエラーハンドリング機構により例外がスローされる可能性がある。

保守担当者向けメモ

  • 本関数はトランザクション開始のシリアライズを担当する。出力フォーマットを変更する場合は、include_xids の条件分岐と文字列フォーマットの両方を更新すること。
