Node

定義

  • 種別: typedef
  • 定義箇所: src/include/parser/parse_node.h:92

目的

PostgreSQL のパーサーサブシステムにおいて、構文木やクエリ木を構成する汎用的なデータ構造の基底型として機能します。

シグネチャまたは形状

typedef Node として定義されています。具体的な構造体のメンバ構成は本ヘッダーファイル内では明示されていませんが、PostgreSQL のノードシステムにおいて、すべてのノード型が継承する共通の基底構造体です。

入力と出力

  • 入力: 任意のノード型へのポインタを Node * 型としてキャストして受け取ることが一般的です。
  • 出力: 特定のノード型へキャストして利用されます。

前提条件と不変条件

  • すべての PostgreSQL ノード構造体は、最初のメンバとして NodeTag 型のフィールドを持つ必要があります。
  • Node 型へのポインタは、実行時に IsA() マクロや nodeTag() 関数を用いて、実際のノード型を識別可能です。

アルゴリズムまたは振る舞い

  • 構文解析およびクエリ解析の過程で、ツリー構造を形成するために使用されます。
  • Node 型のポインタを介して、再帰的なツリーの走査や変換が行われます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/file_fdw/file_fdw.c や contrib/hstore/hstore_io.c など、PostgreSQL の拡張モジュールやコア機能の各所で、データ構造をノードとして扱う際に参照されています。
  • 呼び出し先: 本シンボル自体は基底型であるため、特定の関数を呼び出すものではありません。

関連データ構造

  • ParseState: 解析状態を保持する構造体。
  • ColumnRef: 解析対象となる列参照ノード。
  • ParamRef: パラメータ参照ノード。

失敗モードと境界条件

  • 不適切な型へのキャスト（Node から誤った構造体型へのキャスト）は、実行時のセグメンテーションフォールトやデータ破損の原因となります。

保守担当者向けメモ

  • Node は PostgreSQL のノードシステムの根幹です。この型を直接操作する際は、必ず nodeTag() を確認し、適切な型へのキャストを行ってください。
  • 構造体の詳細定義は nodes/nodes.h 等の関連ヘッダーを参照してください。
