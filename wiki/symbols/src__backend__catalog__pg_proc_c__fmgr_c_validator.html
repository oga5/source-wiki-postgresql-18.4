fmgr_c_validator

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_proc.c:788

目的

C言語で記述された関数の妥当性を検証します。具体的には、指定された共有ライブラリファイルが存在し、かつ関数シンボルが解決可能であることを確認します。

シグネチャまたは形状

Datum fmgr_c_validator(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される関数OID（PG_GETARG_OID(0)）。
  • **出力**: なし（PG_RETURN_VOID()）。検証失敗時はエラーをスローします。

前提条件と不変条件

  • CheckFunctionValidatorAccess により、呼び出し元が対象関数を検証する権限を持っている必要があります。
  • pg_proc システムカタログに、対象の関数OIDに対応するレコードが存在する必要があります。

アルゴリズムまたは振る舞い

1. CheckFunctionValidatorAccess を呼び出し、アクセス権限を検証します。権限がない場合は即座に終了します。 2. SearchSysCache1 を使用して pg_proc から対象関数のレコードを取得します。 3. レコードから prosrc（関数シンボル名）と probin（共有ライブラリパス）の属性値を取り出します。 4. load_external_function を呼び出し、指定されたライブラリをロードします。 5. fetch_finfo_record を呼び出し、ライブラリ内の関数シンボルが正しく存在するかを確認します。 6. ReleaseSysCache でキャッシュを解放し、正常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上の直接的な呼び出し元は明示されていませんが、pg_proc.c 内の関数定義登録処理から間接的に呼び出されることが想定されます。
  • **呼び出し先**:
  • CheckFunctionValidatorAccess
  • SearchSysCache1 / ReleaseSysCache
  • SysCacheGetAttrNotNull
  • load_external_function
  • fetch_finfo_record

関連データ構造

  • pg_proc: 関数定義を保持するシステムカタログ。
  • HeapTuple: pg_proc の行データ。

失敗モードと境界条件

  • SearchSysCache1 が対象OIDを見つけられない場合、elog(ERROR) により処理が中断されます。
  • load_external_function または fetch_finfo_record が失敗した場合（ライブラリ不在、シンボル未解決など）、それらの関数内でエラーがスローされ、検証は失敗します。

保守担当者向けメモ

  • 本関数は check_function_bodies フラグの状態に関わらず検証を実行する設計となっています。これは pg_dump によるリストア時の整合性確保を優先しているためです。
