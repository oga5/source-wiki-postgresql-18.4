ss_get_location

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/syncscan.c:253

目的

テーブルに対するシーケンシャルスキャンを開始する際、他のバックエンドとスキャン位置を同期させるための最適な開始ブロック番号を取得します。

シグネチャまたは形状

BlockNumber ss_get_location(Relation rel, BlockNumber relnblocks)

入力と出力

  • **入力**:
  • rel: スキャン対象の Relation 構造体。
  • relnblocks: 対象テーブルの現在の総ブロック数。
  • **出力**:
  • スキャンを開始すべき BlockNumber（0 から relnblocks - 1 の範囲）。

前提条件と不変条件

  • SyncScanLock（LWLock）を排他モードで獲得して共有メモリ上のスキャン位置情報を検索します。
  • 取得した位置が現在のテーブルサイズ（relnblocks）を超えている場合、0 にリセットされます。

アルゴリズムまたは振る舞い

1. SyncScanLock を排他獲得します。 2. ss_search を呼び出し、共有メモリから当該リレーションの現在のスキャン位置を取得します。 3. SyncScanLock を解放します。 4. 取得した位置が relnblocks 以上（テーブルの切り詰め等による無効な値）であれば、開始位置を 0 に補正します。 5. TRACE_SYNCSCAN が定義されている場合、ログに開始位置を出力します。 6. 最終的な開始位置を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/access/table/tableam.c
  • **呼び出し先**:
  • LWLockAcquire / LWLockRelease
  • ss_search
  • RelationGetRelationName（TRACE_SYNCSCAN 有効時）

関連データ構造

  • Relation: スキャン対象のリレーション情報。
  • BlockNumber: ブロック位置を指す型。
  • SyncScanLock: 同期スキャン情報を保護する LWLock。

失敗モードと境界条件

  • **テーブルの切り詰め**: VACUUM 等によりテーブルが縮小された場合、保存されていたスキャン位置が現在のブロック数を超える可能性があります。この場合、関数は安全のため 0 を返します。

保守担当者向けメモ

  • 本関数は ss_search を介して共有メモリ上の LRU リストを更新するため、呼び出し時には排他ロックが必須です。
  • TRACE_SYNCSCAN はデバッグ用であり、本番環境でのパフォーマンス影響を考慮して有効化してください。
