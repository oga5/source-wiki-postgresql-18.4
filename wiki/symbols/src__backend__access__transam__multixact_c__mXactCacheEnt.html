mXactCacheEnt

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/multixact.c:394

目的

MultiXactId に関連付けられたメンバー（トランザクション ID とロック状態のペア）をメモリ上でキャッシュするためのエントリ構造体です。

シグネチャまたは形状

typedef struct mXactCacheEnt
{
    MultiXactId multi;
    int         nmembers;
    dlist_node  node;
    MultiXactMember members[FLEXIBLE_ARRAY_MEMBER];
} mXactCacheEnt;

入力と出力

  • multi: キャッシュ対象の MultiXactId。
  • nmembers: members 配列に格納されている MultiXactMember の数。
  • members: 可変長配列として保持されるメンバー情報のリスト。

前提条件と不変条件

  • members は FLEXIBLE_ARRAY_MEMBER を使用しており、構造体インスタンスの末尾に動的に割り当てられたメモリ領域として存在します。
  • node を介して二重連結リスト（dlist）の一部として管理されます。

アルゴリズムまたは振る舞い

  • GetMultiXactIdMembers が MultiXactId のメンバー情報を取得する際、まずこの構造体を用いたローカルキャッシュを確認します。
  • キャッシュミスが発生した場合、SLRU から情報を読み取り、新たに mXactCacheEnt を割り当ててキャッシュに追加します。
  • 構造体のサイズ計算には offsetof(mXactCacheEnt, members) が使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/multixact.c 内のキャッシュ管理関数群（GetMultiXactIdMembers 等）。
  • 呼び出し先: dlist 操作関数（dclist_container 等）がこの構造体をリストから取得するために使用します。

関連データ構造

  • MultiXactMember: メンバー情報を構成する個別の要素。
  • dlist_node: キャッシュリスト管理用のノード。

失敗モードと境界条件

  • メモリ不足時: mXactCacheEnt の動的確保に失敗する可能性があります。
  • キャッシュの肥大化: 多数の MultiXactId が参照される場合、キャッシュエントリの管理がメモリ消費に影響します。

保守担当者向けメモ

  • 本構造体は FLEXIBLE_ARRAY_MEMBER を含んでいるため、インスタンス生成時には sizeof ではなく offsetof を用いたサイズ計算が必須です。
  • キャッシュの有効期間やエントリの破棄タイミングについては、multixact.c 内のキャッシュ管理ロジックを参照してください。
