oidparse

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:263

目的

パーサーが生成したノード（T_Integer または T_Float）から Oid 値を抽出します。

シグネチャまたは形状

Oid oidparse(Node *node)

入力と出力

  • **入力**: Node *node - 抽出対象のノード。
  • **出力**: Oid - 抽出されたオブジェクト識別子。

前提条件と不変条件

  • 入力ノードは T_Integer または T_Float である必要があります。
  • T_Float の場合、その値は uint32in_subr が許容する有効な OID 文字列として解釈可能である必要があります。

アルゴリズムまたは振る舞い

1. nodeTag(node) を判定します。 2. T_Integer の場合、intVal(node) を呼び出し、その値を Oid として返します。 3. T_Float の場合、castNode(Float, node)->fval を取得し、uint32in_subr を介して Oid に変換して返します。これは int4 に収まらない大きな値を扱うための処理です。 4. 上記以外のノード型が渡された場合、elog(ERROR) によりエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/objectaddress.c:1048
  • **呼び出し先**:
  • intVal (マクロ)
  • castNode (マクロ)
  • uint32in_subr
  • elog

関連データ構造

  • Node: 構文解析ツリーの基本構造体。
  • Oid: PostgreSQL のオブジェクト識別子型。

失敗モードと境界条件

  • 未知のノード型が渡された場合、elog(ERROR) により処理が中断されます。
  • T_Float の値が uint32in_subr で解析できない場合、uint32in_subr 内部のロジックに従いエラーが発生します。

保守担当者向けメモ

  • T_Float を許容しているのは、レクサーが int4 の範囲を超える数値を浮動小数点定数として扱うためです。
  • InvalidOid を返すコードパスは、default ケースで elog がエラーを投げるため到達不能ですが、コンパイラの警告抑制のために存在します。
