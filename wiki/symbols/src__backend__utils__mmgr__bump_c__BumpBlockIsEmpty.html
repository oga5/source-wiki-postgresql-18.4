BumpBlockIsEmpty

定義

  • 種別: static inline function
  • 定義箇所: src/backend/utils/mmgr/bump.c:551

目的

指定された BumpBlock がメモリ割り当てに使用されていない（空である）かどうかを判定する。

シグネチャまたは形状

static inline bool BumpBlockIsEmpty(BumpBlock *block)

入力と出力

  • **入力**: BumpBlock *block （検査対象のバンプブロックへのポインタ）
  • **出力**: bool （ブロックが空であれば true、そうでなければ false）

前提条件と不変条件

  • block は有効な BumpBlock 構造体を指している必要がある。
  • Bump_BLOCKHDRSZ はブロックヘッダーのサイズとして定義されている。

アルゴリズムまたは振る舞い

ブロック内の現在の割り当て位置を示す freeptr が、ブロックヘッダー直後のアドレス（初期位置）と一致するかどうかを比較する。

  • freeptr == ((char *) block + Bump_BLOCKHDRSZ) が真であれば、ブロックは一度も割り当てに使用されていないと判断する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mmgr/bump.c:671 にて、ブロックが空であるかの確認に使用されている。
  • **呼び出し先**: 本関数は他の関数を呼び出さない。

関連データ構造

  • BumpBlock: バンプアロケータが管理するメモリブロックの構造体。

失敗モードと境界条件

  • block が NULL の場合、ポインタ参照によりクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は static inline で定義されており、コンパイル時に展開されることを想定している。
  • freeptr の値のみで判定しているため、一度割り当てられた後に BumpReset 等でリセットされたブロックの状態については、呼び出し側のコンテキスト管理に依存する。
