ConditionVariableSignal

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/condition_variable.c:258

目的

条件変数で待機しているプロセスのうち、先頭の1つを起床させる。

シグネチャまたは形状

void ConditionVariableSignal(ConditionVariable *cv)

入力と出力

  • **入力**: cv (操作対象の ConditionVariable 構造体へのポインタ)
  • **出力**: なし

前提条件と不変条件

  • cv は初期化済みであること。
  • 内部の wakeup リスト（proclist）はスピンロックによって保護されている。

アルゴリズムまたは振る舞い

1. cv->mutex スピンロックを取得する。 2. cv->wakeup リストが空でない場合、リストの先頭からプロセス（PGPROC）を取り出す。 3. cv->mutex スピンロックを解放する。 4. 取り出したプロセスが NULL でない場合、そのプロセスの procLatch に対して SetLatch を呼び出し、待機状態を解除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtree.c
  • src/backend/access/nbtree/nbtsort.c
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease
  • proclist_is_empty / proclist_pop_head_node
  • SetLatch

関連データ構造

  • ConditionVariable: 待機リストとスピンロックを保持する構造体。
  • PGPROC: 待機中のプロセス情報を保持する構造体。
  • proclist: 待機プロセスを管理するリスト構造。

失敗モードと境界条件

  • 待機リストが空の場合、何も行わず正常終了する。
  • スピンロック取得中に割り込みが発生した場合の挙動は、スピンロックの仕様に依存する。

保守担当者向けメモ

  • 本関数は単一のプロセスのみを起床させる。全プロセスを起床させる必要がある場合は ConditionVariableBroadcast を使用すること。
  • SetLatch を呼び出すことで、待機中のプロセスが WaitLatch から復帰するトリガーとなる。
