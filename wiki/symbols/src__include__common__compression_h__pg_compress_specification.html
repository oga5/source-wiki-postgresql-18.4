pg_compress_specification

定義

  • 種別: struct
  • 定義箇所: src/include/common/compression.h:32

目的

PostgreSQLのバックエンドおよびフロントエンドツール（pg_basebackup等）において、圧縮アルゴリズムとその実行パラメータを一元管理するための構造体です。

シグネチャまたは形状

typedef struct pg_compress_specification
{
	pg_compress_algorithm algorithm;
	unsigned	options;        /* PG_COMPRESSION_OPTION定数のOR */
	int			level;
	int			workers;
	bool		long_distance;
	char	   *parse_error;    /* パース成功時はNULL、失敗時はエラーメッセージ */
} pg_compress_specification;

入力と出力

  • **入力**: parse_compress_specification 関数により、文字列形式の圧縮設定が本構造体の各フィールドにマッピングされます。
  • **出力**: 確定した構造体は、各圧縮ライブラリ（zlib, lz4, zstd）のAPI呼び出し時の設定値として利用されます。

前提条件と不変条件

  • parse_error フィールドが非NULLの場合、パース処理が失敗したことを示します。このメモリは呼び出し側で pfree を用いて適切に解放する必要があります。
  • 特定のアルゴリズム（LZ4やZstd）を使用する場合、ビルド時の構成（--with-lz4 等）が有効である必要があります。

アルゴリズムまたは振る舞い

1. **パース**: parse_compress_specification がユーザー指定の文字列を解析し、構造体を初期化します。 2. **検証**: validate_compress_specification により、設定されたレベルやオプションが選択されたアルゴリズムに対して妥当か確認されます。 3. **適用**: 検証済みの構造体に基づき、バックアップ等の処理で圧縮ライブラリが呼び出されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/backend/backup/basebackup_gzip.c
  • src/backend/backup/basebackup_lz4.c
  • src/backend/backup/basebackup_zstd.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • pg_compress_algorithm: 圧縮アルゴリズムを識別する列挙型。

失敗モードと境界条件

  • parse_error にエラーメッセージが格納される場合、呼び出し側でのメモリ解放漏れに注意が必要です。
  • アルゴリズムごとの許容レベルやオプションの範囲外を指定した場合、検証フェーズでエラーとなります。

保守担当者向けメモ

  • pg_compress_algorithm の列挙順序はディスク上のデータ形式に直結するため、変更や順序の入れ替えは厳禁です。
  • 本構造体はフロントエンドとバックエンドで共有されるため、変更時は両方の互換性に配慮してください。
