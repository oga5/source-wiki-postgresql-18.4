transformCreateTableAsStmt

定義

  • 種別: function
  • 定義箇所: src/backend/parser/analyze.c:3177

目的

CREATE TABLE AS または CREATE MATERIALIZED VIEW 文の解析ツリーを、実行可能なユーティリティクエリツリーへ変換する。

シグネチャまたは形状

Query *transformCreateTableAsStmt(ParseState *pstate, CreateTableAsStmt *stmt)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体。
  • stmt: 変換対象の CreateTableAsStmt 構造体。
  • **出力**:
  • Query: commandType が CMD_UTILITY に設定され、utilityStmt に元の stmt が格納されたクエリツリー。

前提条件と不変条件

  • stmt->query に格納されたクエリは transformStmt によって変換される。
  • OBJECT_MATVIEW（マテリアライズド・ビュー）の場合、以下の制約が課される:
  • データ変更を伴う CTE（hasModifyingCTE）は禁止。
  • 一時テーブルや一時ビューの使用は禁止。
  • 外部パラメータ（query_contains_extern_params）の使用は禁止。
  • RELPERSISTENCE_UNLOGGED（非ログ記録）設定は禁止。

アルゴリズムまたは振る舞い

1. transformStmt を呼び出し、stmt->query をクエリツリーに変換する。 2. stmt->objtype が OBJECT_MATVIEW の場合、上記の前提条件に基づきバリデーションを行い、違反があればエラーを報告する。 3. マテリアライズド・ビューの場合、後続の ON SELECT ルール作成で使用するため、変換後のクエリのコピーを stmt->into->viewQuery に保存する。 4. 最終的に makeNode(Query) で新しい Query ノードを作成し、commandType を CMD_UTILITY に設定して utilityStmt に stmt を格納して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/analyze.c 内の transformStmt（推測されるディスパッチャー）。
  • **呼び出し先**:
  • transformStmt: 内部のクエリ変換用。
  • isQueryUsingTempRelation: マテリアライズド・ビューの制約チェック用。
  • query_contains_extern_params: マテリアライズド・ビューの制約チェック用。
  • copyObject: ビュー定義の保存用。
  • makeNode: クエリノード生成用。
  • ereport: エラー報告用。

関連データ構造

  • ParseState: 解析のコンテキスト。
  • CreateTableAsStmt: CREATE TABLE AS 文の構文解析結果。
  • Query: 解析後のクエリツリー。
  • IntoClause: CREATE TABLE AS のターゲット情報。

失敗モードと境界条件

  • マテリアライズド・ビュー作成時に、データ変更CTE、一時オブジェクト、外部パラメータ、または非ログ記録（UNLOGGED）設定が含まれている場合、ERRCODE_FEATURE_NOT_SUPPORTED でエラー終了する。

保守担当者向けメモ

  • 本関数は CREATE TABLE AS と CREATE MATERIALIZED VIEW の両方を扱う。マテリアライズド・ビュー特有の制約は stmt->objtype で分岐している。
  • stmt->into->viewQuery へのコピーは、後の intorel_startup() で利用されるため、この代入を削除してはならない。
