initcap_wbnext

定義

  • 種別: function
  • 定義箇所: src/common/unicode/case_test.c:49

目的

タイトルケース変換（initcap）処理において、文字列内の単語境界を逐次的に特定するためのイテレータ関数です。

シグネチャまたは形状

static size_t initcap_wbnext(void *state)

入力と出力

  • **入力**: void *state（WordBoundaryState 構造体へのポインタとしてキャストして使用）
  • **出力**: size_t 型のオフセット値を返します。これは次に処理すべき単語境界の開始位置、または文字列の終端（wbstate->len）を示します。

前提条件と不変条件

  • state は WordBoundaryState 構造体として初期化されている必要があります。
  • wbstate->str は有効な UTF-8 文字列であると想定されています。

アルゴリズムまたは振る舞い

1. 現在のオフセットから文字列を走査し、各文字を utf8_to_unicode でデコードします。 2. pg_u_isalnum を用いて、現在の文字が英数字（alnum）かどうかを判定します。 3. 以下のいずれかの場合に、現在のオフセットを単語境界として返します：

  • 初回呼び出し時（wbstate->init が false）。
  • 前の文字との英数字属性（prev_alnum）が変化したとき。

    • 4. 属性の変化を検知した際、wbstate->prev_alnum を更新し、wbstate->init を true に設定します。 5. 文字列の終端に達した場合は、wbstate->len を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale_builtin.c
  • src/common/unicode/case_test.c（unicode_strtitle の引数として使用）
  • **呼び出し先**:
  • utf8_to_unicode
  • pg_u_isalnum
  • unicode_utf8len

関連データ構造

  • WordBoundaryState: 単語境界判定の状態を保持する構造体。

失敗モードと境界条件

  • 文字列が空の場合や、終端に達した場合は wbstate->len を返して終了します。
  • 不正な UTF-8 シーケンスが含まれる場合の挙動は、依存する utf8_to_unicode の実装に依存します。

保守担当者向けメモ

  • 本関数は unicode_strtitle にコールバックとして渡されることを前提としています。
  • wbstate->posix フラグにより、ロケール依存の判定ロジックが切り替わる設計となっています。
