r_remove_category_3

目的

ネパール語のステミング処理において、カテゴリ3に分類される特定の接尾辞を単語末尾から識別し、削除する。

シグネチャまたは形状

static int r_remove_category_3(struct SN_env * z)

入力と出力

  • **入力**: SN_env * z（Snowballランタイム環境へのポインタ）
  • **出力**: int 型。接尾辞が削除された場合は 1 を返し、マッチしなかった場合は 0 を返す。エラー発生時は負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体を指している必要がある。
  • 処理は単語の末尾から前方に向かって行われる。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置（z->c）を z->ket（削除対象の末尾）に設定する。 2. find_among_b を使用し、テーブル a_3（91個の要素）を参照して、単語末尾から後方検索を行う。 3. マッチした場合、現在のカーソル位置を z->bra（削除対象の先頭）に設定する。 4. slice_del(z) を呼び出し、マッチした接尾辞をバッファから物理的に削除する。 5. slice_del の戻り値が負であれば、その値をそのまま返す。正常終了時は 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_nepali.c 内のメインステミング処理フローから呼び出される。
  • **呼び出し先**: find_among_b（ランタイム関数）、slice_del（ランタイム関数）。

関連データ構造

  • SN_env: Snowballランタイムの環境状態を保持する構造体。
  • a_3: カテゴリ3の接尾辞パターンを保持する among 構造体の配列。

失敗モードと境界条件

  • find_among_b がマッチに失敗した場合、0 を返して終了する。
  • slice_del が内部エラー（メモリ操作失敗等）で負の値を返した場合、そのエラーコードを伝播させる。
