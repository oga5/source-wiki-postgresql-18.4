pg_localeconv_free

定義

  • 種別: function
  • 定義箇所: src/port/pg_localeconv_r.c:103

目的

pg_localeconv_r() によって動的に確保された struct lconv 構造体内の文字列メンバを解放し、メモリリークを防ぐ。

シグネチャまたは形状

void pg_localeconv_free(struct lconv *lconv)

入力と出力

  • **入力**: lconv (初期化済みの struct lconv へのポインタ)
  • **出力**: なし（副作用としてメモリを解放）

前提条件と不変条件

  • 引数 lconv は、pg_localeconv_r() によって生成された有効な構造体であること。
  • 内部的に定義された table 配列のメタデータに基づき、文字列型としてマークされたメンバのみが free() の対象となる。

アルゴリズムまたは振る舞い

1. table 配列を走査し、各メンバのメタデータを確認する。 2. is_string フラグが真であるメンバについて、lconv_string_member() を用いて対象の文字列ポインタを取得する。 3. 取得したポインタに対して free() を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale.c:585
  • src/port/pg_localeconv_r.c:134
  • src/port/pg_localeconv_r.c:171
  • **呼び出し先**:
  • lconv_string_member (インライン関数)
  • free (標準ライブラリ)

関連データ構造

  • struct lconv: C 標準ライブラリのロケール情報構造体。
  • table: pg_localeconv_r.c 内で定義された、lconv 構造体の各メンバの管理情報を保持する内部配列。

失敗モードと境界条件

  • lconv が NULL の場合の挙動は、lconv_string_member の実装に依存する（ソース excerpt には明示なし）。
  • 既に解放済みのポインタを渡した場合の二重解放については、呼び出し側の管理責任となる。

保守担当者向けメモ

  • 本関数は pg_localeconv_r() と対になるメモリ管理関数である。table 配列の構造に変更を加える際は、本関数の走査ロジックも同期させる必要がある。
