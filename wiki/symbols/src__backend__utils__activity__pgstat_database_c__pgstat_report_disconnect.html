pgstat_report_disconnect

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_database.c:243

目的

セッション終了時の原因に基づき、データベース単位の統計情報（切断関連の異常終了カウント）を更新する。

シグネチャまたは形状

void pgstat_report_disconnect(Oid dboid)

入力と出力

  • **入力**: dboid (Oid) - 統計を更新する対象データベースの識別子。
  • **出力**: なし。副作用として、内部の統計エントリが更新される。

前提条件と不変条件

  • pgstat_should_report_connstat() が真を返す場合にのみ統計更新が行われる。
  • 統計更新は、グローバル変数 pgStatSessionEndCause の値に基づいて行われる。

アルゴリズムまたは振る舞い

1. pgstat_should_report_connstat() を確認し、偽であれば即座に終了する。 2. pgstat_prep_database_pending(dboid) を呼び出し、対象データベースの統計エントリを取得する。 3. pgStatSessionEndCause の値に応じて以下の通り統計を更新する:

  • DISCONNECT_CLIENT_EOF: sessions_abandoned をインクリメント。
  • DISCONNECT_FATAL: sessions_fatal をインクリメント。
  • DISCONNECT_KILLED: sessions_killed をインクリメント。
  • DISCONNECT_NOT_YET または DISCONNECT_NORMAL: 何も行わない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c:614 (確認済み)
  • **呼び出し先**:
  • pgstat_should_report_connstat() (内部チェック)
  • pgstat_prep_database_pending() (統計エントリ取得)

関連データ構造

  • PgStat_StatDBEntry: データベース単位の統計情報を保持する構造体。
  • pgStatSessionEndCause: セッション終了理由を保持するグローバル変数。

失敗モードと境界条件

  • pgstat_should_report_connstat() が偽の場合、統計は記録されない。
  • pgStatSessionEndCause が DISCONNECT_NOT_YET または DISCONNECT_NORMAL の場合、カウントは更新されない。

保守担当者向けメモ

  • 本関数はセッション終了時の統計報告に特化している。
  • 統計情報の永続化や共有メモリへの同期は、この関数が呼び出された後のプロセス終了処理やフラッシュ処理に依存する。
