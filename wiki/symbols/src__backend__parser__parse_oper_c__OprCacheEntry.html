OprCacheEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_oper.c:59

目的

演算子検索結果をキャッシュするためのハッシュテーブルエントリ構造体です。検索パスと引数型から導出された演算子の OID を保持し、再検索のコストを削減します。

シグネチャまたは形状

typedef struct OprCacheEntry
{
    OprCacheKey key;    /* ハッシュ検索キー (先頭必須) */
    Oid         opr_oid; /* 解決された演算子の OID */
} OprCacheEntry;

入力と出力

  • **入力**: OprCacheKey（検索キー）および解決された Oid。
  • **出力**: OprCacheHash ハッシュテーブル内のエントリとして格納されます。

前提条件と不変条件

  • key フィールドは構造体の先頭に配置しなければなりません（ハッシュテーブル検索の仕様）。
  • OprCacheHash に格納される際、この構造体のサイズが ctl.entrysize として指定されます。

アルゴリズムまたは振る舞い

  • hash_search を用いて OprCacheHash から検索・挿入されます。
  • キャッシュミス時に解決された演算子情報が本構造体に書き込まれ、キャッシュへ登録されます。
  • hash_seq_search を用いて、キャッシュ内の全エントリを走査することが可能です。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_oper.c 内の演算子解決ロジック（make_oper_cache_entry 等のキャッシュ操作関数経由）。
  • **呼び出し先**: 直接的な関数呼び出しはありません。データ構造として hash_search や hash_seq_search 等のハッシュテーブル操作関数によって操作されます。

関連データ構造

  • OprCacheKey: ハッシュ検索のキーとなる構造体。
  • OprCacheHash: 本構造体を要素として保持するハッシュテーブル。

失敗モードと境界条件

  • メモリ不足時には hash_search が失敗する可能性があります。
  • pg_operator や pg_cast の変更に伴い InvalidateOprCacheCallBack が実行されると、本構造体を含むキャッシュ全体がフラッシュされます。

保守担当者向けメモ

  • 本構造体にフィールドを追加する場合は、key が必ず先頭にあることを維持してください。
  • キャッシュの無効化ロジックは InvalidateOprCacheCallBack に依存しているため、キャッシュの整合性に関する変更を行う際は当該関数を確認してください。
