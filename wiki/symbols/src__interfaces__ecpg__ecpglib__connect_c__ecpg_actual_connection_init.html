ecpg_actual_connection_init

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/connect.c:23

目的

スレッド固有のデータ（TSD）としてカレント接続を保持するための pthread_key_t を初期化します。

シグネチャまたは形状

static void ecpg_actual_connection_init(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として actual_connection_key を初期化）

前提条件と不変条件

  • pthread ライブラリが利用可能であること。
  • 本関数は pthread_once を介して一度だけ実行されることが想定されています。

アルゴリズムまたは振る舞い

1. pthread_key_create を呼び出し、グローバル変数 actual_connection_key にスレッド固有のキーを割り当てます。 2. デストラクタ関数には NULL を指定しており、スレッド終了時の自動的なクリーンアップは行われません。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/ecpglib/connect.c 内の pthread_once(&actual_connection_key_once, ecpg_actual_connection_init) により呼び出されます。
  • 呼び出し先: pthread_key_create（POSIXスレッドライブラリ）。

関連データ構造

  • actual_connection_key: スレッドごとのカレント接続を保持するために使用される pthread_key_t 型のグローバル変数。

失敗モードと境界条件

  • pthread_key_create が失敗した場合の挙動については、本ソースコード内では明示的なエラーハンドリングは記述されていません。

保守担当者向けメモ

  • 本関数は static であり、connect.c 内でのみ使用されます。
  • スレッドセーフな接続管理の基盤となるため、キーの生成タイミング（pthread_once）を変更する際は注意が必要です。
