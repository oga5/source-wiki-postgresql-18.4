IndexDeleteCounts

定義

  • 種別: struct
  • 定義箇所: src/backend/access/heap/heapam.c:194

目的

インデックス削除処理において、特定のブロックグループ内に含まれるTID（Tuple Identifier）の統計情報を保持し、削除効率を最適化するために使用されます。

シグネチャまたは形状

typedef struct IndexDeleteCounts
{
    int16       npromisingtids; /* グループ内の「有望な」TID数 */
    int16       ntids;          /* グループ内のTID総数 */
    int16       ifirsttid;      /* グループの最初の削除TIDへのオフセット */
} IndexDeleteCounts;

入力と出力

本構造体は関数への引数として渡されるほか、palloc を用いて配列としてメモリ確保されます。主にインデックス削除の最適化ロジック内で、グループごとの統計情報として入出力されます。

前提条件と不変条件

  • ntids はグループ内のTID総数を表します。
  • npromisingtids は ntids 以下の値をとります。
  • ifirsttid は、関連する削除TID配列内でのインデックス位置を示します。

アルゴリズムまたは振る舞い

インデックス削除処理において、TIDをブロックごとにグループ化する際に使用されます。bottomup_nblocksfavorable 関数などでグループの優先度判定に利用されるほか、qsort を用いたソート処理の対象としても利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c 内のインデックス削除関連ロジック（bottomup_nblocksfavorable 等）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • IndexDeletePrefetchState: インデックス削除時のプリフェッチ状態を管理する構造体。
  • delstate (構造体内部で参照される削除状態管理用データ)。

失敗モードと境界条件

  • palloc によるメモリ確保失敗時にエラーが発生する可能性があります。
  • ntids や npromisingtids の値が int16 の範囲を超えないことが前提となります。

保守担当者向けメモ

  • 本構造体は heapam.c 内のインデックス削除最適化（ボトムアップ削除）に特化した内部構造体です。
  • 構造体のフィールド変更は、bottomup_nblocksfavorable および関連するソート関数のロジックに影響を与えるため注意してください。
