is_valid_option

定義

  • 種別: function
  • 定義箇所: contrib/file_fdw/file_fdw.c:359

目的

指定されたオプション名が、特定のコンテキスト（Oid）において有効な file_fdw オプションであるかを判定します。

シグネチャまたは形状

static bool is_valid_option(const char *option, Oid context)

入力と出力

  • **入力**:
  • option: 検証対象のオプション名（文字列）。
  • context: オプションが定義されたカタログの Oid（例: ForeignTableRelationId 等）。
  • **出力**:
  • bool: オプション名とコンテキストの組み合わせが valid_options 配列内に存在する場合は true、それ以外は false を返します。

前提条件と不変条件

  • valid_options という名前の静的配列がスコープ内に定義されており、各要素は optname（文字列）と optcontext（Oid）を持つ FileFdwOption 構造体である必要があります。
  • valid_options 配列は、optname が NULL である要素を終端として持つ必要があります。

アルゴリズムまたは振る舞い

1. valid_options 配列を先頭から順に走査します。 2. 各要素について、optname が NULL に達するまでループします。 3. 現在の要素の optcontext が引数の context と一致し、かつ optname が引数の option と一致（strcmp）する場合、直ちに true を返します。 4. 配列の最後まで一致が見つからなかった場合、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: file_fdw_validator（contrib/file_fdw/file_fdw.c:225 付近）から呼び出され、ユーザーが指定したオプションの妥当性を検証します。
  • **呼び出し先**: 標準ライブラリの strcmp を使用します。

関連データ構造

  • FileFdwOption: オプション名と適用可能なコンテキストを保持する構造体。

失敗モードと境界条件

  • option が NULL の場合、strcmp によるセグメンテーション違反が発生する可能性があります。
  • valid_options 配列が適切に終端されていない場合、メモリ境界外アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数は contrib/postgres_fdw/option.c に同名の関数が存在しますが、それぞれ独立した実装です。修正時は file_fdw 用の valid_options 定義のみを参照していることを確認してください。
