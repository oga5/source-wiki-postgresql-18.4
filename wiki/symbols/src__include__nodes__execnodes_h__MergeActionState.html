MergeActionState

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/execnodes.h:440

目的

SQLの MERGE 文における各アクション（INSERT, UPDATE, DELETE）の実行時状態を保持します。プランノードが持つ静的な定義を、実行エンジンが処理可能な動的状態へ変換したものです。

シグネチャまたは形状

typedef struct MergeActionState
{
	NodeTag		type;
	MergeAction *mas_action;	/* 関連するMergeActionノード */
	ProjectionInfo *mas_proj;	/* アクションのターゲットリストの射影情報 */
	ExprState  *mas_whenqual;	/* WHEN [NOT] MATCHED AND 条件式 */
} MergeActionState;

入力と出力

  • **入力**: makeNode(MergeActionState) を用いて生成され、MergeAction ノードの情報に基づいて初期化されます。
  • **出力**: 実行エンジン（nodeModifyTable.c 等）が MERGE アクションを評価する際に、射影（mas_proj）や条件判定（mas_proj）の結果を提供します。

前提条件と不変条件

  • mas_action は対応する MergeAction プランノードを指す必要があります。
  • mas_proj および mas_whenqual は、実行開始時に適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

  • MERGE 処理中、各アクションに対して MergeActionState が生成されます。
  • mas_whenqual は WHEN [NOT] MATCHED AND 句で指定された条件を評価するために使用されます。
  • mas_proj は、アクションが実行される際にターゲットリレーションへ挿入または更新する値を計算するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeModifyTable.c: MERGE アクションの初期化および処理時に参照。
  • src/backend/executor/execPartition.c: パーティション分割されたテーブルに対する MERGE 処理時に生成・参照。
  • src/backend/executor/execExprInterp.c: 式評価の文脈で参照。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありませんが、保持する ExprState や ProjectionInfo を通じて式評価エンジンや射影エンジンが呼び出されます。

関連データ構造

  • MergeAction: 対応する静的なプランノード。
  • ProjectionInfo: ターゲットリストの射影を管理。
  • ExprState: WHEN 句の条件式評価状態。

失敗モードと境界条件

  • makeNode によるメモリ確保失敗時はシステム全体のエラーとなります。
  • mas_whenqual が NULL の場合、条件なし（常に真）として扱われます。

保守担当者向けメモ

  • 本構造体は実行時専用であり、copyObject や equal といったノードサポート関数はサポートしていません。
  • MERGE 処理の複雑化に伴い、mas_action 経由でアクセスする情報が増える可能性があるため、変更時は nodeModifyTable.c の整合性に注意してください。
