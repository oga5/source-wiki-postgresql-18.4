typeInheritsFrom

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_inherits.c:405

目的

指定された型（subclassTypeId）が、別の型（superclassTypeId）を継承しているかどうかを判定する。

シグネチャまたは形状

bool typeInheritsFrom(Oid subclassTypeId, Oid superclassTypeId)

入力と出力

  • **入力**:
  • subclassTypeId: 継承元（子）の型OID。
  • superclassTypeId: 継承先（親）の型OID。
  • **出力**:
  • bool: 継承関係にある場合は true、それ以外は false。

前提条件と不変条件

  • 入力された型OIDから対応するリレーションOID（subclassRelid, superclassRelid）が取得できる必要がある。
  • pg_inherits カタログに対する AccessShareLock を内部で取得・解放する。

アルゴリズムまたは振る舞い

1. 入力された型OIDを typeOrDomainTypeRelid および typeidTypeRelid でリレーションOIDに変換する。変換失敗時は false を返す。 2. has_subclass を使用し、親候補にサブクラスが存在しない場合は即座に false を返す。 3. 幅優先探索（BFS）を用いて継承グラフを走査する。

  • queue に subclassRelid を初期値として投入。
  • visited リストで訪問済みリレーションを管理し、無限ループや重複処理を防止する。

    • 4. pg_inherits カタログを InheritsRelidSeqnoIndexId でスキャンし、現在のリレーションが継承している親（inhparent）を順次取得する。 5. inhparent が superclassTypeId に一致すれば true を返し、そうでなければその親を queue に追加して探索を継続する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_coerce.c (型変換の妥当性チェック)
  • **呼び出し先**:
  • typeOrDomainTypeRelid, typeidTypeRelid
  • has_subclass
  • table_open, table_close
  • systable_beginscan, systable_getnext, systable_endscan

関連データ構造

  • pg_inherits: 継承関係を保持するシステムカタログ。
  • List: 探索用の queue および visited リストとして使用。

失敗モードと境界条件

  • 型OIDが複雑型やドメインでない場合、InvalidOid が返され、関数は false を返す。
  • 継承グラフに循環が含まれる場合、visited リストによるチェックで無限ループを回避する。

保守担当者向けメモ

  • 本関数は AccessShareLock を保持してカタログをスキャンするため、並行的な DDL 操作によるカタログ変更の影響を受ける可能性がある。
  • has_subclass は高速なチェック用であり、真を返しても必ずしも継承関係が存在するとは限らない（過去の継承関係を含むため）。本関数は pg_inherits を直接スキャンすることで正確な判定を行う。
