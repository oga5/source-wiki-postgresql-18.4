report_status

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/util.c:31

目的

pg_upgrade において、指定されたログタイプに基づいてメッセージをログ出力するためのインターフェースを提供します。

シグネチャまたは形状

void report_status(eLogType type, const char *fmt, ...);

入力と出力

  • **入力**:
  • type: 出力するログの重要度や種類を指定する eLogType 列挙型。
  • fmt: printf 形式のフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（void）。

前提条件と不変条件

  • pg_log_v 関数が内部で呼び出されるため、メッセージの末尾に改行を含めないことが前提となります。
  • 可変長引数は va_start および va_end によって適切に処理されます。

アルゴリズムまたは振る舞い

1. va_start を使用して可変長引数リストを初期化します。 2. 内部関数 pg_log_v を呼び出し、ログタイプとフォーマット文字列、引数リストを渡して出力処理を委譲します。 3. va_end を使用して引数リストを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_upgrade/check.c
  • src/bin/pg_upgrade/exec.c
  • src/bin/pg_upgrade/tablespace.c
  • **呼び出し先**:
  • pg_log_v (同ファイル内)

関連データ構造

  • eLogType: ログの出力先や挙動を決定する列挙型（pg_upgrade.h で定義）。

失敗モードと境界条件

  • pg_log_v の実装に依存します。フォーマット文字列と引数の不整合がある場合、標準的な printf 系の動作に従います。

保守担当者向けメモ

  • 本関数は pg_attribute_printf(2, 3) 属性が付与されており、コンパイル時にフォーマット文字列のチェックが行われます。
  • ログ出力の詳細は pg_log_v および log_opts グローバル変数の設定に依存します。
