AppendStringToManifest

定義

  • 種別: function
  • 定義箇所: src/backend/backup/backup_manifest.c:383

目的

バックアップ・マニフェストの構築において、指定された文字列を一時ファイルへ書き出し、同時にマニフェスト全体のチェックサム計算を更新する。

シグネチャまたは形状

static void AppendStringToManifest(backup_manifest_info *manifest, const char *s)

入力と出力

  • **入力**:
  • manifest: マニフェスト生成状態を保持する backup_manifest_info 構造体へのポインタ。
  • s: 書き出す対象の文字列。
  • **出力**: なし（副作用として一時ファイルへの書き込みと構造体の状態更新を行う）。

前提条件と不変条件

  • manifest は NULL であってはならない（Assert による検証）。
  • manifest->buffile が有効な BufFile として初期化されていること。

アルゴリズムまたは振る舞い

1. 入力文字列 s の長さを取得する。 2. manifest->still_checksumming が真の場合、pg_cryptohash_update を用いてマニフェスト全体のSHA-256チェックサム計算を更新する。更新に失敗した場合は ERROR を投げる。 3. BufFileWrite を使用して、文字列 s を一時ファイル（manifest->buffile）に書き込む。 4. manifest->manifest_size に書き込んだ長さを加算し、累積サイズを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/backup_manifest.c 内の各関数（マニフェストのJSON構造を構築する箇所で頻繁に呼び出される）。
  • **呼び出し先**:
  • pg_cryptohash_update
  • BufFileWrite
  • pg_cryptohash_error（エラー発生時）

関連データ構造

  • backup_manifest_info: マニフェスト生成のコンテキスト（一時ファイル、チェックサム計算状態、累積サイズを保持）。

失敗モードと境界条件

  • pg_cryptohash_update が負の値を返した場合、elog(ERROR) により処理が中断される。
  • 文字列 s が空の場合、長さ0として処理される。

保守担当者向けメモ

  • 本関数はマニフェスト構築の基盤であり、JSONの構文（カンマや括弧など）の整合性は呼び出し側で制御されている。
  • マニフェスト全体のチェックサム計算は still_checksumming フラグによって制御されており、終了処理の直前まで継続される。
