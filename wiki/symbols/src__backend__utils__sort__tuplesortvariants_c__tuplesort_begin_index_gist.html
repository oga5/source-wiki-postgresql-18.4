tuplesort_begin_index_gist

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:491

目的

GiSTインデックス構築時のソート処理を行うための Tuplesortstate を初期化します。

シグネチャまたは形状

Tuplesortstate *tuplesort_begin_index_gist(Relation heapRel, Relation indexRel, int workMem, SortCoordinate coordinate, int sortopt)

入力と出力

  • **入力**:
  • heapRel: ソート対象のヒープリレーション。
  • indexRel: GiSTインデックスリレーション。
  • workMem: ソートに使用可能なメモリ量（KB）。
  • coordinate: 並列ソート等のための調整用構造体。
  • sortopt: ソートオプション（TUPLESORT_RANDOMACCESS 等）。
  • **出力**:
  • 初期化済みの Tuplesortstate 構造体へのポインタ。

前提条件と不変条件

  • base->nKeys は indexRel のキー属性数に設定されます。
  • base->sortKeys 配列が属性数分確保され、各列に対して PrepareSortSupportFromGistIndexRel が呼び出されます。
  • base->haveDatum1 は true に設定されます。

アルゴリズムまたは振る舞い

1. tuplesort_begin_common を呼び出し、共通のソート状態を初期化します。 2. TuplesortIndexBTreeArg 構造体を割り当て、インデックス情報を格納します。 3. Tuplesortstate の関数ポインタをGiSTインデックス用の実装（comparetup_index_btree 等）で上書きします。 4. 各インデックス列に対して SortSupport を準備し、照合順序（Collation）や属性番号を設定します。 5. 第1キーに対してのみ、省略キー（Abbreviated Key）最適化を有効化します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistbuild.c
  • **呼び出し先**:
  • tuplesort_begin_common
  • TuplesortstateGetPublic
  • MemoryContextSwitchTo
  • palloc / palloc0
  • IndexRelationGetNumberOfKeyAttributes
  • PrepareSortSupportFromGistIndexRel

関連データ構造

  • Tuplesortstate: ソート処理のメイン状態管理構造体。
  • TuplesortIndexBTreeArg: GiSTインデックスソート用の引数保持構造体。
  • SortSupportData: 各列の比較ロジックを保持する構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • indexRel が無効な場合、PrepareSortSupportFromGistIndexRel 内で失敗する可能性があります。

保守担当者向けメモ

  • GiSTインデックスのソートには、内部的にB-Tree用の比較関数（comparetup_index_btree）が流用されています。
  • arg->enforceUnique および arg->uniqueNullsNotDistinct は明示的に false に設定されています。
