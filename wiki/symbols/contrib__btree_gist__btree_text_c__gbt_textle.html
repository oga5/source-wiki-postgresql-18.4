gbt_textle

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_text.c:54

目的

PostgreSQLのGiSTインデックスにおいて、text型の値が比較対象の値以下であるか（less than or equal to）を判定します。

シグネチャまたは形状

static bool gbt_textle(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の text 型データへのポインタ。
  • collation: 照合順序（Collation）のOID。
  • flinfo: 関数マネージャ情報。
  • **出力**:
  • bool: a <= b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は有効な text 型のデータである必要があります。
  • 比較には指定された collation が適用されます。

アルゴリズムまたは振る舞い

1. PointerGetDatum を用いて a と b を Datum 型に変換します。 2. DirectFunctionCall2Coll を介して、PostgreSQL標準のテキスト比較関数 text_le を呼び出します。 3. text_le の戻り値を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_text.c 内の静的配列（gbtree_vinfo 構造体の初期化箇所）から参照されています。
  • **呼び出し先**: PostgreSQLコアの text_le 関数を直接呼び出します。

関連データ構造

  • gbtree_vinfo: text 型のインデックス操作に必要な比較関数群を保持する構造体で使用されます。

失敗モードと境界条件

  • text_le が内部でエラーを発生させた場合、その例外が伝播します。
  • collation が無効な場合、標準の比較関数側の挙動に従います。

保守担当者向けメモ

  • 本関数は text 型専用の比較ロジックを抽象化するためのラッパーです。
  • bpchar 型の同様の処理については、別途定義されている gbt_bpcharle を参照してください。
