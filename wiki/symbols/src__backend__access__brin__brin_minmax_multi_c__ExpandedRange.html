ExpandedRange

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/brin/brin_minmax_multi.c:237

目的

BRIN インデックスの「Multi Min/Max」演算において、複数の間隔（Interval）をマージ・圧縮処理するために正規化された一時的な表現を提供します。

シグネチャまたは形状

typedef struct ExpandedRange
{
    Datum       minval;         /* 下限値 */
    Datum       maxval;         /* 上限値 */
    bool        collapsed;      /* minval == maxval の場合に true */
} ExpandedRange;

入力と出力

  • **入力**: Ranges 構造体から変換されたデータを受け取り、マージ処理の対象となります。
  • **出力**: 圧縮後の ExpandedRange 配列は、最終的に Ranges 構造体へ戻されるか、インデックスの物理格納形式へ変換されます。

前提条件と不変条件

  • collapsed フラグは minval と maxval が等しい場合に true である必要があります。
  • マージ処理の過程で、配列はソート済みであること、および間隔同士に重なりがないことが期待されます（AssertCheckExpandedRanges による検証対象）。

アルゴリズムまたは振る舞い

  • Ranges から ExpandedRange の配列へ展開されます。
  • qsort_arg を用いてソートされ、reduce_expanded_ranges 等の関数によって、最も近い間隔同士がマージされます。
  • マージ処理は、インデックスの保持する値の数が制限を超えた際に実行され、インデックスサイズを縮小します。

呼び出し元と呼び出し先

  • **呼び出し元**: fill_expanded_ranges（Ranges からの変換）、reduce_expanded_ranges（圧縮処理）、および brin_minmax_multi_consistent 等の整合性チェック関数から参照されます。
  • **呼び出し先**: qsort_arg（ソート）、memcpy（配列操作）などが利用されます。

関連データ構造

  • Ranges: メモリ上で間隔を管理する主要な構造体。
  • SerializedRanges: ディスク上の物理格納形式。

失敗モードと境界条件

  • 距離計算が近似値で行われる型（UUID や INET など）の場合、マージの最適化効率が低下する可能性があります。
  • メモリ上のバッファ制限を超えた場合、圧縮処理が強制的に実行されます。

保守担当者向けメモ

  • AssertCheckExpandedRanges を活用して、データ構造の不変条件（ソート順序や重なりの有無）を検証してください。
  • 圧縮処理は高コストであるため、インデックス構築時には MINMAX_BUFFER_FACTOR を用いたバッファ確保が重要です。
