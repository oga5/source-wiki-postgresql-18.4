check_uescapechar

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/parser.c:281

目的

Unicodeエスケープ文字として指定された文字が、SQLの構文規則において有効なエスケープ文字であるかを検証します。

シグネチャまたは形状

static bool check_uescapechar(unsigned char escape);

入力と出力

  • **入力**: escape (検証対象の文字)
  • **出力**: bool 型。指定された文字がエスケープ文字として使用可能な場合は true、使用不可（予約済みまたは無効な文字）の場合は false を返します。

前提条件と不変条件

  • ecpg_isspace() との整合性が求められます。
  • 16進数文字（isxdigit）、+、'、"、および空白文字はエスケープ文字として使用できません。

アルゴリズムまたは振る舞い

1. 引数 escape が以下のいずれかに該当するかを判定します。

  • 16進数文字 (isxdigit)
  • +
  • ' (シングルクォート)
  • " (ダブルクォート)
  • 空白文字 (ecpg_isspace)

    • 2. 上記のいずれかに該当する場合、false を返します。 3. 上記のいずれにも該当しない場合、true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/preproc/parser.c 内で、UESCAPE 句の解析時に呼び出されます。
  • **呼び出し先**: isxdigit (標準ライブラリ) および ecpg_isspace (本モジュール内関数) を呼び出します。
  • 注: src/backend/parser/parser.c にも同名の関数が存在しますが、本ページは src/interfaces/ecpg/preproc/parser.c の定義を対象としています。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • escape がマルチバイト文字の一部である場合や、制御文字である場合の挙動は、isxdigit および ecpg_isspace の実装に依存します。

保守担当者向けメモ

  • 本関数は src/interfaces/ecpg/preproc/parser.c 内の UESCAPE 句の検証ロジックと密接に関連しています。
  • PostgreSQL 本体のパーサ（src/backend/parser/parser.c）にある同名関数と仕様を同期させる必要があります。
