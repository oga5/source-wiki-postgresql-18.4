pg_mul_s64_overflow

定義

  • 種別: function
  • 定義箇所: src/include/common/int.h:292

目的

2つの int64 型の乗算を行い、結果が int64 の範囲内に収まるかを判定する。オーバーフローが発生する場合は true を返し、成功した場合は false を返す。

シグネチャまたは形状

static inline bool pg_mul_s64_overflow(int64 a, int64 b, int64 *result)

入力と出力

  • **入力**:
  • a, b: 乗算対象の int64 値。
  • result: 計算結果を格納する int64 へのポインタ。
  • **出力**:
  • 戻り値: オーバーフローが発生した場合は true、それ以外は false。
  • *result: 演算成功時は積の値、オーバーフロー時はダミー値 0x5EED が書き込まれる。

前提条件と不変条件

  • int64 型が定義されている環境であること。
  • 演算結果が int64 の範囲を超える可能性がある場合、呼び出し側は戻り値を確認し、true の場合は結果を使用しないこと。

アルゴリズムまたは振る舞い

コンパイル環境に応じて以下のいずれかの手法で実装される： 1. **組み込み関数利用**: HAVE__BUILTIN_OP_OVERFLOW が定義されている場合、__builtin_mul_overflow を使用。 2. **128bit 整数利用**: HAVE_INT128 が定義されている場合、int128 にキャストして積を計算し、PG_INT64_MAX / PG_INT64_MIN と比較。 3. **フォールバック**: 上記以外の場合、除算を避けるための事前チェックを行い、範囲外であればオーバーフローと判定。0 や 1 との乗算はオーバーフローしないため個別に除外される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/cash.c
  • src/backend/utils/adt/datetime.c
  • src/backend/utils/adt/int8.c
  • src/backend/utils/adt/numeric.c
  • **呼び出し先**:
  • 外部関数への直接的な呼び出しはない（コンパイラ組み込み関数または言語標準の演算子を使用）。

関連データ構造

  • int64 (PostgreSQL の 64bit 符号付き整数型)

失敗モードと境界条件

  • **オーバーフロー**: 結果が int64 の範囲を超える場合、true を返し *result に 0x5EED を設定する。
  • **ゼロ除算リスク**: フォールバック実装において、0 との乗算チェックにより除算が発生しないよう保護されている。

保守担当者向けメモ

  • HAVE_INT128 が利用可能な環境では、64bit 演算が最適化される。
  • フォールバック実装は、乗算よりもコストの高い除算を避けるための最適化ロジックを含んでいるため、変更時はパフォーマンスへの影響に注意すること。
