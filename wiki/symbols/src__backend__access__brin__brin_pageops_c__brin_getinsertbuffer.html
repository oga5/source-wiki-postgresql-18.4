brin_getinsertbuffer

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_pageops.c:689

目的

BRINインデックスへのタプル挿入に適した物理ページ（バッファ）を検索または確保します。

シグネチャまたは形状

static Buffer brin_getinsertbuffer(Relation irel, Buffer oldbuf, Size itemsz, bool *extended);

入力と出力

  • **入力**:
  • irel: 対象のインデックスリレーション。
  • oldbuf: 現在処理中のバッファ（存在しない場合は InvalidBuffer）。
  • itemsz: 挿入しようとしているタプルのサイズ。
  • extended: ページが新規拡張されたかを示すフラグへのポインタ。
  • **出力**:
  • 挿入可能な状態の Buffer。失敗時や再試行が必要な場合は InvalidBuffer を返す。

前提条件と不変条件

  • itemsz は BrinMaxItemSize 以下であること。
  • 戻り値のバッファは排他ロック（BUFFER_LOCK_EXCLUSIVE）が取得された状態で返される。
  • ページがRevmapページに転用された場合は、ロックを解放し InvalidBuffer を返す。

アルゴリズムまたは振る舞い

1. **ターゲット選定**: RelationGetTargetBlock でキャッシュされたブロックを優先し、なければ GetPageWithFreeSpace でFSMから候補を探す。 2. **バッファ確保**: 候補がない場合はリレーションを拡張（P_NEW）し、ある場合は ReadBuffer で読み込む。 3. **ロックと検証**:

  • ページがRevmapページでないことを確認する。
  • ページ内の空き領域が itemsz を満たすか確認する。

    • 4. **再試行**: 空き領域が不足している場合、RecordAndGetPageWithFreeSpace を呼び出してFSMを更新し、別のページでループを繰り返す。 5. **終了**: 適切なページが見つかった場合、ターゲットブロックを更新し、ロックを保持したままバッファを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: brin_doinsert（src/backend/access/brin/brin_pageops.c 内の複数箇所）。
  • **呼び出し先**: RelationGetTargetBlock, GetPageWithFreeSpace, ReadBuffer, LockBuffer, brin_initialize_empty_new_buffer, br_page_get_freespace, RecordAndGetPageWithFreeSpace 等。

関連データ構造

  • Relation: インデックスリレーション。
  • Buffer: バッファ管理構造体。
  • FSM (Free Space Map): 空き領域管理マップ。

失敗モードと境界条件

  • **サイズ超過**: 挿入しようとするアイテムが BrinMaxItemSize を超える場合、エラー（ERRCODE_PROGRAM_LIMIT_EXCEEDED）を発生させる。
  • **Revmap競合**: ページが並行処理によりRevmapページへ変更された場合、処理を中断し InvalidBuffer を返す。
  • **FSM不整合**: FSMが古い情報を返した場合、ループ内で再試行を行う。

保守担当者向けメモ

  • ページ拡張時に RELATION_IS_LOCAL でない場合は LockRelationForExtension を使用して排他制御を行う必要がある。
  • ページ操作はクリティカルセクション内での実行を前提としており、エラー発生時の整合性に注意すること。
