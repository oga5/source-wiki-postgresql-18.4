cost_qual_eval_context

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/path/costsize.c:167

目的

クエリ内の条件式（Qual）の評価コストを算出する際、再帰的なツリー探索（Walker）を通じてコスト情報を蓄積・保持するために使用されるコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	PlannerInfo *root;
	QualCost	total;
} cost_qual_eval_context;

入力と出力

  • **入力**:
  • root: プランナのグローバル情報（PlannerInfo）。
  • total: 評価コストの累積値（QualCost）。
  • **出力**: 探索終了後、total フィールドに計算されたコストの合計値が格納されます。

前提条件と不変条件

  • cost_qual_eval_walker 関数による探索の過程で、total フィールドが更新されます。
  • root は有効な PlannerInfo を指している必要があります。

アルゴリズムまたは振る舞い

この構造体は、cost_qual_eval_walker 関数がクエリツリーを走査する際の「状態保持」の役割を担います。探索中にノードが見つかるたびに、そのノードの評価コストが total に加算されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/costsize.c 内の関数（cost_qual_eval_walker を呼び出す箇所）でインスタンス化されます。
  • **呼び出し先**: cost_qual_eval_walker 関数がこの構造体を引数として受け取り、内部の total を更新します。

関連データ構造

  • PlannerInfo: プランナの全体的な状態を保持する構造体。
  • QualCost: 評価コスト（スタートアップコストとトータルコスト）を保持する構造体。
  • cost_qual_eval_walker: この構造体を利用してツリーを探索する関数。

失敗モードと境界条件

  • 構造体自体に起因する失敗モードは定義されていません。
  • root が NULL の場合、または total の計算過程でオーバーフローが発生する場合の挙動については、本構造体の定義範囲外です。

保守担当者向けメモ

  • 本構造体は cost_qual_eval_walker と密結合しています。Walker のロジックを変更する際は、本構造体のフィールドが適切に利用されているか確認してください。
