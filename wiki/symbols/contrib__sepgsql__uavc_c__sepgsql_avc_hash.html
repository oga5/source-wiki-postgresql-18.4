sepgsql_avc_hash

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:66

目的

SELinux のユーザー空間アクセスベクタキャッシュ（AVC）において、主体（Subject）のセキュリティコンテキスト、対象（Target）のセキュリティコンテキスト、およびオブジェクトクラスの組み合わせからハッシュ値を計算する。

シグネチャまたは形状

static uint32 sepgsql_avc_hash(const char *scontext, const char *tcontext, uint16 tclass)

入力と出力

  • **入力**:
  • scontext (const char *): 主体のセキュリティコンテキスト文字列。
  • tcontext (const char *): 対象のセキュリティコンテキスト文字列。
  • tclass (uint16): オブジェクトクラス識別子。
  • **出力**:
  • uint32: 計算されたハッシュ値。

前提条件と不変条件

  • scontext および tcontext は有効なヌル終端文字列である必要がある。
  • hash_any 関数（common/hashfn.h）を利用して文字列のハッシュを生成する。

アルゴリズムまたは振る舞い

1. scontext のハッシュ値を hash_any で計算する。 2. tcontext のハッシュ値を hash_any で計算する。 3. 上記2つのハッシュ値と tclass をビット単位の XOR（排他的論理和）で結合し、最終的なハッシュ値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/uavc.c:210
  • contrib/sepgsql/uavc.c:304
  • **呼び出し先**:
  • hash_any (PostgreSQL 内部ユーティリティ)
  • strlen (標準 C ライブラリ)

関連データ構造

  • 本関数は avc_cache エントリのハッシュテーブル検索および挿入時に使用される。

失敗モードと境界条件

  • 入力文字列が非常に長い場合、strlen および hash_any の処理コストが増大する。
  • 文字列が NULL の場合の挙動については、本ソースコード内では明示的なガードは確認できない。

保守担当者向けメモ

  • 本関数はハッシュテーブルのインデックス計算に使用されるため、衝突耐性は hash_any の特性に依存する。
  • キャッシュのキー構成に変更があった場合は、本関数の XOR ロジックを更新する必要がある。
