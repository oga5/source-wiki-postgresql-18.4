extractPageInfo

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/parsexlog.c:388

目的

WAL レコードを解析し、データブロックの変更情報を抽出して process_target_wal_block_change に渡すことで、pg_rewind の同期対象となる変更マップを更新します。

シグネチャまたは形状

static void extractPageInfo(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record （解析対象の WAL レコード状態）
  • **出力**: なし（副作用として process_target_wal_block_change を呼び出し、内部状態を更新する）

前提条件と不変条件

  • record は有効な XLogReaderState を指している必要があります。
  • メインフォーク（MAIN_FORKNUM）以外のフォークに対する変更は、本関数では無視されます。

アルゴリズムまたは振る舞い

1. WAL レコードの RmgrId および info を取得し、レコードの種類を特定します。 2. データベース作成、削除、SMGR 作成・切り詰め、トランザクションコミット/アボートに関連する特定のレコードタイプは、同期対象外としてスキップします。 3. XLR_SPECIAL_REL_UPDATE フラグが立っているが認識できないレコードタイプに遭遇した場合、pg_fatal を呼び出して処理を中断します。 4. レコードに含まれる各ブロックを走査し、XLogRecGetBlockTagExtended でブロックタグ（RelFileLocator, ForkNumber, BlockNumber）を取得します。 5. MAIN_FORKNUM に該当するブロックのみを抽出し、process_target_wal_block_change を呼び出して変更を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/parsexlog.c 内の WAL 読み込みループ（extractPageMap 等から呼び出される）。
  • **呼び出し先**: process_target_wal_block_change（変更の記録）、XLogRecGetRmid、XLogRecGetInfo、XLogRecMaxBlockId、XLogRecGetBlockTagExtended、RmgrName、pg_fatal。

関連データ構造

  • XLogReaderState: WAL レコードの読み込み状態を保持する構造体。
  • RelFileLocator: 変更されたリレーションを識別する構造体。
  • BlockNumber: 変更されたブロック番号。

失敗モードと境界条件

  • **認識不能な更新**: XLR_SPECIAL_REL_UPDATE フラグが設定されている未知のレコードタイプに遭遇すると、pg_fatal によりプログラムが終了します。
  • **フォークの除外**: MAIN_FORKNUM 以外のフォーク（例: FSM_FORKNUM, VISIBILITYMAP_FORKNUM）に対する変更は、本関数では無視されます。

保守担当者向けメモ

  • 本関数は、特定の WAL レコードタイプを無視する判断基準を保持しています。新しい WAL レコードタイプが追加された際、それが pg_rewind の同期に影響するかどうかを判断し、必要に応じて条件分岐を更新してください。
