ResourceElem

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/resowner/resowner.c:64

目的

リソースオーナーによって管理される個々のリソース参照を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct ResourceElem
{
	Datum		item;
	const ResourceOwnerDesc *kind;
} ResourceElem;

入力と出力

  • item: 管理対象リソースの識別子（Datum 型）。
  • kind: リソースの種類を定義する ResourceOwnerDesc へのポインタ。

前提条件と不変条件

  • ハッシュテーブル内で使用される際、kind が NULL であることは、そのスロットが空（未使用）であることを示します。

アルゴリズムまたは振る舞い

  • ResourceOwnerData 内で、固定長配列（arr）またはハッシュテーブル（hash）の要素として格納されます。
  • リソースの解放順序を決定する際、resource_priority_cmp 関数内で kind を参照してソートが行われます。

呼び出し元と呼び出し先

  • 呼び出し元: ResourceOwnerData 構造体（配列およびハッシュテーブルの要素として保持）、および ResourceOwnerSort 関数（ソート処理の対象として参照）。
  • 呼び出し先: ResourceOwnerDesc（リソースの種別定義）。

関連データ構造

  • ResourceOwnerData: 本構造体を保持する親構造体。
  • ResourceOwnerDesc: リソースの特性を定義する構造体。

失敗モードと境界条件

  • ハッシュテーブルとして使用する場合、kind が NULL かどうかで空きスロットを判定するため、有効なリソースの kind に NULL を設定してはなりません。

保守担当者向けメモ

  • 本構造体は resowner.c 内の内部的な管理単位です。
  • ResourceOwnerEnlarge() を呼び出して十分な容量を確保した後に、この要素を格納する必要があります。
