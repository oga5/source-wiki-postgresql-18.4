ginxlogCreatePostingTree

定義

  • 種別: struct
  • 定義箇所: src/include/access/ginxlog.h:21

目的

GINインデックスにおいて、ポスティングツリー（Posting Tree）を新規作成する際のWAL（Write-Ahead Log）レコード構造を定義します。

シグネチャまたは形状

typedef struct ginxlogCreatePostingTree
{
    uint32      size;
    /* A compressed posting list follows */
} ginxlogCreatePostingTree;

入力と出力

  • **入力**: 構造体自体は固定長の size フィールドを持ち、その直後に可変長の圧縮済みポスティングリストデータが続くレイアウトをとります。
  • **出力**: WALレコードの一部として永続化され、リカバリ時にツリー構造の再構築に使用されます。

前提条件と不変条件

  • 構造体の直後に圧縮されたポスティングリストデータが配置されている必要があります。
  • size フィールドは、後続するデータのサイズを正確に反映している必要があります。

アルゴリズムまたは振る舞い

1. **記録**: GINインデックス操作中にポスティングツリーが作成される際、本構造体にメタデータを格納し、XLogRegisterData を通じてWALに書き出されます。 2. **再生**: リカバリ時には XLogRecGetData で本構造体を取得し、直後のメモリ領域から圧縮データを読み取ってポスティングツリーの再構築を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gindatapage.c 内のポスティングツリー作成処理から参照されています。
  • **呼び出し先**: src/backend/access/gin/ginxlog.c 内のREDO処理において、WALレコードからデータを抽出するために使用されます。

関連データ構造

  • ginxlogInsert: GINの挿入操作全般を扱う共通構造体。
  • XLogRecord: WALレコードの汎用ヘッダー。

失敗モードと境界条件

  • 本構造体は可変長データを伴うため、memcpy 等によるデータアクセス時にポインタ演算が正しく行われない場合、メモリ破壊や不正なデータ読み込みが発生するリスクがあります。
  • WALレコードのサイズ制限を超過するような巨大なポスティングリストを扱う場合、適切な分割処理が必要です。

保守担当者向けメモ

  • 本構造体はWALバイナリ形式の一部であるため、フィールドの追加や変更を行う際は、既存のWALログとの後方互換性に細心の注意を払ってください。
  • 構造体のアライメントとパディングが、異なるプラットフォーム間でのバイナリ互換性を損なわないよう確認してください。
