AtEOSubXact_HashTables

定義

  • 種別: function
  • 定義箇所: src/backend/utils/hash/dynahash.c:1940

目的

サブトランザクションの終了（コミットまたはアボート）時に、そのネストレベルで開始されたハッシュテーブルのシーケンシャルスキャンをクリーンアップし、リークを防止する。

シグネチャまたは形状

void AtEOSubXact_HashTables(bool isCommit, int nestDepth)

入力と出力

  • isCommit (bool): トランザクションがコミットされる場合は true、アボートされる場合は false。
  • nestDepth (int): クリーンアップ対象となるサブトランザクションのネストレベル。
  • 出力: なし（副作用としてグローバルなスキャン管理状態を更新）。

前提条件と不変条件

  • num_seq_scans、seq_scan_tables、seq_scan_level がグローバルなスキャン管理状態として存在すること。
  • 削除操作において、配列の順序は維持されない。

アルゴリズムまたは振る舞い

1. num_seq_scans - 1 から 0 へ向かって逆方向にループし、すべてのスキャンエントリを検査する。 2. 各エントリの seq_scan_level が指定された nestDepth 以上である場合、対象のスキャンをクリーンアップする。 3. isCommit が true の場合、リークしたスキャンとして WARNING ログを出力する。 4. 配列の末尾にある要素を現在のインデックス i にコピーし、num_seq_scans をデクリメントすることで、配列から要素を削除する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/xact.c (サブトランザクション終了処理)
  • 呼び出し先:
  • elog (警告ログ出力)

関連データ構造

  • seq_scan_tables: アクティブなハッシュテーブルスキャンの配列。
  • seq_scan_level: 各スキャンが開始されたネストレベルを保持する配列。
  • num_seq_scans: 現在アクティブなスキャン数。

失敗モードと境界条件

  • num_seq_scans が 0 の場合、ループは実行されず何も行われない。
  • 削除対象が配列の末尾にある場合や、複数存在する場合でも、逆方向からの走査により正しく処理される。

保守担当者向けメモ

  • 本関数はサブトランザクションの終了処理に特化している。トップレベルトランザクションの終了処理については AtEOXact_HashTables を参照すること。
