目的

pg_archivecleanup ユーティリティのメインエントリーポイントであり、コマンドライン引数の解析、アーカイブディレクトリの初期化、および不要な WAL ファイルの削除処理を統括する。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数。アーカイブディレクトリのパスと、保持すべき最も古い WAL ファイル名（基準ファイル）を必須とする。
  • オプション: -b (バックアップ履歴削除), -d (デバッグ出力), -n (ドライラン), -x (拡張子除去)。
  • **出力**:
  • 正常終了時は 0 を返す。
  • 引数エラーや処理失敗時は 2 を返し、エラーメッセージを標準エラーに出力する。

前提条件と不変条件

  • Initialize() が呼び出される前に、アーカイブディレクトリのパスが引数から取得されている必要がある。
  • SetWALFileNameForCleanup() により、削除の境界となる exclusiveCleanupFileName が正しく算出されている必要がある。

アルゴリズムまたは振る舞い

1. **環境初期化**: pg_logging_init およびロケール設定を行う。 2. **引数解析**: getopt_long を使用し、オプションフラグおよび必須引数（アーカイブディレクトリ、基準ファイル名）を読み込む。 3. **検証**: 引数の過不足をチェックし、不足している場合はエラーを出力して終了する。 4. **初期化**: Initialize() を呼び出し、アーカイブディレクトリの存在確認等を行う。 5. **境界決定**: SetWALFileNameForCleanup() を呼び出し、削除対象の基準となる WAL ファイル名を確定させる。 6. **削除実行**: CleanupPriorWALFiles() を呼び出し、基準より古いファイルをディレクトリから削除する。 7. **終了**: 処理完了後、exit(0) で終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェルまたはプロセス実行環境。
  • **呼び出し先**:
  • Initialize()
  • SetWALFileNameForCleanup()
  • CleanupPriorWALFiles()
  • usage()
  • pg_logging_init(), getopt_long(), exit() 等の標準/ライブラリ関数。
  • *注記*: ツールが提示した contrib/amcheck 等の参照は、本シンボルとは無関係なコードベース内の同名関数への言及であり、本関数の呼び出し元ではない。

関連データ構造

  • long_options: コマンドライン引数の定義構造体。
  • archiveLocation, restartWALFileName, exclusiveCleanupFileName, dryrun, cleanBackupHistory, additional_ext: 処理の制御に使用されるグローバル変数群。

失敗モードと境界条件

  • **引数不足**: アーカイブディレクトリまたは基準ファイル名が指定されていない場合、エラーメッセージを表示して終了する。
  • **引数過多**: 必須引数以外の余分な引数が渡された場合、エラーを表示して終了する。
  • **ディレクトリ不在**: Initialize() 内でアーカイブディレクトリが存在しないと判断された場合、処理が中断される。

保守担当者向けメモ

  • 本関数はプログラムのライフサイクルを制御する。新しい引数や初期化処理を追加する場合は、getopt_long の定義と Initialize() の呼び出し順序に注意すること。
  • 削除ロジックの変更は CleanupPriorWALFiles() に集約されているため、本関数では引数の整合性チェックに注力する。
