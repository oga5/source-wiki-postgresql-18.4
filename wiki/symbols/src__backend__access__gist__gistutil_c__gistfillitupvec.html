gistfillitupvec

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistutil.c:126

目的

複数の IndexTuple を単一の連続したメモリ領域にコピーし、それらを連結したタプルベクトルを作成します。

シグネチャまたは形状

IndexTupleData *gistfillitupvec(IndexTuple *vec, int veclen, int *memlen)

入力と出力

  • **入力**:
  • vec: IndexTuple の配列（コピー元）。
  • veclen: 配列内のタプル数。
  • **出力**:
  • 戻り値: 連結されたタプル群を格納したメモリ領域の先頭ポインタ。
  • memlen: 連結された全タプルの合計サイズ（バイト単位）が格納される。

前提条件と不変条件

  • vec に含まれる各 IndexTuple は有効である必要があります。
  • 戻り値のメモリ領域は palloc によって確保されるため、呼び出し元で適切に管理（または解放）する必要があります。

アルゴリズムまたは振る舞い

1. veclen で指定された全タプルのサイズを IndexTupleSize マクロを用いて合計し、memlen に格納します。 2. 合計サイズ分のメモリを palloc で確保します。 3. 各タプルを確保したメモリ領域へ memcpy を用いて順次コピーし、ポインタを進めます。 4. 最後に、連結されたメモリ領域の先頭ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c: インデックス挿入やページ分割処理において呼び出されます。
  • src/backend/access/gist/gistbuild.c: インデックス構築時に呼び出されます。
  • **呼び出し先**:
  • palloc (PostgreSQL メモリ管理)
  • memcpy (標準ライブラリ)
  • IndexTupleSize (マクロ)

関連データ構造

  • IndexTuple: GiST インデックス内の各エントリを表すデータ構造。
  • IndexTupleData: IndexTuple の実体となる構造体。

失敗モードと境界条件

  • veclen が 0 の場合、memlen は 0 となり、palloc(0) が実行されます。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は、GiST インデックスのページ操作において、複数のタプルをまとめて扱う際のメモリレイアウトを平坦化するために使用されます。
  • IndexTupleSize マクロに依存しているため、タプル構造の変更時には注意が必要です。
