pg_tablespace_location

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/misc.c:300

目的

指定されたテーブルスペースOIDに対応するファイルシステム上の物理パス（またはシンボリックリンクの参照先）をテキスト形式で取得します。

シグネチャまたは形状

Datum pg_tablespace_location(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) によりテーブルスペースのOIDを受け取ります。
  • **出力**: text 型として、テーブルスペースのパスを返します。

前提条件と不変条件

  • 入力OIDが InvalidOid の場合、現在のデータベースのデフォルトテーブルスペース（MyDatabaseTableSpace）として扱われます。
  • クラスタのデフォルトテーブルスペース（DEFAULTTABLESPACE_OID または GLOBALTABLESPACE_OID）が指定された場合、空文字列を返します。

アルゴリズムまたは振る舞い

1. 入力OIDを検証し、デフォルト値への置換および特殊OIDの判定を行います。 2. pg_tblspc/<oid> パスを構築します。 3. lstat を使用して対象パスの状態を確認します。

  • ディレクトリである場合（allow_in_place_tablespaces が有効な場合など）、そのパスをそのまま返します。
  • シンボリックリンクである場合、readlink を使用してリンク先のパスを取得し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • pg_dumpall
  • pg_rewind
  • pg_upgrade
  • psql (describeコマンド)
  • その他、SQLクエリ経由で呼び出されます。
  • **呼び出し先**:
  • lstat, readlink (システムコール)
  • cstring_to_text, ereport (PostgreSQL内部関数)

関連データ構造

  • Oid: テーブルスペース識別子。
  • struct stat: ファイル属性保持用。

失敗モードと境界条件

  • lstat が失敗した場合、ereport(ERROR, ...) によりエラーを報告します。
  • readlink が失敗した場合、またはリンク先パスが MAXPGPATH を超える場合、エラーを報告します。

保守担当者向けメモ

  • 本関数は pg_tblspc ディレクトリ内のシンボリックリンク構造に強く依存しています。ファイルシステム構成に変更を加える際は注意してください。
  • allow_in_place_tablespaces 設定によるディレクトリ配置のケースを考慮した設計となっています。
