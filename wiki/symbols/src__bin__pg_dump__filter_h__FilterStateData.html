FilterStateData

定義

  • **種別**: struct
  • **定義箇所**: src/bin/pg_dump/filter.h:25

目的

pg_dump および pg_restore におけるフィルタファイル（除外・包含リスト）の解析状態を保持・管理するための構造体です。

シグネチャまたは形状

typedef struct
{
	FILE	   *fp;
	const char *filename;
	exit_function exit_nicely;
	int			lineno;
	StringInfoData linebuff;
} FilterStateData;

入力と出力

  • **入力**: filter_init を通じてファイルパスと終了処理用コールバック関数を受け取ります。
  • **出力**: 解析中の行データやエラー情報を保持し、filter_read_item 等の関数を通じて解析結果を提供します。

前提条件と不変条件

  • StringInfoData を使用して行バッファを動的に管理します。
  • exit_nicely は、エラー発生時にアプリケーション固有の終了処理を実行するために使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: filter_init によりファイルポインタと状態がセットアップされます。 2. **解析**: filter_read_item が呼び出されるたびに、fp から行を読み込み、lineno をインクリメントしながら linebuff に内容を格納します。 3. **終了**: filter_free により、バッファの解放とファイルクローズが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: pg_dump.c 等のアプリケーション本体。
  • **呼び出し先**: filter.c 内の関数群（filter_init, filter_free, pg_log_filter_error, read_quoted_string, read_pattern, filter_read_item）が本構造体を引数として操作します。

関連データ構造

  • StringInfoData (lib/stringinfo.h): 行バッファの管理に使用。
  • exit_function: エラー時のコールバック関数型。

失敗モードと境界条件

  • ファイルオープン失敗や読み込みエラー発生時、pg_log_filter_error を介してエラーログが出力されます。
  • フィルタファイルの構文不正時は、登録された exit_nicely 関数が呼び出され、アプリケーションが終了します。

保守担当者向けメモ

  • 本構造体は pg_dump と pg_restore で共有されるため、変更時は両アプリケーションへの影響を考慮してください。
  • FilterObjectType 列挙型を拡張する場合は、本構造体を利用する解析ロジック（filter.c）の更新が必要です。
