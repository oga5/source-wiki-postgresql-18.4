dsnowball_init

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/dict_snowball.c:224

目的

PostgreSQLの全文検索辞書としてSnowballステマーを初期化し、設定パラメータに基づいて辞書インスタンス（DictSnowball）を構築します。

シグネチャまたは形状

Datum dsnowball_init(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される List *dictoptions（辞書定義パラメータのリスト）。
  • **出力**: 初期化された DictSnowball 構造体へのポインタを Datum 型として返します。

前提条件と不変条件

  • dictoptions には language パラメータが必須です。
  • stopwords および language パラメータはそれぞれ最大1回のみ指定可能です。
  • 成功時には d->stem（ステマー環境）が割り当てられている必要があります。

アルゴリズムまたは振る舞い

1. DictSnowball 構造体をメモリ確保し、初期化します。 2. dictoptions リストを走査し、各パラメータを解析します。

  • stopwords: readstoplist を呼び出し、指定されたファイルからストップワードリストを読み込みます。
  • language: locate_stem_module を呼び出し、指定された言語に対応するステマーを検索・設定します。

    • 3. 未知のパラメータが指定された場合、または必須パラメータが欠落している場合はエラーを報告します。 4. 現在のメモリコンテキストを d->dictCtx に保存し、構築した構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの辞書初期化フレームワーク（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • readstoplist: ストップワードリストの読み込み。
  • locate_stem_module: 言語固有のステマーの割り当て。
  • palloc0, ereport, defGetString, lfirst: PostgreSQLのユーティリティ関数。

関連データ構造

  • DictSnowball: ステマー環境、ストップワードリスト、メモリコンテキストを保持する主要構造体。
  • DefElem: 辞書定義の各パラメータを保持する構造体。

失敗モードと境界条件

  • stopwords または language パラメータが重複して指定された場合、ERRCODE_INVALID_PARAMETER_VALUE でエラー終了します。
  • 未知のパラメータが指定された場合、エラー終了します。
  • language パラメータが指定されなかった場合、エラー終了します。

保守担当者向けメモ

  • 本関数は辞書作成時に一度だけ実行されます。
  • d->dictCtx に現在のメモリコンテキストを保持することで、後のステミング実行時のメモリ管理を制御しています。
