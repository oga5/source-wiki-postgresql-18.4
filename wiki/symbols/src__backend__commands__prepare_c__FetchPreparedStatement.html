FetchPreparedStatement

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:433

目的

セッション内で管理されているプリペアドステートメントを、名前を指定してハッシュテーブルから取得する。

シグネチャまたは形状

PreparedStatement *FetchPreparedStatement(const char *stmt_name, bool throwError)

入力と出力

  • **入力**:
  • stmt_name: 検索対象のプリペアドステートメント名。
  • throwError: 該当するステートメントが見つからない場合にエラーを発生させるかどうかのフラグ。
  • **出力**:
  • 成功時: 該当する PreparedStatement 構造体へのポインタ。
  • 失敗時: throwError が false の場合は NULL を返す。

前提条件と不変条件

  • prepared_queries ハッシュテーブルが初期化されているかどうかにかかわらず、安全に呼び出し可能。
  • throwError が true であり、かつステートメントが存在しない場合、本関数は呼び出し元に戻らず ereport(ERROR) により処理を中断する。

アルゴリズムまたは振る舞い

1. prepared_queries ハッシュテーブルが存在するか確認する。 2. 存在する場合、hash_search を用いて stmt_name をキーに検索を行う（HASH_FIND モード）。 3. ハッシュテーブルが存在しない、または検索結果が NULL の場合、entry を NULL に設定する。 4. entry が NULL かつ throwError が true の場合、ERRCODE_UNDEFINED_PSTATEMENT エラーを報告する。 5. 取得した entry（または NULL）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/prepare.c 内の各コマンド処理関数。
  • src/backend/tcop/postgres.c（拡張クエリプロトコル処理）。
  • src/backend/tcop/pquery.c および src/backend/tcop/utility.c。
  • **呼び出し先**:
  • hash_search（ハッシュテーブル検索用）。
  • ereport（エラー報告用）。

関連データ構造

  • PreparedStatement: プリペアドステートメントの情報を保持する構造体。
  • prepared_queries: セッション内のプリペアドステートメントを格納するハッシュテーブル。

失敗モードと境界条件

  • **ステートメント未存在**: throwError が true の場合、ERRCODE_UNDEFINED_PSTATEMENT でエラー終了する。
  • **ハッシュテーブル未初期化**: prepared_queries が NULL の場合、検索を行わず NULL を返す。

保守担当者向けメモ

  • 本関数はセッションローカルなプリペアドステートメントの検索専用である。
  • 戻り値の PreparedStatement 構造体のライフサイクルは、呼び出し元で適切に管理される必要がある。
