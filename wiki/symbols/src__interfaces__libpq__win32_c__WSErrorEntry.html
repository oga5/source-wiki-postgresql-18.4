WSErrorEntry

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/win32.c:43

目的

Windows環境におけるWinsockエラーコードと、それに対応する人間が読み取り可能な説明文をマッピングするためのデータ構造です。

シグネチャまたは形状

struct WSErrorEntry {
    DWORD       error;
    const char *description;
};

入力と出力

本構造体はデータ保持用であり、直接的な入出力関数ではありません。WSErrors 配列の要素として、エラーコード（DWORD）を入力とし、対応する説明文（const char *）を出力として提供するために使用されます。

前提条件と不変条件

  • error フィールドには、Winsock固有のエラーコードが格納されます。
  • description フィールドは、対応するエラーの簡潔な説明文を指すポインタです。

アルゴリズムまたは振る舞い

本構造体は、winsock_strerror 関数がエラーメッセージを検索する際の参照テーブル（WSErrors）の構成要素として機能します。検索時には、この構造体の error フィールドを線形探索することで、該当するエラーの説明文を特定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/win32.c 内の winsock_strerror 関数が、エラーメッセージ検索のために本構造体を参照します。
  • 呼び出し先: 本構造体自体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • WSErrors: WSErrorEntry 型の要素を持つ静的配列。Winsockエラーコードの全リストを保持します。

失敗モードと境界条件

  • 検索対象のエラーコードが WSErrors 配列内に存在しない場合、winsock_strerror は本構造体による解決を断念し、Windows API（FormatMessage）を用いた動的なメッセージ取得へフォールバックします。

保守担当者向けメモ

  • 新しいWinsockエラーコードを追加する場合は、WSErrors 配列に本構造体のインスタンスを追加してください。
  • 本構造体は静的テーブルの一部であるため、実行時のメモリ割り当ては発生しません。
