DatumGetJsonPathPCopy

定義

  • 種別: function
  • 定義箇所: src/include/utils/jsonpath.h:40

目的

Datum 型として渡された JSON Path データを、メモリ上にコピーして JsonPath 構造体として取得します。

シグネチャまたは形状

static inline JsonPath *DatumGetJsonPathPCopy(Datum d)

入力と出力

  • **入力**: Datum d (JSON Path を保持する Datum)
  • **出力**: JsonPath * (コピーされた JSON Path 構造体へのポインタ)

前提条件と不変条件

  • 入力 d は、PostgreSQL の varlena 型（可変長データ型）として有効な JSON Path を保持している必要があります。
  • PG_DETOAST_DATUM_COPY マクロを使用するため、必要に応じて TOAST データの伸長とメモリへのコピーが行われます。

アルゴリズムまたは振る舞い

1. 引数 d に対して PG_DETOAST_DATUM_COPY を実行します。 2. これにより、データが圧縮または外部化されている場合は伸長され、メモリ上に新しいコピーが作成されます。 3. その結果を JsonPath * 型にキャストして返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/jsonpath.h:47 にて PG_GETARG_JSONPATH_P_COPY(x) マクロ内で利用されています。
  • **呼び出し先**:
  • PG_DETOAST_DATUM_COPY (PostgreSQL のコアマクロ)

関連データ構造

  • JsonPath: 本関数が返すバイナリ形式の JSON Path 構造体。

失敗モードと境界条件

  • メモリ不足時には PG_DETOAST_DATUM_COPY の内部処理によりエラーが発生する可能性があります。
  • 入力 d が有効な JsonPath 形式でない場合、後続の処理で不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数はインライン関数として定義されており、主に SQL 関数インターフェースから JSON Path を安全に取得するために使用されます。
  • PG_GETARG_JSONPATH_P_COPY マクロ経由での利用が推奨されます。
