reduce_outer_joins_partial_state

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/prep/prepjointree.c:99

目的

外結合（Outer Join）の最適化処理において、かつて FULL JOIN であった結合の情報を保持し、その結合の「依然として NULL 可能（nullable）」な側を追跡するために使用されます。

シグネチャまたは形状

typedef struct reduce_outer_joins_partial_state
{
    int         full_join_rti;  /* RT index of a formerly-FULL join */
    Relids      unreduced_side; /* relids in its still-nullable side */
} reduce_outer_joins_partial_state;

入力と出力

本構造体はデータ保持用であり、関数ではありません。palloc を用いてメモリ確保され、reduce_outer_joins 関連の処理内で状態管理のために使用されます。

前提条件と不変条件

  • full_join_rti は、クエリのレンジテーブル（Range Table）内のインデックスを指す必要があります。
  • unreduced_side は、結合の NULL 可能側を特定するためのリレーション ID セットを保持します。

アルゴリズムまたは振る舞い

外結合の削減処理（reduce_outer_joins）において、結合の型が変更された後も、特定の結合ノードが保持する NULL 伝播の性質を追跡するために利用されます。この構造体は、最適化の過程でリストに格納され、後続の処理で参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/prep/prepjointree.c 内の reduce_outer_joins 関連処理。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数呼び出しは行いません。

関連データ構造

  • Relids: リレーション ID の集合を表現する型。
  • reduce_outer_joins_pass1_state, reduce_outer_joins_pass2_state: 同様の最適化フェーズで使用される関連状態構造体。

失敗モードと境界条件

  • メモリ不足時、palloc によるメモリ確保が失敗する可能性があります。
  • full_join_rti が無効なレンジテーブルインデックスを参照している場合、後続のプランニング処理で不正な状態を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は reduce_outer_joins の内部的な状態管理用です。この構造体の変更は、外結合の削減ロジック全体に影響を与えるため、変更時は prepjointree.c 内の関連する最適化ロジックとの整合性を確認してください。
