GeqoPrivateData

定義

  • 種別: struct
  • 定義箇所: src/include/optimizer/geqo.h:75

目的

遺伝的アルゴリズムによるクエリ最適化（GEQO）の実行中に必要な状態情報を保持し、プランナの PlannerInfo 構造体（join_search_private フィールド）を介して共有するためのデータ構造です。

シグネチャまたは形状

typedef struct
{
	List	   *initial_rels;	/* the base relations we are joining */
	pg_prng_state random_state; /* PRNG state */
} GeqoPrivateData;

入力と出力

  • **入力**: GEQO探索開始時に、結合対象となるリレーションのリスト（initial_rels）が格納されます。
  • **出力**: 探索プロセス全体を通じて、乱数生成器の状態（random_state）が更新・保持されます。

前提条件と不変条件

  • GEQOアルゴリズムが有効なクエリにおいて、PlannerInfo の join_search_private にキャスト可能な状態で格納されている必要があります。
  • random_state は、GEQOの確率的な探索プロセスにおいて一貫した乱数生成を保証するために維持されます。

アルゴリズムまたは振る舞い

GEQOの実行中、この構造体は以下の役割を果たします。 1. **リレーション管理**: 最適化対象となるベースリレーションのリストを保持し、結合順序の探索に使用します。 2. **乱数生成**: pg_prng_state を通じて、遺伝的アルゴリズムの交叉や突然変異、選択フェーズで必要となる乱数生成の状態を管理します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_main.c: GEQOのメイン処理でインスタンス化されます。
  • src/backend/optimizer/geqo/geqo_eval.c: root->join_search_private からキャストして参照されます。
  • src/backend/optimizer/geqo/geqo_random.c: root->join_search_private からキャストして参照されます。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。

関連データ構造

  • List: initial_rels を保持するために使用されます。
  • pg_prng_state: common/pg_prng.h で定義される乱数生成器の状態構造体。
  • PlannerInfo: 本構造体を join_search_private フィールドに保持する上位構造体。

失敗モードと境界条件

  • PlannerInfo の join_search_private が初期化されていない状態でアクセスした場合、不正なメモリ参照が発生する可能性があります。
  • 結合対象のリレーション数が極端に少ない場合、GEQO自体が呼び出されないため、本構造体も使用されません。

保守担当者向けメモ

  • random_state の更新はGEQOの探索結果の再現性に直結するため、乱数生成器の仕様変更時には注意が必要です。
  • initial_rels はプランナが構築する RelOptInfo のリストであり、GEQOの探索空間を定義する重要な要素です。
