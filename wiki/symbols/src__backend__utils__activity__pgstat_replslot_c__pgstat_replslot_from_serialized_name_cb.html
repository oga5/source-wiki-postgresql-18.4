pgstat_replslot_from_serialized_name_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_replslot.c:201

目的

シリアライズされたレプリケーションスロット名から、統計情報を識別するための PgStat_HashKey を復元する。

シグネチャまたは形状

bool pgstat_replslot_from_serialized_name_cb(const NameData *name, PgStat_HashKey *key)

入力と出力

  • **入力**:
  • name: 復元対象のスロット名（NameData 型）。
  • key: 復元された統計ハッシュキーを格納する出力用ポインタ。
  • **出力**:
  • 成功時は true を返し、key に PGSTAT_KIND_REPLSLOT、InvalidOid（dboid）、および対応するスロットインデックス（objid）を設定する。
  • 失敗時（スロットが見つからない場合）は false を返す。

前提条件と不変条件

  • スロットのインデックスは再起動時に変化する可能性があるため、永続化された「名前」から現在のインデックスを解決する必要がある。
  • get_replslot_index がスロットの存在確認とインデックス取得に使用される。

アルゴリズムまたは振る舞い

1. get_replslot_index を呼び出し、指定された name に対応する現在のスロットインデックス idx を取得する。 2. idx が -1 の場合、スロットが既に削除されていると判断し false を返す。 3. 成功した場合、key 構造体の kind を PGSTAT_KIND_REPLSLOT に設定する。 4. dboid を InvalidOid に設定する（レプリケーションスロットはデータベースに依存しないため）。 5. objid に取得した idx を格納し、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat.c において、統計情報のシリアライズ復元処理の一部として参照されている。
  • **呼び出し先**: get_replslot_index（同ファイル内）。

関連データ構造

  • NameData: スロット名を保持する構造体。
  • PgStat_HashKey: 統計エントリを特定するためのキー構造体。

失敗モードと境界条件

  • 指定された名前のスロットがシステム上に存在しない場合、get_replslot_index が -1 を返し、本関数は false を返して終了する。

保守担当者向けメモ

  • 本関数は統計サブシステムのシリアライズ復元コールバックとして登録されている。
  • レプリケーションスロットの統計は論理スロットのみを対象としており、物理スロットの扱いは含まれない。
