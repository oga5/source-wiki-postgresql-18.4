MemoryContextAllocAligned

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/mcxt.c:1440

目的

指定されたメモリコンテキストにおいて、要求された境界（alignto）に整列されたメモリ領域を割り当てます。

シグネチャまたは形状

void *MemoryContextAllocAligned(MemoryContext context, Size size, Size alignto, int flags)

入力と出力

  • **入力**:
  • context: 割り当て対象のメモリコンテキスト。
  • size: 要求するメモリサイズ（バイト単位）。
  • alignto: 要求するアライメント境界（2のべき乗であること）。
  • flags: MemoryContextAllocExtended に渡されるフラグ。
  • **出力**:
  • 整列されたメモリ領域へのポインタ。

前提条件と不変条件

  • alignto は 128MB 未満であること。
  • alignto は 2 のべき乗であること。
  • alignto <= MAXIMUM_ALIGNOF の場合、標準の MemoryContextAllocExtended が呼び出されます。

アルゴリズムまたは振る舞い

1. alignto が MAXIMUM_ALIGNOF 以下であれば、標準の割り当て関数へ委譲します。 2. それ以外の場合、要求サイズにアライメント調整用の余分なバイト数と、リダイレクト用 MemoryChunk のサイズを加算して確保します。 3. MemoryContextAllocExtended を呼び出し、未整列のメモリブロックを取得します。 4. TYPEALIGN マクロを使用して、取得した領域内で要求された境界に一致するアドレスを計算します。 5. 計算したアドレスの直前に MCTX_ALIGNED_REDIRECT_ID を持つ MemoryChunk を配置し、元の未整列ポインタを指すように設定します。これにより、後の pfree や repalloc で元のブロックを特定可能にします。 6. MEMORY_CONTEXT_CHECKING が有効な場合、センチネルバイトを配置します。 7. Valgrind を使用して、リダイレクトヘッダーおよび未使用領域へのアクセスを禁止します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/page/bufpage.c
  • src/backend/storage/smgr/bulk_write.c
  • src/backend/utils/mmgr/alignedalloc.c
  • src/backend/utils/mmgr/mcxt.c (自身)
  • **呼び出し先**:
  • MemoryContextAllocExtended
  • PointerGetMemoryChunk
  • MemoryChunkSetHdrMask
  • VALGRIND_MAKE_MEM_NOACCESS

関連データ構造

  • MemoryContext: メモリコンテキストの管理構造体。
  • MemoryChunk: メモリブロックのメタデータ管理用構造体。

失敗モードと境界条件

  • alignto が 2 のべき乗でない場合、アサーションエラーが発生します。
  • alignto が 128MB 以上の場合、アサーションエラーが発生します。
  • メモリ不足時は MemoryContextAllocExtended の挙動に従い、通常はエラーがスローされます。

保守担当者向けメモ

  • 本関数は、pfree 時に元の割り当てブロックを特定するために、整列されたポインタの直前にリダイレクト用の MemoryChunk を埋め込んでいます。この構造を破壊しないよう注意してください。
  • alignto の値は MemoryChunk の value フィールドに格納されており、repalloc 時に再利用されます。
