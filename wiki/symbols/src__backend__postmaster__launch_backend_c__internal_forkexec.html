internal_forkexec

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/launch_backend.c:306

目的

EXEC_BACKEND モードにおいて、子プロセス起動に必要な状態情報を一時ファイルにシリアライズし、fork() および execv() を用いて新しいプロセスを生成する。

シグネチャまたは形状

static pid_t internal_forkexec(const char *child_kind, int child_slot, const void *startup_data, size_t startup_data_len, ClientSocket *client_sock);

入力と出力

  • **入力**:
  • child_kind: 起動する子プロセスの種類を示す文字列。
  • child_slot: 子プロセスのスロット番号。
  • startup_data: プロセス起動に必要な追加データ。
  • startup_data_len: startup_data のサイズ。
  • client_sock: クライアントソケット情報。
  • **出力**:
  • 成功時: 子プロセスのプロセスID (pid_t)。
  • 失敗時: -1。

前提条件と不変条件

  • save_backend_variables が正常に完了し、BackendParameters が初期化されていること。
  • 一時ファイルディレクトリ (PG_TEMP_FILES_DIR) への書き込み権限があること。
  • postgres_exec_path が実行可能なパスを指していること。

アルゴリズムまたは振る舞い

1. save_backend_variables を呼び出し、プロセス間引き継ぎ用のパラメータ構造体を作成する。 2. 一時ファイル名（PG_TEMP_FILES_DIR 配下）を生成し、ファイルを開く。ディレクトリが存在しない場合は作成を試みる。 3. パラメータ構造体を一時ファイルに書き込み、ファイルを閉じる。 4. argv 配列を構築し、--forkchild 引数に child_kind を、続いて一時ファイル名を指定する。 5. fork_process() を実行する。

  • 子プロセス側: execv() を呼び出し、postgres バイナリを再実行する。失敗時は exit(1) する。
  • 親プロセス側: 生成された子プロセスの PID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c 内のプロセス起動関数（internal_forkexec をラップする関数群）。
  • **呼び出し先**: save_backend_variables, AllocateFile, MakePGDirectory, fwrite, FreeFile, fork_process, execv, ereport。

関連データ構造

  • BackendParameters: プロセス間で受け渡される状態情報の構造体。
  • ClientSocket: クライアント接続情報。

失敗モードと境界条件

  • save_backend_variables の失敗: パラメータの保存に失敗した場合、メモリを解放して -1 を返す。
  • 一時ファイルの作成・書き込み失敗: AllocateFile や fwrite が失敗した場合、ログを出力して -1 を返す。
  • execv の失敗: 子プロセス内で実行に失敗した場合、ログを出力して exit(1) する。
  • fork_process の失敗: OSの制限等でプロセス生成ができない場合、-1 を返す。

保守担当者向けメモ

  • 本関数は EXEC_BACKEND モード用であり、Windows環境や特定のテスト環境で利用される。
  • tmpBackendFileNum は静的変数であり、プロセス起動ごとにインクリメントされる。
  • palloc0 を使用してパディングバイトを初期化しており、Valgrind等のツールによる未初期化メモリ警告を回避している。
