permissionsList

定義

  • 種別: function
  • 定義箇所: src/bin/psql/describe.c:1049

目的

データベース内のテーブル、ビュー、シーケンス等のオブジェクトに対するアクセス権限（ACL）および行レベルセキュリティ（RLS）ポリシー情報を取得し、整形して表示する。

シグネチャまたは形状

bool permissionsList(const char *pattern, bool showSystem)

入力と出力

  • **入力**:
  • pattern: オブジェクト名を絞り込むためのパターン文字列（NULL可）。
  • showSystem: システムカタログ（pg_catalog 等）を含めるかどうかを示すフラグ。
  • **出力**:
  • 成功時に true、失敗時に false を返す。
  • 内部で構築した SQL クエリの結果を標準出力（pset.queryFout）へ整形出力する。

前提条件と不変条件

  • pset（psqlの設定状態）が初期化されていること。
  • サーバーバージョンが 9.2 以降であることを想定している。
  • 実行には pg_catalog へのアクセス権限が必要。

アルゴリズムまたは振る舞い

1. PQExpBuffer を初期化し、pg_class を中心とした SQL クエリを構築する。 2. オブジェクトの種類（テーブル、ビュー、マテリアライズドビュー、シーケンス、外部テーブル、パーティションテーブル）を識別する。 3. printACLColumn を呼び出し、オブジェクトレベルの権限を表示する。 4. カラムレベルの権限（pg_attribute）をサブクエリで取得する。 5. サーバーバージョンに応じて、RLS ポリシー（pg_policy）情報をクエリに追加する（9.5以上で有効）。 6. validateSQLNamePattern を使用して、ユーザー入力を安全な SQL 句に変換する。 7. PSQLexec でクエリを実行し、printQuery を用いて表形式で出力する。 8. バッファと結果セットを解放して終了する。

関連データ構造

  • PQExpBufferData: SQL クエリ構築用バッファ。
  • PGresult: クエリ実行結果の保持。
  • printQueryOpt: 出力フォーマット設定。

失敗モードと境界条件

  • PSQLexec が失敗した場合、または validateSQLNamePattern による検証が失敗した場合は error_return ラベルへ遷移し、false を返す。
  • pattern が指定されていない場合、デフォルトでシステムスキーマは除外される。
  • インデックスや TOAST テーブルは権限情報の対象外として無視される。

保守担当者向けメモ

  • サーバーバージョン 9.5 未満、10.0 未満で RLS ポリシーのクエリ生成ロジックが分岐しているため、将来のバージョン追加時はこの条件分岐を確認すること。
  • カラム権限のフォーマットは printACLColumn と整合性を保つ必要がある。
