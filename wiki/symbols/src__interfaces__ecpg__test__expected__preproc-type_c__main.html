定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-type.c:79

目的

ECPGプリプロセッサによる EXEC SQL TYPE 指令およびカスタム型マッピングの変換結果を検証するための回帰テスト用エントリポイントです。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（コマンドライン引数は受け取らない）
  • 出力: 成功時に 0 を返し、エラー発生時に sqlca.sqlcode を終了ステータスとして返す。標準出力にテスト結果を出力する。

前提条件と不変条件

  • データベース ecpg1_regression が接続可能であること。
  • sqlca 構造体がエラーハンドリングに利用可能であること。

アルゴリズムまたは振る舞い

1. **初期化**: ECPGdebug を有効化し、データベースへ接続する。 2. **DDL実行**: create table empl を実行し、テスト用テーブルを作成する。 3. **データ挿入**: insert 文を実行し、ホスト変数 accs_val をバインドする。 4. **データ取得**: select 文を実行し、構造体 TBempl の各メンバおよびカスタム型変数（str, ptr, vc）に結果をマッピングする。 5. **検証**: 取得した値を printf で出力し、期待値との整合性を確認する。 6. **終了**: ECPGdisconnect で接続を解除し、動的に確保したメモリを解放して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: 不明（OSのプロセス起動時に呼び出される）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGdisconnect, printf, exit, free

関連データ構造

  • struct TBempl: テスト用構造体。
  • struct varchar: vc 変数用の型定義。
  • sqlca: SQL通信領域。

失敗モードと境界条件

  • データベース接続失敗時、sqlca.sqlcode を出力して終了する。
  • テーブル作成失敗時、sqlca.sqlcode を出力して終了する。
  • データ挿入または取得失敗時、sqlca.sqlcode を出力して終了する。

保守担当者向けメモ

  • 本関数はプリプロセッサによって生成されたコードであるため、直接編集せず、元の .pgc ファイルを修正すること。
  • ECPGdo の引数リストは、プリプロセッサが生成した型情報とバインド変数のマッピングを直接示しているため、ECPGの内部動作の解析に有用である。
