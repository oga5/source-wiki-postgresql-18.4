r_mark_yU

目的

トルコ語のステミング処理において、特定の接尾辞（yU）の条件付きマッチングと除去を行う。

シグネチャまたは形状

static int r_mark_yU(struct SN_env * z)

入力と出力

  • **入力**: z (Snowball ステミング環境構造体)
  • **出力**: 成功時に 1、失敗または条件不一致時に 0 または負の値を返す。

前提条件と不変条件

  • z は有効な SN_env 構造体である必要がある。
  • 内部で呼び出される r_check_vowel_harmony および r_mark_suffix_with_optional_y_consonant が正常に動作する環境であること。

アルゴリズムまたは振る舞い

1. r_check_vowel_harmony(z) を呼び出し、母音調和のルールを検証する。結果が 0 以下であれば即座に終了する。 2. in_grouping_b_U(z, g_U, 105, 305, 0) を使用し、現在のカーソル位置の文字が特定の母音グループ（g_U）に含まれるかを確認する。含まれない場合は 0 を返す。 3. r_mark_suffix_with_optional_y_consonant(z) を呼び出し、接尾辞の除去を試みる。 4. 最終的に成功した場合は 1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/snowball/libstemmer/stem_UTF_8_turkish.c:1773（他のステミングルール内から呼び出される）。
  • **呼び出し先**:
  • r_check_vowel_harmony

関連データ構造

  • struct SN_env: ステミングの実行状態を保持する構造体。
  • g_U: 母音グループを定義する静的データ（in_grouping_b_U で使用）。

失敗モードと境界条件

  • r_check_vowel_harmony が失敗した場合、処理は中断される。
  • in_grouping_b_U による母音チェックが不一致の場合、接尾辞の除去は行われず 0 が返される。
