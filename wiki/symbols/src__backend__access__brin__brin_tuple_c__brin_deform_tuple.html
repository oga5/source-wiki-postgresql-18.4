brin_deform_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_tuple.c:552

目的

ディスク上の物理的な BrinTuple を、メモリ上で操作可能な BrinMemTuple 構造体へ展開（デシリアライズ）します。

シグネチャまたは形状

BrinMemTuple *brin_deform_tuple(BrinDesc *brdesc, BrinTuple *tuple, BrinMemTuple *dMemtuple)

入力と出力

  • **入力**:
  • brdesc: インデックスの記述子。
  • tuple: 変換対象のオンディスク BrinTuple。
  • dMemtuple: 再利用する BrinMemTuple（NULLの場合は新規割り当て）。
  • **出力**: 展開された BrinMemTuple へのポインタ。

前提条件と不変条件

  • dMemtuple が提供される場合、そのメモリコンテキストは有効である必要があります。
  • brdesc は tuple の構造と一致している必要があります。
  • BrinMemTuple の各カラム値は、dtup->bt_context にコピーされます。

アルゴリズムまたは振る舞い

1. **初期化**: dMemtuple が指定されていれば初期化し、なければ新規作成します。 2. **メタデータ抽出**: tuple のヘッダーからプレースホルダーフラグや空範囲フラグ、ブロック番号をコピーします。 3. **デコンストラクト**: brin_deconstruct_tuple を呼び出し、ディスク上のデータ領域から各カラムの Datum 値とNULLビットマップを抽出します。 4. **値のコピー**: dtup->bt_context にメモリコンテキストを切り替え、各カラムの値を datumCopy を用いてメモリ上に複製します。 5. **状態設定**: 各カラムの bv_hasnulls や bv_allnulls を設定し、シリアライズ用ポインタをリセットします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pageinspect/brinfuncs.c
  • src/backend/access/brin/brin.c
  • **呼び出し先**:
  • brin_memtuple_initialize / brin_new_memtuple
  • brin_deconstruct_tuple
  • datumCopy
  • MemoryContextSwitchTo

関連データ構造

  • BrinTuple: オンディスク形式のタプル。
  • BrinMemTuple: メモリ上の操作用タプル。
  • BrinDesc: インデックスのメタデータ。

失敗モードと境界条件

  • dMemtuple が NULL の場合、メモリ不足時に brin_new_memtuple 内でエラーが発生する可能性があります。
  • datumCopy は、型定義（typbyval や typlen）に基づいてメモリ割り当てを行うため、型情報が不整合な場合は予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は、ループ内でのメモリ割り当てを避けるために dMemtuple の再利用をサポートしています。パフォーマンスが重要なパスでは、可能な限り既存の BrinMemTuple を渡してください。
  • datumCopy によるコピーは dtup->bt_context に対して行われるため、メモリリークを防ぐには BrinMemTuple 全体のライフサイクル管理が重要です。
