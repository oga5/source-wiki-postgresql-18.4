DefineQueryRewrite

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteDefine.c:223

目的

PostgreSQLにおけるクエリ書き換えルール（Rewrite Rule）の定義および更新を行うメインエントリポイントです。指定されたリレーションに対してルールを適用し、システムカタログへの登録と整合性チェックを統括します。

シグネチャまたは形状

ObjectAddress DefineQueryRewrite(
    const char *rulename,
    Oid event_relid,
    Node *event_qual,
    CmdType event_type,
    bool is_instead,
    bool replace,
    List *action
)

入力と出力

  • **入力**: ルール名、対象リレーションのOID、イベント条件（event_qual）、コマンド種別、INSTEADフラグ、置換フラグ、アクションリスト。
  • **出力**: 作成または更新されたルールの ObjectAddress を返します。

前提条件と不変条件

  • 対象リレーションに対して AccessExclusiveLock を保持する必要があります。
  • システムカタログへの変更権限が必要です。
  • ON SELECT ルールの場合、対象はビューまたはマテリアライズドビューである必要があります。
  • ON SELECT ルールは _RETURN という名前である必要があります（後方互換性のため一部例外あり）。

アルゴリズムまたは振る舞い

1. **ロック取得**: table_open を呼び出し、対象リレーションに AccessExclusiveLock を取得します。 2. **妥当性検証**: リレーションの種類（テーブル、ビュー等）がルール適用可能か確認し、システムテーブルへの変更制限をチェックします。 3. **権限確認**: object_ownercheck により、呼び出し元が対象リレーションの所有者であることを確認します。 4. **アクション解析**: ルールアクション内に OLD や NEW を不正に参照するクエリが含まれていないか走査します。 5. **ON SELECT 特殊処理**: ON SELECT ルールの場合、単一の SELECT アクションであること、INSTEAD であること、ターゲットリストがリレーションと一致すること等を厳格に検証します。 6. **カタログ登録**: 妥当性確認後、内部的にカタログ更新処理へ移行します（ソースコードの続きで実行）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/view.c
  • src/backend/parser/parse_utilcmd.c
  • src/backend/rewrite/rewriteDefine.c（DefineRule経由）
  • **呼び出し先**:
  • table_open
  • object_ownercheck
  • checkRuleResultList
  • getInsertSelectQuery
  • RelationGetRelationName 等のアクセサ

関連データ構造

  • Relation: 対象リレーションの定義情報。
  • Query: ルールアクションを構成するクエリツリー。
  • RewriteRule: ルール定義を保持する構造体。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • **権限エラー**: 所有者でない場合、ACLCHECK_NOT_OWNER でエラー終了します。
  • **型不一致**: ルール適用不可なリレーションや、ON SELECT ルールをテーブルに定義しようとした場合に ERRCODE_WRONG_OBJECT_TYPE でエラー終了します。
  • **実装制限**: OLD/NEW への不正なアクセスや、ON SELECT ルールにおける複数アクション、INSTEAD NOTHING 等は ERRCODE_FEATURE_NOT_SUPPORTED で拒否されます。
  • **重複定義**: ON SELECT ルールが既に存在する場合、replace が偽であればエラーとなります。

保守担当者向けメモ

  • 本関数はカタログを書き換えるため、ロックの取得順序とレベルが重要です。DefineRule との整合性を維持してください。
  • ON SELECT ルールはビューの定義そのものであるため、変更時にはキャッシュ無効化（CacheInvalidateRelcache）が必須です。
  • ソースコードの後半（_RETURN 名のチェック以降）で、実際のカタログ挿入処理が続くため、修正時はカタログの依存関係管理に注意してください。
