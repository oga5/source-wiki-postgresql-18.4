BulkInsertStateData

定義

  • 種別: struct
  • 定義箇所: src/include/access/hio.h:29

目的

大量挿入（バルク挿入）操作において、バッファの再利用やページ拡張を効率化するための状態を保持する構造体です。

シグネチャまたは形状

typedef struct BulkInsertStateData
{
    BufferAccessStrategy strategy;    /* BULKWRITE 戦略オブジェクト */
    Buffer        current_buf;        /* 現在の挿入ターゲットページ */
    BlockNumber   next_free;          /* 次の空きページ候補 */
    BlockNumber   last_free;          /* 最後に拡張された時点での空きページ範囲の終端 */
    uint32        already_extended_by; /* バルク挿入によって拡張されたページ数 */
} BulkInsertStateData;

入力と出力

  • 本構造体は heapam.c 等で palloc され、hio.c の各関数（RelationGetBufferForTuple 等）へポインタとして渡されます。
  • 出力として、挿入対象のバッファ状態や拡張状況が更新され、後続の挿入処理で再利用されます。

前提条件と不変条件

  • BulkInsertStateData は heapam.c と hio.c 間で共有される内部的な状態管理用です。
  • 挿入時のバッファロックは呼び出し側が管理する責任があります。
  • RelationPutHeapTuple 内でエラーが発生した場合は PANIC を使用する必要があります。

アルゴリズムまたは振る舞い

1. **ページ選択**: RelationGetBufferForTuple が本構造体を参照し、効率的な挿入先ページを特定します。 2. **拡張管理**: already_extended_by を用いて、大量挿入時のページ拡張の積極性を調整します。 3. **再チェック**: next_free から last_free までのページは、前回の拡張時に未使用だったページですが、使用中に他プロセスによって利用されている可能性があるため、必要に応じて再チェックが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c（palloc によるインスタンス生成）、および tableam.h で定義されたテーブルアクセスメソッドのインターフェース経由で各所から参照されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を直接呼び出すことはありません。hio.c 内の関数が本構造体のメンバを読み書きします。

関連データ構造

  • BufferAccessStrategy: バッファ管理戦略を定義。
  • Buffer: バッファプール内のページを指すハンドル。
  • RelationData: 将来的に next_free 等のフィールドを保持する候補として言及されています。

失敗モードと境界条件

  • ページ拡張が必要な状況で、ディスク容量不足やロック競合が発生する可能性があります。
  • next_free から last_free の範囲にあるページが既に他プロセスによって使用されている場合、再チェックにより別のページが選択されます。

保守担当者向けメモ

  • 本構造体は外部モジュールによる直接操作を想定していません。
  • 将来的には、next_free 等の状態を RelationData に移行することが検討されています。
