TimeoutId

定義

  • 種別: enum
  • 定義箇所: src/include/utils/timeout.h:23

目的

PostgreSQL バックエンドにおいて、管理対象となるタイムアウトイベントの種類を識別するために使用されます。

シグネチャまたは形状

typedef enum TimeoutId
{
    STARTUP_PACKET_TIMEOUT,
    DEADLOCK_TIMEOUT,
    LOCK_TIMEOUT,
    STATEMENT_TIMEOUT,
    STANDBY_DEADLOCK_TIMEOUT,
    STANDBY_TIMEOUT,
    STANDBY_LOCK_TIMEOUT,
    IDLE_IN_TRANSACTION_SESSION_TIMEOUT,
    TRANSACTION_TIMEOUT,
    IDLE_SESSION_TIMEOUT,
    IDLE_STATS_UPDATE_TIMEOUT,
    CLIENT_CONNECTION_CHECK_TIMEOUT,
    STARTUP_PROGRESS_TIMEOUT,
    USER_TIMEOUT,
    MAX_TIMEOUTS = USER_TIMEOUT + 10
} TimeoutId;

入力と出力

  • 入力: タイムアウト設定関数（enable_timeout_after 等）や登録関数（RegisterTimeout）の引数として渡されます。
  • 出力: 内部的なタイムアウト管理配列のインデックスや、アクティブなタイムアウトの識別子として機能します。

前提条件と不変条件

  • 優先順位: 複数のタイムアウトが同時に発生した場合、列挙型で定義された順序に従って処理されます。
  • 拡張性: USER_TIMEOUT 以降の領域を使用して、ユーザー定義のタイムアウトを追加可能です。

アルゴリズムまたは振る舞い

  • タイムアウトの識別子として機能し、src/backend/utils/misc/timeout.c 内の all_timeouts 配列や active_timeouts リストの管理に使用されます。
  • RegisterTimeout を通じて、特定の TimeoutId に対して timeout_handler_proc（コールバック関数）を紐付けます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/timeout.c 内の各関数（find_active_timeout, insert_timeout, enable_timeout, RegisterTimeout, enable_timeout_after, enable_timeout_every, enable_timeout_at, disable_timeout）がこの列挙型を参照しています。
  • 呼び出し先: 本シンボル自体は列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • timeout_handler_proc: タイムアウト発生時に実行される関数ポインタ型。
  • EnableTimeoutParams / DisableTimeoutParams: 複数のタイムアウトを操作するための構造体。

失敗モードと境界条件

  • MAX_TIMEOUTS: 定義された最大数を超えるタイムアウト理由を登録しようとした場合、配列の範囲外アクセス等の問題が発生する可能性があります。

保守担当者向けメモ

  • タイムアウトの優先順位を変更する場合は、列挙型の定義順序を慎重に検討してください。
  • 新しいタイムアウト理由を追加する際は、MAX_TIMEOUTS の値が適切に確保されているか確認してください。
