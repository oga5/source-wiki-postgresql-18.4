SerializedClientConnectionInfo

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/init/miscinit.c:1076

目的

クライアント接続に関連する認証情報（認証IDの長さおよび認証方式）を、プロセス間での受け渡しやシリアライズが可能な形式で保持するために使用されます。

シグネチャまたは形状

typedef struct SerializedClientConnectionInfo
{
    int32       authn_id_len;   /* strlen(authn_id), or -1 if NULL */
    UserAuth    auth_method;
} SerializedClientConnectionInfo;

入力と出力

本構造体はデータ保持用であり、関数ではありません。入力として認証IDの長さ（int32）と認証方式（UserAuth）を格納し、出力としてそれらをシリアライズされたメモリ領域から読み出すために利用されます。

前提条件と不変条件

  • authn_id_len が -1 の場合、認証IDは NULL であるとみなされます。
  • 構造体全体のサイズは sizeof(SerializedClientConnectionInfo) によって決定されます。

アルゴリズムまたは振る舞い

この構造体は、接続情報のシリアライズ処理において、必要なメモリサイズを計算する際や、構造体インスタンスを初期化して値を格納する際に利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/init/miscinit.c 内の関数群が、接続情報のシリアライズサイズ計算（add_size）や、構造体のインスタンス化（{0} による初期化）のために参照しています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • UserAuth: 認証方式を定義する列挙型（本構造体のメンバとして使用）。

失敗モードと境界条件

  • authn_id_len に不正な値が設定された場合、後続の認証ID読み取り処理でメモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は miscinit.c 内のシリアライズ処理に密接に関連しています。構造体のメンバを変更する場合は、関連するサイズ計算ロジックおよびシリアライズ/デシリアライズ処理の整合性を確認してください。
