pgstat_get_beentry_by_proc_number

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:1222

目的

指定されたプロセス番号（ProcNumber）に対応するバックエンドのステータス情報（PgBackendStatus）を取得します。

シグネチャまたは形状

PgBackendStatus *pgstat_get_beentry_by_proc_number(ProcNumber procNumber)

入力と出力

  • **入力**: procNumber (対象プロセスの識別番号)
  • **出力**: 対応する PgBackendStatus 構造体へのポインタ。該当するプロセスが見つからない場合は NULL を返します。

前提条件と不変条件

  • 内部で pgstat_get_local_beentry_by_proc_number を呼び出し、ローカルメモリ上のステータスエントリを検索します。
  • 返されるポインタは、LocalPgBackendStatus 構造体内の backendStatus フィールドを指します。

アルゴリズムまたは振る舞い

1. pgstat_get_local_beentry_by_proc_number(procNumber) を呼び出し、ローカルのステータスエントリを取得します。 2. 取得結果が有効（非NULL）であれば、そのエントリ内の backendStatus メンバへのポインタを返します。 3. 取得結果が NULL の場合は、そのまま NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: pgstat_get_local_beentry_by_proc_number
  • **呼び出し元**:
  • src/backend/utils/activity/pgstat_backend.c: pgstat_get_beentry_by_proc_number(procNumber)
  • src/backend/utils/adt/pgstatfuncs.c: pg_stat_get_backend_idset 等の統計情報取得関数群から複数回参照されています。

関連データ構造

  • PgBackendStatus: バックエンドの活動状況を保持する共有メモリ上の構造体。
  • LocalPgBackendStatus: 共有メモリ上のステータスをローカルで管理するためのラッパー構造体。
  • ProcNumber: プロセスを識別するための数値型。

失敗モードと境界条件

  • 指定された procNumber に対応するバックエンドが存在しない場合、関数は NULL を返します。呼び出し側はこの戻り値が NULL である可能性を考慮する必要があります。

保守担当者向けメモ

  • 本関数は、共有メモリ上の情報を直接操作するのではなく、ローカルにコピーされた（あるいはマッピングされた）ステータス情報へのアクセスを提供します。
  • 統計情報ビュー（pg_stat_activity）の実装において、特定のプロセス情報を抽出する際の標準的なインターフェースとして利用されています。
