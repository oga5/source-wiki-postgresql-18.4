descriptor_free

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/descriptor.c:735

目的

SQL 記述子（struct descriptor）に関連するすべてのメモリおよびリソースを解放し、記述子オブジェクト自体を破棄します。

シグネチャまたは形状

static void descriptor_free(struct descriptor *desc);

入力と出力

  • 入力: desc - 解放対象となる struct descriptor へのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • desc が指すメモリ領域は有効である必要があります。
  • 内部の descriptor_item リストおよび desc->name、desc->result は、ecpg_free および PQclear で安全に解放可能な状態である必要があります。

アルゴリズムまたは振る舞い

1. 記述子に保持されている descriptor_item のリストを順次走査し、各アイテムの data メンバーおよびアイテム構造体自体を ecpg_free で解放します。 2. 記述子の名前 (desc->name) を ecpg_free で解放します。 3. 記述子に関連付けられた PGresult オブジェクトを PQclear で解放します。 4. 最後に、記述子構造体本体 (desc) を ecpg_free で解放します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/ecpglib/descriptor.c 内の ECPGdeallocate_desc や、記述子リストの破棄処理から呼び出されます。
  • 呼び出し先: ecpg_free（メモリ解放）、PQclear（libpq の結果セット解放）。

関連データ構造

  • struct descriptor: 記述子のメイン構造体。
  • struct descriptor_item: 記述子内の各項目を保持するリンクリスト構造体。

失敗モードと境界条件

  • desc が NULL の場合、ecpg_free や PQclear の実装に依存しますが、一般的に未定義の動作を引き起こす可能性があります。
  • 内部の各ポインタが既に解放済みである場合、二重解放（double free）が発生するリスクがあります。

保守担当者向けメモ

  • 本関数は static であり、descriptor.c 内でのみ使用されます。
  • PQclear を呼び出しているため、libpq のライフサイクル管理と整合している必要があります。
  • 記述子内のアイテムリストを走査する際、next ポインタを正しく追跡してメモリリークを防ぐ構造になっています。
