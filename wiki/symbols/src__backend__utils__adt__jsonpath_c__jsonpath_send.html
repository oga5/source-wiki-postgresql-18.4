jsonpath_send

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonpath.c:146

目的

jsonpath 型のデータを、クライアントへの送信に適したバイナリ形式（bytea）に変換します。

シグネチャまたは形状

Datum jsonpath_send(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して jsonpath 型のデータ（JsonPath *）を受け取ります。
  • **出力**: bytea 型のバイナリデータとして Datum を返却します。

前提条件と不変条件

  • 入力データは PG_GETARG_JSONPATH_P(0) により正当な JsonPath 構造体として取得される必要があります。
  • 内部的に jsonPathToCstring を使用して文字列形式へ変換し、その後にバイナリプロトコル用のヘッダを付与します。

アルゴリズムまたは振る舞い

1. 入力された JsonPath 構造体を jsonPathToCstring を用いて文字列形式（jtext）に変換します。 2. pq_begintypsend を呼び出し、送信用のバッファを初期化します。 3. バッファの先頭に JSONPATH_VERSION（1バイト）を書き込みます。 4. 変換した文字列データを pq_sendtext でバッファに追加します。 5. 一時的な文字列バッファ jtext を解放します。 6. pq_endtypsend を呼び出し、構築されたバイナリデータを bytea として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の型システム（jsonpath 型の送信処理として呼び出される）。
  • **呼び出し先**:
  • jsonPathToCstring: バイナリから文字列への変換。
  • pq_begintypsend, pq_sendint8, pq_sendtext, pq_endtypsend: libpq プロトコル用のバイナリ構築関数。
  • pfree: メモリ解放。

関連データ構造

  • JsonPath: 入力されるデータ構造。
  • StringInfoData: 文字列およびバイナリ構築用の一時バッファ。

失敗モードと境界条件

  • メモリ不足が発生した場合、PostgreSQL のメモリ管理機構によりエラーがスローされます。
  • jsonPathToCstring が失敗した場合、その例外が伝播します。

保守担当者向けメモ

  • 本関数は jsonpath 型のバイナリ送信プロトコルを定義しています。プロトコルバージョン（JSONPATH_VERSION）を変更する場合は、受信側の jsonpath_recv との整合性に注意してください。
  • 現在の実装では、一度文字列形式を経由して送信を行っています。
