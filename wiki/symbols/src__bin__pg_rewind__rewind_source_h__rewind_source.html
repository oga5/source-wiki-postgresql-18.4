rewind_source

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/rewind_source.h:23

目的

pg_rewind において、データソース（同期元）がリモート（libpq）かローカルかに関わらず、同一のインターフェースでファイル操作やデータ取得を行うための抽象レイヤーを提供します。

シグネチャまたは形状

typedef struct rewind_source
{
    void (*traverse_files) (struct rewind_source *, process_file_callback_t);
    char *(*fetch_file) (struct rewind_source *, const char *, size_t *);
    void (*queue_fetch_range) (struct rewind_source *, const char *, off_t, size_t);
    void (*queue_fetch_file) (struct rewind_source *, const char *, size_t);
    void (*finish_fetch) (struct rewind_source *);
    XLogRecPtr (*get_current_wal_insert_lsn) (struct rewind_source *);
    void (*destroy) (struct rewind_source *);
} rewind_source;

入力と出力

  • **入力**: 各関数ポインタは rewind_source 構造体へのポインタを第一引数として受け取ります。
  • **出力**: fetch_file は取得したファイル内容（malloc 済み、ヌル終端）を返し、get_current_wal_insert_lsn はソースの現在のWAL挿入位置（XLogRecPtr）を返します。

前提条件と不変条件

  • fetch_file が返すバッファは常にヌル終端されています。
  • queue_fetch_range および queue_fetch_file で要求された操作は、finish_fetch が呼び出されるまで実行が保留される場合があります。

アルゴリズムまたは振る舞い

1. **走査**: traverse_files によりソース内の全ファイルを列挙し、コールバックを適用します。 2. **取得要求**: queue_fetch_range / queue_fetch_file を用いて必要なデータ範囲をキューに蓄積します。 3. **一括実行**: finish_fetch を呼び出すことで、キューイングされた全てのI/O要求を確定的に実行します。 4. **終了**: destroy を呼び出し、関連リソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/libpq_source.c 内の各関数がこの構造体の実装を提供しています。
  • **呼び出し先**: process_file_callback_t（ファイル走査時のコールバック）が呼び出されます。

関連データ構造

  • XLogRecPtr: WAL位置の表現に使用。
  • process_file_callback_t: traverse_files で使用されるコールバック型。

失敗モードと境界条件

  • queue_fetch_file において、ソースが稼働中のサーバーである場合、ファイルが並行して変更される可能性があります。実装は可能な限り指定された長さ以上のデータをコピーし、切り詰められたファイルの転送を避けることが推奨されます。

保守担当者向けメモ

  • 新しいソースタイプ（例: 新しいプロトコルやストレージ）を追加する場合は、本構造体の全ての関数ポインタを実装する必要があります。
  • queue_fetch_range は即時実行ではなくキューイングを意図した設計であるため、実装時はネットワーク効率やI/O最適化を考慮してください。
