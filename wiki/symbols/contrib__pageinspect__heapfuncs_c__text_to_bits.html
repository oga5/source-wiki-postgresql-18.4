text_to_bits

定義

  • 種別: function
  • 定義箇所: contrib/pageinspect/heapfuncs.c:84

目的

'0' および '1' の文字で構成された文字列を、PostgreSQL の内部的なビットマップ表現（bits8 配列）に変換します。

シグネチャまたは形状

static bits8 *text_to_bits(char *str, int len)

入力と出力

  • **入力**:
  • str: 変換対象の文字列（'0' または '1' のみを含む）。
  • len: 文字列の長さ。
  • **出力**:
  • 変換後のビット列を格納した bits8 型のメモリ領域（palloc により確保）。

前提条件と不変条件

  • 入力文字列 str は '0' または '1' の文字のみで構成されている必要があります。
  • len は文字列の実際の長さを正しく反映している必要があります。

アルゴリズムまたは振る舞い

1. len + 1 バイトのメモリを palloc で確保します。 2. 文字列を先頭から走査し、8文字ごとに1バイトの bits8 値を構築します。 3. 各文字が '0' または '1' であることを確認し、その値をビット位置（off % 8）に応じてシフトし、現在のバイト byte に加算します。 4. 8ビット（off % 8 == 7）に達するごとに、構築した byte を結果配列の該当位置に格納します。 5. 文字列の走査完了後、生成されたビット配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pageinspect/heapfuncs.c:488 にて、t_bits_str を変換するために呼び出されています。
  • **呼び出し先**: palloc（メモリ確保）、ereport（エラー報告）、pg_mblen_cstr（エラーメッセージ生成用）。

関連データ構造

  • bits8: PostgreSQL で定義される 8 ビット符号なし整数型。

失敗モードと境界条件

  • **エラー**: 入力文字列に '0' または '1' 以外の文字が含まれている場合、ERRCODE_DATA_CORRUPTED を伴うエラーが発生し、処理が中断されます。

保守担当者向けメモ

  • 本関数は palloc を使用しているため、呼び出し元でメモリコンテキストが適切に管理されていることを確認してください。
  • エラー発生時の pg_mblen_cstr は、不正な文字を含む文字列の長さを安全に取得するために使用されています。
