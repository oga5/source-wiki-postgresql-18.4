TSQueryParserStateData

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsquery.c:78

目的

tsquery 型の文字列を解析し、ポーランド記法（前置記法）のツリー構造へ変換する過程で、解析状態、トークン、スタック、およびオペランドバッファを保持するために使用されます。

シグネチャまたは形状

struct TSQueryParserStateData {
    ts_tokenizer gettoken;
    char *buffer;
    char *buf;
    int count;
    ts_parserstate state;
    List *polstr;
    char *op;
    char *curop;
    int lenop;
    int sumlen;
    TSVectorParseState valstate;
    Node *escontext;
};

入力と出力

  • **入力**: 解析対象のクエリ文字列（buffer）およびトークナイザ関数（gettoken）。
  • **出力**: 解析結果として polstr（ポーランド記法リスト）および op（収集されたオペランド文字列）が構築されます。

前提条件と不変条件

  • escontext は valstate の escontext と一致している必要があります。
  • count は括弧のネストレベルを追跡し、( でインクリメント、) でデクリメントされます。
  • op バッファは sumlen（使用サイズ）と lenop（確保サイズ）によって管理されます。

アルゴリズムまたは振る舞い

1. **トークン化**: gettoken を用いて入力文字列をスキャンします。 2. **スタック構築**: push* 系関数により、演算子とオペランドを polstr リストへ順次追加します。 3. **オペランド収集**: クエリ内の各オペランド文字列を op バッファに格納し、curop ポインタで末尾を管理します。 4. **エラーハンドリング**: escontext を介して、構文エラー発生時に即座に elog(ERROR) を投げず、呼び出し元で制御可能なソフトエラー処理をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery.c 内の解析関数（parse_tsquery 等）。
  • **呼び出し先**: valstate を通じた TSVectorParseState 関連関数、および push* 系関数。

関連データ構造

  • TSQuery: 解析結果を格納する最終的な構造体。
  • TSVectorParseState: オペランド解析用の状態構造体。
  • List: ポーランド記法リストを保持する PostgreSQL のリスト構造。

失敗モードと境界条件

  • **スタックオーバーフロー**: 再帰的な解析を行うため、check_stack_depth() による深さ制限のチェックが必要です。
  • **メモリ不足**: op バッファの拡張時にメモリ不足が発生する可能性があります。
  • **構文エラー**: 括弧の不整合や不正な演算子配置が発生した場合、escontext を通じてエラーが報告されます。

保守担当者向けメモ

  • 本構造体は tsquery の解析におけるプライベートな状態保持用です。src/include/tsearch/ts_utils.h では不透明ポインタ（TSQueryParserState）として定義されています。
  • 演算子ノードは左オペランドへの相対オフセットを持つため、polstr の構築時にはこのオフセット計算が正しく行われるよう注意してください。
