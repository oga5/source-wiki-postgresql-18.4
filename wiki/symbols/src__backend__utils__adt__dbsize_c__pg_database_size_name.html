pg_database_size_name

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/dbsize.c:190

目的

指定されたデータベース名に対応する物理的なディスクサイズ（バイト単位）を算出する。

シグネチャまたは形状

Datum pg_database_size_name(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_NAME(0) を介してデータベース名（Name 型）を受け取る。
  • **出力**: データベースの合計サイズを int64 型で返す。サイズが 0 の場合は NULL を返す。

前提条件と不変条件

  • 指定されたデータベース名がシステムカタログに存在する必要がある。
  • 実行ユーザーは対象データベースへのアクセス権限、または pg_read_all_stats ロールを保持している必要がある（権限不足時はエラーまたは NULL）。

アルゴリズムまたは振る舞い

1. get_database_oid を呼び出し、入力されたデータベース名から対応する OID を取得する。 2. calculate_database_size(dbOid) を呼び出し、対象データベースの全テーブルスペース上の物理サイズを集計する。 3. 集計結果が 0 の場合は PG_RETURN_NULL() を実行する。 4. それ以外の場合は PG_RETURN_INT64(size) を実行し、バイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（直接の呼び出し元情報はソースコード上では未特定）。
  • **呼び出し先**:
  • get_database_oid: データベース名から OID を解決する。
  • calculate_database_size: データベースの物理サイズを計算する。

関連データ構造

  • Name: データベース名を保持する型。
  • Oid: データベースを一意に識別する識別子。
  • int64: サイズ計算結果を保持する型。

失敗モードと境界条件

  • データベース名が存在しない場合、get_database_oid によりエラーが発生する可能性がある。
  • 権限チェックに失敗した場合、アクセス拒否エラーが発生する。
  • データベースの物理サイズが 0 の場合、関数は NULL を返す。

保守担当者向けメモ

  • 本関数は calculate_database_size に依存しているため、サイズ計算ロジックの変更時には影響を受ける。
  • 長時間のファイル走査が発生する可能性があるため、calculate_database_size 内部での CHECK_FOR_INTERRUPTS() の呼び出しがクエリの応答性に寄与している。
