SendBaseBackup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup.c:989

目的

PostgreSQLのベースバックアップ処理のメインエントリーポイントであり、クライアントからの BASE_BACKUP コマンドに基づき、データディレクトリおよびテーブルスペースのバックアップをストリーミング送信する。

シグネチャまたは形状

void SendBaseBackup(BaseBackupCmd *cmd, IncrementalBackupInfo *ib)

入力と出力

  • **入力**:
  • cmd: バックアップコマンドのオプション情報を含む BaseBackupCmd 構造体。
  • ib: 増分バックアップ用の情報を含む IncrementalBackupInfo 構造体（フルバックアップ時は NULL）。
  • **出力**: なし（バックアップデータを bbsink を通じて送信）。

前提条件と不変条件

  • セッション内で既にバックアップが実行中の場合、エラーが発生する。
  • 増分バックアップが要求されている場合、ib が NULL であってはならない。
  • bbsink インターフェースを介してデータが処理されるため、エラー発生時も bbsink_cleanup が確実に呼び出される必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: parse_basebackup_options でコマンドオプションを解析し、プロセス状態を WALSNDSTATE_BACKUP に設定。 2. **パイプライン構築**: bbsink を使用して、転送先（クライアントまたは外部ターゲット）、スロットリング、圧縮（GZIP/LZ4/ZSTD）、進捗報告の各レイヤーを順次構築。 3. **実行**: perform_base_backup を呼び出し、バックアップのメインループを実行。 4. **クリーンアップ**: PG_TRY / PG_FINALLY ブロックにより、処理終了時またはエラー発生時に必ず bbsink_cleanup を実行してリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c
  • **呼び出し先**:
  • parse_basebackup_options
  • bbsink 関連関数（bbsink_copystream_new, BaseBackupGetSink, bbsink_throttle_new, bbsink_gzip_new 等）
  • perform_base_backup
  • bbsink_cleanup

関連データ構造

  • basebackup_options: バックアップ設定を保持。
  • bbsink: データ送信パイプラインを抽象化する構造体。
  • BaseBackupCmd: コマンド引数。
  • IncrementalBackupInfo: 増分バックアップのメタデータ。

失敗モードと境界条件

  • **セッション状態**: SESSION_BACKUP_RUNNING の場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE でエラー終了。
  • **増分バックアップの不備**: 増分バックアップ指定時にマニフェストが提供されていない場合、エラー終了。
  • **リソース解放**: エラー発生時も PG_FINALLY により bbsink のクリーンアップが保証される。

保守担当者向けメモ

  • 本関数はバックアップの「オーケストレーター」として機能する。個別の圧縮アルゴリズムや転送先の詳細は bbsink 実装に委譲されているため、新しい転送先や圧縮形式を追加する場合は bbsink の拡張を検討すること。
  • perform_base_backup への引数として渡される ib は、増分バックアップの要件に応じて適切にフィルタリングされる。
