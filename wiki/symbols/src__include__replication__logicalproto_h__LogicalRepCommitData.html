LogicalRepCommitData

定義

  • 種別: struct
  • 定義箇所: src/include/replication/logicalproto.h:134

目的

論理レプリケーションにおけるトランザクションコミットメッセージのデータを保持する構造体です。パブリッシャーからサブスクライバーへコミット情報を転送するために使用されます。

シグネチャまたは形状

typedef struct LogicalRepCommitData
{
    XLogRecPtr    commit_lsn;    /* コミット時のLSN */
    XLogRecPtr    end_lsn;       /* トランザクション終了時のLSN */
    TimestampTz   committime;    /* コミット時刻 */
} LogicalRepCommitData;

入力と出力

本構造体は、論理レプリケーションのワイヤプロトコルにおける COMMIT メッセージのデコード結果を格納するコンテナとして機能します。

前提条件と不変条件

  • commit_lsn および end_lsn は有効な XLogRecPtr である必要があります。
  • committime は有効なタイムスタンプ値である必要があります。

アルゴリズムまたは振る舞い

論理レプリケーションの受信側（ワーカー）において、ストリームから読み取られたコミット情報を保持し、apply_handle_commit_internal 等の関数を通じてトランザクションの適用処理に渡されます。

呼び出し元と呼び出し先

  • **呼び出し元（デコード）**:
  • logicalrep_read_commit (src/backend/replication/logical/proto.c)
  • logicalrep_read_stream_commit (src/backend/replication/logical/proto.c)
  • **呼び出し先（利用）**:
  • apply_handle_commit_internal (src/backend/replication/logical/worker.c)
  • その他 src/backend/replication/logical/worker.c 内のコミット処理ロジック

関連データ構造

  • XLogRecPtr: WAL上の位置を指すポインタ型。
  • TimestampTz: PostgreSQLのタイムスタンプ型。

失敗モードと境界条件

  • ストリームからの読み取り時にデータが破損している場合、デコード関数がエラーを発生させる可能性があります。
  • commit_lsn が end_lsn より大きい場合など、論理的に矛盾する値が含まれる場合の挙動は、呼び出し側のバリデーションに依存します。
