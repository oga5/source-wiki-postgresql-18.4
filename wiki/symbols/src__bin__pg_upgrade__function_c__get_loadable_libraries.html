get_loadable_libraries

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/function.c:78

目的

旧クラスタで使用されている共有ライブラリ名および論理レプリケーションの出力プラグイン名を収集し、os_info.libraries に格納する。

シグネチャまたは形状

void get_loadable_libraries(void)

入力と出力

  • **入力**: グローバル変数 old_cluster（旧クラスタのデータベース情報およびレプリケーションスロット情報）。
  • **出力**: グローバル変数 os_info.libraries（収集されたライブラリ情報の配列）および os_info.num_libraries（ライブラリの総数）。

前提条件と不変条件

  • old_cluster が初期化済みであること。
  • upgrade_task を介して旧クラスタの各データベースから probin（ライブラリパス）をクエリ可能であること。

アルゴリズムまたは振る舞い

1. UpgradeTask を作成し、pg_proc カタログから probin を取得するクエリを登録する。 2. upgrade_task_run を実行し、各データベースからライブラリパスを収集する。 3. 収集したライブラリ数と、count_old_cluster_logical_slots() で得られる論理レプリケーションスロットのプラグイン数の合計分、os_info.libraries をメモリ確保する。 4. 各データベースを走査し、以下の情報を os_info.libraries にコピーする。

  • pg_proc から取得したライブラリパス。
  • 有効な論理レプリケーションスロットのプラグイン名。

    • 5. 一時的なクエリ結果およびタスク用メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/check.c:610
  • **呼び出し先**:
  • upgrade_task_create, upgrade_task_add_step, upgrade_task_run, upgrade_task_free
  • process_loadable_libraries (コールバックとして使用)
  • count_old_cluster_logical_slots
  • PQntuples, PQgetvalue, PQclear (libpq)
  • pg_malloc, pg_strdup, pg_free

関連データ構造

  • UpgradeTask: データベースクエリの実行管理。
  • loadable_libraries_state: クエリ結果を一時保持する構造体。
  • LibraryInfo: os_info.libraries の要素型。
  • old_cluster: クラスタ情報。

失敗モードと境界条件

  • pg_malloc 失敗時はメモリ不足で終了する。
  • 重複するプラグイン名が含まれる可能性があるが、後続の check_loadable_libraries で重複排除されるため、本関数内では考慮しない。
  • 無効なレプリケーションスロットはスキップされる。

保守担当者向けメモ

  • 本関数はライブラリのパスを収集するのみであり、実際のロード検証は check_loadable_libraries が担当する。
  • os_info.libraries のメモリ確保サイズは、pg_proc の結果数とスロット数の合計に依存する。
