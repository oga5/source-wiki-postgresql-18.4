ReplicationKind

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/replnodes.h:20

目的

PostgreSQLのレプリケーションプロトコルにおいて、物理レプリケーションと論理レプリケーションを識別するために使用されます。

シグネチャまたは形状

typedef enum ReplicationKind
{
    REPLICATION_KIND_PHYSICAL,
    REPLICATION_KIND_LOGICAL,
} ReplicationKind;

入力と出力

  • 入力: なし（列挙型定義）
  • 出力: REPLICATION_KIND_PHYSICAL または REPLICATION_KIND_LOGICAL のいずれかの値。

前提条件と不変条件

  • レプリケーションコマンドのパラメータを保持する構造体（StartReplicationCmd 等）のメンバとして使用されます。

アルゴリズムまたは振る舞い

  • 実行エンジンがレプリケーションの種類を判定する際のフラグとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: 構文解析器（src/include/nodes/replnodes.h 内の構造体定義を通じて参照）。
  • 呼び出し先: 実行エンジン（src/backend/replication/ 配下の実装がこの値に基づいて処理を分岐させる）。
  • 参照箇所:
  • src/include/nodes/replnodes.h:56
  • src/include/nodes/replnodes.h:94
  • src/include/replication/walsender_private.h:78

関連データ構造

  • StartReplicationCmd: レプリケーション開始コマンドのパラメータを保持する構造体。
  • Walsender: レプリケーション送信プロセスの状態を管理する構造体（walsender_private.h 経由）。

失敗モードと境界条件

  • 定義された2種類以外の値が代入された場合の挙動は未定義です。

保守担当者向けメモ

  • 新しいレプリケーションの種類を追加する場合は、この列挙型を拡張し、対応するバックエンドの処理ロジックを更新する必要があります。
