ResetReindexState

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/index.c:4212

目的

REINDEX操作に関連するグローバルな状態をリセットし、サブトランザクションの失敗時などに上位レベルの状態が破壊されるのを防ぐために使用されます。

シグネチャまたは形状

void ResetReindexState(int nestLevel)

入力と出力

  • **入力**: nestLevel (int) - 現在のトランザクションのネストレベル。
  • **出力**: なし（副作用としてグローバル変数を更新）。

前提条件と不変条件

  • REINDEX操作は再入不可能（non-reentrant）であるという前提に基づいています。
  • reindexingNestLevel が現在の nestLevel 以上である場合にのみ、状態のリセットが実行されます。

アルゴリズムまたは振る舞い

1. 引数 nestLevel とグローバル変数 reindexingNestLevel を比較します。 2. 条件を満たす場合、以下のグローバル変数を初期化します。

  • currentlyReindexedHeap を InvalidOid に設定。
  • currentlyReindexedIndex を InvalidOid に設定。
  • pendingReindexedIndexes を NIL に設定。
  • reindexingNestLevel を 0 に設定。

    • 3. pendingReindexedIndexes の内容はトランザクション寿命のメモリコンテキストにあるため、明示的な解放は行いません。

  • **呼び出し先**:

関連データ構造

  • currentlyReindexedHeap (Oid)
  • currentlyReindexedIndex (Oid)
  • pendingReindexedIndexes (List)
  • reindexingNestLevel (int)

失敗モードと境界条件

  • サブトランザクション内での失敗時に呼び出されることを想定しており、ネストレベルの比較により、意図しない上位レベルの状態リセットを回避する設計となっています。

保守担当者向けメモ

  • 本関数はREINDEX処理のクリーンアップを担います。状態管理の複雑さを避けるため、再入不可能な設計であることを維持してください。
