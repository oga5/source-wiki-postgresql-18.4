gbt_oidge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_oid.c:33

目的

2つの Oid 型の値に対し、左辺が右辺以上であるか（>=）を判定します。

シグネチャまたは形状

static bool gbt_oidge(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の Oid 値へのポインタ。
  • b: 比較対象の Oid 値へのポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • a が b 以上であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は、有効な Oid 型の値へのポインタである必要があります。

アルゴリズムまたは振る舞い

1. ポインタ a および b を const Oid * にキャストします。 2. それぞれのポインタが指す値を参照し、>= 演算子を用いて大小比較を行います。 3. 比較結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_oid.c 内の静的配列（tinfo 構造体の初期化箇所など）から参照されています。
  • **呼び出し先**: PostgreSQLの標準的な比較演算子 >= を使用します。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子型。

失敗モードと境界条件

  • ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける Oid 型の比較サポート関数の一つです。
  • btree_utils_num.h で定義される汎用的な数値比較ロジックと整合するように設計されています。
