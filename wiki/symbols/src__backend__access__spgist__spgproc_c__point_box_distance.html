point_box_distance

定義

  • src/backend/access/spgist/spgproc.c:30

目的

点と軸平行なボックス（BOX）間の最短距離を計算します。SP-GiSTインデックスにおける最近傍探索（KNN）の距離計算を支援します。

シグネチャまたは形状

static double point_box_distance(Point *point, BOX *box)

入力と出力

  • **入力**:
  • point: 距離計算の起点となる Point 構造体へのポインタ。
  • box: 距離計算の対象となる BOX 構造体へのポインタ。
  • **出力**:
  • double: 点とボックス間の最短距離。いずれかの座標が NaN の場合は NaN を返します。

前提条件と不変条件

  • box は軸に平行である必要があります。
  • point および box は有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. 入力された点またはボックスの座標に NaN が含まれるか確認し、含まれる場合は NaN を返します。 2. 点の X 座標がボックスの範囲外にある場合、ボックスの境界までの水平距離 dx を算出します。範囲内の場合は dx = 0.0 とします。 3. 点の Y 座標がボックスの範囲外にある場合、ボックスの境界までの垂直距離 dy を算出します。範囲内の場合は dy = 0.0 とします。 4. HYPOT(dx, dy) を用いて、算出した dx と dy から斜辺（最短距離）を計算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgproc.c 内の関数（spg_key_orderbys_distances 等から参照されることを示唆する証拠あり）。
  • **呼び出し先**: isnan、get_float8_nan、HYPOT などの数学ユーティリティ。

関連データ構造

  • Point: 幾何学的な点（utils/geo_decls.h で定義）。
  • BOX: 軸平行な矩形（utils/geo_decls.h で定義）。

失敗モードと境界条件

  • 入力座標に NaN が含まれる場合、計算結果として NaN を返します。
  • 点がボックスの内部または境界上にある場合、距離は 0.0 となります。

保守担当者向けメモ

  • 本関数はボックスが軸に平行であることを前提としています。
  • HYPOT マクロは math.h の機能を利用しており、オーバーフローやアンダーフローを考慮した距離計算を行います。
