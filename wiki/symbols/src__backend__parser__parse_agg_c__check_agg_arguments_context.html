check_agg_arguments_context

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_agg.c:36

目的

集約関数の引数に対する意味解析を行う際、クエリレベルのネスト状況や変数の参照範囲を追跡するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct
{
	ParseState *pstate;
	int			min_varlevel;
	int			min_agglevel;
	int			min_ctelevel;
	RangeTblEntry *min_cte;
	int			sublevels_up;
} check_agg_arguments_context;

入力と出力

本構造体は関数（check_agg_arguments 等）の引数として渡され、解析過程で各フィールドの値が更新・参照されます。

前提条件と不変条件

  • pstate は有効な ParseState を指している必要があります。
  • sublevels_up は、現在解析中の式がクエリのどのレベルにあるかを示す非負の整数です。

アルゴリズムまたは振る舞い

集約関数の引数を走査する際、この構造体を使用して以下の情報を保持します。

  • min_varlevel, min_agglevel: 走査中に見つかった変数や集約関数の最小レベル。
  • min_ctelevel, min_cte: 走査中に見つかったCTE（共通テーブル式）のレベルとエントリ。

    • これらは、集約関数が許可されていない場所で使用されていないか、あるいは正しいクエリレベルを参照しているかを検証するために使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/parser/parse_agg.c 内の check_agg_arguments および関連する解析関数。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ParseState: パーサーの状態を保持する構造体。
  • RangeTblEntry: クエリの範囲テーブルエントリ。

失敗モードと境界条件

  • sublevels_up の値が不適切な場合、集約関数のネスト判定やスコープ判定が正しく行われない可能性があります。

保守担当者向けメモ

  • 本構造体は check_agg_arguments_walker 等の再帰的な走査処理において、状態を維持するために不可欠です。
  • sublevels_up の管理は複雑なため、変更を加える際は expression_tree_walker 等の再帰的な走査ロジックとの整合性に注意してください。
