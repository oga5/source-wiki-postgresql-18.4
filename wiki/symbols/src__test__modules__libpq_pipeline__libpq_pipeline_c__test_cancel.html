test_cancel

定義

  • 種別: function
  • 定義箇所: src/test/modules/libpq_pipeline/libpq_pipeline.c:243

目的

PostgreSQL の libpq が提供する各種クエリキャンセル API（PQcancel, PQrequestCancel, PQcancelBlocking, PQcancelPoll）の動作を検証するテスト関数です。

シグネチャまたは形状

static void test_cancel(PGconn *conn)

入力と出力

  • 入力: PGconn *conn（テスト対象のデータベース接続）
  • 出力: なし（失敗時は pg_fatal を呼び出しプロセスを終了する）

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • 関数内で PQsetnonblocking(conn, 1) が実行され、接続が非ブロッキングモードに設定される。
  • キャンセル対象のクエリ実行中、monitorConn を用いて並行して監視を行う。

アルゴリズムまたは振る舞い

1. **初期化**: 接続を非ブロッキングモードにし、監視用の monitorConn を作成する。 2. **PQcancel テスト**: PQgetCancel で取得したオブジェクトを使用し、クエリをキャンセルする。再利用可能であることを確認する。 3. **PQrequestCancel テスト**: 非同期的なキャンセル要求を送信し、結果を検証する。 4. **PQcancelBlocking テスト**: PQcancelCreate で作成した接続を用い、ブロッキング形式でキャンセルを実行する。 5. **PQcancelPoll テスト**: PQcancelStart と select() を用いた I/O 多重化による非同期キャンセル処理を検証する。 6. **PQcancelReset テスト**: キャンセル接続をリセットし、再利用可能であることを確認する。 7. **終了**: 各種リソース（PGcancel, PGcancelConn）を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/libpq_pipeline/libpq_pipeline.c 内のメインテストルーチン（行 2358 等）。
  • 呼び出し先: copy_connection, send_cancellable_query, confirm_query_canceled, pg_fatal 等のヘルパー関数、および libpq の各種 API（PQcancel, PQcancelPoll, select 等）。

関連データ構造

  • PGconn: データベース接続ハンドル。
  • PGcancel: キャンセル要求用オブジェクト。
  • PGcancelConn: キャンセル接続用オブジェクト。

失敗モードと境界条件

  • PQsetnonblocking 失敗時: pg_fatal により終了。
  • PQcancel 等の API がエラーを返した場合: errorbuf に格納されたメッセージと共に pg_fatal で終了。
  • select() 実行時: EINTR（割り込み）は無視して継続するが、その他のエラーは pg_fatal で終了。
  • PQcancelStatus が CONNECTION_OK 以外の場合: pg_fatal で終了。

保守担当者向けメモ

  • 本関数は libpq のキャンセル API の網羅的なテストケースとして機能します。新しいキャンセル関連 API が追加された場合は、本関数にテストケースを追加してください。
  • select() を使用したポーリングループは、タイムアウト（3秒）を設けており、デッドロックを防止しています。
