pg_comp_crc32c_loongarch

定義

  • 種別: function
  • 定義箇所: src/port/pg_crc32c_loongarch.c:19

目的

LoongArch アーキテクチャのハードウェア命令（CRCC）を活用し、データバッファに対する CRC-32C チェックサムを高速に計算する。

シグネチャまたは形状

pg_crc32c pg_comp_crc32c_loongarch(pg_crc32c crc, const void *data, size_t len)

入力と出力

  • **入力**:
  • crc: 計算の初期値（または前回の計算結果）。
  • data: 計算対象のデータバッファへのポインタ。
  • len: データバッファの長さ（バイト単位）。
  • **出力**:
  • 更新後の pg_crc32c 型のチェックサム値。

前提条件と不変条件

  • LoongArch コンパイラの組み込み関数（__builtin_loongarch_crcc_*）が利用可能であること。
  • メモリアライメントは必須ではないが、8バイト境界に揃えることでパフォーマンスが向上する。

アルゴリズムまたは振る舞い

1. **先頭の端数処理**: 入力ポインタ p が 8 バイト境界に揃っていない場合、1/2/4 バイト単位で処理を行い、ポインタを 8 バイト境界に調整する。 2. **メインループ**: 8 バイト単位で __builtin_loongarch_crcc_w_d_w を呼び出し、高速に計算を行う。 3. **残余処理**: ループ終了後、残った 7 バイト以下のデータを 4/2/1 バイト単位で順次処理し、最終的な値を算出する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/include/port/pg_crc32c.h 内で参照されている（直接の呼び出し元コードの詳細は提供されていない）。
  • **呼び出し先**: LoongArch 組み込み関数 __builtin_loongarch_crcc_w_b_w, __builtin_loongarch_crcc_w_h_w, __builtin_loongarch_crcc_w_w_w, __builtin_loongarch_crcc_w_d_w。

関連データ構造

  • pg_crc32c: チェックサム値を保持する型（port/pg_crc32c.h で定義）。

失敗モードと境界条件

  • len が 0 の場合、入力された crc 値がそのまま返される。
  • バッファの長さが 8 バイト未満の場合、メインループはスキップされ、残余処理のみが実行される。

保守担当者向けメモ

  • 本実装はハードウェア命令に依存しているため、コンパイラが LoongArch の組み込み関数をサポートしている環境でのみ有効である。
  • パフォーマンス最適化のため、アライメント調整の順序（1/2/4バイト）は厳密に維持すること。
