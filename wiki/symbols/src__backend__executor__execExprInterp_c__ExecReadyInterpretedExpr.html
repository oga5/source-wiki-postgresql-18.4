ExecReadyInterpretedExpr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execExprInterp.c:249

目的

式評価（Expression Evaluation）の準備段階において、インタープリタ型実行エンジンの初期化を行い、単純な式に対してはオーバーヘッドを削減するための高速パス（Fast-path）関数を割り当てる。

シグネチャまたは形状

void ExecReadyInterpretedExpr(ExprState *state)

入力と出力

  • **入力**: ExprState *state（評価対象の式状態構造体）
  • **出力**: なし（副作用として state の evalfunc および evalfunc_private フィールドを更新する）

前提条件と不変条件

  • state->steps_len >= 1 であること。
  • ステップリストの末尾が EEOP_DONE_RETURN または EEOP_DONE_NO_RETURN であること。
  • EEO_FLAG_DIRECT_THREADED フラグが設定されていないこと。

アルゴリズムまたは振る舞い

1. ExecInitInterpreter() を呼び出し、インタープリタの初期化を行う。 2. state->flags に EEO_FLAG_INTERPRETER_INITIALIZED を設定し、二重初期化を防止する。 3. state->evalfunc に ExecInterpExprStillValid を設定し、初回実行時の検証用コールバックを登録する。 4. 式のステップ数（steps_len）と各ステップの opcode を検査し、特定のパターン（変数参照、定数評価、ハッシュデータ操作など）に合致する場合、state->evalfunc_private に最適化された ExecJust* 系関数を割り当てて終了する。 5. 高速パスに該当しない場合、デフォルトのインタープリタ実行が継続される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execExpr.c 内の式コンパイルプロセスから呼び出される。
  • **呼び出し先**: ExecInitInterpreter を呼び出す。また、最適化パスとして ExecJust* 系関数（ExecJustInnerVar, ExecJustConst 等）を evalfunc_private に代入する。

関連データ構造

  • ExprState: 式の実行状態を保持する構造体。
  • ExprEvalStep: 式を構成する個別の命令ステップ。
  • ExprEvalOp: 命令の種類を定義する列挙型。

失敗モードと境界条件

  • steps_len が不正な場合、Assert により停止する。
  • 既に初期化済みの state に対しては、何もせず即座に復帰する。

保守担当者向けメモ

  • 本関数は式評価のホットパスにおける初期化処理である。新しい高速パスを追加する場合は、steps_len ごとの分岐内にパターンを追加すること。
  • ExecJust* 系関数は、PostgreSQL の式評価におけるオーバーヘッドを最小化するために重要である。複雑なロジックを追加する際は、既存の ExecInterpExpr の仕組みを優先的に検討すること。
