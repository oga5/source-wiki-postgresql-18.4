ScanDirection

定義

  • 種別: enum
  • 定義箇所: src/include/access/sdir.h:24

目的

テーブルやインデックスのスキャン処理における進行方向（順方向、逆方向、停止）を抽象化し、アクセス手法間で一貫した状態管理を行うために定義されています。

シグネチャまたは形状

typedef enum ScanDirection
{
	BackwardScanDirection = -1,
	NoMovementScanDirection = 0,
	ForwardScanDirection = 1
} ScanDirection;

入力と出力

  • 入力: スキャン実行関数（amgettuple 等）において、スキャン方向を制御する引数として受け取られます。
  • 出力: 該当しません（列挙型定義のため）。

前提条件と不変条件

  • ForwardScanDirection は 1、BackwardScanDirection は -1 に固定されています。この数値は、乗算による方向反転（ScanDirectionCombine マクロ等）を可能にするための設計上の不変条件です。

アルゴリズムまたは振る舞い

  • スキャン実行時、この列挙値に基づいて次に取得すべきタプルの位置が決定されます。
  • NoMovementScanDirection はスキャンの初期化や停止状態を表しますが、実際のデータ取得処理では通常使用されません。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/auto_explain、contrib/pg_stat_statements、および各アクセス手法の実行関数（gistgettuple、hashgettuple 等）から引数として参照されています。
  • 呼び出し先: 本シンボルは列挙型定義であり、直接的な呼び出し先はありません。

関連データ構造

  • ScanDesc（src/include/access/relscan.h）: スキャン状態を保持する構造体であり、メンバとして ScanDirection を保持します。

失敗モードと境界条件

  • 有効な値の範囲外を指定した場合の挙動は、各アクセス手法の実装に依存します。ScanDirectionIsValid マクロを使用して、値の正当性を検証することが推奨されます。

保守担当者向けメモ

  • 値（-1, 0, 1）を変更すると、ScanDirectionCombine を利用した演算ロジックが破壊されるため、変更には注意が必要です。
