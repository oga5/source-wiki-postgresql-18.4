sepgsql_avc_check_perms

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/uavc.c:419

目的

指定されたオブジェクトに対するSELinux権限チェックを実行します。ユーザー空間アクセスベクタキャッシュ（AVC）を利用して判定を行い、必要に応じて監査ログの生成や違反時の処理を行います。

シグネチャまたは形状

bool sepgsql_avc_check_perms(const ObjectAddress *tobject, uint16 tclass, uint32 required, const char *audit_name, bool abort_on_violation)

入力と出力

  • **入力**:
  • tobject: チェック対象のオブジェクトアドレス。
  • tclass: オブジェクトクラス（SEPG_CLASS_*）。
  • required: 要求される権限ビットマスク。
  • audit_name: 監査ログ出力時に使用する識別名。
  • abort_on_violation: 権限違反時にエラーを発生させるかどうかのフラグ。
  • **出力**:
  • bool: 権限が許可されている場合は true、拒否されている場合は false。

前提条件と不変条件

  • GetSecurityLabel を介して対象オブジェクトのセキュリティラベルが取得可能であること。
  • 取得したセキュリティラベルは、関数終了時に pfree で解放される。

アルゴリズムまたは振る舞い

1. GetSecurityLabel を呼び出し、対象オブジェクトのセキュリティラベル文字列を取得します。 2. 取得したラベル文字列を引数として sepgsql_avc_check_perms_label を呼び出し、実際の権限判定を行います。 3. 取得したセキュリティラベル文字列が非NULLであれば、pfree を実行してメモリを解放します。 4. sepgsql_avc_check_perms_label の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/database.c
  • contrib/sepgsql/dml.c
  • contrib/sepgsql/label.c
  • contrib/sepgsql/proc.c
  • **呼び出し先**:
  • GetSecurityLabel (PostgreSQL内部関数)
  • sepgsql_avc_check_perms_label (contrib/sepgsql/uavc.c)
  • pfree (PostgreSQLメモリ管理関数)

関連データ構造

  • ObjectAddress: PostgreSQLのオブジェクトを特定するための構造体。

失敗モードと境界条件

  • GetSecurityLabel がNULLを返した場合、sepgsql_avc_check_perms_label はラベルなしとして処理を継続します。
  • abort_on_violation が true の場合、権限違反時に sepgsql_avc_check_perms_label 内部でエラーが発生し、呼び出し元へ戻らない可能性があります。

保守担当者向けメモ

  • 本関数はラッパーであり、実質的なキャッシュ検索や計算ロジックは sepgsql_avc_check_perms_label に委譲されています。
  • 権限チェックのループ処理やポリシー更新の検証が必要な場合は、呼び出し先である sepgsql_avc_check_perms_label の実装を確認してください。
