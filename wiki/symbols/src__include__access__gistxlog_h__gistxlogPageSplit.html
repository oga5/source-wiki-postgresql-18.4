gistxlogPageSplit

定義

  • 種別: struct
  • 定義箇所: src/include/access/gistxlog.h:68

目的

GiSTインデックスのページ分割操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。リカバリ時に分割前の状態を再現し、インデックスの整合性を維持するために使用されます。

シグネチャまたは形状

typedef struct gistxlogPageSplit
{
    BlockNumber origrlink;      /* 分割前のページの右リンク */
    GistNSN     orignsn;        /* 分割前のページのNSN */
    bool        origleaf;       /* 分割されたページがリーフページか否か */
    uint16      npage;          /* 分割後のページ数 */
    bool        markfollowright; /* F_FOLLOW_RIGHTフラグを設定するか */
} gistxlogPageSplit;

入力と出力

  • **入力**: GiSTインデックスのページ分割発生時に、この構造体に現在のページ状態が格納されます。
  • **出力**: WALレコードの一部として書き込まれ、リカバリ時に gist_redo がこの構造体を読み取ってページ再構築を行います。

前提条件と不変条件

  • 本構造体の直後に、gistxlogPage 構造体と各ページに対応する IndexTupleData の配列が続くレイアウトを前提としています。
  • 構造体のメンバを変更した場合は、対応する gist_redo および gist_desc の実装も更新する必要があります。

アルゴリズムまたは振る舞い

1. **記録**: GiSTインデックスのページ分割時、分割前のページ情報（origrlink, orignsn, origleaf）と分割後の構成情報（npage, markfollowright）を本構造体に設定します。 2. **WAL挿入**: XLogRegisterData を介してWALレコードに登録されます。 3. **リプレイ**: リカバリ時に gist_redo がレコードを解析し、XLogReaderState の block_id を用いて分割後の各ページを再構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistxlog.c 内のGiST WAL記録処理。
  • **呼び出し先**: src/backend/access/rmgrdesc/gistdesc.c 内の out_gistxlogPageSplit 関数（デバッグ出力用）。

関連データ構造

  • gistxlogPage: ページ分割後の各ページ情報を保持する構造体。
  • IndexTupleData: ページ内に格納されるインデックスエントリ。

失敗モードと境界条件

  • 複数のページにまたがる操作であるため、XLogReaderState によるバッファ管理が不適切な場合、ページ整合性が損なわれる可能性があります。

保守担当者向けメモ

  • 本構造体はWALのバイナリレイアウトと直結しています。フィールドの追加や削除を行う際は、バイナリ互換性に注意してください。
  • gist_redo および gist_desc の実装と常に同期を取る必要があります。
