eary

定義

  • 種別: struct
  • 定義箇所: contrib/oid2name/oid2name.c:21

目的

コマンドライン引数で指定された複数の検索対象（OID、テーブル名、ファイルノード番号）を動的に保持し、SQL クエリの IN 句を生成するためのリスト構造体です。

シグネチャまたは形状

typedef struct
{
	char	  **array;
	int			num;
	int			alloc;
} eary;

入力と出力

  • 入力: add_one_elt 関数を通じて文字列要素が追加されます。
  • 出力: get_comma_elts 関数を通じて、保持している要素をカンマ区切りの文字列として取得します。

前提条件と不変条件

  • alloc は現在確保されている array の容量（要素数）を示します。
  • num は現在格納されている有効な要素数を示します。
  • num が alloc に達した場合、動的なメモリ再割り当てが必要です。

アルゴリズムまたは振る舞い

  • 要素追加時、alloc が 0 の場合は初期値として 8 要素分のメモリを確保します。
  • num が alloc を超える場合、容量を拡張して要素を格納します。
  • 最終的に格納された要素を連結し、SQL クエリ構築用の文字列を生成します。

呼び出し元と呼び出し先

  • 呼び出し元: main 関数内で tables、oids、filenumbers としてインスタンス化されます。
  • 呼び出し先: add_one_elt が要素の追加を行い、get_comma_elts が SQL 構築用の文字列生成を行います。

関連データ構造

  • options: 実行時のオプションを保持する構造体であり、eary 型のメンバを保持して検索対象を管理します。

失敗モードと境界条件

  • メモリ不足時: pg_malloc が失敗した場合、プログラムは終了します。
  • 空のリスト: num が 0 の場合、get_comma_elts は空の文字列または適切な初期値を返す必要があります。

保守担当者向けメモ

  • 本構造体は oid2name ユーティリティにおける動的配列の汎用的な実装です。要素の追加や文字列変換を行う際は、関連する add_one_elt および get_comma_elts 関数の実装と整合性を保つ必要があります。
