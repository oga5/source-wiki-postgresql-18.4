logicalrep_rel_att_by_name

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/relation.c:208

目的

リモートリレーションの定義情報から、指定された列名に対応する列番号（インデックス）を検索する。

シグネチャまたは形状

static int logicalrep_rel_att_by_name(LogicalRepRelation *remoterel, const char *attname)

入力と出力

  • **入力**:
  • remoterel: リモートリレーションの構造体ポインタ。
  • attname: 検索対象の列名（文字列）。
  • **出力**:
  • 成功時: 見つかった列のインデックス（0 以上の整数）。
  • 失敗時: -1。

前提条件と不変条件

  • remoterel は有効な LogicalRepRelation 構造体を指している必要がある。
  • remoterel->attnames 配列が初期化されていることを前提とする。

アルゴリズムまたは振る舞い

1. remoterel が保持する列数（natts）分だけループする。 2. 各列名 remoterel->attnames[i] と引数 attname を strcmp で比較する。 3. 一致した時点でそのインデックス i を即座に返す。 4. 全ての列を確認しても一致しない場合は -1 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/relation.c:453（直接的な呼び出しを確認）
  • **呼び出し先**:
  • 標準ライブラリ関数 strcmp を呼び出す。

関連データ構造

  • LogicalRepRelation: リモートリレーションのメタデータ（列名配列 attnames や列数 natts を含む）を保持する構造体。

失敗モードと境界条件

  • 指定された列名がリモートリレーション内に存在しない場合、-1 を返す。
  • remoterel->natts が 0 の場合、ループは実行されず即座に -1 を返す。

保守担当者向けメモ

  • 本関数は線形探索を行うため、列数が多いテーブルでは呼び出し頻度に注意すること。
  • LogicalRepRelation 構造体の定義については src/include/replication/logicalrelation.h を参照のこと。
