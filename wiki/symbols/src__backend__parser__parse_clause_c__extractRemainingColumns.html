extractRemainingColumns

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:252

目的

入力テーブルの列リストから、既にマージ済みの列（USING 句などで処理された列）を除外した残りの列を抽出し、出力用の列リストおよび名前空間情報を構築します。

シグネチャまたは形状

static int extractRemainingColumns(ParseState *pstate,
                                   ParseNamespaceColumn *src_nscolumns,
                                   List *src_colnames,
                                   List **src_colnos,
                                   List **res_colnames, List **res_colvars,
                                   ParseNamespaceColumn *res_nscolumns)

入力と出力

  • **入力**:
  • pstate: 解析状態。
  • src_nscolumns: 入力テーブルの列メタデータ配列。
  • src_colnames: 入力テーブルの列名リスト。
  • src_colnos: 既にマージ済みの列番号リスト（入出力両用）。
  • **出力**:
  • src_colnos: 更新された列番号リスト。
  • res_colnames: 抽出された列名のリスト。
  • res_colvars: 抽出された列に対応する Var ノードのリスト。
  • res_nscolumns: 抽出された列のメタデータ配列。
  • **戻り値**: 抽出された列の総数（int）。

前提条件と不変条件

  • src_colnos に含まれる列番号は、既に処理済みとしてスキップされます。
  • colname が空文字列（'\0'）である列（削除された列）は無視されます。

アルゴリズムまたは振る舞い

1. src_colnos を走査し、既存の列番号を Bitmapset (prevcols) に格納して高速な検索を可能にします。 2. src_colnames を順次走査し、以下の条件を満たす列を抽出対象とします。

  • 列名が空でないこと。
  • prevcols に含まれていないこと。

    • 3. 抽出対象の列について、以下の処理を行います。

  • src_colnos に列番号を追加。
  • res_colnames に列名を追加。
  • buildVarFromNSColumn を呼び出し、対応する Var ノードを生成して res_colvars に追加。
  • res_nscolumns に列メタデータをコピー。

    • 4. 抽出した列の総数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_clause.c 内の関数（行 51, 253, 1516, 1521 での参照を確認）。
  • **呼び出し先**: buildVarFromNSColumn（Var ノード生成用）。

関連データ構造

  • ParseState: 解析状態を保持する構造体。
  • ParseNamespaceColumn: 名前空間内の列情報を保持する構造体。
  • Bitmapset: 処理済み列の追跡に使用。

失敗モードと境界条件

  • 入力テーブルが空の場合、ループは実行されず 0 を返します。
  • 全ての列が既にマージ済みの場合、抽出される列はなく 0 を返します。

保守担当者向けメモ

  • prevcols を使用することで、列数が多い場合でも $O(N)$ の計算量で処理できるよう最適化されています。
  • res_nscolumns は呼び出し元で十分な領域が確保されている必要があります。
