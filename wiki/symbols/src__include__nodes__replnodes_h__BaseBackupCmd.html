BaseBackupCmd

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/replnodes.h:41

目的

レプリケーションプロトコルにおける BASE_BACKUP コマンドのパラメータを保持し、構文解析器から実行エンジンへ情報を伝達するためのパースノード構造体です。

シグネチャまたは形状

typedef struct BaseBackupCmd
{
    NodeTag     type;
    List       *options;
} BaseBackupCmd;

入力と出力

  • **入力**: フロントエンドから送信された BASE_BACKUP コマンドの構文解析結果。
  • **出力**: 実行エンジン（SendBaseBackup 等）が参照するコマンドオプションのリスト。

前提条件と不変条件

  • type フィールドは PostgreSQL のノードシステムの一部として、適切な NodeTag を保持する必要があります。
  • options は List 構造体へのポインタであり、コマンド固有のパラメータを格納します。

アルゴリズムまたは振る舞い

1. **パース時**: レプリケーションプロトコルで BASE_BACKUP が受信されると、構文解析器が本構造体をインスタンス化し、オプションを options リストに格納します。 2. **実行時**: 実行エンジンが本構造体を受け取り、options を解析してバックアップ処理（basebackup.c 等）を実行します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/replication/walsender.c: SendBaseBackup を呼び出す際に本構造体を使用。
  • **呼び出し先**:
  • src/backend/backup/basebackup.c: SendBaseBackup 関数が本構造体を引数として受け取り、バックアップ処理を実行。
  • src/include/backup/basebackup.h: SendBaseBackup の宣言で本構造体を参照。

関連データ構造

  • NodeTag: ノードの種類を識別するためのタグ。
  • List: コマンドオプションを保持するための PostgreSQL 汎用リスト構造。

失敗モードと境界条件

  • options リストが空、または不正なオプションが含まれている場合、実行エンジン側でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/nodes/replnodes.h で定義されており、PostgreSQL のノードシステムの一部として扱われます。
  • 新しいバックアップオプションを追加する場合は、本構造体の options リストを介して拡張可能です。
  • コピーやデバッグ出力が必要な場合は、src/backend/nodes/copyfuncs.c や outfuncs.c への対応が必要となる場合があります。
