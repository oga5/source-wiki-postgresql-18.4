ExecDelete

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeModifyTable.c:1571

目的

テーブル（またはパーティション）に対する行削除操作を実行する。トリガー処理、外部テーブル（FDW）連携、および同時実行制御（EPQ: EvalPlanQual）を伴う削除ロジックを統括する。

シグネチャまたは形状

TupleTableSlot *ExecDelete(ModifyTableContext *context, ResultRelInfo *resultRelInfo, ItemPointer tupleid, HeapTuple oldtuple, bool processReturning, bool changingPart, bool canSetTag, TM_Result *tmresult, bool *tupleDeleted, TupleTableSlot **epqreturnslot)

入力と出力

  • **入力**:
  • context: 実行コンテキスト（ModifyTableContext）。
  • resultRelInfo: ターゲットリレーション情報。
  • tupleid: 削除対象タプルの物理位置（ItemPointer）。
  • oldtuple: 削除対象のタプルデータ。
  • **出力**:
  • 戻り値: 処理されたタプルを保持する TupleTableSlot、または削除がスキップされた場合は NULL。
  • tmresult: テーブルアクセスメソッドによる操作結果（TM_Result）。
  • tupleDeleted: 削除が成功したかを示すフラグ。
  • epqreturnslot: EPQ処理時に更新後のタプルを返すためのスロット。

前提条件と不変条件

  • ExecDeletePrologue が正常終了していること。
  • 外部テーブルの場合、ri_FdwRoutine が定義されていること。
  • TM_SelfModified が発生した場合、現在のコマンドによる二重削除は無視されるが、トリガーによる変更が絡む場合はエラーとなる。

アルゴリズムまたは振る舞い

1. **前処理**: ExecDeletePrologue を呼び出し、BEFORE ROW トリガーを実行する。 2. **トリガー/FDW分岐**:

  • INSTEAD OF トリガーが存在する場合、それを実行し、不要であれば終了する。
  • 外部テーブルの場合、ExecForeignDelete を呼び出し、FDW に削除を委譲する。

    • 3. **削除実行**:

  • ExecDeleteAct を呼び出し、ストレージ層で削除を実行する。
  • TM_Result に基づき、同時更新（TM_Updated）や自己更新（TM_SelfModified）をハンドリングする。

    • 4. **同時実行制御 (EPQ)**:

  • 他セッションによる更新が検出された場合、EvalPlanQual を用いて最新状態での再評価を行う。

    • 5. **後処理**: 必要に応じて epqreturnslot に結果を格納し、呼び出し元へ戻る。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeModifyTable.c 内のメイン実行ループ（ExecModifyTable 等）。
  • **呼び出し先**:
  • ExecDeletePrologue, ExecDeleteAct
  • ExecIRDeleteTriggers
  • EvalPlanQualBegin, EvalPlanQual, EvalPlanQualSlot
  • table_tuple_lock

関連データ構造

  • ModifyTableContext: 実行状態を保持。
  • ResultRelInfo: ターゲットテーブルのメタデータとトリガー情報。
  • TupleTableSlot: タプルデータの受け渡しに使用。
  • TM_Result: テーブルアクセスメソッドの操作結果ステータス。

失敗モードと境界条件

  • **シリアライズ失敗**: IsolationUsesXactSnapshot() 下で同時更新が発生した場合、ERRCODE_T_R_SERIALIZATION_FAILURE を送出。
  • **トリガー違反**: TM_SelfModified 時に現在のコマンドと異なる CID で変更が行われている場合、ERRCODE_TRIGGERED_DATA_CHANGE_VIOLATION を送出。
  • **Qual不一致**: EPQ 実行後にタプルが条件を満たさなくなった場合、NULL を返して終了。

保守担当者向けメモ

  • 本関数は nodeModifyTable.c の中核であり、パーティションルーティングや FDW 連携と密接に関係している。
  • TM_SelfModified のハンドリングは、トリガーとクエリの相互作用による複雑なケースを想定しているため、変更時は注意が必要。
  • ExecDeletePrologue, ExecDeleteAct, ExecDeleteEpilogue といったサブ関数への分割が進んでいるため、ロジックの修正は各サブ関数を確認すること。
