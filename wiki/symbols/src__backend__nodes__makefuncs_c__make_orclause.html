make_orclause

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/makefuncs.c:742

目的

論理和（OR）演算を表す BoolExpr ノードを生成し、初期化する。

シグネチャまたは形状

Expr *make_orclause(List *orclauses)

入力と出力

  • **入力**: orclauses (List *) - OR演算の対象となる式のリスト。
  • **出力**: Expr * - 生成された BoolExpr ノードへのポインタ。

前提条件と不変条件

  • makeNode(BoolExpr) を介してメモリが確保されるため、呼び出し時に適切なメモリコンテキストがアクティブである必要がある。
  • 生成されたノードの boolop フィールドは常に OR_EXPR に設定される。
  • location フィールドはデフォルト値として -1（位置情報なし）に設定される。

アルゴリズムまたは振る舞い

1. makeNode(BoolExpr) を呼び出し、BoolExpr 構造体分のメモリを確保する。 2. expr->boolop に OR_EXPR を代入する。 3. 引数 orclauses を expr->args に格納する。 4. expr->location を -1 に初期化する。 5. 構築したノードを Expr * 型にキャストして返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/plan/createplan.c
  • src/backend/optimizer/plan/subselect.c
  • src/backend/optimizer/prep/prepqual.c
  • **呼び出し先**:
  • makeNode マクロ（src/backend/nodes/makefuncs.c 内で定義されたノード生成用マクロ）。

関連データ構造

  • BoolExpr: 本関数が生成する論理演算ノードの構造体。
  • List: 引数として渡される式のリスト。

失敗モードと境界条件

  • メモリ不足時には makeNode 経由でエラーが発生する可能性がある。
  • orclauses が空リストの場合でもノードは生成されるが、その後のクエリ処理における妥当性は呼び出し側の責務となる。

保守担当者向けメモ

  • 本関数は BoolExpr の標準的なコンストラクタである。location 情報が必要な場合は、本関数呼び出し後に手動でフィールドを更新する必要がある。
