strnxfrm_icu

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_icu.c:497

目的

ICU（International Components for Unicode）の照合順序を用いて、文字列のソートキー（Sort Key）を生成します。

シグネチャまたは形状

size_t strnxfrm_icu(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • **入力**:
  • dest: ソートキーを格納するバッファ（NULL可）。
  • destsize: dest バッファのサイズ。
  • src: 変換対象の文字列。
  • srclen: src の長さ。
  • locale: ICUプロバイダーが設定されたロケール構造体。
  • **出力**:
  • 戻り値: 生成されたソートキーのバイト数（NULL終端文字を含まない）。

前提条件と不変条件

  • locale->provider が COLLPROVIDER_ICU であること（Assert で検証）。
  • dest が指定されている場合、結果はNULL終端されること。
  • ICUの変換器（icu_converter）が初期化されていること。

アルゴリズムまたは振る舞い

1. init_icu_converter() を呼び出し、エンコーディング変換環境を準備します。 2. uchar_length を用いて、データベースエンコーディングからUTF-16（UChar）への変換に必要な長さを算出します。 3. TEXTBUFLEN を超える場合は palloc_array でメモリを確保し、それ以外はスタック上の sbuf を使用します。 4. uchar_convert で文字列をUTF-16に変換します。 5. ucol_getSortKey を呼び出し、ICUの照合順序に基づいたソートキーを生成します。 6. ICUが返すソートキー長にはNULL終端文字が含まれるため、戻り値から1を減算して調整します。 7. 確保したメモリがあれば解放し、結果のバイト数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pg_locale_icu.c 内のメソッドテーブル（collate_methods_icu および collate_methods_icu_utf8）から参照されています。
  • **呼び出し先**:
  • init_icu_converter
  • uchar_length
  • uchar_convert
  • ucol_getSortKey
  • palloc_array / pfree

関連データ構造

  • pg_locale_t: ロケール情報およびICUの UCollator を保持する構造体。
  • UChar: ICUで使用されるUTF-16文字型。

失敗モードと境界条件

  • ulen >= lengthof(sbuf) の場合、動的メモリ確保が行われます。
  • ucol_getSortKey が失敗した場合、PostgreSQLのトランザクションメカニズムによりエラーが報告されます。
  • dest がNULLの場合、ucol_getSortKey の仕様に従い、必要なバッファサイズが計算されます。

保守担当者向けメモ

  • 本関数は ucol_getSortKey の戻り値から1を引いて返しますが、これはPostgreSQLの慣習に合わせた調整です。
  • icu_converter はセッション単位で共有される静的オブジェクトである点に注意してください。
