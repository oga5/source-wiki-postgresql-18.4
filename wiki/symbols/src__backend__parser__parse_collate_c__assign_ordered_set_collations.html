assign_ordered_set_collations

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_collate.c:918

目的

順序付き集合集約（Ordered-set aggregate）における照合順序の解決を行います。集約関数の直接引数（aggdirectargs）および集約引数（args）に対して、適切な照合順序を割り当てます。

シグネチャまたは形状

static void assign_ordered_set_collations(Aggref *aggref, assign_collations_context *loccontext)

入力と出力

  • **入力**:
  • aggref: 処理対象の Aggref ノード。
  • loccontext: 照合順序の解決状態を保持する assign_collations_context。
  • **出力**: なし（副作用として aggref 内のノードに照合順序を適用）。

前提条件と不変条件

  • aggref は順序付き集合集約を表すノードである必要があります。
  • loccontext は有効な解析コンテキストを指している必要があります。

アルゴリズムまたは振る舞い

1. **マージ条件の判定**: 集約引数が単一であり、かつ可変長引数ではない場合、照合順序を親コンテキストへマージするフラグ merge_sort_collations を有効にします。 2. **直接引数の処理**: aggref->aggdirectargs に対して assign_collations_walker を再帰的に呼び出します。 3. **集約引数の処理**: aggref->args 内の各 TargetEntry を走査します。

  • merge_sort_collations が真の場合: assign_collations_walker を使用して再帰的に処理します。
  • それ以外の場合: assign_expr_collations を呼び出し、個別に照合順序を割り当てます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_collate.c 内の集約関数処理ロジック（assign_collations_walker 等から間接的に呼び出される）。
  • **呼び出し先**:
  • assign_collations_walker
  • assign_expr_collations
  • list_length
  • get_func_variadictype
  • lfirst_node

関連データ構造

  • Aggref: 集約関数呼び出しを表すノード。
  • assign_collations_context: 照合順序の伝播状態を保持する構造体。
  • TargetEntry: 集約引数を保持するノード。

失敗モードと境界条件

  • 引数リストが空の場合や、aggfnoid が無効な場合、get_func_variadictype の挙動に依存します。
  • 照合順序の競合が発生した場合、呼び出し先の assign_collations_walker または assign_expr_collations がエラーを投げる可能性があります。

保守担当者向けメモ

  • 本関数は順序付き集合集約特有の複雑な照合順序解決を担います。merge_sort_collations の条件分岐は、集約の引数構成に基づいているため、集約関数の定義変更時には注意が必要です。
