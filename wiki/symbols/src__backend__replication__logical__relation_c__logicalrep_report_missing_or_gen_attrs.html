logicalrep_report_missing_or_gen_attrs

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/relation.c:254

目的

論理レプリケーションのターゲットテーブルにおいて、必須列の欠落や生成列との不整合を検出し、エラーを報告する。

シグネチャまたは形状

static void logicalrep_report_missing_or_gen_attrs(LogicalRepRelation *remoterel, Bitmapset *missingatts, Bitmapset *generatedatts)

入力と出力

  • remoterel: 対象となるリモートリレーション情報（名前空間およびテーブル名を含む）。
  • missingatts: 欠落している列を示すビットマップセット。
  • generatedatts: 不整合な生成列を示すビットマップセット。
  • 出力: なし。条件に合致する場合、ereport(ERROR, ...) により処理を中断する。

前提条件と不変条件

  • missingatts または generatedatts が空でない場合、本関数は必ずエラーを発生させる。
  • エラーメッセージには logicalrep_get_attrs_str を用いて、該当する列名が文字列として付与される。

アルゴリズムまたは振る舞い

1. missingatts が空でない場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE を伴うエラーを発生させ、欠落している列名を報告する。 2. generatedatts が空でない場合、同様のエラーコードで、互換性のない生成列を報告する。 3. 両方のビットマップセットが空であれば、何も行わず正常に終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/relation.c 内の関数（具体的な関数名はソース抜粋からは特定不可）。
  • 呼び出し先: logicalrep_get_attrs_str（列名の文字列化に使用）。

関連データ構造

  • LogicalRepRelation: リモートリレーションのメタデータ（名前空間、テーブル名など）を保持。
  • Bitmapset: 欠落または不整合な列のインデックスを管理。

失敗モードと境界条件

  • missingatts または generatedatts にメンバーが含まれる場合、常に ERROR レベルで終了する。
  • 複数の列が該当する場合、errmsg_plural を用いて単数形または複数形のメッセージが適切に選択される。

保守担当者向けメモ

  • 本関数は、レプリケーションの適用前にテーブル構造の整合性を検証するガード節として機能する。
  • 呼び出し元の詳細なコンテキストについては、ソース抜粋からは限定的な情報しか得られていないため、必要に応じて relation.c の呼び出し箇所を参照すること。
