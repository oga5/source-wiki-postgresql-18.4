flatten_grouping_sets

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_clause.c:2257

目的

GROUP BY 句に含まれる入れ子状の GroupingSet ノードやリスト構造を再帰的に平坦化し、正規化された形式に変換します。

シグネチャまたは形状

Node *flatten_grouping_sets(Node *expr, bool toplevel, bool *hasGroupingSets)

入力と出力

  • **入力**:
  • expr: 処理対象の Node（GroupingSet, RowExpr, List 等）。
  • toplevel: 現在のノードが最上位レベルかどうかを示すフラグ。
  • hasGroupingSets: GroupingSet が検出された場合に true を設定するためのポインタ（最上位呼び出し時のみ使用）。
  • **出力**:
  • 平坦化された Node（GroupingSet または List）。

前提条件と不変条件

  • check_stack_depth() により、再帰呼び出し時のスタック深さがチェックされます。
  • toplevel が true の場合、空の GroupingSet (GROUPING_SET_EMPTY) は NIL として返されます。

アルゴリズムまたは振る舞い

1. **RowExpr**: 暗黙的なキャストを持つ場合、その引数に対して再帰的に処理を行います。 2. **GroupingSet**:

  • hasGroupingSets が指定されていれば true をセットします。
  • content リストを走査し、各要素を再帰的に平坦化します。
  • GROUPING_SET_SETS 型の入れ子構造はリストとして結合し、それ以外は lappend で追加します。
  • 最上位レベルまたは GROUPING_SET_SETS 以外の場合は makeGroupingSet でノードを再構築し、それ以外は平坦化したリストを返します。

    • 3. **List**: リスト内の各要素を再帰的に処理し、結果を結合して返します。 4. **その他**: 該当しないノード型はそのまま返されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_clause.c 内の flatten_grouping_sets 自身（再帰呼び出し）。
  • 同ファイル内の grouplist 処理箇所（src/backend/parser/parse_clause.c:2648）。
  • **呼び出し先**:
  • check_stack_depth()
  • makeGroupingSet()
  • list_concat(), lappend()

関連データ構造

  • GroupingSet: グループ化セットを表すノード。
  • RowExpr: 行式。
  • List: ノードのリスト。

失敗モードと境界条件

  • **再帰の深さ**: 非常に深い入れ子構造に対しては check_stack_depth() によるスタックオーバーフロー保護が働きます。
  • **空の入力**: NIL が入力された場合は NIL を返します。
  • **空のグループ化セット**: 最上位レベルで GROUPING_SET_EMPTY に遭遇した場合、NIL を返して呼び出し元での処理に委ねます。

保守担当者向けメモ

  • 本関数は GROUP BY 句の複雑な構文解析結果をプランナが扱いやすいフラットな構造に変換する重要な役割を担っています。
  • toplevel フラグの管理が再帰構造におけるノードの保持・展開の判断基準となっているため、変更時は注意が必要です。
