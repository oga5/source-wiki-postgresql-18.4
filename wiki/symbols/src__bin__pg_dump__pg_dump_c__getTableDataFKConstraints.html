getTableDataFKConstraints

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_dump.c:3186

目的

データベースのダンプ処理において、外部キー制約（FK constraint）に基づき、参照先テーブルのデータダンプが参照元テーブルのデータダンプよりも先に完了するように依存関係を構築する。

シグネチャまたは形状

static void getTableDataFKConstraints(void)

入力と出力

  • 入力: なし（内部で getDumpableObjects を呼び出し、全ダンプ対象オブジェクトを取得する）。
  • 出力: なし（副作用として、addObjectDependency を介して DumpableObject 間の依存関係リストを更新する）。

前提条件と不変条件

  • getDumpableObjects が呼び出し可能であること。
  • 処理対象の制約（DO_FK_CONSTRAINT）に関連するテーブル情報（contable および confrelid で特定されるテーブル）が、ダンプ対象として有効（dataObj が存在）であること。

アルゴリズムまたは振る舞い

1. getDumpableObjects を呼び出し、全ダンプ対象オブジェクトのリストを取得する。 2. リストを走査し、objType が DO_FK_CONSTRAINT であるオブジェクトを抽出する。 3. 各制約について、以下の条件を確認する：

  • 参照元テーブル（contable）のデータダンプオブジェクトが存在すること。
  • 参照先テーブル（confrelid）が特定でき、かつそのデータダンプオブジェクトが存在すること。

    • 4. 条件を満たす場合、addObjectDependency を使用し、参照元テーブルのデータダンプオブジェクトが、参照先テーブルのデータダンプオブジェクトに依存するように登録する。 5. 最後に、取得したオブジェクトリストのメモリを free する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_dump.c 内のメイン処理フロー（main 関数周辺のダンプ構築フェーズ）。
  • 呼び出し先: getDumpableObjects, findTableByOid, addObjectDependency, free。

関連データ構造

  • DumpableObject: ダンプ対象オブジェクトの基底構造体。
  • ConstraintInfo: 制約情報を保持する構造体。
  • TableInfo: テーブル情報を保持する構造体。

失敗モードと境界条件

  • findTableByOid が NULL を返す場合（参照先テーブルがダンプ対象外である場合など）、その制約に関する依存関係の追加はスキップされる。
  • contable または dataObj が NULL の場合、処理はスキップされる。

保守担当者向けメモ

  • 本関数は、データダンプの順序を制御するための依存関係解決ロジックの一部である。
  • 依存関係の追加は addObjectDependency を介して行われるため、ダンプ順序のソート処理（sortDumpableObjects）に影響を与える。
