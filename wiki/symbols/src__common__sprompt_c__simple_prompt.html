simple_prompt

定義

  • 種別: function
  • 定義箇所: src/common/sprompt.c:37

目的

対話的なユーザー入力（パスワードやユーザー名など）を安全かつポータブルに取得するためのインターフェースを提供します。

シグネチャまたは形状

char *simple_prompt(const char *prompt, bool echo);

入力と出力

  • **入力**:
  • prompt: 表示するプロンプト文字列。
  • echo: true の場合は入力内容を端末に表示し、false の場合はエコーバックを抑止します。
  • **出力**:
  • ユーザーが入力した文字列を格納した char * 型のポインタを返します。
  • メモリは malloc されているため、呼び出し側で解放する必要があります。

前提条件と不変条件

  • 呼び出し側は、返された文字列に対して free() を実行する責任を負います。
  • 端末がリダイレクトされている場合でも動作しますが、その場合はエコー抑止機能は無効となります。

アルゴリズムまたは振る舞い

1. 内部的に simple_prompt_extended(prompt, echo, NULL) を呼び出します。 2. 端末デバイス（POSIXでは /dev/tty、Windowsでは CONIN$/CONOUT$）を優先的に使用して対話環境を確立します。 3. echo が false の場合、プラットフォーム固有の API を用いて端末のエコー設定を一時的に無効化します。 4. プロンプトを表示し、pg_get_line を介して入力を取得します。 5. 入力文字列から改行コードを除去し、端末設定を復元した上で結果を返します。

関連データ構造

  • PromptInterruptContext: 割り込み発生時の制御に使用される構造体（本関数は NULL を渡して使用）。

失敗モードと境界条件

  • メモリ不足時には malloc 失敗により終了する可能性があります。
  • 端末オープンに失敗した場合は stdin/stderr にフォールバックしますが、対話的な制御（エコー抑止等）が制限される場合があります。

保守担当者向けメモ

  • 本関数は simple_prompt_extended の簡易版です。割り込み処理（longjmp 等）が必要な場合は simple_prompt_extended を直接使用してください。
  • Windows 環境では CONIN$/CONOUT$ を w+ モードで開く特殊な実装が内部で行われています。
