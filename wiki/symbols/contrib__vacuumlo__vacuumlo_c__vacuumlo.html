vacuumlo

定義

  • 種別: function
  • 定義箇所: contrib/vacuumlo/vacuumlo.c:61

目的

指定されたデータベースに接続し、どのテーブルからも参照されていない「孤立したラージオブジェクト（LO）」を特定して削除する。

シグネチャまたは形状

static int vacuumlo(const char *database, const struct _param *param);

入力と出力

  • **入力**:
  • database: 接続対象のデータベース名。
  • param: 接続設定や実行オプション（ホスト、ポート、ユーザー、トランザクション制限等）を保持する _param 構造体。
  • **出力**:
  • 成功時は 0、失敗時は -1 を返す。

前提条件と不変条件

  • libpq を介したデータベース接続が確立可能であること。
  • 実行ユーザーが対象データベースに対して適切な権限を持っていること。
  • param->pg_prompt が TRI_YES の場合、パスワード入力が求められる。
  • システムカタログ（pg_ で始まるスキーマ）は解析対象から除外される。

アルゴリズムまたは振る舞い

1. **接続確立**: PQconnectdbParams を使用してデータベースに接続。パスワードが必要な場合は対話的に取得。 2. **環境設定**: search_path を安全な値に設定。 3. **一時テーブル作成**: vacuum_l を作成し、全 LO の OID を格納（サーバーバージョンに応じて pg_largeobject_metadata または pg_largeobject を参照）。 4. **解析準備**: ANALYZE vacuum_l を実行し、クエリプランを最適化。 5. **参照チェック**: pg_class 等をクエリし、oid または lo 型の列を持つ全テーブルを特定。 6. **削除処理**: 特定されたテーブルを走査し、参照されている LO を vacuum_l から除外。残った LO を lo_unlink で削除する（トランザクション制限数ごとにコミットを実行）。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（contrib/vacuumlo/vacuumlo.c:540）。
  • **呼び出し先**: libpq 関数群（PQconnectdbParams, PQexec, PQstatus, PQfinish 等）、simple_prompt。

関連データ構造

  • struct _param: 実行オプションを保持。
  • PGconn, PGresult: libpq の接続および結果保持用構造体。

失敗モードと境界条件

  • データベース接続失敗時に -1 を返す。
  • search_path の設定失敗や、一時テーブル作成失敗時に -1 を返す。
  • PQescapeIdentifier による識別子のエスケープに失敗した場合、エラーを出力して -1 を返す。

保守担当者向けメモ

  • transaction_limit はロックテーブルの枯渇を防ぐための重要なパラメータである。
  • pg_ スキーマを除外することで、システムカタログや一時テーブル自身の誤削除を防いでいる。
  • -n オプション（ドライラン）時は、実際の削除処理をスキップするロジックが後続のコードに含まれる。
