init_spin_delay

定義

  • 種別: static inline function
  • 定義箇所: src/include/storage/s_lock.h:739

目的

スピンロックの待機状態を追跡するための SpinDelayStatus 構造体を初期化します。

シグネチャまたは形状

static inline void init_spin_delay(SpinDelayStatus *status, const char *file, int line, const char *func);

入力と出力

  • **入力**:
  • status: 初期化対象となる SpinDelayStatus 構造体へのポインタ。
  • file: 呼び出し元のソースファイル名（文字列）。
  • line: 呼び出し元の行番号。
  • func: 呼び出し元の関数名（文字列）。
  • **出力**: なし（status が指す構造体を直接更新）。

前提条件と不変条件

  • status は有効なメモリ領域を指している必要があります。
  • 初期化後、status 内の spins、delays、cur_delay はすべて 0 に設定されます。

アルゴリズムまたは振る舞い

1. status 構造体のカウンタメンバ（spins, delays, cur_delay）を 0 にリセットします。 2. 渡されたデバッグ情報（file, line, func）を構造体の対応するメンバに格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/s_lock.c:102
  • src/include/storage/s_lock.h:751（init_local_spin_delay マクロ経由）
  • **呼び出し先**:
  • なし（本関数は純粋なデータ初期化関数です）。

関連データ構造

  • SpinDelayStatus: スピン待機の試行回数や遅延時間を保持する構造体。

失敗モードと境界条件

  • status が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こします。

保守担当者向けメモ

  • 本関数は init_local_spin_delay マクロを通じて利用されることが一般的です。
  • 構造体のメンバ構成に変更があった場合は、本関数も併せて更新してください。
