format_node_dump

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/print.c:96

目的

ノードのダンプ文字列を、端末表示に適した幅（78文字）で折り返し、改行を挿入して整形した文字列を生成する。

シグネチャまたは形状

char *format_node_dump(const char *dump)

入力と出力

  • **入力**: dump (整形対象のノードダンプ文字列)
  • **出力**: 整形済みの新しい文字列（StringInfo バッファから抽出された char *）。呼び出し元で解放が必要。

前提条件と不変条件

  • 入力 dump はヌル終端文字列であること。
  • 内部で StringInfo を使用するため、メモリ不足時には PostgreSQL のエラーハンドリング機構が動作する。

アルゴリズムまたは振る舞い

1. StringInfoData を初期化する。 2. 入力文字列を最大 78 文字ずつ読み込む。 3. 行末が文字列の終端でない場合、単語の区切り（スペース）を探す。

  • スペースが見つかれば、その位置で改行する。
  • スペースが見つからなければ、そのまま 78 文字で強制的に改行する。

    • 4. 整形した各行を appendStringInfo でバッファに追加する。 5. 最後にバッファ内の文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/print.c 内の他の関数（print や pprint 等の内部処理から参照されている）。
  • **呼び出し先**: initStringInfo, appendStringInfo（lib/stringinfo.h）。

関連データ構造

  • StringInfoData: 整形済み文字列を構築するための動的バッファ。

失敗モードと境界条件

  • 入力文字列が空の場合、空の文字列が返される。

保守担当者向けメモ

  • 本関数は LINELEN マクロ（78）をハードコードしている。
  • 戻り値の文字列は palloc されたメモリ領域であるため、呼び出し元で適切に管理（pfree）すること。
