InitConflictIndexes

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/conflict.c:137

目的

論理レプリケーションにおいて、競合チェックの対象となる一意制約インデックスを特定し、ResultRelInfo 構造体に登録します。

シグネチャまたは形状

void InitConflictIndexes(ResultRelInfo *relInfo)

入力と出力

  • **入力**: relInfo (ResultRelInfo *) - 処理対象のテーブル情報を含む構造体。
  • **出力**: なし（副作用として relInfo->ri_onConflictArbiterIndexes を更新）。

前提条件と不変条件

  • relInfo は有効なポインタである必要があります。
  • 競合チェック対象として登録されるインデックスは、一意（Unique）かつ即時制約（Non-deferrable）である必要があります。

アルゴリズムまたは振る舞い

1. relInfo に含まれるすべてのインデックスを走査します。 2. 各インデックスについて以下の条件を確認します。

  • インデックスが存在すること。
  • ii_Unique フラグが立っていること（一意インデックスであること）。
  • indimmediate が真であること（遅延制約ではないこと）。

    • 3. 上記条件を満たすインデックスの OID をリスト化し、relInfo->ri_onConflictArbiterIndexes に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/logical/worker.c 内の各所（InitConflictIndexes(relinfo) または InitConflictIndexes(partrelinfo) として呼び出し）。
  • **呼び出し先**:
  • RelationGetRelid（インデックスの OID 取得に使用）。
  • lappend_oid（リストへの追加に使用）。

関連データ構造

  • ResultRelInfo: 実行時のリレーション情報。ri_onConflictArbiterIndexes フィールドに結果が保持されます。

失敗モードと境界条件

  • インデックスが一つも条件を満たさない場合、ri_onConflictArbiterIndexes は NIL となります。
  • relInfo->ri_IndexRelationDescs[i] が NULL の場合はスキップされます。

保守担当者向けメモ

  • 本関数は競合解決の「解決」ロジックではなく、あくまで「どのインデックスを使って競合を検知するか」の準備を行うものです。
  • 遅延制約（deferrable）インデックスはサポート対象外であるため、本関数では明示的に除外されています。
