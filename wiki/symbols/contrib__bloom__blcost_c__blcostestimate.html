blcostestimate

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blcost.c:21

目的

PostgreSQL のクエリプランナに対し、Bloom インデックスを使用したスキャンのコスト見積もり（起動コスト、総コスト、選択率、相関、ページ数）を提供します。

シグネチャまたは形状

void blcostestimate(PlannerInfo *root, IndexPath *path, double loop_count,
                    Cost *indexStartupCost, Cost *indexTotalCost,
                    Selectivity *indexSelectivity, double *indexCorrelation,
                    double *indexPages)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • path: インデックススキャンのパス情報。
  • loop_count: ループ回数。
  • **出力**:
  • indexStartupCost, indexTotalCost: コスト見積もり値。
  • indexSelectivity: インデックスの選択率。
  • indexCorrelation: インデックスの相関。
  • indexPages: インデックスのページ数。

前提条件と不変条件

  • Bloom インデックスは、スキャン時にすべてのインデックスタプルを走査する必要があるという前提で計算されます。

アルゴリズムまたは振る舞い

1. path->indexinfo からインデックス内のタプル数（tuples）を取得します。 2. GenericCosts 構造体の numIndexTuples に取得したタプル数を設定します。 3. genericcostestimate を呼び出し、PostgreSQL 標準の汎用的なコスト計算ロジックを実行します。 4. 計算されたコスト結果を、引数として渡された各ポインタ変数にコピーして返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: IndexAmRoutine 構造体の amcostestimate フィールドを通じてプランナから呼び出されます（contrib/bloom/blutils.c:139 にて登録）。
  • **呼び出し先**: genericcostestimate（utils/selfuncs.h 定義）。

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • IndexPath: インデックススキャンのパス情報。
  • IndexOptInfo: インデックスの統計情報（タプル数など）。
  • GenericCosts: genericcostestimate に渡すコスト計算用の中間構造体。

失敗モードと境界条件

  • 提供されたソースコード内には明示的なエラーハンドリングや境界条件のチェックは含まれていません。

保守担当者向けメモ

  • 本関数は独自の複雑な計算ロジックを持たず、標準の genericcostestimate に依存しています。
  • Bloom インデックス特有の最適化が必要になった場合は、GenericCosts の設定値を調整するか、計算ロジックを拡張してください。
