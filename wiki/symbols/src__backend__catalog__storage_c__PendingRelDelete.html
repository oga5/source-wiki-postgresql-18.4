PendingRelDelete

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/storage.c:62

目的

トランザクションのコミットまたはアボート時に、物理的なリレーション（テーブルやインデックス）の削除を遅延実行するための予約情報を保持する。

シグネチャまたは形状

typedef struct PendingRelDelete
{
    RelFileLocator rlocator;    /* 削除対象のリレーション */
    ProcNumber    procNumber;   /* 一時リレーションの場合はプロセス番号、それ以外は INVALID_PROC_NUMBER */
    bool          atCommit;     /* true=コミット時に削除, false=アボート時に削除 */
    int           nestLevel;    /* リクエスト時のトランザクションネストレベル */
    struct PendingRelDelete *next; /* 連結リスト用のポインタ */
} PendingRelDelete;

入力と出力

  • 本構造体は MemoryContextAlloc(TopMemoryContext, ...) を介してメモリ確保される。
  • TopMemoryContext に配置されるため、トランザクション終了後も生存可能である。

前提条件と不変条件

  • pendingDeletes リストの先頭要素として管理される。
  • トランザクションのネストレベル（nestLevel）に基づき、サブトランザクションの終了時に適切な削除処理が行われる。

アルゴリズムまたは振る舞い

1. **登録**: RelationCreateStorage や RelationDropStorage 等の処理において、物理削除が必要な場合に本構造体が生成され、pendingDeletes リストに追加される。 2. **実行**: トランザクション終了時（コミットまたはアボート）に smgrDoPendingDeletes が呼び出され、リストを走査して atCommit フラグと現在のトランザクション状態を照合し、物理的な unlink を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/storage.c 内の RelationCreateStorage、RelationDropStorage、および関連するトランザクション管理関数。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出さない。

関連データ構造

  • RelFileLocator: 削除対象リレーションの識別子。
  • pendingDeletes: src/backend/catalog/storage.c 内で定義される、本構造体を要素とする連結リストのヘッドポインタ。

失敗モードと境界条件

  • メモリ不足時: MemoryContextAlloc が失敗した場合、システム全体に影響が及ぶ可能性がある。
  • 一時リレーションの扱い: procNumber が INVALID_PROC_NUMBER でない場合、一時リレーションとして特別な処理が行われる。

保守担当者向けメモ

  • pendingDeletes リストは TopMemoryContext に保持されているため、メモリリークを防ぐためにリストの適切な解放（smgrDoPendingDeletes による処理）が必須である。
  • atCommit フラグの判定ロジックは、トランザクションのコミット/アボートの整合性に直結するため、変更時には注意が必要である。
