GetStrictOldestNonRemovableTransactionId

定義

  • 種別: function
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:625

目的

破損チェックにおいて「誤った破損報告（偽陽性）」を避けるため、通常の GetOldestNonRemovableTransactionId よりも厳格（保守的）なトランザクションID（XID）の境界（horizon）を計算する。

シグネチャまたは形状

static TransactionId GetStrictOldestNonRemovableTransactionId(Relation rel)

入力と出力

  • **入力**: Relation rel （対象のリレーション）
  • **出力**: TransactionId （計算された最も古い削除不可能なトランザクションID）

前提条件と不変条件

  • 呼び出し元はリレーションのロックを保持している必要がある。
  • 一時リレーションの場合、TransamVariables->latestCompletedXid および MyProc->xid が後退しないことが前提となる。

アルゴリズムまたは振る舞い

リレーションの種類に応じて以下の通り計算を行う： 1. **リカバリ中**: XidGenLock を取得し、nextXid を返す。 2. **共有リレーション (rel == NULL または relisshared)**: GetRunningTransactionData() を使用し、実行中のすべてのXIDを考慮した oldestRunningXid を返す。 3. **通常の永続リレーション**: 現在のデータベース内で実行中のXIDを考慮した oldestDatabaseRunningXid を返す。 4. **一時リレーション**: GetOldestNonRemovableTransactionId(rel) を呼び出し、標準的な計算結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pg_visibility/pg_visibility.c:734
  • contrib/pg_visibility/pg_visibility.c:848
  • **呼び出し先**:
  • RecoveryInProgress()
  • LWLockAcquire / LWLockRelease
  • XidFromFullTransactionId
  • GetRunningTransactionData()
  • GetOldestNonRemovableTransactionId()

関連データ構造

  • Relation
  • RunningTransactions
  • TransamVariables

失敗モードと境界条件

  • rel が NULL の場合、共有リレーションとして扱われる。
  • リカバリ中とそれ以外でロック取得の挙動が異なるため、実行環境の状態に依存する。

保守担当者向けメモ

  • 本関数は pg_check_frozen や pg_check_visible 等の整合性チェック処理において、誤検知を防止するための重要なガードレールとして機能する。
  • 巨大なテーブルに対して実行する場合、ReadStream を介した全ページ走査に伴うI/O負荷に注意すること。
