PostmasterDeathSignalInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/pmsignal.c:406

目的

Postmaster（親プロセス）が異常終了した際に、子プロセスがそれを検知して即座に停止するためのシグナル設定を行う。

シグネチャまたは形状

void PostmasterDeathSignalInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（成功時）。失敗時は elog(ERROR) によりプロセスを終了させる。

前提条件と不変条件

  • USE_POSTMASTER_DEATH_SIGNAL マクロが定義されている環境でのみ有効。
  • OSが PR_SET_PDEATHSIG または PROC_PDEATHSIG_CTL のいずれかをサポートしている必要がある。

アルゴリズムまたは振る舞い

1. pqsignal を使用し、POSTMASTER_DEATH_SIGNAL に対して postmaster_death_handler をシグナルハンドラとして登録する。 2. OS固有の機能（Linuxの prctl または FreeBSD等の procctl）を呼び出し、親プロセスが終了した際に当該シグナルを自プロセスへ送信するようOSに要求する。 3. 設定完了後、念のため親プロセスが既に存在しない可能性を考慮し、postmaster_possibly_dead フラグを true に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/init/miscinit.c:158
  • 呼び出し先: pqsignal（シグナル登録）、prctl または procctl（OSへのシグナル要求）、elog（エラー報告）

関連データ構造

  • postmaster_possibly_dead: Postmasterの生存状態を示すグローバル変数。

失敗モードと境界条件

  • OSが親プロセスの終了通知機能をサポートしていない場合、コンパイルエラーとなる。
  • prctl または procctl の呼び出しが失敗した場合、elog(ERROR) を発行して停止する。

保守担当者向けメモ

  • 本関数はプロセス初期化の段階で呼び出されることを想定している。
  • postmaster_possibly_dead を true に設定する処理は、シグナル設定のタイミングと親プロセスの終了が競合する可能性を考慮した安全策である。
