dintdict_init

定義

  • 種別: function
  • 定義箇所: contrib/dict_int/dict_int.c:34

目的

PostgreSQL の全文検索辞書テンプレート intdict の初期化を行い、ユーザー指定のオプションに基づいて DictInt 構造体を構築する。

シグネチャまたは形状

Datum dintdict_init(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して List *dictoptions を受け取る。これは CREATE TEXT SEARCH DICTIONARY で指定されたオプションのリスト。
  • **出力**: DictInt * を Datum 型として返却する。

前提条件と不変条件

  • maxlen オプションが指定された場合、その値は 1 以上である必要がある。
  • DictInt 構造体は palloc0 によりゼロ初期化され、デフォルト値として maxlen=6, rejectlong=false, absval=false が設定される。

アルゴリズムまたは振る舞い

1. DictInt 構造体をメモリ上に確保し、デフォルト値を設定する。 2. dictoptions リストを反復処理し、各 DefElem を解析する。

  • maxlen: 整数値に変換し、d->maxlen に格納する。
  • rejectlong: 真偽値に変換し、d->rejectlong に格納する。
  • absval: 真偽値に変換し、d->absval に格納する。

    • 3. 未知のパラメータが指定された場合は ERROR を発生させる。 4. 構築した DictInt 構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の全文検索エンジン（CREATE TEXT SEARCH DICTIONARY 実行時）。
  • **呼び出し先**: palloc0, defGetString, defGetBoolean, ereport, strcmp, atoi 等の PostgreSQL 内部関数および標準 C ライブラリ関数。

関連データ構造

  • DictInt: 辞書の設定値を保持する構造体。
  • DefElem: 辞書オプションの定義を保持する構造体。

失敗モードと境界条件

  • maxlen が 1 未満の場合: ERRCODE_INVALID_PARAMETER_VALUE でエラー終了する。
  • 未知のオプション名が渡された場合: ERRCODE_INVALID_PARAMETER_VALUE でエラー終了する。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_INFO_V1(dintdict_init) マクロにより、PostgreSQL の関数呼び出し規約に適合するように登録されている。
  • DictInt 構造体のメモリ管理は PostgreSQL のメモリコンテキストに依存している。
