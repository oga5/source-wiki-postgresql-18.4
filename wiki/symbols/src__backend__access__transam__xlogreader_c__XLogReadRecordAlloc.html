XLogReadRecordAlloc

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogreader.c:438

目的

WALレコードをデコードするためのメモリ領域を確保します。循環バッファ（circular decode buffer）内での空き領域の割り当てを試み、不足している場合は必要に応じて動的メモリ確保を行います。

シグネチャまたは形状

DecodedXLogRecord *XLogReadRecordAlloc(XLogReaderState *state, size_t xl_tot_len, bool allow_oversized)

入力と出力

  • **入力**:
  • state: XLogReaderState 構造体へのポインタ。
  • xl_tot_len: 確保が必要なレコードの合計サイズ。
  • allow_oversized: 循環バッファに空きがない場合に、個別のメモリ確保を許可するかどうかを示すフラグ。
  • **出力**:
  • 確保された DecodedXLogRecord 構造体へのポインタ。確保に失敗した場合は NULL を返します。

前提条件と不変条件

  • state->decode_buffer が未初期化の場合、本関数内で DEFAULT_DECODE_BUFFER_SIZE を用いて初期化されます。
  • 確保された DecodedXLogRecord の oversized フィールドは、循環バッファから確保された場合は false、個別に確保された場合は true に設定されます。

アルゴリズムまたは振る舞い

1. DecodeXLogRecordRequiredSpace を呼び出し、必要なメモリサイズを算出します。 2. 循環バッファが未初期化であれば、palloc を用いて確保します。 3. 循環バッファの現在の head と tail の位置関係に基づき、以下のいずれかの領域に空きがあるか確認します。

  • tail が head より右側にある場合: tail からバッファ末尾までの空き、またはバッファ先頭から head までの空き。
  • tail が head より左側にある場合: tail から head までの空き。

    • 4. 空き領域が見つかれば、そのアドレスを DecodedXLogRecord として返します。 5. 空き領域がなく、かつ allow_oversized が true の場合、palloc で必要なサイズを直接確保して返します。 6. 上記いずれも該当しない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogreader.c 内のレコード読み込み処理（684行目、891行目付近）から呼び出されます。
  • **呼び出し先**:
  • DecodeXLogRecordRequiredSpace（サイズ計算）
  • palloc（メモリ確保）

関連データ構造

  • XLogReaderState: リーダーの状態と循環バッファの管理情報を保持。
  • DecodedXLogRecord: デコードされたレコードを格納する構造体。

失敗モードと境界条件

  • 循環バッファに十分な空きがなく、かつ allow_oversized が false の場合、関数は NULL を返します。
  • 循環バッファの初期化時に palloc が失敗した場合、PostgreSQLの標準的なメモリ管理ルールに従いエラーが発生します。

保守担当者向けメモ

  • 本関数は循環バッファのポインタ操作（head, tail）に依存しているため、バッファの境界条件を変更する際は、ポインタの計算ロジックに注意してください。
  • allow_oversized が true の場合に返されるメモリは、循環バッファの一部ではないため、解放時に特別な処理が必要になる可能性があります。
