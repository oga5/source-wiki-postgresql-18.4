rbt_rotate_left

定義

  • 種別: function
  • 定義箇所: src/backend/lib/rbtree.c:262

目的

赤黒木の平衡を維持するために、指定されたノード x を中心として左回転操作を行う。

シグネチャまたは形状

static void rbt_rotate_left(RBTree *rbt, RBTNode *x)

入力と出力

  • **入力**:
  • rbt: 操作対象の赤黒木を管理する RBTree 構造体へのポインタ。
  • x: 左回転の支点となる RBTNode へのポインタ。
  • **出力**: なし（副作用として木構造を更新）。

前提条件と不変条件

  • x の右子ノード（x->right）が存在すること（RBTNIL でないこと）。
  • RBTNIL は静的な番兵として扱われ、NULLチェックの代わりに使用される。

アルゴリズムまたは振る舞い

1. y を x の右子ノードとする。 2. y の左部分木を x の右側に付け替える。 3. x の親ノードから見て、x があった位置に y を配置する（x がルートの場合は rbt->root を更新）。 4. x を y の左子ノードとして再配置する。 5. 各ノードの parent ポインタを適切に更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/lib/rbtree.c 内の挿入・削除時の平衡修正処理（rbt_insert_fixup や rbt_delete_fixup 等の文脈で呼び出される）。
  • **呼び出し先**:
  • 直接的な関数呼び出しはないが、RBTNode 構造体のメンバ（right, left, parent）および RBTree 構造体の root メンバを操作する。

関連データ構造

  • RBTree: 木全体を管理する制御構造体。
  • RBTNode: 木の各ノード。
  • RBTNIL: 葉ノードを表す静的な番兵。

失敗モードと境界条件

  • x が RBTNIL である場合や、x->right が RBTNIL である場合の挙動については、ソースコード上の明示的なガードは記述されていない。
  • x がルートノードである場合、rbt->root が正しく更新される。

保守担当者向けメモ

  • 本関数は侵入型データ構造の一部であり、ノードのメモリ管理は呼び出し側に依存する。
  • 回転操作はポインタの付け替えのみを行い、ノードの色の変更は行わない。色の変更は呼び出し側の修正関数（fixup）で行う必要がある。
