tbm_attach_shared_iterate

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/tidbitmap.c:1464

目的

共有メモリ上に存在する TBMSharedIteratorState をアタッチし、ローカルプロセスで反復処理を行うための TBMSharedIterator 構造体を初期化する。

シグネチャまたは形状

TBMSharedIterator *tbm_attach_shared_iterate(dsa_area *dsa, dsa_pointer dp)

入力と出力

  • **入力**:
  • dsa: 共有メモリ領域を管理する dsa_area へのポインタ。
  • dp: 共有メモリ上の TBMSharedIteratorState を指す dsa_pointer。
  • **出力**: 初期化された TBMSharedIterator 構造体へのポインタ。

前提条件と不変条件

  • dp は有効な TBMSharedIteratorState を指している必要がある。
  • dsa は istate が割り当てられた領域と一致している必要がある。

アルゴリズムまたは振る舞い

1. palloc0 を使用して TBMSharedIterator 構造体をメモリ上に確保する。 2. dsa_get_address を用いて、共有メモリ上の TBMSharedIteratorState (istate) のアドレスを取得し、イテレータに格納する。 3. istate からページテーブルのベースアドレスを取得し、イテレータの ptbase に設定する。 4. istate->npages が存在する場合、istate->spages を解決して ptpages に設定する。 5. istate->nchunks が存在する場合、istate->schunks を解決して ptchunks に設定する。 6. 初期化済みのイテレータを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/tidbitmap.c:1582 (直接的な呼び出しを確認)。
  • **呼び出し先**: dsa_get_address (DSA API)。

関連データ構造

  • TBMSharedIterator: 反復処理の状態を保持するローカル構造体。
  • TBMSharedIteratorState: 共有メモリ上に配置された反復処理の制御情報。
  • dsa_area / dsa_pointer: 動的共有メモリ管理用構造体。

失敗モードと境界条件

  • dsa_get_address が無効なポインタを解決しようとした場合の挙動は、DSA ライブラリの仕様に依存する。
  • istate->npages や istate->nchunks が 0 の場合、対応するポインタフィールドは初期化されない。

保守担当者向けメモ

  • 本関数は並列クエリ実行時にワーカープロセスが共有ビットマップをスキャンする際に使用される。
  • TBMSharedIterator のメモリ確保には palloc0 を使用しているため、呼び出し元のメモリコンテキストが適切であることを確認すること。
