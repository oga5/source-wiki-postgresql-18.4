AlterSubscriptionOwner

定義

  • 種別: function
  • 定義箇所: src/backend/commands/subscriptioncmds.c:2029

目的

指定されたサブスクリプションの所有者を変更する。

シグネチャまたは形状

ObjectAddress AlterSubscriptionOwner(const char *name, Oid newOwnerId)

入力と出力

  • **入力**:
  • name: 変更対象のサブスクリプション名。
  • newOwnerId: 新しい所有者のロール OID。
  • **出力**:
  • 変更されたサブスクリプションの ObjectAddress を返す。

前提条件と不変条件

  • pg_subscription システムカタログに対する RowExclusiveLock を保持する必要がある。
  • 指定された名前のサブスクリプションが現在のデータベース内に存在しなければならない。

アルゴリズムまたは振る舞い

1. SubscriptionRelationId に対して table_open を実行し、ロックを取得する。 2. SUBSCRIPTIONNAME システムキャッシュを使用して、指定された名前のサブスクリプションタプルを検索する。 3. タプルが見つからない場合は ERROR を発生させる。 4. AlterSubscriptionOwner_internal を呼び出し、所有権変更の内部処理（カタログ更新等）を実行する。 5. ObjectAddressSet を使用して、対象サブスクリプションの ObjectAddress を構築する。 6. 検索したタプルを解放し、テーブルをクローズして ObjectAddress を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/alter.c:871 (SQL コマンド解析経由の呼び出し)
  • **呼び出し先**:
  • table_open / table_close
  • SearchSysCacheCopy2 / heap_freetuple
  • AlterSubscriptionOwner_internal
  • ObjectAddressSet

関連データ構造

  • ObjectAddress: 変更対象オブジェクトの識別子。
  • HeapTuple: pg_subscription カタログの行データ。
  • Form_pg_subscription: サブスクリプションの属性構造体。

失敗モードと境界条件

  • 指定された名前のサブスクリプションが存在しない場合、ERRCODE_UNDEFINED_OBJECT でエラーとなる。

保守担当者向けメモ

  • 本関数はラッパーであり、実際のカタログ更新ロジックは AlterSubscriptionOwner_internal に委譲されている。
  • トランザクション内での実行を前提としており、RowExclusiveLock を使用して整合性を確保している。
