GetFdwRoutine

定義

  • 種別: function
  • 定義箇所: src/backend/foreign/foreign.c:325

目的

指定された外部データラッパー（FDW）のハンドラ関数を呼び出し、FDW の実行に必要なコールバック関数群を格納した FdwRoutine 構造体を取得する。

シグネチャまたは形状

FdwRoutine *GetFdwRoutine(Oid fdwhandler)

入力と出力

  • **入力**: fdwhandler (Oid) - FDW ハンドラ関数の OID。
  • **出力**: FdwRoutine * - 外部データソースへのアクセスに必要なメソッドを保持する構造体へのポインタ。

前提条件と不変条件

  • restrict_nonsystem_relation_kind に RESTRICT_RELKIND_FOREIGN_TABLE が設定されている場合、非システム外部テーブルへのアクセスは制限され、エラーが発生する。
  • fdwhandler が指す関数は、有効な FdwRoutine 構造体へのポインタを返す必要がある。

アルゴリズムまたは振る舞い

1. 外部テーブルへのアクセス制限を確認し、制限されている場合はエラーを報告する。 2. OidFunctionCall0 を使用して、指定された fdwhandler 関数を実行する。 3. 戻り値が NULL であるか、IsA(routine, FdwRoutine) チェックに失敗した場合はエラーを報告する。 4. 取得した FdwRoutine 構造体へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/foreigncmds.c:1525
  • src/backend/foreign/foreign.c:411
  • **呼び出し先**:
  • OidFunctionCall0（FDW ハンドラ関数の実行）
  • ereport / elog（エラー報告）

関連データ構造

  • FdwRoutine: 外部データソース操作のためのインターフェース定義（src/include/foreign/fdwapi.h で定義）。

失敗モードと境界条件

  • **アクセス制限**: RESTRICT_RELKIND_FOREIGN_TABLE が有効な場合、呼び出しは ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーとなる。
  • **不正なハンドラ**: ハンドラ関数が NULL を返すか、FdwRoutine 型ではないデータを返した場合は elog(ERROR) により処理が中断される。

保守担当者向けメモ

  • 本関数は FDW のハンドラを動的に呼び出すため、ハンドラ側の実装が FdwRoutine 構造体のメモリレイアウトを正しく維持していることに依存している。
  • GetFdwRoutineForRelation と異なり、本関数はキャッシュを行わないため、呼び出し側で必要に応じて管理を行うこと。
