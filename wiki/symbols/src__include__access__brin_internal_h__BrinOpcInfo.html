BrinOpcInfo

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_internal.h:25

目的

BRIN（Block Range Index）インデックスにおいて、特定の演算子クラス（opclass）に関連付けられたメタデータおよび型情報を保持するために使用されます。

シグネチャまたは形状

typedef struct BrinOpcInfo
{
    uint16      oi_nstored;
    bool        oi_regular_nulls;
    void       *oi_opaque;
    TypeCacheEntry *oi_typcache[FLEXIBLE_ARRAY_MEMBER];
} BrinOpcInfo;

入力と出力

  • **入力**: インデックス構築時やスキャン時に、演算子クラスのプロシージャがこの構造体を参照して型情報や設定を取得します。
  • **出力**: 構造体内の各フィールド（oi_nstored 等）が、BRIN インデックスの操作ロジックに対して演算子クラス固有の振る舞いを提供します。

前提条件と不変条件

  • oi_typcache は可変長配列として定義されており、インデックス列の数に応じてメモリが割り当てられます。
  • 演算子クラスのプロシージャが呼び出される際、この構造体は適切に初期化されている必要があります。

アルゴリズムまたは振る舞い

1. brin.c 等の処理において、インデックスの各列に対して BrinOpcInfo が割り当てられます。 2. 演算子クラスのプロシージャ（BRIN_PROCNUM_ADDVALUE 等）が実行される際、この構造体から取得した TypeCacheEntry を用いて、ディスク上のデータとメモリ上の Datum 表現の変換が行われます。 3. oi_opaque フィールドは、各演算子クラスが独自に必要とする状態情報を保持するためのポインタとして機能します。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/access/brin/brin.c: インデックスの初期化および操作時に BrinOpcInfo を割り当て、参照しています。
  • src/backend/access/brin/brin_bloom.c, brin_inclusion.c, brin_minmax.c, brin_minmax_multi.c: 各演算子クラスの実装において、結果の保持や情報取得に使用されています。
  • contrib/pageinspect/brinfuncs.c: インデックス情報の検査に使用されています。
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、関数呼び出しの直接的な起点ではありません。

関連データ構造

  • BrinDesc: インデックスリレーションのメタデータを保持する構造体であり、BrinOpcInfo を内部で参照します。
  • TypeCacheEntry: oi_typcache 配列に格納される、データ型のキャッシュ情報。

失敗モードと境界条件

  • インデックス列の数と oi_typcache のサイズが不一致の場合、メモリ破壊や不正なアクセスが発生する可能性があります。
  • 演算子クラスが oi_opaque を適切に初期化していない場合、後続のプロシージャ呼び出しで予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は BRIN インデックスの内部実装に深く関与しています。新しい演算子クラスを追加する際は、oi_nstored や oi_typcache が正しく設定されていることを確認してください。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体の割り当て時には palloc_array 等を用いて適切なサイズを確保する必要があります。
