wait_on_slots

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/task.c:364

目的

複数のデータベース接続（UpgradeTaskSlot）を監視し、I/O 準備が整った接続を特定して、後続の処理で利用可能な状態にする。

シグネチャまたは形状

static void wait_on_slots(UpgradeTaskSlot *slots, int numslots);

入力と出力

  • **入力**:
  • slots: 監視対象となる UpgradeTaskSlot 構造体の配列。
  • numslots: 配列内のスロット数。
  • **出力**: なし（副作用として slots 配列内の各要素の ready フラグおよび sock フィールドを更新する）。

前提条件と不変条件

  • slots 配列は有効なメモリ領域を指していること。
  • process_slot() が呼び出されるたびに、前回の wait_on_slots() で ready とマークされたスロットの処理が完了していることが前提とされる。
  • FREE 状態のスロットは、処理すべきデータベースが存在しないものとして無視される。

アルゴリズムまたは振る舞い

1. 全スロットの ready フラグを false に初期化する。 2. FREE 状態以外の各スロットについて、PQsocket() でソケットを取得し、select_mode に応じて input または output の fd_set に追加する。 3. select_loop() を呼び出し、いずれかのソケットが準備完了になるまで待機する。 4. 待機終了後、各スロットのソケットが input または output セットに含まれているかを確認し、準備完了であれば slots[i].ready を true に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_upgrade/task.c 内の並列実行ループから呼び出される（具体的な関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: PQsocket (libpq), select_loop (同ファイル内), FD_ZERO, FD_SET, FD_ISSET, Max, pg_fatal。

関連データ構造

  • UpgradeTaskSlot: 接続状態、ソケット、および準備完了フラグを保持する構造体。

失敗モードと境界条件

  • PQsocket() が負の値を返した場合、pg_fatal により異常終了する。
  • select_loop() が -1 を返した場合、pg_fatal により異常終了する。
  • numslots が 0 の場合、ループは実行されず即座に終了する。

保守担当者向けメモ

  • 本関数は libpq の非同期 I/O を多重化するための低レベルな待機ループである。
  • src/fe_utils/parallel_slot.c にも同名の関数が存在するが、本ページは src/bin/pg_upgrade/task.c の実装を対象としている点に注意すること。
