numeric_random

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pseudorandomfuncs.c:173

目的

指定された範囲（rmin から rmax）内で、numeric 型の疑似乱数を生成して返却します。

シグネチャまたは形状

Datum numeric_random(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_NUMERIC(0): 乱数範囲の最小値 (rmin)
  • PG_GETARG_NUMERIC(1): 乱数範囲の最大値 (rmax)
  • **出力**:
  • PG_RETURN_NUMERIC(result): 生成された numeric 型の乱数値

前提条件と不変条件

  • 内部状態 prng_state が初期化されている必要があります（関数内で initialize_prng() を呼び出し保証）。
  • rmin および rmax は numeric 型として有効である必要があります。

アルゴリズムまたは振る舞い

1. initialize_prng() を呼び出し、プロセス内の乱数生成器の状態を初期化します。 2. random_numeric 関数を呼び出し、現在の prng_state を用いて rmin から rmax の範囲内の乱数を算出します。 3. 算出された結果を Datum 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（直接的な呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**:
  • initialize_prng()
  • random_numeric()

関連データ構造

  • prng_state: プロセス単位で保持される乱数生成器の状態構造体。
  • Numeric: PostgreSQLの数値型データ構造。

失敗モードと境界条件

  • initialize_prng() が内部的にシード生成に失敗した場合の挙動は、当該関数の実装に依存します。
  • rmin と rmax の大小関係や、numeric 型としての妥当性については random_numeric 側のバリデーションに依存します。

保守担当者向けメモ

  • 本関数は numeric 型専用の乱数生成器です。他の型（int4, int8 等）については同ファイル内の別関数を参照してください。
  • prng_state はプロセス固有の静的変数であるため、並列クエリ間での同期は行われません。
