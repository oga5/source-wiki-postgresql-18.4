VarBit

定義

  • 種別: struct
  • 定義箇所: src/include/utils/varbit.h:30

目的

PostgreSQLの BIT(n) および BIT VARYING(n) 型のデータをメモリ上で保持するための構造体。

シグネチャまたは形状

typedef struct {
    int32    vl_len_;    /* varlenaヘッダー */
    int32    bit_len;    /* 有効ビット数 */
    bits8    bit_dat[FLEXIBLE_ARRAY_MEMBER]; /* ビット列本体 */
} VarBit;

入力と出力

  • 入力: PG_GETARG_VARBIT_P マクロ等を通じて Datum から変換されたポインタとして受け取られる。
  • 出力: VarBitPGetDatum マクロ等を通じて Datum に変換され、SQLエンジンへ返される。

前提条件と不変条件

  • vl_len_ は varlena ヘッダーであり、直接操作してはならない。
  • bit_len が8の倍数でない場合、最後のバイトの未使用ビット（パディング）は必ずゼロでなければならない。この不変条件により、比較処理等の最適化が可能となる。

アルゴリズムまたは振る舞い

  • varlena 構造体と互換性を持つが、bit_len フィールドが追加されている。
  • ビット列の操作には、VARBITLEN、VARBITBYTES、VARBITPAD などの専用マクロを使用する。
  • メモリレイアウトは bit_dat が先頭バイトから順に格納される。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/varbit.c 内の関数（bit_catenate, bitsubstring, bit_overlay 等）がこの構造体を操作する。
  • 呼び出し先: 具体的な呼び出し先（関数呼び出しの連鎖）については、提供されたソース情報からは特定できない。

関連データ構造

  • varlena: PostgreSQLの可変長データ型の基底構造体。
  • Datum: PostgreSQLの汎用データ表現。

失敗モードと境界条件

  • bit_len が8の倍数でない際のパディング処理を怠ると、比較演算等の結果が不正になる可能性がある。
  • メモリ確保時には VARBITTOTALLEN マクロを使用して必要なサイズを正しく算出する必要がある。

保守担当者向けメモ

  • 直接メモリを操作せず、必ず src/include/utils/varbit.h で提供されているマクロ経由でアクセスすること。
  • bit_dat はフレキシブル配列メンバーであるため、構造体のサイズは動的に決定される。
