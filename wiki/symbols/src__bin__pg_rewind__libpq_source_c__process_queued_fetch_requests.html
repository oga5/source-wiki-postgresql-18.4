process_queued_fetch_requests

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/libpq_source.c:425

目的

蓄積されたファイル取得リクエストをバッチ処理し、リモートサーバーからデータチャンクを取得してローカルのターゲットファイルへ書き込む。

シグネチャまたは形状

static void process_queued_fetch_requests(libpq_source *src)

入力と出力

  • **入力**: libpq_source *src（リクエストキュー request_queue および接続情報を含む構造体）
  • **出力**: なし（副作用としてターゲットファイルシステムを更新）

前提条件と不変条件

  • src->num_requests が 0 より大きいこと。
  • リモートサーバー側で fetch_chunks_stmt プリペアドステートメントが準備されていること。
  • リクエストされたファイルパス、オフセット、長さが src 内の StringInfo バッファに正しくエンコードされていること。

アルゴリズムまたは振る舞い

1. src->request_queue 内の全リクエストを配列形式の文字列に変換し、fetch_chunks_stmt のパラメータとして設定する。 2. PQsendQueryPrepared を用いてクエリを送信し、PQsetSingleRowMode でストリーミング受信を有効化する。 3. PQgetResult をループし、各タプル（パス、オフセット、チャンクデータ）を処理する。 4. チャンクデータが NULL の場合、リモートで削除されたファイルとみなし、remove_target_file を呼び出す。 5. データが存在する場合、リクエスト内容と整合性を検証し、open_target_file および write_target_range を通じてローカルへ書き込む。 6. 全リクエストの処理完了後、PQclear で結果を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • libpq_queue_fetch_range (バッチ実行トリガーとして)
  • libpq_finish_fetch (終了処理として)
  • **呼び出し先**:
  • PQsendQueryPrepared, PQsetSingleRowMode, PQgetResult, PQresultStatus, PQnfields, PQntuples, PQftype, PQfformat, PQgetisnull, PQgetlength, PQgetvalue, PQclear (libpq API)
  • remove_target_file, open_target_file, write_target_range (ファイル操作)
  • pg_fatal, pg_log_debug, pg_malloc, pg_free (ユーティリティ)

関連データ構造

  • libpq_source: リクエストキューと接続状態を保持するメイン構造体。
  • fetch_range_request: 個別のファイル取得リクエスト（パス、オフセット、長さ）。

失敗モードと境界条件

  • **通信エラー**: PQsendQueryPrepared 失敗時に pg_fatal を発行。
  • **データ不整合**: 受信したチャンクのパスやオフセットがリクエストと一致しない場合、または型情報が期待と異なる場合に pg_fatal を発行。
  • **ファイル削除**: リモートでファイルが削除されている場合、remove_target_file によりローカル側でも削除を同期する。
  • **切り詰め**: ソース側でファイルが切り詰められた場合、受信データがリクエストより少なくなる可能性があるが、これは許容される。

保守担当者向けメモ

  • MAX_CHUNKS_PER_QUERY の設定がスループットに直結する。
  • PQgetResult のループ内での chunkno 管理は、リクエスト順序とレスポンス順序が一致することを前提としている。
