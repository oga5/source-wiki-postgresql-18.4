removable_cutoff

定義

  • 種別: function
  • 定義箇所: src/test/modules/injection_points/regress_injection.c:41

目的

テスト環境において、指定されたリレーションに基づき、削除不可能な最も古いトランザクションID（XID）を安定して取得する。

シグネチャまたは形状

Datum removable_cutoff(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_FUNCTION_ARGS: オプションでリレーションのOIDを引数として受け取る。
  • **出力**:
  • Datum: 取得したXIDと現在のエポックを組み合わせた FullTransactionId を返す。

前提条件と不変条件

  • 引数としてリレーションOIDが提供された場合、そのリレーションに対して AccessShareLock を取得する。
  • 非共有リレーションかつ autovacuum_start_daemon が有効な場合、バックグラウンドプロセスにより値が後退する可能性があるため警告を発する。

アルゴリズムまたは振る舞い

1. 引数があればリレーションを開き、ロックを取得する。 2. ReadNextFullTransactionId で現在のトランザクションIDを取得する。 3. do-while ループ内で以下を実行する:

  • CHECK_FOR_INTERRUPTS() を呼び出し、割り込みをチェックする。
  • GetOldestNonRemovableTransactionId を呼び出し、削除不可能な最も古いXIDを取得する。
  • 再度 ReadNextFullTransactionId を呼び出し、取得前後でトランザクションIDが進行していないか確認する。

    • 4. トランザクションIDの進行がなくなるまでループを繰り返す（エポックの不整合を防ぐための防衛的実装）。 5. リレーションを開いた場合は閉じる。 6. FullTransactionIdFromAllowableAt を用いて結果を構築し、返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから呼び出されることを想定（PG_FUNCTION_INFO_V1 による登録あり）。
  • **呼び出し先**:
  • table_open, table_close
  • GetOldestNonRemovableTransactionId
  • ReadNextFullTransactionId
  • FullTransactionIdEquals, FullTransactionIdFromAllowableAt
  • CHECK_FOR_INTERRUPTS

関連データ構造

  • Relation: リレーション情報を保持。
  • FullTransactionId: 64bitトランザクションID。

失敗モードと境界条件

  • 非共有リレーションに対してオートバキュームが有効な場合、値が後退する可能性がある。
  • 実行中にトランザクションIDが進行した場合、ループによるリトライで整合性を確保する。

保守担当者向けメモ

  • 本関数は syscache-update-pruned.spec 等の回帰テストでの利用を主目的としている。
  • do-while ループは、関数実行中に oldestXid が進行し、誤ったエポックに割り当てられることを防ぐために必須である。
