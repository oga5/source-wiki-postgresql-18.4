PLyMapping_ToJsonbValue

定義

  • **種別**: function
  • **定義箇所**: contrib/jsonb_plpython/jsonb_plpython.c:264

目的

Python のマッピングオブジェクト（辞書等）を PostgreSQL の JsonbValue 構造体へ変換し、JSONB オブジェクトとして構築します。

シグネチャまたは形状

JsonbValue *PLyMapping_ToJsonbValue(PyObject *obj, JsonbParseState **jsonb_state)

入力と出力

  • **入力**:
  • obj: 変換対象の Python マッピングオブジェクト。
  • jsonb_state: JSONB 構築状態を保持する JsonbParseState へのポインタ。
  • **出力**: 構築された JsonbValue 構造体へのポインタを返します。

前提条件と不変条件

  • obj は PyMapping プロトコルをサポートしている必要があります。
  • jsonb_state は有効な JsonbParseState を指している必要があります。
  • 変換処理は PG_TRY ブロック内で実行され、終了時に items（Python 辞書のアイテムリスト）の参照カウントが適切に解放されることが保証されます。

アルゴリズムまたは振る舞い

1. PyMapping_Size および PyMapping_Items を使用して、Python オブジェクトからキーと値のペアを取得します。 2. pushJsonbValue を呼び出し、WJB_BEGIN_OBJECT を挿入して JSONB オブジェクトの開始を宣言します。 3. 各要素をループ処理します：

  • キーが None の場合は空文字列として扱います。
  • それ以外のキーは PLyUnicode_ToJsonbValue を介して文字列に変換します。
  • WJB_KEY を挿入し、続いて PLyObject_ToJsonbValue を再帰的に呼び出して値を変換・挿入します。

    • 4. WJB_END_OBJECT を挿入して終了し、結果の JsonbValue を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PLyObject_ToJsonbValue（ソースコード内の参照より推測）。
  • **呼び出し先**: PyMapping_Size, PyMapping_Items, pushJsonbValue, PyList_GetItem, PyTuple_GetItem, PLyUnicode_ToJsonbValue, PLyObject_ToJsonbValue, Py_DECREF。

関連データ構造

  • JsonbValue: JSONB の各要素を表現する構造体。
  • JsonbParseState: JSONB 構築中の状態を管理する構造体。
  • PyObject: Python オブジェクト。

失敗モードと境界条件

  • Python 側でマッピングの取得や要素のアクセスに失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断されます。
  • PG_FINALLY ブロックにより、エラー発生時であっても items のメモリリークは防止されます。

保守担当者向けメモ

  • 本関数は再帰的に PLyObject_ToJsonbValue を呼び出すため、深いネストを持つ Python オブジェクトを変換する際はスタックオーバーフローのリスクを考慮してください。
  • Python の None キーを空文字列に変換する仕様は、JSONB のキーが文字列でなければならないという制約に基づいています。
