ReassignOwnedObjects

定義

  • 種別: function
  • 定義箇所: src/backend/commands/user.c:1610

目的

指定されたロール（オーナー）が所有するすべてのデータベースオブジェクトの所有権を、別のロールへ一括で移転する。

シグネチャまたは形状

void ReassignOwnedObjects(ReassignOwnedStmt *stmt)

入力と出力

  • **入力**: ReassignOwnedStmt *stmt（移転対象のロールリストおよび移転先のロールを指定する構文解析ノード）
  • **出力**: なし（成功時はカタログ更新を実行し、失敗時はエラーをスローする）

前提条件と不変条件

  • 実行ユーザーは、移転元となるすべてのロールに対して has_privs_of_role（権限の継承または直接の所有権）を満たしている必要がある。
  • 実行ユーザーは、移転先となるロールに対しても同様に has_privs_of_role を満たしている必要がある。

アルゴリズムまたは振る舞い

1. roleSpecsToIds を呼び出し、stmt->roles に含まれるロール名を対応する Oid のリストに変換する。 2. 移転元ロールのリストを走査し、現在のユーザーが各ロールに対する権限を持っているか検証する。権限がない場合は ERRCODE_INSUFFICIENT_PRIVILEGE でエラーをスローする。 3. get_rolespec_oid を用いて移転先ロールの Oid を取得し、同様に実行ユーザーの権限を検証する。 4. 検証完了後、shdepReassignOwned を呼び出し、共有カタログおよび各データベース内のオブジェクト所有権の移転処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（SQLコマンド実行時）
  • **呼び出し先**: roleSpecsToIds, has_privs_of_role, GetUserId, get_rolespec_oid, GetUserNameFromId, shdepReassignOwned

関連データ構造

  • ReassignOwnedStmt: SQLの REASSIGN OWNED 文を表現するノード。
  • Oid: PostgreSQLのオブジェクト識別子。

失敗モードと境界条件

  • **権限不足**: 移転元または移転先のロールに対して十分な権限がない場合、ERROR を発生させる。
  • **ロール不在**: 指定されたロール名がシステムに存在しない場合、get_rolespec_oid 等の内部処理によりエラーとなる。

保守担当者向けメモ

  • 本関数は高レベルな権限チェックを担当し、実際のオブジェクト所有権変更のロジックは shdepReassignOwned に委譲している。
  • 権限チェックには has_privs_of_role を使用しており、ロールの継承関係を考慮した厳格な検証が行われる。
