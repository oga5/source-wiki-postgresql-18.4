add_one_elt

定義

  • 種別: function
  • 定義箇所: contrib/oid2name/oid2name.c:234

目的

動的配列構造体 eary に新しい文字列要素を追加し、必要に応じて配列のメモリ領域を拡張します。

シグネチャまたは形状

void add_one_elt(char *eltname, eary *eary)

入力と出力

  • **入力**:
  • eltname: 追加する文字列（char *）。
  • eary: 要素を格納する eary 構造体へのポインタ。
  • **出力**: なし（void）。eary 構造体の内部状態（array, num, alloc）が更新されます。

前提条件と不変条件

  • eary ポインタは有効である必要があります。
  • eary->num は現在格納されている要素数を、eary->alloc は確保済みのスロット数を保持している必要があります。

アルゴリズムまたは振る舞い

1. eary->alloc が 0 の場合、初期容量として 8 スロットを確保します。 2. eary->num が eary->alloc に達している場合、eary->alloc を 2 倍に拡張し、pg_realloc でメモリを再確保します。 3. eltname を pg_strdup で複製し、eary->array の末尾（eary->num 番目）に格納します。 4. eary->num をインクリメントします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/oid2name/oid2name.c 内の get_opts 関数（130行目、146行目、171行目）。
  • **呼び出し先**:
  • pg_malloc（メモリ初期確保時）
  • pg_realloc（メモリ拡張時）
  • pg_strdup（文字列の複製）

関連データ構造

  • eary: 検索対象のリストを管理する動的配列構造体。

失敗モードと境界条件

  • メモリ不足が発生した場合、pg_malloc または pg_realloc が内部でエラーを発生させ、プログラムが終了する可能性があります。

保守担当者向けメモ

  • 本関数は eary 構造体のメモリ管理を担っています。eary 構造体の定義や他の操作関数（get_comma_elts 等）と併せて確認してください。
