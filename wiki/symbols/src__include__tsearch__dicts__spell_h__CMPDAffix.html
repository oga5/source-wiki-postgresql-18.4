CMPDAffix

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/dicts/spell.h:147

目的

複合語（Compound）処理において、単語に付与可能な接辞（Affix）の情報を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	const char *affix;
	int			len;
	bool		issuffix;
} CMPDAffix;

入力と出力

本構造体はデータ保持用であり、関数ではありません。IspellDict 構造体内の CompoundAffix フィールドを通じて、辞書構築時および検索時に参照されます。

前提条件と不変条件

  • affix は接辞の文字列を指します。
  • len は affix の長さを保持します。
  • issuffix が true の場合は接尾辞、false の場合は接頭辞として扱われます。

アルゴリズムまたは振る舞い

辞書構築時に palloc を用いてメモリが確保され、CheckCompoundAffixes 関数において、対象単語が特定の接辞ルールに適合するかを判定するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/spell.c 内の辞書構築ロジックおよび CheckCompoundAffixes 関数。
  • **呼び出し先**: 本構造体自体はデータ構造であるため、関数を呼び出すことはありません。

関連データ構造

  • IspellDict: CompoundAffix フィールドを通じて本構造体の配列を保持します。

失敗モードと境界条件

  • 辞書構築時に Conf->naffixes に基づいてメモリが確保されるため、メモリ不足時には palloc によるエラーが発生する可能性があります。
  • repalloc による動的なサイズ変更が行われるため、ポインタの整合性に注意が必要です。

保守担当者向けメモ

  • IspellDict の一部として管理されており、辞書のライフサイクル（構築から破棄まで）に従ってメモリ管理が行われます。
  • CheckCompoundAffixes 関数でこの構造体がどのように走査・比較されているかを確認する際は、spell.c の該当箇所を参照してください。
