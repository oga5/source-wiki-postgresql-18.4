getDatumCopy

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginbulk.c:127

目的

GINインデックス構築時に、キーとなる Datum のコピーを作成し、必要に応じてメモリ使用量を BuildAccumulator に記録する。

シグネチャまたは形状

static Datum getDatumCopy(BuildAccumulator *accum, OffsetNumber attnum, Datum value)

入力と出力

  • **入力**:
  • accum: 構築状態を管理する BuildAccumulator 構造体へのポインタ。
  • attnum: 対象属性のオフセット番号。
  • value: コピー対象の Datum。
  • **出力**:
  • コピーされた Datum（値渡し型の場合は元の値、参照型の場合はメモリ確保されたコピー）。

前提条件と不変条件

  • accum->ginstate が有効であり、origTupdesc が初期化されていること。
  • 参照型（attbyval が偽）の場合、datumCopy によってメモリが確保される。

アルゴリズムまたは振る舞い

1. TupleDescCompactAttr を使用して、指定された属性のメタデータ（attbyval, attlen）を取得する。 2. 属性が値渡し（attbyval が真）であれば、value をそのまま返す。 3. 属性が参照型であれば、datumCopy を呼び出してメモリ上にコピーを作成する。 4. 参照型の場合、確保されたメモリサイズを GetMemoryChunkSpace で算出し、accum->allocatedMemory に加算する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/ginbulk.c 内の ginInsertBAEntry（ea->key = getDatumCopy(...) として呼び出し）。
  • **呼び出し先**: datumCopy, TupleDescCompactAttr, GetMemoryChunkSpace。

関連データ構造

  • BuildAccumulator: GIN構築状態を保持する構造体。
  • CompactAttribute: 属性の物理的特性を保持する構造体。

失敗モードと境界条件

  • datumCopy 内部でメモリ不足が発生した場合、PostgreSQLのメモリ管理機構によりエラーがスローされる可能性がある。

保守担当者向けメモ

  • 本関数は BuildAccumulator 内のメモリ消費量を追跡する重要な役割を担っている。参照型を扱う際は、必ず allocatedMemory の更新が漏れないように注意すること。
