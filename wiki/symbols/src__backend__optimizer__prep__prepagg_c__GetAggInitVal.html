GetAggInitVal

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/prep/prepagg.c:520

目的

集約関数の初期値（text型として保持されているもの）を、指定された遷移型（transtype）の内部表現（Datum）に変換して取得します。

シグネチャまたは形状

Datum GetAggInitVal(Datum textInitVal, Oid transtype)

入力と出力

  • **入力**:
  • textInitVal: 初期値を表す text 型の Datum。
  • transtype: 変換先のデータ型を表す Oid。
  • **出力**:
  • 変換後の Datum 型の初期値。

前提条件と不変条件

  • transtype に対応する入力関数（Input Function）がシステムカタログから取得可能であること。
  • textInitVal は有効な text 型のデータであること。

アルゴリズムまたは振る舞い

1. getTypeInputInfo を呼び出し、transtype に対応する入力関数（typinput）と型パラメータ（typioparam）を取得します。 2. TextDatumGetCString を使用して、textInitVal を C 文字列に変換します。 3. OidInputFunctionCall を呼び出し、取得した入力関数を用いて文字列を transtype の Datum 形式へ変換します。 4. 変換に使用した一時的な C 文字列を pfree で解放します。 5. 変換された Datum を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeWindowAgg.c
  • src/backend/optimizer/prep/prepagg.c
  • **呼び出し先**:
  • getTypeInputInfo
  • TextDatumGetCString
  • OidInputFunctionCall
  • pfree

関連データ構造

  • Datum: PostgreSQL の汎用データ表現。
  • Oid: 型識別子。

失敗モードと境界条件

  • transtype が無効な型である場合、getTypeInputInfo がエラーを発生させる可能性があります。
  • 入力文字列が transtype の形式として不正な場合、OidInputFunctionCall がエラーを発生させます。

保守担当者向けメモ

  • 本関数は nodeAgg.c および nodeWindowAgg.c からも利用されており、集約の初期化処理における共通ユーティリティとして機能しています。変更時はこれらの実行エンジン側の動作にも影響を与える点に注意してください。
