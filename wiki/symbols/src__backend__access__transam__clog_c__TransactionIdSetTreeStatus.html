TransactionIdSetTreeStatus

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/clog.c:182

目的

トランザクション（親）とそのサブトランザクション群の状態を、CLOG（Commit Log）上にまとめて記録する。

シグネチャまたは形状

void TransactionIdSetTreeStatus(TransactionId xid, int nsubxids, TransactionId *subxids, XidStatus status, XLogRecPtr lsn)

入力と出力

  • xid: 親トランザクションID。
  • nsubxids: サブトランザクションの数。
  • subxids: サブトランザクションIDの配列。
  • status: 設定する状態（TRANSACTION_STATUS_COMMITTED または TRANSACTION_STATUS_ABORTED）。
  • lsn: この更新に関連付けられる WAL の LSN。

前提条件と不変条件

  • status は TRANSACTION_STATUS_COMMITTED または TRANSACTION_STATUS_ABORTED であること（Assert で検証）。
  • サブトランザクションが複数ページにまたがる場合、整合性を保つため、まず TRANSACTION_STATUS_SUB_COMMITTED を経由する。

アルゴリズムまたは振る舞い

1. 親トランザクションのページを特定する。 2. 全てのサブトランザクションが親と同じページに収まるか判定する。 3. **単一ページの場合**: TransactionIdSetPageStatus を呼び出し、一括で状態を更新する。 4. **複数ページにまたがる場合**:

  • ステータスが COMMITTED の場合、最初のページ以外のサブトランザクションを TRANSACTION_STATUS_SUB_COMMITTED に設定する。
  • 親と最初のページにあるサブトランザクションを TransactionIdSetPageStatus で更新する。
  • 残りのページにあるサブトランザクションを set_status_by_pages を用いて順次更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/transam.c
  • **呼び出し先**:
  • TransactionIdToPage
  • TransactionIdSetPageStatus
  • set_status_by_pages (内部ヘルパー)

関連データ構造

  • XidStatus: トランザクションの状態を定義する列挙型。
  • TransactionId: トランザクション識別子。

失敗モードと境界条件

  • サブトランザクションが複数の CLOG ページにまたがる場合、中間状態（SUB_COMMITTED）を経由しないとクラッシュリカバリ時に不整合が生じる可能性がある。
  • ページ境界を跨ぐ際の処理順序が、アトミック性を維持するための重要な境界条件となる。

保守担当者向けメモ

  • 本関数は clog.c の主要なエントリポイントであり、サブトランザクションのコミット処理において整合性を保証する責務を持つ。
  • set_status_by_pages は本関数のヘルパーとして定義されている。
  • ページを跨ぐ更新時の SUB_COMMITTED への遷移は、クラッシュリカバリ時の整合性確保に不可欠であるため、変更時は注意が必要。
