consider_index_join_outer_rels

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/indxpath.c:502

目的

インデックス結合条件（indexjoinclauses）に基づき、パラメータ化されたインデックススキャンのための有効なリレーションIDの組み合わせを探索し、パス生成を試みる。

シグネチャまたは形状

static void consider_index_join_outer_rels(
    PlannerInfo *root, 
    RelOptInfo *rel,
    IndexOptInfo *index,
    IndexClauseSet *rclauseset,
    IndexClauseSet *jclauseset,
    IndexClauseSet *eclauseset,
    List **bitindexpaths,
    List *indexjoinclauses,
    int considered_clauses,
    List **considered_relids
)

入力と出力

  • **入力**:
  • root, rel, index: プランナ情報、対象リレーション、インデックス情報。
  • rclauseset, jclauseset, eclauseset: 制約条件、結合条件、等価クラス条件のセット。
  • indexjoinclauses: 適用可能な結合条件のリスト。
  • considered_clauses: ヒューリスティックな制限のための基準値。
  • **出力**:
  • bitindexpaths: 生成されたパスが追加されるリスト。
  • considered_relids: 試行済みのリレーションIDセットのリスト（更新される）。

前提条件と不変条件

  • considered_relids は、既に試行済みのリレーションIDセットを保持しており、重複した組み合わせの再試行を避けるために使用される。
  • 計画作成時間の爆発を防ぐため、considered_relids の長さは 10 * considered_clauses に制限される。

アルゴリズムまたは振る舞い

1. indexjoinclauses 内の各結合条件を走査する。 2. 既に試行済みのリレーションIDセット（considered_relids）と、現在の結合条件のリレーションIDとの和集合を生成する。 3. 以下の条件を満たす場合、get_join_index_paths を呼び出してパス生成を試みる：

  • 既存のセットと包含関係にないこと（bms_subset_compare が BMS_DIFFERENT）。
  • 同一の等価クラス（parent_ec）から派生した条件が既に考慮されていないこと。
  • 試行回数の上限に達していないこと。

    • 4. 最後に、現在の結合条件単独でのパス生成を get_join_index_paths で試みる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/indxpath.c 内の create_index_paths から呼び出される。
  • **呼び出し先**: get_join_index_paths を呼び出す。また、eclass_already_used を使用して条件の重複判定を行う。

関連データ構造

  • IndexClause: 結合条件を保持する構造体。
  • Relids: リレーションIDの集合（ビットマップ）。
  • EquivalenceClass: 等価クラス。

失敗モードと境界条件

  • considered_relids の長さが制限（10 * considered_clauses）を超えた場合、新しい組み合わせの生成を停止する。
  • 包含関係にあるリレーションIDセットはスキップされるため、冗長なパス生成が抑制される。

保守担当者向けメモ

  • 本関数は、パラメータ化されたインデックススキャンの探索空間を制御する重要なヒューリスティックを含んでいる。
  • bms_subset_compare によるチェックは簡易的なものであり、完全な冗長性テストではない点に注意すること。
