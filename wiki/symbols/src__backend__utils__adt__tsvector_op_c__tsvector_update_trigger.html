tsvector_update_trigger

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/tsvector_op.c:2742

目的

テーブルの特定のテキストカラムの内容に基づき、tsvector 型カラムを自動的に更新するトリガー関数。

シグネチャまたは形状

static Datum tsvector_update_trigger(PG_FUNCTION_ARGS, bool config_column)

入力と出力

  • **入力**:
  • fcinfo: トリガー実行コンテキスト（TriggerData を含む）。
  • config_column: true の場合、トリガー引数の2番目をテキスト検索設定名が格納されたカラム名として扱う。false の場合は設定名そのものとして扱う。
  • **出力**:
  • Datum: 更新された行データ（HeapTuple）を返す。

前提条件と不変条件

  • **トリガー制約**: BEFORE トリガーとして、かつ行レベル（FOR EACH ROW）で実行される必要がある。
  • **イベント制約**: INSERT または UPDATE イベントでのみ動作する。
  • **引数制約**: トリガー引数は最低3つ必要（tsvector カラム名、設定名/設定カラム名、対象テキストカラム名...）。
  • **型制約**: 指定された tsvector カラムは tsvector 型とバイナリ互換であること。テキストカラムは text 型とバイナリ互換であること。

アルゴリズムまたは振る舞い

1. **コンテキスト検証**: トリガーマネージャ経由の呼び出しであるか、および実行タイミングを確認する。 2. **引数解析**: トリガー引数から対象カラム名と設定情報を取得する。 3. **設定取得**: config_column が true なら対象カラムから設定OIDを取得し、false ならスキーマ修飾された名前からOIDを解決する。 4. **テキスト解析**: 指定された全テキストカラムを走査し、parsetext を用いて語彙を抽出する。 5. **更新判定**: UPDATE の場合、対象カラムが更新されたかを確認し、更新が必要な場合のみ make_tsvector で新しい tsvector を生成する。 6. **タプル更新**: heap_modify_tuple_by_cols を使用して、対象行の tsvector カラムを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • tsvector_update_trigger_byid (引数 config_column=true)
  • tsvector_update_trigger_bycolumn (引数 config_column=false)
  • **呼び出し先**:
  • SPI インターフェース関数群（SPI_fnumber, SPI_getbinval 等）
  • parsetext
  • make_tsvector
  • heap_modify_tuple_by_cols

関連データ構造

  • TriggerData: トリガー実行時のメタデータ。
  • ParsedText: テキスト解析結果を保持する構造体。
  • HeapTuple: 更新対象の行データ。

失敗モードと境界条件

  • **エラー**:
  • トリガーが BEFORE 以外、または行レベル以外で実行された場合。
  • 指定されたカラムが存在しない、または型が不適切な場合。
  • 設定カラムが NULL の場合。
  • 設定名がスキーマ修飾されていない場合。
  • **境界条件**:
  • tsvector の位置情報は uint16 に制限されるため、解析対象のテキストが長すぎる場合や語彙が多すぎる場合に制約を受ける可能性がある。

保守担当者向けメモ

  • 本関数は tsvector の自動更新を担うため、トリガー定義の引数順序（tsvector_field, ts_config, text_field1...）の変更は破壊的変更となる。
  • palloc で確保した ParsedWord 配列は関数終了時に自動的に解放されるが、make_tsvector の結果は pfree で適切に管理する必要がある。
