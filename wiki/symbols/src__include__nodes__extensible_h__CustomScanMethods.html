CustomScanMethods

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/extensible.h:112

目的

カスタムスキャン計画（CustomScan）から、実行時に必要な状態（CustomScanState）を生成するためのインターフェースを定義します。

シグネチャまたは形状

typedef struct CustomScanMethods
{
    const char *CustomName;
    Node *(*CreateCustomScanState) (CustomScan *cscan);
} CustomScanMethods;

入力と出力

  • **入力**: CreateCustomScanState は CustomScan 構造体へのポインタを受け取ります。
  • **出力**: CreateCustomScanState は、生成された実行状態（CustomScanState）を Node * として返します。

前提条件と不変条件

  • CustomName は一意である必要があります。
  • 拡張モジュールは、この構造体を RegisterCustomScanMethods を通じてコアシステムに登録する必要があります。

アルゴリズムまたは振る舞い

1. プランナーがカスタムスキャンを計画に組み込む際、このメソッド群が参照されます。 2. エグゼキュータが実行を開始する際、CreateCustomScanState が呼び出され、プランノードから実行用の状態ノードが生成されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/nodes/extensible.c 内の登録・検索関数によって管理されます。
  • **呼び出し先**: 拡張モジュール側で実装された関数が、関数ポインタを通じて呼び出されます。

関連データ構造

  • CustomScan: プランノード。
  • CustomScanState: 実行状態ノード。
  • src/include/nodes/plannodes.h: CustomScanMethods への参照を保持する CustomScan 構造体が定義されています。

失敗モードと境界条件

  • 登録されていない名前で検索を行った場合の挙動については、ソースコードの断片からは明示されていません。

保守担当者向けメモ

  • 本構造体は拡張機能のインターフェースの一部であり、RegisterCustomScanMethods を介してコアシステムに登録されます。
  • CustomName は EXTNODENAME_MAX_LEN（64バイト）以内で一意である必要があります。
