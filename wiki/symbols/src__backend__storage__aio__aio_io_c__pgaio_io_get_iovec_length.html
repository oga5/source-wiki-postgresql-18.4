pgaio_io_get_iovec_length

定義

  • src/backend/storage/aio/aio_io.c:218

目的

非同期I/Oハンドル（PgAioHandle）に関連付けられた iovec 配列の開始アドレスを取得し、その要素数を返します。

シグネチャまたは形状

int pgaio_io_get_iovec_length(PgAioHandle *ioh, struct iovec **iov)

入力と出力

  • **入力**:
  • ioh: 操作対象の PgAioHandle へのポインタ。
  • iov: iovec 配列の開始アドレスを格納するための出力用ポインタ。
  • **出力**:
  • 戻り値: iovec 配列の要素数（int）。

前提条件と不変条件

  • ioh->state が PGAIO_HS_DEFINED 以上である必要があります（Assert により検証）。

アルゴリズムまたは振る舞い

1. ioh->iovec_off を使用して、グローバルな pgaio_ctl->iovecs 配列内の該当ハンドルに対応する iovec の開始位置を特定し、*iov に代入します。 2. ioh->op の値（PGAIO_OP_READV または PGAIO_OP_WRITEV）に基づき、ハンドル内の op_data から対応する iov_length を取得して返します。 3. 定義されていない操作型が指定された場合は pg_unreachable() を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/method_worker.c:552 での呼び出しが確認されています。
  • **呼び出し先**: 本関数は直接的な関数呼び出しを行わず、構造体メンバへのアクセスと条件分岐のみを行います。

関連データ構造

  • PgAioHandle: I/O操作の状態とパラメータを保持する構造体。
  • iovec: POSIX標準のI/Oベクタ構造体。
  • pgaio_ctl: システム全体のAIO制御構造体。

失敗モードと境界条件

  • ioh が指すハンドルが初期化されていない、または状態が PGAIO_HS_DEFINED 未満である場合、アサーション違反が発生します。
  • ioh->op が PGAIO_OP_READV または PGAIO_OP_WRITEV 以外の場合、プログラムは到達不能として処理されます。

保守担当者向けメモ

  • 本関数は pgaio_ctl のグローバルな iovecs 配列に依存しています。この配列のメモリレイアウトやライフサイクルに変更を加える際は注意してください。
