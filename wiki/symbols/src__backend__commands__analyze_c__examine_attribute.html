examine_attribute

定義

  • 種別: function
  • 定義箇所: src/backend/commands/analyze.c:1035

目的

ANALYZE 処理において、特定のテーブル列またはインデックス式に対する統計情報収集の準備（初期化）を行う。

シグネチャまたは形状

static VacAttrStats *examine_attribute(Relation onerel, int attnum, Node *index_expr)

入力と出力

  • **入力**:
  • onerel: 対象リレーションの Relation 構造体。
  • attnum: 対象列の番号。
  • index_expr: インデックス式（列単体分析の場合は NULL）。
  • **出力**:
  • 成功時: 初期化済みの VacAttrStats 構造体へのポインタ。
  • 失敗時: NULL（削除済み列、仮想生成列、または統計対象外と指定された列の場合）。

前提条件と不変条件

  • 削除済み列（attisdropped）および仮想生成列は分析対象外とする。
  • pg_attribute システムカタログから取得した attstattarget が 0 の場合、分析をスキップする。
  • インデックス式が提供された場合、列のデータ型ではなく式の型情報を優先して使用する。

アルゴリズムまたは振る舞い

1. **属性チェック**: 対象列が分析可能か判定し、不可なら NULL を返す。 2. **統計ターゲット取得**: pg_attribute システムカタログから attstattarget を読み取る。 3. **構造体初期化**: VacAttrStats をメモリ確保し、型情報（OID、typmod、照合順序）を設定する。 4. **型分析関数の呼び出し**:

  • データ型に対応する typanalyze 関数を検索し実行する。
  • 指定がない場合はデフォルトの std_typanalyze() を使用する。

    • 5. **検証**: typanalyze の結果が不正、または compute_stats が設定されていない場合はメモリを解放して NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/analyze.c 内の do_analyze_rel（テーブル列およびインデックス列の分析時）。
  • src/backend/statistics/extended_stats.c（拡張統計情報の分析時）。
  • **呼び出し先**:
  • SearchSysCache2, SysCacheGetAttr, ReleaseSysCache（システムカタログ操作）。
  • exprType, exprTypmod, exprCollation（式情報の抽出）。
  • std_typanalyze（デフォルトの分析関数）。
  • 各データ型固有の typanalyze 関数。

関連データ構造

  • VacAttrStats: 列ごとの統計情報を保持する構造体。
  • Relation: リレーションのメタデータ。
  • Form_pg_attribute, Form_pg_type: システムカタログの行定義。

失敗モードと境界条件

  • **キャッシュルックアップ失敗**: pg_attribute または pg_type の検索に失敗した場合、elog(ERROR) により処理を中断する。
  • **分析スキップ**: ユーザーが統計ターゲットを 0 に設定している場合、または型固有の分析関数が失敗した場合は NULL を返す。

保守担当者向けメモ

  • 本関数は typanalyze 関数ポインタを通じて型ごとの分析ロジックを委譲するプラグイン構造をとっている。
  • anl_context を使用してメモリ管理を行っているため、戻り値の VacAttrStats は呼び出し側で適切に扱う必要がある。
