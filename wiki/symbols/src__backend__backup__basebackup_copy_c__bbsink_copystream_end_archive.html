bbsink_copystream_end_archive

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_copy.c:240

目的

アーカイブ終了時に、現在のバックアップ進捗状況をクライアントへ通知し、ストリームの同期を行う。

シグネチャまたは形状

static void bbsink_copystream_end_archive(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - bbsink_copystream 構造体としてキャストされるシンクインスタンス。
  • **出力**: なし。副作用としてネットワーク経由で進捗メッセージを送信する。

前提条件と不変条件

  • sink は bbsink_copystream 型のインスタンスであること。
  • sink->bbs_state が有効なバックアップ状態を保持していること。

アルゴリズムまたは振る舞い

1. 現在のバックアップ完了バイト数 (state->bytes_done) を mysink->bytes_done_at_last_time_check に保存する。 2. 現在時刻を GetCurrentTimestamp() で取得し、mysink->last_progress_report_time を更新する。 3. PqMsg_CopyData メッセージを開始する。 4. 進捗報告を示すタイプバイト 'p' を送信する。 5. 現在の完了バイト数を 8 バイト整数として送信する。 6. メッセージを終了し、pq_flush_if_writable() を呼び出してネットワークバッファをフラッシュする。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_copystream_ops 構造体を通じて、basebackup サブシステムのメインループから間接的に呼び出される。
  • **呼び出し先**:
  • GetCurrentTimestamp()
  • pq_beginmessage, pq_sendbyte, pq_sendint64, pq_endmessage (libpq プロトコル関数)
  • pq_flush_if_writable()

関連データ構造

  • bbsink_copystream: 本関数が操作するメインのコンテキスト構造体。
  • bbsink_state: 進捗状況 (bytes_done) を参照するために使用。
  • StringInfoData: libpq メッセージ構築用バッファ。

失敗モードと境界条件

  • ネットワーク接続が切断されている場合、pq_flush_if_writable 等の libpq 関数内でエラーが発生する可能性がある。
  • sink が NULL または不正な型である場合の動作は未定義。

保守担当者向けメモ

  • 本関数はアーカイブ終了時の進捗報告を強制的に実行する。
  • bbsink_copystream_ops の end_archive フィールドに登録されている。
  • 進捗報告のタイプバイト 'p' は、クライアント側でストリームを解釈する際のプロトコルの一部である。
