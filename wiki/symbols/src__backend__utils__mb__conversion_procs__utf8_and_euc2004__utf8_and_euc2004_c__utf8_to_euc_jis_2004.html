utf8_to_euc_jis_2004

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_euc2004/utf8_and_euc2004.c:62

目的

UTF-8 エンコーディングの文字列を EUC_JIS_2004 エンコーディングへ変換する PostgreSQL の変換プロシージャです。

シグネチャまたは形状

Datum utf8_to_euc_jis_2004(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の引数を受け取ります。
  • src (CSTRING): 変換元の UTF-8 文字列。
  • dest (CSTRING): 変換結果を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 変換失敗時にエラーを投げるか否かのフラグ。
  • **出力**: 変換されたバイト数を Datum 型（INT32）として返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_UTF8, PG_EUC_JIS_2004) により、呼び出し時にエンコーディングIDが正しいことが検証されます。
  • 変換には内部で定義された euc_jis_2004_from_unicode_tree および ULmapEUC_JIS_2004_combined マップデータを使用します。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認します。 3. UtfToLocal 関数を呼び出し、UTF-8 から EUC_JIS_2004 への変換を実行します。この際、変換マップとツリー構造を参照してバイト単位の変換を行います。 4. 変換されたバイト数を PG_RETURN_INT32 で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の変換フレームワーク（直接の呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: UtfToLocal 関数。

関連データ構造

  • euc_jis_2004_from_unicode_tree: 変換用の検索ツリー構造。
  • ULmapEUC_JIS_2004_combined: 変換用のマッピングテーブル。

失敗モードと境界条件

  • noError 引数が true の場合、変換失敗時にエラーを発生させず処理を継続します。
  • 変換対象のエンコーディングが不一致の場合、CHECK_ENCODING_CONVERSION_ARGS によりエラーが発生します。

保守担当者向けメモ

  • 本関数は変換ロジックの「接着剤」であり、実際の変換テーブルはインクルードされている .map ファイルに依存しています。
  • 変換テーブルの更新が必要な場合は、Cファイルを直接編集せず、対応するマップ生成スクリプトを確認してください。
