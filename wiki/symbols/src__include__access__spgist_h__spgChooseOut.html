spgChooseOut

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist.h:74

目的

SP-GiSTインデックスへのデータ挿入時、演算子クラスが「現在のノードに対してどのような操作を行うべきか」を決定した結果を格納するために使用されます。

シグネチャまたは形状

typedef struct spgChooseOut
{
    spgChooseResultType resultType;
    union {
        struct { int nodeN; int levelAdd; Datum restDatum; } matchNode;
        struct { Datum nodeLabel; int nodeN; } addNode;
        struct {
            bool prefixHasPrefix; Datum prefixPrefixDatum;
            int prefixNNodes; Datum *prefixNodeLabels; int childNodeN;
            bool postfixHasPrefix; Datum postfixPrefixDatum;
        } splitTuple;
    } result;
} spgChooseOut;

入力と出力

  • **入力**: 演算子クラスの choose サポート関数が、挿入対象の値と現在のノード状態に基づいて本構造体を設定します。
  • **出力**: SP-GiSTの挿入処理（spgdoinsert.c等）がこの構造体を参照し、ノードの降下、新規ノードの追加、またはタプルの分割を実行します。

前提条件と不変条件

  • resultType に応じて、result 共用体内の適切な構造体（matchNode, addNode, splitTuple）が初期化されている必要があります。
  • nodeN は0から始まるインデックスとして指定されます。

アルゴリズムまたは振る舞い

resultType に基づき、以下のいずれかの動作を指示します。 1. **spgMatchNode**: 指定された子ノード（nodeN）へ降下します。levelAdd でレベルを調整し、restDatum を次のノードへ渡します。 2. **spgAddNode**: 現在のタプルに新しいノード（nodeLabel）を nodeN の位置に追加します。 3. **spgSplitTuple**: 現在のタプルを分割し、上位レベルと下位レベルの内部タプルを再構築します。プレフィックスの有無やノードラベルの再配置情報を保持します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c が挿入処理のメインフローで本構造体を利用します。また、各データ型の演算子クラス実装（spgkdtreeproc.c, spgquadtreeproc.c, spgtextproc.c, geo_spgist.c, network_spgist.c, rangetypes_spgist.c, spgist_name_ops.c）が本構造体に値を設定します。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • spgChooseResultType: resultType の値を定義する列挙型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • nodeN が現在のノード数を超えて指定された場合、インデックスの破損やセグメンテーション違反を招く可能性があります。
  • splitTuple 使用時に prefixNodeLabels が適切に割り当てられていない場合、メモリ不正アクセスの原因となります。

保守担当者向けメモ

  • 本構造体はSP-GiSTの挿入ロジックの核となるため、変更時は spgdoinsert.c の処理フローへの影響を十分に検証してください。
  • splitTuple の各フィールドは複雑なタプル再構築を制御するため、追加や削除を行う際は spgxlog.h で定義されるWALレコードとの整合性に注意してください。
