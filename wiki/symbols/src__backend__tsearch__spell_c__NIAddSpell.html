NIAddSpell

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/spell.c:488

目的

辞書データ構造（IspellDict）に新しい単語エントリを追加し、必要に応じてメモリ領域を動的に拡張します。

シグネチャまたは形状

static void NIAddSpell(IspellDict *Conf, const char *word, const char *flag)

入力と出力

  • **入力**:
  • Conf: 辞書の状態を保持する IspellDict 構造体へのポインタ。
  • word: 追加する単語文字列。
  • flag: 単語に関連付けられた接辞フラグ文字列。
  • **出力**: なし（Conf が指す構造体の内部状態を更新）。

前提条件と不変条件

  • Conf は有効な IspellDict 構造体を指している必要があります。
  • Conf->Spell 配列は、Conf->nspell（現在の要素数）と Conf->mspell（確保済み容量）によって管理されます。

アルゴリズムまたは振る舞い

1. **容量確認**: Conf->nspell が Conf->mspell に達している場合、配列を拡張します。

  • 初回（mspell == 0）は 20,480 要素分を tmpalloc で確保します。
  • 2回目以降は mspell を2倍にし、repalloc でメモリを再確保します。

    • 2. **メモリ確保**: SPELLHDRSZ と単語長を合わせたサイズを tmpalloc で確保し、Conf->Spell[Conf->nspell] に割り当てます。 3. **データ格納**:

  • word を構造体の word フィールドにコピーします。
  • flag が空文字でない場合は cpstrdup で複製して格納し、空文字の場合は VoidString を設定します。

    • 4. **インクリメント**: Conf->nspell を 1 増やします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/spell.c 内の辞書読み込み処理（NIImportDictionary 等）から呼び出されます。
  • **呼び出し先**: repalloc, tmpalloc, strlen, strcpy, cpstrdup。

関連データ構造

  • IspellDict: 辞書全体を管理する構造体。
  • SPELL: 単語とフラグ情報を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: tmpalloc または repalloc が失敗した場合、PostgreSQL のメモリ管理機構によりエラーがスローされます。
  • flag が空文字の場合: VoidString が代入されます。

保守担当者向けメモ

  • 本関数は辞書構築時に大量のメモリを消費するため、tmpalloc を使用して構築用メモリコンテキストで管理されています。
  • Conf->mspell の初期値（20,480）は、辞書読み込み時の再割り当て回数を減らすための最適化です。
