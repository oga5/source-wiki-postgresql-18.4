sepgsql_init_client_label

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/label.c:403

目的

SELinux セキュリティコンテキストの初期化および、PostgreSQL の各種フック（認証、プロシージャ実行、トランザクション管理）の登録を行う。

シグネチャまたは形状

void sepgsql_init_client_label(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルなフック変数および client_label_peer を設定）

前提条件と不変条件

  • getcon_raw() が成功し、サーバープロセスのセキュリティコンテキストが取得可能であること。
  • バックグラウンドワーカー（autovacuum 等）の場合、認証ステップを経由しないため、初期状態として SEPGSQL_MODE_INTERNAL が想定される。

アルゴリズムまたは振る舞い

1. getcon_raw() を呼び出し、サーバープロセスのセキュリティコンテキストを client_label_peer に格納する。失敗時は ERROR を発生させる。 2. ClientAuthentication_hook を sepgsql_client_auth に置き換え、元のフックを next_client_auth_hook に退避する。 3. needs_fmgr_hook および fmgr_hook を、それぞれ sepgsql_needs_fmgr_hook および sepgsql_fmgr_hook に置き換え、元の値を退避する。 4. RegisterXactCallback を用いて sepgsql_xact_callback を登録する。 5. RegisterSubXactCallback を用いて sepgsql_subxact_callback を登録する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/sepgsql/hooks.c:466
  • 呼び出し先: getcon_raw, RegisterXactCallback, RegisterSubXactCallback

関連データ構造

  • client_label_peer: クライアントの初期ラベルを保持する変数。
  • ClientAuthentication_hook, needs_fmgr_hook, fmgr_hook: PostgreSQL コアのフック変数。

失敗モードと境界条件

  • getcon_raw() が失敗した場合、ereport(ERROR, ...) によりトランザクションまたはプロセスが中断される。

保守担当者向けメモ

  • 本関数はセッション開始時に実行される初期化処理である。
  • 認証を伴わないバックグラウンドプロセスと、ユーザーセッションの両方で正しく動作するよう設計されている。
  • フックの連鎖（next_..._hook）を保持しているため、他のモジュールとの共存が可能である。
