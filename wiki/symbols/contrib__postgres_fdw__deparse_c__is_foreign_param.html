is_foreign_param

定義

  • 種別: function
  • 定義箇所: contrib/postgres_fdw/deparse.c:1079

目的

与えられた式（Expr）が、リモートサーバーへ送信すべきパラメータ（Param）として扱うべきか、あるいはリモート側のテーブルの列（Var）として直接参照可能かを判定する。

シグネチャまたは形状

bool is_foreign_param(PlannerInfo *root, RelOptInfo *baserel, Expr *expr)

入力と出力

  • **入力**:
  • root: プランニング情報へのポインタ。
  • baserel: 対象となるリレーション情報。
  • expr: 判定対象の式。
  • **出力**:
  • bool: true の場合、その式はリモートサーバーへパラメータとして送信する必要がある。false の場合、リモート側の列として解決可能であるか、パラメータではない。

前提条件と不変条件

  • expr が NULL の場合は常に false を返す。
  • T_Param 型のノードは、常にリモートサーバーへ送信されるべきパラメータとみなされる。

アルゴリズムまたは振る舞い

1. expr のノードタグを評価する。 2. T_Var の場合:

  • baserel が UPPER_REL かどうかを確認し、対象のリレーション ID を特定する。
  • 変数が現在のリレーションに含まれ、かつサブクエリレベルが 0（varlevelsup == 0）であれば、それはリモート側の列であるため false を返す。
  • それ以外の場合はパラメータとして扱うため true を返す。

    • 3. T_Param の場合:

  • 無条件で true を返す。

    • 4. 上記以外のノード型については false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/postgres_fdw/postgres_fdw.c 内の複数箇所（集約処理や条件判定に関連するロジック）。
  • **呼び出し先**:
  • 本関数内から特定の関数呼び出しは行われていない（マクロ nodeTag および bms_is_member を使用）。

関連データ構造

  • PlannerInfo: プランニングの全体状態。
  • RelOptInfo: リレーションの最適化情報。
  • PgFdwRelationInfo: baserel->fdw_private に格納されている FDW 特有のリレーション情報。
  • Var, Param: PostgreSQL の式ノード。

失敗モードと境界条件

  • baserel->fdw_private が適切に初期化されていない場合、キャスト時に予期せぬ動作を引き起こす可能性がある。
  • expr が NULL の場合は安全に false を返す。

保守担当者向けメモ

  • 本関数は postgres_fdw における式評価の安全性とパラメータ化の境界を定義する重要な関数である。
  • 新しいノード型が追加された場合、switch 文に適切なハンドリングを追加する必要がある。
