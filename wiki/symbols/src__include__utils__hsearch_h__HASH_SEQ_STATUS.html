HASH_SEQ_STATUS

定義

  • 種別: struct
  • 定義箇所: src/include/utils/hsearch.h:120

目的

PostgreSQL の動的ハッシュテーブル（dynahash）において、全要素を順次走査（シーケンシャルスキャン）するための状態を保持する構造体です。

シグネチャまたは形状

typedef struct
{
	HTAB	   *hashp;          /* 対象のハッシュテーブル */
	uint32		curBucket;      /* 現在のバケットインデックス */
	HASHELEMENT *curEntry;      /* 現在のバケット内のエントリ */
	bool		hasHashvalue;   /* ハッシュ値指定の有無 */
	uint32		hashvalue;      /* スキャン開始時のハッシュ値 */
} HASH_SEQ_STATUS;

入力と出力

  • 本構造体は hash_seq_init() に渡され、初期化されます。
  • その後、hash_seq_search() の引数として繰り返し渡され、スキャンの進行状況が更新されます。

前提条件と不変条件

  • hashp は有効な HTAB 構造体を指している必要があります。
  • スキャン中にハッシュテーブルに対して挿入や削除を行うと、スキャン結果に影響を与える可能性があります。

アルゴリズムまたは振る舞い

1. hash_seq_init() により、スキャン開始位置（バケットの先頭など）が設定されます。 2. hash_seq_search() を呼び出すたびに、curBucket と curEntry を更新しながら次の有効なエントリを探索します。 3. 全バケットを走査し終えると、hash_seq_search() は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/pg_stat_statements/pg_stat_statements.c
  • contrib/pg_trgm/trgm_regexp.c
  • contrib/postgres_fdw/connection.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数を直接呼び出しません。操作には hash_seq_init() および hash_seq_search() が使用されます。

関連データ構造

  • HTAB: ハッシュテーブル本体。
  • HASHELEMENT: ハッシュテーブル内の個々の要素。

失敗モードと境界条件

  • ハッシュテーブルが空の場合、最初の hash_seq_search() 呼び出しで直ちに NULL が返されます。
  • スキャン中にハッシュテーブルが再構築（拡張）された場合、スキャン状態の整合性が失われる可能性があります。

保守担当者向けメモ

  • 本構造体は不透明（opaque）な HTAB の内部走査を補助するものです。直接メンバを操作せず、dynahash が提供する API を介して利用してください。
