ecpg_is_type_an_array

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/execute.c:163

目的

指定された PostgreSQL の型 OID が ECPG において配列として扱われるべきかを判定し、その結果（ECPG_ARRAY_NONE, ECPG_ARRAY_VECTOR, ECPG_ARRAY_ARRAY）を返します。

シグネチャまたは形状

enum ARRAY_TYPE ecpg_is_type_an_array(int type, const struct statement *stmt, const struct variable *var)

入力と出力

  • **入力**:
  • type: 判定対象の PostgreSQL 型 OID。
  • stmt: 現在の SQL 文のコンテキスト情報。
  • var: 関連付けられたホスト変数の情報。
  • **出力**:
  • enum ARRAY_TYPE: 判定結果。エラー時は ECPG_ARRAY_ERROR を返します。

前提条件と不変条件

  • stmt->connection->cache_head が初期化されていない場合、既知の型情報をキャッシュに事前登録します。
  • キャッシュに存在しない型については、pg_type システムカタログに対してクエリを発行し、typlen が -1 の場合は ECPG_ARRAY_ARRAY、それ以外で typelem <> 0 の場合は ECPG_ARRAY_VECTOR と判定します。

アルゴリズムまたは振る舞い

1. キャッシュ内に該当する型 OID が存在するか確認し、あればその値を返します。 2. キャッシュにない場合、pg_type テーブルから typlen を取得する SQL を実行します。 3. 取得結果に基づき配列型を判定します。ただし、SQL3_CHARACTER または SQL3_CHARACTER_VARYING 型の場合は、配列であっても ECPG_ARRAY_NONE として扱います。 4. 判定結果をキャッシュに保存し、ログを出力した後に結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/ecpglib/execute.c 内のクエリ結果処理ロジックから呼び出されます。
  • **呼び出し先**:
  • ecpg_type_infocache_push (キャッシュ更新)
  • ecpg_alloc / ecpg_free (メモリ管理)
  • PQexec / PQresultStatus / PQntuples / PQgetvalue / PQclear (libpq)
  • ecpg_check_PQresult (エラーチェック)
  • ecpg_dynamic_type (型判定)
  • ecpg_log (ログ出力)

関連データ構造

  • struct statement: SQL 実行コンテキスト。
  • struct variable: ホスト変数情報。
  • ECPGtype_information_cache: 型情報のキャッシュ構造体。
  • enum ARRAY_TYPE: 配列判定結果の列挙型。

失敗モードと境界条件

  • ecpg_alloc によるメモリ確保失敗時は ECPG_ARRAY_ERROR を返します。
  • PQexec が失敗した場合や、結果セットが不正な場合は ECPG_ARRAY_ERROR を返します。
  • キャッシュ更新関数 ecpg_type_infocache_push が失敗した場合も ECPG_ARRAY_ERROR を返します。

保守担当者向けメモ

  • not_an_array_in_ecpg マクロは、PostgreSQL 側では配列とみなされる型であっても、ECPG では配列として扱わない型を明示するために使用されています。
  • 文字列型の配列は現在未対応であるため、明示的に ECPG_ARRAY_NONE にフォールバックするロジックが含まれています。
