bbsink_throttle_begin_backup

定義

  • **種別**: function
  • **定義箇所**: src/backend/backup/basebackup_throttle.c:95

目的

バックアップの転送レート制限（スロットリング）処理を開始する。バックアップの「実データ」転送フェーズの開始をマークし、スロットリング計算の基準となるタイムスタンプを初期化する。

シグネチャまたは形状

static void bbsink_throttle_begin_backup(bbsink *sink)

入力と出力

  • **入力**: bbsink *sink - スロットリング機能を持つ bbsink_throttle 構造体へのポインタ。
  • **出力**: なし（副作用として内部状態を更新）。

前提条件と不変条件

  • 引数 sink は bbsink_throttle 型としてキャスト可能であること。
  • 本関数は bbsink インターフェースの begin_backup コールバックとして呼び出されること。

アルゴリズムまたは振る舞い

1. sink を bbsink_throttle 型にキャストする。 2. bbsink_forward_begin_backup(sink) を呼び出し、後続のシンクへバックアップ開始を通知する。 3. GetCurrentTimestamp() を使用して、現在の時刻を mysink->throttled_last に格納する。これにより、以降の転送量に対する経過時間の計測が開始される。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink インターフェースを介して呼び出される（直接的な呼び出し元はソースコード上では明示されていないが、bbsink_throttle 構造体のメソッドテーブル begin_backup に登録されている）。
  • **呼び出し先**: bbsink_forward_begin_backup（後続のシンクへの転送）、GetCurrentTimestamp（時刻取得）。

関連データ構造

  • bbsink_throttle: スロットリング状態を保持する構造体。throttled_last メンバを更新する。
  • bbsink: バックアップ処理のパイプラインを構成する基底構造体。

失敗モードと境界条件

  • 特筆すべき失敗モードは定義されていない。

保守担当者向けメモ

  • 本関数はバックアップのヘッダ処理をスキップし、実データの転送開始時点からスロットリングの計測を開始する設計となっている。
  • bbsink_forward_begin_backup を経由してチェーン内の他のシンクへ処理を伝播させるため、順序の変更には注意が必要である。
