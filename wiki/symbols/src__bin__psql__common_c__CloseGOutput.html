CloseGOutput

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:109

目的

psql におけるクエリ結果の出力先（ファイルまたはパイプ）を安全にクローズし、必要に応じて終了ステータスを環境変数に反映させる。

シグネチャまたは形状

void CloseGOutput(FILE *gfile_fout, bool is_pipe)

入力と出力

  • gfile_fout (FILE*): クローズ対象のファイルポインタ。NULLの場合は何もしない。
  • is_pipe (bool): gfile_fout がパイプ経由で開かれたものであるかを示すフラグ。

前提条件と不変条件

  • gfile_fout が有効なファイルポインタであるか、NULLであること。
  • is_pipe が true の場合、gfile_fout は popen 等で開かれたパイプである必要がある。

アルゴリズムまたは振る舞い

1. gfile_fout が NULL でないことを確認する。 2. is_pipe が true の場合:

  • pclose(gfile_fout) を実行してパイプを閉じ、その終了ステータスを SetShellResultVariables に渡してシェル変数に反映させる。
  • restore_sigpipe_trap() を呼び出し、SIGPIPE のトラップ状態を復元する。

    • 3. is_pipe が false の場合:

  • fclose(gfile_fout) を実行してファイルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/common.c:2172 (具体的な関数名はソース抜粋からは不明)
  • **呼び出し先**:
  • SetShellResultVariables
  • restore_sigpipe_trap
  • pclose (標準ライブラリ)
  • fclose (標準ライブラリ)

関連データ構造

  • FILE (標準ライブラリ)

失敗モードと境界条件

  • gfile_fout が NULL の場合、関数は何もせず正常に終了する。
  • pclose または fclose の失敗時のエラーハンドリングは、本関数内では明示的に行われていない。

保守担当者向けメモ

  • 本関数は \o や \g コマンドによる出力先切り替えの終了処理で使用される。
  • パイプ処理時に restore_sigpipe_trap を呼び出すことで、出力先が閉じられた後のシグナル状態を適切に管理している点に注意すること。
