SyncRepGetStandbyPriority

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:859

目的

現在のWALSenderプロセスが同期レプリケーションにおいて持つ優先順位（Priority）を判定します。

シグネチャまたは形状

static int SyncRepGetStandbyPriority(void)

入力と出力

  • **入力**: なし（グローバル変数 am_cascading_walsender、SyncRepConfig、application_name を参照）
  • **出力**: int 型の優先順位値。
  • SYNC_REP_PRIORITY モードの場合: 設定リスト内での順位（1以上）。
  • クォーラム方式の場合: 1。
  • 該当しない場合: 0。

前提条件と不変条件

  • カスケードレプリケーション（am_cascading_walsender が真）の場合、優先順位は常に 0 となります。
  • SyncRepConfig が初期化されていない場合、優先順位は 0 となります。

アルゴリズムまたは振る舞い

1. カスケードレプリケーション中であれば 0 を返します。 2. SyncRepConfig が定義されていない場合は 0 を返します。 3. SyncRepConfig->member_names に格納されたスタンバイ名のリストを走査し、現在の application_name と一致するか、またはワイルドカード * が指定されているかを確認します。 4. 一致が見つかった場合、SYNC_REP_PRIORITY モードであればリスト内のインデックス（1始まり）を優先順位として返し、クォーラム方式であれば一律 1 を返します。 5. 一致するものがない場合は 0 を返します。

関連データ構造

  • SyncRepConfig: 同期レプリケーションの設定を保持する構造体。

失敗モードと境界条件

  • application_name が空または設定リストに含まれない場合、優先順位 0 が返されます。
  • SyncRepConfig が NULL の場合、安全に 0 を返します。

保守担当者向けメモ

  • 本関数は application_name に依存するため、WALSenderが正しく識別されている環境で呼び出される必要があります。
  • クォーラム方式（SYNC_REP_QUORUM）への拡張に伴い、優先順位の計算ロジックが分岐している点に注意してください。
