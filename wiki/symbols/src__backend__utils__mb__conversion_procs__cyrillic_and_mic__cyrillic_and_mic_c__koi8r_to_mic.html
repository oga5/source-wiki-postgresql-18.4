koi8r_to_mic

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:309

目的

KOI8-R エンコーディングの文字列を PostgreSQL 内部のマルチバイト表現（MULE_INTERNAL）へ変換する。

シグネチャまたは形状

Datum koi8r_to_mic(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下を取得。
  • src (CSTRING): 変換元の KOI8-R 文字列。
  • dest (CSTRING): 変換後の文字列を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): エラー発生時の挙動フラグ。
  • **出力**: Datum 型として変換されたバイト数を返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_KOI8R, PG_MULE_INTERNAL) マクロにより、ソースエンコーディングが PG_KOI8R、宛先エンコーディングが PG_MULE_INTERNAL であることが検証される。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. CHECK_ENCODING_CONVERSION_ARGS でエンコーディングの整合性を確認する。 3. latin2mic 関数を呼び出し、LC_KOI8_R ロケール設定を用いて変換を実行する。 4. 変換されたバイト数を PG_RETURN_INT32 で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: latin2mic（mb/pg_wchar.h で定義された変換ユーティリティ）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQL の関数呼び出し規約で使用される引数構造体。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断される。
  • noError フラグが latin2mic に渡され、変換エラー時の挙動が制御される。

保守担当者向けメモ

  • 本関数は latin2mic を利用しており、KOI8-R のキリル文字表現が MULE_INTERNAL と同一であることを前提としている。
  • 変換ロジックの変更時は、latin2mic の仕様および LC_KOI8_R の定義との整合性に注意すること。
