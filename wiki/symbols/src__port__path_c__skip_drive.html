skip_drive

定義

  • 種別: function
  • 定義箇所: src/port/path.c:68

目的

ファイルパス文字列から、Windows 形式のドライブレターまたは UNC パスプレフィックスをスキップし、パスの本体部分を指すポインタを返します。

シグネチャまたは形状

char *skip_drive(const char *path);

入力と出力

  • **入力**: path (ヌル終端文字列へのポインタ)
  • **出力**: ドライブプレフィックスを除去した後のパスの先頭位置を指す char *。プレフィックスが存在しない場合は、入力されたポインタをそのまま返します。

前提条件と不変条件

  • path は有効なヌル終端文字列である必要があります。
  • 内部で IS_DIR_SEP マクロを使用しており、実行環境の OS に依存したパス区切り文字の判定が行われます。

アルゴリズムまたは振る舞い

1. **UNC パス判定**: パスの先頭が2つのディレクトリ区切り文字（例: \\）で始まる場合、UNC パスとみなし、最初のコンポーネント（サーバー名等）をスキップします。 2. **ドライブレター判定**: 先頭がアルファベットで、2文字目がコロン（:）である場合（例: C:）、ドライブレターとみなし、その2文字分をスキップします。 3. **戻り値**: 上記のいずれにも該当しない場合、入力されたポインタをそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/path.c 内の複数の箇所（canonicalize_path 関連の処理やパスの解析処理など）から呼び出されています。
  • **呼び出し先**: IS_DIR_SEP マクロおよび isalpha 関数（ctype.h）を呼び出します。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • path が空文字列の場合、そのまま空文字列を返します。
  • path がドライブレターや UNC プレフィックスを含まない場合、入力ポインタをそのまま返します。
  • 不正な形式のパス（例: C のみ、あるいは \ のみ）に対しては、プレフィックスとして認識せず、入力をそのまま返します。

保守担当者向けメモ

  • 本関数は Windows 環境でのパス操作を主眼としていますが、クロスプラットフォームなコードベースの一部であるため、Unix 系 OS 上では常に元のポインタを返す挙動となります。
  • src/port/path.c 内では、この関数を直接呼び出す代わりに、マクロ #define skip_drive(path) (path) が定義されている環境（Unix 等）もあるため、利用時にはプリプロセッサの定義に注意してください。
