pgaio_io_can_reopen

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_target.c:102

目的

指定された PgAioHandle が保持するI/Oターゲットにおいて、プロセス間移行時などにファイル記述子（FD）の再オープンが可能かどうかを判定します。

シグネチャまたは形状

bool pgaio_io_can_reopen(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh（I/Oハンドルのポインタ）
  • **出力**: bool 型。ターゲットが再オープン処理（reopen 関数ポインタ）を実装している場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • ioh->target は PGAIO_TID_INVALID より大きく、かつ PGAIO_TID_COUNT より小さい有効な範囲内である必要があります（Assert による検証）。

アルゴリズムまたは振る舞い

1. ioh->target を使用して、静的レジストリ pgaio_target_info から該当するターゲット情報を参照します。 2. 取得したターゲット情報の reopen フィールドが NULL でないかを確認します。 3. reopen 関数ポインタが定義されていれば true を、定義されていなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/method_worker.c:240
  • **呼び出し先**:
  • 直接的な関数呼び出しはありませんが、pgaio_target_info 配列の要素を参照します。

関連データ構造

  • PgAioHandle: I/O操作の状態を保持する構造体。
  • PgAioTargetInfo: 各I/Oターゲットの機能（reopen 等）を定義する内部構造体。

失敗モードと境界条件

  • ioh->target が無効な値である場合、Assert によりデバッグビルドで停止します。

保守担当者向けメモ

  • 本関数は、新しいI/Oターゲットを追加する際に、そのターゲットがプロセス間でのFD再構築をサポートするかどうかを制御するインターフェースです。
  • pgaio_target_info 配列の定義が変更された場合、本関数の判定結果も連動して変化します。
