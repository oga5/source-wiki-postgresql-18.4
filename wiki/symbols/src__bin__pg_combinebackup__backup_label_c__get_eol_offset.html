get_eol_offset

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/backup_label.c:200

目的

StringInfo バッファ内の現在のカーソル位置から、次の改行文字（\n）までのオフセットを特定し、行の終端位置を返します。

シグネチャまたは形状

static int get_eol_offset(StringInfo buf)

入力と出力

  • **入力**: StringInfo buf（解析対象のバッファ）
  • **出力**: int（改行文字の直後の位置、またはバッファの終端位置）

前提条件と不変条件

  • buf->cursor はバッファ内の有効な開始位置を指している必要があります。
  • バッファのデータは buf->data に格納されており、buf->len がバッファの総長を示している必要があります。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 buf->cursor を初期オフセット eo として保持します。 2. eo がバッファの長さ buf->len に達するまで、バッファ内を順次走査します。 3. buf->data[eo] が改行文字（\n）である場合、その直後の位置（eo + 1）を返します。 4. 改行文字が見つからないままバッファの終端に達した場合、現在の eo（バッファの終端位置）を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_combinebackup/backup_label.c 内の解析処理（parse_backup_label 等）から呼び出されています。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • StringInfo: PostgreSQLの動的文字列バッファ構造体。data（文字列データ）、len（現在の長さ）、cursor（現在の読み取り位置）を保持します。

失敗モードと境界条件

  • バッファが空の場合、buf->len を返します。
  • 改行文字が含まれない場合、バッファの終端位置（buf->len）を返します。
  • buf->cursor が buf->len と等しい場合、ループは実行されず buf->len が返されます。

保守担当者向けメモ

  • 本関数はバッファ内のポインタを直接操作する補助関数です。呼び出し側で buf->cursor を更新する際の基準点として利用されます。
  • StringInfo の内部構造に依存しているため、バッファのメモリレイアウトが変更された場合は注意が必要です。
