PathClauseUsage

定義

  • 種別: struct
  • 定義箇所: src/backend/optimizer/path/indxpath.c:62

目的

インデックスパス（IndexPath、BitmapAndPath、または BitmapOrPath）が使用する WHERE 句や部分インデックスの述語を追跡・分類するために使用されます。

シグネチャまたは形状

typedef struct {
    Path       *path;           /* 使用されるパス */
    List       *quals;          /* 使用される WHERE 句 */
    List       *preds;          /* 部分インデックスの述語 */
    Bitmapset  *clauseids;      /* quals と preds を表現するビットマップセット */
    bool        unclassifiable; /* 処理対象外の句が多すぎるかどうかのフラグ */
} PathClauseUsage;

入力と出力

  • 本構造体は classify_index_clause_usage 関数によって生成されます。
  • path フィールドには、インデックススキャンに関連するパスノードへのポインタが格納されます。

前提条件と不変条件

  • clauseids は、quals および preds に含まれる要素をビットマップ形式で保持します。
  • unclassifiable が true の場合、そのパスは複雑すぎて詳細な解析が困難であることを示します。

アルゴリズムまたは振る舞い

  • インデックスパスの生成過程において、どの条件（quals）や述語（preds）がそのパスに寄与しているかを記録します。
  • 複数のパスを比較・ソートする際に、各パスが使用している条件の重複や差異を判定するためのメタデータとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: classify_index_clause_usage 関数が本構造体を生成します。また、インデックスパスのソート処理（qsort）において、PathClauseUsage 型の配列が参照されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • Path: インデックススキャンの基底構造体。
  • List: quals や preds を保持するリスト。
  • Bitmapset: clauseids を構成するビットマップセット。

失敗モードと境界条件

  • unclassifiable フラグが true に設定された場合、そのパスに関する詳細な条件解析はスキップされます。

保守担当者向けメモ

  • 本構造体は、インデックスパスのコスト見積もりや最適化の過程で、条件の重複排除やパスの選択ロジックを支える重要な役割を担っています。
  • clauseids のビットマップ操作が正しく行われているかを確認する際は、classify_index_clause_usage の実装を参照してください。
