rte_visible_if_qualified

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:3911

目的

指定された RangeTblEntry (RTE) が、名前空間において「リレーションとしては可視だが、列としては可視ではない」状態であるかを判定する。

シグネチャまたは形状

static bool rte_visible_if_qualified(ParseState *pstate, RangeTblEntry *rte)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体へのポインタ。
  • rte: 判定対象の RangeTblEntry 構造体へのポインタ。
  • **出力**:
  • bool: 条件を満たす場合は true、それ以外（RTEが見つからない場合を含む）は false を返す。

前提条件と不変条件

  • pstate は有効な ParseState を指している必要がある。
  • rte は pstate の名前空間内に存在するか、あるいは findNSItemForRTE によって解決可能である必要がある。

アルゴリズムまたは振る舞い

1. findNSItemForRTE(pstate, rte) を呼び出し、対応する ParseNamespaceItem を取得する。 2. アイテムが見つからない場合は false を返す。 3. アイテムが見つかった場合、その nsitem->p_rel_visible が真であり、かつ nsitem->p_cols_visible が偽である場合にのみ true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_relation.c:3813 (エラーハンドリングまたは条件分岐の文脈)
  • **呼び出し先**:
  • findNSItemForRTE (同ファイル内)

関連データ構造

  • ParseState: 解析のコンテキストを保持する。
  • RangeTblEntry: クエリ内の各リレーションを表現する。
  • ParseNamespaceItem: 名前空間内での可視性フラグ（p_rel_visible, p_cols_visible）を保持する。

失敗モードと境界条件

  • findNSItemForRTE が NULL を返した場合、本関数は安全に false を返す。
  • p_rel_visible と p_cols_visible の両方が真、あるいは両方が偽である場合は false となる。

保守担当者向けメモ

  • 本関数は static であり、主に parse_relation.c 内での名前解決やエラーメッセージ生成の補助として使用される。
  • p_cols_visible が偽であることは、そのリレーションの列が直接参照できない（修飾名が必要な）状況を指す。
