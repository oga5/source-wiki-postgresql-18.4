GISTNodeBuffer

定義

  • 種別: struct
  • 定義箇所: src/include/access/gist_private.h:298

目的

GiSTインデックス構築時におけるバッファリング処理において、特定のインデックスブロックに対応する一時的なバッファ状態を保持するために使用されます。

シグネチャまたは形状

typedef struct
{
	BlockNumber nodeBlocknum;	/* index block # this buffer is for */
	int32		blocksCount;	/* current # of blocks occupied by buffer */

	BlockNumber pageBlocknum;	/* temporary file block # */
	GISTNodeBufferPage *pageBuffer; /* in-memory buffer page */

	bool		queuedForEmptying; /* is this buffer queued for emptying? */
	bool		isTemp;			/* is this a temporary copy, not in the hash table? */

	int			level;			/* 0 == leaf */
} GISTNodeBuffer;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。ただし、gistbuildbuffers.c 内のハッシュテーブル管理や、インデックス構築時のバッファリング処理において、この構造体へのポインタが状態の受け渡しに使用されます。

前提条件と不変条件

  • level フィールドは、インデックスツリー内の階層を示し、0 はリーフノードを指します。
  • isTemp が true の場合、そのインスタンスはハッシュテーブルに登録されていない一時的なコピーであることを示します。

アルゴリズムまたは振る舞い

GiSTインデックスのバッファリング構築アルゴリズムにおいて、大量の挿入を効率化するために利用されます。queuedForEmptying フラグは、バッファが空になる（ディスクへフラッシュされる）ためのキューに登録されているかどうかを追跡するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/access/gist/gistbuild.c: インデックス構築時のノードバッファ操作。
  • src/backend/access/gist/gistbuildbuffers.c: バッファのハッシュテーブル管理およびメモリ割り当て。
  • **呼び出し先**:
  • GISTNodeBufferPage 構造体（pageBuffer フィールドを通じて参照）。

関連データ構造

  • GISTNodeBufferPage: バッファ内のページデータを保持する構造体。
  • GISTBuildBuffers: インデックス構築時のバッファリング処理全体を管理する構造体。

失敗モードと境界条件

  • palloc によるメモリ割り当て失敗の可能性がある（gistbuildbuffers.c での palloc 使用箇所）。
  • ハッシュテーブルのサイズ制限や、一時ファイルへの書き出し失敗が境界条件となります。

保守担当者向けメモ

  • 本構造体は gistbuildbuffers.c で定義されるハッシュテーブルの entrysize として使用されています。変更を加える際は、ハッシュテーブルの整合性に注意してください。
  • isTemp フラグの管理状態が、メモリリークや二重解放を招かないよう注意深く扱う必要があります。
