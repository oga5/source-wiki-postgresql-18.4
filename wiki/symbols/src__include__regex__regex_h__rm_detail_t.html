rm_detail_t

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regex.h:170

目的

正規表現の実行時における詳細なマッチング情報（特に REG_EXPECT フラグに関連する情報）を保持するための構造体です。

シグネチャまたは形状

typedef struct
{
	pg_regmatch_t rm_extend;
} rm_detail_t;

入力と出力

  • **入力**: pg_regmatch_t 型のメンバ rm_extend を保持します。
  • **出力**: 正規表現実行関数（pg_regexec 等）において、マッチングの補足情報として呼び出し元へ返されます。

前提条件と不変条件

  • REG_EXPECT フラグが指定された実行時に使用されることを想定しています。

アルゴリズムまたは振る舞い

正規表現エンジンがマッチング処理を行う際、期待されるマッチング範囲や詳細な位置情報を rm_extend に格納し、呼び出し元に伝達する役割を担います。

呼び出し元と呼び出し先

  • **呼び出し元（直接的な参照の証拠）**:
  • src/backend/regex/regexec.c: 実行関数内でポインタとして使用。
  • src/include/regex/regex.h: 関数プロトタイプ（pg_regexec 等）の引数として使用。
  • src/test/modules/test_regex/test_regex.c: テストコード内での変数定義および引数としての使用。
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • pg_regmatch_t: rm_extend メンバの型であり、マッチした部分文字列の開始・終了位置を定義します。

失敗モードと境界条件

  • 構造体へのポインタが NULL の場合、正規表現エンジンが詳細情報を書き込もうとするとセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • REG_EXPECT フラグに関連する機能拡張を行う際は、本構造体のメンバ定義を確認してください。
  • テストコード src/test/modules/test_regex/test_regex.c が本構造体の利用例として参照可能です。
