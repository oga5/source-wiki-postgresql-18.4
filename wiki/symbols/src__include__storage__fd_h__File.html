File

定義

  • 種別: typedef
  • 定義箇所: src/include/storage/fd.h:51

目的

PostgreSQLの仮想ファイル記述子（VFD）を表現するための型です。OSのファイル記述子（FD）を直接扱うのではなく、本型を介して抽象化することで、プロセスごとのFD制限を回避し、LRUアルゴリズムによる動的なファイル管理を実現します。

シグネチャまたは形状

typedef int File;

入力と出力

  • 入力: 整数値（VFD識別子）。
  • 出力: なし（型定義）。

前提条件と不変条件

  • OSのファイル記述子（FD）とは直接互換性がありません。
  • PostgreSQL内部でファイル操作を行う際は、標準の open(2) や fopen(3) ではなく、本型を扱う PathNameOpenFile 等の専用インターフェースを使用する必要があります。

アルゴリズムまたは振る舞い

  • File 型の変数は、VFD管理システムによって管理される仮想的なハンドルとして機能します。
  • 物理的なファイルアクセス時には、LRUキャッシュを通じて実FDが割り当てられ、必要に応じて他のファイルのFDがクローズされることで、リソースの枯渇を防ぎます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/backup/basebackup_server.c や src/backend/backup/walsummary.c 等で変数型として使用されています。
  • 呼び出し先: 本型を引数として受け取る FileRead, FileWrite, FileClose 等の関数群が src/backend/storage/file/fd.c に実装されています。

関連データ構造

  • iovec（port/pg_iovec.h 経由）: FileRead/FileWrite でのデータ転送に使用されます。

失敗モードと境界条件

  • EMFILE エラー: BasicOpenFile 等で直接OSのFDを取得し、AcquireExternalFD による通知を怠った場合、VFD管理システムがFDの空き数を誤認し、システム制限に達するリスクがあります。

保守担当者向けメモ

  • 本型は抽象化レイヤーの核心であるため、直接的なOSコールを混在させないでください。
  • 一時ファイルとして使用する場合、トランザクション終了時に自動的にクローズ・削除されるライフサイクル管理が適用されます。
