t_starts_with

定義

  • 種別: function
  • 定義箇所: src/tutorial/funcs.c:110

目的

2つのテキスト型引数を受け取り、最初のテキストが2番目のテキストで始まるかどうかを判定するPostgreSQLのユーザー定義関数（UDF）です。

シグネチャまたは形状

Datum t_starts_with(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される2つの text 型引数。
  • **出力**: Datum 型として返される bool 値（PG_RETURN_BOOL マクロを使用）。

前提条件と不変条件

  • PostgreSQLのV1呼び出し規約に従う必要があります。
  • PG_GETARG_TEXT_PP を使用しているため、入力テキストはTOAST圧縮されている場合でも自動的に展開（またはポインタ参照）されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して、第1引数および第2引数を text * として取得します。 2. PG_GET_COLLATION を使用して、現在の照合順序（collation）を取得します。 3. DirectFunctionCall2Coll を呼び出し、組み込み関数 text_starts_with に引数と照合順序を渡して実行します。 4. 実行結果を DatumGetBool で論理値に変換し、PG_RETURN_BOOL を通じて呼び出し元へ返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（PG_FUNCTION_INFO_V1(t_starts_with) マクロにより登録）。
  • **呼び出し先**: 組み込み関数 text_starts_with。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型表現。
  • Oid: 照合順序を識別するためのオブジェクト識別子。

失敗モードと境界条件

  • 入力引数がNULLの場合、PG_GETARG_TEXT_PP の仕様に基づき、呼び出し元でNULLチェックが行われていない場合はエラーとなる可能性があります。
  • 内部で呼び出される text_starts_with がエラーを発生させた場合、その例外が伝播します。

保守担当者向けメモ

  • 本関数は text_starts_with へのラッパーです。ロジックの変更が必要な場合は、組み込み関数側の仕様を確認してください。
  • PG_GETARG_TEXT_PP を使用しているため、メモリ管理はPostgreSQLのメモリコンテキストに依存しており、手動のメモリ解放は不要です。
