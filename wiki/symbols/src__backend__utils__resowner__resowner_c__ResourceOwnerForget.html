ResourceOwnerForget

定義

  • 種別: function
  • 定義箇所: src/backend/utils/resowner/resowner.c:563

目的

指定されたリソースオーナーから、特定のリソース（value）の追跡を解除する。

シグネチャまたは形状

void ResourceOwnerForget(ResourceOwner owner, Datum value, const ResourceOwnerDesc *kind)

入力と出力

  • owner: 対象のリソースオーナー構造体へのポインタ。
  • value: 解除対象のリソースを識別する値（通常はポインタ）。
  • kind: リソースの種類を定義するディスクリプタ。
  • 出力: なし。対象が見つからない場合はエラーを発生させる。

前提条件と不変条件

  • リソースの解放処理（owner->releasing）が開始されていないこと。
  • リソースリストがソート済み（owner->sorted）ではないこと。

アルゴリズムまたは振る舞い

1. **解放状態の確認**: owner->releasing が真であればエラーを投げる。 2. **配列の検索**: まず固定長配列（owner->arr）を逆順に走査し、一致する項目を探す。見つかった場合、末尾の要素を当該位置に移動して配列サイズを減らし、正常終了する。 3. **ハッシュテーブルの検索**: 配列で見つからない場合、ハッシュテーブル（owner->hash）を線形探索する。一致する項目が見つかれば、該当スロットを空にしてハッシュカウントを減らし、正常終了する。 4. **エラー処理**: 上記のいずれでも見つからない場合、プログラマの誤用としてエラーを発生させる。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/openssl.c, src/backend/access/common/tupdesc.c, src/backend/jit/llvm/llvmjit.c, src/backend/storage/file/fd.c, src/backend/storage/ipc/dsm.c, src/backend/storage/ipc/waiteventset.c, src/backend/utils/cache/catcache.c, src/backend/utils/cache/plancache.c, src/backend/utils/cache/relcache.c など。
  • 呼び出し先: elog（エラー発生時）。

関連データ構造

  • ResourceOwnerData: リソースオーナーの内部状態を保持する構造体。
  • ResourceOwnerDesc: リソースの種類を識別するための記述子。

失敗モードと境界条件

  • **解放開始後の呼び出し**: owner->releasing が設定されている場合に呼び出すと ERROR となる。
  • **ソート済み状態での呼び出し**: owner->sorted が真の場合、アサーション違反となる。
  • **未所有リソースの解除**: 指定された value と kind の組み合わせがオーナー内に存在しない場合、ERROR となる。

保守担当者向けメモ

  • 本関数はリソースの追跡を解除するのみであり、リソース自体の解放（クローズやメモリ解放など）は行わない。
  • 配列検索において、見つかった要素を末尾の要素で上書きする手法をとっているため、順序は保持されない。
