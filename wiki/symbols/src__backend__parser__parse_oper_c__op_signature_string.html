op_signature_string

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_oper.c:601

目的

演算子名と引数の型情報を人間が読みやすい文字列形式（例: int4 + int4）に変換し、エラーメッセージやデバッグ出力用として提供する。

シグネチャまたは形状

const char *op_signature_string(List *op, Oid arg1, Oid arg2)

入力と出力

  • **入力**:
  • op: 演算子名を表す List（NameListToString で文字列化される）。
  • arg1: 左辺の型を表す Oid。
  • arg2: 右辺の型を表す Oid。
  • **出力**:
  • 演算子のシグネチャを含む palloc された文字列（const char *）。

前提条件と不変条件

  • arg1 が OidIsValid でない（無効な）場合、左辺の型は出力文字列に含まれない。
  • 戻り値は palloc されたメモリ領域であるため、呼び出し元で適切に管理する必要がある。

アルゴリズムまたは振る舞い

1. StringInfoData を初期化する。 2. arg1 が有効な場合、format_type_be(arg1) を使用して型名を文字列化し、バッファに追加する。 3. NameListToString(op) を呼び出し、演算子名をバッファに追加する。 4. format_type_be(arg2) を使用して右辺の型名を文字列化し、バッファに追加する。 5. 構築された文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_oper.c 内の演算子解決エラー発生箇所（例: 120, 474, 630, 638, 703, 816 行目）。
  • src/backend/commands/operatorcmds.c（422, 430 行目）。
  • **呼び出し先**:
  • initStringInfo, appendStringInfo, appendStringInfoString（lib/stringinfo.h）。
  • format_type_be（utils/format_type.h）。
  • NameListToString（parser/parse_oper.h）。

関連データ構造

  • List: 演算子名を保持するノードリスト。
  • Oid: PostgreSQL の型識別子。
  • StringInfoData: 文字列構築用の一時バッファ。

失敗モードと境界条件

  • arg1 が無効な Oid の場合、単項演算子のような形式で出力される。
  • メモリ不足時には palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数はエラーメッセージ生成に特化している。演算子解決のデバッグにおいて、どの型で不一致が起きているかを特定する重要な役割を果たす。
  • format_type_be を使用しているため、システムカタログの変更や型名の表示形式変更の影響を受ける。
