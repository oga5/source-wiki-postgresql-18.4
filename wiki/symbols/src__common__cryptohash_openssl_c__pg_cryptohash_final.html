pg_cryptohash_final

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash_openssl.c:254

目的

OpenSSLのEVP APIを使用してハッシュ計算を完了させ、結果をバッファに書き出します。

シグネチャまたは形状

int pg_cryptohash_final(pg_cryptohash_ctx *ctx, uint8 *dest, size_t len);

入力と出力

  • ctx: ハッシュ計算状態を保持する pg_cryptohash_ctx 構造体へのポインタ。
  • dest: 計算結果を格納するバッファへのポインタ。
  • len: dest バッファのサイズ（バイト単位）。
  • 戻り値: 成功時は 0、失敗時は -1 を返します。

前提条件と不変条件

  • ctx が NULL の場合、関数は即座に -1 を返します。
  • len は、選択されたハッシュアルゴリズム（MD5, SHA1, SHA2系列）の出力長以上である必要があります。不足している場合、ctx->error に PG_CRYPTOHASH_ERROR_DEST_LEN が設定され、-1 が返されます。

アルゴリズムまたは振る舞い

1. ctx->type に基づき、指定された len が各アルゴリズムの最小必要長を満たしているか検証します。 2. EVP_DigestFinal_ex を呼び出し、OpenSSLのコンテキストから最終的なハッシュ値を dest に書き出します。 3. OpenSSLの戻り値が 0 以下（失敗）の場合、ERR_get_error() でエラーコードを取得し、SSLerrmessage を通じて ctx->errreason にエラー詳細を格納します。その後、ctx->error に PG_CRYPTOHASH_ERROR_OPENSSL を設定し、-1 を返します。 4. 成功時は 0 を返します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/libpq/auth-scram.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/checksum_helper.c
  • src/common/cryptohash.c
  • 呼び出し先:
  • EVP_DigestFinal_ex (OpenSSL)
  • ERR_get_error (OpenSSL)
  • SSLerrmessage (本ファイル内で定義されたエラー変換関数)

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態を保持する構造体。OpenSSLの EVP_MD_CTX を内包します。

失敗モードと境界条件

  • ctx が NULL の場合。
  • len がアルゴリズムごとの必要長（MD5_DIGEST_LENGTH 等）に満たない場合。
  • OpenSSL内部での計算エラー（EVP_DigestFinal_ex が失敗した場合）。

保守担当者向けメモ

  • 本関数はOpenSSLがリンクされている環境でのみ有効です。
  • エラー発生時は ctx->error および ctx->errreason が更新されるため、呼び出し側はこれらを参照して詳細な診断が可能です。
