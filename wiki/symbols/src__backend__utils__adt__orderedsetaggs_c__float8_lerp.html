float8_lerp

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/orderedsetaggs.c:502

目的

2つの float8 値の間で、指定された割合（パーセンタイル）に基づいた線形補間（Linear Interpolation）を行う。

シグネチャまたは形状

static Datum float8_lerp(Datum lo, Datum hi, double pct)

入力と出力

  • **入力**:
  • lo: 補間範囲の下限値（float8 を保持する Datum）。
  • hi: 補間範囲の上限値（float8 を保持する Datum）。
  • pct: 補間係数（0.0 から 1.0 の範囲の double）。
  • **出力**:
  • 補間結果を Datum 型として返す。

前提条件と不変条件

  • 入力 lo および hi は、float8 型の Datum である必要がある。
  • 内部で DatumGetFloat8 および Float8GetDatum マクロを使用しており、型変換が適切に行われることを前提とする。

アルゴリズムまたは振る舞い

1. DatumGetFloat8 を使用して、lo と hi を double 型の loval および hival に変換する。 2. 線形補間の公式 loval + (pct * (hival - loval)) を計算する。 3. 計算結果を Float8GetDatum で Datum 型に変換して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/orderedsetaggs.c:615 (percentile_cont_final_common への引数として渡される)
  • src/backend/utils/adt/orderedsetaggs.c:1012
  • **呼び出し先**:
  • DatumGetFloat8 (マクロ)
  • Float8GetDatum (マクロ)

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。

失敗モードと境界条件

  • pct が 0.0 の場合は loval を、1.0 の場合は hival を返す。
  • 入力値が NaN や無限大の場合、標準的な浮動小数点演算のルールに従う。

保守担当者向けメモ

  • 本関数は percentile_cont 集約関数の最終計算ロジック（percentile_cont_final_common）において、連続的な値を補間するために利用される。
  • 類似の補間ロジックとして interval_lerp が存在するため、修正時は整合性に注意すること。
