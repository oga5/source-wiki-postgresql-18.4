Subscription

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/pg_subscription.h:109

目的

論理レプリケーションにおけるサブスクリプションの設定情報および接続情報を、バックエンドのメモリ上で保持するためのデータ構造です。

シグネチャまたは形状

typedef struct Subscription {
    Oid         oid;
    Oid         dbid;
    XLogRecPtr  skiplsn;
    char       *name;
    Oid         owner;
    bool        ownersuperuser;
    bool        enabled;
    bool        binary;
    char        stream;
    char        twophasestate;
    bool        disableonerr;
    bool        passwordrequired;
    bool        runasowner;
    bool        failover;
    char       *conninfo;
    char       *slotname;
    char       *synccommit;
    List       *publications;
    char       *origin;
} Subscription;

入力と出力

本構造体は、システムカタログ pg_subscription から読み取られた行データをメモリ上に展開したものです。直接的な関数入出力ではなく、レプリケーションワーカーやコマンド処理プロセスが参照する状態保持用オブジェクトとして機能します。

前提条件と不変条件

  • pg_subscription は共有カタログであるため、dbid によって対象データベースを識別します。
  • stream は LOGICALREP_STREAM_xxx 定数、twophasestate は LOGICALREP_TWOPHASE_STATE_xxx 定数に従う必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: GetSubscription() 等を通じてシステムキャッシュから情報を取得し、本構造体に割り当てます。 2. **メモリ管理**: palloc 等で確保された動的メモリ（name, conninfo 等）を含むため、不要になった際は FreeSubscription() 等による解放が必要です。 3. **レプリケーション制御**: レプリケーションワーカーは、この構造体の enabled フラグや conninfo を参照してパブリッシャーへの接続とデータ同期を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_subscription.c
  • src/backend/commands/subscriptioncmds.c
  • src/backend/replication/logical/launcher.c
  • src/backend/replication/logical/worker.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、特定の関数を呼び出すものではありません。

関連データ構造

  • FormData_pg_subscription: カタログ上の物理的な行データ構造。
  • List: publications メンバで使用されるリスト構造。
  • XLogRecPtr: skiplsn で使用されるログシーケンス番号。

失敗モードと境界条件

  • conninfo にパスワードが含まれる場合、システムビュー等での公開範囲に注意が必要です。
  • disableonerr が有効な場合、ワーカーエラー発生時に自動的に enabled が無効化される可能性があります。

保守担当者向けメモ

  • 新しいカラムを追加した際は、pg_subscription.c 内の構造体初期化ロジックおよび FreeSubscription 関数を更新してください。
  • 共有カタログであるため、データベース間での整合性に留意してください。
