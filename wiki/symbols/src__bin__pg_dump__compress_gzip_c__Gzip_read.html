Gzip_read

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_gzip.c:263

目的

CompressFileHandle を介して GZIP 圧縮ファイルからデータを読み込み、解凍してバッファへ格納する。

シグネチャまたは形状

size_t Gzip_read(void *ptr, size_t size, CompressFileHandle *CFH)

入力と出力

  • **入力**:
  • ptr: 読み込んだデータを格納するバッファへのポインタ。
  • size: 読み込むバイト数。
  • CFH: 圧縮ファイルのハンドル情報を含む CompressFileHandle 構造体。
  • **出力**:
  • 実際に読み込まれたバイト数を size_t 型で返す。

前提条件と不変条件

  • CFH->private_data は有効な gzFile ハンドルとしてキャスト可能であること。
  • size が 0 の場合、本関数は何もせず 0 を返す。

アルゴリズムまたは振る舞い

1. CFH->private_data から gzFile ハンドルを取得する。 2. gzread を呼び出し、データを読み込む。 3. gzread の戻り値が 0 以下の場合、gzeof を確認する。EOF に到達している場合は 0 を返す。 4. EOF 以外でエラーが発生した場合は、gzerror を用いてエラー情報を取得し、pg_fatal を呼び出してプロセスを終了させる。 5. 正常時は読み込んだバイト数を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/compress_gzip.c 内で CFH->read_func に割り当てられている。具体的な呼び出し元コードの詳細は提供されていない。
  • **呼び出し先**: gzread, gzeof, gzerror, pg_fatal (すべて zlib または pg_backup_utils.h 由来)。

関連データ構造

  • CompressFileHandle: 圧縮ファイルの抽象化インターフェース。
  • gzFile: zlib が提供する GZIP ファイルハンドル。

失敗モードと境界条件

  • size == 0: 処理を行わず 0 を返す。
  • gzread がエラーを返した場合: gzerror で詳細を取得し、pg_fatal により致命的エラーとして報告する。
  • Z_ERRNO の場合: strerror(errno) を用いてシステムエラーメッセージを出力する。

保守担当者向けメモ

  • gzgetc マクロは構造体レイアウトの不一致を避けるため無効化されている。
  • gzread は EOF とエラーを区別する必要があるため、gzeof による判定が必須である。
