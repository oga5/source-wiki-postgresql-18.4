SH_DISTANCE_FROM_OPTIMAL

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/lib/simplehash.h:385

目的

Robin Hoodハッシュ法において、現在のバケット位置がその要素の本来の最適位置（ハッシュ値から算出される位置）からどれだけ離れているか（距離）を計算します。

シグネチャまたは形状

static inline uint32 SH_DISTANCE_FROM_OPTIMAL(SH_TYPE * tb, uint32 optimal, uint32 bucket)

入力と出力

  • **入力**:
  • tb: ハッシュテーブル構造体へのポインタ。
  • optimal: 要素が本来配置されるべき最適バケットインデックス。
  • bucket: 現在のバケットインデックス。
  • **出力**:
  • uint32: 最適位置から現在の位置までの距離。テーブルが循環していることを考慮した値が返されます。

前提条件と不変条件

  • tb->size はハッシュテーブルの現在のサイズ（バケット数）を保持している必要があります。
  • optimal および bucket は、有効なバケットインデックスの範囲内である必要があります。

アルゴリズムまたは振る舞い

1. optimal <= bucket の場合、単純な差分 bucket - optimal を返します。 2. optimal > bucket の場合、テーブルの末尾を跨いで配置されていると判断し、(tb->size + bucket) - optimal を計算して循環距離を算出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/lib/simplehash.h 内のハッシュテーブル操作ロジック（行681、1103付近）から、要素の配置状態を確認するために呼び出されます。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • SH_TYPE: 生成されるハッシュテーブルの構造体。size メンバを介してテーブルサイズを参照します。

失敗モードと境界条件

  • tb が NULL の場合、または tb->size が正しく初期化されていない場合は未定義の動作となります。
  • インデックスがテーブルサイズを超過している場合、誤った距離が算出されます。

保守担当者向けメモ

  • 本関数はマクロ展開によって生成されるコードの一部であり、SH_DISTANCE_FROM_OPTIMAL という名前は SH_MAKE_NAME(distance) によって動的に生成されます。
  • Robin Hoodハッシュの「要素の移動（シフト）」ロジックにおいて、衝突の深刻度を判定するための重要な指標となります。
