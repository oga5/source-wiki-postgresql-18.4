push_assignment

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:20

目的

SQL記述子操作（GET/SET DESCRIPTOR）における代入項目を一時的なスタックリストに蓄積し、後のコード生成フェーズで利用可能にする。

シグネチャまたは形状

void push_assignment(const char *var, enum ECPGdtype value)

入力と出力

  • **入力**:
  • var: 代入対象となる変数名（文字列）。
  • value: 代入される値の型を示す ECPGdtype 列挙型。
  • **出力**: なし（副作用としてグローバルな assignments リストを更新）。

前提条件と不変条件

  • mm_alloc および mm_strdup によるメモリ確保が成功すること。
  • 蓄積されたリストは、コード生成完了後に drop_assignments を呼び出して解放される必要がある（メモリリーク防止のため）。

アルゴリズムまたは振る舞い

1. struct assignment 型の新しいノードをメモリ上に確保する。 2. 新しいノードの variable フィールドに var を複製して格納する。 3. 新しいノードの value フィールドに value を格納する。 4. 新しいノードの next ポインタを現在の assignments リストの先頭に設定する。 5. グローバル変数 assignments を新しいノードで更新し、スタックの先頭を差し替える。

呼び出し元と呼び出し先

  • **呼び出し元**: 構文解析（パーサー）処理中に呼び出される。具体的な呼び出し元は提供されたソース内では明示されていない。
  • **呼び出し先**: mm_alloc（メモリ確保）、mm_strdup（文字列複製）。

関連データ構造

  • struct assignment: 記述子操作の項目を保持する連結リスト構造体。
  • assignments: push_assignment が操作するグローバルなリストのヘッドポインタ。

失敗モードと境界条件

  • メモリ不足時には mm_alloc 等がエラーを発生させる可能性がある。
  • 記述子操作の文脈外で呼び出された場合、リストの整合性が損なわれる可能性がある。

保守担当者向けメモ

  • 本関数は assignments リストをスタックとして操作する。リストのライフサイクル管理は drop_assignments に依存しているため、本関数を呼び出すコードパスでは必ず対となる解放処理が保証されていることを確認すること。
