ReScanExprContext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execUtils.c:442

目的

式評価コンテキスト（ExprContext）をリセットし、再利用可能な状態に戻す。これには、登録済みのクリーンアップコールバックの実行と、タプル単位のメモリ領域の解放が含まれる。

シグネチャまたは形状

void ReScanExprContext(ExprContext *econtext)

入力と出力

  • **入力**: econtext (ExprContext *) - リセット対象の式評価コンテキスト。
  • **出力**: なし（副作用として econtext の状態が更新される）。

前提条件と不変条件

  • econtext は有効な ExprContext 構造体を指している必要がある。
  • econtext->ecxt_per_tuple_memory が初期化されていること。

アルゴリズムまたは振る舞い

1. ShutdownExprContext(econtext, true) を呼び出し、econtext に登録されたすべてのクリーンアップコールバックを実行する。 2. MemoryContextReset(econtext->ecxt_per_tuple_memory) を呼び出し、タプル単位のメモリコンテキストをリセットしてメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execAmi.c
  • src/backend/executor/nodeAgg.c
  • **呼び出し先**:
  • ShutdownExprContext
  • MemoryContextReset

関連データ構造

  • ExprContext: 式評価の状態とメモリコンテキストを保持する構造体。

失敗モードと境界条件

  • econtext が NULL の場合、メモリ保護違反が発生する可能性がある。
  • 登録されたコールバック関数内でエラーが発生した場合、そのエラーが伝播する。

保守担当者向けメモ

  • ResetExprContext と異なり、本関数は登録されたコールバックの実行を伴うため、リソースのクリーンアップが必要な場合に選択すること。
  • nodeAgg.c 等で、集約状態のリセットや再スキャン時に利用されている。
