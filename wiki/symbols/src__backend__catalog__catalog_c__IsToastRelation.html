IsToastRelation

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/catalog.c:205

目的

指定されたリレーションがTOASTテーブルであるかを判定します。

シグネチャまたは形状

bool IsToastRelation(Relation relation)

入力と出力

  • **入力**: Relation relation（対象のリレーション記述子）
  • **出力**: bool（対象がTOASTテーブルであれば true、そうでなければ false）

前提条件と不変条件

  • 対象のリレーションが pg_toast 名前空間に属していることを判定基準とします。
  • 他のセッションの一時テーブルに属するTOASTテーブルに対しては false を返します。
  • ユーザーリレーションを pg_toast 名前空間に作成したり、移動させたりすることは他の箇所で制限されているため、この判定は一意に機能します。

アルゴリズムまたは振る舞い

1. RelationGetNamespace(relation) を呼び出し、対象リレーションの名前空間OIDを取得します。 2. 取得したOIDを IsToastNamespace に渡し、その結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam.c
  • src/backend/replication/logical/reorderbuffer.c
  • src/backend/utils/cache/inval.c
  • **呼び出し先**:
  • IsToastNamespace (同ファイル内)
  • RelationGetNamespace (ユーティリティ関数)

関連データ構造

  • Relation: リレーション記述子。

失敗モードと境界条件

  • 入力 relation が無効なポインタである場合、RelationGetNamespace の挙動に依存します。
  • 一時テーブルのTOASTテーブルについては、本関数では判定対象外となります。

保守担当者向けメモ

  • 本関数はカタログアクセスを行わない設計となっており、ロック取得中などのデッドロックを回避すべき文脈でも安全に使用可能です。
  • 判定ロジックを変更する場合は、pg_toast 名前空間の定義および管理ルールとの整合性に注意してください。
