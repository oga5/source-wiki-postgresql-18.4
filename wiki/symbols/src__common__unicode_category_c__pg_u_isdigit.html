pg_u_isdigit

定義

  • 種別: function
  • 定義箇所: src/common/unicode_category.c:210

目的

指定された Unicode コードポイントが数字であるかを判定します。POSIX 互換の動作と、Unicode 規格に基づく動作を切り替えて判定可能です。

シグネチャまたは形状

bool pg_u_isdigit(pg_wchar code, bool posix)

入力と出力

  • **入力**:
  • code: 判定対象の Unicode コードポイント（pg_wchar 型）。
  • posix: true の場合、ASCII 数字（'0'-'9'）のみを対象とする POSIX 互換モードで動作します。false の場合、Unicode の一般カテゴリを参照します。
  • **出力**:
  • bool: 数字であれば true、そうでなければ false を返します。

前提条件と不変条件

  • posix が true の場合、入力 code が ASCII 範囲外であっても、'0' から '9' の範囲内にあるかのみを評価します。
  • posix が false の場合、Unicode 規格の PG_U_DECIMAL_NUMBER カテゴリに該当するかを判定します。

アルゴリズムまたは振る舞い

1. posix 引数が true の場合、'0' <= code && code <= '9' の評価結果を返します。 2. posix 引数が false の場合、内部関数 unicode_category(code) を呼び出し、その戻り値が PG_U_DECIMAL_NUMBER と一致するかを判定して返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regc_pg_locale.c:307
  • src/common/unicode/category_test.c:102
  • src/common/unicode_category.c:228 (内部の pg_u_isalnum からの参照)
  • **呼び出し先**:
  • unicode_category() (ソースコードより直接的な呼び出しを確認)

関連データ構造

  • pg_wchar: ワイド文字コードポイントを表す型。
  • PG_U_DECIMAL_NUMBER: Unicode 一般カテゴリを示す定数。

失敗モードと境界条件

  • pg_wchar の値が有効な Unicode コードポイント範囲外である場合の挙動については、本ソース抜粋からは判断できません。

保守担当者向けメモ

  • 本関数は src/include/common/unicode_category.h で宣言されており、フロントエンドおよびバックエンドの両方から利用可能です。
  • Unicode 規格の安定性ポリシーにより、カテゴリ判定ロジックは固定されています。変更が必要な場合は unicode_category_table.h の生成スクリプトを確認してください。
