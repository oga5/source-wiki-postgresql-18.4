check_eme_pkcs1_v15

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/pgp-pubdec.c:41

目的

PKCS#1 v1.5 パディング形式でエンコードされたデータからパディングを除去し、ペイロード部分へのポインタを返します。

シグネチャまたは形状

static uint8 *check_eme_pkcs1_v15(uint8 *data, int len)

入力と出力

  • **入力**:
  • data: 復号対象のバイト配列へのポインタ。
  • len: data の長さ。
  • **出力**:
  • 成功時: パディング後のペイロード開始位置を指す uint8*。
  • 失敗時: NULL。

前提条件と不変条件

  • len は最低でも 10 バイト（ブロックタイプ 1 バイト + 最小パディング 8 バイト + 区切り 1 バイト）必要です。
  • データの先頭バイトは 0x02 である必要があります。
  • パディングの後に 0x00 の区切り文字が存在する必要があります。

アルゴリズムまたは振る舞い

1. 長さが 10 バイト未満の場合は NULL を返します。 2. 先頭バイトが 0x02 でない場合は NULL を返します。 3. 0x00 に到達するまでバイトを読み進め、パディング長（rnd）をカウントします。 4. 0x00 が見つからない、またはパディング長が 8 バイト未満の場合は NULL を返します。 5. 0x00 の直後のアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pgcrypto/pgp-pubdec.c 内の復号処理フロー（pgp_parse_pubenc_sesskey 経由の呼び出しが確認されています）。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • uint8: バイト単位のデータ操作に使用される型。

失敗モードと境界条件

  • **失敗モード**:
  • パディング形式が PKCS#1 v1.5 に準拠していない場合（先頭バイト不一致、区切り文字欠落、パディング長不足）。
  • 入力長が最小要件を満たさない場合。
  • **境界条件**:
  • パディング長がちょうど 8 バイトの場合が最小の有効ケースです。

保守担当者向けメモ

  • 本関数は PKCS#1 v1.5 パディングの検証のみを行います。復号後のデータに対するチェックサム検証は別途 control_cksum 等で行う必要があります。
  • 戻り値が NULL の場合、呼び出し元は PXE_PGP_WRONG_KEY 等のエラーとして処理することが一般的です。
