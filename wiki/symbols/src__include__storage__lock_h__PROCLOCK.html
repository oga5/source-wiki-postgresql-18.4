PROCLOCK

定義

  • 種別: struct
  • 定義箇所: src/include/storage/lock.h:370

目的

特定のバックエンドプロセスと、ロック対象（LOCK）との関連付けを保持する共有メモリ上のデータ構造です。プロセスがどのロックをどのモードで保持しているかを追跡します。

シグネチャまたは形状

typedef struct PROCLOCK
{
    PROCLOCKTAG tag;        /* 一意な識別子 */
    PGPROC     *groupLeader; /* プロセスのロックグループリーダー、またはプロセス自身 */
    LOCKMASK    holdMask;    /* 現在保持しているロックモードのビットマスク */
    LOCKMASK    releaseMask; /* 解放予定のロックモードのビットマスク */
    dlist_node  lockLink;    /* LOCK構造体内のリストへのリンク */
    dlist_node  procLink;    /* PGPROC構造体内のリストへのリンク */
} PROCLOCK;

入力と出力

  • 入力: PROCLOCKTAG により一意に識別されます。
  • 出力: holdMask および releaseMask を通じて、プロセスが保持するロック状態を外部（ロックマネージャ）へ提供します。

前提条件と不変条件

  • 共有メモリ上のハッシュテーブル（LockMethodProcLockHash）で管理されます。
  • lockLink は対応する LOCK オブジェクトのリストに、procLink は対応する PGPROC オブジェクトのリストにそれぞれリンクされます。

アルゴリズムまたは振る舞い

  • ロック取得時（LockAcquire）に、プロセスとロック対象を紐付けるために生成されます。
  • holdMask はプロセスが現在保持しているロックモードを累積し、releaseMask は解放処理の過程で一時的に使用されます。
  • ロック解放時（LockRelease）に、holdMask が更新され、必要に応じてハッシュテーブルから削除されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/lmgr/lock.c 内の FastPathGetRelationLockEntry、SetupLockInTable、CleanUpLock などが本構造体を操作します。
  • 呼び出し先: dlist API を使用して、LOCK および PGPROC のリスト操作を行います。

関連データ構造

  • PROCLOCKTAG: 本構造体を一意に識別するタグ。
  • LOCK: ロック対象の共有メモリ上の表現。
  • PGPROC: ロックを保持するプロセス。
  • LOCKMASK: ロックモードを保持するビットマスク。

失敗モードと境界条件

  • 共有メモリ不足により、ハッシュテーブルへの挿入が失敗する可能性があります。
  • 複数のプロセスが同一の PROCLOCK を参照することはありません（プロセスとロック対象のペアごとに一意であるため）。

保守担当者向けメモ

  • PROCLOCK は LOCK と PGPROC の両方のリストにリンクされているため、削除時には両方のリストから適切にデタッチする必要があります。
  • lockLink および procLink の操作には dlist API を使用してください。
  • PROCLOCK のハッシュテーブルは ShmemInitHash で初期化されます。
