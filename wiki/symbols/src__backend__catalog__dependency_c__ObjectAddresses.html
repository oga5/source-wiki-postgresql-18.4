ObjectAddresses

定義

  • 種別: struct
  • 定義箇所: src/backend/catalog/dependency.c:113

目的

削除対象となるデータベースオブジェクトのリストを保持し、カスケード削除処理におけるオブジェクトの蓄積および管理を行うための拡張可能なリスト構造体です。

シグネチャまたは形状

struct ObjectAddresses
{
    ObjectAddress *refs;        /* pallocされた配列 */
    ObjectAddressExtra *extras; /* pallocされた配列、または未使用時はNULL */
    int         numrefs;        /* 現在の参照数 */
    int         maxrefs;        /* 配列の現在の容量 */
};

入力と出力

  • 本構造体は、findDependentObjects 等の関数において、削除対象のオブジェクト群を蓄積するコンテナとして使用されます。
  • refs および extras は動的にメモリ確保（palloc）される配列です。

前提条件と不変条件

  • numrefs は常に maxrefs 以下である必要があります。
  • extras フィールドは、追加のオブジェクト状態が必要ない場合には NULL に設定可能です。
  • 依存関係の探索中、循環参照を避けるために本構造体を用いたリスト管理が行われます。

アルゴリズムまたは振る舞い

  • 削除対象のオブジェクトをリストに順次追加し、必要に応じて配列の容量を拡張します。
  • 削除実行時には、このリストに基づき依存関係の順序を考慮した処理が行われます。
  • eliminate_duplicate_dependencies 関数により、リスト内の重複依存関係を排除する処理が適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: performDeletion、performMultipleDeletions、findDependentObjects 等の依存関係管理関数から参照・操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ObjectAddress: 個々のオブジェクトを識別する構造体。
  • ObjectAddressExtra: 各オブジェクトの削除処理に関連するフラグや状態を保持する補助構造体。
  • ObjectAddressStack: 再帰的な依存関係探索において、循環参照を検出するために使用されるスレッド状のリスト構造。

失敗モードと境界条件

  • メモリ不足時: palloc による配列拡張時にエラーが発生する可能性があります。
  • スタックオーバーフロー: 依存関係が極めて深い場合、再帰探索時に check_stack_depth() による制限を受ける可能性があります。

保守担当者向けメモ

  • 本構造体は src/include/catalog/dependency.h でも定義されており、モジュール間で共有されています。
  • 新しいオブジェクトタイプを追加する際は、本構造体で管理されるリストが正しく処理されるよう、doDeletion 内のロジックを確認してください。
