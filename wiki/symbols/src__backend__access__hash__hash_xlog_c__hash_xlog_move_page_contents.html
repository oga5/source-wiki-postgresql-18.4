hash_xlog_move_page_contents

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash_xlog.c:500

目的

ハッシュインデックスのページ間でタプルを移動させる操作（ページ内容の移動）をWALリプレイ時に再適用する。

シグネチャまたは形状

static void hash_xlog_move_page_contents(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）
  • **出力**: なし（副作用としてバッファの状態を更新し、ディスクへの書き込み準備を行う）

前提条件と不変条件

  • リプレイ操作中、プライマリバケットページに対してクリーンアップロックを保持する必要がある（スキャンとの競合を防ぐため）。
  • 挿入されるタプル数は、WALレコード内のメタデータ ntups と一致しなければならない。

アルゴリズムまたは振る舞い

1. **ロック取得**: is_prim_bucket_same_wrt の値に応じて、プライマリバケットページおよび書き込み先バッファのクリーンアップロックを取得する。 2. **書き込み処理**:

  • WALレコードから挿入対象のタプルデータを読み込む。
  • PageAddItem を使用して、書き込み先ページにタプルを順次追加する。
  • 処理完了後、ページにLSNをセットし、バッファをダーティにする。

    • 3. **削除処理**:

  • WALレコードから削除対象のオフセット情報を読み込む。
  • PageIndexMultiDelete を使用して、対象ページからタプルを一括削除する。
  • 処理完了後、ページにLSNをセットし、バッファをダーティにする。

    • 4. **クリーンアップ**: 取得したすべてのバッファのロックを解放し、バッファを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hash_xlog.c:1095（hash_redo 関数内でのディスパッチ）
  • **呼び出し先**:
  • XLogReadBufferForRedoExtended, XLogReadBufferForRedo
  • XLogRecGetData, XLogRecGetBlockData
  • PageAddItem, PageIndexMultiDelete
  • PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer

関連データ構造

  • xl_hash_move_page_contents: WALレコードのデータ構造。
  • Buffer: PostgreSQLのバッファ管理構造体。
  • XLogReaderState: WAL読み込み状態を保持する構造体。

失敗モードと境界条件

  • PageAddItem が InvalidOffsetNumber を返した場合、elog(ERROR) により処理を中断する。
  • 挿入されたタプル数が xldata->ntups と一致しない場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は冪等（べきとう）である必要がある。
  • 処理の簡略化のため、リプレイ操作の終了までプライマリバケットページのロックを保持し続ける設計となっている。
