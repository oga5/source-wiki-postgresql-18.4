SlruInternalDeleteSegment

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/slru.c:1502

目的

指定された SLRU セグメントファイルをファイルシステムから物理的に削除し、関連する保留中の同期リクエストを破棄する。

シグネチャまたは形状

static void SlruInternalDeleteSegment(SlruCtl ctl, int64 segno)

入力と出力

  • ctl (SlruCtl): 対象となる SLRU インスタンスの制御構造体。
  • segno (int64): 削除対象のセグメント番号。
  • 戻り値: なし。

前提条件と不変条件

  • ctl が有効な SlruCtl 構造体を指していること。
  • 呼び出し元は、対象セグメントが既に不要であることを確認済みであること。

アルゴリズムまたは振る舞い

1. **同期リクエストの破棄**: ctl->sync_handler が SYNC_HANDLER_NONE でない場合、対象セグメントに対する保留中の fsync リクエストを RegisterSyncRequest (引数 SYNC_FORGET_REQUEST) を用いて削除する。 2. **パス生成**: SlruFileName を使用して、対象セグメントのファイルパスを生成する。 3. **削除実行**: unlink() を呼び出し、ファイルを物理的に削除する。削除の試行は DEBUG2 レベルでログ出力される。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/slru.c 内の SlruDeleteSegment および関連するディレクトリ走査関数から呼び出される。
  • **呼び出し先**:
  • RegisterSyncRequest
  • SlruFileName
  • unlink (システムコール)

関連データ構造

  • SlruCtl: SLRU のディレクトリパスや同期ハンドラを保持する制御構造体。
  • FileTag: 同期リクエストを識別するためのタグ構造体。

失敗モードと境界条件

  • unlink が失敗した場合（権限不足やファイルが存在しない場合など）、本関数はエラーを抑制せず、呼び出し元に伝播する可能性がある。
  • ctl->sync_handler が正しく設定されていない場合、同期リクエストの破棄がスキップされる。

保守担当者向けメモ

  • 本関数はセグメントの物理削除のみを担当する。セグメントが現在使用中かどうかの論理的な判定は、呼び出し元（SlruDeleteSegment 等）で行う必要がある。
