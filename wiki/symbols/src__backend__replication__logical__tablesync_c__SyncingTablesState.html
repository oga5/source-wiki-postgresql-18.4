SyncingTablesState

定義

  • 種別: enum
  • 定義箇所: src/backend/replication/logical/tablesync.c:126

目的

論理レプリケーションにおけるテーブル同期状態のキャッシュの有効性を管理するための列挙型です。

シグネチャまたは形状

typedef enum
{
    SYNC_TABLE_STATE_NEEDS_REBUILD,
    SYNC_TABLE_STATE_REBUILD_STARTED,
    SYNC_TABLE_STATE_VALID,
} SyncingTablesState;

入力と出力

本シンボル自体は状態を示す定数定義であり、直接的な入力や出力はありません。

前提条件と不変条件

  • table_states_validity 変数の初期値として SYNC_TABLE_STATE_NEEDS_REBUILD が使用されます。
  • 同期状態の整合性を保つため、キャッシュの再構築が必要な場合は SYNC_TABLE_STATE_NEEDS_REBUILD から状態遷移を開始する必要があります。

アルゴリズムまたは振る舞い

  • SYNC_TABLE_STATE_NEEDS_REBUILD: キャッシュが古く、再構築が必要な状態。
  • SYNC_TABLE_STATE_REBUILD_STARTED: 再構築処理が開始された状態。
  • SYNC_TABLE_STATE_VALID: キャッシュが最新であり、利用可能な状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/tablesync.c 内の静的変数 table_states_validity の型として参照されています。
  • 呼び出し先: 本シンボルは列挙型定義であるため、呼び出し先はありません。

関連データ構造

  • table_states_validity: 本列挙型を保持する静的変数。

失敗モードと境界条件

  • 状態遷移の不整合が発生した場合、同期ワーカーが古いキャッシュを参照し続ける可能性があります。

保守担当者向けメモ

  • 本列挙型は、pg_subscription_rel カタログから読み込まれる同期状態のキャッシュ管理に特化しています。
  • 共有メモリ上の状態とキャッシュの同期が取れない場合は、table_states_validity を SYNC_TABLE_STATE_NEEDS_REBUILD にリセットすることで強制的な再構築が可能です。
