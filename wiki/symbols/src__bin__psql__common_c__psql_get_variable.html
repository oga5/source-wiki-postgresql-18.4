psql_get_variable

定義

  • 種別: function
  • 定義箇所: src/bin/psql/common.c:187

目的

psql の変数管理システムから指定された変数の値を取得し、指定された引用（クォート）形式に従って文字列をエスケープして返します。

シグネチャまたは形状

char *psql_get_variable(const char *varname, PsqlScanQuoteType quote, void *passthrough);

入力と出力

  • **入力**:
  • varname: 取得対象の変数名。
  • quote: 適用する引用形式（PQUOTE_PLAIN, PQUOTE_SQL_LITERAL, PQUOTE_SQL_IDENT, PQUOTE_SHELL_ARG）。
  • passthrough: 条件付きスタック（ConditionalStack）へのポインタ。
  • **出力**:
  • 成功時: 引用処理済みの変数値（呼び出し元で free() が必要）。
  • 失敗時: NULL。

前提条件と不変条件

  • passthrough が提供され、かつ conditional_active が偽（非アクティブな \if ブランチ内）の場合、変数は展開されず NULL を返します。
  • PQUOTE_SQL_LITERAL または PQUOTE_SQL_IDENT を指定する場合、pset.db が有効なデータベース接続を保持している必要があります。

アルゴリズムまたは振る舞い

1. GetVariable を用いて pset.vars から変数値を検索します。値が存在しない場合は NULL を返します。 2. quote 引数に基づき以下の処理を行います：

  • PQUOTE_PLAIN: 値をそのまま複製して返します。
  • PQUOTE_SQL_LITERAL / PQUOTE_SQL_IDENT: PQescapeLiteral または PQescapeIdentifier を使用してエスケープします。接続がない場合はエラーをログ出力し NULL を返します。
  • PQUOTE_SHELL_ARG: appendShellStringNoError を使用してシェル引数として安全な形式に変換します。改行や復帰文字が含まれる場合はエラーをログ出力し NULL を返します。

    • 3. 最終的に生成された文字列を呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/mainloop.c（スキャン処理の一部として呼び出される）。
  • **呼び出し先**: GetVariable, PQescapeLiteral, PQescapeIdentifier, PQerrorMessage, pg_strdup, PQfreemem, initPQExpBuffer, appendShellStringNoError, conditional_active。

関連データ構造

  • pset.vars: psql の変数格納構造体。
  • PsqlScanQuoteType: 引用形式を定義する列挙型。
  • PQExpBufferData: シェル引数生成時に使用されるバッファ。

失敗モードと境界条件

  • 変数が未定義の場合: NULL を返します。
  • データベース接続なしで SQL 引用を要求した場合: エラーをログ出力し NULL を返します。
  • エスケープ処理（libpq）が失敗した場合: エラーメッセージをログ出力し NULL を返します。
  • シェル引数に不正な文字（改行等）が含まれる場合: エラーをログ出力し NULL を返します。

保守担当者向けメモ

  • 本関数は libpq のエスケープ関数を使用するため、接続のクライアントエンコーディングに依存します。
  • PQUOTE_SHELL_ARG の処理では、appendShellStringNoError の結果を pg_strdup せずに直接 buf.data を返している点に注意してください。
