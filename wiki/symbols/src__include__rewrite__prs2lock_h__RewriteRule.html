RewriteRule

定義

  • 種別: struct
  • 定義箇所: src/include/rewrite/prs2lock.h:24

目的

PostgreSQLのルールシステムにおいて、リレーション（テーブルやビュー）に適用される個別の書き換えルールをメモリ上で保持するためのデータ構造です。

シグネチャまたは形状

typedef struct RewriteRule
{
	Oid			ruleId;
	CmdType		event;
	Node	   *qual;
	List	   *actions;
	char		enabled;
	bool		isInstead;
} RewriteRule;

入力と出力

本構造体は関数ではなくデータ保持のための構造体です。システムカタログ pg_rewrite から読み出されたルール情報がこの構造体にマッピングされ、リレーションキャッシュ（RelCache）の一部として保持されます。

前提条件と不変条件

  • actions フィールドは List 型であり、ルール実行時に適用されるクエリツリーのリストを保持します。
  • isInstead フラグは、当該ルールが INSTEAD ルールであるかどうかを示します。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持のみを行い、直接的な実行ロジックは持ちません。クエリ書き換え処理（rewriteHandler）において、クエリツリーの変換を行う際のメタデータとして参照されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/rewrite/rewriteHandler.c: クエリ書き換えロジック内でルールを評価・適用するために参照されます。
  • src/backend/utils/cache/relcache.c: リレーションキャッシュの構築時に RuleLock を介して本構造体が管理されます。
  • src/backend/rewrite/rewriteDefine.c および src/backend/commands/matview.c: ルールの定義や操作時に参照されます。
  • **呼び出し先**:
  • 本構造体はデータ構造であり、関数を呼び出すことはありません。

関連データ構造

  • RuleLock: RewriteRule の配列を管理するコンテナ構造体。
  • CmdType: ルールが適用されるイベントタイプ（SELECT, INSERT等）を定義する列挙型。
  • List: actions を保持するために使用される nodes/pg_list.h で定義されたリスト構造。

失敗モードと境界条件

  • ルール数が多いテーブルでは、RuleLock を介して保持される RewriteRule の数が増加するため、メモリ消費量に注意が必要です。

保守担当者向けメモ

  • 本構造体の名称にある「Lock」は、歴史的経緯によるものであり、マルチスレッドの排他制御を行うロック機構ではありません。
  • ルールシステムにおけるデータ保持の標準的な単位であるため、ルール定義の変更や拡張を行う際は本構造体のフィールド定義を確認してください。
