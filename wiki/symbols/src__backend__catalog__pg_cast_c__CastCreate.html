CastCreate

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_cast.c:48

目的

PostgreSQLのシステムカタログ pg_cast に新しい型変換（キャスト）ルールを登録し、関連する依存関係を記録します。

シグネチャまたは形状

ObjectAddress CastCreate(Oid sourcetypeid, Oid targettypeid,
                         Oid funcid, Oid incastid, Oid outcastid,
                         char castcontext, char castmethod, DependencyType behavior)

入力と出力

  • **入力**:
  • sourcetypeid, targettypeid: キャスト元およびキャスト先の型OID。
  • funcid: 変換に使用する関数OID（存在しない場合は InvalidOid）。
  • incastid, outcastid: 関数に関連する補助的なキャストOID。
  • castcontext, castmethod: キャストのコンテキストおよびメソッドを示す文字コード。
  • behavior: 依存関係の削除動作（DependencyType）。
  • **出力**:
  • 作成されたキャストオブジェクトの ObjectAddress を返します。

前提条件と不変条件

  • 呼び出し元で権限チェックおよび型チェックが完了している必要があります。
  • 同一のソース型とターゲット型を持つキャストが既に存在する場合、エラー（ERRCODE_DUPLICATE_OBJECT）が発生します。

アルゴリズムまたは振る舞い

1. pg_cast テーブルを RowExclusiveLock で開きます。 2. SearchSysCache2 を使用して、重複するキャスト定義がないか確認します。 3. GetNewOidWithIndex で新しいキャスト用OIDを取得し、heap_form_tuple でタプルを構築します。 4. CatalogTupleInsert でカタログへ挿入します。 5. ObjectAddresses を使用して、ソース型、ターゲット型、変換関数、および補助キャストへの依存関係を収集し、record_object_address_dependencies で記録します。 6. 現在の拡張（Extension）に対する依存関係を記録します。 7. InvokeObjectPostCreateHook を呼び出し、後処理としてテーブルをクローズして作成したオブジェクトのアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/functioncmds.c
  • src/backend/commands/typecmds.c
  • **呼び出し先**:
  • table_open, table_close, heap_form_tuple, CatalogTupleInsert
  • SearchSysCache2, GetNewOidWithIndex
  • new_object_addresses, add_exact_object_address, record_object_address_dependencies, free_object_addresses
  • recordDependencyOnCurrentExtension, InvokeObjectPostCreateHook

関連データ構造

  • pg_cast: キャスト定義を保持するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトを識別する構造体。
  • ObjectAddresses: 依存関係のリストを管理する構造体。

失敗モードと境界条件

  • 同一のソース・ターゲット型ペアが存在する場合、ereport(ERROR, ...) によりトランザクションが中断されます。
  • カタログ操作中にメモリ不足やロック競合が発生した場合、PostgreSQLの標準的なエラー処理に従い中断されます。

保守担当者向けメモ

  • 本関数は定義の追加のみを担います。実行時の型変換ロジックは src/backend/utils/adt/coerce.c 等を参照してください。
  • incastid および outcastid は、関数を用いたキャストにおいて、関数の引数や戻り値の型変換に依存関係がある場合にのみ使用されます。
