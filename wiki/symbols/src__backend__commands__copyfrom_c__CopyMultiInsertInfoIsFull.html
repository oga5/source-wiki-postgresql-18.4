CopyMultiInsertInfoIsFull

定義

  • 種別: static inline function
  • 定義箇所: src/backend/commands/copyfrom.c:424

目的

CopyMultiInsertInfo 構造体が管理するバルク挿入用バッファが、挿入可能な上限（タプル数またはバイト数）に達しているかを判定します。

シグネチャまたは形状

static inline bool CopyMultiInsertInfoIsFull(CopyMultiInsertInfo *miinfo)

入力と出力

  • **入力**: CopyMultiInsertInfo *miinfo（バルク挿入の状態を保持する構造体へのポインタ）
  • **出力**: bool 型。バッファが満杯であれば true、そうでなければ false を返します。

前提条件と不変条件

  • miinfo は初期化済みである必要があります。
  • 判定基準として MAX_BUFFERED_TUPLES（タプル数の上限）および MAX_BUFFERED_BYTES（バイト数の上限）が使用されます。

アルゴリズムまたは振る舞い

1. miinfo->bufferedTuples が MAX_BUFFERED_TUPLES 以上であるかを確認します。 2. または、miinfo->bufferedBytes が MAX_BUFFERED_BYTES 以上であるかを確認します。 3. いずれかの条件が真であれば true を返し、両方とも偽であれば false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/copyfrom.c 内の COPY 実行ループ（行番号 1389 付近）にて、バッファをフラッシュすべきタイミングの判断に使用されています。
  • **呼び出し先**: 本関数は他の関数を呼び出しません。

関連データ構造

  • CopyMultiInsertInfo: バルク挿入の状態を管理する構造体。本関数はこの構造体のメンバ変数を参照します。

失敗モードと境界条件

  • miinfo が NULL の場合、未定義の動作（セグメンテーション違反）を引き起こす可能性があります。
  • 閾値（MAX_BUFFERED_TUPLES, MAX_BUFFERED_BYTES）はコンパイル時に定義された定数に依存します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを考慮して頻繁に呼び出されることを想定しています。
  • バッファの閾値条件を変更する場合は、本関数のロジックと CopyMultiInsertBuffer の初期化ロジックの整合性に注意してください。
