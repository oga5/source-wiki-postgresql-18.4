StartupStatusEnum

定義

  • 種別: enum
  • 定義箇所: src/backend/postmaster/postmaster.c:272

目的

Postmaster プロセスにおけるスタートアッププロセスの現在の実行状態を追跡・管理するために使用されます。

シグネチャまたは形状

typedef enum
{
	STARTUP_NOT_RUNNING,
	STARTUP_RUNNING,
	STARTUP_SIGNALED,
	STARTUP_CRASHED,
} StartupStatusEnum;

入力と出力

  • 本列挙型は状態値として機能し、入力や出力の関数インターフェースは持ちません。

前提条件と不変条件

  • postmaster.c 内の静的変数 StartupStatus の型として使用されます。
  • 初期値は STARTUP_NOT_RUNNING です。

アルゴリズムまたは振る舞い

  • STARTUP_NOT_RUNNING: プロセスが起動していない状態。
  • STARTUP_RUNNING: プロセスが実行中の状態。
  • STARTUP_SIGNALED: プロセスに対して SIGQUIT または SIGKILL を送信済みの状態。
  • STARTUP_CRASHED: プロセスが異常終了した状態。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/postmaster/postmaster.c 内のスタートアッププロセス管理ロジック。
  • 呼び出し先: 情報なし。

関連データ構造

  • StartupStatus: postmaster.c で定義される static StartupStatusEnum 型の変数。

失敗モードと境界条件

  • 状態遷移の整合性は、Postmaster のメインループおよびシグナルハンドラによる StartupStatus 変数の更新に依存します。

保守担当者向けメモ

  • 本列挙型はスタートアッププロセスのライフサイクル管理に直結しています。状態遷移のロジックを変更する際は、postmaster.c 内のプロセス監視ループとの整合性に注意してください。
