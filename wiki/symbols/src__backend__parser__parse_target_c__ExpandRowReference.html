ExpandRowReference

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_target.c:1425

目的

複合型（行型）の式を個別のフィールド参照のリストに展開します。主に SELECT * や (table.*).* のような構文において、行全体を構成する各列を個別の式として抽出するために使用されます。

シグネチャまたは形状

List *ExpandRowReference(ParseState *pstate, Node *expr, bool make_target_entry)

入力と出力

  • **入力**:
  • pstate: 現在のパーサ状態。
  • expr: 展開対象の式（行型）。
  • make_target_entry: true の場合、結果を TargetEntry ノードのリストとして返し、false の場合は FieldSelect ノードのリストとして返します。
  • **出力**: 展開されたフィールド参照の List。

前提条件と不変条件

  • expr は複合型（RECORD 型を含む）として評価可能である必要があります。
  • tupleDesc が取得できない場合、関数は失敗します（Assert(tupleDesc) により検証）。

アルゴリズムまたは振る舞い

1. **特殊ケースの最適化**: expr が全行参照（Var で varattno が InvalidAttrNumber）である場合、ExpandSingleTable を呼び出して効率的に展開します。 2. **型情報の取得**: expr が RECORD 型の場合は expandRecordVariable を使用し、それ以外の場合は get_expr_result_tupdesc を使用して対象の TupleDesc を取得します。 3. **フィールド展開**: 取得した TupleDesc に基づき、各属性を走査します。

  • 削除済み属性（attisdropped）はスキップします。
  • 各フィールドに対して FieldSelect ノードを生成し、元の式を引数として設定します。

    • 4. **結果の構築**: make_target_entry が true の場合、各 FieldSelect を TargetEntry でラップし、列名を付与してリストに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_target.c 内の複数の箇所から呼び出されています（行展開のロジックの一部として）。
  • **呼び出し先**: ExpandSingleTable, expandRecordVariable, get_expr_result_tupdesc, makeTargetEntry, copyObject 等。

関連データ構造

  • ParseState: パーサのコンテキスト情報。
  • TupleDesc: 複合型の構造定義。
  • FieldSelect: 複合型から特定のフィールドを抽出するノード。
  • TargetEntry: ターゲットリストの各要素を保持するノード。

失敗モードと境界条件

  • expr が複合型でない場合、get_expr_result_tupdesc が失敗する可能性があります。
  • 非常に複雑な式に対して本関数を適用すると、copyObject による式の複製が繰り返されるため、パフォーマンス上の懸念が生じる場合があります。

保守担当者向けメモ

  • 本関数は、式が非自明な計算を含む場合に FieldSelect を多用するため、効率が低下する可能性があります。
  • RECORD 型の処理において expandRecordVariable に依存しているため、型解決ロジックの変更時には注意が必要です。
