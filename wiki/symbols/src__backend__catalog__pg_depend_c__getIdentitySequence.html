getIdentitySequence

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:1017

目的

指定されたリレーションと属性（列）に関連付けられた「アイデンティティシーケンス（Identity Sequence）」の OID を取得する。

シグネチャまたは形状

Oid getIdentitySequence(Relation rel, AttrNumber attnum, bool missing_ok)

入力と出力

  • **入力**:
  • rel: 対象のリレーション（Relation）。
  • attnum: 対象の属性番号（AttrNumber）。
  • missing_ok: シーケンスが見つからない場合にエラーを発生させるか（false）、InvalidOid を返すか（true）を制御するフラグ。
  • **出力**:
  • 該当するシーケンスの Oid。見つからない場合は InvalidOid（missing_ok が真の場合）。

前提条件と不変条件

  • 対象のリレーションがパーティションテーブルである場合、内部的に最上位の親テーブルまで遡り、属性名に基づいて属性番号を再解決する。
  • 依存関係の種類として DEPENDENCY_INTERNAL を使用して検索を行う。

アルゴリズムまたは振る舞い

1. 入力 rel がパーティションである場合、get_partition_ancestors を用いて最上位の親リレーションを特定し、属性名を用いて attnum を再解決する。 2. getOwnedSequences_internal を呼び出し、指定されたリレーションと属性に紐付く DEPENDENCY_INTERNAL 型の依存関係リストを取得する。 3. 取得したリストの長さを検証する：

  • リストが 1 より大きい場合、複数のシーケンスが所有されていると見なしエラー（elog(ERROR)）を投げる。
  • リストが空（NIL）の場合、missing_ok に応じてエラーを投げるか InvalidOid を返す。

    • 4. リストが単一であれば、そのシーケンスの OID を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c
  • src/backend/parser/parse_utilcmd.c
  • src/backend/rewrite/rewriteHandler.c
  • **呼び出し先**:
  • get_partition_ancestors
  • get_attname
  • get_attnum
  • getOwnedSequences_internal
  • list_free
  • llast_oid
  • linitial_oid
  • elog

関連データ構造

  • Relation: 操作対象のリレーション。
  • Oid: オブジェクト識別子。
  • List: 依存関係にあるシーケンスのリスト。

失敗モードと境界条件

  • **エラー**:
  • パーティションテーブルにおいて、親テーブルで属性名が解決できない場合。
  • 1つの属性に対して複数の所有シーケンスが存在する場合。
  • シーケンスが見つからず、かつ missing_ok が false の場合。

保守担当者向けメモ

  • 本関数は DEPENDENCY_INTERNAL を明示的に指定して検索を行うため、他の依存関係タイプ（DEPENDENCY_AUTO 等）との混同を避けること。
  • パーティションテーブルの処理において、親テーブルへの遡り処理が含まれているため、パーティション構造の変更時にはこのロジックの整合性に注意が必要である。
