json_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/json.c:151

目的

バイナリ形式の入力ストリームからJSONデータを受信し、妥当性を検証した上で text 型の Datum として返却します。

シグネチャまたは形状

Datum json_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して StringInfo 形式のバイナリバッファを受け取ります。
  • **出力**: 検証済みのJSON文字列を保持する text 型の Datum を返します。

前提条件と不変条件

  • 入力バッファは pq_getmsgtext を介してテキストとして抽出可能である必要があります。
  • データベースのエンコーディング（GetDatabaseEncoding()）に基づいた妥当性検証が行われます。

アルゴリズムまたは振る舞い

1. pq_getmsgtext を使用して、入力バッファから文字列データとバイト数を取得します。 2. makeJsonLexContextCstringLen を呼び出し、取得した文字列に対してJSON字句解析コンテキストを初期化します。 3. pg_parse_json_or_ereport を実行し、JSON構文の妥当性を検証します。不正な構文の場合はエラーを報告します。 4. 検証に成功した場合、cstring_to_text_with_len を使用して、入力されたバイト列から text 型のオブジェクトを生成して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの内部関数マネージャ（SQLインターフェース経由）。
  • **呼び出し先**:
  • pq_getmsgtext
  • makeJsonLexContextCstringLen
  • pg_parse_json_or_ereport
  • cstring_to_text_with_len
  • 注: 呼び出し元の具体的なSQL文脈については、提供されたソース内では明示されていません。

関連データ構造

  • StringInfo: 入力バイナリデータ保持用。
  • JsonLexContext: JSON解析用コンテキスト。

失敗モードと境界条件

  • 入力データがJSONとして不正な場合、pg_parse_json_or_ereport によりエラーが発生し、処理が中断されます。
  • メモリ不足や不正なエンコーディングが含まれる場合、PostgreSQLの標準的なエラー処理機構により例外がスローされます。

保守担当者向けメモ

  • 本関数は json 型の受信専用です。jsonb 型の受信処理とは実装が異なります。
  • pg_parse_json_or_ereport は nullSemAction を使用しており、JSONの構造解析のみを行い、特定のセマンティクス処理は行いません。
