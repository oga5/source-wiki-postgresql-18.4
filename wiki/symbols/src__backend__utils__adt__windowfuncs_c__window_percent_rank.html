window_percent_rank

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/windowfuncs.c:260

目的

SQL標準のウィンドウ関数 PERCENT_RANK() を実装し、現在の行の相対的な順位を計算します。

シグネチャまたは形状

Datum window_percent_rank(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介してウィンドウ関数実行コンテキストを受け取ります。
  • **出力**: float8 型の値を返します。これは、パーティション内での相対的な順位（0から1の範囲）を表します。

前提条件と不変条件

  • パーティション内の総行数（totalrows）が0より大きい必要があります（Assert により検証）。
  • rank_context を使用してパーティションごとの計算状態を保持します。

アルゴリズムまたは振る舞い

1. WinGetPartitionRowCount でパーティション内の総行数を取得します。 2. rank_up を呼び出し、現在の行が前の行と異なる順位（ピアではない）かどうかを判定します。 3. WinGetPartitionLocalMemory を介して rank_context を取得・更新します。順位が更新された場合、現在の行位置（0始まり）に1を加算した値を context->rank に格納します。 4. 総行数が1以下の場合は 0.0 を返します。 5. それ以外の場合、(rank - 1) / (totalrows - 1) の計算結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのウィンドウ関数実行エンジン（src/backend/executor/nodeWindowAgg.c 等から間接的に呼び出されます）。
  • **呼び出し先**:
  • WinGetPartitionRowCount: パーティションの行数取得。
  • rank_up: 順位更新の判定。
  • WinGetPartitionLocalMemory: パーティション固有のメモリ確保。
  • WinGetCurrentPosition: 現在の行位置の取得。
  • PG_RETURN_FLOAT8: 結果の返却。

関連データ構造

  • rank_context: パーティションごとの順位状態を保持する構造体。
  • WindowObject: ウィンドウ関数実行用のAPIハンドル。

失敗モードと境界条件

  • totalrows が 0 の場合、Assert により停止します。
  • totalrows が 1 の場合、計算を行わず 0.0 を返します。

保守担当者向けメモ

  • 本関数は rank_up を利用してピア判定を行っています。順位の計算ロジックを変更する場合は、rank_context の定義と rank_up の仕様を確認してください。
  • オプティマイザ向けに window_percent_rank_support 関数が定義されており、フレーム指定の最適化をサポートしています。
