ReservoirStateData

定義

  • 種別: struct
  • 定義箇所: src/include/utils/sampling.h:46

目的

リザーバーサンプリングアルゴリズム（VitterのアルゴリズムZなど）を実行するための状態管理構造体です。ストリームデータから一定の確率で要素を選択する際の計算状態を保持します。

シグネチャまたは形状

typedef struct
{
    double      W;                  /* 現在のサンプリング重み */
    pg_prng_state randstate;        /* 乱数生成器の状態 */
} ReservoirStateData;

入力と出力

  • **入力**: 構造体内の randstate は乱数生成のシードとして機能し、W はサンプリングの進行に伴い更新される浮動小数点値です。
  • **出力**: 本構造体はサンプリングアルゴリズムの内部状態を保持し、後続のサンプリング処理で参照・更新されます。

前提条件と不変条件

  • randstate は common/pg_prng.h で定義された pg_prng_state 型であり、使用前に適切に初期化されている必要があります。
  • W はサンプリングの進行に応じて確率的に変化する値であり、アルゴリズムの不変条件としてサンプリングの公平性を維持します。

アルゴリズムまたは振る舞い

本構造体は、ストリームデータからランダムにサンプルを選択する際の状態を保持します。具体的なサンプリングロジックは src/backend/utils/misc/sampling.c に実装されており、本構造体の W と randstate を用いて、各要素を選択するかどうかの判定を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: 以下のコンポーネントでインスタンスが宣言・利用されています。
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/commands/analyze.c
  • src/backend/utils/misc/sampling.c
  • **呼び出し先**: 乱数生成のために common/pg_prng.h の関数群を利用します。

関連データ構造

  • ReservoirState: ReservoirStateData * 型のエイリアス。
  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • 浮動小数点演算を含むため、極端なケースでの丸め誤差が発生する可能性があります。
  • 乱数生成器が適切に初期化されていない場合、サンプリングの偏りや予測可能な結果を招く恐れがあります。

保守担当者向けメモ

  • 本構造体は ANALYZE コマンドや外部データラッパー（FDW）による統計情報収集で利用されます。
  • 新規開発においては、古い anl_ 系APIではなく、本構造体および関連する ReservoirState APIを使用してください。
