main

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:54

目的

ECPGプリプロセッサが構造体へのポインタをホスト変数として扱う際の挙動を検証する回帰テストのエントリポイントです。

シグネチャまたは形状

int main()

入力と出力

  • 入力: なし（プログラム引数は受け取らない）
  • 出力: データベース操作の結果およびフェッチした構造体データを標準出力（stdout）へ出力。デバッグログを標準エラー出力（stderr）へ出力。

前提条件と不変条件

  • 実行環境において ecpg1_regression データベースへの接続が可能であること。
  • malloc によるメモリ確保が成功すること。
  • プリプロセッサにより生成された構造体定義（customer, customer2, customer3, customer4）がメモリレイアウトの期待値と一致していること。

アルゴリズムまたは振る舞い

1. **初期化**: malloc を使用して、テスト用構造体（custs1〜custs4）およびインジケータ変数（inds）のメモリをヒープ上に確保します。 2. **接続**: ECPGconnect を呼び出し、データベースに接続します。 3. **テーブル構築**: ECPGdo を介して customers テーブルを作成し、サンプルデータを挿入します。 4. **データフェッチ**: 複数の ECPGdo 呼び出しにより、SQLクエリを実行し、結果を確保した構造体ポインタへ直接フェッチします。この際、sizeof を用いてメモリサイズを明示的に指定します。 5. **検証**: フェッチしたデータを printf で出力し、期待値との比較を可能にします。 6. **終了**: 処理終了後、プロセスを終了します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）
  • 呼び出し先:
  • malloc: メモリ確保
  • ECPGdebug: デバッグログの有効化
  • ECPGconnect: データベース接続
  • ECPGdo: SQL文の実行
  • sqlprint: エラーハンドリング
  • printf: 結果の出力

関連データ構造

  • customer, customer2, customer3, customer4: テスト対象の構造体。
  • cust_ind: SQLのNULL値等を判定するためのインジケータ構造体。
  • sqlca: SQL通信領域（エラーハンドリングに使用）。

失敗モードと境界条件

  • sqlca.sqlcode < 0: SQL実行エラーが発生した場合、sqlprint() が呼び出されます。
  • sqlca.sqlcode == ECPG_NOT_FOUND: データが見つからない場合に発生します。
  • sqlca.sqlwarn[0] == 'W': SQL警告が発生した場合に sqlprint() が呼び出されます。

保守担当者向けメモ

  • 本ファイルはプリプロセッサの出力結果であるため、直接編集しないでください。
  • 構造体のメンバに varchar 型が含まれる場合、len と arr の構造がECPGの期待する形式と一致している必要があります。
