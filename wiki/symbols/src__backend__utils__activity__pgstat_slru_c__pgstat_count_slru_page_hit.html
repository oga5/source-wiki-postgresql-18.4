pgstat_count_slru_page_hit

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_slru.c:64

目的

SLRU（Simple Least Recently Used）キャッシュにおいてページヒットが発生した際に、該当するSLRU統計のヒットカウンタをインクリメントする。

シグネチャまたは形状

void pgstat_count_slru_page_hit(int slru_idx)

入力と出力

  • **入力**: slru_idx (int型) - 統計を更新する対象のSLRUを識別するインデックス。
  • **出力**: なし。

前提条件と不変条件

  • slru_idx は、有効なSLRUエントリを指すインデックスである必要がある。
  • 本関数は、各バックエンドが保持するローカルな統計カウンタ（pending_SLRUStats）を更新する。

アルゴリズムまたは振る舞い

1. get_slru_entry(slru_idx) を呼び出し、指定されたインデックスに対応するSLRU統計エントリを取得する。 2. 取得したエントリの blocks_hit メンバを 1 増加させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/slru.c:539
  • src/backend/access/transam/slru.c:627
  • **呼び出し先**:
  • get_slru_entry（内部ヘルパー関数）

関連データ構造

  • pending_SLRUStats: 各バックエンドが保持するローカルな統計カウンタ配列。
  • PgStatShared_SLRU: 共有メモリ上のSLRU統計構造体（本関数が更新するローカル統計の同期先）。

失敗モードと境界条件

  • slru_idx が不正な値（範囲外）である場合、get_slru_entry の内部実装に依存する（ソースコード上では明示的なエラーハンドリングは記述されていない）。

保守担当者向けメモ

  • 本関数はクリティカルセクション内で呼び出されることを想定しており、メモリ割り当てを行わない設計となっている。
  • 統計の反映は、後続の pgstat_slru_flush_cb による共有メモリへの同期タイミングで行われる。
