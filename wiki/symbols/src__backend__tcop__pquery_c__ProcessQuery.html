ProcessQuery

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/pquery.c:136

目的

単一の計画済みクエリ（PlannedStmt）を実行し、実行エンジンの初期化から結果の配送、リソースの解放までの一連のライフサイクルを完結させる。

シグネチャまたは形状

static void ProcessQuery(PlannedStmt *plan,
                         const char *sourceText,
                         ParamListInfo params,
                         QueryEnvironment *queryEnv,
                         DestReceiver *dest,
                         QueryCompletion *qc)

入力と出力

  • **入力**:
  • plan: 実行対象の計画済みステートメント。
  • sourceText: クエリのソース文字列。
  • params: クエリパラメータ。
  • queryEnv: クエリ実行環境。
  • dest: 結果の出力先（DestReceiver）。
  • qc: コマンド完了情報を格納する構造体（NULL許容）。
  • **出力**: なし（void）。qc が提供されている場合、実行結果のステータスが書き込まれる。

前提条件と不変条件

  • GetActiveSnapshot() により取得されたスナップショットが有効であること。
  • 実行終了時に ExecutorEnd および FreeQueryDesc が呼び出され、割り当てられたリソースが確実に解放されること。

アルゴリズムまたは振る舞い

1. **初期化**: CreateQueryDesc を呼び出し、実行に必要な QueryDesc オブジェクトを生成する。 2. **実行準備**: ExecutorStart を呼び出し、実行計画を準備する。 3. **実行**: ExecutorRun を呼び出し、前方スキャン方向（ForwardScanDirection）でクエリを完了まで実行する。 4. **完了通知**: qc が指定されている場合、queryDesc->operation に基づいてコマンドタグ（SELECT, INSERT, UPDATE, DELETE, MERGE等）と処理件数を設定する。 5. **終了処理**: ExecutorFinish でスキャンを閉じ、ExecutorEnd でリソースを解放し、最後に FreeQueryDesc で QueryDesc を破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/pquery.c 内の他の関数から呼び出されている（直接的な呼び出し元関数の特定は、提供されたソース抜粋からは不可）。
  • **呼び出し先**: CreateQueryDesc, ExecutorStart, ExecutorRun, SetQueryCompletion, ExecutorFinish, ExecutorEnd, FreeQueryDesc。

関連データ構造

  • PlannedStmt: 実行されるクエリ計画。
  • QueryDesc: クエリ実行状態を保持する構造体。
  • DestReceiver: 結果の配送先インターフェース。
  • QueryCompletion: コマンド完了ステータス。

失敗モードと境界条件

  • qc が NULL の場合、コマンド完了情報の構築はスキップされる。
  • queryDesc->operation が定義されたコマンドタイプ（SELECT, INSERT等）に該当しない場合、CMDTAG_UNKNOWN が設定される。

保守担当者向けメモ

  • 本関数は単一クエリの実行に特化している。複雑なポータル実行フロー（複数クエリやユーティリティコマンド）については PortalRun 等の関連関数を参照すること。
  • 実行中にエラーが発生した場合のクリーンアップは、呼び出し元で PG_TRY/CATCH ブロック等を用いて適切に管理する必要がある。
