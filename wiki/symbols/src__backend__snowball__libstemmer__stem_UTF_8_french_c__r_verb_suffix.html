r_verb_suffix

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_french.c:1031

目的

フランス語のステミング処理において、単語末尾の動詞活用接尾辞を識別し、必要に応じて削除する。

シグネチャまたは形状

static int r_verb_suffix(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体。現在の文字列バッファ、カーソル位置、領域情報を含む）
  • 出力: int（成功時に 1、失敗または適用対象外の場合に 0、エラー時に負の値を返す）

前提条件と不変条件

  • z->I[2] に格納された領域（R1またはR2相当）が、接尾辞削除の境界として設定されている必要がある。
  • 処理対象の文字列は SN_env 内のバッファに保持されている。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c が z->I[2] より前にある場合、即座に 0 を返す。 2. z->lb を z->I[2] に一時的に更新し、接尾辞の探索範囲を制限する。 3. find_among_b を使用し、a_6 配列（38個の接尾辞パターン）に対して後方マッチングを行う。 4. マッチした接尾辞の種類（among_var）に応じて以下の処理を行う：

  • case 1: r_R2 を呼び出し、領域内であることを確認した上で slice_del を実行。
  • case 2: slice_del を実行。
  • case 3: slice_del を実行後、直前の文字が 'e' であればそれも削除する。

    • 5. 最後に z->lb を復元し、1 を返して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: french_UTF_8_stem（直接の言及はないが、ファイル概要の制御フローより推測される）。
  • 呼び出し先: find_among_b, r_R2, slice_del。

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_6: 接尾辞マッチングに使用される among 構造体の配列。

失敗モードと境界条件

  • z->c < z->I[2]: 領域外のため処理を中断する。
  • find_among_b がマッチしない場合: 0 を返し、何も変更しない。
  • slice_del 等の内部関数が負の値を返した場合: その値をそのまま伝播させる。

保守担当者向けメモ

  • 本関数は Snowball コンパイラによって自動生成されたコードである。ロジックの変更が必要な場合は、直接編集せず、対応する Snowball ソース言語（.sbl）を修正すること。
  • z->I[2] は r_mark_regions で計算された領域境界を指す。デバッグ時はこの値が適切に設定されているか確認すること。
