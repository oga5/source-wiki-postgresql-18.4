gbt_int4le

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int4.c:42

目的

2つの int4（4バイト整数）値を比較し、左辺が右辺以下であるかどうかを判定します。

シグネチャまたは形状

bool gbt_int4le(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の int32 値へのポインタ。
  • b: 比較対象の int32 値へのポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: *a <= *b が真であれば true、そうでなければ false を返します。

前提条件と不変条件

  • a および b は、有効な int32 型の値へのポインタである必要があります。
  • 本関数は int4 型の比較ロジックとして提供されており、GiST インデックスの比較操作において完全な結果を返します。

アルゴリズムまたは振る舞い

1. ポインタ a および b を const int32 * にキャストします。 2. それぞれのポインタが指す値を参照し、C 言語の <= 演算子を用いて比較します。 3. 比較結果をブール値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_int4.c:85 において、gbtree_ninfo 構造体（tinfo）のメンバとして参照されています。
  • **呼び出し先**: 直接的な関数呼び出しはありません。

関連データ構造

  • int32: PostgreSQL で定義される 4 バイト整数型。
  • FmgrInfo: PostgreSQL の関数管理構造体。

失敗モードと境界条件

  • ポインタが NULL の場合、セグメンテーション違反が発生する可能性があります。
  • 整数値の範囲外（オーバーフロー等）については、C 言語の標準的な整数比較の挙動に従います。

保守担当者向けメモ

  • 本関数は btree_utils_num.c で定義された汎用的な GiST ユーティリティから、tinfo 構造体を介して呼び出されることを想定しています。
  • 比較ロジックは単純な値の比較であるため、変更が必要な場合は int4 型の定義変更と整合性を取る必要があります。
