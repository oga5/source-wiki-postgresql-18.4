begin_heap_rewrite

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/rewriteheap.c:233

目的

テーブル（ヒープ）の物理的な再構築処理を開始し、再構築に必要な状態管理構造体（RewriteState）を初期化する。

シグネチャまたは形状

RewriteState begin_heap_rewrite(Relation old_heap, Relation new_heap, TransactionId oldest_xmin, TransactionId freeze_xid, MultiXactId cutoff_multi);

入力と出力

  • **入力**:
  • old_heap, new_heap: 再構築対象の旧リレーションと、書き込み先の新リレーション。
  • oldest_xmin, freeze_xid, cutoff_multi: タプルの凍結および可視性判定に使用するトランザクションID情報。
  • **出力**:
  • RewriteState: 再構築のライフサイクル全体で利用される不透明な状態管理構造体。

前提条件と不変条件

  • 呼び出し元は、対象となるテーブルに対して AccessExclusiveLock を保持している必要がある。
  • new_heap は空である必要はないが、ロックされている必要がある。
  • 関数内で専用のメモリコンテキスト（rw_cxt）が作成され、すべての関連データはそのコンテキスト下に配置される。

アルゴリズムまたは振る舞い

1. 再構築専用のメモリコンテキストを作成し、以降のメモリ割り当てをそこに切り替える。 2. RewriteStateData 構造体を割り当て、引数で渡されたリレーション情報やトランザクションIDを格納する。 3. 新しいリレーションの現在のブロック数を取得し、バルク書き込み状態（rs_bulkstate）を初期化する。 4. 更新チェーン（ctid）の追跡および旧TIDと新TIDのマッピング管理用に、2つのハッシュテーブル（rs_unresolved_tups, rs_old_new_tid_map）を作成する。 5. 論理デコーディング用の初期化関数 logical_begin_heap_rewrite を呼び出す。 6. 初期化済みの state を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c:724
  • **呼び出し先**:
  • AllocSetContextCreate
  • palloc0

関連データ構造

  • RewriteStateData: 再構築中の状態を保持するメイン構造体。
  • TidHashKey, UnresolvedTupData, OldToNewMappingData: ハッシュテーブルで使用されるキーおよびエントリ構造体。

失敗モードと境界条件

  • メモリ不足: 大規模なテーブルの再構築において、更新チェーンの解決用ハッシュテーブルがメモリを大量に消費する可能性がある。
  • ロック不足: 呼び出し元が適切なロックを保持していない場合、競合が発生する可能性がある。

保守担当者向けメモ

  • 本関数で作成された RewriteState は、最終的に end_heap_rewrite で解放される必要がある。
  • 論理レプリケーションが有効な場合、マッピングファイルが生成されるため、チェックポイント時の整合性に注意すること。
