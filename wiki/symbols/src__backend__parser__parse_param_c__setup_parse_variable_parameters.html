setup_parse_variable_parameters

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_param.c:83

目的

クエリ解析中に型が未定であるパラメータ（$1, $2 など）を動的に解決するためのフック機構を ParseState に初期化・登録します。

シグネチャまたは形状

void setup_parse_variable_parameters(ParseState *pstate, Oid **paramTypes, int *numParams)

入力と出力

  • **入力**:
  • pstate: 解析状態を保持する ParseState 構造体へのポインタ。
  • paramTypes: パラメータの型情報を格納する Oid 配列へのポインタ。
  • numParams: パラメータ数を保持する整数へのポインタ。
  • **出力**: なし（副作用として pstate を更新）。

前提条件と不変条件

  • pstate は有効なメモリ領域を指している必要があります。
  • palloc を使用して VarParamState を割り当てるため、メモリコンテキストが適切に設定されている必要があります。

アルゴリズムまたは振る舞い

1. VarParamState 構造体用のメモリを palloc で確保します。 2. 引数で受け取った paramTypes および numParams を VarParamState に格納します。 3. pstate->p_ref_hook_state に作成した VarParamState を設定します。 4. pstate->p_paramref_hook に variable_paramref_hook を登録します。 5. pstate->p_coerce_param_hook に variable_coerce_param_hook を登録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/analyze.c:157
  • src/backend/parser/analyze.c:3151
  • **呼び出し先**:
  • palloc (メモリ確保)
  • variable_paramref_hook (登録のみ)
  • variable_coerce_param_hook (登録のみ)

関連データ構造

  • ParseState: 解析状態を保持する構造体。
  • VarParamState: 可変パラメータの型情報と数を保持する内部構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • pstate が NULL の場合、セグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は、解析中にパラメータの型が推論される動的なシナリオでのみ使用してください。固定型のパラメータには setup_parse_fixed_parameters を使用してください。
  • p_ref_hook_state に格納された VarParamState は、解析終了後の型検証フェーズでも参照されるため、解析終了まで生存している必要があります。
