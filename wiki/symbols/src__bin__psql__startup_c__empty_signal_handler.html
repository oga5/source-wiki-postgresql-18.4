empty_signal_handler

定義

  • 種別: function
  • 定義箇所: src/bin/psql/startup.c:114

目的

シグナルハンドラとして登録され、特定のシグナルを受信した際に何のアクションも起こさずに復帰することで、シグナルのデフォルト動作（プロセスの終了など）を抑制するために使用されます。

シグネチャまたは形状

static void empty_signal_handler(SIGNAL_ARGS)
  • SIGNAL_ARGS: シグナルハンドラに渡される標準的な引数セット（プラットフォーム依存）。

入力と出力

  • 入力: シグナル発生時にシステムから渡される引数。
  • 出力: なし（void）。

前提条件と不変条件

  • 本関数は pqsignal を介してシグナルハンドラとして登録されることを前提としています。
  • 関数内部で状態変更を行わないため、再入可能性（reentrancy）に関する懸念はありません。

アルゴリズムまたは振る舞い

1. 関数が呼び出される。 2. 処理を行わずに即座に復帰する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/startup.c 内の main 関数（またはその初期化プロセス）において、pqsignal(SIGCHLD, empty_signal_handler) および pqsignal(SIGALRM, empty_signal_handler) として登録されています。
  • 呼び出し先: なし。

関連データ構造

  • なし。

失敗モードと境界条件

  • 本関数自体は失敗しません。
  • シグナルハンドラとしての登録が失敗する場合、それは pqsignal 側の問題であり、本関数のロジックには依存しません。

保守担当者向けメモ

  • SIGCHLD や SIGALRM を無視または空のハンドラで捕捉する必要がある場合にのみ使用してください。
  • 他のシグナルハンドラと混同しないよう注意してください。
