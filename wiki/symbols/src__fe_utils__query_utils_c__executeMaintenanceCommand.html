executeMaintenanceCommand

定義

  • 種別: function
  • 定義箇所: src/fe_utils/query_utils.c:73

目的

PostgreSQLのメンテナンスコマンドをデータベースに対して実行し、割り込み（Ctrl-C）を適切に管理しつつ、実行結果の成否を呼び出し元へ返却します。

シグネチャまたは形状

bool executeMaintenanceCommand(PGconn *conn, const char *query, bool echo)

入力と出力

  • **入力**:
  • conn: データベース接続を保持する PGconn オブジェクト。
  • query: 実行するSQLコマンド文字列。
  • echo: 真の場合、実行前に標準出力へクエリを表示する。
  • **出力**:
  • bool: コマンドが正常に実行され、ステータスが PGRES_COMMAND_OK であれば true、それ以外は false を返す。

前提条件と不変条件

  • conn は有効な接続状態である必要があります。
  • 実行後、内部で生成された PGresult は必ず PQclear によって解放されます。

アルゴリズムまたは振る舞い

1. echo が真であれば、query を標準出力に書き出します。 2. SetCancelConn(conn) を呼び出し、コマンド実行中の割り込みを有効化します。 3. PQexec(conn, query) を実行し、結果を保持します。 4. ResetCancelConn() を呼び出し、割り込み設定をリセットします。 5. 結果オブジェクトが非NULLであり、かつステータスが PGRES_COMMAND_OK であるかを判定します。 6. PQclear で結果を解放し、判定結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/scripts/clusterdb.c:210
  • **呼び出し先**:
  • SetCancelConn (割り込みハンドラ設定)
  • PQexec (libpqによるクエリ実行)
  • ResetCancelConn (割り込みハンドラ解除)
  • PQclear (結果メモリの解放)

関連データ構造

  • PGconn: libpqの接続ハンドル。
  • PGresult: libpqのクエリ実行結果構造体。

失敗モードと境界条件

  • PQexec が失敗した場合（res が NULL またはステータスが不一致）、関数は false を返します。
  • 割り込みが発生した場合の挙動は SetCancelConn の実装に依存します。

保守担当者向けメモ

  • 本関数は executeQuery 等と異なり、失敗時にプログラムを終了（exit(1)）させません。呼び出し側で戻り値を確認し、必要に応じてエラーハンドリングを行う設計です。
  • 複雑なトランザクション制御が必要な場合は、本関数を使用せず直接 libpq を使用してください。
