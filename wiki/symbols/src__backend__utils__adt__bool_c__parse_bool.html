parse_bool

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:30

目的

文字列形式のブール値を解析し、内部的な bool 型へ変換します。

シグネチャまたは形状

bool parse_bool(const char *value, bool *result);

入力と出力

  • **入力**:
  • value: 解析対象の文字列（ヌル終端）。
  • result: 解析結果を格納するための bool 型へのポインタ。
  • **出力**:
  • 戻り値: 解析が成功した場合は true、失敗した場合は false を返します。
  • result 引数: 解析成功時に、対応するブール値が書き込まれます。

前提条件と不変条件

  • value は有効なヌル終端文字列である必要があります。
  • result は書き込み可能な有効なメモリ領域を指している必要があります。

アルゴリズムまたは振る舞い

1. strlen(value) を使用して文字列の長さを取得します。 2. 取得した長さと文字列ポインタを parse_bool_with_len に渡し、実際の解析処理を委譲します。 3. parse_bool_with_len の戻り値をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し先**: parse_bool_with_len
  • **呼び出し元（直接的な証拠）**:
  • contrib/test_decoding/test_decoding.c
  • src/backend/access/common/reloptions.c
  • src/backend/backup/basebackup.c
  • src/backend/commands/extension.c

関連データ構造

  • bool (C言語の論理型)

失敗モードと境界条件

  • 文字列が true, false, yes, no, on, off, 1, 0 といった認識可能な形式に一致しない場合、解析は失敗し false を返します。
  • value が NULL の場合の挙動は、strlen が呼び出されるため未定義（セグメンテーション違反の可能性）です。

保守担当者向けメモ

  • 本関数は parse_bool_with_len のラッパーです。解析ロジックの変更が必要な場合は、主に parse_bool_with_len を参照してください。
  • 設定ファイルや拡張機能のオプション解析など、PostgreSQL 全体で広く利用されているユーティリティ関数です。
