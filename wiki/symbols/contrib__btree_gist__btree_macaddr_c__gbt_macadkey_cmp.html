gbt_macadkey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_macaddr.c:60

目的

GiSTインデックス構築時における macKEY 構造体同士の比較を行い、ソート順序を決定する。

シグネチャまたは形状

static int gbt_macadkey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ（内部で macKEY にキャストされる）。
  • flinfo: 関数呼び出し情報（macaddr_cmp の実行に使用）。
  • **出力**:
  • int: a が b より小さい場合は負、等しい場合は 0、大きい場合は正の値を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体を経由して macKEY を参照可能であること。
  • macKEY 構造体内の lower および upper フィールドが有効な macaddr データとして初期化されていること。

アルゴリズムまたは振る舞い

1. a と b の macKEY 構造体から、それぞれ lower 境界値を抽出する。 2. macaddr_cmp を呼び出し、lower 境界値同士を比較する。 3. 比較結果が 0（等しい）でない場合、その結果を直ちに返す。 4. 比較結果が 0 の場合、同様に upper 境界値同士を macaddr_cmp で比較し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の静的解析によれば、contrib/btree_gist/btree_macaddr.c:85 で参照されている。
  • **呼び出し先**: macaddr_cmp（PostgreSQLの標準関数）。

関連データ構造

  • macKEY: lower と upper の境界値を持つ構造体。
  • Nsrt: ソート処理のために macKEY をラップする構造体。

失敗モードと境界条件

  • macaddr_cmp が内部でエラーを発生させた場合、呼び出し元へ伝播する。
  • 入力ポインタが NULL の場合の挙動は定義されていない。

保守担当者向けメモ

  • 本関数は macaddr 型のインデックスソート順を定義する重要なコンポーネントである。
  • macaddr_cmp を直接呼び出す設計であるため、PostgreSQLのコア側の macaddr 比較ロジックの変更が本関数に直接影響する。
