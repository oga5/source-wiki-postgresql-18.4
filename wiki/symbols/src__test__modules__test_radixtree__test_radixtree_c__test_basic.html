test_basic

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_radixtree/test_radixtree.c:162

目的

適応型基数木（ART）の特定のノードクラスおよびシフト量に対して、挿入、検索、更新、削除、およびイテレーションの一連のCRUD操作を検証する。

シグネチャまたは形状

static void test_basic(rt_node_class_test_elem *test_info, int shift, bool asc)

入力と出力

  • **入力**:
  • test_info: テスト対象のノードクラス情報（nkeys を含む）。
  • shift: 基数木の深さを制御するシフト量。
  • asc: キーの挿入順序（true なら昇順、false なら降順）。
  • **出力**: なし（検証失敗時は elog(ERROR) によりアボート）。

前提条件と不変条件

  • TEST_SHARED_RT マクロの定義有無により、共有メモリ（DSA）またはローカルメモリコンテキストのいずれかで基数木が初期化されること。
  • 挿入されるキーは shift 値に基づいて生成され、木構造の整合性を維持すること。

アルゴリズムまたは振る舞い

1. **初期化**: rt_create を呼び出し、基数木インスタンスを生成する。 2. **挿入**: test_info->nkeys 分のキーを rt_set で挿入し、初期状態ではキーが存在しないことを確認する。 3. **検索**: rt_find を使用し、挿入したすべてのキーが期待値通りに取得できるか検証する。 4. **更新**: rt_set を使用して既存キーの値を更新し、成功を確認する。 5. **削除・再挿入**: 各キーを rt_delete で削除後、再度 rt_set で挿入し、整合性を確認する。 6. **イテレーション**: rt_begin_iterate を使用し、キーが昇順で取得されることを確認する。 7. **クリーンアップ**: 全キーを削除し、rt_find で存在しないことを確認した後、rt_free でメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/test_radixtree/test_radixtree.c 内の test_radixtree 関数から、異なるシフト量と順序の組み合わせで複数回呼び出される。
  • **呼び出し先**: rt_create, rt_set, rt_find, rt_delete, rt_begin_iterate, rt_iterate_next, rt_end_iterate, rt_stats, rt_free 等の基数木ライブラリ API。

関連データ構造

  • rt_radix_tree: テスト対象の基数木インスタンス。
  • rt_node_class_test_elem: テスト対象のノードクラス名とキー数を保持する構造体。
  • rt_iter: イテレーション状態を保持する構造体。

失敗モードと境界条件

  • EXPECT_* マクロの検証に失敗した場合、elog(ERROR) が発行され、テストが中断される。
  • shift 値が大きすぎる場合や、メモリ不足時には基数木操作が失敗する可能性がある。

保守担当者向けメモ

  • 本関数は TEST_SHARED_RT マクロの有無でメモリ管理戦略が分岐するため、共有メモリ環境でのテスト時は LWLock の管理に注意すること。
  • イテレーションの検証ロジックは、asc フラグに関わらずイテレータが常に昇順でキーを返すことを前提としている。
