DeallocateQuery

定義

  • 種別: function
  • 定義箇所: src/backend/commands/prepare.c:504

目的

SQLの DEALLOCATE 文を処理し、セッション内で保持されているプリペアドステートメントを破棄する。

シグネチャまたは形状

void DeallocateQuery(DeallocateStmt *stmt)

入力と出力

  • **入力**: DeallocateStmt *stmt - 実行対象の DEALLOCATE 文の解析ツリー。stmt->name に破棄対象のステートメント名が格納される（全削除の場合は NULL）。
  • **出力**: なし（副作用としてステートメントの削除を実行）。

前提条件と不変条件

  • prepared_queries ハッシュテーブルが初期化されている必要がある。
  • stmt が指す構造体は有効であること。

アルゴリズムまたは振る舞い

1. stmt->name が存在する場合、DropPreparedStatement を呼び出し、指定された名前のプリペアドステートメントをハッシュテーブルから削除する。 2. stmt->name が NULL の場合、DropAllPreparedStatements を呼び出し、セッション内のすべてのプリペアドステートメントを破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（DEALLOCATE 文の実行時）
  • **呼び出し先**:
  • DropPreparedStatement（名前指定時）
  • DropAllPreparedStatements（全削除時）

関連データ構造

  • DeallocateStmt: DEALLOCATE 文の解析結果を保持する構造体。
  • prepared_queries: セッションローカルのプリペアドステートメントを格納するハッシュテーブル。

失敗モードと境界条件

  • 指定された名前のプリペアドステートメントが存在しない場合、DropPreparedStatement 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は単なるディスパッチャであり、実際の削除ロジックは DropPreparedStatement および DropAllPreparedStatements に依存している。
  • プリペアドステートメントの削除に伴い、関連する CachedPlan の参照カウントが適切に減算されることを確認すること。
