xl_heap_update

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/heapam_xlog.h:218

目的

ヒープテーブルにおけるタプル更新操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。リカバリ時に、古いタプルの状態変更と新しいタプルの挿入を再現するために使用されます。

シグネチャまたは形状

typedef struct xl_heap_update
{
    TransactionId old_xmax;
    OffsetNumber old_offnum;
    uint8        old_infobits_set;
    uint8        flags;
    TransactionId new_xmax;
    OffsetNumber new_offnum;
} xl_heap_update;

入力と出力

  • **入力**: 更新対象の古いタプル情報（old_xmax, old_offnum）および新しいタプルの配置情報（new_offnum）、更新フラグ。
  • **出力**: WALレコードの一部としてディスクへ書き出されるバイナリデータ。

前提条件と不変条件

  • flags に XLH_UPDATE_CONTAINS_OLD_TUPLE または XLH_UPDATE_CONTAINS_OLD_KEY が設定されている場合、構造体の直後に xl_heap_header およびタプルデータが続く必要があります。
  • SizeOfHeapUpdate マクロにより、構造体のサイズが定義されています。

アルゴリズムまたは振る舞い

1. **記録**: heapam.c が更新時に本構造体を初期化し、WALレコードとして書き込みます。 2. **再生**: リカバリ時、heapam_xlog.c の heap_redo 等が XLogRecGetData を介して本構造体を読み込みます。 3. **適用**: 読み取ったオフセット情報に基づき、対象ページ内のタプルに対して xmax の更新や新しいタプルの挿入を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c（WAL生成時）、src/backend/access/rmgrdesc/heapdesc.c（デコード時）、src/backend/replication/logical/decode.c（論理デコード時）。
  • **呼び出し先**: 直接的な関数呼び出し関係はソースに明示されていませんが、XLogRecGetData を通じてWALリーダーからデータが供給されます。

関連データ構造

  • xl_heap_header: 更新時に古いタプルデータが含まれる場合に付随するヘッダー。
  • OffsetNumber: ページ内のタプル位置を特定する型。
  • TransactionId: トランザクション識別子。

失敗モードと境界条件

  • flags の設定と実際のデータペイロードの不一致は、リカバリ時のデータ破損や読み取りエラーを引き起こす可能性があります。
  • 8ビットのフラグ領域が定義されており、拡張時にはこの制約に注意が必要です。

保守担当者向けメモ

  • 本構造体は可変長データ（old_tuple）を伴う可能性があるため、解析やデバッグ時には flags フィールドの値を必ず確認してください。
  • XLOG_HEAP_UPDATE オペコードに関連付けられており、heapam_xlog.c の REDO ロジックと密接に結合しています。
