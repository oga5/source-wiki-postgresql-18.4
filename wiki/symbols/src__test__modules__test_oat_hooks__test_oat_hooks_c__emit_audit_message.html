emit_audit_message

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_oat_hooks/test_oat_hooks.c:229

目的

PostgreSQLのオブジェクトアクセスフックの実行状況（試行、完了、拒否）を NOTICE レベルのメッセージとして出力し、テスト結果の監査を行う。

シグネチャまたは形状

static void emit_audit_message(const char *type, const char *hook, char *action, char *objName)

入力と出力

  • **入力**:
  • type: 監査の種類（"attempting", "finished", "denied" 等）。
  • hook: フックの名称。
  • action: 実行された操作内容（呼び出し元で palloc された文字列）。
  • objName: 対象オブジェクト名（呼び出し元で palloc された文字列、または NULL）。
  • **出力**: なし（void）。

前提条件と不変条件

  • action および objName は、呼び出し元で動的に割り当てられたメモリ領域を指している必要があり、本関数内で pfree される。
  • 並列クエリ実行時、ログの重複を防ぐためリーダープロセスでのみ出力を行う。

アルゴリズムまたは振る舞い

1. REGRESS_audit が有効かつ現在のプロセスが並列ワーカーではないことを確認する。 2. 現在のユーザーがスーパーユーザーかどうかを判定し、文字列 "superuser" または "non-superuser" を取得する。 3. objName の有無に応じて、ereport(NOTICE, ...) を使用して監査メッセージを構築・出力する。 4. action および objName が NULL でない場合、それぞれ pfree を呼び出してメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • audit_attempt
  • audit_success
  • audit_failure
  • **呼び出し先**:
  • IsParallelWorker()
  • superuser_arg()
  • GetUserId()
  • ereport()
  • pfree()

関連データ構造

  • REGRESS_audit: 監査ログ出力を制御するグローバルなフラグ変数。

失敗モードと境界条件

  • **並列ワーカー**: IsParallelWorker() が真を返す場合、監査メッセージは出力されない。
  • **メモリ管理**: action や objName が NULL の場合、pfree はスキップされる。
  • **ログ出力**: ereport のエラーコードとして ERRCODE_INTERNAL_ERROR が固定で使用される。

保守担当者向けメモ

  • 本関数はテスト専用モジュールの一部であり、pfree を行うことで呼び出し元でのメモリ解放漏れを防ぐ設計となっている。
  • 並列クエリ環境下でのテスト結果の決定性を維持するため、IsParallelWorker() によるフィルタリングは必須である。
