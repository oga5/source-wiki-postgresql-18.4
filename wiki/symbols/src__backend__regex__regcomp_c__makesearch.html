makesearch

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regcomp.c:621

目的

正規表現の NFA（非決定性有限オートマトン）を検索用 NFA に変換する。具体的には、パターンの先頭に暗黙的な .* を付与することで、文字列内の任意の場所からマッチングを開始できるようにする。

シグネチャまたは形状

static void makesearch(struct vars *v, struct nfa *nfa)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体。
  • nfa: 変換対象の NFA 構造体。
  • **出力**: なし（nfa 構造体を直接書き換える）。

前提条件と不変条件

  • nfa は有効な構造体であること。
  • nfa->pre（開始状態）が存在すること。
  • 変換後、パターンが MATCHALL であれば nfa->maxmatchall が DUPINF に設定される。

アルゴリズムまたは振る舞い

1. **先頭の最適化**: パターンがアンカー（^ や \A）で固定されていない場合、pre 状態に対して .* に相当する遷移（rainbow）および bos（行頭/文字列先頭）への遷移を追加する。 2. **状態の分離**: pre 状態以外からも遷移先を持つ状態（ループバックの可能性がある状態）を特定する。 3. **状態の分割**: 特定された状態を「進行」と「非進行」の状態に分割する。これにより、pre 状態を経由せずに到達可能なパスを正しく処理し、検索エンジンがマッチングの進行状況を適切に追跡できるようにする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/regcomp.c 内のコンパイルプロセス（pg_regcomp 等のフロー中）。
  • **呼び出し先**:
  • rainbow, newarc, newstate, copyouts, cparc, freearc, NOERR（マクロ）。

関連データ構造

  • struct vars: コンパイルコンテキスト。
  • struct nfa: NFA の状態と遷移を管理。
  • struct state, struct arc: NFA の構成要素。

失敗モードと境界条件

  • NOERR() マクロにより、メモリ不足等のエラー発生時に処理を中断する。
  • パターンが既にアンカーで固定されている場合、.* の付与はスキップされる。

保守担当者向けメモ

  • 本関数は正規表現エンジンにおける「検索モード」への変換を担う重要な最適化ステップである。
  • 状態分割のロジックは、pre 状態からの遷移とそれ以外の遷移を区別するために tmp フィールドを一時的に利用している。処理終了時に tmp フィールドをクリアしている点に注意すること。
