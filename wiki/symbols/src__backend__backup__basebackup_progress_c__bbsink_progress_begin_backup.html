bbsink_progress_begin_backup

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_progress.c:83

目的

ベースバックアップのデータストリーミングフェーズ開始を pg_stat_progress_basebackup ビューに通知し、バックアップ全体の統計情報を初期化する。

シグネチャまたは形状

static void bbsink_progress_begin_backup(bbsink *sink);

入力と出力

  • **入力**: bbsink *sink （進捗追跡用シンクの状態を保持する構造体）
  • **出力**: なし（副作用として pgstat 統計情報の更新および後続シンクへの処理委譲を行う）

前提条件と不変条件

  • sink->bbs_state が有効であり、バックアップのメタデータ（テーブルスペースリストや合計サイズ情報）が参照可能であること。

アルゴリズムまたは振る舞い

1. PROGRESS_BASEBACKUP_PHASE_STREAM_BACKUP を用いて、現在のフェーズを「データストリーミング中」に設定する。 2. sink->bbs_state->bytes_total_is_valid が真であれば推定合計サイズを、偽であれば -1 を統計情報にセットする。 3. sink->bbs_state->tablespaces のリスト長からテーブルスペース数を取得する。 4. pgstat_progress_update_multi_param を呼び出し、上記情報をシステム統計ビューへ反映させる。 5. bbsink_forward_begin_backup(sink) を呼び出し、パイプライン内の後続シンクへ処理を転送する。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_progress_ops 構造体（bbsink インターフェースのコールバックテーブル）経由で呼び出される。
  • **呼び出し先**: pgstat_progress_update_multi_param（統計更新）、bbsink_forward_begin_backup（後続シンクへの委譲）。

関連データ構造

  • bbsink: バックアップのデータ転送パイプラインを構成する基本構造体。
  • bbsink_state: バックアップの進行状況やメタデータを保持する構造体。

失敗モードと境界条件

  • bytes_total_is_valid が偽の場合、合計サイズとして -1 が報告される。
  • tablespaces リストが空の場合、テーブルスペース数は 0 として報告される。

保守担当者向けメモ

  • 本関数は bbsink チェーンのデコレータとして機能するため、必ず最後に bbsink_forward_begin_backup を呼び出し、チェーンの断絶を防ぐ必要がある。
