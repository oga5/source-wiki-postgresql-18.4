tfunc_lower

定義

  • 種別: function
  • 定義箇所: src/common/unicode/case_test.c:293

目的

PostgreSQL の Unicode 小文字変換関数 unicode_strlower をテスト用インターフェースとしてラップし、test_convert 関数から呼び出せるようにする。

シグネチャまたは形状

static size_t tfunc_lower(char *dst, size_t dstsize, const char *src, ssize_t srclen);

入力と出力

  • **入力**:
  • dst: 変換後の文字列を格納するバッファ。
  • dstsize: dst バッファのサイズ。
  • src: 変換元の文字列。
  • srclen: src の長さ（バイト単位）。
  • **出力**:
  • 変換後の文字列の長さ（バイト単位）を size_t で返す。

前提条件と不変条件

  • unicode_strlower が利用可能であること。
  • unicode_strlower の第 5 引数（is_lower）に true を渡して呼び出す。

アルゴリズムまたは振る舞い

1. 受け取った引数をそのまま unicode_strlower に転送する。 2. unicode_strlower の第 5 引数に true を指定し、小文字変換であることを明示して実行する。 3. unicode_strlower の戻り値をそのまま呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/unicode/case_test.c 内の test_convert 関数（複数のテストケースで呼び出される）。
  • **呼び出し先**:
  • unicode_strlower (PostgreSQL 内部の Unicode 変換関数)

関連データ構造

  • なし（関数ポインタとして test_convert に渡される）。

失敗モードと境界条件

  • unicode_strlower の仕様に依存する。dstsize が不足している場合や src が不正なエンコーディングである場合の挙動は unicode_strlower の実装に従う。

保守担当者向けメモ

  • 本関数はテスト専用のラッパーである。変換ロジック自体を変更する場合は unicode_strlower を修正すること。
  • test_convert を通じて、多様な文字列（特殊文字やギリシャ文字など）に対する変換結果の検証に使用されている。
