gbt_tskey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_ts.c:100

目的

GiSTインデックスのソート処理において、2つの tsKEY（タイムスタンプ範囲）を比較し、順序を決定する。

シグネチャまたは形状

static int gbt_tskey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部の t メンバが tsKEY としてキャストされる。
  • flinfo: 関数呼び出し情報（本関数内では直接使用されていない）。
  • **出力**:
  • 比較結果を示す整数。a < b なら負、a == b なら 0、a > b なら正を返す。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体を指している必要がある。
  • tsKEY 構造体には lower および upper メンバが存在し、有効なタイムスタンプ値が格納されていること。

アルゴリズムまたは振る舞い

1. a と b の lower（範囲の下限）を timestamp_cmp 関数で比較する。 2. lower の比較結果が 0（等しい）でない場合、その結果を直ちに返す。 3. lower が等しい場合、upper（範囲の上限）を timestamp_cmp 関数で比較し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の静的初期化子（tinfo 構造体の定義箇所）から参照されている。
  • **呼び出し先**: timestamp_cmp（PostgreSQLのコア関数）。

関連データ構造

  • tsKEY: タイムスタンプの範囲（lower, upper）を保持する構造体。
  • Nsrt: ソート対象のノード情報を保持する構造体。

失敗モードと境界条件

  • timestamp_cmp が内部でエラーを発生させた場合、呼び出し元に伝播する。
  • a または b が NULL の場合の挙動は定義されていない。

保守担当者向けメモ

  • 本関数は timestamp 型の比較ロジックに依存している。timestamptz 型を扱う場合も、事前にGMTへ正規化された値が tsKEY に格納されていることを前提としている。
