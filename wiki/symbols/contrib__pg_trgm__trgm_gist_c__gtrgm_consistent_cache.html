gtrgm_consistent_cache

定義

  • 種別: struct
  • 定義箇所: contrib/pg_trgm/trgm_gist.c:24

目的

GiST インデックスの検索処理（gtrgm_consistent）において、クエリのトリグラム抽出や正規表現グラフの生成結果をキャッシュし、CPU コストを削減するために使用されます。

シグネチャまたは形状

typedef struct {
    StrategyNumber strategy;
    text          *query;
    TRGM          *trigrams;
    TrgmPackedGraph *graph;
} gtrgm_consistent_cache;

入力と出力

  • **入力**: gtrgm_consistent 関数実行時に、fcinfo->flinfo->fn_extra を介して取得・保持されます。
  • **出力**: キャッシュされた trigrams および graph は、インデックスノードとの照合処理において再利用されます。

前提条件と不変条件

  • query および trigrams は、本構造体と同じ palloc ブロック内に MAXALIGN 境界で配置されます。
  • graph は構造体とは別のメモリ領域に配置されます。

アルゴリズムまたは振る舞い

1. gtrgm_consistent 呼び出し時に fn_extra を確認し、既存のキャッシュがあればそれを利用します。 2. キャッシュが存在しない場合、クエリからトリグラムを抽出し、必要に応じて正規表現グラフを生成してキャッシュに格納します。 3. インデックスの再スキャン時など、キャッシュのライフサイクル管理においてメモリリークが発生する可能性があることが指摘されています。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_gist.c 内の gtrgm_consistent 関数が、インデックス検索の最適化のために本構造体を初期化および参照します。
  • **呼び出し先**: 直接的な関数呼び出し関係はありませんが、本構造体内のデータは trgm_op.c や trgm_regexp.c で定義されたロジックによって生成・利用されます。

関連データ構造

  • StrategyNumber: 検索戦略を識別します。
  • TRGM: 抽出されたトリグラムの配列。
  • TrgmPackedGraph: 正規表現演算子使用時のグラフ構造。

失敗モードと境界条件

  • インデックスの再スキャン時にキャッシュが適切に解放されない場合、メモリリークが発生する可能性があります。
  • graph フィールドは構造体本体とは別のメモリ領域にあるため、メモリ管理時に注意が必要です。

保守担当者向けメモ

  • 本構造体は fn_extra を通じて永続化されるため、メモリ管理の整合性に注意してください。
  • 正規表現グラフのキャッシュについては、現在の実装においてリークの懸念が示唆されているため、修正や最適化の際には trgm_gist.c のキャッシュ更新ロジックを精査してください。
