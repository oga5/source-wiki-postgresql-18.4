macaddr_gt

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac.c:245

目的

2つの macaddr 型の値を受け取り、第1引数が第2引数より大きいかどうかを判定する比較関数です。

シグネチャまたは形状

Datum macaddr_gt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの macaddr 構造体へのポインタを受け取ります。
  • **出力**: 第1引数が第2引数より大きい場合は true、それ以外の場合は false を Datum 型として返します。

前提条件と不変条件

  • 入力される macaddr ポインタは有効である必要があります。
  • 比較ロジックは macaddr_cmp_internal に依存しており、6バイトのMACアドレスを辞書順で比較します。

アルゴリズムまたは振る舞い

1. PG_GETARG_MACADDR_P マクロを使用して、2つの入力引数 a1 と a2 を取得します。 2. macaddr_cmp_internal(a1, a2) を呼び出し、比較結果を取得します。 3. 比較結果が 0 より大きい（a1 > a2）場合に true を、そうでない場合に false を PG_RETURN_BOOL を通じて返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_macaddr.c:33 (直接呼び出し)
  • **呼び出し先**:
  • macaddr_cmp_internal (比較ロジックの実行)

関連データ構造

  • macaddr: 6バイトのEUI-48 MACアドレスを保持する構造体。

失敗モードと境界条件

  • 引数が NULL の場合、PG_GETARG_MACADDR_P の挙動に従い、呼び出し元で適切に処理される必要があります。
  • MACアドレスが等しい場合、この関数は false を返します。

保守担当者向けメモ

  • 本関数は標準的な比較演算子 > の実装です。比較ロジックを変更する場合は macaddr_cmp_internal を修正してください。
  • btree_gist などの外部モジュールから利用されているため、シグネチャの変更には注意が必要です。
