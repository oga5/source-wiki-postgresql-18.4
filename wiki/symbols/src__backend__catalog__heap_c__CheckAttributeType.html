CheckAttributeType

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:547

目的

テーブルや複合型の列定義において、指定されたデータ型がその列で使用可能かどうかを検証する。再帰的に型構造を探索し、制約違反やサポートされていない型構成を検出する。

シグネチャまたは形状

void CheckAttributeType(const char *attname,
                        Oid atttypid, Oid attcollation,
                        List *containing_rowtypes,
                        int flags)

入力と出力

  • attname: 検証対象の列名。
  • atttypid: 検証対象の型のOID。
  • attcollation: 型の照合順序（Collation）のOID。
  • containing_rowtypes: 再帰呼び出し時の自己参照（循環参照）を検出するための型OIDリスト。
  • flags: 検証ルールを制御するビットフラグ（CHKATYPE_ANYARRAY, CHKATYPE_ANYRECORD, CHKATYPE_IS_PARTKEY, CHKATYPE_IS_VIRTUAL 等）。
  • 出力: なし。検証失敗時は ereport(ERROR, ...) によりトランザクションを中断する。

前提条件と不変条件

  • 本関数は再帰的に呼び出されるため、スタックオーバーフローを防ぐために check_stack_depth() を実行する。
  • カタログ操作を伴うため、適切なトランザクションコンテキストで実行される必要がある。

アルゴリズムまたは振る舞い

1. **擬似型チェック**: TYPTYPE_PSEUDO の場合、許可されたフラグ（ANYARRAY 等）と一致しない限りエラーとする。 2. **ドメイン型チェック**: TYPTYPE_DOMAIN の場合、仮想生成列であればエラーとし、それ以外は基底型に対して再帰的に検証を行う。 3. **複合型チェック**: TYPTYPE_COMPOSITE の場合、containing_rowtypes を用いて自己参照を検出し、エラーとする。問題なければ、その型の各属性に対して再帰的に検証を行う。 4. **範囲型/マルチ範囲型チェック**: それぞれ基底型または範囲型に対して再帰的に検証を行う。 5. **配列型チェック**: 要素型を取得し、再帰的に検証を行う。 6. **最終検証**: 仮想生成列でユーザー定義型が使用されている場合はエラーとする。また、照合順序が必要な型で attcollation が無効な場合はエラーとする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/heap.c 内のテーブル作成処理、src/backend/catalog/index.c、src/backend/commands/tablecmds.c 等から呼び出される。
  • **呼び出し先**: get_typtype, check_stack_depth, getBaseType, get_typ_typrelid, RelationGetDescr, get_range_subtype, get_element_type 等のカタログアクセス関数。

関連データ構造

  • List *containing_rowtypes: 再帰的な型定義の深さを追跡するためのOIDリスト。
  • TupleDesc: 複合型の属性情報を保持する構造体。

失敗モードと境界条件

  • **循環参照**: 複合型が自身をメンバーとして含む場合、containing_rowtypes により検出されエラーとなる。
  • **サポート外の型**: 仮想生成列におけるドメイン型やユーザー定義型の使用はエラーとなる。
  • **照合順序の欠落**: 照合順序が必要な型で指定がない場合、エラーとなる。

保守担当者向けメモ

  • 本関数は再帰的であるため、新しい型カテゴリを追加する際は、無限再帰が発生しないよう注意が必要である。
  • 「UGLY HARD CODED BADNESS」と評されるシステム属性定義との整合性に留意すること。
  • 仮想生成列の制約チェックは、将来的な機能拡張（ユーザー定義型のサポート等）により変更される可能性がある。
