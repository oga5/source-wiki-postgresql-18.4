LogicalTapeFreeze

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/logtape.c:980

目的

書き込みフェーズを終了し、論理テープを読み取り専用（かつランダムアクセス可能）な状態に固定する。

シグネチャまたは形状

void LogicalTapeFreeze(LogicalTape *lt, TapeShare *share)

入力と出力

  • lt: 固定対象の LogicalTape 構造体へのポインタ。
  • share: 並列ソート時にテープセットを共有するための TapeShare 構造体（不要な場合は NULL）。
  • 出力なし（副作用として lt の状態が更新される）。

前提条件と不変条件

  • lt->writing が真であること。
  • lt->offsetBlockNumber が 0 であること。

アルゴリズムまたは振る舞い

1. **書き込み終了処理**: テープがダーティ（未フラッシュのデータあり）な場合、バッファの残りを物理ファイルへフラッシュする。 2. **状態遷移**: lt->writing を偽、lt->frozen を真に設定する。 3. **バッファ再構成**: 読み取り効率のため、バッファサイズを BLCKSZ に調整する。 4. **読み取り準備**: テープの先頭ブロックを読み込み、curBlockNumber、pos、nbytes を初期化する。 5. **チェーン情報の更新**: 最初のブロックのトレーラーから次のブロック番号を取得し、nextBlockNumber を設定する。 6. **共有処理**: share が指定されている場合、BufFileExportFileSet を呼び出し、共有用のメタデータを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/sort/tuplesort.c（tuplesort.c:2186, tuplesort.c:3031）
  • **呼び出し先**: ltsWriteBlock, ltsReadBlock, pfree, palloc, TapeBlockSetNBytes, TapeBlockIsLast, TapeBlockGetTrailer, TapeBlockGetNBytes, BufFileExportFileSet

関連データ構造

  • LogicalTape: テープの現在の状態（書き込み中か、凍結済みか等）を保持。
  • LogicalTapeSet: テープセット全体を管理。
  • TapeShare: 並列処理におけるテープ共有用構造体。

失敗モードと境界条件

  • テープが空（firstBlockNumber == -1L）の場合、nextBlockNumber を -1 に設定して適切に処理する。
  • バッファサイズが BLCKSZ と異なる場合は再割り当てを行う。

保守担当者向けメモ

  • 本関数を呼び出すと、以降の読み込みでブロックが解放されなくなるため、ソートの最終マージパスでのみ使用すること。
  • VALGRIND_MAKE_MEM_DEFINED を使用して、バッファの未初期化領域に対する警告を抑制している。
