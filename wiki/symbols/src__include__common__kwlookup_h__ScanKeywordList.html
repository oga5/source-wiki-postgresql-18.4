ScanKeywordList

定義

  • 種別: struct
  • 定義箇所: src/include/common/kwlookup.h:25

目的

SQL キーワードや識別子を効率的に検索・照合するための、パーフェクトハッシュを用いたルックアップ用データ構造を提供します。

シグネチャまたは形状

typedef struct ScanKeywordList
{
	const char *kw_string;		/* 全キーワード（\0区切り） */
	const uint16 *kw_offsets;	/* 各キーワードの開始オフセット */
	ScanKeywordHashFunc hash;	/* パーフェクトハッシュ関数 */
	int			num_keywords;	/* キーワード総数 */
	int			max_kw_len;		/* 最長キーワードの長さ */
} ScanKeywordList;

入力と出力

本構造体は、ScanKeywordLookup() 関数への引数として入力され、指定された文字列がリスト内に存在するかを判定するために使用されます。

前提条件と不変条件

  • kw_string は、すべてのキーワードが \0 で区切られた文字列の集合である必要があります。
  • hash 関数は、リスト内のキーワードに対して衝突が発生しない「パーフェクトハッシュ」である必要があります。
  • 通常、src/tools/gen_keywordlist.pl によって生成された静的データで初期化されます。

アルゴリズムまたは振る舞い

1. **フィルタリング**: max_kw_len を用いて、入力文字列の長さによる事前選別を行います。 2. **ハッシュ計算**: hash 関数ポインタを介して入力文字列のハッシュ値を算出します。 3. **照合**: ハッシュ値から特定された候補に対し、ASCII ベースの小文字変換を行いながら逐次比較を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/keywords.c、src/common/kwlookup.c、src/include/parser/scanner.h、src/interfaces/ecpg/preproc/ 配下の各ファイル、src/pl/plpgsql/src/pl_scanner.c などで参照されています。
  • **呼び出し先**: 構造体内の hash フィールドは ScanKeywordHashFunc 型の関数を指します。

関連データ構造

  • ScanKeywordHashFunc: ハッシュ計算を行う関数ポインタ型。
  • ScanKeywordLookup(): 本構造体を利用して検索を行う関数。
  • GetScanKeyword(): インデックスからキーワード文字列を取得するインライン関数。

失敗モードと境界条件

  • ハッシュ関数がパーフェクトハッシュとして正しく生成されていない場合、検索結果の不整合が発生します。
  • ロケール依存の tolower() を避けているため、SQL 標準に準拠しない文字変換は行われません。

保守担当者向けメモ

  • 本構造体は src/tools/gen_keywordlist.pl に強く依存しています。構造を変更する場合は、生成スクリプトの更新が必要です。
  • tolower() を使用するとトルコ語などの特定のロケールで誤動作するため、比較ロジック内では使用しないでください。
