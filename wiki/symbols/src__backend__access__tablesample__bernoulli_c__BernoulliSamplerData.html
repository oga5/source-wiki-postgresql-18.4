BernoulliSamplerData

定義

  • 種別: struct
  • 定義箇所: src/backend/access/tablesample/bernoulli.c:37

目的

TABLESAMPLE BERNOULLI メソッドによるスキャン実行中の状態を保持するためのプライベート構造体です。

シグネチャまたは形状

typedef struct
{
	uint64		cutoff;			/* select tuples with hash less than this */
	uint32		seed;			/* random seed */
	OffsetNumber lt;			/* last tuple returned from current block */
} BernoulliSamplerData;

入力と出力

本構造体は関数ではなくデータ構造であるため、直接的な入力・出力はありません。実行時には tsm_state としてメモリ上に確保され、サンプリングの判定基準として利用されます。

前提条件と不変条件

  • cutoff は、タプルごとに計算されたハッシュ値と比較され、選択の閾値として機能します。
  • seed は、クエリ間で一貫したサンプリング結果を保証するために使用されます。
  • lt は、現在のブロック内で最後に返されたタプルのオフセットを保持し、スキャン状態を追跡します。

アルゴリズムまたは振る舞い

スキャン中、bernoulli_nextsampletuple が各タプルに対してハッシュ値を生成し、その値が cutoff 未満である場合にタプルを選択します。本構造体はこの判定に必要な閾値と現在のスキャン位置を保持することで、物理的なテーブル構造に依存しない確率的なサンプリングを実現します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • bernoulli_initsamplescan (メモリ確保時)
  • bernoulli_beginsamplescan (状態へのアクセス)
  • bernoulli_nextsampletuple (状態の読み取りおよび更新)
  • **呼び出し先**:
  • 情報なし（本構造体はデータ保持用であり、関数を呼び出すことはありません）。

関連データ構造

  • TsmRoutine: TABLESAMPLE メソッドのインターフェースを定義する構造体。
  • OffsetNumber: PostgreSQL のタプルオフセット型。

失敗モードと境界条件

  • メモリ割り当て失敗時: palloc0 によるメモリ確保が失敗した場合、プロセスは終了します。
  • 境界条件: lt がブロック内の最終タプルに達した際、スキャンは次のブロックへ移行します。

保守担当者向けメモ

  • 本構造体は bernoulli.c 内でのみ使用されるプライベートな状態保持用です。
  • ハッシュ計算には hash_any が使用されており、マシンアーキテクチャに依存しない再現性が担保されています。変更を加える際は、サンプリングの統計的性質に影響を与えないよう注意してください。
