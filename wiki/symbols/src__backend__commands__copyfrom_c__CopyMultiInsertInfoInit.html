CopyMultiInsertInfoInit

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfrom.c:399

目的

COPY FROM コマンド実行時におけるバルク挿入管理構造体 CopyMultiInsertInfo を初期化します。

シグネチャまたは形状

static void CopyMultiInsertInfoInit(CopyMultiInsertInfo *miinfo, ResultRelInfo *rri, CopyFromState cstate, EState *estate, CommandId mycid, int ti_options)

入力と出力

  • **入力**:
  • miinfo: 初期化対象の CopyMultiInsertInfo 構造体へのポインタ。
  • rri: ターゲットテーブルの ResultRelInfo。
  • cstate: COPY 実行状態を保持する CopyFromState。
  • estate: 実行時状態 EState。
  • mycid: 現在のコマンド ID。
  • ti_options: テーブル挿入オプション。
  • **出力**: なし（miinfo が指す構造体の各フィールドを初期化）。

前提条件と不変条件

  • miinfo は有効なメモリ領域を指している必要があります。
  • パーティションテーブル以外のテーブルに対しては、この関数内で CopyMultiInsertInfoSetupBuffer が呼び出され、バッファの初期化が行われます。

アルゴリズムまたは振る舞い

1. miinfo の各フィールド（multiInsertBuffers, bufferedTuples, bufferedBytes, cstate, estate, mycid, ti_options）を初期化します。 2. ターゲットテーブルがパーティションテーブル（RELKIND_PARTITIONED_TABLE）でない場合、CopyMultiInsertInfoSetupBuffer を呼び出してバッファをセットアップします。 3. パーティションテーブルの場合は、バッファのセットアップを遅延させます（後続の挿入時に行われます）。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c:400 (定義内での再帰的参照ではなく、同ファイル内の初期化処理)
  • src/backend/commands/copyfrom.c:1073
  • **呼び出し先**:
  • CopyMultiInsertInfoSetupBuffer (条件付き)

関連データ構造

  • CopyMultiInsertInfo: バルク挿入の状態を管理する構造体。
  • ResultRelInfo: ターゲットリレーションの情報。
  • CopyFromState: COPY コマンドの実行状態。

失敗モードと境界条件

  • rri が NULL の場合や、rri->ri_RelationDesc が適切に初期化されていない場合は、関数内でセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • パーティションテーブルに対するバッファ初期化の遅延は、パーティションルーティングの動的な性質に対応するための設計です。このロジックを変更する際は、CopyMultiInsertInfoSetupBuffer の呼び出しタイミングに注意してください。
