XLogReleasePreviousRecord

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogreader.c:249

目的

XLogReaderState が保持するデコード済みWALレコードのキューから、最も古いレコードを解放し、メモリを再利用可能にする。

シグネチャまたは形状

XLogRecPtr XLogReleasePreviousRecord(XLogReaderState *state)

入力と出力

  • **入力**: state (XLogReaderState *) - WALリーダーの状態構造体。
  • **出力**: 解放されたレコードの next_lsn を返す。レコードが存在しない場合は InvalidXLogRecPtr を返す。

前提条件と不変条件

  • state->record が指すレコードは、decode_queue_head と一致していなければならない。
  • oversized ではないレコードは、decode_buffer 内の decode_buffer_head に配置されている必要がある。

アルゴリズムまたは振る舞い

1. state->record が空であれば InvalidXLogRecPtr を返して終了する。 2. 対象レコードを decode_queue_head から取り除き、next_lsn を保存する。 3. decode_queue_tail が当該レコードと一致する場合は、NULL にリセットする。 4. レコードが oversized の場合、pfree でメモリを直接解放する。 5. oversized でない場合、decode_buffer_head を次のレコードへ進める。

  • 次のレコードが存在しない（キューが空になった）場合は、decode_buffer_head と decode_buffer_tail をバッファの先頭にリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xlogprefetcher.c:1022
  • src/backend/access/transam/xlogreader.c:329
  • src/backend/access/transam/xlogreader.c:398
  • **呼び出し先**:
  • pfree (メモリ解放)

関連データ構造

  • XLogReaderState: リーダーの状態管理構造体。
  • DecodedXLogRecord: デコード済みWALレコードの構造体。

失敗モードと境界条件

  • state->record が NULL の場合、処理を行わず InvalidXLogRecPtr を返す。
  • decode_buffer が空になった際、ポインタを初期位置へリセットすることでメモリの断片化を防ぐ。

保守担当者向けメモ

  • 本関数は、XLogReaderState のデコードバッファ管理と密接に結合している。oversized レコードと通常のバッファ内レコードで解放パスが異なる点に注意すること。
