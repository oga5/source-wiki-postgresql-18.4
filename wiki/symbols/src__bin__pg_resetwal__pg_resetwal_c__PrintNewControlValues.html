PrintNewControlValues

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:817

目的

pg_resetwal 実行時に、ユーザーの指定によって変更される予定のデータベース制御パラメータ（ControlFile）を標準出力に表示する。

シグネチャまたは形状

static void PrintNewControlValues(void);

入力と出力

  • **入力**: グローバル変数 ControlFile（制御データ）、および mxid_given, mxoff_given, set_oid, set_xid 等のフラグ変数。
  • **出力**: なし（標準出力へのメッセージ表示のみ）。

前提条件と不変条件

  • ControlFile 構造体が適切に初期化または読み込まれていること。
  • ユーザーがコマンドライン引数で値を上書き指定している場合、対応するフラグ（例: set_oid）が真であること。

アルゴリズムまたは振る舞い

1. 「Values to be changed:」というヘッダーを表示する。 2. XLogFileName を使用して、リセット後の最初の WAL セグメント名を表示する。 3. 各フラグ（mxid_given, set_oid, set_xid 等）を確認し、該当する値が変更対象である場合にのみ、ControlFile.checkPointCopy 内の値を整形して表示する。 4. set_xid や set_xid_epoch の表示時には、XidFromFullTransactionId や EpochFromFullTransactionId を用いて FullTransactionId から値を抽出する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_resetwal/pg_resetwal.c:491, 501）から呼び出される。
  • **呼び出し先**: XLogFileName, XidFromFullTransactionId, EpochFromFullTransactionId（いずれも PostgreSQL の内部マクロまたは関数）。

関連データ構造

  • ControlFileData（グローバル変数 ControlFile の型）: データベースの制御情報を保持する。

失敗モードと境界条件

  • 本関数自体はエラーを返さない。
  • 表示される値は、あくまで ControlFile 構造体に現在保持されている値であり、ディスクへの書き込みが成功することを保証するものではない。

保守担当者向けメモ

  • 新しい制御パラメータが ControlFile に追加された場合、本関数にも対応する表示ロジックを追加する必要がある。
  • printf で使用されている _() マクロにより国際化対応されているため、文字列変更時は注意すること。
