get_assignment_input

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteHandler.c:1233

目的

代入操作を含むノードから、代入対象となる入力ノード（ソース）を抽出します。

シグネチャまたは形状

static Node *get_assignment_input(Node *node)

入力と出力

  • **入力**: Node *node （解析対象のクエリツリーノード）
  • **出力**: Node * （抽出された入力ノード、または該当しない場合は NULL）

前提条件と不変条件

  • 入力 node が NULL の場合、NULL を返します。
  • FieldStore または SubscriptingRef ノードのみが有効な入力ノードを保持します。

アルゴリズムまたは振る舞い

1. node が FieldStore 型の場合、その arg フィールドを返します。 2. node が SubscriptingRef 型の場合、refassgnexpr が存在するか確認します。

  • refassgnexpr が NULL であれば、代入操作ではないと判断し NULL を返します。
  • refassgnexpr が存在する場合、refexpr を返します。

    • 3. 上記以外のノード型については NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteHandler.c 内の以下の箇所から呼び出されています。
  • 1147行目: src_input = get_assignment_input(src_expr);
  • 1148行目: prior_input = get_assignment_input(prior_expr);
  • 1163行目: Node *newbottom = get_assignment_input(priorbottom);
  • **呼び出し先**: 該当する関数呼び出しはソース内にありません。

関連データ構造

  • FieldStore: フィールド代入を表現するノード。
  • SubscriptingRef: 配列やサブスクリプトの参照・代入を表現するノード。

失敗モードと境界条件

  • node が NULL の場合は安全に NULL を返します。
  • SubscriptingRef において代入式（refassgnexpr）が欠落している場合、代入操作ではないとみなして NULL を返します。

保守担当者向けメモ

  • 本関数は rewriteHandler.c 内のターゲットリスト正規化処理において、代入の連鎖を辿るために使用されています。
  • 新しい代入型ノードが追加された場合は、本関数の条件分岐を拡張する必要があります。
