hash_bytes_uint32_extended

定義

  • 種別: function
  • 定義箇所: src/common/hashfn.c:630

目的

32ビットの値を64ビットのハッシュ値に変換します。シード値を受け取ることで、ハッシュの衝突を回避し、分布を調整可能なハッシュ計算を提供します。

シグネチャまたは形状

uint64 hash_bytes_uint32_extended(uint32 k, uint64 seed)

入力と出力

  • **入力**:
  • k: ハッシュ化対象の uint32 値。
  • seed: ハッシュ計算の初期状態に影響を与える uint64 値。
  • **出力**:
  • uint64: 計算されたハッシュ値。上位32ビットと下位32ビットを結合して返却されます。

前提条件と不変条件

  • 内部状態 a, b, c は、定数 0x9e3779b9、型サイズ、および固定値 3923095 を用いて初期化されます。
  • seed が 0 以外の場合、シード値の上位・下位32ビットが内部状態に加算され、mix マクロによる攪拌が行われます。

アルゴリズムまたは振る舞い

1. 内部状態 a, b, c を初期化します。 2. seed が 0 でない場合、シード値を内部状態に反映させ mix マクロで攪拌します。 3. 入力値 k を a に加算します。 4. final マクロを適用して最終的なビット混合を行います。 5. 攪拌された b と c をそれぞれ上位・下位32ビットとして結合し、uint64 値として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_bloom.c: Bloomフィルタのインデックス計算に使用。
  • **呼び出し先**:
  • mix マクロ（内部状態の攪拌）
  • final マクロ（最終的なハッシュ値の確定）

関連データ構造

  • uint32, uint64: PostgreSQL の基本型定義（postgres.h 経由）。

失敗モードと境界条件

  • 本関数はエラーを投げない設計であり、入力値 k や seed の全範囲に対して安全に動作します。
  • seed が 0 の場合はシードによる攪拌ステップがスキップされます。

保守担当者向けメモ

  • mix および final マクロはハッシュの分布特性（アバランチ効果）に直結するため、変更時は慎重に評価してください。
  • パフォーマンスが重要なパスで使用されるため、インライン化の恩恵を受ける設計となっています。
