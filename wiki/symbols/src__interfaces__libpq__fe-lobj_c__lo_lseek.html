lo_lseek

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-lobj.c:343

目的

データベース上のラージオブジェクトに対して、指定されたオフセットと基準位置（whence）に基づいて読み書き位置を移動させる。

シグネチャまたは形状

int lo_lseek(PGconn *conn, int fd, int offset, int whence);

入力と出力

  • **入力**:
  • conn: データベース接続を管理する PGconn 構造体へのポインタ。
  • fd: 操作対象のラージオブジェクト記述子。
  • offset: 移動先のオフセット値。
  • whence: シークの基準位置（SEEK_SET, SEEK_CUR, SEEK_END 等）。
  • **出力**:
  • 成功時: 移動後の新しいオフセット位置（int）。
  • 失敗時: -1。

前提条件と不変条件

  • 呼び出し前に lo_initialize(conn) が実行され、サーバー側の関数OIDがキャッシュされている必要がある（関数内で自動的に呼び出される）。
  • ラージオブジェクト操作は現在のトランザクション内で実行される。

アルゴリズムまたは振る舞い

1. lo_initialize を呼び出し、必要なサーバー側関数のOIDがキャッシュされていることを確認する。 2. fd、offset、whence を PQArgBlock 配列に格納する。 3. PQfn を使用して、サーバー側の lo_lseek 関数を呼び出す。 4. サーバーからの応答（PGresult）を確認し、PGRES_COMMAND_OK であれば retval を返し、それ以外の場合は -1 を返す。 5. 処理終了後、PQclear で結果セットを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/examples/testlo.c 等のクライアントアプリケーション。
  • **呼び出し先**: lo_initialize（初期化）、PQfn（サーバー側関数実行）、PQresultStatus、PQclear（libpq API）。

関連データ構造

  • PGconn: 接続状態およびラージオブジェクト関数OIDのキャッシュを保持。
  • PQArgBlock: サーバーへ送信する引数構造体。

失敗モードと境界条件

  • lo_initialize が失敗した場合、-1 を返す。
  • サーバー側でエラーが発生した場合（PQresultStatus が PGRES_COMMAND_OK 以外）、-1 を返す。
  • 非常に大きなファイルサイズを扱う場合、int 型の制限によりオーバーフローする可能性がある（lo_lseek64 の使用が推奨される）。

保守担当者向けメモ

  • 本関数は int を使用しているため、32ビットを超えるオフセットには対応していません。64ビット対応が必要な場合は lo_lseek64 を参照してください。
  • サーバー側の関数OIDは conn->lobjfuncs->fn_lo_lseek に格納されています。
