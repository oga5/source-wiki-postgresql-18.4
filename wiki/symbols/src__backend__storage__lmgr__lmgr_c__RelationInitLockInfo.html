RelationInitLockInfo

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lmgr.c:69

目的

リレーションオブジェクトのロック管理に必要な識別子（LockRelId）を初期化し、リレーションキャッシュエントリに設定する。

シグネチャまたは形状

void RelationInitLockInfo(Relation relation)

入力と出力

  • **入力**: Relation relation （初期化対象のリレーション記述子）
  • **出力**: なし（副作用として relation->rd_lockInfo を更新）

前提条件と不変条件

  • relation が有効であること（RelationIsValid）。
  • relation が有効な OID を保持していること（OidIsValid）。

アルゴリズムまたは振る舞い

1. relation から OID を取得し、rd_lockInfo.lockRelId.relId に代入する。 2. リレーションが共有されているか（relisshared）を確認する。

  • 共有されている場合: rd_lockInfo.lockRelId.dbId を InvalidOid に設定する。
  • 共有されていない場合: rd_lockInfo.lockRelId.dbId を現在のデータベース ID (MyDatabaseId) に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/relcache.c （複数の箇所からリレーションキャッシュ構築時に呼び出される）
  • **呼び出し先**:
  • RelationGetRelid （マクロ）

関連データ構造

  • Relation （リレーション記述子）
  • LockRelId （ロック対象を特定するための構造体）

失敗モードと境界条件

  • relation が NULL または無効な OID を持つ場合、アサーション違反により異常終了する。

保守担当者向けメモ

  • 本関数はリレーションキャッシュの初期化プロセスの一部として実行される。
  • 共有リレーション（relisshared）の判定ロジックは、ロックのスコープをデータベース単位にするか、クラスタ全体にするかを決定する重要な箇所である。
