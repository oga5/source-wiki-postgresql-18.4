ltree_compare

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:45

目的

2つの ltree 型データ間の順序を比較し、辞書順および階層構造に基づいた比較結果を返します。

シグネチャまたは形状

int ltree_compare(const ltree *a, const ltree *b)

入力と出力

  • **入力**: 比較対象となる2つの ltree 構造体へのポインタ a および b。
  • **出力**: 比較結果を示す整数値。
  • 0: 両者が等しい。
  • 正の値: a > b。
  • 負の値: a < b。

前提条件と不変条件

  • a および b は有効な ltree 構造体である必要があります。
  • 内部で LTREE_FIRST および LEVEL_NEXT マクロを使用して各ラベルレベルを走査します。

アルゴリズムまたは振る舞い

1. 両方の ltree の先頭レベルから順に、各ラベルを memcmp で比較します。 2. ラベルの内容が一致する場合、ラベルの長さ (len) を比較します。長さが異なる場合は、その差分と残りのレベル数に基づいた重み付け値を返します。 3. ラベルの内容が異なる場合、memcmp の結果を正規化（-1 または 1）し、残りのレベル数に基づいた重み付け値を乗算して返します。 4. 一方のパスが他方のパスの接頭辞である場合、最後にレベル数 (numlevel) の差分に基づいた値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/ltree/ltree_gist.c 内の GiST インデックス操作関数群（ISEQ マクロ、ノード比較処理など）。
  • **呼び出し先**:
  • memcmp (標準ライブラリ)
  • LTREE_FIRST, LEVEL_NEXT (マクロ)
  • Min (マクロ)

関連データ構造

  • ltree: 階層的なラベルパスを保持する構造体。
  • ltree_level: ltree 内の個々のラベルを構成する構造体。

失敗モードと境界条件

  • 両方のパスが空、または一方が空の場合、numlevel の差分に基づいた結果を返します。
  • ラベルの長さが異なる場合、単純な辞書順比較だけでなく、ラベルの長さの差が結果に反映されます。

保守担当者向けメモ

  • 本関数は ltree の比較ロジックの基盤であり、インデックスの構築や検索の正当性に直結します。
  • 戻り値は単純な -1/0/1 ではなく、レベル数に応じた重み付けが行われているため、比較結果の絶対値には意味があります。変更時は GiST インデックスへの影響を考慮してください。
