WaitEventCustomEntryByName

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/activity/wait_event.c:76

目的

カスタム待機イベントの名前（文字列）と、それに対応する待機イベントID（uint32）を共有メモリ上のハッシュテーブルで管理するためのエントリ構造体です。

シグネチャまたは形状

typedef struct WaitEventCustomEntryByName
{
    char        wait_event_name[NAMEDATALEN]; /* ハッシュキー */
    uint32      wait_event_info;              /* 待機イベントID */
} WaitEventCustomEntryByName;

入力と出力

本構造体はデータ保持用であり、関数ではないため直接的な入力・出力はありません。ハッシュテーブルの検索や挿入操作において、wait_event_name がキーとして使用されます。

前提条件と不変条件

  • wait_event_name は NAMEDATALEN バイト以内の文字列である必要があります。
  • 共有メモリ上のハッシュテーブル（WaitEventCustomHashByName）内で一意であることが期待されます。

アルゴリズムまたは振る舞い

本構造体は、WaitEventCustomNew 等の関数によって共有メモリ上のハッシュテーブルに格納されます。名前からIDを解決する際、この構造体を通じてマッピングが行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/wait_event.c 内のハッシュテーブル操作関数（hash_seq_search を含む検索処理や、カスタムイベント登録処理）から参照されています。
  • **呼び出し先**: 構造体であるため、関数呼び出しは行いません。

関連データ構造

  • WaitEventCustomEntryByInfo: IDから名前を逆引きするための対となる構造体。
  • WaitEventCustomHashByName: 本構造体を要素として格納する共有メモリ上のハッシュテーブル。

失敗モードと境界条件

  • NAMEDATALEN を超える名前を指定した場合の挙動は、文字列の切り詰めやバッファオーバーフローのリスクを伴うため、登録側の実装に依存します。
  • 共有メモリ上のハッシュテーブルが満杯（WAIT_EVENT_CUSTOM_HASH_MAX_SIZE）の場合、新しいエントリの追加は失敗します。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、構造体のサイズ変更は共有メモリのレイアウト設計に影響を与えます。変更時は WaitEventCustomShmemSize の計算ロジックを必ず更新してください。
