output_get_descr_header

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/descriptor.c:155

目的

SQLの GET DESCRIPTOR 文（ヘッダー部）をECPGランタイムライブラリの ECPGget_desc_header 関数呼び出しに変換し、Cコードとして出力する。

シグネチャまたは形状

void output_get_descr_header(const char *desc_name)

入力と出力

  • **入力**:
  • desc_name: 記述子名を指定する文字列。
  • assignments (グローバル変数): 記述子ヘッダーに対して操作対象となる項目リスト。
  • **出力**:
  • base_yyout (外部ストリーム): 生成されたCコード（ECPGget_desc_header(...)）が書き込まれる。

前提条件と不変条件

  • assignments リストには、記述子ヘッダー操作に関連する項目が格納されている必要がある。
  • 記述子ヘッダー操作では ECPGd_count のみが許可される。

アルゴリズムまたは振る舞い

1. ECPGget_desc_header の関数呼び出しの開始部分を出力する。 2. グローバルな assignments リストを走査する。

  • results->value が ECPGd_count である場合、ECPGnumeric_lvalue を呼び出して変数の型情報を出力する。
  • それ以外の項目が指定された場合、mmerror を通じて警告を発する。

    • 3. drop_assignments() を呼び出し、一時的な割り当てリストを解放する。 4. 関数呼び出しの閉じ括弧とセミコロンを出力する。 5. whenever_action(3) を呼び出し、エラーハンドリング設定を適用する。

呼び出し元と呼び出し先

  • **呼び出し元**: 構文解析器（パーサー）が GET DESCRIPTOR 文を認識した際に呼び出される。
  • **呼び出し先**:
  • ECPGnumeric_lvalue
  • mmerror
  • drop_assignments
  • whenever_action

関連データ構造

  • struct assignment: 記述子操作の項目を保持するリスト構造体。
  • base_yyout: コード生成先の出力ストリーム。

失敗モードと境界条件

  • ECPGd_count 以外の項目が assignments に含まれている場合、ET_WARNING レベルの警告が生成される。

保守担当者向けメモ

  • 本関数はヘッダー専用の処理を行う。個別の記述子項目を取得する場合は output_get_descr を参照すること。
  • assignments リストは本関数内で必ず解放されるため、呼び出し側で二重解放しないよう注意が必要である。
