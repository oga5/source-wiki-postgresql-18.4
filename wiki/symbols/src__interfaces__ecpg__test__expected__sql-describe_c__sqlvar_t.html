sqlvar_t

定義

  • 種別: typedef (構造体 sqlvar_struct の別名)
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-describe.c:31

目的

ECPG（Embedded SQL for C）における動的 SQL 記述領域（SQLDA）の各列情報を保持するためのデータ構造です。

シグネチャまたは形状

typedef struct sqlvar_compat sqlvar_t;

※実体は sqlvar_struct です。

入力と出力

  • 入力: DESCRIBE 文の実行結果として、データベースから取得された列のメタデータ（型、長さ、名前など）が格納されます。
  • 出力: アプリケーション側で列の属性を参照するために使用されます。

前提条件と不変条件

  • sqlda_t 構造体の一部として使用されます。
  • 実行時に malloc 等でメモリを確保し、memset 等で初期化して使用されることが一般的です。

アルゴリズムまたは振る舞い

  • SQLDA 内の列定義を保持し、DESCRIBE 文によってメタデータがロードされます。
  • 複数の列が存在する場合、sqlda_t 内の配列として管理されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接の参照例）:
  • src/interfaces/ecpg/test/expected/compat_informix-sqlda.c (メモリ確保および初期化)
  • src/interfaces/ecpg/test/expected/compat_oracle-char_array.c (列情報の参照)
  • src/interfaces/ecpg/test/expected/sql-sqlda.c (構造体の包含関係)
  • 呼び出し先: 情報なし。

関連データ構造

  • sqlda_t: sqlvar_t を保持する親構造体。
  • sqlvar_struct: sqlvar_t の実体となる構造体定義。

失敗モードと境界条件

  • メモリ割り当て失敗: malloc によるメモリ確保が失敗した場合、sqlvar_t へのアクセスは未定義動作となります。
  • 範囲外アクセス: sqlda_t が保持する列数を超えて sqlvar_t 配列にアクセスした場合、メモリ破壊が発生します。

保守担当者向けメモ

  • 本シンボルは ECPG の互換性レイヤー（Informix/Oracle 互換モード）で頻繁に使用されます。
  • sql-describe.c はテスト用コードであるため、実際のライブラリ実装については src/interfaces/ecpg/include/sqlda.h を参照してください。
