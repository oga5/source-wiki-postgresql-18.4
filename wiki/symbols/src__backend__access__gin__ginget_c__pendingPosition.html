pendingPosition

定義

  • 種別: struct
  • 定義箇所: src/backend/access/gin/ginget.c:29

目的

GINインデックスのPending List（未反映の挿入データ）を走査する際、現在の走査位置と状態を保持するために使用されます。

シグネチャまたは形状

typedef struct pendingPosition
{
	Buffer		pendingBuffer;
	OffsetNumber firstOffset;
	OffsetNumber lastOffset;
	ItemPointerData item;
	bool	   *hasMatchKey;
} pendingPosition;

入力と出力

本構造体は関数間で状態を共有するためのコンテナとして機能します。

  • **入力**: 走査対象のバッファ、オフセット範囲、および現在のアイテムポインタ。
  • **出力**: 走査の進捗に伴い、各フィールドが更新されます。

前提条件と不変条件

  • pendingBuffer は有効なバッファである必要があります。
  • hasMatchKey は、検索キーの数に応じたブール値配列を指すポインタである必要があります。

アルゴリズムまたは振る舞い

Pending Listの走査において、現在のページ位置（pendingBuffer）、処理範囲（firstOffset から lastOffset）、および現在注目しているアイテム（item）を追跡します。hasMatchKey を通じて、各キーが現在のアイテムに一致するかどうかの状態を管理します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • scanGetCandidate (src/backend/access/gin/ginget.c:1467)
  • collectMatchesForHeapRow (src/backend/access/gin/ginget.c:1622)
  • pendingPosition 型の変数が src/backend/access/gin/ginget.c:1844 で宣言されています。
  • **呼び出し先**: 構造体であるため、直接的な呼び出し先はありません。

関連データ構造

  • Buffer: バッファ管理用。
  • OffsetNumber: ページ内のオフセット位置。
  • ItemPointerData: 物理的なタプル位置（TID）。

失敗モードと境界条件

  • pendingBuffer が無効な場合や、firstOffset が lastOffset を超えるような不正な範囲指定が行われた場合の挙動は、この構造体を利用する側のロジックに依存します。

保守担当者向けメモ
