reduce

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/parse.c:259

目的

スタック上のトークンを評価し、定義された構文ルールに基づいて複合文を単純な文（stmt）へ畳み込み（還元）、インデントレベルを更新する。

シグネチャまたは形状

void reduce(void)

入力と出力

  • **入力**: グローバル変数 ps（parser_state 構造体）が保持する構文スタック ps.p_stack およびスタックポインタ ps.tos。
  • **出力**: なし（副作用として ps の状態を更新）。

前提条件と不変条件

  • ps.p_stack が有効なトークン列を保持していること。
  • 無限ループ内で動作するため、スタックが還元不可能な状態に達した時点で必ず return される必要がある。

アルゴリズムまたは振る舞い

1. スタックのトップ（ps.tos）にあるトークンを評価し、還元可能なパターンを探索する。 2. stmt がトップにある場合、その直前のトークン（ps.tos - 1）との組み合わせに応じて以下のように還元する：

  • stmt または stmtl の場合: stmtl に統合。
  • dolit の場合: dohead に変換し、インデントレベルを更新。
  • ifstmt の場合: ifhead に変換し、対応するインデントレベルをスタックから検索して更新。
  • swstmt, decl, elsehead, forstmt, whilestmt の場合: stmt に変換し、インデントレベルを更新。

    • 3. whilestmt がトップにある場合、直前が dohead であれば do-while 文の終了とみなし、スタックを2つ分ポップする。 4. いずれのパターンにも一致しない場合、関数を終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: parse() 関数（src/tools/pg_bsd_indent/parse.c 内）。
  • **呼び出し先**: 直接的な関数呼び出しはない。グローバル変数 ps を介して状態を操作する。
  • *注: ツールが提示した他の reduce という名称の関数（pg_stat_statements 等）は、本シンボルとは無関係な別物である。*

関連データ構造

  • parser_state (ps): 構文解析の現在の状態を保持するグローバル構造体。
  • p_stack: ps 内の構文スタック。
  • il: インデントレベルを保持する配列。

失敗モードと境界条件

  • スタックが空の状態や、不正なトークン順序での呼び出しに対する明示的なエラーハンドリングは記述されていない。
  • ループ内で還元が進行しない場合、無限ループに陥る可能性がある。

保守担当者向けメモ

  • 本関数は indent の構文解析の核心部である。新しい構文ルールを追加する際は、switch 文の網羅性と、スタックポインタ ps.tos の操作によるスタックアンダーフローの防止に注意すること。
  • ifstmt の還元において、else の有無は後続の処理で修正される前提となっている。
