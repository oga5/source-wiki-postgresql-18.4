do_set_block_offsets

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_tidstore/test_tidstore.c:181

目的

指定されたブロック番号とオフセットの配列を TidStore に格納し、同時に検証用データ構造（items）を更新してテストの整合性を維持する。

シグネチャまたは形状

Datum do_set_block_offsets(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_INT64(0): 対象のブロック番号 (BlockNumber)。
  • PG_GETARG_ARRAYTYPE_P_COPY(1): 格納するオフセット番号の配列 (ArrayType*)。
  • **出力**:
  • PG_RETURN_INT64(blkno): 処理対象のブロック番号を返す。

前提条件と不変条件

  • check_tidstore_available() により、TidStore が初期化済みであることを確認する必要がある。
  • sanity_check_array(ta) により、入力配列が有効であることを確認する必要がある。
  • TidStoreSetBlockOffsets の仕様上、渡されるオフセットは昇順である必要があるため、関数内で qsort を実行する。

アルゴリズムまたは振る舞い

1. 入力配列からオフセット番号を抽出し、offsetnumber_cmp を用いて昇順にソートする。 2. TidStoreLockExclusive で排他ロックを取得し、TidStoreSetBlockOffsets を呼び出して TidStore にデータを書き込む。 3. TidStoreUnlock でロックを解放する。 4. purge_from_verification_array(blkno) を呼び出し、検証用配列から当該ブロックの既存データを削除する。 5. 検証用配列 items に新しい TID を追加する。必要に応じて repalloc で配列サイズを拡張する。 6. items.num_tids を更新し、統計情報を同期させる。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース経由で呼び出される（PG_FUNCTION_INFO_V1 による登録）。
  • **呼び出し先**:
  • check_tidstore_available
  • sanity_check_array
  • qsort (標準ライブラリ)
  • TidStoreLockExclusive / TidStoreUnlock
  • TidStoreSetBlockOffsets
  • purge_from_verification_array
  • repalloc

関連データ構造

  • TidStore (グローバル変数 tidstore)
  • ItemArray (グローバル変数 items)

失敗モードと境界条件

  • TidStore が初期化されていない場合、check_tidstore_available がエラーを発生させる。
  • 入力配列が不正な場合、sanity_check_array がエラーを発生させる。
  • メモリ不足時、repalloc が失敗する可能性がある。

保守担当者向けメモ

  • 本関数は TidStore の状態と検証用配列 items の状態を同期させる役割を持つ。items 構造体への変更は、check_set_block_offsets による整合性検証に直接影響するため、変更時は注意が必要である。
