ProcessSSLStartup

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/tcop/backend_startup.c:400

目的

クライアントからの接続開始時に、SSLハンドシェイクメッセージ（0x16）を検出し、SSL接続の確立およびALPN（Application-Layer Protocol Negotiation）の検証を行う。

シグネチャまたは形状

static int ProcessSSLStartup(Port *port);

入力と出力

  • **入力**: Port *port（接続情報を保持する構造体）
  • **出力**: int 型。成功時は STATUS_OK、失敗またはSSL非対応時は STATUS_ERROR を返す。

前提条件と不変条件

  • port->ssl_in_use が false であること（アサーションにより保証）。
  • SSLハンドシェイクメッセージの先頭バイトが 0x16 である場合にのみ、SSL接続処理が試行される。

アルゴリズムまたは振る舞い

1. pq_peekbyte() を使用して通信ストリームの先頭バイトを確認する。 2. データが存在しない（EOF）場合は STATUS_ERROR を返す。 3. 先頭バイトが 0x16 でない場合、SSLハンドシェイクではないと判断し STATUS_OK を返す。 4. 0x16 の場合、以下の条件でSSL接続を試行する（USE_SSL 定義時）:

  • SSLが有効かつUnixドメインソケット接続ではないことを確認。
  • secure_open_server(port) を呼び出し、TLSハンドシェイクを実行。
  • port->alpn_used を確認し、ALPNが使用されていない場合はプロトコル違反としてエラーを報告。

    • 5. 成功時は STATUS_OK を返し、失敗時は STATUS_ERROR を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/backend_startup.c 内の接続初期化プロセス（BackendInitialize からの呼び出しが確認されている）。
  • **呼び出し先**: pq_startmsgread, pq_peekbyte, pq_endmsgread, secure_open_server, ereport。

関連データ構造

  • Port: 接続状態やソケット情報を保持する構造体。

失敗モードと境界条件

  • **通信エラー**: pq_peekbyte が EOF を返した場合。
  • **SSL非対応**: ビルド時に USE_SSL が定義されていない、または LoadedSSL が偽の場合。
  • **プロトコル違反**: SSL接続要求においてALPNがネゴシエーションされていない場合。
  • **ハンドシェイク失敗**: secure_open_server が -1 を返した場合。

保守担当者向けメモ

  • 本関数は、バックエンドプロセスの初期化フェーズにおいて、認証前の通信プロトコルネゴシエーションの一部として実行されます。
  • secure_open_server が失敗した場合、既にTLSアラートが送信済みであると仮定して処理を終了します。
  • 共有メモリへのアクセスは許可されていないフェーズであるため、エラー処理には ereport を使用してください。
