spg_text_picksplit

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgtextproc.c:332

目的

SP-GiST 索引において、ページ分割時に複数の文字列データを共通接頭辞に基づいてグループ化し、ノードへ分配するためのロジックを提供します。

シグネチャまたは形状

Datum spg_text_picksplit(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgPickSplitIn（入力データ群）および spgPickSplitOut（出力構造体）を受け取ります。
  • **出力**: void。結果は out 引数を通じて返されます。
  • out->hasPrefix: 共通接頭辞が存在するか。
  • out->prefixDatum: 抽出された共通接頭辞。
  • out->nNodes: 生成されたノード数。
  • out->nodeLabels: 各ノードのラベル（バイト値）。
  • out->mapTuplesToNodes: 各入力タプルがどのノードに属するかのマッピング。
  • out->leafTupleDatums: リーフノードに格納される残りの文字列データ。

前提条件と不変条件

  • 入力データは text 型である必要があります。
  • 共通接頭辞の長さは SPGIST_MAX_PREFIX_LENGTH を超えないよう制限されます。
  • ノードラベル -1 は、接頭辞以降に文字が存在しない（文字列が共通接頭辞と一致する）ことを示します。

アルゴリズムまたは振る舞い

1. **共通接頭辞の特定**: 全入力データ間で最長の共通接頭辞を計算します。 2. **接頭辞の設定**: commonLen が 0 より大きければ out->prefixDatum に設定します。 3. **ノードラベルの抽出**: 各文字列の共通接頭辞の直後のバイトをノードラベルとして抽出します。 4. **ソート**: cmpNodePtr を使用してノードをラベル順にソートし、バイナリサーチを可能にします。 5. **結果の構築**: ソートされた順序に基づき、ノードラベルとリーフタプルを割り当て、out 構造体に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、本関数を直接呼び出す具体的な関数名は記載されていません。SP-GiST のインデックス構築プロセスから呼び出されます。
  • **呼び出し先**:
  • commonPrefix: 共通接頭辞の計算に使用。
  • formTextDatum: 接頭辞およびリーフデータの生成に使用。
  • qsort: ノードのソートに使用。
  • palloc: メモリ確保に使用。

関連データ構造

  • spgPickSplitIn, spgPickSplitOut: SP-GiST インターフェース定義。
  • spgNodePtr: ノードラベルと元のタプルインデックスを保持する内部構造体。

失敗モードと境界条件

  • 文字列がすべて共通である場合、ノードラベルとして -1 が使用されます。
  • commonLen が 0 の場合、接頭辞は設定されません。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は text 型の基数木実装の核となります。ロケール依存の比較を行う場合、パフォーマンスへの影響を考慮してください。
  • nodes[i].c = -1 の扱いは、文字列の終端を表現する特殊なケースとして重要です。
