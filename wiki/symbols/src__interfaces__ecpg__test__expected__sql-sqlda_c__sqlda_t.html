sqlda_t

定義

  • 種別: struct（typedef による別名）
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-sqlda.c:34

目的

ECPG（Embedded SQL for C）において、動的 SQL の実行時にパラメータのバインドや結果セットの取得を行うための SQL Descriptor Area（SQLDA）を表現する構造体です。

シグネチャまたは形状

typedef struct sqlda_compat sqlda_t;

※ _ECPG_INFORMIX_H マクロの定義有無により、内部構造は sqlda-compat.h または sqlda-native.h の定義に切り替わります。

入力と出力

  • **入力**: 動的 SQL 実行時のバインド変数として使用されます。
  • **出力**: FETCH 文などの実行結果を格納するコンテナとして使用されます。

前提条件と不変条件

  • 使用前に malloc 等によるメモリ確保が必要です。
  • sqlvar メンバを通じて、各列のデータ型や値のアドレスを適切にマッピングする必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: malloc でメモリを確保し、memset でゼロクリアします。 2. **バインド/フェッチ**: ECPGdo() 関数に引数として渡され、動的 SQL の実行結果の受け渡しや、入力パラメータのバインドに使用されます。 3. **データ解析**: dump_sqlda() 関数が内部の sqlvar 配列を走査し、sqltype に基づいて値をキャストして標準出力へ書き出します。 4. **解放**: 使用後は DEALLOCATE 等で関連リソースを解放し、free でメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: compat_informix-sqlda.c 等のテストコード内で、動的 SQL 処理の際にインスタンス化および参照が行われています。
  • **呼び出し先**: ECPGdo()（ECPG ライブラリ関数）が本構造体を操作します。また、dump_sqlda() が本構造体の内容を解析します。

関連データ構造

  • sqlvar_t: sqlda_t 内で各列の情報を保持する構造体。
  • sqlca_t: ECPG のエラー情報を保持する構造体。

失敗モードと境界条件

  • メモリ割り当て失敗時（malloc の戻り値が NULL の場合）のハンドリングが必要です。
  • 構造体の定義が環境（Informix 互換モードかネイティブモードか）に依存するため、ビルド時のマクロ設定と不整合がある場合に予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は ECPG プリプロセッサが生成するコードの期待値として使用されています。
  • 新しいデータ型をサポートする際は、dump_sqlda 関数内の型判定ロジックを更新する必要があります。
  • 構造体のレイアウトは sqlda-compat.h および sqlda-native.h で定義されているため、詳細なメンバ構成を確認する場合はそちらを参照してください。
