eliminate_duplicate_dependencies

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/dependency.c:2397

目的

ObjectAddresses リスト内の重複するオブジェクト参照を排除し、リストを整理する。

シグネチャまたは形状

static void eliminate_duplicate_dependencies(ObjectAddresses *addrs)

入力と出力

  • **入力**: ObjectAddresses *addrs （処理対象のオブジェクトアドレスリスト）
  • **出力**: なし（引数の addrs 構造体内の refs 配列と numrefs が直接更新される）

前提条件と不変条件

  • addrs->extras が NULL であること（補助データが存在するとソートによる同期が維持できないため）。
  • addrs->refs に格納されたオブジェクトアドレスは、object_address_comparator でソート可能であること。

アルゴリズムまたは振る舞い

1. リストの要素数が1以下の場合は何もせず終了する。 2. qsort を使用し、object_address_comparator に基づいて addrs->refs をソートし、重複する可能性のある要素を隣接させる。 3. ソートされたリストを走査し、以下の基準で重複を排除する：

  • classId、objectId、objectSubId がすべて一致する場合は重複とみなしてスキップする。
  • classId と objectId が一致し、かつ一方が objectSubId == 0（オブジェクト全体）、もう一方が objectSubId != 0（オブジェクトの一部）である場合、全体参照を部分参照で置き換える（例: テーブルとカラムの両方が指定されている場合、カラム指定のみを残す）。

    • 4. 最終的な有効要素数を addrs->numrefs に再設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/dependency.c 内の複数箇所（findDependentObjects の実行結果整理時など）。
  • **呼び出し先**:
  • qsort（標準ライブラリ）
  • object_address_comparator（比較関数）

関連データ構造

  • ObjectAddresses: 削除対象オブジェクトのリストを保持する構造体。
  • ObjectAddress: 個々のオブジェクトを識別する構造体（classId, objectId, objectSubId を保持）。

失敗モードと境界条件

  • addrs->extras が設定されている場合に Assert 失敗（プログラムの異常終了）が発生する。
  • リストが空、または単一要素の場合は処理をスキップする。

保守担当者向けメモ

  • 本関数は削除順序の安定化に寄与しており、回帰テストの結果を一定にするために重要である。
  • objectSubId の処理ロジックは、オブジェクトの階層構造（全体と部分）に依存しているため、新しいオブジェクトタイプを追加する際は、この階層関係が適切に扱われているか確認が必要である。
