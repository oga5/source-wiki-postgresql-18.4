array_get_element_expanded

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/arrayfuncs.c:1920

目的

拡張配列（Expanded Array）から、指定された添字に対応する要素を抽出します。

シグネチャまたは形状

Datum array_get_element_expanded(Datum arraydatum,
                                 int nSubscripts, int *indx,
                                 int arraytyplen,
                                 int elmlen, bool elmbyval, char elmalign,
                                 bool *isNull)

入力と出力

  • **入力**:
  • arraydatum: 拡張配列の Datum。
  • nSubscripts: 添字の数。
  • indx: 添字の配列。
  • arraytyplen, elmlen, elmbyval, elmalign: 配列および要素の型情報。
  • **出力**:
  • 戻り値: 指定された位置の要素（Datum）。
  • isNull: 要素が NULL の場合、または添字が範囲外の場合に true が設定される。

前提条件と不変条件

  • arraydatum は ExpandedArrayHeader を指す有効なポインタであること。
  • arraytyplen は -1 であること。
  • elmlen, elmbyval, elmalign は、対象の拡張配列のメタデータと一致していること。

アルゴリズムまたは振る舞い

1. DatumGetEOHP を用いて ExpandedArrayHeader を取得し、マジックナンバーを検証します。 2. 次元数および各次元の境界（下限・上限）に基づき、入力された添字が有効範囲内か確認します。範囲外の場合は isNull を true にして終了します。 3. ArrayGetOffset を用いて、多次元添字を線形オフセットに変換します。 4. deconstruct_expanded_array を呼び出し、配列を分解して dvalues および dnulls 配列を準備します。 5. dnulls を確認し、対象要素が NULL であれば isNull を true にして終了します。 6. 正常な場合、dvalues から該当するオフセットの要素を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arrayfuncs.c 内の配列操作関数から呼び出されます（詳細はソースコード内の参照箇所を参照）。
  • **呼び出し先**: deconstruct_expanded_array（配列の分解）、ArrayGetOffset（オフセット計算）。

関連データ構造

  • ExpandedArrayHeader: 拡張配列のメモリ上のヘッダー構造体。
  • Datum: PostgreSQL の汎用データ表現。

失敗モードと境界条件

  • **添字範囲外**: ndim が nSubscripts と一致しない、または添字が下限・上限の範囲外である場合、isNull を true として (Datum) 0 を返します。
  • **NULL 要素**: 対象要素が NULL である場合、isNull を true として (Datum) 0 を返します。

保守担当者向けメモ

  • 本関数は拡張配列専用のアクセサです。通常のフラットな配列（ArrayType）に対する array_get_element とは処理経路が異なります。
  • 戻り値として参照渡し（pass-by-ref）の値を返す場合、そのポインタは拡張配列の生存期間中のみ有効であるという前提で動作します。
