load_categories_hash

定義

  • 種別: function
  • 定義箇所: contrib/tablefunc/tablefunc.c:704

目的

クロス集計（crosstab）処理において、カテゴリ名と属性インデックスをマッピングするためのハッシュテーブルを構築し、初期化する。

シグネチャまたは形状

static HTAB *load_categories_hash(char *cats_sql, MemoryContext per_query_ctx)

入力と出力

  • **入力**:
  • cats_sql: カテゴリ名を取得するための SQL クエリ文字列。
  • per_query_ctx: ハッシュテーブルおよびエントリのメモリ割り当てに使用するメモリコンテキスト。
  • **出力**:
  • 構築された HTAB * 型のハッシュテーブル。

前提条件と不変条件

  • cats_sql は、単一の列（カテゴリ名）を返すクエリである必要がある。
  • カテゴリ名に NULL を含めることはできない。
  • ハッシュテーブルの各エントリは crosstab_HashEnt 構造体として管理される。

アルゴリズムまたは振る舞い

1. hash_create を使用して、指定された per_query_ctx 上にハッシュテーブルを初期化する。 2. SPI_connect を呼び出し、SPI 経由で cats_sql を実行する。 3. クエリ結果が 1 列であることを確認し、各行をループ処理する。 4. 各行からカテゴリ名を取得し、crosstab_cat_desc 構造体に格納してハッシュテーブルに挿入する。 5. SPI_finish を呼び出して SPI 接続を終了し、構築したハッシュテーブルを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/tablefunc/tablefunc.c 内の crosstab_hash 関数から呼び出される。
  • **呼び出し先**: hash_create, SPI_connect, SPI_execute, SPI_getvalue, MemoryContextSwitchTo, palloc, crosstab_HashTableInsert, SPI_finish, ereport, elog。

関連データ構造

  • HTAB: PostgreSQL のハッシュテーブル構造体。
  • crosstab_HashEnt: ハッシュテーブルのエントリ構造体。
  • crosstab_cat_desc: カテゴリ名と属性インデックスを保持する構造体。

失敗モードと境界条件

  • SPI_execute が失敗した場合や、クエリ結果が 1 列でない場合は ereport(ERROR) により処理が中断される。
  • カテゴリ名が NULL の場合、ereport(ERROR) が発生する。
  • SPI_finish が正常終了しない場合、elog(ERROR) により異常終了する。

保守担当者向けメモ

  • 本関数は SPI を使用するため、呼び出し元で適切なメモリコンテキスト管理が行われていることを前提としている。
  • INIT_CATS マクロによる初期サイズ設定が適切かどうかは、対象データのカテゴリ数に応じて検討が必要。
