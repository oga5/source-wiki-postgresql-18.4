ApplyRetrieveRule

定義

  • 種別: function
  • 定義箇所: src/backend/rewrite/rewriteHandler.c:1745

目的

クエリ内のリレーションに対する ON SELECT ルール（ビューの定義など）を適用し、クエリツリーを書き換えて展開する。

シグネチャまたは形状

Query *ApplyRetrieveRule(Query *parsetree, RewriteRule *rule, int rt_index, Relation relation, List *activeRIRs)

入力と出力

  • **入力**:
  • parsetree: 処理対象のクエリツリー。
  • rule: 適用する ON SELECT ルール。
  • rt_index: 範囲テーブル（RangeTable）内の対象リレーションのインデックス。
  • relation: 対象リレーションのディスクリプタ。
  • activeRIRs: 再帰的なビュー展開を防止するためのアクティブリスト。
  • **出力**: 書き換え後の Query 構造体。

前提条件と不変条件

  • ルールのアクションは1つであること。
  • ルールに qual（条件式）が含まれていないこと。
  • restrict_nonsystem_relation_kind 設定により、非システムビューへのアクセスが制限されている場合はエラーを投げる。

アルゴリズムまたは振る舞い

1. **ターゲットリレーションの処理**: 対象リレーションが UPDATE/DELETE/MERGE の結果リレーションである場合、ビューの展開と同時に元のリレーションを保持するための RTE（RangeTblEntry）のコピーを作成し、returningList の変数を調整する。 2. **ロックの確保**: AcquireRewriteLocks を呼び出し、ルールアクション内のリレーションに対して必要なロックを確保する。 3. **行レベルロックの伝播**: FOR [KEY] UPDATE/SHARE 句が存在する場合、markQueryForLocking を用いてルールアクション内のテーブルにロック属性を伝播させる。 4. **再帰的展開**: fireRIRrules を呼び出し、ビュー内のさらなるビュー参照を再帰的に展開する。 5. **サブクエリへの変換**: 対象の RTE を RTE_SUBQUERY に変更し、展開された rule_action をサブクエリとして設定する。 6. **整合性維持**: ビューの列数と RTE の列名リストが一致するようにダミー列名を追加する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/rewrite/rewriteHandler.c 内の fireRIRrules 等から呼び出される（ソースコード内のコメントおよび参照より）。
  • **呼び出し先**: copyObject, rt_fetch, AcquireRewriteLocks, markQueryForLocking, fireRIRrules, ExecCleanTargetListLength, lappend 等。

関連データ構造

  • Query: クエリツリーのルート構造体。
  • RewriteRule: ルール定義。
  • RangeTblEntry: 範囲テーブルエントリ。
  • activeRIRs: 再帰防止用のリスト。

失敗モードと境界条件

  • ルールアクションが1つでない場合、または qual が存在する場合は elog(ERROR) を発生させる。
  • 非システムビューへのアクセス制限に抵触した場合は ereport(ERROR) を発生させる。
  • 未知の commandType が渡された場合はエラーとなる。

保守担当者向けメモ

  • 本関数はビューの展開（RIR: Retrieve-Instead-Retrieve）の中核を担う。
  • ChangeVarNodes によるツリーの直接変更を行うため、操作前に copyObject を用いた深いコピーが必須である。
  • fireRIRrules との相互再帰的な関係に注意すること。
