InitializeTableSpaceCache

定義

  • 種別: function
  • 定義箇所: src/backend/utils/cache/spccache.c:77

目的

テーブルスペース情報をキャッシュするためのハッシュテーブル（TableSpaceCacheHash）を初期化し、システムカタログの更新イベントに対する無効化コールバックを登録します。

シグネチャまたは形状

static void InitializeTableSpaceCache(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数 TableSpaceCacheHash の初期化およびキャッシュ無効化コールバックの登録を行う）

前提条件と不変条件

  • CacheMemoryContext が存在しない場合は、本関数内で CreateCacheMemoryContext() が呼び出され、メモリコンテキストが作成されます。
  • TableSpaceCacheHash は本関数によって生成されるハッシュテーブルを保持します。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体を用いて、キーサイズを Oid、エントリサイズを TableSpaceCacheEntry に設定します。 2. hash_create を呼び出し、TableSpaceCacheHash を生成します（初期サイズ16、HASH_ELEM および HASH_BLOBS フラグを指定）。 3. CacheMemoryContext が未初期化であれば CreateCacheMemoryContext() を実行します。 4. CacheRegisterSyscacheCallback を呼び出し、TABLESPACEOID に対する変更を監視するコールバックとして InvalidateTableSpaceCacheCallback を登録します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/spccache.c 内のキャッシュアクセス関数（get_tablespace 等）から、キャッシュの遅延初期化のために呼び出されます。
  • 呼び出し先: hash_create, CreateCacheMemoryContext, CacheRegisterSyscacheCallback。

関連データ構造

  • TableSpaceCacheHash: キャッシュの実体であるハッシュテーブル。
  • TableSpaceCacheEntry: ハッシュテーブル内の各エントリの構造体。
  • CacheMemoryContext: キャッシュ用メモリコンテキスト。

失敗モードと境界条件

  • メモリ不足時には hash_create や CreateCacheMemoryContext がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数はテーブルスペースキャッシュの初回アクセス時に一度だけ実行されることを想定しています。
  • TABLESPACEOID システムカタログの更新時に InvalidateTableSpaceCacheCallback がトリガーされる仕組みとなっており、キャッシュの一貫性を維持します。
