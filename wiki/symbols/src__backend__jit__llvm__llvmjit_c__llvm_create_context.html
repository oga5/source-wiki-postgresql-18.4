llvm_create_context

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/llvmjit.c:222

目的

LLVM JIT コンパイルに使用する LLVMJitContext を生成し、リソースオーナーへ登録することで、トランザクション終了時のメモリ管理を保証する。

シグネチャまたは形状

LLVMJitContext *llvm_create_context(int jitFlags)

入力と出力

  • **入力**: jitFlags (JIT コンパイルの動作を制御するフラグ)
  • **出力**: 初期化された LLVMJitContext 構造体へのポインタ。

前提条件と不変条件

  • llvm_assert_in_fatal_section() により、致命的なエラーセクション内であることを確認する。
  • TopMemoryContext 上にメモリが割り当てられる。
  • 生成されたコンテキストは、現在のリソースオーナー（CurrentResourceOwner）に紐付けられる。

アルゴリズムまたは振る舞い

1. llvm_session_initialize() を呼び出し、セッションの初期化を行う。 2. llvm_recreate_llvm_context() を呼び出し、LLVM コンテキストを再生成する。 3. ResourceOwnerEnlarge() を呼び出し、リソースオーナーの容量を確保する。 4. TopMemoryContext から LLVMJitContext 分のメモリをゼロ初期化して割り当てる。 5. 入力された jitFlags をコンテキストに設定する。 6. ResourceOwnerRememberJIT() を呼び出し、リソースオーナーにコンテキストを登録してクリーンアップを保証する。 7. llvm_jit_context_in_use_count をインクリメントする。 8. 生成したコンテキストを返す。

関連データ構造

  • LLVMJitContext: JIT コンパイル状態を保持する主要構造体。
  • ResourceOwner: トランザクション単位のリソース管理を行う構造体。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocZero がエラーを発生させる。
  • llvm_session_initialize や llvm_recreate_llvm_context 内部で LLVM 関連の致命的なエラーが発生した場合、PostgreSQL のエラー処理系へ伝播する。

保守担当者向けメモ

  • 本関数は TopMemoryContext を使用するため、明示的な解放（llvm_release_context 等）を行わないとメモリリークの原因となる。リソースオーナーによる管理が必須である。
