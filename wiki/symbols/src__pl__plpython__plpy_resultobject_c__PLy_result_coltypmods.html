PLy_result_coltypmods

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_resultobject.c:191

目的

PLyResult オブジェクトが保持するクエリ結果の各列に対応する型修飾子（typmod）を、Pythonのリストとして取得する。

シグネチャまたは形状

static PyObject *PLy_result_coltypmods(PyObject *self, PyObject *unused)

入力と出力

  • **入力**:
  • self: PLyResultObject へのポインタ（Pythonオブジェクトとして渡される）。
  • unused: 使用されない引数（METH_NOARGS 呼び出し用）。
  • **出力**:
  • 成功時: 各列の型修飾子を格納した PyListObject。
  • 失敗時: NULL（エラー発生時）。

前提条件と不変条件

  • self は PLyResultObject 型である必要がある。
  • ob->tupdesc が有効（NULLでない）であること。

アルゴリズムまたは振る舞い

1. ob->tupdesc が存在するか確認し、存在しない場合は PLy_exc_error を送出し NULL を返す。 2. ob->tupdesc->natts（列数）のサイズで新しいPythonリストを作成する。 3. 各列に対してループ処理を行い、TupleDescAttr を用いて属性情報を取得する。 4. 取得した atttypmod を PyLong_FromLong でPython整数に変換し、リストの各インデックスに格納する。 5. 完成したリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: Pythonインタープリタ（PLyResult オブジェクトのメソッドとして呼び出される）。src/pl/plpython/plpy_resultobject.c:30 にてメソッドテーブルに登録されている。
  • **呼び出し先**:
  • PLy_exception_set（エラー時）
  • PyList_New
  • TupleDescAttr
  • PyLong_FromLong
  • PyList_SetItem

関連データ構造

  • PLyResultObject: クエリ結果を保持するC構造体。
  • TupleDesc: PostgreSQLの行記述子。

失敗モードと境界条件

  • ob->tupdesc が NULL の場合: 「command did not produce a result set」というエラーメッセージと共に例外を送出する。
  • メモリ不足: PyList_New が失敗した場合、NULL を返す。

保守担当者向けメモ

  • 本関数は METH_NOARGS として定義されており、Python側からは引数なしで呼び出されることを前提としている。
  • atttypmod は PostgreSQL の pg_attribute 構造体から取得される。型修飾子の意味については PostgreSQL のカタログ定義を参照のこと。
