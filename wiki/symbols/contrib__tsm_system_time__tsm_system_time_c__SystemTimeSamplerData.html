SystemTimeSamplerData

定義

  • 種別: struct
  • 定義箇所: contrib/tsm_system_time/tsm_system_time.c:45

目的

TABLESAMPLE SYSTEM_TIME メソッドにおけるサンプリング実行中の状態を保持し、時間制限に基づいたブロックサンプリングの進捗を管理する。

シグネチャまたは形状

typedef struct {
    uint32      seed;           /* 乱数シード */
    double      millis;         /* サンプリングの時間制限 */
    instr_time  start_time;     /* スキャン開始時刻 */
    OffsetNumber lt;            /* 現在のブロックから最後に返されたタプル */
    BlockNumber doneblocks;     /* 走査済みブロック数 */
    BlockNumber lb;             /* 最後に訪問したブロック */
    BlockNumber nblocks;        /* リレーションの総ブロック数 */
    BlockNumber firstblock;     /* サンプリング開始ブロック */
    BlockNumber step;           /* ステップサイズ（未設定時は0） */
} SystemTimeSamplerData;

入力と出力

  • 本構造体は palloc0 によってメモリ確保され、スキャン状態として保持される。
  • nblocks, firstblock, step は再スキャン（rescan）時にも変更されない不変値として扱われる。

前提条件と不変条件

  • step が 0 の場合、まだサンプリングのステップサイズが計算されていない状態を示す。
  • nblocks はリレーションの総ブロック数と一致する必要がある。

アルゴリズムまたは振る舞い

  • 時間経過を監視し、指定された millis に達するまでブロックを巡回する。
  • ブロックの選択には、nblocks と互いに素な step を用いた線形合同法的なアプローチを採用し、リレーション全体を網羅する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/tsm_system_time/tsm_system_time.c 内の system_time_initsamplescan（メモリ確保）
  • system_time_beginsamplescan（キャストして利用）
  • system_time_nextsampleblock（キャストして利用）
  • system_time_nextsampletuple（キャストして利用）
  • 呼び出し先:
  • 本構造体はデータ保持用であり、関数を直接呼び出すことはない。

関連データ構造

  • TsmRoutine: テーブルサンプリング API のルーチン定義。
  • instr_time: PostgreSQL の時間計測用型。

失敗モードと境界条件

  • 時間制限に達した場合、または全ブロックを走査し終えた場合、スキャンは終了する。
  • step が適切に計算されない場合、サンプリングの網羅性に影響が出る可能性がある。

保守担当者向けメモ

  • 本構造体は TABLESAMPLE の実行状態を保持するため、再スキャン時の挙動（rescan）において、どのフィールドがリセットされ、どのフィールドが維持されるかを厳密に管理すること。
  • step の計算ロジックは nblocks に依存するため、リレーションのサイズ変更が発生する環境下での整合性に注意が必要。
