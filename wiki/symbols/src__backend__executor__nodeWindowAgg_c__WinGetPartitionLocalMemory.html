WinGetPartitionLocalMemory

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeWindowAgg.c:3283

目的

ウィンドウ関数がパーティションの処理中に使用する、永続的なローカルメモリ領域を確保または取得します。

シグネチャまたは形状

void *WinGetPartitionLocalMemory(WindowObject winobj, Size sz)

入力と出力

  • **入力**:
  • winobj: ウィンドウ関数の実行コンテキストを保持する WindowObject。
  • sz: 確保するメモリ領域のサイズ（バイト単位）。
  • **出力**:
  • 確保または既存のメモリ領域へのポインタ。

前提条件と不変条件

  • winobj は WindowObjectIsValid マクロにより有効であることが保証されている必要があります。
  • メモリは winobj->winstate->partcontext（パーティションごとのメモリコンテキスト）から割り当てられます。

アルゴリズムまたは振る舞い

1. winobj->localmem が既に割り当て済みかを確認します。 2. 未割り当て（NULL）の場合、MemoryContextAllocZero を使用して sz バイトのゼロ初期化されたメモリを確保し、winobj->localmem に格納します。 3. 既に割り当て済みの場合は、既存のポインタをそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/windowfuncs.c 内の各種ウィンドウ関数（rank, ntile 等）から呼び出されます。
  • **呼び出し先**: MemoryContextAllocZero（PostgreSQLのメモリ管理API）。

関連データ構造

  • WindowObject: ウィンドウ関数の実行状態を管理する構造体。localmem メンバを介してメモリ状態を保持します。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocZero がエラーを発生させ、処理が中断されます。
  • sz に 0 を指定した場合の挙動は MemoryContextAllocZero の仕様に依存します。

保守担当者向けメモ

  • 本関数は、ウィンドウ関数がパーティションをまたいで状態を保持するための仕組みです。
  • 確保されたメモリは partcontext に紐付いているため、パーティション終了時に自動的に解放されます。
