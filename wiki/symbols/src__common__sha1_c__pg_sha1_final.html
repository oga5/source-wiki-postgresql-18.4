pg_sha1_final

定義

  • 種別: function
  • 定義箇所: src/common/sha1.c:364

目的

SHA-1 ハッシュ計算の最終処理を行い、メッセージダイジェストを生成する。

シグネチャまたは形状

void pg_sha1_final(pg_sha1_ctx *ctx, uint8 *dest)

入力と出力

  • ctx: SHA-1 コンテキスト構造体へのポインタ。計算状態を保持する。
  • dest: 最終的な 20 バイトのダイジェスト値を格納するためのバッファへのポインタ。

前提条件と不変条件

  • ctx は pg_sha1_init によって初期化され、pg_sha1_update によってデータが供給されている必要がある。
  • dest は少なくとも 20 バイトの書き込み可能なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. sha1_pad(ctx) を呼び出し、RFC 3174 に基づくパディング処理（メッセージ長情報の付与など）をコンテキストに適用する。 2. sha1_result(dest, ctx) を呼び出し、コンテキスト内の最終的なハッシュ状態を dest が指すメモリ領域に書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/cryptohash.c:193 で確認されている。
  • **呼び出し先**: sha1_pad および sha1_result を呼び出す。

関連データ構造

  • pg_sha1_ctx: sha1_int.h で定義される SHA-1 計算用の内部コンテキスト構造体。

失敗モードと境界条件

  • dest が NULL または十分な領域を確保していない場合、メモリ破壊が発生する可能性がある。本関数自体はエラーコードを返さない。

保守担当者向けメモ

  • 本関数は RFC 3174 に準拠したフォールバック実装である。
  • WORDS_BIGENDIAN マクロによりエンディアン処理が分岐するため、移植性を考慮する際は注意すること。
  • 現代的なハードウェアアクセラレーション（SHA-NI 等）は含まれていない。
