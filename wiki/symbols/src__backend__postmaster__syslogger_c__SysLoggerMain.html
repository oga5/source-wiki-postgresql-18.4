SysLoggerMain

定義

  • **種別**: function
  • **定義箇所**: src/backend/postmaster/syslogger.c:164

目的

PostgreSQL のログ収集プロセス（Syslogger）のメインループを実行し、全バックエンドプロセスからのログ出力を集約・管理する。

シグネチャまたは形状

void SysLoggerMain(const void *startup_data, size_t startup_data_len)

入力と出力

  • **入力**:
  • startup_data: プロセス起動時に渡されるデータ（EXEC_BACKEND 環境下では SysloggerStartupData 構造体）。
  • startup_data_len: データのサイズ。
  • **出力**: なし（プロセス終了までループを継続）。

前提条件と不変条件

  • SysLogger_Start によってパイプおよびログファイルが初期化されていること。
  • PostmasterContext が存在する場合、本関数内で解放される。
  • 終了処理において、全アップストリームプロセスが終了し、パイプが EOF になるまでプロセスは生存し続ける。

アルゴリズムまたは振る舞い

1. **初期化**: 起動データからファイル記述子を復元し、PostmasterContext を解放。 2. **環境設定**: stderr を /dev/null にリダイレクトし、不要なパイプ書き込み端を閉じる。 3. **シグナル設定**: SIGHUP（設定再読み込み）や SIGUSR1（ログローテーション）を登録し、終了シグナルを無視する。 4. **スレッド起動**: Windows 環境ではデータ転送用スレッド (pipeThread) を生成。 5. **メタデータ更新**: ログファイル名やローテーション時間を初期化し、current_logfiles を更新。 6. **イベント待機**: WaitEventSet を作成し、MyLatch およびパイプからの読み込みイベントを監視するループへ移行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/postmaster/launch_backend.c（B_LOGGER プロセス起動時）。
  • **呼び出し先**:
  • syslogger_fdopen, MemoryContextDelete, init_ps_display, pqsignal, logfile_getname, set_next_rotation_time, update_metainfo_datafile, CreateWaitEventSet, AddWaitEventToSet 等。
  • Windows 環境では _beginthreadex を介して pipeThread を起動。

関連データ構造

  • SysloggerStartupData: 起動時に渡されるファイル記述子情報。
  • WaitEventSet: イベント監視用オブジェクト。
  • syslogPipe: プロセス間通信用パイプ。

失敗モードと境界条件

  • EXEC_BACKEND 環境下で syslogger_fdopen が失敗した場合、NULL ポインタ参照によりクラッシュする可能性がある。
  • Windows 環境でスレッド生成に失敗した場合、FATAL エラーでプロセスが終了する。

保守担当者向けメモ

  • 本プロセスは他の全プロセスが終了したことを確認するために、パイプの EOF を待機する設計となっている。
  • Windows では名前付きパイプの制限を回避するため、専用スレッドでデータ転送を行っている点に注意すること。
