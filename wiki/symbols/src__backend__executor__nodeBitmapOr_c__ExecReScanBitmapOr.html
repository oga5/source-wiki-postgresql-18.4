ExecReScanBitmapOr

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapOr.c:218

目的

BitmapOr ノードの再スキャン（ReScan）処理を行い、保持しているすべてのサブプランに対してパラメータ変更の伝播と再初期化を制御する。

シグネチャまたは形状

void ExecReScanBitmapOr(BitmapOrState *node)

入力と出力

  • **入力**: BitmapOrState *node （再スキャン対象の BitmapOr ノードの状態構造体）
  • **出力**: なし（副作用としてサブプランの状態を更新）

前提条件と不変条件

  • node は有効な BitmapOrState 構造体を指している必要がある。
  • node->bitmapplans には node->nplans 個のサブプラン状態が格納されている。

アルゴリズムまたは振る舞い

1. node->nplans 個のサブプランを順次走査する。 2. BitmapOr ノード自体にパラメータ変更（ps.chgParam）がある場合、UpdateChangedParamSet を呼び出し、各サブプランに対して変更を通知する。 3. 各サブプランの chgParam が NULL である（＝即時再スキャンが必要な）場合、ExecReScan を呼び出してサブプランを再初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:162 （ExecReScan 経由での呼び出しが確認されている）
  • **呼び出し先**:
  • UpdateChangedParamSet
  • ExecReScan

関連データ構造

  • BitmapOrState: BitmapOr ノードの実行状態を保持する構造体。
  • PlanState: 各サブプランの実行状態を保持する構造体。

失敗モードと境界条件

  • node が NULL の場合の挙動は定義されていない。
  • サブプラン数が 0 の場合、ループは実行されず何も行われない。

保守担当者向けメモ

  • ExecReScan は BitmapOr ノードの内部構造（サブプランの管理）を直接認識しないため、本関数内で明示的に UpdateChangedParamSet を呼び出し、パラメータ変更を伝播させる必要がある。
  • サブプランの chgParam が設定されている場合、そのプランは後続の ExecProcNode 呼び出し時に自動的に再スキャンされるため、本関数ではスキップされる。
