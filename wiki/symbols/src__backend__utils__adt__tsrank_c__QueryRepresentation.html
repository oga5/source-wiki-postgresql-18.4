QueryRepresentation

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/tsrank.c:555

目的

全文検索における近接度（Cover Density）ランキング計算時に、TSQuery の各要素と文書内の出現位置情報を関連付けて保持するための構造体です。

シグネチャまたは形状

typedef struct {
    TSQuery                     query;
    QueryRepresentationOperand *operandData;
} QueryRepresentation;

入力と出力

  • **入力**: TSQuery オブジェクトおよび、クエリ内の各オペランドに対応する QueryRepresentationOperand 配列。
  • **出力**: 本構造体自体が、ランキング計算プロセスにおけるクエリの内部表現として機能します。

前提条件と不変条件

  • query フィールドは有効な TSQuery を指している必要があります。
  • operandData は、クエリ内のオペランド数に対応するメモリ領域を確保している必要があります。

アルゴリズムまたは振る舞い

  • ts_rank_cd（近接度ランキング）の計算過程において、文書内の単語位置情報とクエリの構造を照合するために使用されます。
  • get_docrep 関数によって文書が解析される際、この構造体にクエリの適合情報が格納されます。
  • Cover 関数がこの構造体を参照し、クエリをカバーする文書内の範囲を再帰的に探索してスコアを算出します。

呼び出し元と呼び出し先

  • **呼び出し元**: ts_rank_cd 実装関数群、get_docrep、fillQueryRepresentationData、resetQueryRepresentation など。
  • **呼び出し先**: TS_execute コールバック（クエリのオペランドマッチング用）。

関連データ構造

  • TSQuery: 検索条件を保持するデータ構造。
  • QueryRepresentationOperand: クエリ内の各オペランドの出現位置や状態を管理する構造体。
  • DocRepresentation: 文書をクエリ適合形式に変換したデータ構造。
  • CoverExt: 近接度計算時の範囲情報を保持する構造体。

失敗モードと境界条件

  • クエリが空の場合や、文書内にクエリの単語が含まれない場合、ランキング計算はゼロスコアを返します。
  • 非常に複雑なクエリや長い文書の場合、Cover 関数の再帰処理によりスタックオーバーフローが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は tsrank.c 内の近接度ランキング専用の内部表現です。
  • resetQueryRepresentation を使用して、計算の合間に状態を適切にリセットする必要があります。
