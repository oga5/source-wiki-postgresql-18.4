add_outer_joins_to_relids

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinrels.c:792

目的

外部結合（Outer Join）の結合順序制約を考慮し、現在の結合リレーション集合（Relids）に、論理的に結合可能となった外部結合の識別子（ojrelid）を追加する。

シグネチャまたは形状

Relids add_outer_joins_to_relids(PlannerInfo *root, Relids input_relids,
                                 SpecialJoinInfo *sjinfo,
                                 List **pushed_down_joins)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • input_relids: 現在の結合リレーション集合。
  • sjinfo: 評価対象の外部結合に関する制約情報。
  • pushed_down_joins: 押し下げられた外部結合を記録するためのリスト（出力用）。
  • **出力**: 外部結合の ojrelid が追加された更新済みの Relids。

前提条件と不変条件

  • sjinfo が NULL または ojrelid が 0 の場合、本関数は input_relids を変更せずに返す。
  • JOIN_LEFT 以外の結合タイプでは、構文上の順序に従うため、ojrelid を単純に追加する。

アルゴリズムまたは振る舞い

1. **初期チェック**: sjinfo が有効な外部結合でない場合は即座に終了する。 2. **左結合の判定**: JOIN_LEFT でない場合は、input_relids に ojrelid を追加して終了する。 3. **制約チェック**: commute_below_l が input_relids のサブセットでない場合、結合の押し下げが不可能なため、何も追加せずに終了する。 4. **OJ追加**: sjinfo->ojrelid を input_relids に追加する。 5. **連鎖的な追加**: commute_above_l が存在する場合、root->join_info_list を走査し、条件（min_lefthand, min_righthand, commute_below_l がすべて input_relids に含まれること）を満たす他の外部結合があれば、それらも再帰的に input_relids へ追加する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/equivclass.c:1545, 1572
  • src/backend/optimizer/path/joinrels.c:725
  • **呼び出し先**:
  • bms_add_member, bms_is_subset, bms_is_member, bms_copy, bms_add_members（ビットマップ操作関数群）
  • lappend（リスト操作）

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • Relids: リレーションIDの集合（ビットマップ）。
  • SpecialJoinInfo: 外部結合の制約情報（ojrelid, jointype, commute_below_l, commute_above_l 等を保持）。

失敗モードと境界条件

  • sjinfo が NULL の場合、安全に早期リターンする。
  • 結合条件が満たされない場合、input_relids は更新されない。
  • 押し下げられた結合の連鎖において、join_info_list の順序がボトムアップであることを前提としており、一度の走査で解決する。

保守担当者向けメモ

  • 本関数は、外部結合の同一性（OJ Identity）に基づいた結合順序の柔軟性を管理している。
  • commute_above_l を介した連鎖的な追加処理は、複雑な外部結合のネストを正しく扱うために重要である。
