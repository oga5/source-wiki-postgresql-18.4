index_recheck_constraint

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execIndexing.c:980

目的

インデックスの制約（排他制約など）に対して、既存のタプル値と新しいタプル値を比較し、制約が維持されているかを再検証する。

シグネチャまたは形状

static bool index_recheck_constraint(Relation index, const Oid *constr_procs,
                                     const Datum *existing_values, const bool *existing_isnull,
                                     const Datum *new_values)

入力と出力

  • **入力**:
  • index: 対象のインデックスリレーション。
  • constr_procs: 制約チェックに使用する演算子関数の OID 配列。
  • existing_values: 既存タプルのキー値配列。
  • existing_isnull: 既存タプルの各属性が NULL かどうかを示すフラグ配列。
  • new_values: 新規挿入タプルのキー値配列。
  • **出力**:
  • bool: 制約を満たしていれば true、違反していれば false を返す。

前提条件と不変条件

  • 排他演算子は厳密（strict）であると仮定している。
  • existing_isnull のいずれかが true の場合、制約違反とはみなさず false を返す。

アルゴリズムまたは振る舞い

1. インデックスのキー属性数（indnkeyatts）分だけループする。 2. 各属性について、existing_isnull を確認し、NULL であれば即座に false を返す。 3. constr_procs に格納された演算子関数を OidFunctionCall2Coll で呼び出し、既存値と新規値を比較する。 4. 比較結果が false（制約違反）であれば、即座に false を返す。 5. 全属性のチェックを通過した場合、true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execIndexing.c 内の制約チェックロジックから呼び出される。
  • **呼び出し先**: OidFunctionCall2Coll（PostgreSQL の関数呼び出しインターフェース）。

関連データ構造

  • Relation: インデックスのメタデータ（rd_indcollation など）を参照するために使用。

失敗モードと境界条件

  • 演算子関数がエラーを発生させた場合、呼び出し元に伝播する。
  • 属性値が NULL の場合は制約チェックをスキップし false を返す。

保守担当者向けメモ

  • 本関数は排他制約の再チェックに特化している。演算子が厳密であるという前提に依存しているため、演算子の変更時には注意が必要。
