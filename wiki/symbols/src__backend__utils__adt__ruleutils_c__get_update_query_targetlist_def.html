get_update_query_targetlist_def

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/adt/ruleutils.c:7202

目的

UPDATE 文のターゲットリスト（SET 句の内容）を解析し、人間が読める SQL 文字列として再構築（デパース）します。特に、複数列代入（Multi-assignment）構文の復元をサポートします。

シグネチャまたは形状

static void get_update_query_targetlist_def(Query *query, List *targetList, deparse_context *context, RangeTblEntry *rte);

入力と出力

  • **入力**:
  • query: 対象となる Query 構造体。
  • targetList: UPDATE 文のターゲットリスト（TargetEntry のリスト）。
  • context: デパース状態を保持する deparse_context。
  • rte: 対象テーブルに対応する RangeTblEntry。
  • **出力**: なし（副作用として context->buf に SQL 文字列を追記）。

前提条件と不変条件

  • targetList 内の resjunk フラグが立っていないエントリが、UPDATE の対象列として出力されます。
  • 複数列代入（MULTIEXPR_SUBLINK）が含まれる場合、それらは targetList 内で ID 順に並んでいる必要があります。
  • 列名の解決には、tle->resname ではなく get_attname を使用し、システムカタログから最新の列名を取得します。

アルゴリズムまたは振る舞い

1. **複数列代入の準備**: targetList を走査し、MULTIEXPR_SUBLINK 型の resjunk エントリを抽出してリスト化します。 2. **ターゲットリストの反復処理**:

  • resjunk エントリをスキップします。
  • 複数列代入の開始を検知した場合、( を出力し、対応する SubLink を特定します。
  • get_attname を用いて列名を取得し、quote_identifier でエスケープして出力します。
  • processIndirection を呼び出し、フィールドアクセスや添字（サブスクリプト）を処理します。
  • 複数列代入の末尾に達した場合、) を出力し、代入式として SubLink を出力します。
  • 通常の代入の場合、 = を出力し、get_rule_expr を再帰的に呼び出して値をデパースします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/ruleutils.c 内の get_update_query_def および pg_get_ruledef_worker 等から呼び出されます。
  • **呼び出し先**:
  • get_attname, processIndirection, get_rule_expr, strip_implicit_coercions, appendStringInfoString, appendStringInfoChar 等。

関連データ構造

  • Query: SQL クエリの抽象構文木。
  • TargetEntry: ターゲットリストの各要素。
  • deparse_context: デパース処理の共有状態。
  • RangeTblEntry: テーブル参照情報。
  • SubLink: サブクエリや複数列代入の定義。

失敗モードと境界条件

  • get_attname が列名を取得できない場合、システムカタログとの不整合によりエラーが発生する可能性があります。
  • 複数列代入の Param ID が期待値と一致しない場合、Assert 失敗が発生します。

保守担当者向けメモ

  • 本関数は UPDATE 文の SET 句の再構築に特化しています。列の追加・削除・リネームが発生しても正しく SQL を復元できるよう、システムカタログを直接参照する設計になっています。
  • 複雑な FieldStore や SubscriptingRef が含まれる場合、processIndirection が適切に処理を行うため、この関数内ではそれらの構造を深く追う必要はありません。
