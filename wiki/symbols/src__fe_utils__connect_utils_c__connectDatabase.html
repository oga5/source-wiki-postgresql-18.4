connectDatabase

定義

  • 種別: function
  • 定義箇所: src/fe_utils/connect_utils.c:31

目的

PostgreSQLフロントエンドツールにおいて、ConnParams構造体に基づきデータベースへの接続を確立し、必要に応じて対話的なパスワード認証およびセキュリティ設定（search_pathの保護）を行う。

シグネチャまたは形状

PGconn *connectDatabase(const ConnParams *cparams, const char *progname,
                        bool echo, bool fail_ok, bool allow_password_reuse);

入力と出力

  • **入力**:
  • cparams: 接続パラメータ（dbnameは必須）。
  • progname: アプリケーション名。
  • echo: クエリ実行時にログ出力を行うか。
  • fail_ok: 接続失敗時にpg_fatalで終了せずNULLを返すか。
  • allow_password_reuse: 静的変数に保持されたパスワードを再利用するか。
  • **出力**: 確立されたPGconnポインタ、または失敗時にNULL（fail_okが真の場合）。

前提条件と不変条件

  • cparams->dbnameが供給されていること。
  • 接続成功後、デフォルトでALWAYS_SECURE_SEARCH_PATH_SQLが実行され、検索パスが保護される。

アルゴリズムまたは振る舞い

1. **パスワード管理**: allow_password_reuseが偽であれば既存の静的パスワードを解放。必要に応じてsimple_promptでパスワードを取得。 2. **接続試行**: PQconnectdbParamsを使用して接続を試行。 3. **認証ループ**: 接続失敗時にパスワードが必要と判定された場合、パスワードを再入力して接続を再試行。 4. **エラー判定**: 接続失敗時にfail_okが偽であればpg_fatalで終了。 5. **セキュリティ設定**: 接続成功後、executeQueryを用いてALWAYS_SECURE_SEARCH_PATH_SQLを実行し、結果を破棄。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/scripts/clusterdb.c
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c

関連データ構造

  • ConnParams: 接続先ホスト、ポート、ユーザー名、DB名等を保持する構造体。
  • PGconn: libpqの接続ハンドル。

失敗モードと境界条件

  • **メモリ不足**: PQconnectdbParamsがNULLを返した場合、pg_fatalで終了。
  • **接続失敗**: PQstatusがCONNECTION_BADの場合、fail_okフラグに従い終了するかNULLを返す。
  • **パスワード再利用**: 静的変数を使用するため、異なる接続先で誤って使用されないよう注意が必要。

保守担当者向けメモ

  • allow_password_reuseをtrueにする際は、接続先ホストやユーザーが同一であることを確認すること。
  • 接続直後に実行されるALWAYS_SECURE_SEARCH_PATH_SQLは、セキュリティ上の要件であるため変更時は注意が必要。
