HEAP_XMAX_IS_EXCL_LOCKED

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/htup_details.h:272

目的

ヒープタプルの infomask を検査し、タプルが排他的ロック（Exclusive Lock）状態にあるかどうかを判定します。

シグネチャまたは形状

static inline bool HEAP_XMAX_IS_EXCL_LOCKED(int16 infomask)

入力と出力

  • **入力**: infomask (タプルの t_infomask フィールド値)
  • **出力**: 排他的ロックが設定されている場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力される infomask は、ヒープタプルのヘッダーから取得された有効なビットマスクである必要があります。
  • HEAP_LOCK_MASK と HEAP_XMAX_EXCL_LOCK の定義に基づき、ビット演算による判定が行われます。

アルゴリズムまたは振る舞い

1. 引数 infomask に対して HEAP_LOCK_MASK を適用し、ロック関連のビットのみを抽出します。 2. 抽出された値が HEAP_XMAX_EXCL_LOCK と一致するかを比較し、その結果を真偽値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgrowlocks/pgrowlocks.c
  • src/backend/access/heap/heapam.c（複数箇所でタプルのロック状態確認に使用）
  • **呼び出し先**:
  • 本関数はインライン関数であり、特定の関数を呼び出すのではなく、マクロ定数（HEAP_LOCK_MASK, HEAP_XMAX_EXCL_LOCK）を参照します。

関連データ構造

  • HeapTupleHeaderData: タプルヘッダー構造体（t_infomask を保持）。
  • t_infomask: タプルの状態を示すビットフラグ。

失敗モードと境界条件

  • 不正な infomask 値が渡された場合、誤った判定結果を返す可能性があります。
  • 本関数自体は単純なビット比較であるため、関数内部での失敗モードはありません。

保守担当者向けメモ

  • 本関数は htup_details.h で定義されており、頻繁に呼び出されるためインライン化されています。
  • ロック状態の定義（HEAP_XMAX_EXCL_LOCK 等）が変更された場合、この判定ロジックも影響を受ける可能性があります。
