s_lock

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/s_lock.c:97

目的

プラットフォーム非依存のロジックを用いて、スピンロック（slock_t）の取得を試行し、競合発生時には待機および再試行を制御する。

シグネチャまたは形状

int s_lock(volatile slock_t *lock, const char *file, int line, const char *func)

入力と出力

  • **入力**:
  • lock: 取得対象のスピンロックへのポインタ。
  • file, line, func: ロック取得を要求したソースコードの場所情報（デバッグおよび統計用）。
  • **出力**:
  • 戻り値: ロック取得までにかかった総待機回数（delayStatus.delays）。

前提条件と不変条件

  • lock は volatile で修飾されており、ハードウェアによる原子操作（TAS）が可能な状態であること。
  • s_lock.h で定義される TAS_SPIN マクロが、対象アーキテクチャにおける Test-and-Set 操作を提供していること。

アルゴリズムまたは振る舞い

1. init_spin_delay を呼び出し、スピン待機用の状態構造体 SpinDelayStatus を初期化する。 2. TAS_SPIN(lock) が真（ロック中）である限り、perform_spin_delay を繰り返し呼び出して待機する。 3. TAS_SPIN が偽（ロック取得成功）となった時点でループを抜ける。 4. finish_spin_delay を呼び出し、今回の試行結果に基づいて次回のスピン閾値を更新する。 5. 最終的な待機回数を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/storage/s_lock.h 内の TAS_SPIN マクロ経由で呼び出される。
  • src/backend/storage/lmgr/s_lock.c 内のテストコード（test_lock_struct）から直接呼び出しあり。
  • **呼び出し先**:
  • init_spin_delay
  • TAS_SPIN (マクロ)
  • perform_spin_delay
  • finish_spin_delay

関連データ構造

  • slock_t: スピンロックの実体。
  • SpinDelayStatus: スピン待機の回数や統計情報を保持する構造体。

失敗モードと境界条件

  • **スタック検出**: perform_spin_delay 内部で一定回数（NUM_DELAYS）を超えてもロックが取得できない場合、致命的なエラー（PANIC）が発生する可能性がある。

保守担当者向けメモ

  • 本関数はスピンロックの「ソフトウェア側」の待機ロジックを担当する。ハードウェア依存の低レベルな TAS 操作は s_lock.h に委譲されている。
  • パフォーマンス最適化のため、spins_per_delay の調整ロジック（finish_spin_delay 内）と密接に関連している。
