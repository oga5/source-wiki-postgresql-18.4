gbt_float4_fetch

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_float4.c:121

目的

GiSTインデックス内の float4 型データに対して、インデックスエントリから元の値をフェッチ（取得）します。

シグネチャまたは形状

Datum gbt_float4_fetch(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（GISTENTRY 構造体へのポインタを第1引数として含む）
  • 出力: Datum（フェッチされた float4 値）

前提条件と不変条件

  • PG_GETARG_POINTER(0) により、有効な GISTENTRY 構造体が取得できること。
  • tinfo（gbtree_ninfo 型の静的変数）が float4 型の型情報および比較関数で適切に初期化されていること。

アルゴリズムまたは振る舞い

1. 引数から GISTENTRY 構造体を取得します。 2. btree_utils_num.c で定義されている汎用関数 gbt_num_fetch を呼び出します。 3. その際、float4 型固有の設定テーブルである tinfo を引数として渡し、型に応じたフェッチ処理を実行させます。 4. gbt_num_fetch の戻り値を PG_RETURN_POINTER を介して返します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLのGiSTインデックス管理機構（直接的な呼び出し元コードはソース内に明示されていません）。
  • 呼び出し先: gbt_num_fetch（btree_utils_num.c 内の関数）。

関連データ構造

  • GISTENTRY: GiSTインデックスのノードエントリを保持する構造体。
  • tinfo: float4 型の比較関数や型情報を集約した gbtree_ninfo 型の静的変数。

失敗モードと境界条件

  • entry が NULL の場合、または tinfo が正しく初期化されていない場合、gbt_num_fetch 内で予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_num.c の汎用ロジックを呼び出すためのラッパーです。ロジックの変更が必要な場合は、まず btree_utils_num.c の gbt_num_fetch を確認してください。
  • PG_FUNCTION_INFO_V1(gbt_float4_fetch) により、PostgreSQLの関数マネージャに登録されています。
