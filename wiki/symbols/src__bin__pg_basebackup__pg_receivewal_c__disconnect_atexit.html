disconnect_atexit

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_receivewal.c:67

目的

プログラム終了時に、PostgreSQL サーバーへのレプリケーション接続を安全に切断するために使用されます。

シグネチャまたは形状

static void disconnect_atexit(void)

入力と出力

  • 入力: なし
  • 出力: なし（戻り値なし）

前提条件と不変条件

  • グローバル変数 conn が有効な PGconn オブジェクトを指しているか、あるいは NULL である必要があります。
  • atexit() を介して登録されることを想定しています。

アルゴリズムまたは振る舞い

1. グローバル変数 conn が NULL でないかを確認します。 2. NULL でない場合、PQfinish(conn) を呼び出してサーバーとの接続をクローズします。

呼び出し元と呼び出し先

  • 呼び出し先: PQfinish (libpq)
  • 呼び出し元: atexit() を通じてプログラム終了時に呼び出されます。ソースコード内の直接的な呼び出し箇所は src/bin/pg_basebackup/pg_receivewal.c:835 で atexit(disconnect_atexit) として登録されています。

関連データ構造

  • PGconn: libpq の接続ハンドル（グローバル変数 conn として参照）。

失敗モードと境界条件

  • conn が NULL の場合、何もしません。
  • PQfinish 自体の内部的な失敗については、本関数内では制御していません。

保守担当者向けメモ

  • 本関数は pg_receivewal だけでなく、pg_basebackup や pg_recvlogical など複数のツールで同様の目的で定義・登録されています。修正時は各ツール間での一貫性に注意してください。
