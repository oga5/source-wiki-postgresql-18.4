TidHashKey

定義

  • 種別: struct
  • 定義箇所: src/backend/access/heap/rewriteheap.c:161

目的

ヒープ再構築処理において、更新チェーン（タプルの連鎖）を解決するために使用されるハッシュテーブルのキーとして機能します。古いヒープ内のタプルを一意に識別するために使用されます。

シグネチャまたは形状

typedef struct
{
	TransactionId xmin;			/* tuple xmin */
	ItemPointerData tid;		/* tuple location in old heap */
} TidHashKey;

入力と出力

  • 入力: xmin（トランザクションID）および tid（旧ヒープ内のタプル位置）。
  • 出力: ハッシュテーブルのキーとして利用され、対応するタプル情報の検索に使用されます。

前提条件と不変条件

  • xmin と tid の組み合わせにより、旧ヒープ内のタプルが特定可能である必要があります。
  • ハッシュテーブルのキーサイズとして sizeof(TidHashKey) が使用されます。

アルゴリズムまたは振る舞い

  • rewriteheap.c 内のハッシュテーブル操作において、特定のタプルが更新チェーンの一部であるかを確認するための照合キーとして利用されます。
  • rewrite_heap_tuple 等の処理フローにおいて、旧ヒープから新ヒープへのタプル移行時に、更新チェーンの整合性を維持するための検索キーとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/rewriteheap.c 内の各関数（rewrite_heap_tuple 等）が、ハッシュテーブル操作の際にこの構造体をインスタンス化して使用しています。
  • 呼び出し先: 直接的な関数呼び出しではなく、ハッシュテーブル管理関数（hash_search 等）の引数として渡されます。

関連データ構造

  • UnresolvedTupData: 本構造体をキーとして管理されるデータ構造。
  • RewriteStateData: 本構造体を利用するハッシュテーブルを保持する再構築状態構造体。

失敗モードと境界条件

  • ハッシュテーブルのキーとして使用されるため、メモリ不足時にはハッシュテーブルの構築や検索が失敗する可能性があります。
  • 非常に巨大なテーブルで未解決の更新チェーンが大量に滞留した場合、メモリ消費量が増大します。

保守担当者向けメモ

  • 本構造体は rewriteheap.c 内のハッシュテーブル専用です。変更を加える際は、ハッシュテーブルのキーサイズ計算（sizeof(TidHashKey)）との整合性に注意してください。
