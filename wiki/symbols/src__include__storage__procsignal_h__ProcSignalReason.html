ProcSignalReason

定義

  • 種別: enum
  • 定義箇所: src/include/storage/procsignal.h:30

目的

PostgreSQLのプロセス間通信（IPC）において、バックエンドプロセスや補助プロセスに対して送信される非同期シグナルの発生理由を識別するために使用されます。

シグネチャまたは形状

typedef enum {
    PROCSIG_CATCHUP_INTERRUPT,
    PROCSIG_NOTIFY_INTERRUPT,
    PROCSIG_PARALLEL_MESSAGE,
    PROCSIG_WALSND_INIT_STOPPING,
    PROCSIG_BARRIER,
    PROCSIG_LOG_MEMORY_CONTEXT,
    PROCSIG_PARALLEL_APPLY_MESSAGE,
    PROCSIG_RECOVERY_CONFLICT_FIRST,
    /* ... (リカバリ競合関連の列挙値) ... */
    PROCSIG_SLOTSYNC_MESSAGE
} ProcSignalReason;

入力と出力

  • 本シンボル自体は列挙型であり、関数への引数として渡されることで、受信側プロセスが実行すべき処理を特定する識別子として機能します。

前提条件と不変条件

  • SIGUSR1 シグナルを多重化して利用するため、同じ理由のシグナルが短時間に連続して送信された場合、受信側で一度しか観測されない可能性があります。この挙動を前提とした設計が必要です。

アルゴリズムまたは振る舞い

1. **送信**: SendProcSignal() を通じて、対象プロセスの共有メモリ上のスロットにフラグを立てる際に指定されます。 2. **受信**: 受信側プロセスの SIGUSR1 ハンドラ（procsignal_sigusr1_handler）が、共有メモリ上のフラグを確認し、対応する ProcSignalReason に基づいて処理を分岐させます。 3. **処理**: インデックスの無効化、クエリのキャンセル、バリア同期、リカバリ競合の解決など、理由に応じたアクションが実行されます。

呼び出し元と呼び出し先

  • **主な呼び出し元**:
  • src/backend/storage/ipc/procarray.c: CancelVirtualTransaction, SignalVirtualTransaction, CancelDBBackends
  • src/backend/storage/ipc/standby.c: SendRecoveryConflictWithBufferPin, LogRecoveryConflict
  • src/backend/storage/ipc/procsignal.c: SendProcSignal
  • **主な呼び出し先**:
  • src/backend/storage/ipc/procsignal.c: CheckProcSignal（理由の確認に使用）

関連データ構造

  • ProcSignalSlot: プロセスごとのシグナル状態を保持する共有メモリ上の構造体（procsignal.c で定義）。

失敗モードと境界条件

  • 理由が PROCSIG_RECOVERY_CONFLICT_FIRST から PROCSIG_RECOVERY_CONFLICT_LAST の範囲にある場合、リカバリ中の競合処理として扱われます。
  • 範囲外の理由や未定義の理由が渡された場合の挙動は、各ハンドラの実装に依存します。

保守担当者向けメモ

  • 新しいシグナル理由を追加する場合は、procsignal_sigusr1_handler 内の処理ロジックを更新し、受信側で適切に処理されることを確認してください。
  • SIGUSR1 の多重化によるイベント欠落の可能性を考慮し、状態の累積ではなく、最新の状態を反映する設計を維持してください。
