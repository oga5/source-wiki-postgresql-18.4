gtsquery_compress

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_gist.c:26

目的

GiST インデックス登録時、tsquery 型の値を固定長のビットマップ（TSQuerySign）に変換し、インデックスサイズを削減および最適化する。

シグネチャまたは形状

Datum gtsquery_compress(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して GISTENTRY 構造体を受け取る。
  • **出力**: Datum 型として、圧縮された TSQuerySign を含む GISTENTRY へのポインタを返す。

前提条件と不変条件

  • 入力された GISTENTRY がリーフノード（entry->leafkey が真）である場合にのみ圧縮処理が行われる。
  • リーフノードでない場合は、入力された entry がそのまま返される。

アルゴリズムまたは振る舞い

1. entry->leafkey を確認し、リーフノードであれば以下の処理を行う。

  • palloc を使用して新しい GISTENTRY をメモリ確保する。
  • DatumGetTSQuery で entry->key を tsquery 型に変換する。
  • makeTSQuerySign を呼び出し、tsquery からビットマップ（TSQuerySign）を生成する。
  • gistentryinit を使用し、生成したシグネチャを新しい GISTENTRY に初期化して格納する。

    • 2. リーフノードでない場合は、元の entry をそのまま保持する。 3. 最終的に PG_RETURN_POINTER を介して GISTENTRY へのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの構築プロセス（PostgreSQL の GiST インデックスインターフェース）。
  • **呼び出し先**:
  • makeTSQuerySign: tsquery からシグネチャを生成する。
  • gistentryinit: GISTENTRY 構造体を初期化する。
  • palloc: メモリ確保を行う。
  • DatumGetTSQuery: Datum を tsquery 型へキャストする。
  • TSQuerySignGetDatum: シグネチャを Datum 型へ変換する。

関連データ構造

  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。
  • TSQuerySign: tsquery を表現するビットマップ型。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる可能性がある。
  • entry->leafkey が偽の場合、圧縮はスキップされ、元のエントリがそのまま返される。

保守担当者向けメモ

  • 本関数は GiST インデックスの compress メソッドとして登録されることを想定している。
  • makeTSQuerySign によるシグネチャ生成ロジックが変更された場合、インデックスの再構築が必要になる可能性がある。
