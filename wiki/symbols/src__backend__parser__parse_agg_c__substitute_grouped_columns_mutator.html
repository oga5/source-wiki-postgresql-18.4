substitute_grouped_columns_mutator

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_agg.c:1381

目的

クエリ内の式ツリーを走査し、グループ化された列（GROUP BY 句に含まれる列や関数的に依存する列）を RTE_GROUP への参照に置換します。また、グループ化されていない列が不正な位置で使用されている場合にエラーを報告します。

シグネチャまたは形状

static Node *substitute_grouped_columns_mutator(Node *node, substitute_grouped_columns_context *context);

入力と出力

  • **入力**:
  • node: 変換対象の式ツリーノード。
  • context: 変換状態、グループ化句、およびエラー報告用のパーサー状態を保持する substitute_grouped_columns_context 構造体。
  • **出力**: 変換後の Node *。置換が行われた場合は新しいノード、それ以外は元のノードまたはコピーされたノードを返します。

前提条件と不変条件

  • context->sublevels_up は現在のクエリレベルを追跡しており、これと一致するレベルの Var や Aggref のみを対象とします。
  • context->in_agg_direct_args は、順序付き集合集約の直接引数内を走査しているかを示すフラグとして使用されます。

アルゴリズムまたは振る舞い

1. **Aggref の処理**: 現在のクエリレベルと一致する場合、その集約の直接引数（aggdirectargs）に対して再帰的に本関数を適用します。集約の内部引数や ORDER BY 句には再帰しません。 2. **GroupingFunc の処理**: 現在のレベルの GroupingFunc はスキップします。 3. **グループ化式のマッチング**: GROUP BY 句に単純な Var 以外の式が含まれる場合、式全体が GROUP BY 項目と一致するかを確認し、一致すれば buildGroupedVar で置換します。 4. **Var の処理**:

  • 現在のクエリレベルの Var であれば、GROUP BY 句との照合を試みます。
  • 一致しない場合、check_functional_grouping を用いて関数的依存関係を検証します。
  • 依存関係も確認できない場合、ereport(ERROR) を発行して構文エラーを通知します。

    • 5. **再帰**: expression_tree_mutator を介してツリー全体を走査します。

呼び出し元と呼び出し先

  • **呼び出し元**: substitute_grouped_columns から呼び出されます。また、自身の再帰呼び出しとして agg->aggdirectargs の処理時に使用されます。
  • **呼び出し先**: expression_tree_mutator（ツリー走査）、buildGroupedVar（ノード置換）、check_functional_grouping（依存関係検証）、ereport（エラー報告）。

関連データ構造

  • substitute_grouped_columns_context: 変換に必要なグループ化情報やパーサー状態を保持。
  • Aggref, Var, GroupingFunc, TargetEntry: PostgreSQL のパーサーノード。

失敗モードと境界条件

  • **グループ化エラー**: GROUP BY 句に含まれない列が選択リストや他の句で使用されている場合、ERRCODE_GROUPING_ERROR でエラー終了します。
  • **順序付き集合集約**: 直接引数内にグループ化されていない列が含まれる場合、専用のエラー詳細メッセージと共にエラーとなります。
  • **ネストされたクエリ**: varlevelsup を用いて、現在のクエリレベルに属さない Var は無視されます。

保守担当者向けメモ

  • 本関数は expression_tree_mutator を使用して再帰的にツリーを書き換えます。
  • sublevels_up の管理が重要であり、サブクエリ内の集約や変数を誤って置換しないよう注意してください。
  • check_functional_grouping は高コストな処理であるため、func_grouped_rels リストを使用して一度検証したリレーションはスキップする最適化が行われています。
