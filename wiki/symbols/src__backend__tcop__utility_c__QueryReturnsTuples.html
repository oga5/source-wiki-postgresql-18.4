QueryReturnsTuples

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/utility.c:2136

目的

与えられた Query 構造体が、実行結果としてタプルをクライアントに返すかどうかを判定する。

シグネチャまたは形状

bool QueryReturnsTuples(Query *parsetree)

入力と出力

  • **入力**: Query *parsetree （解析済みのクエリツリー）
  • **出力**: bool （タプルを返す場合は true、それ以外は false）

前提条件と不変条件

  • parsetree は有効なポインタであること。
  • parsetree->commandType に基づいて判定を行う。

アルゴリズムまたは振る舞い

1. parsetree->commandType を評価する。 2. CMD_SELECT の場合は常に true を返す。 3. CMD_INSERT, CMD_UPDATE, CMD_DELETE, CMD_MERGE の場合、returningList が存在すれば true を返し、存在しなければ false を返す。 4. CMD_UTILITY の場合、UtilityReturnsTuples を呼び出し、その結果を返す。 5. 上記以外（CMD_UNKNOWN, CMD_NOTHING 等）は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c 内で定義されている（ソースコード上の直接的な参照）。
  • src/backend/executor/functions.c:2195 にて、将来的な置換候補として言及されている。
  • **呼び出し先**:
  • UtilityReturnsTuples （CMD_UTILITY の場合に呼び出される）。

関連データ構造

  • Query: パーサーが生成するクエリツリー構造体。
  • CmdType: commandType を保持する列挙型。

失敗モードと境界条件

  • CMD_UNKNOWN や CMD_NOTHING が渡された場合、デフォルトの false が返される。
  • CMD_UTILITY の判定は、内部で呼び出される UtilityReturnsTuples の実装に依存する。

保守担当者向けメモ

  • 新しいコマンドタイプが追加された場合、本関数の switch 文に適切な判定ロジックを追加する必要がある。
  • CMD_UTILITY の判定については、UtilityReturnsTuples の更新と同期させること。
