range_gist_same

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_gist.c:777

目的

2つの範囲型（RangeType）がGiSTインデックス上で同一であるかを判定します。

シグネチャまたは形状

Datum range_gist_same(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_RANGE_P(0): 比較対象の範囲型 r1
  • PG_GETARG_RANGE_P(1): 比較対象の範囲型 r2
  • PG_GETARG_POINTER(2): 結果を格納する bool 型のポインタ result
  • **出力**:
  • result ポインタを通じて、両者が等価であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力される範囲型は、比較前に適切に正規化されていることが期待されます。
  • RANGE_CONTAIN_EMPTY フラグを含むすべてのフラグビットが一致する必要があります。

アルゴリズムまたは振る舞い

1. range_get_flags を使用して、2つの範囲型のフラグバイトを比較します。フラグが異なる場合は即座に false と判定します。 2. フラグが一致する場合、range_get_typcache を介して型キャッシュを取得します。 3. range_eq_internal を呼び出し、範囲の内容が等価であるかを判定して結果に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: GiSTインデックスのメソッドとして呼び出されます（具体的な呼び出し元コードの詳細は提供されていません）。
  • **呼び出し先**: range_get_flags, range_get_typcache, range_eq_internal を呼び出します。

関連データ構造

  • RangeType: 比較対象の範囲型データ構造。
  • TypeCacheEntry: 範囲型の型情報を保持するキャッシュ構造体。

失敗モードと境界条件

  • range_eq 関数は RANGE_CONTAIN_EMPTY フラグを無視するため、本関数で明示的にフラグをチェックすることで、空範囲の判定漏れを防いでいます。

保守担当者向けメモ

  • 本関数はGiSTインデックスの same メソッドとして機能します。
  • 範囲の等価性判定において、フラグの不一致は即座に非等価とみなす最適化が行われています。
