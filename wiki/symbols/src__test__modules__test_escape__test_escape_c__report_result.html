report_result

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_escape/test_escape.c:118

目的

テスト実行結果を標準出力に記録し、テスト構成オブジェクト内の統計情報を更新する。

シグネチャまたは形状

static void report_result(pe_test_config *tc, bool success, const char *testname, const char *details, const char *subname, const char *resultdesc)

入力と出力

  • **入力**:
  • tc: テスト設定および状態を保持する pe_test_config 構造体へのポインタ。
  • success: テストの成否を示すブール値。
  • testname, subname, resultdesc: テスト名、サブテスト名、結果説明を示す文字列。
  • details: 詳細な出力情報。
  • **出力**: なし（標準出力への書き込みおよび tc 構造体の更新）。

前提条件と不変条件

  • tc は有効な pe_test_config 構造体を指している必要がある。
  • tc->test_count は呼び出しごとにインクリメントされる。

アルゴリズムまたは振る舞い

1. tc->test_count をインクリメントする。 2. success が false の場合、tc->failure_count をインクリメントする。 3. tc->verbosity 設定に基づき、詳細出力および結果出力の要否を判定する。

  • success が true かつ verbosity <= 0 の場合、詳細出力を抑制する。
  • success が true かつ verbosity < 0 の場合、結果出力を抑制する。

    • 4. 判定結果に従い、printf を用いて標準出力へテスト結果を書き出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/test/modules/test_escape/test_escape.c 内の各テスト実行ロジック（PQescapeLiteral の検証、JSON エスケープ検証、メモリ境界検証など）。
  • **呼び出し先**:
  • 標準ライブラリ関数: printf。

関連データ構造

  • pe_test_config: テストの統計情報（test_count, failure_count）および冗長度設定（verbosity）を保持する。

失敗モードと境界条件

  • tc が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • testname 等の文字列引数が NULL の場合、printf の実行時に未定義動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は TAP (Test Anything Protocol) 形式に準拠した出力を生成する。
  • verbosity レベルによる出力制御は、テスト実行時のログ量を調整するために使用される。
