RecordPageWithFreeSpace

定義

  • 種別: function
  • 定義箇所: src/backend/storage/freespace/freespace.c:193

目的

指定されたリレーションの特定のヒープブロックにおける空き容量を更新し、FSM（Free Space Map）の木構造に反映させる。

シグネチャまたは形状

void RecordPageWithFreeSpace(Relation rel, BlockNumber heapBlk, Size spaceAvail)

入力と出力

  • **入力**:
  • rel: 対象のリレーション。
  • heapBlk: 更新対象のブロック番号。
  • spaceAvail: ページ内の現在の空き容量（バイト単位）。
  • **出力**: なし（void）。

前提条件と不変条件

  • spaceAvail は fsm_space_avail_to_cat を通じて 256 段階のカテゴリ値に変換される。
  • FSM はヒント情報として機能するため、更新後の木構造の整合性は検索時にリーフノードで再確認される。

アルゴリズムまたは振る舞い

1. fsm_space_avail_to_cat を呼び出し、spaceAvail を FSM 管理用のカテゴリ値に変換する。 2. fsm_get_location を使用して、対象の heapBlk に対応する FSM 上の論理アドレス（addr）とスロット位置（slot）を特定する。 3. fsm_set_and_search を呼び出し、特定した位置の値を新しいカテゴリ値で更新し、必要に応じて親ノードへの伝播を行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin.c
  • src/backend/access/brin/brin_pageops.c
  • src/backend/access/heap/hio.c
  • src/backend/access/heap/vacuumlazy.c
  • **呼び出し先**:
  • fsm_space_avail_to_cat
  • fsm_get_location
  • fsm_set_and_search

関連データ構造

  • Relation: リレーション情報。
  • FSMAddress: FSM 木構造上の論理アドレス。
  • BlockNumber: 物理的なブロック識別子。

失敗モードと境界条件

  • heapBlk がリレーションの範囲外である場合、fsm_get_location の挙動に依存する。
  • FSM の更新は WAL ログを出力しないため、クラッシュリカバリ時には物理的なページ状態が優先される。

保守担当者向けメモ

  • 本関数は FSM の更新を行う主要なインターフェースである。
  • パフォーマンス向上のため、WAL 記録は行われない設計となっている。
  • fsm_set_and_search の第 5 引数に 0 が渡されており、検索を伴う更新処理が実行される。
