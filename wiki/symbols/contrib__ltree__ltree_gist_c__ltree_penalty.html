ltree_penalty

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_gist.c:260

目的

GiST インデックスにおいて、新しいキーを既存のインデックスノードに挿入する際のコスト（ペナルティ）を計算する。この値は、インデックスのバランスを最適化するために使用される。

シグネチャまたは形状

Datum ltree_penalty(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): 既存のインデックスエントリ (GISTENTRY)。
  • PG_GETARG_POINTER(1): 挿入対象の新しいインデックスエントリ (GISTENTRY)。
  • PG_GETARG_POINTER(2): 計算されたペナルティ値を格納する float 型のポインタ。
  • **出力**:
  • PG_RETURN_POINTER(penalty): 計算されたペナルティ値へのポインタを返す。

前提条件と不変条件

  • origval および newval は ltree_gist 構造体としてキャスト可能であること。
  • LTREE_GET_SIGLEN() により、現在のシグネチャ長が正しく取得できること。

アルゴリズムまたは振る舞い

1. origval（既存ノード）と newval（挿入値）から、それぞれ左境界ノード (LNODE) と右境界ノード (RNODE) を抽出する。 2. ltree_compare を用いて、以下の比較を行う。

  • cmpl: 既存ノードの左境界と新ノードの左境界の比較。
  • cmpr: 新ノードの右境界と既存ノードの右境界の比較。

    • 3. 各比較結果の最大値（0未満の場合は0）を合計し、それをペナルティ値とする。

  • *penalty = Max(cmpl, 0) + Max(cmpr, 0);

    • 4. この計算により、挿入によってノードの範囲がどれだけ拡大されるかを定量化する。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの挿入処理（PostgreSQL の GiST インデックス基盤から呼び出される）。
  • **呼び出し先**:
  • ltree_compare
  • LTG_GETLNODE / LTG_GETRNODE (マクロ)
  • LTREE_GET_SIGLEN (マクロ)
  • Max (マクロ)

関連データ構造

  • ltree_gist: インデックスノードの構造体。
  • GISTENTRY: GiST インデックスの各エントリを保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、または ltree_gist 構造体のメモリレイアウトが不正な場合は未定義の動作となる。
  • ltree_compare が予期しない値を返した場合、ペナルティ計算が不正確になる可能性がある。

保守担当者向けメモ

  • 本関数は GiST インデックスの挿入コストを決定する重要なロジックである。インデックスの効率（検索速度）に直結するため、変更時は ltree_picksplit との整合性に注意すること。
  • siglen の値はインデックス作成時に固定されるため、実行時に変更されないことを前提としている。
