CmdType

定義

  • 種別: enum
  • 定義箇所: src/include/nodes/nodes.h:268

目的

PostgreSQLのクエリ処理において、実行されるコマンドの種類（SQLステートメントの種別）を識別するために使用されます。パーサーからプランナー、実行エンジンに至るまで、処理の分岐条件として広く利用されます。

シグネチャまたは形状

typedef enum CmdType
{
    CMD_UNKNOWN,
    CMD_SELECT,
    CMD_UPDATE,
    CMD_INSERT,
    CMD_DELETE,
    CMD_MERGE,
    CMD_UTILITY,
    CMD_NOTHING
} CmdType;

入力と出力

  • 入力: 該当なし（列挙型定義のため）。
  • 出力: クエリ処理の各フェーズにおいて、現在の操作がどのコマンドに該当するかを示す識別子として機能します。

前提条件と不変条件

  • CmdType はクエリの性質を決定づける重要なメタデータであり、ノード構造体の一部として保持されます。
  • 列挙値の順序や値の変更は、バイナリ互換性やシリアライズ形式に影響を与える可能性があるため注意が必要です。

アルゴリズムまたは振る舞い

  • CMD_SELECT, CMD_UPDATE, CMD_INSERT, CMD_DELETE, CMD_MERGE: 各種DML操作を識別します。
  • CMD_UTILITY: CREATE, VACUUM, COPY などのユーティリティコマンドを識別します。
  • CMD_NOTHING: ルールシステムにおいて、条件付きで「何もしない」ことを指示するダミーコマンドとして機能します。
  • CMD_UNKNOWN: 初期状態や未定義のコマンドを示します。

呼び出し元と呼び出し先

  • 呼び出し元（参照箇所）:
  • contrib/postgres_fdw/postgres_fdw.c: 外部テーブルに対する操作の判定に使用。
  • src/backend/commands/trigger.c: トリガーの実行制御（before_stmt_triggers_fired 等）において、対象のコマンド種別を判断するために使用。
  • 呼び出し先: 該当なし（列挙型定義のため）。

関連データ構造

  • Query 構造体（parsenodes.h）: クエリのトップレベル構造体として commandType フィールドで CmdType を保持します。
  • PlannedStmt 構造体（plannodes.h）: 実行計画のルートとして commandType を保持します。

失敗モードと境界条件

  • 未定義の CmdType が渡された場合、後続の処理（switch 文など）で予期せぬ挙動やエラーが発生する可能性があります。

保守担当者向けメモ

  • 新しいコマンド種別を追加する必要がある場合は、本列挙型に値を追加するだけでなく、関連する copyfuncs.c や equalfuncs.c 等のノードサポート関数が適切に更新されているか確認してください。
  • トリガーやFDWなど、コマンド種別に依存するロジックが多岐にわたるため、変更時は影響範囲の広さに留意してください。
