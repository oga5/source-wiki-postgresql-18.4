gbt_tsge

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_ts.c:56

目的

2つの Timestamp 値を比較し、第一引数が第二引数以上（greater than or equal to）であるかを判定する。

シグネチャまたは形状

bool gbt_tsge(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a: 比較対象の Timestamp へのポインタ。
  • b: 比較対象の Timestamp へのポインタ。
  • flinfo: 関数呼び出し情報（使用されず）。
  • **出力**:
  • bool: *a >= *b であれば true、そうでなければ false。

前提条件と不変条件

  • 入力ポインタ a および b は、有効な Timestamp 値を指している必要がある。
  • 内部的に timestamp_ge 関数を呼び出すため、PostgreSQLのタイムスタンプ比較ルールに従う。

アルゴリズムまたは振る舞い

1. 引数 a と b を Timestamp 型のポインタとしてキャストする。 2. TimestampGetDatumFast マクロを使用して、各値を Datum 形式に変換する。 3. DirectFunctionCall2 を介して PostgreSQL のコア関数 timestamp_ge を呼び出し、比較結果を取得する。 4. DatumGetBool を使用して結果を論理値に変換し、呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • timestamp_ge (PostgreSQL コア関数)
  • **呼び出し元**:
  • contrib/btree_gist/btree_ts.c 内の静的構造体 tinfo の初期化箇所（136行目付近）で参照されている。

関連データ構造

  • Timestamp: PostgreSQL のタイムスタンプ型。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、セグメンテーション違反が発生する可能性がある。
  • timestamp_ge が内部でエラーを発生させた場合、その例外が伝播する。

保守担当者向けメモ

  • 本関数は GiST インデックスの比較ロジックの一部として使用される。
  • timestamptz 型を扱う場合も、内部的に GMT へ正規化された Timestamp 値が渡されることを前提としている。
