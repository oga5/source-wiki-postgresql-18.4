copy_auth_token

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/hba.c:289

目的

既存の AuthToken 構造体の内容を複製し、新しいメモリ領域にコピーを作成します。

シグネチャまたは形状

static AuthToken *copy_auth_token(AuthToken *in)

入力と出力

  • **入力**: in (AuthToken *) - 複製対象の AuthToken 構造体へのポインタ。
  • **出力**: 複製された AuthToken 構造体へのポインタ。

前提条件と不変条件

  • 入力 in は有効な AuthToken 構造体を指している必要があります。
  • make_auth_token が内部で呼び出されるため、メモリ割り当てが成功することが前提となります。

アルゴリズムまたは振る舞い

1. 入力 in が保持する文字列 (in->string) とクォート状態 (in->quoted) を取得します。 2. make_auth_token を呼び出し、新しい AuthToken 構造体を生成して初期化します。 3. 生成された新しい構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/hba.c:1451
  • src/backend/libpq/hba.c:1476
  • src/backend/libpq/hba.c:2781
  • src/backend/libpq/hba.c:2789
  • **呼び出し先**:
  • make_auth_token

関連データ構造

  • AuthToken: hba.c で使用される認証トークン情報を保持する構造体。

失敗モードと境界条件

  • 入力 in が NULL の場合、in->string へのアクセスによりセグメンテーション違反が発生します。
  • メモリ不足時には make_auth_token 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は AuthToken のディープコピーを行うためのユーティリティです。設定解析中にトークンを別のリストや構造体へ保持する際に使用されます。
