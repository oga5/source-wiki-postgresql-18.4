CState

定義

  • 種別: struct
  • 定義箇所: src/bin/pgbench/pgbench.c:598

目的

pgbench における個々のクライアントの状態を保持する構造体です。データベース接続、スクリプトの実行位置、変数、統計情報、およびステートマシンの状態を管理します。

シグネチャまたは形状

typedef struct {
    PGconn *con;
    int id;
    ConnectionStateEnum state;
    ConditionalStack cstack;
    pg_prng_state cs_func_rs;
    int use_file;
    int command;
    int num_syncs;
    Variables variables;
    pg_time_usec_t txn_scheduled;
    pg_time_usec_t sleep_until;
    pg_time_usec_t txn_begin;
    pg_time_usec_t stmt_begin;
    bool **prepared;
    EStatus estatus;
    pg_prng_state random_state;
    uint32 tries;
    int64 cnt;
} CState;

入力と出力

本構造体は関数への引数として渡され、ステートマシンの遷移や式評価の結果によって内部状態が更新されます。直接的な入力/出力関数はありません。

前提条件と不変条件

  • estatus は、エラーが発生していない場合 ESTATUS_NO_ERROR である必要があります。
  • state は ConnectionStateEnum で定義された有効な遷移状態を保持する必要があります。

アルゴリズムまたは振る舞い

  • pgbench のメインループにおいて、advanceConnectionState() 関数が本構造体の state を参照・更新し、ステートマシンを駆動します。
  • スクリプト内の式評価（evaluateExpr 等）やメタコマンド実行（executeMetaCommand）において、クライアント固有の変数や実行位置を保持・更新するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: pgbench.c 内の threadRun()（メインループ）や、式評価・コマンド実行に関連する各関数（evaluateExpr, executeMetaCommand, evalFunc 等）から参照されます。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出す主体ではありません。

関連データ構造

  • ConnectionStateEnum: ステートマシンの状態定義。
  • Variables: クライアントごとの変数保持用。
  • EStatus: トランザクション実行時のエラー状態。
  • TState: スレッドごとの状態（本構造体は TState に含まれるクライアント配列の一部として管理されることが一般的です）。

失敗モードと境界条件

  • estatus がエラーを示す場合、トランザクションの再試行（tries のカウントアップ）や失敗処理がトリガーされます。
  • prepared 配列は、各コマンドのプリペア状態を管理し、接続ごとの準備状況を追跡します。

保守担当者向けメモ

  • pgbench.c 内の巨大な switch 文（advanceConnectionState）を理解する上で最も重要な構造体です。
  • クライアントごとの乱数生成器（cs_func_rs, random_state）が複数存在するため、乱数生成の文脈を混同しないよう注意してください。
