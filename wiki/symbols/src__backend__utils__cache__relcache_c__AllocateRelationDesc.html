AllocateRelationDesc

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/cache/relcache.c:412

目的

リレーション記述子（Relation 構造体）のメモリ領域を確保し、システムカタログ pg_class の行データに基づいて初期化を行う。

シグネチャまたは形状

static Relation AllocateRelationDesc(Form_pg_class relp)

入力と出力

  • **入力**: relp (Form_pg_class) - リレーションのメタデータを含む pg_class 行のポインタ。
  • **出力**: 初期化された Relation 構造体へのポインタ。

前提条件と不変条件

  • リレーション記述子は CacheMemoryContext 上に割り当てられる必要がある。
  • relp からコピーされるのは固定長フィールド（CLASS_TUPLE_SIZE）のみであり、可変長フィールド（relacl, reloptions）は含まれない。

アルゴリズムまたは振る舞い

1. CacheMemoryContext へメモリコンテキストを切り替える。 2. RelationData 構造体分のメモリを palloc0 で確保し、rd_smgr を NULL に初期化する。 3. CLASS_TUPLE_SIZE 分のメモリを確保し、relp の内容をコピーして rd_rel に格納する。 4. CreateTemplateTupleDesc を使用して属性記述子（rd_att）を生成し、参照カウントを 1 に設定する。 5. 元のメモリコンテキストへ戻し、構築した Relation を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/relcache.c 内の複数箇所（RelationBuildDesc 等から呼び出されることが示唆されている）。
  • **呼び出し先**: MemoryContextSwitchTo, palloc0, palloc, memcpy, CreateTemplateTupleDesc。

関連データ構造

  • RelationData: リレーション記述子のメイン構造体。
  • Form_pg_class: pg_class システムカタログの行形式。
  • CacheMemoryContext: リレーションキャッシュが保持されるメモリコンテキスト。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させる。
  • 可変長フィールド（relacl 等）はキャッシュされないため、これらが必要な場合は別途 syscache から取得する必要がある。

保守担当者向けメモ

  • 本関数はリレーションの物理的なファイルハンドル（rd_smgr）を初期化しないため、後続の処理で適切に設定する必要がある。
  • relacl や reloptions の元データはキャッシュに含まれないため、これらに依存するロジックを修正する際は注意すること。
