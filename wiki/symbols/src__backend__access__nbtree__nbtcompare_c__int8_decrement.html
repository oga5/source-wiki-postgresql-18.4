int8_decrement

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtcompare.c:311

目的

B-tree インデックスのスキップスキャンにおいて、int64 型の値を 1 減算するための補助関数です。

シグネチャまたは形状

static Datum int8_decrement(Relation rel, Datum existing, bool *underflow)

入力と出力

  • **入力**:
  • rel: 対象となるリレーション（未使用）。
  • existing: 減算対象の int64 値（Datum 型）。
  • underflow: アンダーフロー発生時に状態を書き込むためのポインタ。
  • **出力**:
  • 減算後の Datum 値。アンダーフロー時は (Datum) 0 を返します。

前提条件と不変条件

  • existing は int64 型として解釈可能である必要があります。
  • アンダーフローが発生した場合、*underflow は true に設定され、戻り値は未定義となります。

アルゴリズムまたは振る舞い

1. existing を int64 に変換します。 2. 値が PG_INT64_MIN と等しい場合、アンダーフローと判定し、*underflow を true に設定して (Datum) 0 を返します。 3. それ以外の場合、*underflow を false に設定し、値から 1 を減算した結果を Datum 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtcompare.c 内で、SkipSupport 構造体の decrement フィールドに本関数が代入されています。
  • **呼び出し先**: 直接的な関数呼び出しはソース内に見当たりません。

関連データ構造

  • SkipSupport: インデックススキップスキャン用のロジックを保持する構造体。

失敗モードと境界条件

  • **アンダーフロー**: 入力値が PG_INT64_MIN の場合、減算が不可能なためアンダーフローとして処理されます。

保守担当者向けメモ

  • 本関数は SkipSupport インターフェースの一部として使用されます。新しい整数型を追加する際は、同様のパターンに従ってください。
