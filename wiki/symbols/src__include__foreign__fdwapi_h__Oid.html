Oid

定義

  • 種別: typedef
  • 定義箇所: src/include/foreign/fdwapi.h:27

目的

PostgreSQL システム内において、データベースオブジェクト（テーブル、インデックス、関数など）を一意に識別するための識別子（Object Identifier）を表現します。

シグネチャまたは形状

typedef として定義された符号なし整数型です。

入力と出力

  • 入力: 該当なし（型定義のため）
  • 出力: 該当なし

前提条件と不変条件

  • システムカタログ内の各オブジェクトに対して一意な値が割り当てられます。
  • 内部的には符号なし整数として扱われます。

アルゴリズムまたは振る舞い

特定のアルゴリズムは持ちませんが、データベースエンジン全体でオブジェクトを検索、参照、および管理するためのキーとして使用されます。例えば、fdwapi.h 内の GetForeignRelSize_function では、対象となる外部テーブルを特定するために引数 foreigntableid として利用されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/amcheck モジュール内の各関数（amcheck_lock_relation_and_check、index_checkable 等）や、PG_GETARG_OID マクロを介した関数呼び出しで頻繁に使用されています。
  • 呼び出し先: 型定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • RelOptInfo: クエリ計画作成時に Oid を用いてリレーションを特定します。
  • Relation: データベース内のリレーションを表す構造体で、Oid を保持します。

失敗モードと境界条件

  • InvalidOid（通常 0）は、有効なオブジェクトを指さない識別子として予約されています。
  • システムカタログの再構築やオブジェクトの削除に伴い、古い Oid が無効になる可能性があります。

保守担当者向けメモ

  • Oid は PostgreSQL のコア全体で多用される基本型です。FDW API のシグネチャを変更する際は、この型がオブジェクトの識別子として一貫して使用されていることを確認してください。
