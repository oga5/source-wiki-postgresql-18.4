getCopyDataMessage

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:1793

目的

COPY OUT モード中にサーバーから送信されるデータ行（CopyData メッセージ）を取得します。また、非同期メッセージ（通知やパラメータ変更など）を処理し、COPY モードの終了を検知します。

シグネチャまたは形状

static int getCopyDataMessage(PGconn *conn);

入力と出力

  • **入力**: PGconn *conn （PostgreSQL 接続状態を保持する構造体）
  • **出力**:
  • > 0: 取得した CopyData メッセージの長さ。
  • -1: COPY モードの終了を検知。
  • -2: エラー発生（同期喪失またはバッファ拡張失敗）。

前提条件と不変条件

  • conn->inBuffer にメッセージの断片が含まれていることを前提とします。
  • 非同期呼び出しを簡略化するため、CopyData 以外のメッセージであっても、メッセージ全体がバッファに揃うまでは 0 を返し続けます。

アルゴリズムまたは振る舞い

1. **メッセージ読み込み**: pqGetc でメッセージタイプ、pqGetInt でメッセージ長を読み取ります。 2. **バッファ検証**: メッセージ長が不足している場合、pqCheckInBufferSpace でバッファを拡張し、必要に応じて 0 を返して待機します。 3. **メッセージディスパッチ**:

  • PqMsg_CopyData: メッセージ長を返し、呼び出し元にデータを渡します。
  • PqMsg_NotificationResponse, NoticeResponse, ParameterStatus: それぞれのハンドラを呼び出し、処理を継続します。
  • PqMsg_CopyDone またはその他: COPY モードを終了し、conn->asyncStatus を更新して -1 を返します。

    • 4. **ループ**: CopyData 以外のメッセージを処理した後は、pqParseDone でバッファカーソルを進め、次のメッセージを読み込みます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-protocol3.c 内の COPY モード処理箇所（行 1910, 2024）。
  • **呼び出し先**: pqGetc, pqGetInt, handleSyncLoss, pqCheckInBufferSpace, handleFatalError, getNotify, pqGetErrorNotice3, getParameterStatus, pqParseDone。

関連データ構造

  • PGconn: 接続状態および入力バッファ（inBuffer, inCursor, inStart, inEnd）を管理。
  • PqMsg_...: メッセージタイプを識別する定数群。

失敗モードと境界条件

  • **同期喪失**: メッセージ長が 4 未満の場合、handleSyncLoss を呼び出し -2 を返します。
  • **メモリ不足**: pqCheckInBufferSpace が失敗した場合、handleFatalError を呼び出し接続を終了して -2 を返します。

保守担当者向けメモ

  • 本関数は非ブロッキング I/O を前提としています。データが足りない場合は即座にリターンするため、呼び出し側は戻り値が 0 の場合の再試行ロジックを適切に実装する必要があります。
  • COPY_BOTH モードと COPY_IN モードの遷移ロジックが含まれており、状態遷移の変更時には conn->asyncStatus の整合性に注意してください。
