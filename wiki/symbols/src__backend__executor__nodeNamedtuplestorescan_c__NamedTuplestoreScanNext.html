NamedTuplestoreScanNext

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeNamedtuplestorescan.c:30

目的

名前付き一時タプルストア（ENR）から次のタプルをフェッチし、スキャンノードのタプルスロットへ格納する。

シグネチャまたは形状

static TupleTableSlot *NamedTuplestoreScanNext(NamedTuplestoreScanState *node)

入力と出力

  • **入力**: NamedTuplestoreScanState *node（スキャン状態を保持するノード構造体）
  • **出力**: TupleTableSlot *（取得したタプルを格納したスロット、またはタプルがない場合は NULL）

前提条件と不変条件

  • 本関数は前方スキャンのみをサポートする。実行時のスキャン方向が前方であることを Assert で検証している。

アルゴリズムまたは振る舞い

1. ノード内のスキャンタプルスロットを取得する。 2. tuplestore_select_read_pointer を呼び出し、当該ノードに割り当てられた読み取りポインタをアクティブにする。 3. tuplestore_gettupleslot を呼び出し、タプルストアからスロットへタプルを読み込む。 4. 読み込んだスロットを呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecScan（ExecNamedTuplestoreScan を介してコールバックとして呼び出される）。
  • **呼び出し先**:
  • tuplestore_select_read_pointer
  • tuplestore_gettupleslot

関連データ構造

  • NamedTuplestoreScanState: スキャンノードの状態を管理する構造体。
  • TupleTableSlot: タプルを保持するメモリ領域。
  • Tuplestore: 物理的なタプル保持構造。

失敗モードと境界条件

  • タプルストアが空の場合、tuplestore_gettupleslot の挙動に従い NULL が返される。
  • 後方スキャンが要求された場合、Assert によりデバッグビルド環境では停止する。

保守担当者向けメモ

  • 本関数は ExecScan のコールバックとして設計されている。
  • 読み取りポインタの管理は tuplestore 側の機能に依存しており、本関数は現在のスキャン位置を維持する役割を担う。
