PQconninfoFree

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-connect.c:7458

目的

PQconninfoOption 配列によって確保されたメモリ領域を解放し、関連する動的メモリリークを防ぐ。

シグネチャまたは形状

void PQconninfoFree(PQconninfoOption *connOptions)

入力と出力

  • **入力**: PQconninfoOption *connOptions - 解放対象の接続オプション配列へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • connOptions が NULL の場合、関数は何もしない。
  • connOptions が指す配列は、keyword メンバが NULL である要素を終端として持つ必要がある。

アルゴリズムまたは振る舞い

1. connOptions が NULL であれば直ちに終了する。 2. 配列の先頭から keyword が NULL になるまで反復処理を行う。 3. 各要素の val メンバ（動的に確保された文字列）に対して free() を実行する。 4. 最後に、配列自体を保持しているメモリ領域に対して free() を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • src/backend/replication/libpqwalreceiver/libpqwalreceiver.c
  • src/bin/pg_basebackup/pg_createsubscriber.c
  • src/bin/pg_basebackup/streamutil.c
  • src/bin/pg_dump/connectdb.c
  • **呼び出し先**: free()（標準ライブラリ関数）。

関連データ構造

  • PQconninfoOption: 接続パラメータのキーワードと値のペアを保持する構造体。

失敗モードと境界条件

  • connOptions が指すメモリが既に解放済みである場合、二重解放（double free）が発生する。
  • 配列が keyword が NULL である要素で正しく終端されていない場合、メモリ破壊が発生する。

保守担当者向けメモ

  • 本関数は PQconndefaults や PQconninfoParse 等で生成された PQconninfoOption 配列の破棄に使用される。
  • 構造体内の val メンバのみが動的確保されている前提で実装されているため、構造体定義に変更を加える際は注意が必要である。
