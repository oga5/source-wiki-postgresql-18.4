gettoken_query_plain

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery.c:509

目的

tsquery の解析において、入力文字列を単純なトークン（値）として抽出します。

シグネチャまたは形状

static ts_tokentype gettoken_query_plain(
    TSQueryParserState state, 
    int8 *operator, 
    int *lenval, 
    char **strval, 
    int16 *weight, 
    bool *prefix
)

入力と出力

  • **入力**:
  • state: 解析状態を保持する TSQueryParserState 構造体。
  • **出力**:
  • 戻り値: PT_VAL（トークンが見つかった場合）または PT_END（終端に達した場合）。
  • strval: 現在のバッファ位置を指すポインタ。
  • lenval: 現在のバッファ内の文字列長。
  • weight: 0 に初期化されます。
  • prefix: false に初期化されます。
  • state->buf: トークン分だけ進められます。
  • state->count: インクリメントされます。

前提条件と不変条件

  • state->buf が有効な文字列を指している必要があります。
  • weight および prefix は常にデフォルト値（0, false）に設定されます。

アルゴリズムまたは振る舞い

1. state->buf が終端文字（\0）であれば PT_END を返します。 2. weight を 0、prefix を false に設定します。 3. state->buf の現在位置を strval に格納します。 4. strlen(state->buf) を計算して lenval に格納します。 5. state->buf を lenval 分だけ進め、state->count をインクリメントします。 6. PT_VAL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/tsquery.c 内で、TSQueryParserState の gettoken フィールドにこの関数が代入されています（src/backend/utils/adt/tsquery.c:838）。
  • **呼び出し先**: strlen（標準Cライブラリ）。

関連データ構造

  • TSQueryParserState: 解析中の状態を保持する構造体。
  • ts_tokentype: トークンの種別を示す列挙型。

失敗モードと境界条件

  • 空文字列が渡された場合、即座に PT_END を返します。
  • state->buf が指す文字列が非常に長い場合、strlen の計算コストが発生します。

保守担当者向けメモ

  • 本関数は「プレーン」な解析モード用であり、演算子や修飾子（重みやプレフィックス）を一切考慮しません。
  • operator 引数はシグネチャに含まれていますが、本関数内では使用されていません。
