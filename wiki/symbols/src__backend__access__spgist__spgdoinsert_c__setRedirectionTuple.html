setRedirectionTuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgdoinsert.c:567

目的

SP-GiSTインデックスページ内の既存のリダイレクトタプルを更新し、新しい物理的な位置（ブロック番号とオフセット番号）を指し示すように変更します。

シグネチャまたは形状

static void setRedirectionTuple(SPPageDesc *current, OffsetNumber position, BlockNumber blkno, OffsetNumber offnum)

入力と出力

  • current: 操作対象のページ情報を保持する SPPageDesc 構造体へのポインタ。
  • position: 更新対象となるリダイレクトタプルのページ内オフセット番号。
  • blkno: リダイレクト先となる新しいブロック番号。
  • offnum: リダイレクト先となる新しいオフセット番号。
  • 戻り値: なし。

前提条件と不変条件

  • current->page 上の position にあるタプルが SPGIST_REDIRECT 状態であること。
  • 更新前のタプルのポインタが SPGIST_METAPAGE_BLKNO を指していること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. PageGetItem を使用して、指定された position のタプルを SpGistDeadTuple として取得します。 2. タプルの状態が SPGIST_REDIRECT であることを確認します。 3. タプルが保持する ItemPointer を、引数で指定された blkno および offnum に更新します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/spgist/spgdoinsert.c 内の以下の箇所から呼び出されています。
  • 1286行目
  • 1323行目
  • 呼び出し先: PageGetItem、PageGetItemId、ItemPointerSet などのアクセサマクロを使用しています。

関連データ構造

  • SPPageDesc: 挿入中のページ状態を管理する構造体。
  • SpGistDeadTuple: リダイレクト情報を含むタプル構造体。

失敗モードと境界条件

  • position が無効な場合や、対象タプルが SPGIST_REDIRECT 状態でない場合はアサーション失敗により異常終了します。

保守担当者向けメモ

  • 本関数は、SP-GiSTのリーフチェーン移動やページ分割に伴うリダイレクト処理の一部として使用されます。
  • WALログの生成は本関数の呼び出し元で行われる必要があるため、本関数単体ではWALの整合性は保証されません。
