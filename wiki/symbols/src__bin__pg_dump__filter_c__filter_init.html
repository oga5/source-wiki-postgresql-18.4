filter_init

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/filter.c:35

目的

フィルタ処理の状態管理構造体（FilterStateData）を初期化し、指定されたファイルまたは標準入力からの読み込み準備を行う。

シグネチャまたは形状

void filter_init(FilterStateData *fstate, const char *filename, exit_function f_exit)

入力と出力

  • fstate: 初期化対象の FilterStateData 構造体へのポインタ。
  • filename: 読み込み対象のファイルパス。"-" の場合は標準入力（stdin）が使用される。
  • f_exit: エラー発生時に呼び出される終了処理関数ポインタ。
  • 出力: なし（fstate の各フィールドが初期化される）。

前提条件と不変条件

  • fstate は呼び出し元で確保された有効なメモリ領域であること。
  • f_exit は有効な関数ポインタであること。
  • initStringInfo により fstate->linebuff が初期化される。

アルゴリズムまたは振る舞い

1. fstate の filename、lineno（0にリセット）、exit_nicely を引数で設定する。 2. initStringInfo を呼び出し、行バッファを初期化する。 3. filename が "-" であれば、fstate->fp に stdin を割り当てる。 4. それ以外の場合、fopen でファイルを開く。

  • ファイルオープンに失敗した場合は pg_log_error でエラーを出力し、f_exit を呼び出して終了する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/bin/pg_dump/pg_dump.c
  • src/bin/pg_dump/pg_dumpall.c
  • src/bin/pg_dump/pg_restore.c
  • 呼び出し先:
  • initStringInfo
  • fopen
  • pg_log_error
  • fstate->exit_nicely (コールバック)

関連データ構造

  • FilterStateData: フィルタ処理の状態を保持する構造体。
  • exit_function: 終了処理用関数ポインタ型。

失敗モードと境界条件

  • 指定されたファイルが存在しない、または読み取り権限がない場合、pg_log_error を経由して f_exit が呼び出され、プログラムが終了する。

保守担当者向けメモ

  • exit_nicely は外部から注入されるコールバックであるため、デバッグ時には各実行ファイル（pg_dump 等）でどのように定義されているかを確認すること。
  • fstate->linebuff は initStringInfo で初期化されるため、使用後は適切に解放される必要がある（通常は filter_free が担当）。
