sepgsql_schema_relabel

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/schema.c:141

目的

PostgreSQL のスキーマ（pg_namespace）に対するセキュリティラベルの変更（RELABEL）操作において、SELinux ポリシーに基づいた権限検証を行う。

シグネチャまたは形状

void sepgsql_schema_relabel(Oid namespaceId, const char *seclabel)

入力と出力

  • namespaceId (Oid): 対象となるスキーマの識別子。
  • seclabel (const char *): 新しく適用されるセキュリティラベル文字列。
  • 戻り値: なし。権限チェックに失敗した場合はエラーを発生させる。

前提条件と不変条件

  • namespaceId は有効な pg_namespace カタログの OID であること。
  • 呼び出し元は、対象スキーマに対するラベル変更操作の直前に本関数を呼び出す必要がある。

アルゴリズムまたは振る舞い

1. namespaceId から ObjectAddress を構築し、getObjectIdentity を用いて監査用の識別名を取得する。 2. sepgsql_avc_check_perms を呼び出し、現在のラベルに対して db_schema:{setattr relabelfrom} 権限があるか検証する。 3. sepgsql_avc_check_perms_label を呼び出し、新しいラベルに対して db_schema:{relabelto} 権限があるか検証する。 4. 監査用文字列 audit_name を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/sepgsql/label.c:504
  • 呼び出し先:
  • getObjectIdentity
  • sepgsql_avc_check_perms
  • sepgsql_avc_check_perms_label
  • pfree

関連データ構造

  • ObjectAddress: スキーマオブジェクトを特定するために使用。
  • NamespaceRelationId: pg_namespace カタログを参照するための定数。

失敗モードと境界条件

  • 権限チェック（sepgsql_avc_check_perms または sepgsql_avc_check_perms_label）で拒否された場合、内部的にエラーが発生し処理が中断される。
  • getObjectIdentity が失敗した場合、PostgreSQL の標準的なエラー処理に従う。

保守担当者向けメモ

  • 本関数は RELABEL 操作の認可のみを担う。実際のラベル更新処理は呼び出し元で行われる。
  • db_schema クラスに対する権限チェックは、SELinux の AVC（Access Vector Cache）を介して行われるため、ポリシーの変更が即座に反映される。
