on_exit_nicely

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_utils.c:64

目的

プログラム終了時に実行すべきクリーンアップ関数を登録し、終了シーケンスを管理するためのインフラを提供します。

シグネチャまたは形状

void on_exit_nicely(on_exit_nicely_callback function, void *arg);

入力と出力

  • function: 終了時に呼び出されるコールバック関数。
  • arg: コールバック関数に渡される任意の引数へのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • 登録可能なコールバック数は最大20個までです。
  • 登録された関数は、exit_nicely が呼び出された際に、登録時とは逆順（インデックスの大きい順）で実行されます。

アルゴリズムまたは振る舞い

1. 現在の登録インデックス（on_exit_nicely_index）が上限（MAX_ON_EXIT_NICELY）に達していないか確認します。 2. 上限に達している場合は pg_fatal を呼び出し、エラーメッセージを出力して異常終了します。 3. 上限に達していない場合、指定された function と arg を内部リスト（on_exit_nicely_list）の現在のインデックス位置に格納します。 4. インデックスをインクリメントします。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/parallel.c（データベース接続の切断処理等の登録に使用）。
  • 呼び出し先: pg_fatal（エラー発生時）。

関連データ構造

  • on_exit_nicely_list: 登録されたコールバック関数と引数を保持する内部配列。
  • on_exit_nicely_index: 現在の登録数を追跡するカウンタ。

失敗モードと境界条件

  • 登録数が20個を超えた場合、pg_fatal によりプログラムが強制終了されます。

保守担当者向けメモ

  • 本関数で登録された処理は、exit_nicely を通じて実行されます。リソース解放の依存関係がある場合は、登録順序に注意してください。
  • Unix環境では fork() 後に子プロセスへ引き継がれるため、一貫性を保つために可能な限り fork() 前に登録を完了させる設計が推奨されます。
