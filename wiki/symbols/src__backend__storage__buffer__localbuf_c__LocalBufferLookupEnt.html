LocalBufferLookupEnt

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/buffer/localbuf.c:34

目的

一時テーブル用バッファマネージャにおいて、ディスク上のページ（BufferTag）と、プロセスローカルなバッファ配列のインデックス（id）を紐付けるためのハッシュテーブル用エントリ構造体です。

シグネチャまたは形状

typedef struct {
    BufferTag key;    /* ディスクページのタグ */
    int       id;     /* 対応するローカルバッファのインデックス */
} LocalBufferLookupEnt;

入力と出力

本構造体はデータ保持を目的としており、関数ではありません。ハッシュテーブル操作時にキーとして key が使用され、検索結果として id が返されます。

前提条件と不変条件

  • id は LocalBufferDescriptors 配列の有効なインデックスを指す必要があります。
  • LocalBufHash ハッシュテーブルの各エントリとして使用されます。

アルゴリズムまたは振る舞い

ハッシュテーブルの検索キーとして BufferTag を使用し、対応するローカルバッファ ID を高速に特定するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/localbuf.c 内の各関数（LocalBufferAlloc 等）が、ハッシュテーブル検索の結果として本構造体へのポインタを取得します。
  • **呼び出し先**: 構造体定義であるため、直接的な呼び出し先はありません。

関連データ構造

  • BufferTag: ハッシュテーブルの検索キー。
  • LocalBufHash: 本構造体を要素として格納するハッシュテーブル。
  • LocalBufferDescriptors: id が参照するバッファメタデータ配列。

失敗モードと境界条件

  • ハッシュテーブルに存在しないタグで検索を行った場合、検索関数は NULL を返します。
  • sizeof(LocalBufferLookupEnt) はハッシュテーブルの初期化時（info.entrysize）に利用されます。

保守担当者向けメモ

  • 本構造体はプロセスローカルなメモリ上にのみ存在します。
  • 構造体のサイズ変更やフィールド追加を行う際は、LocalBufHash の初期化ロジックと整合性が取れているか確認してください。
