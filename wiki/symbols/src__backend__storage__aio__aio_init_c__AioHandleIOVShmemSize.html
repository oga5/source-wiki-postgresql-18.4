AioHandleIOVShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_init.c:69

目的

非同期I/O（AIO）サブシステムにおいて、全I/Oハンドルが必要とする iovec オブジェクト群を格納するための共有メモリサイズを算出します。

シグネチャまたは形状

Size AioHandleIOVShmemSize(void)

入力と出力

  • **入力**: なし（グローバル変数 io_max_combine_limit、io_max_concurrency および関数 AioProcs() を参照）
  • **出力**: Size 型のバイト数

前提条件と不変条件

  • io_max_combine_limit、AioProcs()（プロセス数）、および io_max_concurrency の積に基づき計算されます。
  • 共有メモリの初期化フェーズで呼び出されることを前提としています。

アルゴリズムまたは振る舞い

以下の計算式により、必要なメモリサイズを算出します。 sizeof(struct iovec) * io_max_combine_limit * AioProcs() * io_max_concurrency この関数は、各I/Oハンドルが保持可能な iovec オブジェクトの総数分を確保するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio_init.c: AioShmemSize() 内で合計サイズ算出のために呼び出されます。
  • src/backend/storage/aio/aio_init.c: AioShmemInit() 内で ShmemInitStruct を用いたメモリ確保時に呼び出されます。
  • **呼び出し先**:
  • AioProcs()
  • mul_size()（PostgreSQLのメモリサイズ計算用ユーティリティ）

関連データ構造

  • struct iovec: I/O操作で使用されるデータ構造。
  • AioProcs(): システム内のAIOプロセス数。

失敗モードと境界条件

  • mul_size を使用しているため、計算結果が Size 型の範囲を超える場合はエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は io_max_combine_limit や io_max_concurrency といった設定値に依存しています。これらの設定値を変更する際は、共有メモリ消費量への影響を考慮してください。
