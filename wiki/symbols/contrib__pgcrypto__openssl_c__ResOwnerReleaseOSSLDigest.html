ResOwnerReleaseOSSLDigest

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/openssl.c:215

目的

PostgreSQL のリソース所有者（ResourceOwner）機構と連携し、トランザクション終了時やエラー発生時に、ハッシュ計算用コンテキスト OSSLDigest を安全に解放する。

シグネチャまたは形状

static void ResOwnerReleaseOSSLDigest(Datum res)

入力と出力

  • **入力**: res (Datum) - 解放対象の OSSLDigest 構造体へのポインタを保持する Datum 型の値。
  • **出力**: なし（void）。

前提条件と不変条件

  • res は有効な OSSLDigest 構造体を指している必要がある。
  • 本関数は ResourceOwner のコールバックとして呼び出されることを想定している。

アルゴリズムまたは振る舞い

1. DatumGetPointer(res) を使用して、Datum から OSSLDigest 構造体へのポインタを取得する。 2. 構造体内の owner フィールドを NULL に設定し、リソース所有者との関連付けを解除する。 3. free_openssl_digest(digest) を呼び出し、OpenSSL のコンテキスト破棄およびメモリ解放を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: ResourceOwner 機構（contrib/pgcrypto/openssl.c 内の ResourceOwner コールバック定義にて参照）。
  • **呼び出し先**: free_openssl_digest（内部的な解放関数）。

関連データ構造

  • OSSLDigest: ハッシュ計算の状態を保持する構造体。ResourceOwner に登録される対象。

失敗モードと境界条件

  • res が NULL または無効なポインタである場合、未定義の動作を引き起こす可能性がある。

保守担当者向けメモ

  • 本関数は ResourceOwner の ReleaseResource コールバックとして登録されている。新しいハッシュアルゴリズムやコンテキスト管理を追加する際は、この解放フローが維持されていることを確認すること。
