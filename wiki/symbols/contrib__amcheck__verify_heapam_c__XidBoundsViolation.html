XidBoundsViolation

定義

  • 種別: enum
  • 定義箇所: contrib/amcheck/verify_heapam.c:45

目的

ヒープリレーションの検証プロセスにおいて、トランザクションID（XID）またはマルチトランザクションID（MXID）が許容範囲内にあるかを判定するための状態を表現する。

シグネチャまたは形状

typedef enum XidBoundsViolation
{
    XID_INVALID,
    XID_IN_FUTURE,
    XID_PRECEDES_CLUSTERMIN,
    XID_PRECEDES_RELMIN,
    XID_BOUNDS_OK,
} XidBoundsViolation;

入力と出力

  • 本シンボルは列挙型であり、関数が検証結果を返す際の戻り値として使用される。

前提条件と不変条件

  • XID_BOUNDS_OK は、検証対象のIDが現在のシステム状態およびリレーションのメタデータに対して整合性が取れていることを示す。
  • その他の値は、IDが範囲外であるか、無効であることを示すエラー状態である。

アルゴリズムまたは振る舞い

  • check_mxid_in_range、check_mxid_valid_in_rel、get_xid_status などの関数において、IDの妥当性を評価した結果を返すために使用される。
  • 検証ロジックは、この列挙値に基づいて破損の有無や種類を判断する。

呼び出し元と呼び出し先

  • 呼び出し元: check_mxid_in_range, check_mxid_valid_in_rel, get_xid_status 等の検証関数がこの型を戻り値として生成する。
  • 呼び出し先: 本シンボル自体はデータ型であり、呼び出し先は存在しない。

関連データ構造

  • HeapCheckContext: 検証中の状態を保持する構造体であり、この列挙値と組み合わせて使用される。

失敗モードと境界条件

  • XID_INVALID: IDが不正。
  • XID_IN_FUTURE: IDが未来のトランザクションを指している。
  • XID_PRECEDES_CLUSTERMIN: IDがクラスタ全体の最小値より古い。
  • XID_PRECEDES_RELMIN: IDがリレーションの最小値より古い。

保守担当者向けメモ

  • 新たな検証ルールを追加する際は、この列挙型に新しい状態を追加する必要があるか検討すること。
  • 本シンボルは verify_heapam.c 内のヒープ検証ロジックに特化している。
