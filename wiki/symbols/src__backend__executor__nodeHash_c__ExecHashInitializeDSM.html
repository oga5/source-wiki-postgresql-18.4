ExecHashInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeHash.c:2779

目的

並列ハッシュ結合実行時において、各ワーカーの実行統計情報を格納するための共有メモリ領域（DSM: Dynamic Shared Memory）を初期化する。

シグネチャまたは形状

void ExecHashInitializeDSM(HashState *node, ParallelContext *pcxt)

入力と出力

  • **入力**:
  • node: HashState 構造体へのポインタ。初期化対象のハッシュノード状態。
  • pcxt: ParallelContext 構造体へのポインタ。並列実行コンテキスト。
  • **出力**: なし（副作用として node->shared_info が設定され、共有メモリ領域が確保される）。

前提条件と不変条件

  • node->ps.instrument が有効であり、かつ pcxt->nworkers > 0 である場合にのみ処理が実行される。
  • 確保された共有メモリ領域は、初期化時に memset によってゼロクリアされる。

アルゴリズムまたは振る舞い

1. インストゥルメンテーションが不要、またはワーカーが存在しない場合は直ちに終了する。 2. SharedHashInfo 構造体と、ワーカー数分の HashInstrumentation 配列を保持するために必要なサイズを計算する。 3. shm_toc_allocate を使用して、共有メモリコンテキスト（pcxt->toc）から指定サイズの領域を割り当てる。 4. 割り当てた領域をゼロクリアする。 5. node->shared_info に割り当てた領域のアドレスを格納し、ワーカー数を設定する。 6. shm_toc_insert を呼び出し、プランノードIDをキーとして共有メモリ内の領域を登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:517（直接的な呼び出しが確認されている）。
  • **呼び出し先**: 共有メモリ管理関数（shm_toc_allocate, shm_toc_insert）およびメモリ操作関数（memset）。

関連データ構造

  • HashState: ハッシュノードの実行状態を保持。
  • SharedHashInfo: 並列実行時のハッシュ結合に関する共有情報を保持する構造体。
  • HashInstrumentation: 各ワーカーの実行統計情報を保持する構造体。
  • ParallelContext: 並列実行のコンテキスト情報。

失敗モードと境界条件

  • pcxt->toc が無効な場合や、共有メモリの割り当てに失敗した場合は、基盤となる shm_toc_allocate の挙動に従いエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は並列実行時の統計収集に特化している。並列処理の理解には Barrier および DSA（Dynamic Shared Area）の概念が不可欠である。
  • node->shared_info へのアクセスは、並列ワーカー間での同期が適切に行われていることを前提としている。
