GenericXLogState

定義

  • 種別: struct（不透明な構造体として宣言）
  • 定義箇所: src/include/access/generic_xlog.h:30

目的

汎用的な WAL（Write-Ahead Logging）記録プロセスにおいて、現在進行中のセッション状態を保持するために使用されます。

シグネチャまたは形状

typedef struct GenericXLogState GenericXLogState; （詳細は src/backend/access/transam/generic_xlog.c 内で定義される不透明な型です）

入力と出力

  • 入力: GenericXLogStart() により初期化され、GenericXLogRegisterBuffer() を通じてバッファの登録に使用されます。
  • 出力: GenericXLogFinish() に渡されることで、登録された変更の確定と WAL レコードの書き出しを制御します。

前提条件と不変条件

  • GenericXLogStart で開始し、GenericXLogFinish で終了するライフサイクルに従う必要があります。
  • GenericXLogRegisterBuffer で取得したページポインタに対してのみ変更を行う必要があります。それ以外の場所でページを直接変更すると、WAL 記録が漏れ、リカバリ時にデータ破損を引き起こします。

アルゴリズムまたは振る舞い

  • ページレベルの変更を追跡し、GenericXLogFinish 呼び出し時に登録されたバッファの変更内容を比較して差分（デルタ）を計算します。
  • デルタ計算には MATCH_THRESHOLD に基づく最適化が適用され、隣接する変更はマージされます。
  • GENERIC_XLOG_FULL_IMAGE フラグが指定された場合、差分ではなくページ全体のイメージを記録します。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照の証拠）:
  • contrib/bloom/blinsert.c
  • contrib/bloom/blutils.c
  • contrib/bloom/blvacuum.c
  • 呼び出し先:
  • src/backend/access/transam/generic_xlog.c 内の GenericXLogRegisterBuffer および GenericXLogFinish 等の API が本構造体を操作します。

関連データ構造

  • Buffer: 変更対象のバッファ。
  • GENERIC_XLOG_FULL_IMAGE: 記録モードを制御するフラグ。

失敗モードと境界条件

  • ページ内の変更が大規模な場合や頻繁な更新が発生する場合、汎用 WAL のオーバーヘッドが大きくなる可能性があります。
  • デルタ計算の最適化により、隣接する変更はマージされます。

保守担当者向けメモ

  • 本構造体は不透明な型として扱われるため、実装の詳細については src/backend/access/transam/generic_xlog.c を参照してください。
  • ページ変更の記録漏れはリカバリ時のデータ破損に直結するため、API の使用ルールを厳守してください。
