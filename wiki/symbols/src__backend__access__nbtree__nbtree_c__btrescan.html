btrescan

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtree.c:384

目的

B-tree インデックススキャンの状態をリセットし、新しいスキャンキーを使用してスキャンを再開する。

シグネチャまたは形状

void btrescan(IndexScanDesc scan, ScanKey scankey, int nscankeys, ScanKey orderbys, int norderbys)

入力と出力

  • **入力**:
  • scan: スキャン対象のインデックス記述子。
  • scankey: 新しいスキャンキーの配列（NULL の場合もある）。
  • nscankeys: スキャンキーの数。
  • orderbys, norderbys: 並び替えキー（本関数内では未使用）。
  • **出力**: なし（副作用として scan 構造体内部の状態を更新）。

前提条件と不変条件

  • scan->opaque は BTScanOpaque 型としてキャスト可能であること。
  • so->dropPin はリスキャンを通じて変更されないこと。
  • インデックス専用スキャン（Index-only scan）において、タプルワークスペースが未割り当ての場合、BLCKSZ * 2 分のメモリが確保される。

アルゴリズムまたは振る舞い

1. **現在のスキャン状態のクリーンアップ**: 現在のページにピンがある場合、_bt_killitems を呼び出して削除対象アイテムを処理し、ピンを解放（Unpin）して無効化する。 2. **スキャンフラグの更新**: so->dropPin をスキャン条件（MVCC スナップショット、WAL 記録の有無等）に基づいて再計算する。 3. **内部状態のリセット**: markPos のピン解放、markItemIndex、needPrimScan、scanBehind、oppositeDirCheck などのフラグを初期化する。 4. **ワークスペースの確保**: インデックス専用スキャンが必要で、かつ currTuples が未割り当ての場合、メモリを割り当てる。 5. **スキャンキーの更新**: scankey が提供されている場合、scan->keyData を更新し、内部のキーカウントをリセットする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtree.c 内の amroutine->amrescan として登録されており、PostgreSQL の実行エンジンから呼び出される。nbtpreprocesskeys.c に関連する言及あり。
  • **呼び出し先**: _bt_killitems（削除対象アイテムがある場合）、palloc（ワークスペース確保時）、BTScanPosUnpinIfPinned、BTScanPosInvalidate。

関連データ構造

  • IndexScanDesc: スキャン全体の記述子。
  • BTScanOpaque: B-tree 特有のスキャン状態（currPos, markPos, currTuples 等を保持）。
  • ScanKey: 検索条件を保持する構造体。

失敗モードと境界条件

  • scankey が NULL の場合、キーのコピー処理はスキップされる。
  • インデックス専用スキャンにおいて、name_ops を使用する場合、NAMEDATALEN の読み取りが markTuples 領域に及ぶ可能性があるが、メモリ保護の観点から安全に設計されている。
  • 未ログ（unlogged）リレーションや非 MVCC スナップショット使用時は、デッドロックや競合回避のためピンの早期解放を行わない。

保守担当者向けメモ

  • 本関数は btendscan と対になるメモリ管理（currTuples の割り当て）を行っている。
  • so->dropPin のロジックは、VACUUM との競合（TID リサイクル）を回避するための重要な機構である。変更時は nbtree/README の TID リサイクルに関するセクションを参照すること。
