heapam_index_fetch_reset

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_handler.c:92

目的

インデックススキャンに関連付けられたヒープバッファを解放し、スキャン状態をリセットする。

シグネチャまたは形状

static void heapam_index_fetch_reset(IndexFetchTableData *scan)

入力と出力

  • **入力**: scan - IndexFetchTableData 構造体へのポインタ（実際には IndexFetchHeapData としてキャストされる）。
  • **出力**: なし。

前提条件と不変条件

  • scan は IndexFetchHeapData 型のデータ構造としてキャスト可能であること。
  • hscan->xs_cbuf が有効なバッファであるか、あるいは InvalidBuffer であること。

アルゴリズムまたは振る舞い

1. 引数 scan を IndexFetchHeapData 型のポインタ hscan にキャストする。 2. hscan->xs_cbuf が有効なバッファ（BufferIsValid）であるかを確認する。 3. 有効であれば ReleaseBuffer を呼び出してバッファを解放し、hscan->xs_cbuf を InvalidBuffer に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c 内のテーブルAMインターフェース実装の一部として呼び出される可能性がある。
  • **呼び出し先**:
  • ReleaseBuffer (ストレージマネージャ)

関連データ構造

  • IndexFetchTableData: テーブルAMの汎用スキャンデータ構造。
  • IndexFetchHeapData: ヒープAM固有のスキャンデータ構造。xs_cbuf（現在のバッファ）を保持する。

失敗モードと境界条件

  • xs_cbuf が既に InvalidBuffer である場合、ReleaseBuffer は呼び出されず、何も行われない（安全に終了する）。

保守担当者向けメモ

  • 本関数は TableAmRoutine の index_fetch_reset コールバックとして登録されている。
  • バッファの解放のみを行うシンプルな関数であり、複雑な状態遷移は含まない。
