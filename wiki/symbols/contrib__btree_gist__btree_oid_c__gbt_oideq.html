gbt_oideq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_oid.c:38

目的

GiSTインデックスにおいて、2つの Oid（Object Identifier）値が等しいかどうかを判定します。

シグネチャまたは形状

static bool gbt_oideq(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Oid 値を指すポインタ。
  • flinfo: 関数管理情報（本関数内では未使用）。
  • **出力**:
  • bool: a と b が等しい場合は true、そうでない場合は false を返します。

前提条件と不変条件

  • a および b は、有効な Oid 型の値を指している必要があります。
  • 本関数は Oid 型の直接比較を行うため、メモリ上の値が正しく初期化されていることを前提とします。

アルゴリズムまたは振る舞い

1. a と b をそれぞれ const Oid * 型にキャストします。 2. ポインタが指す値をデリファレンスし、C言語の等価演算子 == を用いて比較します。 3. 比較結果をそのまま戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_oid.c:91 において、gbtree_ninfo 構造体の初期化時に参照されています。
  • **呼び出し先**: PostgreSQLの組み込み演算子 == を使用します。

関連データ構造

  • Oid: PostgreSQLのオブジェクト識別子型。
  • gbtree_ninfo: Oid 型の比較ロジックを保持する構造体（本関数が登録先）。

失敗モードと境界条件

  • ポインタ a または b が NULL の場合、デリファレンスによりセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は btree_gist モジュールにおける Oid 型の比較アダプターです。
  • flinfo 引数は現在の実装では使用されていませんが、GiSTのサポート関数インターフェースとの互換性のために保持されています。
