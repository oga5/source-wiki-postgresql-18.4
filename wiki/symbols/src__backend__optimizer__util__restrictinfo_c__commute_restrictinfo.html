commute_restrictinfo

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/restrictinfo.c:349

目的

既存の RestrictInfo ノードの結合条件（OpExpr）を入れ替え（交換法則の適用）、左右の引数や関連するメタデータを反転させた新しい RestrictInfo ノードを生成する。

シグネチャまたは形状

RestrictInfo *commute_restrictinfo(RestrictInfo *rinfo, Oid comm_op)

入力と出力

  • **入力**:
  • rinfo: 入れ替え対象の RestrictInfo 構造体。
  • comm_op: 入れ替え後の演算子に対応する Oid。
  • **出力**:
  • 入れ替えられた新しい RestrictInfo 構造体へのポインタ。

前提条件と不変条件

  • rinfo->clause は OpExpr 型である必要がある。
  • clause->args は引数を2つ持つ必要がある（Assert により検証）。
  • rinfo->orclause は NULL であること（Assert により検証）。
  • rinfo_serial は元のノードから引き継がれる。

アルゴリズムまたは振る舞い

1. 元の OpExpr をフラットコピーし、演算子を comm_op に変更、引数の順序を入れ替える。 2. 元の RestrictInfo をフラットコピーし、以下のフィールドを反転・更新する:

  • left_relids / right_relids
  • left_ec / right_ec
  • left_em / right_em
  • left_bucketsize / right_bucketsize
  • left_mcvfreq / right_mcvfreq

    • 3. hashjoinoperator が元の演算子と一致する場合、comm_op に更新する。 4. scansel_cache はクリアし、left_hasheqoperator および right_hasheqoperator は InvalidOid にリセットする。 5. コスト見積もりや選択率のキャッシュは、入れ替え後も有効であるとみなして保持する。

呼び出し元と呼び出し先

  • **呼び出し先**:

関連データ構造

  • RestrictInfo: 変換対象および生成される構造体。
  • OpExpr: 結合条件式を保持するノード。

失敗モードと境界条件

  • clause->args が2つでない場合、Assert が有効なビルドではアボートする。
  • orclause が存在する場合、Assert が有効なビルドではアボートする。

保守担当者向けメモ

  • 本関数は RestrictInfo のメタデータを手動で反転させるため、新しいフィールドが RestrictInfo 構造体に追加された場合は、本関数内でのコピー・反転処理が漏れていないか確認が必要である。
  • scansel_cache の更新はコストに見合わないため意図的にスキップされている。
