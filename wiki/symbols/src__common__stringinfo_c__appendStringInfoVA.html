appendStringInfoVA

定義

  • 種別: function
  • 定義箇所: src/common/stringinfo.c:186

目的

StringInfo バッファに対して、printf 形式のフォーマット済み文字列を安全に追加します。バッファ容量が不足している場合は、必要なサイズを算出して呼び出し元に通知します。

シグネチャまたは形状

int appendStringInfoVA(StringInfo str, const char *fmt, va_list args)

入力と出力

  • **入力**:
  • str: 操作対象の StringInfo 構造体へのポインタ。
  • fmt: フォーマット文字列。
  • args: 可変長引数リスト (va_list)。
  • **出力**:
  • 0: 成功。
  • 0 以外: 失敗（バッファ不足）。戻り値は、書き込みに必要なバッファサイズの推定値（バイト単位）を表します。

前提条件と不変条件

  • str は NULL であってはなりません（Assert で検証）。
  • StringInfo の内部バッファは常にヌル終端が維持されます。
  • 戻り値が 0 以外の場合、バッファの内容は変更されません（書き込み試行前にヌル終端を復元するため）。

アルゴリズムまたは振る舞い

1. 現在のバッファの空き容量 (avail) を確認します。 2. avail が 16 バイト未満の場合、フォーマット処理を行わずに 32 を返して呼び出し元に拡張を促します。 3. pvsnprintf を使用してバッファへ書き込みます。 4. 書き込みがバッファ内に収まった場合、str->len を更新して 0 を返します。 5. 書き込みが収まらなかった場合、元のヌル終端を復元し、pvsnprintf が要求したサイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/stringinfo.c (appendStringInfo)
  • src/backend/backup/basebackup_incremental.c
  • src/backend/backup/walsummary.c
  • src/backend/utils/error/elog.c
  • src/pl/plpython/plpy_elog.c
  • **呼び出し先**:
  • pvsnprintf (フォーマット書き込み用)

関連データ構造

  • StringInfoData: data（文字列バッファ）、len（現在長）、maxlen（最大容量）を保持する構造体。

失敗モードと境界条件

  • **バッファ不足**: avail が不足している場合、書き込みを行わずに必要なサイズを返します。
  • **読み取り専用バッファ**: maxlen == 0 の場合、Assert により停止する可能性があります。

保守担当者向けメモ

  • appendStringInfo は va_list の再走査を避けるため、本関数を呼び出す設計になっています。
  • 頻繁な追加が発生する箇所では、事前に enlargeStringInfo を呼び出すことで、本関数内での失敗と再試行のオーバーヘッドを削減可能です。
