RT_NODE

定義

  • 種別: struct
  • 定義箇所: src/include/lib/radixtree.h:374

目的

適応型基数木（ART）における各ノードの共通ヘッダーとして機能し、ノードの種別、容量、および現在の格納子ノード数を管理する。

シグネチャまたは形状

typedef struct RT_NODE
{
    uint8    kind;    /* ノード種別 */
    uint8    fanout;  /* 現在のサイズクラスにおける最大容量 */
    uint8    count;   /* 子ノード数 */
} RT_NODE;

入力と出力

  • 本構造体は、基数木の各階層におけるノードのメタデータを保持する。
  • count はノードの成長や縮小の判断基準として利用される。

前提条件と不変条件

  • count が閾値を下回るとノードは縮小され、上回ると成長する。
  • node256 型の場合、count が 0 のときは 256 個の要素を持つものとして解釈される。
  • node256 型の fanout はオーバーフローして 0 となる。

アルゴリズムまたは振る舞い

  • 基数木の階層的な検索や挿入において、ノードのサイズクラスを識別し、後続の処理（成長・縮小）を決定するために参照される。
  • RT_NODE_SEARCH などの関数により、特定のキーチャンクに対するノード内の遷移が制御される。

呼び出し元と呼び出し先

  • 呼び出し元: RT_VERIFY_NODE、RT_NODE_SEARCH、および各ノード型（RT_NODE_4, RT_NODE_16 等）の基底構造体として利用されている。
  • 呼び出し先: 具体的な関数呼び出し関係はソースコードの断片からは特定できない。

関連データ構造

  • RT_RADIX_TREE: 基数木のメイン構造体。
  • RT_NODE_4, RT_NODE_16, RT_NODE_48, RT_NODE_256: 本構造体を基底として内包する各サイズのノード構造体。

失敗モードと境界条件

  • fanout および count は uint8 型であるため、ノード種別ごとの最大容量を超えないよう設計されている。
  • node256 における count のオーバーフロー処理（0 を 256 と解釈）が境界条件として存在する。

保守担当者向けメモ

  • 本構造体はマクロ展開によって生成されるテンプレートの一部である。
  • RT_NODE を含む構造体は、メモリレイアウトがノード種別ごとに異なるため、キャストやポインタ演算を行う際は kind フィールドによる型チェックが必須である。
