bytea_1

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-bytea.c:54

目的

ECPG（Embedded SQL for C）において、SQL の bytea 型データを C 言語側で保持・送受信するための構造体定義です。

シグネチャまたは形状

struct bytea_1 {
    int len;
    char arr[512];
};

入力と出力

  • **入力**: SQL 文の実行結果（SELECT 等）を格納するバッファとして使用されます。
  • **出力**: SQL 文（INSERT 等）へバイナリデータを渡すためのホスト変数として使用されます。

前提条件と不変条件

  • len メンバには、arr に格納された有効なバイナリデータのバイト数を正確に設定する必要があります。
  • arr は固定長（512 バイト）のバッファとして確保されています。

アルゴリズムまたは振る舞い

ECPG ランタイムライブラリの ECPGdo 関数を通じて、SQL の bytea 型と C 言語の構造体間のマッピングが行われます。テストコード内では、この構造体の配列 send_buf が定義され、バイナリデータの送受信テストに使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/sql-bytea.c 内の複数の SQL 実行箇所（ECPGdo 呼び出し時）。
  • **呼び出し先**: ECPG ランタイムライブラリ（ECPGt_bytea 型指定子を介して参照）。

関連データ構造

  • send_buf: bytea_1 型の 2 要素配列。
  • ECPGt_bytea: ECPG が bytea 型を識別するために使用する型識別子。

失敗モードと境界条件

  • arr のサイズ（512 バイト）を超えるデータを格納しようとした場合、バッファオーバーフローのリスクがあります。
  • len が不適切な値（負の値や 512 を超える値）に設定された場合、SQL 実行時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はプリプロセッサ生成コードの一部であるため、直接の編集は避けてください。
  • bytea 型のテストにおいて、バッファサイズ不足の挙動を確認する際は LACK_SIZE 関連のテストケースを参照してください。
