pg_strerror_r

定義

  • 種別: function
  • 定義箇所: src/port/strerror.c:45

目的

プラットフォーム間で差異のある strerror_r の挙動を抽象化し、スレッドセーフな方法でエラー番号に対応するメッセージ文字列を取得する。

シグネチャまたは形状

char *pg_strerror_r(int errnum, char *buf, size_t buflen)

入力と出力

  • **入力**:
  • errnum: 変換対象のエラー番号（errno）。
  • buf: メッセージを格納する呼び出し元が提供するバッファ。
  • buflen: buf のサイズ。
  • **出力**:
  • エラーメッセージを格納した buf へのポインタを返す。

前提条件と不変条件

  • buf は buflen 以上のサイズを持つ有効なメモリ領域であること。
  • Windows 環境では、Winsock エラーコード（10000〜11999）が渡された場合、専用の処理が行われる。

アルゴリズムまたは振る舞い

1. **Windows 特殊処理**: WIN32 定義下で Winsock エラー範囲内の場合、win32_socket_strerror を呼び出しその結果を返す。 2. **標準取得**: gnuish_strerror_r を呼び出し、OS 標準のメッセージ取得を試みる。 3. **フォールバック**: 取得した文字列が空、NULL、または '?' で始まる場合、get_errno_symbol を用いてシンボル名（例: "ENOENT"）の取得を試みる。 4. **最終手段**: それでも有効な文字列が得られない場合、snprintf を使用して "operating system error %d" という形式の文字列を buf に書き込み、そのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/strerror.c 内の他の関数（pg_strerror 等）から呼び出される。
  • **呼び出し先**:
  • win32_socket_strerror (Windows 環境のみ)
  • gnuish_strerror_r
  • get_errno_symbol
  • snprintf

関連データ構造

  • c.h: PG_STRERROR_R_BUFLEN などの定数定義が含まれる。

失敗モードと境界条件

  • errnum がシステムで定義されていない場合、数値ベースのフォールバックメッセージが生成される。
  • buflen が極端に小さい場合、snprintf の仕様に従いメッセージが切り詰められる可能性がある。

保守担当者向けメモ

  • 本関数は strerror のスレッドセーフではない問題を回避するために設計されている。
  • get_errno_symbol は網羅的ではないため、未知のエラーコードに対しては数値出力が最終的な保護策となる。
