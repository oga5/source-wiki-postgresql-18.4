pg_switch_wal

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogfuncs.c:176

目的

現在のWAL（Write-Ahead Log）セグメントを強制的に切り替え、新しいWALファイルを開始します。

シグネチャまたは形状

Datum pg_switch_wal(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQLインターフェース経由の引数）。
  • **出力**: Datum 型として、切り替えが発生した時点のWAL位置（LSN: Log Sequence Number）を返却します。

前提条件と不変条件

  • サーバーがリカバリ中（RecoveryInProgress()）ではないこと。リカバリ中の場合はエラーが発生します。

アルゴリズムまたは振る舞い

1. RecoveryInProgress() を呼び出し、サーバーがリカバリ状態にあるかを確認します。リカバリ中の場合は ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE エラーを発生させます。 2. RequestXLogSwitch(false) を呼び出し、WALの強制切り替えを実行します。 3. 切り替えが完了した時点のWAL位置（switchpoint）を取得します。 4. 取得した switchpoint を PG_RETURN_LSN マクロを使用して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（ユーザーによるSQLコマンド実行）。
  • **呼び出し先**: RequestXLogSwitch（src/backend/access/transam/xlog.c 内のWAL管理ロジック）。

関連データ構造

  • XLogRecPtr: WAL上の位置を示すデータ型。

失敗モードと境界条件

  • **リカバリ中の実行**: サーバーがスタンバイモードやリカバリ中の場合、関数はエラーで終了します。
  • **内部エラー**: RequestXLogSwitch 内部でWALの書き込みやファイルシステム操作に失敗した場合、PostgreSQLの標準的なエラー処理メカニズムによりトランザクションがアボートされます。

保守担当者向けメモ

  • 本関数はSQL層とWAL管理層を繋ぐブリッジです。WALの切り替えロジック自体は RequestXLogSwitch に依存しているため、切り替えの挙動を変更する場合は xlog.c を参照してください。
  • 戻り値のLSNは、バックアップの終了位置やリストアポイントの特定など、他のWAL関連関数と組み合わせて利用されることが想定されています。
