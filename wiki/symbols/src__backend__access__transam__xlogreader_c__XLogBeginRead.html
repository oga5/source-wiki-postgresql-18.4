XLogBeginRead

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogreader.c:231

目的

WAL読み取り処理を開始するために、XLogReaderState を指定されたLSN（Log Sequence Number）の位置に初期化します。

シグネチャまたは形状

void XLogBeginRead(XLogReaderState *state, XLogRecPtr RecPtr)

入力と出力

  • state: 操作対象の XLogReaderState 構造体へのポインタ。
  • RecPtr: 読み取りを開始する対象のWALレコード位置（XLogRecPtr）。
  • 出力: なし（副作用として state の内部状態が更新されます）。

前提条件と不変条件

  • RecPtr は有効な値（!XLogRecPtrIsInvalid(RecPtr)）である必要があります。
  • 本関数を呼び出すことで、リーダーは指定された位置からレコードを読み取る準備状態となります。

アルゴリズムまたは振る舞い

1. ResetDecoder(state) を呼び出し、デコーダーの状態をリセットします。 2. state->EndRecPtr および state->NextRecPtr を指定された RecPtr に設定します。 3. state->ReadRecPtr および state->DecodeRecPtr を InvalidXLogRecPtr に初期化し、読み取り位置をリセットします。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/access/transam/twophase.c
  • src/backend/access/transam/xlogprefetcher.c
  • src/backend/access/transam/xlogreader.c（内部の再初期化処理など）
  • 呼び出し先:
  • ResetDecoder

関連データ構造

  • XLogReaderState: WAL読み取りのコンテキストを保持するメイン構造体。
  • XLogRecPtr: WAL上の位置を示す64ビット整数。

失敗モードと境界条件

  • 無効な RecPtr が渡された場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は、WALのシーケンシャルな読み取りを開始する際の必須ステップです。
  • XLogFindNextRecord とは異なり、指定された RecPtr がレコードの開始位置であることを前提としています。
