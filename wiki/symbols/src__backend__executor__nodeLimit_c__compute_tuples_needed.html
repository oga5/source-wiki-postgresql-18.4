compute_tuples_needed

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeLimit.c:430

目的

下位ノード（サブプラン）に対して、クエリの LIMIT および OFFSET を満たすために必要なタプルの総数を通知し、実行効率を最適化する。

シグネチャまたは形状

static int64 compute_tuples_needed(LimitState *node)

入力と出力

  • **入力**: LimitState *node（現在の Limit ノードの状態構造体）
  • **出力**: int64 型の必要なタプル数。計算不能または無制限の場合は -1 を返す。

前提条件と不変条件

  • node は有効な LimitState 構造体を指している必要がある。
  • 計算結果がオーバーフローした場合、負の値が返されるが、これは呼び出し側で許容される。

アルゴリズムまたは振る舞い

1. node->noCount が真（制限なし）であるか、node->limitOption が LIMIT_OPTION_WITH_TIES である場合、最適化が適用できないため -1 を返す。 2. 上記以外の場合、node->count（取得数）と node->offset（スキップ数）の合計値を算出して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeLimit.c 内の ExecSetTupleBound を呼び出す箇所（423行目付近）。
  • **呼び出し先**: 本関数は他の関数を呼び出さない。

関連データ構造

  • LimitState: src/include/nodes/execnodes.h で定義される、Limit ノードの実行時状態を保持する構造体。

失敗モードと境界条件

  • LIMIT や OFFSET の値が非常に大きい場合、加算結果がオーバーフローして負の値になる可能性がある。この挙動は意図されており、呼び出し側で適切に処理される。
  • WITH TIES 指定時は、正確な必要数が事前に確定できないため -1 を返す。

保守担当者向けメモ

  • 本関数は最適化のためのヒントを生成するものであり、正確なタプル数が不明な場合は安全側に倒して -1 を返す設計となっている。
  • ExecSetTupleBound への入力として使用されるため、計算ロジックを変更する際は下位ノード（Sort 等）への影響を考慮すること。
