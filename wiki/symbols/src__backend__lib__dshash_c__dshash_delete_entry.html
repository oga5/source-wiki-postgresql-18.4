dshash_delete_entry

定義

  • 種別: function
  • 定義箇所: src/backend/lib/dshash.c:540

目的

共有ハッシュテーブルから指定されたエントリを削除し、対応するパーティションロックを解放する。

シグネチャまたは形状

void dshash_delete_entry(dshash_table *hash_table, void *entry)

入力と出力

  • hash_table: 操作対象の dshash_table 構造体へのポインタ。
  • entry: 削除対象のエントリ（dshash_table_item に対応するデータ領域）へのポインタ。
  • 出力: なし（void）。

前提条件と不変条件

  • hash_table->control->magic が DSHASH_MAGIC であること。
  • 呼び出し元が、対象エントリが属するパーティションの LWLock を LW_EXCLUSIVE モードで保持していること。

アルゴリズムまたは振る舞い

1. ITEM_FROM_ENTRY マクロを用いて、渡された entry から内部管理用の dshash_table_item を特定する。 2. PARTITION_FOR_HASH マクロを用いて、アイテムのハッシュ値から該当するパーティション番号を算出する。 3. 呼び出し元が当該パーティションの排他ロックを保持していることを Assert で検証する。 4. delete_item を呼び出し、ハッシュテーブルからアイテムを削除する。 5. 最後に LWLockRelease を呼び出し、当該パーティションのロックを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/activity/pgstat_shmem.c 内の複数箇所。
  • src/backend/lib/dshash.c 内の内部ドキュメント参照。
  • 呼び出し先:
  • delete_item（内部関数）
  • LWLockRelease（storage/lwlock.h）

関連データ構造

  • dshash_table: ハッシュテーブルの制御構造体。
  • dshash_table_item: ハッシュテーブル内の各エントリを管理する構造体。

失敗モードと境界条件

  • ロックを保持せずに本関数を呼び出した場合、Assert が有効なビルドではアサーション失敗によりプロセスが停止する。
  • 無効な entry ポインタを渡した場合の挙動は未定義。

保守担当者向けメモ

  • 本関数は、呼び出し元がロックの取得と解放の責任を一部負う設計となっている。ロックの解放は本関数内で行われるため、呼び出し側で二重に解放しないよう注意が必要。
