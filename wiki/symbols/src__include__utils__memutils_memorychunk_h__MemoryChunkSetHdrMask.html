MemoryChunkSetHdrMask

定義

  • 種別: function
  • 定義箇所: src/include/utils/memutils_memorychunk.h:168

目的

メモリコンテキストによって割り当てられたチャンクのヘッダー情報（hdrmask）を初期化し、チャンクと親ブロック間のオフセット、およびコンテキスト固有のメタデータをエンコードする。

シグネチャまたは形状

static inline void MemoryChunkSetHdrMask(MemoryChunk *chunk, void *block, Size value, MemoryContextMethodID methodid)

入力と出力

  • chunk: 初期化対象の MemoryChunk 構造体へのポインタ。
  • block: チャンクを含む親ブロックの開始アドレス。
  • value: チャンクに関連付ける任意のサイズまたは値。
  • methodid: メモリコンテキストのメソッド識別子。
  • 出力: なし（chunk->hdrmask を直接更新する）。

前提条件と不変条件

  • chunk は block 以降のアドレスを指していること（Assert により検証）。
  • blockoffset（chunk - block）は MEMORYCHUNK_BLOCKOFFSET_MASK の範囲内に収まること。
  • value は MEMORYCHUNK_MAX_VALUE 以下であること。
  • methodid は MEMORY_CONTEXT_METHODID_MASK 以下であること。

アルゴリズムまたは振る舞い

1. chunk と block の差分から blockoffset を算出する。 2. 以下の各値を指定されたビット位置へシフトし、ビット論理和（OR）をとって hdrmask を生成する。

  • blockoffset: MEMORYCHUNK_BLOCKOFFSET_BASEBIT 分シフト。
  • value: MEMORYCHUNK_VALUE_BASEBIT 分シフト。
  • methodid: シフトなし。

    • 3. 生成した hdrmask を chunk->hdrmask に格納する。

呼び出し元と呼び出し先

  • 呼び出し元: aset.c、bump.c、generation.c、mcxt.c、slab.c 等のメモリコンテキスト実装において、チャンク割り当て時に呼び出される。
  • 呼び出し先: なし（インライン関数であり、内部で Assert マクロを使用する）。

関連データ構造

  • MemoryChunk: チャンクヘッダーを定義する構造体。
  • MemoryContextMethodID: メモリコンテキストの型を識別する列挙型または定数。

失敗モードと境界条件

  • blockoffset が MEMORYCHUNK_BLOCKOFFSET_MASK を超える場合、アサーション違反が発生する。
  • value が MEMORYCHUNK_MAX_VALUE を超える場合、アサーション違反が発生する。
  • methodid が MEMORY_CONTEXT_METHODID_MASK を超える場合、アサーション違反が発生する。

保守担当者向けメモ

  • hdrmask は 64 ビットに情報を詰め込んでいるため、ビット配置の変更には MEMORYCHUNK_BLOCKOFFSET_BASEBIT 等の定数定義との整合性が必須である。
  • ブロックオフセットの最下位ビットは常に 0 である（MAXALIGN 制約）という前提に依存しているため、この制約を破る変更は不可。
