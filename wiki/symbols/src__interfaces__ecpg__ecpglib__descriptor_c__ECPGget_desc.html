ECPGget_desc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/ecpglib/descriptor.c:233

目的

SQL 記述子（Descriptor）から指定された属性やデータを取得し、アプリケーション側の変数へコピーします。

シグネチャまたは形状

bool ECPGget_desc(int lineno, const char *desc_name, int index, ...);

可変長引数（va_list）を使用して、取得対象の属性タイプおよび格納先変数の情報を順次受け取ります。

入力と出力

  • **入力**:
  • lineno: エラー報告用行番号。
  • desc_name: 取得対象の記述子名。
  • index: 取得する列のインデックス（1始まり）。
  • ...: 属性タイプ（enum ECPGdtype）と、それに対応する格納先変数情報（型、ポインタ、サイズ等）のペア。
  • **出力**: 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • ECPGget_sqlca() が有効な sqlca_t 構造体を返すこと。
  • 指定された desc_name に対応する記述子が、現在のスレッドのコンテキスト内に存在すること。
  • index が 1 から列数（PQnfields）の範囲内であること。

アルゴリズムまたは振る舞い

1. ecpg_result_by_descriptor を呼び出し、記述子に関連付けられた PGresult を取得します。 2. index の妥当性を検証し、内部的な 0 始まりのインデックスへ変換します。 3. 可変長引数を走査し、ECPGd_EODT（終了マーカー）に達するまで以下の処理を繰り返します。

  • ECPGd_indicator または ECPGd_data の場合、指定された変数情報へデータを格納します。
  • その他の属性（ECPGd_name, ECPGd_scale, ECPGd_precision 等）の場合、get_int_item または get_char_item を介して PGresult から値を抽出し、変数へコピーします。

    • 4. 処理の過程でエラーが発生した場合は ecpg_raise を呼び出し、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ECPG プリプロセッサが生成するコードから呼び出されます（具体的な呼び出し箇所はソースコードの静的解析からは特定できません）。
  • **呼び出し先**:
  • ecpg_result_by_descriptor, ECPGget_sqlca, ecpg_init_sqlca, ecpg_raise, ecpg_log
  • get_int_item, get_char_item
  • PQntuples, PQnfields, PQfname, PQfmod, PQfsize, PQftype (libpq 関数)

関連データ構造

  • PGresult: データベースからの結果セットを保持する libpq の構造体。
  • variable: データの格納先や型情報を保持する内部構造体。
  • sqlca_t: SQL 通信領域。

失敗モードと境界条件

  • sqlca が取得できない場合、メモリ不足エラーとして false を返します。
  • desc_name に対応する記述子が見つからない場合、false を返します。
  • index が範囲外の場合、ECPG_INVALID_DESCRIPTOR_INDEX エラーを発生させ false を返します。
  • 属性取得関数（get_int_item 等）が失敗した場合、処理を中断し false を返します。

保守担当者向けメモ

  • 可変長引数の解析順序は ECPG プリプロセッサの生成ロジックと密接に結合しています。変更時はプリプロセッサ側の出力との整合性に注意してください。
  • 数値変換時には LC_NUMERIC ロケールが "C" に設定される前提で動作します。
