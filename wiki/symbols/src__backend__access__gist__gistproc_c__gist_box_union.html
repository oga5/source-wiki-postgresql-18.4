gist_box_union

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistproc.c:163

目的

GiSTインデックスにおいて、複数の矩形（BOX）を包含する最小境界矩形（MBR）を計算する。

シグネチャまたは形状

Datum gist_box_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector へのポインタ。包含対象となる複数のエントリを保持。
  • PG_GETARG_POINTER(1): int へのポインタ（sizep）。計算結果のサイズを格納する出力用引数。
  • **出力**:
  • 戻り値: Datum としてキャストされた、全入力矩形を包含する新しい BOX 構造体へのポインタ。
  • *sizep: sizeof(BOX) が設定される。

前提条件と不変条件

  • 入力 entryvec は有効な GistEntryVector であること。
  • 入力される各キーは DatumGetBoxP を介して BOX 型として解釈可能であること。

アルゴリズムまたは振る舞い

1. entryvec から最初の矩形を取得し、それを初期の pageunion（MBR）とする。 2. 残りの全エントリを順次走査する。 3. 各エントリに対して adjustBox を呼び出し、現在の pageunion をそのエントリを包含するように拡張する。 4. 最終的な pageunion のサイズを *sizep に書き込み、ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には明示的な呼び出し元の記述なし。
  • **呼び出し先**:
  • DatumGetBoxP: キーの抽出。
  • adjustBox: 矩形の包含計算。
  • palloc: メモリ確保。

関連データ構造

  • GistEntryVector: 入力エントリの配列を保持。
  • BOX: 2次元矩形を表すデータ構造。

失敗モードと境界条件

  • entryvec->n が0の場合の挙動はソースコード上では明示されていない。
  • メモリ不足時には palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は BOX 型専用の処理を行う。polygon や circle 型もインデックス内では BOX に圧縮された状態で本関数に渡される。
  • adjustBox は src/backend/access/gist/gistproc.c 内で定義されている基本演算であり、MBRの更新ロジックの詳細はそちらを参照すること。
