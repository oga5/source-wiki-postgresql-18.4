BloomOptions

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/brin/brin_bloom.c:153

目的

BRIN（Block Range Index）におけるブルームフィルタの構築パラメータを保持するための構造体です。ページ範囲ごとの推定値数と許容誤判定率を定義し、フィルタのサイズ決定に利用されます。

シグネチャまたは形状

typedef struct BloomOptions
{
    int32       vl_len_;            /* varlenaヘッダー */
    double      nDistinctPerRange;  /* 範囲ごとの推定値数 */
    double      falsePositiveRate;  /* 許容誤判定率 */
} BloomOptions;

入力と出力

  • **入力**: インデックス作成時に指定されるオプション値（nDistinctPerRange, falsePositiveRate）。
  • **出力**: 本構造体は、BRINインデックスのメタデータとして保持され、フィルタの動的サイズ計算の入力として使用されます。

前提条件と不変条件

  • vl_len_ フィールドは varlena ヘッダーとして機能するため、直接操作してはなりません。
  • 本構造体は、BRINインデックスのページ範囲要約におけるブルームフィルタのパラメータを規定します。

アルゴリズムまたは振る舞い

本構造体で定義されたパラメータに基づき、brin_bloom_opcinfo 等の関数がブルームフィルタのサイズを計算します。誤判定率を低く設定するほど精度は向上しますが、フィルタサイズが増大し、インデックスの効率に影響を与える可能性があります。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソースコード内では、本構造体の具体的な呼び出し元は特定されていません。
  • **呼び出し先**: 本構造体はデータ保持用であり、関数を呼び出すことはありません。
  • **注意**: contrib/bloom モジュール内にも同名の構造体が存在しますが、本シンボル（src/backend/access/brin/brin_bloom.c）とは定義が異なります。混同しないよう注意してください。

関連データ構造

  • BloomFilter: 本構造体で設定されたパラメータを用いて初期化される、実際のブルームフィルタ構造体。

失敗モードと境界条件

  • ブルームフィルタのサイズが BloomMaxFilterSize（1つのインデックスページサイズである8kB）を超える場合、インデックス構築時にエラーが発生します。

保守担当者向けメモ

  • contrib/bloom 内の BloomOptions と名前が重複しているため、参照先が src/backend/access/brin/brin_bloom.c であることを確認してください。
  • 本構造体のフィールドを変更する場合は、brin_bloom_opcinfo におけるサイズ計算ロジックへの影響を評価してください。
