NumericVar

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/numeric.c:313

目的

PostgreSQL の NUMERIC 型に対する算術演算を行うための、メモリ上の作業用構造体です。ディスク上の格納形式（NumericData）から展開され、加減乗除などの計算処理に使用されます。

シグネチャまたは形状

typedef struct NumericVar
{
	int			ndigits;		/* digits[] 内の桁数 (0 も可) */
	int			weight;			/* 先頭桁の重み */
	int			sign;			/* NUMERIC_POS, _NEG, _NAN, _PINF, _NINF */
	int			dscale;			/* 表示スケール */
	NumericDigit *buf;			/* digits[] 用に palloc された領域の先頭 */
	NumericDigit *digits;		/* 基数 NBASE の数値配列 */
} NumericVar;

入力と出力

  • **入力**: numeric_in 等により、ディスク上の Numeric 形式から本構造体へ展開されます。
  • **出力**: 演算完了後、make_result() 等を通じて再び Numeric 形式にパックされます。

前提条件と不変条件

  • digits は palloc されたメモリ領域を指す必要があります。
  • メモリリークを防ぐため、使用後は free_var() を呼び出して buf を解放する必要があります。
  • digits 配列の各要素は基数 NBASE（10000）に基づきます。
  • 非有限値（NaN, PINF, NINF）は、この構造体でのみ表現されます。

アルゴリズムまたは振る舞い

  • 数値の値を「符号」「重み」「桁配列」の組み合わせで保持します。
  • 最初の桁は NBASE のべき乗で重み付けされます。
  • 算術演算関数（add_var, sub_var 等）は、この構造体を入力として受け取り、計算結果を別の NumericVar に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: numeric_in、算術演算関数（numeric_add 等）、集約関数。
  • **呼び出し先**: palloc（メモリ確保）、free_var（メモリ解放）、および各算術演算関数。
  • *注: 具体的な関数呼び出しの網羅的なリストは、提供されたソース excerpt の範囲外です。*

関連データ構造

  • NumericData: NUMERIC 型のオンディスク表現。
  • NumericDigit: digits 配列を構成する型。
  • NumericSumAccum: 集約計算の中間状態を保持する構造体。

失敗モードと境界条件

  • ndigits が 0 の場合、数値が 0 であることを示します。
  • メモリ不足時には palloc がエラーを発生させます。
  • 非有限値（NaN 等）が入力された場合、演算ロジックはそれに応じた特殊な処理を行います。

保守担当者向けメモ

  • NumericVar はメモリ上で動的に拡張される可能性があるため、buf の管理には注意が必要です。
  • NBASE=10000 であるため、1 桁あたり 4 桁の 10 進数を扱います。
  • NumericVar の各フィールドの整合性は、演算関数側で維持される必要があります。
