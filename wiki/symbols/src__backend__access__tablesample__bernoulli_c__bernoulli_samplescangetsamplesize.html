bernoulli_samplescangetsamplesize

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/bernoulli.c:85

目的

TABLESAMPLE BERNOULLI を使用したクエリのプランニング段階において、サンプリング後の推定ページ数および推定タプル数を算出します。

シグネチャまたは形状

static void bernoulli_samplescangetsamplesize(PlannerInfo *root,
                                              RelOptInfo *baserel,
                                              List *paramexprs,
                                              BlockNumber *pages,
                                              double *tuples)

入力と出力

  • **入力**:
  • root: プランナのコンテキスト情報。
  • baserel: 対象リレーションの統計情報。
  • paramexprs: サンプリング率を指定する式リスト。
  • **出力**:
  • pages: 推定される訪問ページ数（baserel->pages が代入される）。
  • tuples: 推定されるサンプリング後のタプル数。

前提条件と不変条件

  • paramexprs の先頭要素はサンプリング率（パーセンテージ）として評価可能である必要があります。
  • サンプリング率が 0 から 100 の範囲外、または評価不能な場合はデフォルト値 0.1 が使用されます。

アルゴリズムまたは振る舞い

1. estimate_expression_value を用いて paramexprs の先頭からサンプリング率を抽出します。 2. 抽出した値が定数かつ有効な範囲（0〜100）であれば、それを 100 で除算して samplefract とします。無効な場合は 0.1 を適用します。 3. *pages に baserel->pages を設定し、全ページを走査することを前提とします。 4. *tuples に baserel->tuples * samplefract を計算し、clamp_row_est を適用して結果を格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: tsm_bernoulli_handler 内で TsmRoutine 構造体の SampleScanGetSampleSize フィールドに登録され、プランナから間接的に呼び出されます。
  • **呼び出し先**: estimate_expression_value（式評価用）、clamp_row_est（推定値の正規化用）。

関連データ構造

  • PlannerInfo: プランナのグローバル情報。
  • RelOptInfo: リレーションの統計情報。
  • TsmRoutine: サンプリングメソッドのインターフェース構造体。

失敗モードと境界条件

  • サンプリング率が NULL または評価不能な定数の場合、デフォルト値 0.1 にフォールバックします。
  • samplefract が NaN の場合も同様にデフォルト値が使用されます。

保守担当者向けメモ

  • 本関数はプランナがコスト見積もりを行う際に使用されます。実行時の実際のサンプリングロジック（bernoulli_nextsampletuple）とは独立した統計計算である点に注意してください。
