sepgsql_client_auth

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/label.c:229

目的

クライアント認証完了時に、接続元プロセスのSELinuxセキュリティコンテキストを取得し、SEPGSQLの動作モードを初期化する。

シグネチャまたは形状

static void sepgsql_client_auth(Port *port, int status)

入力と出力

  • port: 接続中のクライアントポート情報。
  • status: 認証結果（STATUS_OK 等）。
  • 戻り値: なし。

前提条件と不変条件

  • ClientAuthentication_hook に登録され、認証プロセスの一部として呼び出される。
  • 認証失敗時（status != STATUS_OK）は、ソケットがクローズされるため処理を中断する。

アルゴリズムまたは振る舞い

1. **フックの連鎖**: next_client_auth_hook が設定されている場合、それを呼び出す。 2. **認証確認**: status が STATUS_OK でない場合は即座に終了する。 3. **ラベル取得**: getpeercon_raw を使用し、クライアントプロセスのセキュリティラベルを client_label_peer に取得する。失敗した場合は FATAL エラーを発生させる。 4. **モード設定**: sepgsql_get_permissive() の結果に基づき、sepgsql_set_mode を呼び出して動作モードを SEPGSQL_MODE_PERMISSIVE または SEPGSQL_MODE_DEFAULT に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ClientAuthentication_hook を通じて認証プロセスから呼び出される（contrib/sepgsql/label.c:423 でフック登録）。
  • **呼び出し先**:
  • next_client_auth_hook（存在する場合）
  • getpeercon_raw（libselinux API）
  • sepgsql_get_permissive
  • sepgsql_set_mode

関連データ構造

  • Port: PostgreSQLの接続ポート構造体。
  • client_label_peer: クライアントのセキュリティラベルを保持する変数。

失敗モードと境界条件

  • getpeercon_raw が失敗した場合、FATAL エラーによりプロセスが終了する。
  • 認証が成功していない場合、ラベル取得処理はスキップされる。

保守担当者向けメモ

  • 本関数は ClientAuthentication_hook の一部として動作するため、認証フロー全体の整合性に影響を与える。
  • getpeercon_raw の呼び出しは、クライアントプロセスがSELinux環境下にあることを前提としている。
