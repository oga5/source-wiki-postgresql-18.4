findprefix

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regprefix.c:115

目的

コンパイル済みの正規表現（NFA）を走査し、マッチングの先頭に必ず現れる共通の接頭辞を抽出します。

シグネチャまたは形状

static int findprefix(struct cnfa *cnfa, struct colormap *cm, chr *string, size_t *slength)

入力と出力

  • **入力**:
  • cnfa: 解析対象のコンパイル済みNFA構造体。
  • cm: カラーマップ。
  • string: 接頭辞を格納するバッファ。
  • slength: 接頭辞の長さを保持するポインタ。
  • **出力**:
  • REG_EXACT: 完全一致する文字列が特定された場合。
  • REG_PREFIX: 共通の接頭辞が特定された場合。
  • REG_NOMATCH: 接頭辞が特定できなかった場合。

前提条件と不変条件

  • 正規表現は左側にアンカー（BOS/BOL）されている必要があります。
  • 抽出される接頭辞は、カラーマップ上の単一の文字（singleton）である必要があります。
  • 抽出対象のカラーは、高位の文字マップ（high-color-map）を含まない必要があります。

アルゴリズムまたは振る舞い

1. **開始状態の検証**: NFAの開始状態からBOS/BOLアークを辿り、単一の遷移先を特定します。 2. **状態遷移の走査**:

  • 各状態において、単一の文字（カラー）への遷移のみが存在するかを確認します。
  • EOS/EOL、RAINBOW、または複数のカラーへの遷移に遭遇した時点で走査を終了します。
  • カラーが単一の文字（singleton）であることを確認し、string に追加します。

    • 3. **終了判定**: 走査終了時の状態が終了状態（post）に直結している場合、REG_EXACT を返します。それ以外で接頭辞が抽出されていれば REG_PREFIX を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regprefix.c 内の pg_regprefix（推論される直接の呼び出し元）。
  • **呼び出し先**: GETCOLOR マクロ（カラーマップの検証に使用）。

関連データ構造

  • cnfa: 正規表現のNFA表現。
  • colormap: 文字とカラーの対応関係を保持。
  • carc: NFAのアーク構造体。

失敗モードと境界条件

  • 左側がアンカーされていない場合、REG_NOMATCH を返します。
  • 複数の遷移先がある場合や、特殊なアーク（EOS/EOL等）に遭遇した場合は走査を中断します。
  • firstchr が現在のカラーと一致しない場合、接頭辞の追加を中断します。

保守担当者向けメモ

  • 本関数はバックリファレンスや複雑な先読みを考慮しません。
  • firstchr を使用したヒューリスティックな文字特定を行っているため、極めて特殊な文字セット構成では接頭辞の抽出が中断される可能性があります。
