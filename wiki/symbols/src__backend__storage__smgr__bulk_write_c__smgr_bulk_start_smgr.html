smgr_bulk_start_smgr

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/bulk_write.c:99

目的

リレーションのバルク書き込み操作を開始し、書き込み状態を管理するための BulkWriteState 構造体を初期化・構築します。

シグネチャまたは形状

BulkWriteState *smgr_bulk_start_smgr(SMgrRelation smgr, ForkNumber forknum, bool use_wal)

入力と出力

  • **入力**:
  • smgr: 対象リレーションのストレージマネージャハンドル。
  • forknum: 書き込み対象のフォーク番号。
  • use_wal: WAL（Write-Ahead Logging）を記録するかどうかを示すフラグ。
  • **出力**:
  • 初期化済みの BulkWriteState 構造体へのポインタ。

前提条件と不変条件

  • smgr は有効な SMgrRelation である必要があります。
  • 呼び出し時の CurrentMemoryContext が、後続のバッファ割り当てに使用されるメモリコンテキストとして BulkWriteState に記録されます。

アルゴリズムまたは振る舞い

1. BulkWriteState 構造体用にメモリを確保します。 2. 引数として受け取った smgr、forknum、use_wal を構造体に格納します。 3. 保留中の書き込み数 (npending) を 0 に初期化します。 4. smgrnblocks を呼び出し、現在のリレーションのブロックサイズを relsize として記録します。 5. GetRedoRecPtr() を呼び出し、現在の Redo ログポインタを start_RedoRecPtr に保存します。 6. 現在のメモリコンテキストを memcxt に保存し、呼び出し元へ構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/storage.c:504
  • src/backend/storage/smgr/bulk_write.c:89 (smgr_bulk_start_rel 経由)
  • **呼び出し先**:
  • palloc (メモリ確保)
  • smgrnblocks (リレーションのブロック数取得)
  • GetRedoRecPtr (WAL ポインタ取得)

関連データ構造

  • BulkWriteState: 本関数が初期化する、バルク書き込み操作の状態を保持する主要構造体。
  • SMgrRelation: ストレージマネージャのハンドル。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させます。
  • smgrnblocks が失敗した場合、そのエラーが伝播します。

保守担当者向けメモ

  • 本関数で初期化された BulkWriteState は、最終的に smgr_bulk_finish で解放される必要があります。
  • memcxt に現在のメモリコンテキストを保持しているため、このコンテキストが生存している間に書き込み操作を完了させる必要があります。
