ReorderBufferTXNByIdEnt

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/logical/reorderbuffer.c:129

目的

ReorderBuffer 内でトランザクションID (TransactionId) をキーとして、対応するトランザクション状態 (ReorderBufferTXN) を高速に検索するためのハッシュテーブル用エントリ構造体です。

シグネチャまたは形状

typedef struct ReorderBufferTXNByIdEnt
{
    TransactionId xid;
    ReorderBufferTXN *txn;
} ReorderBufferTXNByIdEnt;

入力と出力

  • 入力: ハッシュテーブルのキーとして TransactionId を保持。
  • 出力: 関連付けられた ReorderBufferTXN 構造体へのポインタを保持。

前提条件と不変条件

  • ReorderBuffer が管理するハッシュテーブルの entrysize として本構造体のサイズが指定されている必要があります。
  • xid は一意のトランザクション識別子であり、ハッシュテーブル内での検索キーとして機能します。

アルゴリズムまたは振る舞い

  • ReorderBuffer 内のハッシュテーブルにおいて、特定のトランザクションIDに関連する状態を格納・取得するために使用されます。
  • ハッシュテーブルの構築時に sizeof(ReorderBufferTXNByIdEnt) が使用され、メモリレイアウトが決定されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/reorderbuffer.c 内のハッシュテーブル操作関数（hash_create 等の初期化時や、検索・挿入処理時）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • ReorderBufferTXN: 本構造体が参照するトランザクションのメイン状態構造体。
  • ReorderBuffer: 本構造体を利用するハッシュテーブルを保持する親構造体。

失敗モードと境界条件

  • ハッシュテーブルの初期化やメモリ確保に失敗した場合、本構造体を用いた検索や挿入は実行できません。
  • 存在しない xid で検索を行った場合、ハッシュテーブルの仕様に従い NULL が返されます。

保守担当者向けメモ

  • 本構造体は reorderbuffer.c 内のハッシュテーブル専用です。構造を変更する場合は、関連する hash_create の設定箇所（entrysize 指定）と整合性を取る必要があります。
