plan_elem_desc

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/heapdesc.c:75

目的

ヒープのFREEZE操作計画（xlhp_freeze_plan）を人間が読める形式に変換し、StringInfo バッファへ出力する。

シグネチャまたは形状

static void plan_elem_desc(StringInfo buf, void *plan, void *data)

入力と出力

  • buf: 出力先の StringInfo。
  • plan: xlhp_freeze_plan 構造体へのポインタ。
  • data: OffsetNumber 配列へのポインタ（OffsetNumber **）。処理後に配列ポインタが ntuples 分だけ進められる。

前提条件と不変条件

  • plan は xlhp_freeze_plan 型としてキャスト可能であること。
  • data は OffsetNumber 配列を指す有効なポインタであること。

アルゴリズムまたは振る舞い

1. plan を xlhp_freeze_plan にキャストする。 2. xmax、t_infomask、t_infomask2、ntuples を buf に書き込む。 3. array_desc を呼び出し、offsets 配列の内容を整形して buf に追加する。 4. data が指すポインタを、処理したタプル数（ntuples）分だけ進める。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/rmgrdesc/heapdesc.c:317（heap2_desc 内の処理から参照されている）。
  • 呼び出し先: appendStringInfo、appendStringInfoString、array_desc。

関連データ構造

  • xlhp_freeze_plan: FREEZE操作の計画情報を保持する構造体。
  • OffsetNumber: ヒープページ内のオフセットを指す型。

失敗モードと境界条件

  • plan または data が NULL の場合、デリファレンスによりクラッシュする可能性がある。
  • ntuples が不適切な値である場合、array_desc の処理においてメモリ境界外アクセスが発生する可能性がある。

保守担当者向けメモ

  • 本関数は array_desc を利用して配列を走査するため、data ポインタの更新が呼び出し側の状態と同期している必要がある。
  • 新しいフィールドを xlhp_freeze_plan に追加した場合は、本関数の出力フォーマットも更新すること。
