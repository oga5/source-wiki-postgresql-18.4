inet_hist_inclusion_join_sel

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_selfuncs.c:756

目的

2つのヒストグラム（hist1_values と hist2_values）を用いて、ネットワークアドレス型（inet/cidr）の結合における包含関係の選択率を推定します。

シグネチャまたは形状

static Selectivity inet_hist_inclusion_join_sel(
    Datum *hist1_values, 
    int hist1_nvalues,
    Datum *hist2_values, 
    int hist2_nvalues,
    int opr_codenum
)

入力と出力

  • **入力**:
  • hist1_values, hist1_nvalues: 第1引数のヒストグラムデータと要素数。
  • hist2_values, hist2_nvalues: 第2引数のヒストグラムデータと要素数。
  • opr_codenum: 演算子の種類を示す数値コード。
  • **出力**:
  • Selectivity（double）: 推定された選択率。

前提条件と不変条件

  • hist2_nvalues が 2 以下の場合は、内部ヒストグラム要素が存在しないとみなされ、常に 0.0 を返します。
  • MAX_CONSIDERED_ELEMS を用いて、計算コストを抑制するための間引き処理が行われます。

アルゴリズムまたは振る舞い

1. hist2_nvalues が 2 を超える場合、MAX_CONSIDERED_ELEMS に基づいてヒストグラム要素を間引くステップ数 k を算出します。 2. hist2_values の両端を除いた要素に対してループ処理を行います。 3. 各要素に対し inet_hist_value_sel を呼び出し、hist1_values との包含関係に基づく選択率を累積します。 4. 累積した選択率を試行回数 n で除算し、平均値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/network_selfuncs.c 内の結合選択率推定ロジック（例: networkjoinsel 経由のフロー）から参照されています。
  • **呼び出し先**: inet_hist_value_sel を呼び出します。

関連データ構造

  • Datum: PostgreSQL の汎用データ表現。
  • Selectivity: 選択率を表す double 型のエイリアス。
  • MAX_CONSIDERED_ELEMS: 統計計算時に考慮する要素数の上限（1024）。

失敗モードと境界条件

  • hist2_nvalues <= 2: 境界条件として 0.0 を即座に返します。
  • ヒストグラムが空または極端に小さい場合、推定精度が低下する可能性があります。

保守担当者向けメモ

  • 本関数は結合推定における $O(N^2)$ の計算量を抑制するために間引きを行っています。MAX_CONSIDERED_ELEMS の変更はプランナの計算コストと推定精度のトレードオフに直結するため注意してください。
  • 包含関係の判定ロジックは inet_hist_value_sel に依存しているため、ロジックの修正時はそちらも併せて確認してください。
