HeapTupleHeaderData

定義

  • 種別: struct
  • 定義箇所: src/include/access/htup_details.h:153

目的

PostgreSQLのヒープタプル（テーブルの行データ）の先頭に配置される固定長ヘッダー構造体です。ディスク上の物理レイアウトとメモリ上のタプル操作の共通インターフェースとして機能します。

シグネチャまたは形状

struct HeapTupleHeaderData
{
    union { HeapTupleFields t_heap; DatumTupleFields t_datum; } t_choice;
    ItemPointerData t_ctid;
    uint16 t_infomask2;
    uint16 t_infomask;
    uint8 t_hoff;
    bits8 t_bits[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: タプル作成時や更新時に、トランザクション情報（Xmin/Xmax等）や属性情報が本構造体に書き込まれます。
  • **出力**: GETSTRUCT() マクロ等を通じて、ヘッダー直後のユーザーデータ領域へのポインタが提供されます。

前提条件と不変条件

  • t_infomask および t_infomask2 は、タプルの状態（NULLの有無、ロック状態、HOT更新の有無等）を保持します。
  • t_hoff は、ヘッダー（ビットマップおよびパディングを含む）のサイズを示します。
  • MinimalTupleData とメモリレイアウトを意図的に一致させており、ポインタのキャストによる相互運用が可能です。
  • t_bits 以降の領域は可変長であり、NULLビットマップとして使用されます。

アルゴリズムまたは振る舞い

  • heap_getattr 等のアクセサ関数は、t_infomask を確認してNULLビットマップを走査し、属性のオフセットを計算してデータを取り出します。
  • t_ctid は、現在のタプルまたは更新後のタプルを指す行ポインタとして機能し、更新連鎖やスペック的挿入のトークンとしても利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/common/heaptuple.c (タプル長計算や初期化)
  • src/backend/access/heap/heapam.c (更新XID取得)
  • src/backend/access/heap/heapam_xlog.c (WALログ処理)
  • contrib/pageinspect/heapfuncs.c (OID取得)
  • src/backend/jit/llvm/llvmjit_types.c (JITコンパイル用定義)
  • **呼び出し先**:
  • 本構造体自体はデータ構造であり、直接的な関数呼び出しは行いません。

関連データ構造

  • HeapTupleFields: トランザクションIDやコマンドIDを保持。
  • DatumTupleFields: Datum形式のタプル用フィールド。
  • ItemPointerData: t_ctid の型。
  • MinimalTupleData: メモリ節約版のタプル構造。

失敗モードと境界条件

  • t_bits は可変長配列であるため、構造体サイズを計算する際は offsetof(HeapTupleHeaderData, t_bits) を使用する必要があります。
  • t_ctid の解釈は、更新連鎖の状況やスペック的挿入の有無によって異なるため、注意が必要です。

保守担当者向けメモ

  • 本構造体は頻繁にインクルードされるため、パディングやアライメント（MAXALIGN）に細心の注意を払ってください。
  • MinimalTupleData との互換性を維持するため、t_infomask2 以降のフィールド順序を変更する際は注意が必要です。
