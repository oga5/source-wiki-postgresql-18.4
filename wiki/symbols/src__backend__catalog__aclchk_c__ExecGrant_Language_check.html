ExecGrant_Language_check

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/aclchk.c:2251

目的

プログラミング言語オブジェクトに対する GRANT または REVOKE 操作の妥当性を検証し、信頼されていない言語（untrusted language）に対する権限変更を禁止する。

シグネチャまたは形状

static void ExecGrant_Language_check(InternalGrant *istmt, HeapTuple tuple)

入力と出力

  • **入力**:
  • istmt: 実行される GRANT/REVOKE 文の内部表現。
  • tuple: 対象となる pg_language システムカタログのタプル。
  • **出力**: なし（正常時は終了）。エラー時は ereport により例外を発生させる。

前提条件と不変条件

  • tuple は pg_language カタログの有効なタプルであること。
  • 信頼されていない言語（lanpltrusted が偽）に対しては、スーパーユーザーのみが操作可能であるという前提に基づき、通常の権限変更を拒否する。

アルゴリズムまたは振る舞い

1. tuple から Form_pg_language 構造体を取得する。 2. lanpltrusted フィールドを確認する。 3. lanpltrusted が偽（信頼されていない言語）の場合、ERRCODE_WRONG_OBJECT_TYPE エラーを発生させ、処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c 内の ExecGrant_common から呼び出される。
  • **呼び出し先**: ereport（PostgreSQL のエラー報告メカニズム）。

関連データ構造

  • InternalGrant: 権限操作の情報を保持する構造体。
  • Form_pg_language: pg_language カタログの行構造。

失敗モードと境界条件

  • **失敗モード**: 対象言語が信頼されていない言語である場合、常にエラーとなる。
  • **境界条件**: 信頼されている言語（lanpltrusted が真）の場合は、エラーを発生させず正常に終了する。

保守担当者向けメモ

  • 本関数は ExecGrant_common を介して呼び出されるコールバック関数である。
  • 信頼されていない言語に対する権限変更の禁止は、セキュリティ上の制約によるものである。
