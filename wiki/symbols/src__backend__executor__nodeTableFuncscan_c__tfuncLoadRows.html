tfuncLoadRows

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTableFuncscan.c:434

目的

テーブル関数（XMLTABLE や JSON_TABLE 等）の実行において、入力ドキュメントから行を抽出し、列値の生成、デフォルト値の適用、NOT NULL 制約の検証を行った上で、結果を tuplestore に格納する。

シグネチャまたは形状

static void tfuncLoadRows(TableFuncScanState *tstate, ExprContext *econtext)

入力と出力

  • **入力**:
  • tstate: テーブル関数スキャンノードの実行状態を保持する TableFuncScanState 構造体。
  • econtext: 式評価に使用するコンテキスト。
  • **出力**: なし（副作用として tstate->tupstore にタプルが格納される）。

前提条件と不変条件

  • tstate->routine が適切に初期化されていること。
  • econtext->ecxt_per_tuple_memory がタプル処理ごとのメモリ管理に使用可能であること。
  • tstate->tupstore がタプル格納先として準備されていること。

アルゴリズムまたは振る舞い

1. routine->FetchRow を呼び出し、入力ドキュメントから行がなくなるまでループする。 2. 各行に対して ExecClearTuple でスロットを初期化する。 3. 列ごとに以下の処理を行う:

  • ordinalitycol の場合は、現在の行番号（tstate->ordinal）を格納する。
  • それ以外の場合は routine->GetValue で値を取得する。
  • 値が NULL であり、かつ列定義にデフォルト式が存在する場合は ExecEvalExpr で評価・適用する。
  • NOT NULL 制約が定義されている列で値が NULL の場合はエラーを報告する。

    • 4. 処理した行を tuplestore_putvalues で tuplestore に書き込む。 5. 各行の処理終了ごとに econtext->ecxt_per_tuple_memory をリセットし、メモリ消費を抑制する。

呼び出し元と呼び出し先

  • **呼び出し元**: tfuncFetchRows（src/backend/executor/nodeTableFuncscan.c 内）。
  • **呼び出し先**:
  • routine->FetchRow
  • routine->GetValue
  • ExecEvalExpr
  • tuplestore_putvalues
  • MemoryContextReset

関連データ構造

  • TableFuncScanState: スキャンノードの実行状態。
  • TableFuncRoutine: XML/JSON 解析のためのルーチン定義。
  • TupleTableSlot: タプルデータを保持するスロット。
  • ExprContext: 式評価コンテキスト。

失敗モードと境界条件

  • **NOT NULL 制約違反**: 列に NOT NULL 制約があり、かつ値が NULL の場合に ERRCODE_NULL_VALUE_NOT_ALLOWED でエラー終了する。
  • **メモリ制約**: 全行を tuplestore にロードするマテリアライズ型であるため、入力データが巨大な場合はメモリ消費が大きくなる。

保守担当者向けメモ

  • 本関数は tfuncFetchRows から呼び出される。
  • opaque ポインタや perTableCxt の管理は、呼び出し元のライフサイクルに依存している。
  • CHECK_FOR_INTERRUPTS() がループ内に含まれており、クエリキャンセルに応答する設計となっている。
