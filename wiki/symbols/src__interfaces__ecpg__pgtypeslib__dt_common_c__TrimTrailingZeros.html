TrimTrailingZeros

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/dt_common.c:721

目的

文字列形式の日付・時刻データにおいて、末尾の不要なゼロを除去し、フォーマットを整理する。

シグネチャまたは形状

void TrimTrailingZeros(char *str)

入力と出力

  • **入力**: str (変更対象のヌル終端文字列へのポインタ)
  • **出力**: なし（引数 str を直接書き換える）

前提条件と不変条件

  • str は有効なヌル終端文字列であること。
  • 小数点以下の桁数が少なくとも2桁残るように調整される。

アルゴリズムまたは振る舞い

1. 文字列の長さを取得する。 2. 文字列の末尾が '0' であり、かつ小数点から数えて3文字目以降（*(str + len - 3) != '.'）である限り、末尾の '0' を削除し、ヌル終端文字を前方に移動させる。 3. この処理により、小数点以下が少なくとも2桁（例: .00）確保されるまで末尾のゼロが切り詰められる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/pgtypeslib/dt_common.c 内の複数箇所
  • src/interfaces/ecpg/pgtypeslib/interval.c:748
  • **呼び出し先**:
  • 標準ライブラリ関数: strlen

関連データ構造

  • 特になし。

失敗モードと境界条件

  • 文字列内に小数点が存在しない場合、あるいは小数点以下が2桁以下の場合は、末尾のゼロは除去されない。
  • str が極端に短い場合や、不正なフォーマットの文字列が渡された場合の動作は定義されていない。

保守担当者向けメモ

  • 本関数は dt_common.c および interval.c で利用されており、日付・時刻の文字列出力の正規化において重要な役割を果たす。
  • 小数点以下の精度を維持するロジック（len - 3）は、固定的なフォーマットを前提としているため、出力形式の変更時には注意が必要である。
