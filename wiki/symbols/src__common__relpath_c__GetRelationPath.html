GetRelationPath

定義

  • 種別: function
  • 定義箇所: src/common/relpath.c:142

目的

PostgreSQLのデータディレクトリ内における、リレーション（テーブルやインデックス等）の物理ファイルパス文字列を生成します。

シグネチャまたは形状

RelPathStr GetRelationPath(Oid dbOid, Oid spcOid, RelFileNumber relNumber, int procNumber, ForkNumber forkNumber);

入力と出力

  • **入力**:
  • dbOid: データベースOID。
  • spcOid: テーブルスペースOID。
  • relNumber: リレーション番号。
  • procNumber: プロセス番号（一時テーブルの場合に使用、通常は INVALID_PROC_NUMBER）。
  • forkNumber: フォーク番号（MAIN_FORKNUM 等）。
  • **出力**:
  • RelPathStr: 構築されたファイルパスを保持する構造体。

前提条件と不変条件

  • spcOid が GLOBALTABLESPACE_OID の場合、dbOid は 0、procNumber は INVALID_PROC_NUMBER である必要があります（Assertによる検証）。
  • 構築されたパス文字列は REL_PATH_STR_MAXLEN を超えてはなりません（Assertによる検証）。

アルゴリズムまたは振る舞い

1. テーブルスペースの種類（Global, Default, Custom）に応じてパスの基底ディレクトリを決定します。 2. 一時テーブルか否か（procNumber の有無）に応じて、ファイル名のプレフィックス（t<procNumber>_）を制御します。 3. forkNumber が MAIN_FORKNUM 以外の場合、forkNames 配列から対応するサフィックス（_fsm, _vm 等）を取得し、パスに付与します。 4. sprintf を使用して RelPathStr.str バッファにパスを書き込みます。 5. 最後にパスの長さを検証し、構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c
  • src/bin/pg_rewind/filemap.c
  • src/test/regress/regress.c
  • **呼び出し先**:
  • sprintf（標準ライブラリ）
  • Assert（マクロ）

関連データ構造

  • RelPathStr: パス文字列を保持する構造体。
  • ForkNumber: フォークの種類を定義する列挙型。
  • forkNames: フォーク番号と文字列の対応を保持する配列。

失敗モードと境界条件

  • パス長が REL_PATH_STR_MAXLEN を超える場合、Assert により異常終了します。
  • 不正な forkNumber が渡された場合、forkNames 配列の範囲外アクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数はスタック上の構造体を返すため、呼び出し側でのメモリ解放は不要です。
  • 新しいフォークを追加する場合は、ForkNumber 列挙型および forkNames 配列の更新が必要です。
  • パスフォーマットは外部ツールとの互換性に直結するため、変更には細心の注意を払ってください。
