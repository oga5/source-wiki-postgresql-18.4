astreamer_extractor

定義

  • 種別: struct
  • 定義箇所: src/fe_utils/astreamer_file.c:32

目的

アーカイブストリーミングフレームワークにおいて、アーカイブ内の各メンバーをファイルシステム上の個別のファイルとして展開・保存するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct astreamer_extractor
{
    astreamer    base;
    char        *basepath;
    const char *(*link_map) (const char *);
    void        (*report_output_file) (const char *);
    char        filename[MAXPGPATH];
    FILE        *file;
} astreamer_extractor;

入力と出力

  • **入力**: astreamer 基底構造体を通じて受け取るアーカイブメンバーのヘッダー情報およびデータストリーム。
  • **出力**: 指定された basepath 配下へのファイル作成、ディレクトリ作成、シンボリックリンク作成、およびファイル内容の書き込み。

前提条件と不変条件

  • base フィールドとして astreamer 構造体を保持し、ストリーマーのインターフェースに適合している必要があります。
  • filename は MAXPGPATH の長さを超えないパスである必要があります。

アルゴリズムまたは振る舞い

  • astreamer_extractor_content 関数により、アーカイブのメンバー種別（ヘッダー、コンテンツ、トレーラー）に応じて状態遷移します。
  • ASTREAMER_MEMBER_HEADER 受信時にファイルやディレクトリの作成を行い、ASTREAMER_MEMBER_CONTENTS 受信時に file ハンドルへデータを書き込みます。
  • ASTREAMER_MEMBER_TRAILER 受信時に開いている file をクローズします。

呼び出し元と呼び出し先

  • **呼び出し元**: astreamer_extractor_new（インスタンス生成）、astreamer_extractor_content（処理実行）、およびファイル内の各操作関数（extract_directory 等）から参照されます。
  • **呼び出し先**: palloc0（メモリ確保）、および fwrite 等の標準ファイル操作関数（直接の呼び出し先詳細はソースコードのロジックに依存）。

関連データ構造

  • astreamer: 基底となるストリーマー構造体。
  • astreamer_archive_context: 処理の分岐に使用されるアーカイブメンバー情報。

失敗モードと境界条件

  • パスが MAXPGPATH を超える場合、またはファイルシステム権限により作成が拒否された場合にエラーが発生します。
  • 既存ディレクトリの取り扱いについては should_allow_existing_directory の判定ロジックに従い、特定のディレクトリ（pg_wal 等）以外での重複はエラーとなります。

保守担当者向けメモ

  • 本構造体はアーカイブのフォーマット（tar等）に依存せず、抽象的なメンバー情報のみを扱います。
  • 展開時のパス安全性検証は extract_directory や create_file_for_extract 内で行われるため、変更時はこれらの関数との整合性に注意してください。
