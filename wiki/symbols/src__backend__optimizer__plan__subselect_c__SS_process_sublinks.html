SS_process_sublinks

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/subselect.c:2025

目的

式ツリー内の SubLink（サブクエリ）を再帰的に探索し、実行可能な SubPlan または InitPlan に変換して置換する。

シグネチャまたは形状

Node *SS_process_sublinks(PlannerInfo *root, Node *expr, bool isQual)

入力と出力

  • **入力**:
  • root: 現在のクエリレベルの PlannerInfo。
  • expr: 処理対象の式ツリー。
  • isQual: 式がクエリの修飾子（WHERE句など）として使用されるかを示すフラグ。
  • **出力**:
  • SubLink が置換された後の新しい式ツリー（Node *）。

前提条件と不変条件

  • process_sublinks_mutator を介して式ツリーを走査する。
  • 呼び出し元は PlannerInfo が適切に初期化されていることを保証する必要がある。

アルゴリズムまたは振る舞い

1. process_sublinks_context 構造体を初期化し、root および isQual を設定する。 2. process_sublinks_mutator を呼び出し、式ツリーを再帰的に探索する。 3. 探索中に SubLink ノードを発見した場合、それを SubPlan または InitPlan に変換する処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/plan/subselect.c
  • **呼び出し先**:
  • process_sublinks_mutator（内部的な再帰処理）

関連データ構造

  • PlannerInfo: クエリの計画作成に関する情報を保持。
  • process_sublinks_context: 走査中の状態を保持するコンテキスト構造体。
  • SubLink: 変換対象となるサブクエリノード。
  • SubPlan: 変換後の実行可能ノード。

失敗モードと境界条件
