AtStart_Memory

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xact.c:1175

目的

トランザクション開始時に必要なメモリコンテキストの初期化および切り替えを行う。

シグネチャまたは形状

static void AtStart_Memory(void)

入力と出力

  • 入力: なし（グローバルな CurrentTransactionState を参照）
  • 出力: なし（副作用としてメモリコンテキストの状態を更新）

前提条件と不変条件

  • CurrentTransactionState が有効であること。
  • TopMemoryContext が初期化済みであること。

アルゴリズムまたは振る舞い

1. 現在のメモリコンテキストを s->priorContext に保存する。 2. TransactionAbortContext が未作成の場合、TopMemoryContext 配下に 32KB の固定サイズで作成する（アボート時のメモリ不足回避用）。 3. TopTransactionContext が未作成の場合、TopMemoryContext 配下にデフォルト設定で作成する。 4. CurTransactionContext を TopTransactionContext に設定し、現在のトランザクション状態にも保持する。 5. MemoryContextSwitchTo を呼び出し、CurTransactionContext をアクティブなコンテキストに切り替える。

呼び出し元と呼び出し先

  • 呼び出し元: StartTransaction（src/backend/access/transam/xact.c:2154 等）
  • 呼び出し先: AllocSetContextCreate, MemoryContextSwitchTo

関連データ構造

  • TransactionStateData (CurrentTransactionState)
  • TopMemoryContext
  • TransactionAbortContext
  • TopTransactionContext
  • CurTransactionContext

失敗モードと境界条件

  • AllocSetContextCreate がメモリ不足により失敗した場合、PostgreSQL の標準的なエラー処理（elog）によりトランザクション開始が中断される。

保守担当者向けメモ

  • 本関数はトランザクション開始時のメモリ管理の基盤であり、ここで作成される TransactionAbortContext はエラー発生時の安全なメモリ領域として機能する。
  • TopTransactionContext はトランザクション終了時にリセットされ、後続のトランザクションで再利用される設計となっている。
