ExtractSetVariableArgs

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/guc_funcs.c:166

目的

VariableSetStmt 構文解析ノードから、GUC設定処理に適した文字列形式の値を抽出します。

シグネチャまたは形状

char *ExtractSetVariableArgs(VariableSetStmt *stmt)

入力と出力

  • **入力**: VariableSetStmt *stmt（SQLの SET 文等の解析結果構造体）
  • **出力**: char *（抽出された設定値の文字列、または NULL）

前提条件と不変条件

  • 入力 stmt は有効なポインタである必要があります。
  • stmt->kind に応じて適切な処理が分岐します。

アルゴリズムまたは振る舞い

stmt->kind の値に基づき、以下のいずれかの処理を行います。 1. VAR_SET_VALUE: flatten_set_variable_args を呼び出し、stmt->name と stmt->args から設定値を文字列化して返します。 2. VAR_SET_CURRENT: GetConfigOptionByName を呼び出し、現在の設定値を取得して返します。 3. その他: NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_db_role_setting.c:32
  • src/backend/commands/functioncmds.c:672
  • src/backend/utils/misc/guc.c:4631
  • src/backend/utils/misc/guc_funcs.c:63
  • **呼び出し先**:
  • flatten_set_variable_args
  • GetConfigOptionByName

関連データ構造

  • VariableSetStmt: SQLの SET 文を表現するパーサーノード。

失敗モードと境界条件

  • stmt->kind が VAR_SET_VALUE または VAR_SET_CURRENT 以外の場合、関数は NULL を返します。
  • GetConfigOptionByName が失敗した場合の挙動は、当該関数の仕様に依存します。

保守担当者向けメモ

  • 本関数は guc_funcs.c 内のユーティリティであり、SQLコマンドの実行とGUC内部ロジックの橋渡しを行います。
  • VAR_SET_VALUE 以外のケース（VAR_SET_DEFAULT 等）が将来追加された場合、switch 文の拡張が必要です。
