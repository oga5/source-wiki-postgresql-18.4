pg_popcount_masked_choose

定義

  • 種別: function
  • 定義箇所: src/port/pg_bitutils.c:208

目的

バッファ内の各バイトに対してマスクを適用した上で、ビットが立っている（1である）箇所の総数をカウントする処理において、実行時のCPU機能に応じた最適な実装を選択・実行する。

シグネチャまたは形状

uint64 pg_popcount_masked_choose(const char *buf, int bytes, bits8 mask)

入力と出力

  • buf: カウント対象のデータが格納されたバッファへのポインタ。
  • bytes: バッファのサイズ（バイト単位）。
  • mask: 各バイトに対して適用するビットマスク。
  • 戻り値: マスクされたバッファ内の1の総数（uint64）。

前提条件と不変条件

  • 本関数が最初に呼び出された際、内部で choose_popcount_functions() が実行され、CPU機能判定に基づいた関数ポインタの切り替えが行われる。
  • 以降の呼び出しでは、判定済みの最適化された実装が使用されることを前提とする。

アルゴリズムまたは振る舞い

1. choose_popcount_functions() を呼び出し、ハードウェア命令（POPCNT等）の利用可否を判定し、最適化された実装へのディスパッチ準備を行う。 2. 判定完了後、pg_popcount_masked() を呼び出して実際のカウント処理を実行し、その結果を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/pg_bitutils.c 内で pg_popcount_masked_optimized 関数ポインタの初期値として参照されている。また、src/port/pg_popcount_aarch64.c においても同様の参照および呼び出しが行われている。
  • 呼び出し先: choose_popcount_functions() および pg_popcount_masked()。

関連データ構造

  • pg_popcount_masked_optimized: 本関数を初期値として保持する関数ポインタ。

失敗モードと境界条件

  • バッファサイズ bytes が 0 の場合、カウント結果は 0 となる。
  • CPU機能判定の成否に関わらず、フォールバック実装により常に正しいカウント結果を返すことが保証されている。

保守担当者向けメモ

  • 本関数は動的ディスパッチの初期化用ラッパーである。
  • pg_popcount_masked_optimized が実行時に書き換えられた後は、本関数は直接呼び出されない設計となっている。
