geqo_randint

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_random.c:35

目的

GEQO（遺伝的クエリ最適化）アルゴリズムにおいて、指定された範囲内の整数乱数を生成します。

シグネチャまたは形状

int geqo_randint(PlannerInfo *root, int upper, int lower);

入力と出力

  • **入力**:
  • root: 現在のクエリ計画情報を保持する PlannerInfo 構造体へのポインタ。
  • upper: 生成する乱数の上限値（含む）。
  • lower: 生成する乱数の下限値（含む）。
  • **出力**:
  • lower 以上 upper 以下の範囲の整数値を返します。

前提条件と不変条件

  • root->join_search_private に GeqoPrivateData が格納されている必要があります。
  • 現在の実装では、lower は負の値ではないことが前提となっています。

アルゴリズムまたは振る舞い

1. root->join_search_private から GeqoPrivateData を取得し、その内部にある random_state を参照します。 2. PostgreSQL標準の擬似乱数生成器である pg_prng_uint64_range を呼び出し、指定された範囲の乱数を生成します。 3. 生成された値を int 型にキャストして返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_cx.c
  • src/backend/optimizer/geqo/geqo_erx.c
  • src/backend/optimizer/geqo/geqo_mutation.c
  • src/backend/optimizer/geqo/geqo_ox1.c
  • src/backend/optimizer/geqo/geqo_ox2.c
  • **呼び出し先**:
  • pg_prng_uint64_range（src/backend/utils/misc/pg_prng.c で定義）

関連データ構造

  • PlannerInfo: クエリ最適化のコンテキストを保持。
  • GeqoPrivateData: GEQOセッション固有のデータ（乱数状態 random_state を含む）。

失敗モードと境界条件

  • lower に負の値を指定した場合の動作は保証されていません。
  • upper < lower の場合の pg_prng_uint64_range の挙動に依存します。

保守担当者向けメモ

  • 本関数は pg_prng をラップしたものです。将来的に負の範囲をサポートする必要が生じた場合は、pg_prng_uint64_range の仕様確認とキャストの安全性を再検討してください。
