gbt_bpchareq

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_text.c:115

目的

GiSTインデックスにおいて、bpchar（空白埋め固定長文字）型同士の等価性比較を行うためのサポート関数です。

シグネチャまたは形状

static bool gbt_bpchareq(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のデータへのポインタ。
  • collation: 比較に使用する照合順序（Collation）のOID。
  • flinfo: 関数マネージャ情報（使用されません）。
  • **出力**:
  • bool: a と b が等しい場合に true、それ以外の場合に false を返します。

前提条件と不変条件

  • a および b は、PostgreSQLの bpchar 型として解釈可能な有効なデータポインタである必要があります。
  • 内部で bpchareq 関数を呼び出すため、データベースの標準的な比較ルールに従います。

アルゴリズムまたは振る舞い

1. DirectFunctionCall2Coll を使用して、PostgreSQLの標準関数 bpchareq を呼び出します。 2. その際、引数 a と b を PointerGetDatum で Datum 型に変換して渡します。 3. bpchareq の戻り値（Datum）を DatumGetBool で bool 型に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_text.c 内の bptinfo 構造体（gbtree_vinfo 型）の初期化時に参照されています。
  • **呼び出し先**: bpchareq（PostgreSQL標準の比較関数）。

関連データ構造

  • gbtree_vinfo: bpchar 型のインデックス操作に必要な比較関数を保持する構造体。

失敗モードと境界条件

  • a または b が無効なポインタである場合、セグメンテーション違反が発生する可能性があります。
  • collation が不適切な場合、bpchareq の内部ロジックに従ってエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は bpchar 型専用のラッパーです。text 型の比較には gbt_texteq が使用されます。
  • DirectFunctionCall2Coll を介した呼び出しであるため、標準の比較関数に変更があった場合、自動的にその挙動が反映されます。
