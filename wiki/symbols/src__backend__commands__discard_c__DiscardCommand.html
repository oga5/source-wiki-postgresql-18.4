DiscardCommand

定義

  • 種別: function
  • 定義箇所: src/backend/commands/discard.c:30

目的

SQL の DISCARD コマンドのメインエントリーポイントとして、セッション状態のリセット処理をターゲットに応じて適切な内部関数へディスパッチする。

シグネチャまたは形状

void DiscardCommand(DiscardStmt *stmt, bool isTopLevel)

入力と出力

  • stmt: 実行対象の DISCARD 文を表す DiscardStmt 構造体へのポインタ。
  • isTopLevel: コマンドがトップレベルで実行されているかを示すフラグ。
  • 出力: なし（戻り値は void）。

前提条件と不変条件

  • stmt->target に基づいて処理が分岐される。
  • 未定義のターゲットが指定された場合、エラーが発生する。

アルゴリズムまたは振る舞い

stmt->target の値に応じて以下の処理を実行する： 1. DISCARD_ALL: DiscardAll(isTopLevel) を呼び出し、セッション全体を初期化する。 2. DISCARD_PLANS: ResetPlanCache() を呼び出し、プランキャッシュをリセットする。 3. DISCARD_SEQUENCES: ResetSequenceCaches() を呼び出し、シーケンスキャッシュをリセットする。 4. DISCARD_TEMP: ResetTempTableNamespace() を呼び出し、一時テーブルの名前空間をリセットする。 5. 上記以外: elog(ERROR) を発行し、処理を中断する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/tcop/utility.c（DiscardStmt を解析した後に呼び出される）。
  • 呼び出し先: DiscardAll, ResetPlanCache, ResetSequenceCaches, ResetTempTableNamespace。

関連データ構造

  • DiscardStmt: DISCARD コマンドの構文解析結果を保持する構造体。

失敗モードと境界条件

  • 未知の target 値が渡された場合、elog(ERROR) によりトランザクションがアボートされる。
  • DISCARD ALL の内部処理（DiscardAll）において、トランザクションブロック内での実行は禁止されている。

保守担当者向けメモ

  • 本関数はディスパッチャとして機能するため、新しい DISCARD ターゲットを追加する場合は switch 文にケースを追加し、対応するリセット関数を呼び出す必要がある。
