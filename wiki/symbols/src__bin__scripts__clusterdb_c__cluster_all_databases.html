cluster_all_databases

定義

  • 種別: function
  • 定義箇所: src/bin/scripts/clusterdb.c:226

目的

PostgreSQLクラスタ内の接続可能な全データベースに対して、順次 CLUSTER コマンドを実行する。

シグネチャまたは形状

static void cluster_all_databases(ConnParams *cparams, SimpleStringList *tables, const char *progname, bool verbose, bool echo, bool quiet);

入力と出力

  • **入力**:
  • cparams: 接続パラメータを保持する構造体。
  • tables: クラスタリング対象のテーブルリスト（空の場合はDB全体）。
  • progname: 実行中のプログラム名。
  • verbose, echo, quiet: 実行時の出力制御フラグ。
  • **出力**: なし（副作用としてデータベースへのコマンド発行を行う）。

前提条件と不変条件

  • cparams を用いてメンテナンス用データベースへの接続が確立可能であること。
  • pg_database システムカタログからデータベース名を取得するクエリが成功すること。

アルゴリズムまたは振る舞い

1. connectMaintenanceDatabase を呼び出し、メンテナンスDBへの接続を確立する。 2. pg_database に対して datallowconn が真かつ datconnlimit が -2 でないデータベース一覧を取得するクエリを実行し、接続を閉じる。 3. 取得した各データベース名に対し、以下の処理をループで繰り返す。

  • cparams->override_dbname を対象DB名に更新する。
  • tables リストが空でない場合は、リスト内の各テーブルに対して cluster_one_database を呼び出す。
  • tables リストが空の場合は、データベース全体に対して cluster_one_database を呼び出す。

    • 4. 最後に PQclear でクエリ結果を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/scripts/clusterdb.c:152）。
  • **呼び出し先**:
  • connectMaintenanceDatabase
  • executeQuery
  • PQfinish, PQntuples, PQgetvalue, PQclear
  • cluster_one_database

関連データ構造

  • ConnParams: 接続情報を管理する構造体。
  • SimpleStringList: 対象テーブル名を保持するリスト構造。
  • PGconn, PGresult: libpq の接続およびクエリ結果オブジェクト。

失敗モードと境界条件

  • メンテナンスDBへの接続失敗やクエリ実行失敗時は、executeQuery 等の内部でエラーハンドリングが行われる。
  • データベース一覧が空の場合、ループは実行されない。
  • tables リストの有無により、対象がテーブル単位かデータベース全体かに分岐する。

保守担当者向けメモ

  • 本関数は clusterdb --all オプション指定時に実行される。
  • cluster_one_database を再利用して各DBへの処理を委譲しているため、個別のDBに対する処理ロジックを変更する場合は cluster_one_database を参照すること。
