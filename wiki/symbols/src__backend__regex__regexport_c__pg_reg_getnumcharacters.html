pg_reg_getnumcharacters

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:229

目的

指定されたカラー ID に対応する文字数を取得します。正規表現の最適化やインデックススキャンにおいて、特定のカラーが保持する文字の集合サイズを判定するために使用されます。

シグネチャまたは形状

int pg_reg_getnumcharacters(const regex_t *regex, int co);

入力と出力

  • **入力**:
  • regex: コンパイル済みの正規表現オブジェクトへのポインタ。
  • co: 問い合わせ対象のカラー ID。
  • **出力**:
  • 指定されたカラーに含まれる文字数（整数）。
  • 失敗または不確定な場合は -1 を返します。

前提条件と不変条件

  • regex は NULL であってはならず、REMAGIC マジックナンバーで検証済みである必要があります。
  • 内部の colormap 構造体を通じてカラー情報にアクセスします。

アルゴリズムまたは振る舞い

1. regex から colormap を取得します。 2. co が有効範囲外（<= 0）であるか、PSEUDO フラグが立っている場合は -1 を返します。 3. cm->cd[co].nuchrs が 0 でない場合（高位カラーマップに該当する場合）、文字数は不確定とみなし -1 を返します。 4. 上記の条件に該当しない場合、cm->cd[co].nschrs に格納されている既知の文字数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_regexp.c:781 での呼び出しが確認されています。
  • **呼び出し先**: 直接的な関数呼び出しはありませんが、regex_t 構造体内の guts メンバおよび colormap 構造体を参照します。

関連データ構造

  • regex_t: 正規表現のコンパイル済み状態を保持する構造体。
  • guts: regex_t の内部実装詳細を保持する構造体。
  • colormap: 文字クラス（カラー）の定義と属性を管理する構造体。

失敗モードと境界条件

  • co <= 0: WHITE や RAINBOW などの特殊カラーは対象外として -1 を返します。
  • PSEUDO フラグ: BOS などの擬似カラーは文字数を持たないため -1 を返します。
  • 高位カラーマップ: nuchrs != 0 の場合、文字数が不確定であるため -1 を返します。

保守担当者向けメモ

  • 本関数は、文字数が確定している場合にのみ値を返します。不確定なケース（高位カラーマップ）での挙動は設計上の制約であり、詳細な文字リストが必要な場合は pg_reg_getcharacters の使用を検討してください。
