cb_options

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_combinebackup/pg_combinebackup.c:71

目的

pg_combinebackup コマンドの実行設定およびコマンドライン引数から得られたパラメータを保持する構造体です。

シグネチャまたは形状

typedef struct cb_options
{
	bool		debug;
	char	   *output;
	bool		dry_run;
	bool		no_sync;
	cb_tablespace_mapping *tsmappings;
	pg_checksum_type manifest_checksums;
	bool		no_manifest;
	DataDirSyncMethod sync_method;
	CopyMethod	copy_method;
} cb_options;

入力と出力

本構造体は、コマンドライン引数の解析結果を格納するコンテナとして機能します。出力先パス、コピー手法、同期方式、テーブルスペースマッピング情報などが含まれます。

前提条件と不変条件

  • tsmappings は cb_tablespace_mapping 型のリストを指す必要があります。
  • copy_method および sync_method は、プログラム実行中に決定される有効な列挙値である必要があります。

アルゴリズムまたは振る舞い

本構造体は、プログラムの初期化フェーズで生成され、以降のバックアップ統合処理（ディレクトリ走査、ファイルコピー、マニフェスト生成）において、各関数の設定パラメータとして参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_combinebackup/pg_combinebackup.c:159）でインスタンス化されます。
  • **呼び出し先**: add_tablespace_mapping、create_output_directory、scan_for_existing_tablespaces 等の関数に引数として渡されます。

関連データ構造

  • cb_tablespace_mapping: テーブルスペースのパス変換ルールを保持。
  • CopyMethod: ファイルコピー手法（copy, link, clone）を定義。
  • DataDirSyncMethod: データディレクトリの同期方式を定義。
  • pg_checksum_type: マニフェストのチェックサムアルゴリズム。

失敗モードと境界条件

  • output パスが指定されていない場合や、無効な copy_method が設定された場合の動作は、本構造体自体ではなく、これを利用する関数側のバリデーションに依存します。

保守担当者向けメモ

  • 本構造体は pg_combinebackup のグローバルな設定状態を保持するため、変更を加える際は、これを利用するすべてのサブ関数（reconstruct.c や copy_file.c 関連の呼び出し）への影響を考慮してください。
