SnapBuildState

定義

  • 種別: enum
  • 定義箇所: src/include/replication/snapbuild.h:22

目的

論理デコーディングにおけるカタログスナップショット構築プロセスの状態を追跡し、WALストリームから整合性のあるデータベース状態を再現するための制御状態を定義する。

シグネチャまたは形状

typedef enum {
    SNAPBUILD_START = -1,
    SNAPBUILD_BUILDING_SNAPSHOT = 0,
    SNAPBUILD_FULL_SNAPSHOT = 1,
    SNAPBUILD_CONSISTENT = 2,
} SnapBuildState;

入力と出力

  • 入力: WALレコードの処理に伴う状態遷移（SnapBuildProcessChange 等による）。
  • 出力: 現在の構築フェーズを示す列挙値。

前提条件と不変条件

  • SNAPBUILD_CONSISTENT に到達するまでは、コミットコールバックの呼び出しは保留される。
  • SNAPBUILD_FULL_SNAPSHOT 到達後も、それ以前に開始されたトランザクションが実行中である可能性があるため、変更の適用は即座に行われない。

アルゴリズムまたは振る舞い

  • SNAPBUILD_START: 初期状態。処理開始直後。
  • SNAPBUILD_BUILDING_SNAPSHOT: 初期カタログスナップショットを構築するため、コミット済みトランザクションを収集するフェーズ。
  • SNAPBUILD_FULL_SNAPSHOT: 後のトランザクションをデコードするために十分な情報が収集された状態。変更の収集を開始する。
  • SNAPBUILD_CONSISTENT: SNAPBUILD_FULL_SNAPSHOT 時点で実行中だった全トランザクションが終了した時点。この状態に達して初めて、デコードされた変更の適用が可能となる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/replication/logical/snapbuild.c 内のロジック、および contrib/pg_logicalinspect/pg_logicalinspect.c（状態説明の取得）。
  • 呼び出し先: 直接的な関数呼び出し関係は本シンボル自体には存在しない。

関連データ構造

  • SnapBuild: 構築器のメイン構造体。内部で SnapBuildState を保持する。
  • src/include/replication/snapbuild_internal.h: SnapBuild 構造体の詳細定義。

失敗モードと境界条件

  • 状態遷移の順序が不正な場合、論理デコーディングの整合性が損なわれる可能性がある。
  • SNAPBUILD_CONSISTENT に到達するまでの間、システムはカタログの可視性を正しく判定できない。

保守担当者向けメモ

  • 状態遷移図が src/backend/replication/logical/snapbuild.c の冒頭に記載されているため、変更を加える際は必ず参照すること。
  • SnapBuildCurrentState() 関数を通じて現在の状態を取得可能。
