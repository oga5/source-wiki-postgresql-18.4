is_code_in_table

定義

  • 種別: function
  • 定義箇所: src/common/saslprep.c:982

目的

指定されたUnicodeコードポイントが、ソート済みの範囲テーブル内に存在するかを効率的に判定します。

シグネチャまたは形状

static bool is_code_in_table(pg_wchar code, const pg_wchar *map, int mapsize)

入力と出力

  • **入力**:
  • code: 検索対象の pg_wchar 型コードポイント。
  • map: 範囲の開始値と終了値が交互に並んだ pg_wchar 型の配列。
  • mapsize: 配列 map の要素数。
  • **出力**:
  • bool: コードポイントがテーブル内のいずれかの範囲に含まれる場合は true、それ以外は false を返します。

前提条件と不変条件

  • mapsize は偶数である必要があります（Assert(mapsize % 2 == 0) により検証）。
  • map 配列は、codepoint_range_cmp 関数が期待する形式でソートされている必要があります。

アルゴリズムまたは振る舞い

1. コードポイントが map の最小値未満、または最大値を超えている場合は即座に false を返します。 2. bsearch を使用し、map 配列に対してバイナリサーチを実行します。 3. 比較には codepoint_range_cmp 関数を使用し、コードポイントが定義された範囲内に存在するかを確認します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/saslprep.c 内の IS_CODE_IN_TABLE マクロ経由で呼び出されます。
  • **呼び出し先**:
  • bsearch (標準ライブラリ)
  • codepoint_range_cmp (比較関数)

関連データ構造

  • pg_wchar: 文字コードを保持する型。
  • prohibited_output_ranges 等の範囲テーブル（map 引数として渡される）。

失敗モードと境界条件

  • mapsize が奇数の場合、アサーションエラーが発生します。
  • 範囲外のコードポイントに対しては、バイナリサーチを行う前に早期リターンすることで計算量を削減しています。

保守担当者向けメモ

  • 本関数は static であり、saslprep.c 内でのみ使用されます。
  • パフォーマンス向上のため、バイナリサーチを採用しています。テーブル定義を変更する際は、必ず mapsize が偶数かつソート済みであることを維持してください。
