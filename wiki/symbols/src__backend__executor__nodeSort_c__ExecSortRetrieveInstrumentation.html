ExecSortRetrieveInstrumentation

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSort.c:475

目的

並列クエリ実行時において、各ワーカーから収集されたソート統計情報（TuplesortInstrumentation）を共有メモリからローカルメモリへコピーし、集約可能な状態にする。

シグネチャまたは形状

void ExecSortRetrieveInstrumentation(SortState *node)

入力と出力

  • **入力**: SortState *node - ソートノードの実行状態を保持する構造体。
  • **出力**: なし（副作用として node->shared_info が更新される）。

前提条件と不変条件

  • node->shared_info が NULL の場合、本関数は何も行わず即座に復帰する。
  • node->shared_info には、並列ワーカー数 (num_workers) とそれに対応する統計情報が含まれている必要がある。

アルゴリズムまたは振る舞い

1. node->shared_info が NULL であれば終了する。 2. SharedSortInfo 構造体のサイズを計算する（ヘッダーサイズ + ワーカー数分の TuplesortInstrumentation サイズ）。 3. 計算したサイズ分を palloc でメモリ確保する。 4. memcpy を使用して、共有メモリ上の統計情報を確保したローカルメモリ領域へコピーする。 5. node->shared_info ポインタを、コピー先のローカルメモリ領域へ更新する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c:1087
  • **呼び出し先**:
  • 直接的な関数呼び出しの証拠はソース抜粋に含まれていない。

関連データ構造

  • SortState: ソートノードの実行状態を管理する構造体。
  • SharedSortInfo: 並列ソートの統計情報を共有するための構造体。
  • TuplesortInstrumentation: 個別のソート処理に関する統計データ。

失敗モードと境界条件

  • node->shared_info が NULL の場合、安全に早期リターンする。
  • メモリ割り当て失敗時は palloc の標準的な挙動（エラー発生）に従う。

保守担当者向けメモ

  • 本関数は並列実行時の統計収集に特化している。node->shared_info のメモリ管理を共有メモリからプロセスローカルへ切り替える役割を持つため、メモリリークや二重解放が発生しないよう注意すること。
