checkAllTheSame

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgdoinsert.c:598

目的

SP-GiSTインデックスのページ分割時において、すべてのタプルが同一のノードに分類される（allTheSame）状態を検出し、必要に応じて分割ロジックを強制的に上書きする。

シグネチャまたは形状

static bool checkAllTheSame(spgPickSplitIn *in, spgPickSplitOut *out, bool tooBig, bool *includeNew)

入力と出力

  • **入力**:
  • in: 分割対象のタプル情報を含む入力構造体。
  • out: picksplit 関数による現在の分割決定結果。
  • tooBig: 現在のタプルセットがページ容量を超過しているかを示すフラグ。
  • **出力**:
  • includeNew: 新しいリーフタプルを分割に含めるべきかを示すフラグ。
  • 戻り値: allTheSame モードを適用した場合は true、それ以外は false。

前提条件と不変条件

  • in->nTuples が1以下の場合は、常に false を返す。
  • tooBig が真の場合、新しいタプルはテスト対象から除外される。

アルゴリズムまたは振る舞い

1. out->mapTuplesToNodes を走査し、すべてのタプルが同一のノードに分類されているかを確認する。 2. すべてのタプルが同一ノードであれば、out の設定を上書きする：

  • out->nNodes を8に設定する。
  • タプルを i % 8 の規則でノードに再分配する。
  • out->nodeLabels が存在する場合、元のノードラベルを全ノードに複製する。

    • 3. tooBig かつ新しいタプルが既存のノードと異なる場所に分類される場合、includeNew を false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c 内の doPickSplit 関数から呼び出される。
  • **呼び出し先**: palloc（メモリ割り当て）。

関連データ構造

  • spgPickSplitIn: 分割入力データ。
  • spgPickSplitOut: 分割出力データ（本関数により変更される）。

失敗モードと境界条件

  • タプル数が1以下の場合は allTheSame モードを適用しない。
  • tooBig が真の場合、新しいタプルが既存のノードと一致しないときは分割から除外される。

保守担当者向けメモ

  • 本関数は picksplit プロシージャの決定を強制的に上書きする特殊なロジックである。
  • out->nNodes = 8 は恣意的な値であり、将来的な調整や定数化の検討が必要な場合がある。
  • nodeLabels の複製処理においてメモリ割り当てを行うため、呼び出し側でのメモリ管理に注意すること。
