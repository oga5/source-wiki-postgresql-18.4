gbt_intvkey_cmp

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_interval.c:61

目的

interval 型の範囲を保持する intvKEY 構造体同士を比較し、GiST インデックス構築時のソート順序を決定します。

シグネチャまたは形状

int gbt_intvkey_cmp(const void *a, const void *b, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象の Nsrt 構造体へのポインタ。内部で intvKEY にキャストされます。
  • flinfo: 関数呼び出し情報（interval_cmp の実行に使用）。
  • **出力**:
  • int: a が b より小さい場合は負、等しい場合は 0、大きい場合は正の値を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、Nsrt 構造体を経由して intvKEY を指している必要があります。
  • interval_cmp 関数が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. a と b の intvKEY 構造体から、それぞれの下限値（lower）を取り出します。 2. interval_cmp を呼び出し、下限値を比較します。 3. 下限値が異なる場合、その比較結果を返します。 4. 下限値が等しい場合、上限値（upper）を取り出し、再度 interval_cmp で比較した結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/btree_gist/btree_interval.c 内のソートサポート設定（gbt_intv_sortsupport 等の文脈）から参照されていることが示唆されています。
  • **呼び出し先**: interval_cmp（PostgreSQL の interval 型比較関数）。

関連データ構造

  • intvKEY: interval 型の範囲（下限・上限）を保持する構造体。
  • Nsrt: ソート処理のために intvKEY をラップする構造体。

失敗モードと境界条件

  • interval_cmp がエラーを発生させた場合、本関数もその例外を伝播します。
  • 入力ポインタが NULL の場合の挙動は定義されていません。

保守担当者向けメモ

  • 本関数は interval 型の二段階比較（下限優先、等しければ上限）を実装しています。
  • btree_gist の他の型と同様に、interval_cmp を直接呼び出すことで一貫した順序付けを保証しています。
