RelationPreserveStorage

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/catalog/storage.c:251

目的

トランザクション終了時に物理ファイルの削除が予約されているリレーション（pendingDeletes リスト）から、指定されたリレーションの削除予約を取り消します。これにより、コミットまたはアボート時に物理ファイルが削除されるのを防ぎます。

シグネチャまたは形状

void RelationPreserveStorage(RelFileLocator rlocator, bool atCommit)

入力と出力

  • **入力**:
  • rlocator (RelFileLocator): 削除予約を解除したいリレーションの識別子。
  • atCommit (bool): 削除予約がコミット時（true）かアボート時（false）のいずれであるかを指定するフラグ。
  • **出力**: なし（副作用として pendingDeletes リストを更新）。

前提条件と不変条件

  • pendingDeletes リストは TopMemoryContext に保持されており、トランザクション終了後もアクセス可能です。
  • 指定された rlocator と atCommit の組み合わせに一致するエントリのみがリストから削除されます。

アルゴリズムまたは振る舞い

1. グローバル変数 pendingDeletes を先頭から走査します。 2. 現在の要素が rlocator と atCommit の両方に一致する場合、その要素をリストから切り離し、pfree を用いてメモリを解放します。 3. 一致しない場合は、次の要素へ進みます。 4. リストの走査完了までこれを繰り返し、該当するすべての予約エントリを削除します。

関連データ構造

  • PendingRelDelete: 削除待ちリレーションを保持するリスト構造体。
  • pendingDeletes: 削除待ちリレーションを管理するグローバルリスト。
  • RelFileLocator: リレーションの物理的な場所を特定するための識別子。

失敗モードと境界条件

  • 該当する rlocator と atCommit の組み合わせがリスト内に存在しない場合、何も行わずに正常終了します。
  • リストが空の場合、走査は即座に終了します。

保守担当者向けメモ

  • 本関数は、トランザクションの進行中にリレーションの削除予約を無効化する必要がある場合（例: リレーションの再利用や、削除対象からの除外）に使用されます。
  • atCommit フラグを正確に指定しないと、意図しないタイミングの削除予約が残る可能性があるため注意してください。
