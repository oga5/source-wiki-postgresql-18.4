ScanSourceDatabasePgClassPage

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dbcommands.c:328

目的

データベース複製プロセスにおいて、指定されたページ内のタプルを走査し、コピー対象となるリレーション情報を抽出してリストに蓄積する。

シグネチャまたは形状

List *ScanSourceDatabasePgClassPage(Page page, Buffer buf, Oid tbid, Oid dbid,
                                    char *srcpath, List *rlocatorlist,
                                    Snapshot snapshot)

入力と出力

  • **入力**:
  • page: 走査対象のページデータ。
  • buf: ページに対応するバッファ。
  • tbid, dbid: ターゲットのテーブルスペースOIDおよびデータベースOID。
  • srcpath: ソースデータベースのパス。
  • rlocatorlist: 抽出した CreateDBRelInfo オブジェクトを保持するリスト。
  • snapshot: 可視性判定に使用するスナップショット。
  • **出力**:
  • 更新された rlocatorlist（コピー対象のリレーション情報リスト）。

前提条件と不変条件

  • page は有効なヒープページであること。
  • ItemId が Used かつ Normal であるタプルのみが処理対象となる。
  • HeapTupleSatisfiesVisibility により、指定されたスナップショットから可視なタプルのみが処理される。

アルゴリズムまたは振る舞い

1. ページ内の全オフセットを FirstOffsetNumber から maxoff まで順次走査する。 2. 各オフセットについて ItemId を取得し、使用されていない、または削除済み（Dead/Redirected）の項目をスキップする。 3. 有効なタプルに対して ItemPointerSet で位置を特定し、HeapTupleData を初期化する。 4. HeapTupleSatisfiesVisibility で可視性を確認する。 5. 可視なタプルに対し ScanSourceDatabasePgClassTuple を呼び出し、コピーが必要な場合は CreateDBRelInfo オブジェクトを取得して rlocatorlist に追加する。 6. 最終的に更新されたリストを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dbcommands.c 内のデータベース複製処理フローから呼び出される（詳細はソースコード上の直接的な参照関係に基づく）。
  • **呼び出し先**: ScanSourceDatabasePgClassTuple（タプルごとの詳細処理）、lappend（リストへの追加）。

関連データ構造

  • CreateDBRelInfo: データベース複製時にコピー対象となるリレーションの情報を保持する構造体。
  • HeapTupleData: 抽出されたタプルのヘッダ情報。
  • Page: ページ管理構造。

失敗モードと境界条件

  • ページが空の場合、ループは実行されず空のリストが返される。
  • ItemId が Dead または Redirected の場合はスキップされ、エラーにはならない。
  • ScanSourceDatabasePgClassTuple が NULL を返した場合、そのタプルはコピー対象外として無視される。

保守担当者向けメモ

  • 本関数は pg_class のページを走査する低レベルな処理を担当する。
  • データベース複製戦略（WAL_LOG 等）におけるカタログスキャンの一部として機能する。
  • rlocatorlist のメモリ管理は呼び出し元に依存する。
