RequestXLogStreaming

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiverfuncs.c:245

目的

スタートアッププロセスからWALレシーバーに対し、指定されたタイムラインと位置からのWALストリーミング開始または再開を指示します。

シグネチャまたは形状

void RequestXLogStreaming(TimeLineID tli, XLogRecPtr recptr, const char *conninfo, const char *slotname, bool create_temp_slot)

入力と出力

  • **入力**:
  • tli: 受信を開始するタイムラインID。
  • recptr: 受信を開始するWAL位置（セグメント境界に切り捨てられます）。
  • conninfo: プライマリへの接続文字列。
  • slotname: 使用するレプリケーションスロット名。
  • create_temp_slot: スロット名が指定されない場合に一時スロットを作成するかどうかのフラグ。
  • **出力**: なし（共有メモリの状態更新およびシグナル送信による副作用）。

前提条件と不変条件

  • 共有メモリ上の WalRcv 構造体へのアクセスは mutex で保護されます。
  • 呼び出し時点で walRcvState は WALRCV_STOPPED または WALRCV_WAITING である必要があります。
  • WAL位置 recptr は、常にセグメントの先頭位置に調整されます。

アルゴリズムまたは振る舞い

1. recptr をセグメント境界に切り捨てます。 2. WalRcv->mutex を取得し、接続情報やスロット設定を共有メモリに書き込みます。 3. 現在の状態に応じて walRcvState を WALRCV_STARTING（停止時）または WALRCV_RESTARTING（待機時）に更新します。 4. 初回起動時またはタイムライン変更時は、受信開始位置等のメタデータを初期化します。 5. mutex を解放します。 6. WALRCV_STOPPED からの遷移であれば PMSIGNAL_START_WALRECEIVER をPostmasterへ送信し、それ以外であればWALレシーバープロセスのラッチをセットして処理を促します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlogrecovery.c（スタートアッププロセス）。
  • **呼び出し先**: SendPostmasterSignal（Postmasterへの通知）、SetLatch（WALレシーバーの起動・再開通知）。

関連データ構造

  • WalRcvData: 共有メモリ上のWALレシーバー制御構造体。
  • XLogRecPtr: WAL上の位置を示すデータ型。

失敗モードと境界条件

  • conninfo がNULLの場合、接続情報は空文字列として扱われます。
  • slotname が指定されている場合、create_temp_slot の値は無視され、永続スロットとして扱われます。
  • 状態遷移の不整合が発生した場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数はスタートアッププロセスからのみ呼び出される設計です。
  • 共有メモリの更新とシグナル送信の順序は、競合を防ぐためにスピンロックの解放前後で厳密に制御されています。
