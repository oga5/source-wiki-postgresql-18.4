r_Step_2

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_serbian.c:5608

目的

セルビア語のステミング処理における第2段階として、単語末尾の特定の接尾辞をルールテーブルに基づいて削除または置換する。

シグネチャまたは形状

static int r_Step_2(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境構造体）
  • 出力: 成功時は 1 または 0、失敗（メモリ不足や不正な操作）時は負の整数を返す。

前提条件と不変条件

  • z->c が単語の末尾を指していること。
  • r_R1 関数が呼び出され、対象の接尾辞が領域 R1 内にあることが検証されること。

アルゴリズムまたは振る舞い

1. z->ket を現在のカーソル位置に設定する。 2. find_among_b を使用し、テーブル a_2（2035個の要素）から後方一致する接尾辞を検索する。 3. 一致が見つからない場合は終了する。 4. z->bra を現在のカーソル位置に設定する。 5. r_R1 を呼び出し、一致した接尾辞が領域 R1 内にあるか確認する。 6. 一致した接尾辞のインデックス（among_var）に基づき、slice_from_s を用いて対応する文字列への置換を行う。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_serbian.c:6523（メインのステミング処理フロー内）
  • 呼び出し先: find_among_b, r_R1, slice_from_s

関連データ構造

  • SN_env: ステミング環境を保持する構造体。
  • a_2: ステミングルールを定義した among 構造体の配列。

失敗モードと境界条件

  • find_among_b が一致を返さない場合、処理は何も行わず 0 を返す。
  • r_R1 が領域外と判断した場合、その戻り値をそのまま返す。
  • slice_from_s が内部的なメモリ操作やバッファ操作で失敗した場合、負の戻り値が伝播される。

保守担当者向けメモ

  • 本関数はSnowballコンパイラによる自動生成コードである。ロジックの変更が必要な場合は、直接Cファイルを編集せず、対応する .sbl ソースファイルを修正すること。
  • among_var に基づく switch 文は、ルールテーブルのインデックスと密結合しているため、テーブル定義の変更時には注意が必要である。
