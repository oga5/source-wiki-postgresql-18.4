makeDependencyGraph

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_cte.c:647

目的

WITH 句（CTE）に含まれる各クエリを走査し、CTE間の依存関係を特定して、解析順序を決定するためのグラフを構築します。

シグネチャまたは形状

static void makeDependencyGraph(CteState *cstate)

入力と出力

  • **入力**: CteState *cstate（解析対象のCTEリストおよび状態を保持する構造体へのポインタ）
  • **出力**: なし（副作用として cstate 内の依存関係情報が更新され、ソートが実行されます）

前提条件と不変条件

  • cstate は初期化済みであること。
  • 各ループの終了時に cstate->innerwiths が NIL であること（Assert により検証）。

アルゴリズムまたは振る舞い

1. cstate->items に格納された各CTEを順次処理します。 2. 各CTEのクエリ本体（cte->ctequery）に対して makeDependencyGraphWalker を呼び出し、依存関係を抽出します。 3. 走査中、現在のCTEインデックスを cstate->curitem に設定し、ネストされた WITH 句の状態をリセットします。 4. 全てのCTEの走査完了後、TopologicalSort を呼び出し、依存関係に基づいて解析順序を確定させます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_cte.c 内の transformWithClause（182行目）。
  • **呼び出し先**:
  • makeDependencyGraphWalker（各CTEクエリの走査）
  • TopologicalSort（依存関係に基づくソート）

関連データ構造

  • CteState: CTE解析の状態管理用構造体。
  • CommonTableExpr: CTEの定義ノード。

失敗モードと境界条件

  • cstate->innerwiths が NIL でない場合、Assert によりアボートします。
  • 循環参照が存在する場合、TopologicalSort 内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は WITH 句の解析における依存関係解決の起点です。
  • makeDependencyGraphWalker がネストされた WITH 句を正しく処理しているため、変更を加える際は名前空間の可視性ルールに注意してください。
