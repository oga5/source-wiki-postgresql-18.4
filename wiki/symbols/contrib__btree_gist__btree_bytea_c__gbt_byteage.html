gbt_byteage

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bytea.c:31

目的

bytea型に対する「以上（greater than or equal to）」比較をGiSTインデックスのサポート関数として提供します。

シグネチャまたは形状

bool gbt_byteage(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のbyteaデータへのポインタ。
  • collation: 照合順序（本関数では使用されません）。
  • flinfo: 関数呼び出し情報。
  • **出力**: a >= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は有効なbyteaデータ構造を指している必要があります。
  • PostgreSQLの標準関数 byteage が利用可能である必要があります。

アルゴリズムまたは振る舞い

1. a と b を PointerGetDatum を用いてPostgreSQLのDatum形式に変換します。 2. DirectFunctionCall2 を介して、PostgreSQLの標準比較関数 byteage を呼び出します。 3. byteage の戻り値を DatumGetBool で論理値に変換して返します。

呼び出し元と呼び出し先

  • **呼び出し先**: PostgreSQL標準の byteage 関数。
  • **呼び出し元**: contrib/btree_gist/btree_bytea.c 内の静的配列（GiSTサポート関数テーブル）から参照されています。具体的な実行時の呼び出し元はGiSTインデックスの検索・評価プロセスです。

関連データ構造

  • bytea: PostgreSQLの可変長バイナリデータ型。

失敗モードと境界条件

  • 入力ポインタがNULLの場合、DirectFunctionCall2 の挙動に依存します。
  • byteage 関数がエラーを発生させた場合、その例外が伝播します。

保守担当者向けメモ

  • 本関数は単なるラッパーであり、実質的な比較ロジックはPostgreSQLコアの byteage に依存しています。
  • btree_gist モジュール内での一貫性を保つため、比較ロジックを変更する場合は byteage の仕様を確認してください。
