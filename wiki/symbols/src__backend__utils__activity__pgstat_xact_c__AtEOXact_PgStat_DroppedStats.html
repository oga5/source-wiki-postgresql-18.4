AtEOXact_PgStat_DroppedStats

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_xact.c:66

目的

トランザクション終了時（コミットまたはアボート）に、トランザクション内で保留されていた統計情報の作成・削除処理を確定させ、不要な統計エントリを物理的に削除する。

シグネチャまたは形状

static void AtEOXact_PgStat_DroppedStats(PgStat_SubXactStatus *xact_state, bool isCommit)

入力と出力

  • xact_state: 処理対象のトランザクション状態（PgStat_SubXactStatus）。保留中の統計アイテムリストを保持する。
  • isCommit: トランザクションがコミットされた場合は true、アボートされた場合は false。
  • 出力なし（副作用として統計エントリの削除およびメモリ解放を行う）。

前提条件と不変条件

  • xact_state->pending_drops リスト内の各アイテムは、トランザクションのライフサイクルに関連付けられている。
  • isCommit が true の場合、トランザクション内で「削除」された統計を物理削除する。
  • isCommit が false の場合、トランザクション内で「作成」された統計を物理削除する。

アルゴリズムまたは振る舞い

1. xact_state->pending_drops が空であれば即座に終了する。 2. リスト内の各 PgStat_PendingDroppedStatsItem を順次走査する。 3. コミット時かつ削除対象の統計であれば pgstat_drop_entry を実行する。 4. アボート時かつ作成対象の統計であれば pgstat_drop_entry を実行する。 5. 処理したアイテムをリストから削除し、pfree でメモリを解放する。 6. pgstat_drop_entry が失敗したアイテムが存在する場合、最後に pgstat_request_entry_refs_gc() を呼び出し、ガベージコレクションを要求する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/activity/pgstat_xact.c 内のトランザクション終了処理（AtEOXact_PgStat など）。
  • **呼び出し先**: pgstat_drop_entry（統計エントリの物理削除）、pgstat_request_entry_refs_gc（参照カウントのGC要求）、pfree（メモリ解放）。

関連データ構造

  • PgStat_SubXactStatus: トランザクションごとの統計状態を管理する構造体。
  • PgStat_PendingDroppedStatsItem: 保留中の統計変更アイテムを保持する構造体。

失敗モードと境界条件

  • pgstat_drop_entry が false を返した場合、そのアイテムは即座に解放されず、not_freed_count がインクリメントされる。この場合、最後に pgstat_request_entry_refs_gc() が呼び出される。

保守担当者向けメモ

  • 本関数は、統計の「作成」と「削除」がトランザクションの成否と逆の論理で処理される点に注意すること（コミット時は削除を反映し、アボート時は作成を無効化する）。
  • dclist_foreach_modify を使用してリストを走査・削除しているため、反復中のリスト操作は安全である。
