HandleFunctionRequest

定義

  • 種別: function
  • 定義箇所: src/backend/tcop/fastpath.c:187

目的

クライアントからの Fastpath（PQfn）関数実行リクエストを処理するメインエントリーポイント。SQL パーサーやプランナーを介さず、指定された OID に基づいてサーバー上の関数を直接実行する。

シグネチャまたは形状

void HandleFunctionRequest(StringInfo msgBuf)

入力と出力

  • **入力**: msgBuf (クライアントから送信された関数呼び出しメッセージを含む StringInfo バッファ)
  • **出力**: なし（実行結果は SendFunctionResult を介してクライアントへ送信される）

前提条件と不変条件

  • トランザクションがアボート状態でないこと（アボート状態の場合はエラーを発生させる）。
  • 実行には Namespace および Procedure に対する適切な権限（ACL_USAGE, ACL_EXECUTE）が必要。
  • 照合順序（Collation）を必要とする関数や、集合を返す関数（proretset）には非対応。

アルゴリズムまたは振る舞い

1. **初期化**: トランザクションスナップショットをアクティブ化。 2. **メタデータ取得**: msgBuf から関数 OID を読み取り、fetch_fp_info で関数情報をロード。 3. **権限検証**: スキーマおよび関数に対する実行権限をチェック。 4. **引数解析**: parse_fcall_arguments を使用してメッセージから引数をデシリアライズ。 5. **実行判定**: 関数が strict であり、かつ引数に NULL が含まれる場合は実行をスキップ。 6. **実行**: FunctionCallInvoke を呼び出し、結果を取得。 7. **応答**: SendFunctionResult で結果をクライアントへ返送。 8. **終了処理**: 割り込みチェックを行い、スナップショットを解放。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/postgres.c（PostgresMain 内で F メッセージを受信した際に呼び出される）
  • **呼び出し先**:
  • fetch_fp_info
  • object_aclcheck / aclcheck_error
  • parse_fcall_arguments
  • FunctionCallInvoke
  • SendFunctionResult
  • PushActiveSnapshot / PopActiveSnapshot

関連データ構造

  • StringInfo: 通信メッセージバッファ。
  • fp_info: 関数実行に必要なメタデータ（OID、名前、型情報等）を保持する構造体。
  • FunctionCallInfoData: 関数呼び出しの引数および実行状態を管理する構造体。

失敗モードと境界条件

  • **トランザクションエラー**: アボート中のトランザクションで呼び出された場合、ERRCODE_IN_FAILED_SQL_TRANSACTION でエラー終了。
  • **権限エラー**: スキーマまたは関数へのアクセス権がない場合、aclcheck_error によりエラー終了。
  • **メッセージ形式エラー**: pq_getmsgend により、メッセージの末尾が期待通りでない場合にエラー終了。

保守担当者向けメモ

  • 本関数内での palloc は MessageContext で行われるため、明示的な解放は不要。
  • fp_info のキャッシュは現在行われておらず、呼び出しごとにシステムカタログを検索する設計となっている。
  • 実行結果のシリアライズには SendFunctionResult が使用される。
