freetree

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_cleanup.c:114

目的

tsquery の処理に使用される NODE 構造体のツリーを再帰的に走査し、メモリを解放する。

シグネチャまたは形状

static void freetree(NODE *node)

入力と出力

  • 入力: NODE *node（解放対象のツリーのルートノード）
  • 出力: なし（void）

前提条件と不変条件

  • node が NULL の場合、何も行わずに終了する。
  • 再帰処理を行うため、スタックオーバーフローを防止する check_stack_depth() が呼び出される。

アルゴリズムまたは振る舞い

1. check_stack_depth() を呼び出し、スタックの深さを確認する。 2. node が NULL であれば即座にリターンする。 3. node->left が存在する場合、再帰的に freetree を呼び出す。 4. node->right が存在する場合、再帰的に freetree を呼び出す。 5. 最後に pfree(node) を呼び出し、現在のノードのメモリを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/utils/adt/tsquery_cleanup.c 内の各関数（clean_NOT、cleanup_tsquery_stopwords 等の処理終了時やエラーハンドリング時）。
  • 呼び出し先:
  • check_stack_depth()
  • freetree()（再帰呼び出し）
  • pfree()

関連データ構造

  • NODE: tsquery のクエリツリーを表現する構造体。

失敗モードと境界条件

  • node が NULL の場合は安全に処理される。
  • 極端に深いツリー構造の場合、check_stack_depth() によりスタックオーバーフローが未然に防がれる。

保守担当者向けメモ

  • 本関数はツリーの末端から順にメモリを解放する再帰的なクリーンアップ処理である。
  • pfree を使用しているため、PostgreSQL のメモリコンテキスト管理下で実行されることを前提としている。
