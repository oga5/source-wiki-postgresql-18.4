flagInhTables

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/common.c:268

目的

データベース内のテーブル継承関係を解決し、子テーブルから親テーブルへのリンク構築および、属性取得やインデックス生成に必要な「興味深い（interesting）」フラグの付与、パーティションテーブルに対する TableAttachInfo の生成を行う。

シグネチャまたは形状

static void flagInhTables(Archive *fout, TableInfo *tblinfo, int numTables, InhInfo *inhinfo, int numInherits)

入力と出力

  • **入力**:
  • fout: アーカイブハンドル。
  • tblinfo: 全テーブル情報の配列。
  • numTables: テーブル数。
  • inhinfo: 継承関係情報の配列。
  • numInherits: 継承関係の数。
  • **出力**: なし（副作用として tblinfo 内の構造体および TableAttachInfo オブジェクトを更新・生成する）。

前提条件と不変条件

  • tblinfo 配列は有効な TableInfo オブジェクトを保持していること。
  • inhinfo 配列は pg_inherits カタログから取得された継承関係情報であること。
  • パーティションテーブルの場合、親テーブルは必ず1つであること（それ以外は致命的エラー）。

アルゴリズムまたは振る舞い

1. **リンク構築**: inhinfo を走査し、各子テーブルの parents 配列に親テーブルへのポインタを追加する。findTableByOid で対象テーブルを検索し、見つからない場合はパーティションインデックスとみなしてスキップする。 2. **フラグ付与**: ダンプ対象のテーブルについて、その親テーブルの interesting フラグを true に設定する。 3. **パーティション処理**: ダンプ対象かつパーティションであるテーブルに対し、TableAttachInfo オブジェクトを生成する。

  • AssignDumpId を呼び出し、一意なダンプIDを割り当てる。
  • 親テーブルおよびパーティションテーブルへの依存関係を addObjectDependency で登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/common.c 内の getSchemaData（推測される文脈）。
  • **呼び出し先**: findTableByOid, pg_realloc_array, pg_malloc_array, pg_fatal, palloc, AssignDumpId, pg_strdup, addObjectDependency。

関連データ構造

  • TableInfo: テーブルのメタデータと継承関係を保持。
  • InhInfo: 継承関係の定義。
  • TableAttachInfo: パーティションの親子関係をダンプするための補助構造体。

失敗モードと境界条件

  • findTableByOid が親テーブルを見つけられない場合、pg_fatal を呼び出し処理を中断する。
  • パーティションテーブルの親数が1でない場合、pg_fatal を呼び出し処理を中断する。

保守担当者向けメモ

  • 本関数は getSchemaData の一部として実行され、ダンプ順序決定の前提となる依存関係グラフを構築する。
  • TableAttachInfo の生成は、ダンプ出力時にパーティションを親にアタッチする命令を正しく順序付けるために不可欠である。
  • 継承関係の解決ロジックは、PostgreSQL のバージョンアップに伴うカタログ構造の変更に影響を受けやすいため注意が必要。
