CreateAccessMethod

定義

  • 種別: function
  • 定義箇所: src/backend/commands/amcmds.c:42

目的

SQL コマンド CREATE ACCESS METHOD を実行し、新しいアクセスメソッドをシステムカタログ pg_am に登録する。

シグネチャまたは形状

ObjectAddress CreateAccessMethod(CreateAmStmt *stmt)

入力と出力

  • **入力**: CreateAmStmt *stmt（アクセスメソッド名、ハンドラ関数名、AM タイプを含む構文解析木）。
  • **出力**: ObjectAddress（作成されたアクセスメソッドのカタログ ID とクラス ID を含む構造体）。

前提条件と不変条件

  • 実行にはスーパーユーザー権限が必要。
  • 指定されたアクセスメソッド名が既存の pg_am エントリと重複していないこと。
  • lookup_am_handler_func により、指定されたハンドラ関数が AM タイプに対して妥当であると検証されていること。

アルゴリズムまたは振る舞い

1. pg_am リレーションを RowExclusiveLock でオープンする。 2. 呼び出し元がスーパーユーザーであることを確認し、権限不足の場合はエラーを投げる。 3. GetSysCacheOid1 を使用して、指定された名前のアクセスメソッドが既に存在しないか確認する。 4. lookup_am_handler_func を呼び出し、ハンドラ関数の OID を取得し、AM タイプとの適合性を検証する。 5. GetNewOidWithIndex で新しい OID を生成し、heap_form_tuple を用いて pg_am 用のタプルを構築する。 6. CatalogTupleInsert でカタログへ書き込み、タプルを解放する。 7. recordDependencyOn を使用して、ハンドラ関数との依存関係（DEPENDENCY_NORMAL）を記録する。 8. recordDependencyOnCurrentExtension を呼び出し、現在の拡張機能との依存関係を記録する。 9. InvokeObjectPostCreateHook を実行し、リレーションをクローズして作成したオブジェクトのアドレスを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c（CreateAmStmt の実行時）。
  • **呼び出し先**: table_open, superuser, GetSysCacheOid1, lookup_am_handler_func, GetNewOidWithIndex, heap_form_tuple, CatalogTupleInsert, heap_freetuple, recordDependencyOn, recordDependencyOnCurrentExtension, InvokeObjectPostCreateHook, table_close。

関連データ構造

  • CreateAmStmt: SQL 文の解析結果を保持する構造体。
  • pg_am: アクセスメソッドを格納するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトを識別するための構造体。

失敗モードと境界条件

  • **権限エラー**: スーパーユーザーでない場合、ERRCODE_INSUFFICIENT_PRIVILEGE で失敗する。
  • **重複エラー**: 同名のアクセスメソッドが既に存在する場合、ERRCODE_DUPLICATE_OBJECT で失敗する。
  • **検証エラー**: ハンドラ関数が見つからない、または型が適合しない場合、lookup_am_handler_func 内でエラーが発生する。

保守担当者向けメモ

  • 本関数は DDL 処理の一部であり、実行時のアクセスメソッド呼び出しロジックは含まない。
  • pg_am のスキーマ変更や依存関係管理のルール変更時には、タプル構築部分と recordDependencyOn の呼び出しを確認すること。
