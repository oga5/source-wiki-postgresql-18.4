pg_lsn_ge

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_lsn.c:162

目的

2つのログシーケンス番号（LSN）を比較し、第1引数が第2引数以上であるか（Greater than or Equal）を判定します。

シグネチャまたは形状

Datum pg_lsn_ge(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの pg_lsn 型の値を受け取ります。
  • **出力**: bool 型（Datum 形式）。第1引数が第2引数以上の場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力される pg_lsn は内部的に XLogRecPtr（64ビット整数）として扱われます。
  • PG_GETARG_LSN マクロにより、入力値が正しく XLogRecPtr 型へ変換されることが前提です。

アルゴリズムまたは振る舞い

1. PG_GETARG_LSN(0) および PG_GETARG_LSN(1) を呼び出し、2つのLSN値を XLogRecPtr 型の変数 lsn1 と lsn2 に取得します。 2. C言語の比較演算子 >= を用いて lsn1 >= lsn2 を評価します。 3. PG_RETURN_BOOL マクロを使用して、評価結果を Datum 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェースから演算子として呼び出されます。具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**: PG_GETARG_LSN および PG_RETURN_BOOL マクロを使用します。

関連データ構造

  • XLogRecPtr: LSNの内部表現である64ビット整数型。src/include/access/xlogdefs.h で定義されています。

失敗モードと境界条件

  • 入力値が有効な pg_lsn ではない場合、マクロ経由でエラーが発生する可能性があります。
  • 64ビット整数の範囲内であれば、LSNの値に関わらず正しく比較が行われます。

保守担当者向けメモ

  • 本関数は単純な比較演算であり、pg_lsn 型のB-treeインデックス等で使用される比較ロジックの一部です。
  • 算術演算とは異なり、numeric 型への変換を伴わないため、オーバーヘッドは最小限です。
