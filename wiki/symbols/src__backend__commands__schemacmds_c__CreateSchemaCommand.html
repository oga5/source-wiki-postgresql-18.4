CreateSchemaCommand

定義

  • 種別: function
  • 定義箇所: src/backend/commands/schemacmds.c:51

目的

SQLの CREATE SCHEMA 文を処理し、新しい名前空間の作成、所有権の設定、およびスキーマ内に含まれるサブコマンド（テーブル作成等）の逐次実行を行う。

シグネチャまたは形状

Oid CreateSchemaCommand(CreateSchemaStmt *stmt, const char *queryString, int stmt_location, int stmt_len)

入力と出力

  • **入力**:
  • stmt: CreateSchemaStmt 構造体（解析済みの CREATE SCHEMA 文）。
  • queryString: 元のSQLクエリ文字列。
  • stmt_location, stmt_len: クエリ内でのコマンドの位置情報。
  • **出力**:
  • 作成されたスキーマの Oid を返す。IF NOT EXISTS が指定され、かつスキーマが既に存在する場合は InvalidOid を返す。

前提条件と不変条件

  • 現在のデータベースに対する ACL_CREATE 権限が必要。
  • ターゲットロールへの切り替え権限が必要。
  • pg_ で始まるスキーマ名は、allowSystemTableMods が有効でない限り予約済みとして拒否される。

アルゴリズムまたは振る舞い

1. **所有者の特定**: stmt->authrole があればそのロール、なければ現在のユーザーを所有者とする。 2. **権限チェック**: データベース作成権限とロール切り替え権限を検証。 3. **存在確認**: IF NOT EXISTS 指定時、スキーマが存在すれば通知を出して終了。 4. **所有者切り替え**: 必要に応じて SetUserIdAndSecContext で一時的に所有者へ権限を昇格。 5. **スキーマ作成**: NamespaceCreate を呼び出し、CommandCounterIncrement で可視化。 6. **検索パスの更新**: 作成中のスキーマを一時的に search_path の先頭に追加。 7. **イベントトリガー通知**: EventTriggerCollectSimpleCommand を呼び出し。 8. **サブコマンド実行**: transformCreateSchemaStmtElements でサブコマンドを順序立て、ProcessUtility をループで呼び出して順次実行。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/extension.c
  • src/backend/tcop/utility.c
  • **呼び出し先**:
  • NamespaceCreate
  • ProcessUtility
  • transformCreateSchemaStmtElements
  • get_rolespec_oid, object_aclcheck, check_can_set_role 等の権限・カタログ関連関数。

関連データ構造

  • CreateSchemaStmt: CREATE SCHEMA 文の構文解析結果を保持。
  • pg_namespace: スキーマ情報を格納するシステムカタログ。

失敗モードと境界条件

  • 権限不足: aclcheck_error によりエラー終了。
  • 予約済み名称: pg_ で始まる名前は ERRCODE_RESERVED_NAME でエラー終了。
  • 存在チェック: IF NOT EXISTS がない場合、既存スキーマ作成試行は NamespaceCreate 内でエラーとなる。

保守担当者向けメモ

  • サブコマンドの実行は ProcessUtility を再帰的に呼び出すことで実現されている。
  • search_path の一時的な変更は、エラー発生時に自動的に復元されるよう GUC_ACTION_SAVE を使用している。
  • 所有者変更時の権限移行には aclnewowner が関与する。
