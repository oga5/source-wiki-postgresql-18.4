join_is_legal

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/joinrels.c:349

目的

クエリ内の外部結合（Outer Join）、セミ結合（Semi Join）、アンチ結合（Anti Join）などの制約に基づき、2つのリレーション（rel1, rel2）の結合が論理的に許容されるかを判定する。

シグネチャまたは形状

static bool join_is_legal(PlannerInfo *root, RelOptInfo *rel1, RelOptInfo *rel2,
                          Relids joinrelids,
                          SpecialJoinInfo **sjinfo_p, bool *reversed_p)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel1, rel2: 結合対象のリレーション。
  • joinrelids: 結合後のリレーションIDセット。
  • **出力**:
  • 戻り値: 結合が正当であれば true、そうでなければ false。
  • sjinfo_p: 該当する SpecialJoinInfo 構造体へのポインタ（見つからない場合は NULL）。
  • reversed_p: 結合順序が SpecialJoinInfo の定義に対して逆転している場合に true。

前提条件と不変条件

  • 出力パラメータ sjinfo_p と reversed_p は、関数開始時に必ず初期化される。
  • 結合が複数の SpecialJoinInfo に該当する場合、その結合は無効とみなされ false を返す。

アルゴリズムまたは振る舞い

1. root->join_info_list を走査し、提案された結合が SpecialJoinInfo の制約に抵触しないか確認する。 2. 結合対象が SpecialJoinInfo の右辺（RHS）と重複する場合、以下の判定を行う:

  • 既に結合済み、またはRHS内に完全に含まれる場合はスキップ。
  • セミ結合の場合、RHSがユニーク化可能であれば結合を許可する。
  • min_lefthand と min_righthand の包含関係に基づき、結合の正当性を検証する。

    • 3. 結合が LEFT JOIN のRHSに関連する場合、JOIN_LEFT 以外のタイプ（SEMI, ANTI 等）や、LHSとの重複がある場合は false を返す。 4. 最終的に、制約に違反しない場合にのみ true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/joinrels.c 内の join_search_one_level 等、結合探索を行う各箇所から呼び出される。
  • **呼び出し先**: bms_overlap, bms_is_subset, bms_equal などのビットマップ操作関数、および create_unique_path（セミ結合のユニーク化判定用）。

関連データ構造

  • PlannerInfo: クエリ全体のプランニング情報。
  • RelOptInfo: 結合対象のリレーション情報。
  • SpecialJoinInfo: 外部結合やセミ結合の制約情報。

失敗モードと境界条件

  • 複数の SpecialJoinInfo に同時に適合する場合、不正な結合パスとして false を返す。
  • JOIN_LEFT 以外の特殊な結合において、RHSへの不正な結合が試みられた場合は false を返す。
  • 結合条件がない場合や、ヒューリスティックによる結合順序の制限により、探索が打ち切られる場合がある。

保守担当者向けメモ

  • 本関数はSQLの結合制約（特に LATERAL 参照や外部結合の順序）を厳密に管理している。
  • SpecialJoinInfo の min_lefthand / min_righthand の判定ロジックは、Bushy Planの生成可否に直結するため、変更時は結合順序の制約に注意すること。
  • 戻り値が false の場合でも、プランナが他のパスを探索できるよう設計されている。
