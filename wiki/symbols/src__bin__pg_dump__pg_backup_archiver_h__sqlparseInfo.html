sqlparseInfo

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.h:160

目的

INSERT データの解析状態を保持し、断片的な入力行をバッファリングして SQL 構文解析を支援するための構造体です。

シグネチャまたは形状

typedef struct
{
	sqlparseState state;		/* 解析状態 */
	bool		backSlash;		/* 次の文字がバックスラッシュでエスケープされているか */
} sqlparseInfo;

入力と出力

  • **入力**: sqlparseState による状態遷移および PQExpBuffer を介した文字列データ。
  • **出力**: 解析対象の SQL 文の断片を保持する curCmd バッファ。

前提条件と不変条件

  • curCmd が NULL の場合、現在処理中のコマンド行は存在しません。
  • backSlash フラグは、エスケープシーケンスの継続性を制御するために使用されます。

アルゴリズムまたは振る舞い

  • INSERT データのストリームを読み込み、行が完結していない場合に curCmd に蓄積します。
  • バックスラッシュによるエスケープ状態を追跡し、SQL の構文境界を正しく判定します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_dump/pg_backup_archiver.h:230 にて ArchiveHandle 構造体の一部として定義されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数呼び出し関係は特定できません。

関連データ構造

  • sqlparseState: 解析の現在のフェーズを定義する列挙型（定義詳細は本構造体外）。
  • PQExpBuffer: 文字列バッファ操作用構造体。

失敗モードと境界条件

  • curCmd のメモリ確保失敗時、PQExpBuffer の動作に依存します。
  • 入力データが不完全な状態で終了した場合、curCmd に残存するデータが適切に処理される必要があります。

保守担当者向けメモ

  • 本構造体は pg_dump における INSERT データの解析という特定の用途に特化しています。
  • sqlparseState の定義と併せて参照してください。
