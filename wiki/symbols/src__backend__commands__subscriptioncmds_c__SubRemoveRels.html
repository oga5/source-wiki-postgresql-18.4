SubRemoveRels

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/subscriptioncmds.c:833

目的

論理レプリケーションにおいて、サブスクリプションから削除対象となるリレーション（テーブル）の識別子と、その現在の同期状態を一時的に保持するために使用されます。

シグネチャまたは形状

typedef struct SubRemoveRels
{
    Oid         relid;
    char        state;
} SubRemoveRels;

入力と出力

  • 本シンボルはデータ構造であり、関数ではありません。
  • palloc を用いて SubRemoveRels 型の配列としてメモリ確保され、削除対象のリレーション情報を格納するために利用されます。

前提条件と不変条件

  • relid は有効なテーブルの OID である必要があります。
  • state はサブスクリプションに関連付けられたリレーションの同期状態を表す文字コードです。

アルゴリズムまたは振る舞い

  • src/backend/commands/subscriptioncmds.c 内において、削除対象となるリレーションのリストを管理する際に使用されます。
  • subrel_count に基づいてメモリを動的に確保し、対象となるリレーションの情報を保持するバッファとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/subscriptioncmds.c 内のロジック（sub_remove_rels 変数への割り当て箇所）。
  • 呼び出し先: 本シンボルはデータ構造であるため、直接的な呼び出し先はありません。

関連データ構造

  • pg_subscription_rel: サブスクリプションとリレーションの関連付けを管理するシステムカタログ。

失敗モードと境界条件

  • メモリ不足時: palloc が失敗した場合、PostgreSQL の標準的なエラーハンドリングによりトランザクションがアボートされます。
  • データの不整合: 保持される relid がシステムカタログ上に存在しない場合、後続の処理でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は subscriptioncmds.c 内での一時的なデータ保持を目的としています。
  • 構造体のフィールド（relid, state）を変更する場合は、これらを参照しているループ処理やメモリ確保ロジックに影響がないか確認してください。
