RI_FKey_noaction_upd

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ri_triggers.c:675

目的

外部キー制約における NO ACTION 更新操作を処理するトリガー関数です。更新時に制約違反が発生していないかを検証します。

シグネチャまたは形状

Datum RI_FKey_noaction_upd(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（トリガー呼び出しのコンテキスト情報を含む）。
  • **出力**: Datum 型（トリガーの実行結果。通常は PointerGetDatum(NULL)）。

前提条件と不変条件

  • 本関数は、UPDATE イベントに対するトリガーとして呼び出される必要があります。
  • 内部で ri_CheckTrigger を呼び出し、呼び出しタイミングとイベントが RI_TRIGTYPE_UPDATE に適合しているかを検証します。

アルゴリズムまたは振る舞い

1. ri_CheckTrigger を呼び出し、トリガーの呼び出しコンテキストが適切かを確認します。 2. ri_restrict 関数を呼び出し、実際の制約チェックを実行します。この際、第2引数に true を渡すことで、RESTRICT 動作と同様のロジックを共有します。 3. ri_restrict の戻り値をそのまま返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/tablecmds.c（外部キー制約定義時にトリガー関数として登録される）。
  • **呼び出し先**:
  • ri_CheckTrigger（トリガーの妥当性検証）。
  • ri_restrict（制約チェックの実行）。

関連データ構造

  • TriggerData: fcinfo->context からキャストして取得される、トリガー実行時のデータ構造。

失敗モードと境界条件

  • ri_CheckTrigger による検証で、呼び出しタイミングやイベントが不適切と判断された場合、エラーが発生します。
  • ri_restrict 内で制約違反が検出された場合、エラーが報告されます。

保守担当者向けメモ

  • 本関数は RESTRICT 動作とロジックを共有しています。NO ACTION の挙動を変更またはデバッグする際は、ri_restrict の実装も併せて確認してください。
