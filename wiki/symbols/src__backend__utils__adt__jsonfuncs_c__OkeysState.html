OkeysState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/jsonfuncs.c:55

目的

JSONオブジェクトのキーを列挙する関数（json_object_keys 等）の実行中における状態を保持するために使用されます。

シグネチャまたは形状

typedef struct OkeysState
{
	JsonLexContext *lex;
	char	  **result;
	int			result_size;
	int			result_count;
	int			sent_count;
} OkeysState;

入力と出力

  • lex: JSON解析のための字句解析コンテキスト。
  • result: 抽出されたキー文字列の配列。
  • result_size: result 配列の確保済みサイズ。
  • result_count: 現在までに格納されたキーの総数。
  • sent_count: 既に呼び出し元へ返却済みのキー数。

前提条件と不変条件

  • result_count は result_size を超えないこと。
  • sent_count は result_count を超えないこと。

アルゴリズムまたは振る舞い

本構造体は、集合を返す関数（SRF）のコンテキスト内で、JSONオブジェクトからキーを抽出し、メモリ上に保持して順次返却するためのバッファとして機能します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/jsonfuncs.c 内の json_object_keys 関連関数群（行571, 596, 625, 735, 747, 773, 788, 811, 826）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • JsonLexContext: JSONの字句解析状態を管理する構造体。

失敗モードと境界条件

  • メモリ不足: palloc によるメモリ確保時に発生する可能性があります。
  • JSON解析エラー: lex コンテキストが不正なJSONを指している場合に発生します。

保守担当者向けメモ

  • 本構造体は funcctx->user_fctx に格納され、SRFの複数呼び出し間で状態を維持するために利用されます。
  • result 配列の拡張が必要な場合は、result_size を更新し、適切にメモリを再確保する必要があります。
