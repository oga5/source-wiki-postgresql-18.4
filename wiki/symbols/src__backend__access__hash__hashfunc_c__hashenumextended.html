hashenumextended

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:134

目的

列挙型（enum）の値をハッシュ結合等のためにハッシュ化します。シード値を受け取ることで、ハッシュ結合のパフォーマンスを向上させる拡張ハッシュ関数として機能します。

シグネチャまたは形状

Datum hashenumextended(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて以下の引数を受け取ります。
  • 第1引数: OID 型（列挙型の内部表現）。
  • 第2引数: int64 型（ハッシュ計算に使用するシード値）。
  • **出力**: Datum 型として計算された32ビットのハッシュ値を返します。

前提条件と不変条件

  • 入力された OID は uint32 にキャスト可能である必要があります。
  • ハッシュ関数は、同一の入力に対して常に同一のハッシュ値を返す必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID(0) により列挙型の値を uint32 として取得します。 2. PG_GETARG_INT64(1) によりシード値を取得します。 3. hash_uint32_extended を呼び出し、列挙値とシード値を組み合わせてハッシュ値を計算し、その結果を返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのハッシュアクセスメソッド（pg_amproc に登録された演算子クラス経由で呼び出されます）。
  • **呼び出し先**: hash_uint32_extended（src/include/common/hashfn.h で定義）。

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • PG_FUNCTION_ARGS: SQL関数呼び出しの引数コンテナ。

失敗モードと境界条件

  • 入力引数が不足している場合や型が不一致な場合、PG_GETARG_* マクロの内部でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は hashenum の拡張版です。新しい列挙型関連のハッシュ処理を追加または変更する際は、一貫性を保つために両方の関数を確認してください。
  • hash_uint32_extended を利用しているため、基盤となるハッシュアルゴリズムの変更は common/hashfn.h 側の仕様に依存します。
