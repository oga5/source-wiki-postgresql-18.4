AddNewAttributeTuples

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:847

目的

新規リレーション作成時に、そのリレーションの属性（列）情報をシステムカタログ pg_attribute に登録し、データ型や照合順序（Collation）に対する依存関係を記録する。

シグネチャまたは形状

static void AddNewAttributeTuples(Oid new_rel_oid, TupleDesc tupdesc, char relkind)

入力と出力

  • **入力**:
  • new_rel_oid: 新規作成するリレーションの OID。
  • tupdesc: 属性定義を含むタプル記述子。
  • relkind: リレーションの種類（pg_class.relkind に相当）。
  • **出力**: なし（副作用として pg_attribute への挿入と依存関係の記録を行う）。

前提条件と不変条件

  • 本関数はトランザクション内で呼び出される必要がある。
  • pg_attribute に対する RowExclusiveLock を保持して実行される。

アルゴリズムまたは振る舞い

1. pg_attribute を開き、関連するインデックスをオープンする。 2. InsertPgAttributeTuples を呼び出し、tupdesc に基づく属性タプルを pg_attribute に挿入する。 3. 各属性について、データ型 (atttypid) への依存関係を recordDependencyOn で記録する。 4. デフォルト以外の照合順序が指定されている場合、その照合順序への依存関係も記録する。 5. relkind がビューや複合型でない場合、システム属性（ctid 等）を定義する SysAtt を用いて同様に pg_attribute へ挿入する。 6. オープンしたインデックスとリレーションをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/heap.c:429
  • src/backend/catalog/heap.c:1456
  • **呼び出し先**:
  • table_open, table_close
  • CatalogOpenIndexes, CatalogCloseIndexes
  • InsertPgAttributeTuples
  • recordDependencyOn
  • CreateTupleDesc, FreeTupleDesc

関連データ構造

  • TupleDesc: 属性定義の保持に使用。
  • pg_attribute: 挿入対象のシステムカタログ。
  • SysAtt: システム属性の定義配列。

失敗モードと境界条件

  • ビューや複合型の場合はシステム属性の追加をスキップする。
  • デフォルトの照合順序 (DEFAULT_COLLATION_OID) は固定（pinned）されているため、依存関係の記録対象外とする。

保守担当者向けメモ

  • システム属性の定義は SysAtt にハードコーディングされており、変更には注意が必要。
  • pg_attribute への挿入は InsertPgAttributeTuples に委譲されているため、カタログスキーマの変更時はそちらも確認すること。
