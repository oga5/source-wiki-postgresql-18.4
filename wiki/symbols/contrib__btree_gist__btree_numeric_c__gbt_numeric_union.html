gbt_numeric_union

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_numeric.c:125

目的

GiST インデックス構築時、複数の numeric 型キーを包含する最小の範囲（Union）を計算する。

シグネチャまたは形状

Datum gbt_numeric_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GistEntryVector *entryvec（結合対象のインデックスエントリ群）。
  • PG_GETARG_POINTER(1): int32 *size（結果のサイズを格納するポインタ）。
  • **出力**:
  • Datum: 結合された範囲を表す numeric 型のデータ構造へのポインタ。

前提条件と不変条件

  • tinfo（gbtree_vinfo 構造体）が、numeric 型の比較関数群を保持する設定情報として定義されていること。
  • btree_utils_var.c で定義された汎用的な可変長型処理関数 gbt_var_union が利用可能であること。

アルゴリズムまたは振る舞い

1. entryvec から入力された複数の numeric キーを抽出する。 2. btree_utils_var.c で提供される汎用関数 gbt_var_union を呼び出し、numeric 型の比較ロジック（tinfo）を用いてそれらを包含する最小の範囲を計算する。 3. 計算結果を Datum として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GiST インデックス構築プロセス（直接の呼び出し元はソースコード上では明示されていません）。
  • **呼び出し先**:
  • gbt_var_union（contrib/btree_gist/btree_utils_var.c 内の汎用関数）。

関連データ構造

  • GistEntryVector: GiST インデックスの結合対象エントリを保持する構造体。
  • gbtree_vinfo (tinfo): numeric 型の比較関数群を保持する設定情報。

失敗モードと境界条件

  • 入力データが空の場合や不正な numeric 型データが含まれる場合の挙動は、依存先の gbt_var_union の実装に依存する。

保守担当者向けメモ

  • 本関数は btree_utils_var.h で定義された汎用ロジックへのアダプターとして機能している。
  • numeric 型固有の比較ロジックを変更する場合は、tinfo の定義を確認すること。
