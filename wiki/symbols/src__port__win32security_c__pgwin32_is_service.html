pgwin32_is_service

定義

  • 種別: function
  • 定義箇所: src/port/win32security.c:119

目的

現在のプロセスが Windows サービスとして実行されているかどうかを判定します。

シグネチャまたは形状

int pgwin32_is_service(void);

入力と出力

  • **入力**: なし
  • **出力**:
  • 1: サービスとして実行されている場合
  • 0: サービスとして実行されていない場合
  • -1: 判定処理中にエラーが発生した場合

前提条件と不変条件

  • 初回実行時に判定結果を静的変数 _is_service にキャッシュし、以降の呼び出しではキャッシュ値を返します。
  • 標準エラー出力（stderr）が有効なハンドルを持つ場合、サービスではないとみなします。

アルゴリズムまたは振る舞い

1. **キャッシュ確認**: _is_service が初期値（-1）以外であれば、その値を即座に返します。 2. **標準エラー出力チェック**: GetStdHandle(STD_ERROR_HANDLE) を呼び出し、有効なハンドルが存在する場合はサービスではない（0）と判定します。 3. **LocalSystem アカウント確認**: SECURITY_LOCAL_SYSTEM_RID を用いて、現在のプロセスが LocalSystem アカウントで実行されているか CheckTokenMembership で確認します。該当する場合はサービス（1）と判定します。 4. **サービスグループ確認**: SECURITY_SERVICE_RID を用いて、現在のプロセスがサービスグループに属しているかを確認します。該当する場合はサービス（1）、それ以外はサービスではない（0）と判定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/error/elog.c
  • src/bin/pg_ctl/pg_ctl.c
  • **呼び出し先**:
  • Windows API: GetStdHandle, AllocateAndInitializeSid, CheckTokenMembership, FreeSid, GetLastError
  • 標準ライブラリ: fprintf

関連データ構造

  • SID_IDENTIFIER_AUTHORITY: SID 生成に使用される構造体。

失敗モードと境界条件

  • AllocateAndInitializeSid または CheckTokenMembership が失敗した場合、標準エラー出力にエラーメッセージを出力し、-1 を返します。
  • 判定はプロセス単位で行われるため、実行中にサービス状態が変化するケースは考慮されていません。

保守担当者向けメモ

  • 本関数は初期化フェーズで呼び出されることを想定しており、ereport() が利用できない環境でも動作するように fprintf(stderr, ...) を使用しています。
  • 判定ロジックは Windows のセキュリティトークンに基づいているため、OS の仕様変更や権限モデルの変更時には注意が必要です。
