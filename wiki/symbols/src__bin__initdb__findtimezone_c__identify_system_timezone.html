identify_system_timezone

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/findtimezone.c:330

目的

ホストシステムの現在のタイムゾーン設定を検出し、PostgreSQLが利用可能なIANAタイムゾーンデータベース形式の名称、またはPOSIX形式のタイムゾーン文字列を特定して返します。

シグネチャまたは形状

static const char *identify_system_timezone(void);

入力と出力

  • **入力**: なし（内部で time(NULL) および tzset() を使用してシステム状態を取得）。
  • **出力**: 成功時は特定されたタイムゾーン名（const char *）、失敗時は NULL（GMTへのフォールバックを意味する）。

前提条件と不変条件

  • tzset() が呼び出され、OSのタイムゾーンライブラリが初期化されていること。
  • pg_TZDIR() が有効なタイムゾーンデータベースディレクトリを指していること。
  • 戻り値のバッファは静的領域（resultbuf）を指しており、呼び出し元で即座に利用またはコピーする必要がある。

アルゴリズムまたは振る舞い

1. **サンプリングデータの生成**: 現在時刻を基準に、過去100年分の週次データ（木曜日）を tztry 構造体に格納し、システムとデータベースの照合用テストセットを作成する。 2. **直接探索**: /etc/localtime のシンボリックリンクを check_system_link_file で確認し、一致すればその名称を返す。 3. **ブルートフォース探索**: scan_available_timezones を呼び出し、データベース内の全ゾーンとサンプリングデータを比較して最もスコアの高いものを選択する。 4. **POSIX形式の構築**: 上記で特定できない場合、localtime() を用いて標準時（STD）および夏時間（DST）の名称とオフセットを特定し、STD<ofs>DST 形式等の文字列を生成して score_timezone で妥当性を検証する。 5. **フォールバック**: 最終的に一致するものがない場合は NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/findtimezone.c 内の他関数（select_default_timezone 等から呼び出されることが示唆されている）。
  • **呼び出し先**: tzset, time, localtime, build_time_t, check_system_link_file, pg_TZDIR, scan_available_timezones, strftime, get_timezone_offset, score_timezone。

関連データ構造

  • struct tztry: タイムゾーン照合用のテスト時刻セットを保持。
  • struct tm: システム時刻の分解に使用。

失敗モードと境界条件

  • localtime() が失敗した場合、NULL を返す。
  • データベース内に一致するゾーンが見つからない場合、NULL を返す。
  • 「Factory」ゾーンが検出された場合、IANAの仕様に従い NULL（GMT）として扱う。
  • 過去のタイムゾーン定義が不正確なシステムでは、スコアが低い場合に誤ったゾーンが選択される可能性がある。

保守担当者向けメモ

  • 本関数は initdb 専用のロジックであり、バックエンドのタイムゾーン処理とは独立しています。
  • Unix系プラットフォームでの探索は、過去のサンプリングデータとの比較という計算コストの高い手法をとるため、データベースの規模やOSの仕様変更の影響を受けやすい点に注意してください。
