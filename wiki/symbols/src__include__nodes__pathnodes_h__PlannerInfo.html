PlannerInfo

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/pathnodes.h:212

目的

PostgreSQLのクエリプランナにおけるルート構造体であり、現在のクエリレベルの最適化状態を保持します。

シグネチャまたは形状

typedef struct PlannerInfo PlannerInfo;

※構造体の詳細なメンバ定義は、提供されたソース抜粋には含まれていません。

入力と出力

  • 入力: クエリの構文木（Query）を基に初期化されます。
  • 出力: 最適化プロセスを経て、実行可能な Plan ノードツリーの生成に使用されます。

前提条件と不変条件

  • プランナの再帰的な処理において頻繁に参照されます。
  • RelOptInfo や Path との間に複雑なリンクを持つため、copyObject() 等の汎用関数によるコピーはサポートされていません。

アルゴリズムまたは振る舞い

1. クエリの最適化状態を管理し、ベースリレーションの探索や結合探索の過程で情報を蓄積します。 2. RelOptInfo を通じてリレーションの統計情報やコスト見積もりを管理します。 3. 最終的に create_plan() を通じて、選択された最適な Path を実行計画へ変換します。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）の例:
  • contrib/bloom/blcost.c: blcostestimate
  • contrib/file_fdw/file_fdw.c: fileGetForeignRelSize, fileGetForeignPaths, fileGetForeignPlan, fileIsForeignScanParallelSafe, estimate_size, estimate_costs
  • 呼び出し先: 提供されたソース抜粋には、本構造体が呼び出す具体的な関数情報は含まれていません。

関連データ構造

  • PlannerGlobal: クエリ全体の最適化状態を保持。
  • RelOptInfo: 特定のリレーションに関する最適化情報。
  • Path: 実行計画の候補。
  • RestrictInfo: 制限句の最適化用メタデータ。
  • EquivalenceClass: 等価な式グループ。

失敗モードと境界条件

提供されたソース抜粋には、失敗モードや境界条件に関する記述はありません。

保守担当者向けメモ

  • 本構造体はプランナの再帰処理で多用されるため、変更時は最適化ロジック全体への影響を考慮してください。
  • 構造体のメンバ定義については、src/include/nodes/pathnodes.h の該当行以降を参照してください。
