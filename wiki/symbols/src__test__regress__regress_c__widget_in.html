widget_in

定義

  • 種別: function
  • 定義箇所: src/test/regress/regress.c:174

目的

PostgreSQL のテスト用カスタムデータ型 WIDGET の入力関数として、文字列形式の表現を内部構造体へ変換する。

シグネチャまたは形状

Datum widget_in(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して、WIDGET 型の文字列表現（例: (x,y,radius) 形式）を受け取る。
  • **出力**: PG_RETURN_POINTER(result) を介して、メモリ上に確保された WIDGET 構造体へのポインタを Datum として返す。

前提条件と不変条件

  • 入力文字列は LDELIM で始まり、RDELIM で終わる形式である必要がある。
  • 内部で atof を使用して数値を解析するため、入力文字列の各要素は数値として解釈可能であること。
  • 構文エラー時は ereport(ERROR, ...) を呼び出し、トランザクションを中断させる（ハードエラー）。

アルゴリズムまたは振る舞い

1. 入力文字列を走査し、区切り文字（DELIM）に基づいて座標および半径の各要素を特定する。 2. 必要な要素数（NARGS）が不足している場合、ERRCODE_INVALID_TEXT_REPRESENTATION を伴うエラーを発生させる。 3. palloc を使用して WIDGET 構造体用のメモリを確保する。 4. atof を用いて文字列を数値に変換し、構造体の各メンバ（center.x, center.y, radius）に代入する。 5. 構築した構造体ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの CREATE FUNCTION を通じて呼び出される。ソースコード上では PG_FUNCTION_INFO_V1(widget_in) マクロによりエクスポートされている。
  • **呼び出し先**: palloc（メモリ確保）、ereport（エラー報告）、atof（文字列変換）などの PostgreSQL 内部 API および標準 C ライブラリ関数。

関連データ構造

  • WIDGET: 本関数が生成するテスト用データ型構造体。

失敗モードと境界条件

  • 入力文字列の形式が不正（区切り文字の欠落や要素数不足）な場合、ereport(ERROR) により処理が停止する。
  • atof が変換できない文字列が渡された場合、数値変換結果が 0.0 になる可能性がある。

保守担当者向けメモ

  • 本関数はテスト専用であり、意図的にハードエラー（ereport(ERROR)）を使用している。エラーハンドリングのテストケースとして機能するため、安易に「ソフト」なエラー処理へ変更しないこと。
