gbt_bitle

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:49

目的

BIT 型および VARBIT 型のデータに対して「以下（less than or equal to）」の比較を行い、その結果をブール値で返します。

シグネチャまたは形状

bool gbt_bitle(const void *a, const void *b, Oid collation, FmgrInfo *flinfo)

入力と出力

  • **入力**:
  • a, b: 比較対象のビット列データへのポインタ。
  • collation: 照合順序（本関数では使用されません）。
  • flinfo: 関数呼び出し情報（本関数では使用されません）。
  • **出力**:
  • a <= b であれば true、そうでなければ false を返します。

前提条件と不変条件

  • 入力ポインタ a および b は、PostgreSQL の Datum として解釈可能な有効なメモリ領域を指している必要があります。
  • 内部的に bitle 関数を呼び出すため、bitle が定義するビット列の順序付け規則に従います。

アルゴリズムまたは振る舞い

1. PointerGetDatum を用いて、入力ポインタ a および b を Datum 型に変換します。 2. DirectFunctionCall2 を介して、PostgreSQL 標準の bitle 関数を呼び出します。 3. bitle の戻り値を DatumGetBool でブール型に変換し、呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • bitle (PostgreSQL 標準のビット列比較関数)
  • **呼び出し元**:
  • contrib/btree_gist/btree_bit.c 内の tinfo 構造体（GiST インデックスの比較テーブル）から参照されています。

関連データ構造

  • tinfo: gbtree_vinfo 構造体。本関数を含む比較関数群が登録されています。

失敗モードと境界条件

  • 入力が NULL の場合、DirectFunctionCall2 の挙動に依存します。
  • bitle 関数がサポートしないデータ型や不正なビット列が渡された場合、例外が発生する可能性があります。

保守担当者向けメモ

  • 本関数は DirectFunctionCall2 を使用したラッパーです。ロジックの変更が必要な場合は、基底となる bitle 関数の仕様を確認してください。
  • 照合順序（collation）や FmgrInfo は現在の実装では無視されていますが、将来的な拡張やインターフェースの互換性のためにシグネチャに含まれています。
