utf8_to_uhc

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/utf8_and_uhc/utf8_and_uhc.c:62

目的

PostgreSQLの文字コード変換フレームワークにおいて、UTF-8エンコーディングからUHC（Unified Hangul Code）エンコーディングへの変換を実行する。

シグネチャまたは形状

Datum utf8_to_uhc(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて以下の引数を受け取る。
  • src (CSTRING): 変換元のUTF-8文字列。
  • dest (CSTRING): 変換後のUHC文字列を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 変換失敗時にエラーを報告するかどうかのフラグ。
  • **出力**: PG_RETURN_INT32 を通じて、実際に変換されたバイト数を返す。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_UTF8, PG_UHC) により、変換元がUTF-8、変換先がUHCであることを検証する。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得する。 2. UtfToLocal 関数を呼び出し、uhc_from_unicode_tree（utf8_to_uhc.map由来）を使用してUTF-8からUHCへの変換を行う。 3. 変換結果のバイト数を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの文字コード変換フレームワーク（直接的な呼び出し元情報はソースコード上では明示されていない）。
  • **呼び出し先**: UtfToLocal（PostgreSQLのマルチバイト処理ライブラリ関数）。

関連データ構造

  • uhc_from_unicode_tree: 変換テーブル（utf8_to_uhc.map）で定義された変換ツリー構造。

失敗モードと境界条件

  • noError 引数が false の場合、変換不可能な文字が含まれているとエラーが発生する可能性がある。
  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQLの標準的なエラー処理が実行される。

保守担当者向けメモ

  • 本関数は定型的なラッパーであり、実際の変換ロジックは UtfToLocal および utf8_to_uhc.map に依存している。
  • 変換テーブルの詳細は src/backend/Unicode/utf8_to_uhc.map を参照すること。
