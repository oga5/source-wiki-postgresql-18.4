make_append_rel_info

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/appendinfo.c:50

目的

継承テーブルやパーティションテーブルの親子関係を管理するための AppendRelInfo 構造体を生成し、初期化します。

シグネチャまたは形状

AppendRelInfo *make_append_rel_info(Relation parentrel, Relation childrel, Index parentRTindex, Index childRTindex)

入力と出力

  • **入力**:
  • parentrel: 親テーブルのリレーション記述子。
  • childrel: 子テーブルのリレーション記述子。
  • parentRTindex: 親リレーションのレンジテーブルインデックス。
  • childRTindex: 子リレーションのレンジテーブルインデックス。
  • **出力**:
  • 初期化済みの AppendRelInfo 構造体へのポインタ。

前提条件と不変条件

  • parentrel および childrel は有効な Relation 構造体である必要があります。
  • make_inh_translation_list を内部で呼び出し、親子の属性マッピングを構築します。

アルゴリズムまたは振る舞い

1. makeNode(AppendRelInfo) を使用してメモリを確保します。 2. 引数から parent_relid、child_relid、parent_reltype、child_reltype を設定します。 3. make_inh_translation_list を呼び出し、親から子への属性変換リストを構築して appinfo に格納します。 4. RelationGetRelid を用いて親の OID を取得し、parent_reloid に設定します。 5. 完成した AppendRelInfo 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/inherit.c:518
  • **呼び出し先**:
  • make_inh_translation_list (内部関数)
  • makeNode (マクロ)
  • RelationGetRelid (マクロ)

関連データ構造

  • AppendRelInfo: 親子関係の変換情報を保持する主要な構造体。

失敗モードと境界条件

  • 属性の型や照合順序が継承関係として不整合な場合、内部で呼び出される make_inh_translation_list 等がエラーを報告する可能性があります。

保守担当者向けメモ

  • 本関数はプランナが継承関係を解決する際のメタデータ作成の起点となります。
  • 属性変換の詳細は make_inh_translation_list に依存しているため、変換ロジックの変更時はそちらも確認してください。
