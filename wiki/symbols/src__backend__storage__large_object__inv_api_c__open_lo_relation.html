open_lo_relation

定義

  • 種別: function
  • 定義箇所: src/backend/storage/large_object/inv_api.c:72

目的

ラージオブジェクト（LO）操作に必要なシステムカタログ pg_largeobject テーブルおよびそのインデックスを、現在のトランザクション内でオープンし、キャッシュされたリレーション参照（lo_heap_r, lo_index_r）を初期化する。

シグネチャまたは形状

static void open_lo_relation(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバルなリレーションキャッシュを更新）

前提条件と不変条件

  • lo_heap_r および lo_index_r が既にオープンされている場合、関数は即座にリターンする。
  • リレーションの所有権は、現在のリソースオーナーではなく TopTransactionResourceOwner に割り当てられる。

アルゴリズムまたは振る舞い

1. キャッシュされたリレーション（lo_heap_r, lo_index_r）が既に存在するか確認し、存在すれば終了する。 2. 現在のリソースオーナーを一時的に TopTransactionResourceOwner に切り替える。 3. table_open を用いて LargeObjectRelationId を RowExclusiveLock モードでオープンし、lo_heap_r に格納する。 4. index_open を用いて LargeObjectLOidPNIndexId を RowExclusiveLock モードでオープンし、lo_index_r に格納する。 5. リソースオーナーを元の値に戻す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/large_object/inv_api.c 内の各LO操作関数（inv_create, inv_open, inv_read, inv_write, inv_truncate 等）から呼び出される。
  • 呼び出し先:
  • table_open
  • index_open

関連データ構造

  • lo_heap_r: pg_largeobject テーブルへのリレーション参照。
  • lo_index_r: pg_largeobject のインデックスへのリレーション参照。

失敗モードと境界条件

  • トランザクション外で呼び出された場合や、カタログへのアクセス権限がない場合はエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数でオープンされたリレーションは、トランザクション終了時に close_lo_relation を呼び出して適切にクローズする必要がある。
  • RowExclusiveLock を使用しているため、読み取りだけでなく書き込みを伴う操作も想定されている。
