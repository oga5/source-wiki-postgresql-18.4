ExecCopySlotMinimalTuple

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/executor/tuptable.h:495

目的

TupleTableSlot に保持されているタプルを MinimalTuple 形式でコピーし、呼び出し元のコンテキストで割り当てられたメモリ領域として返します。

シグネチャまたは形状

MinimalTuple ExecCopySlotMinimalTuple(TupleTableSlot *slot)

入力と出力

  • **入力**: slot (TupleTableSlot *) - コピー対象のタプルを保持するスロット。
  • **出力**: MinimalTuple - 複製された最小タプル。

前提条件と不変条件

  • slot は有効な TupleTableSlot を指している必要があります。
  • slot->tts_ops が適切に初期化されており、copy_minimal_tuple 関数ポインタが実装されている必要があります。

アルゴリズムまたは振る舞い

1. 指定された slot の tts_ops（仮想関数テーブル）にアクセスします。 2. copy_minimal_tuple メソッドを呼び出し、スロット内のタプルを MinimalTuple 形式に変換・コピーします。 3. 複製されたタプルを呼び出し元へ返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execTuples.c
  • src/backend/executor/nodeMemoize.c
  • src/backend/executor/nodeSetOp.c
  • src/backend/utils/sort/tuplesortvariants.c
  • src/backend/utils/sort/tuplestore.c
  • **呼び出し先**:
  • slot->tts_ops->copy_minimal_tuple（スロットの実装型に応じた具体的なコピー関数）

関連データ構造

  • TupleTableSlot: タプルを保持する抽象コンテナ。
  • MinimalTuple: PostgreSQL 内部で利用される、ヘッダーを最小化したタプル形式。
  • TupleTableSlotOps: スロットの操作を定義する関数ポインタテーブル。

失敗モードと境界条件

  • メモリ不足時には、基盤となるメモリ割り当て関数によりエラーが発生する可能性があります。
  • スロットが空（TTS_EMPTY）の場合の挙動は、各スロット実装の copy_minimal_tuple 実装に依存します。

保守担当者向けメモ

  • 本関数はインライン化されており、スロットの型によらず一貫したインターフェースを提供します。
  • 戻り値の MinimalTuple は呼び出し元でメモリ管理（解放）を行う必要があります。
