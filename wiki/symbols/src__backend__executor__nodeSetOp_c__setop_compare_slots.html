setop_compare_slots

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSetOp.c:377

目的

2つの TupleTableSlot を比較し、SetOpState で定義されたソートキーに基づいて、それらが等しいか、あるいはどちらが大きいかを判定する。

シグネチャまたは形状

static int setop_compare_slots(TupleTableSlot *s1, TupleTableSlot *s2, SetOpState *setopstate)

入力と出力

  • **入力**:
  • s1, s2: 比較対象の TupleTableSlot。
  • setopstate: 実行状態を保持する SetOpState 構造体。
  • **出力**:
  • int: 比較結果。0 は両者が等しいことを示し、それ以外は ApplySortComparator の戻り値に従う。

前提条件と不変条件

  • s1 および s2 は有効なタプルスロットであること。
  • setopstate->numCols が比較対象の列数を正しく保持していること。
  • setopstate->sortKeys が初期化済みであること。

アルゴリズムまたは振る舞い

1. slot_getallattrs を呼び出し、両スロットの全属性をメモリ上に展開する。 2. setopstate->numCols で指定された回数だけループし、各列の値を比較する。 3. sortKey から属性番号 (ssup_attno) を取得し、対応する Datum と isNull フラグを抽出する。 4. ApplySortComparator を使用して列単位の比較を行う。 5. 比較結果が 0 でない場合、直ちにその値を返す。 6. 全ての列で差が見つからなかった場合、0 を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeSetOp.c 内の複数箇所（setop_retrieve_sorted 等のロジック内で、タプル間の等価性判定に使用）。
  • **呼び出し先**:
  • slot_getallattrs
  • ApplySortComparator

関連データ構造

  • TupleTableSlot: 比較対象のタプルデータを保持する構造体。
  • SetOpState: 実行ノードの状態およびソートキー情報を保持する構造体。
  • SortSupport: 各列の比較ロジックを定義する構造体。

失敗モードと境界条件

  • スロット内の属性番号が範囲外である場合、メモリ破壊や不正なアクセスが発生する可能性がある。
  • setopstate->sortKeys が適切に設定されていない場合、未定義の動作を引き起こす。

保守担当者向けメモ

  • 本関数は SetOp ノードの比較ロジックの中核であり、パフォーマンスに直結する。
  • slot_getallattrs を無条件に呼び出しているため、必要に応じて最適化の余地があるか検討すること。
