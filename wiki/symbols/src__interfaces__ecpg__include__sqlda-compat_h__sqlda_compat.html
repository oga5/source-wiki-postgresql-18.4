sqlda_compat

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/include/sqlda-compat.h:37

目的

Informix 互換の SQLDA（SQL Descriptor Area）構造を提供し、ECPG における動的 SQL 文の実行時にホスト変数とデータベース間でのデータ受け渡しを仲介します。

シグネチャまたは形状

struct sqlda_compat
{
    short        sqld;
    struct sqlvar_compat *sqlvar;
    char         desc_name[19];
    short        desc_occ;
    struct sqlda_compat *desc_next;
    void        *reserved;
};

入力と出力

  • **入力**: ECPG ランタイムライブラリが SQL 文を実行する際、アプリケーションによって準備された本構造体がバインド変数として利用されます。
  • **出力**: データベースからの結果セットが本構造体を通じてアプリケーションに格納されます。

前提条件と不変条件

  • Informix の仕様に準拠したフィールド配置とサイズを維持する必要があります。
  • reserved フィールドは内部利用または将来の拡張用であり、アプリケーション側で直接操作してはなりません。

アルゴリズムまたは振る舞い

本構造体はデータ定義のみを含みます。ECPG ランタイムは、この構造体内の sqlvar ポインタを介して sqlvar_compat 配列にアクセスし、メタデータやデータ値の読み書きを行います。desc_next を用いて複数の SQLDA を連結リストとして管理可能です。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/interfaces/ecpg/ecpglib/descriptor.c
  • src/interfaces/ecpg/ecpglib/execute.c
  • src/interfaces/ecpg/ecpglib/sqlda.c
  • **呼び出し先**:
  • ecpg_build_compat_sqlda (関数)
  • ecpg_set_compat_sqlda (関数)

関連データ構造

  • struct sqlvar_compat: 各列やパラメータの情報を保持する構造体。
  • src/interfaces/ecpg/include/sqlda.h: 汎用的な SQLDA 定義。
  • src/interfaces/ecpg/include/sqlda-native.h: ECPG ネイティブ形式の SQLDA 定義。

失敗モードと境界条件

  • sqld（保持する変数の数）と desc_occ（構造体のサイズ）の整合性が取れていない場合、メモリ操作時に不正なアクセスが発生する可能性があります。
  • 32KB を超えるデータを取り扱う場合は、拡張フィールドである sqlilongdata（sqlvar_compat 内）の適切な利用が求められます。

保守担当者向けメモ

  • 本構造体は Informix との互換性を最優先としています。変更を加える際は、既存の Informix アプリケーションとのバイナリ互換性に影響がないか確認してください。
  • desc_name は 19 バイトの固定長配列である点に注意してください。
