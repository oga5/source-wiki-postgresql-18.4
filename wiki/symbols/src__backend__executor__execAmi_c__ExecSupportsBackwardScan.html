ExecSupportsBackwardScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execAmi.c:510

目的

指定されたプランノードが、実行時に後方スキャン（逆方向へのタプル取得）をサポートしているかを判定する。

シグネチャまたは形状

bool ExecSupportsBackwardScan(Plan *node)

入力と出力

  • **入力**: Plan *node（判定対象のプランノード）
  • **出力**: bool（サポートしている場合は true、それ以外は false）

前提条件と不変条件

  • node が NULL の場合は false を返す。
  • node->parallel_aware が true の場合、並列実行時の状態管理が困難であるため、常に false を返す。

アルゴリズムまたは振る舞い

プランノードの型（nodeTag）に基づき、以下のルールで再帰的または直接的に判定を行う。

1. **再帰的判定**: Result, Append, SubqueryScan, LockRows, Limit ノードは、子プラン（outerPlan や subplan）のサポート状況に依存する。

  • Append ノードは、非同期プラン（nasyncplans > 0）が存在する場合、後方スキャンをサポートしない。

    • 2. **直接判定**:

  • IndexScan, IndexOnlyScan: IndexSupportsBackwardScan を呼び出し、インデックスの特性を確認する。
  • CustomScan: CUSTOMPATH_SUPPORT_BACKWARD_SCAN フラグが立っている場合のみ true。
  • SeqScan, TidScan, TidRangeScan, FunctionScan, ValuesScan, CteScan, Material, Sort: これらは後方スキャンをサポートする。

    • 3. **拒否**: SampleScan, Gather, IncrementalSort は、仕様または実装上の制約により後方スキャンをサポートしない。 4. **デフォルト**: 上記以外のノードは false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/portalcmds.c:147
  • src/backend/executor/spi.c:1700
  • src/backend/optimizer/plan/planner.c:467
  • src/backend/executor/execAmi.c 内の再帰呼び出し
  • **呼び出し先**:
  • IndexSupportsBackwardScan
  • ExecSupportsBackwardScan（再帰呼び出し）

関連データ構造

  • Plan: 実行プランの基本構造体。
  • Append, IndexScan, IndexOnlyScan, SubqueryScan, CustomScan: 各種プランノードの派生構造体。

失敗モードと境界条件

  • プランツリーが深い場合、再帰呼び出しによるスタックオーバーフローのリスクがある。
  • Append ノードにおいて、子プランのいずれか一つでも後方スキャンをサポートしない場合、全体として false となる。

保守担当者向けメモ

  • 新しいプランノードを追加する際は、本関数の switch 文に適切なケースを追加すること。
  • ExecSupportsBackwardScan はプランツリーを再帰的に走査するため、ノードの追加時には後方スキャンを阻害する要因がないか慎重に検討すること。
