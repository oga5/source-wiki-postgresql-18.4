dump_variables

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/variable.c:460

目的

SQL文の入出力引数リストを再帰的に走査し、各変数およびインジケータ変数の型情報を出力（コード生成）します。

シグネチャまたは形状

void dump_variables(struct arguments *list, int mode)

入力と出力

  • **入力**:
  • list: 処理対象となる arguments 構造体のリンクリスト。
  • mode: リスト要素の解放を制御するフラグ（0以外の場合、処理後に要素を free する）。
  • **出力**: なし（副作用として ECPGdump_a_type を呼び出し、Cコードを生成）。

前提条件と不変条件

  • list が NULL の場合、関数は何もせず即座に復帰します。
  • リストは先頭から構築されているため、再帰呼び出しによってリストの末尾から順に処理されます。

アルゴリズムまたは振る舞い

1. リストが NULL であれば終了します。 2. dump_variables(list->next, mode) を再帰的に呼び出し、リストの末尾まで到達します。 3. 現在の要素について ECPGdump_a_type を呼び出し、変数名、型、スコープ情報、およびインジケータ変数の情報を出力します。 4. mode が 0 でない場合、現在のリスト要素を free します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/output.c:157 (argsinsert を引数として呼び出し)
  • src/interfaces/ecpg/preproc/output.c:160 (argsresult を引数として呼び出し)
  • src/interfaces/ecpg/preproc/variable.c:475 (再帰呼び出し)
  • **呼び出し先**:
  • ECPGdump_a_type (コード生成用)
  • mm_strdup (文字列メモリ確保)
  • free (メモリ解放)

関連データ構造

  • struct arguments: SQL文の引数リストを保持する構造体。
  • ECPGtype: 変数の型情報を保持する構造体。

失敗モードと境界条件

  • list が NULL の場合は安全に処理されます。
  • mode が 0 の場合、リスト要素のメモリ解放は行われません。

保守担当者向けメモ

  • 本関数は再帰的にリストを処理するため、リストが非常に長い場合にスタックオーバーフローのリスクがないか留意してください。
  • ECPGdump_a_type への引数として渡される str_zero は、常に "0" という文字列を指す一時的なメモリとして確保・解放されます。
