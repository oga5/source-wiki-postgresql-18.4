index_beginscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/indexam.c:255

目的

インデックススキャンを開始し、指定されたインデックスリレーションに対するスキャン記述子（IndexScanDesc）を初期化および構築します。

シグネチャまたは形状

IndexScanDesc index_beginscan(Relation heapRelation,
                              Relation indexRelation,
                              Snapshot snapshot,
                              IndexScanInstrumentation *instrument,
                              int nkeys,
                              int norderbys)

入力と出力

  • **入力**:
  • heapRelation: スキャン対象のヒープリレーション。
  • indexRelation: スキャン対象のインデックスリレーション。
  • snapshot: スキャンに使用する MVCC スナップショット。
  • instrument: 実行統計用構造体（NULL 可）。
  • nkeys: インデックス検索キーの数。
  • norderbys: ソートキーの数。
  • **出力**:
  • 初期化済みの IndexScanDesc 構造体へのポインタ。

前提条件と不変条件

  • snapshot は InvalidSnapshot であってはなりません（Assert により検証）。

アルゴリズムまたは振る舞い

1. index_beginscan_internal を呼び出し、インデックス AM 固有の初期化処理を実行して IndexScanDesc を生成します。 2. 生成されたスキャン記述子に、ヒープリレーション、スナップショット、および計測用ポインタを格納します。 3. table_index_fetch_begin を呼び出し、ヒープからインデックス一致タプルを取得するための準備を行います。 4. 構築された IndexScanDesc を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c
  • src/backend/access/index/genam.c
  • src/backend/catalog/index.c
  • src/backend/executor/execIndexing.c
  • **呼び出し先**:
  • index_beginscan_internal
  • table_index_fetch_begin

関連データ構造

  • IndexScanDesc: スキャン状態を保持する主要な構造体。
  • Relation: リレーション記述子。
  • Snapshot: MVCC スナップショット。
  • IndexScanInstrumentation: スキャン実行の計測情報。

失敗モードと境界条件

  • snapshot が InvalidSnapshot の場合、アサーション違反によりプロセスが停止します。
  • インデックス AM が ambeginscan を実装していない場合、index_beginscan_internal 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数で取得したリレーションの参照カウントは、後続の index_endscan で解放される必要があります。
  • index_beginscan_internal がスキャン記述子の大部分を初期化するため、パラメータの追加や変更を行う際は両関数の整合性に注意してください。
