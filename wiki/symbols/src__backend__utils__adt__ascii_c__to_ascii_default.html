to_ascii_default

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/ascii.c:155

目的

SQL関数 to_ascii(text) のバックエンド実装として、現在のデータベースエンコーディングに基づき、入力されたテキストデータをASCII近似表現に変換します。

シグネチャまたは形状

Datum to_ascii_default(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る text 型のデータ。
  • **出力**: Datum 型として返される、変換済みの text 型データ。

前提条件と不変条件

  • 入力データは PG_GETARG_TEXT_P_COPY(0) によりコピーされ、関数内で安全に処理されます。
  • 現在のデータベースエンコーディング（GetDatabaseEncoding()）が変換の基準として使用されます。

アルゴリズムまたは振る舞い

1. 入力された text データをコピーして取得します。 2. GetDatabaseEncoding() を呼び出し、現在のデータベースのエンコーディングを取得します。 3. encode_to_ascii 関数を呼び出し、取得したエンコーディングに基づいてデータをインプレース変換します。 4. 変換結果を PG_RETURN_TEXT_P を通じて返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（to_ascii 関数）から呼び出されます。
  • **呼び出し先**:
  • GetDatabaseEncoding()
  • encode_to_ascii()
  • PG_GETARG_TEXT_P_COPY()
  • PG_RETURN_TEXT_P()

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • 入力データが NULL の場合、PG_GETARG_TEXT_P_COPY のマクロ定義に従った挙動となります。
  • encode_to_ascii が内部で対応していないエンコーディングの場合の挙動は、当該関数の実装に依存します。

保守担当者向けメモ

  • 本関数は to_ascii SQL関数のデフォルト引数版を実装しています。
  • encode_to_ascii は入力データを直接書き換えるため、変換前後で文字列長が変化しない前提で動作します。
