assign_locale_numeric

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:383

目的

lc_numeric 設定が変更された際に、キャッシュされた数値フォーマット情報を無効化するために使用されます。

シグネチャまたは形状

void assign_locale_numeric(const char *newval, void *extra)

入力と出力

  • **入力**:
  • newval: 設定される新しいロケール値（使用されません）。
  • extra: GUC フック用の追加データ（使用されません）。
  • **出力**: なし（void）。

前提条件と不変条件

  • CurrentLocaleConvValid というグローバルなフラグを制御します。
  • 本関数が呼び出されると、数値フォーマットのキャッシュ状態が「無効」となります。

アルゴリズムまたは振る舞い

1. CurrentLocaleConvValid 変数に false を代入します。 2. これにより、次回数値フォーマットが必要になった際に、システムが新しいロケール設定に基づいて情報を再取得することを強制します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/misc/guc_tables.c: GUC（Grand Unified Configuration）パラメータ lc_numeric の設定変更時に呼び出されます。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • CurrentLocaleConvValid: 数値フォーマット情報のキャッシュ有効性を保持するグローバル変数。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。常に成功します。

保守担当者向けメモ

  • 本関数は GUC フックとして機能します。lc_numeric 設定が変更された際にキャッシュをクリアする役割に特化しているため、ロケール設定自体の適用ロジックは含まれていません。
  • PGLC_localeconv() 等の数値フォーマット取得関数と対になる存在です。
