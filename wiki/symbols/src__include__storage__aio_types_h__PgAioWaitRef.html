PgAioWaitRef

定義

  • 種別: struct
  • 定義箇所: src/include/storage/aio_types.h:32

目的

非同期I/O（AIO）操作の完了を待機するための参照ハンドルです。I/Oハンドルが再利用される環境において、古いI/O操作を誤って待機することを防ぐための世代管理機能を提供します。

シグネチャまたは形状

typedef struct PgAioWaitRef
{
    uint32      aio_index;        /* PgAioHandle を特定する内部ID */
    uint32      generation_upper; /* 世代番号（上位32ビット） */
    uint32      generation_lower; /* 世代番号（下位32ビット） */
} PgAioWaitRef;

入力と出力

  • **入力**: I/O発行時に pgaio_io_get_wref を通じて初期化されます。
  • **出力**: pgaio_io_from_wref を介して、対応する PgAioHandle と世代番号の照合に使用されます。

前提条件と不変条件

  • generation_upper と generation_lower は、ハンドル再利用時の競合を回避するために使用されます。
  • 構造体サイズを抑えるため、64ビットの世代番号を2つの32ビットフィールドに分割して保持しています。

アルゴリズムまたは振る舞い

1. **識別**: aio_index により対象のハンドルを特定します。 2. **検証**: I/O完了待機時に、現在のハンドル世代と PgAioWaitRef 内の世代番号を比較します。 3. **再利用対策**: ハンドルが再利用されるたびに世代番号が増加するため、古い参照を持つ呼び出し元は、新しいI/O操作を誤って待機することなく、不一致を検知できます。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/buffer/localbuf.c
  • **呼び出し先（操作）**:
  • src/backend/storage/aio/aio.c: pgaio_io_from_wref, pgaio_io_get_wref, pgaio_wref_clear, pgaio_wref_valid, pgaio_wref_get_id, pgaio_wref_wait, pgaio_wref_check_done

関連データ構造

  • PgAioHandle: 本構造体が参照する実体。
  • PgAioResult, PgAioReturn: I/O完了後のステータス管理に使用される関連構造体。

失敗モードと境界条件

  • ハンドルが再利用された場合、世代番号の不一致により pgaio_wref_valid 等が失敗を報告します。
  • 世代番号がオーバーフローした場合の挙動については、本ソース excerpt には明記されていません。

保守担当者向けメモ

  • 本構造体は aio_types.h で定義されており、メモリ効率を考慮して設計されています。
  • 世代管理のロジックを変更する場合は、src/backend/storage/aio/aio.c 内の pgaio_io_from_wref 等の検証ロジックと整合性を保つ必要があります。
