NUMDesc

定義

  • **種別**: struct
  • **定義箇所**: src/backend/utils/adt/formatting.c:310

目的

数値フォーマット変換（TO_CHAR や TO_NUMBER 等）において、数値の属性（精度、符号、ロケール設定など）を保持し、フォーマット処理の制御を行うための構造体です。

シグネチャまたは形状

typedef struct {
    int pre;            /* 小数点前の桁数 */
    int post;           /* 小数点後の桁数 */
    int lsign;          /* ロケール固有の符号を使用するか */
    int flag;           /* 数値パラメータフラグ */
    int pre_lsign_num;  /* lsign 用の一時値 */
    int multi;          /* 'V' 指定子による乗数 */
    int zero_start;     /* 最初のゼロの位置 */
    int zero_end;       /* 最後のゼロの位置 */
    int need_locale;    /* ロケール設定が必要か */
} NUMDesc;

入力と出力

本構造体は関数間で受け渡されるデータ保持用であり、直接的な入出力関数ではありません。主に NUMDesc_prepare によって初期化され、NUM_processor や NUM_cache 等の数値変換処理で参照されます。

前提条件と不変条件

  • NUMDesc_prepare 関数を介して、FormatNode から適切に初期化されている必要があります。
  • 数値変換処理の実行中、この構造体はフォーマット文字列の解析結果と整合している必要があります。

アルゴリズムまたは振る舞い

  • 数値フォーマット文字列を解析した結果（FormatNode）を、数値処理エンジンが理解できる形式に変換・保持します。
  • pre および post フィールドは、数値の有効桁数や配置を決定するために使用されます。
  • multi フィールドは、フォーマット内の 'V' 指定子（スケーリング）を処理するために使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • NUMDesc_prepare (初期化)
  • NUM_cache (キャッシュ検索)
  • NUM_processor (数値変換処理)
  • **呼び出し先**:
  • 構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • FormatNode: フォーマット文字列の解析結果を保持するツリー構造。
  • NUMCacheEntry: 数値フォーマットの解析結果をキャッシュするための構造体。

失敗モードと境界条件

  • フォーマット文字列が複雑すぎる場合や、NUMDesc の各フィールドが許容範囲を超える数値設定を持つ場合、変換処理が予期しない挙動を示す可能性があります。
  • キャッシュサイズ（DCH_CACHE_SIZE）を超えるフォーマット文字列が指定された場合、キャッシュは利用されず、毎回解析が行われます。

保守担当者向けメモ

  • 新しい数値フォーマット指定子を追加する場合は、NUMDesc 構造体のフィールド定義と、NUMDesc_prepare 内での初期化ロジックを更新する必要があります。
  • lsign や need_locale はセッションのロケール設定（LC_NUMERIC）に依存するため、ロケール変更時の整合性に注意してください。
