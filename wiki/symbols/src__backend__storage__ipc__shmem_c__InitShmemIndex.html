InitShmemIndex

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/shmem.c:282

目的

共有メモリ上の全データ構造を管理するためのハッシュテーブル（ShmemIndex）を初期化します。

シグネチャまたは形状

void InitShmemIndex(void)

入力と出力

  • 入力: なし
  • 出力: なし（グローバル変数 ShmemIndex に初期化されたハッシュテーブルを代入）

前提条件と不変条件

  • ShmemInitHash を呼び出す際、内部で ShmemInitStruct が呼び出されます。
  • ShmemInitStruct は本来 ShmemIndex が存在することを前提としますが、本関数では "ShmemIndex" という特別な名前を指定することで、この循環依存を回避し、ハッシュテーブルを自己構築します。

アルゴリズムまたは振る舞い

1. HASHCTL 構造体にキーサイズ (SHMEM_INDEX_KEYSIZE) とエントリサイズ (sizeof(ShmemIndexEnt)) を設定します。 2. ShmemInitHash を呼び出し、名前 "ShmemIndex" でハッシュテーブルを作成します。 3. 作成されたハッシュテーブルのポインタをグローバル変数 ShmemIndex に格納します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c 内の共有メモリ初期化シーケンスから呼び出されます。
  • 呼び出し先: ShmemInitHash を呼び出します。

関連データ構造

  • ShmemIndex: 共有メモリ上の全データ構造を管理するハッシュテーブル。
  • ShmemIndexEnt: ハッシュテーブルのエントリ構造体。
  • HASHCTL: ハッシュテーブルの初期化パラメータを保持する構造体。

失敗モードと境界条件

  • 共有メモリの割り当てが不足している場合、ShmemInitHash 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は共有メモリ管理のブートストラップにおいて極めて重要な役割を果たします。ShmemIndex の初期化順序を変更する際は、ShmemInitStruct との循環依存関係に注意してください。
