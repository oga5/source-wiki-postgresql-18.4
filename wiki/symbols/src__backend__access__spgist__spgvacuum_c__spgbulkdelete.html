spgbulkdelete

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgvacuum.c:949

目的

SP-GiST インデックスに対する VACUUM 処理のメインエントリポイントとして、指定されたヒープタプルを参照するインデックスエントリを削除し、インデックスの肥大化を抑制する。

シグネチャまたは形状

IndexBulkDeleteResult *spgbulkdelete(IndexVacuumInfo *info, IndexBulkDeleteResult *stats, IndexBulkDeleteCallback callback, void *callback_state)

入力と出力

  • **入力**:
  • info: VACUUM 実行に関する情報（インデックスリレーション等）。
  • stats: 統計情報を保持する構造体（NULL の場合は新規割り当て）。
  • callback: 不要なタプルを判定するためのコールバック関数。
  • callback_state: コールバック関数に渡す状態データ。
  • **出力**:
  • 処理結果を格納した IndexBulkDeleteResult 構造体へのポインタ。

前提条件と不変条件

  • stats が NULL の場合、内部で palloc0 を用いてメモリが確保される。
  • spgBulkDeleteState 構造体に引数で受け取った情報を集約し、後続の走査処理へ引き継ぐ。

アルゴリズムまたは振る舞い

1. stats が NULL の場合、IndexBulkDeleteResult 用のメモリを確保する。 2. IndexVacuumInfo、stats、callback、callback_state を spgBulkDeleteState 構造体に格納する。 3. spgvacuumscan を呼び出し、インデックスの物理走査と不要タプルの削除処理を実行する。 4. 最終的な統計情報 stats を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgutils.c: amroutine->ambulkdelete への代入を通じて呼び出される。
  • **呼び出し先**:
  • spgvacuumscan: インデックスの走査と削除処理の実行。

関連データ構造

  • spgBulkDeleteState: VACUUM 実行中の状態（コールバック、統計情報、保留リスト等）を保持する構造体。
  • IndexBulkDeleteResult: VACUUM 処理の結果統計を保持する構造体。
  • IndexVacuumInfo: VACUUM 実行時のメタ情報。

失敗モードと境界条件

  • メモリ不足時: palloc0 によるメモリ確保が失敗した場合、PostgreSQL の標準的なエラーハンドリングによりアボートする。
  • コールバックの挙動: callback 関数が異常終了した場合、処理は中断される。

保守担当者向けメモ

  • 本関数は SP-GiST の ambulkdelete インターフェースを実装している。
  • 実際の削除ロジックは spgvacuumscan およびその配下の関数群（vacuumLeafPage 等）に委譲されているため、詳細なページ操作についてはそちらを参照すること。
