execute_attr_map_slot

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/tupconvert.c:191

目的

TupleTableSlot 間で属性の再配置（マッピング）を行い、物理構造の異なるタプル間でのデータ変換を実行する。

シグネチャまたは形状

TupleTableSlot *execute_attr_map_slot(AttrMap *attrMap, TupleTableSlot *in_slot, TupleTableSlot *out_slot)

入力と出力

  • **入力**:
  • attrMap: 属性の対応関係を定義するマップ構造体。
  • in_slot: 変換元の TupleTableSlot。
  • out_slot: 変換先の TupleTableSlot。
  • **出力**:
  • 変換後の out_slot を返す。

前提条件と不変条件

  • in_slot および out_slot の tts_tupleDescriptor が NULL でないこと。
  • in_slot および out_slot の tts_values が NULL でないこと。

アルゴリズムまたは振る舞い

1. slot_getallattrs(in_slot) を呼び出し、入力スロットの全属性値を展開する。 2. ExecClearTuple(out_slot) を呼び出し、出力スロットの既存内容をクリアする。 3. attrMap に基づき、入力スロットの値をループで走査して出力スロットへ転送する。

  • attrMap->attnums[i] が 0 の場合、出力側の属性は NULL として扱う。
  • それ以外の場合、対応する入力属性値をコピーする。

    • 4. ExecStoreVirtualTuple(out_slot) を呼び出し、出力スロットを仮想タプルとしてマークする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c
  • src/backend/commands/trigger.c
  • src/backend/executor/execMain.c
  • **呼び出し先**:
  • slot_getallattrs
  • ExecClearTuple
  • ExecStoreVirtualTuple

関連データ構造

  • AttrMap: 属性のインデックス対応を保持する構造体。
  • TupleTableSlot: 変換対象となるタプル保持用スロット。

失敗モードと境界条件

  • attrMap が指すインデックスが in_slot の属性数を超過している場合、メモリ不正アクセスの可能性がある（呼び出し側で整合性が保証されている必要がある）。

保守担当者向けメモ

  • 本関数は仮想タプルを操作するため、物理的なタプル変換（HeapTuple）とは異なるパスを通る。
  • attrMap->attnums[i] は 1 始まりのインデックスであるため、内部で -1 して 0 始まりの配列アクセスに変換している。
