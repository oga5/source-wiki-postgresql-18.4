oidvectorin

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:137

目的

スペース区切りの文字列（例: "123 456 789"）を、PostgreSQLの内部形式である oidvector 構造体に変換します。

シグネチャまたは形状

Datum oidvectorin(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_CSTRING(0) を介して、Oidのリストを表す文字列を受け取ります。
  • **出力**: oidvector 構造体へのポインタを Datum として返します。エラー発生時は NULL を返します。

前提条件と不変条件

  • 入力文字列はスペース区切りの数値列である必要があります。
  • oidvector は1次元配列として扱われ、インデックスの下限（lbound1）は常に0です。
  • oidvector 内に NULL 要素は存在しません。

アルゴリズムまたは振る舞い

1. 初期サイズ32で oidvector 構造体をメモリ確保します。 2. 文字列を走査し、空白をスキップしながら uint32in_subr を用いて各数値を Oid に変換します。 3. 配列の要素数が確保済みサイズを超えた場合、repalloc で領域を拡張します。 4. 変換完了後、SET_VARSIZE で構造体のサイズを確定し、ndim（1）、dataoffset（0）、elemtype（OIDOID）、dim1（要素数）、lbound1（0）を設定します。 5. 変換中にエラーが発生した場合は PG_RETURN_NULL() を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていませんが、PostgreSQLの型システム（oidvector 型の入力関数）として登録されることを想定しています。
  • **呼び出し先**:
  • uint32in_subr: 文字列から Oid への変換に使用。
  • palloc0, repalloc: メモリ管理に使用。
  • SOFT_ERROR_OCCURRED: エラー判定に使用。

関連データ構造

  • oidvector: Oid の固定長配列を保持する構造体。
  • Oid: PostgreSQLのオブジェクト識別子型。

失敗モードと境界条件

  • 文字列が空の場合、要素数0の oidvector を生成します。
  • 数値変換時に不正な文字列が渡された場合、uint32in_subr を通じてエラーが通知され、SOFT_ERROR_OCCURRED により NULL が返されます。

保守担当者向けメモ

  • oidvector は歴史的な経緯により、一般的な配列型とは異なりインデックスが0から始まる点に注意してください。
  • nalloc の初期値（32）は経験的な推測値です。
