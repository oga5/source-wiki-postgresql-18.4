getSchemaData

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/common.c:97

目的

データベース内の全オブジェクトのメタデータを収集し、ダンプ可能な状態に構築する pg_dump のメインオーケストレーション関数です。

シグネチャまたは形状

TableInfo *getSchemaData(Archive *fout, int *numTablesPtr)

入力と出力

  • **入力**:
  • fout: データベース接続およびアーカイブ状態を保持する Archive 構造体。
  • numTablesPtr: 収集されたテーブル数を格納するための出力用ポインタ。
  • **出力**:
  • 収集された全テーブル情報を保持する TableInfo 構造体の配列。

前提条件と不変条件

  • 呼び出し前にデータベースへの接続が確立されている必要があります。
  • 依存関係の解決順序（拡張機能 → スキーマ → テーブル → 関数 → 型 → その他オブジェクト）が厳密に守られる必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: 拡張機能、スキーマ、テーブル情報を順次取得します。 2. **収集**: 関数、型、演算子、アクセス手法、全文検索設定、外部データラッパー、権限、照合順序、変換、キャストなどを順次取得します。 3. **関係構築**:

  • getInherits で継承情報を取得。
  • flagInhTables でテーブル間の継承関係を構築。
  • getTableAttrs および flagInhAttrs で属性の継承状態を解決。
  • getPartitioningInfo でパーティション情報を取得。
  • getIndexes および flagInhIndexes でインデックス情報を構築。

    • 4. **制約とルール**: 制約、トリガー、ルール、ポリシー、パブリケーション、サブスクリプション情報を取得します。 5. **終了処理**: 一時的な継承情報メモリを解放し、テーブル数とテーブル配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内のメイン処理フローから呼び出されます。
  • **呼び出し先**: getExtensions, getNamespaces, getTables, getFuncs, getInherits, flagInhTables, flagInhAttrs, getIndexes, flagInhIndexes 等、多数のメタデータ取得・処理関数。

関連データ構造

  • TableInfo: テーブルのメタデータを保持する構造体。
  • Archive: ダンプ処理のコンテキストを保持。
  • ExtensionInfo, InhInfo: それぞれ拡張機能および継承関係を保持する構造体。

失敗モードと境界条件

  • データベース接続が切断された場合、各内部関数でエラーが発生します。
  • 依存関係の解決中にカタログ情報が不整合な場合、後続の flag 系関数で予期せぬ動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は pg_dump のデータ収集フェーズの司令塔です。新しいオブジェクトタイプを追加する場合は、依存関係を考慮した適切な順序で関数呼び出しを追加してください。
  • inhinfo は処理の最後に free() されるため、それ以前に参照を保持しないよう注意してください。
