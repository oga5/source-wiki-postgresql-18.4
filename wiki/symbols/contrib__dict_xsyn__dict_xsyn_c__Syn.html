Syn

定義

  • 種別: struct
  • 定義箇所: contrib/dict_xsyn/dict_xsyn.c:28

目的

全文検索におけるシノニム展開の単位として、検索キーとなる単語と、それに対応するシノニムのリストを保持する。

シグネチャまたは形状

typedef struct
{
	char	   *key;
	char	   *value;
} Syn;

入力と出力

  • key: 検索対象となる単語（文字列）。
  • value: 展開されるシノニムのリストを含む未パースの文字列。

前提条件と不変条件

  • value には、キー自身を含むシノニムのリストが含まれる。
  • 辞書データとしてメモリ上にロードされた後、key を基準としてソートされた状態で配列として管理される。

アルゴリズムまたは振る舞い

  • dxsyn_init において、外部ファイルから読み込まれた各行がこの構造体に格納される。
  • qsort を用いて key の昇順にソートされる。
  • dxsyn_lexize において、入力単語をキーとして bsearch による $O(\log n)$ の高速検索が行われる。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/dict_xsyn/dict_xsyn.c: read_dictionary（配列の構築）、compare_syn（ソート・検索用比較）、dxsyn_lexize（検索）。
  • 呼び出し先:
  • 構造体自体はデータ保持用であり、直接的な関数呼び出しは行わない。

関連データ構造

  • DictSyn: 辞書全体の設定と、Syn 構造体の配列を保持する上位の構造体。

失敗モードと境界条件

  • 辞書ファイルが空の場合、d->len が 0 となり検索は行われない。
  • bsearch で該当する key が見つからない場合、found は NULL を返す。

保守担当者向けメモ

  • src/backend/tsearch/dict_synonym.c にも同名の Syn 構造体が存在するが、本シンボルは contrib/dict_xsyn/dict_xsyn.c 内の定義である。混同しないよう注意すること。
  • value フィールドはパース前の文字列であるため、利用時には dxsyn_lexize 内でのパース処理が必要となる。
