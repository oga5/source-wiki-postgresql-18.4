DecodeInterval

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/interval.c:325

目的

文字列トークンのリストを解析し、時間間隔（interval）の各単位（年、月、日、時、分、秒）に分解して struct tm および小数秒（fsec_t）へ格納する。

シグネチャまたは形状

int DecodeInterval(char **field, int *ftype, int nf, int *dtype, struct tm *tm, fsec_t *fsec);

入力と出力

  • **入力**:
  • field: 解析対象の文字列トークン配列。
  • ftype: 各トークンの型を示す配列。
  • nf: トークンの数。
  • **出力**:
  • dtype: 解析されたデータ型（DTK_DELTA が設定される）。
  • tm: 解析結果を格納する struct tm 構造体。
  • fsec: 解析結果の小数秒（マイクロ秒単位）。
  • **戻り値**: 成功時は 0、エラー時は DTERR_FIELD_OVERFLOW や DTERR_BAD_FORMAT などのエラーコードを返す。

前提条件と不変条件

  • ClearPgTm(tm, fsec) を呼び出し、出力先を初期化してから処理を開始する。
  • トークンリストを末尾から先頭に向かって走査する。
  • -ffast-math コンパイルオプションは使用しないこと（浮動小数点演算の精度維持のため）。

アルゴリズムまたは振る舞い

1. DTK_DELTA 型として初期化し、出力構造体をクリアする。 2. トークンリストを逆順（nf - 1 から 0）に走査する。 3. ftype に応じて以下の処理を行う：

  • DTK_TIME: DecodeTime を呼び出し、時刻情報を分解する。
  • DTK_TZ: タイムゾーン形式（hh:mm等）を判定し、必要に応じて符号を反転させる。
  • DTK_DATE / DTK_NUMBER: 数値として解析し、YEAR-MONTH 形式や小数を含む形式を処理する。

    • 4. 解析した値を tm 構造体の各フィールド（tm_sec, tm_min 等）および fsec に加算する。 5. AdjustFractSeconds 等の補助関数を用いて、単位間の繰り上げ・繰り下げを調整する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/pgtypeslib/interval.c:1032 内で呼び出されている。
  • **呼び出し先**: DecodeTime, ClearPgTm, AdjustFractSeconds, strtoint, strtod, rint。
  • 注: PostgreSQL バックエンド側の datetime.c や timestamp.c にも同名の関数が存在するが、本ページは ECPG ライブラリ内の実装を対象としている。

関連データ構造

  • struct tm: 日付・時刻の各単位を保持する標準構造体。
  • fsec_t: 小数秒を保持する型。
  • interval: ECPG で使用される時間間隔の内部表現構造体。

失敗モードと境界条件

  • strtoint が範囲外の値を検出した場合: DTERR_FIELD_OVERFLOW を返す。
  • 文字列フォーマットが不正な場合: DTERR_BAD_FORMAT を返す。
  • 月単位の解析において、月数が MONTHS_PER_YEAR（12）以上の場合: DTERR_FIELD_OVERFLOW を返す。

保守担当者向けメモ

  • 本関数は PostgreSQL バックエンドのロジックを移植したものである。
  • struct pg_tm ではなく struct tm を使用している点に注意すること。
  • IntervalStyle は INTSTYLE_POSTGRES_VERBOSE にハードコードされている。
