ExplainOneUtility

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:389

目的

EXPLAIN コマンドにおいて、クエリ以外のユーティリティ文（CREATE TABLE AS、DECLARE CURSOR、EXECUTE、NOTIFY 等）の実行計画や情報を生成・出力する。

シグネチャまたは形状

void ExplainOneUtility(Node *utilityStmt, IntoClause *into, ExplainState *es, ParseState *pstate, ParamListInfo params)

入力と出力

  • **入力**:
  • utilityStmt: 処理対象のユーティリティ文のノード。
  • into: CREATE TABLE AS 等で使用される IntoClause。
  • es: 出力状態を保持する ExplainState。
  • pstate: パーサー状態。
  • params: クエリ実行時のパラメータリスト。
  • **出力**: なし（ExplainState のバッファに結果を蓄積する）。

前提条件と不変条件

  • utilityStmt が NULL の場合は即座にリターンする。
  • CREATE TABLE AS および DECLARE CURSOR の場合、内部のクエリを書き換えた後のリスト長は 1 である必要がある（Assert による検証）。

アルゴリズムまたは振る舞い

1. **CREATE TABLE AS**: 対象リレーションの存在を確認し、存在する場合は ExplainDummyGroup で警告を出力して終了する。存在しない場合はクエリをコピー・書き換えし、ExplainOneQuery を呼び出す。 2. **DECLARE CURSOR**: クエリをコピー・書き換えし、ExplainOneQuery を呼び出す。ANALYZE オプションがある場合はクエリが実際に実行される。 3. **EXECUTE**: ExplainExecuteQuery に処理を委譲する。 4. **NOTIFY**: ExplainState のフォーマットに応じて "NOTIFY" という文字列またはグループを出力する。 5. **その他**: 計画構造を持たないユーティリティ文として、テキスト形式ならメッセージを出力し、それ以外なら ExplainDummyGroup を呼び出す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c (クエリ実行フロー内)
  • src/backend/commands/prepare.c (EXECUTE 文の処理)
  • **呼び出し先**:
  • ExplainOneQuery
  • ExplainExecuteQuery
  • ExplainDummyGroup
  • QueryRewrite
  • JumbleQuery
  • post_parse_analyze_hook (フックが設定されている場合)

関連データ構造

  • ExplainState: 出力形式やバッファを管理する構造体。
  • Node: ユーティリティ文の基底型。
  • Query: 書き換え対象のクエリ構造体。

失敗モードと境界条件

  • CREATE TABLE AS において、対象オブジェクトが既に存在する場合、実行計画の生成を回避し ExplainDummyGroup で終了する。
  • 未対応のユーティリティ文が渡された場合、計画構造を持たない旨のメッセージを出力する。

保守担当者向けメモ

  • ANALYZE オプション付きで DECLARE CURSOR を実行すると、カーソルは作成されないがクエリ自体は実行される点に注意すること。
  • 新しいユーティリティ文のサポートを追加する場合は、IsA マクロを用いた条件分岐にケースを追加する。
