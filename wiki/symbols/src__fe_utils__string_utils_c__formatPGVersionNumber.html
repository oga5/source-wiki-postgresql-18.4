formatPGVersionNumber

定義

  • 種別: function
  • 定義箇所: src/fe_utils/string_utils.c:312

目的

PostgreSQLの整数形式のバージョン番号を、人間が読み取り可能な文字列形式（メジャーバージョンおよびマイナーバージョン）に変換する。

シグネチャまたは形状

char *formatPGVersionNumber(int version_number, bool include_minor, char *buf, size_t buflen);

入力と出力

  • version_number: 変換対象の整数値（例: 170001）。
  • include_minor: マイナーバージョンを含めるかどうかのフラグ。
  • buf: 結果を格納する文字バッファ。
  • buflen: buf のサイズ。
  • 戻り値: buf へのポインタを返す。

前提条件と不変条件

  • version_number が 100,000 以上の場合、新しい2部構成（メジャー.マイナー）の形式として処理される。
  • version_number が 100,000 未満の場合、古い3部構成（メジャー.マイナー.パッチ）の形式として処理される。
  • buf は snprintf が書き込み可能な十分なサイズを持っている必要がある。

アルゴリズムまたは振る舞い

1. version_number >= 100000 の場合:

  • include_minor が真なら major.minor（version/10000.version%10000）を生成。
  • 偽なら major（version/10000）のみを生成。

    • 2. version_number < 100000 の場合:

  • include_minor が真なら major.minor.patch（version/10000.(version/100)%100.version%100）を生成。
  • 偽なら major.minor（version/10000.(version/100)%100）を生成。

    • 3. 生成した文字列を buf に書き込み、buf を返す。

呼び出し元と呼び出し先

  • 呼び出し元: pgbench.c、psql/command.c、psql/describe.c など、複数のフロントエンドツールから参照されている。
  • 呼び出し先: 標準ライブラリの snprintf を使用。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • buflen が不足している場合、snprintf の仕様に従い文字列が切り詰められる可能性がある。
  • 負の version_number に対する挙動は定義されていない。

保守担当者向けメモ

  • バージョン番号の計算ロジックはPostgreSQLの歴史的なバージョン体系に依存しているため、将来的なバージョン体系の変更時にはロジックの更新が必要となる可能性がある。
