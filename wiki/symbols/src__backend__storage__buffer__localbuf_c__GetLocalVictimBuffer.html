GetLocalVictimBuffer

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/localbuf.c:222

目的

一時テーブル用ローカルバッファが不足した際に、クロックスイープアルゴリズムを用いて置換対象となるバッファを選定し、再利用可能な状態にする。

シグネチャまたは形状

Buffer GetLocalVictimBuffer(void)

入力と出力

  • **入力**: なし
  • **出力**: 再利用可能になったバッファの識別子（Buffer 型）を返す。

前提条件と不変条件

  • CurrentResourceOwner が有効であること。
  • ローカルバッファの全領域が使用中の場合、エラーが発生する。
  • 選択されたバッファがダーティな場合は、再利用前にディスクへフラッシュされる。

アルゴリズムまたは振る舞い

1. **クロックスイープ**: nextFreeLocalBufId を起点にバッファを走査する。 2. **使用カウントの減衰**: 参照カウントが 0 のバッファに対し、使用カウント（UsageCount）が 1 以上であれば減衰させ、走査を継続する。 3. **バッファの選定**: 使用カウントが 0 のバッファを発見した場合、PinLocalBuffer を実行してピン留めする。 4. **遅延メモリ確保**: バッファのデータ領域が未確保であれば、GetLocalBufferStorage() を呼び出して確保する。 5. **フラッシュと無効化**: 選択したバッファがダーティ（BM_DIRTY）であれば FlushLocalBuffer で書き出し、タグが有効（BM_TAG_VALID）であれば InvalidateLocalBuffer でハッシュテーブルから削除する。

呼び出し元と呼び出し先

  • **呼び出し元**: LocalBufferAlloc 内でバッファの割り当てが必要な際に呼び出される。
  • **呼び出し先**: GetLocalBufferDescriptor, PinLocalBuffer, GetLocalBufferStorage, FlushLocalBuffer, InvalidateLocalBuffer, pgstat_count_io_op 等。

関連データ構造

  • LocalBufferDescriptors: バッファメタデータを保持する配列。
  • LocalRefCount: 各バッファの参照カウント配列。
  • BufferDesc: バッファの状態（state）を管理する構造体。

失敗モードと境界条件

  • **リソース枯渇**: 全てのローカルバッファがピン留めされており、走査を一周しても空きが見つからない場合、ERRCODE_INSUFFICIENT_RESOURCES でエラー終了する。

保守担当者向けメモ

  • 本関数はプロセスローカルなメモリ管理を行うため、共有バッファマネージャのようなロックは不要だが、pg_atomic を用いた状態更新が行われている点に注意すること。
  • 走査中に BM_DIRTY フラグを確認し、必要に応じて同期書き込みを行うため、I/O 負荷が発生する可能性がある。
