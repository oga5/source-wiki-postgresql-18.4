inet_semi_join_sel

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/network_selfuncs.c:807

目的

セミジョイン（EXISTS 句など）におけるネットワーク型（inet/cidr）の選択率を推定します。左辺の単一値が右辺の集合（MCVまたはヒストグラム）のいずれかに適合する確率を算出します。

シグネチャまたは形状

Selectivity inet_semi_join_sel(
    Datum lhs_value,
    bool mcv_exists, Datum *mcv_values, int mcv_nvalues,
    bool hist_exists, Datum *hist_values, int hist_nvalues,
    double hist_weight,
    FmgrInfo *proc, int opr_codenum
)

入力と出力

  • **入力**:
  • lhs_value: 比較対象の左辺値。
  • mcv_exists, mcv_values, mcv_nvalues: 右辺の最頻値（MCV）リストとその存在フラグ。
  • hist_exists, hist_values, hist_nvalues: 右辺のヒストグラムとその存在フラグ。
  • hist_weight: ヒストグラムが占める母集団の割合。
  • proc: 比較演算を実行するための関数情報。
  • opr_codenum: 演算子の種類を示す数値コード。
  • **出力**:
  • Selectivity（double）: 0.0 から 1.0 の範囲で、条件が真となる確率を返します。

前提条件と不変条件

  • mcv_exists が真の場合、mcv_values には有効な Datum 配列が格納されている必要があります。
  • hist_weight が 0 より大きい場合のみ、ヒストグラムを用いた計算が行われます。

アルゴリズムまたは振る舞い

1. **MCVの走査**: mcv_exists が真の場合、mcv_values 内の各要素に対して proc（演算子）を適用します。いずれか一つでも真（TRUE）を返す要素があれば、直ちに 1.0 を返します。 2. **ヒストグラムの計算**: MCVで判定できなかった場合、hist_exists かつ hist_weight > 0 であれば inet_hist_value_sel を呼び出します。

  • 演算子を反転（-opr_codenum）させて lhs_value を右辺として評価します。
  • 算出された選択率に hist_weight を乗じ、Min(1.0, ...) で正規化して返します。

    • 3. **デフォルト**: いずれにも該当しない場合、0.0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/network_selfuncs.c 内の結合選択率推定ロジックから呼び出されます（行489, 521）。
  • **呼び出し先**:
  • FunctionCall2: proc を実行するために使用。
  • inet_hist_value_sel: ヒストグラムに基づく選択率計算に使用。

関連データ構造

  • Datum: PostgreSQLの汎用データ表現。
  • FmgrInfo: 関数マネージャ情報。
  • Selectivity: double 型のエイリアス。

失敗モードと境界条件

  • hist_weight が 0 以下の場合、ヒストグラムの計算はスキップされます。
  • mcv_nvalues や hist_nvalues が 0 の場合、それぞれの計算フェーズは実行されません。
  • 演算子関数がエラーを発生させた場合、呼び出し元に伝播します。

保守担当者向けメモ

  • 本関数はセミジョイン専用のロジックです。inet_opr_codenum で定義された演算子コードの符号反転を利用して、ヒストグラム計算関数との整合性を保っています。
  • 包含関係の判定には inet_hist_value_sel が依存するビット単位のヒューリスティックが関与するため、推定精度を調整する場合は inet_hist_match_divider の挙動を確認してください。
