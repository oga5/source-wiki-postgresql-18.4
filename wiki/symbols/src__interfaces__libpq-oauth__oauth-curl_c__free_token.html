free_token

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq-oauth/oauth-curl.c:201

目的

token 構造体が保持する動的メモリを解放し、リソースリークを防ぐ。

シグネチャまたは形状

static void free_token(struct token *tok);

入力と出力

  • **入力**: tok - 解放対象の token 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • tok は NULL であってはならない（NULL チェックは関数内で行われない）。
  • tok 内の各メンバ（access_token, token_type）は、malloc 等で確保されたメモリ領域を指しているか、あるいは NULL である必要がある。

アルゴリズムまたは振る舞い

1. tok->access_token を解放する。 2. tok->token_type を解放する。 3. tok->err を引数として free_token_error を呼び出し、エラー関連のリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq-oauth/oauth-curl.c:2606 での呼び出しが確認されている。
  • **呼び出し先**: free_token_error を呼び出す。

関連データ構造

  • token: OAuth トークン情報を保持する構造体。
  • token_error: トークン取得時のエラー情報を保持する構造体。

失敗モードと境界条件

  • tok が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • tok 内のメンバが初期化されていない（不定なポインタである）場合、free 呼び出しにより未定義動作を引き起こす。

保守担当者向けメモ

  • 本関数は token 構造体自体のメモリは解放しない。構造体自体を解放する必要がある場合は、呼び出し側で別途 free(tok) を行う必要がある。
