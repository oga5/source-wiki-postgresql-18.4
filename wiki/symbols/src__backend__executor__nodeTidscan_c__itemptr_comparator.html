itemptr_comparator

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidscan.c:282

目的

ItemPointerData（TID）の配列をソートするために使用される比較関数です。ブロック番号およびオフセット番号に基づいた昇順ソートを提供します。

シグネチャまたは形状

static int itemptr_comparator(const void *a, const void *b)

入力と出力

  • **入力**: 比較対象となる2つの ItemPointerData へのポインタ（const void *）。
  • **出力**:
  • a < b の場合: -1
  • a > b の場合: 1
  • a == b の場合: 0

前提条件と不変条件

  • 入力ポインタ a および b は、有効な ItemPointerData 構造体を指している必要があります。
  • 比較はまずブロック番号 (BlockNumber) で行われ、一致する場合のみオフセット番号 (OffsetNumber) で比較されます。

アルゴリズムまたは振る舞い

1. ItemPointerGetBlockNumber を用いて、両方のポインタからブロック番号を取得し比較します。 2. ブロック番号が異なる場合、その大小関係を即座に返します。 3. ブロック番号が等しい場合、ItemPointerGetOffsetNumber を用いてオフセット番号を取得し比較します。 4. オフセット番号の大小関係を返します。両者が等しい場合は 0 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeTidscan.c 内の TidListEval 等で、TID リストのソート処理（qsort 等）の引数として参照されています。
  • **呼び出し先**: ItemPointerGetBlockNumber および ItemPointerGetOffsetNumber マクロを呼び出します。

関連データ構造

  • ItemPointerData: 物理的なタプル位置（ブロック番号とオフセット番号）を保持する構造体。

失敗モードと境界条件

  • 入力ポインタが NULL の場合の挙動は定義されていません。
  • ブロック番号およびオフセット番号が最大値である場合でも、整数比較により正しく順序付けられます。

保守担当者向けメモ

  • 本関数は qsort 等の標準的な比較関数インターフェースに準拠しています。
  • TID リストのソートは、物理的なブロック順序に近いアクセスを促し、I/O 効率を向上させるために不可欠です。変更を加える際は、ソート順序がスキャン効率に与える影響を考慮してください。
