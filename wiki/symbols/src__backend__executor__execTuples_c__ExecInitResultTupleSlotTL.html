ExecInitResultTupleSlotTL

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execTuples.c:1987

目的

プランノードの実行結果を保持するためのタプルスロットを初期化し、関連する型情報を設定する。

シグネチャまたは形状

void ExecInitResultTupleSlotTL(PlanState *planstate, const TupleTableSlotOps *tts_ops)

入力と出力

  • **入力**:
  • planstate: 初期化対象のプランノード状態（PlanState）。
  • tts_ops: スロットの振る舞いを定義する関数テーブル（TupleTableSlotOps）。
  • **出力**: なし（void）。副作用として planstate 内のタプルスロットおよび型情報が初期化される。

前提条件と不変条件

  • planstate は有効なメモリ領域に存在すること。
  • tts_ops は、使用するタプル形式（TTSOpsVirtual, TTSOpsMinimalTuple 等）に対応する有効な関数テーブルであること。

アルゴリズムまたは振る舞い

1. ExecInitResultTypeTL(planstate) を呼び出し、プランノードの出力型情報を初期化する。 2. ExecInitResultSlot(planstate, tts_ops) を呼び出し、指定された tts_ops を用いてプランノードの結果用タプルスロットを割り当て・初期化する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • nodeAgg.c, nodeAppend.c, nodeCustom.c, nodeGroup.c, nodeHash.c, nodeHashjoin.c, nodeIncrementalSort.c, nodeMaterial.c 等の各プランノード初期化関数。
  • **呼び出し先**:
  • ExecInitResultTypeTL
  • ExecInitResultSlot

関連データ構造

  • PlanState: プランノードの実行状態を保持する構造体。
  • TupleTableSlotOps: タプルスロットの操作（格納、抽出、クリア等）を抽象化する関数テーブル。

失敗モードと境界条件

  • メモリ不足時には、内部で呼び出される MakeTupleTableSlot 等によりエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は、プランノードの初期化プロセスにおいて「型定義」と「スロット確保」を連続して行うためのラッパーである。
  • 物理的なタプル形式（TTSOpsHeap）や仮想タプル（TTSOpsVirtual）の切り替えは、呼び出し元で渡される tts_ops によって制御される。
