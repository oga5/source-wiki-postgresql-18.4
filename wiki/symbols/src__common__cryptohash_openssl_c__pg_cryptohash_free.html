pg_cryptohash_free

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash_openssl.c:325

目的

暗号学的ハッシュ計算に使用したコンテキストを破棄し、関連するリソースを解放する。

シグネチャまたは形状

void pg_cryptohash_free(pg_cryptohash_ctx *ctx)

入力と出力

  • **入力**: pg_cryptohash_ctx *ctx（解放対象のハッシュコンテキストへのポインタ）
  • **出力**: なし（void）

前提条件と不変条件

  • ctx が NULL の場合、関数は何もしない。
  • バックエンド環境（FRONTEND が未定義）では、ctx->resowner が設定されている場合、ResourceOwnerForgetCryptoHash を通じてリソース所有者から登録を解除する必要がある。

アルゴリズムまたは振る舞い

1. ctx が NULL であれば即座に終了する。 2. OpenSSLの EVP_MD_CTX_destroy を呼び出し、内部の EVP_MD_CTX を破棄する。 3. バックエンド環境の場合、ResourceOwnerForgetCryptoHash を呼び出してリソース管理を解除する。 4. explicit_bzero を使用して ctx 構造体全体をゼロクリアし、機密情報をメモリから消去する。 5. FREE(ctx) を呼び出し、コンテキストメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/libpq/auth-scram.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/checksum_helper.c
  • **呼び出し先**:
  • EVP_MD_CTX_destroy (OpenSSL)
  • ResourceOwnerForgetCryptoHash (バックエンドのみ)
  • explicit_bzero
  • FREE

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態を保持する構造体。

失敗モードと境界条件

  • ctx が NULL の場合、安全に無視される。
  • explicit_bzero によるメモリ消去は、機密情報の漏洩を防ぐための必須ステップとして実行される。

保守担当者向けメモ

  • 本関数は pg_cryptohash_create で確保されたリソースの対となる解放処理である。
  • バックエンド環境とフロントエンド環境でメモリ管理の責務が異なるため、#ifndef FRONTEND による条件分岐が重要である。
