parse_compress_algorithm

定義

  • 種別: function
  • 定義箇所: src/common/compression.c:48

目的

圧縮アルゴリズム名を表す文字列を、内部的な列挙型 pg_compress_algorithm に変換する。

シグネチャまたは形状

bool parse_compress_algorithm(char *name, pg_compress_algorithm *algorithm);

入力と出力

  • **入力**:
  • name: 比較対象のアルゴリズム名文字列（例: "gzip", "zstd"）。
  • algorithm: 変換結果を格納する pg_compress_algorithm 型へのポインタ。
  • **出力**:
  • 成功時に true を返し、algorithm に対応する値を設定する。
  • 不明なアルゴリズム名の場合は false を返す。

前提条件と不変条件

  • name はヌル終端文字列である必要がある。
  • algorithm は有効なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. name を以下の文字列と比較する。

  • "none" -> PG_COMPRESSION_NONE
  • "gzip" -> PG_COMPRESSION_GZIP
  • "lz4" -> PG_COMPRESSION_LZ4
  • "zstd" -> PG_COMPRESSION_ZSTD

    • 2. 一致した場合は algorithm に値を代入し true を返す。 3. いずれにも一致しない場合は false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c:904
  • src/bin/pg_basebackup/pg_basebackup.c:2655
  • src/bin/pg_basebackup/pg_receivewal.c:800
  • src/bin/pg_dump/pg_dump.c:938
  • **呼び出し先**:
  • 標準ライブラリ関数 strcmp を使用。

関連データ構造

  • pg_compress_algorithm: 圧縮アルゴリズムを識別する列挙型（src/include/common/compression.h で定義）。

失敗モードと境界条件

  • name が上記リストに含まれない文字列の場合、false を返し、algorithm の値は変更されない。

保守担当者向けメモ

  • 新しい圧縮アルゴリズムをサポートする場合は、この関数に条件分岐を追加する必要がある。
  • 大文字・小文字の区別は strcmp により厳密に行われるため、呼び出し側で正規化が必要な場合がある。
