member_copy_control_data

定義

  • 種別: function
  • 定義箇所: src/bin/pg_verifybackup/astreamer_verify.c:333

目的

バックアップストリームから読み込まれた制御ファイル（pg_control）のデータを、検証用バッファに蓄積する。

シグネチャまたは形状

static void member_copy_control_data(astreamer *streamer, astreamer_member *member, const char *data, int len)

入力と出力

  • streamer: 現在のストリーマーコンテキスト。
  • member: 現在処理中のアーカイブメンバー情報。
  • data: 読み込まれたデータブロックのポインタ。
  • len: 読み込まれたデータのバイト長。
  • 出力: なし（副作用として astreamer_verify 構造体内のバッファおよびカウンタを更新）。

前提条件と不変条件

  • mystreamer->verify_control_data が真であること（制御ファイル処理中であること）。
  • ControlFileData 構造体のサイズを超えるデータはバッファにコピーされないが、総バイト数は正確にカウントされる。

アルゴリズムまたは振る舞い

1. streamer を astreamer_verify 型にキャストする。 2. verify_control_data フラグをアサートで確認する。 3. 現在のバッファ蓄積量（control_file_bytes）が sizeof(ControlFileData) 未満である場合、バッファの残り容量分だけ data を mystreamer->control_file へ memcpy する。 4. 実際にバッファへコピーしたかに関わらず、読み込んだデータ長 len を control_file_bytes に加算し、総受信量を更新する。

呼び出し元と呼び出し先

  • 呼び出し元: astreamer_verify_content（src/bin/pg_verifybackup/astreamer_verify.c:111 付近）。
  • 呼び出し先: memcpy（標準ライブラリ）。

関連データ構造

  • astreamer_verify: 検証状態を保持する構造体。
  • ControlFileData: 制御ファイルの構造体定義。

失敗モードと境界条件

  • control_file_bytes が sizeof(ControlFileData) を超える場合、バッファへの書き込みは停止するが、カウンタは増加し続ける。これにより、制御ファイルのサイズが期待値と異なる場合でも、後続の検証フェーズで検知可能にする。

保守担当者向けメモ

  • 本関数は、制御ファイルのデータがストリーム経由で断片的に送られてくることを前提としている。
  • バッファサイズは sizeof(ControlFileData) に固定されているため、将来的に制御ファイルの構造体が変更された場合は注意が必要。
