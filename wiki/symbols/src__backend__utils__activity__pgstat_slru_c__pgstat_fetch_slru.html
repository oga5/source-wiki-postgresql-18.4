pgstat_fetch_slru

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_slru.c:104

目的

SLRU（Simple Least Recently Used）キャッシュに関する現在の統計情報のスナップショットを取得し、呼び出し元へ提供します。

シグネチャまたは形状

PgStat_SLRUStats *pgstat_fetch_slru(void)

入力と出力

  • **入力**: なし
  • **出力**: PgStat_SLRUStats 構造体へのポインタ。共有メモリから取得された最新のSLRU統計スナップショットを指します。

前提条件と不変条件

  • 呼び出し前に、統計サブシステムが初期化されている必要があります。
  • 返されるポインタは pgStatLocal.snapshot.slru を参照しており、後続の統計取得操作によって内容が更新される可能性があるため、長期的な保持には適しません。

アルゴリズムまたは振る舞い

1. pgstat_snapshot_fixed(PGSTAT_KIND_SLRU) を呼び出し、共有メモリ上のSLRU統計をローカルのスナップショット領域へ同期させます。 2. 同期完了後、pgStatLocal.snapshot.slru に格納されている統計データへのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/pgstatfuncs.c:1725 (SQL関数経由での統計取得)
  • **呼び出し先**:
  • pgstat_snapshot_fixed (統計サブシステム内部関数)

関連データ構造

  • PgStat_SLRUStats: SLRUの各操作（ヒット、読み込み、書き込み等）のカウンタを保持する構造体。
  • pgStatLocal: 各バックエンドが保持する統計情報のローカルキャッシュ構造体。

失敗モードと境界条件

  • 共有メモリへのアクセス時にロック競合が発生した場合、pgstat_snapshot_fixed の内部実装に従い待機が発生する可能性があります。
  • 統計データがまだ一度も収集されていない場合、初期化済みのゼロ値を含む構造体が返されます。

保守担当者向けメモ

  • 本関数はSQLインターフェースから統計を参照する際の標準的なエントリポイントです。
  • 統計の整合性を保証するため、必ず pgstat_snapshot_fixed を経由して最新のスナップショットを更新してから参照する設計となっています。
