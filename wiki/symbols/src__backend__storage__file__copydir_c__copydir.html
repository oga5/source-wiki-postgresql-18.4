copydir

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/copydir.c:47

目的

指定されたソースディレクトリの内容をターゲットディレクトリへ再帰的に複製し、ファイルシステムレベルでのデータ永続化を保証する。

シグネチャまたは形状

void copydir(const char *fromdir, const char *todir, bool recurse)

入力と出力

  • fromdir (const char*): コピー元のディレクトリパス。
  • todir (const char*): コピー先のディレクトリパス。
  • recurse (bool): true の場合、サブディレクトリも再帰的にコピーする。
  • 戻り値: なし。失敗時は ereport(ERROR, ...) により処理を中断する。

前提条件と不変条件

  • todir は存在しないか、空である必要がある（MakePGDirectory が失敗するため）。
  • enableFsync が有効な場合、コピー完了後にすべてのファイルとディレクトリに対して fsync が実行される。
  • 処理中にシグナルを受信した場合、CHECK_FOR_INTERRUPTS() により中断される。

アルゴリズムまたは振る舞い

1. **ディレクトリ作成**: MakePGDirectory を使用して todir を作成する。 2. **走査とコピー**: AllocateDir/ReadDir で fromdir を走査する。

  • . および .. はスキップする。
  • サブディレクトリの場合、recurse が true なら再帰的に copydir を呼び出す。
  • 通常ファイルの場合、file_copy_method に応じて clone_file または copy_file を呼び出す。

    • 3. **永続化**: enableFsync が有効な場合、todir 内の全ファイルを再度走査し、fsync_fname で同期する。 4. **ディレクトリ同期**: 最後に todir 自体に対して fsync_fname(..., true) を実行し、メタデータを確定させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dbcommands.c 内のデータベースコピー処理などで呼び出される。
  • **呼び出し先**:
  • MakePGDirectory, AllocateDir, ReadDir, FreeDir (ディレクトリ操作)
  • get_dirent_type (ファイル種別判定)
  • clone_file / copy_file (ファイルコピー)
  • fsync_fname (永続化)

関連データ構造

  • DIR (ディレクトリストリーム)
  • struct dirent (ディレクトリ内のエントリ情報)
  • PGFileType (ファイル種別定数)

失敗モードと境界条件

  • MakePGDirectory 失敗時: ディレクトリ作成権限不足やパスの競合によりエラー終了。
  • ReadDir 失敗時: 読み取り権限不足等によりエラー終了。
  • fsync 無効時: enableFsync が false の場合、コピー後の同期処理をスキップして終了する。

保守担当者向けメモ

  • fsync の実行は、クラッシュリカバリの整合性を維持するために不可欠である。
  • clone_file は OS の機能（Reflink 等）に依存するため、プラットフォームごとの挙動に注意すること。
  • 長時間の処理となる可能性があるため、CHECK_FOR_INTERRUPTS() の位置は適切に維持する必要がある。
