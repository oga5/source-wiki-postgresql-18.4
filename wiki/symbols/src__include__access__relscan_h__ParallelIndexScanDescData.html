ParallelIndexScanDescData

定義

  • 種別: struct
  • 定義箇所: src/include/access/relscan.h:195

目的

並列インデックススキャンにおいて、複数のバックエンド（ワーカー）間で共有されるスキャン状態を保持するための構造体です。

シグネチャまたは形状

typedef struct ParallelIndexScanDescData
{
    RelFileLocator ps_locator;        /* スキャン対象の物理テーブル */
    RelFileLocator ps_indexlocator;   /* スキャン対象の物理インデックス */
    Size           ps_offset_ins;     /* SharedIndexScanInstrumentation へのオフセット */
    Size           ps_offset_am;      /* AM固有構造体へのオフセット */
    char           ps_snapshot_data[FLEXIBLE_ARRAY_MEMBER]; /* スナップショットデータ */
} ParallelIndexScanDescData;

入力と出力

  • **入力**: 並列クエリ実行時にエグゼキュータによって初期化され、共有メモリ上に配置されます。
  • **出力**: 各ワーカーは本構造体を参照し、スキャン対象の識別子やスナップショット情報を取得します。

前提条件と不変条件

  • 共有メモリ上に配置されるため、メンバの配置やサイズは並列環境下で一貫性を保つ必要があります。
  • ps_snapshot_data は可変長メンバであり、構造体の末尾に配置されます。

アルゴリズムまたは振る舞い

1. **初期化**: エグゼキュータが共有メモリ領域を確保し、ps_locator および ps_indexlocator を設定します。 2. **オフセット管理**: ps_offset_ins および ps_offset_am を用いて、共有メモリ上の関連データ（計測情報やAM固有の制御データ）を特定します。 3. **スナップショットの保持**: ps_snapshot_data にスキャンに必要なスナップショット情報を格納し、全ワーカーで共有します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/index/indexam.c において、並列スキャンのセットアップ時に本構造体のサイズ計算やオフセット算出が行われています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。各アクセスメソッド（AM）がこの構造体を参照してスキャンを制御します。

関連データ構造

  • RelFileLocator: テーブルおよびインデックスの物理的な位置を特定するために使用されます。
  • ParallelIndexScanDesc: 本構造体へのポインタ型（src/include/access/genam.h で定義）。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、並列スキャンの初期化が失敗します。
  • ps_snapshot_data のサイズが不適切な場合、メモリ破壊や不正なアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は共有メモリ上に配置されるため、メンバの追加や変更を行う際は、並列処理への影響およびアライメントを十分に考慮してください。
  • ps_snapshot_data は FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には offsetof を使用することが推奨されます。
