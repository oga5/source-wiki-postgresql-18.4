pg_hmac_free

定義

  • 種別: function
  • 定義箇所: src/common/hmac.c:288

目的

HMAC 計算に使用した pg_hmac_ctx コンテキストおよび関連する内部リソースを解放し、メモリを安全に消去します。

シグネチャまたは形状

void pg_hmac_free(pg_hmac_ctx *ctx)

入力と出力

  • **入力**: ctx - 解放対象の pg_hmac_ctx 構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • ctx が NULL の場合、本関数は何も行わず正常に終了します。
  • ctx が有効なポインタである場合、内部のハッシュコンテキストおよび構造体全体が解放・消去されます。

アルゴリズムまたは振る舞い

1. ctx が NULL かどうかを確認し、該当する場合は早期リターンします。 2. 内部保持されているハッシュコンテキスト ctx->hash に対して pg_cryptohash_free を呼び出し、関連リソースを解放します。 3. explicit_bzero を使用して pg_hmac_ctx 構造体全体をゼロクリアし、機密情報（鍵など）の残留を防ぎます。 4. FREE(ctx) を呼び出し、コンテキスト構造体自体のメモリを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c
  • src/common/scram-common.c
  • **呼び出し先**:
  • pg_cryptohash_free
  • explicit_bzero
  • FREE (マクロ)

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する不透明な構造体。

失敗モードと境界条件

  • ctx が NULL の場合は安全に処理されます。
  • 既に解放済みのポインタを渡した場合の動作は未定義です。

保守担当者向けメモ

  • 本関数はフロントエンド環境において、メモリリークを防ぐために呼び出し側で必ず呼び出す必要があります。
  • explicit_bzero を使用しているため、セキュリティ上の観点からメモリ上の鍵データは確実に消去されます。
