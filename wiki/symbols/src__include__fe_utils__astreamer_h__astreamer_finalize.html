astreamer_finalize

定義

  • 種別: function
  • 定義箇所: src/include/fe_utils/astreamer.h:146

目的

ストリーム処理パイプラインの各ステージにおける終了処理（クリーンアップ）を実行します。

シグネチャまたは形状

static inline void astreamer_finalize(astreamer *streamer)

入力と出力

  • **入力**: astreamer *streamer - 終了処理対象となるストリーム構造体へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • streamer は NULL であってはならない（Assert により検証）。
  • streamer->bbs_ops が有効であり、かつ finalize 関数ポインタが実装されている必要がある。

アルゴリズムまたは振る舞い

1. streamer が NULL でないことを確認する。 2. streamer に関連付けられた bbs_ops（操作関数群）内の finalize コールバックを呼び出し、当該ステージの終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/astreamer_inject.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • src/fe_utils/astreamer_gzip.c
  • src/fe_utils/astreamer_lz4.c
  • src/fe_utils/astreamer_tar.c
  • src/fe_utils/astreamer_zstd.c
  • **呼び出し先**:
  • streamer->bbs_ops->finalize（各ストリーム実装固有の終了処理関数）。

関連データ構造

  • astreamer: ストリーム処理の基底構造体。
  • astreamer_ops: finalize コールバックを保持する操作関数群の構造体。

失敗モードと境界条件

  • streamer が NULL の場合、アサーションエラーが発生する。
  • bbs_ops または finalize 関数ポインタが初期化されていない場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数はパイプラインの終了処理をトリガーする。全データの供給が完了した後に呼び出すこと。
  • メモリ解放を行う astreamer_free とは役割が異なるため、終了処理後に必要に応じて astreamer_free を呼び出す必要がある。
