test_empty

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_radixtree/test_radixtree.c:120

目的

適応型基数木（ART）が空の状態における、基本的な検索、削除、およびイテレーション操作の正当性を検証する。

シグネチャまたは形状

static void test_empty(void)

入力と出力

  • 入力: なし
  • 出力: なし（検証失敗時は EXPECT_* マクロにより elog(ERROR) を発行し、処理を中断する）

前提条件と不変条件

  • TEST_SHARED_RT マクロの定義有無に応じて、共有メモリ（DSA）またはメモリコンテキスト上に基数木インスタンスが生成されること。
  • テスト終了時に rt_free が呼び出され、メモリが解放されること。

アルゴリズムまたは振る舞い

1. **初期化**: rt_create を呼び出し、空の基数木を生成する。 2. **検索・削除検証**: 任意のキー（0, 1, PG_UINT64_MAX）に対して rt_find を実行し、結果が NULL であることを確認する。また、rt_delete が false を返し、rt_num_entries が 0 であることを確認する。 3. **イテレーション検証**: rt_begin_iterate でイテレータを作成し、rt_iterate_next が直ちに NULL を返すことを確認する。 4. **終了処理**: rt_end_iterate でイテレータを破棄し、rt_free で基数木を解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/test/modules/test_radixtree/test_radixtree.c 内の test_radixtree 関数から呼び出される。
  • 呼び出し先: rt_create, rt_find, rt_delete, rt_num_entries, rt_begin_iterate, rt_iterate_next, rt_end_iterate, rt_free（および TEST_SHARED_RT 有効時は dsa_create 等の DSA API）。

関連データ構造

  • rt_radix_tree: テスト対象の基数木構造体。
  • rt_iter: イテレーション用構造体。

失敗モードと境界条件

  • EXPECT_* マクロの条件を満たさない場合、elog(ERROR) によりトランザクションがアボートされる。
  • 共有メモリモード時、dsa_create や LWLock の確保に失敗した場合はシステムエラーとなる。

保守担当者向けメモ

  • 本関数は test_radixtree の初期化フェーズで実行される。
  • TEST_SHARED_RT マクロの切り替えにより、メモリ管理の挙動が大きく異なるため、修正時は両方のパスを確認すること。
