BarrierAttach

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/barrier.c:235

目的

動的バリアに対してプロセスを新たに参加させ、現在のバリアフェーズを取得します。

シグネチャまたは形状

int BarrierAttach(Barrier *barrier);

入力と出力

  • **入力**: barrier (対象となる Barrier 構造体へのポインタ)
  • **出力**: 現在のバリアフェーズ（int 型）を返します。

前提条件と不変条件

  • barrier->static_party が false であること（静的バリアに対しては使用不可）。
  • 呼び出し側は、この関数を呼び出すことでバリアの参加者として登録されます。

アルゴリズムまたは振る舞い

1. barrier->mutex を使用してスピンロックを獲得します。 2. barrier->participants（参加者数）をインクリメントします。 3. 現在の barrier->phase を読み取ります。 4. barrier->mutex を解放します。 5. 読み取った phase を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHash.c (複数箇所)
  • src/backend/executor/nodeHashjoin.c
  • **呼び出し先**:
  • SpinLockAcquire
  • SpinLockRelease

関連データ構造

  • Barrier: 同期状態を保持する構造体。mutex、participants、phase を含みます。

失敗モードと境界条件

  • 静的バリア（static_party が設定されているもの）に対して呼び出した場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は動的バリア専用です。参加・離脱が頻繁に発生するため、呼び出し側は戻り値の phase を確認し、現在のフェーズに応じた処理を行う必要があります。
