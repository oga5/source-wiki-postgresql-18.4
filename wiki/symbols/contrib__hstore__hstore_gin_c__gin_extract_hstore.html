gin_extract_hstore

定義

  • 種別: function
  • 定義箇所: contrib/hstore/hstore_gin.c:43

目的

hstore 型のデータを GIN 索引に格納可能な形式（キーと値のペア）に変換し、索引付けのためのエントリ配列を生成します。

シグネチャまたは形状

Datum gin_extract_hstore(PG_FUNCTION_ARGS)
  • 引数: PG_FUNCTION_ARGS (内部的に HStore ポインタと int32 ポインタを抽出)
  • 戻り値: Datum (変換されたエントリ配列へのポインタ)

入力と出力

  • **入力**:
  • PG_GETARG_HSTORE_P(0): 索引対象の hstore データ。
  • PG_GETARG_POINTER(1): エントリ数を格納するための int32 へのポインタ。
  • **出力**:
  • *nentries: 抽出されたエントリの総数（キー数 × 2）。
  • 戻り値: Datum 配列。各キーと値が makeitem によってフラグ付きの text 型に変換されて格納されます。

前提条件と不変条件

  • hstore データが有効であること。
  • エントリ配列のサイズは常に 2 * count（キーと値のペア数）となります。
  • 抽出された各エントリには、KEYFLAG、VALFLAG、または NULLFLAG のいずれかの識別フラグが付与されます。

アルゴリズムまたは振る舞い

1. 入力 hstore からキーと値のペア数 count を取得し、2 * count 個分の Datum 配列をメモリ確保します。 2. hstore 内の各要素をループで走査します。 3. 各キーに対して makeitem を呼び出し、KEYFLAG を付与して配列に格納します。 4. 各値に対して、値が NULL の場合は NULLFLAG を、それ以外の場合は VALFLAG を付与して makeitem を呼び出し、配列に格納します。 5. 最終的に生成されたエントリ配列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_gin.c 内の PG_FUNCTION_INFO_V1 マクロによる登録。
  • gin_extract_hstore_query 関数内から、クエリが hstore 型である場合に DirectFunctionCall2 を介して呼び出されます。
  • **呼び出し先**:
  • makeitem (エントリのフラグ付き変換に使用)。
  • palloc (メモリ確保)。

関連データ構造

  • HStore: 入力される hstore 型の内部構造。
  • HEntry: hstore 内の各要素へのアクセスに使用される構造体。
  • KEYFLAG, VALFLAG, NULLFLAG: インデックス項目を識別するためのプレフィックス定数。

失敗モードと境界条件

  • hstore が空の場合、count が 0 となり、エントリ配列は作成されず nentries は 0 となります。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は PostgreSQL 9.1 以前からのディスクフォーマット互換性を維持するため、エントリに特定のフラグを付与しています。変更時はインデックスの再構築が必要になる可能性がある点に注意してください。
  • recheck の要否については、本関数ではなく gin_consistent_hstore が判定を担います。
