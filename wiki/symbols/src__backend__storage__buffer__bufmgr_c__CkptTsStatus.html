CkptTsStatus

定義

  • **種別**: struct
  • **定義箇所**: src/backend/storage/buffer/bufmgr.c:106

目的

チェックポイント処理中における、テーブルスペースごとの進捗状況と統計情報を保持するための構造体です。

シグネチャまたは形状

typedef struct CkptTsStatus
{
    Oid         tsId;             /* テーブルスペースのOID */
    float8      progress;         /* 現在の進捗値 */
    float8      progress_slice;   /* 1ページあたりの進捗増分 */
    int         num_to_scan;      /* チェックポイント対象の総ページ数 */
    int         num_scanned;      /* 処理済みのページ数 */
    int         index;            /* CkptBufferIds 内の現在のオフセット */
} CkptTsStatus;

入力と出力

本構造体は関数への引数や戻り値として直接使用されるのではなく、チェックポイント処理の実行中にメモリ（palloc 等）へ確保され、テーブルスペースごとの状態管理に使用されます。

前提条件と不変条件

  • progress は 0 から num_to_scan に相当する範囲で管理されます。
  • progress_slice は、テーブルスペース間での進捗比較を可能にするために使用されます。

アルゴリズムまたは振る舞い

チェックポイント処理において、各テーブルスペース内のページをスキャンする際、処理済みのページ数（num_scanned）を更新し、progress を progress_slice ずつ加算することで、全体のチェックポイント進捗を追跡します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/bufmgr.c 内のチェックポイント関連関数（3352, 3441, 3460, 3463, 3465, 3510, 3530 行付近）で、動的なメモリ確保および状態更新が行われています。
  • **呼び出し先**: 構造体自体はデータ保持用であり、特定の関数を呼び出すことはありません。

関連データ構造

  • CkptBufferIds: チェックポイント対象のバッファIDを保持する配列。本構造体の index メンバは、この配列を参照するために使用されます。

失敗モードと境界条件

  • メモリ不足時: palloc または repalloc が失敗した場合、PostgreSQL の標準的なエラーハンドリングによりアボートします。
  • num_to_scan が 0 の場合、progress_slice の計算においてゼロ除算が発生しないよう注意が必要です。

保守担当者向けメモ

  • 本構造体はチェックポイントの進捗監視に特化しています。進捗計算ロジックを変更する場合は、progress_slice の算出根拠と num_scanned の更新タイミングを整合させる必要があります。
