bbsink_forward_manifest_contents

定義

  • **種別**: function
  • **定義箇所**: src/backend/backup/basebackup_sink.c:88

目的

バックアップマニフェストのコンテンツデータを、現在のシンクからチェーン内の次のシンク（bbs_next）へ転送します。

シグネチャまたは形状

void bbsink_forward_manifest_contents(bbsink *sink, size_t len)

入力と出力

  • **入力**:
  • sink: 現在の bbsink オブジェクトへのポインタ。
  • len: 転送するデータのバイト長。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • sink->bbs_next が NULL であってはならない。
  • 現在のシンクのバッファ（bbs_buffer）とバッファ長（bbs_buffer_length）が、後続のシンクと完全に一致している必要がある。

アルゴリズムまたは振る舞い

1. sink->bbs_next が存在することをアサートで確認します。 2. 現在のシンクと後続のシンク間でバッファおよびバッファ長が共有されていることをアサートで検証します。 3. 後続のシンクの bbsink_manifest_contents メソッドを呼び出し、処理を委譲します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/basebackup_to_shell/basebackup_to_shell.c
  • src/backend/backup/basebackup_server.c
  • src/backend/backup/basebackup_throttle.c
  • **呼び出し先**:
  • bbsink_manifest_contents（sink->bbs_next のメソッド）

関連データ構造

  • bbsink: バックアップデータの転送パイプラインを構成する抽象構造体。

失敗モードと境界条件

  • bbs_next が初期化されていない場合、アサーション失敗によりプロセスが異常終了します。
  • バッファの不整合がある場合、アサーション失敗によりプロセスが異常終了します。

保守担当者向けメモ

  • 本関数は、転送のみを行うシンクの実装においてコールバックとして直接割り当て可能です。
  • バッファの整合性が呼び出し元で保証されていることが前提となるため、カスタムシンクを実装する際は注意してください。
