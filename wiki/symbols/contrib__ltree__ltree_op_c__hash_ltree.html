hash_ltree

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltree_op.c:137

目的

ltree 型のデータに対してハッシュ値を生成し、ハッシュインデックスでの利用を可能にする。

シグネチャまたは形状

Datum hash_ltree(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを通じて渡される ltree 型のポインタ。
  • **出力**: uint32 型のハッシュ値。PG_RETURN_UINT32 を介して返される。

前提条件と不変条件

  • 入力された ltree は有効な構造体であること。
  • PG_GETARG_LTREE_P(0) により、第1引数から ltree 構造体を取得する。
  • 処理終了後、PG_FREE_IF_COPY により必要に応じてメモリが解放される。

アルゴリズムまたは振る舞い

1. 初期値 result = 1 を設定する。 2. ltree 内の各レベル（ltree_level）を LTREE_FIRST から順次走査する。 3. 各レベルのラベル名（al->name）に対して hash_any を実行し、ハッシュ値 levelHash を取得する。 4. result = (result << 5) - result + levelHash の計算式を用いて、各レベルのハッシュ値を累積的に結合する。 5. 全レベルの走査が完了するまで LEVEL_NEXT で次のレベルへ移動する。 6. 最終的な result を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数登録マクロ PG_FUNCTION_INFO_V1(hash_ltree) により SQL インターフェースから呼び出される。
  • **呼び出し先**: hash_any（ハッシュ計算用ユーティリティ）。

関連データ構造

  • ltree: 階層構造を持つラベルパスを保持するデータ構造。
  • ltree_level: ltree 内の各階層のラベル情報。

失敗モードと境界条件

  • ltree が空（レベル数 numlevel が 0）の場合、ループは実行されず初期値 1 が返される。
  • メモリ確保が必要な場合、PostgreSQL のメモリ管理機構に依存する。

保守担当者向けメモ

  • 本関数は hash_array() と同様のハッシュ結合アルゴリズムを採用している。
  • hash_ltree_extended との関連性があり、ハッシュ計算のシード値の扱いに注意すること。
