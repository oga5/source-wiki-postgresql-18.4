GetExtensibleNodeMethods

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/extensible.c:124

目的

指定された名前（extnodename）に対応する拡張ノードのメソッド構造体（ExtensibleNodeMethods）をレジストリから取得します。

シグネチャまたは形状

const ExtensibleNodeMethods *GetExtensibleNodeMethods(const char *extnodename, bool missing_ok)

入力と出力

  • **入力**:
  • extnodename: 検索対象の拡張ノード名（最大64バイト）。
  • missing_ok: true の場合、ノードが見つからないときに NULL を返します。false の場合はエラーを発生させます。
  • **出力**:
  • 成功時: 該当する ExtensibleNodeMethods 構造体へのポインタ。
  • 失敗時: missing_ok が true であれば NULL。

前提条件と不変条件

  • 内部ハッシュテーブル extensible_node_methods が初期化済みである必要があります。
  • extnodename は登録時に使用された名前と一致する必要があります。

アルゴリズムまたは振る舞い

1. 内部関数 GetExtensibleNodeEntry を呼び出し、extensible_node_methods ハッシュテーブルから指定された名前のノードエントリを検索します。 2. 取得したエントリから ExtensibleNodeMethods 構造体へのポインタをキャストして返します。 3. missing_ok フラグの制御は GetExtensibleNodeEntry に委譲されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/copyfuncs.c
  • src/backend/nodes/equalfuncs.c
  • src/backend/nodes/outfuncs.c
  • src/backend/nodes/readfuncs.c
  • **呼び出し先**:
  • GetExtensibleNodeEntry (内部関数)

関連データ構造

  • ExtensibleNodeMethods: 拡張ノードのコピー、比較、シリアライズ等の関数ポインタを保持する構造体。
  • extensible_node_methods: 登録されたメソッドを管理する内部ハッシュテーブル。

失敗モードと境界条件

  • missing_ok が false の状態で未登録の extnodename を指定した場合、エラーが発生します。
  • extnodename が EXTNODENAME_MAX_LEN を超える場合や、ハッシュテーブルが未初期化の場合の挙動は GetExtensibleNodeEntry の実装に依存します。

保守担当者向けメモ

  • 本関数はコアシステム（copyfuncs.c 等）が拡張ノードの処理を行う際のゲートウェイです。
  • 拡張機能側で RegisterExtensibleNodeMethods が正しく呼び出されていない場合、本関数はノードを解決できません。
