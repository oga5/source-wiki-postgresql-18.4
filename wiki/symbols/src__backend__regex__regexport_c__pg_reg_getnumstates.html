pg_reg_getnumstates

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regexport.c:35

目的

コンパイル済みの正規表現オブジェクトが内部で保持する NFA（非決定性有限オートマトン）の総状態数を取得する。

シグネチャまたは形状

int pg_reg_getnumstates(const regex_t *regex)

入力と出力

  • **入力**: regex_t 構造体へのポインタ。
  • **出力**: NFA が保持する状態数を示す整数値。

前提条件と不変条件

  • 入力される regex ポインタは NULL であってはならない。
  • regex->re_magic が REMAGIC と一致している必要がある（assert による検証）。

アルゴリズムまたは振る舞い

1. regex->re_guts を struct guts * 型にキャストし、内部の search フィールド（cnfa 構造体）にアクセスする。 2. cnfa->nstates に格納されている状態数を読み取り、その値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pg_trgm/trgm_regexp.c:2133 にて呼び出しを確認。
  • **呼び出し先**: 本関数は他の関数を呼び出さない。

関連データ構造

  • regex_t: 正規表現のコンパイル結果を保持する構造体。
  • guts: 正規表現エンジンの内部構造体。
  • cnfa: 正規表現の NFA 構造を定義する構造体。

失敗モードと境界条件

  • regex が NULL または不正なマジックナンバーを持つ場合、assert によりプログラムが異常終了する可能性がある。

保守担当者向けメモ

  • 本関数は regex/regguts.h で定義された内部構造体 cnfa に直接依存している。
  • 正規表現のコンパイル状態が正しく初期化されていることを前提としているため、コンパイル前のオブジェクトを渡さないよう注意が必要である。
