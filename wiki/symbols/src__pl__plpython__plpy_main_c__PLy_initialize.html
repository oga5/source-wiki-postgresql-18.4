PLy_initialize

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_main.c:97

目的

PostgreSQL セッション内での Python インタプリタの初回起動、環境設定、および複数バージョンの競合チェックを行う。

シグネチャまたは形状

static void PLy_initialize(void)

入力と出力

  • 入力: なし（グローバル変数 plpython_version_bitmask_ptr を参照）
  • 出力: なし（副作用としてインタプリタの状態を更新）

前提条件と不変条件

  • セッション内で単一の Python メジャーバージョンのみが使用可能であること。
  • 複数回呼び出された場合、2 回目以降は初期化処理をスキップする（inited フラグによる制御）。

アルゴリズムまたは振る舞い

1. **競合チェック**: plpython_version_bitmask_ptr を確認し、現在の Python メジャーバージョンと不一致であれば FATAL エラーを発生させる。 2. **初期化済み確認**: inited が true であれば即座にリターンする。 3. **Python インタプリタ設定**:

  • plpy モジュールをインタブに追加し、Py_Initialize() を実行。
  • plpy モジュールをインポートし、PLy_init_interp() および PLy_init_plpy() を呼び出して環境を構築。

    • 4. **エラー検証**: PyErr_Occurred() を確認し、初期化中にエラーがあれば FATAL で終了。 5. **状態リセット**: 手続きキャッシュの初期化 (init_procedure_caches)、サブトランザクションリストのクリア (NIL)、実行コンテキストスタックの初期化 (NULL) を行う。 6. **完了**: inited を true に設定する。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpython/plpy_main.c 内の各ハンドラ（plpython3_call_handler 等）から呼び出される。
  • 呼び出し先: PyImport_AppendInittab, Py_Initialize, PyImport_ImportModule, PLy_init_interp, PLy_init_plpy, PLy_elog, init_procedure_caches。

関連データ構造

  • plpython_version_bitmask_ptr: バージョン競合検出に使用。
  • explicit_subtransactions: 初期化時に NIL にリセット。
  • PLy_execution_contexts: 初期化時に NULL にリセット。

失敗モードと境界条件

  • **FATAL**: 異なる Python バージョンがセッション内で検出された場合、または初期化中に Python 側で未捕捉のエラーが発生した場合にプロセスを終了させる。

保守担当者向けメモ

  • 本関数はセッションごとに一度だけ実行される設計である。
  • plpython_version_bitmask_ptr によるチェックは、動的にロードされるライブラリの競合を防ぐための重要な安全装置である。
