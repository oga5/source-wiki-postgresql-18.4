mic_to_win866

定義

  • **種別**: function
  • **定義箇所**: src/backend/utils/mb/conversion_procs/cyrillic_and_mic/cyrillic_and_mic.c:421

目的

MULE_INTERNAL エンコーディングから WIN866 エンコーディングへ文字列を変換する PostgreSQL 変換プロシージャです。

シグネチャまたは形状

Datum mic_to_win866(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS マクロを介して以下の引数を受け取ります。
  • src (CSTRING): 変換元の MULE_INTERNAL 文字列。
  • dest (CSTRING): 変換結果を格納するバッファ。
  • len (INT32): 変換対象のバイト長。
  • noError (BOOL): 変換エラー発生時の挙動フラグ。
  • **出力**: 変換されたバイト数を Datum 型（int32）として返します。

前提条件と不変条件

  • CHECK_ENCODING_CONVERSION_ARGS(PG_MULE_INTERNAL, PG_WIN866) により、ソースエンコーディングが PG_MULE_INTERNAL であり、宛先エンコーディングが PG_WIN866 であることが検証されます。

アルゴリズムまたは振る舞い

1. PG_GETARG_* マクロを使用して引数を取得します。 2. mic2latin_with_table 関数を呼び出し、変換を実行します。

  • 変換テーブルとして koi2win866 を使用します。
  • 内部的な文字セット識別子として LC_KOI8_R を指定します。

    • 3. 変換されたバイト数を PG_RETURN_INT32 で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1(mic_to_win866) により登録）。
  • **呼び出し先**: mic2latin_with_table（変換処理の実行）。

関連データ構造

  • koi2win866: 変換に使用される static const unsigned char 型のマッピングテーブル。

失敗モードと境界条件

  • CHECK_ENCODING_CONVERSION_ARGS の検証に失敗した場合、PostgreSQL のエラーハンドリング機構により処理が中断されます。
  • noError フラグの設定により、変換不可能な文字が出現した際の挙動（エラー発生またはスキップ）が制御されます。

保守担当者向けメモ

  • 本関数は PG_FUNCTION_ARGS を引数に取るため、SQL インターフェース経由での呼び出しを前提としています。直接呼び出す場合は PostgreSQL の関数呼び出し規約に従ってください。
  • 変換ロジックの変更が必要な場合は、マッピングテーブル koi2win866 の整合性を確認してください。
