ExecInitBitmapAnd

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapAnd.c:54

目的

BitmapAnd ノードの初期化を行い、実行に必要な BitmapAndState 構造体を構築する。

シグネチャまたは形状

BitmapAndState *ExecInitBitmapAnd(BitmapAnd *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の BitmapAnd プランノード。
  • estate: 実行状態を保持する EState オブジェクト。
  • eflags: 実行フラグ（EXEC_FLAG_BACKWARD および EXEC_FLAG_MARK はサポート外）。
  • **出力**:
  • 初期化済みの BitmapAndState 構造体へのポインタ。

前提条件と不変条件

  • eflags に EXEC_FLAG_BACKWARD または EXEC_FLAG_MARK が含まれていないこと（Assert で検証）。
  • BitmapAnd ノードは ExecQual や ExecProject を呼び出さないため、式コンテキストやタプルスロットの初期化は不要。

アルゴリズムまたは振る舞い

1. makeNode(BitmapAndState) を使用して状態構造体を割り当てる。 2. BitmapAnd ノード内のサブプラン数に基づき、PlanState ポインタの配列を確保する。 3. BitmapAndState の各フィールド（ps.plan, ps.state, ps.ExecProcNode など）を初期化する。 4. node->bitmapplans リストを走査し、各サブプランに対して ExecInitNode を再帰的に呼び出す。 5. 初期化された各サブプランの PlanState を bitmapplanstates 配列に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:197 において、ExecInitNode 経由で呼び出される。
  • **呼び出し先**: ExecInitNode（各サブプランの初期化のため）。

関連データ構造

  • BitmapAndState: 本ノードの実行状態を保持する構造体。
  • BitmapAnd: プランツリー上のノード構造体。
  • PlanState: サブプランの実行状態を保持する構造体。

失敗モードと境界条件

  • eflags に禁止されたフラグが渡された場合、Assert によりデバッグビルドで停止する。
  • メモリ不足時には palloc0 がエラーを発生させる。

保守担当者向けメモ

  • 本ノードは ExecProcNode 呼び出し規約をサポートしていないため、実行時には MultiExecProcNode を使用すること。
  • サブプランの初期化順序は、プランナーが決定した最適化順序に従う。
