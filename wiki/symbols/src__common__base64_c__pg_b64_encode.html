pg_b64_encode

定義

  • 種別: function
  • 定義箇所: src/common/base64.c:48

目的

バイナリデータをBase64エンコードされた文字列に変換します。

シグネチャまたは形状

int pg_b64_encode(const uint8 *src, int len, char *dst, int dstlen);

入力と出力

  • **入力**:
  • src: エンコード対象のバイナリデータ。
  • len: src のバイト長。
  • dst: エンコード結果を格納するバッファ。
  • dstlen: dst バッファのサイズ。
  • **出力**:
  • 成功時: エンコードされた文字列の長さ（バイト数）。
  • 失敗時: -1（バッファ不足時）。

前提条件と不変条件

  • dst バッファは、エンコード結果を格納するのに十分なサイズである必要があります。
  • 変換中にバッファオーバーフローを検知した場合、出力バッファ dst は memset によりゼロクリアされます。
  • 処理終了時に (p - dst) <= dstlen が成立することを保証します。

アルゴリズムまたは振る舞い

1. 入力バイトを3バイトずつ読み込み、ビットシフト操作で4つの6ビット値に分割します。 2. _base64 テーブルを参照して、各6ビット値を対応するBase64文字に変換します。 3. 入力データに端数がある場合、パディング文字 = を付与します。 4. 処理の各段階で、書き込み先のバッファ残量をチェックし、不足している場合はエラー処理へ移行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/postgres_fdw/connection.c
  • src/backend/libpq/auth-scram.c
  • src/common/scram-common.c
  • **呼び出し先**:
  • memset（標準ライブラリ）
  • Assert（マクロ）
  • 内部参照: _base64（変換テーブル）

関連データ構造

  • _base64: エンコード用の変換テーブル。

失敗モードと境界条件

  • **バッファ不足**: dstlen がエンコード後のデータ長に対して不足している場合、error ラベルへジャンプし、-1 を返します。
  • **空白文字**: 本実装は空白文字を一切許容しません。

保守担当者向けメモ

  • 本関数は厳格な仕様であり、空白文字が含まれる入力はエラーとなります。
  • 呼び出し側は事前に pg_b64_enc_len を使用して、必要なバッファサイズを確保してください。
  • FRONTEND マクロの切り替えにより、サーバーおよびクライアントの両方で利用可能です。
