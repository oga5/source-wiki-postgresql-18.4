process_psqlrc_file

定義

  • 種別: function
  • 定義箇所: src/bin/psql/startup.c:812

目的

指定されたパスを基に、PostgreSQL のバージョン情報（マイナー/メジャー）を付与した設定ファイル（psqlrc）の存在を確認し、読み込みを実行する。

シグネチャまたは形状

static void process_psqlrc_file(char *filename)

入力と出力

  • **入力**: filename (読み込むべき設定ファイルのベースパス)
  • **出力**: なし（戻り値は void）

前提条件と不変条件

  • filename は有効な文字列ポインタであること。
  • process_file 関数が利用可能であること。
  • Windows 環境下では R_OK マクロが未定義の場合に 4 として定義される。

アルゴリズムまたは振る舞い

1. 入力された filename に基づき、以下の順序でファイルパスを生成する。

  • filename + "-" + PG_VERSION (マイナーバージョン付き)
  • filename + "-" + PG_MAJORVERSION (メジャーバージョン付き)
  • filename (元のパス)

    • 2. access 関数を用いて、生成されたパスに対して読み取り権限（R_OK）があるか順次確認する。 3. 最初に存在が確認されたファイルに対して process_file(path, false) を呼び出し、内容を処理する。 4. 生成した一時的なパス文字列（psqlrc_minor, psqlrc_major）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/startup.c 内の複数箇所（rc_file や envrc_alloc を引数として呼び出し）。
  • **呼び出し先**: process_file（ファイル読み込み実行）、psprintf（パス文字列生成）、access（ファイル存在確認）、free（メモリ解放）。

関連データ構造

  • PG_VERSION, PG_MAJORVERSION: PostgreSQL のバージョン情報を保持するマクロ。

失敗モードと境界条件

  • 指定されたいずれのファイルも存在しない場合、何も処理を行わずに終了する。
  • psprintf によるメモリ確保に失敗した場合の挙動は、当該関数の実装に依存する。

保守担当者向けメモ

  • 本関数は psqlrc の読み込み順序（バージョン固有設定から汎用設定へ）を制御する。新しいバージョン対応を追加する際は、この優先順位が意図通りか確認すること。
  • Windows 環境における R_OK の定義は、プラットフォーム固有の挙動を吸収するためのものである。
