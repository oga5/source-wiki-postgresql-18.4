binary_upgrade_set_next_index_relfilenode

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_upgrade_support.c:129

目的

pg_upgrade 実行中、次に作成されるインデックスに対して特定の物理ファイル番号（RelFileNumber）を強制的に割り当てるためのグローバル変数を設定する。

シグネチャまたは形状

Datum binary_upgrade_set_next_index_relfilenode(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_OID(0) を介して RelFileNumber（OID 型として受け取り）を指定する。
  • **出力**: なし（PG_RETURN_VOID()）。

前提条件と不変条件

  • サーバーが binary_upgrade モードで起動されている必要がある。CHECK_IS_BINARY_UPGRADE マクロにより、この条件が満たされない場合はエラーが発生する。

アルゴリズムまたは振る舞い

1. 引数から RelFileNumber を取得する。 2. CHECK_IS_BINARY_UPGRADE を実行し、アップグレードモードであることを検証する。 3. グローバル変数 binary_upgrade_next_index_pg_class_relfilenumber に取得した値を代入する。 4. 処理を終了し、void を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/pg_dump.c 内の SQL クエリ生成箇所から呼び出される。
  • **呼び出し先**: CHECK_IS_BINARY_UPGRADE マクロ（定義元は catalog/binary_upgrade.h 経由の可能性が高いが、本ソース内ではマクロとして展開される）。

関連データ構造

  • binary_upgrade_next_index_pg_class_relfilenumber: 次回のインデックス作成時に使用される物理ファイル番号を保持するグローバル変数。

失敗モードと境界条件

  • binary_upgrade モード以外で呼び出された場合、CHECK_IS_BINARY_UPGRADE によりエラーが発生する。

保守担当者向けメモ

  • 本関数は pg_upgrade によるメタデータ移行専用の管理インターフェースである。
  • 誤用はシステムカタログの整合性を破壊するため、通常の運用環境では使用されないよう制限されている。
