ExplainPrintJITSummary

定義

  • 種別: function
  • 定義箇所: src/backend/commands/explain.c:874

目的

クエリ実行計画の EXPLAIN 出力において、JIT（Just-In-Time）コンパイルに関する統計情報の集計と表示を行う。

シグネチャまたは形状

void ExplainPrintJITSummary(ExplainState *es, QueryDesc *queryDesc)

入力と出力

  • **入力**:
  • es: 出力状態を管理する ExplainState 構造体。
  • queryDesc: 実行されたクエリの記述子。estate を通じて JIT 統計情報にアクセスする。
  • **出力**: なし（ExplainState を通じてバッファへ出力内容を蓄積する）。

前提条件と不変条件

  • queryDesc->estate->es_jit_flags に PGJIT_PERFORM が設定されている場合にのみ処理を実行する。
  • 統計情報の集計にはローカルの JitInstrumentation 構造体を使用し、元の estate 内の統計情報を直接変更しない。

アルゴリズムまたは振る舞い

1. PGJIT_PERFORM フラグを確認し、JIT が有効でない場合は即座に終了する。 2. ローカル変数 ji を初期化する。 3. queryDesc->estate->es_jit が存在する場合、InstrJitAgg を呼び出してメインの JIT 統計を ji に集計する。 4. queryDesc->estate->es_jit_worker_instr が存在する場合、並列ワーカーの JIT 統計を ji に加算（マージ）する。 5. 最終的に ExplainPrintJIT を呼び出し、集計された統計情報を出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/explain.c (メインの EXPLAIN 処理フロー内)
  • contrib/auto_explain/auto_explain.c
  • **呼び出し先**:
  • InstrJitAgg (統計情報の集計)
  • ExplainPrintJIT (最終的な出力処理)

関連データ構造

  • ExplainState: 出力形式やバッファを保持する。
  • QueryDesc: クエリ実行状態（EState）を保持する。
  • JitInstrumentation: JIT 実行時間や最適化回数などの統計情報を保持する構造体。

失敗モードと境界条件

  • JIT が無効な場合、何も出力せずに正常終了する。
  • 並列ワーカーが存在しない場合、ワーカー側の統計集計はスキップされる。
  • 本関数は複数回呼び出される可能性があるため、元の estate を破壊しない設計となっている。

保守担当者向けメモ

  • ExplainPrintJIT との役割分担に注意すること。本関数は統計の「集計」を担い、ExplainPrintJIT は「表示」を担う。
  • 並列クエリ実行時の統計マージロジックを変更する場合は、InstrJitAgg の挙動と整合性を取ること。
