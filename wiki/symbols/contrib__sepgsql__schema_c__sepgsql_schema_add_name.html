sepgsql_schema_add_name

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/schema.c:216

目的

指定されたスキーマに対して、新しいオブジェクトを追加する権限（SEPG_DB_SCHEMA__ADD_NAME）があるかを検証します。

シグネチャまたは形状

void sepgsql_schema_add_name(Oid namespaceId)

入力と出力

  • **入力**: namespaceId (Oid) - 権限チェック対象となるスキーマの OID。
  • **出力**: なし。権限チェックに失敗した場合はエラーを発生させます。

前提条件と不変条件

  • namespaceId は有効な pg_namespace カタログの OID である必要があります。
  • 内部で呼び出される check_schema_perms により、権限違反時にはトランザクションを中断するエラーが発生します。

アルゴリズムまたは振る舞い

1. 引数として受け取った namespaceId を使用します。 2. check_schema_perms 関数を呼び出し、SEPG_DB_SCHEMA__ADD_NAME 権限を要求します。 3. 第3引数に true を渡しているため、権限がない場合は即座にエラーが送出されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/sepgsql/proc.c:279
  • contrib/sepgsql/relation.c:681
  • **呼び出し先**:
  • check_schema_perms (内部ユーティリティ関数)

関連データ構造

  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • 指定された namespaceId がシステム上に存在しない場合、check_schema_perms 内部の処理に依存します。
  • SELinux ポリシーにより ADD_NAME 権限が拒否されている場合、エラーが発生し処理が停止します。

保守担当者向けメモ

  • 本関数はスキーマへのオブジェクト追加を伴う操作のフックとして使用されます。
  • 権限チェックのロジックを変更する場合は、check_schema_perms の実装を確認してください。
