getJsonEncodingConst

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_expr.c:3248

目的

SQL/JSON関数で使用されるエンコーディング情報を、PostgreSQLの定数ノード（Const）として生成する。

シグネチャまたは形状

static Const *getJsonEncodingConst(JsonFormat *format)

入力と出力

  • **入力**: JsonFormat *format（JSONのフォーマット情報を含む構造体）
  • **出力**: Const *（エンコーディング名「UTF8」「UTF16」「UTF32」のいずれかを保持する定数ノード）

前提条件と不変条件

  • format が NULL であるか、format_type または encoding が JS_ENC_DEFAULT の場合は、デフォルトで JS_ENC_UTF8 が選択される。
  • 戻り値の Const ノードは NAMEOID 型として生成される。

アルゴリズムまたは振る舞い

1. 入力された format を確認し、エンコーディングを決定する。 2. 決定したエンコーディング（JS_ENC_UTF16, JS_ENC_UTF32, JS_ENC_UTF8）に基づき、対応する文字列リテラルを選択する。 3. 未定義のエンコーディングが指定された場合は elog(ERROR) を発生させる。 4. namestrcpy を使用して文字列を NameData にコピーし、makeConst を用いて Const ノードを構築して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/parser/parse_expr.c 内の複数箇所（getJsonEncodingConst を呼び出す文脈が確認されている）。
  • **呼び出し先**: palloc, namestrcpy, makeConst, NameGetDatum 等のPostgreSQL内部ユーティリティ関数。

関連データ構造

  • JsonFormat: エンコーディング情報を含む入力構造体。
  • Const: 戻り値として生成されるノード型。
  • NameData: エンコーディング名を保持するために使用される構造体。

失敗モードと境界条件

  • format が無効なエンコーディング値を持つ場合、elog(ERROR) により処理が中断される。
  • メモリ割り当て（palloc）失敗時はPostgreSQLの標準的なエラーハンドリングに従う。

保守担当者向けメモ

  • 本関数はSQL/JSONのエンコーディング指定を定数ノードに変換する専用のヘルパーである。
  • 新しいエンコーディングをサポートする場合は、switch 文へのケース追加が必要となる。
