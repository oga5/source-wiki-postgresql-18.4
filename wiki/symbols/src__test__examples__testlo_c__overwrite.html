overwrite

定義

  • 種別: function
  • 定義箇所: src/test/examples/testlo.c:107

目的

PostgreSQL のラージオブジェクト内の指定された範囲を文字 'X' で上書きする。

シグネチャまたは形状

static void overwrite(PGconn *conn, Oid lobjId, int start, int len)

入力と出力

  • conn: データベース接続ハンドル (PGconn *)。
  • lobjId: 操作対象のラージオブジェクトの OID (Oid)。
  • start: 上書きを開始するオフセット位置 (int)。
  • len: 上書きするバイト数 (int)。
  • 出力: なし（標準エラー出力への進捗・エラーメッセージ出力のみ）。

前提条件と不変条件

  • conn は有効なデータベース接続であること。
  • lo_open が INV_WRITE モードで成功すること。
  • len 分のメモリ確保が成功すること。

アルゴリズムまたは振る舞い

1. lo_open を使用してラージオブジェクトを書き込みモードで開く。 2. lo_lseek を使用して、指定された start 位置へシークする。 3. len バイト分のバッファを確保し、すべて 'X' で埋める。 4. lo_write をループ内で呼び出し、バッファの内容をラージオブジェクトに書き込む。 5. 書き込み完了後、バッファを解放し、lo_close でラージオブジェクトを閉じる。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコード内では、main 関数から呼び出されることが示唆されている。
  • 呼び出し先: lo_open, lo_lseek, lo_write, lo_close (libpq API)、および malloc, free, fprintf (標準ライブラリ)。
  • 注意: ツールが検出した他のファイル（contrib/ 等）内の overwrite という単語は、本関数への参照ではなく、単なる文字列の一致である可能性が高い。

関連データ構造

  • PGconn: libpq のデータベース接続オブジェクト。
  • Oid: PostgreSQL のオブジェクト識別子。

失敗モードと境界条件

  • lo_open が失敗した場合、エラーメッセージを標準エラー出力に出力するが、処理は継続される（その後の lo_lseek 等でエラーが発生する可能性がある）。
  • lo_write が 0 以下の値を返した場合、書き込み失敗としてループを中断し、エラーメッセージを出力する。

保守担当者向けメモ

  • 本関数はサンプルコードの一部であり、エラーハンドリングは最小限である。実運用環境では lo_open の戻り値チェック後に早期リターンするなどの堅牢な処理が必要である。
  • 64ビットオフセットを扱う必要がある場合は、src/test/examples/testlo64.c の実装を参照すること。
