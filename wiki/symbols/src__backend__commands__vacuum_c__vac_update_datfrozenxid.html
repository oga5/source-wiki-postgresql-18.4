vac_update_datfrozenxid

定義

  • src/backend/commands/vacuum.c:1623

目的

データベース全体の凍結トランザクションID（datfrozenxid）および最小マルチトランザクションID（datminmxid）を更新し、pg_database カタログに永続化する。これにより、pg_xact などの SLRU 領域の切り詰めを安全に行えるようにする。

シグネチャまたは形状

void vac_update_datfrozenxid(void)

入力と出力

  • **入力**: なし（内部的に MyDatabaseId やシステムカタログを参照）
  • **出力**: なし（副作用として pg_database の該当行を更新）

前提条件と不変条件

  • データベースごとに一度に一つのバックエンドのみが実行可能（LockDatabaseFrozenIds(ExclusiveLock) による排他制御）。
  • pg_class をスキャンし、各リレーションの relfrozenxid および relminmxid を集計して最小値を算出する。
  • 算出された値が「未来」の値（ReadNextTransactionId() 等を超える値）である場合、データ破損の可能性があると判断し、更新を中止する。

アルゴリズムまたは振る舞い

1. LockDatabaseFrozenIds で排他ロックを取得する。 2. GetOldestNonRemovableTransactionId() 等を用いて、初期の最小値を設定する。 3. pg_class をシーケンシャルスキャンし、各リレーションの relfrozenxid と relminmxid を確認する。

  • RELKIND_RELATION, MATVIEW, TOASTVALUE 以外のリレーションは無視する。
  • 未来のトランザクションIDが検出された場合は bogus フラグを立てて処理を中断する。
  • 正常な値であれば、最小値を更新する。

    • 4. pg_database を RowExclusiveLock で開き、systable_inplace_update_begin を使用して対象データベースのタプルを更新する。 5. datfrozenxid および datminmxid が現在の値より進んでいる場合のみ値を書き換える。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/vacuum.c（vacuum() 関数内）
  • src/backend/postmaster/autovacuum.c
  • **呼び出し先**:
  • LockDatabaseFrozenIds, GetOldestNonRemovableTransactionId, GetOldestMultiXactId, ReadNextTransactionId, ReadNextMultiXactId
  • table_open, systable_beginscan, systable_getnext, systable_endscan, table_close
  • systable_inplace_update_begin

関連データ構造

  • pg_database (カタログ)
  • pg_class (カタログ)
  • FormData_pg_database

失敗モードと境界条件

  • **データ破損**: pg_class 内に「未来」のトランザクションIDが存在する場合、更新を行わずに終了する。
  • **ロック競合**: 他のバックエンドが既に LockDatabaseFrozenIds を保持している場合、待機が発生する。
  • **タプル不在**: pg_database 内に現在のデータベースに対応するタプルが見つからない場合、elog(ERROR) を発生させる。

保守担当者向けメモ

  • 本関数は pg_database をインプレース更新するため、システムカタログの整合性に直接影響する。
  • vac_truncate_clog() との競合を防ぐため、LockDatabaseFrozenIds による保護が必須である。
  • pg_upgrade 等で不正な値が混入した場合の安全策として、未来のIDに対するチェックが実装されている。
