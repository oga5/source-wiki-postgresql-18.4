r_Step_1a

目的

Porter Stemming アルゴリズムの第 1 段階（Step 1a）を実行し、単語末尾の特定の接尾辞（主に複数形や過去形の語尾）を削除または置換する。

シグネチャまたは形状

static int r_Step_1a(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: int（成功時に 1、失敗またはマッチしない場合に 0、エラー発生時に負の値を返す）

前提条件と不変条件

  • z->c が現在のカーソル位置を指していること。
  • z->lb が文字列の開始境界を指していること。
  • z->p が文字列バッファを保持していること。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置 z->c を z->ket に設定する。 2. カーソルの直前（z->c - 1）が文字 's'（ASCII 115）であるかを確認し、境界条件を満たさない場合は終了する。 3. find_among_b を使用して、定義された接尾辞テーブル a_0 と照合する。 4. マッチした場合、z->bra を現在のカーソル位置に設定し、マッチした接尾辞の種類（among_var）に応じて以下のいずれかの操作を行う：

  • case 1: s_0 に置換。
  • case 2: s_1 に置換。
  • case 3: 接尾辞を削除。

    • 5. 操作の結果（slice_from_s または slice_del の戻り値）を返す。

関連データ構造

  • SN_env: ステミング処理の状態管理構造体。
  • a_0: 接尾辞マッチング用の among 構造体配列。

失敗モードと境界条件

  • カーソルが境界 z->lb にある場合、または直前の文字が 's' でない場合は 0 を返す。
  • 接尾辞がマッチしない場合は 0 を返す。
  • slice_from_s または slice_del が負の値を返した場合、そのエラーコードをそのまま伝播させる。
