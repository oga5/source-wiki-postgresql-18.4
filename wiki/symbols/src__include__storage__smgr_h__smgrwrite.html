smgrwrite

定義

  • 種別: function
  • 定義箇所: src/include/storage/smgr.h:130

目的

単一のデータブロックを物理ストレージへ書き込むためのインターフェースです。

シグネチャまたは形状

static inline void smgrwrite(SMgrRelation reln, ForkNumber forknum, BlockNumber blocknum, const void *buffer, bool skipFsync)

入力と出力

  • reln (SMgrRelation): 書き込み対象のリレーションハンドル。
  • forknum (ForkNumber): 書き込み対象のフォーク番号。
  • blocknum (BlockNumber): 書き込み対象のブロック番号。
  • buffer (const void *): 書き込むデータを含むメモリ領域。
  • skipFsync (bool): true の場合、書き込み後の fsync をスキップします。
  • 戻り値: なし。

前提条件と不変条件

  • reln は有効な SMgrRelation である必要があります。
  • 内部的に smgrwritev を呼び出すため、ベクトル I/O インターフェースの制約に従います。

アルゴリズムまたは振る舞い

単一ブロックの書き込み要求を smgrwritev へ転送します。具体的には、buffer ポインタを配列として扱い、要素数 1 で smgrwritev(reln, forknum, blocknum, &buffer, 1, skipFsync) を実行します。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/storage/buffer/bufmgr.c
  • src/backend/storage/buffer/localbuf.c
  • src/backend/storage/smgr/bulk_write.c
  • src/test/modules/test_aio/test_aio.c
  • 呼び出し先:
  • smgrwritev (直接の呼び出し先)

関連データ構造

  • SMgrRelation: ストレージマネージャのリレーションハンドル。
  • ForkNumber: リレーション内のフォーク（メイン、FSM、VMなど）を識別する型。
  • BlockNumber: ブロックを識別する型。

失敗モードと境界条件

  • 無効な reln が渡された場合、ストレージマネージャのディスパッチ先でエラーが発生する可能性があります。
  • skipFsync が true の場合、システムクラッシュ時にデータが永続化されないリスクがあるため、呼び出し側で適切な同期処理（チェックポイント等）を保証する必要があります。

保守担当者向けメモ

  • 本関数は smgrwritev の単一ブロック用ラッパーです。新規実装やパフォーマンスが重要な箇所では、可能な限りベクトル I/O インターフェース（smgrwritev）の直接利用を検討してください。
  • smgr.c の関数はリエントラントではないため、割り込みによる smgrreleaseall の発生に注意してください。
