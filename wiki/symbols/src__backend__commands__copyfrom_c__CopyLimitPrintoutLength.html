CopyLimitPrintoutLength

定義

  • 種別: function
  • 定義箇所: src/backend/commands/copyfrom.c:332

目的

COPY コマンド実行時のエラーメッセージ等で表示される文字列を、一定の長さ（100バイト）に切り詰め、末尾に省略記号 ... を付与して整形する。

シグネチャまたは形状

char *CopyLimitPrintoutLength(const char *str)

入力と出力

  • **入力**: str (切り詰め対象のヌル終端文字列)
  • **出力**: 切り詰めおよび ... 付与後の新しい文字列（palloc でメモリ確保される）。元の文字列が制限以下の場合は、その複製を返す。

前提条件と不変条件

  • 入力文字列 str は有効なヌル終端文字列であること。
  • 戻り値は呼び出し元で解放（pfree）する必要がある。
  • MAX_COPY_DATA_DISPLAY 定数（100）が切り詰め基準として使用される。

アルゴリズムまたは振る舞い

1. 入力文字列の長さを取得する。 2. 長さが MAX_COPY_DATA_DISPLAY 以下であれば、pstrdup で複製して即座に返す。 3. 長さが制限を超える場合、pg_mbcliplen を使用してマルチバイト文字の境界を考慮しながら切り詰め長を計算する。 4. 切り詰め後の長さ + 3バイト（... 用）のメモリを palloc で確保する。 5. 切り詰めた文字列をコピーし、末尾に ... を付与して返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c (行284, 311)
  • src/backend/commands/copyfromparse.c (行1054)
  • **呼び出し先**:
  • strlen
  • pstrdup
  • pg_mbcliplen
  • palloc
  • memcpy
  • strcpy

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **メモリ不足**: palloc が失敗した場合、PostgreSQL のエラーハンドリングによりアボートする。

保守担当者向けメモ

  • 本関数は COPY コマンドのログ出力やエラーコンテキスト生成において、過大なデータがログを埋め尽くすのを防ぐために使用される。
  • MAX_COPY_DATA_DISPLAY を変更する場合は、表示上の制約とログの可読性のバランスを考慮すること。
