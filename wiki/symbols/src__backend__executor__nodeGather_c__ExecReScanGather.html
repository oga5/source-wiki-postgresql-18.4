ExecReScanGather

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeGather.c:441

目的

Gather ノードの再スキャン処理を実行し、並列実行状態をリセットして次回の実行に備える。

シグネチャまたは形状

void ExecReScanGather(GatherState *node)

入力と出力

  • **入力**: GatherState *node（Gather ノードの実行状態構造体）
  • **出力**: なし（副作用としてノードの状態を更新）

前提条件と不変条件

  • node は有効な GatherState を指している必要がある。
  • ExecReScanGather 呼び出し後、次回の ExecProcNode 呼び出し時に共有状態が再構築される。

アルゴリズムまたは振る舞い

1. **ワーカーの停止**: ExecShutdownGatherWorkers(node) を呼び出し、既存のバックグラウンドワーカーを終了させる。 2. **初期化フラグのリセット**: node->initialized を false に設定し、次回の実行で共有状態が再構築されるようにする。 3. **パラメータ更新**: gather->rescan_param が設定されている場合、子プランの chgParam に当該パラメータを追加し、リーダープロセスでの再スキャンを通知する。 4. **子ノードの再スキャン**: 子プランの chgParam が空であれば、ExecReScan(outerPlan) を呼び出して子ノードを再スキャンする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execAmi.c:174
  • **呼び出し先**:
  • ExecShutdownGatherWorkers
  • ExecReScan (条件付き)

関連データ構造

  • GatherState: Gather ノードの実行状態を保持する構造体。
  • Gather: プランノードの定義構造体。
  • PlanState: 子プランの実行状態。

失敗モードと境界条件

  • rescan_param が負の値の場合、パラメータの追加はスキップされる。
  • 子ノードの chgParam が設定されている場合、ExecReScan は呼び出されない（次回の ExecProcNode で処理されるため）。

保守担当者向けメモ

  • ReInitializeDSM（共有状態のリセット）と ReScan（ローカル状態のリセット）の順序依存性に注意すること。両者に依存する処理は、最初の ExecProcNode 呼び出しまで待機する必要がある。
