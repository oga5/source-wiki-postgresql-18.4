lex_peek

定義

  • 種別: function
  • 定義箇所: src/common/jsonapi.c:304

目的

JSON字句解析コンテキストが現在保持しているトークン型を、解析を進めることなく参照（peek）する。

シグネチャまたは形状

static inline JsonTokenType lex_peek(JsonLexContext *lex)

入力と出力

  • **入力**: JsonLexContext *lex - 現在の解析状態を保持するコンテキスト構造体へのポインタ。
  • **出力**: JsonTokenType - 現在のトークン型を示す列挙値。

前提条件と不変条件

  • lex ポインタは有効な JsonLexContext を指している必要がある。
  • 本関数はインライン関数であり、呼び出し側のコンテキストで lex->token_type が常に最新の状態に更新されていることを前提とする。

アルゴリズムまたは振る舞い

1. 引数として受け取った JsonLexContext 構造体のメンバ token_type を直接参照する。 2. 参照した値を戻り値として返す。 3. 解析位置の移動やトークンの消費は行わない。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/jsonapi.c 内の複数の箇所で、現在のトークンを確認するために使用されている（例: 319, 773, 827, 895, 925, 1257, 1271, 1328, 1351, 1424 行）。
  • **呼び出し先**:
  • 呼び出し先は存在しない（構造体メンバへの直接アクセスのみ）。

関連データ構造

  • JsonLexContext: 解析状態を保持する主要な構造体。
  • JsonTokenType: トークンの種類を定義する列挙型。

失敗モードと境界条件

  • lex が NULL の場合、未定義の動作を引き起こす可能性がある。
  • 本関数自体はエラーチェックを行わないため、呼び出し元でコンテキストの有効性を保証する必要がある。

保守担当者向けメモ

  • 本関数は非常に軽量なアクセサであるため、パフォーマンスへの影響は最小限である。
  • lex_expect 等のトークン消費を伴う関数と対で使用されることが多いため、ロジック変更時はトークン状態の整合性に注意すること。
