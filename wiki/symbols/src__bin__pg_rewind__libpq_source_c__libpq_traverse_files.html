libpq_traverse_files

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/libpq_source.c:231

目的

リモートの PostgreSQL サーバーからデータディレクトリ内の全ファイルおよびディレクトリのリストを再帰的に取得し、各エントリに対してコールバック関数を実行する。

シグネチャまたは形状

static void libpq_traverse_files(rewind_source *source, process_file_callback_t callback)

入力と出力

  • **入力**:
  • source: libpq_source 構造体へのポインタ（rewind_source としてキャスト）。
  • callback: ファイルパス、型、サイズ、リンク先情報を受け取るコールバック関数。
  • **出力**: なし（副作用として callback を呼び出す）。

前提条件と不変条件

  • source は libpq_source 型として初期化済みであること。
  • リモートサーバー上で pg_ls_dir および pg_stat_file 関数が利用可能であること。

アルゴリズムまたは振る舞い

1. WITH RECURSIVE 句を用いた SQL クエリを構築し、リモートサーバーのデータディレクトリを再帰的に走査する。 2. pg_tablespace テーブルを結合し、pg_tblspc/ 配下のシンボリックリンク情報を取得する。 3. PQexec でクエリを実行し、結果セットを走査する。 4. 各行に対し、以下の判定を行う：

  • link_target が存在する場合: 絶対パスなら FILE_TYPE_SYMLINK、それ以外は FILE_TYPE_DIRECTORY。
  • isdir が真の場合: FILE_TYPE_DIRECTORY。
  • その他: FILE_TYPE_REGULAR。

    • 5. 判定した属性を引数として callback を呼び出す。 6. クエリ実行中に削除されたファイルは無視する。

呼び出し元と呼び出し先

  • **呼び出し元**: init_libpq_source 内で rewind_source インターフェースの関数ポインタとして登録される。
  • **呼び出し先**: PQexec, PQresultStatus, PQnfields, PQntuples, PQgetisnull, PQgetvalue, PQclear, pg_fatal, atoll, strcmp, is_absolute_path, および引数として渡された callback。

関連データ構造

  • libpq_source: PGconn を保持する構造体。
  • rewind_source: インターフェース定義。
  • file_type_t: ファイル種別（FILE_TYPE_REGULAR, FILE_TYPE_DIRECTORY, FILE_TYPE_SYMLINK）。

失敗モードと境界条件

  • クエリ実行失敗時: pg_fatal により終了する。
  • 結果セットの列数が期待値（4）と異なる場合: pg_fatal により終了する。
  • カスタムシンボリックリンク: バックエンド側にリンク先を取得する汎用関数がないため、標準的なテーブルスペース以外のカスタムリンクは正しくコピーされない可能性がある。

保守担当者向けメモ

  • 本関数は libpq 接続を介してリモートのファイルシステム構造を同期するために使用される。
  • link_target の判定ロジックは、インプレーステーブルスペースのディレクトリ構造に依存している。
