libpq_gettext

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-misc.c:1352

目的

libpq ライブラリ内で使用されるエラーメッセージや文字列を、現在のロケール設定に基づいて翻訳して取得する。

シグネチャまたは形状

char *libpq_gettext(const char *msgid);

入力と出力

  • **入力**: msgid (翻訳対象のメッセージ文字列)
  • **出力**: 翻訳された文字列へのポインタ。翻訳が見つからない場合は msgid 自体が返される。

前提条件と不変条件

  • 呼び出し前に libpq_binddomain() が実行され、メッセージドメインが正しくバインドされている必要がある（本関数内で呼び出し済み）。

アルゴリズムまたは振る舞い

1. libpq_binddomain() を呼び出し、国際化のためのドメイン設定を保証する。 2. dgettext を使用し、libpq ドメインから msgid に対応する翻訳済み文字列を取得して返す。

呼び出し元と呼び出し先

  • **呼び出し先**: libpq_binddomain (同ファイル内)
  • **呼び出し元**: src/interfaces/libpq-oauth/oauth-curl.c 内の各所（エラーメッセージの生成やユーザーへの通知出力時に利用）。

関連データ構造

  • なし

失敗モードと境界条件

  • dgettext の仕様に基づき、指定されたロケールで翻訳が存在しない場合は、入力された msgid がそのまま返される。

保守担当者向けメモ

  • 本関数は libpq の国際化対応の基盤である。メッセージの追加や変更を行う際は、翻訳カタログとの整合性に注意すること。
  • libpq-oauth 関連のコードから頻繁に呼び出されているため、OAuth 認証フローのデバッグ時には本関数経由のメッセージ出力を確認すること。
