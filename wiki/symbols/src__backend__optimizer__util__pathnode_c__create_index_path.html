create_index_path

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/pathnode.c:1051

目的

インデックススキャンを実行するための IndexPath ノードを生成し、そのコストを算出する。

シグネチャまたは形状

IndexPath *create_index_path(PlannerInfo *root,
                             IndexOptInfo *index,
                             List *indexclauses,
                             List *indexorderbys,
                             List *indexorderbycols,
                             List *pathkeys,
                             ScanDirection indexscandir,
                             bool indexonly,
                             Relids required_outer,
                             double loop_count,
                             bool partial_path)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • index: 対象インデックスのメタ情報。
  • indexclauses, indexorderbys, indexorderbycols: インデックスで使用する句や順序付け情報。
  • pathkeys: パスのソート順序。
  • indexscandir: スキャン方向。
  • indexonly: インデックスのみスキャン（Index-Only Scan）を行うか否か。
  • required_outer: パラメータ化されたパスに必要な外部リレーション。
  • loop_count: 推定ループ回数。
  • partial_path: 並列クエリ用の部分パスかどうか。
  • **出力**: 初期化およびコスト計算済みの IndexPath 構造体へのポインタ。

前提条件と不変条件

  • index 引数は有効な IndexOptInfo を指している必要がある。
  • pathnode->path.parallel_aware は常に false に初期化される。
  • pathnode->path.parallel_safe は index->rel->consider_parallel に基づいて設定される。

アルゴリズムまたは振る舞い

1. makeNode(IndexPath) を使用してメモリを確保する。 2. indexonly フラグに基づき、pathtype を T_IndexOnlyScan または T_IndexScan に設定する。 3. get_baserel_parampathinfo を呼び出し、パラメータ化情報を設定する。 4. インデックス関連の各フィールド（indexinfo, indexclauses 等）を引数からコピーする。 5. cost_index を呼び出し、当該パスのコストを算出・設定する。 6. 完成した pathnode を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/indxpath.c
  • src/backend/optimizer/plan/planner.c
  • **呼び出し先**:
  • makeNode (マクロ)
  • get_baserel_parampathinfo
  • cost_index

関連データ構造

  • IndexPath: 本関数が生成する主要な構造体。
  • IndexOptInfo: インデックスの物理的特性を保持する構造体。
  • PlannerInfo: プランナのコンテキスト情報。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性がある。
  • index が NULL の場合、rel へのアクセスでクラッシュする可能性がある。

保守担当者向けメモ

  • 本関数は IndexPath の生成とコスト計算をカプセル化している。コスト計算ロジックを変更する場合は cost_index を参照すること。
  • parallel_workers は本関数内では 0 に固定されている。並列実行を考慮した拡張が必要な場合はこの初期化箇所を確認すること。
