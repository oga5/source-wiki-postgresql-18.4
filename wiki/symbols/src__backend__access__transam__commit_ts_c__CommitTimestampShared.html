CommitTimestampShared

定義

  • 種別: struct
  • 定義箇所: src/backend/access/transam/commit_ts.c:98

目的

コミットタイムスタンプ機能の共有メモリ状態を保持するための構造体です。

シグネチャまたは形状

typedef struct CommitTimestampShared
{
	TransactionId xidLastCommit;
	CommitTimestampEntry dataLastCommit;
	bool		commitTsActive;
} CommitTimestampShared;

入力と出力

本構造体自体はデータ保持用であり、関数ではありません。共有メモリ上に配置され、コミットタイムスタンプの有効状態および直近のコミット情報を管理します。

前提条件と不変条件

  • commitTsActive が true の場合、コミットタイムスタンプの記録機能が有効であることを示します。
  • xidLastCommit および dataLastCommit は、システム内で最後に処理されたコミットの情報を保持します。

アルゴリズムまたは振る舞い

  • 本構造体は共有メモリ領域の一部として確保されます。
  • commit_ts.c 内の静的変数 commitTsShared を通じてアクセスされます。
  • 初期化時および共有メモリサイズ計算時に sizeof(CommitTimestampShared) が参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/commit_ts.c 内の共有メモリ初期化処理およびコミットタイムスタンプ管理関数群。
  • **呼び出し先**: 構造体内部のフィールドは、コミット情報の記録および検索処理において直接読み書きされます。

関連データ構造

  • CommitTimestampEntry: dataLastCommit フィールドの型として使用されます。
  • TransactionId: xidLastCommit フィールドの型として使用されます。

失敗モードと境界条件

  • 共有メモリの確保に失敗した場合、本構造体の初期化が正常に行われません。
  • track_commit_timestamp GUCパラメータの設定変更により、commitTsActive の値が動的に切り替わります。

保守担当者向けメモ

  • 本構造体は CommitTsShmemInit 等の共有メモリ初期化処理において、ShmemInitStruct を介して割り当てられます。
  • 構造体のサイズ変更を行う場合は、共有メモリの計算ロジック（CommitTsShmemSize）との整合性に注意してください。
