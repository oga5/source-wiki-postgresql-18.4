oauth_json_object_field_start

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth-oauth.c:229

目的

サーバーから受信したOAuthエラー応答のJSON解析において、特定のフィールド（status、scope、openid-configuration）を識別し、対応するコンテキスト変数への格納準備を行う。

シグネチャまたは形状

JsonParseErrorType oauth_json_object_field_start(void *state, char *name, bool isnull)

入力と出力

  • **入力**:
  • state: json_ctx 構造体へのポインタ。
  • name: 現在解析中のJSONフィールド名。
  • isnull: フィールド値がnullかどうかを示すフラグ。
  • **出力**:
  • JSON_SUCCESS: 常に成功を返す。

前提条件と不変条件

  • state は json_ctx 型としてキャスト可能であること。
  • 解析対象のJSONはトップレベルのオブジェクト構造であること（ctx->nested == 1 の場合にのみ処理が実行される）。

アルゴリズムまたは振る舞い

1. ctx->nested が 1（トップレベル）であることを確認する。 2. name を以下の既知のフィールド名と比較する：

  • ERROR_STATUS_FIELD: 一致する場合、ctx->target_field を ctx->status に設定。
  • ERROR_SCOPE_FIELD: 一致する場合、ctx->target_field を ctx->scope に設定。
  • ERROR_OPENID_CONFIGURATION_FIELD: 一致する場合、ctx->target_field を ctx->discovery_uri に設定。

    • 3. 一致したフィールド名を ctx->target_field_name に記録する。 4. JSON_SUCCESS を返して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: JSONパーサーのコールバックとして登録され、パーサーから呼び出される。
  • **呼び出し先**: strcmp（標準ライブラリ）。
  • 注: 呼び出し元・呼び出し先の詳細なスタックトレース情報は提供されていません。

関連データ構造

  • json_ctx: 解析状態を保持する構造体。status、scope、discovery_uri などのフィールドを保持する。

失敗モードと境界条件

  • トップレベル以外のネストされたフィールドは無視される。
  • 定義されていないフィールド名が渡された場合、target_field は更新されず、何も起こらない。

保守担当者向けメモ

  • 本関数は json_ctx の状態遷移を制御するコールバックの一部です。新しいエラーフィールドを追加する場合は、比較対象の定数と ctx 構造体のメンバを更新してください。
