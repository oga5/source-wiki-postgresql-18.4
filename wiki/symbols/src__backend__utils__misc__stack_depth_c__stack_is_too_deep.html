stack_is_too_deep

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/stack_depth.c:108

目的

現在のプロセスにおけるスタック消費量が、設定された上限（max_stack_depth_bytes）を超過しているかどうかを判定します。

シグネチャまたは形状

bool stack_is_too_deep(void)

入力と出力

  • **入力**: なし（グローバル変数 stack_base_ptr および max_stack_depth_bytes を参照）。
  • **出力**: スタックが上限を超えている場合は true、そうでない場合（または基準点が未設定の場合）は false を返します。

前提条件と不変条件

  • stack_base_ptr が初期化されている必要があります。
  • スタックの伸長方向（増加または減少）に関わらず、絶対値を用いて正しく差分を計算します。

アルゴリズムまたは振る舞い

1. ローカル変数 stack_top_loc のアドレスを取得し、stack_base_ptr との差分を計算します。 2. 計算された差分 stack_depth が負の値であれば、絶対値に変換します。 3. stack_depth が max_stack_depth_bytes を超え、かつ stack_base_ptr が NULL でない場合に true を返します。 4. 上記条件を満たさない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c
  • src/backend/nodes/queryjumblefuncs.c
  • src/backend/regex/regcomp.c
  • src/backend/utils/misc/stack_depth.c (check_stack_depth 内)
  • src/backend/utils/mmgr/mcxt.c
  • **呼び出し先**: 本関数はポインタ演算のみを行い、他の関数を呼び出しません。

関連データ構造

  • stack_base_ptr: スタックの基準位置を保持するグローバルポインタ。
  • max_stack_depth_bytes: 許容されるスタック深さの最大値（バイト単位）。

失敗モードと境界条件

  • stack_base_ptr が NULL の場合（初期化前）、スタック超過エラーを回避するために常に false を返します。
  • スタックの伸長方向がプラットフォームによって異なる場合でも、絶対値計算により正しく動作します。

保守担当者向けメモ

  • 本関数は頻繁に呼び出される可能性があるため、計算コストを最小限に抑える設計になっています。
  • stack_base_ptr のチェックを最後に行うことで、正常系での処理効率を優先しています。
