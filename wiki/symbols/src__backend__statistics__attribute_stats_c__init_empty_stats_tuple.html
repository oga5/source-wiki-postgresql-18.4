init_empty_stats_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/attribute_stats.c:902

目的

システムカタログ pg_statistic に新しい行を挿入するための初期データ構造（values 配列、nulls 配列、replaces 配列）を構築する。

シグネチャまたは形状

static void init_empty_stats_tuple(Oid reloid, int16 attnum, bool inherited,
                                   Datum *values, bool *nulls, bool *replaces)

入力と出力

  • **入力**:
  • reloid: 対象リレーションの OID。
  • attnum: 対象属性の番号。
  • inherited: 継承された統計かどうかを示すフラグ。
  • values, nulls, replaces: pg_statistic の行を構成するための出力用バッファ（呼び出し元で確保済みであること）。
  • **出力**: なし（引数で渡された配列を直接書き換える）。

前提条件と不変条件

  • values, nulls, replaces は Natts_pg_statistic 分の要素を持つ配列であること。
  • pg_statistic の必須列（starelid, staattnum, stainherit）およびデフォルト値を持つ列（stanullfrac, stawidth, stadistinct）が適切に初期化される。

アルゴリズムまたは振る舞い

1. nulls と replaces 配列の全要素を true に初期化する。 2. starelid, staattnum, stainherit に引数値を設定し、対応する nulls を false にする。 3. 統計情報の基本指標（stanullfrac, stawidth, stadistinct）にデフォルト値を設定し、nulls を false にする。 4. STATISTIC_NUM_SLOTS 分のループを回し、各スロットの stakind, staop, stacoll を初期値（0 または InvalidOid）で埋め、nulls を false に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/attribute_stats.c 内の統計更新処理（attribute_statistics_update 等）から呼び出される。
  • **呼び出し先**: 直接的な関数呼び出しは行わず、memset やマクロ（ObjectIdGetDatum 等）を使用する。

関連データ構造

  • pg_statistic: 本関数が初期化対象とするシステムカタログ。
  • Natts_pg_statistic: pg_statistic の属性数。
  • STATISTIC_NUM_SLOTS: 統計スロットの最大数。

失敗モードと境界条件

  • 渡された配列バッファのサイズが Natts_pg_statistic 未満の場合、メモリ破壊が発生する。
  • 呼び出し元で適切なメモリ領域が確保されていない場合、セグメンテーション違反を引き起こす。

保守担当者向けメモ

  • 本関数は pg_statistic のスキーマ定義に依存している。カタログ定義に変更があった場合は、初期化ロジックの整合性を確認すること。
  • デフォルト値（DEFAULT_NULL_FRAC 等）は statistics.h 等で定義されている定数に依存している。
