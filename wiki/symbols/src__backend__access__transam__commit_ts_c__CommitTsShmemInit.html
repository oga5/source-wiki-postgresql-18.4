CommitTsShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/commit_ts.c:529

目的

コミットタイムスタンプ機能で使用する共有メモリ領域の初期化およびSLRU（Simple Least Recently Used）キャッシュのセットアップを行う。

シグネチャまたは形状

void CommitTsShmemInit(void)

入力と出力

  • 入力: なし（グローバル変数 commit_timestamp_buffers および CommitTsCtl を参照）
  • 出力: なし（副作用として共有メモリの確保と初期化を行う）

前提条件と不変条件

  • commit_timestamp_buffers が0の場合、自動チューニングが実行される。
  • 実行後、commit_timestamp_buffers は0以外であることが保証される。
  • Postmaster プロセス以外で実行される場合、共有メモリの初期化状態が正しく維持されている必要がある。

アルゴリズムまたは振る舞い

1. **バッファ数の自動チューニング**: commit_timestamp_buffers が0の場合、CommitTsShmemBuffers() を呼び出して適切な値を算出し、SetConfigOption を用いて設定を適用する。 2. **SLRUの初期化**: CommitTsCtl に対して SimpleLruInit を実行し、pg_commit_ts ディレクトリを対象としたSLRU管理を開始する。 3. **ページ順序判定の登録**: CommitTsPagePrecedes を CommitTsCtl->PagePrecedes に設定し、SlruPagePrecedesUnitTests で検証を行う。 4. **共有メモリ構造体の確保**: ShmemInitStruct を使用して CommitTimestampShared 構造体を共有メモリ上に確保する。 5. **初期状態の設定**: Postmaster 配下でない（単独起動またはバックエンド起動の）場合、xidLastCommit を InvalidTransactionId に、commitTsActive を false に初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c 内の CreateSharedMemoryAndSemaphores から呼び出される。
  • 呼び出し先: CommitTsShmemBuffers, SetConfigOption, SimpleLruInit, SlruPagePrecedesUnitTests, ShmemInitStruct, CommitTsPagePrecedes。

関連データ構造

  • CommitTsCtl: SLRU管理用の制御構造体。
  • CommitTimestampShared: コミットタイムスタンプの共有状態を保持する構造体。
  • LWTRANCHE_COMMITTS_BUFFER, LWTRANCHE_COMMITTS_SLRU: ロック用トランシェ。

失敗モードと境界条件

  • commit_timestamp_buffers の設定失敗時、PGC_S_OVERRIDE を用いて強制的に適用を試みる。
  • Postmaster 配下で ShmemInitStruct が既存のメモリ領域を見つけられない場合、Assert により停止する。

保守担当者向けメモ

  • 本関数はPostgreSQLの共有メモリ初期化シーケンスの一部であるため、呼び出し順序の変更には注意が必要。
  • track_commit_timestamp の動的変更に伴う再初期化ロジックは本関数外（ActivateCommitTs 等）で管理される。
