RELCACHECALLBACK

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/inval.c:288

目的

リレーションキャッシュ（relcache）の変更イベントが発生した際に実行されるコールバック関数とその引数を保持するための構造体です。

シグネチャまたは形状

static struct RELCACHECALLBACK
{
    RelcacheCallbackFunction function;
    Datum       arg;
} relcache_callback_list[MAX_RELCACHE_CALLBACKS];

入力と出力

  • 入力: RelcacheCallbackFunction 型の関数ポインタおよび Datum 型の引数。
  • 出力: なし（構造体として保持される）。

前提条件と不変条件

  • relcache_callback_list 配列のサイズは MAX_RELCACHE_CALLBACKS によって制限されています。
  • 本構造体は inval.c 内で静的に定義されており、リレーションキャッシュの無効化イベントに関連するコールバックを管理します。

アルゴリズムまたは振る舞い

リレーションキャッシュの無効化処理において、登録されたコールバック関数を順次呼び出すための基盤となります。具体的な実行ロジックは、この構造体の配列を走査する関数によって制御されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/cache/inval.c 内の関数（802行目および860行目付近）において、配列要素へのポインタとして参照されています。
  • 呼び出し先: 登録された RelcacheCallbackFunction が実行時に呼び出されます。

関連データ構造

  • relcache_callback_list: 本構造体を要素とする固定長配列。
  • RelcacheCallbackFunction: コールバック関数の型定義。

失敗モードと境界条件

  • MAX_RELCACHE_CALLBACKS を超える数のコールバックを登録しようとした場合の挙動については、本ソース抜粋からは判断できません。

保守担当者向けメモ

  • 本構造体は inval.c 内で静的に管理されているため、外部モジュールから直接操作することはできません。コールバックの登録・削除を行う関数経由でのアクセスが必要です。
