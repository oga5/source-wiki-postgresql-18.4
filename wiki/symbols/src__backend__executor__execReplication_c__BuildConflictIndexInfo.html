BuildConflictIndexInfo

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execReplication.c:437

目的

論理レプリケーションにおいて、競合が発生した際に使用するインデックスに対して、推測的（speculative）な挿入処理に必要な情報を構築します。

シグネチャまたは形状

static void BuildConflictIndexInfo(ResultRelInfo *resultRelInfo, Oid conflictindex)

入力と出力

  • **入力**:
  • resultRelInfo: 対象テーブルの実行時リレーション情報。
  • conflictindex: 競合が発生したインデックスの OID。
  • **出力**: なし（副作用として resultRelInfo 内のインデックス情報が更新されます）。

前提条件と不変条件

  • resultRelInfo は有効なリレーション情報を含んでいる必要があります。
  • 指定された conflictindex が resultRelInfo に含まれるインデックスのいずれかと一致する必要があります。
  • BuildSpeculativeIndexInfo が同一インデックスに対して二重に呼び出されないこと（indexRelationInfo->ii_UniqueOps == NULL であることがアサートされます）。

アルゴリズムまたは振る舞い

1. resultRelInfo が保持するすべてのインデックスをループで走査します。 2. 現在のインデックスの OID が conflictindex と一致するか確認します。 3. 一致した場合、BuildSpeculativeIndexInfo を呼び出し、当該インデックスに対する推測的挿入用のメタデータを構築します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execReplication.c 内の論理レプリケーション適用処理（競合検出時）。
  • **呼び出し先**: BuildSpeculativeIndexInfo。

関連データ構造

  • ResultRelInfo: 実行時のリレーション情報。
  • IndexInfo: インデックスのメタデータ。
  • Relation: リレーション記述子。

失敗モードと境界条件

  • 指定された conflictindex が resultRelInfo に存在しない場合、何も行わずに終了します。
  • ii_UniqueOps が既に設定されている場合、アサート失敗によりプロセスが停止する可能性があります。

保守担当者向けメモ

  • 本関数は論理レプリケーションの競合処理フローの一部です。インデックスの再構築やメタデータ更新を行うため、同時実行制御下での整合性に注意してください。
