moveLeafs

定義

  • **種別**: function
  • **定義箇所**: src/backend/access/spgist/spgdoinsert.c:386

目的

ページ分割を行うほどではないが、現在のリーフページに空き容量が不足している場合に、リーフチェーン全体を別の新しいリーフページへ移動させ、挿入対象のタプルを収容する。

シグネチャまたは形状

static void moveLeafs(Relation index, SpGistState *state,
                      SPPageDesc *current, SPPageDesc *parent,
                      SpGistLeafTuple newLeafTuple, bool isNulls)

入力と出力

  • **入力**:
  • index: 対象のインデックスリレーション。
  • state: SP-GiSTの操作状態。
  • current: 現在のリーフページおよびタプル情報。
  • parent: 親インナータプル情報。
  • newLeafTuple: 挿入する新しいリーフタプル。
  • isNulls: NULL値を含むかどうかを示すフラグ。
  • **出力**: なし（副作用としてインデックス構造を更新）。

前提条件と不変条件

  • ルートページに対しては実行不可（parent->buffer が InvalidBuffer でないことを要求）。
  • parent->buffer と current->buffer は異なる必要がある。
  • 処理対象のタプル状態は SPGIST_LIVE または SPGIST_DEAD であること。

アルゴリズムまたは振る舞い

1. **対象特定**: 現在のリーフページ内のリーフチェーンを辿り、移動対象のタプルを特定して必要なサイズを計算する。 2. **ページ確保**: SpGistGetBuffer を呼び出し、全タプルを収容可能な新しいリーフページを確保する。 3. **データ移行**:

  • START_CRIT_SECTION() を開始。
  • 既存のライブタプルと新しいタプルを新しいページへコピーし、チェーンリンクを更新する。
  • spgPageIndexMultiDelete を使用して、元のページから移動済みタプルを削除し、リダイレクトまたはプレースホルダーを配置する。
  • saveNodeLink を呼び出し、親インナータプルのダウンリンクを新しいページへ更新する。

    • 4. **WAL記録**: インデックスビルド時を除き、構造変更をWALに記録する。 5. **終了処理**: バッファをダーティにし、ロックを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c 内の挿入ロジック（行2133付近）。
  • **呼び出し先**: SpGistGetBuffer, spgPageIndexMultiDelete, saveNodeLink, SpGistPageAddNewItem, MarkBufferDirty, UnlockReleaseBuffer, XLogInsert 等。

関連データ構造

  • SPPageDesc: 挿入中のページおよびタプル情報を保持。
  • SpGistLeafTuple: リーフタプルの構造体。
  • spgxlogMoveLeafs: WAL記録用の構造体。

失敗モードと境界条件

  • SPGIST_LIVE または SPGIST_DEAD 以外のタプル状態が検出された場合、elog(ERROR) を発生させる。
  • ページ容量不足時は SpGistGetBuffer が適切なページを確保する。
  • インデックスビルド時はリダイレクトを行わず、プレースホルダーを使用する。

保守担当者向けメモ

  • 本関数はページ内のリーフチェーンを物理的に移動させるため、ロック取得順序とWALの整合性に注意が必要。
  • isBuild フラグの状態により、リダイレクト処理の挙動が変化する点に留意すること。
