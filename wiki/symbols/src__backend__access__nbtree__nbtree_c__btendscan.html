btendscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtree.c:469

目的

B-tree インデックススキャンの終了処理を行い、スキャンに関連するリソースを解放する。

シグネチャまたは形状

void btendscan(IndexScanDesc scan)

入力と出力

  • **入力**: IndexScanDesc scan（終了対象のスキャン記述子）
  • **出力**: なし（副作用としてメモリ解放およびバッファのアンピンを実行）

前提条件と不変条件

  • scan->opaque は BTScanOpaque 型としてキャスト可能であること。
  • スキャン中に保持していたバッファピンは、本関数内で適切に解放される。

アルゴリズムまたは振る舞い

1. **バッファの解放**: currPos が有効な場合、未処理の削除済み項目（numKilled）があれば _bt_killitems を呼び出して処理し、バッファのピンを解除する。 2. **マーク位置の解放**: markPos のピンを解除し、markItemIndex をリセットする。 3. **メモリ解放**:

  • keyData が存在すれば pfree する。
  • arrayContext が存在すれば MemoryContextDelete を実行する。
  • killedItems および currTuples が存在すれば pfree する。

    • 4. **構造体の解放**: 最後に BTScanOpaque 構造体自体を pfree する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtree.c 内の amroutine->amendscan に登録されており、インデックスアクセスメソッドの終了時に呼び出される。
  • **呼び出し先**: _bt_killitems（削除済み項目の処理）、BTScanPosUnpinIfPinned（バッファピン解除）、pfree、MemoryContextDelete。

関連データ構造

  • IndexScanDesc: スキャン全体の記述子。
  • BTScanOpaque: B-tree スキャン固有の状態を保持する構造体。

失敗モードと境界条件

  • so->currPos や so->markPos がピンされていない場合、BTScanPosUnpinIfPinned は何もしないため安全である。
  • 各ポインタフィールドが NULL の場合、解放処理はスキップされる。

保守担当者向けメモ

  • so->markTuples は btrescan で再利用される可能性があるため、本関数では解放されない点に注意すること。
  • 並列スキャン時の状態管理については、本関数が並列スキャン記述子のクリーンアップを直接担当しているか、あるいは呼び出し元で別途処理されているかを確認すること（本ソースからは並列状態の明示的な解放処理は確認できない）。
