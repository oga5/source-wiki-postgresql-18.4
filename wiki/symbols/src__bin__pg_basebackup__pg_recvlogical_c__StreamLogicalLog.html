StreamLogicalLog

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_recvlogical.c:214

目的

論理レプリケーションスロットを介したデータストリーミングのメインループを実行し、サーバーから受信した論理デコーディングデータをファイルまたは標準出力へ書き出す。

シグネチャまたは形状

static void StreamLogicalLog(void)

入力と出力

  • **入力**: グローバル変数（conn, startpos, replication_slot, noptions, options, verbose, standby_message_timeout, fsync_interval, outfile, time_to_abort 等）を介して設定を受け取る。
  • **出力**: なし（void）。ただし、ファイルシステムへの書き込みおよびサーバーへのフィードバック送信という副作用を持つ。

前提条件と不変条件

  • conn が確立されていること（未接続の場合は GetConnection() が試行される）。
  • START_REPLICATION ... LOGICAL コマンドがサーバーに対して正常に発行され、PGRES_COPY_BOTH 状態へ移行していること。

アルゴリズムまたは振る舞い

1. **初期化**: output_written_lsn 等の LSN 関連変数を初期化し、サーバーへ START_REPLICATION コマンドを送信する。 2. **メインループ**: time_to_abort が真になるまで以下を繰り返す。

  • **定期処理**: fsync_interval に基づくディスクへのフラッシュ、および standby_message_timeout に基づくサーバーへのフィードバック送信を行う。
  • **ファイル管理**: 出力先ファイルが未オープンであればオープンし、output_reopen フラグに応じて再オープン処理を行う。
  • **データ受信**: PQgetCopyData を使用してサーバーからデータを受信する。データがない場合は select() を用いてソケットを監視し、タイムアウトまで待機する。

    • 3. **終了処理**: ループ終了時、エラー発生時またはシグナル受信時に接続を閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_basebackup/pg_recvlogical.c:1018）。
  • **呼び出し先**:
  • GetConnection, createPQExpBuffer, appendPQExpBuffer, PQexec, PQclear, feGetCurrentTimestamp, feTimestampDifferenceExceeds, OutputFsync, sendFeedback, PQgetCopyData, PQsocket, pg_log_info, pg_log_error 等。

関連データ構造

  • PGresult: libpq のクエリ結果保持用。
  • PQExpBuffer: START_REPLICATION コマンド文字列の構築に使用。
  • XLogRecPtr: 受信位置（LSN）の管理に使用。

失敗モードと境界条件

  • **接続失敗**: GetConnection() が失敗した場合、関数は即座に終了する。
  • **コマンドエラー**: START_REPLICATION が PGRES_COPY_BOTH を返さない場合、エラーログを出力して終了する。
  • **I/O エラー**: ファイルのオープンや fsync に失敗した場合、エラーログを出力して終了する。
  • **ソケットエラー**: PQsocket() が無効な場合、エラーとして処理される。

保守担当者向けメモ

  • 本関数は COPY プロトコルを直接制御しているため、libpq の PQgetCopyData の挙動に強く依存する。
  • fsync のタイミング制御はデータ整合性に直結するため、変更時は fsync_interval の仕様を再確認すること。
  • Windows 環境でのシグナルハンドリングやファイルオープン時の PG_BINARY 指定など、プラットフォーム固有の挙動に注意が必要。
