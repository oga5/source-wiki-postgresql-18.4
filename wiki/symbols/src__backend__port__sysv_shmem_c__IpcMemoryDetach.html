IpcMemoryDetach

定義

  • 種別: function
  • 定義箇所: src/backend/port/sysv_shmem.c:285

目的

プロセスのアドレス空間から System V 共有メモリセグメントをデタッチする。

シグネチャまたは形状

static void IpcMemoryDetach(int status, Datum shmaddr)

入力と出力

  • **入力**:
  • status: on_shmem_exit コールバック規約に従う整数値（未使用）。
  • shmaddr: デタッチ対象の共有メモリセグメントの開始アドレス（Datum 型）。
  • **出力**: なし。

前提条件と不変条件

  • shmaddr は、プロセスが現在アタッチしている有効な System V 共有メモリセグメントのアドレスである必要がある。

アルゴリズムまたは振る舞い

1. DatumGetPointer を用いて shmaddr をポインタに変換する。 2. shmdt() システムコールを呼び出し、該当する共有メモリセグメントをプロセスのアドレス空間から切り離す。 3. shmdt() が負の値を返した場合、elog(LOG, ...) を使用してエラーログを出力する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/sysv_shmem.c 内の on_shmem_exit 登録処理（on_shmem_exit(IpcMemoryDetach, ...)）。
  • **呼び出し先**:
  • shmdt() (システムコール)
  • elog()

関連データ構造

  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • shmdt() が失敗した場合、ログにエラーメッセージが出力されるが、関数自体は終了する。

保守担当者向けメモ

  • 本関数は on_shmem_exit コールバックとして登録されることを前提としている。
  • 共有メモリの削除（shmctl による IPC_RMID）は本関数では行わず、別途 IpcMemoryDelete 等の役割となる。
