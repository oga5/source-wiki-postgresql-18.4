GetReplicationTransferLatency

定義

  • 種別: function
  • 定義箇所: src/backend/replication/walreceiverfuncs.c:394

目的

プライマリサーバーからWALデータを受信する際の、転送レイテンシ（ミリ秒単位）を算出する。

シグネチャまたは形状

int GetReplicationTransferLatency(void)

入力と出力

  • **入力**: なし（共有メモリ上の WalRcv 構造体を参照）
  • **出力**: int 型。プライマリがメッセージを送信した時刻と、スタンバイがそれを受信した時刻の差分（ミリ秒）。

前提条件と不変条件

  • 共有メモリ上の WalRcv 構造体が初期化されていること。
  • 共有メモリへのアクセスは WalRcv->mutex スピンロックによって保護される。

アルゴリズムまたは振る舞い

1. 共有メモリ上のグローバル変数 WalRcv を取得する。 2. WalRcv->mutex スピンロックを獲得する。 3. 構造体内の lastMsgSendTime（最後に受信したメッセージの送信時刻）と lastMsgReceiptTime（そのメッセージの受信時刻）をローカル変数にコピーする。 4. スピンロックを解放する。 5. TimestampDifferenceMilliseconds を使用して、上記2時刻の差分をミリ秒単位で計算し、その結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walreceiver.c 内の関数から呼び出されていることが確認されている。
  • **呼び出し先**:
  • SpinLockAcquire / SpinLockRelease（共有メモリ保護用）
  • TimestampDifferenceMilliseconds（時刻差分計算用）

関連データ構造

  • WalRcvData: 共有メモリ上でWALレシーバーの状態を保持する構造体。本関数はこの中のタイムスタンプフィールドを使用する。

失敗モードと境界条件

  • タイムスタンプがまだ設定されていない場合、あるいはメッセージの送受信が一度も行われていない場合の挙動は、TimestampDifferenceMilliseconds の実装に依存する。

保守担当者向けメモ

  • 本関数は共有メモリのロックを保持する時間が短いため、パフォーマンスへの影響は最小限である。
  • 呼び出し元が walreceiver.c に限定されているかについては、他のモジュールからの参照可能性を考慮し、必要に応じて src/include/replication/walreceiver.h の宣言を確認すること。
