oidsend

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/oid.c:70

目的

Oid 型の値を、クライアントとの通信に適したバイナリ形式（bytea）に変換します。

シグネチャまたは形状

Datum oidsend(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して Oid 型の値を受け取ります。
  • **出力**: bytea 型の Datum を返します。

前提条件と不変条件

  • PG_GETARG_OID(0) を使用して、第1引数が Oid 型として正しく取得できる必要があります。
  • pq_begintypsend および pq_endtypsend を使用したバイナリメッセージ構築のライフサイクルに従う必要があります。

アルゴリズムまたは振る舞い

1. pq_begintypsend を呼び出し、バイナリ送信用のバッファ（StringInfoData）を初期化します。 2. PG_GETARG_OID(0) で取得した Oid 値を、pq_sendint32 を用いてネットワークバイトオーダー（ビッグエンディアン）の4バイト整数としてバッファに書き込みます。 3. pq_endtypsend を呼び出してバッファを bytea 形式に変換し、その結果を PG_RETURN_BYTEA_P で返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/regproc.c 内の複数の関数（regprocsend, regprocsend, regtypesend, regoperatorsend, regclasssend など）から、コード共有の目的で直接呼び出されています。
  • **呼び出し先**:
  • pq_begintypsend
  • pq_sendint32
  • pq_endtypsend

関連データ構造

  • Oid: 変換対象の識別子型。
  • StringInfoData: バイナリメッセージ構築用の一時バッファ。
  • bytea: 出力されるバイナリデータ型。

失敗モードと境界条件

  • メモリ不足時、pq_begintypsend 等の内部関数がエラーを発生させる可能性があります。
  • 入力値が Oid として妥当でない場合、PG_GETARG_OID マクロの仕様に従いエラーが発生します。

保守担当者向けメモ

  • 本関数は Oid だけでなく、regproc や regtype 等の Oid を内部表現とする他の型でも共通のバイナリ変換ロジックとして再利用されています。変更を加える際は、src/backend/utils/adt/regproc.c への影響を考慮してください。
