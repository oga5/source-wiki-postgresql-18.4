目的

PostgreSQL のラージオブジェクト（Large Object）機能の基本的な操作（インポート、読み取り、書き込み、エクスポート）を実演するコマンドラインツールです。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • argv[1]: 接続対象のデータベース名
  • argv[2]: インポートするローカルファイルパス
  • argv[3]: エクスポート先のファイルパス
  • **出力**:
  • 成功時: 0 を返却し、標準出力に操作状況を表示。
  • 失敗時: 標準エラー出力にエラーメッセージを表示し、exit(1) または exit_nicely(conn) を呼び出して終了。

前提条件と不変条件

  • コマンドライン引数として正確に3つの値が渡されること。
  • データベースへの接続が成功すること。
  • ラージオブジェクトの操作は BEGIN から END までのトランザクションブロック内で行われること。

アルゴリズムまたは振る舞い

1. **接続確立**: PQsetdb でデータベースに接続し、PQstatus で接続を確認。 2. **セキュリティ設定**: search_path を空に設定し、不正な操作を防止。 3. **トランザクション開始**: BEGIN コマンドを発行。 4. **インポート**: lo_import を使用してローカルファイルをラージオブジェクトとして登録し、OID を取得。 5. **操作**:

  • pickout: ラージオブジェクトの特定範囲（1000-2000バイト）を読み取り。
  • overwrite: 同範囲を 'X' で上書き。

    • 6. **エクスポート**: lo_export を使用してラージオブジェクトの内容をファイルへ書き出し。 7. **終了処理**: END コマンドでトランザクションをコミットし、PQfinish で接続を閉じて終了。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（エントリーポイント）。
  • **呼び出し先**:
  • libpq API: PQsetdb, PQstatus, PQerrorMessage, PQexec, PQresultStatus, PQclear, PQfinish, lo_import, lo_export
  • 補助関数: exit_nicely, pickout, overwrite
  • **注記**: ツールによる main の呼び出し元検索結果（amcheck 等）は、本ファイルとは無関係なコードベース内の同名シンボルへの参照であり、本関数を呼び出しているものではありません。

関連データ構造

  • PGconn: データベース接続ハンドル。
  • PGresult: クエリ実行結果の格納用。
  • Oid: ラージオブジェクトを一意に識別するオブジェクト識別子。

失敗モードと境界条件

  • 引数の数が不足している場合、使用法を表示して終了。
  • データベース接続に失敗した場合、エラーメッセージを表示して終了。
  • lo_import が失敗（OID 0 を返却）した場合、エラーメッセージを表示。
  • lo_export が失敗（負の値を返却）した場合、エラーメッセージを表示。

保守担当者向けメモ

  • 本関数は libpq のラージオブジェクト API の使用例として設計されています。
  • エラーハンドリングは最小限であるため、実運用コードでは PQerrorMessage を用いたより詳細な診断が必要です。
  • 64ビットオフセットを扱う場合は、src/test/examples/testlo64.c を参照してください。
