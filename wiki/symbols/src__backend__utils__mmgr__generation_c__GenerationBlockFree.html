GenerationBlockFree

定義

  • **種別**: static inline function
  • **定義箇所**: src/backend/utils/mmgr/generation.c:663

目的

GenerationContext が管理するメモリブロックをシステムから解放し、コンテキストのメモリ割り当て統計を更新する。

シグネチャまたは形状

static inline void GenerationBlockFree(GenerationContext *set, GenerationBlock *block)

入力と出力

  • **入力**:
  • set: 対象となる GenerationContext へのポインタ。
  • block: 解放対象の GenerationBlock へのポインタ。
  • **出力**: なし（void）。

前提条件と不変条件

  • block は KeeperBlock（コンテキスト作成時に確保される永続ブロック）であってはならない。
  • block は現在 set->freeblock として保持されているブロックであってはならない。
  • 上記条件は Assert により実行時に検証される。

アルゴリズムまたは振る舞い

1. IsKeeperBlock および freeblock との重複チェックを行い、不正な解放を防止する。 2. dlist_delete を呼び出し、対象ブロックをコンテキストのブロックリストから除外する。 3. set の mem_allocated 統計から、解放するブロックのサイズ（block->blksize）を減算する。 4. CLOBBER_FREED_MEMORY が定義されている場合、wipe_mem を使用して解放領域を上書きする。 5. free(block) を呼び出し、メモリをシステムに返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/mmgr/generation.c 内の GenerationFree およびコンテキストのクリーンアップ処理から呼び出される。
  • **呼び出し先**:
  • dlist_delete (lib/ilist.h)
  • free (標準ライブラリ)
  • wipe_mem (デバッグ時のみ)

関連データ構造

  • GenerationContext: メモリコンテキストの管理構造体。
  • GenerationBlock: メモリ割り当ての単位となるブロック構造体。
  • dlist: GenerationBlock を管理する双方向リスト。

失敗モードと境界条件

  • KeeperBlock や freeblock を誤って渡した場合、Assert が有効なビルドではプロセスが停止する。
  • block が無効なポインタである場合、dlist_delete または free の実行時に未定義動作が発生する。

保守担当者向けメモ

  • 本関数はブロックを即座に free() するため、再利用を目的とした freeblock への移動とは明確に区別されている。
  • mem_allocated の更新は、メモリ使用量の正確な追跡に不可欠であるため、変更時は注意すること。
