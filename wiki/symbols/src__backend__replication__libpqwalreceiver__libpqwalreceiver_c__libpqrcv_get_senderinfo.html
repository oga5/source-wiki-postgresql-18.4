libpqrcv_get_senderinfo

定義

  • 種別: function
  • 定義箇所: src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:396

目的

アクティブなレプリケーション接続（WalReceiverConn）から、接続先プライマリサーバーのホスト名およびポート番号を取得する。

シグネチャまたは形状

static void libpqrcv_get_senderinfo(WalReceiverConn *conn, char **sender_host, int *sender_port)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する WalReceiverConn 構造体へのポインタ。
  • sender_host: ホスト名を格納する文字列ポインタの格納先（出力用）。
  • sender_port: ポート番号を格納する整数ポインタの格納先（出力用）。
  • **出力**:
  • なし（引数経由で値を書き込む）。sender_host には pstrdup された文字列が、sender_port には数値が設定される。

前提条件と不変条件

  • conn が指す WalReceiverConn 構造体内の streamConn が NULL であってはならない（Assert により検証）。

アルゴリズムまたは振る舞い

1. 出力引数 sender_host を NULL、sender_port を 0 に初期化する。 2. conn->streamConn を使用して libpq の PQhost() を呼び出し、ホスト名を取得する。値が存在し空文字列でなければ、pstrdup を用いてメモリを確保し sender_host に格納する。 3. 同様に PQport() を呼び出し、ポート番号を取得する。値が存在し空文字列でなければ、atoi で整数に変換し sender_port に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: PQWalReceiverFunctions 構造体を通じて、レプリケーションサブシステムから間接的に呼び出される（src/backend/replication/libpqwalreceiver/libpqwalreceiver.c:102）。
  • **呼び出し先**: libpq ライブラリの PQhost および PQport 関数。

関連データ構造

  • WalReceiverConn: 接続状態（PGconn）を保持する構造体。

失敗モードと境界条件

  • streamConn が NULL の場合、アサーション違反によりプロセスが異常終了する可能性がある。
  • PQhost または PQport が空文字列を返した場合、出力引数は初期値（NULL または 0）のままとなる。

保守担当者向けメモ

  • sender_host には pstrdup でメモリが割り当てられるため、呼び出し元で適切に解放する必要がある。
  • 本関数は libpq の接続状態に依存しているため、接続が確立されていない状態での呼び出しは避けること。
