spg_quad_picksplit

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgquadtreeproc.c:168

目的

SP-GiSTインデックスのノード分割時において、重心（centroid）を基準に点データを4つの象限に分類し、子ノードへのマッピングを決定する。

シグネチャまたは形状

Datum spg_quad_picksplit(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgPickSplitIn（入力タプル群）および spgPickSplitOut（出力構造体）を受け取る。
  • **出力**: spgPickSplitOut 構造体の各フィールド（hasPrefix, prefixDatum, nNodes, mapTuplesToNodes, leafTupleDatums）を更新する。戻り値は void。

前提条件と不変条件

  • in->nTuples 個の点データが提供されていること。
  • USE_MEDIAN マクロの定義有無により、重心計算アルゴリズム（中央値または平均値）が切り替わる。
  • 象限は1から4まで定義され、軸上の点は「より番号の小さい象限」に分類される。

アルゴリズムまたは振る舞い

1. **重心の算出**:

  • USE_MEDIAN が定義されている場合、X座標とY座標それぞれのソート済み中央値を用いて重心を決定する。
  • 定義されていない場合、全点の座標平均値を重心とする。

    • 2. **プレフィックス設定**: 算出した重心を out->prefixDatum に格納し、out->hasPrefix を true に設定する。 3. **ノード構成**: out->nNodes を4に設定し、ノードラベルは使用しない（NULL）。 4. **タプル割り当て**: 各入力タプルに対し、getQuadrant 関数を用いて所属する象限（0〜3のインデックス）を特定し、out->mapTuplesToNodes に記録する。同時に out->leafTupleDatums を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTのインデックス構築プロセス（具体的な呼び出し元関数は提供されたソースからは特定不可）。
  • **呼び出し先**: getQuadrant（象限判定用）、x_cmp / y_cmp（中央値計算時のソート用）、palloc 等のメモリ管理関数。

関連データ構造

  • spgPickSplitIn, spgPickSplitOut: SP-GiSTの分割インターフェース用構造体。
  • Point: 2次元座標データ。

失敗モードと境界条件

  • in->nTuples が0の場合の挙動については、本ソースからは明示的なガード条件は確認できない。
  • メモリ不足時には palloc がエラーを発生させる可能性がある。

保守担当者向けメモ

  • USE_MEDIAN の定義により計算コストが変動するため、パフォーマンスチューニング時は注意すること。
  • 象限の定義（1〜4）と getQuadrant が返す値の整合性が維持されていることを前提としている。
