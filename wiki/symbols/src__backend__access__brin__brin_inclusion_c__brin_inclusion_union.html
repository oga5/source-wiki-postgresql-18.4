brin_inclusion_union

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_inclusion.c:473

目的

2つのブロック範囲（BrinValues）の集約値を結合し、包含インデックスにおける新しい集約値を算出する。

シグネチャまたは形状

Datum brin_inclusion_union(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): BrinDesc *（インデックス記述子）
  • PG_GETARG_POINTER(1): BrinValues *col_a（結合先となる集約値）
  • PG_GETARG_POINTER(2): BrinValues *col_b（結合元となる集約値）
  • **出力**: なし（PG_RETURN_VOID()）。結果は col_a の bv_values 配列を直接更新することで反映される。

前提条件と不変条件

  • col_a->bv_attno == col_b->bv_attno であること。
  • col_a および col_b の bv_allnulls が false であること。

アルゴリズムまたは振る舞い

1. **空要素の伝播**: col_b が空要素を含む場合、col_a の INCLUSION_CONTAINS_EMPTY フラグを true に設定する。 2. **結合不可状態の伝播**: col_a または col_b が既に INCLUSION_UNMERGEABLE（結合不可）状態であれば、col_a をその状態に維持して終了する。 3. **結合可能性の検証**: PROCNUM_MERGEABLE 関数を呼び出し、両者の値が結合可能か判定する。結合不可と判断された場合、col_a の INCLUSION_UNMERGEABLE を true に設定して終了する。 4. **和集合の計算**: PROCNUM_MERGE 関数を呼び出し、col_a と col_b の値を結合する。 5. **メモリ管理**: 結果がポインタ型（attbyval が偽）の場合、古い値を解放し、必要に応じて新しい値をコピーして col_a->bv_values[INCLUSION_UNION] に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は確認されていません。
  • **呼び出し先**:
  • inclusion_get_procinfo: 結合可能性判定および和集合計算のための関数を取得。
  • FunctionCall2Coll: 取得した PROCNUM_MERGEABLE および PROCNUM_MERGE 関数を実行。

関連データ構造

  • BrinValues: ブロック範囲の集約値を保持する構造体。
  • InclusionOpaque: インデックスのメタデータ。
  • CompactAttribute: 属性の物理的特性（attbyval, attlen 等）。

失敗モードと境界条件

  • PROCNUM_MERGEABLE が false を返した場合、その範囲は結合不可としてマークされ、以降の集約処理から除外される。
  • attbyval が偽のデータ型において、メモリの再割り当てやコピーに失敗した場合はシステムエラーとなる可能性がある。

保守担当者向けメモ

  • 本関数は brin_inclusion_add_value 等と同様に、INCLUSION_UNMERGEABLE および INCLUSION_CONTAINS_EMPTY の状態管理を厳密に行う必要がある。
  • PROCNUM_MERGE の結果が既存のポインタと同一である場合のメモリ解放処理（pfree）は、datumCopy を伴うため慎重に扱うこと。
