ResetUnloggedRelationsInDbspaceDir

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/reinit.c:160

目的

指定されたデータベースディレクトリ内のアンログド（unlogged）リレーションに対し、初期化フォーク（_init）に基づいたクリーンアップおよび再構築を行う。

シグネチャまたは形状

static void ResetUnloggedRelationsInDbspaceDir(const char *dbspacedirname, int op)

入力と出力

  • dbspacedirname: 処理対象のデータベースディレクトリパス。
  • op: 実行する操作のビットマスク。UNLOGGED_RELATION_CLEANUP（不要なフォークの削除）および UNLOGGED_RELATION_INIT（_init フォークからの復元）を指定可能。
  • 戻り値: なし。

前提条件と不変条件

  • op には UNLOGGED_RELATION_CLEANUP または UNLOGGED_RELATION_INIT の少なくとも一方が含まれている必要がある。
  • _init フォークは決して削除されない。
  • ファイル名解析は parse_filename_for_nontemp_relation を介して厳格に行われ、一時ファイルは対象外となる。

アルゴリズムまたは振る舞い

1. **クリーンアップ処理 (UNLOGGED_RELATION_CLEANUP)**:

  • 2パス方式で実行。第1パスでディレクトリを走査し、_init フォークを持つリレーションの OID をハッシュテーブルに記録する。
  • ハッシュテーブルが空であれば処理を終了する。
  • 第2パスで再度ディレクトリを走査し、ハッシュテーブルに存在する OID に一致する非 _init フォークファイルを unlink で削除する。

    • 2. **初期化処理 (UNLOGGED_RELATION_INIT)**:

  • クリーンアップ完了後（またはクリーンアップがスキップされた場合）に実行。
  • ディレクトリを走査し、_init フォークファイルを対応するメインフォークファイルへ copy_file で複製する。

呼び出し元と呼び出し先

  • 呼び出し元: ResetUnloggedRelations (src/backend/storage/file/reinit.c:151)。
  • 呼び出し先: AllocateDir, ReadDir, FreeDir, parse_filename_for_nontemp_relation, hash_create, hash_search, hash_destroy, unlink, copy_file, ereport, elog。

関連データ構造

  • unlogged_relation_entry: クリーンアップ対象のリレーションを追跡するためのハッシュテーブル用構造体。

失敗モードと境界条件

  • unlink 失敗時: ereport(ERROR, ...) により処理を中断し、エラーを報告する。
  • _init フォークが存在しない場合: クリーンアップ処理はハッシュテーブルが空であることを検知し、早期リターンする。

保守担当者向けメモ

  • クリーンアップ時のハッシュテーブル使用により、多数のリレーションが存在する場合でも O(N) の計算量で効率的に処理される。
  • copy_file 後の fsync は、リカバリプロセスにおいてチェックポイントが機能しない状況を考慮した設計である。
