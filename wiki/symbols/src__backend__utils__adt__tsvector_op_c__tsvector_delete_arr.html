tsvector_delete_arr

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector_op.c:577

目的

指定された tsvector から、配列に含まれる語彙（lexemes）を削除した新しい tsvector を生成する。

シグネチャまたは形状

Datum tsvector_delete_arr(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TSVECTOR(0): 処理対象の TSVector。
  • PG_GETARG_ARRAYTYPE_P(1): 削除対象の語彙を含む text[] 配列。
  • **出力**:
  • PG_RETURN_POINTER(tsout): 削除後の新しい TSVector を返す。

前提条件と不変条件

  • 入力配列内の NULL 要素は無視される。
  • tsvector は可変長データ型であるため、操作後に適切なメモリ管理とサイズ更新が行われる必要がある。

アルゴリズムまたは振る舞い

1. deconstruct_array_builtin を使用して、入力配列を Datum 配列に分解する。 2. 削除対象の各語彙に対し、tsvector_bsearch を用いて tsin 内のインデックスを特定する。 3. 特定されたインデックスを skip_indices 配列に格納する。 4. tsvector_delete_by_indices を呼び出し、該当するインデックスを除外した新しい tsvector を構築する。 5. 確保したメモリ（skip_indices）を解放し、入力引数のコピーを適切に解放して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 外部からの SQL 呼び出し（直接的な C 言語レベルの呼び出し元情報は提供されていない）。
  • **呼び出し先**:
  • deconstruct_array_builtin
  • tsvector_bsearch
  • tsvector_delete_by_indices
  • palloc0 / pfree

関連データ構造

  • TSVector: 全文検索用ベクトル型。
  • ArrayType: 入力語彙を保持する配列型。

失敗モードと境界条件

  • 入力配列が空の場合、削除は行われず元の tsvector が返される。
  • 存在しない語彙が配列に含まれている場合、その語彙は無視される（tsvector_bsearch が負の値を返すため）。

保守担当者向けメモ

  • 本関数は、削除対象の語彙数が少ない場合に効率的となるようバイナリサーチを採用している。
  • tsvector_delete_by_indices が内部で新しいメモリ領域を確保するため、メモリリークを防ぐために pfree の呼び出し順序に注意すること。
