ExecHashSubPlan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSubplan.c:100

目的

サブクエリの結果をメモリ上のハッシュテーブルに保持し、式評価の一部として高速なルックアップ（IN, ANY 等のサブリンク処理）を行う。

シグネチャまたは形状

static Datum ExecHashSubPlan(SubPlanState *node, ExprContext *econtext, bool *isNull)

入力と出力

  • **入力**:
  • node: SubPlanState 構造体（ハッシュテーブルや投影情報を含む）。
  • econtext: 式評価用のコンテキスト。
  • **出力**:
  • 戻り値: Datum 型のブール値（TRUE または FALSE）。
  • isNull: bool へのポインタ。結果が UNKNOWN（NULL）となる場合に true が設定される。

前提条件と不変条件

  • サブクエリに直接的な相関（parParam や args）が含まれていないこと。
  • 結合演算子は、両方の入力が非NULLの場合にNULLを返さないと仮定される。
  • 呼び出しごとに econtext を使用して投影を行い、終了時に ExecClearTuple でスロットをクリアし、hashtempcxt をリセットする必要がある。

アルゴリズムまたは振る舞い

1. **初期化**: ハッシュテーブルが未作成、またはプランの状態が変更されている場合、buildSubPlanHash を呼び出してハッシュテーブルを構築する。 2. **空判定**: サブクエリの結果が空の場合、即座に FALSE を返す。 3. **投影**: node->projLeft を使用して、左辺（LHS）の式を評価しタプルを形成する。 4. **ルックアップ**:

  • **LHSが非NULL**: メインハッシュテーブルを検索し、一致すれば TRUE。一致せず、かつNULLを含む行が存在する場合は findPartialMatch で UNKNOWN を判定する。
  • **LHSがNULL**: TRUE は返せない。UNKNOWN の判定が必要な場合、メインおよび部分NULLテーブルをスキャンし、一致すれば UNKNOWN を返す。

    • 5. **後処理**: ExecClearTuple でスロットを解放し、MemoryContextReset で一時メモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ExecSubPlan（src/backend/executor/nodeSubplan.c 内のロジックより推測）。
  • **呼び出し先**:
  • buildSubPlanHash
  • ExecProject
  • slotNoNulls / slotAllNulls
  • FindTupleHashEntry
  • findPartialMatch
  • ExecClearTuple
  • MemoryContextReset

関連データ構造

  • SubPlanState: サブプランの実行状態とハッシュテーブルを保持する主要構造体。
  • SubPlan: 実行対象のサブクエリ定義。
  • TupleTableSlot: 投影されたタプルを保持する。

失敗モードと境界条件

  • **エラー**: 直接的な相関（parParam または args が存在）がある場合、elog(ERROR) を発生させる。
  • **境界条件**:
  • サブクエリが空の場合、常に FALSE を返す。
  • LHSがNULLの場合、三値論理に基づき UNKNOWN または FALSE を返す。

保守担当者向けメモ

  • 本関数は econtext を一時的に書き換える「ハック」を行っているため、投影処理の前後関係には注意が必要。
  • hashtempcxt のリセットは、メモリリークを防ぐために必須である。
