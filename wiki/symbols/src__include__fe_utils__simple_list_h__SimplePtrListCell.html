SimplePtrListCell

定義

  • 種別: struct
  • 定義箇所: src/include/fe_utils/simple_list.h:46

目的

PostgreSQL のフロントエンドコードにおいて、汎用ポインタ（void *）を格納するための単方向連結リストの各要素（セル）を保持します。

シグネチャまたは形状

typedef struct SimplePtrListCell
{
    struct SimplePtrListCell *next;
    void       *ptr;
} SimplePtrListCell;

入力と出力

  • **入力**: リストへの追加時に void * 型のポインタを受け取ります。
  • **出力**: リスト走査時に ptr メンバを通じて格納されたポインタを返します。

前提条件と不変条件

  • next はリストの次のセルを指すか、末尾の場合は NULL です。
  • 本構造体自体は pg_malloc で動的に確保されます。
  • 格納される ptr が指すメモリ領域の管理は、本構造体ではなく呼び出し側の責務です。

アルゴリズムまたは振る舞い

  • リストの末尾への追加（append）時に新しいセルとして割り当てられます。
  • リストの破棄（destroy）時には、各セルが pg_free で解放されますが、ptr が指す先のメモリは解放されません。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/simple_list.c 内のリスト操作関数（append, destroy 等）から直接参照されます。また、pg_amcheck.c、pg_dump.c、pg_verifybackup.c 等のフロントエンドツールでリスト走査用ポインタとして使用されています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、特定の関数を呼び出すことはありません。

関連データ構造

  • SimplePtrList: 本セルをリストとして管理する構造体（head および tail ポインタを保持）。

失敗モードと境界条件

  • メモリ不足時には pg_malloc がエラーを発生させます。
  • リストが空の場合、head および tail は NULL となります。

保守担当者向けメモ

  • バックエンドの List とは異なり、非常に軽量な実装です。複雑な操作が必要な場合は、他のデータ構造を検討してください。
  • ptr が指すメモリのライフサイクル管理は本構造体に含まれないため、メモリリークには注意が必要です。
