gistvacuumpage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistvacuum.c:307

目的

GiSTインデックスの単一ページに対する VACUUM 処理を実行します。不要なタプルの削除、空ページの特定、およびインデックス分割に伴う再帰的なページ走査を行います。

シグネチャまたは形状

static void gistvacuumpage(GistVacState *vstate, Buffer buffer)

入力と出力

  • **入力**:
  • vstate: GistVacState 構造体へのポインタ。スキャン状態、コールバック関数、統計情報、空ページ管理用セットを保持。
  • buffer: 処理対象のインデックスページを保持するバッファ。
  • **出力**: なし（副作用としてバッファの更新、統計情報の加算、空ページセットへの登録を行う）。

前提条件と不変条件

  • vstate が有効であること。
  • buffer はロックされていない状態で渡される（関数内で GIST_EXCLUSIVE ロックを取得する）。
  • IntegerSet に追加するブロック番号は昇順である必要があるため、再帰呼び出し時はセットへの追加をスキップする。

アルゴリズムまたは振る舞い

1. **ロックと初期化**: 対象ページを排他ロックし、ページ状態を確認。 2. **ページ種別ごとの処理**:

  • **リサイクル可能/削除済み**: 統計情報を更新。
  • **リーフページ**:
  • 分割発生時の再帰チェック: NSN や rightlink を確認し、必要に応じて過去のページへ戻る（goto restart によるループ処理）。
  • タプル削除: callback を使用して不要タプルを特定し、PageIndexMultiDelete で削除。WAL 記録を行う。
  • 空ページ判定: 全タプル削除後に空になった場合、vstate->empty_leaf_set に登録。
  • **内部ページ**:
  • 不正タプルチェック: 旧バージョン由来の無効なタプルを検出しログ出力。
  • ページセット登録: 後段の gistvacuum_delete_empty_pages 用に vstate->internal_page_set に登録。

    • 3. **終了**: バッファのロックを解放。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gist/gistvacuum.c 内の gistvacuumscan から呼び出される。
  • **呼び出し先**:
  • RecordFreeIndexPage, intset_add_member (データ構造操作)
  • PageIndexMultiDelete, GistMarkTuplesDeleted (ページ操作)
  • gistXLogUpdate (WAL 記録)
  • callback (ユーザー定義の削除判定関数)

関連データ構造

  • GistVacState: スキャン中の状態管理。
  • GISTPageOpaque: GiST ページヘッダー情報（rightlink, NSN など）。
  • IntegerSet: 空ページ番号の管理に使用。

失敗モードと境界条件

  • **再帰的走査**: インデックス分割によりタプルが移動した場合、過去のページへ戻る必要がある。スタックオーバーフローを避けるため、再帰呼び出しではなく goto を用いたループ構造を採用している。
  • **不正タプル**: 過去のクラッシュリカバリに起因する無効な内部タプルが存在する場合、ログを出力し REINDEX を推奨する。

保守担当者向けメモ

  • 本関数は VACUUM の中心的なページ処理ロジックです。goto restart はスタック消費を抑えるための最適化であり、変更時は注意してください。
  • ページが空になった際の IntegerSet への追加は、昇順制約のため非再帰的な初回訪問時のみに行われます。
