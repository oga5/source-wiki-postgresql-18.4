BufferLookupEnt

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/buffer/buf_table.c:27

目的

共有バッファプールにおいて、ディスク上のページ（BufferTag）とメモリ上のバッファスロット（buffer ID）を対応付けるためのハッシュテーブルエントリを定義します。

シグネチャまたは形状

typedef struct
{
	BufferTag	key;			/* Tag of a disk page */
	int			id;				/* Associated buffer ID */
} BufferLookupEnt;

入力と出力

本構造体はデータ保持用であり、関数ではありません。ハッシュテーブル操作時にキーとして key（BufferTag）が入力され、値として id（buffer ID）が読み書きされます。

前提条件と不変条件

  • key はディスク上のページを一意に識別します。
  • id は有効なバッファスロット番号を指します。
  • 本構造体を含むハッシュテーブルの操作には、呼び出し側による BufMappingLock の適切な管理が必須です。

アルゴリズムまたは振る舞い

ハッシュテーブルの各エントリとして、特定のページタグとバッファIDのペアを保持します。buf_table.c 内の検索、挿入、削除操作において、この構造体のサイズがハッシュテーブルのメモリ確保やエントリ管理の基準として使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/buf_table.c 内の BufTableShmemSize（サイズ計算）、BufTableLookup、BufTableInsert、BufTableDelete が本構造体を参照・操作します。
  • **呼び出し先**: 本構造体自体はデータ構造であり、関数を呼び出しません。

関連データ構造

  • BufferTag: key フィールドの型。ディスク上のページを特定するために使用されます。
  • SharedBufHash: 本構造体を要素として格納する共有メモリ上のハッシュテーブル。

失敗モードと境界条件

  • ハッシュテーブル操作時にエントリが見つからない場合、BufTableDelete 等の関数は共有メモリ構造の破損と見なし elog(ERROR) を発生させます。
  • 構造体自体の境界条件は、ハッシュテーブルの容量および BufferTag の有効性に依存します。

保守担当者向けメモ

  • 本構造体は buf_table.c のハッシュテーブル管理専用です。
  • 構造体のメンバを変更する場合は、SharedBufHash の初期化および関連するハッシュ検索ロジックへの影響を考慮してください。
