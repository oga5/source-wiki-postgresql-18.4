bbsink_end_archive

定義

  • 種別: static inline function
  • 定義箇所: src/include/backup/basebackup_sink.h:215

目的

バックアップアーカイブのデータ転送終了を通知し、シンクチェーンにおけるアーカイブ処理の完了フェーズを実行する。

シグネチャまたは形状

void bbsink_end_archive(bbsink *sink)

入力と出力

  • **入力**: sink (bbsink *) - 操作対象のシンク構造体へのポインタ。
  • **出力**: なし。

前提条件と不変条件

  • sink は NULL であってはならない (Assert(sink != NULL) により検証)。
  • sink->bbs_ops が有効であり、かつ end_archive コールバック関数が実装されている必要がある。

アルゴリズムまたは振る舞い

1. 引数として渡された sink が NULL でないことを確認する。 2. sink が保持する bbs_ops（操作テーブル）内の end_archive 関数ポインタを呼び出し、当該シンク固有の終了処理を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c (複数箇所)
  • src/backend/backup/basebackup_sink.c (シンクチェーンの委譲処理)
  • **呼び出し先**:
  • sink->bbs_ops->end_archive (シンクの具体的な実装に応じた関数)

関連データ構造

  • bbsink: シンクの基本構造体。
  • bbsink_ops: 各種バックアップイベントに対するコールバック関数を保持するテーブル。

失敗モードと境界条件

  • sink が NULL の場合、アサーション違反によりプロセスが停止する。
  • sink->bbs_ops が初期化されていない、または end_archive が NULL の場合、セグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数はデコレータパターンにおける「終了」イベントの伝播を担う。チェーン内の各シンクで処理を完結させた後、必要に応じて bbsink_forward_end_archive 等を通じて次段へ通知を伝播させる設計となっている。
