SN_env

定義

  • 種別: struct
  • 定義箇所: src/include/snowball/libstemmer/api.h:14

目的

Snowball ステミングアルゴリズムの実行に必要な状態（コンテキスト）を保持する構造体です。

シグネチャまたは形状

struct SN_env {
    symbol * p;      // 現在の文字列バッファ
    int c;           // 現在のカーソル位置
    int l;           // 文字列の長さ
    int lb;          // 文字列の境界位置
    int bra;         // ステミング対象範囲の開始位置
    int ket;         // ステミング対象範囲の終了位置
    symbol * * S;    // 動的メモリ領域（文字列用）
    int * I;         // 整数領域（作業用変数）
};

入力と出力

  • 入力: SN_create_env により初期化され、SN_set_current を通じて処理対象の文字列がロードされます。
  • 出力: ステミング処理の結果は、構造体内のバッファ p に反映されます。

前提条件と不変条件

  • SN_create_env によってメモリが確保されている必要があります。
  • ステミング処理中は、カーソル位置 c や境界 bra, ket がアルゴリズムのルールに従って動的に更新されます。

アルゴリズムまたは振る舞い

1. **環境構築**: SN_create_env で作業用メモリ（S, I）が確保されます。 2. **データロード**: SN_set_current で対象文字列がバッファにコピーされ、カーソルがリセットされます。 3. **ステミング実行**: 各言語固有のステミング関数が本構造体を参照し、カーソルや境界を操作して文字列を変換します。 4. **終了処理**: SN_close_env により確保されたメモリが解放されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/dict_snowball.c および各言語のステミング関数（例: basque_ISO_8859_1_stem）。
  • 呼び出し先: calloc（api.c 内で環境生成時に使用）。

関連データ構造

  • symbol: ステミング対象の文字型（デフォルトは unsigned char）。
  • header.h: メモリ配置やアライメントに関連する定数定義。

失敗モードと境界条件

  • メモリ確保失敗: SN_create_env での calloc 失敗時に NULL が返される可能性があります。
  • スレッドセーフ性: 本構造体は動的な状態を保持するため、マルチスレッド環境ではスレッドごとに独立したインスタンスが必要です。

保守担当者向けメモ

  • symbol 型を変更する場合は、header.h で定義される HEAD とのアライメント制約に注意してください。
  • 構造体内の各フィールドはステミングアルゴリズムの実行中に頻繁に書き換えられるため、外部からの直接操作には注意が必要です。
