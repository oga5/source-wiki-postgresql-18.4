print_gen

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/geqo/geqo_misc.c:90

目的

遺伝的アルゴリズム（GEQO）の特定の世代における個体群の適応度統計（最良、最悪、中央値、平均）を、指定されたファイルストリームへ出力する。

シグネチャまたは形状

void print_gen(FILE *fp, Pool *pool, int generation)

入力と出力

  • fp: 出力先のファイルストリーム。
  • pool: 統計対象となる Pool 構造体へのポインタ。
  • generation: 現在の世代番号。
  • 出力: fp に対して、世代番号と適応度統計を整形した文字列を書き出し、バッファをフラッシュする。

前提条件と不変条件

  • GEQO_DEBUG マクロが定義されている環境でのみ有効。
  • pool は有効な Pool 構造体を指している必要がある。
  • pool->data 配列は、適応度順にソートされていることを前提としている（インデックス0が最良、pool->size - 2 が最悪の個体として扱われるため）。

アルゴリズムまたは振る舞い

1. pool->size に基づき、最悪の個体を示すインデックス lowest を算出する（バッファ領域を考慮し size - 2 を使用）。 2. fprintf を使用し、以下の値を一行で出力する:

  • generation
  • pool->data[0].worth (最良)
  • pool->data[lowest].worth (最悪)
  • pool->data[pool->size / 2].worth (中央値)
  • avg_pool(pool) の戻り値 (平均)

    • 3. fflush(fp) を呼び出し、出力を即座に確定させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/optimizer/geqo/geqo_main.c:229
  • 呼び出し先: avg_pool (同ファイル内のヘルパー関数)

関連データ構造

  • Pool: 個体群を管理する構造体。data 配列および size メンバを参照する。

失敗モードと境界条件

  • pool->size が 1 以下の場合、lowest は 0 となり、最良と最悪の個体が同一とみなされる。
  • fp が無効なストリームである場合、書き込みエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はデバッグ目的の補助ルーチンである。本番環境のパフォーマンスに影響を与えないよう、呼び出し側で GEQO_DEBUG マクロによるガードを行うこと。
  • pool->data のソート順序が崩れている場合、出力される統計値が不正確になる点に注意。
