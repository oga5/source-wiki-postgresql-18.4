pg_clear_relation_stats

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/relation_stats.c:201

目的

テーブル（リレーション）の統計情報をデフォルト値にリセットするためのSQL関数インターフェース。

シグネチャまたは形状

Datum pg_clear_relation_stats(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQL経由で渡される引数）。
  • **出力**: PG_RETURN_VOID() により、戻り値なし（void）。

前提条件と不変条件

  • relation_statistics_update を呼び出すため、同関数が要求する ShareUpdateExclusiveLock の取得およびリカバリ状態のチェックが内部的に行われる必要がある。

アルゴリズムまたは振る舞い

1. LOCAL_FCINFO を使用して、6つの引数を持つ関数呼び出し情報をスタック上に初期化する。 2. 呼び出し元から受け取った最初の2つの引数（リレーション識別子等）をそのまま引き継ぐ。 3. 残りの4つの統計用引数に対して、リセット用のデフォルト値を設定する。

  • relpages: 0
  • reltuples: -1.0
  • relallvisible: 0
  • relallfrozen: 0

    • 4. 構築した引数情報を relation_statistics_update に渡し、pg_class の統計情報を更新する。 5. 処理完了後、voidを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（直接の呼び出し元情報はソースコード上では明示されていない）。
  • **呼び出し先**: relation_statistics_update（統計更新のコアロジック）。

関連データ構造

  • FunctionCallInfoData: 関数呼び出し情報を保持する構造体。
  • pg_class: 更新対象となるシステムカタログ。

失敗モードと境界条件

  • リカバリ中の実行は禁止されており、relation_statistics_update 内部でエラーが発生する可能性がある。
  • 対象リレーションに対するロック取得に失敗した場合、デッドロックまたはエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は relation_statistics_update へのラッパーであり、統計リセット時の固定値を定義している。統計カラムの仕様変更時は、本関数の引数設定値と relation_statistics_update の整合性を確認すること。
