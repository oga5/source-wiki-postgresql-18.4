thesaurus_lexize

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/dict_thesaurus.c:787

目的

シソーラス辞書を用いて、入力されたトークンストリームに対してフレーズ置換処理を行い、検索用のトークン列を生成する。

シグネチャまたは形状

Datum thesaurus_lexize(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): DictThesaurus 構造体（辞書定義）。
  • PG_GETARG_POINTER(3): DictSubState 構造体（マッチング状態）。
  • PG_GETARG_DATUM(1), PG_GETARG_DATUM(2): 辞書処理用の引数。
  • **出力**:
  • TSLexeme*: 置換後のトークン列、またはマッチしない場合は NULL。

前提条件と不変条件

  • 引数の数が4つであること、および dstate が NULL でないことが必須。
  • dstate->isend が真の場合、即座に NULL を返す。
  • 内部で使用するサブ辞書（subdict）が有効でない場合、キャッシュから再取得する。

アルゴリズムまたは振る舞い

1. **サブ辞書による正規化**: subdict を呼び出し、入力単語を正規化する。 2. **マッチング探索**:

  • 正規化された単語が辞書内のルールに合致するかを findVariant で探索する。
  • ストップワードの場合は findTheLexeme(d, NULL) を使用して処理する。

    • 3. **状態更新**: 探索結果（LexemeInfo）を dstate->private_state に保存する。 4. **置換実行**: checkMatch を呼び出し、合致したフレーズを置換後のトークン列に変換する。 5. **制御フラグ**: dstate->getnext を更新し、後続のトークン処理が必要かどうかを制御する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は特定されていない。
  • **呼び出し先**: lookup_ts_dictionary_cache, FunctionCall4 (サブ辞書の lexize), findTheLexeme, findVariant, checkMatch。

関連データ構造

  • DictThesaurus: シソーラス辞書のメイン構造体。
  • DictSubState: フレーズマッチングの進行状態を保持する構造体。
  • LexemeInfo: 辞書内の単語情報を保持する構造体。
  • TSLexeme: 全文検索用のトークン構造体。

失敗モードと境界条件

  • PG_NARGS() != 4 または dstate == NULL の場合、ERROR を発生させる。
  • 単語が認識できない場合、info を NULL に設定し、dstate->getnext を false にして終了する。
  • 辞書ルールに合致しない場合、NULL を返す。

保守担当者向けメモ

  • 本関数はフレーズを扱うため、単一の単語ではなく DictSubState を介した状態遷移が重要である。
  • dstate->private_state に格納される LexemeInfo は、マッチングの進捗を保持するために不可欠である。
