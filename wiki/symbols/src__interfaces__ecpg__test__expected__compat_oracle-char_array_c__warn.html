warn

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/compat_oracle-char_array.c:51

目的

ECPG（Embedded SQL for C）のテスト実行時において、データ取得時にカラムの切り詰め（truncation）が発生したことを示す警告メッセージを標準エラー出力へ通知する。

シグネチャまたは形状

static void warn(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用として標準エラー出力へ文字列を出力）

前提条件と不変条件

  • 本関数は static で定義されており、定義ファイル内でのみ呼び出し可能である。
  • 呼び出し元は、SQL実行後の警告状態（例: sqlca.sqlwarn[0] == 'W'）を検知している必要がある。

アルゴリズムまたは振る舞い

1. fprintf を使用して、標準エラー出力（stderr）に "Warning: At least one column was truncated\n" という固定メッセージを出力する。 2. 処理を終了し、呼び出し元へ制御を戻す。

呼び出し元と呼び出し先

  • 呼び出し元: 提供されたソースコードの断片からは、具体的な呼び出し箇所は特定できない。
  • 呼び出し先: fprintf（標準ライブラリ関数）。

関連データ構造

  • sqlca_t（SQL通信領域）: 本関数が通知する「切り詰め」の判定基準となる警告フラグを保持する構造体。

失敗モードと境界条件

  • 標準エラー出力がクローズされている場合、出力は失敗する可能性がある。
  • 本関数自体はエラーハンドリングを行わず、メッセージ出力のみを行う。

保守担当者向けメモ

  • 本関数はECPGプリプロセッサが生成するテストコードの一部であり、手動編集は推奨されない。
  • 警告メッセージの内容を変更する場合は、テストの期待値（expected output）との整合性に注意すること。
