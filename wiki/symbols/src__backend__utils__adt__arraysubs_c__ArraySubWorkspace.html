ArraySubWorkspace

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/arraysubs.c:30

目的

配列型の添字操作（サブスクリプト）を実行する際、式コンパイル時に決定された型情報および実行時の添字境界値を保持するためのワークスペース構造体です。

シグネチャまたは形状

typedef struct ArraySubWorkspace
{
    Oid         refelemtype;    /* 配列要素型のOID */
    int16       refattrlength;  /* 配列型のtyplen */
    int16       refelemlength;  /* 要素型のtyplen */
    bool        refelembyval;   /* 要素型は値渡し（pass-by-value）か */
    char        refelemalign;   /* 要素型のtypalign */
    int         upperindex[MAXDIM]; /* 添字の境界値（上限） */
    int         lowerindex[MAXDIM]; /* 添字の境界値（下限） */
} ArraySubWorkspace;

入力と出力

  • 本構造体は array_exec_setup 内で palloc を用いて割り当てられ、実行計画の sbsrefstate->workspace に格納されます。
  • 各フィールドは、式コンパイル時に取得された型情報および実行時の添字計算結果によって初期化されます。

前提条件と不変条件

  • upperindex および lowerindex 配列は、実際の次元数が MAXDIM 未満であっても、常に MAXDIM のサイズを保持する必要があります。これは array_get_slice や array_set_slice が境界外の領域を操作する可能性があるためです。

アルゴリズムまたは振る舞い

  • 実行計画の初期化時に割り当てられ、以降の添字操作（fetch/assign）において、型情報（サイズ、アライメント、値渡し可否）の参照および添字の妥当性検証に使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/arraysubs.c 内の array_exec_setup（割り当て）、および array_subscript_check_subscripts、array_subscript_fetch、array_subscript_assign 等の各操作関数（参照）。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • SubscriptingRef: 添字操作を表現するノード。本構造体はこのノードの実行時状態（sbsrefstate）の一部として保持されます。
  • MAXDIM: 配列の最大次元数。upperindex および lowerindex のサイズを決定します。

失敗モードと境界条件

  • MAXDIM を超える次元数を持つ配列操作は、本構造体の固定長配列の範囲外となるためサポートされません。
  • 実行時に sbsrefstate->workspace が適切に初期化されていない状態でアクセスした場合、未定義の動作を引き起こします。

保守担当者向けメモ

  • 本構造体は array_get/set_slice が MAXDIM に基づいて操作を行うため、配列の次元数制限に関する不変条件を厳守してください。
  • raw_array_subscript_handler と標準的な配列ハンドラの両方で利用されるため、変更時は両方の実行パスへの影響を考慮してください。
