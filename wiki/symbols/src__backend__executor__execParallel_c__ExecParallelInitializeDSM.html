ExecParallelInitializeDSM

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execParallel.c:446

目的

並列クエリ実行において、各プランノードが動的共有メモリ（DSM）上に必要な共有状態を確保・初期化するための再帰的なセットアップを行う。

シグネチャまたは形状

static bool ExecParallelInitializeDSM(PlanState *planstate, ExecParallelInitializeDSMContext *d);

入力と出力

  • **入力**:
  • planstate: 初期化対象のプランノード状態。
  • d: 初期化コンテキスト（DSMのTOC、インストゥルメンテーション情報、ノードカウンタ等を含む）。
  • **出力**:
  • 戻り値: planstate_tree_walker の結果（通常は false）。

前提条件と不変条件

  • ExecParallelEstimate による事前のメモリ見積もりが完了していること。
  • d->pcxt（ParallelContext）が有効であり、DSMのTOC（Table of Contents）へのアクセスが可能であること。

アルゴリズムまたは振る舞い

1. planstate が NULL の場合は直ちに false を返す。 2. インストゥルメンテーションが有効な場合、現在のノードIDを d->instrumentation に記録する。 3. d->nnodes をインクリメントする。 4. nodeTag(planstate) に基づき、各プランノード固有の初期化関数（例: ExecSeqScanInitializeDSM, ExecHashJoinInitializeDSM 等）を呼び出す。これらは shm_toc_allocate や shm_toc_insert を使用してDSM領域を割り当て、キーを登録する。 5. planstate_tree_walker を再帰的に呼び出し、プランツリー全体に対して同様の処理を適用する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c 内の並列実行初期化プロセスから呼び出される（直接の呼び出し元関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: 各プランノード固有の InitializeDSM 関数（ExecSeqScanInitializeDSM, ExecHashJoinInitializeDSM 等）および planstate_tree_walker。

関連データ構造

  • PlanState: 実行プランのノード状態。
  • ExecParallelInitializeDSMContext: 初期化に必要な共有メモリコンテキストや統計情報を保持する構造体。
  • ParallelContext: DSM管理のための基盤構造体。

失敗モードと境界条件

  • planstate が NULL の場合は何も行わず終了する。
  • 特定のノードタイプが switch 文に含まれていない場合、そのノードのDSM初期化はスキップされる。

保守担当者向けメモ

  • 新しい並列対応プランノードを追加する際は、本関数の switch 文に適切な初期化関数呼び出しを追加する必要がある。
  • EXPLAIN ANALYZE 用のインストゥルメンテーションが必要なノードは、並列対応か否かに関わらず初期化処理が必要となる点に注意すること。
