sepgsql_proc_post_create

定義

  • 種別: function
  • 定義箇所: contrib/sepgsql/proc.c:36

目的

新規作成されたPostgreSQL関数オブジェクトに対して、SELinuxのセキュリティコンテキストを計算し、デフォルトのセキュリティラベルを適用する。

シグネチャまたは形状

void sepgsql_proc_post_create(Oid functionId)

入力と出力

  • **入力**: functionId (作成された関数のOID)
  • **出力**: なし（副作用としてセキュリティラベルをシステムカタログに永続化する）

前提条件と不変条件

  • pg_proc カタログがトランザクション内で更新済みであること。
  • SnapshotSelf を用いて、作成直後のタプルを読み取る必要がある。
  • 関数が属する名前空間（スキーマ）の db_schema:{add_name} 権限が必須である。

アルゴリズムまたは振る舞い

1. **カタログ検索**: SnapshotSelf を使用して pg_proc から対象関数の情報を取得する。 2. **スキーマ権限確認**: 関数が作成された名前空間に対し、sepgsql_avc_check_perms を呼び出し db_schema:{add_name} 権限を検証する。 3. **ラベル計算**: クライアントのコンテキストと親名前空間のコンテキストから、sepgsql_compute_create を用いて新規関数のデフォルトラベルを導出する。 4. **関数権限確認**: 監査用文字列を生成し、sepgsql_avc_check_perms_label を呼び出す。proleakproof 属性が真の場合は SEPG_DB_PROCEDURE__INSTALL 権限も要求する。 5. **ラベル適用**: SetSecurityLabel を呼び出し、計算されたラベルを対象関数に付与する。 6. **クリーンアップ**: スキャンを終了し、割り当てたメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/sepgsql/hooks.c:143 (フック経由)
  • **呼び出し先**:
  • table_open, systable_beginscan, systable_getnext, systable_endscan (カタログ操作)
  • sepgsql_avc_check_perms, sepgsql_avc_check_perms_label (権限判定)
  • sepgsql_get_client_label, sepgsql_get_label, sepgsql_compute_create (ラベル計算)
  • SetSecurityLabel (ラベル適用)

関連データ構造

  • Form_pg_proc: pg_proc カタログのタプル構造。
  • ObjectAddress: 操作対象のオブジェクト識別子。
  • StringInfoData: 監査ログ生成用バッファ。

失敗モードと境界条件

  • systable_getnext でタプルが見つからない場合、elog(ERROR) によりトランザクションを中断する。
  • 権限チェックで拒否された場合、sepgsql_avc_check_perms 等が例外を発生させる。

保守担当者向けメモ

  • SnapshotSelf の使用は、トランザクション内で作成されたばかりのタプルを可視化するために必須である。
  • proleakproof 属性の変更はセキュリティ上の重要度が高いため、SEPG_DB_PROCEDURE__INSTALL 権限を要求する設計となっている。
  • db_language:{implement} の権限チェックが将来的に必要となる可能性がある（ソース内の XXX コメント参照）。
