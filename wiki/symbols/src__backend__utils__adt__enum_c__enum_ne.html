enum_ne

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/enum.c:332

目的

2つの列挙型（ENUM）値が等しくないかどうかを判定するSQL演算子関数です。

シグネチャまたは形状

Datum enum_ne(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る2つの Oid（列挙型の内部表現）。
  • **出力**: bool 型（Datum として返却）。2つのOIDが一致しない場合に true、一致する場合に false を返します。

前提条件と不変条件

  • 入力される2つの Oid は、同一の列挙型に属している必要があります。
  • 列挙型の比較において、OIDの不一致は値の不一致と等価であるという前提に基づいています。

アルゴリズムまたは振る舞い

1. PG_GETARG_OID マクロを使用して、第1引数および第2引数から Oid を取得します。 2. 取得した2つの Oid を直接比較（a != b）します。 3. 比較結果を PG_RETURN_BOOL マクロで返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLレベルの不一致演算子（<> または !=）から呼び出されます。
  • **呼び出し先**: 直接的な関数呼び出しはありません。OIDの比較のみを行うため、他の列挙型比較関数（enum_cmp_internal 等）は経由しません。

関連データ構造

  • Oid: PostgreSQLのシステムカタログにおけるオブジェクト識別子。列挙型の各値は一意のOIDを持ちます。

失敗モードと境界条件

  • 引数が列挙型以外のOIDである場合、または型が異なる列挙型同士の比較である場合の動作については、本関数内では検証されません。

保守担当者向けメモ

  • 本関数はOIDの直接比較のみを行うため、非常に高速です。
  • 列挙型の順序が変更された場合でも、OIDの等価性判定には影響しないため、キャッシュの再検証や複雑な比較ロジックは不要です。
