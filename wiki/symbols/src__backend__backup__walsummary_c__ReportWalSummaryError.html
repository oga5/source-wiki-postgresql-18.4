ReportWalSummaryError

定義

  • 種別: function
  • 定義箇所: src/backend/backup/walsummary.c:321

目的

WAL要約ファイルの読み取り中に発生したエラーを、PostgreSQLの標準的なエラー報告メカニズム（ereport）へ変換して通知するブリッジ関数です。

シグネチャまたは形状

void ReportWalSummaryError(void *callback_arg, char *fmt, ...);
  • pg_attribute_printf(2, 3) により、フォーマット文字列と可変長引数の型チェックが行われます。

入力と出力

  • **入力**:
  • callback_arg: コールバック用の汎用ポインタ（現在は使用されていない模様）。
  • fmt: エラーメッセージのフォーマット文字列。
  • ...: フォーマット文字列に対応する可変長引数。
  • **出力**: なし（関数は ERROR レベルの例外を発生させて終了します）。

前提条件と不変条件

  • ereport(ERROR, ...) を呼び出すため、本関数が呼び出されると現在のトランザクションまたはプロセスは中断されます。

アルゴリズムまたは振る舞い

1. StringInfoData バッファを初期化します。 2. appendStringInfoVA を使用して、可変長引数をフォーマット済みの文字列としてバッファに格納します。バッファサイズが不足する場合は enlargeStringInfo で動的に拡張します。 3. 最終的に ereport を呼び出し、ERRCODE_DATA_CORRUPTED エラーコードと共に構築したメッセージを出力します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup_incremental.c:589
  • src/backend/backup/walsummaryfuncs.c:105
  • **呼び出し先**:
  • initStringInfo, appendStringInfoVA, enlargeStringInfo (PostgreSQLの文字列操作ユーティリティ)
  • ereport (PostgreSQLのエラー報告マクロ)

関連データ構造

  • StringInfoData: 可変長文字列を保持するための構造体。

失敗モードと境界条件

  • メモリ不足が発生した場合、enlargeStringInfo 内部でメモリ割り当てエラーが発生し、本関数自体が例外を投げる可能性があります。

保守担当者向けメモ

  • 本関数は WalSummaryIO などのコールバックインターフェースと組み合わせて使用されることを想定しています。
  • callback_arg は現在の実装では使用されていませんが、将来的な拡張のためにシグネチャに含まれています。
