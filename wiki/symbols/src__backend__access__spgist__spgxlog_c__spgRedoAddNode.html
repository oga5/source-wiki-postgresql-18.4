spgRedoAddNode

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgxlog.c:283

目的

SP-GiSTインデックスにおけるノード追加操作のWALリプレイを実行し、インデックスページの状態を復元する。

シグネチャまたは形状

static void spgRedoAddNode(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（リプレイ対象のWALレコード）。
  • **出力**: なし（副作用としてバッファページを更新し、LSNを更新する）。

前提条件と不変条件

  • record から取得したタプルデータはアライメントされていないため、アクセス前に memcpy で SpGistInnerTupleData へコピーする必要がある。
  • ページ更新時は、親ノードの更新よりも子ページ（新しいノード）の挿入を優先し、リンクの整合性を維持しなければならない。

アルゴリズムまたは振る舞い

1. WALレコードから spgxlogAddNode 構造体とタプルデータを抽出する。 2. fillFakeState を用いてリプレイ用の SpGistState を構築する。 3. ブロック参照が1つのみの場合（XLogRecHasBlockRef(record, 1) が偽）:

  • 対象ページ上でタプルを置換し、LSNを更新してダーティにする。

    • 4. ブロック参照が複数ある場合:

  • 新しいノードを挿入し、必要に応じて親ノードのリンクを更新する。
  • 古いタプルを削除し、プレースホルダーまたはリダイレクトタプルで置換する。
  • 親ノードが更新対象に含まれる場合（parentBlk が0, 1, 2のいずれか）、spgUpdateNodeLink を呼び出してダウンリンクを更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgxlog.c 内の spg_redo 関数（WALレコードタイプに基づくディスパッチ経由）。
  • **呼び出し先**: fillFakeState, XLogRecGetData, XLogReadBufferForRedo, XLogInitBufferForRedo, SpGistInitBuffer, addOrReplaceTuple, spgUpdateNodeLink, spgFormDeadTuple, PageIndexTupleDelete, PageAddItem, PageSetLSN, MarkBufferDirty, UnlockReleaseBuffer。

関連データ構造

  • spgxlogAddNode: WALレコードのデータ構造。
  • SpGistInnerTupleData: インデックス内部タプルのヘッダー。
  • SpGistState: リプレイ時に使用する一時的な状態管理構造体。

失敗モードと境界条件

  • PageAddItem が期待するオフセットを返さない場合、elog(ERROR) を発生させる。
  • XLogReadBufferForRedo が BLK_NEEDS_REDO を返さない場合、ページ更新はスキップされる。
  • parentBlk の値に応じて、親ノードの更新場所（ページ0, 1, 2）が動的に決定される。

保守担当者向けメモ

  • リプレイ処理は、スタンバイサーバーでの読み取り時にリンク切れが発生しないよう、厳密な順序でページを更新する必要がある。
  • state.isBuild の状態によって、削除されたタプルの扱い（プレースホルダーかリダイレクトか）が分岐するため注意すること。
