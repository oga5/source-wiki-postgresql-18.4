TextPositionState

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/varlena.c:55

目的

文字列検索関数（text_position 関連）において、検索対象（haystack）と検索文字列（needle）の状態、およびBoyer-Moore-Horspool法に必要なスキップテーブルを保持するために使用されます。

シグネチャまたは形状

typedef struct {
    pg_locale_t locale;
    bool        is_multibyte_char_in_char;
    bool        greedy;
    char       *str1;
    char       *str2;
    int         len1;
    int         len2;
    int         skiptablemask;
    int         skiptable[256];
    char       *last_match;
    int         last_match_len;
    int         last_match_len_tmp;
    char       *refpoint;
    int         refpos;
} TextPositionState;

入力と出力

本構造体自体はデータ保持用ですが、text_position_setup 関数によって初期化され、text_position_next 等の関数で状態が更新・参照されます。

前提条件と不変条件

  • skiptable はBoyer-Moore-Horspoolアルゴリズムに基づく検索スキップ距離を格納します。
  • refpoint と refpos は、バイト位置から文字位置への変換を効率化するためのキャッシュとして機能し、前回の変換位置から継続して計算を行います。

アルゴリズムまたは振る舞い

  • text_position_setup で検索のための初期設定とスキップテーブルの構築が行われます。
  • text_position_next を呼び出すことで、反復的にマッチングを実行します。
  • 非決定論的なロケール（nondeterministic collations）を使用する場合、マッチした文字列の長さが入力された needle の長さと一致しない可能性があるため、last_match_len で個別に管理されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/varlena.c 内の text_position 関連関数群（text_position_setup, text_position_next 等）から参照・操作されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • pg_locale_t: ロケール情報を保持する構造体。

失敗モードと境界条件

  • skiptable は256要素で構成されており、検索文字のバイト値に基づいてインデックスされます。
  • 文字位置変換（refpoint）において、文字列の先頭からではなく前回の位置から計算を行うため、状態の整合性が維持されている必要があります。

保守担当者向けメモ

  • text_position_next_internal 等の内部関数と密接に結合しています。変更時はこれらの関数が TextPositionState の各フィールドをどのように更新しているかを確認してください。
  • TOASTされたデータが渡される可能性があるため、本構造体を利用する前段で PG_DETOAST_DATUM 等による展開が適切に行われているか注意が必要です。
