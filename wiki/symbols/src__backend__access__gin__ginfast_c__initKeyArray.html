initKeyArray

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginfast.c:674

目的

KeyArray 構造体を初期化し、指定された最大要素数分のメモリを割り当てる。

シグネチャまたは形状

static void initKeyArray(KeyArray *keys, int32 maxvalues)

入力と出力

  • **入力**:
  • keys: 初期化対象となる KeyArray 構造体へのポインタ。
  • maxvalues: 保持可能な最大要素数 (int32)。
  • **出力**: なし（keys が指す構造体のメンバを更新）。

前提条件と不変条件

  • keys は有効なメモリ領域を指している必要がある。
  • maxvalues は正の整数であること。
  • 実行後、keys->nvalues は必ず 0 に初期化される。

アルゴリズムまたは振る舞い

1. palloc_array を使用して、Datum 型の配列を maxvalues 分だけ割り当て、keys->keys に格納する。 2. 同様に GinNullCategory 型の配列を maxvalues 分だけ割り当て、keys->categories に格納する。 3. keys->nvalues を 0 に設定する。 4. keys->maxvalues に引数 maxvalues を代入する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginfast.c:868
  • src/backend/access/gin/ginfast.c:993
  • **呼び出し先**:
  • palloc_array (メモリ割り当て関数)

関連データ構造

  • KeyArray: GINインデックスのキーとカテゴリ情報を保持するための構造体。

失敗モードと境界条件

  • メモリ不足時には palloc_array がエラーを発生させる。
  • maxvalues に 0 以下の値を指定した場合の挙動は、palloc_array の実装に依存する。

保守担当者向けメモ

  • 本関数は KeyArray のメモリ管理を担うため、使用後のメモリ解放（pfree 等）が呼び出し側で適切に行われているか確認すること。
