PLy_result_dealloc

定義

  • 種別: function
  • 定義箇所: src/pl/plpython/plpy_resultobject.c:116

目的

PLyResultObject インスタンスのメモリを解放し、保持しているPythonオブジェクトの参照カウントを適切に減少させることで、リソースのクリーンアップを行います。

シグネチャまたは形状

static void PLy_result_dealloc(PLyResultObject *self)

入力と出力

  • 入力: PLyResultObject *self（解放対象のPL/Python結果オブジェクト）
  • 出力: なし（void）

前提条件と不変条件

  • self は有効な PLyResultObject へのポインタである必要があります。
  • Python 3.8以降の環境では、型オブジェクトの参照カウント管理が追加で行われます。

アルゴリズムまたは振る舞い

1. Python 3.8以降の場合、対象オブジェクトの型情報を取得します。 2. self が保持する nrows、rows、status の各Pythonオブジェクトに対して Py_XDECREF を実行し、参照カウントを減らします。 3. tupdesc が存在する場合、FreeTupleDesc を呼び出して解放し、ポインタを NULL に設定します。 4. PyObject_Free(self) を呼び出し、構造体自体のメモリを解放します。 5. Python 3.8以降の場合、手順1で取得した型オブジェクトの参照カウントを Py_DECREF で減らします。

呼び出し元と呼び出し先

  • 呼び出し元: Pythonの型システム（Py_tp_dealloc スロット経由）および src/pl/plpython/plpy_spi.c からの参照が確認されています。
  • 呼び出し先: Py_XDECREF, FreeTupleDesc, PyObject_Free, Py_DECREF（いずれもPython C APIまたはPostgreSQL内部関数）。

関連データ構造

  • PLyResultObject: 本関数が管理・解放する主要な構造体。

失敗モードと境界条件

  • tupdesc が NULL の場合、FreeTupleDesc はスキップされます。
  • Python 3.8未満の環境では、型オブジェクトの参照カウント操作は行われません。

保守担当者向けメモ

  • 本関数はPythonのC APIにおけるデストラクタとして機能します。
  • Python 3.8以降の PyTypeObject 参照カウント管理（Python issue 35810対応）が含まれているため、将来的なPythonバージョンアップ時にもこの条件付きコンパイルの整合性に注意してください。
