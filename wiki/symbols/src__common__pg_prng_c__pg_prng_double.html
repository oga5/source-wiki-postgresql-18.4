pg_prng_double

定義

  • 種別: function
  • 定義箇所: src/common/pg_prng.c:267

目的

擬似乱数生成器の状態から、[0.0, 1.0) の範囲で一様に分布する倍精度浮動小数点数（double）を生成する。

シグネチャまたは形状

double pg_prng_double(pg_prng_state *state)

入力と出力

  • **入力**: pg_prng_state *state（乱数生成器の状態を保持する構造体へのポインタ）
  • **出力**: double 型の乱数値

前提条件と不変条件

  • state は初期化済みであること。
  • PostgreSQLがIEEE浮動小数点演算をサポートしている環境であることを前提としている。

アルゴリズムまたは振る舞い

1. xoroshiro128ss(state) を呼び出し、64ビットの乱数値 v を取得する。 2. v の上位52ビットを抽出する（v >> (64 - 52)）。 3. ldexp 関数を用い、抽出した値を $2^{-52}$ 倍することで、[0.0, 1.0) の範囲にスケーリングして返す。

呼び出し元と呼び出し先

  • **呼び出し先**: xoroshiro128ss（内部的な乱数生成コア）、ldexp（数学ライブラリ関数）。
  • **呼び出し元**:
  • contrib/auto_explain/auto_explain.c
  • contrib/tablefunc/tablefunc.c
  • src/backend/access/gin/ginget.c
  • src/backend/access/transam/xact.c
  • src/backend/optimizer/geqo/geqo_random.c
  • src/backend/storage/lmgr/s_lock.c
  • src/backend/tcop/postgres.c
  • src/backend/utils/adt/pseudorandomfuncs.c
  • src/backend/utils/misc/sampling.c

関連データ構造

  • pg_prng_state: 乱数生成器の内部状態を保持する構造体。

失敗モードと境界条件

  • **境界**: 生成される値は [0.0, 1.0) の範囲内である。
  • **精度**: 倍精度浮動小数点数の仮数部が52ビットであることを前提としており、環境によっては1.0に丸められる可能性がある。
  • **ゼロ値**: 0.0 が返される可能性があるため、サンプリング等で 0.0 を除外する必要がある場合は呼び出し側で処理が必要。

保守担当者向けメモ

  • 本関数は暗号学的に安全ではないため、パスワードや暗号鍵の生成には使用しないこと。
  • 内部で xoroshiro128ss を使用しており、状態がすべて0になると0を返し続ける不動点を持つ。初期化時のチェックには pg_prng_seed_check を参照すること。
