s_lock_stuck

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/s_lock.c:78

目的

スピンロックの取得試行が一定回数を超えて失敗し、デッドロックまたは異常な競合状態（スタック）と判断された場合に、エラーを報告してプロセスを停止させる。

シグネチャまたは形状

static void s_lock_stuck(const char *file, int line, const char *func)

入力と出力

  • **入力**:
  • file: 呼び出し元のソースファイル名。
  • line: 呼び出し元の行番号。
  • func: 呼び出し元の関数名（NULL の場合は "(unknown)" として扱われる）。
  • **出力**: なし（関数は復帰せず、プロセスを終了させる）。

前提条件と不変条件

  • 本関数が呼び出されることは、システムが回復不能なスピンロックの競合状態にあることを意味する。

アルゴリズムまたは振る舞い

1. func が NULL の場合、デフォルト値 (unknown) を代入する。 2. S_LOCK_TEST マクロが定義されている場合:

  • 標準エラー出力にスタックした旨のメッセージを出力し、exit(1) でプロセスを終了する。

    • 3. S_LOCK_TEST マクロが定義されていない場合:

  • elog(PANIC, ...) を呼び出し、致命的エラーとしてプロセスを異常終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/lmgr/s_lock.c:135（スピンロックの待機ループ内で、試行回数が上限に達した際に呼び出される）。
  • **呼び出し先**: fprintf（テスト時）、exit（テスト時）、または elog（通常時）。

関連データ構造

  • なし。

失敗モードと境界条件

  • func 引数が NULL の場合でも安全に動作するよう設計されている。
  • 本関数自体が失敗モード（PANIC）のトリガーであるため、本関数内での境界条件は存在しない。

保守担当者向けメモ

  • 本関数は PANIC を発生させるため、呼び出し元はロック取得の最終手段としてのみ使用すること。
  • S_LOCK_TEST マクロによる条件分岐は、主にスタンドアローンでのデバッグ用途を想定している。
