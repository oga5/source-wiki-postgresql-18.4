bbsink_throttle

定義

  • 種別: struct
  • 定義箇所: src/backend/backup/basebackup_throttle.c:23

目的

ベースバックアップの転送速度を制限（スロットリング）するための bbsink 実装において、スロットリング状態を保持するために使用されます。

シグネチャまたは形状

typedef struct bbsink_throttle
{
    bbsink      base;
    uint64      throttling_sample;
    int64       throttling_counter;
    TimeOffset  elapsed_min_unit;
    TimestampTz throttled_last;
} bbsink_throttle;

入力と出力

本構造体はデータそのものではなく、転送レート制御のための状態を保持します。

  • throttling_sample: スロットリング判定を行うためのデータ量（バイト単位）。
  • throttling_counter: 累積転送量。
  • elapsed_min_unit: throttling_sample を転送するために必要な最小時間。
  • throttled_last: 最後に転送レートを確認した時刻。

前提条件と不変条件

  • bbsink を継承しており、パイプラインの一部として機能します。
  • 転送レートの計算は整数演算で行われ、端数は次回の呼び出しに持ち越されます。

アルゴリズムまたは振る舞い

1. throttle() 関数により、累積転送量が throttling_sample を超えたか判定されます。 2. 転送が速すぎる場合、WaitLatch を用いて elapsed_min_unit に基づく待機時間をスリープします。 3. 待機中、CHECK_FOR_INTERRUPTS() を呼び出し、停止要求に対して即座に応答します。 4. 調整後、データを後続のシンクへ転送します。

呼び出し元と呼び出し先

  • **呼び出し元**: bbsink_throttle_new（初期化時）、bbsink_throttle_archive_contents、bbsink_throttle_manifest_contents（データ転送時）。
  • **呼び出し先**: throttle()（内部関数）、および bbsink_forward_* 関数群（後続シンクへの転送）。

関連データ構造

  • bbsink: 基底構造体。
  • TimeOffset, TimestampTz: 時間計測に使用される型。

失敗モードと境界条件

  • 転送レートが極端に高い場合、待機時間が長くなる可能性があります。
  • 停止シグナル受信時は CHECK_FOR_INTERRUPTS() により処理が中断されます。

保守担当者向けメモ

  • 本構造体は「デコレータ」パターンで実装されており、既存のシンクチェーンの間に挿入可能です。
  • 整数演算によるスロットリングのため、微小な誤差が発生する可能性があります。
