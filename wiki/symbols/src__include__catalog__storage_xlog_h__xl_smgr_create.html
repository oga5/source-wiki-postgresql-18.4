xl_smgr_create

定義

  • 種別: struct
  • 定義箇所: src/include/catalog/storage_xlog.h:33

目的

データベースのファイル作成操作をWAL（Write-Ahead Logging）に記録するための構造体です。リカバリ時に物理的なファイル作成を再実行（Redo）するために必要な情報を保持します。

シグネチャまたは形状

typedef struct xl_smgr_create
{
    RelFileLocator rlocator;
    ForkNumber    forkNum;
} xl_smgr_create;

入力と出力

  • **入力**: ストレージ層でのファイル作成時に、対象リレーションの識別子（RelFileLocator）とフォーク番号（ForkNumber）が本構造体に格納されます。
  • **出力**: WALレコードの一部としてディスクへ書き込まれ、リカバリプロセスで読み取られます。

前提条件と不変条件

  • 本構造体は、物理的なファイル作成操作と対になるWALレコードとして使用されます。
  • リカバリ時には、保持されている RelFileLocator と forkNum を用いて、対応する物理ファイルが正しく作成される必要があります。

アルゴリズムまたは振る舞い

1. **記録**: ストレージ層でファイルが作成される際、log_smgrcreate() 等を通じて本構造体に情報がセットされ、WALに書き込まれます。 2. **リカバリ**: システム起動時のリカバリプロセスにおいて、smgr_redo() がWALレコードを読み取り、本構造体の内容を参照して物理的なストレージ操作を再実行します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/access/rmgrdesc/smgrdesc.c
  • src/backend/access/transam/xlogprefetcher.c
  • src/backend/catalog/storage.c
  • src/backend/postmaster/walsummarizer.c
  • **呼び出し先**:
  • 本構造体自体はデータ保持用であり、関数呼び出しは含みません。

関連データ構造

  • RelFileLocator: リレーションの物理的な位置を特定するための識別子。
  • ForkNumber: ファイルのフォーク（メインデータ、FSM、VMなど）を識別する番号。

失敗モードと境界条件

  • WALレコードの読み取り時にデータが破損している場合、リカバリプロセスが失敗する可能性があります。
  • 存在しないリレーションや無効なフォーク番号が指定された場合、smgr_redo による再実行時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は「作成」操作専用です。ファイル「削除」のログ記録は xact.c 側で管理されているため、本構造体には含まれません。
  • リカバリ時の物理的な一貫性を保つため、RelFileLocator の変更や ForkNumber の定義変更時には本構造体の整合性に注意してください。
