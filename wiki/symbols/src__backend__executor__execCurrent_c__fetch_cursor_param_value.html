fetch_cursor_param_value

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execCurrent.c:257

目的

WHERE CURRENT OF 句で使用されるカーソル名がパラメータとして渡された場合に、その値を解決して文字列として取得します。

シグネチャまたは形状

static char *fetch_cursor_param_value(ExprContext *econtext, int paramId);

入力と出力

  • **入力**:
  • econtext: 式の評価コンテキスト (ExprContext)。
  • paramId: 解決対象のパラメータID。
  • **出力**:
  • 成功時: カーソル名を表す char * 型の文字列。
  • 失敗時: エラーを報告し、呼び出し元へは戻りません。

前提条件と不変条件

  • econtext が有効な ParamListInfo を保持していること。
  • 指定された paramId が有効な範囲内であること。
  • 取得されたパラメータの型が REFCURSOROID であること。

アルゴリズムまたは振る舞い

1. econtext からパラメータリストを取得します。 2. paramFetch フックが定義されていればそれを呼び出し、そうでなければ params 配列から直接パラメータデータを取得します。 3. パラメータが有効かつ非 NULL であることを確認します。 4. パラメータの型が REFCURSOROID であるか検証し、一致しない場合はエラーを発生させます。 5. TextDatumGetCString を使用して、refcursor 型の値を C 文字列に変換して返します。 6. パラメータが見つからない、または型が不適切な場合は ereport(ERROR, ...) を呼び出します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execCurrent.c 内の関数（WHERE CURRENT OF 処理の過程で呼び出される）。
  • **呼び出し先**:
  • paramInfo->paramFetch (フックが設定されている場合)
  • ereport (エラー発生時)
  • TextDatumGetCString (値の変換時)

関連データ構造

  • ExprContext: 式評価の実行時状態を保持。
  • ParamListInfo: パラメータのリストと取得用フックを保持。
  • ParamExternData: 個別のパラメータデータ。

失敗モードと境界条件

  • **パラメータ未定義**: paramId が範囲外、または値が取得できない場合に ERRCODE_UNDEFINED_OBJECT でエラー。
  • **型不一致**: パラメータの型が REFCURSOROID ではない場合に ERRCODE_DATATYPE_MISMATCH でエラー。
  • **NULL 値**: パラメータが NULL の場合、エラーとして扱われます。

保守担当者向けメモ

  • 本関数は refcursor 型が text 型の I/O ルーチンを共有しているという前提に依存しています。
  • 動的パラメータを扱うためのフック (paramFetch) が適切に処理されるよう設計されています。
