copy_file

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/file/copydir.c:132

目的

指定されたソースファイルをコピー先へ複製します。OSキャッシュへの負荷を抑えつつ、データ整合性を考慮したバッファリングコピーを実行します。

シグネチャまたは形状

void copy_file(const char *fromfile, const char *tofile)

入力と出力

  • **入力**:
  • fromfile: コピー元ファイルのパス（文字列）。
  • tofile: コピー先ファイルのパス（文字列）。
  • **出力**: なし（成功時は正常終了、失敗時は ereport(ERROR, ...) により例外をスロー）。

前提条件と不変条件

  • fromfile は読み取り可能であること。
  • tofile は新規作成可能であること（既存ファイルがある場合は失敗する）。
  • palloc を使用するため、PostgreSQL のメモリコンテキストが有効であること。

アルゴリズムまたは振る舞い

1. **初期化**: COPY_BUF_SIZE（8 * BLCKSZ）のバッファを確保。 2. **ファイルオープン**: OpenTransientFile を使用し、ソースを読み取り専用、コピー先を書き込み専用（新規作成）で開く。 3. **データコピー**:

  • ループ内で read と write を繰り返し、データを転送。
  • CHECK_FOR_INTERRUPTS() を呼び出し、シグナルによる中断を監視。
  • FLUSH_DISTANCE（プラットフォームにより1MBまたは32MB）ごとに pg_flush_data を呼び出し、OSキャッシュをフラッシュ。
  • 転送中、pgstat_report_wait_start/end を使用して待機イベントを記録。

    • 4. **終了処理**: 最後に残りのデータをフラッシュし、ファイルを閉じてバッファを解放。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/basic_archive/basic_archive.c からの直接呼び出しが確認されています。その他の src/backend/commands/copyfrom.c 等の参照は、本関数ではなく FILE* 型の変数名との衝突である可能性が高く、本関数の呼び出し元としては不確定です。
  • **呼び出し先**: OpenTransientFile, CloseTransientFile, palloc, pfree, read, write, pg_flush_data, CHECK_FOR_INTERRUPTS, pgstat_report_wait_start/end。

関連データ構造

  • COPY_BUF_SIZE: コピー用バッファサイズ。
  • FLUSH_DISTANCE: キャッシュフラッシュを行う間隔。

失敗モードと境界条件

  • **ファイルオープン失敗**: ソースが存在しない、またはコピー先への書き込み権限がない場合に ERROR を発生。
  • **読み書き失敗**: I/Oエラー発生時に ERROR を発生。特に書き込み時に errno が設定されない場合はディスクフルとみなして ENOSPC を設定。
  • **中断**: CHECK_FOR_INTERRUPTS() により、処理中にシグナルを受け取ると即座に終了。

保守担当者向けメモ

  • 本関数は fsync を直接実行しません。呼び出し側でコピー完了後に fsync_fname 等を用いて永続化を保証する必要があります。
  • FLUSH_DISTANCE はプラットフォーム（特にmacOS）の特性を考慮したチューニング値であるため、変更時は注意してください。
