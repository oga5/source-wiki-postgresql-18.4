prepare_room

定義

  • 種別: function
  • 定義箇所: contrib/pgcrypto/mbuf.c:73

目的

MBuf 構造体が保持するバッファ領域に対し、指定されたバイト数（block_len）のデータを追加するために十分な空き容量を確保する。

シグネチャまたは形状

static void prepare_room(MBuf *mbuf, int block_len)

入力と出力

  • **入力**:
  • mbuf: 操作対象の MBuf 構造体へのポインタ。
  • block_len: 追加予定のデータサイズ（バイト単位）。
  • **出力**: なし（副作用として mbuf の内部状態を更新）。

前提条件と不変条件

  • mbuf は有効な MBuf 構造体を指している必要がある。
  • repalloc を使用するため、PostgreSQL のメモリコンテキスト内で実行される必要がある。

アルゴリズムまたは振る舞い

1. 現在のバッファの空き容量（mbuf->buf_end - mbuf->data_end）が block_len 以上であれば、何もせず終了する。 2. 空き容量が不足している場合、STEP 定数に基づき、現在のバッファサイズに block_len を加えた分を拡張する。 3. repalloc を呼び出し、メモリを再確保する。 4. 拡張後の新しいメモリ領域に合わせて、mbuf 内の各ポインタ（buf_end, data_end, read_pos, data）を再計算し、オフセットを維持したまま更新する。

呼び出し元と呼び出し先

  • **呼び出し元**: mbuf_append（contrib/pgcrypto/mbuf.c:102）から呼び出される。
  • **呼び出し先**: repalloc（PostgreSQL のメモリ管理関数）。

関連データ構造

  • MBuf: メモリバッファを管理する構造体。data（開始位置）、data_end（データ終了位置）、buf_end（バッファ終了位置）、read_pos（読み込み位置）を保持する。

失敗モードと境界条件

  • メモリ不足時には repalloc がエラーを発生させる。
  • block_len が負の値や極端に大きい値の場合の挙動は、repalloc の制約に依存する。

保守担当者向けメモ

  • 本関数は mbuf_append の内部ヘルパーとして機能する。
  • ポインタの再計算時に data を基準とした相対位置を維持しているため、メモリ再配置後もデータ整合性が保たれる設計となっている。
