pgaio_io_call_stage

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_callback.c:198

目的

非同期I/O（AIO）操作の開始前に、PgAioHandle に登録された各コールバックのステージング処理（stage）を順次実行します。

シグネチャまたは形状

void pgaio_io_call_stage(PgAioHandle *ioh)

入力と出力

  • **入力**: PgAioHandle *ioh - 処理対象のAIOハンドル。
  • **出力**: なし（副作用として各コールバックの stage 関数を実行）。

前提条件と不変条件

  • ioh->target が PGAIO_TID_INVALID より大きく PGAIO_TID_COUNT 未満であること。
  • ioh->op が PGAIO_OP_INVALID より大きく PGAIO_OP_COUNT 未満であること。
  • 上記条件は Assert により検証されます。

アルゴリズムまたは振る舞い

1. ioh に登録されたコールバックを、登録順序の逆順（スタック形式）でループ処理します。 2. 各コールバックIDに対応する PgAioHandleCallbacksEntry を aio_handle_cbs 配列から取得します。 3. 取得したエントリに stage 関数が定義されている場合、その関数を (ioh, cb_data) を引数として呼び出します。 4. pgaio_debug_io を介して、デバッグレベル DEBUG3 で実行状況を記録します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/aio/aio.c:447
  • **呼び出し先**:
  • 登録された各コールバックの stage 関数（PgAioHandleCallbacksEntry 経由）。
  • pgaio_debug_io（デバッグログ出力）。

関連データ構造

  • PgAioHandle: I/O操作の状態とコールバック情報を保持する構造体。
  • PgAioHandleCallbacksEntry: コールバック関数ポインタとメタデータを保持する構造体。
  • aio_handle_cbs: 利用可能な全コールバック定義を保持する静的配列。

失敗モードと境界条件

  • ioh->num_callbacks が 0 の場合、ループは実行されず何も行われません。
  • 特定のコールバックに stage 関数が定義されていない場合、そのコールバックはスキップされます。

保守担当者向けメモ

  • コールバックは登録順序の逆順で実行されます。これは、後から登録された処理が優先的に結果を加工できるようにするためです。
  • 本関数はI/O開始前に呼び出されるため、共有リソースのロックや準備が必要な処理をここに集約してください。
