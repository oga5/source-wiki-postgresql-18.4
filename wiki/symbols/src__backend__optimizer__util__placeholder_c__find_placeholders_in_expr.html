find_placeholders_in_expr

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/placeholder.c:258

目的

指定された式ツリー内を走査し、含まれるすべての PlaceHolderVar を特定して、対応する PlaceHolderInfo エントリをクエリの PlannerInfo に登録する。

シグネチャまたは形状

static void find_placeholders_in_expr(PlannerInfo *root, Node *expr)

入力と出力

  • **入力**:
  • root: クエリの最適化情報を保持する PlannerInfo 構造体へのポインタ。
  • expr: 走査対象となる式ツリー（Node）。
  • **出力**: なし（副作用として root 内の PlaceHolderInfo リストが更新される）。

前提条件と不変条件

  • root は有効な PlannerInfo 構造体を指している必要がある。
  • root->placeholdersFrozen が真である場合、新しい PlaceHolderInfo の追加は制限される可能性がある（モジュール概要の注意書きに基づく）。

アルゴリズムまたは振る舞い

1. pull_var_clause を使用し、式ツリーから PlaceHolderVar を含む変数リストを抽出する。この際、集約関数やウィンドウ関数内も再帰的に探索する。 2. 抽出されたリストを反復処理する。 3. 各要素が PlaceHolderVar であることを確認し、それ以外（通常の Var など）は無視する。 4. find_placeholder_info を呼び出し、該当する PlaceHolderVar に対応する PlaceHolderInfo が存在しない場合は新規作成・登録を行う。 5. 最後に、抽出した変数リストをメモリから解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/placeholder.c 内の以下の箇所から呼び出される:
  • PlaceHolderInfo の再帰的処理時（174行目）
  • f->quals の処理時（233行目）
  • j->quals の処理時（246行目）
  • **呼び出し先**:
  • pull_var_clause
  • find_placeholder_info
  • list_free

関連データ構造

  • PlannerInfo: クエリ全体の最適化状態を保持。
  • PlaceHolderVar: 式ツリー内に埋め込まれたプレースホルダノード。
  • PlaceHolderInfo: PlaceHolderVar の評価レベルや依存関係を管理する構造体。

失敗モードと境界条件

  • 式が NULL の場合や、PlaceHolderVar を含まない式の場合は、何も行わず正常終了する。
  • pull_var_clause がメモリ不足等で失敗した場合は、その例外が伝播する可能性がある。

保守担当者向けメモ

  • pull_var_clause は必要以上の情報を抽出する可能性があるが、実装の簡便さのために採用されている。
  • 本関数は PlaceHolderVar の存在を保証するための初期化フェーズで重要となる。デバッグ時は find_placeholder_info の呼び出し状況を追跡すること。
