prepare_sql_fn_parse_info

定義

  • 種別: function
  • 定義箇所: src/backend/executor/functions.c:250

目的

SQL関数の解析に必要なメタデータ（引数型、引数名、照合順序など）を収集し、SQLFunctionParseInfo 構造体を構築する。

シグネチャまたは形状

SQLFunctionParseInfoPtr prepare_sql_fn_parse_info(HeapTuple procedureTuple, Node *call_expr, Oid inputCollation)

入力と出力

  • **入力**:
  • procedureTuple: 対象関数の pg_proc システムカタログタプル。
  • call_expr: 関数呼び出し式（多相型解決に使用）。
  • inputCollation: 関数の入力照合順序。
  • **出力**:
  • SQLFunctionParseInfoPtr: 構築された解析情報構造体へのポインタ。

前提条件と不変条件

  • procedureTuple は有効な pg_proc タプルであること。
  • 多相型（Polymorphic type）が含まれる場合、call_expr を用いて実際の型が解決可能であること。

アルゴリズムまたは振る舞い

1. SQLFunctionParseInfo 構造体をメモリ確保し、関数名をコピーする。 2. 入力照合順序を保存する。 3. pg_proc から引数型をコピーし、多相型が含まれる場合は get_call_expr_argtype を用いて実際の型に解決する。解決できない場合はエラーを報告する。 4. SysCacheGetAttr を使用して pg_proc から引数名（proargnames）と引数モード（proargmodes）を取得する。 5. get_func_input_arg_names を呼び出し、引数名を解析情報に格納する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_proc.c
  • src/backend/executor/functions.c
  • src/backend/optimizer/util/clauses.c
  • **呼び出し先**:
  • palloc0, palloc, pstrdup (メモリ管理)
  • GETSTRUCT, SysCacheGetAttr (システムカタログアクセス)
  • IsPolymorphicType, get_call_expr_argtype (型解決)
  • get_func_input_arg_names (引数名抽出)
  • ereport (エラー報告)

関連データ構造

  • SQLFunctionParseInfo: 本関数が構築する主要なデータ構造。
  • HeapTuple: pg_proc エントリを保持する。
  • Form_pg_proc: pg_proc タプルの構造体定義。

失敗モードと境界条件

  • 多相型の解決に失敗した場合、ERRCODE_DATATYPE_MISMATCH でエラー終了する。
  • 引数名やモードがシステムカタログ上で NULL の場合、適切に処理（NULL として扱う）される。
  • 引数名の配列エントリ数が引数の総数より少ない場合、安全のため argnames は NULL に設定される。

保守担当者向けメモ

  • 本関数は SQL 関数の解析フェーズにおける初期化を担当する。
  • 多相型の解決ロジックは src/backend/optimizer/util/clauses.c 等の他の箇所でも参照されるため、変更時は影響範囲に注意すること。
