pgstat_wal_flush_cb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_wal.c:90

目的

プロセスローカルなWAL使用量統計（pgWalUsage）の増分を計算し、共有メモリ上のWAL統計構造体へ反映（フラッシュ）します。

シグネチャまたは形状

bool pgstat_wal_flush_cb(bool nowait);

入力と出力

  • **入力**:
  • nowait (bool): true の場合、共有メモリのロック取得に失敗した際に待機せず即座に制御を戻します。
  • **出力**:
  • bool: 共有メモリへの書き込みがスキップされた場合（保留中の変更がない、または nowait 指定でロック取得失敗時）に true を返します。正常にフラッシュが完了した場合は false を返します。

前提条件と不変条件

  • IsUnderPostmaster または !IsPostmasterEnvironment が真であること。
  • pgStatLocal.shmem が初期化済みであり、シャットダウン状態ではないこと。

アルゴリズムまたは振る舞い

1. pgstat_wal_have_pending() を呼び出し、更新すべき差分があるか確認します。ない場合は false を返して終了します。 2. WalUsageAccumDiff を使用し、現在の pgWalUsage と前回の報告値 prevWalUsage との差分を算出します。 3. nowait 引数に応じて共有メモリのロック（stats_shmem->lock）を取得します。

  • nowait=true でロック取得に失敗した場合は true を返して終了します。

    • 4. WALSTAT_ACC マクロを用いて、算出した差分を共有メモリ上の wal_records, wal_fpi, wal_bytes, wal_buffers_full に加算します。 5. ロックを解放します。 6. 次回の計算基準とするため、prevWalUsage を現在の pgWalUsage で更新します。 7. false を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/activity/pgstat.c: 統計システム内のコールバックとして登録されています。
  • src/backend/utils/activity/pgstat_wal.c: pgstat_report_wal 等の内部処理から呼び出されます。
  • **呼び出し先**:
  • pgstat_wal_have_pending()
  • WalUsageAccumDiff()
  • LWLockAcquire() / LWLockConditionalAcquire()
  • LWLockRelease()

関連データ構造

  • PgStatShared_Wal: 共有メモリ上のWAL統計保持構造体。
  • WalUsage: WAL使用量カウンタを保持する構造体。
  • pgWalUsage (グローバル変数): プロセスローカルな現在のWAL使用量。
  • prevWalUsage (静的変数): 前回のフラッシュ時点でのWAL使用量。

失敗モードと境界条件

  • **ロック競合**: nowait=true の場合、ロックが取得できなければ統計の更新をスキップし、呼び出し元へ true を返します。
  • **変更なし**: 更新すべき統計がない場合、ロックを取得せずに false を返します。

保守担当者向けメモ

  • 本関数は pgstat モジュールのコールバックとして動的に呼び出される設計です。
  • WALSTAT_ACC マクロは関数スコープ内で定義されており、共有メモリへの加算処理を簡略化しています。
  • prevWalUsage の更新は、共有メモリへの書き込みが成功した直後に行われる必要があります。
