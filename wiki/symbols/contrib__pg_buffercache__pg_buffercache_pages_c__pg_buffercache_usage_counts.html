pg_buffercache_usage_counts

定義

  • **種別**: function
  • **定義箇所**: contrib/pg_buffercache/pg_buffercache_pages.c:619

目的

共有バッファキャッシュ内のバッファ使用回数（usage count）ごとの分布統計を収集し、各カウント値におけるバッファ数、ダーティバッファ数、およびピン留め（pinned）されているバッファ数を算出する。

シグネチャまたは形状

Datum pg_buffercache_usage_counts(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（SQLインターフェース経由の呼び出し引数）。
  • **出力**: Datum（戻り値は 0 だが、tuplestore を通じて (usage_count, count, dirty, pinned) の形式で結果セットを返す）。

前提条件と不変条件

  • NBuffers（全バッファ数）の全走査を行うため、システム負荷が高い環境では実行時間に影響を与える可能性がある。
  • バッファ状態は pg_atomic_read_u32 を用いてアトミックに読み取られるが、関数全体でシステム全体の一貫したスナップショットを保証するものではない。

アルゴリズムまたは振る舞い

1. InitMaterializedSRF を呼び出し、結果セットを格納するメモリ領域を初期化する。 2. 0 から NBuffers - 1 までループし、各バッファのヘッダー（BufferDesc）を取得する。 3. pg_atomic_read_u32 でバッファ状態を取得し、BUF_STATE_GET_USAGECOUNT で使用回数を抽出する。 4. 抽出した使用回数に基づき、usage_counts（総数）、dirty（ダーティフラグ有無）、pinned（参照カウント有無）の各配列をインクリメントする。 5. BM_MAX_USAGE_COUNT + 1 までの各インデックスについて、集計結果を tuplestore_putvalues で出力用タプルに格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PG_FUNCTION_INFO_V1 マクロによる登録経由）。
  • **呼び出し先**:
  • GetBufferDescriptor
  • pg_atomic_read_u32
  • CHECK_FOR_INTERRUPTS
  • tuplestore_putvalues

関連データ構造

  • BufferDesc: バッファヘッダー情報。
  • ReturnSetInfo: SRF（Set-Returning Function）の結果セット情報。
  • BM_MAX_USAGE_COUNT: 使用回数の最大値。

失敗モードと境界条件

  • 割り込みが発生した場合、CHECK_FOR_INTERRUPTS() により処理が中断される可能性がある。
  • NBuffers が非常に大きい場合、メモリ消費量が増大する可能性がある。

保守担当者向けメモ

  • 本関数は統計情報の集計を行うため、バッファヘッダーのロックを保持し続けることはない。
  • ダーティフラグやピン留め状態は、集計の瞬間の値であり、関数実行中に変化する可能性がある。
