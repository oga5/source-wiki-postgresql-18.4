FindEndOfXLOG

定義

  • 種別: function
  • 定義箇所: src/bin/pg_resetwal/pg_resetwal.c:939

目的

既存の WAL セグメントファイルをスキャンし、新しい WAL の開始位置（セグメント番号）を決定する。

シグネチャまたは形状

static void FindEndOfXLOG(void)

入力と出力

  • **入力**: グローバル変数 ControlFile（既存の制御情報）および pg_wal ディレクトリ内のファイル群。
  • **出力**: グローバル変数 newXlogSegNo を更新する。

前提条件と不変条件

  • ControlFile が読み込み済みであること。
  • pg_wal ディレクトリが存在し、アクセス可能であること。
  • タイムラインに関わらず、発見されたすべての WAL セグメントのうち最大のセグメント番号を基準とする。

アルゴリズムまたは振る舞い

1. ControlFile.checkPointCopy.redo から初期のセグメント番号を算出する。 2. pg_wal ディレクトリを走査し、IsXLogFileName または IsPartialXLogFileName に合致するファイル名からセグメント番号を抽出する。 3. 発見されたセグメント番号の最大値を newXlogSegNo に保持する。 4. 最後に、現在の ControlFile.xlog_seg_size から新しい WAL セグメントサイズ（WalSegSz）へ換算し、新しい WAL の開始位置として安全な「未使用領域」まで進める（newXlogSegNo++）。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（src/bin/pg_resetwal/pg_resetwal.c:424）
  • **呼び出し先**: opendir, readdir, closedir, XLogFromFileName, XLByteToSeg, pg_fatal

関連データ構造

  • ControlFileData (グローバル変数 ControlFile): 制御情報。
  • XLogSegNo: WAL セグメント番号。

失敗モードと境界条件

  • pg_wal ディレクトリが開けない場合、または読み取り中にエラーが発生した場合は pg_fatal により異常終了する。
  • 既存の WAL ファイルが存在しない場合、ControlFile のチェックポイント位置を基準に計算される。

保守担当者向けメモ

  • 本関数は、古い WAL セグメントサイズから新しいサイズへの変換を伴うため、ControlFile.xlog_seg_size と WalSegSz の両方の値が正しく設定されていることが重要である。
  • タイムラインを無視して最大値を採用する設計は、安全側に倒した保守的なアプローチである。
