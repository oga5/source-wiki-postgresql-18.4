ConstraintNameExists

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_constraint.c:456

目的

指定された名前空間（スキーマ）内において、特定の名前を持つ制約が既に存在するかどうかを確認する。

シグネチャまたは形状

bool ConstraintNameExists(const char *conname, Oid namespaceid)

入力と出力

  • **入力**:
  • conname: 検索対象の制約名（const char *）。
  • namespaceid: 制約が属する名前空間の OID（Oid）。
  • **出力**:
  • 制約が存在する場合は true、存在しない場合は false を返す（bool）。

前提条件と不変条件

  • pg_constraint システムカタログに対する AccessShareLock を取得してスキャンを行う。
  • 呼び出し元は、名前の競合を適切に処理するために必要なロックを保持していることが推奨される。

アルゴリズムまたは振る舞い

1. table_open を使用して pg_constraint カタログを AccessShareLock モードで開く。 2. ScanKeyInit を用いて、制約名（conname）と名前空間（connamespace）を条件とするスキャンキーを構築する。 3. ConstraintNameNspIndexId インデックスを使用して systable_beginscan を開始する。 4. systable_getnext で該当するタプルが存在するかを確認し、その結果を found に格納する。 5. systable_endscan および table_close を呼び出し、リソースを解放して結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/indexcmds.c:2656
  • **呼び出し先**:
  • table_open, table_close
  • ScanKeyInit
  • systable_beginscan, systable_getnext, systable_endscan

関連データ構造

  • pg_constraint: 制約情報を保持するシステムカタログ。
  • ConstraintNameNspIndexId: 制約名と名前空間による検索用インデックス。

失敗モードと境界条件

  • table_open が失敗した場合、PostgreSQL の標準的なエラー処理により中断される可能性がある。
  • 指定された名前空間が存在しない場合、あるいは制約名が NULL の場合の挙動は、基盤となるカタログアクセスの仕様に依存する。

保守担当者向けメモ

  • 本関数は制約名の重複チェックに使用される。名前の競合を防ぐための重要なユーティリティであるため、変更時にはカタログスキャンの整合性に注意すること。
