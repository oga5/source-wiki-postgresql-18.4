AioShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio_init.c:112

目的

PostgreSQL の非同期 I/O (AIO) サブシステムが共有メモリ上に必要とする総バイト数を計算します。

シグネチャまたは形状

Size AioShmemSize(void)

入力と出力

  • **入力**: なし（グローバル変数 io_max_concurrency および pgaio_method_ops を参照）。
  • **出力**: Size 型の共有メモリ総サイズ。

前提条件と不変条件

  • io_max_concurrency が -1 の場合、本関数内で AioChooseMaxConcurrency() を呼び出し、GUC 設定を動的に更新します。
  • 共有メモリの計算には add_size() を使用し、オーバーフローを防止します。

アルゴリズムまたは振る舞い

1. **GUC の自動調整**: io_max_concurrency が -1 の場合、AioChooseMaxConcurrency() で算出した値を io_max_concurrency に適用します。 2. **基本構造体の加算**: 以下の各コンポーネントに必要なサイズを合算します。

  • AioCtlShmemSize()
  • AioBackendShmemSize()
  • AioHandleShmemSize()
  • AioHandleIOVShmemSize()
  • AioHandleDataShmemSize()

    • 3. **メソッド固有リソースの加算**: pgaio_method_ops->shmem_size が定義されている場合、その戻り値を加算します。 4. **戻り値**: 計算された総サイズを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c:152（PostgreSQL の共有メモリ初期化シーケンス内）。
  • **呼び出し先**:
  • AioChooseMaxConcurrency()
  • SetConfigOption()
  • AioCtlShmemSize(), AioBackendShmemSize(), AioHandleShmemSize(), AioHandleIOVShmemSize(), AioHandleDataShmemSize()
  • pgaio_method_ops->shmem_size()

関連データ構造

  • PgAioCtl
  • PgAioBackend
  • PgAioHandle

失敗モードと境界条件

  • io_max_concurrency の設定適用に失敗した場合、PGC_S_OVERRIDE を使用して強制的に適用を試みます。
  • pgaio_method_ops が NULL の場合、メソッド固有リソースの加算はスキップされます。

保守担当者向けメモ

  • 本関数は共有メモリのレイアウトを決定する重要なエントリポイントです。io_max_concurrency の値がメモリ消費量に直結するため、変更時は各構造体のサイズ計算ロジックとの整合性に注意してください。
