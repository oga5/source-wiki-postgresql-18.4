OperatorGet

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_operator.c:123

目的

システムカタログ pg_operator から、指定された名前と型を持つ演算子の OID を検索し、その演算子が完全に定義されているか（シェル演算子ではないか）を判定する。

シグネチャまたは形状

Oid OperatorGet(const char *operatorName,
                Oid operatorNamespace,
                Oid leftObjectId,
                Oid rightObjectId,
                bool *defined)

入力と出力

  • **入力**:
  • operatorName: 演算子名。
  • operatorNamespace: 演算子が属する名前空間の OID。
  • leftObjectId: 左オペランドの型 OID（単項演算子の場合は InvalidOid）。
  • rightObjectId: 右オペランドの型 OID（単項演算子の場合は InvalidOid）。
  • defined: 演算子の定義状態を格納する出力用ポインタ。
  • **出力**:
  • 戻り値: 見つかった場合は演算子の OID、見つからない場合は InvalidOid。
  • defined: 演算子が存在し、かつ oprcode が有効な場合は true、それ以外は false。

前提条件と不変条件

  • OPERNAMENSP システムキャッシュを使用して検索を行うため、キャッシュが初期化されている必要がある。
  • defined ポインタは呼び出し元で有効なメモリ領域を指している必要がある。

アルゴリズムまたは振る舞い

1. SearchSysCache4 を使用し、OPERNAMENSP インデックス経由で pg_operator を検索する。 2. 該当するタプルが存在する場合:

  • GETSTRUCT で Form_pg_operator を取得する。
  • 戻り値として oid を保持する。
  • RegProcedureIsValid(oprform->oprcode) を評価し、演算子が実体を持つか（シェルではないか）を defined に設定する。
  • ReleaseSysCache でタプルを解放する。

    • 3. 該当するタプルが存在しない場合:

  • 戻り値として InvalidOid を返す。
  • defined に false を設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_operator.c 内の他関数から呼び出されている（具体的な関数名はソースコードの断片からは特定不可）。
  • **呼び出し先**: SearchSysCache4, GETSTRUCT, RegProcedureIsValid, ReleaseSysCache（いずれも PostgreSQL 内部 API）。

関連データ構造

  • pg_operator: 演算子の定義を保持するシステムカタログ。
  • Form_pg_operator: pg_operator カタログの行構造体。

失敗モードと境界条件

  • 演算子が見つからない場合、InvalidOid を返し、defined は false となる。
  • oprcode が InvalidOid である場合（シェル演算子）、defined は false となる。

保守担当者向けメモ

  • 本関数は pg_operator の検索における標準的なインターフェースである。
  • defined フラグは、PostgreSQL における「シェル演算子（プレースホルダー）」と「実体化された演算子」を区別するために重要である。
