ECPGfree_type

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/preproc/type.c:631

目的

ECPGtype 構造体によって保持されている型情報およびその関連データを再帰的に解放し、メモリリークを防ぐ。

シグネチャまたは形状

void ECPGfree_type(struct ECPGtype *type)

入力と出力

  • **入力**: struct ECPGtype *type （解放対象の型情報構造体へのポインタ）
  • **出力**: なし（void）

前提条件と不変条件

  • 引数 type は有効な ECPGtype 構造体を指している必要がある。
  • 構造体内のメンバー（type_name, size, struct_sizeof）は、動的に割り当てられたメモリであるか、あるいは NULL である必要がある。

アルゴリズムまたは振る舞い

1. type->type が単純型（IS_SIMPLE_TYPE）でない場合、型タグに応じて以下の処理を行う：

  • ECPGt_array: 配列要素（type->u.element）に対して再帰的に ECPGfree_type を呼び出す。多次元配列が検出された場合はエラーを報告する。
  • ECPGt_struct / ECPGt_union: ECPGfree_struct_member を呼び出し、構造体または共用体のメンバーリストを解放する。
  • その他: mmerror を通じてパースエラーを報告する。

    • 2. 構造体自身のメンバである type_name、size、struct_sizeof をそれぞれ free する。 3. 最後に type 構造体自体を free する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/preproc/type.c (複数箇所)
  • src/interfaces/ecpg/preproc/variable.c:358
  • **呼び出し先**:
  • ECPGfree_type (再帰呼び出し)
  • ECPGfree_struct_member
  • base_yyerror
  • mmerror
  • free (標準ライブラリ)

関連データ構造

  • ECPGtype: 型情報を保持する主要な構造体。
  • ECPGstruct_member: 構造体や共用体のメンバー定義を保持する構造体。

失敗モードと境界条件

  • **多次元配列**: ECPGt_array の要素がさらに ECPGt_array である場合、base_yyerror によりエラーが報告される。
  • **未知の型**: IS_SIMPLE_TYPE に該当せず、かつ定義済みの switch ケースに一致しない型コードが渡された場合、mmerror によりパースエラーが報告される。

保守担当者向けメモ

  • 本関数は再帰的に構造体を解放するため、ECPGtype の定義に変更を加えた場合は、本関数の switch 文および解放ロジックが最新の構造体定義と整合しているか確認すること。
  • メモリ管理は mm_alloc 等のラッパー経由で行われているため、free の呼び出しが適切であることを維持すること。
