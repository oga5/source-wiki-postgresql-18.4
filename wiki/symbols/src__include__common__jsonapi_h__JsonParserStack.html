JsonParserStack

定義

  • 種別: struct
  • 定義箇所: src/include/common/jsonapi.h:63

目的

JSON 解析プロセスにおいて、解析の予測状態を管理するためのスタック構造を提供します。

シグネチャまたは形状

不透明な構造体として定義されています。

typedef struct JsonParserStack JsonParserStack;

入力と出力

  • 入力: src/common/jsonapi.c 内の各操作関数（push_prediction, pop_prediction 等）に対して、JsonParserStack へのポインタが渡されます。
  • 出力: 解析の予測状態を保持・更新し、必要に応じてスタックのトップ要素を返します。

前提条件と不変条件

  • JsonLexContext 内の pstack メンバとして保持され、解析のライフサイクルを通じて管理されます。
  • スタック操作関数（push/pop/next/have）を介してのみアクセスされることが想定されています。

アルゴリズムまたは振る舞い

JSON の構造（オブジェクトや配列のネスト）に応じた予測状態をスタック形式で管理します。

  • push_prediction: 新しい予測状態をスタックに積みます。
  • pop_prediction: スタックのトップから予測状態を取り除きます。
  • next_prediction: 次の予測状態を参照します。
  • have_prediction: スタックに要素が存在するかを確認します。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/jsonapi.c 内の解析ロジック。
  • 呼び出し先: src/common/jsonapi.c 内の push_prediction, pop_prediction, next_prediction, have_prediction 関数。
  • 補足: JsonLexContext 構造体（src/include/common/jsonapi.h:114）がこの構造体へのポインタを保持しています。

関連データ構造

  • JsonLexContext: 本構造体を保持し、解析のコンテキストを管理するメイン構造体。
  • td_entry: スタックに格納される予測エントリの型（src/common/jsonapi.c 内で定義）。

失敗モードと境界条件

  • メモリ不足: ALLOC0 によるスタックの初期化時に失敗する可能性があります。
  • スタックアンダーフロー: 空のスタックに対して pop や next を呼び出した場合の挙動は、実装側のガード条件に依存します。

保守担当者向けメモ

  • 本構造体の内部メンバは jsonapi.h では非公開です。詳細な実装は src/common/jsonapi.c を参照してください。
  • JsonLexContext のライフサイクルと同期してメモリが管理されるため、個別の解放処理は不要ですが、freeJsonLexContext が適切に呼ばれることを確認してください。
