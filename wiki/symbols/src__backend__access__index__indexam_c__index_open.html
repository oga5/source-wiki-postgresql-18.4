index_open

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/indexam.c:132

目的

インデックスリレーションをオープンし、指定されたロックモードを適用した上で、対象が有効なインデックスであることを検証する。

シグネチャまたは形状

Relation index_open(Oid relationId, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relationId (Oid): オープンするインデックスリレーションのOID。
  • lockmode (LOCKMODE): リレーション取得時に適用するロックモード。
  • **出力**:
  • Relation: オープンされたインデックスリレーションへのポインタ。

前提条件と不変条件

  • 指定された relationId は、システムカタログ上で有効なインデックスリレーションを指している必要がある。
  • validate_relation_kind による検証が成功すること。

アルゴリズムまたは振る舞い

1. relation_open(relationId, lockmode) を呼び出し、リレーションを取得する。 2. validate_relation_kind(r) を呼び出し、取得したリレーションがインデックスとして適切か（インデックス種別であるか）を検証する。 3. 検証済みの Relation ポインタを返す。

関連データ構造

  • Relation: インデックスの物理的および論理的属性を保持する構造体。

失敗モードと境界条件

  • relation_open がリレーションを見つけられない場合、またはロック取得に失敗した場合はエラーが発生する。
  • validate_relation_kind がリレーションがインデックスではないと判断した場合、エラーが発生する。

保守担当者向けメモ

  • 本関数はインデックスアクセスの入り口であり、必ず validate_relation_kind を通すことで、誤ったリレーション型に対する操作を防止している。
  • index_close と対で使用されることが想定されている。
