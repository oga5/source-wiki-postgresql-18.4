pgstat_report_wal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_wal.c:45

目的

WAL（Write-Ahead Logging）およびI/O統計情報を収集し、共有メモリへフラッシュするためのメインエントリポイントです。

シグネチャまたは形状

void pgstat_report_wal(bool force)

入力と出力

  • **入力**:
  • force (bool): true の場合、ロック取得時に待機を行います。false の場合、ロック取得に失敗しても待機せず即座に処理を中断します。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • 本関数は、統計情報の更新頻度が高い環境で呼び出されることを想定しています。
  • force=false の場合、パフォーマンスへの影響を最小限にするため、ロック取得の待機を回避する設計となっています。

アルゴリズムまたは振る舞い

1. force 引数に基づき、ロック待機フラグ nowait を設定します。 2. pgstat_wal_flush_cb(nowait) を呼び出し、WAL統計の差分を計算して共有メモリへ反映します。 3. pgstat_flush_backend(nowait, PGSTAT_BACKEND_FLUSH_WAL) を呼び出し、WAL関連のバックエンド統計をフラッシュします。 4. pgstat_flush_io(nowait) を呼び出し、I/O統計をフラッシュします。 5. pgstat_flush_backend(nowait, PGSTAT_BACKEND_FLUSH_IO) を呼び出し、I/O関連のバックエンド統計をフラッシュします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/bgwriter.c
  • src/backend/postmaster/checkpointer.c
  • src/backend/postmaster/walsummarizer.c
  • src/backend/postmaster/walwriter.c
  • src/backend/replication/walreceiver.c
  • **呼び出し先**:
  • pgstat_wal_flush_cb
  • pgstat_flush_backend
  • pgstat_flush_io

関連データ構造

  • WalUsage: プロセスごとのWAL使用量を保持する構造体。
  • pgWalUsage: プロセスローカルなWAL使用量カウンタ。

失敗モードと境界条件

  • force=false の場合、内部のロック取得が競合すると統計のフラッシュがスキップされる可能性があります。

保守担当者向けメモ

  • 本関数は pgstat モジュールの統計フラッシュ処理を統合的に実行します。
  • _cb 接尾辞を持つ関数は pgstat モジュールから動的に呼び出される設計であるため、変更時はコールバックインターフェースとの整合性に注意してください。
