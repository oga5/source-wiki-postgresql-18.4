AnalyzeAttrFetchFunc

定義

  • 種別: typedef
  • 定義箇所: src/include/commands/vacuum.h:111

目的

ANALYZE コマンド実行時、統計情報の計算対象となる列のデータを取得するための関数ポインタ型です。

シグネチャまたは形状

typedef bool (*AnalyzeAttrFetchFunc) (VacAttrStatsP stats,
                                      int rownum,
                                      Datum *value,
                                      bool *isNull);

※シグネチャの詳細は、本型を利用する AnalyzeAttrComputeStatsFunc の定義および analyze.c 等の実装文脈から推測される標準的な形状です。

入力と出力

  • 入力:
  • stats: 統計対象の列情報（VacAttrStatsP）。
  • rownum: 取得対象の行番号。
  • 出力:
  • value: 取得した値（Datum）を格納するポインタ。
  • isNull: 値が NULL かどうかを示すフラグを格納するポインタ。
  • 戻り値: 取得に成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • ANALYZE 処理のコンテキスト内で呼び出される必要があります。
  • VacAttrStatsP は有効なリレーションおよび列情報を含んでいる必要があります。

アルゴリズムまたは振る舞い

ANALYZE 処理において、サンプリングされた行から特定の列の値を抽出する抽象化レイヤーとして機能します。具体的なデータ取得ロジックは、データ型やストレージ形式に応じて実装側（analyze.c や各型の typanalyze 実装）で提供されます。

呼び出し元と呼び出し先

  • 呼び出し元（直接的な参照先）:
  • src/backend/commands/analyze.c
  • src/backend/tsearch/ts_typanalyze.c
  • src/backend/utils/adt/array_typanalyze.c
  • src/backend/utils/adt/rangetypes_typanalyze.c
  • 呼び出し先:
  • ソースコードの断片からは、具体的な呼び出し先関数は特定できません。

関連データ構造

  • VacAttrStats: ANALYZE 中の列統計情報を保持する構造体。
  • AnalyzeAttrComputeStatsFunc: 本関数型を引数として受け取り、統計計算を実行する関数型。

失敗モードと境界条件

  • 行番号が範囲外の場合や、データ取得中にエラーが発生した場合は false を返すことが想定されます。

保守担当者向けメモ

  • 本型は ANALYZE の拡張性を支えるインターフェースです。新しいデータ型に対して統計収集ロジックを追加する場合、この関数型に適合する取得関数を実装する必要があります。
  • 呼び出し元と呼び出し先の詳細なフローについては、src/backend/commands/analyze.c の compute_stats 関連のロジックを参照してください。
