build_mss

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/mcv.c:346

目的

多変量統計情報の構築プロセスにおいて、複数の列を同時にソートするための MultiSortSupport 構造体を初期化および設定する。

シグネチャまたは形状

static MultiSortSupport build_mss(StatsBuildData *data)

入力と出力

  • **入力**: StatsBuildData *data（統計構築対象の列情報や型情報を含むデータ構造）
  • **出力**: MultiSortSupport（設定済みのソートサポートオブジェクト）

前提条件と不変条件

  • data->nattnums が有効な列数であること。
  • 各列の型情報（attrtypid）に対して、TYPECACHE_LT_OPR を用いたキャッシュルックアップが成功し、有効な比較演算子（lt_opr）が取得できること。

アルゴリズムまたは振る舞い

1. multi_sort_init を呼び出し、指定された列数分の MultiSortSupport を初期化する。 2. 各列に対してループ処理を行い、lookup_type_cache で型の比較演算子を取得する。 3. 比較演算子が InvalidOid の場合はエラーを発生させる。 4. multi_sort_add_dimension を使用して、各列のインデックス、比較演算子、および照合順序（attrcollid）を MultiSortSupport に追加する。 5. 設定済みの mss を返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/statistics/mcv.c 内の統計構築ロジック（行194付近）。
  • **呼び出し先**:
  • multi_sort_init
  • lookup_type_cache
  • multi_sort_add_dimension
  • elog（エラー発生時）

関連データ構造

  • MultiSortSupport: 複数列のソートを管理するための構造体。
  • StatsBuildData: 統計構築に必要なメタデータを保持する構造体。
  • TypeCacheEntry: 型のキャッシュ情報。

失敗モードと境界条件

  • lookup_type_cache が指定された型に対して有効な比較演算子を返せない場合、elog(ERROR, ...) により処理が中断される。

保守担当者向けメモ

  • 本関数は多変量統計のソート順序を決定する基盤である。新しいデータ型をサポートする場合やソートロジックを変更する場合は、multi_sort_add_dimension への引数を確認すること。
