TransactionIdOlder

定義

  • 種別: static inline function
  • 定義箇所: src/include/access/transam.h:333

目的

2つのトランザクションID（TransactionId）を比較し、より古い（先に発生した）方のIDを返します。

シグネチャまたは形状

static inline TransactionId TransactionIdOlder(TransactionId a, TransactionId b)

入力と出力

  • **入力**: 比較対象となる2つの TransactionId 型の値 a および b。
  • **出力**: a と b のうち、TransactionIdPrecedes の判定に基づき「より古い」と判断された方の TransactionId。

前提条件と不変条件

  • 入力されるIDは TransactionIdIsValid マクロによって有効性がチェックされます。
  • トランザクションIDの循環（ラップアラウンド）を考慮した比較ロジックである必要があります。

アルゴリズムまたは振る舞い

1. a が無効な場合、b を返します。 2. b が無効な場合、a を返します。 3. TransactionIdPrecedes(a, b) が真であれば a を返し、そうでなければ b を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/procarray.c 内の複数の箇所で、xmin や oldest_considered_running などのトランザクション境界を更新する際に呼び出されています。
  • **呼び出し先**: TransactionIdIsValid および TransactionIdPrecedes を内部で呼び出しています。

関連データ構造

  • TransactionId: 32ビットのトランザクション識別子。

失敗モードと境界条件

  • いずれか一方または両方の引数が InvalidTransactionId である場合、有効な方のIDを優先して返します。両方とも無効な場合の挙動は、TransactionIdIsValid の定義に依存します。

保守担当者向けメモ

  • 本関数は src/include/access/transam.h で定義されたインライン関数であり、パフォーマンスが重視される箇所で使用されています。
  • 比較ロジックは TransactionIdPrecedes に依存しているため、XIDのラップアラウンドに関する仕様変更時には当該マクロと併せて確認が必要です。
