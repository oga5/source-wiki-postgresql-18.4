pg_get_line_append

定義

  • 種別: function
  • 定義箇所: src/common/pg_get_line.c:123

目的

指定された FILE ストリームから1行を読み込み、既存の StringInfo バッファの末尾に追記する。継続行の処理や、動的に拡張可能なバッファへの逐次読み込みに適している。

シグネチャまたは形状

bool pg_get_line_append(FILE *stream, StringInfo buf, PromptInterruptContext *prompt_ctx);

入力と出力

  • **入力**:
  • stream: 読み込み対象のファイルストリーム。
  • buf: 読み込んだデータを追記する StringInfo 構造体。
  • prompt_ctx: 割り込み制御用コンテキスト（NULL指定可能）。
  • **出力**:
  • 戻り値: 成功時は true、EOFまたはエラー発生時は false を返す。
  • buf: 読み取った内容が追記され、末尾にヌル文字が挿入される。

前提条件と不変条件

  • buf は初期化済みであること。
  • 割り込みが発生した場合、関数はバッファの状態を呼び出し前の状態（orig_len）にロールバックする。
  • 読み込み成功時、バッファの末尾には必ずヌル文字が保持される。

アルゴリズムまたは振る舞い

1. 現在のバッファ長を orig_len として記録する。 2. prompt_ctx が提供されている場合、sigsetjmp を設定し、割り込み時にバッファを復元して false を返す準備をする。 3. ループ内で fgets を使用し、バッファの空き領域にデータを読み込む。 4. 読み込みのたびに buf->len を更新し、改行文字 \n が含まれていれば即座に true を返す。 5. 改行が見つからない場合は enlargeStringInfo でバッファを拡張し、読み込みを継続する。 6. fgets が NULL を返した場合、または I/O エラーが発生した場合は、バッファを orig_len に戻し false を返す。

関連データ構造

  • StringInfo: 動的文字列バッファを管理する構造体。
  • PromptInterruptContext: 割り込み発生時のジャンプ先と状態を保持する構造体。

失敗モードと境界条件

  • **I/O エラー**: ferror が真を返す場合、読み込みを中断し false を返す。
  • **データ未取得**: EOF に到達した際、一度もデータを読み取れなかった場合は false を返す。
  • **割り込み**: SIGINT 等により longjmp が発生した場合、収集したデータは破棄され false を返す。
  • **改行なしの EOF**: EOF に到達したが、それまでに何らかのデータが読み取られていた場合は、改行がなくても true を返す。

保守担当者向けメモ

  • fgets と同様、末尾の改行文字は削除されずにバッファに残る。
  • 呼び出し元で pfree を行う必要はない（StringInfo の管理方針に従う）。
  • 大量読み込み時のパフォーマンスを考慮する場合、バッファの再利用を検討すること。
