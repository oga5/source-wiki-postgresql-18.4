extractPageMap

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/parsexlog.c:65

目的

指定された範囲のWAL（Write-Ahead Log）を読み込み、各レコードを解析してデータブロックの変更情報を抽出・記録する。

シグネチャまたは形状

void extractPageMap(const char *datadir, XLogRecPtr startpoint, int tliIndex,
                    XLogRecPtr endpoint, const char *restoreCommand)

入力と出力

  • **入力**:
  • datadir: WALファイルが格納されているデータディレクトリのパス。
  • startpoint: WAL解析を開始するLSN（Log Sequence Number）。
  • tliIndex: タイムラインインデックス。
  • endpoint: WAL解析を終了するLSN。
  • restoreCommand: WALセグメントをアーカイブから復元するためのコマンド文字列。
  • **出力**: なし（副作用として内部的な変更マップを更新する）。

前提条件と不変条件

  • endpoint は、WALレコードの境界と正確に一致している必要がある。一致しない場合は致命的エラーとなる。
  • XLogReader の割り当てに失敗した場合は致命的エラーとなる。

アルゴリズムまたは振る舞い

1. XLogReaderAllocate を使用し、SimpleXLogPageRead をコールバックとしてWALリーダーを初期化する。 2. XLogBeginRead で開始位置を設定する。 3. XLogReadRecord をループ内で呼び出し、endpoint に到達するまでWALレコードを順次読み込む。 4. 読み込んだ各レコードに対し extractPageInfo を呼び出し、変更されたブロック情報を抽出する。 5. 読み込み完了後、XLogReaderFree でリーダーを解放し、開いているファイルディスクリプタ xlogreadfd を閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c:488
  • **呼び出し先**:
  • XLogReaderAllocate
  • XLogBeginRead
  • XLogReadRecord
  • extractPageInfo
  • XLogReaderFree
  • SimpleXLogPageRead (コールバックとして登録)

関連データ構造

  • XLogReaderState: WAL読み込みの状態を管理する構造体。
  • XLogPageReadPrivate: WAL読み込み時に使用されるプライベートデータ（tliIndex, restoreCommand を保持）。

失敗モードと境界条件

  • **メモリ不足**: XLogReaderAllocate が失敗した場合、pg_fatal を呼び出す。
  • **読み込みエラー**: XLogReadRecord がレコードを返さない場合、エラーメッセージと共に pg_fatal を呼び出す。
  • **境界不一致**: 最終的な EndRecPtr が endpoint と一致しない場合、pg_fatal を呼び出す。

保守担当者向けメモ

  • 本関数は pg_rewind のWAL解析フェーズのメインループである。
  • xlogreadfd というグローバル変数（またはファイルスコープ変数）を直接操作してファイルディスクリプタを管理している点に注意すること。
