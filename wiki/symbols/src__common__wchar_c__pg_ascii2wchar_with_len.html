pg_ascii2wchar_with_len

定義

  • 種別: function
  • 定義箇所: src/common/wchar.c:72

目的

ASCII文字列を pg_wchar 型のワイド文字配列へ変換します。

シグネチャまたは形状

int pg_ascii2wchar_with_len(const unsigned char *from, pg_wchar *to, int len)

入力と出力

  • **入力**:
  • from: 変換元のASCII文字列（unsigned char *）。
  • to: 変換先となる pg_wchar 型のバッファ。
  • len: 処理対象とする最大バイト数。
  • **出力**:
  • 戻り値: 実際に変換された文字数（int）。

前提条件と不変条件

  • to バッファは、少なくとも len + 1 個の pg_wchar を格納できる十分な領域が必要です。
  • 入力文字列がヌル終端されている場合、ヌル文字に到達した時点で変換を停止します。

アルゴリズムまたは振る舞い

1. len が 0 より大きく、かつ from が指す文字がヌル文字でない間、ループを繰り返します。 2. from の各バイトを to にコピーし、ポインタをインクリメントします。 3. len をデクリメントし、変換した文字数 cnt をインクリメントします。 4. ループ終了後、to の末尾にヌル文字（0）を書き込みます。 5. 変換した文字数 cnt を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/wchar.c:2087 において、PG_SQL_ASCII エンコーディングの関数テーブル定義の一部として参照されています。
  • **呼び出し先**: 外部関数への呼び出しはありません。

関連データ構造

  • pg_wchar: PostgreSQL内部でワイド文字を表現するための型。

失敗モードと境界条件

  • len が 0 の場合、変換は行われず 0 を返します。
  • 入力文字列が len より短い場合、ヌル文字に到達した時点で処理を終了します。

保守担当者向けメモ

  • 本関数は PG_SQL_ASCII エンコーディング専用の変換ロジックです。
  • 変換先バッファのオーバーフローを防ぐため、呼び出し側で適切なバッファサイズを確保してください。
