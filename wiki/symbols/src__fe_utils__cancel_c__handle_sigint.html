handle_sigint

定義

  • 種別: function
  • 定義箇所: src/fe_utils/cancel.c:152

目的

ユーザーからの割り込み（SIGINT）を捕捉し、実行中のデータベースクエリに対してキャンセル要求を送信するシグナルハンドラです。

シグネチャまたは形状

static void handle_sigint(SIGNAL_ARGS)

入力と出力

  • **入力**: SIGNAL_ARGS（シグナルハンドラとしての標準引数）。
  • **出力**: なし（void）。副作用として標準エラー出力へメッセージを出力し、グローバル状態を更新します。

前提条件と不変条件

  • cancelConn が有効な場合、PQcancel を実行してバックエンドへキャンセル信号を送信します。
  • CancelRequested フラグは本関数内で true に設定されます。
  • シグナルハンドラ内での実行に耐えうるよう、write_stderr を介した安全な出力が行われます。

アルゴリズムまたは振る舞い

1. CancelRequested フラグを true に設定します。 2. cancel_callback が登録されている場合、それを実行します。 3. cancelConn が存在する場合、PQcancel を呼び出します。

  • 成功時: cancel_sent_msg を標準エラー出力に書き出します。
  • 失敗時: cancel_not_sent_msg とエラー内容（errbuf）を標準エラー出力に書き出します。

呼び出し元と呼び出し先

  • **呼び出し元**: pqsignal(SIGINT, handle_sigint) を通じて、システムシグナルハンドラとして登録されます（src/fe_utils/cancel.c:189）。
  • **呼び出し先**:
  • cancel_callback()（登録されている場合）
  • PQcancel()（libpq ライブラリ）
  • write_stderr()

関連データ構造

  • CancelRequested: キャンセル要求の状態を保持するグローバルフラグ。
  • cancelConn: キャンセル対象の接続を保持する PGcancel オブジェクト。
  • cancel_callback: 割り込み時に実行されるオプションのコールバック関数。

失敗モードと境界条件

  • cancelConn が NULL の場合、キャンセル要求の送信はスキップされます。
  • PQcancel が失敗した場合、エラーメッセージが errbuf に格納され、標準エラー出力へ出力されます。

保守担当者向けメモ

  • 本関数はシグナルハンドラとして動作するため、非同期シグナル安全な関数のみを使用する必要があります。
  • CancelRequested フラグは自動リセットされないため、呼び出し側で適切に管理してください。
  • ポインタ操作（cancelConn 等）を行う際は、他の関数による解放処理と競合しないよう注意が必要です。
