MultiXactStatus

定義

  • 種別: enum
  • 定義箇所: src/include/access/multixact.h:37

目的

タプルに対して複数のトランザクションが同時に保持するロックの種類（共有行ロック等）を定義・識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	MultiXactStatusForKeyShare = 0x00,
	MultiXactStatusForShare = 0x01,
	MultiXactStatusForNoKeyUpdate = 0x02,
	MultiXactStatusForUpdate = 0x03,
	MultiXactStatusNoKeyUpdate = 0x04,
	MultiXactStatusUpdate = 0x05,
} MultiXactStatus;

入力と出力

  • 入力: heapam.c 等のロック取得処理において、ロックモードから本列挙型への変換が行われます。
  • 出力: MultiXactIdWait や ConditionalMultiXactIdWait 等の関数へ引数として渡され、ロックの待機判定や状態確認に利用されます。

前提条件と不変条件

  • 各値は特定のロック強度や更新タイプに対応しており、0x00 から 0x05 までの整数値として表現されます。
  • MultiXactStatusNoKeyUpdate は「キー列に影響しない更新」、MultiXactStatusUpdate は「その他の更新および削除」を指します。

アルゴリズムまたは振る舞い

  • heapam.c 内の get_mxact_status_for_lock 関数により、タプルロックモードから本列挙型へのマッピングが行われます。
  • MultiXactId が保持するメンバーのロック状態を判定する際、この列挙値が比較基準として使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam.c
  • MultiXactIdWait
  • ConditionalMultiXactIdWait
  • get_mxact_status_for_lock (内部での変換処理)
  • **呼び出し先**: 本シンボル自体は列挙型定義であり、関数を呼び出す側です。

関連データ構造

  • MultiXactId: マルチトランザクションを識別するID。
  • MultiXactMember: トランザクションIDと MultiXactStatus を保持する構造体。

失敗モードと境界条件

  • 定義外の整数値がキャストされて渡された場合、期待されるロック判定ロジックが正しく動作しない可能性があります。

保守担当者向けメモ

  • MultiXactStatus の値を追加・変更する場合は、heapam.c 内の変換関数（get_mxact_status_for_lock 等）およびロックモードとの整合性を必ず確認してください。
