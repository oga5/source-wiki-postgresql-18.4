sql_conn

定義

  • 種別: function
  • 定義箇所: contrib/oid2name/oid2name.c:294

目的

PostgreSQL データベースへの接続を確立し、初期設定としてセキュアな search_path を適用する。

シグネチャまたは形状

PGconn *sql_conn(struct options *my_opts)

入力と出力

  • **入力**: my_opts (struct options *) - 接続に必要なホスト名、ポート、ユーザー名、データベース名、プログラム名を含む構造体。
  • **出力**: PGconn * - 確立されたデータベース接続ハンドル。接続失敗時はエラーメッセージを出力してプロセスを終了する。

前提条件と不変条件

  • my_opts は有効なメモリ領域を指している必要がある。
  • 接続時にパスワードが必要な場合、標準入力からパスワードを読み取る対話的な処理が行われる。

アルゴリズムまたは振る舞い

1. my_opts から接続パラメータ（ホスト、ポート、ユーザー、DB名等）を抽出し、PQconnectdbParams を呼び出す。 2. 接続が拒否され、かつパスワードが必要な場合は、simple_prompt を用いてパスワードを取得し、再接続を試みる（ループ処理）。 3. 接続状態を PQstatus で確認し、CONNECTION_BAD の場合はエラーを出力して終了する。 4. 接続成功後、ALWAYS_SECURE_SEARCH_PATH_SQL を実行して search_path を安全な値に設定する。 5. 設定が成功すれば接続ハンドルを返し、失敗すれば接続を閉じて終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/oid2name/oid2name.c:606 (main関数内)
  • **呼び出し先**: PQconnectdbParams, PQstatus, PQconnectionNeedsPassword, PQfinish, simple_prompt, pg_fatal, pg_log_error, PQexec, PQresultStatus, PQerrorMessage, PQclear, exit

関連データ構造

  • struct options: 接続設定を保持する構造体。
  • PGconn: libpq の接続ハンドル。
  • PGresult: libpq のクエリ実行結果。

失敗モードと境界条件

  • データベース接続失敗時: pg_fatal または pg_log_error を経由してプロセスが終了する。
  • search_path 設定失敗時: PQclear および PQfinish を実行後、プロセスが終了する。

保守担当者向けメモ

  • 本関数は libpq の接続ループを内包しており、パスワード入力が必要なケースを自動的に処理する。
  • ALWAYS_SECURE_SEARCH_PATH_SQL の実行は、セキュリティ上の不変条件を維持するために必須である。
