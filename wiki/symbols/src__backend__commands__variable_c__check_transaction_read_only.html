check_transaction_read_only

定義

  • 種別: function
  • 定義箇所: src/backend/commands/variable.c:545

目的

トランザクションの読み取り専用モード（transaction_read_only）への変更要求が、SQL標準および現在のシステム状態に照らして妥当であるかを検証します。

シグネチャまたは形状

bool check_transaction_read_only(bool *newval, void **extra, GucSource source)

入力と出力

  • **入力**:
  • newval: 設定しようとしている新しい値（読み取り専用か否か）。
  • extra: GUCサブシステム用の拡張データ（本関数では未使用）。
  • source: 設定のソース（GUCサブシステムより提供）。
  • **出力**:
  • 成功時は true を返し、検証に失敗した場合は false を返します。失敗時には GUC_check_errmsg 等を通じてエラー詳細が報告されます。

前提条件と不変条件

  • 本関数は、読み取り専用モードから読み書き可能（read-write）モードへ変更しようとする場合にのみ、制約チェックを実行します。
  • InitializingParallelWorker が真である場合、検証はスキップされます。

アルゴリズムまたは振る舞い

1. newval が false（読み書き可能への変更）であり、かつ現在読み取り専用トランザクション中である場合に以下のチェックを行います。 2. **サブトランザクションの確認**: IsSubTransaction() が真であれば、サブトランザクション内でのモード変更は許可されず、エラーを返します。 3. **スナップショットの確認**: FirstSnapshotSet が真であれば、既にクエリが実行されていると見なし、モード変更は許可されずエラーを返します。 4. **リカバリ状態の確認**: RecoveryInProgress() が真であれば、リカバリ中のモード変更は許可されずエラーを返します。 5. 上記のいずれにも該当しない場合、または newval が true の場合は true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: GUCサブシステム（src/backend/utils/misc/guc_tables.c）からフックとして呼び出されます。
  • **呼び出し先**:
  • IsTransactionState(), IsSubTransaction(), RecoveryInProgress() などの状態確認関数。
  • GUC_check_errcode(), GUC_check_errmsg() などのエラー報告マクロ。

関連データ構造

  • XactReadOnly: 現在のトランザクションが読み取り専用かどうかを示すグローバル変数。
  • FirstSnapshotSet: 現在のトランザクションで最初のスナップショットが取得済みかを示すグローバル変数。

失敗モードと境界条件

  • **ERRCODE_ACTIVE_SQL_TRANSACTION**: サブトランザクション内での変更、または最初のクエリ実行後の変更を試みた場合に発生します。
  • **ERRCODE_FEATURE_NOT_SUPPORTED**: リカバリ中に読み書き可能モードへの変更を試みた場合に発生します。

保守担当者向けメモ

  • 本関数は SET TRANSACTION READ WRITE 等のコマンド実行時に呼び出されます。
  • トランザクションの進行状況に依存するため、並列ワーカーの初期化やリカバリ処理との整合性に注意してください。
