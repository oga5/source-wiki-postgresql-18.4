window_row_number

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/windowfuncs.c:83

目的

SQLの ROW_NUMBER() ウィンドウ関数を実装し、現在のパーティション内における行の連番（1から始まる）を算出します。

シグネチャまたは形状

Datum window_row_number(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（ウィンドウ関数実行用の標準インターフェース）。
  • **出力**: int64 型の現在の行番号（1-based index）。

前提条件と不変条件

  • windowapi.h で定義された WindowObject コンテキストが有効である必要があります。
  • WinGetCurrentPosition を使用して、現在のパーティション内での相対的な行位置を取得します。

アルゴリズムまたは振る舞い

1. PG_WINDOW_OBJECT() を介して現在のウィンドウオブジェクトを取得します。 2. WinGetCurrentPosition(winobj) を呼び出し、現在の行のパーティション内での位置（0-based）を取得します。 3. WinSetMarkPosition(winobj, curpos) を呼び出し、現在の位置をマークします。 4. 取得した位置に1を加算し、PG_RETURN_INT64 を通じて結果を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLのウィンドウ関数実行エンジン（src/backend/executor/nodeWindowAgg.c 等）。
  • **呼び出し先**:
  • WinGetCurrentPosition（ウィンドウAPI）
  • WinSetMarkPosition（ウィンドウAPI）
  • 注: 呼び出し元・呼び出し先の詳細なコールグラフは、提供されたソース抜粋の範囲外です。

関連データ構造

  • WindowObject: ウィンドウ関数の実行状態を保持する不透明なハンドル。

失敗モードと境界条件

  • パーティションの先頭行では curpos が 0 となり、結果として 1 が返されます。
  • ウィンドウ関数が適切に初期化されていない場合、WinGetCurrentPosition 等のAPI呼び出しが失敗する可能性があります。

保守担当者向けメモ

  • 本関数は window_row_number_support 関数と対になっており、オプティマイザがフレーム指定を最適化する際に参照されます。
  • WinSetMarkPosition は、ウィンドウ関数の実行状態を追跡するために重要です。変更を加える際は、ウィンドウAPIの仕様に従っているか確認してください。
