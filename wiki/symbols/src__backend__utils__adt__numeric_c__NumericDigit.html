NumericDigit

定義

  • 種別: typedef
  • 定義箇所: src/backend/utils/adt/numeric.c:82

目的

PostgreSQL の NUMERIC 型において、数値を構成する個々の「桁（digit）」を格納するための基本データ型です。

シグネチャまたは形状

typedef signed char NumericDigit;

※注: ソースコード内には typedef int16 NumericDigit; との記述も存在し、コンパイル環境や構成に応じて定義が選択される可能性があります。

入力と出力

  • 入力: 0 から NBASE - 1 までの範囲の整数値。
  • 出力: Numeric 型の内部表現の一部として、数値の各桁を保持します。

前提条件と不変条件

  • 各 NumericDigit は、基数 NBASE（現在は 10000）の範囲内に収まる必要があります。
  • 符号付きの型として定義されており、数値の表現において正負の管理に使用されます。

アルゴリズムまたは振る舞い

  • NUMERIC 型の内部構造体（NumericData 等）において、n_data 配列の要素として使用されます。
  • 算術演算時には、この型を保持する配列が NumericVar 構造体を通じて展開・操作され、計算結果が再びこの形式にパックされます。

呼び出し元と呼び出し先

  • 呼び出し元: numeric.c 内の算術演算関数（add_var, sub_var 等）および数値変換関数。
  • 呼び出し先: 直接的な関数呼び出し関係ではなく、NumericVar 構造体や NumericData 構造体のメンバとして参照されます。

関連データ構造

  • NumericData: NUMERIC 型のオンディスク表現。
  • NumericVar: 算術演算に使用されるメモリ上の作業用構造体。
  • NBASE: 数値計算の基数（10000）。

失敗モードと境界条件

  • NBASE を超える値を格納しようとした場合、数値表現の整合性が失われます。
  • NumericVar 構造体でメモリを確保する際、NumericDigit の配列サイズが不足するとバッファオーバーランのリスクがあります。

保守担当者向けメモ

  • NumericDigit の定義が signed char と int16 の両方で言及されているため、変更を加える際は src/backend/utils/adt/numeric.c 内の条件付きコンパイル設定を確認してください。
  • NBASE=10000 であるため、1 つの NumericDigit に 4 桁の 10 進数が格納されるという設計上の不変条件を維持してください。
