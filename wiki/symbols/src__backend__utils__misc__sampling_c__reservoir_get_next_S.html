reservoir_get_next_S

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/sampling.c:146

目的

リザーバーサンプリングにおいて、現在処理中のレコード数（t）に基づき、次に選択対象となるレコードまでのスキップ数（S）を計算する。

シグネチャまたは形状

double reservoir_get_next_S(ReservoirState rs, double t, int n);

入力と出力

  • **入力**:
  • rs: サンプリング状態を保持する ReservoirState 構造体。乱数生成器の状態および内部変数 W を含む。
  • t: 現在までに処理したレコード数。
  • n: 抽出対象のサンプル数。
  • **出力**:
  • S: 次に選択すべきレコードまでのスキップ数（double）。

前提条件と不変条件

  • t >= n であることが想定される。
  • sampler_random_fract は 0 を含まない範囲の乱数を生成する。
  • rs->W はアルゴリズムの実行状態を保持し、呼び出し間で更新される。

アルゴリズムまたは振る舞い

Vitterのアルゴリズムに基づき、t の値に応じて処理を切り替える。 1. **Algorithm X**: t <= 22.0 * n の場合、単純な反復処理により S を決定する。 2. **Algorithm Z**: t が大きい場合、より効率的な計算を行う。

  • 乱数 U と X を生成し、S を floor(X) で初期化する。
  • 収束条件を満たすまで W を更新しながら反復し、最終的な S を算出する。
  • 内部状態 rs->W を更新して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/commands/analyze.c
  • **呼び出し先**:
  • sampler_random_fract (乱数生成)
  • math.h の exp, log, floor

関連データ構造

  • ReservoirState: サンプリングの状態（乱数生成器の状態 randstate および W）を保持する構造体。

失敗モードと境界条件

  • t が小さい場合、Algorithm X が適用される。
  • t が大きい場合、Algorithm Z が適用される。
  • 浮動小数点演算（log, exp）を含むため、入力値の極端な偏りや精度限界に依存する。

保守担当者向けメモ

  • 本関数は Vitter の論文に基づくアルゴリズムを実装している。
  • rs->W は状態保持変数であり、呼び出し元で適切に管理された ReservoirState を渡す必要がある。
  • 浮動小数点数の対数計算が含まれるため、変更時は精度への影響を考慮すること。
