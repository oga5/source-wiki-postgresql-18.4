ltxtq_out

定義

  • 種別: function
  • 定義箇所: contrib/ltree/ltxtquery_io.c:612

目的

PostgreSQL の ltxtquery 型を、人間が読みやすい中置記法（infix notation）の文字列形式に変換して出力する。

シグネチャまたは形状

Datum ltxtq_out(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して渡される ltxtquery 型のポインタ。
  • **出力**: 変換されたクエリ文字列（char *）。PG_RETURN_POINTER マクロにより Datum として返却される。

前提条件と不変条件

  • 入力された ltxtquery 構造体の size が 0 であってはならない（0 の場合は構文エラーとして処理される）。
  • 内部処理において INFIX 構造体を使用し、動的にメモリを確保して文字列を構築する。

アルゴリズムまたは振る舞い

1. 入力された ltxtquery のサイズを検証し、0 の場合はエラーを報告する。 2. INFIX 構造体を初期化し、クエリのオペランドおよび演算子配列へのポインタを設定する。 3. infix 関数を呼び出し、内部的な逆ポーランド記法（ITEM 配列）を再帰的に走査して中置記法の文字列を生成する。 4. 生成された文字列バッファを返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の SQL インターフェース（PG_FUNCTION_INFO_V1 により登録）。
  • **呼び出し先**: infix 関数（再帰的に呼び出される）。

関連データ構造

  • ltxtquery: 検索クエリを保持する構造体。
  • INFIX: 文字列変換中の状態（バッファ、現在位置、演算子・オペランドへの参照）を保持する構造体。
  • ITEM: ltxtquery 内部の演算子・オペランド要素。

失敗モードと境界条件

  • query->size == 0: 空のクエリとして ERRCODE_SYNTAX_ERROR を発生させる。
  • メモリ不足: palloc が失敗した場合、PostgreSQL のエラーハンドリングによりアボートされる。

保守担当者向けメモ

  • 本関数は ltxtquery のバイナリ表現をテキストへ復元する役割を持つ。
  • infix 関数による再帰処理が含まれるため、スタック深さの制限に注意すること。
  • 文字列バッファの動的拡張が必要な場合は infix 側の実装を確認すること。
