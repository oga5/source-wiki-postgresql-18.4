pg_signal_thread

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/signal.c:273

目的

Windows環境において、名前付きパイプを介して外部プロセスからのシグナル要求を待機・受信し、PostgreSQLバックエンドへシグナルを伝達するバックグラウンドスレッドとして機能します。

シグネチャまたは形状

static DWORD WINAPI pg_signal_thread(LPVOID param);

入力と出力

  • **入力**: param (LPVOID) - スレッド起動時に渡される引数（本実装では未使用）。
  • **出力**: DWORD - スレッド終了コード（無限ループのため通常は戻らない）。

前提条件と不変条件

  • pgwin32_initial_signal_pipe が初期化済みであるか、または INVALID_HANDLE_VALUE であること。
  • プロセスIDに基づいた一意な名前付きパイプ (\\.\pipe\pgsignal_<PID>) が利用可能であること。

アルゴリズムまたは振る舞い

1. **パイプの準備**: プロセスIDを含む名前付きパイプを作成または再接続します。 2. **接続待機**: ConnectNamedPipe を使用して、シグナル送信側からの接続を待機します。 3. **シグナル受信**: パイプ経由で1バイトのシグナル番号 (sigNum) を読み取ります。 4. **キューイング**: pg_queue_signal(sigNum) を呼び出し、シグナルを内部キューへ格納します。これにより、送信側の kill() 完了後に受信側が確実にシグナルを検知できることを保証します。 5. **応答**: 送信側の CallNamedPipe() が終了できるよう、パイプへ応答を書き込み、FlushFileBuffers で送信完了を待ちます。 6. **切断**: DisconnectNamedPipe を呼び出し、次の接続に備えてパイプを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/port/win32/signal.c 内の pgwin32_signal_initialize において CreateThread により起動されます。
  • **呼び出し先**:
  • pg_queue_signal (シグナルのキューイング)
  • Windows API: CreateNamedPipe, ConnectNamedPipe, ReadFile, WriteFile, FlushFileBuffers, DisconnectNamedPipe, CloseHandle

関連データ構造

  • pg_signal_queue: 受信したシグナルを保持する共有ビットマスク。
  • pgwin32_initial_signal_pipe: 初期化時に使用されるパイプハンドル。

失敗モードと境界条件

  • **パイプ作成失敗**: CreateNamedPipe が失敗した場合、エラーログを出力し、500ms待機後に再試行します。
  • **接続失敗**: ConnectNamedPipe が失敗した場合、ハンドルをクローズして再作成を試みます。
  • **読み取り失敗**: パイプからの読み取りが失敗した場合、クライアント側の問題とみなし、接続を切断して次の待機へ移行します。

保守担当者向けメモ

  • 本関数は無限ループで動作します。
  • pg_queue_signal への呼び出しは、Windows特有のタイミング問題を回避するための強力な同期保証として機能しています。
  • パイプの再作成ロジックは、シグナル受信の欠落を防ぐためのフォールバックとして実装されています。
