BufferManagerShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/buf_init.c:161

目的

PostgreSQLの共有メモリ領域において、バッファ管理サブシステムが必要とする総メモリサイズを計算する。

シグネチャまたは形状

Size BufferManagerShmemSize(void)

入力と出力

  • **入力**: なし（グローバル変数 NBuffers および BLCKSZ を参照）
  • **出力**: Size 型（計算された共有メモリの合計バイト数）

前提条件と不変条件

  • NBuffers が正しく設定されていること。
  • add_size および mul_size 関数がオーバーフローチェックを行い、安全な加算・乗算を提供すること。

アルゴリズムまたは振る舞い

以下の各コンポーネントのサイズを合計し、アライメントのためのパディングを加算して総サイズを算出する。 1. **バッファ記述子**: NBuffers 分の BufferDescPadded サイズと、キャッシュラインアライメント用の PG_CACHE_LINE_SIZE。 2. **データページ**: PG_IO_ALIGN_SIZE と NBuffers 分の BLCKSZ。 3. **戦略管理**: StrategyShmemSize() が返すサイズ（freelist.c 関連）。 4. **I/O同期**: NBuffers 分の ConditionVariableMinimallyPadded サイズと、アライメント用の PG_CACHE_LINE_SIZE。 5. **チェックポイント**: NBuffers 分の CkptSortItem サイズ。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c（共有メモリ初期化プロセスの一部として呼び出される）
  • **呼び出し先**: StrategyShmemSize()（freelist.c 内の関数）

関連データ構造

  • BufferDescPadded: バッファ記述子。
  • ConditionVariableMinimallyPadded: I/O待機用変数。
  • CkptSortItem: チェックポイント時のソート用構造体。

失敗モードと境界条件

  • NBuffers が極端に大きい場合、add_size / mul_size 内でオーバーフローが発生する可能性がある（PostgreSQLのメモリ計算ユーティリティにより保護される）。

保守担当者向けメモ

  • 本関数で計算されるサイズは、BufferManagerShmemInit で実際に確保されるメモリレイアウトと完全に一致している必要がある。構成要素を追加・変更した場合は、必ず両方の関数を更新すること。
