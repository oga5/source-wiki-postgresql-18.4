HeapTupleHeaderGetOidOld

定義

  • 種別: static inline function
  • 定義箇所: contrib/pageinspect/heapfuncs.c:45

目的

古いバージョンの PostgreSQL からアップグレードされたデータにおいて、ヒープタプルヘッダ内に格納されている OID を抽出する。

シグネチャまたは形状

static inline Oid HeapTupleHeaderGetOidOld(const HeapTupleHeaderData *tup)

入力と出力

  • **入力**: tup (ヒープタプルヘッダへのポインタ)
  • **出力**: タプルが HEAP_HASOID_OLD フラグを持つ場合は格納されている Oid を返し、それ以外の場合は InvalidOid を返す。

前提条件と不変条件

  • 入力される tup は有効な HeapTupleHeaderData 構造体を指している必要がある。
  • t_hoff フィールドは、タプルヘッダの直後にデータが開始されるオフセットを正しく保持している必要がある。

アルゴリズムまたは振る舞い

1. タプルの t_infomask を確認し、HEAP_HASOID_OLD ビットが立っているか判定する。 2. フラグが立っている場合、タプルヘッダの開始位置から t_hoff 分進んだ位置の直前（t_hoff - sizeof(Oid)）を OID の格納位置とみなし、その値を読み取って返す。 3. フラグが立っていない場合は InvalidOid を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/pageinspect/heapfuncs.c 内の heap_page_items 関数（259行目付近）から呼び出されている。
  • **呼び出し先**: 直接的な関数呼び出しはない（ポインタ演算とビットマスク操作のみ）。

関連データ構造

  • HeapTupleHeaderData: ヒープタプルのヘッダ構造。
  • Oid: オブジェクト識別子。
  • HEAP_HASOID_OLD: access/htup_details.h で定義される、古い OID 保持を示すフラグ。

失敗モードと境界条件

  • t_hoff が不正な値（小さすぎるなど）である場合、メモリ上の誤った位置を読み取る可能性がある。
  • ページが破損している場合、返される OID は意味をなさない値となる可能性がある。

保守担当者向けメモ

  • 本関数は pageinspect モジュールにおいて、破損したページや古い形式のデータを解析するために使用される。
  • 読み取り専用の関数であり、データ構造の変更は行わない。
