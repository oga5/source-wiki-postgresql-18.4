heap_prune_record_dead

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/pruneheap.c:1245

目的

ヒープページ内の剪定（pruning）処理において、削除対象として特定されたタプルを PruneState に記録し、統計情報を更新する。

シグネチャまたは形状

static void heap_prune_record_dead(PruneState *prstate, OffsetNumber offnum, bool was_normal)

入力と出力

  • prstate: 剪定処理の状態を保持する PruneState 構造体へのポインタ。
  • offnum: 削除対象となるタプルのオフセット番号。
  • was_normal: 削除対象が通常のタプルであるかを示すフラグ。
  • 出力: なし（prstate の内部状態を更新）。

前提条件と不変条件

  • 対象の offnum が既に処理済みでないこと（!prstate->processed[offnum]）。
  • prstate->ndead が MaxHeapTuplesPerPage を超えないこと。

アルゴリズムまたは振る舞い

1. prstate->processed[offnum] を true に設定し、二重処理を防止する。 2. offnum を prstate->nowdead 配列に追加し、prstate->ndead をインクリメントする。 3. offnum を prstate->deadoffsets 配列に追加し、prstate->lpdead_items をインクリメントする（VACUUM用）。 4. was_normal が true の場合、prstate->ndeleted をインクリメントする。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/heap/pruneheap.c 内の剪定ロジック（heap_prune_chain 等）。
  • 呼び出し先: なし。

関連データ構造

  • PruneState: 剪定処理中の状態（削除対象オフセットや統計情報）を保持する構造体。

失敗モードと境界条件

  • ndead が MaxHeapTuplesPerPage を超過する場合、アサーションにより停止する。

保守担当者向けメモ

  • 本関数は all_visible フラグの解除を即座に行わない。これは、削除可能なデッドタプルの存在がページの凍結を妨げるべきではないという設計意図による。
  • was_normal が false の場合（リダイレクト先など）、削除カウントには含まれない点に注意すること。
