getvacant

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:981

目的

DFA（決定性有限オートマトン）の状態キャッシュにおいて、再利用可能な「空き」状態セット（sset）を確保および初期化します。

シグネチャまたは形状

static struct sset *getvacant(struct vars *v, struct dfa *d, chr *cp, chr *start)

入力と出力

  • **入力**:
  • v: 正規表現の実行コンテキスト（vars 構造体）。
  • d: 対象の DFA 構造体。
  • cp: 現在の文字ポインタ。
  • start: マッチング開始位置のポインタ。
  • **出力**: 再利用のために解放・初期化された struct sset *。確保に失敗した場合は NULL を返します。

前提条件と不変条件

  • 取得対象の sset は LOCKED フラグが立っていないこと（assert により検証）。
  • 呼び出し元は pickss を介して候補となる sset を選定済みであること。

アルゴリズムまたは振る舞い

1. pickss を呼び出し、置換対象となる sset を特定します。 2. 対象 sset に向かう既存の入力アーク（ins）をすべて削除し、参照を解除します。 3. 対象 sset から出力されるアークを辿り、接続先の sset の入力チェーンから当該 sset を取り除きます。 4. POSTSTATE（成功状態）または NOPROGRESS（進捗なし状態）フラグを持つ場合、その位置情報を DFA 構造体の lastpost または lastnopr に退避させます。 5. 最終的に、クリーンアップされた sset ポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/regex/rege_dfa.c 内の miss 関数（キャッシュミス時の状態生成処理）から呼び出されます。
  • **呼び出し先**:
  • pickss: 置換対象の sset を選択します。

関連データ構造

  • struct sset: DFA の状態セット（キャッシュエントリ）。
  • struct dfa: DFA の実行状態を保持する主要構造体。
  • struct arcp: アーク（遷移）のポインタ構造体。

失敗モードと境界条件

  • pickss が NULL を返した場合、本関数も NULL を返します。
  • 削除対象の sset が LOCKED 状態である場合、アサーションエラーが発生します。
  • 接続先 sset の入力チェーンを辿る際、整合性が取れない場合はアサーションエラーが発生します。

保守担当者向けメモ

  • 本関数はキャッシュの入れ替え処理の根幹であり、アークのリンク解除が不完全だとメモリリークや不正なポインタ参照を招くため、修正時は inchain の更新ロジックに注意してください。
