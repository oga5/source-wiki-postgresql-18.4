ExecSetOp

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSetOp.c:159

目的

SQLの集合演算（INTERSECT および EXCEPT）を実行し、入力リレーションから演算ルールに基づいた結果タプルを生成する。

シグネチャまたは形状

TupleTableSlot *ExecSetOp(PlanState *pstate)

入力と出力

  • **入力**: PlanState *pstate（SetOpState 構造体としてキャストされる実行ノードの状態）。
  • **出力**: 演算結果となる TupleTableSlot、または処理すべきタプルがない場合は NULL。

前提条件と不変条件

  • pstate は SetOpState 型としてキャスト可能であること。
  • 割り込みチェック（CHECK_FOR_INTERRUPTS()）が実行の都度行われる。
  • node->numOutput > 0 の場合、以前に返したタプルを再送出する。

アルゴリズムまたは振る舞い

1. **再送出処理**: numOutput > 0 の場合、現在の resultTupleSlot を返し、カウンタを減算する。 2. **終了判定**: node->setop_done が真であれば NULL を返す。 3. **戦略の選択**: plannode->strategy に基づき分岐する。

  • SETOP_HASHED: table_filled が偽であれば setop_fill_hash_table を呼び出し、その後 setop_retrieve_hash_table で結果を取得する。
  • SETOP_HASHED 以外: setop_retrieve_sorted を呼び出し、ソート済み入力から結果を取得する。

呼び出し元と呼び出し先

  • **呼び出し元**: 実行エンジン（ExecProcNode 経由での呼び出しが想定される）。
  • **呼び出し先**:
  • setop_fill_hash_table
  • setop_retrieve_hash_table
  • setop_retrieve_sorted
  • 注: 呼び出し元・呼び出し先の詳細なコールグラフは、提供されたソース excerpt の範囲外の情報を含みます。

関連データ構造

  • SetOpState: 実行ノードの状態を保持する構造体。
  • SetOp: プランノード。strategy（SETOP_HASHED か否か）を保持する。
  • TupleTableSlot: タプルを保持するスロット。

失敗モードと境界条件

  • 入力が空の場合、setop_done に基づき適切に NULL を返す。
  • SETOP_HASHED 戦略において、ハッシュテーブル構築が完了していない場合は初回に構築処理が実行される。

保守担当者向けメモ

  • 本関数は射影（Projection）やクオリファイア（Qual）のチェックを行わないため、それらの処理は上位ノードに依存する。
  • UNION 演算はこのノードでは処理されない。
  • メモリ管理は SetOpState 内のコンテキストに依存しており、再スキャン時には適切にリセットされる必要がある。
