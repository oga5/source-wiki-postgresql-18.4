hashint8extended

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hashfunc.c:103

目的

64ビット整数（int64）に対して、シード値を用いた拡張ハッシュ計算を行い、ハッシュ結合等のパフォーマンスを向上させるためのハッシュ値を生成します。

シグネチャまたは形状

Datum hashint8extended(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して以下の2つの引数を受け取ります。
  • 第1引数: ハッシュ対象の int64 値。
  • 第2引数: ハッシュ計算に使用する int64 型のシード値。
  • **出力**: 計算されたハッシュ値を Datum 型として返します。

前提条件と不変条件

  • 入力値が等価であれば、同一のシード値に対して常に同一のハッシュ値を返します。
  • 内部的に int64 を上位32ビットと下位32ビットに分割して処理します。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT64(0) で対象値を取得します。 2. 値を上位32ビット（hihalf）と下位32ビット（lohalf）に分割します。 3. 値が負か否かに応じてビット反転を行い、lohalf と hihalf を XOR 演算で合成します。 4. 合成された値と、PG_GETARG_INT64(1) から取得したシード値を hash_uint32_extended に渡し、最終的なハッシュ値を算出します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/date.c
  • src/backend/utils/adt/pg_lsn.c
  • src/backend/utils/adt/timestamp.c
  • src/backend/utils/adt/xid.c
  • **呼び出し先**:
  • hash_uint32_extended（src/include/common/hashfn.h 経由）

関連データ構造

  • Datum: PostgreSQLの汎用データ型。
  • PG_FUNCTION_ARGS: 関数呼び出し時の引数コンテキスト。

失敗モードと境界条件

  • 負の整数値に対しては、ビット反転処理を行うことで正の整数値と一貫性のあるハッシュ化を試みます。
  • 64ビット整数の全範囲を扱いますが、ハッシュ値の衝突については hash_uint32_extended のアルゴリズムに依存します。

保守担当者向けメモ

  • hashint8 と同様のロジックを共有しています。修正時は両関数の整合性に注意してください。
  • 拡張ハッシュ関数であるため、新しいデータ型を追加する際は、通常のハッシュ関数と対になる形で本関数のような ...extended バージョンを実装することが推奨されます。
