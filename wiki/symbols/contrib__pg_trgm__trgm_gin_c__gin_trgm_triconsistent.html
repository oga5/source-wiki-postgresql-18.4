gin_trgm_triconsistent

定義

  • 種別: function
  • 定義箇所: contrib/pg_trgm/trgm_gin.c:268

目的

GIN インデックスにおける部分一致検索（GIN_MAYBE）をサポートするための 3 値論理判定関数です。インデックスから得られたトリグラムの候補が、検索条件を満たす可能性があるかを判定します。

シグネチャまたは形状

Datum gin_trgm_triconsistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GinTernaryValue *check（各キーの整合性状態）
  • PG_GETARG_UINT16(1): StrategyNumber strategy（検索戦略）
  • PG_GETARG_INT32(3): int32 nkeys（キーの数）
  • PG_GETARG_POINTER(4): Pointer *extra_data（追加の検索情報）
  • **出力**:
  • PG_RETURN_GIN_TERNARY_VALUE(res): GIN_TRUE, GIN_FALSE, GIN_MAYBE のいずれか。

前提条件と不変条件

  • 本関数が返す結果は常に「不正確（inexact）」であるため、Assert(res != GIN_TRUE) が成立します。
  • Similarity 関連の戦略では、トリグラムの出現数に基づき閾値判定を行います。
  • 正規表現検索では、extra_data に格納された TrgmPackedGraph を使用して判定を行います。

アルゴリズムまたは振る舞い

1. **戦略による分岐**:

  • **類似度検索 (Similarity, WordSimilarity 等)**: index_strategy_get_limit で取得した閾値と、check 配列内の GIN_FALSE でないキーの割合を比較し、GIN_MAYBE または GIN_FALSE を返します。
  • **等価・LIKE 検索**: すべてのキーが GIN_FALSE でなければ GIN_MAYBE を返し、一つでも GIN_FALSE があれば GIN_FALSE を返します。
  • **正規表現検索**: trigramsMatchGraph を呼び出し、グラフ構造に基づいて整合性を判定します。GIN_MAYBE を GIN_TRUE とみなして保守的に判定します。

    • 2. **戻り値**: 常に GIN_MAYBE または GIN_FALSE を返し、呼び出し元に対して再チェックを促します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の GIN インデックスエンジン（直接の呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**: index_strategy_get_limit, trigramsMatchGraph, palloc, pfree, elog。

関連データ構造

  • GinTernaryValue: GIN インデックスの 3 値論理状態。
  • StrategyNumber: 検索戦略を識別する数値。
  • TrgmPackedGraph: 正規表現検索時に使用されるグラフ構造。

失敗モードと境界条件

  • 未知の StrategyNumber が渡された場合、elog(ERROR) を発生させます。
  • 大文字小文字を区別しない検索（ILike, RegExpICase）において、IGNORECASE が定義されていない環境で実行された場合、エラーとなります。
  • nkeys が 0 の場合、類似度検索では GIN_FALSE を返します。

保守担当者向けメモ

  • 本関数は常に recheck を必要とする結果（GIN_MAYBE）を返す設計です。
  • 正規表現検索のロジックは trgm_regexp.c で定義された trigramsMatchGraph に依存しています。
