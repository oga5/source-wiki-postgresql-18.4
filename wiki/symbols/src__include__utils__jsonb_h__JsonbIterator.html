JsonbIterator

定義

  • 種別: struct
  • 定義箇所: src/include/utils/jsonb.h:341

目的

jsonb 構造体（バイナリ形式のJSONデータ）を再帰的に走査するためのメモリ内イテレータ構造体です。

シグネチャまたは形状

typedef struct JsonbIterator
{
    JsonbContainer *container;
    uint32         nElems;
    bool           isScalar;
    JEntry        *children;
    char          *dataProper;
    int            curIndex;
    uint32         curDataOffset;
    uint32         curValueOffset;
    JsonbIterState state;
    struct JsonbIterator *parent;
} JsonbIterator;

入力と出力

  • **入力**: JsonbIteratorInit を介して初期化され、対象となる JsonbContainer を保持します。
  • **出力**: JsonbIteratorNext を呼び出すことで、JSON要素を順次探索し、各ノードの状態を返します。

前提条件と不変条件

  • Jsonb 構造体は4バイト境界でアライメントされています。
  • nElems は子要素の数（オブジェクトの場合はペア数）を示します。
  • curIndex は 0 から nElems - 1 の範囲で推移します。

アルゴリズムまたは振る舞い

  • JsonbIteratorInit でイテレータを初期化し、JsonbIteratorNext を繰り返し呼び出すことでツリー構造を探索します。
  • オブジェクトを走査する場合、curDataOffset が現在のキーを、curValueOffset が対応する値を指すように交互に返します。
  • parent ポインタにより、再帰的な構造の走査をサポートします。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/jsonb_plperl, contrib/jsonb_plpython, src/backend/utils/adt/jsonb.c, src/backend/utils/adt/jsonb_gin.c, src/backend/utils/adt/jsonb_op.c 等の各実装ファイルで利用されています。
  • **呼び出し先**: 具体的な関数呼び出し関係は本ソース抜粋からは特定できません。

関連データ構造

  • JsonbContainer: 走査対象のコンテナ。
  • JEntry: 各要素の型と位置を管理するヘッダー。
  • JsonbIterState: 走査の内部状態を保持する列挙型。

失敗モードと境界条件

  • JEntry のフラグ（JENTRY_HAS_OFF）に基づいたオフセット計算が必要であり、不適切な計算はメモリ破壊を招く可能性があります。
  • 可変長データ領域へのアクセス時には、パディングを考慮したアライメントの管理が不可欠です。

保守担当者向けメモ

  • 本構造体は src/backend/utils/adt/jsonb.c の走査ロジックと密接に結合しています。
  • Jsonb のオンディスク形式は圧縮効率を優先しているため、イテレータの走査ロジックを変更する際は JEntry のオフセット計算ロジックに十分注意してください。
