create_seqscan_path

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/pathnode.c:985

目的

リレーションに対するシーケンシャルスキャン（SeqScan）の実行計画パス（Path ノード）を生成し、そのコストを算出する。

シグネチャまたは形状

Path *create_seqscan_path(PlannerInfo *root, RelOptInfo *rel, Relids required_outer, int parallel_workers)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: スキャン対象のリレーション情報。
  • required_outer: このパスが依存する外部リレーションの集合（パラメータ化パス用）。
  • parallel_workers: 使用する並列ワーカー数。
  • **出力**:
  • 初期化およびコスト計算済みの Path 構造体へのポインタ。

前提条件と不変条件

  • pathnode->pathtype は T_SeqScan に設定される。
  • pathnode->pathkeys は常に NIL となる（シーケンシャルスキャンは順序を保証しないため）。
  • parallel_workers > 0 の場合、parallel_aware が true に設定される。

アルゴリズムまたは振る舞い

1. makeNode(Path) により新しいパスノードを割り当てる。 2. リレーション情報（rel）からターゲットリストやパラメータ情報を取得し、パスノードに設定する。 3. 並列実行設定（parallel_workers）に基づき、並列実行の可否およびワーカー数を設定する。 4. cost_seqscan を呼び出し、当該パスの起動コストおよび総コストを計算してノードに格納する。 5. 完成した Path ノードを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/allpaths.c
  • src/backend/optimizer/plan/planner.c
  • src/backend/optimizer/util/pathnode.c（内部的な再帰呼び出し等）
  • **呼び出し先**:
  • makeNode（メモリ確保）
  • get_baserel_parampathinfo（パラメータ化情報の取得）
  • cost_seqscan（コスト計算）

関連データ構造

  • Path: 生成される実行計画の基本構造体。
  • PlannerInfo: プランナの全体状態。
  • RelOptInfo: スキャン対象となるリレーションのメタデータ。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性がある。
  • parallel_workers が 0 の場合、非並列のシーケンシャルスキャンとして構成される。

保守担当者向けメモ

  • 本関数はシーケンシャルスキャンの標準的なファクトリである。コストモデルの変更が必要な場合は cost_seqscan を参照すること。
  • pathkeys が常に NIL である点は、このパスがソート順序を保持しないという不変条件に基づいている。
