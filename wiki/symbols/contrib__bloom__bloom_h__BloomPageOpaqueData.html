BloomPageOpaqueData

定義

  • 種別: struct
  • 定義箇所: contrib/bloom/bloom.h:33

目的

PostgreSQL の Bloom インデックスにおける各ページの末尾に配置されるメタデータ構造体です。ページの種類（メタページかデータページか）の識別および、ページ内に格納されているインデックスタプル数の管理に使用されます。

シグネチャまたは形状

typedef struct BloomPageOpaqueData
{
	OffsetNumber maxoff;    /* ページ内のインデックスタプル数 */
	uint16       flags;     /* ビット定義によるフラグ */
	uint16       unused;    /* アライメント調整用パディング */
	uint16       bloom_page_id; /* BLOOM インデックス識別用ID */
} BloomPageOpaqueData;

入力と出力

本構造体はデータ構造の定義であり、関数ではないため入力と出力はありません。ただし、PageInit 関数（blutils.c）において、ページ初期化時の引数として本構造体のサイズが渡されます。

前提条件と不変条件

  • ページ内の空き領域を計算する際は、MAXALIGN を考慮したマクロ（BloomPageGetFreeSpace）を使用する必要があります。
  • bloom_page_id はページ末尾に配置されることが保証されています。

アルゴリズムまたは振る舞い

  • インデックスページが初期化される際、PageInit を通じてページ末尾に本構造体が割り当てられます。
  • maxoff フィールドは、ページ内のタプル走査時に現在のタプル数を追跡するために参照されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/bloom/blutils.c の PageInit が本構造体のサイズを使用してページを初期化します。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • BloomPageOpaque: BloomPageOpaqueData へのポインタ型。
  • BloomMetaPageData: インデックスのルート（メタページ）構造。
  • BloomTuple: インデックスページ内に格納される実際のデータ構造。

失敗モードと境界条件

  • ページ末尾に配置されるため、ページサイズ（BLCKSZ）を超過したデータ配置を行うとメモリ破壊を引き起こします。
  • unused フィールドはアライメント調整用であり、直接的な操作は行われません。

保守担当者向けメモ

  • BloomTuple は可変長であるため、ページ内の空き領域計算には必ず MAXALIGN を考慮したマクロを使用してください。
  • bloom_page_id は pg_filedump 等の外部ツールがインデックス種別を判別するために使用するため、変更時には注意が必要です。
