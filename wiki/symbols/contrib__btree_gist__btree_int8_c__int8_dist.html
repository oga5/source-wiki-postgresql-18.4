int8_dist

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_int8.c:95

目的

2つの int8（64ビット整数）値間の絶対距離を計算し、PostgreSQLのGiSTインデックスにおける距離計算関数として提供する。

シグネチャまたは形状

Datum int8_dist(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_INT64(0) および PG_GETARG_INT64(1) から取得される2つの int64 値。
  • **出力**: 2値間の差の絶対値（int64）を PG_RETURN_INT64 で返す。

前提条件と不変条件

  • 入力値の減算においてオーバーフローが発生しないこと。
  • 差分が PG_INT64_MIN にならないこと（絶対値変換時に表現不能となるため）。

アルゴリズムまたは振る舞い

1. 第1引数から第2引数を減算する。 2. pg_sub_s64_overflow を使用して減算時のオーバーフローをチェックする。 3. 減算結果が PG_INT64_MIN であるか、オーバーフローが発生した場合はエラーを報告する。 4. 正常な場合、i64abs を用いて差分の絶対値を算出し、結果を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上の直接的な呼び出し元は明示されていませんが、GiSTインデックスの距離計算処理（gbt_int8_distance 等）から利用される設計です。
  • **呼び出し先**: pg_sub_s64_overflow（オーバーフロー判定）、i64abs（絶対値計算）、ereport（エラー報告）。

関連データ構造

  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に従う引数コンテナ。

失敗モードと境界条件

  • **数値範囲外**: 減算結果が int64 の範囲を超える場合、または PG_INT64_MIN となる場合は ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE エラーが発生する。

保守担当者向けメモ

  • 本関数は int8 型専用の距離計算ユーティリティです。
  • PG_INT64_MIN を除外しているのは、abs(PG_INT64_MIN) が正の int64 として表現できないためです。
