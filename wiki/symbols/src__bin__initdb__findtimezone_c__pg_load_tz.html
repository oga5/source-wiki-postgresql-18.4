pg_load_tz

定義

  • 種別: function
  • 定義箇所: src/bin/initdb/findtimezone.c:90

目的

指定されたタイムゾーン名に対応する pg_tz 構造体を読み込み、初期化して返します。

シグネチャまたは形状

static pg_tz *pg_load_tz(const char *name)

入力と出力

  • **入力**: name (タイムゾーン名を表す文字列)
  • **出力**: 成功時は静的に確保された pg_tz 構造体へのポインタ、失敗時は NULL を返します。

前提条件と不変条件

  • name の長さは TZ_STRLEN_MAX 以下である必要があります。
  • 内部で静的変数 static pg_tz tz を使用しているため、本関数は再入可能ではありません。

アルゴリズムまたは振る舞い

1. **長さ検証**: name が TZ_STRLEN_MAX を超える場合、NULL を返します。 2. **GMTの処理**: name が "GMT" の場合、tzparse を用いて初期化を試みます。 3. **標準読み込み**: それ以外の場合、tzload を用いてタイムゾーンファイルを読み込みます。 4. **フォールバック**: tzload が失敗し、かつ name が : で始まらない場合、tzparse による解析を試みます。 5. **名前の保存**: 成功した場合、tz.TZname に name をコピーし、構造体へのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/initdb/findtimezone.c 内の複数箇所で呼び出されています。
  • **呼び出し先**: tzparse, tzload（いずれも外部のタイムゾーン処理ロジック）。

関連データ構造

  • pg_tz: タイムゾーン情報を保持する構造体。

失敗モードと境界条件

  • name が長すぎる場合、NULL を返します。
  • tzload および tzparse の両方が失敗した場合、NULL を返します。
  • 静的変数を使用しているため、連続して呼び出すと以前の呼び出し結果が上書きされます。

保守担当者向けメモ

  • 本関数は静的領域を再利用するため、呼び出し側で結果を保持する必要がある場合は、ポインタを保持するのではなく内容をコピーしてください。
  • tzload や tzparse の挙動については src/timezone/ 配下の実装を参照してください。
