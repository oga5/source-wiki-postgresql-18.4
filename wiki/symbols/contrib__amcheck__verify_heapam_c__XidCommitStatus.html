XidCommitStatus

定義

  • 種別: enum
  • 定義箇所: contrib/amcheck/verify_heapam.c:54

目的

ヒープ検証プロセスにおいて、特定のトランザクションID（XID）のコミット状態を分類・保持するために使用されます。

シグネチャまたは形状

typedef enum XidCommitStatus
{
    XID_COMMITTED,        /* コミット済み */
    XID_IS_CURRENT_XID,   /* 現在のトランザクションID */
    XID_IN_PROGRESS,      /* 実行中 */
    XID_ABORTED,          /* アボート済み */
} XidCommitStatus;

入力と出力

  • 本シンボルは列挙型であり、関数ではありません。
  • 主に check_tuple_visibility 等の関数内で、トランザクション状態の判定結果として変数に代入されます。

前提条件と不変条件

  • 常に上記4つの状態のいずれかをとります。
  • ヒープ検証中、タプルの可視性判定において、トランザクションの整合性を評価する際の基準値として機能します。

アルゴリズムまたは振る舞い

  • verify_heapam モジュール内でのタプル検証時、各タプルの xmin や xmax がどの状態にあるかを判定し、その結果をこの型で保持します。
  • 判定結果に基づき、破損の有無や可視性の妥当性が評価されます。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/amcheck/verify_heapam.c 内の複数の検証関数（check_tuple_visibility 等）が、トランザクション状態の戻り値としてこの型を使用しています。
  • 呼び出し先: 本シンボル自体はデータ型であるため、関数呼び出しは発生しません。

関連データ構造

  • HeapCheckContext: 検証中の状態を保持する構造体であり、この型をメンバとして保持します。
  • MaxOffsetNumber: ページ内のタプル数に応じた配列サイズを確保する際に併用されます。

失敗モードと境界条件

  • 該当なし（列挙型のため、値の範囲外へのアクセスはコンパイル時または実行時の型安全性に依存します）。

保守担当者向けメモ

  • 本列挙型は verify_heapam.c 内の可視性チェックロジックに深く依存しています。トランザクション状態の判定ロジックに変更を加える場合は、この列挙値の網羅性を確認してください。
