libpqsrv_cancel

定義

  • 種別: function
  • 定義箇所: src/include/libpq/libpq-be-fe-helpers.h:385

目的

実行中のクエリに対してキャンセル要求を安全に発行します。バックエンドの割り込み処理と統合され、タイムアウト付きで非同期にキャンセルを完了させるためのヘルパー関数です。

シグネチャまたは形状

const char *libpqsrv_cancel(PGconn *conn, TimestampTz endtime);

入力と出力

  • **入力**:
  • conn: キャンセル対象の PGconn オブジェクト。
  • endtime: キャンセル処理の期限を示す TimestampTz。
  • **出力**:
  • 成功時は NULL を返します。
  • 失敗時はエラー内容を示す文字列（const char *）を返します。

前提条件と不変条件

  • PG_TRY/PG_FINALLY ブロックにより、エラー発生時や中断時であっても PGcancelConn リソースが確実に解放されることを保証します。
  • 処理中に CHECK_FOR_INTERRUPTS() を呼び出し、バックエンドのシグナルに応答します。

アルゴリズムまたは振る舞い

1. PQcancelCreate でキャンセル用接続オブジェクトを作成します。 2. PQcancelStart でキャンセル処理を開始します。 3. ループ内で PQcancelPoll を呼び出し、状態を監視します。

  • PGRES_POLLING_OK に達すれば成功として終了します。
  • タイムアウト（endtime）を超過した場合はエラーを返します。
  • 読み込み/書き込みが必要な場合は WaitLatchOrSocket を使用して、ラッチまたはソケットの準備ができるまで待機します。

    • 4. 待機解除後、ResetLatch を行い、CHECK_FOR_INTERRUPTS() で割り込みをチェックします。 5. 最後に PQcancelFinish でリソースを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c:1370
  • contrib/postgres_fdw/connection.c:1453
  • contrib/postgres_fdw/connection.c:1654
  • **呼び出し先**:
  • PQcancelCreate, PQcancelStart, PQcancelPoll, PQcancelErrorMessage, PQcancelSocket, PQcancelFinish
  • GetCurrentTimestamp, TimestampDifferenceMilliseconds
  • WaitLatchOrSocket, ResetLatch, CHECK_FOR_INTERRUPTS
  • pchomp (エラーメッセージ整形用)

関連データ構造

  • PGconn: libpq の接続ハンドル。
  • PGcancelConn: キャンセル処理専用の接続ハンドル。

失敗モードと境界条件

  • **メモリ不足**: PQcancelCreate が失敗した場合、"out of memory" を返します。
  • **タイムアウト**: endtime を過ぎた場合、"cancel request timed out" を返します。
  • **通信エラー**: PQcancelStart や PQcancelPoll が失敗した場合、PQcancelErrorMessage から取得したエラー文字列を返します。

保守担当者向けメモ

  • 本関数はエラーメッセージ生成時にメモリを消費するため、一時的なメモリコンテキスト内での呼び出しが推奨されます。
  • libpq の非同期 I/O をバックエンドの WaitLatchOrSocket と統合しているため、修正時は WL_ フラグの整合性に注意してください。
