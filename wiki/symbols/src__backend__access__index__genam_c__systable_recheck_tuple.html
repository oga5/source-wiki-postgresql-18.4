systable_recheck_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/index/genam.c:572

目的

システムカタログスキャンにおいて、直近に取得したタプルの可視性を最新のスナップショットに基づいて再確認する。

シグネチャまたは形状

bool systable_recheck_tuple(SysScanDesc sysscan, HeapTuple tup)

入力と出力

  • **入力**:
  • sysscan: 現在のスキャン状態を保持する SysScanDesc 構造体。
  • tup: 再確認対象の HeapTuple。
  • **出力**:
  • bool: タプルが現在のスナップショットで可視であれば true、そうでなければ false を返す。

前提条件と不変条件

  • tup は sysscan->slot に格納されているヒープタプルと一致している必要がある（Assert により検証）。

アルゴリズムまたは振る舞い

1. GetCatalogSnapshot を呼び出し、対象リレーションの最新のカタログスナップショットを取得する。 2. RegisterSnapshot を用いてスナップショットを登録し、生存期間を確保する。 3. table_tuple_satisfies_snapshot を呼び出し、現在のスナップショットに対してタプルが可視かどうかを判定する。 4. UnregisterSnapshot でスナップショットを解放する。 5. HandleConcurrentAbort を呼び出し、論理ストリーミング中の並行アボートを処理する。 6. 判定結果を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/dependency.c
  • src/backend/catalog/pg_shdepend.c
  • **呼び出し先**:
  • GetCatalogSnapshot
  • RegisterSnapshot / UnregisterSnapshot
  • table_tuple_satisfies_snapshot
  • HandleConcurrentAbort
  • RelationGetRelid
  • ExecFetchSlotHeapTuple

関連データ構造

  • SysScanDesc: システムカタログスキャンの状態を管理する構造体。

失敗モードと境界条件

  • sysscan が無効な場合や、tup が sysscan->slot と不一致である場合は Assert により停止する可能性がある。

保守担当者向けメモ

  • 本関数は MVCC の可視性ルールを再評価するものであり、カタログの整合性を維持するために重要である。
  • HandleConcurrentAbort の呼び出しは、論理レプリケーション等のストリーミング処理における一貫性確保のために必須である。
