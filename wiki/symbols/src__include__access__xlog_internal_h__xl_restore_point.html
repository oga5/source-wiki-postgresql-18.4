xl_restore_point

定義

  • 種別: struct
  • 定義箇所: src/include/access/xlog_internal.h:286

目的

リカバリ処理において、特定の時点（リストアポイント）を識別するための情報を保持するWALレコード構造体です。

シグネチャまたは形状

typedef struct xl_restore_point
{
    TimestampTz rp_time;
    char        rp_name[MAXFNAMELEN];
} xl_restore_point;

入力と出力

  • **入力**: リストアポイント作成時に rp_time（タイムスタンプ）と rp_name（名称）が設定されます。
  • **出力**: リカバリ実行中、WALレコードをデコードする際に、この構造体を通じて rp_time が抽出され、リカバリのターゲット時刻として利用されます。

前提条件と不変条件

  • rp_name は MAXFNAMELEN バイト以内の文字列である必要があります。
  • WALレコードとして書き込まれる際、XLogRegisterData を介して構造体全体が登録されます。

アルゴリズムまたは振る舞い

1. **書き込み**: src/backend/access/transam/xlog.c にて、xl_restore_point 構造体のインスタンスが作成され、XLogRegisterData を通じてWALに記録されます。 2. **読み込み**: リカバリ処理（xlogrecovery.c）において、WALレコードから XLogRecGetData を用いて本構造体へキャストし、rp_time を取得してリカバリの停止条件判定に使用します。

呼び出し元と呼び出し先

  • **呼び出し元（書き込み側）**: src/backend/access/transam/xlog.c がこの構造体を使用してWALレコードを生成します。
  • **呼び出し先（読み込み側）**: src/backend/access/transam/xlogrecovery.c がリカバリ中のデータ抽出に使用します。
  • **デコード**: src/backend/access/rmgrdesc/xlogdesc.c がこの構造体をキャストしてレコード内容を解釈します。

関連データ構造

  • TimestampTz: PostgreSQLのタイムスタンプ型。
  • MAXFNAMELEN: ファイル名等の最大長を定義する定数。

失敗モードと境界条件

  • rp_name が MAXFNAMELEN を超える場合、バッファオーバーフローのリスクがあるため、呼び出し側での適切な切り詰めが必要です。
  • WALレコードが破損している場合、XLogRecGetData によるキャスト後の値は不定となります。

保守担当者向けメモ

  • 本構造体はWALの物理フォーマットの一部であるため、フィールドの順序やサイズを変更すると、過去のWALファイルとの互換性が失われます。変更時は注意してください。
