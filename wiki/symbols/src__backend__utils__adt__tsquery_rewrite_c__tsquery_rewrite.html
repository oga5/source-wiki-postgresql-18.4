tsquery_rewrite

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_rewrite.c:409

目的

tsquery 型のクエリに対し、指定されたパターン（ex）を別のクエリ（subst）に置換する。

シグネチャまたは形状

Datum tsquery_rewrite(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TSQUERY_COPY(0): 対象の TSQuery
  • PG_GETARG_TSQUERY(1): 置換対象のパターン TSQuery
  • PG_GETARG_TSQUERY(2): 置換後の TSQuery
  • **出力**: 置換後の TSQuery を Datum として返す。

前提条件と不変条件

  • 入力された query または ex のサイズが 0 の場合、置換を行わず元のクエリを返す。
  • 内部処理において QT2QTN で木構造へ変換後、QTNTernary および QTNSort による正規化が必須である。
  • subst が空の場合、一致したノードは削除される。

アルゴリズムまたは振る舞い

1. 入力された query、ex、subst をそれぞれ QTNode 形式の木構造に変換する。 2. QTNTernary および QTNSort を呼び出し、木構造を正規化する。 3. findsubquery を呼び出し、木全体を走査してパターン一致と置換を実行する。 4. 置換後の木構造を QTNBinary でバイナリ形式に戻し、QTN2QT で TSQuery 型へ再変換する。 5. 処理終了後、一時的に生成した QTNode およびコピーされた引数をメモリから解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は特定されていない。
  • **呼び出し先**:
  • QT2QTN, QTNTernary, QTNSort, QTNFree, QTNBinary, QTN2QT (ユーティリティ関数)
  • findsubquery (置換ロジック)
  • PG_GETARG_TSQUERY, PG_FREE_IF_COPY, PG_RETURN_POINTER (PostgreSQL 関数インターフェース)

関連データ構造

  • TSQuery: 全文検索クエリのデータ型。
  • QTNode: tsquery を操作するための木構造表現。

失敗モードと境界条件

  • query または ex が空（size == 0）の場合、置換は行われない。
  • findsubquery が NULL を返した場合（置換によりクエリ全体が消滅した場合）、サイズ 0 の空の TSQuery を返す。
  • 再帰的な木構造操作を行うため、非常に深いクエリ構造ではスタックオーバーフローのリスクがある。

保守担当者向けメモ

  • 本関数は tsquery の正規化（QTNSort）に強く依存しているため、正規化ロジックに変更を加える際は注意が必要である。
  • メモリリークを防ぐため、PG_FREE_IF_COPY による引数の解放と QTNFree による中間ノードの解放が適切に行われていることを確認すること。
