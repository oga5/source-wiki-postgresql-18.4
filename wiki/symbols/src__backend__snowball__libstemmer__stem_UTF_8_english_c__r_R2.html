r_R2

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_english.c:483

目的

Porterステミングアルゴリズムにおいて、現在のカーソル位置が単語内の「R2領域」内にあるかどうかを判定する。

シグネチャまたは形状

static int r_R2(struct SN_env * z)

入力と出力

  • **入力**: z (Snowball環境構造体へのポインタ)
  • **出力**: 整数値。現在のカーソル位置 z->c が、事前に計算されたR2領域の開始位置 z->I[0] 以上であれば真（1）、そうでなければ偽（0）を返す。

前提条件と不変条件

  • r_mark_regions 関数が事前に呼び出され、z->I[0] にR2領域の開始インデックスが格納されていること。
  • z->c は現在の処理対象文字のインデックスを示す。

アルゴリズムまたは振る舞い

1. 環境構造体 z 内の整数配列 I のインデックス0（R2領域の開始位置）と、現在のカーソル位置 c を比較する。 2. z->I[0] <= z->c の評価結果を戻り値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: 本関数は、Porterアルゴリズムの各ステップ（r_Step_1a ～ r_Step_5 等）から、接尾辞削除の条件判定として呼び出される。
  • **呼び出し先**: なし（算術比較のみを実行）。

関連データ構造

  • struct SN_env: Snowballランタイムの環境を保持する構造体。I[0]（領域境界）および c（カーソル位置）を含む。

失敗モードと境界条件

  • r_mark_regions が未実行の場合、z->I[0] が初期値（通常は0または無効値）のままであるため、判定結果が正しく機能しない可能性がある。

保守担当者向けメモ

  • 本関数は自動生成コードの一部である。ロジックの変更が必要な場合は、直接編集せず、Snowballのソース定義（.sblファイル）を修正すること。
  • z->I[0] の具体的な意味（R1かR2か）は、r_mark_regions の実装に依存する。
