XidSkip

定義

  • 種別: static inline function
  • 定義箇所: src/test/modules/xid_wraparound/xid_wraparound.c:170

目的

SLRU（Simple LRU）のページ境界をまたぐことなく、安全にスキップ可能なトランザクションID（XID）の最大距離を算出します。

シグネチャまたは形状

static inline uint32 XidSkip(FullTransactionId fullxid)

入力と出力

  • **入力**: FullTransactionId fullxid（現在のトランザクションID）
  • **出力**: uint32（次のページ境界に到達するまでの安全なスキップ量。境界上にある場合は0）

前提条件と不変条件

  • 入力された fullxid が有効な範囲内（5以上かつ UINT32_MAX - 5 未満）であること。
  • COMMIT_TS_XACTS_PER_PAGE、SUBTRANS_XACTS_PER_PAGE、CLOG_XACTS_PER_PAGE の各定数が、PostgreSQLの現在の内部実装と一致していること。

アルゴリズムまたは振る舞い

1. 現在のXIDから UINT32_MAX - 5 までの距離を初期の最大スキップ量として設定します。 2. COMMIT_TS、SUBTRANS、CLOG の各SLRUページサイズに基づき、現在のXIDから次のページ境界までの距離を計算します。 3. 各SLRUの境界までの距離のうち、最も小さい値を Min 関数で選択し、それをスキップ量として返します。 4. いずれかのSLRUページ境界に既に位置している場合（剰余が0の場合）、スキップ不可として0を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/test/modules/xid_wraparound/xid_wraparound.c 内の consume_xids_shortcut 関数（212行目付近）から呼び出されます。
  • **呼び出し先**: XidFromFullTransactionId、Min、および各SLRUページサイズを定義するマクロ。

関連データ構造

  • FullTransactionId: 64bitのトランザクションID表現。
  • SLRU関連定数: COMMIT_TS_XACTS_PER_PAGE, SUBTRANS_XACTS_PER_PAGE, CLOG_XACTS_PER_PAGE。

失敗モードと境界条件

  • low < 5 または low >= UINT32_MAX - 5 の場合、安全のためスキップを行わず 0 を返します。
  • ページ境界に位置している場合、スキップ量は 0 となります。

保守担当者向けメモ

  • 本関数はPostgreSQLの内部構造（SLRUページサイズ等）に強く依存しています。バージョンアップにより内部実装が変更された場合、スキップロジックが正しく機能しなくなる可能性があるため注意してください。
