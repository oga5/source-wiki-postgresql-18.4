getCopyStart

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-protocol3.c:1705

目的

PostgreSQL プロトコル 3.0 における CopyInResponse、CopyOutResponse、または CopyBothResponse メッセージを処理し、COPY モード開始時のメタデータ（バイナリフラグおよび属性記述子）を初期化する。

シグネチャまたは形状

static int getCopyStart(PGconn *conn, ExecStatusType copytype)

入力と出力

  • **入力**:
  • conn: 現在の接続状態を保持する PGconn 構造体へのポインタ。
  • copytype: 生成する PGresult のステータスタイプ（PGRES_COPY_IN, PGRES_COPY_OUT, PGRES_COPY_BOTH）。
  • **出力**:
  • 成功時: 0 を返し、conn->result に初期化済みの PGresult を格納する。
  • 失敗時: EOF を返し、接続状態を維持しつつエラー処理を行う。

前提条件と不変条件

  • conn->inBuffer に、COPY 開始メッセージの残りのデータ（バイナリフラグ 1 バイト、フィールド数 2 バイト、各フィールドのフォーマット 2 バイト）が読み込まれている必要がある。
  • pqGetInt は 2 バイト整数を符号なしとして読み込むため、関数内で int16 へのキャストによる符号付き変換が行われる。

アルゴリズムまたは振る舞い

1. PQmakeEmptyPGresult を呼び出し、指定された copytype で PGresult を作成する。 2. pqGetc でバイナリモードフラグを読み取り、result->binary に設定する。 3. pqGetInt でフィールド数（2 バイト）を読み取り、result->numAttributes に格納する。 4. フィールド数が 0 より大きい場合、pqResultAlloc を使用して属性記述子（PGresAttDesc）の配列を確保し、ゼロクリアする。 5. 各フィールドについて pqGetInt でフォーマットコードを読み取り、符号付き整数に変換して result->attDescs[i].format に格納する。 6. 成功時、conn->result に構築した結果を代入して終了する。失敗時は PQclear でメモリを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-protocol3.c 内のメッセージ解析ロジック（CopyInResponse、CopyOutResponse、CopyBothResponse 受信時）。
  • **呼び出し先**:
  • PQmakeEmptyPGresult
  • pqGetc
  • pqGetInt
  • pqResultAlloc
  • MemSet
  • PQclear

関連データ構造

  • PGconn: 接続状態および受信バッファを管理。
  • PGresult: COPY モードのメタデータ（属性記述子、バイナリフラグ）を保持。
  • PGresAttDesc: 各フィールドのフォーマット情報を保持する構造体。

失敗モードと境界条件

  • PQmakeEmptyPGresult がメモリ不足等で失敗した場合、EOF を返す。
  • pqGetc や pqGetInt によるバッファ読み込みが失敗した場合、EOF を返す。
  • pqResultAlloc が失敗した場合、EOF を返す。
  • いずれの失敗時も、確保済みの PGresult は PQclear により破棄される。

保守担当者向けメモ

  • 本関数はプロトコル 3.0 専用である。
  • pqGetInt が 2 バイト整数を符号なしで扱う仕様のため、フォーマットコードの符号変換（int16 へのキャスト）が必須である点に注意すること。
