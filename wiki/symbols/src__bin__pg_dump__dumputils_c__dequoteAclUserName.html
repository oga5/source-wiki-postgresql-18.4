dequoteAclUserName

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/dumputils.c:618

目的

ACL（アクセス制御リスト）文字列から、引用符で囲まれたユーザー名やエスケープされた引用符を適切に処理し、元のユーザー名文字列を抽出してバッファに格納する。

シグネチャまたは形状

static char *dequoteAclUserName(PQExpBuffer output, char *input)

入力と出力

  • **入力**:
  • output: 抽出されたユーザー名を格納するための PQExpBuffer。
  • input: 解析対象の ACL 文字列（ユーザー名部分から開始）。
  • **出力**:
  • 戻り値: 解析が終了した位置の input ポインタを返す。

前提条件と不変条件

  • output バッファは関数内で resetPQExpBuffer により初期化される。
  • ユーザー名が引用符で囲まれている場合、内部の引用符は "" と記述されるエスケープ規則に従う必要がある。

アルゴリズムまたは振る舞い

1. output バッファをリセットする。 2. input 文字列を走査し、=（ACL の区切り）に到達するまで処理を繰り返す。 3. 引用符で囲まれていない文字はそのまま output に追加する。 4. 引用符で囲まれている場合、以下の処理を行う：

  • 閉じ引用符（エスケープされていないもの）に到達するまで文字を読み込む。
  • "" という連続した引用符は、単一の " として output に追加する。

    • 5. 処理終了後、現在の input 位置を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/dumputils.c:438 (grantee の抽出)
  • src/bin/pg_dump/dumputils.c:450 (grantor の抽出)
  • **呼び出し先**:
  • resetPQExpBuffer
  • appendPQExpBufferChar

関連データ構造

  • PQExpBuffer: PostgreSQL の動的文字列バッファ構造体。

失敗モードと境界条件

  • input が \0 に達した場合、構文エラーとしてその時点のポインタを返す。
  • 引用符の閉じ忘れや不正なエスケープシーケンスに対する厳密なエラーハンドリングは行わず、解析を終了する。

保守担当者向けメモ

  • 本関数の引用符エスケープ規則（" を "" と表現する）は、バックエンド側の src/backend/utils/adt/acl.c 内の putid() 関数と同期させる必要がある。変更時は必ず両方の実装を確認すること。
