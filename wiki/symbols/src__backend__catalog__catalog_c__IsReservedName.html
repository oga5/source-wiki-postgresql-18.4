IsReservedName

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/catalog.c:277

目的

指定された名前が PostgreSQL のシステム予約命名規則（pg_ で始まる）に従っているかを判定します。

シグネチャまたは形状

bool IsReservedName(const char *name)

入力と出力

  • **入力**: name (検証対象の文字列へのポインタ)
  • **出力**: bool (名前が pg_ で始まる場合は true、それ以外は false)

前提条件と不変条件

  • 入力 name はヌル終端文字列である必要があります。
  • 内部で name[0] から name[2] までのインデックスに直接アクセスするため、文字列長が 3 未満の場合は未定義の動作を引き起こす可能性があります。

アルゴリズムまたは振る舞い

  • 文字列の先頭 3 文字が順に 'p', 'g', '_' であるかをチェックします。
  • 処理速度を優先するため、文字列比較関数を使用せず、各文字を直接比較する実装となっています。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/schemacmds.c (スキーマ名、テーブル名の検証)
  • src/backend/commands/tablespace.c (テーブルスペース名の検証)
  • src/backend/commands/user.c (ロール名の検証)
  • src/backend/replication/logical/origin.c (レプリケーションオリジン名の検証)
  • src/backend/utils/adt/acl.c (ロール名の検証)
  • **呼び出し先**:
  • 本関数は標準ライブラリや他の関数を呼び出しません。

関連データ構造

  • 特になし。

失敗モードと境界条件

  • **境界条件**: 3 文字未満の文字列が渡された場合、メモリ安全性の観点から境界外アクセスが発生するリスクがあります。呼び出し側で文字列長を保証する必要があります。

保守担当者向けメモ
