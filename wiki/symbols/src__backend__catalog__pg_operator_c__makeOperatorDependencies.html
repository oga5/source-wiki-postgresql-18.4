makeOperatorDependencies

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_operator.c:852

目的

演算子（pg_operator）に関連するオブジェクト間の依存関係を pg_depend カタログに登録する。

シグネチャまたは形状

ObjectAddress makeOperatorDependencies(HeapTuple tuple, bool makeExtensionDep, bool isUpdate)

入力と出力

  • **入力**:
  • tuple: 演算子の pg_operator 行を表す HeapTuple。
  • makeExtensionDep: 拡張機能への依存関係を記録するかどうかを示すフラグ。
  • isUpdate: 既存の演算子の更新であるかどうかを示すフラグ。
  • **出力**:
  • 登録された演算子の ObjectAddress を返す。

前提条件と不変条件

  • tuple は Form_pg_operator 構造体として解釈可能であること。
  • isUpdate が真の場合、既存の依存関係レコードを削除してから再登録を行う。
  • oprcom（交換子）および oprnegate（否定子）への依存関係は、本関数では記録されない（これらは RemoveOperatorById 等で手動管理される）。

アルゴリズムまたは振る舞い

1. 演算子の OID を基に ObjectAddress を特定する。 2. isUpdate が真であれば、既存の依存関係レコードを削除する。 3. 以下の各属性が有効（OidIsValid）な場合、それらへの依存関係を ObjectAddresses リストに追加する。

  • 名前空間 (oprnamespace)
  • 左オペランド型 (oprleft)
  • 右オペランド型 (oprright)
  • 結果型 (oprresult)
  • 実装関数 (oprcode)
  • 制限選択関数 (oprrest)
  • 結合選択関数 (oprjoin)

    • 4. record_object_address_dependencies を呼び出し、収集した依存関係を永続化する。 5. 所有者 (oprowner) への依存関係を recordDependencyOnOwner で記録する。 6. makeExtensionDep が真であれば、現在の拡張機能への依存関係を記録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_operator.c (演算子作成および更新処理)
  • src/backend/commands/operatorcmds.c (演算子コマンド処理)
  • **呼び出し先**:
  • deleteDependencyRecordsFor, deleteSharedDependencyRecordsFor
  • new_object_addresses, add_exact_object_address, record_object_address_dependencies, free_object_addresses
  • recordDependencyOnOwner, recordDependencyOnCurrentExtension

関連データ構造

  • Form_pg_operator: pg_operator カタログの行構造。
  • ObjectAddress: オブジェクトを一意に識別する構造体。
  • ObjectAddresses: 依存関係のリストを保持する構造体。

失敗モードと境界条件

  • OidIsValid チェックにより、無効な OID を持つ属性は依存関係の対象から除外される。
  • isUpdate 時の削除処理は、拡張機能メンバーシップを除外して実行される。

保守担当者向けメモ

  • 本関数は pg_depend への登録のみを担当する。演算子間の相互参照（交換子・否定子）のリンク更新は OperatorUpd が担うため、依存関係の設計と混同しないこと。
