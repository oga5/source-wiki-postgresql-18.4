AllocateVfd

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/fd.c:1417

目的

VFD（仮想ファイル記述子）キャッシュから未使用のファイルレコード（File）を確保します。キャッシュが枯渇している場合は、動的に領域を拡張します。

シグネチャまたは形状

File AllocateVfd(void)

入力と出力

  • **入力**: なし
  • **出力**: 確保された File 型のインデックス（VfdCache 配列の添字）。

前提条件と不変条件

  • InitFileAccess が呼び出され、SizeVfdCache が 0 より大きいこと。
  • VfdCache[0] は空きリストの管理用アンカーとして機能する。

アルゴリズムまたは振る舞い

1. VfdCache[0].nextFree を確認し、空きスロットがあるか判定する。 2. 空きがない場合、SizeVfdCache を 2 倍（最小 32）に拡張する。realloc に失敗した場合は ERROR を報告する。 3. 拡張された新しいエントリを初期化し、空きリストに連結する。 4. 空きリストの先頭からインデックスを取り出し、リストを更新して返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: PathNameOpenFile 等のファイルオープン処理（src/backend/storage/file/fd.c:1610 など）。
  • **呼び出し先**: realloc, MemSet, elog/ereport（PostgreSQLエラーハンドリング）。

関連データ構造

  • Vfd: 仮想ファイル記述子を保持する構造体。
  • VfdCache: Vfd 構造体のグローバル配列。
  • File: VfdCache のインデックスを表す型。

失敗モードと境界条件

  • **メモリ不足**: realloc が失敗した場合、ERRCODE_OUT_OF_MEMORY でプロセスを終了させる。

保守担当者向けメモ

  • VfdCache[0] は常に空きリストのアンカーとして使用されるため、実際のファイル操作には使用されない。
  • キャッシュ拡張時の realloc は、既存の VfdCache ポインタを上書きする前に一時変数に保持することで、メモリ確保失敗時のデータ破壊を防いでいる。
