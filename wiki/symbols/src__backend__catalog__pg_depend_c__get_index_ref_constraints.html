get_index_ref_constraints

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_depend.c:1116

目的

指定されたインデックス（indexId）を「参照先（reference）」として保持している制約（Constraint）の OID リストを取得します。

シグネチャまたは形状

List *get_index_ref_constraints(Oid indexId)

入力と出力

  • **入力**: indexId (Oid) - 検索対象のインデックスの OID。
  • **出力**: List * - インデックスに依存する制約の OID リスト。該当がない場合は NIL。

前提条件と不変条件

  • pg_depend システムカタログに対する AccessShareLock を取得して読み取りを行います。
  • DependReferenceIndexId インデックスを使用して効率的にスキャンします。

アルゴリズムまたは振る舞い

1. pg_depend テーブルを AccessShareLock で開きます。 2. pg_depend_reference_index を使用し、以下の条件でスキャンを開始します。

  • refclassid が RelationRelationId（インデックス自身）
  • refobjid が指定された indexId
  • refobjsubid が 0

    • 3. スキャン結果をループし、以下の条件を満たすレコードを抽出します。

  • classid が ConstraintRelationId であること。
  • objsubid が 0 であること。
  • deptype が DEPENDENCY_NORMAL であること。

    • 4. 条件に合致したレコードの objid（制約の OID）をリストに追加して返します。 5. スキャン終了後、テーブルをクローズします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/index.c:1660 にて呼び出されています。
  • **呼び出し先**: table_open, ScanKeyInit, systable_beginscan, systable_getnext, lappend_oid, systable_endscan, table_close 等のシステムカタログ操作関数。

関連データ構造

  • pg_depend: 依存関係を格納するシステムカタログ。
  • List: PostgreSQL のリスト構造。
  • Form_pg_depend: pg_depend カタログのタプル構造体。

失敗モードと境界条件

  • 指定された indexId に依存する制約が存在しない場合、空のリスト（NIL）を返します。
  • pg_depend テーブルへのアクセス権限やロック取得に失敗した場合は、PostgreSQL の標準的なエラーハンドリングにより例外が発生します。

保守担当者向けメモ

  • 本関数は「通常の依存関係（DEPENDENCY_NORMAL）」を持つ制約のみを対象としています。他の依存タイプ（DEPENDENCY_AUTO 等）が含まれる可能性がある場合は、ロジックの拡張が必要です。
  • refobjsubid を 0 に固定して検索しているため、インデックス全体に対する依存関係のみを対象としています。
