test_resowner_priorities

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_resowner/test_resowner_basic.c:49

目的

ResourceOwner 機構におけるリソースの解放順序、優先度、および親子階層を跨いだリソース管理が正しく機能することを検証するテスト関数です。

シグネチャまたは形状

Datum test_resowner_priorities(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_INT32(0): リソースの種類数 (nkinds)。0以下はエラー。
  • PG_GETARG_INT32(1): 各種類ごとのリソース数 (nresources)。0以下はエラー。
  • **出力**: PG_RETURN_VOID() により値を返さない。

前提条件と不変条件

  • nkinds および nresources は正の整数であること。
  • ResourceOwnerRelease 呼び出し後のリソース操作は禁止されている。
  • 親子関係にある ResourceOwner は、親が削除される前に適切に管理される必要がある。

アルゴリズムまたは振る舞い

1. CurrentResourceOwner を親とし、その下に子 ResourceOwner を作成する。 2. RESOURCE_RELEASE_BEFORE_LOCKS および RESOURCE_RELEASE_AFTER_LOCKS フェーズに対応する ResourceOwnerDesc をそれぞれ nkinds 分生成する。 3. 子および親の ResourceOwner に対して、指定された数 (nresources) のリソースを ResourceOwnerRemember で登録する。 4. ResourceOwnerRelease を以下の順序で呼び出し、各フェーズでの解放処理をトリガーする。

  • RESOURCE_RELEASE_BEFORE_LOCKS
  • RESOURCE_RELEASE_LOCKS
  • RESOURCE_RELEASE_AFTER_LOCKS

    • 5. 最後に ResourceOwnerDelete を呼び出し、親 ResourceOwner を破棄する。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**: ResourceOwnerCreate, ResourceOwnerEnlarge, ResourceOwnerRemember, ResourceOwnerRelease, ResourceOwnerDelete, palloc, psprintf, CStringGetDatum, elog。

関連データ構造

  • ResourceOwner: リソース所有権を管理する構造体。
  • ResourceOwnerDesc: リソースの解放フェーズや優先度、コールバック関数を定義する構造体。

失敗モードと境界条件

  • nkinds または nresources が 0 以下の場合は elog(ERROR) により中断される。
  • リソース登録時に ResourceOwnerEnlarge を呼び出さない場合や、解放フェーズの制約に違反する操作を行った場合、ResourceOwner API 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はログ出力 (elog(NOTICE, ...)) を通じて解放順序を可視化しているため、テスト実行時のログを確認することで、優先度とフェーズに基づく解放ロジックの正当性を検証できる。
  • ReleaseString および PrintString がリソースの解放・デバッグ出力に使用されている。
