get_distance

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/rangetypes_selfuncs.c:806

目的

2つの範囲境界（RangeBound）間の距離を計算します。主に範囲型の選択率推定において、境界間の間隔を数値化するために使用されます。

シグネチャまたは形状

float8 get_distance(TypeCacheEntry *typcache, const RangeBound *bound1, const RangeBound *bound2)

入力と出力

  • **入力**:
  • typcache: 型固有のキャッシュ情報（rng_subdiff_finfo を含む）。
  • bound1, bound2: 距離を測定する対象の範囲境界。
  • **出力**:
  • float8: 境界間の距離。無限大や計算不能な場合は 1.0 または get_float8_infinity() を返します。

前提条件と不変条件

  • typcache は有効な型キャッシュエントリである必要があります。
  • typcache->rng_subdiff_finfo が有効な場合、その関数は2つの Datum を受け取り、その差を float8 で返す必要があります。

アルゴリズムまたは振る舞い

1. **有限値の場合**: 両方の境界が有限であれば、typcache 内の subdiff 関数を呼び出して差分を計算します。

  • subdiff が未定義の場合は 1.0 を返します。
  • 計算結果が NaN または負数の場合は 1.0 に補正します。

    • 2. **無限値の場合**:

  • 両方が無限で、かつ lower フラグが一致する場合（両方とも正の無限大、または両方とも負の無限大）、距離は 0.0 です。
  • それ以外（一方が有限、または無限の方向が異なる）の場合は get_float8_infinity() を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/rangetypes_selfuncs.c 内の複数箇所（選択率計算ロジック）。
  • src/backend/utils/adt/multirangetypes_selfuncs.c 内の複数箇所。
  • **呼び出し先**:
  • typcache->rng_subdiff_finfo（関数ポインタ経由）。
  • get_float8_infinity()。

関連データ構造

  • RangeBound: 範囲の境界を表現する構造体。
  • TypeCacheEntry: 型の比較関数やサブ差分関数を保持するキャッシュ構造体。

失敗モードと境界条件

  • subdiff 関数が定義されていない場合、デフォルト値 1.0 が返されます。
  • 計算結果が NaN または負数となる異常値は、安全のため 1.0 に置き換えられます。
  • 無限大同士の比較において、方向が一致しない場合は無限大が返されます。

保守担当者向けメモ

  • 本関数は subdiff 関数の実装に強く依存しています。カスタム範囲型を追加する際は、subdiff 関数が適切に定義されているか確認してください。
  • 選択率推定の精度は、この関数が返す距離の正確性に依存します。
