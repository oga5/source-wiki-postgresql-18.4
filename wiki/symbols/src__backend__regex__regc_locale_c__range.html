range

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_locale.c:411

目的

正規表現における文字範囲（例: [a-z]）を表現する文字ベクトル（cvec）を生成します。大文字小文字を区別しないマッチングが指定されている場合は、範囲内の各文字に対応する大文字・小文字の等価文字を補完します。

シグネチャまたは形状

struct cvec *range(struct vars *v, chr a, chr b, int cases)

入力と出力

  • **入力**:
  • v: コンパイル状態を保持する vars 構造体。
  • a: 範囲の開始文字。
  • b: 範囲の終了文字。
  • cases: 大文字小文字を区別しない場合は非ゼロ、区別する場合はゼロ。
  • **出力**:
  • 成功時: 指定された範囲および等価文字を含む cvec 構造体へのポインタ。
  • 失敗時: NULL（範囲指定が不正な場合、またはメモリ不足等のエラー発生時）。

前提条件と不変条件

  • a が b より大きい場合、before(a, b) が偽となり、REG_ERANGE エラーが記録されます。
  • cases が指定された場合、範囲内の文字に対して pg_wc_tolower および pg_wc_toupper を適用します。
  • 大文字小文字を区別しない場合、生成される cvec のサイズは最大 100,000 文字に制限されます。

アルゴリズムまたは振る舞い

1. **範囲検証**: a から b への順序が妥当かを確認します。 2. **単純ケース**: cases が 0 の場合、getcvec で領域を確保し、addrange で範囲を追加して返します。 3. **大文字小文字非依存ケース**:

  • 指定された範囲 [a, b] を cvec に追加します。
  • 範囲内の各文字 c について、pg_wc_tolower(c) および pg_wc_toupper(c) を計算します。
  • 計算結果が元の文字と異なり、かつ元の範囲 [a, b] に含まれない場合、その文字を cvec に追加します。
  • 処理中に cvec の容量制限を超えた場合は REG_ETOOBIG エラーを記録します。
  • 各ループの反復ごとに INTERRUPT(v->re) を呼び出し、割り込みチェックを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内での直接的な呼び出し元は明示されていませんが、モジュール概要より regcomp.c から呼び出されることが示唆されています。
  • **呼び出し先**:
  • before(): 文字の順序比較。
  • getcvec(): cvec 構造体の確保。
  • addrange(): cvec への範囲追加。
  • addchr(): cvec への単一文字追加。
  • pg_wc_tolower() / pg_wc_toupper(): ロケール依存の大文字小文字変換。
  • ERR() / NOERRN(): エラー処理マクロ。
  • INTERRUPT(): 割り込みチェック。

関連データ構造

  • struct vars: コンパイル時のコンテキスト情報。
  • struct cvec: 文字集合を保持する構造体。
  • chr: 文字コードを表す型。

失敗モードと境界条件

  • a > b: REG_ERANGE エラー。
  • cv->nchrs >= cv->chrspace: REG_ETOOBIG エラー（大文字小文字補完時に発生）。
  • getcvec 失敗時: NOERRN マクロによりエラーが伝播し NULL を返します。

保守担当者向けメモ

  • ツールによる呼び出し元検索結果（verify_heapam.c 等）は、本関数 range とは別の文脈（ブロック範囲）を指しており、本関数の直接の呼び出し元ではありません。
  • 大文字小文字の補完ロジックは pg_wc_* 関数に依存しているため、ロケール定義の変更時には注意が必要です。
