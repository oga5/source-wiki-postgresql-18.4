pg_get_wal_records_info_till_end_of_wal

定義

  • 種別: function
  • 定義箇所: contrib/pg_walinspect/pg_walinspect.c:820

目的

指定された開始LSNから現在のWAL末尾までの範囲において、WALレコードの情報を取得・出力するためのSQLインターフェース関数です。

シグネチャまたは形状

Datum pg_get_wal_records_info_till_end_of_wal(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_LSN(0) により、開始位置となる XLogRecPtr を引数として受け取ります。
  • **出力**: PG_RETURN_VOID() を返し、内部で呼び出される GetWALRecordsInfo を通じてテーブル形式のデータを出力します。

前提条件と不変条件

  • 入力された start_lsn が現在のシステムWAL末尾（GetCurrentLSN()）を超えていないこと。超えている場合はエラーが発生します。
  • 本関数は古いバージョンとの互換性のために提供されており、新規開発では範囲指定可能な関数を使用することが推奨されます。

アルゴリズムまたは振る舞い

1. 引数から start_lsn を取得し、GetCurrentLSN() で現在のWAL末尾を取得します。 2. start_lsn > end_lsn の場合、ERRCODE_INVALID_PARAMETER_VALUE でエラーを報告します。 3. GetWALRecordsInfo を呼び出し、指定された範囲のWALレコード情報を処理します。 4. 処理完了後、void を返して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(pg_get_wal_records_info_till_end_of_wal) マクロにより登録されています。
  • **呼び出し先**:
  • GetCurrentLSN()
  • GetWALRecordsInfo()
  • ereport()

関連データ構造

  • XLogRecPtr: WAL上の位置を示すデータ型。
  • PG_FUNCTION_ARGS: PostgreSQLの関数呼び出し規約に基づく引数コンテキスト。

失敗モードと境界条件

  • start_lsn が現在のWAL末尾より大きい場合、ereport によりエラーが発生します。
  • WALの読み取り中に発生するエラー（GetWALRecordsInfo 内の処理に依存）は、PostgreSQLの標準的なエラーハンドリングに従います。

保守担当者向けメモ

  • 本関数は互換性維持のためのラッパーです。ロジックの変更が必要な場合は、共通の処理基盤である GetWALRecordsInfo を確認してください。
  • contrib/pg_walinspect/pg_walinspect.c 内の他の関数と同様、メモリ管理には注意が必要です。
