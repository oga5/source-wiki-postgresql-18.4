Archive

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_backup.h:226

目的

PostgreSQL のバックアップおよびリストア処理におけるメインコンテキストを保持し、ダンプ/リストアセッションの状態管理を行うための構造体です。

シグネチャまたは形状

typedef struct Archive {
    DumpOptions *dopt;
    RestoreOptions *ropt;
    int verbose;
    char *remoteVersionStr;
    int remoteVersion;
    bool isStandby;
    int minRemoteVersion;
    int maxRemoteVersion;
    int numWorkers;
    char *sync_snapshot_id;
    int encoding;
    bool std_strings;
    char *searchpath;
    char *use_role;
    bool exit_on_error;
    int n_errors;
    bool *is_prepared;
} Archive;

入力と出力

  • **入力**: CreateArchive または OpenArchive 関数により初期化され、DumpOptions や RestoreOptions を通じて設定を受け取ります。
  • **出力**: 保持された状態に基づき、WriteData や RestoreArchive などの API を介してアーカイブ形式に応じたデータ処理を実行します。

前提条件と不変条件

  • dopt はダンプ実行時に、ropt はリストア実行時に使用されます。
  • is_prepared は _dumpPreparedQueries 列挙型によってインデックス付けされます。
  • 構造体の末尾部分は非公開（private）として扱われ、直接的な操作は推奨されません。

アルゴリズムまたは振る舞い

1. **初期化**: CreateArchive または OpenArchive でハンドルを生成。 2. **設定**: SetArchiveOptions 等でオプションを適用。 3. **実行**: Archive ポインタを介して、pg_backup_custom.c 等の具体的な実装へ処理を委譲。 4. **終了**: CloseArchive を呼び出し、リソースを解放。

呼び出し元と呼び出し先

  • **呼び出し元**: 提供されたソース情報には、本構造体を直接操作する具体的な呼び出し元は記載されていません。
  • **呼び出し先**: pg_backup_archiver.c などの内部実装モジュールが、本構造体のメンバを参照・更新します。

関連データ構造

  • DumpOptions, RestoreOptions: ダンプ/リストアの設定保持。
  • _dumpPreparedQueries: プリペアドクエリの状態管理用インデックス。
  • ArchiveFormat, ArchiveMode: アーカイバの動作を規定する列挙型。

失敗モードと境界条件

  • exit_on_error が true の場合、SQL エラー発生時に処理を中断します。
  • n_errors はエラー発生回数をカウントし、エラー発生時でも処理を継続する場合の追跡に使用されます。

保守担当者向けメモ

  • Archive 構造体は非常に巨大ですが、外部モジュールは主にポインタを介した API 利用に集中してください。
  • 文字列のエスケープが必要な場合は、Archive のエンコーディング設定を利用する appendStringLiteralAH マクロを使用してください。
  • CatalogId（システムカタログ上のID）と DumpId（ダンプ処理上の論理ID）の混同を避けること。
