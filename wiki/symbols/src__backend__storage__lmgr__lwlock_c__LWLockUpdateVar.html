LWLockUpdateVar

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lwlock.c:1731

目的

LWLock が保護する共有変数（valptr）をアトミックに更新し、その変数の変更を待機しているプロセスを起床させる。

シグネチャまたは形状

void LWLockUpdateVar(LWLock *lock, pg_atomic_uint64 *valptr, uint64 val)

入力と出力

  • lock: 更新対象の LWLock へのポインタ。
  • valptr: 更新対象の 64 ビット共有変数へのポインタ。
  • val: valptr に書き込む新しい値。
  • 出力: なし（副作用として共有メモリの状態変更とプロセスの起床を行う）。

前提条件と不変条件

  • 呼び出し側は、対象の lock を排他モード（LW_EXCLUSIVE）で保持していなければならない（lock->state に LW_VAL_EXCLUSIVE フラグが立っていることが Assert で検証される）。

アルゴリズムまたは振る舞い

1. pg_atomic_exchange_u64 を使用し、メモリバリアを伴って valptr を val に更新する。 2. LWLockWaitListLock を取得し、ロックの待機リストを保護する。 3. 待機リストを走査し、LW_WAIT_UNTIL_FREE モードで待機しているプロセスを特定する。 4. 特定したプロセスを待機リストから削除し、一時的な起床リスト（wakeup）へ移動させる。各プロセスの状態を LW_WS_PENDING_WAKEUP に更新する。 5. LWLockWaitListUnlock を呼び出し、待機リストの保護を解除する。 6. 起床リスト内の各プロセスに対し、pg_write_barrier を実行した上で状態を LW_WS_NOT_WAITING に戻し、PGSemaphoreUnlock を呼び出してプロセスを起床させる。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/access/transam/xlog.c 内の WAL 挿入ロック管理処理などで確認されている。
  • 呼び出し先: LWLockWaitListLock, LWLockWaitListUnlock, proclist 操作関数群（proclist_init, proclist_foreach_modify, proclist_delete, proclist_push_tail）、GetPGProcByNumber, PGSemaphoreUnlock。

関連データ構造

  • LWLock: ロック本体および待機リスト（waiters）を保持。
  • PGPROC: 待機中のプロセス情報を保持する構造体。
  • proclist_head: 待機プロセスを管理する双方向リスト。

失敗モードと境界条件

  • 呼び出し時に排他ロックを保持していない場合、Assert により異常終了する。
  • 待機リストが空の場合、変数の更新のみを行い、起床処理はスキップされる。

保守担当者向けメモ

  • 本関数は LWLock の状態と連動した変数更新を行うための特殊なインターフェースである。
  • pg_atomic_exchange_u64 によるフルバリアが保証されているため、変数の更新は必ず起床処理の前に完了する。
  • LW_WAIT_UNTIL_FREE 以外の待機モードを持つプロセスは、この関数では起床対象とならない点に注意すること。
