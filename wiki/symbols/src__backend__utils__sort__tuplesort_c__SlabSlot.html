SlabSlot

定義

  • 種別: union
  • 定義箇所: src/backend/utils/sort/tuplesort.c:144

目的

マージフェーズにおいて、palloc/pfree のオーバーヘッドを回避するためのスラブアロケータ用メモリブロックとして機能します。

シグネチャまたは形状

typedef union SlabSlot
{
    union SlabSlot *nextfree;
    char        buffer[SLAB_SLOT_SIZE];
} SlabSlot;

入力と出力

  • 入力: なし（構造体定義）
  • 出力: なし（構造体定義）

前提条件と不変条件

  • buffer のサイズは SLAB_SLOT_SIZE で固定されます。
  • 空きリスト（free list）管理時は nextfree ポインタとして使用され、データ格納時は buffer として使用される共用体です。

アルゴリズムまたは振る舞い

  • スラブアロケータの空きリスト管理において、nextfree を介して次の空きスロットを指し示す連結リストを形成します。
  • メモリ割り当て時には、slabFreeHead からスロットを取り出し、解放時にはリストの先頭に繋ぎ直すことで高速なメモリ管理を実現します。

呼び出し元と呼び出し先

  • 呼び出し元: tuplesort.c 内のメモリ初期化およびスラブ管理ロジック（slabFreeHead の操作など）。
  • 呼び出し先: なし（データ構造定義のため）。
  • 補足: 呼び出し元・呼び出し先の詳細な関数呼び出しグラフは、提供されたソース excerpt の範囲外です。

関連データ構造

  • Tuplesortstate: スラブアロケータの管理状態（slabFreeHead 等）を保持するメイン構造体。

失敗モードと境界条件

  • スラブ領域の枯渇: slabFreeHead が NULL になった場合、これ以上のスロット割り当てはできません。
  • 型キャスト: tuple や slabMemoryBegin からのキャスト時に使用されますが、SLAB_SLOT_SIZE との整合性が取れていない場合、メモリ破壊を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は tuplesort.c 内のメモリ管理最適化に特化しています。
  • nextfree と buffer の共用体構造であるため、メモリレイアウトを変更する際は SLAB_SLOT_SIZE との整合性に十分注意してください。
