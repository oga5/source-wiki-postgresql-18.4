pgstat_report_replslot

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_replslot.c:77

目的

レプリケーションスロットに関連する統計情報を、共有メモリ上の統計エントリに累積加算する。

シグネチャまたは形状

void pgstat_report_replslot(ReplicationSlot *slot, const PgStat_StatReplSlotEntry *repSlotStat)

入力と出力

  • slot: 統計を更新する対象のレプリケーションスロット構造体。
  • repSlotStat: 加算する統計値（トランザクション数、バイト数等）を保持する構造体。
  • 出力: なし（共有メモリ上の統計エントリを直接更新する）。

前提条件と不変条件

  • 呼び出し元は、対象のレプリケーションスロットが有効であることを保証する必要がある。
  • 内部で pgstat_get_entry_ref_locked を使用して共有メモリ上の統計エントリをロックするため、呼び出し元で追加のロックを保持する必要はない。

アルゴリズムまたは振る舞い

1. ReplicationSlotIndex(slot) を用いて、対象スロットの統計エントリを共有メモリから取得し、ロックする。 2. REPLSLOT_ACC マクロを使用して、repSlotStat に含まれる以下の値を共有メモリ上の統計エントリに加算する。

  • spill_txns, spill_count, spill_bytes
  • stream_txns, stream_count, stream_bytes
  • total_txns, total_bytes

    • 3. pgstat_unlock_entry を呼び出し、統計エントリのロックを解放する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/replication/logical/logical.c
  • src/backend/replication/slot.c
  • 呼び出し先:
  • pgstat_get_entry_ref_locked
  • pgstat_unlock_entry

関連データ構造

  • ReplicationSlot: レプリケーションスロットの制御構造体。
  • PgStat_StatReplSlotEntry: 統計情報の保持用構造体。
  • PgStatShared_ReplSlot: 共有メモリ上に配置される統計エントリのラッパー。

失敗モードと境界条件

  • 統計エントリの取得に失敗した場合、pgstat_get_entry_ref_locked 内部でエラーが発生する可能性がある。
  • 物理レプリケーションスロットに対しては統計が収集されない（論理レプリケーションスロット専用）。

保守担当者向けメモ

  • 本関数は、レプリケーションの進行に伴い頻繁に呼び出されるため、統計更新のオーバーヘッドを最小限に抑える設計となっている。
  • 統計項目を追加する場合は、REPLSLOT_ACC マクロの対象フィールドを更新すること。
