arcbatch

定義

  • 種別: struct
  • 定義箇所: src/include/regex/regguts.h:319

目的

正規表現のコンパイル過程において、arc 構造体を一括でメモリ確保（バルクアロケーション）するための管理用構造体です。

シグネチャまたは形状

struct arcbatch {
    struct arcbatch *next;
    size_t narcs;
    struct arc a[FLEXIBLE_ARRAY_MEMBER];
};

入力と出力

  • **入力**: ARCBATCHSIZE(n) マクロを使用して、必要なアーク数 n に基づくメモリサイズが計算されます。
  • **出力**: 確保された arcbatch 構造体は、NFA 構築時のアーク格納領域として利用されます。

前提条件と不変条件

  • a メンバは FLEXIBLE_ARRAY_MEMBER として定義されており、構造体の末尾に可変長のアーク配列が続くメモリレイアウトを前提としています。
  • next ポインタにより、複数の arcbatch が連結されたリスト構造を形成します。

アルゴリズムまたは振る舞い

1. NFA 構築時にアークの追加が必要となった際、既存の arcbatch に空きがあるか確認します。 2. 空きがない場合、MALLOC を用いて新しい arcbatch を確保し、既存のリストの先頭または末尾に連結します。 3. ARCBATCHSIZE マクロにより、構造体ヘッダーと可変長配列部分の合計サイズが動的に算出されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_nfa.c 内の NFA 構築ロジックが、アークの追加時に本構造体を動的に確保・参照しています。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行いません。

関連データ構造

  • arc: arcbatch 内に格納される個々のアーク情報。
  • nfa: arcbatch のリストを管理し、コンパイル状態を保持する構造体。
  • guts: lastab メンバを通じて arcbatch のチェーンを保持します。

失敗モードと境界条件

  • メモリ確保失敗時: MALLOC が失敗した場合、正規表現のコンパイル処理は中断されます。
  • narcs の値: 確保時に指定された数を超えて a 配列にアクセスすることはできません。

保守担当者向けメモ

  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体のサイズ計算には必ず ARCBATCHSIZE マクロを使用してください。
  • regc_nfa.c でのメモリ確保と解放のライフサイクルが nfa 構造体の破棄と一致していることを確認してください。
