DefineTSConfiguration

定義

  • 種別: function
  • 定義箇所: src/backend/commands/tsearchcmds.c:898

目的

SQLの CREATE TEXT SEARCH CONFIGURATION コマンドを実行し、新しいテキスト検索設定をシステムカタログ pg_ts_config に登録します。既存の設定をコピーして作成することも可能です。

シグネチャまたは形状

ObjectAddress DefineTSConfiguration(List *names, List *parameters, ObjectAddress *copied)

入力と出力

  • **入力**:
  • names: 設定の名前（スキーマ修飾名）。
  • parameters: parser または copy を指定する DefElem のリスト。
  • copied: コピー元設定の ObjectAddress を格納するためのポインタ（指定時）。
  • **出力**:
  • 作成された設定オブジェクトの ObjectAddress を返します。

前提条件と不変条件

  • 対象スキーマに対する CREATE 権限が必要です。
  • parser オプションと copy オプションを同時に指定することはできません。
  • parser が指定されていない場合、copy 元の設定からパーサーを継承する必要があります。

アルゴリズムまたは振る舞い

1. QualifiedNameGetCreationNamespace で名前空間を特定し、権限チェックを行います。 2. parameters を走査し、parser OID または copy 元の sourceOid を抽出します。 3. sourceOid が指定されている場合、システムキャッシュから設定を読み込み、パーサーを継承します。 4. pg_ts_config カタログをオープンし、新しい設定のタプルを作成して挿入します。 5. sourceOid が指定されている場合、pg_ts_config_map からトークンと辞書のマッピング情報を読み取り、新しい設定に対して一括挿入（CatalogTuplesMultiInsertWithInfo）を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/utility.c (SQLコマンド実行時)
  • **呼び出し先**:
  • QualifiedNameGetCreationNamespace, object_aclcheck, get_ts_parser_oid, get_ts_config_oid, table_open, CatalogTupleInsert, systable_beginscan 等。

関連データ構造

  • pg_ts_config: 設定本体を保持するシステムカタログ。
  • pg_ts_config_map: トークン型と辞書のマッピングを保持するシステムカタログ。
  • ObjectAddress: 作成されたオブジェクトの識別子。

失敗モードと境界条件

  • 指定された名前空間に作成権限がない場合、エラーとなります。
  • parser と copy の両方が指定されている場合、構文エラーとなります。
  • parser が特定できない場合、エラーとなります。
  • sourceOid が無効な場合、キャッシュルックアップ失敗によりエラーとなります。

保守担当者向けメモ

  • 本関数は pg_ts_config_map のコピー時に MAX_CATALOG_MULTI_INSERT_BYTES に基づくバッチ処理を行っています。マッピング数が多い場合のメモリ効率に留意してください。
  • 依存関係（pg_depend）の構築ロジックは本関数の後続処理（ソース抜粋外）で実行される想定です。
