PostPrepare_Locks

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lock.c:3541

目的

準備済みトランザクション（Prepared Transaction）へロックの所有権を移譲する。トランザクションのコミット準備完了後、そのトランザクションが保持していたロックを、共有メモリ上のダミープロセス（TwoPhaseGetDummyProc）へ引き継ぐために使用される。

シグネチャまたは形状

void PostPrepare_Locks(TransactionId xid)

入力と出力

  • **入力**: xid (準備対象のトランザクションID)
  • **出力**: なし（副作用として共有メモリ上のロック所有権が更新される）

前提条件と不変条件

  • 呼び出し前に AtPrepare_Locks が実行され、高速パス（Fast-Path）ロックが既にメインテーブルへ移行済みであること。
  • 呼び出し元プロセスがロックグループのリーダーであるか、グループに属していないこと。
  • セッションレベルのロックとトランザクションレベルのロックを同一オブジェクトに対して同時に保持していないこと。

アルゴリズムまたは振る舞い

1. **ローカルロックの整理**: LockMethodLocalHash を走査し、準備済みトランザクションに引き継ぐべきロックを特定する。セッションレベルのみのロックは無視し、トランザクションレベルのロックに対して releaseMask を設定する。 2. **共有メモリ上のロック移譲**: 各ロックパーティションを順次ロックし、PROCLOCK エントリを走査する。 3. **所有権の更新**: hash_update_hash_key を使用して、PROCLOCK の所有者を現在のプロセスからダミープロセスへ変更する。 4. **リストの再構築**: dlist を操作し、ロックを現在のプロセスのリストからダミープロセスのリストへ移動させる。 5. **クリティカルセクション**: 処理全体を START_CRIT_SECTION() で囲み、エラー発生時の整合性を保護する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xact.c:2697
  • **呼び出し先**: TwoPhaseGetDummyProc, hash_seq_init, hash_seq_search, RemoveLocalLock, LockHashPartitionLockByIndex, LWLockAcquire, LWLockRelease, hash_update_hash_key, dlist_delete, dlist_push_tail 等。

関連データ構造

  • LOCALLOCK: プロセスローカルなロック管理構造体。
  • PROCLOCK: プロセスとロックオブジェクトの関連付けを管理する共有メモリ上の構造体。
  • LOCK: ロックオブジェクトそのものを表す共有メモリ上の構造体。
  • PGPROC: プロセス管理構造体（ダミープロセスを含む）。

失敗モードと境界条件

  • **PANIC**: セッションレベルとトランザクションレベルのロックを混在して保持している場合、またはロックの再割り当て時にハッシュテーブルで重複エントリが検出された場合に発生する。
  • **メモリ不足**: 共有メモリ不足によりロック設定が不完全な場合、ローカルロックエントリを削除して処理を継続する。

保守担当者向けメモ

  • 本関数は AtPrepare_Locks と対になって動作する。AtPrepare_Locks で高速パスを排除しているため、本関数実行中に他バックエンドがリストを操作することはない。
  • hash_update_hash_key を使用しているのは、ハッシュキー（所有者）を変更する際に、メモリ確保の失敗リスクを避けるためである。
