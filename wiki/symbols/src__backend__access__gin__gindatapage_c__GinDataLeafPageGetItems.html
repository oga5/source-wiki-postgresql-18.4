GinDataLeafPageGetItems

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/gindatapage.c:134

目的

GINポスティングツリーのリーフページから、TID（Tuple Identifier）のリストを抽出して返します。

シグネチャまたは形状

ItemPointer GinDataLeafPageGetItems(Page page, int *nitems, ItemPointerData advancePast)

入力と出力

  • **入力**:
  • page: 読み取り対象のGINリーフページ。
  • nitems: 取得したTIDの個数を格納する出力用ポインタ。
  • advancePast: スキップ対象となるTIDの閾値。これより大きいTIDから抽出を開始します。
  • **出力**:
  • 抽出されたTIDの配列（ItemPointer）。呼び出し元でメモリ解放が必要です。

前提条件と不変条件

  • page は有効なGINリーフページである必要があります。
  • advancePast が有効な場合、その値以下のTIDを含むセグメントはスキップされます。

アルゴリズムまたは振る舞い

1. **ページ形式の判定**: GinPageIsCompressed(page) を使用して、ページが圧縮されているか否かを判定します。 2. **圧縮ページの場合**:

  • ページ内のポスティングリストセグメントを走査します。
  • advancePast が指定されている場合、ginCompareItemPointers を用いて、該当するTIDを含むセグメントまでスキップします。
  • ginPostingListDecodeAllSegments を呼び出し、残りのセグメントをデコードしてTIDリストを生成します。

    • 3. **非圧縮ページの場合**:

  • dataLeafPageGetUncompressed を呼び出し、ページから直接TIDリストを取得します。
  • 取得したデータを palloc で確保したメモリ領域にコピーして返します。

    • 4. **戻り値**: 抽出されたTIDの配列を返します。該当データがない場合は NULL を返し、nitems に 0 を設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • src/backend/access/gin/ginget.c
  • **呼び出し先**:
  • GinDataLeafPageGetPostingList / GinDataLeafPageGetPostingListSize
  • GinNextPostingListSegment
  • ginCompareItemPointers
  • ginPostingListDecodeAllSegments
  • dataLeafPageGetUncompressed

関連データ構造

  • ItemPointerData: TIDを保持する構造体。
  • GinPostingList: 圧縮されたポスティングリストのヘッダ構造。

失敗モードと境界条件

  • ページが空、または advancePast 以降にデータが存在しない場合、result は NULL となり nitems は 0 になります。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本関数は圧縮・非圧縮の両方のページ形式を透過的に扱うため、ページレイアウトの変更時には両方のパスを確認してください。
  • advancePast によるスキップ処理は、インデックススキャン時の効率化に直結するため、比較ロジックの変更には注意が必要です。
