revmap_physical_extend

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_revmap.c:521

目的

BRINインデックスのリバースレンジマップ（revmap）を物理的に拡張し、新しいrevmapページを割り当ててメタページを更新する。

シグネチャまたは形状

static void revmap_physical_extend(BrinRevmap *revmap)

入力と出力

  • 入力: BrinRevmap *revmap （対象インデックスのrevmap状態を保持する構造体）
  • 出力: なし（副作用としてメタページおよびインデックスの物理構造を更新）

前提条件と不変条件

  • revmap->rm_metaBuf がメタページを保持していること。
  • メタページの lastRevmapPage が、現在のインデックスの物理的なrevmap末尾と整合していること。
  • 拡張対象のページが既存のBRINページと競合する場合、退避処理が優先されること。

アルゴリズムまたは振る舞い

1. メタページを排他ロックし、キャッシュされた lastRevmapPage が最新か確認する。古い場合はロックを解除して終了する。 2. 次のrevmapページ番号を算出する。 3. 該当ブロックが既に存在する場合は読み込み、存在しない場合は ExtendBufferedRel でインデックスを物理拡張する。 4. 取得したページが既存のBRINページとして使用されている場合、brin_evacuate_page を呼び出してデータを退避させ、呼び出し元に制御を戻す。 5. ページが空または新規の場合、brin_page_init で BRIN_PAGETYPE_REVMAP として初期化する。 6. メタページの lastRevmapPage を更新し、WALログ（XLOG_BRIN_REVMAP_EXTEND）を記録する。 7. 最後にメタページと対象バッファのロックを解除する。

呼び出し元と呼び出し先

  • 呼び出し元: revmap_extend_and_get_blkno （src/backend/access/brin/brin_revmap.c:511）
  • 呼び出し先: LockBuffer, ReadBuffer, ExtendBufferedRel, brin_start_evacuating_page, brin_evacuate_page, brin_page_init, XLogInsert 等

関連データ構造

  • BrinRevmap: revmapのアクセス状態を管理。
  • BrinMetaPageData: インデックスのメタデータ（lastRevmapPage を保持）。
  • Buffer: バッファマネージャの管理単位。

失敗モードと境界条件

  • **同時拡張**: 他のバックエンドが並行してインデックスを拡張した場合、BufferGetBlockNumber の不一致を検知して処理を中断する。
  • **ページ破損**: 取得したページが BRIN_IS_REGULAR_PAGE でもなく新規でもない場合、ERRCODE_INDEX_CORRUPTED を発生させる。
  • **退避発生**: ページが既に使用中の場合、退避処理を実行して終了する。

保守担当者向けメモ

  • 本関数はメタページを排他ロックするため、頻繁な呼び出しはインデックス全体の競合を招く可能性がある。
  • pd_lower の設定は、古いバージョンのPostgreSQLとの互換性およびWALリプレイ時のメタデータ保護のために必須である。
