GucInfoMain

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/misc/help_config.c:45

目的

PostgreSQLの起動オプション --help-config が指定された際に、システム内の全設定パラメータ（GUC）を標準出力へ一覧表示するためのエントリポイントとして機能します。

シグネチャまたは形状

void GucInfoMain(void)

入力と出力

  • **入力**: なし（引数を取らない）
  • **出力**: なし（戻り値は void）。処理完了後、exit(0) によりプロセスを終了します。

前提条件と不変条件

  • 本関数は、GUCシステムが初期化される前の段階で呼び出されることを想定しています。
  • 内部で build_guc_variables() を呼び出し、GUCハッシュテーブルを構築する必要があります。

アルゴリズムまたは振る舞い

1. build_guc_variables() を呼び出し、GUC変数のハッシュテーブルを初期化します。 2. get_guc_variables() を呼び出し、全GUC変数のポインタ配列と個数を取得します。 3. 取得した各変数を mixedStruct 型にキャストし、ループ処理を行います。 4. displayStruct() を用いて、各変数が表示対象であるか（GUC_NO_SHOW_ALL フラグ等に基づき）を判定します。 5. 表示対象であれば printMixedStruct() を呼び出し、型に応じたフォーマットで標準出力へ書き出します。 6. 全ての処理終了後、exit(0) を実行して終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/main/main.c:220
  • **呼び出し先**:
  • build_guc_variables()
  • get_guc_variables()
  • displayStruct()
  • printMixedStruct()
  • exit()

関連データ構造

  • config_generic: GUC変数の基底構造体。
  • mixedStruct: 各種GUC構造体（config_bool, config_int 等）を統一的に扱うための共用体。

失敗モードと境界条件

  • get_guc_variables() が返す変数のリストが空の場合、ループは実行されず即座に exit(0) へ移行します。
  • 表示対象の変数が存在しない場合、出力は行われません。

保守担当者向けメモ

  • 本関数はサーバー起動時の初期化プロセスの一部として動作します。
  • mixedStruct へのキャストは、各GUC構造体の先頭メンバが config_generic であるというメモリレイアウトの前提に依存しています。
  • 特定のパラメータが表示されない場合は、guc_tables.c におけるフラグ設定を確認してください。
