PLpgSQL_rwopt

定義

  • 種別: enum
  • 定義箇所: src/pl/plpgsql/src/plpgsql.h:194

目的

PL/pgSQL における式（expression）の評価時、読み書き（R/W）最適化を適用可能かどうかを判定・管理するための列挙型です。

シグネチャまたは形状

typedef enum PLpgSQL_rwopt
{
    PLPGSQL_RWOPT_UNKNOWN = 0,
    PLPGSQL_RWOPT_NOPE,
    PLPGSQL_RWOPT_TRANSFER,
    PLPGSQL_RWOPT_INPLACE,
} PLpgSQL_rwopt;

入力と出力

本シンボル自体は列挙型であり、入力や出力を持つ関数ではありません。状態値として以下のいずれかをとります。

  • PLPGSQL_RWOPT_UNKNOWN: 最適化の可否が未判定。
  • PLPGSQL_RWOPT_NOPE: 最適化不可。
  • PLPGSQL_RWOPT_TRANSFER: 旧値を式状態へ転送可能。
  • PLPGSQL_RWOPT_INPLACE: 最上位関数へ読み書き可能（R/W）な値として直接渡すことが可能。

前提条件と不変条件

  • PLPGSQL_RWOPT_UNKNOWN が初期値として使用されます。
  • 実行エンジンが最適化の可否を判断する過程で、この値が更新されます。

アルゴリズムまたは振る舞い

PL/pgSQL の実行エンジンが式を評価する際、メモリ効率や処理速度を向上させるために、値をコピーせずに直接操作（R/W）できるかを判定します。この列挙値はその判定結果を保持し、実行時のデータ受け渡し戦略を決定する指標となります。

呼び出し元と呼び出し先

  • 呼び出し元: src/pl/plpgsql/src/plpgsql.h:268 にて PLpgSQL_expr 構造体のメンバ expr_rwopt として定義されています。
  • 呼び出し先: 提供されたソース情報からは、具体的な関数による参照先は特定できません。

関連データ構造

  • PLpgSQL_expr: 式を保持する構造体であり、本列挙型をメンバとして保持します。

失敗モードと境界条件

  • PLPGSQL_RWOPT_UNKNOWN のまま処理が進んだ場合、最適化が適用されない、あるいはデフォルトの動作が選択される可能性があります。

保守担当者向けメモ

  • 本列挙型は PLpgSQL_expr の最適化戦略を制御する重要なフラグです。新しい最適化手法を追加する際は、この列挙型に新しい状態を追加し、実行エンジン側でのハンドリングを更新する必要があります。
