main

定義

  • 種別: function
  • 定義箇所: src/tools/pg_bsd_indent/indent.c:65

目的

BSD indent ユーティリティの実行起点として、プログラムの初期化、コマンドライン引数の解析、およびソースコード整形のためのメイン制御ループを実行します。

シグネチャまたは形状

int main(int argc, char **argv)

入力と出力

  • **入力**:
  • argc, argv: コマンドライン引数（オプション指定および入力/出力ファイル名）。
  • 標準入力（stdin）: 入力ファイルが指定されない場合に使用。
  • **出力**:
  • 戻り値: 成功時に 0、エラー発生時に 1 を返します。
  • 標準出力（stdout）: 整形結果の書き出し先。

前提条件と不変条件

  • グローバル状態 ps (parser_state) がプログラム全体で共有され、解析の進捗を保持します。
  • メモリ確保（malloc）が失敗した場合、err() を呼び出して即座に終了します。
  • 入力ファイルと出力ファイルが同一であることは許可されません。

アルゴリズムまたは振る舞い

1. **初期化**: グローバル変数、バッファ（combuf, labbuf, codebuf, tokenbuf 等）、およびパーサー状態をデフォルト値で設定します。 2. **引数解析**: コマンドライン引数を走査し、プロファイル設定の適用や入出力ファイル名の特定を行います。 3. **ファイル準備**: 入力ファイルを開き、出力先が指定されていない場合はバックアップ作成（bakcopy()）を伴う上書き処理を準備します。 4. **環境設定**: ユーザー指定がない場合のデフォルトのインデント設定やコメント配置ルールを適用します。 5. **実行開始**: fill_buffer() で入力ストリームを読み込み、parse(semicolon) を呼び出して解析を開始します。

呼び出し元と呼び出し先

  • **呼び出し元**: OS のシェル（プログラムのエントリポイント）。
  • **呼び出し先**:
  • malloc, err, errx, fopen, strcmp: 標準ライブラリ関数。
  • alloc_typenames, set_defaults, set_profile, set_option, bakcopy, fill_buffer, parse: 本モジュール内の関数。
  • **注意**: ツールが提示した contrib/amcheck 等の参照は、PostgreSQL ソースツリー内の同名の変数や概念に対するものであり、本関数 main との直接的な呼び出し関係を示すものではありません。

関連データ構造

  • parser_state ps: 解析中のコードの状態（インデントレベル、括弧の深さ等）を保持するグローバル構造体。
  • in_buffer, codebuf, labbuf, combuf, tokenbuf: 入出力およびトークン処理用のバッファ。

失敗モードと境界条件

  • **メモリ不足**: 各種バッファの確保失敗時に err(1, NULL) で終了します。
  • **ファイル操作エラー**: 入力ファイルが開けない場合、または出力ファイル作成に失敗した場合にエラー終了します。
  • **引数エラー**: 入力ファイルと出力ファイルが同一である場合、または不明なパラメータが渡された場合に errx で終了します。

保守担当者向けメモ

  • 本関数は非常に長い初期化処理を含みます。ps 構造体の各フィールドがどのように初期化されているかを確認する際は、indent_globs.h の定義と併せて参照してください。
  • main の後半で呼び出される parse() 以降が、実際の字句解析と整形処理のメインループとなります。
