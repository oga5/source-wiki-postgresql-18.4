ProcArrayEndTransaction

定義

  • **種別**: function
  • **定義箇所**: src/backend/storage/ipc/procarray.c:666

目的

トランザクション終了時に、当該バックエンドプロセスが保持するトランザクションID（XID）を「実行中」の集合から除外し、共有メモリ上のプロセス状態を更新する。

シグネチャまたは形状

void ProcArrayEndTransaction(PGPROC *proc, TransactionId latestXid)

入力と出力

  • **入力**:
  • proc: 対象となるバックエンドの PGPROC 構造体へのポインタ。
  • latestXid: 終了するトランザクションの最新のXID（有効な場合）。
  • **出力**: なし（副作用として共有メモリ上の状態を更新）。

前提条件と不変条件

  • latestXid が有効な場合、proc->xid も有効である必要がある。
  • latestXid が無効な場合、proc->xid は無効であり、サブトランザクションの状態（subxidStatus）は空である必要がある。

アルゴリズムまたは振る舞い

1. **XIDが有効な場合**:

  • 他のプロセスがスナップショットを取得する際に、当該プロセスが「実行中」として誤認されないよう、ProcArrayLock を取得して状態をクリアする。
  • LWLockConditionalAcquire でロックが即時取得できれば ProcArrayEndTransactionInternal を呼び出す。
  • ロックが取得できない場合は、効率化のため ProcArrayGroupClearXid を使用してグループ単位でクリアを行う。

    • 2. **XIDが無効な場合**:

  • ロックを取得せず、proc->vxid.lxid、proc->xmin、proc->delayChkptFlags、proc->recoveryConflictPending をリセットする。
  • プロセスが PROC_VACUUM_STATE_MASK フラグを保持している場合は、ProcArrayLock を排他取得し、共有メモリ上の ProcGlobal->statusFlags を更新してフラグをクリアする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/xact.c (コミット処理時)
  • src/backend/utils/time/snapmgr.c
  • **呼び出し先**:
  • ProcArrayEndTransactionInternal
  • ProcArrayGroupClearXid
  • LWLockAcquire / LWLockRelease / LWLockConditionalAcquire

関連データ構造

  • PGPROC: バックエンドプロセスの状態を保持する構造体。
  • ProcArrayLock: プロセス配列へのアクセスを制御する LWLock。
  • ProcGlobal: 共有メモリ上のプロセス状態を管理するグローバル構造体。

失敗モードと境界条件

  • ProcArrayLock の競合が発生した場合、ProcArrayGroupClearXid を用いたグループ化処理へフォールバックすることで、パフォーマンス低下を抑制する設計となっている。
  • バキューム状態のクリア時には、共有キャッシュラインの不要なダーティ化を避けるため、フラグの変更がある場合のみロックを取得する。

保守担当者向けメモ

  • 本関数は頻繁に呼び出されるため、ロックの取得範囲を最小限に抑えることが重要である。
  • ProcArrayGroupClearXid への分岐は、高負荷時のスケーラビリティを確保するための最適化である。
