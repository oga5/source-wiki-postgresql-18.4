pg_atomic_unlocked_test_flag

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/atomics.h:195

目的

アトミックフラグが現在ロックされていない状態（free）であるかどうかを確認します。

シグネチャまたは形状

bool pg_atomic_unlocked_test_flag(volatile pg_atomic_flag *ptr)

入力と出力

  • **入力**: ptr - 確認対象の pg_atomic_flag 構造体へのポインタ。volatile 修飾されています。
  • **出力**: フラグがロックされていない場合は true、ロックされている場合は false を返します。

前提条件と不変条件

  • ptr は適切にアライメントされている必要があります。
  • 本関数はアトミックな読み取り操作を伴いますが、ロックの取得（セット）は行いません。

アルゴリズムまたは振る舞い

内部的に pg_atomic_unlocked_test_flag_impl(ptr) を呼び出し、プラットフォーム固有の実装へ処理を委譲します。この関数は状態を変化させず、現在のフラグ値に基づいた真偽値を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/autovacuum.c: wi_dobalance フラグの状態確認に使用。
  • src/test/regress/regress.c: アトミック操作のテストケース内で使用。
  • **呼び出し先**:
  • pg_atomic_unlocked_test_flag_impl（プラットフォーム固有の実装）。

関連データ構造

  • pg_atomic_flag: アトミック操作対象のフラグ型。

失敗モードと境界条件

  • ptr が NULL の場合の挙動は未定義です。
  • メモリ配置が不適切な場合、プラットフォームによっては不正なメモリアクセスが発生する可能性があります。

保守担当者向けメモ

  • 本関数は「ロックの確認」のみを行うため、呼び出し直後に別のスレッドがフラグをセットする可能性がある（競合状態）ことに注意してください。状態の変更を伴う排他制御が必要な場合は pg_atomic_test_set_flag を検討してください。
