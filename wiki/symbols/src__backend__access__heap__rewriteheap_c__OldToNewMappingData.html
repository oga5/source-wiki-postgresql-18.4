OldToNewMappingData

定義

  • **種別**: struct
  • **定義箇所**: src/backend/access/heap/rewriteheap.c:179

目的

テーブル再構築（VACUUM FULL や CLUSTER）時、論理レプリケーションの整合性を維持するために、古いヒープ上のタプル位置（TID）と新しいヒープ上のタプル位置の対応関係を保持する。

シグネチャまたは形状

typedef struct {
    TidHashKey    key;          /* actual xmin/old location of B tuple */
    ItemPointerData new_tid;    /* where we put it in the new heap */
} OldToNewMappingData;

入力と出力

  • **入力**: ハッシュテーブルのキーとして TidHashKey を受け取り、値として新しい物理位置 ItemPointerData を格納する。
  • **出力**: なし（データ保持用構造体）。

前提条件と不変条件

  • rewriteheap.c 内のハッシュテーブル（rs_old_new_tid_map）の要素として使用される。
  • key フィールドは、古いヒープにおけるタプルの xmin および物理的な位置を特定する。

アルゴリズムまたは振る舞い

  • テーブル再構築中に、論理デコーディングが必要なタプルが新しいヒープへ書き込まれる際、この構造体にマッピング情報が記録される。
  • rewriteheap.c 内のハッシュテーブル構築時、sizeof(OldToNewMappingData) がエントリサイズとして指定される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/rewriteheap.c 内のハッシュテーブル管理ロジック（hash_ctl.entrysize の設定箇所）。
  • **呼び出し先**: なし（データ構造体のため）。

関連データ構造

  • TidHashKey: マッピングのキーとして使用される。
  • OldToNewMapping: OldToNewMappingData へのポインタ型。

失敗モードと境界条件

  • メモリ不足: 非常に巨大なテーブルで大量の更新チェーンが存在する場合、ハッシュテーブルの肥大化によりメモリ不足が発生する可能性がある。

保守担当者向けメモ

  • 本構造体は論理レプリケーションの整合性に直結する。変更を加える際は、pg_logical/mappings に書き出されるマッピングファイルの形式と整合しているか確認すること。
