pg_cancel_conn

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/fe-cancel.c:31

目的

PostgreSQLサーバーに対するクエリキャンセル要求を送信するための、専用接続状態を保持する構造体です。

シグネチャまたは形状

struct pg_cancel_conn
{
    PGconn      conn;
};

入力と出力

  • 入力: なし（構造体定義）
  • 出力: なし（構造体定義）

前提条件と不変条件

  • PGcancelConn（pg_cancel_conn の型定義）の基盤となる構造体です。
  • 既存の PGconn から抽出された認証情報（PIDおよびキャンセルキー）を保持し、キャンセル専用の通信経路を管理するために使用されます。

アルゴリズムまたは振る舞い

  • 本構造体は、キャンセル要求送信のための接続状態をラップするコンテナとして機能します。
  • PQcancelCreate 等の関数を通じて生成され、非同期キャンセル処理においてソケット設定や接続状態を保持します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/fe-cancel.c 内のキャンセル関連関数群。
  • 呼び出し先: 情報なし。

関連データ構造

  • PGconn: 本構造体が内部で保持する接続情報構造体。
  • PGcancelConn: pg_cancel_conn の typedef 名（src/interfaces/libpq/libpq-fe.h:207）。

失敗モードと境界条件

  • 構造体自体の定義に関する失敗モードはありません。
  • インスタンス化時のメモリ割り当て失敗や、保持する PGconn の初期化失敗が後続の処理に影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体は libpq の内部的なキャンセル通信経路を管理します。
  • 修正を行う際は、fe-cancel.c で定義されている非同期API（PQcancelStart 等）との整合性を維持してください。
