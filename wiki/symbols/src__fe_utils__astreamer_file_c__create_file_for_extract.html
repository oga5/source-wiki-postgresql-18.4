create_file_for_extract

定義

  • 種別: function
  • 定義箇所: src/fe_utils/astreamer_file.c:371

目的

アーカイブ展開処理において、指定されたパスに新しいファイルをバイナリ書き込みモードで作成し、適切なファイル権限を適用する。

シグネチャまたは形状

static FILE *create_file_for_extract(const char *filename, mode_t mode)

入力と出力

  • **入力**:
  • filename: 作成するファイルのパス（文字列）。
  • mode: ファイルに適用する権限（mode_t）。
  • **出力**:
  • 成功時: 開かれたファイルポインタ（FILE *）。
  • 失敗時: 戻り値なし（pg_fatal によりプロセスが終了する）。

前提条件と不変条件

  • 指定された filename に書き込み権限があること。
  • pg_fatal が呼び出された場合、呼び出し元へは戻らずプログラムが終了する。

アルゴリズムまたは振る舞い

1. fopen(filename, "wb") を実行し、バイナリ書き込みモードでファイルを開く。 2. ファイル作成に失敗した場合、pg_fatal を呼び出しエラーメッセージを出力して終了する。 3. 非Windows環境の場合、chmod(filename, mode) を実行し、指定された権限を適用する。 4. chmod に失敗した場合、pg_fatal を呼び出しエラーメッセージを出力して終了する。 5. 開いたファイルポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/fe_utils/astreamer_file.c 内の展開処理ロジック（astreamer_extractor_content 等から参照されている）。
  • **呼び出し先**:
  • fopen (標準ライブラリ)
  • pg_fatal (PostgreSQL共通ログ機能)
  • chmod (POSIXシステムコール)

関連データ構造

  • FILE: 標準Cライブラリのファイルストリーム構造体。

失敗モードと境界条件

  • **ファイル作成失敗**: ディレクトリが存在しない、権限不足、またはディスクフル等の理由で fopen が失敗した場合、pg_fatal により致命的エラーとして処理される。
  • **権限設定失敗**: chmod が失敗した場合（非Windows環境）、同様に pg_fatal により終了する。
  • **プラットフォーム依存**: WIN32 環境では chmod がスキップされる。

保守担当者向けメモ

  • 本関数は astreamer フレームワークの展開処理専用のヘルパーです。
  • pg_dir_create_mode 等の定数と組み合わせて使用されることを想定しています。
  • 呼び出し元でファイルポインタのクローズ漏れが発生しないよう注意してください。
