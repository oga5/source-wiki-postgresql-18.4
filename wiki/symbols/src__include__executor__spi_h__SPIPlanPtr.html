SPIPlanPtr

定義

  • 種別: typedef (不透明なポインタ型)
  • 定義箇所: src/include/executor/spi.h:66
  • 実体: struct _SPI_plan *

目的

PostgreSQL の Server Programming Interface (SPI) において、準備済み（プリペアド）SQL プランを指し示すためのハンドルとして機能します。

シグネチャまたは形状

typedef struct _SPI_plan *SPIPlanPtr;

入力と出力

  • **入力**: SPI 関連関数（SPI_execute_plan 等）に対して、実行対象のプランを指定するために渡されます。
  • **出力**: SPI_prepare 等の関数によって生成され、プランの参照として返されます。

前提条件と不変条件

  • このポインタは不透明（opaque）であり、直接の構造体メンバへのアクセスは推奨されません。
  • SPI 接続が確立されているコンテキスト内で使用される必要があります。

アルゴリズムまたは振る舞い

  • SQL の解析および計画作成の結果を保持し、再利用可能な実行単位として管理します。
  • src/backend/executor/spi.c 内の内部関数（_SPI_prepare_plan や _SPI_execute_plan など）において、実行プランの準備や実行制御の引数として利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • contrib/spi/refint.c: 参照整合性の管理においてプランを保持するために使用。
  • src/backend/executor/spi.c: SPI 内部の実行ロジックにおいて、プランの準備や実行の引数として使用。
  • **呼び出し先**:
  • 本シンボル自体はポインタ型であるため、直接の呼び出し先はありません。

関連データ構造

  • SPIExecuteOptions: プラン実行時のオプションを保持する構造体。
  • _SPI_plan: SPIPlanPtr が指す実際の内部構造体（spi_priv.h 等で定義）。

失敗モードと境界条件

  • 無効なプランポインタを SPI 関数に渡した場合の挙動は、SPI のエラーハンドリング機構に依存します。
  • メモリコンテキストの生存期間を超えてプランを保持した場合、ダングリングポインタとなる可能性があります。

保守担当者向けメモ

  • SPIPlanPtr は不透明なポインタとして設計されているため、内部構造を変更する場合は spi_priv.h 側の定義と整合性を取る必要があります。
  • 拡張機能（contrib）から利用される公開インターフェースであるため、型定義の変更は後方互換性に影響を与える可能性があります。
