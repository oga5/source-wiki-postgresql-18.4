generate_series_fctx

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/int8.c:29

目的

generate_series 関数（64ビット整数版）において、複数回の関数呼び出し間で反復処理の状態を保持するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    int64 current;
    int64 finish;
    int64 step;
} generate_series_fctx;

入力と出力

本構造体は関数呼び出しの内部状態を保持するものであり、直接的な入出力関数ではありません。generate_series_step_int8 等の関数内で palloc を用いてメモリ確保され、FuncCallContext を介して管理されます。

前提条件と不変条件

  • current は現在の反復値を示します。
  • finish は反復の終了条件となる値です。
  • step は各ステップで加算される増分値です。

アルゴリズムまたは振る舞い

generate_series_step_int8 関数において、ステートマシンとして機能します。初回呼び出し時に初期化され、後続の呼び出しで current を step ずつ更新しながら、current が finish を超える（または下回る）まで値を生成し続けます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/int8.c および src/backend/utils/adt/int.c 内の generate_series_step_int8 関数から参照・更新されます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数を呼び出すことはありません。

関連データ構造

  • FuncCallContext: PostgreSQLの集合生成関数（SRF）で状態を保持するために使用されるコンテキスト。

失敗モードと境界条件

  • メモリ割り当て失敗: palloc によるメモリ確保時にエラーが発生する可能性があります。
  • 算術オーバーフロー: current の更新時に int64 の範囲を超える可能性があります（実装側で適切なチェックが必要です）。

保守担当者向けメモ

  • 本構造体は int8 版と int4 版（int.c 内）で同様の定義が使用されています。変更を加える際は両方の実装への影響を確認してください。
