bool_alltrue

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/bool.c:382

目的

集約関数（EVERYなど）の最終計算として、蓄積されたブール値の状態に基づき、すべての非NULL値が真であるかを判定する。

シグネチャまたは形状

Datum bool_alltrue(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して BoolAggState 構造体へのポインタを受け取る。
  • **出力**:
  • 非NULL値が存在しない場合、または状態がNULLの場合は NULL を返す。
  • すべての非NULL値が真であれば true を、一つでも偽が含まれていれば false を返す。

前提条件と不変条件

  • 入力引数が BoolAggState 構造体へのポインタであることを前提とする。
  • state->aggcount は処理された非NULL値の総数を保持している必要がある。

アルゴリズムまたは振る舞い

1. PG_ARGISNULL(0) を確認し、NULLであれば NULL を返す。 2. state がNULL、または aggcount が0（非NULL値が一つも存在しない）の場合、PG_RETURN_NULL() を実行する。 3. state->aggtrue（真と判定された値の数）と state->aggcount（非NULL値の総数）を比較する。 4. 両者が一致する場合に true を、一致しない場合に false を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLエンジン（集約関数の最終計算フェーズ）。
  • **呼び出し先**: 供給されたソースコード内には明示的な呼び出し先は記載されていない。

関連データ構造

  • BoolAggState: 集約計算中の状態（aggcount, aggtrue を含む）を保持する構造体。

失敗モードと境界条件

  • 入力がNULLの場合、または集約対象の非NULL値が一つも存在しない場合は NULL を返す。

保守担当者向けメモ

  • 本関数は bool_accum によって更新される BoolAggState に依存している。集約ロジックを変更する際は、bool_accum および bool_accum_inv との整合性に注意すること。
