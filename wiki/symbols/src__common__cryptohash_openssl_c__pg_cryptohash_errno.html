pg_cryptohash_errno

定義

  • 種別: enum
  • 定義箇所: src/common/cryptohash_openssl.c:50

目的

OpenSSLを用いた暗号学的ハッシュ計算処理において発生したエラーの状態を識別するために使用されます。

シグネチャまたは形状

typedef enum pg_cryptohash_errno
{
    PG_CRYPTOHASH_ERROR_NONE = 0,
    PG_CRYPTOHASH_ERROR_DEST_LEN,
    PG_CRYPTOHASH_ERROR_OPENSSL,
} pg_cryptohash_errno;

入力と出力

  • 本シンボルは列挙型であり、関数ではありません。
  • ハッシュ計算処理の結果として、エラー状態を保持する変数（pg_cryptohash_errno error）に代入されます。

前提条件と不変条件

  • PG_CRYPTOHASH_ERROR_NONE はエラーが発生していない正常な状態を示します。
  • OpenSSLライブラリがリンクされている環境でのみ使用されます。

アルゴリズムまたは振る舞い

  • PG_CRYPTOHASH_ERROR_DEST_LEN: 出力先バッファの長さが不十分な場合に発生します。
  • PG_CRYPTOHASH_ERROR_OPENSSL: OpenSSLの内部関数（EVP_MD_CTX関連など）でエラーが発生した場合に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/cryptohash.c および src/common/cryptohash_openssl.c 内のハッシュ計算関数群。
  • 呼び出し先: 本シンボル自体はデータ型であるため、呼び出し先はありません。

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態を保持する構造体。

失敗モードと境界条件

  • 該当なし（本シンボル自体が失敗モードを定義する列挙型です）。

保守担当者向けメモ

  • 新たなエラー種別を追加する場合は、この列挙型を拡張し、対応するエラーメッセージ変換処理を pg_cryptohash_error 等に実装してください。
