NotificationList

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/async.c:389

目的

トランザクションのネストレベルごとに、保留中の通知（NOTIFY）イベントを管理するためのスタック構造を提供します。

シグネチャまたは形状

typedef struct NotificationList
{
	int			nestingLevel;	/* 現在のトランザクションのネスト深さ */
	List	   *events;			/* Notification構造体のリスト */
	HTAB	   *hashtab;		/* NotificationHash構造体のハッシュテーブル、またはNULL */
	struct NotificationList *upper; /* 上位トランザクションレベルのデータへのポインタ */
} NotificationList;

入力と出力

  • 入力: トランザクション内での NOTIFY コマンド実行時に、events リストへ通知情報が追加されます。
  • 出力: トランザクションコミット時に、この構造体が保持する通知リストが共有メモリ上のキューへ書き出されます。

前提条件と不変条件

  • upper ポインタにより、トランザクションのネスト構造を維持します。
  • nestingLevel は現在のトランザクションの深さを反映します。
  • hashtab は重複排除や効率的な検索のために使用される場合があり、不要な場合は NULL となります。

アルゴリズムまたは振る舞い

1. **蓄積**: トランザクション内で NOTIFY が発行されるたびに、対応する NotificationList にイベントが追加されます。 2. **階層管理**: サブトランザクションが開始されると、新しい NotificationList が作成され、既存のリストを upper に保持してスタックとして機能します。 3. **コミット処理**: トランザクション終了時、この構造体を通じて蓄積された通知が処理され、共有メモリ上のキューへ転送されます。

呼び出し元と呼び出し先

  • 呼び出し元: pendingNotifies 静的変数を通じて、トランザクション管理処理から参照されます。
  • 呼び出し先: Notification 構造体（events リスト内）および NotificationHash 構造体（hashtab 内）を管理します。

関連データ構造

  • Notification: 個々の通知メッセージを保持する構造体。
  • NotificationHash: 通知の重複管理を行うためのハッシュテーブル用構造体。
  • List: PostgreSQLの汎用リスト構造体。

失敗モードと境界条件

  • メモリ不足: TopTransactionContext 上で割り当てられるため、メモリ枯渇時にはトランザクションが中断される可能性があります。
  • ネストの深さ: トランザクションのネストが深い場合、スタックの深さに応じてメモリ消費が増加します。

保守担当者向けメモ

  • 本構造体はトランザクションローカルなメモリ（TopTransactionContext）で管理されます。
  • pendingNotifies が NULL の場合、現在のトランザクションには保留中の通知がないことを示します。
  • upper ポインタを辿ることで、サブトランザクションからメイントランザクションまでの通知を一括して追跡可能です。
