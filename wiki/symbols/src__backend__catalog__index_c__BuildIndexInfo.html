BuildIndexInfo

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/index.c:2427

目的

指定されたインデックスリレーションのカタログ情報（pg_index）に基づき、インデックス操作に必要なメタデータ構造体 IndexInfo を構築して返します。

シグネチャまたは形状

IndexInfo *BuildIndexInfo(Relation index)

入力と出力

  • **入力**: index (Relation) - 対象となるインデックスのリレーション記述子。
  • **出力**: IndexInfo* - 構築されたインデックス情報構造体。

前提条件と不変条件

  • index->rd_index が有効な Form_pg_index を保持していること。
  • インデックスのキー数（indnatts）が 1 以上かつ INDEX_MAX_KEYS 以下であること。これに違反する場合、ERROR が発生します。

アルゴリズムまたは振る舞い

1. インデックスの属性数を確認し、範囲外であればエラーをスローします。 2. makeIndexInfo を呼び出し、pg_index の各フィールド（ユニーク性、述語、式、アクセスメソッドの特性など）を反映した IndexInfo を初期化します。 3. pg_index の indkey から属性番号を ii_IndexAttrNumbers 配列へコピーします。 4. インデックスが排他制約（indisexclusion）を持つ場合、RelationGetExclusionInfo を呼び出して排他演算子、プロシージャ、戦略情報を取得し、IndexInfo に格納します。 5. 構築した IndexInfo を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_nbtree.c
  • src/backend/access/brin/brin.c
  • src/backend/access/gin/gininsert.c
  • src/backend/access/nbtree/nbtsort.c
  • src/backend/catalog/index.c（内部の複数箇所）
  • **呼び出し先**:
  • makeIndexInfo
  • RelationGetIndexExpressions
  • RelationGetIndexPredicate
  • RelationGetExclusionInfo

関連データ構造

  • IndexInfo: インデックス操作の実行時に使用されるメタデータ構造体。
  • Form_pg_index: システムカタログ pg_index の行構造。
  • Relation: リレーション記述子。

失敗モードと境界条件

  • indnatts が 1 未満または INDEX_MAX_KEYS を超える場合、elog(ERROR) により処理が中断されます。

保守担当者向けメモ

  • 本関数は BuildDummyIndexInfo と対比されることがあり、こちらは実際のカタログ定義に基づいた情報を生成します。
  • 排他制約情報の取得は indisexclusion フラグに依存するため、カタログの整合性が保たれていることが前提となります。
