tlist_same_collations

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/tlist.c:290

目的

ターゲットリスト（tlist）内の各非ジャンク（non-junk）エントリの照合順序（collation）が、指定された照合順序リスト（colCollations）と一致するかを検証する。

シグネチャまたは形状

bool tlist_same_collations(List *tlist, List *colCollations, bool junkOK)

入力と出力

  • **入力**:
  • tlist: 検証対象の TargetEntry リスト。
  • colCollations: 照合順序（OID）のリスト。
  • junkOK: true の場合、ジャンクエントリ（resjunk）を無視する。false の場合、ジャンクエントリが含まれていると検証失敗となる。
  • **出力**:
  • bool: 全ての非ジャンクエントリの照合順序が colCollations と一致し、かつリストの長さが等しい場合に true を返す。

前提条件と不変条件

  • tlist 内の各要素は TargetEntry 構造体である必要がある。
  • colCollations は OID のリストである必要がある。

アルゴリズムまたは振る舞い

1. tlist を走査し、各 TargetEntry を確認する。 2. resjunk が true の場合:

  • junkOK が false ならば直ちに false を返す。

    • 3. resjunk が false の場合:

  • colCollations が既に末尾に達している（NULL）場合は、tlist の方が長いため false を返す。
  • exprCollation() を用いて式の照合順序を取得し、colCollations の現在の要素（OID）と比較する。一致しなければ false を返す。
  • colCollations のポインタを次へ進める。

    • 4. 走査終了後、colCollations がまだ残っている（NULL ではない）場合は、tlist の方が短いため false を返す。 5. 全ての条件を満たせば true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/prep/prepunion.c:297
  • **呼び出し先**:
  • list_head, lfirst, lnext, lfirst_oid (リスト操作マクロ)
  • exprCollation (式ノードの照合順序取得)

関連データ構造

  • TargetEntry: tlist の各要素。
  • List: PostgreSQL の汎用リスト構造。

失敗モードと境界条件

  • tlist と colCollations の長さが一致しない場合、false を返す。
  • junkOK が false の状態で resjunk が true のエントリに遭遇した場合、false を返す。
  • 照合順序が一致しない場合、false を返す。

保守担当者向けメモ

  • 本関数は tlist の非ジャンクエントリと colCollations を 1 対 1 で対応させるため、リストの順序が重要である。
  • exprCollation は式ノードに対して呼び出されるため、tlist 内の式が正しく型付けされていることが前提となる。
