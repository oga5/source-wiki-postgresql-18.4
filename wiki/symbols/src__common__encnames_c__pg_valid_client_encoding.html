pg_valid_client_encoding

定義

  • 種別: function
  • 定義箇所: src/common/encnames.c:487

目的

指定されたエンコーディング名が、クライアント側で使用可能なエンコーディングとして妥当であるかを検証し、対応する内部 ID を返します。

シグネチャまたは形状

int pg_valid_client_encoding(const char *name)

入力と出力

  • **入力**: name (検証対象のエンコーディング名文字列)
  • **出力**:
  • 妥当な場合: 対応する内部エンコーディング ID (int)
  • 無効な場合: -1

前提条件と不変条件

  • 入力された name は pg_char_to_encoding によって内部 ID に変換可能である必要があります。
  • 変換された ID は、マクロ PG_VALID_FE_ENCODING(enc) によって定義されるクライアント側でサポートされるエンコーディングの範囲内に含まれている必要があります。

アルゴリズムまたは振る舞い

1. pg_char_to_encoding(name) を呼び出し、文字列を内部 ID に変換します。変換に失敗した場合は -1 を返します。 2. 取得した ID が PG_VALID_FE_ENCODING マクロの条件を満たしているか確認します。 3. 条件を満たさない場合は -1 を返し、満たす場合はその ID を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/variable.c:694 にて、設定値の検証に使用されています。
  • **呼び出し先**:
  • pg_char_to_encoding (内部 ID への変換)
  • PG_VALID_FE_ENCODING (マクロによる妥当性判定)

関連データ構造

  • pg_enc: エンコーディングを識別する内部 ID 型。
  • pg_encname_tbl: エンコーディング名と ID のマッピングテーブル。

失敗モードと境界条件

  • name が未知のエンコーディング名である場合、pg_char_to_encoding が負の値を返し、本関数も -1 を返します。
  • name は有効なエンコーディング名であっても、それがクライアント側でサポートされていない（サーバー専用など）場合、PG_VALID_FE_ENCODING のチェックにより -1 が返されます。

保守担当者向けメモ

  • 本関数は src/include/mb/pg_wchar.h で宣言されており、フロントエンドおよびバックエンドの両方から参照可能です。
  • クライアントエンコーディングのサポート範囲を変更する場合は、PG_VALID_FE_ENCODING マクロの定義を確認してください。
