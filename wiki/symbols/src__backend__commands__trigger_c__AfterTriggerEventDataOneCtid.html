AfterTriggerEventDataOneCtid

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/trigger.c:3731

目的

AFTER トリガーのイベントデータのうち、単一のタプル（ItemPointer）を保持する必要があるイベントを格納するためのデータ構造です。

シグネチャまたは形状

typedef struct AfterTriggerEventDataOneCtid
{
	TriggerFlags ate_flags;		/* status bits and offset to shared data */
	ItemPointerData ate_ctid1;	/* inserted, deleted, or old updated tuple */
} AfterTriggerEventDataOneCtid;

入力と出力

  • **入力**: TriggerFlags（ステータスビットおよび共有データへのオフセット）および ItemPointerData（対象タプルの物理位置）。
  • **出力**: なし（構造体としてイベントキューに格納される）。

前提条件と不変条件

  • ate_ctid1 は、挿入、削除、または更新されたタプルの物理位置（ItemPointer）を指す必要があります。
  • ate_flags は、トリガー実行エンジンがイベントを正しく識別・処理するためのフラグを含んでいる必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、AFTER トリガーのイベントキューにおいて、単一のタプル識別子を必要とするイベントのメモリレイアウトを定義します。
  • sizeof(AfterTriggerEventDataOneCtid) を用いて、イベントキューのメモリ確保やオフセット計算が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/trigger.c 内のイベント保存処理（AfterTriggerSaveEvent 等）において、イベントのサイズ計算やメモリ配置の際に参照されます。
  • **呼び出し先**: 構造体定義であるため、直接的な関数呼び出し先はありません。

関連データ構造

  • TriggerFlags: イベントの状態管理に使用。
  • ItemPointerData: タプルの物理位置を保持。
  • AfterTriggerEventData: 関連するイベントデータ構造の総称。
  • AfterTriggerEventDataNoOids, AfterTriggerEventDataZeroCtids: 他のイベントデータバリエーション。

失敗モードと境界条件

  • メモリ配置時に sizeof の計算が誤っている場合、イベントキューの読み取り時に不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本構造体は、AfterTriggerEventData のバリエーションの一つです。イベントデータのサイズ最適化のために個別に定義されています。変更を加える際は、他の AfterTriggerEventData* 構造体との整合性および trigger.c 内のサイズ計算ロジックを確認してください。
