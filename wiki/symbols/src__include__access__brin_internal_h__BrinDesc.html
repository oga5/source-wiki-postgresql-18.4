BrinDesc

定義

  • 種別: struct
  • 定義箇所: src/include/access/brin_internal.h:44

目的

BRIN（Block Range Index）インデックスの構築、スキャン、およびメンテナンス時に、インデックスリレーションのメタデータとディスク上のタプルをメモリ上で扱うための記述子として機能します。

シグネチャまたは形状

typedef struct BrinDesc
{
    MemoryContext bd_context;
    Relation      bd_index;
    TupleDesc     bd_tupdesc;
    TupleDesc     bd_disktdesc;
    int           bd_totalstored;
    BrinOpcInfo  *bd_info[FLEXIBLE_ARRAY_MEMBER];
} BrinDesc;

入力と出力

  • **入力**: brin_build_desc 関数を介して初期化され、インデックスリレーションのメタデータがロードされます。
  • **出力**: インデックス操作（brininsert, bringetbitmap 等）において、ディスク上のタプルとメモリ上の Datum 表現を相互変換するための情報を提供します。

前提条件と不変条件

  • bd_info は bd_tupdesc->natts 分の要素を持つ可変長配列として扱われます。
  • bd_disktdesc は初回使用時に生成されるキャッシュです。

アルゴリズムまたは振る舞い

1. インデックス操作の開始時に brin_build_desc でメモリ上に構築されます。 2. インデックス内の各カラムに対する演算子クラス固有の情報（BrinOpcInfo）を保持し、型キャッシュ管理（utils/typcache.h）と連携してデータ変換を行います。 3. brininsert や bringetbitmap などの関数が、この構造体を参照して物理的なインデックスページと論理的なデータ表現を橋渡しします。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/brin/brin.c 内の brin_build_desc 等の関数、および contrib/pageinspect/brinfuncs.c から参照されています。
  • **呼び出し先**: 構造体内部で BrinOpcInfo を保持しており、演算子クラスのプロシージャ（BRIN_PROCNUM_ADDVALUE 等）を呼び出す際にその情報が利用されます。

関連データ構造

  • BrinOpcInfo: 各演算子クラス固有の情報を保持する構造体。
  • Relation: インデックスリレーション本体。
  • TupleDesc: タプルの記述子。

失敗モードと境界条件

  • bd_info の要素数はインデックスの属性数に依存するため、メモリ確保時に offsetof(BrinDesc, bd_info) を用いた計算が必要です。
  • 演算子クラスが必須プロシージャを実装していない場合、インデックス操作時に予期せぬ動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は BRIN インデックスの内部実装用であり、コア開発者以外による直接的な操作は想定されていません。
  • 新しい演算子クラスを追加する際は、BRIN_MANDATORY_NPROCS に基づくプロシージャ番号との整合性を確認してください。
