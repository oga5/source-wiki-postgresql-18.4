hemdistsign

定義

  • 種別: function
  • 定義箇所: contrib/ltree/_ltree_gist.c:186

目的

2つのビットベクトル（シグネチャ）間のハミング距離を計算する。

シグネチャまたは形状

static int hemdistsign(BITVECP a, BITVECP b, int siglen)

入力と出力

  • **入力**:
  • a, b: 比較対象のビットベクトルへのポインタ (BITVECP)。
  • siglen: ビットベクトルの長さ（バイト単位）。
  • **出力**:
  • int: 2つのビットベクトル間で異なるビットの総数（ハミング距離）。

前提条件と不変条件

  • a および b は、少なくとも siglen バイト以上の有効なメモリ領域を指している必要がある。
  • pg_number_of_ones 配列がシステム内で利用可能であること。

アルゴリズムまたは振る舞い

1. dist を 0 に初期化する。 2. siglen バイト分ループし、各バイトごとに a[i] と b[i] の排他的論理和（XOR）をとる。 3. XORの結果（diff）をインデックスとして pg_number_of_ones 配列を参照し、セットされているビット数を dist に加算する。 4. 全バイトの走査終了後、累積した dist を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_gist.c
  • contrib/intarray/_intbig_gist.c
  • contrib/ltree/_ltree_gist.c
  • contrib/pg_trgm/trgm_gist.c
  • **呼び出し先**:
  • pg_number_of_ones（グローバルなルックアップテーブル）

関連データ構造

  • BITVECP: ビットベクトルを指すポインタ型。

失敗モードと境界条件

  • siglen が負または不正な値である場合、メモリ境界外アクセスが発生する可能性がある。
  • a または b が NULL の場合の挙動は未定義。

保守担当者向けメモ

  • 本関数は複数の contrib モジュール（hstore, intarray, ltree, pg_trgm）で共通して利用されている。変更時はこれらのモジュールへの影響を考慮すること。
  • pg_number_of_ones を使用したルックアップは、現代的な CPU の popcount 命令よりも効率的であると想定されているが、環境によっては再評価の余地がある。
