gbt_bytea_penalty

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bytea.c:151

目的

GiSTインデックスにおいて、新しいキーを挿入する際にインデックスツリーのどのノードを選択するのが最適か（ペナルティが最小か）を計算します。

シグネチャまたは形状

Datum gbt_bytea_penalty(PG_FUNCTION_ARGS)

入力と出力

  • 入力:
  • PG_GETARG_POINTER(0): GISTENTRY *o（既存のインデックスエントリ）
  • PG_GETARG_POINTER(1): GISTENTRY *n（挿入対象の新しいエントリ）
  • PG_GETARG_POINTER(2): float *result（計算されたペナルティ値を格納する出力先）
  • 出力: PG_RETURN_POINTER を介して result を返します。

前提条件と不変条件

  • tinfo 構造体が bytea 型の比較ロジックを保持している必要があります。
  • fcinfo->flinfo が関数呼び出し情報として有効である必要があります。

アルゴリズムまたは振る舞い

1. 引数から既存エントリ o、新規エントリ n、および結果格納用ポインタ result を取得します。 2. gbt_var_penalty 関数を呼び出し、bytea 型固有の比較設定 tinfo を渡してペナルティ値を算出します。 3. 計算結果を result に格納し、そのポインタを返します。

呼び出し元と呼び出し先

  • 呼び出し元: PostgreSQLのGiSTインデックス管理サブシステム（直接的な呼び出し元コードはソース抜粋に含まれていません）。
  • 呼び出し先: gbt_var_penalty（btree_utils_var.c で定義された汎用関数）。

関連データ構造

  • GISTENTRY: GiSTインデックスのエントリ構造体。
  • tinfo: gbtree_vinfo 構造体のインスタンス。bytea 型の比較関数や型情報を保持します。

失敗モードと境界条件

  • 入力ポインタが NULL の場合、PostgreSQLの標準的なメモリ保護機構によりエラーが発生する可能性があります。
  • tinfo が正しく初期化されていない場合、誤ったペナルティ値が算出される可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_var.c の汎用ロジックをラップするアダプターです。bytea 型特有の比較ロジックを変更する場合は、tinfo の定義を確認してください。
