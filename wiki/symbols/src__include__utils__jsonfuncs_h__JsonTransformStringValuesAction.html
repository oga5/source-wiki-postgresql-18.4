JsonTransformStringValuesAction

定義

  • 種別: typedef
  • 定義箇所: src/include/utils/jsonfuncs.h:38

目的

JSON または JSONB データの反復処理中に、文字列型の値に対して変換処理を適用するためのコールバック関数型を定義する。

シグネチャまたは形状

typedef text *(*JsonTransformStringValuesAction) (void *state, char *elem_value, int elem_len);
  • state: 変換処理の状態を保持するポインタ。
  • elem_value: 対象となる文字列値の先頭ポインタ。
  • elem_len: 文字列値の長さ。
  • 戻り値: 変換後の text 型データへのポインタ。

入力と出力

  • 入力: JSON/JSONB 内の文字列要素（char * および長さ）。
  • 出力: 変換結果としての text * 型。

前提条件と不変条件

  • 呼び出し元は、反復処理のコンテキストにおいて本関数をコールバックとして渡す必要がある。
  • 戻り値の text 型データは、PostgreSQL のメモリ管理規則（palloc 等）に従って割り当てられている必要がある。

アルゴリズムまたは振る舞い

1. JSON/JSONB の走査中、文字列型の値が見つかった際に本関数が呼び出される。 2. state を通じて渡された情報を利用し、elem_value に対して特定の変換（エスケープ処理や型変換など）を行う。 3. 変換後の値を text 型として返し、反復処理の呼び出し元へ戻す。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/tsearch/wparser.c (headline_json_value 経由)
  • src/backend/utils/adt/jsonfuncs.c (JSON/JSONB 反復処理関数内)
  • 呼び出し先:
  • 提供されたソース excerpt には、本関数が呼び出す具体的な関数は記載されていない。

関連データ構造

  • text: PostgreSQL の可変長文字列型。
  • JsonIterateStringValuesAction: 類似の反復処理用コールバック型。

失敗モードと境界条件

  • elem_value が NULL の場合や、elem_len が不正な場合の挙動は、実装側のコールバック関数に依存する。
  • メモリ不足時には、PostgreSQL のエラーハンドリング機構により例外が発生する可能性がある。

保守担当者向けメモ

  • 本関数は jsonfuncs.c 内の反復処理関数（iterate_json_values 等）で利用される。
  • 新しい変換ロジックを追加する場合は、このシグネチャに適合する関数を定義し、action_state 引数として渡すこと。
