PgAioWorkerSubmissionQueue

定義

  • 種別: struct
  • 定義箇所: src/backend/storage/aio/method_worker.c:55

目的

クライアントプロセスとIOワーカープロセス間で共有される、I/Oリクエスト投入用のリングバッファ構造体です。

シグネチャまたは形状

typedef struct PgAioWorkerSubmissionQueue
{
    uint32      size;
    uint32      mask;
    uint32      head;
    uint32      tail;
    int         sqes[FLEXIBLE_ARRAY_MEMBER];
} PgAioWorkerSubmissionQueue;

入力と出力

  • **入力**: クライアントプロセスが pgaio_worker_submission_queue_insert を介してI/O IDをキューに挿入します。
  • **出力**: IOワーカープロセスが pgaio_worker_submission_queue_consume を介してI/O IDを取り出し、実行します。

前提条件と不変条件

  • sqes は可変長配列として定義されており、共有メモリ上に配置される際は size に応じた領域が確保されます。
  • キューへのアクセスは AioWorkerSubmissionQueueLock による排他制御が必要です。

アルゴリズムまたは振る舞い

  • リングバッファとして動作し、head と tail のポインタを用いてI/Oリクエストの追跡を行います。
  • mask を用いて、インデックスのラップアラウンドを効率的に計算します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/aio/method_worker.c 内のI/O投入および消費関数（pgaio_worker_submission_queue_insert, pgaio_worker_submission_queue_consume 等）から参照されます。
  • **呼び出し先**: 構造体定義のため、特定の関数を呼び出すことはありません。

関連データ構造

  • PgAioWorkerControl: 全ワーカーの稼働状況を管理する構造体。
  • AioWorkerSubmissionQueueLock: 本キューへのアクセスを保護するロック。

失敗モードと境界条件

  • キューが満杯の場合の挙動については、本構造体の定義からは直接読み取れません。
  • メモリ割り当てサイズが size と不整合を起こした場合、バッファオーバーフローのリスクがあります。

保守担当者向けメモ

  • offsetof(PgAioWorkerSubmissionQueue, sqes) を使用して、共有メモリ確保時のサイズ計算が行われています。
  • FLEXIBLE_ARRAY_MEMBER を使用しているため、構造体自体のサイズではなく、動的に確保された領域のサイズを考慮する必要があります。
