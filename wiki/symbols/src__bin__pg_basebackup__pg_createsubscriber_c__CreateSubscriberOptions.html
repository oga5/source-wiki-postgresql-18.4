CreateSubscriberOptions

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/pg_createsubscriber.c:35

目的

pg_createsubscriber ユーティリティにおいて、コマンドライン引数や実行時の設定情報を一元管理し、各処理関数へ受け渡すためのデータ構造です。

シグネチャまたは形状

struct CreateSubscriberOptions
{
    char       *config_file;
    char       *pub_conninfo_str;
    char       *socket_dir;
    char       *sub_port;
    const char *sub_username;
    bool        two_phase;
    SimpleStringList database_names;
    SimpleStringList pub_names;
    SimpleStringList sub_names;
    SimpleStringList replslot_names;
    int         recovery_timeout;
    bool        all_dbs;
    SimpleStringList objecttypes_to_clean;
};

入力と出力

本構造体は、主にコマンドライン引数の解析結果を保持する入力コンテナとして機能します。関数呼び出しの引数として渡される際は、読み取り専用（const）として参照されることが一般的です。

前提条件と不変条件

  • SimpleStringList 型のメンバは、適切に初期化および解放される必要があります。
  • 接続情報やポート番号などの文字列メンバは、実行環境のメモリ上に有効な文字列として保持されている必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持に特化しており、特定のアルゴリズムは実装していません。プログラム実行の初期段階で引数解析によって値が設定され、以降の検証、パブリッシャー準備、サブスクライバー昇格、クリーンアップといった各フェーズで参照されます。

呼び出し元と呼び出し先

  • **呼び出し元**: main 関数（引数解析後）
  • **呼び出し先（参照元）**:
  • get_sub_conninfo
  • store_pub_sub_info
  • modify_subscriber_sysid
  • start_standby_server
  • get_publisher_databases
  • wait_for_end_recovery

関連データ構造

  • SimpleStringList: データベース名、パブリケーション名、サブスクリプション名、レプリケーションスロット名、クリーンアップ対象オブジェクトのリスト管理に使用。
  • LogicalRepInfo: 本構造体で保持する情報を元に構築される、データベースごとの詳細なレプリケーション情報構造体。

失敗モードと境界条件

  • 構造体内の文字列ポインタが NULL の場合、参照先の関数でセグメンテーション違反が発生する可能性があります。
  • recovery_timeout が不適切な値（負の値など）に設定された場合、リカバリ待機処理の挙動に影響を与える可能性があります。

保守担当者向けメモ

  • 本構造体に新しい設定項目を追加した場合は、引数解析ロジックおよび cleanup_objects_atexit 等のクリーンアップ処理への影響を確認してください。
  • SimpleStringList を使用しているメンバについては、メモリリークを防ぐため、プログラム終了時に適切に解放されていることを確認してください。
