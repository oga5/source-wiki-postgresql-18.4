BitmapHeapRecheck

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeBitmapHeapscan.c:192

目的

ビットマップヒープスキャンにおいて、損失のある（lossy）ビットマップから取得されたタプルが、本来のクエリ条件を満たしているかを再検証（recheck）する。

シグネチャまたは形状

bool BitmapHeapRecheck(BitmapHeapScanState *node, TupleTableSlot *slot)

入力と出力

  • **入力**:
  • node: ビットマップヒープスキャンの実行状態を保持する BitmapHeapScanState 構造体。
  • slot: 再検証対象となるタプルを格納した TupleTableSlot。
  • **出力**:
  • bool: タプルが条件を満たす場合は true、満たさない場合は false を返す。

前提条件と不変条件

  • node->ss.ps.ps_ExprContext が初期化されていること。
  • node->bitmapqualorig に、再検証に使用する元のクエリ条件式が保持されていること。

アルゴリズムまたは振る舞い

1. node から式コンテキスト（ExprContext）を取得する。 2. 式コンテキストの ecxt_scantuple に、検証対象の slot をセットする。 3. ExecQualAndReset を呼び出し、node->bitmapqualorig に格納された条件式を評価する。 4. 評価結果をそのまま呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は src/backend/executor/nodeBitmapHeapscan.c:218 であり、ExecScanRecheckMtd の一部として登録されている。
  • **呼び出し先**: ExecQualAndReset を呼び出す。

関連データ構造

  • BitmapHeapScanState: ビットマップヒープスキャンの実行状態を管理する構造体。
  • TupleTableSlot: タプルデータを保持するスロット。
  • ExprContext: 式評価のためのコンテキスト。

失敗モードと境界条件

  • node または slot が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • node->bitmapqualorig が適切に設定されていない場合、誤った評価結果を返す可能性がある。

保守担当者向けメモ

  • 本関数は EvalPlanQual におけるアクセスメソッドルーチンとして使用される。
  • 損失のあるビットマップスキャンにおいて、ヒープから読み取ったタプルがインデックス条件を本当に満たしているかを確認する重要なステップである。
