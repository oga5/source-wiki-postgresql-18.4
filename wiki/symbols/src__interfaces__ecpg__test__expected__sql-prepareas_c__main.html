目的

ECPG（Embedded C for PostgreSQL）における PREPARE および EXECUTE 文の動作検証を行うための回帰テストのエントリポイントです。特に、プリペアドステートメントの識別子に変数や特殊文字が含まれる場合の挙動を検証します。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（プログラム引数は使用しません）。
  • 出力: 成功時は 0 を返し、テストの進捗状況を標準出力に書き出します。エラー発生時は sqlprint() を介して標準エラー出力に詳細が出力されます。

前提条件と不変条件

  • データベース ecpg1_regression への接続が成功している必要があります。
  • テスト実行前にテーブル test が作成され、各テストケースの前後で truncate されることで状態がリセットされます。

アルゴリズムまたは振る舞い

1. **初期化**: データベースへ接続し、テスト用テーブル test を作成します。 2. **テスト実行**: 以下のパターンを順次実行します。

  • 通常の INSERT 文の実行。
  • EXECUTE IMMEDIATE による即時実行。
  • PREPARE 文によるステートメントの準備と、その後の EXECUTE による実行。
  • 識別子に変数や特殊文字（スペース、ダブルクォート）を用いた PREPARE の検証。

    • 3. **検証**: 各実行後に check_result_of_insert() を呼び出し、期待値と一致するかを確認します。 4. **終了処理**: トランザクションのコミットとリソースの解放を行います。

呼び出し元と呼び出し先

  • 呼び出し元: OS（プログラムのエントリポイント）。
  • 呼び出し先: ECPGconnect, ECPGtrans, ECPGdo, ECPGprepare, check_result_of_insert, sqlprint, printf, strcpy。

関連データ構造

  • sqlca: SQL通信領域。各SQL操作後のエラーコード（sqlca.sqlcode）の確認に使用されます。
  • ivar1, ivar2: テスト用データとして使用される整数変数。
  • v_include_dq_name, v_include_ws_name, v_normal_name, v_query: プリペアドステートメントの識別子やクエリ文字列を保持する文字配列。

失敗モードと境界条件

  • sqlca.sqlcode < 0 となった場合、sqlprint() が呼び出され、エラー内容が標準エラー出力に記録されます。
  • 識別子に特殊文字（スペースやダブルクォート）が含まれる場合、ECPGプリプロセッサが正しくCコードへ変換できているかを検証する境界条件として機能します。
