UnlockRelationOid

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/lmgr.c:228

目的

指定されたリレーション（OID）に対して保持しているロックを解放します。

シグネチャまたは形状

void UnlockRelationOid(Oid relid, LOCKMODE lockmode)

入力と出力

  • **入力**:
  • relid (Oid): 解放対象のリレーション識別子。
  • lockmode (LOCKMODE): 解放するロックのモード。
  • **出力**: なし（void）。

前提条件と不変条件

  • SetLocktagRelationOid を用いて、リレーションOIDからロックタグ（LOCKTAG）が正しく構築可能であること。
  • 呼び出し側は、指定した relid と lockmode の組み合わせで既にロックを保持している必要があります。

アルゴリズムまたは振る舞い

1. SetLocktagRelationOid を呼び出し、引数の relid から LOCKTAG 構造体を初期化します。 2. LockRelease を呼び出し、構築したタグと指定されたロックモードを使用して、共有メモリ上のロックテーブルから該当するロックを解放します。第3引数には false が渡され、ロックの解放処理が実行されます。

関連データ構造

  • LOCKTAG: ロック対象を識別するための構造体。
  • LOCKMODE: ロックの強度や種類を定義する列挙型。

失敗モードと境界条件

  • 保持していないロックを解放しようとした場合、LockRelease の内部ロジックによりエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は LockRelationOid の対となる操作です。
  • LockRelease に渡される第3引数が false であるため、この関数は通常のロック解放に使用されます。特殊なロック管理（FastPath等）の挙動については LockRelease の実装を参照してください。
