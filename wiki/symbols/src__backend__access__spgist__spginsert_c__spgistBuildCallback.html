spgistBuildCallback

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spginsert.c:40

目的

SP-GiSTインデックス構築時、ヒープスキャンから渡された各タプルをインデックスへ挿入するためのコールバック関数です。

シグネチャまたは形状

static void spgistBuildCallback(Relation index, ItemPointer tid, Datum *values, bool *isnull, bool tupleIsAlive, void *state)

入力と出力

  • **入力**:
  • index: 対象のインデックスリレーション。
  • tid: 挿入対象タプルの物理位置。
  • values, isnull: タプルの属性値とNULLフラグ。
  • state: SpGistBuildState 構造体へのポインタ（void *として渡される）。
  • **出力**: なし（副作用としてインデックスへの挿入と状態更新を行う）。

前提条件と不変条件

  • state 引数は SpGistBuildState 型としてキャスト可能であること。
  • 呼び出し元は table_index_build_scan 等のヒープスキャン処理であること。

アルゴリズムまたは振る舞い

1. buildstate->tmpCtx を現在のメモリコンテキストとして設定し、一時的なメモリ消費を制御します。 2. spgdoinsert を呼び出し、タプルをインデックスに挿入します。 3. spgdoinsert が false を返した場合（バッファロックの競合等）、tmpCtx をリセットして成功するまで再試行します。 4. 挿入成功後、buildstate->indtuples をインクリメントして総タプル数を更新します。 5. 最後にメモリコンテキストを復元し、tmpCtx をリセットして後続のタプル処理に備えます。

呼び出し元と呼び出し先

  • **呼び出し元**: table_index_build_scan（spgbuild 内から間接的に呼び出される）。
  • **呼び出し先**: spgdoinsert。

関連データ構造

  • SpGistBuildState: インデックス構築中の状態（一時メモリコンテキスト、インデックス状態、タプルカウント）を保持する構造体。

失敗モードと境界条件

  • spgdoinsert がバッファロックの競合等で失敗した場合、ループ内でリトライ処理が行われます。
  • メモリコンテキストの切り替えとリセットにより、各タプル処理ごとのメモリリークを防止しています。

保守担当者向けメモ

  • 本関数はインデックス構築のループ内で呼び出されるため、パフォーマンスが重要です。
  • spgdoinsert の戻り値が false の場合のリトライロジックは、並行処理による競合を回避するための重要なガードです。
