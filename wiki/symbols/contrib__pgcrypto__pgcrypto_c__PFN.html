PFN

定義

  • 種別: typedef
  • 定義箇所: contrib/pgcrypto/pgcrypto.c:58

目的

暗号化アルゴリズムやハッシュ関数などのプロバイダーを動的に検索・初期化するための関数ポインタ型です。

シグネチャまたは形状

typedef int (*PFN) (const char *name, void **res);

入力と出力

  • **入力**:
  • name: 検索対象のアルゴリズム名（文字列）。
  • res: 初期化されたプロバイダーのコンテキストを格納するためのポインタのポインタ。
  • **出力**:
  • 成功時に 0、失敗時に非ゼロの整数値を返します。

前提条件と不変条件

  • res は有効なメモリ領域を指している必要があります。
  • name は find_provider を通じて小文字化された文字列が渡されることが想定されています。

アルゴリズムまたは振る舞い

この型は、find_provider 関数が特定の暗号化プロバイダー（px_find_digest, px_find_hmac, px_find_combo など）を呼び出す際のインターフェースとして機能します。呼び出された関数は、指定された name に基づいて適切なプロバイダーを検索し、その結果を res に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**: find_provider 関数がこの型を使用して、動的にプロバイダー検索関数を呼び出します。
  • **呼び出し先**: px_find_digest, px_find_hmac, px_find_combo がこの型に適合する関数として渡されます。

関連データ構造

  • PX_MD, PX_HMAC, PX_Combo（px.h で定義される暗号化コンテキスト構造体）が res を通じて返されます。

失敗モードと境界条件

  • 指定された name に対応するプロバイダーが見つからない場合、呼び出し先の関数は非ゼロの値を返し、res は設定されません。

保守担当者向けメモ

  • 本型は pgcrypto.c 内の抽象化レイヤーの接着剤です。新しい暗号化アルゴリズムやプロバイダーを追加する際は、このシグネチャに適合する関数を実装する必要があります。
