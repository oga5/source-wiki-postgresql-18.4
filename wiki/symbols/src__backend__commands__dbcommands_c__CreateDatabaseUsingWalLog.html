CreateDatabaseUsingWalLog

定義

  • 種別: function
  • 定義箇所: src/backend/commands/dbcommands.c:147

目的

テンプレートデータベースから新しいデータベースへ、WALログベースの戦略を用いて物理的なデータとリレーション情報を複製する。

シグネチャまたは形状

static void CreateDatabaseUsingWalLog(Oid src_dboid, Oid dst_dboid, Oid src_tsid, Oid dst_tsid)

入力と出力

  • **入力**:
  • src_dboid: テンプレートとなるソースデータベースのOID。
  • dst_dboid: 新規作成するターゲットデータベースのOID。
  • src_tsid: ソースデータベースのテーブルスペースOID。
  • dst_tsid: ターゲットデータベースのテーブルスペースOID。
  • **出力**: なし（副作用として物理ディレクトリの作成とデータコピーを実行）。

前提条件と不変条件

  • 呼び出し時点で、ソースおよびターゲットデータベースに対する適切なロックが取得されていること。
  • src_dboid に対応するデータベースディレクトリが存在すること。

アルゴリズムまたは振る舞い

1. GetDatabasePath を用いてソースおよびターゲットのディレクトリパスを取得する。 2. CreateDirAndVersionFile を呼び出し、ターゲットディレクトリの作成と PG_VERSION ファイルの書き込みを行う。 3. RelationMapCopy を実行し、リレーションマップファイルをソースからターゲットへコピーする。 4. ScanSourceDatabasePgClass を実行し、コピー対象となるリレーションのリストを取得する。 5. 取得したリストを反復処理し、各リレーションに対して以下を実行する:

  • ターゲットのテーブルスペースを決定する（ソースと同一か、デフォルトテーブルスペースか）。
  • LockRelationId を使用してソースとターゲットの両リレーションに AccessShareLock を取得する。
  • CreateAndCopyRelationData を呼び出し、ストレージデータをコピーする。
  • UnlockRelationId でロックを解放する。

    • 6. 最後に、割り当てたメモリ（パス文字列、リスト）を解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dbcommands.c 内のデータベース作成ロジックから呼び出される。
  • **呼び出し先**: GetDatabasePath, CreateDirAndVersionFile, RelationMapCopy, ScanSourceDatabasePgClass, LockRelationId, CreateAndCopyRelationData, UnlockRelationId 等。

関連データ構造

  • Oid: データベースおよびテーブルスペースの識別子。
  • RelFileLocator: リレーションの物理的な位置情報。
  • CreateDBRelInfo: コピー対象リレーションの情報を保持する構造体。
  • LockRelId: ロック対象リレーションの識別子。

失敗モードと境界条件

  • ディレクトリ作成やファイルコピーの失敗時には、PostgreSQLの標準的なエラー処理メカニズムによりトランザクションが中断される。
  • 物理的な残骸が残らないよう、呼び出し元で PG_ENSURE_ERROR_CLEANUP 等のクリーンアップ機構が併用されることを前提としている。

保守担当者向けメモ

  • 本関数は非トランザクション的なファイル操作を含むため、エラー発生時の整合性確保には細心の注意が必要である。
  • AccessShareLock を取得する保守的な実装となっているが、データベース全体がロックされている状況下では、リレーション単位のロックが不要である可能性についても検討の余地がある。
