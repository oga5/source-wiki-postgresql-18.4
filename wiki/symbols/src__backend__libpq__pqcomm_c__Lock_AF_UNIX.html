Lock_AF_UNIX

定義

  • 種別: function
  • 定義箇所: src/backend/libpq/pqcomm.c:684

目的

Unixドメインソケットの作成に先立ち、排他制御用のロックファイルを作成し、既存のソケットファイルを安全に削除することで、bind() 呼び出し時の競合や失敗を防止する。

シグネチャまたは形状

static int Lock_AF_UNIX(const char *unixSocketDir, const char *unixSocketPath)

入力と出力

  • **入力**:
  • unixSocketDir: ソケットディレクトリのパス。
  • unixSocketPath: ソケットファイルのパス。
  • **出力**:
  • STATUS_OK: 処理成功。

前提条件と不変条件

  • unixSocketPath が抽象名前空間（@ で始まる）を示す場合、ロックファイルは作成されず即座に STATUS_OK を返す。
  • ロックファイルはソケットファイルそのものではなく、関連付けられた別ファイルとして作成される。

アルゴリズムまたは振る舞い

1. unixSocketPath が @ で始まるか確認し、該当する場合は早期リターンする。 2. CreateSocketLockFile を呼び出し、unixSocketPath に対応するロックファイルを生成する。 3. unlink() を呼び出し、既存のソケットファイルを削除する。 4. sock_paths リストに unixSocketPath を追加し、後続のメンテナンスのためにパスを保持する。 5. STATUS_OK を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/libpq/pqcomm.c 内の関数（ListenServerPort からの呼び出しが確認されている）。
  • **呼び出し先**: CreateSocketLockFile, unlink, lappend, pstrdup。

関連データ構造

  • sock_paths: ソケットファイルのパスを保持するグローバルなリスト（lappend で使用）。

失敗モードと境界条件

  • CreateSocketLockFile が失敗した場合、その内部でエラー処理が行われる（本関数は戻り値として STATUS_OK を返す設計となっている）。
  • 抽象ソケット（@ プレフィックス）の場合、ロックファイル作成および unlink はスキップされる。

保守担当者向けメモ

  • 本関数は bind() 実行前の準備として呼び出される。
  • ソケットファイルそのものではなくロックファイルを使用することで、移植性を高め、競合状態を回避している。
