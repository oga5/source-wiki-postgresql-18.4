smgrunpin

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/smgr.c:310

目的

SMgrRelation オブジェクトのピン留めカウントを減らし、カウントがゼロになった場合に当該オブジェクトをトランザクション終了時の破棄対象リストへ追加する。

シグネチャまたは形状

void smgrunpin(SMgrRelation reln)

入力と出力

  • **入力**: reln (SMgrRelation) - 操作対象のストレージマネージャリレーションハンドル。
  • **出力**: なし（副作用として reln の内部状態およびグローバルな破棄リストを更新）。

前提条件と不変条件

  • reln->pincount > 0 であること（アサーションにより検証）。

アルゴリズムまたは振る舞い

1. 対象 reln の pincount を 1 減算する。 2. 減算後の pincount が 0 になった場合、当該 reln をグローバルな unpinned_relns リストの末尾に追加する。これにより、トランザクション終了時に AtEOXact_SMgr を通じて破棄可能となる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/utils/rel.h (リレーションキャッシュ管理に関連する箇所)
  • **呼び出し先**:
  • dlist_push_tail (リスト操作用ユーティリティ)

関連データ構造

  • SMgrRelation: ストレージマネージャが保持するリレーションの物理ハンドル。pincount フィールドを保持する。
  • unpinned_relns: ピン留めが解除され、破棄待ち状態にある SMgrRelation を管理する双方向リスト。

失敗モードと境界条件

  • pincount が 0 の状態で呼び出された場合、アサーション違反によりプロセスが異常終了する。

保守担当者向けメモ

  • 本関数は、リレーションが不要になった際に呼び出される。ピン留めされていないオブジェクトは、トランザクション境界で自動的にクリーンアップされるため、メモリリークを防ぐために適切なペアリング（smgrpin との対）が重要である。
