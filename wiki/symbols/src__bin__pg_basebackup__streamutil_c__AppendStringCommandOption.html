AppendStringCommandOption

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/streamutil.c:766

目的

レプリケーションコマンドのオプション文字列を構築し、指定されたバッファに追加します。文字列値を持つオプションを安全にエスケープして付与するために使用されます。

シグネチャまたは形状

void AppendStringCommandOption(PQExpBuffer buf, bool use_new_option_syntax,
                               char *option_name, char *option_value)

入力と出力

  • buf: 構築中のコマンド文字列を保持する PQExpBuffer。
  • use_new_option_syntax: PostgreSQL 15以降の括弧付きオプション構文を使用するかどうかを指定するフラグ。
  • option_name: オプション名。
  • option_value: オプションに設定する文字列値（NULL許容）。
  • 出力: なし（buf の内容が更新されます）。

前提条件と不変条件

  • グローバル変数 conn が有効なデータベース接続を保持している必要があります（PQescapeStringConn 内部で使用されるため）。
  • option_value が NULL の場合、値の付与はスキップされます。

アルゴリズムまたは振る舞い

1. AppendPlainCommandOption を呼び出し、option_name を buf に追加します。 2. option_value が NULL でない場合：

  • option_value の長さを取得します。
  • palloc を使用して、エスケープ後の文字列を格納するためのメモリを確保します。
  • PQescapeStringConn を使用して、option_value を現在の接続の文字セットに合わせてエスケープします。
  • エスケープされた値を '...' で囲み、buf に追加します。
  • pfree を使用して、確保したメモリを解放します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_basebackup.c および src/bin/pg_basebackup/streamutil.c 内の複数箇所。
  • 呼び出し先: AppendPlainCommandOption、PQescapeStringConn、appendPQExpBuffer、palloc、pfree。

関連データ構造

  • PQExpBuffer: コマンド文字列を動的に構築するための構造体。

失敗モードと境界条件

  • option_value が NULL の場合、値の追加は行われません。
  • メモリ不足時には palloc がエラーを発生させる可能性があります。
  • conn が未初期化または無効な場合、PQescapeStringConn の呼び出しで失敗する可能性があります。

保守担当者向けメモ

  • 本関数はフロントエンド用のユーティリティであり、バックエンドの関数を直接呼び出すことはできません。
  • 新しいレプリケーションコマンドを追加する際は、use_new_option_syntax に基づく構文の差異を考慮してください。
