dbase_identify

定義

  • 種別: function
  • 定義箇所: src/backend/access/rmgrdesc/dbasedesc.c:56

目的

データベース操作に関連する WAL（Write-Ahead Log）レコードの info フィールドを解析し、操作の種類を識別する文字列を返します。

シグネチャまたは形状

const char *dbase_identify(uint8 info);

入力と出力

  • **入力**: info (uint8) - WAL レコードの info フィールド。
  • **出力**: const char * - 操作の種類を表す文字列（"CREATE_FILE_COPY", "CREATE_WAL_LOG", "DROP" のいずれか）。該当しない場合は NULL。

前提条件と不変条件

  • 入力された info ビットから XLR_INFO_MASK を除外して評価します。

アルゴリズムまたは振る舞い

1. info に対して ~XLR_INFO_MASK を適用し、フラグビットを除去します。 2. マスク後の値に基づき、以下のいずれかに該当するかを switch 文で判定します。

  • XLOG_DBASE_CREATE_FILE_COPY: "CREATE_FILE_COPY" を返します。
  • XLOG_DBASE_CREATE_WAL_LOG: "CREATE_WAL_LOG" を返します。
  • XLOG_DBASE_DROP: "DROP" を返します。

    • 3. いずれにも一致しない場合、初期値である NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/include/access/rmgrlist.h: RM_DBASE_ID リソースマネージャの定義内で参照されています。
  • **呼び出し先**:
  • 本関数は内部で他の関数を呼び出しません。

関連データ構造

  • XLOG_DBASE_CREATE_FILE_COPY, XLOG_DBASE_CREATE_WAL_LOG, XLOG_DBASE_DROP: commands/dbcommands_xlog.h で定義された WAL レコードタイプ定数。

失敗モードと境界条件

  • info が既知のデータベース操作タイプと一致しない場合、NULL を返します。

保守担当者向けメモ

  • dbcommands_xlog.h に新しいデータベース操作タイプが追加された場合は、本関数の switch 文に新しい case を追加する必要があります。
