blinsert

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blinsert.c:174

目的

Bloom 索引に対して、単一のタプルを挿入する。

シグネチャまたは形状

bool blinsert(Relation index, Datum *values, bool *isnull,
              ItemPointer ht_ctid, Relation heapRel,
              IndexUniqueCheck checkUnique,
              bool indexUnchanged,
              IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • index: 挿入対象の Bloom 索引リレーション。
  • values, isnull: 挿入するタプルの属性値と NULL 判定配列。
  • ht_ctid: 挿入対象タプルのヒープ内位置。
  • **出力**: 常に false を返す。

前提条件と不変条件

  • 挿入処理用に一時的な MemoryContext を作成し、処理終了時に破棄する。
  • ページへの変更は GenericXLog を介してアトミックに行われる。
  • メタページ（BLOOM_METAPAGE_BLKNO）は、空きページリスト（notFullPage）の管理に使用される。

アルゴリズムまたは振る舞い

1. **初期化**: 挿入用メモリコンテキストを作成し、BloomFormTuple でタプルを生成する。 2. **高速パス**: メタページを共有ロックし、notFullPage 配列の先頭ページへの挿入を試みる。成功すれば即座に終了する。 3. **探索**: 先頭ページで失敗した場合、メタページを排他ロックし、notFullPage 配列を順次走査して空きのあるページを探す。 4. **新規割り当て**: 既存のページに空きがない場合、BloomNewBuffer で新規ページを割り当て、メタページを更新する。 5. **コミット**: GenericXLogFinish を呼び出し、変更を確定させてリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blutils.c 内の aminsert ルーチンとして登録されている。
  • contrib/bloom/blvacuum.c からの参照が示唆されている。
  • **呼び出し先**:
  • initBloomState, BloomFormTuple, BloomPageGetMeta, BloomInitPage, BloomPageAddItem, BloomNewBuffer
  • GenericXLog 系関数 (Start, RegisterBuffer, Finish, Abort)
  • ReadBuffer, LockBuffer, UnlockReleaseBuffer

関連データ構造

  • BloomState: 索引の状態管理。
  • BloomMetaPageData: メタページ構造。
  • BloomTuple: 挿入されるタプルデータ。
  • GenericXLogState: WAL ログ記録用状態。

失敗モードと境界条件

  • notFullPage 配列内の全ページが満杯の場合、新規ページを割り当てる。
  • 新規ページ割り当て時に BloomPageAddItem が失敗した場合は elog(ERROR) を発生させる。
  • ページが削除済み（BloomPageIsDeleted）または新規（PageIsNew）の場合、BloomInitPage で再初期化される。

保守担当者向けメモ

  • GenericXLog を使用しているため、ページ変更時は必ず GenericXLogRegisterBuffer を経由すること。
  • メタページのロック昇格（共有から排他へ）は、競合を避けるための最適化として実装されている。
  • メモリリークを防ぐため、MemoryContext の切り替えと削除が適切に行われているか注意すること。
