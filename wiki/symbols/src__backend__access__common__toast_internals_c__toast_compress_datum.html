toast_compress_datum

定義

  • 種別: function
  • 定義箇所: src/backend/access/common/toast_internals.c:45

目的

入力された varlena 形式のデータを指定されたアルゴリズムで圧縮し、圧縮によるサイズ削減が十分な場合にのみ圧縮済みデータを返します。

シグネチャまたは形状

Datum toast_compress_datum(Datum value, char cmethod)

入力と出力

  • **入力**:
  • value: 圧縮対象の Datum（varlena 形式）。
  • cmethod: 圧縮アルゴリズムを指定する文字コード。無効な場合はデフォルト値が使用されます。
  • **出力**:
  • 圧縮に成功し、かつサイズ削減効果が認められた場合は圧縮済みデータの Datum を返します。
  • 圧縮が不可能、またはサイズ削減効果が不十分（2バイト以下の削減）な場合は PointerGetDatum(NULL) を返します。

前提条件と不変条件

  • 入力 value は外部データ（VARATT_IS_EXTERNAL）であってはなりません。
  • 入力 value は既に圧縮済み（VARATT_IS_COMPRESSED）であってはなりません。
  • 圧縮後のサイズが元のサイズと比較して3バイト以上小さくない場合、圧縮は「不十分」とみなされ破棄されます。

アルゴリズムまたは振る舞い

1. cmethod が有効か確認し、無効であれば default_toast_compression を適用します。 2. cmethod に応じて pglz_compress_datum または lz4_compress_datum を呼び出します。 3. 圧縮ルーチンが NULL を返した場合は、そのまま NULL を返します。 4. 圧縮後のサイズ（ヘッダー含む）が元のサイズから2バイトを引いた値よりも小さい場合のみ、TOAST_COMPRESS_SET_SIZE_AND_COMPRESS_METHOD マクロでヘッダーを更新し、圧縮データを返します。 5. それ以外の場合は、一時領域を pfree し、NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_tuple.c
  • src/backend/access/common/indextuple.c
  • src/backend/access/table/toast_helper.c
  • **呼び出し先**:
  • pglz_compress_datum
  • lz4_compress_datum
  • pfree

関連データ構造

  • varlena: PostgreSQLの可変長データ型構造体。
  • ToastCompressionId: 圧縮アルゴリズムを識別するID。

失敗モードと境界条件

  • **エラー**: サポートされていない cmethod が渡された場合、elog(ERROR) によりトランザクションが中断されます。
  • **境界条件**: 圧縮によるサイズ削減が1〜2バイト程度の場合、ヘッダーやパディングのオーバーヘッドにより逆効果となる可能性があるため、本関数はこれを「圧縮失敗」として扱い、元のデータを維持します。

保守担当者向けメモ

  • 本関数は圧縮の成否をサイズ削減量で判断する「ガード」の役割を兼ねています。圧縮アルゴリズムを追加する場合は、switch 文へのケース追加が必要です。
