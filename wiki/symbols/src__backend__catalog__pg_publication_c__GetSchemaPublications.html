GetSchemaPublications

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_publication.c:962

目的

指定されたスキーマに関連付けられているすべてのパブリケーションの OID リストを取得します。

シグネチャまたは形状

List *GetSchemaPublications(Oid schemaid)

入力と出力

  • **入力**: schemaid (Oid) - 対象となるスキーマの OID。
  • **出力**: List * - スキーマに関連付けられたパブリケーションの OID を格納したリスト。

前提条件と不変条件

  • PUBLICATIONNAMESPACEMAP システムキャッシュが利用可能であること。
  • 戻り値のリストは lappend_oid によって構築されるため、呼び出し元で適切にメモリ管理（リストの解放）を行う必要があります。

アルゴリズムまたは振る舞い

1. SearchSysCacheList1 を使用し、PUBLICATIONNAMESPACEMAP インデックスを介して指定された schemaid に一致するすべてのタプルを検索します。 2. 取得したリストを反復処理し、各タプルから pg_publication_namespace 構造体の pnpubid フィールド（パブリケーション ID）を抽出します。 3. 抽出したパブリケーション ID を result リストに追加します。 4. ReleaseSysCacheList を呼び出し、検索に使用したキャッシュリストを解放します。 5. 構築したパブリケーション ID のリストを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/catalog/pg_publication.c:380
  • src/backend/replication/pgoutput/pgoutput.c:2096
  • src/backend/utils/cache/relcache.c:5836, 5851
  • **呼び出し先**:
  • SearchSysCacheList1
  • GETSTRUCT
  • lappend_oid
  • ReleaseSysCacheList

関連データ構造

  • pg_publication_namespace: パブリケーションとスキーマの紐付けを保持するシステムカタログ構造体。
  • CatCList: システムキャッシュ検索結果を保持するリスト構造。

失敗モードと境界条件

  • 指定された schemaid に関連付けられたパブリケーションが存在しない場合、空のリスト（NIL）が返されます。
  • システムキャッシュの検索失敗やメモリ不足が発生した場合、PostgreSQL の標準的なエラーハンドリング（elog 等）により処理が中断される可能性があります。

保守担当者向けメモ

  • 本関数は pg_publication_namespace カタログを参照しています。スキーマとパブリケーションの紐付けロジックに変更を加える際は、このカタログのインデックス定義と整合性を確認してください。
