GucShowHook

定義

  • 種別: typedef
  • 定義箇所: src/include/utils/guc.h:195

目的

GUC（Grand Unified Configuration）パラメータの現在の値を、表示用の文字列として取得するためのフック関数を定義します。

シグネチャまたは形状

typedef const char *(*GucShowHook) (void);

入力と出力

  • **入力**: なし（void）。
  • **出力**: 現在の設定値を表す const char * 型の文字列を返します。

前提条件と不変条件

  • 返される文字列は、呼び出し元が解放する必要のない静的メモリまたは永続的なメモリ領域を指している必要があります。
  • フック関数は、現在のGUC状態に基づいて動的に値を生成または選択して返すことが期待されます。

アルゴリズムまたは振る舞い

GUCシステムにおいて、特定のパラメータに対して show_hook が設定されている場合、SHOW コマンドや設定値の取得要求時にこの関数が呼び出されます。標準的な値の参照ではなく、このフックを介して計算された値や、特定の文脈に応じた文字列が提供されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/misc/guc.c 内のGUC値取得処理（show_config_option 等）から呼び出されることが示唆されています。
  • **呼び出し先**: 呼び出し先に関する具体的な情報は提供されていません。

関連データ構造

  • src/include/utils/guc_tables.h: show_hook フィールドを持つ構造体定義が含まれており、GUC変数の定義テーブル内で使用されます。

失敗モードと境界条件

  • フック関数が NULL を返す場合の挙動については、本ソース excerpt からは定義されていません。
  • 呼び出し元が pg_attribute_nonnull を指定している箇所があるため、実装側で NULL を返さないことが強く推奨されます。

保守担当者向けメモ

  • CheckHook や AssignHook と同様に、GUC変数のライフサイクル管理において重要な役割を果たします。
  • src/include/utils/guc.h の 341, 354, 367, 378, 390 行目などで、このフックを利用するインターフェースが宣言されています。
