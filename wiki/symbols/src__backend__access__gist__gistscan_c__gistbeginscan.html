gistbeginscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/gist/gistscan.c:73

目的

GiSTインデックススキャンを開始するためのリソース割り当てと初期化を行います。

シグネチャまたは形状

IndexScanDesc gistbeginscan(Relation r, int nkeys, int norderbys)

入力と出力

  • **入力**:
  • r: スキャン対象のインデックスリレーション。
  • nkeys: スキャンキーの数。
  • norderbys: OrderBy句の数。
  • **出力**:
  • 初期化済みの IndexScanDesc 構造体。

前提条件と不変条件

  • giststate->scanCxt がスキャン全体のメモリ管理コンテキストとして使用されます。
  • gistendscan が呼び出されることで、本関数で割り当てられたすべてのリソースが解放されることが保証されています。

アルゴリズムまたは振る舞い

1. RelationGetIndexScan を呼び出し、スキャン記述子を生成します。 2. initGISTstate を使用して GISTSTATE を初期化します。 3. スキャン専用のメモリコンテキストへ切り替え、GISTScanOpaqueData を割り当てます。 4. 検索キュー (so->queue) を初期化し、距離計算用ワークスペースを確保します。 5. norderbys > 0 の場合、xs_orderbyvals および xs_orderbynulls を初期化します。 6. スキャン状態の不透明ポインタ (scan->opaque) に so を設定します。 7. インデックス専用スキャン（Index-only scan）に必要なフィールドは、本関数では初期化されず、後続の gistrescan で処理されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gist/gist.c: amroutine->ambeginscan に関数ポインタとして登録されています。
  • **呼び出し先**:
  • RelationGetIndexScan
  • initGISTstate
  • MemoryContextSwitchTo
  • palloc0 / palloc
  • createTempGistContext
  • memset

関連データ構造

  • IndexScanDesc: スキャン全体の記述子。
  • GISTSTATE: GiSTインデックスの実行時状態。
  • GISTScanOpaqueData: GiST固有のスキャン状態（検索キュー、距離計算用ワークスペース等）。

失敗モードと境界条件

  • メモリ不足時には palloc 等によりエラーが発生します。
  • norderbys が 0 の場合、xs_orderbyvals 等の割り当てはスキップされます。

保守担当者向けメモ

  • 本関数で作成されたリソースは giststate->scanCxt の子として管理されるため、個別の解放処理は不要です。
  • インデックス専用スキャンの初期化は gistrescan に委譲されているため、変更を加える際は注意してください。
