setop_fill_hash_table

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSetOp.c:406

目的

SetOp ノードのハッシュ戦略において、外部および内部リレーションから全タプルを読み込み、グループごとの出現回数を集計したハッシュテーブルを構築する。

シグネチャまたは形状

static void setop_fill_hash_table(SetOpState *setopstate)

入力と出力

  • **入力**: SetOpState *setopstate（実行状態を保持する構造体。hashtable および子プランナーの状態を含む）。
  • **出力**: なし（副作用として setopstate 内のハッシュテーブルが構築され、table_filled フラグが true に設定される）。

前提条件と不変条件

  • setopstate->hashtable が初期化済みであること。
  • 呼び出し後、setopstate->hashiter がハッシュテーブル走査のために初期化される。
  • 各タプルのハッシュ検索およびカウント処理の前後で、econtext（式コンテキスト）がリセットされること。

アルゴリズムまたは振る舞い

1. **外部リレーションの処理**:

  • outerPlan からタプルを順次取得する。
  • LookupTupleHashEntry を使用してハッシュテーブルエントリを取得（存在しない場合は新規作成）。
  • 新規エントリの場合、numLeft と numRight を 0 に初期化する。
  • numLeft をインクリメントする。

    • 2. **内部リレーションの処理**:

  • 外部リレーションにタプルが存在した場合のみ実行する。
  • innerPlan からタプルを順次取得する。
  • LookupTupleHashEntry で既存エントリを検索する（新規作成は行わない）。
  • エントリが存在する場合のみ numRight をインクリメントする。

    • 3. **終了処理**:

  • setopstate->table_filled を true に設定。
  • ResetTupleHashIterator を呼び出し、ハッシュテーブル走査の準備を行う。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/nodeSetOp.c 内の ExecSetOp（ハッシュ戦略選択時）。
  • **呼び出し先**: outerPlanState, innerPlanState, ExecProcNode, LookupTupleHashEntry, TupleHashEntryGetAdditional, ResetExprContext, ResetTupleHashIterator。

関連データ構造

  • SetOpState: 実行ノードの状態管理。
  • TupleHashTable: タプルをグループ化して保持するハッシュテーブル。
  • SetOpStatePerGroupData: 各グループの出現回数（numLeft, numRight）を保持する構造体。

失敗モードと境界条件

  • 外部リレーションが空の場合、内部リレーションの読み込みはスキップされる。
  • 内部リレーションのタプルがハッシュテーブルに存在しない場合、そのタプルは無視される（entry が NULL の場合はカウントされない）。

保守担当者向けメモ

  • 本関数は SetOp ノードのハッシュベース実行における全データ読み込みフェーズを担う。
  • メモリ管理は setopstate->hashtable が依存するメモリコンテキストに依存するため、再スキャン時のリセットロジックと整合している必要がある。
