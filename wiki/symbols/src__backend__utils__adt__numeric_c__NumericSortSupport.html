NumericSortSupport

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/numeric.c:340

目的

NUMERIC 型のソート処理において、省略形（abbreviated key）を用いた比較の最適化およびカーディナリティ（値の多様性）推定を行うための状態を保持します。

シグネチャまたは形状

typedef struct {
    void           *buf;                /* 短い可変長データのバッファ */
    int64           input_count;        /* 非NULL値のカウント */
    bool            estimating;         /* カーディナリティ推定中か否か */
    hyperLogLogState abbr_card;         /* カーディナリティ推定器 */
} NumericSortSupport;

入力と出力

  • 本構造体は、ソートサポート関数（SortSupport）の ssup_extra フィールドを通じて管理されます。
  • numeric_abbrev_convert_var 関数において、NumericVar を入力として受け取り、ソート用の省略形キーを生成する際に利用されます。

前提条件と不変条件

  • palloc を使用してメモリが割り当てられる必要があります。
  • estimating フラグが true の場合、abbr_card が適切に初期化され、カーディナリティの推定に使用されます。

アルゴリズムまたは振る舞い

1. **初期化**: ソート開始時に palloc でメモリを確保し、状態を初期化します。 2. **変換**: numeric_abbrev_convert_var が呼び出されると、NumericVar の値を比較可能な省略形キーに変換します。 3. **推定**: estimating が有効な場合、hyperLogLogState を用いて入力値のカーディナリティを追跡し、省略形キーによる比較が有効かどうかを判断します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/numeric.c 内のソートサポート関連関数（numeric_abbrev_convert_var 等）。
  • **呼び出し先**: hyperLogLog 関連の関数（abbr_card の操作用）、およびメモリ管理関数（palloc）。

関連データ構造

  • NumericVar: 算術演算および変換の対象となるメモリ上の数値表現。
  • hyperLogLogState: カーディナリティ推定のための状態保持構造体。
  • SortSupport: PostgreSQL のソート最適化フレームワークで使用される上位構造体。

失敗モードと境界条件

  • メモリ不足時に palloc が失敗する可能性があります。
  • estimating 状態の管理が不適切な場合、カーディナリティ推定値が不正確になる可能性があります。

保守担当者向けメモ

  • 本構造体は numeric.c 内のソート最適化ロジックに密結合しています。
  • abbr_card の更新ロジックを変更する際は、hyperLogLog の仕様と numeric_abbrev_convert_var での利用状況を併せて確認してください。
