ExecGrant_Relation

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/aclchk.c:1784

目的

リレーション（テーブル、シーケンス等）に対する GRANT / REVOKE 文の実行処理を行い、システムカタログ pg_class の ACL 情報を更新する。

シグネチャまたは形状

static void ExecGrant_Relation(InternalGrant *istmt)

入力と出力

  • **入力**: InternalGrant *istmt（実行対象の権限操作情報を含む構造体）
  • **出力**: なし（副作用としてシステムカタログを更新）

前提条件と不変条件

  • istmt->objects に含まれる各 OID は、pg_class に存在する有効なリレーションであること。
  • インデックスや複合型に対する権限付与は許可されない。
  • シーケンスに対しては、シーケンス専用の権限のみが許可される。

アルゴリズムまたは振る舞い

1. pg_class および pg_attribute テーブルを RowExclusiveLock でオープンする。 2. 対象リレーションごとに RELOID システムキャッシュを検索し、リレーションの種類（relkind）を確認する。 3. インデックス、複合型、またはシーケンスに対する不適切な操作を検出し、エラーを投げる。 4. istmt->all_privs が指定されている場合、リレーションの種類に応じて ACL_ALL_RIGHTS_SEQUENCE または ACL_ALL_RIGHTS_RELATION を適用する。 5. REVOKE 操作かつ列権限が指定されている場合、expand_all_col_privileges を呼び出し、列レベルの権限を暗黙的に取り消す。 6. pg_class の relacl 属性を取得し、存在しない場合はデフォルトの ACL を生成する。 7. 既存の ACL をコピーし、後続の権限マージ処理へ移行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/aclchk.c 内の権限管理ロジックから呼び出される（直接の呼び出し元はソースコード上の ExecuteGrantStmt 等のフローに関連）。
  • **呼び出し先**: table_open, SearchSysCacheLocked1, expand_all_col_privileges, acldefault, DatumGetAclPCopy, aclmembers, aclcopy 等。

関連データ構造

  • InternalGrant: 権限操作の定義を保持する構造体。
  • Form_pg_class: pg_class カタログのタプル構造。
  • AclMode: 権限ビットマスク。

失敗モードと境界条件

  • **エラー**:
  • 対象がインデックスまたは複合型の場合（ERRCODE_WRONG_OBJECT_TYPE）。
  • シーケンス以外のオブジェクトに対してシーケンス権限を操作しようとした場合。
  • 不正な権限タイプを指定した場合（ERRCODE_INVALID_GRANT_OPERATION）。
  • **警告**:
  • シーケンスに対して非対応の権限を付与しようとした場合、警告を発行し、対応する権限のみに制限して処理を継続する。

保守担当者向けメモ

  • 本関数はリレーションレベルの権限を扱うが、列レベルの権限変更が必要な場合は ExecGrant_Attribute 等と連携する。
  • pg_upgrade 等で利用される binary_upgrade_record_init_privs のような初期権限移行フラグとの整合性に注意すること。
