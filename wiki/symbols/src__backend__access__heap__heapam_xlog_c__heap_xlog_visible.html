heap_xlog_visible

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_xlog.c:181

目的

ヒープページの「全可視（all-visible）」状態をWALリプレイ時に反映し、可視性マップ（Visibility Map）およびフリースペースマップ（FSM）を更新する。

シグネチャまたは形状

static void heap_xlog_visible(XLogReaderState *record)

入力と出力

  • **入力**: XLogReaderState *record（処理対象のWALレコード）
  • **出力**: なし（副作用としてバッファおよび可視性マップを更新）

前提条件と不変条件

  • xlrec->flags は VISIBILITYMAP_XLOG_VALID_BITS の範囲内であること。
  • ホットスタンバイ環境では、ResolveRecoveryConflictWithSnapshot を通じてスナップショット競合の解決が試行される。

アルゴリズムまたは振る舞い

1. **競合解決**: ホットスタンバイ稼働中であれば、スナップショットの競合を解決する。 2. **ヒープページ更新**: XLogReadBufferForRedo で対象ページを取得し、PageSetAllVisible を適用する。必要に応じてLSNを更新し、バッファをダーティにする。 3. **FSM更新**: ページが全可視化されたことに伴い、XLogRecordPageWithFreeSpace を呼び出してFSMの情報を更新する。 4. **可視性マップ更新**: XLogReadBufferForRedoExtended を使用して可視性マップページを読み込み、visibilitymap_set を実行してビットを反映させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/heap/heapam_xlog.c 内のWALリプレイディスパッチャ（heap2_redo 等から間接的に呼び出される）。
  • **呼び出し先**: ResolveRecoveryConflictWithSnapshot, XLogReadBufferForRedo, PageSetAllVisible, PageSetLSN, MarkBufferDirty, XLogRecordPageWithFreeSpace, XLogReadBufferForRedoExtended, visibilitymap_set 等。

関連データ構造

  • xl_heap_visible: WALレコードのデータ構造。
  • XLogReaderState: WALレコード読み取り状態。
  • Buffer: バッファ管理構造。
  • RelFileLocator: リレーション識別子。

失敗モードと境界条件

  • ヒープファイルがリカバリ中に削除または切り詰められた場合、ヒープページの更新はスキップされるが、可視性マップの更新は継続される。
  • 可視性マップページが新規作成される場合は PageInit が行われる。

保守担当者向けメモ

  • 本関数は、ヒープページと可視性マップの両方を更新する。ヒープページ更新のLSNインターロックをスキップしても、可視性マップの更新は安全に行われる設計となっている。
  • FSMはWALログに記録されないため、本関数でヒューリスティックに更新することで、スタンバイ側でのFSMの陳腐化を防いでいる。
