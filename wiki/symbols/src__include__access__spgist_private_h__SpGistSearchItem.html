SpGistSearchItem

定義

  • 種別: struct
  • 定義箇所: src/include/access/spgist_private.h:165

目的

SP-GiSTインデックスのスキャン処理において、探索対象となるノードやリーフタプルの状態をキュー内で管理するための構造体です。

シグネチャまたは形状

typedef struct SpGistSearchItem
{
    pairingheap_node phNode;
    Datum           value;
    SpGistLeafTuple leafTuple;
    void           *traversalValue;
    int             level;
    ItemPointerData heapPtr;
    bool            isNull;
    bool            isLeaf;
    bool            recheck;
    bool            recheckDistances;
    double          distances[FLEXIBLE_ARRAY_MEMBER];
} SpGistSearchItem;

入力と出力

  • **入力**: 検索実行時に spgAllocSearchItem 等を介して生成され、spgAddSearchItemToQueue によりスキャン状態（SpGistScanOpaque）のキューへ追加されます。
  • **出力**: pairingheap を用いた優先度付きキューから取り出され、インデックスの探索処理に使用されます。

前提条件と不変条件

  • distances フィールドは FLEXIBLE_ARRAY_MEMBER を使用しており、numberOfOrderBys に応じた可変長配列として扱われます。
  • isLeaf が true の場合、leafTuple に有効なリーフタプル情報が含まれます。

アルゴリズムまたは振る舞い

  • pairingheap のノードとして機能し、スキャン中の探索順序を制御します。
  • 演算子クラス固有のトラバーサル値（traversalValue）を保持し、ツリー探索時の状態遷移をサポートします。
  • 距離計算が必要な場合、distances 配列を用いて順序付けを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgscan.c 内の各関数（spgAddSearchItemToQueue, spgAllocSearchItem など）から生成・操作されます。
  • **呼び出し先**: pairingheap の比較関数（spgscan.c 内）から、キューの順序決定のために参照されます。

関連データ構造

  • pairingheap_node: キュー管理のための基底構造。
  • SpGistLeafTuple: リーフタプルの詳細情報。
  • SpGistScanOpaqueData: スキャン全体の状態を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: spgAllocSearchItem 呼び出し時にメモリ確保に失敗する可能性があります。
  • 距離計算の不整合: recheckDistances が設定されている場合、後続の再チェック処理が必要です。

保守担当者向けメモ

  • 本構造体は pairingheap に依存しているため、キュー操作時の比較ロジックを変更する際は spgscan.c の比較関数を併せて確認してください。
  • distances は可変長配列であるため、構造体の割り当て時には numberOfOrderBys を考慮したサイズ計算が必須です。
