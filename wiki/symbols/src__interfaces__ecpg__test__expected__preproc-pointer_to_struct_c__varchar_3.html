varchar_3

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c:111

目的

ECPG（Embedded SQL for C）プリプロセッサの型変換テストにおいて、可変長文字列（varchar）を保持するための構造体定義を提供します。

シグネチャまたは形状

struct varchar_3 {
    int len;
    char arr[50];
};

入力と出力

  • 入力: SQLクエリの結果として取得される文字列データ。
  • 出力: len メンバに文字列長、arr メンバに文字列本体が格納されます。

前提条件と不変条件

  • arr は固定長 50 バイトのバッファとして確保されます。
  • ECPGの仕様に基づき、len と arr の順序および構造が維持される必要があります。

アルゴリズムまたは振る舞い

本構造体は、ECPGプリプロセッサがSQLの VARCHAR 型をC言語の構造体にマッピングする際のターゲットとして使用されます。データフェッチ時に、データベースから取得した値がこの構造体のメモリレイアウトに従って書き込まれます。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/ecpg/test/expected/preproc-pointer_to_struct.c 内のメイン処理（main 関数等）から、ホスト変数として参照されます。
  • 呼び出し先: 該当なし（データ保持用の構造体定義のため）。

関連データ構造

  • varchar_1, varchar_2, varchar_4: 同様の目的で定義された他のバリエーション。
  • customer 構造体群: 本構造体をメンバとして含む可能性のあるテスト用構造体。

失敗モードと境界条件

  • 文字列長が 50 バイトを超える場合、arr への書き込み時にバッファオーバーフローが発生する可能性があります。
  • ECPGプリプロセッサが期待するメモリレイアウトと一致しない場合、データフェッチ時に不正な値が読み取られる可能性があります。

保守担当者向けメモ

  • 本ファイルはプリプロセッサの出力結果であるため、直接編集しないでください。
  • 構造体のメンバ構成を変更する場合は、ECPGプリプロセッサ側の型変換ロジックとの整合性を確認してください。
