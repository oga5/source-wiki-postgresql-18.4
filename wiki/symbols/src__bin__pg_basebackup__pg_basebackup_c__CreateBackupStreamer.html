CreateBackupStreamer

定義

  • 種別: function
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:1061

目的

バックアップデータの受信および処理を行うための astreamer パイプラインを構築します。出力フォーマット（plain/tar）や圧縮設定、マニフェスト注入の要否に応じて適切なストリーマーを連結します。

シグネチャまたは形状

static astreamer *CreateBackupStreamer(
    char *archive_name, 
    char *spclocation,
    astreamer **manifest_inject_streamer_p,
    bool is_recovery_guc_supported,
    bool expect_unterminated_tarfile,
    pg_compress_specification *compress
);

入力と出力

  • **入力**: アーカイブ名、テーブルスペースの場所、マニフェスト注入用ポインタ、リカバリ設定のサポート状況、圧縮仕様など。
  • **出力**: 構築された astreamer パイプラインの先頭ポインタ。

前提条件と不変条件

  • format == 'p'（plain形式）の場合、アーカイブの抽出処理が必須となります。
  • format == 't'（tar形式）かつ標準出力への書き出し時、マニフェストの注入が要求されている場合は、単一のtarファイル内にマニフェストを含める必要があります。
  • 圧縮されたtarファイルへのマニフェスト注入はサポートされておらず、エラーとなります。

アルゴリズムまたは振る舞い

1. **形式判定**: アーカイブ名から拡張子を解析し、tar形式および圧縮の種類（gzip, lz4, zstd）を特定します。 2. **バリデーション**: 圧縮済みtarへのマニフェスト注入要求や、解析が必要な形式（plain等）でtar以外が指定された場合にエラー終了します。 3. **ストリーマー構築**:

  • **Plain形式**: astreamer_extractor_new を使用して展開処理を行うストリーマーを生成します。
  • **Tar形式**: 圧縮アルゴリズムに応じたライター（astreamer_plain_writer_new 等）を生成し、必要に応じて圧縮用ストリーマーをラップします。解析が必要な場合は astreamer_tar_archiver_new を追加します。

    • 4. **マニフェスト注入**: 要求がある場合、該当するストリーマーを manifest_inject_streamer_p に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_basebackup/pg_basebackup.c 内のバックアップ処理フロー（行1404, 1614など）。
  • **呼び出し先**: astreamer インターフェースの各コンストラクタ（astreamer_extractor_new, astreamer_plain_writer_new, astreamer_tar_archiver_new 等）。

関連データ構造

  • astreamer: データ処理パイプラインを構成する抽象構造体。
  • pg_compress_specification: 圧縮アルゴリズムおよび設定を保持する構造体。

失敗モードと境界条件

  • **エラー終了**: 圧縮済みtarへのマニフェスト注入要求、または解析不能な形式でのアーカイブ処理要求が発生した場合、pg_log_error を出力して exit(1) します。
  • **未対応**: 圧縮済みtarの解析（展開・再圧縮を伴う処理）は効率の観点からサポート外です。

保守担当者向けメモ

  • 本関数は astreamer パイプラインの組み立て順序を制御する中心的な役割を担います。新しい圧縮形式や処理ステップを追加する場合は、パイプラインの連結順序に注意してください。
  • spclocation の処理において、テーブルスペースマッピングが正しく適用されているか確認してください。
