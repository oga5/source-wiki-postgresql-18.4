getObjectDescription

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaddress.c:2911

目的

システムカタログ内のオブジェクトを識別し、人間が読み取れる形式の説明文（文字列）を生成します。

シグネチャまたは形状

char *getObjectDescription(const ObjectAddress *object, bool missing_ok)

入力と出力

  • **入力**:
  • object: 説明を取得する対象の ObjectAddress 構造体へのポインタ。
  • missing_ok: true の場合、オブジェクトが見つからない場合にエラーを発生させず、呼び出し元で処理を継続可能にするフラグ。
  • **出力**:
  • オブジェクトの説明文を含む動的に割り当てられた文字列（char *）。呼び出し元で pfree する必要があります。

前提条件と不変条件

  • ObjectAddress 構造体の classId に基づいて適切なカタログテーブルを検索します。
  • 戻り値の文字列は palloc されたメモリ領域です。

アルゴリズムまたは振る舞い

1. StringInfoData を初期化し、説明文の構築を開始します。 2. object->classId に基づき、以下のオブジェクトタイプごとに処理を分岐します。

  • **Relation**: リレーションまたはその列の説明を取得します。
  • **Procedure**: 関数名を取得します。
  • **Type**: 型名を取得します。
  • **Cast**: pg_cast を検索し、キャスト元とキャスト先の型名を含む説明を生成します。
  • **Collation**: pg_collation を検索し、名前空間を考慮した照合順序名を取得します。
  • **Constraint**: pg_constraint を検索し、制約名と関連するリレーション名を取得します。

    • 3. 構築された文字列を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/dependency.c（依存関係のログ出力等）、contrib/sepgsql/dml.c、contrib/sepgsql/label.c などから呼び出されます。
  • **呼び出し先**: getRelationDescription, get_attname, format_procedure_extended, format_type_extended, table_open, systable_beginscan, SearchSysCache1 等のカタログアクセス関数。

関連データ構造

  • ObjectAddress: オブジェクトを特定するためのクラスID、オブジェクトID、サブIDを保持。
  • StringInfoData: 文字列構築用バッファ。

失敗モードと境界条件

  • missing_ok が false の場合、オブジェクトがシステムカタログに見つからないと elog(ERROR) を発生させます。
  • missing_ok が true の場合、オブジェクトが存在しない場合は NULL を返すか、空の文字列を生成して処理を終了します。

保守担当者向けメモ

  • 新しいオブジェクトタイプを追加する場合は、switch 文に新しい case を追加し、対応するカタログテーブルの検索ロジックを実装してください。
  • pfree によるメモリリークを防ぐため、StringInfo を使用した一時的な文字列生成後の解放を徹底してください。
