makepoint

定義

  • 種別: function
  • 定義箇所: src/tutorial/funcs.c:47

目的

2つの Point 型引数から、一方の X 座標と他方の Y 座標を組み合わせた新しい Point 型の値を生成して返します。

シグネチャまたは形状

Datum makepoint(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取る2つの Point 型ポインタ（PG_GETARG_POINT_P マクロを使用）。
  • **出力**: 新たに作成された Point 型の Datum（PG_RETURN_POINT_P マクロを使用）。

前提条件と不変条件

  • PostgreSQL の V1 呼び出し規約に従う関数として定義されています。
  • palloc を使用してメモリを確保するため、呼び出し元のメモリコンテキストが有効である必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_POINT_P(0) および (1) により、入力された2つの Point 構造体へのポインタを取得します。 2. palloc(sizeof(Point)) を呼び出し、新しい Point 構造体用のメモリを動的に確保します。 3. 第1引数の x 座標と、第2引数の y 座標を新しい構造体にコピーします。 4. PG_RETURN_POINT_P を使用して、構築した Point を Datum 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（PG_FUNCTION_INFO_V1 マクロにより登録）。
  • **呼び出し先**:
  • palloc（PostgreSQL メモリ管理関数）
  • PG_GETARG_POINT_P / PG_RETURN_POINT_P（PostgreSQL マクロ）

関連データ構造

  • Point: utils/geo_decls.h で定義される座標構造体。

失敗モードと境界条件

  • メモリ不足時には palloc がエラーを発生させ、トランザクションがアボートする可能性があります。
  • 入力引数が NULL の場合、PG_GETARG_POINT_P の挙動に依存します。

保守担当者向けメモ

  • 本関数は src/tutorial/funcs.c 内のサンプル実装です。
  • palloc で確保したメモリは PostgreSQL のメモリコンテキスト管理下にあるため、明示的な解放は不要です。
