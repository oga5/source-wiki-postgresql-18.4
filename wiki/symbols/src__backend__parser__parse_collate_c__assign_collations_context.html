assign_collations_context

定義

  • 種別: struct
  • 定義箇所: src/backend/parser/parse_collate.c:64

目的

式解析の最終段階において、構文木を走査しながら照合順序（Collation）の解決状態を保持・伝播するためのコンテキスト構造体です。

シグネチャまたは形状

typedef struct {
    ParseState *pstate;         /* 解析状態（エラー報告用） */
    Oid         collation;      /* 現在の照合順序のOID */
    CollateStrength strength;   /* 照合順序の派生強度 */
    int         location;       /* 照合順序を設定した式の位置 */
    Oid         collation2;     /* 競合時の照合順序OID */
    int         location2;      /* 競合時の式位置 */
} assign_collations_context;

入力と出力

本構造体は関数引数として渡され、走査中のノード間で共有されます。

  • **入力**: pstate を含む初期化されたコンテキスト。
  • **出力**: 走査完了後、決定された照合順序情報が collation および strength フィールドに格納されます。

前提条件と不変条件

  • strength が COLLATE_CONFLICT の場合のみ、collation2 および location2 フィールドが有効です。
  • pstate はエラー報告のために常に有効である必要があります。

アルゴリズムまたは振る舞い

  • assign_collations_walker 等の再帰的な走査関数によって、子ノードから親ノードへ照合順序をバブルアップさせる際に状態を保持します。
  • merge_collation_state を通じて、現在のノードの照合順序とコンテキスト内の既存の照合順序をマージし、競合が発生した場合は strength を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: assign_query_collations、assign_collations_walker、assign_list_collations、assign_expr_collations、assign_aggregate_collations、assign_ordered_set_collations、assign_hypothetical_collations 等の解析関数から参照・更新されます。
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • CollateStrength: 照合順序の派生強度を定義する列挙型。
  • ParseState: 解析状態を保持する構造体。

失敗モードと境界条件

  • 異なる照合順序を持つ式が混在する場合、strength が COLLATE_CONFLICT に設定され、後続の処理でエラー報告のトリガーとなります。

保守担当者向けメモ

  • 本構造体は、解析中に照合順序の状態を永続的に保持するコストを避けるための「後処理」メカニズムの核となります。
  • Aggref や RowCompareExpr などの特殊なノードを扱う際は、このコンテキストを介した照合順序の解決ロジックが複雑になるため、変更時には注意が必要です。
