HistControl

定義

  • 種別: enum
  • 定義箇所: src/bin/psql/settings.h:86

目的

psqlのコマンド履歴保存時における、重複や空白のフィルタリング動作を制御するための列挙型です。

シグネチャまたは形状

typedef enum
{
	hctl_none = 0,
	hctl_ignorespace = 1,
	hctl_ignoredups = 2,
	hctl_ignoreboth = hctl_ignorespace | hctl_ignoredups,
} HistControl;

入力と出力

  • 入力: 該当なし（列挙型の定義）
  • 出力: 該当なし

前提条件と不変条件

  • hctl_ignoreboth は、hctl_ignorespace と hctl_ignoredups のビット論理和として定義されています。

アルゴリズムまたは振る舞い

  • hctl_none: フィルタリングを行いません。
  • hctl_ignorespace: 空白で始まる行を履歴に保存しません。
  • hctl_ignoredups: 直前の履歴と重複する行を保存しません。
  • hctl_ignoreboth: 空白の無視と重複の無視の両方を適用します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/settings.h 内の PsqlSettings 構造体において、histcontrol フィールドの型として参照されています。
  • 呼び出し先: 該当なし（本シンボルは型定義です）。

関連データ構造

  • PsqlSettings: psql の状態を保持する構造体であり、この型を持つ histcontrol フィールドを含みます。

失敗モードと境界条件

  • 該当なし。

保守担当者向けメモ

  • hctl_ignoreboth はビット演算の結果として定義されているため、この列挙型を拡張する際はビットフラグとしての整合性に注意してください。
