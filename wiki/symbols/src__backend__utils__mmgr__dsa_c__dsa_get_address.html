dsa_get_address

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mmgr/dsa.c:941

目的

共有メモリ内の相対位置を示す dsa_pointer を、現在のプロセスにおける仮想アドレスへ変換する。

シグネチャまたは形状

void *dsa_get_address(dsa_area *area, dsa_pointer dp)

入力と出力

  • **入力**:
  • area: 対象となる dsa_area 構造体へのポインタ。
  • dp: 変換対象の dsa_pointer（セグメント番号とオフセットを含む）。
  • **出力**:
  • 変換後の仮想アドレス（void *）。dp が無効な場合は NULL を返す。

前提条件と不変条件

  • dp が InvalidDsaPointer でないこと。
  • DSA_EXTRACT_SEGMENT_NUMBER(dp) で得られるインデックスが DSA_MAX_SEGMENTS 未満であること。

アルゴリズムまたは振る舞い

1. DsaPointerIsValid を確認し、無効であれば NULL を返す。 2. check_for_freed_segments(area) を呼び出し、解放済みセグメントのデタッチ処理を行う。 3. dp からセグメント番号とオフセットを抽出する。 4. 指定されたセグメントが現在のプロセスにマッピングされていない場合、get_segment_by_index を呼び出してマッピングを確立する。 5. area->segment_maps[index].mapped_address にオフセットを加算した値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execParallel.c
  • src/backend/executor/nodeHash.c
  • **呼び出し先**:
  • check_for_freed_segments
  • get_segment_by_index

関連データ構造

  • dsa_area: DSA 領域の状態を保持する構造体。
  • dsa_pointer: 共有メモリ内の位置を示す擬似ポインタ。
  • dsa_segment_map: セグメントのマッピング情報を保持する構造体。

失敗モードと境界条件

  • dp が無効な場合、NULL を返す。
  • セグメント番号が DSA_MAX_SEGMENTS を超える場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は dsa_pointer を解決する唯一の手段であるため、共有メモリ上のデータにアクセスする際は必ず経由する必要がある。
  • get_segment_by_index の呼び出しにより、必要に応じて動的にセグメントがマッピングされる。
