spgbeginscan

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgscan.c:303

目的

SP-GiSTインデックススキャンを開始するための状態構造体（SpGistScanOpaque）を初期化し、実行エンジンへ IndexScanDesc を提供します。

シグネチャまたは形状

IndexScanDesc spgbeginscan(Relation rel, int keysz, int orderbysz)

入力と出力

  • **入力**:
  • rel: スキャン対象のインデックスリレーション。
  • keysz: スキャンキーの数。
  • orderbysz: Order-by句の数。
  • **出力**:
  • 初期化済みの IndexScanDesc 構造体。内部の opaque フィールドに SpGistScanOpaque が格納されます。

前提条件と不変条件

  • rel は有効なSP-GiSTインデックスである必要があります。
  • SpGistScanOpaque 内の tempCxt および traversalCxt は、スキャン終了までメモリ管理のために維持されます。
  • Order-byスキャン（KNN検索）が要求される場合、numberOfOrderBys > 0 に基づいて距離計算用の配列が確保されます。

アルゴリズムまたは振る舞い

1. RelationGetIndexScan を呼び出し、汎用的なスキャン記述子を生成します。 2. SpGistScanOpaqueData をメモリ確保し、initSpGistState でインデックス状態を初期化します。 3. 検索用の一時メモリコンテキスト（tempCxt, traversalCxt）を作成します。 4. インデックス専用スキャン用のタプル記述子（reconTupDesc）を取得します。 5. Order-byスキャンが有効な場合、距離計算用の配列（zeroDistances, infDistances 等）を初期化し、xs_orderbyvals 等を割り当てます。 6. 演算子クラスの inner_consistent および leaf_consistent メソッドの関数情報を取得し、so に格納します。 7. 最後に scan->opaque に so を設定して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/spgist/spgutils.c: amroutine->ambeginscan に登録されています。
  • **呼び出し先**:
  • RelationGetIndexScan
  • palloc0 / palloc
  • initSpGistState
  • AllocSetContextCreate
  • getSpGistTupleDesc
  • get_float8_infinity
  • fmgr_info_copy
  • index_getprocinfo

関連データ構造

  • SpGistScanOpaque: スキャン中の状態を保持する主要構造体。
  • IndexScanDesc: PostgreSQLの汎用インデックススキャン記述子。
  • ScanKey: 検索条件を保持する構造体。

失敗モードと境界条件

  • メモリ不足時: palloc 等がエラーを発生させます。
  • keysz が0の場合: so->keyData は NULL に設定されます。
  • numberOfOrderBys が0の場合: 距離計算用の配列確保はスキップされます。

保守担当者向けメモ

  • innerConsistentFn および leafConsistentFn は、インデックスの演算子クラスから取得されます。これらはスキャン中の条件判定に不可欠です。
  • KNN検索時の距離計算用配列の初期化ロジックは、spgrescan での再利用を前提としています。
  • メモリコンテキストの分離（tempCxt と traversalCxt）は、スキャン中のメモリリークを防ぐために重要です。
