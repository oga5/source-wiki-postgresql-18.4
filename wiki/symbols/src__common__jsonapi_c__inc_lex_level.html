inc_lex_level

定義

  • 種別: function
  • 定義箇所: src/common/jsonapi.c:561

目的

JSON解析における現在の字句解析レベル（ネスト深度）をインクリメントし、インクリメンタル解析時には必要に応じて解析スタックの領域を拡張します。

シグネチャまたは形状

static inline bool inc_lex_level(JsonLexContext *lex)

入力と出力

  • **入力**: JsonLexContext *lex - 解析状態を保持するコンテキスト構造体。
  • **出力**: bool - 成功した場合は true を返し、メモリ不足等でスタック拡張に失敗した場合は false を返します。

前提条件と不変条件

  • lex は有効な JsonLexContext を指している必要があります。
  • インクリメンタル解析モード（lex->incremental）の場合、lex->pstack が初期化されている必要があります。
  • lex->lex_level は常に現在のネスト深度を正しく反映している必要があります。

アルゴリズムまたは振る舞い

1. **スタック拡張（インクリメンタルモード時）**:

  • 現在のレベルがスタック容量の限界に達している場合、JS_STACK_CHUNK_SIZE 分だけスタックサイズを拡張します。
  • prediction、fnames、fnull の各配列を REALLOC で再割り当てします。
  • JSONAPI_USE_PQEXPBUFFER が定義されている環境では、再割り当て失敗時に false を返します。

    • 2. **レベル更新**:

  • lex->lex_level を 1 増加させます。

    • 3. **初期化（インクリメンタルモード時）**:

  • 新しいレベルの fnames[lex->lex_level] を NULL に設定し、後続のメモリ解放処理（freeJsonLexContext）の安全性を確保します。

    • 4. **戻り値**: 常に true を返します（メモリ割り当て失敗時を除く）。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/jsonapi.c 内の解析ロジック（行962、993など）。
  • **呼び出し先**: REALLOC マクロ（環境に応じて palloc または malloc に展開）。

関連データ構造

  • JsonLexContext: 解析状態を保持する主要構造体。
  • JsonParserStack: インクリメンタル解析時に使用されるスタック構造体。

失敗モードと境界条件

  • **メモリ不足**: REALLOC が失敗した場合、関数は false を返します。これは特にフロントエンド環境（JSONAPI_USE_PQEXPBUFFER 定義時）で発生し得ます。
  • **スタックオーバーフロー**: スタックサイズが size_t の上限に達した場合の挙動は、REALLOC の実装に依存します。

保守担当者向けメモ

  • 本関数は static inline で定義されており、パフォーマンスを重視しています。
