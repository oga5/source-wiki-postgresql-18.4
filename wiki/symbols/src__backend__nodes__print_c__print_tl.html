print_tl

定義

  • 種別: function
  • 定義箇所: src/backend/nodes/print.c:469

目的

クエリのターゲットリスト（TargetList）の内容を、デバッグ目的で人間が読みやすい形式で標準出力にダンプします。

シグネチャまたは形状

void print_tl(const List *tlist, const List *rtable)

入力と出力

  • **入力**:
  • tlist: 表示対象の TargetEntry ノードを含むリスト。
  • rtable: 式の表示時に参照されるレンジテーブル（RangeTblEntry のリスト）。
  • **出力**: なし（標準出力 stdout へ整形されたテキストを出力）。

前提条件と不変条件

  • tlist は TargetEntry ノードのリストである必要があります。
  • rtable は print_expr が式内の Var ノードを解決するために必要なコンテキストを提供します。

アルゴリズムまたは振る舞い

1. 開始記号 ( を出力します。 2. tlist の各要素（TargetEntry）を順に走査します。 3. 各 TargetEntry について、以下の情報を出力します。

  • resno（属性番号）
  • resname（属性名、NULLの場合は <null>）
  • ressortgroupref（ソート/グループ参照番号、存在しない場合は空欄）

    • 4. print_expr を呼び出し、TargetEntry 内の式（expr）を再帰的に表示します。 5. 最後に終了記号 ) を出力します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード上での直接的な呼び出し元は提供されていません。
  • **呼び出し先**:
  • print_expr: 各ターゲットエントリ内の式を表示するために呼び出されます。

関連データ構造

  • List: ノードのリスト構造。
  • TargetEntry: ターゲットリストの各要素を構成する構造体。
  • RangeTblEntry: rtable に含まれるテーブル定義情報。

失敗モードと境界条件

  • tlist が空の場合、単に ( と ) が出力されます。
  • tle->resname が NULL の場合、<null> という文字列で代替表示されます。

保守担当者向けメモ

  • 本関数はデバッグ専用です。本番環境のコードから呼び出さないでください。
  • rtable が適切でない場合、print_expr 内で Var ノードの解決に失敗する可能性があります。
