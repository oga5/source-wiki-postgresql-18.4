r_Step_5a

定義

  • 種別: function
  • 定義箇所: src/backend/snowball/libstemmer/stem_UTF_8_porter.c:514

目的

Porter Stemming アルゴリズムの最終段階（Step 5a）として、単語末尾の 'e' を条件付きで削除する。

シグネチャまたは形状

static int r_Step_5a(struct SN_env * z)

入力と出力

  • 入力: SN_env * z（ステミング環境の状態を保持する構造体）
  • 出力: 成功時に 1、条件不一致時に 0、エラー発生時に負の値を返す。

前提条件と不変条件

  • z->c が文字列の有効範囲内にあること。
  • z->p が UTF-8 文字列バッファを指していること。

アルゴリズムまたは振る舞い

1. 現在のカーソル位置（z->c）を ket に設定し、直前の文字が 'e' であるかを確認する。'e' でなければ終了（0 を返す）。 2. 'e' であればカーソルを一つ戻し、そこを bra とする。 3. 以下のいずれかの条件を満たす場合に slice_del を実行して 'e' を削除する：

  • r_R2(z) が真である場合。
  • r_R1(z) が真であり、かつ r_shortv(z)（短母音判定）が偽である場合。

    • 4. 削除処理が成功すれば 1 を返し、失敗または条件不一致の場合は元の状態を維持して終了する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/snowball/libstemmer/stem_UTF_8_porter.c および src/backend/snowball/libstemmer/stem_ISO_8859_1_porter.c 内のメイン処理フロー。
  • 呼び出し先: r_R2, r_R1, r_shortv, slice_del。

関連データ構造

  • SN_env: ステミング環境（カーソル位置、領域境界、文字列バッファ）を管理する構造体。

失敗モードと境界条件

  • z->c <= z->lb: 文字列の先頭に達している場合は削除を行わない。
  • slice_del 等の内部関数が負の値を返した場合、そのエラーコードをそのまま伝播させる。

保守担当者向けメモ

  • 本関数は Snowball コンパイラにより自動生成されたコードである。ロジックの変更が必要な場合は、直接編集せずソースとなる .sbl ファイルを修正すること。
  • r_R1, r_R2 の領域判定ロジックに依存しているため、それらの定義変更が本関数の挙動に直接影響する。
