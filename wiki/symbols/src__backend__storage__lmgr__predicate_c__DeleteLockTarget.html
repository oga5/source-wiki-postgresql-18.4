DeleteLockTarget

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/predicate.c:2668

目的

指定された述語ロック対象（PREDICATELOCKTARGET）に関連付けられたすべての述語ロックを削除し、対象自体が不要であればメモリから解放する。

シグネチャまたは形状

static void DeleteLockTarget(PREDICATELOCKTARGET *target, uint32 targettaghash)

入力と出力

  • **入力**:
  • target: 削除対象となる述語ロックターゲットへのポインタ。
  • targettaghash: 対象のハッシュ値。
  • **出力**: なし（副作用として共有メモリ上のハッシュテーブルからエントリを削除）。

前提条件と不変条件

  • SerializablePredicateListLock が排他モード（LW_EXCLUSIVE）で保持されていること。
  • PredicateLockHashPartitionLock(targettaghash) が保持されていること。

アルゴリズムまたは振る舞い

1. SerializableXactHashLock を排他モードで取得する。 2. 対象ターゲットの predicateLocks リストを反復処理し、各 PREDICATELOCK を以下の手順で削除する。

  • トランザクション側のリンク（xactLink）およびターゲット側のリンク（targetLink）から削除する。
  • PredicateLockHash から該当するロックエントリを検索し、HASH_REMOVE で削除する。

    • 3. SerializableXactHashLock を解放する。 4. RemoveTargetIfNoLongerUsed を呼び出し、ターゲット自体が不要であれば削除を試みる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/lmgr/predicate.c 内の複数箇所（行 471, 2854 など）。
  • **呼び出し先**:
  • RemoveTargetIfNoLongerUsed
  • hash_search_with_hash_value（PredicateLockHash に対して実行）
  • LWLockAcquire / LWLockRelease

関連データ構造

  • PREDICATELOCKTARGET: 述語ロックの対象（ページやリレーション等）。
  • PREDICATELOCK: 述語ロックの実体。
  • PredicateLockHash: 述語ロックを管理するハッシュテーブル。

失敗モードと境界条件

  • Assert により、ロックが正しく保持されていない場合に異常終了する。
  • hash_search_with_hash_value でエントリが見つからない場合、Assert(found) により異常終了する。

保守担当者向けメモ

  • 本関数は述語ロックのクリーンアップにおいて重要な役割を果たす。ロックの削除順序（リストからの切り離しとハッシュテーブルからの削除）は、メモリ整合性を維持するために厳密である必要がある。
  • SerializableXactHashLock の取得は、ロックの削除中にトランザクション状態との不整合を防ぐために必須である。
