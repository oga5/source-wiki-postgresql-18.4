CatCInProgress

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/cache/catcache.c:52

目的

キャッシュ検索処理中に、現在処理中のエントリを追跡するためのスタック構造体です。再帰的な検索や無効化イベント発生時に、処理中の状態を保持するために使用されます。

シグネチャまたは形状

typedef struct CatCInProgress
{
    CatCache   *cache;          /* 属するキャッシュ */
    uint32      hash_value;     /* エントリのハッシュ値（リストの場合は無視） */
    bool        list;           /* リストエントリかどうか */
    bool        dead;           /* 無効化された場合にセットされる */
    struct CatCInProgress *next; /* スタックの次要素 */
} CatCInProgress;

入力と出力

本構造体は関数ではなくデータ型です。検索処理の開始時にスタックへプッシュされ、終了時にポップされます。

前提条件と不変条件

  • catcache_in_progress_stack を起点とするリンクリストとして管理されます。
  • キャッシュの無効化処理（CatCacheInvalidate 等）において、このスタックを走査することで、現在処理中のエントリが削除対象であるかを判定します。

アルゴリズムまたは振る舞い

1. **登録**: キャッシュ検索関数（SearchCatCache 等）の開始時に、ローカル変数として CatCInProgress が作成され、グローバルなスタック catcache_in_progress_stack に追加されます。 2. **追跡**: キャッシュの無効化処理中に、スタック上のエントリとハッシュ値が一致した場合、そのエントリの dead フラグが true に設定されます。 3. **解除**: 検索処理の終了時に、スタックから該当エントリが取り除かれます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/cache/catcache.c 内の検索および無効化関連関数（SearchCatCache 等）が、この構造体をローカル変数としてスタックに積みます。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。

関連データ構造

  • CatCache: キャッシュ管理のメイン構造体。
  • catcache_in_progress_stack: CatCInProgress を要素とするグローバルなスタックの先頭ポインタ。

失敗モードと境界条件

  • **無効化の競合**: 検索中に無効化が発生した場合、dead フラグが立てられます。このフラグが立っているエントリは、処理終了後に適切に破棄される必要があります。
  • **スタックの不整合**: 検索処理が異常終了した場合、スタックのポインタが正しく復元されないリスクがあります。

保守担当者向けメモ

  • 本構造体は catcache.c 内の静的変数 catcache_in_progress_stack を介して管理されています。
  • 検索処理のホットパスに関わるため、スタック操作のオーバーヘッドを最小限に抑える設計となっています。
  • 参照カウント管理と密接に関連しており、dead フラグが立ったエントリのライフサイクル管理には注意が必要です。
