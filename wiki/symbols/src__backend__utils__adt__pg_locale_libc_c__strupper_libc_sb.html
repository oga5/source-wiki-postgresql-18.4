strupper_libc_sb

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale_libc.c:338

目的

シングルバイト文字エンコーディング環境において、OSのlibcロケール設定に基づき文字列を大文字に変換する。

シグネチャまたは形状

static size_t strupper_libc_sb(char *dest, size_t destsize, const char *src, ssize_t srclen, pg_locale_t locale)

入力と出力

  • dest: 変換後の文字列を格納するバッファ。
  • destsize: dest バッファのサイズ。
  • src: 変換対象のソース文字列。
  • srclen: ソース文字列の長さ（負の場合は strlen(src) が使用される）。
  • locale: ロケール情報を含む pg_locale_t 構造体。
  • 戻り値: 処理された文字列の長さ（srclen）。

前提条件と不変条件

  • srclen + 1 <= destsize を満たす場合のみ変換が実行される。
  • locale が指す構造体は有効である必要がある。
  • locale->is_default が真の場合、PostgreSQL独自のASCII変換（pg_toupper）が適用される。
  • locale->is_default が偽の場合、locale->info.lt を使用した toupper_l が適用される。

アルゴリズムまたは振る舞い

1. srclen が負の場合、strlen(src) で長さを算出する。 2. バッファサイズが十分な場合、memcpy で src を dest にコピーし、終端文字を付与する。 3. 文字列内の各文字に対してループ処理を行う：

  • デフォルトロケールであれば pg_toupper を適用。
  • それ以外であれば toupper_l を使用してロケール固有の大文字変換を行う。

    • 4. 最終的に srclen を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/adt/pg_locale_libc.c 内の関数（直接の参照として行81および149が確認されている）。
  • 呼び出し先: strlen, memcpy, pg_toupper, toupper_l。

関連データ構造

  • pg_locale_t: ロケール情報を保持する構造体。
  • locale_t: OS標準のロケールハンドル（locale->info.lt）。

失敗モードと境界条件

  • srclen + 1 > destsize: バッファ不足時は変換処理をスキップし、何も書き込まずに srclen を返す。
  • srclen が負の入力: strlen に依存するため、src がNUL終端されていない場合は未定義動作となる可能性がある。

保守担当者向けメモ

  • 本関数はシングルバイト専用である。マルチバイトエンコーディングの場合は strupper_libc_mb が使用される。
  • toupper_l の挙動はプラットフォームのlibc実装に依存する。
  • islower_l によるガードは行わず、toupper_l がそのまま適用される設計となっている。
