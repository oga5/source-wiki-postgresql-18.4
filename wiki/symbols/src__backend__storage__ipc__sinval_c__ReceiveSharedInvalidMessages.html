ReceiveSharedInvalidMessages

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/storage/ipc/sinval.c:68

目的

共有メモリ上のキャッシュ無効化（SI）メッセージキューからメッセージを読み込み、各バックエンドのローカルキャッシュを同期させる。

シグネチャまたは形状

void ReceiveSharedInvalidMessages(void (*invalFunction) (SharedInvalidationMessage *msg),
                                  void (*resetFunction) (void))

入力と出力

  • **入力**:
  • invalFunction: 個別の無効化メッセージを処理するコールバック関数。
  • resetFunction: キャッシュの全リセットが必要な場合に呼び出されるコールバック関数。
  • **出力**: なし（副作用としてキャッシュの更新および共有メモリのクリーンアップを行う）。

前提条件と不変条件

  • 静的バッファ messages を使用し、再帰的な呼び出しが発生してもスタック上のメッセージを安全に処理する。
  • SharedInvalidMessageCounter が処理ごとにインクリメントされる。

アルゴリズムまたは振る舞い

1. **再帰処理の継続**: 前回の呼び出しで処理しきれなかったメッセージがある場合、まずそれを処理する。 2. **メッセージ取得**: SIGetDataEntries を呼び出し、共有メモリからメッセージを取得する。 3. **リセット処理**: SIGetDataEntries が負の値を返した場合、resetFunction を実行して終了する。 4. **メッセージ適用**: 取得したメッセージをループで順次 invalFunction に渡す。 5. **ループ継続**: バッファが満杯（MAXINVALMSGS）の場合、追加のメッセージが存在する可能性があるため、取得処理を繰り返す。 6. **キャッチアップ完了**: catchupInterruptPending が真の場合、フラグを解除し SICleanupQueue を呼び出して、次のバックエンドへキャッチアップ信号を伝播させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/cache/inval.c:938 (LocalExecuteInvalidationMessage を引数として呼び出し)
  • **呼び出し先**:
  • SIGetDataEntries (共有メモリからのメッセージ取得)
  • invalFunction (引数で渡されたコールバック)
  • resetFunction (引数で渡されたコールバック)
  • SICleanupQueue (キューの整理)

関連データ構造

  • SharedInvalidationMessage: 処理対象のメッセージ構造体。
  • MAXINVALMSGS: 一度に処理するメッセージの最大数（32）。
  • SharedInvalidMessageCounter: 処理されたメッセージの総数。
  • catchupInterruptPending: キャッチアップが必要かを示すフラグ。

失敗モードと境界条件

  • **キューオーバーフロー**: SIGetDataEntries が MAXINVALMSGS を返した場合、ループにより全メッセージの処理を試みる。
  • **リセット要求**: SIGetDataEntries が負の値を返すと、個別のメッセージ処理を中断し、キャッシュ全体のリセットを実行する。

保守担当者向けメモ

  • nextmsg と nummsgs は volatile 宣言されており、再帰的な呼び出しによる状態破壊を防ぐ設計となっている。
  • キャッチアップ信号の伝播（デイジーチェーン）は、システム負荷の急増を避けるための意図的な設計である。
