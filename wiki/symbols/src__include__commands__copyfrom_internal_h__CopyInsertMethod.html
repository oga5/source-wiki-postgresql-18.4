CopyInsertMethod

定義

  • 種別: enum
  • 定義箇所: src/include/commands/copyfrom_internal.h:46

目的

COPY FROM コマンド実行時における、テーブルへのデータ挿入戦略（単一行挿入かバッチ挿入か）を決定・管理するために使用されます。

シグネチャまたは形状

typedef enum CopyInsertMethod
{
    CIM_SINGLE,
    CIM_MULTI,
    CIM_MULTI_CONDITIONAL,
} CopyInsertMethod;

入力と出力

  • 本シンボルは列挙型であり、関数ではありません。
  • CopyFromStateData 構造体内で、現在の挿入モードを保持する変数として利用されます。

前提条件と不変条件

  • 挿入戦略の選択は、対象テーブルの特性や実行環境の制約に基づきます。
  • CIM_MULTI および CIM_MULTI_CONDITIONAL は、バッチ挿入関数（table_multi_insert または ExecForeignBatchInsert）の利用を前提とします。

アルゴリズムまたは振る舞い

  • CIM_SINGLE: 常に単一行挿入関数（table_tuple_insert または ExecForeignInsert）を使用します。
  • CIM_MULTI: 常にバッチ挿入関数を使用します。
  • CIM_MULTI_CONDITIONAL: バッチ挿入が有効な場合に限り、バッチ挿入関数を使用します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/copyfrom.c にて CopyInsertMethod 型の変数が定義されており、挿入ロジックの制御に使用されています。
  • 呼び出し先: 本シンボル自体は列挙型であり、直接的な呼び出し先はありません。

関連データ構造

  • CopyFromStateData: この列挙型をメンバとして保持し、COPY FROM の実行状態を管理します。

失敗モードと境界条件

  • 適切な挿入メソッドが選択されていない場合、テーブルの制約や外部テーブルの API 仕様により挿入処理が失敗する可能性があります。

保守担当者向けメモ

  • 本列挙型は COPY FROM のパフォーマンス最適化に直結します。新しい挿入戦略を追加する際は、copyfrom.c 内の挿入ロジックとの整合性を確認してください。
