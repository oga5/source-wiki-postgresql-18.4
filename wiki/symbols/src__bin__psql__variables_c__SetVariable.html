SetVariable

定義

  • 種別: function
  • 定義箇所: src/bin/psql/variables.c:280

目的

psql の変数管理システムにおいて、指定された名前の変数を設定、更新、または削除します。

シグネチャまたは形状

bool SetVariable(VariableSpace space, const char *name, const char *value);

入力と出力

  • **入力**:
  • space: 変数を保持する VariableSpace 構造体。
  • name: 設定する変数名。
  • value: 設定する値（文字列）。NULL の場合は変数の削除を意味します。
  • **出力**:
  • bool: 処理が成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • space および name は NULL であってはなりません。
  • 変数名は valid_variable_name の規則に従う必要があります。
  • 変数リストは名前順（strcmp 順）にソートされた単方向連結リストとして保持されます。
  • value が NULL の場合、存在しない変数を削除しようとしてもエラーにはならず true を返します。

アルゴリズムまたは振る舞い

1. **バリデーション**: 変数名が不正な場合、エラーログを出力して false を返します。 2. **検索**: リストを走査し、指定された名前の変数を探索します。 3. **更新**:

  • 変数が存在する場合、substitute_hook（値の正規化）および assign_hook（値の妥当性検証）を順に実行します。
  • フックが成功（confirmed）した場合のみ値を更新し、フックがない場合は即座に更新します。
  • 値が NULL に更新され、かつフックが登録されていない場合は、リストからノードを削除します。

    • 4. **新規作成**: 変数が存在しない場合、value が NULL でなければ新しいノードを作成し、リストの適切な位置（名前順）に挿入します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/psql/command.c 内の各コマンド処理（\set 等）から頻繁に呼び出されます。
  • **呼び出し先**:
  • valid_variable_name（変数名の検証）
  • pg_strdup, pg_malloc, pg_free（メモリ管理）
  • current->substitute_hook / current->assign_hook（登録されている場合）

関連データ構造

  • VariableSpace: 変数リストのヘッダー。
  • struct _variable: 変数名、値、フック関数、次ノードへのポインタを保持する内部構造体。

失敗モードと境界条件

  • space または name が NULL の場合: false を返します。
  • valid_variable_name が不正な場合: エラーログを出力し false を返します。
  • assign_hook が false を返した場合: 値の更新は拒否され、false を返します。
  • メモリ不足: pg_malloc 等が失敗した場合の挙動は、標準的な psql のメモリ管理方針に従います。

保守担当者向けメモ

  • value は内部で複製されるため、呼び出し側で渡した文字列の寿命を気にする必要はありません。
  • 代入フックが失敗した場合、current->value は変更されずに維持されます。
  • 変数リストの順序は strcmp に依存しているため、挿入位置のロジックを変更する際は注意してください。
