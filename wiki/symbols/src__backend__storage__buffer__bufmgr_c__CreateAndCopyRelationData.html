CreateAndCopyRelationData

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/bufmgr.c:5239

目的

既存のリレーション（ソース）の全フォークを新しいリレーション（宛先）へ複製する。データベース作成時などの物理的なデータコピー処理を担う。

シグネチャまたは形状

void CreateAndCopyRelationData(RelFileLocator src_rlocator, 
                                RelFileLocator dst_rlocator, 
                                bool permanent)

入力と出力

  • **入力**:
  • src_rlocator: コピー元リレーションの識別子。
  • dst_rlocator: コピー先リレーションの識別子。
  • permanent: 永続的なリレーションか否かを示すフラグ（true の場合 RELPERSISTENCE_PERMANENT、false の場合 RELPERSISTENCE_UNLOGGED として扱う）。
  • **出力**: なし（副作用としてストレージ上に新しいリレーションが作成され、データがコピーされる）。

前提条件と不変条件

  • src_rlocator および dst_rlocator は有効なリレーション識別子であること。
  • permanent が true の場合、または forkNum が INIT_FORKNUM の場合、WAL ログへの記録が行われる。

アルゴリズムまたは振る舞い

1. permanent フラグに基づき、リレーションの永続性（relpersistence）を決定する。 2. smgropen を使用してソースおよび宛先のストレージマネージャハンドルを取得する。 3. RelationCreateStorage を呼び出し、宛先リレーションのストレージを初期化する。 4. RelationCopyStorageUsingBuffer を使用して MAIN_FORKNUM のデータをコピーする。 5. MAIN_FORKNUM を超える全フォークについてループ処理を行う：

  • smgrexists でソース側にフォークが存在するか確認する。
  • 存在する場合、smgrcreate で宛先にフォークを作成する。
  • 必要に応じて log_smgrcreate で WAL ログを記録する。
  • RelationCopyStorageUsingBuffer を使用してブロック単位でデータをコピーする。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/dbcommands.c:216
  • **呼び出し先**:
  • smgropen, smgrexists, smgrcreate (ストレージマネージャ)
  • RelationCreateStorage
  • RelationCopyStorageUsingBuffer
  • log_smgrcreate

関連データ構造

  • RelFileLocator: リレーションの物理的な場所を特定する構造体。
  • SMgrRelation: ストレージマネージャの管理構造体。
  • ForkNumber: リレーションのフォーク（データ、空き領域マップなど）を識別する列挙型。

失敗モードと境界条件

  • ソースリレーションのフォークが存在しない場合、そのフォークのコピーはスキップされる。
  • データベース作成処理の文脈では、個別のリレーションに対するクリーンアップ登録は行われない（データベース単位のクリーンアップ機構に依存するため）。

保守担当者向けメモ

  • 本関数はブロック単位のコピーに RelationCopyStorageUsingBuffer を利用しており、バッファマネージャを介したデータ移動を行う。
  • permanent フラグの扱いは WAL ログの生成条件に直結するため、変更時はログ出力要件を再確認すること。
