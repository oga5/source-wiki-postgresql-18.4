pg_SSPI_continue

定義

  • 種別: function
  • 定義箇所: src/interfaces/libpq/fe-auth.c:225

目的

SSPI（Security Support Provider Interface）認証プロトコルの継続処理を実行し、サーバーからの認証トークンを処理して、必要に応じて応答を返します。

シグネチャまたは形状

static int pg_SSPI_continue(PGconn *conn, int payloadlen)

入力と出力

  • **入力**:
  • conn: 接続状態を保持する PGconn 構造体。
  • payloadlen: サーバーから受信した認証ペイロードのバイト数。
  • **出力**:
  • STATUS_OK (0): 処理が正常に完了した場合。
  • STATUS_ERROR (-1): メモリ確保失敗、SSPIエラー、または通信エラーが発生した場合。

前提条件と不変条件

  • conn->sspictx が NULL の場合は初回実行とみなされ、コンテキストの初期化が行われます。
  • payloadlen は、サーバーからのメッセージ全体が既に conn の入力バッファに存在していることを前提としています。
  • 認証が完了すると conn->client_finished_auth が true に設定されます。

アルゴリズムまたは振る舞い

1. **入力バッファの準備**: 初回実行時以外は、payloadlen 分のメモリを確保し、pqGetnchar でサーバーからのトークンを読み込みます。 2. **SSPIコンテキストの更新**: InitializeSecurityContext を呼び出し、SSPI認証状態を更新します。 3. **コンテキストの保存**: 初回実行時は、生成された newContext を conn->sspictx にコピーして保持します。 4. **応答の送信**: SSPIが生成した出力バッファ（outbuf）にデータがある場合、pqPacketSend を使用して PqMsg_GSSResponse としてサーバーへ送信します。 5. **終了判定**: SSPIの戻り値が SEC_E_OK であれば、認証完了フラグを立てます。 6. **クリーンアップ**: 使用した一時バッファを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/libpq/fe-auth.c 内の認証フロー制御ロジックから呼び出されます。
  • **呼び出し先**:
  • InitializeSecurityContext (Windows SSPI API)
  • pqGetnchar, pqPacketSend, libpq_append_conn_error (libpq 内部関数)
  • malloc, free, memcpy (標準ライブラリ)

関連データ構造

  • PGconn: 接続情報および認証状態（sspictx, sspicred, sspitarget 等）を保持。
  • SecBufferDesc, SecBuffer: SSPIで使用されるバッファ構造体。
  • CtxtHandle: SSPIのセキュリティコンテキストハンドル。

失敗モードと境界条件

  • **メモリ不足**: malloc が失敗した場合、エラーメッセージを記録して STATUS_ERROR を返します。
  • **SSPIエラー**: InitializeSecurityContext が SEC_E_OK または SEC_I_CONTINUE_NEEDED 以外を返した場合、pg_SSPI_error を経由してエラーを報告します。
  • **不正なバッファ数**: SSPIが期待しない数の出力バッファを返した場合、エラーとして処理されます。

保守担当者向けメモ

  • 本関数はWindows環境のSSPI認証に特化しています。
  • conn->sspictx のメモリ管理は freePGconn() に依存しているため、本関数内ではコンテキストの初期化と更新のみに注力しています。
  • 認証完了後のデータ送信は、サーバーが期待している場合にのみ行われるよう制御されています。
