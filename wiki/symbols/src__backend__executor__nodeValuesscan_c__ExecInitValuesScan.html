ExecInitValuesScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeValuesscan.c:209

目的

VALUES 句をスキャンするための ValuesScanState ノードを初期化し、実行に必要な式リストの準備とメモリコンテキストの割り当てを行います。

シグネチャまたは形状

ValuesScanState *ExecInitValuesScan(ValuesScan *node, EState *estate, int eflags)

入力と出力

  • **入力**:
  • node: 初期化対象の ValuesScan プランノード。
  • estate: クエリ実行状態を保持する EState。
  • eflags: 実行フラグ。
  • **出力**: 初期化済みの ValuesScanState 構造体へのポインタ。

前提条件と不変条件

  • ValuesScan ノードは子プランを持たないこと（outerPlan および innerPlan が NULL であること）。
  • VALUES 句内の各行は、スキャン時に仮想タプルとして評価・提供される。

アルゴリズムまたは振る舞い

1. ValuesScanState を割り当て、ExecValuesScan を実行関数として設定します。 2. ExecAssignExprContext を使用して、行処理用およびスキャン用の式コンテキストを生成します。 3. ExecTypeFromExprList でタプル記述子を生成し、仮想タプル用のスロットを初期化します。 4. ExecInitResultTypeTL および ExecAssignScanProjectionInfo で結果型と射影情報を設定します。 5. node->values_lists を走査し、各行の式リストを配列に格納します。 6. SubPlan を含む式リストについては、ExecInitExprList を呼び出して個別に初期化します。この際、JITコンパイルを一時的に無効化し、オーバーヘッドを抑制します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c 内のプランノード初期化処理から呼び出されます。
  • **呼び出し先**: makeNode, ExecAssignExprContext, ExecTypeFromExprList, ExecInitScanTupleSlot, ExecInitResultTypeTL, ExecAssignScanProjectionInfo, ExecInitQual, ExecInitExprList, contain_subplans 等。

関連データ構造

  • ValuesScan: プランノード。
  • ValuesScanState: 実行時の状態保持構造体。
  • EState: 実行状態。
  • TupleTableSlot: 仮想タプルを格納するスロット。

失敗モードと境界条件

  • VALUES 句が空の場合や、式リストの解析に失敗した場合の挙動は、基盤となる ExecInitExprList 等の依存関数に準拠します。
  • SubPlan を含む複雑な式がある場合、メモリリークやポインタの不整合を防ぐため、明示的な初期化ロジックが適用されます。

保守担当者向けメモ

  • VALUES 句の式は一度しか評価されないため、JITコンパイルを無効化する設計となっています。
  • SubPlan の接続問題を避けるため、contain_subplans を用いた条件付き初期化を行っています。このロジックを変更する際は、EXPLAIN 出力やリソース管理への影響に注意してください。
