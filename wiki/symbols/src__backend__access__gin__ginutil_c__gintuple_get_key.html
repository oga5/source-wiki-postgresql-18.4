gintuple_get_key

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginutil.c:263

目的

GINインデックスのタプルから、格納されているキー値（Datum）と、その値がNULLであるか否かを示すカテゴリ情報を抽出します。

シグネチャまたは形状

Datum gintuple_get_key(GinState *ginstate, IndexTuple tuple, GinNullCategory *category)

入力と出力

  • **入力**:
  • ginstate: GINインデックスの実行時状態を保持する構造体。
  • tuple: 抽出対象の IndexTuple。
  • category: 出力用ポインタ。NULLカテゴリ情報を格納する。
  • **出力**:
  • 戻り値: タプルから抽出されたキー値（Datum）。

前提条件と不変条件

  • ginstate が有効な GinState 構造体を指していること。
  • tuple が有効な IndexTuple であること。
  • category が結果を書き込むための有効なメモリ領域を指していること。

アルゴリズムまたは振る舞い

1. ginstate->oneCol が真（単一列インデックス）の場合、タプルの最初の属性から値を抽出します。 2. ginstate->oneCol が偽（複数列インデックス）の場合、gintuple_get_key を用いて属性番号を特定し、対応する列のタプル記述子を使用して値を抽出します。 3. 抽出した値がNULLであれば、GinGetNullCategory を用いてNULLカテゴリを判定し、category に格納します。 4. NULLでなければ、category に GIN_CAT_NORM_KEY を設定します。 5. 抽出した Datum を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • src/backend/access/gin/ginentrypage.c
  • src/backend/access/gin/ginfast.c

  • **呼び出し先**:

関連データ構造

  • GinState: インデックスの演算クラスや記述子を保持する構造体。
  • IndexTuple: GINインデックス内の物理的なタプル形式。
  • GinNullCategory: NULL値の種別を識別する列挙型。

失敗モードと境界条件

  • tuple が不正な場合や、ginstate 内の記述子とタプルの構造が一致しない場合、index_getattr 経由で予期せぬ動作やエラーが発生する可能性があります。

保守担当者向けメモ

  • 複数列インデックスの場合、属性番号の特定に gintuple_get_attrnum を使用するため、インデックス定義とタプル構造の整合性が重要です。
  • ginstate->oneCol による分岐は、インデックスの構成に依存するため、変更時は注意が必要です。
