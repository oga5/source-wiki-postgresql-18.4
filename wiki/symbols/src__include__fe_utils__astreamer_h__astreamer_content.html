astreamer_content

定義

  • 種別: function
  • 定義箇所: src/include/fe_utils/astreamer.h:136

目的

アーカイブストリームのパイプラインにおいて、現在のストリームコンポーネントに対してデータチャンクを供給し、処理を委譲する。

シグネチャまたは形状

static inline void astreamer_content(astreamer *streamer, astreamer_member *member, const char *data, int len, astreamer_archive_context context)

入力と出力

  • streamer: 処理を実行する astreamer オブジェクト。
  • member: 現在処理中のアーカイブメンバーに関するメタデータ（ファイルパス等）。NULL の場合もある。
  • data: 処理対象のデータチャンク。
  • len: data のバイト長。
  • context: アーカイブ内の現在の位置（ヘッダー、コンテンツ、トレーラー等）を示す astreamer_archive_context。

出力なし（void）。

前提条件と不変条件

  • streamer は NULL であってはならない（Assert により検証）。
  • streamer->bbs_ops が有効であり、content 関数ポインタが実装されている必要がある。

アルゴリズムまたは振る舞い

1. streamer が NULL でないことを確認する。 2. streamer に関連付けられた bbs_ops（操作関数群）の content コールバックを呼び出し、引数として渡された member、data、len、context をそのまま引き渡す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_basebackup/astreamer_inject.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_verifybackup/pg_verifybackup.c
  • src/fe_utils/astreamer_gzip.c
  • src/fe_utils/astreamer_lz4.c
  • **呼び出し先**:
  • streamer->bbs_ops->content（各ストリーム実装固有の関数）

関連データ構造

  • astreamer: ストリーム処理の基底構造体。
  • astreamer_member: アーカイブメンバーのメタデータ構造体。
  • astreamer_archive_context: アーカイブ内のコンテキストを示す列挙型。
  • astreamer_ops: content コールバックを定義する構造体。

失敗モードと境界条件

  • streamer が NULL の場合、アサーション違反によりプログラムが異常終了する可能性がある（デバッグビルド時）。
  • bbs_ops が適切に初期化されていない場合、セグメンテーション違反が発生する。

保守担当者向けメモ

  • 本関数はインライン関数であり、パイプラインの各ステージを透過的に連結するために使用される。
  • 各ストリーム実装の content 関数内では、処理後に必要に応じて bbs_next を介して後続のストリームへデータを転送する設計となっている。
