MessageDLL

定義

  • 種別: struct
  • 定義箇所: src/interfaces/libpq/win32.c:234

目的

Windows環境において、FormatMessage API を用いてシステムエラーメッセージを取得するために必要なDLL情報を保持する構造体です。

シグネチャまたは形状

struct MessageDLL {
    const char *dll_name;
    void       *handle;
    int         loaded; /* BOOL */
};

入力と出力

本構造体はデータ保持用であり、直接的な関数入力・出力はありません。dlls 配列の要素として使用され、各フィールドは以下の役割を持ちます。

  • dll_name: ロード対象のDLL名（文字列）。
  • handle: LoadLibraryEx によって取得されたDLLのハンドル。
  • loaded: ロード状態を示すフラグ（BOOL）。

前提条件と不変条件

  • dlls 配列の要素として定義され、src/interfaces/libpq/win32.c 内で管理されます。
  • handle は LoadLibraryEx の戻り値であり、プロセス終了まで保持されることが想定されています。

アルゴリズムまたは振る舞い

winsock_strerror 関数がエラーメッセージを解決する際、この構造体のリストを順次参照します。loaded フラグが未設定の場合、LoadLibraryEx を呼び出してDLLをロードし、成功すれば handle に格納します。

呼び出し元と呼び出し先

  • 呼び出し元: src/interfaces/libpq/win32.c 内の winsock_strerror がこの構造体の配列（dlls）を走査します。
  • 呼び出し先: LoadLibraryEx (Windows API) を使用してDLLをロードします。

関連データ構造

  • dlls: MessageDLL 型の静的配列。
  • winsock_strerror: 本構造体を利用してエラーメッセージを解決する関数。

失敗モードと境界条件

  • DLLのロードに失敗した場合、handle は無効となり、次のDLLの試行へ移行します。
  • dlls 配列のサイズは DLLS_SIZE マクロによって計算されます。

保守担当者向けメモ

  • 本構造体は libpq のWindows用エラー処理に特化しています。新しいDLLを検索対象に追加する場合は、dlls 配列を更新してください。
  • loaded フィールドは BOOL 型として扱われていますが、C言語の整数型として実装されています。
