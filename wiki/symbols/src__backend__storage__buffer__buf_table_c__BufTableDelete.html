BufTableDelete

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/buf_table.c:147

目的

共有バッファハッシュテーブル（SharedBufHash）から、指定された BufferTag に対応するエントリを削除します。

シグネチャまたは形状

void BufTableDelete(BufferTag *tagPtr, uint32 hashcode)

入力と出力

  • **入力**:
  • tagPtr: 削除対象のページを特定する BufferTag 構造体へのポインタ。
  • hashcode: tagPtr から事前に計算されたハッシュ値。
  • **出力**: なし（void）。

前提条件と不変条件

  • 呼び出し側は、対象のハッシュ値に対応する BufMappingLock を適切に保持している必要があります。本関数自体はロックの取得・解放を行いません。
  • 指定された tagPtr に対応するエントリがハッシュテーブル内に存在することが期待されます。

アルゴリズムまたは振る舞い

1. hash_search_with_hash_value を呼び出し、SharedBufHash に対して HASH_REMOVE アクションを実行します。 2. 削除操作の結果を確認し、エントリが見つからなかった場合は共有メモリ上のハッシュテーブルが破損していると判断し、elog(ERROR) を発生させて異常終了します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c（複数の箇所でバッファの置換や削除時に呼び出されます）
  • **呼び出し先**:
  • hash_search_with_hash_value（PostgreSQLの共有ハッシュテーブル操作関数）

関連データ構造

  • BufferTag: ページを一意に識別するキー。
  • SharedBufHash: バッファマッピングを保持する共有ハッシュテーブル。
  • BufferLookupEnt: ハッシュテーブル内のエントリ構造体。

失敗モードと境界条件

  • **破損検知**: 削除対象のエントリがハッシュテーブル内に存在しない場合、elog(ERROR) によりエラーとなります。これはシステムの一貫性が失われたことを意味します。

保守担当者向けメモ

  • 本関数はスレッドセーフではありません。呼び出し側での BufMappingLock の管理が必須です。
  • ハッシュ計算の重複を避けるため、呼び出し側で計算済みの hashcode を渡す設計になっています。
