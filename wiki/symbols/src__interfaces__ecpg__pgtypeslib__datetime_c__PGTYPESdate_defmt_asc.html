PGTYPESdate_defmt_asc

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/datetime.c:329

目的

指定されたフォーマット文字列に従って、ASCII文字列を解析し、date 型の変数へ値を復元します。

シグネチャまたは形状

int PGTYPESdate_defmt_asc(date *d, const char *fmt, const char *str);

入力と出力

  • **入力**:
  • d: 結果を格納する date 型へのポインタ。
  • fmt: 日付の構成要素（yy, mm, dd）を含むフォーマット文字列。
  • str: 解析対象の日付文字列。
  • **出力**:
  • 成功時は 0 を返します。
  • 失敗時は -1 を返し、errno にエラーコードを設定します。

前提条件と不変条件

  • d, fmt, str はすべて非 NULL である必要があります。
  • fmt には yy, mm, dd の各文字列が少なくとも1回ずつ含まれている必要があります。
  • 入力文字列が数字のみの場合、長さは 6 または 8 である必要があります。

アルゴリズムまたは振る舞い

1. **フォーマット解析**: fmt 内の yy, mm, dd の出現順序を特定し、トークンの順序（例: ymd, dmy）を決定します。 2. **文字列の正規化**:

  • 入力 str が数字のみの場合、長さ（6または8）に応じて各要素を抽出し、スペースで区切った形式に変換します。
  • それ以外の場合は、入力文字列をコピーします。

    • 3. **トークン化と変換**: 内部で tm 構造体を使用し、フォーマット順序に従って文字列を解析・割り当てます。最終的に date 型へ変換します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/compatlib/informix.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-dt_test.c
  • **呼び出し先**:
  • pgtypes_alloc (メモリ確保)
  • pgtypes_strdup (文字列コピー)
  • strstr, strlen, strncpy, isdigit (標準Cライブラリ)

関連データ構造

  • date: ECPG内で使用される日付型（内部的には整数値）。
  • struct tm: 日付要素を保持するための中間構造体。

失敗モードと境界条件

  • 引数が NULL の場合: PGTYPES_DATE_ERR_EARGS を設定。
  • フォーマット文字列に yy, mm, dd が不足している場合: PGTYPES_DATE_ERR_EARGS を設定。
  • 数字のみの入力で長さが 6 または 8 以外の場合: PGTYPES_DATE_ERR_ENOSHORTDATE を設定。
  • メモリ確保失敗時は -1 を返します。

保守担当者向けメモ

  • 本関数は非常に複雑な文字列解析ロジックを含んでおり、デリミタの有無や月名の省略形など、多様な入力形式に対応しています。
  • メモリ割り当てには pgtypes_alloc が使用されており、呼び出し元で適切に解放する必要があります。
  • 2000年1月1日を基準としたオフセット値として日付を扱います。
