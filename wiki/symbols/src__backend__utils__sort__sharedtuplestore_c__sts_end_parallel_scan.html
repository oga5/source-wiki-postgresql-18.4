sts_end_parallel_scan

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/utils/sort/sharedtuplestore.c:280

目的

並列スキャン操作を終了し、SharedTuplestoreAccessor が保持している読み取り用ファイルリソースを解放します。

シグネチャまたは形状

void sts_end_parallel_scan(SharedTuplestoreAccessor *accessor)

入力と出力

  • **入力**: accessor (SharedTuplestoreAccessor *) - 操作対象のアクセサ構造体へのポインタ。
  • **出力**: なし。副作用として accessor->read_file がクローズされ、NULL に設定されます。

前提条件と不変条件

  • accessor は有効な SharedTuplestoreAccessor を指している必要があります。
  • 呼び出し後、accessor->read_file はクローズ済みとなり、再利用不可となります。

アルゴリズムまたは振る舞い

1. accessor->read_file が NULL でないかを確認します。 2. BufFileClose() を呼び出し、読み取り用の一時ファイルを閉じます。 3. accessor->read_file を NULL に設定し、リソース解放済みであることを示します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeHash.c (複数箇所)
  • src/backend/executor/nodeHashjoin.c
  • src/backend/utils/sort/sharedtuplestore.c:258
  • **呼び出し先**:
  • BufFileClose (storage/buffile.h)

関連データ構造

  • SharedTuplestoreAccessor: 並列スキャンの状態とファイルハンドルを保持する構造体。

失敗モードと境界条件

  • accessor->read_file が既に NULL の場合、関数は何も行わず安全に終了します。

保守担当者向けメモ

  • 現在の実装ではファイルクローズのみを行っています。コメントにある通り、SHARED_TUPLESTORE_SINGLE_PASS モードにおいて全スキャナ終了時にファイルを削除する機能の実装には、並列スキャナの参照カウンタ管理が必要です。
