writeListPage

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginfast.c:58

目的

GINインデックスのペンディングリストにおけるページ初期化およびタプルの書き込みを行い、その結果としてページ内の空き容量を返却します。

シグネチャまたは形状

int32 writeListPage(Relation index, Buffer buffer, IndexTuple *tuples, int32 ntuples, BlockNumber rightlink)

入力と出力

  • **入力**:
  • index: 対象のインデックスリレーション。
  • buffer: 書き込み対象のバッファ。
  • tuples: 書き込む IndexTuple の配列。
  • ntuples: 書き込むタプルの数。
  • rightlink: ページチェーンの次ページへのリンク（InvalidBlockNumber の場合は末尾ページとみなす）。
  • **出力**:
  • int32: 書き込み後のページ内の正確な空き容量（PageGetExactFreeSpace の結果）。

前提条件と不変条件

  • size <= BLCKSZ であること（ワークスペースへのコピー時にバッファサイズを超えないこと）。
  • GinInitBuffer により、対象バッファは GIN_LIST として初期化されます。
  • 呼び出し元は、対象のバッファに対して適切なロックを保持している必要があります。

アルゴリズムまたは振る舞い

1. START_CRIT_SECTION() を呼び出し、クリティカルセクションを開始します。 2. GinInitBuffer でページを GIN_LIST として初期化します。 3. 指定された tuples をページに追加し、同時に workspace にコピーします。 4. rightlink をページヘッダーの rightlink に設定します。 5. rightlink が InvalidBlockNumber の場合、GinPageSetFullRow を設定し、maxoff を 1 に初期化します。それ以外の場合は maxoff を 0 に設定します。 6. MarkBufferDirty(buffer) を実行します。 7. インデックスがWALを必要とする場合、ginxlogInsertListPage 構造体を用いてWALレコードを生成・登録し、XLogInsert を実行してLSNを更新します。 8. PageGetExactFreeSpace で空き容量を取得し、UnlockReleaseBuffer でバッファを解放してクリティカルセクションを終了します。

関連データ構造

  • IndexTuple: 書き込まれるインデックスデータ。
  • GinMetaPageData (間接的): GINインデックスのメタページ管理に関連。
  • ginxlogInsertListPage: WAL記録用の構造体。

失敗モードと境界条件

  • PageAddItem が InvalidOffsetNumber を返した場合、elog(ERROR) により処理を中断します。
  • size > BLCKSZ の場合、Assert により異常終了します。

保守担当者向けメモ

  • 本関数は ginHeapTupleFastInsert 等から呼び出され、ペンディングリストの物理的な書き込みを担います。
  • WAL記録の整合性は XLogInsert を通じて保証されており、クラッシュリカバリ時に再実行可能な設計となっています。
