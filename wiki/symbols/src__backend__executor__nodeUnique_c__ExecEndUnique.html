ExecEndUnique

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeUnique.c:167

目的

Unique ノードおよびそのサブノードの実行を終了し、リソースを解放する。

シグネチャまたは形状

void ExecEndUnique(UniqueState *node)

入力と出力

  • **入力**: UniqueState *node （終了対象となる Unique ノードの実行時状態構造体）
  • **出力**: なし（void）

前提条件と不変条件

  • 引数 node は有効な UniqueState 構造体を指している必要がある。
  • outerPlanState(node) が有効なサブプラン状態を保持していること。

アルゴリズムまたは振る舞い

1. outerPlanState(node) を介して、このノードの配下にあるサブプランの状態を取得する。 2. ExecEndNode を呼び出し、サブプランの終了処理（リソース解放等）を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:734 （ExecEndNode 経由での呼び出しが確認されている）
  • **呼び出し先**: ExecEndNode （executor/executor.h で定義される汎用的なノード終了関数）

関連データ構造

  • UniqueState: Unique ノードの実行時状態を保持する構造体。

失敗モードと境界条件

  • node が NULL の場合、あるいは node 内のサブプラン状態が不正な場合、ExecEndNode の内部でセグメンテーション違反が発生する可能性がある。

保守担当者向けメモ

  • 本関数は Unique ノード固有のクリーンアップ処理を拡張する必要が生じた場合に修正対象となる。現状はサブプランの終了処理を委譲するのみである。
