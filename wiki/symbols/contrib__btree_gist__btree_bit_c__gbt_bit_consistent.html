gbt_bit_consistent

定義

  • 種別: function
  • 定義箇所: contrib/btree_gist/btree_bit.c:135

目的

GiST インデックスにおける BIT 型および VARBIT 型の検索条件判定（Consistent）を行います。インデックス内のキーとクエリを比較し、条件に合致するかを判定します。

シグネチャまたは形状

Datum gbt_bit_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_POINTER(0): GISTENTRY * (インデックスエントリ)
  • PG_GETARG_DATUM(1): query (検索条件データ)
  • PG_GETARG_UINT16(2): strategy (比較演算子の戦略番号)
  • PG_GETARG_POINTER(4): recheck (再チェックフラグへのポインタ)
  • **出力**:
  • Datum: bool 型の判定結果（PG_RETURN_BOOL を使用）
  • *recheck: 常に false が設定されます。

前提条件と不変条件

  • 本関数は完全一致判定を行うため、GiST 検索後の再チェックは不要です（*recheck = false）。
  • tinfo（gbtree_vinfo 構造体）がモジュール内で定義されており、比較ロジックに使用されます。

アルゴリズムまたは振る舞い

1. recheck フラグを false に設定します。 2. インデックスノードがリーフノード（GIST_LEAF(entry)）の場合:

  • gbt_var_consistent を呼び出し、インデックス内のキーとクエリを直接比較します。

    • 3. リーフノード以外（内部ノード）の場合:

  • gbt_bit_xfrm を使用してクエリをインデックス格納に適した形式に変換します。
  • 変換後のクエリを用いて gbt_var_consistent を呼び出し、範囲判定を行います。

呼び出し元と呼び出し先

  • **呼び出し元**: GiST インデックスの検索エンジン（直接の呼び出し元コードはソース抜粋に含まれていません）。
  • **呼び出し先**:
  • gbt_var_consistent
  • gbt_bit_xfrm

関連データ構造

  • GISTENTRY: GiST インデックスのエントリ構造体。
  • GBT_VARKEY: 可変長キーの内部表現。
  • tinfo: gbtree_vinfo 構造体（比較関数や変換関数を保持）。

失敗モードと境界条件

  • 入力データが不正な形式である場合、内部で呼び出される gbt_var_consistent や gbt_bit_xfrm がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本関数は btree_utils_var.c で提供される汎用関数に依存しています。
  • recheck が常に false であるため、インデックスの検索結果に対して追加のフィルタリングは不要です。
