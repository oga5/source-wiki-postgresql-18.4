get_record1

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/preproc-outofscope.c:229

目的

カーソル mycur からデータをフェッチし、ECPGランタイムを介して構造体変数へデータを格納する。

シグネチャまたは形状

static void get_record1(void)

入力と出力

  • **入力**: 引数なし。内部的に ECPGget_var を使用して、事前に登録されたメモリ領域（MYTYPE および MYNULLTYPE 構造体）を参照する。
  • **出力**: なし（副作用として、参照先の構造体にフェッチした行データが書き込まれる）。

前提条件と不変条件

  • ECPGset_var により、インデックス 0 に MYTYPE 型のポインタ、インデックス 1 に MYNULLTYPE 型のポインタが登録済みであること。
  • カーソル mycur が既に OPEN 状態であること。

アルゴリズムまたは振る舞い

1. ECPGdo マクロを使用して SQL FETCH mycur を実行する。 2. 取得したデータ（id, t, d1, d2, c）を、ECPGget_var(0) が指す MYTYPE 構造体の各メンバに格納する。 3. 対応する NULL インジケータを ECPGget_var(1) が指す MYNULLTYPE 構造体に格納する。 4. SQL操作の直後に sqlca.sqlcode を確認し、負の値であれば exit(1) を呼び出して異常終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/preproc-outofscope.c 内のメインループ（行 230 および 331 付近）。
  • **呼び出し先**: ECPGdo（ECPGランタイム）、ECPGget_var（ECPGランタイム）、exit（標準ライブラリ）。

関連データ構造

  • MYTYPE: データベースの行データを保持する構造体。
  • MYNULLTYPE: 各フィールドのNULL状態を追跡するインジケータ構造体。
  • sqlca: SQL通信領域（エラーコード確認に使用）。

失敗モードと境界条件

  • sqlca.sqlcode < 0: SQL実行エラーが発生した場合、プロセスは即座に終了する。
  • ECPGget_var が無効なインデックスを参照した場合の挙動はECPGランタイムに依存する。

保守担当者向けメモ

  • 本関数はECPGプリプロセッサが生成したコードである。手動で修正せず、.pgc ソースファイルを修正すること。
  • ECPGget_var の引数（0, 1）は、変数の登録順序と密接に関連しているため、変更時は注意が必要である。
