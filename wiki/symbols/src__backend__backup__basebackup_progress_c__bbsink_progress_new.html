bbsink_progress_new

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_progress.c:58

目的

ベースバックアップの進捗追跡を行うための bbsink オブジェクトを生成し、pgstat サブシステムに対してバックアップ開始の通知と初期パラメータの設定を行う。

シグネチャまたは形状

bbsink *bbsink_progress_new(bbsink *next, bool estimate_backup_size)

入力と出力

  • **入力**:
  • next: bbsink チェーンの次段となるシンクへのポインタ（必須）。
  • estimate_backup_size: バックアップサイズの推定を行うかどうかを示すフラグ。
  • **出力**:
  • 初期化された bbsink オブジェクトへのポインタ。

前提条件と不変条件

  • next 引数は NULL であってはならない（Assert による検証あり）。
  • bbsink_progress_ops がこのシンクの操作テーブルとして設定される。

アルゴリズムまたは振る舞い

1. palloc0 を使用して bbsink 構造体メモリを確保する。 2. bbs_ops に bbsink_progress_ops を割り当て、bbs_next に next を格納する。 3. pgstat_progress_start_command を呼び出し、PROGRESS_COMMAND_BASEBACKUP コマンドの追跡を開始する。 4. pgstat_progress_update_param を呼び出し、PROGRESS_BASEBACKUP_BACKUP_TOTAL を -1（NULL 相当）に初期化する。 5. 生成したシンクオブジェクトを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c:1051
  • **呼び出し先**:
  • palloc0
  • pgstat_progress_start_command
  • pgstat_progress_update_param

関連データ構造

  • bbsink: バックアップ転送パイプラインを構成する基本構造体。
  • bbsink_ops: bbsink の操作コールバックテーブル。

失敗モードと境界条件

  • メモリ不足時には palloc0 がエラーを発生させる。
  • next が NULL の場合、アサーション違反によりプロセスが停止する。

保守担当者向けメモ

  • 本関数は bbsink チェーンの初期化の一部として呼び出される。
  • estimate_backup_size 引数は現在の実装では本関数内での直接的な処理には使用されていないが、後続の進捗更新ロジックにおいて推定値が利用される前提となっている。
