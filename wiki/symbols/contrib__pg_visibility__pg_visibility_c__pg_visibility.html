pg_visibility

定義

  • 種別: function
  • 定義箇所: contrib/pg_visibility/pg_visibility.c:122

目的

指定されたリレーションの特定のブロックについて、可視性マップ（VM）のビット状態と、実際のヒープページ上の可視性フラグを照合し、その結果を返す。

シグネチャまたは形状

Datum pg_visibility(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • relid (Oid): 対象リレーションのOID。
  • blkno (int64): 検査対象のブロック番号。
  • **出力**:
  • Datum: 以下の3つのフィールドを持つタプル（all_visible (bool), all_frozen (bool), pd_all_visible (bool)）。

前提条件と不変条件

  • 対象リレーションは AccessShareLock でオープンされる。
  • blkno は 0 以上かつ MaxBlockNumber 以下である必要がある。
  • リレーションの種類（relkind）は可視性マップをサポートしている必要がある。

アルゴリズムまたは振る舞い

1. 対象リレーションを AccessShareLock でオープンし、relkind を検証する。 2. visibilitymap_get_status を呼び出し、VMから all_visible および all_frozen ビットを取得する。 3. 指定された blkno がリレーションの現在のブロック数未満である場合、該当ページをバッファに読み込み、PageIsAllVisible マクロを用いてページヘッダーの可視性フラグを確認する。 4. blkno がEOFを超えている場合は、ページレベルの可視性は false とみなす。 5. 取得した3つの値をタプルとして構築し、返却する。 6. 最後にリレーションをクローズする。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（直接的なC言語レベルの呼び出し元情報はソース内に明示されていない）。
  • **呼び出し先**: relation_open, check_relation_relkind, pg_visibility_tupdesc, visibilitymap_get_status, ReleaseBuffer, RelationGetNumberOfBlocks, ReadBuffer, LockBuffer, BufferGetPage, PageIsAllVisible, UnlockReleaseBuffer, relation_close, HeapTupleGetDatum, heap_form_tuple。

関連データ構造

  • Relation: リレーション記述子。
  • Buffer: バッファ管理用ハンドル。
  • TupleDesc: 返却されるタプルの構造定義。

失敗モードと境界条件

  • blkno が負または MaxBlockNumber を超える場合、ERROR をスローする。
  • リレーションが可視性マップを持たない種類の場合、check_relation_relkind によりエラーが発生する可能性がある。
  • EOFを超えたブロック番号が指定された場合、ページレベルの可視性は false を返す。

保守担当者向けメモ

  • 本関数は AccessShareLock を使用するため、実行中にVMビットやページ状態が変更される可能性がある。
  • 誤検知を許容する設計となっており、破損チェックの補助として利用される。
