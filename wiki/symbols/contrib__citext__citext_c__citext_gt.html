citext_gt

定義

  • 種別: function
  • 定義箇所: contrib/citext/citext.c:282

目的

2つの citext 型の値を受け取り、左側の値が右側の値よりも大きいかどうかを判定する比較関数です。

シグネチャまたは形状

Datum citext_gt(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの text 型（citext として扱われる）引数を受け取ります。
  • **出力**: PG_RETURN_BOOL を介して bool 型（true または false）を返します。

前提条件と不変条件

  • 入力された text 型データは、PG_GETARG_TEXT_PP により展開された状態で取得されます。
  • 処理終了後、PG_FREE_IF_COPY を用いて入力データのメモリ解放を適切に行う必要があります。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して、2つの入力引数を取得します。 2. citextcmp 関数を呼び出し、現在の照合順序（PG_GET_COLLATION()）に基づいて比較を行います。 3. citextcmp の戻り値が 0 より大きい場合、result を true とします。 4. PG_FREE_IF_COPY を使用して、必要に応じて一時的なメモリを解放します。 5. 最終的な比較結果を PG_RETURN_BOOL で返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PG_FUNCTION_INFO_V1(citext_gt) マクロにより、PostgreSQL の関数マネージャから呼び出されることが示唆されています。
  • **呼び出し先**:
  • citextcmp（比較ロジックの実行）
  • PG_GETARG_TEXT_PP / PG_FREE_IF_COPY / PG_RETURN_BOOL（PostgreSQL 関数インターフェース）

関連データ構造

  • text: PostgreSQL の可変長文字列型。citext はこの型をベースに大文字・小文字を区別しない比較を実現しています。

失敗モードと境界条件

  • 入力が NULL の場合、PostgreSQL の関数呼び出し規約に従い、呼び出し元で適切に処理される必要があります（本関数内では明示的な NULL チェックは行われていません）。
  • 照合順序（Collation）の設定により、比較結果が影響を受ける可能性があります。

保守担当者向けメモ

  • 本関数は citext 型の「より大きい（greater than）」演算子を実装しています。
  • 比較ロジックの変更が必要な場合は、内部で使用している citextcmp を確認してください。
