ArchiveStreamState

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_basebackup/pg_basebackup.c:56

目的

pg_basebackup におけるアーカイブストリーム受信処理中の状態を保持し、テーブルスペース情報、圧縮設定、およびデータ処理パイプライン（astreamer）の管理を行う。

シグネチャまたは形状

typedef struct ArchiveStreamState
{
	int			tablespacenum;
	pg_compress_specification *compress;
	astreamer  *streamer;
	astreamer  *manifest_inject_streamer;
	PQExpBuffer manifest_buffer;
	char		manifest_filename[MAXPGPATH];
	FILE	   *manifest_file;
} ArchiveStreamState;

入力と出力

  • 本構造体は、バックアップデータの受信プロセスにおいて、受信したデータチャンクやマニフェスト情報を処理するためのコンテキストとして機能する。
  • manifest_buffer および manifest_file を通じて、バックアップマニフェストの蓄積と永続化を制御する。

前提条件と不変条件

  • astreamer パイプラインが適切に初期化されている必要がある。
  • manifest_filename は MAXPGPATH の制限に従う。

アルゴリズムまたは振る舞い

  • pg_basebackup.c 内のデータ受信処理において、callback_data として渡され、ストリームの各段階で参照される。
  • streamer および manifest_inject_streamer を介して、受信データの展開、圧縮、マニフェスト注入といったパイプライン処理を維持する。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_basebackup/pg_basebackup.c 内の関数（1287 行目でインスタンス化、1335 行目でコールバックデータとして参照）。
  • 呼び出し先: 本構造体自体はデータ保持用であり、直接的な関数呼び出しは行わないが、保持する astreamer ポインタを通じてパイプライン処理が実行される。

関連データ構造

  • astreamer: データ処理パイプラインの抽象化インターフェース。
  • pg_compress_specification: 圧縮設定を保持する構造体。
  • PQExpBuffer: マニフェストデータの一時バッファ。

失敗モードと境界条件

  • manifest_file のオープン失敗や、manifest_buffer のメモリ確保失敗が想定される。
  • tablespacenum が不正な値である場合、後続の処理で不整合が生じる可能性がある。

保守担当者向けメモ

  • astreamer パイプラインの構築と破棄のライフサイクルが本構造体の生存期間と一致していることを確認すること。
  • マニフェスト処理に関連するフィールド（manifest_buffer, manifest_file, manifest_filename）の整合性は、バックアップの完全性に直結するため注意が必要。
