sql_function_parse_error_callback

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/pg_proc.c:998

目的

SQL関数の解析中に発生したエラーに対し、エラーコンテキスト情報を付与または変換して、ユーザーに分かりやすいエラーメッセージを提供します。

シグネチャまたは形状

static void sql_function_parse_error_callback(void *arg)

入力と出力

  • **入力**: void *arg - parse_error_callback_arg 構造体へのポインタ。
  • **出力**: なし（副作用としてエラーコンテキストを更新）。

前提条件と不変条件

  • 引数 arg は parse_error_callback_arg 型としてキャスト可能であること。
  • callback_arg->prosrc および callback_arg->proname が有効な値を持っていること。

アルゴリズムまたは振る舞い

1. 引数 arg を parse_error_callback_arg 型にキャストします。 2. function_parse_error_transpose(callback_arg->prosrc) を呼び出し、エラーが構文エラーであるか判定します。 3. 構文エラーであれば、エラー位置を CREATE FUNCTION 文のコンテキストに変換します。 4. 構文エラーでない場合は、errcontext を使用して「SQL function "関数名"」という情報をエラーコンテキストスタックに追加します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/catalog/pg_proc.c 内で、sqlerrcontext.callback にこの関数が代入されています。
  • **呼び出し先**: function_parse_error_transpose（エラー位置変換用）、errcontext（エラーコンテキスト追加用）。

関連データ構造

  • parse_error_callback_arg: エラー発生時のコンテキスト情報（関数名やソースコード）を保持する構造体。

失敗モードと境界条件

  • callback_arg が NULL の場合、デリファレンスによりクラッシュする可能性があります。
  • function_parse_error_transpose がエラー位置を特定できない場合、汎用的な関数名ベースのコンテキスト情報のみが出力されます。

保守担当者向けメモ

  • 本関数はエラーハンドリングのコールバックとして登録されるため、直接呼び出すのではなく、エラー発生時にシステムによって呼び出されることを前提としています。
  • function_parse_error_transpose のロジック変更時には、本関数が正しくエラー位置をマッピングできているか確認してください。
