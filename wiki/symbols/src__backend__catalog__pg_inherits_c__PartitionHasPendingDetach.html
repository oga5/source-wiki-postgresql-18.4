PartitionHasPendingDetach

定義

  • **種別**: function
  • **定義箇所**: src/backend/catalog/pg_inherits.c:619

目的

指定されたリレーション（パーティション）が、現在デタッチ処理中（inhdetachpending フラグが真）であるかどうかを判定します。

シグネチャまたは形状

bool PartitionHasPendingDetach(Oid partoid)

入力と出力

  • **入力**: partoid (Oid) - 検査対象のリレーション識別子。
  • **出力**: bool - パーティションがデタッチ処理中であれば true、そうでなければ false。

前提条件と不変条件

  • 呼び出し元は、対象のリレーションに対して適切なロックを保持している必要があります。
  • 対象の partoid は pg_inherits カタログに存在している必要があります。存在しない場合、本関数はエラーを発生させます。

アルゴリズムまたは振る舞い

1. pg_inherits システムカタログを RowExclusiveLock でオープンします。 2. InheritsRelidSeqnoIndexId インデックスを使用して、指定された partoid に一致するエントリを検索します。 3. 該当するタプルが見つかった場合、その inhdetachpending フィールドの値を取得します。 4. スキャンを終了し、カタログをクローズした上で、取得したフラグ値を返します。 5. 該当するエントリが見つからない場合は、elog(ERROR) を呼び出し、処理を中断します。

関連データ構造

  • pg_inherits: 継承関係およびパーティションのデタッチ状態を保持するシステムカタログ。
  • Form_pg_inherits: pg_inherits カタログのタプル構造体。inhdetachpending フィールドを含む。

失敗モードと境界条件

  • **エラー**: 指定された partoid が pg_inherits に存在しない場合、ERROR レベルでログ出力され、トランザクションがアボートされます。
  • **境界条件**: pg_inherits に同一の inhrelid を持つエントリが複数存在することは想定されていません（コード内コメントより）。

保守担当者向けメモ

  • 本関数は、対象が実際にパーティションであるかどうかの検証は行いません。
  • inhdetachpending の値は、カタログ上のフラグとトランザクションの可視性に依存します。並行処理における整合性については、呼び出し元でのロック管理に依存している点に注意してください。
