SlruScanDirCbDeleteAll

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/slru.c:1743

目的

SLRU ディレクトリ内のセグメントファイルを走査し、すべてのセグメントを削除するためのコールバック関数です。

シグネチャまたは形状

bool SlruScanDirCbDeleteAll(SlruCtl ctl, char *filename, int64 segpage, void *data)

入力と出力

  • **入力**:
  • ctl: SLRU 制御構造体。
  • filename: 走査対象のファイル名。
  • segpage: セグメントの開始ページ番号。
  • data: 呼び出し元から渡される補助データ（本関数では未使用）。
  • **出力**:
  • 戻り値: 常に false を返し、走査の継続を指示します。

前提条件と不変条件

  • ctl が有効な SlruCtl 構造体を指している必要があります。
  • SlruInternalDeleteSegment が内部で呼び出されるため、呼び出し元は適切なロックや排他制御を確保している必要があります。

アルゴリズムまたは振る舞い

1. 引数 segpage を SLRU_PAGES_PER_SEGMENT で除算し、セグメント番号を算出します。 2. SlruInternalDeleteSegment を呼び出し、該当するセグメントファイルを物理的に削除します。 3. false を返して、ディレクトリ内の全ファイルを走査し続けるよう指示します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/commit_ts.c: SlruScanDirectory 経由で呼び出し。
  • src/backend/commands/async.c: SlruScanDirectory 経由で呼び出し。
  • src/test/modules/test_slru/test_slru.c: テスト目的で呼び出し。
  • **呼び出し先**:
  • SlruInternalDeleteSegment: セグメント削除の実行。

関連データ構造

  • SlruCtl: SLRU インスタンスの制御構造体。

失敗モードと境界条件

  • SlruInternalDeleteSegment が失敗した場合（ファイル削除の失敗など）、その挙動は SlruInternalDeleteSegment の実装に依存します。
  • 走査対象のディレクトリが空の場合、本関数は一度も呼び出されません。

保守担当者向けメモ

  • 本関数は SlruScanDirectory のコールバックとして設計されています。
  • 戻り値が false であるため、ディレクトリ内のすべてのエントリに対して削除処理が試行されます。
