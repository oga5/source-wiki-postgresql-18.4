目的

ECPGプリプロセッサの回帰テスト用エントリポイントであり、コマンドライン引数で定義されたマクロがファイル境界を越えて汚染されないことを検証する。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（コマンドライン引数によるマクロ定義環境下で実行されることを想定）
  • 出力: 成功時に 0 を返却。標準出力にテスト結果（i の値や CMDLINESYM の状態）を出力。

前提条件と不変条件

  • ECPGランタイムライブラリが利用可能であること。
  • テスト用データベース ecpg1_regression が接続可能であること。
  • 各 EXEC SQL 文の実行後、sqlca.sqlcode が負の値でないこと（エラー発生時は sqlprint() が呼び出される）。

アルゴリズムまたは振る舞い

1. ECPGdebug を有効化し、デバッグログを出力する。 2. ECPGconnect でデータベースに接続する。 3. ECPGdo を使用してテーブル作成およびデータ挿入を行う。 4. SELECT 文を実行し、結果をホスト変数 i および s に格納して表示する。 5. コマンドラインで定義されたシンボル（CMDLINESYM）に関連するクエリを複数回実行し、値の推移を printf で出力して再定義・リセットの挙動を確認する。 6. ECPGdisconnect で接続を解除し、終了する。

呼び出し元と呼び出し先

  • 呼び出し元: テスト実行環境（OSまたはテストハーネス）。
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGdisconnect, sqlprint, printf。
  • 注記: ツールによる静的解析では、本シンボルを呼び出す具体的なソースコード上の箇所は特定されていません。

関連データ構造

  • sqlca: SQL通信領域（sqlca_t 型）。エラーハンドリングに使用。
  • ホスト変数: int i, char s[200]。

失敗モードと境界条件

  • sqlca.sqlcode < 0 となる場合、sqlprint() が呼び出され、エラー情報が標準エラー出力に出力される。
  • マクロ定義が期待通りにリセットされない場合、後続の SELECT 文の結果が想定値と乖離する。
