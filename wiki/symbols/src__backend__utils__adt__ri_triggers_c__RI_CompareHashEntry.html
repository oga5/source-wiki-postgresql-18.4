RI_CompareHashEntry

定義

  • 種別: struct
  • 定義箇所: src/backend/utils/adt/ri_triggers.c:171

目的

参照整合性（RI）チェックにおいて、データ型の比較や型変換に必要な関数情報をキャッシュするためのハッシュテーブルエントリ構造体です。

シグネチャまたは形状

typedef struct RI_CompareHashEntry
{
    RI_CompareKey key;              /* ハッシュキー */
    bool          valid;            /* 初期化成功フラグ */
    FmgrInfo      eq_opr_finfo;     /* 等価演算子の関数情報 */
    FmgrInfo      cast_func_finfo;  /* 型変換（キャスト）の関数情報 */
} RI_CompareHashEntry;

入力と出力

  • 本構造体自体はハッシュテーブルの要素として保持されます。
  • ri_HashCompareOp 関数を介して、Oid（演算子および型ID）をキーとして検索・生成されます。

前提条件と不変条件

  • key フィールドは RI_CompareKey 型であり、ハッシュテーブルの検索キーとして機能します。
  • valid が true の場合、eq_opr_finfo および cast_func_finfo は有効な関数情報として利用可能です。

アルゴリズムまたは振る舞い

  • 参照整合性チェック時に、特定の型に対する比較演算子やキャスト関数を繰り返し検索するオーバーヘッドを避けるために使用されます。
  • hash_search を用いて ri_compare_cache 内を検索し、存在しない場合は ri_HashCompareOp によってエントリが生成・初期化されます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • ri_HashCompareOp (生成・検索)
  • src/backend/utils/adt/ri_triggers.c 内の各トリガー処理関数（キャッシュされた情報の利用）
  • **呼び出し先**:
  • hash_search (ハッシュテーブル操作)
  • fmgr_info 等（FmgrInfo の初期化に関連）

関連データ構造

  • RI_CompareKey: ハッシュテーブルの検索キー。
  • FmgrInfo: PostgreSQLの関数マネージャが使用する関数呼び出し情報。
  • ri_compare_cache: 本構造体を格納するハッシュテーブル（HTAB）。

失敗モードと境界条件

  • ハッシュテーブルの初期化やエントリの生成に失敗した場合、valid フラグが適切に設定されない可能性があります。
  • メモリ不足時には hash_search がエラーを発生させる可能性があります。

保守担当者向けメモ

  • 本構造体は ri_compare_cache ハッシュテーブルの要素サイズとして sizeof(RI_CompareHashEntry) が指定されています。
  • キャッシュされた FmgrInfo はバックエンドの生存期間中保持されるため、システムカタログの変更に伴う無効化ロジックが必要な場合は注意してください。
