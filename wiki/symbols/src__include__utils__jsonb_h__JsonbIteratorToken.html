JsonbIteratorToken

定義

  • 種別: enum
  • 定義箇所: src/include/utils/jsonb.h:20

目的

JsonbIterator を用いて jsonb 構造を走査する際、現在位置の要素タイプや構造上の状態を識別するために使用されます。

シグネチャまたは形状

typedef enum
{
	WJB_DONE,          /* 走査終了 */
	WJB_KEY,           /* オブジェクトのキー */
	WJB_VALUE,         /* オブジェクトの値 */
	WJB_ELEM,          /* 配列の要素 */
	WJB_BEGIN_ARRAY,   /* 配列の開始 */
	WJB_END_ARRAY,     /* 配列の終了 */
	WJB_BEGIN_OBJECT,  /* オブジェクトの開始 */
	WJB_END_OBJECT,    /* オブジェクトの終了 */
} JsonbIteratorToken;

入力と出力

  • 入力: JsonbIteratorNext 関数等の戻り値として生成されます。
  • 出力: 走査アルゴリズムにおいて、次に処理すべきデータ型や構造の境界を決定するための制御フローとして利用されます。

前提条件と不変条件

  • JsonbIterator を介した走査プロセスにおいて、常に有効なトークンが返されます。
  • 走査が完了した状態では WJB_DONE が返されます。

アルゴリズムまたは振る舞い

JsonbIteratorNext を呼び出すことで、jsonb のバイナリ構造を再帰的に探索し、現在のノードに応じたこの列挙型の値を返します。呼び出し元はこの値に基づき、キー、値、あるいはネストされた構造（配列・オブジェクト）の開始・終了を判定します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/adt/jsonb.c、src/backend/utils/adt/jsonb_gin.c、src/backend/utils/adt/jsonb_op.c、および contrib/ 配下の各拡張モジュール（jsonb_plperl, jsonb_plpython 等）で、イテレータの戻り値として参照されています。
  • **呼び出し先**: 本シンボル自体は列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • JsonbIterator: このトークンを用いて走査を行うためのメイン構造体。
  • JsonbValue: 走査中に取得される個々の要素を保持する構造体。

失敗モードと境界条件

  • 走査対象の jsonb が不正な形式である場合、イテレータの挙動は未定義です。
  • 走査終了後に JsonbIteratorNext を呼び出した場合の挙動は、WJB_DONE を返し続けることが期待されます。

保守担当者向けメモ

  • 新しい jsonb 構造の要素タイプを追加する場合は、この列挙型を拡張する必要があります。
  • src/backend/utils/adt/jsonb.c 内の JsonbIteratorNext 実装と密結合しているため、変更時は必ず整合性を確認してください。
