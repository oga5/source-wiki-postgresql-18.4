compareentry

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsvector.c:86

目的

WordEntry 構造体で表現される語彙（lexeme）を、共有バッファ内の文字列データに基づいて辞書順に比較します。主に qsort_arg を用いたソート処理の比較関数として使用されます。

シグネチャまたは形状

static int compareentry(const void *va, const void *vb, void *arg)

入力と出力

  • **入力**:
  • va, vb: 比較対象の WordEntry 構造体へのポインタ。
  • arg: 語彙文字列が格納されたバッファ（char *）へのポインタ。
  • **出力**:
  • 比較結果を整数で返します（tsCompareString の戻り値に準拠）。

前提条件と不変条件

  • arg に渡されるバッファは、WordEntry が保持する pos（オフセット）と len（長さ）から有効な文字列を参照できる必要があります。
  • WordEntry 構造体は、比較対象の文字列データと正しく紐付いている必要があります。

アルゴリズムまたは振る舞い

1. va および vb を WordEntry 型にキャストします。 2. arg を文字列バッファとして取得します。 3. tsCompareString を呼び出し、バッファ内の各オフセット（a->pos, b->pos）と長さ（a->len, b->len）に基づいて文字列比較を行います。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsvector.c: qsort_arg の引数として呼び出されます。
  • src/backend/utils/adt/tsvector.c: tsvector の内部構造の検証やソート状態の確認時に呼び出されます。
  • **呼び出し先**:
  • tsCompareString: 実際の文字列比較を実行します。

関連データ構造

  • WordEntry: tsvector 内の各語彙のメタデータ（位置、長さ、重み）を保持する構造体。

失敗モードと境界条件

  • WordEntry の pos や len がバッファの範囲外を指している場合、メモリ破壊や不正な比較結果を招く可能性があります。

保守担当者向けメモ

  • 本関数は qsort_arg のコールバックとして設計されています。シグネチャを変更する場合は、tsvector.c 内のすべての呼び出し箇所を更新する必要があります。
  • WordEntry 構造体の定義変更（特に pos や len の型や位置）が本関数の動作に直結するため、src/include/tsearch/ts_type.h の変更時には注意してください。
