SyncRepWakeQueue

定義

  • 種別: function
  • 定義箇所: src/backend/replication/syncrep.c:906

目的

同期レプリケーションの待機キューから、指定された条件を満たすバックエンドプロセスを解放し、ウェイクアップさせる。

シグネチャまたは形状

static int SyncRepWakeQueue(bool all, int mode)

入力と出力

  • **入力**:
  • all (bool): true の場合、キュー内の全プロセスを解放する。false の場合、現在のLSN進捗状況に応じて条件を満たすプロセスのみを解放する。
  • mode (int): 待機モード（SYNC_REP_WAIT_WRITE, SYNC_REP_WAIT_FLUSH, SYNC_REP_WAIT_APPLY）。
  • **出力**:
  • int: 解放したプロセス数。

前提条件と不変条件

  • mode は 0 以上 NUM_SYNC_REP_WAIT_MODE 未満であること。
  • SyncRepLock が排他モード（LW_EXCLUSIVE）で保持されていること。
  • 対象の待機キューがLSN順にソートされていること。

アルゴリズムまたは振る舞い

1. 指定された mode の待機キュー（WalSndCtl->SyncRepQueue[mode]）を走査する。 2. all が false の場合、現在のLSN（walsndctl->lsn[mode]）がプロセスの待機LSN（proc->waitLSN）に達していない時点で走査を終了する。 3. 条件を満たす各プロセスについて以下を実行する：

  • キューからプロセスを削除する。
  • pg_write_barrier() を発行し、キューからの削除が完了したことを保証する。
  • プロセスの状態を SYNC_REP_WAIT_COMPLETE に更新する。
  • SetLatch を呼び出し、待機中のプロセスを再開させる。

    • 4. 解放したプロセス数をカウントし、最後にその値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/replication/syncrep.c 内の各箇所（SyncRepReleaseWaiters 等から呼び出され、レプリケーションの進捗に応じた解放を行う）。
  • **呼び出し先**:
  • dlist_delete_thoroughly
  • SetLatch

関連データ構造

  • WalSndCtlData: 同期レプリケーションの制御構造体。
  • PGPROC: バックエンドプロセスの状態を保持する構造体。
  • SyncRepLock: 待機キューを保護するLWLock。

失敗モードと境界条件

  • キューが空の場合、numprocs は 0 を返す。
  • SyncRepLock を保持せずに呼び出した場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は SyncRepWaitForLSN と密接に関連しており、状態遷移（SYNC_REP_WAIT_COMPLETE への変更）とラッチ操作の順序は、メモリバリアを介して厳密に制御されている。変更を加える際は、SyncRepWaitForLSN 側の読み取りロジックとの整合性に注意すること。
