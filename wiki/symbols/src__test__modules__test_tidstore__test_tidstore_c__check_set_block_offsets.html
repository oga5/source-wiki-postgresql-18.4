check_set_block_offsets

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_tidstore/test_tidstore.c:234

目的

TidStore 内に格納されたデータが、検証用データ構造（ItemArray）と整合しているかを検証する。

シグネチャまたは形状

Datum check_set_block_offsets(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（SQL インターフェース経由で呼び出される）
  • 出力: PG_RETURN_VOID()（成功時）
  • 失敗時: 整合性エラーが発生した場合、elog(ERROR, ...) によりトランザクションを中断する。

前提条件と不変条件

  • check_tidstore_available() が呼び出され、tidstore が初期化済みであること。
  • グローバル変数 items（ItemArray 型）が、TidStore に挿入された期待値と同期されていること。

アルゴリズムまたは振る舞い

1. **存在確認**: items 内の全 TID が TidStoreIsMember で存在するか確認する。 2. **全件検索**: items に含まれる各ブロック番号に対し、全オフセットを走査して TidStore 内の全 TID を lookup_tids 配列に収集する。 3. **イテレーション**: TidStoreBeginIterate を使用して TidStore 内の全データを走査し、iter_tids 配列に収集する。 4. **整合性検証**:

  • lookup_tids および iter_tids の件数が期待値と一致するか確認する。
  • 各配列を itemptr_cmp でソートし、items.insert_tids と比較して不整合がないか確認する。

呼び出し元と呼び出し先

  • 呼び出し元: SQL インターフェース（PG_FUNCTION_INFO_V1 による登録）。
  • 呼び出し先: check_tidstore_available, TidStoreIsMember, TidStoreLockShare, TidStoreUnlock, TidStoreBeginIterate, TidStoreIterateNext, TidStoreGetBlockOffsets, TidStoreEndIterate, qsort, itemptr_cmp。

関連データ構造

  • TidStore: テスト対象のデータ構造。
  • ItemArray: 検証用の期待値を保持する構造体。
  • TidStoreIter, TidStoreIterResult: イテレーション用構造体。

失敗モードと境界条件

  • TidStore 内の TID が欠落している場合、エラーを投げる。
  • 検索またはイテレーションで取得した TID の総数が、挿入済み TID の総数と一致しない場合、エラーを投げる。
  • ソート後の配列比較で不一致がある場合、エラーを投げる。

保守担当者向けメモ

  • 本関数は単一プロセスでの実行を前提としており、LWLock を用いた排他制御が含まれています。
  • items 配列の各フィールド（insert_tids, lookup_tids, iter_tids）が正しく管理されていることがテストの前提となります。
