build_replindex_scan_key

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execReplication.c:54

目的

論理レプリケーションにおいて、インデックスを用いた行検索を行うためのスキャンキー（ScanKey）を構築します。

シグネチャまたは形状

static int build_replindex_scan_key(ScanKey skey, Relation rel, Relation idxrel, TupleTableSlot *searchslot)

入力と出力

  • **入力**:
  • skey: 初期化対象の ScanKey 配列。
  • rel: 対象のテーブルリレーション。
  • idxrel: 検索に使用するインデックスリレーション。
  • searchslot: 検索条件となる値を含む TupleTableSlot。
  • **出力**:
  • 戻り値 (int): 生成されたスキャンキーの数。

前提条件と不変条件

  • skey_attoff > 0 であること（少なくとも1つの属性でインデックススキャンを行う必要がある）。
  • 現在の実装では、インデックス内の式（expression）はサポートされていません。

アルゴリズムまたは振る舞い

1. インデックスの属性定義（indkey）および演算子クラス情報を取得します。 2. インデックスの各キー属性に対してループ処理を行います。

  • 式が含まれる属性はスキップします。
  • 演算子ファミリーから等価比較（COMPARE_EQ）演算子を特定します。
  • ScanKeyInit を使用して、searchslot から取得した値を基にスキャンキーを初期化します。
  • 照合順序（sk_collation）を設定します。
  • 値が NULL の場合は、SK_ISNULL および SK_SEARCHNULL フラグを付与します。

    • 3. 構築したスキャンキーの総数を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execReplication.c 内の RelationFindReplTupleByIndex から呼び出されます。
  • **呼び出し先**: SysCacheGetAttrNotNull, IndexRelationGetNumberOfKeyAttributes, get_opclass_input_type, get_opclass_family, IndexAmTranslateCompareType, get_opfamily_member, get_opcode, ScanKeyInit 等のアクセサ関数。

関連データ構造

  • ScanKey: インデックススキャンの条件を保持する構造体。
  • Relation: テーブルおよびインデックスのメタデータ。
  • TupleTableSlot: 検索対象のタプルデータを保持するコンテナ。

失敗モードと境界条件

  • 必要な演算子が見つからない場合、elog(ERROR) により処理を中断します。
  • インデックスに式が含まれる場合、その属性は無視されます。

保守担当者向けメモ

  • 本関数は論理レプリケーションの適用プロセスにおける行特定フェーズの基盤です。
  • 現状、式インデックスには対応していません。将来的な拡張が必要な場合は、table_attno のチェックロジックを修正する必要があります。
