pg_hmac_errno

定義

  • 種別: enum
  • 定義箇所: src/common/hmac.c:42

目的

HMAC（Keyed-Hashing for Message Authentication）処理中に発生するエラー状態を識別するための列挙型です。

シグネチャまたは形状

typedef enum pg_hmac_errno
{
	PG_HMAC_ERROR_NONE = 0,
	PG_HMAC_ERROR_OOM,
	PG_HMAC_ERROR_INTERNAL,
} pg_hmac_errno;

入力と出力

  • **入力**: なし（列挙型の定義）
  • **出力**: pg_hmac_errno 型の値

前提条件と不変条件

  • PG_HMAC_ERROR_NONE は 0 と定義されており、正常終了状態を示します。
  • 状態管理において、エラーが発生した場合はこの列挙型のいずれかの値が保持されます。

アルゴリズムまたは振る舞い

HMAC 計算プロセスにおけるエラー種別を分類します。

  • PG_HMAC_ERROR_NONE: エラーなし。
  • PG_HMAC_ERROR_OOM: メモリ不足（Out of Memory）。
  • PG_HMAC_ERROR_INTERNAL: 内部ハッシュ関数等の予期せぬエラー。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/hmac.c および src/common/hmac_openssl.c 内の HMAC 処理関数が、エラー状態を保持する変数として参照しています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する構造体。この構造体内部でエラー状態を記録するために本列挙型が使用されます。

失敗モードと境界条件

  • 本シンボル自体は定義であるため失敗モードを持ちませんが、これを利用する HMAC API は、メモリ確保失敗時に PG_HMAC_ERROR_OOM を、ハッシュ計算失敗時に PG_HMAC_ERROR_INTERNAL を設定する設計となっています。

保守担当者向けメモ

  • 本列挙型は src/common/hmac.c と src/common/hmac_openssl.c の両方で参照されています。エラーコードを追加・変更する際は、両方の実装における整合性に注意してください。
