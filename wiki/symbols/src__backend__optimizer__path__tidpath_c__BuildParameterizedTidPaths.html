BuildParameterizedTidPaths

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/tidpath.c:425

目的

結合条件（joininfo）から、他テーブルの列に依存するパラメータ化されたTIDスキャンパスを生成し、プランナに登録する。

シグネチャまたは形状

static void BuildParameterizedTidPaths(PlannerInfo *root, RelOptInfo *rel, List *clauses)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報。
  • rel: 対象となるリレーション情報。
  • clauses: 評価対象となる RestrictInfo のリスト。
  • **出力**: なし（副作用として add_path を通じて rel にパスを追加）。

前提条件と不変条件

  • clauses 内の各 RestrictInfo は、pseudoconstant でなく、かつ restriction_is_securely_promotable を満たす必要がある。
  • IsTidEqualClause により、対象の条件が ctid を用いた等価比較（TidEqual）であると判定される必要がある。
  • join_clause_is_movable_to により、対象リレーションへ移動可能である必要がある。

アルゴリズムまたは振る舞い

1. clauses リストを反復処理し、各 RestrictInfo がTIDスキャンに利用可能か検証する。 2. 検証を通過した条件に対し、required_outer（パスの実行に必要な外部リレーションの集合）を計算する。これには rinfo->required_relids と rel->lateral_relids の和集合から対象リレーション自身を除いたものが用いられる。 3. create_tidscan_path を呼び出してパスを作成し、add_path を用いてリレーションのパスリストに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/path/tidpath.c:576

関連データ構造

  • PlannerInfo: プランナのコンテキスト。
  • RelOptInfo: スキャン対象のリレーション情報。
  • RestrictInfo: 評価対象の制約条件。
  • Path: 生成されるアクセスパス。

失敗モードと境界条件

  • TidEqual 以外の演算子（範囲比較や CurrentOfExpr など）は本関数では処理されず、スキップされる。
  • join_clause_is_movable_to の判定に失敗した条件は無視される。

保守担当者向けメモ

  • 本関数は TidEqual 結合条件のみを対象としている。ScalarArrayOpExpr 等の他の形式をサポートする可能性はあるが、現時点ではコスト対効果の観点から実装されていない。
  • RestrictInfoIsTidQual とは異なる判定ロジックを持つため、修正時は整合性に注意すること。
