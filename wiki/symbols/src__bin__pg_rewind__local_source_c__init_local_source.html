init_local_source

定義

  • 種別: function
  • 定義箇所: src/bin/pg_rewind/local_source.c:37

目的

ローカルファイルシステム上のデータディレクトリをソースとして扱うための local_source 構造体を初期化し、rewind_source インターフェースをセットアップするコンストラクタです。

シグネチャまたは形状

rewind_source *init_local_source(const char *datadir)

入力と出力

  • **入力**: datadir (ソースとなるデータディレクトリのパス)
  • **出力**: 初期化された rewind_source 構造体へのポインタ。

前提条件と不変条件

  • pg_malloc0 によりメモリがゼロクリアされた状態で local_source 構造体が確保されます。
  • rewind_source インターフェースの各関数ポインタ（traverse_files, fetch_file, queue_fetch_file, queue_fetch_range, finish_fetch, destroy）がローカル実装関数に割り当てられます。
  • get_current_wal_insert_lsn は NULL に設定されます。

アルゴリズムまたは振る舞い

1. local_source 構造体分のメモリを確保します。 2. 構造体内の common フィールド（rewind_source 型）に対し、ローカルファイル操作用の関数ポインタを登録します。 3. 引数で渡された datadir を構造体に保持します。 4. 最終的に rewind_source 型のポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_rewind/pg_rewind.c 内のメイン処理から呼び出されます。
  • **呼び出し先**: pg_malloc0 (メモリ確保)。各関数ポインタに登録された local_traverse_files, local_fetch_file, local_queue_fetch_file, local_queue_fetch_range, local_finish_fetch, local_destroy は、この関数自体からは直接呼び出されません。

関連データ構造

  • local_source: ローカルディレクトリのパスとインターフェース実装を保持する構造体。
  • rewind_source: ソースデータアクセスの抽象インターフェース。

失敗モードと境界条件

  • メモリ確保に失敗した場合、pg_malloc0 の内部挙動によりプロセスが終了します。

保守担当者向けメモ

  • 本関数は rewind_source インターフェースの具現化を担当しています。将来的に新しいファイル操作関数を追加する場合は、本関数のポインタ登録箇所を更新する必要があります。
