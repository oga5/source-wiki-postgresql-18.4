ExecCheckOneRelPerms

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execMain.c:645

目的

指定されたリレーション（RTEPermissionInfo）に対して、クエリ実行に必要な権限が現在のユーザーに付与されているかを検証します。

シグネチャまたは形状

bool ExecCheckOneRelPerms(RTEPermissionInfo *perminfo)

入力と出力

  • **入力**: perminfo (検証対象のリレーション権限情報構造体)
  • **出力**: 権限が充足されていれば true、不足していれば false を返します。

前提条件と不変条件

  • perminfo->requiredPerms は 0 以外である必要があります（Assert による検証）。
  • ユーザーIDは、perminfo->checkAsUser が有効な場合はその値を使用し、それ以外の場合は GetUserId() を使用します。

アルゴリズムまたは振る舞い

1. **リレーションレベルの権限確認**: pg_class_aclmask を使用し、リレーション全体に対する権限を確認します。 2. **不足権限の特定**: リレーションレベルで満たせない権限（remainingPerms）を算出します。 3. **列レベルの権限確認**:

  • remainingPerms に SELECT, INSERT, UPDATE 以外のビットが含まれる場合は直ちに false を返します。
  • SELECT 権限が必要な場合、perminfo->selectedCols を走査し、各列または全列（InvalidAttrNumber の場合）に対して pg_attribute_aclcheck を実行します。
  • INSERT または UPDATE 権限が必要な場合、ExecCheckPermissionsModified を呼び出して列レベルの変更権限を検証します。

    • 4. すべての検証を通過した場合に true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execMain.c:624
  • src/backend/optimizer/plan/planner.c:897
  • **呼び出し先**:
  • pg_class_aclmask
  • pg_attribute_aclcheck_all
  • pg_attribute_aclcheck
  • ExecCheckPermissionsModified
  • bms_next_member (ビットマップセット走査)

関連データ構造

  • RTEPermissionInfo: リレーションの権限要件、対象列、ユーザーIDを保持する構造体。
  • AclMode: 権限ビットマスク。

失敗モードと境界条件

  • **権限不足**: 必要な権限がリレーションレベルでも列レベルでも確認できない場合、false を返します。
  • **列参照なし**: SELECT COUNT(*) のように列を明示しない場合、いずれかの列に対して SELECT 権限があれば許可されます。
  • **エラー報告**: 現在の実装では、列レベルの権限不足であってもテーブルレベルのエラーとして報告されます。

保守担当者向けメモ

  • GetUserId() は現在十分に高速ですが、将来的にボトルネックとなる場合は ExecCheckPermissions で一度だけ取得して引き回す設計への変更が検討可能です。
  • 列レベルの権限チェック失敗時に、より詳細な列レベルのエラーメッセージを返すことが望ましいとされています。
