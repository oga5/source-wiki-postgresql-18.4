AttStatsSlot

定義

  • 種別: struct
  • 定義箇所: src/include/utils/lsyscache.h:47

目的

システムカタログから統計情報（ヒストグラムや頻度値など）を安全に抽出し、保持するためのコンテナ構造体です。メモリ管理を伴う統計値の取得を容易にします。

シグネチャまたは形状

typedef struct AttStatsSlot
{
    Oid         staop;
    Oid         stacoll;
    Oid         valuetype;
    Datum      *values;
    int         nvalues;
    float4     *numbers;
    int         nnumbers;
    void       *values_arr;
    void       *numbers_arr;
} AttStatsSlot;

入力と出力

  • **入力**: get_attstatsslot 関数に渡され、統計情報の格納先として利用されます。
  • **出力**: 抽出された統計値（values, numbers）およびそのメタデータ（staop, stacoll, valuetype）が格納されます。

前提条件と不変条件

  • values_arr および numbers_arr は get_attstatsslot によって内部的に割り当てられるプライベートな領域です。
  • 構造体内のメモリリークを防ぐため、使用後は必ず free_attstatsslot を呼び出す必要があります。

アルゴリズムまたは振る舞い

1. get_attstatsslot を呼び出す際、取得したい統計情報の種類（ATTSTATSSLOT_VALUES や ATTSTATSSLOT_NUMBERS）を指定します。 2. 指定されたフラグに応じて、values や numbers 配列がメモリ確保され、データがコピーされます。 3. free_attstatsslot が呼び出されると、内部的に確保された values_arr および numbers_arr が解放されます。

呼び出し元と呼び出し先

  • **呼び出し元**: 選択度推定（Selectivity Estimation）を行う各モジュールが主な利用者です。
  • contrib/intarray/_int_selfuncs.c
  • src/backend/executor/nodeHash.c
  • src/backend/tsearch/ts_selfuncs.c
  • src/backend/utils/adt/array_selfuncs.c
  • src/backend/utils/adt/multirangetypes_selfuncs.c
  • src/backend/utils/adt/network_selfuncs.c
  • **呼び出し先**: 構造体自体はデータ保持用であり、直接関数を呼び出すことはありません。操作には get_attstatsslot および free_attstatsslot を介します。

関連データ構造

  • HeapTuple: システムカタログから情報を取得する際の基盤データ。
  • Oid: 統計情報に関連する演算子や型、照合順序の識別子。

失敗モードと境界条件

  • 統計情報が存在しない場合、values や numbers は NULL に設定され、nvalues や nnumbers は 0 となります。
  • メモリ不足時には palloc 経由でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は lsyscache.c 内の関数と密結合しています。フィールドの追加や変更を行う際は、必ず get_attstatsslot および free_attstatsslot の実装を更新してください。
  • values_arr と numbers_arr は外部から直接操作すべきではありません。
