test_thread

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/thread-thread_implicit.c:132

目的

ECPGのマルチスレッド環境における並行データベース接続およびトランザクション処理の整合性を検証する。各スレッドが独立した接続名を用いてデータベースへ接続し、ループ内で INSERT 処理を実行する。

シグネチャまたは形状

void *test_thread(void *arg)

入力と出力

  • **入力**: arg (スレッド識別番号として使用される long 型のポインタ)
  • **出力**: void * (常に NULL を返す)

前提条件と不変条件

  • iterations 変数が定義されており、ループ回数が制御されていること。
  • データベース ecpg1_regression が接続可能であること。
  • 各スレッドは thread_%03ld という形式の一意な接続名を使用する。

アルゴリズムまたは振る舞い

1. **接続確立**: threadnum を基に接続名を生成し、ECPGconnect でデータベースへ接続する。接続失敗時はエラーメッセージを出力して終了する。 2. **トランザクション開始**: ECPGtrans を呼び出し、begin でトランザクションを開始する。 3. **データ挿入**: iterations 回数分、ECPGdo を使用して test_thread テーブルへ thread 名と iteration 番号を挿入する。 4. **終了処理**: ECPGtrans で commit を実行し、ECPGdisconnect で接続を閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/thread-thread.c 内の pthread_create または CreateThread（Windows環境）からスレッドエントリポイントとして呼び出される。
  • **呼び出し先**: ECPGconnect, ECPGtrans, ECPGdo, ECPGdisconnect（ECPGランタイムライブラリ関数）。

関連データ構造

  • sqlca: ECPGのSQL通信領域。各操作後の sqlca.sqlcode をチェックしてエラー判定に使用。

失敗モードと境界条件

  • **接続失敗**: データベース接続が確立できない場合、エラーを出力して NULL を返す。
  • **SQLエラー**: ECPGdo 等の各SQL操作で sqlca.sqlcode < 0 となった場合、sqlprint() が実行される。挿入失敗時はエラーメッセージが出力されるが、スレッドは継続する。

保守担当者向けメモ

  • 本関数は回帰テスト用であり、デバッグ出力は無効化されている。
  • Windows環境とPOSIX環境の差異を吸収するため、接続名生成時に #ifndef _MSC_VER による条件分岐が含まれている。
  • スレッドの実行順序は不定であるため、検証は最終的なテーブルの行数に基づいて行われる。
