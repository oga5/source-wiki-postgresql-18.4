get_rte_attribute_name

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:3357

目的

指定された RangeTblEntry (RTE) における特定の属性番号（列番号）に対応する列名を取得する。

シグネチャまたは形状

char *get_rte_attribute_name(RangeTblEntry *rte, AttrNumber attnum)

入力と出力

  • **入力**:
  • rte: 対象となる RangeTblEntry 構造体へのポインタ。
  • attnum: 取得対象の属性番号 (AttrNumber)。
  • **出力**:
  • 成功時: 列名を表す文字列 (char *)。
  • 特殊ケース: attnum が InvalidAttrNumber の場合は "*" を返す。

前提条件と不変条件

  • attnum が有効な範囲内であること。範囲外の場合はエラーが発生する。
  • rte が指す構造体は初期化済みであること。

アルゴリズムまたは振る舞い

1. attnum が InvalidAttrNumber ならば "*" を即座に返す。 2. rte->alias が存在し、attnum がエイリアスの列リスト範囲内であれば、そのエイリアス名を返す。 3. rte->rtekind が RTE_RELATION である場合、システムカタログから最新の列名を取得する（get_attname を使用）。これにより、ルール等で列名が変更された場合でも正確な名前を取得できる。 4. 上記以外の場合、rte->eref->colnames リストから対応する列名を取得する。 5. いずれにも該当しない（無効な attnum）場合は、elog(ERROR) により処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/nodes/print.c
  • src/backend/parser/parse_agg.c
  • src/backend/utils/adt/ruleutils.c
  • **呼び出し先**:
  • get_attname (システムカタログ検索時)
  • list_nth (リストアクセス時)
  • strVal (ノード値抽出時)
  • elog (エラー発生時)

関連データ構造

  • RangeTblEntry: クエリ内のリレーションを表現する構造体。
  • AttrNumber: 列番号を識別する型。

失敗モードと境界条件

  • attnum が rte の保持する列リストの範囲外である場合、elog(ERROR) が発生し処理が停止する。
  • RTE_RELATION 以外で eref->colnames が空または不整合な場合も同様にエラーとなる。

保守担当者向けメモ

  • 本関数は、クエリ解析の過程で列名を解決する際の標準的な手段である。
  • RTE_RELATION の場合にカタログを参照するのは、eref が構築された後に列名が変更される可能性があるためである。この挙動はルール（Rules）の再書き換え等で重要となる。
