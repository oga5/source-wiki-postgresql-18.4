predicate_refuted_by

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/util/predtest.c:221

目的

与えられた clause_list（制約条件）が predicate_list（述語）を論理的に否定（refute）できるか、すなわち「制約条件が真であるとき、述語は必ず偽となるか」を判定します。

シグネチャまたは形状

bool predicate_refuted_by(List *predicate_list, List *clause_list, bool weak)

入力と出力

  • **入力**:
  • predicate_list: 判定対象となる述語のリスト。
  • clause_list: 矛盾の根拠となる制約条件のリスト。
  • weak: 判定の厳密さを制御するフラグ。
  • **出力**:
  • bool: 矛盾が証明された場合に true、それ以外（証明不能または矛盾しない）の場合に false を返します。

前提条件と不変条件

  • predicate_list または clause_list が NIL の場合、矛盾は成立しないため直ちに false を返します。
  • 入力される式は IMMUTABLE であることが前提です。
  • eval_const_expressions 等により、式ツリーは既に正規化されていることが期待されます。

アルゴリズムまたは振る舞い

1. 入力リストが単一要素の場合、リストのオーバーヘッドを避けるために先頭要素を直接抽出します。 2. 抽出したノード（p および c）を引数として、再帰的な矛盾判定関数 predicate_refuted_by_recurse を呼び出します。 3. predicate_refuted_by_recurse の結果をそのまま返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/util/plancat.c
  • src/backend/partitioning/partprune.c
  • src/test/modules/test_predtest/test_predtest.c
  • **呼び出し先**:
  • predicate_refuted_by_recurse（直接の再帰的処理）

関連データ構造

  • List: PostgreSQL のノードリスト構造。
  • Node: 式ツリーを構成する汎用ノード。

失敗モードと境界条件

  • リストが空（NIL）の場合は false を返します。
  • 複雑な AND/OR 構造を持つ式の場合、再帰の深さによりスタックオーバーフローのリスクがあります。
  • 演算子の包含関係がキャッシュ（OprProofCacheHash）に存在しない場合、カタログ参照が発生します。

保守担当者向けメモ

  • 本関数はトップレベルのラッパーであり、実際の論理推論は predicate_refuted_by_recurse に委譲されています。
  • weak パラメータの挙動については、呼び出し先である predicate_refuted_by_recurse の実装詳細を確認してください。
