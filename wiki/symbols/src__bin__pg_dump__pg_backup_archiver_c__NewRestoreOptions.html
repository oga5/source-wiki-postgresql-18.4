NewRestoreOptions

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_archiver.c:1104

目的

復元処理で使用される RestoreOptions 構造体をメモリ上に確保し、デフォルト値で初期化する。

シグネチャまたは形状

RestoreOptions *NewRestoreOptions(void);

入力と出力

  • **入力**: なし。
  • **出力**: 初期化済みの RestoreOptions 構造体へのポインタ。

前提条件と不変条件

  • pg_malloc0 を使用しているため、メモリ確保失敗時はプログラムが終了する。
  • 戻り値の構造体は、呼び出し元で適切に解放する必要がある。

アルゴリズムまたは振る舞い

1. pg_malloc0 を呼び出し、RestoreOptions サイズのメモリをゼロクリアして確保する。 2. 以下のフィールドに初期値を設定する:

  • format: archUnknown
  • cparams.promptPassword: TRI_DEFAULT
  • dumpSections: DUMP_UNSECTIONED
  • compression_spec.algorithm: PG_COMPRESSION_NONE
  • compression_spec.level: 0
  • dumpSchema, dumpData, dumpStatistics: true

    • 3. 初期化されたポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/pg_backup_tar.c:814
  • src/bin/pg_dump/pg_dump.c:1192
  • src/bin/pg_dump/pg_restore.c:155
  • **呼び出し先**:
  • pg_malloc0 (メモリ確保)

関連データ構造

  • RestoreOptions: 復元処理の動作を制御する設定項目を保持する構造体。

失敗モードと境界条件

  • メモリ不足時には pg_malloc0 内部でエラーが発生し、プロセスが終了する。

保守担当者向けメモ

  • 本関数で設定されるデフォルト値は、pg_restore 等のコマンドライン引数によって後から上書きされることを前提としている。新しいオプションを追加する場合は、ここでの初期化漏れに注意すること。
