parse_backup_label

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/backup_label.c:44

目的

バックアップラベルファイルの内容を解析し、WAL位置、タイムライン（TLI）、および増分バックアップに関連するメタデータを抽出して、呼び出し元へ提供します。

シグネチャまたは形状

void parse_backup_label(char *filename, StringInfo buf,
                        TimeLineID *start_tli, XLogRecPtr *start_lsn,
                        TimeLineID *previous_tli, XLogRecPtr *previous_lsn)

入力と出力

  • **入力**:
  • filename: エラーメッセージに使用するファイル名。
  • buf: 解析対象のデータを含む StringInfo バッファ。
  • **出力**:
  • start_tli, start_lsn: 抽出された開始タイムラインとWAL位置。
  • previous_tli, previous_lsn: 抽出された増分バックアップの起点となるタイムラインとLSN（存在する場合）。

前提条件と不変条件

  • buf は有効なバックアップラベルファイルの内容を保持している必要があります。
  • 解析中に必須項目（START WAL LOCATION, START TIMELINE）が見つからない場合、または増分情報が不完全（片方のみ存在）な場合は pg_fatal によりプロセスが終了します。

アルゴリズムまたは振る舞い

1. 出力引数を初期化します。 2. buf->cursor を進めながら、get_eol_offset を用いて行単位でバッファを走査します。 3. line_starts_with を使用して各行を識別し、以下のキーを検索します：

  • START WAL LOCATION: parse_lsn で値を抽出。
  • START TIMELINE: parse_tli で値を抽出。
  • INCREMENTAL FROM LSN: parse_lsn で値を抽出。
  • INCREMENTAL FROM TLI: parse_tli で値を抽出。

    • 4. 走査終了後、必須項目の有無と増分情報の整合性（LSNとTLIが両方揃っているか）を検証します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_combinebackup/pg_combinebackup.c:551
  • **呼び出し先**: get_eol_offset, line_starts_with, parse_lsn, parse_tli, pg_fatal

関連データ構造

  • StringInfo: PostgreSQLの動的文字列バッファ。解析状態（cursor）の管理に使用されます。
  • TimeLineID, XLogRecPtr: WALおよびタイムライン情報の型。

失敗モードと境界条件

  • 必須項目が欠落している場合、pg_fatal を呼び出し異常終了します。
  • 増分バックアップのメタデータにおいて、LSNとTLIのいずれか一方のみが存在する場合、整合性エラーとして異常終了します。
  • parse_lsn や parse_tli が解析に失敗した場合、エラーメッセージを出力して異常終了します。

保守担当者向けメモ

  • 本関数は buf->cursor を更新しながらバッファを走査します。
  • parse_lsn や parse_tli はバッファ内の文字を一時的に \0 で置換する可能性があるため、バッファの破壊的変更を伴う点に注意してください。
