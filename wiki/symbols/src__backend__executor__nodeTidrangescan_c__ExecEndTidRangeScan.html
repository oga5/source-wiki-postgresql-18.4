ExecEndTidRangeScan

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeTidrangescan.c:338

目的

TID範囲スキャン実行ノードに関連付けられたリソースを解放し、スキャンを終了する。

シグネチャまたは形状

void ExecEndTidRangeScan(TidRangeScanState *node)

入力と出力

  • **入力**: TidRangeScanState *node - 終了対象となるTID範囲スキャンノードの状態構造体。
  • **出力**: なし（void）。

前提条件と不変条件

  • node は有効な TidRangeScanState 構造体を指している必要がある。
  • node->ss.ss_currentScanDesc が有効なスキャン記述子を保持している場合、その記述子は table_endscan を通じて適切にクリーンアップされる。

アルゴリズムまたは振る舞い

1. node から現在のスキャン記述子 ss_currentScanDesc を取得する。 2. スキャン記述子が NULL でない場合、table_endscan(scan) を呼び出し、テーブルアクセス層におけるスキャンリソースを解放する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execProcnode.c:660 から呼び出されることが確認されている。
  • **呼び出し先**: table_endscan（テーブルアクセス層の関数）。

関連データ構造

  • TidRangeScanState: TID範囲スキャンの実行状態を保持する構造体。
  • TableScanDesc: テーブルスキャンの状態を管理する記述子。

失敗モードと境界条件

  • node->ss.ss_currentScanDesc が NULL の場合、table_endscan は呼び出されず、関数は何もせず正常に終了する。

保守担当者向けメモ

  • 本関数は ExecEndNode の呼び出しチェーンの一部として実行される。
  • table_endscan の呼び出しは、Table AM（テーブルアクセス層）のインターフェースに依存しているため、将来的なストレージエンジンの変更時にもこの契約が維持される必要がある。
