BufTableShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/storage/buffer/buf_table.c:40

目的

共有バッファテーブル（SharedBufHash）を共有メモリ上に構築するために必要なメモリサイズを算出します。

シグネチャまたは形状

Size BufTableShmemSize(int size)

入力と出力

  • **入力**: size（ハッシュテーブルに格納するエントリ数）
  • **出力**: Size 型（必要な共有メモリのバイト数）

前提条件と不変条件

  • 引数 size は、バッファプール内のバッファ数およびパーティション数に基づいた適切な値である必要があります。

アルゴリズムまたは振る舞い

hash_estimate_size 関数を呼び出し、指定されたエントリ数 size と各エントリのサイズ sizeof(BufferLookupEnt) に基づいて、共有ハッシュテーブルの構築に必要なメモリ量を推定して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/buffer/freelist.c（共有メモリの総サイズ計算時に使用）
  • **呼び出し先**: hash_estimate_size（PostgreSQLのハッシュテーブル管理ユーティリティ）

関連データ構造

  • BufferLookupEnt: ハッシュテーブルの各エントリの構造体。本関数はこのサイズを基準にメモリ量を算出します。

失敗モードと境界条件

  • hash_estimate_size の内部仕様に依存します。引数 size が負の値や極端に大きい値である場合の挙動については、本ソース excerpt には記載がありません。

保守担当者向けメモ

  • 本関数は共有メモリの割当計画段階で使用されます。バッファ数やパーティション数の定義が変更された場合、呼び出し側の引数計算が正しく行われているか確認してください。
