win32_get_codeset

定義

  • 種別: function
  • 定義箇所: src/port/chklocale.c:201

目的

Windows 環境において、ロケール文字列から PostgreSQL が内部的に使用するコードセット名（例: CP1252 や utf8）を解決する。

シグネチャまたは形状

static char *win32_get_codeset(const char *ctype);

入力と出力

  • **入力**: ctype (ロケール名を表す文字列)
  • **出力**: 解決されたコードセット名を含む動的に確保された文字列（malloc 済み）。解決できない場合やメモリ確保に失敗した場合は NULL を返す。

前提条件と不変条件

  • 入力 ctype は有効なロケール文字列であること。
  • 戻り値の文字列は呼び出し元で free する必要がある。

アルゴリズムまたは振る舞い

1. MultiByteToWideChar を使用して入力ロケール名をワイド文字列に変換する。 2. GetLocaleInfoEx を呼び出し、システムから ANSI コードページ番号を取得する。

  • 取得成功時: コードページが CP_ACP であれば "utf8" を、それ以外は "CP<番号>" という形式の文字列を生成する。

    • 3. GetLocaleInfoEx が失敗した場合のフォールバック:

  • ロケール文字列の最後のドット (.) 以降を抽出する。
  • 抽出部分が数字のみであれば "CP<数字>" とし、それ以外であればそのままの文字列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/port/chklocale.c 内の関数から呼び出される（直接的な上位関数名はソース抜粋からは特定不可）。
  • **呼び出し先**: Windows API (MultiByteToWideChar, GetLocaleInfoEx) および標準ライブラリ関数 (malloc, sprintf, strrchr, strspn, strcpy)。

関連データ構造

  • WCHAR: Windows API で使用されるワイド文字型。
  • LOCALE_NAME_MAX_LENGTH: ロケール名バッファの最大長。

失敗モードと境界条件

  • GetLocaleInfoEx が失敗し、かつロケール文字列にドットが含まれない場合、戻り値は NULL となる。
  • malloc が失敗した場合、戻り値は NULL となる。
  • cp == CP_ACP の場合は ANSI コードページが利用不可と判断し、"utf8" を返す。

保守担当者向けメモ

  • 本関数は Windows 特有のロケール解決ロジックをカプセル化している。
  • GetLocaleInfoEx が失敗した際のフォールバック処理は、Unix 形式のロケール文字列が Windows の setlocale() に渡された場合を想定している。
