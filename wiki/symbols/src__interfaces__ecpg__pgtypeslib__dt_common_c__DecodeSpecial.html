DecodeSpecial

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/pgtypeslib/dt_common.c:634

目的

日付・時刻に関連する特殊なトークン（キーワード）を検索し、その型と値を特定します。

シグネチャまたは形状

int DecodeSpecial(int field, char *lowtoken, int *val);

入力と出力

  • **入力**:
  • field: キャッシュインデックスとして使用する整数値。
  • lowtoken: 検索対象の文字列（小文字化済み）。
  • val: 検索結果の値を格納するポインタ。
  • **出力**:
  • 戻り値: トークンの型（int）。見つからない場合は UNKNOWN_FIELD を返す。
  • val 経由: トークンに対応する数値（見つからない場合は 0）。

前提条件と不変条件

  • lowtoken は検索可能な文字列であること。
  • datecache 配列が field インデックスに対して有効なメモリ領域を指していること。
  • datetktbl および szdatetktbl が初期化されていること。

アルゴリズムまたは振る舞い

1. datecache[field] を確認し、キャッシュされたトークンと lowtoken が一致すればそれを使用する。 2. 一致しない場合、datebsearch を用いて datetktbl から lowtoken をバイナリサーチする。 3. 検索結果を datecache[field] に保存する。 4. トークンが見つかった場合、その型を戻り値とし、値を *val に代入する。 5. 見つからない場合、型として UNKNOWN_FIELD を返し、*val を 0 に設定する。

呼び出し元と呼び出し先

  • **呼び出し元**: ツールによる解析では、src/backend/utils/adt/ 配下の date.c, datetime.c, timestamp.c 等からの参照が示唆されていますが、これらはバックエンド側のコードであり、本ライブラリ（ECPG）との直接的な呼び出し関係については、提供されたソース excerpt 内では明示されていません。
  • **呼び出し先**: datebsearch（トークン検索用）。

関連データ構造

  • datetkn: 日付・時刻のキーワードを保持する構造体。
  • datecache: トークン検索結果を保持するキャッシュ配列。
  • datetktbl: 検索対象となるキーワードテーブル。

失敗モードと境界条件

  • lowtoken がテーブル内に存在しない場合、tp は NULL となり、UNKNOWN_FIELD が返されます。
  • field インデックスが datecache の範囲外である場合、未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本関数は datecache を介して検索を最適化しています。キャッシュの整合性が保たれていることを確認してください。
  • strncmp を使用しているため、トークンの前方一致によるマッチングが行われます。
