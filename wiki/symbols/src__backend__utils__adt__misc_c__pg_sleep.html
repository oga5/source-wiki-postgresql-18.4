pg_sleep

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/misc.c:369

目的

指定された秒数だけ現在のバックエンドプロセスを待機させます。シグナル（SIGALRMやSIGINTなど）に対して即座に応答できるよう設計されています。

シグネチャまたは形状

Datum pg_sleep(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_FLOAT8(0) により、待機秒数（float8）を取得します。
  • **出力**: なし（PG_RETURN_VOID()）。

前提条件と不変条件

  • WaitLatch の制限（最大 INT_MAX ミリ秒）を回避するため、内部的に最大10分単位のループ処理を行います。
  • 待機終了時刻を最初に計算することで、複数回のループによる遅延の累積を防ぎます。
  • CHECK_FOR_INTERRUPTS() をループ内で呼び出し、割り込み要求を監視します。

アルゴリズムまたは振る舞い

1. 現在時刻に指定秒数を加算し、終了時刻（endtime）を算出します。 2. ループ内で以下を繰り返します。

  • CHECK_FOR_INTERRUPTS() で割り込みを確認します。
  • 残り時間を計算し、600秒（10分）を超える場合は600秒に制限します。
  • WaitLatch を呼び出し、指定時間またはシグナル受信まで待機します。
  • ResetLatch でラッチをリセットします。
  • 残り時間が0以下になったらループを終了します。

呼び出し元と呼び出し先

  • **呼び出し元**: 外部からのSQLクエリ（例: SELECT pg_sleep($1)）。ソースコード内では src/test/modules/libpq_pipeline/libpq_pipeline.c:188 での利用が確認されています。
  • **呼び出し先**: WaitLatch, ResetLatch, CHECK_FOR_INTERRUPTS, GetCurrentTimestamp, ceil。

関連データ構造

  • MyLatch: 現在のバックエンドのラッチ。
  • WAIT_EVENT_PG_SLEEP: 待機イベント識別子。

失敗モードと境界条件

  • 負の秒数が指定された場合、ループ条件により即座に終了します。
  • WaitLatch がシグナルによって早期終了した場合でも、終了時刻の計算に基づき、指定された合計時間まで待機を継続します。

保守担当者向けメモ

  • 本関数は WaitLatch を用いたループ処理の典型例です。シグナル割り込みを適切に処理する設計となっており、同様の待機処理を実装する際の参考になります。
