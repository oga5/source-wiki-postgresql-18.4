LZ4Stream_init

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/compress_lz4.c:353

目的

LZ4 圧縮または解凍のためのストリームコンテキストを初期化します。

シグネチャまたは形状

static bool LZ4Stream_init(LZ4State *state, int size, bool compressing)

入力と出力

  • **入力**:
  • state: LZ4State 構造体へのポインタ。
  • size: バッファサイズのヒント。
  • compressing: 圧縮モードであれば true、解凍モードであれば false。
  • **出力**:
  • 成功時に true、失敗時に false を返します。

前提条件と不変条件

  • state->inited が true の場合、初期化はスキップされ直ちに true を返します。
  • 圧縮モードの場合、LZ4State_compression_init が成功し、かつヘッダーの書き込みが完了している必要があります。

アルゴリズムまたは振る舞い

1. state->inited を確認し、既に初期化済みであれば終了します。 2. compressing が true の場合:

  • LZ4State_compression_init を呼び出し、圧縮コンテキストを準備します。
  • state->buffer に格納された LZ4 ヘッダーを state->fp へ書き込みます。

    • 3. compressing が false の場合:

  • LZ4F_createDecompressionContext を呼び出し、解凍コンテキストを作成します。
  • size と DEFAULT_IO_BUFFER_SIZE の大きい方をバッファ長として採用し、state->buffer および state->overflowbuf を割り当てます。

    • 4. 成功時に state->inited を true に設定します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/compress_lz4.c:354 (再帰的または内部参照の可能性あり)
  • src/bin/pg_dump/compress_lz4.c:461 (解凍時)
  • src/bin/pg_dump/compress_lz4.c:580 (圧縮時)
  • **呼び出し先**:
  • LZ4State_compression_init
  • LZ4F_createDecompressionContext
  • LZ4F_isError
  • pg_malloc
  • fwrite

関連データ構造

  • LZ4State: 圧縮・解凍のコンテキスト、バッファ、ストリーム状態を保持する構造体。

失敗モードと境界条件

  • fwrite が期待されるバイト数を書き込めなかった場合、errno を設定し false を返します。
  • LZ4F_createDecompressionContext がエラーを返した場合、state->errcode にエラーコードを格納し false を返します。
  • メモリ割り当て (pg_malloc) が失敗した場合は、標準的なメモリ不足エラーが発生します。

保守担当者向けメモ

  • 本関数は遅延初期化（Lazy Initialization）を採用しています。
  • 圧縮モードと解凍モードで初期化処理が大きく分岐するため、変更時は両方のパスを確認してください。
