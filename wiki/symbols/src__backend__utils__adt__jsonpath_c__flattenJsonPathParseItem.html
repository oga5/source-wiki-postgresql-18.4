flattenJsonPathParseItem

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonpath.c:238

目的

SQL/JSONパス式の抽象構文木（AST）を再帰的に走査し、クエリ実行エンジンが評価可能な4バイトアライメントされたバイナリ形式へ変換（シリアライズ）します。

シグネチャまたは形状

static bool flattenJsonPathParseItem(
    StringInfo buf, 
    int *result, 
    struct Node *escontext,
    JsonPathParseItem *item, 
    int nestingLevel,
    bool insideArraySubscript
)

入力と出力

  • **入力**:
  • buf: シリアライズ結果を格納する StringInfo バッファ。
  • item: 変換対象の JsonPathParseItem（ASTノード）。
  • nestingLevel: 現在のネスト深さ。
  • insideArraySubscript: 配列添字内かどうかのフラグ。
  • escontext: エラー報告用コンテキスト。
  • **出力**:
  • result: 処理されたアイテムのバッファ先頭からの相対オフセットが格納されます。
  • 戻り値: 成功時に true、エラー発生時に false を返します。

前提条件と不変条件

  • buf は JSONPATH_HDRSZ をヘッダーとして保持している必要があります。
  • 再帰呼び出しを行うため、スタック深さのチェック（check_stack_depth()）が実行されます。
  • バイナリ形式は4バイトアライメントを維持し、ノード間のリンクは絶対アドレスではなくバッファ先頭からの相対オフセットで管理されます。

アルゴリズムまたは振る舞い

1. 現在のアイテムタイプをバッファに書き込み、アライメントを調整します。 2. reserveSpaceForItemPointer を呼び出し、後続ノードへのポインタ用領域を予約します。 3. item->type に応じて以下の処理を行います：

  • **文字列/変数/キー**: 値の長さと内容をバッファに追加します。
  • **算術/比較演算子**: 左右の引数を再帰的に処理し、予約した領域に相対オフセットを書き込みます。
  • **正規表現**: フラグ、パターン、および式を順次書き込みます。
  • **単項演算子/フィルタ**: 引数を再帰的に処理し、オフセットを更新します。
  • **配列添字**: 要素数を書き込み、各要素の from/to を再帰的に処理します。

    • 4. @（jpiCurrent）や LAST の使用箇所において、コンテキストに応じた構文チェックを行います。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の jsonpath_in 等のパス式変換プロセスから再帰的に呼び出されます。
  • **呼び出し先**: appendStringInfoChar, alignStringInfoInt, reserveSpaceForItemPointer, appendBinaryStringInfo, check_stack_depth, CHECK_FOR_INTERRUPTS, ereturn 等。

関連データ構造

  • JsonPathParseItem: 変換対象のASTノード。
  • StringInfo: バイナリ構築用バッファ。
  • JsonPathItemType: ノードの種類を定義する列挙型。

失敗モードと境界条件

  • **スタックオーバーフロー**: 複雑なパス式で再帰が深すぎる場合に発生します。
  • **構文エラー**: @ がルート式で使用された場合や、LAST が配列添字以外で使用された場合に ereturn を介してエラーを報告します。
  • **メモリ不足**: StringInfo の拡張失敗時に発生する可能性があります。

保守担当者向けメモ

  • 本関数はバイナリ形式の「連結リスト」構造を構築します。next ポインタや引数へのオフセットは、ノード処理後に確定するため、reserveSpaceForItemPointer で先行予約するパターンを厳守してください。
  • jpiFilter 等でネストレベルが変更される点に注意してください。
