gimme_gene

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/optimizer/geqo/geqo_erx.c:281

目的

エッジ組み換え交叉（ERX）アルゴリズムにおいて、現在の遺伝子（都市）から次に接続すべき遺伝子を選択する。

シグネチャまたは形状

Gene gimme_gene(PlannerInfo *root, Edge edge, Edge *edge_table)

入力と出力

  • **入力**:
  • root: クエリ最適化のコンテキスト情報。
  • edge: 現在の遺伝子が持つ接続先リスト（Edge 構造体）。
  • edge_table: 全遺伝子の接続状況を保持するテーブル。
  • **出力**: 次に選択された Gene（都市ID）。

前提条件と不変条件

  • edge.edge_list には有効な接続先候補が含まれていること。
  • edge_table 内の各都市の unused_edges は、ツアーへの組み込み状況を反映していること。
  • 負の値を持つ friend は共有エッジとして扱われ、優先的に選択される。

アルゴリズムまたは振る舞い

1. **共有エッジの優先**: edge_list 内に負の値（共有エッジ）が存在する場合、即座にその絶対値を返す。 2. **最小接続数の特定**: 共有エッジがない場合、候補の中で edge_table における unused_edges が最小であるものを探す。 3. **ランダム選択**: 最小の unused_edges を持つ候補が複数存在する場合、geqo_randint を用いてその中からランダムに一つを選択する。 4. **終了**: 選択された Gene を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/geqo/geqo_erx.c 内の gimme_tour 関数から呼び出される。
  • **呼び出し先**: 乱数生成のために geqo_randint を呼び出す。

関連データ構造

  • Edge: 都市間の接続関係と未使用エッジ数を保持する構造体。
  • Gene: 都市を識別する型。

失敗モードと境界条件

  • **エラー**: 候補となるエッジが見つからない場合、または minimum_count が正しく設定されない場合、elog(ERROR) により処理を中断する。
  • **境界条件**: 接続先候補が 0 の場合、または全ての候補が既にツアーに組み込まれている場合、本関数は到達不能なコードパスとしてエラーを発生させる。

保守担当者向けメモ

  • minimum_count のロジックは、接続の選択肢が少ない都市を優先するヒューリスティックであり、この条件分岐が失敗することはアルゴリズム上の不整合を意味する。
  • friend < 0 のチェックは、共有エッジを優先するというERXの基本戦略を実装している。
