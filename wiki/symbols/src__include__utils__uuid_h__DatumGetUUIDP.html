DatumGetUUIDP

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/utils/uuid.h:34

目的

PostgreSQLの Datum 型として保持されているUUIDデータを、pg_uuid_t 構造体へのポインタとして安全にキャストして取得する。

シグネチャまたは形状

static inline pg_uuid_t *DatumGetUUIDP(Datum X)

入力と出力

  • **入力**: Datum X（UUIDデータへのポインタを含む Datum 値）
  • **出力**: pg_uuid_t *（UUIDデータ構造体へのポインタ）

前提条件と不変条件

  • 入力 Datum X は、有効な pg_uuid_t 構造体を指すポインタを保持している必要がある。
  • pg_uuid_t は16バイトの固定長データ構造である。

アルゴリズムまたは振る舞い

1. DatumGetPointer(X) を呼び出し、Datum から汎用ポインタ型へ変換する。 2. 変換されたポインタを pg_uuid_t * 型にキャストして返却する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/btree_gist/btree_uuid.c
  • src/backend/access/brin/brin_minmax_multi.c
  • src/backend/utils/adt/uuid.c
  • src/include/utils/uuid.h（PG_GETARG_UUID_P マクロ経由）
  • **呼び出し先**:
  • DatumGetPointer（PostgreSQLのコアマクロ）

関連データ構造

  • pg_uuid_t: UUIDデータを保持する16バイトの構造体。

失敗モードと境界条件

  • 入力 Datum が NULL ポインタを指している場合、返却されたポインタをデリファレンスするとクラッシュする可能性がある。本関数自体はポインタの妥当性チェックを行わない。

保守担当者向けメモ

  • 本関数はインライン定義されているため、変更時は src/include/utils/uuid.h を再コンパイルする必要がある。
  • システム定義の uuid_t との衝突を避けるため、必ず pg_uuid_t を介してアクセスすること。
