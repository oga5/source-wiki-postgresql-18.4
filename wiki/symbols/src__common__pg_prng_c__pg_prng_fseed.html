pg_prng_fseed

定義

  • 種別: function
  • 定義箇所: src/common/pg_prng.c:101

目的

浮動小数点数（double）のシード値を用いて、擬似乱数生成器（PRNG）の状態を初期化します。

シグネチャまたは形状

void pg_prng_fseed(pg_prng_state *state, double fseed)

入力と出力

  • **入力**:
  • state: 初期化対象となる pg_prng_state 構造体へのポインタ。
  • fseed: 0.0 から 1.0 の範囲を想定したシード値（double）。
  • **出力**: なし（state が指す構造体が更新されます）。

前提条件と不変条件

  • fseed は double 型であり、内部で約52ビットの仮数部を利用して int64 型のシード値へ変換されます。
  • 変換後のシード値は pg_prng_seed に渡され、生成器の状態が設定されます。

アルゴリズムまたは振る舞い

1. fseed に (2^52 - 1) を乗算し、int64 型の整数シードへ変換します。 2. 変換された整数値を pg_prng_seed 関数へ渡し、pg_prng_state の初期化を実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/optimizer/geqo/geqo_random.c:24
  • src/backend/utils/adt/pseudorandomfuncs.c:72
  • **呼び出し先**:
  • pg_prng_seed（同ファイル内）

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • fseed が 0.0 から 1.0 の範囲外である場合の挙動については、ソースコード上に明示的なガード条件は記述されていません。
  • 内部で呼び出される pg_prng_seed は、生成器の状態がすべて0になることを防ぐチェック（pg_prng_seed_check）を行うため、初期化後の生成器が不動点（0を返し続ける状態）に陥ることは回避されます。

保守担当者向けメモ

  • 本関数は暗号学的に安全な乱数生成を目的としていません。暗号用途には使用しないでください。
  • fseed から int64 への変換は、浮動小数点数の精度に依存します。より広範なエントロピーが必要な場合は、他のシード関数を検討してください。
