LagTracker

定義

  • 種別: struct
  • 定義箇所: src/backend/replication/walsender.c:226

目的

スタンバイサーバーからのフィードバックに基づき、レプリケーションのラグ（遅延）を追跡・計算するための循環バッファ構造体です。

シグネチャまたは形状

typedef struct {
    XLogRecPtr    last_lsn;
    WalTimeSample buffer[LAG_TRACKER_BUFFER_SIZE];
    int           write_head;
    int           read_heads[NUM_SYNC_REP_WAIT_MODE];
    WalTimeSample last_read[NUM_SYNC_REP_WAIT_MODE];
    WalTimeSample overflowed[NUM_SYNC_REP_WAIT_MODE];
} LagTracker;

入力と出力

  • **入力**: スタンバイから報告される LSN（Log Sequence Number）およびタイムスタンプ情報。
  • **出力**: 各同期待機モード（NUM_SYNC_REP_WAIT_MODE）におけるラグ計算結果。

前提条件と不変条件

  • 循環バッファとして動作し、write_head が read_heads を追い越す可能性がある。
  • overflowed 配列は、書き込みヘッドと読み取りヘッドが衝突した際に、ラグ計算を継続するための退避先として使用される。

アルゴリズムまたは振る舞い

1. **バッファ管理**: write_head が循環バッファを更新し、各 read_heads が個別の追跡位置を保持する。 2. **オーバーフロー処理**: 書き込みヘッドが読み取りヘッドに衝突した場合、該当する読み取りヘッドの現在のサンプルを overflowed に保存し、read_heads[i] を -1 に設定してオーバーフロー状態を示す。 3. **復帰**: スタンバイから報告された LSN が overflowed に保存された LSN を超えた時点で、通常のバッファベースの追跡へ自動的に遷移する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/walsender.c 内で lag_tracker として静的に確保され、MemoryContextAllocZero を通じて初期化される。
  • **呼び出し先**: 情報なし。

関連データ構造

  • WalTimeSample: ラグ計算に使用される時系列サンプルデータ。
  • NUM_SYNC_REP_WAIT_MODE: 同期レプリケーションの待機モード数。
  • XLogRecPtr: WAL の位置を示すポインタ。

失敗モードと境界条件

  • **バッファ衝突**: write_head と read_heads が衝突した場合、overflowed 配列を使用してデータ欠損を防ぐ設計となっている。

保守担当者向けメモ

  • 本構造体は walsender.c 内の lag_tracker ポインタを通じてグローバルに管理されています。
  • NUM_SYNC_REP_WAIT_MODE の変更は構造体のサイズに直接影響するため注意してください。
