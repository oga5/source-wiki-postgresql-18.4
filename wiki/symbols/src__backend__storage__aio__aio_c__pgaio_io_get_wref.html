pgaio_io_get_wref

定義

  • 種別: function
  • 定義箇所: src/backend/storage/aio/aio.c:365

目的

指定された非同期I/Oハンドル（PgAioHandle）を識別するための待機参照（PgAioWaitRef）を生成します。これにより、後続の完了待ち合わせ処理が可能になります。

シグネチャまたは形状

void pgaio_io_get_wref(PgAioHandle *ioh, PgAioWaitRef *iow)

入力と出力

  • **入力**:
  • ioh: 参照対象の PgAioHandle へのポインタ。
  • iow: 情報を格納する PgAioWaitRef 構造体へのポインタ。
  • **出力**: なし（iow が更新されます）。

前提条件と不変条件

  • ioh->state が PGAIO_HS_HANDED_OUT、PGAIO_HS_DEFINED、または PGAIO_HS_STAGED のいずれかであること。
  • ioh->generation が 0 ではないこと。

アルゴリズムまたは振る舞い

1. ioh の状態と世代番号の妥当性をアサーションで検証します。 2. ioh のポインタ演算により、グローバルな pgaio_ctl->io_handles 配列内でのインデックスを計算し、iow->aio_index に格納します。 3. ioh->generation（64ビット）を上位32ビットと下位32ビットに分割し、それぞれ iow->generation_upper および iow->generation_lower に格納します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c
  • src/test/modules/test_aio/test_aio.c
  • src/backend/storage/aio/aio.c（内部ドキュメントによる言及）
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • PgAioHandle: I/O操作のライフサイクルを管理する構造体。
  • PgAioWaitRef: I/O完了を待機するために使用される識別子。
  • pgaio_ctl: AIOサブシステム全体の制御構造体。

失敗モードと境界条件

  • ioh が無効な状態（IDLE 等）である場合、アサーション違反によりプロセスが停止します。
  • pgaio_ctl が初期化されていない状態で呼び出された場合、メモリ参照エラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は、I/Oが開始される前であっても、ハンドルが確保されていれば呼び出し可能です。
  • generation の分割格納は、ハンドル再利用時の誤った完了判定を防ぐための識別子として機能します。
