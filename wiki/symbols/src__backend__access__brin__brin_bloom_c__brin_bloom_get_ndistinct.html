brin_bloom_get_ndistinct

定義

  • 種別: function
  • 定義箇所: src/backend/access/brin/brin_bloom.c:494

目的

BRINインデックスのブルームフィルタ作成時に使用する、ページ範囲あたりの推定異なり値数（n_distinct）を算出する。

シグネチャまたは形状

int brin_bloom_get_ndistinct(BrinDesc *bdesc, BloomOptions *opts)

入力と出力

  • **入力**:
  • bdesc: インデックスの記述子（BrinDesc）。ページ範囲サイズを取得するために使用。
  • opts: ブルームフィルタのオプション設定（BloomOptions）。
  • **出力**:
  • ページ範囲あたりの推定異なり値数（int）。

前提条件と不変条件

  • bdesc から取得される pagesPerRange は有効なブロック番号であること。
  • 算出結果は BLOOM_MIN_NDISTINCT_PER_RANGE 以上、かつページ範囲内の最大タプル数（maxtuples）以下に制限される。

アルゴリズムまたは振る舞い

1. インデックスのページ範囲サイズ（pagesPerRange）と、オプションから得られるベースの ndistinct を取得する。 2. ページ範囲内の最大タプル数（maxtuples）を計算する。 3. ndistinct が負の値の場合、それは maxtuples に対する相対値とみなし、絶対値を乗算して実数に変換する。 4. 算出された値を BLOOM_MIN_NDISTINCT_PER_RANGE で下限補正し、maxtuples で上限補正する。 5. 最終的な値を int 型にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/brin/brin_bloom.c 内の bloom_init 呼び出し時に使用される。
  • **呼び出し先**:
  • BrinGetPagesPerRange (マクロ)
  • BloomGetNDistinctPerRange (マクロ)
  • Max / Min (マクロ)

関連データ構造

  • BrinDesc: インデックスのメタデータ構造体。
  • BloomOptions: ブルームフィルタの構成オプション。

失敗モードと境界条件

  • ndistinct が負の値の場合、maxtuples を基準とした相対計算が行われる。
  • 算出値が極端に小さい場合は BLOOM_MIN_NDISTINCT_PER_RANGE に、大きすぎる場合は maxtuples に丸められるため、メモリ不足や無駄なフィルタ生成を防ぐ境界条件が適用される。

保守担当者向けメモ

  • 本関数はブルームフィルタのサイズ決定の根拠となるため、誤判定率（false positive rate）の調整と連動している。
  • maxtuples の計算には MaxHeapTuplesPerPage が使用されており、ページサイズが固定であることを前提としている。
