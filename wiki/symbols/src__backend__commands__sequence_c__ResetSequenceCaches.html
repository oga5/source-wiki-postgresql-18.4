ResetSequenceCaches

定義

  • 種別: function
  • 定義箇所: src/backend/commands/sequence.c:1944

目的

セッション内で保持されているシーケンスのキャッシュ状態を破棄し、メモリを解放する。

シグネチャまたは形状

void ResetSequenceCaches(void)

入力と出力

  • 入力: なし
  • 出力: なし（副作用としてグローバル変数の状態を変更する）

前提条件と不変条件

  • seqhashtab が存在する場合、そのハッシュテーブルは破棄され、ポインタは NULL にリセットされる。
  • last_used_seq は常に NULL にリセットされる。

アルゴリズムまたは振る舞い

1. seqhashtab が非 NULL であるかを確認する。 2. 存在する場合、hash_destroy を呼び出してハッシュテーブルを破棄し、seqhashtab を NULL に設定する。 3. 最後に使用されたシーケンスへのポインタ last_used_seq を NULL に設定する。

呼び出し元と呼び出し先

  • 呼び出し元:
  • src/backend/commands/discard.c:44
  • src/backend/commands/discard.c:77
  • 呼び出し先:
  • hash_destroy（PostgreSQLのハッシュテーブル管理関数）

関連データ構造

  • seqhashtab: セッション内でアクセスされたシーケンスを追跡するためのハッシュテーブル。
  • last_used_seq: 最後にアクセスされたシーケンスのキャッシュエントリを指すポインタ。

失敗モードと境界条件

  • seqhashtab が既に NULL の場合、関数は何もしない（安全に呼び出し可能）。

保守担当者向けメモ

  • 本関数は主に DISCARD ALL コマンド等のセッション状態のリセット処理から呼び出される。
  • シーケンスのキャッシュを強制的にクリアするため、次回シーケンスアクセス時には再初期化（init_sequence）が発生する。
