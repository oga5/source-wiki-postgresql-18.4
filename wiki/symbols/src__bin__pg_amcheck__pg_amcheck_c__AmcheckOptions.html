AmcheckOptions

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_amcheck/pg_amcheck.c:55

目的

pg_amcheck コマンドラインツールにおけるユーザー指定の実行オプション、検証対象のフィルタリング条件、およびヒープ・B-tree 固有の検証パラメータを保持する構造体です。

シグネチャまたは形状

typedef struct AmcheckOptions
{
    bool        dbpattern;
    bool        alldb;
    bool        echo;
    bool        verbose;
    bool        strict_names;
    bool        show_progress;
    int         jobs;
    bool        install_missing;
    char       *install_schema;
    PatternInfoArray include;
    PatternInfoArray exclude;
    bool        excludetbl;
    bool        excludeidx;
    bool        excludensp;
    bool        allrel;
    bool        no_toast_expansion;
    bool        reconcile_toast;
    bool        on_error_stop;
    int64       startblock;
    int64       endblock;
    const char *skip;
    bool        parent_check;
    bool        rootdescend;
    bool        heapallindexed;
    bool        checkunique;
    bool        no_btree_expansion;
} AmcheckOptions;

入力と出力

本構造体は、コマンドライン引数の解析結果を格納する入力データとして機能します。出力として、プログラム内の各検証関数（prepare_heap_command や prepare_btree_command 等）が参照する設定値を提供します。

前提条件と不変条件

  • excludetbl, excludeidx, excludensp は、exclude リストの内容と常に整合している必要があります。
  • allrel は、include リストが空である場合にのみ真となります。

アルゴリズムまたは振る舞い

プログラムの初期化フェーズで静的変数 opts としてインスタンス化され、コマンドライン引数に基づいて設定されます。以降の処理において、検証対象の絞り込みや、contrib/amcheck 拡張機能へ渡す SQL クエリの構築ロジックの制御に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_amcheck/pg_amcheck.c 内の main 関数にて初期化されます。
  • 呼び出し先: 本構造体自体はデータ保持用であり、関数を呼び出すことはありませんが、prepare_heap_command や prepare_btree_command などの関数によってその値が参照されます。

関連データ構造

  • PatternInfoArray: 検証対象の包含・除外パターンを保持します。

失敗モードと境界条件

  • startblock および endblock の値が不正な場合、検証クエリの構築時に SQL エラーが発生する可能性があります。
  • install_missing が真の場合、対象スキーマが存在しない、あるいは権限不足により拡張機能のインストールに失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は pg_amcheck の全ライフサイクルを通じて参照されるため、新しい検証オプションを追加する際は、本構造体へのフィールド追加と、それに対応する引数解析ロジックの更新が必要です。
  • excludetbl 等の最適化フラグは、exclude リストを走査するコストを削減するために存在します。リストの内容を変更した際は、これらのフラグの同期を忘れないでください。
