wait_for_worker_state_change

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/tablesync.c:230

目的

論理レプリケーションの同期ワーカーにおいて、適用ワーカー（Apply Worker）が特定の状態へ遷移するのを待機します。

シグネチャまたは形状

static bool wait_for_worker_state_change(char expected_state)

入力と出力

  • **入力**: expected_state (待機対象となる char 型の状態コード)
  • **出力**: bool 型。期待する状態に到達した場合は true、適用ワーカーが終了した場合は false を返します。

前提条件と不変条件

  • MyLogicalRepWorker が有効なワーカーコンテキストを指している必要があります。
  • 呼び出し元は同期ワーカーのプロセス内である必要があります。

アルゴリズムまたは振る舞い

1. 現在のワーカー状態 (MyLogicalRepWorker->relstate) が expected_state と一致するか確認し、一致すれば直ちに true を返します。 2. 一致しない場合、LogicalRepWorkerLock を取得し、サブスクリプションに関連付けられた適用ワーカーを検索します。 3. 適用ワーカーが存在する場合、logicalrep_worker_wakeup_ptr を呼び出して通知を送ります。 4. WaitLatch を使用して、適用ワーカーからの通知またはタイムアウト（1秒）を待機します。 5. タイムアウトまたは通知受信後、ループの先頭に戻り状態を再確認します。 6. 適用ワーカーが見つからない（終了している）場合はループを脱出し、false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/tablesync.c:1595 (同期プロセス中の状態遷移待機として使用)
  • **呼び出し先**:
  • CHECK_FOR_INTERRUPTS()
  • LWLockAcquire / LWLockRelease
  • logicalrep_worker_find
  • logicalrep_worker_wakeup_ptr
  • WaitLatch / ResetLatch

関連データ構造

  • LogicalRepWorker: ワーカーの状態管理に使用される構造体。
  • LogicalRepWorkerLock: ワーカー間の同期用ロック。

失敗モードと境界条件

  • **適用ワーカーの消失**: 適用ワーカーが予期せず終了した場合、logicalrep_worker_find が失敗し、関数は false を返して待機を中断します。
  • **割り込み**: CHECK_FOR_INTERRUPTS() により、プロセスへのシグナル送信時に待機が中断される可能性があります。
  • **タイムアウト**: 1秒ごとにラッチを確認するため、適用ワーカーからの通知が失われた場合でも無限ループには陥りません。

保守担当者向けメモ

  • 本関数は MyLogicalRepWorker を介して共有メモリ上の状態を参照します。状態の更新がアトミックであることを前提としていますが、ロックなしでの読み取りが許容されている点に注意してください。
  • 待機イベント WAIT_EVENT_LOGICAL_SYNC_STATE_CHANGE が使用されており、待機中のプロセス状態は pg_stat_activity 等で追跡可能です。
