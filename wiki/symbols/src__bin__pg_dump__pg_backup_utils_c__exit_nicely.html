exit_nicely

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/pg_backup_utils.c:91

目的

プログラムの終了シーケンスを管理し、登録されたクリーンアップ処理を確実に実行した上でプロセスを終了させる。

シグネチャまたは形状

void exit_nicely(int code)

入力と出力

  • **入力**: code (int型) - プログラム終了時に返される終了ステータスコード。
  • **出力**: なし（プロセスを終了させるため戻り値はない）。

前提条件と不変条件

  • on_exit_nicely を通じてクリーンアップ関数が登録されている場合、それらは登録時とは逆順（インデックスの大きい順）に実行される。
  • Windows環境では、parallel_init_done が真かつ現在のスレッドがメインスレッドではない場合、プロセス全体ではなく当該スレッドのみを終了させる。

アルゴリズムまたは振る舞い

1. on_exit_nicely_list に格納された関数を、インデックスの降順でループ処理し、各関数に code と登録時の引数を渡して実行する。 2. Windows環境（WIN32）の場合、メインスレッド以外からの呼び出しであれば _endthreadex(code) を実行してスレッドを終了する。 3. 最後に exit(code) を呼び出し、プロセスを終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_dump/connectdb.c
  • src/bin/pg_dump/filter.c
  • **呼び出し先**:
  • on_exit_nicely_list に登録された各コールバック関数。
  • _endthreadex (Windows環境のみ)。
  • exit (標準ライブラリ)。

関連データ構造

  • on_exit_nicely_list: 終了時に実行すべき関数ポインタと引数を保持する配列。
  • on_exit_nicely_index: 現在登録されているコールバックの数を示すインデックス。
  • mainThreadId: Windows環境においてメインスレッドを識別するための変数。

失敗モードと境界条件

  • コールバックが未登録の場合、ループは実行されず直ちに終了処理へ移行する。
  • Windows環境において、メインスレッド以外から呼び出された場合はプロセス全体ではなくスレッドのみが終了する。

保守担当者向けメモ

  • コールバック関数は登録の逆順で実行されるため、リソース解放の依存関係に注意すること。
  • Unix環境での fork() 使用時は、子プロセスにもコールバックが引き継がれるため、一貫性を保つ設計が必要である。
