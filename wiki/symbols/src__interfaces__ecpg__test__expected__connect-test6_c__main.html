目的

ECPG（Embedded SQL for C）におけるデータベース接続の異常系テストを実行するエントリポイントです。接続切断後の操作や、存在しない接続名に対する操作がセグメンテーションフォールトを引き起こさず、適切なエラーコードを返すことを検証します。

シグネチャまたは形状

int main(void)

入力と出力

  • 入力: なし（引数なし）
  • 出力: 成功時に 0 を返します。標準出力および標準エラー出力へテストの進行状況と sqlca.sqlcode の値を出力します。

前提条件と不変条件

  • ECPGライブラリが正しく初期化されていること。
  • データベース ecpg1_regression が接続可能な状態であること。

アルゴリズムまたは振る舞い

1. **デバッグ有効化**: ECPGdebug(1, stderr) を呼び出し、デバッグ出力を有効にします。 2. **接続**: ECPGconnect を使用して ecpg1_regression データベースへ接続します。 3. **テスト1**: テーブル作成・挿入後、ECPGdisconnect で切断し、その状態で ECPGget_desc を実行してエラーハンドリングを確認します。 4. **テスト2**: 存在しない接続名に対して ECPGdeallocate_all を実行し、エラーコードを確認します。 5. **テスト3**: 切断済み接続に対して ECPGdeallocate_all を実行し、エラーコードを確認します。 6. **テスト4**: 存在しない接続名でプリペアドステートメントの定義とカーソル宣言を行い、エラーハンドリングを検証します。

呼び出し元と呼び出し先

  • 呼び出し元: OS（テスト実行プロセス）
  • 呼び出し先: ECPGdebug, ECPGconnect, ECPGdo, ECPGallocate_desc, ECPGprepare, ECPGset_desc, ECPGtrans, ECPGdisconnect, ECPGget_desc, ECPGdeallocate_all, ECPGprepared_statement（すべて ecpglib 由来）

関連データ構造

  • sqlca: SQL通信領域。実行結果のステータスコード（sqlca.sqlcode）の確認に使用。

失敗モードと境界条件

  • 接続が存在しない、または既に切断されている状態で操作を行った場合、プログラムはクラッシュせず、sqlca.sqlcode にエラー値が格納されることが期待されます。
