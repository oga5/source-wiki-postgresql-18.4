RunObjectPostCreateHookStr

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/objectaccess.c:157

目的

データベースオブジェクトの作成完了後に、名前ベースで識別されるオブジェクトに対するフック処理を実行します。主にセキュリティ拡張や監査ログプラグインが、オブジェクト作成イベントに介入するために使用されます。

シグネチャまたは形状

void RunObjectPostCreateHookStr(Oid classId, const char *objectName, int subId, bool is_internal)

入力と出力

  • **入力**:
  • classId (Oid): 対象オブジェクトのカタログOID。
  • objectName (const char *): オブジェクトの名前。
  • subId (int): サブオブジェクトID（該当しない場合は0）。
  • is_internal (bool): 内部的な作成処理かどうかを示すフラグ。
  • **出力**: なし（戻り値は void）。

前提条件と不変条件

  • グローバル変数 object_access_hook_str が NULL でないことが前提です。関数内で Assert(object_access_hook_str != NULL) が実行され、フックが未登録の場合はアボートします。

アルゴリズムまたは振る舞い

1. ObjectAccessPostCreate 構造体をスタック上に確保し、memset でゼロ初期化します。 2. 入力された is_internal フラグを構造体の is_internal フィールドに設定します。 3. 登録されているフック関数 object_access_hook_str を呼び出します。その際、イベントタイプとして OAT_POST_CREATE を指定し、オブジェクト情報および構造体へのポインタを渡します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の直接的な呼び出し元は提供されていません。
  • **呼び出し先**: グローバル関数ポインタ object_access_hook_str が指す外部プラグイン関数。

関連データ構造

  • ObjectAccessPostCreate: フック関数に渡される引数構造体。
  • OAT_POST_CREATE: フック関数に通知されるイベントタイプ定数。

失敗モードと境界条件

  • object_access_hook_str が NULL の状態で呼び出された場合、Assert によりプログラムが停止します。

保守担当者向けメモ

  • 本関数は名前ベースの識別が必要なケースで使用されます。OIDが確定している場合は RunObjectPostCreateHook の使用を検討してください。
  • フック関数はロードされた共有ライブラリ側に実装されるため、デバッグ時はプラグイン側の実装を確認してください。
