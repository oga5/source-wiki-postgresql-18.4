convert_link_to_directory

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup.c:2098

目的

バックアップ処理において、シンボリックリンクをディレクトリとして扱う必要がある場合に、渡された stat 構造体のモード情報を強制的にディレクトリ属性へ書き換える。

シグネチャまたは形状

static void convert_link_to_directory(const char *pathbuf, struct stat *statbuf)

入力と出力

  • **入力**:
  • pathbuf: 対象ファイルのパス（本関数内では未使用）。
  • statbuf: 対象ファイルの属性情報が格納された struct stat へのポインタ。
  • **出力**: なし（statbuf の st_mode フィールドを直接更新する）。

前提条件と不変条件

  • statbuf は有効なメモリ領域を指している必要がある。
  • pg_dir_create_mode がシステムで定義されている必要がある。

アルゴリズムまたは振る舞い

1. statbuf->st_mode がシンボリックリンク (S_ISLNK) であるかを確認する。 2. シンボリックリンクである場合、st_mode を S_IFDIR（ディレクトリ）と pg_dir_create_mode のビット論理和で上書きする。 3. シンボリックリンクでない場合は、何も変更しない。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c 内の以下の箇所から呼び出されている。
  • 1374行目
  • 1393行目
  • **呼び出し先**: システムマクロ S_ISLNK を使用。

関連データ構造

  • struct stat: POSIX で定義されるファイル属性構造体。

失敗モードと境界条件

  • statbuf が NULL の場合、セグメンテーション違反が発生する可能性がある。
  • シンボリックリンク以外のファイルに対しては、本関数は副作用を及ぼさない。

保守担当者向けメモ

  • 本関数は、バックアップ対象のファイルシステム走査中に、特定の条件下でシンボリックリンクをディレクトリとして再帰的に処理させるために使用される。
  • pathbuf 引数は現在の実装では使用されていないが、将来的なログ出力やデバッグ目的のために保持されている可能性がある。
