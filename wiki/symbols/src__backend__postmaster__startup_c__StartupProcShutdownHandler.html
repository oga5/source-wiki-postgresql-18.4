StartupProcShutdownHandler

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/startup.c:108

目的

起動プロセス（Startupプロセス）に対する SIGTERM シグナルを処理し、プロセスの終了要求を管理する。

シグネチャまたは形状

static void StartupProcShutdownHandler(SIGNAL_ARGS)

入力と出力

  • **入力**: SIGNAL_ARGS（シグナルハンドラの標準引数）。
  • **出力**: なし（副作用としてグローバル変数の更新とプロセスの終了、またはリカバリ処理の起床を行う）。

前提条件と不変条件

  • in_restore_command フラグが外部コマンド実行中かどうかを正しく反映していること。
  • WakeupRecovery() がリカバリ処理を中断または終了させるために呼び出されること。

アルゴリズムまたは振る舞い

1. in_restore_command が真の場合、外部コマンド実行中であると判断し、proc_exit(1) を呼び出して直ちにプロセスを終了する。 2. in_restore_command が偽の場合、shutdown_requested グローバル変数を true に設定し、リカバリ処理に終了要求を通知する。 3. 最後に WakeupRecovery() を呼び出し、待機中のリカバリ処理を起床させる。

呼び出し元と呼び出し先

  • **呼び出し元**: SIGTERM シグナル受信時にシステムによって呼び出される。
  • **呼び出し先**: proc_exit（終了時）、WakeupRecovery。

関連データ構造

  • in_restore_command（外部コマンド実行状態を示すフラグ）
  • shutdown_requested（終了要求フラグ）

失敗モードと境界条件

  • 外部コマンド実行中に SIGTERM を受信した場合、リカバリの整合性を待たずに proc_exit(1) で即時終了する。

保守担当者向けメモ

  • 本関数は pqsignal(SIGTERM, StartupProcShutdownHandler) によってシグナルハンドラとして登録される。
  • in_restore_command の状態管理が、プロセス終了の安全性を担保する重要なガードとなっている。
