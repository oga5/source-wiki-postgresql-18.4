allocateSection

定義

  • 種別: function
  • 定義箇所: src/backend/jit/llvm/SectionMemoryManager.cpp:165

目的

JITコンパイルされたコードやデータセクションに対し、実行可能なメモリ領域を確保または再利用する。

シグネチャまたは形状

uint8_t *allocateSection(
    SectionMemoryManager::AllocationPurpose Purpose, 
    uintptr_t Size, 
    unsigned Alignment
);

入力と出力

  • **入力**:
  • Purpose: 確保するメモリの用途（Code, ROData, RWData）。
  • Size: 要求するメモリサイズ（バイト単位）。
  • Alignment: 要求するアライメント（2のべき乗であること）。
  • **出力**: 確保されたメモリ領域の先頭アドレス（uint8_t*）。失敗時は nullptr を返す。

前提条件と不変条件

  • Alignment が0の場合は16に設定される。
  • Alignment は必ず2のべき乗である必要がある（assertによる検証）。
  • 確保されるメモリは、初期状態では常に読み書き可能（Read-Write）として扱われる。

アルゴリズムまたは振る舞い

1. 指定された Purpose に基づき、対応する MemoryGroup（CodeMem, RODataMem, RWDataMem）を選択する。 2. **再利用**: MemGroup.FreeMem リストを走査し、要求サイズを満たす既存の空きブロックがあれば、それをアライメント調整して割り当てる。 3. **新規確保**: 空きブロックがない場合、MMapper->allocateMappedMemory を使用してOSから新規メモリ領域を確保する。 4. **管理**: 確保したメモリの残余領域が16バイト以上ある場合、それを FreeMem リストに追加して再利用に備える。 5. 確保した領域を PendingMem に登録し、後続の finalizeMemory で権限変更が行えるように追跡する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • allocateCodeSection
  • allocateDataSection
  • src/backend/jit/llvm/SectionMemoryManager.cpp 内の各セクション割り当てメソッド。
  • **呼び出し先**:
  • MMapper->allocateMappedMemory（OSレベルのメモリ確保）

関連データ構造

  • SectionMemoryManager::AllocationPurpose: メモリ用途の列挙型。
  • MemoryGroup: 各用途ごとのメモリ管理構造体（FreeMem, PendingMem, AllocatedMem を保持）。
  • sys::MemoryBlock: LLVMのメモリブロック抽象化クラス。

失敗モードと境界条件

  • allocateMappedMemory がエラーを返した場合、nullptr を返す。
  • アライメントが2のべき乗でない場合、アサーションエラーが発生する。
  • 確保したメモリの残余が16バイト未満の場合、空きブロックとして登録されず断片化の原因となる可能性がある。

保守担当者向けメモ

  • 本メソッドはLLVMの RuntimeDyld から呼び出される。
