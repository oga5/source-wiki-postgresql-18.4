CheckpointWriteDelay

定義

  • 種別: function
  • 定義箇所: src/backend/postmaster/checkpointer.c:771

目的

チェックポイント実行中の書き込み速度を制御（スロットリング）し、システム負荷を平滑化する。

シグネチャまたは形状

void CheckpointWriteDelay(int flags, double progress)

入力と出力

  • **入力**:
  • flags: チェックポイントの動作フラグ（CHECKPOINT_IMMEDIATE 等）。
  • progress: 現在の進捗状況（0.0 から 1.0 の範囲）。
  • **出力**: なし（void）。

前提条件と不変条件

  • 呼び出し元が checkpointer プロセスでない場合、本関数は即座に復帰する。
  • flags に CHECKPOINT_IMMEDIATE が含まれる場合、またはシャットダウン要求がある場合はスリープを行わない。

アルゴリズムまたは振る舞い

1. **プロセス確認**: AmCheckpointerProcess() を確認し、チェックポインター以外であれば処理を終了する。 2. **スロットリング判定**: IsCheckpointOnSchedule(progress) を用いて、チェックポイントが予定通り進行しているかを確認する。 3. **待機処理**: 予定通り進行しており、かつ即時実行やシャットダウンの要求がない場合：

  • 設定変更（ConfigReloadPending）があれば適用する。
  • AbsorbSyncRequests() を呼び出し、保留中の fsync 要求を処理する。
  • CheckArchiveTimeout() を実行し、統計情報を報告する。
  • WaitLatch を使用して 100ms の待機を行う。

    • 4. **強制吸収**: 待機を行わない場合でも、WRITES_PER_ABSORB 回の書き込みごとに AbsorbSyncRequests() を呼び出し、fsync キューの溢れを防止する。 5. **バリア処理**: ProcSignalBarrierPending が真であれば ProcessProcSignalBarrier() を実行する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/buffer/bufmgr.c (バッファ同期処理中)
  • **呼び出し先**:
  • AmCheckpointerProcess()
  • ImmediateCheckpointRequested()
  • IsCheckpointOnSchedule()
  • ProcessConfigFile()
  • UpdateSharedMemoryConfig()
  • AbsorbSyncRequests()
  • CheckArchiveTimeout()
  • pgstat_report_checkpointer()
  • WaitLatch() / ResetLatch()
  • ProcessProcSignalBarrier()

関連データ構造

  • MyLatch: プロセスの待機制御に使用。
  • ConfigReloadPending: 設定再読み込みフラグ。
  • ShutdownXLOGPending / ShutdownRequestPending: シャットダウン状態フラグ。
  • ProcSignalBarrierPending: プロセスシグナルバリアフラグ。

失敗モードと境界条件

  • **キューの溢れ**: AbsorbSyncRequests を定期的に呼び出すことで、fsync 要求キューのオーバーフローを回避する設計となっている。
  • **即時実行**: CHECKPOINT_IMMEDIATE フラグが立っている場合、スリープをスキップして処理を優先する。

保守担当者向けメモ

  • 本関数は bgwriter_delay とは独立しており、現在は 100ms の固定待機を行う。
  • absorb_counter は静的変数であり、書き込み回数に応じた fsync 要求の吸い上げ頻度を管理している。
