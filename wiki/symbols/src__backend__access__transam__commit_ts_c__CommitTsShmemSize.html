CommitTsShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/commit_ts.c:518

目的

コミットタイムスタンプ機能が共有メモリ上で必要とする合計サイズを計算します。

シグネチャまたは形状

Size CommitTsShmemSize(void)

入力と出力

  • **入力**: なし
  • **出力**: Size 型のバイト数。SLRUキャッシュに必要な領域と、制御構造体 CommitTimestampShared の合計値を返します。

前提条件と不変条件

  • 本関数は、共有メモリの初期化プロセスにおいて、システム全体の共有メモリサイズを算出する段階で呼び出される必要があります。

アルゴリズムまたは振る舞い

1. CommitTsShmemBuffers() を呼び出し、設定に基づいたSLRUバッファ数を取得します。 2. SimpleLruShmemSize() を使用して、SLRU管理に必要な共有メモリサイズを算出します。 3. 手順2の結果に sizeof(CommitTimestampShared) を加算して返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/storage/ipc/ipci.c:125（システム全体の共有メモリサイズ算出時）
  • **呼び出し先**:
  • CommitTsShmemBuffers()
  • SimpleLruShmemSize()

関連データ構造

  • CommitTimestampShared: コミットタイムスタンプ機能の制御用共有メモリ構造体。
  • SlruCtlData (間接的): SimpleLruShmemSize を通じて利用されるSLRU管理構造体。

失敗モードと境界条件

  • CommitTsShmemBuffers() が返すバッファ数が0の場合、SimpleLruShmemSize の仕様に従った最小限のサイズが計算されます。

保守担当者向けメモ

  • 本関数は src/include/access/commit_ts.h で公開されています。
  • 共有メモリのレイアウトや CommitTimestampShared 構造体のサイズを変更した場合は、必ず本関数の計算式が整合しているか確認してください。
