AlterTypeRecurseParams

定義

  • 種別: struct
  • 定義箇所: src/backend/commands/typecmds.c:88

目的

ALTER TYPE コマンドにおいて、ベース型の変更を依存するドメインや配列型へ再帰的に伝播させる際に、更新対象の属性と新しい値を保持するためのパラメータ構造体です。

シグネチャまたは形状

typedef struct {
    /* 更新対象を示すフラグ */
    bool updateStorage;
    bool updateReceive;
    bool updateSend;
    bool updateTypmodin;
    bool updateTypmodout;
    bool updateAnalyze;
    bool updateSubscript;
    /* 各属性の新しい値 */
    char storage;
    Oid  receiveOid;
    Oid  sendOid;
    Oid  typmodinOid;
    Oid  typmodoutOid;
    Oid  analyzeOid;
    Oid  subscriptOid;
} AlterTypeRecurseParams;

入力と出力

本構造体は、AlterTypeRecurse 関数等へ渡される入力パラメータとして機能します。出力（戻り値）はありません。

前提条件と不変条件

  • updateXxx フラグが true の場合のみ、対応する XxxOid フィールド（または storage）が有効な値として参照されます。
  • 構造体内の各フィールドは、pg_type カタログの属性に対応しています。

アルゴリズムまたは振る舞い

ALTER TYPE 実行時に、変更対象の属性をフラグで指定し、新しい値を本構造体に格納して再帰処理関数に渡します。これにより、ベース型の変更がドメイン等の依存オブジェクトへ一貫して適用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/typecmds.c 内の AlterType 関連処理（例: 4327行目, 4656行目）。
  • **呼び出し先**: 本構造体自体はデータ保持用であり、関数呼び出しの引数として AlterTypeRecurse 等へ渡されます。
  • ※詳細な呼び出し関係のグラフは提供されていません。

関連データ構造

  • pg_type: 本構造体が更新を試みる対象となるシステムカタログ。

失敗モードと境界条件

  • 特定の属性（例: storage）はドメインに継承されない場合があり、フラグによる制御が必須です。
  • 権限不足や依存関係の制約により、カタログ更新時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は AlterTypeRecurse のインターフェースと密結合しています。属性を追加・変更する際は、フラグと値の両方を更新してください。
  • 継承されない属性の取り扱いについては、typecmds.c 内の再帰ロジックを参照してください。
