smgrshutdown

定義

  • 種別: function
  • 定義箇所: src/backend/storage/smgr/smgr.c:209

目的

ストレージマネージャサブシステムの終了処理を実行し、登録されているすべてのストレージ実装に対してシャットダウン処理を委譲する。

シグネチャまたは形状

static void smgrshutdown(int code, Datum arg)

入力と出力

  • **入力**:
  • code: プロセス終了コード（on_proc_exit のコールバック規約に従う）。
  • arg: 補助データ（現在は使用されていない）。
  • **出力**: なし（void）。

前提条件と不変条件

  • 本関数は on_proc_exit を介してプロセス終了時に呼び出されることを想定している。
  • 実行中は HOLD_INTERRUPTS() により割り込みが禁止され、ストレージ操作の整合性が保護される。

アルゴリズムまたは振る舞い

1. HOLD_INTERRUPTS() を呼び出し、処理中の割り込みを抑止する。 2. 登録されているストレージマネージャの数（NSmgr）分だけループする。 3. 各ストレージマネージャのスイッチ構造体（smgrsw）を確認し、smgr_shutdown 関数ポインタが定義されていればそれを実行する。 4. RESUME_INTERRUPTS() を呼び出し、割り込み禁止状態を解除する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/storage/smgr/smgr.c 内の初期化処理にて on_proc_exit(smgrshutdown, 0) として登録されている。
  • **呼び出し先**:
  • smgrsw 配列に格納された各ストレージ実装の smgr_shutdown 関数。

関連データ構造

  • smgrsw: ストレージマネージャの関数ポインタを保持するスイッチ配列。
  • NSmgr: 登録されているストレージマネージャの総数。

失敗モードと境界条件

  • smgrsw[i].smgr_shutdown が NULL の場合、その実装に対するシャットダウン処理はスキップされる。
  • 割り込み禁止期間中に発生したエラーは、RESUME_INTERRUPTS() が呼ばれるまで保留される。

保守担当者向けメモ

  • 本関数はバックエンド終了時のクリーンアップを担当する。新しいストレージ実装を追加した場合は、smgrsw に適切な smgr_shutdown 関数を登録する必要がある。
