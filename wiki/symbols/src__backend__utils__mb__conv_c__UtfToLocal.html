UtfToLocal

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conv.c:506

目的

UTF-8でエンコードされた文字列を、指定されたローカルエンコーディングへ変換します。

シグネチャまたは形状

int UtfToLocal(const unsigned char *utf, int len,
               unsigned char *iso,
               const pg_mb_radix_tree *map,
               const pg_utf_to_local_combined *cmap, int cmapsize,
               utf_local_conversion_func conv_func,
               int encoding, bool noError)

入力と出力

  • **入力**:
  • utf: 変換元UTF-8文字列のポインタ。
  • len: 入力文字列のバイト長。
  • map: ラディックスツリー形式の変換テーブル。
  • cmap: 結合文字用マップ。
  • conv_func: アルゴリズムによる変換関数。
  • encoding: ターゲットエンコーディングID。
  • noError: 変換失敗時にエラーを報告せず中断するかを指定するフラグ。
  • **出力**: 変換後のバイト数（戻り値）。iso バッファに変換結果が格納されます。

前提条件と不変条件

  • encoding は PG_VALID_ENCODING マクロにより有効性が検証されます。
  • ASCII文字（1バイト）は1対1の変換とみなされます。
  • 結合文字マップ（cmap）は、ASCII文字を含まないことを前提としています。

アルゴリズムまたは振る舞い

1. 入力文字列をバイト単位で走査し、pg_utf_mblen で文字境界を特定します。 2. **結合文字の処理**: cmap が提供されている場合、連続する文字を組み合わせて検索し、一致すれば変換結果を格納します。 3. **ラディックスツリー検索**: pg_mb_radix_conv を使用して変換を試みます。 4. **関数変換**: 上記で変換できない場合、conv_func があればそれを適用します。 5. **エラー処理**: 変換不可または不正なバイト列が検出された際、noError が false であればエラーを報告し、true であれば処理を中断して戻ります。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/utils/mb/conversion_procs/ 配下の各エンコーディング変換プロシージャ（utf8_and_big5.c, utf8_and_cyrillic.c 等）。
  • **呼び出し先**: pg_utf_mblen, pg_utf8_islegal, bsearch, store_coded_char, pg_mb_radix_conv, report_invalid_encoding, report_untranslatable_char。

関連データ構造

  • pg_mb_radix_tree: ラディックスツリーによる変換マップ。
  • pg_utf_to_local_combined: 結合文字変換用構造体。

失敗モードと境界条件

  • 入力文字列が不正なUTF-8シーケンスである場合、または変換テーブルに存在しない文字が含まれる場合、noError フラグに応じてエラー報告または中断が発生します。
  • 入力バッファの残りが文字長より短い場合、処理を中断します。

保守担当者向けメモ

  • 本関数はパフォーマンスに非常に敏感です。新しい変換テーブルを追加する際は pg_mb_radix_tree の構造を遵守してください。
  • noError フラグ使用時は、戻り値（消費されたバイト数）が正しく計算されているか確認が必要です。
  • 結合文字の処理は通常の文字変換より優先されるため、変換順序の変更には注意してください。
