spg_quad_leaf_consistent

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgquadtreeproc.c:406

目的

SP-GiST四分木インデックスにおいて、リーフノードに格納されたデータが検索条件（スキャンキー）を満たすかどうかを判定し、必要に応じて距離計算を行う。

シグネチャまたは形状

Datum spg_quad_leaf_consistent(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して spgLeafConsistentIn（リーフデータ、スキャンキー、順序付けキー等）を受け取る。
  • **出力**: PG_RETURN_BOOL(res) により、条件一致の真偽値を返す。また、spgLeafConsistentOut 構造体を介して、recheck フラグ（常に false）、leafValue、および distances（KNN検索時）を更新する。

前提条件と不変条件

  • out->recheck は常に false に設定される（すべての比較が厳密であるため）。
  • RTContainedByStrategyNumber の場合、クエリ引数は Point ではなく BOX として扱われる。

アルゴリズムまたは振る舞い

1. リーフノードのデータ（leafDatum）を取得する。 2. すべてのスキャンキーを順次評価する。

  • RTLeft, RTRight, RTSame, RTBelow, RTAbove などの戦略番号に基づき、point_left, point_right, point_eq, point_below, point_above 等の関数を用いて比較を行う。
  • RTContainedByStrategyNumber の場合は box_contain_pt を使用する。

    • 3. いずれかの条件で不一致となった場合、直ちに判定を終了する。 4. すべての条件を満たし、かつ norderbys > 0 の場合は、spg_key_orderbys_distances を呼び出して距離計算結果を格納する。

呼び出し元と呼び出し先

  • **呼び出し元**: SP-GiSTの検索エンジン（直接的な呼び出し元はソースコード上では明示されていないが、SP-GiSTのインデックススキャンプロセスから呼び出される）。
  • **呼び出し先**: spg_key_orderbys_distances（KNN検索時）、および point_left, point_right, point_eq, point_below, point_above, box_contain_pt（比較演算用）。
  • **備考**: spg_kd_leaf_consistent は本関数と同一のロジックを使用しているとの記述がある。

関連データ構造

  • spgLeafConsistentIn: 検索入力データ。
  • spgLeafConsistentOut: 検索結果出力データ。
  • Point, BOX: 幾何学データ型。

失敗モードと境界条件

  • 未知の戦略番号（sk_strategy）が渡された場合、elog(ERROR) によりエラーを発生させる。

保守担当者向けメモ

  • 本関数は四分木専用のリーフ判定ロジックである。
  • RTContainedByStrategyNumber 処理において、DatumGetPointP を BOX ポインタに対して使用する「型キャストの回避」が行われている点に注意すること。
