XLogArchiveIsBusy

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogarchive.c:618

目的

指定されたWALファイルがアーカイブプロセスによって現在処理中（ビジー状態）であるかを判定する。

シグネチャまたは形状

bool XLogArchiveIsBusy(const char *xlog)

入力と出力

  • **入力**: xlog (WALファイル名)
  • **出力**: bool
  • true: ファイルがアーカイブ待ち（.readyが存在）であるか、状態が不明なためビジーとみなされる場合。
  • false: ファイルがアーカイブ済み（.doneが存在）であるか、チェックポイント等によりファイル自体が削除されている場合。

前提条件と不変条件

  • pg_wal/archive_status ディレクトリ内に .ready または .done ファイルが存在することを前提に状態を判定する。
  • ファイルシステム上の状態変化（レースコンディション）を考慮した再確認ロジックを含む。

アルゴリズムまたは振る舞い

1. **.done の確認**: 指定されたWALファイルに対応する .done ファイルの存在を確認し、存在すれば false を返す。 2. **.ready の確認**: .ready ファイルの存在を確認し、存在すれば true を返す。 3. **再確認**: アーカイブプロセスが直後に終了した可能性を考慮し、再度 .done ファイルの存在を確認する。 4. **ファイル削除の確認**: pg_wal ディレクトリから対象ファイル自体が消失しているかを確認する。消失している場合（ENOENT）、アーカイブ済みとみなして false を返す。 5. **デフォルト**: 上記のいずれにも該当しない場合、安全側に倒して true を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/transam/xlog.c 内の lastxlogfilename および histfilename の処理において呼び出されている。
  • **呼び出し先**: stat() システムコールおよび StatusFilePath() マクロ（パス構築用）。

関連データ構造

  • struct stat: ファイル状態確認用。
  • MAXPGPATH: パス文字列の最大長定義。

失敗モードと境界条件

  • stat() が ENOENT 以外のエラーを返した場合の挙動は、呼び出し元の環境に依存する。
  • アーカイブプロセスと並行してファイルが削除された場合、stat() の結果に基づき false を返すことで競合を回避する。

保守担当者向けメモ

  • 本関数は XLogArchiveIsReadyOrDone や XLogArchiveIsReady と関連しており、アーカイブ状態の判定ロジックを共有している。
  • XLOGDIR マクロを使用してWALディレクトリを参照しているため、ディレクトリ構成の変更時には注意が必要。
