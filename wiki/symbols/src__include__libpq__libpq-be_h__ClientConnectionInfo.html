ClientConnectionInfo

定義

  • 種別: struct
  • 定義箇所: src/include/libpq/libpq-be.h:86

目的

並列ワーカープロセスへ引き継ぐために、認証済みのクライアントアイデンティティ情報を保持する構造体です。

シグネチャまたは形状

typedef struct ClientConnectionInfo
{
    const char *authn_id;
    UserAuth    auth_method;
} ClientConnectionInfo;

入力と出力

  • **入力**: 認証プロセス中に決定される認証済み識別子（authn_id）および認証方式（auth_method）。
  • **出力**: 並列クエリ実行時などに、ワーカープロセスへ認証状態を伝播させるためのデータセット。

前提条件と不変条件

  • authn_id は、認証サイクル中にユーザーが提示した識別子を指します。
  • auth_method は authn_id が NULL でない場合にのみ有効な意味を持ちます。
  • 「trust」認証方式のように認証が実際に行われていない場合、authn_id は NULL となります。

アルゴリズムまたは振る舞い

本構造体は、バックエンドプロセスから並列ワーカープロセスへの認証情報のシリアライズおよび復元に使用されます。認証済みアイデンティティを保持することで、データベースロールへの割り当て後も、元の認証時の識別子（pg_ident マップ等で使用されるもの）を追跡可能にします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c: 並列ワーカー生成時のシリアライズおよび復元処理。
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、直接的な関数呼び出し先は定義されていません。

関連データ構造

  • Port: クライアント接続の全情報を保持する構造体。ClientConnectionInfo はこの情報を補完する役割を持ちます。
  • UserAuth: auth_method に使用される列挙型。
  • MyClientConnectionInfo: src/backend/utils/init/miscinit.c で定義されるグローバル変数。

失敗モードと境界条件

  • 認証が完了していない場合、authn_id が NULL になるため、後続の処理で auth_method を参照する際は注意が必要です。

保守担当者向けメモ

  • 本構造体は並列処理のコンテキストで重要です。変更を加える際は、parallel.c におけるシリアライズ/デシリアライズ処理との整合性を確認してください。
  • authn_id は pg_ident マップにおける「SYSTEM-USERNAME」に相当しますが、pg_hba の設定により実際の文字列は変動する可能性がある点に留意してください。
