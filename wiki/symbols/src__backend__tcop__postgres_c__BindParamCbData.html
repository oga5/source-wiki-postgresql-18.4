BindParamCbData

定義

  • 種別: struct
  • 定義箇所: src/backend/tcop/postgres.c:114

目的

拡張クエリプロトコルにおけるバインドパラメータ処理中に、ポータル名、パラメータ番号、およびパラメータ値を保持するためのコンテキストデータ構造です。

シグネチャまたは形状

typedef struct BindParamCbData
{
    const char *portalName;
    int         paramno;        /* 0から始まるパラメータ番号、初期値は -1 */
    const char *paramval;       /* テキスト形式の入力文字列（利用可能な場合） */
} BindParamCbData;

入力と出力

  • **入力**: 構造体の各フィールドは、バインド処理の文脈に応じて初期化されます。
  • **出力**: 構造体自体は、バインド処理のコールバック関数内でパラメータ情報を参照するために使用されます。

前提条件と不変条件

  • paramno は初期状態では -1 に設定されることが想定されています。
  • paramval はテキスト形式の入力文字列を指すポインタであり、利用可能でない場合は NULL になる可能性があります。

アルゴリズムまたは振る舞い

本構造体は、クエリ実行のバインドフェーズにおいて、特定のパラメータ情報をコールバック関数へ受け渡すためのコンテナとして機能します。処理の過程で、paramno をインクリメントしたり、paramval を参照して値を解析するロジックで使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/postgres.c 内の関数（例: 1798 行目付近でインスタンス化）。
  • **呼び出し先**: src/backend/tcop/postgres.c 内のコールバック関数（例: 2595 行目付近で arg としてキャストされ参照）。
  • **補足**: 外部モジュールからの呼び出しに関する直接的な証拠はありません。

関連データ構造

  • Portal: ポータル名（portalName）に関連する実行単位。

失敗モードと境界条件

  • paramval が NULL の場合に、文字列として参照しようとするとセグメンテーション違反が発生する可能性があります。
  • paramno が負の値のまま処理が進んだ場合の挙動は、呼び出し側のロジックに依存します。

保守担当者向けメモ

  • 本構造体は postgres.c 内の局所的なバインド処理に特化しています。変更を加える際は、exec_bind_message 周辺のパラメータ処理ロジックへの影響を確認してください。
