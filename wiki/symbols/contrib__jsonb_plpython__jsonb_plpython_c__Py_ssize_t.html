Py_ssize_t

定義

  • 種別: typedef
  • 定義箇所: contrib/jsonb_plpython/jsonb_plpython.c:32

目的

Python C API におけるサイズやインデックスを表現するための符号付き整数型です。

シグネチャまたは形状

typedef Py_ssize_t;

※Python の標準ヘッダーで定義される型であり、本ファイルでは主に外部関数ポインタのシグネチャ内で使用されます。

入力と出力

  • **入力**: データのサイズやインデックスを表す数値。
  • **出力**: 該当するメモリ領域のサイズや、シーケンス内の位置を示す符号付き整数。

前提条件と不変条件

  • Python C API の規約に従い、負の値はエラーや特殊な状態を示すために使用される場合があります。
  • 64ビット環境では通常 ssize_t と同等のサイズを持ちます。

アルゴリズムまたは振る舞い

  • 文字列の長さ指定や、リスト・辞書などのコンテナ要素数のカウントに使用されます。
  • PLyUnicode_FromStringAndSize_t のような関数ポインタ定義において、文字列の長さを渡す引数として機能します。

呼び出し元と呼び出し先

  • **呼び出し元**: contrib/jsonb_plpython/jsonb_plpython.c 内の各変換関数（pcount や i 変数として使用）。
  • **呼び出し先**: Python C API の各種関数（PyList_New, PyList_SetItem 等、src/pl/plpython/ 配下の各ファイルで参照）。

関連データ構造

  • PyObject: Python オブジェクトの基底構造体。
  • PLyUnicode_FromStringAndSize_t: Py_ssize_t を引数に取る関数ポインタ型。

失敗モードと境界条件

  • 負の値を渡した場合の挙動は、呼び出し先の Python C API 関数に依存します。
  • 非常に大きな値を指定した場合、メモリ割り当ての失敗やオーバーフローのリスクがあります。

保守担当者向けメモ

  • 本シンボルは Python の標準的な型定義です。PostgreSQL 側の型（int32 や int64）と混同しないよう注意してください。
  • contrib/jsonb_plpython/jsonb_plpython.c 内では、主に plpython 拡張から動的にロードされる関数のシグネチャ定義の一部として利用されています。
