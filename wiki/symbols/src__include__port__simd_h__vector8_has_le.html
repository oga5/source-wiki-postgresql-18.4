vector8_has_le

定義

  • 種別: static inline function
  • 定義箇所: src/include/port/simd.h:212

目的

Vector8 型のベクトル内に、指定された値 c 以下の要素が含まれているかを判定します。

シグネチャまたは形状

bool vector8_has_le(const Vector8 v, const uint8 c)

入力と出力

  • **入力**:
  • v: 検索対象の Vector8 型データ。
  • c: 比較基準となる uint8 型の値。
  • **出力**:
  • ベクトル内のいずれかの要素が c 以下であれば true、そうでなければ false を返します。

前提条件と不変条件

  • USE_ASSERT_CHECKING が有効な場合、SIMD 演算の結果と、各要素を逐次比較した結果が一致することを Assert で検証します。

アルゴリズムまたは振る舞い

コンパイル時の設定に応じて以下のいずれかの手法をとります。

1. **SIMD 利用時 (!defined(USE_NO_SIMD))**:

  • vector8_ssub（飽和減算）を用いて v から c を減算します。
  • 結果として得られたベクトルに対し vector8_has_zero を実行し、アンダーフロー（＝元の値が c 以下）が発生した箇所を検出します。

    • 2. **SIMD 非利用時 (USE_NO_SIMD)**:

  • 特定の条件（v の最上位ビットが未設定かつ c < 0x80）を満たす場合、ビット演算による最適化手法（Stanford Bit Hacks）を適用します。
  • それ以外の場合は、ベクトル内の各要素をループで逐次比較します。

関連データ構造

失敗モードと境界条件

  • USE_NO_SIMD 環境下でビット演算による最適化を行う際、c >= 0x80 または v の最上位ビットがセットされている場合は、安全のため逐次比較へフォールバックします。

保守担当者向けメモ

  • 本関数は simd.h 内の他のベクトル演算関数（vector8_has_zero 等）に依存しています。修正時はこれらの関数との整合性に注意してください。
  • USE_ASSERT_CHECKING を有効にすると、SIMD 演算の正当性を逐次処理で検証するチェックが挿入されます。開発時には有用ですが、本番環境では無効にしてください。
