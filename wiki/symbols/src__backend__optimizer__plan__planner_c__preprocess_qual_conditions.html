preprocess_qual_conditions

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/plan/planner.c:1409

目的

クエリのジョインツリー（jointree）を再帰的に走査し、各ノードに含まれる条件式（quals）に対して最適化のための前処理（preprocess_expression）を適用する。

シグネチャまたは形状

static void preprocess_qual_conditions(PlannerInfo *root, Node *jtnode)

入力と出力

  • **入力**:
  • root: プランニングの全体状態を保持する PlannerInfo 構造体。
  • jtnode: 走査対象のジョインツリーノード（RangeTblRef, FromExpr, または JoinExpr）。
  • **出力**: なし（副作用として jtnode 内の quals フィールドを更新する）。

前提条件と不変条件

  • jtnode が FromExpr または JoinExpr の場合、その quals フィールドは preprocess_expression によって変換された結果で上書きされる。
  • RangeTblRef の場合は何も行われない。

アルゴリズムまたは振る舞い

1. jtnode が NULL の場合は即座に終了する。 2. jtnode の型に応じて以下の処理を行う：

  • **RangeTblRef**: 何もしない。
  • **FromExpr**: fromlist 内の各要素に対して再帰的に本関数を呼び出し、その後 f->quals に対して preprocess_expression(root, f->quals, EXPRKIND_QUAL) を実行する。
  • **JoinExpr**: 左辺（larg）と右辺（rarg）に対して再帰的に本関数を呼び出し、その後 j->quals に対して preprocess_expression(root, j->quals, EXPRKIND_QUAL) を実行する。

    • 3. 上記以外のノード型が渡された場合は elog(ERROR) を発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/plan/planner.c 内の subquery_planner（parse->jointree を引数として呼び出し）。
  • **呼び出し先**:
  • preprocess_qual_conditions（再帰呼び出し）
  • preprocess_expression（条件式の変換）

関連データ構造

  • PlannerInfo: プランニングのコンテキスト情報。
  • FromExpr, JoinExpr, RangeTblRef: ジョインツリーを構成するノード型。

失敗モードと境界条件

  • 未知のノード型が渡された場合、elog(ERROR) により処理が中断される。
  • jtnode が NULL の場合は安全に無視される。

保守担当者向けメモ

  • 本関数はジョインツリーの構造を辿るための再帰的なユーティリティである。
  • preprocess_expression に渡される EXPRKIND_QUAL は、条件式としての正規化ルールを適用するための定数である。
