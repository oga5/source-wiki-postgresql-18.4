test_radixtree

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_radixtree/test_radixtree.c:433

目的

PostgreSQL の適応型基数木（Adaptive Radix Tree）実装の正当性を検証するためのメインテスト関数です。

シグネチャまたは形状

Datum test_radixtree(PG_FUNCTION_ARGS)

入力と出力

  • 入力: PG_FUNCTION_ARGS（SQL インターフェース経由の引数）
  • 出力: PG_RETURN_VOID()（戻り値なし）

前提条件と不変条件

  • 本関数は PostgreSQL の関数として登録されており、SQL から呼び出されることを前提としています。
  • テスト実行中に EXPECT_* マクロが失敗した場合、elog(ERROR) によりトランザクションがアボートされます。

アルゴリズムまたは振る舞い

1. test_empty() を呼び出し、空の木に対する操作を検証します。 2. rt_node_class_tests 配列を反復処理し、各ノードクラスに対して以下の条件で test_basic() を実行します。

  • 深さ: 0（単一ノード）、8（2レベル）、max_shift（最大レベル）
  • 順序: 昇順および降順の双方

    • 3. test_random() を呼び出し、擬似乱数を用いた大規模な負荷テストを実行します。 4. 最後に PG_RETURN_VOID() で終了します。

呼び出し元と呼び出し先

  • 呼び出し元: SQL インターフェース（PG_FUNCTION_INFO_V1(test_radixtree) により登録）。
  • 呼び出し先: test_empty(), test_basic(), test_random()。

関連データ構造

  • rt_node_class_tests: テスト対象のノードクラスとキー数を定義した配列。
  • rt_node_class_test_elem: 各テストケースのパラメータを保持する構造体。

失敗モードと境界条件

  • テスト対象の基数木実装に不整合がある場合、内部で呼び出される EXPECT_* マクロがエラーを発生させ、処理が中断されます。
  • max_shift は uint64 のビット数に基づいて計算され、基数木の深さの境界を定義します。

保守担当者向けメモ

  • 本関数はテストのエントリポイントです。新しいテストケースを追加する場合は、test_basic の呼び出しパターンを増やすか、新しいテスト関数を作成して本関数から呼び出してください。
  • TEST_SHARED_RT マクロの状態によって、共有メモリ上での動作検証が行われるかどうかが変わる点に注意してください。
