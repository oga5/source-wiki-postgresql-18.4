buffer_call_terminate_io

定義

  • 種別: function
  • 定義箇所: src/test/modules/test_aio/test_aio.c:560

目的

PostgreSQL のバッファ I/O 終了処理（TerminateBufferIO または TerminateLocalBufferIO）を SQL インターフェース経由で強制的に実行し、テスト目的でバッファの状態を操作する。

シグネチャまたは形状

Datum buffer_call_terminate_io(PG_FUNCTION_ARGS)

入力と出力

  • **入力 (PG引数)**:
  • buf (int32): 対象のバッファ ID。
  • for_input (bool): 入力（読み込み）操作の終了か否か。
  • succeed (bool): I/O が成功したとみなすか否か。
  • io_error (bool): BM_IO_ERROR フラグを立てるか否か。
  • release_aio (bool): AIO ハンドルを解放するか否か。
  • **出力**: void (戻り値なし)

前提条件と不変条件

  • 指定された buf は有効なバッファである必要がある。
  • for_input が真の場合、succeed が真であれば BM_VALID フラグが付与される。
  • for_input が偽（出力）の場合、succeed が真であれば clear_dirty が真となり、ダーティフラグがクリアされる。

アルゴリズムまたは振る舞い

1. 引数に基づき、BM_IO_ERROR や BM_VALID などのフラグビットを計算する。 2. 処理前のバッファ参照カウントをログ出力する。 3. BufferIsLocal(buf) の判定に基づき、ローカルバッファであれば TerminateLocalBufferIO を、共有バッファであれば TerminateBufferIO を呼び出す。 4. 処理後のバッファ参照カウントをログ出力する。 5. PG_RETURN_VOID() で終了する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL インターフェース（PG_FUNCTION_INFO_V1 により登録）。
  • **呼び出し先**:
  • TerminateLocalBufferIO
  • TerminateBufferIO
  • DebugPrintBufferRefcount (ログ出力用)

関連データ構造

  • BufferDescriptor (共有バッファ用)
  • LocalBufferDescriptor (ローカルバッファ用)

失敗モードと境界条件

  • buf が不正な値の場合、GetBufferDescriptor 等の内部関数でアサーション失敗やセグメンテーション違反が発生する可能性がある。
  • 共有バッファとローカルバッファの判定ロジックに依存するため、不適切なバッファ ID を渡すと予期せぬ動作を引き起こす。

保守担当者向けメモ

  • 本関数はテスト専用であり、本番環境のコードパスには含まれない。
  • ログ出力には errhidestmt(true) および errhidecontext(true) が設定されており、テスト実行時のノイズを抑える設計となっている。
