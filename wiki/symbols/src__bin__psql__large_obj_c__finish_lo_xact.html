finish_lo_xact

定義

  • 種別: function
  • 定義箇所: src/bin/psql/large_obj.c:97

目的

ラージオブジェクト操作に関連するトランザクションを終了（コミット）します。

シグネチャまたは形状

static bool finish_lo_xact(const char *operation, bool own_transaction)

入力と出力

  • **入力**:
  • operation: 操作名（ログ出力用に使用されるが、本関数内では未使用）。
  • own_transaction: 本関数がトランザクションを管理すべきかを示すフラグ。
  • **出力**:
  • bool: トランザクションの終了（コミット）が成功した場合は true、失敗した場合は false を返します。

前提条件と不変条件

  • pset.autocommit が有効であり、かつ own_transaction が true の場合にのみ、明示的な COMMIT が発行されます。

アルゴリズムまたは振る舞い

1. own_transaction が true かつ pset.autocommit が有効な場合、PSQLexec("COMMIT") を実行します。 2. COMMIT が失敗した場合、PSQLexec("ROLLBACK") を試行し、結果を破棄した上で false を返します。 3. COMMIT が成功した場合は PQclear で結果を解放し、true を返します。 4. 上記条件に該当しない場合は、何もせず true を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • do_lo_export (src/bin/psql/large_obj.c:161)
  • do_lo_import (src/bin/psql/large_obj.c:221)
  • do_lo_unlink (src/bin/psql/large_obj.c:258)
  • **呼び出し先**:
  • PSQLexec
  • PQclear

関連データ構造

  • pset: settings.h で定義されるグローバル設定構造体。autocommit 状態の判定に使用。
  • PGresult: libpq のクエリ結果構造体。

失敗モードと境界条件

  • COMMIT コマンドが失敗した場合、ROLLBACK を実行してエラー状態を返します。
  • operation 引数は関数内で使用されておらず、将来的な拡張やデバッグログ用として保持されています。

保守担当者向けメモ

  • 本関数は start_lo_xact と対になる処理です。トランザクションの整合性を保つため、必ず start_lo_xact が成功した後に呼び出されることを前提としています。
