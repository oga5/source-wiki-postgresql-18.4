BackendStatusShmemSize

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/backend_status.c:80

目的

バックエンドの活動状況を保持するために必要な共有メモリ領域の総サイズを計算する。

シグネチャまたは形状

Size BackendStatusShmemSize(void)

入力と出力

  • 入力: なし
  • 出力: 必要な共有メモリサイズ（Size 型）

前提条件と不変条件

  • NumBackendStatSlots が正しく設定されていること。
  • pgstat_track_activity_query_size が有効な値であること。

アルゴリズムまたは振る舞い

以下の各領域の合計サイズを計算する： 1. PgBackendStatus 構造体の配列（全スロット分） 2. アプリケーション名バッファ（NAMEDATALEN × スロット数） 3. クライアントホスト名バッファ（NAMEDATALEN × スロット数） 4. クエリ文字列バッファ（pgstat_track_activity_query_size × スロット数） 5. (オプション) SSL状態バッファ（USE_SSL 定義時） 6. (オプション) GSSAPI状態バッファ（ENABLE_GSS 定義時）

各加算には add_size および mul_size を使用し、オーバーフローを防止する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c（共有メモリ初期化プロセスの一部として呼び出される）
  • 呼び出し先: add_size, mul_size（メモリ計算用ユーティリティ）

関連データ構造

  • PgBackendStatus: 各バックエンドの状態を保持する構造体。
  • PgBackendSSLStatus, PgBackendGSSStatus: SSL/GSSAPI情報を保持する構造体。

失敗モードと境界条件

  • NumBackendStatSlots が非常に大きい場合、mul_size 等の内部でメモリ制限に達する可能性がある。

保守担当者向けメモ

  • 新しいステータス情報を共有メモリに追加する場合は、本関数でサイズ計算を更新する必要がある。
  • コンパイルオプション（SSL/GSS）によるサイズ変動があるため、条件付きコンパイルの漏れに注意すること。
