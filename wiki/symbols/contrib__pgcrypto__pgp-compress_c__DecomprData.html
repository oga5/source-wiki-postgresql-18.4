DecomprData

定義

  • 種別: struct
  • 定義箇所: contrib/pgcrypto/pgp-compress.c:193

目的

PGPデータストリームの伸長処理（inflate）において、zlib のストリーム状態と中間バッファを保持するためのコンテキスト構造体です。

シグネチャまたは形状

struct DecomprData
{
    int         buf_len;        /* = ZIP_OUT_BUF */
    int         buf_data;       /* 利用可能なデータ量 */
    uint8      *pos;            /* バッファ内の現在位置 */
    z_stream    stream;         /* zlibのストリーム状態 */
    int         eof;            /* 終了フラグ */
    uint8       buf[ZIP_OUT_BUF]; /* 出力用中間バッファ */
};

入力と出力

本構造体は関数引数として直接受け渡されるのではなく、PullFilter のプライベートデータ（priv）として保持され、伸長処理の各ステップで状態の読み書きに使用されます。

前提条件と不変条件

  • buf_len は常に ZIP_OUT_BUF と等しい必要があります。
  • stream は zlib の関数群（inflate 等）で使用される前に、適切に初期化されている必要があります。
  • pos は常に buf の範囲内を指す必要があります。

アルゴリズムまたは振る舞い

伸長処理において、ソースから読み込んだデータを zlib の inflate() に渡し、伸長結果を buf に格納します。decompress_read 呼び出し時に、このバッファから要求されたサイズ分を切り出して提供し、バッファが空になったら再度 inflate() を実行して補充するサイクルを繰り返します。

呼び出し元と呼び出し先

  • 呼び出し元: contrib/pgcrypto/pgp-compress.c 内の decompress_init、decompress_read、decompress_free がこの構造体を操作します。
  • 呼び出し先: zlib ライブラリの関数（inflate 等）を呼び出します。

関連データ構造

  • z_stream: zlib のコア構造体。
  • PullFilter: 本構造体を内部状態として保持するフィルタインターフェース。

失敗モードと境界条件

  • zlib の伸長エラー発生時、またはストリームの終端判定（eof）において処理が終了します。
  • ZIP_OUT_BUF を超えるデータが一度に要求された場合、バッファの再充填ロジックが適切に動作する必要があります。

保守担当者向けメモ

  • 本構造体は palloc を介してメモリ確保されることを前提としています。
  • zlib がインストールされていない環境では、本構造体を利用する関数群はスタブとして動作するため、実質的に使用されません。
