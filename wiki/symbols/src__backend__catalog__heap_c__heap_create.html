heap_create

定義

  • 種別: function
  • 定義箇所: src/backend/catalog/heap.c:284

目的

カタログに登録されていない（uncataloged）ヒープリレーションをメモリ上に構築し、必要に応じて物理ストレージを作成する。

シグネチャまたは形状

Relation heap_create(const char *relname,
                     Oid relnamespace,
                     Oid reltablespace,
                     Oid relid,
                     RelFileNumber relfilenumber,
                     Oid accessmtd,
                     TupleDesc tupDesc,
                     char relkind,
                     char relpersistence,
                     bool shared_relation,
                     bool mapped_relation,
                     bool allow_system_table_mods,
                     TransactionId *relfrozenxid,
                     MultiXactId *relminmxid,
                     bool create_storage)

入力と出力

  • **入力**: リレーション名、名前空間、テーブルスペース、OID、アクセスメソッド、タプル記述子、リレーション種別、永続性フラグ、共有/マッピングフラグ、システム変更許可フラグ、およびストレージ作成フラグ。
  • **出力**: 構築された Relation 構造体へのポインタ。relfrozenxid および relminmxid には初期値が設定される。

前提条件と不変条件

  • relid は有効な OID である必要がある（Assert による検証）。
  • allow_system_table_mods が偽の場合、システムカタログ名前空間への作成は制限される。
  • reltablespace がデータベースのデフォルトテーブルスペースと一致する場合、内部的に InvalidOid に強制変換される。

アルゴリズムまたは振る舞い

1. **権限チェック**: システムカタログへの不正な作成試行をエラーにする。 2. **初期化**: relfrozenxid と relminmxid を無効値で初期化。 3. **属性調整**: リレーション種別に基づき、テーブルスペースやストレージ作成フラグを調整。 4. **リレーション構築**: RelationBuildLocalRelation を呼び出し、リレーションキャッシュエントリを作成。 5. **物理ストレージ作成**: create_storage が真の場合、アクセスメソッドまたは RelationCreateStorage を介してディスク上のファイルを作成。 6. **依存関係記録**: ストレージを作成しない場合でテーブルスペースが指定されていれば、recordDependencyOnTablespace を呼び出す。 7. **統計情報**: pgstat_create_relation を呼び出し、統計情報を初期化。

呼び出し元と呼び出し先

  • **呼び出し元**: heap_create_with_catalog、index_create（src/backend/catalog/index.c）、およびその他テーブル作成処理を行う関数。
  • **呼び出し先**: RelationBuildLocalRelation、table_relation_set_new_filelocator、RelationCreateStorage、recordDependencyOnTablespace、pgstat_create_relation。

関連データ構造

  • Relation: リレーションのメタデータを保持する構造体。
  • TupleDesc: タプルの列定義情報。
  • RelFileNumber: 物理ファイルの識別子。

失敗モードと境界条件

  • allow_system_table_mods が偽の状態でシステムカタログ名前空間に作成しようとすると ERRCODE_INSUFFICIENT_PRIVILEGE でエラーとなる。
  • ストレージ作成が要求されたが、リレーション種別がストレージをサポートしていない場合は Assert により停止する。

保守担当者向けメモ

  • 本関数は「カタログに登録しない」リレーション作成用である。カタログ登録を伴う場合は heap_create_with_catalog を使用すること。
  • reltablespace の強制的な InvalidOid 変換は、データベースクローン時の整合性を保つためのハックである。
  • システム属性の定義は heap.c 内で静的に管理されているため、変更時は注意が必要。
