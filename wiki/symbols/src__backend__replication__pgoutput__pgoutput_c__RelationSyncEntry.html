RelationSyncEntry

定義

  • **種別**: struct
  • **定義箇所**: src/backend/replication/pgoutput/pgoutput.c:126

目的

論理レプリケーションにおいて、特定のテーブルに対するパブリケーション設定、行フィルタ、列リスト、およびスキーマ送信状態を保持するキャッシュ構造体です。

シグネチャまたは形状

typedef struct RelationSyncEntry {
    Oid relid;
    bool replicate_valid;
    bool schema_sent;
    PublishGencolsType include_gencols_type;
    List *streamed_txns;
    PublicationActions pubactions;
    ExprState *exprstate[NUM_ROWFILTER_PUBACTIONS];
    EState *estate;
    TupleTableSlot *new_slot;
    TupleTableSlot *old_slot;
    Oid publish_as_relid;
    AttrMap *attrmap;
    Bitmapset *columns;
    MemoryContext entry_cxt;
} RelationSyncEntry;

入力と出力

  • **入力**: get_rel_sync_entry 関数等を通じて、対象テーブルのOIDおよびパブリケーション情報に基づき初期化されます。
  • **出力**: レプリケーション処理中に、テーブルの同期状態（スキーマ送信済みか、どの列を送信すべきか等）を判定するための情報源として利用されます。

前提条件と不変条件

  • exprstate は NUM_ROWFILTER_PUBACTIONS 分の要素を持ち、各アクション（INSERT/UPDATE/DELETE）に対応する行フィルタを保持します。
  • attrmap は、パーティションテーブルを親テーブルのスキーマで公開する場合の変換に使用されます。
  • entry_cxt は、この構造体に関連する動的なメモリ（行フィルタ式や列リスト等）を管理するための専用メモリコンテキストです。

アルゴリズムまたは振る舞い

  • **行フィルタ**: 各アクションに対して個別の ExprState を保持します。これは、UPDATE/DELETEがレプリカアイデンティティに依存する一方、INSERTは依存しないという制約の違いを吸収するためです。
  • **スキーマ同期**: schema_sent フラグにより、サブスクライバーへ既にテーブル定義が送信されたかを追跡します。
  • **パーティション対応**: publish_via_partition_root が有効な場合、publish_as_relid に親テーブルのOIDが設定され、attrmap を用いてタプルの変換が行われます。

呼び出し元と呼び出し先

  • **呼び出し元**: get_rel_sync_entry が本構造体の取得・生成を行います。また、pgoutput_change 等の変更処理関数が、対象テーブルの同期状態を確認するために参照します。
  • **呼び出し先**: 構造体内部の EState や ExprState を通じて、PostgreSQLの実行エンジン（executor/executor.h）の機能を利用します。

関連データ構造

  • PublicationActions: パブリケーションの対象アクションを定義。
  • ExprState, EState: 行フィルタ評価に使用。
  • AttrMap: パーティションと親テーブル間の属性マッピング。
  • Bitmapset: 公開対象の列リスト。

失敗モードと境界条件

  • システムカタログの変更によりキャッシュが無効化された場合、replicate_valid フラグ等を用いて再同期が必要です。
  • パーティションと親テーブルで TupleDesc が同一の場合、attrmap は NULL となります。

保守担当者向けメモ

  • 本構造体は pgoutput プラグインのキャッシュ管理の核です。システムカタログの変更時にキャッシュを適切に無効化するコールバック（rel_sync_cache_relation_cb 等）との整合性に注意してください。
  • streamed_txns はストリーミングトランザクションのスキーマ状態管理に使用されます。
