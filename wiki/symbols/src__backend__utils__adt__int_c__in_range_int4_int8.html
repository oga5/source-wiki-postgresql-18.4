in_range_int4_int8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/int.c:697

目的

ウィンドウ関数のフレーム境界計算において、int4 型の値を int8 型のオフセットに基づいて比較し、範囲内にあるかを判定します。

シグネチャまたは形状

Datum in_range_int4_int8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_INT32(0): 比較対象の値 (val)
  • PG_GETARG_INT32(1): 基準値 (base)
  • PG_GETARG_INT64(2): オフセット値 (offset)
  • PG_GETARG_BOOL(3): オフセットを減算するか (sub)
  • PG_GETARG_BOOL(4): 範囲の境界が「より小さい」側か (less)
  • **出力**: bool 型の判定結果を Datum として返却。

前提条件と不変条件

  • すべての算術演算は int64 型にキャストして実行されます。
  • offset が負の値である場合、ERRCODE_INVALID_PRECEDING_OR_FOLLOWING_SIZE エラーが発生します。

アルゴリズムまたは振る舞い

1. val と base を int64 に変換します。 2. sub が真の場合、offset を反転させます。 3. pg_add_s64_overflow を使用して base + offset を計算します。

  • オーバーフローが発生した場合: sub が偽なら val は常に sum より小さいとみなし less を返し、sub が真なら !less を返します。

    • 4. オーバーフローしない場合:

  • less が真なら val <= sum を判定。
  • less が偽なら val >= sum を判定。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/int.c:783 (DirectFunctionCall5経由)
  • **呼び出し先**:
  • pg_add_s64_overflow (算術オーバーフローチェック)
  • ereport (エラー報告)

関連データ構造

  • Datum: PostgreSQLの汎用データ表現。
  • PG_FUNCTION_ARGS: 関数呼び出し引数コンテナ。

失敗モードと境界条件

  • offset < 0: ウィンドウ関数のサイズ指定不正としてエラーをスローします。
  • int64 範囲外の加算: pg_add_s64_overflow により検出し、論理的な大小関係に基づいて結果を決定します。

保守担当者向けメモ

  • 本関数は int4 と int8 の混合演算を扱うため、精度維持のためにすべて int64 で計算しています。
  • オーバーフロー時の挙動は、ウィンドウ関数の境界条件（less フラグ）に基づいた論理的な推論に依存しています。
