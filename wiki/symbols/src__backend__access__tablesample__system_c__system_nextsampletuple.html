system_nextsampletuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/tablesample/system.c:235

目的

TABLESAMPLE SYSTEM メソッドにおいて、現在選択されているブロック内のタプルを順次走査し、次に読み込むべきタプルのオフセット番号を決定する。

シグネチャまたは形状

static OffsetNumber system_nextsampletuple(SampleScanState *node, BlockNumber blockno, OffsetNumber maxoffset)

入力と出力

  • **入力**:
  • node: SampleScanState 構造体へのポインタ。内部状態 tsm_state を保持する。
  • blockno: 現在処理中のブロック番号。
  • maxoffset: 現在のブロックにおける最大オフセット番号。
  • **出力**:
  • 次の有効な OffsetNumber を返す。ブロック内の全タプルを走査し終えた場合は InvalidOffsetNumber を返す。

前提条件と不変条件

  • node->tsm_state は SystemSamplerData 型としてキャスト可能であること。
  • sampler->lt には、前回の呼び出しで返されたオフセット番号（または初期状態）が保持されていること。

アルゴリズムまたは振る舞い

1. node->tsm_state から SystemSamplerData を取得し、現在のオフセット lt を読み込む。 2. lt が InvalidOffsetNumber の場合は FirstOffsetNumber から開始し、それ以外の場合は lt をインクリメントして次のオフセットへ進める。 3. 計算されたオフセットが maxoffset を超えた場合、走査終了とみなし InvalidOffsetNumber を設定する。 4. 更新されたオフセットを sampler->lt に保存し、その値を呼び出し元へ返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/tablesample/system.c 内の tsm_system_handler 構造体を通じて、クエリ実行エンジンから呼び出される。
  • **呼び出し先**: 直接的な関数呼び出しは行わない。

関連データ構造

  • SystemSamplerData: スキャン中の状態（現在のオフセット lt を含む）を保持する構造体。
  • SampleScanState: 実行時のスキャン状態を保持する上位構造体。

失敗モードと境界条件

  • maxoffset が 0 の場合、最初のチェックで tupoffset > maxoffset となり、直ちに InvalidOffsetNumber が返される。
  • ブロック内の最終タプルに到達した後の呼び出しでは、常に InvalidOffsetNumber が返される。

保守担当者向けメモ

  • 本関数はブロック内のタプルを線形に走査する役割を持つ。system メソッドの特性上、選択されたブロック内の全タプルを返す必要があるため、このロジックはブロック内の全オフセットを網羅するように設計されている。
