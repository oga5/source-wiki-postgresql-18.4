reservoir_init_selection_state

定義

  • 種別: function
  • 定義箇所: src/backend/utils/misc/sampling.c:132

目的

リザーバーサンプリング（VitterのAlgorithm Z）を実行するための状態を初期化します。

シグネチャまたは形状

void reservoir_init_selection_state(ReservoirState rs, int n)

入力と出力

  • rs (ReservoirState): 初期化対象となるリザーバーサンプリングの状態構造体。
  • n (int): サンプリングする要素数。
  • 出力: なし（rs が指す構造体の内部状態が更新されます）。

前提条件と不変条件

  • rs は有効なメモリ領域を指している必要があります。
  • n は正の整数である必要があります（内部で log 関数の引数として使用されるため）。

アルゴリズムまたは振る舞い

1. pg_global_prng_state から取得した値を用いて、rs->randstate を初期化します。 2. リザーバーサンプリングの Algorithm Z で使用される変数 W を計算します。

  • W = exp(-log(sampler_random_fract(&rs->randstate)) / n)
  • sampler_random_fract は 0 を含まない一様な乱数を生成するため、対数計算は安全に行われます。

呼び出し元と呼び出し先

  • 呼び出し元:
  • contrib/file_fdw/file_fdw.c
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/commands/analyze.c
  • 呼び出し先:
  • sampler_random_init_state
  • pg_prng_uint32
  • sampler_random_fract
  • exp (math.h)
  • log (math.h)

関連データ構造

  • ReservoirState: サンプリングの状態（乱数生成器の状態 randstate および変数 W）を保持する構造体。

失敗モードと境界条件

  • n が 0 または負の場合、log 関数の引数や除算において予期しない動作を引き起こす可能性があります。
  • sampler_random_fract が 0 を返すことはないため、log(0) によるエラーは発生しません。

保守担当者向けメモ

  • 本関数は再現可能な結果を必要としない箇所で使用されるため、乱数生成器はランダムに初期化されます。
  • 新規コードでは、レガシーな anl_ 系関数ではなく、本関数を含む新しいAPIを使用してください。
