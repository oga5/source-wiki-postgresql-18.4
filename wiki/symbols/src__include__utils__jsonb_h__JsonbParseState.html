JsonbParseState

定義

  • 種別: struct
  • 定義箇所: src/include/utils/jsonb.h:319

目的

jsonb データを構築する際の中間状態を保持するための構造体です。pushJsonbValue 関数と組み合わせて使用され、ストリーム状の入力から jsonb ツリーを段階的に構築します。

シグネチャまたは形状

typedef struct JsonbParseState
{
    JsonbValue    contVal;      /* 現在のコンテナ値 */
    Size          size;         /* コンテナのサイズ */
    struct JsonbParseState *next; /* 親コンテナへのポインタ */
    bool          unique_keys;  /* オブジェクトのキーの一意性をチェックするか */
    bool          skip_nulls;   /* nullのオブジェクトフィールドをスキップするか */
} JsonbParseState;

入力と出力

  • **入力**: pushJsonbValue 関数に引数として渡され、構築中の状態を更新します。
  • **出力**: 構築完了後、最終的な JsonbValue が得られ、これを JsonbValueToJsonb でオンディスク形式の Jsonb に変換します。

前提条件と不変条件

  • next ポインタにより、ネストされたJSON構造（オブジェクトや配列）をスタックとして管理します。
  • unique_keys が true の場合、オブジェクト構築時に重複キーの検証が行われます。

アルゴリズムまたは振る舞い

1. **初期化**: 構築開始時に NULL から開始し、pushJsonbValue が必要に応じてメモリを割り当てます。 2. **構築**: pushJsonbValue を繰り返し呼び出すことで、JsonbValue のツリーを構築します。 3. **ネスト処理**: 新しいオブジェクトや配列を開始するたびに、新しい JsonbParseState がスタックにプッシュされます。 4. **終了**: 構築終了時にスタックを遡り、最終的な Jsonb 構造を生成します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/hstore/hstore_io.c
  • contrib/jsonb_plperl/jsonb_plperl.c
  • contrib/jsonb_plpython/jsonb_plpython.c
  • **呼び出し先**:
  • pushJsonbValue（構築の主要関数）

関連データ構造

  • JsonbValue: メモリ上でJSON要素を表現する構造体。
  • Jsonb: オンディスク形式のルート構造体。

失敗モードと境界条件

  • メモリ不足時には palloc 経由でエラーが発生します。
  • unique_keys が有効な場合、重複キーが検出されるとエラーが発生する可能性があります。

保守担当者向けメモ

  • JsonbParseState は再帰的な構造をスタックとして保持するため、メモリ管理は palloc に依存しています。
  • JEntry のオフセット計算など、jsonb のオンディスク形式への変換ロジックと密接に関連しているため、変更時は src/backend/utils/adt/jsonb.c の実装を確認してください。
