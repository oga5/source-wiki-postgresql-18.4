lexeme_match

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/ts_typanalyze.c:489

目的

ハッシュテーブル内での語彙（lexeme）比較を行うためのコールバック関数です。

シグネチャまたは形状

static int lexeme_match(const void *key1, const void *key2, Size keysize)

入力と出力

  • **入力**: 比較対象となる2つのキー（key1, key2）およびキーサイズ（keysize）。
  • **出力**: 2つのキーが一致する場合は 0、それ以外の場合は非ゼロの整数を返します。

前提条件と不変条件

  • keysize 引数は使用されません。これは、各キー構造体自体が自身の長さを保持しているためです。
  • 内部的に lexeme_compare を呼び出すため、比較対象のキーは lexeme_compare が期待する形式（LexemeHashKey）である必要があります。

アルゴリズムまたは振る舞い

1. keysize 引数を無視します。 2. lexeme_compare(key1, key2) を呼び出し、その結果をそのまま戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tsearch/ts_typanalyze.c 内の compute_tsvector_stats 関数にて、ハッシュテーブルの制御構造（hash_ctl.match）に登録されます。
  • **呼び出し先**: lexeme_compare 関数。

関連データ構造

  • LexemeHashKey: 比較対象となる語彙の識別用構造体。

失敗モードと境界条件

  • 特筆すべき失敗モードはありません。lexeme_compare の仕様に依存します。

保守担当者向けメモ

  • 本関数はハッシュテーブルのインターフェース適合のためのラッパーです。比較ロジックを変更する場合は lexeme_compare を修正してください。
