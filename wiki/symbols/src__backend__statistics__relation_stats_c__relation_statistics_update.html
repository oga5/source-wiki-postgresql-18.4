relation_statistics_update

定義

  • 種別: function
  • 定義箇所: src/backend/statistics/relation_stats.c:64

目的

ユーザーが指定した統計値（ページ数、タプル数、可視性情報）を用いて、対象リレーションの pg_class カタログエントリを更新する。

シグネチャまたは形状

static bool relation_statistics_update(FunctionCallInfo fcinfo)

入力と出力

  • **入力**: FunctionCallInfo fcinfo（スキーマ名、リレーション名、および更新対象の統計値を含むSQL引数）。
  • **出力**: bool（更新が成功した場合は true、無効なパラメータ等で更新をスキップした場合は false）。

前提条件と不変条件

  • リカバリ中（RecoveryInProgress()）ではないこと。
  • 対象テーブルに対して ShareUpdateExclusiveLock を取得できること。
  • reltuples 引数が指定された場合、その値は -1.0 以上であること。

アルゴリズムまたは振る舞い

1. 引数からスキーマ名とリレーション名を取得し、RangeVarGetRelidExtended で対象の OID を特定しつつ ShareUpdateExclusiveLock を取得する。 2. 指定された統計値（relpages, reltuples, relallvisible, relallfrozen）を解析する。 3. pg_class システムカタログを RowExclusiveLock でオープンし、対象リレーションのタプルをシステムキャッシュから検索する。 4. 現在の値と異なる場合のみ、heap_modify_tuple_by_cols を用いて変更対象カラムを特定し、CatalogTupleUpdate で更新を適用する。 5. CommandCounterIncrement() を呼び出し、変更を可視化する。 6. ロックを解放して終了する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/statistics/relation_stats.c 内の SQL 関数（pg_restore_relation_stats 等）から呼び出される。
  • **呼び出し先**:
  • stats_check_required_arg, RangeVarGetRelidExtended, table_open, SearchSysCache1, heap_modify_tuple_by_cols, CatalogTupleUpdate, CommandCounterIncrement 等の PostgreSQL 内部 API。

関連データ構造

  • pg_class: 更新対象となるシステムカタログ。
  • relarginfo: 引数のマッピング定義（src/backend/statistics/relation_stats.c 内）。

失敗モードと境界条件

  • **エラー**: リカバリ中の呼び出し、対象リレーションが見つからない場合、または pg_class エントリが見つからない場合はエラー（elog(ERROR)）となる。
  • **警告**: reltuples が -1.0 未満の場合、警告を発行し false を返す。
  • **境界条件**: 値に変更がない場合は更新処理をスキップする。

保守担当者向けメモ

  • vac_update_relstats() と同様のロック戦略を採用しているため、デッドロックの可能性を考慮すること。
  • 更新処理は CatalogTupleUpdate を介して行われるため、カタログの整合性ルールに従う必要がある。
