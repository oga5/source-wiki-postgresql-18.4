parse_basebackup_options

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup.c:697

目的

BASE_BACKUP コマンドで指定されたオプションリストを解析し、basebackup_options 構造体に設定値を格納する。

シグネチャまたは形状

static void parse_basebackup_options(List *options, basebackup_options *opt);

入力と出力

  • **入力**:
  • options: DefElem を要素に持つ List。コマンド引数。
  • opt: 設定値を書き込むための basebackup_options 構造体へのポインタ。
  • **出力**: なし（opt を直接更新する）。

前提条件と不変条件

  • opt は MemSet により初期化され、デフォルト値（manifest は MANIFEST_OPTION_NO、compression は PG_COMPRESSION_NONE 等）が適用される。
  • 各オプションは一度しか指定できない。重複した場合は ERRCODE_SYNTAX_ERROR でエラー終了する。

アルゴリズムまたは振る舞い

1. basebackup_options 構造体をゼロクリアし、デフォルト値を設定する。 2. options リストを走査し、各 DefElem の defname に基づいて以下の処理を行う：

  • label, progress, checkpoint, wait, wal, incremental, max_rate, tablespace_map, verify_checksums, manifest, manifest_checksums, target 等の引数を解析。
  • max_rate は MAX_RATE_LOWER から MAX_RATE_UPPER の範囲内である必要がある。
  • incremental が有効な場合、summarize_wal が有効でなければエラーとなる。
  • verify_checksums はグローバル変数 noverify_checksums を更新する。

    • 3. 不明なオプションや不正な値が指定された場合は ereport によりエラーを発生させる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/backup/basebackup.c 内の SendBaseBackup およびバックアップ処理の初期化フローから呼び出される。
  • **呼び出し先**: defGetString, defGetBoolean, defGetInt64, pg_strcasecmp, pg_checksum_parse_type, parse_bool 等のユーティリティ関数。

関連データ構造

  • basebackup_options: バックアップ設定を保持する構造体。
  • List, ListCell, DefElem: PostgreSQL のノード管理用データ構造。

失敗モードと境界条件

  • **重複指定**: 同一オプションが複数回指定された場合、ERRCODE_SYNTAX_ERROR を送出。
  • **範囲外の値**: max_rate が許容範囲外の場合、ERRCODE_NUMERIC_VALUE_OUT_OF_RANGE を送出。
  • **前提条件違反**: WAL サマライズが無効な状態で増分バックアップを要求した場合、ERRCODE_OBJECT_NOT_IN_PREREQUISITE_STATE を送出。
  • **不正な値**: checkpoint や manifest などで未定義の値を指定した場合、ERRCODE_SYNTAX_ERROR を送出。

保守担当者向けメモ

  • 新しいバックアップオプションを追加する際は、basebackup_options 構造体へのフィールド追加と、本関数内での解析ロジックの追加が必要。
  • noverify_checksums のようにグローバル変数を直接操作するオプションがあるため、並行処理や状態管理には注意すること。
