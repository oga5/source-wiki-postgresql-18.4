gzip_palloc

定義

  • 種別: function
  • 定義箇所: src/backend/backup/basebackup_gzip.c:288

目的

zlib ライブラリが内部メモリ割り当てを行う際に使用する、PostgreSQLのメモリ管理システム（palloc）へのラッパー関数です。

シグネチャまたは形状

static void *gzip_palloc(void *opaque, unsigned items, unsigned size)

入力と出力

  • **入力**:
  • opaque: 未使用（zlib のインターフェース要件による）。
  • items: 割り当てる要素数。
  • size: 各要素のサイズ（バイト単位）。
  • **出力**:
  • 割り当てられたメモリ領域へのポインタ。

前提条件と不変条件

  • palloc が呼び出されるため、PostgreSQLのメモリコンテキストが有効な状態で呼び出される必要があります。
  • items * size の計算結果がメモリ割り当て可能な範囲内である必要があります。

アルゴリズムまたは振る舞い

1. items と size を乗算し、必要な合計バイト数を算出します。 2. PostgreSQLの palloc 関数を呼び出し、メモリを確保してそのポインタを返します。

呼び出し元と呼び出し先

  • **呼び出し元**: zlib の z_stream 構造体（zs->zalloc フィールドに本関数が設定され、deflateInit2 等の内部で呼び出されます）。
  • **呼び出し先**: palloc（PostgreSQLのメモリ割り当て関数）。

関連データ構造

  • z_stream: zlib の圧縮状態を保持する構造体。本関数はこの構造体の zalloc フィールドに代入されます。

失敗モードと境界条件

  • palloc がメモリ不足を検知した場合、PostgreSQLの標準的なエラーハンドリング（elog(ERROR, ...)）により処理が中断されます。

保守担当者向けメモ

  • 本関数は src/fe_utils/astreamer_gzip.c にも同名の関数が存在しますが、本ページは src/backend/backup/basebackup_gzip.c 内の定義を対象としています。
  • opaque 引数は zlib の仕様上必須ですが、本実装では使用されていません。
