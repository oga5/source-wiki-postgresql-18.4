pgstat_report_checkpointer

定義

  • 種別: function
  • 定義箇所: src/backend/utils/activity/pgstat_checkpointer.c:30

目的

プロセスローカルなチェックポイント統計バッファ（PendingCheckpointerStats）の内容を共有メモリ上の統計構造体へ反映し、バッファをリセットする。

シグネチャまたは形状

void pgstat_report_checkpointer(void)

入力と出力

  • **入力**: グローバル変数 PendingCheckpointerStats（プロセスローカルな統計値）。
  • **出力**: なし（共有メモリ上の pgStatLocal.shmem->checkpointer を更新し、PendingCheckpointerStats をゼロクリアする）。

前提条件と不変条件

  • pgStatLocal.shmem->is_shutdown が偽であること。
  • pgstat_assert_is_up() により、統計サブシステムが稼働中であること。
  • 共有メモリへの書き込みは pgstat_begin_changecount_write および pgstat_end_changecount_write で保護されること。

アルゴリズムまたは振る舞い

1. PendingCheckpointerStats がすべてゼロであれば、何もせず終了する。 2. pgstat_begin_changecount_write を呼び出し、共有メモリの更新を開始する。 3. PendingCheckpointerStats に保持されている各カウンタ（num_timed, write_time, buffers_written 等）を、共有メモリ上の対応するフィールドに加算する。 4. pgstat_end_changecount_write を呼び出し、更新を完了する。 5. MemSet を使用して PendingCheckpointerStats をゼロクリアする。 6. pgstat_flush_io(false) を呼び出し、IO統計のフラッシュを行う。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/postmaster/checkpointer.c 内のチェックポインタプロセス（複数箇所）。
  • **呼び出し先**:
  • pgstat_begin_changecount_write / pgstat_end_changecount_write
  • MemSet
  • pgstat_flush_io

関連データ構造

  • PendingCheckpointerStats: プロセスローカルな統計バッファ。
  • PgStatShared_Checkpointer: 共有メモリ上の統計構造体。

失敗モードと境界条件

  • 統計データがすべてゼロの場合、共有メモリへの書き込み処理をスキップする最適化が行われる。
  • 共有メモリの更新はアトミックな changecount 管理下で行われるため、競合が発生した場合は pgstat サブシステムの規定に従う。

保守担当者向けメモ

  • 本関数はチェックポインタプロセスが統計を報告する際のメインインターフェースである。
  • 統計の追加項目が必要な場合は、CHECKPOINTER_ACC マクロの定義箇所にフィールドを追加すること。
