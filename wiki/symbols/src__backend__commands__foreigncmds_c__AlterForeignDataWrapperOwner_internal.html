AlterForeignDataWrapperOwner_internal

定義

  • 種別: function
  • 定義箇所: src/backend/commands/foreigncmds.c:215

目的

外部データラッパー（FDW）の所有者を変更し、システムカタログの更新および所有者依存関係の再構築を行う。

シグネチャまたは形状

static void AlterForeignDataWrapperOwner_internal(Relation rel, HeapTuple tup, Oid newOwnerId)

入力と出力

  • rel: pg_foreign_data_wrapper システムカタログのリレーション（RowExclusiveLock 保持を想定）。
  • tup: 変更対象の FDW を指す HeapTuple。
  • newOwnerId: 新しい所有者の Oid。
  • 出力: なし（副作用としてカタログを更新）。

前提条件と不変条件

  • 呼び出し元はスーパーユーザーである必要がある。
  • newOwnerId で指定される新しい所有者もスーパーユーザーである必要がある。
  • rel は pg_foreign_data_wrapper システムカタログであること。

アルゴリズムまたは振る舞い

1. **権限チェック**: 現在のユーザーおよび newOwnerId がスーパーユーザーであることを確認する。 2. **所有者変更の判定**: form->fdwowner が newOwnerId と異なる場合のみ処理を続行する。 3. **カタログ更新準備**: fdwowner 属性を更新対象としてマークする。 4. **ACL 更新**: FDW に ACL が設定されている場合、aclnewowner を用いて所有者変更に伴う ACL の再構築を行い、更新対象に加える。 5. **カタログ更新**: heap_modify_tuple および CatalogTupleUpdate を使用してシステムカタログを更新する。 6. **依存関係更新**: changeDependencyOnOwner を呼び出し、所有者に関する依存関係を更新する。 7. **フック通知**: InvokeObjectPostAlterHook を呼び出し、変更をサブシステムに通知する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/commands/foreigncmds.c 内の AlterForeignDataWrapperOwner および AlterForeignDataWrapperOwner_oid からの呼び出しが確認されている。
  • 呼び出し先: superuser, superuser_arg, heap_getattr, aclnewowner, heap_modify_tuple, CatalogTupleUpdate, changeDependencyOnOwner, InvokeObjectPostAlterHook。

関連データ構造

  • Form_pg_foreign_data_wrapper: FDW のカタログ定義構造体。
  • Relation: システムカタログのアクセスに使用。
  • HeapTuple: カタログ行の操作に使用。

失敗モードと境界条件

  • スーパーユーザー権限がない場合、ERRCODE_INSUFFICIENT_PRIVILEGE でエラーを発生させる。
  • newOwnerId がスーパーユーザーでない場合、同様にエラーを発生させる。

保守担当者向けメモ

  • 本関数は pg_foreign_data_wrapper カタログの直接的な更新を行うため、呼び出し元で適切なロックが取得されていることを前提としている。
  • ACL の更新ロジックは aclnewowner に依存しており、FDW 固有の権限管理ルールに変更がある場合はここを確認すること。
