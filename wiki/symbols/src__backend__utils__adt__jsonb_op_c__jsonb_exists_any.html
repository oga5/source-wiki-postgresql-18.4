jsonb_exists_any

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/jsonb_op.c:45

目的

jsonb 型のデータ内に、指定されたテキスト配列のいずれかの要素（キーまたは値）が存在するかを判定します。SQLの ?| 演算子を実装します。

シグネチャまたは形状

Datum jsonb_exists_any(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_JSONB_P(0): 検索対象の jsonb オブジェクト。
  • PG_GETARG_ARRAYTYPE_P(1): 検索するキーまたは値のリストを含む text 型の配列。
  • **出力**:
  • bool: 配列内のいずれかの要素が jsonb 内に見つかった場合は true、それ以外は false を返します。

前提条件と不変条件

  • 入力配列の要素は TEXTOID 型として扱われます。
  • 検索はトップレベルのキーまたは要素に対してのみ行われ、再帰的な探索は行われません。
  • 配列要素が NULL の場合はスキップされます。

アルゴリズムまたは振る舞い

1. deconstruct_array_builtin を使用して、入力配列を Datum 配列に分解します。 2. 分解された各要素に対してループ処理を行います。 3. 各要素を JsonbValue（型 jbvString）に変換します。 4. findJsonbValueFromContainer を呼び出し、jsonb のルートコンテナ内に当該文字列が存在するかを確認します。 5. 一致するものが見つかった時点で直ちに true を返し、ループを終了します。 6. 全ての要素を確認しても見つからない場合は false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（?| 演算子経由）。
  • **呼び出し先**:
  • deconstruct_array_builtin
  • findJsonbValueFromContainer
  • PG_GETARG_JSONB_P, PG_GETARG_ARRAYTYPE_P, PG_RETURN_BOOL

関連データ構造

  • Jsonb: 検索対象となるJSONB構造体。
  • JsonbValue: 検索キーを表現するための一時的な構造体。
  • ArrayType: 入力された検索キーの配列。

失敗モードと境界条件

  • 入力配列が空の場合、ループは実行されず false を返します。
  • 配列内に NULL 要素が含まれる場合、その要素は無視されます。
  • jsonb がオブジェクトまたは配列でない場合、findJsonbValueFromContainer の仕様に従い検索が行われます。

保守担当者向けメモ

  • 本関数は再帰的な探索を行わないため、ネストされた構造内の検索には対応していません。
  • 配列要素の toasted 状態については、VARDATA_ANY および VARSIZE_ANY_EXHDR を使用して適切に処理されています。
