tidout

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:118

目的

TID（Tuple Identifier）型の内部表現を、人間が読み取り可能な文字列形式 (block,offset) に変換します。

シグネチャまたは形状

Datum tidout(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_ITEMPOINTER(0) を介して取得される ItemPointer 型のポインタ。
  • **出力**: (block,offset) 形式の文字列（cstring）を Datum として返却。

前提条件と不変条件

  • 入力された ItemPointer は有効なメモリ領域を指している必要があります。
  • 出力文字列は pstrdup を使用してメモリが確保されるため、呼び出し元で適切に管理される必要があります。

アルゴリズムまたは振る舞い

1. ItemPointerGetBlockNumberNoCheck および ItemPointerGetOffsetNumberNoCheck を使用して、ItemPointer からブロック番号とオフセット番号を抽出します。 2. snprintf を用いて、固定長バッファ buf[32] に (block,offset) の形式で文字列を書き込みます。 3. pstrdup を呼び出してバッファの内容をコピーし、そのポインタを PG_RETURN_CSTRING マクロで返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/pgrowlocks/pgrowlocks.c:144 からの呼び出しが確認されています。
  • **呼び出し先**:
  • ItemPointerGetBlockNumberNoCheck
  • ItemPointerGetOffsetNumberNoCheck
  • snprintf
  • pstrdup

関連データ構造

  • ItemPointer: TID の内部表現。BlockIdData（ブロック番号）と OffsetNumber（オフセット番号）で構成されます。

失敗モードと境界条件

  • ブロック番号やオフセット番号が unsigned int の範囲を超えることはありませんが、snprintf のバッファサイズ（32バイト）を超えるような異常な値が入力された場合、文字列が切り詰められる可能性があります。

保守担当者向けメモ

  • 現在の出力形式は (block,offset) です。ソースコード内のコメントにある通り、将来的にレコード形式での出力へ変更する検討の余地があります。
