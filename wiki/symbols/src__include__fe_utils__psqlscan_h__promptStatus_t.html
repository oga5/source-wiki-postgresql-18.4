promptStatus_t

定義

  • 種別: enum
  • 定義箇所: src/include/fe_utils/psqlscan.h:39

目的

SQLコマンドの字句解析中における現在の入力状態を追跡し、対話型シェル（psql等）が適切なプロンプトを表示できるようにするための列挙型です。

シグネチャまたは形状

typedef enum _promptStatus
{
	PROMPT_READY,
	PROMPT_CONTINUE,
	PROMPT_COMMENT,
	PROMPT_SINGLEQUOTE,
	PROMPT_DOUBLEQUOTE,
	PROMPT_DOLLARQUOTE,
	PROMPT_PAREN,
	PROMPT_COPY,
} promptStatus_t;

入力と出力

  • **入力**: 字句解析器が入力ストリームを処理する際、現在のトークン解析状況に応じてこの列挙値が更新されます。
  • **出力**: psql の get_prompt() 関数等に渡され、ユーザーに表示するプロンプト文字列の決定に使用されます。

前提条件と不変条件

  • psql_scan() を含む解析ループにおいて、現在の解析状態を反映する変数として保持されます。
  • 状態は解析の進行に応じて動的に遷移します（例: 引用符の開始で PROMPT_SINGLEQUOTE へ遷移し、終了で PROMPT_READY へ戻る）。

アルゴリズムまたは振る舞い

  • PROMPT_READY: 新しいコマンド入力を待機している状態。
  • PROMPT_COMMENT / PROMPT_SINGLEQUOTE / PROMPT_DOUBLEQUOTE / PROMPT_DOLLARQUOTE: それぞれコメント内、単一引用符内、二重引用符内、ドル引用符内であることを示します。
  • PROMPT_PAREN: 括弧内に存在することを示します。
  • PROMPT_COPY: COPY FROM STDIN 等のデータ入力モードであることを示します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/bin/psql/mainloop.c: 解析状態の保持に使用。
  • src/bin/psql/prompt.c: get_prompt() の引数として使用。
  • src/bin/pgbench/pgbench.c および src/test/modules/test_escape/test_escape.c: 状態管理変数として使用。
  • **呼び出し先**: 本シンボルは列挙型であり、関数ではないため呼び出し先はありません。

関連データ構造

  • PsqlScanState: 解析器の内部状態を保持する構造体。
  • PsqlScanResult: 解析結果の終了状態を示す列挙型。

失敗モードと境界条件

  • 状態遷移の不整合は、字句解析器のバグや不正な入力構文に起因する可能性があります。
  • 境界条件として、引用符のネストやコメントの入れ子状態が正しく処理されない場合、プロンプトが誤った状態を表示する可能性があります。

保守担当者向けメモ

  • psql のプロンプト表示ロジックと密接に関連しています。新しいプロンプト状態を追加する場合は、get_prompt() 側の対応も必要となります。
  • src/bin/psql/prompt.h にも言及がある通り、psqlscan.h が定義の正本です。
