create_plain_partial_paths

定義

  • 種別: function
  • 定義箇所: src/backend/optimizer/path/allpaths.c:805

目的

指定されたリレーションに対して、並列実行可能な順序なしのパス（Partial Path）を生成し、リレーションのパスリストに追加する。

シグネチャまたは形状

static void create_plain_partial_paths(PlannerInfo *root, RelOptInfo *rel)

入力と出力

  • **入力**:
  • root: プランナのグローバル情報（PlannerInfo）。
  • rel: パスを追加対象とするリレーション情報（RelOptInfo）。
  • **出力**: なし（副作用として rel->partial_pathlist にパスが追加される）。

前提条件と不変条件

  • rel->pages に基づいて並列ワーカー数が計算される。
  • max_parallel_workers_per_gather 設定が並列実行の可否に影響する。

アルゴリズムまたは振る舞い

1. compute_parallel_worker を呼び出し、利用可能な並列ワーカー数を算出する。 2. ワーカー数が 0 以下の場合は、並列スキャンが不要と判断して即座に終了する。 3. create_seqscan_path を使用して、並列シーケンシャルスキャンパスを作成する。 4. add_partial_path を呼び出し、生成したパスをリレーションの並列パスリストに登録する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/optimizer/path/allpaths.c 内の関数から呼び出される（直接的な呼び出し元関数の詳細はソース抜粋からは不明）。
  • **呼び出し先**:
  • compute_parallel_worker
  • create_seqscan_path
  • add_partial_path

関連データ構造

  • PlannerInfo: プランナのコンテキスト情報。
  • RelOptInfo: 最適化対象のリレーション情報。

失敗モードと境界条件

  • parallel_workers が 0 以下の場合、パスは追加されず正常に終了する。

保守担当者向けメモ

  • 本関数は並列シーケンシャルスキャンのみを対象としている。他のアクセス手法（インデックススキャン等）の並列化が必要な場合は、別途ロジックの追加が必要となる。
