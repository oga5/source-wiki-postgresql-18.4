float_zero_divide_error

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/float.c:101

目的

浮動小数点演算におけるゼロ除算が発生した際に、PostgreSQLの標準的なエラーメカニズムを通じて例外を報告する。

シグネチャまたは形状

void float_zero_divide_error(void)

入力と出力

  • 入力: なし
  • 出力: なし（本関数はエラーを発生させ、呼び出し元へは復帰しない）

前提条件と不変条件

  • ereport マクロを介して ERROR レベルの例外を発生させるため、呼び出し後は制御が呼び出し元に戻らない。

アルゴリズムまたは振る舞い

1. ereport を呼び出し、ERRCODE_DIVISION_BY_ZERO エラーコードを設定する。 2. エラーメッセージとして "division by zero" を出力する。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/utils/float.h 内のインライン関数（例: 浮動小数点演算を行う算術演算子）から、ゼロ除算を検知した際に呼び出される。
  • 呼び出し先: PostgreSQLの ereport マクロ（エラー報告サブシステム）。

関連データ構造

  • なし

失敗モードと境界条件

  • 本関数自体が失敗することはない。常に ERROR を発生させることで、トランザクションの異常終了をトリガーする。

保守担当者向けメモ

  • pg_noinline 指定により、関数呼び出しのオーバーヘッドを許容しつつ、エラー処理コードをメインの演算ロジックから分離している。
  • src/include/utils/float.h で pg_noreturn 属性付きで宣言されており、コンパイラに対して本関数が復帰しないことを伝えている。
