TupleDescCompactAttr

定義

  • 種別: static inline 関数
  • 定義箇所: src/include/access/tupdesc.h:174

目的

タプル記述子（TupleDesc）から、指定された属性インデックスに対応する CompactAttribute 構造体へのポインタを取得します。

シグネチャまたは形状

CompactAttribute *TupleDescCompactAttr(TupleDesc tupdesc, int i)

入力と出力

  • **入力**:
  • tupdesc: 対象のタプル記述子。
  • i: 取得対象の属性インデックス（0から始まる）。
  • **出力**:
  • 指定された属性の CompactAttribute 構造体へのポインタ。

前提条件と不変条件

  • tupdesc は有効なメモリ領域を指している必要があります。
  • i は 0 以上 tupdesc->natts 未満である必要があります。
  • USE_ASSERT_CHECKING が有効な場合、内部で verify_compact_attribute(tupdesc, i) が呼び出され、属性情報が正しく設定されていることが検証されます。

アルゴリズムまたは振る舞い

1. tupdesc->compact_attrs 配列の i 番目の要素のアドレスを計算します。 2. USE_ASSERT_CHECKING が定義されている場合、verify_compact_attribute を実行して整合性を確認します。 3. 計算したアドレスを返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_heapam.c
  • contrib/pageinspect/heapfuncs.c
  • contrib/postgres_fdw/postgres_fdw.c
  • src/backend/access/brin/brin_inclusion.c
  • src/backend/access/brin/brin_tuple.c
  • src/backend/access/common/attmap.c
  • src/backend/access/common/heaptuple.c
  • **呼び出し先**:
  • verify_compact_attribute（USE_ASSERT_CHECKING 定義時のみ）

関連データ構造

  • TupleDesc: タプル記述子のメイン構造体。
  • CompactAttribute: タプル変形（Deformation）を高速化するための軽量な属性定義構造体。

失敗モードと境界条件

  • インデックス i が範囲外の場合、メモリ破壊やセグメンテーション違反が発生する可能性があります。
  • populate_compact_attribute() による初期化が完了していない TupleDesc に対して呼び出した場合、不整合なデータが返される可能性があります。

保守担当者向けメモ

  • 本関数はパフォーマンスに直結するインライン関数です。ロジックの変更はタプル変形処理全体に影響を与えます。
  • CompactAttribute の構造を変更する場合は、populate_compact_attribute() の実装と整合性が取れていることを確認してください。
