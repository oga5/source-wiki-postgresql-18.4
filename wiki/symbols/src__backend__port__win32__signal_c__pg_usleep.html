pg_usleep

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32/signal.c:52

目的

指定されたマイクロ秒数だけ待機する。Windows環境において、シグナルイベントの発生を監視しつつ、割り込み可能なスリープを実現する。

シグネチャまたは形状

void pg_usleep(long microsec)

入力と出力

  • **入力**: microsec (待機するマイクロ秒数)
  • **出力**: なし（戻り値なし）

前提条件と不変条件

  • pgwin32_signal_event が初期化されている場合、この関数はシグナルイベントを待機する。
  • microsec はミリ秒単位に変換され、500マイクロ秒未満の場合は1ミリ秒として扱われる。

アルゴリズムまたは振る舞い

1. pgwin32_signal_event が未初期化（NULL）の場合、SleepEx を呼び出し、割り込み不可能なスリープを行う。 2. pgwin32_signal_event が初期化済みの場合、WaitForSingleObject を使用して指定時間待機する。 3. 待機中にシグナルイベントが通知された場合（WAIT_OBJECT_0）:

  • pgwin32_dispatch_queued_signals() を呼び出し、キューイングされたシグナルを処理する。
  • errno を EINTR に設定して復帰する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/auth_delay/auth_delay.c
  • src/backend/access/nbtree/nbtpage.c
  • src/backend/access/transam/xlog.c
  • src/backend/access/transam/xlogutils.c
  • src/backend/commands/vacuum.c
  • src/backend/libpq/pqcomm.c
  • src/backend/port/win32/socket.c
  • **呼び出し先**:
  • SleepEx (Windows API)
  • WaitForSingleObject (Windows API)
  • pgwin32_dispatch_queued_signals

関連データ構造

  • pgwin32_signal_event: シグナル到着を通知するためのWindowsイベントオブジェクト。

失敗モードと境界条件

  • pgwin32_signal_event が初期化される前の呼び出しでは、シグナルによる割り込みは発生しない。
  • 待機時間が500マイクロ秒未満の場合、最小待機時間は1ミリ秒となる。

保守担当者向けメモ

  • 本関数はWindows環境におけるPOSIXシグナルエミュレーションの待機ポイントとして機能する。
  • errno = EINTR の設定は、呼び出し元がシグナルによる中断を検知できるようにするためのものである。
