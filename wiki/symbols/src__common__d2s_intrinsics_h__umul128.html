umul128

定義

  • 種別: function
  • 定義箇所: src/common/d2s_intrinsics.h:41

目的

2つの64ビット整数の積を計算し、その結果を128ビット（上位64ビットと下位64ビット）として取得する。Ryuアルゴリズムにおける浮動小数点数から文字列への変換処理をサポートする。

シグネチャまたは形状

static inline uint64 umul128(const uint64 a, const uint64 b, uint64 *const productHi)

入力と出力

  • **入力**:
  • a (uint64): 乗数1
  • b (uint64): 乗数2
  • productHi (uint64*): 積の上位64ビットを格納するためのポインタ
  • **出力**:
  • 戻り値 (uint64): 積の下位64ビット

前提条件と不変条件

  • productHi は有効なメモリ領域を指している必要がある。
  • 本関数は intrin.h に依存しており、環境に応じてハードウェア命令またはソフトウェアエミュレーションが選択される。

アルゴリズムまたは振る舞い

  • 内部的に _umul128 を呼び出すラッパー関数として動作する。
  • 64ビット環境ではCPUのネイティブ命令を利用し、それ以外では環境に応じた実装へ抽象化される。

呼び出し元と呼び出し先

  • **呼び出し元**: src/common/d2s.c 内の複数の箇所で、浮動小数点変換ロジックの一部として呼び出されている。
  • **呼び出し先**: _umul128（Windows環境等の組み込み関数）。

関連データ構造

  • uint64: 64ビット符号なし整数型。

失敗モードと境界条件

  • productHi に NULL を渡した場合の挙動は未定義（呼び出し元に依存）。
  • オーバーフローは発生しない（128ビットの結果として正確に保持されるため）。

保守担当者向けメモ

  • 本関数は d2s.c の浮動小数点変換処理の基盤である。
  • 32ビット環境等でパフォーマンス上の問題が発生した場合は、HAS_64_BIT_INTRINSICS マクロによる分岐実装の最適化を検討すること。
