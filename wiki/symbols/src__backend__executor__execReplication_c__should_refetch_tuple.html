should_refetch_tuple

定義

  • 種別: function
  • 定義箇所: src/backend/executor/execReplication.c:133

目的

論理レプリケーションにおけるテーブル行のロック操作において、同時実行による競合が発生した際に、再試行が必要かどうかを判定し、ログを出力する。

シグネチャまたは形状

static bool should_refetch_tuple(TM_Result res, TM_FailureData *tmfd)

入力と出力

  • **入力**:
  • res: table_tuple_lock から返された TM_Result 型のステータス。
  • tmfd: 失敗時の詳細情報を含む TM_FailureData 構造体へのポインタ。
  • **出力**:
  • bool: 再試行が必要な場合は true、それ以外は false を返す。

前提条件と不変条件

  • table_tuple_lock の実行結果に基づき動作する。
  • TM_Ok の場合は再試行不要（false）とみなす。
  • TM_Updated または TM_Deleted の場合は、シリアライゼーション失敗としてログを出力し、再試行を要求する（true）。

アルゴリズムまたは振る舞い

1. res の値を評価する。 2. TM_Updated の場合:

  • tmfd->ctid がパーティション移動を示しているか確認し、適切なメッセージでログ出力する。
  • refetch を true に設定する。

    • 3. TM_Deleted の場合:

  • 同時削除が発生した旨をログ出力し、refetch を true に設定する。

    • 4. TM_Invisible の場合:

  • 予期せぬ状態として ERROR を発生させる。

    • 5. その他の値の場合:

  • 未定義のステータスとして ERROR を発生させる。

    • 6. 最終的に refetch の値を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/execReplication.c 内の行ロック処理を行う箇所（行 266, 424, 511）。
  • **呼び出し先**:
  • ereport (ログ出力)
  • elog (エラー報告)
  • ItemPointerIndicatesMovedPartitions (マクロ)

関連データ構造

  • TM_Result: テーブルロック操作の結果を示す列挙型。
  • TM_FailureData: ロック失敗時の詳細情報（ctid等）を保持する構造体。

失敗モードと境界条件

  • TM_Invisible: 存在しない、あるいは可視でないタプルをロックしようとした場合に ERROR となる。
  • 未定義の TM_Result 値: default ケースで ERROR となる。

保守担当者向けメモ

  • 本関数は goto retry を含む再試行ループ内で呼び出されることを前提としている。
  • TM_Updated および TM_Deleted 時のログ出力内容は、将来的に改善の余地がある（ソースコード内の XXX コメント参照）。
