PGContextVisibility

定義

  • 種別: enum
  • 定義箇所: src/interfaces/libpq/libpq-fe.h:162

目的

エラーメッセージに含まれる CONTEXT フィールドの表示制御レベルを定義する列挙型です。

シグネチャまたは形状

typedef enum
{
    PQSHOW_CONTEXT_NEVER,   /* CONTEXTフィールドを非表示 */
    PQSHOW_CONTEXT_ERRORS,  /* エラー時のみCONTEXTを表示（デフォルト） */
    PQSHOW_CONTEXT_ALWAYS   /* 常にCONTEXTを表示 */
} PGContextVisibility;

入力と出力

  • 入力: PQsetErrorContextVisibility 関数において、設定値として本列挙型の値を受け取ります。
  • 出力: PQsetErrorContextVisibility 関数は、変更前の設定値を本列挙型として返します。

前提条件と不変条件

  • libpq の内部状態として保持され、エラー発生時のメッセージ構築時に参照されます。
  • 列挙型の値は追加される可能性がありますが、既存の定数の変更や削除は行われません。

アルゴリズムまたは振る舞い

  • PQSHOW_CONTEXT_NEVER: サーバーから CONTEXT 情報が提供されても、クライアント側のエラー出力には含めません。
  • PQSHOW_CONTEXT_ERRORS: エラー発生時のみ CONTEXT を出力します。これがデフォルトの挙動です。
  • PQSHOW_CONTEXT_ALWAYS: エラーの有無に関わらず、利用可能な場合は常に CONTEXT を出力します。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/psql/settings.h にて設定保持用として参照されています。
  • 呼び出し先: PQsetErrorContextVisibility 関数（src/interfaces/libpq/fe-connect.c）が本型を引数および戻り値として使用します。
  • その他: fe-exec.c および fe-protocol3.c の内部処理で参照されています。

関連データ構造

  • PGconn: 接続ごとの設定として本型を保持します（libpq-int.h 内の show_context メンバ）。

失敗モードと境界条件

  • 該当する情報はありません。

保守担当者向けメモ

  • 本型は libpq の公開 API の一部です。不透明ポインタ PGconn を介して設定を変更する際は、必ず PQsetErrorContextVisibility を使用してください。
