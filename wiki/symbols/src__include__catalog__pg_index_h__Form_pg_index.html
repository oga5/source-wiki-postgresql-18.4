Form_pg_index

定義

  • 種別: typedef
  • 定義箇所: src/include/catalog/pg_index.h:70

目的

システムカタログ pg_index の各行データ（FormData_pg_index）を指すためのポインタ型です。データベース内のインデックスに関するメタデータへのアクセスを提供します。

シグネチャまたは形状

typedef FormData_pg_index *Form_pg_index;

入力と出力

  • **入力**: HeapTuple から抽出されたデータ（GETSTRUCT マクロ等を使用）。
  • **出力**: FormData_pg_index 構造体の各フィールド（インデックスの構成列、一意性フラグ、有効性など）へのアクセス。

前提条件と不変条件

  • GETSTRUCT マクロ等を使用して、有効な HeapTuple からキャストされる必要があります。
  • 構造体内の indkey フィールドが 0（InvalidAttrNumber）の場合、その列は indexprs に格納された式として扱われます。

アルゴリズムまたは振る舞い

本シンボル自体は型定義であり、実行時の振る舞いは持ちません。実体である FormData_pg_index は、インデックスの作成、更新、およびプランナによる最適化の過程で、カタログデータの読み書きに使用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • contrib/dblink/dblink.c
  • contrib/tcn/tcn.c
  • src/backend/access/index/genam.c
  • src/backend/catalog/index.c
  • **呼び出し先**:
  • FormData_pg_index（構造体定義）

関連データ構造

  • FormData_pg_index: pg_index テーブルの物理的な行構造体。
  • HeapTuple: カタログ行を保持するデータ構造。

失敗モードと境界条件

  • 不適切なメモリ領域をキャストした場合、構造体フィールドへのアクセス時に不正なメモリ参照が発生します。
  • indkey や indclass などの配列フィールドを扱う際、インデックスの定義数を超えたアクセスを行わないよう注意が必要です。

保守担当者向けメモ

  • 本シンボルは pg_index カタログのスキーマ変更に直接依存します。src/include/catalog/pg_index.h の定義が変更された場合、この型を通じてアクセスしているすべての箇所で整合性を確認してください。
  • 可変長フィールド（CATALOG_VARLEN）が含まれるため、行のシリアライズ・デシリアライズ時には heap_form_tuple 等の適切な関数を経由していることを確認してください。
