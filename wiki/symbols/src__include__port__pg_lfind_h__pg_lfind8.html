pg_lfind8

定義

  • 種別: function
  • 定義箇所: src/include/port/pg_lfind.h:25

目的

uint8 型の配列から指定された値（key）を高速に検索します。SIMD 命令を活用して並列比較を行い、効率的な探索を実現します。

シグネチャまたは形状

static inline bool pg_lfind8(uint8 key, uint8 *base, uint32 nelem)

入力と出力

  • **入力**:
  • key: 検索対象の uint8 値。
  • base: 検索対象となる uint8 配列の先頭ポインタ。
  • nelem: 配列内の要素数。
  • **出力**:
  • bool: 配列内に key が存在する場合は true、存在しない場合は false を返します。

前提条件と不変条件

  • base は少なくとも nelem 個の要素を持つ有効なメモリ領域を指している必要があります。
  • SIMD 命令セット（Vector8）のサイズは 2 のべき乗であることを前提としています。

アルゴリズムまたは振る舞い

1. **SIMD 探索フェーズ**: 配列の要素数 nelem を Vector8 のサイズで切り捨てた範囲まで、SIMD 命令を用いて並列比較を行います。vector8_load でチャンクを読み込み、vector8_has で key が含まれているかを確認します。 2. **逐次探索フェーズ**: SIMD 処理でカバーしきれなかった残りの要素（tail_idx から nelem まで）に対し、単純なループによる逐次比較を行います。 3. **終了**: どちらかのフェーズで key が見つかった時点で true を返し、最後まで見つからなければ false を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/common/jsonapi.c
  • src/test/modules/test_lfind/test_lfind.c
  • **呼び出し先**:
  • vector8_load
  • vector8_has

    • (いずれも port/simd.h で定義される抽象化関数)

関連データ構造

  • Vector8: SIMD 演算に使用されるベクトル型。

失敗モードと境界条件

  • nelem が 0 の場合、ループは実行されず即座に false を返します。
  • nelem が Vector8 のサイズ未満の場合、SIMD 探索フェーズはスキップされ、逐次探索のみが行われます。

保守担当者向けメモ

  • 本関数は static inline で定義されており、ヘッダーファイル内で完結しています。
  • SIMD 命令の利用可否は port/simd.h の抽象化レイヤーに依存しています。
  • 境界処理において、SIMD 処理と逐次処理の重複がないよう tail_idx の計算（nelem & ~(sizeof(Vector8) - 1)）が設計されています。
