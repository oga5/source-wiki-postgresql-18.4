macaddr8_recv

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/mac8.c:253

目的

外部バイナリ形式（EUI-48 または EUI-64）のデータストリームを読み込み、内部表現である macaddr8 型へ変換します。

シグネチャまたは形状

Datum macaddr8_recv(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して StringInfo バッファを受け取ります。
  • **出力**: macaddr8 構造体へのポインタを Datum 型として返します。

前提条件と不変条件

  • 入力バッファは libpq/pqformat.h のプロトコルに従ったバイナリ形式である必要があります。
  • 6バイトの入力が与えられた場合、EUI-64 形式への拡張（0xFF, 0xFE の挿入）が自動的に行われます。

アルゴリズムまたは振る舞い

1. palloc0 を使用して macaddr8 構造体用のメモリを確保します。 2. バッファから最初の3バイトを読み込み、addr->a, addr->b, addr->c に格納します。 3. バッファの長さが6バイトの場合、EUI-48 と判断し、addr->d に 0xFF、addr->e に 0xFE を設定します。 4. バッファの長さが6バイト以外の場合、次の2バイトを読み込み addr->d, addr->e に格納します。 5. 残りの3バイトを読み込み、addr->f, addr->g, addr->h に格納します。 6. 構築した macaddr8 構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: ソースコード内の明示的な呼び出し元は特定されていませんが、PostgreSQL の型システムを通じて SQL レベルのバイナリ入力処理から呼び出されます。
  • **呼び出し先**:
  • palloc0: メモリ確保。
  • pq_getmsgbyte: バッファからのバイト読み込み。
  • PG_RETURN_MACADDR8_P: 結果の返却。

関連データ構造

  • macaddr8: 8バイトの MAC アドレスを保持する構造体。
  • StringInfo: バイナリデータ送受信用のバッファ構造体。

失敗モードと境界条件

  • バッファから読み込むべきバイト数が不足している場合、pq_getmsgbyte がエラーを発生させる可能性があります。
  • 入力長が6バイト以外で、かつ必要なバイト数がバッファ内に存在しない場合の挙動は pq_getmsgbyte の実装に依存します。

保守担当者向けメモ

  • 本関数は macaddr8 型のバイナリ入力インターフェースです。EUI-48 から EUI-64 への変換ロジックがハードコードされているため、将来的に他の拡張形式をサポートする場合は条件分岐の修正が必要です。
