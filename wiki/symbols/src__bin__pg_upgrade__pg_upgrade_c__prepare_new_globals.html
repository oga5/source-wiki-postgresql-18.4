prepare_new_globals

定義

  • 種別: function
  • 定義箇所: src/bin/pg_upgrade/pg_upgrade.c:549

目的

アップグレード後の新クラスタにおいて、グローバルオブジェクト（ロールやテーブルスペース）を復元し、システムテーブルの凍結トランザクションID（frozenxids）を初期化する。

シグネチャまたは形状

static void prepare_new_globals(void)

入力と出力

  • 入力: なし（グローバル変数 new_cluster および log_opts を参照）
  • 出力: なし（副作用として新クラスタへのデータ書き込みおよびログ出力を行う）

前提条件と不変条件

  • set_frozenxids(false) が呼び出される時点で、新クラスタは initdb 直後の状態である必要がある。
  • グローバルオブジェクトのダンプファイル（GLOBALS_DUMP_FILE）が指定されたパスに存在すること。

アルゴリズムまたは振る舞い

1. set_frozenxids(false) を呼び出し、initdb によって作成されたテーブルの frozenxids を設定する。 2. ユーザーへの進捗通知として「Restoring global objects in the new cluster」を表示する。 3. psql を実行し、ダンプファイルからグローバルオブジェクトを新クラスタに復元する。 4. check_ok() を呼び出し、コマンドの実行結果が正常であることを確認する。

呼び出し元と呼び出し先

  • 呼び出し元: main (src/bin/pg_upgrade/pg_upgrade.c:168)
  • 呼び出し先:
  • set_frozenxids
  • prep_status
  • exec_prog
  • cluster_conn_opts
  • check_ok

関連データ構造

  • new_cluster: 新クラスタの接続情報やパスを保持する構造体。
  • log_opts: ログ出力先やダンプディレクトリの情報を保持する構造体。

失敗モードと境界条件

  • exec_prog が失敗した場合、check_ok() 内でエラーが報告され、プロセスが終了する可能性がある。
  • ダンプファイルが存在しない、または権限不足により読み込めない場合、psql の実行が失敗する。

保守担当者向けメモ

  • 本関数は、スキーマ復元（create_new_objects）の前に実行される必要がある。
  • psql 実行時の引数は EXEC_PSQL_ARGS マクロによって制御されているため、接続オプションの変更が必要な場合はここを確認すること。
