ConnParams

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_dump/pg_backup.h:83

目的

PostgreSQL のバックアップおよびリストア処理において、データベース接続に必要なパラメータを保持・管理するための構造体です。

シグネチャまたは形状

typedef struct _connParams
{
    char       *dbname;          /* 接続文字列を含む可能性があるデータベース名 */
    char       *pgport;          /* ポート番号 */
    char       *pghost;          /* ホスト名 */
    char       *username;        /* ユーザー名 */
    trivalue    promptPassword;  /* パスワード入力の要否 */
    char       *override_dbname; /* コマンドライン引数から取得したdbnameを上書きする値 */
} ConnParams;

入力と出力

本構造体は、コマンドライン引数や設定ファイルから読み取った接続情報を格納する入力コンテナとして機能します。出力は、接続処理を行う関数（libpq を利用する関数群）へ渡されるパラメータセットとなります。

前提条件と不変条件

  • dbname フィールドには、単純なデータベース名だけでなく、接続文字列（connstring）が含まれる可能性があります。
  • override_dbname が NULL でない場合、接続先のデータベース名は dbname ではなく override_dbname が優先されます。

アルゴリズムまたは振る舞い

本構造体自体はデータの保持のみを行い、ロジックは持ちません。各ツール（pg_dump, pg_restore, clusterdb 等）が本構造体をインスタンス化し、接続パラメータを格納した後に、接続管理関数へポインタを渡すことで利用されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照・利用）**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/pg_dump/pg_backup.h
  • src/bin/pg_dump/pg_backup_db.c
  • src/bin/scripts/clusterdb.c
  • src/bin/scripts/createdb.c
  • **呼び出し先**:
  • 構造体定義のため、直接的な呼び出し先はありません。本構造体を受け取る関数（例: cluster_one_database 等）が、libpq の接続関数を呼び出す際に利用します。

関連データ構造

  • trivalue: パスワードプロンプトの制御に使用される列挙型。

失敗モードと境界条件

  • dbname に接続文字列を指定した場合、override_dbname による上書きはデータベース名のみに限定され、接続文字列内の他のパラメータ（ホストやポート等）には影響しません。

保守担当者向けメモ

  • 本構造体は pg_dump だけでなく、pg_scripts 配下のツールでも共通利用されています。フィールドの追加や変更を行う際は、依存する複数のツールへの影響を考慮してください。
