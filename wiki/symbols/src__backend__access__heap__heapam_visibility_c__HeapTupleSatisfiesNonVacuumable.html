HeapTupleSatisfiesNonVacuumable

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_visibility.c:1437

目的

タプルが「VACUUMによって削除可能ではない」状態にあるかを判定します。主に、特定のトランザクションスナップショットの観点から、タプルが物理的に削除可能な状態（HEAPTUPLE_DEAD）に達しているかを判断するために使用されます。

シグネチャまたは形状

static bool HeapTupleSatisfiesNonVacuumable(HeapTuple htup, Snapshot snapshot, Buffer buffer)

入力と出力

  • **入力**:
  • htup: 判定対象のヒープタプル。
  • snapshot: 可視性判定に使用するスナップショット。
  • buffer: タプルが格納されているバッファ。
  • **出力**:
  • bool: タプルが削除可能でない（生存しているとみなされる）場合に true、削除可能（HEAPTUPLE_DEAD）と判定された場合に false を返します。

前提条件と不変条件

  • 呼び出し元は、対象のバッファに対して少なくとも共有ロック（Shared Lock）を保持している必要があります。
  • HeapTupleSatisfiesVacuumHorizon が返す dead_after トランザクションIDは、HEAPTUPLE_RECENTLY_DEAD の場合にのみ有効（TransactionIdIsValid）です。

アルゴリズムまたは振る舞い

1. HeapTupleSatisfiesVacuumHorizon を呼び出し、タプルの基本的な可視性結果（res）と、削除可能となるトランザクションID（dead_after）を取得します。 2. 結果が HEAPTUPLE_RECENTLY_DEAD の場合、GlobalVisTestIsRemovableXid を使用して、指定されたスナップショットの観点からそのタプルが削除可能かを確認します。削除可能であれば res を HEAPTUPLE_DEAD に更新します。 3. 結果が HEAPTUPLE_DEAD ではない場合（true）を返します。

呼び出し元と呼び出し先

  • **呼び出し先**:
  • HeapTupleSatisfiesVacuumHorizon
  • GlobalVisTestIsRemovableXid
  • **呼び出し元**:
  • ソースコード内の参照情報に基づくと、src/backend/access/heap/heapam_visibility.c 内の他の可視性判定ロジックから呼び出されています。具体的な上位レイヤーの関数名はソースコードの断片からは特定できません。

関連データ構造

  • HeapTuple: 判定対象のタプル構造体。
  • Snapshot: 可視性判定の基準となるスナップショット構造体。
  • Buffer: タプルが配置されているバッファ。
  • HTSV_Result: 可視性判定結果を示す列挙型。

失敗モードと境界条件

  • HeapTupleSatisfiesVacuumHorizon が HEAPTUPLE_RECENTLY_DEAD 以外を返した場合、dead_after は InvalidTransactionId である必要があります（アサーションで検証）。
  • HEAPTUPLE_RECENTLY_DEAD の場合は、dead_after が有効なトランザクションIDであることが期待されます。

保守担当者向けメモ

  • 本関数は、VACUUM処理がタプルを安全に削除できるかを判断する際の補助的な役割を果たします。
  • GlobalVisTestIsRemovableXid による判定は、スナップショットの vistest メンバーに依存しているため、スナップショットの整合性に注意してください。
