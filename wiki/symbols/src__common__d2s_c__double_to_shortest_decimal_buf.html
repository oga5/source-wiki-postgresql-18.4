double_to_shortest_decimal_buf

定義

  • 種別: function
  • 定義箇所: src/common/d2s.c:1052

目的

IEEE 754 倍精度浮動小数点数（double）を、精度を損なわない最短の 10 進数文字列へ変換し、指定されたバッファに書き込みます。

シグネチャまたは形状

int double_to_shortest_decimal_buf(double f, char *result)

入力と出力

  • **入力**:
  • f: 変換対象の double 型浮動小数点数。
  • result: 変換後の文字列を格納するためのバッファ（呼び出し元で十分な領域を確保する必要がある）。
  • **出力**:
  • 戻り値: 書き込まれた文字列の長さ（終端文字を含まない）。

前提条件と不変条件

  • result バッファは、変換結果を格納するために十分なサイズ（DOUBLE_SHORTEST_DECIMAL_LEN 以上）であること。
  • 変換後の文字列は、この関数によってヌル終端（\0）されます。

アルゴリズムまたは振る舞い

1. 内部関数 double_to_shortest_decimal_bufn を呼び出し、f を 10 進数文字列に変換して result に書き込みます。 2. 戻り値のインデックスが DOUBLE_SHORTEST_DECIMAL_LEN 未満であることを Assert で検証します。 3. result の末尾にヌル終端文字を付与します。 4. 書き込んだ文字数を返します。

呼び出し元と呼び出し先

  • **呼び出し先**: double_to_shortest_decimal_bufn
  • **呼び出し元**:
  • src/backend/nodes/outfuncs.c:221
  • src/backend/utils/adt/float.c:544
  • src/common/d2s.c:1074

関連データ構造

  • DOUBLE_SHORTEST_DECIMAL_LEN: バッファサイズの制約に使用される定数。

失敗モードと境界条件

  • Assert が有効なビルドにおいて、変換結果が DOUBLE_SHORTEST_DECIMAL_LEN を超える場合はアサーション失敗となります。

保守担当者向けメモ

  • 本関数は double_to_shortest_decimal_bufn のラッパーであり、文字列の終端処理を保証する役割を持ちます。
  • 変換ロジックの詳細は d2d 関数および to_chars 関数に依存しています。変更時は IEEE 754 の丸め仕様との整合性に注意してください。
