PredicateLockShmemInit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/lmgr/predicate.c:1144

目的

PostgreSQL の直列化可能スナップショット分離（SSI）に必要な共有メモリ上のデータ構造（ハッシュテーブルおよび管理リスト）を初期化する。

シグネチャまたは形状

void PredicateLockShmemInit(void)

入力と出力

  • 入力: なし
  • 出力: なし（共有メモリ領域の確保とグローバル変数へのポインタ設定を行う）

前提条件と不変条件

  • Postmaster プロセス起動時に一度だけ実行される必要がある。
  • 共有メモリのサイズ計算は PredicateLockShmemSize と整合している必要がある。
  • IsUnderPostmaster が偽の場合、ハッシュテーブルの初期化およびダミーエントリの作成が行われる。

アルゴリズムまたは振る舞い

1. **述語ロック対象ハッシュテーブル (PredicateLockTargetHash)**: PREDICATELOCKTARGET 構造体用のハッシュテーブルを確保し、ページ分割等の際に備えてダミーエントリを登録する。 2. **述語ロックハッシュテーブル (PredicateLockHash)**: PREDICATELOCK 構造体用のハッシュテーブルを確保する。 3. **直列化可能トランザクションリスト (PredXact)**: SERIALIZABLEXACT 構造体の配列を確保し、availableList（空きリスト）と activeList（アクティブリスト）を初期化する。また、古いコミット済みトランザクションを追跡するための OldCommittedSxact を作成する。 4. **XID ハッシュテーブル (SerializableXidHash)**: SERIALIZABLEXID 構造体用のハッシュテーブルを確保する。 5. **RW 競合プール (RWConflictPool)**: トランザクション間の読み書き競合を追跡するためのリスト構造を確保・初期化する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/storage/ipc/ipci.c:305（CreateSharedMemoryAndSemaphores 内から呼び出される）
  • 呼び出し先: ShmemInitHash, ShmemInitStruct, hash_search, dlist_init, LWLockInitialize, CreatePredXact 等

関連データ構造

  • PredicateLockTargetHash
  • PredicateLockHash
  • PredXact
  • SerializableXidHash
  • RWConflictPool

失敗モードと境界条件

  • 共有メモリ不足時には ShmemInitHash や ShmemInitStruct がエラーを発生させる。
  • IsUnderPostmaster が真の場合、一部の初期化処理（ダミーエントリの挿入など）はスキップされる。

保守担当者向けメモ

  • 本関数での計算ロジックを変更した場合は、必ず PredicateLockShmemSize も更新すること。
  • max_table_size の算出には MaxBackends や max_prepared_xacts が関与しており、これらはシステム全体のメモリ使用量に直結する。
