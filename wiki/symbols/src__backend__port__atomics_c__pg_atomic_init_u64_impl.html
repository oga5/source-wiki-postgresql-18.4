pg_atomic_init_u64_impl

定義

  • 種別: function
  • 定義箇所: src/backend/port/atomics.c:23

目的

ハードウェアによるネイティブな 64 ビットアトミック操作がサポートされていない環境において、スピンロックを用いた 64 ビットアトミック変数の初期化を行う。

シグネチャまたは形状

void pg_atomic_init_u64_impl(volatile pg_atomic_uint64 *ptr, uint64 val_)

入力と出力

  • ptr: 初期化対象となる pg_atomic_uint64 構造体へのポインタ。
  • val_: 変数に設定する初期値。
  • 戻り値: なし。

前提条件と不変条件

  • ptr->sema のサイズが slock_t 以上である必要がある（コンパイル時に StaticAssertDecl で検証）。
  • 本関数は、システムが PG_HAVE_ATOMIC_U64_SIMULATION を定義している環境でのみ使用される。

アルゴリズムまたは振る舞い

1. ptr->sema に対して SpinLockInit を実行し、スピンロックを初期化する。 2. ptr->value に引数 val_ を代入する。

呼び出し元と呼び出し先

  • 呼び出し元: src/include/port/atomics.h および src/include/port/atomics/generic.h からの参照が確認されている。
  • 呼び出し先: SpinLockInit（storage/spin.h で定義）。

関連データ構造

  • pg_atomic_uint64: 64 ビットアトミック操作のための構造体。スピンロック（sema）と値（value）を保持する。
  • slock_t: スピンロックの実装型。

失敗モードと境界条件

  • ptr が指すメモリ領域が不正な場合、SpinLockInit の実行時に未定義の動作を引き起こす可能性がある。
  • StaticAssertDecl により、構造体のサイズ不整合が検出された場合はコンパイルエラーとなる。

保守担当者向けメモ

  • 本関数は、ネイティブなアトミック命令が利用できない環境でのフォールバック実装である。
  • パフォーマンスが重要なパスでは、プラットフォームがネイティブなアトミック操作をサポートしているか確認すること。
