print_double

定義

  • 種別: function
  • 定義箇所: src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c:33

目的

プラットフォーム間で一貫した浮動小数点数の出力を保証するためのヘルパー関数です。特にWindows環境における指数表記の差異を吸収し、テスト結果の安定性を担保します。

シグネチャまたは形状

static void print_double(double x)

入力と出力

  • **入力**: double x （出力対象の浮動小数点数）
  • **出力**: なし（標準出力 stdout へ直接書き出されます）

前提条件と不変条件

  • WIN32 環境下では、sprintf で生成された文字列の指数部が3桁（例: e+001）である場合に、これを2桁（例: e+01）に変換する処理が行われます。
  • 非 WIN32 環境では、標準的な printf("%g", x) と同等の出力を行います。

アルゴリズムまたは振る舞い

1. WIN32 定義時:

  • sprintf を使用して double 値を文字列 convert に変換します。
  • 文字列の末尾付近を走査し、指数表記が e+00 のような3桁形式であるかを確認します。
  • 該当する場合、指数部の0を一つ削除して2桁に詰め、文字列を再構成します。
  • 最終的に printf で出力します。

    • 2. それ以外:

  • printf("%g", x) を直接実行します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test2.c
  • src/interfaces/ecpg/test/expected/pgtypeslib-num_test.c
  • src/interfaces/ecpg/test/expected/compat_informix-dec_test.c
  • **呼び出し先**:
  • 標準ライブラリ関数: sprintf, strlen, printf

関連データ構造

  • 特になし（プリミティブな double 型および char 配列を使用）。

失敗モードと境界条件

  • convert バッファ（128バイト）を超える長さの文字列が生成された場合、sprintf によるバッファオーバーフローの可能性があります。
  • 指数表記を含まない数値や、指数部が既に2桁である場合は変換処理をスキップします。

保守担当者向けメモ

  • 本関数は src/interfaces/ecpg/test/printf_hack.h でも参照されており、複数のテストファイルで共通のハックとして利用されています。
  • Windows特有の指数表記の挙動が将来的に変更された場合、このロジックの修正が必要になる可能性があります。
