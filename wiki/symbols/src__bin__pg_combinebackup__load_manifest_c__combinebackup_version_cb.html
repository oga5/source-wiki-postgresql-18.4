combinebackup_version_cb

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/load_manifest.c:242

目的

バックアップマニフェストファイルのバージョンを検証し、増分バックアップをサポートしない古いバージョン（バージョン1）が指定された場合に処理を中断する。

シグネチャまたは形状

static void combinebackup_version_cb(JsonManifestParseContext *context, int manifest_version)

入力と出力

  • context: パース処理の状態を保持する JsonManifestParseContext へのポインタ。
  • manifest_version: マニフェストファイルから読み取られたバージョン番号。
  • 戻り値: なし（void）。

前提条件と不変条件

  • 本関数は JsonManifestParseContext のコールバックとして呼び出されることを前提とする。
  • バージョン2以降のマニフェストのみが、増分バックアップ処理において有効とみなされる。

アルゴリズムまたは振る舞い

1. 引数 manifest_version が 1 であるかを確認する。 2. 1 であれば、pg_fatal を呼び出し、増分バックアップがサポートされていない旨のエラーメッセージを出力してプログラムを終了させる。 3. 1 以外の場合は何も行わず正常に終了する。

呼び出し元と呼び出し先

  • 呼び出し元: マニフェストパース処理の初期化過程において、JsonManifestParseContext 構造体の version_cb フィールドに本関数が代入され、パースロジックから間接的に呼び出される（src/bin/pg_combinebackup/load_manifest.c:145）。
  • 呼び出し先: pg_fatal（エラー発生時）。

関連データ構造

  • JsonManifestParseContext: マニフェスト解析のコンテキストを保持する構造体。

失敗モードと境界条件

  • manifest_version が 1 の場合、pg_fatal によりプロセスが強制終了される。

保守担当者向けメモ

  • 本関数は増分バックアップの互換性チェックを担っている。将来的にバージョン3以降が導入された場合、必要に応じて条件分岐の更新を検討すること。
