ginrescan

定義

  • 種別: function
  • 定義箇所: src/backend/access/gin/ginscan.c:492

目的

GINインデックススキャンの状態をリセットし、新しい検索条件でスキャンを再開するための準備を行う。

シグネチャまたは形状

void ginrescan(IndexScanDesc scan, ScanKey scankey, int nscankeys, ScanKey orderbys, int norderbys)

入力と出力

  • **入力**:
  • scan: 現在のスキャン状態を保持する IndexScanDesc。
  • scankey: 新しい検索条件（ScanKey）。
  • nscankeys: 検索キーの数。
  • orderbys: ソート順序指定（本関数内では未使用）。
  • norderbys: ソート順序指定の数（本関数内では未使用）。
  • **出力**: なし（副作用として scan 構造体内部の状態を更新）。

前提条件と不変条件

  • scan->opaque が GinScanOpaque 型としてキャスト可能であること。
  • ginFreeScanKeys により、既存の検索キーに関連するメモリが解放されること。

アルゴリズムまたは振る舞い

1. scan->opaque から GinScanOpaque 構造体を取得する。 2. ginFreeScanKeys を呼び出し、既存の検索キーに関連するリソースを解放する。 3. 新しい scankey が提供され、かつ scan->numberOfKeys が正の値である場合、memcpy を使用して scan->keyData を新しい検索条件で上書きする。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/gin/ginutil.c: amroutine->amrescan に関数ポインタとして登録されている。
  • **呼び出し先**:
  • ginFreeScanKeys: スキャンキーの解放に使用。

関連データ構造

  • IndexScanDesc: インデックススキャンの共通状態。
  • GinScanOpaque: GIN固有のスキャン状態（scan->opaque に格納）。
  • ScanKeyData: 検索条件のデータ構造。

失敗モードと境界条件

  • scankey が NULL の場合、検索条件のコピーはスキップされる。
  • scan->numberOfKeys が 0 以下の場合、検索条件の更新は行われない。

保守担当者向けメモ

  • 本関数は amrescan インターフェースの一部として呼び出される。
  • orderbys および norderbys 引数は現在の実装では使用されていない。
  • ginFreeScanKeys を呼び出すことで、GinScanOpaque 内のメモリコンテキスト（keyCtx）がリセットされ、メモリリークを防ぐ設計となっている。
