assign_locale_time

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/pg_locale.c:395

目的

LC_TIME ロケール設定が変更されたことをシステムに通知し、キャッシュされた時刻関連のロケール情報を無効化します。

シグネチャまたは形状

void assign_locale_time(const char *newval, void *extra)

入力と出力

  • newval: 設定される新しいロケール名（本関数内では使用されません）。
  • extra: 拡張データ（本関数内では使用されません）。
  • 戻り値: なし。

前提条件と不変条件

  • GUC（Grand Unified Configuration）のフック関数として呼び出されることを想定しています。

アルゴリズムまたは振る舞い

1. グローバル変数 CurrentLCTimeValid を false に設定します。これにより、次回時刻関連のロケール情報が必要になった際に、システムが最新のロケール設定を再読み込みするよう強制します。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/utils/misc/guc_tables.c 内の GUC 設定テーブルから、LC_TIME 設定変更時に呼び出されます。
  • 呼び出し先: なし。

関連データ構造

  • CurrentLCTimeValid: ロケール情報の有効性を保持するグローバルなフラグ変数。

失敗モードと境界条件

  • 特になし。本関数は常に成功し、副作用としてフラグの更新のみを行います。

保守担当者向けメモ

  • 本関数は LC_TIME の変更を検知するための軽量なトリガーです。ロケール情報の再構築自体は、このフラグを参照する別の関数（check_locale_time 等）によって行われます。
