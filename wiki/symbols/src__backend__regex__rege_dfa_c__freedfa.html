freedfa

定義

  • 種別: function
  • 定義箇所: src/backend/regex/rege_dfa.c:699

目的

DFA（決定性有限オートマトン）構造体に関連するメモリリソースを解放する。

シグネチャまたは形状

static void freedfa(struct dfa *d)

入力と出力

  • 入力: struct dfa *d（解放対象のDFA構造体へのポインタ）
  • 出力: なし

前提条件と不変条件

  • d が NULL でないこと。
  • d->arraysmalloced フラグが真の場合、ssets、statesarea、outsarea、incarea がそれぞれ動的に確保されている必要がある。
  • d->ismalloced フラグが真の場合、d 自体が動的に確保されている必要がある。

アルゴリズムまたは振る舞い

1. d->arraysmalloced が真であれば、DFA 内部の各配列（ssets, statesarea, outsarea, incarea）が NULL でないことを確認し、それぞれ FREE() を呼び出して解放する。 2. d->ismalloced が真であれば、d 自身を FREE() で解放する。

呼び出し元と呼び出し先

  • 呼び出し元: src/backend/regex/regexec.c 内の各関数（v->subdfas[i], v->ladfas[i], s, d の解放処理）および src/backend/regex/rege_dfa.c 内の内部処理。
  • 呼び出し先: FREE() マクロ（メモリ解放用）。

関連データ構造

  • struct dfa: DFA の実行状態を保持する主要構造体。

失敗モードと境界条件

  • d が NULL の場合、呼び出し元で適切にガードされていない限り、未定義の動作を引き起こす可能性がある。
  • 各ポインタが NULL でない場合にのみ FREE() が実行されるため、二重解放や不正なポインタ解放は回避される設計となっている。

保守担当者向けメモ

  • 本関数は regexec.c からも呼び出されるため、DFA のライフサイクル管理において重要な役割を果たす。
  • FREE() マクロの定義は regguts.h に依存しているため、メモリ管理ポリシーを変更する場合は注意すること。
