GETTEMP

定義

  • 種別: function
  • 定義箇所: src/port/mkdtemp.c:93

目的

テンプレート文字列に基づいて一意なファイル名またはディレクトリ名を生成し、必要に応じて作成または検証を行う内部補助関数です。

シグネチャまたは形状

static int GETTEMP(char *path, int *doopen, int domkdir)

入力と出力

  • path: 変更対象のテンプレート文字列（末尾に X を含む）。関数内で一意な名前に置換されます。
  • doopen: ファイル記述子を格納するポインタ（ファイル作成時）。NULL の場合はファイル作成を行いません。
  • domkdir: ディレクトリ作成を行うかどうかのフラグ。
  • 戻り値: 成功時は 1、失敗時は 0 を返します。

前提条件と不変条件

  • path は NULL であってはなりません。
  • テンプレート内の X は、プロセスIDおよび静的カウンタ xtra を用いて一意な文字に置換されます。
  • doopen が指定された場合、O_CREAT | O_EXCL | O_RDWR モードでファイル作成を試みます。

アルゴリズムまたは振る舞い

1. **テンプレート解析**: path 末尾の X をカウントし、プロセスIDと静的カウンタ xtra を用いて X を文字に置換します。 2. **ディレクトリ検証**: パス内のディレクトリ部分が実在し、かつディレクトリであることを stat(2) で確認します。 3. **作成試行**: doopen が有効なら open(2) を、domkdir が有効なら mkdir(2) を実行します。 4. **衝突回避**: EEXIST エラーが発生した場合、テンプレート内の文字をインクリメントして別の名前を生成し、成功するまで再試行します。 5. **状態更新**: 次回の呼び出しに備え、静的カウンタ xtra を更新します。

呼び出し元と呼び出し先

  • 呼び出し元: src/port/mkdtemp.c 内の mkdtemp 関数から GETTEMP(path, NULL, 1) として呼び出されます。
  • 呼び出し先: getpid(), stat(), open(), mkdir(), lstat(), isdigit() を呼び出します。

関連データ構造

  • struct stat: ファイル状態の確認に使用。
  • xtra[2]: 衝突回避のための静的カウンタ。

失敗モードと境界条件

  • テンプレート内に X が存在しない、またはパスが不正な場合、失敗する可能性があります。
  • EEXIST 以外のエラーが発生した場合、直ちに 0 を返します。
  • テンプレートの X が少ない場合、衝突回避の試行回数が制限され、処理が終了する可能性があります。

保守担当者向けメモ

  • 本関数は NetBSD 由来の実装であり、暗号学的依存関係を避けるため OpenBSD 版のアルゴリズムは採用されていません。
  • HAVE_MKDTEMP が定義されている環境では、本ファイルの実装は無視されます。
