ConnectionOption

定義

  • 種別: struct
  • 定義箇所: src/backend/foreign/foreign.c:565

目的

外部データラッパー（FDW）の接続オプションを定義し、各オプションがどのシステムカタログ（pg_foreign_data_wrapper、pg_foreign_server、pg_user_mapping 等）で使用可能かを管理するために使用されます。

シグネチャまたは形状

struct ConnectionOption
{
    const char *optname;    /* オプション名 */
    Oid         optcontext; /* オプションが許可されるカタログのOID */
};

入力と出力

  • 本構造体自体はデータ定義であり、関数ではありません。
  • optname はオプションの名称を示す文字列ポインタです。
  • optcontext は、そのオプションが適用可能なシステムカタログを識別する OID です。

前提条件と不変条件

  • optname は、libpq 等の接続文字列として有効な名称である必要があります。
  • optcontext は、PostgreSQL のシステムカタログ OID と一致している必要があります。

アルゴリズムまたは振る舞い

  • 本構造体は、libpq_conninfo_options 配列の要素として静的に定義され、外部サーバーやユーザーマッピングのオプション検証時に参照されます。
  • オプション検証ロジックにおいて、指定されたオプションが現在のコンテキスト（カタログ）で許可されているかを判定するために利用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/foreign/foreign.c 内のオプション検証関数（postgresql_fdw_validator 等）から参照されています。
  • **呼び出し先**: 情報なし。

関連データ構造

  • libpq_conninfo_options: ConnectionOption 型の静的配列。有効な接続オプションのリストを保持します。

失敗モードと境界条件

  • 構造体定義自体に失敗モードはありません。
  • 参照時に optcontext が不適切な場合、オプション検証プロセスで誤ったバリデーション結果を返す可能性があります。

保守担当者向けメモ

  • postgresql_fdw_validator が libpq の接続オプションをハードコードしているため、libpq の仕様変更に伴い、本構造体の配列定義を更新する必要があります。
  • 新しいオプションを追加する際は、適切な optcontext を指定し、該当するカタログでの利用が許可されているかを確認してください。
