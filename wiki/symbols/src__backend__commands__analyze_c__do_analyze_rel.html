do_analyze_rel

定義

  • 種別: function
  • 定義箇所: src/backend/commands/analyze.c:277

目的

指定されたリレーション（テーブル）に対して統計情報を生成・更新するメイン処理を実行します。継承ツリーの処理や、列ごとの統計計算、インデックスの解析を制御します。

シグネチャまたは形状

static void do_analyze_rel(Relation onerel, VacuumParams *params, List *va_cols, 
                           AcquireSampleRowsFunc acquirefunc, BlockNumber relpages, 
                           bool inh, bool in_outer_xact, int elevel);

入力と出力

  • **入力**:
  • onerel: 解析対象のリレーション。
  • params: VACUUM/ANALYZE の実行パラメータ。
  • va_cols: 解析対象列のリスト（指定がない場合は全列）。
  • acquirefunc: サンプル行取得用の関数ポインタ。
  • relpages: 対象リレーションのページ数。
  • inh: 継承ツリーを再帰的に処理するかどうか。
  • in_outer_xact: 外部トランザクション内での実行か。
  • elevel: エラー報告レベル。
  • **出力**: なし（副作用として pg_statistic カタログ等の更新を行う）。

前提条件と不変条件

  • anl_context メモリコンテキストが作成され、処理終了後に解放されることが想定されています。
  • 実行ユーザーはテーブル所有者の権限に切り替えられ、セキュリティ制限付き操作が適用されます。
  • システム属性は解析対象外です。

アルゴリズムまたは振る舞い

1. **初期化**: メモリコンテキストの作成、ユーザー権限の切り替え、検索パスの制限を行います。 2. **計測**: verbose 設定に応じて、I/O タイミングやリソース使用量の初期スナップショットを取得します。 3. **列選択**: va_cols が指定されていればその列を、なければ全列を examine_attribute で解析対象として特定します。 4. **インデックス準備**: 継承ツリーの解析でない場合、vac_open_indexes を呼び出し、インデックスの解析情報を準備します。 5. **後続処理**: 抽出されたサンプル行に基づき、各列の統計計算およびカタログ更新へと進みます（ソース抜粋範囲外）。

呼び出し元と呼び出し先

  • **呼び出し元**: analyze_rel（src/backend/commands/analyze.c 内）。
  • **呼び出し先**: examine_attribute, vac_open_indexes, BuildIndexInfo 等。

関連データ構造

  • VacAttrStats: 列ごとの統計情報を保持。
  • AnlIndexData: インデックスの解析用メタデータ。
  • VacuumParams: 実行オプション。

失敗モードと境界条件

  • 指定された列が存在しない場合、ERRCODE_UNDEFINED_COLUMN でエラー終了します。
  • 列が重複して指定された場合、ERRCODE_DUPLICATE_COLUMN でエラー終了します。
  • 継承ツリー解析時、親テーブルのインデックスは処理されません。

保守担当者向けメモ

  • anl_context によるメモリ管理を行っているため、関数内で割り当てたメモリの解放漏れに注意してください。
  • typanalyze 関数ポインタによりデータ型ごとの統計アルゴリズムが切り替わるため、型固有の挙動を修正する場合は examine_attribute 経由の呼び出しを確認してください。
