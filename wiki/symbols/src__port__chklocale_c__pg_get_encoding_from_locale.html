pg_get_encoding_from_locale

定義

  • 種別: function
  • 定義箇所: src/port/chklocale.c:300

目的

OS のロケール設定（LC_CTYPE）から、PostgreSQL が内部的に使用する文字エンコーディング識別子（pg_enc）を導出します。

シグネチャまたは形状

int pg_get_encoding_from_locale(const char *ctype, bool write_message)

入力と出力

  • **入力**:
  • ctype: ロケール文字列。NULL の場合は現在の環境設定（setlocale(LC_CTYPE, NULL)）が使用されます。
  • write_message: 認識できないエンコーディングに遭遇した際、警告メッセージを出力するかどうかを指定するフラグ。
  • **出力**:
  • 成功時: 対応する PostgreSQL エンコーディング定数（pg_enc）。
  • 失敗時: -1。

前提条件と不変条件

  • C または POSIX ロケールが指定された場合、常に PG_SQL_ASCII を返します。
  • 内部的に encoding_match_list を参照し、システム固有のコードセット名と照合します。

アルゴリズムまたは振る舞い

1. ctype が NULL の場合、現在のロケールを取得します。 2. C / POSIX ロケールであれば PG_SQL_ASCII を即座に返します。 3. プラットフォームに応じたコードセット名の抽出:

  • **非 Windows**: newlocale および nl_langinfo_l(CODESET, ...) を使用してコードセット名を取得します。
  • **Windows**: win32_get_codeset(ctype) を呼び出してコードページを解決します。

    • 4. 取得したコードセット名を encoding_match_list と大文字小文字を区別せずに照合します。 5. 一致した場合、対応するエンコーディングコードを返します。 6. 一致しない場合、プラットフォーム固有の特殊処理（macOS での空文字列に対する PG_UTF8 返却など）を経て、write_message が真であれば警告を出力し、-1 を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/collationcmds.c
  • src/backend/commands/dbcommands.c
  • src/backend/utils/adt/pg_locale.c
  • src/backend/utils/mb/mbutils.c
  • src/bin/initdb/initdb.c
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/interfaces/libpq/fe-connect.c
  • **呼び出し先**:
  • setlocale (標準ライブラリ)
  • newlocale, nl_langinfo_l, freelocale (非 Windows)
  • win32_get_codeset (Windows)
  • pg_strcasecmp
  • ereport / fprintf

関連データ構造

  • encoding_match_list: システムのエンコーディング名と PostgreSQL のエンコーディングコードを対応付ける静的テーブル。
  • pg_enc: src/include/mb/pg_wchar.h で定義されるエンコーディング列挙型。

失敗モードと境界条件

  • メモリ不足によりコードセット名の複製（strdup）に失敗した場合、-1 を返します。
  • ロケールが不正で newlocale が失敗した場合、-1 を返します。
  • 認識できないコードセット名の場合、警告を出力した上で -1 を返します。

保守担当者向けメモ

  • 新しいエンコーディングや別名を追加する場合は、encoding_match_list を更新してください。
  • Windows 環境での挙動は win32_get_codeset に依存しているため、Windows 特有のロケール問題が発生した場合はそちらを確認してください。
