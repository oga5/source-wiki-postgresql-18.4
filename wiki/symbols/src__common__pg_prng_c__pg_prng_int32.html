pg_prng_int32

定義

  • 種別: function
  • 定義箇所: src/common/pg_prng.c:242

目的

擬似乱数生成器の状態から、32ビット符号付き整数（int32）の乱数を生成する。

シグネチャまたは形状

int32 pg_prng_int32(pg_prng_state *state)

入力と出力

  • **入力**: pg_prng_state *state（乱数生成器の状態を保持する構造体へのポインタ）
  • **出力**: int32 型の乱数値

前提条件と不変条件

  • state は初期化済みであること。
  • 暗号学的な安全性は保証されないため、セキュリティが重要な用途（パスワード生成や鍵生成など）には使用しないこと。

アルゴリズムまたは振る舞い

1. 内部関数 xoroshiro128ss(state) を呼び出し、64ビットの乱数値を取得する。 2. 取得した64ビット値を右に32ビットシフトし、上位32ビットを取り出す。 3. その結果を int32 型にキャストして返す。

呼び出し元と呼び出し先

  • **呼び出し先**: xoroshiro128ss
  • **呼び出し元**: src/bin/pg_test_fsync/pg_test_fsync.c:235 にて pg_global_prng_state を引数として呼び出されている。

関連データ構造

  • pg_prng_state: 乱数生成器の状態を保持する構造体。

失敗モードと境界条件

  • pg_prng_state の全ビットが0の場合、生成器は0を返し続ける不動点となる。これを防ぐため、初期化時に pg_prng_seed_check 等による検証が推奨される。

保守担当者向けメモ

  • 本関数は xoroshiro128** アルゴリズムに基づいている。
  • 戻り値の型は int32 であるが、内部的には uint64 の上位ビットを利用している点に注意すること。
