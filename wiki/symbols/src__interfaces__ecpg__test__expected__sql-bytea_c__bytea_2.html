bytea_2

定義

  • 種別: struct
  • 定義箇所: src/interfaces/ecpg/test/expected/sql-bytea.c:57

目的

ECPG（Embedded SQL for C）において、SQL の bytea 型データを C 言語側で保持・送受信するためのデータ構造体です。

シグネチャまたは形状

struct bytea_2 {
    int len;
    char arr[DATA_SIZE];
};

入力と出力

  • **入力**: SQL 文の実行結果として、データベースから取得されたバイナリデータが arr に格納され、そのデータ長が len に設定されます。
  • **出力**: ECPG ランタイムを通じて、この構造体に格納されたバイナリデータが SQL 文のパラメータとしてデータベースへ送信されます。

前提条件と不変条件

  • len メンバには、arr に格納されている有効なバイナリデータのバイト数が正しく設定されている必要があります。
  • arr のサイズは DATA_SIZE で固定されています。

アルゴリズムまたは振る舞い

ECPG プリプロセッサが生成するコードにおいて、SQL の bytea 型と C 言語の構造体とのマッピングを仲介します。ECPGdo 関数等のインターフェースを通じて、バイナリデータの読み書きに使用されます。

呼び出し元と呼び出し先

  • **呼び出し元**: src/interfaces/ecpg/test/expected/sql-bytea.c 内の各 SQL 実行箇所（152, 200, 280, 334 行目付近）から、ECPGt_bytea 型として参照されています。
  • **呼び出し先**: ECPG ランタイムライブラリ（ecpglib）が、この構造体を介してデータベースとのバイナリデータ転送を行います。

関連データ構造

  • bytea_1, bytea_3, bytea_4: 同様の目的で定義された他の構造体。
  • recv_buf: 本構造体のインスタンスを要素として含む配列。

失敗モードと境界条件

  • DATA_SIZE を超えるバイナリデータが格納された場合、バッファオーバーフローのリスクがあります。
  • len が不適切な値（負の値や DATA_SIZE を超える値）に設定された場合、ECPG ランタイムによるデータ転送時にエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体はプリプロセッサ生成コードの一部であるため、直接の編集は避けてください。
  • bytea 型のテストにおいて、バッファサイズ不足の挙動を確認する際は、本構造体の len と DATA_SIZE の関係性に注意してください。
