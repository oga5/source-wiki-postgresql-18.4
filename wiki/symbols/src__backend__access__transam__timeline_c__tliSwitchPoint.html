tliSwitchPoint

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/timeline.c:571

目的

指定されたタイムライン（TimeLineID）が終了し、次のタイムラインへ切り替わった時点のWAL位置（LSN）を履歴リストから取得します。

シグネチャまたは形状

XLogRecPtr tliSwitchPoint(TimeLineID tli, List *history, TimeLineID *nextTLI)

入力と出力

  • **入力**:
  • tli: 検索対象のタイムラインID。
  • history: TimeLineHistoryEntry 構造体のリスト。
  • nextTLI: 次のタイムラインIDを格納するためのポインタ（不要な場合は NULL を指定可能）。
  • **出力**:
  • 戻り値: 指定した tli が終了したWAL位置（XLogRecPtr）。
  • nextTLI（出力引数）: 指定した tli の直後に続くタイムラインID。

前提条件と不変条件

  • history リストは、readTimeLineHistory 等によって構築された有効なタイムライン履歴である必要があります。
  • 指定された tli が履歴リスト内に存在しなければなりません。

アルゴリズムまたは振る舞い

1. nextTLI が指定されている場合、初期値として 0 をセットします。 2. history リストを先頭から順に走査します。 3. 現在の要素の tli が検索対象と一致した場合、その要素の end（終了位置）を戻り値として返します。 4. 一致しない場合、現在の要素の tli を nextTLI に更新し、次の要素へ進みます。 5. リストの末尾まで走査しても一致が見つからない場合、ereport(ERROR) を発行し、処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し先**:

関連データ構造

  • TimeLineHistoryEntry: タイムラインの開始・終了位置とIDを保持する構造体。
  • List: PostgreSQLの標準的なリスト構造。

失敗モードと境界条件

  • **エラー**: 指定された tli が履歴リストに見つからない場合、ERROR レベルでログを出力し、トランザクションをアボートします。
  • **境界条件**: nextTLI に NULL を渡した場合、次のタイムラインIDの取得はスキップされます。

保守担当者向けメモ

  • 本関数は履歴リストの走査を行うため、リストの長さが非常に長い場合、パフォーマンスに影響を与える可能性があります。
  • 戻り値の InvalidXLogRecPtr は、エラー発生時のコンパイラ警告抑制用であり、正常系では到達しません。
