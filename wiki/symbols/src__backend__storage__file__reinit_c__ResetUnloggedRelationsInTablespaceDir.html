ResetUnloggedRelationsInTablespaceDir

定義

  • 種別: function
  • 定義箇所: src/backend/storage/file/reinit.c:105

目的

指定されたテーブルスペースディレクトリ内の全データベースディレクトリを走査し、アンログド（unlogged）リレーションのクリーンアップまたは初期化処理を各データベース単位で実行する。

シグネチャまたは形状

static void ResetUnloggedRelationsInTablespaceDir(const char *tsdirname, int op)

入力と出力

  • tsdirname (入力): 走査対象のテーブルスペースディレクトリパス。
  • op (入力): 実行する操作フラグ（UNLOGGED_RELATION_INIT または UNLOGGED_RELATION_CLEANUP）。
  • 戻り値: なし。

前提条件と不変条件

  • tsdirname が存在しない場合（ENOENT）、エラーログを出力して正常終了する（テーブルスペース削除とディレクトリ削除の競合を許容するため）。
  • データベースディレクトリは数値名を持つもののみが処理対象となる。

アルゴリズムまたは振る舞い

1. AllocateDir を使用して tsdirname を開く。 2. ディレクトリ内のエントリを ReadDir で順次読み込む。 3. エントリ名が数値のみで構成されている場合、それをデータベースディレクトリと見なし、パスを構築する。 4. op に応じて起動進捗ログを出力する。 5. ResetUnloggedRelationsInDbspaceDir を呼び出し、当該データベースディレクトリ内のリレーションを処理する。 6. 最後に FreeDir でディレクトリハンドルを解放する。

呼び出し元と呼び出し先

  • 呼び出し元: ResetUnloggedRelations（src/backend/storage/file/reinit.c 内の各箇所から呼び出し）。
  • 呼び出し先: ResetUnloggedRelationsInDbspaceDir。

関連データ構造

  • DIR: ディレクトリ操作用ハンドル。
  • dirent: ディレクトリ内のファイルエントリ情報。

失敗モードと境界条件

  • AllocateDir が ENOENT 以外のエラーを返した場合、ReadDir 等の標準的なエラー処理により起動プロセスが失敗する。
  • 数値以外の名前を持つディレクトリやファイルはスキップされる。

保守担当者向けメモ

  • 本関数はテーブルスペースレベルの走査を担当し、個別のデータベースディレクトリに対する処理は ResetUnloggedRelationsInDbspaceDir に委譲している。
  • op フラグによる条件分岐は、ログ出力の制御に使用されている。
