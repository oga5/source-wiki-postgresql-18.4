proc_exit

定義

  • 種別: function
  • 定義箇所: src/backend/storage/ipc/ipc.c:104

目的

PostgreSQLのバックエンドプロセス終了時におけるメインエントリポイントとして、リソースの安全な解放とクリーンアップ処理をオーケストレーションする。

シグネチャまたは形状

void proc_exit(int code)

入力と出力

  • **入力**: int code（終了ステータスコード）
  • **出力**: なし（プロセスを終了させるため、呼び出し元には戻らない）

前提条件と不変条件

  • 呼び出し元プロセスが自身のプロセスID（MyProcPid）と一致している必要がある。一致しない場合、PANIC エラーが発生する。
  • 終了処理の整合性を保つため、proc_exit_prepare(code) を経由して終了処理中フラグのセットや割り込み無効化が行われる。

アルゴリズムまたは振る舞い

1. **プロセス検証**: MyProcPid と getpid() を比較し、子プロセスからの不正な呼び出しを防止する。 2. **準備**: proc_exit_prepare(code) を呼び出し、終了処理の準備を行う。 3. **プロファイリング処理 (PROFILE_PID_DIR 定義時)**:

  • gprof 用のプロファイル出力先ディレクトリ（gprof/pid または gprof/avworker）を作成し、カレントディレクトリを移動する。
  • 他の on_proc_exit コールバックによる干渉を避けるため、この処理は最後に実行される。

    • 4. **ログ出力**: DEBUG3 レベルで終了コードを記録する。 5. **終了**: exit(code) を呼び出し、プロセスを終了させる。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/transam/parallel.c
  • src/backend/access/transam/xlogarchive.c
  • src/backend/access/transam/xlogrecovery.c
  • src/backend/bootstrap/bootstrap.c
  • src/backend/libpq/auth.c
  • **呼び出し先**:
  • proc_exit_prepare
  • exit
  • elog (PANIC/DEBUG3)
  • getpid, mkdir, chdir (プラットフォーム依存)

関連データ構造

  • MyProcPid: 現在のプロセスのPIDを保持するグローバル変数。

失敗モードと境界条件

  • **子プロセスからの呼び出し**: MyProcPid と getpid() が不一致の場合、PANIC により即座に停止する。
  • **プロファイリング**: PROFILE_PID_DIR が有効な環境で、ディレクトリ作成や移動に失敗した場合の挙動はOSの mkdir/chdir に依存する。

保守担当者向けメモ

  • 本関数は終了シーケンスの最終段階を担う。コールバックの実行順序を維持するため、特定の処理（プロファイリング等）は on_proc_exit リストではなく、この関数内に直接記述されている。
  • on_proc_exit コールバック内でエラーが発生した場合の再帰防止策は proc_exit_prepare 側で管理されている。
