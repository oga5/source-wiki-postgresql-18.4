EventTriggerData

定義

  • 種別: struct
  • 定義箇所: src/include/commands/event_trigger.h:24

目的

イベントトリガー関数実行時に、トリガーのコンテキスト情報（イベント名やパースツリーなど）を保持し、関数へ伝達するためのデータ構造です。

シグネチャまたは形状

typedef struct EventTriggerData
{
    NodeTag     type;           /* ノードタグ */
    const char *event;          /* イベント名 */
    Node       *parsetree;      /* パースツリー */
    CommandTag  tag;            /* コマンドタグ */
} EventTriggerData;

入力と出力

  • **入力**: イベントトリガー発生時にバックエンドのコマンド実行フローから構築されます。
  • **出力**: fmgr の fcinfo->context を通じて、呼び出されるイベントトリガー関数に渡されます。

前提条件と不変条件

  • fcinfo->context として渡される際、IsA(fcinfo->context, EventTriggerData) が真である必要があります。
  • CALLED_AS_EVENT_TRIGGER(fcinfo) マクロにより、呼び出し元がイベントトリガー経由であるかを確認可能です。

アルゴリズムまたは振る舞い

1. DDLコマンド実行時、バックエンドが EventTriggerData をインスタンス化します。 2. 実行対象のイベントトリガー関数を検索し、EventTriggerInvoke を介して関数を呼び出します。 3. その際、EventTriggerData へのポインタが fcinfo->context に格納されます。 4. イベントトリガー関数は、この構造体を参照してイベントの種類や関連するパースツリーを解析します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/event_trigger.c 内の各フック関数（EventTriggerDDLCommandStart 等）がこの構造体を生成します。
  • **呼び出し先**: EventTriggerInvoke 関数が、この構造体を引数として受け取り、トリガー関数を実行します。

関連データ構造

  • Node: パースツリーの基底構造。
  • CommandTag: 実行されたコマンドの種類を識別する列挙型。
  • fcinfo (FunctionCallInfo): fmgr における関数呼び出し情報。

失敗モードと境界条件

  • fcinfo->context が NULL である場合、または期待される型と異なる場合は、イベントトリガーとしての実行が正しく行われません。

保守担当者向けメモ

  • 本構造体は src/backend/commands/event_trigger.c と密接に連携しています。
  • フィールドの追加や変更を行う際は、EventTriggerInvoke および関連するフック関数の初期化ロジックを更新する必要があります。
