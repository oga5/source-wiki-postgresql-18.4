PGSemaphoreTryLock

定義

  • 種別: function
  • 定義箇所: src/backend/port/win32_sema.c:209

目的

ブロックせずにセマフォの取得を試みます。

シグネチャまたは形状

bool PGSemaphoreTryLock(PGSemaphore sema)

入力と出力

  • **入力**: sema (取得対象のセマフォハンドル)
  • **出力**: セマフォが取得できた場合は true、取得できなかった場合は false を返します。

前提条件と不変条件

  • Windows環境におけるWin32 APIのセマフォハンドルを対象とします。
  • 呼び出しは非ブロッキングであり、即座に結果を返します。

アルゴリズムまたは振る舞い

1. WaitForSingleObject をタイムアウト値 0 で呼び出し、セマフォの状態を確認します。 2. 戻り値が WAIT_OBJECT_0 の場合、セマフォを取得できたと判断し true を返します。 3. 戻り値が WAIT_TIMEOUT の場合、セマフォが利用不可であると判断し、errno を EAGAIN に設定した上で false を返します。 4. 上記以外の戻り値（エラー）が発生した場合は、ereport(FATAL, ...) を呼び出し、プロセスを終了させます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/port/win32_sema.c:122 (ループ内での利用)
  • src/backend/port/posix_sema.c および src/backend/port/sysv_sema.c に同名の宣言が存在しますが、これらはプラットフォーム固有の実装であり、本関数を直接呼び出しているわけではありません。
  • **呼び出し先**:
  • WaitForSingleObject (Win32 API)
  • GetLastError (Win32 API)
  • ereport (PostgreSQLエラー報告マクロ)

関連データ構造

  • PGSemaphore: セマフォハンドルを保持する型。

失敗モードと境界条件

  • **セマフォ取得失敗**: WAIT_TIMEOUT が返された場合、errno を EAGAIN に設定して正常な制御フローとして false を返します。
  • **システムエラー**: WaitForSingleObject が予期せぬエラーを返した場合、FATAL エラーとして処理を中断します。

保守担当者向けメモ

  • 本関数はWindows環境専用の実装です。他プラットフォーム（POSIX/System V）との互換性のため、src/include/storage/pg_sema.h で共通インターフェースとして定義されています。
  • WaitForSingleObject の戻り値が WAIT_OBJECT_0 または WAIT_TIMEOUT 以外の場合に FATAL を投げる設計は、カーネルオブジェクトの状態異常を致命的なエラーとみなす方針に基づいています。
