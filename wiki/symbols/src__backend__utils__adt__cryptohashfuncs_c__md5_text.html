md5_text

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/cryptohashfuncs.c:33

目的

SQLのmd5()関数に対応し、入力されたtext型データからMD5ハッシュ値を計算して16進数文字列として返却する。

シグネチャまたは形状

Datum md5_text(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して受け取るtext型データ。
  • **出力**: 計算されたMD5ハッシュ値を含むtext型（16進数文字列）。

前提条件と不変条件

  • 入力データはtext型として渡される必要がある。
  • pg_md5_hash が内部でハッシュ計算を実行する。

アルゴリズムまたは振る舞い

1. PG_GETARG_TEXT_PP を使用して入力テキストを取得する。 2. VARSIZE_ANY_EXHDR を用いて、ヘッダーを除いたデータ長を算出する。 3. pg_md5_hash を呼び出し、入力データからハッシュ値を生成する。 4. 計算に失敗した場合は ereport(ERROR, ...) を発行し、処理を中断する。 5. 成功した場合、cstring_to_text を介して結果をtext型に変換し、PG_RETURN_TEXT_P で返却する。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース（PostgreSQLの関数呼び出し規約経由）。
  • **呼び出し先**: pg_md5_hash（ハッシュ計算用）、cstring_to_text（戻り値変換用）、ereport（エラー報告用）。

関連データ構造

  • text: PostgreSQLの可変長文字列型。
  • Datum: PostgreSQLの汎用データ型表現。

失敗モードと境界条件

  • pg_md5_hash が false を返した場合、内部エラー（ERRCODE_INTERNAL_ERROR）として処理される。
  • 入力データがNULLの場合の挙動は、PG_FUNCTION_ARGS の標準的なマクロ定義に依存する。

保守担当者向けメモ

  • 本関数はMD5アルゴリズムを使用している。MD5はセキュリティ上の脆弱性が知られているため、用途には注意が必要である。
  • 戻り値はバイナリハッシュではなく16進数文字列である点に留意すること。
