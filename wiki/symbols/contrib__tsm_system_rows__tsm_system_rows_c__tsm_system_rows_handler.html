tsm_system_rows_handler

定義

  • 種別: function
  • 定義箇所: contrib/tsm_system_rows/tsm_system_rows.c:80

目的

PostgreSQLの TABLESAMPLE SYSTEM_ROWS メソッドをクエリ実行エンジンに登録するためのハンドラ関数です。TsmRoutine 構造体を生成し、サンプリングに必要な各コールバック関数を割り当てます。

シグネチャまたは形状

Datum tsm_system_rows_handler(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS (PostgreSQLの関数呼び出し規約に基づく引数)
  • **出力**: Datum 型として TsmRoutine 構造体へのポインタを返す。

前提条件と不変条件

  • makeNode(TsmRoutine) によりメモリが確保されること。
  • parameterTypes に INT8OID が設定され、単一の整数パラメータを受け入れる仕様となっていること。

アルゴリズムまたは振る舞い

1. TsmRoutine 構造体を初期化する。 2. サンプリングの特性を設定する（repeatable_across_queries = false, repeatable_across_scans = true）。 3. 以下の関数ポインタを登録する:

  • SampleScanGetSampleSize: system_rows_samplescangetsamplesize
  • InitSampleScan: system_rows_initsamplescan
  • BeginSampleScan: system_rows_beginsamplescan
  • NextSampleBlock: system_rows_nextsampleblock
  • NextSampleTuple: system_rows_nextsampletuple

    • 4. EndSampleScan は NULL に設定される。 5. 構築した tsm 構造体を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQLの拡張モジュールロード機構（PG_FUNCTION_INFO_V1 マクロ経由で登録）。
  • **呼び出し先**:
  • makeNode (PostgreSQL内部関数)
  • list_make1_oid (PostgreSQL内部関数)
  • system_rows_samplescangetsamplesize 等の各サンプリング用関数（関数ポインタとして代入）。

関連データ構造

  • TsmRoutine: サンプリングメソッドのインターフェースを定義する構造体。

失敗モードと境界条件

  • メモリ不足時には makeNode がエラーを発生させる可能性がある。
  • EndSampleScan が NULL であるため、スキャン終了時のクリーンアップ処理は呼び出されない。

保守担当者向けメモ

  • 本関数は TABLESAMPLE のエントリポイントです。新しいサンプリングロジックを追加または変更する場合は、この関数で登録される関数ポインタの対応関係を確認してください。
  • repeatable_across_queries が false であるため、クエリを跨いだサンプリング結果の再現性は保証されません。
