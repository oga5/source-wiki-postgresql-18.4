SnapBuildBuildSnapshot

定義

  • **種別**: 関数
  • **定義箇所**: src/backend/replication/logical/snapbuild.c:359

目的

論理デコーディングにおいて、カタログ（システムテーブル）の読み取りに使用する SNAPSHOT_HISTORIC_MVCC 型のスナップショットを構築する。

シグネチャまたは形状

static Snapshot SnapBuildBuildSnapshot(SnapBuild *builder)

入力と出力

  • **入力**: SnapBuild *builder （スナップショット構築の状態を保持する構造体）
  • **出力**: Snapshot （構築されたカタログ読み取り用スナップショット）

前提条件と不変条件

  • builder->state が SNAPBUILD_FULL_SNAPSHOT 以上であること。
  • builder->xmin および builder->xmax が有効なトランザクションIDであること。

アルゴリズムまたは振る舞い

1. builder->committed リストのサイズに基づき、必要なメモリ領域を確保する。 2. snapshot_type を SNAPSHOT_HISTORIC_MVCC に設定する。 3. builder->xmin と builder->xmax をスナップショットにコピーする。 4. builder->committed.xip に格納されている「コミット済みとみなすべきトランザクションID」をスナップショットの xip 配列にコピーし、qsort でソートする（これにより bsearch が可能になる）。 5. subxip（サブトランザクション用）は初期状態では空（NULL）に設定する。 6. その他のスナップショットフィールド（curcid 等）を初期化して返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/snapbuild.c 内の各所（行: 467, 586, 666, 1080, 1908）。
  • **呼び出し先**: MemoryContextAllocZero, qsort, xidComparator。

関連データ構造

  • SnapBuild: スナップショット構築のメイン状態構造体。
  • SnapshotData: 生成されるスナップショットの基底構造体。
  • Snapshot: SnapshotData へのポインタ。

失敗モードと境界条件

  • メモリ不足時には MemoryContextAllocZero がエラーを発生させる。
  • builder->committed.xcnt が 0 の場合、xip 配列は空として処理される。

保守担当者向けメモ

  • 本関数は SnapshotData の xip および subxip フィールドを、論理デコーディング特有の用途（カタログ可視性制御）に転用している。
  • subxip は必要に応じて ReorderBufferCopySnap() によって後から埋められる設計である。
  • xip 配列はソート済みであることが bsearch 利用の前提条件となるため、変更時は注意が必要。
