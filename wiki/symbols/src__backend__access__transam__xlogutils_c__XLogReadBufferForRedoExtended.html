XLogReadBufferForRedoExtended

定義

  • 種別: function
  • 定義箇所: src/backend/access/transam/xlogutils.c:339

目的

WALリプレイ処理において、指定されたブロックをバッファキャッシュへ読み込み、必要に応じてフルページイメージの復元やロックの取得、Redo適用の要否判定を行う。

シグネチャまたは形状

XLogRedoAction XLogReadBufferForRedoExtended(
    XLogReaderState *record,
    uint8 block_id,
    ReadBufferMode mode,
    bool get_cleanup_lock,
    Buffer *buf
)

入力と出力

  • **入力**:
  • record: 解析済みのWALレコード状態。
  • block_id: 対象ブロックの識別子。
  • mode: バッファ読み込みモード（ReadBufferMode）。
  • get_cleanup_lock: クリーンアップロックを取得するか否かのフラグ。
  • buf: 読み込んだバッファを格納するポインタ。
  • **出力**:
  • 戻り値: XLogRedoAction（BLK_RESTORED, BLK_DONE, BLK_NEEDS_REDO, BLK_NOTFOUND のいずれか）。

前提条件と不変条件

  • 本関数はリカバリプロセス中のみ呼び出されることを前提とする。
  • WILL_INIT フラグがWALレコードにある場合、mode は必ずゼロ初期化を伴うモード（RBM_ZERO_...）でなければならない。
  • INIT_FORKNUM の場合、共有バッファとディスク状態を同期させるため FlushOneBuffer が実行される。

アルゴリズムまたは振る舞い

1. XLogRecGetBlockTagExtended を使用してブロックの物理位置（リレーション、フォーク、ブロック番号）を特定する。 2. フルページイメージが存在し、適用すべき場合は XLogReadBufferExtended でバッファを取得し、RestoreBlockImage でページを復元する。ページが新規でない場合はLSNを更新し、ダーティマークを付与する。 3. フルページイメージがない場合は XLogReadBufferExtended でバッファを取得する。 4. バッファが有効な場合、mode に応じて排他ロックまたはクリーンアップロックを取得する。 5. ページLSNとレコードのLSNを比較し、適用済みであれば BLK_DONE、未適用であれば BLK_NEEDS_REDO を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/hash/hash_xlog.c および src/backend/access/heap/heapam_xlog.c 等のリソースマネージャのRedoルーチン。
  • **呼び出し先**: XLogRecGetBlockTagExtended, XLogRecBlockImageApply, XLogReadBufferExtended, RestoreBlockImage, BufferGetPage, PageSetLSN, MarkBufferDirty, FlushOneBuffer, LockBuffer, LockBufferForCleanup, PageGetLSN。

関連データ構造

  • XLogReaderState: WALレコードの解析状態を保持。
  • Buffer: バッファキャッシュ内のページを指すハンドル。
  • XLogRedoAction: リプレイアクションの結果を示す列挙型。

失敗モードと境界条件

  • block_id が無効な場合: PANIC で停止。
  • WILL_INIT フラグと mode の不整合: PANIC で停止。
  • フルページイメージの復元失敗: ERROR を報告。
  • ページが存在しない場合: BLK_NOTFOUND を返す。

保守担当者向けメモ

  • 本関数はリカバリ専用のユーティリティであり、通常のバックエンドプロセスからの呼び出しは想定されていない。
  • INIT_FORKNUM に対する FlushOneBuffer は、クラッシュリカバリ終了時の整合性維持のために必須である。
