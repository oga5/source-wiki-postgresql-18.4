tidsend

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tid.c:159

目的

TID (Tuple Identifier) 型の値を、クライアント・サーバー間の通信に適したバイナリ形式に変換します。

シグネチャまたは形状

Datum tidsend(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_GETARG_ITEMPOINTER(0) を介して取得される ItemPointer 型のタプル識別子。
  • **出力**: bytea 型のバイナリデータ。ブロック番号（4バイト）とオフセット番号（2バイト）がネットワークバイトオーダーで格納されます。

前提条件と不変条件

  • 入力された ItemPointer は有効なメモリ領域を指している必要があります。
  • pq_begintypsend および pq_endtypsend を使用するため、PostgreSQL のバイナリ転送プロトコルに準拠した形式で出力されます。

アルゴリズムまたは振る舞い

1. pq_begintypsend を呼び出し、バイナリ送信用のバッファを初期化します。 2. ItemPointerGetBlockNumberNoCheck を用いてブロック番号を取得し、pq_sendint32 でバッファに書き込みます。 3. ItemPointerGetOffsetNumberNoCheck を用いてオフセット番号を取得し、pq_sendint16 でバッファに書き込みます。 4. pq_endtypsend を呼び出し、構築されたバッファを bytea 型として返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL の関数マネージャ（fmgr）経由で呼び出されます。具体的な呼び出し元コードの詳細は提供されていません。
  • **呼び出し先**:
  • pq_begintypsend, pq_sendint32, pq_sendint16, pq_endtypsend（libpq/pqformat.h）
  • ItemPointerGetBlockNumberNoCheck, ItemPointerGetOffsetNumberNoCheck（src/include/storage/itemptr.h）

関連データ構造

  • ItemPointer: TID の内部表現。
  • StringInfoData: バイナリ転送用バッファ。
  • bytea: PostgreSQL の可変長バイナリ型。

失敗モードと境界条件

  • メモリ不足が発生した場合、pq_begintypsend 等の内部関数がエラーを投げる可能性があります。
  • 入力ポインタが NULL の場合、ItemPointerGet... マクロの挙動により不正なメモリ参照が発生する可能性があります。

保守担当者向けメモ

  • 本関数は TID 型のバイナリ表現を定義するものであり、変更時はクライアント側のデコードロジックとの整合性に注意してください。
  • NoCheck 系のマクロを使用しているため、入力値の妥当性検証は呼び出し側で行う必要があります。
