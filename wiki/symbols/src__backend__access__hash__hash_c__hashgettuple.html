hashgettuple

定義

  • 種別: function
  • 定義箇所: src/backend/access/hash/hash.c:289

目的

ハッシュインデックススキャンにおいて、次のタプルを取得する。

シグネチャまたは形状

bool hashgettuple(IndexScanDesc scan, ScanDirection dir)

入力と出力

  • **入力**:
  • scan: インデックススキャンの状態を保持する IndexScanDesc 構造体。
  • dir: スキャン方向を示す ScanDirection。
  • **出力**:
  • 次のタプルが見つかった場合は true、それ以外の場合は false を返す。

前提条件と不変条件

  • ハッシュインデックスはハッシュコードのみを保持するため、常に損失あり（lossy）とみなされる。そのため、本関数は常に scan->xs_recheck = true を設定する。

アルゴリズムまたは振る舞い

1. scan->xs_recheck を true に設定する。 2. スキャンが未初期化（HashScanPosIsValid が偽）の場合、_hash_first を呼び出して最初のアイテムを取得する。 3. スキャンが既に初期化されている場合:

  • scan->kill_prior_tuple が真であれば、直前に取得したタプルを削除対象として so->killedItems に記録する。
  • _hash_next を呼び出して次のアイテムを取得する。

    • 4. 取得結果を呼び出し元に返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/hash/hash.c 内の amroutine->amgettuple に登録されている。
  • **呼び出し先**:
  • _hash_first
  • _hash_next

関連データ構造

  • IndexScanDesc: スキャン状態を管理する構造体。
  • HashScanOpaque: ハッシュインデックス固有のスキャン状態（so->currPos, so->killedItems 等を含む）。

失敗モードと境界条件

  • so->numKilled が MaxIndexTuplesPerPage を超える場合、それ以上の削除対象記録は行われない。
  • スキャン方向が反転した場合、同一アイテムが複数回記録される可能性があるが、本関数では重複排除を行わず、超過分を無視する。

保守担当者向けメモ

  • 本関数はインターフェース層であり、具体的な検索ロジックは _hash_first および _hash_next に委譲されている。
  • 削除対象タプルの処理は、インデックスページを離れる際やスキャン終了時に一括して行われる設計となっている。
