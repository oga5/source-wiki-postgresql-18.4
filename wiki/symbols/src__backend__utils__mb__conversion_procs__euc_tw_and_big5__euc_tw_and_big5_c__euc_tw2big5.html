euc_tw2big5

定義

  • 種別: function
  • 定義箇所: src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:151

目的

EUC_TW エンコーディングのバイト列を BIG5 エンコーディングへ変換する内部関数です。

シグネチャまたは形状

static int euc_tw2big5(const unsigned char *euc, unsigned char *p, int len, bool noError)

入力と出力

  • **入力**:
  • euc: 変換元となる EUC_TW 文字列のポインタ。
  • p: 変換後の BIG5 文字列を格納するバッファ。
  • len: 入力文字列のバイト長。
  • noError: 変換エラー時に例外を投げず処理を中断するかを指定するフラグ。
  • **出力**:
  • 戻り値: 変換処理したバイト数。

前提条件と不変条件

  • 入力文字列は EUC_TW エンコーディングに従っている必要があります。
  • 変換後のバッファ p は、十分なサイズが確保されている必要があります。
  • 処理終了後、p の末尾にはヌル終端文字が付与されます。

アルゴリズムまたは振る舞い

1. 入力文字列をバイト単位で走査し、pg_encoding_verifymbchar を用いて文字境界を検証します。 2. SS2 (シングルシフト2) を検出した場合、後続のバイトから CNS11643 のプレーン番号を特定し、CNStoBIG5 を呼び出して変換します。 3. SS2 以外の場合は CNS11643-1 として扱い、同様に変換します。 4. ASCII 文字（高位ビットが立っていない文字）はそのままコピーします。 5. noError が true の場合、不正な文字や変換不能な文字に遭遇した時点で処理を中断します。

呼び出し元と呼び出し先

  • **呼び出し元**: 同一ファイル内の euc_tw_to_big5 関数（src/backend/utils/mb/conversion_procs/euc_tw_and_big5/euc_tw_and_big5.c:62）から呼び出されます。
  • **呼び出し先**:
  • pg_encoding_verifymbchar: 文字の妥当性検証。
  • CNStoBIG5: CNS11643 コードポイントから BIG5 への変換。
  • report_invalid_encoding / report_untranslatable_char: エラー報告（noError が false の場合）。

関連データ構造

  • PG_EUC_TW, PG_BIG5: エンコーディング識別子。
  • SS2: EUC_TW における補助文字セットを示す定数。
  • LC_CNS11643_x: MULE_INTERNAL における文字セット識別子。

失敗モードと境界条件

  • 入力文字列が不正なマルチバイトシーケンスである場合、noError が false ならエラーを報告し、true ならその時点までのバイト数を返して終了します。
  • 変換不能な文字（CNStoBIG5 が 0 を返す場合）も同様に、noError 設定に従ってエラー報告または中断を行います。
  • 入力文字列が空、またはヌル文字が含まれる場合の挙動は、pg_encoding_verifymbchar の検証結果に依存します。

保守担当者向けメモ

  • 本関数は変換ロジックのコアであり、具体的な変換テーブルは CNStoBIG5（外部定義）に依存しています。
  • noError フラグによる制御は、COPY コマンド等の柔軟なデータロードをサポートするために重要です。
