tuplesort_begin_heap

定義

  • 種別: function
  • 定義箇所: src/backend/utils/sort/tuplesortvariants.c:179

目的

HeapTuple（行データ）のソート処理を開始するための初期化関数です。ソートに必要な比較ロジックや入出力関数を Tuplesortstate に注入し、実行環境を構築します。

シグネチャまたは形状

Tuplesortstate *tuplesort_begin_heap(
    TupleDesc tupDesc,
    int nkeys,
    AttrNumber *attNums,
    Oid *sortOperators,
    Oid *sortCollations,
    bool *nullsFirstFlags,
    int workMem,
    SortCoordinate coordinate,
    int sortopt
)

入力と出力

  • **入力**:
  • tupDesc: ソート対象タプルの記述子。
  • nkeys: ソートキーの数。
  • attNums, sortOperators, sortCollations, nullsFirstFlags: 各ソートキーの属性番号、演算子、照合順序、NULL配置フラグの配列。
  • workMem: ソートに使用可能なメモリ量。
  • coordinate: 並列ソート等の実行コンテキスト。
  • sortopt: ランダムアクセス等のオプションフラグ。
  • **出力**: 初期化済みの Tuplesortstate 構造体へのポインタ。

前提条件と不変条件

  • nkeys > 0 であること。
  • attNums および sortOperators の各要素が 0 でないこと。
  • base->maincontext にメモリコンテキストを切り替えて処理が行われる。

アルゴリズムまたは振る舞い

1. tuplesort_begin_common を呼び出し、共通のソート状態を初期化する。 2. HeapTuple 用の関数ポインタ（comparetup_heap, writetup_heap, readtup_heap 等）を Tuplesortstate に登録する。 3. 各ソートキーに対して PrepareSortSupportFromOrderingOp を呼び出し、比較のためのサポートデータを構築する。 4. 省略キー（Abbreviated Keys）の最適化が適用可能か判定し、単一キーかつ変換器が不要な場合は onlyKey 最適化を有効にする。 5. 最後に元のメモリコンテキストへ復帰し、状態を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/executor/nodeAgg.c
  • src/backend/executor/nodeIncrementalSort.c
  • src/backend/executor/nodeSort.c
  • src/backend/utils/adt/orderedsetaggs.c
  • **呼び出し先**:
  • tuplesort_begin_common
  • PrepareSortSupportFromOrderingOp
  • MemoryContextSwitchTo
  • palloc0

関連データ構造

  • Tuplesortstate: ソート処理の全状態を保持するメイン構造体。
  • SortSupportData: 各カラムの比較ロジックを保持する構造体。
  • TupleDesc: タプルの構造定義。

失敗モードと境界条件

  • nkeys が 0 の場合、アサーションエラーが発生する。
  • attNums や sortOperators に 0 が含まれる場合、アサーションエラーが発生する。

保守担当者向けメモ

  • 本関数は tuplesort.c のプラグインとして機能します。HeapTuple 固有の比較ロジックを変更する場合は、comparetup_heap 等の関数と併せて修正してください。
  • 省略キー最適化と onlyKey 最適化は排他的な関係にあるため、ロジック変更時には注意が必要です。
