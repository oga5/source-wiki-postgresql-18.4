GinBtreeData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/gin_private.h:151

目的

GIN（Generalized Inverted Index）インデックスにおけるエントリツリーおよびポスティングツリーの走査、挿入、および操作を行うための汎用的なメソッドテーブルおよびコンテキストを保持します。

シグネチャまたは形状

typedef struct GinBtreeData {
    /* 検索メソッド */
    BlockNumber (*findChildPage) (GinBtree, GinBtreeStack *);
    BlockNumber (*getLeftMostChild) (GinBtree, Page);
    bool        (*isMoveRight) (GinBtree, Page);
    bool        (*findItem) (GinBtree, GinBtreeStack *);

    /* 挿入メソッド */
    OffsetNumber (*findChildPtr) (GinBtree, Page, BlockNumber, OffsetNumber);
    GinPlaceToPageRC (*beginPlaceToPage) (GinBtree, Buffer, GinBtreeStack *, void *, BlockNumber, void **, Page *, Page *);
    void        (*execPlaceToPage) (GinBtree, Buffer, GinBtreeStack *, void *, BlockNumber, void *);
    void       *(*prepareDownlink) (GinBtree, Buffer);
    void        (*fillRoot) (GinBtree, Page, BlockNumber, Page, BlockNumber, Page);

    bool        isData;
    Relation    index;
    BlockNumber rootBlkno;
    GinState   *ginstate;
    bool        fullScan;
    bool        isBuild;

    /* エントリツリー用検索キー */
    OffsetNumber entryAttnum;
    Datum        entryKey;
    GinNullCategory entryCategory;

    /* データツリー（ポスティングツリー）用検索キー */
    ItemPointerData itemptr;
} GinBtreeData;

入力と出力

  • **入力**: インデックス操作に必要な Relation、GinState、および検索対象のキー情報（entryKey や itemptr）を保持します。
  • **出力**: メソッドポインタを通じて、ツリーの走査結果（GinBtreeStack）や挿入処理の制御（GinPlaceToPageRC）を返します。

前提条件と不変条件

  • ginstate はデータスキャン時には無効です。
  • isData フラグにより、対象がエントリツリーかポスティングツリーかを区別します。

アルゴリズムまたは振る舞い

  • GIN インデックスの各ページ操作に対して、関数ポインタを介した抽象化レイヤーを提供します。
  • 検索時には findChildPage や findItem を使用してツリーを降下し、挿入時には beginPlaceToPage や execPlaceToPage を用いてページ分割やタプル挿入を制御します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/gin/gindatapage.c、src/backend/access/gin/ginentrypage.c、src/backend/access/gin/ginget.c、src/backend/access/gin/gininsert.c などでインスタンス化および利用されています。
  • **呼び出し先**: 構造体内の関数ポインタが指す具体的な実装関数（ginEntryInsert 等）が呼び出されます。

関連データ構造

  • GinBtree: GinBtreeData へのポインタ型。
  • GinBtreeStack: ツリー走査時のパスを保持するスタック構造。
  • GinState: インデックス操作のコンテキスト。

失敗モードと境界条件

  • ginstate が適切に初期化されていない状態でアクセスすると未定義の動作を引き起こします。
  • ページ操作メソッドは、バッファのロック状態やページレイアウト（ginblock.h 準拠）に依存します。

保守担当者向けメモ

  • 本構造体は GIN の内部実装の基盤であり、メソッドテーブルの追加や変更は gin_private.h を参照する全ソースファイルに影響します。
  • isData フラグの切り替えにより、エントリツリーとポスティングツリーで共通のロジックを再利用する設計となっています。
