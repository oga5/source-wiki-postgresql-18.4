CreatePublication

定義

  • 種別: function
  • 定義箇所: src/backend/commands/publicationcmds.c:831

目的

SQLの CREATE PUBLICATION 文を実行し、論理レプリケーション用のパブリケーション定義をシステムカタログ pg_publication に作成する。

シグネチャまたは形状

ObjectAddress CreatePublication(ParseState *pstate, CreatePublicationStmt *stmt)

入力と出力

  • **入力**:
  • pstate: 構文解析状態。
  • stmt: CREATE PUBLICATION 文の解析結果構造体。
  • **出力**: 作成されたパブリケーションの ObjectAddress を返す。

前提条件と不変条件

  • データベースに対する CREATE 権限が必要。
  • FOR ALL TABLES を指定する場合、実行ユーザーはスーパーユーザーである必要がある。
  • FOR TABLES IN SCHEMA を指定する場合、実行ユーザーはスーパーユーザーである必要がある。
  • 指定されたパブリケーション名が既に存在する場合はエラーとなる。
  • wal_level が logical でない場合、警告を発する。

アルゴリズムまたは振る舞い

1. 権限チェックを行い、pg_publication リレーションをオープンする。 2. パブリケーション名の重複を確認し、オプションを解析する。 3. pg_publication に新しいタプルを挿入し、所有者との依存関係を記録する。 4. CommandCounterIncrement() を呼び出し、変更を可視化する。 5. for_all_tables が真の場合は全リレーションのキャッシュを無効化する。 6. それ以外の場合は、指定されたテーブルやスキーマをパブリケーションに関連付け、必要に応じて行フィルタの変換やカラムリストの検証を行う。 7. 最後にオブジェクト作成フックを呼び出し、作成したパブリケーションの ObjectAddress を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/tcop/utility.c (SQL文の実行時)
  • **呼び出し先**:
  • parse_publication_options, ObjectsInPublicationToOids, TransformPubWhereClauses, CheckPubRelationColumnList, PublicationAddTables, PublicationAddSchemas, CacheInvalidateRelcacheAll, InvokeObjectPostCreateHook 等。

関連データ構造

  • CreatePublicationStmt: SQL文の解析結果を保持する構造体。
  • pg_publication: パブリケーション情報を格納するシステムカタログ。
  • PublicationActions: パブリケーションの操作権限（insert, update, delete, truncate）を保持する構造体。

失敗モードと境界条件

  • 権限不足（ERRCODE_INSUFFICIENT_PRIVILEGE）: スーパーユーザー権限が必要な条件を満たさない場合。
  • 重複定義（ERRCODE_DUPLICATE_OBJECT）: 同名のパブリケーションが既に存在する場合。
  • wal_level 不足: 論理レプリケーションが有効でない環境での作成時に警告。

保守担当者向けメモ

  • 本関数は論理レプリケーションの定義の起点となるため、カタログ更新後のキャッシュ無効化処理が重要である。
  • パーティションテーブルを含むパブリケーション作成時の挙動（pubviaroot）については、関連する PublicationAddTables 等のロジックと整合性を保つこと。
