mySrand

定義

  • 種別: function
  • 定義箇所: contrib/bloom/blutils.c:253

目的

擬似乱数生成器の状態を初期化し、シード値を設定します。PostgreSQL 標準の乱数生成器とは独立した実装であり、インデックスのオンディスク構造の再現性を確保するために使用されます。

シグネチャまたは形状

static void mySrand(uint32 seed)

入力と出力

  • **入力**: uint32 seed（乱数生成器の初期化に使用するシード値）
  • **出力**: なし（副作用として静的変数 next を更新）

前提条件と不変条件

  • 内部状態 next を [1, 0x7ffffffe] の範囲内に収める不変条件を持ちます。

アルゴリズムまたは振る舞い

1. 引数 seed を静的変数 next に代入します。 2. next を 0x7ffffffe で除算した余りに 1 を加算することで、値を [1, 0x7ffffffe] の範囲に変換・正規化します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/bloom/blutils.c 内の signValue 関数（276行目および284行目）から呼び出されます。
  • **呼び出し先**:
  • 直接的な関数呼び出しはありません。

関連データ構造

  • next: mySrand および myRand が共有する静的変数（型はソースコード上の文脈から uint32 と推測される）。

失敗モードと境界条件

  • seed が 0 の場合でも、計算式 (0 % 0x7ffffffe) + 1 により、next は 1 に初期化されます。

保守担当者向けメモ

  • 本関数は myRand と対で使用されます。
  • 乱数生成アルゴリズムの変更は、Bloom インデックスの署名生成ロジックに影響を与え、既存のインデックスとの互換性を損なう可能性があるため注意してください。
