ParallelSlotsAdoptConn

定義

  • 種別: function
  • 定義箇所: src/fe_utils/parallel_slot.c:459

目的

既存のデータベース接続（PGconn）を ParallelSlotArray 内の空きスロットに割り当て、管理下に置くために使用されます。

シグネチャまたは形状

void ParallelSlotsAdoptConn(ParallelSlotArray *sa, PGconn *conn)

入力と出力

  • **入力**:
  • sa: 接続を管理する ParallelSlotArray 構造体へのポインタ。
  • conn: 割り当て対象となる既存の PGconn オブジェクト。
  • **出力**: なし（void）。

前提条件と不変条件

  • sa は ParallelSlotsSetup によって初期化済みである必要があります。
  • conn は有効なデータベース接続である必要があります。

アルゴリズムまたは振る舞い

1. find_unconnected_slot(sa) を呼び出し、接続が割り当てられていないスロットを検索します。 2. 空きスロットが見つかった場合（offset >= 0）、そのスロットの connection フィールドに conn を格納します。 3. 空きスロットが見つからなかった場合、disconnectDatabase(conn) を呼び出して渡された接続を破棄します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/bin/pg_amcheck/pg_amcheck.c
  • src/bin/scripts/reindexdb.c
  • src/bin/scripts/vacuumdb.c
  • **呼び出し先**:
  • find_unconnected_slot (内部関数)
  • disconnectDatabase (外部関数)

関連データ構造

  • ParallelSlotArray: 複数の ParallelSlot を保持する配列構造体。
  • PGconn: libpq のデータベース接続ハンドル。

失敗モードと境界条件

  • ParallelSlotArray 内に空きスロットが存在しない場合、引数として渡された conn は即座に切断・解放されます。

保守担当者向けメモ

  • 本関数は、外部で確立された接続を並列処理フレームワークに統合する際に使用されます。
  • 接続の所有権が ParallelSlotArray に移転されるため、呼び出し元は以降その接続を直接操作すべきではありません。
