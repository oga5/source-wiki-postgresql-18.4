datapagemap_iterator

定義

  • 種別: struct
  • 定義箇所: src/bin/pg_rewind/datapagemap.c:18

目的

datapagemap_t 構造体に記録された変更済みデータブロックを順次走査するためのイテレータ状態を保持します。

シグネチャまたは形状

struct datapagemap_iterator
{
    datapagemap_t *map;
    BlockNumber nextblkno;
};

入力と出力

  • 入力: datapagemap_iterate() 関数によって初期化されます。
  • 出力: datapagemap_next() 関数がこの構造体を参照し、次の変更済みブロック番号を返します。

前提条件と不変条件

  • map フィールドは有効な datapagemap_t 構造体を指している必要があります。
  • nextblkno は走査中の現在のブロック位置を保持します。

アルゴリズムまたは振る舞い

  • datapagemap_iterate() により生成され、datapagemap_next() による反復処理の内部状態として使用されます。
  • 呼び出し側は、使用終了後に pg_free() を用いてこの構造体を明示的に解放する必要があります。

呼び出し元と呼び出し先

  • 呼び出し元: src/bin/pg_rewind/datapagemap.h:21 にて datapagemap_iterator_t として型定義されています。
  • 呼び出し先: 具体的な関数呼び出しの連鎖については、提供されたソース情報からは特定できません。

関連データ構造

  • datapagemap_t: 変更済みブロックを保持するビットマップ構造体。
  • datapagemap_iterator_t: 本構造体の別名（typedef）。

失敗モードと境界条件

  • イテレータの解放を怠った場合、メモリリークが発生します。
  • map が指すデータ構造が反復処理中に変更された場合の動作は定義されていません。

保守担当者向けメモ

  • 本構造体は非常に軽量な設計であり、pg_rewind における変更済みブロックの走査という単一の目的に特化しています。
  • メモリ管理は呼び出し側の責任である点に注意してください。
