datum_image_eq

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/datum.c:270

目的

2つの Datum 値が、論理的な意味ではなく、メモリ上のバイナリイメージとして同一であるかを判定します。

シグネチャまたは形状

bool datum_image_eq(Datum value1, Datum value2, bool typByVal, int typLen)

入力と出力

  • **入力**:
  • value1, value2: 比較対象の Datum。
  • typByVal: 値が値渡し（true）か参照渡し（false）かを示すフラグ。
  • typLen: 型の長さ（バイト数）。-1 は可変長、-2 は C 文字列を指す。
  • **出力**:
  • bool: 両者のバイナリ表現が一致すれば true、そうでなければ false。

前提条件と不変条件

  • 呼び出し元は、対象となる型の正しい typByVal および typLen を提供する必要があります。誤った値を提供した場合、メモリ破壊や不正なポインタ参照が発生します。

アルゴリズムまたは振る舞い

型情報に基づき以下のいずれかの手法で比較を行います。 1. **typByVal が true の場合**:

  • typLen に応じて char, int16, int32 としてキャストし直接比較します。それ以外は Datum 値そのものを比較します。

    • 2. **typLen > 0 の場合**:

  • memcmp を使用し、指定されたバイト数分だけバイナリ比較を行います。

    • 3. **typLen == -1 (可変長) の場合**:

  • toast_raw_datum_size でサイズを比較し、一致する場合のみ PG_DETOAST_DATUM_PACKED で展開（必要に応じて）した上で memcmp を行います。一時的に確保したメモリは pfree で解放します。

    • 4. **typLen == -2 (C文字列) の場合**:

  • strlen で長さを取得し、一致する場合のみ memcmp で比較します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtutils.c
  • src/backend/executor/nodeMemoize.c
  • src/backend/utils/adt/ri_triggers.c
  • src/backend/utils/adt/rowtypes.c
  • **呼び出し先**:
  • toast_raw_datum_size, PG_DETOAST_DATUM_PACKED, pfree, memcmp, strlen, elog 等の内部ユーティリティ。

関連データ構造

  • Datum: PostgreSQL の汎用データ型表現。
  • varlena: 可変長データ型の内部構造体。

失敗モードと境界条件

  • typLen が -1, -2, および正の整数以外の場合、elog(ERROR) により処理が中断されます。
  • TOAST されたデータに対しては、内部的に展開処理が行われます。

保守担当者向けメモ

  • 本関数は「バイナリとしての同一性」を保証するものであり、論理的に等価であっても内部表現（TOAST 状態やパディング等）が異なる場合は false を返す可能性があります。
  • datum_image_hash と対になる関数として設計されています。
