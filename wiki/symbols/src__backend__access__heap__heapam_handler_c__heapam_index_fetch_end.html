heapam_index_fetch_end

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam_handler.c:104

目的

インデックススキャンに関連するヒープアクセス用のリソースを解放し、メモリをクリーンアップします。

シグネチャまたは形状

static void heapam_index_fetch_end(IndexFetchTableData *scan)

入力と出力

  • **入力**: scan (IndexFetchTableData *) - 解放対象のインデックススキャン状態データ。
  • **出力**: なし。

前提条件と不変条件

  • scan は IndexFetchHeapData 構造体としてキャスト可能である必要があります。
  • 本関数呼び出し後、scan が指すメモリ領域および関連するバッファ状態は無効となります。

アルゴリズムまたは振る舞い

1. scan を IndexFetchHeapData * 型の hscan にキャストします。 2. heapam_index_fetch_reset(scan) を呼び出し、スキャンに関連付けられたバッファの解放等のリセット処理を実行します。 3. pfree(hscan) を呼び出し、スキャン状態を保持していたメモリ領域を解放します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/heap/heapam_handler.c 内の TableAmRoutine 構造体（heapam_methods）の .index_fetch_end フィールドとして登録されています。
  • **呼び出し先**:
  • heapam_index_fetch_reset
  • pfree

関連データ構造

  • IndexFetchTableData: テーブルAMの汎用的なインデックススキャンデータ構造。
  • IndexFetchHeapData: ヒープAM固有のインデックススキャン状態を保持する構造体。

失敗モードと境界条件

  • scan が NULL の場合、pfree およびキャスト処理により未定義動作が発生する可能性があります。

保守担当者向けメモ

  • 本関数は TableAmRoutine のコールバックとして実装されています。インデックススキャン終了時のメモリリークを防ぐため、heapam_index_fetch_begin で確保されたリソースが heapam_index_fetch_reset で適切に解放されることを保証してください。
