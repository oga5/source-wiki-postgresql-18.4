policy_role_list_to_array

定義

  • 種別: function
  • 定義箇所: src/backend/commands/policy.c:136

目的

ポリシー定義に含まれるロールリスト（List）を、システムカタログ pg_policy で扱われる Datum 型の OID 配列に変換する。

シグネチャまたは形状

Datum *policy_role_list_to_array(List *roles, int *num_roles)

入力と出力

  • **入力**:
  • roles: RoleSpec 構造体のリスト。
  • num_roles: 変換後の配列要素数を格納するための出力用ポインタ。
  • **出力**:
  • Datum *: 割り当てられた OID 配列へのポインタ。

前提条件と不変条件

  • roles が NIL の場合、num_roles に 1 が設定され、ACL_ID_PUBLIC を含む単一要素の配列が返される。
  • roles 内に ROLESPEC_PUBLIC が含まれる場合、それが唯一の要素であるとみなされる。他のロールと混在している場合は警告が発行され、num_roles は 1 に調整される。

アルゴリズムまたは振る舞い

1. roles が空の場合、ACL_ID_PUBLIC を持つ配列を生成して返す。 2. roles が空でない場合、リストの長さに応じてメモリを確保する。 3. リストを走査し、各 RoleSpec を get_rolespec_oid で OID に変換する。 4. ROLESPEC_PUBLIC が検出された場合、他のロール指定を無視して ACL_ID_PUBLIC のみを格納した配列を即座に返す。 5. 変換した OID を配列に格納し、最後に配列ポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/policy.c 内の CreatePolicy および AlterPolicy 関数から呼び出される。
  • **呼び出し先**: get_rolespec_oid を呼び出してロール名を OID に解決する。

関連データ構造

  • List / ListCell: PostgreSQL のリスト構造。
  • RoleSpec: ロール指定を保持する構造体。
  • Datum: PostgreSQL の汎用データ型。

失敗モードと境界条件

  • get_rolespec_oid が失敗した場合、例外が発生する可能性がある。
  • ROLESPEC_PUBLIC が他のロールと混在している場合、WARNING レベルのメッセージがログに出力され、結果は PUBLIC のみに限定される。

保守担当者向けメモ

  • 本関数は palloc を使用してメモリを確保するため、呼び出し側で適切なメモリコンテキスト管理が行われていることを確認すること。
  • pg_policy カタログの polroles 列に格納するデータの準備に使用される。
