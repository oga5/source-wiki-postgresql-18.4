GetBulkInsertState

定義

  • 種別: function
  • 定義箇所: src/backend/access/heap/heapam.c:2012

目的

バルク挿入操作で使用する状態管理オブジェクト（BulkInsertState）を生成および初期化する。

シグネチャまたは形状

BulkInsertState GetBulkInsertState(void);

入力と出力

  • **入力**: なし
  • **出力**: 初期化済みの BulkInsertState 構造体へのポインタ。

前提条件と不変条件

  • palloc を使用してメモリを確保するため、メモリ不足時にはエラーが発生する可能性がある。
  • 返される BulkInsertState は、BAS_BULKWRITE（バルク書き込み）戦略で初期化される。

アルゴリズムまたは振る舞い

1. BulkInsertStateData 構造体分のメモリを palloc で確保する。 2. GetAccessStrategy(BAS_BULKWRITE) を呼び出し、バルク書き込み用のアクセス戦略を取得して strategy フィールドに格納する。 3. 以下のフィールドを初期値で設定する:

  • current_buf = InvalidBuffer
  • next_free = InvalidBlockNumber
  • last_free = InvalidBlockNumber
  • already_extended_by = 0

    • 4. 初期化済みのポインタを返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/commands/copyfrom.c
  • src/backend/commands/createas.c
  • src/backend/commands/matview.c
  • src/backend/commands/tablecmds.c
  • **呼び出し先**:
  • palloc
  • GetAccessStrategy

関連データ構造

  • BulkInsertStateData: バルク挿入の状態を保持する構造体。
  • Buffer: current_buf で使用されるバッファ識別子。
  • BlockNumber: next_free および last_free で使用されるブロック番号。

失敗モードと境界条件

  • メモリ割り当て失敗時に palloc がエラーをスローする。
  • GetAccessStrategy が失敗した場合の挙動は当該関数の仕様に依存する。

保守担当者向けメモ

  • 本関数で生成されたオブジェクトは、使用後に FreeBulkInsertState で解放する必要がある。
  • BAS_BULKWRITE 戦略は、大量のデータ挿入時にバッファリング効率を最適化するために使用される。
