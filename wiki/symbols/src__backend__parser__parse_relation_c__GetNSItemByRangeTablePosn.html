GetNSItemByRangeTablePosn

定義

  • 種別: function
  • 定義箇所: src/backend/parser/parse_relation.c:517

目的

指定されたレンジテーブルインデックス（varno）とネストレベル（sublevels_up）に対応する ParseNamespaceItem を、ParseState の名前空間リストから検索して取得する。

シグネチャまたは形状

ParseNamespaceItem *GetNSItemByRangeTablePosn(ParseState *pstate, int varno, int sublevels_up)

入力と出力

  • **入力**:
  • pstate: 現在の解析状態を保持する ParseState 構造体へのポインタ。
  • varno: 検索対象のレンジテーブルインデックス。
  • sublevels_up: 検索対象のネストレベル（0は現在のクエリ、1以上は外側のクエリ）。
  • **出力**:
  • 該当する ParseNamespaceItem 構造体へのポインタ。

前提条件と不変条件

  • sublevels_up が指定された場合、pstate->parentParseState を辿って対象のスコープまで移動する。
  • 検索対象の pstate は NULL であってはならない（Assert により検証）。

アルゴリズムまたは振る舞い

1. sublevels_up の値に従い、pstate を親方向へ遡る。 2. 到達した pstate の p_namespace リストを走査する。 3. リスト内の各 ParseNamespaceItem の p_rtindex が varno と一致するか確認する。 4. 一致するアイテムが見つかればそれを返す。 5. リストの最後まで見つからない場合は elog(ERROR) を発生させ、処理を中断する。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/parser/parse_coerce.c
  • src/backend/parser/parse_func.c
  • src/backend/parser/parse_merge.c
  • src/backend/parser/parse_target.c
  • **呼び出し先**:
  • lfirst (マクロ)
  • elog (エラー報告)

関連データ構造

  • ParseState: 解析状態を保持する構造体。
  • ParseNamespaceItem: 名前空間内のアイテムを表現する構造体。

失敗モードと境界条件

  • 指定された varno が現在の名前空間リストに存在しない場合、内部エラーとして elog(ERROR) が発生する。
  • sublevels_up が深すぎて pstate が NULL になる場合、Assert によりデバッグビルドで停止する。

保守担当者向けメモ

  • 本関数は、クエリの解析中に特定のレンジテーブルエントリ（RTE）に関連付けられた名前空間情報を取得するための標準的な手段である。
  • エラー発生時は「nsitem not found」という内部エラーとなるため、呼び出し側で varno が正しく解決されていることを保証する必要がある。
