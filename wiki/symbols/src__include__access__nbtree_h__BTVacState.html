BTVacState

定義

  • 種別: struct
  • 定義箇所: src/include/access/nbtree.h:331

目的

B-tree インデックスのバキューム処理（削除およびクリーンアップ）における実行状態を保持する構造体です。

シグネチャまたは形状

typedef struct BTVacState
{
    IndexVacuumInfo *info;
    IndexBulkDeleteResult *stats;
    IndexBulkDeleteCallback callback;
    void       *callback_state;
    BTCycleId   cycleid;
    MemoryContext pagedelcontext;
    int         bufsize;
    int         maxbufsize;
    BTPendingFSM *pendingpages;
    int         npendingpages;
} BTVacState;

入力と出力

  • **入力**: バキューム対象のインデックス情報、統計情報、削除判定用コールバック関数、およびメモリコンテキスト。
  • **出力**: 処理結果として stats（IndexBulkDeleteResult）が更新され、削除されたページ情報が pendingpages に蓄積されます。

前提条件と不変条件

  • pendingpages は BTPendingFSM 型の配列であり、bufsize は確保された要素数、npendingpages は現在有効な要素数を示します。
  • maxbufsize は work_mem の制限を遵守するように設定される必要があります。

アルゴリズムまたは振る舞い

1. **初期化**: _bt_pendingfsm_init を通じてバキューム状態が初期化されます。 2. **ページ処理**: btvacuumpage や _bt_pagedel を介してインデックスページを走査し、不要なタプルの削除やページ削除の判定を行います。 3. **保留処理**: 削除対象となったページは _bt_pendingfsm_add により pendingpages に追加されます。 4. **終了処理**: _bt_pendingfsm_finalize により、保留されていたページ情報が処理され、インデックスのクリーンアップが完了します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/nbtree/nbtree.c（btvacuumpage 等）および src/backend/access/nbtree/nbtpage.c（_bt_pagedel 等）から参照されます。
  • **呼び出し先**: IndexBulkDeleteCallback を介して外部のコールバック関数を呼び出します。

関連データ構造

  • IndexVacuumInfo, IndexBulkDeleteResult: バキュームのメタデータと結果。
  • BTPendingFSM: 削除保留中のページ情報を保持する構造体。

失敗モードと境界条件

  • maxbufsize を超えるページ削除が発生した場合、メモリ管理上の制約を受ける可能性があります。
  • callback が NULL の場合や、無効な MemoryContext が渡された場合の動作は未定義です。

保守担当者向けメモ

  • BTVacState は nbtree のバキューム処理における中心的なコンテキストです。
  • pendingpages の動的確保と work_mem 制限の整合性については、nbtpage.c の実装ロジックを詳細に確認してください。
