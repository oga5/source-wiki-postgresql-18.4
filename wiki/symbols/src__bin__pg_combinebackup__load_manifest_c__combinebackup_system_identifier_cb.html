combinebackup_system_identifier_cb

定義

  • 種別: function
  • 定義箇所: src/bin/pg_combinebackup/load_manifest.c:254

目的

バックアップマニフェストのパース処理中に、システム識別子（system identifier）を抽出し、メモリ上の manifest_data 構造体に格納するコールバック関数です。

シグネチャまたは形状

static void combinebackup_system_identifier_cb(JsonManifestParseContext *context, uint64 manifest_system_identifier)

入力と出力

  • **入力**:
  • context: パース状態を保持する JsonManifestParseContext へのポインタ。private_data メンバを通じて manifest_data 構造体を参照します。
  • manifest_system_identifier: マニフェストから読み取られた 64 ビットのシステム識別子。
  • **出力**: なし（void）。

前提条件と不変条件

  • context->private_data が有効な manifest_data 構造体を指している必要があります。
  • この関数は、マニフェストのパース処理中にコールバックとして呼び出されることを前提としています。

アルゴリズムまたは振る舞い

1. context->private_data から manifest_data 構造体のポインタを取得します。 2. 引数として渡された manifest_system_identifier を、manifest->system_identifier に代入します。 3. 検証処理は本関数内では行わず、後続のフェーズに委ねます。

呼び出し元と呼び出し先

  • **呼び出し元**: マニフェスト解析エンジン（common/parse_manifest.c 経由）。
  • **呼び出し先**: なし。
  • **設定箇所**: src/bin/pg_combinebackup/load_manifest.c 内で、JsonManifestParseContext の system_identifier_cb フィールドに本関数が登録されます。

関連データ構造

  • JsonManifestParseContext: パースのコンテキストを保持する構造体。
  • manifest_data: バックアップマニフェストの情報を保持する構造体（load_manifest.h で定義）。

失敗モードと境界条件

  • 本関数自体は単純な代入を行うため、特有の失敗モードはありません。ただし、context が無効な場合はセグメンテーション違反が発生する可能性があります。

保守担当者向けメモ

  • 本関数は検証を行わない設計です。システム識別子の整合性チェックが必要な場合は、呼び出し元のパース完了後、あるいは manifest_data を利用する段階で実装してください。
