ExecCopySlotMinimalTupleExtra

定義

  • 種別: function
  • 定義箇所: src/include/executor/tuptable.h:507

目的

TupleTableSlot が保持するタプルを MinimalTuple 形式でコピーし、呼び出し元が指定した追加領域（extra）を付与して返します。

シグネチャまたは形状

static inline MinimalTuple ExecCopySlotMinimalTupleExtra(TupleTableSlot *slot, Size extra)

入力と出力

  • **入力**:
  • slot: 対象となる TupleTableSlot へのポインタ。
  • extra: コピー時に確保する追加メモリサイズ（バイト単位）。
  • **出力**:
  • 成功時: 確保されたメモリ領域を含む MinimalTuple へのポインタ。

前提条件と不変条件

  • slot は有効な TupleTableSlot を指している必要があります。
  • slot->tts_ops が copy_minimal_tuple 関数ポインタを実装している必要があります。

アルゴリズムまたは振る舞い

1. 指定された slot の tts_ops（TupleTableSlotOps）構造体を参照します。 2. tts_ops->copy_minimal_tuple 関数を呼び出し、slot の内容を MinimalTuple 形式に変換・コピーします。 3. その際、引数 extra で指定されたサイズの追加領域を確保します。 4. 生成された MinimalTuple を呼び出し元に返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execGrouping.c:504 にて使用されています。
  • **呼び出し先**: slot->tts_ops->copy_minimal_tuple を介して、各スロット型（MinimalTupleTableSlot 等）固有の実装が呼び出されます。

関連データ構造

  • TupleTableSlot: タプルを保持する抽象構造体。
  • TupleTableSlotOps: スロットの操作を定義する仮想関数テーブル。
  • MinimalTuple: PostgreSQL の最小化されたタプル形式。

失敗モードと境界条件

  • メモリ不足時には、基盤となるメモリ割り当て関数によりエラーが発生する可能性があります。
  • slot が空（TTS_EMPTY）の場合の挙動は、各 tts_ops の実装に依存します。

保守担当者向けメモ

  • 本関数はインライン関数であり、TupleTableSlotOps を通じた動的なディスパッチを行います。
  • extra 引数は、特定のコンテキストでタプルに付随するメタデータや追加情報を格納するために使用されます。
