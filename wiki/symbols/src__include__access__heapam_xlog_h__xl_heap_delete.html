xl_heap_delete

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/heapam_xlog.h:113

目的

ヒープテーブルにおけるタプル削除操作をWAL（Write-Ahead Logging）に記録するためのデータ構造です。リカバリ時に特定のページ上のタプルを削除状態へ遷移させるために使用されます。

シグネチャまたは形状

typedef struct xl_heap_delete
{
    TransactionId xmax;         /* 削除されたタプルのxmax */
    OffsetNumber offnum;        /* 削除されたタプルのオフセット番号 */
    uint8        infobits_set;  /* インフォマスクビット */
    uint8        flags;         /* 操作フラグ */
} xl_heap_delete;

入力と出力

  • **入力**: ヒープ操作時に heapam.c によって生成され、WALレコードのデータ部として書き込まれます。
  • **出力**: リカバリ時に heap_redo 関数が XLogRecGetData を介してこの構造体を読み取り、対象ページのタプル状態を更新します。

前提条件と不変条件

  • flags フィールドには、src/include/access/heapam_xlog.h:99 で定義される8ビットのフラグ値が使用可能です。
  • SizeOfHeapDelete マクロ（offsetof(xl_heap_delete, flags) + sizeof(uint8)）により、構造体のサイズが定義されています。

アルゴリズムまたは振る舞い

1. **記録**: ヒープ削除操作時に、削除対象のタプル情報（xmax, offnum 等）を本構造体に格納し、XLogInsert を通じてWALへ書き出します。 2. **再生**: リカバリ時、heap_redo がWALレコードをデコードして本構造体を取得します。その後、指定されたオフセットのタプルに対して xmax の更新やインフォマスクの適用を行い、ページを再構築します。

呼び出し元と呼び出し先

  • **呼び出し元（生成側）**: src/backend/access/heap/heapam.c 内の削除処理ロジック。
  • **呼び出し先（消費側）**: src/backend/access/heap/heapam_xlog.c（heap_redo）、src/backend/access/rmgrdesc/heapdesc.c（デコード表示）、src/backend/replication/logical/decode.c（論理デコード）。

関連データ構造

  • OffsetNumber: ページ内のタプル位置を特定する型。
  • TransactionId: 削除を実行したトランザクションID。
  • XLogRecGetData: WALレコードから本構造体を取り出すためのインターフェース。

失敗モードと境界条件

  • 構造体のサイズやフィールドの配置は、SizeOfHeapDelete マクロに依存します。
  • ページ内のオフセットが不正な場合、リカバリ処理中にページアクセスエラーが発生する可能性があります。

保守担当者向けメモ

  • 本構造体は xl_heap_update や xl_heap_lock と同様の infobits フィールドを保持しており、これらは一貫して扱われる必要があります。
  • 構造体の変更時は、heapdesc.c のデコードロジックおよび heapam_xlog.c のREDOロジックとの整合性を確認してください。
