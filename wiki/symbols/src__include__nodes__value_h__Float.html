Float

定義

  • 種別: struct
  • 定義箇所: src/include/nodes/value.h:47

目的

SQLクエリの構文解析時に出現する浮動小数点数リテラルを、構文解析ツリー内で保持するためのノード構造体です。

シグネチャまたは形状

typedef struct Float
{
	pg_node_attr(special_read_write)

	NodeTag		type;
	char	   *fval;
} Float;

入力と出力

  • **入力**: fval フィールドに浮動小数点数の文字列表現（char*）を格納します。
  • **出力**: castNode(Float, node)->fval を通じて、保持している文字列値を取り出します。

前提条件と不変条件

  • type フィールドには、PostgreSQLのノードシステムで定義された適切な NodeTag が設定されている必要があります。
  • fval が指す文字列は、呼び出し側で palloc されたメモリ領域である必要があります。
  • 数値は double 型ではなく char* として保持され、解析段階での精度維持が保証されます。

アルゴリズムまたは振る舞い

  • パーサーが字句解析で浮動小数点数リテラルを検出した際、makeFloat 関数によってヒープ上に生成されます。
  • 構文解析ツリー（List 等）に格納され、意味解析や計画作成フェーズまで保持されます。
  • 必要に応じて、後続の処理で NUMERIC 型等へ変換されます。

呼び出し元と呼び出し先

  • **呼び出し元（参照箇所）**:
  • src/backend/commands/define.c: リテラル値の抽出に使用。
  • src/backend/commands/tsearchcmds.c: 型判定に使用。
  • src/backend/nodes/outfuncs.c: ノードのシリアライズに使用。
  • src/backend/nodes/readfuncs.c: ノードのデシリアライズに使用。
  • **呼び出し先**:
  • makeNode (マクロ経由でメモリ確保に使用)

関連データ構造

  • NodeTag: ノードの型識別子。
  • List: Float ノードを格納するために使用されるリスト構造。

失敗モードと境界条件

  • fval が指す文字列が有効な浮動小数点形式でない場合、後続の変換処理でエラーが発生する可能性があります。
  • メモリ不足時には palloc がエラーを発生させます。

保守担当者向けメモ

  • 本構造体は double ではなく文字列で値を保持している点に注意してください。
  • src/backend/nodes/value.c にあるコンストラクタ関数 makeFloat が生成の起点となります。
