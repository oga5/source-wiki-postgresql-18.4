tsquery_or

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsquery_op.c:83

目的

2つの tsquery 型の値を論理和（OR）演算子で結合し、新しい tsquery を生成します。

シグネチャまたは形状

Datum tsquery_or(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して2つの TSQuery 型引数を受け取ります。
  • **出力**: Datum 型として、結合された結果の TSQuery を返します。

前提条件と不変条件

  • 入力された TSQuery は PG_GETARG_TSQUERY_COPY によりコピーとして取得されます。
  • 演算終了後、一時的に生成された QTNode ツリーおよび入力された TSQuery のコピーは適切に解放される必要があります。

アルゴリズムまたは振る舞い

1. **空クエリの判定**:

  • 引数 a が空（size == 0）の場合、引数 b をそのまま返します。
  • 引数 b が空の場合、引数 a をそのまま返します。

    • 2. **ツリー構築**: join_tsqueries を呼び出し、OP_OR を用いて2つの tsquery を QTNode ツリー構造に結合します。 3. **シリアライズ**: QTN2QT を使用して、構築したツリーを TSQuery 型のバイト列に変換します。 4. **メモリ解放**: QTNFree でツリー構造を解放し、PG_FREE_IF_COPY で入力引数のコピーを解放します。

呼び出し元と呼び出し先

  • **呼び出し元**: SQL レベルの | 演算子から呼び出されます（直接的な呼び出し元情報はソースコード内に明示されていません）。
  • **呼び出し先**:
  • join_tsqueries (ツリー結合)
  • QTN2QT (ツリーから TSQuery への変換)
  • QTNFree (ツリーのメモリ解放)

関連データ構造

  • TSQuery: 全文検索クエリの内部表現。
  • QTNode: 演算処理用の中間ツリー構造。

失敗モードと境界条件

  • 入力引数のいずれかが空の場合、演算を行わずに他方の引数を返す最適化が行われます。
  • メモリ管理において、PG_RETURN_TSQUERY を実行する前に QTNFree および PG_FREE_IF_COPY が正しく呼び出されない場合、メモリリークが発生する可能性があります。

保守担当者向けメモ

  • 本関数は tsquery の論理演算の標準的な実装パターンに従っています。新しい演算子を追加する際は、join_tsqueries の利用とメモリ解放の順序を本関数に倣ってください。
