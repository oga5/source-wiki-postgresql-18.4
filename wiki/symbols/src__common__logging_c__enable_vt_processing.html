enable_vt_processing

定義

  • 種別: function
  • 定義箇所: src/common/logging.c:53

目的

Windows環境において、標準エラー出力（stderr）に対する仮想端末（VT100）エスケープシーケンスの処理を有効化する。

シグネチャまたは形状

static bool enable_vt_processing(void)

入力と出力

  • 入力: なし
  • 出力: 成功した場合は true、失敗または対象外の場合は false を返す。

前提条件と不変条件

  • Windows環境での実行を想定している。
  • GetStdHandle(STD_ERROR_HANDLE) が有効なハンドルを返す必要がある。

アルゴリズムまたは振る舞い

1. GetStdHandle を用いて標準エラー出力のハンドルを取得する。 2. GetConsoleMode を呼び出し、現在のコンソール設定を確認する。 3. 既に ENABLE_VIRTUAL_TERMINAL_PROCESSING が設定されている場合は、何もせず true を返す。 4. 設定されていない場合、ENABLE_VIRTUAL_TERMINAL_PROCESSING フラグを付与し、SetConsoleMode を呼び出して設定を更新する。 5. 更新が成功すれば true を、失敗すれば false を返す。

呼び出し元と呼び出し先

  • 呼び出し元: src/common/logging.c 内の pg_logging_init（color_terminal 変数の初期化時）。
  • 呼び出し先: Windows API (GetStdHandle, GetConsoleMode, SetConsoleMode)。

関連データ構造

  • HANDLE (Windows API)
  • DWORD (Windows API)

失敗モードと境界条件

  • GetStdHandle が INVALID_HANDLE_VALUE を返す場合、false を返す。
  • GetConsoleMode または SetConsoleMode が失敗した場合、false を返す。

保守担当者向けメモ

  • 本関数はWindowsプラットフォーム固有の処理である。
  • ログ出力の色付け制御において、端末がVT100エスケープシーケンスをサポートしているかを判断する重要な役割を担う。
