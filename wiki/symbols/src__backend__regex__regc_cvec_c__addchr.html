addchr

定義

  • 種別: function
  • 定義箇所: src/backend/regex/regc_cvec.c:81

目的

正規表現コンパイル時に使用される文字セット（cvec）に対し、個別の文字（chr）を追加するために使用されます。

シグネチャまたは形状

static void addchr(struct cvec *cv, chr c)

入力と出力

  • **入力**:
  • cv: 文字を追加する対象の cvec 構造体へのポインタ。
  • c: 追加する文字（chr 型）。
  • **出力**: なし（戻り値なし）。

前提条件と不変条件

  • cv が指す cvec 構造体において、現在格納されている文字数（nchrs）が、確保されている容量（chrspace）未満である必要があります。この条件は assert によって検証されます。

アルゴリズムまたは振る舞い

1. cv->nchrs が cv->chrspace より小さいことを確認します。 2. cv->chrs 配列の現在の末尾（nchrs が指す位置）に文字 c を書き込みます。 3. cv->nchrs をインクリメントし、格納済み文字数を更新します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/regex/regc_locale.c 内の各所（例: 465, 476, 511, 512, 515, 516, 526, 617, 618, 728 行目）。
  • **呼び出し先**: なし。

関連データ構造

  • struct cvec: regguts.h で定義される文字セットを保持する構造体。chrs 配列、nchrs（現在の要素数）、chrspace（最大容量）を保持します。

失敗モードと境界条件

  • **境界条件**: nchrs が chrspace に達している状態で呼び出された場合、assert によりプログラムが停止します。

保守担当者向けメモ

  • 本関数は regcomp.c にインクルードされる regc_cvec.c 内で static 関数として定義されており、コンパイル単位内での最適化を前提としています。
  • cvec 内部の配列を直接操作する設計となっているため、呼び出し元は必ず十分な容量が確保されていることを保証する必要があります。
