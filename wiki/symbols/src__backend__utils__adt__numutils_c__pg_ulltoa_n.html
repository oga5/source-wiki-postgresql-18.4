pg_ulltoa_n

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/numutils.c:1139

目的

符号なし64ビット整数（uint64）を文字列に変換し、その文字数を返します。

シグネチャまたは形状

int pg_ulltoa_n(uint64 value, char *a)

入力と出力

  • **入力**:
  • value: 変換対象の uint64 値。
  • a: 変換結果を格納するバッファへのポインタ。
  • **出力**:
  • 戻り値: 変換された文字列の長さ（文字数）。
  • 副作用: a が指すメモリ領域に、NUL終端を含まない数値文字列が書き込まれます。

前提条件と不変条件

  • a は、変換後の数値文字列を格納するのに十分なサイズのバッファを指している必要があります。
  • 本関数は NUL 終端を書き込みません。呼び出し側で必要に応じて終端処理を行う必要があります。

アルゴリズムまたは振る舞い

1. value が 0 の場合、'0' を書き込み 1 を返します。 2. decimalLength64 を用いて変換後の長さを算出します。 3. value が 100,000,000 以上の場合、ループ内で DIGIT_TABLE を参照し、一度に8桁ずつ文字列バッファへ memcpy で書き込みます。 4. 残りの桁数に対して、32ビット整数へのキャストを行い、同様に DIGIT_TABLE を用いて2桁ずつ効率的に書き込みます。 5. 最後に、残った1桁を文字に変換して書き込みます。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tcop/cmdtag.c:154
  • src/backend/utils/adt/numutils.c:1238
  • **呼び出し先**:
  • decimalLength64 (内部関数)
  • memcpy (標準ライブラリ)

関連データ構造

  • DIGIT_TABLE: 00〜99 の数値ペアを保持するルックアップテーブル。高速な文字列化のために使用されます。

失敗モードと境界条件

  • 入力が 0 の場合、'0' を書き込み 1 を返します。
  • バッファサイズが不足している場合、メモリ破壊が発生します（呼び出し側の責任）。

保守担当者向けメモ

  • 本関数はパフォーマンスを重視しており、標準ライブラリの sprintf よりも高速です。
  • DIGIT_TABLE を利用した最適化が行われているため、変更時はテーブルの整合性に注意してください。
  • NUL 終端を行わない仕様であるため、バッファ操作を行う際は常に長さを考慮してください。
