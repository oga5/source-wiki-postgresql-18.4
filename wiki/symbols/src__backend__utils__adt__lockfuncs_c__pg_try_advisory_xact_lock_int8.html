pg_try_advisory_xact_lock_int8

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/lockfuncs.c:713

目的

ユーザー指定の64ビット整数キーを使用して、トランザクションスコープのアドバイザリーロックを非ブロック的に取得する。

シグネチャまたは形状

Datum pg_try_advisory_xact_lock_int8(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: SQL関数引数として int64 型のキーを1つ受け取る。
  • **出力**: ロック取得に成功した場合は true、既に他のセッションによってロックされている等の理由で取得できなかった場合は false を返す。

前提条件と不変条件

  • ロックは現在のトランザクションに紐付く。トランザクション終了時に自動的に解放される。
  • キーは MyDatabaseId と組み合わされてロックタグが生成されるため、データベースを跨いだロックは行えない。

アルゴリズムまたは振る舞い

1. PG_GETARG_INT64(0) で入力キーを取得する。 2. SET_LOCKTAG_INT64 マクロを使用して、キーを LOCKTAG 構造体に変換する。 3. LockAcquire 関数を呼び出し、ExclusiveLock（排他ロック）を要求する。この際、dontWait 引数を true に設定し、ロックが即座に取得できない場合は待機せずに制御を戻す。 4. LockAcquire の戻り値が LOCKACQUIRE_NOT_AVAIL でないことを確認し、その結果をブール値として返す。

呼び出し元と呼び出し先

  • **呼び出し元**: SQLインターフェース経由で呼び出される（直接的なC言語レベルの呼び出し元情報は提供されていない）。
  • **呼び出し先**:
  • LockAcquire (PostgreSQLロックマネージャのコア関数)

関連データ構造

  • LOCKTAG: ロック対象を識別するための構造体。
  • LockAcquireResult: LockAcquire の実行結果を示す列挙型。

失敗モードと境界条件

  • ロックの取得に失敗した場合（既に他者がロックを保持している場合）、エラーにはならず false を返す。
  • メモリ不足や内部的なロックマネージャの制限に抵触した場合は、LockAcquire 内部でエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数はトランザクションスコープのロックを扱うため、セッションスコープの pg_try_advisory_lock_int8 とは管理体系が異なる点に注意すること。
  • ロックマネージャの詳細は src/backend/storage/lmgr/ を参照のこと。
