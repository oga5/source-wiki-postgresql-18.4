json_to_tsvector_worker

定義

  • 種別: function
  • 定義箇所: src/backend/tsearch/to_tsany.c:363

目的

JSON テキストデータを解析し、全文検索用の TSVector 構造体へ変換する内部ワーカー関数です。

シグネチャまたは形状

static TSVector json_to_tsvector_worker(Oid cfgId, text *json, uint32 flags)

入力と出力

  • **入力**:
  • cfgId: 全文検索設定の OID。
  • json: 変換対象の JSON テキストデータ。
  • flags: 処理オプションを指定するビットフラグ。
  • **出力**:
  • 変換結果として構築された TSVector 型のポインタ。

前提条件と不変条件

  • ParsedText 構造体 prs が初期化され、words が NULL に設定されていること。
  • TSVectorBuildState が cfgId と prs を保持して適切に構成されていること。

アルゴリズムまたは振る舞い

1. ParsedText 構造体（prs）を初期化し、TSVectorBuildState（state）に設定します。 2. iterate_json_values を呼び出し、入力された JSON 内の各値を走査します。その際、各値に対して add_to_tsvector コールバックを適用し、トークンを prs に蓄積します。 3. 最後に make_tsvector(&prs) を呼び出し、蓄積されたトークンから最終的な TSVector を構築して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/tsearch/to_tsany.c 内の JSON 関連変換関数（json_to_tsvector 等）から複数回呼び出されています。
  • **呼び出し先**:
  • iterate_json_values: JSON 構造の走査に使用。
  • make_tsvector: 最終的な TSVector 構造体の構築に使用。
  • add_to_tsvector: iterate_json_values のコールバックとして使用（直接の呼び出し先）。

関連データ構造

  • TSVector: 全文検索用のベクトル型。
  • TSVectorBuildState: 変換過程の状態を保持する構造体。
  • ParsedText: 解析されたトークン列を保持する構造体。

失敗モードと境界条件

  • JSON の形式が不正な場合、iterate_json_values 経由でエラーが発生する可能性があります。
  • メモリ不足時には make_tsvector 等の内部関数でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数は JSON の走査と TSVector の構築を仲介する役割です。JSON 構造の抽出ロジックを変更する場合は iterate_json_values の引数やコールバックを、ベクトル構築ロジックを変更する場合は make_tsvector を参照してください。
