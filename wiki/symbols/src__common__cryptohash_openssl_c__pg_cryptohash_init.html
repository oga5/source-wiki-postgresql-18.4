pg_cryptohash_init

定義

  • 種別: function
  • 定義箇所: src/common/cryptohash_openssl.c:177

目的

OpenSSLの EVP_MD_CTX を使用して、指定された暗号学的ハッシュアルゴリズムの計算コンテキストを初期化します。

シグネチャまたは形状

int pg_cryptohash_init(pg_cryptohash_ctx *ctx);

入力と出力

  • **入力**: pg_cryptohash_ctx *ctx（初期化対象のハッシュコンテキストへのポインタ）。
  • **出力**: 成功時に 0、失敗時に -1 を返します。

前提条件と不変条件

  • ctx は NULL であってはなりません。
  • ctx->type には、サポートされているハッシュアルゴリズム（PG_MD5, PG_SHA1, PG_SHA224, PG_SHA256, PG_SHA384, PG_SHA512 のいずれか）が設定されている必要があります。

アルゴリズムまたは振る舞い

1. ctx->type に基づき、対応するOpenSSLの EVP_DigestInit_ex を呼び出してハッシュコンテキストを初期化します。 2. OpenSSLの戻り値が成功（1）以外の場合、ERR_get_error() を用いてエラーメッセージを取得し、ctx->errreason に格納します。 3. エラー発生時は ERR_clear_error() を呼び出し、OpenSSLのエラーキューをクリアします。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/uuid-ossp/uuid-ossp.c
  • src/backend/backup/backup_manifest.c
  • src/backend/libpq/auth-scram.c
  • src/backend/utils/adt/cryptohashfuncs.c
  • src/common/checksum_helper.c
  • src/common/cryptohash.c
  • **呼び出し先**:
  • EVP_DigestInit_ex (OpenSSL)
  • SSLerrmessage
  • ERR_get_error (OpenSSL)
  • ERR_clear_error (OpenSSL)

関連データ構造

  • pg_cryptohash_ctx: ハッシュ計算の状態を保持する構造体。OpenSSLの EVP_MD_CTX を内包します。

失敗モードと境界条件

  • ctx が NULL の場合、-1 を返します。
  • OpenSSLの初期化関数が失敗した場合、ctx->error に PG_CRYPTOHASH_ERROR_OPENSSL が設定され、-1 を返します。
  • FIPSモード等の環境では、初期化時に複数のエラーがキューに積まれる可能性があるため、ERR_clear_error() による全消去が行われます。

保守担当者向けメモ

  • 本関数はOpenSSLがリンクされているビルド環境でのみ有効です。
  • エラー発生時の SSLerrmessage は、OpenSSLの内部エラーコードをPostgreSQLで利用可能な文字列に変換するために使用されます。
