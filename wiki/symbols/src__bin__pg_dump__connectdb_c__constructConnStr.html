constructConnStr

定義

  • 種別: function
  • 定義箇所: src/bin/pg_dump/connectdb.c:244

目的

データベース接続パラメータの配列から、サブプロセスへの引き継ぎに適した接続文字列を生成します。セキュリティ上の理由から、パスワードやデータベース名などの機密情報や特定のパラメータを除外した文字列を作成します。

シグネチャまたは形状

static char *constructConnStr(const char **keywords, const char **values);

入力と出力

  • **入力**:
  • keywords: 接続パラメータ名の文字列配列（NULL 終端）。
  • values: 対応するパラメータ値の文字列配列。
  • **出力**:
  • char *: 生成された接続文字列（呼び出し元で解放が必要）。

前提条件と不変条件

  • keywords および values 配列は NULL で終端されている必要があります。
  • PQExpBuffer を使用してメモリを動的に確保するため、呼び出し元は返された文字列を pg_free 等で適切に管理する必要があります。

アルゴリズムまたは振る舞い

1. createPQExpBuffer でバッファを初期化します。 2. keywords 配列を順次走査します。 3. 以下のパラメータは除外対象としてスキップします。

  • dbname
  • password
  • fallback_application_name

    • 4. 除外対象以外のパラメータについて、key='value' 形式でバッファに追記します。パラメータ間はスペースで区切られます。 5. 最終的な文字列を pg_strdup で複製し、バッファを破棄して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/bin/pg_dump/connectdb.c 内の接続処理フロー（src/bin/pg_dump/connectdb.c:192 での呼び出しを確認）。
  • **呼び出し先**:
  • createPQExpBuffer, appendPQExpBuffer, appendPQExpBufferChar, destroyPQExpBuffer (libpq ユーティリティ)
  • appendConnStrVal (内部ヘルパー)
  • pg_strdup

関連データ構造

  • PQExpBuffer: 接続文字列構築用の一時バッファ。

失敗モードと境界条件

  • メモリ不足時には libpq のバッファ関数内でエラーが発生する可能性があります。
  • 入力配列が空の場合、空の文字列が返されます。

保守担当者向けメモ

  • 本関数はセキュリティを考慮し、サブプロセスへの接続情報漏洩を防ぐ設計となっています。新しい接続パラメータを追加する際は、機密情報が含まれないか確認し、必要に応じて除外リスト（if 文の条件）を更新してください。
