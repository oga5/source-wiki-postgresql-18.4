checkSplitConditions

定義

  • 種別: function
  • 定義箇所: src/backend/access/spgist/spgdoinsert.c:332

目的

SP-GiSTインデックスのページ分割が必要かどうかを判断するため、指定されたリーフタプルチェーンのサイズとタプル数を算出します。

シグネチャまたは形状

static int checkSplitConditions(Relation index, SpGistState *state, SPPageDesc *current, int *nToSplit)

入力と出力

  • **入力**:
  • index: 対象のインデックスリレーション。
  • state: SP-GiSTの内部状態管理構造体。
  • current: 調査対象のページとタプル情報を保持する SPPageDesc。
  • nToSplit: 分割対象となるライブタプルの数を格納する出力用ポインタ。
  • **出力**:
  • 戻り値: ライブタプルが占有する合計サイズ（バイト単位）。

前提条件と不変条件

  • current->page は有効なページである必要があります。
  • current->offnum から始まるタプルチェーンを走査します。
  • ルートページが指定された場合、強制的に分割を発生させるため BLCKSZ を返します。

アルゴリズムまたは振る舞い

1. **ルート判定**: current->blkno がルートであれば、強制分割フラグとして BLCKSZ を返し終了します。 2. **チェーン走査**: current->offnum から SGLT_GET_NEXTOFFSET を用いてリーフタプルチェーンを順次辿ります。 3. **統計計算**:

  • SPGIST_LIVE 状態のタプルのみをカウントし、サイズ（タプルサイズ + ItemIdData）を累積します。
  • SPGIST_DEAD 状態のタプルはカウントから除外します。

    • 4. **結果格納**: ライブタプルの総数を nToSplit に書き込み、合計サイズを戻り値として返します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/access/spgist/spgdoinsert.c 内の挿入処理フローから呼び出されます。
  • **呼び出し先**: PageGetItem, PageGetItemId, PageGetMaxOffsetNumber, SGLT_GET_NEXTOFFSET などのマクロ/関数を使用します。

関連データ構造

  • SPPageDesc: 挿入中のページ状態を保持する構造体。
  • SpGistLeafTuple: リーフタプルの内部構造。
  • ItemIdData: ページ内のアイテム識別子。

失敗モードと境界条件

  • **エラー**: SPGIST_LIVE でも SPGIST_DEAD でもない未知のタプル状態に遭遇した場合、elog(ERROR) を発生させます。
  • **境界条件**:
  • チェーンの終端（InvalidOffsetNumber）に達した時点で走査を終了します。
  • DEAD タプルがチェーンの先頭にある場合、そのタプルはカウントされません。

保守担当者向けメモ

  • 本関数は分割の必要性を判断するための統計取得のみを行い、実際の分割処理は doPickSplit 等が担います。
  • BLCKSZ を強制分割の閾値として利用しているため、ページサイズ変更時の挙動に注意してください。
