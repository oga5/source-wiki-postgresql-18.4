btbuild

定義

  • 種別: function
  • 定義箇所: src/backend/access/nbtree/nbtsort.c:294

目的

B-Treeインデックスのバルクロード（初期構築）を実行するメインエントリーポイントです。ヒープスキャンからソート、リーフページへの書き込み、およびツリー構造の構築までを統括します。

シグネチャまたは形状

IndexBuildResult *btbuild(Relation heap, Relation index, IndexInfo *indexInfo)

入力と出力

  • **入力**:
  • heap: 対象となるヒープリレーション。
  • index: 構築対象の空のインデックスリレーション。
  • indexInfo: インデックスの属性情報（ユニーク制約やNULLの扱いなど）。
  • **出力**:
  • IndexBuildResult*: 処理されたヒープタプル数およびインデックスタプル数を格納した構造体。

前提条件と不変条件

  • 対象のインデックスリレーションは空（ブロック数 0）でなければなりません。空でない場合、エラーが発生します。
  • 本関数は、任意のインデックスリレーションに対して一度だけ呼び出されることが想定されています。

アルゴリズムまたは振る舞い

1. **初期化**: BTBuildState を初期化し、ユニーク制約等のフラグを設定します。 2. **ヒープスキャン**: _bt_spools_heapscan を呼び出し、ヒープからタプルを読み込んでソート用スプールへ蓄積します。 3. **リーフ構築**: _bt_leafbuild を実行し、ソート済みタプルをリーフページへ書き込みます。 4. **リソース解放**: スプールを破棄し、並列構築が行われていた場合は _bt_end_parallel で終了処理を行います。 5. **統計情報**: BTREE_BUILD_STATS が定義されている場合、構築統計をログ出力します。 6. **結果返却**: 処理したタプル数を集計して返却します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/access/nbtree/nbtree.c: amroutine->ambuild として登録されています。
  • **呼び出し先**:
  • _bt_spools_heapscan
  • _bt_leafbuild
  • _bt_spooldestroy
  • _bt_end_parallel

関連データ構造

  • BTBuildState: 構築中の状態を保持する内部構造体。
  • IndexBuildResult: 構築結果を保持する標準的な構造体。
  • IndexInfo: インデックスのメタデータ。

失敗モードと境界条件

  • インデックスリレーションが既にデータを含んでいる場合、elog(ERROR) により処理が中断されます。
  • 並列構築時にワーカープロセスとの同期やリソース管理で問題が発生した場合、関連するサブ関数内でエラーが発生する可能性があります。

保守担当者向けメモ

  • 本関数はバルクロード専用のロジックであり、通常の挿入処理（nbtinsert.c）とは異なるパスを辿ります。
  • BTREE_BUILD_STATS を有効にすることで、構築パフォーマンスのデバッグが可能です。
  • 並列構築の終了処理（_bt_end_parallel）は、btleader が存在する場合のみ実行される条件付き処理です。
