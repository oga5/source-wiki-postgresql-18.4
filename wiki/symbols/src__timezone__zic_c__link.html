link

定義

  • 種別: struct
  • 定義箇所: src/timezone/zic.c:291

目的

タイムゾーン定義ファイル（.zi）内で指定された、タイムゾーン名間のリンク情報を保持するために使用されます。

シグネチャまたは形状

struct link
{
	const char *l_filename;
	lineno_t	l_linenum;
	const char *l_target;
	const char *l_linkname;
};

入力と出力

  • **入力**: タイムゾーン定義ファイル内の Link 行から解析された情報が各フィールドに格納されます。
  • **出力**: 構造体内の各メンバは、zic コンパイラが最終的にファイルシステム上でシンボリックリンクまたはハードリンクを作成する際の参照情報として利用されます。

前提条件と不変条件

  • l_filename および l_linenum は、エラーメッセージの出力時にソースファイルの場所を特定するために使用されます。
  • l_target はリンク元の既存のタイムゾーン名、l_linkname は作成する新しいリンク名を指します。

アルゴリズムまたは振る舞い

zic.c の解析プロセスにおいて、infile() が定義ファイルを読み込む際にこの構造体のインスタンスが構築されます。その後、コンパイルの最終段階で dolink() 関数がこの情報を参照し、指定されたターゲットに対してリンクを作成します。

呼び出し元と呼び出し先

  • **呼び出し元**: src/timezone/zic.c 内の構文解析関数（infile() 等）がこの構造体をインスタンス化します。
  • **呼び出し先**: dolink() 関数がこの構造体を参照してリンク作成処理を実行します。
  • **注意**: ツールが検出した contrib/amcheck 等の link への言及は、PostgreSQL のインデックス構造に関するものであり、本シンボルとは無関係です。

関連データ構造

  • lineno_t: 行番号を保持するための型定義。
  • struct rule, struct zone: zic.c における他の主要な定義構造体。

失敗モードと境界条件

  • リンク名が既に存在する場合や、ターゲットとなるゾーンが存在しない場合、dolink() によるリンク作成処理が失敗する可能性があります。

保守担当者向けメモ

  • 本構造体は IANA の zic ユーティリティ由来のものです。
  • リンク作成のロジックを変更する際は、src/timezone/zic.c 内の dolink() 関数の実装を確認してください。
