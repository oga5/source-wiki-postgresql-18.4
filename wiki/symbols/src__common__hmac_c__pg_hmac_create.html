pg_hmac_create

定義

  • 種別: function
  • 定義箇所: src/common/hmac.c:76

目的

HMAC（Keyed-Hashing for Message Authentication）計算を行うためのコンテキスト構造体 pg_hmac_ctx を生成・初期化します。

シグネチャまたは形状

pg_hmac_ctx *pg_hmac_create(pg_cryptohash_type type);

入力と出力

  • **入力**: type (pg_cryptohash_type) - 使用するハッシュアルゴリズム（MD5, SHA1, SHA2系）。
  • **出力**: 成功時は初期化済みの pg_hmac_ctx へのポインタ、失敗時は NULL を返します。

前提条件と不変条件

  • 指定された type に基づき、対応するダイジェスト長およびブロック長が設定されます。
  • 内部で pg_cryptohash_create を呼び出し、基盤となるハッシュコンテキストを確保します。

アルゴリズムまたは振る舞い

1. pg_hmac_ctx 構造体用のメモリを確保し、ゼロクリアします。 2. 引数 type に応じて、digest_size と block_size を適切な定数（MD5_BLOCK_SIZE 等）で初期化します。 3. pg_cryptohash_create(type) を呼び出し、ハッシュ計算用コンテキストを生成します。 4. ハッシュコンテキストの生成に失敗した場合は、確保した pg_hmac_ctx を解放して NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/libpq/auth-scram.c
  • src/common/scram-common.c
  • src/interfaces/libpq/fe-auth-scram.c
  • **呼び出し先**:
  • pg_cryptohash_create (基盤ハッシュコンテキストの生成)
  • ALLOC / FREE (メモリ管理マクロ)

関連データ構造

  • pg_hmac_ctx: HMAC 計算の状態を保持する不透明な構造体。
  • pg_cryptohash_type: サポートされるハッシュアルゴリズムの列挙型。

失敗モードと境界条件

  • メモリ不足により ALLOC が NULL を返した場合、関数は NULL を返します。
  • pg_cryptohash_create が失敗した場合、確保済みの pg_hmac_ctx を解放した上で NULL を返します。

保守担当者向けメモ

  • 本関数はフォールバック実装用です。OpenSSL 等の外部ライブラリが利用可能な環境では、ビルド構成に応じて src/common/hmac_openssl.c の実装が優先される場合があります。
  • メモリ管理は postgres.h / postgres_fe.h の定義に従い、バックエンドでは palloc、フロントエンドでは malloc が使用されます。フロントエンドで使用する場合は、呼び出し側で pg_hmac_free を呼び出す必要があります。
