GinMetaPageData

定義

  • **種別**: struct
  • **定義箇所**: src/include/access/ginblock.h:55

目的

GIN（Generalized Inverted Index）インデックスのルートとなるメタページ構造を定義します。インデックス全体の統計情報や、メインのインデックス構造へ統合される前のエントリを保持する「保留中リスト（Pending List）」の管理に使用されます。

シグネチャまたは形状

typedef struct GinMetaPageData {
    BlockNumber head;
    BlockNumber tail;
    uint32      tailFreeSize;
    BlockNumber nPendingPages;
    int64       nPendingHeapTuples;
    BlockNumber nTotalPages;
    BlockNumber nEntryPages;
    BlockNumber nDataPages;
    int64       nEntries;
    int32       ginVersion;
} GinMetaPageData;

入力と出力

本構造体はデータ保持用であり、関数ではありません。インデックス操作ルーチンがメタページの内容を読み書きする際のインターフェースとして機能します。

前提条件と不変条件

  • ginVersion は現在 2 です。
  • バージョン 1 以下のインデックスは互換性を持ちますが、古い形式のデータ（非圧縮のポストリスト等）が含まれる可能性があります。
  • nTotalPages 等の統計情報は、最後に VACUUM が実行された時点での正確な値です。

アルゴリズムまたは振る舞い

  • **保留中リスト管理**: head および tail ポインタにより、高速挿入されたエントリを保持する GIN_LIST ページを管理します。
  • **バージョン管理**: ginVersion フィールドにより、インデックスの初期化時期に応じた機能制限やデータ形式の解釈を決定します。バージョン 0 のインデックスでは、全インデックススキャンを拒否する等の制約があります。

呼び出し元と呼び出し先

  • **呼び出し元（参照）**:
  • src/backend/access/gin/ginfast.c（保留中リストの操作）
  • contrib/pageinspect/ginfuncs.c（インデックス検査）
  • contrib/pgstattuple/pgstatindex.c（統計情報取得）
  • **呼び出し先**: 構造体定義のため、特定の関数呼び出し先はありません。

関連データ構造

  • GinPageOpaqueData: 全GINページ末尾に配置される共通構造体。
  • GIN_LIST: 保留中リストを構成するページタイプ。

失敗モードと境界条件

  • バージョン 0 のインデックスに対して全インデックススキャンを試みた場合、適切に拒否する必要があります。
  • 統計情報は VACUUM 実行時まで更新されないため、最新の正確な値とは限らない点に注意が必要です。

保守担当者向けメモ

  • ginVersion フィールドは構造体の末尾に配置されています。過去の経緯により先頭への移動は不可能なため、レイアウト変更時は注意してください。
  • 構造体サイズは sizeof(GinMetaPageData) で取得可能です。ginfast.c 等でメタページ全体をコピーする際に使用されています。
