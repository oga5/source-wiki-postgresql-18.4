templ

定義

  • 種別: struct
  • 定義箇所: src/tools/pg_bsd_indent/lexi.c:60

目的

C言語の予約語（キーワード）と、それに対応するトークン種別コードを保持するためのデータ構造です。

シグネチャまたは形状

struct templ {
    const char *rwd;
    int         rwcode;
};

入力と出力

  • 入力: rwd（予約語文字列）、rwcode（対応する整数コード）
  • 出力: なし（データ構造定義）

前提条件と不変条件

  • bsearch を用いた二分探索を前提としているため、この構造体を用いた配列（specials[]）はアルファベット順にソートされている必要があります。
  • 検索アルゴリズムの制約により、rwd（文字列）が構造体の先頭メンバである必要があります。

アルゴリズムまたは振る舞い

本構造体自体はデータ保持用ですが、lexi.c 内の字句解析において、識別子が予約語であるかを判定するためのテーブル定義に使用されます。

呼び出し元と呼び出し先

  • 呼び出し元: lexi.c 内の字句解析ロジックが、specials[] 配列を介して本構造体を参照します。
  • 呼び出し先: なし（データ構造のため）。

関連データ構造

  • specials[]: templ 型の配列。予約語のテーブルとして定義されています。

失敗モードと境界条件

  • 予約語テーブルの順序がアルファベット順でない場合、二分探索が失敗し、正しいトークン種別が取得できません。

保守担当者向けメモ

  • specials 配列に新しいキーワードを追加する際は、必ずアルファベット順を維持してください。
  • 構造体のメンバ順序を変更すると、bsearch による検索ロジックが破壊されるため注意してください。
