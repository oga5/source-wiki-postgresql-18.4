CreateSharedComments

定義

  • 種別: function
  • 定義箇所: src/backend/commands/comment.c:237

目的

共有データベースオブジェクト（データベース、ロール、テーブルスペース等）に対するコメントの挿入、更新、または削除を行う。

シグネチャまたは形状

void CreateSharedComments(Oid oid, Oid classoid, const char *comment)

入力と出力

  • oid: 対象オブジェクトのOID。
  • classoid: 対象オブジェクトのクラスID。
  • comment: 設定するコメント文字列。空文字列またはNULLが渡された場合は、既存のコメントを削除する。

前提条件と不変条件

  • pg_shdescription カタログに対する操作を行う。
  • 共有オブジェクトのため、subid（サブオブジェクトID）は考慮しない。
  • 呼び出し元で適切なロック（ShareUpdateExclusiveLock 等）が取得されていることを前提とする。

アルゴリズムまたは振る舞い

1. **入力正規化**: comment が空文字列の場合、NULL に変換して削除処理へ回す。 2. **タプル準備**: comment が NULL でない場合、pg_shdescription の各属性（objoid, classoid, description）を更新用データとして準備する。 3. **検索**: SharedDescriptionRelationId を開き、SharedDescriptionObjIndexId を使用して対象オブジェクトの既存レコードをスキャンする。 4. **更新/削除**: 既存レコードが見つかった場合、comment が NULL なら CatalogTupleDelete を、それ以外なら CatalogTupleUpdate を実行する。 5. **挿入**: 既存レコードが見つからず、かつ comment が NULL でない場合、CatalogTupleInsert を用いて新規レコードを作成する。 6. **終了処理**: 使用したリソースを解放し、テーブルを閉じる。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/commands/comment.c 内の CommentObject 関数から呼び出される。
  • **呼び出し先**: table_open, systable_beginscan, systable_getnext, systable_endscan, CatalogTupleUpdate, CatalogTupleDelete, CatalogTupleInsert, heap_modify_tuple, heap_form_tuple, heap_freetuple, table_close 等のカタログアクセスAPI。

関連データ構造

  • pg_shdescription: 共有オブジェクトのコメントを保持するシステムカタログ。
  • Oid: オブジェクト識別子。

失敗モードと境界条件

  • 対象オブジェクトに対して複数のコメントレコードが存在することは想定外であり、ループ内で最初のマッチのみを処理して break する。
  • comment が NULL の場合、既存レコードの削除のみが行われる。

保守担当者向けメモ

  • 本関数は共有オブジェクト専用である。通常オブジェクトのコメント操作には CreateComments を使用すること。
  • pg_shdescription のインデックススキャンに依存しているため、カタログ定義の変更時には注意が必要。
