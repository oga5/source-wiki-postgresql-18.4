GinState

定義

  • 種別: struct
  • 定義箇所: src/include/access/gin_private.h:57

目的

GIN（Generalized Inverted Index）インデックスの操作中における状態管理およびメタデータ保持を行うための主要な構造体です。インデックスの走査、挿入、およびサポート関数の実行に必要なコンテキストを提供します。

シグネチャまたは形状

typedef struct GinState
{
    Relation    index;
    bool        oneCol;
    TupleDesc   origTupdesc;
    TupleDesc   tupdesc[INDEX_MAX_KEYS];
    FmgrInfo    compareFn[INDEX_MAX_KEYS];
    FmgrInfo    extractValueFn[INDEX_MAX_KEYS];
    FmgrInfo    extractQueryFn[INDEX_MAX_KEYS];
    FmgrInfo    consistentFn[INDEX_MAX_KEYS];
    FmgrInfo    triConsistentFn[INDEX_MAX_KEYS];
    FmgrInfo    comparePartialFn[INDEX_MAX_KEYS];
    bool        canPartialMatch[INDEX_MAX_KEYS];
    Oid         supportCollation[INDEX_MAX_KEYS];
} GinState;

入力と出力

  • **入力**: インデックス操作（挿入、検索、クリーンアップ等）の開始時に、対象となる Relation を基に初期化されます。
  • **出力**: 保持する FmgrInfo を通じて、インデックスの各カラムに対応する演算子クラスのサポート関数を呼び出すためのインターフェースを提供します。

前提条件と不変条件

  • tupdesc 配列は、インデックスの各カラムの型情報を保持します。
  • 単一カラムインデックスの場合、oneCol は true となり、origTupdesc はリーフタプルを直接記述します。
  • 複数カラムインデックスの場合、リーフタプルはカラム番号とキーデータで構成され、tupdesc[i] は (int2, keytype) の形式を記述します。

アルゴリズムまたは振る舞い

  • インデックス操作のコンテキストとして機能し、gininsert や gingetbitmap などの処理において、演算子クラスの関数（consistentFn 等）を呼び出すための準備を行います。
  • 部分一致検索が可能な場合、canPartialMatch が true に設定され、comparePartialFn が利用可能になります。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/amcheck/verify_gin.c
  • src/backend/access/gin/ginentrypage.c
  • src/backend/access/gin/ginfast.c
  • src/backend/access/gin/ginget.c
  • **呼び出し先**:
  • 演算子クラスのサポート関数（FmgrInfo 経由で呼び出される関数群）。

関連データ構造

  • Relation: インデックスの物理的な関係情報。
  • TupleDesc: タプルの記述子。
  • FmgrInfo: PostgreSQL の関数マネージャ情報。

失敗モードと境界条件

  • INDEX_MAX_KEYS を超えるカラム数を持つインデックスには対応していません。
  • 必要なサポート関数が演算子クラスで定義されていない場合、FmgrInfo の呼び出し時にエラーが発生する可能性があります。

保守担当者向けメモ

  • GinState は GIN インデックスの内部実装において頻繁に参照されるため、フィールドの追加や変更は gininsert.c や ginget.c 等の広範なコードに影響を与えます。
  • src/backend/access/gin/README を参照し、リーフタプルの構造と tupdesc のマッピング関係を十分に理解した上で修正を行ってください。
