GetUserMapping

定義

  • 種別: function
  • 定義箇所: src/backend/foreign/foreign.c:200

目的

指定されたユーザーおよび外部サーバーに対応するユーザーマッピング情報をシステムカタログから取得し、メモリ上の構造体として返します。

シグネチャまたは形状

UserMapping *GetUserMapping(Oid userid, Oid serverid)

入力と出力

  • **入力**:
  • userid (Oid): 検索対象のユーザーID。
  • serverid (Oid): 検索対象の外部サーバーID。
  • **出力**:
  • UserMapping*: 取得したユーザーマッピング情報を含む構造体へのポインタ。

前提条件と不変条件

  • USERMAPPINGUSERSERVER システムキャッシュを使用して検索を行います。
  • 呼び出し側は、返された UserMapping 構造体を適切に管理（必要に応じて解放）する必要があります。

アルゴリズムまたは振る舞い

1. 指定された userid と serverid で USERMAPPINGUSERSERVER キャッシュを検索します。 2. 見つからない場合、userid に InvalidOid を指定して PUBLIC ユーザーマッピングを再検索します。 3. いずれも存在しない場合は ERROR を発生させます。 4. 成功した場合、システムカタログ pg_user_mapping から情報を抽出し、UserMapping 構造体に格納します。 5. umoptions（オプション情報）は untransformRelOptions を介してリスト形式に変換されます。 6. 使用したシステムキャッシュタプルを解放し、構造体を返します。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • contrib/dblink/dblink.c
  • contrib/postgres_fdw/postgres_fdw.c
  • **呼び出し先**:
  • SearchSysCache2
  • GetForeignServer
  • ereport
  • palloc
  • SysCacheGetAttr
  • untransformRelOptions
  • ReleaseSysCache

関連データ構造

  • UserMapping: ユーザーマッピング情報を保持する構造体。
  • Form_pg_user_mapping: システムカタログ pg_user_mapping のタプル形式。

失敗モードと境界条件

  • 指定されたユーザーおよび PUBLIC のいずれにもマッピングが存在しない場合、ERRCODE_UNDEFINED_OBJECT エラーが発生します。
  • umoptions が存在しない（isnull）場合は、options フィールドに NIL が設定されます。

保守担当者向けメモ

  • 本関数は pg_user_mapping カタログに依存しています。カタログスキーマの変更時には、SysCacheGetAttr の属性インデックス（Anum_pg_user_mapping_umoptions）が整合しているか確認してください。
