validate_compress_specification

定義

  • 種別: function
  • 定義箇所: src/common/compression.c:343

目的

圧縮設定構造体 pg_compress_specification の内容が、指定されたアルゴリズムの制約（圧縮レベルの範囲、機能の対応状況）を満たしているかを検証する。

シグネチャまたは形状

char *validate_compress_specification(pg_compress_specification *spec);

入力と出力

  • **入力**: pg_compress_specification *spec（検証対象の圧縮設定構造体）
  • **出力**: 検証成功時は NULL を返す。検証失敗時は、エラー内容を説明する文字列（char *）を返す。

前提条件と不変条件

  • spec->parse_error が NULL であること（解析段階でエラーが発生していないこと）。
  • 呼び出し側は、戻り値が NULL でない場合に返されたエラー文字列を適切に解放（pfree 等）する責任を持つ。

アルゴリズムまたは振る舞い

1. **解析エラー確認**: spec->parse_error が設定されていれば、即座にそのエラーを返す。 2. **アルゴリズム別検証**: spec->algorithm に応じて以下のチェックを行う。

  • GZIP: レベル範囲を 1〜9 に設定。
  • LZ4: レベル範囲を 1〜12 に設定。
  • ZSTD: ビルド時に USE_ZSTD が有効な場合、ZSTD_minCLevel() および ZSTD_maxCLevel() を使用して範囲を決定。
  • NONE: level が 0 以外であればエラーを返す。

    • 3. **レベル範囲チェック**: spec->level がアルゴリズムの許容範囲外かつデフォルト値でもない場合、エラーを返す。 4. **オプション機能チェック**:

  • PG_COMPRESSION_OPTION_WORKERS が指定されている場合、ZSTD 以外であればエラーを返す。
  • PG_COMPRESSION_OPTION_LONG_DISTANCE が指定されている場合、ZSTD 以外であればエラーを返す。

    • 5. **正常終了**: すべてのチェックを通過した場合 NULL を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/backup/basebackup.c
  • src/bin/pg_basebackup/pg_basebackup.c
  • src/bin/pg_basebackup/pg_receivewal.c
  • src/bin/pg_dump/pg_dump.c
  • **呼び出し先**:
  • get_compress_algorithm_name
  • psprintf
  • ZSTD_maxCLevel / ZSTD_minCLevel（ビルド依存）

関連データ構造

  • pg_compress_specification: 圧縮アルゴリズム、レベル、オプションフラグ、エラーメッセージを保持する構造体。

失敗モードと境界条件

  • **解析失敗**: spec->parse_error が設定されている場合、検証を行わずにエラーを伝播する。
  • **範囲外指定**: 圧縮レベルが各アルゴリズムの許容範囲外の場合、エラーメッセージを生成する。
  • **非対応オプション**: ZSTD 以外のアルゴリズムで並列ワーカー数やロングディスタンスモードを指定した場合、エラーを返す。
  • **ビルド依存**: HAVE_LIBZ や USE_ZSTD が定義されていない環境では、各ライブラリ固有の機能検証が制限される。

保守担当者向けメモ

  • 本関数は pg_dump や pg_basebackup などのクライアントツールとサーバー側の両方で使用される。
  • 新しい圧縮アルゴリズムを追加する際は、本関数の switch 文に新しいケースを追加し、レベル範囲とサポートするオプションを定義する必要がある。
  • エラーメッセージは psprintf を使用して動的に生成されるため、メモリ管理に注意すること。
