StopList

定義

  • 種別: struct
  • 定義箇所: src/include/tsearch/ts_public.h:100

目的

全文検索において、検索対象から除外すべき単語（ストップワード）のリストを保持・管理するための構造体です。

シグネチャまたは形状

typedef struct
{
	int			len;
	char	  **stop;
} StopList;

入力と出力

  • len: リストに含まれるストップワードの総数。
  • stop: ストップワード文字列を格納する配列へのポインタ。

前提条件と不変条件

  • stop 配列は、readstoplist 関数によって外部ファイルから読み込まれたデータで初期化されることが想定されています。
  • len は stop 配列の有効な要素数と一致している必要があります。

アルゴリズムまたは振る舞い

  • readstoplist: 外部ファイルからストップワードを読み込み、StopList 構造体に格納します。
  • searchstoplist: 指定された単語（key）が StopList 内に存在するかを判定します。

呼び出し元と呼び出し先

  • **呼び出し元（参照元）**:
  • src/backend/snowball/dict_snowball.c
  • src/backend/tsearch/dict_ispell.c
  • src/backend/tsearch/dict_simple.c
  • **呼び出し先**:
  • 本構造体はデータ保持用であり、操作には src/include/tsearch/ts_public.h で宣言された readstoplist および searchstoplist 関数が利用されます。

関連データ構造

  • tsearch/ts_type.h: 全文検索の基本型定義を提供します。

失敗モードと境界条件

  • readstoplist 実行時にファイル読み込みに失敗した場合や、メモリ不足が発生した場合の挙動は、呼び出し側の実装に依存します。
  • searchstoplist は、StopList が適切に初期化されていない場合に未定義の動作を引き起こす可能性があります。

保守担当者向けメモ

  • 本構造体は ts_public.h で定義されており、PostgreSQL の全文検索辞書モジュール間での共通インターフェースとして機能します。
  • 辞書の実装（dict_simple.c 等）において、ストップワードの検索効率は searchstoplist の実装に依存します。
