FindReplTupleInLocalRel

定義

  • 種別: function
  • 定義箇所: src/backend/replication/logical/worker.c:2914

目的

論理レプリケーションにおいて、リモートのタプルに対応するローカルリレーション内のタプルを検索し、排他ロックを取得する。

シグネチャまたは形状

bool FindReplTupleInLocalRel(ApplyExecutionData *edata, Relation localrel,
                             LogicalRepRelation *remoterel,
                             Oid localidxoid,
                             TupleTableSlot *remoteslot,
                             TupleTableSlot **localslot)

入力と出力

  • **入力**:
  • edata: 適用処理の実行状態。
  • localrel: 検索対象のローカルリレーション。
  • remoterel: リモートリレーションの定義情報。
  • localidxoid: 検索に使用するインデックスのOID（無効な場合はシーケンシャルスキャン）。
  • remoteslot: リモートから受信したタプルデータ。
  • **出力**:
  • localslot: 検索結果のタプルを格納するスロット（関数内で生成）。
  • 戻り値: タプルが見つかった場合は true、それ以外は false。

前提条件と不変条件

  • localidxoid が有効であるか、または remoterel->replident が REPLICA_IDENTITY_FULL である必要がある。
  • 実行時に TargetPrivilegesCheck により対象リレーションに対する ACL_SELECT 権限が検証される。

アルゴリズムまたは振る舞い

1. TargetPrivilegesCheck を呼び出し、SELECT権限を確認する。 2. table_slot_create を使用して localslot を初期化する。 3. localidxoid が有効な場合:

  • RelationFindReplTupleByIndex を呼び出し、指定インデックスを用いて検索を行う。

    • 4. localidxoid が無効な場合:

  • RelationFindReplTupleSeq を呼び出し、シーケンシャルスキャンで検索を行う。

    • 5. 検索時は常に LockTupleExclusive を指定し、対象タプルに排他ロックをかける。 6. 検索結果（found）を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/replication/logical/worker.c 内の複数箇所（行388, 2680, 2867, 3071付近）。
  • **呼び出し先**: TargetPrivilegesCheck, table_slot_create, RelationFindReplTupleByIndex, RelationFindReplTupleSeq。

関連データ構造

  • ApplyExecutionData: 適用処理の実行コンテキスト。
  • Relation: ローカルテーブルのメタデータ。
  • LogicalRepRelation: リモートテーブルの定義。
  • TupleTableSlot: タプルデータを保持する構造体。

失敗モードと境界条件

  • インデックス検索時、アサーションチェック（USE_ASSERT_CHECKING有効時）により、インデックスがPK/RIであるか、または REPLICA_IDENTITY_FULL に適格であるかが検証される。
  • 権限不足時は TargetPrivilegesCheck がエラーを発生させる可能性がある。

保守担当者向けメモ

  • 本関数は論理レプリケーションの適用処理におけるタプル検索の抽象化レイヤーである。
  • localslot のメモリ管理は estate->es_tupleTable に依存している点に注意すること。
