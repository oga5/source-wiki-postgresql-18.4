proclist_head

定義

  • struct
  • 定義箇所: src/include/storage/proclist_types.h:38

目的

プロセス管理において、ProcNumber を用いた双方向連結リストの先頭および末尾を保持するためのヘッダー構造体です。

シグネチャまたは形状

typedef struct proclist_head
{
    ProcNumber  head; /* リスト先頭の PGPROC 番号 */
    ProcNumber  tail; /* リスト末尾の PGPROC 番号 */
} proclist_head;

入力と出力

  • 入力: proclist.h で定義される各操作関数（proclist_init, proclist_push_head_offset 等）に対し、リストの状態を保持するポインタとして渡されます。
  • 出力: リストの先頭および末尾の ProcNumber を保持します。

前提条件と不変条件

  • 空リストは INVALID_PROC_NUMBER で表現されます。
  • リストに含まれていないノードは next == prev == 0 である必要があります。
  • リスト内のノードが循環参照を持つことは許可されていません。
  • 構造体のメンバを直接操作せず、必ず proclist.h で提供される API を使用してください。

アルゴリズムまたは振る舞い

  • ProcNumber（整数インデックス）を使用することで、共有メモリ内での配置やプロセス間でのリスト構造の共有を容易にします。
  • proclist.h の関数群を通じて、リストへの要素の挿入、削除、走査が行われます。

呼び出し元と呼び出し先

  • 呼び出し元（参照元）:
  • src/backend/storage/lmgr/lwlock.c (wakeup処理)
  • src/include/storage/condition_variable.h (待機プロセスリスト)
  • src/include/storage/lwlock.h (待機中 PGPROC リスト)
  • 呼び出し先:
  • storage/procnumber.h (ProcNumber 型および INVALID_PROC_NUMBER 定数)

関連データ構造

  • proclist_node: リストの各要素を構成するリンク情報。
  • proclist_mutable_iter: 走査中に要素削除を安全に行うためのイテレータ。
  • PGPROC: リストの各ノードが指し示す対象のプロセス構造体。

失敗モードと境界条件

  • リストが空の場合、head および tail は INVALID_PROC_NUMBER となります。
  • 整合性を維持するため、リスト操作 API を介さない直接的なメンバ操作はリストの破損を招きます。

保守担当者向けメモ

  • 本構造体はポインタではなく ProcNumber を使用しているため、異なるアドレス空間にマップされてもリスト構造が維持されます。
  • リスト操作の詳細は src/include/storage/proclist.h を参照してください。
