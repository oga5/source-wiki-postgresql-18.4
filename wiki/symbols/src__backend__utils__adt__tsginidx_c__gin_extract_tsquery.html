gin_extract_tsquery

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/tsginidx.c:93

目的

GINインデックス検索時、tsqueryからインデックス検索用のキー（語彙）を抽出し、検索モード（全走査かインデックス検索か）を決定する。

シグネチャまたは形状

Datum gin_extract_tsquery(PG_FUNCTION_ARGS)

入力と出力

  • **入力**:
  • PG_GETARG_TSQUERY(0): 検索対象のtsquery。
  • PG_GETARG_POINTER(1): 抽出されたキーの数を格納するint32ポインタ。
  • PG_GETARG_POINTER(3): 前方一致フラグ配列（bool **）へのポインタ。
  • PG_GETARG_POINTER(4): 追加データ（Pointer **）へのポインタ。
  • PG_GETARG_POINTER(6): 検索モード（int32 *）へのポインタ。
  • **出力**:
  • 戻り値: 抽出されたキーのDatum配列。
  • *nentries: 抽出されたキーの数。
  • *searchMode: GIN_SEARCH_MODE_DEFAULT または GIN_SEARCH_MODE_ALL。

前提条件と不変条件

  • query->size > 0 の場合のみキー抽出が行われる。
  • tsquery_requires_match が真を返す場合、GIN_SEARCH_MODE_DEFAULT が設定される。
  • tsquery_requires_match が偽（例: ! foo のような否定のみのクエリ）の場合、GIN_SEARCH_MODE_ALL が設定される。

アルゴリズムまたは振る舞い

1. tsquery内のQI_VAL（値）アイテムを走査し、その数をカウントする。 2. 抽出したキーの数分だけメモリ（entries, partialmatch, extra_data）を確保する。 3. map_item_operand を作成し、クエリ内のアイテム番号と抽出されたキー番号をマッピングする。 4. 各QI_VALから文字列を抽出し、entries配列に格納する。 5. 対応するval->prefixをpartialmatch配列に設定する。 6. 最後にqueryを解放し、entries配列を返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/backend/utils/adt/tsginidx.c 内のスタブ関数（gin_extract_tsquery_oldsig 等）から呼び出される。
  • **呼び出し先**:
  • tsquery_requires_match
  • GETQUERY, GETOPERAND (マクロ)
  • cstring_to_text_with_len
  • palloc, palloc0

関連データ構造

  • TSQuery: 全文検索クエリの内部表現。
  • QueryItem, QueryOperand: tsqueryを構成する要素。
  • GIN_SEARCH_MODE_DEFAULT, GIN_SEARCH_MODE_ALL: GINインデックスの検索モード定数。

失敗モードと境界条件

  • query->size == 0 の場合、キーは抽出されず、nentries は 0 となる。
  • メモリ不足時は palloc によりエラーが発生する。

保守担当者向けメモ

  • 本関数は、インデックス検索のキー抽出ロジックの核となる。
  • extra_data に格納される map_item_operand は、後の consistent メソッドでクエリの論理構造を評価する際に使用されるため、変更時は整合性に注意すること。
  • gin_extract_tsquery_oldsig 等のスタブ関数は、古いシグネチャとの互換性維持のために存在しており、本関数を直接呼び出す設計となっている。
