merge

定義

  • 種別: function
  • 定義箇所: src/backend/lib/pairingheap.c:78

目的

2つのペアリングヒープノード（またはサブヒープ）を統合し、単一のヒープ構造を維持する。

シグネチャまたは形状

static pairingheap_node *merge(pairingheap *heap, pairingheap_node *a, pairingheap_node *b)

入力と出力

  • **入力**:
  • heap: ヒープの管理構造体（比較関数 ph_compare を保持）。
  • a, b: マージ対象の pairingheap_node へのポインタ。
  • **出力**:
  • マージ後の新しいルートとなる pairingheap_node へのポインタ。

前提条件と不変条件

  • heap->ph_compare は、2つのノードを比較し、順序に基づいて整数を返す関数である必要がある。
  • マージの結果、常に値が大きい（または比較関数の定義に従う）ノードがルートとなり、もう一方がその子ノードとして追加される。

アルゴリズムまたは振る舞い

1. **空チェック**: a または b が NULL の場合、他方をそのまま返す。 2. **順序の決定**: heap->ph_compare を使用し、a が b より小さい場合は a と b を入れ替える。これにより、常に a が新しいルートとなる。 3. **リンクの更新**:

  • b を a の子リストの先頭に挿入する。
  • a の既存の first_child が存在する場合、その prev_or_parent を b に更新する。
  • b の prev_or_parent を a に設定し、b の next_sibling を a の現在の first_child に設定する。
  • a の first_child を b に更新する。

    • 4. **戻り値**: 新しいルートである a を返す。

呼び出し元と呼び出し先

  • **呼び出し元**: pairingheap_add が本関数を呼び出すことが示唆されている（ソースファイル概要より）。
  • **呼び出し先**: heap->ph_compare（関数ポインタ経由）。
  • **注意**: ツールによる参照リストには merge という名称の他機能への言及が含まれているが、これらは本シンボルとは無関係である。

関連データ構造

  • pairingheap: ヒープ全体を管理する構造体。
  • pairingheap_node: ヒープの各要素。first_child, next_sibling, prev_or_parent ポインタを持つ。

失敗モードと境界条件

  • a または b が NULL の場合、安全に処理される。
  • 比較関数が正しく定義されていない場合、ヒープの不変条件が破壊される可能性がある。

保守担当者向けメモ

  • 本関数は静的関数であり、pairingheap.c 内の内部操作専用である。
  • 挿入操作の基本単位となるため、効率がヒープ全体の性能に直結する。
