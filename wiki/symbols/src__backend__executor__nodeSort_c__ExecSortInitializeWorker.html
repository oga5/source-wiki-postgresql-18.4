ExecSortInitializeWorker

定義

  • 種別: function
  • 定義箇所: src/backend/executor/nodeSort.c:461

目的

並列クエリ実行時において、ワーカープロセス内の SortState を初期化し、共有メモリ上の統計情報領域を紐付ける。

シグネチャまたは形状

void ExecSortInitializeWorker(SortState *node, ParallelWorkerContext *pwcxt)

入力と出力

  • **入力**:
  • node: 初期化対象の SortState 構造体へのポインタ。
  • pwcxt: 並列ワーカーのコンテキスト情報（共有メモリの TOC を含む）。
  • **出力**: なし（node 構造体の内部状態を更新する）。

前提条件と不変条件

  • pwcxt が有効であり、node に対応するプランノード ID が共有メモリの TOC (pwcxt->toc) に存在すること。

アルゴリズムまたは振る舞い

1. pwcxt->toc から、現在のプランノード ID (node->ss.ps.plan->plan_node_id) をキーとして共有メモリ上の情報を検索する。 2. 検索結果を node->shared_info に格納する。 3. node->am_worker フラグを true に設定し、自身がワーカープロセスであることを明示する。

呼び出し元と呼び出し先

  • **呼び出し元**: src/backend/executor/execParallel.c:1389 にて呼び出される。
  • **呼び出し先**: shm_toc_lookup（共有メモリのテーブル検索関数）。

関連データ構造

  • SortState: ソートノードの実行状態を保持する構造体。
  • ParallelWorkerContext: 並列実行時のワーカー用コンテキスト。
  • SharedExecutorInstrumentation: node->shared_info が指し示す共有メモリ上の統計情報構造体。

失敗モードと境界条件

  • shm_toc_lookup の第3引数が true であるため、指定されたキーが見つからない場合はエラーが発生する可能性がある。

保守担当者向けメモ

  • 本関数は並列クエリのワーカー側でのみ実行される。
  • node->shared_info は並列ワーカー間での統計情報の集約に使用されるため、この初期化が正しく行われないと統計情報の収集が失敗する。
