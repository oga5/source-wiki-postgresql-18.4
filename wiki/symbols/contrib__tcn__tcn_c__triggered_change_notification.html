triggered_change_notification

定義

  • 種別: function
  • 定義箇所: contrib/tcn/tcn.c:58

目的

データベーステーブルの変更（INSERT, UPDATE, DELETE）を検知し、主キー情報を含むペイロードを NOTIFY コマンド経由で非同期通知として送信するトリガー関数です。

シグネチャまたは形状

Datum triggered_change_notification(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS（TriggerData を含む実行コンテキスト）。オプションとして、通知チャネル名を指定する引数を最大1つ受け取ります。
  • **出力**: Datum 型（常に NULL を返します）。

前提条件と不変条件

  • **トリガー設定**: AFTER かつ FOR EACH ROW のトリガーとして呼び出される必要があります。
  • **スキーマ制約**: 対象テーブルには主キーが定義されている必要があります。
  • **引数制約**: 引数は0個（デフォルトチャネル "tcn" を使用）または1個（チャネル名を指定）である必要があります。

アルゴリズムまたは振る舞い

1. **検証**: 呼び出しコンテキストがトリガーとして適切か、AFTER かつ FOR EACH ROW かをチェックします。 2. **操作判定**: INSERT('I'), UPDATE('U'), DELETE('D') のいずれかを判定します。 3. **主キー特定**: RelationGetIndexList と SearchSysCache1 を使用し、対象テーブルの有効な主キーインデックスを検索します。 4. **ペイロード構築**: StringInfo を使用し、"テーブル名",操作,列名='値',... という形式の CSV 文字列を生成します。値の抽出には SPI_getvalue を使用します。 5. **通知**: Async_Notify を呼び出し、指定されたチャネルへペイロードを送信します。 6. **終了**: 主キーが見つからない場合はエラーを発生させます。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のトリガーマネージャ（直接の呼び出し元情報はソースコード上では明示されていません）。
  • **呼び出し先**:
  • Async_Notify（通知送信）
  • SPI_getvalue（タプル値抽出）
  • RelationGetIndexList, SearchSysCache1, ReleaseSysCache（インデックス検索）
  • strcpy_quoted（文字列エスケープ）

関連データ構造

  • TriggerData: トリガー実行コンテキスト。
  • StringInfo: ペイロード構築用バッファ。
  • Form_pg_index: システムカタログのインデックス情報。

失敗モードと境界条件

  • **プロトコル違反**: トリガーとして正しくない呼び出し、または AFTER/FOR EACH ROW ではない場合にエラーとなります。
  • **主キー欠如**: 主キーが存在しないテーブルで実行された場合、エラーとなります。
  • **引数過多**: 2つ以上の引数が指定された場合、エラーとなります。

保守担当者向けメモ

  • 本関数はシステムカタログ（pg_index）へ頻繁にアクセスするため、高頻度な更新が発生するテーブルではパフォーマンスへの影響を考慮してください。
  • strcpy_quoted によるエスケープは単純な実装であるため、特殊文字を含むデータ型の取り扱いには注意が必要です。
