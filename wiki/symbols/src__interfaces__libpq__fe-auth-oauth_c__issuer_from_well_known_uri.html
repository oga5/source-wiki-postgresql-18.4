issuer_from_well_known_uri

定義

  • **種別**: function
  • **定義箇所**: src/interfaces/libpq/fe-auth-oauth.c:365

目的

OAuth 2.0 / OIDC の Discovery URI（.well-known URI）から、対応する発行者（Issuer）識別子を抽出する。

シグネチャまたは形状

char *issuer_from_well_known_uri(PGconn *conn, const char *wkuri)

入力と出力

  • **入力**:
  • conn: エラー報告に使用する PGconn オブジェクト。
  • wkuri: 解析対象の Discovery URI 文字列。
  • **出力**:
  • 成功時: 抽出された発行者識別子を含む動的に確保された文字列（呼び出し元で解放が必要）。
  • 失敗時: NULL（conn にエラーメッセージが設定される）。

前提条件と不変条件

  • URI は原則として https:// で始まる必要がある（PGOAUTHDEBUG=UNSAFE が有効な場合のみ http:// を許可）。
  • URI にクエリパラメータやフラグメント（? や #）を含めることはできない。
  • URI は OIDC Discovery 1.0 または RFC 8414 で定義された .well-known 接尾辞を含む必要がある。

アルゴリズムまたは振る舞い

1. **スキーム検証**: https://（またはデバッグ時の http://）で始まるかを確認。 2. **構造検証**: クエリやフラグメントが含まれていないか確認。 3. **パス解析**: .well-known プレフィックスを検索し、その後に続く有効な接尾辞（openid-configuration または oauth-authorization-server）を特定。 4. **形式検証**: 接尾辞がパスセグメントの末尾にあるか、あるいはパスの先頭に位置しているかを検証。 5. **抽出**: URI 文字列から .well-known 関連のパス部分を除去し、残りの部分を発行者識別子として返す。

呼び出し元と呼び出し先

  • **呼び出し元**:
  • src/interfaces/libpq/fe-auth-oauth.c:603
  • src/interfaces/libpq/fe-auth-oauth.c:1109
  • **呼び出し先**:

関連データ構造

  • PGconn: libpq の接続状態を保持する構造体。

失敗モードと境界条件

  • **HTTPS 必須**: https:// 以外（かつデバッグ設定なし）の場合、エラーを返す。
  • **不正な形式**: クエリ/フラグメントの混入、サポート外の接尾辞、またはパス構造が不正な場合はエラーを返す。
  • **メモリ不足**: strdup が失敗した場合、NULL を返す。

保守担当者向けメモ

  • 本関数は、RFC 8414 および OIDC Discovery 1.0 の仕様に基づき、.well-known パスを削除して発行者 ID を再構築する。
  • memmove を使用して文字列を詰め合わせる際、NULL 終端文字を含めて移動している点に注意。
