array_prepend_support

定義

  • 種別: function
  • 定義箇所: src/backend/utils/adt/array_userfuncs.c:285

目的

array_prepend 関数に対するプランナのサポート関数であり、配列への要素追加操作において、インプレース（破壊的）更新が可能かどうかを判定し、最適化を支援します。

シグネチャまたは形状

Datum array_prepend_support(PG_FUNCTION_ARGS)

入力と出力

  • **入力**: PG_FUNCTION_ARGS を介して SupportRequestModifyInPlace 構造体を含む Node を受け取ります。
  • **出力**: インプレース更新が可能な場合は対象の Param ノードを返し、それ以外の場合は NULL を返します。

前提条件と不変条件

  • 入力された rawreq が SupportRequestModifyInPlace 型である必要があります。
  • インプレース更新の最適化は、関数の配列引数が代入先の配列と一致する場合にのみ適用されます。

アルゴリズムまたは振る舞い

1. rawreq が SupportRequestModifyInPlace 型であるかを確認します。 2. 型が一致する場合、リクエスト内の引数リストから2番目の引数（配列引数）を取得します。 3. その引数が PARAM_EXTERN 型の Param であり、かつ代入先の paramid と一致するかを検証します。 4. 条件を満たす場合、その Param ノードを返却し、最適化を許可します。条件を満たさない場合は NULL を返します。

呼び出し元と呼び出し先

  • **呼び出し元**: PostgreSQL のプランナ（詳細な呼び出し元情報はソースコードから直接特定できません）。
  • **呼び出し先**: IsA マクロ、lsecond マクロなど、ノード操作のための内部ユーティリティ。

関連データ構造

  • SupportRequestModifyInPlace: プランナがインプレース更新を要求する際に使用するノード構造体。
  • Param: クエリ内のパラメータを表現するノード。

失敗モードと境界条件

  • 引数が SupportRequestModifyInPlace ではない場合、最適化は行われず NULL が返されます。
  • 引数が Param ではない、あるいは paramid が一致しない場合、最適化は適用されません。

保守担当者向けメモ

  • 本関数は array_prepend の最適化パスの一部です。インプレース更新のロジックを変更する場合は、SupportRequestModifyInPlace の仕様と整合性を取る必要があります。
  • 呼び出し元に関する情報は、プランナの実行計画生成プロセスに依存するため、本ソースコード単体では限定的な情報しか得られません。
